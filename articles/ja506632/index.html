Python、GDALで作成された火星の美しく詳細な地質図



今週は、USCSが作成した地図に基づいて、火星の地質学的（または地質学的）地図を創造的に再考します。元のデータと同じ地質学的データが使用されましたが、より多くのトポニムと署名が追加され、視覚的なスタイルが再設計され、一般的な理解のためにマップの凡例の用語が簡略化されました。

プロジェクトの最も難しい部分の1つは、シンボルを専門家ではない言語に翻訳することでした。元の地図は地質学者向けに作成されたため、ほとんどの語彙を確認する必要がありました。たとえば、カルデラの端の値の新しい解釈は、「火山噴火後に残っている空のマグマ溜りの境界」です。オリジナルには、「卵形の結節があり、1つまたは複数の部分的または完全に閉じたくぼみが融合した輪郭を描きました。噴火と爆発の可能性のある噴火に関連する火山の崩壊。

多くの場合、翻訳された指定は元の指定よりも不正確または情報提供が少なかったため、地質要素の各タイプの元の略語も含まれていました。科学的な用語で地質構造の各タイプについて突然知りたい場合は、これらのタグにソースデータへのリンクが含まれている場合があります。



火星の注目すべき地質学的特徴のいくつか。 1：太陽系で最大の火山、オリンパス山。 2：マリナーバレー、4000 kmを超える深い峡谷のシステム。 3：太陽系で最大の目に見える衝撃クレーターであるヘラス平原。 4：火星は、地質学的に北部低地（薄緑）と南部高地（茶色）に分かれています。小惑星と彗星（ネオンイエロー）の落下によって形成された衝突クレーターは、惑星全体に散在しています。



次の個別のレイヤーがマップに使用されました。 1-3：USGSデータセットの地質学的要素、地質学的接触、地質学的特徴。 4-5：傾斜レベルと2つのUSGSソースから。 6：IAU（International Astronomical Union）からの命名法。 7-8：Photoshopで開発されたグリッド線と3D効果。

プロジェクトの予測を選択するのに長い時間がかかりました。惑星のどの部分が各地層で構成されているかを正確に示すために、最終的にEckert IVの投影法が使用されました。。このタイプのマップ投影は、オブジェクトの輪郭を歪めますが、惑星の表面全体にわたって図形の相対面積を維持します。このような投影は極域の視覚化には適していないため、火星の各半球（北、南、東、西）を表示するために、コーナーに4つのマップインサートも追加しました。

これらの異なる地図の投影法を比較するには、ティソインディカトリックスを使用できます。これは、地球上のさまざまな場所に描かれた同じサイズの円のセットです。すべての地図投影は空間をゆがめますが、投影によって効果が大きく異なることがわかります。



1：等距離。2：Eckert IV。3-5：異なる経度と緯度を中心とする正投影

(DEM)


地形モデルは、地球上のさまざまなポイントの高さに関する情報を含むデータです。このプロジェクトでは、米国地質調査所から入手したデータを使用しました。GeoTIFFファイルの各ピクセルは、ポイントの高さを表す16ビットの数値です。

注：多くのプログラムはこのファイルを正常に読み取ることができないため、プレビュー形式や他の特殊でないプログラムでは正常に表示されません。プロジェクトのフレームワーク内で、彼らとの作業はGDAL（英語の地理空間データ抽象化ライブラリ-地理空間データの抽象化のライブラリ）を通過しました。

マッププロジェクションの作成


上述のように、エリアを変更しないメインマップにはEckert IV投影が使用され、埋め込み極マップには正投影が使用されました。

DEMファイルのプロジェクションを変更します。ソースファイルは等しいプロジェクションを使用したため、コマンドラインに次のコードを入力して翻訳を行う必要があります。コードは元のintifファイルを使用して、新しいouttifをeck4（Eckert IV）投影形式で作成します。

gdalwarp -t_srs "+proj=eck4" ./path_to_intif.tif ./path_to_outtif.tif

次に、平均的な方法を使用して、各ピクセルのサイズを1500x1500メートルに減らすだけで、DEM解像度を下げます。これにより処理時間が短縮され、この時点での解像度の低下は、後で行うよりも簡単です。

gdalwarp -tr 1500 1500 -r average ./path_to_intif.tif ./path_to_outtif.tif

洗浄および傾斜マップ


次に、結果のファイルを使用して、惑星の各半球の洗浄マップと勾配マップを含むマップを作成します。

洗浄後、マップには架空の光源からの影が表示され、光は上からマップに当たります。現実の世界ではこれは起こりません。単一の光源がボールの異なる部分に異なる角度で影を作成するためです。GDALの組み込みの陰影起伏機能は、マップ全体で光の入射角が同じになるように設定します。このプロジェクトでは、垂直方向の拡大率であるzを20に設定しました。これにより、各高さの値が20倍になり、レリーフのコントラストが増し、影ができました。

gdaldem hillshade -z 20 ./path_to_intif.tif ./path_to_hillshade.tif

スロープマップは、マップの最も急な部分を強調表示し、地形のウォッシュに情報を追加します（急勾配ではなく絶対的な高さを強調します）。

gdaldem slope ./path_to_intif.tif ./path_to_slope.tif



正投影


マップに加えて、火星の半球の4つのマップもEckert IVの投影で作成されます。Eckert IVの投影法は、北極と南極の表示にはあまり適していないため、これらの2つのインセットは、それらを理解するのに非常に役立ちます。ウォッシュアンドスロープマップを取得するには、eck4の代わりにorthoを少し変更してコードを繰り返し、各マップの経度と緯度の中心を指定します（北：+ lat_0 = 90 + lon_0 = 90、南：+ lat_0 = -90 + lon_0 = 90、東： + lat_0 = 0 + lon_0 = 90、西：+ lat_0 = 0 + lon_0 = 270）。

gdalwarp -t_srs "+proj=ortho +lat_0=90 +lon_0=0" ./path_to_intif.tif ./path_to_outtif.tif
gdalwarp -tr 1500 1500 -r average ./path_to_intif.tif ./path_to_outtif.tif
gdaldem hillshade -z 20 ./path_to_intif.tif ./path_to_hillshade.tif
gdaldem slope ./path_to_intif.tif ./path_to_slope.tif



凡例カード


国際天文学連合は地球外の天体の命名に責任があります。各惑星のすべてのオブジェクトを含むCSV形式のファイルをサイトから直接ダウンロードするだけです。これを行うには、高度な検索機能を使用して、惑星のすべての機能タイプ-ターゲット（火星）をダウンロードしますが、承認ステータスはIAUによる採用-MACによって承認されています。[含める列]セクションで、フィーチャーID、フィーチャー名、クリーンフィーチャー名、ターゲット、直径、中心緯度/経度、フィーチャータイプ、およびフィーチャータイプコードを選択します。たとえば、名前が付けられた人を称えるなど、オブジェクトに関する追加情報をプロジェクトに追加する場合は、Origin（origin）を有効にすることもできます。

地質構造と要素


地質図には、さまざまな種類の岩や、断層線や河床などのその他の特徴が表示されます。USGSには、火星の美しい地質図があり、これをソースとして使用しています。このデータを使用するには、USGS地質図からデータベースアーカイブ（790 MB）をダウンロードしてください。Pythonのマップ設計に関する次のセクションでは、このデータベースの各データタイプにアクセスして表示する方法について説明します。

Pythonマップデザイン


地質学的要素、構造、フィーチャ、2種類のテキストラベル、およびグリッドを使用して6つの図面（プロット）を作成します。必要に応じて、PhotoshopまたはIllustratorから効果を簡単に適用するために、処理のためにデータを分離することはしばしば価値があります。 Matplotlibはgridspecを使用してすべての要素をカスタマイズし、各サブプロットが装飾フレーム内の特定の場所を占めるようにしました。



地質学的要素-これらは、惑星の表面を構成するさまざまな種類のレリーフです。異なる岩石要素は通常、異なる色で表されます。 USGSデータセットは、アイテムのタイプを2〜3文字のコードで各品種にマークします。 Mars_geologic_units.csvのコードと色のペアのテーブルをコンパイルして、各文字コードに色を割り当てます。 cartopyとmatplotlibで各要素を作成するときに、このファイルを参照します。グラフィックパラメーターを別のファイルに保存すると、さまざまなカラースキームを使用しやすくなり、デザインをコードから分離できます。



地質学的接触-これらは地層間の境界です。一部の地質学的境界は、おおよそ、またはダストの下に隠れています。地層については、Mars_geologic_boundaries.csv構成ファイルを使用して、各タイプの地質接触を異なる色で表示します。最終的なマップでは、IllustratorでPDFを開き、同じ色のすべてのオブジェクトを選択する（選択->同一->ストロークの色）ことで、地質学的接触の表示を破線の形式で構成します。



地質-これらは、運河、クレーターの端、または山の尾根など、惑星の表面上の他の目に見える線です。データセットには13の異なるタイプがあります。それぞれを特別なスタイルで表示するために、それぞれを異なる色で表示し、Illustratorで線にスタイルを設定しました。これにより、グラジエントテールなどの特殊効果を適用して、流出チャネルと溶岩流の移動方向を示すことができます。



伝説


タグには2つのデータセットがマップに使用されました。最初はIAUからの公式です。ラベルのサイズはオブジェクトのサイズに従って作成されますが、Illustaratorで寸法を確定する必要があります。



2番目のラベルセットは、各主要な地質要素の略語を追加します。これにより、影によって一部の色が変化するため、マップの凡例が使いやすくなります。創造的にコードにアプローチする必要がありました。つまり、地質学的形状の幾何学的中心に各略語を描く必要がありました。これは、すべての場合に理想的ではありません。しかし、このプロジェクトの場合、これは正常でした。計画に従って、すべて同じように、これらの署名は、将来の署名の重複を避けるために手動で移動します。



着陸場所：火星探査車や他の宇宙船の着陸地点を表示するために、NASAからの拡張デジタル画像が背景として使用されました。惑星の4つの半球をすべて表示するには、投影を変更する必要がありました-landing_sites.ipynb。

Matplotlibからの結果の保存


通常、結果はPDF形式で保存され、Illustratorでテキストやフォームを最終編集します。次に、後続の編集を容易にするいくつかの標準コマンドを示します。

import matplotlib
import matplotlib.pyplot as plt
import matplotlib.backends.backend_pdf as pdf

#     ,     :
matplotlib.rcParams['pdf.fonttype'] = 42

#     
matplotlib.rcParams['image.composite_image'] = False

#      subplot
ax.axis('off')

#  padding  margins 
plt.margins(0,0)
plt.subplots_adjust(top=1, bottom=0, right=1, left=0, hspace=0, wspace=0)
plt.gca().xaxis.set_major_locator(plt.NullLocator())
plt.gca().yaxis.set_major_locator(plt.NullLocator())

#  pdf
pp = pdf.PdfPages('./savename.pdf', keep_empty=False)
pp.savefig(fig)
pp.close()

#      
#     PNG     Photoshop:
plt.savefig('./savename.png', format='png', dpi=600, pad_inches=0, transparent=True)


PDFを保存した後、各オブジェクトを個別に編集できるように編集する必要があります。Illustratorで、ファイル内のすべてを選択し、[オブジェクト]-> [クリッピングマスク]-> [リリース]に移動します。この時点で、以前に行った場合は、背景と境界線を削除することも価値があります。

配色開発


プロジェクトの開始時に、14の異なる配色が作成されました。元々のアイデアは、惑星ごとに独自のカラーパレットを持つことでしたが、最終的にはすべてのプロジェクトで同じ色のコレクションを使用しました。



装飾的なイラスト


このプロジェクトでは、アーティストのウィリアムモリスまたはアルフォンスミュシャのスタイルで、大きなデータセットと手の要素を組み合わせたいと考えていました。このために、インスピレーションのための絵画の大規模なコレクションが開発前に集められました。



最初は、地図のフレームワークのさまざまなバリエーションを試したかったのです。18の異なるパターンのコレクションが作成されました。

ただし、実装プロセス中に、データが多すぎることが明らかになり、いずれにしてもすべてが非常に混雑しているように見えました。その結果、主投影を構成するパターンが1つだけ選択されました。
