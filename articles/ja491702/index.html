純粋なCSS：リンクの色をアニメーション化する4つの方法

記事の作成者は、その翻訳を本日公開しており、純粋なCSSを使用して、リンクテキストにカーソルを合わせたときに色を変更するメカニズムを作成することを提案しています。しかし、これは通常の色の変化であってはなりません。新しい色は、リンクを左から右に塗りつぶし、古い色を置き換える必要があります。


これを行うには、この資料で説明されている4つの方法のいずれかに頼ることができます。コンテンツのアクセシビリティ、ソリューションのパフォーマンス、ブラウザのサポートなど、さまざまな重要事項に特に注意を払いながら、これらの方法を検討してください。

方法＃1：background-clip：textを使用する


この記事の執筆時点では、background-clip：textプロパティは実験的なものです。Internet Explorer 11以前ではサポートされていません。この方法では、いわゆる「ノックアウトテキスト」（特定の面で切り取られ、背景が半透明に見えるテキスト）をシャープなグラデーションで作成します。HTMLマークアップは<a>、ハイパーリンクを記述する単一の要素で構成されています。

<a href="#">Link Hover</a>

リンクのスタイルを作成することから始めましょう。を使用overflow: hiddenすると、リンクの外観を変更すると、この要素の範囲を超えるすべてが切り取られます。

a {
  position: relative;
  display: inline-block;
  font-size: 2em;
  font-weight: 800;
  color: royalblue;
  overflow: hidden;
}

リンクの開始色と終了色に50％の値を持つ鋭い線形グラデーションを使用する必要があります。

a {
  /*  ,    */
  background: linear-gradient(to right, midnightblue, midnightblue 50%, royalblue 50%);
}

プロパティbackground-clipを使用してグラデーションをトリミングします。textテキストを表示するための値を与えましょう。background-sizeおよびのプロパティを使用しますbackground-position。これは、初期色を表示するために行われます。

a {
  /*  ,    */
  background-clip: text;
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
  background-size: 200% 100%;
  background-position: 100%;
}

最後に、CSSプロパティをスタイルに追加し、transitionハイパーリンクの疑似クラスをスタイルします:hover。リンクにカーソルを合わせると、新しい色が左から右にリンクを塗りつぶすには、プロパティを使用しますbackground-position。

a {
  /*  ,    */
  transition: background-position 275ms ease;
}
a:hover {
  background-position: 0 100%;
}

ここで CodePen上の例である。この技術は、あなたが所望の効果を達成することができますが、SafariとChromeはトリミングされたテキストと影の要素を。つまり、それらは表示されません。CSSプロパティを使用してテキストに下線を引くなど、テキストスタイルを適用してtext-decorationも機能しません。したがって、リンクに下線を付ける場合は、下線をカスタマイズする他の方法を検討する必要があります。


方法2：幅/高さを適用する


このメソッドは、タグと同じテキストを含むdata- *属性の使用に基づいています<a>。ここでは、プロパティ管理width（リンクを左から右または右から左に色で塗りつぶす）またはプロパティheight（上から下または下から上に効果を適用する）を使用します。たとえば、この例では、プロパティに効果が適用されwidth、リンクの上にマウスを置くと、0から100％に変化します。マークアップは次のとおりです。

<a href="#" data-content="Link Hover">Link Hover</a>

CSSは、背景のプロパティ設定を除いて、前の例で使用したものと似ています。ここで、さらに、プロパティは正常に動作しますtext-decoration：

a {
  position: relative;
  display: inline-block;
  font-size: 2em;
  color: royalblue;
  font-weight: 800;
  text-decoration: underline;
  overflow: hidden;
}

ここで、属性のテキストを使用する必要がありますdata-content。このテキストは、タグのコンテンツの上に配置されます<a>。ここで興味深い小さなトリックを使用できます。これは、属性からテキストをコピーし、リンクの疑似要素のattr()プロパティの関数を使用してテキストを派生させることです。content::before

a::before {
  position: absolute;
  content: attr(data-content); /*    */
  top: 0;
  left: 0;
  color: midnightblue;
  text-decoration: underline;
  overflow: hidden;
  transition: width 275ms ease;
}

テキストが新しい行にジャンプしないようにするために、スタイルが疑似要素に適用されますwhite-space: nowrap。リンクの色を変更するには、color疑似要素のCSSプロパティの値を設定::beforeし、最初にプロパティの値がwidth0になるようにします。

a::before {
  /*  ,    */
  width: 0;
  white-space: nowrap;
}

width疑似要素 の値::beforeを100％に増やして、マウスリンクにカーソルを合わせたときに効果を適用します。

a:hover::before {
  width: 100%;
}

ここでこの方法を適用した例。我々が、要素のプロパティに作用widthし、height私たちが必要なものを達成するため、この方法は、その低いために注目すべきである生産。1秒あたり60フレームで滑らかな色の変化を得るには、プロパティを使用するtransformか、プロパティを使用することをお勧めします。opacityプロパティをtext-decoration使用すると、アニメーションリンクでテキストを強調するさまざまなスタイルを使用できます。ここで、我々は今、検討する第3の技術を、使用して作成し、これを証明する例が。これは、clip-pathの CSSプロパティの使用に基づいています。



方法3：クリップパスを使用する


ここでは、CSSプロパティclip-pathとポリゴン（この場合は長方形）を使用します。長方形には4つのコーナーがあり、リンクの上にカーソルを置くと、2つの辺の長さが増加します。図が拡大し、その側面の1つが左から右に移動します。


前の例と同じマークアップを使用します。

<a href="#" data-content="Link Hover">Link Hover</a>

ここでも、疑似要素を使用します::before。しかし、CSSはここでは異なります。

a::before {
  position: absolute;
  content: attr(data-content);
  color: midnightblue;
  text-decoration: underline;
  clip-path: polygon(0 0, 0 0, 0 100%, 0 100%);
  transition: clip-path 275ms ease;
}

前の方法とは異なり、ここではプロパティをtext-decoration: underline疑似要素に設定する必要があります::before。これは、色の変更がリンクのテキストだけでなく、リンクに下線を引く線にも影響を与えるために必要です。では、プロパティのCSSコードを見てみましょうclip-path：

clip-path: polygon(0 0, 0 0, 0 100%, 0 100%);

プロパティ内のポリゴンの頂点の位置はclip-pathパーセンテージで指定され、ポリゴン上のそれらの配置に対応する順序で座標を決定します。

  • 0 0 =左上隅
  • 0 0 =右上隅
  • 0 100% =右下隅
  • 0 100% =左下隅

座標を変更することで、塗りつぶし効果の適用方向を変更できます。座標がわかったので、マウスリンクにカーソルを合わせると、左から右に移動して図を拡大できます。

a:hover::before {
  clip-path: polygon(0 0, 100% 0, 100% 100%, 0 100%);
}

ここにあります。この例の作業バージョンは、リンクの色をアニメーション化するこの技術は非常にうまく動作しますが、それを使用する前に、考慮の支援でプロパティclip-path別のブラウザを。を使用したCSSトランジションの作成clip-pathは、高さ/幅のテクニックを使用するよりも優れています。ただし、その使用は、ブラウザが非常にリソースを大量に消費するペイント操作（描画）を実行するという事実につながります。

方法＃4：変換を使用する


ここで使用されるマークアップは、要素のマスキング手法を使用しています<span>。要素<span>のリンクのコンテンツと重複するコンテンツを使用するため、コンテンツの可用性を向上させるために属性を使用しますaria-hidden="true"。これにより、繰り返されるテキストがスクリーンリーダーに表示されなくなります。

このようなテキストは2度発声されません。

<a href="#"><span data-content="Link Hover" aria-hidden="true"></span>Link Hover</a>

要素のCSSには<span>、左側から始まる遷移の説明が含まれています。

span {
  position: absolute;
  top: 0;
  left: 0;
  overflow: hidden;
  transform: translateX(-100%);
  transition: transform 275ms ease;
}

<span>次の図のよう に、要素の動きを右に整理する必要があります。


これを行うには、::before要素pseudo- elementを使用します<span>。そして、以前と同様に、属性を使用することにしますdata-content。要素の位置を変更して、変換を適用しtransform: translateX(100%)、軸に沿って移動しますX。

span::before { 
  display: inline-block;
  content: attr(data-content);
  color: midnightblue;
  transform: translateX(100%);
  transition: transform 275ms ease;
  text-decoration: underline;
}

要素<span>と同様に、疑似要素の位置は次::beforeの構文を使用して設定されtransform: translateX(0)ます。

a:hover span::before {
  transform: translateX(0);
}

以下はCodePenの例です。このメソッドは、上記のブラウザ間でのサポートが最も優れていますが、実装するには、より多くのHTMLとCSSが必要です。それにもかかわらず、CSSプロパティの使用はtransform パフォーマンスに悪影響を与えません。その使用は要素の再描画を引き起こさず、その結果、毎秒60フレームでのスムーズなCSS遷移の形成につながります。


概要


CSSを使用して同じ効果を達成するために、4つの異なる方法を検討しました。それぞれに長所と短所がありますが、それらの上にカーソルを置くと、リンクのテキストの色のアニメーションを整理するのに不可能はないことがわかります。これは小さな楽しい効果であり、これを使用すると、リンクがよりインタラクティブであると認識されます。
