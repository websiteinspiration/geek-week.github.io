変更したシャドウフリップアプローチを使用して、最も単純な凹面効果から始めます。この構造を追加します。

struct DarkBackground<S: Shape>: View {

                    .fill(Color.darkEnd)

                    .shadow(color: Color.darkEnd, radius: 10, x: 10, y: 10)
            }
        }
    }
}

いくつかの実験


SwiftUIが具体的に何ができるかをよりよく理解するのに役立つので、今が効果を試す時です。

たとえば、線形グラデーションを追加して滑らかな凸型ボタンを作成し、押すと反転させることができます。

if isHighlighted {
    shape
        .fill(LinearGradient(Color.darkEnd, Color.darkStart))
        .shadow(color: Color.darkStart, radius: 10, x: 5, y: 5)
        .shadow(color: Color.darkEnd, radius: 10, x: -5, y: -5)

} else {
    shape
        .fill(LinearGradient(Color.darkStart, Color.darkEnd))
        .shadow(color: Color.darkStart, radius: 10, x: -10, y: -10)
        .shadow(color: Color.darkEnd, radius: 10, x: 10, y: 10)
}

これを実行すると、ボタンを押して離したときに、ボタンがスムーズに上下にアニメーションすることがわかります。アニメーションが少し煩わしいので、モディファイアが存在する後、このモディファイヤをメソッドmakeBody()from DarkButtonStyleに追加して無効にすることをお勧めしますbackground()。

.animation(nil)



クッションボタンのこの効果は魅力的ですが、それを使用する予定がある場合は、ボタンをもう少し目立たせるために次の3つの変更を試すことをお勧めします。

まず、これがネオモルフィックデザインの低コントラストの原則に反するという事実にもかかわらず、私はそれを目立たせるために灰色のアイコンを白いものに置き換えます。したがって、ContentView次のものが必要です。

Image(systemName: "heart.fill")
    .foregroundColor(.white)

第2に、押された状態のボタンにオーバーレイを追加すると、均等に押された実際の物理的なボタンのように見えるだけでなく、押された状態と押されていない状態を区別するのにも役立ちます。

これを実装するには、次のように、trueのoverlay()後fill()に修飾子を挿入する必要がありisHighlightedます。

if isHighlighted {
    shape
        .fill(LinearGradient(Color.darkEnd, Color.darkStart))
        .overlay(shape.stroke(LinearGradient(Color.darkStart, Color.darkEnd), lineWidth: 4))
        .shadow(color: Color.darkStart, radius: 10, x: 5, y: 5)
        .shadow(color: Color.darkEnd, radius: 10, x: -5, y: -5)



さらにシャープな外観を実現するにshadow()は、押された状態の2つの修飾子を削除して、それ以外の場合はオーバーレイに焦点を合わせることができます。

3番目に、ボタンが押されていない状態にオーバーレイを追加して、ボタンであることを示すこともできます。次のfill()ようにの直後に置きます：

} else {
    shape
        .fill(LinearGradient(Color.darkStart, Color.darkEnd))
        .overlay(shape.stroke(Color.darkEnd, lineWidth: 4))
        .shadow(color: Color.darkStart, radius: 10, x: -10, y: -10)
        .shadow(color: Color.darkEnd, radius: 10, x: 10, y: 10)
}



スイッチスタイルの追加


ボタンスタイルを非モーフィック背景スタイルから分離する利点の1つは、同じ効果を使用してスイッチスタイルを追加できることです。これは、次の点を除いて、にToggleStyle類似した新しいプロトコル準拠の構造を作成することを意味しますButtonStyle。

  1. configuration.isOnスイッチがオンになっているかどうかを判断するために読む必要があります。
  2. 切り替えを処理するためのボタン、または少なくともそのようなものを提供する必要がありますonTapGesture()。

この構造をプロジェクトに追加します。

struct DarkToggleStyle: ToggleStyle {
    func makeBody(configuration: Self.Configuration) -> some View {
        Button(action: {
            configuration.isOn.toggle()
        }) {
            configuration.label
                .padding(30)
                .contentShape(Circle())
        }
        .background(
            DarkBackground(isHighlighted: configuration.isOn, shape: Circle())
        )
    }
}

ContentView自分でテストできるように 、そのうちの1つを配置したいので、まずこのプロパティを追加します。

@State private var isToggled = false

次に、既存のボタンVStackを40の間隔でラップして、下に配置します。

Toggle(isOn: $isToggled) {
    Image(systemName: "heart.fill")
        .foregroundColor(.white)
}
.toggleStyle(DarkToggleStyle())

構造ContentViewは次のようになります。

struct ContentView: View {
    @State private var isToggled = false

    var body: some View {
        ZStack {
            LinearGradient(Color.darkStart, Color.darkEnd)

            VStack(spacing: 40) {
                Button(action: {
                    print("Button tapped")
                }) {
                    Image(systemName: "heart.fill")
                        .foregroundColor(.white)
                }
                .buttonStyle(DarkButtonStyle())

                Toggle(isOn: $isToggled) {
                    Image(systemName: "heart.fill")
                        .foregroundColor(.white)
                }
                .toggleStyle(DarkToggleStyle())
            }
        }
        .edgesIgnoringSafeArea(.all)
    }
}

以上で、2つの場所に共通のネオモーフィックデザインを適用しました。

アクセシビリティの向上


さまざまなネオモーフィックスタイルで遊ぶ時間は十分にありましたが、ここでこのデザインの問題について詳しく説明します。コントラストが極端に不足しているため、ボタンやその他の重要なコントロールが周囲から目立つには不十分であり、不完全なビジョン。

これは私が誤解を認める瞬間なので、事前にいくつかのことを言いたいと思います。

グレーのアイコンを白に変更してコントラストを瞬時に強調する方法はすでに説明しましたが、ボタンとスイッチをさらに使いやすくするには、ボタンとスイッチのコントラストをさらに大きくする必要があります。

したがって、要素を本当に目立たせるために、どのような変更を加えることができるかを検討します。

まず、拡張機能に2つの新しい色を追加してください。

static let lightStart = Color(red: 60 / 255, green: 160 / 255, blue: 240 / 255)
static let lightEnd = Color(red: 30 / 255, green: 80 / 255, blue: 120 / 255)

次に、既存のものDarkBackgroundを複製し、コピーに名前を付けますColorfulBackground。すぐに対処しますが、最初に、いくつかの準備をする必要があります。

3番目に、ボタンとスイッチの暗いスタイルを複製し、名前をColorfulButtonStyleおよびColorfulToggleStyleに変更して、新しいものColorfulBackgroundを背景として使用するようにします。

したがって、次のようになります。

struct ColorfulButtonStyle: ButtonStyle {
    func makeBody(configuration: Self.Configuration) -> some View {
        configuration.label
            .padding(30)
            .contentShape(Circle())
            .background(
                ColorfulBackground(isHighlighted: configuration.isPressed, shape: Circle())
            )
            .animation(nil)
    }
}

struct ColorfulToggleStyle: ToggleStyle {
    func makeBody(configuration: Self.Configuration) -> some View {
        Button(action: {
            configuration.isOn.toggle()
        }) {
            configuration.label
                .padding(30)
                .contentShape(Circle())
        }
        .background(
            ColorfulBackground(isHighlighted: configuration.isOn, shape: Circle())
        )
    }
}

最後に、ボタンを編集してContentView、新しいスタイルを使用するように切り替えます。

Button(action: {
    print("Button tapped")
}) {
    Image(systemName: "heart.fill")
        .foregroundColor(.white)
}
.buttonStyle(ColorfulButtonStyle())

Toggle(isOn: $isToggled) {
    Image(systemName: "heart.fill")
        .foregroundColor(.white)
}
.toggleStyle(ColorfulToggleStyle())

必要に応じてアプリケーションを実行できますが、それは意味がありません-実際には変更されていません。

私たちの生活のカラフルなバージョンを持って、我々は修飾子を変更しようとしている fill()とoverlay()押され、押されていない状態のため。したがって、isHighlightedtrueの場合は、次のようにとのdarkStart両方darkEndに変更します。lightStartlightEnd

if isHighlighted {
    shape
        .fill(LinearGradient(Color.lightEnd, Color.lightStart))
        .overlay(shape.stroke(LinearGradient(Color.lightStart, Color.lightEnd), lineWidth: 4))

アプリケーションを再度実行すると、すでに大幅に改善されていることがわかります。押された状態は明るい青色になり、ボタンが押されてスイッチがアクティブになるとはっきりします。しかし、別のことができます。ボタンが押されていないときにボタンの周りに同じ色を追加して、注意を引くことができます。



これを行うには、既存のoverlay()非ストレス状態を次のように変更します。

.overlay(shape.stroke(LinearGradient(Color.lightStart, Color.lightEnd), lineWidth: 4))


したがって、完成したボタンのスタイルは次のようになります。

ZStack {
    if isHighlighted {
        shape
            .fill(LinearGradient(Color.lightEnd, Color.lightStart))
            .overlay(shape.stroke(LinearGradient(Color.lightStart, Color.lightEnd), lineWidth: 4))
            .shadow(color: Color.darkStart, radius: 10, x: 5, y: 5)
            .shadow(color: Color.darkEnd, radius: 10, x: -5, y: -5)
    } else {
        shape
            .fill(LinearGradient(Color.darkStart, Color.darkEnd))
            .overlay(shape.stroke(LinearGradient(Color.lightStart, Color.lightEnd), lineWidth: 4))
            .shadow(color: Color.darkStart, radius: 10, x: -10, y: -10)
            .shadow(color: Color.darkEnd, radius: 10, x: 10, y: 10)
    }
}

ここでアプリケーションを再度実行すると、ボタンとスイッチの周りに青いリングが表示され、クリックすると青色で塗りつぶされます。
