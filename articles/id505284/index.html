wx. Dapat diakses


Bahkan, dia muncul sebelumnya, kembali ke versi 4.0.4. Namun, dalam versi ini, pengembang wxpython memecah aksesibilitas dasar, yang merupakan kemunduran besar kembali ke aksesibilitas.

Masalah ini dapat dilihat, misalnya, di sini: wx.Accessible: label generik dilihat oleh pembaca layar jika kontrol memilih untuk tidak mengimplementasikan objek IAccessible atau nama kata kunci untuk wx.Kontrol konstruktor tidak diberikan .

Tetapi dalam pembaruan besar terakhir 4.1.0, mereka memperbaiki masalah ini, dan sekarang Anda dapat sepenuhnya melihat alat baru.

Dengan bantuannya, Anda dapat membuat komponen awal yang tidak dapat diakses tersedia, dan memperluas kemampuan komponen yang ada dan cukup mudah diakses. Jadi saya menemukan ketika saya mencari kemungkinan untuk menghasilkan informasi teks tambahan untuk pembaca layar tanpa mengubah representasi grafis.

Karena topik aksesibilitas bukan yang paling relevan, dan komponen ini relatif baru, saya tidak menemukan contoh yang dapat dipahami untuk bekerja dengannya bahkan di segmen Internet berbahasa Inggris, dan lebih lagi di segmen berbahasa Rusia.

Saya akan mencoba memberikan kontribusi kecil untuk bidang ini.

Pertimbangkan contoh bekerja dengannya untuk kejelasan.

Saya tidak akan memberi Anda kode standar ekstra untuk membuat jendela wx.Frame. Contoh-contoh ini penuh untuk setiap selera. Oleh karena itu, saya akan mengutip secara singkat hanya inovasi untuk bekerja dengan wx.Accessible.
Sebagai contoh kita, ambil widget adv.CalendarCtrl.

Ketika Anda menangkap fokus dalam widget ini, Anda dapat menavigasi tanggal dengan panah pada keyboard, tetapi pada saat yang sama itu tidak disuarakan sama sekali, dan pengguna tidak dapat memahami pada tanggal berapa dia saat ini.

Selain itu, widget memiliki masalah lain, misalnya, ketidakmampuan untuk menggunakan keyboard untuk mengubah fokus darinya dan beralih ke komponen antarmuka berikutnya (ps: jika ada yang tahu opsi untuk mengembalikan fokus dari widget tersebut ke jendela, tulis di komentar), tapi sekarang tugasnya bukan untuk membuat widget kalender sepenuhnya dapat diakses, tetapi untuk menunjukkan kerja dengan wx.Accessible.

Jadi dalam program kami, kami memiliki inisialisasi widget ini:

self.date = wx.adv.CalendarCtrl(self.panel, wx.ID_ANY,
    wx.DateTime.Today(),
    style=wx.adv.CAL_MONDAY_FIRST |
          wx.adv.CAL_SEQUENTIAL_MONTH_SELECTION)

Di bawah ini Anda perlu menambahkan yang berikut:

self.date.SetAccessible(AdvCalendarAccessible(self.date))

Kelas AdvCalendarAccessible adalah kelas kami yang sepenuhnya baru, diwarisi dari wx.Accessible, disajikan di bawah ini.

Ini menimpa metode yang diperlukan untuk pembaca layar.
Jadi, buat kelas baru kami:

class AdvCalendarAccessible(wx.Accessible):

    def __init__(self, window):
        self.window = window
        super().__init__(window)

    def GetName(self, index):
        date = self.window.GetDate()
        return (wx.ACC_OK, date.Format('%d.%m.%Y'))

    def GetDescription(self, index):
        text = ''
        date = self.window.GetDate()
        attr = self.window.GetAttr(date.day)
        if attr is not None:
            text = attr.GetBackgroundColour().GetAsString()
        return (wx.ACC_OK, text)

Dalam opsi ini, kami, atas permintaan dari pembaca layar, mengembalikan tanggal kepada mereka dan juga warna latar belakang dari tanggal yang dipilih, jika tentu saja ditandai oleh program.


Jadi itu saja. Sekarang pengguna dapat mendengar semua tanggal saat bernavigasi dengan panah keyboard pada widget ini.

Selain metode ini, ada juga yang lain. Misalnya, untuk menyuarakan nilai komponen atau string prompt-nya.

Dokumentasi lengkap tersedia di: wx.Accessible .

Saya harap seseorang akan menemukan informasi yang berguna, dan program Anda akan menjadi satu langkah lebih mudah diakses.
