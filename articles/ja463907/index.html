GraphQLを使用して8baseバックエンドに接続する


これで最初のフロントエンドが稼働しました。次のステップは、GraphQLを使用して8baseバックエンドと対話するようにクライアントを構成することです。接続に役立つ便利なライブラリはapollo-boostです。 URIを使用してGraphQLバックエンドに接続するためのクライアントを提供します。 URIは8baseが提供するエンドポイントです。index.jsファイルを更新して、ApolloClientを構成しましょう。

import React from "react";
import ReactDOM from "react-dom";
import { ApolloProvider } from "react-apollo";
import ApolloClient from "apollo-boost";
import * as serviceWorker from "./serviceWorker";
    
import "./index.css";
import App from "./App";
    
const client = new ApolloClient({
  uri: "<YOUR_8BASE_ENDPOINT>"
});
    
ReactDOM.render(
  <ApolloProvider client={client}>
    <App />
  </ApolloProvider>,
  document.getElementById("root")
);
serviceWorker.unregister();

ApolloClientは、uriプロパティを持つオブジェクトを引数として受け取ります。同時に、クライアントは、提供されたuriを使用してデータを受信、更新、および削除するためのアクセスを提供します。<YOUR_8BASE_ENDPOINT>の値を実際のエンドポイントに置き換えてください。

次に、propクライアントを受け入れるApolloProviderでアプリケーションをラップします。ApolloProviderは8baseテーブルスキーマをロードします。これにより、フロントエンドコードのすべてのデータモデルプロパティにアクセスできます。

ノートを削除する


すべてのメモを削除して、それらが存在しなかったふりをすることができない場合、どのようなメモ作成アプリも役に立ちます。ノートを削除する手順：

  1. ユーザーは削除ボタンをクリックします。
  2. 確認ウィンドウが表示されます-これにより、悪意のあるユーザーの数が減り、非常に便利です。
  3. 行為をコミットするために残っています-ノートを削除します。

ノートを削除するための設定は、src / note-card / delete-button.jsファイルにあります。ファイルを開き、次のインポートをファイルの上部に追加します。特に、TODO 1が配置されている場所の下：

import gql from 'graphql-tag';
import { graphql } from 'react-apollo';

次に、削除ミューテーションを記述し、それをDELETE_MUTATION変数に割り当てます。これは次のようになります。

const DELETE_MUTATION = gql`
      mutation DeleteNote($data: NoteDeleteInput!) {
        noteDelete(data: $data) {
          success
        }
      }
    `;

次に、graphql関数を使用して、変異をコンポーネントに関連付けます。ファイルのエクスポート行を以下のフラグメントに置き換えます。

export default graphql(DELETE_MUTATION, {
      name: 'deleteNote',
    })(DeleteButton);

次に、graphql関数は、名前付きのdeleteNote関数を渡します。これは、削除ミューテーションを実行するために使用されます。これで、クリックイベントハンドラーを更新し、メモを削除するために非常に必要な設定を追加できます。コンポーネントのonDeleteClick

関数に移動し、関数内に次のフラグメントを配置します。

const onDeleteClick = async () => {
      const data = {
        id: noteId,
      };
      const remove = window.confirm('Are you sure you want to delete this note?');
      if (remove) {
        await deleteNote({ variables: { data } });
        refetch();
      }
    };

削除ボタンをクリックすると、データオブジェクトの本文で、noteIdの値がidフィールドに割り当てられ、このメモを完全に削除するつもりであることを明確に示す確認メッセージがユーザーに表示されます。ユーザーが意図したとおりに続行する場合は、deleteNote関数を呼び出して削除を変更します。これが完了すると、アプリケーションが必要になります。 これがどのように見えるかです。localhost：3000に移動して、メモを削除してみてください。 このアプリケーションのデモ版はこちらで見つかります。
