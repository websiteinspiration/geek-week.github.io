Javaのインストール


もちろん、JavaFXにはJDKがインストールされている必要があります。必要な依存関係の取得は、Javaのバージョンによって異なります。

JavaFXは、Java 8のリリースの一部として導入されましたが、後でJDKから削除され、
Java 11の別のモジュールに移植されました。

つまり、Java 8-10をターゲットにしている場合、JDKの一部として必要な依存関係がすべてすでに存在します。やったー！新しいバージョンのJava（11以上）を使用している場合は、依存関係を個別に取得する必要があります。

メイベン


依存関係を手動で取得および管理することはかなり不便であり、実際のアプリケーションではめったにそれを行いません。MavenやGradleなどの依存関係管理システムを使用することをお勧めします。そうすることで、使用している依存関係とバージョンを宣言するだけで、あとはシステムが処理します。

アーキタイプメイベン


もちろん、Mavenプロジェクトを最初から手動で設定することもできます。ただし、Mavenを使用して、ベースプロジェクトの構造とコンテンツを作成するより便利な方法を好む場合があります。

Mavenにはアーキタイプの概念があります。つまり、テンプレートからさまざまなタイプのプロジェクトを作成できるということです。さまざまなタイプのプロジェクトには多数のアーキタイプがあり、幸い、JavaFXにはいくつかあります。選択できるアーキタイプは、使用しているJavaのバージョンによって異なります。

Mavenアーキタイプの詳細については、次の投稿をご覧ください。Mavenアーキタイプチュートリアル。

Java 8アーキタイプ


com.zenjava：javafx-basic-archetype を使用できます。または、これが自分に合わない場合は、自分で他のアーキタイプを見つけることができます。

上記のアーキタイプを使用してMavenを使用し、コマンドラインからプロジェクトを簡単に生成できます。

mvn archetype:generate -DarchetypeGroupId=com.zenjava -DarchetypeArtifactId=javafx-basic-archetype

または、IDEのアーキタイプから新しいMavenプロジェクトを作成することもできます。

Java 11アーキタイプ


Java 11の場合、org.openjfx：javafx-archetype-simpleを使用できます。

プロジェクトを作成するには、次を実行します。

mvn archetype:generate -DarchetypeGroupId=org.openjfx -DarchetypeArtifactId=javafx-archetype-simple

Mavenの手動チューニング


さらに制御したい場合は、アーキタイプから生成せずに、手動でMavenプロジェクトを構成できます。

まず、2つのコンポーネントが必要です。1つ目は、Java FX用のMavenプラグインです。

以下をpom.xmlに追加してください：

<build>
      <plugins>
            <plugin>
                  <groupId>org.openjfx</groupId>
                  <artifactId>javafx-maven-plugin</artifactId>
                  <version>0.0.3</version>
                  <configuration>
                         <mainClass>com.example.App</mainClass>
                  </configuration>
             </plugin>
       </plugins>
</build>

ご了承ください 
<mainclass>
メインメソッドを持ち、javafx.application.Applicationを拡張するメインクラスを指す必要があります。これについては、シリーズの次の記事で説明します。

2番目の部分では、JavaFXコントロールの依存関係を追加します。

<dependency>
        <groupId>org.openjfx</groupId>
        <artifactId>javafx-controls</artifactId>
        <version>11.0.2</version>
</dependency>

Gradle


Gradleは現在、アーキタイプから直接プロジェクトを作成することをサポートしていません。

非公式のGradle Archetypeプラグインを使用して、上記のMavenアーキタイプを使用できます。
または、アーキタイプからMavenを使用してプロジェクトを作成し、pom.xmlを含むディレクトリで次のコマンドを使用してGradleプロジェクトに変換することもできます。

gradle init


手動Gradle


手動のMavenセットアップと同様に、JavaFXプラグインを追加する必要があります。

plugins {
    id 'application'
    id 'org.openjfx.javafxplugin' version '0.0.8'
}

そして、コントロールの依存関係：

javafx {
    version = "11.0.2"
    modules = [ 'javafx.controls' ]
}

JavaFX SDK


ローカルでJavaFXを使用する別の可能性があります。JavaFX SDKをダウンロードできます。これには、必要なすべてのライブラリが含まれており、IDEでプロジェクトにリンクしたり、クラスパスに追加したりできます。

これは、GradleやMavenに精通しておらず、ローカルでの開発にのみ役立つ場合があります。

アプリケーションを配布する場合、必要な依存関係をすべて含める必要があるため、これは不便になります。

このオプションを使用すると、必要なすべてのファイルを含むIDEで、Maven / Gradleなしでプロジェクトを生成できます。IntelliJ IDEAでは、次の場所に移動できます。

File → New → Project → JavaFX
