PVS-Studioがクラウドに移行：GitLab CI / CD


この記事は、クラウドシステムでのPVS-Studioの使用に関する一連の出版物の続きです。今回は、GitLab Incの製品であるGitLab CIと組み合わせてアナライザーを見ていきます。CIシステムに静的アナライザーを統合すると、プロジェクトのビルドフェーズの直後にバグを特定でき、エラー検出のコストを削減するための非常に効果的な方法です。

クラウドCIシステムへの統合に関する他の記事のリスト：

  • PVS-Studio : Azure DevOps
  • PVS-Studio : Travis CI
  • PVS-Studio : CircleCI


GitLabは、オンラインリポジトリ管理サービスです。アカウントを登録することで、公式ウェブサイトのブラウザで直接使用したり、独自のサーバーにインストールしてデプロイしたりできます。

PVS-Studioは、C、C ++、C＃、およびJavaで記述されたプログラムのエラーおよび潜在的な脆弱性を検出するように設計された静的コード分析ツールです。 Windows、Linux、macOSの64ビットシステムで動作し、32ビット、64ビット、および組み込みARMプラットフォーム用に設計されたコードを分析できます。静的コード分析でプロジェクトをテストするのが初めての場合は、PVS-Studioの最も興味深い警告をすばやく確認し、このツールの機能を評価する方法に関する記事を読むことをお勧めします。

OBSプロジェクトを使用して、クラウドでの静的アナライザーの操作をデモンストレーションします。Open Broadcaster Softwareは、無料でオープンソースのビデオ録画およびストリーミングソフトウェアのスイートです。OBSは、デバイスやソースからリアルタイムで傍受する機能、シーンの構成、デコード、録音、放送を提供します。データ送信は主にReal Time Messaging Protocolを介して行われ、データはRTMPをサポートする任意のソースに転送できます。プログラムには、最も人気のあるストリーミングプラットフォームに直接ブロードキャストするための既成のプリセットがあります。

次に、静的コード分析の本質を示すために、Open Broadcaster Softwareプロジェクトのエラーを示すレポートの警告をいくつか見てみましょう。この記事の目的は、PVS-StudioとGitLab CI / CD間の相互作用の原理を説明することです。そのため、エラーのある興味深いコードフラグメントのみが書き出されています。プロジェクトの作成者に一時的なライセンスを提供する準備ができており、必要に応じて、プロジェクトのより徹底的な分析を行うことができます。または、 PVS-Studioの無料ライセンスのオプションの 1つを使用できます。

また、PVS-Studioの機能を探索し、プロジェクトを確認するためのトライアルキーを誰でも個別に受け取ることができます。

それでは、Open Broadcaster Softwareで見つかったエラーの例をいくつか見てみましょう。

次の行に注意してください：if（data_end_pos> cb-> capacity）の場合、条件が満たされると、下の行で定義されている変数back_sizeが常にゼロより大きいことがわかります。下の別の行は常にtrueになります。過剰な状態は、その下にあるデータを変更するコードである場合、それほど無害ではありません。

警告N2、N3

V629 '1 <<インデント'式を調べることを検討してください。32ビット値のビットシフトと、それに続く64ビットタイプへの拡張。profiler.c（610）

static void profile_print_entry(uint64_t active, unsigned indent, ....)
{
  ....
  active &= (1 << indent) - 1;
  ....
}

ここでは、32ビット型と64ビット型の操作の組み合わせが疑わしく見えます。32ビット型（式（1 <<インデント）-1）を使用して最初のマスクを計算すると、アクティブ＆= ...の観点から暗黙的に64ビット型に拡張されます。ほとんどの場合、マスクを計算するときに、64ビット型の使用も想定されていました。

コードの正しいバージョン：

active &= ((uint64_t)(1) << indent) - 1;

または：

active &= (1ull << indent) - 1;

また、このコードブロックのコピーと貼り付けを以下に示します。アナライザーは次の警告も発行しました：V629 '1 <<インデント'式の検査を検討してください。32ビット値のビットシフトと、それに続く64ビットタイプへの拡張。profiler.c（719）

結論


ご覧のとおり、PVS-Studio静的アナライザーをGitLabプロジェクトに統合するのは非常に簡単です。これを行うには、構成ファイルを1つ記述してクラウドリポジトリに配置するだけです。GitLabには独自の統合仮想マシンがあるため、CIシステムの構成に多くの時間を費やす必要もありません。コードをチェックすると、アセンブリの直後に問題を特定できるため、編集の複雑さとコストがまだ小さい場合に問題を取り除くのに役立ちます。
