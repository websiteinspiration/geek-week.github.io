游戏代码命令与征服：90年代的错误。第一卷


美国公司Electronic Arts Inc（EA）发布了《命令与征服：泰伯利亚黎明》和《命令与征服：红色警戒》游戏的开源代码。此代码应有助于游戏社区开发mod和地图，创建自定义单位以及自定义游戏逻辑。我们每个人都有独特的机会来深入了解发展历史，这与现代的发展历史截然不同。那时没有StackOverflow网站，便利的代码编辑器和强大的编译器。然后就没有静态分析器了，社区将首先遇到的是代码中的数百个错误。但是PVS-Studio团队会为此提供帮助，指出这些错误的位置。

介绍


命令与征服-一系列实时策略类型的计算机游戏。该系列的第一款游戏于1995年发布。电子艺界仅在1998年收购了该游戏的开发工作室。

从那时起，已经发布了一些游戏和许多mod。对于游戏的源代码被公布与释放命令与征服修复集合。

为了找到代码中的错误，使用了PVS-Studio分析仪。这是一个用于检测用C，C ++，C＃和Java编写的程序的源代码中的错误和潜在漏洞的工具。

由于代码中存在大量问题，因此所有错误示例将在两篇文章的周期中给出。

V610未定义的行为。检查移位运算符“ <<”。右操作数（'（32-bits_to_shift）'= [1..32]）大于或等于提升后的左操作数的位长度。MP.CPP 659

#define UNITSIZE 32

void XMP_Shift_Right_Bits(digit * number, int bits, int precision)
{
  ....
  int digits_to_shift = bits / UNITSIZE;
  int bits_to_shift = bits % UNITSIZE;

  int index;
  for (index = digits_to_shift; index < (precision-1); index++) {
    *number = (*(number + digits_to_shift) >> bits_to_shift) |
      (*(number + (digits_to_shift + 1)) << (UNITSIZE - bits_to_shift));
    number++;
  }
  ....
}

分析仪检测到可能会向右移位32位数字的情况。这是怎么回事：

int bits_to_shift = bits % UNITSIZE;

该单位大小恒定具有值32：

int bits_to_shift = bits % 32;

因此，对于32的倍数的所有位值，bits_to_shift变量的值将为零。 因此，在这段代码中：



.... << (UNITSIZE - bits_to_shift) ....

如果从常数32减去0，将发生32位移位。
