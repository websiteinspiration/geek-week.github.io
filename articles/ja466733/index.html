DINS Java EVENINGへようこそ9月18日

私たちは、19歳のStaro-Petergofskyのオフィスで開催される次のオープンなDINS Java EVENINGに、Java開発に関心のあるすべての人を招待します。

今夜、WrikeのDenis Reppが、製品の重要な部分の技術的負債を扱う作業を整理する方法を説明します。InfobipのAndrey Markelovが、Javaサービスの応答時間の分析を整理する方法の具体例を示します。

カットの下-レポートとスピーカーの詳細、会議に参加するために登録するリンク、前回の会議の資料。



報告書


「内燃機関の宇宙船。技術的な借金で戦いを生き延びた」（Denis Repp、Wrike）


-ワープエンジンがAI-95で実行されている場合はどうなりますか？
-キャビン内の唯一のヒーターがトースターである場合はどうなりますか？
-ハニーベイが1つのナットと3つの釘で船の船体に取り付けられている場合はどうなりますか？
-船長、鍵を16に渡すか、最後に技術的負債に対処しましょう！
レポートの過程で、Denisは、製品の重要な部分で技術的負債を処理するプロセスを整理する方法、プロセスを予測可能かつ透過的にする方法、対処しなければならないミスの種類とそれらを回避する方法を理解することを提案しています。

8年以上にわたり、Denisは金融機関やプロジェクトで、お金に関連した何らかの方法で働いていました。この間、彼は、操作の信頼性とフォールトトレランス、技術的負債、アプリケーションの再設計、開発者およびチームリーダーとしての統合に関する作業に関する豊富な知識を蓄積しました。現在、彼はWrikeでeコマースの構築を支援し、開発者の開発、作業の組織化、およびチーム管理を担当しています。

「分散トレース：Javaサービスの応答時間の分析」（Andrey Markelov、Infobip）


最新のシステムでは、何十もの固有のサービスが参加してクライアント要求を処理できます。原則として、これは顧客にとって興味深いことではありませんが、応答時間とスループットは興味深いものです。レポートでは、Andreiが具体例を使用して応答時間の分析を整理する方法を示します。このレポートは、あらゆるレベルの技術者の練習に役立ちます。

Andreiは11年以上にわたり、金融および電気通信の分野でJavaアプリケーションを開発してきました。ビジネスプロジェクトに加えて、コミュニティで非常に人気のあるアトラシアン製品用の独自のプラグインを含む、オープンソース製品の開発にも積極的に関与しています。アトラシアンコミュニティのアクティブメンバーであり、プロメテウス、Docker、Redisを使用したことについての謝罪。
