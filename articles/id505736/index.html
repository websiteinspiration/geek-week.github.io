Jalankan, PHPUnit, jalankan: bagaimana saya mengoptimalkan waktu eksekusi pengujian

Dari awal bekerja di salah satu proyek web, kami bertujuan pada tingkat tinggi cakupan kode dengan tes, dan pada tahap awal pengembangan, saya tidak berpikir tentang mengoptimalkan kecepatan eksekusi mereka. Akibatnya, dengan pertumbuhan proyek, cakupan tes yang terus meningkat dan pertumbuhan tim, waktu pelaksanaan tes meningkat dari beberapa detik menjadi puluhan menit. Dan melakukan tes cepat bisa sama pentingnya dengan kinerja seluruh aplikasi.


Bagaimana saya melawan ini dan apa yang terjadi pada akhirnya?


Apa yang terjadi sebelum optimasi


Aplikasi sedang dikembangkan dalam PHP 7.2 dan Symfony 4.2. Untuk menulis tes, kami menggunakan PHPUnit. Sebagian besar tes adalah tes integrasi untuk REST API.


Jalankan tes:


> bin/phpunit
...
Time: 20.17 minutes, Memory: 400.25 MB
OK (1494 tests, 5536 assertions)

Semua tes memakan waktu sekitar 20 menit. Ini menyebabkan masalah berikut:


  • proses CI otomatis mulai diblokir selama durasi tes, yang menyebabkan kelelahan pelari CI gratis,
  • pengembang dipaksa untuk berbaris dalam antrian virtual, menunggu penyelesaian pipa CI,
  • selama semua tes, Anda bisa pergi secara lokal untuk membuat secangkir kopi dan daun melalui berita, yang hanya memicu keinginan untuk mengabaikan peluncuran mereka.


Mulai mencari hambatan utama dan bereksperimen!

#!/bin/sh

php_no_xdebug() {
  temporaryPath="$(mktemp -t php.XXXXXX).ini"

  # Using awk to ensure that files ending without newlines do not lead to configuration error
  php -i | grep "\.ini" | grep -o -e '\(/[a-z0-9._-]\+\)\+\.ini' | grep -v xdebug | xargs awk 'FNR==1{print ""}1' | grep -v xdebug >"$temporaryPath"

  php -n -c "$temporaryPath" "$@"
  rm -f "$temporaryPath"
}
