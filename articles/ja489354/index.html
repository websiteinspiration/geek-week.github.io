IdentityServer4。基本概念。OpenID Connect、OAuth 2.0、JWT

この投稿では、IdentityServer4専用の記事スレッドを開きます。基本的な概念から始めます。


現在最も有望な認証プロトコルはOpenID Connectであり、OAuth 2.0は認証（許可）プロトコルです。IdentityServer4はこれら2つのプロトコルを実装します。一般的なセキュリティ問題に最適化されています。


OpenID Connectはプロトコルと認証の標準であり、リソース（Web API）へのアクセスを提供しませんが、これはOAuth 2.0承認プロトコルの上に開発されたものであり、UserInfoリソースへのアクセス権があるかのようにユーザープロファイルパラメータを取得できます。

JWT（JSON Web Token）は、ユーザーデータをJSON形式で暗号化された形式で送信する方法を定義するWeb標準です。

OAuth 2.0（RF​​C 6749）は、プロトコルと認証の標準です。これにより、アプリケーションはWeb APIなどの安全なリソースにアクセスできます。
