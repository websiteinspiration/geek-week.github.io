Pemodelan Farmakokinetik di Julia: Penggunaan Praktis dari DifEquations.jl dan Optim.jl

Model farmakokinetik seringkali merupakan deskripsi matematis yang disederhanakan tentang bagaimana konsentrasi (jumlah) suatu zat uji berubah seiring waktu dalam suatu sistem biologis. Disederhanakan karena tubuh merupakan mekanisme kompleks di mana penyerapan, distribusi, metabolisme dan ekskresi zat ( ADME)) dapat terjadi dalam berbagai cara, tergantung pada substansi itu sendiri dan pada kondisi tubuh saat ini dan tidak jarang dengan umpan balik yang tidak jelas. Oleh karena itu, untuk merefleksikan dan menggambarkan semua faktor yang mungkin bekerja pada kinetika suatu zat praktis tidak mungkin dan dalam banyak kasus tidak dianjurkan. Namun demikian, suatu penyajian dimungkinkan di mana suatu zat didistribusikan di area terbatas tubuh, yang akan disebut kamera, dan transisi suatu zat dari satu ruang ke ruang lain dijelaskan oleh persamaan tertentu.







Pemodelan farmakokinetik dapat dianggap sebagai kasus khusus pemodelan nonlinear. Untuk ini, paket komersial banyak digunakan, seperti: Phoenix WinNonlin, NONMEM, Monolix, dll., Yang memungkinkan Anda untuk mendapatkan estimasi parameter model dari kompleksitas yang berbeda (sewenang-wenang). Ada juga paket yang tersedia untuk lingkungan komputasi Proyek R (nlme, nls, saemix) dan Julia (CurveFit.jl, LsqFit.jl, NLreg.jl), yang dapat digunakan untuk menentukan parameter model farmakokinetik. Batasan Proyek R dan paket Julia terkait dengan fakta bahwa peneliti harus menentukan fungsi nonlinear dalam bentuk eksplisit, yang tidak selalu memungkinkan. Selain itu, tidak semua paket ini memiliki fungsionalitas yang memadai untuk mengevaluasi model dengan efek campuran (efek acak dan tetap).







Pumas.jl agak terpisah, seperti Meskipun fungsionalitas dinyatakan serius, paket belum menerima distribusi luas (mungkin untuk saat ini), mungkin ini (distribusi luas) dicegah oleh lisensi penggunaan tidak jelas. Tujuan dari pekerjaan ini: deskripsi pendekatan utama untuk pemodelan farmakokinetik, pembangunan model farmakokinetik dalam bentuk sistem persamaan diferensial, pemilihan parameter model farmakokinetik.

  Kₐ = p[1]
  K₁₂= p[2]
  K₂₁= p[3]
  Kₑ = p[4]

Kombinasi DiffEquations.jl dan Optim.jl memungkinkan Anda membuat model secara visual dan melakukan analisis. Pada saat yang sama, dimungkinkan untuk membangun model populasi individu dan sederhana.







Kode lengkap:







using DifferentialEquations,  Plots, Optim, LossFunctions, Random, Distributions, DiffEqParamEstim

#Function
function pkf!(du, u, p, t)
  Kₐ = p[1]
  K₁₂= p[2]
  K₂₁= p[3]
  Kₑ = p[4]
  Aᵍ = u[1]
  C₁ = u[2]
  C₂ = u[3]
  Vf = p[5]
  du[1] = - Kₐ  * Aᵍ
  du[2] =   Kₐ  * Aᵍ / Vf   - Kₑ * C₁  - K₁₂ * C₁  + K₂₁ * C₂
  du[3] =   K₁₂ * C₁ - K₂₁ * C₂
end

#Start value
u0    = [3.0, 0.0, 0.0]
#Parameters
p     = [0.8, 0.35, 0.20, 0.15, 0.8]
#Time span
tspan = (0.0, 50.0)
#ODE problem
prob  = ODEProblem(pkf!, u0, tspan, p)
#Solution
sol   = solve(prob)
#Plots
#plot(sol)
#plot(sol,vars=(1))
plot(sol,vars=(2))
#plot!(sol,vars=(3))

x = [0.25, 0.5, 1.0, 2.0, 2.5, 3.0, 3.5, 4.0, 4.5, 5.0,  8.0, 12.0, 24.0, 36.0, 48.0, 60.0]
y = hcat(sol.(x)...)'[:,2] .* exp.(rand(Normal(), length(x)) ./ 10)
scatter!(x,y)

function loss_function(sol, x, y)
   tot_loss = 0.0
   if any((s.retcode != :Success for s in sol))
     tot_loss = Inf
   else
     tot_loss = sum(value(L2DistLoss(), hcat(sol.(x)...)'[:,2], y))
     #tot_loss = sum(value(LogitDistLoss(), hcat(sol.(x)...)'[:,2], y))
     #tot_loss = sum(value(HuberLoss(mean(y)), hcat(sol.(x)...)'[:,2], y))
   end
   tot_loss
end

cost_function = build_loss_objective(prob, Tsit5(), f ->  loss_function(f, x, y))

result = optimize(cost_function, [0.4, 0.1, 0.1, 0.1, 1.0], NelderMead(), Optim.Options(allow_f_increases = true, iterations = 5000, time_limit = 60.0))

prob   = ODEProblem(pkf!, u0, tspan, result.minimizer)
sol    = solve(prob)
plot!(sol,vars=(2))
