O que é um hub de dados do Google Ads

Uma dor típica de um profissional de marketing é avaliar a eficácia da publicidade gráfica (porque as interações com ela ocorrem fora do site do anunciante). As ferramentas clássicas de análise da web não podem fazer isso e o restante das ferramentas não fornece uma avaliação transparente.

Para resolver esse problema, o Google lançou o Ads Data Hub , uma ferramenta para recuperar dados de inventário de mídia para avaliar sua eficácia. Permite combinar dados de publicidade gráfica com dados do site e CRM.

Abaixo, falarei sobre o que é o Hub de dados de anúncios e como ele funciona.

Para que serve um hub de dados de anúncios?


Existem conversões de exibição clássicas . A maioria das ferramentas de mídia permite que você saiba quantos usuários daqueles que viram o banner fizeram um pedido. Mas essas ferramentas não mostram todas as etapas do usuário, desde a exibição do banner até a compra, e não levam em consideração os pedidos adquiridos no sistema CRM.

Para que os anunciantes possam combinar as informações da campanha com seus próprios dados, os serviços de publicidade fornecem ferramentas de upload, como o Data Transfe r do Google. Anteriormente, ele podia ser usado para fazer upload de dados brutos de log para cada clique, impressão ou evento, incluindo o ID do usuário, do DoubleClick Campaign Manager (DCM) para o Google BigQuery. No entanto, em 2018 as regras do RGPDproibiu os anunciantes de fazer upload de dados do usuário, pois eles interagem com o site do editor, não com o anunciante.

Portanto, o Google deu aos anunciantes a oportunidade de avaliar de forma transparente a eficácia da publicidade gráfica sem violar os requisitos de privacidade. Agora, os códigos de usuário não são descarregados do DCM e do DPM, mas estão disponíveis para trabalhar no Hub de dados do anúncio.

O que mais causa a aparência do Hub de dados de anúncios? Para a maioria dos anunciantes, a publicidade gráfica é uma alternativa à publicidade na TV ou no exterior. Ou seja, isso não substitui o marketing de desempenho, mas outra ferramenta de cobertura. No entanto, ao contrário da mesma TV, não existem parâmetros de referência estabelecidos na publicidade gráfica. Portanto, um anunciante precisa de uma ferramenta que mostre com qual público-alvo eles interagiram graças à exibição de publicidade.

E o Google criou essa ferramenta. Num futuro próximo, a maioria dos grandes anunciantes que investem em publicidade gráfica enfrentará a necessidade de desenvolver um Hub de dados de anúncios.

Como o Hub de dados do Google Ads funciona


O Ads Data Hub não é um repositório independente. Esta é uma ferramenta para trabalhar com dados no Google BigQuery. Em essência, o Ads Data Hub vincula dois projetos do BigQuery - o seu e o do Google.



Um projeto do Google armazena dados de log do Campaign Manager, Display & Video 360, YouTube e Google Ads. Esta é a mesma informação que não pode ser obtida externamente devido às regras do GDPR.

O segundo projeto, de sua propriedade, armazena todos os dados de marketing (online e offline) que você carregou no BigQuery a partir do Google Analytics, CRM ou outras fontes. A equipe do Google não tem acesso ao seu projeto.

O Ads Data Hub é uma API que permite solicitar dados simultaneamente desses dois projetos sem fazer upload deles no nível do usuário. Graças a isso, você pode associar todos os cliques e conversões a dados de impressão e ver como uma determinada campanha afetou a conversão.

Importante: os resultados da consulta retornados pelo Ads Data Hub são agregados até 50 usuários. Ou seja, cada linha da tabela deve conter dados sobre 50 ou mais usuários. Não é possível detalhar para um usuário específico. Essa restrição é introduzida para cumprir as regras do GDPR.

Um exemplo de relatório que pode ser criado usando o Hub de dados de anúncios


Digamos que você publique campanhas da Rede de Display do Google e queira avaliar sua eficácia. Usando o Hub de dados de anúncios, você pode combinar impressões do Campaign Manager, sessões e pedidos on-line do Google Analytics, pedidos comprados do CRM em uma única consulta SQL.

Como resultado, você receberá um relatório que contém as seguintes informações:

  1. O nome das campanhas publicitárias.
  2. O número de impressões para cada campanha.
  3. O alcance da campanha é o número de usuários que viram o banner.
  4. Sessões de usuários que visitaram o site após visualizar o banner. Além disso, o usuário não precisa clicar no link no anúncio. Isso leva em conta a sessão e aqueles que visitaram o site sem um clique. Por exemplo, vi um anúncio, lembrei-o e encontrei seu site em uma pesquisa.
  5. Clica no link no anúncio.
  6. Conversões online.
  7. Pedidos confirmados do seu CRM.



Ou seja, o Hub de dados de anúncios permite avaliar as conversões pós-visualização. Se você construísse esse relatório sem o ADH, haveria apenas os usuários que visitaram o site clicando no banner. Você não saberia nada sobre quem não clicou, mas visitou o site e também no relatório não haveria vendas reais do CRM (colunas amarelas no relatório).

Como usar o Hub de dados de anúncios


Para usar o Hub de dados de anúncios, você precisará de dados do site e do sistema de CRM no Google BigQuery, que serão combinados com os dados sobre as impressões de campanhas publicitárias.

Você também precisa configurar a transferência do Client Client ID para o Campaign Manager (anteriormente DCM) para que haja um identificador pelo qual você possa combinar dados de visitas ao site com informações de impressão. Isso pode ser feito, por exemplo, usando o Floodlight .

Anteriormente, para trocar informações, os profissionais de marketing criavam arquivos do Excel e os enviavam um ao outro. Com o tempo, chegou-se ao entendimento de que é melhor armazenar dados em um único local e acessar informações relevantes quando necessário. Então, o conceito de nuvem para documentos apareceu.

Obviamente, o Excel continua a ser usado para trabalhar com dados aqui e agora. Mas esses são dois conceitos diferentes - uma coisa é quando os usuários se conectam a uma fonte e a outra é quando todos têm uma cópia dos dados.

Os dados da MarTech evoluíram historicamente no mesmo caminho, quando todos os serviços da MarTech para computação e geração de relatórios exigiam uma cópia dos dados. Se você deseja criar um relatório no GA - faça o upload dos dados para lá, se desejar criar um relatório no DCM - faça o upload dos dados para o BigQuery. O Ads Data Hub é um substituto conceitual para esse paradigma.

Qual é o problema? Agora, um profissional de marketing pode fazer upload de dados do Google Ads para o BigQuery e, depois disso, será alterado retrospectivamente no serviço de publicidade. Como resultado, em seu projeto BigQuery, os dados ficarão obsoletos se não forem atualizados.

O segundo problema é quais dados carregar? Há muitas informações sobre publicidade gráfica. Para que um profissional de marketing crie um relatório uma vez por semana, você precisa carregar gigabytes de informações em seu projeto. Em vez disso, o Ads Data Hub sugere conectar a interface SQL aos dados dos serviços de publicidade do Google. Ou seja, você tem um projeto GBQ. Ele contém seus dados que o Google não possui (por exemplo, do CRM). Agora que você precisa de um relatório com dados de campanhas publicitárias, basta conectá-los à sua solicitação e obter as informações mais recentes aqui e agora. Você não precisa mais armazenar dados de publicidade em seu projeto. Atualizá-los todos os dias, quando algo mudou ao longo de períodos históricos, também não é necessário. Para relatórios, absolutamente todos os dados que você coleta estão disponíveis.
