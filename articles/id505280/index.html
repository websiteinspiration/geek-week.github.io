Magento 2: CSS Kritis



Pada artikel ini, kita akan melihat cara mengkonfigurasi dan mengaktifkan css kritis di Magento 2.

Apa yang kamu, css kritis?


Baik HTML dan CSS memperlambat output halaman. Dokumen HTML perlu diproses untuk membuat model DOM, yang tanpanya tidak akan ada konten pada halaman. Dokumen CSS terlebih dahulu harus dimuat dan kemudian diproses (CSS Object Model).

Untuk mempercepat proses, Anda perlu menyederhanakan CSS dan menggunakan kueri media. Css kritis juga berkontribusi pada peningkatan kinerja.

Seperti yang kami katakan di atas, semua gaya CSS yang dimuat dari file eksternal diperlakukan sebagai rendering pemblokiran. Ini berarti bahwa halaman web tidak akan ditampilkan sampai file-file ini diunduh. Menggunakan css kritis, kami menyediakan "CSS kritis" yang tertanam di dalamnya dan menjadikan pemuatan semua gaya lainnya asinkron.

Pemuatan asinkron dicapai dengan menambahkan arahan preload baruke atribut rel

Untuk mendukung cross-browser rel = "preload", Magento menggunakan polyfill.

Sebuah fitur penting, "critical css" hanya mencakup bagian yang terlihat dari halaman.



Dengan demikian, kami dapat secara signifikan meningkatkan waktu tampilan "pertama" dari halaman kami.

Bagaimana cara menyalakannya?


Toko - Pengaturan - Konfigurasi - Lanjutan - Pengembang

Tab Lanjut hanya tersedia dalam mode pengembang

Setel Gunakan jalur kritis CSS ke ya

Atau gunakan baris perintah: konfigurasi bin / magento: set dev / css / use_css_critical_path 1

CSS kritis harus ditempatkan di app / desain / frontend / vendor_name / theme_name / theme_name /web/css/critical.css



Bagaimana cara menghasilkan?


Proses pembuatan manual sepenuhnya cukup memakan waktu dan memakan waktu.

Generator seperti Penthouse atau Critical dapat digunakan untuk membuat CSS kritis untuk tema Anda.

Atau beralih ke sumber daring yang cukup bagus.



Penting untuk menyisipkan alamat url ke sumber daya di bidang (1), dan menyisipkan css, yang digunakan di situs, dan klik tombol Buat di bidang (2).



Sebagai hasilnya, kita mendapatkan css kritis yang dihasilkan dan diperkecil.



Bagian tertentu dari pekerjaan manual, tentu saja, kita harus:

  • Memeriksa dan menyesuaikan css kritis yang dihasilkan
  • Jangan lupa, juga bahwa critical.css di Magento adalah satu untuk semua halaman, jadi penting untuk tidak melupakan daftar, halaman produk, halaman pendaftaran dan login
