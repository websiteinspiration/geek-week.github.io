背景：QoEとは何か、プロバイダーがQoEを使用する理由

QoEの主なコンポーネントとその決定方法について説明します。また、QoE情報がサービスプロバイダーのサービス品質の向上にどのように役立つかについても説明します。


/ CC BY / Norlando Pobre

QoEの意味


IT業界でのQuality of Experienceの単一の定義はありません。しかし、通常、QoEはインターネットプロバイダーのサービスに対する顧客の一般的な満足度として理解されています。

知覚の質は、主観的および客観的ないくつかの要因に依存します。最初のケースでは、ユーザーの評価は、他のオペレーターとの作業経験に影響されます。価格、テクニカルサポートの比率などを比較します。



客観的なQoE要因の主な指標は、サービスの品質（QoS）です。この場合、加入者の印象は、ネットワークパフォーマンスと安定性、ビデオ視聴時のアーティファクトの有無、およびVoIP会話の不要なクリックとノイズによって形成されます。

QoEの測定方法


QoEに影響を与えるいくつかの要因は主観的ですが、ユーザーの知覚の品質を測定できます。この分野では多くの研究が行われています。
決定方法は、サービスまたはサービス（オーディオ、ビデオ、ストリーミングなど）によって異なります。

スピーチとオーディオ。彼らは、国際電気通信連合（ITU）によって提案された、いわゆるEモデルを使用します。彼女は、機器の特性とネットワークパラメータに基づいて、VoIPでの会話中のユーザーエクスペリエンスを予測します。この情報は式に代入され、その係数は一連の主観的な実験に基づいて選択されます。

R=R0−Is−Id−Ie+A


ここに、 R0 信号対雑音比は、 Is -会話中のスピーチの音量と倍音の存在を考慮し、 Id -信号遅延に起因する歪みのレベル、 Ie-使用されたコーデックと機器の影響を評価します。についてA、これはQoE品質の許容可能な低下であり、クライアントはこれに耐える準備ができています（たとえば、Wi-Fiに接続されているときのモビリティと引き換えに）。各パラメータの正確な数値は、ITUのドキュメントに記載されています。

さらに、係数Rユーザーの知覚の質の平均値を表す平均オピニオンスコア（MOS）で再計算されます。公式は、3ページのエンジニアの作業にあります。

Habréで私たちに：

  • テレコムダイジェスト：オペレーターのネットワークの構築、インターネットプロトコル、情報セキュリティ
  • プロバイダー用ツールキット：トラフィックとその構成を操作するシステムに関するテーマ別ウェビナー
  • ディスカッション：人気のビデオホスティングがハッキングガイドでコンテンツをますます削除している理由

ネットワークゲーム。コンピューターでプレイするときの知覚の質を評価するために、多くのプロバイダーがGモデルを使用しています。これは、オランダ応用科学研究機構（TNO）のエンジニアグループによって提案されました。レイテンシ、ジッター、ネットワーク接続パケットの損失などの要素が考慮されます。式は次のとおりです。

MOS=max(4,33−3,08∙10−9X3+1,18∙10−5X2−1,15∙10−2X,1)


パラメータの後ろ XQoEへのネットワーク停止の影響は隠されています。次のように計算されます。

X=min(PING+0,686∙JITTER,650)


オランダの専門家は、Quake IV、Quake III、Unreal Tournament、Counter Strike、Haloなどのゲームでフォーミュラをテストしました。この方法の精度は約98％でした。

ビデオ。ビデオを見るときのQoEは、画像の明るさやコントラスト、色再現、目に見える歪みやアーティファクトの存在などの要因の影響を受けます。時々、ビデオの知覚の品質は、一連の主観的テストを使用して測定されます。専門家はビデオクリップを見て、その品質を5段階で評価し、結果を平均化します。人的要因を排除するために、ビデオ品質メトリックなどの特殊なソフトウェアを使用して、ぼやけ、不自然な画像の動き、ノイズなど、ビデオ内のさまざまなアーチファクトを検出します。

QoEの測定方法


QoEの評価では、システムも使用できます。ネットワーク機器とユーザーアクティビティを監視するための一連の機能と、アプリケーションを記録して調査を実施するためのシステムを提供します。ここにある2つのメトリック：知覚品質を決定する

RTT（ラウンドトリップ時間）。これは、信号の送信、処理、および受信の確認に必要な時間です。


/ RTT計算/ QoE

再送信に関するVASエキスパートのウェビナースライド。システムは、クライアントがサーバーからのSYN / ACK応答を10秒間待たないたびに発生する、繰り返されるデータ送信試行を登録します。

RTTと再送信のデータは、グラフの形でダッシュボードに表示されます。監視部門は、これらの統計を処理し、RTTインジケータの増加を視覚的に追跡することで対応できます。


/ RTT変更チャート/ VASエキスパートウェビナースライド

なぜこれがインターネットサービスプロバイダーなのか


接続エラーを監視することにより、ネットワークの全体的なステータスを評価できます。それらが体系的である場合、ITインフラストラクチャに問題がないことを確認する価値があります。

QoEを使用すると、他の問題を解決できます。マーケティング担当者は、RTTと再送信機を評価することにより、最高のパフォーマンスとカバレッジを備えた機器を販売する機会を得て、加入者の個人アカウントを通じてチャネルの負荷と品質に関する情報を提供します。このような手順は、QoEを高め、潜在的な顧客チャーンを減らすのに役立ちます。
