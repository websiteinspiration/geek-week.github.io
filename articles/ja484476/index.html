数学ソフトウェアの分野での個人的な関与の話をせずにセージの歴史について書くのは難しいです。覚えている限り、電卓を使うのが大好きでした。1970年代の私がまだ非常に小さい頃、父はどういうわけか私に電気機械式積算機を手に入れました。それを使って紙テープに書き込むのに多くの時間を費やしました。それから彼はフリーマーケットで私にLEDディスプレイを備えた古いTI関数電卓をもらいました。オレゴンからテキサスに引っ越してきた11歳のとき、私はしばらくの間本当に心を奪われる計算尺を購入しました。その後、小学校の成績で、ようやく本当の関数電卓である小さなカシオを手に入れました。これは私の最初の紹介でした三角法、統計、およびその他多くの計算数学。自分のペースで学んだ変わったカリキュラムに従って小学校を通過したが、すぐに10年生レベルで数学を学び始めたが、私は良い仕事をした。

私が今まで使用した最初のコンピュータープログラムはMathematicaで、1992年にWindows 3.1を実行しているPCで、フラッグスタッフにある北アリゾナ大学の17歳の学生でした。

当然のことながら、私のMathematicaのコピーは海賊版でした。なぜなら、多くの学生と同じように、私は当時非常に貧しかったからです。そのとき私がそれが有用だと思った唯一の理由は、3次元グラフの作成（単に娯楽のため）であり、これもインタラクティブに視点を変更できないことを失望させました。また、Mathematicaよりも適していると思われるMATHCADのコピーも入手できました。私はコンピュータエンジニアリングの学生であり、物理学、電気工学、プログラミングなどのコースを受講していたので、これはおそらく驚くべきことではありません。数学は私の主な主題ではありませんでした。多くの変数の分析に関するクラスを終えたとき、「これは私のものです。最後の数学的訓練！」私も使った私の数学コースの一環として、コンピュータラボで約1〜2時間メープルを食べます。当時、私はMaple、Mathematica、およびMATHCADを、特に魅力的な特性を備えた関数電卓を超えないソフトウェアとして認識していました。

1.抽象代数。1994.ケイリー。1997. MATLAB、Linux、C / C ++の経験


1994年初頭に数学ソフトウェアを思いついたとき、私が数学をプロファイルとして選択したとき、コンピューターサイエンスに変装した抽象的な本の中で本を偶然見つけて、グループ、リング、フィールドなどのアイデアに完全に夢中になりました。別のコンピューターラボでは、前のシステムMagmaであるシステムCayleyの印刷物（北アリゾナ大学の教授）Mike Falk（Mike Falk）が利用できたドキュメントに行き詰まりました。Cayleyで具体化されたアイデアが、以前は数学ソフトウェアで一般的に可能と考えていたすべてのことをはるかに先に進めたので、私は驚き、感激しました。私はCayleyを実際に使用したことはないと確信していますが、今のように、ソフトウェアは高価で手が届きませんでした。代わりに、グループ、リング、フィールドを使用した計算の素晴らしい例がいっぱいあるリファレンスマニュアルを読むのに多くの時間を費やしました。主に数学に興味を持った科目です。

実際、Cayleyとの短い会議の後、私は数学ソフトウェアに触れなくなり、3年間重要なことさえプログラミングしませんでした。それは、1994年に理論数学に非常に興味があり、この3年間のほとんどの時間を、線形代数と組み合わせ論の基礎に関する本（アリゾナでは十分以上でした）から始まり、数学の本の体系的な演習に費やしたためです。Hartshornの「代数幾何学」[これは、R。Hartshornによる古典的な本「Algebraic Geometry」を参照します。これは、Mir Publishing Houseによってロシア語でも出版されました。トランス。]。

1997年に私はバークレーの大学院生で、あまり教えたくなく、仕事を見つけました（全米科学財団のVIGREプログラムによって資金提供されました[全米科学財団、全米科学財団-科学技術の開発を担当する米国政府の独立機関、以下-NSF-およそ。 [レーン]）学部で1年間、大学院のフェローであるトムインセルと一緒に、代数の大学コースの資料に基づいてカリキュラムを準備（トム・インゼル）。私がLinuxを数年間使用したという事実にもかかわらず、私はフリーソフトウェアについて考えたことはなく、一部の人々が実際にそれに貢献できるという事実もありませんでした。 Linuxは「自分のコンピューターにインストールできるUNIX」だと思っていましたが、モデム経由のダウンロード速度が非常に遅いため、CDを購入してLinuxの代金を支払う人もいました。トムと私は長い間一緒にプロジェクトに取り組んでおり、Slackware（Linuxディストリビューション）に含まれているソフトウェアの作成方法を教えてくれたため、新しいバージョンがリリースされたときにCDの無料コピーを受け取った。ねえ、誰でもLinuxに貢献できます！

また、MATLABでプログラミングしなければならなかったとトムがよく不平を言ったことも思い出します私たちのプロジェクトのために、そしてJavaを含む私たちのビジネスにより適している多くの代替案に言及しました。ほんの少しの事実に基づく数学を使ってGUIをプログラミングしていたので、MATLABベースのハンドルベースのGUIライティングシステムは本当に大変でした。コンピューターサイエンスの学生以来、C / C ++ / Windows 3.1 GUI開発で多くの経験があり、MATLABが当時私たちが行っていたことに対して完全に扱いにくいことに同意しました。振り返ってみると、私たちがしたことはおそらく意味がなく、おそらく使用されていなかったと言えます。しかし、この経験は私たち2人にとって非常に貴重であり、NSFがこれをサポートしてくれてうれしいです。

時間の経過とともに、博士号の私の論文。 D.は3年の大学院にもかかわらず、どこにも進まなかった。私はその仕事について聞いたら、ケン・リベットが「あり：すべての大学院生を尋ねプライム pがあるようヘッケ代数が [ドイツの数学者にちなんで命名されたエリック・ヘッケ -名前はまだ本書に言及され、約あたりには。]のレベルpされる分枝状のpに？」これは、私の知る限り、コンピュータを使わずに解決することはほぼ確実に不可能だと聞いた最初の研究課題です。クラスメートのJanos Chirik（Janos Csirik、現在はDE Shawにいる）、Matt Bakerと一緒に（マットベーカー、現在はジョージアテック）1970年代半ばから1つの出版物、土方を検索して見つけたため、上記の質問に特定のpで答えることができました。しかし、アルゴリズムを実装するには、多数の2次フィールドのクラス番号を計算する必要がありますが、そのときに聞いたことがある数学的プログラムではこれを実行できませんでした。 JanosとMattはPARIを推奨し、私は自分のコンピューターにインストールしました。実際、その助けにより、クラスグループをすばやく簡単に計算することができました。 PARIは私が最初に出会った無料の数学プログラムでもありました。



ケネスアラン（「ケン」）リベットはアメリカの数学者です。代数的数論と代数的幾何学のスペシャリスト。谷山 - 志村 - ヴェイル（谷山 - 志村 - ウェイル）の推論 とフェルマーの最後の定理、後者の証明におけるアンドリューウィルズ（アンドリューウィルズ）の背後にある理由の間の「橋」を築きました。この記事で言及したJanos Chirik の学問的な「父」、Dimitar Zhechev（後者は第2部で登場）、およびBjorn Mikhail Poonen（学者の息子はWilliam Stein）です。

それで、アルゴリズムのコーディングを始めました（アイヒラートレース式-Selberg（アイヒラー-Selberg） [ 1、2 ] PARIで）。私は、ユーザー定義のデータ型、例外処理、テンプレートなどを備えた本格的なプログラミング言語であるC ++について多くの経験をしました。1992年に、C ++に関する多くの本を最初から最後まで完全に読んだことを覚えています。私はずっと前にC ++で多くのコード（ビデオゲーム！）を書きました。同時に、PARIは完全なショックをもたらしました。これはローカル変数がなく、実際のスコープがなく、組み込み型が少ししかない言語でした。しばらく考えたように（誤って）、すべての関数定義は同じ行にあるはずです。これは、私が見つけたコードサンプルの場合と同じです。ただし、代数的数論の分野のアルゴリズム内部ライブラリに実装されたものは美しく、深く、非常に高速でした。その結果、私はトレース式を実装し、Ribetにすばやく回答を提供するために実現したことを開始しました。プログラムは機能しませんでした-基本的な整合性チェックが失敗しました。クラスグループを計算するアルゴリズムに重大なバグがあることがわかりました。特に、qfbclassno関数では、不正な応答を暗黙的に返していました。
