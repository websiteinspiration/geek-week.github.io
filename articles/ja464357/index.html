問題があります-分散ネットワークで乱数を生成することは困難です。ほとんどすべてのブロックチェーンはすでにこれに遭遇しています。実際、ユーザー間に信頼関係がないネットワークでは、紛れもない乱数を作成することで多くの問題が解決されます。

記事では、例としてゲームを使用して問題を解決する方法を説明しています。これらの最初は、Waves Xmas Treeでした。開発には、乱数ジェネレータが必要でした。



最初は、ブロックチェーンからの情報に基づいて数を生成することを計画していました。しかし、その後、それが明らかになりました。数値は不正である可能性があり、ソリューションが適合しないことを意味します。

私たちは回避策を思いつきました：commit-disclosureスキームを使用してください。サーバーは1から5までの数を「推測」し、それに「塩」を追加して、Keccak関数を使用して結果をハッシュしました。サーバーは、既に保存されている番号でスマートコントラクトを事前に展開しました。ゲームは、ユーザーがハッシュによって隠された数を推測したという事実に縮小されたことがわかります。

プレーヤーは賭けを行い、サーバーは要求された番号と「ソルト」をスマートコントラクトに送信しました。彼は平易な言葉でカードを明らかにした。その後、サーバーは数字を検証し、ユーザーが勝ったか負けたかを決定しました。

サーバーが確認のために番号または「塩」を送信しなかった場合、ユーザーは勝ちました。この場合、ゲームごとに事前にスマートコントラクトを展開し、そこに潜在的な勝利を収める必要がありました。それは不便で、長くて高価であることがわかりました。当時、他に安全な解決策はありませんでした。

# @return 1 ... 100
func GenerateRandInt (gameId,rsaSign) = {
   	# verify RSA signature to proof random
    let rsaSigValid = rsaVerify (SHA256, toBytes(gameId), rsaSign, RSAPUBLIC)
    if (rsaSigValid)
        then {
            let rand = (toInt(sha256(rsaSign)) % 100)
            if ((0 > rand))
                then ((-1 * rand) + 1)
                else (rand + 1)
            }
        else throw ("Invalid RSA signature")
    }

rand-乱数があります。

最初に、秘密鍵（rsaSign）を使用したゲームIDの RSA署名の結果である文字列が取得されます。次に、SHA-256（sha256（rsaSign））を使用してハッシュされます。 署名とそれに続くハッシュの結果を予測することはできません。したがって、乱数の生成に影響を与えることは不可能です。特定の範囲（たとえば、1から100）の数値を取得するには、toIntおよび％100変換関数を使用します（アナログからmodへ）。 記事の冒頭で、公開鍵を使用して秘密鍵でRSA署名の有効性を確認できるrsaVerify（）関数について説明しました。以下は、GenerateRandInt（gameId、rsaSign）の部分です。





rsaVerify (SHA256, toBytes(gameId), rsaSign, RSAPUBLIC)

RSAPUBLIC公開鍵とrsaSign文字列が入力に渡されます。署名の有効性がチェックされます。チェックが成功すると、番号が生成されます。それ以外の場合、システムは署名が無効であると見なします（無効なRSA署名）。

サーバーは秘密鍵でゲームIDに署名し、有効なRsa署名を2880ブロック以内で送信する必要があります。このパラメーターは、スマートコントラクトの展開時に構成されます。割り当てられた時間中に何も起こらなければ、ユーザーが勝ちます。この場合、賞品はご自分の住所に送付する必要があります。これは損失につながるため、サーバーは「チートすることは有益ではありません」と判明しました。以下に例を示します。



ユーザーがサイコロローラーをプレイ。私は立方体の6つの面から2つを選びました、賭けは14波です。サーバーが設定された時間（2880ブロック）内にスマートコントラクトで有効なRSA署名を送信しない場合、ユーザーは34.44波を取得します。

ゲームで数値を生成するために、外部の非ブロッキングシステムであるオラクルを使用します。サーバーは、ゲームIDのRSA署名を実装します。スマートコントラクトは、署名の有効性をチェックし、勝者を決定します。サーバーが何も送信しなかった場合は、ユーザーが自動的に勝ちます。
