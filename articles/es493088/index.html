Analizando un sitio de cine en Golang

Hola a todos, en este artículo veremos un pequeño código de muestra escrito para recopilar datos de un sitio en el popular lenguaje de programación Go today.

Este artículo será especialmente interesante para aquellos que han escuchado mucho sobre Go, pero aún no lo han probado por su cuenta.

Para uno de los proyectos internos, necesitábamos recopilar datos sobre las películas que se alquilan y el calendario de espectáculos, ahora consideraremos la primera versión (más simple) del analizador, con la que todo comenzó.

Para aquellos que son demasiado vagos para leer el código del artículo, les proporciono inmediatamente un enlace al repositorio .

En su primera versión, el analizador pudo recopilar solo la siguiente información:

  • Sobre sesiones en un cine,
  • Descripción detallada,
  • Título,
  • Estudio,

Y guárdelo en JSON. Inicialmente, debe seleccionar la biblioteca adecuada para el análisis.

Google ofrece una gran cantidad de opciones para la consulta `raspado web de golang`, muchas de ellas se reflejan en esta lista , le aconsejo que la lea, elegí geziyor , ya que admite JS Rendering (que, por cierto, no usaremos en este Sin embargo, esta característica es muy útil al analizar sitios) y es bastante simple de usar.

Entonces, la biblioteca está seleccionada, el siguiente paso es instalarla y comenzar a usarla en el código.
Instalar la biblioteca es extremadamente simple:

go get -u github.com/geziyor/geziyor

Ahora pasemos a escribir código.

Dentro del cuerpo de la función principal, llamamos al analizador, le damos la URL de la página desde la que comenzaremos a recopilar datos e indicamos que queremos exportar el resultado a un archivo JSON:


func main() { 
    geziyor.NewGeziyor(&geziyor.Options{ 
        StartURLs: []string{"https://kinoteatr.ru/raspisanie-kinoteatrov/city/#"}, 
        ParseFunc: parseMovies, 
        Exporters: []export.Exporter{&export.JSON{}}, 
    }).Start() 
}

Se ha comenzado, pero no hay suficiente lógica para recopilar datos, para esto necesitamos implementar la función parseMovies .

La lógica de recopilación será la siguiente:

  • Busque un bloque que contenga información de la película,
  • Recopilación de información sobre todas las sesiones dentro de este bloque,
  • Colección del nombre de la película y estudio de cine,
  • Recopilando un enlace a una página con información detallada sobre la película,
  • Colección de descripciones de esta página.

Pasemos a la implementación de esta función.

De esta manera, se recopila información sobre las sesiones y luego se lleva a una forma conveniente para una persona (eliminamos espacios y sangrías adicionales en una nueva línea).


var sessions = strings.Split(s.Find(".shedule_session_time").Text(), " \n ") 
sessions = sessions[:len(sessions)-1] 

for i := 0; i < len(sessions); i++ { 
    sessions[i] = strings.Trim(sessions[i], "\n ") 
}

Este bloque de código es responsable de obtener una página con información detallada sobre la película y obtener su descripción.


if href, ok := s.Find("a.gtm-ec-list-item-movie").Attr("href"); ok {
    g.Get(r.JoinURL(href), func(_g *geziyor.Geziyor, _r *client.Response) {
        description = _r.HTMLDoc.Find("span.announce p.movie_card_description_inform").Text() 
        description = strings.ReplaceAll(description, "\t", "") 
        description = strings.ReplaceAll(description, "\n", "")

Esto llama a la API para exportar los resultados a un archivo JSON.


g.Exports <- map[string]interface{}{
