こんにちは！私の名前はユーリヴラドです。私はBadooのAndroid開発者で、純粋なKotlinでReaktiveライブラリ-Reactive Extensionsの作成に参加しています。


どのライブラリでも、可能な限りバイナリ互換性を監視する必要があります。依存関係にあるライブラリの異なるバージョンに互換性がない場合、結果は実行時にクラッシュします。Reaktiveのサポートを追加するとき私たちは、例えば、このような問題が発生することがありMVICore。



この記事では、バイナリ互換性とその機能がKotlinでどのように機能するか、JetBrains、および現在Badooでどのようにサポートされているかを簡単に説明します。

        basePath: String,
        jarPattern: String,
        publicPackages: List<String> = emptyList(),
        nonPublicPackages: List<String> = emptyList()
    ) {
        val base = File(basePath).absoluteFile.normalize()

        println("Reading binary API from $jarFile")

        val target = File("reference-public-api")
            .resolve(testName.methodName.replaceCamelCaseWithDashedLowerCase() + ".txt")

        api.dumpAndCompareWith(target)
    }
