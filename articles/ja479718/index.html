CI GitHubを使用したArduino環境アプリケーションの構築



CI / CD（gitlab ce）を扱う仕事が少しあったので、最近githubでアクションに出くわし、これがどんな種類の獣かを試すことにしました。

JSやその他のソリューションを構築することはあまり面白くなかったので、esp8266のArduinoスケッチのアセンブリをテストし、結果としてコンパイル済みのバイナリを取得し、ローカルのArduino IDEをインストールする必要なしにマイクロコントローラーに直接ダウンロードする準備をしました。

アクション、またはパイプライン
, — , push git , . github, . Pipeline , .

私はこれすべてのためにドッカーイメージの作成を開始していません。これは、ci / cdを整理するプロセスを複雑にします。また、ライブラリごとにイメージを作成する必要はありません。

これはどのように役立ちますか？メモリの量が異なるコントローラのバージョンごとに、ある種のオープンソースプロジェクトがあり、すべてのプラットフォーム用のコードをコンパイルするにはかなりの時間がかかります。アクションでは、アセンブリが必要なプラットフォームを一度構成できます。プラットフォームの各バージョンのリポジトリ内のコードを変更するたびに、準備ができたファームウェアが存在します。

アクションスクリプト自体のコード：

on: [push]
jobs:
  build:
    runs-on: ubuntu-18.04
    steps:
    - uses: actions/checkout@v1
    - name: install arduino cli
      run:  curl -fsSL https://raw.githubusercontent.com/arduino/arduino-cli/master/install.sh | sh
    - name: MakeSketch
      run: ./bin/arduino-cli sketch new MyFirstSketch
    - name: create dir
      run: mkdir -p /home/runner/.arduino15/packages
    - name: lib init
      run: sudo ./bin/arduino-cli core update-index --additional-urls http://arduino.esp8266.com/stable/package_esp8266com_index.json
    - name: install esp8266
      run: sudo ./bin/arduino-cli core install esp8266:esp8266 --additional-urls http://arduino.esp8266.com/stable/package_esp8266com_index.json  
    - name: install python3
      run: sudo apt-get install -yq python3
    - name: compile
      run: sudo ./bin/arduino-cli compile --fqbn esp8266:esp8266:d1 MyFirstSketch
    - uses: actions/upload-artifact@v1
      with:
        name: esp8266-d1-mini-4m.bin
        path: MyFirstSketch/MyFirstSketch.esp8266.esp8266.d1.bin 
    - uses: ncipollo/release-action@v1
      with:
        artifacts: "MyFirstSketch/MyFirstSketch.esp8266.esp8266.d1.bin"
        tag: "release"
        token: ${{ secrets.GITHUB_TOKEN }}

実際には -

  • ubuntu 18.04のクリーンなシステムにarduino-cliをインストールします
  • esp8266サポートを追加
  • リポジトリでホストされているarduinoコードをコンパイルし、
  • コンパイルされたバイナリを追加するリリースを作成します



オープンソースプロジェクトではgithubのアクションは無料ですが、クローズドリポジトリではアクションも使用できるようになりますが、しばらくするとアクセスできなくなるようです。

私はdevopsを装うつもりはありません、いくつかのことはより簡単に行うことができますが、いずれにせよ、それは機能します。
