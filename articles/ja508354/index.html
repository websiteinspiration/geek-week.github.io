ティムリッド。一度にたくさん。選択と開発の方法

こんにちは！私の名前はアンドレイノビコフです。私は、Exnessの1つの部門の開発責任者です。HRBPのLena Skvortsovaと一緒に、開発チームでチームリーダーをどのように選び、どのように開発し、キプロスの太陽の下で成長させているかをお伝えします。 







一定の成長期にある多くの企業は、次の質問に直面しています。自分で成長するか、市場で準備をしますか？同社が着実に成長しており、キプロスにある私たちの技術オフィスは現在、最も多数の1つです。 





2020年1月6日のインフォグラフィック



ロシアと西洋の両方の多くの企業の経験を調査した結果、最も人気のある決定、つまりチームリーダーの成長と育成、そして例外的には市場からの採用を決定しました。それの訳は： 



  1. 移転は多くの人にとって深刻なステップであるため、キプロスでの従業員の雇用は常にCISでの採用よりも時間がかかります。
  2. 7-9チームリーダーを一度に採用することは、何年もかかる大変な仕事です。
  3. 私たちのエンジニアにとって、チームリーダーになることは、将来管理職に就きたいと考えている子供たちが社内で成長できるようにするためのもう1つの開発経路です。 
  4. チームがすでに知っている人の方が、ストームとグラインドの段階がはるかに速くなります。 


記事は5つのパートで構成されています。プログラムとアプローチを借りることができ、私たちの仕事が同様の課題に直面している他の企業に役立つと私たちは喜んでいます。

選考プロセス



私たちのタスクは、8つのチームのそれぞれで同時にチームリーダーを選択することでした。これにより、新しい形式の管理への移行が開発チームで同じになり、みんなが同時に学習を開始できるようになりました。 



これを行うために、プロセス全体をいくつかの段階に分けました。



  1. 自己準備;
  2. 就職の面接; 
  3. 結果の校正。 






自己準備 



独立した準備として、チームのすべての候補者に書面での割り当てを与えました-「チームリーダーになった場合、今すぐチームで何を変えますか？」、「チームリーダーになりたいのはなぜですか？」、「どのような問題と課題が必要ですか？」私は自分を新しい役割に見ていますか？」



これらの質問への回答は非常に重要です。なぜなら、一方では「本当にチームリーダーになりたいですか？」という質問についてもう一度考えさせるためです。もう1つは、私の経験、長所と短所を振り返り、チームの開発の問題と領域を強調します。現在の役割で表示されます。 



就職の面接



チームリーダーの役割についての内部面接は、外部候補者への面接と大差なかったと言えます。これは、候補者の動機、経験、スキルの評価に役立つ質問に基づいていました。

この段階では、各候補者について次の基本事項を理解することが重要です。 



  1. 従業員がチームリーダーになるための動機は何ですか？お金、キャリア、挑戦、自己啓発、表彰？
  2. チームリードの役割に対する候補者の長所は何ですか。
  3. 何を汲み上げて変更する必要がありますか？ 
  4. この従業員をチームリーダーに任命する際のリスクは何ですか？たとえば、チームには前線が1つなく、すぐに交代ポジションを開く必要がありますか、それとも別の候補者が強い意欲を失うリスクがありますか？ 


すべてのインタビューは非公式の場で行われたため、会話はできる限りオープンで正直でした。



内部候補者を選択することは、外部候補者よりも簡単で信頼性が高くなります。すでにその人に精通しているため、彼の長所と短所がわかっており、パフォーマンス結果と、過去数年間のソフトスキルの評価があります。



結果の調整



結果のキャリブレーションは、利害関係者、製品チームが問題を提起し、ビジネスユニットの責任者の前に選択を防御するものです。選択のリスクを特定し、開発計画について話し合うには、なぜ1人または別の候補者が選ばれたのかを証明する必要があります。キャリブレーションを使用すると、チームを外から見て、ブラインドゾーンの瞬間を強調することができる人々の考えられる困難と視点を考慮することができます。 



人材プール



タレントプール、またはタレントプールは私たちの可能性です。チームリーダーになりたい、この方向に成長したい男性も含まれますが、現在、別のチームリーダーのチームがないか、特定のスキルを伸ばす必要があります。まず第一に、これらはチームリーダーによって選ばれなかった候補者ですが、この役割を担う意欲と可能性があります。 現在のチームで交代のポジションを開くとき、またはチームリーダーを必要とする新しいチームが現れたときに、チームリーダーを招待するのはタレントプール



からです。  タレントプールの開発プログラムはチームリーダーの場合と同じですが、トレーニング形式は、教材の自己開発と自己管理のために設計されています。 





開発プログラム



私たちの内部プログラムは6か月間設計されており、エンジニアがチームの線形管理者としての新しい役割に徐々に没頭することを考慮して設計されています。私たちは、人を役割にすばやく没頭させることと、仕事の最初の日からTLの役割で役立つ学習スキルと、完全に新しい役割およびチームとの新しい関係における燃え尽きからの保護との間のバランスを維持します。 



チーム開発プログラムは、基本管理と運用管理の2つの領域に分かれています。最初の方向は、経営基盤です。これは、どの組織のどのマネージャーにも役立ちます。2番目の方向は、私たちが使用するツールとアプローチです。これらも非常に人気がありますが、どこでも使用されているわけではなく、特定の会社に適応しているわけでもありません。

IDP（個別開発計画）



IDP、または個別の開発計画は、当社の個別の成長ツールです。基本的な管理スキルの開発プログラムが標準的な管理トピックとグループでのスキルの開発をカバーする場合、IDPは特定の従業員の成長分野を考慮して開発されます。

チュートリアルとヒント



結論として、私はあなたに役立つかもしれないいくつかのヒントとレッスンを共有したいと思います。 



私たちが直面している主な問題は、THBの役割のあいまいな理解です。さまざまな企業でのチームリーダーの役割は、さまざまなスキルと責任で構成されます。多くの場合、技術チームの役割と混同されます。したがって、チームリーダーのポジション、職務、責任、機会、および会社での役割を明確かつ体系的に説明し、チームとのミーティングでそれを伝えることが重要です。 



チームリーダーを選択するのは簡単ではありません。あなたはすでにチームの非公式のリーダーである最高の最高の人を選ぶ必要があります。チームリーダーによって選ばれなかった候補者の期待に正しく応え、強力な意欲低下を防ぎ、人材プールでの彼らの成長をサポートする必要があります。



チームが成熟していてスムーズに機能している場合は、チームの選択を自分で進めることができます。チームが集まり、私たちの参加なしに独立してチームリード選挙を行い、この特定の人物がこのポジションの優れた候補者である理由を証明した事例がありました。
