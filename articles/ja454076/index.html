DotNext 2019 Piter：小さなレポート

5月15日〜16日にDotNext 2019 Piterカンファレンスが開催されました。サンクトペテルブルクの天気は異常に暖かかった。町民は異常に青い空の中で奇妙な黄色のボールを信じられないほど見ましたが、それでも夏のコートに切り替える時期であるかどうか判断できませんでした。パークインプルコフスカヤホテルのコングレスセンターは、ロシアの子会社で最大のイベントへの訪問者のための扉を開きました。どうだったか思い出します。



テーマ


多くの人が、DotNextがさまざまな方向にますます寛容になってきていることにすでに気付いていると思います。今回は、建築、トレンド、そしてしばしば仕事に欠けている良い実践のための場所がありました。「しかし、ハードコアはどうですか？！」ハードコア愛好家は叫び、トマトを振った。私はあなたを安心させるために急いで、すべてが筋金入りの秩序であり、彼は常に支配的な賭けを持つでしょう。

報告書


レポートの完全なリストはまだここにあることを思い出させてください。しばらくすると、プレゼンテーションが同じ場所に表示されます。ビデオに関しては、フィードバックフォームに記入した人はすでにリンクを受け取っているはずです。残りは少し幸運ではなく、ビデオがパブリックドメインに投稿されるまで待つ必要があります。通常、これは11月6〜7日にモスクワでWTCコングレスセンターで開催される次の会議の近くで行われます。しかし、一部のレポートのトランスクリプトは夏にハブに表示されます。お見逃しなく！

TOP-10レポートを含む従来の投稿もあり、参加者の大多数の意見が得られますが、今のところ、私が個人的に覚えていることを簡単に説明します。

ミハイル・シェルバコフ（yu5k3）「デシリアライゼーションの脆弱性：過去、現在、未来」




セキュリティの専門家は常に断片的な商品です。そして、そのように深く関連性があり、ホールは眠りに落ちませんでした-そしてまったく火で午後に。ミハイルはデシリアライザの脆弱性のトピックを整理し、デシリアライゼーションを安全に書く方法に関するベストプラクティスを共有しました。

エフゲニーペシコフ（エペシュク）「.NETのマルチスレッド化：パフォーマンスが十分でない場合」




ユージンは、例外の報告とサンクトペテルブルク年前DotNextでデビューした直後に評価の最初の行を取り、彼はあなたに腸を提供してきました第三の連続した会議のために裁判所のテーブル。私はしばしば、低レベルの詳細を研究することの有用性についての論争の相手として行動します。実際、大多数にとって、これは動物園の鐘と笛です。しかし、Zhenyaが働いているContourのインフラストラクチャチームにとって、ハードコアは人生であり、人生はハードコアです。今回は、.NETでマルチスレッドを使用する機能について、理論とチームが決定した実際のケースを上手に組み合わせて話しました。

コンスタンティン・グストフ「マイクロサービスのDDD：複雑さ対複雑さ」




コンスタンティンは、昨年秋にDotNext 2018モスクワで始まったモノリスカットの話を続けています。大規模アプリケーションでのDDDの実用化に関する優れたレポート。これとは別に、過去6か月間でKostyaは講演者として力を注いできました。

ローマのネボリン（ネボロマン）「なぜアーキテクチャが機能するのか、そしてどのようにそれと共存するのか」




Vagif Abilov（ヴァジファビロフ）、マキシムアルシノフ（マルシノフ）「ビジネスオブジェクトの後の人生」




2つのレポートを並べて配置したのはなぜですか？機能的なパラダイムが活況を呈しています。より多くの機能が機能的な世界からC＃に登場します。そして、両方のレポートは、パラダイム間の橋を架けることについてです。F＃、特にHaskellに切り替えずに、関数型のアプローチがもたらす利点を、今ここでどのように使用するか。そして、これはスレッドの安全性とスケーラビリティだけでなく、たとえば、コードの表現力（hi、DDD）でもあります。私の意見では、Romanのレポートは、従来のジョークやトローリングを含む、より堅実で簡単であることがわかりました。しかし、自分自身と議論することは難しいため、パラダイムの対立はペアレポートでより明確に明らかにされます。そして、そのようなレポートを準備することは桁違いに困難ですが、VagifとMaximはクールであることがわかりました。

ディラン・ビーティ（スキル・マター）「コードのコスト」




ディラン・ビーティの基調講演を締めくくります。そしてそれはとても正しいです。魔術師としてのディランは、帽子からモザイクの断片を取り出します（写真の左側にあります）。ミーム、コード、ニュース、事実の形で。そして、あなたがそれを最も期待しないときに、op！マジック！パズルがビッグ哲学思想に追加されました。

エピローグ




DotNext 2019 Piterは終了しました。しかし、これはDotNext 2019モスクワに先立つ不可視のプロセスの開始も意味します。 2019年11月6〜7日にWTCコングレスセンターで開催されることを改めてお伝えします。

ワークショップで同僚に伝えたいことがあれば、今がレポートを提出する絶好の機会です。リンクをたどると、すべてがそこにあり、すべてがペイントされています。

そして、最初の話者の名前はすでに知られています。

Federico Lois（Corvalius）は、Corvalius R＆Dの共同創設者です。過去10年間、CPUとGPUのような特別なハードウェアの両方でアルゴリズムのパフォーマンスに取り組んできました。彼の経験は、銀行ソフトウェアのパフォーマンスの管理からデータベースエンジンの最適化にまで及びます。関心は、高度に分散されたシステム、大規模並列技術、低レベルの最適化、および高性能コンピューターグラフィックスの領域にあります。こことここに彼の過去のレポートのビデオがあります。

Jeff Prosise（Wintellect）は、（Jeffrey RichterおよびJohn Robbinsと共に）Wintellectの共同創設者です。彼はソフトウェア開発に関する9冊の本と数百の記事を書いており、現在はAzureでの作業、AIソリューションの構築、WintellectNOWの管理にほとんどの時間を費やしています。

Victor Gamov（Confluent）-多くのプログラマーに愛されているポッドキャスト「Debriefing Flight」の共同創設者。同時に、Apache Kafkaに基づくプラットフォームを開発するConfluentのDeveloper Advocateです。分散ストリーミングデータ処理システムの設計と開発を支援します。O'Reilly著「Enterprise Web Development」の共著者。

秋のDotNext 2019モスクワのチケットは既に販売されており、6月1日まで最低（アーリーバード）の価格が有効です。さらに、DotNext 2019 Piterのすべての参加者は、メールで個人的なプロモーションコードを受け取っているはずです。これにより、価格がさらに快適になります。秋にお会いしましょう！
