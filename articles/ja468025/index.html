Zimbra OSEでSNIを構成する方法

21世紀の初めには、IPv4アドレスのようなリソースが枯渇の危機に瀕しています。 2011年には、IANAは残りの5つのブロック/ 8をアドレス空間から地域のインターネットレジストラに割り当て、すでに2017年にアドレスを終了しました。 IPv4アドレスの壊滅的な不足への答えは、IPv6の出現だけでなく、1つのIPv4アドレスで膨大な数のWebサイトをホストすることを可能にするSNIテクノロジーにもありました。 SNIの本質は、この拡張機能により、顧客がハンドシェイク中に接続したいサイトの名前をサーバーに通知できることです。これにより、サーバーは複数の証明書を保存できます。つまり、複数のドメインが同じIPアドレスで動作できます。 SNIテクノロジーは、ビジネス向けのSaaSプロバイダーの間で特に人気が高まっています。これに必要なIPv4アドレスの数に関係なく、ほぼ無制限の数のドメインをホストできました。 Zimbra Collaboration Suite Open-Source EditionでSNIサポートを実装する方法を見てみましょう。



SNIは、現在サポートされているすべてのバージョンのZimbra OSEで動作します。 Zimbra Open-Sourceがマルチサーバーインフラストラクチャで実行されている場合、Zimbraプロキシサーバーがインストールされているノードで以下のすべてのアクションを実行する必要があります。さらに、IPv4アドレスに配置する各ドメインについて、一致する証明書とキーのペア、および認証局からの信頼できる証明書チェーンが必要です。 Zimbra OSEでSNIを構成する際のエラーの大部分の理由は、誤った証明書ファイルにあることに注意してください。したがって、直接インストールする前にすべてを注意深く確認することをお勧めします。

まず、SNIを正常に動作させるには、次のコマンドを入力する必要がありますzmprov mcf zimbraReverseProxySNIEnabled ZimbraプロキシのあるノードでTRUEにしてから、zmproxyctl restartコマンドを使用してプロキシサービスを  再起動します。

まず、ドメイン名を作成します。たとえば、company.ruドメインを取得し、ドメインが既に作成された後、Zimbra仮想ホストの名前と仮想IPアドレスを決定します。 Zimbra仮想ホストの名前は、ドメインにアクセスするためにユーザーがブラウザーの行に入力する必要がある名前に対応し、証明書に示されている名前と一致する必要があることに注意してください。たとえば、仮想ホスト名としてZimbra mail.company.ruを使用し、仮想IPv4アドレスとして1.2.3.4を使用します。

その後、コマンドzmprov md company.ru zimbraVirtualHostName mail.company.ru zimbraVirtualIPAddress 1.2.3.4を入力して、Zimbra仮想ホストを仮想IPアドレスにバインドします。サーバーがNATまたはファイアウォールの背後にある場合、ドメインへのすべての要求は、ローカルネットワーク上のアドレスではなく、バインドされている外部IPアドレスに送られるように注意する必要があることに注意してください。

すべてが完了した後は、ドメイン証明書をインストール用に検証および準備してからインストールするだけです。

ドメインの証明書が正しく発行された場合、証明書を含む3つのファイルが手元にあります。2つは認証局からの証明書のチェーンで、1つはドメインへの直接の証明書です。さらに、証明書を取得するために使用したキーを含むファイルが手元にあるはずです。別のフォルダー/tmp/company.ruを作成し、鍵と証明書を含むすべての既存のファイルをそこに置きます。結果は次のようになります。

ls /tmp/company.ru
company.ru.key
 company.ru.crt
 company.ru.root.crt
 company.ru.intermediate.crt

その後、cat company.ru.root.crt company.ru.intermediate.crt >> company.ru_ca.crtコマンドを使用して証明書チェーンを1つのファイルに結合し、/ opt /コマンドを使用して証明書に問題がないことを確認しますzimbra / bin / zmcertmgr verifycrt comm /tmp/company.ru/company.ru.key /tmp/company.ru/company.ru.crt /tmp/company.ru/company.ru_ca.crt。証明書とキーの検証が成功したら、インストールを続行できます。

インストールを開始するには、最初にドメイン証明書と認証局からの信頼できるチェーンを1つのファイルに結合します。また、cat company.ru.crt company.ru_ca.crt >> company.ru.bundleという形式の1つのコマンドを使用して実行され  ます。。 ：その後、あなたはすべての証明書とLDAPへの鍵を書き込むためにコマンドを実行する必要がありますは/ opt / Zimbra社/ libexecに/ zmdomaincertmgr savecrt company.ru company.ru.bundle company.ru.key、その後、使用して証明書をインストールするの/ opt /コマンドをzimbra / libexec / zmdomaincertmgr deploycrts。インストール後、company.ruドメインの証明書とキーは/opt/zimbra/conf/domaincerts/company.ruフォルダーに保存されます。 

異なるドメイン名を使用してこれらの手順を繰り返すことにより、同じIPアドレスを使用して、同じIPv4アドレスで数百のドメインの場所を取得できます。同時に、発行元のさまざまなセンターの証明書を問題なく使用できます。任意のブラウザで実行されたすべてのアクションが正しいことを確認できます。各仮想ホスト名について、独自のSSL証明書が表示されます。
