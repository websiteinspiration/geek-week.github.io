Windowsターミナルの最新プレビューバージョン-0.11の紹介

バージョン1.0のリリース前の最新のメジャーアップデートで戻ってきて嬉しいです！これからv1.0までの問題は、このバージョンのリビジョンになります。いつものように、ターミナルはMicrosoft StoreまたはGitHubのリリースページからダウンロードできます。猫の下で、更新の詳細を詳しく見ていきます。



ローカリゼーション


これはWindowsターミナルの最初のバージョンであり、そのユーザーインターフェイスは英語だけでなく、現在もバグの修正に取り組んでいますので、ご理解のうえよろしくお願いいたします。翻訳の問題を見つけた場合は、GitHubで報告してください。

設定を更新


設定に変更を加えたことに注目してください。ターミナルが自動的に新しいファイルを生成するように、settings.jsonファイルをアプリケーションフォルダーから移動することをお勧めします。その後、設定をコピーして戻すことができます。

Settings.json


profiles.jsonファイルは、settings.jsonに名前が変更されました。すでにターミナルがインストールされている場合、ファイル名は自動的に更新されます。

カスカディアコード


Cascadia Code フォントは、ターミナルによって自動的に生成されるすべてのプロファイルに対してデフォルトで設定されました。フォントを変更する場合は、fontFaceプロパティを任意のプロファイルまたはデフォルトセクションに追加します。

タンゴの配色


現在、端末にはTango DarkとTango Lightの配色が付属しています。





コピーアンドペースト


新しく生成されたsettings.jsonファイルでは、コピー操作と貼り付け操作は、それぞれctrl + cとctrl + vに関連付けられています。新しいキーバインディングはctrl + shift + cおよびctrl + shift + vと連動し、必要に応じて簡単に削除できます。さらに、ctrl + cとctrl + vを破棄することに決めた場合でも、古いキーの組み合わせはdefaults.jsonファイルに含まれています。

注：古いキーの組み合わせを「解放」する場合は、settings.jsonファイルでその値をunboundまたはnullに設定できます。

{ "command": "unbound", "keys": "ctrl+shift+c" },
{ "command": null, "keys": "ctrl+shift+v" }

コピーする


デフォルトのコピー動作をフォーマットなしでコピーするように変更しました。これを行うために、新しく生成されたsettings.jsonファイルに新しいグローバルcopyFormattingパラメータが追加されました。デフォルトでは、その値はfalseに設定されています。

注： defaults.jsonでは、copyFormattingがtrueに設定されています。

削除された設定


バージョン1.0のリリースに備えて、いくつかの古い設定が削除されました。これはおそらくあなたの作業プロセスに影響を与えたと理解していますが、将来これに頼る必要がないことを願っています。

「グローバル」名前空間これ
以降、ターミナルはJSONファイルのグローバルプロパティをサポートしなくなります。グローバルオブジェクト内に設定がある場合、それらは無視されます。この問題を解決するには、グローバルオブジェクトを削除し、グローバルパラメーターをJSONファイルのルートに移動します。

テーマはrequestedTheme
グローバルパラメータがいる名前が変更されたテーマを。これは、将来テーマをインストールするための柔軟性を提供するのに役立ちます。

"theme": "system"

一行は、コピーcopyTextWithoutNewlinesは、
コマンドとtrimWhitespaceの引数のためのコピーアクションが削除され、置き換えられている単一行の引数。この変更は、この機能がどのように機能するかをよりよく説明するのに役立ちます。この引数の値をtrueに設定すると、コピーアクションで、ターミナルからコピーされた内容が1行に貼り付けられます。

{ "command": { "action": "copy", "singleLine": true ), "keys": "ctrl+alt+c" }

特定のプロファイルを使用してタブを作成し
、特定のプロファイルで新しいタブを作成するための最初の反復はからキーバインディングのコマンドを持っていたnewTabProfile0にnewTabProfile8。これらのコマンドを削除して、より柔軟な代替手段を採用しました。

注：ターミナルでのインデックス作成は0から始まります。

{ "command": { "action": "newTab", "index": NUMBER }, "keys": "ctrl+shift+NUMBER" }

特定のタブへの
移動。タブナビゲーションの構造が、新しいタブを作成する場合と同じになりました。switchToTab0 - switchToTab8は、indexプロパティのために削除されました。

注：ターミナルでのインデックス作成は0から始まります。

{ "command": { "action": "switchToTab", "index": NUMBER }, "keys": "ctrl+alt+NUMBER" }

パネルのパーティション分割パネル
をパーティション分割する際により多くのオプションを提供するために、splitHorizo​​ntalとsplitVerticalは新しい形式に置き換えられました。さらに、分離の方向を指定しない場合、ターミナルはデフォルトで自動機能を使用して分割を使用します。これにより、最大表面積を確保するために杭打ちが実行されます。

{ "command": { "action": "splitPane", "split": "horizontal"}, "keys": "alt+shift+-" },
{ "command": { "action": "splitPane", "split": "vertical"}, "keys": "alt+shift+plus" }

パネル間の切り替え
最初は、moveFocusDown、moveFocusUp、moveFocusLeft、moveFocusRightを使用して、パネル間を移動できるキーバインディングを作成できます。それらを削除し、代わりに次の形式を追加しました。

{ "command": { "action": "moveFocus", "direction": "down" }, "keys": "alt+down" },
{ "command": { "action": "moveFocus", "direction": "up" }, "keys": "alt+up" },
{ "command": { "action": "moveFocus", "direction": "left" }, "keys": "alt+left" },
{ "command": { "action": "moveFocus", "direction": "right" }, "keys": "alt+right" }

サイズ変更同じキーバインディングアーキテクチャに従うために、パネルのサイズ変更のための古いコマンド（resizePaneDown、resizePaneUp、resizePaneLeft、resizePaneRight）も削除され、次の形式に置き換えられました。

{ "command": { "action": "resizePane", "direction": "down" }, "keys": "alt+shift+down" },
{ "command": { "action": "resizePane", "direction": "up" }, "keys": "alt+shift+up" },
{ "command": { "action": "resizePane", "direction": "left" }, "keys": "alt+shift+left" },
{ "command": { "action": "resizePane", "direction": "right" }, "keys": "alt+shift+right" }
