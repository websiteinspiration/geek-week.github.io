Yealink W80B IPセルラーDECTシステムを3CXに接続する

2019年9月、Yealinkは最新のYealink W80B IPセルラーIP-DECTシステムを発表しました。この記事では、その機能について簡単に説明し、3CX PBXを操作します。

この場をお借りして、皆様にメリークリスマスと明けましておめでとうございます！

Microcell DECTシステム


マイクロセルラーIP-DECTシステムは、1つの重要な機能で従来のDECT電話とは異なります-基地局（ハンドオーバー）とサブスクライバーのスタンバイモード（ローミング）間の加入者のエンドツーエンドスイッチングのサポート。このようなソリューションは、特定のニッチ、特に大規模な倉庫、ホテル、自動車販売店、産業、スーパーマーケット、および同様の企業で需要があります。ただちに、そのようなDECTシステムは企業のプロフェッショナルコミュニケーションシステムに属し、「携帯電話」で完全に置き換えることはできません（最大の節約が最前線に置かれない限り）。

       
Yealink W80Bは、単一のDECTネットワークで最大30の基地局をサポートし、一緒に最大100のDECT端末に対応できます。同時に、加入者の場所に関係なく、HD品質の通信が提供されます。

企業にDECTシステムを導入する前に、信号品質の予備的なフィールド測定を実行することをお勧めします。このため、Yealinkは、W80B測定ベースステーション、2つのW56H端子、端子を取り付けるための三脚、2つのUH33プロフェッショナルヘッドセットで構成される特別な測定キットを推奨しています。測定手順の詳細をお読みください。
  
W80Bベースステーションは、次の3つのモードで動作できます。

  • DM（DECT Manager）-中規模および大規模ネットワークでの操作モード。この場合、1つの専用データベースが制御ベースとしてのみ機能します（DECT機能なし）。基本モードで動作している最大30のWECTB DECTベースを接続できます。このようなネットワークは、最大100の加入者/ 100の同時コールをサポートします。
  • DM-Base-このモードでは、1つのベースステーションがDECTマネージャーとDECTベースの両方として機能します。この構成は小規模なネットワークで使用され、最大10のベース（ベースモード）、最大50のサブスクライバー/ 50の同時コールを接続できます。
  • ベース-DMまたはDECTベースに接続するマネージドベースモード。


マイクロセルシステム用DECT端子


Yealink W80Bには、アッパークラスとミドルクラスの2つのターミナルが用意されています。

Yealink w56h


大きくて透明な2.4インチディスプレイ、エレガントな工業デザイン、強力なバッテリー、およびプロ用のさまざまなアクセサリー（後で説明します）を備えたハンドセット。ハンドセットの機能：
 
  • 最長30時間の通話時間と最長400時間の待受時間
  • PCの標準USBポートまたはSIP-T29G、SIP-T46G、SIP-T48G電話のポートからの充電。10分の充電で、最大2時間通話できます。
  • 端子をベルトに取り付けるためのヒンジ付きクリップ。これは、ハンドセットが何らかの障害物に引っ掛かった場合に壊れるのではなく、ロールオーバーを可能にします。
  • 3.5mmジャック。ヘッドセットを接続します。


端末を完全に保護するわけではなく、困難な状況を想定していませんが、ハンドセットと一緒に追加の保護カバーを使用できます。

Yealink w53h


主に工業用に設計されたミドルクラスのチューブ。以前のモデルと同様に、HDオーディオを送信するためのCAT-iq2.0 DECT標準をサポートしています。チューブの特徴：

  • 1.8インチカラーディスプレイ
  • リチウムイオン電池と通話時間は最大18時間/待機時間は最大200時間。 
  • どんなサイズの手にも快適にフィットするコンパクト設計。
  •  ベルトクリップと3.5 mmジャック。ヘッドセットを接続します。


この携帯電話は、建設現場や産業などで使用するための完全な住宅保護を備えたプロ用ケースを提供します。

どちらのハンドセットも、ベースステーションから「無線」でのファームウェアの更新、3CXアドレス帳のダウンロード、通話に関するすべての操作（保留、転送、会議など）をサポートしています。
 

Yealink W80Bと3CX PBXの接続


Yealink W80B自動チューニングテンプレートは、3CX v16 Update 4でのみ表示されました。したがって、接続する前にこの更新をインストールしてください。また、データベースのファームウェアが最新であることを確認してください。 W80Bには現在最新のファームウェアが付属していますが、3CX PBXの [ファームウェア]タブのYealinkページでバージョンを確認することをお勧めします。ファームウェアを更新するには、[設定]> [アップグレード]> [ファームウェアのアップグレード]セクションのデータベースインターフェイス（管理者のユーザー名とパスワード）にアクセスします。



個別のDECT端子を更新する必要がないことに注意してください。各ハンドセットは、ベースステーションに接続した直後に「無線で」アップデートの受信を開始します。ただし、同じセクションで（データベースに接続した後）手動で更新できます。

新しいファームウェアをインストールした後、データベースをリセットすることをお勧めします。これを行うには、すべてのインジケータが緑色にゆっくり点滅し始めるまで、ベースのボタンを20秒間押し続けます。インジケーターの点滅が止まるまでボタンを押し続けてから放します-ベースがリセットされます。



データベース操作モードの設定


次に、ベースステーションの適切な動作モードを設定する必要があります。小規模なネットワークがあり、これがネットワークの最初のベースであるため、[ ベースモード]セクションでDM-Baseハイブリッドモードを選択します。次に[OK]をクリックし、データベースが再起動するまで待ちます。再起動後、インターフェースに移動します。DECTマネージャーの多くの設定が表示されます。ただし、今は必要ありません。ベースは自動的に構成されます。  

PBX 3CXの基本構成


前述のように、Yealink W80B接続は、3CXに付属している特別なテンプレートのおかげで自動化されています。

  1. ベースMACアドレスを見つけてコピーし、FXS / DECTデバイスの 3CXセクションに移動して、[ + Add FXS / DECT ]ボタンをクリックします   。
  2. 電話のメーカーとモデルを選択します。
  3. MACを挿入し、[OK]をクリックします。


     
開いたタブで、ベースの接続方法を指定します—ローカルエリアネットワーク、3CX SBC経由のリモート接続、または直接リモートSIP接続。私たちの場合は、ローカルネットワークが使用されているので、ベースと3CXサーバーは同じネットワーク上にあります。



  • 自動構成リンクをコピーして、データベースインターフェースに貼り付けます。
  • 接続要求を受け入れるサーバーネットワークインターフェイスを選択します（サーバーに複数のネットワークインターフェイスがある場合）。
  • また、3CXによって生成された新しい基本インターフェースのパスワードをコミットします。自動調整後、デフォルトのパスワードadminが置き換えられます。
  • ハンドセットはHDサウンドをサポートしているため、最初にG722ワイドバンドコーデックをインストールして、HD品質のVoIPトラフィックを送信できます。         

次に、[ 拡張機能 ]タブに移動して、パイプに割り当てるユーザーを選択します。前述のように、DM-Baseモードでは、最大50人の3CXユーザーを選択できます。
 


[OK]をクリックすると、データベース構成ファイルが自動的に生成されます。これは後でアップロードします。

3CX SBCまたはSTUNを介したベースのリモート接続（SIPを介した直接接続）には追加情報が必要であり、いくつかの機能があります。

3CX SBCによる接続


この場合、リモートネットワーク上のSBCサーバーのローカルIPアドレスとSBCポート（デフォルトでは5060）を追加で指定する必要があります。最初にリモートネットワークに3CX SBCをインストールして設定する必要があることに注意してください。
  


直接SIP接続（STUNサーバー）


この場合、リモートW80Bで構成されるSIPポートとRTPポートの範囲を指定する必要があります。これらのポートは、リモートオフィスのNATルーターのベースのIPアドレスに転送する必要があります。

すべてのDECT端子が正しく機能するためには、W80Bベースに600ポートの範囲を割り当てる必要があることに注意してください。



また、端末に割り当てられた内線の設定で、[ PBXオプションによるプロキシオーディオストリーム ]オプションを有効にする必要があります。


        

データベースでの構成ファイルのリンクの指定


上記では、3CXでベースをセットアップするときに、自動チューニングリンクとW80Bインターフェイスにアクセスするための新しいパスワードを修正しました。ここに行く、データベースインタフェースに行く設定>自動引当金>サーバーのURL、リンク、クリック貼り付けることを確認した後、今、自動プロビジョニング。



端末の登録


ベースを構成した後、必要な数の端末をベースに接続します。これを行うには、[ ハンドセットとアカウント]> [ハンドセットの登録]セクションに移動し、SIPアカウントを編集するためのアイコンをクリックします。



次に、[ Start Register Handset ]をクリックします。 


そして、ハンドセット自体で、イージーペアリングボタンを押します。



また、ハンドセットメニューの[Registration]> [Base 1]に移動してPIN 0000を入力することもできます。登録が

成功すると、ハンドセットは「無線で」ファームウェアの更新を開始します。これにはかなり時間がかかります。

補完されました！追加の基地局をDECTマネージャーに接続する


記事の執筆中に、2つ目のW80Bベースステーションが届きました。ここでは、いわゆるDECTクラスターに接続する方法について説明します。

DECTクラスターと同期レベル


DECTクラスタは、ハンドオーバー（アクティブなサブスクライバーのエンドツーエンドの再接続）、ローミング（基地局間でのスタンバイモードの端末の転送）、およびロードバランシング（サブスクライバーを過負荷のステーションから空きリソースのある最も近いステーションに切り替える）をサポートするベースステーションのグループです。

基地局が無線信号の到達範囲内にある場合、それらは1つのクラスターに結合されます。

ただし、たとえば、2つのオブジェクトが互いにリモートにあり、その間にDECT接続がない場合は、2つのクラスターを作成する必要があります。それらの間の同期とハンドオーバーはサポートされていません。
   
クラスター内の最初の（または中央の）DECTデータベースは、クラスター内の残りのデータベースのシンクロナイザーであり、最高レベルの同期レベル1を持ちます。クラスターに追加されると、新しいデータベースにはより低いレベルの同期が割り当てられます。クラスターに同じレベルの同期を持つ複数のベースがある場合、新しいベースは信号が最も良いベースと同期されます。そうしないと、新しいベースは同期に参加できません。



すぐに、シンクロナイザベースはネットワークの中心にあり、同期の第2レベルのベースと直接接続している必要があることに注意してください。



次に、同期レベルによる（企業内の）データベースの正しい分散のいくつかの例を示します。



DECTクラスターへの新しいW80Bベースの追加


最初は、新しいベースはベースモードで構成されています。ネットワークに接続し、ユーザー名/パスワードadmin / adminでWebインターフェイスに移動し、[ベースモード]セクションでDECTマネージャーのIPアドレスを指定します。



次に、DECTマネージャーに移動し、[ベースステーションの登録]セクションで右側のボタンをクリックして、新しいDECTベースの接続を完了します。



開いたウィンドウで、このデータベースの名前、クラスター番号、同期レベルを指定します（上記を参照）。ベースを既存のクラスター1に接続し、同期レベル2を割り当てます（レベル1には最初のベースがあります）。

 
新しいデータベースがクラスターに正常に追加されました。



これで、新しいデータベースのWebインターフェースに戻ることができます（パスワードはすでにadminにあります-DECTマネージャーのパスワードと一致します）。ステータスでは、DECTマネージャーに接続するためのパラメーターが表示されます。

 
そして、DECTマネージャー自体では、両方のベースが表示されます。
