Goにある要素


ガベージコレクション


このメカニズムは、自動メモリ管理を実行するプログラミング言語に存在します。ガベージコレクションまたはメモリ管理を理解するには、まずメモリ自体のしくみを理解する必要があります。プログラミング言語を使用する場合、コンパイラーは、変数を作成したり、配列をループしたりするなど、データを格納するためにシステム内のさまざまなメモリー空間を割り当てます。メモリーの割り当てと再配分を実装して、プログラムをメモリーでより効率的に動作させる必要があります。

Cのような言語では、メモリは手動で管理されます。Cに精通している場合は、システムにメモリを動的に割り当てるmalloc関数があることを知っています。JavaScriptやPythonなどの高級言語では、この割り当てはガベージコレクターと呼ばれるプログラムによって自動的に実行されます。名前が示すように、その役割は、メモリを管理し、必要なときにスペースを割り当て、割り当てられたメモリが不要になったときに解放することです。Goはガベージコレクションを実装しているため、プログラマーはメモリ管理について心配する必要はありません。

パッケージ


パッケージ化とは、すべてのソースコードとリソースを組み合わせて、パッケージと呼ばれる単一のオブジェクトにすることです。ソフトウェアパッケージは、簡単なインストール、共有、追加、デバッグなど、多くの点で便利です。Goには、ドキュメント、バイナリ、ソースコードファイルを統合する統合パッケージシステムがあります。パッケージ化の目的は、ソースコードを手動でコピーせずに、ソフトウェアで他のソフトウェアプロジェクトを使用することです。

ファーストクラスの機能


ファーストクラスの関数は、他の変数と同様に処理できる関数です。つまり、割り当て、返却、エクスポート、パラメータとして渡すなどができます。Goで記述された次のスニペットを見てください。文字列を出力する関数がhello world first class function変数に割り当てられますa。変数aはメモリ内の実際の値として動作しますが()、最後に追加して関数として呼び出すこともできます。aの値が他の変数と同じように表示されていることもわかります。これがファーストクラスの機能の基本概念です。

package main

import (  
    "fmt"
)

func main() {  
    a := func() {
        fmt.Println("hello world first class function")
    }
    a()
    fmt.Printf("%T", a)
}

字句スコープ


プログラムのスコープは、変数または関数の定義が有効であるブロックまたはスコープに似ています。たとえば、関数内で宣言された変数は、この関数ブロック内、つまり中括弧の間でのみその値を保持します{}。この関数ブロックの外でそのような変数の値にアクセスしようとすると、プログラムはそれを見つけることができません。

package main

import “fmt”

func main() {

    {
        v := 1
        {
            fmt.Println(v)
        }
        fmt.Println(v)
    }

    fmt.Println(v)
    // “undefined: v” compilation error

}

指定されたフラグメントには、4つの可視領域があります。1つ目はグローバル領域、2つ目は関数main ()、3つ目は関数内の最初のブロックmain、4つ目はfmt.Println初めて呼び出される領域です。3つのうち、Println後者はコンパイルエラーを生成します。これは、変数の定義がvエリア3および4でのみ使用できるためです。とPrintln呼ばれるときvパラメータとして渡されると、プログラムはまず現在の領域でその定義を探します。見つからない場合は、親のスコープの外に移動し、定義が見つかるまでこれを続けます。これは字句スコープです。プログラムは、変数と関数の定義を、それらが使用/呼び出されている領域から検索し始め、次に移動します。最後のfmt.Printlnプログラムではv、現在の領域または親領域で定義が見つからなかったため、コンパイルエラーがスローされました。

システムコールインターフェース


Goには、オペレーティングシステムで使用可能なシステムコールへの参照として機能するシステムコールインターフェイスが付属しています。たとえば、ファイルのオープンと読み取り、入出力など。APIの関数呼び出しをインターセプトし、オペレーティングシステムの必要なシステムコールを呼び出します。

不変文字列


Goの構文はCに似ていますが、UTF-8でエンコードされた不変の行の形式でそれよりも優れています。つまり、Goで記述されたプログラムは、多くの言語と文字を使用して文字列を形成できます。簡単に言うと、文字列は、プログラミング言語の文字の組み合わせ/配列/リストです。文字列は文字を組み合わせて構成されているため、その構成を変更できます。文字の追加、削除、移動などが可能です。宣言後の文字列の不変性を保証する方法を検討します。不変文字列の概念は新しいものではなく、Python文字列インスタンスではオブジェクトを変更できません。JavaScriptにも不変文字列があり、Rubyには凍結文字列リテラルが追加されています2.3で。しかし、それにもかかわらず、C ++、PHP、Perlなどの多くの一般的な言語には不変の文字列がありません。
