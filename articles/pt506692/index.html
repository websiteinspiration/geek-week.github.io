Expressar


Estrutura de desenvolvimento web rápida, flexível, minimalista e fácil de aprender.

Ótimo para

  • desenvolvendo aplicativos usando a pilha MEAN (MongoDB, Express, Angular, Node) ou MERN (React em vez de Angular)
  • manipulação fácil de vários tipos de solicitações, como GET, PUT, POST e DELETE
  • implantação rápida de aplicativos de uma página, várias páginas e híbridos

Cheerio


Projetado para processar ("analisar") a marcação, por exemplo, HTML, fornece uma interface para manipular ("atravessar") a estrutura final dos dados.

Ótimo para

  • criando scanners da web ("raspadores")
  • sintaxe e kit de ferramentas fáceis e intuitivos

Nodemailer


Projetado para enviar e-mails.

	const nodemailer = require('nodemailer')

	const testAccount = await nodemailer.createTestAccount()

	const transporter = nodemailer.createTransport({
		host: 'smtp.ethereal.email',
		port: 587,
		secure: false,
		auth: {
			user: testAccount.user,
			pass: testAccount.pass
		}
	})

	const info = await transporter.sendMail({
		from: '"John Smith" <smith@example.com>',
		to: 'foo@example.com, bar@example.com',
		subject: 'hello',
		text: 'hello crazy world',
		html: '<strong>hello crazy world</strong>'
	})

Ótimo para

  • enviando mensagens via SMTP

Socket.io


Projetado para configurar uma conexão bidirecional baseada em eventos em tempo real.

	const server = require('http').createServer()
	const io = require('socket.io')(server)
	io.on('connection', client => {
		client.on('event', data => { ... })
		client.on('disconnect', () => { ... })
	})
	server.listen(3000)

Ótimo para

  • implementação de análises ao vivo, fluxo de dados binários, mensagens, colaboração com documentos

Morgan


É um middleware para registrar solicitações.

	morgan
	(':method:url :status :res[content-length] - :response-time ms')

	---

	const express = require('express')

	const morgan = require('morgan')

	const app = express()

	app.use(morgan('combined'))

	app.get('/', (req, res) => res.send('hello crazy world'))

Ótimo para

  • gravar solicitações no console, arquivo, banco de dados
  • depuração e "log"

Erros de HTTP


Projetado para criar erros específicos para testar aplicativos no Express, Koa, Connect, etc.

	app.use((req, res, next) => {

		next()
	})

Ótimo para

Analisador de corpo


É um middleware para analisar dados.

Analisa solicitações recebidas (req.body) antes de processá-las.

	const express = require('express')
	const bodyParser = require('body-parser')

	const app = express()

	//  application/x-www-form-urlencoded
	app.use(bodyParser.urlencoded({ extended: false }))

	//  application/json
	app.use(bodyParser.json())

Ótimo para

  • interpretações do corpo da consulta
  • contém muitas funções úteis (inflar, digitar, verificar etc.)

Sequelizar


É uma tecnologia baseada em Promise para mapeamento objeto-relacional do Postgres, MySQL, MariaDB, SQLite e Microsoft SQL Server.

Ele suporta transações seguras (confiáveis), troca de dados, carregamento energético e atrasado, replicação de dados, etc.

	const sequelize = new Sequelize(
		'database', 'username', 'password', {
			host: 'localhost',
			dialect: /*   'mysql' | 'mariadb' | 'postgres' | 'mssql' */
		}
	)

Passaporte


É um middleware compatível com Express para autenticação.

A principal tarefa é autenticar solicitações usando um conjunto de plugins - estratégias.

	passport.use(new LocalStrategy(
		(username, password, done) => {
			User.findOne({ username: username }, (err, user) => {
				if(err) { return done(err) }

				if(!user) { return done(null, false) }

				if(!user.verifyPassword(password)) { return done(null, false) }

				return done(null, user)
			})
		}
	))

Ótimo para

  • autenticação
  • integração com OAuth e OpenID (faça login com sua conta do Facebook, Twitter etc.)

810 mil

Dotenv


É um módulo de dependência zero (ponto) para carregar variáveis ​​do arquivo .env no process.env.

Mantém as variáveis ​​separadas do código.

	require('dotenv').config()

	const db = require('db')

	db.connect({
		host: process.env.DB_HOST,
		username: process.env.DB_USER,
		password: process.env.DB_PASS
	})

	//  .env
	DB_HOST = localhost
	DB_USER = root
	DB_PASS=12345

Ótimo para

  • carregando variáveis ​​de ambiente como AWS, nomes de usuário sql, senhas necessárias para implantar aplicativos, conectando-se a outras ferramentas
  • armazenamento das configurações do ambiente separadamente do código

Multer


Middleware para processar dados de várias partes / formulário - carregamento de imagens.

	const express = require('express')

	const multer = require('multer')

	const upload = multer({ desc: 'uploads/' })

	const app = express()

	app.post('/profile', upload.single('avatar'), (req, res, next) => {
		// req.file -   'avatar'
		// req.body     
	})

	app.post('/photos/upload', upload.array('photos', 12), (req, res, next) => {
		// req.files -    'photos'
		// req.body      
	})

Ótimo para

  • download rápido de arquivos multipartes / dados de formulário

CORS


Middleware Connect / Express projetado para definir regras de política de origem comum.

	const express = require('express')

	const cors = require('cors')

	const app = express()

	app.use(cors())

	app.get('/products/:id', (req, res, next) => {
		res.json({ msg: '    !' })
	})

	app.listen(80, () => {
		console.log('ok')
	})

Ótimo para

  • Tarefas relacionadas ao CORS
