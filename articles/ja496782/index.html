平方根の計算を拒否した方法



デジタル信号処理では非常に多くの場合、ベクトルの長さを計算する必要があります。これは通常、式A = SQRT（X ^ 2 + Y ^ 2）によって行われます。ここで値を二乗することは難しくありませんが、特にマイクロコントローラーの場合、平方根を計算する操作は単純な操作ではありません。また、根の計算アルゴリズムは時間とともに安定せず、そのような計算が多いアルゴリズムの場合、計算に必要な時間を予測することが難しくなります。

私もそのような問題に直面しました。根を計算する手順をどのように拒否したかについては、以下をお読みください。

最初に、4つのベクトルを考えます。それらは異なる四分の一にあります。



すべてを4分の1に減らします。これを行うには、Y座標を取り、それが負の場合は正に変換します。それがすでにポジティブである場合、すべてを変更せずに残します。

実際、私は絶対座標と符号なし整数で示された値を持っているので、モジュールを計算します。

Y> 2047の場合、Y = Y-2047
Y <= 2047の場合、Y = 2047-Y。

ここで2047は仮想ゼロです。

何が起きたのか見てみましょう。



座標Xについても同じことを行います

。X> 2047の場合、X = X-2047
X <= 2047の場合、X = 2047-X

ちなみに、2つの12ビット値から2つの11ビット値が存在することは注目に値します。



現在、すべてのベクトルは第1四半期にありますが、私たちが探しているその長さは変わっていません。

変換の次の段階：「45度の軸を中心にねじる」。

ベクトルの角度は0〜90度ですが、ベクトルの座標がY> Xの場合、ベクトルの角度は45〜90度の範囲にあり、Y <Xの場合、ベクトルの角度は45度未満です。 X = Yの場合、ベクトルの角度は45度です。

ベクトルを45度に減らします。

Y> Xの場合、TEMP = Y、Y = X、X = TEMP（XとYを入れ替えます）。
Y <= Xの場合、何も行いません。

何が起こったのか見てみましょう。すべてのベクトルは、0〜45度の角度範囲にあります。



この段階で、角度を22.5度確認し、再度展開するというアイデアがありました。そして、チェックを行い、12.25度ねじります。同様に、ベクトルがX軸上に「横たわる」まで、その長さはX座標によって決定され、Yはゼロになります。

しかし、私はこの方法をとらず、別の方法で賭けをしましたが、後で判明したので、負けませんでした。

YをXで割り、この商から逆正接をとると、角度が得られます。 Xをこの角度の余弦で割ると、ベクトルの長さがわかります。

したがって、ベクトルの長さはA = X / COS（ATAN（Y / X））

です。逆正接のテーブルから角度を取得し、次にコサインのテーブルから係数を取得する必要があると思ったとき、それは難しいように見えました。

しかし、何も起こりませんでした。私はあなたがすぐにY / X比がセルをアドレスするテーブルを作ることができると決めました、そしてそれはベクトルの長さを得るためにXが分割されなければならない係数です。

A = X / K; ここで、K = COS（ATAN（Y / X））ですが

、乗算は除算よりも簡単です。したがって、

A = X * K; ここで、K = 1 / COS（ATAN（Y / X））

すべての（Y / X）のすべてのKを事前に計算する必要があることは明らかです。

計算されたテーブルには2048の値が含まれ、それぞれ16ビットです。これは4kBのデータです。

最低値は0x800hで、20​​48を意味します。最も古い値は0xB50で、2896を意味します。2896
/ 2048 = 1.414は、1 / cos（45）に対応します。

配列のインデックスを取得するには、Yに2048を掛けてXで割る必要があります。

結果のインデックスは配列のセルを指し、このセルから値を抽出し、Xを掛けて2048で割ります。結果はベクトルの長さです。

使用するプラットフォームに乗算および/または除算コマンドがない場合、2048による除算および乗算はシフトによって実行され、別の除算および乗算は特別なコマンドまたはアルゴリズムによって実行されることに注意したい。

私の場合、除算と乗算のコマンドを備えたSTM32F103マイクロコントローラーが使用されました。

アルゴリズムを要約します。

  1. Yで標識を確認し、必要に応じて方向を変えます。
  2. Xのサインを確認し、必要に応じてオンにします。
  3. XとYを比較し、必要に応じて場所を入れ替えます。
  4. Xの0を確認します。
  5. Yに2048をシフトで乗算します。
  6. XによるY除算。
  7. 結果の値を配列のアドレスに追加し、値Kを読み取ります。
  8. XにKを掛けます。
  9. Xをシフトで2048で割ります。

SQRT（17901）= 133（整数に切り捨て）

アルゴリズム：

X = 2047-45 = 2002、Y = 2047-126 = 1921

条項1：2002> 2047？ ->いいえ。 X = 2047-2002 = 45
ポイント2：1921> 2047？ ->いいえ。 Y = 2047-1921 = 126
ポイント3：126> 45->はい。スワップX = 126、Y = 45
ポイント4：X = 0->いいえ。
パラグラフ5を続ける：Y = Y * 2048、Y = 45 * 2048 = 92160
ポイント6：I = Y / X、I = 92160/126 = 731
ポイント7：テーブルには8つの値の255行があり、731番目の値を探しています。
