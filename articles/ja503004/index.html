Fluid Frameworkの更新


1  年前にBuild 2019で  導入された Fluid Framework Interactive Web Collaboration Platformは、   オープンソースソフトウェアになりつつあります。また、間もなく、Fluid Frameworkの機能の一部は開発者だけでなくエンドユーザーも利用できるようになります。そのコンポーネントと作業環境は、Office.comとWebバージョンのOutlookに表示されます。

Microsoft 365での最初のFluid Framework統合により、複数のユーザーが同時にドキュメントに変更を加え、接続されたコンポーネントを作成して、アプリケーションで即座に、そして非常に簡単に共有することができます。たとえば、表、グラフ、タスクリストをWebバージョンのOutlookに貼り付けて、販売データ、プロジェクトタスク、および調査レポートを常に更新して、最新の変更を反映させることができます。 Office.comでは、ドキュメントのアクティビティストリーム、推奨事項のリスト、@メンションに含まれるFluid Frameworkワークスペースを作成または管理したり、Office.comで検索したりできます。また、Fluid Frameworkのコンポーネントは軽量であるため、編集は即座に適用されます。

流動的なネットワーク環境では、アプリケーションを即座にコラボレーションツールに変えることができます。低レイテンシの同期データ構造と、エンドポイントを接続するためのスイッチングサービスが含まれます。静的データ構造をFluidデータ構造に置き換えると、アプリケーションは即座にリアルタイムのコラボレーションのサポートを開始します。

開発者の多文化でオープンでアクティブなコミュニティを作成する場合、Fluid Frameworkの可能性を最大限に引き出すことが可能です。このため、MicrosoftはFluid Frameworkをオープンソース環境に変え、開発者が独自のアプリケーションでコアFluid Frameworkを活用できるようにします。追加のドキュメントとツールをリリースし、Fluid Frameworkの作成とリリースについてマイクロソフトと協力するよう開発者に依頼します。

Microsoft Edge：開発者向けの改善


Microsoft Edgeは、消費者、開発者、および商用顧客のワークフローを改善する一連の新機能を導入しています。

まず第一に、これはサイドバーを使用して検索するためのより速く、より文脈に即した方法です。単語またはフレーズを選択して右クリックし、コンテキストメニューの[サイドバーで検索]を選択するだけです。結果はブラウザウィンドウの右側のパネルに表示されるため、ユーザーは元のコンテンツのコンテキストを失うことはありません。

開発者向けに、アクセシビリティ、手書き、スクロール、ローカリゼーションの改善など、Chromiumプロジェクトに貢献しています。Webview 2.0のプレビューバージョンは、.NETおよびUWPでの開発にも使用できます。また、プログレッシブWebアプリ（PWA）（プレビュー）の新しい機能強化のおかげで、これらのアプリケーションはネイティブであり、共有や通知などのユーザーインターフェイスがより自然になっています。

Project Reunionを起動する-Windowsアプリケーションを開発する新しい方法
