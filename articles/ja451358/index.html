Scalaの暗黙的なパラメーターと変換

Habrに関するこれまでの記事を読み終えたため、tytsとtytsは、Scalaの暗黙の機能をすばやく理解することができませんでした。一緒に考えてみましょう。


そのため、Scalaの暗黙関数を使用すると、メソッドの呼び出しや変数への明示的な参照を回避でき、代わりにコンパイラーが必要な暗黙データ自体を見つけることができます。

たとえば、FloatからInt（FloatToInt）に変換する関数を記述して、この関数を明示的に呼び出す代わりに、コンパイラーが暗黙的にそれを行うことができます。

def double(value: Int) = value * 2
implicit def FloatToInt(value: Float):Int = value.toInt
println(double(2.5F))

混乱していますか？すべてを順番に取得しましょう。

したがって、今日は次の2つの暗黙のカテゴリを見ていきます。

  • 暗黙的なパラメーター（値）。それらは、暗黙的に宣言されたコンパイラー値によって自動的に渡されます。
  • 暗黙的な変換 予期されるパラメーターのタイプが入力パラメーターと一致しない場合、コンパイラーは暗黙的にマークされたスコープ内で、この状況で必要な予期されるパラメーターと入力パラメーターを使用してメソッドを検索し、自動的に渡します。

暗黙的なパラメーター


暗黙的パラメーターは、暗黙的なキーワードが前に付いた関数パラメーターです。つまり、関数が呼び出されたときに値が渡されなかった場合、コンパイラーは個人的に暗黙のパラメーターを検索し、それを渡してくれます。

たとえば、暗黙のパラメーター値を入力として受け取り、それを2倍にする関数：

def double(implicit value: Int) = value * 2

暗黙的なパラメータがないと、エラーで失敗します。

def double(implicit value: Int) = value * 2
println(double) // error: could not find implicit value for parameter value

明示的なパラメーターが渡されると、機能します。

def double(implicit value: Int) = value * 2
println(double(3))  // 6

暗黙のパラメーターを使用すると、次のようになります。

def double(implicit value: Int) = value * 2
implicit val multiplier = 2
println(double) // 4

ただし、注意が必要です。

def double(implicit value: Int) = value * 2
implicit val multiplier = 2
implicit val multiplier2 = 1 
println(double) // error: ambiguous implicit values

そして最後に、暗黙のパラメーターとしてdefを渡す例：

def double(implicit value: Int) = value * 2
val cappucinoLarge: Boolean = false
implicit def cappucinoPrice: Int = if (cappucinoLarge) 200 else 100
println(double) // 200

それら。最後に私たちは成功します
double(cappucinoPrice())

構文メモ：


def func1(implicit value1: Int)                          // value1 
def func2(implicit value1: Int, value2: Int)             // value1  value2 
def func3(value1: Int, implicit value2: Int)             //  
def func4(value1: Int)(implicit value2: Int)             //  value2 
def func5(implicit value1: Int)(value2: Int)             //  
def func6(implicit value1: Int)(implicit value2: Int)    //

暗黙的な変換


FloatからIntの例に戻る：

def double(value: Int) = value * 2
implicit def FloatToInt(value: Float):Int = value.toInt
println(double(2.5F))

doubleを呼び出すと、予期されるパラメーター（Int）と入力パラメーター（Float）の型が一致しません。したがって、コンパイラーは、暗黙的であり、この状況で必要な予期されるパラメーター（Int）および入力パラメーター（Float）がマークされたスコープ内のメソッドを探します。FloatToIntを検索し、変換を実行して、目的の値をさらにdoubleに渡します。

今、私はそれがより明確になったことを願っています。Scalaの習得に成功しました！
