これは改善のほんの一部です。試用するには、WebStorm 2019.3の30日間トライアルバージョンをWebサイトからダウンロードするか、ツールボックスアプリを使用してください。フルバージョンは、WebStormまたはAll Products Packのアクティブなサブスクリプションの保有者、および無料の学生とオープンソースプロジェクトの開発者が使用できます。

次に、各改善点をより詳細に見て、製品の他の重要な変更について説明します。


最も一般的なWebStormパフォーマンスの問題の1つは、起動が遅いことでした。この不便さを修正するために、最後の2つのリリースサイクルを費やしましたが、多くのユーザーから不満がありました。その結果、WebStorm 2019.3はウェルカム画面を表示し、以前にインデックス付けされたプロジェクトを以前のバージョンよりもはるかに速く開きます。

「速く」とはどういう意味ですか？異なるオペレーティングシステムでのWebStorm 2019.3の起動時間を測定し、これらの数値を以前の2つのバージョン（2019.1および2019.2）のパフォーマンスと比較しました。正確な結果を得るために、どこでも同じプロジェクトを使用しました。create -react-appを使用して作成された、エディターでapp.jsファイルを開いたReactアプリケーションです。取得した結果は次のとおりです。



起動時間は以前のバージョンと比較して平均で約20％短縮されましたが、使用したネットワーク設定、ワークスペースのサイズ、コンピューターの古さ、およびウイルス対策の構成によって結果は異なる場合があります。

私たちは栄光に頼るつもりはありません-バージョン2020.1では、インデックス作成プロジェクトの最適化など、この領域でさらに重要な改善がいくつか見られます。

Vue.jsサポートの大きな改善


ローンチの加速に加えて、Vue.jsのサポートが大幅に改善され、WebStormユーザーがこのフレームワークでアプリケーションをすばやく作成できるようになりました。

まず、Vue.jsプロジェクトでのコード補完が大幅に改善されました。まず、WebStormがスロット名を自動的に入力できるようになりました。これは、プロジェクトコンポーネントとVuetify、Quasar、およびBootstrapVueコンポーネントライブラリの両方で定義されたスロット名に対して機能します。次に、WebStormは、異なるファイルにあるコンポーネントの部分間の関係を理解し​​、同じvueファイルで定義されていなくても、入力パラメーター、データ、およびメソッドのオートコンプリートオプションを提供します。



最後に、Vue.jsで定義されたコンポーネントとディレクティブのドキュメント、およびいくつかの一般的なコンポーネントライブラリをWebStormで表示できます。これを行うには、コード内のコンポーネントまたはディレクティブの名前にカーソルを合わせ、F1を押します。



新しいキャラクター名のヒント


JavaScriptまたはTypeScriptファイルに変数名またはパラメーター名を追加し始めるとすぐに、WebStormは、プロジェクトで定義されたクラス、インターフェース、およびタイプエイリアスの名前、および使用する標準ライブラリと依存関係に基づいて、いくつかの適切なオプションを通知します。



新しい変数、関数、クラス、およびインターフェースについては、同じスコープで既に使用されているがまだ定義されておらず、未解決としてマークされていない名前のヒントも表示されます。

たとえば、新しい関数を使用したい場合は、最初にそれを呼び出し、次に入力functionして追加し、提案された名前を表示できます。もちろん、使用場所にカーソルを合わせてAlt-Enterを押すこともできます「クイックフィックス」作成関数を適用します。



一般的に使用されるテクノロジーの最新バージョンのサポート


WebStorm 2019.3には、オプションのチェーン、無効な合体、アサーションシグネチャなど、TypeScript 3.7のコア機能の組み込みサポートがあり、より効率的に作業できます。しかしそれだけではありません。次のYarn 2リリースで利用可能なPlug'n'Play機能のサポートも追加しました。

CDNを介して追加されたCSSライブラリのコード補完


CDNを介してHTMLファイルで接続されたBootstrapまたは別のCSSライブラリを使用していますか？これで、WebStormは、プロジェクトにソースを追加する必要なしに、使用しているライブラリのクラス名のオートコンプリートオプションを提供します。この機能を有効にするには、HTMLファイルのCDNリンクにカーソルを合わせ、Alt-Enterキーを押して、[ ライブラリをダウンロード]を選択します。



ツールと構成を実行するための1つのアクション


新しいRun Anythingアクションを使用すると、npmやYarnなどの構成やツールをすばやく起動したり、最近のプロジェクトを開いたりできます。使用するには、Ctrlキーを 2回押すか、ナビゲーションバーの右側にあるアイコンをクリックします。



すべての新しいプロジェクトにグローバルファイルビューアを接続する


プロジェクトでPrettierなどのツールを使用し、ファイルビューアを使用してそれらを構成する場合は、既存の設定をすべての新しいプロジェクトに自動的に適用したい場合があります。WebStorm 2019.3では、デフォルトでグローバルファイルビューアを有効にすることでこれを実現できます。それらを有効にするには、[設定]> [設定]に移動します。ツール| File Watchers、新しいグローバルファイルビューアを追加し、[ Enabled]をオンにします。

プロジェクトのクローン作成のための更新されたインターフェイス


バージョン管理システムからプロジェクトを複製すると、更新された[ バージョン管理から取得 ]ダイアログボックスが表示されます。このアップデートの主な目的は、GitHubでプロジェクトを取得するエクスペリエンスを改善することです。これで、ダイアログボックスの左側でGitHubを選択し、ダイアログボックスから直接アカウントにログインできます。その後、アカウントおよび所属する組織のリポジトリのリストを参照して、クローンを作成するリポジトリを見つけることができます。
