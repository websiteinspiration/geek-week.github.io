La última revisión de la API de Renga

El sistema para el diseño de edificios y estructuras de Renga se está desarrollando activamente, tiene herramientas para arquitectos, diseñadores de ingenieros de KZh y KM, VK, OV, EO y ES. Con cada versión, el sistema, que ahora se divide en Renga Architecture, Renga Structure y Renga MEP, y pronto se convertirá en uno, crea funcionalidad. Junto con esto, crece y cambia, convirtiéndose en API cada vez más popular. De hecho, en Renga puede crear un modelo de información de un edificio, pero no puede (y no planea) obtener un hermoso render, una estimación para la construcción o el cálculo de tuberías. Por supuesto, puede exportar el modelo de Renga a formatos comunes como OBJ, IFC, STEP y muchos otros, pero para prescindir de intermediarios y evitar posibles interpretaciones erróneas de los datos durante la exportación / importación, es mejor actuar a través de la API.



Inicialmente, cuando surgió la cuestión de crear la API de Renga (después del lanzamiento de la primera versión de Renga Architecture), nos centramos solo en la API de C ++, en función de los requisitos que teníamos en ese momento.

A medida que Renga se desarrolló, comenzamos a recibir preguntas de los desarrolladores sobre si es posible escribir en C # y otros lenguajes en Renga y qué versiones de Visual Studio son compatibles.

Luego, después de analizar la disponibilidad de la API para los usuarios, pensamos en cambiar la tecnología.

El hecho es que la API de C ++ es bastante difícil. Depende mucho del compilador, es decir si el compilador de Visual Studio compila el SDK, entonces probablemente no pueda compilarlo usando, por ejemplo, gcc, ya que el cambio de nombre no está estandarizado, puede haber diferencias en la implementación de tablas virtuales, plantillas, etc. En Visual Studio, también puede haber diferencias en la configuración de compilación y el soporte para el estándar de una versión a otra.

En general, uno podría vivir con esto, por ejemplo, usando funciones virtuales al mínimo (y como consecuencia, herencia), no usando plantillas y arreglando el estándar utilizado y el entorno de desarrollo durante un cierto período de tiempo.

Además, podríamos crear varias versiones de la API para diferentes entornos y / o versiones del sistema.
Pero aún así, decidimos mirar para otro lado. Y después de la investigación, eligieron COM.
Este estándar se desarrolló en 1993, es decir, no es solo antiguo, sino superestrella), pero nos permitió proporcionar a los usuarios una única API, con la que puede escribir complementos en C ++, C # u otro lenguaje compatible con COM, además, puede usar la API COM para acceder a Renga desde aplicaciones de terceros, incluidas las escritas en lenguajes con escritura dinámica, por ejemplo, en Python.



Debo decir que la Ayuda de Renga API detalla cómo comenzar, qué y cómo usarlo. Pero aún le decimos lo que necesita para escribir una extensión para Renga.

Entonces, para escribir una extensión para Renga en cualquier idioma, necesitará:

  1. Renga y comprender qué es exactamente lo que desea automatizar al trabajar con el sistema.
  2. Kit de desarrollo de software Renga.
  3. Microsoft Visual Studio. Cualquier versión anterior a 2012.
  4. Conocimiento de los conceptos básicos de programación.

Puede obtener información sobre el sistema Renga y sus capacidades en el sitio web del sistema , en la ayuda , en el blog de los desarrolladores , así como en grupos en VK y Facebook , donde se publican anuncios y grabaciones de seminarios web, también puede encontrar usuarios reales del sistema y chatear con ellos. Cabe señalar que la API de Renga se está expandiendo a pedido de los usuarios, y si no vio las capacidades de automatización que necesita en esta lista, contáctenos y discutiremos su idea.

Si continuamos trabajando solo en la API de C ++, lo más probable es que el conjunto de características sea el mismo, pero estamos casi seguros de que no nacerán todas las extensiones de Renga. Por cierto, puede ver la lista de extensiones que conocemos en el sitio .

Ahora veamos qué ofrecemos al final a nuestros usuarios en términos de API. Para obtener el SDK de Renga, solo necesita descargarlo. Y para usarlo puedes estudiar y usar los ejemplos que están en el archivo.



Hay ejemplos en el SDK para escribir extensiones en C ++ y C #, así como ejemplos de conexión a Renga en Python. La Ayuda de la API de Renga en las secciones de Cómo se enumeran ejemplos que utilizan las técnicas de esta sección.

Los ejemplos de complementos de C ++ y C # demuestran casi todas las características de la API, en los ejemplos de Python encontrará cómo usarlo para crear una propiedad personalizada en Renga desde una aplicación de terceros, así como también cómo automatizar la apertura, el guardado y el cierre de un proyecto de Renga usando un script.

Actualmente, Renga puede cargar complementos desde archivos DLL binarios de C ++ y ensamblados .NET, y utilizando lenguajes de tipo dinámico como Python, VBScript, 1C y otros, puede acceder a la API de Renga desde programas de terceros para obtener los datos necesarios.
En general, estamos satisfechos con la transición a la API COM y las oportunidades que ha abierto para nuestros usuarios.

Cualquier extensión de Renga que se llame desde Renga debe consistir en un archivo binario para el complemento y un archivo .xml con la extensión .rndesc, con el siguiente contenido:

<RengaPlugin>
   <!--   -->
   <Name>Best Plugin</Name>
   <!--   -->
   <Version>1.0</Version>
   <!--    -->
   <Copyright>Copyright text</Copyright>
   <!--   Renga API -->
   <RequiredAPIVersion>2.3</RequiredAPIVersion>
   <!--   cpp/net -->        
   <PluginType>Plugin type</PluginType>
   <!--    -->                     
   <PluginFilename>Best_Plugin.dll</PluginFilename> ->
   <Vendor>Renga</Vendor>                           
</RengaPlugin>

Para conectar la extensión a Renga, debe instalarla o copiarla a la carpeta de instalación de Renga \ Plugins. La información contenida en el archivo .rndesc se muestra en la pestaña Extensiones de la ventana Configuración de Renga.

Estamos seguros de que, armados con el conocimiento adquirido y nuestro conocimiento de programación, cualquier desarrollador puede escribir una extensión para Renga.
