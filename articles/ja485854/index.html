C ++コンパイラが関数のオーバーロードを解決するのを助ける

場合によっては、C ++コンパイラーは適切なオーバーロードされた関数を選択できないことがあります。たとえば、人間の視点から明らかな状況では、コンパイルエラーが発生します。

void f(int i){}
void f(string s){}

vector<int> int_c = { 1, 2, 3, 4, 5 };
vector<string> string_c = { "Sic" ,"transit" ,"gloria" ,"mundi" };
for_each(begin(int_c), end(int_c), f);//error C2672: "for_each":

問題は、コンパイラーが過負荷を解決できる最後の行にfのパラメーターがないことです。非常に特定のタイプのイテレータがfor_eachパラメータに渡されるという事実は、コンパイラにとって重要ではありません。

問題を解決するには、いくつかの「正面」の方法を提案できます

。1）static_cast <>（）を使用して、目的の型の関数へのポインターにキャストを強制します。

void f（int i）へのポインターにキャストします。

std::for_each(begin(int_c), end(int_c), static_cast<void (*)(int)>(&f));

void f（string i）へのポインターにキャストします。

std::for_each(begin(string_c), end(string_c), static_cast<void (*)(string)>(&f));

2）関数へのポインタの定義とその使用。

void f（int i）を使用します。

 void (*fp_int)(int) = &f;
  for_each(begin(int_c), end(int_c), fp_int);

void f（文字列i）を使用します。

void (*fp_string)(string) = &f;
for_each(begin(string_c), end(string_c), fp_string);

3）for_eachテンプレート関数の明示的な特殊化の作成：

void f（int i）を使用します。

std::for_each<vector<int>::const_iterator, void(*)(int) >(int_c.begin(), int_c.end(), f);

void f（文字列i）を使用します。

std::for_each<vector<string>::const_iterator, void(*)(string)  >(string_c.begin(), string_c.end(), f);

以上のコンパクト：

std::for_each<decltype(int_c.begin()), void(*)(int) >(int_c.begin(), int_c.end(), f);
std::for_each<decltype(string_c.begin()), void(*)(string)  >(string_c.begin(), string_c.end(), f);

上記で提案された方法は普遍的ではなく、呼び出し側で追加のコードを開発する必要があります。最良のアイデアは、関数fのパラメーターを受け取り、それらを変更せずにそれに渡す関数fの上に薄いシェルを作成することです。

また、可能性を考慮する必要があります。

  • 関数fのパラメーターの数が無制限に存在する、
  • 関数の戻り値の存在、
  • 参照渡しのパラメーター、
  • cvパラメータ修飾子の存在
  • 関数fの例外。

そのようなシェルの構造を段階的に分析します。ラムダ関数は良い土台です。コンパイラによるC ++ 11標準のサポートにより、これは次のように実装できます。

for_each(begin(int_c), end(int_c), [](int a) { return f(a); });
for_each(begin(string_c), end(string_c), [](string a) { return f(a); });

またはdecltype（）を使用：

for_each(begin(int_c), end(int_c), [](decltype(*int_c.begin()) a) { return f(a); });
for_each(begin(string_c), end(string_c), [](decltype(*string_c.begin()) a) { return f(a); });

コンパイラーによるC ++ 14標準のサポートにより、多形ラムダに対して次のソリューションがあります。 

auto l = [](auto a) { return f(a); };

for_each(begin(int_c), end(int_c), l);
for_each(begin(string_c), end(string_c), l);

したがって、パラメーターを関数fにパラメーターとしてラムダに渡し、このパラメーターを使用してfを呼び出します。しかし、問題があります：事前にわかりません-関数パラメーターは値または参照ですか？したがって、完璧なギアを使用する必要があります。

auto l = [](auto&& a) { return f(std::forward<decltype(a)>(a)); };
for_each(begin(int_c), end(int_c), l);
for_each(begin(string_c), end(string_c), l);

完全な転送（完全な転送）のトピックについては、マイヤーズ[1]を参照してください。同じ資料のプレゼンテーションのよりわかりやすい（とにかく）バージョンは、記事[2]にあります。その翻訳はHabré[3]にあります。

関数パラメーターの数が無制限の場合、ラムダパラメーターを可変として示し、fに渡される各パラメーターに対してstd :: forwardを呼び出します。

 auto l = [](auto&&... a) { return f(std::forward<decltype(a)>(a)...); };

コンパイラー時間指定子noexcept [4]とnoexcept演算子[5]を追加して、関数fが例外をスローするかどうかをコンパイラーに示します[6]。 

  auto l = [](auto&&... a) \
        noexcept(noexcept(f(std::forward<decltype(a)>(a)...)))\
    { return f(std::forward<decltype(a)>(a)...); };

構築されたラムダの戻り値型の出力を追加します。

   auto l = [](auto&&... a) \
        noexcept(noexcept(f(std::forward<decltype(a)>(a)...)))\
        -> decltype(f(std::forward<decltype(a)>(a)...))\
    { return f(std::forward<decltype(a)>(a)...); };

トリプルコードの繰り返しが煩わしく、マクロを使用することをためらわない場合は、呼び出し側のサイズを減らすことができます。

#define LIFT(foo) \
[](auto&&... x) \
noexcept(noexcept(foo(std::forward<decltype(x)>(x)...))) \
-> decltype(foo(std::forward<decltype(x)>(x)...)) \
{ return foo(std::forward<decltype(x)>(x)...); }

その結果、コンパイラーがオーバーロードされた関数を解決するのを助けるシェルがあります。

for_each(begin(int_c), end(int_c), LIFT(f));
for_each(begin(string_c), end(string_c), LIFT(f));
