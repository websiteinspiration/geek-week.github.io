製品としてのAPI：バックエンドのみを理解しているときに作業を収益化する方法



この記事では、専門の市場でのAPIの作成、デプロイ、および販売における私の経験を再説します。ウェブサイトを作成したり、支払い処理のソリューションを実装する方法を考えたりする必要はありませんでした。コードを記述してAPIをデプロイしただけです。

スタートアップを作成するには、チームが必要です。このチームの全員が、プログラマー、マーケター、セラーなど、あらゆる取引のジャックである必要があります。さらに、それは長く困難な旅なので、成功の可能性はわずかです。ただし、独自のプロジェクトを開始するには、スタートアップは必須ではありません。バックエンド以外は何もせずに商品を売れると確信しています。

昨年、私は自分の製品をリリースする計画を思いつきました。これには、オーディエンスとの連携や支払いの処理は含まれていません。手順全体は3つのステップで構成されます。

  1. ある種の問題を解決するAPIを作成する
  3. API

API, -


約6か月前、Webで公開されたニュース記事から情報を抽出する「手順」を実装する必要がありました。これは、WebサイトのURLを入力として受け取り、記事のタイトル、発行日、著者名、メディアへのリンクなどの情報を含む構造化されたレポートを返すスクリプトであると想定されていました。Vladislav Kuchaevによる



関数の可視化

これをすべて実行できるPythonライブラリを見つけるのに数分かかりました。コードを書くのにあと数分-そして、私はすでに動くプロトタイプの準備ができています。必要な操作を実行するコードが既にある場合は、APIでラップするのは簡単です。ただし、このAPIのデプロイとホスティングは、すでにより困難な場合があります。

サーバーレスアーキテクチャにAPIをデプロイする


シンプルなサーバーレス機能

数百の記事を1時間に数回処理する必要がありました。そして、すべて同時に。つまり、分析する必要があるニュース記事の200〜800のアドレスのピークと、その後の不定期間-何もありません。

AWSについてはすでに決定しましたが、そのようなシナリオにはAWS Lambdaが最良のオプションだと思いました。 AWS Lambdaはサーバーレスツールであり、サービスとして機能します。特定のイベントに応答してコードを実行します。サーバーのサポートを提供する必要はなく、関数が実行されたときの料金のみを支払います。

Lambda関数をデプロイしました。すべてが適切に機能しました。数百の同時通話で、彼女は並行通話に対応しました。そして再び、彼らは実行時間の支払いをするだけで済みました。そのとき、私は次のように考えました。誰かがこの操作を実行するAPIを販売しているのだろうか？」



Googleが「記事抽出API」を検索しました

はい。一見すると、これらはすべて、私がプロジェクトで行ったのと同じことを行います。これは、次の2つの理由から朗報です。

  • そんなことが求められている
  • 誰と比較するか

サーバーレスAPI

次に、サーバーレス関数をサーバーレスAPIに変換する必要があります。私はZappaを使用しました。Zappaは、すべての主要な作業を行うPythonパッケージです。 FlaskでAPIを書き込み、それをZappa経由でAWS LambdaとGateway APIにデプロイしました。完全なAPIコードは、私のGitHubページで公開されています。

多くのプログラミング言語とクラウドサービスプロバイダーで機能するサーバーレスコードをデプロイするためのもう1つの優れたパッケージは、サーバーレスと呼ばれます。

AWSで私のAPIへの50,000回の呼び出しを処理する平均コストは0.6〜0.8ドルで、これには無料期間が考慮されていません。このタイプのソリューションの50,000コールの標準コストは、30〜250ドルです。したがって、主な質問は、APIを配布、宣伝、そして最も重要なこととして販売するために、他にどのくらいの費用がかかるかです。ほとんどの場合、私はこれで資本を積み上げることはしませんが、Netflix、Spotify、Leetcode、および他の何かを購読するコストをカバーする可能性があります。

Marketplace APIを通じて製品を配布する


彼らがRapidAPIに書いたものは次のとおりです。

「Marketplace APIは、他のオンラインマーケットプレイスと同じように機能します。プロバイダーにAPIを公開する機会を提供し、開発者はそれらを使用することができます。他のタイプのマーケットプレイスと同様に、マーケットプレイスAPIは、開発者向けポータルとAPIプロバイダー向けポータルを含むいくつかのコンポーネントで構成されています。

警告：この記事の執筆時点では、私はRapidAPIと提携していません。彼らは私に支払いをせず、特別な条件を提供していません。私にはこのプラットフォームが最適だと思われたので、それを選びました。

APIをRapidAPIマーケットプレイスに配置するために、次の手順を実行しました。

  1. GatewayおよびLambda APIを使用してAWSにデプロイされたAPI
  2. アクセスを制限するためにゲートウェイAPIにx-api-keyを作成しました
  3. そのAPIをRapidAPIマーケットプレイスに結び付けました
  4. x-api-keyをRapidAPI管理に渡しました

エンドユーザーが（私の製品のページから）RapidAPI側でエンドポイント要求を送信すると、必要な数の支払い単位がプランから差し引かれることを確認します。次に、RapidAPIサービスは、提供したx-api-keyを使用してAPIを呼び出します。 RapidAPIを介してサービスを使用する開発者の数は関係ありません。バックエンド側から見ると、すべて同じx-api-keyを使用した一連の同一の呼び出しのように見えます。ですから、ユーザーについてはどうですか、私の頭は痛くありません。料金を徴収し、関税計画を検討し、コントロールパネルを提供します-これはすべて私の心配事ではなくなりました。

このサービスの費用は、各トランザクションの20％です。トランザクションが受信されない場合、何も支払う必要はありません。たとえば、月額100,000コールの料金プランを50ドルで販売すると、この金額の80％、つまり40ドルが得られます。誰もこのプランを購入しなければ、RapidAPIに資金を提供する必要はありません。

発売


私のページに アクセスして、APIを無料で試すことができます。コードは、構成手順を含め、GitHubリポジトリに完全にアップロードされます。4月20日にProduct Huntに製品を投稿したので、この種のソリューションにコミュニティが関心を持っているかどうかをすぐに確認できることを楽しみにしています。

実行できるその他のAPI


確かに、あなたはすでに何らかの機能を持っています-少なくともいくつかのPythonライブラリをまとめる単純なもの（私のものはそのうちの1つです）。あなたが人生で目にしたことのある開発者なら誰でもこのコードを自分で再現できるように思えますが、それは問題ではありません。それを購入する準備ができている何千もの人々がまだあります。ほとんどの場合、コードを記述してデプロイし、維持するよりも、既製のソリューションにお金を払う方が収益性が高くなります。

以下は、需要が高く、多くの開発者が作成できるAPIのリストです。

  • テキスト要約を作成するためのAPI
  • Reformulation API
  • テキスト分類API
  • 画像分類API
  • テキストまたはWebページ上のジオデータを識別および正規化するためのAPI
  • 指定されたURLからメールアドレスを収集するためのAPI

オープンアクセスのライブラリを機能のコアとして使用できます。それらの機能を組み合わせて、独自のコードを作成します。

ちなみに、似たようなものを書いて作品に使用しているということは、聴衆のニーズに精通しているということであり、商品開発の大きなメリットです。

エンドツーエンドのソリューションを作成する


あなたの履歴書に注意を引くための最良の方法は、完全な何かを行うことができることを示すことです。ソフトウェア開発者、バックエンド開発者、データサイエンティスト、またはデータエンジニアとして初めて就職することは非常に難しい問題ですが、すでに頭に浮かんだAPIについて言及し、それを市場に投入し、成功裏にサポートすることで、チャンスが大幅に広がります。多くの雇用主は、GitHubのリポジトリよりもProduct Huntで製品ページを開くことをいとわないです。

最終的に


私が市場で見つけたAPIは、主に独立した開発者ではなく、企業全体の労力の成果です。会社によって開発されたソフトウェアは、エラーを生成する可能性がはるかに低くなります。しかし、彼らはたくさん尋ねます。ブートストラッププロジェクトに取り組んでいるインディーズの開発者は、より洗練されていないがより安価なものに目を向ける可能性が高くなります。

現在、世界経済は景気後退局面に入っています。企業の予算が削減されます。既存のソリューションに代わる低コストのソリューションを提供できるユーザーは、多くの聴衆を獲得します。たぶん私たちの時が来ました。
