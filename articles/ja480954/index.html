フルゲノムに関する以前の詳細な記事では、3つの問題を公開し、最初に3つすべてを正しく解決する問題をテストすることを約束しました。同時に、これらのタスクで遺伝データを処理する方法の例も示します。今日は最初のものを公開します。



最初の記事では、バイオインフォマティクスデータの操作に役立つ有用な情報とリンクを共有しました。見逃した場合は、最初にお読みになることをお勧めします。

免責事項
一部のコマンドとソフトウェアはWindowsでは使用できないため、遺伝的データの操作はUnixシステム（Linux、macOS）で実行されます。したがって、Windowsユーザーにとって最も簡単な解決策の1つは、Linux仮想マシンをレンタルすることです。

以下に説明するすべての操作は、コマンドライン-端末で実行されます。開始する前に、OSを実行している端末で作業する方法とコマンドを使用する方法を学びます。コマンドの一部はOSとデータに害を及ぼす可能性があるためです。


必要なソフトウェア


Yandex.Cloudに必要なすべてのソフトウェアを備えた仮想マシン（VM）のイメージを コンパイルしました。 Yandex.Cloudに登録し、アカウントの[コンピューティングクラウド]セクションで、[VMの作成]をクリックします。公開イメージとして、Atlas Data Analysisカタログから1000 Genomesを選択します。

VM構成：100％2vCPU、8GB RAM、20GB HDD。 VMを作成するときは、支払いデータを入力する必要がありますが、アカウントに書き込まれるものはありません。コードワードの開始と追加の付与で、2019年12月31日までアトラスのVMとイメージを無料で使用できます。タスクを渡すための助成金を受け取るには、コードワード「ATLAS」をYandex.Cloudサポートサービスに送信します。

注：2019年12月18日以降にサインアップしたYandex.Cloudの新規ユーザー、またはまだ試用期間があり、開始時の助成金を持っているユーザーに対して、助成金は有効です。コードワードATLASは一度だけ有効です。

最初に、VMへの接続を計画しているローカルコンピューターでsshキーを作成します。

~/.ssh/yandex-cloud.pubVMを作成するとき に、ファイルの内容を適切なウィンドウにコピーしてください。

ちらつく


Plinkは、遺伝データと広域ゲノム関連検索（GWAS）を操作するためのソフトウェアパッケージです。それは遺伝学者ショーンパーセル（ショーンパーセル）によって開発されました。 2008年以来、Plinkの支援により、世界中で何百ものGWASが実行されており、その最高の結果が、アトラスが疾患リスクを計算するためのアルゴリズムのデータソースとして使用しています。

Plinkは、ジェノタイピングデータを保存および変換し、それらを検索するための一連のツールを提供します。 Plinkは、統計処理、連鎖不平衡（LD）分析、降下による同一性（IBD）と同一性（IBSによる状態）分析、人口層別化および上位性テスト-いくつかの遺伝的変異の相互作用も可能にしますそれらの間。

IBDとIBSは、人口構成を分析し、親族関係を決定するために使用されます。

エピスタシスの例は、APOE遺伝子のrs7412とrs429358のバリエーションです。これらのバリエーションの特定の組み合わせは、アルツハイマー病を発症するリスクを急激に増加させますが、各バリエーションは、個々にリスクへの寄与はわずかです。

公式ウェブサイトからPlinkの安定版をダウンロードしてください。


BCFtools


BCFtools-VCF形式の遺伝子データとそのバイナリ対応BCFを操作するためのユーティリティのセット。BCFtoolsパッケージの可能なアプリケーションのリストには、VCF / BCFファイルの注釈、フィルタリング、マージ、および分割、それらの共通部分の検索、インデックス付け、選択検索、ソート、統計カウントなどが含まれます。

インストールするには、次のようにします。
git clone git://github.com/samtools/htslib.git
git clone git://github.com/samtools/bcftools.git
cd bcftools
# The following is optional:
# autoheader && autoconf && ./configure --enable-libgsl --enable-perl-filters
make

アがが


VCFの生成時にこの情報がわかっていれば、VCF形式を使用して、個人のフィールドに関する情報を単一の数値として保存できます。 X染色体のレコードのGTフィールド（遺伝子型、遺伝子型）には、男性の場合と女性の場合の2つの対立遺伝子に対応する1つの数値が含まれています。シーケンスされたサンプルの生物学的フィールドに関する情報がない場合、GTフィールドにはデフォルトで2つの数値が含まれます（図2で赤く強調表示されています）。

このマニュアルで使用するVCFファイルでは、Y染色体は除外されていますが、VCFファイルにY染色体が存在しても、シーケンスされたサンプルに実際にあるとは限りません。これは、X染色体とY染色体で同一であり、その両端にある偽常染色体領域（PAR）が原因です。

通常、異なる染色体には長い同一（相同）領域はありませんが、X染色体とY染色体には、最初（PAR1）と最後（PAR2）に数百万塩基対の長い領域があります。したがって、PAR領域の男性のNGSデータを分析すると、2つの対立遺伝子が見つかり（各性染色体に1つ）、女性の遺伝子型はY染色体のPAR領域に表示される可能性がありますが、実際にはX染色体の遺伝子型です。

アーカイブされていない形式でタスクを実行するためのVCFファイルは、それぞれ約19ギガバイトを使用するため、スペースを節約するために、アーカイブのみを使用することをお勧めします。上記のプログラムはすべて、圧縮されたVCFデータを処理できます。また、何もする必要はありません。

被験者の性別を決定するには、X染色体の遺伝子型を調べ、最初と最後にある領域PAR1およびPAR2を除外する必要があります。これらは、ゲノムのGRCh37バージョンの位置60001〜2699520および154931044〜155260560の間隔です。遺伝子型に1つの数値指定が含まれる場合、これは男性の生物学的性別、それ以外の場合は女性です。 VCFファイルでの性別の指定は、VCFの生成中の生物学的フィールドに関する情報の可用性に依存するため、このアプローチを常に使用できるとは限らないことに注意してください。

データセットのサンプルごとに次のコマンドを使用します。引数の後にサンプルIDを代入します-s。

(/Data/Tutotrial/CEI.1kg.2019.demo.vcf.gz):
#     
bcftools view -O v -H -s HG00731 -r chrX:2699521-154931043 CEI.1kg.2019.demo.vcf.gz | head

コマンドを実行すると、指定したサンプル識別子のVCFファイルの内容の一部が表示されます。-r chrX:2699521-154931043 BCFtools のパラメーターは、ファイルの内容の表示を、2699521番から154931043番の染色体Xの領域に制限します（図3の非PAR領域）。これらの境界は、この場合には不要な偽常染色体領域を除外します（PAR1およびPAR2）。GTフィールドの数値を使用して、各サンプルの性別を決定します。


図3。性染色体上の偽常染色体領域PAR1およびPAR2の位置。

図1のVCFファイルまたはVCFファイルのヘッダーの最後の行にあるすべてのサンプル識別子のリストを確認できます。それらは、FORMAT列名の後にリストされます。

私たちは関係を決定します


関係を判断するには、すべてのサンプルの遺伝子データをペアで比較する必要があります。完全なゲノムに従ってこれを行うのは困難です。この場合のVCFファイルは数十ギガバイトかかります。使用するVCFは約2ギガバイトしか使用しませんが、イルミナのチップ上で遺伝子型が特定された遺伝的変異識別子（rsID）のリストに従ってフィルタリングします：GSA v1、GSA v2、HumanOmniExpress v1.0、HumanOmniExpress v1.3、InfiniumExome v1。 1およびInfinium OmniExpressExome v1.4。これらは、商業ジェノタイピングで最も人気のあるチップです。

これらのチップの遺伝的変異のすべての識別子のリストを、rsIDのリストを含む別のファイルにまとめました。140万の識別子が含まれています。VCFファイルをフィルタリングするには、次のコマンドを実行します。

bcftools view -O z -i 'ID=@rsids_for_subsetting.txt' CEI.1kg.2019.demo.vcf.gz > CEI.1kg.2019.demo.subset.vcf.gz

BCFtoolsおよびその他のパッケージを使用してVCFファイルを操作するたびに、以前のコマンドの履歴がファイルのヘッダーに追加されます。 VCFファイルをフィルタリングする方法と以前に実行されたコマンドに関係なく、ハッシュ合計を計算することにより、VCFのメインコンテンツの整合性とIDを確認できます。

#     
gunzip -c CEI.1kg.2019.demo.subset.vcf.gz | grep -v "^#" > CEI.1kg.2019.demo.subset.body.vcf
md5sum --tag CEI.1kg.2019.demo.subset.body.vcf
#       d609eb330908d4eb5feb9f1860fd508b

このコマンドgunzip -cは、ファイルの内容をstdoutに出力して解凍します。そこから、開始する #（したがって、コマンドが使用されるgrep -v "^#"）VCFファイルのヘッダー行が削除されます。ヘッダーは、遺伝的データ自体のみの整合性を比較するために削除され、このVCFファイルの操作に使用されたツールと時期に関するメタデータは比較されません。

ハッシュ値が一致する場合は、次に進んでVCFを内部のPlink形式に変換できます（デフォルトでは、Plink形式は、bed、bim、famの拡張子を持つ3つのファイルです）。これらのファイルには、遺伝子型、染色体、位置などのデータのみが残り、残りは削除されます。この形式で作業する方が簡単で、VCFからの追加情報を必要としないさまざまな問題を解決できます。たとえば、GWASを実施します。

#    
plink --vcf CEI.1kg.2019.demo.subset.vcf.gz --make-bed --out CEI.1kg.2019.demo.subset

このコマンドは、フォルダー内に3つのファイルを作成します 。10個のサンプルすべてのペアワイズ関係を決定できます。次のコマンドを使用して、Plinkファイルを分析します。
CEI.1kg.2019.demo.subset.bed
CEI.1kg.2019.demo.subset.bim
CEI.1kg.2019.demo.subset.fam
