この記事がハブレではなく教育雑誌で公開された場合、私は次のように始めます。「ナビゲーションが便利で、インターフェイスが便利な、お気に入りのサイトを思い出してください。しかし、これは氷山の一角にすぎないことをご存知でしたか...」しかし、おそらくすべてがハブレについて知っているので、歌詞はありません。 Webプロジェクトのバックエンドは複雑なシステムであり、アプリケーションは迅速に動作し、効率的に検索し、情報をすばやくユーザーに提供し、データを安全かつ安全に保存し、必要に応じて他の外部システムと連携できます。それはスイスの時計のようなものです。彼らはスタイルと銘板のためにそれを購入しますが、愛好家は拘束された文字盤の下に時計仕掛けの魔法の世界があることを知っています。そして、バックエンドの開発者は、メカニズム、生物、腸、内臓の魔術師、またはWebアプリケーションでそれを呼び出す人です。ええと、歌詞がなければできませんでした！ 

つまり、今日は「職業...」シリーズの別の職業を分析しています。では、バックエンドの開発者に行ったらどうでしょうか？ 


バックエンドは常に少し苦痛です

どなた？


学問的に言えば、バックエンド開発者は、WebサイトまたはWebアプリケーションの内部ロジックと計算ロジック、およびその他のソフトウェアと情報システムを担当するプログラマーです。厳密に言えば、バックエンドはWebソリューションだけでなく、デスクトップアプリケーションとモバイルアプリケーションの両方で利用できます。実際、バックエンドはサービスのハードウェアとソフトウェアに関連するすべてです。しかし、歴史的には、バックエンドとフロントエンドがWeb開発で特に明確に明確に区別され始めました。そして、これは現時点で最も人気のある領域でもあるので、特にWebについて説明します（デスクトップの場合、これらの多くも有効ですが、モバイル開発者については、問題の1つで個別に説明します）。

バックエンド開発者は、インターフェースを介してアプリケーションまたはサイトのユーザーがアクセスできるコンポーネントと機能を作成します。簡単に言うと、backdersは、ユーザーが見たり直接触れたりしないすべてを開発するプログラマーです。つまり、オンラインストアの美しいデータ入力フォームまたはバスケットはフロントエンドであり、DBMSへのデータの格納、一連の登録フォームフィールドとバスケット、支払いゲートウェイとの統合、自動レター、コンテンツのロードと更新はすべてバックエンドです。したがって、登録フォームでデータベースエラーが発生し始めた場合、エラーは...テスター:-)です（理由-次のシリーズでも説明します）。

バックエンド開発者が担当するもの：

  • サイトとその計算ロジックのすべての機能が正しく動作するようにします。
  • DBMSを介してデータベースを編成および操作する。
  • 基本的なロジックとアプリケーション操作アルゴリズムの開発。
  • API
  • 外部サービスとの必要な統合;
  • アプリケーションと個々のコンポーネントのテストとデバッグ。

一般に、サイトの脳と血はバックエンド開発者の良心に基づいています。これがすべての開発の神であり王であると考える必要はありません。プロセスの残りの参加者は彼の行動を見込んでフリーズします-これは完全に真実ではありません（まあ、ほぼ）。


フロントエンドの開発者は、ランプを黄色に塗ってバックエンドをこすり、ランプが機能するようにしますが、日中のみです。


バックエンドの開発者は、誰もがこれらの電球で問題を抱えていることに驚いています。彼がデータベースに光を当てるのを忘れていて、落ち着いてフロントエンドのせいにしていることを思い出します。

どこに必要ですか？


フロントエンド開発の場合のように、絶対にどこにでも、Web上でユーザーと対話するWebサイトまたはアプリケーションがあります。誰もがさまざまなレベルの支援者を必要とすることは別の問題です。条件付きソーシャルネットワークのオフィスや大規模なアナウンスサイトでは、バックエンド開発者は基本的なスキルだけでなく、高負荷、マイクロサービスアーキテクチャ、コンテナ化（Docker、Kubernetes）を操作し、監視システムを知っている必要があります（ Zabbix、Grafanaなど）にはスクラムスキルがあります（特定の企業ではオプション）。小さなオンラインストアの場合、PHP-Ajaxで必要なDBMS-HTMLの束で十分です。賃金も経験に対する要件もまったく異なることは明らかです。 

伝統的にハブレキャリアでバックエンド開発者の求人は444件見つかりました。これはフロントエンドの約2倍です。hh.ru-約1800。一般に、フロントエンドベンダーは、サイト検索が完了するまでに2時間かかるワークステーションであなたを待っています。ここでは、複雑な内部ロジックとビジネスロジック（オンラインストア、CRM-ERP、企業ポータルなど）を使用するアプリケーションについて説明します。

基本的な専門家の要件


バックエンドの要件は、フロントエンドよりも会社とそのビジネスプロセスの機能に敏感です。 「企業としての魚用冷凍機の原理を理解する」、「販売の基本知識または販売経験」、「JavaScript、CSS、HTMLの優れた知識」など、一見すると奇妙な場合があります。しかし、それは最初のアプローチでのみクレイジーで反発的に見えます。実際、バックエンド開発者は本当にビジネスプロセスに没頭しており、それらに応じてコードを開発するだけでなく、特定のタスクに合わせていくつかを調整し、内部からどのように機能するかを理解する必要があります。販売目標到達プロセスを一度も見たことがなく、それがどのようなガベージであるかを理解していない場合、フィルター、スライス、トランジションと一緒にロジックを開発することはできません（優れたTKがあるとしても、それを理解するのは困難です）。まあ、「JavaScript、CSS、HTMLの素晴らしい知識」がある例では、すべてがシンプルです。マネージャーはおそらく他の開発者を本当に信用しておらず、開発へのアプローチを再検討する準備ができています。これは多くの場合、小規模な企業で発生します。 

しかし、バックエンド開発者がほとんどすべての仕事で見るであろう基本的な要件のセットもあります。

  • 少なくとも1つの「サーバー」プログラミング言語に関する知識：PHP、Go、ASP.NET、C / C ++、Python、Ruby、Java。場合によっては、バックエンド（Node.js）のJavaScriptの知識で十分ですが、これは重要なことではありません。
  • APIの知識（REST、SOAP-ますます少なく）。
  • Apache、NGINX、IIS、およびその他のサーバーの原理を理解する。
  • 単体テストを作成し、コードをテストでカバーするスキル。
  • ネットワークセキュリティの基本とそれを保証するツールの知識。
  • 特定のアプリケーションの開発の問題を解決できる一般的なWebフレームワークに関する知識。
  • データベースクエリおよびデータベース設計の作成スキル。
  • フロントエンドの基本を知ることはプラスではありませんが、必須のポイントです。それ以外の場合は、アプリケーションの設計と作成が非常に困難になります。

履歴書とあなたの実際の仕事の両方のための大きなプラスは、別の知識のセットになります。

  • UNIXの管理またはLinuxの知識（任意の1つの配布キットを使用できます）。
  • HTTPの動作原理（キャッシュ、承認、メッセージ構造、ヘッダー、応答コードなど）に関する知識
  • OSIモデル。 
  • 参照条件（TOR）の準備​​と評価のスキルは、ソフトウェア要件に関する最も正確な情報を収集するために必要な非常に重要なスキルです。 

「はい、おかしくなりました！」-あなたは言うでしょう、あなたは正しいでしょう。ただし、良いニュースがあります。バックエンド開発ツールの変更はフロントエンドよりもはるかにゆっくりであり、要件が古くなります。だから私は一度それを習得しました-そして物事を最新に保つためにダイジェストを読みましたが、特にフラットなデザインがファッショナブルになり、あなたのサイトの製品カードが2日間流行しなくなったので沸騰しないでください:)

ちなみに、バックエンドの開発は正確な科学であり、結果の品質は、選択および適用されたアルゴリズム、パターン、サイクル、関数などの検証方法に大きく依存するため、バックエンドには優れた強力な数学者がたくさんいます。

職業の神話


▍バックエンドは非常に難しい


問題点。一部の開発者は、開発ツール、無数のライブラリー、および低レベルのダイナミズムのおかげでより簡単であると主張していますが、他の開発者は原則としてバックエンドにアプローチすることを恐れています。ただし、いつものように、真ん中：むしろ、バックエンドの開発者スタックは多様であり、異種のコンポーネントがありますが、最初に、プロジェクトにすべてが必要なわけではありません。2つ目は、1つのことを理解するだけの価値があり、すべての関係が明らかであり、知識です。一本の棒で縛られて。いずれの場合も、バックエンドで自分自身を試す必要があります。向きを変えて別のタイプの開発を選択した場合でも、得られた知識は作業に役立ちます。


▍バックエンドデベロッパーが数百万ドルを獲得


これは一般にプログラミングに関する一般的な神話です。私が自分の目で見なければならなかったロシアの開発者の最大の実質賃金は54万ルーブルです。 1か月あたり（C / C ++、海外プロジェクト、コンピュータービジョンシステム、PhD、リモート）、2番目-40万ルーブル。 （Delphi、udalenka、外国企業、ネイティブとしてのドイツ語）。残りは、私のプレートの「Habr Careers」から取った数字と非常に似ています。言語の素晴らしい知識+才能+狭い範囲での経験とユニークなスキル+高額の支払いが可能なプロジェクト-それが収束する場合にのみ、数百万を受け取ります。これらの人はほんの少数であり、彼らの運命は美しいとは言えません。彼らの人生の99％は仕事であり、1％は夢です。したがって、別の論文を実現することが重要です。バックエンド開発者は需要があり、多くの専門家よりも優れた収益を上げています。手段試すだけの価値があります。
