さて、そうストア。格納形式が許可するよりも高い精度で計算を実行した後、数値を「格納」するとはどういう意味ですか？これは丸めを意味し、丸めはエラーを起こすことを意味します。エラーはさまざまな方法で推定できます。繰り返さないように、今日ではフーリエ変換を使用したスペクトル分析を使用しています。

非常に短い紹介


正弦波の形の信号を受け取り、それに対してフーリエ変換を実行すると、スペクトルでは単一のピークが得られます。実際、スペクトルには、非線形歪みの結果として得られる基本トーンの倍数である周波数の高調波と、ノイズ、干渉、およびデジタル化の結果として得られるノイズシェルフの両方が含まれる場合があります。ここでは、これらのノイズのレベルを測定します。

開始


さらに興味深いことに、テスト信号として1つの正弦波ではなくいくつかの正弦波を使用します。これらの正弦波の周期が離散フーリエ変換の周期に完全に適合することを確認する必要があります。Wolfram Mathematicaでは、これは例えば次のように行うことができる：

sz = 8192;
data = Table[2 Sum[
Sin[Prime[j] k 2 Pi/sz + j*j]/sz, 
{j, 100, 200, 2}] // N, 
{k, 0, sz - 1}];

ここでは素数は周波数の不均一な間引きに使用されます。そして、j * jは、周波数に応じて正弦波の位相をシフトして、テスト信号の強いスパイクを回避し、ある程度均一な振幅を提供します。視覚的に受信された信号は次のようになります。

結論


奇跡はまだ起こらなかった。スペクトル分析では、Positフォーマットをストレージとして使用するとDoubleに近い精度が得られるという著者の声明は確認されませんでした。最良の条件でさえ、PositのノイズシェルフはFloatよりも20デシベル低いだけでしたが、同時にInt32よりも10デシベル高く、Doubleよりも60デシベルだけ高くなっています。

もちろん、Positは有用なアプリケーションを見つけることができます-許容範囲を超えないようにするための保護として、通常の値を大幅に超える排出がクリッピングまたはオーバーフローにつながらない場合。ただし、このシナリオでも、Positは明らかにより良い数値形式というよりは、IntとFloatの間の妥協のように機能します。
