なぜそれが必要だったのか


私にとって、12インチの画面と約1キログラムの重さの子供用ノートパソコンは、旅行や出張に最適です。これらはテキストを操作するためのタイプライターで、インターネットでネットサーフィンしたり、夜に映画を見たりできます。私は2005年以来、同様のモデルを定期的に利用しています。2017年には、当時の老人をより陽気な、しかしできるだけ予算に優しいものに置き換えるときがきました。

一般的に、それがうまくいったとしたら、まるで取るようなものでした。唯一のことは、ベイトレイルなどの古い「原子核」の割合は欲しくなかったが、少なくともアポロ湖などのアップグレードは進んだ。シングルスレッドモードでは、以前の「アトミック」モードの1.5倍の生産性があり、タイプライターでマルチスレッドが必要になることはほとんどありません。

2017年に何が選ばれたのか


これは、デュアルコアIntel Celeron N3355（Apollo Lake）を搭載した1キロのAcer ES1-132で、値札は16k弱です。



ドライブは32 GBの容量を持つ小さなeMMCでした。ドライブを通常のSSDに交換する予定だったので、ドライブのボリュームは気になりませんでした。幸いにも、彼にとっては簡単にアクセスできる別のコンパートメントがありました。

最初の待ち伏せは、ドライブベイカバーを開いた後に発見されました-ドライブ用のSATAコネクタがありませんでした。後で判明したように、ドライブは名前のないコネクタに接続されており、メモリスロットの下の写真に表示されています。これを行うには、特別なアダプターケーブルが必要です。あなたはこれらをロシアで見つけることができませんが、彼らは800ルーブルの開始価格でAliexpressに出くわします。これは運が良ければ利用可能になります。したがって、安定した値札は2〜3倍高くなります。そして2017年、私はこれについて何も知りませんでした。ただし、これはストーリーのメイントピックとは関係ありません。



その結果、解決策は次のとおりでした：内蔵ストレージデバイスに「Windu」と「Office」のみを残し、残りをケースから忍び出さない小さくて機敏なUSB 3.0フラッシュドライブに配置することです。

災害の規模を評価するために、私はeMMCドライブのテストを行いました（私はドライブと共存する必要があるため）、ここに結果を投稿しました。一般的に耐えられる。少なくとも、一般的なHDDよりもはるかに高速です。特に、小さなファイルを操作する場合はそうです。

次に、追加のメモリバーを取り付けて、RAMの量を6 GBにし、店でフラッシュドライブを探しに行きました。
次は古い格言によって特徴付けられるかもしれません：「何も悪いことを予告しない素晴らしい晴れた日が始まりました-それは13日の金曜日でした」。

ほとんどすべての小さなフラッシュドライブが過熱し、減速し始めることが判明しました


市場には、USBコネクタの境界を超えてほとんどクロールしないフラッシュドライブの赤ちゃんがたくさんいます。そしてその中には、線形の読み取り/書き込み操作で約100/60 MB /秒の速度を示す機敏なモデルがあります。映画や作品データを保存するためには、十分すぎるほどです。

しかし、彼らのテストを調べたところ、40〜50秒の激しい作業の後、ほぼすべてのパンくずが過熱し、コントローラーのスロットルがオンになり、速度が5回ごとに低下することが突然わかりました（いいえ、OSのキャッシュと混同しません）。これは次のようなものです（写真は私のものではありません）。



その結果、Transcendのフラッシュドライブで停止する必要がありました。これは、顕著に突き出して、線形テストで兄弟の過熱よりも少し控えめな結果（約97/29 MB /秒）を示していますが、減速しません。



最初の雷-Win10は彼女が必要とするスペースの量を知りません


最初の組み込み後、ラップトップはほぼ1日更新されました（これは冗談ではありません）。この手順の最後に、ドライブの32ギグのうち、残りの9ギガのみが残りました。はい、Windowsは20 GB以上を消費しました！次に、ライセンスされた「Office」をほんの少しだけ転がしました。5ギガバイトの空き容量が残っており、心配する必要はありませんでした。

2017年の秋、最初のCreators Updateである1703がラップトップに届きましたが、それは要求なしに変動し、ドライブの空き容量は1ギガバイト未満になりました。そして始まりました...
それはインストールのために彼が6 GBを解放するように頼んだという事実から始まりました！それら。それは私がそこにインストールしたすべてのソフトウェアよりも2ギグ多いです！
私は時間をかけてすべてを解体し、ディスクのクリーンアップを行い、Acerから愚かなソフトウェアを削除し、古いリカバリポイントを解体しました。これらすべての試練の後、彼は約950 MBの何かを失っていました。
保証付きでAcerサービスに持ち込み、破損していると言われましたが、更新されず、修復するという素晴らしいアイデアがありました。そして、そこで彼を楽しんでもらいましょう。結局のところ、この会社の誰かが、32ギガバイトのドライブにWinduをインストールするというアイデアを思いつきました。
しかし、私たちはそうではありません。解体の第2フェーズが始まり、システムソフトウェア、あらゆる種類のJava、Microsoft Visual C ++再頒布可能ライブラリ、そして実際に1 MB以上かかるすべてのもの（カジュアルゲームも含む）に影響を与えました。

一般的に、数時間後、適切な場所が見つかりました。アップデートがロールアップされ、ドライブのコントロールポイントが解体された後、残りのギグは5ギガのみでした（まだソフトウェアをインストールしていませんでした）。同時に、このアップデートでのすべてのすばらしいMSの革新は、原則として私には興味がありませんでした。

この悪夢の後、私はブラウザ、無料のオフィス、および一緒に作業したいくつかのコンパクトなグラフィックエディタのみを返しました。
次のFまで、残り6か月余りでした。

ダブルツー：すべては秋のようですが、より厳しいです！


2018年の春に、コードネームRedstone 4（1803）がアップデートされました。2つの無料のギグで、彼はジャンプ用に5つ、次にインストール用にさらに6つを要求します。そして画像を完成させるために-接続されたフラッシュドライブで16 GB！..
一般的に、32 GBドライブを搭載したシステムで、このような更新に青信号が出されたときにMicrosoftが何を考えているかは、私にとっては謎です。まあ、それとも、PCメーカーはそのような「創造物」を発売するときにどう思いますか。
さらに、この更新がダウンロードされない場合、組み込みのDefenderのデータベースの更新を含め、残りの更新アクティビティは完全に中断されます。まあ、特に娘や他の家族がラップトップを使用することがあったという事実を考慮して、更新なしでWinduを離れて穴あきふるいにしたくありませんでした。
一般的に、これらのギガバイトすべてを当時どのように削ったかを言うのは難しいですが、覚えていたライフハックの1つは、スワップを無効にすることです。私の場合、彼は追加のギグをリリースしました。



スワップを外付けUSB 3.0 HDDに切り替える試みもありましたが、Windowsはエラーを再起動し、メインドライブでスワップが行われたと書きました。なぜかわかりませんでした。

ダブルスリー：MSが問題に気づき、修正しましたが、機能しません


1903の数字が付いたもう1つのメガアップデートは、ほぼ1年間の落ち着きを破り、MSは最終的に小さなドライブへのインストールに問題があることを認識し、問題を解決しようとしました。彼らがしたことは次のとおりです：

  1. Windowsはそれを自動的にダウンロードしなくなりましたが、ユーザーが気づき、ダウンロードボタンを押すのを待ちます（この特定の瞬間の調整にはちょうど1年かかったと思います）。
  2. さらに、他のマイナーアップデートがダウンロードされてインストールされます（一般的には画期的なようです）。
  3. メインドライブに十分なスペースがない場合、USBフラッシュドライブまたは他の外部ドライブを取り付けて、インストール中に10 GB以上の空きスペースを借りるように求められます。



しかし、これはすべて理論ですが、実際には回路が機能しないことがわかりました（MS、私はあなたを疑いませんでした）。そもそも、システムはまだメインディスクの多くのスペースを解放するように要求します（まあ、そうです）。彼女がメインディスクで要求されたよりも500 MB以上多く与えられた後、1903パッケージのインストールは沸騰し始めました。1時間後、システムが再起動し、画面上のビーズが長時間オンになり、その後、次のようなすばらしいメッセージが表示されました：



実験者として、アップデートをロールアップするためにさらにいくつかの試みを行い、初期状態を変更しました（さまざまなタイプのドライブを接続し、追加のメモリを解放し、スワップで遊んだ）。しかし、結果は同じでした。

90年代前半のソリューション


エラーコード0xc190020fが失敗した更新のログで点滅し、ディスクスペースがまだ不足していることを示唆しています。それで、ヴィンダは彼女がどれほどのスペースを必要とするか理解していません、そして、彼女はより多くのために解放されるべきです。しかし、どうやって？

脳への要求は30年前に決定を返しました（草がより緑で、4メガを搭載した386DX40は、DOOMが既に実行されていたため、誰にとっても夢でした）。はい、私たちはディスク圧縮について話しています。私の友人の何人かは、40〜80 MBの容量を持つHDDを備えたシステム上で、ほとんど何のためにも十分なスペースがなかったときに、この手順に頼りました。 Win10では、ディスクをバックアップする機能は維持されていました。私の場合、アーカイブによるディスクのクリーンアップにより、6 GBも解放されることが約束されました。この手順は絶望から決めました。その結果、約束された6 GBではなく1.8 GBしか解放されませんでした。



このベンチャーの欠点は、システムがディスク上のデータを常にアーカイブ/解凍するために、追加のプロセッサ時間を必要とすることです。そして、プロセッサー時間に関しては、ここではすべてがあいまいです。

その結果、乗る！そして、ディスク上の古いバージョンのOSとリカバリポイントをすべて削除した後、8 GBが突然解放されました。しかし、冬に近づく可能性のある次のメガアップデートまでしか疑いがあります。

パート2：実際のコアプロセッサとは


予算外のセグメントで普及している古い「アトミック」コア（ベイトレイル、ブラスウェル、チェリートレイル）を備えたプロセッサーのトピックを繰り返しますが、私は触れません。コアあたりのパフォーマンスが非常に低いため、ユーザーエクスペリエンスが決まります。
コアとなるプロセッサを見つけたい- この記事のプレートはあなたを助けるでしょう。
比較的元気の良いコアを備えた唯一の予算のIntelモバイルプロセッサは、Celeron N3350です。2つのApollo Lakeコア（ゴールドモントアーキテクチャ）を備えており、ピークで静かに2.4 GHzまで加速します。ただし、N3350には、Celeron N4000とN4100（Gemini Lake）という2人のフォロワーがいます。

パフォーマンスの次のステップは、Haswellアーキテクチャの古い人々ですが、その消費電力は、ゴールドモントの5ワットに対して15ワットから始まります（つまり、それらを搭載したすべてのラップトップは、価格を含めて、すべての結果ですでに積極的に冷却されます）。

実務経験


したがって、タイプライターの機能を使用する場合、すべてが正常です。原則として、CS：ディテールと解像度を抑えたCondition Zeroでも、かなり許容できます。彼はまた、Facebookフィードのスクリプトをかなりうまく噛み、あらゆる種類のビデオをひねりますが、VKontakte（Firefox経由）からのHDクリップの再生を遅くします。

しかし、このすべての牧歌には1つありますが、それはWindows 10です。定期的な検索と更新のインストールには、2つの弱いCPUカーネルが非常に長い時間（場合によっては20分）にわたって100％の負荷をかけます。また、15〜20分は、要求がなければ、両方のDefenderコア（すべての種類のチェックが無効になっている）にとどまることができます。より高速なシステムでは、これらすべてに気付くことはありません（Core i7-6500Uプロセッサを搭載したラップトップに同じ1903メガアップデートをインストールすると、すべてをダウンロードし、インストールを開始し、再起動したいというメッセージが表示されます）。システムは再び何かにかかっています。同じディフェンダーがひそかに採掘しているように見えることもあります。しかし、あなたがこれらのトリックに注意を払わない場合、それは生きることはかなり可能です。

32ギグとWindows 10で生きる方法（結論）


まず、購入しないでください。まあ、なんらかの理由でこれらのゲームをプレイする準備ができている場合は、（ディスクから余分なものをすべて削除した後）役立つものを以下に示します。

  1. 以前のバージョンのWindowsのアンインストール（「ディスククリーンアップ」を使用）と古い復元コントロールポイントを忘れないでください。
  2. ボリュームをアーカイブします（ディスクのプロパティを確認するか、[ディスクのクリーンアップ]から[システムファイルのクリア]ボタンをクリックしてリストからアーカイブアイテムを選択します）。
  3. システムに4 GBを超えるRAMがある場合は、更新中のスワップを無効にします。
  4. Microsoft WimBootテクノロジーもあります。OSバックアップファイルのコピーをポインタに置き換えますが、多くの手間がかかります。気に誰が、ある細部の多くは。

更新管理を手動モードに切り替えるWindows Update MiniToolユーティリティも役立つ場合があります。
しかし、一般に、「シャーマニズム」がないと、32ギガバイトのドライブを備えたシステムとWindows 10は、それ自体でグローバル更新をローリングすることができないことを覚えておく必要があります。さらに、それらにつまずくと、他のマイナーアップデートをインストールできなくなり、最終的にシステムがセキュリティのための穴の開いたふるいに変わります。
2019年のそのような偽のラップトップは、Irbis、Digma、Krez、Prestigio、Haierから提供されています。そして、これらの会社ですべてが明確である場合、HPとAcer（それぞれ1つのモデルがあります）は驚くべきものであり続けます。
