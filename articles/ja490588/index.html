次に、私はすぐにArduinoの最初のコードを書き始めました。 2つの潜水艦エンジンを制御するためのコードでした。 2つのポテンショメータ：左はエンジンの総出力を制御し、右は潜水艦の回転を制御します（ポテンショメータの位置に応じて、エンジンの1つの出力を低減します）。これらすべてを、別のコントロールパネルを作成することを計画していたため、安価なディスプレイに表示しました（その結果、潜水艦はスマートフォンで制御されます）。



1週間前にポテンショメーターがどのように機能するかわからなかったことを考えると、私の熱意は何とも言えませんでした。そこで止まらず、金物屋や薬局に行きました。建設業界では、さまざまなポリプロピレンパイプ、カップリング、クランプを採点し、薬局ではいくつかのジャネットシリンジを使用しました。

パイプはそれぞれ潜水艦の船体に行き、注射器はモジュールに浮力を変える。浮力を変更するモジュールだけが、私にとって最も問題の多い部分であることがわかりました。

浮力係数


このモジュールのタスクは非常にシンプルで、水を集めて、必要に応じて水を絞り戻します。そして、疑問が生じました-サーボ、モーター、ギア一式を備えた注射器のピストンをどのように押すのですか？そのように押す価値はありません



。ギアやその他のささいなことを操作した最初の経験でした。ちなみに、sg90サーボを360度回転用に作り直すことができました。ポテンショメーターを回すメインギアのラッチを研ぎ、ポテンショメーターシャフトをゼロ位置に接着しました。

ギア写真


これはまだ問題の解決に役立ちませんでした-ギアラックと相互作用するギアを確実に修正できませんでした。工学で得た経験は、浮力変更モジュールを2回目に習得するのに役立ちました。より強力なサーボ、太いねじ付きロッド、およびピストンに固定したナットを使いました。今回はサーボの改造に悩まなかったので、外部ドライバーを使ってサーボモーターに直結した方が簡単だと思いました。

ピストンにレーザー距離計を配置して、リアルタイムで、現在の位置を確認できるようにしました。さて、この距離データに基づいて、ピストンロックが極端な位置にある場合に、それを処方しました。おそらく、ピストンの位置を決定するためのより簡単な方法があるかもしれませんが、私は誤って中国人から非常に安いモジュールを見つけました-VL53L0X距離計で、それを使用することにしました。その結果、私は非常に満足しました。ライブラリはシンプルで、それは正常に機能します。注射器の限られた空間の精度は約5mm程度で、原則的にはそれで十分でした。



テスト中に、別の問題が発生しました-ピストンがシリンジの壁に強く付着しています。何が接続されているのかはわかりませんが、ピストンの動きを開始するにはかなりの力を加える必要があります。最初のジャムの後、正常に作動します。私たちはほとんどすべてのタイプの潤滑剤を試しました-それらの多くはそれを悪化させました。このため、モジュールにアルミフレームを追加する必要がありました。

ハウジング


ケースは興味深いタスクでした-簡単に分解できる密閉された接続を作成することです。私は任務を果たさなかった、すべてをしっかりと接着しなければならなかった。シリンジが水を引くと、体内に圧力が発生し、すべての留め具が押し出されます。その結果、すべての重要なワイヤが密閉されたコネクタに運ばれ、それを介してデバイスを充電したり、オンボードのArduinoをフラッシュしたり、アンテナを接続したりできます。

はい、当社のアンテナはケーブルを使用して接続され、表面位置にあるため、信頼性の高い通信が保証されます。しかしアンテナについては少し後で。



追加の写真







ハウジングは、50mmポリプロピレンパイプとカップリングで構成されています。ジョイントは密封されたペーストで覆われ、その上に強度のためにホットメルトが充填されています。シリンジノズル、気密コネクタ、インクルージョントグルスイッチ、およびサーチライト用の2本のワイヤーが最後まで運ばれました。フラッドライトは浸水部分の船首に取り付けられ、この設計により重心を潜水艦の中心に近づけることができました。

実際、これらは、コレクターモーターの動作により回路で発生するピークを平滑化するための回路で役立ちました。電圧レギュレータを接続するためにも必要です。

最初は、すべてをブレッドボードに集めただけの理由でうまくいかなかった。回路のこの部分またはその部分が機能しない理由を理解できませんでした。その結果、すべてがはんだ付けされ始め、陽性のテスト結果は間もなく実現しました。

アンテナだけでさらに困難が生じました。

アンテナ


アンテナとして、esp8266を使い、スマートフォンからWi-Fi経由で潜水艦を操作することにしました。 ESP8266に基づく多種多様なモジュールを持っているのは中国人だけです。3つの異なるモジュールを購入しましたが、接続してフラッシュできるのは1つだけでした-ESP -01。

理論的には、今注文すると、すでに必要なファームウェアが付属しています。管理はRemoteXYを介して行われ、ATコマンドには特定のファームウェアバージョンが必要です。 ATコマンドを介して制御する適切なファームウェアを見つける問題は、RemoteXYガイドの助けを借りてのみ解決されました。ちなみに、私は広告ではなく、インターフェイスが好きでしたが、それから初めて、あらゆる種類のIoTに対してより便利でよく発達したインターフェイスデザイナーを見つけました。



ファームウェアが正常に動作した後、作業に必要なコンポーネントをモジュールに取り付け、便利な接続のためにUSBコネクタをはんだ付けしました。 USBメイトを通常のボトルの下からコルクに統合し、ケースを変更する機能（ボトルの交換）を備えたシンプルな有線アンテナを得ました。



ファームウェア以外にも問題がありました。

ESP-01ボードは、5 Vではなく3.3 Vで動作する必要があります。さらに、ロジックと電源の両方で動作します。レベルコンバーターを介してロジックを構成した場合、私は怠惰すぎて電力を台無しにせず、小さなラジエーターの子をチップに貼り付けました。 5ボルトから正常に動作しますが、非常に熱くなります。最後のラジエーターは、チップを燃やさないようにします。



もう1つの問題-シールドコネクタの理想的なケーブルをピックアップしましたが、シールド付きのピンは2つしかありませんが、アンテナの場合は4つ必要です（アンテナとオンボードのArduino間の通信用の電源とRXおよびTX）。

ESP + Arduinoが機能するためには共通のグラウンドが必要だったため、アンテナを個別に起動するだけでは不可能であることがわかりました。ケーブルの近くのアースとしてシールドを使用し、アンテナ自体に別のバッテリーを追加する必要がありました。不便ですが、動作します。もちろん、4コア用のケーブルを見つけて、潜水艦から電池をアンテナに給電する方が簡単です。



写真では、ケーブル、シリコンチューブ、およびシールドコネクタの圧着穴の直径がよく一致しています。

管理とファームウェア


管理はスマートフォンからのインターフェースを介して行われます。インターフェースは直接サイトで既製のモジュールで構成され、インターフェースのソースコードを受け取り、インターフェースのさまざまな要素をファームウェア内のアクションにバインドするだけでした。



インターフェースのソースコードを受け取る前に、Arduinoが相互作用する無線モジュールのタイプを設定で指定する必要があります。 Arduino自体をフラッシュするだけです。Wi-Fiモジュールを使用すると、ATコマンドを介して通信が自動的に行われます。アクセスポイントが作成され、スマートフォンからアクセスポイントに接続され、プリインストールされたアプリケーションを介して制御されます。インターフェースはArduinoから提供され、ファームウェアに接続され、スマートフォンのアプリケーション自体によって認識されます。

これは私の最初のコードでした。基本的なプログラミングスキルと基本的な数学のみを使用するので、ここに残しません。私にとっていくつかの困難な瞬間がありました-初めての通常の論理操作を行うことができなかったため、特定の値では、シリンジのサーボドライブが一方向の動きに対してブロックされました。

たとえば、最大の取水量になると、ピストンは後方に停止しますが、前方にブロックしてはなりません。また、その逆の場合、すべての水が絞り出されると、ピストンは前進しないはずですが、問題なく戻りコマンドを実行します。

  if ((RemoteXY.button_1 == 1) && (RemoteXY.button_2 == 0) && (val_f < 100)) {
    pwm_UP = 1;
    pwm_DOWN = 0;
  }

  else if ((RemoteXY.button_1 == 0) && (RemoteXY.button_2 == 1) && (val_f > 25)) {
    pwm_UP = 0;
    pwm_DOWN = 1;
  }
  else {
    pwm_UP = 0;
    pwm_DOWN = 0;
  }

これは、結局のところ、論理的な構成です。RemoteXY.button_＃は、ダイビングまたは浮上のためのインターフェースのボタンです。

また、コードの私にとって難しい部分から、これはレンジファインダーフィルターです（私はネットワークで最も単純なものの1つを採用しました）、電圧計の値を設定します。レンジファインダーの前述のFASTモードのためにフィルターが必要でした、入力値が大幅にジャンプし、フィルターはそれに対処するのに役立ちました。しかし、電圧計はバッテリーの放電を示すのに役立ちました。 Arduinoには基準ピンがあり、1.1ボルト以下の電圧が印加されている場合、Arduinoはこのピンに供給される電圧を正確に決定できます。分圧器の後で8.4 Vを1.1 Vに変換します。この変換は不正確であることが判明し、ファームウェアに変数を追加して電圧値を実験的に修正する必要がありました。

テスト中


テストは、比較的きれいな水で放棄された採石場で行われました。テストでは、カメラを固定し、潜水艦のトリムを調整する必要がありました（基本的な中立浮力とともに）。

最初のタスクは、カメラマウントに適したネジを取り付けるだけで解決しました。カメラの回転を避けるために、小さな粘土が追加されました。



トリマーはバッグによって支配されていましたが、それはクランプに引っ掛けるのに便利であることがわかりました、そしてすでにクランプは潜水艦に沿って簡単に動かすことができます。バッグの中のナットの数によって、中立の浮力を設定し、次に、潜水艦が鼻を噛まないように、クランプの位置をすばやくピックアップしました。このオプションの決定は、採石場に行く直前に行われました。自動トリムシステムを作成する時間が残っていなかっただけです。理論的には、ねじ山のあるロッドに沿って荷重を移動することで、非常に簡単に作成できます。次の潜水艦では、まさにこのオプションを試します。ここはおそらく、潜水艦全体です。
