参照メモリタイプ


データはJavaScriptにどの程度正確に保存されますか？多くのプログラミングコースは、古典から説明を始めます。変数は、データを格納する一種の「ボックス」です。動的型付けを使用する言語にとって、どちらが重要ではないように見えるか：インタープリター自体がデータ型を「飲み込み」、必要に応じて型を動的に変更します。変数の型とそれらの処理方法について考える必要はありません。もちろん、どちらが間違っているかということなので、今日の議論から、脱落することが多い機能について説明します。変数がJavaScriptに格納される方法-プリミティブ（コピー）の形式またはリンクの形式で。

すぐに原始の形式で保存することができる変数の種類を示しています。このboolean、null、undefined、Number、String、Symbol、BigInt。このデータ型で個別に宣言された変数に遭遇した場合、初期初期化中にメモリセルが作成され、値によって割り当て、コピー、転送、返されることを覚えておく必要があります。

JavaScriptの残りの部分は、参照されるメモリ領域に依存しています。彼らは何のために必要ですか？言語の作成者は、メモリが可能な限り経済的に使用される言語を作成しようとしました（これは当時、まったく新しいものではありませんでした）。例として、3人の新しい職場の同僚の名前を覚えておく必要があると想像してください。まったく新しい名前であり、比較を強化するために、インドまたは中国の新しい同僚があなたに変わった名前を付けています。今、あなたの同僚はあなたとあなたの学校の2人の親友と同じように呼ばれていると想像してください。どのような状況で覚えやすいでしょうか？ここでは、人とコンピュータのメモリは同じように動作します。具体的な例をいくつか示します。

let x = 15; //  x
x = 17;//   
console.log(x)//   
//    
let obj = {x:1, y:2} //  
let obj1 = obj; //  obj  obj1
obj1.x = 2; //    ""
console.log(obj1.x); //  ,  
console.log(obj.x) //      obj.x ?

したがって、インタビューで同様のタスクに遭遇した場合は、目の前にあるデータのタイプ-どこでどのように値を取得したのか、プリミティブタイプとして、またはリファレンスとして、すぐに理解するようにしてください。



コンテキスト作業


JSでコンテキストがどのように機能するかを正確に理解するには、いくつかの点を検討する必要があります。

  1. グローバル/ローカルレベルの可視性。
  2. グローバル/ローカルスコープで変数を初期化するときのコンテキストの違い。
  3. 矢印機能。

昔、ES5に戻ると、すべてが非常に簡単でした。varを使用した変数宣言だけがあり、プログラムフローで宣言された場合、グローバルと見なされました（つまり、変数は、windowまたはなどのグローバルオブジェクトのプロパティとして割り当てられていましたglobal）。そして、彼らはに来たシーンletとconst多少異なる動作をする：彼らはグローバルオブジェクトに割り当てられていない、とは異なるメモリに格納され、ブロックスコープに焦点を当てました。現在、varはすでに使用されていないと見なされています。varを使用すると、グローバルスコープの目詰まりが発生する可能性があり、letさらに予測しやすくなります。

データ型と適用対象


すぐに言いましょう：配列は本質的にオブジェクトであり、JavaScriptではこれはオブジェクトの最初のバリエーションではありません-Map、WeakSet、Set、およびコレクションはこれを確認します。

したがって、配列はオブジェクトであり、JSにおける通常のオブジェクトとは異なる、索引付けの最適化を介して、第二のArray.prototypeを連続にまず第一に大きな速度で、Bを提供oを「メソッドのlshy組、それ兄Object.prototype。"

console.log(typeof({}))
console.log(typeof([]))
console.log(typeof(new Set))

次は、データ型の奇妙なキューですnull。JavaScriptにnullのデータの種類を尋ねると、かなり明確な答えが得られます。ただし、ここではいくつかのトリックがないと実行できません。

let x = null;
console.log(typeof(x));
//! , null   objet, ?

//   !      )

null特別なデータ型とは 何かを覚えておく価値があります。ただし、前の例の最初は厳密に別のデータ型を指しています。この特定の型が言語に追加された理由をよりよく理解するには、C ++またはC＃構文の基本を探る価値があるように思えます。

そしてもちろん、インタビューではしばしばそのようなタスクに遭遇しますが、その特異性は動的型付けに関連しています：

console.log(null==undefined);//true
console.log(null===undefined);//     false

JSで比較する場合、多くのトリックが型キャストに関連付けられています。物理的にすべてをここにもたらすことはできません。「What the hell JavaScript」を参照することをお勧めします。



開発プロセス中に言語に残された非論理的な機能


行の追加。実際、数字を含む文字列を追加することは、言語の開発におけるエラーに起因するものではありませんが、JavaScriptのコンテキストでは、これは十分に論理的でないと見なされる有名な例につながりました：

codepen.io/pen/?editors=0011

let x = 15;
let y = "15";
console.log(x+y);//  ""
console.log(x-y); //

plusが単に数値を含む行を追加するという事実は比較的非論理的ですが、覚えておく必要があるだけです。Web開発で広く使用されている他の2つのインタプリタ言語（PHPとPython）は、文字列と数値を追加してそのようなトリックをスローせず、そのような操作ではるかに予測どおりに動作するため、これは特に珍しいことがあります。

同様の例はあまり知られていません。たとえば、NaNの場合：

    console.log(NaN == NaN); //false
    console.log(NaN > NaN); //false
    console.log(NaN < NaN);  //false …   ... ,      NaN?
    console.log(typeof(NaN)); // number

たとえば、型チェックを誤って構成した場合、NaNはしばしば不愉快な驚きをもたらします。

このエラーはIEEE 754形式に関連しているため、0.1 +0.2の例ははるかに有名です。たとえば、このような「数学的な」Pythonでも使用されています。
