差別的なプライバシー：ライブラリの比較



この記事は、実用的な応用比較です。差分プライバシーの詳細をすべて説明しているわけではありません。この質問をより深く学習したい場合は、記事の最後にあるリンクを参照してください。

差分プライバシー（より正確には、ε-差分プライバシー）の約束は、プライベートデータセット内のデータを公​​的に集計するときに、データのプライバシーと正確性のバランスをとる測定可能な方法を提供することです。

設定スライダーがあるとしましょう：

  • それを左に持っていく-データがデータセットにある人々の完全なプライバシーを保ちながら、同時に情報に多くのノイズを追加し、集計統計の正確さを低下させます。
  • それを右に持っていけば、あなたは総統計の完全な正確さを得るが、データセットから人々についての個人情報を明らかにする。

つまり、スライダーはスケールのどこにでも移動でき、特定のユースケースのバランスを選択できます。

データの変換方法


ε差分プライバシーでは、ε（イプシロン）の正の値を使用して、プライバシーと精度のバランスを見つけることができます。 εが小さい場合、プライバシーは維持されますが、精度が低下します。 εが大きいと、正確さのためにプライバシーが損なわれます。 εの値は0から無限大まで変化します。

差分プライバシーライブラリではさまざまな方法が使用され、イプシロンパラメーターを入力値として取り、元のデータセットのε に反比例するランダムノイズを追加します。つまり、イプシロンが小さいほど多くのノイズが追加されます。

一部のライブラリは、追加のパラメーターを使用し、ランダムなノイズ制御ツールを提供します。たとえば、乱数が取得される確率密度（ラプラス分布、正規分布など）です。

いくつかのライブラリもの概念を実装し、プライバシーの予算（プライバシーの予算を）：関数が呼び出されるたびに、ライブラリーは、ユーザーが事前に割り当てられた予算のセットの量を使用しています。理論的根拠は次のとおりです。新しいデータが公開されるたびに、攻撃者がデータセットから人物に関する情報を抽出する可能性が高くなります。プライバシーバジェットの助けを借りて、ライブラリは値ではなくエラーを返す場合があります。

差分プライバシーライブラリの比較


同じ方法を使用して、3つのライブラリとその作業を特定のデータセットと比較します。

  • IBM /差分プライバシーライブラリ（Python）
  • google /差分プライバシー（C ++）
  • ブルビンスタイン/ diffpriv（R言語）

データセットのサイズとプライバシーの望ましいレベル（イプシロン）が精度にどのように影響するかを見ていきます。それぞれのケースで、異なるライブラリで得られた結果を比較します。

テストデータセット


人の体重がキログラムである列を含むデータセットをランダムに生成します。この情報は機密情報と見なされます。秘密にしておく必要があります。各重みは、倍精度実数として表されます。

重量は、データセットの平均重量が70 kgで標準偏差が30である正規分布に従って生成されました。

調査のために、さまざまなサイズのデータ​​セットを生成できるようにします。

重量生成コードは次のとおりです。

import random

def generate_weight_dataset(dataset_size):
    outpath = f"weights_{dataset_size}.csv"
    mu = 70  # mean
    sigma = 30  # standard deviation
    with open(outpath, "w+") as fout:
        for i in range(dataset_size):
            random_weight = random.normalvariate(mu, sigma)  # normal distribution
            random_weight = abs(random_weight)  # make sure weights are positive
            random_weight = round(random_weight, 1)  # round to 1 decimal digit
            line = f"{random_weight}"
            print(line, file=fout)

ライブラリの使用


ライブラリを使用する方法を見てみましょう。常に同じパラメーターを使用してそれらを比較します。たとえば、すべてのケースで同じ入力イプシロン値があります。測定には、データセットの平均値を使用します。考慮されるすべてのライブラリは、ランダムノイズ追加メカニズムを使用して、平均、偏差、合計などの操作を実装します。多くの場合、ラプラスメカニズムを使用しました。平均値の検索操作は任意に選択されるため、同じメカニズムを使用してすべての操作をテストできます。

IBMライブラリー（Python）を使用して、差分プライバシーの平均値を計算します。

import diffprivlib.tools.utils

def dp_mean(weights, epsilon, upper):
    dp_mean = diffprivlib.tools.utils.mean(weights, epsilon=epsilon, range=upper)
    return dp_mean

upper = max(weights)
epsilon = 1.0
dpm = dp_mean(weights, epsilon, upper)

Googleライブラリと同じ（C ++）：

double dp_mean(std::list<double> weights, double epsilon, double lower, double upper) {
    std::unique_ptr<differential_privacy::BoundedMean<double>> mean_algorithm = 
    differential_privacy::BoundedMean<double>::Builder()
    .SetEpsilon(epsilon)
    .SetLower(0.0)
    .SetUpper(upper)
    .Build()
    .ValueOrDie();

    for (auto weight : weights) {
        mean_algorithm->AddEntry(weight);
    }
    double privacy_budget = 1.0;
    auto output = mean_algorithm->PartialResult(privacy_budget).ValueOrDie();
    return differential_privacy::GetValue<double>(output);
}

Googleライブラリの使用ごとに、1.0のプライバシー予算全体を使用することに注意してください。また、IBMライブラリーはこのパラメーターを受け入れません。

次に、diffpriv（言語R）を使用してDPの平均値を計算します。

library(diffpriv)

dp_mean <- function(xs, epsilon) {
  ## a target function we'd like to run on private data X, releasing the result
  target <- function(X) mean(X)

  ## target seeks to release a numeric, so we'll use the Laplace mechanism---a
  ## standard generic mechanism for privatizing numeric responses
  n <- length(xs)
  mech <- DPMechLaplace(target = target, sensitivity = 1/n, dims = 1)

  r <- releaseResponse(mech, privacyParams = DPParamsEps(epsilon = epsilon), X = xs)
  r$response
}

プライバシーが精度に与える影響


平均値の精度は、平均DPと実際の平均との差を計算するだけで測定できます。

イプシロンが変化したときに平均重量の精度がどうなるか見てみましょう。ランダムノイズを追加しているので、100回実行して平均二乗誤差を計算し、ライブラリが歪んだ値を生成しているかどうかを確認します。

結果は、誤差（実際の平均と平均DPの差）がイプシロンに反比例することを示しています。イプシロンが大きい場合、エラーは小さく、逆も同様です。私はe ^ -10からe ^ 10の範囲のイプシロン値をテストしました（対数目盛に注意してください）。

テストでは、10,600行の任意の一定サイズのデータ​​セットを使用しました。

標準誤差



実際、イプシロンが増加するとエラーは減少します。興味深いことに、イプシロン値が非常に小さい場合、Googleライブラリには標準エラー制限があります。IBMおよびdiffprivライブラリでは、これは観察されません。

データセットのサイズが精度に与える影響


DPの平均値の精度への影響について話しています。

小さなデータセットのリスクの1つは、個人が集計された値（平均）に大きな影響を与えることです。したがって、データセットに人物が1人しかいない場合、完全に正確な平均値はその人物の正確な体重と等しくなります。データセットのサイズを変更するときに、ライブラリが個々の情報の開示を回避するのにどのように役立つかを見てみましょう。

イプシロン1.0の任意の定数値を使用します。

標準誤差



データセットのサイズが小さくなると、平均二乗誤差が大きくなり、その逆も同様です。期待通りでした。人が少ない場合は、個人データの開示を避けるためにノイズを追加したいと思います。短所は、データセットが小さい場合、精度が非常に低いため、集計値が完全に無関係になる可能性があることです。

diffprivの場合と同様に、標準エラーはデータセットのサイズの変化により大きく依存しています。ただし、他の2つのライブラリと同様に、パターンは依然として顕著です。データセットが大きくなると、平均二乗誤差が減少します。ただし、これは最大約30,000行のデータセットにのみ当てはまります。その後、エラーはほとんど変わりません。また、データセット17 912のサイズによるエラーのサイズの異常な低下にも注意してください。

繰り返しになりますが、diffprivは一般的に標準偏差が低くなっています。

結論


差別化されたプライバシーの3つの異なるライブラリで講じられた対策は、当初の期待に沿うものでした。ただし、実装が異なると、特に境界の場合、イプシロン値が非常に小さい場合やGoogleライブラリの場合と同様に、結果が異なります。実際には、これは干渉しないはずです。ユーザーが非常に大きくて非常に小さなデータセットや非常に小さなイプシロン値を使用しないためです。ただし、このような状況では、ライブラリが警告を発行する場合があります。

diffprivの動作は他の2つのライブラリと著しく異なるため、イプシロン値を慎重に選択することをお勧めします。
