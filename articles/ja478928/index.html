フィールド800x640。セルサイズは10x10です。

class Square(object):
    def __init__(self, canvas, x, y, size, main):
        self.canvas = canvas
        self.cel = self.canvas.create_rectangle(x, y, x + size, y + size, fill='#EEEEEE')
        self.main = main
        self.color_change = True
        self.canvas.tag_bind(self.cel, '<ButtonPress-1>', self.paint)



そのような小さなフィールドは完全に一杯になる時間はありませんでした。約600のセルがすばやく更新されましたが、セルサイズを4に減らすとすぐに、セルの状態の変化率とインターフェイスの応答が低下しました。

Squareクラスを削除して、画像からピクセルの描画を開始しました。インターフェイスは正常に機能しましたが、200セルを超えると、すべてが非常にゆっくりと更新されました。アルゴリズムに問題が見つかりました。1ピクセルより遅い4ピクセルのセルを描画します。

def fill_black(self, x, y, color):
        for i in range((-self.cell // 2) - 1, (self.cell // 2) - 1):
            for j in range((-self.cell // 2) - 1, (self.cell // 2) - 1):
                xx = (x + i) % self.can_width
                yy = (y + j) % self.can_height
                self.cell_matrix.put(color, (xx, yy))

修正後、通常の速度で600セル、画面上に非常に小さな数字が表示されました。「黒いピクセル」の画像のチェックをやめ、システムの状態をマトリックスに保存し始めたとき、なんとか速度を900セルに引き上げました。

def fill_black(self, coords, color, virt_color):
    x, y = coords
    self.virt_mat[x][y] = virt_color
    self.cell_matrix.put(color, (x, y))

スピード

Lua + Love2dでLifeを作成しました。私は言語とエンジン関数を使用しました。

function Life:draw()
    love.graphics.setCanvas(self.canvas)
    love.graphics.clear()
    for y=1,self.rows do
        for x=1,self.cols do
            if self.cells[y][x]==1 then
                love.graphics.setColor(self.color)
                love.graphics.points(x-1,y-1)
            end
        end
    end
    love.graphics.setCanvas()
    love.graphics.setColor(set.WHITE)
    love.graphics.draw(self.canvas,self.x,self.y)
end

111735生細胞の場合、システムは18 fpsで動作します。
