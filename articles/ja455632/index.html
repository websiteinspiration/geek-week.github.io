注意！Visual Studio 2017でのC ++ std :: map :: mergeおよびstd :: set :: mergeの実装における危険なバグ

MS Visual Studio 2017でC ++ 17標準を使用する場合-注意してください。現在のバージョンには、std :: map :: mergeおよびstd :: set :: mergeの実装に重大なバグがあります。詳細-カットの下。

バグはどのように現れますか？


  1. 標準のN * log（サイズ（）+ N）で宣言されたものではなく、std :: map :: mergeおよびstd :: set :: mergeの複雑さ（Nは追加された部分のサイズ）は、約Nの2乗になります。
  2. マージの助けを借りて、要素が十分に多いコンテナが追加された場合、結果のコンテナが破棄されると、スタックオーバーフローが発生します。
  3. マージの実行後、コンテナーは誤った状態になるため、他の症状が発生する可能性があります。

マイクロソフトは何と言っていますか？


バグレポートは、2か月前に私からマイクロソフトに送信されました。
Visual Studio 2019 Update 2では、プレビュー2が修正されているはずです。

しかし、現在のバージョンのVisual Studio 2017 15.9.12ではこれまでのところ修正されておらず、最新のレポートで判断すると、待機に長い時間がかかります...

バグは、赤黒ツリーの実装に追加されたノードの誤ったカラーマーキングにあります。

再現するには？


//#include "pch.h"

#include <chrono>
#include <string>
#include <iostream>
#include <map>
#include <set>


const size_t mainSize = 50'000;
const size_t additionSize = mainSize;


auto getMainMap()
{
  std::map<size_t, std::string> result;
  while( result.size() < mainSize )
    result.emplace(result.size(), "SomeText");

  return result;
}


auto getAdditionMap()
{
  std::map<size_t, std::string> result;
  while ( result.size() < additionSize )
    result.emplace(mainSize + result.size(), "SomeAdditionText");

  return result;
}


int main()
{
  {
    auto mainMap = getMainMap();
    auto addition = getAdditionMap();

    auto start = std::chrono::steady_clock::now();
    for ( auto const &elem : addition )
      mainMap.emplace(elem);
    auto end = std::chrono::steady_clock::now();

    std::cout << "manual insertion with copy: "
              << std::chrono::duration_cast<std::chrono::microseconds>(end - start).count()
              << std::endl;
  }

  {
    auto mainMap = getMainMap();
    auto addition = getAdditionMap();

    auto start = std::chrono::steady_clock::now();
    mainMap.merge(addition);
    auto end = std::chrono::steady_clock::now();

    std::cout << "std::map::merge: "
              << std::chrono::duration_cast<std::chrono::microseconds>(end - start).count()
              << std::endl;
  } // <---- and stack overflow here because of incorrect mainMap after merge

  return 0;
}

mainSizeの値を変更すると、異なる結果が得られる可能性があります-実行が遅いか、クラッシュするだけです。

何をすべきか？


コードを修正し、マージ呼び出しを手動挿入に置き換えます。または、VS 2019に切り替えます。
