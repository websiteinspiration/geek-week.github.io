エフェメラルコンテナーKubernetesの最初のステップ

Ephemeral Containers Kubernetesは、実行可能バイナリーに加えて何も含まれていないコンテナーの問題をデバッグするための興味深いアイデアです。

一部の情報はKubernetesのウェブサイトで入手でき、いくつかの例は広大な場所で見つけることができます。

この関数は実験的なものであり、cuberberのアルファ版ベータ版の説明の荒野では理解できませんでした。
Kubernetes v1.16.3でテストされました。

一時的なコンテナーを囲炉裏で開始できるようにするには、対応する機能を/機能ゲートを介してアクティブ化する必要があります。

一般的に、機能ゲートは、キューブの必要なコンポーネントを起動するためのオプションです（psを介して実行中のプロセスのパラメーターを調べます）。このパラメーターを指定して起動されたキューブレットだけでは不十分である可能性があります。そうでない場合は、キューブの対応するコンポーネントで異なる機能をアクティブ化する必要があります。

キューブの使用モードに応じて、次のオプションがあります

1。
minikube start --feature-gates="EphemeralContainers=true"

2.セクションに追加してクラスターを拡張します

apiVersion: kubeadm.k8s.io/v1beta2
    kind: InitConfiguration
    localAPIEndpoint: {}
    nodeRegistration:
      kubeletExtraArgs:
        "feature-gates": "EphemeralContainers=true"
---
apiVersion: kubeadm.k8s.io/v1beta2
    kind: ClusterConfiguration
    apiServer:
      extraArgs:
        "feature-gates": "EphemeralContainers=true"
    scheduler:
      extraArgs:
        "feature-gates": "EphemeralContainers=true"
    controllerManager:
      extraArgs:
        "feature-gates": "EphemeralContainers=true"

3.稼働中のクラスター

/var/lib/kubelet/kubeadm-flags.env

パラメータに追加 

--feature-gates=EphemeralContainers=true

それを次のようにする 

KUBELET_KUBEADM_ARGS="--cgroup-driver=cgroupfs --feature-gates=EphemeralContainers=true --network-plugin=cni --pod-infra-container-image=k8s.gcr.io/pause:3.1"

サービスkubeletの再起動を再開します

/etc/kubernetes/manifests/kube-apiserver.yaml

次の変数をcmdリストの最後に追加します。

spec:
  containers:
  - command:
    - --feature-gates=EphemeralContainers=true

同様に、私たちは支配します

kube-scheduler.yaml    
kube-controller-manager.yaml

後 

kubectl get pods -A

名前空間kube-systemで、名前を含むポッドを探します 

kube-apiserver
kube-scheduler   
kube-controller-manager

これらのポッドはkubectl delete podを介して削除
され、修正されたファイルのパラメーターを使用して再作成されます。

4.理論的には、別のオプションがありますが、私にはうまくいきませんでした

kubeadm upgrade plan  --feature-gates EphemeralContainers=true

ポッドコンテナへの接続を試みます。

クーバーのサイトからの小さなコピー。

example-podが穏やかに接続するポッドの名前に置き換えられたec.jsonファイルを作成します。イメージでは、デバッグツール（ubuntuなど）を使用してコンテナーを記述します。

{
    "apiVersion": "v1",
    "kind": "EphemeralContainers",
    "metadata": {
            "name": "example-pod"
    },
    "ephemeralContainers": [{
        "command": [
            "sh"
        ],
        "image": "busybox",
        "imagePullPolicy": "IfNotPresent",
        "name": "debugger",
        "stdin": true,
        "tty": true,
        "terminationMessagePolicy": "File"
    }]
}

接続するカウンターのプロパティにパッチを適用し、名前空間を忘れないでください

kubectl replace --raw /api/v1/namespaces/<b>default</b>/pods/<b>example-pod</b>/ephemeralcontainers  -f ec.json

コンテナーにシェルを接続して実行します。

kubectl attach -it example-pod -c debugger

「ps auxww」を作成すると、面白いことは何も起こりません。共有プロセス名前空間のサポートが必要です。
