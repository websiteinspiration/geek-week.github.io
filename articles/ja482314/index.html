PatroniなどのHAProxyを使用した高可用性PostgreSQLクラスターの構築

たまたま問題が発生したとき、私はこのソリューションを開発して実行するだけの十分な経験がありませんでした。それからグーグルを始めました。


何が問題なのかはわかりませんが、チュートリアルのようにすべてを段階的に実行しても、作成者と同じ環境を準備しても何もうまくいかないという事実に遭遇するのは初めてです。何が問題なのかはわかりませんが、これに遭遇したとき、私は決定しました。すべてがうまくいったら、チュートリアルを書きます。間違いなく機能するもの。


オンラインガイド


たまたま、インターネットは、さまざまなガイド、チュートリアル、ステップバイステップなどの欠如に悩まされていませんでした。偶然にも、私はフェイルセーフなPostgreSQLクラスターの便利な構成と構築のためのソリューションの開発を任されました。その主な要件は、マスターサーバーからすべてのレプリカへのレプリケーションのストリーミングと、マスターサーバーに障害が発生したときの予約の自動入力でした。


この段階で、使用されるテクノロジーのスタックが定義されました。


  • DBMSとしてのPostgreSQL
  • クラスタリングソリューションとしてのPatroni
  • Patroniの分散ストレージとしてのetcd
  • データベースを使用するアプリケーションの単一のエントリポイントを整理するためのHAproxy

取り付け


注意してください-PatroniなどのHAProxyを使用して高可用性PostgreSQLクラスターを構築します。


すべての操作は、Debian 10がインストールされた仮想マシンで実行されました。


etcd


etcdではディスクの負荷が非常に重要であるため、patroniとpostgresqlが配置されるのと同じマシンにetcdをインストールすることはお勧めしません。しかし、トレーニングの目的のために、それだけを行います。
etcdをインストールします。


#!/bin/bash
apt-get update
apt-get install etcd

/ etc / default / etcdファイルにコンテンツを追加します

[member]


ETCD_NAME=datanode1 # hostname
ETCD_DATA_DIR="/var/lib/etcd/default.etcd"


ALL IP ADRESSES SHOULD BE VALID. LISTER PEER, CLIENT etc SHOULD BE SET TO IP ADDRESS OF HOST


ETCD_LISTEN_PEER_URLS="http://192.168.0.143:2380" #
ETCD_LISTEN_CLIENT_URLS="http://192.168.0.143:2379,http://127.0.0.1:2379" #


[cluster]


ETCD_INITIAL_ADVERTISE_PEER_URLS="http://192.168.0.143:2380" #
ETCD_INITIAL_CLUSTER="datanode1=http://192.168.0.143:2380,datanode2=http://192.168.0.144:2380,datanode3=http://192.168.0.145:2380" # etcd
ETCD_INITIAL_CLUSTER_STATE="new"
ETCD_INITIAL_CLUSTER_TOKEN="etcd-cluster-1"
ETCD_ADVERTISE_CLIENT_URLS="http://192.168.0.143:2379" #


コマンドを実行する


systemctl restart etcd
