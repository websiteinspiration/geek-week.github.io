Pythonを使用してMicrosoft Wordでドキュメントに入力します。パート1

私たちは受益者に関する情報を入手する義務を果たします


小さな紹介


2016年12月21日から、ロシア連邦の連邦法「犯罪の収益の合法化（ロンダリング）とテロ資金の調達に対抗すること」の改正が、その有権者所有者に関する情報を開示する法人の義務に関して発効しました。この点で、多くの企業は彼らの有利な所有者を見つけるために所有権チェーンに沿って問い合わせを送信します。誰かが紙で要求を出し、誰かがメールを送信します。

私たちの意見では、「Know Your Beneficial Owner」の義務が履行されていることの適切な証拠は、送信/配信についてのメモが付いた紙の手紙の存在です。これらの手紙は、少なくとも年に1回は作成するのが理想的です。弁護士が少数の会社しかない場合、手紙を書くことは難しくありません。しかし、3ダース以上の企業がある場合、手紙を書くことは消滅的な前向きなルーチンに変わります。手紙の詳細が絶えず変化しているという事実によって問題は悪化します：署名者は終了し、企業は再登録し、住所を変更します。このすべてを考慮に入れる必要があります。ここでPythonプログラミングスキルはどのように役立ちますか？

それは非常に単純です-それ自体が必要な詳細を文字で置き換えるプログラムがあるといいでしょう。手動で文書の後に文書を作成することを強制せずに、文字自体を形成することを含みます。試してみよう。

単語の文字の構造。Python docxtplモジュール


プログラムコードを記述する前に、データを配置するレターテンプレートがどのようになるかを見てみましょう。

会社からその参加者/株主への手紙のテキストは、おおよそ次のようになります



。作業の原理を理解するために、テンプレートの1つのフィールドに入力する簡単なプログラムを作成します。

まず、Wordのレターテンプレート自体に、署名者などのフィールドの1つではなく、変数を設定します。この変数は英語である必要があります。言語、またはロシア語ですが、1つの単語で記述します。また、変数は必ず二重中括弧で囲む必要があります。これは次のようになります



。プログラム自体は次の形式になります。

from docxtpl import DocxTemplate
doc = DocxTemplate(".docx")
context = { 'director' : ".."}
doc.render(context)
doc.save("-final.docx")

まず、Word形式のドキュメントを操作するためのモジュールをインポートします。次に、テンプレートを開き、テンプレート自体で先に示したディレクターのフィールドに、ディレクターの名前を入力します。最後に、ドキュメントは新しい名前で保存されます。

したがって、Wordテンプレートファイルのすべてのフィールドに入力するには、まず、テンプレート自体のすべての入力フィールドを変数と共に角かっこ{}で定義し、プログラムを記述する必要があります。コードは次のようになります。

from docxtpl import DocxTemplate
doc = DocxTemplate(".docx")
context = { 'emitent' : ' ', 'address1' : '. , . , . 0', '': ' ', '_': '. , . , . 0', 'director': '.. '}
doc.render(context)
doc.save("-final.docx")

出口で、プログラムを実行すると、完成した完成したドキュメントを受け取ります。
