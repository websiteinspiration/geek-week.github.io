Windows PowerShellとは何で、何を食べますか？パート4：オブジェクト、カスタムクラスの操作



PowerShellインタープリターウィンドウのコマンドのテキスト出力は、人間が読める形式で情報を表示するための方法にすぎません。実際、環境はオブジェクトの操作に重点が置かれています。コマンドレットと関数は入力でオブジェクトを受け取り、出力で返します。対話モードとスクリプトで使用できる変数のタイプは.NETクラスに基づいています。サイクルの4番目の記事では、オブジェクトの操作について詳しく説明します。

オブジェクトは、データフィールド（プロパティ、イベントなど）とその処理メソッド（メソッド）のコレクションであることを思い出してください。その構造はタイプによって決まります。タイプは通常、.NET Core統合プラットフォームで使用されるクラスに基づいています。 COM、CIM（WMI）およびADSIオブジェクトを操作することもできます。プロパティとメソッドは、データに対してさまざまなアクションを実行するために必要です。PowerShellに加えて、オブジェクトを引数として関数やコマンドレットに渡し、値を変数に割り当てることができます。また、コマンドを構成するためのメカニズム（パイプラインまたはパイプライン）もあります。パイプライン内の各コマンドは、オブジェクトごとに出力を次のコマンドに渡します。処理には、コンパイルされたコマンドレットを使用するか、独自の高度な関数を作成できますパイプライン内のオブジェクトを使用して、フィルタリング、並べ替え、グループ化、さらには構造の変更など、さまざまな操作を実行します。この形式でデータを転送することには重大な利点があります。受信側のチームはバイト（テキスト）のストリームを解析する必要がなく、対応するプロパティとメソッドを参照することで必要なすべての情報を簡単に抽出できます。

オブジェクト構造を表示する


たとえば、システムで実行されているプロセスに関する情報を取得できるGet-Processコマンドレットを実行し



ます。返されたオブジェクトとそのメソッドのプロパティがわからない、フォーマットされたテキストデータが表示されます。出力を微調整するには、オブジェクトの構造を調べる方法を学ぶ必要があります。Get-Memberコマンドレットがこれを支援します。

Get-Process | Get-Member



ここでは、タイプと構造がすでに表示されており、追加のパラメーターを使用して、たとえば、入力を入力したオブジェクトのプロパティのみを表示できます。

Get-Process | Get-Member -MemberType Property

この知識は、管理タスクを対話的に解決したり、独自のスクリプトを作成したりするために必要になります。たとえば、応答プロパティによってハングしたプロセスに関する情報を取得します。

オブジェクトのフィルタリング


PowerShellでは、特定の条件を満たすオブジェクトをパイプ処理できます。

Where-Object {   }

演算子の括弧内のスクリプトブロックの実行結果は、論理値である必要があります。true（$ true）の場合、Where-Objectコマンドレットに入ったオブジェクトはパイプラインに渡され、それ以外の場合（値$ false）は削除されます。たとえば、停止しているWindows Serverサービスを一覧表示します。StatusプロパティがStoppedに設定されているもの：

Get-Service | Where-Object {$_.Status -eq "Stopped"}



ここでもテキスト表現を確認していますが、パイプラインを通過するオブジェクトのタイプと内部構造を理解したい場合は簡単です。

Get-Service | Where-Object {$_.Status -eq "Stopped"} | Get-Member



オブジェクトの並べ替え


オブジェクトをパイプライン処理する場合、オブジェクトを並べ替える必要があることがよくあります。プロパティ名（並べ替えキー）はSort-Objectコマンドレットに渡され、値で並べ替えられたオブジェクトを返します。実行中のプロセスの出力は、費やされたプロセッサー時間（cpuプロパティ）によって簡単にソートできます。

Get-Process | Sort-Object –Property cpu

-Propertyパラメーターは、Sort-Objectコマンドレットを呼び出すときに省略できます。これはデフォルトで使用されます。逆ソートの場合は、-Descendingパラメーターを使用します。

Get-Process | Sort-Object cpu -Descending



オブジェクトとそのパーツの選択


Select-Objectコマンドレットでは、-Firstまたは-Lastパラメーターを使用して、パイプラインの最初または最後で特定の数のオブジェクトを選択できます。これにより、単一のオブジェクトまたは特定のプロパティを選択し、それらに基づいて新しいオブジェクトを作成できます。簡単な例を使用してコマンドレットの動作を見てみましょう。

次のコマンドは、最大量のRAM（WSプロパティ）を消費している10個のプロセスに関する情報を表示します。

Get-Process | Sort-Object WS -Descending | Select-Object -First 10



パイプラインを通過するオブジェクトの特定のプロパティのみを選択し、それらに基づいて新しいプロパティを作成できます。

Get-Process | Select-Object ProcessName, Id -First 1

パイプラインの結果として、新しいオブジェクトを取得します。その構造は、Get-Processコマンドレットによって返される構造とは異なります。Get-Memberを使用してこれを確認します。

Get-Process | Select-Object ProcessName, Id -First 1 | Get-Member



Select-Objectは単一のオブジェクト（-First 1）を返すことに注意してください。これには、2つのフィールドしか指定されていません。それらの値は、Get-Processコマンドレットでパイプラインに渡された最初のオブジェクトからコピーされました。Select-Objectを使用すると、PowerShellスクリプトでオブジェクトを作成する方法の1つがベースになります。

$obj = Get-Process | Select-Object ProcessName, Id -First 1
$obj.GetType()



Select-Objectを使用すると、ハッシュテーブルとして表す必要があるオブジェクトに計算プロパティを追加できます。この場合、最初のキーの値はプロパティの名前に対応し、2番目のキーの値はパイプラインの現在の要素のプロパティの値に対応します。

Get-Process | Select-Object -Property ProcessName, @{Name="StartTime"; Expression = {$_.StartTime.Minute}}



パイプラインを通過するオブジェクトの構造を見てみましょう。

Get-Process | Select-Object -Property ProcessName, @{Name="StartTime"; Expression = {$_.StartTime.Minute}} | Get-Member



ForEach-Object、Group-Object、Measure-Object


オブジェクトを操作するための他のコマンドレットがあります。たとえば、最も有用な3つについて説明します

。ForEach-Objectを使用すると、パイプライン内の各オブジェクトに対してPowerShellコードを実行できます。

ForEach-Object {   }

Group-Objectは、プロパティ値によってオブジェクトをグループ化します。

Group-Object PropertyName

-NoElementパラメーターを指定して実行すると、グループ内の要素の数を確認できます。

Measure-Objectは、パイプライン内のオブジェクトのフィールドの値によってさまざまな要約パラメーターを集計します（合計を計算し、最小値、最大値、または平均値も見つけます）：

Measure-Object -Property PropertyName -Minimum -Maximum -Average -Sum

通常、考慮されるコマンドレットは対話的に使用され、スクリプトは多くの場合、Begin、Process、およびEndブロックを使用して関数を作成します。

.NETおよびCOMオブジェクトの作成（New-Object）


システム管理者に役立つ.NET CoreおよびCOMインターフェイスを備えた多くのソフトウェアコンポーネントがあります。System.Diagnostics.EventLogクラスを使用すると、Windows PowerShellから直接システムログを管理できます。-TypeNameパラメーターを指定したNew-Objectコマンドレットを使用して、このクラスのインスタンスを作成する例を見てみましょう。

New-Object -TypeName System.Diagnostics.EventLog



特定のイベントログを指定しなかったため、クラスの結果のインスタンスにはデータが含まれていません。これを変更するには、作成時に-ArgumentListパラメーターを使用して特別なコンストラクターメソッドを呼び出す必要があります。アプリケーションログにアクセスする場合は、文字列「Application」を引数としてコンストラクタに渡します。

$AppLog = New-Object -TypeName System.Diagnostics.EventLog -ArgumentList Application
$AppLog



注：コマンドの出力を$ AppLog変数に保存しました。パイプラインは一般的にインタラクティブモードで使用されますが、スクリプトを作成するには、オブジェクト参照を維持する必要があります。さらに、メインの.NET CoreクラスはSystem名前空間に含まれています。PowerShellはデフォルトで、指定されたタイプを検索します。そのため、System.Diagnostics.EventLogの代わりにDiagnostics.EventLogを書き込むことは非常に正確です。

ジャーナルを操作するには、適切な方法を参照できます。

$AppLog | Get-Member -MemberType Method



アクセス権付きのClear（）メソッドを使用してクリアするとします。

$AppLog.Clear()

New-Objectコマンドレットは、COMコンポーネントの操作にも使用されます。Windowsスクリプティングサーバーに付属のライブラリから、Internet ExplorerなどのActiveXアプリケーションに至るまで、かなりの数があります。COMオブジェクトを作成するには、目的のクラスのプログラム識別子ProgIdで-ComObjectパラメーターを指定する必要があります。

New-Object -ComObject WScript.Shell
New-Object -ComObject WScript.Network
New-Object -ComObject Scripting.Dictionary
New-Object -ComObject Scripting.FileSystemObject

New-Objectを使用して独自のオブジェクトを任意の構造で作成するには、古すぎて扱いにくいように見えます。このコマンドレットは、PowerShellの外部のソフトウェアコンポーネントを操作するために使用されます。以降の記事では、この問題について詳しく説明します。.NETおよびCOMオブジェクトに加えて、CIM（WMI）およびADSIオブジェクトについても学習します。

静的メソッドの呼び出し


一部の.NET Coreクラスのインスタンスを作成することはできません。これらには、System.EnvironmentおよびSystem.Mathが含まれます。これらは静的であり、静的なプロパティとメソッドのみが含まれます。本質的に、これらはオブジェクトを作成せずに使用される参照ライブラリです。型クラス名を角括弧で囲んで、リテラルを通じて静的クラスを参照できます。さらに、Get-Memberを使用してオブジェクトの構造を見ると、System.Environmentの代わりにSystem.RuntimeType型が表示されます。

[System.Environment] | Get-Member



静的要素のみを表示するには、-Staticパラメーターを指定してGet-Memberを呼び出す必要があります（オブジェクトのタイプに注意してください）。

[System.Environment] | Get-Member -Static



静的プロパティとメソッドにアクセスするには、リテラルの後のピリオドの代わりに2つの連続したコロンを使用します。

[System.Environment]::OSVersion

または

$test=[System.Math]::Sqrt(25) 
$test
$test.GetType()



タイプPSCustomObject


PowerShellで使用できる多くのデータ型の中で、任意の構造を持つオブジェクトを格納するように設計されたPSCustomObjectについて言及する価値があります。New-Objectコマンドレットを使用してこのようなオブジェクトを作成することは、古典的ですが面倒で時代遅れの方法と考えられています。

$object = New-Object  –TypeName PSCustomObject -Property @{Name = 'Ivan Danko'; 
                                          City = 'Moscow';
                                          Country = 'Russia'}

オブジェクトの構造を見てみましょう：

$object | Get-Member



PowerShell 3.0以降では、別の構文を使用できます。

$object = [PSCustomObject]@{Name = 'Ivan Danko'; 
                                          City = 'Moscow';
                                          Country = 'Russia'
}

同等の方法のいずれかでデータにアクセスできます。

$object.Name

$object.'Name'

$value = 'Name'
$object.$value

次に、既存のハッシュテーブルをオブジェクトに変換する例を示します。

$hash = @{'Name'='Ivan Danko'; 'City'='Moscow'; 'Country'='Russia'}
$hash.GetType()
$object = [pscustomobject]$hash
$object.GetType()



このタイプのオブジェクトの欠点の1つは、プロパティの順序が変更される可能性があることです。これを回避するには、[ordered]属性を使用する必要があります。

$object = [PSCustomObject][ordered]@{Name = 'Ivan Danko'; 
                                          City = 'Moscow';
                                          Country = 'Russia'
}

オブジェクトを作成するための他のオプションがあります。上記では、Select-Objectコマンドレットの使用について説明しました。要素の追加と削除についてはまだ扱いません。前の例のオブジェクトに対してこれを行うのは非常に簡単です。

$object | Add-Member –MemberType NoteProperty –Name Age  –Value 33
$object | Get-Member



Add-Memberコマンドレットを使用すると、 "-MemberType ScriptMethod"構成を使用して、プロパティだけでなく、以前に作成した$オブジェクトオブジェクトにメソッドを追加することもできます。

$ScriptBlock = {
    #  
}
$object | Add-Member -Name "MyMethod" -MemberType ScriptMethod -Value $ScriptBlock
$object | Get-Member

注：新しいメソッドのコードを保存するために、ScriptBlockタイプの$ ScriptBlock変数を使用しました。



プロパティを削除するには、適切な方法を使用します。

$object.psobject.properties.remove('Name')

独自のクラスを作成する


PowerShell 5.0では、オブジェクト指向プログラミング言語の特徴的な構文を使用してクラスを定義する機能が導入されています。サービスワードClassはこれを目的としています。その後、クラス名を指定し、演算子の括弧でその本体を説明する必要があります。

class MyClass
{
    #  
}

これは、真のタイプの.NET Coreであり、その本体はそのプロパティ、メソッド、およびその他の要素を記述します。簡単なクラス定義の例を考えてみましょう：

class MyClass 
{
     [string]$Name
     [string]$City
     [string]$Country
}

オブジェクト（クラスのインスタンス）を作成するには、New-Objectコマンドレットまたは[MyClass]タイプのリテラルと疑似静的 new メソッド（デフォルトのコンストラクター）を使用します。

$object = New-Object -TypeName MyClass

または

$object = [MyClass]::new()

オブジェクトの構造を分析しましょう：

$object | Get-Member



スコープについて忘れないでください。型名を文字列として参照したり、クラスが定義されているスクリプトやモジュールの外で型リテラルを使用したりすることはできません。同時に、関数は、モジュールまたはスクリプトの外部で使用できるクラス（オブジェクト）のインスタンスを返すことができます。

オブジェクトを作成したら、そのプロパティを入力します。

$object.Name = 'Ivan Danko'
$object.City = 'Moscow'
$object.Country = 'Russia'
$object



クラスの説明では、プロパティタイプだけでなくデフォルト値も設定されていることに注意してください。

class Example
{
     [string]$Name = 'John Doe'
}

クラスメソッドの説明は関数の説明に似ていますが、機能語functionを使用していません。関数と同様に、必要に応じてパラメーターがメソッドに渡されます。

class MyClass 
{
     [string]$Name
     [string]$City
     [string]$Country
     
     # 
     Smile([bool]$param1)
     {
         If($param1) {
            Write-Host ':)'
         }
     }
}

これで、クラスの代表者が笑顔にする方法を理解しました。

$object = [MyClass]::new()
$object.Smile($true)

メソッドはオーバーロードできます。さらに、クラスには静的なプロパティとメソッド、およびクラス自体の名前と同じ名前のコンストラクターがあります。スクリプトまたはPowerShellモジュールで定義されたクラスは、別のクラスのベースとして機能できます。これが継承の実装方法です。同時に、既存の.NETクラスの使用が基本クラスとして許可されています。

class MyClass2 : MyClass
{
      #  ,     MyClass
}
[MyClass2]::new().Smile($true)
