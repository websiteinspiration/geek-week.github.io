Java 14がやってくる



Oracleによると、Java 14のリリースは3月17日に予定されています。リリース日は聖パトリックの日（この日に祝われます）に関連しているかどうかは、すぐにわかりますか。新しいJavaで利用できるイノベーションを見て、喜びと悲しみのどちらでビールを飲むかを決めましょう。

記録


Java 14のプレビューモードで利用できる、根本的に新しい言語機能。主な目的は、コード内の多くの春雨を取り除くことです。レコードは、フィールドにデータを格納するためにのみ使用されるクラスを、説明されている動作なしで置き換える必要があります。Recordを定義すると、コンパイラー自体がコンストラクター、ゲッター、equals（）およびhashCode（）、toString（）メソッドを生成します。どこかですでに見ましたよね、ロンボクさん？エントリは次のとおりです。

public record Person(String name, int age){}

Record をコンパイルするには、jdk14をダウンロードして次のコマンドを入力する必要があります。

javac —enable-preview —release 14 Person.java

逆コンパイルして、コンパイラが何をしたかを確認します。

public final class Person extends java.lang.Record {
    private final java.lang.String name;
    private final int age;

    public Person(java.lang.String name, int age) { /* compiled code */ }

    public java.lang.String toString() { /* compiled code */ }

    public final int hashCode() { /* compiled code */ }

    public final boolean equals(java.lang.Object o) { /* compiled code */ }

    public java.lang.String name() { /* compiled code */ }

    public int age() { /* compiled code */ }
}

そのため、Recordの代わりに、新しい抽象Recordクラスを継承する最終クラスを取得しました。そして予想通り、生成されたゲッター、コンストラクター、equals（）、hashCode（）、toString（）を取得しました。すべてのフィールドがfinalとしてマークされていることに注意してください。つまり、フィールドを再定義することはできません。ただし、レコード自体は完全に変更されていません。フィールドに格納されているオブジェクトは変更可能です。詳細な議論の記録を見つけることができるここに。

Talkative NullPointerExceptions


よりフレンドリーなのはNullPointerExceptionsです。いいえ、コンパイラーはNPEを修正する方法をまだ知りませんが、今では例外の説明がより有益になります。状況をシミュレートします。Java11とJava 14でNPEを呼び出します。たとえば、Entityに典型的な複雑な構成があり、データを取得するには、目的のフィールドに到達するためにソースからいくつかのオブジェクトを呼び出す必要があります。

var street = message.getAuthor().getAddress().getStreet();

Java 11では、主な質問である「誰がnullか？メッセージ、著者、住所？

Exception in thread "main" java.lang.NullPointerException
	at Main.main(Main.java:11)

Java 14で実行するには、上記のRecordと同じ方法でクラスをコンパイルする必要があります。

javac —enable-preview —release 14 Main.java

そして、特別なフラグを追加して実行します：

java -XX:+ShowCodeDetailsInExceptionMessages --enable-preview Main

実行の結果、次の出力が得られます。

Exception in thread "main" java.lang.NullPointerException: Cannot invoke "Address.getStreet()" because the return value of "Author.getAddress()" is null
	at Main.main(Main.java:11)

ご覧のとおり、メッセージの情報量が増え、コールチェーンのどこでNPEが発生したかを正確に確認できます。

テキストブロック


Java 13で導入されたテキストブロックは、プレビュー機能としてJava 14でも使用できます。それらの主なタスクは、複数行リテラルでの作業を簡素化することであることを思い出させてください。SQLクエリ、HTMLコード、JSONを記述するための非常に便利な機能。これは非常に役立つ機能のひとつだと思います。構文を思い出してください。たとえば、SQLクエリを記述する必要があります。Java 13より前は、文字列連結を使用して読み取り可能なクエリを記述していましたが、クエリは次のようになります。

String sql = "SELECT name, age " +
                 "FROM PERSON" +
                 "WHERE name = \'Igor\'" +
                 "AND car=\'Niva\'";

Java 13以降では、次のように三重二重引用符の間に行を記述することでテキストブロックを使用できます。


    String sql = """ 
                 SELECT name, age 
                 FROM PERSON
                 WHERE name = 'Igor'
                   AND car='Niva'
                 """;

Java 14では、テキストブロックで使用できる2つの新しい区切り文字が追加されています。最初は単一のスペースです：\ s。2番目は改行文字：\です。

スイッチ式


Java 14以降、スイッチ式はプレビュー機能から完全な機能に移行します。新しい演算子の機能を簡単に思い出してみましょう。

  • ラムダ構文
  • 1つのケースで複数の式を使用する機能
  • 可能なswitch（）値のセットのカバレッジが不完全な場合のエラー出力。つまり、列挙型を使用する場合：

    public enum  Car {
    NIVA, NEXIA, KIA
}

    次に、switch（）を次のように記述すると、コンパイラは列挙からのすべての可能なケースがリストされているわけではないことを示すエラーをスローします。

    switch (car) {
    case NIVA -> System.out.println("Niva");
    case NEXIA -> System.out.println("Nexia");
}
// error: the switch expressions does not cover all possible input values.

  • 値を返す機能：

    var avto = switch (car) {
    case NIVA, KIA -> "Niva are better";
    case NEXIA -> "Nexia";
    default -> "Niva still better";
};


パターンマッチング


パターンマッチングは、 Java 14で利用できる新しいプレビュー機能です。多くの話があり、多くの人が待っていました。目標は、instanceof演算子でオブジェクト型チェックとその変換を組み合わせることです。言い換えると、Java 14以前は次のように記述していました。

Object obj = "Igor";

if (obj instanceof String) {
    String s = (String) obj;
    System.out.println(s.length());
}

それら。オブジェクトであるクラスのメソッドを使用するには、オブジェクトをこのクラスにキャストする必要があります。Java 14以降、instanceofが検証と変換の機能を組み合わせた新しい構文が利用可能になりました。

Object obj = "Igor";

if (obj instanceof String s){
    System.out.println(s.length());
}
