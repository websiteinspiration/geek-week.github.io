CentOS 8をVDS 256にインストールする

CentOS 8は、私たちの好むと好まざるとに関わらず、ついに日が明けました。近い将来、コミュニティからの企業オペレーティングシステムのフォロワーは、それと友達になる必要があります。

「ワイルド」で8つを試すことに決めたところ、ホスティングプロバイダーはCentOS 8で仮想サーバーイメージを作成するのに急いでいなかったことがわかりました。現代のアナコンダ（anaconda）。

新しいCentOSの展開されたイメージを直接VDSハードドライブにコピーし、「ネットワークを介して」サーバーをダウンロードするために残りました。この方法は目新しいものではありませんが、たとえば、その実装に関する詳細で完全な指示を見つけることができませんでした。したがって、私はすぐにこのギャップを埋め、そのようなVDSへのCentOS 8のインストールについて話したかったのです。

1. 3つの大切なファイル

メモリの量とドナーディスクのサイズは重要ではありません、主なものはそれにオペレーティングシステムをインストールすることです。私は断固として、1つのルートパーティションでドナーディスクをマークすることをお勧めします。

この場合、KVM VDSがあるため、ドナーは自宅で、libvirtを使用して任意のLinuxディストリビューションで焼き上げることができます。私は追加の最小限の初期設定を好みます。これは、タスクのコンテキストで重要な助けになります。で、この記事小さなCentOSイメージを作成する標準的な方法について話しました。スポイラーの下で、対応するキックスタートファイルとその使用方法：

ドナーを作成したら、ディスクからすべてのファイルを1つのアーカイブ（centos8.tgzなど）に取り込む必要があります。これはさまざまな方法で実行できます。ドナーディスクを別の仮想マシンに接続するか、ハイパーバイザーにマウントするか、仮想ディレクトリ（/ dev、/ procなど）を除くすべてのファイルをドナー自体にパックします。

次に、ドナーからさらに2つのファイルが必要です。 ：カーネル（vmlinuz）と変更されたramdrive（initrd）カーネルはドナーの/ブートディレクトリ（vmlinuz-something-there）で取得でき、NFSサーバーからネットワーク経由でブートする機能を含めることにより、ramdriveを特別にする必要があります。これを行うには、動作しているドナーオペレーティングシステムで、次のことを行う必要があります。

# dnf install dracut-network nfs-utils
# echo 'add_dracutmodules+="nfs"' >> /etc/dracut.conf
# dracut -f /initrd

そして、ディスクのルートから必要なinitrdを取得します。

その結果、将来必要になる3つのファイル：

  • centos8.tgz-インストールされているCentOS 8のすべてのファイルを含むアーカイブ
  • initrd -NFSをサポートする一時ファイルシステム
  • vmlinuz -OSカーネル

2. NFSサーバー


ローカルディスクを使用せずにネットワーク経由でVDSをダウンロードするには、NFSリソースを備えたサーバーが必要です。このようなサーバーは何でもかまいません。主なものは、VDSのIPアドレスからのネットワークの可用性です。自宅の機器（「ルーターでのポートフォワーディング」の可能性あり）または他のVDSを、たとえばテストのために試すことができます。最後に、NFS（冗談）で管理できる場合は、「ドナー」を使用します。CentOS 8をインストールすると、ドナーと同様にNFSサーバーは不要になります。

エクスポートされたNFSリソースには、ルートにcentos8.tgzファイルとその解凍されたコンテンツが含まれている必要があります。以下は、例としてCentOS 7を使用したクイックプロダクションガイドです。

# mkdir /nfsroot
# cp .../centos8.tgz /nfsroot
# tar xzf /nfsroot/centos8.tgz -C /nfsroot

# yum install nfs-utils
# systemctl enable rpcbind nfs-server

# echo '/nfsroot 100.100.100.100/32(rw,sync,no_root_squash,no_all_squash)'>/etc/exports

# systemctl restart rpcbind nfs-server

100.100.100.100はもちろん、将来のCentOS 8でのVDS IPアドレスです。

3. VDSを「ネットワーク経由」でダウンロードします（NFSroot）


まず、最初の手順で取得したramdrive（initrd）ファイルとカーネル（vmlinuz）ファイルを VDSルートにコピーする必要があります。次に、GRUBを構成します

。/etc/grub.d/40_custom

#!/bin/sh
exec tail -n +3 $0
# This file provides an easy way to add custom menu entries.  Simply type the
# menu entries you want to add after this comment.  Be careful not to change
# the 'exec tail' line above.
menuentry "NFSRoot" {
    linux  /vmlinuz root=/dev/nfs rw nfsroot=200.200.200.200:/nfsroot \
           ip=100.100.100.100::100.100.100.1:255.255.255.0:::off
    initrd /initrd
}

200.200.200.200-NFSサーバーのIPアドレス。

/ etc / default / grub

. . .
GRUB_TIMEOUT=0
. . .
GRUB_DEFAULT="NFSRoot"
. . .

VDSを再起動します。

# grub2-mkconfig --output=/boot/grub2/grub.cfg
# reboot

すべてが順調に進むと、VDSがCentOS 8ドナーをロードし、ローカルドライブが完全にマウント解除されます。

以来、ネットワークマネージャが理解遺産を、あなたは、ネットワークを構成するために、適切なifcfgファイルを作成することができます。インターフェースの名前は次のとおりです。

# ls /sys/class/net/
enp1s0  lo


# cat > /etc/sysconfig/network-scripts/ifcfg-enp1s0 << EOF
> DEVICE=enp1s0
> NAME=enp1s0
> TYPE=Ethernet
> ONBOOT=y
> BOOTPROTO=none
> IPADDR=100.100.100.100
> NETMASK=255.255.255.0
> GATEWAY=100.100.100.1
> EOF

以上で、ローカルドライブからVDSを再起動できます。
