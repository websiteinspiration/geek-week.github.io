Cara menggunakan rakitan pihak ketiga di plug-in

Masalah



Saya, dan saya pikir banyak pengembang MS Dynamic CRM (dalam kasus saya ini adalah 2016 on premise) mengalami masalah, untuk menggunakan majelis pihak ketiga, Anda perlu menggabungkan mereka dengan majelis utama dan kemudian mendaftarkannya di Plugin Pendaftaran Util.



Sebagai contoh, banyak yang menggunakan ILMerge atau ILRepack.



Cari



Saya mencari opsi solusi di berbagai situs, tetapi di mana-mana kecuali menggunakan ILMerge atau ILRepack, saya tidak menemukan opsi lain. Saya ingin mengimplementasikan tugas sehingga majelis dapat bergerak bersama dengan database ke server lain, dan semuanya akan bekerja tanpa masalah.



Tidak ada penemuan



Saya pikir opsi yang saya jelaskan tidak akan membawa sesuatu yang baru, tetapi di satu sisi itu bukan solusi sepele yang membantu teman dan kolega saya Yuriy Yurkovskiy membantu saya menemukannya .

using System;

using System.Data.SqlClient;

using System.Reflection;

using System.Web;



namespace CrmExternalAssemblyLoader

{

public class AssemblyLoaderModule: IHttpModule

{

private static bool IsLoaded;

private readonly object _lock = new object();

public void Dispose()

{

throw new NotImplementedException();

}



public void Init(HttpApplication context)

{

lock (_lock)

{

if (!IsLoaded)

{

AppDomain.CurrentDomain.AssemblyResolve += CurrentDomain_AssemblyResolve;

IsLoaded = true;

}

}

}



private Assembly CurrentDomain_AssemblyResolve(object sender, ResolveEventArgs args)

{

var assemblyName = new AssemblyName(args.Name);

var assembly = GetAssembly(assemblyName.Name + ".dll");

if (assembly != null)

{

return Assembly.Load(assembly);

}

return null;

}

Selesai



Ketika eksekusi plugin dimulai, CRM akan mencoba untuk menemukan perakitan dalam memori, tidak menemukannya dan merangkak ke dalam AppDomain.CurrentDomain.AssemblyResolve, di mana perakitan dikurangi dari database dan dimuat ke dalam memori domain. Majelis akan hidup sampai situs dimuat ulang.



Pro kontra



+ Anda tidak perlu menggabungkan rakitan ke dalam plugin, sehingga ukuran rakitan tidak akan bertambah, tidak akan ada banyak beban pada SQL ketika plugin dimulai, terutama jika plugin tidak terdaftar.

+ Kecepatan akses ke perakitan lebih tinggi karena sudah dimuat ke dalam domain.

+ Semua majelis pihak ketiga ada di database Anda dan bermigrasi dengan basis.

+ Kemampuan untuk menyimpan rakitan dari versi yang berbeda ( Momen ini tidak diuji dengan baik! Jadi masih ada sesuatu yang perlu diperiksa)



- Mungkin saat memperbarui CRM, Anda perlu menambahkan baris modul ke web.config CRM lagi.
