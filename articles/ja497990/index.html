PostgreSQL：C言語での拡張（関数）の開発

この記事は数年前に書かれたもので、どこに置くことができるのかわからず、忘れていました。

解釈された（スクリプトされた）言語と比較してPostgreSQLの拡張機能を開発するときにC言語を使用する意味は、パフォーマンスと機能の2つに減らすことができます。ただし、単純に、Cで記述されたコードは、たとえば、各レコードのリクエストで関数が100万回呼び出された場合、はるかに速く機能します。さらに具体的には、一部のPostgreSQL機能は、Cを除いてまったく実行できません。たとえば、他の言語のタイプ（特に、関数から値を返す場合）では、ANYELEMENT、ANYARRAY、特に重要なVARIADICはサポートされていません。

単純なC関数


たとえば、2つの引数を取り、それらを追加する関数を記述します。この例はPostgreSQLのドキュメントで説明されていますが、少し改善して収集します。次に、PostgreSQLにロードして、SQL関数からの呼び出しを記述します。

#include "postgres.h"
#include "fmgr.h"
PG_MODULE_MAGIC;
PG_FUNCTION_INFO_V1(add_ab);
Datum add_ab(PG_FUNCTION_ARGS)
{
    int32 arg_a = PG_GETARG_INT32(0);
    int32 arg_b = PG_GETARG_INT32(1);
    PG_RETURN_INT32(arg_a + arg_b);
}

add_func.c ファイルは、より複雑な機能を開発するためのテンプレートとして使用できます。また、コードを通過します

  • #include "postgresql.h"：常に含める必要があるこのヘッダーファイルには、さまざまな基本タイプと関数が含まれています。
  • #include "fmgr.h"：ヘッダーファイルには、さまざまなPG_ *マクロが含まれています。
  • PG_MODULE_MAGIC：バージョン8.2以上のPostgreSQL用のモジュールを開発していることを決定するマクロ。
  • PG_FUNC_INFO_V1：PostgreSQL内の関数呼び出し規約を定義するマクロ。関数を宣言せずに関数を宣言すると、バージョン0を呼び出すことに同意し、それ以外の場合はバージョン1を呼び出すことになります。

  • add_ab(PG_FUNCTION_ARGS): . - . , .
  • int32 arg_a = PG_GETARG_INT32(0): ( ).
  • PG_RETURN_INT32(arg_a + arg_b): .

これを正しくコンパイルしてコンパイルする必要があります。出力は、動的にロード可能な共有ライブラリ（* .so）になります。そのためには、Makefileを使用する方が便利です。ドキュメントには、登録が必要なキーとパスが記載されていますが、PGXSを使用して収集します。これは拡張機能開発者向けの環境です。つまり、PostgreSQLに必要なすべての-devおよび-develパッケージをシステムにインストールする必要があります。

MODULES = add_func
PG_CONFIG = pg_config
PGXS := $(shell $(PG_CONFIG) --pgxs)
include $(PGXS)

SQL関数を作成します


次に、以前に作成した拡張ライブラリから呼び出されるSQL関数を記述します。
CREATE FUNCTION add(int, int)
  RETURNS int
AS '/usr/lib/postgresql/9.4/lib/add_func', 'add_ab'
LANGUAGE C STRICT;

それで全部です！これでこの関数を使用できるので

SELECT add(1, 2);

インストールを自動化する


ここで、インストールを少し自動化します。これは、PostgreSQLのどのバージョンが使用され、どのようにインストールされているかが事前にわからないときに非常に役立ちます。これを行うには、次のファイルを作成します。

CREATE FUNCTION add(int, int)
  RETURNS int
AS 'MODULE_PATHNAME', 'add_ab'
LANGUAGE C STRICT;

Makefileに行 を追加し、

MODULES = add_func
DATA_built = add_funcs.sql
PG_CONFIG = pg_config
PGXS := $(shell $(PG_CONFIG) --pgxs)
include $(PGXS)

引数の確認


STRICTキーワードを使用したadd（int、int） 関数のSQL定義を覚えていますか？つまり、引数の少なくとも1つがNULLの場合、関数は機能せず、単にNULLを返します。これはSQLステートメントの動作に似ています。たとえば、+演算子に少なくとも1つのNULL引数がある場合、結果はNULLになります。 SQL集計関数sum（）と同様に、NULL値を無視して機能し続ける引数チェックを関数に追加します。このために私たちはする必要があります



  • 引数の1つがNULLであっても、関数が機能することを確認してください
  • 両方の引数がNULLの場合、NULLを返します

PG_FUNCTION_INFO_V1(add_ab_null);
Datum add_ab_null(PG_FUNCTION_ARGS)
{
    int32 not_null = 0;
    int32 sum = 0;
    if (!PG_ARGISNULL(0))
    {
        sum += PG_GETARG_INT32(0);
        not_null = 1;
    }
    if (!PG_ARGISNULL(1))
    {
        sum += PG_GETARG_INT32(1);
        not_null = 1;
    }
    if (not_null)
    {
        PG_RETURN_INT32(sum);
    }
    PG_RETURN_NULL();
}

そして今、それをチェックしてください、

CREATE FUNCTION add(int, int)
 RETURNS int
AS '$libdir/add_func', 'add_ab_null'
LANGUAGE C;

SELECT add(NULL, NULL) AS must_be_null, add(NULL, 1) AS must_be_one;
-[ RECORD 1 ]+--
must_be_null |
must_be_one  | 1

関数内の任意の数の引数


すでにお気づきのように、マクロを使用して引数の値を取得しました。したがって、任意の数の引数を渡して、ループでそれらの値を読み取るだけです。

if (!PG_ARGISNULL(i))
{
    sum += PG_GETARG_INT32(i);
    not_null = 1;
}

すぐに質問があるかもしれませんが、配列を使用して引数の値を渡すことができます。そして実際、これは実行する必要があることですが、残念ながらPostgreSQLには独自のメモリマネージャが存在するため、これはそれほど簡単な作業ではありません。しかし、それを解決してみてください。PostgreSQLのドキュメントに、テキスト[]文字配列が渡される例がありますが、これは必ずしも必要なものではありません。私たちの目的に合わせてみましょう

#include "utils/array.h"     //       
#include "catalog/pg_type.h" // INT4OID
PG_MODULE_MAGIC;
Datum add_int32_array(PG_FUNCTION_ARGS);
PG_FUNCTION_INFO_V1(add_int32_array);
Datum add_int32_array(PG_FUNCTION_ARGS)
{
    //     .
    //          ,       int.
    ArrayType *input_array;
    int32 sum = 0;
    bool not_null = false;

    Datum *datums;
    bool *nulls;
    int count;
    int i;
    input_array = PG_GETARG_ARRAYTYPE_P(0); //     .  *_P  
                                            //        ,   INT32

    //          INT32 (INT4)
    Assert(ARR_ELEMTYPE(input_array) == INT4OID);

    //      
    if (ARR_NDIM(input_array) > 1)
        ereport(ERROR, (errcode(ERRCODE_ARRAY_SUBSCRIPT_ERROR), errmsg("1-dimensional array needed")));

    deconstruct_array(input_array, //  
                      INT4OID,     //  
                      4,           //    
                      true,        // int4   
                      'i',         //  'i'
                      &datums, &nulls, &count); //   

    for(i = 0; i < count; i++)
    {
        //     
        if (nulls[i])
            continue;

        // ,       
        sum += DatumGetInt32(datums[i]);
        not_null = true;
    }
    if (not_null)
        PG_RETURN_INT32(sum);

    PG_RETURN_NULL();
}

いつものように、コードを見ていきましょう。

  • 整数配列には特別なタイプはないため、あらゆるタイプの配列で機能するジェネリックタイプArrayTypeを使用します。
  • 最初の引数で配列を初期化するために、特別なマクロPG_GETARG_ARRAYTYPE_Pを使用しました
  • 配列が本当に1次元であるかどうかを確認するチェックもありますARR_NDIM
  • int4（= 23）のOIDタイプはINT4OIDとして定義されます。他のタイプの定義を表示するには、SQLを使用できます。

    select oid, typlen, typbyval, typalign from pg_type
where typname = 'int4';
-[ RECORD 1 ]
oid | 23
typlen | 4
typbyval | t
typalign | i

推奨事項


次に、拡張機能を作成するための主な推奨事項のいくつかを要約します。

メモリを操作する


PostgreSQL DBMSの作成者と作成者は、メモリの操作とリークの防止に特に注意を払っています。これは、DBMSとその拡張機能を開発するための基本的なルールの1つです。これは彼ら自身の決定、つまり記憶の文脈に基づいています。

palloc（）およびpfree（）の使用


PostgreSQLのメモリに関するすべての作業は、非標準の関数palloc（）およびpfree（）を使用することから成ります。

構造の初期化


常に新しい宣言された構造を初期化します。たとえば、palloc（）の後にmemset（）を呼び出します。

接続されたファイル


各拡張機能には、少なくとも2つのファイル（postgres.hおよびfmgr.h）が含まれている必要があります。
