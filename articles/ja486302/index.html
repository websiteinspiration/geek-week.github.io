NSIS：$ 0〜$ 9、$ R0〜$ R9の問題

:)

ターゲットハブが記事の内容に対応していませんが、何をすべきか...-目的に応じたテーマはありませんが、最も適切なのは「C」です。それを読んだ人は書かれたものを最も簡単に知覚することができるでしょう。

記事の目的


NSIS を使用してインストーラー/アンインストーラーを作成するときにグローバル変数を制御する問題を解決するオプション。

小さな紹介


nsisコードを書くとき、プログラマーはプロジェクト全体をスコープとするグローバル変数を使用する機会があります。それらの20のみがあります：$ 0- $ 9および$ R0- $ R9。
これらを使用すると非常に便利です。これにより、対象の問題を解決するために正式に意図された追加の変数を導入する必要がなくなります。正式に言うと すべてのnsis変数にはグローバルスコープがあり、それらが何か異なる場合は、修飾子const（C / C ++の用語で言えば）がある場合のみです。

!define variable1 ''value 1''

特徴：

  • グローバルスコープの静的変数。
  • 宣言時に初期化されます。
  • 連絡方法：$ {variable1}

Var variable2

特徴：

  • グローバルスコープの動的変数。
  • コードによってどこでも宣言されますが、最初の呼び出しよりも上です。
  • 必要な回数だけ初期化されますが、関数ブロックとマクロの内部のみです。
  • アクセス方法：$ variable2

したがって、上記の機能を考えると、static以外の変数をターゲットにして作成することは、正当化するのが難しいことが明らかになります。グローバル変数はデフォルトですでに存在し、ニーズを満たすのに十分です（そうでない場合は、プログラマーに「適切な」質問をする必要があります）。他の変数はコードを詰まらせるだけです。

問題の説明


しかし問題は、このコードは常にエラーを生成することです：


Function F1
    StrCpy $0 "vladimir"
FunctionEnd

Function F2
    StrCpy $0 "alexei"
    Call F1
    ${If} $0 == "alexei" ; //  $0  "vladimir"
        ; // -  ...
    ${EndIf}
FunctionEnd

そして、あなたはこれの多くの例を思い付くことができます、そして、nsisプログラミング職人は彼らが可能な限りこの問題を解決しました：

  • 誰かがすべての変数を注意深く見ました（そのような人々は記念碑を建てる必要があります））;
  • 誰か、主にさまざまなプラグインの作成者が、提示された変数のいずれかを使用するたびに、それをスタックにプッシュし、使用の終了後、Popを呼び出してプライマリ値を返しました。
  • 上記のように、誰かがVarを介して変数を生成しました。

いずれにせよ、彼らは皆、グローバル変数の状態を監視するという問題を解決する普遍的なメカニズムの欠如という問題に直面しました。

ここで私は彼女（問題）に普遍的な解決策を提供することを許可します。

問題の解決策


まず、NSISListへのリンクは、リストタイプのデータ構造を使用する機能を導入するプラグインであり、C ++のstd :: listと同様の方法で取得する必要があります。

2番目のステップは、問題解決メカニズムのソースコードです。


!verbose 3
!include NSISList.nsh	;      *

/*
  Dump, :

!define F1 "!insertmacro _F1"
!macro _F1 [paramIn1 paramIn2 ... paramInN] [paramOut1 paramOut1 ... paramOutM]
    ${Dump}
    [Push ${paramIn1}
	Push ${paramIn2}
	...
	Push ${paramInN}]
    !ifdef __UNINSTALL__
        Call un.F1
    !else
        Call F1
    !endif
    ${Undump}
    [Pop ${paramOut1}
	Pop ${paramOut2}
	...
	Pop ${paramOutM}]
!macroend

!macro Func_F1 un
    Function ${un}F1
        ;  ...
        ;         $0-$9, $R0-$R9.
        ;        .
    FunctionEnd
!macroend
!insertmacro Func_F1 ""
!insertmacro Func_F1 "un."

*/

/**   Dump */
!define InitDump "!insertmacro _InitDump"
!macro _InitDump
    !ifdef __UNINSTALL__
        Call un.InitDump
    !else
        Call InitDump
    !endif
!macroend

!macro Func_InitDump un
    Function ${un}InitDump
        #  $0-$9
        ${List.Create} d0
        ${List.Create} d1
        ${List.Create} d2
        ${List.Create} d3
        ${List.Create} d4
        ${List.Create} d5
        ${List.Create} d6
        ${List.Create} d7
        ${List.Create} d8
        ${List.Create} d9
        #  $R0-$R10
        ${List.Create} dR0
        ${List.Create} dR1
        ${List.Create} dR2
        ${List.Create} dR3
        ${List.Create} dR4
        ${List.Create} dR5
        ${List.Create} dR6
        ${List.Create} dR7
        ${List.Create} dR8
        ${List.Create} dR9
    FunctionEnd
!macroend
!insertmacro Func_InitDump ""
!insertmacro Func_InitDump "un."


/**     nsis-  Dump */
!define Dump "!insertmacro _Dump"
!macro _Dump
    !ifdef __UNINSTALL__
        Call un.Dump
    !else
        Call Dump
    !endif
!macroend

!macro Func_Dump un
    Function ${un}Dump
        # $0-$9
        ${List.Add} d0 $0
        ${List.Add} d1 $1
        ${List.Add} d2 $2
        ${List.Add} d3 $3

!macroend
!insertmacro Func_Dump ""
!insertmacro Func_Dump "un."


/**     nsis-  Dump */
!define Undump "!insertmacro _Undump"
!macro _Undump

        Call un.Undump
    !else
        Call Undump
    !endif
!macroend

そして3番目のステップは、その 機能の簡単な説明です。

前のメインコードコメントで説明されたルールに従って作成および設計された各関数は、グローバル変数$ 0- $ 9、$ R0- $ R9を上書きする危険性の観点から完全に保護されます。 -呼び出し関数のロジックの違反から。

それはこのように動作します：

  1. F1関数自体は直接呼び出されません。 
    Call F1
    マクロラッパーを介して 
    ${F1} [paramIn1 [paramIn2]...[paramInN]] [paramOut1 [paramOut2]...[paramOutM]]
  2. 関数が直接呼び出される前であっても、呼び出されます 
    ${Dump}
    その結果、変数$ 0- $ 9、$ R0- $ R9の現在の値がダンプに保存されます。内部的には、多くのリストがあり、それぞれがターゲット変数に関連付けられています。
  3. 関数のすべてのターゲット変数をスタックにロードする 
    Push ${paramIn1} Push ${paramIn2} ... Push ${paramInN}
    （必要な場合）;
  4. 関数が呼び出されます。
  5. 関数はそのロジックを実行して終了します。
  6. 通話が発生します 
    ${Undump}
    その結果、$ {Dump}の最後の呼び出しで保存されたグローバル変数の値が復元されます。
  7. スタックからの関数結果のアンロード 
    Pop ${paramOut1} Pop ${paramOut2} ... Pop ${paramOutM}
    （必要な場合）;
  8. F1関数のマクロラッパーが作業を完了します。

結論


その結果、nsisコードを安全に書き込むためのユニバーサルデザインが得られました。

コンパイルされたコードが約30ミリ秒動作することを除いて、マイナスにはならないと言うこともできます。もっとゆっくり。
