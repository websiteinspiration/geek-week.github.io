Vite-Vueにバンドルなしで開発をリード

こんにちは、ハブロフスク。6月に、OTUSはVue.js開発者コースを開始します。コースの開始を見越して、私たちは伝統的に興味深い資料の翻訳を準備しました。


ある日、Ewan Yu（Vue.jsの作成者）は夜寝ないことを決め、最終的にViteを作成しました。



Viteを使用すると、バンドルのアセンブリ手順なしで、単一ファイルのコンポーネントを使用してVueでアプリケーションを作成できます。インポートは、ネイティブESモジュールとしてブラウザーによって要求されます。開発サーバーは.vueファイルのリクエストをインターセプトし、オンザフライでコンパイルします。そして、それは即座に起こります。

Viteはまだ実験的な解決策であり、今後何が起こるか、またこの記事がどれくらいの期間有効であるかはわかりません。食前酒とか、楽しいものとか。

やってみよう


次のファイルを作成します。

Comp.vue

<template>
  <button @click="count++"> {{ count }}</button> 
</template>

<script>
export default {
  data: () => ({ count: 0 })
}
</script>

<style scoped>
button{
  font-size: 2rem;
}
</style>

index.html

<div id="app"></div>
<script type="module">
  import { createApp } from 'vue'
  import Comp from './Comp.vue'

  createApp(Comp).mount('#app')
</script>

次に、に

npx vite

移動しhttp://localhost:3000、ファイル.vueを編集して、即座に変更を確認します。

生産組立


次に、本番用のアプリケーションを作成しましょう。

作成するpackage.json

{
  "scripts": {
    "build": "vite build"
  }
}

実行：

npm i -D vite
npm run build


コードでディレクトリを確認してください。
私が遭遇した問題：

  • エラー：モジュール 'tslib'が見つかりません
    npm i -D tslibを実行して解決
  • エラー：ENOENT：そのようなファイルまたはディレクタはありません
    空のディレクトリを手動で作成することで解決しました

考え


変更の結果をすぐに見たので、開始は本当に簡単で、開発はより速くなりました。その場でのコンパイルのようなもの。したがって、それは本当に有望に見えます。おそらくしばらくしてから、Vueのこのような開発方法に移ります。

エヴァン・ユー、魔法をかけてください。

おもしろい事実：ビューはフランス語からビューとして翻訳され、すぐに読みます。

この時点で、Viteの作業は完了しており、Vue 3以降も続行されるようです。最も簡単な方法は、Vue 3ベータ版です。Evan Yuさんから

のツイートをもとに完成したとのことですが、リポジトリのコミットを見ると、まだ作業が進んでいると言えます。
