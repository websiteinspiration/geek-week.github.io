初心者の目から見たWebix JavaScriptライブラリ



私は初心者のJSフロントエンド開発者です。現在、ミンスクのIT企業で勉強とトレーニングをしています。Web-uiの基本を学ぶことはWebix JSライブラリの例で行われます。私は最初の経験を共有し、このエレガントで強力なライブラリの開発のための一種のトレーニングマニュアルを手配したいと思います。

最初のタスク


Webixライブラリを使用してWebアプリケーションインターフェイスを作成します。ライブラリ自体は、組み込みのCSSスタイルを提供します。デフォルトでは、マテリアルデザインスキンが使用され、ライブラリには合計5つの異なるスキンが用意されているため、CSSを編集する必要はほとんどありません。

以下のスクリーンショットは、参照インターフェース-TKを示しています。TKは、実際の演習として「組み立てる」必要があります。

ステップ1.作業環境のセットアップ


作業環境を作ります。ファイルが必要です：

  • index.html;
  • script.js;
  • data.js;
  • style.css;

style.cssファイルはオプションであり、それなしで実行できますが、私はプロセスに従って厳密に動作することを学ぶので、それを動作させます。

次に、ライブラリを接続します。ドキュメントには、 3つの接続方法を示すWebixインストールガイドがあります。

  • npmパッケージマネージャーを使用してダウンロードします。
  • サイトから直接パッケージをダウンロードしてください。
  • Webix CDN GPLファイルへの直接リンクを介して。

3番目の方法はより視覚的であり、index.htmlファイルの数行のコードで実行できるため、次の方法を使用します。

<link rel="stylesheet" type="text/css" href="http://cdn.webix.com/edge/webix.css">
<script type="text/javascript" src="http://cdn.webix.com/edge/webix.js"></script>

ここでは、例に示されているのと同じ順序で残りのファイルを接続します。これは、script.jsファイルがdata.jsに含まれているデータを使用するためです。

<!DOCTYPE html>
<html lang="en">
<head>
    <script type="text/javascript" src="http://cdn.webix.com/edge/webix.js"></script>
    <link rel="stylesheet" type="text/css" href="http://cdn.webix.com/edge/webix.css">

    <link rel="stylesheet" href="style.css">
    <title>My first page</title>
</head>
<body>

    <script src="data.js"></script>
    <script src="script.js"></script>
</body>
</html>

ステップ2.初期化。プロジェクトのワイヤーフレーム


Webixインターフェース全体は、webix.ui（）コンストラクターによって作成されます。すべてが正しく構成され、正常に機能していることを確認するために、「Hello world！」という文字列を画面に表示します。

私は組み込みのtempalteテンプレートエンジンを使用します。これにより、必要なレコードを含む既製のコンポーネントが生成されます。

webix.ui({
    template: "Hello world!"
});

ライブラリの主な機能はレイアウトコンポーネントです。このコンポーネントを使用すると、ページを他のコンポーネントの行と列に分割できます。これで、3つの行で構成されるページのスケルトンを定義します。

  1. ヘッダ
  2. フッター
  3. メイン（本体）。

組み込みのCSSテーマもここで接続します。

コード：

webix.ui({
    rows:[
        {template:"Header"},
        {template:"Main"},
        {template:"Footer"}
    ]
});

以下は何が起こったかのスクリーンショットです。rows配列を使用して、3つの行を作成しました。


図2

インターフェースの主要部分は、3つの列で構成されます：
-リスト、
-テーブル
-フォーム。
それを描画するには、cols配列を使用して、必要な列を作成します。

コード：

webix.ui({
    rows:[
        {template:"Header"},
        {cols:[
            {template:"List"},
            {template:"Table"},
            {template:"Form"}
        ]},
        {template:"Footer"}
    ]
});

ステップ3.ヘッダーを作成する


ページフレームが作成され、目的のコンテンツで埋めるために残ります。私のTKでは、ヘッダーにさまざまなコントロールが含まれているため、テンプレートの代わりに「ヘッダー」、「ツールバー」ウィジェットを使用しています。これで、タイトルとボタンを配置できる必要な要素が作成されました。

            view:"toolbar",
            css:"webix_dark",
            cols:[
                { view:"label", label:"My app"},
                {},
                {height: 40, type:"icon", icon:"wxi-user",  view:"button", label:"Profile", width:100, css:"webix_transparent"}
            ]

次に、cols配列を使用して、3つの列が作成されます。最初の列では、コンポーネント、view：“ label”が静的テキストを作成し、labelキーがテキストタイトルを表示します。

2列目はスペーサーです。すべての空きスペースを埋め、最初と最後の要素をページの端まで押します。

3列目はボタンであり、view：“ button”コンポーネントを使用して作成されます。widthプロパティとheightプロパティは、ボタンの高さと幅をピクセル単位で設定し、type： "icon"およびicon： "wxi-user"設定は、ドキュメントで提供されているリストから必要なアイコンを表示します。

ステップ4.リストウィジェットを作成する


リスト ウィジェットは標準のWebixウィジェットであり、メニューの作成に使用されます。私の仕事は、select：trueプロパティを使用してクリック可能にすることです。また、このウィジェットでは、クラス-css： "list_color"を使用してスタイルの変更を使用しています。テンプレート「リスト」を記述する

代わりに、次のコードを記述します。

{
    view: "list",
    id:"mylist",
    scroll:false,
    select:true,
    width:200,
    css:"list_color",
    data:[
        {value:"Dashboard",},
        {value:"Users"},
        {value:"Products"},
        {value:"Location"}
    ]
},
{view: "resizer"},
{template:"Table"},
{template:"Form"}

リスト ウィジェットでは、デフォルトはscrollで、コマンド-scroll：falseで無効にしています。

リストの長さとその値は、データ配列を使用して設定されます。
デフォルトのスタイルをオーバーライドするには、次のように記述する必要があります。

.list_color{
  background-color: #EEEEEE;
  font-weight: 500;
}
.list_color .webix_selected{
  background-color: #dddfe2;
}

デフォルトのウィジェットの背景は白で、.list_color {...}と書くことで取得します。ただし、要素のスタイルを設定する必要がある場合は、クラスの名前の後に、要素のデフォルトクラスを指定する必要があります。リスト

ウィジェットの最後のタッチで、それとテーブル列の間にドラッグされた境界線- {view： "resizer"}を設定します。4番目のステップの結果： 図5




ステップ5.テーブルを作成し、別のファイルからデータを抽出する


データがdata.jsファイルから取得されていることを思い出させてください。それらをテンプレートではなくテーブルの形式で表示するには、「テーブル」行、次のコードを記述します。

    view:"datatable",
    id:"film_list",
    scroll:"y",
    autoConfig: true,
    data:small_film_set

コードは次のように機能します。view： "datatable"はテーブルを作成し、autoConfig：trueを使用して、small_film_set配列の最初に作成されたデータに基づいて列を構成します。この配列へのパスは、data - smal_film_setという行で示されます。

ストックスクロール：「y」は水平スクロールを削除し、垂直スクロールのみを残します。結果は、列見出しをクリックしてソートできるテーブルです。


図6

ステップ6.フォーム


プロパティビュー： "フォーム" - ウィジェット "フォーム"を作成します。作業明細書によると、これは要素配列に配置された入力フィールドとボタンで構成されています：[{...}、...]。

文字列テンプレート「Table」を次のコードに置き換えます。

view:"form", id:'film_form', width: 350,
elements:[
    { type:"section", template:"EDIT FILMS"},
    { view:"text", name:"title", label:"Title" },
    { view:"text", name:"year", label:"Year" },
    { view:"text", name:"rating", label:"Rating" },
    { view:"text", name:"votes", label:"Votes" },
    {
        margin:10,
        cols:[
                { view:"button", id:"btn_add", minWidth:65, value:"Add new", css:"webix_primary", click:addItem},
                { view:"button", id:"btn_clear", minWidth:65, value:"Clear", click:clearForm}
        ]
    },
    {}
]

type： "section" プロパティを持つ行は、フォームのタイトルです。入力フィールドはプロパティによって設定されます-view： "text"、フィールド名はラベルに書き込まれます。今後、フォームは値を渡す必要があります。これには、view： "text"プロパティと組み合わせて、name： "..."プロパティが設定されます。

ウィジェットはボタンであり、プロパティビューで設定されます： "button"。ボタンはフォームの不可欠な部分ではなく、スタンドアロン要素として使用できます。標準のHTML入力タグと同様に、ボタン名はvalueプロパティによって割り当てられます。青色は組み込みのcssスタイル"webix_primary"によって設定されます。

配列の最後の要素はスペーサーで、フッターまでのフリースペースを埋めます。フッターは画面の下部に押されます。これは、後続のすべての要素が押し込まれるフォーム要素のデフォルトの高さのために使用されます。コード結果：

最終段階


ページはほぼ準備ができており、フッターを作成するためだけに残っています。行テンプレート「Footer」を次のコードに置き換えます。

cols:[
{height: 30, template:"The software is provided by <a href='#'>webix.com</a>. All rights reserved (c)", css:"center_text"}
]

コードは、テキストに加えてテンプレートプロパティを使用すると、HTMLタグを設定できることを示しています。テキストを中央揃えにするために、カスタムのcssクラス「center_text」を使用しました。

.center_text{
  text-align:center;
}

ページの最終結果は、作業指示書で宣言された内容に完全に準拠する必要があります（図1）。

汎化


この例では、Webixの機能のごく一部のみが使用されました。結果は、わずか60行のコードで、多くの複雑な要素を含むページを作成できることを示しています。既製のライブラリソリューションとそのコンポーネントの使いやすさにより、スタイルをいじる時間を無駄にすることなく、完全に異なるページレイアウトを自由に設計できます。
