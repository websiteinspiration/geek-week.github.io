単純な割り当てでクロージャー変数を変更することはできません。Pythonは代入を関数の本体内の定義と見なし、通常はクロージャーを作成しません。 

正常に動作し、ディスプレイ2：

def make_closure(x):
    def closure():
        print(x)

    return closure

make_closure(2)()

そして、このコードはスローしUnboundLocalError: local variable 'x' referenced before assignmentます：

def make_closure(x):
    def closure():
        print(x)
        x *= 2
        print(x)

    return closure

make_closure(2)()


コードを機能させるには、を使用しますnonlocal。これは、割り当てを定義と見なさないようにインタープリターに明示的に指示します。

def make_closure(x):
    def closure():
        nonlocal x
        print(x)
        x *= 2
        print(x)

    return closure

make_closure(2)()


反復中に、最初または最後に処理されている要素を見つける必要がある場合があります。これは、明示的なフラグを使用して簡単に判断できます。

def sparse_list(iterable, num_of_zeros=1):
    result = []
    zeros = [0 for _ in range(num_of_zeros)]

    first = True
    for x in iterable:
        if not first:
            result += zeros
        result.append(x)

        first = False

    return result

def sparse_list(iterable, num_of_zeros=1):
    result = []
    zeros = [0 for _ in range(num_of_zeros)]

    iterator = iter(iterable)
    try:
        result.append(next(iterator))
    except StopIteration:
        return []

    for x in iterator:
       result += zeros

2つのイベント間の経過時間を測定する必要がある場合は、time.monotonic()代わりにを使用してくださいtime.time()。time.monotonic()システムクロックを更新する場合でも、小さな方向に変化することはありません。

from contextlib import contextmanager
import time


@contextmanager
def timeit():
    start = time.monotonic()
    yield
    print(time.monotonic() - start)

def main():
    with timeit():
           time.sleep(2)

main()


ネストされたコンテキストマネージャは通常、それらがネストされていることを知りません。外部マネージャーを使用して内部マネージャーを作成することで、これについて彼らに伝えることができます。

from contextlib import AbstractContextManager
import time


class TimeItContextManager(AbstractContextManager):
    def __init__(self, name, parent=None):
        super().__init__()

        self._name = name
        self._parent = parent
        self._start = None
        self._substracted = 0

    def __enter__(self):
        self._start = time.monotonic()
        return self
        
    def __exit__(self, exc_type, exc_value, traceback):
        delta = time.monotonic() - self._start
        if self._parent is not None:
            self._parent.substract(delta)

    print(self._name, 'total', delta)
    print(self._name, 'outer', delta - self._substracted)

    return False

    def child(self, name):
        return type(self)(name, parent=self)

    def substract(self, n):
        self._substracted += n


timeit = TimeItContextManager


def main():
    with timeit('large') as large_t:
        with large_t.child('medium') as medium_t:
            with medium_t.child('small-1'):
                time.sleep(1)
            with medium_t.child('small-2'):
                time.sleep(1)
        time.sleep(1)
    time.sleep(1)


main()


一連の呼び出しに関する情報を渡す必要がある場合、最初に頭に浮かぶのは、関数の引数の形式でデータを渡すことです。

場合によっては、チェーン内のすべての関数を変更して新しいデータを転送する方がはるかに便利です。代わりに、チェーン内のすべての関数で使用されるコンテキストを指定できます。どうやってするの？

最も簡単な解決策は、グローバル変数を使用することです。 Pythonでは、厳密にはグローバル変数でもあるため、コンテキストキーパーとしてモジュールとクラスを使用することもできます。おそらく、たとえばジャーナリングのために、おそらくこれを定期的に行っています。

アプリケーションがマルチスレッド化されている場合、通常のグローバル変数はスレッドセーフではないため機能しません。それぞれの時点で、呼び出しのチェーンをいくつか持つことができ、それぞれに独自のコンテキストが必要です。モジュールはあなたを助けthreading、threading.local()スレッドセーフなオブジェクトを提供します。データをそこに格納すると、属性への単純な参照になりますthreading.local().symbol = '@'。

ただし、説明されている両方のアプローチは、同時実行に対して安全ではありません。つまり、システムが関数を呼び出すだけでなく、それらが実行されることも想定しているコルーチン呼び出しチェーンには適していません。コルーチンの実行中awaitに、イベントフローが別のチェーンの別のコルーチンをトリガーすることがあります。これは機能しません：

import asyncio
import sys

global_symbol = '.'

async def indication(timeout):
    while True:
        print(global_symbol, end='')
        sys.stdout.flush()
        await asyncio.sleep(timeout)

async def sleep(t, indication_t, symbol='.'):
    loop = asyncio.get_event_loop()

    global global_symbol
    global_symbol = symbol
    task = loop.create_task(
            indication(indication_t)
    )
    await asyncio.sleep(t)
    task.cancel()

loop = asyncio.get_event_loop()
loop.run_until_complete(asyncio.gather(
    sleep(1, 0.1, '0'),
    sleep(1, 0.1, 'a'),
    sleep(1, 0.1, 'b'),
    sleep(1, 0.1, 'c'),
))

これを修正するには、コルーチンを切り替えるたびに、コンテキストを強制的に設定および復元します。この動作contextvarsは、Python 3.7以降で利用可能なモジュールを使用して実装できます。

import asyncio
import sys
import contextvars

global_symbol = contextvars.ContextVar('symbol')

async def indication(timeout):
    while True:
        print(global_symbol.get(), end='')
        sys.stdout.flush()
        await asyncio.sleep(timeout)

async def sleep(t, indication_t, symbol='.'):
    loop = asyncio.get_event_loop()

    global_symbol.set(symbol)
    task = loop.create_task(indication(indication_t))
    await asyncio.sleep(t)
    task.cancel()

loop = asyncio.get_event_loop()
loop.run_until_complete(asyncio.gather(
    sleep(1, 0.1, '0'),
    sleep(1, 0.1, 'a'),
    sleep(1, 0.1, 'b'),
    sleep(1, 0.1, 'c'),
))
