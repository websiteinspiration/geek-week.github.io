A / Bテストとは何ですか？


A / Bテストは、市場調査のよく知られた方法です。簡単に言えば、サイトのオーディエンスをいくつかの等しいまたは等しくないコホートに分割し、サイトで1つの変更/改善を実行します。その結果、どのページがビジネスおよび製品全体のタスクをより効果的に解決するかがわかります。1つのテストのフレームワークで1つの仮説（最大2つ）のみをチェックします。

この例では、一部の製品カテゴリのワンクリック購入ボタンを公開しました。実験中、彼らは指標と目標到達プロセスを調べ、ユーザーが「ゴミ箱」を「通過」せずに、つまり不要なジェスチャーをせずにスクリプトをワンクリックで使用する方がよいかどうかについての仮説をテストしました。製品カテゴリが偶然に選択されたのではなく、統計によると、1つの製品だけで購入されることが最も多い製品に関するものでした。



原則として、A / Bテストでは、次のような指標を確認します。

  • 変換（実行された主要なアクションの共有）;
  • 財務指標（GMVの成長、収益、平均請求額）;

Alexander Povarov氏によると、Booking.comの成功はもちろん称賛に値します。しかし、品質テストについて話している場合、それらは多くのデータを必要とします。 Booking.comには多くのユーザーがいて、一度に多くのことを試すことができます。中規模のプロジェクトについて話している場合、そのようなプロジェクトではデータとトラフィックが十分でないことがよくあります。トラフィックが屋根より高い場合、これは万能薬ではありません。これは、ユーザーがサイトにアクセスして、製品の詳細に関連するさまざまなユーザーケースを解決できるためです。これがインターネットバンクである場合、誰かが電話の代金を支払うようになり、誰かがそのステートメントを確認します。等。さらに、多くの場合、1つのA / Bテストで1つのユーザーアクションのみをテストします。

簡単な結論を述べると、A / Bテストには以下があります制限：

  • 大量のデータが必要です。
  • 同種のユーザーの場合にのみ適しています。
  • 1つのステップ（クリック）に焦点を当てています。

次のポイント-A / Bテストは複数の成長を与えません：

  • 1 ppの変換増加（たとえば、1％→2％）;
  • 30回繰り返しても1％→31％成長しない;
  • 隣接する漏斗が劣化する可能性があります。
  • 変換は改善されますが、1つのステップで行われます。

複雑な製品について話している場合、A / Bテストで遊ぶと、隣接する目標到達プロセスをフックする可能性が高く、それらへの変換が失敗する可能性があります。最も一般的な例は、大規模なサービスのプロモーションページでいくつかの機能が販売されている場合です。一つを強調すると、他への変換は確実にパスします。しかし、すべてが順調に進んでも、毎日製品を使用するユーザーとのやり取りの歴史が始まり、何らかの形で改善したコンバージョンは製品にまったく影響しません。

A / Bテストの機能とアプリケーション制限：

  1. 多数の聴衆（DAU、MAU）;
  2. テストでは1つ（最大2つ）の仮説のみ。
  3. 1％の成長レベルで10％のテストで製品が成功し、90％のテストで結果が得られなかった。
  4. これらのビジネス指標の指標への影響は限定的です。
  5. B2B製品には適用不可、フィンテック製品には適用不可（アカウントをクリックするたびに）。

もちろん、同時に、スプリットテストは、あらゆる質問に対する最も透過的な答えであり、直感に頼らず、ユーザーのことを考えない機能です。さらに、A / Bテストはまた、あなたが既存の製品の洞察を得ることができることに基づいて、良い友達です。それでも、A / Bテストのおかげで、製品のメトリックとNPSが改善されたため、安価で、既存の製品の成長点を探している場合は、常にテストを使用することをお勧めします。しかし、それでも、A / Bテストは、現在のビジネスモデルと製品から最大限を引き出すことに関するものです。
