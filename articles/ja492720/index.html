diskussion：プロジェクトファイルサービス

GitHub / GitLabを自由に使えるプログラマーはうらやましいだけです-共同開発は非常に便利です。


たとえば、建設（エンジニアリングネットワーク）の設計者は、労力の結果として「ファイルのセット」も見るのでしょうか。原則として、これは「昔ながらの方法」です。「サーバー」上のファイル、フラッシュドライブ、メールまたはファイル共有、および会議-電話や電子メールによるディスカッション。請負業者がまだ関与している場合は、作業が終了するまでに、すべてがゆっくりと混乱に陥っています。重要な文字がスキップされたり、リクエストが届かないなどです。したがって、調整されていない作業が原因で締め切りに間に合わないことはよくあることであり、誰もがこれに同意しているようです。
