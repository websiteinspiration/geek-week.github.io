GitlabCIで使用される依存関係チェックライブラリでの脆弱性スキャナーの使用

私はルクソフト出身です。
脆弱性管理の重要な部分は、最新のシステムを構成するソフトウェアコンポーネントのサプライチェーンのセキュリティを理解して保証することです。アジャイルチームとDevOpsは、オープンソースライブラリとフレームワークを広範囲に利用して、開発時間とコストを削減します。しかし、このメダルには欠点があります。それは、他の人の過ちや脆弱性を引き継ぐ能力です。


明らかに、チームはどのオープンソースコンポーネントがアプリケーションに含まれているかを明確に把握し、明らかに信頼できるソースから明らかに信頼できるバージョンをダウンロードし、新しく発見された脆弱性を修正した後にコンポーネントの更新バージョンをダウンロードする必要があります。


この投稿では、OWASP依存関係チェックを使用して、コードで深刻な問題が発生した場合にビルドを中断する方法について説明します。


「アジャイルプロジェクトにおける開発セキュリティ」という本は次のように説明されています。OWASP Dependency Checkは、アプリケーションで使用されているすべてのオープンソースコンポーネントをカタログ化し、それらに存在する脆弱性を示す無料のスキャナーです。Java、.NET、Ruby（gemspec）、PHP（composer）、Node.js、Python用のバージョンのほか、一部のC / C ++プロジェクト用のバージョンがあります。Dependency Checkは、Ant、Maven、Gradleなどの一般的なビルドツールや、Jenkinsなどの継続的インテグレーションサーバーと統合します。


依存関係チェックは、NIST National Vulnerability Database（NVD）から既知の脆弱性を持つすべてのコンポーネントを報告し、NVDニュースチャネルからのデータに基づいて更新されます。


幸いなことに、このすべては、このようなOWASP依存性チェックのプロジェクト、またはのような商業用プログラムなどのツール使用して自動的に行うことができブラックダック、JFrog X線、Snyk、Sonatypeのネクサスライフサイクル、またはSourceClearを。


これらのツールをアセンブリパイプラインに含めて、オープンソースの依存関係リストを自動的にコンパイルし、古いバージョンのライブラリと既知の脆弱性を含むライブラリを特定し、深刻な問題が見つかった場合はビルドを中断することができます。

mvnが開始される長い行を作成しないようにするために、別の変数DEPENDENCY_OPTSでパラメーターを発行します。


最終的な最小限の構成.gitlab-ci.ymlは次のようになります。 


variables:
  MAVEN_OPTS: "-Dhttps.protocols=TLSv1.2 -Dmaven.repo.local=$CI_PROJECT_DIR/.m2/repository -Dorg.slf4j.simpleLogger.log.org.apache.maven.cli.transfer.Slf4jMavenTransferListener=WARN -Dorg.slf4j.simpleLogger.showDateTime=true -Djava.awt.headless=true"
  MAVEN_CLI_OPTS: "--batch-mode --errors --fail-at-end --show-version -DinstallAtEnd=true -DdeployAtEnd=true"
  DEPENDENCY_OPTS: "-DfailBuildOnCVSS=7 -DcveUrlModified=http://localhost:12345/nvdcve-1.1-modified.json.gz -DcveUrlBase=http://localhost:12345/nvdcve-1.1-%d.json.gz"

cache:
  paths:
    - .m2/repository

verify:
  stage: test
  script:
    - set +e
    - mvn $MAVEN_CLI_OPTS install org.owasp:dependency-check-maven:check $DEPENDENCY_OPTS || EXIT_CODE=$?
    - export PATH_WITHOUT_HOME=$(pwd | sed -e "s/\/home\/gitlab-runner\/builds//g")
    - echo "************************* URL Dependency-check-report.html *************************"
    - echo "http://$HOSTNAME:9999$PATH_WITHOUT_HOME/target/dependency-check-report.html"
    - set -e
    - exit ${EXIT_CODE}
  tags:
    - shell
