Implantação rápida do vm ESXi com Terraform

Olá pessoal, meu nome é Ivan e sou administrador de sistemas alcoólicos (OPS).

Gostaria de dizer como implantar máquinas virtuais no ESXi sem o vCenter usando o Terraform.

Muitas vezes, você precisa implantar / recriar máquinas virtuais para testar este ou aquele aplicativo. Por causa da preguiça, pensei em automatizar o processo. Minhas pesquisas me levaram a um maravilhoso produto hashicorp , terraform .

Acho que muitas pessoas sabem o que é o Terraform, e quem não sabe, esse é um aplicativo para gerenciar qualquer nuvem, infraestrutura ou serviço usando o conceito de IasC ( Infraestrutura como código ).

Como ambiente de virtualização, eu uso o ESXi. Muito simples, conveniente e confiável.
Eu prevejo uma pergunta.
Por que terraforma se você pode usar o vCenter Server?
Você pode, é claro, mas. Em primeiro lugar, esta é uma licença adicional; em segundo lugar, este produto consome muitos recursos e simplesmente não se encaixa no meu servidor doméstico e, em terceiro lugar, a capacidade de atualizar as habilidades.

O servidor é a plataforma Intel NUC:

CPU: 2 CPUs x Intel(R) Core(TM) i3-4010U CPU @ 1.70GHz
RAM: 8Gb
HDD: 500Gb
ESXi version: ESXi-6.5.0-4564106-standard (VMware, Inc.)

E então, as primeiras coisas primeiro.

Por enquanto, vamos configurar o esxi, a saber, abrir a porta VNC nas configurações do firewall.

Por padrão, o arquivo está protegido contra gravação. Realizamos as seguintes manipulações:

chmod 644 /etc/vmware/firewall/service.xml
chmod +t /etc/vmware/firewall/service.xml
vi /etc/vmware/firewall/service.xml

adicione o seguinte bloco ao final do arquivo:

<service id="1000">
  <id>packer-vnc</id>
  <rule id="0000">
    <direction>inbound</direction>
    <protocol>tcp</protocol>
    <porttype>dst</porttype>
    <port>
      <begin>5900</begin>
      <end>6000</end>
    </port>
  </rule>
  <enabled>true</enabled>
  <required>true</required>
</service>

Saia, salve. Mudamos os direitos de volta e reiniciamos o serviço:

chmod 444 /etc/vmware/firewall/service.xml
esxcli network firewall refresh

Nesse caso, o VNC é necessário para conectar-se à máquina virtual e especificar o caminho para o arquivo kickstart.

Na verdade, antes de reiniciar o host. Depois disso, essa manipulação terá que ser repetida.

Além disso, executarei todo o trabalho em uma máquina virtual no mesmo servidor.

Características:

OS: Centos 7 x86_64 minimal
RAM: 1GB
HDD: 20GB
Selinux: disable
firewalld: disable

Em seguida, precisamos do empacotador , também um produto da HashiCorp.

Ele é necessário para a montagem automática da imagem "dourada". Que usaremos no futuro.

yum install unzip git -y
curl -O https://releases.hashicorp.com/packer/1.5.5/packer_1.5.5_linux_amd64.zip
unzip packer_1.5.5_linux_amd64.zip -d /usr/bin && rm -rf packer_1.5.5_linux_amd64.zip
packer version
Packer v1.5.5

Pode ocorrer um erro na etapa da versão do empacotador , pois um pacote com o mesmo nome pode estar no RedHat.

which -a packer
/usr/sbin/packer

Para resolver, você pode criar um link simbólico ou usar o caminho absoluto / usr / bin / packer.

Em um gita, preparei tudo o que precisava.

git clone https://github.com/letnab/create-and-deploy-esxi.git && cd create-and-deploy-esxi

Na pasta iso, você precisa colocar a distribuição do sistema operacional. No meu caso, é o centos 7.

Também é necessário editar o arquivo centos-7-base.json :

variables:     
iso_urls:  
iso_checksum:

Após todas as alterações, execute a montagem:

/usr/bin/packer build centos-7-base.json

Após a conclusão bem-sucedida, o arquivo ova será localizado na pasta output-packer-centos7-x86_64 .

Instale o Terraform:

curl -O https://releases.hashicorp.com/terraform/0.12.24/terraform_0.12.24_linux_amd64.zip
unzip terraform_0.12.24_linux_amd64.zip -d /usr/bin/ && rm -rf terraform_0.12.24_linux_amd64.zip
terraform version
Terraform v0.12.24

Como o Terraform não possui um provedor para ESXi, é necessário construí-lo.

Coloque ir:

cd /tmp
curl -O https://dl.google.com/go/go1.14.2.linux-amd64.tar.gz
tar -C /usr/local -xzf go1.14.2.linux-amd64.tar.gz && rm -rf go1.14.2.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin
go version
go version go1.14.2 linux/amd64

Em seguida, coletamos o provedor:

go get -u -v golang.org/x/crypto/ssh
go get -u -v github.com/hashicorp/terraform
go get -u -v github.com/josenk/terraform-provider-esxi
export GOPATH="$HOME/go"
cd $GOPATH/src/github.com/josenk/terraform-provider-esxi
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-w -extldflags "-static"' -o terraform-provider-esxi_`cat version`
cp terraform-provider-esxi_`cat version` /usr/bin

Estamos na linha de chegada. Vamos divulgar nossa imagem.

Vá para a pasta:

cd /root/create-and-deploy-esxi/centos7

Primeiro de tudo, edite o arquivo variable.tf . Você deve especificar uma conexão com o servidor ESXi.

O arquivo network_config.cfg contém as configurações de rede da futura máquina virtual. Mudamos para nossas necessidades e executamos uma linha:

sed -i -e '2d' -e '3i "network": "'$(gzip < network_config.cfg| base64 | tr -d '\n')'",' metadata.json

Bem, no arquivo main.tf, altere o caminho do arquivo ova para o seu próprio, se diferente.
