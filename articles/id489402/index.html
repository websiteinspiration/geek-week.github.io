Pratinjau Terminal Windows v0.9

Pelepasan versi 0.9 Terminal Windows telah terjadi. Ini adalah versi terbaru dari Terminal, yang akan menyertakan fitur-fitur baru sebelum rilis v1. Anda dapat mengunduh Terminal Windows dari Microsoft Store atau dari halaman rilis di GitHub . Mari kita melihat lebih dekat pada detail pembaruan!



Argumen baris perintah


Alias WT sekarang mendukung perintah argumen baris. Sekarang Anda dapat meluncurkan Terminal dengan tab dan panel baru, dibagi sesuka Anda, dengan profil yang Anda sukai, dimulai dengan direktori yang Anda sukai! Kemungkinannya tidak terbatas! Berikut adalah beberapa contoh:

wt -d.
Buka terminal dengan profil default di direktori kerja saat ini.

wt -d.; new-tab -d C: \ pwsh.exe
Membuka Terminal dengan dua tab. Di yang pertama - profil default, mulai dari direktori kerja saat ini. Yang kedua adalah profil default dengan pwsh.exe sebagai "commandline" (bukan profil default "commandline") yang dimulai di direktori C: \.

wt -p "Windows PowerShell" -d.; split-pane -V wsl.exe
Buka Terminal dengan dua panel yang dipisahkan secara vertikal. Profil dengan nama "Terminal Windows" diluncurkan di panel atas, dan profil default digunakan di panel bawah, menggunakan wsl.exe sebagai "commandline" (bukan profil default "commandline").

wt -d C: \ Users \ cinnamon \ GitHub \ WindowsTerminal; split-pane -p "Command Prompt"; split-pane -p "Ubuntu" -d \\ wsl $ \ Ubuntu \ home \ cinnak -H
Lihat di bawah.



Jika Anda ingin mengetahui semua yang dapat Anda lakukan dengan argumen baris perintah baru kami, lihat dokumentasi lengkapnya di sini .

PowerShell Autodiscover


Jika Anda penggemar berat PowerShell Core , kami memiliki kabar baik untuk Anda. Terminal Windows sekarang mendeteksi versi PowerShell dan secara otomatis membuat profil baru untuk Anda. Versi PowerShell, yang, menurut kami, adalah yang terbaik (mulai dari nomor versi terbesar, hingga versi GA, ke versi dengan bundel terbaik), akan disebut "PowerShell" dan akan menempati slot PowerShell Core asli dalam daftar drop-down.



Konfirmasikan penutupan semua tab


Apakah Anda orang yang tidak ingin ditanya setiap kali menutup semua tab? Jika Anda menjawab ya, maka fitur baru ini pasti untuk Anda! Parameter global baru telah dibuat yang memungkinkan Anda untuk selalu menyembunyikan kotak dialog konfirmasi "Tutup Semua Tab". Untuk melakukan ini, atur parameter “confirmCloseAllTabs” menjadi false di bagian atas file profiles.json, dan Anda tidak akan pernah melihat pop-up ini lagi! Terima kasih @ rstat1 untuk kontribusi parameter baru ini.
