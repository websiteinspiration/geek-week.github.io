[ピーター]クリフクリックでJUG.ruに会う-JVMはそれを行うのか？

Hydra 2019カンファレンスの2日前の7月9日火曜日、JUG.ruは、有名なスタートアップ（Rocket Realtime School、Neurensic、H2O.ai）の創設者であり共同創設者であるCratusのCTOであるCliff Click（Cliff Click）と面会しますスピーカーとコンサルタント。

Cliff ClickはJavaの世界の伝説です。彼は、JVMの初期開発に参加し、有名なHotSpot Server Compiler（C2）の作成者の1人であり、Azulテクノロジースタックの重要な部分を改善し、JITコンパイルには生きる権利があることを世界に証明しました。

参加は無料ですが、登録が必要です。こちらから登録できます。会議のトピックとスピーカーに関する詳細情報はカットされています。

レポートは英語で読まれます。


クリフ自身がこの会議について書いている内容は次のとおりです。

「私はHotSpot JVMに約15年間取り組んできましたが、内部の状況には常に驚かされています。高品質のGC、コンパイラ、メモリとマルチスレッドのユニバーサルモデル、コードコストモデル、動的なクラスルーム、イントロスペクションとリフレクション、オペレーティングシステムとハードウェアリソースへのアクセス、巨大な標準ライブラリ。これはどこから来たのですか？なぜ彼らはこれらすべてをそこに追加したのでしょうか...そして、最も重要なのは、いつ失敗するのですか？私たちが考慮し、一緒に暮らす必要があるJVMの制限は何ですか？どのようなサービスを希望しますか（ただし、サービスは提供されません-その理由は？）」

これらすべての質問には、いくつかの新興企業（Rocket Realtime School、Neurensic、H2O.aiなど）の創設者であり共同創設者であるCratus（プロセス改善のためのIoTセンサー）のCTOであるCliff Clickが回答します。クリフは15歳（TRS Z-80のパスカル）で最初のコンパイラーを書きました。 Java（Sea of​​ Nodes IR）でのC2の作業で最もよく知られています。このコンパイラーは、JITが高品質のコードを生成できることを世界に示し、これがJavaを主要な最新プラットフォームの1つにした要因の1つでした。 Cliffは、Azul Systemsが500ギガバイトのヒープで10ミリ秒のGC一時停止をサポートし、JVMのすべての面で一般的に機能する純粋なJavaソフトウェアを備えた864コアのメインフレームの構築を支援しました。クリフは定期的に学会などで報告を受け、実践的なエンジニア向けの会議だけでなく、彼が個人的に書いたHotSpotに関する多くの資料を見つけることができます。
