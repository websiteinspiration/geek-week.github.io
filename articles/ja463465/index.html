. reduce


配列でよく行われるもう1つの一般的なことは、それらに基づいて単一の値を計算することです。この特殊なケースは、数値セットの合計ですでに使用した例です。別の例は、最大数の文字を含むフォントを見つけることです。

このパターンを実装するより高次の演算は、略記法と呼ばれます（たたみ込みとも呼ばれます）。この操作は、配列から1つの要素を繰り返し取得し、それを現在の値と組み合わせることにより、値を構築します。数値を合計するときは、ゼロから開始して、後続の各要素を合計に追加します。

配列に加えて、reduce関数のパラメーターは、結合関数と初期値です。この関数はフィルターとマップよりも少し複雑なので、詳しく見てみましょう。

function reduce(array, combine, start) {
   let current = start;
   for (let element of array) {
      current = combine(current, element);
   }
   return current;
}

console.log(reduce([1, 2, 3, 4], (a, b) => a + b, 0));
// → 10

もちろん、この関数に対応するreduce配列を操作する標準的な方法には、さらに便利な機能があります。配列に少なくとも1つの要素が含まれている場合は、開始引数を省略できます。このメソッドは、配列の最初の要素を初期値として選択し、2番目の要素から削減を開始します。

console.log([1, 2, 3, 4].reduce((a, b) => a + b));
// → 10

reduce（2回）を使用して文字数が最も多いフォントを見つけるには、次のように記述します。

function characterCount(script) {
   return script.ranges.reduce((count, [from, to]) => {
      return count + (to — from);
   }, 0);
}

console.log(SCRIPTS.reduce((a, b) => {
   return characterCount(a) < characterCount(b) ? b : a;
}));
// → {name: "Han", ...}

characterCount関数は、サイズの合計を計算することにより、このフォントに割り当てられた範囲を減らします。リダクション関数のパラメーターリストでの構造化の使用に注意してください。次に、reduceへの2番目の呼び出しは、前の結果を使用して最大のフォントを見つけ、2つのフォントを繰り返し比較して、大きい方のフォントを返します。

Hanフォントには、Unicode標準で89,000文字以上が割り当てられており、データセットの中で最大の書記体系になっています。漢字は、中国語、日本語、韓国語のテキストに使用されるフォントです。彼らの言語は異なる方法で書かれていますが、多くの共通の特徴があります。（米国にある）Unicodeコンソーシアムは、文字コードを保存するために、そのような文字を単一の記録システムと見なすことを決定しました。これは漢統一と呼ばれ、まだ一部の人々にとって非常に迷惑です。

構成可能性


考えてみましょう：高次関数なしで前の例（最大のフォントを見つける）をどのように書き直すことができますか？次のコードはそれほど悪くありません。

let biggest = null;
for (let script of SCRIPTS) {
   if (biggest == null ||
       characterCount(biggest) < characterCount(script)) {
     biggest = script;
   }
}
console.log(biggest);
// → {name: "Han", ...}

いくつかの追加のバインディングが表示され、プログラムは4行長くなりました。しかし、このコードはまだかなり明確です。

高次関数は、操作を構成する必要がある場合に非常に役立ちます。例として、データセット内の生きている言語と死んだ言語のフォントを作成する平均年を計算するコードを書いてみましょう。

function average(array) {
   return array.reduce((a, b) => a + b) / array.length;
}

console.log(Math.round(average(
   SCRIPTS.filter(s => s.living).map(s => s.year))));
// → 1188
console.log(Math.round(average(
   SCRIPTS.filter(s => !s.living).map(s => s.year))));
// → 188

したがって、Unicodeでの死んだ言語のスクリプトは、生きている言語のスクリプトよりも平均して古くなっています。

これらは特に重要な、または驚くべき統計ではありません。しかし、うまくいけば、それを計算するために使用されるコードが読みやすいことに同意します。これはコンベヤと考えることができます。まず、すべてのフォントを分析し、生きている（または死んでいる）フィルタを取り、作成された年数を取り、平均値を計算し、結果を四捨五入します。

この計算は、1つの大きなサイクルとして表すこともできます。

let total = 0, count = 0;
for (let script of SCRIPTS) {
   if (script.living) {
     total += script.year;
     count += 1;
   }
}
console.log(Math.round(total / count));
// → 1188

しかし、このコードでは、何がどのように計算されるのかを理解することがより困難です。また、中間結果は一貫した値として提示されないため、平均などを別の関数に分離するには、さらに多くの作業を行う必要があります。

コンピュータが実際に行うことに関しては、2つのアプローチも根本的に異なります。1つ目はフィルターとマップの実行時に新しい配列を作成し、2つ目は一部の数値のみを計算するため、処理量が少なくなります。通常は、より読みやすいオプションを用意できますが、非常に大きな配列を処理して何度も処理する必要がある場合は、抽象的なスタイルではないほど速度が向上します。

文字列と文字コード


データセットの1つの使用法は、特定のテキストが入力されるフォントを決定することです。これを行うプログラムを見てみましょう。

各フォントには文字コード範囲の配列があることを思い出してください。したがって、文字コードがわかれば、次の関数を使用して対応するフォント（存在する場合）を見つけることができます。

function characterScript(code) {
   for (let script of SCRIPTS) {
      if (script.ranges.some(([from, to]) => {
        return code >= from && code < to;
      })) {
        return script;
      }
   }
      return null;
}

console.log(characterScript(121));
// → {name: "Latin", ...}

いくつかの方法は別の高次関数です。テスト関数を受け取り、配列のいずれかの要素に対してtrueを返すかどうかを報告します。

しかし、文字コードを文字列として取得するにはどうすればよいでしょうか。

第1章で、JavaScriptでは文字列が16ビットの数値のシーケンスとして表されることを述べました。これらは、いわゆるコード単位です。当初、Unicodeでは、文字コードはそのようなブロック（65,000文字強）に配置されると想定されていました。これでは不十分であることが明らかになったとき、多くの人が1つの文字を格納するためにより多くのメモリを使用する必要性に反対し始めました。この問題を解決するために、JavaScript文字列で使用されるUTF-16形式が発明されました。その中で、最も一般的な文字は1つの16ビットコード単位を占め、残りは2つのコード単位です。

今日では、UTF-16は悪い考えであると一般に認められています。エラーを発生させるために作成されたようです。コード単位と文字が同一のプログラムを簡単に書くことができます。また、母国語が2つのコード単位を占める文字を使用しない場合、このプログラムは正常に動作します。しかし、誰かがそのようなプログラムをあまり一般的でないアルファベット、たとえば漢字に使用しようとするとすぐに、すぐに壊れてしまいます。幸いなことに、絵文字の出現後、2つのコード単位がどこでも文字エンコードに使用されるようになり、そのような問題を解決する負担がより公平に分散されました。

残念ながら、JavaScript文字列を使用した明らかな操作（長さプロパティから長さを取得する、角かっこで内容にアクセスするなど）は、コード単位のみを扱います。



JavaScriptのcharCodeAtメソッドは、完全な文字コードではなくコード単位を返します。後で表示されるcodePointAtメソッドは、完全なUnicode文字を返します。したがって、これを使用して文字列から文字を取得できます。ただし、codePointAtに渡される引数は、引き続きコードユニットのシーケンスのインデックスです。したがって、文字列内のすべての文字を反復処理するために、1つまたは2つのコード単位が文字を占めるかどうかの問題を解決する必要があります。

前の章で、for / ofループは文字列にも使用できることを述べました。codePointAtのように、このタイプのループは、プログラマーがUTF-16の問題を明確に認識したときに現れました。このループを文字列に適用すると、コード単位ではなく実際の文字が得られます。



文字（1つまたは2つのコード単位の文字列）がある場合、そのコードを取得するには、codePointAt（0）を使用できます。

テキスト認識


characterScript関数と、ループ内の文字を正しく列挙する方法があります。次のステップは、各フォントに属する文字の数を数えることです。ここでは、カウントの抽象化が必要です。

function countBy(items, groupName) {
   let counts = [];
   for (let item of items) {
      let name = groupName(item);
      let known = counts.findIndex(c => c.name == name);
      if (known == -1) {
        counts.push({name, count: 1});
      } else {
        counts[known].count++;
      }
   }
   return counts;
}

console.log(countBy([1, 2, 3, 4, 5], n => n > 2));
// → [{name: false, count: 2}, {name: true, count: 3}]

countBy関数は、コレクション（for / ofループでソートできるすべてのもの）と特定の要素のグループ名を計算する関数を受け入れます。 countBy関数は、オブジェクトの配列を返します。各オブジェクトには、グループの名前とそのグループで見つかった要素の数が含まれています。

この関数は、配列を操作する別の方法-findIndexを使用します。このメソッドはindexOfに似ていますが、特定の値を検索する代わりに、指定された関数がtrueを返す最初の値を見つけます。アイテムが見つからない場合、findIndexはindexOfと同様に-1を返します。

countByを使用して、このテキストで使用されているフォントを通知する関数を作成できます。



この関数は、最初にフォント名で文字をカウントし、characterScriptを使用してそれらに名前を付け、どのフォントにも属さない文字については文字列「none」を返します。filterを呼び出すと、これらの文字には関心がないため、結果の配列から「none」エントリが削除されます。

パーセンテージを計算できるようにするには、まず、reduceメソッドを使用して計算できるフォントに属する文字の総数を取得する必要があります。そのような文字が見つからない場合、関数は特定の文字列を返します。それ以外の場合は、マップを使用してカウント結果を読み取り可能な文字列に変換し、結合を使用してそれらを結合します。

概要


関数値を他の関数に渡す機能は、JavaScriptの非常に便利な側面です。これにより、スペースを使用した計算をシミュレートする関数を作成できます。その後、そのような関数をコードで呼び出すと、これらの「ギャップ」は関数値で埋められます。

配列には、いくつかの便利な高次メソッドがあります。 forEachメソッドを使用して、配列の要素をループできます。 filterメソッドは、述語関数の条件を満たす要素のみを含む新しい配列を返します。各要素に対して関数を実行することによる配列変換は、mapを使用して行われます。配列のすべての要素を1つの値に結合するには、reduceを使用できます。 someメソッドは、要素が指定された述語関数と一致するかどうかを確認します。最後に、findIndexメソッドは、述語に一致する最初の要素の位置を見つけます。
