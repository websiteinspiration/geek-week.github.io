いいえ、（2つの変数の）汎用の乗数がなければ、確かに実行できませんでした。私はペンでそれを作らなければならなかった、そして彼は水晶の半分を取った。問題は、彼が私のすべてのステップで脱穀したことです。窓はありません。変数の乗算に加えて、アルゴリズムには固定係数があり、これも乗算する必要がありました。そして、これに乗数を使用する方法はありませんでした。窓が残っていないだけです。そして、プロジェクトの闘争は各セルを通過したので、もちろん回避方法に非常に興味があり、これらの係数のプロパティを使用して、それらを乗算する最適なスキームを作成しました（汎用の乗算器ではなく、指定された数の乗算デバイス！）。まあ、完全な夢が実現したとき-異なる係数による乗算のスキームがいくつかの一般的な水晶リソースを最大限に活用できるように。

それで、1年生の仕事からエンジニアと数学者の仕事にスムーズに移りました。与えられた特定の数に最適な乗算スキームを構築する方法は？

通常、AからBまでの数を乗算するため、少し覚えておきましょう。

  1. 最初に、結果をゼロに設定します。
  2. 放電Aを実行します。
  3. 放電0の場合、何もしません。
  4. ビット1の場合、Bを対応するビット数だけシフトし、結果に追加します。

合計では、Aの桁数が少ないほど、乗算が容易になります。 2、4、または8を掛けるのは簡単です。シフトだけが必要です。 3、5、または10を掛けるのは少し難しいです。1加算します。 11を掛けるのはさらに困難です。単位が多いほど、困難になります。

さて、255を掛けるとどうなるでしょうか。最大8ユニット、悪夢のようなタスクですよね？そしてここに置物があります！耳でトリッキーなフェイントを作ってみましょう。Bに256を掛けて（つまり、8桁シフトして）、結果から減算します。見事な外観にもかかわらず、255を掛けるのは10を掛けるのと同じくらい簡単です。254、240、および224を掛けるのも同じくらい簡単です（必要に応じてチェックしてください）。

全体として、減算は乗算に役立ちます。この貴重な考えを思い出してください。その間、私たちは難易度を数値は、それを乗算するために必要な加算（または減算）演算の最小数です。シフトは考慮されません。問題のコンテキストでは（プログラミングではなく回路があります！）それらは無料で入手できます。明白だが有用な1つのステートメントを定式化しましょう。

数値の難しさは、その2進表記の単位数以下です。
実際、フェアを差し引くことで乗算するトリックがない場合、両方とも学校で教えられたので、加算の数はバイナリ表記Aのユニットの数に等しくなります。そして、もし私たちがだまし始めたら、運が良ければ、操作の数を減らします。

このステートメントの興味深い結果：
nビットの数値の難易度は常に厳密にnよりも小さくなります。
結局のところ、n桁の数字の最大単位数はn、255などの数値。一方、フォーカスは、255を掛ける場合と同様に、任意の容量の数値に対して繰り返すことができます。これにより、汎用の乗算器を最適化する可能性が広がります。

推論をより定期的に見てみましょう。はじめに、乗算スキームAをすでに持っているとしましょう。1つのように取ります。次に、すべての条件が移動し、発展し、差し引かれ、結果は同じAになります。全体として、Aによる乗算のスキームは、2の累乗の合計の形式での数Aの表現に過ぎず、項は+記号と-記号の両方で使用できます。。そして、正と負の項を一緒にグループ化することが可能であり、その差回路はAに等しい特定の数A +とA-を記述しました。これは悪いことです...違いのメンバーは任意に大きくなる可能性があります。それらを制限する考慮事項があるかどうか考えてみましょうか？
まず、2の累乗はそれぞれ1度しか回路に入力できないことに注意してください。実際、彼女が同じ記号で2回入力した場合、これは2の累乗でより大きい単位に置き換えることができます。彼女が異なる記号で2回入力した場合、それらは相互に減算されます。全体として、このスキームは通常のバイナリ表現Aに非常に似ています。唯一の違いは、彼の「ビット」（以降、私たちはそれらをトリットと呼ぶ））、2つではなく、3つの値を取ることができます：1、0および-1。

合計 Aがnビットの2進数である場合、それによる乗算のスキームは合計で記述されます。

A=tm2m+tm−12m−1+....+t12+t0


どこ ti値、1、0および-1を取るトリット、およびmこれまでに未知の数。

以下では、このような三項図の式、またはAの数の回路または回路の乗算と呼びます。m を見つけよう。 255による乗算の例からわかるように、mは決してn未満ではありません。それがn + 1と等しくなるかどうか見てみましょう。 ましょうAが、以前のように、ポジティブであることN-桁の番号。次に、乗算スキームは次のように定義されます。



A=tn+12n+1+tn2n+tn−12n−1....+t12+t0


それを思い出します 2k+2k−1+....+2+1=2k+1−1
ため、決してトップトリット缶は-1に等しくなります。実際、他のすべてが1であっても、合計は-1のままです。そして条件によってAは正です。ここで、一番上のtritを1にします。ただし、この場合、次のtritは-1にする必要があります。そうしないと、量が多すぎます。実際、次のtritが0の場合、合計は少なくとも 間nは-ビットAは、もはや2n+1−(2n−1+...+1)=2n+1−(2n−1)=2n+1
。しかし、上位のtritが1、次の-1の場合、2つの上位メンバーの合計は2n−1。したがって、トップのトリットは0です。 したがって、重要なステートメントが証明されます：m = n。 言い換えると、nビットAによる乗算のスキームを表すには、n + 1の2のべき乗で十分です。0からn（バイナリ表現の場合は1より大きい）で、すぐに無限大から解放されます。 今、あなたは困難を計算しようとすることができます2n+1−2n=2n

class TriScheme():     
    def __init__(self, buf:list): #   
        self.buf=[]
        for i in range(0, len(buf)):
            s=0
            if (type(buf[i]) == int) or (type(buf[i]) == float):
                if buf[i]   > 0: s=1
                elif buf[i] < 0: s=-1
            self.buf.append(s)
            
    def zero(self): # 
        self.buf=[0]*len(self.buf)
        
    def __repr__(self): #    
        s=""
        for i in range(1, len(self.buf)+1):            
            if   self.buf[-i]==1:  s=s+"+"
            elif self.buf[-i]==0:  s=s+"0"
            elif self.buf[-i]==-1: s=s+"-"
        return s
            
    def inc(self):  #  (  )
        for i in range(0, len(self.buf)):
            if (self.buf[i]+1) < 2:
                self.buf[i]+=1
                break
            for j in range(0, i+1):
                self.buf[i]=-1
                
    def __int__(self):  #  
        m=1
        s=0
        for i in range(0, len(self.buf)):
            s+=self.buf[i]*m
            m=m*2
        return s
    
    def isMax(self): #    
        s=0
        for i in range(0, len(self.buf)):
            s+=self.buf[i]
        return (s == len(self.buf))
    
    def isMaxDig(self): # ,   ,  n- 
        s=[0]*len(self.buf)
        s[0]=-1
        s[-1]=1
        return (self.buf == s)
        
    def ones(self): #     ( )
        s=0
        for i in range(0, len(self.buf)):
            if self.buf[i] != 0:
                s+=1
        return s
    
    def minPos(self):  #     
        for i in range(0, len(self.buf)):
            if self.buf[i] != 0:
                return i
                         
def allSchemes(dig): #        dig
    sch=[]
    for i in range(0, 2**dig): sch.append([])

        if ts.isMaxDig():
            break
        ts.inc()
    return sch

すべてうまくいくでしょうが、上記のアルゴリズムは非常に高価です。その複雑さは3n数字の長さに応じて、N。 8桁の場合は瞬時にカウントされます。 14分間。 16時間。悲しいかな、すべてのnビット数の回路を読み取る必要がある場合は、それをCに書き換えてより強力なコンピュータにする以外に何もできません。ただし、アルゴリズムは完全に並列化されており、GPUまたはクラスター上で確実に実行できます。

特定の腺を設計するには、通常、特定の数のスキームのリストが必要です。そして、ALLは望ましいだけでなく、最適でもありません。どちらを選択するかは、状況に応じて異なります（たとえば、いくつかの定数を乗算するデバイス）。そしてここでは、より人道的なアルゴリズムを提供できます。もう一度、私たちは驚くべき平等を思い出します：2k+2k−1+....+2+1=2k+1−1。

タスクのコンテキストでは、これは以下を意味します。いくつかの上級サーキットを知ってみましょう。位置kで始まるすべてのマイナートライは不明であり、以前はゼロと見なされていました。次に、未知のトリットを何らかの方法で変更し、回路の値をプラス/マイナスだけ変更します2k+1−1。さらに、下層階級への進歩に伴い、この不確実性の価値は減少します。これにより、逐次近似によってスキームを検索できます。

正の数Aが与えられます。それのためのすべてのスキームをnビット数の中から見つける必要があります。Aと特定の回路の値の差の絶対値は、回路の誤差と呼ばれます。まず、nビットの数値を記述するn + 1ビットのスキームを採用します。これらはすべて不明であり、最初はゼロに設定されています。そして、古いものから始めて、1つずつトリットを定義していきましょう。ではK目の位置は、スキーム3つの新しいスキームを生成することができる-の値とK-番目トリット1、0、-1。継続するスキームのリストには、エラーが超えないスキームのみを残します2k−1。結果の回路リストを使用して、位置k-1のステップに進みましょう。したがって、結果リスト（位置0）ではALL回路が可能であり、その値はAに等しくなります。そのような関数calcSchemesを書いてみましょう。

calcSchemes
def calcSchemes(a, n=-1):
    m=1
    nn=1
    while m < a:
        nn+=1
        m=m*2
    if n < nn:
        n=nn
    sch=[TriScheme([0]*n)]
    for i in range(0, n):
        tmp=[]
        pos=n-i-1
        for j in range(0, len(sch)):
            for k in range(-1, 2):
                ts=sch[j]
                ts.buf[pos]=k
                d=abs(a - int(ts))
                if d <= (2**pos - 1):
                    tmp.append(TriScheme(ts.buf))
        sch=tmp   
    return sch
