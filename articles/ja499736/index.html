サンクトペテルブルクHSEでソフトウェアエンジニアリングの修士号を取得して学んだこと

多くの人は、ソフトウェアエンジニアリングの修士号で何を教えることができるかを理解しておらず、学部を卒業したら、そこで仕事に行き、そこですべてを学ぶ必要があると信じています。SEの修士課程に行く価値はありますか？これは難しい質問です。この記事では答えませんが、強力な学士の卒業生が私たちに教えていることを教えます。



私の名前はオリヤクニャフスカヤです。ソフトウェアエンジニアリングトラックで、高等経済学部-サンクトペテルブルクのデータプログラミングと分析の修士プログラムの1年目です。では、前のシリーズ、私はすでに私のHSEへのパスと、私はここに来たと動機について話しました。

私たちの研究はどうですか


概念的に教えられていること


修士課程にはITの強力な学士号を取得した卒業生のみが受け入れられます。したがって、学部の研究では、数学、アルゴリズム、プログラミングに必要なすべての基本的な知識をすでに取得していると考えられます。ソフトウェアエンジニアリングトラックの目標は、開発におけるさまざまな基本的な方向性を学生に理解させることです。したがって、私たちのコースは、開発に関連するさまざまな知識の分野から選択されます。

学習に影響を与える能力


私たちのプログラムは新しく、コースの内容に影響を与えることができます。分野のセットは年々変化しており、私たちのプログラムはすでに1年前の同僚のプログラムとはかなり異なっています。

私たちは、希望、どのコースに参加したいか（できれば、誰がどこでこのコースを読んだかを具体的に示して）を表明し、行政は同意しようとします。私たちの意見は、来年のコースを誰がどのように読むかにも影響します。各学期の終わりに、アンケートに回答します。その後、キュレーターが結果について話し合い、コースを変更する最善の方法と、来年に残す価値があるかどうかをアドバイスします。多くのコースが初めて教えられるので、このフィードバックはカリキュラムの開発にとって重要です。 

学習機能


HSEに特有の珍しい事柄のうち、研究は他の大学のように学期ではなくモジュールで分類されています。モジュールは2〜3か月で、1年間に4つのトレーニングモジュールがあります。原則として、各モジュールでは約3つの科目を扱います。 

一部の科目は学期が続き、一部は1つのモジュールです。一部の科目は、私たちと私たちのサブディレクションの2年目の学生のみが読むことができます（4人から6人がクラスに参加しています）。Yandex Data Analysis Schoolでいくつかの科目を履修し、他の地域の子供たちと一緒にコースの一部を受講します。 

私たちは理論的なコースとかなり応用されたコースの両方、そしてソフトスキルの開発に関連するコースを用意しました。 

私たちは何を教えられましたか


ソフトウェアエンジニアリングトラックの目標は、開発に関連するさまざまな分野の経験を提供することです。全体像を説明できるかわからないので、今年私たちに教えられたコースについてお話しします。各コースで何が起こったのか、どのような印象を受けましたか。

データ管理アルゴリズム


データベースに関連するアルゴリズムに関する理論的なコース。コースは1モジュール、週に3組続きました。分散データベースでのトランザクション処理アルゴリズムについて話されました。たとえば、トランザクション実行スケジュール、同時実行制御プロトコル、フォール後のデータベース回復の正確さの基準について。

毎週2つの講義と1つのセミナーがありました。各セミナーでは、3人が以前に作成した記事の1つについて20分間のレポートを作成しました。コースに参加したのは7人だけだったので、2週間ごとにレポートを作成する必要がありました。同時に、作成されたレポートは常に積極的に扱われました。 

私にとってこのコースの主な利点は、主な資料とは関係ありませんでした。私が改善した主なスキルは、英語で科学的レポートを作成する能力です。 

コースはBoris Novikovによって英語で完全に読まれました。すべての学生がロシア語を話すという事実にもかかわらず、講義は英語で与えられました、質問は英語で尋ねられるべきでした、そして報告もそれぞれ英語で準備されるべきです。私の生活の中で、私は英語を十分に理解していないため、私は隠されて沈黙する必要があるように見えますが、コースのルールの1つは、「誰でも任意に愚かなことや任意に文盲を言うことができる」でした。したがって、非常に前向きな雰囲気が生まれました。

このコースを終えた後、私は最初に、重要でない情報を英語で問題なく長く伝えることができると信じていました。次に、科学記事から重要な情報を分離するスキルを得ました。  

コンピューターグラフィックス


このコースは1学期向けに設計されており、主にUnityでのレンダリングに専念しました。それはによって読み取られたミハイル・パニン、でtechlide Playneta.gg。主にシェーダーの作成とメッシュの生成に専念する多くの実用的な割り当てがありました。 

このコースを受講している間、オブジェクトの視覚的な認識に関して意識が拡大しました。最後の課題はクラウドの生成でした。そして、それをしている間に、実際には雲を実際に見たことはなく、雲がどのように見えるのか疑問に思いました。なんとかきれいな空を作れるかどうか検証できなかったので、かなりの時間を要し、結局最高点まで合格しませんでした。ある種の雲を生成しています。それは通常の雲であるように見え、教​​師に送信しています-そして、すぐに答えが返ってきます。 。約20の異なるバージョンのクラウドを送信したと思います。


今、私は仕事に行き、空を見て、雲を見て、すぐにくっついて考えます：「うーん...そして、興味深いことに、雲が上下で暗く、逆に明るいのはなぜですか？」それから夜明けで光源が下にあることがわかります。 


拡散反射モデルとその間の何かを実装する必要があった宿題。

コンピューターグラフィックスのコースを終えた後、今は雲をよく見ています。また、レンガの壁や鏡面性の異なるオブジェクトにも適用されます。 

ネットワーク攻撃防御


コースは私たちのサブディレクションだけのために読まれました、合計6人の学生がいました。ネットワークを持ち上げ、Apacheとnginxを設定してロードし、脆弱性を検出して攻撃から防御する方法を学びました。

コースは完全に実用的で、主に私たちの質問への教師の回答と私たちが何をすべきかについての詳細な指示が与えられた実験室の仕事で構成されていました。このコースで理解した主なことは、これが私が扱いたい領域ではないということです。実験室を決めるために座るたびに、最初はすべてが非常にはっきりしているように見えます。その後、多くの落とし穴が発見されます。たとえば、プログラムバージョンの不整合、開いている接続数の予期しない制限、突然、十分な構成がない場所が判明します。さらに、コースを受講したすべての人がさまざまな場所で問題を抱えており、教師の助けがなければ何が問題であるかを理解できなかった。

それはすべてが正常に進んでいるとき、それは主題に関して単純であり、何かが突然故障した場合、それは非常に困難になりました。

有限体代数とコーディング理論


コースは非常にインタラクティブに構築されています。講義や練習はたった5人で行われ、たくさんのコミュニケーションがありました。講義では、次の思考への移行は、出席者全員が前の思考を理解したときにのみ起こりました。私の記憶によると、これは学部の研究の数学コースで行われたため、アイデアや抽象的な概念ではなく、実際のオブジェクトについて議論しました。講義では、最初にいくつかの新しいオブジェクトの例が与えられ、それらを調べ、手でひねり、どのように操作できるかを理解し、いくつかのパターンを見つけました。そして、私たちが新しいオブジェクトに触発されて初めて、教師は定理を定式化して証明し始めました。 

私たちが5人いたので、練習中に誰もがボードで問題を解決することができました。タスクは非常に実用的でした。たとえば、有限体の作成、逆元の決定、何かの暗号化と復号化などです。そのため、知識はより活発になりました。

タスクの例：十分に大きな有限体があり、それについて多くのことを計算する必要があります：要素の次数、逆数、円分クラスなど。最初から、すべてを数える方法がわからず、最も簡単な方法で考えます。数回の反復の後、1桁速く計算できることに気付き、いくつかのプロパティを計算で使用し始めます。あなたはあなた自身、これまたはそのプロパティがどのように機能するか、そしてなぜそれが機能するかを理解しました、そしてあなたはこの理由のためにそれを使用します。 

直感が単に1つまたは別のオブジェクトで動作するように開発されているようなアイテムは本当に十分ではないようです。コースが1モジュールだけ続いたのは残念です。 

ゲーム開発


ゲーム開発の入門コースはPlaykotの支援を受けて、初めて、つまり私たちのために、そして一般的に教えられました。彼はゲーム業界を代表する数人の異なる人々から教えられました。


このコースは、ゲーム開発のさまざまな側面（アニメーション、物理、AI、レンダリングと最適化の高度な側面、ネットワーク）に特化した部分に分かれていました。さらに、関連分野（ゲームデザイン、ゲームの分析、ユーザーインタラクション、アート）に関する概要講義もありました。それを扱う人はそれぞれの領域について話しました。たとえば、Unityに関する一連の講義は、Anton Yakovlevによって行われました。彼はUnityで物理学のリーダーとして働いており、PointOmega Gamesチームに代わってゲームを作り、さまざまな講義を行っています。 Unityでアニメーションを作成する方法、プレーヤーの制御を追加する方法、ボットを作成する方法について説明しました。

これは、コースに生徒数よりも教師数が多い場合に当てはまります。私たちは5人しかいないので、コース全体のタスクとして、5人はネットワークの相互作用、物理学、およびいくつかのボットを使用して小さなゲームを作成します。最終的に何が得られるか見てみましょう:) 

C ++


私は本当にこの特定のコースを聞きたかったです。学部では、毎年C ++コースを受講しましたが、これは2014年でした。それ以来、言語の標準と機能はすでに変更されています。私はC ++で開発しており、この言語をある程度知っていますが、私の知識は徐々に古くなっており、更新したいと考えています。 

このコースはC ++コースの2番目の部分であり、私たちとコンピュータサイエンスセンターの学生のために一緒に行われます。それは1学期続き、Valera Lesinが読み、特定の分野に精通している講師を招待しました。 20番目の標準、つまりコース開始の数か月前に登場したあらゆる種類のクールなチップについて話されています。たとえば、モジュール、概念、コルーチンについてです。その後、実際には各トピックの問題を解決し、言語の新しい概念を試す機会があります。 

ゲーム理論


このコースは、私たちの学部の2つの修士号を取得した1年生（約30人）を対象に行われました。多くの教師がいて、彼らはサンクトペテルブルクHSE のゲーム理論と意思決定の国際研究所の従業員でした。コースはロシア語と英語で同時に教えられました：一部の外国人教師はロシア語を話さず、英語で講義しました、ロシア語を話せる人もいましたが、彼らは英語で書きました。

先生はいつも変わっていたので、とてもクールで面白かった講義もあり、中には眠ってしまった講義もありました。そして、私は断固として十分な練習をしませんでした。簡単なテストが1つと宿題が2つありましたが、テストではこれら3つのタスクのうち2つに合格すれば十分であることがわかりました。そのため、2つ目の宿題を真ん中に投げて完了しませんでした。彼女は緊急ではないオプションのタスクの列に移動しました。

私の意見では、あまりにも過酷なコースです。

IT管​​理


IT管理上のコースは、一つのモジュールを継続し、イリヤアンドレーエPshenichnyによって読み取られた：で、事業開発のディレクターを演技Playkot。このコースには、私たちのプログラムと「情報システムと人間とコンピュータの相互作用」プログラムの2つの学部生が参加しました。   

講義は主に講義であり、IT業界特有のさまざまな管理面について少しお話を伺いました。たとえば、開発へのアプローチ、開発の段階、使用できる段階、リスクの管理方法、リーダーの主な機能は何か、チームの動機付けの方法、従業員の採用と解雇の方法、そしてコースの最後には自己啓発に関するいくつかの推奨事項がありました。

講義はとても簡単で面白かったです。通常、管理の各領域のストーリーは、個人的な経験の例と、何が正しく機能し、どのような場合にうまく機能するかを説明することによって、積極的にサポートされました。 

このコースの実践的なタスクは1つだけでした。私たちは申請書を受け取り、開発計画と予算を立てる必要がありました。作業は2週間で完了する必要があり、それがチェックされ、フィードバックを聞いたり何かを修正したりしたいすべての人と一緒に、教師が直接電話して話しました。その後、さらに1週間が修正のために与えられました。 

講義では、チームとの連携方法や、従業員のやる気に影響を与えるものについて個別に取り上げました。たとえば、リーダーの重要なタスクの1つは、部下に有能なフィードバックを与えることであると言われました。 

この点で、イリヤアンドレイエヴィッチが私の最終的な割り当てについてフィードバックを与えた方法についてお話ししたいと思います。最初は、タスクの10点中7点を得ました（5点満点で、これは4です）。一般的に、これはまあまあですが、災害ではありません。修正すべきかどうか疑問に思いました。他にもあります。通話中に、Ilya Andreevichは当然のことながら作業の何が悪いのか教えてくれました。従業員の給与は市場ではなく、税金は考慮されておらず、アプリケーションのテストは通常​​計画のどこかで失われました。最後に、彼は間違いなく私がこの仕事を訂正することを期待していると付け加えました。そんな自信はどこから来たのだろうと思いました。彼は答えた：「あなたは義務的であり、勤勉で、完璧を目指しているからです。」そして、私はこれらの特性を満たしたいので、間違いなくそれを修正しました！ 

これは、人が仕事を修正するモチベーションが高まるような方法で否定的なフィードバックを与える方法の非常に美しい例であると私には思えました。そして、はい、このタスクでは合計10です。    

並行性の理論と実践


このコースは、Roman Lipovskyが教えているYandex Data Analysis Schoolでリモートで受講しています。このコースは、競争力のあるシステムを専門としています。これは、メモリモデルのしくみ、コルーチン、ファイバーのしくみ、さまざまな同期プリミティブのデバイス、および非同期システムの操作について説明します。 

コースは遠隔地にあり、多くの生徒が参加するため、教師との個別のコミュニケーションがうまくいきません。しかし、実践と宿題は非常に考え抜かれています。 

宿題はDockerコンテナー内のコンピューターで自動的にチェックされます。変更をClionからDockerにフラッシュし、簡単なコマンドでテストを開始できます。同時に、テストは非常にうまく行われ、これは競争力のあるプログラミングのコースにとって非常に困難です。私が正しく理解していれば、標準のプリミティブの代わりに、作業の正しさをチェックできる教師からの独自の実装を使用しているというのが事実です。たとえば、特定の方法でフローを切り替えたり、偽のウェイクアップを実行したりします。

宿題もとても上手です。さまざまな難易度の一連のタスクが定期的に表示されます。さらに、通常はインフラストラクチャ全体がすでに実装されており、それに時間を費やす必要はありません。課題が与えられた本質を直接理解することだけが必要です。C ++で非同期に動作する方法を理解し、コルーチンを実装し、プリミティブを実装するタスクがありました。 

MAGUEGO


専門分野のコースに加えて、経営学で学ぶコース中に、選択した方向に関係のないモジュールのサイズの小さなコースを1つ取る必要があります。サンクトペテルブルクHSEでは、そのようなコースはコード名MAGOLEGOの下に隠れています。私たちは、経済学、法律、心理学、経営など、さまざまな分野に関連する約20の分野の選択肢を与えられました。代わりに、最初に同意した場合は、当社が選択したオンラインコースを受講できます。 

私はフルタイムのコースであるPsychology of Professional Successを受講しましたが、私の選択にはとても満足しています。私たちは、紛争が起こりそうなこと、フィードバックに建設的に対応する方法、専門家の機能障害とは何か、そして感情的な燃え尽き症候群で何ができるかについて話されました。さらに、スピーチとライティングのスタイルによって、人について何かを理解する方法、およびインタビュー中に注意を払うことができる点についても話しました。 

このコースの興味深い考えに別の記事を割くことができるように思えます。このコースはElena Igorevna Kudryavtsevaによって読まれました。聞くのはとても簡単で面白かったです。良い物語の構造、興味深い場所のイントネーションに明確に重点を置いたしっかりとした声と、意味を説明する多くの例。 

レポートタスクとして、独自のポートフォリオを作成する必要がありました。ポートフォリオは履歴書に似ていますが、それらの違いは、各プロジェクト/作業場所/実績にストーリーのある個別のページがあることです。自分のことを話す方がはるかに簡単であり、面接担当者が何ができるかを理解しやすいので、就職の面接時にそのような父親がいると非常に便利だという意見があります。 

コースを修了した後も、ポートフォリオの形でアーティファクトがまだ残っているのが好きでした。これは、将来役立つかもしれません。新しい経験を積んだら補足するつもりです。
