SIM7600E-Hモジュールの一部としてのOpenLinux





カスタムアプリケーションを開発してモジュールにロードするメカニズムは、LinuxおよびWindowsオペレーティングシステムの両方で使用できます。この記事では、SIMCom Wireless Solutionsが提供するSDKの例を使用して、ユーザーアプリケーションをコンパイルし、モジュールにロードする方法を詳しく説明します。



この記事を書く前に、私の友人は、Linuxの開発からは程遠いのですが、SIM7600E-Hモジュール用の自分のアプリケーションを開発するプロセスを説明する質問にできるだけ遠ざけるように頼みました。資料の提出可能性を評価する基準は、「理解できるように」という語句でした。



何が起こったのかを理解してください。

記事は定期的に更新され、更新されます。




プレリュード



通常、セルラー通信モジュールは、データ送信、音声通話、SMSなどにのみ使用されます。これはすべて、外部制御マイクロコントローラーから送信されるATコマンドによって行われます。ただし、外部からロードされたカスタムコードを実行できるモジュールのカテゴリがあります。場合によっては、これによりデバイスの全体的な予算が大幅に削減され、ボードをより単純な（および予算の）マイクロコントローラーにしたり、完全に破棄したりできます。 AndroidまたはLinuxによって制御されるLTEモジュールとその強力なリソースの出現により、一般的なプロセッサーで利用可能な問題を解決できます。この記事では、Linuxが管理するSIM7600E-Hについて説明します。実行可能なアプリケーションをダウンロードして実行する方法を見ていきます。



多くの点で、この資料は「SIM7600 Open Linux開発クイズ」というドキュメントに基づいていますが、いくつかの追加、特にロシア語版は役に立ちます。この記事は、モジュールの学習を始めたばかりの人がデモアプリケーションをダウンロードできるようにし、今後の作業に必要なスキルを提供するのに役立ちます。



SIM7600E-Hの概要



SIM7600E-Hは、クアルコムのARM Cortex-A7 1.3GHzプロセッサをベースにしたモジュールで、内部にLinuxオペレーティングシステム（カーネル3.18.20）があり、ヨーロッパ（ロシアを含む）周波数範囲2G / 3G / Cat.4対応のLTE。最大ダウンロード速度は最大150Mbps、アップロード速度は最大50Mbpsです。豊富な周辺機器、工業用温度範囲、および統合されたGPS / GLONASSナビゲーションの存在は、M2Mテーマの最新のモジュラーソリューションの要件をすべてカバーします。



システムの概要



SIM7600E-Hモジュールは、Linuxオペレーティングシステム（カーネル3.18.20）に基づいています。次に、ファイルシステムは、ソートされていないブロックイメージファイルシステム（UBIFS）ジャーナルファイルシステムに基づいています。



このファイルシステムの重要な機能は次のとおりです。



  • パーティションで動作し、パーティションを作成、削除、またはサイズ変更できます。
  • 媒体のボリューム全体に記録調整を提供します。
  • 不良ブロックで動作します。
  • 緊急停電またはその他の障害時のデータ損失の可能性を最小限に抑えます。
  • ロギング。


説明はここから取られ、同じ場所で、そのようなファイルシステムについてより詳細に説明されています。



それら。このタイプのファイルシステムは、モジュールの過酷な動作条件や起こり得る電源の問題に最適です。しかし、これは、不安定な電力の状態がモジュールの予想される動作モードになることを意味するものではなく、これはデバイスの実行可能性の向上を示すだけです。

SDKの解凍とコンパイル



シェルにアクセスし、モジュールのコマンドラインで作業を開始できるようになったら、最初のアプリケーションをコンパイルしてモジュールにロードします。



多くの場合、これで問題が発生する可能性があります。なぜなら Windowsでコードをコンパイルするときの衝突を避けるために、モジュールはLinuxオペレーティングシステムで実行されます。ネイティブ環境でコンパイルするのが最適です-Linux。



Linuxがない場合や、Linuxをマシンにインストールしたい場合は、仮想マシンにインストールする方法については説明しません。VirtualBoxを使用し、Ubuntuバージョン20.04（執筆時点での現在のバージョン）をインストールします。その下で、コンパイラ、SDKなどの作業を開始します。



Linux環境に切り替えて、ディストリビューターから受け取ったアーカイブを解凍します。 



simcom@VirtualBox:~/Desktop/OpenLinux$ sudo tar -xzf MDM9x07_OL_2U_22_V1.12_191227.tar.gz


sim_open_sdkディレクトリに移動して、環境を追加します。 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ cd sim_open_sdk
simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ source sim_crosscompile/sim-crosscompile-env-init


同じフォルダーにとどまり、その中にある後続のコマンドを実行します。

libncurses5-devライブラリがインストールされていない場合は、インストールします。 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get update && sudo apt-get install libncurses5-dev -y


Python、それもインストールされていない場合： 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get install python -y


およびgcc： 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get install gcc

次に、いくつかのファイルをコンパイルし、次のコマンドを順番に実行する必要があります。

simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make

simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make aboot

simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel_menuconfig
simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel

simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make rootfs

simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel_module

simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make demo
