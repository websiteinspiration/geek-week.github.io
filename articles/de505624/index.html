Wie verarbeite ich einen Datenrahmen mit Milliarden von Datensätzen in Sekunden?

Die Big Data-Analyse von Python befindet sich in einer Renaissance-Ära. Es begann mit der NumPy-Bibliothek. Diese Bibliothek ist übrigens eine der Komponenten der Werkzeuge, die in diesem Material behandelt werden. Im Jahr 2006 gewann das Thema Big-Data-Verarbeitung allmählich an Bedeutung. Dieser Prozess beschleunigte sich mit dem Aufkommen von Hadoop. Dann kam die Pandas-Bibliothek mit ihren DataFrame-Datenstrukturen, die normalerweise einfach als „Datenrahmen“ bezeichnet werden. Im Jahr 2014 wurde Big Data zum Mainstream, im selben Jahr erschien die Apache Spark-Plattform. Im Jahr 2018 wurden die Dask-Bibliothek und andere Tools zur Analyse von Daten in Python veröffentlicht.



Jeden Monat stoße ich auf neue Tools für die Datenanalyse in Python, die ich unbedingt lernen möchte. Wenn Sie ein oder zwei Stunden damit verbringen, sie zu studieren, können Sie auf lange Sicht viel Zeit sparen. Darüber hinaus ist es wichtig, die neuen Entwicklungen im Bereich der Technologie zu überwachen, die Sie interessieren. Vielleicht denken Sie, dieser Artikel handelt von der Dask- Bibliothek . Aber das ist nicht so. Heute werde ich Ihnen von einer kürzlich entdeckten Python-Bibliothek erzählen, die für diejenigen, die sich mit Datenanalyse beschäftigen, wissenswert ist.

Vaex Python-Bibliothek



Vaex ist eine leistungsstarke Python-Bibliothek, die zur „verzögerten“ Verarbeitung von Datenrahmen (ähnlich wie Pandas-Datenrahmen) entwickelt wurde und zur Visualisierung und Recherche großer Mengen tabellarischer Daten dient. Diese Bibliothek kann die wichtigsten statistischen Indikatoren für die untersuchten Daten berechnen. Gleichzeitig kann die Verarbeitung von einer Milliarde Datensätzen ungefähr eine Sekunde dauern. Die Bibliothek unterstützt viele Visualisierungstools, mit denen Big Data interaktiv untersucht werden kann.

Vaex und Dask vergleichen



Die Vaex-Bibliothek ähnelt nicht Dask, aber die darin verwendeten Datenstrukturen ähneln den in Dask verwendeten DataFrame-Entitäten. Sie basieren auf Pandas-Datenrahmen. Dies bedeutet, dass Dask von Pandas bestimmte Einschränkungen beim Arbeiten mit Datenrahmen erbt . Beispielsweise muss eine Anforderung, dass zu verarbeitende Daten vollständig in den Speicher geladen werden müssen. Im Fall von Vaex ist dies nicht der Fall.

Vaex erstellt keine Kopien von Datenrahmen. Mit dieser Bibliothek können Sie daher große Datenrahmen auf Computern mit wenig RAM verarbeiten.

Sowohl Vaex als auch Dask verwenden Lazy Data Processing Engines. Der Hauptunterschied zwischen ihnen besteht darin, dass Vaex die Feldwerte berechnet, wenn sie benötigt werden, und in Dask müssen Sie die Wertberechnungsfunktion explizit verwenden.

Um das volle Potenzial von Vaex auszuschöpfen, müssen die Daten im HDF5- oder Apache Arrow-Format dargestellt werden.

Installieren Sie Vaex


Die Installation von Vaex ist so einfach wie bei jedem anderen Python-Paket:

pip install vaex

Experimente mit Vaex


Erstellen Sie einen Pandas-Datenrahmen, der 1 Million Zeilen und 1000 Spalten enthält. Dadurch können wir eine große Datendatei erhalten.

import vaex
import pandas as pd
import numpy as np
n_rows = 1000000
n_cols = 1000
df = pd.DataFrame(np.random.randint(0, 100, size=(n_rows, n_cols)), columns=['col%d' % i for i in range(n_cols)])
df.head()

Informationen zur Speichernutzung Speichern

Sie diesen Datenrahmen auf der Festplatte, um ihn später mit Vaex zu lesen.

file_path = 'big_file.csv'
df.to_csv(file_path, index=False)

Wenn Sie diese CSV-Datei direkt mit Vaex lesen, werden wir nicht viel davon profitieren. Die Lesegeschwindigkeit entspricht der von Pandas bereitgestellten Geschwindigkeit. Auf meinem Laptop lesen sowohl Vaex als auch Pandas diese Daten in etwa 85 Sekunden.

Um die Funktionen von Vaex zu sehen, müssen CSV-Daten in das HDF5-Format (Hierarchical Data Format Version 5) konvertiert werden. Vaex verfügt über eine Funktion zum Durchführen einer solchen Konvertierung, mit der Sie mit Daten arbeiten können, deren Volumen die Menge des verfügbaren Arbeitsspeichers überschreitet. Dies wird dadurch erreicht, dass die Quelldaten in Fragmente unterteilt sind.

Wenn es aufgrund von Speichermangel nicht möglich ist, eine ausreichend große Datei mit Pandas zu öffnen, kann diese Datei in das HDF5-Format konvertiert und mit Vaex verarbeitet werden.

dv = vaex.from_csv(file_path, convert=True, chunk_size=5_000_000)

Die obige Funktion erstellt automatisch eine HDF5-Datei und speichert sie auf der Festplatte.

Überprüfen Sie den Variablentyp dv:

type(dv)
# 
vaex.hdf5.dataset.Hdf5MemoryMapped

Lesen wir nun den 7,5-GB-Datensatz mit Vaex. Wir müssen es jetzt nicht lesen - Tatsache ist, dass es bereits durch eine Variable dargestellt wird dv. Wir führen jedoch immer noch den folgenden Befehl aus, um herauszufinden, wie schnell diese Operation ist:

dv = vaex.open('big_file.csv.hdf5')

Vaex benötigt weniger als eine Sekunde, um diesen Befehl auszuführen. Tatsächlich liest Vaex diese Datei jedoch nicht, da Mechanismen der "faulen" Datenverarbeitung verwendet werden.

Lassen Sie Vaex nun die Datei lesen, indem Sie die Summe der Werte in der Spalte berechnen col1:

suma = dv.col1.sum()
suma
# array(49486599)

Aber dieses Team hat mich wirklich überrascht. Die Berechnung des Vaex-Betrags dauerte weniger als eine Sekunde. Wie ist das möglich? Und vielleicht liegt das an der Speicherzuordnung.

Datenvisualisierung


Vaex zeigt auch eine gute Geschwindigkeit bei der Datenvisualisierung. Die Bibliothek hat eine spezielle Funktion : plot1d, plot2dund plot2d_contour.

dv.plot1d(dv.col2, figsize=(14, 7))

Virtuelle Spalten


Vaex erstellt beim Hinzufügen einer neuen Spalte zum Dataset eine virtuelle Spalte. Er nimmt keine Erinnerung. Die darin gespeicherten Werte werden wie gesagt "on the fly" berechnet.

dv['col1_plus_col2'] = dv.col1 + dv.col2
dv['col1_plus_col2']



Neue Spalte

Effektive Datenfilterung


Vaex erstellt beim Filtern von Daten keine Kopien von Datenrahmen. Dies trägt zu einer effizienteren Speichernutzung bei.

dvv = dv[dv.col1 > 90]

Datenaggregation


Vaex aggregiert Daten etwas anders als Pandas. Das Wichtigste dabei ist jedoch, dass diese Operation in Vaex sehr schnell ist.

Erstellen Sie eine neue Spalte mit den Vergleichsergebnissen dv.col1 >= 50:

dv['col1_50'] = dv.col1 >= 50

Vaex kombiniert die Operationen zum Gruppieren und Aggregieren von Daten in einem Team. Der folgende Befehl gruppiert die Daten nach Spalten col1_50und berechnet die Summe der Spalten col3:

dv_group = dv.groupby(dv['col1_50'], agg=vaex.agg.sum(dv['col3']))
dv_group

Datenverbindungen


Mit Vaex können Sie Daten kombinieren, ohne Datenkopien im Speicher zu erstellen. Dies trägt wie andere Funktionen der Bibliothek zur wirtschaftlichen Nutzung des Speichers bei. Die joinunten gezeigte Funktion wird Pandas-Benutzern bekannt vorkommen:

dv_join = dv.join(dv_group, on=’col1_50')
