React


Reactコンポーネントは単なるJavaScriptオブジェクトです。Reactは独自の仮想DOMを作成します。これは、ユーザーインターフェイス構造全体のツリービューです。Reactは仮想DOMツリーをメモリに格納します。仮想DOMの内容がブラウザーウィンドウに物理的に表示される前に、Reactは多くの操作を実行して、仮想DOMからアイテムを追加、更新、および削除できます。ユーザーインターフェイス要素のレンダリングに関連

しないものには、コンポーネントメソッドrender()を使用しないでください。コンポーネントの状態やプロパティを変更する必要がある場合は、Reactコンポーネントのライフサイクルの標準的な方法を使用してください。

render（）メソッドは常に純粋な関数のままである必要があります。


このメソッドrender()は、仮想コンポーネントDOMを更新します。新しい仮想DOMツリーが以前に表示されたツリーと異なる場合、Reactは仮想DOMの更新に加えて、実際のブラウザーDOMも更新します。開発者はブラウザのDOMを直接更新する必要はありません。このルールは、Reactアプリケーションのコード内のすべての場所に適用されます。関数に適用する場合は特に重要render()です。


何らかの方法でDOMを直接更新する関数呼び出しでrender（）メソッドを汚染しないでください。

このメソッドrender()はコンポーネントの状態を（それを使用していてもsetState）変更したり、HTTPリクエストを行ったりしてはなりません。このメソッドからjQueryにアクセスしないでください。特定のデータをロードするリクエストを実行しないでください。実際には、メソッドrender()は純粋な関数になる状態に維持する必要があります。このメソッドは常に、コンポーネントメカニズムの最終段階で呼び出されます。その実装の過程で、ユーザーインターフェイスを更新するだけで済みます。仮想DOMへのすべての更新がすでに完了していると想定されています。
