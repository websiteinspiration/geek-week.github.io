ソースコードのトップコメント

こんにちは、ハブロフスク。ここにはまだあまり書いていませんが、いくつかの形式の記事を試してみました。ここでは、私が働いている会社のITスペシャリストとのインタビュー、翻訳、ガイドなどを紹介しました。そして、今日、私は自分のために新しいフォーマットを試すことにしました。



自由な時間には、Quoraに座って時々好きです-私は他の参加者からの質問に答えたり、興味深い質問への回答を読んだりします。先日、かなり人気のある質問を見つけました。スレッドには、1,000を超えるサブスクライバーと100以上の応答があります。そして問題は、あなたがソースコードで見た中で最高のコメントは何ですか？（元。これまでに遭遇したソースコードの中で最も良いコメントは何ですか？）

私は、stackoverflowとredditの両方にそのようなスレッドがあることを知っていますが、Quoraからの回答に刺激を受け、そこから選択することにしました。私は答えのほとんどを調べ、最も多く来たカップルを選びました。

サーシャ・クラソフスキーによる返信


DigiPenで見つけたいくつかのランダムな学生向けゲームのソースコードを見ていたら、突然これを目にしました。

/* Do NOT delete this comment */

当然、コメントを削除するとどうなるかを確認する必要がありました。削除して再コンパイルしてみました。動作しませんでした。コメントを返すと、まるで魔法のようにコンパイルされました。

エラーLNK1000がありました： "不明なエラー。テクニカルサポートオプションについては、ドキュメントを参照してください。」なぜこのコメントが必要だったのかは永遠に謎のままです。

Somnath Mishraからの応答から


むかし、Stackoverflowで同様のスレッドを見て、出会った面白いコメントのいくつかを共有しました。はい、私は面白いコメントをたくさん書いていますが、明白な理由でここではそれらを共有しません。

このコメントは楽しいだけでなく素晴らしいです。一部の人々は、プログラマーとして働いているときに文学的夢をあきらめます：

1. // Replaces with spaces the braces in cases where braces
2. // in places cause stasis                                                           
3. $str = str_replace(array("\{","\}")," ",$str);

（レーン。うっ滞を引き起こす場所で、ブラケット間のスペースを置き換えます）
それは英語で印象的に聞こえます-2行の解説、そしてそれらには6つの韻を踏む単語があります。

少し黒いユーモア：

1. double penetration; // ouch

コードは人を無神論者にすることができます：

1. //When I wrote this, only God and I understood what I was doing    
2. //Now, God only knows                                                  
3. // Update: I am an atheist now!

古いコードをクリーンアップするとき、プログラマーは結果の可能性を恐れて、使用されないが削除されない部分に遭遇することがよくあります。

1. // I am not sure if we need this, but too scared to delete.

イライラするプログラマ：

1. // I am not responsible of this code.
2. // They made me write it, against my will.

（レーン。私はこのコードの責任を負いません。彼らは私の意志に反してそれを書くように強制しました）

怒っているプログラマ：

1. // no comments for you
2. // it was hard to write
3. // so it should be hard to read
