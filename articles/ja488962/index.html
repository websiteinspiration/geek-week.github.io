C ++でスクリプトを作成できますか？


最近、私はLinuxスクリプトプログラミングの素晴らしい世界に再び突入しなければなりませんでした。基本的にはそれほど難しいことではありませんが、そのような仕事に出くわすことはあまりないので、毎回勉強します。明日はすっかり忘れてしまい、1か月後には、これをどうやってやるのか、もう一度ググるでしょう。問題は依然として、多くの場合、スクリプトを再度作成しないが、誰かがすでに作成した既存のスクリプトを変更することが判明しました。そして、それはbashではないかもしれませんが、shまたは何か他のもの...構文に違いがあります、shで機能するものはbashで機能するはずですが、常に逆ではありません。そして、ダッシュや灰がある場合はどうなりますか？わかりません...これらのスクリプト言語にはまだ違いがあり、混乱しています。そしてもちろん、個人的には、スクリプトがsedまたはawkを呼び出すときのケーキのチェリーであり、コマンドラインにそのようなパラメーターがあるので、それらを見て不思議に思います。これはすべてプログラマーの資格に依存することは明らかですが、すべてが私の頭に収まるわけではありません。そして今、私の忍耐力がなくなったので、これからはc ++でスクリプトを記述してみたいと思いました...

真のシステム管理者にとって、私の考えは慎重に思えるかもしれません。しかし、なぜですか？

したがって、アイデアは非常に単純です。通常のスクリプトと同じ方法でC ++スクリプトを記述したいと思います。つまり、スクリプトの最初の行にシバンと「インタープリター」へのパスの表示を含める必要があります。

#!/bin/c++

スクリプトの後続の行は、通常のc ++プログラムになります。

C ++スクリプトの「インタプリタ」を用意する必要があります。少なくともbashでは正確に書けます（正確ではありませんが、今回が最後です）。もちろん、それはインタプリタではなくコンパイラです。

それが私がしたことです：

#!/bin/bash

msg_file=/dev/null
#msg_file=/dev/stdout

tmp_path=$HOME"/.cache/c++/"
mkdir -p $tmp_path
tmp_file=$1".c++"
exe_file=$1".bin"
if test $1 -nt  $tmp_path$exe_file; then
    echo "Need to recompile.." > $msg_file
    tail -n +2 $1 > $tmp_path$tmp_file
    eval "g++ -o $tmp_path$exe_file $tmp_path$tmp_file > /dev/null 2>&1"
    if [ $? -eq 0 ]
    then
	echo "Compiled ok" > $msg_file
    else
	echo "Compile error" > $msg_file
	exit 255
    fi
fi
eval "$tmp_path$exe_file $@1"

このスクリプトは、必要なすべてを行います。一時フォルダーとして、〜/ .cache / c ++フォルダーを選択しました。元のスクリプトはこのフォルダーにコピーされますが、シバンのある最初の行はありません。これは、tailコマンドによって行われます。新しいファイルの名前は元のスクリプトのようになりますが、拡張子はc ++になります。拡張子が.binのバイナリーは、同じフォルダーに収集されます。ただし、最初に、もちろん、バイナリの作成時に「ifテスト」チェックが行われます。コンパイルは、既存のバイナリが元の「スクリプト」よりも古い場合にのみ発生します。バイナリはevalコマンドで起動され、すべての初期パラメータがそれに渡されます。

このc ++ファイルを/ binフォルダーにコピーして実行可能にする必要があります（chmod a + x）。

最初の「c ++スクリプト」を書いてみます。

#!/bin/c++

#include <stdio.h>
#include <iostream>

using namespace std;

int main( int argc, char *argv[] )
{
    cout << "hello world!\n";
    for( int i=0; i<argc; i++)
    {
	cout << "Param" << i <<  " is " << argv[i] << "\n";
    }
    return 60+argc;
}

このプログラムは、単に私が入力パラメータのリストを出力し、その数+ 60を返し
、私の「スクリプト」を実行します。



それは動作します!!!

C ++コードでエラーが発生した場合、プログラムはコンパイルされないため起動しませんが、$？をエコーし​​ます。 255を返します。しかし、それは意図されたものでした。

C ++を使用すると、途方もない機会が提供されます。まず、おなじみの構文。次に、std :: vector、std :: map、std :: stringなどの標準クラスは、かけがえのないものです。同じ行-何をしたいのか、行を見て、部分文字列に分割し、切断して征服し、配列を取得します。 sedやawkは必要ありません。第三に、デバッガー-神様！なんという祝福でしょう！スクリプト用のgdbデバッガーがあります。さらに、std ::ファイルシステムを使用できます（コンパイラーが許可する場合）。あなたは続けることができます...
