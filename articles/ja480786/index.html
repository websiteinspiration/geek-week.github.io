この場合、数値の配列を宣言してコンソールに表示するだけです。これを新しい配列を返す関数に渡しますが、2番目の引数に値を追加して、新しい配列の最後に追加します。

function insertValToArr(arr, val) {
  const newArr = arr;
  newArr.push(val); 
  return newArr;
}

コンソールの結論からわかるように、新しいアレイが変更されただけでなく、古いアレイも変更されました。これは、関数insertValToArrで1つの配列を別の配列に割り当てたconst newArr = arrため、既存の配列へのリンクを作成し、新しい配列を変更しようとしたときに、古い配列のメモリ領域を参照し、大まかに言えばそれを変更したために発生しました。また、両方の配列が同じメモリ領域を参照するため、それらの値は同じになります。古い配列を変更できないように関数を変更しましょう。

const arr = [0, 1, 2, 3, 4, 5];
const newArr =  insertValToArr(arr, 15);
console.log("New array: ", newArr); // output: "New array: " [0, 1, 2, 3, 4, 5, 15]
console.log("Old array: ", arr); // output: "Old array: " [0, 1, 2, 3, 4, 5]
function insertValToArr(arr, val) {
  const newArr = [];
  arr.forEach((value, ind) => { newArr[ind] = value});
  newArr.push(val);
  return newArr;
}

古い配列は変更されていません。これは、その各要素を受け取り、要素の値を新しい配列の要素に個別に割り当てたためです。これで、後者には別個のメモリ領域があり、それを変更しても、古い配列には影響しません。しかし、これらはすべて単純な例であり、実際のプログラムでは、おそらく1次元配列だけでなく、2次元、3次元ではなく、4次元である場合もあります。ほとんどの場合、それらは連想配列（ハッシュテーブル）の形式で見つかります。JavaScriptでは、これらはほとんどの場合オブジェクトです。

JavaScriptが提供するオブジェクトをコピーするための標準的な方法を見てみましょう-Object.assign（）は、1つ以上のソースオブジェクトからターゲットオブジェクトに、それ自体のすべての列挙型プロパティの値をコピーするために使用されます。コピー後、ターゲットオブジェクトを返します。考慮して下さい：

const obj = { a: 1 };
const newObj = Object.assign({}, obj);
console.log(newObj); // output: { a: 1, b: 2 }
console.log(obj); // output: { a: 1, b: 2 }
