Vuex - pecahkan perselisihan lama dengan metode baru

Vuex memiliki satu praktik populer - tidak menggunakan mapStatedan mapMutations. Umumnya. Sebagai gantinya, kami segera mendapatkan pengambil untuk itu ketika menciptakan nilai baru di toko, dan tindakan untuk setiap mutasi.


Sesuatu seperti:


export default new Vuex.Store({
  state: {
    // 1
    count: 0
  },
  getters: {
    // 1
    count: (state) => state.count
  },
  mutations: {
    // 2
    increment(state) {
      state.count++;
    }
  },
  actions: {
    // 2
    increment({ commit }) {
      commit('increment');
    }
  }
});
