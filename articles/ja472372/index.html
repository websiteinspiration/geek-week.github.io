Androidオートメーション 初めてのエスプレッソテストを作成するための超簡単ガイド

皆さん、こんにちは。モバイルQAエンジニアコースの開始を見越して、興味深い資料の翻訳を共有します。



エスプレッソとは何ですか？


いいえ、これはあなたが元気づけるために毎日飲む飲み物ではありません。Espressoは、Googleが開発したオープンソースのテストフレームワークです。実際のデバイスまたはエミュレーターで複雑なユーザーインターフェイステストを実行できます。Android向けの複雑なテストの作成には時間がかかりますか？

多分。しかし、最初の一歩を踏み出し、Espressoフレームワークを使用してAndroidの簡単なテストケースを作成する方法を今すぐ学ぶことはできます。

自動化について詳しく教えてください。


もちろん。自動化は、テストを高速化し、テストをより効率的かつスケーラブルにする方法です。手動テストは非常に重要ですが、自動テストは将来的にははるかに優れたオプションになります。

テストフレームワークには、基本的に2つのタイプがあります。

  1. ソースコードにアクセスする必要がなく、プロジェクトの一部として統合されていないフレームワーク。たとえば、WebDriver、Appium、QTPなどです。
  2. ソースコードにアクセスする必要があるフレームワーク。たとえば、Espresso、KIF（Keep It Functional）です。

Espressoはソースコードへのアクセスを必要とするフレームワークなので、自動化のためにプロジェクトコードへのアクセスが必要です。機能するプロジェクトが必要なので、作成しましょう！

エスプレッソの主要コンポーネント


Espressoで使用できるメソッドには3つのタイプがあります。

  1. ViewMatchers-ビューの現在の階層でオブジェクトを見つけることができます
  2. ViewAssertions-オブジェクトの状態を確認し、状態が基準を満たしていることを確認できます
  3. ViewActions-これらのメソッドを使用すると、オブジェクトを使用してさまざまなアクションを実行できます。

より深く掘り下げ、実際の例でどのように機能するかを見てみましょう。

簡単な自動化アプリケーションを作成する


最初に行う必要があるのは、自動化するアプリケーションを作成することです。 Android Studioでプロジェクトを作成しましょう。もちろん、これを行うには、Android Studioがコンピューターにインストールされている必要があります。

ご覧のとおり、そこには1つのテストしか記述されていません。これはUIテストではなく、JUnitテストです。

次に、簡単なユーザーインターフェイステストを追加する必要があります。これを行うには、まずMainActivityを開くルールを作成します。

JUnit Rule アノテーションのインポートを追加しましょう：

import org.junit.Rule;

次に行うことは、RunWithアノテーションに加えて、以下のコード行を追加することです。

@Rule
public ActivityTestRule<MainActivity> activityActivityTestRule = new ActivityTestRule<>(MainActivity.class);



これで、ActivityTestRuleが赤で強調表示されていることがわかります。つまり、ActivityTestRule関数をインポートする必要があります。しかし、最初に、これを実行できるライブラリを追加する必要があります。簡単に言えば、このタスクには、この目的のために特別に作成されたアセンブリ自動化システムであるGradleの助けが必要です。Gradle

構成ファイルに移動して、このライブラリを追加しましょう。build.gradle（モジュール：app）というファイルを開き、以下の行を追加します。

androidTestImplementation 'com.android.support.test:rules:1.0.2'


Android Studio。依存関係を追加します。

追加後、ボタンをクリックしてプロジェクトを同期し、テスト実装を含むファイルに戻る必要があります。

ライブラリーが追加されたので、次のステップは、ライブラリーを機器テストとともにファイルにインポートすることです。

import androidx.test.rule.ActivityTestRule;


Android Studio。ActivityTestRuleをインポートします。

これで、最初のテストを追加する準備ができました。ExampleInstrumentedTestに次のコードを入力します。

@Test
public void clickButtonHome(){
   onView(withId(R.id.navigation_home)).perform(click()).check(matches(isDisplayed()));
}


Android Studio。テストの追加と追加要素のインポート

テストでは、動作を開始する前に追加要素をインポートする必要があります。[ OK]をクリックすると、実際にテストを開始する準備が整いました。
