1日で学生をウダレンカに編入


早くも3月16日、私たちは快適なオープンスペースに座っていました。翌日、オフィス全体が遠隔地の自宅に移動しました。十分に似たような話を持っています。しかし、私たちの状況の詳細は、従業員の異動ではなく、顧客の異動です。これらはフルタイムで勉強するようになった三千人以上の学生ですが、一方では通信になってしまいました。

最初と2番目のサポートの従業員に多大な労力がかかりました。突然、数千の個人ユーザーデバイスがサポートに表示されました。必要なソフトウェアのインストール、特定のソフトウェアのライセンスの発行を支援する必要があり、突然、クリミア（ズームをそこに残した）、統合されたVPNヘルプを備えたOperaではZoomが機能しなくなりました。

メインプラットフォームとしてズームが選択されました。

当初、問題はクラスの計画、会議の作成、すべての生徒と教師への通知で生じました。


これは、ズームの講義スケジュールが最初の数日間のようになったものです。すべての会議が手動で作成された


そして、それが1秒後に再び始まったとき

、私の同僚のおかげで、彼はすぐにZoom APIをスモークし、スケジュールに従って会議の自動作成を台無しにしました。また、それに関する自動情報。

そして、ここにズームAPIを使用して会議を作成する方法に関するセルゲイの作品へのリンクがあります。

次に、Androidアプリケーションでのプラットフォーム統合の経験を共有したいと思います。
ユーザーケースでは、これは非常に便利であることがわかりました。会議はサーバー上で作成され、それらの識別子は電話のスケジュールに転送されます。


Zoom

Now と統合されたスケジュールインターフェースがどのように機能するかを見てみましょう。
ソースコード

1. SDKから2つのライブラリーcommonlibとmobilertcをインポートします。これはおそらく最も悲しい段階です。あなたのアプリケーションは80メガバイト重いです。

2.アクティビティまたはフラグメントにInitAuthSDKCallback、MeetingServiceListenerを実装します。ズームを機能させるには、最小21のAPIが必要です（minSdkVersion 16はサイトのドキュメントに示されていますが、すぐには更新されません。SDKからのサンプルの例に集中することをお勧めします）。

フラグメントを作成するときに、SDKを初期化します

Zoom SDKを初期化する
private View view;
private ZoomSDK mZoomSDK;
@Override
public View onCreateView(
            LayoutInflater inflater, ViewGroup container,
            Bundle savedInstanceState) {
        view = inflater.inflate(R.layout.fragment_first, container, false);

        mZoomSDK = ZoomSDK.getInstance();
        if (mZoomSDK.isLoggedIn()) {
            getActivity().onBackPressed();
            return null;
        }

        InitAuthSDKHelper.getInstance().initSDK(getContext(), this);
        if (mZoomSDK.isInitialized()) {

            ZoomSDK.getInstance().getMeetingService().addListener(this);
            ZoomSDK.getInstance().getMeetingSettingsHelper().enable720p(true);
        }

        return view;
}


これを行うには、InitAuthSDKHelperクラスを作成し、関数を呼び出します。

initSDK
public void initSDK(Context context, InitAuthSDKCallback callback) {
        if (!mZoomSDK.isInitialized()) {
            mInitAuthSDKCallback = callback;

            ZoomSDKInitParams initParams = new ZoomSDKInitParams();
//            initParams.jwtToken = SDK_JWTTOKEN;
            initParams.appKey = SDK_KEY;
            initParams.appSecret = SDK_SECRET;
            initParams.enableLog = true;
            initParams.logSize = 50;
            initParams.domain = WEB_DOMAIN;
            initParams.videoRawDataMemoryMode = ZoomSDKRawDataMemoryMode.ZoomSDKRawDataMemoryModeStack;
            mZoomSDK.initialize(context, this, initParams);
        }
}


ここでは、次のパラメーターに注意します

。appKeyおよびappSecret-これは、開発者として登録された、ズームサイトで作成するアプリケーションの識別子と秘密鍵です（約1分かかります）。

実際には、それらを識別に使用することは強くお勧めしません。
代わりに、ここに記述されているように、jwtTokenパラメータを使用する必要があります。
