A.1。C言語の基本


Cプログラムは、一連のプロシージャ（値を返さない場合でも関数と呼ばれることが多い）で構成されています。これらのプロシージャには、宣言、演算子、およびコンピュータに何をすべきかを一緒に伝える他の要素が含まれています。図A-1は、3つの整数変数が宣言され、値が割り当てられる小さな手順を示しています。プロシージャの名前がメインです。プロシージャ名の後ろの大括弧の間に識別子がないことで示されるように、プロシージャには仮パラメータがありません。プロシージャの本体は中括弧（{}）で囲まれています。この例は、Cに変数があり、これらの変数は使用前に宣言する必要があることを示しています。 Cにも演算子があります。この例では、これらは代入演算子です。すべてのステートメントはセミコロンで終了する必要があります（Pascalとは異なり、演算子の後にではなく、演算子の間にコロンを使用しています）。

コメントは文字「/ *」で始まり、文字「* /」で終わり、複数行にまたがることができます。

{

     j  =  i + 015; 	/*  j   i + 015 ( ) */
     k = j * j + 0xFF;   /*  k  j * j + 0xFF ( ) */

A.2。基本的なデータ型


Cには、主に整数と文字の2つのタイプのデータ（変数）があり、それぞれintとcharとして宣言されています。個別のブール変数はありません。 int変数はブール変数として使用されます。この変数に0が含まれている場合、それはfalse / falseを意味し、他の値はtrue / trueを意味します。 Cにも浮動小数点型がありますが、MINIXはそれらを使用しません。

値の範囲（コンパイラに依存する範囲）を定義するint型に、短い、長い、または符号なしの「形容詞」を適用できます。ほとんどの8088プロセッサは、intおよびshort intには16ビット整数を使用し、long intには32ビット整数を使用します。 8088プロセッサの符号なし整数（unsigned int）の範囲は0〜65535であり、通常の整数（int）の場合のように-32768〜+32767ではありません。文字は8ビットです。

レジスター指定子は、intとcharの両方に対しても許可され、宣言された変数をレジスターに配置してプログラムをより高速に実行する必要があることをコンパイラーに通知します。

A.3。複合型とポインタ


このセクションでは、より複雑なデータ型を構築する4つの方法（配列、構造体、共用体、ポインター）について説明します。配列は、同じタイプの要素のコレクション/セットです。Cのすべての配列は要素0で始まります。

宣言

int a [10];

[0]から[9]までの配列の要素に格納される10個の整数を持つ配列aを宣言します。次に、配列は3次元以上にすることができますが、MINIXでは使用されません。
構造は、通常はさまざまなタイプの変数のコレクションです。Cの構造は、Pascalのレコードに似ています。オペレーター

struct {int i; char c;} s;

は、整数iと文字cの2つのメンバーを含む構造体としてsを宣言します。

構造体sのメンバーiを6に割り当てるには、次の式を記述します。

s.i = 6;

ここで、ドット演算子は、要素iが構造sに属していることを示しています。
ユニオンも、構造に似たメンバーのセットです。ただし、ユニオンに所属できるのは常に1つだけです。発表

union {int i; char c;} u;

つまり、整数または文字を使用できますが、両方を使用することはできません。コンパイラーは、結合の最大の要素（占有メモリーの観点から）に対応できるように、結合に十分なスペースを割り当てる必要があります。ユニオンは、MINIXの2つの場所でのみ使用されます（メッセージをいくつかの異なる構造の結合として定義し、ディスクブロックをデータブロック、iノードブロック、カタログブロックなどの結合として定義します）。

ポインターは、Cでマシンアドレスを格納するために使用されます。それらは非常に頻繁に使用されます。アスタリスク（*）は、広告内のポインターを示すために使用されます。発表

int i, *pi, a [10], *b[10], **ppi;

A.6。プログラムの構造


Cプログラムは、プロシージャと宣言を含む1つ以上のファイルで構成されます。
これらのファイルを個別にオブジェクトファイルにコンパイルし、（リンカーを使用して）相互にリンクして実行可能プログラムを形成できます。
Pascalとは異なり、プロシージャ宣言はネストできないため、すべてプログラムファイルの「トップレベル」に記述されます。

プロシージャの最初の宣言の前のファイルの先頭など、プロシージャの外部で変数を宣言することができます。これらの変数はグローバルであり、staticキーワードが宣言の前にない限り、プログラム全体の任意のプロシージャで使用できます。この場合、これらの変数は別のファイルでは使用できません。手続きにも同じ規則が適用されます。プロシージャ内で宣言された変数は、プロシージャに対してローカルです。
プロシージャは、別のファイルで宣言された整数変数vにアクセスでき（変数が静的でない場合）、それ自体が外部であると宣言します。

extern int v;

各グローバル変数は、メモリを割り当てるために、extern属性なしで1回だけ宣言する必要があります。

変数は宣言時に初期化できます。

int size = 100;

配列と構造体も初期化できます。明示的に初期化されていないグローバル変数は、デフォルト値のゼロを受け取ります。

A.8。イディオム


このセクションでは、Cに典型的であるが他のプログラミング言語では一般的ではないいくつかの構成体を見ていきます。まず、ループについて考えます。

while (n--) *p++  =  *q++;

変数pとqは通常文字ポインターであり、nはカウンターです。ループは、n文字の文字列をqが指す場所からpが指す場所にコピーします。ループの各反復で、カウンターは0に達するまで減少し、各ポインターは増加するため、より高い数値のメモリセルを順に指します。

別の一般的なデザイン：

for (i = 0; i < N; i++) a[i] = 0;

これは、aの最初のN個の要素を0に設定します。このループを作成する別の方法は次のとおりです。

for (p = &a[0]; p < &a[N]; p++) *p = 0;

このステートメントでは、整数ポインタpは、配列のゼロ要素を指すように初期化されています。ループは、pが配列のN番目の要素のアドレスに到達するまで続きます。ポインタ構成は配列構成よりもはるかに効率的であるため、通常は使用されます。

割り当て演算子が予期しない場所に表示される場合があります。例えば、

if (a = f (x))  <  >;

最初に関数fを呼び出し、次に関数aの呼び出し結果を割り当て、
最後にそれが真（ゼロ以外）または偽（ゼロ）であるかどうかを確認します。aがゼロに等しくない場合、条件は満たされます。オペレーター
