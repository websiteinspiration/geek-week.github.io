Persiapan data untuk penyetelan dan pelatihan jaringan saraf untuk deteksi objek.



Kami melanjutkan topik deteksi objek . Agar algoritma dapat membedakan antara objek atau untuk dapat mendeteksi penyimpangan dalam proses, itu harus dilatih pada kumpulan data yang sesuai dengan gambar keseluruhan bahan video yang dianalisis. Mempersiapkan perangkat pelatihan adalah proses yang sangat memakan waktu, yang tergantung pada kecepatan pelatihan dan kualitas algoritme. Pertimbangkan penggunaan beberapa alat yang akan membantu mempersiapkan data dengan benar.



Tahap persiapan pertama adalah kompilasi dari set data awal. Sebagai aturan, semakin besar volume data dan semakin tinggi keragaman objek yang dianalisis, semakin tinggi kualitas algoritma. Jika Anda bermaksud untuk menganalisis video dari DVR, maka menyusun dataset lebih baik untuk mengambil screenshot dari materi video yang ada, karena dalam hal ini, lebih sedikit data dan lebih sedikit waktu untuk pelatihan ulang akan diperlukan. Dalam kasus lain, kumpulan data dikompilasi dengan mempertimbangkan sudut pandang dari mana objek ditangkap, ukurannya, dll., Berapa banyak objek dari set pelatihan mirip dengan objek dari bahan yang akan dianalisis oleh algoritma. Kumpulan data harus terdiri dari berbagai variasi, mis. di mana satu objek dan beberapa akan ditampilkan, misalnya, seseorang dan telepon dalam satu bingkai.



Jika kompilasi set data dilakukan dengan menggunakan kamera atau telepon, kemungkinan besar gambar akan memiliki resolusi tinggi, yang secara negatif mempengaruhi proses pembelajaran. Masalah utama dengan gambar resolusi tinggi adalah kesalahan "Kehabisan memori", bahkan jika parameter batch_size kecil. Untuk mengurangi kemungkinan kesalahan ini, perlu untuk mengurangi ukuran gambar. Program IrfanView akan membantu kami dalam hal ini. Program ini memungkinkan Anda untuk mengelompokkan gambar, mis. Anda hanya perlu menentukan jalur ke gambar sumber, mengatur parameter konversi dan memulai proses konversi. Hasilnya, kami mendapatkan gambar "lite" di folder yang ditentukan. Untuk melakukan ini, di jendela awal program, tekan tombol B (tata letak Rusia "I") atau di tab File, pilih "Konversi / Ubah Nama Batch",sebuah jendela muncul sebagai berikut:

Ada banyak transformasi berbeda yang tersedia di opsi lanjutan, dalam contoh ini kami hanya mengubah ukuran gambar asli, mengurangi ukuran dengan 10 kali (sebutkan 10% dari nilai asli):







Jadi, jika gambar asli adalah 4160 x 3120, gambar yang dihasilkan akan menjadi 416 x 312 piksel.



Pembentukan sampel pelatihan adalah proses yang agak melelahkan, apakah itu adalah pembentukan tangkapan layar dari bahan video atau pengambilan gambar dengan kamera dan telepon. Untuk meningkatkan volume set data secara signifikan, berbagai metode augmentasi digunakan. Mari kita lihat beberapa metode ini diimplementasikan dalam bahasa pemrograman Python menggunakan perpustakaan Pillow. Salah satu metode augmentasi yang paling populer adalah "refleksi cermin".



Untuk memulai, kami mengimpor perpustakaan yang diperlukan: 



from PIL import Image, ImageOps
import os
import numpy


Kami menunjukkan lokasi direktori dengan gambar sumber dan jalur untuk hasil konversi: 



path_to_origin = "images/" 
output_folder = "Augmentation_result/"
img_names = os.listdir(path_to_origin)

if not os.path.exists(output_folder):
    os.makedirs(output_folder)


Kami akan menulis fungsi untuk mengonversi gambar menjadi "gambar cermin": 



def mirror(path_to_origin, img_name, output_folder):
    im = Image.open(path_to_origin + img_name)
    ImageOps.mirror(im).save(output_folder+"mirror"+img_name, "JPEG")


Metode augmentasi populer lainnya adalah rotasi gambar. Perlu dipertimbangkan dengan objek berukuran penuh bahwa dengan metode augmentasi ini, bagian dari gambar hilang. Kami menulis fungsi untuk memutar gambar: 



def rotate(path_to_origin, img_name, output_folder):
    im = Image.open(path_to_origin + img_name)
    im.rotate(-30).save(output_folder+"rotate_"+img_name, "JPEG")


Nilai (-30) adalah sudut rotasi gambar, yang dipilih berdasarkan jenis objek. Misalnya, rotasi 180 derajat, jika ada telepon di gambar, maka ini adalah pilihan yang baik untuk augmentasi, tetapi jika ada orang di gambar, maka 180 derajat untuk gambar ini adalah contoh yang buruk, karena Anda tidak mungkin melihat video di mana orang berjalan terbalik.



Dan akhirnya, kita akan menghabiskan 2 jenis augmentasi untuk setiap gambar: 



for img in img_names:
    rotate(path_to_origin, img, output_folder)
    mirror(path_to_origin, img, output_folder)

Untuk markup kita akan menggunakan program LabelImage, karena menurut saya itu cukup mudah digunakan dan multifungsi. Program ini dapat mengembalikan hasil markup dalam dua format (PascalVOC dan YOLO). Format pertama disajikan dalam bentuk file XML di mana parameter gambar ditunjukkan, serta koordinat persegi panjang yang menggambarkan lokasi objek dalam gambar:







File XML tersebut dihasilkan untuk setiap gambar di mana semua objek yang ditandai dijelaskan. Nama file markup sama dengan nama gambar. Format YOLO disajikan dalam bentuk file teks di mana kelas objek dan koordinat dari persegi panjang dijelaskan, dijelaskan oleh nilai x, y, w, h dalam bentuk normal:







Selain file teks dengan markup untuk format YOLO, file classes.txt juga dihasilkan, di mana nama-nama objek ditunjukkan. Nomor urut nama-nama benda tergantung pada urutan penampilan saat menandai gambar. Perlu dicatat bahwa file classes.txt dapat ditimpa ketika aplikasi dibuka kembali, oleh karena itu, perlu untuk memantau urutan memasukkan nama dalam program, jika tidak maka akan terjadi ketidakkonsistenan koordinat. Markup objek terjadi dalam urutan berikut:







  1. Pilih direktori dengan gambar;
  2. Pilih direktori tempat file markup akan disimpan;
  3. Pilih format tata letak (PascalVOC atau YOLO);
  4. Pilih alat objek "Kunci W";
  5. Pilih objek dalam gambar;
  6. Atur nama objek (selama operasi berulang, Anda dapat memilih dari daftar, di bawah tombol OK);
  7. Konfirmasi
  8. Simpan "Ctrl + s" (ulangi langkah 4-8 untuk setiap objek yang diperlukan);
  9. Pergi ke gambar berikutnya.


Setelah selesai, jumlah file markup harus sesuai dengan jumlah gambar dengan nama yang identik. Dengan demikian, kami memeriksa alat dan metode yang memungkinkan Anda untuk mengubah ukuran gambar, menambah jumlah data, serta menandai objek pada gambar yang akan terdeteksi oleh jaringan saraf. Pembentukan dan persiapan data merupakan langkah penting dan memakan waktu di mana kesuksesan dalam menciptakan algoritma pembelajaran yang mendalam tergantung.
