VCF変換


以下は、VCFファイルを変換し、受信データを最近無料になったPrometheaseサービスにアップロードする手順です。サンプルについて受け取ったPrometheaseレポートについて理解することをお勧めします。タスク1で取得したバリエーションのリストでフィルタリングされたVCFファイルを使用します  。

#     
bcftools query -s HG00731 -f '[%SAMPLE]\t%ID\t%CHROM\t%POS\t%REF\t%ALT\t[%GT]\n' -e '%ID=="."' CEI.1kg.2019.demo.subset.vcf.gz | create_23andme.sh > HG00731.subset.23andme.txt
#      HG00731.subset.23andme.txt

このコマンドをbcftools query使用すると、ユーザーがフラグの後に指定した形式で、VCFファイルから使用可能な情報を抽出できます-f。フラグ-sは、HG00731データを抽出するサンプル（）の識別子を示します。 -eフラグは、除外基準を示すために使用されます。この場合’%ID=="."’、rsIDを持たないエントリは除外されます。からの出力  bcftools queryはcreate_23andme.sh、データを4列（rsID、染色体、位置、遺伝子型）のTSV形式に変換し、それらをファイルに書き込むスクリプトへの入力に転送されます。create_23andme.sh独自のゲノム全体のシーケンスデータを操作するためのスクリプトをダウンロードして保存できます。

脚本create_23andme.shVCFファイルから抽出されたフィールドを使用して、遺伝的変異のタイプ（SNVの単一ヌクレオチド変異、INSの挿入またはDELの削除）を決定stdoutし、特定の種類の変異（A、G、TおよびCが有効な対立遺伝子）に従って識別子rsID、染色体、位置および対立遺伝子を  記録しますタイプSNV、IおよびDの場合-タイプINSおよびDELの許容対立遺伝子指定）。

変換プロセスには長い時間がかかることに注意してください。1つのサンプルでファイルあたり約4時間、約100万のバリエーションがあります。並行性BCFtoolsはサポートされていません。

移動し  promethease.comと登録します。 [生データのアップロード]ボタン（図2）をクリックして、ファイルをアップロードしますHG00731.subset.23andme.txt。ダウンロードが完了したら、[無料のレポートを作成]ボタンをクリックして、データに基づいて生成されるレポートの名前を入力します。レポートを作成すると、電子メールで通知が届き、レポートの内容を理解できます。各サンプルのレポートで、Promethease解釈システムがAB0 / Rhシステム（Rh-Rh係数）で決定した血液型を見つけます。表1に準拠して、あなたの結果を確認してください

このマニュアルで使用されているファイルには常にALTフィールド（代替対立遺伝子）が含まれているため、各バリエーションがどのタイプ（INS、DEL、SNV）に属するかを理解し、23andMe形式でエントリを正しく作成できます。 Atlasのゲノム全体のシーケンスデータには、この対立遺伝子が見つかった場所にのみ、満たされたALT対立遺伝子が含まれます。それ以外の場合、VCFファイルの生成時にALTフィールドを埋めるための情報は存在しません。ホモ接合型参照サイト（参照対立遺伝子が見つからなかったゲノム内の位置）でのデータ出力は、ヌクレオチドシーケンスで検出された変化だけでなく、それらの欠如もあるため、必要です。

ゲノムのそのような位置にALT対立遺伝子が存在しないため、参照（REF）対立遺伝子のみが見つかった遺伝子変異のタイプを特定できません。このような場合の遺伝子型の記録は、この変異の可能性のある対立遺伝子に関する情報源を使用する必要があるため複雑であり、このガイドでは扱いません。このマニュアルとスクリプトcreate_23andme.shを使用して、ゲノム全体のシーケンス後に取得したVCFファイルをAtlasに変換する可能性がある場合、スクリプトはcreate_23andme.shそのようなレコードを明示的にフィルターして挿入と削除のレコードを作成するときにエラーを除外するため、変換されたファイルには参照ホモ接合遺伝子型が含まれません。

スクリプトを作成するにはcreate_23andme.sh 依然として参照ホモ接合遺伝子型が生成されている場合は、その25〜28行目の内容を置き換える必要があります。

...
  if [ "$ALT" == "." ] || [[ "$ALT" == *"*"* ]]
  then
    continue
  fi
...

に

...
  if [[ "$ALT" == *"*"* ]]
  then
    continue
  fi
  
  if [ "$ALT" == "." ]
  then
    echo -e "$RSID\t$CHR\t$POS\t$REF$REF"
  fi
...

この置換によりstdout、ホモ接合型参照遺伝子型のレコードを表示できます  。挿入と削除に使用される形式の有効な対立遺伝子はIとDであり、スクリプトはA、G、T、またはCの対立遺伝子を使用するため、挿入と削除のそのようなエントリは正しくないことに注意してください。挿入と削除を行うには、ALT対立遺伝子が検出されなかったゲノムの特定の位置にどのような種類の変異があるかを事前に知っておく必要があります。この情報は、ALT対立遺伝子がある場合（すでにで実装されている場合）を分析するcreate_23andme.shか、外部データベース、たとえばdbSNP（にはないcreate_23andme.sh）を使用して  取得できます  。

Atlasで全ゲノムシーケンスの完全なVCFファイルのPrometheaseレポートを取得するには、VCFファイル自体をPrometheaseにロードできますが、圧縮されたAtlas VCFファイルのサイズは約8ギガバイトですが、Prometheaseでは次のファイルしかアップロードできません。 4ギガバイト。この問題の解決策の説明は、ここにあり  ます。別の解決策は、VCFファイルをいくつかの部分（それぞれ4ギガバイト未満）に分割し、それぞれをPrometheaseデータダウンロードメニューの追加ファイルとしてロードすることです。


コンテストの3番目のタスク


最初のタスクのバリエーションのリストに従ってフィルタリングしたテストデータセットの12サンプルのそれぞれの変換データをPrometheaseにダウンロードし、サンプル識別子-Promethease解釈システムによって定義された血液型AB0 / Rhの対応表（Rh-Rh因子）をコンパイルします。確率論的に特定され、Prometheaseレポートで接頭辞「prob」が付けられた血液型は、接頭辞なしで記述します。未定義の値を不明として記録します（定義されている場合は、不明な血液型のアカゲザル係数を書き込む必要があります）。例を表1に示します。

VCFを上記の提案された実装で使用されている形式に変換することは非常に簡単ですが、かなりの時間が必要です。最適化のために、このデータを自動的に生成し、一連の識別子を反復するループを含むスクリプトを作成できます。このようなスクリプトをいくつか作成し、それぞれが異なるサンプル識別子のセットを並列実行するために転送することは可能ですが、並列実行スクリプトの数は、コンピューター/仮想マシンのCPUの数を超えてはなりません。このようなループを作成する方法については、こちらをご覧ください。 Yandex.Cloudで作業する場合、必要に応じて、多数の仮想CPUを備えた別の仮想マシンを作成できます。これにより、タスクの完了にかかる時間が比例して削減されます。
