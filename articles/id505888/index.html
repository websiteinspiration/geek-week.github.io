Parsing YouTube, termasuk data yang diunggah, tanpa API YouTube

pengantar


Untuk memuat data konten di YouTube, mereka biasanya menggunakan Selenium atau YouTube API. Namun, ada kerugian di mana-mana.

  1. Selenium terlalu lambat untuk diurai. Bayangkan parsing daftar putar ~ 1000 klip dengan selenium.
  2. YouTube API, tentu saja, adalah pilihan terbaik jika Anda memiliki beberapa jenis aplikasi atau proyek Anda, tetapi di sana Anda perlu mendaftarkan aplikasi, dll. Dalam versi "percobaan" Anda harus terus-menerus masuk untuk menggunakan api, ada juga kuota yang berakhir cepat.
  3. Dalam metode kami, saya akan mengatakan struktur data yang sangat rumit yang dihasilkan oleh YouTube. Memilah pencarian YouTube sangat tidak stabil.

Bagaimana cara memuat data di YouTube?


Ada token untuk ini, yang dapat ditemukan dalam kode html halaman. Kemudian di masa mendatang kami akan menggunakannya sebagai parameter untuk permintaan ke YouTube, yang memberi kami konten baru. YouTube sendiri mengunduh konten menggunakan permintaan tempat token ini digunakan.

Ada parameter keluar tambahan yang akan kita butuhkan di langkah berikutnya.



Mendapatkan token melalui skrip


Kami menyusun parameter tajuk untuk permintaan ke YouTube. Selain agen-pengguna, kami memasukkan dua agen tambahan, yang Anda lihat di bawah.

headers = {
    'User-Agent': '      ,      ',
    'x-youtube-client-name': '1',
    'x-youtube-client-version': '2.20200529.02.01'
}

Kami membuat permintaan menggunakan perpustakaan permintaan. Letakkan tautan ke halaman yang ingin Anda muat, dan tambahkan juga header.

token_page = requests.get(url, headers=headers)

Token tidak dapat ditemukan oleh parser, karena disembunyikan di dalam tag tag. Untuk menyimpannya ke variabel, saya menulis kode yang jelek:

nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]

Biasanya ini adalah token 80 karakter.

Membuat permintaan untuk konten


service = 'https://www.youtube.com/browse_ajax'
params = {
"ctoken": nextDataToken,
"continuation": nextDataToken
}
r = requests.post(service, params=params, headers=headers)

Berbagai jenis data yang dimuat memiliki tautan layanan yang berbeda. Milik kami cocok untuk daftar putar dan video dari saluran.

YouTube mengirim data dalam format json. Oleh karena itu, kami menulis r.json (), tetapi daftar akan terbang kepada Anda, kami membutuhkan elemen kedua dari daftar, oleh karena itu r.json () [1]. Selanjutnya, kami sudah memiliki data. Masih mengurai.

Mengurai respons json


Anda dapat melihat rantai panjang kamus, tetapi kami akan mempersingkatnya sehingga lebih atau kurang jelas.

r_jsonResponse = r_json['response']
dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]

Di sini kita mendapatkan token baru untuk permintaan lebih lanjut. Jika data yang dimuat selesai, maka Anda tidak akan melihat token.
for content in dataContainer["contents"]:
	videoId = content['playlistVideoRenderer']['videoId']

Ini adalah bagaimana Anda dapat mengekstrak id video dengan menambahkan bagian template, Anda akan mendapatkan tautan ke video.

Untuk mendapatkan data berikut, Anda harus melakukan hal yang sama - meminta token, parsing dan sekali lagi.

Kode yang berfungsi sepenuhnya terlihat seperti ini:

import requests, json

url = input('url: ')
headers = {
		"User-Agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/81.0.4044.122 Safari/537.36",
		'x-youtube-client-name': '1',
		'x-youtube-client-version': '2.20200429.03.00',
		}
token_page = requests.get(url, headers=headers)
nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]
sleep = False #  ,    
ids = []
while not sleep:
	service = 'https://www.youtube.com/browse_ajax'
	params = {
	"ctoken": nextDataToken,
	"continuation": nextDataToken
	}
	r = requests.post(service, params=params, headers=headers)
	r_json = r.json()[1]
	r_jsonResponse = r_json['response']
	dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
	try: #  
		nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]
	except:
                #  . ,    .    
		sleep = True
	for content in dataContainer["contents"]:
		videoId = content['playlistVideoRenderer']['videoId']
		ids.append(videoId)
print(len(ids))
