結果：2019年の9つの主要な技術革新

Alexander Chistyakovが連絡しています。私はvdsina.ruの伝道者であり、2019年の9つの最高の技術イベントについて話します。

評価では、専門家の意見よりも自分の好みに頼っていました。したがって、このテクノロジーには根本的に新しく驚くべきものは何もないため、たとえば無人車両はこのリストには含まれていません。

リストのイベントを重要度またはすごい効果で並べ替えることはしませんでした。それらの重要性は10年で明らかになり、すごい効果はあまりにも短期間であるため、このストーリーを一貫性のあるものにしようとしました。

1. WebAssemblyでのRustプログラミング言語のポータブルサーバーアプリケーション


レビューは2つのレポートから始めます

。1. Brian Cantrillのレポート 「RustでOSを書き直す時間はありますか？」 2018年に彼が読み返しました。

レポートを読んでいた時点で、ブライアンカントリルはジョイエントのCTOに勤務しており、彼とジョイエントにとって2019年の結果はどうなるかわかりませんでした。

2. Rust言語のコアチームのメンバーであり、書籍「The Rustプログラミング言語」の著者であり、Cloudflareで作業しているSteve Klabnikによるレポート。Rust言語の機能と、アプリケーションを起動するためのプラットフォームとしてWebブラウザーを使用できるWebAssemblyテクノロジーについて語っています。

2019年、WebAssemblyとWASIインターフェイス、ファイルやソケットなどのオペレーティングシステムのオブジェクトへのアクセスを提供することで、ブラウザを超えてサーバーソフトウェア市場を目指しています。

ブレークスルーの本質は明らかです。人類には、Web用のポータブルアプリケーションを実行できるもう1つのランタイムがあります（Java言語の作者が発明したWORAの原則を覚えている人はいますか？）。

さらに、Rust言語のおかげで、これらのアプリケーションを作成する比較的安全な方法があります。その意味は、コンパイル段階でエラーのクラス全体を破棄することです。

WebAssemblyはゲームをひっくり返すので、Dockerの作成者の1人であるSolomon Hykesは、WebAssemblyとWASIが2008年に存在していたら、Dockerは生まれなかっただろうと書いています。



Rustが新しいポータブルテクノロジーの達人であったことは当然のことです。そのエコシステムは動的に発展しており、StackOverflowが実施した調査の結果によると、Rustは数年の間、最もお気に入りのプログラミング言語であり続けています。

これはスティーブのレポートのスライドです。これは、Rustを使用すると完全に回避できるセキュリティエラーの数と、過去10年半に検出されたMS Windowsのエラーの総数の比率を明確に示しています。



マイクロソフトはこのような課題に何らかの形で対応する必要があり、それに応えました。

2. MicrosoftのProjectVerona。Windowsを保存し、任意のOSの新しい履歴ページを開きます。


Microsoft Windowsカーネルお​​よびほとんどのユーザープログラムのエラー数は、過去12年間でほぼ直線的に増加しています。



2019年、マイクロソフトのマシューパーキンソンはプロジェクトヴェローナを一般に紹介しましたが、これはこれに終止符を打つ可能性があります。

これは、Rust言語のアイデアに基づいて安全なプログラミング言語を作成するというMicrosoftの取り組みです。MicrosoftResearchの同僚は、セキュリティの問題のほとんどは、ほとんどのWindowsが記述されているC言語の大きな遺産に関連していることを発見しました。ベロナの錆びたような言語は、ゼロコストの抽象化の原則を使用して、メモリとリソースへの同時アクセスを管理します。それがどのように機能するかを詳細に理解したい場合は、パーキンソン自身のレポートをチェックしてください。

興味深いことに、グラスゴーハスケルコンパイラの主な開発者であるSimon Peyton-Jonesが特にMicrosoftで働いているという事実にもかかわらず、Microsoftは伝統的に悪の帝国であり、すべての新しい敵であると見なされてきました。



最初のパラグラフからのブライアンカントリルの質問：「Rustのオペレーティングシステムのカーネルを書き直す時がきましたか？」は予期しない回答を受け取りました。オペレーティングシステムのカーネルがまだ書き換えられていないことは明らかですが、ユーザー空間で実行されているプログラムはすでに書き換えられています。止められないプロセスが始まりました。これにより、すべてのオペレーティングシステムの将来に向けて新しいページが開きます。

3. FlutterフレームワークによるDartプログラミング言語の人気の高まり


次のニュースは私たちと一般の人々だけでなく、その形成過程に直接参加しているほとんどの人にとっても大きな驚きだと思います。 8年前にGoogleに登場したDartプログラミング言語は、今年、人気が急速に高まっています。

私は、Githubのリポジトリを分析し、テーブルのデータを月に1回更新することで、プログラミング言語の人気を評価する方法を使用しています。年の初めにDartで人気のリポジトリが100しかなかった場合、今日はすでに313です

。Dartは、Erlang、PowerShell、R、Perl、Elixir、Haskell、Lua、CoffeeScriptの人気を上回っています。より速く、それは今年、他のプログラミング言語が成長していないようです。なぜそれが起こったのですか？

今年の画期的なレポートの1つ聴衆によると、HackerNewsはRichard Feldmanによって読まれ、「なぜ関数型プログラミングが標準ではないのですか？」話の多くは、プログラミング言語がどのように普及しているかを分析することです。 Richardによると、主な理由の1つは、人気のあるアプリケーションまたはフレームワーク、つまりキラーアプリの存在です。

Dart言語では、Flutterモバイルアプリケーション開発フレームワークが人気の理由になりました。Googleトレンドによると、Flatterの人気の上昇は今年の初めに過ぎませんでした。



私たちはモバイル開発に関与していないため、Dartについては何も知りませんが、静的型付けを備えた別のプログラミング言語を歓迎します。

4.仮想マシンeBPFのおかげで、Linuxカーネルとそのコミュニティが存続する可能性


私たちはVDSinaでのカンファレンスが大好きです。今年はサンクトペテルブルクで開催されたDevOopsカンファレンスに参加し、業界のトレンドや注目すべきことに関する円卓会議に参加しました。2019年、そのような会話は意見が支配的でした：

  • Dockerは退屈すぎるので死んでいます
  • Kubernetesは生きており、約1年間続きます-彼らは2020年の会議でそれについて話します
  • それまでの間、Linuxカーネルを長い間調べてきた人はいません。

Linuxカーネルの開発では、興味深い点だけでなく、革命的なことが起こっているという私の観点から、私は最後のポイントを共有しません。最も注目すべきは、eBPF仮想マシンです。これは、もともとネットワークパケットのフィルター処理という退屈なタスクを解決するために作成され、その後、汎用のカーネルレベルの仮想マシンに成長しました。

eBPFのおかげで、カーネルの外部で部分的に処理できるイベントの発生を報告します-インターフェイスにより、ユーザースペースからカーネルと安全かつ効率的に対話し、Linuxカーネルの機能を拡張および補完して、すべてを見るのを回避できます。 Linus Torvaldsの目。

eBPFが登場する以前は、Linuxカーネルとの対話に密接に関連するアクティビティを持つプログラムの開発は困難でした。ユーザー空間で非常に高速ではないデバイスのドライバーやファイルシステムのインターフェイスなどを作成するには、経験豊富なLinuxカーネル開発者による正式なレビュー手順を実行する必要がありました。

eBPFインターフェースの出現により、そのようなプログラムを作成するプロセスが大幅に簡略化されました。入力のしきい値が減少し、開発者が増え、コミュニティが再び活気づくようになります。

私は熱心なだけではありません。長年にわたるコア開発者のデビッドミラーは、コア開発エコシステムの存続（！）のためのeBPFの重要性を宣言しています。同じく有名な別の開発者であるブレンダングレッグ（私は彼の大ファンです）がeBPFを画期的なものと呼びます50歳ではないに等しい。

5. Linuxカーネルのio_uring非同期インターフェースのおかげで、LinuxはFreeBSD棺のほぼ最後の釘を打ちました


Linuxカーネルについて話しているので、今年のもう1つの重要な改善は注目に値します。Facebookの Jens Exbowによって書かれた新しい高性能非同期io_uring非同期I / O APIがカーネルに含まれています。

長年にわたり、FreeBSDのシステム管理者と開発者は、非同期I / OがLinuxよりもFreeBSDの方が優れていたという事実に基づいて選択を下していました。たとえば、NginxのGleb Smirnovは、2014年の彼のレポートでこの議論を使用しました。

今、ゲームは逆さまになっています。Ceph分散ファイルシステムはすでにio_uringに切り替えられていますパフォーマンステストの結果は印象的です。1秒あたりのI / O操作数の増加は、ブロックサイズに応じて14％から102％です。PostgreSQLで非同期I / Oを使用するプロトタイプがあり（少なくともバックグラウンドライター用）、PostgreSQLを非同期I / Oに変換するためのさらなる作業が計画されています。しかし、開発者コミュニティの保守主義を考えると、2020年にはこれらの変化は見られません。



6. RyzenプロセッサラインでのAMDの勝利


珍しいことではなく、AMDだけが長年の業界の共演者であり、記録を塗り替えています。

新しいRyzenプロセッサラインは、驚異的な価格/パフォーマンス比を示しました。それらは、Amazonのベストセラープロセッサのリストを独占しており、一部の地域では、AMDプロセッサの売上がIntelの売上を上回りました。競争では、インテルは非常に不人気な措置をとる必要があります。それは、独自のコンパイラーで作成されたプログラムを競合他社のプロセッサーで効率的に機能させないようにします。 Intelの汚い戦い方にもかかわらず、AMDの市場予測は2000年の最高記録に近づきました。

新しいiPadOSの発売に伴い、アップルは「破壊的イノベーション」と呼ばれるIntelに対する戦術を使用しました。破壊的イノベーションです。

ウィキペディアの「破壊的イノベーション」の定義は

、市場価値の関係を変えるイノベーションです。同時に、以前に行われた競争に基づいたパラメーターがその重要性を失うため、古い製品は競争力がなくなります。

「破壊的な革新」の例としては、電話（電信の代わり）、蒸気船（帆船の代わり）、半導体（真空装置の代わり）、デジタルカメラ（フィルムの代わり）、電子メール（従来の郵便物）があります。


Appleは独自のエネルギー効率の高いARMベースのプロセッサを使用しており、これはIntel x86のわずかなパフォーマンスよりもユーザーにとって重要であることが証明されています。

Appleはなんとか市場の一部を奪い取り、iPadをエンターテインメントターミナルから本格的な作業ツールに変えました-最初にコンテンツを作成する人のために、そして今は開発者のために。もちろん、近い将来、ARMベースのMacBookは見当たらないでしょうが、MacBook Proキーボードの設計に関する小さな問題は、代替ソリューションを見つけるのに役立ち、それらの1つはiPadOSを備えたiPad Proになると約束されています。

ゲイツとマイクロソフトはどうですか？

かつて、ゲイツはIBMとまったく同じトリックを行いました。

1970年代、IBMはサーバー市場を独占し、巨人の自信は素人向けのパソコンに注意を向けなかった。 1980年代に、ゲイツはお金のためにIBMを作成し、そのためにMS-DOSのライセンスを取得し、オペレーティングシステムの権利はそれ自体に任せていました。お金を受け取ったマイクロソフトは、MS-DOS用のグラフィカルインターフェイスを作成し、Windowsが誕生しました。最初はDOS用のグラフィカルアドオン、次にPC用の最初のオペレーティングシステムであり、大衆が使用するのに便利です。大規模な低迷企業であるIBMは、若くて速いMicrosoftにPC市場を奪っています。私はこの素晴らしい話を非常に簡潔に述べたので、2020年にiPadOSを使用してAppleがIntelと対戦する方法に興味がある場合は、全体を読むことを強くお勧めします。
