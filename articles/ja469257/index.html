PowerShellとWSLを使用してWindowsでLinuxコマンドを統合する

典型的なWindows開発者の質問：「なぜまだここにないのです< LINUX>か？」強力なスクロールlessツールでgrepあれsed、使い慣れたツールであれ、Windowsの開発者は、日常業務でこれらのコマンドに簡単にアクセスできることを望んでいます。

Linux用のWindowsサブシステム（WSL）は、この点で大きな前進を遂げました。これにより、WindowsからLinuxコマンドを呼び出して、それらをプロキシすることができますwsl.exe（たとえば、wsl ls）。これは大幅な改善ですが、このオプションには多くの欠点があります。

  • ユビキタスな追加はwsl疲れて不自然です。
  • 円記号はディレクトリセパレータではなくエスケープ文字として解釈されるため、引数内のWindowsパスは常に機能するとは限りません。
  • 引数のWindowsパスは、WSLの対応するマウントポイントに変換されません。
  • エイリアスと環境変数を含むWSLプロファイルのデフォルト設定は考慮されません。
  • Linuxパス補完はサポートされていません。
  • コマンド補完はサポートされていません。
  • 引数の補完はサポートされていません。

その結果、LinuxコマンドはWindowsではセカンドクラスの市民として認識され、ネイティブチームよりも使用が難しくなります。彼らの権利を平等にするには、これらの問題を解決する必要があります。

PowerShellシェル


PowerShell関数ラッパーを使用すると、wslWindowsのパスをWSLパスに変換することで、コマンド補完を追加し、プレフィックスの必要性をなくすことができます。シェルの基本要件：

  • 各Linuxコマンドには、同じ名前の関数のシェルが1つ必要です。
  • シェルは、引数として渡されたWindowsパスを認識し、それらをWSLパスに変換する必要があります。
  • シェルはwsl、適切なLinuxコマンドを使用して任意のパイプライン入力を呼び出し、関数に渡されるコマンドライン引数を渡す必要があります。

このテンプレートは任意のコマンドに適用できるため、これらのシェルの定義を抽象化し、インポートするコマンドのリストから動的に生成できます。

# The commands to import.
$commands = "awk", "emacs", "grep", "head", "less", "ls", "man", "sed", "seq", "ssh", "tail", "vim"
 
# Register a function for each command.
$commands | ForEach-Object { Invoke-Expression @"
Remove-Alias $_ -Force -ErrorAction Ignore
function global:$_() {
    for (`$i = 0; `$i -lt `$args.Count; `$i++) {
        # If a path is absolute with a qualifier (e.g. C:), run it through wslpath to map it to the appropriate mount point.
        if (Split-Path `$args[`$i] -IsAbsolute -ErrorAction Ignore) {
            `$args[`$i] = Format-WslArgument (wsl.exe wslpath (`$args[`$i] -replace "\\", "/"))
        # If a path is relative, the current working directory will be translated to an appropriate mount point, so just format it.
        } elseif (Test-Path `$args[`$i] -ErrorAction Ignore) {
            `$args[`$i] = Format-WslArgument (`$args[`$i] -replace "\\", "/")
        }
    }
 
    if (`$input.MoveNext()) {
        `$input.Reset()
        `$input | wsl.exe $_ (`$args -split ' ')
    } else {
        wsl.exe $_ (`$args -split ' ')
    }
}
"@
}

リスト$commandは、インポートするコマンドを定義します。次に、コマンドを使用して、それぞれの関数ラッパーを動的に生成しますInvoke-Expression（最初に、関数と競合するエイリアスを削除します）。

この関数は、コマンドライン引数をループし、Windowsのコマンドへのパスを定義Split-Pathし、Test-Pathその後、WSLの道へのパスに変換します。Format-WslArgument後で定義するヘルパー関数を介してパスを実行します。スペースや角かっこなどの特殊文字はエスケープされますが、そうでなければ誤解されます。

最後に、パイプラインwsl入力とコマンドライン引数を渡します。

これらのラッパーを使用すると、プレフィックスを追加せずに、より自然な方法でお気に入りのLinuxコマンドを呼び出すことができます。wsl パスがどのように変換されるかについて心配する必要はありません。

  • man bash
  • less -i $profile.CurrentUserAllHosts
  • ls -Al C:\Windows\ | less
  • grep -Ein error *.log
  • tail -f *.log

ここでは基本的なコマンドセットを示していますが、リストに追加するだけで、任意のLinuxコマンドのシェルを作成できます。このコードをPowerShell プロファイルに追加すると、ネイティブコマンドと同様に、これらのコマンドをすべてのPowerShellセッションで使用できます。

デフォルトのオプション


Linuxでは、頻繁に使用されるコマンド（alias ls=ls -AFhまたはなどexport LESS=-i）のデフォルトパラメータを設定して、プロファイル（ログインプロファイル）にエイリアスや環境変数を定義するのが慣例です。非対話型シェルを介したプロキシの欠点の1つは、wsl.exeプロファイルが読み込まれないため、これらのパラメーターがデフォルトで使用できないことです（つまり、lsWSL では、wsl ls上記で定義されたエイリアスでは動作が異なります）。

PowerShellは$ PSDefaultParameterValuesを提供します。これは、既定のパラメーターを定義するための標準的なメカニズムですが、コマンドレットと高度な関数に対してのみです。もちろん、あなたが私たちの殻から高度な機能を作ることができますが、この紹介不要な合併症（例えば、PowerShellは一部のパラメータ名を関連付ける（例えば、それは-aに関し、-ArgumentList）、これは部分的な名前を引数として受け入れるLinuxコマンドと競合します）、およびデフォルト値を定義するための構文は最適ではありません（デフォルトの引数を定義するには、コマンド名だけでなく、キーのパラメーター名が必要です）。

ただし、シェルを少し変更するだけで、同様のモデル$PSDefaultParameterValuesを実装してLinuxコマンドのデフォルトオプションを有効にすることができます。

function global:$_() {
    …
 
    `$defaultArgs = ((`$WslDefaultParameterValues.$_ -split ' '), "")[`$WslDefaultParameterValues.Disabled -eq `$true]
    if (`$input.MoveNext()) {
        `$input.Reset()
        `$input | wsl.exe $_ `$defaultArgs (`$args -split ' ')
    } else {
        wsl.exe $_ `$defaultArgs (`$args -split ' ')
    }
}

$WslDefaultParameterValuesコマンドラインに 渡して、を介してパラメーターを送信しますwsl.exe。以下は、PowerShellプロファイルに指示を追加してデフォルト設定を構成する方法を示しています。今、それを行うことができます！

$WslDefaultParameterValues["grep"] = "-E"
$WslDefaultParameterValues["less"] = "-i"
$WslDefaultParameterValues["ls"] = "-AFh --group-directories-first"

パラメータは後でモデル化されるため、キーを値に設定することで、一時的に簡単にオフに$PSDefaultParameterValuesすることができます。個別のハッシュテーブルの追加の利点は、から個別に切断できることです。"Disabled"$true$WslDefaultParameterValues$PSDefaultParameterValues

引数の補完


PowerShellでは、コマンドを使用して引数ターミネータを登録できますRegister-ArgumentCompleter。Bashには強力なプログラム可能な補完ツールがあります。WSLでは、PowerShellからbashを呼び出すことができます。PowerShell関数ラッパーの引数ターミネーターを登録し、bashを呼び出して終了を作成できる場合、bash自体と同じ精度で引数を完全に完成できます。

# Register an ArgumentCompleter that shims bash's programmable completion.
Register-ArgumentCompleter -CommandName $commands -ScriptBlock {
    param($wordToComplete, $commandAst, $cursorPosition)
 
    # Map the command to the appropriate bash completion function.
    $F = switch ($commandAst.CommandElements[0].Value) {
        {$_ -in "awk", "grep", "head", "less", "ls", "sed", "seq", "tail"} {
            "_longopt"
            break
        }
 
        "man" {
            "_man"
            break
        }
 
        "ssh" {
            "_ssh"
            break
        }
 
        Default {
            "_minimal"
            break
        }
    }
 
    # Populate bash programmable completion variables.
    $COMP_LINE = "`"$commandAst`""
    $COMP_WORDS = "('$($commandAst.CommandElements.Extent.Text -join "' '")')" -replace "''", "'"
    for ($i = 1; $i -lt $commandAst.CommandElements.Count; $i++) {
        $extent = $commandAst.CommandElements[$i].Extent
        if ($cursorPosition -lt $extent.EndColumnNumber) {
            # The cursor is in the middle of a word to complete.
            $previousWord = $commandAst.CommandElements[$i - 1].Extent.Text
            $COMP_CWORD = $i
            break
        } elseif ($cursorPosition -eq $extent.EndColumnNumber) {
            # The cursor is immediately after the current word.
            $previousWord = $extent.Text
            $COMP_CWORD = $i + 1
            break
        } elseif ($cursorPosition -lt $extent.StartColumnNumber) {
            # The cursor is within whitespace between the previous and current words.
            $previousWord = $commandAst.CommandElements[$i - 1].Extent.Text
            $COMP_CWORD = $i
            break
        } elseif ($i -eq $commandAst.CommandElements.Count - 1 -and $cursorPosition -gt $extent.EndColumnNumber) {
            # The cursor is within whitespace at the end of the line.
            $previousWord = $extent.Text
            $COMP_CWORD = $i + 1
            break
        }
    }
 
    # Repopulate bash programmable completion variables for scenarios like '/mnt/c/Program Files'/<TAB> where <TAB> should continue completing the quoted path.
    $currentExtent = $commandAst.CommandElements[$COMP_CWORD].Extent
    $previousExtent = $commandAst.CommandElements[$COMP_CWORD - 1].Extent
    if ($currentExtent.Text -like "/*" -and $currentExtent.StartColumnNumber -eq $previousExtent.EndColumnNumber) {
        $COMP_LINE = $COMP_LINE -replace "$($previousExtent.Text)$($currentExtent.Text)", $wordToComplete
        $COMP_WORDS = $COMP_WORDS -replace "$($previousExtent.Text) '$($currentExtent.Text)'", $wordToComplete
        $previousWord = $commandAst.CommandElements[$COMP_CWORD - 2].Extent.Text
        $COMP_CWORD -= 1
    }
 
    # Build the command to pass to WSL.
    $command = $commandAst.CommandElements[0].Value
    $bashCompletion = ". /usr/share/bash-completion/bash_completion 2> /dev/null"
    $commandCompletion = ". /usr/share/bash-completion/completions/$command 2> /dev/null"
    $COMPINPUT = "COMP_LINE=$COMP_LINE; COMP_WORDS=$COMP_WORDS; COMP_CWORD=$COMP_CWORD; COMP_POINT=$cursorPosition"
    $COMPGEN = "bind `"set completion-ignore-case on`" 2> /dev/null; $F `"$command`" `"$wordToComplete`" `"$previousWord`" 2> /dev/null"
    $COMPREPLY = "IFS=`$'\n'; echo `"`${COMPREPLY[*]}`""
    $commandLine = "$bashCompletion; $commandCompletion; $COMPINPUT; $COMPGEN; $COMPREPLY" -split ' '
 
    # Invoke bash completion and return CompletionResults.
    $previousCompletionText = ""
    (wsl.exe $commandLine) -split '\n' |
    Sort-Object -Unique -CaseSensitive |
    ForEach-Object {
        if ($wordToComplete -match "(.*=).*") {
            $completionText = Format-WslArgument ($Matches[1] + $_) $true
            $listItemText = $_
        } else {
            $completionText = Format-WslArgument $_ $true
            $listItemText = $completionText
        }
 
        if ($completionText -eq $previousCompletionText) {
            # Differentiate completions that differ only by case otherwise PowerShell will view them as duplicate.
            $listItemText += ' '
        }
 
        $previousCompletionText = $completionText
        [System.Management.Automation.CompletionResult]::new($completionText, $listItemText, 'ParameterName', $completionText)
    }
}
 
# Helper function to escape characters in arguments passed to WSL that would otherwise be misinterpreted.
function global:Format-WslArgument([string]$arg, [bool]$interactive) {
    if ($interactive -and $arg.Contains(" ")) {
        return "'$arg'"
    } else {
        return ($arg -replace " ", "\ ") -replace "([()|])", ('\$1', '`$1')[$interactive]
    }
}

コードはbashの内部の一部を理解していないと少しタイトですが、基本的には次のようにします。

  • 私たちは、リストを渡し、関数のすべての私たちのラッパーの引数のファイナライザを登録する$commandsパラメータに-CommandNameするためにRegister-ArgumentCompleter。
  • 私たちは、シェル関数への各コマンドをマップ自動補完のためのbashの用途（bashが仕様を定義するために自動補完の仕様を使用していること$F、のために短いがcomplete -F <FUNCTION>）。
  • PowerShellの引数を変換$wordToComplete、$commandAstおよび$cursorPosition形式で仕様に応じてのbash補完機能を期待プログラマブル自動のバッシュ。
  • に転送するためのコマンドラインを作成しwsl.exeます。これにより、正しい環境設定が提供され、適切なオートコンプリート機能が呼び出され、結果が改行とともに表示されます。
  • 次に、それをwslコマンドラインから呼び出し、出力を行区切り記号で区切り、それぞれを生成しCompletionResults、それらを並べ替えて、スペースやブラケットなどの文字をエスケープします。

その結果、Linuxコマンドシェルはbashとまったく同じオートコンプリートを使用します！例えば：

  • ssh -c <TAB> -J <TAB> -m <TAB> -O <TAB> -o <TAB> -Q <TAB> -w <TAB> -b <TAB>

各オートコンプリートは、既知のホストなどの構成データをWSLから読み取ることにより、前の引数に固有の値を提供します！

<TAB>パラメータを循環します。<Ctrl + >利用可能なすべてのオプションが表示されます。

また、bashオートコンプリートが機能するようになったため、PowerShellで直接Linuxパスをオートコンプリートできます。

  • less /etc/<TAB>
  • ls /usr/share/<TAB>
  • vim ~/.bash<TAB>

bashの完了で結果が得られない場合、PowerShellはWindowsパスを持つデフォルトのシステムに戻ります。したがって、実際には、これらの方法と他の方法を自由に同時に使用できます。

結論


PowerShellとWSLを使用すると、LinuxコマンドをネイティブアプリケーションとしてWindowsに統合できます。 Win32ビルドやLinuxユーティリティを探したり、Linuxシェルに切り替えてワークフローを中断したりする必要はありません。ただ、WSLをインストールし、構成するあなたのPowerShellプロファイル、およびインポートするコマンドのリストを！ LinuxおよびWindowsファイルのコマンドおよびパスパラメーターの豊富なオートコンプリートは、現在でもネイティブのWindowsコマンドにはない機能です。
