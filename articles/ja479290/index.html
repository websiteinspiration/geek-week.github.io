おそらく誰もがこのアルゴリズムを知っていますが、「当局は私から隠していました」。彼の言葉による説明は、英語フォーラムの自動翻訳のアーカイブの検索エンジンの3ページ目にありました。その詳細な説明（およびコードを含む）は、ハブロスタにふさわしいようです。

したがって、たとえば、立ち上がっていない人を取り除くために、おもちゃやプロセスのどこかにモブを生成する必要があります。これを行うには、Mobの重心を見つけ（これはそのボリュームを見つけることとほぼ同じです）、Mobの脚の上のどこかにあることを確認する必要があります。



モブは多面体です。簡単にするために、多面体は三角形のみで構成されていると見なします（内部のアルゴリズムにはガウス面積の公式があるため、任意の多面体に拡張できますが、なぜ...）。さらに、多面体はまともな多面体にふさわしいので、自己交差がなく、閉じたボリュームを制限するべきではありません。

アルゴリズムの本質は、xy平面に「落ちる」多面体の面を形成する図形のボリュームを考慮することです。これを行うには、三角形の投影領域と、フィギュアのボリュームを追加するための記号（切頭プリズム）を知っている必要があります。実際、三角形が事前に順序付けられている場合、体積と符号の両方が1つの計算に削減されます。

したがって、再帰関数が最初に三角形を収集するのは入力です。これは、多面体を「外側」で見たときに、三角形の周りを移動する方向が同じになるように収集されます（理想的には反時計回り。時計回りに方向を取ると、結果は正しくなりますが、負になるため、体積弾性率が戻り値に与えられます）。

これは非常に簡単に実現できます。三角形（ポイントa1、a2、a3）を取り、その近傍を探し、一致する2つの頂点を逆の順序でリストします（たとえば、a2、a1、b1など）。

ます。このような三角形をxy平面に投影すると、「上」の三角形の投影のトラバース順序は最初に選択したものと一致し、「下」の三角形の投影のトラバース順序は方向を変更します。その結果、ガウスの公式によって計算されたこの三角形の符号と面積が変更されます。ここで、「下の」三角形-条件付きの概念-は、そのすぐ下のボリュームが多面体のボリュームに含まれていないことを意味します。非凸多面体の「下」の三角形は、「上」の三角形よりも高くなることがあります。

これらの準備手順の後、多面体の総体積を計算するには、面から収集された切頭プリズムのすべての体積と、xy平面上のこれらの面の投影を追加する必要があります。そして、プリズムの体積は、面積（ガウス、符号あり）と三角形の頂点の算術平均z座標の積と見なされます。

多面体がxy平面と交差する場合、体積を計算するときにすべての符号が互いに相殺され、結果は正しいままです（モジュールなしでプリズムの高さを取得するだけで済みます）。

重心を検索すると、すべてがほぼ同じになります。同様に、各切り詰められたプリズムの質量の中心を見つけ、プリズムの体積を掛けて座標的に要約する必要があります（質量は体積全体に均一に分布し、1つを別のものに置き換えることができると想定されています）。切頭プリズムの重心を見つけるには、2つの四面体（+1関数）と1つの通常のプリズムの重心を計算する必要があります。多面体がxy平面と交差する場合も、アルゴリズムは「劣化しません」（また、マグリットの複製である場合もあります）。

def TetrV(mas):#dot1, dot2, dot3, dot4):
    """   """
    M = np.zeros((3,3),float)
    for i in range(1,4):
        for j in range(0,3):
            M[i-1][j] = mas[i][j] - mas[0][j]
    #print(M)
    return math.fabs(np.linalg.det(M)/6)
