メディアクエリのないflexboxでのアイテムの重要なレイアウト

2019年のCSS Flexboxについてはどのような投稿になるでしょうか？レイアウト設計者はこのテクノロジーを数年間積極的に使用しており、すべての秘密を解明する必要があります。

しかし、最近、1つの重要な、そしてflexboxに関連する便利なトリックを共有する必要があると強く感じました。4〜6行しかかからないものの、フレックスボックスに関連する問題を解決できるのは1人の知人（学生、タイプセッター、およびWebに近い人だけ）ではなかったという事実です。

したがって、最初に問題の説明、次に背景、最後に解決策です。

問題の定式化


  1. 大画面では、2つの要素が水平方向に配置されていますが、スマートフォンでは垂直方向に配置されています。
  2. 同時に、大きな画面ではそれらを端に押して（justify-content：space-betweenのように）、小さな画面では中央に配置する必要があります（justify-content：center）。

どうやら、何を決めるか！？ただし、もう1つ条件があります。メディアクエリなしで、自動モードでこの再構築を行う必要があります。

なぜ聞くの？メディアクエリの禁止によりこれらのトリックが行われる理由、フレックスボックスのためにフレックスボックスが選ばれる理由

そして重要なのは、コンテンツの幅を動的にすることができるということです。たとえば、ニュースの見出しと日付は互いに隣接しています。日付の形式は明確に定義されており、ヘッダーの長さは大きく異なります。要素が同じ線上に近づき始めた投稿のみに転送を移動してください。



そして、ここで結局のところ、再構築が行われる画面の幅はテキストの長さに依存するため、メディアクエリはまったく役に立ちません。通常、この場合、メディアリクエストは、最も長いタイトルに焦点を当てて、マージンをとって配置されます。短い見出しの場合、これはまあまあです。

別の印象的な例：ヘッダーのロゴとメニュー。ロゴの幅はわかっていますが、メニューに問題があります。実際、正式に完成したテンプレートにレイアウトすることがよくありますが、実際にはメニューはサイト管理パネルから形成されます。ポイント数-誰にもわからないため、再構築を実行する幅が明確ではありません。



実際のサイトでは、メニューがモバイルバージョンのボタンに変わることは明らかです。しかし、ポイント数がわからない場合、この変換はどのような幅で行う必要がありますか？もし私たちがこの変革に遅れたら、私はメニューがしばらくの間美しく飛んでいきたいと思っています。

そのような問題を解決するとき、人々はしばしば緊張しません。事前にメディアリクエストを行い、インパクトの点までかなりのマージンを確保するか、2つのうち1つの要素のみを中央に配置するコンテンツです。

方法1は単に不便であり、方法2は見出しと日付の例では多かれ少なかれ正常に見えますが、ロゴとメニューにとってはひどいものです。

, . flexbox media- : ? ( , - text-align flex-grow , )


数年前にflexboxに出会ってすぐに、この要素の動作を実現したかったのです。ただし、実験やマニュアルの読み取りでは結果は得られませんでした。その結果、ジャンプ中に1つの要素のみが中央に配置され、他の要素は端に留まることができると結論付けられました。

完璧な振る舞いを実現するために常に何かが欠けていました。フレックスグローは、ある画面では役立ち、別の画面では害を及ぼします。マージン：autoおよびtext-alignのヘルプが、結果を達成するのに十分ではありません。

その結果、ほとんどのプログラマーが考えるように、私はflexがその方法を知らないだけだと判断しました。そして、レイアウトで他の手段とテクニックを使って管理する方法を学んだので、このタスクを忘れてしまいました。

洞察は2019年の夏に来ました。レイアウトコースで宿題を整理しているときに、まったく関係のないことがわかりました。flex-grow：0.5。その例では、欲望の0.5は問題の解決にあまり役立ちませんでしたが、すぐにそれがここにあるとすぐに感じました-いつも欠けていた秘密の成分です。成長が分数になる可能性があることを知りませんでした。 0.5が完璧な配置を実現するのに役立つことは、私には思いもよらないことでした！flex-grow：0.5は空きスペースのちょうど半分のキャプチャであるため、これは明らかなようです

。一般的に、flex-grow：0.5を見ると、10分で要素を配置する問題を完全に解決しました。コードが非常にシンプルで簡潔であることを見て、私は長い間知られている自転車を発明したと判断しました。それにもかかわらず、数日前、私は彼がレイアウトデザイナーにどれだけ知られているかを確認し、45,000人のサブスクライバーがいるWeb開発用の対話型開発をYouTubeチャンネルで開始することにしました。

私は仕事を設定しました、私は待ち始めました。そして、どの加入者もそれを決定しませんでした。それを解決しないでください、そしておなじみのタイプセッター。フレックスを使用して要素を完璧に配置する方法を人々が単に知らなかったという感覚が高まりました。そして、この投稿は熟成しています。結局のところ、多くのタイプセッターは、問題のステートメントで説明されているように、メディアクエリなしで要素を配置することは単に不可能であると信じているように思えます。

以下で説明する解決策は非常に普遍的です。テキストを中央に配置したり、背景を正しく表示したりするために、追加のラッパーを作成する必要がある場合があります。しかし、多くの場合、構造を変更することなく、わずか6行のCSSコードで問題を解決することができます。

基本的な解決策


基本として、小さなHTMLを使用します。これは、最初のgif画像で見た例です。日付とタイトルは端まで押されており、ジャンプすると中央に配置されます。

<div class="blog">
  <div class="blogItem">
    <div class="blogItem__cat">Article posted in: PHP Programming</div>
    <div class="blogItem__dt">21.10.2019</div>
  </div>
  <div class="blogItem">
    <div class="blogItem__cat">Article posted in: Javascript</div>
    <div class="blogItem__dt">22.10.2019</div>
  </div>
  <div class="blogItem">
    <div class="blogItem__cat">Article posted in: wft my programm don't work</div>
    <div class="blogItem__dt">23.10.2019</div>
  </div>
</div>

背景、フレームなどに関連する面白くないスタイルは省略します。CSSの主要部分：

.blogItem {
  display: flex;
  flex-wrap: wrap;
}

.blogItem__cat {
  flex-grow: 0.5;
  margin-left: auto;
}

.blogItem__dt {
  flex-grow: 0.5;
  text-align: right;
}

このコードを初めて見ると、コードが人ではなく、ランダムなCSSプロパティの妄想ジェネレータによって作成されたように感じるかもしれません。したがって、それがどのように機能するかを理解する必要があります！

まず画像を注意深く検討し、その後でその下にあるテキストの説明に進むことをお勧めします。



最初は、フレックスアイテムは横に並んでおり、コンテンツに応じてサイズが決まります。flex-grow：0.5は、各要素に空きスペースの半分を与えます。彼らが近くに立っているとき、空きスペースはありません。margin-left：autoは要素が同じ行に配置されている間はまったく機能しないため、これはめちゃくちゃ重要です。

最初の要素はすぐに完璧になり、2番目のニーズの内容はtext-align：rightで右端に釘付けします。その結果、justify-contentをまったく記述せずに、justify-contentの類似物であるspace-betweenを取得しました。

しかし、要素が別の行にジャンプすると、さらに大きな奇跡が始まります。マージン左autoは、最初の要素を左端から可能な限り遠くにプッシュします。さあ、さあ、どれくらいのスペースが残っていますか？100％-要素サイズ-成長から半分自由=半分自由！ここに配置があります！

2番目の要素は左端にあり、その基本サイズに半分を加えたものに等しいスペースを占めます。信じられないですが、text-align：rightは完全に中央に要素を送信します！

誰かがまだ信じていない場合は、ここに実用的な例へのリンクがあります。要素が適切なタイミングでジャンプし、理想的には中央に配置されていることがはっきりとわかります。

そのようなソリューションを作成する鍵はflex-grow：0.5です。これがないと、大画面でmargin- [left-right] autoを無効にすることができません。もう1つの素晴らしい事実は、正当化コンテンツがまったく使用されないことです。状況の全体的な逆説は、コンテンツをjustify-contentに揃えようとすると、スキームが壊れるということです。

まったく同じプロパティを使用して、たとえば、メニューやロゴ、サンドボックスへのリンクなど、より大きく複雑な要素を組み合わせることができます。メニューをボタンに変える必要があることを改めて強調しますが、今では必要以上に早くメニューが飛んでくることを心配する必要がなくなりました。結局、飛んでもメニューはとても美味しそう！

改善されたソリューション


YouTubeのインタラクティブに関するコメントでは、タイトルが小さな画面の1行に配置されるまで、テキストが正しく配置されていることにすぐに気付きました。新しい行では、テキストは左にプッシュされます。



タイトルが元々2番目の要素（日付の右側にある）だった場合、text-align：rightがそれに作用します。そのため、テキストはさらに奇妙に見えます。ちなみにflex-grow 0.5のため、背景と日付を設定できません。

ただし、これはすべて、プロパティを持つ子を作成することで簡単に解決できます

some-selector {
  display: inline-block;
  text-align: center
}

行ブロッキングのために、このような要素はコンテンツのサイズで正確に取得されます。非常に重要なのは、親のテキスト配置により、適切な場所に配置されることです。すでに要素の内部では、必要なテキスト配置を使用しています。ほとんどの場合、中央揃えです。

flex-elementに1行しかないまで、この内部text-alignの動作を感じないことが重要です。

ちなみに、インラインフレックスも結構です！

結論


この手法は、特定の問題に対する興味深い解決策にすぎません。もちろん、メディアリクエストを拒否せず、メディアのないフレックスボックスを信じてください。しかし、検討した例では、私の意見では、この解決策は非常に興味深いようです。
