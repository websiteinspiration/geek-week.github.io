Gradleチートシート

ほとんどの人は、プロジェクトに何かを追加する必要がある場合、または何かが突然壊れた場合にのみ、gradleに対処し始めるように見えます。「過労によって得られた」問題を解決した後、経験は安全に忘れられます。さらに、インターネット上の多くの例は、何が起こっているのか理解を深めない高度に専門化された呪文に似ています。


android {
    compileSdkVersion 28
    defaultConfig {
        applicationId "com.habr.hello"
        minSdkVersion 20
        targetSdkVersion 28
    }
    buildTypes {
        release {
            minifyEnabled false
        }
    }
}

上記の各行の詳細は説明しません。これらは、Androidプラグインの実装に関するプライベートな詳細です。より価値のあるものがあります-すべてがどのように編成されているかを理解すること。情報はさまざまなサイト/公式ドキュメント/クレードルのソースとそのプラグインに散在しています-一般に、これは私が忘れたくないもう少し普遍的な知識です。


追加のテキストは、単にGradleを習得している、またはすでに忘れている場合のチートシートと見なすことができます。
