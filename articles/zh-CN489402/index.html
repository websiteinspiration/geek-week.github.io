Windows Terminal预览版v0.9

Windows Terminal 0.9版已经发布。这是终端的最新版本，它将在v1发行之前包含新功能。您可以从Microsoft Store或从GitHub的发布页面下载Windows Terminal 。让我们仔细看看更新细节！



命令行参数


别名wt现在支持命令行参数。现在，您可以启动带有新选项卡和面板的终端，这些选项卡和面板根据您喜欢的配置文件从您喜欢的目录开始按自己的喜好进行划分！可能性是无止境！以下是一些示例：

wt -d。
在当前工作目录中打开一个具有默认配置文件的终端。

wt -d。; new-tab -d C：\ pwsh.exe
打开带有两个选项卡的终端。在第一个-默认配置文件中，从当前工作目录开始。第二个是从C：\目录开始的默认配置文件，其中pwsh.exe为“命令行”（而不是默认配置文件“ commandline”）。

wt -p“ Windows PowerShell” -d。分割窗格-V wsl.exe
打开一个终端，其中两个面板垂直分开。在顶部面板上启动名称为“ Windows Terminal”的配置文件，在底部面板上使用默认配置文件，并将wsl.exe用作“命令行”（而不是默认配置文件“ commandline”）。

wt -d C：\用户\肉桂\ GitHub \ WindowsTerminal; split-pane -p“命令提示符”；拆分窗格-p“ Ubuntu” -d \\ wsl $ \ Ubuntu \ home \ cinnak -H

PowerShell自动发现


如果您是PowerShell Core的忠实拥护者，那么我们对您来说是个好消息。Windows终端现在可以检测到任何版本的PowerShell，并自动为您创建一个新的配置文件。我们认为，最好的PowerShell版本（从最大的版本号开始，直到GA版本，再到具有最佳捆绑包的版本）将被称为“ PowerShell”，并将占据下拉列表中的原始PowerShell Core插槽。



确认关闭所有标签


您是不想每次都被要求关闭所有标签的人吗？如果您回答“是”，那么此新功能绝对适合您！已创建一个新的全局参数，使您始终可以隐藏“关闭所有选项卡”确认对话框。为此，请将profiles.json文件顶部的“ confirmCloseAllTabs”参数设置为false，您将再也看不到此弹出窗口！感谢@ rstat1为这个新参数的贡献。
