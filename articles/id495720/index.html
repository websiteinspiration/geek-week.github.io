Instal desktop Linux di Android

Halo, Habr! Saya mempersembahkan kepada Anda terjemahan sebuah artikel dari majalah APC.


Artikel ini membahas instalasi lengkap dari lingkungan operasi Linux bersama dengan lingkungan desktop grafis pada perangkat Android.

Salah satu teknologi utama yang banyak digunakan sistem Linux pada Android adalah pRoot . Ini adalah implementasi ruang pengguna dari utilitas chroot, yang sangat populer di desktop dan server Linux. Namun, alat chroot membutuhkan hak pengguna root, yang tidak tersedia secara default di Android. pRoot memberikan keuntungan ini dengan membuat ikatan ke direktori.

Terminal Linux


Tidak semua emulator terminal Linux untuk Android memiliki satu set utilitas BusyBox, tidak seperti, misalnya, Termux. Alasan untuk ini adalah kenyataan bahwa seluruh poin dari sistem tersebut adalah untuk menyediakan instalasi "lengkap" dari semua komponen OS, sementara BusyBox dirancang untuk mengurangi semua utilitas umum menjadi file biner tunggal. Linux di Android


Seperti disebutkan sebelumnya, paket perangkat lunak yang akan kita instal berjalan di ruang pengguna.

Ini berarti bahwa mereka hanya memiliki izin untuk pengguna saat ini, yang dalam kasus Android OS selalu merupakan pengguna biasa, mis. tidak memiliki hak administrator. Namun, untuk menginstal desktop Linux, kita perlu menginstal server grafis, misalnya, X atau Wayland. Jika kami melakukan ini di lingkungan operasi Linux, maka itu akan dimulai dengan hak istimewa pengguna biasa, tanpa memiliki akses ke tingkat grafis OS Android. Dan oleh karena itu, kita harus melihat ke arah menginstal server dengan cara Android "standar", sehingga memiliki akses ke perangkat keras dan kemampuan untuk mendukung lingkungan grafis.

Orang-orang pintar dari komunitas pengembang telah mengusulkan dua solusi untuk masalah ini. Yang pertama adalah menggunakan versi Linux Anda sendiri (biasanya server X). Setelah mereka mulai bekerja di latar belakang, Anda akan memiliki akses ke proses latar belakang ini melalui VNC. Jika perangkat Android Anda sudah memiliki program penampil VNC untuk berinteraksi dari jarak jauh dengan komputer lain, gunakan saja untuk mendapatkan akses jarak jauh ke host lokal. Ini adalah solusi yang mudah digunakan, tetapi beberapa pengguna melaporkan mengalami kesulitan dengan program ini.

Opsi kedua adalah menginstal server yang dirancang khusus untuk perangkat Android. Beberapa server tersedia di Play Store dalam versi gratis dan berbayar. Sebelum instalasi, Anda perlu memeriksa apakah opsi yang dipilih didukung atau setidaknya berfungsi dengan paket perangkat lunak Linux untuk Android yang akan Anda instal. Kami lebih suka sistem X-Server, dan karena itu menggunakan paket perangkat lunak XServer XSDL ( tautan ). Artikel ini akan menjelaskan proses instalasi untuk server ini, meskipun mungkin sedikit berbeda jika Anda memiliki aplikasi lain yang diinstal atau menggunakan VNC.

Pemilihan sistem


Seperti halnya X-Server, ada beberapa aplikasi untuk menginstal distribusi sistem Linux di Play Store. Di sini, seperti halnya dengan Termux, kami akan berkonsentrasi pada opsi yang tidak memerlukan otoritas pengguna super, yang pada gilirannya melibatkan tingkat risiko tertentu. Aplikasi ini menyediakan semua fungsi yang diperlukan untuk sebagian besar pengguna, sekaligus menjaga data Anda tetap aman. Berikut ini adalah contoh aplikasi seperti itu di Play Store:

- UserLAnd : Pilihan pengguna yang sangat populer. Aplikasi ini mencakup sekumpulan distribusi umum: Debian, Ubuntu, Arch dan Kali. Anehnya, meskipun kurangnya pilihan berbasis RPM, UserLAnd menyertakan Alpine Linux untuk perangkat dengan memori lebih sedikit.

- AnLinux: Aplikasi ini membantu dalam menginstal satu atau lebih daftar distribusi volume besar dan mungkin termasuk Ubuntu / Debian, Fedora / CentOS, openSUSE, dan bahkan Kali. Di sana Anda juga dapat memilih opsi desktop berbiaya rendah: Xfce4, MATE, LXQtand LXDE. Agar berfungsi, Anda perlu menginstal Termux, dan sistem operasi Android harus dari 5.0 ke atas.

- Andronix sangat mirip dengan AnLinux. Ini mungkin memiliki desain yang lebih baik daripada aplikasi sebelumnya, tetapi mendukung distribusi yang lebih sedikit.

- GNURoot WheezyX: Proyek ini dimulai sebagai varian Linux di Android dan dikembangkan untuk program sumber terbuka. Seperti namanya, ini berfokus pada distribusi Debian, sedangkan 'X' pada akhirnya berarti bahwa aplikasi difokuskan pada desktop grafis. Dan terlepas dari kenyataan bahwa pencipta menghentikan pengembangan proyek demi UserLAnd, GNURoot WheezyX masih tersedia di Play Store, jika ada yang membutuhkannya.

Penulis artikel ini akan menggunakan aplikasi UserLAnd untuk menginstal desktop Linux di Android, dan ada beberapa alasan untuk ini. Pertama, aplikasi ini open source (walaupun AnLinux juga memilikinya). Kedua, ia menawarkan set distribusi yang baik (meskipun tidak termasuk Fedora atau CentOS), juga dengan itu Anda dapat menginstal distribusi dengan persyaratan sistem minimal yang tidak akan memakan banyak ruang dalam memori ponsel cerdas. Tetapi keuntungan utama dari UserLAnd adalah ia memiliki alat pendukung untuk menginstal aplikasi individual alih-alih seluruh distribusi. Kami akan mencari tahu apa artinya ini bagi kami, nanti. Sekarang mari kita instal UserLAnd pada perangkat Anda.

Aplikasi UserLAnd


Unduh aplikasi dari Google Play atau F-Droid ( tautan ) di OS Android. Itu diinstal seperti aplikasi lain - tidak ada yang perlu dilakukan di sini. Setelah itu, luncurkan dari panel aplikasi.

Hal pertama yang akan Anda lihat adalah daftar distribusi. Pada akhirnya, Anda dapat menemukan beberapa opsi desktop: LXDE dan Xfce4. Menutup aplikasinya, Firefox, beberapa game dan beberapa utilitas kantor: GIMP, Inkscape, dan LibreOf ce. Tab ini disebut "Aplikasi". Ini dirancang untuk menginstal aplikasi.

Setelah Anda menginstal sesuatu, entri yang sesuai akan muncul pada tab Session. Di sini Anda dapat memulai atau menghentikan sesi saat ini, serta melihat proses yang berjalan.

"Filesystems" adalah tab terakhir yang menunjukkan instalasi yang sudah selesai. Perlu dicatat bahwa setelah Anda menghapus item apa pun dari Filesystem, informasi tentangnya akan dihapus dari tab Sesi, yang, bagaimanapun, tidak membuktikan sebaliknya. Ini berarti Anda dapat membuat sesi baru berdasarkan sistem file saat ini. Memahami bagaimana hubungan ini bekerja jauh lebih mudah jika Anda melihatnya dalam tindakan, dan oleh karena itu kami akan mulai dengan menginstal aplikasi di lingkungan sistem UserLAnd.


Sebelum Anda menginstal distribusi pada ponsel cerdas Anda, Anda harus memberikan akses UserLAnd ke repositori.

Distribusi di UserLAnd


Pilih salah satu distribusi yang terletak di layar Aplikasi untuk diinstal pada perangkat. Sebagai contoh, kita akan menggunakan Ubuntu. Ketika Anda mengklik ikon, kotak dialog muncul meminta nama pengguna VNC, kata sandi dan kata sandi. Kemudian pilih cara di mana Anda akan mengakses distribusi. Pengunduhan akan dimulai, di mana gambar dasar dari distribusi yang dipilih akan terlibat. File akan dibongkar di direktori UserLAnd.

Ketika unduhan selesai, kembali ke emulator terminal xterm. Anda dapat memasukkan perintah utilitas untuk mengetahui versi Linux yang Anda instal:

uname â€“a

Langkah selanjutnya adalah menginstal desktop menggunakan perintah utilitas Ubuntu:

sudo apt install lxde

Langkah terakhir adalah memastikan lingkungan desktop baru Anda siap dijalankan. Untuk melakukan ini, Anda perlu mengedit file .xinitrcfile, yang saat ini hanya memiliki satu baris / usr / bin / twm . Itu harus diubah ke / usr / bin / startlxde. Sekarang keluar dari sesi XSDL (pastikan untuk menekan tombol STOP di area notifikasi), tahan tombol "daftar Ubuntu" pada tab Sesi, lalu klik "Stop Sesi" dan mulai kembali sesi. Setelah beberapa detik, lingkungan sistem LXDE akan muncul. Anda dapat melakukan hal yang sama seperti di desktop biasa. Mungkin hanya sedikit lebih kecil dan bekerja sedikit lebih lambat: untuk menekan tombol pada perangkat, Anda harus menunggu lebih lama daripada jika Anda melakukannya dengan keyboard dan mouse. Mari kita lihat bagaimana tepatnya kita dapat meningkatkan lingkungan sistem Linux pada smartphone.

Panduan Cepat UserLAnd


Dengan mempelajari isi desktop secara cermat, Anda dapat menemukan rekreasi yang tepat dari versi desktop. Jika Anda menggunakan UserLAnd pada perangkat dengan keyboard dan mouse (terhubung melalui koneksi Bluetooth atau dengan cara lain), akan mudah bagi Anda untuk beradaptasi menggunakan lingkungan sistem Linux dalam format ini. Dengan pengecualian sedikit keterlambatan karena fakta bahwa kursor X-Windows disinkronkan dengan kursor perangkat Android, semuanya bekerja dengan lancar.

Mungkin hal pertama yang ingin Anda lakukan adalah menyesuaikan sistem font default, karena ukuran font desktop terlalu besar untuk layar ponsel. Seperti disebutkan di atas, perintah utilitas tidak akan berfungsi dalam kasus ini, jadi jangan ragu untuk menggunakan alat yang benar-benar tak tergantikan yang diinstal di lingkungan sistem UserLAnd, yang disebut ASAP:

Anda mungkin juga membutuhkan cara alternatif untuk membangun koneksi untuk distribusi Anda. Fakta bahwa Anda awalnya menginstal XSDL tidak berarti bahwa itu harus tidak berubah sepanjang waktu.
