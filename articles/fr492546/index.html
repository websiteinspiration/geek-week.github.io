Vérification de la vulnérabilité de tout site utilisant Nikto

Bonjour à tous! En mars, OTUS lance un nouvel atelier Kali Linux . En prévision du début du cours, nous avons préparé une traduction de matériel utile pour vous. Nous voulons également inviter tout le monde à une leçon gratuite sur le sujet: «Attaques par déni de service et défense contre elles» .




Avant d'attaquer un site, un hacker ou un pentester dresse d'abord une liste d'objectifs. Après avoir effectué une bonne reconnaissance et trouvé des points faibles pour «viser», il aura besoin d'un outil d'analyse de serveur Web tel que Nikto, qui l'aidera à trouver des vulnérabilités - des vecteurs d'attaque potentiels.

Nikto est un simple scanner de serveur Web open source qui analyse un site Web et signale les vulnérabilités trouvées qui peuvent être exploitées ou piratées. De plus, il est l'un des outils les plus utilisés pour analyser les sites Web à la recherche de vulnérabilités dans l'industrie, et dans de nombreux milieux, il est considéré comme la norme de l'industrie.

Malgré le fait que cet outil soit extrêmement efficace, il n'agit pas secrètement. Tout site doté d'un système de détection d'intrusion ou d'autres mesures de sécurité comprendra qu'il est en cours d'analyse. Nikto a été conçu pour tester la sécurité, et personne n'a pensé au secret de son travail.

Comment utiliser Nikto


Si vous exécutez simplement Nikto sur le site Web cible, vous ne comprendrez peut-être pas quoi faire avec les informations obtenues après l'analyse. Nikto ressemble en fait plus à un pointeur laser, ce qui implique une prise de vue, et après un certain temps, vous verrez comment cela fonctionne.

Pour commencer, parlons des cibles. La cible peut être presque n'importe quel endroit où un pirate peut frapper, comme des imprimantes réseau ou un serveur Web. Lorsque nous utiliserons Nikto un peu plus tard, nous devrons lui fournir l'un des trois types d'informations: l'adresse IP du service local, le domaine d'attaque ou le site Web SSL / HTTPS.

Avant de commencer à numériser avec Nikto, il est préférable d'effectuer une exploration préliminaire avec un outil ouvert comme Maltego. Ces outils peuvent être utiles pour créer un profil et former une liste plus précise d'objectifs sur lesquels se concentrer. Une fois cela fait, vous pouvez utiliser Nikto pour rechercher des vulnérabilités potentielles pour les cibles de votre liste.

Si vous êtes chanceux, une vulnérabilité avec un exploit bien connu sera trouvée, ce qui signifie qu'il existe déjà un outil qui aidera à exploiter ce point faible. À l'aide d'un outil approprié qui exploite automatiquement la vulnérabilité, un pirate peut accéder à la cible pour exécuter un nombre illimité d'attaques cachées, comme par exemple l'ajout de code malveillant.

Étape 1: installer Nikto


Si vous utilisez Kali Linux, Nikto sera préinstallé, vous n'avez donc pas besoin de télécharger et d'installer quoi que ce soit. Il sera situé dans la catégorie Analyse de vulnérabilité. Si pour une raison quelconque vous ne l’avez pas, vous pouvez télécharger Nikto depuis son référentiel sur GitHub ou simplement utiliser la commande apt install.

apt install nikto

Si vous utilisez un Mac, vous pouvez utiliser Homebrew pour installer Nikto.

brew install nikto

Étape 3: utilisez la syntaxe de base


Comme vous pouvez le voir à l'étape précédente, Nikto a de nombreux cas d'utilisation, mais pour nos besoins, nous utiliserons la syntaxe de base <IP ou nom d'hôte> avec l'adresse IP réelle ou le nom d'hôte sans crochets.

nikto -h <IP or hostname>

Cependant, Nikto est capable de scanner SSL et le port 443, qui est utilisé par les sites HTTPS (HTTP utilise le port 80 par défaut). Ainsi, nous ne sommes pas limités uniquement à l'exploration d'anciens sites, nous pouvons évaluer les vulnérabilités des sites utilisant SSL, qui est aujourd'hui une exigence presque obligatoire pour l'indexation dans les résultats de recherche.

Si nous savons que le site cible a SSL, nous pouvons le spécifier dans Nikto pour gagner du temps sur l'exploration en ajoutant -ssl à la fin de la commande.

nikto -h <IP or hostname> -ssl

Étape 5: Numériser l'adresse IP


Maintenant que nous avons effectué une analyse rapide du site Web, nous pouvons essayer d'utiliser Nikto sur le réseau local pour trouver des serveurs intégrés, tels que la page de connexion du routeur ou le service HTTP sur une autre machine, qui est simplement un serveur sans site Web. Pour connaître l'adresse IP que nous utiliserons ifconfig.

L'adresse IP dont nous avons besoin fait référence à inet. Nous pouvons l'utiliser ipcalcpour obtenir la portée du réseau. Si vous ne disposez pas d'ipcalc, vous pouvez l'installer à l'aide de la commande apt install ipcalc, puis réessayer. La plage sera après "Réseau", dans mon cas c'est 192.168.0.0/24.

Maintenant, nous voulons démarrer Nmap pour trouver des services qui fonctionnent dans cette gamme de réseaux. Analysons le port 80 à l'aide de notre plage, pour cela, nous ajouterons -oG (sortie grepable) pour obtenir uniquement les hôtes qui sont opérationnels, c'est-à-dire ceux qui répondent en disant que le port 80 est ouvert. Ensuite, nous enregistrons tout cela dans un fichier, que je nommerai nullbyte.txt, vous, à son tour, pouvez le nommer comme vous le souhaitez.

ipcalc 192.168.0.48

Il existe une petite astuce qui aidera à envoyer tous les hôtes directement à Nikto pour analyse. Nous utilisons catpour lire les entrées stockées dans notre document nullbyte.txt(ou comme vous l'avez appelé vous-même). Après cela, nous utilisons awk , un outil spécial sous Linux qui vous aidera à trouver le modèle suivant, où Up signifie que l'hôte est en place, et print $ 2 signifie que vous devez imprimer le deuxième mot sur chaque ligne, c'est-à-dire uniquement l'adresse IP. Ensuite, nous enverrons les données reçues dans un fichier appelé targetIP.txt(ou ce que vous voulez).

cat nullbyte.txt | awk '/Up$/{print $2}' | cat >> targetIP.txt

Maintenant, nous pouvons afficher le contenu de notre nouveau fichier avec cat pour voir toutes les adresses IP qui ont le port 80 ouvert.

cat targetIP.txt

192.168.0.5
192.168.0.11

Ce format est idéal pour Nikto, car il peut facilement interpréter de tels fichiers. Ainsi, nous pouvons envoyer cette sortie à Nikto avec la commande suivante.

nikto -h targetIP.txt

Les résultats seront similaires à ceux obtenus lors de la numérisation avec SSL.

D'après les informations ci-dessus, nous comprenons qu'il existe un serveur Varnish et certains en-têtes qui peuvent indiquer comment le site Web est configuré. Cependant, des informations plus utiles sont découvertes dans les répertoires qui peuvent aider à intercepter les fichiers de configuration contenant des informations d'identification ou d'autres éléments qui n'ont pas été configurés correctement et qui étaient involontairement disponibles.

Les éléments préfixés avec OSVDB sont des vulnérabilités signalées dans la base de données de vulnérabilités Open Source (un site qui a fermé en 2016). Il est similaire à d'autres bases de données de vulnérabilité telles que SecurityFocus , Microsoft Technet et Vulnerabilities and Exposures (https://cve.mitre.org/). Personnellement, je suis plus proche de la National Vulnerability Database .

Bien que notre analyse n'ait révélé aucune vulnérabilité critique pouvant être exploitée, vous pouvez utiliser l'outil d'aide CVE pour traduire l'identifiant OSVDB en un enregistrement CVE afin de pouvoir utiliser l'un des sites répertoriés ci-dessus.

Supposons que vous trouviez quelque chose qui mérite d'être exploré, par exemple CVE-2018-10933 , la vulnérabilité Libssh, dont nous avons discuté plus en détail précédemment . Le CVE contient des informations sur la façon de l'exploiter, la gravité de la vulnérabilité (par exemple, une vulnérabilité critique) et d'autres informations qui peuvent aider à déterminer le vecteur d'attaque. Si cela vaut la peine, vous pouvez effectuer une recherche avec Metasploit, car quelqu'un a probablement déjà développé un module qui aidera à exploiter facilement cette vulnérabilité.



Étape 7: Numérisez avec Metasploit


L'une des meilleures choses à propos de Nikto est que vous pouvez simplement exporter les informations que vous avez reçues de la numérisation vers un format que Metasploit peut lire. Pour ce faire, utilisez simplement les commandes pour effectuer l'analyse ci-dessus, mais ajoutez-y les indicateurs -Format msf + à la fin . Ce format peut vous aider à faire correspondre rapidement les données obtenues grâce à l'exploit.

nikto -h <IP or hostname> -Format msf+

Donc, dans le guide d'aujourd'hui, nous sommes passés de la définition d'une cible à la recherche de vulnérabilités, puis nous avons associé les vulnérabilités à l'exploit afin de ne pas avoir à faire tout le travail manuellement. Étant donné que Nikto ne fonctionne pas secrètement, il est sage d'exécuter ces analyses via un VPN, Tor ou tout autre type de service afin que votre adresse IP ne soit pas marquée comme suspecte.
