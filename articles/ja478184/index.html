マウスの代わりのKDE接続、または最初の接続の落とし穴



私は旅行中で、古い忠実な友人- ＃！++が搭載されたASER Aspire one AOA110ネットブックと一緒に過ごした時間を過ごすことがわかりました。 長い間使っていなかったので、タッチパッドのループ不良をすっかり忘れていました。 当然、私はマウスを持っていませんでしたが、本当に快適なブラウザを使いたかったので、マウスの代わりにKDE Con​​nectアプリケーションを使用することにしました。それはすべてが単純であるように思われます：接続された両方のデバイスにアプリケーションをインストールし、そしてあなたの健康を楽しんでください。しかし、それはありました...一般的に、すべての順序について。 スマートフォンのPlayマーケットからアプリケーションをインストールして起動します






KDE Con​​nect（スクリーンショット）


アプリケーションをネットブックにインストールして実行します。

sudo apt install kdeconnect
kdeconnect-indicator

デバイスを接続します（重要！）1つのローカルネットワーク。それがwi-fiであるかUSB接続であるかは関係ありません（私のスマートフォンはネットブック用のUSBモデムでした）。

コンソールでkdeconnect-indicatorを起動し、電話で利用可能なデバイスのリストを更新します-そして...

何もない...（スクリーンショット）


Googleが答えを見つける手助けをしてくれます。私が事前に有効にして忘れていた通常のファイアウォール。不要な場合はオフにします。ファイアウォールを有効にしたままにする必要がある人。

sudo ufw disable


ネットブックが利用可能なデバイスのリストに表示されていることがわかります（スクリーンショット）。


デバイスを選択し、[ペアリングをリクエスト]をクリックします。

ペアリングのリクエストを送信します。ネットブックモニターに次のように表示されます。

ペアリングリクエスト：（スクリーンショット）


表示されますが、何もできません。TABボタンが切り替わらない、アクティブ化されない、他のキーも同様です。

アプリケーション開発者は、コンソールアプリケーションを使用してペアリングを提供します
kdeconnect-cli -a
