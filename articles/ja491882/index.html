スクリプトを使用してSecureCRTへの入力を自動化する

多くの場合、ネットワークエンジニアは、メモ帳からコンソールに特定のフラグメントをコピーして貼り付ける作業を行います。通常、いくつかのパラメーターをコピーする必要があります：ユーザー名/パスワードおよびその他。スクリプトを使用すると、このプロセスを高速化できます。しかし、スクリプトの作成とスクリプトの実行というタスクの合計は、手動構成よりも時間がかかります。そうでない場合、スクリプトは役に立ちません。

この記事の目的は何ですか？この記事はファストスタートサイクルからのものであり、複数のデバイスに機器（1つのタスク）をセットアップするときにネットワークエンジニアが時間を節約することを目的としています。SecureCRTソフトウェアと組み込みのスクリプト実行機能を使用します。

前書き


スクリプト実行エンジンは、ボックスからSecureCRTプログラムに組み込まれています。ターミナルにスクリプトが必要な理由：

  • 自動化された入力と出力、および入力/出力の正確性の最小限のチェック。
  • 日常的なタスクのパフォーマンスをスピードアップするには-機器のセットアップ間の一時停止を減らします。（同じ機器でのコピー/過去のアクションの時間によって引き起こされる一時停止の事実上の削減。機器で使用するコマンドのフラグメントが3つ以上あります。）

このドキュメントでは、次のタスクについて説明します。

  • 簡単なスクリプトを作成する。
  • SecureCRTでのスクリプトの実行。
  • シンプルで高度なスクリプトの使用例。（実生活からの実践。）


簡単なスクリプトを作成する。


最も単純なスクリプトは、2つのSendコマンドとWaitForStringコマンドのみを使用します。この機能は、実行されるタスクの90％（またはそれ以上）に十分です。

スクリプトは、Python、JS、VBS（Visual Basic）、Perlなどで機能します。

パイソン


# $language = "Python"
# $interface = "1.0"
def main():
  crt.Screen.Synchronous = True
  crt.Screen.Send("\r")
  crt.Screen.WaitForString("name")
  crt.Screen.Send("admin\r")
  crt.Screen.WaitForString("Password:")
  crt.Screen.Send("Password")
  crt.Screen.Synchronous = False
main()

ネットワーク機器へのFTPコピー。


このスクリプトはコマンドウィンドウ（シェル）を起動し、FTP経由でデータをコピーしました。完了すると、彼はセッションを閉じました。コピーには非常に長い時間がかかり、FTPバッファーのデータはあまり保存されないため、このためにメモ帳を使用することはできません。

# $language = "Python"
# $interface = "1.0"

# Connect to a telnet server and automate the initial login sequence.
# Note that synchronous mode is enabled to prevent server output from
# potentially being missed.

def main():
	crt.Screen.Synchronous = True
	crt.Screen.Send("ftp 192.168.1.1\r")
	crt.Screen.WaitForString("Name")
	crt.Screen.Send("admin\r")
	crt.Screen.WaitForString("Password:")
	crt.Screen.Send("Password\r")
	crt.Screen.WaitForString("ftp")
	crt.Screen.Send("binary\r")
	crt.Screen.WaitForString("ftp")
	crt.Screen.Send("put S5720LI-V200R011SPH016.pat\r")
	crt.Screen.WaitForString("ftp")
	crt.Screen.Send("quit\r")
	crt.Screen.Synchronous = False
main()


スクリプトを使用してユーザー名/パスワードを入力する


ある顧客では、ネットワーク機器へのアクセスが直接閉鎖されました。最初にデフォルトゲートウェイに接続し、次にそこから接続されている機器に接続することで、機器にアクセスできます。接続には、IOS /ソフトウェア機器に組み込まれたsshクライアントを使用しました。したがって、コンソールでユーザー名とパスワードが要求されました。以下のスクリプトを使用して、ユーザー名とパスワードが自動的に入力されました。

# $language = "Python"
# $interface = "1.0"

# Connect to a telnet server and automate the initial login sequence.
# Note that synchronous mode is enabled to prevent server output from
# potentially being missed.

def main():
	crt.Screen.Synchronous = True
	crt.Screen.Send("snmpadmin\r")
	crt.Screen.WaitForString("assword:")
	crt.Screen.Send("Password\r")
	crt.Screen.Synchronous = False
main()

注：スクリプトは2でした。1つは管理者アカウント用で、2つ目はeSIGHTアカウント用です。

スクリプトの実行中にデータを直接追加する機能を持つスクリプト。


タスクは、すべてのネットワーク機器に静的ルートを追加することでした。ただし、各機器のインターネットゲートウェイには独自のゲートウェイがありました（デフォルトゲートウェイとは異なりました）。次のスクリプトはルーティングテーブルを表示し、構成モードに入り、コマンド（インターネットのゲートウェイIPアドレス）を完全には完了しませんでした-この部分を追加しました。Enterキーを押した後、スクリプトはコマンドを実行し続けました。

# $language = "Python"
# $interface = "1.0"

# Connect to a telnet server and automate the initial login sequence.
# Note that synchronous mode is enabled to prevent server output from
# potentially being missed.

def main():
	crt.Screen.Synchronous = True
	crt.Screen.Send("Zdes-mogla-bit-vasha-reklama\r")
	crt.Screen.WaitForString("#")
	crt.Screen.Send("show run | inc ip route\r")
	crt.Screen.WaitForString("#")
	crt.Screen.Send("conf t\r")
	crt.Screen.WaitForString("(config)#")
	crt.Screen.Send("ip route 10.10.10.8 255.255.255.252 ")
	crt.Screen.WaitForString("(config)#")
	crt.Screen.Send("end\r")
	crt.Screen.WaitForString("#")
	crt.Screen.Send("copy run sta\r")
	crt.Screen.WaitForString("[startup-config]?")
	crt.Screen.Send("\r")
	crt.Screen.WaitForString("#")
	crt.Screen.Send("exit\r")
	crt.Screen.Synchronous = False
main()

このスクリプトの行crt.Screen.Send（「ip route 10.10.10.8 255.255.255.252」）では、ゲートウェイのIPアドレスは追加されず、改行文字はありません。スクリプトは、「（config）＃」という文字を含む次の行が表示されるのを待っています。これらの文字は、IPアドレスを入力して入力した後に表示されました。
