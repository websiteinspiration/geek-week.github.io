Web Sip client based on JsSIP + FreePBX

FreePBX has a Web Sip client. I was wondering how it works. Starting to learn WebRTC technology, I came across 3 libraries - these are JsSIP, its fork SIP.js, and also sipml5.



JsSIP and SIP.js could not be started the first time, there was not enough experience. But sipml5 was the easiest to use. Having downloaded its test version, it turned out to make the first call.



All 3 libraries are operational. Each has its own strengths and weaknesses. Having tested all 3 libraries, I chose JsSIP.



The main pros and cons that I encountered:



1. JsSIP



Pros:



  • lightweight;
  • actively supported by developers.


Minuses:



  • at the initial stage, more difficult to start;
  • I had to edit the sources to work with asterisk.


2. Sip.js



Pros:



  • lightweight;
  • emphasis is placed on working with asterisk.


Minuses:



  • Could not start sound for Early media (sound is not connected for operator pre-message messages).


3. Sipml5



Pros:



  • relatively simple for beginners.


Minuses:



  • complex and confusing code, there was no desire to understand it.


Training



The web client only works on a page with encryption support. We need to create a self-signed certificate. How is this done there are ready-made articles on the Internet.



Firefox allows you to work with self-signed certificates. To work in Chrome, you will have to do a root certificate first, and based on it, already sign a certificate for the phone to work. The root certificate must be imported into the certificate store.



Enabling support for web sip client on FreePBX



Administrator -> User Managment -> Tab ‚ÄúGroups‚Äù -> Tab ‚ÄúUCP‚Äù -> Tab ‚ÄúPhone‚Äù

For the ‚ÄúEnable Phone‚Äù property, select ‚ÄúYes‚Äù.



If you had internal numbers in FreePBX, say 200, then when you activate the web sip client, additional numbers with the prefix 99 will appear, i.e. 99200.



On the client browser, after you have given permission to use a self-signed certificate, you still need to give permission for the asterisk certificate. To do this, type in the browser 



https://192.168.1.70:8089/ws


The IP address must be specified on which asterisk is running.



Editing JsSIP source



In the sources of the JsSIP library, it is necessary to replace transport = ws with transport = wss. Without this edit, the client will not work. You do not need to edit anything in Sip.js, this moment was provided for there.

var socket = new JsSIP.WebSocketInterface('wss://192.168.1.70:8089/ws');

ua.on('disconnected', function(e) { /*   */ });

var eventHandlers = {

		console.log('call is in progress');

			console.log(e);

	'eventHandlers'    : eventHandlers,
	'mediaConstraints' : { 'audio': true, 'video': false }

</script>

Summary



Over the past 2 years of using the Web sip client, there have been problems that appeared when updating the browser.



For example, in version 63 of Firefox, I had to change the SDP protocol so that the session would not be beaten. The browser stopped communicating without the parameter a = mid: 0. In the latest versions it is no longer relevant, the error has been fixed, in later versions there is no such problem.



In version 65 of Firefox with ref. the interlocutor‚Äôs voice began to disappear on the call, there were no such problems in LAN. The problem is only when calling out through the gateway. In chrome, such a problem was not observed, everything works stably.



Mostly users work in Firefox, and for this reason it was necessary to remove the renegotiation of the SDP protocol in the library. Problems with the loss of sound after the answer ceased to bother.
