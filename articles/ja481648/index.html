Junos PyEZは、無料のIPv4サブネットを見つけるためのタスクの例として

Junos PyEZの操作に関する記事-「Junos OSを実行するデバイスの管理と自動化を可能にするPythonマイクロフレームワーク」の自動化と制御。この記事で説明するスクリプトを書くことは、Pythonを学習し、情報を収集するタスクを自動化したり、Junos OSを実行している機器の構成を変更したりするなど、いくつかの目標を追求しました。このPython + Junos PyEZバンドルの選択は、Pythonプログラミング言語を入力するための低いしきい値とJunos OSの専門知識を必要としないJunos PyEZライブラリの使いやすさのために行われました。

仕事


会社所有のipv4フリーサブネットの監査。サブネットが空いているという基準は、Junos OSを実行しているルーターとして機能するスイッチのルートに、そのサブネットに関するレコードがないことです。

実装


Python + Junos PyEZ、ただしparamikoとssh.exec_commandを介してそれを実行する誘惑がありましたが、その結果、調査対象の機器でnetconfデバイスのネットワーク管理プロトコルを構成する必要があります。NetconfはリモートプロシージャコールRPCを介して機器と連携し、この例ではXMLを使用して受信した情報を提供します。

Junos PyEZの現在のバージョンをPyPIからインストールするには、次のコマンドを使用します。

$ pip install junos-eznc

次のコマンドを使用して、GitHubのプロジェクトのメインブランチからインストールすることもできます。

$ pip install git+https://github.com/Juniper/py-junos-eznc.git

そしてもう1つのオプション

$ pip install -r requirements.txt

このコマンドは、システムに存在しない作業に必要なライブラリをインストールします。私のバージョンのrequirements.txtには2つしかありません。最後のバージョンは、スクリプトの作成時に示されています。

junos-eznc
netaddr

デフォルトのスクリプトはシステム内の現在のユーザーの名前を取得します。キーshow_route.py -u <user_name>を使用して別のユーザーの名前でログインできます。getpass.getpassはstdinからのパスワードを受け入れるため、パスワードはシステムに残りません。機器に接続するには、要求に応じてホスト名またはIPアドレスを入力する必要もあります。デバイスの認証に必要なすべてのデータが受信されました。

Junos PyEZは、sshを介したコンソール、telnet、またはnetconfを使用したJunos OSを実行している機器への接続をサポートしています。この記事では、最後のオプションを検討しています。

機器に接続するには、jnpr.junosモジュールのDeviceクラスが使用されます。

with jnpr.junos.Device(host=router,
                           user=args.name,
                           passwd=password) as dev:

要求は、リモートプロシージャコールまたはリモートプロシージャへの呼び出しを介して、ルーターにとって既知のすべてのルートについて行われます。 

data = dev.rpc.get_route_information()

Junos OSでの同様のコマンド 

user@router> show route | display xml

コマンドの最後にrpcコマンドを追加すると、リクエストタグが取得され、それをRPCメソッドの名前と一致させることができます。これにより、興味のある他の名前を見つけることができます。リクエストタグを記述するための構文はメソッド名とは異なります。つまり、ハイフンをアンダースコアに置き換えます。

user@router> show route | display xml rpc
<rpc-reply xmlns:junos="http://xml.juniper.net/junos/15.1R1/junos">
    <rpc>
        <get-route-information>
        </get-route-information>
    </rpc>
</rpc-reply>

ルートデータをxml形式で受け取り、その中から<rt-destination> xxx.xxx.xxx.xxx/yy </ rt-destination>タグで興味のあるものだけを選択して、文字列形式のリストとして変数に書き込み、リストを取得しました。忙しいサブネット。

route_list = data.xpath("//rt-destination/text()")

ルーターが既に知っているサブネットとは異なるサブネットをチェックする必要がある場合に、ルーターへのリクエストを再実行しないように、残りはwhileループにラップされます。私が要求するルーターはOSPFを介してのみルートを認識しているため、境界ルーターの場合、要求を少し変更してスクリプトの時間を短縮することをお勧めします。

data = dev.rpc.get_ospf_route_information()

次に、whileループの内容を見てみましょう


最初に、ユーザーは同じサブネットのネットワークから3オクテット以下のマスクでサブネットを入力するように求められます。これは、検索範囲を設定するために必要です。基準と検索範囲を設定するこの実装はあまり好きではありませんが、これまでのところ、これ以上の解決策はありません。次に、route_listサブネットの結果リストから、3オクテット以下の変数を使用して、関心のあるサブネットを選択します。

tmp = re.search(r'^%s\S*' % subnet_search, route_list[i])

IPNetwork、netaddrモジュールを使用して、サブネットをipv4アドレスのリストとして取得します 

range_subnet = netaddr.IPNetwork(tmp.group(0))

ユーザーがマスクで入力したネットワークのIPNetworkを使用して、アドレスの範囲を取得し、ビジーアドレスのリストと比較するために、この範囲からすべてのアドレスのリストを作成します。 

for i in set(net_list).difference(set(busyip)):
        freeip.append(i)

受信した無料アドレスのリストをサブネットの形式で推測します 

print(netaddr.IPSet(freeip))

以下は、ルータとして使用されているスイッチでテストされた完全なスクリプトです。ex4550、ex4600モデル


#!/usr/bin/env python3
# -*- coding: utf-8 -*-
import argparse
import getpass
import netaddr
import re
import sys

import jnpr.junos

parser = argparse.ArgumentParser()
parser.add_argument('-u', '--user',
                    action='store',
                    dest='name',
                    help='Enter login from tacacs if it differs from the '
                         'username in the system.')
args = parser.parse_args()

if not args.name:
    args.name = getpass.getuser()    # Return the “login name” of the user.
router = input("Full routers name: ")
password = getpass.getpass("Password: ")

try:
    # Authenticates to a device running Junos, for get information about routs
    # into xml format and selects by tag.
    route_list = []
    with jnpr.junos.Device(host=router,
                           user=args.name,
                           passwd=password) as dev:
        data = dev.rpc.get_route_information()
    route_list = data.xpath("//rt-destination/text()")
except (jnpr.junos.exception.ConnectRefusedError,
        jnpr.junos.exception.ConnectUnknownHostError) as err:
    print("Equipment name or password wrong.")
    sys.exit(1)

while True:
    subnet = input("Net with mask: ")
    subnet_search = input("Input no more three octet: ")
    # Gets a list of busy IP addresses from the received subnets.
    busyip = []
    for i in range(len(route_list)):
        tmp = re.search(r'^%s\S*' % subnet_search, route_list[i])
        if tmp:
            range_subnet = netaddr.IPNetwork(tmp.group(0))
            for ip in range_subnet:
                busyip.append("%s" % ip)
    range_subnet = netaddr.IPNetwork(subnet)
    # Gets list ip adresses from subnetworks lists.
    net_list = []
    for ip in range_subnet:
        net_list.append("%s" % ip)
    # omparing lists.
    freeip = []
    for i in set(net_list).difference(set(busyip)):
        freeip.append(i)
    print(netaddr.IPSet(freeip))

    request = input("To run request again enter yes or y, "
                    "press 'enter', complete request: ")
    if request in ("yes", "y"):
        continue
    else:
        print('Bye')
        break
