Installieren Sie den Linux-Desktop unter Android

Hallo Habr! Ich präsentiere Ihnen die Übersetzung eines Artikels aus dem APC-Magazin.


Dieser Artikel beschreibt die vollständige Installation der Linux-Betriebsumgebung zusammen mit der grafischen Desktop-Umgebung auf Android-Geräten.

Eine der Schlüsseltechnologien, die viele Linux-Systeme unter Android verwenden, ist pRoot . Dies ist eine User-Space-Implementierung des Dienstprogramms chroot, das auf Linux-Desktops und -Servern sehr beliebt ist. Das Chroot-Tool benötigt jedoch Root-Benutzerrechte, die unter Android standardmäßig nicht verfügbar sind. pRoot bietet diesen Vorteil, indem eine Bindung zum Verzeichnis hergestellt wird.

Linux-Terminals


Im Gegensatz zu Termux verfügen nicht alle Linux-Terminalemulatoren für Android über eine Reihe von BusyBox-Dienstprogrammen. Der Grund dafür ist die Tatsache, dass der Sinn solcher Systeme darin besteht, eine „vollständige“ Installation aller Betriebssystemkomponenten bereitzustellen, während BusyBox so konzipiert ist, dass alle gängigen Dienstprogramme in einer einzigen Binärdatei zusammengefasst werden. Auf Systemen, auf denen BusyBox nicht installiert ist, wird Bootstrapping für Linux verwendet, das Vollversionen von Programmen enthält.

Diese Systeme verfügen jedoch über zusätzliche Technologie, für die Termux nicht erforderlich ist. Dieser Artikel behandelt die vollständige Installation der Linux-Distribution sowie des GUI-Desktops. Zunächst müssen Sie jedoch die Installationsmethode des Grafiksystems auswählen.

Linux auf Android


Wie bereits erwähnt, werden die Softwarepakete, die wir installieren, im Benutzerbereich ausgeführt.

Dies bedeutet, dass sie nur die Berechtigung für den aktuellen Benutzer haben, der im Fall von Android OS immer ein regulärer Benutzer ist, d. H. hat keine Administratorrechte. Um den Linux-Desktop zu installieren, müssen wir jedoch einen grafischen Server installieren, z. B. X oder Wayland. Wenn wir dies in der Linux-Betriebsumgebung tun würden, würde dies mit den Berechtigungen eines normalen Benutzers beginnen, ohne Zugriff auf die grafische Ebene des Android-Betriebssystems zu haben. Daher sollten wir uns darum bemühen, den Server auf die „normale“ Android-Art zu installieren, damit er Zugriff auf Hardware hat und die grafische Umgebung unterstützen kann.

Kluge Köpfe aus der Entwicklergemeinde haben zwei Lösungen für dieses Problem vorgeschlagen. Die erste besteht darin, eine eigene Linux-Version zu verwenden (normalerweise einen X-Server). Nachdem sie im Hintergrund arbeiten, haben Sie über VNC Zugriff auf diesen Hintergrundprozess. Wenn Ihr Android-Gerät bereits über ein VNC-Viewer-Programm für die Remote-Interaktion mit anderen Computern verfügt, verwenden Sie es einfach, um Remotezugriff auf den lokalen Host zu erhalten. Dies ist eine benutzerfreundliche Lösung, aber einige Benutzer berichten, dass sie Probleme mit dem Programm haben.

Die zweite Möglichkeit besteht darin, einen Server zu installieren, der speziell für Android-Geräte entwickelt wurde. Einige Server sind im Play Store sowohl als kostenlose als auch als kostenpflichtige Version erhältlich. Vor der Installation müssen Sie überprüfen, ob die ausgewählte Option unterstützt wird oder zumindest mit dem Linux-Softwarepaket für Android funktioniert, das Sie installieren möchten. Wir bevorzugten das X-Server-System und verwendeten daher das XServer XSDL-Softwarepaket ( Link ). In diesem Artikel wird der Installationsprozess für diesen Server beschrieben. Er kann jedoch geringfügig abweichen, wenn Sie eine andere Anwendung installiert haben oder VNC verwenden.

Systemauswahl


Wie bei X-Servern gibt es im Play Store verschiedene Anwendungen zum Installieren von Linux-Systemdistributionen. Wie bei Termux konzentrieren wir uns auch hier auf Optionen, für die keine Superuser-Berechtigung erforderlich ist, was wiederum ein gewisses Risiko mit sich bringt. Diese Anwendungen bieten alle für die meisten Benutzer erforderlichen Funktionen und schützen gleichzeitig Ihre Daten. Im Folgenden finden Sie Beispiele für solche Anwendungen im Play Store:

- UserLAnd : Eine sehr beliebte Auswahl an Benutzern. Die Anwendung enthält eine Reihe gängiger Distributionen: Debian, Ubuntu, Arch und Kali. Seltsamerweise enthält UserLAnd trotz des Fehlens von RPM-basierten Optionen Alpine Linux für Geräte mit weniger Speicher.

- AnLinux: Diese Anwendung hilft bei der Installation einer oder mehrerer großvolumiger Verteilerlisten und kann Ubuntu / Debian, Fedora / CentOS, openSUSE und sogar Kali enthalten. Dort können Sie auch kostengünstige Desktop-Optionen auswählen: Xfce4, MATE, LXQt und LXDE. Um zu arbeiten, muss Termux installiert sein und das Android-Betriebssystem muss ab 5.0 verfügbar sein.

- Andronix ist AnLinux sehr ähnlich. Es hat möglicherweise ein besseres Design als die vorherige Anwendung, unterstützt jedoch weniger Distributionen.

- GNURoot WheezyX: Dieses Projekt startete als Linux-Variante unter Android und wurde für Open Source-Programme entwickelt. Wie der Name schon sagt, konzentriert es sich auf Debian-Distributionen, während das 'X' am Ende bedeutet, dass sich die Anwendung auf den grafischen Desktop konzentriert. Und trotz der Tatsache, dass die Entwickler die Entwicklung des Projekts im Interesse von UserLAnd eingestellt haben, ist GNURoot WheezyX weiterhin im Play Store erhältlich, falls jemand dies benötigt.

Die Autoren dieses Artikels verwenden die UserLAnd-Anwendung, um den Linux-Desktop unter Android zu installieren. Dafür gibt es mehrere Gründe. Erstens ist die Anwendung Open Source (obwohl AnLinux es auch hat). Zweitens bietet es eine Reihe guter Distributionen (obwohl Fedora oder CentOS nicht enthalten sind). Es ist auch möglich, Distributionen mit den minimalen Systemanforderungen zu installieren, die nicht viel Speicherplatz im Speicher des Smartphones beanspruchen. Der Hauptvorteil von UserLAnd besteht jedoch darin, dass es Support-Tools zum Installieren einzelner Anwendungen anstelle ganzer Distributionen bietet. Wir werden später herausfinden, was genau dies für uns bedeutet. Jetzt installieren wir UserLAnd auf Ihrem Gerät.

UserLAnd-Anwendung


Laden Sie die Anwendung von Google Play oder F-Droid ( Link ) unter dem Android-Betriebssystem herunter . Es wird wie jede andere Anwendung installiert - hier muss nichts Besonderes getan werden. Starten Sie es anschließend über das Anwendungsfenster.

Das erste, was Sie dort sehen werden, ist eine Liste von Distributionen. Am Ende finden Sie einige Desktop-Optionen: LXDE und Xfce4. Schließt seine Anwendung Firefox, einige Spiele und mehrere Office-Dienstprogramme: GIMP, Inkscape und LibreOf. Diese Registerkarte heißt "Anwendungen". Es wurde entwickelt, um Anwendungen zu installieren.

Nachdem Sie etwas installiert haben, wird auf der Registerkarte Sitzung ein entsprechender Eintrag angezeigt. Hier können Sie die aktuelle Sitzung starten oder stoppen sowie die laufenden Prozesse anzeigen.

"Dateisysteme" ist die letzte Registerkarte, auf der bereits abgeschlossene Installationen angezeigt werden. Es ist zu beachten, dass nach dem Löschen eines Elements aus Dateisystemen Informationen darüber auf der Registerkarte Sitzung gelöscht werden, was jedoch nichts anderes beweist. Dies bedeutet, dass Sie eine neue Sitzung basierend auf dem aktuellen Dateisystem erstellen können. Das Verständnis der Funktionsweise dieser Beziehung ist viel einfacher, wenn Sie sie in Aktion sehen. Daher beginnen wir mit der Installation der Anwendung in der UserLAnd-Systemumgebung.


Bevor Sie die Distribution auf Ihrem Smartphone installieren, müssen Sie UserLAnd Zugriff auf das Repository gewähren.

Distributionen in UserLAnd


Wählen Sie eine der Distributionen auf dem Anwendungsbildschirm aus, die auf dem Gerät installiert werden sollen. Als Beispiel werden wir Ubuntu verwenden. Wenn Sie auf das Symbol klicken, wird ein Dialogfeld angezeigt, in dem Sie nach dem VNC-Benutzernamen, dem Kennwort und dem Kennwort gefragt werden. Wählen Sie dann die Art und Weise aus, wie Sie auf die Distribution zugreifen möchten. Der Download beginnt, wobei das Basis-Image der ausgewählten Distribution beteiligt ist. Die Datei wird im UserLAnd-Verzeichnis entpackt.

Wenn der Download abgeschlossen ist, kehren Sie zum xterm-Terminalemulator zurück. Sie können einen Dienstprogrammbefehl eingeben, um herauszufinden, welche Linux-Version Sie installiert haben:

uname –a

Der nächste Schritt besteht darin, den Desktop mit dem Befehl des Ubuntu-Dienstprogramms zu installieren:

sudo apt install lxde

Der letzte Schritt besteht darin, sicherzustellen, dass Ihre neue Desktop-Umgebung betriebsbereit ist. Dazu müssen Sie die .xinitrcfile-Datei bearbeiten , die derzeit nur eine Zeile / usr / bin / twm enthält . Es muss in / usr / bin / startlxde geändert werden. Beenden Sie nun die XSDL-Sitzung (drücken Sie im Benachrichtigungsbereich unbedingt die STOP-Taste), halten Sie die Schaltfläche "Ubuntu-Liste" auf der Registerkarte "Sitzungen" gedrückt, klicken Sie dann auf "Sitzungen stoppen" und starten Sie die Sitzungen neu. Nach einigen Sekunden sollte die LXDE-Systemumgebung angezeigt werden. Sie können darin dasselbe tun wie auf einem normalen Desktop. Es ist möglicherweise etwas kleiner und arbeitet etwas langsamer: Um eine Taste auf dem Gerät zu drücken, müssen Sie länger warten, als wenn Sie es mit Tastatur und Maus getan hätten. Mal sehen, wie genau wir die Linux-Systemumgebung auf einem Smartphone verbessern können.

UserLAnd Kurzanleitung


Wenn Sie den Inhalt des Desktops sorgfältig studieren, können Sie die genaue Neuerstellung der Desktop-Version finden. Wenn Sie UserLAnd auf einem Gerät mit Tastatur und Maus verwenden (über eine Bluetooth-Verbindung oder auf andere Weise verbunden), können Sie sich leicht an die Verwendung der Linux-Systemumgebung in diesem Format anpassen. Mit Ausnahme einer leichten Verzögerung aufgrund der Tatsache, dass der X-Windows-Cursor mit dem Cursor des Geräts unter Android synchronisiert ist, funktioniert alles reibungslos.

Möglicherweise möchten Sie als Erstes das Standardschriftartensystem anpassen, da die Desktop-Schriftgröße für den Telefonbildschirm zu groß ist. Gehen Sie zum Hauptmenü und wählen Sie "Einstellungen" → "Erscheinungsbild und Widgets anpassen" → "Widget". Hier können Sie die Standardschriftgröße in eine für Ihr Telefon geeignetere Option ändern.

Möglicherweise möchten Sie Ihre Lieblingsprogramme später in der Linux-Systemumgebung installieren. Wie oben erwähnt, funktionieren Dienstprogrammbefehle in diesem Fall nicht. Verwenden Sie daher das wirklich unersetzbare Tool, das in der UserLAnd-Systemumgebung namens ASAP installiert ist:

sudo apt install emacs



Verteilungen in der Anwendung werden in Form von Sitzungen dargestellt. Sie können sie starten und schließen.



Nach der Installation der Distribution können Sie eine Desktop-Umgebung mit Standardbefehlen hinzufügen.

Sie benötigen wahrscheinlich auch alternative Möglichkeiten, um eine Verbindung für Ihre Distribution herzustellen. Die Tatsache, dass Sie XSDL ursprünglich installiert haben, bedeutet nicht, dass es ständig unverändert bleiben sollte. Sie können auf der Registerkarte Sitzung ein anderes Konto erstellen und einen anderen Server auswählen. Stellen Sie einfach sicher, dass Sie es im selben Dateisystem angeben. UserLAnd wird versuchen, Sie zur richtigen Anwendung zu leiten, um einen neuen Verbindungstyp herzustellen: entweder XSDL, ConnectBot für SSH oder bVNC.

Die Beharrlichkeit, mit der die Anwendung Sie beim Versuch, die Verbindung wiederherzustellen, automatisch zum Play Store weiterleitet, kann jedoch ärgerlich sein. Um dies zu stoppen, ändern Sie einfach den Server, indem Sie eine spezielle Anwendung installieren. Wählen Sie zum Installieren von SSH den alten bewährten VX ConnectBot aus. Geben Sie einfach Port 2022 auf Ihrer Workstation mit Ihrem Benutzernamen und Passwort ein. Um eine Verbindung zum VNC-Server herzustellen, reicht es aus, die kommerzielle, aber in vielerlei Hinsicht erweiterte Jump Desktop-Anwendung zu installieren und die Adresse 127.0.0.1►951 zu wählen.

Wir hoffen, dass Sie sich an das VNC-Passwort erinnern, das beim Erstellen des Dateisystems festgelegt wurde.
Sie können auch mit ähnlichen Tools auf einem anderen Computer in Ihrem Netzwerk auf die aktuelle UserLAnd-Sitzung zugreifen. Es reicht aus, SSH über ein Linux-Terminal, z. B. Konsole, mit einer Arbeitssitzung (natürlich mit einem SSH-Verbindungstyp) zu verbinden oder über KRDC eine Verbindung zu einer VNC-Sitzung herzustellen. Ersetzen Sie einfach die lokalen Adressen auf dem Computerbildschirm durch die IP-Adressen Ihres Android.

In Kombination mit zwei tragbaren Anwendungen bietet diese Installation ein praktisches tragbares Linux-System, mit dem Sie über jeden Computer, der Ihnen derzeit zur Verfügung steht, eine Verbindung herstellen können.
