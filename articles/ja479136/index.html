非常に単純なブロックチェーンは次のように説明できます。これは、特定のタイプのトランザクションが記録されるレジストリです。ブロックチェーンは、整数の因数分解などの数学関数を使用します。このような機能は一方向で簡単に解決できますが、反対方向では困難です。つまり、セキュリティが確保されます。

ブロックチェーントランザクションは、ブロックと呼ばれるデータベースに追加され、ブロックは「ハッシュ関数」と呼ばれる数学的ツールで暗号化されます。次に、ハッシュは次のトランザクションのセットを含む次のブロックに含まれます。次のステップでは、ハッシュ関数によって再び暗号化され、次のハッシュが提供されます。新しいハッシュが次のブロックに追加されます。このようにして、ブロックのチェーンが形成され、それらはすべて互いに順番にネストされているように見えます-したがって「ブロックチェーン」という名前です。

PythonのMD5アルゴリズムによって生成されたハッシュの例を次に示します。 MD5は、128ビットハッシュを提供する広く使用されているハッシュ関数です。

>>> import hashlib
>>> def hash(mystring):
...     hash_object=hashlib.md5(mystring.encode())
...     print(hash_object.hexdigest())
...
>>>
>>> hash("Kellogg first block")
10a4826ea290595ef96e945b31054254

従来のコンピューターの元の値（この場合は「最初のケロッグブロック」）を復元することは非常に困難です。これはブルートフォースでのみ実行できます-考えられるすべてのオプションを整理し、次の各候補が問題のステートメントを満たしているかどうかを確認することにより、体系的にソリューションを探します。同様に、ビットコインブロックのハッシュソリューション-多くのゼロから開始-は、非常に大量の計算を必要とします。そのため、ビットコインネットワーク内のすべてのコンピューターの総計算能力を使用する場合でも、ブロックを解決するのに約7分かかります。

量子コンピューターとは何ですか？


コンピューティングパワーのほとんどのブレークスルーの基礎の1つは、トランジスタのサイズを継続的に縮小するプロセスです。トランジスタはコンピュータで情報を処理する論理ゲートを形成するため、計算のフォームファクタの本質はトランジスタにあります。



過去数十年にわたって、チップメーカー、特にIntelは、CPU内のトランジスタの数を絶えず増加させ、そのサイズを縮小してきました。最近のCPUでは、各トランジスタはHIVウイルスよりもさらに小さくなっています。実際、トランジスタのサイズが原子と比較される限界に近づいています。そのため、ムーアの法則は将来なくなり、コンピューティングパワーはこれまでのペースで成長しなくなると広く信じられています。

この問題の1つの解決策は、量子コンピューティングです。

情報がトランジスタに基づいて処理される従来のコンピュータでは、ビットは0と1にのみ等しい可能性があります。ただし、量子計算は、それぞれが1、0、または2つのキュービットの状態の量子重ね合わせを表すことができるキュービットのシーケンスに基づいています。一般に、n量子ビットを持つ量子コンピューターは、同時に2 ^ nまでの状態で任意の重ね合わせを行うことができます。

クラシックコンピュータは常に、2 ^ nの状態のいずれかにしかなりません。したがって、量子コンピューターは古典的なコンピューターよりも指数関数的に高速です。

このような重ね合わせはどのようにして可能ですか？実際のところ、量子世界は定義上、平行しています。トーマスヤングが設定した2つのスリットを使用した有名な実験では、粒子は2つのスリットを同時に貫通する可能性があります。最高の量子コンピューターは膨大な量の計算を実行でき、従来のコンピューターよりもはるかに高速です。

量子コンピューティングの原則


私たちのほとんどは学校で古典力学に精通しており、逆に量子コンピューティングの現象は直感的とは言えません。大学のコースで知っている最も簡単な例の1つを紹介してみます。



立方体のどの面が正面かを推測してください。おそらくあなたは確信が持てません。しかし、あなたが自分で決めるとすぐに、混乱は終わります。可能な状態の不確実性は、量子力学の最も深い原理の1つである重ね合わせの原理を説明します。
この原理のおかげで、nキュビットは任意の重ね合わせを表すことができ、同時に最大2 ^ nの異なる状態を含みます。
今この絵を考えてください。正面はどっち？



それは面白いです。ここでも、答えがわからない場合がありますが、最初のキューブを決定すると、すぐに2番目のキューブが決定されます。どういうわけか、空間で分離されたこれらの2つの立方体は接続されています。

量子エンタングルメントは、2つ以上のオブジェクトの量子状態を、個々のオブジェクトが空間で互いに離れていても、互いに関連して記述する必要がある現象です。

量子エンタングルメントは、量子コンピューターの操作に関する超臨界的な詳細です。量子コンピューターはエンタングルメントを設定し、出力を測定して、重ね合わせを0または1（クラシック状態）に折りたたみます。量子世界の多くのアルゴリズムは、一定の確率で正しい答えを出します。ただし、量子コンピューターの結果を繰り返し開始、実行、および測定することにより、正しい答えが得られる可能性を高めることができます。

ブロックチェーンの脅威


上述したように、ブロックチェーン技術は暗号技術の使用に基づいており、巨大な計算能力を使用したブルートフォースを除いて、ハッキングに対してほとんど無力であると考えられています。概念的には、このような暗号化は、インターネット上の通信を提供する技術に似ています。

ただし、量子コンピューターは、その巨大なコンピューティング能力により、理論的には公開鍵暗号を解読する必要があるため、ブロックチェーンに脅威をもたらします。

確かに、量子コンピューティングのスケーリングに関連する問題があります。
キュービットは非常に壊れやすいものです。バックグラウンドノイズでさえ、デコヒーレンスにつながり、粒子の量子的性質を破壊する可能性があります。
有用なキュビットごとに、エラーを修正するためにさらに10〜100キュビットが必要です。この分野の研究は現在進行中であり、提案されているソリューションの1つはトポロジカル量子コンピューターです。

量子コンピューティングの世界におけるブロックチェーンの未来は何ですか？


ブロックチェーンの問題について共同創設者のAndreessen HorowitzのMark Andersonから尋ねられたとき、彼は言った：
ベンチャーキャピタルに大きな問題があると伝えると、大騒ぎになるだけです。したがって、この問題を解決する賢い人を見つける必要があります。

しかし、彼の楽観主義にもかかわらず、これは複雑な問題です。

解決策の1つは、量子暗号に基づく量子ブロックチェーンです。ニュージーランドのウェリントンにあるビクトリア大学のDel RajanとMatt Visserによって提案されました。考え方は単純です。コンピュータがあまりにも早くカウントし始めた場合、タスクを複雑にする必要があります。時間に絡み合った量子粒子に基づいてブロックチェーンを作成します。したがって、1つだけの量子粒子で、そのすべての前身の履歴をエンコードするのに十分であり、このチェーンを破壊せずにこのチェーンを解読することは不可能です。
ただし、ここではすでに制限されているブロックチェーンスケーリングの分野で新しい問題に直面する必要があります。
