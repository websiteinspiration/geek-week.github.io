GDPRコンプライアンスについてCookieを確認する方法-新しいオープンツールが役立ちます

CovenantSQLデータベースの開発者は、サイトがどのCookieをどのような目的で設定するかを決定するオープンソースユーティリティCookie Scannerを導入しました。

このツールが必要な理由とその機能について説明します。


/ Flickr / YannCœuru / CC BY /写真が変更されました

法律が言うこと


一般データ保護規則（GDPR）の第30条では、ユーザーの個人データを使用するサイトに、Cookieのインストールについてユーザーに通知し、同意を得ることを義務付けています。

EUでのCookieの使用は、2009年から施行されている別の法律（eプライバシー指令）によっても規制されています（今年は、以前の資料の1つで説明した、より厳格なeプライバシー規制に置き換わります）。また、Cookieの処理について通知するようにWebサイトの所有者に義務付けています。

GDPR要件およびePrivacy指令に違反した場合、リソースの所有者は、最高2000万ユーロまたは組織の年間売上高の4％の多額の罰金を科せられることがあります。

難しさは何ですか


罰金にもかかわらず、多くのサイトはユーザーの同意なしにいわゆるオプションのCookieを設定します。

サイトが正しく機能するために必要な場合、ユーザーがCookieを設定するための同意を得るためにインターネットリソースは必要とされていません。ただし、法律の表現はかなりあいまいであり、この規則がいつ適用されるかは必ずしも明確ではありません。

これに基づいて、紛争状況が発生します。たとえば、フランスの出版社であるEditions Croque Futurは、マーケティングキャンペーンのために訪問者をサイトのCookieに設定しました。所有者は、これらのCookieがサイトの収益性を保証するため、サービスの運用に不可欠であることを指摘しました。しかし、規制当局は彼らの主張に同意せず、25千ユーロの罰金を会社に送った。

別の問題は、GDPR では、サイト上のすべてのサードパーティのCookie（ソーシャルネットワークプラグインの操作に必要なCookieなど）を指定および説明する必要がないことです。英国情報委員会（Information Commissioner's Office）のガイド（PDF、p.17）の代表者によると、企業はサードパーティのCookieをカテゴリに分類し、その目的を説明するだけで十分です。

イタリアの規制当局は、これは必要な措置であると説明しています。それ以外の場合、所有者はサイト上のサードパーティのCookieを常に監視し、その目的を監視する必要があります。WebマスターがWebサイトにCookieを設定する責任を負うすべての組織と直接連絡をとらないことが多いため、このタスクを達成することは困難です。

実際には、サイトにアクセスすると、ユーザーのブラウザーがサードパーティのCookieをダウンロードできることがわかりますが、その目的ははっきりしていません。

問題を解決する方法


特定のリソースによって設定されたCookieの完全なリストを取得するには（同時に、GDPRの要件に違反しているサイトを見つけます）、Cookieスキャナーユーティリティを使用します。ネットワークには類似のサービスがかなり多数ありますが、その多くは有料です。 Cookie Scannerのコードはオープンで、GitHubのリポジトリにあります。

Cookie ScannerはCookieのステータスを分析し、それらのタスクを説明するレポートを生成します。このツールは、特殊なcookiepediaリソースに関する情報を取得し、CQLデータベースを使用してそれを表します。現在、1万種類のCookieに関する情報が含まれています。 CQLデータベースの操作を開始する方法について-CovenantSQL 開発者が作成したクイックスタートガイド。

Cookie ScannerにはMacOS / LinuxオペレーティングシステムとChromeブラウザが必要です。または、コードとレイアウトのテストに使用されるヘッドレスバージョンを参照することもできます。画面にコンテンツをレンダリングしないため、より高速に動作し、メモリ消費量も少なくなります。

重要なプロパティは、ベアLinuxサーバーにインストールできることです。パッケージを置くだけで、ブラウザはそのまま動作します。

コンテナでヘッドレスクロムを実行するには、次のコマンドを実行します。

$ docker container run -d -p 9222:9222 zenika/alpine-chrome --no-sandbox \
 --remote-debugging-address=0.0.0.0 --remote-debugging-port=9222

Cookie Scannerの設定はgetを使用して行われます：

$ go get github.com/CovenantSQL/CookieScanner

このツールは、PDF、JSON、およびHTML形式のCookieに関するレポートを生成します。PDF形式で情報を出力するためのパーサー関数は、次のようになります。

func outputAsPDF(remote *godet.RemoteDebugger, htmlFile string) (pdfBytes []byte, err error) {
	var tab *godet.Tab

	htmlFile, _ = filepath.Abs(htmlFile)
	fileLink := "file://" + htmlFile

	if tab, err = remote.NewTab(fileLink); err != nil {
		return
	}
	if err = remote.ActivateTab(tab); err != nil {
		return
	}

	// wait for page to load
	time.Sleep(time.Second)

	return remote.PrintToPDF(godet.PortraitMode())
}

これは、CLIモードを使用してHTMLレポートを生成する方法です。

$ CookieScanner cli \
    --headless \
    --classifier "covenantsql://050cdf3b860c699524bf6f6dce28c4f3e8282ac58b0e410eb340195c379adc3a?config=./config/config.yaml" \
    --html cql.html covenantsql.io

プログラムが生成できるレポートの例は、リポジトリの対応するセクションにあります。
