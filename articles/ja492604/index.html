コルーチンを使用して非同期でビューを操作する

レンダリングの前に最初に準備する必要があるビュー（ImageViewなど）がある状況を見てみましょう。たとえば、サイズ、形状を計算したり、青い効果を適用したりします。これらの計算はコストのかかる操作になる可能性があるため、バックグラウンドスレッドに転送することをお勧めします。

ハビスタの祖父は実行可能ファイルを作成し、ハンドラーを使用して結果をメインストリームに転送し、ビューに適用します（最初に頭に浮かぶこと）。

コルーチンを備えた大釜でこれをすばやく簡単に行う方法：

まず、kotlin-extension関数を作成します。

inline fun <T> View.doAsync(
        crossinline backgroundTask: (scope: CoroutineScope) -> T, 
        crossinline result: (T?) -> Unit) {
    val job = CoroutineScope(Dispatchers.Main)
    //  ,    
    // ,   
    val attachListener = object : View.OnAttachStateChangeListener {
        override fun onViewAttachedToWindow(p0: View?) {}
        override fun onViewDetachedFromWindow(p0: View?) {
            job.cancel()
            removeOnAttachStateChangeListener(this)
        }
    }
    this.addOnAttachStateChangeListener(attachListener)
    //  Job,      
    job.launch {
        //  Deferred     
        val data = async(Dispatchers.Default) {
            try {
                backgroundTask(this)
            } catch (e: Exception) {
                e.printStackTrace()
                return@async null
            }
        }
        if (isActive) {
            try {
                result.invoke(data.await())
            } catch (e: Exception) {
                e.printStackTrace()
            }
        }
        //     Job
        this@doAsync.removeOnAttachStateChangeListener(attachListener)
    }
}

次に、状況をシミュレートします。RecyclerViewがあり、各アイテムに画像があります。ショーの前に、この写真をぼかしたい（ぼかし）。非同期なしの場合は次のようになります。

inner class PostHolder(itemView: View) : RecyclerView.ViewHolder(itemView) {
    private val ivTest = itemView.iv_test

次に、関数を使用します。

inner class PostHolder(itemView: View) : RecyclerView.ViewHolder(itemView) {
    private val ivTest = itemView.iv_test
    fun bind() {
        val bitmap = ... 
        itemView.doAsync({ scope -> 
            //        
            return@doAsync bitmap?.addBlurShadow(Color.CYAN, 50.dp, 50.dp)
        }, { it ->
            //            
            ivTest.setImageBitmap(it)
        })
    }
}

結果：



全体として、ビューのライフサイクルに依存しています。したがって、ビューが親から切り離され、アイテムが失われたときにリサイクラーで、またはフラグメント/アクティビティが破棄されたときにこれが常に発生すると、コルーチンを安全に停止してキャンセルできます。結果が表示される場所がないため、ビューを破棄する準備ができています。
