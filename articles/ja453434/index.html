カートリッジの多様性：ゲームボーイ用ゲームのセンサー


90年代の終わりまでに、ほとんどのゲーム用家庭用コンソールはカートリッジから光ディスクに切り替わりました。ただし、いくつかの例外を除いて、ほとんどのポータブルコンソールはドライブに切り替えられませんでした。彼らは引き続きカートリッジの作業を行い、徐々にダウンロード可能なコンテンツ1に移行しました。カートリッジは非常に小さく、ゲームに必要なものだけが含まれており、頻繁に移動するデバイスの小さな回転ディスクにレーザーを正確に配置する必要はありませんでした。時間の経過とともに、小型化の進歩により、さらに高度なハードウェア機能をカートリッジに組み込むことができるようになりました。この傾向は徐々に、機能を拡張するセンサーを備えたカートリッジの登場につながりました。

この記事では、3つの異なる世代のニンテンドーゲームボーイプラットフォーム用の3つのカートリッジを見ていきます。それぞれに追加の機器が含まれています。これらの失敗したカートリッジは、任天堂によれば、テクノロジーがカートリッジを導いたはずの場所を理解することを可能にします。

これは好奇心旺盛なビデオゲームカートリッジに関する3番目の記事です。第一及び第二の翻訳：1、2。

ゲームボーイカメラ


1998年に最初のゲームボーイのために発売されたゲームボーイカメラは、フル機能のデジタルカメラと30枚の写真用のメモリを備えた、変わった形のカートリッジでした。結果として得られた写真は、現代の基準では微視的に小さく、粒子が粗かったという事実にもかかわらず、世紀の変わり目に、ゲームボーイカメラは多くの注目を集め、最小のデジタルカメラ2によって1999年版の「ギネスブック」として認められさえしました。

Game Boy CameraのソフトウェアはGame Freakによって作成されました。この名前は、ポケモンのファンにはおなじみかもしれません。ゲームフリークは、ポケモンシリーズのゲームの主な開発者です。彼女はPokémonRed and Blue（1996）からSwordまでのすべてのゲームを開発し、シールド（2019年末に登場）。 Game Boy Camera用のソフトウェアを作成することは、3社の歴史の中で奇妙なページであり、他のすべてのプロジェクトは非常に人気の高いフランチャイズのバージョンでした。

Game Boy Cameraが作成されたゲームプラットフォームのスタイルに合わせて、WarioWareニンテンドーシリーズの美学に似た楽しいユーザーインターフェイスを備えていました。各画面は混沌としたシンボルで装飾されており、インタラクション中にほとんどすべてのものが動き、ひきつりました。カートリッジは所有者が写真を撮るだけでなく、他のものと同じように狂ったように見えるステッカーや落書きでそれらを飾ることもできました。

ゲームボーイカメラの発売当時、スマートフォンにカメラが登場するまで、デジタル写真は完全に知られたものではなかった4、アマチュア写真の扉を開いた、まだ数年ありました。そして当時、ゲームボーイカメラは研究と創造性のための比較的安価な媒体でした、そしてそれは後に社会に大きな影響を与えた運動の起源の興味深い見方となりました。

数年前、ブロガーのDavid Friedmanがニューヨークの写真の宝庫を共有し、2000年にGame Boy Cameraを作りました5。これは素晴らしくプレーンでピクセル化されたタイムカプセルです。 Flickrには、他にもGame Boy Camera 6のサンプル写真があります。

続編がないことを考えると、ゲームボーイカメラが任天堂にとって大ヒットにならなかったと考えるのは当然です。しかし、写真は明らかに会社の弱点でした。2006年にリリースされたニンテンドーWiiにはデジタルフォトギャラリーアプリケーションがあり、DSi（2009）、3DS（2011）およびWii U（2012）にはカメラが内蔵されていました。そして、今日のほとんどの人々は、低品質の低解像度の白黒カメラであまりメリットを感じていませんが、ゲームボーイカメラは引き続き注目を集めています。発明的なユーザーは、カメラからコンピューター7に画像を転送する多くの方法を考え出しましたが、私にとっては、スマートフォンを使用してゲームボーイの画面の写真を撮るだけの方がいつも簡単でした。

カービーティルト 'n'タンブル


Kirby Tilt 'n' Tumbleは、Game Boy 8カートリッジに組み込まれた新しい種類のゲーム入力の最初の例でした。2000年にゲームボーイカラー用にリリースされたこのホットピンクカートリッジには、ゲームボーイのカービィの傾きをさまざまな方向に制御できる（または必要な）加速度計が組み込まれていました。

ゲームプレイ自体は、2000年代後半の無数のスロープ駆動のモバイルゲームとほぼ同じです。カービィボールの囚人は出口に向かい、途中でオブジェクトを収集する必要があります。キャラクターの代わりに現実世界のオブジェクトを管理することで、ゲームに斬新さが加わり、全体的にサポート的な歓迎を受けました。しかし、良いレビューにもかかわらず、加速度計を備えたゲームはGame Boy Color用にリリースされていませんでした（ただし、そのようなゲームがGame Boy Advance 9に2つ登場しました））


プレイヤー自身が加速度計を較正する必要がありました。

Tumbleは、長年に渡って任天堂と密接な関係を築いてきた10パートナーの開発者であるHAL Laboratoryによって作成されました。 HAL研究所の岩田聡社長は、このゲームがリリースされた頃に任天堂を歩き回り、2年後には同社の社長に就任しました。イベントの順序を考えると、任天堂をジェスチャーコントロールに興味を持ったのは岩田だったと考えられます。結局、2006年にリリースされたWiiは、ジェスチャーコントロールに完全に依存しており、タンブルで使用されているものと同様の加速度計は言うまでもありません。もちろん、この仮定を額面どおりに行う義務はありません。たとえば、2007年にリリースされた最初のiPhone も同様です。彼は加速度計を積極的に使用し、それによって彼の人生の数年の間に、モーションコントロールされたモバイルゲームの巨大なシャフトが登場しました。おそらく、加速度計を使用してモーションを制御するという考えの時が来たのでしょう。

WarioWare：ツイスト！


2004年にリリースされたWarioWareの長いシリーズの第2部であるWarioWare：Twisted！ Game Boy Advanceでは、それ自体の入力タイプだけでなく、特別な出力方法もありました：ジャイロスコープがプレーヤーがコンソールをどのように握るかを決定し、小さな振動モーターが騒々しい触覚フィードバックを作成しました。振動機能は目新しいものではありませんでしたが、ジャイロスコープでした11。

先代同様、Twisted！いわゆる「マイクロゲーム」のコレクションでした。説明、理解、およびほんの数秒で完了する必要のある小さなタスクです。ゲームのこの部分のほとんどのマイクロゲームでは、ジャイロスコープが使用されていたため、プレーヤーは厳しい時間制限でコンソールを回して回す必要がありました。ツイスト！任天堂は大ヒットとなり、WarioWareのコンセプトを1つの成功したゲームから長いシリーズに変える最初のステップであることがわかりました。その登場から15年後、さらに6つの続編がリリースされました。


マイクロゲームの1つ：バリオの湾曲した脊椎を交換する必要があります。

ツイスト！、Kirby Tilt 'n' Tumbleのように、Wiiへの第一歩と考えることができます。 Wiiがリリースされてから数年後、その加速度計ベースのコントローラーにMotionPlusジャイロスコープが追加されました。Kirby Tilt 'n' Tumbleと最初のWiiコントローラーで使用されている加速度計は、加速度を測定します。これらは、向きの変化や突然の動きを測定するのに役立ちますが、プレーヤーがアイテムをどのように保持しているかは判断できません。ジャイロスコープは動きを測定しませんが、向きを測定します。これにより、動きをより正確に制御できます。 MotionPlusで行ったように、これら2つのタイプのセンサーを組み合わせると、非常に正確なモーションコントロールを実装できます。

ツイスト！は専用機器を備えた唯一のWarioWareシリーズのゲームでしたが、ゲームプレイスタイルの実験が続けられました：Touched！（2004）DSスタイラスが使用され、Smooth Moves（2006）はWiiモーションコントロールを使用し、Snapped！（2008）-DSiカメラ。WarioWareシリーズは、好奇心旺盛なゲームプレイアイデアの一種のテストプラットフォームになり、新しいタイプの入力を使用した実験が始まりました。

結論


これらのカートリッジはそれぞれ、コンソールに新しい興味深いものをもたらしましたが、追加の機器をカートリッジに統合することには欠点がありました。明らかに、これらのカートリッジは製造コストがより高くなりました。つまり、追加の機器はそれ自体を正当化する必要がありました。実際、これが起こることはめったにありませんが、そのような実験が新しいユニークなゲームプレイにつながる場合もあります。
