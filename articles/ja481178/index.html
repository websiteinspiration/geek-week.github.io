アナライザーは正しいようです。多くの場合、一部の引数は、たとえば一部の変数を交換するために、意図的に逆の順序で渡されます。しかし、これは正確にそうであるようには見えません。まず、ブール型の変数が混乱しています。次に、メソッドの名前も一般的です。「スワップ」や「リバース」はありません。さらに、そのような間違いをすることはそれほど難しくありません。人々は、行と列のペアの順序を異なる方法で認識することがよくあります。

7位：「見落とされたばかり」


V3078 'OrderBy'メソッドを繰り返し呼び出すと、元の並べ替え順序が失われます。'ThenBy'メソッドを使用して、元の並べ替えを保持します。GridModel.Selection.cs 107

internal partial class GridModel
{
  private void BuildCellSelectionRegions(....)
  {
    ....
    this.MergeCellSelectionRegions(selectedItemsInView
        .OrderBy(c => c.Column.ItemInfo.LayoutInfo.Line)
        .OrderBy(c => c.RowItemInfo.LayoutInfo.Line));
    }
}

「UWPのTelerik UIを確認してPVS-Studioを理解する」という記事のエラー。並べ替え済みのコレクションでOrderByを再利用するため、以前の並べ替えの結果は失われます。ThenByを使用する必要があります。

2位：「順番になってください！」


V3070初期化されていない変数「LANG_USER_DEFAULT」は、「LOCALE_USER_DEFAULT」変数を初期化するときに使用されます。NativeMethods.cs 890

internal static class NativeMethods
{
  ....
  public static readonly int LOCALE_USER_DEFAULT =
    MAKELCID(LANG_USER_DEFAULT);
  public static readonly int LANG_USER_DEFAULT = 
    MAKELANGID(LANG_NEUTRAL, SUBLANG_DEFAULT);
  ....
}

記事「WinForms：エラー、ホームズ」のエラー。クラスフィールドの初期化順序が混乱しています。値を算出するLOCALE_USER_DEFAULTのフィールドを、使用LANG_USER_DEFAULTの分野はまだ初期化され、0の値を有していない、LANG_USER_DEFAULTの変数であるコードのどこにも使用されていません。このエラーの原因を理解するために、WinFormsコードのメソッドを含むテストプログラムを作成しました。簡単にするために、使用されているいくつかの定数の代わりに、それらの実際の値が置き換えられました：

internal static class NativeMethods
{
  public static readonly int LOCALE_USER_DEFAULT = 
    MAKELCID(LANG_USER_DEFAULT);
  public static readonly int LANG_USER_DEFAULT = 
    MAKELANGID(0x00, 0x01);
  
  public static int MAKELANGID(int primary, int sub)
  {
    return ((((ushort)(sub)) << 10) | (ushort)(primary));
  }
  public static int MAKELCID(int lgid)
  {
    return MAKELCID(lgid, 0x0);
  }
  public static int MAKELCID(int lgid, int sort)
  {
    return ((0xFFFF & lgid) | (((0x000f) & sort) << 16));
  }
}
class Program
{
  static void Main()
  {
    System.Console.WriteLine(NativeMethods.LOCALE_USER_DEFAULT);
  }
}

打ち上げの結果、次のようにコンソールに表示されます：0は、今、私たちは宣言交換することによって、エラーを修正しますLOCALE_USER_DEFAULTとLANG_USER_DEFAULTフィールドを。この形式のプログラムの結果：1024。
