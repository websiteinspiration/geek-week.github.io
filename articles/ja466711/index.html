脆弱性DaOfficeがソーシャルネットワークから任意のユーザーを削除することを許可

前書き


2018年の夏に、私はWebアプリケーションの脆弱性の検索とは何かを知りたくて、友人にそれがどのように機能するかを示すように依頼しました。私たちは被害者として、当社の企業ソーシャルネットワーク（DaOfficeが開発）を選びました。
新世代のイントラネット。ソーシャル。モバイル。関与する。
調査を開始し、XSS、CSRFなどに対する保護の欠如に関連する重大でない脆弱性を発見しました。しかし、記事はそれらについてはまったく取り上げられていません。そして、あなたが社内にソーシャルネットワークを持っている場合は、それらに慣れることができます。



ユーザーの自己削除


ソーシャルネットワークにログインすると、ブラウザーとサーバーの間で交換される要求の調査が始まりました。ページのコードの1つで、「SelfDelete」という名前のページに言及していることに気づきました（完全なアドレスhttps：// <your social network site> / net_home / PrivateRoom / SelfDelete）。インターフェイスのリンク/ボタンはどこにも見つかりませんでした。結局私はクリックしただけです。その後、私のネットワークはソーシャルネットワークから削除されました。実際には、ブラウザに表示されたCookieは、ユーザーからの確認なしにアカウントを削除するのに十分でした-CSRFはGETリクエストを介して操作されます。

重要度の評価


削除については、結局のところ、ユーザーがアカウントにログインしていて、彼のCookieが腐っていないことで、ユーザーにブラウザからリンクをたどるように強制するだけで十分でした。

Common Vulnerability Scoring Systemによると、脆弱性は次のように評価できます。







最初のアピール


残念ながら、開発者のWebサイトの連絡先からは電話番号のみが示されています。これは、ほのめかしており、意味のあるものは何もありません。そのため、有名な社内連絡先を通じて連絡を取り、拒否リクエストを受け取りました。

第二の魅力


2018年8月上旬に、従業員は「奇跡的に」このリンクの存在を知り、積極的に共有し始めました。ユーザーはソーシャルネットワークで大規模な再登録を開始しました。







これがその様子です。その結果、ライセンスがないためにユーザーに同僚を削除し、無料のライセンスが表示されたときに登録することを開発者に通知し、開発者はそれを排除し、4以降月の脆弱性が修正されました。

一部のユーザーがグループ、コンテンツ、履歴の管理者権限を失っているのは残念ですが、ユーザーのアドレスに誇らしげに誇らしげな数字があります：https：// <your social network site> / net_home / People / <user name> 2
2019年には、MS Yammerのこのソーシャルネットワークを置き換えるという私の提案を誰も聞いていませんが、すべては同じです。www.cvedetails.comで必要な権限を持つアカウントを持っている人がいる場合は、この記事へのリンクを記載した脆弱性を公開することをお勧めします。
