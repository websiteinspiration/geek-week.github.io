Windows Terminal Preview v0.9

El lanzamiento de la versión 0.9 de Windows Terminal ha tenido lugar. Esta es la última versión de la Terminal, que incluirá nuevas características antes del lanzamiento de v1. Puede descargar Windows Terminal desde Microsoft Store o desde la página de lanzamiento en GitHub . ¡Echemos un vistazo más de cerca a los detalles de la actualización!



Argumentos de línea de comando


El alias wt ahora admite argumentos de línea de comandos. ¡Ahora puede iniciar la Terminal con nuevas pestañas y paneles, divididos a su gusto, con perfiles que le gusten, comenzando con los directorios que le gusten! ¡Las posibilidades son infinitas! Aquí hay algunos ejemplos:

wt -d.
Abre una terminal con un perfil predeterminado en el directorio de trabajo actual.

wt -d. new-tab -d C: \ pwsh.exe
Abre una Terminal con dos pestañas. En el primero, el perfil predeterminado, a partir del directorio de trabajo actual. El segundo es el perfil predeterminado con pwsh.exe como "línea de comando" (en lugar del perfil predeterminado "línea de comando") que comienza en el directorio C: \.

wt -p "Windows PowerShell" -d.; panel dividido -V wsl.exe
Abre una Terminal con dos paneles separados verticalmente. El perfil con el nombre "Windows Terminal" se inicia en el panel superior, y el perfil predeterminado se usa en el panel inferior, usando wsl.exe como la "línea de comando" (en lugar de la "línea de comando" del perfil predeterminado).

wt -d C: \ Users \ cinnamon \ GitHub \ WindowsTerminal; split-pane -p "Símbolo del sistema"; panel dividido -p "Ubuntu" -d \\ wsl $ \ Ubuntu \ home \ cinnak -H
Ver más abajo.



Si desea saber todo lo que puede hacer con nuestros nuevos argumentos de línea de comandos, consulte la documentación completa aquí .

Detección automática de PowerShell


Si eres un gran fanático de PowerShell Core , tenemos buenas noticias para ti. Windows Terminal ahora detecta cualquier versión de PowerShell y crea automáticamente un nuevo perfil para usted. La versión de PowerShell, que, en nuestra opinión, es la mejor (desde el número de versión más grande, hasta la versión de GA, hasta la versión con el mejor paquete), se llamará "PowerShell" y ocupará la ranura original de PowerShell Core en la lista desplegable.



Confirme el cierre de todas las pestañas


¿Eres el que no quiere que se te pida que cierre todas las pestañas cada vez? Si respondió que sí, entonces esta nueva característica es definitivamente para usted. Se ha creado un nuevo parámetro global que le permite ocultar siempre el cuadro de diálogo de confirmación "Cerrar todas las pestañas". Para hacer esto, configure el parámetro "confirmCloseAllTabs" en falso en la parte superior del archivo profiles.json, ¡y nunca volverá a ver esta ventana emergente! Gracias @ rstat1 por la contribución de este nuevo parámetro.
