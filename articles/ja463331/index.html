仕事の始まり


新しいApolloプロジェクトを作成する場合は、Apolloクライアントインスタンスを構成した後で、次のパッケージをインストールすることをお勧めします。

npm install @apollo/react-hooks

すでにApolloを使用するプロジェクトがある場合、フックに切り替えるにはいくつかの方法があることを意味します。特定の方法の選択は、移行をどの程度正確に行いたいかによって異なります。これに関する詳細はここにあります。

なぜフックは未来なのですか？


Apolloクライアントは、より高次のコンポーネントとレンダリングプロップに基づくAPIを引き続きサポートします。これらのAPIは、しばらくの間システムに存在します。将来的には、フックがApolloクライアントを使用してデータをロードするための最良のメカニズムになると考えています。すでにApolloを使用しているユーザーは、今すぐフックに切り替える必要はありませんが、新しいコンポーネントにはフックを使用する必要があります。この推奨には、これから検討するいくつかの理由があります。

hookフックを使用すると、データの処理に使用するコードが少なくて済みます


フックは、データの操作に使用される定型コードの量を減らします。これにより、コンポーネントのサイズが縮小され、そのようなコンポーネントが理解しやすくなります。その結果、開発者は高次コンポーネントの配置方法を詳しく調べたり、レンダープロップの複雑なロジックを分析したりする必要がなくなります。データをロードするには、単一の関数を呼び出すだけで十分ですuseQuery。

const LAST_LAUNCH = gql`
  query lastLaunch {
    launch {
      id
      timestamp
    }
  }
`;

export function LastLaunch() {
  const { loading, data } = useQuery(LAST_LAUNCH);
  return (
    <div>
      <h1>Last Launch</h1>
      {loading ? <p>Loading</p> : <p>Timestamp: {data.launch.timestamp}</p>}
    </div>
  );
}

▍複数の変異


1つのコンポーネントで複数のミューテーションを実行する場合、高次コンポーネントまたはレンダプロップメカニズムを使用すると、コードが理解しにくくなる可能性があります。 APIレンダープロップ（相互に埋め込まれたコンポーネントで構成される構造の構築）を使用すると、Mutation構造化されたコードとその明確な階層に誤った感覚が与えられます。新しいフックをuseMutation使用すると、この問題を完全に回避できます。実際には、その使用は単純な関数呼び出しに限定されています。次の例は、いくつかのミューテーションとクエリが相互にどのように相互作用するかを示しています。これはすべて同じコンポーネント内で行われます。

function Message() {
  const [saveMessage, { loading }] = useMutation(SAVE_MESSAGE);
  const [deleteMessage] = useMutation(DELETE_MESSAGE);
  const { data } = useQuery(GET_MESSAGE);

  return (
    <div>
      <p>
        {loading
          ? 'Loading ...'
          : `Message: ${data && data.message ? data.message.content : ''}`}
      </p>
      <p>
        <button onClick={() => saveMessage()}>Save</button>
        <button onClick={() => deleteMessage()}>Delete</button>
      </p>
    </div>
  );
}

ここではフックを使用していますuseMutation。ここで複数のミューテーションが動作しているのを確認できます。このサンプルアプリケーションには、レンダープロップを使用して作成された同様のコンポーネントも含まれています。これにより、このコンポーネントをフックを使用して作成されたコンポーネントと比較する機会が与えられます。

Type TypeScriptサポートの改善


TypeScriptの大ファンであることは明らかです。新しいフックの機能は、Apollo CLIによって生成される自動タイプ定義とうまく連動します。これにより、Reactコンポーネントのタイプセーフなコードの記述が大幅に容易になります。フックuseQueryとTypeScript を使用したデータの読み込みは次のようになります。

import { RocketData, RocketVars } from './types';

const GET_ROCKET_INVENTORY = gql`
  query getRocketInventory($year: Int!) {
    rocketInventory(year: $year) {
      id
      year
    }
  }
`;

export function RocketInventoryList() {
  const { loading, data } = useQuery<RocketData, RocketVars>(
    GET_ROCKET_INVENTORY,
    { variables: { year: 2019 } }
  );
  return (/* ... show data ... */);
}

ApolloおよびTypeScriptフックを使用すると、厳密に型指定されたReactコンポーネントを簡単に開発できます。

queryクエリ実行の遅延


フックuseQueryは、関数が呼び出されたときに要求を実行します。しかし、これは正確には常に必要ではないシステムの動作です。たとえば、ユーザーにプロンプ​​トを表示する検索クエリを入力するフィールドを想像してみてください。ユーザーが入力した内容を受け入れる準備ができているようなフィールドを持つコンポーネントを表示する必要がある場合があります。しかし同時に、ユーザーがフィールドに何かを入力し始めるまで、検索サーバーへのクエリは遅延します。このシナリオを実装useLazyQueryするにexecuteは、2番目の位置に関数を持つタプルを返すフックを使用できます。

const [execute, { loading, data }] = useLazyQuery(GET_INVENTORY);

関数を呼び出すまでリクエストは実行されませんexecute。この時点で、コンポーネントは再レンダリングされ、を使用して実装された通常のクエリ実行スキームが適用されuseQueryます。
