Mentransfer file menggunakan pipa dan hal-hal kecil lainnya di Delphi

Entah bagaimana mereka membuat saya tugas menulis beberapa unit dalam Delphi yang menyediakan akses ke fitur Windows berikut:

  1. Memori bersama .
  2. Pipa .

Tidak mungkin untuk menggunakan komponen pihak ketiga. Memori bersama seharusnya digunakan untuk menyortir file teks besar - beberapa puluh atau bahkan ratusan megabyte tempat TStringList jatuh dari EOutOfMemory . Pipa - untuk mentransfer file dari satu aplikasi ke aplikasi lainnya.

Untuk menulis ulang kode contoh dari dokumentasi Microsoft dari C ke Delphi bahkan tidak sampai setengahnya, tetapi menjadi 10 persen. Hal utama adalah untuk mengikat semuanya, menambahkan fungsi tambahan jika diperlukan dan menjadikannya berfungsi, dan tidak hanya ada sebagai kode sumber. Akibatnya, monster jelek seperti itu lahir sehingga itu bukan sesuatu untuk ditunjukkan kepada siapa pun, tetapi hanya menatapnya itu menakutkan. Namun, mungkin seseorang akan menemukan sesuatu yang berguna untuk dirinya sendiri di perutnya (bagaimana jika?), Jadi saya memutuskan untuk mempostingnya di sini. Untuk celaan umum. Kode sumber lengkap untuk kekejian ini tersedia di GitHub . Teks artikel akan memberikan nama file di mana Anda dapat melihat satu atau beberapa bagian dari implementasi.

Tugas pertama adalah menyortir file teks - diputuskan seperti ini:

  1. Buka file ( CreateFile ).
  2. Kami menggunakan CreateFileMapping dan MapViewOfFile (common \ MemoryMappedFileUnit.pas).
  3. Kami menentukan penyandian file. Di sini, saya membatasi diri saya, pada kenyataannya, hanya untuk dua pengkodean: UTF-16 dan ACP - pengodean byte tunggal dengan halaman kode default. Di Windows berbahasa Rusia, Windows-1251 biasanya digunakan .
  4. Kami memecah teks menjadi beberapa baris. Yaitu, kami mencari penanda garis akhir dan, karenanya, kami menganggap teks dari penanda ke penanda sebagai satu baris.
  5. Urutkan array string yang dihasilkan (SortingTest \ MergeSorterUnit.pas).
  6. Kami menulis array string yang diurutkan kembali ke file.
  7. KEUNTUNGAN!

Pelanggan pasti ingin membuat penyortiran multithreaded, jadi Gabung Sortir diambil sebagai algoritma penyortiran . (Terima kasih kepada guru yang menciptakan implementasi kerja dan mempostingnya di Stackoverflow .)

Tugas kedua - mentransfer file dari satu aplikasi ke aplikasi lain - ternyata menjadi jauh lebih sulit. Mendapatkan implementasi pipa pada Delphi berdasarkan contoh Microsoft adalah satu hal, tetapi membuat transfer file yang berfungsi adalah hal lain. Sebagai program server, Microsoft menawarkan beberapa opsi:

  1. Server Pipa Multithreaded .
  2. Pipa Server Menggunakan Dinamakan Overlapped I / O .
  3. Server Pipa Bernama Menggunakan Rutin Penyelesaian .

Ternyata sebagai hasil dari dua minggu siksaan, implementasi mekanisme transfer file berdasarkan versi pertama dari server ( Multithreaded Pipe Server ) terlalu rumit. Lebih masuk akal untuk menggunakan opsi kedua ( Named Pipe Server Using Overlapped I / O ), dengan itu implementasinya jauh lebih sederhana (meskipun pelanggan benar-benar menginginkan versi pertama server karena beberapa alasan ). Tetapi saya masih harus menyiksa diri sendiri untuk membawa semua ini ke dalam pikiran. Akibatnya, pemancar file berbasis pipa lahir: TCommunicationEndpointBase (CommunicationTest \ SimpleCommunicationUnit.pas).
