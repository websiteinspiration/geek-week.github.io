OpenCVとディープラーニングでDobbleをプレイするようにコンピューターに教えた方法

みなさんこんにちは！あなたはおそらく私たちが新しいコース「コンピュータビジョン」を立ち上げたことをすでにご存じでしょう。クラスはこれから数日で始まります。クラスの開始を予想して、CVの世界に没頭するための興味深い翻訳をもう1つ用意しました。



趣味はボードゲームで、畳み込みニューラルネットワークに少し慣れているので、カードゲームで人を倒すことができるアプリケーションを作成することにしました。私は自分のデータセットを使用してモデルをゼロから構築し、小さなデータセットでどのように機能するかを確認したいと思いました。シンプルなDobbleゲーム（Spot itとも呼ばれます）から始めることにしました。

Dobbleが何であるかわからない場合は、ゲームのルールを簡単に思い出します。Dobbleは、プレーヤーが2枚のカードに同時に描かれた絵を見つけようとする単純なパターン認識ゲームです。オリジナルのDobbleゲームの各カードには8つの異なるキャラクターが含まれており、異なるカードではそれらのサイズが異なります。2枚のカードには共通のシンボルが1つしかありません。最初にシンボルを見つけたら、カードを手に取ります。55枚のカードのデッキが終了すると、最も多くのカードを持つものが勝ちます。


自分で試してみてください：これらの2つのカードに共通する記号は何ですか？

どこから始めますか？


データ分析タスクを解く最初のステップは、データを収集することです。私は電話で各カードの写真を6枚撮りました。合計で330枚の写真が見つかりました。そのうちの4つを以下に示します。あなたは尋ねるかもしれません、これは良いたたみ込みニューラルネットワークを作成するのに十分ですか？これに戻ります！



画像処理


さて、私たちが持っているデータ、次は何ですか？おそらく、成功への道の最も重要な部分は画像処理です。各画像から文字を取得する必要があります。ここでいくつかの困難が待っています。上の写真では、一部の文字が他の文字よりも区別しづらいことがわかります。雪だるまとゴースト（3番目の写真）と針（4番目の写真）の明るい色、しみ（2番目の写真）と感嘆符（4番目の写真）はいくつかの部分で構成されています。明るい文字を処理するために、コントラストを追加します。その後、画像のサイズを変更して保存します。

コントラストを追加


コントラストを追加するために、Lab色空間を使用します。Lは明度、aは緑からマゼンタの範囲の色成分、bは青から黄色の範囲の色成分です。OpenCVを使用してこれらのコンポーネントを簡単に抽出できます。

import cv2
import imutils
imgname = 'picture1'
image = cv2.imread(f’{imgname}.jpg’)
lab = cv2.cvtColor(image, cv2.COLOR_BGR2LAB)
l, a, b = cv2.split(lab)


左から右へ：元の画像、明度コンポーネント、コンポーネントaおよびコンポーネントb

次に、明度コンポーネントにコントラストを追加し、すべてのコンポーネントを組み合わせて、通常の画像に変換します。

clahe = cv2.createCLAHE(clipLimit=3.0, tileGridSize=(8,8))
cl = clahe.apply(l)
limg = cv2.merge((cl,a,b))
final = cv2.cvtColor(limg, cv2.COLOR_LAB2BGR)


左から右へ：元の画像、明度コンポーネント、高コントラストの画像、RGBに変換された画像

カードと文字の認識


画像が処理されたので、画像内のカードを検出できます。OpenCVを使用して、外部輪郭を探しています。次に、画像をハーフトーンに変換し、しきい値（この場合は190）を選択して白黒画像を作成し、パスを検索します。コード：

image = cv2.imread(f’{imgname}processed.jpg’)
gray = cv2.cvtColor(image, cv2.COLOR_RGB2GRAY)
thresh = cv2.threshold(gray, 190, 255, cv2.THRESH_BINARY)[1]
# find contours
cnts = cv2.findContours(thresh.copy(), cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
cnts = imutils.grab_contours(cnts)
output = image.copy()
# draw contours on image
for c in cnts:
    cv2.drawContours(output, [c], -1, (255, 0, 0), 3)


しきい値と外部輪郭の選択を使用してハーフトーンに変換された処理済み画像

外部輪郭を領域で並べ替えると、面積が最大の輪郭が見つかります-これが私たちのカードです。文字を抽出するには、白い背景を作成します。

# sort by area, grab the biggest one
cnts = sorted(cnts, key=cv2.contourArea, reverse=True)[0]
# create mask with the biggest contour
mask = np.zeros(gray.shape,np.uint8)
mask = cv2.drawContours(mask, [cnts], -1, 255, cv2.FILLED)
# card in foreground
fg_masked = cv2.bitwise_and(image, image, mask=mask)
# white background (use inverted mask)
mask = cv2.bitwise_not(mask)
bk = np.full(image.shape, 255, dtype=np.uint8)
bk_masked = cv2.bitwise_and(bk, bk, mask=mask)
# combine back- and foreground
final = cv2.bitwise_or(fg_masked, bk_masked)


マスク、背景、前景画像、最終画像

文字認識の時間です。結果の画像を使用して、外部輪郭を再度検出できます。これらの輪郭はシンボルになります。各シンボルの周りに正方形を作成すると、この領域を抽出できます。ここでコードは少し長くなります：

# just like before (with detecting the card)
gray = cv2.cvtColor(final, cv2.COLOR_RGB2GRAY)
thresh = cv2.threshold(gray, 195, 255, cv2.THRESH_BINARY)[1]
thresh = cv2.bitwise_not(thresh)
cnts = cv2.findContours(thresh.copy(), cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
cnts = imutils.grab_contours(cnts)
cnts = sorted(cnts, key=cv2.contourArea, reverse=True)[:10]
# handle each contour
i = 0
for c in cnts:
    if cv2.contourArea(c) > 1000:
        # draw mask, keep contour
        mask = np.zeros(gray.shape, np.uint8)
        mask = cv2.drawContours(mask, [c], -1, 255, cv2.FILLED)
        # white background
        fg_masked = cv2.bitwise_and(image, image, mask=mask)
        mask = cv2.bitwise_not(mask)
        bk = np.full(image.shape, 255, dtype=np.uint8)
        bk_masked = cv2.bitwise_and(bk, bk, mask=mask)
        finalcont = cv2.bitwise_or(fg_masked, bk_masked)
        # bounding rectangle around contour
        output = finalcont.copy()
        x,y,w,h = cv2.boundingRect(c)
        # squares io rectangles
        if w < h:
            x += int((w-h)/2)
            w = h
        else:
            y += int((h-w)/2)
            h = w
        # take out the square with the symbol
        roi = finalcont[y:y+h, x:x+w]
        roi = cv2.resize(roi, (400,400))
        # save the symbol
        cv2.imwrite(f"{imgname}_icon{i}.jpg", roi)
        i += 1

モデルのアーキテクチャ


1つのラベルを持つマルチクラス分類のタスクがあります。文字ごとに1つのラベルが必要です。そのため、57個のノードを含む出力ソフトマックスレイヤーをアクティブ化する関数と、損失関数としてカテゴリカルクロスエントロピーが必要です。

最終的なモデルのアーキテクチャは次のとおりです。

# imports
from keras import layers
from keras import models
from keras import optimizers
from keras.preprocessing.image import ImageDataGenerator
import matplotlib.pyplot as plt
# layers, activation layer with 57 nodes (one for every symbol)
model = models.Sequential()
model.add(layers.Conv2D(32, (3, 3), activation='relu', input_shape=(400, 400, 3)))
model.add(layers.MaxPooling2D((2, 2)))  
model.add(layers.Conv2D(64, (3, 3), activation='relu'))
model.add(layers.MaxPooling2D((2, 2)))
model.add(layers.Conv2D(128, (3, 3), activation='relu'))
model.add(layers.MaxPooling2D((2, 2)))
model.add(layers.Conv2D(256, (3, 3), activation='relu'))
model.add(layers.MaxPooling2D((2, 2)))
model.add(layers.Conv2D(256, (3, 3), activation='relu'))
model.add(layers.MaxPooling2D((2, 2)))
model.add(layers.Conv2D(128, (3, 3), activation='relu'))
model.add(layers.Flatten())
model.add(layers.Dropout(0.5)) 
model.add(layers.Dense(512, activation='relu'))
model.add(layers.Dense(57, activation='softmax'))
model.compile(loss='categorical_crossentropy',       optimizer=optimizers.RMSprop(lr=1e-4), metrics=['acc'])

データ増強


パフォーマンスを向上させるために、データ拡張を使用しました。データ拡張は、入力データの量と種類を増やすプロセスです。これは、既存の画像を回転、シフト、拡大縮小、切り取り、反転することで実行できます。Kerasはデータを簡単に増強できます：

# specify the directories
train_dir = 'symbols/train'
validation_dir = 'symbols/validation'
test_dir = 'symbols/test'
# data augmentation with ImageDataGenerator from Keras (only train)
train_datagen = ImageDataGenerator(rescale=1./255, rotation_range=40, width_shift_range=0.1, height_shift_range=0.1, shear_range=0.1, zoom_range=0.1, horizontal_flip=True, vertical_flip=True)
test_datagen = ImageDataGenerator(rescale=1./255)
train_generator = train_datagen.flow_from_directory(train_dir, target_size=(400,400), batch_size=20, class_mode='categorical')
validation_generator = test_datagen.flow_from_directory(validation_dir, target_size=(400,400), batch_size=20, class_mode='categorical')

2枚のカードの共通シンボルの認識


これで、2枚のカードで共通のシンボルを探し始めることができます。2枚の写真を使用し、各画像を個別に予測し、セットの共通部分を使用して、両方のカードにあるシンボルを見つけます。3つの作業オプションがあります。

  • 予測中に問題が発生しました：一般的な文字が見つかりませんでした。
  • 交差点にシンボルが1つあります（予測はtrueまたはfalseです）。
  • 交差点に複数のキャラクターがいます。この場合、最も高い確率（両方の予測の平均）を持つシンボルを選択します。

結論


それは完璧なモデルではありませんか？残念だけど違う。カードの新しい写真を撮り、それらに予測のためのモデルを与えたとき、雪だるまにいくつかの問題がありました。時々、彼は目やシマウマを雪だるまとして認識しました！その結果、時々結果がおかしいことがあります。


まあ、ここの雪だるまはどこですか？

このモデルは人間より優れていますか？必要なものに応じて：人々は完全に認識しますが、モデルはそれをより速く行います！コンピューターが処理している時間に気づきました。55枚のカードのデッキを与え、2枚のカードの各組み合わせに共通のシンボルを取得する必要がありました。合計で、これらは1485の組み合わせです。コンピュータはそれを140秒未満で行いました。彼はいくつかのミスを犯しましたが、スピードに関しては間違いなく誰にも負けません！



100％機能するモデルを作成するのは難しいとは思いません。これは、転送トレーニングを通じて達成できます。モデルの機能を理解するために、テスト画像のレイヤーを視覚化できます。次回もできます！
