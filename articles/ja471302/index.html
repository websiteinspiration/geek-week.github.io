中国でのユーザー認証とソーシャルクレジット

中国の法律によると、インターネットポータルまたはサービスに登録するには、ユーザーを実在の人物として識別する必要があります。これを行う方法、これに必要なもの、およびそれがもたらす結果について話し、この記事で説明します。

2004年5月18日、これを定義するガイドラインが最初に発行されました。各サイトの各ユーザーは実在の人物である必要があります。当時、技術開発のレベル上、これは事実上不可能であったため、最初のチェック段階では、通常、IDカード番号とユーザー名を入力する必要がありました。


もちろん、これは十分ではありませんでした。中国語（第2世代のIDカード）（参照による彼らの歴史の詳細な話）は18桁の番号で、最初の6桁は地域コード、次の8は生年月日、次の3は性別とシリアル番号、最後は検証です。最近、数字が不足しているため、一部の数字は最後に文字が付いた19桁になっていますが、本質的に根本的な変更はありません。インターネット上に数多くの数字生成サイトが登場し、保護されていませんでした。

2008年、産業情報省の創設後、すべてがはるかに迅速になり、最近まで2014年から2018年まで、次の種類のチェックが中国に存在しました（そして現在存在しています）。これらは、公安部や中国人民銀行などと共同で作成されたものであり、基盤を提供しますが、最終的な開発者と直接仲介することはありません。
Alibaba、Tencent、NetEase、Sohuなどの大規模なIT企業や数十に上る小規模なIT企業は、ユーザー検証APIを販売するための独自の市場を持っています。最初で最も基本的なのは、IDカードの番号と名前の照合です。



したがって、サービスのパッケージを購入することにより、開発者は、ユーザーが入力したデータをサーバーに転送でき、サーバーは「0」または「1」と答えます。IDカード番号が指定した名前と一致するかどうか。

{
"code": "10000",
"message": "成功",
"data": {
"result": "1"
},
"seqNo": "4XU29Z4D1704061618"
}

これも明らかに不十分です。カード番号と人物の名前を知っていても、ユーザーとドキュメントの所有者が同じ人物であるとは限りません。テクノロジーは進化しており、電話番号によるチェックもあります。つまり、サービスのユーザーが、彼が示す番号の所有者であるかどうか。したがって、これを確認SMSの送信と一緒に使用するのが最も論理的です。



ここで答えははるかに広範囲です-「0」または「1」の答えと同時にサービスはオペレーター、登録の地域についての情報を私たちに与えます：

そして、同じ基準に従って銀行カード番号による検証。すべてがまったく同じですが、電話番号の代わりに、銀行カード番号を入力する必要があります。サービスは偶然に答えを出します。当然、カードからの確認支払いと一緒にこのようなチェックを実行する方がより論理的です-登録時に指定されたカードと支払いが確認されたカードが一致する場合、ユーザーとカード所有者が同じ人物である可能性が高くなります。

UnionPayカードとVisa \ Mastercardの間には基本的な違いがあるため、カード番号を入力してもカードからの資金の盗難につながることは決してないため、このような小切手もユーザーにとって非常に安全です。

しかし、これらすべてのチェックはすぐに時代遅れになり、ブラシ脸验证の外観のため、現在は事実上請求されていません。



現在、ほとんどすべてのアプリケーションがこの特定の方法を使用しています。ここではいくつかのマーカーが使用されています。まず、記事の冒頭で説明したように、IDカードの番号と名前がチェックされます。その後、ユーザーは顔をスキャンし、画面に表示される数字を読み取ります。サービスはアプリケーションに、「1」または「0」ではなく、1から100までの数字を示す回答を元のパーセンテージで提供します。ガイドでは、85以上の数値で合格したチェックを検討することをお勧めしますが、開発者が自分の信頼度を確立することに煩わされることはありません。

ソーシャルクレジットについては、誰も（評判の良いニュース出版物であっても）何も知らないので、少し理論を述べる義務があります。
ソーシャルクレジットは、個人の信用体系です。そして、それがすべてに影響を与えると言うのは愚かでしょう。例として、最近ICBCクレジットカードを受け取りました。これは、金融関係の当事者としての私の信用に影響を与えましたか？もちろん。これは他に何か影響しましたか？もちろん違います。

したがって、ここ-行政上の不正行為（上記の例）により、知人は失信（信頼の喪失）ステータスを取得しました。しかし、この特定の場合にのみ-罰は常に不正行為に対応するべきです。この場合-「行動する方法がわからない-バスに乗る」の原則に基づく快適な車両による旅行の禁止。しかし、これは彼が私の10倍以上持っている銀行での彼の信用限度に影響を与えませんでした（そして影響を与えることはできませんでした）。
このシステムの次の段階は、被执行（生産中）です。これは次の段階であり、行政の不正行為のためにそこに落ちることはありません。これらは犯罪者、詐欺師などです。手続きの終了と評決の後、裁判所は犯人をリストに載せ、彼は自信を失ったものの新しい品質の最初の段階に移ります。

違いは、軽度の不正行為については、人が行政上のプロトコルに基づいて一時的にそこに到着するため、裁判所はここに参加しないことです。つまり、他の行政事務と同様に、決定は「200元の罰金を科し、XX日間信頼を失った人々のリストに次の制限を加えることです：A、B、C」と述べています。そして、任期の満了時に、レコードは自動的に削除され、それはそれ以上の履歴に影響を与えず、その人は何の制限も経験しません。

裁判所命令によるリストの誰もが完全に異なる規制に従います。この文では、罰の種類と期間に加えて、信頼を失った人のリストに含める期間が示され、その後、その人を確認してリストから削除することができます。

つまり、世界的な違いは、行政犯罪では人は自動的にリストから削除され、それ以上の結果はありません、そして刑事事件では、手動でのみ、適切な裁判所の決定後にのみです。
