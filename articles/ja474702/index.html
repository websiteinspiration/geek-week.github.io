関数型プログラミングの概要


関数型プログラミング（FP）は、一連の関数を記述してコードを編成する方法です。

EcmaScriptはマルチパラダイムプログラミング言語であり、とりわけ関数型パラダイムを実装しています。つまり、ESの関数はデータであり、関数に渡したり、関数から返したり、関数自体を受け取ったりすることができます。それら。 ESの関数はファーストクラスの関数です。

以下の定義が続きます。

関数引数（funarg） -値が関数である引数。

高階関数（FWP、高階関数、hof）は、関数を引数として受け入れる関数です。

関数値関数関数を返す関数です。

これらすべてのタイプの関数は、条件付きで最初のクラスの関数に結合されます。上記の定義から以下のように、ESではすべての関数が最初のクラスのオブジェクトです。

純粋な関数-関数型プログラミングに最適


純粋関数（PF） -常に予測結果を返します。
PFプロパティ：

  • PF実行の結果は、渡された引数とPFを実装するアルゴリズムにのみ依存します
  • グローバル値を使用しないでください
  • 外部の値や渡された引数を変更しないでください
  • ファイル、データベース、または他の場所にデータを書き込まないでください

純関数の例：

const add = (x,y) => x+y;

関数の不純性の良い例は次のとおりです。 

var first;
var second;

function testFn() {
  var a = 10;
  
  first = function() {
    return ++a;
  }

  second = function() {
   return --a;
  }

  a = 2;
  first();//3
}

testFn();

first();//4
second();//3

この例のテストを作成するのがどれほど複雑で、純粋な関数をどれだけ単純化できるか想像してみてください。

不純な関数は、コードの保守、理解、およびテストを複雑にする、時間とともに変化する外部状態によって特徴付けられます。

それどころか、純粋な関数は常に読み取り可能でテスト可能であり、計算の並列化を単純化し、再利用が簡単です。

純粋な関数の例では、ES6構文に切り替えたことに気づいたと思います。これは意図的に行われました。この関数構文は「矢印関数」と呼ばれますが、実際には、かなり前に発明された数学的抽象化の実装です。それについてさらに。

ラムダ-関数


これは、この矢印形式の書き込みが数学および他のいくつかのプログラミング言語で呼ばれているものです。関数型プログラミングはマットと非常に密接に関連しています。分析なので、驚かないでください。

ラムダ計算という用語は、1930年代にアロンツォ教会によって導入されました。本質的に、ラムダ計算は数学の方程式を記述する正式な形式に過ぎません。詳細はこちら。

ESでは、ラムダ関数がクロージャーを実装することがよくあります。

const add = x => y => x + y;

短くて簡潔。add関数は、引数xを取り、クロージャーに格納して関数を返すラムダです。

このコードと比較してください：

funtion add(x) {
  return function (y) {
   return x + y;
  }
}

明らかに、最初のオプションの方が良く見えます。

免疫


不変（immutable、immunity）は、作成後に状態を変更できないオブジェクトです。そのようなオブジェクトの変更の結果は常に新しいオブジェクトになりますが、古いオブジェクトは変更されません。

不変性は、関数型プログラミングの根幹です。

例を考えてみましょう：

const impureAddProp = (key, value, object) => {
  object[key] = value;//  
};
const User= {
  name: 'Alex'
};
impureAddProp ('isAdmin', true, User);

ご覧のとおり、この例では、Userオブジェクトにプロパティを追加して変更しました。これで、Userオブジェクトは、impureAddProp関数とそれを変更する他の関数の一種の「共有状態」になります。このアプローチは、次の理由でテストが困難です。共有状態と相互作用する関数を変更する場合、他の関数で起こり得るエラーを常に念頭に置いてください。

関数型プログラミングの観点からは、次のように正しいでしょう。

const pureAddProp = (key, value, object) => ({
  ...object,
  [key]: value
});
const User= {
  name: 'Alex'
};
const Admin= pureAddProp ('isAdmin', true, User);

したがって、Userオブジェクトは変更されません。常に安全なデータのコピーを変更しています。
