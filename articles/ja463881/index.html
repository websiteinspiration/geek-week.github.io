人質を撮った写真。WiFi（PTP / IPプロトコル）を介したデジタルカメラの最初のハッキング



現在、ネットワーク接続を備えたデバイスは潜在的なハッキングから保護されていません。カメラも。

DEF CON 2019ハッカー会議で、チェック・ポイント・ソフトウェア・テクノロジーズのEyal Itkinが、PTP（Picture Transfer Protocol）プロトコルの世界初のライブエクスプロイトを明らかにしました。最近のDSLRの多くは、WiFi経由のファイル転送をサポートしています。この場合、周囲のカメラが自動的に接続される悪意のあるWiFiアクセスポイントが記述されています。 このエクスプロイトは、カメラがPTP / USB経由でコンピューターに物理的に接続されている場合にも機能します。どちらの場合も、攻撃者はカメラとの接続を確立し、そこに実行可能コードをロードできます。たとえば、カード上のすべての画像を暗号化します。



たとえば、Check Point Eyal Itkinは、ランサムウェアマルウェアのデモバージョンを、フラッシュカード上のファイルを暗号化するCanon EOS 80Dカメラにアップロードしました。Itkinは、これが非常に現実的な収益化モデルであることを示唆しています。重要なライフイベントの写真はしばしばそこに保存されるため、人々はそれらを取り戻すためにお金を払いたくなるでしょう。



ランサムウェアに加えて、他のエクスプロイトの悪用方法が理論的に可能です。

  • 完全な無能力化
  • スパイ活動のためのカメラの使用（彼女はすでに写真を撮っています）

彼らがこのモデルを選択したのは、このメーカーがグローバルデジタルカメラ市場の50％以上を管理しているためです。さらに、キヤノンのカメラを中心に広範なモッディング運動が発達しました。活動家は、機能強化された無料のファームウェアをリリースするためにファームウェアとハ​​ードウェアをリバースエンジニアリングします。MagicLatternおよびCanon Hack Development Kit（CHDK）を参照してください。

ただし、他のカメラはすべてPTPを標準的な方法でサポートしているため、同様の方法が他のカメラにも適用されます。そして、このプロトコルには、ハッキングに有利な重要な特性があります。

  • 認証および暗号化なし（プロトコルはもともとUSBの下で作成されたため、PTP / IPの存在は想定されていませんでした）;
  • 豊富な機能を備えた数十の複雑なチームのサポート。
  • WiFiサポート。

ハッキングに関連して、キヤノンはパッチとセキュリティ警告を発行しました。ユーザーに予防策を講じ、不審なWiFiに接続せず、ネットワーク機能が使用されていない場合は無効にすることをお勧めします。

攻撃の技術的詳細


ハッカーが最初に行ったのは、リバースエンジニアリングとカメラのファームウェアの分析でした。 AESによって暗号化されていることが判明しましたが、キーはありません。しかし、Magic Latternコミュニティはすでにこの問題を解決しています。彼らはROMダンパーを使用し、キーを取得しました。



ダンプはIDAにロードされ、Magic Latternには、この関数またはその関数の呼び出し元のアドレスが文書化された優れたデータベースがあります。キヤノンのカメラは、独自のDryOSリアルタイムOSを使用しています。彼女のおかげで、デバイスはわずか3秒で再起動できます。

以下は、DryOSのオペコードを含むPTPコマンドのリストです。ファームウェアで必要なコードを見つけるには、各コマンドのリストに示されている対応する定数の検索を開始するだけで十分です。

PTPコマンドで動作するいくつかの関数の表面的な分析では、それらを実装するのはかなり難しく（1つの関数に148の一意のハンドラー）、バッファオーバーフローに明らかな脆弱性があることがわかりました。



Itkinは、4つのPTPコマンド（SendObjectInfo、NotifyBtStatus、BLERequest、SendHostInfo）を呼び出すときに4つのバッファオーバーフローの脆弱性を発見しました：脆弱性番号CVE-2019-5994、CVE-2019-5998、CVE-2019-5999、CVE-2019-6000）。興味深いことに、このカメラモデルはBluetoothをサポートしていませんが、Bluetooth処理に関連する2つの脆弱性があります。

現在、適切なエクスプロイトを収集するために残っていましたが、行われました。しかし、作者はそこで止まらず、より視覚的なデモンストレーションのために、フラッシュカード上のファイルを暗号化するランサムウェアプログラムを作成することにしました。独自の暗号化を実装するのではなく、Canonファームウェア自体に組み込まれたAES暗号化機能を使用するという考えでした。 Iskinは暗号化機能を検索する際に、暗号化キー自体も見つけました。

独自の実装のため、ファームウェアでキーが生成される方法を理解するのは困難でした。しかし、この理解は暗号関数を使用するために必要ではない、とItkinは言った。マルウェアの指揮下で、マップ上の写真を正常に暗号化し、正しい署名を計算します。

Itkinは、ユーザーの介入なしにカメラに悪意のあるファームウェアアップデートをインストールする方法も開発しました（CVE-2019-5995）。

DEF CONに関するプレゼンテーションで、著者はこれがPTPプロトコルの最初の悪用であり、「脆弱性を悪用せずにほとんどのDSLRをキャプチャする信頼できる方法」であると述べています。
