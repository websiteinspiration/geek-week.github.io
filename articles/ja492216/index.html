ビルド段階で.Netアプリケーションに値を保存する

ランタイムで使用するための一連の定数を渡すアプリケーションを構築するときに私は必要でした。たとえば、アセンブリ時に既知の文字列値をアプリケーションに「縫い込み」たいとします。

C ++の世界では、定義とコンパイラオプションを使用して、このようなことを非常に簡単に解決しました。しかし、.Netでは、defineにはtrue / false以外の値はありません。それらは識別されるかされないかのどちらかです。私が理解している限り、彼らの目標は最も単純な条件付きコンパイルです。

解決策を気にする人、猫へようこそ。

私を訪れた最初の考えは、ビルドの前にテンプレートに従って定数を含むファイルを生成することでしたが、テンプレートエンジンの大砲を使わずに実行したいと思います。

調べてみると、面白いことがわかりました。.Netには属性メカニズムがあります。これらの属性は、クラス、メソッド、フィールド、およびあらゆる種類の異なるエンティティに固執できます。アセンブリ全体にフックできることがわかりました。

プロジェクトファイル（.csproj）では、アセンブリ中にこれらの属性の値を設定できます。また、MSBuildでは、プロパティのメカニズムを通じて外部からパラメーターを渡すことができます。すべてが合うようです、あなたは試さなければなりません。

新しいコンソールアプリケーションを作成します。

% mkdir Example && cd Example
% dotnet new console

属性の定義を含むファイルExampleAttribute.csを作成します。

using System;

namespace Example
{

[AttributeUsage(AttributeTargets.Assembly)] //     
public class ExampleAttribute : Attribute
{
    public string Value { get; set; }

    public ExampleAttribute(string value)
    {
        Value = value;
    }
}
 
}

次に、Example.csprojファイルは次の形式に縮小されます。変更の本質が明確になるようにコメントを追加しました。

<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp3.1</TargetFramework>
    <RootNamespace>Example</RootNamespace>

    <ExampleValue>default</ExampleValue> <!--        -->
  </PropertyGroup>

  <!--          -->
  <ItemGroup>
    <AssemblyAttribute Include="Example.ExampleAttribute"> <!--    -->
      <_Parameter1>$(ExampleValue)</_Parameter1> <!--      -->
    </AssemblyAttribute>
  </ItemGroup>

</Project>

まあ、実際には、Project.csで実行時に値を取得する

using System;
using System.Reflection;

namespace Example
{
    class Program
    {
        static void Main(string[] args)
        {
            var assembly = Assembly.GetExecutingAssembly();
            var attr = (ExampleAttribute) assembly.GetCustomAttribute(typeof(ExampleAttribute));
            Console.WriteLine($"Assembly attribute value = '{attr.Value}'");
        }
    }
}

それで、私たちは私たちが持っているものを見るために組み立てて走ります。

% dotnet build .
% dotnet run --no-build .
Assembly attribute value = 'default'

そして今パラメーターを使って：

% dotnet build . /p:ExampleValue="NOT DEFAULT"
% dotnet run --no-build .
Assembly attribute value = 'NOT DEFAULT'

出来上がり、目標は達成されました。健康に使用します。
