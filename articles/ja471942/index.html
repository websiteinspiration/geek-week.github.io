要件に応じて：Kingston DC500RおよびDC500M SSDのプロフェッショナルテスト

企業のSSDの実際の例と専門的なテストを求めました。True SystemパートナーのKingston DC500RおよびDC500M SSDの詳細な概要を提供します。True Systemのエキスパートが実際のサーバーを組み立て、すべてのエンタープライズクラスのSSDが直面する絶対的に実際のタスクをエミュレートしました。彼らがしたことを読んでみましょう！



2019キングストンラインナップ


まず、少しドライな理論。すべてのキングストンSSDは、4つの大きなグループに分けることができます。同じドライブが一度に複数のファミリーに分類されるため、この分割は条件付きです。

  • システムビルダー向けSSD： 2.5インチSATA SSD、M.2およびmSATA Kingston UV500、および2つのNVMeストレージモデル-Kingston A1000およびKingston KC2000;
  • ユーザー向けSSD。前のグループと同じモデル、さらにSATA SSD Kingston A400。
  • 企業向けSSD：UV500およびKC2000;
  • 企業SSD。このレビューのヒーローになったDC500シリーズを駆動します。DC500ラインは、DC500R（1次読み取り値、0.5 DWPD）とDC500M（混合負荷、1.3 DWPD）に分かれています。

テスト方法


基礎となったのは、人気のあるSNIAソリッドステートストレージパフォーマンスパフォーマンステスト仕様v2.0.1テストスイートでしたが、テスターはそれを調整して、2019年にワークロードを企業SSDの実際の使用に近づけました。各テストの説明では、何が変更されたのか、その理由を説明しています。

I / O運用テスト（IOPS）

このテストでは、さまざまなサイズ（1024 KB、128 KB、64 KB、32 KB、16 KB、8 KB、4 KB、0.5 KB）のブロックと、読み取り/書き込み比率が異なるランダムアクセスについて、1秒あたりのI / O操作の数を測定しますレコード（100 / 0、95 / 5、65 / 35、50 / 50、35 / 65、5 / 95、0 / 100）。 True Systemの専門家は、次のテストパラメータを使用しました：16スレッド、キューの深さは8。同時に、0.5 KBブロック（512バイト）は、サイズが小さすぎてドライブに深刻な負荷をかけることができないため、まったく実行されませんでした。

IOPSテストは飽和モードに入ることを意味しないため、非常に簡単に実行できます。どちらのドライブも完全に対応し、宣言された工場の特性を完全に満たしています。被験者は、4 KBのブロックの書き込みで優れたパフォーマンスを示しました：70および88千IOPS。これは特に読書志向のKingston DC500Rにとって素晴らしいものです。読み取り操作自体については、これらのSSDは工場出荷時の値を超えるだけでなく、SATAインターフェイスの上限性能に近づきます。

このテストでは、シーケンシャルアクセスのスループットを調べます。つまり、両方のSSDが1 MBおよび128 KBのブロックで順次読み取りおよび書き込み操作を実行します。スレッドあたりのキュー深度が16の8スレッド。

シーケンシャルレコーディングはわずかに遅れています。これは、読み取り集中型クラスに属するKingston DC500Rで特に顕著です。つまり、集中的な読み取り用に設計されています。したがって、テストのこの部分のキングストンDC500Rは、記載されている値よりもさらに低い値を示しました。しかし、True Systemの専門家は、そのような負荷に対してまったく設計されていないドライブ（DC500Rのリソースは0.5 DWPDであることを思い出してください）では、これらの400以上のMB /秒は依然として良い結果であると考えています。

遅延テスト


すでに述べたように、これは企業のドライブにとって最も重要なテストです。実際、SSDドライブの毎日の長い動作中に発生する問題を判別するために使用できます。SNIA PTS標準テストは、最小のキューの深さで、さまざまなブロックサイズ（8 KB、4 KB、0.5 KB）および読み取り/書き込み比率（100 / 0、65 / 35、0 / 100）の平均および最大遅延を測定します（QD = 1の1つのストリーム）。ただし、True Systemの編集者は、より現実的な値を取得するために真剣に変更することを決定しました。

  • 0.5 KBブロックを除外しました。
  • キュー1と32のシングルスレッドロードの代わりに、負荷はフローの数（1、2、4）とキューの深さ（1、2、4、8、16、32）で異なります。
  • 65/35の比率の代わりに、70/30がより現実的な比率として使用されます。
  • 平均値と最大値だけでなく、パーセンタイル99％、99.9％も表示されます。
  • 選択したストリーム数の値について、IOPSに対する遅延の依存性（99％、99.9％、および平均値）のグラフが、すべてのブロックと読み取り/書き込み比率についてプロットされます。

データは、35秒間続く5回のウォームアップ+ 30秒の負荷である25ラウンドのうち4回の平均です。グラフの場合、True Systemエディターは、キューの深さが1から32で、スレッドが1〜4の一連の値を選択しました。これは、遅延を考慮したドライブのパフォーマンスを評価するために行われたものであり、これは最も現実的な指標です。

平均遅延インジケータ：



このグラフは、DC500RとDC500Mの違いを示しています。Kingston DC500Rは集中的な読み取り操作用に設計されているため、書き込み操作の数は負荷の増加に伴って増加せず、25,000のままです。
混合負荷（書き込みの70％と読み取りの30％）を見ると、DC500RとDC500Mの違いも顕著です。 400マイクロ秒の遅延に対応する負荷をかけると、ユニバーサルDC500Mのパフォーマンスが3倍高いことがわかります。これも非常に自然で、ドライブの特性に起因します。
興味深いのは、DC500Mが100％の読み取りでもDC500Rよりも先行しているため、同じ量のIOPSで遅延が少ないことです。違いはわずかですが、非常に興味深いものです。

これらのグラフによると、True Systemの専門家は、QoS遅延によって宣言された特性の信頼性をチェックしました。仕様では、キューの深さが1の4 KBブロックの読み取りが0.5ミリ秒、書き込みが2ミリ秒のインジケーターが示されました。これらの数値が確認されたことも、大きなマージンでお知らせできることを誇りに思います。興味深いことに、最小の読み取り遅延（DC500Rの場合は280〜290μs、DC500Mの場合は250〜260μs）は、QD = 1ではなく2〜4で達成されます。
QD = 1での書き込み遅延は50μsでした（このような低遅延は、低負荷でドライブキャッシュが解放されることが保証されているために得られ、キャッシュへの書き込み時には常に遅延が発生します）。この指標は宣言された値の40分の1です！

継続的なパフォーマンステスト

別の非常に現実的なテストでは、長時間の集中的な作業中のパフォーマンスの変化（IOPSと待機時間）を調査します。作業シナリオとして、4 KBブロックのランダム記録が600分間選択されました。このテストの意味は、そのような負荷があると、コントローラーがガベージを継続的に収集して書き込み用のメモリの空きブロックを準備しているときに、SSDドライブが飽和モードになることです。つまり、これは最も使い果たされるモードであり、実際のサーバーで直面しているエンタープライズクラスのSSDが直面しているものとまったく同じです。

真のシステムテストの結果によると、次のパフォーマンス指標が得られました。



テストのこの部分の主な結果：実際の作業でのKingston DC500RとKingston DC500Mは、どちらも工場出荷時の値を超えています。準備されたブロックが完了すると、tが飽和モードを開始し、Kingston DC500Rは22,000 IOPS（20,000 IOPSではなく）に維持されます。 Kingston DC500Mの範囲は77〜78,000ですが、ドライブプロファイルでは75,000 IOPSが宣言されています。ドライブ間の違いはこのテストでもはっきりとわかります。ドライブのワークフローで書き込み操作の割合が高いことが予想される場合、Kingston DC500Mは3倍以上の生産性があることがわかります（待ち時間の観点から、DC500Mは読み取り操作でパフォーマンスが優れていたことも覚えています）。
