今日は、動的DTPトランキングプロトコルとVTP-VLANトランキングプロトコルについて見ていきます。前のレッスンで述べたように、ICND2試験のトピックは、シスコのWebサイトに表示される順序で説明します。



前回、項目1.1を確認し、今日は1.2を確認します。ネットワークスイッチ接続のセットアップ、チェック、トラブルシューティング：トランクからのVLANの追加と削除、DTPおよびVTPプロトコルバージョン1および2です。

スイッチのすべてのポートは、デフォルトでDTPプロトコルの動的自動モードを使用するように構成されています。つまり、異なるスイッチの2つのポートがそれらの間に接続されている場合、ポートの1つがトランクモードまたはdesirableモードの場合、トランクは自動的にオンになります。両方のスイッチのポートが動的自動モードの場合、トランクは形成されません。



したがって、2つのスイッチそれぞれの動作モードの設定にすべて依存します。わかりやすくするために、2つのスイッチのDTPモードの可能な組み合わせの表を作成しました。両方のスイッチがダイナミックオートを使用している場合、トランクは形成せず、アクセスモードのままであることがわかります。したがって、2つのスイッチ間にトランクを作成する場合は、少なくとも1つのスイッチをトランクモードにプログラムするか、トランクポートをプログラムしてDynamic Desirableモードを使用する必要があります。表からわかるように、各スイッチポートは、アクセス、動的自動、動的望ましい、またはトランクの4つのモードのいずれかにすることができます。

両方のポートがアクセス用に構成されている場合、接続されているスイッチはアクセスモードを使用します。一方のポートがダイナミック自動用に設定され、もう一方がアクセス用に設定されている場合、両方がアクセスモードで機能します。 1つのポートがアクセスモードで動作し、もう1つのポートがトランクモードで動作する場合、スイッチを接続できないため、このモードの組み合わせは使用できません。

したがって、トランキングが機能するためには、スイッチポートの1つがトランクに、もう1つがトランクにプログラムされている必要があります。両方のポートがDynamic Desirable用に構成されている場合も、トランクが形成されます。

Dynamic DesirableとDynamic Autoの違いは、最初のモードでは、ポート自体が2番目のスイッチのポートにDTPフレームを送信することによってトランクを開始することです。 2番目のモードでは、スイッチポートは誰かが彼と話し始めるまで待機します。両方のスイッチのポートがダイナミックオート用に構成されている場合、トランクはそれらの間に形成されません。 Dynamic Desirableの場合、逆の状況があります-両方のポートがこのモード用に構成されている場合、トランクは必然的にそれらの間に形成されます。

相互に接続されたスイッチを適切に構成するのに役立つので、この表を覚えておくことをお勧めします。 Packet Tracerでこの側面を見てみましょう。 3つのスイッチを順番に接続し、これらの各デバイスのCLIコンソールウィンドウを画面に表示します。



show int trunkコマンドを入力すると、すべてのスイッチが動的自動モードに構成されているため、必要な設定がない場合はまったく自然なトランクは表示されません。中央のスイッチのf0 / 1インターフェースのパラメーターを表示するように依頼すると、管理設定モードで動的自動パラメーターが表示されます。



3番目と1番目のスイッチの設定は類似しています-動的自動モードのf0 / 1ポートもあります。テーブルを覚えている場合、トランキングでは、すべてのポートがトランクモードであるか、ポートの1つがDynamic Desirableモードである必要があります。

最初のスイッチSW0の設定に入り、ポートf0 / 1を構成しましょう。 switchport modeコマンドを入力すると、システムは可能なモードパラメータ（アクセス、ダイナミック、トランク）のヒントを提供します。私はswitchport mode dynamic desirableコマンドを使用していますが、このコマンドを入力した後、2番目のスイッチのトランクポートf0 / 1が最初にダウンし、最初のスイッチのDTPフレームを受信した後、アップしたことを確認できます。



ここで、スイッチSW1のCLIコンソールでshow int trunkコマンドを入力すると、ポートf0 / 1がトランキング状態にあることがわかります。スイッチSW1のコンソールに同じコマンドを入力すると、同じ情報が表示されます。つまり、スイッチSW0とSW1の間にトランクがインストールされています。最初のスイッチのポートはdesirableモードで、2番目のポートはautoモードです。



2番目と3番目のスイッチは接続されていないため、3番目のスイッチの設定に移動し、switchport mode dynamic desirableコマンドを入力します。2番目のスイッチで同じダウンアップ状態の変化が発生し、3つのスイッチが接続されているポートf0 / 2に接続していることがわかります。これで、2番目のスイッチには2つのトランクがあります。1つはf0 / 1インターフェイス上に、2つ目はf0 / 2上にあります。これは、show int trunkコマンドを使用して確認できます。



2番目のスイッチの両方のポートは自動状態です。つまり、隣接するスイッチとのトランキングの場合、トランクをインストールするモードは2つしかないため、ポートはトランクモードまたはdesirableモードである必要があります。このテーブルを使用すると、スイッチ間のポートを構成するように、スイッチのポートをいつでも構成できます。これは、動的DTPトランキングプロトコルを使用することの本質です。

VLANトランキングプロトコル、つまりVTPに取り掛かりましょう。このプロトコルは、異なるネットワークデバイスのVLANデータベースの同期を提供し、更新されたVLANデータベースを1つのデバイスから別のデバイスに転送します。 3つのスイッチのスキームに戻りましょう。 VTPは、サーバー、クライアント、透過の3つのモードで機能します。 VTP v3にはオフと呼ばれる別のモードがありますが、Cisco試験のトピックでは、VTPの最初と2番目のバージョンのみがカバーされています。



サーバーモードは、スイッチのコマンドラインから新しいVLANを作成したり、ネットワークを削除または変更したりするために使用されます。クライアントモードでは、VLANで操作を実行できません。このモードでは、VLANデータベースのみがサーバーから更新されます。透過モードは、VTPプロトコルが無効にされているかのように機能します。つまり、スイッチは独自のVTPメッセージを発行しませんが、他のスイッチから更新を送信します。 。透過モードでは、スイッチは独自のVLANデータベースを更新せずに、他の人のメッセージの送信機として機能します。
このスライドでは、グローバル構成モードで入力されたVTPプロトコル構成コマンドを示しています。最初のコマンドでは、使用するプロトコルバージョンを変更できます。 2番目のコマンドは、VTP動作モードを選択します。



VTPドメインを作成する場合は、vtp domain <domain name>コマンドを使用し、VTPパスワードを設定するには、vtp password <PASSWORD>コマンドを入力する必要があります。最初のスイッチのCLIコンソールに移動し、show vtp statusコマンドを発行してVTPの状態を確認します。



VTPプロトコルのバージョンが表示されます。2番目に、サポートされるVLANの最大数は255、既存のVLANの数は5、VLANの動作モードはサーバーです。これらはすべてデフォルト設定です。 VTPについては、30日目のレッスンですでに説明しました。何かを忘れた場合は、戻ってこのビデオをもう一度確認してください。

VLANデータベースを表示するには、show vlan briefコマンドを入力します。ここに示されているのは、VLAN1とVLAN1002-1005です。デフォルトでは、すべての空きスイッチインターフェイスが最初のネットワークに接続されています。23個のファストイーサネットポートと2個のギガビットイーサネットポート、残りの4個のVLANはサポートされていません。他の2つのスイッチのVLANデータベースはまったく同じように見えますが、f1 / 1とf0 / 2はトランクで占有されているため、SW1にはVLAN用に23個、22個のファストイーサネットポートが空いていません。レッスン「30日目」で説明した内容をもう一度思い出させてください。VTPプロトコルはVLANデータベースの更新のみをサポートしています。

スイッチポートアクセスおよびスイッチポートモードアクセスコマンドVLAN10、VLAN20、またはVLAN30を使用してVLANと連携するように複数のポートを設定すると、VTPはVLANデータベースのみを更新するため、これらのポートの設定はVTPを使用して複製されません。
したがって、SW1ポートの1つがVLAN20で動作するように構成されているが、このネットワークがVLANデータベースにない場合、ポートは無効になります。一方、データベースの更新は、VTPプロトコルを使用している場合にのみ行われます。

show vtp statusコマンドを使用すると、3つのスイッチがすべてサーバーモードになっていることがわかります。中間スイッチSW1をvtp mode transparentコマンドでトランスペアレントモードに切り替え、3番目のスイッチSW2をvtp mode clientコマンドでクライアントモードに切り替えます。

次に、最初のスイッチSW0に戻り、vtp domain <domain name>コマンドを使用してnwking.orgドメインを作成します。トランスペアレントモードである2番目のスイッチのVTPの状態を確認すると、ドメインの作成にまったく反応しなかったことがわかります。VTPドメイン名フィールドは空のままです。ただし、クライアントモードの3番目のスイッチはデータベースを更新し、ドメイン名はVTP-nwking.orgです。したがって、スイッチデータベースSW0の更新はSW1を通過し、SW2に影響を与えました。

ここで、指定されたドメイン名を変更しようとします。このドメイン名については、SW0の設定に移動して、vtp domain NetworKingコマンドを入力します。ご覧のとおり、今回は更新が行われませんでした。3番目のスイッチのVTPドメイン名は同じままでした。実際、このようなドメイン名の更新は、デフォルトのドメインが変更されたときに1回だけ行われます。その後、VTPドメイン名が再び変更された場合、他のスイッチでは手動で変更する必要があります。

次に、最初のスイッチのCLIコンソールで新しいVLAN100ネットワークを作成し、それをIMRANと呼びます。最初のスイッチのVLANデータベースには表示されましたが、3つ目のスイッチのデータベースには表示されませんでした。これらは異なるドメインだからです。VLANデータベースが更新されるのは、両方のスイッチのドメインが同じである場合、または前に示したように、デフォルト名の代わりに新しいドメイン名が設定されている場合のみです。

3つのスイッチの設定に進み、vtpモードとvtpドメインのNetworKingコマンドを順に入力します。名前の入力では大文字と小文字が区別されるため、ドメイン名のスペルは両方のスイッチでまったく同じである必要があります。次に、vtp mode clientコマンドを使用して、SW2を再びクライアントモードにします。しばらく様子を見てみましょう。ご覧のように、ドメイン名が一致すると、SW2データベースが更新され、新しいVLAN100 IMRANがその中に表示されます。透過モードであるため、これらの変更は平均的なスイッチに影響を与えません。

不正アクセスから身を守りたい場合は、VTPパスワードを作成できます。この場合にのみ、VTPアップデートを受信できるため、反対側のデバイスが完全に同じパスワードを持っていることを確認する必要があります。

次に検討するのは、VTPプルーニング、つまり未使用のVLANの「トリミング」です。ネットワーク上でVTPプロトコルを使用する100台のデバイスがある場合、1つのデバイスのVLANデータベースを更新すると、残りの99台のデバイスに自動的に複製されます。ただし、これらのデバイスのすべてがアップデートで言及されているVLANを備えているわけではないため、それらに関する情報は必要ない場合があります。

VTPを使用してデバイスにVLANデータベースの更新を配信すると、すべてのデバイスのすべてのポートが、追加、削除、および変更されたVLANに関する情報を受信しますが、それらのVLANには何もすることができません。同時に、ネットワークは過剰なトラフィックで詰まっています。これを防ぐには、VTPプルーニングの概念を使用します。スイッチで無関係なVLANの「トリミング」モードを有効にするには、vtp pruningコマンドを使用します。その後、スイッチは実際に使用しているVLANについて自動的に相互に通知し、接続されていないネットワークに更新を送信する必要がないことをネイバーに警告します。

たとえば、SW2にVLAN10ポートがない場合、このネットワークのトラフィックを送信するためにSW1は必要ありません。同時に、スイッチSW1にはVLAN10トラフィックが必要です。そのポートの1つがこのネットワークに接続されているため、このトラフィックをスイッチSW2に送信する必要がないだけです。
したがって、SW2がvtpプルーニングモードを使用している場合、SW1に「このネットワークは私に接続されておらず、ポートの1つがこのネットワークで動作するように構成されていないため、VLAN10のトラフィックを送信しないでください」と伝えます。 vtp pruningコマンドを使用すると、次のようになります。

特定のインターフェースのトラフィックをフィルタリングする別の方法があります。これにより、特定のVLANを使用してトランク上のポートを構成できます。この方法の欠点は、各トランクポートを手動で構成する必要があることです。これにより、許可されるVLANと禁止されるVLANを示す必要があります。これには、3チームのシーケンスが使用されます。 1つ目はこれらの制限が関係するインターフェースを示し、2つ目はこのインターフェースをトランクポートに変換し、3つ目はスイッチポートトランク許可vlan <all / none / add / remove / VLAN number>-このポートで許可されるVLANを示します：すべて、なし1つは、追加されたVLANまたは削除されたVLAN。

特定の状況に応じて、使用するVTPプルーニングまたはトランク許可を選択します。一部の組織は、セキュリティ上の理由からVTPを使用しないことを好むため、トランキングを手動で構成することを選択します。 vtp pruningコマンドはPacket Tracerでは機能しないため、GNS3エミュレーターで表示します。

SW2の設定に移動し、vtp pruningコマンドを入力すると、システムはこのモードがオンになっていることをすぐに通知します。プルーニングがオンになっている、つまり、VLANトリミングが1つのコマンドだけで有効になっています。

show vtp statusコマンドを入力すると、vtpプルーニングモードが有効になっていることがわかります。



スイッチサーバーでこのモードを構成する場合は、その設定に移動し、vtp pruningコマンドを入力します。つまり、サーバーに接続されているデバイスは、vtpプルーニングを自動的に使用して、古いVLANのトランキングトラフィックを最小限に抑えます。

このモードを使用しない場合は、特定のインターフェイス（e0 / 0など）を入力してから、switchport trunk allowed vlanコマンドを入力する必要があります。システムは、このコマンドの可能なパラメータのヒントを提供します。

-WORD-トランクモードのこのインターフェイスで解決されるVLAN番号。
-追加-VLANデータベースリストに追加するVLAN。
-all-すべてのVLANを有効にします。
-以外-指定されたものを除くすべてのVLANを許可します。
-なし-すべてのVLANを無効にします。
-削除-VLANデータベースのリストからVLANを削除します。

たとえば、VLAN10で有効になっているトランクがあり、VLAN20で有効にしたい場合は、switchport trunk allowed vlan add 20コマンドを入力する必要があります。



他の情報を表示したいので、show interface trunkコマンドを使用します。デフォルトでは、トランクに対してすべてのVLAN 1〜1005が許可され、VLAN10がそれらに追加されていることに注意してください。
