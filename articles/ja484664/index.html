Android向けのもう1つの[ほぼ]削除できないトロイの木馬

昨年末、システム領域の変更検出機能を使用して、一部のユーザーがシステムファイル/system/lib/libc.soに変更を記録しました。これは、Linuxベースのオペレーティングシステムの主要なライブラリの1つであり、システムコールと基本的な機能を担当します。このケースの詳細な調査により、2015年以降私たちに知られているAndroid.Xinyファミリーのトロイの木馬からの新しいサンプルが明らかになりました。

初めて、その代表者のファイルに「不変」属性がインストールされることを確認しました。これにより、デバイスからのトロイの木馬の削除が非常に複雑になりました。

指定された属性はインストールされたアプリケーションのapkファイルに置かれ、このアプリケーションを削除する試みは成功したように見え、データは削除されましたが、apkファイル自体はそのまま残りました。デバイスを再起動すると、アプリケーションが再び「表示」されました。2016年に、これらのトロイの木馬の1つについて話しました。このような脅威に対抗するために、アンチウイルスにファイルリセット属性を追加しました。これは、ユーザーがアンチウイルスにルート権限を付与した場合に機能します。

この記事では、Android.Xinyの新しいバージョンで使用されている別の興味深い自己防衛の方法を検討します。

Android 5.1？2019年に？



この記事で取り上げたトロイの木馬は、バージョン5.1までのAndroid OSで動作します。そのような「古代」バージョンのAndroid用に設計されたマルウェアがまだアクティブであるのは奇妙に思われるかもしれません（バージョン5.1は2015年にリリースされました）。しかし、その年齢にもかかわらず、古いバージョンはまだ使用されています。グーグルによると2019年5月7日の時点で、デバイスの25.2％がAndroid 5.1以下を実行しています。ユーザーに関する統計では、わずかに大きい数値が得られます-約26％。これは、すべてのAndroidデバイスの約4分の1が潜在的なターゲットであり、それほど小さくはないことを意味します。これらのデバイスが修正されることのない脆弱性の影響を受けやすいことを考えると、古いバージョンのAndroid OSが依然としてウイルス作成者の関心を集めていることは当然のことです。実際、前述の脆弱性を悪用することで取得できるroot権限は、ウイルス作成者の手を解き放ちます-彼らの助けを借りて、デバイス上で何でもできます。ほとんどの場合、それはアプリケーションの平凡なインストールに帰着します。

トロイの木馬の主な機能


初期のバージョンから始めて、Android.Xinyトロイの木馬の主な機能は、ユーザーの許可なしにデバイスに任意のアプリケーションをインストールすることです。したがって、攻撃者は、インストールの料金を支払うアフィリエイトプログラムに参加することでお金を稼ぐことができます。判断できる限り、これはこの家族の作成者にとって主な収入源の1つです。いくつかの代表的な製品を起動した後、数分で実際に動作しないデバイスを取得できます。そのデバイスには、多くの無害だが不要なユーザーアプリケーションがインストールされて起動されます。さらに、これらのトロイの木馬はマルウェアもインストールする可能性があります—すべて管理サーバーから受信したコマンドに依存します。

Android.Xinyトロイの木馬の新しいバージョンを際立たせる最も興味深いのは、削除に対する保護です。2つのコンポーネントがそれを担当します。より詳細に検討してみましょう。

インストーラ


sha1：f9f87a2d2f4d91cd450aa9734e09534929170c6c
検出：Android.Xiny.5261

このコンポーネントは、root権限を取得した後に起動します。システムファイル/ system / bin / debuggerdと/ system / bin / ddexeが自動的に起動されるように置き換え、元のファイルを_serverという名前で保存し、従来のコンパニオンウイルスのように機能します。また、コマンドラインパラメータで渡されたフォルダからいくつかの実行可能ファイルをシステムパーティションにコピーします。さらに、特別なパラメータを使用して実行し、新しいバージョンが配置されているフォルダを指定すると、トロイの木馬は、システムパーティションにインストールしたコンポーネントを更新できます。

Android.Xiny.5261には、削除するファイルの印象的なリストが含まれています。これには、システムパーティションにインストールされた、トロイの木馬のファミリーの古いメンバーに特有のパスが含まれます。たとえば、Triadaなど。



さらに、Android.Xiny.5261は、事前にインストールされた一部のアプリケーションを削除します-おそらくスペースを解放するためです。最後に、SuperSU、KingRootなどの有名なルート権限管理アプリケーションを削除します。したがって、ルート権限を使用する機能をユーザーから奪うため、システムパーティションにインストールされているトロイの木馬コンポーネントを削除します。

ここでは、起動されたファイルへのパスが「/ data /」で始まるかどうか、および「/ sdcard」が含まれているかどうかがチェックされます。いずれかの条件が満たされた場合、開始はブロックされます。パスに沿って/データ/データ/がアプリケーションディレクトリであることを思い出してください。これにより、通常のアプリケーションがファイルを作成できるすべてのディレクトリからの実行可能ファイルの実行がブロックされます。

libc.soシステムライブラリに加えられた変更は、root権限を取得するように設計されたアプリケーションを破壊します。 exec *関数の変更により、通常、エクスプロイトはネットワークからアプリケーションディレクトリにダウンロードされて実行される実行可能ファイルであるため、このようなアプリケーションはシステムの権限を高めるためにエクスプロイトを起動できません。それでも特権の昇格に成功した場合、変更されたマウント機能では、システムパーティションを書き込み用に再マウントできません。つまり、システムパーティションに変更を加えることはできません。

その結果、トロイの木馬の自己防衛は2つの部分で構成されます：インストーラーはルート権限を管理するためのアプリケーションをアンインストールし、変更されたlibc.soライブラリーは再度インストールされるのを防ぎます。さらに、この保護は「競合他社」、つまり「適切な」ルート権限アプリケーションと同じ原理で機能するため、ルート権限を取得してシステムパーティションにインストールされる他のトロイの木馬からも機能します。

このようなトロイの木馬に対処する方法は？


Android.Xiny.5260を取り除くために、デバイスに再フラッシュすることができます-そのためのオープンアクセスファームウェアがある場合。しかし、別の方法でマルウェアを削除することは可能ですか？難しいですが可能です-いくつかの方法があります。 root権限を取得するには、so-librariesの形式でエクスプロイトを使用できます。実行可能ファイルとは異なり、このトロイの木馬はダウンロードをブロックしません。トロイの木馬自体のコンポーネントを使用することもできます。このコンポーネントは、他の部分にルート権限を提供するように設計されています。パス/ dev / socket / hs_linux_work201908091350に沿ってソケットを介してコマンドを受信します（パスはバージョンによって異なる場合があります）。マウントロックのバイパスについては、mountflagsパラメータの「マジック」値を使用するか、対応するsyscallを直接呼び出すことができます。
