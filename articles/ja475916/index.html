ロボットに目を追加

時々、ロボットは何かをつかむ必要があります。あたかも手なしのように目のないロボット。文字通り。結局、おいしいものがどこにあるのかわからないので、ロボットはロボットアームでそれに到達することができません。または他のマニピュレータ。

この記事では、ロボット座標系と3Dカメラを切り替えることができるようにロボットを較正する方法を説明します。



ステップ1.ロボット

を利用するDobot Magicianがオブジェクトを移動します。オブジェクトがどのように移動するかをご覧ください-Intel Realsense D435。そして、キャリブレーションオブジェクトは赤いボールでなければなりません。

これは、彼らが一緒に見える方法です。



赤いボールは、誰かがクリスマスツリーを飾るだけのものではありません。赤-イメージ内で簡単に見つかるように（別のネットワークを学習せずに）。ボール-画像の中心をより正確に計算します。しかし、後でこれに戻ります。

ステップ2.ロボットがボールを動かす場所

キャリブレーションアルゴリズムは次のように説明できます。

  1. 空間内の位置のリストを生成します
  2. ロボットはボールをリスト内で動かし、それを使って画像を撮影します。
  3. 各画像について、ボールの座標を見つけます
  4. カメラ座標からロボット座標への変換を計算します

配置は、可能な限り多くのスペースをカバーするように選択する必要があります。小さなボールをとらない方が良いです。これは校正の精度を低下させます。

ロボット自体、カメラ、すべてが実行される環境（たとえば、ROSで実行した環境）に依存しすぎるため、ここではキャリブレーションを記録するためのコードは示しません。重要なことは、さらなるキャリブレーションのために、少なくとも手動モードで写真を撮り、ロボットを位置に送るためのステップを送信し、写真を保存できることです。

唯一の要件は、カメラとロボットの相対位置が常に変化しないことです。検証のために2番目の画像セットをすぐに撮影して、キャリブレーションがどの程度正確に計算されたかを評価することもお勧めします。

ステップ3.座標を取得する

リストで指定された位置に特別にroborukを送信するため、マニピュレーターの座標がわかります。自由度が多いロボットの場合、より多くのパラメーターを指定する必要がありますが、4軸ロボットアームでは、位置はロボットアームの先端の座標によって一意に決まります。だから私たちは、手が行った位置を保持するだけです。



ボール画像の場合、その座標はまだ計算されていません。適切に選択されたオブジェクトをキャリブレーションに使用します。ボールを見つけるタスクを「赤の最大領域を見つける」として再定式化します。これはpython + opencvでは次のようになります。

def get_center(image):
    #      
    image = cv2.GaussianBlur(image, (7, 7), 0)
    #     HSV
    hsv = cv2.cvtColor(image, cv2.COLOR_BGR2HSV)
    #   
    saturation = hsv[...,1]
    saturation[(hsv[..., 0] > 15) & (hsv[..., 0] < 165)] = 0
    _, image1 = cv2.threshold(saturation, 92, 255, cv2.THRESH_BINARY)
    #    
    contours = cv2.findContours(image1, cv2.RETR_LIST, cv2.CHAIN_APPROX_SIMPLE)[0]
    contour = max(contours, key=cv2.contourArea)
    #   
    b_circle = cv2.minEnclosingCircle(contour)
    return b_circle[0]

次に、球の中心の座標を見つける必要があります。さまざまな方法があります。たとえば、球の3Dモデルを点群にカスタマイズできます。ただし、realsense D435では、オブジェクトの境界で深度値がかなりうるさいので、別の方法を使用します。

ここでボールが最大の接続領域であると想定します。次に、その中心を見つけ、チャネルの深さからボールの表面上のこの点までの深さを見つけます。そして、ステレオメトリーの助けを借りて、ボールの表面の可視点からその中心に移動しましょう。

ボールの中心を見つけるには、投影が円でなければならないという知識を使用し、中心の公平な推定値を取得します（ただし、ロボットの吸盤が私たちを妨げようとしています）。ここでは、この領域を説明する最小領域の円を見つけるだけで十分です。これは、opencv -minEnclosingCircleですでに決定されています。

ポイントu、vの2つの座標（ピクセル）とそれに対するミリメートル（mm）を受け取ったら、それらをカメラSKの物理座標に変換します。

def get_world_coords(u, v, depth, camera_matrix):
    f = np.linalg.inv(camera_matrix)
    l = np.array([u,v,1]) * depth
    return np.dot(f,l)

camera_matrix- 式による、カメラの内部パラメータの行列。

ステップ4.キャリブレーションを実行

する現時点では、空間内のさまざまな位置の2セットのポイントを受け取りました。球の上部の座標-ロボットのSCでボールがロボットを吸うポイントの座標と、ボールの可視中心の座標-カメラに最も近い球のポイントSCカメラ。それらを比較するには、最初にそれらをボールの1つの物理的なポイントに持ってくる必要があります。

かなり残っています-同じ物理点に対応する2つの3D座標セットがある場合、最初のセットから2番目のセットへの変換を見つけます。これにはopencv cv2.estimateAffine3D関数を使用します。理想的に見つかる座標の場合、アフィン変換はもちろん列挙型です。ポイントの変換を説明するには、回転と変位で十分ですが、ストレッチは不要です。ただし、アフィン変換を使用すると、内部カメラパラメーターの計算が不十分な行列を使用して、不正確さを補正できます。それだけでなく、まったく知らなくてもキャリブレーションを取得できます。

transformation = cv2.estimateAffine3D(camera_coords, robot_coords)

出力は3x4変換行列で、最初の3x3コンポーネントは軸拡張と組み合わせた回転行列です。正しいカメラキャリブレーションと適切な入力データがあれば、回転行列に近い行列が得られます。残りの3つの数値は、カメラとロボット間の変位ベクトルです。

ステップ6.適用された問題で使用

するキャリブレーションが完了したら、実際の問題の解決に進むことができます。たとえば、VRヘルメットをコントローラーに接続して、ロボットを制御し、バーチャルリアリティでキューブを動かすことができました。


しかし、この校正方法は非常に一般的です。使用するカメラとロボットは関係ありません。説明した方法により、ロボットとカメラの座標系間の関係を簡単に計算できます。さらに、わずかな変更で、カメラはロボットに対して相対的に移動するため、メソッドは自律的であり、ロボットは自動的にキャリブレーションを実行できます。
