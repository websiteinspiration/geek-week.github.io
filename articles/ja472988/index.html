Fast.aiを使用してテキストの調性を分析します

この記事では、ロシア語のテキストメッセージの調性の分類について説明します（基本的に、同じテクノロジーを使用したテキストの分類）。この記事を基礎として、Word2vecモデルを使用したCNNアーキテクチャ上の調性の分類が検討されました。この例では、ULMFitモデルを使用して、同じデータセットでツイートをポジティブとネガティブに分離する同じ問題を解決します。記事の結果（平均F1スコア= 0.78142）はベースラインとして受け入れられます。

前書き


ULMFITモデルは、2018年にfast.ai開発者（Jeremy Howard、Sebastian Ruder）によって導入されました。このアプローチの本質は、事前トレーニング済みモデルを使用するときにNLPタスクで転移学習を使用し、モデルのトレーニング時間を短縮し、ラベル付きテストサンプルのサイズの要件を削減することです。

この場合のトレーニングスキーム



は次のようになります。言語モデルの意味は、次の単語を順番に予測できることです。このように長い接続されたテキストを取得することは問題ですが、それでも言語モデルは言語のプロパティをキャプチャし、単語の使用のコンテキストを理解できるため、テクノロジーの基礎となるのは言語モデル（たとえば、単語のベクトル表示ではない）です。 ULMFitは、シミュレーションタスクにAWD-LSTMアーキテクチャを使用します。可能な限りドロップアウトを積極的に使用することを意味します。ここでのラベルは次の単語であり、手で何もマークする必要がないため、言語モデルトレーニングのタイプは、半教師あり学習と呼ばれることもあります。

事前トレーニング済みの言語モデルとして、公的に入手可能なほとんどすべてを使用します。

トレーニング用のデータを準備します


同様に、Twitterからのロシア語メッセージに基づいて作成された、Yulia Rubtsovaによる短いテキストRuTweetCorpの本文についてトレーニングを実施します。本文には、CSV形式で114,991件の肯定的なツイートと111,923件の否定的なツイートが含まれています。さらに、SQL形式で17 639 674レコードのボリュームを持つ未割り当てのツイートのデータベースがあります。私たちの分類子のタスクは、ツイートがポジティブかネガティブかを判断することです。

以来、それは1700万ツイートに言語モデルを再教育するのに長い時間だったと転移学習を示すの仕事だった怠惰、我々は完全に未割り当てのつぶやきのベースを無視して、トレーニングデータセットからのテキストの一部の言語モデルを再教育します。おそらく、このベースを使用して言語モデルをシャープにすることで、全体的な結果を向上させることができます。

予備のワープロでトレーニングとテストを行うためのデータセットを作成します。元の記事からコードを取得します。

#  
n = ['id', 'date', 'name', 'text', 'typr', 'rep', 'rtw', 'faw', 'stcount', 'foll', 'frien', 'listcount']
data_positive = pd.read_csv('data/positive.csv', sep=';', error_bad_lines=False, names=n, usecols=['text'])
data_negative = pd.read_csv('data/negative.csv', sep=';', error_bad_lines=False, names=n, usecols=['text'])

#   
sample_size = min(data_positive.shape[0], data_negative.shape[0])
raw_data = np.concatenate((data_positive['text'].values[:sample_size],
                           data_negative['text'].values[:sample_size]), axis=0)
labels = [1] * sample_size + [0] * sample_size

def preprocess_text(text):
    text = text.lower().replace("", "")
    text = re.sub('((www\.[^\s]+)|(https?://[^\s]+))', 'URL', text)
    text = re.sub('@[^\s]+', 'USER', text)
    text = re.sub('[^a-zA-Z--1-9]+', ' ', text)
    text = re.sub(' +', ' ', text)
    return text.strip()


data = [preprocess_text(t) for t in raw_data]

df_train=pd.DataFrame(columns=['Text', 'Label'])
df_test=pd.DataFrame(columns=['Text', 'Label'])

df_train['Text'], df_test['Text'], df_train['Label'], df_test['Label'] = train_test_split(data, labels, test_size=0.2, random_state=1)

df_val=pd.DataFrame(columns=['Text', 'Label'])

df_train, df_val = train_test_split(df_train, test_size=0.2, random_state=1)

混同マトリックスを見てみましょう：

interp = ClassificationInterpretation.from_learner(learn)
interp.plot_confusion_matrix()



精度、再現率、およびf1スコアパラメータを計算します。

neg_precision = interp.confusion_matrix()[0][0] / (interp.confusion_matrix()[0][0] + interp.confusion_matrix()[1][0])
neg_recall = interp.confusion_matrix()[0][0] / (interp.confusion_matrix()[0][0] + interp.confusion_matrix()[0][1])
pos_precision = interp.confusion_matrix()[1][1] / (interp.confusion_matrix()[1][1] + interp.confusion_matrix()[0][1])
pos_recall = interp.confusion_matrix()[1][1] / (interp.confusion_matrix()[1][1] + interp.confusion_matrix()[1][0])
neg_f1score = 2 * (neg_precision * neg_recall) / (neg_precision + neg_recall)
pos_f1score = 2 * (pos_precision * pos_recall) / (pos_precision + pos_recall)

print('          F1-score')
print('    Negative   {0:1.5f}    {1:1.5f}   {2:1.5f}'.format(neg_precision, neg_recall, neg_f1score))
print('    Positive   {0:1.5f}    {1:1.5f}   {2:1.5f}'.format(pos_precision, pos_recall, pos_f1score))
print('    Average    {0:1.5f}    {1:1.5f}   {2:1.5f}'.format(statistics.mean([neg_precision, pos_precision]), 
                                                              statistics.mean([neg_recall, pos_recall]), 
                                                              statistics.mean([neg_f1score, pos_f1score])))
