Xamarin.FormsのXAMLホットリロードがリリースされました

先週のXamarin Developer Summitで、Xamarin.FormsのXAML Hot Reloadを発表しました。これにより、XAMLユーザーインターフェイスに変更を加えて、リアルタイムで表示できます。別のビルドやデプロイは必要ありません。

Xamarin.FormsのXAMLホットリロードは、開発をスピードアップし、ユーザーインターフェイスでのアセンブリ、実験、反復を簡素化します。つまり、ユーザーインターフェイスを構成するたびにアプリケーションを再構築する必要がなくなり、すべての変更が実行中のアプリケーションに即座に表示されます。

アプリケーションがXAMLホットリロードを使用してコンパイルされると、すべてのライブラリとサードパーティのコントロールで動作します。この機能は、Visual Studio 2019およびVisual Studio 2019 for MacのiOSおよびAndroidで利用できます。すべては、シミュレーター、エミュレーター、物理デバイスを含むすべての有効な展開目標で機能します。

XAMLのホットリブートは2019年後半に利用可能になりますが、プレビューに参加するために登録することができます：

何がサポートされていますか？


Xamarin.FormsのXAMLホットリブートは、既存のワークフローに接続して時間を節約し、生産性を向上させます。今日、XAMLに変更を加える場合は、ビルド、デプロイ、およびデバッグを再度行う必要があります。次に、XAMLファイルを保存して、変更が展開ターゲットにリアルタイムでどのように反映されるかを確認します。これにより、モバイルアプリケーションの作成と展開により多くの時間を費やすことができます。

Hot Reloadを使用するために特別なことをする必要はありません。追加のパッケージやコードはありません。 Visual StudioまたはVisual Studio for Macで通常どおり、iOS、Androidエミュレーター、シミュレーター、または物理デバイスにアプリケーションをビルドしてデプロイします。 Xamarin.FormsのXAML Hot Reloadはネットワーク接続ではなくデバッガーを使用するため、複雑な企業ネットワークや非接続環境で確実に機能します。 XAMLを編集して[保存]をクリックします。再起動すると、ナビゲーションステータスが保存されます。 MVVMテンプレートを使用する場合、ビューモデルにバインドされているUI状態は、再起動しても変更されません。



この新製品は、ページ、アプリケーション、シェルなど、すべての有効なXAMLタイプの更新もサポートしています。 MyControl.xamlを編集して保存すると、このコントロールにリンクするすべてのページが、変更に応じて自動的に更新されます。

XAMLホットリブートは、タイプミスやIntelliSense高速アクションなどのサポートされていない変更に耐性があります。変更を再読み込みする前に、Visual Studioは変更が有効であることを確認します。編集に無効な変更が含まれている場合、その変更はアプリケーションに適用されず、XAMLエディターは曲線を表示して、それを認識できるようにします。再起動がVisual Studioの[出力]ウィンドウを通過したかどうかを確認することもできます。

Xamarin.FormsのXAMLホットリロードは、複雑さに関係なく、すべてのタイプのアプリケーションで機能することが重要です。Xamarin.FormsのXAML Hot Reloadを使用して問題が発生しないように、Telerik、Infragistics、Grial UI Kit、Prism、Syncfusionなどのコントロールプロバイダーおよびオープンソースプロジェクト開発者とのパートナーシップを確立しています。 。

レビューを重視


Xamarin.FormsのXAML Hot Reloadの開発者とさまざまな概念の50以上のテストを実施し、概念が内部開発サイクルの生産性を高めることを確認しました。また、いくつかの主要なXamarinクライアント、Visual Studioパートナー、およびXamarin MVPパートナーと提携して、このエクスペリエンスがお客様の期待に応えることを保証しています。最後に、シアトルのユーザーエクスペリエンスラボに開発者を招待し、Xamarin.FormsのXAMLホットリロードを使用して、すべての開発者にとって直感的なエクスペリエンスであることを確認しました。

次は何ですか？


プライベートプレビュー段階では、XAMLホットリブートに引き続き取り組み、ニーズに対応し、品質やパフォーマンスなどの要素に設定した期待を超えることを確認します。準備が整ったことをお知らせいただければ、Visual StudioおよびVisual Studio for Macの将来のリリースに組み込まれる予定です。

品質と生産性の目標を達成することに加えて、次の項目もXAMLホットリロードロードマップに含まれています。

  • 増分リロード：変更されたコントロールまたはプロパティのみをリロードします
  • 同時再起動：複数のデバッグ対象プラットフォームのXAMLを同時に再起動します
  • ステータス監視の改善：「粗い」編集または再読み込みする無効な変更を含む編集を表示するためのUXの改善
  • CSS：CSSスタイルの更新はリロード時に適用されます

XAMLはXamarinのホットリブートの始まりにすぎません。今後も、お客様にとって最も重要と思われる項目にロードマップを集中していきます。XAMLに焦点を当てるだけでなく、Visual Studioで作業する際の生産性を向上させるための追加の方法を引き続き模索しています。
