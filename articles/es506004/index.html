Primeros pensamientos sobre Deno

Se preparó una traducción del artículo antes del lanzamiento del curso Node.js Developer .


En Codegram, nos encanta experimentar con nuevas tecnologías, ¡y Deno salió hace apenas una semana! Lea el artículo si desea conocer este nuevo tiempo de ejecución de JavaScript y qué piensa Codegram al respecto.

Es posible que ya haya oído hablar de Deno, o tal vez no, pero ahora Internet está lleno de información al respecto y hay una razón para eso. Ha aparecido un nuevo tiempo de ejecución de JavaScript con un lindo dinosaurio como logotipo (algunas personas piensan que el logotipo tiene un calcetín triste, pero no ...). Incluso tuiteamos sobre él una semana antes del primer lanzamiento, pero, para decir la verdad, escuchamos sobre Deno hace dos años después de ver esta increíble actuación de Ryan Dahl .

En este artículo, repasaremos Deno superficialmente y compartiremos nuestros pensamientos al respecto. ¡Empecemos!

Hola deno


Deno es un tiempo de ejecución de JavaScript como Node. Para usarlo, primero debe instalar la CLI deno. Para hacer esto, siga las instrucciones para su sistema operativo en deno.land (¡nuestro nombre de dominio se complementa!). Deno no es un tenedor de Nodo, fue creado desde cero en Rust. Escribí un artículo sobre Rust para programadores de JS , ¡puedes leerlo si quieres saber más sobre el lenguaje!

Después de instalar la CLI, puede ejecutar el programa clásico "¡Hola, mundo!" de la siguiente manera:

deno run https://deno.land/std/examples/welcome.ts

¿Estamos ejecutando TypeScript fuera de la caja? ¡Si! TS es un invitado bienvenido en Deno, por lo que no necesita nada más para ejecutar el código en TS. También puede ejecutar el código JS, pero soy un gran admirador de TS, así que fui el primero en hablar de ello.

Tenga en cuenta que el código de nodo no es compatible con Deno. Sin embargo, en este momento están trabajando para garantizar la compatibilidad , de modo que en el futuro pueda usar sus paquetes NPM favoritos.

Hay una cosa más que vale la pena mencionar que Deno es un sandbox por defecto. Es decir, inicialmente no tiene acceso a la red o al sistema de archivos. Si su programa necesita acceso, debe permitirlo explícitamente (es decir, usar --allow-net para otorgar privilegios de red).

No mas package.jsononode_modules
¿Quién no ha visto este chiste en Internet?



A pesar de una ligera exageración, todavía hay algo de verdad en esta broma. A medida que su aplicación crece, la lista de dependencias crece y usted termina con un gran árbol de dependencias. El problema es que cada aplicación tiene una copia de la versión package @ que está utilizando.
Uno de los principales objetivos de Deno es duplicar el navegador. Cuando desee importar algo en un navegador, escriba una etiqueta <script>y use la URL. Deno funciona de la siguiente manera:

import { assertEquals } from "https://deno.land/std/testing/asserts.ts";
assertEquals("hello", "hello");
assertEquals("world", "world");
console.log("Asserted! ");

Estoy de acuerdo, se ve un poco extraño. Como no tenemos ningún package.json, no hay forma de especificar dependencias externas, por lo que las importamos cuando son necesarias. Al principio, Deno descargará el archivo y lo guardará en el caché. Puede especificar el indicador --reload si desea descargar el paquete nuevamente.

Probablemente tenga muchas preguntas sobre este tema (yo también las tuve), por lo que lo invito a visitar esta página de preguntas frecuentes en el manual de Deno para obtener más información sobre el tema.

Tiempo de ejecución compatible con la API del navegador


Otro objetivo importante de Deno era la compatibilidad del navegador, por lo que tiene un montón de características disponibles desde el ámbito global, como fetch o addEventListener . También puede usar el objeto de ventana global (sin embargo, por ahora, recomiendo usar globalThis). Puede obtener más información sobre las funciones globales en la documentación .

El tiempo de ejecución también tiene un Deno global para API que no son un estándar web. Puede usarlo para operaciones de bajo nivel, como leer un archivo, abrir un socket TCP, etc.

Biblioteca estándar rica


Los desarrolladores de Deno han creado una colección de módulos estándar listos para usar. La biblioteca también está alojada en el dominio deno.land , por lo que puede importar cualquier módulo a su aplicación utilizando la siguiente URL:

import { v4 } from "https://deno.land/std/uuid/mod.ts";

console.log(v4.generate());

La biblioteca estándar incluye muchos módulos útiles, pero hay muchos de terceros, y esta lista se actualiza todos los días. Hay una buena noticia más: puede importar cualquier módulo colocado en una URL pública, lo que significa que puede importar fácilmente módulos desde GitHub. Deno tiene un servicio de reescritura de URL que puede usar para que sus módulos estén disponibles. Puede obtener más información sobre esta mecánica en esta página .

Herramientas incorporadas


Deno CLI también incluye herramientas integradas para realizar algunas tareas comunes, como ejecutar pruebas, por ejemplo, ¡o incluso crear un paquete! ¿Esto significa que Deno puede reemplazar broma, más bonita y webpack? Probablemente no. Deno es una solución bastante nueva y aún no tiene un ecosistema tan extenso, pero la presencia de todas estas herramientas, cuyo acceso se puede obtener mediante la instalación de un binario, sigue siendo impresionante.

Conclusión


¡Salta sobre esta ola de bombo! Estoy realmente impresionado con Deno. Después de dos años de desarrollo y un largo camino hacia el éxito, el nuevo tiempo de ejecución de JavaScript se ve muy bien y estoy ansioso por comenzar a experimentar con él. Node apareció hace unos 11 años y se convirtió en un nuevo hito en la historia de JavaScript. Él mejoró el idioma, y ​​creo que hemos alcanzado un nuevo hito, ¡es hora de Deno!
