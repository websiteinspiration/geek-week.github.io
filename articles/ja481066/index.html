こんにちは

画像分類


以前の記事と同様に、Ubuntu 16.04を使用します。仕事には、Kerasライブラリが必要です。こちらの手順に従ってUbuntuにインストールできます。ページに

移動し、コードのダウンロードボタンを押してください！ページの下。ソースが記載されたメールが届きます。 アーカイブをダウンロードして解凍し、フォルダに移動します。最初に、事前トレーニング済みのGoogleNetモデルに基づいてオブジェクト分類スクリプトを実行します。



cd pi-deep-learning/
python pi_deep_learning.py --prototxt models/bvlc_googlenet.prototxt \
    --model models/bvlc_googlenet.caffemodel --labels synset_words.txt \
    --image images/barbershop.png

オブジェクト検出


次に、魚眼カメラを備えたRaspberry Piでオブジェクトを検出してみます。ページに移動し、コードのダウンロードボタンを押してください！ページの下。ソースが記載されたメールが届きます。

分類と同様に、ダウンロード、解凍、フォルダに移動します。オブジェクトを検出するためのスクリプトを実行します。

python pi_object_detection.py --prototxt MobileNetSSD_deploy.prototxt.txt --model MobileNetSSD_deploy.caffemodel

結論： 

[INFO] loading model...
[INFO] starting process...
[INFO] starting video stream...
[ INFO:0] Initialize OpenCL runtime...
libEGL warning: DRI3: failed to query the version
libEGL warning: DRI2: failed to authenticate

スクリプトを実行すると、次のエラーが発生する可能性があります。

Error: AttributeError: ‘NoneType’ object has no attribute ‘shape’ (comment to post)

この問題を解決するには、ファイルpi_object_detection.pyを開き、74行目をコメントアウトします。

vs = VideoStream(src=0).start()

75行目をコメント解除します


vs = VideoStream(usePiCamera=True).start()





すべてのオブジェクトが正しく認識されているわけではありませんが、スクリプトがいくつかのオブジェクトを検出していることがわかります。私の観察によると、スクリプトはオブジェクトの検出をかなり速く実行します（残念ながら、fpsは修正できませんでした）。

real_time_object_detection.pyスクリプトを実行することもできます。エラーが発生した場合は、pi_object_detection.pyスクリプトの手順を繰り返します。38行目をコメントアウトし、39行目をコメント解除



します。結果これで、すべてです。皆さん、頑張ってくださいね！
