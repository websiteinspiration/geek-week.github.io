openHABとHome Assistantのホームオートメーション


最初は、openHABですべてのホームオートメーションを開始しました。そして、私はすでにopenHABの経験がありましたが、それはクイックスタートではありませんでした。このホームオートメーションシステムはどのようなものですか？

openHAB（オープンホームオートメーションバスの略）は、建物のオートメーション用のオープンプラットフォームとしてドイツのKai Kreuzerによって開発された2010年にさかのぼります。 2010年には、そのようなソリューションは事実上存在せず、openHABは主に現在観察しているさまざまなスマートホームシステムのプロトタイプになりました。彼のアイデアは単純です。プロトコルや技術的機能に関係なく、さまざまなメーカーのソリューションを1つのオープンソフトウェアプラットフォームに組み合わせます。これにより、特定のメーカーから離れ、単一の管理インターフェースですべての製品を使用できます。


Visual Studio Code。openHAB VS Code Extension

アパートのホームオートメーションで最も重要なエグゼクティブデバイスは、MegaD-2561有線コントローラーでした。ライトをオン/オフし、すべてのセンサーとメーターから読み取り値を受け取るのは彼です。

これは、ピアに比べてかなり低い価格です3,500ルーブル。（2017年末現在）コントローラーの場合、操作に2つの追加モジュールが必要です。たとえば、次のとおりです。

  • 最初のモジュール：7つの標準入力、7つのリレー出力0-220V（7 * 2300W / 10A）：〜3 000ルーブル（2017年末）。
  • 2番目のモジュール：14のユニバーサルハードウェア構成可能な入力+ 1つのリレー出力、ボタンとデジタルセンサーの両方を接続する機能I2C、1線など：〜3 000ルーブル（2017年の終わりに）;

私のアパートには2つのセットがインストールされています。つまり、これらは2つのコントローラーと4つの追加モジュールです。

その比較的低価格を見ると、これがホームオートメーションに最適なデバイスであると判断できますが、これは完全に正しいわけではありません。これは主にオタクのDIYデバイスであり、初期設定と物理的な接続に十分な時間と忍耐力がない場合、それはあなたのためではありません。また、Xiaomi Mi Homeのように、MegaD-2561はそのままでは機能しません。

そして、あなたのアパートや家の自動化があなたによってではなく、特別な組織によって行われているなら、それはあなたがこのデバイスを提供される可能性が低いです。

しかし、私は自分ですべてを理解し、同時に「この」デバイスが適切な設定で提供できる「大人の」機能を手に入れたいという欲望と時間を持っていました。私が最初に見たKNXによると、私はそのような価格の機器のみを数えていました。私は最終的に、すべての修理、家具、およびオートメーション用の機器やセンサーを含むすべての電気工事に支払いました。KNXによれば、設置と構成を行わない場合の機器の価格のみでした。
