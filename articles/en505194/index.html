Windows Defender SmartScreen harms independent developers

Suppose you are an independent developer and have released your application. To simplify the installation, you create an installer and start distributing it. Some courageous experimenter downloads, launches the installer - but sees a strict warning:


Windows SmartScreen blocks the launch of the executable file

Indeed, in the current environment, Windows actively blocks the launch of binary files through the SmartScreen function.

But what is a smartscreen?


SmartScreen collects installation data from all Windows users to determine the ‚Äúreputation‚Äù of the application. If the program does not have a good reputation, you receive such a warning. Most users in this situation will delete .exe, considering it to be malware, but you can bypass SmartScreen by clicking on the "Details" button and then on "Run Anyway".

Digitally Signed Racket


But how to build a reputation? First of all, Microsoft should be able to collect information about who published the application, and this is done using a code signing certificate. The most obvious conclusion is that unsigned applications will always call SmartScreen. A more insidious conclusion is that acquiring a code signing certificate is a big expense for an individual developer. There is currently no equivalent of Let's Encrypt for signing a code, so you will have to buy a certificate from reputable certification authorities. The price range is wide, but a one-year certificate usually costs about $ 100.


SSL.com offers one of the ‚Äúcheapest‚Äù code signing certificates on the market, priced at $ 129 per year.

Suppose you pecked at a hook, bought yourself an overestimated number of primes multiplied by a computer, signed your code, and re-published your application. Now users will begin to install your application, right? Wrong.

Building a reputation is a trick 22


Even with a digital signature, SmartScreen will still work. After all, you are an unknown new publisher, and the SmartScreen ‚Äúreputation building‚Äù process is an absolute black box. So, here's the trick 22: to create a reputation, you need people to install your software - and Microsoft will collect data with installation statistics. But in order to get users to install your software, you need to get rid of a message that clearly suggests that your code will damage their computers.

And then worse.

Let's say you are finally approved by the algorithms as a reliable publisher. Your certificate will expire and you will renew it. A common thing on the Internet. But in the world of Windows software, your reputation will be nullified. SmartScreen does not recognize certificate updates, it considers you a new publisher, and you have to go through this painful process again.

This can be mitigated by buying a certificate with a longer validity period, but then the cost becomes just an insurmountable barrier for most independent developers.

EV Code Signing Certificates


For independent developers, this is the end of the road. For publishers, there is another solution: an advanced verification certificate for code signing (EV). By signing the code using one of them, you automatically bypass SmartScreen. It is reserved only for corporations and, as the name implies, provides for additional verification of information. It is conducted by certification centers.

The popular certification authority Digicert.com sells such certificates for a whopping $ 699 per year.

An independent developer will have to go through the difficulties of forming a legal entity and fork out at outrageous prices for these certificates. Of course, for most this is not realistic.
