About the death of dual boot and the unity of Windows and Linux

My work machine used to be a laptop created by Apple. I could do almost anything on it: develop programs, write texts, compose music, and much more. But I was haunted by the thought that I was attached to the ecosystem of Apple, that I was dependent on the whims of this company. So I started looking for something new.



I began to assemble a workstation for machine learning tasks. I put in it, among other things, an excellent processor, a lot of memory, a decent video card. Almost all of my tasks I solved in Ubuntu. True, I needed Microsoft Office to work with texts. Online Office had not yet appeared, and let's call a spade a spade, LibreOffice is just some kind of horror. For me, the solution was the double boot in the Ubuntu configuration - Windows 10. I really liked the feeling of freedom that you experience when switching from an OS from Apple to Ubuntu. And the possibilities that open to those who assemble their own computer are almost endless.







Dual boot for a long time completely suited me. And when I used it a million times, the WSL (Windows Subsystem for Linux, Windows Subsystem for Linux) technology appeared. When this happened, I started to solve some of my Linux problems on Windows. True, even so, I still did not have much for full work. But now, with the release of WSL 2, I get the feeling that the new version of WSL is able to radically change the situation. Today I propose to talk about how, using WSL 2, to transfer the task of developing programs from Linux to Windows 10. I will talk about the new features of WSL 2, and what can be expected from this subsystem in the future.



WSL 2 Review



WSL 2 is the new version of the Windows subsystem for Linux. There are some changes in this version that determine how Linux distributions interact with Windows.









Microsoft loves Linux.



In this WSL release, the user has improved file system performance and full compatibility with system calls. Of course, we have the ability to run Linux either on WSL 1 or on WSL 2, and, moreover, you can switch between different versions of WSL at any time. WSL 2 is the result of a major improvement in the basic architecture of the system; its new features are based on virtualization technology and on the functionality of the Linux kernel. At the same time, Microsoft takes care of the small details, so the WSL user can simply go about their business without distraction.



Installation



Microsoft promises that in the near future, installing WSL 2 will be very easy, and that Linux can be updated using the Windows update mechanism. In the meantime, the process of installing WSL 2 requires some effort, but, in fact, there is nothing to worry about.



In this example, we will install Ubuntu 20.04 on Windows 10. It should be noted that the installation process will be the same for all Linux distributions available in the Microsoft Store. First you need to enable the Windows Subsystem for Linux component. To do this, open PowerShell as administrator and run the following command: 



dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart


Next, you need to upgrade WSL to WSL 2. For this, Windows 10 must be updated to version 2004. Intel virtualization technology must be enabled in the BIOS. Again, use PowerShell with administrative privileges and run the following command: 



dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart


To complete the WSL installation and upgrade to WSL 2, restart the computer. Next, you need to install WSL 2 as the default version of WSL when installing new Linux distributions. To do this, all the same, using PowerShell with administrator privileges, we execute this command: 



wsl --set-default-version 2


After you run this command, the following message may appear: 



WSL 2 requires an update to its kernel component. For information please visit https://aka.ms/wsl2kernel








Message displayed when trying to install WSL 2 as the default Windows subsystem for Linux



Follow the link and install the appropriate .msi file, thanks to which the Linux kernel for WSL 2 will be installed on your machine. After the kernel is installed, run the above command again. Now it should complete successfully without giving a similar message.



Now all that remains is to install the required Linux distribution. To do this, open the Microsoft Store and look for Ubuntu 20.04 LTS there . After installing the distribution, a shortcut should appear on the Start menu to start Ubuntu. Start the system and follow the instructions to complete the installation (in general, the completion of the installation is to create a new user).



In order to check whether the distribution is really installed on WSL 2, run the following command: 



wsl --list --verbose


If it turned out that you are using WSL 1, then you can switch to WSL 2 using this command: 



wsl --set-version <distribution name> <versionNumber>


That's all. You now have a complete Ubuntu distribution running on Windows 10.



Setting up a working environment for a programmer



Now that you have working Ubuntu at your disposal, you can install everything you may need. For example, if you are a data scientist, you can install the most recent Anaconda distribution. If you are a front-end developer, then you, for example, may be interested in Angular, npm and much more. Here I would like to focus on two tools. This is Visual Studio Code and a bunch of Docker + Kubernetes.

VS Code is a code editor that many developers prefer. One of the strengths of this editor is its support for an infinite number of extensions. And now that we have included WSL 2, Remote Development can be called an absolutely necessary extension for VS Code .



This extension allows you to remotely work on code that is available in an environment created by WSL 2, in a container, or even on a remote virtual machine accessed via SSH. This extension allows, for example, creating a project in Linux running WSL 2 and using the VS Code editor installed in Windows 10 to work on this project.



As a matter of fact, VS Code differs in huge opportunities. This is the IntelliSense intelligent system, and integration with git, and a debugger, and a terminal, and a lot more. I believe that you will manage to configure VS Code and very soon you will be able to make exactly what you need from this editor.

WSL Development Plans



WSL 2 still lacks some features, but the Windows subsystem for Linux is developing in the right direction. In the near future, you can install WSL with a single command. To do this, it will be enough to open the Windows terminal and do the following in it: 



wsl.exe --install


In addition, WSL 2 will be the version of the subsystem that, when you first install WSL, is installed by default.



It should be noted that developers are looking forward to the following features from WSL: support for graphics cards and support for applications with a graphical interface.



Since the release of WSL 1, the development team of this subsystem has most often asked to implement CUDA or GPU Compute support in WSL. In the latest code, the development team of WSL, virtualization systems, DirectX, Windows Driver are working on this feature. Therefore - stay tuned.
