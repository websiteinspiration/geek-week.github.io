Denoとは何ですか？Node.jsを置き換えますか？



良い一日。あなたにI現在の記事の翻訳「電王対 Node.js- Louis Petrikによる最も重要な違いは次のとおりです。

Denoは新しいNode.jsですか？それとも良い代替案ですか？この記事では、これらの質問に答えてみます。

デノの歴史


DenoとNode.jsは一人の仕事です-Ryan Dahl

Dahlは2009年からNode.jsに取り組んでいますが、数年後にチームを去りました。 2018年、彼は「Node.jsについて後悔する10のこと」というプレゼンテーションを行いました。この講演では、DahlがJavaScriptとTypeScriptの新しい「エンジン」であるDenoを紹介しました。

楽しい事実：「Deno」は「Node」のアナグラムです。

開発者は、Denoが近い将来Node.jsを置き換えることを恐れるべきでしょうか？

ダール自身はそうではないと信じている。ビデオへの



リンク。

Dahlが彼のスピーチで述べているように、Denoは完全性（安定性）においてNode.jsよりも劣っています。

ただし、どちらもブラウザの外部、たとえばWebサーバー上のJSコードランタイムです。しかし、Node.jsとDenoの間の「関係」が数年でどのように発展するかを誰も言うことはできません。

DenoとNode.js：内部デバイス


Node.jsはC ++で記述され、V8を使用してJSコードを実行します。 V8は、ブラウザでJSコードをできるだけ早く実行できるようにGoogle Chrome用に特別に設計されました。現在、Microsoft Edgeを含むほとんどすべてのブラウザーがV8に切り替えられています。

DenoもV8に基づいていますが、C ++の代わりにRustを使用しています。Rustは、C ++に劣らない優れたパフォーマンスだけでなく、セキュリティ：メモリアクセスエラーやバッファオーバーフローを排除することも目標としています。

セキュリティはNode.jsの主な欠点の1つです。アプリケーションを起動すると、ファイルシステムまたはネットワークにアクセスできるようになるためです。

Denoは、ユーザーにアクセス許可を要求することで、これを修正しようとします。

良い例は、Denoの公式Webサイトからの次のTSコードです。

import { serve } from 'https://deno.land/std@0.50.0/http/server.ts'

const s = serve({ port: 8000 })

for await (const req of s) {
	req.respond({ body: 'Hello World\n'})
}

ご覧のとおり、これは次のように起動する非常に単純なWebサーバーです。

deno run app.ts

ただし、Denoにはネットワークへのアクセス許可が付与されていないため、次のメッセージが表示されます。



アクセスを提供するには、次のことを行う必要があり

deno run --allow-net app.ts

ます。ただし、許可するまで、アプリケーションはファイルシステムにアクセスできません。

さようならNPM、こんにちはES6インポート


私たちのアプリケーションのもう1つのセキュリティ問題はNPMです。NPMを使用すると巨大な「node_modules」フォルダーが作成されるだけでなく、アプリケーションを動作させるインストール済みパッケージにも理論上は悪意のあるコードが含まれる可能性があります。

DenoはNPMを使用しません。代わりに、例に示すように、ライブラリはURL経由でインポートされます。

import { serve } from 'https://deno.land/std@0.50.0/http/server.ts'

ライブラリは最初の実行時に読み込まれ、キャッシュされます。



これは、Webサーバーのコードをコピーしてサーバーまたはコンピューターに個別に保存して再利用する方法と似ています。ライブラリをリロードする必要はありません。

ブラウザ外のウィンドウオブジェクト


ウィンドウは、いくつかの非常に重要なJS関数を含むブラウザのグローバルオブジェクトです。

そのような機能の1つはフェッチです。FecthはNode.jsでは使用できません。これには特別なライブラリがありますが、Denoでは使用できます。

これは、ドキュメントから次のように、DenoがWindowにアクセスできるという事実によるものです。

次のコードは、ブラウザとDenoの両方で実行できます。

fetch('https://jsonplaceholder.typicode.com/todos/1')
	.then(response => response.json)
	.then(data => console.log(data))

すぐに使えるTypeScriptサポート


もちろん、TypeScriptの使用は好みの問題です。好きな人もいれば、そうでない人もいます。とはいえ、DenoはそのままでTypeScriptをサポートし、もちろんJavaScriptもサポートします。

これを確認するには、コードを記述し、ファイルを.tsまたは.jsとして保存し、で実行しdeno runます。TypeScriptのサポートをNode.jsに簡単に追加できるため、これは重要な利点ではないと私は考えています。
