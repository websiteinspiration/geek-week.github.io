二分探索木




ツリーの外観は、配列の要素が最初にどのように配置されるかに大きく依存します。ランダムデータに基づくと、確率の高いツリーは、最適なツリーと比較してレベル数がはるかに多く、不均衡であることがわかります。この例では、7つのレベルが形成されました。配列の要素を配置した場合、例えば、この順序で：

25、28、29、23、30、27、24、22、21、26

、探索木は、レベル（= 4）の最小可能数と、バランスされるであろう。バランスの取れたツリーを使用してソートすると、時間は複雑になります：O（n log n）。しかし、不均衡なツリーの場合、それはO（n 2）に劣化する可能性があります。

以前のhabrastatiaでは、バイナリ検索ツリーによる並べ替えと、その興味深い変更により、時間の複雑さがO（n log n）の保証された並べ替えを可能にすることが検討されていました。

検索ツリーのバランスが悪いほど、それを走査するのにコストがかかります。

一方、検索ツリーにデータを格納すると、組織の観点から非常に便利です。これにより、ソートなど、このデータを使用して多くの操作を実行できます。

二分探索木の親は、必ずしも子供以上のものではありません。ただし、同時に、左の子は常に親よりも小さく、右の子は常に親よりも大きい（または等しい）。

検索ツリーを作成するときの配列の最大要素は、右側のサブツリーのどこかにあります。この最大値を抽出するには、検索ツリーを完全に回避する必要があります。

バイナリヒープ


通常のヒープでは、親子関係は非常に単純に編成されており、要素のインデックス自体に関連付けられています。



元々ランダム配列に基づいて構築されたそのようなツリーはまだソートされていないため、図は実際にはまだ束ではありません。各要素を回って、ふるいにかける必要があります。その結果、配列の要素が再構築され、各親がその子孫よりも大きくなります。



最大の要素はツリーのルートにあります。その場所はわかっているので、探す必要はありません。一方、ヒープから最大値を抽出する場合（代わりに、配列のその他の要素をツリーのルートに配置する必要がある場合）、ヒープはソートツリーではなくなり、再度ふるいにかける必要があります。

最終的なアルゴリズム：

  • I.ソートされていない配列に基づいて、デカルトツリーを作成します。
    • I.1。配列の要素を左から右に繰り返します。
    • I.1。これが配列の最初の要素である場合、これは単にデカルトツリーの最初のノードです。
    • I.2。これが配列の最初の要素でない場合は、これを1つの要素で構成されるデカルトツリーと見なします。このツリーを、前の要素から構築されたツリーとマージします。
  • II。現在の最大要素をツリーから削除し、その後、デカルトツリーを復元します。
    • II.1。現在の最大要素は、ツリーのルートです。
    • II.2。ツリーからルートを削除し、最大の要素を追加の配列に転送します。
    • II.3。ルートを削除すると、構造は2つのデカルトツリーに分割されます。
    • II.4。これらの2つのツリーに対して、マージ操作を実行します。新しい現在の最大値は、結合されたツリーのルートの1つです。
    • II.5。新しい高値の場合は、ステップII.1-II.4を再度繰り返します。




C ++での実装：

/*
 * c++ 11 code to construct a Cartesian tree. The method cartesianTreeSort
 * will sort the contents of the array.
 */

#include <iostream>
#include <vector>
#include <queue>

struct Node{
    int value;
    Node *left, *right;
    Node *parent;

    Node(){
        value = 0;
        parent = NULL;
        left = NULL;
        right = NULL;
    }
};

// Used by priority queue
struct compare{
    bool operator()(Node *left, Node *right){
        return left->value > right->value;
    }
};

class CartesianTree{
private:
    // last pointer to keep track of last node added
    Node *root, *last;

    Node * findLowestNode(Node *node, int x){
        if(node->value < x)
            return node;
        else if(node->parent != NULL)
            return findLowestNode(node->parent, x);
        else
            return NULL;
    }

public:
    Node * getRoot(){
        return root;
    }

    void addNode(int x){
        Node *new_node = new Node;
        new_node->value = x;
        if(root == NULL){
            last = new_node;
            root = new_node;
            return;
        }
        Node *z = findLowestNode(last, x);
        if(z == NULL){
            new_node->left = root;
            root->parent = new_node;
            root = new_node;
        }
        else{
            new_node->left = z->right;
            z->right = new_node;
            new_node->parent = z;
        }
        last = new_node;
    }

    CartesianTree(std::vector<int> ar){
        root = NULL;
        last = NULL;
        // Call addNode function for each element of the array
        for(auto x : ar){
            addNode(x);
        }
    }

    void InorderTraversal(Node *node){
        // To print inorder traversal of the tree
        if(node == NULL)
            return;
        InorderTraversal(node->left);
        std::cout << node->value << ' ';
        InorderTraversal(node->right);
    }

    // Function to sort and store values in array
    void cartesianTreeSort(std::vector<int> &sorted_ar){
        // Initialize input array
        sorted_ar.assign(0, 0);

        // Initialize priority queue
        std::priority_queue<Node *, std::vector<Node *>, compare> p_queue;
        p_queue.push(root);
        Node *temp = NULL;
        while(!p_queue.empty()){
            temp = p_queue.top();
            p_queue.pop();
            sorted_ar.push_back(temp->value);
            if(temp->left){
                p_queue.push(temp->left);
            }
            if(temp->right){
                p_queue.push(temp->right);
            }
        }
    }
};

int main(){
    std::vector<int> ar = {1, 14, 5, 0, 8};

    CartesianTree tree(ar);
    std::cout << "Inorder Traversal\n";
    tree.InorderTraversal(tree.getRoot());
    std::cout << '\n';

    std::vector<int> sorted;
    tree.cartesianTreeSort(sorted);
    std::cout << "Sorted array is\n";
    for(auto x : sorted)
        std::cout << x << ' ';
    std::cout << '\n';
}


複雑


最小コストのデカルトツリーは、配列の並べ替えられたセクションとほとんど並べ替えられた（昇順または降順では関係ありません）セクションを処理します。たとえば、ここでは逆順配列のプロセスです。



アルゴリズムの最も高価な部分はMerge操作です。これは、n要素のそれぞれに対して2回実行する必要があります。ランダムデータでは、1回のMerge操作のコストはO（log n）ですが、配列の順序付けられた領域にある要素の場合、O（1）操作のコストです。

したがって、ソートの平均および最悪の時間の複雑さはO（n log n）であり、最適なものはO（n）です。

追加のメモリによると、物事は悪いです-O（n）デカルトツリーが一種のヒープである場合、コストはO（1）になります。ただし、デカルトツリーはバイナリ検索ツリーでもあるため、n個のすべての要素について、親子関係を個別に保存する必要があります。

さらに、この形式での実装は、それが外部ソートであることを意味します-ソートされた要素は別の配列に収集されます。そして、これは追加のメモリのO（n）です。
