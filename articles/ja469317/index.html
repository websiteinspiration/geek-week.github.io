プレセールにおけるPMaの財務：MS Projectで予算とキャッシュフローをすばやく計算する方法

プロジェクトに取り組むことを決定する前に会社の管理者が回答する必要がある主な質問の中で、次のことが際立っています。

  1. 提案されている価格で議論中のプロジェクトを実行することは有益ですか？
  2. プロジェクト中の財務状況はどのようになりますか？キャッシュギャップはありますか？

もちろん、これはRMaのような頭痛の種ではないと考えることもできますが、会社が小規模の場合、会社のすべての財務上の問題が給与の遅延という形で最も直接的にあなたとチームに影響を及ぼします。会社がしっかりと立ち上がったとしても、プロジェクトがネガティブになった場合、チームの雰囲気に必然的に悲しみの要素がもたらされます。したがって、プロジェクトの利益という意味では、わずかで1回限りの利益があり、利益は大きくて一定であるため、最初は財務的な部分を考慮に入れることをお勧めします。したがって、リーダーのリーダーとしてのカルマです。まあ、またはプリセール段階で財政的に失敗したプロジェクトを認識し、可能な限りそれらに参加しないようにする機能。


興味がある-猫へようこそ

あなたが始める前に。

この記事を書いているとき、先輩から育った平均的なRMが財務の詳細をしっかりと掘り下げることを絶対に望んでいないことが考慮されました。したがって、プレセールの財務計算を最小限の時間と精神的な努力で得る方法が示されています。たとえば、技術系大学で「エコロジー」という科目の単位を取得するなど、生態学者は私を許します。

タスクの入力、タスク間の接続の構築、階層の編成など、MS Projectでスケジュールを自分で実行できることが前提となっています。私たちが話していることは、オンラインで「正確な」質問に答えることができるプレセールの実用的な財務モデルを取得するための、MS Projectでの財務情報の迅速な準備と入力です。

記事には金銭的な影響はなく、すべて標準であり、2003年以降、MS Projectのどのバージョンでも使用できます。記事ではMS Project 2019を使用しましたが、インターフェースに大きな違いはありません。悲しいかな、作品は英語版を使用しています。私はリスクを冒さず、この記事を書くためだけにロシア語に変更しました。
要点に行きましょう。

パート1。料金と将来の費用


Linuxで鉄の真空中で球形の馬 を開発するプロジェクトの事前販売を見てみましょう。これは、顧客の重要なパラメーターを監視し、受け取ったデータの分析に基づいて制御アクションを発行します。開発には、ボード、システム、アプリケーションソフトウェアの開発が含まれ、FPGAは含まれません。ケースは準備されました。私たちの目標は、キャッシュギャップに入らないように、利益のパーセンテージを考慮し、プロジェクトの支払いスケジュールを評価して、適切な金額を顧客に展開することです。

この教育プロジェクトに必要な専門家：

  1. プログラマー。
  2. 回路技術者。
  3. ボードブリーダー。
  4. テスター
  5. 実装者。
  6. あなた自身が首相です。あなたの時間もお金がかかります。

これには、秘書、簿記、弁護士、購入者、販売者、オフィスレンタル、電気、クッキー付きのお茶は直接含まれません-それはあなたに依存せず、管理することはできません。あなたの力のすべては、これにコストを追加することです、これは開発者の時給の増加という形でです。これが今から行うことです。

個人的には、私はこの場合、給与に「正味」を2倍するだけの方が好きです。はい、それはとても簡単です。同社は手持ち給与に対して43％の税金を支払い、残りの57％は間接費として支払われるためです。これらの残りの57％には、最終的なレートに影響する暗黙の詳細も含まれています。28日間の休暇があり、開発者は定期的に病気であり、時間を求めています-彼らは年間10.5ヶ月働いていると想定できます。取締役/金融業者が57％に同意しない場合、これは間違いなく頭痛の種ではありません。会社の経理部門全体にアクセスできず、原則としてアクセスできないため、取締役/金融業者に番号を提供するよう依頼してください。ただし、現時点では設定されていません-57を入力してください％

計算例：適用されたプログラマーは、手あたり82 000ルーブルを受け取ります。平均164労働時間（休日を含む）、つまり 彼の時間給は82,000 / 164 = 500ルーブルです。1000r /時のレートを設定します。すべて。

続行する前に、間接費を計算する別の方法を説明する価値があります。すでにお気づきのように、オフィス、Cookie、および弁護士の諸経費はジュニアとシニアで同じであるため、請求書を給与の％として計算することは完全に正しくありません。したがって、財務担当者がその期間の1時間あたりの平均オフィス経費数を計算するようにディレクターに依頼することができます（年/四半期/月-重要ではありません。時間数で割ります）。次に、たとえば、請求書の300r / hと給与（税込み）の時間ごとの費用を別々に82,000 * 1.43 / 164 = 715r / hの数値を取得します。これはより正確になります。715+ 300 = 1015r /になります。時間ここで唯一の大きな問題は、簿記からその期間のオーバーヘッドコストの正確な量を取得することが非常に難しいことです。私は一度だけ成功しましたしたがって、給与に関心を持たせることを好みます。はい、そしてプロジェクトは主に中程度の給与のミドルによって引っ張られているため、2つの会計方法の違いはそれほど大きくありません。最後に、ここでも給与請求書の57％を投入することに同意できます。また、（より正確には）経理部門が1時間あたりの請求書番号を提供する場合-いくつかのレートをすばやく変更すると、強制から5分かかり、すべてが自動的に再計算されます。
