HackTheBox。ウォークスルーTraverxec。nostromo WebサーバーのRCE、GTFOBinsテクニック


HackTheBox サイトからさらに処理するために送信されたソリューションを引き続き公開しています。

この記事では、nostromo WebサーバーでRCEを取得し、アクティブなメタスプロイトセッションの下からmeterpreterシェルを取得し、nostromo configを調べ、SSHキー暗号化パスワードを調整し、GTFOBinsテクニックを使用して特権を増やします。

実験室への接続はVPN経由です。情報セキュリティの分野で何かを知っている人とプライベートネットワークに行き着くため、仕事用のコンピュータや、重要なデータが利用できるホストから接続しないことをお勧めします。

偵察


このマシンのIPアドレスは10.10.10.165で、/ etc / hostsに追加します。

10.10.10.165    traverxec.htb

まず、開いているポートをスキャンします。nmapですべてのポートをスキャンするには時間がかかるため、最初にmasscanでこれを実行します。tun0インターフェースからすべてのTCPおよびUDPポートを毎秒1000パケットの速度でスキャンします。

masscan -e tun0 -p1-65535,U:1-65535 10.10.10.165 --rate=1000



ここで、ポートで動作するサービスの詳細については、-Aオプションを指定してスキャンを実行します。

nmap -A traverxec.htb -p22,80



したがって、SSHとNostromo Webサーバーがあります。

Nostromo Webサーバーでバージョン1.9.6の既知のエクスプロイトを確認してみましょう。これらの目的のために、searchsploitを使用できます。このプログラムは、exploit-dbデータベースにあるエクスプロイトを便利に処理する機能を提供します。



結果からわかるように、2番目のエクスプロイトは適していません。最初のエクスプロイトにはMetasploitマークが付いています。これは、このモジュールがMetasploit Frameworkコンテキスト用に実装されていることを示しています。

エントリーポイント


msfconsoleをダウンロードして、エクスプロイト検索を実行します。



これで、エクスプロイトの完全な名前がわかり、使用できるようになりました。エクスプロイトを読み込んだ後、その情報を確認します。したがって、その使用の基本的なオプションとエクスプロイト自体の説明を見つけます。





いいね！このエクスプロイトは、私たちのバージョンのnostromoに適しています。基本的なオプションを設定し、エクスプロイトの動作を確認します。



便宜上、meterpreterシェルを取得します。これを行うには、バックグラウンドで動作するようにセッションを送信します-Ctrl + Zで確定します。そして、作業セッションを見ると、最小化されたセッションが表示されます。



実行中のセッションでmeterpreterシェルを取得するには、操作後モジュールpost / multi / manage / shell_to_meterpreterを使用できます。このモジュールを次のようにすばやく使用します。



Meterpreterシェルが2番目のセッションで開いていることが通知されます。したがって、2番目のセッションを開始します。



ユーザー


そして、あなたがホストを見回す必要がある最初の事柄。これを行うには、Linuxの列挙スクリプトをマシンにロードします。



次に、シェルを呼び出し、スクリプトを実行する権利を与えます。



出力を分析すると、.htpasswdからパスワードハッシュが見つかります。



さらに回転させることは時間の無駄です...これを実現し、どこへ進むべきかわからないため、Webサーバーの構成を調べることにしました。幸い、ディレクトリはこのファイルへのパスに示されています。



次に、nhttp.confファイルを確認します。



実際、nostromoのルートディレクトリは、設定ファイルでhomedirs_publicとして指定されています。つまり、これはpublic_wwwディレクトリです。このディレクトリは、ユーザーのホーム（homedirs-/ home）ディレクトリ（serveradmin-david@traverxec.htb）にあります。上記を要約すると、index.htmlファイルは/ home / david / public_wwwディレクトリにあります。ユーザーがこのページにアクセスできるようにするには、ディレクトリにすべてのユーザーの読み取り権限が必要です。



推測が確認されました。さらに、ある種のディレクトリが表示されているので、そこに何があるかを見てみましょう。



そこでSSHファイルのバックアップを見つけます。それらをダウンロードしてください。



ファイルを解凍します。



また、秘密のユーザーキーがありますが、接続しようとするとパスワードの入力を求められます。これは、キーが暗号化されていることを意味します。



しかし、スクロールすることができます。最初に、目的の形式にします。



ハッシュをファイルに保存し、JohnTheRipperを使用してrockyou.txt辞書をスクロールします。



したがって、鍵暗号化パスワードを見つけます。それとキーを使用して、SSH経由で接続します。
