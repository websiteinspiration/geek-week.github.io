気づかないかもしれない最新のJavaScriptの機能



過去7年間、JavaScriptでほぼ毎日稼働しているという事実にもかかわらず、ESの革新に関するニュースにはほとんど注意を払っていないことを認めざるを得ません。async / awaitやプロキシなどの主な機能は1つですが、毎年もっと重要なことを研究する必要があるため、毎年私の視野に当てはまらない小さな段階的な変化の流れがあります。

この記事では、最近のJSの機能についてまとめました。これらの機能は、登場時にほとんど話題になりませんでした。それらのいくつかは使いやすさを向上させるだけであり、いくつかは信じられないほど実用的であり、たくさんのコードを書く手間を省くことができます。

ES2015


2進および8進リテラル


多くの場合、JavaScriptはバイナリ操作を使用する必要はありません。しかし、他の方法では解決できないタスクが発生する場合があります。たとえば、弱いデバイス用の高性能コードを作成する場合、ビットをローカルストレージに圧縮するか、ブラウザーでRGBピクセル操作を実行するか、または密にパックされたバイナリデータ形式で作業します。

これには、2進数の非表示/結合に多くの作業が必要になる場合があります。彼らは小数で隠されているのは無駄だといつも思っていました。ES6でこのような場合のために、フォーマットのバイナリリテラルを追加しました0b。

const binaryZero = 0b0;
const binaryOne  = 0b1;
const binary255  = 0b11111111;
const binaryLong = 0b111101011101101;

これにより、バイナリフラグの操作が大幅に簡略化されます。

// Pizza toppings
const olives    = 0b0001;
const ham       = 0b0010;
const pineapple = 0b0100;
const artechoke = 0b1000;

const pizza_ham_pineapple = pineapple | ham;
const pizza_four_seasons  = olives | ham | artechoke;

8進数でも同じです。JSの世界では、これはニッチな機会ですが、ネットワークや一部のファイル形式によく使用されます。構文を使用して8進数を書き込むことができます0o。

Number.isNaN（）


と混同しないでくださいwindow.isNaN()。これは、はるかに直感的な動作を備えた新しい方法です。

クラシックにisNaNはいくつかの興味深いトリックがあります：

isNaN(NaN)        === true
isNaN(null)       === false
isNaN(undefined)  === true
isNaN({})         === true
isNaN('0/0')      === true
isNaN('hello')    === true

これは私たちに何を与えますか？まず、これらのパラメーターはどれも実際にはありませんNaN。いつものように、問題はすべての「お気に入りの」JavaScriptプロパティである型キャストです。の引数はwindow.isNaN、関数を使用して数値にキャストされますNumber。

この問題は、新しい静的メソッドによって解決されNumber.isNaN()ます。彼はかつて彼に渡された議論の平等を返しますNaN。これは完全に明らかです：

Number.isNaN(NaN)       === true
Number.isNaN(null)      === false
Number.isNaN(undefined) === false
Number.isNaN({})        === false
Number.isNaN('0/0')     === false
Number.isNaN('hello')   === false

署名： Number.isNaN : (value: any) => boolean

Array.prototype.includes（）


見逃すのは大変でしたが、過去3年間に書いた場合array.indexOf(x) !== -1は、新しい方法で喜んでincludesください。

[1, 2, 3].includes(2)    === true
[1, 2, 3].includes(true) === false

includes同じ値ゼロアルゴリズムを使用します。これは===、値を処理できることを除いて、厳密な等価性（）のチェックとほぼ同じですNaN。このアルゴリズムは、コンテンツではなくリンクによってオブジェクトも比較します。

const object1 = {};
const object2 = {};

const array = [object1, 78, NaN];

array.includes(object1) === true
array.includes(object2) === false
array.includes(NaN)     === true

includesfromIndexシフト値を提供できるようにする2番目のパラメーターを使用できます。

// positions   0  1  2  3  4
const array = [1, 1, 1, 2, 2];

array.includes(1, 2) === true
array.includes(1, 3) === false

有用

署名： Array.prototype.includes : (match: any, offset?: Int) => boolean

ES2018


正規表現の金鉱


ES2018には、多くの新しい正規表現機能があります。

後読み一致（前の文字との一致）


これをサポートするランタイムでは、一致する前に文字を探す正規表現を記述できるようになりました。たとえば、ドル記号が前に付いているすべての数値を検索するには、次のようにします。

const regex = /(?<=\$)\d+/;
const text  = 'This cost $400';
text.match(regex) === ['400']

新しい先読みグループであるツイン先読みグループについてです。

Look ahead:  (?=abc)
Look behind: (?<=abc)

Look ahead negative:  (?!abc)
Look behind negative: (?<!abc)

残念ながら、今日は新しい後読み構文を古いブラウザに変換できないため、しばらくの間Nodeでしか使用できない可能性があります。

名前付きキャプチャグループ


正規表現でサブサンプルを選択して、簡単な解析に使用できるようになりました。最近まで、このようなフラグメントは番号でのみ参照できました。次に例を示します。

const getNameParts  = /(\w+)\s+(\w+)/g;
const name          = "Weyland Smithers";
const subMatches    = getNameParts.exec(name);

subMatches[1]     === 'Weyland'
subMatches[2]     === 'Smithers'

そして今、これらのサブサンプル（またはレコードグループ）に名前を割り当てるための構文があります。?<titl>グループに名前を付けたい場合は、最初の角括弧内に配置します。

const getNameParts  = /(?<first>\w+)\s(?<last>\w+)/g;
const name          = "Weyland Smithers";
const subMatches    = getNameParts.exec(name);

const {first, last} = subMatches.groups
first             === 'Weyland'
last              === 'Smithers'

残念ながら、これはChromeとNodeでのみ機能します。

ドットは新しい行をマークできるようになりました


フラグのみを置く必要があり/s、たとえば、/someRegex./s、/anotherRegex./sg。

ES2019


Array.prototype.flat（）およびflatMap（）


MDNでこれを見てとても嬉しかったです。

簡単に言うflat()と、指定された最大深度（depth）で多次元配列を1次元配列に変換します。

const multiDimensional = [
    [1, 2, 3],
   [4, 5, 6],
   [7,[8,9]]
];

multiDimensional.flat(2) === [1, 2, 3, 4, 5, 6, 7, 8, 9]

flatMap-このmap後にflat深さ1 が続きます。これは、配列を返す関数をマップする必要があるが、結果がネストされたデータ構造である必要がない場合に便利です。

const texts = ["Hello,", "today I", "will", "use FlatMap"];

// with a plain map
const mapped = texts.map(text => text.split(' '));
mapped === ['Hello', ['today', 'I'], 'will', ['use', 'FlatMap']];

// with flatmap
const flatMapped = texts.flatMap(text => text.split(' '));
flatMapped === ['Hello', 'today', 'I', 'will', 'use', 'FlatMap'];
