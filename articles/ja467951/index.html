Habrastatistics：読者のコメントを分析します。パート2、質問への回答

こんにちはHabr。

で前の部分、このサイトのユーザーのメッセージは、様々なパラメータ（メッセージの数、評価、「カルマ」など）の話題ではなく、活発な議論を引き起こした、分析しました。そのような質問は、2番目の部分を作成するのに十分に蓄積されています。



今年のコメントで最大の議論の長さ、ユーザーの「カルマ」の最大値と最小値、その他の統計に興味がある人は、猫にお願いします。

データ収集


まず、パーサーを補足して、より多くのデータを収集する必要がありました。メッセージの評価とスレッド内のメッセージの位置に関する情報はすでにHTMLに含まれていますが、ユーザーの「カルマ」を取得するために、追加のリクエストが行われました。もちろん、同じユーザーが何度もデータを要求しないように値はキャッシュされました。

更新されたデータセットは次のようになります（ユーザーのニックネームは削除されました）。 追加のフィールドから、回答フィールド（回答数）、ユーザーの「カルマ」、および正と負の評価が追加されました。 たとえば、ユーザーのニックネームによって「カルマ」を取得する関数は、次のようになります。

https://habr.com/ru/post/322900/,comment_19707920,comment_19706258,UserXXXX,karma:112.2,answers:1,2019-02-04 20:26:00,rating:1,up:1,down:0, ?
https://habr.com/ru/company/mailru/blog/351212/,comment_19794710,comment_19794310,UserXXXX,karma:-10.0,answers:1,2019-02-23 18:16:00,rating:3,up:5,down:-2,
...





def get_karma(user: str):
        data_html = get_as_str("https://habr.com/ru/users/%s/" % user)
        karma = data_html.find_between('info/help/karma/', '</a>').find_between('stacked-counter__value', '/div>').find_between('>', '<').replace(",", ".").replace('–', '-')
        return float(karma) if len(karma) > 0 else 0.0

カルマ


前のパートで示したように、「カルマの質問」が最も興味深いので、それから始めましょう。ご存じのように、「カルマ」はこのサイトのアクティブユーザーの属性であり、参加者は増加または減少を選択できます。率直に言って、それが実際にどのように機能し、どのような制限があるのか​​については決して質問しなかったので、特別なコメントなしでデータを提供します。

合計で、評価を収集した時点のサイトでは、25,109人のユーザーが少なくとも1回はコメントを残したユーザーです。これらのうち、9973人のユーザー（39％）のカルマはありません。12,346人のユーザー（49％）が正のカルマを持ち、そのうち5384人のユーザー（21％）がカルマ<= 4、> = 40がカルマを持っています（このレベルでは、「報酬プログラム」に参加して、記事の支払いを受け取ることができます）には、1,522人のユーザー（6％）がいます。2790人のユーザー（11％）が負のカルマを持っています。

ディスカッション


次の興味深い質問は、メッセージの中で最も長いスレッドを見つけることでした。これを行うのは非常に簡単であることがわかりました。HTMLの各メッセージには一意の識別子があり、data-parent_idパラメータもあります。残りは技術的な問題です。

したがって、2019年のディスカッションの中で最も長いがらくた：619回答、618回答、614回答、556回答、553回答。興味深いことに、これらのスレッドの原因となった5つのメッセージのうち4つは、カルマが-20未満のユーザーによって書き込まれました。

コメントへの回答数に応じてコメントをグループ化します。
