メッセージはトランザクションです


ブロックチェーンでのトランザクションがトークン（コイン）をあるユーザーから別のユーザーに転送するという事実には誰もがすでに慣れています。ビットコインのように。メッセージを送信するための特別なタイプのトランザクションを作成しました。

ブロックチェーンのメッセンジャーでメッセージを送信するには、いくつかの段階を経る必要があります：

  1. メッセージテキストを暗号化する
  2. 暗号文をトランザクションに入れる
  3. トランザクションに署名
  4. トランザクションを任意のホストに送信します
  5. ノードの分散システムがメッセージの「信頼性」を決定します
  6. すべて問題なければ、メッセージを含むトランザクションは次のブロックに含まれます。
  7. 受信者はメッセージトランザクションを取得して復号化します

ステージ1〜3および7はクライアントでローカルに実行され、5〜6はネットワークノードで実行されます。

メッセージの暗号化


メッセージは送信者の秘密鍵と受信者の公開鍵で暗号化されます。ネットワークから公開鍵を取得しますが、そのためには受信者のアカウントを初期化する必要があります。つまり、少なくとも1つのトランザクションが必要です。RESTリクエストを使用できGET /api/accounts/getPublicKey?address={ADAMANT address}ます。チャットをダウンロードするとき、対話者の公開鍵はすでに利用可能です。



メッセンジャーは、curve25519xsalsa20poly1305アルゴリズム（NaCl Box）を使用してメッセージを暗号化します。アカウントにはEd25519キーが含まれているため、ボックスを形成するには、まずキーをCurve25519 Diffie-Hellmanに変換する必要があります。

JavaScriptの例を次に示します。

/**
 * Encodes a text message for sending to ADM
 * @param {string} msg message to encode
 * @param {*} recipientPublicKey recipient's public key
 * @param {*} privateKey our private key
 * @returns {{message: string, nonce: string}}
 */
adamant.encodeMessage = function (msg, recipientPublicKey, privateKey) {
  const nonce = Buffer.allocUnsafe(24)
  sodium.randombytes(nonce)

  if (typeof recipientPublicKey === 'string') {
    recipientPublicKey = hexToBytes(recipientPublicKey)
  }

  const plainText = Buffer.from(msg)
  const DHPublicKey = ed2curve.convertPublicKey(recipientPublicKey)
  const DHSecretKey = ed2curve.convertSecretKey(privateKey)

  const encrypted = nacl.box(plainText, nonce, DHPublicKey, DHSecretKey)

  return {
    message: bytesToHex(encrypted),
    nonce: bytesToHex(nonce)
  }
}

トランザクション署名


送信者と受信者の信頼性を全員が確認できるように、送信時とメッセージの内容で、トランザクションに署名します。デジタル署名により、公開鍵を使用してトランザクションの信頼性を検証できます。これには秘密鍵は必要ありません。

2-データブロックのSHA-256を考慮します

/**
 * Creates hash based on transaction bytes.
 * @implements {getBytes}
 * @implements {crypto.createHash}
 * @param {transaction} trs
 * @return {hash} sha256 crypto hash
 */
adamant.getHash = function (trs) {
  return crypto.createHash('sha256').update(this.getBytes(trs)).digest()
}

3-取引に署名します

adamant.transactionSign = function (trs, keypair) {
  var hash = this.getHash(trs)
  return this.sign(hash, keypair).toString('hex')
}

/**
 * Creates a signature based on a hash and a keypair.
 * @implements {sodium}
 * @param {hash} hash
 * @param {keypair} keypair
 * @return {signature} signature
 */
adamant.sign = function (hash, keypair) {
  return sodium.crypto_sign_detached(hash, Buffer.from(keypair.privateKey, 'hex'))
}

トランザクションの検証


ノードの分散コンセンサスベースのシステムは、メッセージトランザクションの「信頼性」を決定します。誰から、誰に、いつ、メッセージが別のメッセージに置き換えられたか、送信時刻が正しく表示されたか。これはブロックチェーンの非常に重要な利点です。チェックを担当する中心的な構造がなく、メッセージのシーケンスとその内容を偽造することはできません。

最初に、1つのノードが信頼性をチェックし、それを他のノードに送信します。ほとんどがすべてが正常であると言う場合、トランザクションはチェーンの次のブロックに含まれます-これはコンセンサスです。



チェックを担当するノードコードの部分は、GitHub- validator.jsおよびverify.jsで確認できます。ええ、ノードはNode.jsで実行されます。

メッセージを含むトランザクションをブロックに含める


コンセンサスに達した場合、メッセージを含むトランザクションは、他の信頼できるトランザクションとともに次のブロックに分類されます。

ブロックには厳密なシーケンスがあり、後続の各ブロックは、前のブロックのハッシュに基づいて形成されます。



結論としては、メッセージもこのシーケンスに含まれており、「再配置」することはできません。複数のメッセージがブロックに分類される場合、それらの順序はtimestampメッセージによって決定されます。

メッセージを読む


メッセンジャーアプリケーションは、宛先に送信されたトランザクションをブロックチェーンから取得します。このため、エンドポイントを作成しましたapi/chatrooms。

すべてのトランザクションは誰でも利用できます-暗号化されたメッセージを受信できます。ただし、受信者だけが自分の秘密鍵と送信者の公開鍵で復号化できます。

**
 * Decodes the incoming message
 * @param {any} msg encoded message
 * @param {string} senderPublicKey sender public key
 * @param {string} privateKey our private key
 * @param {any} nonce nonce
 * @returns {string}
 */
adamant.decodeMessage = function (msg, senderPublicKey, privateKey, nonce) {
  if (typeof msg === 'string') {
    msg = hexToBytes(msg)
  }

  if (typeof nonce === 'string') {
    nonce = hexToBytes(nonce)
  }

  if (typeof senderPublicKey === 'string') {
    senderPublicKey = hexToBytes(senderPublicKey)
  }

  if (typeof privateKey === 'string') {
    privateKey = hexToBytes(privateKey)
  }

  const DHPublicKey = ed2curve.convertPublicKey(senderPublicKey)
  const DHSecretKey = ed2curve.convertSecretKey(privateKey)
  const decrypted = nacl.box.open(msg, nonce, DHPublicKey, DHSecretKey)

  return decrypted ? decode(decrypted) : ''
}
