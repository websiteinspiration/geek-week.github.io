Algoritmo EM en general


En el curso, en funci√≥n del cual se escribi√≥ el art√≠culo, el algoritmo EM se presenta como uno de los m√©todos de agrupaci√≥n. En otras palabras, este es un m√©todo de aprendizaje autom√°tico sin un maestro, cuando no sabemos las respuestas verdaderas de antemano. En nuestro art√≠culo, tambi√©n consideraremos el algoritmo como parte de uno de los m√©todos de agrupaci√≥n de una variable aleatoria con una distribuci√≥n normal. Sin embargo, debe tenerse en cuenta que el algoritmo tiene una aplicaci√≥n m√°s amplia. El algoritmo EM (maximizaci√≥n de expectativas en ingl√©s) es un m√©todo general para encontrar estimaciones de la funci√≥n de probabilidad en modelos con variables ocultas, que a partir de una mezcla de distribuciones le permite construir distribuciones de probabilidad complejas (aproximadas).

El algoritmo EM en problemas de agrupamiento se utiliza como un algoritmo iterativo, que realiza dos pasos en cada iteraci√≥n:

E-step. En el primer paso E, de alguna manera, por ejemplo, al azar, seleccionamos variables ocultas, en nuestro caso ser√° una expectativa matem√°tica:Œº y desviaci√≥n est√°ndar -œÉ . Usando las variables seleccionadas, calculamos la probabilidad de asignar cada objeto a un grupo particular. Los pasos E posteriores utilizan las variables ocultas definidas en los pasos M.

M-step. En el paso M, de acuerdo con las probabilidades de asignar cada objeto a un grupo particular obtenido en el paso E, recalculamos las variables ocultasŒº yœÉ

iteraciones se repiten hasta que ocurre la convergencia. Podemos suponer que la convergencia ocurri√≥ cuando los valores de las variables ocultas no cambian significativamente, por ejemplo, dentro de una constante dada. En el √∫ltimo ejemplo, que se considera en el art√≠culo, no estableceremos constantes y, en consecuencia, no determinaremos cambios en los valores de las variables ocultas en cada iteraci√≥n. Hag√°moslo m√°s simple: limitaremos nuestro algoritmo a un n√∫mero fijo de pasos.

El paso M es lo suficientemente simple como para entenderlo, y lo veremos en el √∫ltimo ejemplo. Nos centraremos en el paso E del art√≠culo.

El paso E se basa en el teorema bayesiano, en base al cual determinamos la probabilidad de que cada objeto sea asignado a uno u otro grupo. Recordemos de qu√© trata este teorema.

Recordemos la f√≥rmula de Bayes


Pxi(Bj)=P(Bj)‚ãÖPBj(xi)P(xi)



d√≥nde Pxi(Bj) - la probabilidad de que con el eventoxi una hip√≥tesisBj

P(Bj) - probabilidad de hip√≥tesisBj

PBj(xi) - probabilidad de ocurrencia de un eventoxi sujeto a la hip√≥tesisBj

P(xi) - probabilidad de ocurrencia de un eventoxi siempre que cada hip√≥tesis se cumple (calculado por la f√≥rmula para la probabilidad total del evento)

A condici√≥n de que el eventoxi ya ha ocurrido, la probabilidad de que se cumpla la hip√≥tesisBj reval√∫a utilizando la f√≥rmula anterior. Podemos decir esoP(Bj) es la probabilidad a priori de la hip√≥tesis (antes de la prueba), yPxi(Bj) es la probabilidad posterior de la misma hip√≥tesis evaluada despu√©s del eventoxi , es decir, teniendo en cuenta que el eventoxi fiable sucedi√≥.

def gaus_func_01(mu,sigma,x):
    return math.e**(-(x-mu)**2/(2*sigma**2)) / (sigma*(2*math.pi)**0.5)

#        
def proba_x(X, w1, w2, mu_1, mu_2, sigma_1, sigma_2):
    for i in X:
        P1_x = gaus_func_01(mu_1,sigma_1,i)
        P2_x = gaus_func_01(mu_2,sigma_2,i)
        P_x = w1*P1_x + w2*P2_x
        P_x_1 = (w1*P1_x)/P_x
        P_x_2 = (w2*P2_x)/P_x
        proba_temp = []
        proba_temp.append(P_x_1)
        proba_temp.append(P_x_2)
        proba_X.append(proba_temp)
    return proba_X

#        
def pred_x(proba_X, limit_proba):
    pred_X = []
    for x in proba_X:
        if x[0] >= limit_proba:
            pred_X.append(1)
        else:
            pred_X.append(2)
    return np.array(pred_X)

def gaus_func_02(k, m, x, mu, sigma):
    pj_xi = []
    for j in range(k):
        det_sigma_j = np.linalg.det(sigma[j])
        factor_1 = 1 / (((2 * math.pi)**(k/2)) * ((det_sigma_j)**0.5))
        factor_2 = []
        for i in x:

            -0.5 * np.matrix(i - mu[j]) * np.matrix(np.linalg.inv(sigma[j])) * np.matrix(i - mu[j]).T))
        pj_xi.append(factor_1 * np.array(factor_2))
    return np.array(pj_xi)

Conoce el algoritmo EM


Continuamos nuestro ejemplo con las piezas que llegaron al almac√©n. Pero esta vez solo sabremos que los productos se fabricaron en dos m√°quinas diferentes, hay 10,000 de ellos, cada parte tiene un di√°metro y un tama√±o, y no sabemos nada m√°s. Pero la tarea no ha cambiado: nosotros, como antes, de toda la pila grande, elementos aleatoriamente mezclados, tendremos que determinar a qu√© m√°quina pertenece esta o aquella parte.

A primera vista, parece poco realista, pero de hecho tenemos en nuestras manos un poderoso conjunto de herramientas: la f√≥rmula de Bayes y la f√≥rmula de densidad de probabilidad de una variable aleatoria. Tomemos toda esta bondad.

Soluci√≥n

¬øQu√© haremos? Como deber√≠a ser en el algoritmo EM, primero inicializamos los par√°metros: la

probabilidad de la hip√≥tesis es extraer la parte producida en la m√°quina No. 1 -w1 Determinamos la probabilidad de una hip√≥tesis para extraer la pieza producida en la m√°quina No. 2 - w2. Solo hay dos hip√≥tesis, lo que significa que cada una de ellas en el primer paso ser√° igual a 0.5.

Expectativa matem√°tica de variables aleatoriasŒºdefinir de la siguiente manera. Mezclamos todos los productos usando la funci√≥n aleatoria, dividimos el conjunto en dos partes iguales, para cada parte para cada par√°metro (di√°metro, peso) determinamos el valor promedio.

Desviaci√≥n Est√°ndarœÉtoma lo que se llama desde el techo: ponlo igual a la unidad en todos los aspectos. Escribimos en el formato de la matriz de covarianza.

Estamos listos para hacer el primer paso E del algoritmo. Usando los par√°metros inicializados de variables aleatorias, determinamos la probabilidad de que cada parte se asigne a la m√°quina No. 1 o a la m√°quina No. 2.

En realidad, de esta manera, tomamos el primer E-step.

Ahora depende del paso M. Todo es simple aqu√≠. Despu√©s de determinar la probabilidad de que cada parte se produzca en una m√°quina en particular, podemos volver a calcular la probabilidad de cada hip√≥tesis:w1, w2y Œº y œÉ.

Haremos 15 de tales iteraciones, en dos pasos cada una.

#   E-
def e_step(x, k, m, n, w, mu, sigma):
    #      i-     j-  
    pj_xi = []
    for j in range(k):
        det_sigma_j = np.linalg.det(sigma[j])
        factor_1 = 1 / (((2 * math.pi)**(k/2)) * ((det_sigma_j)**0.5))
        factor_2 = []
        for i in x:
            factor_2.append(math.e**float(
                -0.5 * np.matrix(i - mu[j]) * np.matrix(np.linalg.inv(sigma[j])) * np.matrix(i - mu[j]).T))
        pj_xi.append(factor_1 * np.array(factor_2))
    pj_xi = np.array(pj_xi)
    
    #     ,  i-    j- 
    pj_xi_w = []
    for j in range(k):
        pj_xi_w.append(pj_xi[j] * w[j])
    pj_xi_w = np.array(pj_xi_w)
    
    #     i-      
    sum_pj_xi_w = np.sum(pj_xi_w, axis = 0)
    
    #         
    proba_xi = []
    for j in range(k):
        proba_xi.append(pj_xi_w[j] / sum_pj_xi_w)
    
    return np.array(proba_xi)

def m_step(x, proba_xi,N):
    w_new = np.sum(proba_xi, axis = 1) / N
    
    #   
    mu_new = (np.array((np.matrix(proba_xi) * np.matrix(X))).T / np.sum(proba_xi, axis = 1)).T
    
    #  
    cov_new = []
    for mu in range(mu_new.shape[0]):
        X_cd = []
        X_cd_proba = []
        count = 0
        for x_i in x:
            cd = np.array(x_i - mu_new[mu])
            X_cd.append(cd)
            X_cd_proba.append(cd * proba_xi[mu][count])
            count += 1
        X_cd = np.array(X_cd)
        X_cd = X_cd.reshape(N, m)
        X_cd_proba = np.array(X_cd_proba)
        X_cd_proba = X_cd_proba.reshape(N, m)

        cov_new.append(np.matrix(X_cd.T) * np.matrix(X_cd_proba))
    cov_new = np.array((np.array(cov_new) / (np.sum(proba_xi, axis = 1)-1)))
    #             ,    
    #       ,        
    if cov_new[0][0][1] < 0:
        cov_new[0][0][1] = 0
    if cov_new[0][1][0] < 0:
        cov_new[0][1][0] = 0
    
    if cov_new[1][0][1] < 0:
        cov_new[1][0][1] = 0
    if cov_new[1][1][0] < 0:
        cov_new[1][1][0] = 0
    
    #   
    sigma_new = cov_new**0.5
    return w_new, mu_new, sigma_new
