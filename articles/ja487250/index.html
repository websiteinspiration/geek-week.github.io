遅延アルファブレンディング

この記事では、ラスタ化されたジオメトリを混合する方法について説明します。半透明オブジェクトの従来のミキシングモデル-Alpha、Additive、Multiplicative-は、同じ描画原理によって統合されています。フラグメントシェーダーの出力で受け取ったピクセルを現在のバッファーにあるものと混合して、プリミティブを次々に描画します。新しいプリミティブはそれぞれ、それが描画されるバッファーの領域を更新します。アルファブレンディングの場合、より高いアップシェードのオブジェクトは以前に描画されたものです。しかし、シーンの上部に描画されたオブジェクトのグループを使用して何かを実行したい場合はどうでしょうか。たとえば、マスクでオブジェクトをトリミングしたり、オブジェクトをハイライトしたりしますか？ここで2つの決定がすぐに思い浮かびます。たとえば、別のテクスチャの投影を追加するなど、マテリアルに変更を加える（つまり、シェーダーを変更する、テクスチャのセットを拡張する）か、これは透明マスクを担当します。ただし、まだら状のオブジェクトが多数ある場合、それぞれの固有のマテリアルを変更するのは不便であり、エラーが発生します。 2番目のオプションは、関心のあるすべてのオブジェクトを別のフルスクリーンターゲットに描画し、それを最終シーンに既に描画することです。ここでは、そのコンテンツを使用して必要なことをすべて実行できますが、これには追加のメモリの割り当てが必要であり、最も不愉快なことに、ターゲットレンダリングを切り替える必要があります。これは、モバイルデバイスでの「最も簡単な」操作ではなく、2回実行する必要があります。そして、あなたがこのようにいくつかのレイヤーで作業したいなら？ここでは、そのコンテンツを使用して必要なことをすべて実行できますが、これには追加のメモリの割り当てが必要であり、最も不愉快なことに、ターゲットレンダリングを切り替える必要があります。これは、モバイルデバイスでの「最も簡単な」操作ではなく、2回実行する必要があります。そして、あなたがこのようにいくつかのレイヤーで作業したいなら？ここでは、そのコンテンツを使用して必要なことをすべて実行できますが、これには追加のメモリの割り当てが必要であり、最も不愉快なことに、ターゲットレンダリングを切り替える必要があります。これは、モバイルデバイスでの「最も簡単な」操作ではなく、2回実行する必要があります。そして、あなたがこのようにいくつかのレイヤーで作業したいなら？

トリックは何ですか？ 


リバースレンダリング、または遅延混合と呼ばれるテクノロジーの本質は次のとおりです。別のブレンド式を使用してシーンを逆方向に描画します。さらに、最終的なイメージは、従来のアプローチとまったく同じままです。

使い方？


私たちが描くイメージの透明度チャネルを介して混合する方法は、上で説明されています。今、私たちはそれを逆にします：私たちはすでに描かれたピクセルの透明度を使用します（またはむしろ、描かれた透明度を既に描かれたものと混合します）。つまり、AlphaSrcの代わりにAlphaSaturateを使用し、OneMinusAlphaSrc-Oneの代わりに使用します。バッファに透明度= 1のものがすでにある場合、寄与はゼロになり、そのようなピクセルの色は変化しないことがわかります。透明度がゼロの場合は、両方の色を一緒に追加しましょう（これを行うには、ゼロでフレームバッファーをクリアするか、透明度がゼロである黒をクリアする必要があります）。この追加により、結果の色は描画された色と等しくなります。最終的な式は次のようになります。
 

（約AlphaSaturate = min（AlphaSrc、1-AlphaDst））

追加する必要がある透明度の値：レイヤーごとに累積する必要があります。つまり、アルファチャネルのミックス変数にOneとOneがあります。ColorDstを変更して、バッファをゼロでクリアしてみませんか？これは、加算混合に必要です。AdditiveBlendingは、AlphaSrc変数にゼロがある点でのみ異なります。透明度は変更せず、色のみを変更してください。

わかりやすくするために、逆レンダリングスキームは次のようになります。 

まず、フレームバッファーをクリアします。次に、上記の混合関数を設定し、最上位のオブジェクトから描画を開始し（従来のアプローチでは、最後に描画されます）、下のオブジェクトまで続きます。背景画像は最後に描画されます。

Ambient occlusion




欠点、またはむしろ制限があります。そのような手法ですべてのブレンドを繰り返すことができるわけではありません。アルファミキシングと添加剤は間違いなく可能ですが、独自の特別なブレンドを採用するか使用しない必要があります。ただし、解決策があります。シーンのレンダリングの段階を分離できます。それの一部は逆の方法で行われるべきであり、それの一部は通常のものであり、これはフィールドとポストプロセスの上に特殊効果を与えるために私たちが行ったものです。

アディティブレンダリングと混合レンダリングのテクニックの重要なポイント：逆レンダリングのパッセージの前に描画され、テクスチャに透明度情報がない場合（「黒い背景に白い点」のようなテクスチャ）、そのようなオブジェクトは透明度を上書きします。 「リターン」の通路では、このセクションに関する情報は失われ、視覚的には「暗い四角」または明るい加点の周りの黒い境界線のように見えます。


これは、アルファチャネルの混合に関して加法混合を変更することで克服できます。

AlphaSrc = GL_ONE_MINUS_DST_ALPHA
AlphaDst = GL_ONE

つまり、カラーチャネルの明るさを透明度に変換し、透明度とは逆に色を引き伸ばす必要があります。結果の古いテクスチャは、黒の背景で同じに見えるはずです。手動でこれが成功する可能性は低く、自動コンバーターを作成することは理にかなっています。この場合、チャネル変換疑似コードは次のようになります。

RGB_old = Texel_in.rgb
A_old = Texel_in.a
A_middle = 1.0 / ((RGB_old) / 3.0) * A_old // linear color space
RGB_new = RGB_old * A_middle;
A_shift = minimum( 1.0 / RGB_new.r, 1.0)
A_shift = minimum( 1.0 / RGB_new.g, A_shift)
A_shift = minimum( 1.0 / RGB_new.b, A_shift)
RGB_result = RGB_new * A_shift; 
A_result = (RGB_result) / 3.0)
Texel_out = Vector4(RGB_result, A_result)
