新的Azure考试

在过去的三个月中，微软宣布了对认证和考试的许多更改。本文中对其中一些进行了描述。

本文不重点介绍全新的考试，而是针对现有考试的更新版本：管理员使用AZ-104，开发人员使用AZ-204，建筑师使用AZ-303和AZ-304。

2020年1月（距离Ignite2018首次宣布基于角色的考试仅1.5年，距az-203和az-103出现1年零1个月），微软宣布对开发人员，管理员，架构师和ai工程师进行考试变更（后者不如前三个重要）。

Beta版中已经提供了某些考试（az-204 / az-104），稍后将提供体系结构考试。可以在2020年8月底之前通过旧的（当前）考试（由于Covid-19而推迟了截止日期）

，似乎微软从上一次宣布中获悉了“轰炸”和“焚烧”非常辛苦的时候。特别是，在引入新考试之后，不会引入新证书：由于它们是管理员/开发人员/架构师，因此仍然存在。通常，乘以零不会发生，对于外部观察者来说，好像什么都没有改变，因为以前，证书是可见的，而不是考试。

在本文中，我将告诉您考试中有哪些变化。

社区很少喜欢更改考试，因为 每次这会将他们以前的考试乘以零（即使不是完全正确，社区仍然会意识到这一点）。但是在这里，“并非一切都那么简单。”

让我们继续学习AZ-104

AZ-103 / AZ-104


与Az-204不同，Az-204的整个部分都被去除了，并且重排很小，而Az-104中则是连续的重排。

除了订阅管理部分，我们现在拥有一个引以为傲的治理功能，即 订阅，资源组以及有关身份管理的一些知识。

隐藏文字

“存储”部分包含以下更改：

  • 从存储中删除了备份（这是合理的，因为它不是存储），并将其移至单独的备份和监视部分
  • 他们删除了数据框，这也是正常的，因为 很少有人在实践中与他一起工作，并且在考试中对他的询问绝对不清楚。
  • 删除了CDN，因为 它也不存储。
  • 删除了活动日志，因为 这也不是特定于存储的内容，移至单独的部分进行备份和监视

隐藏文字

虚拟机部分已转换为计算部分，现在不仅包含虚拟机，而且还包含托管Kubernetes集群，容器实例和Web应用程序

隐藏文字

同时，DSC也被删除，尽管最后它可能被伪装成自定义脚本扩展名。当然，DSC是一项有趣的技术，但是却很琐碎，在Linux更大的世界中，它并不是那么有趣（以我个人的观点）。

网络部分已得到大量补充。显然出现了Azure应用程序网关，Azure堡垒，Azure WAN。

隐藏文字




出现了一个单独的监视部分，该部分以前是“ Azure资源利用”的一部分，坦率地说，这与本质不符。

隐藏文字


身份/ Azure Active Directory

非常重要的是，Azure Active Directory部分失去了作用并发生了变化。我宁愿支持这个决定，因为 好吧，不是每天我们都配置MFA（通常是一次针对整个租户的操作，然后不是每个人都使用天蓝色的MFA）。通过广告连接进行同步也很好，因为他们删除了同步，因为 尽管许多客户都在使用它，但他们通常将其设置为1次并且不再使用它（99％的管理员从未见过，尽管他们一直在全天候管理azure）总的来说

，就我的主观观点而言，对知识的要求并没有太大变化，但是结构内容变得更加合乎逻辑。

迈向最大的变革-建筑师考试。

AZ-300和AZ301-> AZ-303 / AZ-304


要通过建筑认证，您必须通过2项设计与实施考试。从这个意义上说，什么都没有改变：因为有两次考试，所以它仍然存在。但是，考试的结构已大大修改。我认为，知识和经验的集合实际上并没有改变，但是考试的结构却发生了根本性的变化。结论是根据新的技能度量1和2

得出的。这些部分当然可以让真正的架构师了解，但是它们不是Azure特定的，或者很难直接验证。例如，az-301中有一个部分：“收集要求”。

隐藏文字

在考试中，使用“案例研究”脚本，您可以了解架构师是否可以隔离重要需求。但是，要真正收集需求，您需要进行白板会议，问卷调查，黑客马拉松和研讨会。在测试中，这没有完成。

在301项考试中，他们还“锯掉”了有关创建角色，人员等的部分。根据我在解决方案架构师中的经验，我可以说这些事情对于业务分析是可行的，并且只有当项目中没有任何内容时，您才可以自己做。与Azure无关。

隐藏文字

删除了要求架构师能够编写SQL代码的部分。这个东西很有用，但是有不同的方言，通常，体系结构在哪里？架构师可以扮演Tech Lead项目的角色，但是在这种情况下，有关基本sql语法的问题不是架构师的问题。但是，即使他不是非常庞大的数据专家，高可用性架构师也应该知道。

隐藏文字

SSL / TLS系列中有关安全性的相当抽象的词已删除，并替换为特定服务，例如Azure Key Vault和Azure AD托管身份。

隐藏文字

很高兴在Az-303考试中对迁移部分进行了描述，否则在更新之前太简单了。

隐藏文字

但是在304年，它只是重新制定

网络部分已变​​得更加蔚蓝。实际上，我们被告知要知道的事情。它曾经是非常抽象的：网络安全性，但是其中包含的内容尚不清楚。

隐藏文字

设计数据管理，设计存储策略，数据保护，归档策略（来自业务连续性）已减少到选择合适的数据库和存储帐户。我认为，这不仅仅是内容的变化，而是更多的重新包装。许多点都可以减少，事实如此。

隐藏文字

“设计和文档数据流”部分成为“设计数据集成”，并且服务也直接列出。很好，删除了诸如“创建数据流程图”之类的未经检查的问题。再一次，在我看来，az-301的描述是关于抽象架构师的，在304年，他们决定只写可以验证的内容以及特定于Azure的内容。

隐藏文字

但是授权部分（查看主题列表）已完全更改。以前，它是一个抽象的“ blabla”。在新版本中，他们要求提供Azure AD和Azure的特定服务和功能。尽管作为新问题的一部分，他也可以参与治理。

隐藏文字

实际上，“计算”部分没有改变，只是他们添加了一个有关AKS的项目，但之前在其他部分中没有，因此可以说没有任何变化。

隐藏文字

集成API部分已移至“设计应用程序体系结构”部分。从Az-300中移除的事件/消息（最终定居在Az-304中）落入同一部分。

在那里，在“设计应用程序体系结构”中，移动了“设计部署”部分。

隐藏文字

高可用性似乎并没有改变，只是部分问题与业务连续性有关。

隐藏文字

灾难恢复部分还吸收了备份部分。这变得更加合乎逻辑。

隐藏文字

因此，我们可以自信地说，在Az-303 / AZ-304部分中进行了非常大的重新排列和重构。但是，我认为经过验证的知识的数量没有改变。那些为过去的考试做准备的人应该不会有问题通过新的考试。

如果您查看全部4项考试，那么考试的结构（或至少是技能测量的结构）将得到显着改善。但是，如果您创建了培训课程，那么您将不得不重新设计和重组它们。
