ロシア語の文を解析する

この記事では、文脈自由文法とLR分析アルゴリズムを使用してロシア語の文を解析するプロセスについて説明します。

自然言語処理は、人工知能と数学言語学の一般的な方向性です。コンピュータ分析と自然言語の合成の問題を研究しています。

一般に、自然言語文を分析するプロセスは次のとおりです。（1）文を構文単位に分割します-単語と句。（2）各ユニットの文法パラメータの決定。（3）ユニット間の構文関係の定義。出力は抽象的な構文解析ツリーです。

1.文を構文単位に分割する


自然言語文は、単語形式と強力なフレーズで構成されています。特定の単語のいくつかの単語形式は、パラダイムと呼ばれます。

例えば、

   "": [, , , , ,  ]

語句-複合結合、述語、または安定した表現-は変化せず、意味を失うことなく小さな単位に分解することはできません。さらに、単語とは、あらゆる構文単位-単語形式または句を意味します。

文の各単語は、トリプルによって決定されます。

  1. 単語形式/単語文字列（「書いた」）
  2. 単語の通常形（ "write"）
  3. 文法パラメータのセット（['VERB'、 'sing'、 'musc'、 'tran'、 'past']）

したがって、「明らかに、彼は会議に来ません」という文の内訳は次の形式になります。

[' ', '', '', '', '', '']
 ' ' -  ,

2.文法パラメータ（grammems）の定義


グラムは、文法範疇の要素です。同じカテゴリーの異なる文法は相互に排他的であり、一緒に表現することはできません。各単語形式について、7つの文法のセットを定義します。

3.単語間の構文関係の定義


単語間の構文関係を決定するために、文脈自由文法とLR分析を使用します。

文法とLR分析


形式文法は、いわゆるプロダクションの形式で言語を記述する方法です。例えば：

a -> ab | ac

ルール「a」が「ab」または「ac」を生成することを意味します。

非終端記号は、言語の本質（文、式など）を表すオブジェクトです。ターミナル -文法に対応する言語で直接存在し、特定の不変の意味（文字、単語、数式など）を持つオブジェクト。文脈自由文法は、すべての製品の左側が単一の非終端である文法です。

ロシア語を記述するために、コンポーネントの文法理論（句構造文法）を使用します。これは、複雑な文法単位は、直接コンポーネントと呼ばれる、2つのより単純で交差しない単位で構成されると主張します。以下のコンポーネントが区別されます。

（1）名義グループ（NP）

NP[case='nomn'] -> N[case='nomn'] | ADJ[case='nomn'] NP[case='nomn'] | …

つまり、主格名詞句は、主格の場合の名詞、または主格の場合の形容詞+主格の名詞句、または別のものです。

（2）言語グループ（VP）

VP[tran] -> V[tran] NP[case='ablt'] | ADJ VP[tran] | …

言い換えると、他動詞グループは、他動詞+除去名詞グループOR短い形容詞+他動詞グループOR別のものです。

（3）前置詞グループ（PP）

 PP -> PREP NP[case='datv'] | ...

前置詞グループは、前置詞+名詞の与格グループまたは別のグループです。

（4）完全なオファー（S）

S -> NP[case='nomn'] VP[tran]

名詞と動詞のグループの数、人、性別が一致する場合にのみ、完全な文が存在します。 

  def agreement(self, node_left, node_right):
                       
                       ...
                       if (numb1 and numb2): 
				if (numb1 != numb2): return False;
			if (per1 and per2):
				if (per1 != per2): return False;
			if (gend1 and gend2):
				if (gend1 != gend2): return False;
		return True;

不完全な文は、名目部分が省略された文です。原則として、そのような文では、動詞グループは非人格的な動詞によって表現されます。たとえば、「歩きたい」「軽くなってきました」など。楕円文は、動詞部分を省略してダッシュに置き換えた文です。たとえば、「後ろには森があります。左右に沼があります。」

この文が文法言語に属しているかどうかを判断するために、LR分析アルゴリズムを使用します。このアルゴリズムには、ボトムアップ（葉からルート）までの構文解析ツリーの構築が含まれます。アルゴリズムの重要な要素は、「転送畳み込み」（英語のshift-reduce）の方法です。

（1）いずれかのルールの右側に一致するチェーンが見つかるまで入力行の文字を読み取り、見つかったチェーンをスタックに入れます（転送）。
（2）文法からのルールによって見つかったチェーンを置き換えます（畳み込み）。

すべての文字列チェーンがラップされている場合、この文は文法言語に属し、少なくとも1つの解析ツリーが存在します。

ツリー

二分木は文中で使用され、構文的関係を表します。ここで、葉は一連のグラムを持つ単語（終端）であり、ノードはルール（前終端）です。語根は文（非終端）です。

ツリーノードは次のように定義されます。

class Node:
	def __init__(self, word=None, tag=None, grammemes=None, leaf=False):
		self.word = word; #  
		self.tag = tag; #   - , 
      
		self.grammemes = grammemes; #  
		self.leaf = leaf;
		self.l = None;
		self.r = None;
		self.p = None;

ツリーの構築は、単語またはフレーズの文字列とその一連の文法が割り当てられた葉から始まります。 

	def build(self, sent):
		for word in sent:
			new_node = Node(word[0], word[1], word[2], leaf=True)
			self.nodes.append(new_node)

次に、LR分析が実行されます。各畳み込みは、共通の祖先の下にある2つのノードまたはリーフの和集合に対応します。祖先ノードには、文法規則に対応する前終端子タグが割り当てられます。さらに、祖先は、グループの主要メンバーの文法を受け取ります。たとえば、動詞グループV [tran] PRCL（「にしたい」など）では、推移的な動詞V [tran]から記号が取得されます。 PRCLの粒子からではありません。名詞グループNP [case = 'nomn'] NP [case = 'gent']（たとえば、「father of children」）では、主格の名詞から記号が取得されます。

畳み込みは確立された順序で発生することに注意することが重要です。

	def reduce(self):
		self.reduce_ADJ() # 
		self.reduce_NP() #  
		self.reduce_PP() # 
		self.reduce_VP() # 
		self.reduce_S() #

この順序は、提案の一部のメンバーを「見落とす」可能性を排除するため、重要です。最初に、形容詞は修飾子（例えば、めちゃくちゃ美しい）と一緒に形成され、次に名詞群、前置詞、そして最後に口頭で形成されます。その後、完全な/不完全な文が検索されます。存在しない場合、ツリーにはルートがありません。つまり、文は文法言語に属していません。
