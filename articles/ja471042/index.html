2番目のセクションを続けます。最初より難しいとすぐに言いますが、今回はプログラムのソースコードを提供してくれます。ここ（https://t.me/RalfHackerPublicChat）およびここ（https://t.me/RalfHackerChannel）の議論を忘れないでください。始めましょう。

署名otpのアイコンをクリックします。接続用のアドレスとポートが提供されます。



サーバーに接続して見回します。



プログラムとそのソースコードを読み取ることができないフラグ。ソースを見てみましょう。



分析してみましょう。プログラムはパスワードを引数として受け取ります。



次に、ランダムな16バイトがotp変数に格納されます。



ランダムな名前のファイル（otpの最初の8バイト）がtmpフォルダーに作成され、ランダムな8バイト（otpの2番目の8バイト）がそこに書き込まれます。



何らかの理由で、作成されたファイルが値を読み取り、入力されたパスワードと比較します。



これは、脆弱性が存在する場所です。これは、ファイル内の生成された番号の中間ストレージで構成されます。たとえば、ファイルサイズを0に制限して、パスワード0で書き込みと読み取りを比較すると、次のようになります。

# ulimit -f 0



プログラムを実行します。



エラーが発生します。同じpythonを使用して処理できるため、問題ではありません。

python -c "import os, signal; signal.signal(signal.SIGXFSZ, signal.SIG_IGN); os.system('./otp 0')"
