3Dの概要：Three.jsの基本

Webの複雑さは日々変化しており、その機能は、特に3Dレンダリングと同様に急速に成長しています。3Dテーマに参加し始めたばかりの方-猫へようこそ。



遠くから行こう


WebGLは、ブラウザーで動作する3Dグラフィックスを作成できるJavaScript用のソフトウェアライブラリです。このライブラリは、OpenGLライブラリのアーキテクチャに基づいています。 WebGLは、Cのような構文を持つGLSLシェーダープログラミング言語を使用します。 WebGLは、コードがブラウザーで直接モデル化されるという点で興味深いものです。このため、WebGLはHTML5で導入されたキャンバスオブジェクトを使用します。

WebGL、特にシェーダーの操作は、かなり時間がかかるプロセスです。開発プロセスでは、各点、線、面などを記述する必要があります。これらすべてを視覚化するために、かなり大量のコードを登録する必要があります。開発速度を上げるために、Three.jsライブラリが開発されました。

Three.jsは、WebGLでインタラクティブな3Dグラフィックを作成および表示するための事前定義されたクラスのセットを含むJavaScriptライブラリです。

Three.js for WebGLはjQuery for JavaScriptと同じです。ライブラリは、宣言型の構文を提供し、ブラウザの3Dに関連する頭痛の種を抽象化します。一般的な概要を見て、3Dの世界に不慣れな場合の開始方法を見てみましょう。

Three.jsの詳細


Three.jsライブラリーは、すでに述べたように、WebGLでの作業を容易にします。Three.jsを使用する場合、シェーダーを記述する必要はありませんが（可能性は残ります）、使い慣れた概念で操作することが可能になります。

多くの開発者がライブラリに取り組んでいます。主なイデオロギーと開発者は、創造的な仮名Mr.Doobで知られるRicardo Cobello です。

Three.jsを使用したグラフィックスのモデリングは、シーン、ライト、カメラ、オブジェクト、およびそれらのマテリアルなどの概念を操作する機会があるため、セットと比較できます。

3つのいわゆるThree.jsクジラは次のとおりです。

  • シーン -作成するすべてのオブジェクトが配置される一種のプラットフォーム。
  • カメラ -実際には-これはシーンに向けられる「目」です。カメラは、ステージ上にあるオブジェクトを取得して表示します。
  • レンダラー -カメラでキャプチャしたシーンを表示できるビジュアライザ。

Three.jsにはいくつかのタイプのカメラがあります。

  • 遠近法カメラ
  • ステレオカメラ
  • 正射カメラ
  • キューブカメラ

最も一般的なものは、パースビューカメラと正投影カメラです。

遠近法カメラ


これは、3Dシーンのレンダリングに使用される最も一般的な投影モードです。

パースビューカメラは、人間の目が見ているものをシミュレートするように設計されています。カメラはすべてのオブジェクトを透視投影で認識します。つまり、オブジェクトが遠くにあるほど小さく見えます。

Orthographic Camera


この投影モードでは、表示された画像内のオブジェクトのサイズは、カメラからの距離に関係なく一定のままです。つまり、オブジェクトから無限の距離にあるリモートカメラです。

この場合、すべての垂直線は垂直のままで、すべて平行-平行です。カメラを動かしても、線やオブジェクトが歪むことはありません。

これは、2DシーンとUI要素を表示するときに役立ちます。

ステージでオブジェクトを作成する


ステージで作成されたオブジェクトはメッシュと呼ばれます。

Meshは、三角形のポリゴンメッシュに基づくオブジェクトを表すクラスです。

このクラスは2つの引数を取ります。

  • 形状 -形状（頂点の位置、面、半径など）を説明します
  • マテリアル -オブジェクトの外観（色、テクスチャ、透明度など）を説明します

立方体と球体の2つの単純な形状を作成してみましょう。

まず、サイトthree.jsにアクセスし、ライブラリの最新バージョンをダウンロードします。次に、ライブラリをヘッドセクションまたはドキュメントのボディセクションの先頭に接続します。これで完了です。

<!DOCTYPE html>
<html>

    <title>First Three.js app</title>
    <style>
      body { margin: 0; }
      canvas { width: 100%; height: 100% }
      </style>
  </head>
  <body>
    <script src="js/three.js"></script>
    <script>
      //  Javascript .
    </script>
  </body>
</html>

さらに、作成したオブジェクトを表示するには、シーンを作成し、カメラを追加して、レンダリングを構成する必要があります。

シーンを追加：
var scene = new THREE.Scene();

パースビューカメラを追加します。

var camera = new THREE.PerspectiveCamera( 75, window.innerWidth / window.innerHeight, 0.1, 1000 );

カメラは、上記の4つのパラメータを取ります。

  • 画角またはFOV、ここでは標準の75度です。
  • 2番目のパラメーターはアスペクト比またはアスペクト比です。
  • 3番目と4番目のパラメーターは、カメラからの最小距離と最大距離であり、レンダリングに含まれます。

レンダーを追加して設定します。

var renderer = new THREE.WebGLRenderer();
renderer.setSize( window.innerWidth, window.innerHeight );
document.body.appendChild( renderer.domElement );

何をしたか：最初にレンダーオブジェクトを作成し、次に表示領域のサイズに従ってサイズを設定し、最後にそれをページに追加して、操作する空のキャンバス要素を作成します。

レンダリングを作成したら、キャンバスタグを表示する場所を指定します。この例では、bodyタグに追加しました。

立方体自体を作成するには、まずジオメトリを設定します。

var geometry = new THREE.BoxGeometry( 10, 10, 10);

キューブは、BoxGeometryクラスを使用して作成されます。これは、立方体の頂点と面を含むクラスです。私たちはサイズを転送します：

  • width：キューブの幅、x軸に沿った辺のサイズ
  • 高さ：立方体の高さ、つまり Y側のサイズ
  • depth：立方体の深さ、つまり Z軸の辺のサイズ

立方体に色を付けるには、材料を設定します。

var material = new THREE.MeshBasicMaterial( { color: 0x00ff00 } );

この例では、MeshBasicMaterialが設定され、カラーパラメータ0x00ff00が渡されます。緑色。基本的にこの素材は、フィギュアに均一な色を与えるために使用されます。欠点は、図が深く消えることです。ただし、このマテリアルは{wireframe：true}パラメータを使用してワイヤフレームをレンダリングするときに非常に役立ちます。

次に、ジオメトリを取得してそれにマテリアルを適用するMeshオブジェクトが必要です。

var cube = new THREE.Mesh( geometry, material );
scene.add( cube );

camera.position.z = 25;

シーン にメッシュを追加し、カメラとキューブが同じ点になるため、シーンにadd。）（）メソッドが座標（0,0,0）でデフォルトで追加された後、カメラを移動します。

立方体をアニメーション化するには、requestAnimationFrameを使用してレンダーループ内にすべてを描画する必要があります。

function render() {
        requestAnimationFrame( render );
  cube.rotation.x += 0.01;
  cube.rotation.y += 0.01;      
  renderer.render( scene, camera );
}
render();

requestAnimationFrameは、何かをアニメーション化したいブラウザへのリクエストです。呼び出す関数、つまりrender（）関数を渡します。

ここでは、回転速度のパラメータを設定します。その結果、ループはシーンを毎秒60回レンダリングし、立方体を回転させます。


次に球を描きましょう：

var geometry = new THREE.SphereGeometry(1, 32, 32);

球を作成するには、SphereGeometryクラスを使用します。

  • 半径（デフォルト値は1）
  • widthSegments-水平セグメント（三角形）の数。最小値は3、デフォルト値は8です。
  • heightSegments-垂直セグメントの数。最小値は2、デフォルト値は6です。

ちなみに、三角形の数を指定すればするほど、球の表面は滑らかになります。

次に、別のマテリアル-MeshNormalMaterial -RGBカラーの法線ベクトルを比較するマルチカラーマテリアルを使用してみます。

var material = new THREE.MeshNormalMaterial();
var sphere = new THREE.Mesh( geometry, material );
scene.add( sphere );
camera.position.z = 3;

素材の種類はたくさんあります。一部のマテリアルは、組み合わせて1つの図に同時に適用できます。詳細はこちら。

最後のステップは、レンダリングサイクルを設定することです。

function render() {
	requestAnimationFrame( render );
	sphere.rotation.x += 0.01;
  sphere.rotation.y += 0.01;
  renderer.render( scene, camera );
}

より複雑な図を作成して、より複雑なマテリアルを適用してみましょう。

例として、照明を考慮したマテリアルMeshPhongMaterialを取り上げます。したがって、最初にシーンにライトを追加する必要があります。以下に黄色の色合いのスポットライトを追加し、座標軸上の位置を設定します。

var scene = new THREE.Scene();
var spotLight = new THREE.SpotLight(0xeeeece);
spotLight.position.set(1000, 1000, 1000);
scene.add(spotLight);
var spotLight2 = new THREE.SpotLight(0xffffff);
spotLight2.position.set( -200, -200, -200);
scene.add(spotLight2);

上記のSpotLightは、円錐に沿って1点から1方向に放射し、光源から離れるにつれて広がります。スポットライトは、色に加えて、強度、距離、角度、周縁部、減衰、および影付けの引数を取ることができます。

他の種類のライトとその機能については、こちらをご覧ください。

次に、形状自体を定義します。

var geometry = new THREE.TorusGeometry( 10, 3, 16, 100 );

TorusGeometry クラスは、トーラスまたは「ローラー」を構築するように設計されています。このクラスは、次のパラメーターを受け取ります。

  • 半径、デフォルトは1。
  • パイプの直径、デフォルトは0.4;
  • radiusSegmentsまたは三角形のセグメントの数。デフォルトは8です。
  • 管状セグメントまたは面セグメントの数、デフォルトは6

材料を追加：

var material = new THREE.MeshPhongMaterial( {
color: 0xdaa520,
specular: 0xbcbcbc,
 } );

この材料は、光沢のある表面を対象としています。黄金色にして、鏡面特性を追加し、素材の光沢と色に影響を与えます。デフォルトの色は0x111111-濃い灰色です。

Three.jsの機能についてもう少し


プロジェクトにThree.jsを含めるには、npm install threeを実行するだけです。WebpackまたはBrowserify

を使用してファイルを結合すると、ブラウザーでrequire（ 'modules'）を実装でき、すべての依存関係を結合できるので、モジュールをソースファイルにインポートして、通常モードで引き続き使用できます。

var THREE = require('three');

var scene = new THREE.Scene();
...

ES6構文インポート を使用することもできます。

import * as THREE from 'three';

const scene = new THREE.Scene();
...

または、たとえばSceneのように、Three.jsライブラリの特定の部分のみをインポートする場合：

import { Scene } from 'three';

const scene = new Scene();
...
