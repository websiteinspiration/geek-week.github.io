Oracle VM Server for SPARC for Dummies（ハウツー）

初心者がSolarisおよびOracle VM Server for SPARC仮想化プラットフォーム（以前はSun Logical Domainsと呼ばれていたもの）を操作するための小さなハウツー。



Logical Domainsは、Sun Microsystemsが2007年5月に発表した、SPARC V9ベースのUNIXサーバーライン向けの仮想化および物理リソース共有テクノロジーです。

コマンドラインまたはビジュアル制御インターフェースからハイパーバイザーを制御することは可能ですが、後者はここでは考慮されません。Solaris 10

OSの例が示されていますが、Solaris 11ではネットワーク構成の機能を検討する価値があります。

Sunの用語では、仮想マシンはドメインと呼ばれ、いくつかのタイプのドメインがあります。

通常、デバイスサービスは制御ドメインで作成されますが、このために別のドメイン（サービスドメイン）を選択できます。

  • primary-vcc0という名前と5000〜5100で使用されるポートの範囲を使用して、プライマリドメインにコンソールコンセントレータを作成します。

    # /usr/sbin/ldm add-vcc port-range=5000-5100 primary-vcc0 primary

  • primary-vds0と呼ばれるプライマリドメインに仮想ハードディスクサーバーを作成します。

    # /usr/sbin/ldm add-vds primary-vds0 primary

  • primary-vsw0という名前のプライマリドメインに仮想スイッチを作成し、それをnxge0ネットワークインターフェースにバインドします。ネットワークカードのMACアドレスを使用：

    # /usr/sbin/ldm add-vsw net-dev=nxge0 primary-vsw0 primary

    指定されたMACアドレスが使用されます：

    # /usr/sbin/ldm add-vsw mac-addr=2:04:4f:fb:9f:0d net-dev=nxge0 primary-vsw0 primary

  • プライマリドメインサービスのリストを表示します。

    # /usr/sbin/ldm list-services primary


制御ドメインを設定し、そこから他のドメインのためにリソースを解放する


  • 暗号化デバイスのリストを表示します。SPARCシステムで使用される場合、ドメイン内にこれらのデバイスが存在すると、CPUの数を動的に再構成できません。

    # /usr/sbin/ldm list -o crypto primary
  • このパラメータを自然に0に設定するか、このコマンドにまったく触れないでください。

    # /usr/sbin/ldm set-mau 0 primary
  • コアの数をプライマリドメインに設定します。

    # /usr/sbin/ldm set-vcpu 8 primary
  • プライマリドメインの再構成を開始するか、サーバーを再起動します。

    # /usr/sbin/ldm start-reconf primary
  • プライマリドメインのRAM容量を設定します。

    # /usr/sbin/ldm set-memory 4G primary
  • 現在のドメイン構成を初期として保存します。

    # /usr/sbin/ldm add-config initial
  • ドメイン構成のリストを表示します（最初の[次の電源オン]は、ハイパーバイザーが次にロードされるときに構成が使用されることを意味します）：

    # /usr/sbin/ldm list-config
  • ハイパーバイザーを再起動します。

    # shutdown -y -g0 -i6

デフォルトでは、制御ドメインと他のドメインの間にネットワーク接続はありません。これは、制御ドメインが物理インターフェース（例：nxge0）を使用し、残りのドメインが仮想（例：vsw0）であるためです。

イメージからOracle Solaris 10をインストールする


  • 必要なドメインを停止します。

    # /usr/sbin/ldm stop ldg1
  • 管理対象ドメインのリストから除外します：

    # /usr/sbin/ldm unbind-domain ldg1
  • solaris10.isoイメージをiso_volという名前のprimary-vds0仮想ディスクサーバーに追加します。

    # /usr/sbin/ldm add-vdsdev /export/solaris10.iso iso_vol@primary-vds0
  • 仮想化ドライブをvdisk_isoとしてldg1ドメインに接続します。

    # /usr/sbin/ldm add-vdisk vdisk_iso iso_vol@primary-vds0 ldg1
  • ldg1ドメインに関連するすべてのリソースを表示します。

    # /usr/sbin/ldm list-bindings ldg1
  • ldg1ドメインを制御ドメインに関連付けます。

    # /usr/sbin/ldm bind-domain ldg1
  • ldg1ドメインを起動します。

    # /usr/sbin/ldm start-domain ldg1
  • ドメインコンソールに接続します。

    # /usr/bin/telnet localhost 5000
  • vdisk_isoのドメインデバイスエイリアスを参照します。

    ok devalias
  • イメージから起動します。

    ok boot vdisk_iso:f

ドメインの削除


  • ldg1ドメインを停止します。

    # /usr/sbin/ldm stop-domain ldg1
  • ldg1ドメインのリンクを解除します。

    # /usr/sbin/ldm unbind-domain ldg1
  • ldg1ドメインを削除します。

    # /usr/sbin/ldm remove-domain ldg1
