それをより早く連想化します（PyMorphy2、PyMystem3およびいくつかの魔法）

私はプログラマーとして働いており、テキスト分析に関連する機械学習も行っています。自然言語を処理する場合、ドキュメントの事前準備が必要です。その方法の1つは、テキストのすべての単語をコンテキストを考慮して通常の形式にすることである、見出し語化です。

最近、このプロセスに多大な時間を費やすという問題に直面しました。特定のタスクで100,000を超えるドキュメントがあり、その平均長は約1000文字であり、計算のためにサーバーではなく、通常のローカルコンピューターで処理を実装する必要がありました。インターネットで解決策を見つけることはできませんでしたが、私たちはそれを自分自身で見つけました。私はそれを共有したいと思います-この記事で最も人気のある2つの見出し語化ライブラリの比較分析を示します。



ピモフィ2


最も人気のあるものの1つはPyMorphy2です。これは、ネットワーク上で見つけることができるほとんどすべてのソリューションに含まれています。このライブラリも使用しましたが、データベース全体の見出し語化が必要になるまで、完全に表示されました（上で書いたように、これらは10万以上の小さなドキュメントです）。このような大量のドキュメントを分析するために、PyMorphy2はほぼ10時間かかりますが、この間ずっとプロセッサの負荷は平均で約30％です（Intel Core i7 7740X）。

Pymystem3


別の解決策を探すために、Yandex PyMystem3のライブラリを分析しましたが、結果はPyMorphy2のほぼ2倍（時間的に）劣っていました。10万件のドキュメントを処理するには16時間かかります。

魔法


プロセッサの負荷がほとんどゼロであるのは不思議に思えました。また、1つのテキスト、つまり大きなテキスト（3〜4千文字）から結果を取得するために、PyMystem3が約1秒かかったのも奇妙でした。したがって、テキストの間に何らかのセパレータを追加してテキストを結合することにしました。これにより、ドキュメントリストの構造を再び返し、それらを見出し語化することができます。

Pythonソリューションコード：

def checkExecTimeMystemOneText(texts):
    lol = lambda lst, sz: [lst[i:i+sz] for i in range(0, len(lst), sz)]
    txtpart = lol(texts, 1000)
    res = []
    for txtp in txtpart:
        alltexts = ' '.join([txt + ' br ' for txt in txtp])

        words = mystem.lemmatize(alltexts)
        doc = []
        for txt in words:
            if txt != '\n' and txt.strip() != '':
                if txt == 'br':
                    res.append(doc)
                    doc = []
                else:
                    doc.append(txt)

ユニオンごとに1000個のドキュメントを取得し、それらの間の区切り文字は「br」でした（ロシア語のテキスト、および以前に削除したラテン語のアルファベットと特殊文字に注意してください）。このソリューションは、見出し語化を大幅に加速しました。平均すると、すべての10万のドキュメントで約25分であり、プロセッサの負荷は20〜40％でした。確かに、このソリューションはRAMをより多くロードします。平均すると、約3〜7 GBですが、これは良い結果です。十分なメモリがない場合は、結合するドキュメントの数を変更できますが、これにより処理が遅くなりますが、一度に1つのテキストよりもはるかに高速になります。また、メモリの容量が許す場合は、この値を増やして、結果をさらに速く取得できます。
