こんにちはハブル！今日は再び認識について話します。つまり、カスケード分類器としてのこのような単純な認識モデルについてです。そのカスケードは、ViolaとJonesの人気のある方法で使用されています。これについては、Habréで何度も書いています（たとえば、ここ、ここ、およびここ）。悲しいことに、記事が豊富であるにもかかわらず、カスケード分類子を真剣に研究した人はいません。そして、ハブレだけでなく、科学界にも。カスケード分類子は単純に見えますが、多くの落とし穴と興味深い機能があります。したがって、私たちはあなたと私たちの知識を共有することを急いでいます。だから、興味があれば猫へようこそ。

カスケード分類子は非常に単純なモデルです。これは、いくつかの連続するレベルで構成され、各レベルはバイナリ分類子として表すことができます。調査された前例は、最初のレベルの入力に供給され、次にレベルごとに「移動」します。次のレベルで分類子が前例を否定として認識した場合、カスケードの残りの分類子によって分析されなくなります。このような単純なモデルは、Viola and Jonesの方法[1]の発表後に普及しました。しかし、それはこれだけですか？それは単なるカスケード分類器ですか？それを理解しましょう！

Habréに関する本日の記事を、新しい形式で作成します。私たちは詳細に明らかにするいくつかのステートメントを選択し、どこかに反論さえします。それでは始めましょう！

Viola and Jonesメソッドのカスケード分類器は、オブジェクト検出器の動作を高速化するために使用されます


最初のページの元の記事[1]には、そのようなフレーズがあります。
この論文の3番目の主要な貢献は、画像の有望な領域に注目することで検出器の速度を劇的に向上させるカスケード構造で、より複雑な分類子を連続的に組み合わせる方法です。

実際、ビオラとジョーンズのオリジナルの方法は、画像内のオブジェクトを検索するように設計されています。さらに、検出問題は、調査対象の画像の各ポイントに異なるスケールで適用されるバイナリ分類器を使用したスライディングウィンドウ法によって解決されます。検出段階での調査データの不均衡（調査中の各画像に目的のオブジェクトがない「空の」領域、オブジェクトのある領域よりも数百万または数十億倍も多い）は、カスケードの使用を促しました-カスケード-「空の」領域をすばやく遮断できるメカニズム。

しかし、それはすでに訓練された分類器を使用することに関するものでした。次に、分類子のトレーニング手順に移りましょう。サンプルの不均衡というまったく同じ問題があることがわかります。負の例の数は、正の例の数よりも何倍も（何百万倍も、何十億倍も）多いです。しかし、そのアーキテクチャのおかげで、カスケードの新しい各レベルは、負のトレーニングサンプル全体ではなく、カスケードの以前のレベルからの限られたエラーセットに対してのみ、AdaBoostメソッドによってトレーニングされます。これにより、バランスのとれた限られたサンプルでAdaBoostトレーニングマシンを実行できます。

ご覧のように、Viola and Jonesメソッドでカスケード分類器を使用すると、2回撮影されます。

  1. 分類器を簡単にトレーニングでき、「無限」のトレーニングセットの問題を自然に回避できます。
  2. オブジェクトの検出時に「空の」領域を高速でクリッピングするため、平均的な生産性が高くなります。

カスケード分類器は一貫して高いパフォーマンスを提供し、リアルタイム認識システムで簡単に使用できます


上で述べたように、カスケードスキームでは、「空の」領域の数がオブジェクトを含む領域の数よりも数桁多いため、「空の」領域の迅速なスクリーニングにより、高いパフォーマンスを実現できます。 「空」の領域の処理時間は、オブジェクトがある領域の処理時間と数倍異なります（カスケードの長さ-カスケードのレベル数に比例）。

オブジェクトを含む領域の数は画像ごとに異なるため、各フレームの処理時間も異なります。フレーム上にオブジェクトがある領域がオブジェクトのない領域よりも大幅に少ないため、処理時間の違いは数十倍ではなく数十パーセント測定されますが、それでも産業用認識システムでは重要です。

したがって、異なる画像でのカスケード分類器の動作時間は大幅に異なる可能性があります。したがって、分類子のパフォーマンスの深刻な測定を実行する場合、測定は平均で最悪の場合の動作時間で行う必要があります。また、カスケード分類子を使用する場合は、このような一時的な「不整合」に常に備えてください。

私たちの実践では、平均的なケースと最悪のケースでカスケード動作時間に大きな差異があるため、すでに深刻な問題に直面しています。有料道路自動化プロジェクトの一環として、車のタイプを認識する問題を解決しました。主なサブタスクの1つは、ホイールセットのカウントの問題でした。もちろん、ビオラとジョーンズの方法を使用して、個々のフレームのホイールを検出しました。ホイールの変動が大きいため（下図を参照）、トレーニングされたカスケードは非常に長くなりました（20レベル）。フレームごとに異なる処理時間に関連する不快な問題をライブで観察したため、リアルタイムで認識システムを構築することができませんでした。

ご覧のように、カスケードの複雑さは、提示されているトレーニングアルゴリズムには関与していないため、パフォーマンスに効果があるとは言えません。同時に、世界中の科学者が効果的なカスケードのためのアルゴリズムを学習することの重要性を確信していることは確かです。
The overall training process involves two types of tradeoffs. In most cases classifiers with more features will achieve higher detection rates and lower false positive rates. At the same time classifiers with more features require more time to compute. In principle one could define an optimization framework in which
– the number of classifier stages,
– the number of features, ni, of each stage,
– the threshold of each stage
are traded off in order to minimize the expected number of features N given a target for F and D. Unfortunately finding this optimum is a tremendously difficult problem.

興味深いのは、2016年に作成された関連文献のレビューで、その時点では分類器をカスケードするための効果的なトレーニングアルゴリズムがないことを示していたことです。ちなみに当時、Smart Enginesがこの問題を解決していました。第1種の検出エラーに依存する次の関数を調査しました（E1）、第2種の検出エラー（E2）および中程度の難易度（N）：

F(E1,  E2, N)=  β1⋅ E1+ β2⋅E2+  β3⋅N.


パラメータの選択 β1、 β2 そして β3、第1種と第2種の誤差の相対コスト、および結果として得られる検出器の複雑さを設定します。さらに、カスケード分類器をトレーニングするプロセスでは、選択された汎関数を最大化するカスケードを得るために、各レベルでパラメーターを列挙する貪欲なアルゴリズムが使用されました。開発されたアルゴリズムの詳細な説明はこの記事の範囲を超えていますが、書誌リンク[5]を提供することにより、読者である読者といつでもそれを共有する準備ができています。
