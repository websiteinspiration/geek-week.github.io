タプルのリストがPythonで異常な動作をするのはなぜですか？

プログラミング言語では、私は常にそれらの内部構造に興味を持っていました。1人または別のオペレーターはどのように機能しますか？なぜこのように書いた方がよいのでしょうか。このような質問は、「今ここで」問題を解決するのに常に役立つとは限りませんが、長期的には、プログラミング言語の全体像を形成します。今日、私はこれらのダイビングの1つの結果を共有し、tuple「a in list」eを変更するとどうなるかという質問に答えたいと思います。

Pythonにはデータ型があることは誰でも知っていますlist。

a = []
a.append(2)

list単なる配列です。アイテムを追加、削除、変更することができます。また、さまざまな興味深い演算子もサポートしています。たとえば、項目をに追加する+ =演算子list。+=現在のリストを変更しますが、新しいリストは作成しません。これはここではっきりと見えます：

>>> a = [1,2]
>>> id(a)
4543025032
>>> a += [3,4]
>>> id(a)
4543025032

Pythonには、もう1つの優れたデータ型がありますtuple。-不変のコレクション。要素の追加、削除、変更はできません：

>>> a = (1,2)
>>> a[1] = 3
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

演算子を使用する+=と、新しい演算子が作成されtupleます。

>>> a = (1,2)
>>> id(a)
4536192840
>>> a += (3,4)
>>> id(a)
4542883144

注意、問題は次のとおりです：次のコードは何をしますか？

a = (1,2,[3,4])
a[2] += [4,5]

オプション：

  1. アイテムがリストに追加されます。
  2. タプル不変性例外がスローされます。
  3. どちらも。
  4. どちらでもない。

答えを一枚の紙に書いて、少し確認しましょう。

>>> a = (1,2,[3,4])
>>> a[2] += [4,5]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

じゃあ！だから私たちはそれを理解しました！正解は2です。少し待ってください。

>>> a
(1, 2, [3, 4, 4, 5])

実際、正しい答えは3です。つまり、要素が追加され、例外が出ました-ワット？！


これがなぜ起こるか見てみましょう。そして素晴らしいモジュールがこれを助けてくれますdis：

import dis

def foo():
    a = (1,2,[3,4])
    a[2] += [4,5]

dis.dis(foo)

最初のブロックは、tuple'aを作成し、それを変数に格納しますa。それから楽しみが始まります：

       21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)

変数aと定数2 へのポインタをスタックにロードします。

       27 DUP_TOP_TWO

それらを複製し、同じ順序でスタックに入れます。

       28 BINARY_SUBSCR

この演算子は、スタックの最上位要素（TOS）と次の要素（TOS1）を取ります。そして、スタックの一番上に新しい要素を書き込みますTOS = TOS1[TOS]。したがって、スタックから上位2つの値を削除し、2番目の要素tuple'a（配列）へのリンクを入れます。

       29 LOAD_CONST      4 (4)
       32 LOAD_CONST      5 (5)
       35 BUILD_LIST      2

要素4と5のリストを作成し、スタックの一番上に配置します。

       38 INPLACE_ADD

+=スタックの上位2つの要素に 適用します（重要！これらは2つのリストです！1つは4と5で構成され、もう1つはから取得されますtuple）。ここではすべて問題なく、命令はエラーなしで実行されます。それはので、+=元のリストを変更し、内のリストtupleのE「は既に（その瞬間に）変更されました。

       39 ROT_THREE
       40 STORE_SUBSCR

ここでは、スタックの上位3つの要素（そこにありtuple、その中に配列のインデックスと新しい配列）を入れ替え、新しい配列をtupleインデックスで書き込みます。ここで例外が発生します！

まあ、それだけです！実際、リストは変更できますが、すべてオペレーターに任され=ます。

最後に、例外なくこのコードを書き直す方法を見つけましょう。すでに理解したように、必要なのはのエントリを削除することだけですtuple。ここにいくつかのオプションがあります：

>>> a = (1,2,[3,4])
>>> b = a[2]
>>> b += [4,5]
>>> a
(1, 2, [3, 4, 4, 5])

>>> a = (1,2,[3,4])
>>> a[2].extend([4,5])
>>> a
(1, 2, [3, 4, 4, 5])
