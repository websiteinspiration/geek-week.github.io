Web Apps: Micro Frontend Module Federation Supported Framework

I want to introduce a microfront framework with support for Webpack Module Federation. The framework allows you to link applications written in any library, vanilla JS, and even IFrame, if things are really bad.


Recently there was an article where it was described in some detail what is on the market. However, most solutions impose some additional concepts, whether it be routing or the application registry, although in fact sometimes none of this is necessary, but only a simple and easy way to implement one application into another is needed.


But let's step back a second, why do we need a micro-frontend approach? At some point, any large application outgrows itself, the build process slows down, every small change results in waiting minutes in watch mode, there is too much code, and it is no longer possible to effectively support it.


The problem is exacerbated if the team wants to switch to a new stack / framework. We tried to smoothly switch from one framework to another, replacing one monolith with another. The entire transition period had to support both products, and when we finally finished the final monolith was absolutely huge. At some point, the idea was expressed to start splitting it into sub-applications, but each of them was developed by different teams, had a different interface for communicating with the host application, and some sub-applications were generally IFrame, because needed maximum isolation. This approach, despite the minuses of the zoo, worked for some time and was very successful. But for the long term, he did not work.


How to include an application written on one framework in another application on another framework? ** Need an abstract orchestra! **

Before we go directly to the framework, we outline the basic terms:



  • A host is an application that contains smaller applications.
  • A sub-application (or just an Application) is an IFrame, Global Application or Web Component Application that are included in the host, they, in turn, can be hosts for more embedded applications
  • Orchestrator: Web Apps Framework - an agent that allows the host to communicate with applications, download them, etc.
  • Applications do not speak directly to each other, but the host can connect them

Web Apps Framework


Web Apps framework is a framework-independent way to implement any application into any application, with a generalized communication interface and other bonuses.

import React from "react";
import {render, unmountComponentAtNode} from "react-dom";
import {App} from './app';

const template = document.createElement('template');

template.innerHTML = `
    <style>/* shadow CSS */</style>
    <div class="container"></div>
`;

customElements.define('web-app-%appId%', class extends HTMLElement {
    constructor() {
        super();
        this.attachShadow({mode: 'open'});
        this.shadowRoot.appendChild(
            document.importNode(template.content, true)
        );
        this.mount = this.shadowRoot.querySelector('.container');
    }

    connectedCallback() { render(
        <App 
            authtoken={this.getAttribute('authtoken')}
            node={this}
        />, 
        this.mount
    ) }

    disconnectedCallback() { unmountComponentAtNode(this) }    
});
