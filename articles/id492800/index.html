Penyedia IoT GSM di Perumahan dan Utilitas Publik (Bagian 1)

Saya membaca artikel penulis Mengganggutentang kesulitan di Iot dan memutuskan untuk berbicara tentang pengalamannya sebagai penyedia IOT.

Artikel pertama bukan iklan, sebagian besar materi tanpa model peralatan. Saya akan mencoba menuliskan detailnya di artikel berikut ini.

Saya tidak melihat masalah dalam menggunakan modem GSM untuk mengumpulkan data dari perangkat pengukuran, karena saya mengambil bagian dalam menciptakan sistem pengumpulan dari 795 bangunan tempat tinggal, frekuensi survei setiap jam, stabilitas dikonfirmasi oleh tiga tahun operasi.

Geografi jaringan tempat kerja sekarang memiliki lebih dari 5.000 titik pengumpulan di Rusia.

Kesulitan


  • Koneksi timeout - di ruang bawah tanah dengan kualitas komunikasi, secara halus, tidak terlalu, ada dua pilihan: biarkan apa adanya atau ubah tempat titik koneksi tetapi kirimkan sejumlah kecil data.

  • Personil - ya ada beberapa dari mereka, tetapi masalahnya sedikit berbeda, tidak semua spesialis yang baik setuju untuk naik ke ruang bawah tanah tanpa insentif materi.

  • Peralatan kebun binatang bisa menjadi banyak masalah, terutama ketika ada lebih dari 100 model dan modifikasi yang berbeda, tetapi ada solusi yang sudah jadi.

  • Server pengumpulan data - Saya tidak menyarankan menciptakan kembali roda dan menulis dari awal, kecuali tentu saja ini bukan proyek untuk 100.000 poin dan peralatan di situs satu model.

  • Kurangnya kualitas instalasi di fasilitas, kruk di setiap fasilitas, saya akan memberikan contoh dari arsip pribadi, menarik kesimpulan Anda sendiri bagaimana Anda dapat terhubung ke sana tanpa saraf (contoh di bawah)
