製品アナリスト：何をするか、どれだけ稼ぐか、ビジネスがもたらすメリット

Product Analyst-ビジネスとデータの架け橋。彼は製品マネージャーと協力して、製品チームが正しい決定を行うのを助けます。Netisの作者であるDenis Vikharevが、製品分析とは何か、ビジネスで製品アナリストが評価される理由、製品アナリストになることができる理由、自分のタスク、給与、およびツールについて語っています。

製品分析とは


製品分析を使用すると、ユーザーが製品と対話する方法を確認できます。従来、製品分析の2つのタスクは区別することができます。データ収集とその解釈です。

まず、製品アナリストはさまざまなソースから多数の数値を収集します。ユーザーがクリックするボタン、製品を使用する頻度、製品の製品の人気のある機能、そうでないものなどです。これらの測定値は製品に何が起こるかを示していますが、その理由は説明していません。

2番目のステップでは、アナリストはユーザーの行動を説明する数値から洞察を引き出します。これのおかげで、製品チームは自分が作った製品と次に進むべき場所を理解しています。
製品分析は、チームが「誰が」「何をいつ、どこで」行ったかを理解するのに役立ちます。そして、これを使って次に何をすべきか。

製品アナリストがビジネスで評価される理由


製品アナリストは、製品内のユーザーイベントを追跡し、数値の意味をビジネスの言語に変換し、問題を解決するための運用上の推奨事項を提供します。企業が製品アナリストの手によって解決する4つのタスクを特定しました。

製品のユーザーを維持する


お金は企業に爆発的な成長をもたらし、多くの新しい顧客をもたらすことができます。製品分析は、彼らの行動を知り、彼らが直面する問題と彼らが製品で受け取る価値を処理することにより、既存のユーザーを保護するのに役立ちます。

新規ユーザーは常に既存のユーザーよりもコストがかかるため、製品分析にリソースを投資することは企業にとって有益です。
ベンチャー投資家のTomasz Tunguz氏の意見：「一方で、成長は投資のラウンドを上げるのに役立ち、製品の需要を示しています。一方、顧客の流出は、製品の市場への適合性について疑問を投げかける。

「市場の要件を十分に満たしていない製品でビジネスの成長を刺激すると、企業は数百万ドルを調達し、「ロールアウト」する必要があります。

友達を商品や市場にする


製品の市場価値（文字通り「製品から市場へ」）を保証する製品のコアバリューを理解せずにユーザーのロイヤルティを獲得することは不可能です。同じ「あはの瞬間」を見つけるには、忠実な顧客と失われた顧客をどのような行動で分けるかを知る必要があります。
「Aha Moment」は、ユーザーが製品の価値を理解する瞬間である成長の鍵です。それを見つけることは、顧客データに基づく正しい結論に役立ちます。Facebookにとって、これは  「10日間で7人の友達」という目標の達成でした。
ユーザーに関連する製品指標を知ることは、市場に一致させるのが簡単です。SnowplowのAnthony Mandelliは、Airbnbの創設者であるJoe Gebbia がポッドキャスト「How I Built It」でポッドキャストでどのように  データを使用して見たのかについて語ったことを思い出します  。その後、Airbnbが問題の解決策を引き継ぎ、会社の収益を時々増加させました。

写真の品質に関する問題を解決することで、Airbnbの収益が数倍増加しました。出典：Airbnbウェブサイト

競合他社をしのぐ


マッキンゼーの調査に  よると、データと分析を賢く使用することで、企業は大規模に成長することができます。これのおかげで、業界のリーダーと遅れているプレーヤーの間のギャップは絶えず拡大しています。

調査時、市場をリードする企業の回答者は、データイニシアチブと分析により、3年間で利益の少なくとも20％をもたらしたと述べています。

分析を使用したスポット作業は、望ましい効果をもたらしません。リーダーになるには、データを操作するための長期的な戦略を作成する必要があります。

ユーザーエクスペリエンスを向上させる


製品チームは製品を盲目的に変更できます。しかし、結果を分析しなければ、成功または失敗に具体的につながった原因を特定することはできません。製品分析では、ユーザーの行動データをリアルタイムで調べます。これは、チームが次の反復サイクル中に製品のビジョンを再考し、必要なアクションを実行するのに役立ちます。

製品アナリストは何をしますか？


製品アナリストは製品の状態を分析し、製品の開発を支援します。これにより、製品のメトリックが低下していないこと、および製品の決定が成功することが保証されます。

データの整理、目標の設定、製品アナリスト向けのテストの実施に関する統一された規則は見つかりませんでした。各企業には独自の規則があります。問題のある製品のライフサイクルは重要です。ローンチしたばかりのスタートアップでは、アナリストはデータ収集システムを整頓するよう求められ、成熟したシステムでは成長ポイントを見つけて競合他社を検討します。

そのため、私たちは何十もの空席を分析し、アナリストが直面する可能性のあるタスクのリストをまとめました  。レビューには、IT企業だけでなく、「従来のビジネス」である航空会社、マスマーケット、ロジスティクス企業も含まれていました。


製品の成長ポイントとボトルネックを見つける


製品チームがボタンをクリックした回数に関するデータに依存することは難しく、人間の行動の理由を説明していません。したがって、チームはデータに基づいて結論を出し、製品のパターンと異常を見つけるアナリストに行きます。

レポートと製品モニタリング指標を開発する


ダッシュボードには、チームとリーダーの主要な製品指標、依存関係、および傾向が表示されます。アナリストは、ダッシュボードに表示する必要のあるレポートとメトリックを決定します。これにより、主要なものから注意をそらさず、管理上の決定を行うのに役立ちます。チームが必要とするメトリックの普遍的なセットはなく、ビジネスの目標と製品のタイプに応じて選択されます。
プロダクトマネージャーのSergey Tikhomirovは、一連のメトリックを製品のライフサイクルと関連付け  ます。また、AppMetrica製品のVladislav Prishchepovは、目標から始めて、製品の主要なメトリックに焦点を当てることを推奨  しています。したがって、食品配達アプリケーションの場合、これは「食べる時間」、つまり注文を完了してからそれを受け取るまでの時間です。

チームの問題と解決策を検証する


アナリストは定量的なデータを「掘り下げ」て、チームの仮説をテストし、タスクに正しく優先順位を付けます。詳細なインタビュー中にチームが問題を特定した場合、アナリストはそれを確認するか、異議を唱えることができます。たとえば、音声分析ツールを使用して、キーワードでコールセンターのマネージャーと顧客との何百もの会話を分析したことがあります。

A / Bテストを実施する


作業仮説は、ユーザーのコントロールセグメントでテストされます。アナリストは、テストが外部および内部の要因（休日、屋外の天気、コールセンターのマネージャーによる顧客への電話）の影響を受けないことを確認します。これらすべてがパフォーマンスを歪める可能性があります。

テスト結果は統計的に有意でなければなりません-Appsumoサービスによると、  これは12.5％のケースでのみ発生します。制御セグメントが仮説を確認すると、スケーリングされます。別の問題は、トラフィックが少ないb2bでのテスト後に決定することです。

仮説のテストとスケーリング


仮説のテストは、条件付きで4つの段階に分けられます。1）影響を与えたいメトリックを探しています。2）調査を実施する。3）フィードバックを分析します。4）仮説またはスケールを削除します。アナリストはすべての段階で製品チームと協力し、「なぜこれが起こったのか」と「それに対して何をすべきか」という質問に答え、チームが不人気な決定をサポートするのを防ぎます。アナリストの作業の主な価値は、結果の解釈にあります。

データを分析する


製品アナリストのタスクの大きなプールは、ハードスキルの分野にあります。彼は統計の基本を理解し、プログラミング言語を話します。これは、分析者がデータを収集して処理し、その品質を評価してパターンを探すのに役立ちます。

新規クライアント（クライアントではない）の操作：

多くの時間を費やしています：

  • BIシステムで定期的なレポートを設定する。
  • サイト上のイベントをマークアップし、一般に、さまざまなソースからのデータ収集をセットアップします。
  • エンドツーエンドの分析、データマートをクラスター上に構築します。

製品分析ツール



Python  （アナログ：R、Java）

単純な構文を備えたプログラミング言語、多数のライブラリ、および必要に応じて役立つ開発されたコミュニティ。

Excelで処理できない大量のデータの処理に適しています。データを視覚化し、情報分析のタスクを自動化し、顧客の流出を予測するためのモデルを構築し、クラスタリングを実行できます。


Google Analytics  （アナログ：Yandex.Metrica、Heap）

ウェブ解析のための無料ツール。これは、サイトでのトラフィックとユーザーアクションのソース、訪問者数、ビュー、コンバージョン、カスタマイズされたイベントのレポートを示し、コホート分析の実施に役立ちます。


Tableau  （アナログ：Power BI、QlikSense、Looker）

明確なインターフェースでデータを分析および視覚化するためのプラットフォーム。効果的なグラフィックを構築し、さまざまなソースからのデータを組み合わせるのに役立ちます。 MS Excel、MySQL、SQL、Google BigQuery、Microsoft Azureで動作します。レポートの自動更新と配布を構成し、それらを電子メールで送信し、サーバーにリンクを公開し、参照によりレポートにアクセスすることができます。


Mixpanel  （アナログ：Amplitude、Flurry、KissMetrics）

リアルタイムの分析と分析のためのシステム。登録後のユーザーの行動を理解するのに役立ちます。各イベントの条件を含む目標到達プロセスを構築し、プッシュを送信し、A / Bテストを実施できます。


SQL

製品エコシステム内のデータベースを操作するためのツール。これにより、アナリストは開発者なしで必要なデータを受信、処理、および作成します。動的な期間でレポートを作成し、テーブルを結合し、必要な基準に従って値を切り捨てることができます。
