Les 14 bibliothèques Node.js les plus populaires en 2020



Bonjour mes amis!

Je vous présente la traduction de l'article «14 bibliothèques NodeJS les plus utiles en 2020» .

Dans cet article, l'auteur partage ses découvertes parmi les nombreux packages Nodejs.

Express


Cadre de développement Web rapide, flexible, minimaliste et facile à apprendre.

Idéal pour

  • développement d'applications à l'aide de la pile MEAN (MongoDB, Express, Angular, Node) ou MERN (React au lieu d'Angular)
  • gestion facile de divers types de demandes, telles que GET, PUT, POST et DELETE
  • déploiement rapide d'applications monopages, multipages et hybrides

Nombre de téléchargements par semaine

11 millions

Alternatives

Koa, Hapi, Meteor

Cheerio


Conçu pour traiter ("analyser") le balisage, par exemple, HTML, fournit une interface pour manipuler ("parcourir") la structure de données finale.

	const cheerio = require('cheerio')
	const $ = cheerio.load('<ul id="fruits">...</ul>')

Idéal pour

  • création de scanners web ("scrapers")
  • syntaxe et boîte à outils simples et intuitives

Téléchargements hebdomadaires

4,2 millions

Alternatives

marionnettiste jsdom

Nodemailer


Conçu pour envoyer des e-mails.

	const nodemailer = require('nodemailer')

	const testAccount = await nodemailer.createTestAccount()

	const transporter = nodemailer.createTransport({
		host: 'smtp.ethereal.email',
		port: 587,
		secure: false,
		auth: {
			user: testAccount.user,
			pass: testAccount.pass
		}
	})

	const info = await transporter.sendMail({
		from: '"John Smith" <smith@example.com>',
		to: 'foo@example.com, bar@example.com',
		subject: 'hello',
		text: 'hello crazy world',
		html: '<strong>hello crazy world</strong>'
	})

Idéal pour

  • envoi de messages via SMTP

Téléchargements hebdomadaires

980 mille

Alternatives

sendmail, emailjs

Socket.io


Conçu pour configurer une connexion bidirectionnelle basée sur les événements en temps réel.

	const server = require('http').createServer()
	const io = require('socket.io')(server)
	io.on('connection', client => {
		client.on('event', data => { ... })
		client.on('disconnect', () => { ... })
	})
	server.listen(3000)

Idéal pour

  • implémentation d'analyses en direct, flux de données binaires, messagerie, collaboration avec des documents

Téléchargements hebdomadaires

3 millions

Alternatives

poussoir, websockets

Faker


Conçu pour recevoir de fausses informations dans le navigateur et le «nœud».

	const faker = require('faker')

	const randomName = faker.name.findName()
	const randomEmail = faker.internet.email()
	const randomCard = faker.helpers.createCard() //

Idéal pour

  • tester le frontend en termes de travail avec les données jusqu'au développement du backend
  • Un grand nombre de méthodes, y compris l'obtention d'une adresse arbitraire, nom de l'entreprise, base de données, image, prénom, nom de famille, etc.

Téléchargements hebdomadaires

1,4 million

Alternatives

décontractée

Morgan


Il s'agit d'un middleware pour l'enregistrement des requêtes.

	morgan
	(':method:url :status :res[content-length] - :response-time ms')

	---

	const express = require('express')

	const morgan = require('morgan')

	const app = express()

	app.use(morgan('combined'))

	app.get('/', (req, res) => res.send('hello crazy world'))

Idéal pour

  • écrire des requêtes dans la console, le fichier, la base de données
  • débogage et «journalisation»

Téléchargements hebdomadaires

2 millions

Erreurs HTTP


Conçu pour créer des erreurs spécifiques afin de tester des applications sur Express, Koa, Connect, etc.

	app.use((req, res, next) => {
		if(!req.user) return next(createError('401', ',    .'))
		next()
	})

Idéal pour

  • erreurs d'émulation
  • un grand nombre de propriétés de l'objet Error sont disponibles

	expose
	headers
	message
	status
	statusCode

Téléchargements hebdomadaires

27 millions

Analyseur de corps


Il s'agit d'un middleware pour l'analyse des données.

Analyse les demandes entrantes (corps requis) avant de les traiter.

	const express = require('express')
	const bodyParser = require('body-parser')

	const app = express()

	//  application/x-www-form-urlencoded
	app.use(bodyParser.urlencoded({ extended: false }))

	//  application/json
	app.use(bodyParser.json())

Idéal pour

  • Interprétation des corps de requête
  • contient de nombreuses fonctions utiles (gonfler, taper, vérifier, etc.)

Téléchargements hebdomadaires

13 millions

Sequelize


Il s'agit d'une technologie basée sur Promise pour le mappage relationnel-objet de Postgres, MySQL, MariaDB, SQLite et Microsoft SQL Server.

Il prend en charge les transactions sécurisées (fiables), l'échange de données, le chargement énergétique et différé, la réplication de données, etc.

	const sequelize = new Sequelize(
		'database', 'username', 'password', {
			host: 'localhost',
			dialect: /*   'mysql' | 'mariadb' | 'postgres' | 'mssql' */
		}
	)

Téléchargements hebdomadaires

720 mille

Passeport


Il s'agit d'un middleware compatible Express pour l'authentification.

La tâche principale consiste à authentifier les requêtes à l'aide d'un ensemble de plugins - stratégies.

	passport.use(new LocalStrategy(
		(username, password, done) => {
			User.findOne({ username: username }, (err, user) => {
				if(err) { return done(err) }

				if(!user) { return done(null, false) }

				if(!user.verifyPassword(password)) { return done(null, false) }

				return done(null, user)
			})
		}
	))

Idéal pour

  • authentification
  • intégration avec OAuth et OpenID (connectez-vous avec votre compte Facebook, Twitter, etc.)

Téléchargements hebdomadaires

810 mille

Dotenv


C'est un module de dépendance zéro (point) pour charger des variables du fichier .env dans process.env.

Garde les variables séparées du code.

	require('dotenv').config()

	const db = require('db')

	db.connect({
		host: process.env.DB_HOST,
		username: process.env.DB_USER,
		password: process.env.DB_PASS
	})

	//  .env
	DB_HOST = localhost
	DB_USER = root
	DB_PASS=12345

Idéal pour

  • chargement des variables d'environnement telles que AWS, noms d'utilisateur sql, mots de passe nécessaires pour déployer des applications, connexion à d'autres outils
  • stockage des paramètres d'environnement séparément du code

Téléchargements hebdomadaires

10 millions

Multer


Middleware pour le traitement de plusieurs parties / données de formulaire - chargement d'images.

	const express = require('express')

	const multer = require('multer')

	const upload = multer({ desc: 'uploads/' })

	const app = express()

	app.post('/profile', upload.single('avatar'), (req, res, next) => {
		// req.file -   'avatar'
		// req.body     
	})

	app.post('/photos/upload', upload.array('photos', 12), (req, res, next) => {
		// req.files -    'photos'
		// req.body      
	})

Idéal pour

  • téléchargement rapide de fichiers multipart / form-data

Téléchargements hebdomadaires

92 mille

Axios


Client HTTP basé sur des promesses pour navigateur et nœud.

	const axios = require('axios')

	//     ID
	axios.get('/user?ID=12345')
		.then(response => {
			// 
			console.log(response)
		}).catch(error => {
			//  
			console.log(error)
		}).finally(() => {
			//  
		})

Idéal pour

  • envoi de demandes depuis le nœud
  • envoi de requêtes depuis le navigateur
  • Support de promesse

Téléchargements hebdomadaires

9,6 millions

CORS


Intergiciel Connect / Express conçu pour définir des règles de politique d'origine commune.

	const express = require('express')

	const cors = require('cors')

	const app = express()

	app.use(cors())

	app.get('/products/:id', (req, res, next) => {
		res.json({ msg: '    !' })
	})

	app.listen(80, () => {
		console.log('ok')
	})

Idéal pour

  • Tâches liées au CORS

Téléchargements hebdomadaires

3,7 millions
