Webpackツリーの振とうと再エクスポートで2分

前書き


始めましょう。私たちは素晴らしい遺産を持つモノリシックなフロントエンドを持っていました。サービスは、コンポーネントと同じファイルにありました。すべてがファサードのスローガンと混同されていました：「すべてを手元に置いてください-必要なものを見つけるのが簡単です。」そして、ファイルの長さが200 +、300 +、500 +またはそれ以上のコード行であることは問題ではありません。


目的


すべてを読みやすく、より小さく、より速くします。


実装


可能なすべてのものをファイルに分離し、ここでの黄金の弾丸が唯一の責任の原則です。ファイル内にコンポーネントと純粋な関数がある場合は、それらを分離します。


ES6 +の登場により、import ... from構文を使用できるようになりました。これは、export ... fromも使用できるため、優れた機能です。


リファクタリング


次の構造のファイルを想像してください。


// ,  ,      
function multiply (a, b) {
  return a * b;
}
function sum (a, b) {
  return a + b;
}
function calculateSomethingSpecial(data) {
  return data.map(
    dataItem => sum(dataItem.param1, dataItem.param2)
  );
}

このコードを次の方法でファイルに分割できます。


構造：


utils
  multiply.js
  sum.js
  calculateSomethingSpecial.js

およびファイル：


// multiply.js

export default function multiply (a, b) {
  return a * b;
}
or
const multiply (a, b) => a * b;
//    –     .

// sum.js

export default function sum (a, b) {
  return a + b;
}

// calculateSomethingSpecial.js

import sum from "./sum";

export default function calculateSomethingSpecial(data) {
  return data.map(
    dataItem => sum(dataItem.param1, dataItem.param2));
}

これで、関数を個別にインポートできます。しかし、余分な行とインポートにこれらの長い名前があるため、それはまだひどいように見えます。


// App.js

import multiply from '../utils/multiply';
import sum from '../utils/sum';
import calculateSomethingSpecial from '../utils/calculateSomethingSpecial';
...

しかし、このために、再エクスポート（re-export）と呼ばれる新しいJS構文の登場で登場した素晴らしい機能があります。このフォルダーで、すべての機能を組み合わせるためにindex.jsファイルを作成する必要があります。これで、コードを次のように書き換えることができます。


// utils/index.js

export { default as sum } from './sum';
export { default as multiply } from './multiply';
export { default as calculateSomethingSpecial } from './calculateSomethingSpecial';

少しpodshamanim App.js：


// App.js

import { multiply, sum, calculateSomethingSpecial } from '../utils';
