Ubah dan hapus Azure VM menggunakan PowerShell

Menggunakan PowerShell, insinyur dan administrator TI berhasil mengotomatiskan berbagai tugas ketika bekerja tidak hanya dengan di tempat, tetapi juga dengan infrastruktur cloud, khususnya, dengan Azure. Bekerja melalui PowerShell dalam beberapa kasus jauh lebih nyaman dan lebih cepat daripada melalui portal Azure. Berkat kemampuan lintas platformnya, PowerShell dapat digunakan untuk OS apa pun.



Tidak masalah jika Anda bekerja dengan Ubuntu, Red Hat atau Windows - PowerShell akan membantu mengendalikan sumber daya cloud. Menggunakan modul Azure PowerShell , misalnya, Anda dapat mengatur properti apa pun dari mesin virtual.



Pada artikel ini, kita akan melihat bagaimana Anda dapat menggunakan PowerShell untuk mengubah ukuran VM di cloud Azure, dan juga untuk menghapus VM dan objek terkait.







Penting! Jangan lupa untuk menyeka tangan Anda dengan pembersih untuk bersiap-siap untuk bekerja:



  • Anda akan membutuhkan Modul PowerShell Azure - Anda dapat mengunduhnya dari Galeri PowerShell dengan perintah Install-Module Az



    .
  • Anda perlu mengautentikasi di Azure cloud, di mana mesin virtual berputar dengan menjalankan perintah Connect-AzAccount



    .


Pertama, buat skrip yang mengubah ukuran Azure VM. Buka VS Code dan simpan skrip PowerShell baru yang disebut Resize-AzVirtualMachine.ps1 - kita akan menambahkan potongan-potongan kode ke dalamnya saat kita melihat contoh.



Kami meminta ukuran VM yang tersedia



Sebelum mengubah ukuran VM, Anda perlu mencari tahu ukuran apa yang secara umum dapat diterima untuk mesin virtual di Azure cloud. Untuk melakukan ini, jalankan perintah Get-AzVMSize



.



Jadi, untuk mesin virtual devvm01 dari grup sumber daya dev , kami meminta semua jenis ukuran yang diizinkan: 



Get-AzVMSize -ResourceGroupName dev -VMName devvm01


(Dalam tugas nyata, tentu saja, alih-alih ResourceGroupName = dev dan VMName = devvm01 Anda akan menentukan nilai-nilai Anda untuk parameter ini.)



Perintah akan mengembalikan daftar seperti ini:







Ini adalah semua opsi ukuran yang memungkinkan yang dapat ditetapkan untuk mesin virtual ini.



Ubah ukuran mobil



Sebagai contoh, kami akan mengubah ukuran ke ukuran baru Standard_B1ls - ini berada di tempat pertama dalam daftar di atas. (Dalam tugas yang sebenarnya, tentu saja, Anda memilih ukuran yang Anda butuhkan.)



  1. Pertama, menggunakan perintah, Get-AzVM



    kami mendapatkan informasi tentang objek kami (mesin virtual), menyimpannya dalam variabel $virtualMachine



    : 



    $virtualMachine = Get-AzVM -ResourceGroupName dev -VMName devvm01
  2. Kemudian kami mengambil properti dari objek ini .HardwareProfile.VmSize



    dan menetapkan nilai baru yang diinginkan: 



    $virtualMachine.HardwareProfile.VmSize = "Standard_B1ls"
  3. Dan sekarang jalankan perintah pembaruan VM - Update-AzVm



    : 



    Update-AzVM -VM devvm01 -ResourceGroupName dev
  4. Kami memastikan semuanya berjalan dengan baik - untuk ini, sekali lagi, kami meminta informasi tentang objek kami dan melihat properti $virtualMachine.HardwareProfile



    : 



    $virtualMachine = Get-AzVM -ResourceGroupName dev -VMName devvm01
$virtualMachine.HardwareProfile


Jika kita melihat Standard_B1ls di sana , maka semuanya beres, ukuran mesin telah berubah. Anda dapat melangkah lebih jauh dan mengembangkan kesuksesan dengan mengubah ukuran beberapa VM sekaligus menggunakan sebuah array.



Bagaimana dengan menghapus VM di Azure?



Dengan penghapusan, tidak semuanya sesederhana dan semudah kelihatannya. Bagaimanapun, Anda perlu menghapus sejumlah sumber daya yang terkait dengan mesin ini, termasuk:



  • Simpan wadah penyimpanan diagnostik
  • Antarmuka jaringan
  • Alamat IP Publik
  • Disk sistem dan gumpalan di mana statusnya disimpan
  • Disk Data


Oleh karena itu, kami akan membuat fungsi dan menamainya Remove-AzrVirtualMachine



- dan itu tidak hanya akan menghapus Azure VM, tetapi juga semua hal di atas.



Kami mengikuti jalur standar dan pertama-tama kami mendapatkan objek kami (VM) menggunakan perintah Get-AzVm



. Sebagai contoh, biarkan itu menjadi mesin WINSRV19 dari grup sumber daya MyTestVMs .



Kami menyimpan objek ini dengan semua propertinya dalam sebuah variabel $vm



: 



$vm = Get-AzVm -Name WINSRV19 -ResourceGroupName MyTestVMs


Hapus wadah dengan file diagnostik boot



Saat membuat VM di Azure, pengguna juga diminta untuk membuat wadah untuk menyimpan diagnostik boot (wadah diagnostik boot), sehingga jika ada masalah dengan boot, ada sesuatu yang harus diubah untuk pemecahan masalah. Namun, ketika VM dihapus, wadah ini tetap melanjutkan keberadaannya yang sekarang tanpa tujuan. Perbaiki situasi ini.



  1. Pertama, cari tahu di akun penyimpanan mana wadah ini berada - untuk ini kita perlu menemukan properti storageUri



    di dalam perut objek DiagnosticsProfile



    VM kami. Untuk ini, saya menggunakan ungkapan reguler ini: 



    $diagSa = [regex]::match($vm.DiagnosticsProfile.bootDiagnostics.storageUri, '^http[s]?://(.+?)\\.').groups[1].value
  2. Sekarang kita perlu mencari tahu nama kontainer, dan untuk ini kita perlu mendapatkan ID VM menggunakan perintah Get-AzResource



    : 



    
if ($vm.Name.Length -gt 9) {
    $i = 9
} else {
    $i = $vm.Name.Length - 1
}
 
$azResourceParams = @{
    'ResourceName' = WINSRV
    'ResourceType' = 'Microsoft.Compute/virtualMachines'
    'ResourceGroupName' = MyTestVMs
}
 
$vmResource = Get-AzResource @azResourceParams
$vmId = $vmResource.Properties.VmId
$diagContainerName = ('bootdiagnostics-{0}-{1}' -f $vm.Name.ToLower().Substring(0, $i), $vmId)
  3. Selanjutnya, kami mendapatkan nama grup sumber daya tempat wadah itu berada: 



    $diagSaRg = (Get-AzStorageAccount | where { $_.StorageAccountName -eq $diagSa }).ResourceGroupName
  4. Dan sekarang kita memiliki semua yang kita butuhkan untuk menghapus wadah dengan perintah Remove-AzStorageContainer



    : 



    $saParams = @{
    'ResourceGroupName' = $diagSaRg
    'Name' = $diagSa
}
 
Get-AzStorageAccount @saParams | Get-AzStorageContainer | where { $_.Name-eq $diagContainerName } | Remove-AzStorageContainer -Force


Hapus VM



Sekarang kita akan menghapus mesin virtual itu sendiri, karena kita telah membuat variabel $vm



untuk objek yang sesuai. Nah, jalankan perintah Remove-AzVm



: 



$null = $vm | Remove-AzVM -Force


Hapus antarmuka jaringan dan alamat IP publik



VM kami memiliki satu (atau bahkan beberapa) antarmuka jaringan (NIC) - untuk menghapusnya jika tidak perlu, buka properti NetworkInterfaces



objek VM kami dan hapus perintah NIC Remove-AzNetworkInterface



. Jika ada lebih dari satu antarmuka jaringan, kami menggunakan loop. Pada saat yang sama, untuk setiap NIC, kami memeriksa properti IpConfiguration



untuk melihat apakah antarmuka memiliki alamat IP publik. Jika ditemukan, hapus dengan perintah Remove-AzPublicIpAddress



.



Berikut ini adalah contoh kode seperti itu, di mana kami mengulangi semua NIC, menghapusnya, memeriksa apakah ada IP publik. Jika ada, maka kami mengurai properti PublicIpAddress



, mendapatkan nama sumber daya yang sesuai dengan ID dan menghapusnya: 




foreach($nicUri in $vm.NetworkProfile.NetworkInterfaces.Id) {
    $nic = Get-AzNetworkInterface -ResourceGroupName $vm.ResourceGroupName -Name $nicUri.Split('/')[-1]
    Remove-AzNetworkInterface -Name $nic.Name -ResourceGroupName $vm.ResourceGroupName -Force

    foreach($ipConfig in $nic.IpConfigurations) {
        if($ipConfig.PublicIpAddress -ne $null) {
            Remove-AzPublicIpAddress -ResourceGroupName $vm.ResourceGroupName -Name $ipConfig.PublicIpAddress.Id.Split('/')[-1] -Force
        }
    }
}


Hapus drive sistem



Disk OS adalah gumpalan, untuk menghilangkan yang ada perintah Remove-AzStorageBlob



- tetapi sebelum Anda menjalankannya, Anda perlu mengatur nilai yang diperlukan untuk parameternya. Untuk ini, khususnya, Anda perlu mendapatkan nama wadah penyimpanan yang berisi disk sistem, dan kemudian mentransfernya ke perintah ini bersama dengan akun penyimpanan yang sesuai. 



$osDiskUri = $vm.StorageProfile.OSDisk.Vhd.Uri
$osDiskContainerName = $osDiskUri.Split('/')[-2]
$osDiskStorageAcct = Get-AzStorageAccount | where { $_.StorageAccountName -eq $osDiskUri.Split('/')[2].Split('.')[0] }
$osDiskStorageAcct | Remove-AzStorageBlob -Container $osDiskContainerName -Blob $osDiskUri.Split('/')[-1]


Hapus Status Disk Sistem Gumpalan



Untuk melakukan ini, seperti yang mungkin sudah Anda duga, kami mengambil wadah penyimpanan tempat disk ini disimpan, dan, dengan asumsi bahwa gumpalan berisi di akhir status



, kami meneruskan parameter yang sesuai ke perintah hapus Remove-AzStorageBlob



: 



$osDiskStorageAcct | Get-AzStorageBlob -Container $osDiskContainerName -Blob "$($vm.Name)*.status" | Remove-AzStorageBlob


Dan akhirnya, hapus cakram data



VM kami dapat memiliki disk dengan data yang melekat padanya. Jika tidak diperlukan, kami juga akan menghapusnya. Pertama kita parse StorageProfile



VM kita dan temukan propertinya Uri



. Jika ada beberapa disk, kami mengatur siklus dengan URI



. Untuk setiap URI, kami akan menemukan akun tersimpan yang sesuai dengannya Get-AzStorageAccount



. Kemudian, mari kita uraikan URI penyimpanan untuk mendapatkan nama gumpalan yang Anda butuhkan dan meneruskannya ke perintah hapus Remove-AzStorageBlob



bersama dengan akun Anda. Begini cara tampilannya dalam kode: 



if ($vm.DataDiskNames.Count -gt 0) {
    foreach ($uri in $vm.StorageProfile.DataDisks.Vhd.Uri) {
        $dataDiskStorageAcct = Get-AzStorageAccount -Name $uri.Split('/')[2].Split('.')[0]
        $dataDiskStorageAcct | Remove-AzStorageBlob -Container $uri.Split('/')[-2] -Blob $uri.Split('/')[-1]
    }
}
