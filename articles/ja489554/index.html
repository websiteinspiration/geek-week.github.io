式"class '" + getClassName() + "'"は同じであり、二重ループで同じラインを作成したくないので、ループの外側で1回作成することをお勧めします。以前は、このような例を見つけるのは偶然の問題でした。アプリケーションのデバッグ中に、ソース内でこの例が正常に失敗したことがわかりました。IDEA-230889のおかげで、これは自動化できます。もちろん、パッセージに関係なく、常にループ内に新しい行を作成することにはほど遠いですが、これらの場合でも、永続的な定数部分があるものは区別できます。


// org.springframework.beans.factory.support.BeanDefinitionReaderUtils

public static String uniqueBeanName(String beanName, BeanDefinitionRegistry registry) {
  String id = beanName;
  int counter = -1;

  // Increase counter until the id is unique.
  while (counter == -1 || registry.containsBeanDefinition(id)) {
    counter++;
    id = beanName + GENERATED_BEAN_NAME_SEPARATOR + counter;
  }
  return id;
}

ここではプレフィックスはあるbeanName + GENERATED_BEAN_NAME_SEPARATORので、それをすることができ、常に同じもたらしたアウト。
