レーベンシュタインの最短距離文字認識

最近、アプリケーションプログラムでの文字認識のタスクは特に難しくありません。多くのほぼ完成している既製のOCRライブラリを使用できます。それにもかかわらず、サードパーティの「洗練された」OCRライブラリを使用せずに独自の認識アルゴリズムを開発するタスクが発生する場合があります。


これはまさに私の作業中に発生したタスクであり、既製のライブラリを使用しない方がよい理由はいくつかあります。クローズドプロジェクト、その追加の認定、コードの行数と接続されたライブラリのサイズに対する特定の制限、サブジェクト領域で十分に認識する必要があるため、なおさらです。特定の文字セット。


認識アルゴリズムは単純であり、もちろん、最も正確で高速かつ効果的であるように見せかけることはありませんが、その小さなタスクにはうまく対応します。


構造化された形式のドキュメントのスキャンされた画像の形式で入力があるとします。これらのドキュメントには、左上隅に特別な1文字のコードがあります。私たちの仕事は、この記号を認識して、いくつかのアクションを実行することです。たとえば、指定されたルールに従ってソースドキュメントを分類します。

レーベンシュタイン距離を求める関数は、標準アルゴリズムによる方法として実装されています。

public static int levenshtein(String targetStr, String sourceStr) {
        int m = targetStr.length(), n = sourceStr.length();
        int[][] delta = new int[m + 1][n + 1];
        for (int i = 1; i <= m; i++)
            delta[i][0] = i;
        for (int j = 1; j <= n; j++)
            delta[0][j] = j;
        for (int j = 1; j <= n; j++)
            for (int i = 1; i <= m; i++) {
                if (targetStr.charAt(i - 1) == sourceStr.charAt(j - 1))
                    delta[i][j] = delta[i - 1][j - 1];
                else
                    delta[i][j] = Math.min(delta[i - 1][j] + 1,
                            Math.min(delta[i][j - 1] + 1, delta[i - 1][j - 1] + 1));
            }
        return delta[m][n];
    }

結果のfindSymbolは、認識される文字になります。

このアルゴリズムを最適化してパフォーマンスを向上させ、さまざまなチェックを追加して認識効率を向上させることができます。多くのインジケーターは、解決される問題の特定の主題領域（文字数、フォントの種類、画質など）に依存します。

この方法は、文字の「類似性」、たとえば「L」などの問題のある問題にも、定性的に対処できることがわかりました。<-> "P"、 "5" <-> "S"、 "O" <-> "0"。たとえば、バイナリ文字列「L」と「P」の間の距離は、「不規則性」があっても、認識された「L」と参照文字列「L」の間の距離より常に大きくなります。
