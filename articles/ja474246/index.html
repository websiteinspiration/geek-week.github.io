JavaScript Meetup SuperJob：ビデオレポート

10月10日に、JavaScript Meetup SuperJobがありました。参加者が話し合ったことを話し、ビデオやプレゼンテーションを共有します。



SuperJobの主任開発者であるAlexander Kukhtaは、「JSONの経験：クライアントでのAPI」というトピックについて話しました。

Alexanderは、同社のJavaScriptアプリケーションのアーキテクチャとパフォーマンスを担当するプラットフォームチームを代表しています。

最初のスピーカーとして、アレクサンダーはスーパージョブのフロントエンドがどのように機能するかを共有しました。彼はさらに、クライアントが約1,500個の未加工のエンティティをどのように格納するか、Reactコンポーネントでそれらをどのように処理するか、そしてアプリケーションがスローダウンしないようにどのように実行するかについても話しました。


→ アレクサンダーのプレゼンテーション
→ スピーチのデモ


SuperJob開発チームのリーダーであるPeter Solopovは、「タイプシステムとフロントエンドタイピング」について話しました。

ピーターのレポートでは、質問に対する回答を見つけることができます。

  • どんな型システムが存在し、それらはどのように進化しましたか？
  • FlowとTypeScriptの主な違いは何ですか？
  • SuperJobのような大きなプロジェクトでFlowはどのように使用されますか？

そしてそこには、多くの学生がベンチで忘れていた可能性が高い数学がたくさんあります。


→ ピーターのプレゼンテーション


SuperJobフロントエンド開発マネージャーのAlexander Savinは、「JSONサーバーの作成：Node.jsとPostgreSQL上のAPI」というトピックについて話しました。

Alexanderは、JSON：JavaScript APIに従ってサーバーを作成した経験を共有しました。
彼は、既存のデータベースのAPIをすばやく取得するのに役立つツールと、Node.jsとPostgreSQLを使用してそのようなツールを自分で作成する方法を説明しました。
Alexanderはまた、「最初のDB」アプローチを使用するときに起こりうる問題について触れ、それらを最小限に抑えるためにアプリケーションアーキテクチャで何を提供する必要があるかを説明しました。


→ アレクサンダーのプレゼンテーション
