Spring BootでのJsonシリアル化ユニットテスト



前書き


各Webサービスの主なタスクの1つは、モデルをクライアント側に戻すことです。この場合、Spring Bootは便利なレベルの抽象化を提供し、開発者がモデルを操作するレベルに留まり、モデルのシリアル化プロセスをプログラムのソースコードの外部に残すことができます。しかし、シリアル化自体がアプリケーションのビジネスロジックの一部になり、テストケースのカバレッジが必要になった場合はどうなるでしょうか。

この記事では、シリアル化中にアプリケーションのビジネスロジックの特殊性（金額を丸めるシナリオ）を考慮する必要がある場合のシナリオの1つを検討し、Spring Bootでシリアル化メカニズムに遭遇する例と、可能なテスト方法について説明します。

問題の定式化


Webサービスが顧客の費用に関する情報を提供する責任を負うようにします。構成可能な精度でデータを提供する必要があります。論理的な解決策は、丸めロジックを適用する可視性を維持しながら、すべてのモデルをサービスの周辺に変換することです。

可能な解決策を検討する


アプリケーションのコントローラーを想像してみてください。これにより、目的のモデルが返されます。

カスタマイズのために他の注釈を処理する必要があったかもしれません。この注釈の機能は、注釈付きモデルフィールドのシリアル化を担当するサービスを決定する機能です。

シリアライザが何であるかを見る前に、タスクを複雑にします。動的パラメータ解決のあらゆる症状を抽象化する内部サービスを通じて丸めパラメータを構成できるようにします。

この複雑さは、私たちが考慮したい重要なポイントです。モデルの実装からわかるように、フレームワークのAPIは引数にシリアル化クラスを使用します。これは、シリアライザーのライフサイクルがシリアライザーフレームワークの制御下に置かれることを意味します。これは、アプリケーションのコンテキストからシリアライザに依存関係を注入したい場合はどうすればよいかという疑問を引き起こします。これを行うには、上記のシリアライザの実装を検討してください。

//     ,
//  Jackson   Spring,
// API   
//    Spring DI
@JsonComponent
public class MoneySerializer extends JsonSerializer<BigDecimal> {

    //  ,   ,
    //    Spring Boot    Bean'.
    private RoundingHolder roundingHolder;

    @Autowired
    public MoneySerializer(RoundingHolder roundingHolder) {
        this.roundingHolder = roundingHolder;
    }

    //       ,
    // ,   ,      -
    //      .
    @Override
    public void serialize(BigDecimal value, JsonGenerator jsonGenerator, SerializerProvider serializerProvider) throws IOException {
        jsonGenerator.writeNumber(value.setScale(roundingHolder.getPrecision(), roundingHolder.getRoundingMode()));
    }
}

サービスの準備は整っていますが、責任ある開発者として、組み立てたキッチンが機能することを確認したいと考えています。

この瞬間を詳しく見てみましょう。ジャクソンはObjectMapperクラスを使用して、モデルをシリアル化および逆シリアル化します。これはまさにモデルの変換を担当するコンテキストオブジェクトです。したがって、モデルがどのように表示されるかを確認するには、ObjectMapperがコンテキストからそれを処理する方法を確認する必要があります。

独自のカスタムObjectMapperを作成する場合は、次の典型的な例に出くわすことができます：ObjectMapper mapper = new ObjectMapper。ただし、Spring Bootがデフォルトでこのクラスのインスタンスを作成する方法を見てください。これを行うには、オブジェクトの作成を担当する元のJacksonAutoConfiguration自動構成コードを使用します。

@Bean
public ObjectMapper jacksonObjectMapper(Jackson2ObjectMapperBuilder builder) {
    return builder.createXmlMapper(false).build();
}

さらに進んでbuild（）を見ると、デフォルトのマッパー（サービスをカスタムシリアライザーに挿入するなど）での作業に慣れている可能性があるシリアル化については、Beanマッパーを作成するだけでは不十分であり、提供されているビルダーを使用する必要があります。 。ちなみに、Spring Bootのドキュメント自体では、これが明示的に述べられています。余談ですが、JacksonTester

への参照を追加したいと思います。MockitoのコンテキストでのBDDシリアル化テストのシェルの代表として。

要約する


  • Spring Bootは、JsonSerializerアノテーションを介してモデルのシリアル化をカスタマイズする機能を提供します
  • シリアル化をテストするには、アプリケーションと同じ構成でマッパーを使用します
  • Spring Beanの自動構成からBeanをオーバーライドするときは、デフォルトのBeanが持っていた機会を逃さないように、このBeanがSpring Boot自体を作成する方法に注意してください。
  • JsonTestアノテーションを使用して、シリアル化のテストに必要な限定されたコンテキストを指定できます。
