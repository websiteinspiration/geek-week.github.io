Faktanya adalah menjadi budak integrasi ini, kami harus mendukung antarmuka pertukaran data dari luar. Yaitu - file data biner "file data COBOL".


Ternyata (dan kami tidak punya pengalaman dengan COBOL sebelumnya), file data di dalamnya memiliki struktur non-linear . Yaitu:


  • Satu dan blok data yang sama dapat diartikan sebagai "kelompok" yang berbeda (format data dalam terminologi COBOL), semacam polimorfisme data: direktif redefines. Apalagi dengan semua kelompok yang bersarang dan bidangnya.
  • Sebuah blok dapat diulang N kali: direktif occurs
  • Blok berjalan berurutan, tetapi permulaannya dihitung dari akhir blok sebelumnya - karena itu, Anda tidak dapat membaca blok terpisah tanpa membaca blok sebelumnya.
