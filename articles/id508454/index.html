Basis kode. Memperluas RecyclerView



Halo!



Nama saya Anton Knyazev , pengembang senior Android di Omega-R. Selama tujuh tahun terakhir, saya telah mengembangkan aplikasi seluler secara profesional dan memecahkan masalah kompleks pengembangan asli.



Saya ingin berbagi metode ekstensi RecyclerView yang dikembangkan oleh tim kami dan saya. Mereka akan menjadi basis yang andal untuk membuat daftar khusus dalam aplikasi.



Setiap aplikasi unik dengan caranya sendiri karena ide, desain, dan tim spesialisnya. Keputusan yang sukses seringkali ingin ditransfer dari satu proyek ke proyek lainnya. Oleh karena itu, alih-alih hanya menyalin, logis untuk membuat perpustakaan terpisah yang akan digunakan dan ditingkatkan oleh seluruh tim.



Tim memutuskan untuk membuat perpustakaan kecil yang meningkatkan dan mempercepat pengembangan aplikasi, dan menempatkannya di repositori GitHub publik . Ini membuatnya mudah untuk menghubungkan perpustakaan dalam proyek-proyek melalui JitPack dan memberi pelanggan jaminan bahwa tidak ada "kriminal" dalam kode.



Pustaka pertama yang kami posting di GitHub adalah ekstensi sederhana dari RecyclerView.



Mari kita mulai dengan masalah yang dia selesaikan:



  1. Tidak ada layoutManager default - ini merepotkan, karena seringkali Anda harus memilih LinearLayoutManager yang sama;
  2. Tidak ada cara untuk menambahkan pembagi dan ruang item melalui xml - ini juga merepotkan, karena Anda perlu menambahkan tata letak item atau melalui ItemDecorator;
  3. Anda tidak bisa hanya menambahkan header dan footer melalui xml - ini hanya dimungkinkan melalui ViewHolder terpisah.


Masalahnya tidak kritis, tetapi mereka menciptakan ketidaknyamanan dan meningkatkan waktu pengembangan.



1. Masalah: tidak ada manajer tata letak default



Pengembang RecyclerView tidak menyediakan kemampuan untuk memilih LayoutManager default. Anda dapat mengatur layoutManager dengan cara berikut:



1. Via XML di atribut aplikasi: layoutManager = "LinearLayoutManager": 



<?xml version="1.0" encoding="utf-8"?>
<androidx.recyclerview.widget.RecyclerView
   ...
    app:layoutManager="LinearLayoutManager"/>


2. melalui kode: 



recyclerView.layoutManager = LinearLayoutManager(this)


Dalam pengalaman kami, dalam banyak kasus, LinearLayoutManager diperlukan.



Berikut adalah beberapa contoh daftar seperti itu dari aplikasi ITProTV, Simple World, dan Dexen kami:







Solusi: tambahkan layoutManager default



Hanya 3 baris yang ditambahkan ke OmegaRecyclerView: 



 if (layoutManager == null)  {
            layoutManager = LinearLayoutManager(context, attrs, defStyleAttr, 0)
 }


Jadi, ketika LinearLayoutManager diperlukan, maka Anda tidak perlu menambahkan apa pun, yaitu, Anda bisa melupakan layoutManager. 



<?xml version="1.0" encoding="utf-8"?>
<com.omega_r.libs.omegarecyclerview.OmegaRecyclerView
    android:id="@+id/recyclerview"
    android:layout_width="match_parent"
    android:layout_height="match_parent" />


2. Masalah: tidak ada cara untuk menambahkan pembagi dan ruang item melalui xml



Anda harus menambahkan pembagi cukup sering saat menggunakan RecyclerView. Misalnya, dalam proyek "Dunia Sederhana", salah satu layar dengan pembagi non-standar:







Dari tata letak ini dapat dilihat bahwa:



  • digunakan pembagi antara elemen dan di bagian paling akhir;
  • ruang barang bekas.


Bagaimana ini bisa diterapkan di Android dengan cara standar?



Metode 1



Cara yang paling jelas adalah memasukkan pembagi sebagai elemen ImageView: 



 <RelativeLayout
   ...
   android:paddingStart="20dp"
   android:paddingTop="12dp"
   android:paddingEnd="20dp"
   android:paddingBottom="12dp">

...

   <ImageView
       ...
       android:layout_alignParentBottom="true"
       android:src="@drawable/divider"/>

</RelativeLayout>


Mungkin saja Anda hanya perlu melakukan pembagi antar elemen. Dalam hal ini, Anda harus menghapus pembagi terakhir dan menambahkan kode untuk menyembunyikannya di adaptor.



Metode 2



Cara lain adalah dengan menggunakan DividerItemDecoration, yang dapat ditarik oleh pembagi ini. Untuk itu, Anda juga harus membuat drawable: 



<?xml version="1.0" encoding="utf-8"?>
<layer-list xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:left="32dp">

        <shape android:shape="rectangle">

            <size
                    android:width="1dp"
                    android:height="1dp" />

            <solid android:color="@color/gray_dark" />

        </shape>

    </item>

</layer-list>


Untuk menambahkan indentasi, Anda perlu menulis ItemDecoration Anda: 



class VerticalSpaceItemDecoration(private val verticalSpaceHeight: Int): RecyclerView.ItemDecoration {

    override fun getItemOffsets(outRect: Rect, view: View, parent: RecyclerView,
            state: RecyclerView.State) {
        outRect.bottom = verticalSpaceHeight
    }

}


DividerItemDecoration sederhana: selalu menarik pembagi di bawah setiap item daftar.

Tetapi dalam hal terjadi perubahan persyaratan, seseorang harus mencari solusi lain.



Solusi: tambahkan kemampuan untuk menambahkan pembagi dan ruang item melalui xml



Jadi, OmegaRecyclerView kami harus dapat menambahkan pembagi menggunakan atribut berikut:



  1. divider ‚Äì drawable, ;
  2. dividerShow (beginning, middle, end) ‚Äì , , ;
  3. dividerHeight ‚Äì divider, ;
  4. dividerPadding, dividerPaddingStart, dividerPaddingEnd ‚Äì : , , ;
  5. dividerAlpha ‚Äì ;
  6. itemSpace ‚Äì .


Semua atribut ini diterapkan langsung ke OmegaRecyclerView kami menggunakan dua ItemDecoration khusus.



Salah satu ItemDecoration menambahkan padding antar item, yang kedua menarik pembagi itu sendiri. Ingatlah bahwa jika ada lekukan di antara elemen-elemen, ItemDecoration kedua menarik pembagi di tengah indentasi. Tidak ada cara untuk mendukung semua opsi LayoutManager yang mungkin, jadi kami hanya mendukung LinearLayoutManager dan GridLayoutManager. Perhatikan juga orientasi daftar untuk LinearLayoutManager.



Untuk menyederhanakan kode, kami memilih DividerDecorationHelper khusus, yang akan membaca dan menulis data relatif dalam Rect, tergantung pada orientasi dan urutan (terbalik atau langsung). Ini akan memiliki metode setStart, setEnd dan getStart, getEnd.



Mari kita buat ItemDecoration dasar untuk dua kelas, yang akan berisi logika umum, yaitu:



  1. memeriksa bahwa layoutManager adalah turunan dari LinearLayoutManager;
  2. perhitungan orientasi dan urutan saat ini;
  3. menentukan DividerDecorationHelper yang sesuai.


Di SpaceItemDecoration, kami hanya menimpa satu metode getItemOffset, yang akan menambahkan lekukan: 



override fun getItemOffset(outRect: Rect, parent: RecyclerView, helper: DividerDecorationHelper, position: Int, itemCount: Int) {
        if (isShowBeginDivider() || countBeginEndPositions <= position) helper.setStart(outRect, space)
        if (isShowEndDivider() && position == itemCount - countBeginEndPositions) helper.setEnd(outRect, space)
    }


DividerItemDecoration selanjutnya akan menarik pembagi secara langsung. Ini harus memperhitungkan lekukan di antara elemen-elemen dan menggambar pembagi di tengah. Pertama, mendefinisikan kembali metode getItemOffset untuk kasus ketika indentasi tidak ditentukan, tetapi pembagi diperlukan untuk menggambar. 



    override fun getItemOffset(outRect: Rect, parent: RecyclerView, helper: DividerDecorationHelper, position: Int, itemCount: Int) {
        if (position == 0 && isShowBeginDivider()) {
            helper.setStart(outRect, dviderSize)
        }
        if (position != 0 && isShowMiddleDivider()) {
            helper.setStart(outRect, dividerSize)
        }
        if (position == itemCount - 1 && isShowEndDivider()) {
            helper.setEnd(outRect, dividerSize)
        }
    }


Kami juga menambahkan opsi yang memungkinkan DividerItemDecoration untuk bertanya pada adaptor apakah mungkin untuk menggambar di atas atau di bawah item yang dipilih. Untuk menerapkan fitur ini, buat adaptor Anda sendiri yang mewarisi dari yang standar dengan metode berikut: 



   open fun isDividerAllowedAbove(position: Int): Boolean {
        return true
    }

    open fun isDividerAllowedBelow(position: Int): Boolean {
        return true
    }


Selanjutnya, mendefinisikan kembali metode onDrawOver untuk menggambar pembagi di atas elemen yang ditarik. Dalam metode ini, Anda harus melihat semua elemen yang terlihat di layar (via getChildAt) dan, jika perlu, gambar pembagi ini. Perlu juga dicatat bahwa warna yang tidak memiliki ketinggian dapat berasal dari atribut dividerDrawable. Untuk kasus ini, ketinggian dapat diambil dari atribut dividerHeight.



3. Masalah: Anda tidak bisa langsung menambahkan header dan footer melalui xml







Di RecyclerView tidak mungkin untuk menambahkan tampilan melalui xml, tetapi ada cara lain untuk melakukan ini.



Metode 1



Satu cara yang jelas untuk menambahkan tampilan adalah melalui adaptor. Selain itu, perlu untuk membedakan antara header dan footer di adaptor saat memasukkan pengenal Anda untuk tipe tampilan. 



 fun onCreateViewHolder(parent: ViewGroup, viewType: Int): RecyclerView.ViewHolder? {
	  val inflater = LayoutInflater.from(parent.context)
        return when (viewType) {
		 TYPE_HEADER -> {
                val headerView: View = inflater.inflate(R.layout.item_header, parent, false)
                HeaderViewHolder(itemView)
            }
            TYPE_ITEM -> {
                val itemView: View = inflater.inflate(R.layout.item_view, parent, false)
                ItemViewHolder(itemView)
            }
           
            else -> null
        }
    }


Metode 2



Cara yang sedikit berbeda, tetapi juga melalui adaptor. Dimulai dengan recyclerview: 1.2.0-alpha02, MergeAdapter telah muncul, yang memungkinkan Anda untuk menggabungkan beberapa adapter menjadi satu, membuat kode lebih bersih. 



val mergeAdapter = MergeAdapter(headerAdapter, itemAdapter,  footerAdapter)
recyclerView.adapter = mergeAdapter


Solusi: tambahkan kemampuan untuk hanya menambahkan header dan footer melalui xml



Hal pertama yang harus dilakukan adalah mencegat penambahan tampilan di OmegaRecyclerView kami saat proses mengembang sedang berlangsung. Untuk melakukan ini, ganti metode addView dan tambahkan semua tampilan header dan footer ke diri Anda sendiri. Metode ini digunakan oleh RecyclerView sendiri untuk menambahkan item daftar yang terlihat. Tetapi tampilan yang ditambahkan melalui xml tidak akan memiliki ViewHolder, yang pada akhirnya akan melempar NullPointerException.



Jadi, kita perlu menentukan kapan tampilan ditambahkan saat mengembang. Untungnya, ada metode onFinishInflate yang dilindungi yang dipanggil saat proses penggelembungan berakhir. Karenanya, saat memanggil metode ini, kami menandai bahwa proses penggelembungan selesai. 



  protected override fun onFinishInflate() {
        super.onFinishInflate()
        finishedInflate = true
    }


Dengan demikian, metode addView akan terlihat seperti ini: 



  override fun addView(view: View, index: Int, params: ViewGroup.LayoutParams) {
        if (finishedInflate) {
            super.addView(view, index, params)
        } else {
            // save header and footer views
        }
    }


Selanjutnya, Anda perlu mengingat semua tampilan tambahan ini dan mentransfernya ke adaptor khusus sebagai MergeAdapter.



Kami juga berhasil memecahkan masalah lain: ketika metode findViewById dipanggil, pandangan kami tidak akan dikembalikan. Untuk mengatasi masalah ini, kami mendefinisikan kembali metode findViewTraversal: perlu untuk membandingkan id tampilan yang kami temukan dan mengembalikan tampilan jika cocok. Karena metode ini disembunyikan, kita cukup menulisnya tanpa menunjukkan bahwa itu menimpa.



Anda dapat berkenalan dengan fitur-fitur ini dan fitur berguna lainnya dengan deskripsi terperinci di perpustakaan OmegaRecyclerView kami :



  1. Pada tahun 2018, kami menciptakan ViewPager kami dari RecyclerView. Selain itu, di ViewPager kami ada gulir yang tak ada habisnya;
  2. ExpandableRecyclerView - kelas khusus untuk menambahkan daftar drop-down, dengan kemampuan untuk memilih animasi ekspansi;
  3. StickyHeader adalah item daftar spesifik yang dapat ditambahkan melalui adaptor.
