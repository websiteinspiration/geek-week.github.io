GitHubリポジトリセキュリティツールの概要



前書き


GitHubでリポジトリの作成を開始するとき、最初に考える必要があるのはセキュリティです。

独自のGitHubリポジトリを作成している場合、またはリポジトリに頻繁に貢献している場合は、コードに脆弱性が含まれているかどうかを知る必要があります。過去のリポジトリの脆弱性がセキュリティ問題を引き起こしていました。これは、最近の最大のデータリークの2つ（EquifaxとHeartbleed SSL Exploit）が、将来使用される可能性のある対応するオープンソースコンポーネントの脆弱性から始まったという事実によって強調されました。

この投稿では、GitHubリポジトリの脆弱性を特定するために使用できる4つの個別のツールを見て分析します。これらの4つのツールにはそれぞれ独自の超能力がありますが、それぞれに弱点があります。この記事は、オープンソースプロジェクトに適したツールを選択するのに役立ちます。

この記事は、ジュニアに実用的なアドバイスを提供する EDISON Softwareのサポート、およびソフトウェアの設計とTKのロシア語と英語での翻訳で翻訳されました。


ガードレール




GuardRailsは、GitHubの市場で入手できるフリーミアムセキュリティアプリケーションです。 GuardRailsは、静的なコード分析と脆弱な依存関係の識別を提供できます。脆弱性を含むコメントをリクエストプールに書き込みます。

アプリケーション自体がリアルタイムでユーザーコードの新しいエントリをスキャンするため、ユーザーは脆弱性が現れた直後にすぐに脆弱性を排除するためのアクションを実行できます。これは、侵入者からリポジトリとコードを保護するのに役立ちます。リクエストプールに関して、GuardRailsはセキュリティの問題を検出したときに各リクエストにコメントを書き込みます。ブランチを使用すると、この情報はGuardRailsダッシュボードに表示されます。

GuardRailsサービスの基本原則は、包括的で迅速なセットアップであり、ユーザーはGuardRailsをすべてのリポジトリと数分で統合できます。GuardRailsをSlackと統合して、通知がより効率的に届くようにすることもできます。

GuardRailsは現在、Python、Ruby、JavaScript、Solidity、Go、Java、PHPをサポートしています。

ホワイトソースボルト




WhiteSource Boltは、GitHubユーザーがリポジトリのスキャンを作成できるようにし、コードに現れる可能性のあるオープンソースの脆弱性を識別できるようにします。これは、オープンソースの分野におけるセキュリティ、ライセンス、レポートの分野の専門家であるWhiteSourceによって提供されています。彼らは2011年以来市場で活動しており、210万人以上の異なる開発者の助けを期待できます。

彼らのサービスは、プッシュアクションが発生するたびにBoltがリポジトリのスキャンを開始し、検出された脆弱性ごとに問題を作成するように機能します。また、既存のオープンソースコードコンポーネントで発見された新しい脆弱性の問題も引き起こします。さらに、脆弱性を含むリクエストのプールを自動的にキャンセルすることで、脆弱なコンポーネントがコードに入るのを防ぐことができます。

また、Boltはユーザーに独自のWhiteSource脆弱性データベースへのアクセスを提供します。これは広範囲であり、最も高価なオープンソースセキュリティ市場であると多くの人が考えています。CVEおよびCVSSデータ、提案された修正、脆弱なコンポーネントへのパス、ヘルプへのリンクなど、発見された脆弱性に関する情報を受け取ります。

Boltは現在、Java、Python、PHP、C＃、C ++など、200以上の異なるプログラムをサポートしています。

LGTM




LGTMは無料のオープンソースプロジェクトアプリであり、ユーザーがコードの潜在的な脆弱性を特定し、そもそもそれらが発生するのを防ぎます。特に、LGTMはゼロデイ脆弱性の発見に焦点を当てたセキュリティ調査チームが収集したデータを使用します。 700,000人を超える開発者と135,000人を超えるオープンソースプロジェクトがLGTMサービスの恩恵を受けており、このレベルの経験がサービスの品質を証明しています。 LGTM GitHubアプリはGitHubマーケットプレイスで入手できます。

リポジトリで作業する場合、LGTMはコードを自動的にスキャンして、表示される可能性のある脆弱性とCVEをチェックします。経験豊富なLGTM開発者と研究者の大規模なコミュニティのおかげで、彼らが提供するサービスがリポジトリのセキュリティに大きな利益をもたらす可能性があることを理解しています。これにより、クエリログを維持するよりもさらに簡単になり、潜在的な脆弱性がコードベースに入る前に検出できるようになります。

LGTMには現在、サポートされているプログラミング言語が多数あり、そのサポートはC、C ++まで拡張されています。COBOL、Python、Javascript、およびJava。

Githubセキュリティアラート




GitHub Security Alertsは、依存関係のあるGitHubリポジトリの所有者とメンバーに提供される無料のサービスです。ユーザーは独自の依存関係グラフを使用して、依存関係に脆弱性があるかどうかを確認でき、これらの脆弱性を修正するための提案をユーザーに提供します。

GitHubが潜在的な脆弱性を通知すると、GitHubが更新する必要のある依存関係についてアドバイスする更新を受け取ります。依存関係の既知の安全なバージョンがある場合、GitHubは機械学習を使用してそれを選択し、推奨に含まれます。
各脆弱性に関する情報に関して、GitHubは、影響を受ける脆弱性、影響を受けるバージョンの範囲、CVE ID、および脆弱性データベースに含まれる修正案を通知します。

このサービスは現在、JavaScript、Ruby、Pythonをサポートしています。
