アイドル状態のHDDからのクラフト-ミニポンプ



どういうわけか私は将来の自家製製品のための水ポンプを必要としていました。はい、それは単純ではありません-寸法に制限があります-厚さ25mmまで、幅50mmまで（長さ-すでに異なる場合があります）。望ましい特性-1mの圧力と100l / hの流量。（主にサイズの点で）販売で彼が欲しかったものを見つけるのではなく、彼の頑固で頑固な性質の中で、彼はこの問題への彼のソリューションの実装に取り掛かりました！

注意-たくさんの写真！

「脳」と小さな背景：


厳密に言うと、ポンプにHDDモーターを使用するという考えは新しいものではありません。 2009年以降、ブランチ全体が1つの有名なフォーラムで維持されています。そのため、もともとは「ラップトップ」ハードドライブからポンプを製造し、電源キーとセンサーレスコントロールが統合された適切なドライバーを見つけることを目的としていました。

しかし、「そのまま」私は経験から学ぶことができませんでした。さまざまなドライバー（MTD6501C、DRV11873、およびその他の中国の工芸品）を使用した一連のテストでは、残念な結果が得られました。3.5ドライブの大型モーターは完全に動作します。しかし、小さなモーターでは、せいぜい、ユニットを始動することは可能であり、それらは非常に不安定に動作します。そのような残念な結果で、古いアイデアは放棄され、忘却の瀬戸際にありました。

しかし、比較的最近、TIからかなり興味深いドライバーに出会いました。DRV10987。その控えめな次元で、それは非常に印象的な可能性を持っています：

  • 6Vから28Vまでの動作電圧
  • 統合された5v降圧コンバーター（MKを制御用に給電できます）
  • 最大2Aの一定動作電流（ピーク-3A）
  • モーターの動作を制御するための膨大な数のソフトウェア定義パラメーター（I2Cバス上の構成レジスターの値を設定）
  • 緊急停止/故障後のモーターの自動再始動（故障の条件が過ぎた場合）
  • 過電流保護
  • サージ保護
  • ローターストップ/ロック検出
  • コントローラーが過熱したときにシャットダウン

arduinka（ローカルエレクトロニクスエンジニアがこれを許してくれる）を装備 して、このドライバー用のLUTボードによって作成されたパラメーターを設定し、モーターの始動に関する実験をさらに詳しく行いました。さて、このコントローラーは私を失望させませんでした！パラメータの選択における「科学的突破」の適用方法にもかかわらず、HDDから任意のモーターへのアプローチを見つけることができました！

この自作のオンライン設定コンフィギュレーターで私を助けました。健康に使用してください！）

これはarduinoを介してパラメータを注ぐことについてのスケッチです：
#include <Arduino.h>
#include <Wire.h>

#define I2C_DRV10983_Q1_ADR 0x52
#define Fault_Reg 0x00 
#define MotorSpeed_Reg 0x01 
#define DeviceIDRevisionID_Reg 0x08
#define SpeedCtrl_Reg 0x30 
#define EEPROM_Access_Code_Reg 0x31
#define EEPROM_EeReady_Reg 0x32
#define EEPROM_Iindividual_Access_Adr_Reg 0x33
#define EEPROM_Individual_Access_Data_Reg 0x34
#define EEPROM_Access_Reg 0x35
#define EECTRL_Reg 0x60

void setup() {
    Serial.begin(9600);
    Wire.begin(); 
}

byte readByAdress(byte reg_adr,  unsigned int &result) { //I2C write 2-byte register
    byte i = 0, err = 0;
    byte bytes[2] = {0, 0}; 
    Wire.beginTransmission(I2C_DRV10983_Q1_ADR); 
    Wire.write(reg_adr); 
    err = Wire.endTransmission();
    if(err!=0)
        return err;
    Wire.requestFrom(I2C_DRV10983_Q1_ADR, 2);
    while(Wire.available()) 
    { 
        bytes[i] = Wire.read();
        i++;
    }
    result = ((bytes[0] << 8) | bytes[1]);
    return 0;
}

byte writeByAdress(byte reg_adr,  unsigned int value) { //I2C read 2-byte register
    byte bytes[2]; 
    bytes[1] = value & 0xFF;
    bytes[0] = (value >> 8) & 0xFF;
    Wire.beginTransmission(I2C_DRV10983_Q1_ADR); 
    Wire.write(reg_adr); 
    Wire.write(bytes,2);
    return Wire.endTransmission();
}

boolean flag = true;
void loop() {
    if(flag){
        unsigned int onReady = 0;
        writeByAdress(EECTRL_Reg, 0xFFFF);
        writeByAdress(EEPROM_Access_Code_Reg, 0x0000); //Reset EEPROM_Access_Code_Reg
        writeByAdress(EEPROM_Access_Code_Reg, 0xC0DE); //Set EEPROM_Access_Code_Reg
        while(onReady == 0){ // Wait EEPROM ready
            readByAdress(EEPROM_EeReady_Reg, onReady);
        }
        Serial.println("EEPROM_Access.");
        onReady = 0;
        //Write values on shadow registers
        //writeByAdress(EEPROM_Access_Reg, 0x1000); //Not use EEPROM storage. Store values in shadow registers
        writeByAdress(0x90, 0x154F);
        writeByAdress(0x91, 0x042C);
        writeByAdress(0x92, 0x0090);
        writeByAdress(0x93, 0x09EA);
        writeByAdress(0x94, 0x3FAF);   
        writeByAdress(0x95, 0xFC33);
        writeByAdress(0x96, 0x016A);
        writeByAdress(EEPROM_Access_Reg,0x0006); //EEPROM mass access enabled && update 
        while(onReady == 0 ){ // Wait EEPROM ready
            readByAdress(EEPROM_EeReady_Reg, onReady);
        }
        Serial.println("EEPROM_Update.");
        writeByAdress(EECTRL_Reg, 0x0000); //Run motor
        flag = false;
    }    
}
