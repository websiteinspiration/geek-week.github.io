Erfahrung in der Verwendung der Rutoken-Technologie zur Registrierung und Autorisierung von Benutzern im System (Teil 2)

Guten Tag! Wir werden uns weiterhin mit diesem Thema befassen ( den vorherigen Teil finden Sie hier ).

Heute wenden wir uns dem praktischen Teil zu. Beginnen wir mit der Einrichtung unserer Zertifizierungsstelle basierend auf einer vollständigen OpenSSL-Open-Source-Bibliothek. Dieser Algorithmus wurde unter Windows 7 getestet.

Nach der Installation von openSSL können wir verschiedene kryptografische Vorgänge (z. B. das Erstellen von Schlüsseln und Zertifikaten) über die Befehlszeile ausführen.

Der Aktionsalgorithmus lautet wie folgt:

  1. Laden Sie das Installationsverteilungspaket openssl-1.1.1g herunter.
    OpenSSL hat verschiedene Versionen. In der Dokumentation zu Rutoken heißt es, dass openSSL Version 1.1.0 oder höher erforderlich ist. Ich habe die Version openssl-1.1.1g verwendet. Sie können openSSL von der offiziellen Website herunterladen. Für eine einfachere Installation müssen Sie jedoch die Installationsdatei für Windows im Netzwerk suchen. Ich habe es für Sie getan : slproweb.com/products/Win32OpenSSL.html Sie
    müssen die Seite nach unten scrollen und Win64 OpenSSL v1.1.1g EXE 63MB Installer herunterladen.
  2. Installieren Sie openssl-1.1.1g auf dem Computer.
    Die Installation muss gemäß dem Standardpfad erfolgen, der automatisch im Ordner C: \ Programme angegeben wird. Das Programm wird im OpenSSL-Win64-Ordner installiert.
  3. Um openSSL nach Bedarf zu konfigurieren, gibt es eine openssl.cfg-Datei. Diese Datei befindet sich im Pfad C: \ Programme \ OpenSSL-Win64 \ bin, wenn Sie openSSL wie im vorherigen Absatz beschrieben installiert haben. Wechseln Sie in den Ordner, in dem openssl.cfg gespeichert ist, und öffnen Sie diese Datei beispielsweise mit Notepad ++.
  4. Sie haben wahrscheinlich vermutet, dass das Zertifizierungscenter durch Ändern des Inhalts der Datei openssl.cfg eingerichtet werden würde, und Sie haben absolut Recht. Konfigurieren Sie dazu den Befehl [ca]. In der Datei openssl.cfg befindet sich der Anfang des Textes, an dem Änderungen vorgenommen werden, wie folgt: [ca].
  5. Jetzt werde ich ein Konfigurationsbeispiel mit seiner Beschreibung geben:
