这是发送邮件的示例功能：
public function email($emails, $body, $subject)
{
    if (empty($emails)) {
        return false;
    }

    $prefix = getenv('NATS_PREFIX');
    $encoder = new JSONEncoder();
    $options = new ConnectionOptions([
        'token' => getenv('NATS_TOKEN')
    ]);
    $client = new EncodedConnection($options, $encoder);

    try {
        $client->connect();
        $client->publish(
            $prefix . 'email',
            [
                'Emails' => $emails,
                'HtmlBody' => $body,
                'TextBody' => strip_tags($body),
                'Subject' => $subject,
                'Sender' => 'info@' . getenv('DOMAIN_NAME')
            ]
        );
    } catch (\Exception $exception) {
        return false;
    }

    return true;
}

我们的产品面向欧洲，因此价格以欧元计。

我们遇到了什么困难：

  1. 您可以指望具有货币终端的手指支付网关。它是什么？这种东西可以让您接受欧元付款，而不是卢布付款。为什么这个这么重要？是的，因为欧洲人对俄罗斯人保持警惕，所以有很多骗子。支付时，卢布会减少转换并增加支持票证，您必须在其中解释您的身份和身份。
  2. 货币终端的协调需要时间。准备好从5天开始。在实践中大约为10。因为您将被仔细检查，而欧洲人不愿隔离。此外，我们的国内银行也会提出问题，为什么您需要它
  3. 尚未取消3d安全（SMS，带有付款确认码）。因此，如果欧洲的银行不支持欧洲客户的这种可能性，那么他将无法付款。在这种情况下，您必须在PayPal上用笔打败客户
  4. 只有当所有内容都已存在时，才进行站点的协调。那些。该网站已完全准备就绪，包括所有法律文件（例如报价）。因为 我们没有在线商店，通常没有一个典型的解决方案-我们求助律师为我们的产品及其后续翻译草拟报价

碰巧我们首先接受了卢布付款。在同意支付网关后，他们转而使用欧元。开始时这稍微减少了转换。

流媒体组织


如果通过授权一切都很简单，那么计费就可以自己搞定。此类任务通常在一般的统计开发人员中找到。但是使用流技术时，一切都不像乍看起来那样清晰。怎么了？

  1. 我们从未与他打交道。我需要一个服务器+客户端的客户端
  2. 图片需要镜像，即 进行垂直显示（舞蹈视频的功能）
  3. 我们不知道营地中会有多少参与者。而且我们还不能为任何地方（在现成的流媒体解决方案中）的流量\月份\年份\数量购买一个帐户。那会吞噬所有的利润
  4. 我至少需要通过引荐来源网址来防止内容盗用的iframe保护

作为解决方案，我们选择了一堆

Larix Broadcaster应用程序-> nginx + rtmp-> cdn->客户端，在搜索过程中，用于从移动设备广播RTMP的Larix Broadcaster应用程序被证明是最可靠，最稳定的。在单独的小型虚拟服务器上，我们提出了nginx + rtmp以进一步流向CDN。

这个方案是最便宜的。比使用现成的流媒体平台的服务便宜得多。您不需要购买任何\月\年的高级帐户\黄金\超级帐户。您只需为花费的流量付费。在我们的情况下，这是完美的解决方案。
