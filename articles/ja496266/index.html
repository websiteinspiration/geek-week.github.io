死だけを数える


この疾患に関連する人の死亡数におけるCOVID-19の被害を評価することに注意してください。記事の大部分は、コロナウイルスによる死者数の予測に当てられます。コロナウイルスの患者数を考慮してその数を予測することは、一般的に拒否されました。これにはいくつかの理由があります。主なことは、異なる国でのCOVID-19の症例数に関する統計を比較することは不可能であることです。一部の国では、迅速な試験方法が他の国よりもはるかに多く利用できます。一部の国では、人口のほぼ普遍的なスクリーニングが実施されていますが、他の国では、重篤な症状のある人だけが検査されます。かなりの数の症例において、この疾患はほとんど無症候性であることを考えると、患者の死亡数には、0.5％未満から3.5％を超えるまで、非常に大きな広がりがあります。

おそらく、死亡率データのばらつきは、主にCOVID-19症例の検出によるものです。

この場合の死亡率統計ははるかに信頼できるように見えます。当然のことながら、死因がコロナウイルスとして示されていない場合の死者数の過小評価と、付随する疾患の両方、およびCOVID-19が誤って診断された場合の過大評価と、季節性インフルエンザなどによる人の死亡の両方が発生する可能性があります。ただし、死亡に関する統計はより信頼できると予想できます。確率の高い無症候性疾患は、一般に専門家の注意を通過する可能性があり、医師はすべての死亡例を分析する必要があります。

また、社会への実際の被害は、そのメンバーの死が原因であり、比較的早く通過する軽度の病気が原因ではないことにも注意してください。

問題は、世界は出展者ではなく、ロジスティック曲線によって支配されているということです。

出展者とは異なり、ロジスティック曲線は学校だけでなく、主要な技術大学（たとえば、モスクワ州立大学の物理学部やPhysTech）でも通過します。したがって、物理学者や技術者は一般にそれについて何も知らないことがよくあります。

それにもかかわらず、経済学、生物学、社会学、科学、技術における膨大な数の現象が、この数学的モデルに完全に沿って発展しています。それを詳しく見てみましょう。彼女が来た。時間はx軸にプロットされ、yには、私たちが調査した現象を特徴付ける数値（この場合、コロナウイルスによる死亡数）がプロットされます。



誇大宣伝全体を壊す陰湿な曲線


ロジスティック曲線は、2つの安定状態間の遷移を示します。低い方の状態は、伝統的にゼロに等しいと考えられています。上限状態は、原則として、調査された現象が達成できる最大値です。曲線は任意に近づきますが、最大値に達することはありません。

曲線の最も重要な点は変曲点です。最小値と最大値のちょうど中間に位置しています。この時点で、曲線の最大成長率になります。しかし、変曲がそこで起こります。この時点まで、曲線の成長は加速するだけです。後-フェードのみ。
通常、最初は特定の現象に誰も気づきませんでした（コロナウイルスは1月末まで気付かなかったため）。この時点で、曲線の値はゼロに近いです。徐々に、現象は成長し、彼らはそれに気づき始め、その周りに誇大宣伝が発生します。誇大広告は、ポジティブ（たとえば、ガガーリンの飛行時、世界中の人々が宇宙で病気になったため）とネガティブ（現在のコロナウイルスの状況）の両方です。誇大宣伝の時に、誰もが現象が信じられないほどの割合を獲得し、世界をひっくり返すと予測しています。そのため、ガガーリンの飛行時、専門家でさえ太陽系の征服は20世紀に起こると考えていました。そして、彼らは、宇宙飛行のすべての成功が1969年に人が月に着陸することで終わるとはまったく期待していませんでした。

曲線が将来の最大値の半分に達するのは、最大の誇大宣伝の時点です。その後、成長は衰退し、この現象はそれに対する期待（および恐怖）を完全に正当化しません。

最初は、新しい死者の数がほぼ指数関数的に増加していることがわかります。次に、成長が鈍化し始め、ロジスティック曲線がその最大値の半分に達したときに、新しい死亡の曲線が変曲点で最大に達します。その後、新たな死者の数は減少し、急いでゼロになります。赤い点は新しい死を示し、青い曲線はそれらを近似するロジスティック曲線の導関数です。

ロジスティック曲線は、その変曲点を中心に対称であり、その1次導関数は、この点を通る垂直線を基準にしています。また、実際のデータは理想的にはロジスティック曲線上にありますが、それらは1次導関数に対して「ダンス」していることにも注意してください。実際のところ、ある時点での死亡率はばらつきが大きく、総死亡率はそのような指標の合計です。これは、中心極限定理に従って平滑化されます。

したがって、ロジスティック曲線はコロナウイルスによる死亡を予測するために効果的に使用できることがわかります。ここで重要な特性はその対称性です。変曲点に到達すると、曲線の半分から高精度で残りの半分を復元できます。

犠牲者は正当化されますか？


そして最後に、記事の冒頭で提起された最後の質問：世界のほとんどの国の政府が実施した厳格な検疫措置はどの程度正当化されますか？
世界中で毎年約5800万人が亡くなっています。悲観的な予測によれば、200万人がコロナウイルスで死亡すると、この数字は3.5％増加します。一方、大規模な世界の検疫は、大恐慌以来の最大の経済危機へとエスカレートする恐れがあります。その結果、数千人、さらには数億人もの人々が仕事なしで留まることになります。人口の収入は減少し、多くは飢餓または医療費の支払い不能で死亡します。

一部の世界の指導者も含めて、人々を彼らの運命に任せ、経済を台無しにしない方が良いだろうという意見がしばしば表明されます。結局、毎年300〜65万人が季節性インフルエンザで亡くなっていますが、経済を破壊するような制限的な措置を取る人は誰もいません。

私たちのモデルでは、次のことを述べることができます。COVID-19は季節性インフルエンザではありません。このウイルスは比類のないほど危険です。実際には、ロジスティック曲線の変曲点はそれ自体では存在しません。流行の状況に大きく影響されます。流行の経過はロジスティック曲線で表されますが、ロジスティック曲線は家族全体です。変曲点はカーブの最大値のちょうど半分であることを覚えています。したがって、死亡率のピークが遅れて通過するほど、最終的な死亡数は多くなります。変曲点の近くでロジスティック曲線が可能な限り速く成長することがわかりました。したがって、病気のピークが10日後に過ぎると、犠牲者の数は数倍に増える可能性があります。

ロジスティック曲線の変曲点について受け取った予測には、すでに世界各国の政府が実施したすべての検疫措置が含まれています。そのような対策がなかった場合、変曲点はずっと後に到達しました。この場合、死亡率は全世界の人口の0.4-0.5％のレベルに達する可能性があります。

人口の0.4〜0.5％のチェックされていない流行から死亡率を推定するのはなぜですか。この場合、何らかの形で、地球の全人口がウイルスに罹患すると想定しています。しかし、かなりの数の人々において、この疾患は無症候性です。したがって、コロナウイルスの人口について可能な限り幅広い検査を組織し、実際の症例の大部分を特定することに成功した韓国やドイツなどの国の統計を使用します。他の国では、私たちの意見では、症例数が大幅に過小評価されているため、統計は正確に歪んでいます。したがって、3.5％という超高死亡率です。

世界の人口の0.4〜0.5％は2,800〜3,500万人です。人類がその歴史の中で繰り広げてきたすべての戦争について、第二次世界大戦での損失の数だけがこの数字を超えています。世界の国々の圧倒的多数の国の政府が人々を救うために前例のない経済的犠牲を払ったという事実は、人間の生命の価格が世界でどれほど成長したかを示しています。ヒューマニズムの考え方と個人の利益の優先順位がどれほど広がっているか。そして、これは著者に人類の誇りとすべての人類のより良い未来への希望を抱かせます。

この記事の最大の欠点


そして、これがこの記事の最大の欠点です。残念ながら、従来の統計では、ロジスティック関数に基づいて予測誤差を推定するためのツールは提供されていません。これは、変曲を持つ関数の形式によるものです。この変曲がなく、曲線が単調である場合、Box-Cox変換を使用して、関数を最初に線形形式にします。その後、線形回帰のエラー方程式を使用して、エラーの上限と下限を作成し、次に逆ボックスコックス変換を使用して、曲線エラー境界を取得します。これに基づいて、コロナウイルスによる死亡数の最大および最小予測を作成します。

悲しいかな、古典統計のツールは、変曲のある曲線の場合にエラー境界を構築することを不可能にします。ただし、機械学習の手法が役立つ場合があります。次の記事では、この場合にエラーの境界がどのように構成されるかを示し、上記で検討した各国および世界全体の死亡数の最小および最大予測を作成します。

入力引数が3桁の配列であるresLogistic関数を定義し、出力はロジスティック曲線の値と中国での流行による実際の死亡数の差の2乗の合計です。 

def resLogistic(coefficents):
    A0 = coefficents[0]
    A1 = coefficents[1]
    A2 = coefficents[2]
    teor = A0 / (1 + np.exp(A1 * (X.ravel() - A2)))
    
    return np.sum((teor - chi) ** 2)

死亡数は各日に累積合計として取得されます。ロジスティック曲線は、入力ベクトルの3つのコンポーネントによって決定されます。コンポーネントのゼロ（Pythonの配列要素の数はゼロから始まります）は最大値に関与し、最初は関数の成長率、2番目は時間軸上の変曲点の位置を示します。
teor-入力パラメータベクトルに基づく、毎日の死亡数、chi-中国での実際の死亡数。この関数は、理論値と実際の値の差の2乗の合計を返します。

次に、scipy.optimizeライブラリの最小化メソッドを使用して、偏差の2乗の合計を最小化するベクトルを見つけます。このベクトルに基づいて作成されたロジスティック曲線は、最小二乗法に基づいて行われた望ましい死亡率予測です。

最小化メソッドには開始点が必要であることを付け加えます。これは、既知のロジスティック関数の特性に基づいて選択します（最大値は、経験に基づく死亡数よりも大きく、変曲点は、1日あたりの新しい死亡数を表すdtot曲線の最大値に近い必要があります）。通常、最小化メソッドの結果は開始点とは関係ありませんが、例外があります。
mimim.xは、最小化ベクトルの値です。
