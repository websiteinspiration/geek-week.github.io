パート1. OAuth 2.0とOpenID Connectに関するストーリー


普遍的であり、21世紀の今日、誰もが好きなオープンアクセスの委任と認証プロトコルのバンドルはOAuth + OIDCと呼ばれています。大量使用に向いているため、まだ何も考え出されていません。 HTTP（S）プロトコルの上を歩き、JWT（JSON Web Token）コンテナーを使用するため、フロントエンドベンダーに特に人気があります。 OpenID Connectは、その作業にOAuthを使用します。つまり、OIDCはOAuthのラッパーです。

OpenID-認証とデジタル識別システムの作成のためのオープンスタンダードは、開発者にとって新しいものではありません。今年、彼は14歳になります。現在の3番目のバージョンでは、完全な名前はOpenID ConnectまたはOIDCより短いです。 Webとモバイルの両方の開発、および企業システムの両方で人気があります。

彼のパートナーであるOAuth Open Access Delegation Standardは12歳になりました。また、対応するRFC 5849標準が登場してから9年、最新バージョンのOAuth 2.0プロトコルと現在のRFC 6749に依存します。 OAuth 2.0は、以前のOAuth 1.0と互換性がないことに注意してください。

厳密に言えば、OAuthはプロトコルではなく、ソフトウェアシステムにアクセス制御のアーキテクチャを実装する際に、ユーザー識別操作を分離して別の信頼できるサーバーに転送するための一連のルール（スキーム）です。

そして、注意、OAuthは特定のユーザーについて何も言うことができません！彼が誰であるか、彼が今どこにいるのか、彼が今コンピューターで働いているのかどうかさえも。しかし、それにより、すでに発行されているアクセストークンを使用して、ユーザーの介入なしにシステムと対話することが可能になります。これは重要なポイントです。
それは役に立ちます：

  • OAuthはXACML属性アクセス記述言語の使用と完全に組み合わされます
  • OAuthは、ユーザー管理アクセス（UMA）にも依存しています

OAuthの使用例の完全な動作コード：
Class OAuthClient.REST Extends %CSP.REST
{

Parameter OAUTH2APPNAME = "OAuthClient";

Parameter OAUTH2CLIENTREDIRECTURI = "https://52773b-76230063.labs.learning.intersystems.com/oauthclient/";

// to keep sessionId
Parameter UseSession As Integer = 1;

XData UrlMap [ XMLNamespace = "http://www.intersystems.com/urlmap" ]
{
    <Routes>
        <Route Method="GET" Url = "/" Call = "Do" />
    </Routes>
}

ClassMethod Do() As %Status
{
    // Check for accessToken
    set isAuthorized = ##class(%SYS.OAuth2.AccessToken).IsAuthorized(
        ..#OAUTH2APPNAME,
        .sessionId,
        "scope1",
        .accessToken,
        .idtoken,
        .responseProperties,
        .error)
 
    // to show accessToken
    if isAuthorized {
        set valid = ##class(%SYS.OAuth2.Validation).ValidateJWT(
            ..#OAUTH2APPNAME,
            accessToken,
            "scope1",
            .aud,
            .JWTJsonObject,
            .securityParameters,
            .sc
        )
        &html< Hello!<br> >
            w "You access token = ", JWTJsonObject.%ToJSON()
        &html< </html> >
        quit $$$OK
    }

    // perform the process of user and client identification and get accessToken       
    set url = ##class(%SYS.OAuth2.Authorization).GetAuthorizationCodeEndpoint(
        ..#OAUTH2APPNAME,
        "scope1",
        ..#OAUTH2CLIENTREDIRECTURI,
        .properties,
        .isAuthorized,
        .sc)
 
    if $$$ISERR(sc) {
          w "error handling here"
          quit $$$OK
    }
    
    // url magic correction: change slashes in the query parameter to its code
    set urlBase = $PIECE(url, "?")
    set urlQuery = $PIECE(url, "?", 2)
    set urlQuery = $REPLACE(urlQuery, "/", "%2F")
    set url = urlBase _ "?" _ urlQuery
    
    &html<
        <html>
            <h1>  IRIS   OAuth2</h1>    
            <a href = "#(url)#">  <b>IRIS</b></a>
        </html>
    >

    quit $$$OK
}
}
