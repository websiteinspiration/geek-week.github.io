Acronis True Image 2020：新しいレプリケーションスキームと保護の強化

Acronis True Imageの新しいバージョンがリリースされました-個人ユーザー向けのデータ保護システム。また、True Image 2020がTrue Image 2019とどのように異なるかについてお話ししたいと思います。



Acronis True Imageは、個人ユーザー向けの個人データ保護ソリューションです。 True Imageには、バックアップおよび災害復旧システム、データ認証検証ツール、マルウェア対策ツールが含まれています。毎年8月末に新しいバージョンのTrue Imageを発表し、今年はソリューションのパフォーマンスを向上させる多数の追加機能が登場しました。一部のチップは、モバイルデバイスやMacコンピュータのユーザーに高く評価されます。

店頭に登場したAcronis True Image 2020がリリースされ、保護システムのさまざまな部分に関連する100を超える改善が行われました。それらの一部は小さな修正または最適化ですが、私たちの投稿では、製品のユーザーの生活をより良く改善できる最も重要で興味深い更新のみに焦点を当てます。

Acronisデュアルプロテクション


このリリースの最も興味深い機能の1つは、バックアップの複製です。レプリケーションとは、利用可能な最新バージョンのデータをバックアップからAcronisクラウドに転送することです。その結果、ユーザーには2つのバックアップがあります。1つはローカルメディア（外部ドライブなど）にあり、もう1つはクラウドにあります。



このような「二重」のバックアップ計画をセットアップした後、データはローカルバックアップ操作の完了後に自動的にクラウドにコピーされます。したがって、ユーザーは「高速な」ローカルバックアップを使用しているため、誤ってファイルを削除するなどの偶発的なデータ損失から、クラウドアーカイブからデータを回復できるため、クラッシュや機器の損失から保護されます。



Acronis Dual Protectionスキームには、ローカルストレージとクラウドへの2つの連続した従来のバックアップよりも多くの技術的な利点があります。まず、ディスク上のデータを分析する必要がないため、レプリケーションに費やされるシステムリソースが少なくて済みます。バックアップする必要があるものと、アーカイブに既にあるものを検討してください。次に、クラウドへのバックアップは通常ローカルバックアップよりも時間がかかるため、アプリケーションはファイルシステムのスナップショットを保持する必要があります。これはシステム自体の負荷であり、一部のスナップショットは操作が終了するまで存続しない場合があります。レプリカの場合、スナップショットは必要ありません。

アーカイブと重複排除の向上


重複排除機能を含め、アーカイブメカニズム自体が改善および改善されています。ユーザーは、バックアップの作成にかかる時間の短縮と、アーカイブ自体のサイズの縮小によって、これに気づくことができます。さらに、改良されたアーキテクチャにより、ローカルアーカイブとクラウドコピーの両方のコンテンツを、追加のリソースなしで、オンデマンドで個々のファイルをダウンロードするまで表示できます。

改善されたアーカイバの最近のテストでは、パフォーマンスが大幅に向上しました。そのため、ディスクにローカルバックアップの増分バージョンを作成する時間は45％削減され、クラウドバックアップの増分時間は、以前のバージョンのAcronis True Image 2019と比較して80％削減されました。

新しい通知および情報システム


全体的なグラフィックインターフェイスの全体的な改良に加えて、通知センターも追加しました。不要な移行を行わずに、バックアッププロセスに関する最新の情報や、マルウェア対策システムの動作に関するレポートを取得できます。



通知パネルには最新の30個のイベントが表示されるため、ユーザーはプロセスの正常な完了、または逆にバックアップとレプリケーション中の失敗とエラーを追跡できます。このため、「レポート」にアクセスする必要はありません。すべての問題とエラーを1か所で確認でき、必要に応じて修正できます。

高度なランサムウェア対策


ちなみに、新しい通知パネルには、ランサムウェアに対する保護システムであるActive Protectionの動作に関する情報があります。現在、通知には、ブロックされたプロセスとその理由（これは、ソフトウェアが誤ってブロックされた理由を開発者が理解するのに役立ちます）、および個々のプロセスを監視する理由に関する情報も表示されます。



Active Protectionシステム自体は、より「敏感」になっています。AIの継続的なトレーニングにより、誤検知の数が大幅に削減され、エンジンの効率も向上しました。個別の改善が行われ、暗号化機能が作業をブロックする試みに対するAcronisシステムプロセスの安定性が向上しました

ポータブルデバイスの最適化


ほとんどのユーザーは据え置き型だけでなくポータブルデバイスにもAcronis True Imageをインストールするため、ラップトップのデータ保護システムに多くの改良を加えました。

以前は、バッテリー電源で実行しているときにバックアップをオフにするだけだったため、ユーザーはラップトップのバッテリー寿命とデータの安全性のどちらかを選択する必要がありました。設定メニューには、リソースを大量に消費するコピーとアーカイブのプロセスにリソースを費やしたくない場合に、特定のバッテリーレベルに設定できる「スライダー」があり、最長のモバイル作業を優先します。



もう1つの側面は、クラウドにバックアップするためのネットワークの選択です。据え置き型PCのすべてが明確で、ダウンロードが家庭またはオフィスのインターネットチャネルを介して行われる場合、モバイルデバイスはさまざまなワイヤレスネットワークに接続します。新しいバージョンのAcronis True Image 2020では、バックアップデータを転送できるネットワークを構成できます。原則として、これにはホームネットワークとオフィスネットワークが使用されます。このアプローチにより、たとえばカフェや空港でネットワークに接続している場合に、「弱い」クラウドや有料Wi-Fiで明らかに運命を破るバックアップの試みを回避できます。チャネル保護の観点から見ると、暗号化はどのような場合でも情報をAcronisクラウドに送信するために使用されますが、一部のユーザーはまだオープンネットワークでのデータ転送を恐れていました。彼らにとって、使用されるWi-Fi接続のリストを制限する機能も必要でした。

MACの高度なセキュリティ
