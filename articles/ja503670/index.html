Mac OSは遅くなりますか？-Catalinaは起動時にインターネット経由で署名されていないコードをチェックします

カタリナに切り替えた後、多くのユーザーがブレーキについて不満を言い始めました。西側の同僚たちはその理由を突き止めた。MacOSは署名のないコードのハッシュをAppleサーバーに送る。

これにより、実行に重大な遅延が生じ、簡単に確認できます。

echo $'#!/bin/sh\necho Hello' > /tmp/test.sh && chmod a+x /tmp/test.sh
time /tmp/test.sh && time /tmp/test.sh

結果は驚くべきものです。

$ time /tmp/test.sh && time /tmp/test.sh
Hello

real    0m1.417s
user    0m0.001s
sys 0m0.003s
Hello

real    0m0.003s
user    0m0.001s
sys 0m0.002s

なぜそれが起こるのですか？Appleはかなり前から公証メカニズムを使用しています。これにより、ユーザーは有効なDeveloper-IDで署名されたプログラムに悪意のあるコンポーネントが含まれていないことを確認できます。つまり、アプリケーションが署名されているという事実に加えて、アップルはマルウェアの要素についても自動的にチェックします。
興味深いことに、将来的には、Appleは通常、公証されていないプログラムの立ち上げを禁止する予定です。見積もり：
重要

macOS 10.14.5以降、新しい開発者ID証明書で署名されたソフトウェアと、すべての新規または更新されたカーネル拡張機能を実行するには、公証を受ける必要があります。macOS 10.15以降、2019年6月1日以降にビルドされ、開発者IDで配布されるすべてのソフトウェアは公証を受ける必要があります。

明らかに、侵入するために、実行する新しい未署名のコードはAAPLサーバーに送信されます。これは、ネットワークアナライザーを実行し、api.apple-cloudkit.comドメインへの呼び出しを確認することで簡単に確認できます。送信されるのはコード自体ではなく、ハッシュ量であることに注意してください。これは2つの方法で確認できます。異なるサイズのスクリプトについて、ネットワーク経由で送信されるデータの量を比較します。データの送信を担当するデーモンの内容を確認する（otool -tV / usr / libexec / syspolicyd）。ただし、インターネット接続が遅い場合、遅延は数秒になる可能性があります。中国のユーザーが5.47秒の遅延について書いています。

一方では、Appleは理解できます。ただし、ユーザーのセキュリティに配慮していると同時に、プラットフォームでの開発プロセスをすべて地獄に変えています。

しかし、最初に問題に注意を向けた開発者を引用したいと思います。
オペレーティングシステムのこの動作は、低レベルのシステムAPIからの2つのメソッド（たとえば、execとgetxattr）が結果を返す前に同期ネットワーク呼び出しを実行するときに、設計に重大な問題があることを示しています。
