Bagaimana saya mentransfer dedic.ru ke halaman github dan merasa puas

Sekali waktu saya cukup dekat terlibat dalam organisasi berbagai jenis hosting. Dalam istilah teknis, tentu saja - menyiapkan server, mengkonfigurasi panel kontrol dan hal-hal terkait lainnya.

Kemudian saya mempunyai ide untuk menuliskan berbagai keputusan dan pemikiran cerdas di situs untuk mempopulerkan topik ini, karena pada saat itu tidak ada apa-apa tentang hosting hosting di RuNet. Dan dia menyebutnya dedic.ru.

Untuk mesin situs, untuk beberapa alasan saya memilih Drupal daripada Wordpress, nanti saya tidak bisa menjelaskan pilihan ini kepada diri saya sendiri. Kemudian saya merasa bahwa Drupal lebih fleksibel, nyaman, dan logis.
Kekecewaan pertama saya adalah pada saat rilis Drupal versi keenam. Saya benar-benar tidak dapat mentransfer proyek saya saat ini di sana - terus-menerus sesuatu jatuh, lalu taksonomi, lalu forum, lalu sesuatu yang lain.

Akibatnya, saya harus meninggalkan dedic.ru dalam mode hanya baca dan mencoba untuk melanjutkan apa yang saya mulai di situs lain. Tetapi pada saat itu, cukup banyak situs tentang topik ini telah muncul dan usaha saya tidak berjalan. Ya, dan saya sendiri mulai mengubah minat - awan, keamanan dan otomatisasi.

Dan kemudian suatu malam saya berpikir - dan bagaimana sebenarnya sebuah situs dapat dibuat mothball sedemikian rupa sehingga tidak membayar kontennya?

Solusinya ditemukan cukup cepat - letakkan situs di halaman github. Ini hanya layanan untuk menerbitkan situs statis, dan sepenuhnya gratis.

Untuk menggunakannya, cukup buat repositori publik dan ubah cabang dari master menjadi gh-pages. Setelah itu, konten repositori dapat diproses sebagai konten untuk server web dan menontonnya di browser.

Namun, muncul pertanyaan - bagaimana sebenarnya mentransfer konten ke repositori? Tangan? - Bodoh!

Solusinya ditemukan cukup cepat: Ada dua plugin bagus untuk Wordpress - FG Drupal ke WordPress dan WP2Static. Yang pertama memudahkan untuk mengimpor konten dari Drupal ke Wordpress melalui kueri ke database Drupal. Dan yang kedua adalah mempublikasikan situs statis dari Wordpress ke berbagai repositori, termasuk halaman github.

Tentu saja, impor tidak mentransfer taksonomi dan forum, tetapi karena saya tidak berpikir tentang pengembangan situs lebih lanjut, saya memutuskan untuk hanya skor beberapa konten untuk kerugian, dan saya redid struktur secara manual - karena tidak ada banyak entri di sana.

Setelah mengekspor situs ke halaman github, saya melampirkan domain asli dedic.ru di sana dan hampir puas. Dua hal yang tidak saya miliki adalah https dan semacam interaksi bagi pengunjung.

Untuk https, saya dengan mudah mengadaptasi CloudFlare, menetapkan sebagai CNAME alamat internal situs pada halaman github, mengaktifkan proxy dan menetapkan metode TLS / SSL sebagai fleksibel.

Untuk interaksi, saya tidak menghasilkan sesuatu yang lebih pintar daripada menempatkan donat standar dari paypal dan menambahkan kemampuan bagi donor untuk mengganti konten blok dengan milik mereka sendiri melalui permintaan tarik. Mungkin perlu menambahkan opsi komentar melalui layanan Disqus, tetapi jujur ​​- saya sudah terlalu malas untuk melakukan ini lebih lanjut.

Saya harap informasi ini bermanfaat bagi seseorang, dan konten saya pernah mendapat manfaat dari pengaturan server.
