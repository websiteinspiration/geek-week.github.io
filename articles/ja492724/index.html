だから、私は百万の整数のベクトルをチェックして、得ました：

念のため、これを数回実行しました。はい、この現象は本物です。私のメインコードは：

void run(vector<int>& v, const string& label)
{
    auto t0 = system_clock::now();
    sort(v.begin(), v.end());
    auto t1 = system_clock::now();
    cout << label 
         << duration_cast<microseconds>(t1 — t0).count() 
         << " milliseconds\n";
}

void tst()
{
    vector<int> v(1'000'000);
    iota(v.begin(), v.end(), 0);
    run(v, "already sorted ");
    std::shuffle(v.begin(), v.end(), std::mt19937{ std::random_device{}() });
    run(v, "shuffled    ");
}


少なくとも、この現象は、このコンパイラー、標準ライブラリー、およびオプティマイザーの設定で実際に発生しています。実装が異なれば、答えも異なる（そして与える）可能性があります。実際、誰かがより体系的な調査を行い（インターネットでのクイック検索はそれを見つけるのに役立ちます）、ほとんどの実装はこの効果を示しています。

理由の1つは分岐予測です。並べ替えアルゴリズムの主要な操作は、「if（v [i] <pivot]）...」または同等のものです。ソートされたシーケンスの場合、このテストは常に真ですが、ランダムシーケンスの場合、選択されたブランチはランダムに変化します。

別の理由は、ベクトルがすでにソートされている場合、要素を正しい位置に移動する必要がないためです。これらの細部の影響により、約5または6の係数が得られました。

クイックソート（および一般的なソート）は、コンピュータサイエンスの最高の心を集めた複雑な研究​​です。優れたソート機能は、優れたアルゴリズムを選択し、その実装中に機器のパフォーマンスに注意を払った結果です。

C ++の->演算子とは何ですか？




これは古いトリックの質問です。C ++では、演算子はありません->。以下を検討してください。

if (p-->m == 0) f(p);


もちろん、これはある種の演算子があるように見えます->そして適切な宣言pとmがあれば、これをコンパイルして実行することさえできます：

int p = 2;
int m = 0;
if (p-->m == 0) f(p);


これは実際には、p--がm（それがそうである方法）より大きいかどうかを確認し、結果（true）を0と比較します。まあ、true！= 0なので、結果はfalseであり、f（）は呼び出されません。言い換えると：

if ((p--) > m == 0) f(p);


そのような質問にはあまり時間をかけないでください。それらは、C ++が発明される前から、混乱する初心者に人気がありました。

＃4。C ++のポインター変数と参照変数の違いは何ですか？



リンクとポインタの詳細については、Learn C ++をご覧ください。

どちらもメモリ内ではマシンアドレスとして表されます。違いはそれらの使用法です。
ポインタを初期化するには、オブジェクトのアドレスを指定します。

int x = 7;
int* p1 = &x;
int* p2 = new int{9};


ポインターを介して読み書きするには、逆参照演算子（*）を使用します。

*p1 = 9;       // write through p1
int y = *p2;   // read through p2


あるポインタを別のポインタに割り当てると、どちらも同じオブジェクトを指します。

p1 = p2;       //  p1  p2    int   9
*p2 = 99;      //  99  p2
int z = *p1;   //   p1, z  99 (  9)


ポインターは、そのライフサイクル中に別のオブジェクトを指すことができます。これがリンクとの主な違いです。リンクは作成時にオブジェクトにアタッチされ、別のリンクに変換することはできません。

参照の場合、逆参照は暗黙的です。オブジェクトでリンクを初期化すると、リンクはそのアドレスを取得します。

int x = 7;
int& r1 = x;
int& r2 = *new int{9};


新しい演算子はポインターを返すので、割り当ての前に逆参照し、それを使用してリンクを初期化する必要がありました。

リンクを介して読み書きするには、リンク名を使用します（明示的な逆参照なし）。

r1 = 9;        //   r1
int y = r2;    //   r2


あるリンクを別のリンクに割り当てると、指定された値がコピーされます。

r1 = r2;       //  p1  p2     9
r1 = 99;       //  99  r1
int z = r2;    //   r2, z  9 (  99)


参照とポインタの両方が関数の引数としてよく使用されます。

void f(int* p)

{
    if (p == nullptr) return;
    // ...
}

void g(int& r)
{
    // ...
}

int x = 7;
f(&x);
g(x);


ポインターがある可能性があるnullptrため、何かを指しているかどうかを確認する必要があります。リンクについては、最初は何かを参照していると想定できます。

＃5。文字列の単語を反復する方法は？




を使用stringstreamしますが、「単語」をどのように定義しますか？「メアリーには小さな子羊がいた」と考えてください。最後の言葉は「ラム」または「ラム」ですか？句読点がない場合、これは簡単です。

vector<string> split(const string& s)
{
    stringstream ss(s);
    vector<string> words;
    for (string w; ss>>w; ) words.push_back(w);
    return words;
}

auto words = split("here is a simple example");   // five words
for (auto& w : words) cout << w << '\n';


または単に：

for (auto& w : split("here is a simple example")) cout << w << '\n';


デフォルトでは、>>演算子はスペースをスキップします。区切り文字の任意のセットが必要な場合、状況は少し混乱します。

template<typename Delim>
string get_word(istream& ss, Delim d)
{
    string word;
    for (char ch; ss.get(ch); )    //  
        if (!d(ch)) {
            word.push_back(ch);
            break;
        }
    for (char ch; ss.get(ch); )    //  
        if (!d(ch))
            word.push_back(ch);
        else
            break;
    return word;
}


dは、文字が区切り文字であるかどうかを通知する操作です。 ""（空の文字列）を返して、返す単語がないことを示します。

vector<string> split(const string& s, const string& delim)
{
    stringstream ss(s);
    auto del = [&](char ch) { for (auto x : delim) if (x == ch) return true; return false; };

    vector<string> words;
    for (string w; (w = get_word(ss, del))!= ""; ) words.push_back(w);
    return words;
}

auto words = split("Now! Here is something different; or is it? ", "!.,;? ");
for (auto& w : words) cout << w << '\n';
