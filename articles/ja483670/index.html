MACアドレスについて知りたいすべて

これらは6バイトであり、通常は16進形式で表示され、工場でネットワークカードに割り当てられていることを誰もが知っています。一見するとランダムです。一部の人々は、アドレスの最初の3バイトが製造元の識別子であり、残りの3バイトがそれらに割り当てられていることを知っています。自分で任意のアドレスを設定できることも知られています。多くの人がWi-Fiの「ランダムアドレス」について聞いたことがあります。


それが何であるかを理解しましょう。


MACアドレス（メディアアクセスコントロールアドレス）-ネットワークアダプターに割り当てられた一意の識別子で、主にイーサネット、Wi-Fi、BluetoothなどのIEEE 802標準ネットワークで使用されます。正式には「EUI-48型識別子」と呼ばれています。名前から、アドレスが48ビットの長さであることは明らかです。 6バイト。 （オクテットが常にドットで区切られているIPv4アドレスとは対照的に）普遍的に受け入れられているアドレスの標準はありません。通常、コロンで区切られた6つの16進数として書き込まれます：00：AB：CD：EF：11：22。 -AB-CD-EF-11-22、さらには00ab.cdef.1122。


歴史的に、アドレスはネットワークカードチップセットのROMにスティッチされ、フラッシュプログラマーなしでアドレスを変更することはできませんでしたが、現在のところ、アドレスはオペレーティングシステムからプログラムで変更できます。LinuxおよびMacOS（常に）、Windows（ドライバーが許可している場合は、ほとんどの場合）、Android（ルートのみ）では、ネットワークカードのMACアドレスを手動で設定できます。iOS（ルートなし）では、同様のトリックは不可能です。

private static final long VALID_LONG_MASK = (1L << 48) - 1;
private static final long LOCALLY_ASSIGNED_MASK = MacAddress.fromString("2:0:0:0:0:0").mAddr;
private static final long MULTICAST_MASK = MacAddress.fromString("1:0:0:0:0:0").mAddr;

public static @NonNull MacAddress createRandomUnicastAddress(MacAddress base, Random r) {
        long addr;
        if (base == null) {
            addr = r.nextLong() & VALID_LONG_MASK;
        } else {
            addr = (base.mAddr & OUI_MASK) | (NIC_MASK & r.nextLong());
        }
        addr |= LOCALLY_ASSIGNED_MASK;
        addr &= ~MULTICAST_MASK;
        MacAddress mac = new MacAddress(addr);
        if (mac.equals(DEFAULT_MAC_ADDRESS)) {
            return createRandomUnicastAddress(base, r);
        }
        return mac;
    }

誰かがあなたのデバイスのMACアドレスを見つけた場合、何が危険なのでしょうか？有線および無線ネットワークでは、サービス拒否攻撃を構成できます。また、無線機器の場合、センサーが設置されている場所の出現の瞬間を固定できる可能性もあります。アドレスをスプーフィングすることにより、デバイスのふりをしようとすることができます。これは、追加のセキュリティ機能（承認や暗号化）を使用しない場合にのみ機能します。ここの人々の99.9％は心配する必要がありません。


MACアドレスは、思ったより複雑ですが、それよりも単純です。
