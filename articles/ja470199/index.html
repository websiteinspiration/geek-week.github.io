この記事は、小学生、学生、およびこれまで試したことはないが、実際にコンピュータゲームを書き始めたいと思っている年配の人を対象としています。子供の頃、誰がコンピュータゲームをプレイせず、GTAやクライシスよりも優れた自分のゲームを書きたくなかったのですか。そして私は欲しかった、そして奇妙なことに、私はまだ望んでいる。同じプログラマーを教える数年間の練習で、どこに行くべきか、何をするべきかについていくつかのメモを得ました。

記事は完全に正確で正確であるとは主張しておらず、ほんの数年で経験を積んだ一人だけが書いています。また、それは実用的でロマンチックな側面のみを考慮します。収益化の問題や、愛する人を満足させること以外のゲームから利益を得ること、およびおそらく他のことについては取り上げません。

1.どこから始めますか？


練習から。あなたは純粋なCとOpenGL / DirectXから始めることができます、これはサムライの方法です、それは抽象化なしでそれをゼロから学ぶことはかなり難しいのでです。

SDLのようなフレームワーク、またはC / C ++があまりにも混乱しているようであれば、より高いレベルのLibGDX / PyGame /LÖVE2Dを使用できます。ここでは抽象化のレベルが少し高いので、作業は少し簡単ですが、ゲームの内部構造の組織全体を自分で作成し、ゲームサイクルとは何か、それにイベントシステムを編成する方法を理解する必要があります。このアプローチは、十分なレベルの知識とスキルを提供しますが、最初は、鉄の荒野の奥深くに登るインセンティブを提供しません。

そして最後に、Unity / Unreal / CryEngine / Godotのようなゲームエンジンを使って、おもちゃを作ることができます。
