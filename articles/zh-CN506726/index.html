今天我们转向实践部分。让我们从基于完整的openSSL加密开放源代码库设置证书颁发机构开始。该算法已在Windows 7上进行了测试

。安装了openSSL之后，我们可以通过命令行执行各种加密操作（例如，创建密钥和证书）。

动作算法如下：

  1. 下载安装分发包openssl-1.1.1g。
    OpenSSL具有各种版本。Rutoken的文档说，需要openSSL版本1.1.0或更高版本。我使用的是openssl-1.1.1g版本。您可以从官方站点下载openSSL，但是为了简化安装，您需要在网络上找到Windows的安装文件。我为您做到了：slproweb.com/products/Win32OpenSSL.html您
    需要向下滚动页面并下载Win64 OpenSSL v1.1.1g EXE 63MB安装程序。
  2. 在计算机上安装openssl-1.1.1g。
    必须根据标准路径进行安装，该路径在C：\ Program Files文件夹中自动指定。该程序将安装在OpenSSL-Win64文件夹中。
  3. 为了根据需要配置openSSL，有一个openssl.cfg文件。如果您按照上一段所述安装了openSSL，则此文件位于路径C：\ Program Files \ OpenSSL-Win64 \ bin中。转到存储openssl.cfg的文件夹，然后使用例如记事本++打开此文件。
  4. 您可能猜到了将通过更改openssl.cfg文件的内容以某种方式设置认证中心，您是完全正确的。为此，请配置[ca]命令。在openssl.cfg文件中，我们将在此处进行更改的文本开头可以找到：[ca]。
  5. 现在，我将给出一个配置示例及其说明：
