Angulareact

私は問題があります。アプリケーションはAngularで記述され、コンポーネントライブラリはReactで記述されています。ライブラリのクローン作成は高すぎます。したがって、最小限のコストでAngularアプリケーションでReactコンポーネントを使用する必要があります。それを行う方法を見つけます。


免責事項


私はAngularのスペシャリストではありません。私は2017年に最初のバージョンを試してから、AngularDartを少し見ましたが、現在、フレームワークの最新バージョンでのアプリケーションサポートに遭遇しました。決定がおかしい、または「別の世界から」であると思われる場合、それはあなたには見えません。


この記事で紹介するソリューションは、実際のプロジェクトではまだテストされておらず、概念にすぎません。自己責任で使用してください。


問題


現在、Angular 8でかなり大きなアプリケーションをサポートおよび開発しています。さらに、Reactには2、3の小さなアプリケーションがあり、さらに12を（Reactでも）ビルドする予定です。すべてのアプリケーションは会社の内部ニーズに使用され、同じスタイルである必要があります。唯一の論理的な方法は、アプリケーションをすばやく構築できるコンポーネントライブラリを作成することです。


しかし、現状では、Reactでライブラリを取得して作成することはできません。最大のアプリケーションでは使用できません-Angularで記述されています。この問題に遭遇したのは私が最初ではありません。「角度で反応する」ことをググる最初のことは、Microsoftリポジトリです。残念ながら、このソリューションにはドキュメントがありません。さらに、プロジェクトのreadmeでは、パッケージがMicrosoftによって内部的に使用されていることが明確に述べられており、チームには開発の明確な計画がなく、使用するのはユーザー自身の責任です。このようなあいまいなパッケージを本番環境にドラッグする準備ができていないので、自転車を作成することにしました。

import { createElement } from 'react';
import { render } from 'react-dom';
import { Hello } from './Hello';

@Component({
  selector: 'app-hello',
  template: `<div #react></div>`,
})
export class HelloComponent implements OnInit {
  @ViewChild('react', { read: ElementRef, static: true }) ref: ElementRef;

  @Input() name: string;
  @Output() click = new EventEmitter<string>();

  ngOnInit() {
    this.renderReactElement();
  }

  private renderReactElement() {
    const props = {
      name: this.name,
      onClick: () => this.lick.emit(`Hello, ${this.name}!`),
    };

    const reactElement = createElement(Hello, props);

    render(reactElement, this.ref.nativeElement);
  }
}
