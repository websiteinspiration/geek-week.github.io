Personalice su entorno Python con pyenv, virtualenvwrapper, tox y pip-compile





Estas herramientas simplificarán la configuración y automatizarán las operaciones de rutina. Ahorrarán al desarrollador de muchas dificultades que le impiden centrarse en la resolución de problemas y en la escritura cómoda del código.



Hay muchas formas de personalizar su entorno Python. Este artículo es sobre uno de ellos. Pero esto, por supuesto, no es la única solución.



Python es un lenguaje de programación de propósito general que a menudo se recomienda para principiantes. Durante los veinte años de su existencia, se han escrito varios libros sobre Python. Aunque a menudo se lo denomina simple, configurar herramientas de desarrollo de Python está lejos de ser una tarea fácil.









Configurar el entorno Python es bastante complicado: xkcd



Use pyenv para el control de versiones



En mi opinión, este es el mejor administrador de versiones de Python. Sin embargo, vale la pena señalar que pyenv se ejecuta en Linux, Mac OS X y WSL2: es decir, tres entornos "similares a UNIX".



Instalar pyenv en sí mismo a veces puede ser difícil. Pero un instalador especial de pyenv que use curl puede ayudar. bash para bootstrap.



Si está utilizando una Mac (u otro sistema en el que está ejecutando Homebrew), puede leer las instrucciones para instalar y usar pyenv aquí .



Después de instalar y configurar pyenv, puede usar pyenv global para instalar la versión predeterminada de Python. Puedes elegir tu versión "favorita". Por lo general, esta es la última versión estable, pero no es precisa -)



Use virtualenvwrapper para configurar un entorno virtual



Para Python, puede crear entornos virtuales, es decir, colocar cada proyecto en un entorno aislado. Esto se puede hacer usando virtualenvwrapper. Entonces es posible cambiar entre entornos virtuales en cualquier momento.



Lea más sobre cómo configurar un entorno virtual aquí .



Si necesita crear y usar repetidamente el mismo entorno virtual, tiene sentido automatizar el proceso. Hoy mi elección es tox.



Usa tox para automatizar



Tox es una gran herramienta para automatizar tus pruebas. En cada entorno de Python, creo un archivo tox.ini. No importa qué sistema use para la integración continua, todo funcionará. Y puedo ejecutarlo localmente usando virtualenvwrapper: la cuestión es que estoy probando mi código en varias versiones de Python y varias versiones de dependencias de la biblioteca. Esto significa que tox funcionará en múltiples entornos. Algunos de ellos tendrán dependencias reales. En algunos - congelados. También podría generarlos localmente usando pip-compile. Recientemente, he estado considerando el nox como un reemplazo para el tox. En este artículo no explicaré las razones, pero debe prestarle atención.

Use pip-compile para administrar dependencias



Cada vez que Python carga sus dependencias en la etapa de ejecución del código. Comprender y controlar qué versión de las dependencias se utiliza ayudará a evitar bloqueos inesperados mientras se ejecuta el programa. Esto significa que debemos pensar en las herramientas de gestión de dependencias.



Para cada nuevo proyecto, incluyo el archivo require.in, que (como regla) contiene solo un carácter:



.







Sí, aquí todo es correcto. Una sola cadena de puntos. En el archivo setup.pyfile para dependencias, escribo Twisted> = 17.5 en lugar de Twisted == 18.1, lo que dificulta la actualización a nuevas versiones de la biblioteca.



"." significa "directorio actual" en el que se utiliza el archivo setup.py correspondiente como fuente de información de dependencia.



Esto significa que cuando se utiliza pip-compile require.in> require.txt, se creará un archivo con dependencias congeladas. Puede usar este archivo de dependencia en un entorno virtual creado por virtualenvwrapper o en tox.ini.



A veces, el archivo require-dev.txt generado a partir de require-dev.in (contenido :. [Dev]) o require-test.txt generado a partir de require-test.in (contenido :. [Test]) puede ser útil.



Alternativamente, pip-compile puede usar dephell . La herramienta dephell tiene muchas cosas interesantes, por ejemplo, el uso de solicitudes HTTP asincrónicas para cargar dependencias.



Salida



Considero que Python es un lenguaje poderoso y hermoso. Sin embargo, para trabajar productivamente, utilizo un cierto conjunto de herramientas que han demostrado su efectividad, al menos para mí. Así que usa estas cuatro herramientas: pyenv, virtualenvwrapper, tox y pip-compile, y estarás feliz =)
