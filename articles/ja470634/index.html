Instagramでコミュニティを特定してユーザーの興味を特定する

ソーシャルネットワークのプロファイルは、個人についてどの程度説明できますか？写真、投稿、コメント、サブスクリプション-分析のための手付かずのフィールド。今日は、Instagramのサブスクリプションに基づいてユーザーの興味を判断する方法について説明します。

ソース

明白なアプローチは、個人のサブスクリプションを分類することです。たとえば、ユーザーIvanのサブスクリプションの中で、2つのオートブロガーと3つの面白い写真付きのアカウントを見つけましたが、残りのプロファイルはテーマ別ではありません（これらは友人、親戚、同僚などと想定しています）。これから私たちは結論付けます：イヴァンは陽気な人で、車が好きです。利益必要なのは「ブロガー+コンテンツテーマ」の一致だけですが、ここではそれほど単純ではありません。

ブロガーって誰？ Instagramのブロガーは通常のアカウントと同じです（そして一般的に、それは心の状態のようなものです）。理想的には、ライブオーディエンスの大きなシェアを持つテーマ別アカウントに関心がありますが、単にサブスクライバーの数を減らしたいのではなく、何かを見逃す可能性があります。

良い分類法はどこで入手できますか？自分でカテゴリを思いつくと、「何か違うもの」に興味をそそられるものが必ず見つかります。すべての準備が整うと、カテゴリが広すぎることがわかります。自然なパーティションを取得したいので、ブロガーをクラスター化します。

1人のブロガーが複数のカテゴリに属する​​ことはできますか？そしてどうやって！非常に多くの場合、彼は同時にスイス人、死神、そして男を演じる男です（旅行、ライフスタイル、妻と母親と同様に）。
したがって、非常に多数のブロガーを含むデータセットと、出力で交差するクラスターを含むクラスター化アルゴリズムが必要です。

すべての始まり

Instagramのオピニオンリーダーに最も効果的に広告を掲載したいと考えているお客様からのアプローチです。時にはブロガーのリストがあり、それから私たちは最適化問題を解決します-より少ないお金でより多くの人々にリーチするため、そして時にはリストがなく、関連するブロガーを見つける必要がある場合もあります。今回彼らは眼科医のプロフィールを検索しました。

私たちは推奨システムに似たアプローチを使用しました：

  1. 手で3〜7個の適切なプロファイルを選択してください（「コアコミュニティ」）

  2. サブスクライバーの共通部分を使用して、トピックに関心のある人々（「熱狂的」）を特定します

  3. 「熱狂的」な方のご契約で新しい医師を募集しています


  4. 必要に応じて、手順1〜3を繰り返します（「コア」に新しい医師を追加するか、変更します）

例について詳しく説明します。

ステップ1. Instagramの検索を使用して、ロシアのさまざまな地域から眼科医の6つのアカウントを選択しました。


リストはかなりランダムですが、ブログが本当にトピックに一致し、購読者が1000人以上であるということで十分でした。

ステップ2.すべてのサブスクライバーを通過した後、分布を調べます。

やったー！交差点があります。

さらに、6つのプロファイルすべてに登録している2人がいます！したがって、ここで、「熱狂的な」眼科を誰が考えるかを決定する必要があります。 3つのサブスクリプションから始めて、全員を対象とすることにしました。彼らが言うように、1回は事故、2回は偶然、3回は規則性です。したがって、131人（2 + 3 + 24 + 102）の「熱狂的な」ユーザーが見つかりました。

ステップ3.「熱狂的な」人々のサブスクリプションをダウンロードし、可能な限りあらゆる方法で再び集計します。


印象的なアカウントのリストを取得しました。ここから、新しい眼科医を選択します。右の列は、2番目のステップで選択された131人のうち、プロファイルに登録されている人数を示しています。一番上には、驚くべきことではない、私たちの最初の6つのプロファイルがありました。ここでしきい値を選択して「カウント値が10を超えるすべてのプロファイルは新しい医師です」と言う場合、Olga Buzovaのような人気のあるアカウントを眼科医に帰属させるリスクがあり、同時に、小さいが関連するプロファイルを失うため、さらに1つ追加しますサブスクライバーの数を列に入れ、サブスクライバー間の熱狂的なユーザーの割合を計算します。


便宜上、端数には100を掛けています（実際には、シェアはパーセンテージです）。その結果、条件カウント> = 6およびシェア> = 0.05で、166人の新しい眼科医が受診しました。クラス！

ステップ4. 172のアカウントがコミュニティの新しい「コア」になりました。これにより、約500人の新しい眼科医を見つけることができましたが、結果はすでに外国のアカウントにシフトしています。

Instagramのおすすめの類似アカウント

もちろん、これはすべて興味深いものであり、相互に類似しているアカウントを見つける方法はわかったが、すべてのインストーラーをカテゴリーに分類したい。それから、既成のInstagramの推奨事項を見るというアイデアが浮かびました。


実際のところ、テーマ別アカウントには推奨事項がありますが、一般の人には推奨されていません。ブロガーを検討し、頭痛の種を1つ差し引くという質問に対する答えは次のとおりです。やったー！

推奨事項では、1人のブロガーが最大80個の同様のプロファイルを持っています。 1000のロシアのブログのリストがあり、それから始めました。彼らは同様のものを取り、次に同様のものを取り、そして...まあ、あなたはそれを手に入れました:)

計画は、数百人のサブスクライバーを持つ小さなプロファイルが推奨事項に含まれないため、いつかこのプロセスが終了するという計画でしたが、実際には忍耐が終わったときに停止しました。いつでもデータセットを収集し続けることができますが、これまでのところ、ほとんどすべてのロシアのブロガーをカバーしている可能性が高いため、これは役に立ちませんでした。

これで、3,428,453の頂点（ブロガー）と96,967,974のエッジ（2つのブロガーの類似性）のグラフができました。

どうやら、Instagramの推奨事項は、このプロファイルのフォロワーがまだフォローしている人に基づいて形成されているため、Bryanskの医師は、人気のあるBryanskアカウントとその他の医師を推奨事項に含めるでしょう。これは眼科医の検索と似ていますが、ブロガーが所属するすべてのコミュニティを同時に考慮することができます。さらに、ブロガーのすべてのサブスクライバーを分析する必要がないため、処理速度が大幅に向上し、80の推奨アカウントを取得するだけで十分です。

アルゴリズムの選択と説明

いいね！データが準備されたので、このような大きなグラフに、できれば適切な時間内に対処できるツールとアルゴリズムを選択する必要があります。出力では、1人のブロガーが複数のコミュニティに所属できるコミュニティのセットを取得したいことを思い出させてください。

Stanford Network Analysis Platform（SNAP）ライブラリを選択しました
スタンフォードネットワーク分析プラットフォーム（SNAP）は、汎用のネットワーク分析およびグラフマイニングライブラリです。これはC ++で書かれており、数億のノードと数十億のエッジを持つ大規模なネットワークに簡単に拡張できます。大きなグラフを効率的に操作し、構造特性を計算し、通常のランダムグラフを生成し、ノードとエッジの属性をサポートします。
私たちの注意はAGMfitアルゴリズム（AGM-Affilated Graph Model）によって引き付けられ、最終的にはBIGCLAM（ビッグネットワークのクラスターアフィリエーションモデル）を使用しました。これらの違いは、最初のケースでは最適化問題が組み合わせで解決されるため、スケーラビリティが低くなり、2番目のケースでは、このようなグラフを入力に送信できるようになるだけです。

2つのノードに共通のコミュニティーがあるほど、これらの2つのノード間の通信の可能性が高くなります。どちらのアルゴリズムもAffilated Graph Modelに基づいているので、さらに詳しく

見ていきましょう。丸い頂点がコミュニティである2部グラフがあるとします（A そして B）、正方形-ソーシャルネットワークのユーザー、および各人（u）特定の重みを持つさまざまなコミュニティを指します（FuA そして FuB）重みが大きいほど、参加者がコミュニティの他のメンバーとつながりがある（親しんでいる）可能性が高くなります。


例：同じ大学で勉強した同僚がInstagramでお互いをフォローしている。どちらも同じ大学（コミュニティ1）の卒業生で、同じ会社（コミュニティ2）で働いているため、ネットワーク上の「友情」の原因と出会った時期はわかりません。1人は大学を卒業して就職し、2人目は5年前に卒業してずっと会社で働いていたとしたら、最初のコミュニティ1に所属するほうが重くなり、コミュニティ2に所属するほうが重くなり、逆もまた同様です。

数人の共通のコミュニティごとに、親しみを深める独立した機会を得ます。

1−exp(−Fuc⋅Fvc)


したがって、コミュニティが一般的であるほど、お互いを知る可能性が高くなります。

1−exp(−∑cFuc⋅Fvc)


ユーザーの各ペア間の接続の可能性を計算すると、出力では、頂点が人物であり、それらの間のエッジがお互いの知り合いまたはサブスクリプションであるというグラフが表示されます。

したがって、左側のビューから、右側の写真に移ります。


では、このモデルがコミュニティの発見にどのように使用されるかを理解しましょう。

同様のブロガーのグラフがあり、私たちの目標は発見することですK 上記のモデルを使用して、最も適切な重み行列を選択することにより、コミュニティ F （その次元 N 上のノード Kコミュニティ）出力で得られるグラフが元のグラフと同じになるようにします。これは、尤度関数を最大化することによって行われます。

l(F)=logP(G|F)


それでも、この全体は非負行列分解として表すことができます。これは、AGMfitと比較してBIGCLAMアルゴリズムのスケーラビリティにプラスを与えるだけです。

コミュニティの番号付けと起動

BIGCLAMでは、コミュニティの正確な数と間隔の両方を指定できます。アルゴリズムは、テストのためにノードのペアの20％を選択します。80％は、コミュニティの数が異なるモデルの適合です。

エッジのリストを含む入力ファイルを作成して実行します。


./bigclam -o:bloggers -i:query_graph_all.edgelist -c:-1 -nc:20 -mc:50000 -xc:200000 -nt:10

パラメータを使用すると、試行回数、コミュニティの最小数と最大数（または必要に応じて正確）、並列化するスレッド数、および出力ファイルのプレフィックスを指定できます。この例では、10スレッドで20回試行して、コミュニティを50から20万コミュニティに設定しました。このすべての素晴らしさは、2つのIntel®Xeon®Gold 6150 CPU @ 2.70GHzで5日間カウントされました。出力は2つのファイルを受け取りました。1つはGephi、2番目のテキストで、各行はコミュニティです。

実際、アルゴリズムを2回実行し、初めて上限の5万コミュニティに出くわしましたが、結果は悪くありませんでしたが、混合コミュニティに出くわすことがよくありました。今回もコミュニティの最大値は20万コミュニティに達しましたが、コミュニティははるかに優れていたので、そこで停止しました。結果をさらに良くすることはおそらく可能でしたが、20万人の名前のないコミュニティが私たちの想像力を脅かしました。

美しい絵

これでどうするの？

一方では、結果として生じるコミュニティは、（当時の眼科医と同様に）ブロガーの狭いグループをすばやく検索するのに役立ちます。例として2人のブロガーがいる場合、私たちは彼らが属しているすべてのコミュニティを取り上げ、不要なコミュニティを破棄します。それだけです。たとえば、装飾やインテリアデザインに関するコミュニティの1つなど、名前による検索を使用できます。


一方、ユーザーの興味を判断するには、これらのコミュニティすべてを何らかの意味で呼び出す必要があります。または、おそらくすべてが必要なわけではありません。

クランク

残念ながら、受け取ったすべてのコミュニティが同じように良いわけではありません。最も興味深いコミュニティを見つけるために、同じSNAPライブラリのCRank優先順位付け方法を使用することを決定しました。グラフの構造に基づいて自動的に呼び出され、各コミュニティランクに一致します。値が大きいほど、コミュニティは優れています。

プログラムの入力時に、以前に見つけたコミュニティを送信します。コミュニティごとに、優先順位メトリック（4個）が計算され、これらのメトリックが集計されて、見積もりが取得されます。


詳細はこちら。

発売

./crank -i:bloggers_cmtyvv.txt -c:query_graph_all.edgelist -o:bloggers_prioritization.txt

ローンチ中に、Ubuntu 18.04との互換性の問題が発生しました。ここでソリューションを提供しています。

私たちのコミュニティのblogger_cmtyvv.txtファイルとquery_graph_all.edgelistエッジのリストには、最初はニックネーム（文字列型）が含まれていたため、ハッシュ化してintにし、それ以外の場合は悪態をつく必要がありました。

例でそれがどのように機能するか見てみましょう。

名前に「lokomotiv」という文字列が含まれているプロファイルがあるいくつかのコミュニティを選択します。これは、Lokomotivクラブのフットボールファンのコミュニティである必要があります。コミュニティのメンバーは赤でマークされ、各写真の下に対応するスコアがあります。

コミュニティ1、スコア0.4
コミュニティ2、スコア0.41
コミュニティ3、スコア0.34
コミュニティ4、スコア0.13

次に、このサブグラフのすべてのプロファイルを確認した後、目的のコミュニティに実際に属している人を選択すると、次のようになります。

最初の2つのコミュニティには実際に必要なものが含まれており、事実上エラーはありません（ロシアのフットボールチームロシアのプロファイルを除く）。3番目のコミュニティは適切ですが、カバーするアカウントは最初の2つより少ないため、スコアは小さく、4番目のコミュニティはコミュニティ、それはフットボールのトピックに関するアカウントで構成されていますが（ここでは他のクラブの選手、妻、マネージャーです）、しかし、それでは私たちには合いません。
CRankは非常にうまく機能していると結論付けることができます。

着色と結果のための手動マーキング。

ユーザーを興味ごとに色分けする問題を解決するために、投稿のテキストを分析し、結果がより良い/より興味深い（テキストまたはサブスクリプション）か、一致するかどうかを確認したいと考えました。少数の人々のサンプルでは、​​約800のコミュニティを個別にラベル付けする必要がありました。結果に満足しただけでなく、このアプローチを開発する必要があると判断しました。コミュニティ検出方法を使用すると、非常に狭く珍しい興味カテゴリを見つけることができます。たとえば、（小さな灰色のげっ歯類の意味での）ラットのプロフィールの購読やその他の驚くべきこと、人が興味を持っていると思われるスポーツの種類を見つけ、すべてを「極端な」カテゴリに押し込むのではない等

20万のコミュニティすべてにマークを付けることはおそらくありませんが、CRankの結果をいじって、コミュニティだけを高速に残し、必要に応じてマークを付けることもできます。そんなこと:)
