データの整列


コンピュータのRAM内のデータの配置は、より高速なアクセスのためのメモリ内のデータの特別な配置です。メモリを使用する場合、プロセスはマシンワードをメインユニットとして使用します。プロセッサーのタイプが異なれば、サイズも異なります（1、2、4、8など）。バイト。オブジェクトをメモリに保存するときに、一部のフィールドがこれらの単語の境界を超える場合があります。一部のプロセッサは、境界整列されていないデータを、境界整列されたデータよりも長時間処理できます。また、非コアプロセッサは通常、非境界整列データを処理できません。

アラインされたデータとアラインされていないデータのモデルをよりよく想像するために、次のオブジェクト、つまりデータ構造の例を考えてみましょう。

struct Data{
	int var1;
	void* mas[4];
};

x32プロセッサとx64プロセッサのint型の変数のサイズは4バイトではなく、void *型の変数の値はそれぞれ4バイトと8バイトであるため、x32プロセッサとx64プロセッサのこの構造は次のようにメモリで表されます。



このような構造のX64プロセッサは、データが整列されていないため機能しません。データアライメントのために、構造に別の4バイトフィールドを追加する必要があります。

struct Data{
	int var1;
	int addition;
	void* mas[4];
};

したがって、x64プロセッサのデータ構造データはメモリ内で整列されます。

char* slow_memcpy(char* dest, const char* src, size_t len){
	int i;
	for (i=0; i<len; i++) {
		dest[i] = src[i];
	}
	return dest;
}

char* fast_memcpy(char* dest, const char* src, size_t len){
	size_t i;
	// 64-byte block fast copy
	if(len >= 64){
		i = len / 64;
		len &= (64-1);
		while(i-- > 0){
			__asm__ __volatile__ (
			"movdqa (%0), %%xmm0\n"
			"movdqa 16(%0), %%xmm1\n"
			"movdqa 32(%0), %%xmm2\n"
			"movdqa 48(%0), %%xmm3\n"
			"movntps %%xmm0, (%1)\n"
			"movntps %%xmm1, 16(%1)\n"
			"movntps %%xmm2, 32(%1)\n"
			"movntps %%xmm3, 48(%1)\n"
			::"r"(src),"r"(dest):"memory");
			dest += 64;
			src += 64;
		}
	}

	// byte-to-byte slow copy
	if(len) slow_memcpy(dest, src, len);
	return dest;
}
