カットの下には、抜粋した「データクラス」があります。

データを操作する


時間を無駄にして、すでに行われたことをやり直したくはありません。ほとんどのアプリケーションは、データを格納するためにクラスを使用します。作業を簡略化するために、Kotlinの作成者はデータクラスの概念を提案しました。この章では、データクラスを使用して、以前は夢にも思わなかったよりエレガントで簡潔なコードを作成する方法を学びます。データクラスのヘルパー関数を見て、データオブジェクトをコンポーネントに分解する方法を学びます。同時に、デフォルトのパラメーター値によってコードがどのように柔軟になるかを説明し、すべてのスーパークラスの祖先であるAnyも紹介します。

==演算子は、equalsという関数を呼び出します


ご存知のように、==演算子を使用して、等しいかどうかを確認できます。==ステートメントが実行されるたびに、equalsという関数が呼び出されます。各オブジェクトにはequals関数が含まれており、この関数の実装によって==演算子の動作が決まります。

デフォルトでは、等価性をチェックするためのequals関数は、2つの変数が同じオブジェクトを参照しているかどうかをチェックします。

それがどのように機能するかを理解するために、w1とw2という名前の2つのWolf変数を想像してください。w1とw2に1つのWolfオブジェクトへの参照が含まれている場合、それらを==演算子で比較すると、結果はtrueになります。


ただし、w1とw2に異なるWolfオブジェクトへの参照が含まれている場合、それらを==演算子と比較すると、オブジェクトに同じプロパティ値が含まれていても、結果はfalseになります。


前述のように、作成したすべてのオブジェクトにequals関数が自動的に含まれます。しかし、この関数はどこから来るのでしょうか？

等しいスーパークラスAnyから継承


クラスはAnyという名前のクラスから関数を継承するため、各オブジェクトにはequalsという関数が含まれています。Anyクラスは、すべてのクラスの祖先、つまりすべての結果のスーパークラスです。定義する各クラスはAnyのサブクラスであり、プログラムでこれを指摘する必要はありません。したがって、次のようなmyClassというクラスコードを記述したとします。

class MyClass {
    ...
}

コンパイラーはそれを自動的に次の形式に変換します。

各クラスはAnyのサブクラスであり、その動作を継承します。各クラスはAnyのサブクラスであり、プログラムでこれを報告する必要はありません。

すべてからの継承の重要性


結果のスーパークラスとしてAnyを含めることには、2つの重要な利点があります。

  • これにより、すべてのクラスが共通の動作を確実に継承します。Anyクラスは、システムの動作が依存する重要な動作を定義します。また、各クラスはAnyのサブクラスであるため、この動作は作成したすべてのオブジェクトに継承されます。したがって、Anyクラスはequalsという関数を定義しているため、この関数はすべてのオブジェクトによって自動的に継承されます。
  • つまり、ポリモーフィズムは任意のオブジェクトで使用できます。各クラスはAnyのサブクラスであるため、作成するすべてのオブジェクトには、最終的なスーパータイプとしてAnyクラスがあります。これは、任意の型のオブジェクトで機能するAnyパラメータまたはAny戻り値型の関数を作成できることを意味します。これは、次の形式のコードを使用して、任意のタイプのオブジェクトを格納するためのポリモーフィック配列を作成できることも意味します。

val myArray = arrayOf(Car(), Guitar(), Giraffe())

コンパイラーは、配列内の各オブジェクトがAnyの共通プロトタイプを持っていることを認識し、そのためArray型の配列を作成します。

Anyクラスによって継承される一般的な動作は、よく見る価値があります。

Anyから継承された共通の動作


Anyクラスは、各クラスによって継承されるいくつかの関数を定義します。基本的な機能とその動作の例を次に示します。

  • equals（any：Any）：ブール値
    2つのオブジェクトが「等しい」と見なされるかどうかをチェックします。デフォルトでは、関数は1つのオブジェクトのチェックに使用されている場合はtrueを返し、異なるオブジェクトの場合はfalseを返します。舞台裏では、==演算子がプログラムで使用されるたびに、equals関数が呼び出されます。

val w1 = Wolf()                                 val w1 = Wolf()
val w2 = Wolf()                                 val w2 = w1
println(w1.equals(w2))                          println(w1.equals(w2))

false (equals  false,                 true (equals  true,
  w1  w2                                w1  w2
                                    
  .)                                  —   ,
                                                  w1 == w2.

  • hashCode（）：Int
    オブジェクトのハッシュコードを返します。ハッシュコードは、値を効率的に格納および取得するために、一部のデータ構造でよく使用されます。

val w = Wolf()
println(w.hashCode())

523429237（ハッシュコードwの値）

  • toString（）：String
    オブジェクトを表すStringメッセージを返します。デフォルトでは、メッセージにはクラスの名前と、通常は気にしない番号が含まれています。

val w = Wolf()
println(w.toString())

ウルフ@ 1f32e575

デフォルトでは、equals関数は、2つのオブジェクトが同じ実際のオブジェクトであるかどうかをチェックします。

equals関数は、==演算子の動作を決定します。

Anyクラスは、リストされたすべての関数にデフォルトの実装を提供し、これらの実装はすべてのクラスに継承されます。ただし、これらの実装をオーバーライドして、リストされているすべての関数のデフォルトの動作を変更できます。

2つのオブジェクトの単純な等価性チェック


==演算子の動作を変更するには、equals関数の実装を変更する必要がある場合があります。

レシピを保存するためのオブジェクトを作成できるRecipeクラスがあるとします。この状況では、2つのRecipeオブジェクトに同じレシピの説明が含まれている場合、それらは等しい（または同等の）オブジェクトと見なされる可能性があります。Recipeクラスが、titleとisVegetarianという2つのプロパティで定義されているとします。

class Recipe(val title: String, val isVegetarian: Boolean) {
}

==演算子は、同じプロパティ、titleとisVegetarianを持つ2つのRecipeオブジェクトを比較するために使用される場合、trueを返します。

val r1 = Recipe("Chicken Bhuna", false)
val r2 = Recipe("Chicken Bhuna", false)


==演算子の動作は、equals関数をオーバーライドする追加のコードを記述することで変更できますが、Kotlin開発者はより便利なソリューションを提供しました。データクラスの概念を作成したことです。これらのクラスとは何か、どのように作成されるかを見てみましょう。

データクラスを使用すると、データオブジェクトを作成できます。


データクラスは、データを格納するためのオブジェクトを作成するためのクラスです。これには、データを操作するときに役立つツールが含まれます。たとえば、2つのデータオブジェクトに同じプロパティ値が含まれているかどうかをチェックするequals関数の新しい実装などがあります。2つのオブジェクトに同じデータが含まれている場合、それらは等しいと見なすことができます。

データクラスを定義するには、通常のデータ定義の前にdataキーワードを付けます。次のコードは、前に作成したRecipeクラスをデータクラスに変換します。

data class Recipe(val title: String, val isVegetarian: Boolean) {
}

データ接頭辞は、通常のクラスをデータクラスに変換します。

データクラスに基づいてオブジェクトを作成する方法


データクラスオブジェクトは、通常のクラスのオブジェクトと同じ方法で作成されます。つまり、このクラスのコンストラクターを呼び出します。たとえば、次のコードは新しいレシピデータオブジェクトを作成し、それをr1という名前の新しい変数に割り当てます。

val r1 = Recipe("Chicken Bhuna", false)

データクラスは自動的にそれらのequals関数をオーバーライドして==演算子の動作を変更し、オブジェクトの等価性が各オブジェクトのプロパティ値に基づいてチェックされるようにします。たとえば、同じプロパティ値で2つのRecipeオブジェクトを作成した場合、2つのオブジェクトを==演算子で比較すると、同じデータが格納されるため、結果はtrueになります。

val r1 = Recipe("Chicken Bhuna", false)
val r2 = Recipe("Chicken Bhuna", false)
//r1 == r2  true

2つのRecipeオブジェクトに同じデータが含まれているため、r1とr2は「等しい」と見なされます。

Anyスーパークラスから継承されたequals関数の新しい実装に加えて、データクラス
はhashCode関数とtoString関数もオーバーライドします。これらの関数がどのように実装されているか見てみましょう。

クラスオブジェクトは継承された動作を再定義します


データを処理するために、データクラスはオブジェクトを必要とするため、Anyスーパークラスから継承されたequals、hashCode、およびtoString関数の次の実装を自動的に提供します。

equals関数はプロパティ値を比較します


データクラスを定義するときに、リンクが同じオブジェクトを指している場合、その等号関数（したがって==演算子）は引き続きtrueを返します。ただし、オブジェクトがコンストラクターで定義された同じプロパティ値を持っている場合もtrueを返します。

val r1 = Recipe("Chicken Bhuna", false)
val r2 = Recipe("Chicken Bhuna", false)
println(r1.equals(r2))
       true

プロパティに同じ値が含まれている場合、データオブジェクトは等しいと見なされます。

等しいオブジェクトの場合、同じhashCode値が返されます


2つのデータオブジェクトが等しいと見なされた場合（つまり、2つのデータオブジェクトが同じプロパティ値を持っている場合）、hashCode関数はこれらのオブジェクトに対して同じ値を返します。

val r1 = Recipe("Chicken Bhuna", false)
val r2 = Recipe("Chicken Bhuna", false)
println(r1.hashCode())
println(r2.hashCode())

241131113
241131113

toStringはすべてのプロパティの値を返します


最後に、toString関数はクラスの名前とそれに続く番号を返さなくなりましたが、データクラスのコンストラクターで定義されたすべてのプロパティの値を含む有用な文字列を返します。

val r1 = Recipe("Chicken Bhuna", false)
println(r1.toString())
       Recipe(title=Chicken Bhuna, isVegetarian=false)

Anyスーパークラスから継承された関数をオーバーライドすることに加えて、データクラスは、データオブジェクトをコピーする機能など、データをより効率的に処理する追加のツールも提供します。これらのツールがどのように機能するか見てみましょう。

コピーを使用したデータオブジェクトのコピー


一部のプロパティを変更してデータオブジェクトのコピーを作成する必要があるが、他のプロパティは元の状態のままにする場合は、copy関数を使用します。このために、コピーされるオブジェクトに対して関数が呼び出され、新しい値を持つすべての変更可能なプロパティの名前がそれに渡されます。

次のようなコードで定義されているr1という名前のRecipeオブジェクトがあるとします。

val r1 = Recipe("Thai Curry", false)


isVegetarianプロパティの値をtrueに置き換えて、Recipeオブジェクトのコピーを作成する場合、これは次のように行われます。


本質的に、これは「r1のコピーを作成し、そのisVegetarianプロパティの値をtrueに変更し、新しいオブジェクトをr2という名前の変数に割り当てる」ことを意味します。これにより、オブジェクトの新しいコピーが作成され、元のオブジェクトは変更されません。

コピー関数に加えて、データクラスは、データオブジェクトをそのプロパティの値のセットに分割するための関数のセットも提供します。このプロセスは、構造化解除と呼ばれます。これがどのように行われるか見てみましょう。

データクラスは関数componentNを定義します...


データクラスを定義すると、コンパイラは、オブジェクトのプロパティ値にアクセスするための代替メカニズムとして使用できる関数のセットをクラスに自動的に追加します。これらの関数は、componentN関数の総称として知られています。Nは、取得したプロパティの数（宣言順）です。

componentN関数の動作を確認するために、次のRecipeオブジェクトがあるとします。

val r = Recipe("Chicken Bhuna", false)

オブジェクトの最初のプロパティ（titleプロパティ）の値を取得する場合は、オブジェクトのcomponent1（）関数を次のように呼び出すことができます。

val title = r.component1()

component1（）は、データクラスのコンストラクタで定義された最初のプロパティに含まれる参照を返します。

この関数は次のコードと同じ働きをします：

val title = r.title

関数を含むコードはより普遍的です。ComponentN関数がデータクラスで非常に役立つのはなぜですか？

...データオブジェクトを再構築するように設計されています


ジェネリックなcomponentN関数は、データオブジェクトをプロパティ値に分割したり、破棄したりするためのシンプルで便利な方法を提供するので、主に役立ちます。

Recipeオブジェクトのプロパティの値を取得し、その各プロパティの値を個別の変数に割り当てたいとします。コードの代わりに

val title = r.title
val vegetarian = r.isVegetarian

各プロパティの順次処理では、次のコードを使用できます。

val (title, vegetarian) = r

タイトルを最初のプロパティrに割り当て、ベジタリアンを2番目のプロパティに割り当てます。

このコードは、「titleとvegetarianの2つの変数を作成し、各変数のrプロパティの1つの値を割り当てる」ことを意味します。彼は次の断片と同じことをします

val title = r.component1()
val vegetarian = r.component2()

しかし、それはよりコンパクトになります。

===演算子は、2つの変数が同じオブジェクトを参照しているかどうかを常にチェックします。

2つの変数が型に関係なく同じオブジェクトを参照しているかどうかを確認する場合は、==の代わりに===演算子を使用します。===演算子は、2つの変数に1つの実際のオブジェクトへの参照が含まれている場合にのみ、結果をtrueにします。2つの変数xとyがあり、次の式があるとします。

x === y

結果がtrueになると、変数xとyが同じオブジェクトを参照する必要があることがわかります。

==演算子とは異なり、===演算子の動作は、equals関数から独立しています。===演算子は、クラスのタイプに関係なく常に同じように動作します。

レシピプロジェクトの作成


JVMの新しいKotlinプロジェクトを作成し、「Recipes」という名前を付けます。次に

Recipeというプロジェクトに新しいデータクラスを追加し、Recipeデータオブジェクトを作成します。以下はコードです。Recipes.ktのバージョンを更新して、それを私たちのバージョンに合わせてください：

data class Recipe(val title: String, val isVegetarian: Boolean) (  {} ,        .)

fun main(args: Array<String>) {
      val r1 = Recipe("Thai Curry", false)
      val r2 = Recipe("Thai Curry", false)
      val r3 = r1.copy(title = "Chicken Bhuna") (  r1    title)
      println("r1 hash code: ${r1.hashCode()}")
      println("r2 hash code: ${r2.hashCode()}")
      println("r3 hash code: ${r3.hashCode()}")

}
