WireGuardとRaspberry Piをサーバーとして使用してシンプルなVPNをセットアップする

WireGuard は将来のLinux 5.6カーネルの一部となるため、このVPNをRaspberry Pi上のLTEルーター/アクセスポイントと統合する方法を検討することにしました。

装置


  • LTEモジュールとパブリックIPアドレスを備えたRaspberry Pi 3。VPNサーバー（以降、edgewalkerと呼びます）があります。
  • すべての通信にVPNを使用する必要があるAndroidフォン
  • ネットワーク内でのみVPNを使用する必要があるLinuxラップトップ

VPNに接続する各デバイスは、他のすべてのデバイスに接続できる必要があります。たとえば、両方のデバイスがVPNの一部である場合、電話はラップトップ上のWebサーバーに接続できる必要があります。設定が非常に簡単な場合は、VPNとデスクトップに（イーサネット経由で）接続することを検討できます。

（時間にわたり確保有線および無線接続が少なくなってきていることを考えると標的型攻撃、WPA2にKRACK攻撃ハッキングとWPA3に対するDragonblood攻撃）、私は真剣にかかわらず、彼らは仕事で、環境の、私のすべてのデバイスに対してWireGuardを使用しての可能性を検討してください。

ソフトウェアのインストール


WireGuardは、ほとんどのLinux、Windows、およびmacOSディストリビューション用に事前にコンパイルされたパッケージを提供します。AndroidおよびiOSアプリケーションは、アプリケーションディレクトリを通じて配信されます。

私は最新のFedora Linux 31を使用しています。インストールする前に、マニュアルを読むのが面倒でした。パッケージを見つけてwireguard-toolsインストールしたところ、何も機能しない理由がわかりませんでした。さらなる調査の結果、パッケージwireguard-dkms（ネットワークドライバー付き）がインストールされておらず、パッケージが私のディストリビューションのリポジトリにないことがわかりました。

手順を読んだら、正しい手順を踏みます。

$ sudo dnf copr enable jdoss/wireguard
$ sudo dnf install wireguard-dkms wireguard-tools

Raspbian PiにRaspbian Buster配布キットをインストールしました。すでにそこwireguardにパッケージがあります。インストールしてください。

$ sudo apt install wireguard

Androidフォンに、公式のGoogle App StoreディレクトリからWireGuard VPNアプリケーションをインストールしました。

キーのインストール


ホスト認証の場合、WireguardはVPNホストの認証に単純な秘密/公開鍵スキームを使用します。次のコマンドでVPNキーを簡単に作成できます。

$ wg genkey | tee wg-laptop-private.key |  wg pubkey > wg-laptop-public.key
$ wg genkey | tee wg-server-private.key |  wg pubkey > wg-server-public.key
$ wg genkey | tee wg-mobile-private.key |  wg pubkey > wg-mobile-public.key

これにより、3つの鍵ペア（6つのファイル）が提供されます。設定内のファイルは参照しませんが、ここに内容をコピーします。各キーはbase64の1行です。

VPNサーバー（Raspberry Pi）の構成ファイルの作成


設定はかなり簡単です、私は次のファイルを作成しました/etc/wireguard/wg0.conf：

[Interface]
Address = 10.200.200.1/24
ListenPort = 51820
PrivateKey = <copy private key from wg-server-private.key>
PostUp   = iptables -A FORWARD -i %i -j ACCEPT; iptables -t nat -A POSTROUTING -o wwan0 -j MASQUERADE
PostDown = iptables -D FORWARD -i %i -j ACCEPT; iptables -t nat -D POSTROUTING -o wwan0 -j MASQUERADE

[Peer]
# laptop
PublicKey = <copy public key from wg-laptop-public.key>
AllowedIPs = 10.200.200.2/32

[Peer]
# mobile phone
PublicKey = <copy public key from wg-mobile-public.key>
AllowedIPs = 10.200.200.3/32

いくつかのコメント：

  • キーを使用してファイルから行を挿入する必要がある適切な場所
  • 私のVPNは内部範囲を使用しています 10.200.200.0/24
  • コマンドの場合PostUp/ PostDown外部ネットワークインターフェイスwwan0がありますが、別のインターフェイス（eth0など）がある場合があります。

VPNネットワークは、次のコマンドで簡単に解除できます。

$ sudo wg-quick up wg0

1つの小さな詳細：dnsmasqネットワークインターフェイスへのバインドで使用したDNSサーバーとして、許可されたデバイスのリストにbr0デバイスも追加しましたwg0。dnsmasqでは、これは、ネットワークインターフェースを含む新しい行を構成ファイル/etc/dnsmasq.confに追加することによって行われます。次に例を示します。

interface=br0
interface=wg0

さらに、リスニングUDPポート（51280）へのトラフィックを許可するiptableルールを追加しました。

$ sudo iptables -I INPUT -p udp --dport 51820 -j ACCEPT

すべてが機能しているので、VPNトンネルを自動的に開始するように設定できます。

$ sudo systemctl enable wg-quick@wg0.service

ラップトップのクライアント構成


ラップトップ/etc/wireguard/wg0.confで、同じ設定で構成ファイルを作成します。

[Interface]
Address = 10.200.200.2/24
PrivateKey = <copy private key from wg-laptop-private.key>

[Peer]
PublicKey = <copy public key from wg-server-public.key>
AllowedIPs = 10.200.200.0/24
Endpoint = edgewalker:51820

ノート：

  • Edgewalkerの代わりに、VPNサーバーのパブリックIPまたはホストを指定する必要があります
  • に設定AllowedIPsする10.200.200.0/24と、VPNは内部ネットワークへのアクセスにのみ使用されます。他のすべてのIPアドレス/サーバーへのトラフィックは、引き続き「通常の」オープンチャネルを通過します。ラップトップの事前構成済みDNSサーバーも使用されます。

テストと自動起動には、同じコマンドwg-quickとsystemd次のコマンドを使用します。

$ sudo wg-quick up wg0
$ sudo systemctl enable wg-quick@wg0.service

Androidフォンでのクライアントのセットアップ


Androidフォンの場合は、非常によく似た構成ファイルを作成します（これをとしましょうmobile.conf）。

[Interface]
Address = 10.200.200.3/24
PrivateKey = <copy private key from wg-mobile-private.key>
DNS = 10.200.200.1
        
[Peer]
PublicKey = <copy public key from wg-server-public.key>
AllowedIPs = 0.0.0.0/0
Endpoint = edgewalker:51820

ラップトップの設定とは異なり、電話機はVPNサーバーをDNSサーバー（文字列DNS）として使用し、すべてのトラフィックをVPNトンネルを通過させる必要があります（AllowedIPs = 0.0.0.0/0）。

ファイルをモバイルデバイスにコピーする代わりに、QRコードに変換できます。

$ sudo apt install qrencode
$ qrencode -t ansiutf8 < mobile.conf

QRコードはASCIIとしてコンソールに出力されます。Android VPNアプリからスキャンして、VPNトンネルを自動的に構成できます。

結論


WireGuardの設定は、OpenVPNに比べて魔法のようなものです。
