constexprラムダ式

C ++ 17以降、標準ではoperator()ラムダ型をconstexpr可能な限り暗黙的に定義しています。
expr.prim.lambdaから＃4：
対応するラムダ式の条件パラメーターの宣言の後にconstexprが続く場合、または関数呼び出し演算子がconstexpr関数の要件を満たす場合、関数呼び出し演算子はconstexpr関数です。

例えば：

constexpr auto Square = [] (int n) { return n*n; }; // implicitly constexpr
static_assert(Square(2) == 4);

C ++ 17では、constexpr関数は次の規則に従う必要があることを思い出してください。

  • 仮想であってはなりません。

    • その戻り型はリテラル型でなければなりません。
    • パラメータの各タイプはリテラルタイプである必要があります。
    • その本体は=削除、=デフォルト、または含まない複合ステートメントでなければなりません
      • asmの定義
      • goto式、
      • タグ
      • ブロックを試すか
      • 初期化が実行されない非リテラル変数、静的変数、またはストリーミングメモリ変数の定義。

より実用的な例はどうですか？

template<typename Range, typename Func, typename T>
constexpr T SimpleAccumulate(const Range& range, Func func, T init) {
    for (auto &&elem: range) {
        init += func(elem);
    }
    return init;
}

int main() {
    constexpr std::array arr{ 1, 2, 3 };

    static_assert(SimpleAccumulate(arr, [](int i) { 
            return i * i; 
        }, 0) == 14);
}

ラムダテンプレート

C ++ 14では、一般化されたラムダを得ました。つまり、autoとして宣言されたパラメーターはテンプレートパラメーターです。

ラムダの場合：

[](auto x) { x; }

コンパイラは、次のボイラープレートメソッドに一致するcallステートメントを生成します。

template<typename T>
void operator(T x) { x; }

ただし、このテンプレートパラメータを変更して実際のテンプレート引数を使用する方法はありませんでした。C ++ 20では、これが可能になります。

たとえば、ラムダをあるタイプのベクトルでのみ機能するように制限するにはどうすればよいですか？

一般的なラムダを書くことができます：

auto foo = []<typename T>(const auto& vec) { 
        std::cout<< std::size(vec) << '\n';
        std::cout<< vec.capacity() << '\n';
    };

ただし、intパラメータ（たとえばfoo(10);）を指定して呼び出すと、ある種の読みにくいエラーが発生する可能性があります。

prog.cc: In instantiation of 'main()::<lambda(const auto:1&)> [with auto:1 = int]':
prog.cc:16:11:   required from here
prog.cc:11:30: error: no matching function for call to 'size(const int&)'
   11 |         std::cout<< std::size(vec) << '\n';

C ++ 20では、次のように記述できます。

auto foo = []<typename T>(std::vector<T> const& vec) { 
        std::cout<< std::size(vec) << '\n';
        std::cout<< vec.capacity() << '\n';
    };

上記のラムダはテンプレート呼び出しステートメントを許可します：

<typename T>
void operator(std::vector<T> const& s) { ... }

テンプレートパラメータは、キャプチャオファーに従います[]。

で呼び出すとint (foo(10);)、より良いメッセージが表示されます。

note:   mismatched types 'const std::vector<T>' and 'int'

上記の例では、コンパイラーは本体内のコードよりもラムダインターフェイスの不整合について警告することができます。

もう1つの重要な側面は、ユニバーサルラムダでは、テンプレートの型ではなく、変数しか持たないことです。したがって、それにアクセスする場合は、decltype（x）を使用する必要があります（引数（auto x）のラムダ式の場合）。これにより、一部のコードがより冗長で複雑になります。

例（P0428のコードを使用）：

auto f = [](auto const& x) {
    using T = std::decay_t<decltype(x)>;
    T copy = x;
    T::static_function();
    using Iterator = typename T::iterator;
}

今、あなたは次のように書くことができます：

auto f = []<typename T>(T const& x) {
    T::static_function();
    T copy = x;
    using Iterator = typename T::iterator;
}

上記のセクションでは、C ++ 20の概要を説明しましたが、もう1つ別のユースケースがあります。この手法はC ++ 14でも可能です。読み続けてください。

ボーナス-ラムダを使用したリフト

現在、関数のオーバーロードがあり、それらを標準のアルゴリズム（または、呼び出されたオブジェクトを必要とするもの）に渡したい場合、問題が発生します。

// two overloads:
void foo(int) {}
void foo(float) {}

int main()
{
  std::vector<int> vi;
  std::for_each(vi.begin(), vi.end(), foo);
}

GCC 9（トランク）から次のエラーを受け取ります。

error: no matching function for call to 
for_each(std::vector<int>::iterator, std::vector<int>::iterator,
 <unresolved overloaded function type>)
   std::for_each(vi.begin(), vi.end(), foo);
                                       ^^^^^

ただし、ラムダを使用して目的のオーバーロード関数を呼び出すことができるトリックがあります。

基本的な形式では、単純な値タイプの場合、2つの関数について、次のコードを記述できます。

std::for_each(vi.begin(), vi.end(), [](auto x) { return foo(x); });

そして最も一般的な形式では、もう少しタイプする必要があります：

#define LIFT(foo) \
  [](auto&&... x) \
    noexcept(noexcept(foo(std::forward<decltype(x)>(x)...))) \
   -> decltype(foo(std::forward<decltype(x)>(x)...)) \
  { return foo(std::forward<decltype(x)>(x)...); }

かなり複雑なコード...そうですか？ :)

それを解読してみましょう：

一般化されたラムダを作成し、取得したすべての引数を渡します。それを正しく判断するには、noexceptと戻り値の型を指定する必要があります。そのため、正しいタイプを取得するために、呼び出しコードを複製する必要があります。
このようなLIFTマクロは、C ++ 14をサポートするすべてのコンパイラーで機能します。
