sekitar teknologi: QATOK # 2



Pada tanggal 5 Maret, di kantor Odnoklassniki di St. Petersburg, sekitar Tech akan diadakan: QATOK # 2 - pertemuan yang didedikasikan untuk pengujian dan segala sesuatu yang berkaitan dengan jaminan kualitas dalam proyek-proyek TI. Kami sedang menunggu 3 laporan teknis dari perwakilan OK, Yandex dan VK, mereka akan berbicara tentang masalah swa-uji Espresso-Android di dunia nyata, berbagi pengalaman mereka menggunakan generator Openapi untuk menulis swa-uji di REST-API dan menunjukkan cara mengevaluasi cakupan uji API tanpa menghapus Kode Overover.

Daftar di ok.tech: QATOK # 2

Memecahkan masalah tes otomatis Espresso-Android di dunia nyata -
Alexey Tyurin, VKontakte



Seringkali, proyek yang telah mendapatkan momentum dan memiliki basis kode yang besar, sampai pada kesimpulan bahwa Anda perlu menulis autotests. Termasuk UI-autotests. Dalam situasi ini, Anda tidak memiliki kesempatan untuk membuat kembali arsitektur proyek untuk pengujian secara signifikan. Akibatnya, tes yang dibuat menjadi tidak stabil dan mengalihkan perhatian tim pengembangan produk lebih dari yang mereka bantu.
Tujuan dari laporan: untuk menunjukkan kepada orang-orang bagaimana masalah khusus diselesaikan dalam otomatisasi pengujian UI dari aplikasi Android (kasus per kasus). Selain itu, solusi ini tidak memerlukan pengerjaan ulang arsitektur aplikasi atau revisi signifikan untuk autotest.

Menilai Cakupan Tes API Tanpa Menghapus Cakupan Kode -
Sergey Alekseev, Classmates



API adalah salah satu layanan terpenting di Odnoklassniki, diperlukan untuk pekerjaan platform pengguna dan sejumlah besar mitra eksternal, oleh karena itu menguji API adalah tugas terpenting yang membutuhkan perhatian khusus dan banyak solusi non-standar.
Dalam laporan itu, saya akan berbicara tentang bagaimana pengujian API diorganisasikan di Odnoklassniki, berbagi pengalaman saya dalam mengembangkan alat internal yang membantu kami menemukan titik-titik yang tidak ditemukan / lemah dalam pengujian otomatis API, maka saya akan menunjukkan bagaimana Anda dapat mengumpulkan alat analisis untuk mencakup metode API dengan tes otomatis untuk infrastruktur Anda untuk persyaratan khusus, dan pada akhirnya saya akan memberikan perhatian khusus pada pendekatan pengumpulan dan analisis cakupan pengujian.

Cara menggunakan generator OpenAPI untuk menguji REST API -
Viktor Orlovsky, Yandex.Verticals



Dalam laporan saya, saya akan memberi tahu Anda bagaimana kami menggunakan Openapi-generator di Yandex.Verticals untuk menulis autotest pada REST-API, berbagi pengalaman saya tentang bagaimana kami dapat dengan cepat dan mudah menutupi sejumlah besar API layanan baru dengan autotests, dan juga memperhatikan masalah apa yang kami temui di sepanjang jalan dan bagaimana menyelesaikannya. 
Kami akan beralih dari tes yang diwarisi pada klien http ke API ke generasi tes nyata berdasarkan spesifikasi.
