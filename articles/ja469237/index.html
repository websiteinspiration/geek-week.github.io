それはすべて、Gradle設定の微妙な点を学び、Android開発における（そして実際に）その機能を理解したかったという事実から始まりました。私はライフサイクルと本から始め、徐々にシンプルなタスクを書き、最初のGradleプラグイン（buildSrc内）を作成しようと試み、それが始まりました。


Android開発の現実の世界に近いことをすることに決め、レイアウトxmlマークアップファイルを解析して、ビューへのリンクを含むJavaオブジェクトを作成するプラグインを作成しました。その後、彼はアプリケーションマニフェストの変換に専念しました（これは作業ドラフトの実際のタスクで必要でした）。変換後、マニフェストは約5k行を要し、そのようなxmlファイルを使用してIDEで作業することは非常に困難です。


そこで、Androidプロジェクト用のコードとリソースを生成する方法を見つけましたが、やがてもっと何かが欲しくなりました。Groovyがすぐに使えるように、AST（抽象構文ツリー）をコンパイル時間に変換するのはクールだという考えがありました。そのようなメタプログラミングは多くの可能性を切り開いて、空想です。


理論は単なる理論ではないので、Androidの開発に役立つものを作成することで、このトピックの研究を強化することにしました。最初に頭に浮かんだのは、システムコンポーネントを再作成するときの状態の保持です。大まかに言えば、Bundleでの変数の保存は最小限のボイラープレートで可能な限り簡単です。

private static void transformDir(File input, File dest) {
    if (dest.exists()) {
      FileUtils.forceDelete(dest)
    }
    FileUtils.forceMkdir(dest)
    String srcDirPath = input.getAbsolutePath()
    String destDirPath = dest.getAbsolutePath()
    for (File file : input.listFiles()) {
      String destFilePath = file.absolutePath.replace(srcDirPath, destDirPath)
      File destFile = new File(destFilePath)
      if (file.isDirectory()) {
        transformDir(file, destFile)
      } else if (file.isFile()) {
        if (file.name.endsWith(".class")
            && !file.name.endsWith("R.class")
            && !file.name.endsWith("BuildConfig.class")
            && !file.name.contains("R\$")) {
          transformSingleFile(file, destFile)
        } else {
          FileUtils.copyFile(file, destFile)
        }
      }
    }
  }

private static void transformClass(String inputPath, String outputPath) {
    FileInputStream is = new FileInputStream(inputPath)
    ClassReader classReader = new ClassReader(is)
    ClassWriter classWriter = new ClassWriter(ClassWriter.COMPUTE_FRAMES)

    StaterClassVisitor adapter = new StaterClassVisitor(classWriter)

    classReader.accept(adapter, ClassReader.EXPAND_FRAMES)
    byte [] newBytes = classWriter.toByteArray()
    FileOutputStream fos = new FileOutputStream(outputPath)
    fos.write(newBytes)
    fos.close()
  }
