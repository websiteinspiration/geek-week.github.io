Modélisation d'un modèle Ising bidimensionnel en C ++ (à l'aide du package graphique OpenGL)

Ising Model


Le modèle d'Ising a été introduit pour comprendre la nature du ferromagnétisme et a influencé l'étude des transitions de phase et des phénomènes critiques. Le ferromagnétisme décrit l'apparition d'une aimantation spontanée dans les ferromagnétiques en dessous d'une certaine température - le point de Curie. Au point de Curie (une plage de températures étroite), l'ordre se produit, dans ce cas, l'alignement des moments magnétiques, ce qui entraîne une transition de phase, c'est-à-dire que les propriétés de la substance changent brusquement.

Initialement, le modèle a été proposé par V. Lenz en 1920, et étudié par Ising pour un boîtier unidimensionnel, en l'honneur duquel il a obtenu son nom. Dans ce modèle, chaque sommet du réseau cristallin prend un nombre: +1 ou -1, - champ «haut» ou «bas». Le nombre qui correspond aux sommets est appelé un spin. Ainsi, un réseau de N spins peut être dans 2 états N. Chaque état correspond à l'énergie obtenue à partir de l'interaction par paires des spins des atomes voisins:

De plus, J> 0 décrit le comportement d'un ferromagnet, J <0 est un antiferromagnet.

Méthode Monte Carlo


Comme mentionné précédemment, un total de 2 N états est possible, et pour un nombre suffisamment grand de N spins, il est difficile d'obtenir des résultats numériques. Par exemple, pour N = 10, nous obtenons 2 10 états qui ne sont pas si simples à modéliser directement, par conséquent, une approche statistique est utilisée pour la modélisation.

Dans cette approche, le système est considéré dans un état d'équilibre thermodynamique à une certaine température T. Lors de l'échange d'énergie avec l'environnement, l'énergie changera près de l'état d'équilibre, et l'énergie moyenne d'une particule est proportionnelle à T.La mise en œuvre d'un changement aléatoire constant autour de l'état d'équilibre utilise la méthode de Monte Carlo et la modélisation peut être divisée en étapes:

  1. Jouez les états α i du système (par exemple, au hasard) à un T fixe;
  2. Considérer pour ces états les caractéristiques thermodynamiques proches de l'équilibre (énergie E, aimantation M);
  3. Moyenne des valeurs obtenues


Algorithme Metropolis.


Dans les problèmes de mécanique statistique, les expressions «méthode Monte Carlo» et «méthode d'échantillonnage Metropolis» sont presque synonymes. Nous donnons la forme la plus générale de l'algorithme Metropolis comme exemple de système de spins ou de particules:

  1. Nous formons la configuration initiale.
  2. Nous effectuons un changement de test aléatoire dans la configuration initiale. Par exemple, nous sélectionnons au hasard une rotation et essayons de la renverser. Ou sélectionnez une particule aléatoire et essayez de la déplacer vers une distance aléatoire.
  3. Nous calculons ∆, c'est-à-dire la variation de l'énergie du système due à la modification de la configuration de test.
  4. Si ∆ est inférieur ou égal à zéro, nous acceptons la nouvelle configuration et passons à l'étape 8.

  6. Nous générons un nombre aléatoire rnd dans l'intervalle (0, 1)
  7. Si rnd≤W, nous acceptons la nouvelle configuration, sinon, sauvegardez la configuration précédente.
  8. Nous déterminons les valeurs des grandeurs physiques requises.
  9. Répétez les étapes 2 à 8 pour obtenir suffisamment de configurations ou de «tests».
  10. Nous calculons des moyennes sur des configurations statistiquement indépendantes les unes des autres.

Modèle d'Ising bidimensionnel en C ++


Nous allons écrire notre propre programme qui implémente le modèle d'Ising bidimensionnel en utilisant la méthode Monte Carlo et l'algorithme Metropolis. Nous implémentons les fonctionnalités suivantes:

  • L'utilisateur définit la taille du côté de la grille SIZE, la température maximale admissible T_MAX.
  • ISING SIZE×SIZE, , (0), , , (1), .
  • T ISING. , W S ( ). , – 0,01℃.
  • – . T=2,269℃ .
  • Un changement de configuration se produit directement sur l'écran toutes les 10 ms (millisecondes); à chaque changement de configuration, le nombre d'étapes Monte Carlo augmentera.
  • L'utilisateur peut changer la température avant chaque nouveau changement dans la configuration du réseau de spin.
  • Toutes les 10 s (secondes), des données seront transmises à la console (coefficient d'acceptation, énergie moyenne par rotation, carré moyen d'énergie par rotation, magnétisation moyenne, carré moyen de magnétisation).

#include <iostream>
#include <random>
#include <locale>
#include "glut.h"

using namespace std;

#define SIZE 256
#define SIZE_PX 800
#define T_MAX 4
#define CURIE_SCALE 1000. 

double quadSize = SIZE_PX / (double)SIZE;

short lattice[SIZE][SIZE];
double w[5];
double T = T_MAX, M, E;

int ratio = 0;
size_t nmcs = 0;
double ecum = 0, e2cum = 0, mcum = 0, m2cum = 0;

void display()
{
	glClear(GL_COLOR_BUFFER_BIT);
	
	glBegin(GL_QUADS);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			glColor3f(lattice[i][j], lattice[i][j], lattice[i][j]);
			glVertex2d(i, j);
			glVertex2d(i, j + 1);
			glVertex2d(i + 1, j + 1);
			glVertex2d(i + 1, j);
		}
	glEnd();

	glBegin(GL_QUADS);
	{
		glColor3f(1, 0, 0);
		glVertex2d(SIZE, 0);
		glVertex2d(SIZE, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, 0);

		glColor3f(1, 1, 1);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
	}
	glEnd();

	glutSwapBuffers();
	glFlush();
}

void calcW()
{
	double e4 = exp(-4 / T), e8 = e4 * e4;
	w[0] = w[4] = e8;
	w[1] = w[3] = e4;
	w[2] = 0;
}

void init()
{
	M = E = 0;
	static std::default_random_engine generator;
	std::uniform_int_distribution<int> distribution(0, RAND_MAX);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			lattice[i][j] = ((distribution(generator) / (double)RAND_MAX >= 0.5) - 1) * 2 + 1;
			M += lattice[i][j];
		}

	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			E += (i + 1 != SIZE) ? lattice[i][j] * lattice[i + 1][j] : 0;
			E += (j + 1 != SIZE) ? lattice[i][j] * lattice[i][j + 1] : 0;
		}

	calcW();
}

void metropolis()
{
	int x, y, sum;
	for (int i = 0; i < SIZE * SIZE; i++)
	{
		x = rand() % SIZE;
		y = rand() % SIZE;
		sum = lattice[(x - 1 + SIZE) % SIZE][y] +
			lattice[(x + 1 + SIZE) % SIZE][y] +
			lattice[x][(y - 1 + SIZE) % SIZE] +
			lattice[x][(y + 1 + SIZE) % SIZE];

		if (sum * lattice[x][y] <= 0 || (rand() / (double)RAND_MAX) < w[sum / 2 + 2])
		{
			lattice[x][y] = -lattice[x][y];
			::ratio++;
			M += 2 * lattice[x][y];
			E -= 2 * lattice[x][y] * sum;
		}
	}
}

void timer1(int)
{
	display();

	metropolis();

	nmcs++;

	ecum += E;
	e2cum += E * E;
	mcum += M;
	m2cum += M * M;


	glutTimerFunc(10, timer1, 0);
}

void outputData(int)
{
	double norm = 1 / (double)(nmcs * SIZE * SIZE);

	cout << "  = " << ::ratio * norm << endl;
	cout << "    = " << ecum * norm << endl;
	cout << "     = " << e2cum * norm << endl;
	cout << "  = " << mcum * norm << endl;
	cout << "   = " << m2cum * norm << endl << endl;

	glutTimerFunc(10000, outputData, 0);
}

void keyboard(unsigned char key, int x, int y)
{
	switch (key)
	{
	case 'w': case 'W':
	{
		T += 0.01;
		T = (T >= T_MAX) ? T_MAX : T;
		calcW();
		break;
	}
	case 's': case 'S':
	{
		T -= 0.01;
		T = (T <= 0.01) ? 0.01 : T;
		calcW();
		break;
	}
	}
}

void motion(int x, int y)
{
	double k = 1 - y / (double)SIZE_PX;
	T = T_MAX * k;
	T = (T >= T_MAX) ? T_MAX : T;
	T = (T <= 0.01) ? 0.01 : T;
	calcW();
}

int main()
{
	setlocale(LC_ALL, "ru");
	init();
	glutInitDisplayMode(GLUT_SINGLE | GLUT_RGB);
	glutInitWindowSize(SIZE_PX + 100, SIZE_PX);
	glutCreateWindow("ISING");
	glMatrixMode(GL_PROJECTION);
	glLoadIdentity();
	gluOrtho2D(0, SIZE + SIZE / 8, 0, SIZE);
	glutDisplayFunc(display);
	glutKeyboardFunc(keyboard);
	glutMotionFunc(motion);
	glutTimerFunc(1, timer1, 0);
	glutTimerFunc(10000, outputData, 0);

	glutMainLoop();
}

Notez le but de la création de certaines fonctions dans ce produit logiciel:

  • init () - crée une configuration initiale;
  • display () - rendu du réseau de spins et échelle de température;
  • calcW () - calcul des probabilités de transition;
  • metropolis () - implémentation de l'algorithme Metropolis;
  • timer1 () - mise à jour des données après chaque changement de configuration, augmentant le nombre d'étapes de Monte Carlo;
  • outputData () - calcul de la moyenne par rotation, sortie des données nécessaires;
  • clavier () et mouvement () - la possibilité de changer la température en utilisant l'échelle de deux manières - boutons du clavier et clic de souris, respectivement;

Cela vaut la peine de tirer quelques conclusions:

  • À une température critique T = 2,269 ℃, une transition de phase se produit d'un état ferromagnétique désordonné à un état ferromagnétique ordonné.
  • Lorsque la température baisse, les caractéristiques physiques et statistiques suivantes changent comme suit:
    • Le taux d'acceptation est réduit;
    • L'énergie moyenne par rotation diminue;
    • Le carré moyen d'énergie par rotation augmente;
    • L'aimantation moyenne diminue;
    • Le carré moyen de l'aimantation augmente.
