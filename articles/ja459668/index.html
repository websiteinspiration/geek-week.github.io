写真からテキストを認識する方法：Visionフレームワークの新機能

これで、Visionフレームワークはテキストを以前とは異なり、実際に認識することができます。 Dodo ISにいつ適用できるか楽しみです。その間、ボードゲームMagic The Gatheringからのカードの認識とそれらからのテキスト情報の抽出に関する記事の翻訳。




Visionフレームワークは、2017年にiOS 11とともにWWDCで一般に初めて導入されました。Visionは、

開発者がオブジェクト、水平面、バーコード、顔の表情、およびテキストを分類および識別できるように作成されました。

ただし、テキスト認識に問題がありました。ビジョンはテキストのある場所を見つけることができましたが、実際のテキスト認識は行われませんでした。もちろん、個々のテキストフラグメントの周囲に境界ボックスを表示するのは良かったですが、それらを個別に引き出して認識する必要がありました。

この問題は、iOS 13に含まれていたVisionのアップデートで解決されました。Visionフレームワークは、真のテキスト認識を提供します。

これをテストするために、Magic The Gatheringボードゲームのカードを認識し、そこからテキスト情報を抽出できる非常にシンプルなアプリケーションを作成しました。

  • カード名;
  • リリースコード;
  • コレクション番号（別名インデックス）。

これは、受け取りたい地図と選択したテキストの例です。



カードを見ると、「このテキストはかなり小さいですが、カードには干渉する可能性のある他のテキストがたくさんあります」と考えるかもしれません。しかし、ビジョンに関しては、これは問題ではありません。

まず、を作成する必要がありますVNRecognizeTextRequest。本質的に、これは私たちが認識したいことの説明に加えて、認識言語の設定と正確さのレベルです：

let request = VNRecognizeTextRequest(completionHandler: self.handleDetectedText)
request.recognitionLevel = .accurate
request.recognitionLanguages = ["en_GB"]

完了ブロックの形式はhandleDetectedText(request: VNRequest?, error: Error?)です。それをコンストラクタに渡しVNRecognizeTextRequest、残りのプロパティを設定します。

認識精度の2つのレベルが用意されています.fastと.accurate。私たちのカードの下部にはかなり小さいテキストがあるため、私はより高い精度を選択しました。より速いオプションは、おそらく大量のテキストに適しています。

すべてのカードが含まれているため、認識をイギリス英語に限定しました。複数の言語を指定できますが、追加の言語ごとにスキャンと認識に少し時間がかかることを理解する必要があります。
言及する価値のあるプロパティがさらに2つあります。

  • minimumTextHeight: float-. , . , , , . , . , .

これでリクエストが作成されたので、画像と一緒にリクエストハンドラーに渡す必要があります。

let requests = [textDetectionRequest]
let imageRequestHandler = VNImageRequestHandler(cgImage: cgImage, orientation: .right, options: [:])
DispatchQueue.global(qos: .userInitiated).async {
    do {
        try imageRequestHandler.perform(requests)
    } catch let error {
        print("Error: \(error)")
    }
}

カメラからの画像を直接使用して、からに変換UIImageしCGImageます。これはVNImageRequestHandler、オリエンテーションフラグと組み合わせて使用され、ハンドラーが認識する必要のあるテキストをハンドラーが理解できるようにします。

このデモの一部として、私は電話を縦向きでのみ使用します。だから当然、私は向きを追加し.rightます。だからパダージ！

デバイスのカメラの向きはデバイスの回転とは完全に分離されており、常に左側と見なされます（2009年のデフォルトのように、写真を撮るには、スマートフォンを横向きにする必要があります）。もちろん、時代は変わり、写真やビデオは基本的に縦向きで撮影しますが、カメラはまだ左に配置されています。

ハンドラーが構成されたらすぐに、優先スレッドに移動して.userInitiated、要求の実行を試みます。これが一連のクエリであることに気付くでしょう。これは、1つのパスで複数のデータを引き出そうとするためです（つまり、同じ画像から顔とテキストを識別します）。エラーがない場合は、テキストが検出された後で、リクエストを使用して作成されたコールバックが呼び出されます。

func handleDetectedText(request: VNRequest?, error: Error?) {
    if let error = error {
        print("ERROR: \(error)")
        return
    }
    guard let results = request?.results, results.count > 0 else {
        print("No text found")
        return
    }

    for result in results {
        if let observation = result as? VNRecognizedTextObservation {
            for text in observation.topCandidates(1) {
                print(text.string)
                print(text.confidence)
                print(observation.boundingBox)
                print("\n")
            }
        }
    }
}

ハンドラーはクエリを返し、クエリにはresultsプロパティが含まれています。各結果はVNRecognizedTextObservation、結果に対していくつかのオプションがあるものです（以下-候補）。

認識されたテキストのユニットごとに最大10個の候補を取得でき、それらは信頼度の降順でソートされます。これは、パーサーが最初の試行で誤って認識する特定の用語がある場合に役立ちます。しかし、結果の正確さに自信がなくても、後で正しく判断します。

この例では、最初の結果のみが必要なので、ループしてobservation.topCandidates(1)テキストと信頼度の両方を抽出します。候補者自身は異なるテキストと自信を持っています.boundingBoxが、同じままです。.boundingBoxは左下隅を原点とする正規化された座標系を使用するため、UIKitでさらに使用される場合は、便宜上変換する必要があります。

それで十分です。これを通してカードの写真を実行すると、iPhone XS Maxで0.5秒未満で次の結果が得られます。

これは信じられない！各テキストが認識され、独自の境界ボックスに配置され、結果として信頼度1.0で返されました。

非常に小さな著作権でもほとんど正しいです。これはすべて、重量が3.1 MBの3024x4032イメージで行われました。最初に画像を縮小すると、プロセスはさらに速くなります。特別なニューラルエンジンを備えた新しいA12バイオニックチップでは、このプロセスがはるかに高速であることも注目に値します。

テキストが認識されたら、最後に行うことは、必要な情報を引き出すことです。ここにすべてのコードを配置するわけではありませんが、主要なロジックは、それぞれを反復し.boundingBoxて位置を決定することです。これにより、左下隅と左上隅のテキストを選択して、それより右側にあるものはすべて無視できます。
最終結果は、カードをスキャンして1秒未満で結果を返すアプリケーションです。
