Cuidando da lição de casa: faça backup do seu laptop ou desktop usando os Veeam Agents gratuitos

Na vida cotidiana, eu, como muitos de vocês, queridos leitores, trabalho no escritório a maior parte do tempo, e isso me convém perfeitamente. Hoje, porém, a situação é tal que eu e meus colegas em diferentes partes do mundo e, penso, a maioria das pessoas que lê este artigo, todos trabalhamos em casa.

E aqui será bastante impróprio lembrar a necessidade de fazer backup de um computador ou laptop doméstico. Para quando o ambiente de trabalho muda, a maneira usual de trabalhar com dados também muda. Tudo é óbvio com o trabalho no escritório: acesso direto aos recursos necessários, backups centralizados, se houver, você sempre pode entrar em contato com o suporte técnico. Com o trabalho em casa, você precisa fornecer muitas coisas, incluindo backup.

Acredito que muitos de vocês já estão usando o Veeam Agent para Microsoft Windows ou o Veeam Agent para Linux para isso. Se ainda não, talvez esteja interessado na breve revisão de hoje. Bem-vindo ao gato.



Em 2014, quando a primeira versão do nosso produto, agora conhecida como Veeam Agent para Microsoft Windows , foi lançada , era chamada Veeam Endpoint Backup Free e tinha a única edição não comercial.

Obviamente, a solução mudou muito desde então, e uma versão comercial foi lançada, na qual existem muitos recursos avançados - gerenciamento centralizado, driver CBT, suporte a cluster, processamento de disco paralelo e muito mais. O Veeam Agent for Linux também está em desenvolvimento, o que permite fazer backup de máquinas físicas e restaurar dados dos backups. Mas agora, quando no quintal de 2020, as novas versões do V eeam Agent para Microsoft Windows e Veeam Agent for Linuxhá também uma edição gratuita projetada para computadores individuais.

Portanto, se você precisar fazer backup de sua área de trabalho, laptop ou tablet que roda no Windows, use sua saúde.

Para o Veeam Agent for Linux, a história é semelhante:

  1. Uma captura instantânea dos volumes necessários é criada - isso é feito usando o módulo Veeamsnap especial. (Uma captura instantânea é criada em um volume com espaço livre suficiente.)

    Quando o backup é feito em um volume, o Veeam Agent copia a tabela de partição e o carregador de inicialização para o local de destino.
  2. O Veeam Agent for Linux lê dados do volume, realiza compactação e cópia no local de destino.
  3. Depois que todos os dados necessários são copiados, o instantâneo é excluído. Os dados são salvos em um arquivo de backup.

Veeam Agent para Microsoft Windows: SO suportado


As versões de SO de 64 e 32 bits (onde aplicável) são suportadas:

  • Microsoft Windows Server 2019
  • Microsoft Windows Server 2016
  • Canal semestral do Microsoft Windows Server (incluindo 1909)
  • Microsoft Windows Server 2012 R2
  • Microsoft Windows Server 2012
  • Microsoft Windows Server 2008 R2 SP1
  • Microsoft Windows 10 (da versão 1607 a 1909)
  • Microsoft Windows 8.1
  • Microsoft Windows 7 SP1

Notas:

  • Para sistemas operacionais de servidor, as edições Small Business Server e Server Essentials são suportadas.
  • As versões Server Core para sistemas de servidor em um cenário de implantação independente não são suportadas.
  • SO Windows Embedded / Windows IoT suportado (exceto compilações personalizadas).
  • Os Clusters de Failover da Microsoft são suportados quando o Veeam Agent for Microsoft Windows está executando o Veeam Backup & Replication (não há suporte para cenários de implantação autônomos).
  • O Microsoft Windows 10 Education é suportado a partir da compilação 10586.

Os demais requisitos do sistema são descritos em detalhes na documentação (em inglês).

Trabalho de backup no Veeam Agent for Microsoft Windows


Para backup, configure a tarefa de backup iniciando o assistente Nova Tarefa de Backup . Nele, selecione o tipo de backup:

  • Computador inteiro - a máquina inteira.
  • Backup em nível de volume - backup de volumes selecionados.
  • Backup em nível de arquivo - backup de arquivos específicos.

O Veeam Agent para Microsoft Windows pode usar diferentes tipos de sistemas de armazenamento como um local de armazenamento de backup:

  • Armazenamento local - unidade externa (USB, Firewire ou eSata). A principal coisa aqui, como você sabe, não é fazer backup no disco embutido na máquina.
  • Pasta compartilhada - compartilhamento SMB em um servidor de arquivos ou NAS.
  • Repositório de backup Veeam - repositório Veeam (local).
  • O repositório Veeam Cloud Connect é um repositório baseado em nuvem fornecido pelo provedor de serviços Veeam.
  • Microsoft OneDrive

No meu exemplo, o backup irá para a unidade USB - para isso, na etapa Destino , selecione Armazenamento local :



depois definimos as configurações específicas para o tipo de armazenamento selecionado.

Importante! Ao trabalhar com discos USB removíveis, lembre-se de que, se esse disco estiver formatado para o FAT32, ele não suporta o armazenamento de arquivos maiores que 4 GB. Portanto, não é recomendável usar discos como o armazenamento de backup de destino.

Em seguida, configure o agendamento na etapa Agendamento . Aqui você pode especificar se deseja fazer backup regularmente em determinados horários ou quando os eventos selecionados ocorrem:

  • Bloquear - quando você trancou o carro
  • Logoff - quando desconectado da conta atual
  • Quando o destino de backup está conectado - quando o armazenamento de backup de destino está conectado

Se a terceira opção for especificada, você também poderá instruir o Veeam Agent a terminar de desconectar (Ejetar) o armazenamento de destino no final do backup, para que o backup fique offline. Isso reduzirá o risco de exposição a uma cópia de backup do vírus de criptografia, se um for detectado.

Para fazer isso, selecione a opção Quando o destino do backup estiver conectado → Ejetar armazenamento removível após a conclusão do backup (proteção ao CryptoLocker) :



Veeam Agent for Linux


O Veeam Agent for Linux foi projetado para proteger os dados de máquinas em execução no Linux, e esta solução também possui uma edição gratuita. O kernel Linux 2.6.32 e superior é suportado.

São suportadas as versões de distribuições de 64 e 32 bits (quando aplicável):

  • Debian 8.0 - 10.3
  • Ubuntu 14.04, 16.04, 18.04, 19.10
  • RHEL 6.0 - 8.1
  • CentOS 6.0 - 8.1
  • Oracle Linux 6 - 8.1 (RHCK)
  • Oracle Linux 6 (começando com UEK R1) - Oracle Linux 7 (até UEK R5 U2)
  • SLES 11 SP4, 12 SP1 - 15 SP1
  • SLES para SAP 11 SP4, 12 SP1 - 15 SP1
  • Fedora 30 - 31
  • openSUSE Leap 15.1
  • openSUSE Tumbleweed

Notas:

  • O Fedora é suportado até e incluindo a versão 5.4.14 do kernel.
  • O openSUSE Tumbleweed é suportado até e incluindo a versão 5.1.7 do kernel.
  • O kernel Linux 2.6.32-754.6.3 não é suportado no CentOS / RHEL e Oracle Linux (RHCK).

Para mais informações, consulte a documentação (em inglês).

Os mesmos tipos de backups são suportados como no Veeam Agent for Microsoft Windows, incluindo backups em nível de arquivo. As configurações de backup podem ser definidas usando as etapas no assistente ou na linha de comando.

Mais informações sobre cada uma das soluções podem ser encontradas no guia do usuário para Windows (em inglês) ou Linux (também em inglês).

Sobre as limitações da edição gratuita


Para o Veeam Agent for Linux (comparado à edição comercial do Server):

  1. Apenas 1 tarefa de backup pode ser criada.
  2. O repositório do Veeam Cloud Connect não é suportado como repositório de backup de destino.
  3. Os scripts de pré-congelamento e pós-descongelamento não são suportados para a tarefa de backup.
  4. O backup levando em consideração o trabalho do DBMS Oracle, MySQL, PostgreSQL não é suportado.

Mais detalhes estão na tabela aqui . E sobre o que está acontecendo "nos bastidores", como nossos engenheiros trabalham para fornecer suporte a esse "zoológico" e à compatibilidade da solução Veeam, leia o artigo de Sergey Shtepa, desenvolvedor em um blog em Habré.

As restrições para o Agent for Windows estão listadas em detalhes na mesma tabela . Dos principais, notamos o seguinte:

  1. Os backups baseados em aplicativos não são suportados.
  2. O repositório do Veeam Cloud Connect não é suportado como repositório de backup de destino.
  3. O Microsoft OneDrive não é suportado como armazenamento de backup para máquinas que executam o sistema operacional Windows server.
  4. O Microsoft OneDrive pode ser usado apenas para backups do status do sistema. (Para mais informações sobre tipos de backup, consulte aqui .)
