パターンテンプレートメソッド（テンプレートメソッド）


パターンの名前と分類

テンプレートメソッドは、クラスの動作のパターンです。

目的

テンプレートメソッドは、アルゴリズムの基礎を定義し、サブクラスが全体として構造を変更せずにアルゴリズムの一部のステップを再定義できるようにします。

動機

ApplicationクラスとDocumentクラスを持つアプリケーションフレームワークについて考えます。 Applicationクラスは、外部形式（ファイルなど）で保存されている既存のドキュメントを開く責任があります。 Documentクラスのオブジェクトは、ファイルから読み取った後のドキュメント情報を表します。

このフレームワークに基づいて構築されたアプリケーションは、特定のニーズを満たすApplicationクラスとDocumentクラスのサブクラスを生成できます。

たとえば、グラフィカルエディターはDrawApplicationとDrawDocumentのサブクラスを定義し、スプレッドシートはSpreadsheetApplicationとSpreadsheetDocumentのサブクラスを定義します。


抽象クラスApplicationは、OpenDocument操作でドキュメントを開いて読み取るためのアルゴリズムを定義します。

void Application::OpenDocument (const char* name) {
      if (!CanOpenDocument(name)) {
          //   
          return;
      }

      Document* doc = DoCreateDocument();

      if (doc) {
          _docs->AddDocument(doc);
          AboutToOpenDocument(doc);
          doc->Open();
          doc->DoRead();
      }
}

OpenDocument操作は、ドキュメントを開くためのすべてのステップを定義します。ドキュメントを開くことが可能かどうかをチェックし、Documentクラスのオブジェクトを作成して、ドキュメントのセットに追加し、ファイルからドキュメントを読み取ります。

OpenDocumentフォームの操作は、望ましい動作を得るためにサブクラスで置き換えられる抽象操作のカテゴリでアルゴリズムを記述するテンプレートメソッドと呼ばれます。 Applicationクラスのサブクラスは、それらが開くことができ（CanOpenDocument）、ドキュメントを作成できる（DoCreateDocument）ことを確認します。 Documentクラスのサブクラスはドキュメントを読み取ります（DoRead）。テンプレートメソッドは、ドキュメントが開かれようとしているという情報をアプリケーションサブクラスが取得できるようにする操作も定義します（AboutToOpenDocument）。

抽象演算を使用してアルゴリズムのいくつかのステップを定義すると、テンプレートメソッドはそれらのシーケンスをキャプチャしますが、それらをApplicationおよびDocumentクラスのサブクラスに実装できます。

適用性

パターンテンプレート法を適用するための基本的な条件：

  • アルゴリズムの不変部分の1回の使用で、動作の変更の実装はサブクラスの裁量のままです。
  • コードの重複を回避するために、すべてのサブクラスに共通の動作を1つのクラスに分離してローカライズする必要性。これは、William OpdykeとRalph Johnson [OJ93]によって記述された「一般化するブラケット」手法の良い例です。まず、既存のコードの違いが明らかになり、それが別の操作に転送されます。最終的に、さまざまなコードフラグメントは、新しい操作が呼び出されるテンプレートメソッドに置き換えられます。
  • サブクラス拡張管理。テンプレートメソッドは、特定のポイントでフック（「結果」セクションを参照）をトリガーし、それらのポイントでのみ拡張を許可するように定義できます。

構造


参加者の

AbstractClass（アプリケーション）-抽象クラス：

  • アルゴリズムのステップを実装するために特定のサブクラスで置き換えられる抽象プリミティブ操作を定義します。
  • アルゴリズムのスケルトンを定義するテンプレートメソッドを実装します。テンプレートメソッドは、プリミティブ操作と、AbstractClassクラスまたは他のオブジェクトで定義された操作を呼び出します。

ConcreteClass（MyApplication）-特定のクラス：

  • サブクラスに依存する方法でアルゴリズムのステップを実行するプリミティブ操作を実装します。



ConcreteClass 関係は、アルゴリズムの不変ステップがAbstractClassで実行されることを前提としています。

結果

テンプレートメソッドは、コードを再利用するための基本的な手法の1つです。これらは、ライブラリクラスで共通の動作を引き出す機能を提供するため、クラスライブラリで特に重要な役割を果たします。

テンプレートメソッドは、ハリウッドの原則と呼ばれることもある逆のコード構造につながり、この映画帝国でよく使用される「私たちに電話しないで、私たちはあなたに電話します」というフレーズを暗示しています[Swe85]。この場合、これは、親クラスがサブクラス操作を呼び出すことを意味し、その逆は行いません。

テンプレートメソッドは、次のタイプのオペレーションを呼び出します。

  • 特定の操作（ConcreteClassクラスまたはクライアントクラスのいずれか）
  • AbstractClassクラスの特定の操作（つまり、すべてのサブクラスに役立つ操作）。
  • 基本操作（つまり、抽象操作）;
  • ファクトリー・メソッド（ファクトリー・メソッド・パターン（135）を参照）。
  • サブクラスで拡張できるデフォルトの動作を実装するフック操作。多くの場合、このようなデフォルトの操作は何もしません。

テンプレート操作では、フック操作（置き換え可能）と抽象操作（置き換えが必要）を明確に区別することが重要です。抽象クラスを最大の効率で再利用するには、サブクラスの作成者は、どの操作が置き換えられるかを理解する必要があります。

サブクラスは、操作を置き換え、親クラスから操作を明示的に呼び出すことにより、操作の動作を拡張できます。

void DerivedClass::Operation () {
      //   DerivedClass
      ParentClass::Operation();
}

残念ながら、継承されたオペレーションを呼び出す必要性を忘れるのは非常に簡単です。このような操作をボイラープレートメソッドに変換して、サブクラスがそれを拡張する方法を親が制御できるようにすることができます。アイデアは、親クラスのテンプレートメソッドからフック操作を呼び出すことです。その後、サブクラスはこの操作を正確に再定義できます。

void ParentClass::Operation () {
      //  ParentClass
      HookOperation();
}

親クラスParentClassでは、HookOperation操作は何もしません。

void ParentClass::HookOperation () { }

サブクラスはHookOperationをオーバーライドして動作を拡張します。

void DerivedClass::HookOperation () {
   //    
}

コード例

次のC ++の例は、親クラスがサブクラスに不変条件を課す方法を示しています。例は、NeXT AppKitライブラリ[Add94]から取得されます。画面上での描画をサポートするViewクラスについて考えてみましょう。これは、サブクラスがフォーカスされている場合にのみビューを変更できるという事実から成り立っています。これには、特定の描画コンテキスト（たとえば、色やフォント）を設定する必要があります。

テンプレートのDisplayメソッドを使用して状態を設定できます。Viewクラスは、描画コンテキストをそれぞれ設定およびリセットする2つの特定の操作（SetFocusおよびResetFocus）を定義します。ViewクラスのViewフック操作は、描画自体を行います。Displayは、DoDisplayの前にSetFocusを呼び出してコンテキストを準備し、DoDisplayの後にResetFocusを呼び出してコンテキストをリセットします。

void View::Display () {
      SetFocus();
      DoDisplay();
      ResetFocus();
}

不変条件をサポートするために、Viewクライアントは常にDisplayを呼び出し、Viewサブクラスは常にDoDisplayを置き換えます。

Viewクラスでは、DoDisplay操作は何もしません。

void View::DoDisplay () { }

サブクラスはそれをオーバーライドして、特定の描画動作を追加します。

void MyView::DoDisplay () {
      //   
}
