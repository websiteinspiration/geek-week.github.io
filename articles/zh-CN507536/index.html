成长过程的说明“从假设搜索到A / B测试”

读者好！



通过观察许多营销团队的工作，我注意到许多人在假设的搜索，优先级划分和检验中犯了同样的错误。



在本文中，我将讨论生长过程，尝试使复杂的材料尽可能简单。



我们将在文章中谈论什么



1.主意假设。

2.我们形成了积压的假设。

4.对ICE和PXL框架的假设进行优先级排序和评分。

5.实验设计。

6.准备测试。

7.进行实验。

8.分析和统计意义。

9.实施。



让我们开始吧。







免责声明：在材料上主要思想和工作方案，但在不加深其微妙之处的前提下，假定读者将理解在哪里进行挖掘，因为已经给出了主要框架。



在撰写本文时，我专注于营销团队和成长团队，请记住，对于其他团队而言，该过程可能不合适。



观念≠假设



在开始主要过程之前，您需要将基于假设的思维引入团队。重要的是要理解，这个想法最初的形式还不是一个假设。



假设的主要优点是它们是可测量的。您确切地知道如果开始检验假设，将会提高哪些指标。



假设公式



“动作[X]将使指标[Y]增加[Z]，因为[N]。”



让我们看一个差异的例子。



想法：

“我们需要将在线申请表格从5个步骤缩短到4个步骤，我认为我们将增加注册的转换。”



假设：

“如果我们将在线申请表格从5分减少到4分，我们会将注册转换率从2％增加到3％，增加1个百分点，因为在分析系统中，我们看到一些用户离开了第4步，并且尚未完成注册。”



如果不能将这个想法变成假设的形式，那么就应该放弃它。正确制定的​​假设会杀死毫无意义的行动，这些行动不会以任何方式影响指标。



继续。



我们形成了积压的假设







我们确定了一个假设，以及它与一个概念有何不同。

现在该用假设填充待办事项了。



假设的积压是我们假设以有序形式存在的单一基础。



请记住，假设检验会花费金钱（开发人员，分析师，营销人员，设计师等的时间），并且如果您了解假设不会以任何方式影响指标，则不要添加假设。



如果我们正在谈论IT产品，则所有假设都可以输入AAARRR漏斗中（在上图中，我已对其进行了示意性描绘）。

通过分析渠道的每个步骤，我们可以为每个步骤找到更多的增长点。



例如，我们采取了收购步骤，并按渠道/活动/组/广告素材等分析了所有吸引力渠道，在分析过程中，我们写下了可以显着提高新用户吸引力的假设。



还有哪些其他工具可以帮助我们找到增长点：



  • CJM（产品中的客户路径）。
  • JTBD。
  • 产品的定量分析。
  • UX分析和CustDev。


我不会更详细地介绍每种方式，因为这些主题非常广泛，值得单独撰写。



积压建议：



  • 团队必须有一个单一的假设积压，以便透明的工作和避免付出。
  • 指出该假设适用于AAARRR程序的哪一部分。
  • 因此，乍一看，您将了解它会影响漏斗的哪个级别。


ICE和PXL的假设优先级排序和评分



我们按照AAARRR漏斗的步骤填写了待办事项并分发了假设，现在重要的是要了解我们首先要测试的假设。



因此，我们顺利进行了优先级排序和评分。



想象一下，部门的所有员工都将假设添加到积压中，每个人都希望他的假设成为测试队列中的第一个。为避免误解和冲突，对假设进行优先排序和评分的过程应尽可能透明。这将有助于ICE评分框架。



ICE得分-这是一种



影响 -展示了这个想法将如何积极影响您要改进的关键指标。



信心 -展示您对评估影响和实施容易程度的信心。



缓解（易于实施）是对实施此想法所需的精力和资源的评估。



ICE公式：



ICE得分=影响力x信心x轻松程度



。



团队聚在一起，其每个成员以1到10的等级来评估影响力，信心和轻松度。此外，每个员工的得分在他们之间相乘并加到其他员工的价值上，因此我们为每个假设获得ICE得分。

影响：



一切都很简单，对关键指标的影响越大，表中的得分越高。



置信度：



只有一种计算置信度的方法-这是寻找支持证据的方法。为此，您可以创建自己的卡片来按1到10的等级分配等级，这将有助于团队更透明地进行等级分配，并消除一个雇员放1而另一雇员-10时的等级分散情况。



该卡片列出了常见的测试和证据类型，您可能拥有的信息以及它们所提供的可信度：测试结果，发布日期，个人信心，市场数据，其他人的意见等。

实验设计



我们对假设进行了优先排序，现在我们需要设计实验。



实验的设计是检验假设的详细技术任务，为了理解我们正在测试的内容，需要考虑的样本和指标，我们需要进行此项设计。



实验的设计基于这样的模板：



  • 假设。
  • 我们在做什么？
  • 用户正在测试什么。
  • 评估实验的关键指标。
  • 预期的效果。
  • 行动计划取决于实验结果。


让我们更详细地分析一个例子：



假设：



我们从积压中获取假设。



在我们的例子中，它看起来像这样：



如果我们将在线申请表格从5点减少到4点，我们会将注册转换率从2％增加到3％，增加1个百分点，因为在分析系统中，我们看到一些用户离开了4，无需完成注册。



我们的工作：



控制版本：我们将一切保持不变。



测试版本：在线申请表从5分减少到4分。



我们



针对哪些用户进行测试：仅针对新用户。



指标：



转换为注册。



行动计划：



如果我们的实验是积极的，并且我们看到指标有所改善（注册的注册展示率（CR）> 3％）-我们可以扩展到所有用户。



如果指标下降，则回滚。



如果度量标准不变，则保持不变。



在开始实验之前，请务必讨论并同意在收到结果后打算采取的进一步措施的计划。因此，您可以摆脱测试通过的情况，而最终产品负责人将无法实施该实验。



测试准备



开始实验之前，您需要执行一些准备步骤。



  1. 确定测试的类型。
  2. 计算样本量，置信区间和功效。
  3. 计算实验的持续时间。
  4. 准备测试工具。
  5. 确保实验干净。
  6. 设置分析系统。
  7. 设置细分进行测试。


让我们按顺序执行所有步骤。



1.确定测试类型



选择最适合您的测试选项。



以下是一些选择：



  • A / B测试。
  • MVT（多变量测试）。
  • 恶化的A / B测试。
  • A / A / B测试。


在我们的示例中，我们将考虑最常用的选项-A / B测试。



2.计算样本量，置信区间和功效



在开始之前，请确定您需要吸引多少用户使用每个A / B测试选项，以便最终的数据具有统计意义。



我们将使用的埃文米勒计算器来计算样本（见下图）。



在“基准转化率”中，我们表示对照版本的转化，在本例中为2％。



在“最小可检测效果”中，我们指出了对照版本的转化率将增加多少百分比，在我们的示例中为+ 1％。



显着性水平α（可信度水平）是您对第一类错误（如果是正确的，请拒绝测试的对照版本）承担的风险水平，通常α= 0.05。



这意味着在5％的情况下，您会发现A和B之间的差异，这实际上是偶然的。

重要性水平越低，发现偶然性差异的风险就越低。



统计功效1 −β（统计功效）是第二种误差，如果它是真的，我们将接受样本上的测试版本的概率（如果是真的，则有机会检测到效果）。在计划实验时，必须记住，功率必须相当高才能检测出与对照版本的偏差。如果您不知道应显示指标的百分比，请保留默认值（80％）。



结果，我们知道，对于正确的测试，每个A / B测试变体需要3,292个用户。







接下来，我们可以计算要获得具有统计意义的结果，我们需要获得多少次转化，以及它们之间的区别是多少。



我们将使用顺序采样计算器，







为了进行正确的分析，我们应该获得170次转换，并且A / B测试版本之间的差异应大于26次转换。



4.计算实验持续时间



将在计算中获得的样本数量用于测试每个版本，然后将其除以您的每日流量，即可得出测试所需的天数。



至少测试一整周 周末数据可能与工作日不同。



5.准备测试工具



我们计算了具有统计意义的实验所需的流量和转化次数，现在我们需要选择测试工具包。



该工具包包含两个主要模块：



1.流量来源。



准备流量来源，设置广告系列。



2.版本A和版本B之间的流量分配系统。



在这里，您可以通过两种方式进行操作：创建自己的工具，以便在开发人员的帮助下在版本之间分配流量，或者使用第三方服务，例如：



  • Google Optimize（位于GA的免费服务）
  • Kameleoon。
  • 优化地






6.确保实验干净。



在开始测试之前，请确保两个测试版本之间没有差异。

版本或流量上不应存在技术差异，因为任何差异都会影响实验的纯度。



为了获得最大的清洁度，请在开始之前进行A / A测试。与A / B测试不同，在A / A中，我们测试两个相同的页面（元素）。实验的目的不是看到它们的性能差异。



7.建立分析系统



重新检查分析系统，确保实验所需的所有指标均能正常工作。



一个常见的错误是在不检查分析和指标系统的情况下运行测试，结果是专家花费时间，预算，而所有工作都是徒劳的。



8.配置细分进行测试



回到实验的设计，让我们回顾进行实验的用户。



我们指出：“仅针对新用户。”



在流量分配系统或广告系统中，我们需要清除回头用户。



并进行完全相同的广告系列（如果您不对有机食品进行测试）。



实验



在开始之前，请检查一切是否真的正常工作。再次检查并运行实验。



随着实验的进行，监视流量问题和测试系统，并在出现问题时对其进行修复。



对实验的建议。



1.不要得出过早的结论。



即使第一天您看到测试版本在统计上有显着改进，也要结束实验。



过早的决策问题称为“偷看问题”或“偷窥问题”。



您越频繁地查看愿意根据其做出决定的A / B测试的中间结果，则该标准在不存在时会显示出显着差异的可能性就越大。



看下面的图片，它完美地说明了“偷看问题”的全部实质。







如果我们不了解侦听的问题，我们会在第四天完成实验，认为测试版本要好得多，因此没有必要进一步进行实验。

但是，完成实验到最后，我们将了解到，各组之间没有可测量的稳定差异。



您需要了解，即使我们测试相同的版本，转换的差异可能会随着观察值的累积而周期性地超出不可区分区域的边界。

这是完全正常的，因为以这样的方式形成边界：在测试相同版本时，仅在95％的情况下，它们之间会出现差异。



为了不被“偷看问题”迷住，在开始之前，我们先计算实验的样本量和持续时间。按照您的计划。



2.有一个测试紧急关闭系统



如果您要在自助服务上进行测试，则建议使用紧急关机系统来测试和分析异常，因为该实验会破坏产品。



分析和统计意义



我们进行了一项实验并收到了数据。



现在，我们需要了解，我们在版本A和版本B之间的度量标准中收到的更改与我们的假设或偶然性有关。



换句话说，该变化是否具有统计意义。



在我们的示例中，我们收到了以下数据：



样本A：3,292次会话，67次转换。

样本B：3200次会话，99次转换。



我们将使用计算器来计算统计显着性。







P值= 0.000691，也就是说，在相同的测试组中观察到差异的可能性很小。这意味着可以高度确定地将指标的增长与我们的更改相关联。



A / B测试证实了我们的假设，测试版本在统计学上明显优于对照组。



具有统计意义的结果，请不要急于将其扩展到所有用户。不要冒险，并仔细检查收到的数据。



确保没有样本和异常的交集。



请勿使同一用户陷入两个样本，否则您将冒着接收到的数据可靠性的风险。



但是，如果两个版本之间没有差异，或者测试版本明显更差，该怎么办？





为我们的实验版本可能会丢失这一事实做好准备，这在频繁测试中是很正常的。



在实验结束时，回顾分析为什么您的假设丢失了。



深入数据，进行用户调查，尝试了解为什么测试版本无法按预期工作。与团队保持共同的知识库（Confluence），在这里概述实验的最初期望和后续结果。所有这些数据将依次帮助您进行以下实验。



我们都错了，这是正常的。



人们通常倾向于高估其理性和正确决策的能力。



对此有一个极好的格言：“错误是一种发展税。”



实验实施



经过积极的实验后，它的实现应继续进行（将控制版本替换为测试版本）。



没有进一步行动就进行实验是没有意义的。



经常发生的情况是，当我们获得具有统计意义的结果（我们的假设得到确认）时，我们将无法执行该结果，因为 最初，协调的所有阶段都没有完成。预先考虑实施过程中的所有可能问题。



而不是结论



不管您的测试是否成功，都应将每个实验视为学习的机会。



使用您在以下假设中所学的知识。



祝您实验顺利！
