ステップ1.プロジェクトをモジュールに分割する


コードの混乱を避けるため、プロジェクトをモジュールに分割する方が信頼性が高くなります。これを行うには、次のファイルを作成し、ウィジェットコードをそれらに転送します。

  • header.js- ツールバーウィジェット;
  • aside.js- リストウィジェット;
  • table.js-ウィジェットのデータテーブル。
  • form.js- フォームのウィジェット。
  • footer.js-「ソフトウェアは...」というテキストを持つ要素。

新しいファイルでは、ウィジェット構成の説明は変数で発生します...

const footer = {
    height: 30,
    template:"The software is provided by <a href='#'>webix.com</a>. All rights reserved (c)",
    css:"bottom-link"
}

作成されたファイルは、次の順序でindex.htmlファイルに含まれます。

<body>
    <script src="functions.js"></script>
    <script src="users_module.js"></script>
    <script src="products_module.js"></script>
    <script src="data.js"></script>
    <script src="header.js"></script>
    <script src="table.js"></script>
    <script src="form.js"></script>
    <script src="aside.js"></script>
    <script src="footer.js"></script>
    <script src="script.js"></script>
</body>

結果のモジュールは、script.jsファイルに結合されます。このファイルには、シンプルで簡潔なアプリケーション初期化コードが含まれています。

webix.ui({
    rows:[
        header,
        {
            cols:[
            aside, {view: "resizer"}, table, form
            ]
        },
        footer
    ]
});

ステップ2.タブとその切り替え


ページに十分な空き領域がない場合、またはアプリケーションのコンテンツをテーマ別に分離する必要がある場合は、タブを使用するのが合理的です。

タブ間の切り替えは、マルチビューコンポーネントによって実行されます。このコンポーネントを使用すると、アプリケーションに必要な数のタブを作成し、特定の時点で1つだけを表示できます。

aside.jsファイルにマルチビューを作成します。

const multi = {
    view: "multiview",
    cells:[
        { id:"dashboard", cols:[table, form ] },
        { id:"users", template:"Users" },
        { id:"products", template:"Products" }
    ]   
}

セル配列には、タブのコードが含まれています。各タブには、アクセスして表示できるように、IDを追加する必要があります。これで、マルチビューには3つのタブが含まれ、最初のタブが移動され、その前に作成されたテーブルとフォームが作成されます。

script.jsファイルで、TableウィジェットとFormウィジェットをMultiモジュールに置き換えます。

webix.ui({
    rows:[
        header,
        {
            cols:[
            aside, {view: "resizer"}, /*table, form*/multi
            ]
        },
        footer
    ]
});

タブの切り替えの役割は、リストウィジェットが果たします。要素をクリックして開くには、対応するタブが必要です。

便宜上、リストウィジェットの要素をマルチビュータブと同じIDに設定します。

const aside = {
    view: "list",
    id:"mylist",
    scroll:false,
    select:true,
    width:200,
    css:"list_color",
    data:[
        {value:"Dashboard", id:"dashboard"},
        {value:"Users", id:"users"},
        {value:"Products", id:"products"}
    ],
    on:{
        onAfterSelect:function(id){ 
            $$(id).show();
        }
    }
}

リスト ウィジェットの要素をクリックすると、onAfterSelectがトリガーされます。ハンドラー内では、選択した要素のIDを取得し、同じ名前のタブを表示して、IDでアクセスします。覚えているように、これらは同じです。

ステップ3.「ダッシュボード」タブ-テーブルの設定


この時点まで、プロジェクトでは、フィールドが自動的に生成されるテーブルを使用していました。列を削除したり、新しい列を追加したりする場合は、テーブル設定が使用されます。これを行うには、Datatableウィジェットで、autoConfig ：trueプロパティを各列の設定を含む列配列に置き換える必要があります。

const table = {
    view:"datatable", 
    id:"film_list",
    scroll:"y",
    select:true,
    url:"data/data.js",
    hover:"myhover",
    columns:[
        { id:"rank", header:"", width:50, css:"rank"},
        { id:"title", header:"Film title", fillspace:true},
        { id:"year",  header:"Released", width:100},
        { id:"votes", header:"Votes", width:100},
        { id:"rating", header:"Rating", width:100}
    ]
}

  • idプロパティ値は、この列に表示されるデータアイテムのフィールドを示します。
  • 要素のheaderプロパティは列ヘッダーです。
  • すべての列は固定幅に設定されますが、2番目の列はfillspaceプロパティを使用します。これにより、タイトル列がすべての空き領域を占めることができます。

表ではカスタムCSS設定が使用されています。線にホバーが設定され、最初の列の背景が変更されています。すべての設定は、style.cssファイルで事前定義されており、クラス名を挿入するだけです。

また、テーブルにさらに多くのデータが必要なため、urlプロパティを使用してポイントすることにより、data / data.jsからデータをロードします。

ステップ4.「ユーザー」タブ-リストと図の描画


2番目のタブでは、リストと図を作成します。これを行うには、2番目のマルチビュー要素で、モジュールの名前（「ユーザー」）を指定します。

const multi = {
    view: "multiview",
    cells:[
        { id:"dashboard", cols:[table, form ] },
        { id:"users", rows:[users] },
        { id:"products", template:"Products" }
    ]   
}

ウィジェット自体については、新しいusers_module.jsファイルを作成します。

リスト。リストはリストウィジェットとして理解され、以前はメニューの作成時に使用されていました。リストの行は、ユーザー名と国名で構成されている必要があります。

リストウィジェットコード：

const users = {
    rows:[
        {
            view: "list",
            id:"user_list",
            select:true,
            url:"data/users.js",
            template:"#name# from #country#"
        },
        {
             template:"Chart"
        }
    ]
}

rows配列は、ワークスペースを2つの部分に分割するために使用されます。2番目の部分はチャートに使用されます。

テンプレートプロパティの＃記号の間には、users.jsファイルから読み込まれたデータアイテムから値が取得されるフィールドが指定されています。

結果：



グラフ。ライブラリは、一般的なグラフの種類をサポートしています：線、円、レーダー、ドーナツ、縦棒など。これらはすべて、グラフウィジェットによって作成されます。チャートの外観はtypeプロパティを定義します。

設定template:”Chart”をウィジェットコードに置き換えます。

const users = {
    rows:[
        {
            view: "list",
            id:"user_list",
            select:true,
            url:"data/users.js",
            template:"#name# from #country#"
        },
        {
            view:"chart",
            type:"bar",
            value:"#age#",
            url:"data/users.js",
            xAxis:{
                template:"#age#",
                title:"Age"
            }
        }
    ]
}

  • この設定 type: "bar"は、グラフの線種を定義します。
  • フィールドの名前は値で渡され、必ず `＃...＃`で示されます。これはチャートウィジェットの機能です。
  • xAxis設定は、X軸に沿ってチャートの下に表示される情報を決定します。
  • テンプレートは、グラフの線の下に年齢の桁があることを示しています。
  • タイトルにはチャートのタイトルが含まれます"Age"。

ステップ5.「製品」タブ-ツリー表


このコンポーネントを初期化するために、products_module.jsファイルを作成し、マルチビューウィジェットの3番目の要素で、モジュール「products」の名前を示します。

const multi = {
    view: "multiview",
    cells:[
        { id:"dashboard", cols:[table, form ] },
        { id:"users", rows:[users] },
        { id:"products", rows:[products] }
    ]   
}

ツリーテーブルを作成するには、Treetableウィジェットを使用します。ウィジェットの列の1つには、必要なテンプレート{common.treetable（）}が含まれている必要があります。そうでない場合は、ツリービューではなく標準のテーブルが取得されます。テンプレートを使用すると、特徴的な要素をすぐに描画できます。

  • ネストされたレコードを折りたたむ/展開するアクティブなアイコン。
  • ファイル/フォルダアイコン;
  • エントリのレベルに応じたインデント。

const products = {
    view:"treetable",
    scrollX:false,
    columns:[
        { id:"id", header:"", width:50 },
        { id:"value", header:"Title", fillspace:true, 
           template:"{common.treetable()} #title#" },
        { id:"price", header:"Price", width:200 }
    ],
    select:"row",
    url:"data/products.js"
}

ツリーテーブルコンポーネントには、products.jsファイルから取得する階層データが入力されています。
