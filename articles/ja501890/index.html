React Native-写真とビデオをデバイスギャラリーに保存します

写真やビデオをandroid / iosデバイスに保存することは、多くのReact Native開発者にとって課題です。この記事では、URLを介してデバイスに写真を簡単かつ簡単に保存する方法を紹介します。

まず、2つのライブラリが必要です。

  1. @ react-native-community / cameraroll-ギャラリーへのアクセスを提供します（バージョン1.4.0でのみ成功しました。「より新しい」バージョンでよく知られているエラーがありました。別のバージョンで成功した場合は書き込んでください）
  2. rn-fetch-blob-ローカルアプリケーションファイルへのアクセス。このライブラリは、reactネイティブバージョン0.60以降で動作することに注意してください。


iOS


iOSでは、すべてが非常にシンプルである

ことがわかりました。1. Info.plistに追加します。
<key>NSCameraUsageDescription</key>
<string></string>
<key>NSPhotoLibraryAddUsageDescription</key>
<string></string>
<key>NSPhotoLibraryUsageDescription</key>
<string></string>

2.実装 
import CameraRoll from '@react-native-community/cameraroll';

const saveImageInDevice = async (url) => {
    await CameraRoll.saveToCameraRoll(url, 'photo');
}

それで全部です！saveImageInDevice関数は、リンクアドレスを入力として受け入れます。CameraRoll.saveToCameraRollはファイルをダウンロードし、デバイスギャラリーに保存します。saveToCameraRollメソッドの2番目のパラメーターも「ビデオ」にすることができます

アンドロイド


Androidデバイスはもう少し複雑です。実際のところ、Androidでは、saveToCameraRollメソッドは外部URLの操作方法を認識していません。最初にファイルを外部ストレージにアップロードし、次にギャラリーにアップロードする必要があります。

import {PermissionsAndroid} from 'react-native';
import CameraRoll from '@react-native-community/cameraroll';
import RNFetchBlob from 'rn-fetch-blob';

//      
const checkAndroidPermission = async () => {
    const permission = PermissionsAndroid.PERMISSIONS.WRITE_EXTERNAL_STORAGE;
    await PermissionsAndroid.request(permission);
};

const saveImageInDevice = async (url) => {
     await checkAndroidPermission();
        let res = await RNFetchBlob
          .config({
            fileCache : true,
            appendExt : 'jpg' 
          })
          .fetch('GET', url);
     url = res.path();
     await CameraRoll.saveToCameraRoll(url, 'photo');
}


GETリクエストによるRNFetchBlob.fetchは、ファイルを外部ストレージにアップロードし、このファイルのURLを渡します。次に、CameraRoll.saveToCameraRollがこのファイルをギャラリーにアップロードします。

それで全部です。ご清聴ありがとうございました！
