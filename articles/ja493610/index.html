DispmanX APIを使用してRaspberry Piでビデオレイヤーをプログラミングする


この記事では、Raspberry Single Board ComputerのDispmanX APIの使用方法を説明したいと思います。 DispmanX APIは、Raspberryデスクトップに新しいレンダリングレイヤーを作成する機能を提供します。レイヤーはレイヤーの上にぶら下がっています。これらは動的に作成、削除、移動でき、スケーリングできます。同時に、ビデオコントローラー自体がそれらを組み合わせてモニター画面に表示します。興味深いことに、レイヤーにはアルファチャネルを設定でき、すべてのレイヤーの画像が混合されます。また、32の2ビットARGBレイヤーに加えて、たとえばYUV420レイヤーや他のタイプのレイヤーを作成できます。ラズベリーには、デフォルトですでに2つのレイヤーがあります。一番下の画像にはデスクトップ画像が含まれています。 Xを介したすべての出力は、このレイヤーに送られます。また、マウスカーソルの画像が存在する2番目の最上位のレイヤーがあります。

新しいレイヤーの作成方法、レイヤーへの画像の書き込み方法、画面上での移動方法を示します。実際、上のデモビデオはそのようなプログラムの動作を示しています。 4つの新しい32ビットARGBレイヤーがここで作成されます。各レイヤーで、準備済みのビットマップからピクセルを書き込みます。私のビットマップは雲、太陽、風船の画像です。レイヤーはさまざまな速度で画面内を移動し、一番下のXレイヤーの上にぶら下がっています。
したがって、DispmanX API。これは、Raspberryとそのビデオコントローラーに固有の比較的低レベルなものです。そして、これは「非標準」のことです。一般に、LinuxカーネルとLinuxグラフィックサブシステムには、DRM、Direct Rendering Managerを介してビデオレイヤーをプログラミングする場所がありますが、何らかの理由でRaspberryの作成者は自分の自転車を作成することにしました。一方、これは複雑な自転車ではありませんが、乗ることはかなり可能です。 DispmanXは、Raspberry Pi / Pi2 / Pi3およびPi4のPi-Zeroで実行されます。一般的に、すべてのラズベリー。実際には、Pi4にはすでに通常のOpenGLESv3が搭載されています。ここではそのようなタンバリンはもはや必要ないかもしれません。ただし、その一方で、DispmanXはOpenGLESよりもはるかに単純です（機能は少ないですが）。

このDispmanX APIを使用してプログラムを作成するには、ヘッダーファイル/opt/vc/include/bcm_host.hをインクルードする必要があります。さらに、/ opt / vc / libフォルダーにあるlibbcm_host.soライブラリにプログラムをリンクする必要もあります。

必要なAPIのすべての関数は、vc_dispmanx_ * ...で始まり

ます。新しいレイヤーを作成するために最初に行うことは、次の関数のペアを使用してディスプレイにアクセスすることです。

bcm_host_init();
DISPMANX_DISPLAY_HANDLE_T display = vc_dispmanx_display_open( 0 );

これで、レイヤーの画像を含む「リソース」を作成できます。

VC_IMAGE_TYPE_T type = VC_IMAGE_ARGB8888;
uint32_t UnusedImagePtr;
int SrcImageWidth = 512; //image must be 32 bytes aligned size
int SrcImageWidth = 196;
DISPMANX_RESOURCE_HANDLE_T resource = vc_dispmanx_resource_create( type, SrcImageWidth, SrcImageHeight,  &UnusedImagePtr );

ここでは、レイヤータイプは32ビットで、アルファチャネルが付いています。しかし、私がすでに書いたように、YUVでさえ他のタイプがあるかもしれません。YUVレイヤーの使用は、ビデオの再生時などの動的レイヤーに意味があります。次に、画面に書き込まれるデータの量が大幅に削減され、YUVをRGBにトランスコードする必要がないため、貴重なプロセッサクロックサイクルを節約できます。

新しいリソースを作成した後、前に取得した表示ハンドラーを使用して、新しい要素レイヤーを表示に追加できます。

VC_DISPMANX_ALPHA_T alpha;
	alpha.flags =
		(DISPMANX_FLAGS_ALPHA_T)(DISPMANX_FLAGS_ALPHA_FROM_SOURCE | DISPMANX_FLAGS_ALPHA_MIX);
	alpha.opacity = 255;
	alpha.mask = 0;
int OutLayer = 200;
DISPMANX_ELEMENT_HANDLE_T vc_element = vc_dispmanx_element_add(
		update,
		display,
		OutLayer,
		&dst_rect, resource, &src_rect, DISPMANX_PROTECTION_NONE, &alpha, NULL, DISPMANX_NO_ROTATE );

別の非常に重要な更新パラメーターがあります。表示の内容を変更したり、レイヤーを追加または削除したり、レイヤーを移動したり、レイヤーに新しい画像を書き込んだりする必要があるたびに、変更の開始と変更の終了をマークする必要があります。

ディスプレイ上の要素の変更を開始するには、次のことを行う必要があります。

DISPMANX_UPDATE_HANDLE_T update =  vc_dispmanx_update_start( Priority );

次に、必要なすべてを変更し、レイヤーを作成または移動し、レイヤーに新しいピクセルを書き込み、関数を使用して変更を閉じます。

vc_dispmanx_update_submit_sync( update );

関数の名前から理解できるように、変更はビデオスキャンの次のフレームパルスの後に有効になります。

レイヤーへのピクセルの書き込みは、次の関数によって実行されます。

vc_dispmanx_resource_write_data(
		resource,
		type,
		pitch,
		Pixels, //pointer to ARGB pixels
		&rect );

残念ながら、rectパラメータがあるにもかかわらず、画像内の任意のフラグメントを更新することはできません。更新できるのは「ストリップ」だけです。つまり、長方形の上部と下部を指定できますが、左は常に0で、右は常にイメージの幅になります。

奇妙なことに、これはおそらく、レイヤーの操作に必要な最小限の知識です。

私のプログラムでは、DispmanX APIの小さなラッパーを作成しました。これにより、各レイヤーを個別の構造で説明できます。

struct DISPMANX_ELEMENT {
    DISPMANX_RESOURCE_HANDLE_T  res_;
    DISPMANX_ELEMENT_HANDLE_T   vc_element_;
    VC_IMAGE_TYPE_T type_;
    uint32_t src_width_;
    uint32_t src_height_;
    uint32_t dst_layer_;
    uint32_t dst_width_;
    uint32_t dst_height_;
    int32_t  dst_x_;
    int32_t  dst_y_;
};

まあ、私のラッパー関数の多くは、そのような構造体へのポインターをパラメーターとして受け取ります。 

void dispmanx_init();
void dispmanx_element_init(struct DISPMANX_ELEMENT* Element);
struct DISPMANX_ELEMENT dispmanx_element_create( VC_IMAGE_TYPE_T type, int SrcW, int SrcH, int OutX, int OutY, int OutW, int OutH, int OutLayer );
void dispmanx_element_delete(struct DISPMANX_ELEMENT* Element);
void dispmanx_element_write(struct DISPMANX_ELEMENT* Element, char* Pixels);
void dispmanx_element_move( DISPMANX_UPDATE_HANDLE_T update, struct DISPMANX_ELEMENT* Element, int32_t NewX, int32_t NewY );
DISPMANX_UPDATE_HANDLE_T dispmanx_start_update( int Priority );
void dispmanx_sync( DISPMANX_UPDATE_HANDLE_T Update );

このようにして、いくつかのレイヤーを作成し、それらを簡単に操作できます。

ビットマップを読み取るための関数も書きました。さらに、関数はトリッキーです-純粋な緑のピクセル0x00FF00がビットマップで見つかった場合、それを画像の透明ピクセルと見なし、それに応じてこのピクセルのアルファバイトをゼロに設定します。

プログラムは4つのレイヤーを作成します。最初の2つのレイヤーはクラウドの同じビットマップから作成されますが、レイヤーのサイズを変更し、スケーリングを使用しています。2番目のクラウドは最初のクラウドよりも大きくなっています。3番目のレイヤーは風船で、4番目のレイヤーでは太陽を読み込みます。

typedef struct OBJ{
	int width_;
	int height_;
	int x_;
	int y_;
	int layer_;
	int speed_;
	char* pixels_;
} OBJ_;

int main(int argc , char *argv[])
{
	cout << "Hello Raspberry DispmanX API!\n";
	dispmanx_init();

	OBJ_ cloud1;
	cloud1.pixels_ = LoadBitmap( (char*)"clouds.bmp", &cloud1.width_, &cloud1.height_ );
	cloud1.layer_ = 100;
	cloud1.x_ = 100;
	cloud1.y_ = 120;
	cloud1.speed_ = 3;

	struct DISPMANX_ELEMENT cloud1_element = dispmanx_element_create(
		VC_IMAGE_ARGB8888, cloud1.width_, cloud1.height_, cloud1.x_, cloud1.y_, cloud1.width_, cloud1.height_, cloud1.layer_ );
	dispmanx_element_write( &cloud1_element, cloud1.pixels_ );

	OBJ_ cloud2;
	cloud2.pixels_ = LoadBitmap( (char*)"clouds.bmp", &cloud2.width_, &cloud2.height_ );
	cloud2.layer_ = 101;
	cloud2.x_ = 10;
	cloud2.y_ = 230;
	cloud2.speed_ = 2;

	struct DISPMANX_ELEMENT cloud2_element = dispmanx_element_create(
		VC_IMAGE_ARGB8888, cloud2.width_, cloud2.height_, cloud2.x_, cloud2.y_, cloud2.width_*1.3, cloud2.height_*1.4, cloud2.layer_ );
	dispmanx_element_write( &cloud2_element, cloud2.pixels_ );

	OBJ_ balls;
	balls.pixels_ = LoadBitmap( (char*)"balls.bmp", &balls.width_, &balls.height_ );
	balls.layer_ = 102;
	balls.x_ = -100;
	balls.y_ = 351;
	balls.speed_ = 5;

	struct DISPMANX_ELEMENT balls_element = dispmanx_element_create(
		VC_IMAGE_ARGB8888, balls.width_, balls.height_, balls.x_, balls.y_, balls.width_, balls.height_, balls.layer_ );
	dispmanx_element_write( &balls_element, balls.pixels_ );

	OBJ_ sun;
	sun.pixels_ = LoadBitmap( (char*)"sun.bmp", &sun.width_, &sun.height_ );

永久サイクルでは、レイヤーを画面上でさまざまな速度で移動します。


while(1)
	{
		this_thread::sleep_for( chrono::milliseconds(20) );
		cloud1.x_ += cloud1.speed_;
		if( cloud1.x_>= 1920 )
			cloud1.x_ = 10 - cloud1.width_;

		cloud2.x_ += cloud2.speed_;
		if( cloud2.x_>= 1920 )
			cloud2.x_ = 133 - cloud2.width_;

		balls.x_ += balls.speed_;
		if( balls.x_>= 1920 )
			balls.x_ = 200 - balls.width_;

		sun.x_ += sun.speed_;
		if( sun.x_>= 1920 )
			sun.x_ = 250 - sun.width_;

		DISPMANX_UPDATE_HANDLE_T update = dispmanx_start_update(10);
		dispmanx_element_move( update, &cloud1_element, cloud1.x_, cloud1.y_ );
		dispmanx_element_move( update, &cloud2_element, cloud2.x_, cloud2.y_ );
		dispmanx_element_move( update, &balls_element, balls.x_, balls.y_ );
		dispmanx_element_move( update, &sun_element,   sun.x_,   sun.y_ );
		dispmanx_sync( update );
	}
