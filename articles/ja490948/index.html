クイックソートについて、複雑さ2 * N

複雑さを2 * N（Nの2倍）として定義するクイックソートアルゴリズムが最近開発されました。

この記事では、並べ替えアルゴリズム自体は分析しませんが、一連のテストを実行した後で、それをVisual Studioで使用される並べ替え（std :: sort関数）と比較します。彼らが言うように：事実と最小限の感情のみ。

テキスト情報を並べ替え、単語の配列を考慮します。この配列は、文学作品に基づいて形成されます。これを行うには、これらの作品のすべての単語をこの配列に並べます。単語分割は近代化なしで発生し、新しい単語の始まりの兆候はギャップです。配列内の単語は、テキスト内の単語に完全に対応しています。句読点（ピリオド、コンマなど）と制御文字はスペースなしで単語の後ろに書き込まれるため、結果としてそれらは単語の一部になります。別々の句読点と制御文字は別々の単語として扱われます。ファイルには、これらのファイルが配置されているサイトのアドレスが含まれています。アドレスデータは1ワードです。最長の単語は95文字で構成されます。検討中の作品にはラテンアルファベットの文字があり、キリル文字、句読点、制御文字。

並べ替えられた情報は任意であり、順序付けされていないため、特別な並べ替えは適用されません。

2つの並べ替え方法が比較されます。2* Nの複雑さで新しく開発された並べ替えアルゴリズムとVisual Studio 2017で使用されるアルゴリズム（std :: sort関数）。彼らは、ソート時間と、各テストでこのメモリに使用される各メソッドの量に関心があります。

プログラムは、環境-Visual Studio 2017、プラットフォーム-Windows x64で記述されています。テストは2つの構成で実行されました。デバッグ構成とリリース構成のテスト結果を表1と表2にそれぞれ示します。次の表は、2つのソート方法をテストした結果を示しています。

計算時間の結果は変動するため（値は10％以内で変動）、テストごとに一連の計算が実行され、そこから最小値が選択され、これらの値がテーブルに入れられました。列4および6に示されている時間は、ソート時間のみです。追加のプロセス（ファイルの読み取りと書き込み、配列の形成など）は、指定された時間には含まれません。
