今日のレッスンでは、VLAN設定に専念します。つまり、前のレッスンで説明したすべてのことを実行しようとします。次に、3つの質問を見ていきます。VLANの作成、VLANポートの割り当て、VLANデータベースの表示です。

Cisco Packerトレーサープログラムウィンドウを開いて、自分のネットワークの論理トポロジを描画します。



最初のスイッチSW0は、IPアドレスの範囲が192.168.10.0/24のVLAN10ネットワークに接続された2台のコンピューターPC0とPC1に接続されています。したがって、これらのコンピューターのIPアドレスは192.168.10.1と192.168.10.2になります。通常、人々はVLAN番号をIPアドレスの3番目のオクテットで識別します。私たちの場合は10ですが、これはネットワークを指定するための前提条件ではありません。任意のVLAN識別子を割り当てることができますが、この手順はネットワーク構成を容易にするため、大企業で採用されています。

次はスイッチSW1で、これは2つのラップトップLaptop1とLaptop2でIPアドレス192.168.20.0/24のVLAN20ネットワークに接続されています。

VLAN10は会社のオフィスの1階にあり、販売管理のネットワークです。VLAN20に属するマーケティング担当者のラップトップ0ラップトップは、同じスイッチSW0に接続されています。このネットワークは、他の従業員がいる2階まで拡張されており、別の建物または同じオフィスの3階にある販売部門に接続されています。さらに3台のコンピューター（PC2.3と4）がインストールされています。これらはVLAN10ネットワークの一部です。

VLAN20と同様に、VLAN10は、従業員が別のフロアにいるか別の建物にいるかに関係なく、すべての従業員に中断のない通信を提供する必要があります。これが今日検討するネットワークの概念です。

それをセットアップして、PC0から始めましょう。アイコンをクリックして、コンピューターのネットワーク設定を入力し、IPアドレス192.168.10.1とサブネットマスク255.255.255.0を入力します。デフォルトゲートウェイアドレスは入力しません。これは、あるローカルネットワークから別のネットワークに終了する必要があるためです。この場合、OSIレベル3の設定は扱いません。レベル2のみに関心があり、別のネットワークへのトラフィックルーティングは考慮しません。通信網。



イントラネットと、その一部であるホストのみを構成します。次に、PC2に移動し、最初のPCと同じことを行います。 PC0からPC1にpingできるかどうかを確認します。ご覧のとおり、pingは成功し、IPアドレスが192.168.10.2のコンピューターは自信を持ってパケットを返します。このようにして、スイッチを介したPC0とPC1間の通信の確立に成功しました。

成功した理由を理解するには、スイッチの設定に移動し、VLANテーブルを確認します。



技術的には、このスイッチには5つのVLANがあります。デフォルトではVLAN1、1002、1003、1004、1005です。最後の4つのネットワークを見ると、それらがサポートされておらず、サポートされていないことが示されています。これらは、fddi、fddinet、trnetなどの古いテクノロジーの仮想ネットワークです。現在、使用されていませんが、技術的な要件により、新しいデバイスに含まれています。したがって、実際には、スイッチにはデフォルトで1つの仮想ネットワーク（VLAN1）しかありません。そのため、箱から出してすぐにCiscoスイッチのすべてのポートがこのネットワークで構成されます。これらは24個のファストイーサネットポートと2個のギガビットイーサネットポートです。これにより、デフォルトですべて同じVLAN1の一部となっているため、新しいスイッチの互換性が大幅に向上します。

VLAN1で動作するようにデフォルトで設定されているポートを再割り当てして、VLAN10で動作するようにする必要があります。 Packet Tracerは、私たちの場合、これらがFa0およびFa0 / 2ポートであることを示しています。



スイッチSW0に戻り、これらの2つのポートを構成します。これを行うには、configure terminalコマンドを使用してグローバルコンフィギュレーションモードを開始し、このインターフェイスのコンフィギュレーションコマンドであるint fastEthernet 0/1を入力します。このポートはアクセスポートであるため、アクセスモードを設定する必要があります。また、switchport mode accessコマンドを使用します。



このポートは静的アクセスポートとして構成されていますが、別のスイッチを接続すると、DTPプロトコルを使用して動的トランクモードになります。デフォルトでは、このポートはVLAN1に属しているため、switchport access vlan 10コマンドを使用する必要があります。この場合、VLAN10は存在せず、作成する必要があるというメッセージが表示されます。覚えていれば、VLANデータベースにはVLAN1という1つのネットワークしかなく、VLAN10ネットワークはありません。しかし、スイッチにVLAN10へのアクセスを提供するように要求したため、エラーメッセージを受け取りました。

したがって、VLAN10を作成し、このアクセスポートを割り当てる必要があります。その後、VLANデータベースにアクセスすると、新しく作成されたVLAN0010が表示されます。これはアクティブ状態であり、Fa0 / 1ポートが属しています。



コンピュータには何の変更も加えず、コンピュータが接続されているスイッチのポートを構成しただけです。次に、数分前に成功したIPアドレス192.168.10.2にpingを送信してみましょう。 PC0が接続されているポートはVLAN10に属し、PC1に接続されているポートは引き続きVLAN1に属しており、2つのネットワーク間の接続がないため、成功しませんでした。これらのコンピューター間の通信を確立するには、両方のポートをVLAN10で動作するように構成する必要があります。私は再びグローバル構成モードに入り、スイッチポートf0 / 2に対して同じことを行います。



VLANテーブルをもう一度見てみましょう。これで、VLAN10がFa0 / 1およびFa0 / 2ポートで構成されていることがわかります。ご覧のとおり、デバイスが接続されているSW0スイッチの両方のポートが同じネットワークに属しているため、pingは成功しています。目的を示すためにネットワークの名前を変更してみましょう。VLANを変更する場合は、このネットワークの構成に入る必要があります。

これを行うには、vlan 10と入力します。コマンドプロンプトがSwitch（config）＃からSwitch（config-vlan）＃に変更されています。疑問符を入力すると、システムは3つの可能なコマンド（exit、name、no）のみを表示します。nameコマンドを使用してネットワーク名を割り当てるか、noと入力してコマンドをデフォルトの状態に戻すか、exitコマンドを使用して変更を保存できます。したがって、SALESという名前を入力してコマンドを終了します。



VLANデータベースを見ると、私たちのチームが実行され、以前のVLAN10がSALES-sales departmentと呼ばれていることを確認できます。そこで、私たちはオフィスの2台のコンピュータを、作成した営業部門のネットワークに接続しました。次に、マーケティング部門のネットワークを作成する必要があります。 Laptop0ラップトップをこのネットワークに接続するには、ネットワーク設定を入力し、IPアドレス192.168.20.1とサブネットマスク255.255.255.0を入力する必要があります。デフォルトゲートウェイは必要ありません。次に、スイッチ設定に戻り、int fa0 / 3コマンドでポート設定を入力し、switchport mode accessコマンドを入力する必要があります。次のコマンドは、switchport access vlan 20です。

そのようなVLANは存在せず、作成する必要があるというメッセージを再度受け取ります。他の方法でもかまいません。スイッチポートの構成（config-if）を終了し、スイッチ（config）に移動してvlan 20コマンドを入力すると、VLAN20ネットワークが作成されます。つまり、最初にVLAN20ネットワークを作成し、それにMARKETINGという名前を付け、exitコマンドで変更を保存してから、ポートを構成できます。

sh vlanコマンドでVLANデータベースを入力すると、作成したMARKETINGネットワ​​ークと対応するFa0 / 3ポートが表示されます。このラップトップからコンピューターにpingできないのは、2つの理由があります。VLANが異なり、デバイスが異なるサブネットに属しているためです。それらは異なるVLANに属しているため、VLAN20に属するポートがないため、スイッチは別のネットワークに向けられたラップトップパケットを破棄します。

先ほど申し上げたように、会社は拡大しており、1階には小さなオフィスが足りないため、建物の2階にマーケティング部門を配置し、そこに2人の従業員用のコンピューターを設置し、1階のマーケティング部門と連絡を取りたいと考えています。これを行うには、最初に2つのスイッチ間にトランクを作成する必要があります。最初のスイッチのポートFa0 / 4と2番目のスイッチのポートFa0 / 1です。これを行うには、SW0の設定を入力し、コマンドint f0 / 4とスイッチポートモードトランクを入力します。

switchport trunk encapsulation encapsulationコマンドがありますが、デフォルトでは802.1qカプセル化テクノロジーを使用するため、新しいスイッチでは使用されません。ただし、すべてのスイッチが.1Qプロトコルを理解するようになったため、以前のモデルのCiscoスイッチは、使用されなくなった独自のISLプロトコルを使用していました。したがって、switchport trunk encコマンドを使用する必要はありません。

ここでVLANデータベースに入ると、Fa0 / 4ポートがそこから消えていることがわかります。これは、VLANに固有のアクセスポートのみがこの表にリストされているためです。スイッチのトランクポートを表示するには、sh int trunkコマンドを使用する必要があります。



コマンドウィンドウで、Fa0 / 4ポートがオンになり、802.1qプロトコルを使用してカプセル化され、ネイティブVLAN 1に属していることがわかります。ご存知のように、このトランクポートがタグなしトラフィックを受け入れると、ネイティブVLAN 1ネットワークに自動的にルーティングされます。次のレッスンでは、ネイティブVLANの構成について説明します。ここでは、このデバイスのトランク設定がどのように見えるかを覚えておきます。

次に、2番目のスイッチSW1に移動して、設定モードint f0 / 1に入り、前のケースと同じ方法でポート設定のシーケンスを繰り返します。マーケティング部門のラップトップが接続されている2つのポートFa0 / 2およびFa0 / 3は、アクセスモード用に構成され、VLAN20ネットワークに割り当てられている必要があります。

前のケースでは、各スイッチポートを個別に構成しました。次に、コマンドラインテンプレートを使用してこのプロセスを高速化する方法を説明します。コマンドを入力して、インターフェイスの範囲をint range f0 / 2-3に設定できます。その結果、コマンドラインリクエストはSwitch（config-if-range）＃という形式になり、同じパラメータを入力するか、同じコマンドを指定されたポート範囲。たとえば、同時に20ポート。

前の例では、複数のスイッチポートに対して同じswitchport mode accessおよびswitchport access vlan 10コマンドを数回使用しました。これらのコマンドは、ポートの範囲を使用する場合に1回入力できます。次に、選択したポート範囲に対してswitchport mode accessおよびswitchport access vlan 20コマンドを入力します。



VLAN20はまだ存在しないため、システムによって自動的に作成されます。 exitと入力して変更を保存し、VLANテーブルを表示するように要求します。ご覧のように、ポートFa0 / 2とFa0 / 3は新しく作成されたVLAN20の一部です。

次に、オフィスの2階にあるラップトップのIPアドレスを構成します。Laptop1はアドレス192.168.20.2とサブネットマスク255.255.255.0を受け取り、Laptop2はIPアドレス192.168.20.3を受け取ります。最初のラップトップを2番目のラップトップからpingして、ネットワークパフォーマンスを確認します。ご覧のとおり、両方のデバイスが同じVLANの一部であり、同じスイッチに接続されているため、pingは成功しています。



ただし、1階と2階のマーケティング部門のラップトップは、同じVLAN上にありますが、異なるスイッチに接続されています。それらの間の接続がどのように確保されているかを確認してみましょう。これは、1階にあるラップトップ2のラップトップにIPアドレス192.168.20.1でpingを送信しています。ご覧のとおり、ラップトップがさまざまなスイッチに接続されているにもかかわらず、すべてが問題なく動作します。この接続は、両方のスイッチがトランクによって接続されているためです。

Laptop2とPC0間の接続を確立できますか？いいえ、できません。これらは異なるVLANに属しているためです。次に、PC2、3、4コンピューターのネットワークを構成します。このネットワークでは、最初に2番目のスイッチFa0 / 4と3番目のスイッチFa0 / 1の間にトランクを作成します。

SW1の設定に移動し、config tコマンドを入力してから、int f0 / 4を呼び出してから、スイッチポートモードのトランクに入り、コマンドを終了します。同様に、3番目のスイッチSW2を構成します。トランクを作成しました。設定が有効になると、ポートの色がオレンジから緑に変わっています。次に、VLAN10ネットワークに属する営業部門のコンピューターが接続されているFa0 / 2.0 / 3.0 / 4ポートを構成する必要があります。これを行うには、SW2スイッチの設定に移動し、ポート範囲f0 / 2-4を選択して、switchport mode accessおよびswitchport access vlan 10コマンドをそれらに適用します。これらのポートにはVLAN10ネットワークがないため、システムによって自動的に作成されます。このスイッチのVLANデータベースを見ると、Fa0 / 2.0 / 3.0 / 4ポートがVLAN10ネットワークに属していることがわかります。

その後、IPアドレスとサブネットマスクを入力して、これら3台のコンピューターそれぞれのネットワークを構成する必要があります。PC2はアドレ​​ス192.168.10.3を受け取り、PC3はアドレス192.168.10.4を受け取り、PC4はIPアドレス192.168.10.5を受け取ります。



ネットワークが機能しているかどうかの質問に答えるために、3階または別の建物にあるPC4コンピューターから1階のPC0コンピューターにpingを送信します。pingが失敗したので、これを実行できなかった理由を調べてみましょう。

Laptop2からLaptop0をpingしようとしたとき、ラップトップが異なるスイッチに接続されているにもかかわらず、すべてが正常に機能しました。なぜ今、営業部門のコンピュータがトランクで接続された別のスイッチにも接続されていると、pingが失敗するのですか？問題の原因を理解するには、スイッチの動作を覚えておく必要があります。

PC4からスイッチSW2にパケットを送信すると、パケットがポートFa0 / 4に到着していることがわかります。スイッチはデータベースをチェックし、Fa0 / 4ポートがVLAN10ネットワークに属していることを検出します。その後、スイッチはフレームにネットワーク番号のタグを付けます。つまり、VLAN10ヘッダーをトラフィックパケットに付加し、トランクを介して2番目のスイッチSW1に送信します。このスイッチはヘッダーを「読み取り」、パケットがVLAN10用であることを確認し、そのVLANデータベースを調べ、そこにVLAN10がないことを確認して、パケットを破棄します。したがって、PC2.3と4のデバイスは問題なく相互に通信できますが、スイッチSW1はVLAN10について何も認識していないため、コンピューターPC0およびPC1との通信を確立しようとしても失敗します。

SW1の設定に移動し、vlan 10コマンドを使用してVLAN10ネットワークを作成し、その名前にMARKETINGと入力することで、この問題を簡単に修正できます。もう一度pingを実行してみましょう。最初の3つのパケットが破棄され、4番目のパケットが正常に通過します。これは、スイッチが最初にIPアドレスをチェックしてMACアドレスを決定したため、一定の時間がかかったため、最初の3つのパケットがタイムアウトによってドロップされたためです。スイッチがMACアドレスのテーブルを補足し、パケットを目的のアドレスに直接送信するため、接続が確立されます。
私が問題を修正するために行ったのは、中間スイッチの設定に行って、そこでVLAN10ネットワークを作成することだけでした。したがって、ネットワークがスイッチに直接接続されていなくても、ネットワーク接続に関係するすべてのネットワークを認識している必要があります。ただし、ネットワークに数百のスイッチがある場合、物理的に各設定に入り、手動でVLAN識別子を構成することはできません。そのため、VTPプロトコルを使用します。その構成については、次のビデオチュートリアルで説明します。

そのため、今日は、VLANを作成する方法、VLANポートを割り当てる方法、およびVLANデータベースを表示する方法を検討しました。ネットワークを作成するには、スイッチのグローバル構成モードに入り、vlan <number>コマンドを使用します。また、name <name>コマンドを使用して、作成したネットワークの名前を割り当てることもできます。



インターフェイスモードに入り、switchport access vlan <number>コマンドを使用して、別の方法でVLANを作成することもできます。この番号のネットワークがない場合は、システムによって自動的に作成されます。初期設定を変更した後は、必ずexitコマンドを使用してください。変更しないと、VLANデータベースに保存されません。次に、適切なコマンドを使用して、ポートを特定のVLANに割り当てることができます。



switchport mode accessコマンドは、インターフェイスをスタティックアクセスポートモードにします。その後、switchport access vlan <number>コマンドを使用して、対応するVLAN番号がポートに割り当てられます。 VLANデータベースを表示するには、show vlanコマンドを使用します。このコマンドは、EXECユーザーモードで入力する必要があります。トランクポートのリストを表示するには、show int trunkコマンドを使用します。
