サーバーHTTPヘッダー検証サービス

どのサイトでも、HTTPヘッダーを適切に構成することが重要です。見出しをテーマに多くの記事が書かれています。ここでは、学んだ教訓、RFC文書を要約しました。一部の見出しは必須であり、いくつかは時代遅れであり、いくつかは混乱し、矛盾する可能性があります。WebサーバーのHTTPヘッダーを自動的にチェックする解析を行いました。ヘッダーのみを表示する他の多くのサービスとは異なり、このサービスでは次のことができます。

  1. 一般的なヘッダーの値を設定します。
  2. 独自のカスタムヘッダーを追加します。
  3. HTTPプロトコルのバージョンを示します：1.0、1.1、2（HTTP / 2がサポートされているかどうかをチェックします）;
  4. サーバーに送信する要求メソッド、タイムアウト、およびpostdataデータを指定します。
  5. puzomerkaは、サーバー応答にLast-ModifiedまたはETagが含まれている場合、If-Modified-Since、If-None-Match要求への応答が正しいかどうかもチェックします。

私たちは究極の真実であると主張しません。もちろん、個々のコンテンツや個々のプロジェクトについては、偏差がある場合があります。しかし、このサービスはあなたが何に注意を払うべきかを確実に教えてくれます、それはあなたがヘッダーを編集するのに役立つかもしれません。以下は、検証サービスが注意する点のリストです。なぜそうなのか、Habreに関する記事を読んでください。

必須ヘッダー


  • 日付
  • テキストコンテンツの文字セットを指定するContent-Type、できればutf-8
  • テキストコンテンツのコンテンツエンコーディング圧縮

古くて不要なヘッダー


  • Webサーバーの詳細バージョンを備えたサーバー
  • X-Power-by
  • X_ASPNET-Version
  • 期限切れ
  • プラグマ
  • P3p
  • 経由
  • X-UA互換

セキュリティに必要なヘッダー


  • X-Content-Type-Options
  • X-XSS保護
  • 厳格な輸送セキュリティ
  • リファラーポリシー
  • 機能ポリシー
  • Content-Security-PolicyまたはContent-Security-Policy-Report-Onlyは、インラインスクリプトとスタイルを無効にします。

ヘッダーのキャッシュ


キャッシュ期間が長い静的コンテンツに必要で、キャッシュ期間が短い動的コンテンツには非常に望ましい。

  • 最終更新日
  • ETag
  • キャッシュ制御
  • 変化する
  • サーバーがヘッダーに正しく応答することが重要です：If-Modified-SinceおよびIf-None-Match

HTTP / 2


サーバーはHTTP / 2をサポートする必要があります。デフォルトでは、サービスはサーバーのHTTP / 2をチェックします。サーバーがHTTP / 2をサポートしていない場合は、HTTP / 1.1を選択します。
