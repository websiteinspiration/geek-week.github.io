束の構成


メモリはビジー（割り当て済み）で解放されている場合があります。図は動的メモリを示しています。
  • SSize-メモリが空いている場合、前のメモリブロックのサイズ。
  • サイズ-指定されたメモリブロックのサイズ。2つのステータスビットが追加されます。
  • データ-ユーザーデータ。
  • Fdは、次の空きブロックへのポインタです。
  • Bk-前の空きブロックへのポインター。
  • 無料-メモリを解放します。



したがって、2つの空きブロックを隣接させることはできません。さらに、占有および空きシステムメモリの境界に、特別に処理された空きWブロックがあります。



リスト（バスケット）内のブロックの表現は次のとおりです。



unlinkメソッドは、リストから空きブロックを削除するために使用されます。
void unlink(S, BK, FD){
BK = S->bk;
FD = S->fd;
FD->bk=BK;
FD->fd=FD;
}

メモリの割り当てと解放


mmapの動作を見てみましょう。最初のステップで、必要なサイズの配列（24バイトなど）がチェックされます。必要なブロックがある場合は、リンク解除を使用して分離されます。



2番目のステップでは、このブロックが十分に大きい場合、2つの部分に分割されます。最初の部分が割り当てられ、2番目の部分は別のアレイに再配布されます。



3番目のステップでは、必要なサイズのブロックが割り当てられていない場合、Wブロックがチェックされます。彼が満足した場合、ステップ2が彼と一緒に行われます。Wブロックが小さい場合は、sbrk（）とmmap（）を使用して、使用可能なメモリを拡張します。Freeメソッドはmmapとは完全に逆です。

ヒープバッファオーバーフロー


ヒープオーバーフローは、ヒープデータ領域で発生する一種のバッファオーバーフローです。ヒープ上のメモリは、実行時にアプリケーションによって動的に割り当てられ、通常はプログラムデータが含まれます。悪用は、このデータを特別な方法で破壊することによって実行され、アプリケーションがリンクリストへのポインタなどの内部構造を上書きします。正規のヒープオーバーフローメソッドは、動的メモリ割り当て関係（mallocメタデータなど）を上書きし、ポインター交換を使用してプログラム関数へのポインターを書き換えます。

例として、Unlink関数では、FD-> bkを使用して、メモリ内の任意のワードの値を変更できます。たとえば、GOTアドレスを変更するシェルコードを配置します。オーバーフローの例がその例です。
