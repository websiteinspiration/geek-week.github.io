ECMAScript 2019（ES10）の機能


ES10標準には、多くの新機能があります。ここではそれらの一部のみを検討します。つまり、いくつかの新しい配列メソッドです。

▍Array.prototype.flatメソッド


この方法を使用すると、他の配列を含む配列を作成し、より「フラット」にし、特定の深度レベルに「圧縮」できます。

const array = [1, 2, [3, 4]];
array.flat(); // [1, 2, 3, 4];

これは、特にネストされた配列を操作する必要がある場合に、非常に便利な機能です。データ構造内の配列のネストの深さが1を超える場合、メソッドを1回呼び出すflatだけで配列を完全に「フラット」にすることはできません。このメソッドはdepth、処理される配列の次元を削減する必要があるネストレベルの数を指定できるオプションのパラメーターを受け入れます。

//  
const crazyArray = [1, 2, [3, 4], [[5], [6, [7,8]]]];
crazyArray.flat(Infinity); // [1, 2, 3, 4, 5, 6, 7, 8];
// ,  flat,

アレイが深いほど、それを処理するためにより多くのコンピューティングリソースが必要になります。IEとEdgeはこの機能をサポートしていないことに注意してください。

▍メソッドArray.prototype.flatMap


このメソッドはまず、渡された関数を使用して配列の要素を処理し、次に配列をフラット構造に変換します。

const arr = ["it's Sunny in", "", "California"];
arr.flatMap(x => x.split(" "));
// ["it's","Sunny","in", "", "California"]

差flat及びflatMap方法があるという事実にあるflatMap元の配列の要素を変換する独自の機能を通過することができます。これに加えて、配列の要素を1レベルだけ「上げる」flatMapとは異なりflatます。このメソッドは新しい配列を返します。これは、特定の配列を「フラット」にする前に、何らかの方法でその要素を処理する必要がある場合に役立ちます。

aコードの最上位でawaitを使用する


awaitコードのトップレベルで キーワードを使用すると、モジュールを大きな非同期関数として機能させることができます。この機能により、ECMAScriptモジュールはいくつかのリソースが表示されることを期待できます。これは、それらをインポートする他のモジュールが、インポートされたモジュールの本体が作業可能になるまで待機するという事実につながります。
