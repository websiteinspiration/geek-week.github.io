Masalah Akses Kunci


gpg tidak melihat kuncinya. Lebih tepatnya, saya melihatnya, tetapi hanya sebagai root. Sebuah studi panjang tentang masalah dan googling diikuti. Hasilnya, ternyata


  • Saya tidak punya aturan norak untuk akses pengguna ke perangkat
  • Akses ke peta dapat dilakukan oleh lebih dari satu metode dari gpg

Masalah pertama diselesaikan dengan menambahkan file /etc/udev/rules.d/60-gnuk.rules


SUBSYSTEMS=="usb", ATTRS{idVendor}=="234b", ATTRS{idProduct}=="0000",   \
ENV{ID_SMARTCARD_READER}="1", ENV{ID_SMARTCARD_READER_DRIVER}="gnupg"

Dan reboot udev


devadm control --reload-rules

Ini lebih berlaku untuk mereka yang bukan debian . Itu dan turunannya tidak memiliki masalah ini, tapi saya punya fedora. Setelah itu, kuncinya terlihat.


Masalah kedua bisa diselesaikan dengan dua cara.


  • Nonaktifkan layanan pcscd
  • Konfigurasikan penggunaan layanan pcscd di gnupg

Anda dapat menonaktifkan layanan dengan menghapusnya atau dengan menghentikan layanan secara banally


systemctl stop pcscd
systemctl disable pcscd

Opsi kedua adalah mengkonfigurasi layanan gnupg untuk menggunakan pcscd


touch ~/.gnupg/scdaemon.conf
echo -e "pcsc-driver /usr/lib64/libpcsclite.so.1\ndisable-ccid]\n" > ~/.gnupg/scdaemon.conf

scdaemon gnupg pcscd.
