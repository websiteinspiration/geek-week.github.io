Wi-Fi 6を導入するときに人工知能を使用する理由

Wi-Fi 6への移行を計画するときは、Wi-Fi 6機能によってネットワークパフォーマンスとユーザー接続品質が大幅に向上する領域から始めることが重要です。これは、ネットワーク管理センターであるCisco DNA Centerに実装された人工知能/機械学習（AI / MO）によって支援されます。AI / MOソフトウェアは構成を監視し、テレメトリを収集し、ユーザー、デバイス、アプリケーションの興味深い測定を可能にします。MOアルゴリズムは、イベントの複雑な相関を作成し、コンテキストを考慮して状況を評価できるため、特定の問題の解決に役立ちます。

Wi-Fi 6は既存のワイヤレスネットワークのどのような問題を解決しますか？

  • 混雑したエリアでの貧弱なネットワークパフォーマンス
  • アップリンクでのモバイルデバイスのパフォーマンスの低下
  • 高レベルの電波干渉
  • IoTトラフィックによるネットワークの混雑

Cisco DNA CenterのAI / MOは、ワイヤレスネットワークの問題領域を特定するのに役立ちます。

過負荷領域


MU-MIMOを備えた802.11ac / Wi-Fi 5は、混雑したエリアでうまく機能します。アクセスポイントをWi-Fi 6に置き換える必要があるのは、輻輳によってパフォーマンスが低下し、ユーザーの接続の品質が低下する領域のみです。

Wi-Fi 6で状況が改善されるのはなぜですか？

OFDMAはWi-Fi 6の基礎であり、一度に複数のクライアントに情報を同時に送信できます。これが、Wi-Fi 6とWi-Fi領域で以前に作成されたすべてのものとの根本的な違いです。ネットワークのパフォーマンスは約4倍に向上します。
パフォーマンスの低下は、最大電力で情報を送信するクライアントデバイスが原因で発生する場合もあり、それらのアクセスポイントだけでなく、隣接する無線リソースにも共通の無線リソースを使用します。多くの場合、無線リソースはビジーに見えます。 BSS Coloringテクノロジーはモバイル通信から借用したもので、アクセスポイントが顧客のグループを作成し、放射電力を削減し、ネットワーク全体のノイズレベルを削減し、他のグループの信号を無視するように指示します。以前は放流を待たされていたお客様も同時に情報発信。
最後に、MU-MIMOメカニズムの改善-802.11acとは異なり、ダウンリンクだけでなくアップリンクでも機能するようになり、アップリンク伝送に費やされる非効率的な時間も削減されます。

Cisco DNA Centerの「Trends and Insights」セクションにある保証メニューでは、AI / MOを使用して、ほとんどすべてをキャンパスネットワーク上のすべてのものと比較できます。たとえば、建物、フロア、特定のアクセスポイント間のワイヤレスネットワークパフォーマンスなどです。無線チャネルの使用率スケジュールは、アクセスポイントを最高から最低にランク付けします。無線チャネルの使用率が高い場合、無線リソースはほぼ常に占有され、アクセスポイントは非効率的です。チャネルの利用率が高いポイントでのパケット損失のレベルを見てみましょう。その結果、（1）チャネル使用率と（2）リレーのレベルが高いアクセスポイントが取得されます。これらのアクセスポイントをWi-Fi 6に置き換えることをお勧めします。ちなみに、Cisco DNA Centerを使用すると、一般的なキャンパスの日に画像を巻き戻して、そこから統計を取得できます。



モバイル機器にとって容易ではない領域


モバイルデバイスは、アクセスポイントよりも少ない電力でWi-Fiインターフェイスで送信します（通常、15mW対100mW）。このため、モバイルデバイスがアクセスポイントから適切な信号レベルでアップリンクデータを送信できない場合、非対称チャネルの状況が発生します。ユーザーは困惑している-私は良いWi-Fiを使用しており、インジケーターに4本のスティックがある！良好な信号レベルは、アップリンクに既存の問題があるアクセスポイント（ダウンリンク）から測定されます。この問題はさまざまな形で現れます。各部屋の干渉パターン（干渉）は異なり、コンクリートと金属の構造がアップリンクの問題を補強します。 Wi-Fi 6のOFDMAにより、モバイルデバイスは送信をより狭いチャネルに集中させて電力を増やすことができます。これは、夏のコテージの散水ノズルとほぼ同じように機能し、水の流れの圧力を高めます。その結果、Wi-Fi 6を使用すると、低電力デバイスでアップリンクの信号レベルが向上し、SNR（信号対雑音比）が向上します。これは、複数の反射がある環境では重要です。 Wi-Fiクライアントがアップリンク伝送品質の問題を経験している領域を特定する方法は？



Trends and InsightsメニューのAI / MOを使用して、キャンパス内のすべてのアクセスポイントのクライアントデバイスの平均RSSI（受信信号強度インジケーター）を表示します。ドットがワイヤレスクライアントをどのように聞くかについての写真を入手してください。RSSIが平均を下回っているアクセスポイントは、Wi-Fi 6に置き換えるのが理にかなっています。

高ノイズ領域


ワイヤレスネットワークでの干渉のレベルが高いため、ユーザーはネットワーク接続が長く、アプリケーションの応答が悪く、クラウドへの接続が困難になる可能性があります。 AI Network Analytics Cisco DNA CenterのAI機能は、干渉を自動的に検出し、メインコントロールパネルの「Top 10 Issues」ウィンドウでアラートを発行します。 「Trends and Insights」AI / IOメニューで、干渉レベルでアクセスポイントを並べ替えることができます。

アクセスポイントをクリックし、「インテリジェントキャプチャ」ツールを確認します。パケット、フレーム、無線環境の高度な分析を実行します。スペクトル分析をクリックします。波は、干渉が存在するチャネルと、特定のアクセスポイントの動作に対するこの干渉の影響を示します。インテリジェントキャプチャにより、ソースがWi-Fiでない場合でも干渉を検出して分析できます。



写真は2.4GHz帯でのスペクトル分析結果です。チャネル1および2では、チャネル3および4とは異なり、高レベルの干渉があります。干渉が1つまたは2つのWi-Fiチャネルに制限されている場合は、それらを使用しないようにアクセスポイントを構成できます。ただし、干渉がすべてのチャネルに存在する場合は、Wi-Fi 6の交換に適した候補になります。Wi-Fi6のOFDMAはチャネル内干渉を最小限に抑え、さらにWi-Fi 6クライアントはより狭いチャネルでより多くの電力で情報を送信できるため、外部干渉に対する耐性。

小さなIoTパッケージの問題


この問題は、マシン間トラフィック（M2M）またはビデオ監視に使用されるWi-Fiネットワークで知られています。このタイプの通信は、少量のデータを高周波で転送することを意味します。ほとんどの場合、M2Mはデータを64バイトのUDPパケットにカプセル化しますが、通常のIPファイル転送は1,500バイトの大きなパケットを使用します。 Wi-Fiホットスポットは、チップセットが処理できる1秒あたりのパケット数（PPS）によって制限されます。 30,000 PPSを処理できるWi-Fiチップセットを想像してみてください。 1,500バイトの通常のパケットの場合、このデバイスは360 Mb /秒（30,000 x 1500 x 8）を転送します。ただし、64バイトのパケットの場合、最大パフォーマンスは45 Mb /秒に低下します。に。 20 Mb /秒のM2Mトラフィックは、アクセスポイントのパフォーマンスのほぼ半分を占めます。

小さなパケットの問題を検出するには、「Trends and Insights」のAI / MOメニューに移動し、アクセスポイントをトラフィック（「Traffic」）でソートします。これにより、パケット送信で最もビジーなアクセスポイントが決定されます。インテリジェントキャプチャを使用しますが、今回はフレームカウンターとフレームエラーを確認します。高トラフィック負荷、高フレーム、およびフレームエラーのあるアクセスポイントは、Wi-Fi 6の優れた候補です。
