ちょっとした数学


機械学習では、四角コアを使用した畳み込みが最もよく使用されますが、プレゼンテーションを簡略化するために、最初に1次元の場合を検討します。サイズの入力1次元画像があるとします。1×4：

d=[d0d1d2d3]T,


とフィルターサイズ 1×3：

g=[g0g1g2]T,


畳み込みの結果は次のようになります。

F(2,3)=[d0g0+d1g1+d2g2d1g0+d2g1+d3g2].


これらの式は、マトリックス形式で都合よく書き直されます。

F(2,3)=[d0d1d2d1d2d3][g0g1g2]=[d0g0+d1g1+d2g2d1g0+d2g1+d3g2]=[m1+m2+m3m2−m3−m4],(1)


表記が使用される場所：

m1=(d0−d2)g0,m2=(d1+d2)g0+g1+g22,m3=(d2−d1)g0−g1+g22,m4=(d1−d3)g2.


一見すると、最後の置換は少し無意味に見えます-明らかにより多くの操作があります。しかし、表現g0+g1+g22 そして g0−g1+g22一度だけ計算する必要があります。これを念頭に置いて、乗算の4つの演算のみを実行する必要がありますが、元の式では6を実行する必要があります。
式（1）を再度書き換えます。

F(2,3)=AT[(Gg)⊙(BTd)],(2)


どこ ⊙ 要素ごとの乗算を示し、次の表記も使用されます。

BT=[10−1001100−110010−1],(3)G=[10012121212−1212001],(4)AT=[111001−1−1].(5)


式（2）は、2次元の場合に一般化できます。

F(2×2,3×3)=AT[[GgGT]⊙[BTdB]]A.(6)


に必要な乗算演算の数 F(2×2,3×3) から縮小する 2×2×3×3=36 前 4×4=16。したがって、次の計算ゲインが得られます。3616=2.25回。あなたが視覚化した場合、実際には、画像の各ポイントの畳み込みを個別に計算する代わりに、サイズのブロックに対して直ちに計算します2×2：

count=(m+r−1)×(n+s−1),(7)

k=m×n×r×s(m+r−1)×(n+s−1).(8)


限界では、十分に大きい m そして nたたみ込みの場合は、ポイントによる乗算は1回の演算で十分です。残念ながら、ブロックサイズの増加は、入力オーバーヘッド変換の急速な増加につながりますBTdB そして週末 AT[...]A 画像なので、実際には通常より大きなブロックサイズを使用しないでください 4×4。

Vinogradアルゴリズムの実用的な実装では、最も単純なケース-カーネルとの畳み込みを検討します 3×3 ブロック用 2×2。プレゼンテーションをさらに簡略化するために、入力画像のパディングがなく、入力画像と出力画像のサイズが2

float relu(float value)
{
    return value > 0 ? return value : 0;
}

void gemm_nn(int M, int N, int K, float alpha, const float * A, int lda, 
    float beta, const float * B, int ldb, float * C, int ldc)
{
  for (int i = 0; i < M; ++i)
    for (int j = 0; j < N; ++j)
    {
      C[i*ldc + j] = beta;
      for (int k = 0; k < K; ++k)
        C[i*ldc + j] += alpha * A[i*lda + k] * B[k*ldb + j];
    }
}

void im2col(const float * src, int srcC, int srcH, int srcW,
    int kernelY, int kernelX, float * buf)
{
  int dstH = srcH - kernelY + 1;
  int dstW = srcW - kernelX + 1;
  for (int sc = 0; sc < srcC; ++sc)
    for (int ky = 0; ky < kernelY; ++ky)
      for (int kx = 0; kx < kernelX; ++kx)
        for (int dy = 0; dy < dstH; ++dy)
          for (int dx = 0; dx < dstW; ++dx)
            *buf++ = src[((b * srcC + sc)*srcH + dy + ky)*srcW + dx + kx];
}

void convolution(const float * src, int batch, int srcC, int srcH, int srcW,
    int kernelY, int kernelX, int dilationY, int dilationX,
    int strideY, int strideX, int padY, int padX, int padH, int padW, int group,
    const float * weight, const float * bias, float * dst, int dstC, float * buf)
{
  int dstH = srcH - kernelY + 1;
  int dstW = srcW - kernelX + 1;
  int M = dstC;
  int N = dstH * dstW;
  int K = srcC * kernelY * kernelX;
  for (int b = 0; b < batch; ++b)
  {
    im2col(src, srcC, srcH, srcW, kernelY, kernelX, buf);
    gemm_nn(M, N, K, 1, weight, K, 0, buf, N, dst, N));
    for (int i = 0; i < M; ++i)
      for (int j = 0; j < N; ++j)
        dst[i*N+ j] = relu(dst[i*N + j] + bias[i]);
    src += srcC*srcH*srcW;
    dst += dstC*dstH*dstW;
  }
}

strideY = strideX = dilationY = dilationX = group = 1, padY = padX = padH = padW = 0.

変更された畳み込みアルゴリズム


ここでもう一度式（6）を与えます。

F(2×2,3×3)=AT[[GgGT]⊙[BTdB]]A.


出力画像サイズから切り替えると 2×2 任意のサイズの画像にするには、それをサイズのブロックに分割する必要があります 2×2。そのようなブロックごとに、入力画像が形成されます(2+3−1)×(2+3−1)=16 変換された16に含まれる値 BTdB入力画像を半分のサイズに縮小しました。次に、これらの16個の行列のそれぞれに、変換された重みの対応する行列が乗算されます。GgGT。16個の結果の行列が出力画像に変換されますAT[...]A。このプロセスの下に図が描かれています。



これらのコメントを念頭に置いて、畳み込みアルゴリズムは次の形式を取ります。

void convolution(const float * src, int batch, int srcC, int srcH, int srcW,
    int kernelY, int kernelX, int dilationY, int dilationX,
    int strideY, int strideX, int padY, int padX, int padH, int padW, int group,
    const float * weight, const float * bias, float * dst, int dstC, float * buf)
{
  const int block = 2, kernel = 3;
  int count = (block + kernel - 1)*(block + kernel - 1); // 16
  int dstH = srcH - kernelY + 1;
  int dstW = srcW - kernelX + 1;
  int tileH = dstH / block;
  int tileW = dstW / block;
  int sizeW = srcC * dstC;
  int sizeS = srcC * tileH * tileW;
  int sizeD = dstC * tileH * tileW;
  int M = dstC;
  int N = tileH * tileW;
  int K = srcC;
  float * bufW = buf;
  float * bufS = bufW + sizeW*count;
  float * bufD = bufS + sizeS*count;
  set_filter(weight, sizeW, bufW);
  for (int b = 0; b < batch; ++b)
  {
    set_input(src, srcC, srcH, srcW, bufS, sizeS);
    for (int i = 0; i < count; ++i)
      gemm_nn(M, N, K, 1, bufW + i * sizeW, K, 
        bufS + i * sizeS, N, 0, bufD + i * sizeD, N));
    set_output(bufD, sizeD, dst, dstC, dstH, dstW);
    for (int i = 0; i < M; ++i)
      for (int j = 0; j < N; ++j)
        dst[i*N+ j] = relu(dst[i*N + j] + bias[i]);
    src += srcC*srcH*srcW;
    dst += dstC*dstH*dstW;
  }
}

入力画像と出力画像の変換を考慮せずに必要な操作の数：〜srcC * dstC * dstH * dstW * count。次に、重み、入力画像、出力画像を変換するアルゴリズムについて説明します。

たたみ込みスケールを変換する


式（6）からわかるように、畳み込みの重みに対して次の変換を実行する必要があります。 GgGTマトリックスはどこですか G式（4）で定義されています。このような変換のコードは次のようになります。

void set_filter(const float * src, int size, float * dst)
{
  for (int i = 0; i < size; i += 1, src += 9, dst += 1)
  {
    dst[0 * size] = src[0];
    dst[1 * size] = (src[0] + src[2] + src[1])/2;
    dst[2 * size] = (src[0] + src[2] - src[1])/2;
    dst[3 * size] = src[2];
    dst[4 * size] = (src[0] + src[6] + src[3])/2;
    dst[5 * size] = ((src[0] + src[6] + src[3]) + 
        (src[2] + src[8] + src[5]) + (src[1] + src[7] + src[4]))/4;
    dst[6 * size] = ((src[0] + src[6] + src[3]) + 
        (src[2] + src[8] + src[5]) - (src[1] + src[7] + src[4]))/4;
    dst[7 * size] = (src[2] + src[8] + src[5])/2;
    dst[8 * size] = (src[0] + src[6] - src[3])/2;
    dst[9 * size] = ((src[0] + src[6] - src[3]) + 
        (src[2] + src[8] - src[5]) + (src[1] + src[7] - src[4]))/4;
    dst[10 * size] = ((src[0] + src[6] - src[3]) + 
        (src[2] + src[8] - src[5]) - (src[1] + src[7] - src[4]))/4;
    dst[11 * size] = (src[2] + src[8] - src[5])/2;
    dst[12 * size] = src[6];
    dst[13 * size] = (src[6] + src[8] + src[7])/2;
    dst[14 * size] = (src[6] + src[8] - src[7])/2;
    dst[15 * size] = src[8];
  }
}

幸い、この変換は1回だけ実行する必要があります。したがって、最終的なパフォーマンスには影響しません。

入力画像変換


式（6）からわかるように、入力画像に対して次の変換を実行する必要があります。 BTdBマトリックスはどこですか BT式（3）で定義されています。このような変換のコードは次のようになります。

void set_input(const float * src, int srcC, int srcH, int srcW, 
    float * dst, int size)
{
  int dstH = srcH - 2;
  int dstW = srcW - 2;
  for (int c = 0; c < srcC; ++c)
  {
    for (int row = 0; row < dstH; row += 2)
    {
      for (int col = 0; col < dstW; col += 2)
      {
        float tmp[16];
        for (int r = 0; r < 4; ++r)
          for (int c = c; c < 4; ++c)
            tmp[r * 4 + c] = src[(row + r) * srcW + col + c];
        dst[0 * size] = (tmp[0] - tmp[8]) - (tmp[2] - tmp[10]);
        dst[1 * size] = (tmp[1] - tmp[9]) + (tmp[2] - tmp[10]);
        dst[2 * size] = (tmp[2] - tmp[10]) - (tmp[1] - tmp[9]);
        dst[3 * size] = (tmp[1] - tmp[9]) - (tmp[3] - tmp[11]);
        dst[4 * size] = (tmp[4] + tmp[8]) - (tmp[6] + tmp[10]);
        dst[5 * size] = (tmp[5] + tmp[9]) + (tmp[6] + tmp[10]);
        dst[6 * size] = (tmp[6] + tmp[10]) - (tmp[5] + tmp[9]);
        dst[7 * size] = (tmp[5] + tmp[9]) - (tmp[7] + tmp[11]);
        dst[8 * size] = (tmp[8] - tmp[4]) - (tmp[10] - tmp[6]);
        dst[9 * size] = (tmp[9] - tmp[5]) + (tmp[10] - tmp[6]);
        dst[10 * size] = (tmp[10] - tmp[6]) - (tmp[9] - tmp[5]);
        dst[11 * size] = (tmp[9] - tmp[5]) - (tmp[11] - tmp[7]);
        dst[12 * size] = (tmp[4] - tmp[12]) - (tmp[6] - tmp[14]);
        dst[13 * size] = (tmp[5] - tmp[13]) + (tmp[6] - tmp[14]);
        dst[14 * size] = (tmp[6] - tmp[14]) - (tmp[5] - tmp[13]);
        dst[15 * size] = (tmp[5] - tmp[13]) - (tmp[7] - tmp[15]);
        dst++;
      }
    }
    src += srcW * srcH;
  }
}

この変換に必要な操作の数は、〜srcH * srcW * srcC * countです。

出力画像変換


式（6）からわかるように、出力画像に対して次の変換を実行する必要があります。 AT[...]Aマトリックスはどこですか BT式（5）で定義されています。このような変換のコードは次のようになります。

void set_output(const float * src, int size, 
    float * dst, int dstC, int dstH, int dstW)
{
  for (int c = 0; c < dstC; ++c)
  {
    for (int row = 0; row < dstH; row += 2)
    {
      for (int col = 0; col < dstW; col += 2)
      {
        float tmp[8];
        tmp[0] = src[0 * size] + src[1 * size] + src[2 * size];
        tmp[1] = src[1 * size] - src[2 * size] - src[3 * size];
        tmp[2] = src[4 * size] + src[5 * size] + src[6 * size];
        tmp[3] = src[5 * size] - src[6 * size] - src[7 * size];
        tmp[4] = src[8 * size] + src[9 * size] + src[10 * size];
        tmp[5] = src[9 * size] - src[10 * size] - src[11 * size];
        tmp[6] = src[12 * size] + src[13 * size] + src[14 * size];
        tmp[7] = src[13 * size] - src[14 * size] - src[15 * size];
        dst[col + 0] = tmp[0] + tmp[2] + tmp[4];
        dst[col + 1] = tmp[1] + tmp[3] + tmp[5];
        dst[dstW + col + 0] = tmp[2] - tmp[4] - tmp[6];
        dst[dstW + col + 1] = tmp[3] - tmp[5] - tmp[7];
        src++;
      }
      dst += 2*dstW;
    }
  }
}

この変換に必要な操作の数は、〜dstH * dstW * dstC * countです。

ブドウアルゴリズムの長所と短所


まず、もちろん、長所：

  • このアルゴリズムでは、数回（多くの場合2〜3回）畳み込みの計算を高速化できます。したがって、「理論的な」限界よりも速く畳み込みを計算することが可能です。
  • このアルゴリズムは、標準の行列乗算アルゴリズムに基づいて実装されています。
  • これは、さまざまな入力画像フォーマット（NCHW、NHWC）に実装できます。
  • 中間値を格納するためのバッファーサイズは、行列ベースのアルゴリズムに必要なサイズよりも小さくなっています。

さて、マイナスなしの場合：

  • このアルゴリズムでは、畳み込みカーネルのサイズごと、およびブロックサイズごとに、変換関数を個別に実装する必要があります。たぶん、これがカーネルとのたたみ込みのためだけに実装されているほとんどすべての理由の1つです。3×3。
  • ブロックサイズが大きくなると、変換関数の実装の複雑さが急速に大きくなります。したがって、実際には、ブロックサイズが4×4。
