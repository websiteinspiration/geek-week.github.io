1つのウィンドウで複数のLinux端末を実行する

Linuxは、ターミナルウィンドウをいくつかのミニ画面に分割する多くの方法を提供します。これにより、ある意味で「マルチタスク」モードの操作が可能になります。これは、少なくとも5つの方法で実行できます。


Linuxターミナルのあたたかいちらつきよりも優れていると思われるものは何でしょうか。 -）
これは判明しました：2つの端末の暖かいちらつき！実際には、より多くの陽気です。

端末はかつて物理的なデバイスでしたが、現在ではコンピュータ上のアプリケーションにすぎません。ターミナルを頻繁に使用している場合は、1つのターミナルだけでは不十分なことがよくあります。そして、新しいターミナル（新しいウィンドウ）を開いて、最初のターミナルがコンパイル、変換、またはその他のデータ処理でビジー状態になっているときに作業できるようにする必要があります。

あなたがシステム管理者である場合、最も可能性が高いのは、いくつかのオープンターミナルが必要になるでしょう：1つはWebサーバーの管理用、もう1つはデータベースの管理用、3つ目はファイルのコピー用などです。開いているタブが多数あると、すべてのプロセスを同時に追跡することが難しくなり、作業が非常に複雑になります。

Linuxには、複数のタブを開くことができるターミナルアプリケーションが古くから存在し、幸いにも、この傾向はやがて発展し始めました。今では誰もが当然のことながらマルチタブ端末を利用しています。それでも、タブを切り替える必要は多くの人にとって迷惑です。

したがって、次のステップは画面に分割することです。2つ以上のミニターミナルを同じウィンドウで同時に開くことができます。ほとんどのLinuxビルドには、そのようなパーティションの編成に役立つ多くのツールがあります。

tmux



これはおそらく最も柔軟なツールです。tmuxはターミナルマルチプレクサーであり、完全にホットキーに基づいて動作します。マウスを探すためにキーから手を離す必要はありませんが、特定のキーの組み合わせが何をするかを調べる必要があります。

あるタブを別のタブに「オーバーレイ」して、それらを切り替えることができます。タブを2つ（3つ、4つなど）の画面に分割することもできます。

主に画面への分割にtmuxを使用する場合、いくつかの組み合わせを学習するだけで十分です。

  • Ctrl-B％垂直方向に分割（左側に1つの画面、右側にもう1つの画面）
  • Ctrl-B "水平方向に分割（1つの画面は上、もう1つの画面は下）
  • Ctrl-B Oパネル間の切り替え
  • Ctrl-B？オープンヘルプ
  • Ctrl-B d tmuxを終了し、バックグラウンドで実行したままにします

Tmuxには多くの興味深い機能があります。たとえば、1つのコンピューターでtmuxセッションを開始し、別のコンピューターから同じセッションにリモートで接続します。たとえば、Piでtmuxを実行しているおかげで、どこからでもIRCに接続できます。Piでtmuxを実行し、アクセス可能な任意のコンピューターからログインします。ログアウトしても、tmuxは機能し続け、別のマシンからセッションに接続するのを辛抱強く待っています。

GNUスクリーン




GNU Screenの tmuxと同様に、すでに実行中のセッションに接続および切断し、ウィンドウを水平および垂直に画面に分割できます。

ただし、このツールはtmuxほど柔軟ではありません。 GNU Screen Ctrl-Aの紹介の組み合わせは、Bashの行の先頭にジャンプするコマンドでもあります。したがって、マルチプレクサーの実行中に行の先頭に移動するには、Ctrl-Aを1回ではなく2回押す必要があります。したがって、個人的には、通常、この組み合わせを$ HOME / .screenrcのCtrl-Jに変更します。

escape ^jJ

Screenでの画面分割機能はうまく機能しますが、tmuxにはない欠点がいくつかあります。たとえば、ターミナルウィンドウを分割すると、分割後に表示されるパネルでターミナルの新しいコピーが開始されません。Ctrl-A Tab（または私のようにキーボードショートカットを再定義する場合はCtrl-J Tab）を押し、Ctrl-A Cを使用して手動で新しいシェルを作成する必要があります

。tmuxとは異なり、ターミナルを終了しても分割は消えません。これは設計機能であり、場合によっては非常に便利です。パーティションのリセットを手動で管理する必要があるため、不便な場合もあります。

ただし、GNU Screenは信頼できる強力なアプリケーションであり、何らかの理由でtmuxを使用できない場合に実行できます。

基本的なGNU Screenコマンドは次のとおりです。

  • Ctrl-A S ( — , — )

  • Ctrl-A ?
  • Ctrl-A d Screen ( , screen -r)

Konsole



Konsoleは、標準のKDEプラズマデスクトップターミナルです。 KDE自体と同様に、Konsoleには広範なカスタマイズオプションがあります。

その多くの機能の中には、ウィンドウを画面に分割する機能があります。 Konsoleはグラフィカル端末であるため、キーボードではなくマウスで制御できます。

これを行うには、[表示]メニューに移動します。ウィンドウを水平または垂直に分割できます。アクティブなパネルを切り替えるには、マウスで必要なパネルを選択するだけです。各パネルは固有の端末であるため、独自のテーマとタブを持つことができます。

tmuxやGNU Screenとは異なり、Konsoleセッションを切断して再接続することはできません。ほとんどのグラフィカルアプリケーションと同様に、物理的にはKonsoleを使用します。そのため、リモートアクセスには特別なソフトウェアを使用する必要があります。

Emacs



Emacsはターミナルマルチプレクサではありませんが、そのインターフェースはウィンドウの分割とサイズ変更をサポートしています。それはまた統合されたターミナルを持っています。

いずれにしても、毎日Emacsで作業している場合、これはワークスペースを便利に整理できる機会を高く評価したことを意味します。さらに、Emacs eshellモジュールはeLISPに実装されているため、Emacs自体で使用するのと同じコマンドを使用してモジュールと対話でき、長いファイルパスや出力コマンドを簡単にコピーおよび抽出できます。

グラフィックスウィンドウでEmacsを使用する場合は、マウスでいくつかのアクションを実行できます。たとえば、アクティブなパネルをクリックして選択したり、ウィンドウを分割した後で画面のサイズを変更したりできます。

キーボードショートカットを使用する方が速い場合もあります：

  • Ctrl-X 3垂直方向に分割（1つの画面が左側にあり、別の画面が右側にある）
  • Ctrl-X 2を水平方向に分割（1つの画面は上、もう1つの画面は下）
  • Ctrl-X Oパネル間の切り替え（これはマウスで実行できます）
  • Ctrl-X 0（0はゼロ）現在のパネルを閉じる

tmuxおよびGNU Screenと同様に、emacs-clientを介してEmacsセッションを切断および接続できます。

ウィンドウマネージャー



ウィンドウを画面に分割し、それぞれの画面でターミナルを起動できるテキストエディターが不思議な性質であると考える場合、デスクトップが同じタスクを実行できることがわかったときの驚きは何でしょう。 Ratpoison、Herbsluftwm、i3、AwesomeなどのLinuxデスクトップ、さらにはKDEプラズマデスクトップも、各アプリケーションウィンドウがデスクトップグリッドに固定タイルとして表示されるように構成できます。

デスクトップの「上」にあるウィンドウの代わりに、ウィンドウは指定された場所に留まるため、一方から他方に切り替えることができます。端末マルチプレクサをエミュレートすることにより、ネットワーク上の任意の数の端末を開くことができます。実際、デスクトップマルチプレクサーでターミナルマルチプレクサーを開くこともできます。

そして、マルチプレクサモードで内部のEmacsを開くことを妨げるものは何もありません。同じ流れを続けると何が起こるか誰にもわかりません。ほとんどのLinuxユーザーは、これを自宅で繰り返さないことが最善であることに同意しています。

tmuxやGNU Screenとは異なり、特別なソフトウェアを使用せずにデスクトップの「セッション」を切断して再接続することはできません。
