Electronic walk-through: biometrics and multi-format

The electronic checkpoint is a turnstile with built-in readers, a controller and software - a ready-made kit for organizing access control. An electronic gatehouse can be used both as an independent solution and as an element of a large-scale ACS.



Multi-format


The use of modern multi-format readers allows simultaneous use different identifiers in electronic pass-throughs:

  • EMM / HID format access cards ((HID ProxCard II, ISO prox II; EM-Marin IL-05ELR, EM4100))
  • MIFARE format access cards with reading either a unique UID or crypto-protected data from the card‚Äôs internal memory, additional programming of readers and user cards themselves is required
  • bank cards with PayPass contactless payment technology with reading UID
  • Android smartphones with NFC function with reading a unique identifier (installation and launch of a free application on the phone is required)
  • Apple smartphones with NFC function with reading a unique identifier (Token) associated with a bank card (when linking several bank cards, the Token of the card that is currently active is read).

Built-in electronic pass-through multi-format RFID readers can significantly simplify the transition from EMM / HID cards to MIFARE cards with copy protection. All types of identifiers can be used by employees and visitors at the same time. Readers are connected to the controller via the RS-485 interface.

On the market there are electronic checkpoints based on the curbstile turnstile, including those with a built-in card reader and tripod turnstile.



Biometric identification


Access by biometric data is one of the main trends in the modern security systems market. The widespread use of this method is caused by the desire for maximum protection of identification.

In accordance with current demands, new electronic walk-through models with the possibility of biometric identification have appeared on the market.

The updated model of the electronic gate KT02 is equipped with 2 fingerprint scanners that connect to the controller via a USB interface. In order to move to more reliable identification methods, the already installed previous-generation electronic walk-through KT02 can be updated without a complete replacement.

To use access cards of various formats, payment cards, and smartphones with an NFC module as identifiers, it‚Äôs enough to install multi-format readers in an existing electronic checkpoint. To apply biometric identification, multi-format readers need to be supplemented with a cover with fingerprint scanners and a controller with biometric support.



The electronic checkpoint controller communicates with a PC and other system controllers via an Ethernet interface. Support is provided for the TCP / IP protocol stack (ARP, IP, ICMP, TCP, UDP, DHCP.

In ACS based on the electronic pass-through, face recognition can also be used for identification. For this, the electronic pass-through stand is complemented by the face recognition terminal installed on a special bracket.



The interaction of the server access control system and the terminal is carried out online. The system allows you to quickly add data for employees and visitors to use face recognition as an additional way of identification. Face recognition on the terminal occurs automatically - without the participation of the server.

For additional control, a security officer can view information about a passing employee in verification mode. Also, to strengthen access control, an employee can be assigned a pass using several identification methods at once, for example, a face recognition pass with confirmation by an access card or fingerprint. All events with frames of face recognition passes are stored in the access control system and can be viewed.

System without a dedicated server


Based on the electronic checkpoint, you can build an ACS without a dedicated server, which will be enough to control access at the entrance and to several internal rooms.

The memory of the new generation of controllers is pre-installed software. In addition to the basic functions of access control, the software allows accounting for working hours, integration with 1C, verification, and ordering passes. The system can process the data of 500 employees and 500 visitors and have up to 10 controllers. To select the required functionality, a 60-day test period is provided, after which a license is required.

If for solving access control tasks in an enterprise with up to 100 employees, basic functionality is enough, including time-based access control, zoning control, access with commissioning, you can use a free software package. After the trial period, you will need to get a free license.

Up to 8 lock controllers with an integrated reader can be connected to the electronic checkpoint, which allows equipping eight rooms with an access control system.



The emergence of a new generation of controllers and universal readers allows you to build an access control system based on an electronic pass-through system using reliable identification methods with minimal cost.
