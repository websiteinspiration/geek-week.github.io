根证书过时的问题。下一步让我们加密和智能电视



为了使浏览器对网站进行身份验证，它会将自己显示为有效的证书链。上面显示了一个典型的链，它可以具有多个中间证书。有效链中证书的最小数量为三。

根证书是证书颁发机构的核心。它实际上嵌入在您的OS或浏览器中，实际上存在于您的设备中。您不能在服务器端进行更改。您需要强制更新设备上的操作系统或固件。

安全专家斯科特·赫尔姆（ Scott Helme）写道，“加密”证书颁发机构将带来主要问题，因为今天它是Internet上最受欢迎的CA，其根证书很快就会失效。更改根目录让我们加密定于2020年7月8日。

证书颁发机构（CA）的最终证书和中间证书已从服务器传递给客户端，并且客户端已经具有根证书，因此，使用此证书集合，您可以链接和认证网站。

问题在于每个证书都有一个有效期，在此之后需要更换它。例如，从2020年9月1日开始，Safari计划对服务器TLS证书的有效期引入限制，最长为398天。

这意味着我们所有人都必须至少每12个月更换一次服务器证书。此限制仅适用于服务器证书，它并不适用于根CA证书。

CA证书受一组不同的规则约束，因此它们具有不同的到期限制。有效期为5年的中间证书和使用寿命长达25年的根证书非常常见！

中间证书通常没有问题，因为它们是由服务器交付给客户端的，服务器本身通常会更改其自己的证书，因此在此过程中只需替换中间证书即可。与根CA证书不同，用服务器证书替换非常容易。

就像我们已经说过的那样，根CA直接嵌入在客户端设备本身，操作系统，浏览器或其他软件中。无法更改根CA网站。这需要在客户端上进行更新，无论是更新操作系统还是软件。

一些根CA已经存在很长时间了，我们正在谈论20-25年。很快，一些最古老的根CA将接近其自然寿命，它们的时间快用完了。对于我们大多数人来说，这根本就不是问题，因为CA创建了新的根证书，并且这些根证书已经在OS和浏览器更新中分发了很多年。但是，如果有人很长时间没有更新OS或浏览器，这是一种问题。

格林威治标准时间2020年5月30日10:48:38出现了这种情况。这是来自Comodo证书颁发机构（Sectigo）的AddTrust根证书被删除的确切时间。

它用于交叉签名，以确保与在其存储库中没有新的USERTrust根证书的旧式设备兼容。

不幸的是，问题不仅出现在过时的浏览器中，而且出现在基于OpenSSL 1.0.x，LibreSSL和GnuTLS的非浏览器客户端中。例如，在Roku电视机顶盒中，Heroku服务在Fortinet，Chargify应用程序中，在适用于Linux的.NET Core 2.0平台上以及其他许多应用程序中。

由于现代浏览器可以使用第二个USERTRust根证书，因此认为该问题仅会影响旧系统（Android 2.3，Windows XP，Mac OS X 10.11，iOS 9等）。但是实际上，崩溃始于使用免费的OpenSSL 1.0.x和GnuTLS库的数百个Web服务。由于证书过时错误，已停止建立安全连接。

下一步-让我们加密


即将发生的根CA更改的另一个很好的例子是“让我们加密”证书颁发机构。早在2019年4月，他们计划从Identrust链切换到自己的ISRG Root链，但这没有发生。



“由于担心ISRG根在Android设备上的分布不充分，我们决定将过渡到我们自己的根的日期从2019年7月8日推迟到2020年7月8日，”让我们加密在一份正式声明中说。

由于根CA并非在所有客户端上都过于普及，因此由于“根传播”问题，或者缺少根传播，不得不推迟日期。

现在，让我们加密使用具有到IdenTrust DST根CA X3根的链的交叉签名中间证书。该根证书的发布日期为2000年9月，有效期至2021年9月30日。在此之前，Let's Encrypt计划切换到自己的自签名根ISRG Root X1。



ISRG根目录于2015年6月4日发布。之后，开始批准其为认证中心的过程，该过程于2018年8月6日结束。从现在开始，通过更新操作系统或软件，所有客户都可以使用根CA。所有要做的就是安装更新。

但这就是问题所在。

如果您的手机，电视或其他设备两年未更新，它将如何知道新的ISRG Root X1根证书？而且，如果系统上未安装该证书，则一旦Let's Encrypt切换到新的根目录，您的设备将使所有Let's Encrypt服务器证书无效。 Android生态系统中有很多已经过很长时间没有更新的过时设备。


Android生态系统，

这就是为什么“让我们加密”延迟了过渡到其自己的ISRG根目录并仍使用向下到IdenTrust根目录的中间链接的原因。但是无论如何都必须完成过渡。根更改的日期是2020年7月8日。

要验证设备（电视，机顶盒或其他客户端）上是否安装了ISRG X1根目录，请打开测试站点https://valid-isrgrootx1.letsencrypt.org/。如果没有出现安全警告，则通常一切正常。

让我们加密不是唯一需要解决切换到新根目录的问题的人。互联网上的密码术开始使用已有20多年了，所以就在现在，有很多根证书过期的时代。

多年未更新智能电视软件的智能电视的所有者可能会遇到此问题。例如，新的GlobalSign R5根目录根发布于2012年，并且在某些旧的智能电视无法建立链接之前，因为它们根本没有此根CA。特别是，这些客户端无法建立与bbc.co.uk的安全连接。为了解决该问题，BBC管理员不得不欺骗：他们使用尚未变质的R3 Root和R1 Root的旧根，通过附加的中间证书为这些客户端构建了替代链。

www.bbc.co.uk（叶）
GlobalSign ECC OV SSL CA 2018（中级）
GlobalSign根CA-R5（中级）
GlobalSign根CA-R3（中级）

这是一个临时解决方案。如果不更新客户端软件，该问题将不会解决。智能电视本质上是基于Linux的功能有限的计算机。而且，如果不进行更新，其根证书将不可避免地变得腐烂。

这适用于所有设备，而不仅仅是电视机。如果您有连接到Internet的任何设备，并被宣传为“智能”设备，那么几乎可以肯定的是，证书存在问题。如果设备未更新，则CA根存储库将随着时间的推移而过时，最终问题将浮出水面。问题发生的时间取决于根存储的最新更新时间。实际发布设备的日期可能要提前几年。

顺便说一下，这就是为什么某些大型媒体平台无法使用现代的自动证书颁发机构（例如，Let's Encrypt）的问题，Scott Helme写道。它们不适用于智能电视，并且根数太少，无法保证在传统设备上支持证书。否则，电视将根本无法启动现代流媒体服务。

最新的AddTrust事件表明，即使是大型IT公司也没有为根证书的到期做好准备。

解决问题的方法只有一种-更新。智能设备开发人员必须提供一种用于预先更新软件和根证书的机制。另一方面，对于制造商来说，在保修期结束时确保其设备的运行是无利可图的。
