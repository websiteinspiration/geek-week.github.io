Merampingkan Check Point API dengan Python SDK

Semua kekuatan berinteraksi dengan API terungkap ketika digunakan bersama dengan kode program, ketika dimungkinkan untuk secara dinamis menghasilkan permintaan dan alat API untuk menganalisis respons API. Namun, Kit Pengembangan Perangkat Lunak Python (selanjutnya disebut sebagai Python SDK) untuk Check Point Management API , tetapi sia-sia , tetap tidak terlihat . Ini secara signifikan menyederhanakan kehidupan pengembang dan pecinta otomatisasi. Python telah mendapatkan popularitas yang luar biasa akhir-akhir ini dan saya memutuskan untuk menjembatani kesenjangan dan memberikan gambaran tentang fitur-fitur utama dari Check Point API Python Development Kit . Artikel ini adalah pelengkap yang sangat baik untuk artikel lain di Habr√© Check Point R80.10 API. Manajemen melalui CLI, skrip dan banyak lagi. Kami akan mempertimbangkan cara menulis skrip menggunakan Python SDK dan membahas lebih rinci tentang fungsionalitas API Manajemen baru di versi 1.6 (didukung sejak R80.40). Untuk memahami artikel ini, Anda perlu pengetahuan dasar tentang bekerja dengan API dan Python.

Check Point aktif mengembangkan API dan saat ini ada:

  • Periksa Point Management API (versi saat ini 1.6) - bekerja dengan server manajemen melalui API (dan kemampuan untuk mengeksekusi skrip pada gateway di bawah kendali server manajemen)
  • Check Point GAIA API (versi saat ini 1.4) - bekerja dengan gateway keamanan
  • Threat Prevention API 1.0 - Sandboxing di Check Point Cloud
  • API Kesadaran Identitas - bekerja dengan blade Kesadaran Identitas di gateway
  • API Portal Manajemen Keamanan - bekerja dengan portal manajemen gateway SMB ( Pelajari lebih lanjut tentang gateway SMB )
  • API IoT - interaksi dengan pengontrol IoT
  • CloudGuard Connect API - bekerja dengan CloudGuard Connect (solusi keamanan SD-WAN)
  • Dome9 API - bekerja dengan Dome9

Python SDK saat ini hanya mendukung interaksi dengan API Manajemen dan API Gaia . Kami akan membahas kelas, metode, dan variabel terpenting dalam modul ini.



Pemasangan modul


Modul cpapi menginstal dengan cepat dan mudah dari repositori Titik Periksa resmi di github menggunakan pip . Petunjuk instalasi terperinci ada di README.md . Modul ini disesuaikan untuk bekerja dengan versi Python 2.7 dan 3.7. Pada artikel ini, contoh akan diberikan menggunakan Python 3.7. Namun, Python SDK dapat dijalankan langsung dari server manajemen Titik Periksa (Manajemen Cerdas), tetapi mereka hanya mendukung Python 2.7, jadi bagian terakhir akan menyediakan kode untuk versi 2.7. Segera setelah menginstal modul, saya sarankan melihat contoh di examples_python2 dan examples_python3 direktori .

Awal pekerjaan


Agar kami dapat bekerja dengan komponen-komponen modul cpapi , setidaknya dua kelas yang diperlukan harus diimpor dari modul

cpapi : APIClient dan APIClientArgs

from cpapi import APIClient, APIClientArgs

Kelas APIClientArgs bertanggung jawab untuk menghubungkan ke server API, dan kelas APIClient bertanggung jawab untuk berinteraksi dengan API.

Kami menentukan parameter koneksi


Untuk menentukan berbagai parameter untuk menghubungkan ke API, Anda perlu membuat turunan dari kelas APIClientArgs . Pada prinsipnya, parameternya sudah ditentukan sebelumnya dan Anda bisa menghilangkannya saat menjalankan skrip di server manajemen.

client_args = APIClientArgs()

Tetapi ketika memulai pada host pihak ketiga, Anda harus menentukan setidaknya alamat IP atau nama host dari server API (ini juga merupakan server manajemen). Dalam contoh di bawah ini, kami mendefinisikan parameter koneksi server dan menetapkannya alamat IP dari server manajemen sebagai string.

client_args = APIClientArgs(server='192.168.47.241')

Mari kita lihat semua parameter dan nilai defaultnya yang dapat digunakan saat menghubungkan ke server API:

Argumen untuk metode __init__ dari kelas APIClientArgs 
class APIClientArgs:
    """
    This class provides arguments for APIClient configuration.
    All the arguments are configured with their default values.
    """

    # port is set to None by default, but it gets replaced with 443 if not specified
    # context possible values - web_api (default) or gaia_api
    def __init__(self, port=None, fingerprint=None, sid=None, server="127.0.0.1", http_debug_level=0,
                 api_calls=None, debug_file="", proxy_host=None, proxy_port=8080,
                 api_version=None, unsafe=False, unsafe_auto_accept=False, context="web_api"):
        self.port = port
        # management server fingerprint
        self.fingerprint = fingerprint
        # session-id.
        self.sid = sid
        # management server name or IP-address
        self.server = server
        # debug level
        self.http_debug_level = http_debug_level
        # an array with all the api calls (for debug purposes)
        self.api_calls = api_calls if api_calls else []
        # name of debug file. If left empty, debug data will not be saved to disk.
        self.debug_file = debug_file
        # HTTP proxy server address (without "http://")
        self.proxy_host = proxy_host
        # HTTP proxy port
        self.proxy_port = proxy_port
        # Management server's API version
        self.api_version = api_version
        # Indicates that the client should not check the server's certificate
        self.unsafe = unsafe
        # Indicates that the client should automatically accept and save the server's certificate
        self.unsafe_auto_accept = unsafe_auto_accept
        # The context of using the client - defaults to web_api
        self.context = context


Saya percaya bahwa argumen yang dapat digunakan dalam contoh kelas APIClientArgs intuitif untuk administrator Titik Periksa dan tidak perlu komentar tambahan.

Terhubung melalui APIClient dan manajer konteks


Kelas APIClient paling mudah digunakan melalui manajer konteks. Yang perlu Anda lewati untuk instance kelas APIClient adalah parameter koneksi yang ditentukan pada langkah terakhir.

with APIClient(client_args) as client:

Manajer konteks tidak akan secara otomatis membuat panggilan masuk ke server API, tetapi akan membuat panggilan keluar saat keluar. Jika karena alasan tertentu logout tidak diperlukan di akhir pekerjaan dengan panggilan API, Anda harus mulai bekerja tanpa menggunakan manajer konteks:

client = APIClient(clieng_args)


Pemeriksaan koneksi


Cara termudah untuk memeriksa apakah koneksi melewati parameter yang diberikan adalah dengan menggunakan metode check_fingerprint . Jika verifikasi hash sha1 untuk sidik jari dari sertifikat API server tidak lulus (metode mengembalikan False ), maka ini biasanya disebabkan oleh masalah koneksi dan kami dapat menghentikan program (atau mengizinkan pengguna untuk memperbaiki data koneksi):

    if client.check_fingerprint() is False:
        print("Could not get the server's fingerprint - Check connectivity with the server.")
        exit(1)

Perhatikan bahwa di masa mendatang, kelas APIClient akan memeriksa dengan setiap panggilan API ( api_call dan metode api_query , kita akan membicarakannya sedikit lebih jauh) sha1 sertifikat sidik jari di server API. Tetapi jika kesalahan terdeteksi ketika memeriksa sidik jari dari sertifikat API server (sertifikat tidak diketahui atau telah diubah), metode check_fingerprint akan memberikan kemampuan untuk menambah / mengubah informasi tentangnya di mesin lokal dalam mode otomatis. Pemeriksaan ini dapat dimatikan sama sekali (tetapi ini hanya dapat direkomendasikan jika skrip dijalankan pada server API itu sendiri, ketika terhubung ke 127.0.0.1) menggunakan argumen APIClientArgs - unsafe_auto_accept (lihat detail tentang APIClientArgs sebelumnya di ‚ÄúMenentukan Parameter Koneksi‚Äù).

client_args = APIClientArgs(unsafe_auto_accept=True)

Masuk ke server API


Dalam APIClient memiliki sebanyak 3 metode login pada server API, dan masing-masing dari mereka nilai zaponimaet sid (id sesi), yang digunakan secara otomatis dalam setiap panggilan API berikutnya dalam header (nama judul dalam pengaturan ini - X-CHKP-sid ), jadi Anda tidak perlu memproses parameter ini lebih lanjut.

Metode masuk


Opsi menggunakan nama pengguna dan kata sandi (dalam contoh, nama pengguna admin dan kata sandi 1q2w3e dilewatkan sebagai argumen posisional):

     login = client.login('admin', '1q2w3e')

Parameter opsional opsional juga tersedia dalam metode login. Saya mencantumkan nama dan nilai standarnya: 

continue_last_session=False, domain=None, read_only=False, payload=None

Metode login_with_api_key


Opsi menggunakan kunci api (didukung sejak versi manajemen R80.40 / API Manajemen v1.6, ‚Äú3TsbPJ8ZKjaJGvFyoFqHFA ==‚Äù adalah nilai kunci API untuk salah satu pengguna di server manajemen dengan metode otorisasi kunci API):

     login = client.login_with_api_key('3TsbPJ8ZKjaJGvFyoFqHFA==')

Dalam metode login_with_api_key , parameter opsional yang sama tersedia seperti pada metode login .

Metode login_as_root


Opsi masuk ke mesin lokal dengan server API:

     login = client.login_as_root()

Hanya dua parameter opsional yang tersedia untuk metode ini:

domain=None, payload=None

api_query


Segera buat reservasi bahwa metode ini hanya berlaku untuk panggilan yang hasilnya melibatkan offset (shift). Kesimpulan ini terjadi ketika mengandung atau mungkin mengandung sejumlah besar informasi. Sebagai contoh, ini mungkin permintaan untuk daftar semua objek yang dibuat dari jenis host di server manajemen. Untuk permintaan semacam itu, API mengembalikan daftar 50 objek secara default (Anda dapat meningkatkan batas hingga 500 objek dalam respons). Dan agar tidak menarik informasi beberapa kali, mengubah parameter ofset dalam permintaan API, ada metode api_query yang melakukan ini secara otomatis. Contoh panggilan di mana metode ini diperlukan: show-sesi, show-host, show-network, show-wildcard, show-grup, show-address-range, show-simple-gateways, show-simple-cluster, show-access- peran, pertunjukan-tepercaya-klien, pertunjukan-paket. Bahkan, atas nama panggilan API ini kita melihat kata-kata jamak, sehingga panggilan ini akan lebih mudah ditangani melalui api_query

show_hosts = client.api_query('show-hosts')

Memproses Hasil Panggilan API


Setelah itu, Anda bisa menggunakan variabel dan metode kelas respons APIR (baik di dalam konteks manager dan di luar). Kelas APIResponse memiliki 4 metode dan 5 variabel yang telah ditentukan, kita akan membahas yang paling penting secara lebih rinci.



keberhasilan


Untuk memulainya, alangkah baiknya untuk memastikan bahwa panggilan API berhasil dan mengembalikan hasilnya. Ada metode sukses untuk ini :

In [49]: api_versions.success                                                   
Out[49]: True

Mengembalikan Benar jika panggilan API berhasil (kode respons adalah 200) dan Salah jika tidak berhasil (kode respons lainnya). Lebih mudah digunakan segera setelah panggilan API, sehingga tergantung pada kode respons, Anda dapat menampilkan informasi yang berbeda.

if api_ver.success: 
    print(api_versions.data) 
else: 
    print(api_versions.err_message)

set_success_status


Mungkin ada kebutuhan untuk mengubah nilai status keberhasilan. Secara teknis, Anda dapat meletakkan apa pun di sana, bahkan string biasa. for task in task_result.data["tasks"]:

Contoh yang berguna


Berikut ini adalah contoh yang menggunakan panggilan API yang telah ditambahkan ke API Manajemen 1.6.

Untuk memulai, pertimbangkan operasi panggilan add-host dan add-address-range . Misalkan kita perlu membuat semua alamat ip dari subnet 192.168.0.0/24 sebagai objek dari tipe host, oktet terakhir di antaranya adalah 5, dan menulis semua alamat ip lainnya sebagai objek kisaran alamat. Dalam hal ini, alamat subnet dan alamat broadcast harus dikecualikan.

Jadi, di bawah ini adalah skrip yang memecahkan masalah ini dan membuat 50 objek dari tipe host dan 51 objek dari berbagai jenis alamat. Untuk mengatasi masalah ini, diperlukan 101 panggilan API (tidak termasuk panggilan publikasi terakhir). Juga, dengan menggunakan modul timeit, kami menghitung waktu yang diperlukan untuk menjalankan skrip hingga perubahan diposting.

Script menggunakan add-host dan add-address-range 
import timeit
from cpapi import APIClient, APIClientArgs

start = timeit.default_timer()

first_ip = 1
last_ip = 4

client_args = APIClientArgs(server="192.168.47.240")

with APIClient(client_args) as client: 
     login = client.login_with_api_key('3TsbPJ8ZKjaJGvFyoFqHFA==')
     for ip in range(5,255,5):
         add_host = client.api_call("add-host", {"name" : f"h_192.168.0.{ip}", "ip-address": f'192.168.0.{ip}'})
     while last_ip < 255:
         add_range = client.api_call("add-address-range", {"name": f"r_192.168.0.{first_ip}-{last_ip}", "ip-address-first": f"192.168.0.{first_ip}", "ip-address-last": f"192.168.0.{last_ip}"})
         first_ip+=5
         last_ip+=5
     stop = timeit.default_timer() 
     publish = client.api_call("publish")
     
print(f'Time to execute batch request: {stop - start} seconds')


Di lingkungan lab saya, dibutuhkan 30 hingga 50 detik untuk menjalankan skrip ini, tergantung pada beban pada server manajemen.

Sekarang mari kita lihat bagaimana menyelesaikan masalah yang sama menggunakan add-objects-batch call API , yang dukungannya telah ditambahkan ke API versi 1.6. Panggilan ini memungkinkan Anda membuat banyak objek sekaligus dalam satu permintaan API. Selain itu, ini bisa berupa objek dari tipe yang berbeda (misalnya, host, subnet, dan rentang alamat). Dengan demikian, tugas kami dapat diselesaikan dalam rangka panggilan API tunggal.

Script menggunakan add-objects-batch 
import timeit
from cpapi import APIClient, APIClientArgs

start = timeit.default_timer()

client_args = APIClientArgs(server="192.168.47.240")

objects_list_ip = []
objects_list_range = []

for ip in range(5,255,5):
    data = {"name": f'h_192.168.0.{ip}', "ip-address": f'192.168.0.{ip}'}
    objects_list_ip.append(data)
    
first_ip = 1
last_ip = 4


while last_ip < 255:
    data = {"name": f"r_192.168.0.{first_ip}-{last_ip}", "ip-address-first": f"192.168.0.{first_ip}", "ip-address-last": f"192.168.0.{last_ip}"}
    objects_list_range.append(data)
    first_ip+=5
    last_ip+=5

data_for_batch = {
  "objects" : [ {
    "type" : "host",
    "list" : objects_list_ip
}, {
    "type" : "address-range",
    "list" : objects_list_range
  }]
}


with APIClient(client_args) as client: 
     login = client.login_with_api_key('3TsbPJ8ZKjaJGvFyoFqHFA==')
     add_objects_batch = client.api_call("add-objects-batch", data_for_batch)
     stop = timeit.default_timer() 
     publish = client.api_call("publish")
     
print(f'Time to execute batch request: {stop - start} seconds')


Dan dibutuhkan 3 hingga 7 detik untuk menjalankan skrip ini di lingkungan lab saya, tergantung pada beban pada server manajemen. Artinya, secara rata-rata, pada objek 101 API, panggilan tipe batch memenuhi 10 kali lebih cepat. Pada lebih banyak objek, perbedaannya akan lebih mengesankan.

Sekarang mari kita lihat bagaimana cara bekerja dengan set-object-batch . Menggunakan panggilan API ini, kami dapat secara besar-besaran mengubah parameter apa pun. Mari kita atur bagian pertama dari alamat dari contoh sebelumnya (hingga .124 host, dan rentang juga) sienna warna, dan bagian kedua dari alamat kita akan menetapkan khaki warna.

Ubah warna objek yang dibuat pada contoh sebelumnya 
from cpapi import APIClient, APIClientArgs

client_args = APIClientArgs(server="192.168.47.240")

objects_list_ip_first = []
objects_list_range_first = []

import getpass
import sys, os
sys.path.append(os.path.abspath(os.path.join(os.path.dirname(__file__), '..')))
from cpapi import APIClient, APIClientArgs

def main():
    with APIClient() as client:
       # if client.check_fingerprint() is False:

        login_res = client.login_as_root()

        if login_res.success is False:
            print("Login failed:\n{}".format(login_res.error_message))
            exit(1)

        gw_name = raw_input("Enter the gateway name:")
        gw_ip = raw_input("Enter the gateway IP address:")
        if sys.stdin.isatty():
            sic = getpass.getpass("Enter one-time password for the gateway(SIC): ")
        else:
            print("Attention! Your password will be shown on the screen!")
            sic = raw_input("Enter one-time password for the gateway(SIC): ")
        version = raw_input("Enter the gateway version(like RXX.YY):")
        add_gw = client.api_call("add-simple-gateway", {'name' : gw_name, 'ipv4-address' : gw_ip, 'one-time-password' : sic, 'version': version.capitalize(), 'application-control' : 'true', 'url-filtering' : 'true', 'ips' : 'true', 'anti-bot' : 'true', 'anti-virus' : 'true', 'threat-emulation' : 'true'})
        if add_gw.success and add_gw.data['sic-state'] != "communicating":
            print("Secure connection with the gateway hasn't established!")
            exit(1)
        elif add_gw.success:
            print("The gateway was added successfully.")
            gw_uid = add_gw.data['uid']
            gw_name = add_gw.data['name']
        else:
            print("Failed to add the gateway - {}".format(add_gw.error_message))
            exit(1)
