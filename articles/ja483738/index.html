Micropost：PostgreSQLのUNICODE文字列をカット

時々、データベースに「不可解」を入れたいという欲求があります-例えば、非常に長い線を張ります。いいえ、それをテーブルフィールドに書き込みます-PostgreSQLでは問題ありませんが、インデックス...

問題は、インデックスの行全体（ROW）が1つのデータページ（8KB）に完全に収まる必要があることです。それ以外の場合は、次のようになります。
エラー：インデックス行のサイズ...がインデックス...の最大値...を超えています
つまり、単一行のインデックスの最も単純なケースでも、レーキを踏むことができます。それらにどのように対処しますか？

もちろん、substringで関数インデックスを作成することもできますが、同じ式をすべてのクエリにドラッグする必要があるため、幸福感はまったくありません。NOTIFYで大きすぎる（8000バイトを超える）ペイロード

を送信しようとすると、同様のレーキが待機します。

すべてが非常に単純であるように思われます-8Kのラインを取り、カットします（または必要に応じてスライスにカットします）。ここにあるのは8Kバイトだけで、JavaScriptの行（好みの言語に置き換えます）はすべてUnicode文字で構成されており、バイト長も可変である場合があります。そして、データベースに「半無効」の行を入力することは、どういうわけか完全に悪いです。

しかし、文字の「真ん中」の文字列をカットするとどうなりますか？ここでUNICODE規格が役立ちます：
U + FFFD REPLACEMENT CHARACTERは、不明な、認識できない、または表現できない文字を置き換えるために使用されます
それは最後に我々が得る、私たちは、このようなA「チョップバイト」文字列の背中を変換しようとすることが判明したコードFFFDの文字：



場合は、合計を全体の文字列全体が私たちのために非常に高価ではありません（例えば、始まりに過ぎないが、私達にとって重要であるから、エラーテキスト、と）、最後の整数文字を必要なサイズ（バイト単位）にトリミングします。

const MAX_BYTE_LEN = 8191;
let buf = Buffer.from(errorString);
if (buf.length > MAX_BYTE_LEN) {
  errorString = buf.slice(0, MAX_BYTE_LEN).toString().replace(/\uFFFD$/, '');
}
