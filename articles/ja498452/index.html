Django：国際化のクイックガイド

アプリケーションのさまざまな言語への翻訳とそのローカリゼーションは、すべての開発者が対処しなければならないものです。本日の翻訳は、Djangoアプリケーションの国際化に関する簡単なガイドです。

ここで説明する内容の一部は、Pythonプロジェクトのローカリゼーションに適用されます。基本を確認したら、国際化の取り組みを加速することについて話します。特に、Phraseプラットフォームの使用について。 この資料の読者には、動作するDjangoアプリケーションがあり、パッケージがインストールされていることを前提としています（たとえば、コマンドを使用してインストールできます）。これまでにDjango（Pythonベースの人気のWebフレームワーク）を使用したことがない場合は、最初にこれを確認することをお勧めします。



gettextpip install gettext公式ガイド、そしてこの記事に戻ります。

基本的な作業環境設定


それでは、Djangoプロジェクトmysiteとと呼ばれるアプリケーションがあるとしpollsます。プロジェクトの構造は次のようになります。

/
    manage.py
    mysite/
        __init__.py
        settings.py
        urls.py
        wsgi.py
    polls/
        migrations/
        __init__.py
        admin.py
        models.py
        tests.py
        views.py

作業の最初のステップは、プロジェクト構成で国際化オプションがアクティブになっているかどうかを確認することです。これを行うには、に次の変更を加える必要がありますmysite/settings.py。

# mysite/settings.py
LANGUAGE_CODE = 'en-us'
TIME_ZONE = 'UTC'
USE_I18N = True
USE_L10N = True
USE_TZ = True

テンプレートの国際化


次に、異なる言語に翻訳する必要があるすべての行をマークする必要があります。次のテンプレートファイルがあるとしますpolls/templates/polls/index.html。

<!-- polls/templates/polls/index.html -->
<h1>Welcome to our site!</h1>
<p>Here you find polls.</p>

このファイルは次のように処理する必要があります。

<!-- polls/templates/polls/index.html -->
{% load i18n %}
<h1>{% trans 'WelcomeHeading' %}</h1>
<p>{% trans 'WelcomeMessage' %}</p>

ここでは、ローカライズ用のパッケージをインポートし、すべてのテキストをビュー構成に置き換えましたtrans 'SomeTranslationKeyName'。さらに、このアプローチの代わりに、デフォルトの翻訳テキストを翻訳キーの形式で使用できます。このアプローチでは、特定のキーで使用できる翻訳がない場合でも、常に適切なデフォルトのテキストが得られます。

Pythonコードの国際化


Pythonコード内（ファイル内などpolls/views.py）で文字列をローカライズする場合は、関数をファイルにインポートする必要がありますugettext。非常に正常な彼女には、エイリアスを割り当てることができます_。ファイルからの単純な関数のローカライズバージョンは次のようになりますviews.py。

# polls/views.py
from django.http import HttpResponse
from django.utils.translation import ugettext as _
def index(request):
    output = _('StatusMsg')
    return HttpResponse(output)

翻訳ファイルの作成


次に、サポートする言語設定ごとに翻訳ファイルを作成する必要があります。これを行うには、ディレクトリpolls/localeを作成し、そのディレクトリでpolls次のコマンドを実行します。

django-admin makemessage -l de

ここでde、アプリケーションに追加する予定の言語のロケールコードを置き換えることができます。この例では、このコマンドを実行するgettextと、polls/locale/de/LC_MESSAGES/django.po次の内容の-fileが作成されます。

# polls/locale/de/LC_MESSAGES/django.po
...
#: templates/polls/index.html:3
msgid "WelcomeHeading"
msgstr ""
#: templates/polls/index.html:4
msgid "WelcomeMessage"
msgstr ""

これで、このファイルに改行を入力できます。

# polls/locale/de/LC_MESSAGES/django.po
...
#: templates/polls/index.html:3
msgid "WelcomeHeading"
msgstr "Willkommen auf unserer Seite!"
#: templates/polls/index.html:4
msgid "WelcomeMessage"
msgstr "Hier findet Ihr Umfragen."

翻訳の準備ができたら、次のコマンドを実行してすべてをコンパイルする必要があります。

$ django-admin compilemessages

ディレクトリで、このコマンドを再度実行しますpolls。

翻訳の操作性をすばやく確認するには、ファイル内の言語コードを変更する必要がありますmysite/settings.py。たとえば、次のようになります。

# mysite/settings.py
LANGUAGE_CODE = 'de'

ここでブラウザでアプリケーションを開く場合は、ドイツ語に翻訳する必要があります。

フレーズを使用してアプリケーションのローカライズを高速化する


フレーズを使用してアプリケーションのローカライズに関する作業を整理する場合は、* .poファイルを手動で作成して編集する必要はありません。キーWelcomeHeadingとWelcomeMessageフレーズを作成して転送し、エクスポート機能を使用して* .poファイルをダウンロードするだけです。

コマンドラインツールをインストールするとPhrase Client、作業がさらに簡単になります。プロジェクトのルートディレクトリに.phraseapp.yml次の内容の構成ファイルを作成するだけで十分です。

# .phraseapp.yml
phraseapp:
  access_token: <your access token>
  project_id: <your project's id on PhraseApp>
  file_format: po
  pull:
    targets:
        file: "polls/locale/<locale_code>/LC_MESSAGES/django.po

次に、プロジェクトのルートディレクトリで次のコマンドを実行する必要があります。

phraseapp pull && django-admin compilemessages

このおかげで、プロジェクトのすべての翻訳が更新されます。

さらに、DjangoプロジェクトでIn-Context Editorを使用すると、作業をさらに簡略化できます。これを行うには、次django-phraseを使用してインストールしpipます。

pip install django-phrase

次に、使用する予定のテンプレートを編集するだけですIn-Context-Editor。

<!-- polls/templates/polls/index.html -->
{% load i18n %}
{% load phrase_i18n %}
{% phrase_javascript %}
<h1>{% trans 'WelcomeHeading' %}</h1>
<p>{% trans 'WelcomeMessage' %}</p>

ロードphrase_i18nはの後に行う必要があることに注意してくださいi18n。

最後に、以下を構成ファイルに追加します。

# mysite/settings.py
PHRASE_ENABLED = True
PHRASE_PROJECT_ID = 'YOUR_PROJECT_ID'
PHRASE_PREFIX = '{{__'
PHRASE_SUFFIX = '__}}'

その後、すべての準備が整います。

ロケールの選択


通常、アプリケーション開発者は、ユーザーのブラウザー設定に基づいてロケールを設定します。これを行うには、ファイルmysite/settings.pyを次の形式にする必要があります。

# mysite/settings.py
from django.utils.translation import ugettext_lazy as _
...
MIDDLEWARE_CLASSES = (
    ...,
    'django.middleware.locale.LocaleMiddleware',
    ...,
)
...
LANGUAGE_CODE = 'en-us'
LANGUAGES = (
    ('en-us', _('English')),
    ('de', _('German')),
)

この方法では、ユーザーがロケールを持っている場合、German翻訳オプションが表示されますde。それ以外の場合は、デフォルトの変換が使用されen-usます。次を使用して、このメカニズムの正しい動作を確認できますcurl。

curl http://localhost:8000/polls -H "Accept-Language: de"

このコマンドは、次のようなものを返します。

<h1>Willkommen auf unserer Seite!</h1>
<p>Hier findet Ihr Umfragen.</p>
