Niktoを使用して任意のサイトの脆弱性をチェックする

みなさん、こんにちは！3月に、OTUSは新しいKali Linux Workshopを開始します。コースの開始を見越して、役立つ資料の翻訳を用意しました。また、「サービス拒否攻撃とその防御」というトピックに関する無料のレッスンを皆さんにご紹介したいと思います。




サイトを攻撃する前に、ハッカーまたはペンテスターは最初に目標のリストを作成します。彼が適切な偵察を行い、「狙う」ことの弱点を見つけた後、彼は脆弱性（潜在的な攻撃ベクトル）を見つけるのに役立つNiktoなどのWebサーバースキャンツールが必要になります。

Niktoは、Webサイトをスキャンして、悪用またはハッキングされる可能性のある脆弱性を報告するシンプルなオープンソースのWebサーバースキャナーです。さらに、業界全体の脆弱性についてWebサイトをスキャンするために最も広く使用されているツールの1つであり、多くの分野で業界標準と見なされています。

このツールは非常に効果的であるという事実にもかかわらず、それはひそかに機能しません。侵入検知システムやその他のセキュリティ対策を備えたサイトは、スキャンされていることを理解します。Niktoはセキュリティをテストするように設計されており、その作業の機密性については誰も考えていませんでした。

Niktoの使い方


ターゲットWebサイトでNiktoを実行するだけでは、スキャン後に取得した情報をどう処理するか理解できない場合があります。 Niktoは実際にはレーザーポインターのようなもので、ショットを伴います。しばらくすると、その動作を確認できます。

まず、ターゲットについて話しましょう。ターゲットは、ネットワークプリンターやWebサーバーなど、ハッカーが攻撃できるほぼすべての場所です。少し後でNiktoの使用に進むときは、ローカルサービスのIPアドレス、攻撃ドメイン、またはSSL / HTTPS Webサイトの3種類の情報のいずれかを提供する必要があります。

Niktoでスキャンを開始する前に、Maltegoなどのオープンツールを使用して予備調査を行うことをお勧めします。このようなツールは、プロファイルを作成し、焦点を当てる目標のより具体的なリストを形成するのに役立ちます。これを実行したら、Niktoを使用して、リストからターゲットの潜在的な脆弱性を検索できます。

運が良ければ、よく知られているエクスプロイトの脆弱性が見つかります。つまり、この弱点を悪用するのに役立つツールがすでに存在しています。脆弱性を自動的に悪用する適切なツールを使用して、ハッカーはターゲットにアクセスし、悪意のあるコードの追加など、あらゆる数の隠れた攻撃を実行できます。

手順1：Niktoをインストールする


Kali Linuxを使用する場合、Niktoがプリインストールされるため、何もダウンロードしてインストールする必要はありません。これは脆弱性分析カテゴリに配置されます。何らかの理由でそれがない場合は、GitHubのリポジトリからNiktoをダウンロードするか、コマンドを使用しapt installます。

apt install nikto

Macを使用している場合は、Homebrewを使用してNiktoをインストールできます。

brew install nikto

ステップ3：基本的な構文を使用する


前のステップからわかるように、Niktoには多くの使用例がありますが、私たちの目的のために、山かっこなしの実際のIPアドレスまたはホスト名で基本構文<IPまたはホスト名>を使用します。

nikto -h <IP or hostname>

ただし、NiktoはSSLおよびHTTPSサイトで使用されるポート443をスキャンできます（HTTPはデフォルトでポート80を使用します）。したがって、古いサイトをクロールするだけでなく、SSLを使用してサイトの脆弱性を評価できます。これは、現在、検索結果でのインデックス作成の必須要件です。

ターゲットサイトにSSLがあることがわかっている場合は、Niktoでこれを指定して、コマンドの最後に-sslを追加することで、クロールにかかる時間を節約できます。

nikto -h <IP or hostname> -ssl

手順5：IPアドレスをスキャンする


これでウェブサイトのクイックスキャンが完了したので、ローカルネットワークでNiktoを使用して、ルーターのログインページや、ウェブサイトのないサーバーである別のマシンのHTTPサービスなどの埋め込みサーバーを見つけます。 IPアドレスを見つけるために使用しますifconfig。

必要なIPアドレスはinetを指します。その上でipcalc、ネットワーク範囲を取得するために使用できます。 ipcalcがない場合は、コマンドを使用してインストールし、apt install ipcalc再試行できます。範囲は「ネットワーク」の後にあり、私の場合は192.168.0.0/24です。

ifconfig

        inet 192.168.0.48  netmask 0xffffff00  broadcast 192.168.0.255
        inet6 XXXX::XXX:XXXX:XXXX:XXXX%en0  prefixlen 64  secured scopeid 0x8
        ether XX:XX:XX:XX:XX:XX  txqueuelen 1000  (Ethernet)
        inet6 XXXX::XXX:XXXX:XXXX:XXXX%en0  prefixlen 64 autoconf secured
        inet6 XXXX::XXX:XXXX:XXXX:XXXX%en0  prefixlen 64 autoconf temporary
        nd6 options=201<PERFORMNUD,DAD>
        media: autoselect
        status: active

        options=60<TS04,TS06>
        ether XX:XX:XX:XX:XX:XX
        media: autoselect <full-duplex>
        status: inactive

次に、Nmapを起動して、このネットワーク範囲で機能するサービスを見つけます。範囲を使用してポート80をスキャンしてみましょう。このため、-oG（grepable出力）を追加して、稼働中のホスト、つまりポート80が開いていると応答するホストのみを取得します。次に、これらすべてをファイルに保存しnullbyte.txtます。名前を付けます。あなたは、好きな名前を付けることができます。

ipcalc 192.168.0.48

=>
Network:   192.168.0.0/24       11000000.10101000.00000000. 00000000

スキャンのためにすべてのホストをNiktoに直接送信するのに役立つ小さなトリックがあります。catドキュメントに格納された入力を読み取るために使用しますnullbyte.txt（または自分で呼び出したとおり）。その後、次のテンプレートを見つけるのに役立つLinuxの特別なツールであるawkを使用します。ここで、Upはホストが稼働していることを意味し、print $ 2は各行の2番目の単語、つまりIPアドレスのみを印刷する必要があることを意味します。次に、受信したデータをというファイルtargetIP.txt（または必要なもの）に送信します。

cat nullbyte.txt | awk '/Up$/{print $2}' | cat >> targetIP.txt

これで、catを使用して新しいファイルの内容を表示し、ポート80が開いているすべてのIPアドレスを確認できます。

cat targetIP.txt

192.168.0.2
192.168.0.4
192.168.0.5
192.168.0.11
192.168.0.24
192.168.0.31
192.168.0.48
192.168.0.60

この形式は、このようなファイルを簡単に解釈できるため、Niktoに最適です。したがって、次のコマンドでこの出力をNiktoに送信できます。

nikto -h targetIP.txt

結果は、SSLでスキャンしたときに得られるものと同様になります。

OSVDBで始まる要素は、オープンソース脆弱性データベース（2016年に閉鎖されたサイト）で報告された脆弱性です。SecurityFocus、Microsoft Technet、 Vulnerabilities and Exposures（https://cve.mitre.org/）などの他の脆弱性データベースに似ています。個人的には、National Vulnerability Databaseが私に近いです。

スキャンでは、悪用される可能性のある重大な脆弱性は明らかになりませんでしたが、CVEヘルプツールを使用してOSVDB識別子をCVEレコードに変換し、上記のサイトのいずれかを使用できます。先に詳しく説明したLibsshの脆弱性であるCVE-2018-10933など、

調査に値するものを見つけたとします。 CVEには、それを悪用する方法、脆弱性の重大度（重大な脆弱性など）に関する情報、および攻撃経路の特定に役立つその他の情報が含まれています。これに価値がある場合は、Metasploitで検索できます。おそらく、誰かがこの脆弱性を簡単に悪用するのに役立つモジュールをすでに開発しているためです。



ステップ7：Metasploitでスキャンする


Niktoの最も優れた点の1つは、スキャンから受け取った情報をMetasploitが読み取れる形式にエクスポートできることです。これを行うには、コマンドを使用して上記のスキャンを実行しますが、最後に-Format msf +フラグを追加します。このフォーマットは、エクスプロイトを通じて取得したデータをすばやく照合するのに役立ちます。

nikto -h <IP or hostname> -Format msf+

そのため、今日のガイドでは、ターゲットの定義からその中の脆弱性を見つけることまで行ってから、脆弱性をエクスプロイトに関連付けたため、すべての作業を手動で行う必要がなくなりました。Niktoはひそかに機能しないため、VPN、Tor、またはその他の種類のサービスを介してこれらのスキャンを実行し、IPアドレスに疑わしいフラグが付けられないようにすることをお勧めします。
