理论介绍




对于那些在坦克中的人，如果您没有读过上述文章之一，我将告诉您什么是Deno。 Deno是一个运行时运行时，可在JavaScript和TypeScript上运行（开箱即用支持TypeScript，TypeScript编译器版本硬连线到Deno版本，只有更改Deno版本才可以更改它），它基于V8引擎和Rust编程语言。 Deno由Node.js的创建者Ryan Dahl创建，Deno的主要品质是性能和安全性。 Deno于2018年发布，许多消息来源忘记提及-受Golang影响很大（实际上它最初是用Golang编写的，但后来不得不用Rust重写）。

标准库是在Go标准库的模型上创建的，例如，许多工具或其实现已从Golang切换到Deno，例如，缺少像npm这样的生态系统，并且在初始构建阶段就直接从Web资源中抽取了库（这在Node中造成了一些麻烦）。很自然地对Go的类似系统不熟悉的开发人员的js）。那么，迪诺现在是谁？如果您喜欢打字语言（例如Golang的想法和工具），但想在更简单的东西上写后盾，那么该是在实际中尝试Deno的时候了。但是，如果您要等待平台成熟，则会出现更多的库和StackOverflow的答案，也许您应该稍等一下。

开始创造


尽管Deno很年轻，但仍围绕着它形成了某种生态系统（当然，尚无法与Node.js生态系统相比），这使您可以满足创建自己的API的基本需求。让我们来决定我们要创建的内容：我想在后端使用Deno和TypeScript创建一个简单的CRUD Api（因为有TypeScript支持，为什么不使用它），PostgreSQL作为数据库（通常，我想使用MySQL，但无意中破坏了我的访问权限）一个奇怪的事实是，在外部ip上，内置的PHP 7.4服务器能够连接到我的MySQL数据库，而Deno显示数据包混乱），我们的请求将如下所示：

发布 / api / todos / post 开创新业务
补丁 / api / todos /：id 更新特定案例

好吧，我们决定使用查询方案，启动了PostgreSQL服务器（如果您可以从此处下载它，这是一个非常好的Postico GUI客户端），我们创建了crud_api数据库。
下一步是什么？您当然可以使用sql查询或客户端来创建表，但是我想看看Deno的迁移情况如何。在这种情况下，nessia库很有用，它使您可以创建类似于Laravel中的迁移的迁移。但是在进行迁移之前，让我们先谈谈组织开发环境。

就在前几天，官方的JetBrains Deno开发插件发布了。你可以在这里找到它。。在Visual Studio上，您需要安装Deno插件，然后在项目文件夹中创建.vscode settings 文件夹，在settings.json中打开deno支持（但如果没有它，它似乎对我有用）：

    "deno.enable":true

在准备阶段就这些了，我们可以返回迁移了。首先，我们需要输入初始化命令，取自他们的库文档：

deno run --allow-net --allow-read --allow-write https://deno.land/x/nessie/cli.ts init

我们为我们的库创建一个配置文件，借助它我们可以确定您要连接到哪个DBMS和数据库。
最初，在文件中创建了三个模板-用于PostgreSQL，MySQL和Sqlite（默认情况下，导出了PostegreSQL的设置）。

结果，我们的nessie.config.ts文件将如下所示：

const migrationFolder = "./migrations";

const configPg = {

好了，现在您可以创建一个迁移文件，以便您可以通过指定要创建的表和数据来对其进行编辑。为此，请输入以下命令：

deno run --allow-net --allow-read --allow-write https://deno.land/x/nessie/cli.ts make create_todo

也许经验不足的读者已经对为什么这么多标志感到疑问。这在Deno中是安全的-要执行命令，您需要指定授予它的访问级别（这是我们正在等待的巧妙的安全系统）。这些标志在许多情况下会引起强烈的燃烧-是否需要打印？附加bash脚本，您总是在其中以完全权限调用它们吗？我认为这个系统有时会改变，但就目前而言，还是会改变。因此，.migrations迁移文件应出现在我们的文件夹中，您可以随时对其进行编辑。我对其进行了如下编辑：

export const up = (): string => {
  return "CREATE TABLE todo (id serial, text text, done boolean)";
};

export const down = (): string => {
  return "DROP TABLE todo"
};

之后，我们可以使用以下命令迁移到数据库：

deno run --allow-net --allow-read https://deno.land/x/nessie/cli.ts migrate

nessia中有一个更高级的迁移选项，但我决定使事情变得简单一些。此外，我们可以继续创建自己的模型。

我们与模型一起工作


我们创建一个文件夹models，其中将包含请求本身。首先，您需要创建一个文件，config.ts在其中将添加用于连接数据库的设置（是的，我们已经在中提到了它们nessie.config.ts，但是该文件用于迁移）。对于文件中的连接，config.ts我们将使用deno-posgres库。结果，配置文件将如下所示：

import { Client } from "https://deno.land/x/postgres/mod.ts";

const client = new Client({
  user: "isakura313",
  database: "deno_crud",
  hostname: "localhost",
  port: 5432,
});

export default client;

精细！现在我们直接传递给模型。要构建sql查询，我将使用Dex库，即Deno上Knex库的端口。它将允许我以编程方式确定要执行的SQL查询。让我们首先定义这次必须使用哪种方言Dex，并定义待办事项的接口：

const dex = Dex({ client: "postgres" });

interface Todo {
  id?: number;   //? -     TypeScript
  text: string;
  done: boolean;
}

好了，现在我们可以进行纸浆本身了。首先，定义一个将接收所有待办事项的获取请求。我将使用异步函数执行来通过await连接到数据库并获取查询结果：

async function getAllTodo() {
  await client.connect();
  const getQuery = dex.queryBuilder().select("*").from("todo").toString();
  const result = await client.query(getQuery);
  return result;
}

如果您甚至对TypeScript和csync / await有点熟悉，这里将没有任何问题。但是，更进一步-我们需要编写一个后请求，将案例添加到数据库中。

async function addTodo(todo: Todo) {
  await client.connect();
  const insertQuery = dex.queryBuilder().insert([todo]).into("todo").toString();
  return client.query(insertQuery).then(async () => {
    const getQuery = dex.queryBuilder().select("*").from("todo").where(
      { text: todo.text },
    ).toString();
    const result = await client.query(getQuery);
    const result_data = result.rows ? result.rows[0] : {};
    return result_data;
  });
}

上面的代码可以缩短，但是，我试图使其尽可能地简单和富有表现力，甚至损害了行数。一般而言，将在此发生以下情况-在函数内部，我们创建与数据库的连接，内置了insert-请求，发生请求，在内部创建新请求，该请求返回新创建的大小写或空对象。

其他功能-编辑和删除，我将组合在一起。在编辑过程中，除了在editTodo中进行了更新外，与添加案例时相同。在delete中，我们只删除大小写并不返回任何内容：

async function editTodo(id: number, todo: Todo) {
  await client.connect();
  const editQuery = dex.queryBuilder().from("todo").update(todo).where({ id })
    .toString();
  return client.query(editQuery).then(async () => {
    const getQuery = dex.queryBuilder().select("*").from("todo").where(
      { text: todo.text },
    ).toString();
    const result = await client.query(getQuery);
    const result_data = result.rows ? result.rows[0] : {};
    return result_data;
  });
}

async function deleteTodo(id: number) {
  await client.connect();
  const deleteQuery = dex.queryBuilder().from("todo").delete().where({ id })
    .toString();
  return client.query(deleteQuery);
}

哇。它仅保留导出我们的功能，您可以开始配置服务器和路由：

export {
  addTodo,
  getAllTodo,
  editTodo,
  deleteTodo,
};

路由和服务器操作


创建routes文件夹，其中将包含routes.ts。要使用我们的路线，我们将使用denotrain，它是一个由expressJS启发而来的库，可用于处理url请求，配置路由和许多有用的东西。我们导入功能和路由器：

import { Router } from "https://deno.land/x/denotrain@v0.5.0/mod.ts";
import { addTodo, getAllTodo, editTodo, deleteTodo } from "../models/models.ts";

const api = new Router();

添加get和post方法：

api.get("/", (ctx) => {
  return getAllTodo().then((result) => {
    return result.rows;
    // 
  });
});

api.post("/", (ctx) => {
  const body = {
  //  
    text: ctx.req.body.text,
    done: ctx.req.body.done,
  };

  return addTodo(body).then((newTodo) => {
    ctx.res.setStatus(201); //   "Created"
    return newTodo;
  });
});

Ctx是负责响应上下文的变量。我不知道要在评论中添加什么，我认为一切都已经非常明显了。仅添加补丁和删除方法并导出我们的api仍然是：

api.patch("/:id", (ctx) => {
  const todo = {
    text: ctx.req.body.text,
    done: ctx.req.body.done,
  };

  return editTodo(ctx.req.params.id as number, todo).then((result) => {
    return result;
  });
});

api.delete("/:id", (ctx) => {
  return deleteTodo(ctx.req.params.id as number).then(() => {
    ctx.res.setStatus(204);
    return true;
  });
});

export default api;

它仍然只是提高我们的服务器。在根文件夹中，创建一个文件，server.ts在该文件中，我们还从denotrain导入Application以提升服务器：

import {Application} from "https://deno.land/x/denotrain@v0.5.0/mod.ts";
import api from "./routes/routes.ts";

const app = new Application({port: 1337}) //      1337

app.use("/api/todos", api) //  ,      
app.run() //

您可能在应用程序中遇到问题。我不想每次都手动重新启动已编译的服务器，因此我使用nodemon- Denomon的类似物，它将忠实地重新编译代码并重新启动服务器。在我们的情况下，以下命令是合适的：

denomon --allow net,read server.ts
