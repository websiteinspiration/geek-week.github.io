Lines 9753：Groundhog Day

            あなたがプレイした場合のためにキューブを長い時間、
            彼らがに変わり、ボール

ゲームLines 9753の名前を見ると、この物語は有名なゲームColor Linesの9753番目のクローンについてであると思うかもしれません。同じ色の行で5つのボールを作る必要があります。はい、それはクローンです。さて、他に何が新しい可能性がありますか？HTML + CSS + JavaScriptでの実装のアイデアといくつかの微妙な点を共有します。9753という数字の意味をお話しします。数字の別の組み合わせを見つけて、ゲームの変化を確認する方法を提案します。それはもっと面白くなり、退屈になります-あなたが決めます。9753の意味を推測しましたか？それでは、猫の下で。

さまざまなゲームオプション


元のゲームのどこにフィールドサイズ9x9があるか疑問に思ったことはありませんか？ゲームの大きさだけでなく、他のパラメーターもどれだけうまく選択されているかは驚くべきことです。色の数は7、連続するボールの数は5、ターンごとに表示されるボールの数は3です。これらの4つのパラメーターはゲーム名に含まれています-Lines 9753概して、ゲームの複雑さを決定するのはこの4つであり、これはおそらくすべての人に適しているわけではありません。


おそらく誰かが別のセットを好きになるでしょう。例えば：

  • 3333-ソリティアのような孤独。行き止まりに入るのを恐れず、一度にできるだけ多くのボールを折りたたむことを試みることはできません（5はまだ簡単ですが、6はより難しく、7と8はほとんど非現実的です）。まったく考えたくない場合に適しています。
  • 5643-5x5ボードでは、セルは9x9ほど小さくないため、電話で再生するのが便利ですが、難易度は9753よりも少し高いようです。
  • 5543は5643よりも少し簡単なオプションです。これは、異なる色の数が少ないため、同じ色の一連のボールを作る可能性が高いためです。
  • 4443-複雑度は5643に匹敵しますが、ボールはさらに大きく、画面の短い側には4つのボールしかありません。



誰もが自分に適したゲームオプションを見つけると思います。ネットワークから切断しないでください。一緒にサイトの表に最も人気のあるオプションが表示されます。

戻ってくる


後退することが許可されています。むかしむかし、木々が高く、私がまだ学生だったとき、私はカラーラインをプレイしなければなりませんでした。私は本当にゲームが好きだった。ある日、半日を失い、数千ポイントを獲得したため、私は立ち止まりました。それは事故であったこと、ボールがうまく落ちたこと、そして近い将来、私はこれまで行くことができなかったことに気づきました。自分の記録を破るには、さらに10倍の時間を殺さなければなりません。それ以来、私はこのゲームを長い間プレイしていません。普遍的な不正の感覚は私の魂の奥深くにあります。そして、復讐の陰湿な計画が具体化するまでそこに座っていました-同じゲームを作るために、巻き戻す能力は少なくとも最初に戻って動きます。私がこの方法でゲームを書く主な動機付けの理由となったのは、このオプションでした。

固定ランダム性


一部のプレーヤーは、車が通路を複雑にするために不快な場所に意図的にボールを置くこともあると述べました。特にフリーセルがほとんど残っていない場合、正しいボールが正しい場所に来るのを待つことは不可能です。つまり、ゲームのランダム性はランダムではありません。巻き戻すための新しい機会があれば、新しいボールの新しい場所を毎回提供するのは論理的ではありません。したがって、正しいボールが正しい場所に現れるまで、前後に移動することができます。したがって、ゲームでは、マシンのすべての動きが決定され、動きの数とフィールド上のボールの位置にランダムに依存します。さらに、誰もが同じゲームをプレイします。つまり、ゲームは同じ配置（車の最初の動き）で始まり、その後、車の動きはプレーヤーの移動方法に依存します。プレーヤーが同じ方法で移動した場合、その後、フィールドでは同じ配置のボールが配置されます。これにより、さまざまなソリューションを比較する機会に関係なく、客観的にすることができます。一部の人々は、次は誰で、誰がより速いかを測定したい...プレーヤーの動きのシーケンスの形での決定は、サーバーに送信されます。車の動きは、同じランダムアルゴリズムに従って計算されます。サーバーをだますことはできません。シーケンス全体が正しいかどうかがチェックされ、その結果のみがテーブルに入力されます。すべてのレコードは、プレーヤーの動きのシーケンスとともに保存されます。次に、9999ポイントを獲得した人の最良の決定を分析して公開します。最適なものを選択するための基準を考え出す必要があります。車の動きは、同じランダムアルゴリズムに従って計算されます。サーバーをだますことはできません。シーケンス全体が正しいかどうかがチェックされ、その結果のみがテーブルに入力されます。すべてのレコードは、プレーヤーの動きのシーケンスとともに保存されます。次に、9999ポイントを獲得した人の最良の決定を分析して公開します。最適なものを選択するための基準を考え出す必要があります。車の動きは、同じランダムアルゴリズムに従って計算されます。サーバーをだますことはできません。シーケンス全体が正しいかどうかがチェックされ、その結果のみがテーブルに入力されます。すべてのレコードは、プレーヤーの動きのシーケンスとともに保存されます。次に、9999ポイントを獲得した人の最良の決定を分析して公開します。最適なものを選択するための基準を考え出す必要があります。

促す


以下のボールの位置と色を示します。ゲームは前後に巻き戻すことができるため、次の動きを隠すことは意味がありません。いくつかの動きを前方に向けることは可能ですが、単純なインターフェースでは発生しませんでした（プレイヤーのアクションの結果としてセルが占有されている場合、車の動きは変化することに注意してください）。

得点


スコアリングは、古典的な公式N *（1 + N-DIMENSION）に従って行われます。ここで、Nはライン内のボールの数、DIMENSIONは競技場の寸法です。つまり、集められたボールの5、6、7、8は、それぞれ5、12、21、32ポイントを獲得します。ゲームオプション3333では、3、4、5、および6つのボールが崩壊すると、3、8、15、24が獲得されます。

グラフィックアート


グラフィックスはベクトルのみです-アーチファクトのスケーリング、特にぼやけた境界線は好きではありません。アニメーションの主なグラフィック要素：ビンテージテープレコーダーと、ハイライトとグラデーションライティングを備えた跳ねるボール。CSSを使用して作られ、JavaScriptで味付けされています。最初は、暗い配色が行われました。設定でオンにしてみてください。しかし、すべてが暗く見えます。軽い方が好きです。

立体ボール


逆説的ですが、互いに入れ子になった3つの長方形から、3次元のボールが得られます。


  1. 画面上のサイズと位置を指定するコンテナ
  2. 丸みを帯びた正方形、2次元の形状を作成し、跳ねる間に収縮して伸びる
  3. ボールの色と照明を定義する楕円形のグラデーションの塗りつぶしを持つ正方形で、3番目の次元を追加します。

コンテナを使用すると、すべてが簡単です。CSS属性の位置であるleftとtopは、フィールド上のボールの座標を指定し、font-sizeは、他の2つの正方形が拡大または縮小されるサイズを設定します。丸い正方形は、私たちのWebページでは、ほとんどの円形と同様に、角が最も丸い正方形です。マウスでそれをクリックすると、それは平らになり、上から絞って、側面を広げます。そして、クリックがマウスのクリックと同じくらい速いことがわかった場合、ボールは跳ね始めます。


@keyframes flat {
            0%   {height: 1.0em; width: 1.0em; top: 0; left: 0em;}
            100% {height: 0.9em; width: 1.05em; top: 0.1em; left: -0.025em;}
        }

@keyframes jumping {

色の最大数は15です。


良いモニターでもすべての色がかなり滑らかなグラデーションを持っているわけではないので、目の色を選択しました。色が多いほど、フィールド上でボールを色で区別することが難しくなります。15色で十分だそうです。

カウンター


カウンターの主な秘密は数字のテープにあります。各リボンは、1桁しか表示されない長方形の中に隠されています。この長方形のエッジはわずかに暗くなっており、まるでテープではなくディスクのように丸みを帯びています。スクロールアニメーションの速度は、カウンターの残りの増分に比例し、適切な値によって制限されます。最後の数字は1秒回転しています。


音


サウンド（爆発とジャンプ）は合成です。それらは複雑でない数式によって生成されます。これにより、ゲームのサウンドのほとんどが単一周波数の周波数発生器を使用して生成された80年代の後味が生まれます。プラスには、プログラムのコンパクトさが含まれます。JavaScriptで記述された、そのようなサウンドエフェクトの単純なジェネレーターは、mp3ファイルよりもはるかに小さいです。ボールごとに、lowFreqとhighFreqのノートの独自のペアを選択し（ウィキペディアを参照）、次のようにオーディオバッファーを入力します。

    var soundUpBuffer = audioCtx.createBuffer(1, frameCount, sampleRate);
    for (var i = 0; i < frameCount; i++) {
      var x = i / frameCount; // from 0.0 to 1.0
      var time = duration * x;
      var volume = Math.sin(x * Math.PI);
      var freq = lowFreq + Math.sqrt(x) * (highFreq - lowFreq);
      buffer[i] = volume * Math.sin(2 * Math.PI * freq * time);
    }

ここで、曲線Xに沿っhighFreqへlowFreqから周波数変更半分、およびボリューム罪（πX）。

爆発時には、式は1 /（1 + 10 * x + 1000 * x * x）に従って減衰し、周波数は1〜5オクターブからランダムに選択されます。

アダプティブダイアログボックスのサイズ


ダイアログボックスのサイズを選択するアルゴリズムを実装しました。これにより、ダイアログボックスができるだけ多くのスペースを占め、表示領域の境界を超えないようになります。問題は、デバイスごとにダイアログのサイズが異なることです。場合によっては、ダイアログが小さすぎたり、画面に入らないことがあります。ダイアログのサイズをフォントサイズ（CSS単位em）に依存するようにしました。次に、フォントサイズを並べ替えて、ダイアログが画面のサイズの95％以下になるようにします。したがって、アダプティブCSSの事例を作成する必要はありませんでした。


小さなプログラムサイズ


ゲーム全体は、約150Kbの1つのhtmlファイルに収まります。約3000行のソースコード（もちろん、サーバー側はありません）。JavaScriptライブラリーやフレームワークは使用されませんでした-石器時代のように、すべてが自然です。サードパーティのAPIを使いたくない場合もあれば、自転車の発明家や新しいレーキテスターの寿命を単純化するフレームワークの量を理解することが役立つ場合もあります。最初にこれまたはそのアイデアを実装する方法を独自に考え、次に他の人がそれをどのように実行したかと比較することは有用です。たとえば、Androidのバージョンは 250Kbしかかかりません。これは、薄いWebViewラッパーと、さまざまな電話解像度用のpng-shekのパックで構成されています。ちなみに、png-shkiは1つのベクターxml-koyに置き換えることができます。それはまさにミニマリズムの精神で判明したでしょう。

フォーラム
