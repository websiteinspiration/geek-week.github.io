Cython：30倍を超えるPythonコードアクセラレーション

Pythonは多くのプログラマーが愛する言語です。この言語は非常に使いやすいです。問題は、Pythonで書かれたコードは直感的で読みやすいということです。ただし、Pythonについての会話では、この言語に関する同じ不満をよく耳にします。特にCの専門家がPythonについて話しているときは、「Pythonは遅い」と言われます。そして、そう言う人々は真実に対して罪を犯しません。

他の多くのプログラミング言語と比較して、Pythonは確かに遅いです。さまざまな問題の解決におけるさまざまなプログラミング言語のパフォーマンスを比較したテスト結果は次のとおりです。 Pythonプログラムを高速化する方法はいくつかあります。たとえば、ライブラリを使用できます



複数のプロセッサコアを使用するように設計されています。 Numpy、Pandas、またはScikit-Learn を使用している場合は、科学計算でGPUを使用できるRapidsソフトウェアパッケージを確認することをお勧めします。

これらの高速化手法はすべて、Pythonを使用して解決できるタスクを並列化できる場合に適しています。たとえば、これらは、行列を使用した予備的なデータ処理または操作のタスクです。

しかし、コードが純粋なPythonの場合はどうでしょうか。for絶対に使用しなければならない大きなループがあり、そのループ内で処理されるデータを順次処理する必要があるため、その実行を単純に並列化できない場合はどうでしょうか。Python自体を高速化する方法はありますか？

この質問に対する答えはCython（Pythonで書かれたコードを大幅に高速化できるプロジェクト）によって与えられます。

Cythonとは何ですか？


Cythonは、そのコアにおいて、PythonとC / C ++の間の中間層です。Cythonを使用すると、いくつかのマイナーな変更を加えた通常のPythonコードを記述できます。これを直接Cコードに変換します。

Pythonコードに対する唯一の変更は、その型に関する情報を各変数に追加することです。通常のPythonコードを記述する場合、次のように変数を宣言できます。

x = 0.5

変数を宣言するときにCythonを使用する場合、その型を指定する必要があります。

cdef float x = 0.5

この構成は、変数が浮動小数点数であることをCythonに伝えます。同じ原理で、変数はCで宣言されます。通常のPythonを使用して、変数の型は動的に定義されます。Cythonで使用される明示的な型宣言は、PythonコードをCコードに変換することを可能にするものです。重要なのは、Cでは変数の型を明示的に宣言する必要があるということです。

Cythonのインストールは非常に簡単です。

pip install cython

Cythonのタイプ


Cythonを使用する場合、2種類のタイプを区別できます。1つは変数用、2つ目は関数用です。

変数の場合は、次のタイプを使用できます。

  • cdef int a, b, c
  • cdef char *s
  • cdef float x = 0.5 （単精度数）
  • cdef double x = 63.4 （倍精度数）
  • cdef list names
  • cdef dict goals_for_each_play
  • cdef object card_deck

ここでは、実際にはC / C ++の型が示されていることに注意してください。

関数を使用する場合、次のタイプを使用できます。

  • def -Pythonからのみ呼び出される通常のPython関数。
  • cdef-通常のPythonコードから呼び出すことができないCython関数。このような関数は、Cythonコード内でのみ呼び出すことができます。
  • cpdef -CとPythonの両方からアクセスできる関数。

Pythonのタイプが判明したので、Pythonコードの速度を取り上げます。

Cythonを使用したコードの高速化


Pythonベンチマークの作成から始めましょう。これはfor、数値の階乗が計算されるサイクルになります。対応する純粋なPythonコードは次のようになります。

def test(x):
    y = 1
    for i in range(1, x+1):
        y *= i
    return y

この関数に相当するcythonは、元のバージョンと非常によく似ています。対応するコードは、拡張子がのファイルに配置する必要があります.pyx。コードに加える必要がある唯一の変更は、変数と関数のタイプに関する情報をコードに追加することです。

cpdef int test(int x):
    cdef int y = 1
    cdef int i
    for i in range(1, x+1):
        y *= i
    return y

この関数にはキーワードがあることに注意してくださいcpdef。これにより、この関数をPythonから呼び出すことができます。さらに、型はiループカウンターの役割を果たす変数に割り当てられます。関数で宣言されたすべての変数を入力する必要があることを忘れないでください。これにより、Cコンパイラは使用する型を認識します。

次にsetup.py、CythonコードをCコードに変換するのに役立つファイルを作成します。

from distutils.core import setup
from Cython.Build import cythonize

setup(ext_modules = cythonize('run_cython.pyx'))

コンパイルしましょう：

python setup.py build_ext --inplace

これでCコードが使用できるようになりました。

Cythonコードが置かれているフォルダーを見ると、Cコードを実行するために必要なすべてのファイル（ファイルを含む）が見つかりますrun_cython.c。興味がある場合は、このファイルを開いて、Cythonが生成したCコードを確認してください。

これで、超高速Cコードをテストする準備ができました。以下は、プログラムの2つのバージョンをテストおよび比較するために使用されるコードです。

import run_python
import run_cython
import time

number = 10

start = time.time()
run_python.test(number)
end =  time.time()

py_time = end - start
print("Python time = {}".format(py_time))

start = time.time()
run_cython.test(number)
end =  time.time()

cy_time = end - start
print("Cython time = {}".format(cy_time))

print("Speedup = {}".format(py_time / cy_time))

このコードは非常に単純です。必要なファイルをインポートします-通常のPythonファイルがインポートされるのと同じように、対応する関数を呼び出します。これは、通常のPython関数を常に使用する場合と同じ方法で行います。
