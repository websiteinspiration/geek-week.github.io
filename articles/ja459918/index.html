バッファオーバーフロー


バッファオーバーフローは、メモリに割り当てられたバッファの外側にデータを書き込む機能に基づくコンピュータプログラムの脆弱性であり、原則として、外部からのデータの制御されていない受信と処理が原因で発生します。言語による高レベルのスタックフレームテクノロジーの使用は、制御データとプログラムデータの混合につながります。

この記事では、スタックのバッファオーバーフローのみを分析します。このタイプのバッファオーバーフローはスタックスマッシングと呼ばれ、次の方法で悪用できます。

  • バッファの隣のメモリにあるローカル変数を上書きします。
  • スタックフレームの戻りアドレスを書き換えます。
  • 関数ポインタまたは例外ハンドラを書き換えます。
  • 別のスタックフレームからパラメーターを上書きする。

このタスクでは、ローカル変数を書き換える方法を使用します。次の例でその本質を検討してください。

#include <stdio.h>
#include <string.h>

int main(){
        char pass[9] = "p@ssw0rd\x00";
        char buf[9];
        printf("Input password: ");
        scanf("%s", buf);
        if(!strcmp(pass, buf)) 
                printf("Login ok!!!\n");
        else 
                printf("FAIL...\n");
        return 0;
}

pass 変数は以前に定義されているため、buf変数はオーバーフローする可能性があります。bufに9バイトを超える値を入力すると、pass変数のデータが上書きされます。したがって、たとえば11111111 \ x0011111111 \ x00のようなプログラムを渡すことにより、パスワードを自分のものに「変更」することができます。





仕事の解決策


署名bofの付いたアイコンをクリックすると、ソースコード、プログラム自体、およびTCP接続のアドレスとポートが提供されます。



ソースコードを見てみましょう。



コードからは、プログラムが文字列を受け入れることがわかりますが、既に配線されているキーと制御値が比較されます。しかし、入力は制御されておらず、キーはバッファの前に定義されているため、バッファをオーバーフローさせてキーを上書きすることができます。このためには、メモリ内の変数の相対位置を決定する必要があります。

プログラムの分析には、カッターを使用します。 Cutterを開き、実行ファイルへのパスを指定します。







Cutterはすぐにエントリポイントに送ります。関数のリストで、mainを選択します。



メインでは、関数の呼び出しを確認し、関数の名前をダブルクリックして開きます。



関数で使用される変数と現在のスタックフレーム（ebp）のベースを基準にしたそれらのアドレスを反映する関数コードの前にコメントがあります。ご存知のとおり、バッファはvar_2ch変数で、キーはarg_8hです。





上書きする必要があるバイト数を計算します。これを行うには、アドレス間の違いを見つけます。



したがって、プログラム0x34に任意のバイトを送信してから、例として参照値を追加する必要があります。便宜上、pwntoolsライブラリを使用します。

from pwn import *

conn = remote('pwnable.kr', 9000)

payload = 'A' * 0x34
payload += '\xbe\xba\xfe\xca'

conn.send(payload)

conn.interactive()

シェルを取得してフラグを表示します。



その結果、ポイントを獲得します。
