Angularプロジェクトをゼロから作成および構成する方法


記事は2020年2月29日にベストプラクティスに従って更新されました。


短い紹介


開発の開始時にプロジェクトが十分に厳密に構成されていない状況に遭遇することがよくあります。これは大きな間違いです。リンターの設定を変更したり、将来スクリプトの種類を確認するための追加オプションを組み込んだりすると、チーム全体にとって本当に苦痛になる可能性があるからです。したがって、この一般的な間違いを犯さないで、最初からできるだけ厳密にプロジェクトを構成してください。


開発環境の準備


ギット


開発環境に最初にインストールする必要があるのはGitです。オペレーティングシステムにGitをダウンロードしてインストールします。


その後、個人データを設定する必要があります。この情報は各コミットに含まれ、チームメンバーはプロジェクトのコード行の作成者を見つけることができます。


//    (  )  
$ git config --global user.name "John Doe"
$ git config --global user.email johndoe@example.com

//           
$ git config user.name "John Doe"
$ git config user.email johndoe@example.com

./tools/commit-msg.js


#!/usr/bin/env node

const fs = require('fs');
const { execSync } = require('child_process');

const message = fs.readFileSync(process.env.HUSKY_GIT_PARAMS, 'utf8').trim();
const currentBranch = getCurrentBranch();

fs.writeFileSync(process.env.HUSKY_GIT_PARAMS, `${currentBranch}: ${message}`);
process.exit(0);

function getCurrentBranch() {
  const branches = execSync('git branch', { encoding: 'utf8' });
  return branches
    .split('\n')
    .find((b) => b.charAt(0) === '*')
    .trim()
    .substring(2);
}
