前書き


弊社では、フリーランスとクライアント間のライセンス契約やその他の契約を自動的に作成、管理、安全に保存できるサービスを作成しています。

この問題を解決するために、オープンソースソリューションを含む自然言語処理の分野で数十のソリューションを試してみました。名前付きエンティティを認識するためのライブラリであるオープンソースPythonでの経験を共有したいと思います。

認識される名前付きエンティティ


問題自体について一言。Named Entity Recognition（NER）は、人間の言語処理テクノロジのブランチであり、そのソフトウェア実装により、音声およびテキスト内の目的のカテゴリの単語やフレーズを見つけることができます。最初は、地理的な名前、人の名前、組織、住所でしたが、現在、この概念は大幅に拡張されており、NERの助けを借りて、テキストで相対および絶対日付、数値、数値などを調べています。
名前付きエンティティの識別は、人間の言語への「ゲートウェイ」です。これにより、人の意図を識別して処理し、彼のスピーチと現実世界の単語のつながりを確立することができます。


言語の不平等


まず、さまざまな言語のソフトウェアソリューションにおける明らかな不平等に注目したいと思います。したがって、ほとんどの開発（ロシアのプログラマーが作成したものを含む）は英語で動作します。バハサ、ヒンディー語、またはアラビア語の既製のモデルを見つけることは、恩知らずの仕事です。

ヨーロッパの言語は、最も人気のあるライブラリに少なくとも表示されます;アフリカの言語は、原則として現代の自然言語処理には存在しません。一方、私自身の経験から、アフリカ大陸は巨大で豊かな市場であり、この態度はおそらく市場の惰性であると私は知っています。

品質に驚くべきロシア語の解決策はいくつかありますが、英語を処理するために「構築された」開発されたライブラリーほどの商業力や学術的可能性を感じていません。

NLTKライブラリ


NLTKは自然言語処理のための古典的なライブラリであり、使いやすく、長期間の学習を必要とせず、学生の問題を解決するときに発生する可能性のあるタスクの99％を実行します。 
import nltk
nltk.download('punkt')
nltk.download('averaged_perceptron_tagger')
nltk.download('maxent_ne_chunker')
nltk.download('words')
for sent in nltk.sent_tokenize(english_text):
   for chunk in nltk.ne_chunk(nltk.pos_tag(nltk.word_tokenize(sent))):
      if hasattr(chunk, 'label'):
         print(chunk)

スタンフォードコアNLP




NLTKの機能を拡張する1つの方法は、Stanford CoreNLPのクラシックJavaライブラリをクラシックPythonライブラリとともに使用することです。要件は比較的低く、品質は大幅に向上します。
from nltk.tag.stanford import StanfordNERTagger
jar = "stanford-ner-2015-04-20/stanford-ner-3.5.2.jar"
model = "stanford-ner-2015-04-20/classifiers/" 
st_3class = StanfordNERTagger(model + "english.all.3class.distsim.crf.ser.gz", jar, encoding='utf8') 
st_4class = StanfordNERTagger(model + "english.conll.4class.distsim.crf.ser.gz", jar, encoding='utf8') 
st_7class = StanfordNERTagger(model + "english.muc.7class.distsim.crf.ser.gz", jar, encoding='utf8')
for i in [st_3class.tag(english_text.split()), st_4class.tag(english_text.split()), st_7class.tag(english_text.split())]:
  for b in i:
    if b[1] != 'O':
        print(b)

StanfordNlpのStanzaは、スタンフォード大学から2020年に開発者に贈られたものです。Spacyに欠けていたのは、多言語性、使いやすさと言語への深い没入感でした。
コミュニティがこのライブラリをサポートしている場合は、最も人気のあるライブラリになる可能性が高くなります。
import stanza
stanza.download('en')
def stanza_nlp(text):
  nlp = stanza.Pipeline(lang='en', processors='tokenize,ner')
  doc = nlp(text)
  print(*[f'entity: {ent.text}\ttype: {ent.type}' for sent in doc.sentences for ent in sent.ents], sep='\n')
stanza_nlp(english_text)
