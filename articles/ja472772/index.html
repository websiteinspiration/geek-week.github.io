同様のインシデントとクレームを検索します。指標と最適化

前回の記事では、同様のアプリケーション用の検索エンジンについて説明しました。ローンチ後、最初のレビューを受け始めました。アナリストはいくつかの推奨事項を気に入り推奨しましたが、推奨しませんでした。


次に進んでより良いモデルを見つけるには、まず現在のモデルのパフォーマンスを評価する必要がありました。また、2つのモデルを相互に比較できる基準を選択する必要がありました。


カットの下で、私は話します：


  • おすすめに関するフィードバックの収集
  • 推奨事項の質を評価するための指標の開発
  • モデル最適化サイクルの構築
  • 受け取った洞察と新しいモデル

フィードバック集


アナリストから明確なフィードバックを収集するのが理想的です。提案された各インシデントの推奨はどれほど適切か。これにより、現在の状況を把握し、定量的な指標に基づいてシステムを改善し続けることができます。


非常にシンプルな形式でレビューを収集することが決定されました：


  • 分析しているインシデントの数
  • 推奨インシデント数
  • おすすめレビュー：良い/悪い

「投票」（パラメーター付きのGETリクエストを受け入れ、情報をファイルに保存する小さなプロジェクト）は、推奨ブロックに直接配置されたので、アナリストは「良い」または「悪い」のいずれかのリンクをクリックするだけでフィードバックをすぐに残すことができます。

class BinarizerClassifier(Binarizer):

    def transform(self, x):
        return np.array([-1 if _x > self.threshold else 1 for _x in np.array(x, dtype=float)]).reshape(-1, 1)

    def predict_proba(self, x):
        z = self.transform(x)

        return np.array([[0 if _x > 0 else 1, 1 if _x > 0 else 0] for _x in z.ravel()])

    def predict(self, x):
        return self.transform(x)
