TeXにおけるユークリッドのかなり空想的な「始まり」



「1847年にアイルランドのクリビンによって書かれた奇妙な骨董品h██nyaはもちろん、これが起こるのは良いことです」 Misha Verbitsky

16年目に、オリバーバーンの解釈におけるユークリッドの「始まり」が私の目を引きました。この本の特徴は、「ABCトライアングル」のようなレタリングの代わりに、例えば、対応するトライアングルのある画像など、建設パーツのテキストサムネイルに直接配置されていることです。そのような本を想像することができる限り、それは19世紀半ばの地獄のような作品であり、適切な道具を使って同じくらい簡単に、今あるはずです。そして、一般的に、私はこれを確認することにしました。



イラストレーターで単にイラストを描き、デザインでレイアウトを構成するオプションは、すぐに退廃的なものとしてマークされました。Illustratorで幾何学的構成を実行および編集することは、それが非常に便利であると言っているわけではありませんが、サムネイルをメイン構成に自動的にリンクする美しい方法を考えたことさえありません。このデザインは、このタイプのレイアウトには適していますが、非常に多くの写真が表示されているリンクバーの表示を抑制することが約束されています。それで、私はためらうことなく、すでにおなじみのMetaPostに目を向けました。幾何学の構築は非常に簡単です（もちろん、geogebraのようなものではありません）。ラテックスは、構成方法が比較的明確です。ただし、Latechはメタポスト用のライブラリとの互換性がないため、すぐに使えるメタポストに対応するConTeXtに置き換えられました。

一般的にどのように機能しますか？


一般に13の部分で「開始」-本ですが、バーンは最初の6部だけを処理しました。本は主に「文」（定理と問題）で構成されています。それぞれは構造（通常は1つ）とこの構造を参照するテキストで構成されています。

構造を説明するために、メタポストの新しいインスタンスを作成するマクロをConTeXtに導入しました。メタポストには、構造を説明するためのあらゆる種類の関数があります。次のようになります。

\defineNewPicture{ %     
    pair A, B, C, D; %       
    numeric d;
    d := 2u;
    A := (0, 0);                %
    B := A shifted (d, 0);      %    
    C := A shifted (0, -d);     %
    D := A shifted (d, -d);     %
    byAngleDefine(B, A, C, byblack, 0);     %   
    byAngleDefine(D, B, A, byblue, 0);      %  ,
    byAngleDefine(C, D, B, byred, 0);       %  ,
    byAngleDefine(A, C, D, byyellow, 0);    %  
    draw byNamedAngleResized();     %     
    byLineDefine(A, B, byred, 0, 0);        %   
    byLineDefine(B, D, byyellow, 0, 0);     %  ,
    byLineDefine(D, C, byblack, 0, 0);      %  ,  
    byLineDefine(C, A, byblue, 0, 0);       %  
    draw byNamedLineSeq(0)(AB,BD,DC,CA);    %    
}
\drawCurrentPicture %



文のテキストについては、メインの構成と同じインスタンスで画像を描画および表示するマクロを作成しました。それらは、一般的なケースでは、任意のメタポストコードを実行しますが、ほとんどの場合、必要な要素の名前（角度、セグメントなど）を引数として受け取ります。たとえば、次のようになります。

いくつかの機能


本の写真は気取らないが、特別な注意が必要なものもある。

セグメントがその端で互いに接触している場合、それは何らかの形でうまく配置されている必要があります。現在、2つのセグメントのみのコンパウンドが特に処理され、残りはそのような接続の下に置くことができます。オリジナルには線を接続する方法が少なくとも2つありますが、再現する必要はないように思えました。



角度はセクターを使用して描かれます。角度が十分に小さい場合、同じ半径を使用すると、セクターが短すぎるように見え、それを大きくすることには意味があります。これで、半径は60度の角度で一定のままになり、角度が小さい場合の式は次のようになります。r/a/60。



美しくするために、破線は塗りつぶされた部分で開始および終了する必要があります。そのため、ハッチングはセグメントに整数回フィットするようにわずかにスケーリングされます（イラストレーターにも同様のものがありますが、行の最初と最後に半分の長さのストロークがあります）。



テキストのセグメントをさまざまな方法で表示できます。セグメントをすべて同じ長さにして、色だけを変更できますが、長さをどのように再生してもかまいません。バーンは両方の方法で見つかります。私はこのためのこの式を持っています：L′=Lal1−aどこ L′ -テキスト内のセグメントの長さ L -最初のセグメントの長さ l -セグメントの希望の長さと a -0から1までの特定の数。 a=0その後 L′=l、つまり、どのセグメントも長さで表示されます l。もしa=1、 L′=L。もし0<a<1次に L<l、 L′>L と L>l、 L′<Lつまり、短いセグメントは長くなり、長いセグメントは短くなります。これは、セグメントの相対的なサイズを維持したいが、2センチのセグメントの横にある0.5ミリメートルのセグメントを取得したくない場合に必要です。 （実際、すべてが少し複雑ですが、意味はこれです）。



バーンは手紙を拒否しましたが、有名な退屈なエドワードタフティの本の 1つで、バーンは元気だったとのことでしたが、手紙を返してくれたら嬉しいです。そして、すべて同じなので、ほとんどすべてが自動なので、文字の場所がありました。デフォルトでは、ポイントには、それらの座標が格納されている変数と同じ方法で名前が付けられます。ラベルは、ポリゴンの頂点、セグメントの端、またはその他の方法で配置できます。もちろん、それらは自由にオンとオフを切り替えることができます（もちろん、レイアウトは同時にクリープします）。



しかし、ほとんどのコードは、読者よりも作成者/編集者にとってより目立つものに専念しています。これは、たとえば、角度とセグメントの同義名の正しい認識、いくつかのポリゴンの周りの文字の自動配置、テキスト内の特定の角度でのセグメントの断片の自動描画などです。

オリジナルには頭文字とビネットがあります。一方で、それらはほとんど作業なしで（少なくとも精神的に）描画できますが、ランダムなパターンの最初の文字とビネットを自動的に生成するという、より興味深い（絶望的ではありますが）タスクを取り上げることにしました。 1つ目は楽しいことです。2つ目は、ロシア語（以下で説明）への翻訳では、キリル文字に文字の最初のスタイルを適合させる必要があり、その活動は私の計画には含まれていません。したがって、本を組み立てると、すべての文字とビネットのリストが作成され、このリストから別のメタポストスクリプトが、これらすべての完全なセットを（非常にゆっくりと）描画します。最初の文字はすべて異なり、パターンは繰り返されません。



原理は簡単です。可能な限り大きなカールを文字の部分とフレームに配置します。これは数回発生し、すでに描かれた渦巻きは次の反復に含まれます。次に、「葉」も同じように表示されます。さまざまなタイプのプロセスの形状とプロパティを変更できます。



結果に満足しているとは言えませんが、アルゴリズムを徐々に改善し、最高のものを期待します（ただし、生成されたものの代わりに任意の文字で画像を吐き出したり、スリップしたりすることができます）。おまけとして、スクリプトはカールからランダムタイルを生成できます。



転送


エラーを見つけるために、そして実際、私はロシア語に翻訳し始めました。私の参照点は、モルデューカイボルトフスキーの「始まり」の翻訳でしたが、バーンが証拠に大きく変化しすぎて多くのことに役立っていないことがすぐに明らかになりました。正直なところ、最初は袖の後ですべてを翻訳しました。特に導入部は、それを手に入れるために完全に翻訳しました。最近、私の手が少し粗さを滑らかにするために手を伸ばしました。

不愉快な5冊目の本（それが判明したように、他の本とは別に、やや早い段階で組み立てられた）を除いて、それは特に難しくありませんでした。翻訳（プロセス自体）は、私と著者の両方の多くの異なる学校を見つけるのに本当に役立ちました。命題9の構成を考えてみましょうバーンの6冊目の本はテキストに対応していないため、証明が機能しないため、そこですべてをやり直す必要がありました。

好奇心から：著者が彼の方法を称賛する序文で、彼は最初にホレスに言及し、議論として他の感情に対する視覚の優位性についての彼の詩を引用しました。そして数ページ後、彼は説得力のために「現代の詩人」も追加することに決めました。実際、「現代詩人の詩」はホレスの同じ行の別の翻訳であることが判明しました。ここではもちろん、すべてをそのままにする必要があり、脚注に著者の間違いを示していますが、幸い妻は適切な節を見つけました英語で、ビジネスを始めた本当に現代的なバーンの作家（たとえあなたがこれを行うことができないと言っていても）。英語版ではそのままに。



ロシア語版はまた、その中の小さな文字がデフォルトで含まれているという点で異なり、一般的に英語版よりもオリジナルと比べてより小さな変更です。確かに、ここで誰かが私に文字も含めて英語版を作るように頼んだ、そして私もそこにそれらを含めることを考えている：それらなしで、この本はすでに存在している。

別の同志がこの本をポーランド語に翻訳し始め、問題について多くの貴重なことを書きました（たとえば、角度のスケーリングについて-これは彼の提案です）。翻訳すれば素晴らしい。

小計


この本をいじくり回して、私は言うべき次のことを持っています。本文中の写真はとても読みやすいので、今では他の場所でもこのテクニックを使っています。私がこのプロジェクトに丸一日取り組んだ後の夜にプロジェクトに携わっていたことを考えると、それはかなりすぐに判明しました。最初の英語版は約6か月、すべての計画とプログラミング、ロシア語翻訳の初版は3か月でした。確かに、私はまだあちこちで浅瀬を修正します（さらに最近では、妻が誤字のためにロシア語を読み始め、最初のプルリクエストとほぼ同じように赤面しています）。

githubに最初のバージョンが公開されて以来、キャンペーンは開始および終了しました「Byrneの仕事を完了する」ための資金を調達するためのキックスターターで、つまり、著者が理解しているように、13冊すべての「Beginnings」の本のデザインは同じ脈絡にあります。どうやら、彼らが描くデザインのレイアウトは不明確です。これまでのところ、書籍は発送されていませんが、すぐに発送されます。うまくいくといいですね。さらに最近では、アメリカ人デザイナーの1人が最初の6冊の本の素晴らしいWebバージョンをリリースしました。作品についてのストーリーの記事から判断すると、彼はすべてをイラストレーターに手作業で描いた。

予定


バーンでは、最初からそれのためのツールが予見できなかったので、それのためのツールが11から13まで徐々にそれらに取り組むためのツールにbirnifitsirovatの本「プリンシピア」を徐々に決定することになりました。これまでのところ、11冊目の本の半分以上が完成しているため、3次元構造と平面への投影を記述するための関数がすでにいくつかあります。ただし、平均してボリュームのある構造はフラットな構造よりもはるかに複雑であり、これまでのところ、バーンのアプローチがどれだけ機能するか、またはメタポストでそれらを作成するのに十分便利であることが判明するかどうかについては、はっきりとはわかりません。



テキスト内の画像は、多くの場合カーニングが必要です。似ていますが、より複雑です。隣接するライン上の大きな画像はラインを非常に広く広げます。これは、ラインが互いに重なり合っている場合にのみ正当化されます。これらのことを自動化する方法はまだわかりません。手動ですべてを取得するのは非常に退屈です。



メタポストは、latexからでも個別にでも起動できます。したがって、将来的には、より一般的な環境で同じことができるように、ConTeXtと同じ、latex用のマクロを作成する予定です。理論的には、デザインなどでメタポストを使用することに煩わされる人はいません。これは、画像を個別に収集して手動でリンクするだけで実行できますが、IDのスクリプトをいくつか作成して、これが自然に発生するようにします。後者はある種の倒錯のように見えますが、そのような可能性を考慮しないとあまりにしばしば思い浮かびます。

さて、そして最も重要なことですが、このすべての機械を「始まり」よりも現代的で実用的なものに適用したいと考えています。
