UnityのMVCとスクリプト可能なオブジェクト。パート1

MVC（Model-View-Controller）は、アプリケーションデータ、ユーザーインターフェイス、および制御ロジックを3つの個別のコンポーネントに分割して、それぞれを個別に変更できるようにするスキームです。開発者Cem Ugur Karacamは、Unityでのプログラミング経験を共有し、スクリプト可能なオブジェクトについて簡単に話しました。dev.toで公開された彼の記事の翻訳を紹介します。



スクリプト可能なオブジェクトを使用してUnityにMVCを実装することについての私の話の最初の部分へようこそ。この記事では、スクリプト可能なオブジェクトの操作の基本についてのみ書きました。一般的に、テキストは単純で明確です。

まず、Unity開発者がスクリプト可能なオブジェクトについて何を言っているかを調べます。

  • これは、シーン内の別のオブジェクトにアタッチしてはならないオブジェクトを作成するために継承できるクラスです。
  • このような作成されたオブジェクトの最大の利点は、データを保存するためだけに使用する場合です。

質問するのは理にかなっています。なぜスクリプト可能なオブジェクトなのか？

スクリプト可能なオブジェクトはUnityエンジンの機能の一部であるため、そのようなオブジェクトには、エディターを操作するための多くの組み込み機能があります。

Unityでプロジェクトを開発するときに使用するアプローチが好きです。独自のツールを作成し、デフォルトでインストールされるツールを簡単にカスタマイズできます。スクリプト可能なオブジェクトを使用すると、データ用のカスタムコンテナーを作成し、それらをアセットとしてプロジェクトに保存できるため、機能が大幅に拡張されます。



上の画像はUnityでスクリプト可能なオブジェクトを使用する例です。プロジェクトで作成された単一の.assetファイルには、複数のシステムの操作に必要な情報が含まれており、それらの設定の保持が保証されている場合があります。詳細を知りたい場合は、「輝かしいスクリプト可能なオブジェクト革命でMonoBehaviour専制政治を打倒する」と呼ばれるSO（スクリプト可能なオブジェクト）に関するプレゼンテーションを含む優れたビデオがYouTubeにあります。



別の素晴らしい例を上に示します。ここで、ゲームオブジェクトはInventoryオブジェクトを参照し、Save Systemはそれらすべてを管理します。この画像は、別の優れた講義「スクリプト可能なオブジェクトを使用したゲームアーキテクチャ」からのものです）、それを読むことを強くお勧めします。

もう1つの優れた機能は、スクリプト可能なオブジェクトを理解している場合、JSONやXMLなどの他のデータストレージ形式を使用したくないことです。SOは、Unityにデータを保存するための最も便利な形式であるためです。

それでは、スクリプト可能なオブジェクトの動作を見てみましょう。

using UnityEngine;

public class ItemData : ScriptableObject
{
    public string itemName;
    public ItemType type;
    public float attack;        
}

public enum ItemType
{
    Dagger,
    Axe,
    Sword,
    Staff
}

私たちはItemDataRPGゲームに典型的な名前といくつかのプロパティを持つクラスを書きました。次に、プロジェクトにそのようなアイテムをいくつか作成し、スクリプトからそれらのアイテムにアクセスして、値をコンソールに表示します。しかし、最初に、コードに1つの重要な行を追加します。

using UnityEngine;

[CreateAssetMenu]
public class ItemData : ScriptableObject
{
    public string itemName;
    public ItemType type;
    public float attack;        
}

public enum ItemType
{
    Dagger,
    Axe,
    Sword,
    Staff
}

CreateAssetMenuクラスの前に追加した 属性はItemData、メニューからクラスが保存される.assetファイルを作成することをUnityに伝えます。それ以外の場合は、マウスの右ボタンまたはプロジェクトフォルダーの[作成]ボタンを使用してこれを行うことはできません。





名前の付いたフォルダを作成し、このフォルダにItemsオブジェクトを作成してみてItemDataください。

情報をコンソールに出力するメソッドをそれに追加します。また、配列の境界を超えないようにチェックを実行して、配列を円でバイパスします。

using UnityEngine;

public class Inventory : MonoBehaviour
{
    public ItemData[] inventory;

    int index = 0;

    public void NextItemInfo()
    {
        if (index > inventory.Length)
        {
            index = 0;
        }

        Debug.Log("Item name: " + inventory[index].name);
        Debug.Log ("Attack power: " + inventory[index].attack);

        switch(inventory[index].type)
        {
            case ItemType.Axe: 
            Debug.Log("Item type: Axe"); 
            break;

            case ItemType.Dagger: 
            Debug.Log("Item type: Dagger");
            break;

            case ItemType.Staff: 
            Debug.Log("Item type: Staff");
            break;

            case ItemType.Sword: 
            Debug.Log("Item type: Sword"); 
            break;
        }

        index ++;
    }

}

メソッドへの呼び出しをスペースバーに割り当てます。Unityでは、これはとても簡単です。クラスのおかげで、必要なキーが押されたかどうかをInputメソッドでチェックできUpdateます。

using UnityEngine;

public class Inventory : MonoBehaviour
{
    public ItemData[] inventory;

    int index = 0;

    private void Update() 
    {
        if (Input.GetKeyDown(KeyCode.Space))    
        {
            NextItemInfo();
        }
    }

動作します！次に、スクリプト可能なオブジェクトがデータだけでなく、それらを操作するためのメソッドも格納できることを説明します。たとえば、クラスの価格を計算するメソッドを追加してみましょうItemData。詳細については、こちらとこちらのドキュメントをご覧ください。

using UnityEngine;

[CreateAssetMenu]
public class ItemData : ScriptableObject
{
    public string itemName;
    public ItemType type;
    public float attack;        

    public float GetPrice()
    {
        return attack * 40;
    }
}

次に、クラスで新しいメソッドを使用しますInventory。

using UnityEngine;

public class Inventory : MonoBehaviour
{
    public ItemData[] inventory;

    int index = 0;

    private void Update() 
    {
        if (Input.GetKeyDown(KeyCode.Space))    
        {
            NextItemInfo();
        }
    }

    public void NextItemInfo()
    {
        if (index == inventory.Length)
        {
            index = 0;
        }

        Debug.Log("Item name: " + inventory[index].name);
        Debug.Log ("Attack power: " + inventory[index].attack);

        switch(inventory[index].type)
        {
            case ItemType.Axe: 
            Debug.Log("Item type: Axe"); 
            break;

            case ItemType.Dagger: 
            Debug.Log("Item type: Dagger");
            break;

            case ItemType.Staff: 
            Debug.Log("Item type: Staff");
            break;

            case ItemType.Sword: 
            Debug.Log("Item type: Sword"); 
            break;
        }
        Debug.Log("Item price: " + inventory[index].GetPrice());

        index ++;
    }

}
