ZabbixでSNMPv3を介してネットワーク機器を監視する

この記事では、SNMPv3プロトコルを使用してネットワーク機器を監視する機能について説明します。SNMPv3についてお話しします。Zabbixで本格的なテンプレートを作成した経験を共有し、大規模ネットワークで分散アラートを整理するときに何ができるかを示します。SNMPはネットワーク機器を監視するためのメインプロトコルであり、Zabbixは多数のオブジェクトを監視し、大量の受信メトリックを要約するのに優れています。

SNMPv3について一言


SNMPv3の予約とその使用の機能から始めましょう。SNMPタスクは、簡単なコマンドをネットワークデバイスに送信してネットワークデバイスと基本的な管理を監視します（たとえば、ネットワークインターフェイスのオンとオフを切り替えたり、デバイスを再起動したりします）。

SNMPv3プロトコルと以前のバージョンの主な違いは、次のような従来のセキュリティ機能[1-3]です。

  • 認証。要求が信頼できるソースから受信されたことを決定します。
  • 暗号化、第三者によって傍受されたときに送信されたデータの開示を防止します。
  • 完全性、つまり、パケットが送信中に改ざんされなかったことの保証。

SNMPv3は、セキュリティモデルの使用を意味します。認証戦略は、特定のユーザーとユーザーが属するグループに対して設定されます（以前のバージョンのSNMPでは、サーバーから監視オブジェクトへのリクエストで、「コミュニティ」のみが比較され、「パスワード」のテキスト文字列がクリアテキストで送信されました。 （プレーンテキスト））。

SNMPv3には、セキュリティレベルの概念が導入されています。これは、機器の構成と監視オブジェクトのSNMPエージェントの動作を決定する許容可能なセキュリティレベルです。セキュリティモデルとセキュリティレベルの組み合わせにより、SNMPパケットの処理時に使用されるセキュリティメカニズムが決まります[4]。

SNMPv3を構成する


ネットワーク機器の監視には、監視サーバーと監視対象オブジェクトの両方で、SNMPv3プロトコルの同じ構成が含まれます。

シスコのネットワークデバイスの設定から始めましょう。最低限必要な設定は次のとおりです（CLIを使用する設定では、混乱を避けるために名前とパスワードを簡略化しています）。

snmp-server group snmpv3group v3 priv read snmpv3name 
snmp-server user snmpv3user snmpv3group v3 auth md5 md5v3v3v3 priv des des56v3v3v3
snmp-server view snmpv3name iso included

snmp-serverグループの最初の行-SNMPv3ユーザーのグループ（snmpv3group）、読み取りモード（読み取り）、および監視オブジェクトのMIBツリーの特定のブランチを表示するためのsnmpv3groupグループのアクセス権を定義します（以下の構成のsnmpv3nameは、グループのMIBツリーのブランチを決定しますsnmpv3groupがアクセスできるようになります）。

2行目のsnmp-server user-ユーザーsnmpv3user、snmpv3groupのメンバーシップ、認証md5（md5のパスワードはmd5v3v3v3）と暗号化des（desのパスワードはdes56v3v3v3）の使用を定義します。もちろん、desの代わりにaesを使用することをお勧めします。ここでは、例としてそれを示します。また、ユーザーを定義するときに、このデバイスを監視する権限を持つ監視サーバーのIPアドレスを規制するアクセスリスト（ACL）を追加できます。これもベストプラクティスですが、例を複雑にすることはしません。

snmp-serverビューの3行目は、snmpv3name MIBツリーブランチを定義するコード名を定義して、snmpv3groupユーザーグループがそれらを要求できるようにします。ISOでは、単一のブランチを厳密に定義する代わりに、snmpv3groupユーザーグループが監視オブジェクトのMIBツリーのすべてのオブジェクトにアクセスできるようにします。

Huawei機器の同様の構成（CLIでも）は次のとおりです。

snmp-agent mib-view included snmpv3name iso
snmp-agent group v3 snmpv3group privacy read-view snmpv3name
snmp-agent usm-user v3 snmpv3user group snmpv3group
snmp-agent usm-user v3 snmpv3user authentication-mode md5 
            md5v3v3v3
snmp-agent usm-user v3 snmpv3user privacy-mode des56
            des56v3v3v3

ネットワークデバイスをセットアップした後、SNMPv3プロトコルを介して監視サーバーからのアクセスを確認する必要があります。私はsnmpwalkを使用します。

snmpwalk -v 3 -u snmpv3user -l authPriv -A md5v3v3v3 -a md5 -x des -X des56v3v3v3 10.10.10.252



MIBファイルを使用して特定のOIDオブジェクトを要求するためのより視覚的なツールはsnmpgetです。

ネットワークインターフェイスの検出に進みましょう。ネットワーク機器の場合、これは最も重要な監視機能です。ネットワークデバイスには数百のインターフェイスが存在する可能性があるため、視覚化が乱雑になったりデータベースが乱雑になったりしないように、不要なインターフェイスを除外する必要があります。

より柔軟なフィルタリングのために、多数の検出可能なパラメーターを備えたSNMPの標準検出機能を使用します。
