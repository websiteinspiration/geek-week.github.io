誰がDNSルートゾーンの分散化を提案したか

エンジニアのグループは、ピアツーピアのルートDNSサーバーをブロックチェーンベースのネットワークに置き換えることを提案しました。ITコミュニティがこのイニシアチブについてどう思っているかをお伝えします。


写真- マーカスベングソン-Unsplash

DNSでブロックチェーンはどのように役立ちますか


証明センター（CA）のタスクは、サーバーへの接続が安全であること、およびこのサイトまたはそのサイトに対して発行されたSSL証明書が正当であることを確認することです。各認証センターには、証明書の検証の責任を他の組織に委任する権利がありますが、ブラウザユーザーは、特定のCAの信頼性や、CA /ブラウザフォーラムコンソーシアムのセキュリティ規制に準拠しているかどうかを検証できません。

証明書が危険にさらされている場合、これはMITM攻撃の機会を開きます。これはすでに起こっています-2011年にイランのハッカーが入れ替わりましたDigiNotar認証局が発行した500を超えるSSL証明書。その中には、Mozilla、Googleおよびその他の企業の証明書が含まれていました。 1か月以内に、攻撃者は30万人のユーザーのトラフィックを傍受しました。

一部のIT専門家は、1つの企業ICANNがDNSルートゾーンの暗号化キーを管理することを心配しています。彼女は独占者として行動し、彼女の条件を決定します-どのトップレベルドメイン名（TLD）が登録され、どのくらいの費用がかかりますか。したがって、新しいTLDの申請には185,000 ドルの費用がかかります。

証明機関への信頼の問題を解決し、ルートゾーンを分散させるために、Namebaseのエンジニアは、DNSシステムを編成するための代替アプローチに取り組み始めました。彼らは、ルートサーバーをハンドシェイクブロックチェーンネットワークに置き換えることを提案しました。

使い方


ブロックチェーンは、ドメイン情報を含むファイルのリポジトリとして機能します。分散ネットワークでそれらを保護するために、ビットコインの場合と同様に、作業証明アルゴリズムが使用されます。ドメインを登録するには、ユーザーがblockchainに対応するリクエストを送信する-これはどのようになります example.comの：

$ hsw-rpc createclaim example
{
  "name": "example",
  "target": "example.com.",
  "value": 1133761643,
  "size": 3583,
  "fee": 17900,
  "address": "ts1qd6u7vhu084494kf9cejkp4qel69vsk82takamu",
  "txt": "hns-testnet:aakbvmygsp7rrhmsauhwlnwx6srd5m2v4m3p3eidadl5yn2f"
}

とりわけ、リクエストは、サイト名、ドメイン名、およびブロックチェーンにエントリを登録するためにユーザーが採掘者に支払う意思がある金額を示します。支払いは、実用的なHNSトークンを使用して行われます。マイニングが完了すると（5〜20分かかります）、システムはドメインへの所有者権限を付与します。また、ウェブマスターは自分で暗号署名を付けることができるキーを受け取ります。このアプローチにより、従来の認定センターを放棄することができます。

HNSトークンは、ドメインの販売にも使用されます。トランザクションは公開オークションの形式で行われます-名前は、最も高い入札を行ったユーザーに転送されます。サイバースクワットを回避するために、ハンドシェイク開発者は、ブロックチェーンネットワークのAlexaレーティングに含まれる最初の10万サイトのドメイン名をすでに保護しています。。彼らの実際の所有者は、いつでもブロックチェーンネットワークに移行し、それに対する報酬を受け取ることさえできます。

意見


ハンドシェイクの作者によると、彼らのプラットフォームの機能は、TCP / IPプロトコルスタックWinton Cerfの開発者の1人によって積極的に評価されました。 1年前、彼自身が証明機関への信頼を高めるソリューションを実装することを提案しました。そして一般的に、ITコミュニティにおける分散ルートDNSシステムのアイデアはサポートされました。それだけでいくつかの興味深い可能性が開かれるからです。

ハンドシェイクにより、IPアドレスを新しいTLDに関連付け、トップレベルドメインをフルネームとして使用できます。たとえば、アドレスバーに「namebase」と入力して「namebase.io」に移動します。一部のハッカーニュース居住者は、機能が人気になる可能性は低いと言いますが。ドットのないサイトアドレスは異常に見え、ユーザーを混乱させます。


写真-Kaley Dykstra -Unsplash

また、HNでは、Handshakeなどのプロジェクトがすでに開始されていることが指摘されていました。NamecoinとENSがありました。そして、それらは広まっているわけではありません。 4年前、Namecoinデータベースに登録されている12万のドメイン名のうち、アクティブだったのは28のみでした。信じハンドシェイクは同じ運命を期待していること。

Namebaseのスペシャリストによると、アナログとは異なり、プラットフォームは従来のドメインネームシステムと競合せず、互換性があるとのことです。ユーザーが所有者がブロックチェーンネットワークに登録していない最も人気のある10万のサイトのいずれかのアドレスを入力しようとすると、ソフトウェアは要求を従来のDNSサーバーにリダイレクトします。

開発者分散システムとICANNプロトコルとの透明性と完全な互換性を維持する予定です。そして、ハンドシェイクの将来は、大企業が別のDNSソリューションに切り替えたいかどうかにかかっています。
