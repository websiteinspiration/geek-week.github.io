1. AWSローン


初めにすること！何かを始める前に、会社がAWSローンの対象であることを確認してください。これらのローンを取得する方法は次のとおりです。

AWSにローンを依頼する


だから私たちの場合でした！ 2016年、当社は急速に成長しました。 AWSの請求額が増え始め、AWSの使用を最適化するためのアイデアや時間もありませんでした。資金がないため、AWSからローンを受け取るか、行方不明になることに気づきました。 AWSのマネージャー向けのアカウントのケースを準備し、成長の速さとAWSがサイトでサポートする利点を示しました。 AWSからローンを受け取らなかった場合、資金がなく、Google Cloud（AWSへの移行のために100,000ドルのローンを提供した）に強制されることを彼に示しました。応答には約1週間かかり、AWSは12か月間使用するために100,000ドルを提供してくれました。 AWSにとってもお得でした！ローンの期限が切れたため、AWSに750,000ドル以上を費やしています。

プルーフオブコンセプトをテストおよび起動するためのクレジット


AWSまたはインスタンスタイプの新しいサービスをテストする場合は、アカウントマネージャーに融資を依頼してください。ローンの数は、毎月の費用とPOCを立ち上げるための全体的な努力によって異なります。これらのローンは通常3か月間のみ有効であり、POCの速度を確認する必要があります。私たちのケースでは、インスタンスのi3ファミリーをテストするために3,000ドルのローンを得ることができました。

スタートアップクレジット


会社を設立したり、スタートアップを立ち上げたりした場合、これは前進するための素晴らしい方法です。ローンの金額は、AWSケースの提示方法に応じて、1,000ドルから100,000ドルの範囲です。これらのローンを取得するにはいくつかの方法があります。

加速器


Startup Acceleratorが提供するボーナスの主な利点の1つ。これらのボーナスの1つは、クラウドプロバイダーからのローンです。受け取ることができる量は、あなたが表されているアクセラレータによって異なります。たとえば、YC、Alchemist、Wayra、500 Startupsなどの最高のものは、100,000ドルのAWSローンを提供します。私たちの場合、Alchemist Acceleratorに受け入れられ、10万ドルのローンを発行しました。残念ながら、それ以前はすでに他のローンを利用していたため、これらのローンを利用することができませんでした。たとえば、100,000ドルごとに2つのアクセラレータを受け入れる場合、200,000ドルは使用できず、100,000ドルしか使用できません。

ベンチャーファンドと天使のグループ


一部のWF /エンジェルグループは、ボーナスプログラムの一環としてローンを提供しています。私たちの場合、投資家の1人から2年間有効な$ 10,000のマーケティングローンをなんとか手に入れました。残念ながら使用できませんでした。以前に同額またはそれ以上のローンを受け取っている場合、企業はAWSプロモーションクレジットを使用できません。

すべての主要なクラウドプロバイダーがスタートアップにローンを提供


クラウド市場のすべての主要なプレーヤーは、スタートアップにローンを提供しています。AWS / Googleを使用すると、ローンは最大100,000ドルになります。Azureでは、3年間で360,000ドルを獲得できます。

2. AWSにコスト削減を実装する


簡単な部分は終わったので、今度は課題に取り掛かりましょう！ AWSでコストを削減する魔法の薬はありません。コストを監視するために一生懸命働き、定期的な手順に従う必要があります！クラウドのコストを削減することは、あなたの日常業務であるべきです！ 365日でどれだけ節約できるか想像してみてください。投資する時間は、組織の規模とAWSのコストによって異なります。私は1日約30分かけて自分のアカウントを監視し、コスト削減アクションを優先します。さらに、Back4Appエンジニアは、コスト削減の取り組みに1日あたり少なくとも1時間を費やしています。

最初に完了したイニシアチブ：コスト削減イニシアチブに優先順位を付けるために、Excelで簡単なワークシートを作成しました。

節約額を決定する主な基準：

  • 節約の可能性-節約による最高の収益
  • 難易度-労力と必要な作業を適用するのはどれほど難しいか
  • 所有者-誰がアクションを実行するか

私たちの場合、最大の節約はElastic Compute Cloud（EC2）の使用であり、これが私たちが理解し始めた最初の主題でした。

3. EC2-Elastic Compute Cloud


未使用のインスタンスをオフにする


EC2のコストを削減するために取られる最初のポイントは、未使用のインスタンスをオフにすることです。ここでは何も複雑ではなく、ほとんどの作業は非常に簡単です。すべてのインスタンスのリストを作成し、例からExcelファイルをロードして、それぞれを定義するだけです。この例では、インスタンスを3つの主要なカテゴリに分けました。

  • 継続的に動作する必要があるインスタンス
  • 停止できるインスタンス
  • 削除できるインスタンス

停止せずに、インスタンスを

削除し、可能であれば未使用のインスタンスを削除してください。それらを停止するだけで、EBSの使用に対する支払いは継続されます。

過剰なリソースを削減


このアクションはもう少し複雑で、インスタンスサイズを正しく選択することが絶対に必要です。正しいインスタンスが変更される可能性を高めるために、少なくとも1週間のデータを評価してください。このプロセスを制御するためのアプローチは、インスタンスを3つの主要なカテゴリに分類することでした。

  • 正しいサイズ
  • インスタンスサイズを縮小する
  • インスタンスサイズを増やす

インスタンスの履歴データを確認する方法は非常に簡単です。AWSコントロールパネルで、[サービス]> [EC2]> [インスタンス]> [実行中のインスタンス]に移動し、[インスタンス]> [モニタリング]を選択します。たとえば、CPU使用率メト​​リックを確認できます。

インスタンスサイズを順次

小さくするインスタンスサイズを段階的に小さくします。たとえば、サイズをt2.xlargeからt2.mediumに縮小しないでください。まず、t2.2xlargeからt2.xlargeに減らします。1週間を見て、使用されているリソースの数を確認し、新しいリソースで手順を繰り返します。これらの手順を実行することにより、エラーやダウンタイムの可能性を最小限に抑えることができます。

リザーブドインスタンスを使用する


インスタンスを予約すると、最大70％節約できます。インスタンスを予約すると、予約期間全体を通じて24時間年中無休の料金が発生することに注意してください。インスタンスを使用するかどうかに関係なく、お支払いいただきます。

冗長キャパシティには次の2つのタイプがあります。

標準-AWS Marketplaceで販売でき、コンバーチブルインスタンスよりも安価な別のタイプのインスタンスに変換できません。

コンバーチブル-AWS Marketplaceで販売することはできませんが、標準のインスタンスよりも高価な別のタイプのインスタンスに変換できます。

コンバーチブルインスタンスは混乱する可能性があります

コンバーチブルインスタンスは非常に困難です。インスタンスを変換するたびに、インスタンスのサイズを増やすことしかできません。次の状況を考え

ます。変換可能なリザーブドインスタンスであるt3.mediumを取得します。
t3.mediumをm5.largeインスタンスに変換することにしました。

あなたは確かに成功します！しばらくm5.largeインスタンスを使用した後、t3.mediumに戻る必要があります。

そして、これはすでに不可能です！

コンバーチブルインスタンスの使用量を減らすことはできませんが、そのために支払う金額を増やすことができます。そのため、小さな予約済みインスタンスから始めて、時間の経過とともにそれを増やしてください。消費量を削減したり、AWS Marketplace'eで販売したりできないため、大きなインスタンスから始めないでください。コンバーチブルリザーブドインスタンスを間違えた場合は、12か月または36か月の月額支払いが購読されます。Back4Appでは、常に最初にt2.nanoを購入し、必要に応じてそれを増やします。

次の理由により、コンバーチブルよりも標準のリザーブドインスタンスを使用します。

  • 彼らは安いです
  • 流動性が高く、AWSマーケットプレイスでいつでも販売できます。
  • AWS Marketplaceで割引

予約済みインスタンスにAWS Marketplaceを使用する


リザーブドインスタンスは素晴らしいですが、AWS Marketplaceでリザーブドインスタンスを購入することはさらに優れています！

AWS Marketplaceははるかに安くなる


AWS Marketplaceは証券市場のように見えます！AWSから直接購入するよりもはるかに手頃な価格で、標準のリザーブドインスタンスを購入できる場合があります。理由は簡単です。企業は、使用されなくなった冗長なインスタンスを所有することがありますが、その代金は引き続き支払います。配置価格を下げると、購入プロセスが大幅に加速します。

ここでは、Back4Appで何が起こったかの例を示します。4つのr4.largeリザーブドインスタンスを探していました。AWS Marketplaceで、オンデマンドの価格と比較して実効レートが75％安いインスタンスを見つけることができました。AWSは同じタイプの予約済みインスタンスを販売しましたが、62％の割引がありました。

AWS Marketplaceには柔軟な購入条件があります


AWS Marketplaceのもう1つの利点は、柔軟な使用期間でインスタンスを購入できることです。AWS自体は、リザーブドインスタンスの購入に12か月と36か月のオプションのみを提供しています。AWS Marketplaceははるかに柔軟です！たとえば、15か月の予約済みインスタンスを36か月と同じ価格で購入できます。一部の企業は長期的な義務（36か月）を好み、義務が満了する前にAWSマーケットプレイスで販売します。新しい顧客は、残りの期間だけインスタンスを使用します。次に例を示します。

  • 総取得時間：36か月
  • 最初の購入者：21か月間インスタンスを使用します
  • 2番目の購入者：マーケットプレイスで購入し、残りの15か月間使用します。

主な問題は、AWS Marketplaceで適切なインスタンスを見つけることです。インスタンスの検索を簡略化する優れた方法は、自分がいる場所だけでなく、いくつかのアクセスゾーンでインスタンスを探すことです。インスタンスを取得したら、それを変更して別のアクセシビリティゾーンに割り当てるか、リージョンインスタンスに変換できます。これは難しいことではありませんが、以下の指示に従ってください。

  • 手順1-リザーブドインスタンスダッシュボードに移動
  • ステップ2-リザーブドインスタンスの購入をクリックします
  • ステップ3-オファリングクラスで[標準]を選択します
  • ステップ4-ポップアップウィンドウの右上隅で、[容量を予約するオファリングのみを表示]を選択します
  • ステップ5-[検索]をクリックします
  • ステップ6-アベイラビリティーゾーン1bからインスタンスを取得する
  • ステップ7-インスタンスがアクティブになるのを待つ
  • 手順8-購入したインスタンスを選択します
  • 手順9-[アクション]ボタンをクリックします
  • ステップ10-[リザーブドインスタンスの変更]をクリックします
  • ステップ11-スコープ-アベイラビリティーゾーンまたはリージョンの変更
  • ステップ12-[続行]をクリックします
  • ステップ13-[変更を送信]をクリックします

これらの手順に従うと、任意のアベイラビリティーゾーンで標準リザーブドインスタンスを購入し、それをアベイラビリティーゾーン用に変更できます。

米国外の企業向けのAWSマーケットプレイス


米国の管轄外のユーザーに対する重要な説明。州の法的エンティティがない限り、マーケットプレイスで標準リザーブドインスタンスを販売することはできません。米国外で会社を見つけた場合、マーケットプレイスでインスタンスを購入できますが、販売することはできません。

スポットインスタンスを使用する


スポットインスタンスが大好きです！それらを使用して最大90％を節約でき、多額の投資を必要としません。スポットインスタンスの唯一のマイナス点は、AWSがこれらのインスタンスをすぐに、または2分の警告後に削除できることです。この点で、このタイプのインスタンスは、フォールトトレラントまたは柔軟な負荷に対してのみ使用できます。

EBSを節約する2つ目のポイントは、接続されている各ボリュームのサイズを決定することです。過剰なボリュームと未使用のギガバイトがよく使用されます。

インスタンスの削除-停止してもEBS支払いには影響しません


インスタンスを停止しても、EBSのコストから保護されません。ボリュームは引き続き保存され、支払いに含まれます！インスタンスとデータが不要になった場合は、インスタンスを削除してください。このアクションにより、EC2とEBSのコストが削減されます。

AWSでは、EBSのサイズを連続的に増やすことはできますが、減らすことはできません。一般に、ボリュームを大きくするよりもボリュームを小さくして、必要な場合にのみEBSを使用することをお勧めします。

それがどれほど高くつくことができるかを推定してみましょう。10個のt3.smallインスタンスを使用し、各インスタンスが50GB SSD EBSストレージを使用するとします。

毎月の請求書は次のとおりです。

  • EC2の月額費用= 10 x 15.23ドル= 152.30
  • EBS合計容量：10 x 50 GB = 500 GB
  • GBあたりの価格：$ 0.10-月
  • EBS月額費用：$ 50.00
  • 総費用：$ 182.30

この簡単な例は、EBSがEC2のコストの最大32％を占め、アカウントの重要な部分を占める可能性があることを示しています。これは、制御する必要がある一定の隠された値です。

エフェメラルストレージの詳細を確認するエフェメラルストレージ

の長所と短所を評価してください。利点のうち、EBSに隠れたコストを支払う必要はありません。遅延が少なくなり、リザーブドインスタンスの割引を受けることができます。マイナス点としては、（引き続き支払いを行う）すべての内部ストレージを使用しない可能性が高く、スポットインスタンスの使用がはるかに困難になります。

未使用のボリュームを削除する


費用を節約するもう1つのアクションは、切断された未使用のボリュームの削除です。

S3を使用して、

EBS バックアップをS3の少なくとも2倍のコストで維持します。実際、バックアップはEBSではなくS3に保存する必要があります。最初に、バックアップが段階的に保存され、古いバージョンが定期的に削除されることを確認してください。

6.データ転送


このコストラインはかなり早く手に負えなくなる可能性があります！データがどこからどこから来ているのかを制御するのは十分に難しく、追跡はさらに困難です。さらに、いくつかのタイプのデータ転送があります。たとえば、クラウドからのデータ転送、リージョン間のデータ転送、アクセスゾーン間のデータ転送、S3データ転送などです。

Back4Appがデータ転送コストを削減した方法とそれを追跡した方法について説明します。

AWSのデータ転送コストを削減するにはどうすればよいですか？


データコストを削減するためにできることをいくつか紹介します。

S3エンドポイントをCloudflareおよびその他のCDNサービスに接続します。


CDN（コンテンツ配信ネットワークまたはデータネットワーク）この単純なアクションにより、保持するキャッシュの量に応じて、データ転送コストを最大5倍削減できます。

Elastic IP-可能な場合は独自のIPアドレスを使用する


これは良い例です！AWS計算機によると、EC2インスタンスではクラウドへのデータ転送は無料ですが、「パブリックIP / Elastic IPデータ転送」のフィールドにも入力する必要があります。パブリックIPアドレスまたはElastic IPアドレスを使用する接続、またはAmazonネットワーク内のElastic Load Balancerを選択した場合、インスタンスが同じアベイラビリティーゾーン内にある場合でも、リージョン間のデータ転送料金を支払います。1つのアクセスゾーン内でデータを転送するには、可能な場合は個人のIPアドレスを使用することで、このようなコストを簡単に回避（およびネットワークパフォーマンスを向上）できます。

1つのアベイラビリティーゾーンを使用する-アベイラビリティーゾーン


複数のアベイラビリティーゾーンの代わりに1つのアベイラビリティーゾーンを使用すると、大幅に節約できます。リージョン間のデータ転送を回避または停止します。

GraphQL APIを使用する


GraphQLの重要な利点の1つは、データの過剰配信が減少し、結果として、転送されるデータの総量が減少することです。アプリケーションの構造によっては、RESTの置き換えが困難な作業になる場合があります。また、そのような移行の実現可能性を事前に評価することも価値があります。もう1つの方法は、新しい製品や機能にGraphQLを順次使用することです。

データ転送割引


割引に必要なボリュームが非常に大きいため、これはほとんど適用できません。これは、大企業や大規模なアカウントでのみ役立ちます。

CloudFrontを使用してください！


このアクションを適用すると、あなたは財産を節約することができます！また、アプリケーションの速度を上げて、一般的にレイテンシを減らすこともできます。これについては、以下で少し説明します。

7. CloudFront


Cloudfrontのオンデマンド価格は、量が増えるにつれて急速に減少しています。100 Tbトラフィックの伝送を想像してみましょう。これは、0.060ドル/ GBのコストで、ELBを介した同じ量のデータと比較して約15％少なくなります。

クラウドフロントのプライベート契約とボリューム価格


データコストを削減する最善の方法は次のとおりです。AWSはCloudFrontサービスを使用することを好み、ボリュームによっては、送信CloudFrontトラフィックの価格を0.02ドル/ GBに下げることができます。ほとんどの場合、これには2年間のコミットメントが必要になります。AWSを長期間使用することを計画している場合、これは素晴らしい提案です。このような手配をするには、アカウントマネージャーに連絡して、CloudFrontのプライベート契約について尋ねる必要があります。

Amazon CloudFront SSL-Cert-Custom


この証明書を使用しているかどうかを確認してください。1日あたり20ドルで、同じものをCloudflareから月額25ドルで購入できます。プライベート＆ボリューム料金契約では、他のCDN（クラウド配信ネットワーク）サービスを使用できなくなり、このアクションは利用できなくなります。

8. S3-シンプルストレージサービス


S3ストレージの価格を下げるためのいくつかの推奨事項。

S3 VPCエンドポイント


少数の人々が与えられるという素晴らしい節約の考えは、S3 VPCエンドポイントの使用です。VPC（仮想プライベートクラウド）にS3エンドポイントを配置すると、S3へのデータアクセスが可能になり、クラウド内に独自のパスが提供されます。AWSは、パブリックトラフィックであるかのように課金しません。ボリュームにもよりますが、この手法では月に数千ドルも節約できます。

正しいストレージモデルと適切なストレージクラスを使用してください。
