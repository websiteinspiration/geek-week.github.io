Linux dan WYSIWYG

Dalam esai yang sangat singkat ini, saya akan berbicara tentang nuansa, karena skala 100% pada editor grafis di layar mungkin tidak sesuai dengan ukuran sebenarnya.

Saya perlu mencetak sedikit teks dan gambar ke selembar kertas yang ada. Saya memutuskan untuk benar-benar pas. Saya mengatur skala ke 100%, saya lampirkan lembar ke layar, dan sementara itu, di layar dan di atas kertas, persentase berjalan adalah 20%.

Ini terjadi ketika resolusi nyata layar (dalam titik per inci) tidak sesuai dengan apa yang dipikirkan program. Ketika sebuah program memiliki tugas untuk menggambar sesuatu yang panjang di layar, misalnya, L cm, itu menghitung berapa banyak pikselnya. Untuk melakukan ini, ia menerima DPI, dan kemudian kira-kira rumus berikutLpx=L⋅DPI/2.54mendapatkan panjang dalam piksel.

Sebenarnya, masalahnya adalah bagaimana program bisa mendapatkan DPI ini, dan kemudian tarian dengan rebana dimulai. Secara umum, monitor dapat memberikan kartu video, dan pada gilirannya OS, dimensi liniernya. Untuk melakukan ini, gunakan antarmuka DDC (Display Data Channel) dan protokol EDID. Dan kemudian masalahnya dimulai.

Masalah nomor 1 - protokol EDID mentransmisikan dimensi linier layar ke cm terdekat. Tampaknya tidak ada masalah, tetapi setengah sentimeter pada layar kecil (seperti laptop) adalah 3-4%. Bukan bencana, tapi bisa dihindari.

Masalah 2 - X.org, meskipun faktanya membaca dimensi melalui EDID, untuk beberapa alasan, dalam kasus saya, itu tidak menerapkannya dan mengatur DPI ke 96. Anda dapat mengetahui hal ini dengan perintah berikut di terminal grafik:

$ xdpyinfo | grep resolution
  resolution:    96x96 dots per inch

Masalah nomor 3 - aplikasi dapat menggunakan sumber lain untuk mendapatkan DPI. Ini terjadi dengan aplikasi menggunakan perpustakaan GTK3. Mengapa demikian? Saya tidak tahu. Mari kita serahkan ini pada hati nurani para pengembang GTK3.

Jadi, kencangkan semua ini.

1) Ukur penggaris monitor. Dalam kasus saya, ternyata 347 mm x 195 mm.

2) Hitung DPI. Untuk melakukan ini, kita perlu mengetahui resolusi monitor. Saya memiliki 1600x900. Kemungkinan besar DPI adalah sama untuk pengukuran vertikal dan horizontal, tetapi ini tidak selalu terjadi.DPIx=1600∗25.4/347≈117. DPIy=900∗25.4/195≈117.

3) Buka file konfigurasi X.org /etc/X11/xorg.conf. Jika tidak ada file seperti itu, maka dari root Anda perlu menjalankan:

X.org -configure

dan tulis file yang dihasilkan ke jalur yang ditentukan. Di bagian Monitor, tambahkan baris DisplaySize, sesuatu akan terlihat seperti ini:

Bagian "Monitor"
Identifier "Monitor0"
VendorName "Monitor Vendor"
ModelName "Monitor Model"
DisplaySize 347 195
EndSection

Ini akan menyelesaikan masalah 1 dan 2.

4) Buat file ~ / .Xresources atau tambahkan ke pengaturan DPI yang ada:

Xft.dpi: 117

Ini kami akan memecahkan masalah 3.

Kemudian restart X-sesi dan semuanya harus diambil.
