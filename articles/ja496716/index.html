Power Automate VS Logic Apps。パワー自動化ケース

全てにおいて良い日！Power AutomateとLogic Appsに関する以前の記事では、Power AutomateとLogic Appsの主な違いについて説明しました。今日は先に進み、これらの製品で実現できる興味深い機会を示したいと思います。この記事では、Power Automateで実装できるいくつかのケースを紹介します。

Microsoft Power Automate


この製品は、特定のイベントの発生によるフローの自動および即時の開始のために、さまざまなサービスへの幅広いコネクターとトリガーを提供します。また、スケジュールまたはボタンによるスレッドの起動もサポートしています。

1.通話の自動登録


ケースの1つは、コールの自動登録の実装です。この場合、ストリームのトリガーは、特定のメールボックスへのメール通知の受信であり、その後、さらにロジックが処理されます。



「新しいメールが届いたとき」のトリガーを設定する場合、さまざまなフィルターを使用して、トリガーを開始するために必要なイベントを決定できます。



たとえば、添付ファイルがあるメッセージまたは最大の重要性を持つメッセージに対してのみストリームを開始できます。メッセージがメールボックスの特定のフォルダに到着した場合は、ストリームを開始することもできます。さらに、件名行の目的の部分文字列で文字をフィルタリングすることができます。
必要な計算が行われ、すべての必要な情報が受信されたら、他のアクションからの置換を使用してSharePointリストにアイテムを作成できます。



このストリームを使用すると、必要なメール通知を簡単に取得してコンポーネントに解析し、他のシステムにエントリを作成できます。

2. PowerApps


標準シナリオの1つは、オブジェクトを承認のために承認者に送信することです。このシナリオを実装するには、PowerAppsでボタンを作成し、それをクリックすると、Power Automateストリームを開始します。



ご覧のように、このストリームでは、開始トリガーはPowerAppsです。このトリガーの特徴は、Power Automateストリーム内でPowerAppsから情報を要求できることです。



これは次のように機能します。PowerAppsから情報を取得する必要がある場合は、[Ask in PowerApps]項目をクリックします。その後、このPower Automateストリームのすべてのアクションで使用できる変数が作成されます。 PowerAppsからストリームを開始して、この変数の値をストリーム内に転送する場合にのみ残ります。

3. HTTPリクエストを使用してストリームを開始する


私が話したい3番目のケースは、HTTPリクエストを使用してPower Automateストリームを起動することです。場合によっては、特にさまざまな統合ストーリーに当てはまる場合、HTTPリクエストを介してPower Automateストリームを開始し、さまざまなパラメーターをストリームに渡す必要があります。これは非常に簡単です。トリガーとして、「HTTPリクエストの受信時」アクションが使用されます：HTTP：



HTTP POST URLは、ストリームが最初に保存されるときに自動的に生成されます。このストリームで開始するためにPOSTリクエストを送信する必要があるのは、このアドレスです。パラメータとして、起動時にさまざまな情報を転送できます。たとえば、この場合、SharePointID属性は外部から転送されます。同様の入力回路を作成するには、「ペイロードの例を使用して回路を作成する」という項目をクリックし、ストリームに入るJSONの例を挿入する必要があります。



「完了」をクリックした後、このリクエストテキストのJSONスキーム行動。これで、SharePointID属性をこのストリームのすべてのアクションで代替として使用できます。



「HTTPリクエストの受信時」トリガーはプレミアムコネクタセクションに含まれており、この製品の別のプランを購入した場合にのみ使用できることに注意してください。
