Ide



Pada Januari 2019, saya mulai membaca 2-5 buku sebulan. Setelah selusin karya seni, saya menyadari bahwa saya lebih suka buku tentang bisnis dan peningkatan diri. Jadi di beberapa titik saya membaca tentang kebiasaan orang sukses dan tidur - dan mulai bangun jam 5 pagi. Ngomong-ngomong, saya menulis artikel tentang ini .



Daftar buku saya termasuk Getting Things Done, Eat That Frog, Willpower, dan Atomic Habits. Keempatnya menceritakan kisah hebat tentang dampak psikologi kognitif pada motivasi dan produktivitas. Saya pikir, kata mereka, apa yang tidak bercanda - dan memutuskan untuk secara konsisten mencoba semua tips dari buku. Dan setiap kali ada yang salah.



Setiap kali saya menemukan diri saya sendiri dengan daftar tugas raksasa yang menekan saya secara psikologis (saya hanya menambah jumlah pekerjaan), atau dengan awan tugas yang tidak akan pernah saya mulai. Proyek ‚Äúolahraga‚Äù, ‚Äúpengembangan diri‚Äù, ‚Äúmembaca‚Äù dan ‚Äúhobi‚Äù menyampaikan salam. Sangat mengecewakan ketika saya memiliki daftar tugas yang panjang dan banyak tugas yang tidak akan pernah saya mulai.



Tapi saya keras kepala, jadi saya mencoba mencari tahu apa yang salah. Sebagai hasilnya, saya mengumpulkan daftar aturan dari semua metodologi, menelusuri setiap poin dan menilai pada skala dari "positif mempengaruhi produktivitas" hingga "membahayakan produktivitas". Dia memilih hanya mereka yang membantu menjadi efektif. Barang yang dihapus yang tidak memberikan apa pun atau memperburuk situasi.



Sebagai contoh, secara eksperimental, saya menyingkirkan konsep "proyek". Ternyata saat tidak ada folder tempat Anda bisa menyembunyikan tugas, itu akan berjalan! Sekarang jelas bagi saya, tetapi saya tidak mengerti mengapa tidak jelas bagi penganut GTD yang sama.



Akibatnya, saya sampai pada daftar aturan sederhana, dengan demikian 20% dari pembatasan, yang memberikan peningkatan produktivitas 80%. Dan dia menggambarkan sejumlah aturan ini, pertama dalam artikel tentang produktivitas pribadi - dan kemudian dia mencoba menerapkannya di perusahaan tempat saya bekerja untuk mengelola programmer. Saya harus sedikit memodifikasi dan melengkapi aturan, tetapi bahkan di sana, sistem ini bekerja dengan sempurna.



Meluncurkan



Ketika saya mengkonfirmasi bahwa daftar aturan berfungsi, saya mulai kehilangan alat yang ada untuk memelihara daftar tugas - mereka terlalu sederhana (misalnya, tidak mungkin untuk menambahkan tanggal atau sebulan untuk tugas) atau rumit kembali (ketika iblis patah kakinya di pengaturan). Jadi saya mengambilnya dan memotong prototipe pertama dalam seminggu. Bahkan selang waktu pengembangan tetap:





Butuh waktu sekitar 36 jam untuk mengembangkan MVP pertama "Tuduranta" (saya menghitung jam dari selang waktu). Aplikasi web acungan jempol telah dirilis pada Vue, Vuetify, Node, Koa, Mongo, semuanya dengan saus TypeScript, agar tidak usang karena dokumentasi jenis.



Secara umum, salah satu kekuatan saya adalah saya menemukan alat dan teknologi pengembangan yang keren, yang dengannya Anda dapat menulis kode minimum untuk mendapatkan fungsi yang diperlukan pada output. Tumpukan teknologi ini dapat dilihat pada permulaan yang saya gunakan terus-menerus:



  • Starter untuk frontend
  • Starter untuk backend
  • Starter untuk bot Telegram


Saya juga menulis beberapa artikel tentang cara menggunakan permulaan ini dan bagaimana memulainya dari awal di server atau host secara gratis di GitHub Pages .



Karena saya memiliki permulaan yang begitu bagus, waktu dari ide hingga peluncuran produk telah dikurangi menjadi 6 hari. Sebagai contoh, saya meluncurkan " Localizer " - layanan penerjemahan sumber terbuka mandiri yang di-hosting-sendiri.



Saya berhasil, karena saya lelah melempar string terjemahan dalam banyak format berbeda kepada orang yang berbeda yang ingin membantu menerjemahkan layanan saya ke bahasa lain. Sebuah contoh dapat ditemukan di localize.borodutch.com dan localize.todorant.com .



Aplikasi seluler telah menjadi masalah tersendiri bagi saya. Awalnya, saya adalah pengembang iOS (saya mulai pada 2012). Saya sangat skeptis tentang solusi lintas-platform karena mereka ‚Äúlambat‚Äù. Namun pada tahun 2018, ia mulai memutar-mutar React Native sendiri. Saya melihat betapa dia telah berkembang selama dua tahun terakhir.



Tetapi saya masih memutuskan untuk memulai dengan aplikasi asli. Aplikasi iOS di Swift (dan bahkan SwiftUI), aplikasi Android di Kotlin. Jika saya menulis sebagian besar kode di iOS, maka di Android saya bukan rubah, jadi saya meminta seorang teman untuk dengan cepat membungkus API saya di shell untuk platform favorit saya.



Ya, pada akhirnya ternyata meluncurkan versi iOS dan Android - tetapi berapa biayanya? SwiftUI sangat mentah, Anda harus memotong kode dari kruk di sana, kruk pada kruk dan kruk itu.



Di bawah Android untuk setiap fitur, saya harus menjelaskan kepada teman saya bagaimana itu harus bekerja dan membantu menyelidiki sifat aplikasi, apa yang menjadi semakin sulit, saya melihat bagaimana orang yang dipercaya perlahan-lahan kehilangan minat dalam mengembangkan produk ini. Akibatnya, ia meninggalkan proyek.



Dan saya ditinggalkan dengan aplikasi iOS yang berfungsi (tetapi aneh karena SwiftUI) dan aplikasi Android yang setengah bekerja. Saya sudah berpikir untuk mulai menulis ulang aplikasi Android ke logika baru - tetapi kemudian saya ingat, saya Bereaksi Asli-pengembang sejak 2018, saya dapat mengajukan solusi lintas-platform saya sendiri!



Dan saya duduk dan bersabar. Hanya saja kali ini saya melakukan semuanya dengan bijak, dengan arsitektur keren, dengan mode offline, dengan mode demo tanpa registrasi, dengan sinkronisasi di soket, sehingga semuanya indah. Ya, aplikasi ini sedikit lebih lambat (dalam milidetik) daripada yang asli - tetapi pengguna tidak mungkin memperhatikan ini bukan pada pohon apel terbaru dan pohon ember.



Selain itu, saya punya dua fitur yang sangat keren:



  • Segera setelah ide fitur muncul, implementasikan dan lepaskan pada dua platform sekaligus
  • Tidak bergantung pada orang lain


Ya, saya mulai melakukan semuanya sendiri. Di sini saya adalah orang yang buruk yang tidak ingin mendelegasikan tugas pada tahap awal startup. Sebagai hasilnya, saya membasmi server, front-end, aplikasi iOS, dan aplikasi Android - Saya memiliki akses ke semuanya, saya mengerti dengan sempurna setiap baris kode, saya bisa melakukannya dengan sangat cepat.



Ini seperti mengendarai gearbox manual - lebih efisien dan tinggi, tetapi tidak terlalu santai. Anda harus mengambil kaki Anda di tangan Anda dan memikirkannya sendiri.



Sebagai bonus - saya mulai memaksa diri untuk mengotomatiskan semua proses secara umum. Sebagai contoh, saya menemukan Fastlane dan secara otomatis mengotomatiskan seluruh siklus penerapan aplikasi ke toko-toko. Yah, saya menulis instruksi tentang bagaimana melakukan hal yang sama, di mana itu tanpa itu.



Singkatnya, pada akhirnya, saya memiliki kendali penuh atas seluruh siklus pengembangan, tetapi pengembangannya sedikit lebih lambat (saya tidak cukup untuk semuanya tentang semuanya). Ketika saya mengumpulkan cukup pendapatan untuk Tudurant, saya akan mempekerjakan lebih banyak pengembang.



Pemasaran



Seperti semua proyek saya yang lain, hal pertama yang saya lakukan tentang Tudurant adalah di saluran Telegram saya Gold of the Beard . Semua statistik waktu nyata untuk semua proyek saya dapat dilihat di situs web saya . Saya akan segera mengunggah grafik dari situs di sini sehingga saya tidak harus mengikuti tautan - di sebelah kiri adalah jumlah tugas baru, di sebelah kanan adalah jumlah pengguna baru:







Saya melakukan langkah kecil dengan kuda - mengatakan bahwa 1500 pengguna pertama akan menerima langganan gratis ke layanan selamanya. Ini adalah transaksi sederhana: Saya memberi orang akses gratis - dan mereka memberi saya umpan balik produk. Masuknya pertama pengguna di tangga lagu sebelum Oktober 2019 adalah 1.500 pengguna pertama.



Langkah selanjutnya adalah mengirimkan proyek saya yang lain - Telegram bot Voicy. Ngomong-ngomong, ada artikel tentang dia juga - di awal, ketika hanya ada sedikit lebih dari 10 ribu pengguna.



Saya menggambarkan hasil mailing list Voysi di artikel terpisah . Puncak terbesar dan puncak kecil berikutnya pada grafik adalah akibat langsung dari milis Voysi. Ngomong-ngomong, dalam proyek saya, saya tidak pernah menjual, saya tidak menjual dan saya tidak bermaksud menjual iklan - ada banyak uang sehingga menjadi konyol bahwa orang lain mengorbankan reputasi mereka dengan murah.



Lalu saya menemukan daftar tempat di mana Anda dapat berbicara tentang startup Anda. Karena pada saat itu saya sudah memiliki Tudurant, saya baru saja memasukkan 4-6 posisi dari daftar untuk posting setiap hari selama sekitar empat minggu. Dan setiap hari, perlahan tapi pasti, saya memasang Tudurant di situs-situs ini.



Ngomong-ngomong, saya melakukan ini bukan tanpa menulis catatan pada masing-masing layanan. Sebagai hasilnya, saya membuat tabel PostYourStartup sederhana dan nyaman dengan filter dan barang lainnya. Dan tentang ini saya juga menulis artikel .



Knalpot terbesar berasal dari Reddit dan Indie Hacker. Kebanyakan subreddit dan Peretas Indie perlu mendapatkan karma sebelum memposting - yang saya lakukan.



Setiap kali tugas posting mendekati meninggalkan pos di mana ada "dinding karma", saya membuat 5 tugas per hari selama seminggu dari jenis "Tinggalkan komentar di xyz subbreddit" dan tugas "Posting tentang" Tudurant "ke xyz subbreddn" seminggu kemudian . Jadi, secara sistematis, saya mengisi karma saya dan akhirnya meninggalkan posting tentang Tudurant.



Puncak terbaru di tangga lagu adalah tepatnya hasil memposting "Tudurant" ke semua situs dari daftar. Saya belum menyelesaikan percobaan, saya akan memperbarui tabel ketika data baru tiba. Omong-omong, artikel ini juga merupakan bagian dari eksperimen. Jadi ikuti tautan di artikel ini, instal Tudurant dan coba sehingga Habr terlihat luar biasa dalam statistik di antara alat pemasaran gratis lainnya.



Apa berikutnya?



Hingga saat ini, Tudurant memiliki sekitar 5.000 pendaftaran dan 2.000 pengguna aktif. Seperti yang mungkin telah Anda perhatikan di atas, sejumlah kecil tidak mengganggu saya, Anda selalu perlu memulai perjalanan Anda di suatu tempat. Sebelum itu, saya berhasil menumbuhkan Voysi dari 100 obrolan pertama yang ditambahkan menjadi lebih dari 1.100.000 obrolan yang terhubung.



Saya melakukan seluruh pengembangan Tudurant di Tudurant sendiri, dengan duplikasi beberapa tugas dalam tabel Trello publik, sehingga pengguna akan secara kasar memahami apa yang diharapkan dari layanan di masa depan. Secara harfiah setiap tugas dalam mengembangkan Tudurant ada pada saya, seperti tugas di Tudurant. Dan itu keren - saya sendiri menggunakan layanan yang saya kembangkan.



Inovasi keren berikut yang saya miliki dalam pekerjaan saya adalah delegasi, epos, dan templat. Saya mencoba untuk fokus bukan pada fungsi yang dimiliki pesaing (saya percaya bahwa mereka semua terlihat dengan cara yang salah), tetapi pada fungsi yang secara langsung akan meningkatkan produktivitas pengguna.



Delegasi adalah ketika pengguna dapat menambahkan pengguna "tepercaya" lainnya, yang dapat membuang tugas ke folder kotak masuknya dengan satu klik tombol dan melihat status tugas-tugas ini.



Epik- Ini adalah elemen gamification tertentu. Tudurant sudah memiliki "jajaran" produktivitas (menyelesaikan 1.000 tugas, menerima lencana) dan tagar (cara mengatur tugas menjadi proyek tanpa menyembunyikan tugas). Saya memutuskan untuk menggabungkannya dengan fungsi "epik", ketika Anda dapat memilih tagar dan tujuan tertentu (misalnya, menyelesaikan 100 tugas dengan tagar "olahraga" tahun ini) dan melihat kemajuan yang jelas pada tujuan.



Quests - sehingga Anda dapat membuat daftar linier yang sama untuk program pelatihan (mendorong 20 kali, menarik 10 kali, melompat 5 kali setiap 2 hari), membagikannya dan membuka akses hanya ke lingkaran pengguna tertentu (misalnya, siapa yang akan membeli program dari seseorang di samping) . Saya mengklik tombol, memilih tanggal, dan daftar tugas Anda dilengkapi dengan semua tugas yang diperlukan dari pelatihan.



Secara umum, pengembangan Tudurant didasarkan pada umpan balik pengguna. Siapa pun dapat menulis saya atau di obrolan Telegram kami "Tuduranta" setiap tawaran atau melaporkan bug apa pun. Jika proposal tersebut dapat ditoleransi dan sesuai dengan metodologi, saya segera menambahkan pada diri saya tugas melakukan ini.



Di sisi pemasaran, seperti biasa, tugasnya sangat sederhana - untuk mengurangi persentase orang yang keluar dari layanan sebelum membayar, dan menambah jumlah orang yang tahu tentang Tudurant. Mungkin saya akan mencoba saluran iklan berbayar - karena saya tahu dengan jelas bahwa 18% tampilan halaman rumah dikonversi menjadi pengguna terdaftar.



Kesimpulan



Terima kasih banyak sudah membaca sampai akhir! Layanan ini baru di awal perjalanan - dan saya senang bahwa orang sudah mendapatkan manfaat yang cukup dari itu untuk membenarkan label harga yang lebih tinggi daripada pesaing. Tentu saja, saya tidak akan berhenti - hanya ada cakrawala baru dan masa depan yang cerah di depan!



Pada akhirnya, saya hanya ingin mencatat bahwa kadang-kadang seluruh pasar tidak memahami sesuatu dan tidak berfokus pada apa yang penting bagi pengguna akhir. Dalam kasus Tuduranta, ini adalah fokus pesaing untuk menyimpan tugas, dan bukan pada peningkatan produktivitas. Dan bahkan kerajinan sederhana seperti itu sebagai buku tugas sederhana (semua programmer setidaknya sekali membuat Hello World) dapat sangat diperlukan dan bermanfaat bagi banyak orang.



Saya akan sangat senang menjawab pertanyaan apa pun di komentar untuk posting ini. Saya akan berusaha bersikap sopan dan tidak beracun! Harap dicatat bahwa jika saya terdengar negatif dalam komentar, ini tidak jahat, saya tidak bermaksud buruk. Sulit menyampaikan senyumku dengan teks.



Nah, pastikan untuk mencoba layanan ini! Ada versi web , aplikasi iOS dan aplikasi Android . Jika itu membantu Anda meningkatkan produktivitas, saya akan sangat bahagia. Jika ternyata metodologi ini tidak cocok untuk Anda, cukup keluarkan Tudurant. Saya ingatkan Anda bahwa bulan pertama gratis - tetapi aturannya satu: jika dalam 30 hari pertama Tudurant tidak memberi Anda manfaat $ 5, hapus saja.
