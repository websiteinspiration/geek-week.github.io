IntelliJ IDEAのトップ20ナビゲーション機能。パート1

記事の翻訳は、コース「Android-developer。上級コース。」この記事は主にJavaに焦点を当てていますが、資料はAndroid開発者にとって有用です。



IntelliJ IDEAの主な機能をカバーする一連の記事（コード追加とリファクタリングの主な機能については前に書きました）を続けて、ナビゲーションについてお話ししたいと思います。これは、IDEのパフォーマンスを改善するもう1つの機会であり、多くの時間を節約できます。もちろん、正しく使用してください。

IntelliJ IDEAユーザーがそれを最大限に活用するために理解しなければならない主なことの1つは、このIDEがコードに焦点を合わせるように設計されていることです。ほとんどの時間はコードエディタで費やされます。ナビゲーションは、時間を無駄にすることなく、コード内のある場所から別の場所に移動するのに役立つツールにすぎません。

今日の記事は、プロジェクトファイル、コードブロック、ユーザーインターフェイス要素（ツールウィンドウ、エディタータブなど）の間を移動できる機能に特化しています。

最近使用したファイル


ちょっと立ち止まってナビゲーションパターンを考えると、ほとんどの場合、ファイルの有限セットを操作することに気付くでしょう。これは、最近のファイルコマンドがIntelliJ IDEAユーザーの間で広く使用されている機能である理由を明確に説明しています。慣れると、ほとんどのナビゲーション操作に使用できるようになります。このコマンドを呼び出すには、Ctrl + E（OS Xの場合はCmd + E）を押します。



アクセスされた最後のファイルがデフォルトで選択されているため、ワンクリックでそのファイルに移動できることに注意してください。表示されたファイルのリストを絞り込むには、いわゆるスピード検索を使用できます（クイック検索）。ファイル名の一部を入力し始めるだけで、IDEは探しているものを強調表示します。



構造ポップアップ


あなたが毎日遭遇する別のテンプレートは、現在開いているファイルをナビゲートしています。各ファイルには独自の構造があります。Javaクラスは、フィールド、メソッドなどのメンバーで構成されます。XMLファイルとHTMLファイルは、タグツリーで構成されています。この構造の任意の場所にすばやくジャンプするには、Ctrl + F12（OS Xの場合はCmd + F12）を押して目的の項目を選択します（ここでも、クイック検索を使用してリストを絞り込むことができます）。



構造は、見方によって異なる場合があります。たとえば、Javaクラスの構造には、スーパークラスまたはインターフェースの構造が含まれる場合があります。デフォルトでは、IntelliJ IDEAはスーパークラスとインターフェースのメンバーを非表示にします。ただし、同じホットキーをもう一度クリックすると、それらが表示されます。



次にこのコマンドを使用するとき、IDEは設定を記憶します。


ほとんどの場合、画面に表示されるのはエディター（およびコード自体）だけであるという事実を受け入れることは非常に役立ちます。一部の人々は、プロジェクトツリーを表示したり、現在のコンテキストを理解したり、すばやく変更したりできるようにする必要があると主張するかもしれません。実際、これがIntelliJ IDEAがプロジェクトツリーとエディターを簡単に切り替えることができる理由です。ただし、プロジェクトツリーは実際には最適なオプションではありません。経験豊富なIntellIJ IDEAユーザーは、ナビゲーションバーというはるかに優れたオプションを知っています。新しいユーザーは通常、これは小さな機能だと思っていますが、経験豊富な開発者は、これを使用してほとんどすべてのことを行う方法を知っています。ナビゲーションバーをアクティブにするには、次のコマンドでJump to navigation barコマンドを呼び出します。Alt + Home（Alt + for OS X）。このコマンドを使用すると、プロジェクト構造全体をナビゲートできます。ファイルの作成と変更、ファイルの内容の表示など。



ナビゲーションバー自体が非表示になっている場合でも、ナビゲーションバーへの遷移を使用できることに注意してください。



授業に行く


Go to a classの ような基本的な操作はどうですか？IntelliJ IDEAでCtrl + N（OS Xの場合はCmd + N）を押すたびに、このコマンドがどれほど強力であるかを疑うことはできません。 IDEでサポートされている単純なワイルドカード、キャメル表記、およびパケットプレフィックスに加えて、IntelliJ IDEAのGo to a classコマンドは、ミッドネームマッチング、ポストフィックス行番号、クイックドキュメント、およびポップアップ定義を提供します。



デフォルトでは、このコマンドは、事業;一致が見つからない場合は、プロジェクトライブラリをスキャンします。エリアを切り替えるには、同じホットキーを使用します。

IntelliJ IDEAのコマンドの利点は、マウスがなくても非常に使いやすいことです。ポップアップウィンドウを表示した後、クエリを入力して変更し、結果を上下に移動できます。そして、それはすべてです。他のIDEで適切なコマンドを使用したことがある場合は、私が理解していることがわかります。

ファイルまたはフォルダに移動します


Ctrl + Shift + N（OS Xの場合はCmd + Shift + N）に割り当てられたGo to a file コマンドを使用すると、プロジェクトで特定のファイルを開く必要があるときに時間を節約できます。たとえば、ワイルドカード文字、ラクダ記法、フォルダープレフィックスなどの標準機能に加えて、フォルダーを見つけることができます。



シンボルに移動


そして最後に、Go to ...ファミリーの最後の、しかしそれほど重要ではないコマンドはGo to a symbol（Ctrl + Shift + Alt + N（OS XではCmd + Shift + Alt + Nに割り当てられています）です。これは、その名前が示すように、全文検索なしで任意の文字にジャンプできるようにします。IDEはプロジェクト（およびライブラリ）のすべての文字のインデックスを常に更新するため、プロジェクトが非常に大きい場合でも、式を入力して適切なメソッドまたはフィールドを数秒で見つけることができます。



で選ぶ


ここで、特定のツールウィンドウ（プロジェクトビューや変更ビュー、エクスプローラー/ファインダーなど）で現在選択されているファイルを選択する必要があると想像してください。ここでは、Alt + F1でSelect inコマンドを使用できます。



ツールウィンドウに切り替えます


IntelliJ IDEAのほとんどのツールウィンドウには、便利なキーボードショートカットがあります。対応するホットキーをクリックすると、ツールウィンドウが表示され、フォーカスが移ります。同じホットキーをもう一度クリックした後、フォーカスがツールウィンドウにある場合は非表示になり、フォーカスがエディターに戻ります。これは非常に便利で、マウスを完全に取り除くことができます。ツールウィンドウが必要な場合や、エディターに戻りたい場合は、ホットキーをクリックするだけです。

少なくとも次のホットキーを覚えておいてください。

  • プロジェクトツールボックス：Alt + 1（OS Xの場合はCmd + 1）
  • ツールウィンドウの変更：Alt + 9（OS Xの場合はCmd + 9）
  • ターミナルツールウィンドウ：Alt + F12

エディターに切り替え


誰もが知っているとは限らない最もトリッキーな機能の1つは、Escapeキーを押して（どこからでも）エディターにフォーカスを切り替えることです。切り替え元のツールウィンドウ。この場合、表示状態が保存されます。すべてのツールウィンドウを非表示にしてエディターに切り替えるには、Ctrl + Alt + F12（OS XではCmd + Alt + F12）の別のキーの組み合わせを使用します。
