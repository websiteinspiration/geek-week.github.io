コードを最初から構築します。レベルを上げます



ホイールを本当に理解するには、自分でそれを再発明する必要があります。

先日、JavaScriptの上級開発者の立場についてインタビューを行いました。インタビューにも参加した私の同僚は、候補者にHTTPリクエストを実行し、失敗した場合にそれを数回繰り返す関数を書くように頼みました。

彼がこれをボードに書いたので、疑似コードで十分でしょう。彼が問題をよく理解していれば、私たちは満足するでしょう。しかし、残念ながら、彼は良い解決策を見つけることができませんでした。

彼は緊張していると思い、少し作業を楽にすることにし、コールバックに基づく関数をプロミスに基づく関数に変換するように頼みました。

不運。

私は彼が以前に同様のコードを見たことがあると言えるでしょう。彼は多かれ少なかれこれがどのように機能するかを知っていました。彼がこの概念を理解していることを示す疑似コードで十分でしょう。

しかし、彼がボードに書いたコードは意味がありませんでした。彼はJavaScriptでのPromiseの概念について漠然とした理解しか持っておらず、彼について明確に話すことはできませんでした。

初心者の開発者の場合は言い訳になりませんが、リーダーシップの立場を申請した場合、これでは不十分です。彼はどのようにして複雑なプロミスチェーンをデバッグしてから、他の人に彼がしたことを説明しますか？


記事はEDISONソフトウェアの支援を受けて書かれた、プログラマーの健康とその朝食の世話をし、また、カスタムソフトウェアを開発します。

開発者は当たり前のことを抽象化します


開発者として、私たちは抽象化を扱います。それ以外の場合は複製する必要があるコードを抽象化します。したがって、より重要な点に焦点を当てる場合、私たちは当然のことながら作業している抽象化を受け入れ、単純にそれらが正常に機能すると仮定します。

通常はそうですが、状況が複雑になる場合は、これらの抽象化がどのように機能するかを知っておく必要があります。
上級開発者候補は、Promiseの抽象化を当然のことと考えていました。もし彼がコードのどこかでそれと出会ったなら、彼はおそらく彼と一緒に働く方法を知っていたでしょうが、彼はその概念を理解していなかったため、インタビューでそれを再現できませんでした。
彼はコードを覚えているだけでした。それほど難しいことではありません。

return new Promise((resolve, reject) => {
 functionWithCallback((err, result) => {
  return err ? reject(err) : resolve(result);
 });
});

私がやった。私たちはおそらくこれをすべて行います。コードを覚えておくだけで作業できます。あなたはこれがどのように機能するか多かれ少なかれ理解しています。

しかし、もし彼がその概念を本当に理解していれば、彼はそれを覚える必要はないでしょう。彼は単にこれを知っていて、問題なくそれを再現しました。

あなたの情報源を知る


2012年に、フロントエンドフレームワークが支配し、jQueryが世界を支配する前に、jQueryの作成者であるJohn Rezigによる「JavaScript忍者の秘密」を読みました。

この本では、独自のjQueryを最初から作成する方法を説明し、このライブラリの作成の背後にある思考プロセスについて独自の理解を提供します。近年、jQueryがバックグラウンドに陥っていますが、この本を読むことを強くお勧めします。

この本で私を驚かせたのは、自分でそれを考えることができる絶え間ない感覚でした。この本で説明されている手順は非常に論理的でシンプルなので、このような目標を設定すればjQueryを作成できると本当に感じました。

実際、これを行うことはできません-複雑すぎると思います。私の決断は単純すぎて世間知らずだと思い、あきらめるだけです。私はjQueryを当然のことと考えて、それを信頼します。その後、私はおそらくそれがどのように機能するか理解していません。ブラックボックスとして使用します。

この本を読んで私は変わりました。私はソースコードを理解し始めて、多くの実装がかなり簡単で、明白でさえあることがわかりました。

これらの解決策を自分で考え出すことは、まったく別の問題です。しかし、ソースコードを読んで既存のソリューションを自己実装することこそが、独自のソリューションを思いつくのに役立ちます。

あなたが受け取るインスピレーションとあなたが発見するパターンは、開発者としてあなたを変えます。マジックを使用して検討するこの素晴らしいライブラリは、実際にはマジックではなく、単純でスマートなソリューションであることがわかります。

ステップごとにコードを解析することに時間を費やすことができます。また、作成者がコードを作成するために行ったのと同じ小さな段階的なステップを実行する必要があります。これにより、プログラミングプロセスをより深く理解し、独自の決定を実装する上での自信が高まります。

JavaScriptでPromiseを使い始めたとき、それは魔法だと思いました。それから、それらがコールバックに基づいていることがわかり、プログラミングの私のビジョンは永遠に変わりました。
コールバックを取り除くことになっていたこのテンプレートは...コールバックを使用して実装されましたか？

それは私を変えました。これにより、これらは信じられないほど複雑なコードスニペットではないことがわかり、理解できるようになりました。好奇心があり、飛び込みたいという気持ちがあれば、簡単に理解できるモデルです。

これが、プログラミングを実際に学ぶ方法です。これがあなたがより良い開発者になる方法です。

車輪を再発明する


それでは、車輪を再発明しましょう。あなたの書くバインディング独自のデータを、独自の約束を、あるいはあなたの状態管理ソリューション。

誰もそれを使用しないことは問題ではありません。あなたは自分のために何か新しいことを学びます。そして、あなたがあなた自身のプロジェクトの一つでそれを使うことができれば、それは素晴らしいでしょう。このコードをさらに開発し、さらに学習します。

重要なのは、本番環境でソリューションを使用することではなく、学ぶことです。既存のソリューションの独自の実装を作成することは、最良のものから学ぶための優れた方法です。
