Roslynアナライザー。コードをすばやく正確に書く方法

.NETコンパイラプラットフォームであるRoslynは、コードが実行される前でもエラーを検出するのに役立ちます。1つの例は、Visual Studioに組み込まれているRoslyn Spell Checkerです。静的メソッドを作成し、誤って静的という単語をstatcとして書き込んだとします。コードを実行する前でも、このスペルエラーを確認できます。Roslynは、行を入力する前にテキストを入力すると、コードに警告を表示するためです。つまり、間違いを見つけ出すためにコードをビルドする必要はありません。



Roslynアナライザーは、Visual Studio電球アイコンを使用して自動コード修正を表示することもできます。これにより、コードをすぐに修正できます。



しかし、さらに多くの間違いを見つけられたらどうでしょうか？


Roslynアナライザーパッケージを紹介します。これらのアナライザーコレクションは、より詳細な分析を提供しますが、デフォルトのVisual Studioツールには付属していません。お気に入りのRoslynアナライザーの詳細については、GitHubのRoslynアナライザーリポジトリにアクセスしてください。このリポジトリにはFxCopルールが含まれており、現在も最新のソフトウェア開発に適用されますが、現在はRoslynベースの最新のコード分析プラットフォームを対象としています。続けてこのパッケージをインストールして、優れたコードを書く生産性と速度を向上させましょう！

FxCopアナライザーのインストール：

  1. , Visual Studio 2017 15.8 . , Microsoft.CodeAnalysis.FxCopAnalyzers.
  2. Visual Studio, Package Manager UI.



パッケージをインストールしたら、ソリューションエクスプローラーでアナライザー診断を構成するだけです。アナライザーノードは、ソリューションエクスプローラーの[参照]または[依存関係]ノードの下に表示されます。最初にアナライザーを展開し、次にアナライザーアセンブリの1つを展開すると、アセンブリで利用可能なすべての診断を表示できます。



[プロパティ]ウィンドウで、説明やデフォルトの重大度などの項目を含む診断プロパティを表示できます。プロパティを表示するには、ルールを右クリックして[ プロパティ ]を選択するか、ルールを選択してAlt + Enterキーを押します。



ソリューションエクスプローラーの各診断タイプの横にあるアイコンは、エディターで開いたときに一連のルールに表示されるアイコンに対応しています。

  • “i” Info
  • “!” Warning
  • “x” Error
  • “i” Hidden
  • “↓”



次に、ソリューションエクスプローラーで重大度を設定できます。まず、これを行うには、ソリューションエクスプローラーで[ 依存関係] > [ アナライザー]を展開し、重要度パラメーターを設定するルールを含むアセンブリを展開する必要があります。その後、ルールを右クリックし、[ Set Rule Set Severity]を選択し、ポップアップメニューで重大度パラメーターの1つを選択します。



警告を設定すると、この特定のルールセットのコードで警告が表示されます。



アナライザーの動作を理解したので、パフォーマンスを向上させ、優れたコードをより速く書くことができます！

よくある質問：


Q：Visual Studioで作業しているときに次の警告に出くわしました。「ビルド中に実行されるFxCopアナライザーのために、Run Code Analysisは非推奨になりました。FxCopアナライザーに移行するには、aka.ms / fxcopanalyzersを参照してください。どういう意味ですか？
A：FxCopは、Roslynに10年近く先行してきたコード分析ツールです。コンパイラー（たとえば、Roslynの導入）を移植したのと同様に、コード分析技術もRoslynプラットフォームに移植しました。 .NETコンパイラプラットフォーム上で実行されるため、入力時にコードで警告が表示される場合があります。つまり、エラーを見つけるためにコードをビルドする必要はありません。

Q：レガシーFxCopアナライザーとFxCopアナライザーの違いは何ですか？
A：FxCopアナライザーはリアルタイムでコンパイル時にソースコードを分析しますが、レガシーFxCopは静的コード分析であり、ビルドの完了後にバイナリファイルを分析します。詳細については、Roslynアナライザーvs.を参照してください。静的コード分析とFxCopアナライザーFAQ。

Q：独自のアナライザーを作成できますか？
A：もちろんです！アナライザーを作成するためのドキュメントはこちらです。

Q：コードスタイルにはRoslynまたは.editorconfigパーサーを使用する必要がありますか？
A：Roslynアナライザーと.editorconfigファイルは連携して機能します。.editorconfigファイルでコードスタイルを定義する場合または、テキストエディターのオプションページで、Visual Studioに組み込まれたRoslynアナライザーを実際にカスタマイズしています。

Q：アナライザーは継続的インテグレーション（CI）アセンブリで機能しますか？
A：はい、NuGetパッケージとしてインストールされたアナライザーはCIアセンブリで使用できます。

Q：アナライザーのリクエストやエラーの報告はどこでできますか？
A：GitHubのRoslynリポジトリで、より多くのアナライザーを要求し、バグを報告できます。
