最近、Habré に関する記事を見つけました。C＃とJavaScriptを比較します。私の意見では、それらを比較することは月と太陽を比較するようなものであり、古典的には空中では不調和ではない。この記事は私に別の出版物を思い出させました。それは予期せぬ、そして明白でないJavaScriptの振る舞いのシナリオを扱います、そしてC＃は単語から全く言及されていませんが、活発な好奇心が私にこの振る舞いを別の言語で繰り返すことを試みるよう促しました。

素材は本質的に研究と娯楽であり、説明されている手法は、タイトルから明らかなように、製品開発での使用は推奨されていません。

一番下の行は何ですか？



JavaScriptでは、演算子を使用して余りの除算を浮動小数点数に取得できます。それはこのように動作します： 



3.14 % 5 // 3.14
13.14 % 5 // 3.1400000000000006


同時に、数値はNumber型であり、IEEE 754標準に従って64ビットで格納されます。.NETでは、この型には兄弟のSystem.Doubleまたはdoubleのみがあります。数値リテラルに浮動小数点が含まれる場合、デフォルトでdoubleになります。数値に接尾辞dまたはDを追加することで、意図を明示的に表現できます。だから試してみてください。そして…ビンゴ！ 



Console.WriteLine(3.14d % 5); // 3,14
Console.WriteLine(13.14d % 5); // 3,1400000000000006

次の例では、条件は真です。 



const x = { i: 1, toString: function() { return this.i++; } };
if (x == 1 && x == 2 && x == 3)
    document.write("This will be printed!");


これは、等価性をチェックするときに、オペランドが同じ型にキャストされるために実現されます。この場合、オーバーライドされたtoString（）メソッドがxで呼び出され、何かを返すだけでなく、オブジェクトの状態を変更します。純粋な関数は素晴らしいものであることに気づきましたが、興味を引くために、例からの概念を実装しようとしています。 



Trickster x = new Trickster();
if (x == 1 && x == 2 && x == 3)
    Console.WriteLine("This will be printed!");


.NETでは、対応するキャストが定義されている場合、可能であればメソッドとキャストをオーバーライドできます。 



class Trickster
{
    private int value = 1;

    public override string ToString() =>
        value++.ToString();

    public static implicit operator int(Trickster trickster) =>
        int.Parse(trickster.ToString());
}


やった、それは動作します。しかし、Tricksterがintに適応しているように見えることは混乱しています。交換する 



public static implicit operator int(Trickster trickster) =>
    int.Parse(trickster.ToString());


に 



public static implicit operator double(Trickster trickster) =>
    double.Parse(trickster.ToString());


とにかく動作します。現在、Tricksterとintはダブルにキャストされています。



実際、ToString（）の再定義は、最終結果を達成するために不可欠ではなく、JavaScriptの例を最大限に模倣するために行われます。ダブルキャストを定義するのに十分です。もちろん、副作用の重要性を忘れないでください。 



class Trickster
{
    private int value = 1;

    public static implicit operator double(Trickster trickster) =>
        trickster.value++;
}


演算子のオーバーロードを思い出すと、何にも等しくない、または等しくないオブジェクトを実装できます 



Trickster x = new Trickster();
if (x == 1 && x == 2 && x == 3 && x == new[] { 1, 2, 3 } &&
    x != 1 && x != 2 && x != 3 && x != new[] { 1, 2, 3 })
    Console.WriteLine("This will be printed!");

class Trickster
{

        true;

    public static bool operator !=(Trickster trickster, object o) =>
        true;

C＃にはすぐに使用できるsetTimeout関数がないため、アナログを自分で実装し、まったく同じ結果を得ます。 



void SetTimeout(Action action, int delay) =>
    Task.Run(async () =>

        await Task.Delay(delay);
        action();
    });

Date() && {property: 1}; // {property: 1}

C＃では、演算子は通常、デフォルトではユーザー定義型に適用されません（例外==および！=参照型の場合）。しかし、それらのいくつかは型で明示的にオーバーロードできます。次に、可換性の懸念は開発者にあります。

ハリネズミとハリネズミの交差



JavaScriptでは、文字列と数値に対して数学演算を一緒に実行できます。 



var a = "41";
a += 1; // "411"
var b = "41";
b -=- 1; // 42


C＃では、これは加算でのみ可能です。 



var a = "41" + 1; // 411

// Compilation Error:
// Operator '-' cannot be applied to operands of type 'string' and 'int'
var b = "41" - (-1);


しかし、言語の愛国者に障害はありません。ソーサリーを少し追加することで、このコードでさえ「正しく」機能するようになります。 



Trickster a = "41";
Console.WriteLine(a += 1); // 411
Trickster b = "41";
Console.WriteLine(b -=- 1); // 42


暗黙的な型変換のペアを実装し、ステートメントのペアをオーバーロードする必要があるだけです。 



class Trickster
{
    private string value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = s };

    public static implicit operator Trickster(int i) =>
        new Trickster { value = i.ToString() };

    public static string operator +(Trickster trickster, int i) =>
        trickster.value + i;

    public static int operator -(Trickster trickster, int i) =>
        int.Parse(trickster.value) - i;

    public override string ToString() =>
        value;
}

ジャグリングバナナ



JavaScriptでは、文字列「banana」は次の方法で取得できます。 



('b' + 'a' + + 'a' + 'a').toLowerCase(); // "banana"
('b'+'a'++'a'+'a').toLowerCase(); // Uncaught SyntaxError: Invalid left-hand side expression in postfix operation


ここで、2番目の 'a'に単項+を適用すると、残りの文字列に追加するために文字列 'NaN'にキャストされるNaNが返され、結果は 'baNaNa'となります。



C＃でクラスを作成します 



class Trickster
{
    private string value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = s };

    public static double operator +(Trickster trickster) =>
        double.TryParse(trickster.value, out double result) ? result : double.NaN;
}

class Trickster
