繰り返しますが、再帰降下法を使用した式の分析について

出版社Dialog MEPhIの「Beginner Course C and C ++」という本を手にしたとき、私はまだ男子生徒でした。この本からオブジェクト指向プログラミングの基礎を学び、また、しばらくの間解決できないという問題も抱えていました。その最後には、アプリケーションリストプログラムがあり、その1つは再帰計算機と呼ばれていました。これは、再帰的降下法を使用した算術式の分析との最初の出会いでした。少なくともこのコードで何かを理解するために費やされた長い時間を今でも覚えています。

この記事は主に、初めて式を解析するタスクに直面している人を対象としています。

したがって、算術式のパーサーを作成する前に、これらの式の構造を理解する必要があります。簡単にするために、ここでは単項マイナス演算と式で関数を使用する可能性については考慮しません。

構造を説明するために、Beckus-Naur形式を使用します

E-> T + E | T-E | T
T-> F * T | F / t | F
F-> N | （E）

これが縦棒「|」です 論理演算「または」、およびE、T、F、N-これらの文を使用するときに元の式が分割されるさまざまなタイプの部分式。この場合のNは数値を示します。

最初の文は、タイプEの式が、タイプTの式とタイプEの式、またはタイプTの式からタイプEの式またはタイプTの式

を差し引いたもので構成できることを意味します。分割。

3番目の文は、タイプFの式が数値であるか、大括弧で囲まれたタイプEの式であることを意味します。

これらの3つの文を算術式の例に適用してみましょう

2 + 3 *（4-5）+ 6-7

最初の文を数回適用します

E（2 + 3 *（4-5）+ 6-7）-> 
T（2）+ E（3 *（4-5）+ 6-7）-> 
T（2）+ T（3 *（4-5））+ E（6-7）-> 
T（2）+ T（3 *（4-5））+ T（6）-T（7）

これで、式はT型の部分式のみで構成されているため、それぞれに2番目の文を適用します。

T（2）-> F（2）
T（3 *（4-5））-> F（3）* T（（4-5））-> F（3）* F（（4-5））
T（6）-> F（6）
T（7）-> F（7）

3番目の文の時間です

F（2）-> N（2）-> 2
F（3）-> N（3）-> 3
F（（4-5））-> E（4-5）

括弧内の式（4-5）については、最初の文に再帰的に移動し、式の残りの部分と同様に、すべてのアクションを繰り返す必要があります。式にネストされたブラケットが含まれている場合、そのような再帰的な遷移が多くなります。3つすべての文を、たとえば2 + 2 *（3 + 4 *（5-6））のようにネストされたレベルの角かっこを含む式に個別に適用して、それらが正しく機能することを確認することをお勧めします。

これで、提案によって実行されるアクションをコードの形式で実装する必要があります。これは、再帰的降下法を使用した式の分析になりますが、コードを書き始める前に、最初の2つの再帰を取り除くことで、文を少し単純化しましょう。

最初の文をそれ自体に際限なく置き換えると、

E-> T±T±T±T±...±T

2番目の文も同様の方法で変換できます。

T-> F * / F * / F * / * / ... * / F

実際、これは再帰の代わりに、ここでループを使用できることを意味します。同様に、再帰は3番目の文から除外できません。

コードはJavaで記述します。これは、現時点で私にとって最も身近なプログラミング言語です。コードを書き始める前の最後の発言-トークン化のタスクは省きます。式を分割できない重要な要素（トークン）に分割する。このようなトークンは、数値、算術演算の符号、および角括弧です。式からの入力としてすでにトークン配列があるとします。

String[] tokens = {"2", "+", "3", "*", "(", "4", "-", "5", ")", "+", "6", "-", "7"};

メソッドを含む次のパーサークラスを作成します

  • parse（）は式の解析を開始します
  • E型の式を処理する式（）
  • T型の式を処理するための項（）
  • タイプFおよびNの式を処理するための因数（）

public class RecursiveDescentParser {

    private final String[] tokens;
    private int pos = 0; //   

    public RecursiveDescentParser(String[] tokens) {
        this.tokens = tokens;
    }

    public Double parse() {
        Double result = expression();
        if (pos != tokens.length) {
            throw new IllegalStateException("Error in expression at " + tokens[pos]));
        }
        return result;
    }

    // E -> T±T±T±T± ... ±T
    private Double expression() {
        //   
        Double first = term();

        while (pos < tokens.length) {
            String operator = tokens[pos];
            if (!operator.equals("+") && !operator.equals("-")) {
                break;
            } else {
                pos++;
            }

            //    ()
            Double second = term();
            if (operator.equals("+")) {
                first += second;
            } else {
                first -= second;
            }
        }
        return first;
    }

    // T -> F*/F*/F*/*/ ... */F
    private Double term() {
        //   
        Double first = factor();

        while (pos < tokens.length) {
            String operator = tokens[pos];
            if (!operator.equals("*") && !operator.equals("/")) {
                break;
            } else {
                pos++;
            }

            //    ()
            Double second = factor();
            if (operator.equals("*")) {
                first *= second;
            } else {
                first /= second;
            }
        }
        return first;
    }
    
    // F -> N | (E)
    private Double factor() {
        String next = tokens[pos];
        Double result;
        if (next.equals("(")) {
            pos++;
            //    ,        
            result = expression();  
            String closingBracket;
            if (pos < tokens.length) {
                closingBracket = tokens[pos];
            } else {
                throw new IllegalStateException("Unexpected end of expression");
            }
            if (pos < tokens.length && closingBracket.equals(")")) {
                pos++;
                return result;
            }
            throw new IllegalStateException("')' expected but " + closingBracket);
        }
        pos++;
        //       
        return Double.parseDouble(next);
    }
}

それで全部です。必要に応じて、このパーサーは、式の単項マイナスまたは関数で動作するように教えることができます。最初に行うことは、元の文を変更して、式の構造の新しい要素を考慮に入れるようにすることです。これから、さらに多くの提案がある可能性があります。それらが適切に変更されると、実装はほぼ正式になります。
