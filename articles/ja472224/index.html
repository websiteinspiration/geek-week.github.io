レガシBIOS（任意のOS用）を備えた古いシステムで新しいNVMe SSDをブートディスクとして使用する

短いマニュアル-レガシBIOSを備え、Cloverを使用している古いマザーボードのブート可能なNVMe SSDのサポートを実装する方法（任意のOS）。この記事をきっかけに、私の意見では、ソリューションはそれほどエレガントではなく、すべてのBIOSおよびOSに対応しているわけではありません。


問題の本質


EFI（UEFI）とは異なり、古いBIOSは新しいNVMe SSDを認識しません。最新のOSは、原則としてこれらのディスクを参照します。OSをロードした後、ディスクで作業することは可能ですが、古いBIOSがないため、新しい高速ディスクから起動することはできません。それは、表示されていないものをダウンロードするコマンドを与えることができないからです。

プレリュード


NVMe SSDドライブは、原則としてM.2コネクタを備えており、PCI-Eバスを介してプロセッサと直接接続しているかのように動作します。

したがって、マザーボードにM.2コネクタがない場合は、まず、PCI-E >> M.2アダプタ、またはM.2 SSDフォーマットのPCI-E RAIDコントローラを入手することをお勧めします。

独自のレガシーエミュレーターを搭載した高度なNVMe SSDはほとんどありません。たとえば、Samsung 950 Proには、古いBIOSでレガシーデバイスとして起動する追加のROMがあります。しかし、EVOシリーズ全体にはそのような機会はありません。

決定


ちょっとした歴史


何年も前に、IntelがMacで使用するプロセッサとコンポーネントをAppleに提供し始めたとき、MacOSとの互換性のためにハードウェアの予備テストが必要でした。次に、IntelエンジニアがトリッキーなDUETブートローダーを作成しました。これは、レガシーBIOSの上にロードされ、UEFIを必要とするオペレーティングシステム（MacOSなど）を実行できるUEFIエミュレーターです。

ただし、DUETを「直接」使用すると、問題は非常に重要です（例、またはここに別の b-rrrがあります）。特定のIronとOSに応じて、多くの手動操作が必要であり、実行していることを理解しています。

将来、DUETは、マルチブートローダーを作成するいくつかのプロジェクトの基礎として採用されました。その1つがHackintoshです。これは、最新のMacOSをIntelおよびそれ以降のAMDマシンにインストールするプロジェクトです。Hackintoshの開発の結果として、多機能ブートローダーClover（クローバー）が登場しました。もちろん、MacOSとIntelをロードするためにシャープ化されていますが、今では何でも、事実上、何でも起動するために正常に使用できます。ここでは、クローバー、問題を解決するために使用します。

NVMeドライバー


NVMe SSDがすでにコンピュータに挿入されている場合は、クローバー設定でそれを見つけようとしないでください-そこにはありません。実際、CloverはデフォルトでNVMeデバイスのドライバーをロードしません。ドライバーはダウンロードに手動で含める必要があります。

これを行うには、コンダクターでUSBフラッシュドライブを開きます。

フォルダーG：\ EFI \ CLOVER \ drivers \ offに移動します（「G」ドライブはフラッシュドライブ、「off」フォルダーはクローバードライバーが接続されていないフォルダーです）。

ファイル検索NvmExpressDxe.efi

押してCtrlキー+ Cは、クリップボードにファイルをコピーすると、フォルダにファイルをコピーG：\ EFI \ CLOVER \ドライバ \ BIOS およびG：\ EFI \ CLOVER \ドライバ \ UEFI。念のため、両方のフォルダーにコピーして、現在使用しているBIOS、古いBIOSまたは古いUEFIを理解できないようにします。

コンピューターを再起動し、Cloverフラッシュドライブから起動し、起動オプションを示すアイコンの数を画面に表示します。CloverがNVMe SSDを認識できるようになったため、アイコンがさらに表示されるはずです。
物語の半分は終わりました！

新しいNVMe SSDへの新しいOSのインストール


さらに、高速NVMe SSDからの起動を利用するには、実際にシステムをインストールする必要があります。そして、ニュアンスがあります。将来的には、クローバーからNVMeディスクから起動しようとして問題が発生しないようにするため、現在のシステムを新しいディスクに転送せずに、NVMeディスクに新しいシステムを最初からインストールすることを強くお勧めします。問題は、コンピューターにインストールされたときに選択および使用されたオペレーティングシステムのブートローダーのタイプに戻ります。NVMeをサポートしないBIOSを搭載したシステムがある場合、現在のブートローダーはEFIではなくBIOSに重点を置いています。
Cloverが制御を移すのはこのブートローダーであり、EFIエミュレーションモードです。参考までに、私のWin64クローバーローダーはここのクローバーフラッシュドライブにありますG：\ EFI \ BOOT \ BOOTX64.efi
そのようなOSをNVMeディスクに転送し、それを使用してEFIモードで起動するだけでは機能せず、OSローダーを手動で変更する必要があります。これは時間がかかり、完全に正当化されていない、IMHOではありません。

そのため、新しいOSのライセンスキーを大胆に購入し（まあ、または古いOSがどこにあるかを見つけます）、メディア作成ツールのダウンロードページに移動して、Windowsオペレーティングシステムのブートディスクを作成します。事実、しばらくの間、Microsoftはメディア作成ツールによって作成されたイメージにBIOSとEFIの両方のローダーを含め始めました。そして今、この起動可能なフラッシュドライブは、レガシーBIOS、UEFI、EFIを備えたすべてのシステムで起動できます。そして以前は、結果のブートイメージを処理し、Rufusを統合してEFIモードでブートする必要がありました（例、中央までスクロールします）。

注意


はい、NVMe SSDにOSをインストールするには、2番目の起動可能なUSBフラッシュドライブが必要です。
いいえ、WindowsディストリビューションをCloverを使用してフラッシュドライブに書き込むことはできません。

OSインストール


新しいNVMe SSDへのオペレーティングシステムのインストールは、非常に定期的かつ迅速です。
2つのフラッシュドライブを挿入して起動する必要があり、Windowsディストリビューションからではなく、クローバーフラッシュドライブから起動する必要があります。混同しないでください。

Cloverインターフェイスで、OSのディストリビューションを備えたフラッシュドライブからのブートを見つけます。次に、OSのインストールが開始されます。ある時点で、システムが再起動し（Windowsをインストールします）、NVMe SSDからのダウンロードを続行することを選択する必要がありますが、混乱しないでください。 OSディストリビューションから再起動することを選択しないでください。再起動しないと、インストールが再開されるだけで、SSDを続行し、できれば完了する必要があります。

新しいNVMe SSDに新しいシステムをインストールすると、Cloverに新しいOSがインストールされたという記録が表示されます。また、新しいOSの起動アイコンがダウンロードのリストの最初に表示され、何も混乱させない場合は、「Boot Microsoft EFI Boot from EFI」と呼ばれます。



そして、OSの最終インストール後、USBフラッシュドライブとその配布を削除して、クローバーと「マイコンピューター」に表示されないようにします。

また、BIOSでは、ハードウェアをアップグレードしてEFIをサポートするまで、ディスクの起動順序は常にクローバーフラッシュドライブから開始する必要があります。

そして、それから初めて、誇らしげにあなたの最初の製品を手に入れることができるようになります。。

実際はそれだけです
