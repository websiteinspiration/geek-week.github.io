分布で並べ替え



分布ソートでは、配列がソートされるまで、要素がクラスに分散および再分散されます。

最も一般的なケースでは、これはほぼ同じ方法で発生します。何らかの理由で要素がクラスに分散しています。これが配列の順序付けにつながらない場合は、クラスの属性が洗練され、要素は洗練されたクラスに再び分散されます。そして、これは配列が注文されるまで起こります。

分布による分類では、ほとんどの場合、それらの間の要素の比較とそれらの交換はありません。主なことは、要素が特定のクラスに属しているかどうかであり、他の要素との比較がほとんど役割を果たすことはありません。

通常、これらの並べ替えは線形の時間の複雑さを持っています（交換、マージ、選択、または挿入の効率的な並べ替えのような対数ではありません）。また、クラスごとにグループ化された要素はどこかに保存する必要があるため、このクラスのアルゴリズムでは、ほとんどの場合、大量の追加メモリが必要になります。

分布ソートは、整数と文字列の順序付けに適しています。それらを使って実数をソートすることは通常不便です。また、分布は繰り返し数で構成される配列を完全に並べ替えます。繰り返しが多いほど、必要なクラスが少なくなります。

バケットソート::バケットソート


他の名前はバスケットソート、ブロックソート、ポケットソートです。

バスケットに数値を散布し、次に各バスケットに小さいバスケットに散布する、というように、バスケット内のあるレベルでは同じ要素のみが存在するようになります。次に、このような最下位レベルのバスケットから、アレイを順序付けられた状態に復元するのは簡単です。

具体例で説明します。順序付けられていない配列があるとします。この配列には1から8までの数字が含まれていることが知られています。



これらの数値を2つのグループに分けます。1から4の数値は1つのグループに分類され、5から8は2番目のグループに分類されます。次に、最初のバスケットの数値を2つのバスケットに分散します。また、これらのバスケットを、同じサイズの数がすでに含まれている靱皮バスケットに配布します。5から8までの数字を含む大きなバスケットに、同様の再帰を適用します。

次に、それぞれが同じ番号を含む小さなバスケットから、優先順位に従って要素をメイン配列に返します。

この形式での核の選別は実際には特に適用可能ではありませんが、流通によるすべての選別が一般的にどのように機能するかを標準的に示しています。

タノスソート::タノスソート


時々著者のソートが私に送られますが、これはまさにそのような場合です。著者Andrei Danilinはそれを「ロシアの半分での並べ替え」と呼んでいましたが、私はそれをThanosの並べ替えと呼んでいました。または、使用した方法から正式に進む場合、その算術平均ソートと呼ぶことができます。



要素の算術平均が配列で計算され、すべての要素が2つのグループに分散されます。算術平均より小さい（または等しい）要素は1つのグループに移動し、算術平均より大きい要素は2番目のグループに移動します。次に、同じアクションが両方のグループに再帰的に適用されます。

そしてクレイジーチタンはどうですか？これがランダムな配列である場合、概して、要素は、算術平均と比較すると、2つのグループのいずれかに移動する可能性が50/50の可能性があります。

ちなみに、インターネットで同じ名前の別のコミックアルゴリズムを見つけました。配列が並べ替えられていない場合は、インフィニティグローブをクリックして、ランダムに選択された配列要素の半分を非存在に送信します。生存者が秩序だった配列を形成する場合、これで彼らの大きな使命は達成されたと見なすことができます。そうでない場合は、さらに数回クリックすることができます。



しかし、私たちの分布の分類に戻ります。それらはすべて、カウントソートとビットごとのソートという 2つのグループにのみ分類できます。必要に応じて、カウントビットの並べ替えを強調表示することもできます。両方に起因する可能性があります。

ハイブリッドアルゴリズムもあります（これらは、異なるクラスのメソッドを使用するものです。たとえば、Timの並べ替えは、マージによる並べ替えと挿入による並べ替えの組み合わせ、内省的な並べ替えは、束による並べ替えに入るクイック並べ替えなどです）。ただし、ハイブリッドは別のセクションです。それらについては後で。

ソートのカウント


主なアイデアは、各クラスの数を数えることです。

カウントソート::カウントソート


配列で1つまたは別の数が発生する回数をカウントします。これらの数量がわかっているため、すぐに注文済みの配列を形成します。



この並べ替えでは、配列の最小値と最大値を知る必要があります。次に、補助配列のキーが生成されます。そこでは、何が何回出会ったかを修正します。

Pythonコード：

def CountingSort(array, mn, mx):

	count = defaultdict(int)
	
	for i in array:
		count[i] += 1
		
	result = []
	
	for j in range(mn,mx+1):
		result += [j]* count[j]
		
	return result


ピジョンソート::ピジョンホールソート


配列を調べ、新しい番号が見つかった場合は、この番号の（補助リストのキーとして）カウンターを開始します。初めてではない数値が検出された場合、このカウンターの増分は単にトリガーされます。



前の方法との違いは、カウントによるソートでは、配列で発生する可能性のあるすべての可能性のある数のカウンターをすぐに開始することです（配列の最大値と最小値がわかっている場合は、余裕があります）。一部の数値は決して発生せず、それらのカウンターはゼロを示します。ハトの並べ替えでは、配列で実際に発生する数値に対してのみカウンターを開始します。配列はカウンターの並べ替えのカウントに使用され、二重リンクリストは鳩の並べ替えに使用されます。これにより、外出先で新しいカウンターを追加できます。

このメソッドは、代わりに呼び出されることもありますアルゴリズム自体がディリクレ原理のさまざまな結果を示すものであるため、ディリクレ分類。
N個のオブジェクトがM個のコンテナに分散され、N> Mの場合、少なくとも1つのコンテナに複数の要素が含まれます。

Pythonコード：

def PigeOnHoleSort(a):

	mi = min(a)
	size = max(a) - mi + 1
	holes = [0] * size
	
	for x in a:
		holes[x - mi] += 1
		
	i = 0
	for count in range(size):
		while holes[count] > 0:
			holes[count] -= 1
			a[i] = count + mi
			i += 1


ビットソート


数字の特定のカテゴリにある数字に応じて、数字を分配します。異なる数字に対してこれを数回行うと、ソートされた配列が突然得られます。

低次ビットごとの並べ替え:: LSD基数並べ替え



下の桁から古い桁に移動し、各反復で、桁に含まれる桁に応じて配列の要素を分散します。

次の配布後、次の再配布時に要素がクラスに分類された順序で要素をメイン配列に返します。

ビットごとの並べ替えでは、要素が同じ桁数として扱われることが重要です。実際の桁数が異なる場合は、上位の桁としてゼロを追加することで問題が解決します。

高次ビットごとの並べ替え:: MSD基数並べ替え



まず、シニアランクに分配し、そこから若いランクに移動します。

下位桁への遷移は、配列のすべての要素間ではなく、クラス内で再帰的に実行されるため、このオプションの実装はより困難です。

しかし、この複雑さは、MSDがLSDよりも速いという事実によって報われます。下の桁から上の桁に渡す場合、正しくソートするには、すべての数字のすべての桁を処理する必要があります。先輩から後輩に移る場合、実際にはすべての数字のすべての桁を処理する必要はありません。ソートされた状態は、原則として、早くなります。

ほとんどのビット単位の並べ替えは、より効率的なMSDのバリエーションです。これは文字列をソートするのに特に役立ちます;このため、通常は接尾辞ツリーが使用されます。以下の記事のいずれかで分析します。

ビーズソート::ビーズソート



アルゴリズムの他の名前：そろばんソート、重力ソート。

私はすでに、これは（数回の選別について書いた1、2）ので、私は簡単な、唯一の本質だろう。

配列の各数値がボールのセットであるとすると、ボールの数は数値の値です。これらのボールの水平行として数字を並べてから、垂直方向にプッシュすると、順序付けられた配列になります。

ここでの秘訣は、単項数体系でボールを使用して各数値を表すことです。そして実際には、すべての数値が各桁に何回あるかを数えるだけです。

PythonのBeadSortを1行で：

#!/bin/python3
from itertools import zip_longest
 
 
def beadsort(l):
    return list(map(sum, zip_longest(*[[1] * e for e in l], fillvalue=0)))
