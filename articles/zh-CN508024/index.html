XGBoost使用Apache Spark进行分布式学习和并发预测

哈Ha！已经处于七月底，领角启动一个新的过程中，“工业ML的大数据。” 传统上，由于期望新课程的开始，我们为您准备了有用材料的翻译。







一般信息



在增强（来自一组机器学习模型）中，算法执行顺序过程（与并行处理的装袋相反），生成弱学习算法并将其与强学习算法结合（与所有集成方法一样）。在提升过程的每次迭代时，该模型都尝试自适应地校正前一次迭代的错误，这与装袋不同，在装袋中，弱学习算法是独立训练的。







梯度提升是一种增强算法，它使用梯度下降直接在这些顺序模型中最小化损失函数（与AdaBoost算法不同，后者通过更改训练实例的权重进行训练）。



在训练期间的梯度提升期间创建的弱学习算法通常被实现为决策树。梯度增强中最无效的是创建树的顺序过程，因为在这种情况下，一次只能创建一棵树。



为了克服此限制，Tianji Chen和Carlos Gestrin提出了一种名为XGBoost的梯度增强算法的改进，该算法代表“极限梯度增强”或“极限梯度增强”。这是对类固醇的一种梯度增强，主要用于分类，但有时也用于回归和排名。



与标准梯度提升相比，由于超参数，GPU支持，算法的交叉验证和正则化，新方法大大提高了性能。通常，该模型效率更高，学习速度更快，并且不易再训练。



最近，XGBoost受到广泛欢迎，并在Kaggle赢得了许多机器学习比赛。相信它具有强大的计算能力和准确性。







XGBoost和Apache Spark



在ML中的标准工作流程中，使用Spark之类的系统创建机器学习管道，在该管道中进行预处理和清理数据，然后将结果传递到机器学习阶段，如果您已经在使用Spark，则通常使用Spark MLlib。



在本文的上下文中，重要的是XGBoost必须并行化树的构建过程，从而允许在节点之间进行分布式训练和预测。也就是说，如果我作为Apache Spark MLlib的用户可以使用它来扩展学习XGBoost和进行生产的可能性，那么实际上，我可以享受XGBoost的高性能以及Spark用于工程特性和构建ML管道的强大机制。



认识XGBoost4J-Spark，该项目集成了XGBoost和Apache Spark，并将XGBoost添加到Apache Spark MLlib框架中。



XGBoost4J-Spark使构建MLlib管道成为可能，该管道在训练XGBoost模型之前对其进行预处理，然后对其进行训练，并可用于生产的并行预测。使用该库，每个XGBoost工作者都将变成一个Spark任务，而来自Spark内存的训练数据集将发送给在Spark美术师中不可见地存在的XGBoost工作者。







要开始在XGBoost4J-Spark上编写机器学习应用程序，您首先需要添加适当的依赖项： 



<dependency>
    <groupId>ml.dmlc</groupId>
    <artifactId>xgboost4j-spark</artifactId>
    <version>0.90</version>
</dependency>


数据准备（虹膜示例）



如前所述，XGBoost4J-Spark允许您在XGBoost接口下“拟合”数据。



一旦考虑了DataFrame中的Iris Flowers数据集，我们将需要：



  • 将列从String转换为Double；
  • 将特征列组合成向量，以便数据与Spark机器学习框架接口匹配。


import org.apache.spark.ml.feature.StringIndexer
import org.apache.spark.ml.feature.VectorAssembler
val stringIndexer = new StringIndexer().
  setInputCol("class").
  setOutputCol("classIndex").
  fit(irisDF)
val labelTransformed = stringIndexer.transform(irisDF).drop("class")
val vectorAssembler = new VectorAssembler().
  setInputCols(Array("sepal length", "sepal width", "petal length", "petal width")).
  setOutputCol("features")
val xgbInput = vectorAssembler.transform(labelTransformed).select("features", "classIndex")


在上面的DataFrame中，结果将是两列，“功能”：代表虹膜符号的矢量，“ classIndex”：类型Double的标签。这样的DataFrame可以安全地馈送到XGBoost4J-Spark学习引擎。



分布式培训



import ml.dmlc.xgboost4j.scala.spark.XGBoostClassifier

val xgbClassifier = new XGBoostClassifier().
      setFeaturesCol("features").
      setLabelCol("classIndex").
      setObjective("multi:softmax")
      setMaxDepth(2).
      setNumClass(3).
      setNumRound(100).
      setNumWorkers(10).


XGBoost选项的完整列表可以在此处找到。请注意，在XGBoost4J-Spark中，您也可以使用camelase，如上例所示。

现在，我们可以通过在输入DataFrame上训练XGBoost分类器来创建转换器。作为学习过程的结果，我们获得了可用于进行预测的模型。 



val xgbClassificationModel = xgbClassifier.fit(xgbInput)


并行预测



XGBoost4j-Spark支持批量预测和现场预测。



对于批量预测，模型采用一个DataFrame，其中的一列包含特征向量，对每个特征向量进行预测，并显示包含结果的新DataFrame。在此过程中，XGBoost4J-Spark使用XGBoost工作程序为输入DataFrame的每个部分启动Spark任务，以进行并行批处理预测。 



val predictionsDf = xgbClassificationModel.transform(inputDF)
predictionsDf.show()
