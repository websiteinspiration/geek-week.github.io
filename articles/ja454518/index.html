Wasmer：WebAssemblyコードを実行するための最速のGoライブラリ

WebAssembly（wasm）は、移植可能なバイナリ命令フォーマットです。同じコードwasmコードは、どの環境でも実行できます。このステートメントをサポートするには、すべての言語、プラットフォーム、およびシステムがそのようなコードを実行でき、可能な限り高速かつ安全である必要があります。


WasmerはRustで書かれたwasmランタイムです。もちろん、wasmerはどのRustアプリケーションでも使用できます。今日私たちが翻訳を公開している資料の著者は、彼とWasmerプロジェクトの他の参加者が、このwasm-codeランタイムを他の言語で正常に実装したと述べています。

  • でCおよびC ++、これはバインディングを介して実装されます。
  • PHPでは、これはphp-ext-wasmの拡張です。
  • Pythonでは、これはpython-ext-wasmリポジトリで作業中のPyPiで公開されたwasmerパッケージです。
  • Rubyでは、このwasmer gem はRubyGemsにあります。そのコードはruby-ext-wasmリポジトリにあります。

ここでは、バイナリwasm-codeを実行するように設計されたGoのライブラリである新しいプロジェクトgo-ext-wasmについて説明します。結局のところ、go-ext-wasmプロジェクトは他の同様のソリューションよりもはるかに高速です。しかし、自分たちより先を行かないようにしましょう。彼と一緒に働く方法の話から始めましょう。

Goからwasm関数を呼び出す


まず、wasmerをGo環境（cgoサポートあり）にインストールします。

export CGO_ENABLED=1; export CC=gcc; go install github.com/wasmerio/go-ext-wasm/wasmer

ゴー-EXT-wasmプロジェクトは、通常の囲碁ライブラリです。このライブラリを使用する場合、構造が使用されimport "github.com/wasmerio/go-ext-wasm/wasmer"ます。

それでは、練習を始めましょう。wasmでコンパイルする簡単なプログラムを作成します。これには、たとえばRustを使用します。

#[no_mangle]
pub extern fn sum(x: i32, y: i32) -> i32 {
    x + y
}

このプログラムsimple.rsでファイルに名前を付けます。このプログラムのコンパイルの結果、ファイルsimple.wasmが取得されます。

Goで記述された次のプログラムはsum、wasmファイルから関数を実行し、引数として5と37を渡します。

package main

import (
    "fmt"
    wasm "github.com/wasmerio/go-ext-wasm/wasmer"
)

func main() {
    //   WebAssembly.
    bytes, _ := wasm.ReadBytes("simple.wasm")

    //    WebAssembly.
    instance, _ := wasm.NewInstance(bytes)
    defer instance.Close()

    //    `sum`   WebAssembly.
    sum := instance.Exports["sum"]

    //        Go.
    //   ,      ,  .
    result, _ := sum(5, 37)

    fmt.Println(result) // 42!
}

ここでは、Goで記述されたプログラムが、Rustで記述されたコードをコンパイルして得られたwasmファイルから関数を呼び出します。

したがって、実験は成功しました。GoでWebAssemblyコードを正常に実行しました。データ型変換は自動化されていることに注意してください。wasmコードに渡されるGo値は、WebAssembly型にキャストされます。wasm関数が返すものはGoタイプにキャストされます。その結果、Goでのwasmファイルの関数の操作は、通常のGo関数の操作と同じように見えます。

WebAssemblyコードからGo関数を呼び出す


前の例で見たように、WebAssemblyモジュールは外部から呼び出すことができる関数をエクスポートできます。これは、さまざまな環境でwasmコードを実行できるようにするメカニズムです。

同時に、WebAssemblyモジュール自体がインポートされた関数で動作できます。 Rustで書かれた次のプログラムを考えてみましょう。

extern {
    fn sum(x: i32, y: i32) -> i32;
}

#[no_mangle]
pub extern fn add1(x: i32, y: i32) -> i32 {
    unsafe { sum(x, y) } + 1
}

それでファイルに名前を付けましょうimport.rs。それをWebAssemblyにコンパイルすると、ここにあるコードが生成されます。

エクスポートされた関数add1は関数を呼び出しますsum。この関数の実装はなく、シグネチャのみがファイルで定義されています。これは、いわゆるextern関数です。WebAssemblyの場合、これはインポートされた関数です。その実装をインポートする必要があります。Goを使用し

て関数を実装しますsum。これには、cgoを使用する必要があります。これが結果のコードです。メインコードフラグメントの説明であるコメントには、番号が付けられています。以下では、それらについて詳しく説明します。

package main

// // 1.    `sum` (   cgo).
//
// #include <stdlib.h>
//
// extern int32_t sum(void *context, int32_t x, int32_t y);
import "C"

import (
    "fmt"
    wasm "github.com/wasmerio/go-ext-wasm/wasmer"
    "unsafe"
)

// 2.    `sum`    ( cgo).
//export sum
func sum(context unsafe.Pointer, x int32, y int32) int32 {
    return x + y
}

func main() {
    //   WebAssembly.
    bytes, _ := wasm.ReadBytes("import.wasm")

    // 3.     WebAssembly.
    imports, _ := wasm.NewImports().Append("sum", sum, C.sum)

    // 4.     WebAssembly  .
    instance, _ := wasm.NewInstanceWithImports(bytes, imports)

    //    WebAssembly.
    defer instance.Close()

    //    `add1`   WebAssembly.
    add1 := instance.Exports["add1"]

    //   .
    result, _ := add1(1, 2)

    fmt.Println(result)
    //   add1(1, 2)
    // = sum(1 + 2) + 1
    // = 1 + 2 + 1
    // = 4
    // QED
}

このコードを解析してみましょう：

  1. 関数のシグネチャはsumC で定義されています（コマンドの上のコメントを参照import "C"）。
  2. 関数の実装はsumGo で定義されています（行に注意してください。//exportこのようなcgoメカニズムは、Goで書かれたコードとCで書かれたコードの接続を確立するために使用されます）。
  3. NewImportsWebAssemblyインポートの作成に使用されるAPIです。このコードでは"sum"、これはWebAssemblyによってインポートされた関数の名前でありsum、Go関数C.sumへのポインターであり、cgo関数へのポインターです。
  4. 最後に、NewInstanceWithImportsこれはインポートでWebAssemblyモジュールを初期化するように設計されたコンストラクターです。

メモリからのデータの読み取り


WebAssemblyインスタンスには線形メモリがあります。そこからデータを読み取る方法について話しましょう。いつものように、名前を付けるRustコードから始めましょうmemory.rs。

#[no_mangle]
pub extern fn return_hello() -> *const u8 {
    b"Hello, World!\0".as_ptr()
}

このコードをコンパイルした結果は、memory.wasm以下で使用されるファイルにあります。

関数return_helloは文字列へのポインタを返します。 Cの場合と同様に、行はヌル文字で終了します。

次にGo側に移動します。

bytes, _ := wasm.ReadBytes("memory.wasm")
instance, _ := wasm.NewInstance(bytes)
defer instance.Close()

//    `return_hello`.
//      .
result, _ := instance.Exports["return_hello"]()

//      .
pointer := result.ToI32()

//    .
memory := instance.Memory.Data()

fmt.Println(string(memory[pointer : pointer+13])) // Hello, World!

この関数return_helloは、ポインタを値として返しますi32。この値を取得するには、を呼び出しToI32ます。次に、を使用してメモリからデータを取得しますinstance.Memory.Data()。

この関数は、WebAssemblyインスタンスのメモリスライスを返します。Goスライスと同じように使用できます。

幸い、読みたい行の長さがわかっているので、必要な情報を読み取るには、構文を使用するだけで十分memory[pointer : pointer+13]です。次に、読み取られたデータが文字列に変換されます。
