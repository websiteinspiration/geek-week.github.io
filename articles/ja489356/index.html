AWS、Azure、GitlabのCI / CD。OTUSからの新しいコース

注意！この記事はエンジニアリングではなく、CI / CDの分野の教育に関心のある読者を対象としています。ほとんどの場合、学習に関心がない場合、この資料はあなたにとって興味がありません。



あなたが設定を担当、開発者や管理者の場合は、継続的インテグレーション/連続配信のプロセスを、そしてOTUSはあなたのための特別コースのセットをオープンしました：実用的な集中コースを継続的発展と継続的インテグレーションの配信と連続配信ソフトウェアの一般的な方法でアマゾンウェブサービス、Azure、GitLab、およびJenkinsプラットフォーム全体。

トレーニング中、学生は、3つの主要プロバイダーによるアプリケーションのアセンブリおよびテストプロセスとインストールプロセスの構成方法を学習し、クラウドプロバイダーのアーキテクチャについて理解し、コード分析と脆弱性検索の自動化について学びます。

トレーニングの結果、各学生は卒業作品を作成します。これは、オープンソースプロジェクトが選択できるCI / CDプロセスの実装になります。もちろん、トレーニング後、各生徒はすべてのクラスの資料、修了証明書を受け取ります。最も重要なのは、アプリケーションの構成とテストを構成し、脆弱性を見つけることができるようにすることです。



もちろん、このコースはすべての人を対象としているわけではありません。しかし、経験がある場合：

  • gitを使用する
  • LinuxまたはWindowsシステム管理
  • 開発または運用
  • クラウドプロバイダーと連携する

その後、OTUSがあなたを待っています！あなたはできる入学試験を受け、あなたがAWI、アズール、およびGitlab CI / CDのコースを完了するのに十分な知識を持っているかどうかを判断します。

開始の前夜に「AWS、AzureのとGitlabのためのCI / CD」もちろん、OTUSはオープンデーを開催しました。教師はコースプログラムについてより詳細に話し、生徒からの質問に答え、学習プロセスについても説明しました。


また、AWS、Azure、GitlabのCI / CD コースの講師であるBoris Nikolaevが実施した「K8SでのJenkinsの使用」に関するオープンウェビナーの無料視聴も利用できます。


コース「CI / CD on AWS、Azure、Gitlab」のトレーニングプロセスは、オンラインウェビナーの形式で行われます。トレーニング中（3か月間）、生徒は常に連絡を取り合っている経験豊富な教師に質問できます。演習は、Google Cloud Platform（GCP）、Amazon Web Service、およびMicrosoft Azureを使用して完了します。

コースプログラムは4つのメインモジュールで構成されています。

  1. クラウドでの開発（コード）
  2. 組み立てとテストの自動化（継続的インテグレーション）
  3. インストールの自動化（継続的デリバリー）
  4. 要約モジュール

それぞれがオンラインウェビナーの形式でクラスで完全に理解され、宿題は得られた知識を統合するのに役立ち、必要に応じて、教師から詳細なフィードバックを得ることができます。

多くの専門家は、CI / CDを現代のタスクに最適なソフトウェア開発手法の1つと呼んでいます。この声明に同意しますか？
