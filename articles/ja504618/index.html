Opera Miniブラウザ


Nokia 3310では、Opera Miniブラウザを使用してWebサイトを閲覧できます。このブラウザにはさまざまなバージョンがあります。ページのレンダリング方法は、オペレーティングシステム、デバイス、および使用する設定によって異なります。このブラウザーをスマートフォンにインストールすると、Opera MiniはAndroidまたはiOSで利用可能なブラウザーエンジンを使用するため、新しいものは何も表示されない場合があります。このブラウザには1つの興味深い機能があります。実際には、トラフィックを節約するためのさまざまなパラメーターを設定できます。節約はオフにすることができ、自動的に行うことができ、節約は非常に大きくなる可能性があります。極端なトラフィック節約モードでページを表示すると、リクエストはOperaのプロキシサーバーの1つに送られます。このサーバーは、ページをロードして処理し、圧縮してから、ユーザーに送信します。オペラの代表は言う送信されるデータの量が最大90％削減されます。 JavaScriptコードはサーバーによってのみ処理され、デバイスは完成したページのみを表示するため、これによりページのインタラクティブ機能が制限されます。

Opera MiniでのJavaScript処理に関するその他の注目すべき事実は次のとおりです。

  • すべてのスクリプトの実行時間は2秒に制限されています。
  • 機能setIntervalおよびsetTimeout無効。
  • スクリプトの実行をトリガーするイベントの数には制限があります。

Opera Miniはパフォーマンスとトラフィックの節約に非常に重点を置いています。これがこのブラウザがNokia 3310にインストールされた理由の1つだと思います。ただし、この電話にインストールされているブラウザは、スマートフォンにインストールされているブラウザとは異なります。これは、caniuse.comがその機能を説明するために通常赤を使用するブラウザと同じです。


Opera Mini はCSSとJS機能の限られたセットのみをサポートします。

漸進的改善方法論の使用方法


Web開発における「進歩的な改善」とは何かを理解するのに役立ついくつかの実用的な例があります。

▍グリッドレイアウト


フロントエンドブックマーク プロジェクトのフロントページの2列のレイアウトを作成するために、CSSグリッドテクノロジーを使用しました。ブラウザがそれが何であるかを知らない場合display: gridは、単一列レイアウトの形式のフォールバックが単に使用されます。ブラウザは理解できないプロパティを無視するだけなので、CSSで@ supports-requestsを使用する必要がないことに注意してください。

ul {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(29rem,1fr));
  grid-gap: .7rem;
}

▍検索


最近のブラウザーでは、プロジェクトページの上部にコンボボックスが表示され、ページの移動、フィルター、およびページの選択ができます。私のJavaScriptコードでは、矢印関数、テンプレートリテラル、および言語の他の現代的な機能を使用したいと思っていました。同時に、組み込みのサポートがないブラウザーでこれらの機能をサポートするために、ポリフィルを使用したくありません。そのため、ES2015 +を理解しているブラウザにのみJSコードを送信します。タグに<script>属性を追加することでこれを実現しますtype=«module»。

<script src="/assets/js/scripts.min.js" type="module"></script>

ブラウザは、ESモジュールをサポートする場合にのみスクリプトを実行し、対応するブラウザもES2015 +機能をサポートします。フィリップウォルトンは、2017年にこの記事でこの手法を紹介しました。

▍Webp画像


他のグラフィック形式に対するwebp形式の利点は、ファイルサイズにあります。Webpファイルは通常、他の形式で保存されたファイルよりもはるかに小さくなります。残念ながら、SafariとIE はこの形式をサポートしていないため、すべてのjpg画像を取得してwebpバージョンで置き換えることはできません。しかし、要素を使用して適切な画像を個別に選択する機能をブラウザに与えることができます<picture>：

<picture>
  <source srcset="image.webp" type="image/webp"> 
  <img src="image.jpg" alt="image description"> 
</picture>

ブラウザは要素のコンテンツを<picture>上から下に読み取ります。ブラウザがwebp形式をサポートしている場合は、要素で説明されている対応する画像を使用します<source>。サポートしていない場合は、この要素をスキップして、要素で記述された画像を表示します<img>。
