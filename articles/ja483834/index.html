Debian：単にi386をamd64に変える

これは、再インストールせずに、32ビットDebian / Deabianベースのディストリビューション（64ビットの代わりに誤ってダウンロードする可能性があります）で64ビットアーキテクチャを整理する方法に関する短い記事です。

*ハードウェアは最初にamd64をサポートする必要があります。誰も魔法をかけるつもりはありません。
*これはシステムに損傷を与える可能性があるため、十分注意してください。
*すべてがDebian10-buster-i386でテストされました。
*ここで少なくとも何かが理解されていない場合は、これを行わないでください。

Dpkg、apt、sources.list


すぐに、すべてを秤量したクレイジーな人であれば、パッケージの準備を始めます（原則として、ここでは順序は関係ありませんが、ポイントの方が便利です）

1. /etc/apt/sources.listでamd64を選択し、間に[[arch = amd64] 'を挿入しますdeb \ deb-srcおよびURL

例
# Base reps
deb [arch=amd64] http://deb.debian.org/debian/ buster main contrib non-free
deb-src [arch=amd64] http://deb.debian.org/debian/ buster main contrib non-free

# Update reps
deb [arch=amd64] http://deb.debian.org/debian/ buster-updates main
deb-src [arch=amd64]  http://deb.debian.org/debian/ buster-updates main

# Security reps
deb [arch=amd64] http://security.debian.org/debian-security/ buster/updates main
deb-src [arch=amd64] http://security.debian.org/debian-security/ buster/updates main


これは、将来64ビットのパケットのみがダウンロードされるようにするために必要です。

2. dpkgにamd64を追加して、誓わないようにします。

$ sudo dpkg --add-architecture amd64

3.パッケージリストを更新します。

$ sudo apt update

芯


もちろん、これはすべて64ビットカーネルなしでは意味がありません。インストールしてください。

$ sudo apt install linux-headers-$VERSION-amd64 linux-image-amd64

$ VERSIONを配置して、正しいカーネルバージョンを置き換えます。

カーネルのインストール後、grubは自動的に再構成します。

完了


再起動後、システムはすでにamd64で動作しますが、パッケージで問題が発生する可能性があります。それらを解決するには、次のコマンドを実行するだけで十分です。

$ sudo apt --fix-broken install
$ sudo apt full-upgrade

これについてもあまり気にする必要はありませんが、必要なパッケージはすべて依存関係としてインストールされ、不要なパッケージは次のように削除されます。

$ sudo apt autoremove

上記の操作の後、 '[arch = amd64]'を削除してsources.listを再度修正
し、パッケージに問題がないようにすることができます。
これで、64ビットシステムを自由に使用できます。
