Windows 10にApache Airflowをインストールした経験

前文：学術科学（医学）の世界からの運命の意思によって、私は情報技術の世界に入りました。そこでは、実験を構築する方法論と実験データを分析するための戦略に関する知識を使用する必要がありますが、新しい技術スタックを適用してください。これらの技術を習得する過程で、幸いにもこれまでに克服された多くの困難に直面しています。おそらくこの投稿は、Apacheプロジェクトでの作業を始めたばかりの人にも役立つでしょう。

つまり、要点は。記事に触発されましたYuri Emelyanovは、分析手順の自動化の分野におけるApache Airflowの機能について、提案されたライブラリーのセットを自分の作業で使用したいと思いました。 Apache Airflowにまったく詳しくない人は、国立図書館のウェブサイトにちなんで名付けられた短いレビュー記事に興味があるかもしれません。N.E.バウマン。

Airflowを起動するための通常の手順は明らかにWindows環境には適用されないため、この問題を解決するためにdockerを使用することは冗長になるため、他の解決策を探し始めました。幸いなことに、私は最初にこの道を進んでいなかったので、すばらしいビデオの説明を見つけることができましたDockerを使用せずにWindows 10にApache Airflowをインストールする方法。しかし、よくあることですが、推奨される手順を実行すると、問題が発生します。したがって、Apache Airflowのインストールの経験についてお話ししたいと思います。これにより、誰かの時間を節約できるでしょう。

指示のステップ（スポイラー-5番目のステップ、すべてがうまくいった）を見ていきましょう

。1. Linuxディストリビューションの後続のインストールのためのLinux用のWindowsサブシステムのインストール

問題が少ないほど、

コントロールパネル→プログラム→プログラムと機能→Windowsコンポーネントのオンとオフを切り替えます。 →Linux用のWindowsサブシステム

2.任意のLinuxディストリビューションをインストールするUbuntu

アプリケーションを使用しました。3.インストールと更新のpip



sudo apt-get install software-properties-common
sudo apt-add-repository universe
sudo apt-get update
sudo apt-get install python-pip

4. Apache Airflowをインストールする

export SLUGIFY_USES_TEXT_UNIDECODE=yes
pip install apache-airflow

5.データベースの初期化

これが私の小さな困難の始まりです。命令は、コマンドを入力airflow initdbして次のステップに進むように指示します。しかし、私はいつも答えを得ましたairflow: command not found。Apache Airflowのインストール段階で問題が発生し、必要なファイルが存在しないと想定するのは当然です。すべてが適切な場所にあることを確認したので、私はairflowファイルへのフルパスを指定することにしました（次のようになります）////airflow initdb。しかし、奇跡は起こらず、答えは同じairflow: command not foundでした。私はファイルへの相対パス（./.local/bin/airflow initdb）を使用しようとしましたが、これによりwerkzeugModuleNotFoundError: No module named json'ライブラリを更新することで解決できる新しいエラーが表示されました（私の場合はバージョン0.15.4に）：

pip install werkzeug==0.15.4

werkzeugの詳細については、こちらをご覧ください。

この簡単な操作の後、コマンド./.local/bin/airflow initdbは正常に完了しました。

6. Airflowサーバー

の起動Airflowへのアクセスに関する問題はまだ解消されていません。コマンドを実行./.local/bin/airflow webserver -p 8080するとエラーが発生しましたNo such file or directory。おそらく、経験豊富なUbuntuユーザーは、コマンドを適用してexport PATH=$PATH:~/.local/bin/（つまり、ディレクトリ/.local/bin/を実行可能ファイルの検索への既存のパスに追加して）、ファイルへのアクセスに関するこのような困難をすぐに克服しようとしますが、この投稿は、主にWindowsで動作し、そのようなソリューションを明白とは見なさない場合があります。

上記の操作の後、コマンド./.local/bin/airflow webserver -p 8080は正常に完了しました。

7. URL：localhost：8080 /

前の手順ですべてがうまくいけば、分析のピークを征服する準備が整います。
