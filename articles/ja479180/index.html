準備段階



Discrod
ボットの作成を始める前に、Discord用にボットを作成する必要があります。必要なもの：

  1. 不和のアカウントにログインで
  2. 「アプリケーション」タブで、「新しいアプリケーション」ボタンをクリックし、ボットに名前を付けます
  3. ボットにログインし、[設定]リストで[ボット]タブを見つけて、ボットトークンを取得します。
  4. どこかにトークンを保存

Wargaming

また、Wargaming APIにアクセスするには、Wargamingでアプリケーションを作成する必要があります。ここでも、すべてが簡単です。

  1. このリンクからWargamingアカウントに移動します
  2. 「マイアプリケーション」に移動し、「新しいアプリケーションを追加」ボタンをクリックして、アプリケーションの名前を入力し、そのタイプを選択します
  3. アプリケーションIDを保存

ソフトウェア

スペルロジック




アプリケーションのロジック全体は考慮せず、ボットによるメッセージインターセプトの操作方法とWargaming APIの操作方法のみを示します。

Discordボットの操作は、関数static async Task MainTask（string [] args）を介して行われます。
この関数を呼び出すには、Mainで登録する必要があります

MainTask(args).ConfigureAwait(false).GetAwaiter().GetResult();

次に、ボットを初期化する必要があります。

discord = new DiscordClient(new DiscordConfiguration
{
    Token = token,
    TokenType = TokenType.Bot,
    UseInternalLogHandler = true,
    LogLevel = LogLevel.Debug
});

ここで、tokenはボットのトークンです。
次に、ラムダを使用して、ボットが実行する必要のあるコマンドを記述します。
discord.MessageCreated += async e =>
{
    string message = e.Message.Content;
    if (message.StartsWith("&"))
    {
        await e.Message.RespondAsync(“Hello, ” + e.Author.Username);
    }
};

場所e.Author.Username-ユーザーのニックネームを取得します。

したがって、＆で始まるメッセージを送信すると、ボットはあなたを歓迎します。

この関数の最後に、await discord.ConnectAsync（）;を登録する必要があります。そしてTask.Delay（-1）を待ちます。

これにより、メインスレッドを使用せずにバックグラウンドでコマンドを実行できます。

次に、Wargaming APIを処理する必要があります。ここではすべてが簡単です。CURLリクエストを記述し、JSON文字列の形式で回答を取得し、そこから必要なデータを引き出して、それらを操作します。

WargamingAPIの使用例
public Player FindPlayer(string searchNickname)
        {
            //https://api.worldoftanks.ru/wot/account/list/?application_id=y0ur_a@@_id_h3r3search=nickname
            urlRequest = resourceMan.GetString("url_find_player") + appID + "&search=" + searchNickname;
            Player player = null;
            string resultResponse = GetResponse(urlRequest);
            dynamic parsed = JsonConvert.DeserializeObject(resultResponse);

            string status = parsed.status;
            if (status == "ok")
            {
                int count = parsed.meta.count;
                if (count > 0)
                {
                    player = new Player
                    {
                        Nickname = parsed.data[0].nickname,
                        Id = parsed.data[0].account_id
                    };
                }
                else
                {
                    throw new PlayerNotFound("  ");
                }
            }
            else
            {
                string error = parsed.error.message;
                if (error == "NOT_ENOUGH_SEARCH_LENGTH")
                {
                    throw new PlayerNotFound("   ");
                }
                else if (error == "INVALID_SEARCH")
                {
                    throw new PlayerNotFound(" ");
                }
                else if (error == "SEARCH_NOT_SPECIFIED")
                {
                    throw new PlayerNotFound(" ");
                }
                else
                {
                    throw new Exception("Something went wrong.");
                }
            }

            return player;
        }


注意！すべてのトークンとアプリケーションIDをクリアテキストで保存することは、絶対にお勧めできません。少なくとも-Discordは、そのようなトークンがワールドワイドネットワークに到達したときに最大で-ロボットを禁止します-ボットはサイバー犯罪者を使用し始めます。

VPSにデプロイ-サーバー




ボットを使い終わったら、24時間365日稼働しているサーバーにボットを配置する必要があります。これは、アプリケーションが機能するとボットも機能するという事実によるものです。アプリケーションをオフにするとすぐに、ボットはスリープ状態になります。

WindowsとLinuxの両方で、この世界には多くのVPSサーバーが存在しますが、ほとんどの場合、Linuxの方が数倍安価にホストできます。

Discordサーバーでは、vscale.ioのアドバイスを受け、Ubuntuですぐに仮想サーバーを作成してボットをアップロードしました。このサイトの仕組みについては説明しませんが、すぐにボットの設定に進みます。

まず、.NET Coreで記述されたボットを実行するために必要なソフトウェアをインストールする必要があります。これを行う方法はここで説明されています。

次に、ボットをGitHubなどのGitサービスにアップロードし、VPSサーバーに複製するか、別の方法でボットをダウンロードする必要があります。コンソールのみがあり、GUIがないことに注意してください。もちろんです。

ボットをダウンロードしたら、それを実行する必要があります。これを行うには、次のことを行う必要があります。

  • すべての依存関係を復元：dotnet restore
  • ビルドアプリケーション：dotnet build name_project.sln -c Release
  • ビルドされたDLLに移動します。
  • dotnet name_of_file.dll

おめでとう！ボットが実行されています。ただし、ボットは残念ながらコンソールを占有しているため、VPSサーバーを終了するのはそれほど簡単ではありません。また、サーバーを再起動する場合は、ボットを再起動する必要があります。状況から抜け出す方法はいくつかあります。これらはすべて、サーバーの起動時の起動に関連しています。

  • /etc/init.dにスクリプト実行を追加します
  • 起動時に開始するサービスを作成します。

それらを詳しく説明することには意味がありません。すべてがインターネットで十分に詳しく説明されています。
