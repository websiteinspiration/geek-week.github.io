Codebasis. RecyclerView erweitern



Hallo!



Mein Name ist Anton Knyazev , leitender Android-Entwickler bei Omega-R. In den letzten sieben Jahren habe ich professionelle mobile Anwendungen professionell entwickelt und komplexe Probleme der nativen Entwicklung gelöst.



Ich möchte die von unserem Team und mir entwickelten RecyclerView-Erweiterungsmethoden teilen. Sie werden zu einer zuverlässigen Basis für die Erstellung benutzerdefinierter Listen in Anwendungen.



Jede Anwendung ist aufgrund ihrer Idee, ihres Designs und ihres Spezialistenteams auf ihre Weise einzigartig. Erfolgreiche Entscheidungen möchten oft von einem Projekt auf ein anderes übertragen werden. Anstatt einfach zu kopieren, ist es daher logisch, eine separate Bibliothek zu erstellen, die das gesamte Team verwenden und verbessern würde.



Das Team beschloss, kleine Bibliotheken zu erstellen, die die Anwendungsentwicklung verbessern und beschleunigen, und sie in das öffentliche GitHub-Repository aufzunehmen . Dies erleichtert das Verbinden der Bibliothek in Projekten über JitPack und gibt Kunden die Garantie, dass der Code nichts „Kriminelles“ enthält.



Die erste Bibliothek, die wir auf GitHub gepostet haben, ist eine einfache Erweiterung von RecyclerView.



Beginnen wir mit den Problemen, die sie gelöst hat:



  1. Es gibt keinen Standard-layoutManager - dies ist unpraktisch, da Sie häufig denselben LinearLayoutManager auswählen müssen.
  2. Es gibt keine Möglichkeit, Teiler- und Elementbereich über XML hinzuzufügen. Dies ist auch unpraktisch, da Sie entweder das Elementlayout oder über ItemDecorator hinzufügen müssen.
  3. Sie können Kopf- und Fußzeilen nicht einfach über XML hinzufügen - dies ist nur über einen separaten ViewHolder möglich.


Die Probleme sind unkritisch, verursachen jedoch Unannehmlichkeiten und verlängern die Entwicklungszeit.



1. Problem: Es gibt keinen StandardlayoutManager



Die Entwickler von RecyclerView haben nicht die Möglichkeit bereitgestellt, den Standard-LayoutManager auszuwählen. Sie können layoutManager auf folgende Arten festlegen:



1. Über XML im App-Attribut: layoutManager = ”LinearLayoutManager”: 



<?xml version="1.0" encoding="utf-8"?>
<androidx.recyclerview.widget.RecyclerView
   ...
    app:layoutManager="LinearLayoutManager"/>


2. durch den Code: 



recyclerView.layoutManager = LinearLayoutManager(this)


Nach unserer Erfahrung wird in den meisten Fällen LinearLayoutManager benötigt.



Hier einige Beispiele für solche Auflistungen aus unseren ITProTV-, Simple World- und Dexen-Anwendungen:







Lösung: Fügen Sie den StandardlayoutManager hinzu



OmegaRecyclerView werden nur 3 Zeilen hinzugefügt: 



 if (layoutManager == null)  {
            layoutManager = LinearLayoutManager(context, attrs, defStyleAttr, 0)
 }


Wenn also LinearLayoutManager erforderlich ist, müssen Sie nichts hinzufügen, dh Sie können layoutManager vergessen. 



<?xml version="1.0" encoding="utf-8"?>
<com.omega_r.libs.omegarecyclerview.OmegaRecyclerView
    android:id="@+id/recyclerview"
    android:layout_width="match_parent"
    android:layout_height="match_parent" />


2. Problem: Es gibt keine Möglichkeit, Teiler- und Elementbereich über XML hinzuzufügen



Bei Verwendung von RecyclerView müssen Sie häufig Teiler hinzufügen. Zum Beispiel war in dem Projekt „Simple World“ einer der Bildschirme mit einem solchen nicht standardmäßigen Teiler versehen:







Aus diesem Layout ist ersichtlich, dass:



  • benutzte Trennlinie zwischen Elementen und ganz am Ende;
  • Platz für gebrauchte Gegenstände.


Wie kann dies standardmäßig in Android implementiert werden?



Methode 1



Der naheliegendste Weg besteht darin, den Teiler als ImageView-Element einzuschließen: 



 <RelativeLayout
   ...
   android:paddingStart="20dp"
   android:paddingTop="12dp"
   android:paddingEnd="20dp"
   android:paddingBottom="12dp">

...

   <ImageView
       ...
       android:layout_alignParentBottom="true"
       android:src="@drawable/divider"/>

</RelativeLayout>


Es kann vorkommen, dass Sie nur zwischen Elementen teilen müssen. In diesem Fall müssen Sie den letzten Teiler entfernen und den Code hinzufügen, um ihn im Adapter auszublenden.



Methode 2



Eine andere Möglichkeit besteht darin, die DividerItemDecoration zu verwenden, die dieser Teiler zeichnen kann. Dazu müssen Sie zusätzlich ein Drawable erstellen: 



<?xml version="1.0" encoding="utf-8"?>
<layer-list xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:left="32dp">

        <shape android:shape="rectangle">

            <size
                    android:width="1dp"
                    android:height="1dp" />

            <solid android:color="@color/gray_dark" />

        </shape>

    </item>

</layer-list>


Um einen Einzug hinzuzufügen, müssen Sie Ihre ItemDecoration schreiben: 



class VerticalSpaceItemDecoration(private val verticalSpaceHeight: Int): RecyclerView.ItemDecoration {

    override fun getItemOffsets(outRect: Rect, view: View, parent: RecyclerView,
            state: RecyclerView.State) {
        outRect.bottom = verticalSpaceHeight
    }

}


DividerItemDecoration ist einfach: Es zeichnet immer einen Teiler unter jedem Listenelement.

Im Falle einer Änderung der Anforderungen muss jedoch nach einer anderen Lösung gesucht werden.



Lösung: Fügen Sie die Möglichkeit hinzu, Teiler- und Elementbereich über XML hinzuzufügen



Daher sollte unser OmegaRecyclerView in der Lage sein, einen Teiler mit den folgenden Attributen hinzuzufügen:



  1. divider – drawable, ;
  2. dividerShow (beginning, middle, end) – , , ;
  3. dividerHeight – divider, ;
  4. dividerPadding, dividerPaddingStart, dividerPaddingEnd – : , , ;
  5. dividerAlpha – ;
  6. itemSpace – .


Alle diese Attribute werden mit zwei speziellen ItemDecoration direkt auf unsere OmegaRecyclerView angewendet.



Eine der ItemDecoration fügt eine Auffüllung zwischen den Elementen hinzu, die zweite zeichnet den Teiler selbst. Beachten Sie, dass bei einem Einzug zwischen den Elementen die zweite ItemDecoration einen Teiler in der Mitte des Einzugs zeichnet. Es gibt keine Möglichkeit, alle möglichen LayoutManager-Optionen zu unterstützen, daher unterstützen wir nur LinearLayoutManager und GridLayoutManager. Beachten Sie auch die Listenausrichtung für LinearLayoutManager.



Um den Code zu vereinfachen, wählen wir einen speziellen DividerDecorationHelper aus, der je nach Ausrichtung und Reihenfolge (umgekehrt oder direkt) relative Daten in Rect liest und schreibt. Es wird die Methoden setStart, setEnd und getStart, getEnd haben.



Erstellen wir eine grundlegende ItemDecoration für zwei Klassen, die die allgemeine Logik enthält, nämlich:



  1. Überprüfen, ob layoutManager der Nachkomme von LinearLayoutManager ist;
  2. Berechnung der aktuellen Ausrichtung und Reihenfolge;
  3. Bestimmen des geeigneten DividerDecorationHelper.


In SpaceItemDecoration überschreiben wir nur eine getItemOffset-Methode, die Einrückungen hinzufügt: 



override fun getItemOffset(outRect: Rect, parent: RecyclerView, helper: DividerDecorationHelper, position: Int, itemCount: Int) {
        if (isShowBeginDivider() || countBeginEndPositions <= position) helper.setStart(outRect, space)
        if (isShowEndDivider() && position == itemCount - countBeginEndPositions) helper.setEnd(outRect, space)
    }


Die nächste DividerItemDecoration zeichnet den Teiler direkt. Es sollte den Einzug zwischen den Elementen berücksichtigen und einen Teiler in der Mitte zeichnen. Definieren Sie zunächst die Methode getItemOffset für den Fall neu, dass der Einzug nicht angegeben ist, der Teiler jedoch zum Zeichnen erforderlich ist. 



    override fun getItemOffset(outRect: Rect, parent: RecyclerView, helper: DividerDecorationHelper, position: Int, itemCount: Int) {
        if (position == 0 && isShowBeginDivider()) {
            helper.setStart(outRect, dviderSize)
        }
        if (position != 0 && isShowMiddleDivider()) {
            helper.setStart(outRect, dividerSize)
        }
        if (position == itemCount - 1 && isShowEndDivider()) {
            helper.setEnd(outRect, dividerSize)
        }
    }


Wir fügen auch eine Option hinzu, mit der DividerItemDecoration den Adapter fragen kann, ob über oder unter dem ausgewählten Element gezeichnet werden kann. Um diese Funktion zu implementieren, erstellen Sie Ihren eigenen Adapter, der mit den folgenden Methoden vom Standardadapter erbt: 



   open fun isDividerAllowedAbove(position: Int): Boolean {
        return true
    }

    open fun isDividerAllowedBelow(position: Int): Boolean {
        return true
    }


Definieren Sie als Nächstes die onDrawOver-Methode neu, um den Teiler über die gezeichneten Elemente zu zeichnen. Bei dieser Methode müssen Sie alle auf dem Bildschirm sichtbaren Elemente (über getChildAt) durchgehen und gegebenenfalls diesen Teiler zeichnen. Es sollte auch beachtet werden, dass eine Farbe ohne Höhe aus dem Attribut dividerDrawable stammen kann. In diesem Fall kann die Höhe dem Attribut dividerHeight entnommen werden.



3. Problem: Sie können Kopf- und Fußzeilen nicht direkt über XML hinzufügen







In RecyclerView ist es nicht möglich, eine Ansicht über XML hinzuzufügen, es gibt jedoch andere Möglichkeiten, dies zu tun.



Methode 1



Eine naheliegende Möglichkeit, eine Ansicht hinzuzufügen, ist der Adapter. Darüber hinaus muss bei der Eingabe Ihrer Kennung für viewType im Adapter zwischen Kopf- und Fußzeile unterschieden werden. 



 fun onCreateViewHolder(parent: ViewGroup, viewType: Int): RecyclerView.ViewHolder? {
	  val inflater = LayoutInflater.from(parent.context)
        return when (viewType) {
		 TYPE_HEADER -> {
                val headerView: View = inflater.inflate(R.layout.item_header, parent, false)
                HeaderViewHolder(itemView)
            }
            TYPE_ITEM -> {
                val itemView: View = inflater.inflate(R.layout.item_view, parent, false)
                ItemViewHolder(itemView)
            }
           
            else -> null
        }
    }


Methode 2



Etwas anders, aber auch über den Adapter. Beginnend mit recyclerview: 1.2.0-alpha02 wurde der MergeAdapter angezeigt, mit dem Sie mehrere Adapter zu einem kombinieren können, um den Code sauberer zu gestalten. 



val mergeAdapter = MergeAdapter(headerAdapter, itemAdapter,  footerAdapter)
recyclerView.adapter = mergeAdapter


Lösung: Fügen Sie die Möglichkeit hinzu, einfach Kopf- und Fußzeilen über XML hinzuzufügen



Das erste, was Sie tun müssen, ist, das Hinzufügen von Ansichten in unserem OmegaRecyclerView abzufangen, wenn der Aufblasvorgang ausgeführt wird. Überschreiben Sie dazu die addView-Methode und fügen Sie sich die gesamte Kopf- und Fußzeilenansicht hinzu. Diese Methode wird von RecyclerView selbst verwendet, um sichtbare Listenelemente hinzuzufügen. Über XML hinzugefügte Ansichten haben jedoch keinen ViewHolder, wodurch letztendlich eine NullPointerException ausgelöst wird.



Wir müssen also bestimmen, wann die Ansicht während des Aufblasens hinzugefügt wird. Glücklicherweise gibt es eine geschützte onFinishInflate-Methode, die aufgerufen wird, wenn der Aufblasvorgang endet. Wenn wir diese Methode aufrufen, markieren wir daher, dass der Aufblasvorgang abgeschlossen ist. 



  protected override fun onFinishInflate() {
        super.onFinishInflate()
        finishedInflate = true
    }


Die addView-Methode sieht also folgendermaßen aus: 



  override fun addView(view: View, index: Int, params: ViewGroup.LayoutParams) {
        if (finishedInflate) {
            super.addView(view, index, params)
        } else {
            // save header and footer views
        }
    }


Als Nächstes müssen Sie sich alle diese zusätzlichen Ansichten merken und sie als MergeAdapter auf einen speziellen Adapter übertragen.



Es ist uns auch gelungen, ein anderes Problem zu lösen: Wenn die findViewById-Methode aufgerufen wird, werden unsere Ansichten nicht zurückgegeben. Um dieses Problem zu lösen, definieren wir die findViewTraversal-Methode neu: Es ist erforderlich, die ID der gefundenen Ansicht zu vergleichen und die Ansicht zurückzugeben, wenn sie übereinstimmt. Da diese Methode ausgeblendet ist, schreiben wir sie einfach, ohne anzuzeigen, dass sie überschrieben wird.



Sie können diese und andere nützliche Funktionen anhand einer detaillierten Beschreibung in unserer OmegaRecyclerView- Bibliothek kennenlernen :



  1. 2018 haben wir unseren ViewPager aus RecyclerView erstellt. Darüber hinaus gibt es in unserem ViewPager eine endlose Schriftrolle.
  2. ExpandableRecyclerView - eine spezielle Klasse zum Hinzufügen einer Dropdown-Liste mit der Möglichkeit, eine Erweiterungsanimation auszuwählen.
  3. StickyHeader ist ein bestimmtes Listenelement, das über einen Adapter hinzugefügt werden kann.


All dies ist das Ergebnis der Omega-R-Erfahrung. Die Entwicklung der Entwicklerfähigkeiten durchläuft mehrere Phasen. Zunächst möchten Sie den Code aus einem anderen Projekt kopieren oder etwas Ähnliches tun. Dann kommt die Phase, in der es erforderlich ist, die gesammelten Erfahrungen aufzuzeichnen und ein separates Repository zu erstellen.



In der nächsten Phase beginnen Sie mit der gezielten Erstellung von Funktionen, die in Projekten nur zögerlich ausgeführt wurden. Dies kann einige Zeit in Anspruch nehmen, ermöglicht es Ihnen jedoch, eine Reserve für die Zukunft zu schaffen und die Entwicklung neuer Projekte zu beschleunigen. Ich lade alle ein, die auf Entwicklungsschwierigkeiten stoßen, sich mit unseren Lösungen im Omega-R GitHub-Repository vertraut zu machen .
