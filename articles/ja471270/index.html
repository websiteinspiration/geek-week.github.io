Google PageSpeed Insightsからのサイトの読み込み速度に関するデータの受信を自動化する[テンプレート]

数十または数百のURLのダウンロード速度を定期的に監視しますか？業界のサイトのダウンロード速度を分析しますか、それともサイトのページごとに分析しますか？PageSpeed Insightsに毎回URLを入力してデータをExcelに転送するのは長い時間です。


シンプルなソリューションが見つかりました。これにより、PageSpeed Insightsから一定のスケジュールで、フルマシンで指標を取得できます。セットアップには5〜10分かかりますが、時間を節約できます。



eコマースでのサイトの読み込み速度の調査を準備しているときに、PageSpeed Insightsからデータを取得する自動化に非常に戸惑いました。サービス内のすべてのURLを開き、分析を実行し、メトリックを転送します。これには多くの時間がかかります。


MOZ のJames_McNultyの記事で、解決策を見つけました。この記事では、PageSpeed Insights API V5を使用して5つの指標でデスクトップデータを取得するGoogleスプレッドシートテンプレートを紹介します。テンプレートを変更しました。これにより、8つのメトリックでデータを取得できるようになりました（デスクトップだけでなく、モバイルについても）。
