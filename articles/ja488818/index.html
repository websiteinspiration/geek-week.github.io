#DeepPavlovは結果を目指しています


最近、対話システムは人間と機械の相互作用の標準になりました。チャットボットはほとんどすべての業界で使用されており、人とコンピュータ間のやり取りを簡素化しています。 Webサイト、メッセージングプラットフォーム、デバイスにシームレスに統合されます。今日、多くの企業は、日常的なタスクを複数のユーザー要求を同時に処理できるインタラクティブシステムに委任することを好んでおり、人件費を節約しています。

ただし、多くの場合、企業は、ビジネスのニーズを満たすボットを開発するときにどこから始めればよいかわかりません。歴史的に、チャットボットは2つの大きなグループに分けることができます。ルールとデータに基づいています。最初のタイプは、事前定義されたコマンドとテンプレートに依存しています。これらの各コマンドは、チャットボット開発者が正規表現とテキストデータ分析を使用して作成する必要があります。対照的に、データ駆動型チャットボットは、対話データで事前トレーニングされた機械学習モデルに依存しています。

オープンソースライブラリ-DeepPavlovインタラクティブシステムを構築するための無料で使いやすいソリューションを提供します。 DeepPavlovには、自然言語処理（NLP）に関連する問題を解決するための事前トレーニング済みのコンポーネントがいくつか付属しています。DeepPavlovは、テキストの分類、タイプミスの修正、名前付きエンティティの認識、知識ベースに関する質問への回答など、さまざまな問題を解決します。また、次のコマンドを実行すると、DeepPavlovを1行でインストールできます。

pip install -q deeppavlov

*フレームワークを使用すると、モデルをトレーニングおよびテストしたり、ハイパーパラメーターをカスタマイズしたりできます。ライブラリは、LinuxおよびWindowsプラットフォームをサポートしています。ライブラリのデモ版で、このモデルや他のモデルを試すことができます。

現在、BERTに基づくモデルを使用することにより、多くのタスクにおける最新の結果が得られています。DeepPavlovチームは、BERTを次の3つのタスクに統合しました：テキスト分類、名前付きエンティティの認識、質問への回答。その結果、これらすべてのタスクが大幅に改善されました。

1. BERT DeepPavlovモデル


テキスト分類のための

BERT BERT DeepPavlovに基づくテキスト分類モデルは、例えば、侮辱を検出する問題を解決するのに役立ちます。モデルには、パブリックディスカッションで公開されたコメントが参加者の1人にとって不快であると見なされるかどうかを予測することが含まれます。この場合、分類は、侮辱と非侮辱の2つのクラスでのみ実行されます。

コマンドラインインターフェース（CLI）とPythonの両方で、事前トレーニング済みのモデルを出力に使用できます。モデルを使用する前に、次のコマンドを使用して、必要なすべてのパッケージがインストールされていることを確認してください。

python -m deeppavlov install insults_kaggle_bert
python -m deeppavlov interact insults_kaggle_bert -d

名前付きエンティティ認識のためのBERT

テキスト分類モデルに加えて、DeepPavlovには名前付きエンティティ認識（NER）のためのBERTベースのモデルが含まれています。これは、NLPで最も一般的なタスクの1つであり、ライブラリで最も使用されているモデルです。同時に、NERには多くのビジネスアプリケーションがあります。たとえば、モデルは履歴書から重要な情報を抽出して、人材スペシャリストの作業を容易にすることができます。さらに、NERは、製品の仕様、会社名、会社の支店情報など、顧客の要求に関連するエンティティを識別するために使用できます。

DeepPavlovチームは、PER（人）、LOC（場所）、ORG（組織）など、19種類のマークアップを持つ英語のOntoNotesパッケージでNERモデルをトレーニングしました。対話するには、次のコマンドでインストールする必要があります。

python -m deeppavlov install ner_ontonotes_bert_mult
python -m deeppavlov interact ner_ontonotes_bert_mult [-d]

質問に回答するためのBERT質問への

コンテキスト回答は、特定のコンテキスト（Wikipediaの段落など）で質問への回答を見つけるタスクであり、各質問への回答はコンテキストセグメントです。たとえば、以下のコンテキスト、質問と回答のトリプルは、質問に答えるタスクの正しいトリプレットを形成します。


デモでの質問応答システムの作業のプレゼンテーション。

質問に対する回答のシステムは、ビジネスの多くのプロセスを自動化できます。たとえば、これは雇用主が社内のドキュメントに基づいて回答を得るのに役立ちます。さらに、このモデルは、学習プロセスでテキストを理解する学生の能力をテストするのに役立ちます。しかし最近では、文脈に基づいた質問に答える作業が科学者の大きな注目を集めています。この領域の重要なターニングポイントの1つは、スタンフォード質問回答セット（SQuAD）のリリースでした。SQuADデータセットは、質問と回答のシステム問題を解決するための無数のアプローチにつながりました。最も成功したものの1つは、DeepPavlov BERTモデルです。それは他のすべてを凌駕し、現在、人間の特性に接する結果を生み出しています。

DeepPavlovでBERTベースのQAモデルを使用するには、以下を行う必要があります。

python -m deeppavlov install squad_bert
python -m deeppavlov interact squad_bert -d

より多くのモデルがドキュメントにあります。また、ライブラリコンポーネントの使用に関するチュートリアルが必要な場合は、公式ブログで探してください。

2. DeepPavlov Agent-マルチタスクチャットボットを作成するためのプラットフォーム


今日、インタラクティブエージェントの開発にはいくつかのアプローチがあります。会話エージェントを開発する場合、モジュラーアーキテクチャは主に、スクリプトが展開される集中的な対話に使用されます。ただし、多くの場合、ユーザーは、たとえば、質問に答えたり、情報を検索したり、会話を維持したりする別の機能と、集中した対話を組み合わせる必要があります。したがって、理想的な対話エージェントは、さまざまなタイプのエージェントを組み合わせ、使用するタスクに応じて機能とキャラクターを切り替えるパーソナルアシスタントです。同時に、エージェントはその本質に関する情報を蓄積し、アルゴリズムを特定のユーザーに合わせて調整する必要があります。一方、外部サービスと統合できる必要があります。例えば、外部データベースへのクエリを作成し、そこから情報を取得して処理し、重要なものを強調表示してユーザーに送信します。この問題を解決するために、2019年10月に、マルチタスクチャットボットを作成するためのプラットフォームであるDeepPavlov Agent 1.0の最初のリリースがリリースされました。このエージェントは、本番チャットボットの開発者が複数のNLPモデルを1つのパイプラインに編成するのに役立ちます。

3. DeepPavlov NLP SaaSの実装


2019年9月に、DeepPavlovの事前トレーニング済みのNLPモデルでの作業を簡素化するために、SaaSサービスが開始されました。DeepPavlov Cloudを使用すると、テキストを分析したり、ドキュメントをクラウドに保存したりできます。モデルを使用するには、サービスに登録し、個人アカウントのトークンセクションでトークンを取得する必要があります。現在、このサービスはロシア語で事前にトレーニングされたいくつかのNLPモデルをサポートしており、システムのテストを行っています。

4. DSCT8または対象を絞った対話システムへの参加


Amazon AlexaやGoogleアシスタントなどの仮想アシスタントを使用することで、タクシーの注文、レストランでのテーブルの予約など、多くの日常的なタスクの実装を簡素化できるアプリケーションを開発する機会が開かれました。このような問題を解決するために、集中的な対話システムが使用されます。

対話状態追跡（DST）は、このような対話システムの重要なコンポーネントです。 DSTは、人間の言語での発話を言語の意味論的表現に変換し、特に、ユーザーの目標に対応するインテントとスロット値のペアを抽出します。DSTC8

へのチーム参加中GOLOMBモデル（GOaL指向のマルチタスクBERTベースの対話状態トラッカー）が開発されました-対話の状態を追跡するためのBERTに基づく目標指向のマルチタスクモデル。対話の状態を予測するために、モデルはいくつかの分類問題と部分文字列を見つけるタスクを解決します。まもなく、このモデルはDeepPavlovライブラリに表示されます。それまでは、こちらの記事全体をご覧ください。

6. Powered by TFチャレンジへの参加


前述のように、DeepPavlovには、TensorFlowとKerasを利用した事前トレーニング済みのコンポーネントがいくつか付属しています。そして今年、DeepPavlovチームは、TensorFlowライブラリを使用する最高の機械学習プロジェクトのGoogle Powered by TFチャレンジコンテストで優勝しました。600人以上のコンテスト参加者のうち、Googleは5つの最高のプロジェクトを選択しました。そのうちの1つはDeepPavlovライブラリです。プロジェクトはTensorFlowの公式ブログで紹介されました。TensorFlowの柔軟性により、考えられるあらゆるニューラルネットワークアーキテクチャを作成できることは注目に値します。特に、BERTベースのモデルとのシームレスな統合にはTensorFlowを使用します。



7.コミュニティ開発


私たちのプロジェクトのグローバルな目標は、会話型人工知能の分野の開発者と研究者が次世代のインタラクティブシステムを作成するための最先端のツールを使用できるようにすることと、経験の交換と最先端のテクノロジーの教育のためのAIの分野で国際的に重要なプラットフォームになることです。
これを達成するために、DeepPavlovの従業員はコンピュータサイエンスに携わる学生とスタッフを対象とした無料の学期トレーニングコースを実施する。その1つがコースです。「自然言語処理のディープラーニング」」で、セミナーやワークショップが含まれています。クラスには、対話システムの構築、応答を生成する機能を備えた対話システムを評価する方法、対話システムのさまざまなフレームワーク、対話ポリシーの最適化による報酬額の見積もり方法、ユーザー要求のタイプ、コールセンター呼び出しのモデル化の検討などのトピックが含まれます。 2020年には、新規採用を開始し、すでに900人の学生と従業員が無料でトレーニングを受けています。あなたは私たちのウェブサイトでこのコースと他のコースのニュースとセットをフォローすることができます。そして、コースを欠場したが、もっと学びたい場合は、YouTubeチャンネルでは、いつでもレコードでそれらを見つけることができます。

今日、DeepPavlovライブラリは、世界92か国で使用されている、テキストを操作するためのAI対応コンポーネントを提供しています。2020年2月の時点で、ライブラリのダウンロード数は10万件に達し、インストールのダイナミクスが勢いを増しています。また、ロシアの30社以上の企業がすでに実装しており、DeepPavlovに基づくソリューションをうまく使用しています。これは、そのようなソリューションが世界中で非常に人気があることを示しています。
