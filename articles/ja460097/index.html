マトリックスはあなたを持っています：MITER ATT＆CKを使用するプロジェクトの概要



すでに長い間、MITER ATT＆CKのマトリックス（敵対的戦術、テクニックや常識）は、様々な場所で議論されている記事の全シリーズを含め、ここで Habré自体に。コミュニティがこのモデルを採用したと言うことができ、さらに、多くの人がそれを使い始めました。今日、私たちはあなたのためにさまざまなプロジェクトの概要をまとめました：

  • マトリックスを操作することの認識と利便性を向上させる。
  • 分析をATTおよびCKと一致させる。
  • マトリックスに記載されているさまざまな攻撃をチェックする。

私たちは沿岸で合意します：監視されているユーティリティは完全にテストされていません。詳細についてはソースを参照することをお勧めします。

前書き


最初のATT＆CKモデルは2013年9月に作成され、主にWindowsに焦点を当てていました。それ以来、サイバーセキュリティコミュニティの貢献により、ATTとCKは大きく進化しました。これに加えて、攻撃の準備とモバイルデバイス用のATT＆CKを説明する追加のPRE-ATT＆CKナレッジベースが作成されました。

2019年7月の時点で、Enterprise ATT＆CKには、Windows、Linux、Mac用の314の攻撃方法が含まれています。マトリックス構造は11の戦術で構成されています。最初のアクセスから、C＆Cによる制御とデータの引き出しまでです。攻撃ライフサイクルの各フェーズは、組織のネットワークを危険にさらすためにサイバー犯罪者のさまざまなグループによって成功裏に使用されてきた多くの手法で構成されています。セキュリティをテストするとき、RedTeamは基本的に同じことを行うので、そのようなナレッジベースを使用しないことは大きな省略です。

この記事では、ATT＆CKマトリックス自体については説明しません。詳細はすべてWebサイトで確認できます開発者。このナレッジベースの使用と適用性は、大規模なベンダーではなく頻繁に使用されていることで示されています。ほとんどすべての脅威検出およびハンティングソリューションには、すでにこのマトリックスとのイベントの相関関係があります。

すべてのプラットフォームマトリックスの基本的なナビゲーションと注釈を提供するオープンソースのWebアプリケーション。そのシンプルさにもかかわらず、アプリケーションはマトリックスでの作業を大幅に簡素化し、メインマトリックスの上にレイヤーを作成することができます。特定の層は、特定のグループ化のテクニックを示したり、その逆の場合、保護コーティングを視覚化したりします。Navigatorは、RedTeamまたはBlueTeamの作業を計画するのに役立ちます。基本的に、このアプリケーションでは、マトリックス内のセルを操作するだけです。色分け、コメントの追加、数値の割り当てなどです。

RedTeamの場合


カルデラ


→ プロジェクトページ

CALDERAは、MITRE ATT＆CKプラットフォーム上に構築された、攻撃者のアクションをエミュレートする自動システムです。その主な目的は、エンドポイントセキュリティソリューションのテストとネットワークセキュリティステータスの評価です。ガートナーの用語によれば、このシステムは侵害および攻撃シミュレーション（BAS）製品に起因する可能性があります。 CALDERAは、ATTおよびCKモデルを使用して、あたかも実際の侵入が行われているかのように、敵の動作を識別して複製します。これにより、日常的な作業が回避され、複雑な問題を解決するための時間とリソースが増えます。

最近のシステムアップデートにより構造が変更されました。以前は、敵をエミュレートするサーバー、エージェント、実行可能ファイルで構成されていた場合、プラグインアーキテクチャが使用されます。新しい機能と動作を基本システムに接続します。CALDERAには、Stockpileプラグインを使用して事前に設計されたいくつかの敵対パターンが付属していますが、独自のパターンを追加するのも簡単です。

ATT＆CK-ツール




→  リポジトリプロジェクトページ

には以下が含まれます。

  • ATT＆CK View：敵のエミュレーション計画ツール。
  • ATT＆CKデータモデル：リレーショナルデータモデル。

Viewは、防御側がATT＆CKフレームワークに基づいて敵対的なエミュレーション計画を開発できるように設計されています。良い例として、MITREが開発したAPT3の完全な敵エミュレーション計画があります。これにより、プロジェクトをより早く開始できます。
データモデルの主な目的は、ATTとCKの新しいプロジェクトへの統合を簡素化することです。データベースはSQLiteに基づいており、シンプルさと機動性に優れています。データベースへのクエリの例は、プロジェクトページにあります。

Red Team Automation（RTA）




→  RTA プロジェクトページ

は、ATTおよびCKマトリックス方式に従って悪意のあるアクティビティを実行しようとする38のスクリプトおよびサポート実行可能ファイルのコレクションです。現在、RTAは50の戦術をカバーしています。可能な場合、RTAはスクリプトで記述された悪意のあるアクティビティを実行しようとしますが、それ以外の場合はそれをエミュレートします。

EDRテストスクリプト


→ プロジェクトページ

このリポジトリには、Miter ATT＆CK / LOLBAS / Invoke-CradleCrafterプラットフォームに基づくEDRソリューションをテストするための簡単なスクリプトが含まれています。実際には、EDRによってどれだけ多くの悪意のある攻撃が正しく識別および防止されているかを確認することは困難です。この目的のために、このスクリプトを作成して実行し、EDRコンソールに送信されるメッセージを監視します。ほとんどのテストは単にcalc.exeを実行しますが、簡単に変更できます（たとえば、Mimikatzをダウンロードして実行してみてください）。このスクリプトはWindowsでのみ機能し、ほとんどのEDRソリューションで機能するはずです。

このプロジェクトはまだ始まったばかりです。

Splunk用アプリ


Splunkはログの保存および分析システムです。これには、Webインターフェイスと、パネル（ダッシュボード）を作成する機能があります。これは、独自のSplunkアプリケーションです。

脅威狩り





→  ThreatHunting プロジェクト

ページは、ATTおよびCKマトリックスに従って脅威を監視するように設計されたSplunk用のアプリケーションです。アプリケーションはSysmonデータに基づいています。これは無料の強力なホストレベルのトレースツールで、デバイスドライバーとサービスを使用して、バックグラウンドで実行され、起動プロセスの非常に早い段階で読み込まれます。このサービスでは、登録するものを構成することもできます。 ThreatHuntingアプリケーションを開くと、過去24時間の各ATT＆CKカテゴリのすべてのトリガーの数が記載された概要ページに移動します。また、検出数が最も多く、最も脆弱なホストの技術も表示されます。このアプリケーションを使用すると、ATTおよびCKに関連するイベントを追跡し、データに基づいてイベントツリーを構築し、レポートをコンパイルできます。詳しくはお問い合わせください著者のブログ。

VECTR





→ githubのプロジェクトページ
→  プロジェクトページ

これは、MITRE ATT＆CKマトリックスのデータに従って、RedTeamおよびBlueTeamのテストアクティビティを簡単に追跡して、さまざまなシナリオでの攻撃の検出および防止の可能性を測定できる一元化されたダッシュボードです。以下の機能が含まれています。

  • リアルタイムでテストを追跡。
  • 実行されたテストの進捗状況を測定します。
  • RedTeamメソッドとBlueTeam機能の集中化。
  • カスタムテストスクリプトの追加。
  • 詳細なテストレポートを作成します。

VECTRは、RedTeamのタスクとツール、BlueTeam検出の第1レベルと第2レベル、検出の成功基準とテスト結果を文書化しています。得られた結果に基づいて、検出および応答機能をさらに改善するために使用できるツールボックスの一般的なインジケーターおよび特定の構成に関する推奨事項が提供されます。

ATT-CK_Analysis




→ プロジェクトページ

MITER ATT＆CKからのデータ分析を含む科学分析リポジトリ。独立したアナリストは、次のような多くの質問への回答を求めています。

  • メソッドの大部分を使用するグループ間に、以前は未知の関係がありましたか？
  • 各グループが使用する方法の数は、これらのグループの機能がどの程度進んでいるかを示す妥当な指標ですか？
  • データセットから直接、または外部ソースから機能の階層を確立できる場合、特定のグループが特定の方法を回避している（使用していないだけ）という証拠はありますか？

狩猟エルク（HELK）




→  プロジェクトページ

新しいハンティングELKプロジェクト（Elasticsearch、Logstash、Kibana）。これは、脅威を検出するエージェントの機能、Elastic ELKスタックの機能を拡張するという主な目的と連携して機能する、いくつかのオープンソースプラットフォームのエコシステムです。分析検索機能は、SparkおよびGraphframesテクノロジーの実装によって提供されます。これは、ELKスタックのデータ処理関数を無料で使用できるようにする最初の公的に利用可能なアセンブリの1つです。さらに、Jupyter Notebookがプロジェクトに統合され、ビッグデータや機械学習を使用してプロトタイプが作成されます。このスタックは、視覚化、グラフィカルなリレーショナルクエリ、高度な分析を組み合わせた全文検索エンジンを提供します。プロジェクトは開発段階にあり、コードと機能は変更されます。近い将来、ATTおよびCKからのデータを使用してダッシュボードを追加します。プロジェクトの詳細については、著者のブログをご覧ください。

結論


MITER ATT＆CKマトリックスを積極的に使用するプロジェクトの数は増え続けています。これは、情報セキュリティの脅威のモデルを一新した、アナリストにとっての優れた知識ベースであることに注意してください。ただし、一部の研究者は、マトリックスのいくつかの欠点を指摘しています。たとえば、テクニックの非常にあいまいな説明の場合があり、それによりそれらの作業が非常に複雑になります。

あなたが忘れてはならない主なこと：このマトリックスは、成功した攻撃に基づいて構築されています。それら。概して、これは、使用された手法と方法に関する歴史的な参照です。間違いなく、これは優れた知識ベースであり、操作には便利ですが、それでも敵の可能なテクニックすべてを完全に説明することはできません。
