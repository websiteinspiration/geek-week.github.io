ディレクトリのサイズは私たちの努力の価値はありません

これは完全に役に立たず、実際の使用には不要ですが、* nixシステムのディレクトリに関する楽しい小さな投稿です。金曜日も同じです。

インタビューでは、inodeに関する退屈な質問は、多くの場合、all-is-filesをすり抜けて、正直に答えられる人はほとんどいません。しかし、もう少し深く掘り下げると、興味深いものを見つけることができます。

投稿を理解するためにいくつかのポイント：

  • すべてがファイルです。ディレクトリもファイルです
  • ina inodeメタデータはファイルから保存されますが、ファイル名はそこに保存されません
  • ファイル名はディレクトリデータに保存されます
  • ディレクトリのサイズ。lsで表示されるものと同じで、デフォルトは4KBです。ディレクトリ内のファイルの数と名前の長さによって異なります。
  • 明らかに、より多くのファイル、より大きなディレクトリサイズ

そして興味深い部分は、100万のファイルを含むディレクトリを作成し、ディレクトリのサイズを確認してから、すべてのファイルをワイプしてディレクトリのサイズを確認することです。

$ mkdir niceDir && cd niceDir
#     ,     2-10 
$ for ((i=1;i<133700;i++)); do touch long_long_looong_man_sakeru_$i ; done
$ ls -lhd .
drwxr-xr-x 2 user user 8.1M Aug 2 13:37 .
$ find . -type f -delete
$ ls -l
total 0
$ ls -lhd .
drwxr-xr-x 2 user user 8.1M Aug  2 13:37 .

ご覧のように、ディレクトリのサイズは変更されていませんが、

アンマウントされた状態でfsck（および-Dオプション）を使用するだけで、ディレクトリのサイズを（削除せずに）修正できます。

しかし、なぜそうなのかを探しに行ったところ、10年前にこの動作はすでにlkmlで議論されていることがわかりました。そして開発者によると、修正は単に努力する価値はありません。
