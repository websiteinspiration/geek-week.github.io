Visual Studio for MacでのソリューションレベルのNuGetパッケージ管理の紹介

Visual Studio 2019 for Macバージョン8.3には、このブログ投稿に要約されている多くの新機能が含まれ  ています。ソリューションレベルでパッケージを管理する機能は、Visual Studio for Macのほとんどに欠けていた機能の1つでした。新しいソリューションレベルのNuGetパッケージマネージャーは、Visual Studio 2019 for Macバージョン8.3の優れた新機能の1つです。

パッケージを簡単に見つけられるように改善しました。これには、新しいパッケージを検索するときの改善された機能、プロジェクトに既にインストールされているパッケージの理解、および利用可能な更新を含むパッケージの検索が含まれます。この投稿では、ソリューションのパッケージ管理機能に焦点を当てます。ただし、インストール済みパッケージのインストール、更新、表示を含むほとんどの機能には、プロジェクトレベルで同様の新機能があります。



ソリューションのNuGetパッケージマネージャーを起動するには、ソリューションのコンテキストメニューに移動して、[NuGetパッケージの管理...]を選択します。


 

新しいパッケージの追加


新しいパッケージを検索して追加しようとすると、パッケージをインストールするプロジェクトを選択できます。

いつでも、「インストール済み」タブに移動して、ソリューションにインストールされているすべてのパッケージのリストを表示できます。これにより、パッケージを削除して更新することができます。


 

パッケージの更新


[ 更新 ]タブには、更新が利用可能なソリューション（またはプロジェクトレベルでコマンドを呼び出した場合はプロジェクト）のすべてのパッケージが表示されます。



パッケージの統合


多くの場合、さまざまなプロジェクトがさまざまなバージョンのパッケージを参照している状況では、大きな決定が見られます。これらのバージョンを単一のパッケージバージョンに結合するには、ソリューションで使用する場合があります。ソリューションノードで呼び出されるNuGetパッケージマネージャーの統合タブに移動し、すべてのソリューションプロジェクトで使用するパッケージバージョンを選択して、パッケージ統合を選択できます。 （パッケージを統合）：
