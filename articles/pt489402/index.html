Visualização do Terminal do Windows v0.9

O lançamento da versão 0.9 do Windows Terminal ocorreu. Esta é a versão mais recente do Terminal, que incluirá novos recursos antes do lançamento da v1. Você pode baixar o Terminal do Windows na Microsoft Store ou na página de lançamento no GitHub . Vamos dar uma olhada nos detalhes da atualização!



Argumentos da linha de comando


O alias wt agora suporta argumentos de linha de comando. Agora você pode iniciar o Terminal com novas guias e painéis, divididos como desejar, com perfis que você gosta, começando com os diretórios que você gosta! As possibilidades são infinitas! Aqui estão alguns exemplos:

wt -d.
Abre um terminal com um perfil padrão no diretório de trabalho atual.

em peso; new-tab -d C: \ pwsh.exe
Abre um terminal com duas guias. No primeiro - o perfil padrão, iniciando no diretório de trabalho atual. O segundo é o perfil padrão com pwsh.exe como "linha de comando" (em vez do perfil padrão "linha de comando") iniciando no diretório C: \.

wt -p "Windows PowerShell" -d.; painel de divisão -V wsl.exe
Abre um terminal com dois painéis separados verticalmente. O perfil com o nome “Windows Terminal” é iniciado no painel superior e o perfil padrão é usado no painel inferior, usando wsl.exe como a “linha de comando” (em vez do perfil padrão “linha de comando”).

w -d C: \ Usuários \ canela \ GitHub \ WindowsTerminal; painel dividido -p “Prompt de Comando”; painel de divisão -p "Ubuntu" -d \\ wsl $ \ Ubuntu \ home \ cinnak -H

Descoberta Automática do PowerShell


Se você é um grande fã do PowerShell Core , temos ótimas notícias para você. O Terminal do Windows agora detecta qualquer versão do PowerShell e cria automaticamente um novo perfil para você. A versão do PowerShell, que, em nossa opinião, é a melhor (a partir do maior número de versão, até a versão GA, até a versão com o melhor pacote), será chamada de "PowerShell" e ocupará o slot original do PowerShell Core na lista suspensa.



Confirme o fechamento de todas as guias


Você é quem não quer que seja solicitado a cada vez que feche todas as guias? Se você respondeu sim, esse novo recurso é definitivamente para você! Um novo parâmetro global foi criado que permite ocultar sempre a caixa de diálogo de confirmação "Fechar todas as guias". Para fazer isso, defina o parâmetro "confirmCloseAllTabs" como false na parte superior do arquivo profiles.json e você nunca mais verá esse pop-up! Obrigado @ rstat1 pela contribuição deste novo parâmetro.
