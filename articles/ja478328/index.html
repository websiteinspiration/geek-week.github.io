データサイエンティストが車を購入した方法

オランダで長年生活し、働いた後、家族と私はアメリカに戻り、自転車を車と交換する時が来ました。 アメリカでは車なしで生活するのは非常に難しく、引っ越し前に車を売っていたので、今では家族用の車を買わなければなりませんでした。データ処理と分析の優れたスペシャリストが近づくので、私はこの問題の解決に取り組むことにしました。データを使うことにしました。





データ収集


多くの場合、データ調査プロジェクトの最も難しい部分は、データ収集フェーズです。特に、ある種のホームプロジェクトに関しては。 CSVファイルとしてフォーマットされた、処理の準備ができているものをインターネットからダウンロードすると、失敗する可能性が非常に高いです。そしてデータがなければ、モデルも予測も構築できません。

自動車市場のデータを取得するには？ Webスクレイピングは、この問題の解決に役立ちます。


Google Chromeデベロッパーツールの

ウェブスクレイピングは、ウェブサイトからデータを収集するための自動化されたプロセスです。私はWebスクレイピングの専門家ではありません。この記事は、サイトからデータを収集するためのガイドではありません。しかし、少し練習していくつかのPythonモジュールの機能を詳しく調べれば、データ収集に関するクレイジーな空想を満たすことができます。

このプロジェクトでは、ユーザーインターフェイスのないブラウザーであるSelenium Pythonパッケージを使用しました。このようなブラウザーは、ユーザーがページを操作するのと同じ方法でページを開いて操作します（単にHTMLコードを読み取るBeautiful Soup Pythonライブラリのようなものとは異なります）。

Google Chromeデベロッパーのツールのおかげで、データ収集の準備プロセスが大幅に簡略化されました。それは、私たちが関心のあるWebページ要素をマウスの右ボタンでクリックして、それらをコピーするかxpath、element_idその他のことです。これらすべてはセレンによってよく知られています。しかし、必要な要素とテキストを見つけるために、少し手を加える必要がある場合があります。

何百台もの車の情報を収集するために使用したコードは、ここにあります。

データクレンジング


Webスクレイピングによって収集されたデータの問題の1つは、このデータがかなり乱雑である可能性が非常に高いことです。ここでのポイントは、それらは特定のリポジトリから取得されたのではなく、人々に情報を表示するために作成されたWebサイトから収集されたということです。その結果、そのようなデータは深刻なクリーニングを必要とします。多くの場合、これは必要なデータを取得するために文字列を解析する必要があることを意味します。

Webスクレイピングの結果をCSVファイルに保存した後、このデータをPandasデータフレームに読み込んでクリーニングを行うことができました。


クリーニング前のデータ

正規表現のようなものを使用して、収集したデータから必要な情報を抽出し始め、モデルを構築するための基礎となる機能を作成できます。

まず、Pandasメソッドに渡すことができる再利用に適した正規表現関数を作成しました.apply。

def get_regex_item(name, pattern):
    item = re.search(pattern, name)
    if item is None:
        return None
    else:
        return item.group()

列nameには多くの有用なデータがあります。これに基づいて、私が作成した関数を使用して、4つの新しいフィーチャーを作成できました。

df['4wd'] = df.name.apply(lambda x: 1 if get_regex_item(x, pattern= r'4[a-zA-Z]{2}') else 0)
df['year'] = df.name.apply(lambda x: get_regex_item(x, pattern= r'\d{4}'))
df['type'] = df.name.apply(lambda x: x.split()[-1])
df['certified'] = df.name.apply(lambda x: 1 if get_regex_item(x.lower(), r'certified') else 0)

df['price']  = df.price.apply(lambda x: x[:6].replace(",", "")).astype('int')
df['mileage'] = df.mileage.apply(lambda x: x.split()[0].replace(",", "")).astype('int')

独立した文字エンコーディング


モデルを構築する前に実行する必要があったもう1つのステップは、独立した機能をコーディングすることでした。サイン4wdとcertifiedすでに論理的な形で提示されています。追加の処理は必要ありません。しかし、兆候typeとyearエンコードする必要があります。この問題を解決するために.get_dummies、1つのアクティブな状態でエンコードできるPandasメソッドを使用しました。

Pandasはかなりインテリジェントなシステムで、すべての属性を渡して独立変数のみをエンコードでき、作成したばかりの列の1つを削除して、相互に完全に関連する属性を取り除くこともできます。このようなデータを使用すると、モデルとその解釈にエラーが発生します。コラム_2015が削除されたため、最終的にはマシンの製造年に関する情報を含む2つのコラムしかなかったことに注意してください。列についても同様typeです。

これで、私のデータは、モデルを構築するための基礎となる形式に変換されました。

モデル作成


上の図の分析と、私が自動車について知っていることに基づいて、私は私の場合線形モデルがそうすることに決めました。

私はOLS線形回帰モデル（通常の最小二乗法、通常の最小二乗法）を使用しました。モデルを構築するために、statsmodel Pythonパッケージが使用されました。

import statsmodels.api as sm
y = model_df.price

シミュレーションの結果、標準誤差は1556.09になりました。

得られた係数は、探索的データ分析中に図で確認できるものに対応しています。マイレージは価格に悪影響を及ぼします。平均して、1,000マイル走行するごとに1,600ドルの値下げになります。これは、GMCユーコンの車に当てはまります。おそらく、最も衝撃的な結果は、SLE（ローエンドのユーコンカー）とデナリ（ハイエンドのユーコンカー）の価格差を示したものでしょう。違いは9000ドル以上です。これは、高級車の購入者が受け取るすべてのプレミアムオプションに対して多額のお金を支払わなければならないことを意味します。その結果、彼が本当にこれらすべてを必要とするなら、それは素晴らしいことです。

どんな車を買う価値がありますか？


そこで、データを収集、クリーンアップ、解析、調査、視覚化し、それに基づいてモデルを構築しました。しかし、結局のところ、何を買うべきかという最初の質問には答えがありませんでした。特定のオファーを受け入れる価値があることを確認するにはどうすればよいですか？

この質問に対する答えを見つける方法は非常に簡単です。モデルを使用して予測を作成し、予測価格と自動車ディーラーが提供する価格の差を見つける必要があります。次に、最も過小評価されているオファーが一番上になるように、価格差のリストをソートする必要があります。これにより、最も有利なオファーのリストが表示されます。

いくつかの追加の制限（私の予算と妻の好み）を考慮した後、オファーのソートされたリストに移動し、販売者に電話をかけることができました。




購入した車は青色でハイライトされているため、2015年のGMCユーコンデナリは32,000ドルで購入しました。モデルが発表した予測よりも3000ドル安かった。

市場にはより有利なオファーがありましたが、購入した車は私たちからそれほど遠くないため、購入プロセスが大幅に簡素化され、加速されました。当時、私たちは車を持っていなかったので、この問題を解決するための速度と利便性は私たちにとって重要でした。


私たちの購入は図で青色で強調表示され

、最終的に、以前にコンパイルされた図の1つを見て、購入した車に関する情報を表示することができました。これは、私たちが確かに良い選択肢を見つけたことを示しています。

私の妻は本当に彼女の新しい車が好きですが、私はこの車の価格が好きです。以上を総合すると、車の購入に成功したと言えます。
