スプライトとPictureBox


ご存知のように、PictureBoxには2つの画像レイヤーがあります。BackgroundImage＆Image。エンジンの最初のバージョンでは、約5つのボックスを使用してスプライトを描画しました。このようなシステムには、いくつかの大きな欠点があります。

  • マルチレベルの継承による透明性の問題
  • 更新時にフォームをレイアウトする

その後、グラフィックスを介してアルゴリズムを作成し、いつでもどこでもスプライトを描画できるようにしました。 

PictureBox ALeft; 
Bitmap SpriteListPic; //      

// FreeMovePicture - ,   
// posX, posY -   
// Scale -  
private void SpriteBoxesHolder(Image FreeMovePicture, int posX, int posY, float Scale = 2)
{
       using(Graphics SpGr = Graphics.FromImage(SpriteListPic))
       {
              SpGr.DrawImage(FreeMovePicture, posX * 2, posY * 2, FreeMovePicture.Size.Width / Scale, FreeMovePicture.Size.Height / Scale);
       }
       ALeft.Image = SpriteListPic;
}

LuaInterfaceでの経験について少し：

  • テーブルを操作するときのメモリリークの問題
    string TableReaderS(string Table, string Key)
{
    string Ret = "";
    using (LuaTable tabx = lua.GetTable(Table))
    {
        Ret = (string)tabx[Key];
    }
    return Ret;
}

int TableReaderI(string Table, string Key)
{
    int Ret = -1;
    using (LuaTable tabx = lua.GetTable(Table))
    {
        Ret = (int)(double)tabx[Key]; //   int ...
    }
    return Ret;
}

    GetTable を呼び出すたびに、このグローバルLua変数によって参照されるLuaテーブルを参照する新しいCLRオブジェクトを取得します。
  • テーブルの値がまったくわからない場合：

    try
{
    Size = TableReaderI("Scene", "Image" + Convert.ToString(num) + "Scale");
}
catch (Exception ex)
{
    Size = 2;
}

    Luaは値を返しますが、数値であるという事実は返しません。したがって、キャッチ（例外）。double.TryParseのオプションは、ここでは適していません。lua.GetTableは文字列を返しませんが、特定のタイプ（LuaTable）を返します。このような値が割り当てられている場合は文字列に変換することもできます。
