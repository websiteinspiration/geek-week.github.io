Matériel


Donc, pour créer un prototype fonctionnant au minimum, nous avons besoin de:

  1. Un morceau de planche ou quoi que ce soit auquel vous pouvez attacher tous les composants
  2. Presque n'importe quel microcontrôleur. J'ai pris Arduino Uno comme l'option la plus simple
  3. Module GPS (nous en prenons la date, l'heure et les coordonnées). Théoriquement, vous pouvez prendre le module d'horloge en temps réel à la place, mais vous devrez alors entrer vos coordonnées manuellement et l'appareil ne fonctionnera pas hors de la boîte, mais l'heure du démarrage à froid sera considérablement réduite
  4. Deux servos, ou mieux encore deux moteurs pas à pas
  5. Pointeur laser
  6. Une bagatelle différente: un fer à souder, un adhésif thermofusible, une planche à pain, des fils, un bouton, un condensateur et des mains directes.

Il n'y a pas d'astuces à assembler: il suffit de fixer tous les composants sur la base avec un adhésif thermofusible.
Essayez simplement de vous assurer que les axes des servos et du laser sont aussi perpendiculaires que possible, cela réduira l'erreur de pointage.

Connexion


Rien de compliqué ici non plus:

  • Le GPS est connecté via uart (seul Rx est nécessaire, car nous n'avons pas besoin d'envoyer quoi que ce soit au module
  • servos - dans les broches 10 (axe d'azimut) et 11 (axe de hauteur)
  • bouton - 2 broches
  • alimentation de tous les modules
  • en option - condensateur de puissance

void GpsGetData()
{
  while (Serial.available() > 0)
    gps.encode(Serial.read());
  if (gps.location.isUpdated())
  {
    phi = gps.location.lat(); //   (double)
    lambda = gps.location.lng(); //    (double)
    yr = gps.date.year(); //  (2000+) (u16)
    mo = gps.date.month(); //  (1-12) (u8)
    d = gps.date.day(); //  (1-31) (u8)
    h = gps.time.hour(); //  (0-23) (u8)
    m = gps.time.minute(); //  (0-59) (u8)
    s = gps.time.second(); //  (0-59) (u8)
  }
}
void CalculateParams()
{
  float _time = (h + (float)m / 60 + (float)s / 3600) * 360.0 / 24.0;//   
  float sunlambda = (float)Days(d, mo, yr) * 360 / 365; //      
  sundelta = asin(0.398749 * sin(sunlambda * 2 * PI / 360));//    
  sunalpha = asin(tan(sundelta) / 0.434812); //      
  float tS = _time + lambda - 180 + sunalpha * 360 / (2 * PI) - alpha; //     
  height = 360 / 2 / PI * asin(sin(phi * 2 * PI / 360) * sin(delta * 2 * PI / 360) + cos(phi * 2 * PI / 360) * cos(delta * 2 * PI / 360) * cos(tS * 2 * PI / 360));// 
  az = 360 / 2 / PI * asin(sin(tS * 2 * PI / 360) * cos(delta * 2 * PI / 360) / cos(height * 2 * PI / 360)); //   (   )
  if (tS > 90 || tS < -90)
  {
    az = 180 - az;
  }
 // if (az > 180 && az < 270)

}

void setup() {
  Serial.begin(9600);
  yaw.attach(10, 90);
  pitch.attach(11, 180);
  yaw.setSpeed(20);
  yaw.setAccel(0.1);
  pitch.setSpeed(20);
  pitch.setAccel(0.1);
  yaw.setAutoDetach(false);
  pitch.setAutoDetach(false);
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  but.tick();
  yaw.tick();
  pitch.tick();
  if (but.isSingle())
  {
    counter += 1;
    if (counter == 9)
      counter = 0;
  }
  alpha = Coordinates[counter][0];
  delta = Coordinates[counter][1];
  GpsGetData();
  CalculateParams();
  digitalWrite(LED_BUILTIN, gps.location.isValid());
  if (az > 90 && az < 180)
  {
    yaw.setTargetDeg(270 - az);
    pitch.setTargetDeg(height);
  }
  if (az < -90 && az > -180)
  {
    yaw.setTargetDeg(-90 - az);
    pitch.setTargetDeg(height);
  }
  if (az < 90 && az > -90)
  {
    yaw.setTargetDeg(90.0 - az);
    pitch.setTargetDeg(180 - height);
  }
}

Leçon d'astronomie


À propos des astrotermines et d'autre chose
- , – .

, .

... et il n'y a qu'un seul instant - la fonction CalculateParams ()
Ce que cette fonction devrait faire: accepter les coordonnées de l'étoile dans le système équatorial (ascension droite et déclinaison), le temps et les coordonnées de l'observateur, et donner la hauteur et l'azimut de l'objet, c'est-à-dire traduire essentiellement les coordonnées de l'étoile du système équatorial (dans lequel les étoiles sont immobiles) en horizontal (dans lequel les étoiles se déplacent pendant la journée).

Ceci est réalisé en utilisant les formules de trigonométrie sphérique , ainsi que l'astronomie sphérique dans le vide .

L'algorithme est le suivant:

  1. calculer la longitude écliptique du soleil
  2. calculer la déclinaison du soleil
  3. calculer le lever de soleil direct
  4. calculer l'angle horaire d'une étoile
  5. calculer l'altitude et l'azimut

Voici comment il est implémenté:

void CalculateParams()
{
  float _time = (h + (float)m / 60 + (float)s / 3600) * 360.0 / 24.0;//   
  float sunlambda = (float)Days(d, mo, yr) * 360 / 365; //      
  sundelta = asin(0.398749 * sin(sunlambda * 2 * PI / 360));//    
  sunalpha = asin(tan(sundelta) / 0.434812); //      
  float tS = _time + lambda - 180 + sunalpha * 360 / (2 * PI) - alpha; //     
  height = 360 / 2 / PI * asin(sin(phi * 2 * PI / 360) * sin(delta * 2 * PI / 360) + cos(phi * 2 * PI / 360) * cos(delta * 2 * PI / 360) * cos(tS * 2 * PI / 360));// 
  az = 360 / 2 / PI * asin(sin(tS * 2 * PI / 360) * cos(delta * 2 * PI / 360) / cos(height * 2 * PI / 360)); //   (   )
  if (tS > 90 || tS < -90)
  {
    az = 180 - az;
  }
}

Estimation des erreurs


Évaluons la contribution de chaque facteur à l'erreur (triée par contribution à l'inexactitude)

  1. Courbure de la main - c'est le facteur le plus important pour moi
  2. Position d'horizon inexacte
  3. position inexacte de l'azimut zéro (je rappelle que les astronomes comptent l'azimut du sud dans le sens des aiguilles d'une montre)
  4. Le fait que les servos ne peuvent pas tourner à un angle fractionnaire
  5. inexactitudes dans l'algorithme de traduction (la position du soleil n'est pas déterminée très précisément: par exemple, la longitude écliptique du soleil est considérée comme beaucoup plus compliquée (une version simplifiée dans le code); le jour de l'équinoxe vernal ne se produit pas toujours en même temps;
    aussi, je ne prends pas en compte l'équation du temps) mais l'erreur de - pour cela, une petite fuite.
