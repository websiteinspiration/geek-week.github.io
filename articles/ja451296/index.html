ML.NET 1.0によって発表されました

最近、ML.NET 1.0のリリースを発表しました。ML.NETは、.NETアプリケーションの機械学習（ML）機能を活用するように設計された無料のクロスプラットフォームのオープン機械学習フレームワークです。



github.com/dotnet/machinelearning
開始：dot.net/ml

ML.NETを使用すると、感情分析、問題の分類、予測、推奨事項などのシナリオで、C＃またはF＃を使用してカスタム機械学習モデルをトレーニング、作成、配信できます。これらの一般的なシナリオとタスクは、ML.NETサンプルリポジトリで確認できます。

ML.NETは当初Microsoft Researchの一部として開発されましたが、その後、Windows Defender、Microsoft Office（PowerPointでの設計のアイデア、Excelの推奨図）、Azure Machine Learning、主要な要素の視覚化など、多くのMicrosoft製品で使用される重要なフレームワークに変わりましたPowerBIなどに影響を与えます！

ML.NETは発売以来、次のような多くの組織で使用されています。SigParser（メールスパム検出）、William Mullens（法的分類）、Evolution Software（ヘーゼルナッツ水分検出）。ML.NET Customer ShowcasでML.NETを使用すると、これらの組織や他の多くの組織のパスをたどることができます。提示されたクライアントは、ML.NETの使いやすさ、.NETスキルを再利用する機能、および.NETでのテクニカルスタックの完全な保持が、ML.NETを使用するための主要なドライバーであることを私たちに教えています。

ML.NET 1.0のリリースに伴い、自動機械学習（AutoML）などの新しいプレビュー機能と、ML.NET CLIやML.NETモデルビルダーなどの新しいツールも追加されます。つまり、機械学習モデルをこれで、右クリックするだけでアプリケーションが実行されます！

ML.NETの主要コンポーネント


ML.NETは、機械学習のさまざまな段階（前処理、設計機能、モデリング、評価、試運転）で.NETアプリケーションでMLを使用するための最終的なワークフローを提供することを目的としています。ML.NET 1.0は、次の主要コンポーネントを提供します。

  1. データの提示

    • マルチクラス分類

    • 異常検出

    • テキスト

    • 欠損値の正規化と処理

Automated Machine Learning (Preview)


今日の機械学習の開始には、急な学習曲線が含まれています。カスタム機械学習モデルを構築するときは、シナリオに選択する機械学習タスク（分類や回帰など）、MLアルゴリズムが理解できる形式にデータを変換する方法（テキストデータ->数値ベクトルなど）、およびこれらのMLアルゴリズムを調整して、最高のパフォーマンスを実現します。 MLを初めて使用する場合、これらの各ステップは非常に難しい場合があります。

自動機械学習は、入力データを変換する方法を自動的に把握し、適切な設定で最適なMLアルゴリズムを選択することにより、MLでの旅を簡素化し、クラス最高のカスタムMLモデルを簡単に作成できるようにします。

ML.NETでのAutoMLサポートはプレビューモードになり、現在サポートされているのは主要なMLタスクのみです-回帰（価格予測などのスクリプトに使用）と分類（感情分析、ドキュメント分類などのスクリプトに使用）スパム検出など）。

ML.NETでAutoMLを試すには、ML.NETモデルビルダー、ML.NET CLIを使用する、またはAutoML APIを直接使用するという3つのフォームファクターを使用できます（例はこちらにあります）。

機械学習の初心者には、Visual StudioのML.NETモデルビルダーと任意のプラットフォームのML.NET CLIから始めることをお勧めします。AutoML APIは、その場でモデルを作成するスクリプトにも非常に便利です。

モデルビルダー（プレビュー）


.NET開発者がMLモデルを作成する方法を簡略化するために、ML.NETモデルビルダーを導入することもできます。ML.NETモデルビルダーを使用すると、アプリケーションに機械学習を追加するだけで、右クリックするだけです！

モデルビルダーは、AutoMLを使用して、提供されたデータセットを使用してクラス最高のMLモデルを作成する開発者向けのシンプルなUIツールです。これに加えて、モデルビルダーは、最も効率的なモデルのモデルトレーニングコードとモデル消費コードも生成します。これにより、既存のアプリケーションにMLをすばやく追加できます。

ML.NET CLI（プレビュー）


ML.NET CLI（コマンドラインインターフェイス）は、導入するもう1つの新しいツールです。

ML.NET CLIは、AutoMLとML.NETを使用してML.NETモデルを生成できるドットネットツールです。 ML.NET CLIは、特定のMLタスク（現在は回帰と分類をサポートしています）のデータセットをすばやく繰り返し処理して、最適なモデルを作成します。

CLIは、最高のモデルを作成するだけでなく、モデルトレーニングとモデル消費コードを生成して、最も効率的なモデルを作成することもできます。

ML.NET CLIは、.NET CLIのクロスプラットフォームでシンプルなアドオンです。ちなみに、機会を提供するために、Visual Studioのモデルビルダー拡張もML.NET CLIを使用します。

次のコマンドでML.NET CLIをインストールできます。
dotnet tool install -g mlnet

今すぐ始めましょう！


まだ行っていない場合は、知っておいてください。ML.NETの使用は簡単で、以下に示すように、いくつかの簡単な手順で実行できます。以下のサンプルは、ML.NETを使用して感情分析を実行する方法を示しています。

// 1.  ML Context
var ctx = new MLContext();
 
// 2. Read  input   
IDataView dataReader = ctx.Data
    .LoadFromTextFile<MyInput>(dataPath, hasHeader: true);
 
// 3.   
IEstimator<ITransformer> est = ctx.Transforms.Text
    .FeaturizeText("Features", nameof(SentimentIssue.Text))
    .Append(ctx.BinaryClassification.Trainers
        .LbfgsLogisticRegression("Label", "Features"));
 
// 4.   
ITransformer trainedModel = est.Fit(dataReader);
 
// 5.  ,   
var predictionEngine = ctx.Model
    .CreatePredictionEngine<MyInput, MyOutput>(trainedModel);
 
var sampleStatement = new MyInput { Text = "This is a horrible movie" };
 
var prediction = predictionEngine.Predict(sampleStatement);
