JavaScriptアプリケーション間でモジュールを交換するためのプラグインであるWebpack 5のモジュールフェデレーション、説明、例

Webpackビルダーの5番目のバージョンでは、Javascriptアプリケーション間でモジュールを交換するための一連のプラグインが登場しました。

この記事は、ReactJSフレームワーク上に構築された2つの小さなアプリケーションに基づく短い説明と例です。

Module Federationプラグインを使用すると、アプリケーションは1つ以上のモジュールを個別のJSファイルにエクスポートできます。マイクロフロントエンドアプリケーションを構築する優れた方法。サードパーティのアプリケーションは、既製のモジュール（reactコンポーネントなど）をインポートできます。さらに、依存関係のインポートWebpackが引き継ぎます。NPMとの違いは、ランタイムにインポートされることです。



webpack構成のソースアプリケーションは、明示的に次のことを示します。

  • エクスポート用のモジュール
  • エクスポートファイル名（export.jsなど）
  • エクスポートする必要のない依存関係（export.jsなど）

受信側アプリケーションのHTMLでは、ソースアプリケーションのJSファイルが通常のJSスクリプトとしてインポートされます。 

<script src="http://source-app.com/export.js">

そして、webpack設定では、どのモジュールがファイルから取得されます。

こんな感じ


ソースアプリケーションは、webpack.config.js設定でエクスポートを登録する必要があります。
プラグイン設定で、エクスポートされるコンテナーの名前、コンテナーに含めるモジュール、およびファイルの呼び出し方法を指定します。

設定例 
const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
 
module.exports = {
 ...
 plugins: [
   new ModuleFederationPlugin({
     name: 'home',
     library: { type: 'var', name: 'home' },
     filename: 'export.js',
     exposes: {
       ProductCarousel: './src/ProductCarousel'
     },
     shared: ['react', 'react-dom', '@material-ui/core', '@material-ui/icons']
   }),
 ]
};

受信者アプリケーション

  • javascript.htmlでは、ソースアプリケーションのjsファイルをインポートします

    例 
    <!DOCTYPE html>
<html lang="en">
 <head>
   <script src="http://source-app.com/export.js"></script>

  • webpack.config。

    設定例 
    const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
 
module.exports = {
...
 plugins: [
   new ModuleFederationPlugin({
     remotes: {home: 'home' } // 
   }),
 ]
};

  • JSXで、通常のローカルモジュールであるかのように、目的のコンテナーからモジュールをインポートします。

    コード例 
    import React from 'react';
import ProductCarousel from 'home/ProductCarousel';
 
function App() {
 return (
     <ProductCarousel />
 );
}
 
export default App;
