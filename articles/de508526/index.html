Was ist das Endergebnis?



In JavaScript ist es möglich, den Operator zu verwenden, um den Rest der Division in Gleitkommazahlen zu erhalten. Es funktioniert so: 



3.14 % 5 // 3.14
13.14 % 5 // 3.1400000000000006


Gleichzeitig sind Zahlen vom Typ Number und werden gemäß dem IEEE 754-Standard in 64 Bit gespeichert. In .NET verfügt dieser Typ über ein Twin-Brother-System. Double oder nur Double. Wenn ein numerisches Literal einen Gleitkomma enthält, wird standardmäßig verdoppelt. Die Absicht kann explizit ausgedrückt werden, indem der Zahl das Suffix d oder D hinzugefügt wird. Die Möglichkeit, das Doppelte mit dem Rest zu teilen, ist ebenfalls verfügbar. Also versuche. Und ... Bingo! 



Console.WriteLine(3.14d % 5); // 3,14
Console.WriteLine(13.14d % 5); // 3,1400000000000006

Komm, unrein, denn in dir ist keine Wahrheit!



Im folgenden Beispiel ist die Bedingung wahr. 



const x = { i: 1, toString: function() { return this.i++; } };
if (x == 1 && x == 2 && x == 3)
    document.write("This will be printed!");


Dies wird dadurch erreicht, dass bei der Überprüfung der Gleichheit die Operanden in denselben Typ umgewandelt werden. In diesem Fall wird für x eine überschriebene toString () -Methode aufgerufen, die zusätzlich zur Rückgabe etwas den Status des Objekts ändert. Wir stellen uns fest, dass reine Funktionen wunderbar sind, aber aus Gründen des Interesses versuchen wir, das Konzept anhand des Beispiels umzusetzen. 



Trickster x = new Trickster();
if (x == 1 && x == 2 && x == 3)
    Console.WriteLine("This will be printed!");


Mit .NET können Sie Methoden und Casts nach Möglichkeit überschreiben, wenn die entsprechende Cast definiert ist. 



class Trickster
{
    private int value = 1;

    public override string ToString() =>
        value++.ToString();

    public static implicit operator int(Trickster trickster) =>
        int.Parse(trickster.ToString());
}


Hurra, es funktioniert. Aber es verwirrt, dass Trickster sich an int anzupassen scheint. Ersetzen 



public static implicit operator int(Trickster trickster) =>
    int.Parse(trickster.ToString());


auf der 



public static implicit operator double(Trickster trickster) =>
    double.Parse(trickster.ToString());


Es funktioniert trotzdem. Jetzt werden Trickster und int verdoppelt.



Tatsächlich ist die Neudefinition von ToString () für das Erreichen des Endergebnisses nicht unbedingt erforderlich und dient der maximalen Nachahmung eines Beispiels in JavaScript. Es reicht aus, eine Besetzung zu definieren, um sie zu verdoppeln. Natürlich nicht zu vergessen die Bedeutung der Nebenwirkung. 



class Trickster
{
    private int value = 1;

    public static implicit operator double(Trickster trickster) =>
        trickster.value++;
}


Und wenn Sie sich an die Überlastung der Operatoren erinnern, können Sie ein Objekt implementieren, das für alles gleich und ungleich ist 



Trickster x = new Trickster();
if (x == 1 && x == 2 && x == 3 && x == new[] { 1, 2, 3 } &&
    x != 1 && x != 2 && x != 3 && x != new[] { 1, 2, 3 })
    Console.WriteLine("This will be printed!");


auf so einfache Weise 



class Trickster
{
    public static bool operator ==(Trickster trickster, object o) =>
        true;

    public static bool operator !=(Trickster trickster, object o) =>
        true;

Eleganter Griff



Dieser JavaScript-Code wird dreimal ausgegeben. 



for (i = 0; i <= 2; ++i)
  setTimeout(() => console.log(i), 0);


Da es in C # keine sofort einsatzbereite setTimeout-Funktion gibt, implementieren wir das Analogon selbst und erhalten genau das gleiche Ergebnis. 



void SetTimeout(Action action, int delay) =>
    Task.Run(async () =>
    {
        await Task.Delay(delay);
        action();
    });

for (int i = 0; i <= 2; ++i)
    SetTimeout(() => Console.WriteLine(i), 0);

//        
Console.ReadKey();


Der Effekt der Variablenerfassung ist nur dann offensichtlich, wenn Sie wissen, wie diese Erfassung implementiert ist. Fragen zu diesem Thema sind übrigens in Interviews mit Partnern sehr beliebt. Nach meiner Erfahrung fragt etwa jeder Fünfte.



Niemand versprach Kommutativität



Operatoren in JavaScript sind manchmal nicht kommutativ. 



Date() && {property: 1}; // {property: 1}
{property: 1} && Date(); // Uncaught SyntaxError: Unexpected token '&&'


In C # sind Operatoren im Allgemeinen nicht standardmäßig auf benutzerdefinierte Typen anwendbar (Ausnahme == und! = Für Referenztypen). Einige von ihnen können jedoch explizit vom Typ überladen werden. Dann liegt die Sorge um die Kommutativität beim Entwickler. 



class Trickster
{
    //  C#  && 
    public static object operator +(Trickster trickster, object o) =>
        null;
}


Und es wird nicht unbedingt implementiert. 



var a = new Trickster() + new object(); // OK

// Compilation Error:
// Operator '+' cannot be applied to operands of type 'object' and 'Trickster'
var b = new object() + new Trickster();


Einen Igel mit einem Igel kreuzen



In JavaScript können Sie mathematische Operationen zusammen an Zeichenfolgen und Zahlen ausführen. 



var a = "41";
a += 1; // "411"
var b = "41";
b -=- 1; // 42


In C # ist dies nur mit Addition möglich. 



var a = "41" + 1; // 411

// Compilation Error:
// Operator '-' cannot be applied to operands of type 'string' and 'int'
var b = "41" - (-1);


Aber es gibt keine Hindernisse für die Patrioten der Sprache. Indem Sie ein wenig Zauberei hinzufügen, können Sie dafür sorgen, dass auch dieser Code „richtig“ funktioniert: 



Trickster a = "41";
Console.WriteLine(a += 1); // 411
Trickster b = "41";
Console.WriteLine(b -=- 1); // 42


Sie müssen nur ein Paar impliziter Typkonvertierungen implementieren und das Anweisungspaar überladen. 



class Trickster
{
    private string value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = s };

    public static implicit operator Trickster(int i) =>
        new Trickster { value = i.ToString() };

    public static string operator +(Trickster trickster, int i) =>
        trickster.value + i;

    public static int operator -(Trickster trickster, int i) =>
        int.Parse(trickster.value) - i;

    public override string ToString() =>
        value;
}


Sie können zusätzliche Überlastung ersetzen 



public static string operator +(Trickster trickster, int i) =>
    trickster.value + i;


implizit in einen String umwandeln 



public static implicit operator string(Trickster trickster) =>	
    trickster.value;


und erhalten das gleiche Ergebnis.



Wenn Sie sich nicht die Mühe machen, die Nummer aus der Subtraktionsoperation zurückzugeben und diese Nummer in Trickster zu platzieren, können Sie sie ersetzen 



public static int operator -(Trickster trickster, int i) =>	
    int.Parse(trickster.value) - i;


auf der 



public static string operator -(Trickster trickster, int i) =>
    (int.Parse(trickster.value) - i).ToString();


und der Int-Cast für Trickster wird entfernt.



Bananen jonglieren



In JavaScript kann die Zeichenfolge "Banane" folgendermaßen abgerufen werden: 



('b' + 'a' + + 'a' + 'a').toLowerCase(); // "banana"
('b'+'a'++'a'+'a').toLowerCase(); // Uncaught SyntaxError: Invalid left-hand side expression in postfix operation


Wenn Sie hier unary + auf das zweite 'a' anwenden, wird NaN zurückgegeben, das zur Addition mit den restlichen Strings in die Zeichenfolge 'NaN' umgewandelt wird. Das Ergebnis ist 'baNaNa', und der Schönheit halber wird alles auf Kleinbuchstaben reduziert.



In C # erstellen wir eine Klasse 



class Trickster
{
    private string value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = s };

    public static double operator +(Trickster trickster) =>
        double.TryParse(trickster.value, out double result) ? result : double.NaN;
}


und versuche 



//  double.NaN.ToString()  "NaN",   " "
Thread.CurrentThread.CurrentCulture = new CultureInfo("en-us");
Console.WriteLine(("b" + "a" + + (Trickster)"a" + "a").ToLower());


Leider kann der Compiler trotz des impliziten Castings die Zeichenfolge string -> Trickster -> double -> string nicht erstellen, und Sie müssen es ihm explizit mitteilen. (Wenn Sie darüber nachdenken, würde eine solche Besetzung mehr als seltsam aussehen.)



Trickster kann anders implementiert werden: 



class Trickster
{
    private double value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = double.TryParse(s, out var d) ? d : double.NaN };

    public static double operator +(Trickster trickster) =>
        +trickster.value;
}


In diesem Fall können Sie sogar die unäre + Überladung durch eine implizite Umwandlung in double ersetzen: 



public static implicit operator double(Trickster trickster) =>
    trickster.value;


Als Bonus erhalten wir, wie im JavaScript-Beispiel, einen Kompilierungsfehler, nicht nur einen, sondern eine ganze Reihe, wenn Sie kein Leerzeichen zwischen die beiden Pluspunkte setzen.
