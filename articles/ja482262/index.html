Talend Open Studioにログインする方法

Talend Open Studio for ESBは、アプリケーションとビジネスシステムを異種エンタープライズインフラストラクチャに統合するように設計されたビジュアルコンポーネントに基づいた無料でダウンロードできるツールです。


Open Studio for ESBは、条件付きで2つの部分に分かれています。


  • Studio-ソリューションが開発されたEclipseベースのIDE（Talend Job）;
  • ランタイムは、マイクロサービスのコンテナーであるApache Karafベースのビルドです。

Talend製品には、ロギング用の特別なコンポーネントtLogRowがありますが、log4jへのエクスポートは、サブスクリプションによって配布された製品でのみ意図的に利用できます。


スタジオログ


プロジェクトのデバッグ時にコンポーネントログを取得するには、構成ファイルで優先度を変更し、必要に応じて構成ファイルで必要なアペンダーを有効にする必要があります。ファイル→編集→プロジェクトプロパティの編集→Log4j


そのため、tRESTRequestコンポーネントのログを取得するには、コンポーネントで[詳細設定]→[ログメッセージ]をオンにし、優先度を「info」に変更します。


<!-- Root Logger --> 
<root>
    <priority value ="info"/>
    ...
</root>

タスクの実行中に、3種類のイベントを傍受できます。


  • tWarnコンポーネントイベント-ワークフロー内のイベント。
  • tDieコンポーネントイベント-ワークフローを終了するイベント。
  • Java例外イベント-ワークフローの内部例外。

イベントをインターセプトするために、インターセプトされたイベントのタイプの組み合わせを選択する機能を持つコンポーネントtLogCatcherがあります。

Karafでログを設定するには、公式ドキュメントに慣れることをお勧めします：http : //karaf.apache.org/manual/latest/#_log


必要に応じて、ローカルログを外部システムにエクスポートできます。たとえば、外部filebeatを使用したり、内部ツールKaraf Apache Decanterを使用したりできます。
