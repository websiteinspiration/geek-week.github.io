使用Python SDK简化Check Point API

当与程序代码结合使用时，当可以动态生成API请求和用于分析API响应的工具时，就会显示出与API交互的所有功能。但是，用于Check Point Management API的Python软件开发套件（以下称为Python SDK）却徒劳无功。它极大地简化了开发人员和自动化爱好者的生活。 Python最近获得了极大的欢迎，我决定缩小差距，并概述Check Point API Python开发工具包的主要功能。本文是有关HabréCheck Point R80.10 API的另一篇文章的出色补充。通过CLI，脚本等进行管理。我们将研究如何使用Python SDK编写脚本，并详细介绍1.6版（自R80.40开始受支持）中的新Management API功能。要理解本文，您将需要使用API​​和Python的基本知识。

Check Point正在积极开发API，目前有：

  • Check Point管理API（当前版本1.6） -通过API与管理服务器一起使用（以及在管理服务器的控制下在网关上执行脚本的功能）
  • Check Point GAIA API（当前版本1.4） -使用安全网关
  • 威胁防护API 1.0 -Check Point Cloud中的沙箱
  • 身份识别API-与网关上的身份识别刀片一起使用
  • 安全管理门户API-使用SMB网关管理门户（了解有关SMB网关的更多信息）
  • IoT API-与IoT控制器的交互
  • CloudGuard Connect API-与CloudGuard Connect一起使用（SD-WAN安全解决方案）
  • Dome9 API-使用Dome9

Python SDK当前仅支持与Management API和Gaia API的交互。在本模块中，我们将介绍最重要的类，方法和变量。



模块安装


cpapi 模块可以使用pip 从github上的官方Check Point存储库中快速轻松地安装。详细的安装说明位于README.md中。该模块适用于Python 2.7和3.7版本。在本文中，将使用Python 3.7给出示例。但是，Python SDK可以直接从Check Point（智能管理）管理服务器运行，但是它们仅支持Python 2.7，因此最后一部分将提供2.7版的代码。安装模块后，我建议您立即查看examples_python2和examples_python3目录中的示例。

开始工作


为了使我们能够使用cpapi模块的组件，必须从cpapi模块导入至少两个必需的类：

APIClient和APIClientArgs

from cpapi import APIClient, APIClientArgs

APIClientArgs 类负责连接到API服务器，而APIClient类负责与API进行交互。

我们确定连接参数


要定义用于连接到API的各种参数，您需要创建APIClientArgs类的实例。原则上，其参数是预定义的，并且在管理服务器上运行脚本时可以省略它们。

client_args = APIClientArgs()

但是，在第三方主机上启动时，您必须至少指定API服务器（它也是管理服务器）的IP地址或主机名。在下面的示例中，我们定义服务器连接参数，并将其分配给管理服务器的IP地址作为字符串。

client_args = APIClientArgs(server='192.168.47.241')

让我们看一下连接到API服务器时可以使用的所有参数及其默认值：

APIClientArgs类的__init__方法的参数 
class APIClientArgs:
    """
    This class provides arguments for APIClient configuration.
    All the arguments are configured with their default values.
    """

    # port is set to None by default, but it gets replaced with 443 if not specified
    # context possible values - web_api (default) or gaia_api
    def __init__(self, port=None, fingerprint=None, sid=None, server="127.0.0.1", http_debug_level=0,
                 api_calls=None, debug_file="", proxy_host=None, proxy_port=8080,
                 api_version=None, unsafe=False, unsafe_auto_accept=False, context="web_api"):
        self.port = port
        # management server fingerprint
        self.fingerprint = fingerprint
        # session-id.
        self.sid = sid
        # management server name or IP-address
        self.server = server
        # debug level
        self.http_debug_level = http_debug_level
        # an array with all the api calls (for debug purposes)
        self.api_calls = api_calls if api_calls else []
        # name of debug file. If left empty, debug data will not be saved to disk.
        self.debug_file = debug_file
        # HTTP proxy server address (without "http://")
        self.proxy_host = proxy_host
        # HTTP proxy port
        self.proxy_port = proxy_port
        # Management server's API version
        self.api_version = api_version
        # Indicates that the client should not check the server's certificate
        self.unsafe = unsafe
        # Indicates that the client should automatically accept and save the server's certificate
        self.unsafe_auto_accept = unsafe_auto_accept
        # The context of using the client - defaults to web_api
        self.context = context


我相信可以在APIClientArgs类实例中使用的参数对Check Point管理员来说是直观的，不需要其他注释。

通过APIClient和上下文管理器连接


通过上下文管理器最方便地使用APIClient 类。传递给APIClient类的实例所需要做的就是在上一步中定义的连接参数。

with APIClient(client_args) as client:

上下文管理器不会自动对API服务器进行登录调用，但是在退出时将进行注销调用。如果由于某种原因在使用API​​调用结束工作时不需要注销，则需要在不使用上下文管理器的情况下开始工作：

client = APIClient(clieng_args)


连接检查


检查连接是否通过给定参数的最简单方法是使用check_fingerprint方法。如果未通过服务器API证书的指纹的sha1哈希验证（方法返回False），则通常是由于连接问题引起的，我们可以停止程序（或允许用户修复连接数据）：

    if client.check_fingerprint() is False:
        print("Could not get the server's fingerprint - Check connectivity with the server.")
        exit(1)

请注意，将来，APIClient类将在API服务器上检查每个API调用（api_call和api_query方法，我们将进一步讨论它们）sha1指纹证书。但是，如果在检查服务器API证书的sha1指纹时检测到错误（证书未知或已更改），则check_fingerprint方法将提供以自动模式在本地计算机上添加/更改有关它的信息的功能。可以使用API​​ClientArgs参数-unsafe_auto_accept完全关闭此检查（但是仅当脚本在API服务器本身上运行（连接到127.0.0.1时）才建议这样做）。 （请参阅前面的“定义连接参数”中有关APIClientArgs的详细信息）。

client_args = APIClientArgs(unsafe_auto_accept=True)

登录到API服务器


在APIClient中，API服务器上最多有3种登录方法，并且每个zaponimaet值sid（会话ID）都会在标头（此设置中标题的名称-X-CHKP-sid）中的每个后续API调用中自动使用。您不需要进一步处理此参数。

登录方式


使用用户名和密码的变体（在示例中，用户名admin和密码1q2w3e作为位置参数传递）：

     login = client.login('admin', '1q2w3e')

可选的可选参数在登录方法中也可用，我列出了它们的名称和默认值： 

continue_last_session=False, domain=None, read_only=False, payload=None

Login_with_api_key方法


使用api密钥的变体（自管理版本R80.40 / Management API v1.6开始支持，“ 3TsbPJ8ZKjaJGvFyoFqHFA ==”是具有API密钥授权方法的管理服务器上用户之一的API密钥的值）：

     login = client.login_with_api_key('3TsbPJ8ZKjaJGvFyoFqHFA==')

在login_with_api_key方法中，可以使用与login方法相同的可选参数。

Login_as_root方法


使用API​​服务器登录到本地计算机的选项：

     login = client.login_as_root()

此方法只有两个可选参数：

domain=None, payload=None

处理API调用结果


之后，您可以使用APIResponse类的变量和方法（在上下文管理器内部和外部）。APIResponse类具有 4个方法和5个预定义的变量；我们将更详细地介绍最重要的方法。



成功


首先，最好确保API调用成功并返回结果。有一种成功的方法：

In [49]: api_versions.success                                                   
Out[49]: True

如果API调用成功（响应代码为200），则返回True；否则，则返回False（其他任何响应代码）。在API调用之后立即使用很方便，因此可以根据响应代码显示不同的信息。

if api_ver.success: 
    print(api_versions.data) 
else: 
    print(api_versions.err_message)

状态码


进行API调用后返回响应代码。

In [62]: api_versions.status_code                                               
Out[62]: 400

可能的响应代码：200,400,401,403,404,409,500,501。

set_success_status


可能需要更改成功状态的值。从技术上讲，您可以在其中放置任何内容，甚至是普通的字符串。但是一个真实的例子是在某些伴随条件下将此参数重置为False。下面，请注意在管理服务器上执行任务时的示例，但是我们将认为此请求是不成功的（尽管API调用成功并且返回了代码200，但将成功变量设置为False）。

for task in task_result.data["tasks"]:
    if task["status"] == "failed" or task["status"] == "partially succeeded":
        task_result.set_success_status(False)
        break

有用的例子


以下是使用已添加到Management API 1.6的API调用的示例。

首先，请考虑add-host和add-address-range调用的操作。假设我们需要将子网192.168.0.0/24的所有IP地址创建为主机类型的对象，该主机的最后一个八位位组为5，并将所有其他IP地址写为地址范围对象。在这种情况下，应该排除子网地址和广播地址。

因此，下面是解决此问题的脚本，该脚本创建了50个类型为host的对象和51个类型为地址范围的对象。要解决此问题，需要101个API调用（不计算最终的发布调用）。另外，使用timeit模块，我们可以计算执行脚本直到发布更改所花费的时间。

使用add-host和add-address-range的脚本 
import timeit
from cpapi import APIClient, APIClientArgs

start = timeit.default_timer()

first_ip = 1
last_ip = 4

client_args = APIClientArgs(server="192.168.47.240")

with APIClient(client_args) as client: 
     login = client.login_with_api_key('3TsbPJ8ZKjaJGvFyoFqHFA==')
     for ip in range(5,255,5):
         add_host = client.api_call("add-host", {"name" : f"h_192.168.0.{ip}", "ip-address": f'192.168.0.{ip}'})
     while last_ip < 255:
         add_range = client.api_call("add-address-range", {"name": f"r_192.168.0.{first_ip}-{last_ip}", "ip-address-first": f"192.168.0.{first_ip}", "ip-address-last": f"192.168.0.{last_ip}"})
         first_ip+=5
         last_ip+=5
     stop = timeit.default_timer() 
     publish = client.api_call("publish")
     
print(f'Time to execute batch request: {stop - start} seconds')


在我的实验室环境中，根据管理服务器上的负载，执行此脚本需要30到50秒。

现在，让我们看看如何使用add-objects-batch调用API解决相同的问题，该支持已添加到API版本1.6中。通过此调用，您可以在一个API请求中创建多个对象。此外，这些对象可以是不同类型的对象（例如，主机，子网和地址范围）。因此，我们的任务可以在单个API调用的框架中解决。

使用add-objects-batch的脚本 
import timeit
from cpapi import APIClient, APIClientArgs

start = timeit.default_timer()

client_args = APIClientArgs(server="192.168.47.240")

objects_list_ip = []
objects_list_range = []

for ip in range(5,255,5):
    data = {"name": f'h_192.168.0.{ip}', "ip-address": f'192.168.0.{ip}'}
    objects_list_ip.append(data)
    
first_ip = 1
last_ip = 4


while last_ip < 255:
    data = {"name": f"r_192.168.0.{first_ip}-{last_ip}", "ip-address-first": f"192.168.0.{first_ip}", "ip-address-last": f"192.168.0.{last_ip}"}
    objects_list_range.append(data)
    first_ip+=5
    last_ip+=5

data_for_batch = {
  "objects" : [ {
    "type" : "host",
    "list" : objects_list_ip
}, {
    "type" : "address-range",
    "list" : objects_list_range
  }]
}


with APIClient(client_args) as client: 
     login = client.login_with_api_key('3TsbPJ8ZKjaJGvFyoFqHFA==')
     add_objects_batch = client.api_call("add-objects-batch", data_for_batch)
     stop = timeit.default_timer() 
     publish = client.api_call("publish")
     
print(f'Time to execute batch request: {stop - start} seconds')


根据管理服务器上的负载，在我的实验室环境中执行此脚本需要3到7秒钟。也就是说，平均而言，在101 API对象上，批处理类型调用的执行速度快10倍。在更多对象上，差异将更加显着。

现在，让我们看看如何使用set-objects-batch。使用此API调用，我们可以大规模更改任何参数。让我们设置上一个示例中地址的前半部分（最多0.124台主机，并且还有范围），以彩色西耶那色，然后将地址的后半部分分配为卡其色。
