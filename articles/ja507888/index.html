オブジェクト検出用のニューラルネットワークを調整およびトレーニングするためのデータ準備

オブジェクト検出用のニューラルネットワークを調整およびトレーニングするためのデータ準備。オブジェクト検出



のトピックを続けます。アルゴリズムがオブジェクトを区別したり、プロセスの逸脱を検出したりできるようにするには、分析されたビデオ素材の全体像に対応するデータセットでアルゴリズムをトレーニングする必要があります。トレーニングセットの準備は非常に時間のかかるプロセスであり、トレーニングの速度とアルゴリズムの品質の両方に依存します。データを正しく準備するのに役立ついくつかのツールの使用を検討してください。



準備の最初の段階は、初期データセットのコンパイルです。原則として、データ量が多く、分析対象の多様性が高いほど、アルゴリズムの品質は高くなります。 DVRからビデオを分析する場合は、既存のビデオ素材からスクリーンショットを撮るために、データセットをコンパイルすることをお勧めします。この場合、必要なデータが少なくなり、再トレーニングに必要な時間が短くなります。他の場合では、データセットは、オブジェクトがキャプチャされる視野角、そのサイズなどを考慮してコンパイルされます。トレーニングセットのオブジェクトが、アルゴリズムによって分析されるマテリアルのオブジェクトとどれだけ類似しているかが考慮されます。データセットは、さまざまなバリエーションで構成する必要があります。たとえば、1つのフレームに人と電話など、1つのオブジェクトと複数のオブジェクトの両方が表示されます。



データセットの編集がカメラまたは電話を使用して実行された場合、おそらく画像の解像度が高くなり、学習プロセスに悪影響を及ぼします。高解像度画像の主な問題は、batch_sizeパラメータが小さい場合でも、「メモリ不足」エラーです。このエラーの可能性を減らすには、画像のサイズを小さくする必要があります。 IrfanViewプログラムはこれに役立ちます。このプログラムを使用すると、画像をバッチ処理できます。ソース画像へのパスを指定し、変換パラメーターを設定して、変換プロセスを開始するだけです。その結果、指定したフォルダーに「ライト」画像が取得されます。これを行うには、プログラムの開始ウィンドウでBキー（ロシア語のレイアウト "I"）を押すか、[ファイル]タブで[バッチ変換/名前の変更]を選択します。次のようなウィンドウが表示されます。







次に必要なもの：



  1. バッチ変換モードを選択します。
  2. ソース画像があるフォルダを選択します。
  3. 詳細な詳細設定を構成します。
  4. すべての画像を追加します。
  5. 変換結果のフォルダーを選択します。
  6. バッチ変換を開始します。


詳細オプションではさまざまな変換を利用できます。この例では、元の画像のサイズを変更し、サイズを10倍に縮小します（元の値の10％を指定）。







したがって、元の画像が4160 x 3120の場合、結果の画像は416 x 312になります。ピクセル。



トレーニングサンプルの作成は、ビデオ素材からのスクリーンショットの作成であろうと、カメラと電話での撮影であろうと、かなり退屈なプロセスです。データセットのボリュームを大幅に増やすには、さまざまな拡張方法が使用されます。 Pillowライブラリを使用してPythonプログラミング言語で実装されたこれらのメソッドのいくつかを見てみましょう。最も一般的な拡張方法の1つは「ミラー反射」です。



まず、必要なライブラリをインポートします。 



from PIL import Image, ImageOps
import os
import numpy


ソース画像のあるディレクトリの場所と変換結果のパスを示します。 



path_to_origin = "images/" 
output_folder = "Augmentation_result/"
img_names = os.listdir(path_to_origin)

if not os.path.exists(output_folder):
    os.makedirs(output_folder)


画像を「鏡像」に変換する関数を書きます。 



def mirror(path_to_origin, img_name, output_folder):
    im = Image.open(path_to_origin + img_name)
    ImageOps.mirror(im).save(output_folder+"mirror"+img_name, "JPEG")


もう1つの一般的な拡張方法は、画像の回転です。フルサイズのオブジェクトでは、この拡張方法では画像の一部が失われることを考慮する価値があります。画像を回転させる関数を書きます： 



def rotate(path_to_origin, img_name, output_folder):
    im = Image.open(path_to_origin + img_name)
    im.rotate(-30).save(output_folder+"rotate_"+img_name, "JPEG")


値（-30）は、オブジェクトのタイプに基づいて選択される画像の回転角度です。たとえば、180度回転は、電話が画像内にある場合、これは拡張に適したオプションですが、人物が画像内にある場合、この画像の180度は悪い例です。人々が逆さまに歩くビデオを見ることはほとんどありません。



そして最後に、各画像に2種類の拡張を使用します。 



for img in img_names:
    rotate(path_to_origin, img, output_folder)
    mirror(path_to_origin, img, output_folder)

さまざまな拡大方法（ノイズの追加、コントラストの変更など）があり、すべて処理されるデータのタイプによって異なります。したがって、2つの拡張方法を使用して、トレーニングデータのサイズを3倍に増やしました。トレーニングセットの形成が完了し、画像内のオブジェクトをマークするために残ります。



マークアップには、LabelImageプログラムを使用します。、私の意見ではそれは非常に使いやすく、多機能なので。このプログラムは、マークアップ結果を2つの形式（PascalVOCおよびYOLO）で返すことができます。最初の形式は、画像パラメーターが示されているXMLファイルと、画像内のオブジェクトの場所を示す長方形の座標として表示







されます。このようなXMLファイルは、すべてのマークされたオブジェクトが記述されている各画像に対して生成されます。マークアップファイル名はイメージ名と同じです。 YOLO形式は、オブジェクトのクラスと四角形の座標が記述されたテキストファイルの形式で提示され、正規化された形式で値x、y、w、hによって記述されます。







YOLO形式のマークアップを含むテキストファイルに加えて、classes.txtファイルも生成され、オブジェクトの名前が示されます。オブジェクトの名前のシーケンス番号は、イメージをマークするときの出現順序によって異なります。アプリケーションを再度開いたときに、classes.txtファイルが上書きされる可能性があることに注意してください。プログラムに名前を入力する順序を監視する必要があります。そうしないと、座標の不整合が発生します。オブジェクトのマークアップは次の順序で行われます。







  1. 画像のあるディレクトリを選択します。
  2. マークアップファイルを保存するディレクトリを選択します。
  3. レイアウト形式（PascalVOCまたはYOLO）を選択します。
  4. 「Wキー」オブジェクト選択ツールを選択します。
  5. 画像内のオブジェクトを選択します。
  6. オブジェクトの名前を設定します（操作を繰り返す間、[OK]ボタンの下のリストから選択できます）。
  7. 確認
  8. 「Ctrl + s」を保存します（必要なオブジェクトごとに手順4〜8を繰り返します）。
  9. 次の画像に移動します。


完了すると、マークアップファイルの数は、同じ名前の画像の数に対応します。そこで、画像のサイズ変更、データ量の増加、ニューラルネットワークによって検出される画像上のオブジェクトへのマーク付けを可能にするツールと方法を検討しました。データの形成と準備は、ディープラーニングアルゴリズムの作成が成功するための重要で時間のかかる手順です。
