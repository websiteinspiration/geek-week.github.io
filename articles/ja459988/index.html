彼はニューラルネットワークで犬を食べました

ほら、通りに犬を飼っている。あなたはいつも通りで犬を見ます。誰かがそれらにまったく気づいておらず、問題もありません。これらの精神的苦痛はありません。一般的に、彼らは路上で規範を持っていると想定できます。しかし、あなた！そして彼はそれに気づきました、そしてあなたの目はぬれた場所にいます、あなたはそう思います：ええ、とてもかわいい、私はそれを取るでしょうが、私はできません。しかし、それは直接的ですが、決してそうではありません。」そして、彼はここまで行きました。しかし、彼は写真を撮り、それを撫でました。


ここでは、現時点での動物の地理的位置といくつかの写真をシステムに投入する必要のある親切な小さな動物を紹介します。

逃亡した仲間や他の小さな犬を本当に見つけたい人がいます。私は本当にこれでペットを見つけたいです。彼ら自身が広告を配置し、門、地下室、さまざまなサイトを通過します。

2019年は、より多くの技術的ソリューションに貢献しています。そして、PetSI（PetSearchInstrument）という実用的なタイトルのプロジェクトはまさにそれです。

概念


オープンデータサイエンスコミュニティのソーシャルグッドのため の機械学習の一部として、9851754そして、私たちのチームは行方不明の動物を検索するサービスを作成します。所有者は、動物の写真、紛失の住所、その他の特性を指定でき、見返りに見つかった、または見られた動物について、アルゴリズムの観点から最も適切なメッセージを受け取ります。

私たちのサービスの簡単なアルゴリズム：複数のサイトからのデータ（写真、場所、品種など）を集約し、ニューラルネットワークで画像をベクトルに変換し、knnをトレーニングして、入力された写真の最近傍を表示します。あなたは失われた動物を見つけ、動物は家に帰ります。みんなが幸せだ）



行方不明のペットを検索することに加えて、新しい飼い主へのシェルター動物の追加を加速するための推奨システムを開発しています。サイトでのユーザーの行動に関する統計情報はありませんが、視覚的な類似性に基づくコンテンツベースの推奨事項を使用しています。

フードの下


Pythonで開発が進行中です。次のテクノロジースタックを使用します。

  • Docker、Gitlab CI /アプリケーションのデプロイ用CD。
  • Google Kubernetes Engine ;
  • Scrapy, RabbitMQ ;
  • Sklearn, keras ML;
  • Django, Flask, Bootstrap ;
  • Elasticsearch .

ワークフローのすべてのステージを同期するには、Airflow：

Multi-stage data collection を使用します。まず、スパイダーが情報を収集し、そのままの形式でキューに送信します。キューの反対側では、特別なプロセッサがデータを目的の形式に変換し（たとえば、テキストをアドレスに変換）、それをDBMSに追加します。



収集されたデータは検証され、トレーニングモデルに送信されます。新しいデータとモデルをダウンロードし、各ユーザーの検索結果の再集計を開始する特別なサービスがサイト用に作成されました。サイクル全体には約8時間かかります。

機械学習とデータ分析の使用方法：

  • テキストによる性決定;
  • テキストによる品種の定義;
  • メッセージからのアドレスの割り当て。
  • 画像のセグメンテーション;
  • 画像のベクトルとANNへの変換。
