Vuexでゲッターを操作するためのいくつかのシンプルだが便利なヒント

Vuexはデータを操作するための便利なツールを提供しますが、一部の開発者は必ずしもそれらを意図した目的に使用したり、より明確かつ簡潔に記述できる冗長な構造を作成したりしません。これは、開発者がこれらのツールに精通している場合にも発生します。この記事では、組織のゲッター（ゲッター）に関するいくつかの推奨事項を示します。


ゲッターは、より具体的には、Vuexリポジトリ、計算プロパティの一部です。たとえば、あるパラメータでフィルタリングされたデータを受け取ることができます。しかし、一部の開発者はこのツールの名前を文字通り使用し、状態から直接データを取得する代わりにこのツールを使い始めます。ここからは、ゲッターを使用する際の最初の間違いに続きます。


ゲッターを使用してストレージからデータを簡単に取得する


簡単なコード例を見てみましょう：


state: {
  films: [
    { id: 1, name: ' ' },
    { id: 2, name: ' ' },
    { id: 3, name: ' 60' },
  ],
},
getters: {
  films: state => state.films,
},

このゲッターの使用は十分に一般的で、これは悪いことです。コンポーネントの状態にアクセスするには、計算された値を計算します。次に例を示します。


computed: {
  films() {
    return this.$store.state.films;
  },
},

または、mapStateを使用したさらに便利なオプション：


computed: {
  ...mapState(['films']),
},

データを取得するこのような方法を採用すると、追加のコードでコードをオーバーロードすることはありません。
