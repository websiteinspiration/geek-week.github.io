AdonisJS 5 - Laravel-like framework on nodeJS and with Typescript worthy of your attention

AdonisJS 5 was released, at the moment the developers call it the preview version, although in their Discord they claim that in essence it is a production-ready version of the framework, and it is called preview because not all the necessary packages have been rewritten for version 5 yet, their modesty It‚Äôs worth envy, I‚Äôll say for myself that version 5 really deserves wide attention, in my opinion it is one of the best and simplest frameworks on which it is worth doing a backend in 2020+.
AdonisJS is a Laravel-like framework on NodeJS and now also on Typescript

A key feature of AdonisJS 5 is Typescript out of the box; otherwise, little has changed architecturally. And it‚Äôs good, because AdonisJS version 4 has long found a middle ground, in many ways by the way adopting it from the Laravel framework.

There are:

  • ORM
  • ready integration with popular databases
  • Database migrations
  • Models (much like TypeORM)
  • Controllers
  • View (optional)
  • Routing
  • Middleware
  • Custom CLI Commands
  • Analog artisan - ace

All the same simplicity and grace. The developers of the framework try to adhere to common sense regarding architecture and do not make overhead features. For example, AdonisJS routing is located in a special file like that of Laravel, but if you look at the competitive NodeJS framework - NestJS and see how routing is announced there, you can see a significant difference, and with NestJS it is somehow more beautiful, routes are declared inside the controller using decorators. And this is convenient enough for small projects, but when your application grows up, routing like NestJS becomes disastrously inconvenient, AdonisJS is stable in this sense, it is equally convenient to make both small and large projects in it.
