バナー付きの無限スクロール、または3つのビューを使用する方法


モバイルプラットフォームの各開発者は、1つの方法では解決できないタスクに常に直面しています。常にいくつかの方法があり、いくつかは迅速で、いくつかは複雑であり、それぞれに独自の長所と短所があります。

iOSのエンドレス/サイクリックスクロールは、標準のツールを使用して実装することはできません。さまざまなトリックに進む必要があります。この記事では、問題を解決するためのどのオプションが表面にあり、どのオプションが最終的に実装されるかを説明します。

仕事


用意された画像、タイトル、サブタイトルの形の要素で無限の循環スクロールを行う必要がありました。入力：中央の画像は画面から16.0ポイントでインデントされます。中央の画像の両側に「耳」が突き出ています。バナー間の距離は8.0ポイントです。


私たちは同僚によって何が行われたかを研究します



Dodo-バナーは循環的ではなく、中央のバナーは常に特定の距離で左端に隣接しています。

Auto.ru-バナーは周期的ではありません。スワイプすると、バナーが非常に長い間めくられます。

オゾン-バナーは周期的ですが、タッチでは制御できません。スワイプの方向が決定されると、画像を停止できなくなります。

ワイルドベリー-バナーは循環的ではなく、中央揃えが発生し、スクロールアニメーションが長く完了します。

最後の願い：

  • バナーは中央に配置する必要があります。
  • スクロールはアニメーションを長時間待つことなく終了する必要があります。
  • バナーの管理性：指でスクロールアニメーションを制御する機能が必要です。

実装オプション


まだ解決されていない新しいタスクが発生した場合、既存のソリューションとアプローチを検討する価値があります。

UICollectionView


額で行動すると、を作成するオプションが表示されますUICollectionView。要素の数を作成し、Int.max初期化中に中央を表示し、メソッドをdataSource-で呼び出すときに表示しfunc collectionView(UICollectionView, cellForItemAt: IndexPath) -> UICollectionViewCellます。ゼロ要素がthisであると想定して、対応する要素を返しInt.max / 2ます。のような多くの機能を備えたこのようなモンスターは、UICollectionView単純なタスクでの使用には不適切です。

UIScrollViewおよび（n + 2）UIView


UIScrollViewが作成されるオプションはまだあります。絶対にすべてのバナーがその上に配置され、バナーによって最初と最後に追加されます。最後まで完了すると、ユーザーが気付かないうちにオフセットを変更して、最初の要素に戻ります。そして、後ろにスクロールすると、その逆になります。その結果、多数の要素がある場合、それらを再利用せずに一連のビューが作成されます。


ソース

独自のやり方


UIScrollView + 3つのUIViewを作ることにしました。これらのUIViewは再利用されます。スクロール時にcontentOffset中央のバナーに戻り、3つすべてのUIViewのコンテンツを置き換えます。そして、このタスクを閉じるライトコンポーネントを取得する必要があります。

ただし、スクロール中にコンテンツのなりすましがユーザーに気付かれることが懸念されます。これについては、実装時に学習します。

実装


UIScrollViewと3つのUIImageViewの準備


相続人を作成し、そのUIView上にUIScrollView3つ配置しますUIImageView。

final class BannersView: UIView {
    private let scrollView = UIScrollView()

    private let leftItemView = UIImageView()
    private let centerItemView = UIImageView()
    private let rightItemView = UIImageView()

    init() {
        super.init(frame: .zero)
        self.setup()
    }

    required init?(coder aDecoder: NSCoder) {
        fatalError("init(coder:) has not been implemented")
    }

    private func setup() {
        self.addSubview(self.scrollView)
        self.setupScrollView()

        let imageViews = [self.leftItemView, self.centerItemView, self.rightItemView]
        imageViews.forEach(self.scrollView.addSubview)
    }
}

次の設定でメソッドの実装を追加しますscrollView。

  • decelerationRate-このパラメータは、スクロールアニメーションの速度を示します。私たちの場合、それが最善.fastです。
  • showsHorizontalScrollIndicator -このパラメーターは、水平スクロールバーを表示します。 

    private func setupScrollView() {
    self.scrollView.decelerationRate = .fast
    self.scrollView.showsHorizontalScrollIndicator = false
}

基本的なセットアップの後、レイアウトと配置を行うことができますImageView：

override func layoutSubviews() {
    super.layoutSubviews()
    self.scrollView.frame = self.bounds

    let horizontalItemOffsetFromSuperView: CGFloat = 16.0
    let spaceBetweenItems: CGFloat = 8.0
    let itemWidth = self.frame.width - horizontalItemOffsetFromSuperView * 2
    let itemHeight: CGFloat = self.scrollView.frame.height

    var startX: CGFloat = 0.0

    let imageViews = [self.leftItemView, self.centerItemView, self.rightItemView]
    imageViews.forEach { view in
        view.frame.origin = CGPoint(x: startX, y: 0.0)
        view.frame.size = CGSize(width: itemWidth, height: itemHeight)
        startX += itemWidth + spaceBetweenItems
    }

    let viewsCount: CGFloat = 3.0
    let contentWidth: CGFloat = itemWidth * viewsCount + spaceBetweenItems * (viewsCount - 1.0)
    self.scrollView.contentSize = CGSize(width: contentWidth, height: self.frame.height)
}

スクロール時に画像を中央に配置


スクロールを制御するには、を使用しますUIScrollViewDelegate。setupのデリゲートをメソッドUIScrollViewに設定しcontentInset、最初と最後の画像を両側にインデントを設定します。

self.scrollView.contentInset = UIEdgeInsets(top: 0.0, left: 16.0, bottom: 0.0, right: 16.0)
self.scrollView.delegate = self

メソッドの1つ を作成extensionしますBannersView。デリゲートメソッドfunc scrollViewWillEndDraggingは、ユーザーがスクロールを停止したときに呼び出されます。このメソッドではtargetContentOffset、最終スクロールオフセット（スクロールが停止するポイント）を担当する変数に関心があります。


extension BannersView: UIScrollViewDelegate {

  func scrollViewWillEndDragging(_ scrollView: UIScrollView, withVelocity velocity: CGPoint, targetContentOffset: UnsafeMutablePointer<CGPoint>) {
    let gap: CGFloat = self.centerItemView.frame.width / 3

    let targetRightOffsetX = targetContentOffset.pointee.x + self.frame.width
    if (self.rightItemView.frame.minX + gap) < targetRightOffsetX {
      targetContentOffset.pointee.x = self.rightItemView.frame.midX - self.frame.midX
    }
    else if (self.leftItemView.frame.maxX - gap) > targetContentOffset.pointee.x {
      targetContentOffset.pointee.x = self.leftItemView.frame.midX - self.frame.midX
    }
    else {
      targetContentOffset.pointee.x = self.centerItemView.frame.midX - self.frame.midX
    }
  }

}

gap-これは、ビューが中心であると想定する距離です。オレンジ色の画像の幅の3分の1が画面に表示される場合は、オレンジ色の画像が中央になるように最終オフセットを設定します。


targetRightOffsetX -この点は、正しいビューが中心であるかどうかを判断するのに役立ちます。

スクロール中にオフセットを管理する


ここで、スクロールの最中に、を変更しcontentOffset、画面の中央の中央ビューに戻ります。これにより、ユーザーは無限のスクロールに気付かれないように錯覚させることができます。

デリゲートメソッドを追加しfunc scrollViewDidScroll(_ scrollView: UIScrollView)ます。これは、contentOffsety が変更されたときに呼び出されますUIScrollView。


func scrollViewDidScroll(_ scrollView: UIScrollView) {
    guard
            self.leftItemView.frame.width > 0,
            self.centerItemView.frame.width > 0,
            self.rightItemView.frame.width > 0
            else {
        return
    }

    let gap: CGFloat = self.centerItemView.frame.width / 3
    let spacing: CGFloat = 8.0

    let currentRightOffset: CGFloat = scrollView.contentOffset.x + self.frame.width + scrollView.contentInset.left

    if (self.rightItemView.frame.maxX - gap) < currentRightOffset {
        scrollView.contentOffset.x -= self.centerItemView.frame.width + spacing
    } else if (self.leftItemView.frame.minX + gap) > scrollView.contentOffset.x {
        scrollView.contentOffset.x += self.centerItemView.frame.width + spacing
    }
}

gap-これは、変位の必要性を決定する距離contentOffsetです。我々はのためのポイントを計算するrightItemView：、self.rightItemView.frame.maxX — gap我々はシフトするの交差点の後contentOffset。たとえば、フルディスプレイrightItemViewにスクロールするために100.0ポイントが残っている場合contentOffsetは、バナー間の距離（間隔）を考慮して、1つのバナーの幅だけ後方にシフトし、centerItemView所定の位置に表示しrightItemViewます。同様に、次のことをleftItemView行います。ポイントを計算し、その交点の後に変更しcontentOffsetます。


func set(imageURLs: [URL])外部に表示するデータを公開 するメソッドを追加します。そこで、コードの一部をから転送しsetupます。

また、行を追加して、コンテンツを設定するとcenterItemViewすぐに中央に配置されるようにします。horizontalItemOffsetFromSuperViewで既に使用しているため、layoutSubviews定数に入れて再度使用します。

コンテンツを差し替え


タスクは、オフセットcontentOffsetでビューのコンテンツを同時に置き換えることです。右にスクロールすると、右の画像が中央になり、中央が左になり、左が右になります。1-2-3 | 2から3 - 1.

便宜上、ViewModelにを作成します。

現在どの要素が中央にあるかを確認するにはBannersView、各ビューのコンテンツに変数と変数を追加します。

    private var currentCenterItemIndex: Int = 0

    private var viewModel: BannersViewModel?

    private var leftItemViewModel: URL {
        guard let items = self.viewModel?.items else { fatalError("not ready") }
        let leftIndex = items.index(before: self.currentCenterItemIndex)
        return leftIndex < 0 ? items.last! : items[leftIndex]
    }
    private var centerItemViewModel: URL {
        guard let items = self.viewModel?.items else { fatalError("not ready") }
        return items[self.currentCenterItemIndex]
    }
    private var rightItemViewModel: URL {
        guard let items = self.viewModel?.items else { fatalError("not ready") }
        let rightIndex = items.index(after: self.currentCenterItemIndex)
        return rightIndex >= items.count ? items.first! : items[rightIndex]
    }

ビューのコンテンツを変更する必要がある場合に呼び出されるメソッドを追加します。

    func nextItem() {
        self.currentCenterItemIndex += 1
        if self.viewModel?.items.count == self.currentCenterItemIndex {
            self.currentCenterItemIndex = 0
        }
        self.updateViews()
    }

    func prevItem() {
        self.currentCenterItemIndex -= 1
        if self.currentCenterItemIndex == -1 {
            self.currentCenterItemIndex = self.viewModel?.items.indices.last ?? 0
        }
        self.updateViews()
    }

    private func updateViews() {
        self.leftItemView.setImage(with: self.leftItemViewModel)
        self.centerItemView.setImage(with: self.centerItemViewModel)
        self.rightItemView.setImage(with: self.rightItemViewModel)
    }

最後のステップ


UIView単純な絵の代わりに カスタムを作成することに変わりはありません。タイトル、サブタイトル、画像になります。

延長ViewModel：

struct BannersViewModel {
    let items: [Item]

    struct Item {
        let title: String
        let subtitle: String
        let imageUrl: URL
    }
}

バナー実装を記述します。

extension BannersView {
    final class ItemView: UIView {
        private let titleLabel = UILabel()
        private let subtitleLabel = UILabel()
        private let imageView = UIImageView()

        init() {
            super.init(frame: .zero)
            self.setup()
        }

        required init?(coder aDecoder: NSCoder) {
            fatalError("init(coder:) has not been implemented")
        }

        private func setup() {
            self.addSubview(self.imageView)
            self.addSubview(self.titleLabel)
            self.addSubview(self.subtitleLabel)

            self.imageView.contentMode = .scaleAspectFill

            self.layer.masksToBounds = true
            self.layer.cornerRadius = 8.0
        }

        func set(viewModel: BannersViewModel.Item) {
            self.titleLabel.text = viewModel.title
            self.subtitleLabel.text = viewModel.subtitle
            self.imageView.setImage(with: viewModel.imageUrl)
        }

        override func layoutSubviews() {
            super.layoutSubviews()
            self.imageView.frame = self.bounds

            self.titleLabel.frame.origin = CGPoint(x: 16.0, y: 16.0)
            self.titleLabel.frame.size = CGSize(width: self.bounds.width - 32.0, height: 20.0)

            self.subtitleLabel.frame.origin = CGPoint(x: 16.0, y: self.titleLabel.frame.maxY + 4.0)
            self.subtitleLabel.frame.size = self.titleLabel.frame.size
        }
    }
}
