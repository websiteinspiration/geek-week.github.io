电子医疗信息档案库将如何帮助更有效地诊断疾病



根据IDC的预测，到2025年，医疗保健组织中存储的数据总量将增加到2.3 ZB，各种医学图像将占已用存储容量的80-90％。通过以下示例，有效存储医学图像的重要性很重要。

在亚利桑那州图森市一家医院的诊断部门，使用MRI乳房X线照片（基于X射线，超声和MRI的乳腺癌诊断），为两只哺乳动物拍摄了多达40张X射线图像。以及两到五个活检。在80％的病例中，使用两年前或更早拍摄的老病人照片来解释乳房X线照片的结果，在困难情况下，可能需要十年前拍摄的照片。为了从档案中快速检索旧图像，使用了PACS（图像存档和通信系统）数字医院系统。

使用存储在PACS中的旧图像可大大降低恶性肿瘤诊断错误的风险，并使良性肿瘤患者无需再次进行乳房X线照片甚至活检即可，以确保他们的肿瘤不是恶性的。同时，与旧图像进行比较可减少对恶性肿瘤图像进行错误解释的风险，并使您可以快速为患者分配适当的病理治疗方法和其他分析。

长期保存医学图像档案的功能


用作PACS的存储系统的理想存储系统应该是什么？显然，鉴于医学图像的大尺寸，它必须具有很高的可扩展性，以便它可以存储几十年来积累的每个患者的分析结果。第二个要求是确保从档案中快速搜索和检索数据，否则就无法使用旧图像来解释新调查的结果。

最后，在存储医学图像时，必须完全消除“数据泄漏”，丢失以及意外或有意删除或损坏的可能性。在PACS中存储数据的功能是对I / O性能的要求相对适中：很明显，分析结果仅写入存档一次且永不更改，并且只有少数用户可以发送请求以从存档中提取此数据。通常要求患者数据不超过一年一次。



传统的企业级存储系统不适合用于PACS，主要是因为数据存储成本太高，这在很大程度上是由于服务于事务性应用程序时医学图像档案所不需要的高性能以及较便宜的入门级存储系统不具有PACS档案所需的可伸缩性。

也许是存储医学图像的最佳解决方案


在医疗保健组织中存储医学图像和其他非结构化内容的主要解决方案是使用文件和对象存储系统，这些系统具有很高的可伸缩性和较低的每千兆字节存储成本。 Scality是该存储领域的领导者之一，它正在推广其软件定义的存储Scality RING。 Scality RING的第一版于2010年发布。这是一种横向扩展解决方案，它使用对等通信和部署在标准x86服务器上的无共享分布式体系结构。 Scality RING支持S3和Swift对象数据访问协议，简单的HTTP API和文件访问。去年，Scality设法将其系统在医疗保健领域的安装数量增加了一倍。

Scality RING软件部署在由三个存储节点的最低配置组成的群集上，并实现了一组智能数据访问服务以及数据保护和系统管理。较高级别的是可伸缩数据访问服务（连接器），可通过SMB，NFS和S3协议向应用程序提供数据，以及用于集中管理和监视系统状态的管理器。连接器通常直接安装在存储节点上，但也可以部署在专用服务器上。

中级存储Scality RING是一个分布式虚拟文件系统，具有多种数据保护机制，系统自我修复过程以及系统管理和监视服务。堆栈的最底层是虚拟存储节点和I / O守护程序形成的分布式存储级别，它抽象了物理存储服务器和磁盘接口。

存储层的核心是基于第二代对等路由协议的可伸缩键值分布式对象存储。路由提供了有效的水平存储扩展和跨大量节点的搜索。具有这些存储服务的软件将以所需的计算能力和磁盘子系统的容量部署在所有服务器上。部署Scality RING软件的服务器（节点）通过基于IP的标准网络工厂进行连接，例如，使用10/25/40/100千兆以太网。

Scality RING包括以下软件组件：连接器服务器，用于存储元数据的分布式MESA DBMS，存储节点，I / O守护程序以及基于Web的管理门户。 MESA提供在Scality横向扩展文件系统（SOFS）抽象级别使用的对象索引和元数据管理。

Scality RING连接器使应用程序可以访问服务器上存储的数据。它们支持许多数据访问协议，包括基于代表性状态传输（REST）标准的Amazon Web Services（AWS）S3对象以及NFS，SMB和FUSE文件协议。如果您需要水平扩展I / O或并行服务多个用户，则单个应用程序可以同时使用多个RING连接器访问数据。

存储节点是一个虚拟进程，负责与与RING的分布式key hash的分配部分相关联的对象。存储节点的守护程序（所谓的bizoid）可确保存储在低级本地文件系统中磁盘上的数据的不变性。一台物理服务器（主机）上部署了六个虚拟存储节点。每个biziod都是一个低级过程的实例，该过程控制特定物理磁盘上的输入/输出操作，并保持对象键与该磁盘上特定对象地址的对应关系。

为了确保对象存储的高可用性（最多14个9），Scality RING使用了针对分布式系统优化的各种数据保护机制，而不是传统的RAID技术，包括本地和地理上分布式的复制和擦除编码，可以与复制结合使用一个连接器中的擦除编码。当存储小对象（大小最大为60 KB）时，复制是一种更具成本效益的保护解决方案，而对于大对象，则采用擦除编码，其中不需要复制大数据集。在复制过程中，使用了六个级别的服务等级（CoS）服务等级（从0到5），这对应于保存对象的3-5个副本，并且所有副本都存储在不同的磁盘上。

在使用擦除编码的情况下，将使用Reed-Solomon纠错机制，在该机制中，不是存储对象的多个副本，而是将其划分为与奇偶校验块一起写入的“数据块”。这些片段分布在RING节点之间，当一个或多个节点发生故障时，您可以从其中恢复数据。同样，无共享架构可确保较高的RING容错能力，其中不存在主（“主”）节点，该主节点的故障会导致整个系统的故障。

Scality RING生态系统


尽管Scality RING可以部署在任何标准的x86服务器体系结构上，但Gartner在其“分布式文件系统和对象存储的魔力象限”中指出，其实施需要仔细的设备选择和详细的项目设计，以及深入的IT沉浸-客户的Scality技术专家。

自2014年以来，Scality的战略合作伙伴Hewlett Packard Enterprise提供了HPE Apollo 4000 Gen10系列的两种服务器模型，这些服务器模型是专为大数据分析开发的，作为Scality RING软件定义对象存储联合解决方案的硬件平台。和对象存储：HPE Apollo 4200提供超高存储密度（在2U高的机柜中最多可容纳392 TB，可容纳28个全尺寸LFF磁盘或54个2.5英寸SFF驱动器），并设计用于基于4U机箱的HPE Apollo 4510超容量扩展（每个机箱68个全尺寸磁盘，更多）标准42U服务器机架中为9 PB）。

两种HP Apollo 4000 Gen10型号均允许您灵活配置磁盘子系统，以满足特定的存储节点性能和容量要求，并支持HP ProLiant服务器用户熟悉的HP iLO 5服务器远程管理工具，可帮助您快速部署大量Scality RING存储节点和有效地管理它们。

HPE和Scality的联合解决方案定位为非结构化数据（包括档案）的全局存储库，这时大带宽和大容量比访问存储数据的最小延迟要重要得多。它可以扩展到数千个数据存储和访问节点，在一个命名空间中提供数百PB的数据存储和数万亿个对象的存储。

为了进一步保护存储的信息，您可以使用各种备份软件包，因为Scality认证了其云存储与VEEAM，Commvault，Microfocus Data Protector，Cloudera，MAPR和WEKA.IO产品的兼容性，并在医疗保健中使用Scality RING作为医学图像的存档。提供与PACS系统的兼容性认证Fujifilm，GE Healthcare，Philips和其他几个供应商。

Healthcare Scality Ring案例研究


当前，仅在法国，就有十几家最大的医院使用400 TB至6 PB的Scality Ring存储来存储医学图像档案。例如，在马赛援助公共医院（AP-HM）实施了HPE和Scality的大型联合安装，该联合医院将马赛的四家医院合并为3400张病床，在法国排名第三。在AP-HM医院，有2,000名医生和8,500名其他医务人员在工作。

直到2011年，AP-HM都使用EMC Centera来存储PACS映像。到此时，医学图像的总容量为60 TB，但是为了保护数据而对其进行了复制，因此它们占用了120 TB的容量。每年，医院PACS都会生成另外20 TB的新图像。在2011年，AP-HM用NAS系统取代了Centera，到2017年，图像量增加到320 TB，并且每年增长到40 TB。由于NAS的保修期已满，并且由于数据量的快速增长，该存储设备的存储容量不再足够，因此AP-HM医院的管理层决定再次更换。

选择新的存储系统时，必须确保与医院中使用的所有应用程序兼容，包括对CIFS和NFS文件协议的支持，扩展到数PB的容量，可靠的保护和数据安全性。AP-HM选择了HPE和Scality RING，并从六个HPE Apollo 4510存储服务器以及两个存储META元数据数据库的HPE ProLiant DL360服务器建立了一个分布在三个数据中心的RING集群。主要应用是GE Healthcare提供的Carestream PACS系统，该系统记录了放射学研究和基因组学档案获得的图像。使用Commvault软件备份医学图像和其他数据。

Scality戒指在俄罗斯的可用性


HPE和Scality在协作项目中积累了丰富的经验。通过在俄罗斯的医疗机构中使用Scality RING，HPE莫斯科办事处的认证工程师随时可以为客户提供帮助。

在Factory Express下，HPE提供了为满足客户要求而预先配置的Apollo 4000 Gen10，以快速部署Scality RING集群，并且还为RING集群提供了这些服务器系统的参考体系结构。自去年5月以来，HPE一直在提供初始容量为240 TB的Apollo 4200 Gen10服务器的Base Bundle入门套件和预先安装的Scality RING软件。要部署基本捆绑包，您只需设置网络速度和所需的存储容量。
