しかし、この傾向はいつまで続くのでしょうか。Pythonが最終的に他の言語に置き換えられるのはいつですか？なぜこれが避けられないのですか？

今日のPythonの人気の理由


Pythonの人気は、投稿内のタグの数を見ればStackOverflowで判断できます。StackOverflowの規模を考えると、これは言語の人気のかなり客観的な指標です。


いくつかの人気のあるプログラミング言語のStackOverflowについて言及-長年にわたる変化のダイナミクス。Pythonのパフォーマンスは着実に成長していますが、競合他社は減少しています。

Rは過去数年にわたって停滞状態にあり、他の多くの言語は着実に衰退していますが、Pythonの成長は止められないようです。StackOverflowのすべての質問のほぼ14％に「python」というラベルが付けられており、この傾向は成長しているだけです。これにはいくつかの理由があります。

時間テスト済み


Pythonは90年代から存在しています。彼が成長するために十分な時間を持っているというだけではありません。そしてまた、大規模で支援的なコミュニティが形成されました。

したがって、Pythonコードの作成中に問題が発生した場合は、Google検索を使用してすばやく解決できる可能性が高くなります。誰かがすでに同様の問題に遭遇し、これについて何か有用なものを書いたからです。

初心者にもやさしい


そして重要なのは、この言語が数十年にわたって存在していたことだけではなく、そのためにプログラマーは多くの素晴らしい教科書を書いています。 Python構文も非常に読みやすいです。

まず、データ型を指定する必要はありません。変数を宣言するだけです。コンテキストから、Pythonはそれが整数、浮動小数点値、ブール値、その他のどれであるかを理解します。これは初心者にとって非常に役立ちます。 C ++でプログラミングしなければならなかった場合は、浮動小数点数の代わりに整数が代入されるため、プログラムがコンパイルされないときがどれほど悲しいかわかるでしょう。

また、PythonコードとC ++コードを比較しなければならなかった場合は、Pythonがどれほど理解しやすいかがわかります。C ++は英語を考慮して設計されているという事実にもかかわらず、これはPythonコードに比べて非常に読みにくいものです。

Pythonは古くから存在しており、その間、開発者はあらゆる場面で多くのライブラリを作成してきました。今日では、ほとんどすべてのものに必要なツールを見つけることができます。

どのような計算タスクが実行されても、そのためのPythonパッケージがある可能性があります。これにより、言語を最新の状態に保つことができます。これは、過去数年間で機械学習の人気が高まっていることで顕著です。

Pythonの欠陥-彼らは言語を埋めることができますか


現在の状況に触発されて、Pythonは深刻で長期的なものであると想像できます。しかし、他のテクノロジーと同様に、Pythonにも弱点があります。最も重要な欠点を順番に調べ、致命的かどうかを評価しましょう。

速度


Pythonが遅い。いいえ、まあ、本当に遅いです。平均すると、タスクは競合する言語よりも2〜10倍長く実行されます。

これには理由があります。最初に、動的な型付け-他の言語のように、データ型を指定する必要がない場合に非常に便利であると上記で述べたことを思い出してください。コインの裏側は、プログラムが各変数のために十分なスペースを確保して、とにかく機能するようにする必要があるため、大量のメモリを使用する必要があることです。また、「余裕を持って」メモリを使用すると、計算時間の消費量が増加します。

次に、Pythonは一度に1つのタスクしか実行できません。これは、柔軟なデータ型の結果でもあります。Pythonは、各変数に対応するデータ型が1つだけであることを確認する必要があり、並列プロセスがこれに干渉する可能性があります。

しかし、概して、速度は重要ではありません。生産的なコンピューターとサーバーは非常にアクセスしやすいので、ほんの数秒しか話せません。エンドユーザーは、自分のアプリケーションが0.001秒でロードされるか、0.01秒でロードされるかを気にしません。

可視領域


Pythonは当初、動的に制限されていました。これは基本的に、式を評価するために、コンパイラは最初に現在のブロックを検索し、次にすべての呼び出し関数を順番に検索することを意味します。

動的スコープの問題は、すべての式をすべての可能なコンテキストでチェックする必要があることです。これは面倒です。これが、最新のプログラミング言語のほとんどが静的スコープを使用する理由です。

Pythonは静的スコープに移動しようとしましたが、失敗しました。通常、関数内の関数などの内部スコープは、外部スコープを表示および変更できます。Pythonでは、内側の領域は外側の領域のみを表示でき、変更はできません。これは多くの混乱を招きます。

ラムダ関数


Pythonのすべての柔軟性にもかかわらず、ラムダの使用はかなり制限されています。これらは、Pythonでは式のみであり、演算子ではありません。

一方、変数宣言と演算子は常に演算子です。つまり、ラムダはそれらに使用できません。

式とステートメントのこの区別はかなり恣意的であり、他の言語では発生しません。


厳密なインデントはコードを読みやすくしますが、必然的にサポートが損なわれます。

スペースとタブ


Pythonでは、スペースとタブを使用して、さまざまなレベルのコードを示します。これにより、視覚的に魅力的で直感的になります。

C ++などの他の言語では、ネストされた構造にセミコロン付きの中括弧が使用されます。これは初心者にとってはそれほど美しく不便ではないかもしれませんが、コードの保守がより便利になります。プロジェクトの成長に伴い、このアプローチの重要性も高まっています。

Haskellのような新しい（比較的）言語はこの問題を解決します。それらはスペースに依存しますが、スペースなしで実行したい人のために代替構文を提供します。

モバイル開発


デスクトップPCからスマートフォンへの大幅な移行があるため、モバイルソフトウェアを作成するための信頼性の高い言語の必要性が高まっていることは明らかです。

しかし、Pythonのモバイルアプリケーションは約2回と誤って計算されました。これは、これがまったく不可能ではないという意味ではありません。これには、Kivyというパッケージがあります。

Pythonはモバイルデバイス用に作成されていません。この領域の基本的な問題を解決するのに許容できる結果が得られたとしても、モバイルアプリケーションの開発に合わせて調整された言語を使用するのが最善です。一般的に使用されるモバイルプログラミングプラットフォームには、React Native、Flutter、Iconic、Cordovaなどがあります。

もちろん、ラップトップとデスクトップPCは何年も使用されます。ただし、モバイルデバイスはデスクトップトラフィックをはるかに上回っています。Pythonの学習は、経験豊富で包括的な開発者と見なされるには不十分だと言っても安全です。

実行時エラー


最初に個別にコンパイルし、次に実行します-他のいくつかの言語では、Pythonではできません。代わりに、コードは実行されるたびにコンパイルされるため、プログラムの実行中にコード内のエラーが表示されます。これは、パフォーマンスの低下、不必要な時間の浪費、および多数のテストの必要性につながります。テストの神へのより多くのテスト！

テストだけで多くのことがわかるので、これは初心者には最適です。しかし、経験豊富な開発者にとって、Pythonで複雑なプログラムをデバッグしなければならないことは、それらを間違ったものにします。このパフォーマンスの欠如は、Pythonが近い将来に成功することを示す最も深刻な要因です。
