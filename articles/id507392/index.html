OpenLinux sebagai bagian dari modul SIM7600E-H





Mekanisme untuk mengembangkan aplikasi khusus dan memuatnya ke dalam modul tersedia di bawah sistem operasi Linux dan Windows. Pada artikel ini, kita akan memeriksa secara rinci bagaimana menggunakan contoh-contoh dari SDK yang disediakan oleh SIMCom Wireless Solutions untuk mengkompilasi dan memuat aplikasi pengguna ke dalam modul.



Sebelum menulis artikel ini, seorang teman saya, jauh dari pengembangan untuk Linux, meminta saya untuk melangkah sejauh mungkin ke pertanyaan menjelaskan proses mengembangkan aplikasi saya sendiri untuk modul SIM7600E-H. Kriteria untuk menilai ketersediaan penyerahan materi adalah ungkapan "agar saya mengerti."



Saya mengundang Anda untuk membiasakan diri dengan apa yang terjadi.

Artikel ini diperbarui dan diperbarui secara berkala.




Pendahuluan



Biasanya, modul komunikasi seluler hanya digunakan untuk transmisi data, panggilan suara, SMS, dan sejenisnya. Semua ini dilakukan melalui perintah AT yang dikirim dari mikrokontroler kontrol eksternal. Tetapi ada kategori modul yang memungkinkan Anda untuk mengeksekusi kode kustom yang diambil dari luar. Dalam beberapa kasus, ini secara signifikan mengurangi anggaran keseluruhan perangkat, memungkinkan Anda untuk menggunakan mikrokontroler yang lebih sederhana (dan juga anggaran) atau mengabaikannya sama sekali. Dengan munculnya modul LTE yang dikendalikan oleh Android atau Linux dan sumber dayanya yang kuat, Anda dapat memecahkan masalah apa pun yang tersedia untuk prosesor populer. Artikel ini akan berbicara tentang SIM7600E-H, yang dikelola oleh Linux. Kami akan melihat cara mengunduh dan menjalankan aplikasi yang dapat dieksekusi.



Dalam banyak hal, materi ini didasarkan pada dokumen “SIM7600 Open Linux development quide”, tetapi beberapa tambahan, dan terutama versi Rusia, akan bermanfaat. Artikel ini akan membantu mereka yang baru mulai mempelajari modul untuk mengunduh aplikasi demo dan memberikan keterampilan yang diperlukan untuk pekerjaan lebih lanjut.



Secara singkat tentang siapa SIM7600E-H



SIM7600E-H adalah modul yang didasarkan pada prosesor Qualcomm ARM Cortex-A7 1.3GHz, memiliki sistem operasi Linux (kernel 3.18.20) di dalam, mampu bekerja dengan frekuensi Eropa (termasuk Rusia) rentang 2G / 3G / LTE yang didukung oleh Cat.4, memberikan kecepatan unduh maksimum hingga 150Mbps dan unggahan hingga 50Mbps. Periferal yang kaya, kisaran suhu industri dan keberadaan navigasi GPS / GLONASS terintegrasi mencakup semua persyaratan untuk solusi modular modern dalam tema M2M.



Tinjauan Sistem



Modul SIM7600E-H didasarkan pada sistem operasi Linux (kernel 3.18.20). Pada gilirannya, sistem file ini didasarkan pada sistem file journaled sistem file gambar yang tidak disortir (UBIFS).



Fitur penting dari sistem file ini meliputi:



  • bekerja dengan partisi, memungkinkan Anda untuk membuat, menghapus, atau mengubah ukurannya;
  • memberikan keselarasan perekaman di seluruh volume media;
  • bekerja dengan blok yang buruk;
  • meminimalkan kemungkinan kehilangan data selama pemadaman listrik darurat atau kegagalan lainnya;
  • logging.


Deskripsi diambil dari sini , di tempat yang sama dijelaskan secara lebih rinci tentang sistem file seperti itu.



Itu Jenis sistem file ini sangat ideal untuk kondisi operasi modul yang keras dan kemungkinan masalah daya. Tetapi ini tidak berarti bahwa kondisi daya yang tidak stabil akan menjadi mode operasi yang diharapkan dari modul, ini hanya menunjukkan kelayakan perangkat yang lebih besar.

Buka kemasan dan kompilasi SDK



Setelah kami mendapatkan akses ke shell dan dapat mulai bekerja dengan baris perintah modul, kami akan mencoba untuk mengkompilasi aplikasi pertama kami untuk memuat ke dalam modul.



Banyak yang mungkin mengalami kesulitan dengan ini! Karena modul ini berjalan pada sistem operasi Linux, untuk menghindari tabrakan saat mengkompilasi kode di bawah Windows - yang terbaik adalah mengkompilasi di lingkungan asli - Linux.



Kami tidak akan membahas bagaimana jika tidak ada Linux dan keinginan untuk menginstalnya di mesin Anda, Anda dapat menginstalnya di virtual. Kami akan menggunakan VirtualBox, menginstal Ubuntu versi 20.04 (versi saat ini pada saat penulisan) dan di bawahnya kami akan mulai bekerja dengan kompiler, SDK, dll.



Kami beralih ke lingkungan Linux dan membongkar arsip yang diterima dari distributor. 



simcom@VirtualBox:~/Desktop/OpenLinux$ sudo tar -xzf MDM9x07_OL_2U_22_V1.12_191227.tar.gz


Buka direktori sim_open_sdk dan tambahkan lingkungan: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ cd sim_open_sdk
simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ source sim_crosscompile/sim-crosscompile-env-init


Kami tetap di folder yang sama dan menjalankan perintah selanjutnya ketika berada di dalamnya.

Instal pustaka libncurses5-dev jika belum diinstal: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get update && sudo apt-get install libncurses5-dev -y


Python, jika belum diinstal juga: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get install python -y


dan gcc: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get install gcc


Kompilasi:



Sekarang kita perlu mengkompilasi beberapa file, jalankan perintah berikut secara berurutan.



Jika jendela konfigurasi kernel muncul selama kompilasi - cukup pilih Keluar dan kembali ke konsol, kami tidak perlu mengonfigurasi kernel sekarang.



Kami melakukan: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make


Kompilasi bootloader: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make aboot


Kompilasi kernel: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel_menuconfig
simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel


Kompilasi sistem file root: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make rootfs


Untuk pengguna Linux, akan relevan untuk mengkompilasi driver modul: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel_module


Kompilasi demo: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make demo


Setelah itu, beberapa file baru akan muncul di direktori sim_open_sdk / output: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ ls output/
appsboot.mbn  boot.img  demo_app  helloworld  system.img

Memuat

C:\>adb push C:\demo_app /data/


Konsol akan memberi tahu kami: 



C:\demo_app: 1 file pushed, 0 skipped. 151.4 MB/s (838900 bytes in 0.005s)

C:\>adb shell

/ # cdhmod 777 /data/demo_app
