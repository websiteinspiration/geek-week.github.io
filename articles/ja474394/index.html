Javaでのネイティブコルーチンの使用

多くの現代の言語は、コルーチンを言語レベルで操作することをサポートしています。Javaは現在コルーチンをサポートしていませんが、将来的に変更される可能性があります。

C ++ 20では、コルーチンの操作のサポートが導入される予定です。
JNIを使​​用して、C ++でコルーチンを記述し、Javaコードで使用できます。

ネイティブコルーチンを記述できるものと、Javaコードでそれらを使用する方法を考えてみましょう。

ジェネレーターを使用すると、特定のタイプの値のシーケンスを作成できますが、値は遅延して同期的に生成されます。

  /* C++ code */
  generator<int> generate(int count) {
    for (int i = 0; i < count; i++) {
       co_yield i;
    }
  }


  /* Java code */
  Generator<Integer> gen1 = Coroutine.yield(5);
  Generator<Float>   gen2 = Coroutine.yield(1f, 5);
  Generator<Double>  gen3 = Coroutine.yield(v -> v * 2, 1d, 5);
		
  for (int item : gen1) {
     System.out.println("yield value: " + item);
  }

非同期ジェネレーターを使用すると、特定のタイプの値のシーケンスを作成できますが、値は遅延して非同期で生成されます。

  /* C++ code */
  async_generator<int> generate(int count) {
    for (int i = 0; i < count; i++) {
       co_await 1s;
       co_yield i;
    }
  }


  /* Java code */
  Generator<Integer> gen1 = Coroutine.yieldAsync(5);
  Generator<Float>   gen2 = Coroutine.yieldAsync(1f, 5);
  Generator<Double>  gen3 = Coroutine.yieldAsync(v -> v * 2, 1d, 5);
		
  for (int item : gen1) {
     System.out.println("yield value: " + item);
  }

タスクは遅延計算で実行される非同期計算を実行しますが、コルーチンはタスクが明示的に開始されるまで実行されません。

コルーチンは、ライトフローとして使用できます。この場合、システムで実行中のスレッドの数は、たとえば1000以下に制限できます。また、コルーチンは好きなだけ実行できます。

コルーチンを開始すると、準備ができているかどうかがチェックされます。そうでない場合、コルーチンは一時停止され、OSがハンドラを渡します。この時点で、有用なコードが実行されます。コルーチンの準備ができると、コルーチンの更新が実行されます。

  /* C++ code */
  struct awaiter {
     bool await_ready() const { return false; }
     void await_resume() {}

     void await_suspend(std::coroutine_handle<> handler) {
        /* invoke java/jni code */

        if (!handler.done()) {
           handler.resume();
        }
     }
  };

  co_await awaiter{};

スレッドを開始するときと同様に、コルーチンにはRunnableまたはCallableを渡すことができます。

   /* Java code */
   Coroutine.await(() -> {
      int sum = 5 + 10;
   });

   Task<Integer> task = Coroutine.await(() -> {
      int sum = 5 + 10;
      return sum;
   });

タイマーは、必要な期間、現在のタスクを一時停止します。

   auto operator co_await(const std::chrono::system_clock::duration& duration) {
      return timer{duration};
   }
   
   co_await 10ms;

Thread.sleep（）の代替として使用できます。

   Coroutine.await(10, TimeUnit.MILLISECONDS);

コルーチンは、ファイルシステム、ネットワークなどを操作するための非ブロッキングコードの記述にも使用できます。

ご覧のとおり、コルーチンを使用すると非同期コードを簡単に記述できるようになり、ストリームをブロックせずにコードの一部を実行できます。

C ++ 20で作成する予定のコルーチンは、純粋な言語機能として表示されます。
ジェネレーター、タスク、およびその他のコルーチンは、C ++ 23標準以降に追加される予定です。
独自のコルーチンを自分で作成することも、既成のライブラリ（cppcoroなど）を使用することもできます。

コンパイラMVSC、Clangはコルーチンを拡張機能として既にサポートしており、GCCは開発段階にあります。
