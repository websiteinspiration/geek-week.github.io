オンラインストアの作成における私たちの遠隔勤務経験



今日では、検疫とコロナウイルスのために、多くの企業が従業員にリモート作業を提供する方法を考える必要があります。ほとんど毎日、リモートワークへの移行の技術的側面と心理的側面の両方を明らかにする記事が表示されます。同時に、フリーランサーや世界中の従業員や顧客と長い間関わってきたIT企業など、そのような仕事にはすでに膨大な経験が蓄積されています。

大規模なIT企業を遠隔地の仕事に移すことは容易ではないかもしれません。ただし、多くの場合、よく知られているツールと手法を使用できます。この記事では、技術面からのリモートエクスペリエンスを確認します。この情報が企業が新しい条件に適応するのに役立つことを願っています。コメント、提案、追加はありがたいです。

会社のリソースへのリモートアクセス


IT企業がオフィスで働いている場合、原則として、システムユニット、ラップトップ、サーバー、プリンター、スキャナー、および電話があります。これらはすべてルーターを介してインターネットに接続されています。私たちの会社はその存在の初期には、まさにそのような機器をオフィスに収容していました。

ここで、1〜2日ですべての従業員を家に帰らせる必要があり、プロジェクトの作業が停止しないようにする必要があることを想像してください。この場合はどうしますか？

ラップトップを使用すると、すべてが明確です。従業員はノートパソコンを持って行くだけです。システムユニットとモニターは輸送がより困難ですが、それでも輸送は可能です。

しかし、サーバー、プリンター、電話はどうすればよいでしょうか。

オフィスのサーバーへのアクセスの問題を解決する


従業員が家に帰り、サーバーがオフィスに残り、監視する人がいる場合、会社のサーバーへの従業員の安全なリモートアクセスを組織化するという問題を解決するだけです。これはシステム管理者の仕事です。

Microsoft Windows Serverがオフィスサーバーにインストールされている場合（初期の頃と同様）、管理者がRDPを介してターミナルアクセスをセットアップするとすぐに、従業員は自宅からサーバーで作業できるようになります。端末アクセス用の追加ライセンスを購入する必要がある場合があります。いずれの場合も、従業員は自宅にMicrosoft Windowsがインストールされたコンピューターを必要とします。

Linux OSを搭載したサーバーには、自宅からアクセスでき、ライセンスを購入する必要はありません。会社の管理者は、SSH、POP3、IMAP、SMTPなどのプロトコルを介してアクセスを設定するだけで十分です。

これがまだ行われていない場合は、管理者への不正アクセスからサーバーを保護するために、少なくともオフィスサーバーにファイアウォール（ファイアウォール）をインストールし、VPNを使用して従業員のリモートアクセスを構成することをお勧めします。私たちは、ほとんどすべてのプラットフォームとオペレーティングシステムで利用可能なOpenVPNソフトウェアを使用しています。

しかし、すべてのサーバーをオフにしてオフィスが完全に閉鎖されたらどうなるでしょうか。残りの4つのオプションがあります。

  • 可能であれば、クラウドテクノロジーに完全に切り替えます-クラウドCRMシステムを使用する、Googleドキュメントに一般的なドキュメントを保存するなど。
  • サーバーをシステム管理者のホームに移動します（彼は喜んでいます...）;
  • サーバーを受け入れることに同意するデータセンターにサーバーを移動します。
  • データセンターまたはクラウドでサーバー容量を借りる

サーバーを転送してインストールする必要がないため、最初のオプションが適しています。クラウドテクノロジーへの移行の結果は、さらに役立つでしょう。それらは、メンテナンスとメンテナンスの費用と労力を節約します。

2番目のオプションは、サーバーが24時間稼働し、かなりのノイズが発生するため、自宅でシステム管理者に問題を引き起こします。また、オフィスにサーバーが1台ではなく、ラック全体がある場合はどうでしょうか。



サーバーをデータセンターに転送することも簡単ではありません。原則として、ラックへの設置に適したサーバーのみをデータセンターに配置できます。同時に、オフィスでは、Big Towerタイプのエンクロージャー内のサーバーや、通常のデスクトップコンピューターさえ使用することがよくあります。同様の機器を自宅でホストすることに同意するデータセンターを見つけるのは困難です（たとえば、このようなデータセンターはありますが、PlanetaHostデータセンターに配置しました）。もちろん、適切な数のラックをレンタルして、そこに機器を取り付けることもできます。

サーバーをデータセンターに移動する際のもう1つの問題は、ほとんどの場合、サーバーのIPアドレスを変更する必要があることです。そのため、IPアドレスに関連付けられている場合は、サーバーソフトウェアを再構成するか、一部のソフトウェアライセンスを変更する必要があります。

データセンターでサーバー容量をレンタルするオプションは、サーバーをどこかに輸送する必要がないという点で簡単です。ただし、システム管理者は、すべてのソフトウェアを再インストールし、オフィスにインストールされているサーバーから必要なデータをコピーする必要があります。

オフィステクノロジーがMicrosoft Windowsの使用に基づいている場合は、必要な数のターミナルライセンスを持つMicrosoft Windowsサーバーをデータセンターでレンタルできます。サーバーをリモートで使用する従業員ごとに1つのライセンスを取得します。

物理サーバーをレンタルすると、クラウドで仮想サーバーをレンタルするよりも2〜3倍の費用がかかります。ただし、サーバー全体ではなく、非常に小さな容量が必要な場合は、クラウドを使用したオプションの方が安上がりです。

クラウドリソースの価格の上昇は、クラウド内のハードウェアリソースの予約の結果です。その結果、クラウドはおそらくレンタルされた物理サーバーよりも信頼性が高くなります。しかし、ここではすでにリスクを評価し、お金を数える必要があります。

オンラインストアの作成に携わっている当社については、必要なすべてのリソースは長い間データセンターにあり、リモートで利用できます。これらは、ストアのホスティングに使用される専用の専用物理サーバーであり、ソフトウェア開発者、レイアウト設計者、およびテスター向けの仮想マシンでもあります。

オフィスから自宅への転職


すでに述べたように、従業員は仕事用のコンピューターを持ち運ぶことができます-ラップトップまたはモニター付きのシステムユニット。必要に応じて、従業員が自宅配達を注文することにより、新しいラップトップを購入できます。もちろん、新しいコンピュータは必要なソフトウェアをインストールする必要があり、追加の時間コストが発生します。

従業員がすでにMicrosoft Windowsを備えた自宅のコンピューターを持っている場合は、それらをMicrosoft Windows Serverターミナルとして使用したり、Linuxサーバーにアクセスしたりできます。 VPNアクセスを構成するのに十分です。

私たちの従業員はWindowsとLinuxの両方で働いています。Microsoft Windowsサーバーは非常に少ないため、このOSのターミナルライセンスを購入する必要はありません。データセンターにあるリソースへのアクセスについては、VPNを使用して編成され、さらに各サーバーにインストールされたファイアウォールによって制限されます。

自宅で働く従業員にヘッドセット（マイク付きヘッドホン）とビデオカメラを提供することを忘れないでください。これにより、オフィスと同じように、リモートで非常に効率よく通信できます。

多くの人々は、コンピューターにさまざまな専用モニターをインストールすることにより、自宅の勤務時間中に従業員が何をするかを制御しようとします。私たちはこれをしたことはありません、私たちは仕事の結果だけを制御しました。原則として、これで十分です。

プリンターとスキャナーをどうするか


Webサイトのソフトウェア開発者は、プリンターやスキャナーをほとんど必要としません。ただし、従業員がそのような機器を必要とする場合、遠隔地の作業に切り替える際に問題が発生します。


通常、オフィスMFPは、高速で、大きくて重いものが設置されています。はい、ほとんどの場合、印刷とスキャンを行う必要がある従業員に家に帰らせることができます。もちろん、この従業員は彼をホストする機会がないでしょう。

しかし、多くの従業員が頻繁にドキュメントをスキャンして印刷する場合は、MFPを購入して自宅に設置するか、会社のビジネスプロセスを変更する必要があります。

新しいMFPの輸送と取得の代替として-可能な限り電子ドキュメント管理への移行を加速。

紙や電子文書で作業する


リモートの作業に切り替える前に、ドキュメントフロー全体を電子形式に変換することをお勧めします。たとえば、会計ドキュメントの交換にはDIADOKを使用し、顧客銀行を通じて請求書を支払います。

このようなシステムを導入する場合、電子文書管理に関与するすべての従業員（たとえば、会計士）に、強化された資格のある電子署名を備えたキーフォブを提供する必要があります。そのような装身具を手に入れるにはしばらく時間がかかるかもしれないので、この質問に事前に戸惑うのが良いでしょう。

DIADOKでは（同様のサービスと同様）、他の電子ドキュメント管理オペレーターとのローミングを構成できます。これは、請負業者があなた以外のワークフローシステムを使用している場合に必要になります。

あなたまたはあなたの請負業者の一部が古い方法で文書を扱う場合、郵便局への訪問または宅配便業者への訪問で通常の紙の手紙を送受信する必要があります。検疫の場合、そのような操作は最小限に抑える必要があります。

テレフォニーをどうするか


最初の数年間は、固定電話と携帯電話を使用していました。しかし、私たちはすぐに、多数の従業員と顧客には、さらに適切なソリューションが必要であることを認識しました。

私たちにとって最も便利なのは、MangoTelecomの仮想PBXでした。その助けを借りて、市の電話番号（つまり、オフィスの物理的な場所）への拘束を取り除きました。また、PBXをCRMに統合したり、カスタマーサービスの会話を録音したり、コール転送を設定したりする機会を得ました。

次に、仮想PBXアプリケーションをスマートフォン、ラップトップ、またはデスクトップコンピューターにインストールできます。これにより、ロシアの番号に電話をかけたり、海外からでも国内料金で電話を受けたりすることができます。

したがって、仮想PBXを使用すると、オフィスから自宅への従業員の移動をビジネス継続性の観点からほとんど見えなくすることができます。

オフィスのPBXを使用していて、電源がオフになっているときにシャットダウンすることが避けられない場合は、仮想PBXへの切り替えを検討してください。オフィスPBXの都市番号から仮想PBXの着信番号へのコール転送を有効にできるかどうか、電話プロバイダーに確認してください。この場合、仮想PBXに切り替えても、着信通話が失われることはありません。

従業員同士の通話については、仮想電話交換を利用する場合、原則として通話料はかかりません。

従業員のリモート選択とトレーニング


スタッフを補充するために、会社の初期には候補者を常にオフィスに招待し、古典的な面接を行い、割り当てを行いました。次に、初心者向けの個別研修室を開設しました。

しかし、時間の経過とともに、私たちは完全にリモートの人員選択に切り替えました。

最初の選択は、HHウェブサイトの空室に添付されているテストまたはその他の求人サービスを使用して実行できます。これらのテストでは、適切な準備を行うことで、要件を満たしていないかなりの数の候補者をフィルターに掛けることができます。

そして、すべてが簡単です-私たちはSkypeを使用しています。 Skypeを使用し、常にビデオカメラをオンにしておくと、候補者がテーブルの隣に座っていた場合と同じくらい効果的に面接を実施できます。



特定の欠点がある場合、Skypeには類似のシステムに比べて非常に重要な利点があります。まず第一に、Skypeを介してコンピューターのデスクトップのデモンストレーションを行うことができます。これは、トレーニングや作業上の問題について話し合うときに非常に必要です。さらに、Skypeは無料で、すべての主要なプラットフォームで利用でき、コンピューターやスマートフォンに簡単にインストールできます。

会議を開催したり、複数の従業員をトレーニングしたりする必要がある場合は、Skypeでグループを作成するだけです。デスクトップのデモンストレーションを使用して、スピーカーまたは教師は必要なすべての資料を会議の参加者に持ち込むことができます。同時に、チャットウィンドウでは、リンクの公開、テキストメッセージの送信、ファイルの交換、ダイアログの実施を行うことができます。

Skypeのクラスに加えて、トレーニング映画を準備しています（Camtasia Studioプログラムを使用しますが、慣れているものを使用できます）。これらの映画が社内でのみ使用される場合は、サーバーに投稿し、すべての場合はYouTubeに投稿します。

ほとんどの場合、このトレーニングフィルムの組み合わせ、Skypeグループクラスでの対話とデスクトップのデモンストレーション、教師と生徒間の個別のコミュニケーションにより、リモートでトレーニングを実施できます。

はい、デスクトップをユーザーのグループにデモしたり、ウェビナーを実施したり、トレーニングプラットフォーム（無料のものを含む）を実施したりするためのサービスがあります。しかし、これらすべてについて、プラットフォームでの作業方法の研究に費やされた時間またはお金で支払う必要があります。同時に、無料のプラットフォームが有料になる場合があります。同時に、多くの場合、Skypeの機能で十分です。

プロジェクトコラボレーション


プロジェクトでの共同作業の過程で、私たちは毎日および毎週の会議を開催し、ペアプログラミングとコードレビューを使用します。会議とコードの表示用にSkypeグループが作成されています。必要に応じて、デスクトップデモが使用されます。コードについては、データセンターにあるGitLabサーバーに格納されています。

Googleドキュメントを使用してドキュメントのコラボレーションを整理します。

これらすべてに加えて、アプリケーション処理およびリソース計画システム（CRMおよびERP）と統合された内部Klondikeナレッジベースがあります。これらのツールは、データセンターのサーバーに配置され、長年にわたって作成および改善されてきました。これにより、クライアントからの多数のアプリケーションを効率的に処理し、実行者を任命し、アプリケーションについてのディスカッションを実施し、労働時間を考慮に入れ、さらに多くのことができます。

ほとんどの場合、あなたの会社はすでに同じようなものを使用しており、従業員のリモート作業に切り替える場合、関連リソースへのリモートアクセスを提供するだけで十分です。

リモートユーザーサポート


私たちのユーザーは、ロシアのほぼすべての地域で営業しているオンラインストアの所有者および管理者です。もちろん、リモートでサポートします。

当社のサポートサービスは、チケットシステムを介して機能し、電子メールと電話で質問に回答し、オンラインストアの管理Webサイトと当社のWebサイトを介してチャットします。

タスクについて話し合う段階では、Telegram、WhatsApp、Skypeなど、クライアントが利用できるメッセンジャーを使用します。

ときどき、クライアントが自分のコンピュータで何をしているかを確認する必要があります。これは、デスクトップデモモードでSkypeを介して実行できます。

必要に応じて、TeamViewer、Ammee Admin、AnyDeskなどのツールを使用して、ユーザーのコンピューターでリモートで作業できます。これらのツールを使用するには、クライアントは適切なソフトウェアをコンピューターにインストールする必要があります。

VPNアクセス設定


OpenVPNサーバーは、さまざまなデータセンター（Debian 10 OSを使用）にある仮想マシンにインストールされています。OpenVPNクライアントは、Debian、Ubuntu、MacOS、Microsoft Windowsの環境にある従業員の稼働中のコンピューターにインストールされます。

インターネットには、OpenVPNサーバーとクライアントをインストールするための多くの指示があります。OpenVPNインストールおよび構成ガイドも使用できます。

従業員用のキーを作成するための手動の手順は非常に疲れると言わざるを得ません。10秒以内に新しいユーザーを接続するには、次のようなスクリプトをスポイラーの下で使用します。

キーを作成するためのスクリプト
#!/bin/bash

if [ -z "$1" ]
then
echo "============================================================="
echo "VPN -- Generate crt key pair"
echo "============================================================="
echo "Usage:  bash gen.sh username"
exit
fi

echo "============================================================="
echo "VPN -- Generate crt key pair for user: $1"
echo "============================================================="

ADMIN_EMAIL="admin@mycompany.ru"
USER=$1

RSA="/home/ca/easy-rsa-master/easyrsa3/"
PKI="$RSA"pki/
PKI_KEY="$PKI"private/
PKI_CRT="$PKI"issued/
USR_CRT="/home/ca/cert_generation/user_crt/"
USR_DISTR="/home/ca/cert_generation/user_distr/"

# If user key does not exists, create it

if [ ! -f "$PKI_KEY$USER.key" ]
then
  echo "File $PKI_KEY$USER.key does not exists, creating..."
  cd "$RSA"
  ./easyrsa build-client-full $USER nopass
fi

# Removing user folder, if exists

if [ -e "$USR_CRT$USER/" ]
then
echo "Already exists, removing user folder $USR_CRT$USER..."
rm -r -f "$USR_CRT$USER/"
fi

# Create user folder for key and other files

mkdir $USR_CRT/$USER/

# Copy OpenVPN key, cert and config files to user folder

cp "$PKI_KEY$USER.key" "$USR_CRT$USER/$USER.key"
cp "$PKI_CRT$USER.crt" "$USR_CRT$USER/$USER.crt"
cp "$PKI"ca.crt "$USR_CRT$1"

cp "$USR_DISTR"ta.key "$USR_CRT$USER"
cp "$USR_DISTR"openssl.cnf "$USR_CRT$USER"

# Copy Manual files

cp "$USR_DISTR"readme_vpn_win.txt "$USR_CRT$USER"

# Replace string "change_me" in configuration files whis user name $USER

cp "$USR_DISTR"server.conf "$USR_CRT$USER"/server.conf.1
cp "$USR_DISTR"mycompany_vpn.ovpn "$USR_CRT$USER"/mycompany_vpn_$USER.ovpn.1
cp "$USR_DISTR"readme_vpn_win.txt "$USR_CRT$USER"/readme_vpn_win.txt.1

sed "s/change_me/$USER/g" "$USR_CRT$1"/server.conf.1 > "$USR_CRT$1"/server.conf
rm "$USR_CRT$USER"/server.conf.1

sed "s/change_me/$USER/g" "$USR_CRT$1"/mycompany_vpn_$USER.ovpn.1 > "$USR_CRT$1"/mycompany_vpn_$USER.ovpn
rm "$USR_CRT$USER"/mycompany_vpn_$USER.ovpn.1

sed "s/change_me/$USER/g" "$USR_CRT$1"/readme_vpn_win.txt.1 > "$USR_CRT$1"/readme_vpn_win.txt
rm "$USR_CRT$USER"/readme_vpn_win.txt.1

# Create tar.gz and send it to administrator e-mail

tar -cvzf "$USR_CRT$USER/$USER.tar.gz" "$USR_CRT$USER/"
echo "VPN: crt, key and configuration files for user $USER" | mutt $ADMIN_EMAIL -a $USR_CRT/$USER/$USER.tar.gz -s "VPN: crt, key and configuration files for user $USER"

echo "--------->  DONE!"
echo "Keys fo user $USER sent to $ADMIN_EMAIL"


このスクリプトには、起動時にパラメーターとしてユーザーID（ラテン文字を使用）が渡されます。

スクリプトは、OpenVPNサーバーのインストール時に作成されるパスワード認証局を要求します。さらに、このスクリプトは、OpenVPNクライアントに必要なすべての証明書と構成ファイル、およびOpenVPNクライアントをインストールするためのドキュメントファイルを含むディレクトリを形成します。

構成ファイルとドキュメントファイルを作成するとき、change_meはユーザーIDに置き換えられます。

次に、必要なすべてのファイルを含むディレクトリがパッケージ化され、管理者に送信されます（アドレスはスクリプトで直接指定されます）。結果のアーカイブをユーザーのメールアドレスのユーザーに転送するだけです。
