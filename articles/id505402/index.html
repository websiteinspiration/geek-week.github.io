Menumpangkan tekstur 2d pada objek 3d menggunakan p5.js (bagian 2 - menerapkan pola ke kubus)



Ini adalah bagian kedua dari proyek di mana kita memaksakan pola 2d dinamis pada objek 3d. Anda dapat mulai dengan bagian pertama ( Bagian 1 ) atau cukup menyalin kode yang dihasilkan beberapa paragraf di bawah ini.

Beberapa waktu lalu, saya membuat pola yang dihasilkan menggunakan beberapa baris kode di p5.js library.
p5.js adalah perpustakaan Javascript yang dibuat pada 2014. Tujuan utamanya adalah untuk menjadi jendela ke dunia pemrograman bagi perancang, seniman, guru, dan perwakilan profesi kreatif lainnya. Ini mendukung animasi dan diagram, dengan mudah diterjemahkan ke format web.

Sekarang saya akan memberi tahu Anda cara membuat bagian terakhir - menerapkan pola ke objek 3d (dalam hal ini, kubus).

Untuk mulai bekerja dari awal, Anda perlu membuat proyek di editor online editor.p5js.org. Kami terus mengubah kode berikut:

  background(220);
  createCanvas(400, 400);

Untuk membuat kanvas kita banyak, kita perlu menambahkan parameter WEBGL ke fungsi yang membuat kanvas kita: createCanvas (400.400, WEBGL). Karena titik referensi untuk objek semacam itu sudah ada di tengah, kita dapat (dan harus!) Menghapus terjemahan (200.200), yang melakukan koreksi dalam gambar datar.

Untuk mendiversifikasi skema warna dan membuat gambar yang lebih spektakuler, mari kita ganti parameter fill () dengan variabel kita r, a, c - sekarang dengan setiap iterasi baru dari loop draw tak terbatas (), warna elips akan sedikit berubah. Selain itu, mari kita tambahkan jarak antara lingkaran dan sudut kemiringan dengan mengubah laju perubahan variabel c dan a.

   =  + 0.06
  a = a + 0.3

Sebagai peningkatan terakhir, mari tambahkan push () sebelum kode pola dan pop () setelahnya. Membingkai dengan cara ini membuat kode terisolasi - tidak akan terpengaruh oleh objek lain yang akan kita buat.

let r=10
let a=0
let c=20

function setup() {
  background(220);
  createCanvas(400, 400, WEBGL);
  
}

function draw() {
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  fill(r,a,c);
  ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
}



Pada akhirnya, kita mendapatkan sesuatu seperti ini. Saatnya membuat kubus.

Dalam fungsi draw () yang sama, setelah pop () kita menambahkan kode kotak:

  push()
  box(200)
  pop()

Sekarang, kotak dengan ukuran 200 itu menghadap kami hanya di satu sisi dan terlihat seperti kotak. Untuk memperkirakan volume, kita perlu menggunakannya. Untuk melakukan ini, tambahkan variabel keempat, atas, ke r, a, c yang sudah ada, sudut dengan nilai default 0. Agar parameter ini memengaruhi kubus kami, setelah dorongan kedua (), tetapi sebelum membuat kotak. tambahkan tiga fungsi: rotateX (), rotateY (), rotateZ () dengan parameter sudut untuk masing-masing.

Adalah logis bahwa suatu objek dengan sudut rotasi 0 derajat tidak akan berputar, oleh karena itu, kami menambahkan perubahan nilai sudut variabel menjadi 0,003 (untuk permulaan itu akan cukup). Hasilnya, kami mendapatkan kode berikut:

let r=10
let a=0
let c=20
let angle = 0

function setup() {
  background(220);
  createCanvas(400, 400, WEBGL);
  
}

function draw() {
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  fill(r,a,c);
  ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
  
  push()
  rotateX(angle), 
  rotateY(angle), 
  rotateZ(angle)
  box(200)
  angle = angle + 0.003
  pop()
}

Karena latar belakang kami dibuat hanya satu kali (sedang dalam pengaturan) - kami menangkap bug pertama, kubus kami menghapus latar belakang saat diputar.



Untuk memperbaikinya, Anda cukup mentransfer latar belakang (200) di dalam fungsi draw () .Namun, sekarang kita menangkap masalah lain - lingkaran kita mulai menghilang, seperti yang mereka lakukan di awal, karena sekarang mereka ditimpa dengan setiap siklus baru dan hanya muncul selama sepersekian detik dari ketiadaan.

Untuk mengatasi masalah ini, kita perlu membuat variabel baru (misalnya, seni) tanpa nilai default di awal di samping variabel tingkat atas lainnya.

Di dalam fungsi setup (), kami menyamakannya dengan fungsi createGraphics (400.400), dan kemudian menggunakannya di tempat lain.
Pastikan Anda mengatur parameter yang sama seperti saat membuat Kanvas untuk mempertahankan dimensi.

Sekarang di dalam kode pola, kita menambahkan variabel seni melalui titik ke elips dan propertinya (seperti fill () dan lainnya, jika Anda membuatnya). 

art.fill(r,a,c);
  art.ellipse (x,y,10,10);

Setelah itu, tetap menerapkan pola pada kubus sebagai tekstur. Untuk melakukan ini, tambahkan properti tekstur (seni) di dalam kode kotak-kubus. Logikanya adalah sebagai berikut: kita membuat esensi dari sebuah objek seni grafis terisolasi, yang memanggil fungsi untuk membuat pola dengan properti yang ditentukan, dan setelah itu kita memberikan seni sebagai parameter properti tekstur untuk kubus.

let r=10
let a=0
let c=20
let angle = 0
let art

function setup() {
  createCanvas(400, 400, WEBGL);
  art = createGraphics(400,400);
}

function draw() {
  background(220);
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  art.fill(r,a,c);
  art.ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
  
  push()
  texture(art)
  rotateX(angle), 
  rotateY(angle), 
  rotateZ(angle)
  box(200)
  angle = angle + 0.003
  pop()
}



Tetap untuk menyelesaikan masalah terakhir: pola kita mulai muncul dari sudut kubus. Untuk memulai dari tengah, kita cukup menambahkan setengah lebar dan panjang kubus ke parameter x dan y untuk menggeser bagian tengah mulai menggambar pola.

art.ellipse (x+200,y+200,10,10);



Jika Anda ingin mendiversifikasi pola, Anda dapat mencoba hanya mengubah parameter atau bahkan menambahkan beberapa pola berpotongan, misalnya, dari proyek lain - Cara membuat ulang efek dari pola moire di perpustakaan p5.js untuk pemula (panduan cepat)
