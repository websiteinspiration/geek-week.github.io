効果的および非効果的なPythonコーディング技術

こんにちは、ハブル！Duomlyによる記事「Pythonでのコーディングのグッドプラクティスとバッドプラクティス」の翻訳に注目します。

Pythonは読みやすさに重点を置いた高水準プログラミング言語です。これは、Zen of PythonまたはPEP 20に従って開発、保守、および使用されることがよくあり

ます。この記事では、Pythonでよく見られるコーディング手法の良い例と悪い例をいくつか紹介します。

展開を使用したコンパクトコードの記述


パッケージ化と解凍は強力なPythonツールです。解凍を使用して、変数に値を割り当てることができます：

>>> a, b = 2, 'my-string'
>>> a
2
>>> b
'my-string'

これを使用して、おそらく変数の値を交換する最もコンパクトなコードを書くことができます

>>> a, b = b, a
>>> a
'my-string'
>>> b
2

これは素晴らしいです！

解凍は、より複雑なケースで複数の変数に値を割り当てるためにも使用できます。たとえば、次のように書くことができます：

>>> x = (1, 2, 4, 8, 16)
>>> a = x[0]
>>> b = x[1]
>>> c = x[2]
>>> d = x[3]
>>> e = x[4]
>>> a, b, c, d, e
(1, 2, 4, 8, 16)

ただし、代わりに、より簡潔で読みやすい方法を使用することもできます。

>>> a, b, c, d, e = x
>>> a, b, c, d, e
(1, 2, 4, 8, 16)

かっこいいですよね？しかし、もっとクールに書くことができます：

>>> a, *y, e = x
>>> a, e, y
(1, 16, [2, 4, 8])

コツは、*付きの変数が他の変数に割り当てられていない値を収集することです。

チェーンを使用したコンパクトコードの記述


Pythonでは、比較演算子のチェーンを使用できます。したがって、2つ以上の比較が真であるかどうかを確認する必要はありません。

>>> x = 4
>>> x >= 2 and x <= 8
True

代わりに、よりコンパクトな形式の書き込みを使用できます。

>>> 2 <= x <= 8
True
>>> 2 <= x <= 3
False

Pythonは変数値の連鎖もサポートしています。したがって、複数の変数に同じ値を同時に割り当てる場合は、簡単な方法で行うことができます。

>>> x = 2
>>> y = 2
>>> z = 2

よりコンパクトな方法は、アンパックを使用することです：

>>> x, y, z = 2, 2, 2

ただし、チェーニングを使用すると、さらにクールに見えます。

>>> x = y = z = 2
>>> x, y, z
(2, 2, 2)

値が異なる場合は注意してください！すべての変数は同じ値を参照します。

なしを確認


NoneはPythonのユニークなオブジェクトではありません。Cライクな言語ではnullなどの類似体があります。

変数がNoneを参照しているかどうかは、比較演算子==および！=を使用して確認できます。

>>> x, y = 2, None
>>> x == None
False
>>> y == None
True
>>> x != None
True
>>> y != None
False

しかし、使用してはあるとないの好適：

>>> x is None
False
>>> y is None
True
>>> x is not None
True
>>> y is not None
False

さらに、xはNoneではありませんが、可読性が低くない構成を使用することをお勧めします（xはNoneです）。

シーケンスとマッピング


Pythonでループを実装するには、いくつかの方法があります。Pythonは、実装を簡素化するためにいくつかの組み込みクラスを提供しています。

ほとんどの場合、範囲を使用して整数を返すループを取得できます。

>>> x = [1, 2, 4, 8, 16]
>>> for i in range(len(x)):
...     print(x[i])
... 
1
2
4
8
16

ただし、これを行うにはより良い方法があります。

>>> for item in x:
...     print(item)
... 
1
2
4
8
16

しかし、ループを逆の順序で開始したい場合はどうでしょうか。もちろん、範囲を再び使用できます。

>>> for i in range(len(x)-1, -1, -1):
...     print(x[i])
... 
16
8
4
2
1

ただし、シーケンスを反転する方がよりコンパクトな方法です。

>>> for item in x[::-1]:
...     print(item)
... 
16
8
4
2
1

Pythonの方法は、逆のシーケンスを使用して、シーケンスの要素を逆の順序で返すループを取得することです。

>>> for item in reversed(x):
...     print(item)
... 
16
8
4
2
1

シーケンス要素とそれに対応するインデックスの両方が必要になる場合があります。

>>> for i in range(len(x)):
...     print(i, x[i])
... 
0 1
1 2
2 4
3 8
4 16

列挙を使用して、インデックスと要素の値を返す別のループを取得することをお勧めします。

>>> for i, item in enumerate(x):
...     print(i, item)
... 
0 1
1 2
2 4
3 8
4 16

カッコいい。しかし、2つ以上のシーケンスを分類する必要がある場合はどうでしょうか。もちろん、範囲を再び使用できます。

>>> y = 'abcde'
>>> for i in range(len(x)):
...     print(x[i], y[i])
... 
1 a
2 b
4 c
8 d
16 e

この場合、Pythonもより良いソリューションを提供します。あなたはzipを適用することができます：

>>> for item in zip(x, y):
...     print(item)
... 
(1, 'a')
(2, 'b')
(4, 'c')
(8, 'd')
(16, 'e')

この方法とアンパックを組み合わせることもできます。

>>> for x_item, y_item in zip(x, y):
...     print(x_item, y_item)
... 
1 a
2 b
4 c
8 d
16 e

範囲は非常に役立つことに注意してください。それにもかかわらず、（上記のように）代替を使用する方が最適な場合があります。
辞書を列挙すると、そのキーが得られます。

>>> z = {'a': 0, 'b': 1}
>>> for k in z:
... print(k, z[k])
... 
a 0
b 1

ただし、.items（）メソッドを使用して、キーとそれに対応する値を取得できます。

>>> for k, v in z.items():
...     print(k, v)
... 
a 0
b 1

.keys（） メソッドと.values（）メソッドを使用して、それぞれキーと値を反復処理することもできます。

ゼロとの比較


数値データがあり、数値がゼロに等しいかどうかを確認する必要がある場合は、比較演算子==および！=を使用できます（ただし、常にそうである必要はありません）。

>>> x = (1, 2, 0, 3, 0, 4)
>>> for item in x:
...     if item != 0:
...         print(item)
... 
1
2
3
4

Pythonが提案する方法は、ゼロをFalseとして解釈し、他のすべての数値をTrueとして解釈することです。

>>> bool(0)
False
>>> bool(-1), bool(1), bool(20), bool(28.4)
(True, True, True, True)

それを念頭に置いて、ifアイテムの代わりにifアイテムを使用できます！= 0：

>>> for item in x:
...     if item:
...         print(item)
... 
1
2
3
4

同じロジックに従い、if item == 0の代わりにif not itemを使用できます。

変更可能なオプションの引数を回避する


Pythonには、関数やメソッドに引数を提供するための非常に柔軟なシステムがあります。オプションの引数はこれの一部です。ただし、注意が必要です。通常、変更可能なオプションの引数を使用する必要はありません。次の例を考えてみます。

>>> def f(value, seq=[]):
...     seq.append(value)
...     return seq

一見すると、seqを指定しない場合、f（）は空のリストに値を追加し、[値]のようなものを返します。

>>> f(value=2)
[2]

いいですね 番号！次の例を検討してください。

>>> f(value=4)
[2, 4]
>>> f(value=8)
[2, 4, 8]
>>> f(value=16)
[2, 4, 8, 16]

驚いた？もしそうなら、あなたは一人ではありません。

関数が呼び出されるたびに、オプションの引数（この場合はリスト）の同じインスタンスが提供されるようです。多分これはあなたが必要とするものになるでしょう。ただし、これを回避する必要がある可能性がはるかに高くなります。1つの方法は次のとおりです。

>>> def f(value, seq=None):
...     if seq is None:
...         seq = []
...     seq.append(value)
...     return seq

よりコンパクトなバージョン：

>>> def f(value, seq=None):
...     if not seq:
...         seq = []
...     seq.append(value)
...     return seq

今、私たちは別の結論を得ます：

>>> f(value=2)
[2]
>>> f(value=4)
[4]
>>> f(value=8)
[8]
>>> f(value=16)
[16]

ほとんどの場合、これが必要です。

古典的なゲッターとセッターの回避


Pythonでは、C ++およびJavaと同じ方法でゲッターメソッドとセッターメソッドを定義できます。

>>> class C:
...     def get_x(self):
...         return self.__x
...     def set_x(self, value):
...         self.__x = value

これらを使用する方法は次のとおりです。

>>> c = C()
>>> c.set_x(2)
>>> c.get_x()
2

場合によっては、これが最善の方法です。ただし、特に単純なケースでは、プロパティを使用する方がよい場合がよくあります。

>>> class C:
...     @property
...     def x(self):
...         return self.__x
...     @x.setter
...     def x(self, value):
...         self.__x = value

プロパティは、従来のゲッターやセッターよりもPythonicと見なされます。C＃と同じように、つまり通常のデータ属性と同じように使用できます。

>>> c = C()
>>> c.x = 2
>>> c.x
2

したがって、一般的には、可能な場合はプロパティを使用する方が適切です。

保護されたクラスメンバーへのアクセスを避ける


Pythonでは、そのようなプライベートクラスメンバーはありません。ただし、要素名の先頭に（_）を記述した場合、クラス外で変更するためのアクセスは拒否されます。

たとえば、次のコードを考えます。

>>> class C:
...     def __init__(self, *args):
...         self.x, self._y, self.__z = args
... 
>>> c = C(1, 2, 4)

クラスCインスタンスには、.x、.y、._ Czの3つのデータ要素があります。メンバー名が二重下線（dunder）で始まる場合、変更されます。そのため、._ z要素の代わりに._Czを使用します。

これで、直接.xにアクセスまたは変更できます。

>>> c.x  # OK
1

クラス外の._yにアクセスしたり変更したりすることもできますが、これはマナー違反と見なされます。

>>> c._y  # Possible, but a bad practice!
2

変数が変更されているため、.zにはアクセスできませんが、._ Czにはアクセスまたは変更できます。

>>> c.__z # Error!
Traceback (most recent call last):
File "", line 1, in 
AttributeError: 'C' object has no attribute '__z'
>>> c._C__z # Possible, but even worse!
4
>>>

これは避けなければなりません。クラスの作成者は、おそらくクラスの外ではこの要素を使用しない方がよいことを強調するために、名前をアンダースコアで始めるでしょう。

コンテキストマネージャを使用してリソースを解放する


リソースを適切に管理するためのコードを記述する必要がある場合があります。これは、ファイル、データベース接続、または管理されていないリソースを持つ他のオブジェクトを操作するときによく発生します。たとえば、ファイルを開いて処理することができます。

>>> my_file = open('filename.csv', 'w')
>>> # do something with `my_file`

メモリを正しく管理するには、作業の完了後にこのファイルを閉じる必要があります。

>>> my_file = open('filename.csv', 'w')
>>> # do something with `my_file and`
>>> my_file.close()

この方法で行うことは、まったく行わないよりも優れています。しかし、ファイルの処理中に例外が発生した場合はどうなりますか？その後、my_file.close（）は実行されません。これは、例外処理またはコンテキストマネージャで処理できます。2番目の方法は、コードをwithブロックに配置することを意味します。

>>> with open('filename.csv', 'w') as my_file:
...     # do something with `my_file`

特別なそのブロック手段を使って.enter（）と.exitを（）メソッドは、さえ例外のケースで呼び出されます。これらのメソッドはリソースを処理する必要があります。
コンテキストマネージャと例外処理を組み合わせることで、特に堅牢な構成を実現できます。
