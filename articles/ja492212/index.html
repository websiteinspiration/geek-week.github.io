最初の部分、我々はロシアで最大の雇用者のためのポータルソリューションを作成する方法についての記事、アーキテクチャは、バックエンドで記述されました。この記事では、フロントエンドに渡します。

の再利用性

は主にVue.jsで記述されており、ポータル全体がポートレットに分割されているため、それぞれが独自のサイド（Vuex）、ルート（Vueルーター）、およびコンポーネントを持つ個別のVueインスタンスです。そのようなインスタンスはそれぞれ独自のリポジトリに移動されます。

ポートレットはリポジトリにあるため、異なるポートレットに対して同じタイプのコードを大量に記述しないようにする必要があるという疑問が生じます。この問題を解決するために、再利用可能なすべてのものを別のリポジトリに転送し、.gitmodulesを介して接続します。現在、このようなサブモジュールは2つあります。

1つは共通の機能を格納します。これらは共通のコンポーネント、サービス、定数などです。 Vue-commonと呼ばれるこのモジュールがあります。

2番目のサブモジュールには、アセンブリの設定が含まれ、webpackの構成、およびアセンブリ中に必要なローダーとヘルパーが格納されます。このモジュールはVue-bundlerと呼ばれます。

APIでの作業の便宜上、RESTメソッドも一般的なものとローカルなものに分けられました。 Vue-commonでは、ポータルユーザーのリスト、管理方法、ポータルファイルシステムへのアクセスなどを取得するためのメソッドが導入されました。すべてのAPIエンドポイントは、エントリポイントに登録され、Vueインスタンスに接続されている個別のサービスに移動されました。その後、アプリケーションのどこでも使用できます。

個々のサービスはプラグイン内に登録されています。Vueには、プラグインを接続するための組み込みの使用機能があります。Vueのプラグインについて詳しくは、こちらをご覧ください。

プラグイン自体は次のように初期化されます：

class Api {
    constructor () {

    }
}

export default new Api()

スケーリング

上記のように、ポータルごとに個別のバンドルが収集され、各バンドルには独自のエントリポイントがあります。それらのそれぞれで、コンポーネントとアセットが登録され、ローカル開発用に承認が構成され、プラグインが接続されます。

コンポーネントは、ローカルと一般の両方のアプリケーションごとにグローバルに登録されます。

コンポーネントの登録は次のようになります。

import _ from “lodash”

const requireComponent = require.context('@/components', true, /^[^_].+\.vue$/i)

requireComponent.keys().forEach(filename => {
    const componentConfig = requireComponent(filename)

    // Get PascalCase name of component
    const componentName = _.upperFirst(
        _.camelCase(/\/\w+\.vue/.exec(filename)[0].replace(/^\.\//, '').replace(/\.\w+$/, ''))
    )

    Vue.component(componentName, componentConfig.default || componentConfig)
})

独自の機能を追加するために開発中のポータルが必要になる場合があります。そのため、独自のコンポーネントを作成するか、単に別の方法でコンポーネントを実装する必要があります。 / components-portal / *ポータル名* / *のように、別のフォルダーにコンポーネントを作成するだけで十分です。Vueを目的のエントリーポイントに登録し、1つのフォルダーではなく複数のrequire.contextを追加します。

const contexts = [
    require.context('@/components', true, /^[^_].+\.vue$/i),
   require.context('@/components-portal/example', true, /^[^_].+\.vue$/i)
]

contexts.forEach(requireComponent => {
    requireComponent.keys().forEach(filename => {
        const componentConfig = requireComponent(filename)

        // Get PascalCase name of component
        const componentName = _.upperFirst(
            _.camelCase(/\/\w+\.vue/.exec(filename)[0].replace(/^\.\//, '').replace(/\.\w+$/, ''))
        )

        Vue.component(componentName, componentConfig.default || componentConfig)
    })
})

特定のポータルのコンポーネントに、コンポーネントの一般的なライブラリと同じ名前を設定すると、オブジェクトのプロパティとして単に書き換えられ、このポータルのコンポーネントとして使用されます。

svgアイコンなどのアセットもグローバルに登録されます。svg-sprite-loaderを使用してsvgアイコンからスプライトを作成し、<use：xlink：href = "＃* icon name *" />
