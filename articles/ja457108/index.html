WWDC19：XCTestのテスト計画の開始

こんにちは、ハブル！Shashikant Jagtapによる記事「WWDC19：Getting Started with Test Plan for XCTest」の翻訳を紹介します。



前回のWWDCカンファレンスで、Appleは開発者向けのクールな新機能を披露しました。Xcode 11には、素晴らしい機能もいくつか追加されています。それらについてはリリースノートで読むことができます。今年発表された最も重要なツールの1つは、XCTestおよびXcode UIテストのテスト計画です。この記事では、XCTestでテスト計画機能がどのように機能するかを詳しく見ていきます。詳細を知りたい場合は、ビデオセッション「Testing on Xcode」をご覧ください。

Xcodeテスト計画を使用すると、Xcodeスキーマとは関係なくテストスイートを構成できます。これは、テストを次のレベルに進めることができるテストテクノロジの大幅な改善です。次に、実際のXcode Testプランを見てみましょう。

シナリオ


次の要件を持つ機能テストのテスト計画を作成するとします。

  • 機能テストには、煙と回帰テスト用のキットを含める必要があります。
  • 煙のテストはロンドン（英国）で、回帰テストはサンフランシスコ（米国）で実行する必要があります。
  • 煙のテストはアルファベット順に実行し、回帰テストはランダムな順序で実行して、テスト間の問題を検出する必要があります。

さまざまなアップルプラットフォーム用のXcodeのこのテスト計画を作成します。

Xcodeでのテスト計画の作成


Xcode 11では、これらの要件をテスト計画で定義できます。Xcodeを使用すると、現在のスキーマ構成からテスト計画を作成できます。回路を編集して現在のテスト手順に進むと、Xcodeはテスト計画を作成するオプションを表示します。次の設定でテスト計画を作成できます。

  • 空のテスト計画：空のテンプレートを作成し、独自の計画を作成できます。
  • スキームからのテスト計画：回路構成を取り、テスト計画を作成します。
  • テスト計画の選択：すでにテスト計画を作成している場合は、それを選択してチャートに追加できます。

慣れるために、回路構成からテスト計画を作成します。



テスト計画に名前を付け、正しいテストターゲットを指定する必要があります。テスト計画が作成されると、回路に接続されます。ただし、スキーマは、Xcodebuildコマンドが呼び出されたときにフラグを使用した後に作成されたテスト計画を上書きできます。テスト計画を作成した後、それを特定のターゲットに添付できます。最新計画をUIテストターゲットに追加して、名前を付けましょう。



拡張子がfunctional.xctestplanの新しいファイルが見つかります。これがテスト計画です。必要に応じて編集できます。テストのニーズに応じて、同じターゲットで複合テスト計画を作成できます。この場所には、UIテストを備えたターゲット用の手頃な価格のテスト計画があります。

テスト計画ファイル


テスト計画ファイルは、回路とは独立して実行するようにテストを構成する方法に関する情報を含む、JSONに似た単純な構成ファイルです。以下は、テスト計画で使用できるさまざまなテスト構成設定です。基本的なテスト計画には、3つの重要な要素が含まれています。

  • 構成
  • テストターゲット
  • 設定

テスト計画の各構成には、さまざまな設定で関連付けられた一連のテストが含まれています。テスト計画の要件に従って、2つの構成が必要になります。1つはスモークテスト用で、もう1つは回帰テスト用です。
XcodeでFunctional.xctestplanファイルを開き、configuration1の名前をsmoke-ukに変更し、プランの上部にある[+]ボタンをクリックして、別の回帰usa設定を追加します。
これでテスト計画に2つの構成があり、テスト計画はUIテストでターゲットに既に関連付けられています。各構成にいくつかの設定を追加します。 smoke-uk構成を選択すると、場所をロンドンに変更し、その他の設定を変更できます。同様に、regression-usaの設定を変更します。これは、回帰をトリガーするためのテスト計画です。



テスト計画で利用可能な設定：

  • 引数-テストの起動引数と起動環境を上書きできます。
  • ローカリゼーション-さまざまな地理位置情報、ロケール、言語のテストを構成できます。これは、国際化とローカリゼーションをテストするための非常に優れた設定です。
  • UIテスト/アタッチメント/テスト実行/コードカバレッジ-UIテストおよびテストアーティファクトの特定のオプション。アーティファクトを残すか削除するかは、できるだけ早く決定できます。テストを実行する順序とコードカバレッジカウントを選択することもできます。
  • Xcodeコード診断ツール-ランタイムエラーを検索するために、Address SanitizerやThread SanitizerなどのいくつかのXcodeコード診断ツールを使用できます。メモリリークなどを探すのに役立つ他のオプションを使用することもできます。

最初からテスト計画を作成する


Xcodeスキームを使用してテスト計画を作成したくない場合は、Xcode→製品→テスト計画を使用して、常に新しいテスト計画を作成できます。



このようにしてテスト計画を作成したら、それにテストを追加することによって、それをいくつかのテストターゲットにアタッチします。以下の例では、新しいテスト計画を作成し、それにテストを追加しました。



ソースからテスト計画を作成する


前述のように、Xcodeテスト計画は構成ファイルにすぎず、ここで定義したすべての構成をインストールしました。このシナリオでは、functional.xctestplanを作成しました。そのソースコードは次のようになります。

{
  "configurations" : [
    {
      "name" : "smoke-uk",
      "options" : {
        "language" : "en-GB",
        "locationScenario" : {
          "identifier" : "London, England",
          "referenceType" : "built-in"
        },
        "region" : "GB",
        "testExecutionOrdering" : "lexical"
      }
    },
    {
      "name" : "Regression-usa",
      "options" : {
        "addressSanitizer" : {
          "enabled" : true
        },
        "language" : "en",
        "locationScenario" : {
          "identifier" : "San Francisco, CA, USA",
          "referenceType" : "built-in"
        },
        "region" : "US",
        "testExecutionOrdering" : "random"
      }
    }
  ],
  "defaultOptions" : {
    "codeCoverage" : false
  },
  "testTargets" : [
    {
      "target" : {
        "containerPath" : "container:XCTestPlan.xcodeproj",
        "identifier" : "9B0E393A22A6EB5400F666A1",
        "name" : "XCTestPlanUITests"
      }
    }
  ],
  "version" : 1
}

この方法で構成を作成し、Xcode 11で拡張子.xctestplanのファイルに保存できます。

テスト計画を開始


Xcodeでテスト計画ファイルを作成するプロセスを理解しました。Xcodeでテスト計画を実行してレポートを取得する方法を見てみましょう。スキーマで添付のテスト計画を実行することにより、Xcode-> Product-> Testを通常通りテストできます。私たちの場合、これは両方のテスト構成（スモークとリグレッション）を起動します。
Xcodebuildには、テスト計画を表示および実行するためのコマンドラインオプションがいくつかあります。次のコマンドを実行できます。

$ Xcodebuild -showTestPlans -scheme YOUR_SCEME

現在の回路のすべてのテスト計画が表示されます。別のテストを実行することもできます。

$ Xcodebuild -project XCTestPlan.Xcodeproj/ -scheme XCTestPlan -destination 'platform=iOS Simulator,OS=13.0,name=iPhone XR'  test -testPlan functional

このスクリプトは、テスト計画構成を使用してテストを実行します。

テスト計画レポート


特定のテスト計画のテストを実行した後、Xcodeで生成された適切なレポートを確認できます。レポートは、構成ごとに個別に生成されます。



Xcodeのテストナビゲーターセクションでは、テスト計画でレポートをフィルターできます。Xcodeテスト計画レポートは、Xcodeサーバーで構成できます。

Xcodeテスト計画：アプリケーション


Xcodeテスト計画を使用できる場所と、回路の構成に応じてどのように変化するかについて簡単に説明します。Xcodeテスト計画の使用例をいくつか示します。

  • デモに示されているさまざまなタイプのテスト：煙、回帰など。
  • アプリケーションの国際化とローカリゼーションをテストします。たとえば、アプリケーションを複数の言語でテストする場合は、さまざまな構成を作成できます。
  • パフォーマンステストやストレステストなどの重いテストを実行し、機能テストから分離する。コード診断ツールやその他の機能しないテストと一緒に、作業用のテストを構成できます。

Xcodeテスト計画がテストプロセスの拡張に役立つ他の多くのシナリオも可能です。

結論


Xcode Test Planを使用すると、構成を管理してテストを実行する機能により、XCTestを次のレベルに進めることができます。Xcode Test Plansを使用すると、Appleベースのテストプラットフォームをより適切に制御できます。
