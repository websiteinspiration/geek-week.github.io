RTOSに関する真実。記事＃29。Nucleus SEの中断

通常の制御された割り込み


Nucleus SEは、割り込みを処理する2つの方法を提供します。「ネイティブ」または「通常」（ネイティブ）であり、割り込みは特別なものではなく、OSとの対話がある程度制限されています（少なくとも優先度スケジューラを使用している場合）。マネージド（Managed）。割り込みハンドラーから、はるかに多数の利用可能なAPI呼び出しにアクセスできます。

Nucleus SE割り込みハンドラーは、I / Oマクロを使用して、標準モードまたは管理モードで使用できます。

スタッフの割り込み


Nucleus SEスタッフ割り込みは標準の割り込みハンドラであり、「管理されていない」と見なすことができます。これらは通常、中断が高頻度で発生する可能性があり、コンピューティングリソースの使用量が少ない処理が必要な場合に使用されます。最新の組み込みコンパイラの多くは、interruptキーワードを使用した割り込みハンドラの開発をサポートしているため、このようなハンドラはCで記述されている可能性が高いです。コンパイラーが必要と考えるコンテキスト情報のみが保存されます。これにより、標準の割り込みハンドラーが実行できる機能が大幅に制限されます。

Nucleus SEで通常の割り込みハンドラーを作成するには、マクロNUSE_NISR_Enter（）の呼び出しを含む通常の割り込みハンドラーを記述します。最初にNUSE_NISR_Exit（）を呼び出します。これらのマクロは、nuse_typesファイルで定義されています。hそして、グローバル変数NUSE_Task_StateをNUSE_NISR_CONTEXTに設定します。

ガイド付き割り込み


割り込みハンドラー操作にさらに柔軟性が必要な場合は、Nucleus SEが管理する割り込みが解決策になる場合があります。標準の割り込みとの主な違いは、コンテキストの維持です。コンパイラーがスタックにいくつかのレジスターを保存できるようにする代わりに、制御された割り込みは、タスクのコンテキスト全体を（独自のコンテキストブロックに）入力に保存します。次に、現在のタスクのコンテキストが出力のコンテキストブロックから復元されます。これにより、割り込みハンドラーコードによって現在のタスクを変更する機能が提供されます。これは、優先度スケジューラーを使用しているときに可能です。 Nucleus SEでのコンテキストの保存と復元の詳細については、以前の記事（＃10）で説明しています。

明らかに、コンテキストの完全な保持は、標準の割り込み中に発生するスタック上のいくつかのレジスタのストレージと比較して、コンピューティングリソースの使用の増加を伴います。その代償は追加の柔軟性のために支払われる必要があり、これが割り込み処理アプローチの選択が提供される理由です。

管理された割り込みは、nuse_types.hに記述されているNUSE_MANAGED_ISR（）マクロを使用して構築されます。このマクロは、次のアクションを含む関数を作成します。

  • タスクのコンテキストを維持する。
  • 割り当てNUSE_Task_State値NUSE_MISR_CONTEXT ;
  • ユーザー提供の割り込みハンドラー関数コード。
  • NUSE_Task_Stateを以前の状態に復元します。
  • タスクのコンテキストを復元します。

マクロは2つのパラメーターを取ります。生成されたハンドラーの関数名として使用される割り込みの名前と、割り込みハンドラーのユーザーロジックを含む関数の名前です。

割り込みハンドラーからのAPI呼び出し


標準またはマネージの割り込みハンドラーから呼び出すことができるAPI関数のセットは、使用するスケジューラーによって異なります。一般的に、優先順位スケジューラを使用すると、API関数呼び出しを介してスケジューラにアクセスするための多くのオプションが提供されます。これは、標準の割り込みハンドラを使用する場合は困難です。

優先度スケジューラを使用する場合の標準の割り込みハンドラへのAPI呼び出し

リアルタイムクロック（RTC）割り込みハンドラーは、Nucleus SEで唯一の完全な割り込みハンドラーです。Nucleus SEで時間管理に必要なすべての機能を提供するだけでなく、マネージ割り込みハンドラーを作成する例としても機能します。

Nucleus RTOS


Nucleus SE割り込みの実装はNucleus RTOSとは非常に異なるため、この点で互換性を期待するべきではありません。Nucleus RTOSには、標準/低レベル/高レベルの割り込みスキームがあります。これは、Nucleus SEの標準/制御された割り込みスキームに少し似ています。

低レベルおよび高レベルの割り込みハンドラー


低レベル割り込みハンドラー
低レベル割り込みハンドラー（低レベル割り込みサービスルーチン、は、現在のスタックの使用を含め、従来のプロセッサーと同じです。 Nucleus RTOSは、低レベルの割り込みハンドラーが呼び出されるまでコンテキストを維持し、ハンドラーの完了後にコンテキストを復元します。したがって、低レベルの割り込みハンドラーをCで作成して、Cの他のハンドラーを呼び出すことができます。ただし、低レベルのハンドラーで使用できるNucleus RTOSサービスはごくわずかです。割り込み処理で追加のNucleus RTOSサービスが必要な場合は、高レベルの割り込みハンドラーをアクティブにする必要があります。 Nucleus RTOSは、いくつかの低レベルの割り込みハンドラの使用をサポートしています。

高レベル割り込みハンドラー
高レベル割り込みサービスルーチン（HISR）は動的に作成および削除されます。各高レベルプロセッサには、独自のスタックスペースと独自の制御ユニットがあります。メモリはアプリケーションによって割り当てられます。そしてもちろん、低レベルの割り込みハンドラーがアクティブにする前に、高レベルの割り込みハンドラーを作成する必要があります。

高レベルの割り込みハンドラーには独自のスタックと制御ユニットがあるため、現在使用中のNucleus RTOSデータ構造にアクセスしようとすると、一時的に無効にすることができます。

高レベルの割り込みハンドラーには、3つの優先レベルがあります。優先度の低いハンドラーの処理中に優先度の高い上位レベルのハンドラーがアクティブ化されると、タスクの完了時に優先度の低いハンドラーが実行されます。同じ優先度の高レベルの割り込みハンドラは、アクティブにされた順に実行されます。通常モードでタスクのスケジュールを続ける前に、アクティブ化されたすべての高レベル割り込みハンドラーを完了する必要があります。

Nucleus RTOS APIユーティリティの割り込み呼び出し


Nucleus RTOSには、割り込みをサポートするいくつかのAPI呼び出しがあります。Nucleus SEには実装されていません。

標準の割り込みの場合、API呼び出しは次の機能を提供します。

  • 中断の制御（アクティブ化/非アクティブ化）（ローカルおよびグローバル）。
  • 割り込みベクトルの設定。

低レベルの割り込みの場合：

  • カーネルに低レベルの割り込みハンドラを登録します。

高レベルの割り込みの場合：

  • 高レベルの中断の作成/削除;
  • 高レベルの割り込み起動;
  • アプリケーションの（現時点での）高レベルの割り込みの数を取得します。
  • すべての高レベル割り込みの制御ユニットへのポインタを取得します。
  • 現在の高レベル割り込みの制御ユニットへのポインタを取得します。
  • 高レベルの割り込み情報を取得しています。

グローバル割り込み制御

この呼び出しは、タスクに関係なく、割り込みをアクティブまたは非アクティブにします。したがって、この呼び出しによって非アクティブ化された割り込みは、この呼び出しを再利用してアクティブ化されるまでそのままになります。

サービス呼び出しプロトタイプ：

INT NU_Control_Interrupts (INT new_level);

パラメータ：

new_level-システムの新しい割り込みレベル。常に値NU_DISABLE_INTERRUPTS（すべての割り込みを非アクティブ化）およびNU_ENABLE_INTERRUPTS（すべての割り込みをアクティブ化）を取ることができます。アーキテクチャによっては、他の値を使用できる場合があります。

戻り値：

このサービスコールは、アクティブ化された以前のレベルの割り込みを返します。

ローカル割り込み制御

このサービスコールを使用すると、タスクに応じて割り込みをアクティブまたは非アクティブにできます。この呼び出しは、ステータスレジスタを指定された値に変更します。ステータスレジスタは、次にコンテキストが変更されたときに、NU_Control_Interrupts（）への最後の呼び出しで指定された値に戻ります。

サービス呼び出しプロトタイプ：

INT NU_Local_Control_Interrupts (INT new_level);

パラメータ：

new_level-現在のタスクの新しい割り込みレベル。常に値NU_DISABLE_INTERRUPTS（すべての割り込みを非アクティブ化）およびNU_ENABLE_INTERRUPTS（すべての割り込みをアクティブ化）を取ることができます。アーキテクチャによっては、他の値を使用できる場合があります。

戻り値：
このサービスコールは、アクティブ化された以前のレベルの割り込みを返します。

割り込みベクトル Iをインストールすると

、このサービスコールは、割り込みハンドラーによって制御される割り込みベクトルを置き換えます。

サービス呼び出しプロトタイプ：

VOID *NU_Setup_Vector (INT vector, VOID *new);

パラメータ：

vector-割り込みが登録される割り込みベクトル。
newは、ベクター用に作成された割り込みハンドラーです。

低レベル割り込みの登録

このサービスコールは、低レベル割り込みハンドラー関数を割り込みベクトルに関連付けます。システムコンテキストは、指定された低レベルの割り込みハンドラーが呼び出される前に自動的に保存され、割り込みハンドラーの完了後に復元されます。

サービス呼び出しプロトタイプ：

STATUS NU_Register_LISR (INT vector, VOID (*lisr_entry) (INT), VOID (**old_lisr) (INT);

パラメータ：

lisr_entry-ベクトルに登録される関数。値NU_NULLはベクトルをクリアします。
old_lisrは、指定されたベクトルに対して以前に登録された関数です。

NU_SUCCESS-呼び出しは正常に完了しました。
