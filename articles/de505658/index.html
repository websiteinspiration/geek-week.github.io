Erstellen des Effekts eines schnellen Flugs durch den Weltraum in 10 Minuten auf p5.js



Vor kurzem habe ich mich unter der Inspiration von The Coding Train entschieden, an einer der 10-minütigen Herausforderungen teilzunehmen, bei denen ich die Illusion erzeugen musste, mit hoher Geschwindigkeit durch den Weltraum zu fliegen.

Um das Projekt umzusetzen, habe ich das mir bekannte p5.js ausgewählt - eine Bibliothek für JavaScript, mit der Kunst auf algorithmische Weise erstellt werden kann. Warum war es unmöglich, Standardgrafikpakete von Adobe zu verwenden?

Erstens ist es schön, dies mit Code zu tun.
Zweitens ermöglicht der Code im laufenden Betrieb, indem nur wenige Parameter geändert werden, um ein anderes Ergebnis zu erzielen. Die Geschwindigkeit von Experimenten und Iterationen nimmt erheblich zu.
Und drittens lässt sich dieser Code dann einfach in Form eines Skripts in JS in jedes Webprojekt integrieren.

Nun, bis ins Unendliche und darüber hinaus ...


Um ein Projekt zu starten, gehen Sie am einfachsten zum Online-Editor unter editor.p5js.org.

Als erstes sollten wir Sterne erstellen. Es ist klar, dass wir eine große Anzahl von Sternen brauchen, obwohl es technisch nicht schwierig ist, einen Raum im Weltraum zu finden, in dem man mit bloßem Auge überhaupt keine Sterne sehen kann, aber unsere Idee dieses Effekts wurde mithilfe von Science-Fiction-Filmen erstellt. Ich denke, Sie können die Genauigkeit aus Gründen der Effektivität vernachlässigen. Damit. Wir müssen einen zufälligen Sterngenerator machen.

Erstellen Sie zunächst ganz oben ein Array, in dem unsere Sterne gespeichert werden:

let stars = [];


Lassen Sie uns eine neue Funktion namens Star () erstellen, die vier Schlüsselparameter haben sollte: x-, y-Position und z-Variable, über die wir alles andere ändern werden. Um ein zufälliges Auftreten der Anfangspunkte zu erstellen, können wir die Funktion random () verwenden und als Parameter die Grenzen unseres Bildschirms in Breite und Höhe festlegen.

Die Funktion random () arbeitet mit 0, 1 oder 2 Argumenten. Ohne Argumente - gibt eine Zufallszahl von 0 bis 1 zurück, mit eins - von 0 bis zum Argument, mit zwei - vom ersten bis zum zweiten.


Da wir eine „Fabrik“ für die Herstellung von Sternen bauen werden, führen wir sofort das Schlüsselwort this ein, das auf Informationen von dem Objekt zugreift und darauf zugreift, von dem aus wir diese Funktion (oder besser gesagt die Methode) aufrufen.

function Star() {
  this.x = random(-width, width);
  this.y = random(-height, height);
  this.z = width;


Erfahrene Leute haben vielleicht eine Frage - werden wir echtes 3D machen? Tatsächlich ist es jedoch viel einfacher, Pseudo-3D zu erstellen, als bei Spielen der alten Schule. Die Variable z wird hier für andere Zwecke benötigt. Sie wird nur für die Illusion der dritten Dimension verantwortlich sein.

Fertig, obwohl die Sterne noch nicht sichtbar sind, aber wir können sie bereits erstellen. Es ist Zeit, ihnen Bewegung zu geben. Zu diesem Zweck erweitern wir unsere Star () - Funktion um eine neue Methode (verschachtelte Funktion) update (), die für einen endlosen Zyklus von Änderungen verantwortlich ist.

Wir müssen die Bewegungsgeschwindigkeit durch die Sterne bestimmen. Erstellen Sie dazu eine variable Geschwindigkeit und machen Sie sie gleich 25. Die Logik lautet wie folgt: Wir ändern den Parameter z, der sich auf die Parameter x und y auswirkt, so dass wir durch Ändern eines Parameters alles ändern. Sobald dieser Parameter einen Wert von weniger als Eins erreicht, setzen wir die Parameter wieder auf zufällige Anfangswerte zurück und starten einen neuen Bewegungszyklus des Sterns auf dem Bildschirm.

Nun, Sterne entstehen und bewegen sich sogar, es ist Zeit, sie zu sehen. Dazu verwenden wir die Funktion show (), um dadurch einen Aufruf des Objekts zu erstellen und unsere Sterne weiß zu malen.

    fill("white");
}


Jetzt müssen wir eine Form für unseren Stern erstellen, um eine Vielzahl von Flugbahnen und Größen zu erhalten. Wir werden dies durch dieselbe Variable z tun, die anfänglich gleich der Breite des Bildschirms (Breite) ist, aber mit jedem Bild um 25 abnimmt, bis sie kleiner als 1 wird, wonach sich der Zyklus wiederholt.

Um eine schöne Bewegung der Sterne von der Mitte zum Rand zu erhalten, wollen wir sehen, wie sich die Position ändern sollte, nämlich die Variablen x und y. Jeder Stern sollte auf einem imaginären Strahl erscheinen und entlang des Bildschirms verlaufen.

Stellen Sie sich vor, der Startpunkt ist die Koordinate 200.200. Wir müssen dafür sorgen, dass sich beide Indikatoren allmählich nach unten ändern

Da z immer gleich der Breite des Bildschirms ist (z. B. 600) und x und y zwischen -600 und 600 liegen, sind die Variablen x und y am Anfang immer kleiner oder gleich z. Dies legt nahe, dass wir die Werte in einfach zu verwaltende Werte von 0 bis 1 übersetzen können, indem wir einfach x oder y durch z teilen.

Als nächstes können wir eine Standardzuordnungsfunktion erstellen, um Zahlen von 0 bis 1 proportional zur Breite oder Höhe des Bildschirms gleichzusetzen. Und genau diese Werte werden bereits als Parameter an unseren Ballstern übergeben: ellipse ()

Stellen Sie sich vor, dass Breite = 600 unser Kosmos ist. x = -300, y = -300. In diesem Fall ist der Startpunkt -300/600 = -0,5, nach 4 Bildern, wenn z um 4 · 25 = 100 abnimmt, ist die Koordinate bereits -300 / 500 = -0,6, d. H. Die Koordinaten sind LINEAR von der Mitte des Bildschirms entfernt (Koordinaten 0,0) nach oben und links, bis z kleiner als 1 wird.

Wenn sie positiv sind, geschieht alles genau gleich, nur die Bewegung erfolgt nach unten und rechts.

Falls sie sich unterscheiden, sind die Optionen unten und links (mit positivem y und negativem x) oder oben und rechts (mit negativem y und positivem x).

Die Ellipsenfunktion verwendet vier Parameter: Position entlang x, Position entlang y, Durchmesser entlang x und Durchmesser entlang y, auf deren Grundlage die Ellipse aufgebaut wird.

Nun, es ist Zeit, die Sterne anzuzünden und Raum zu schaffen.

Unter Start und Geschwindigkeit erstellen wir eine Standardfunktion, die die Umgebung erstellt (sie funktioniert einmal) - setup (). Im Inneren erstellen wir unsere Zeichenfläche - createCanvas (600.600) und starten unsere Fabrik zum Erstellen von Sternen durch eine for-Schleife. Zunächst beschränken wir uns auf 800.

function setup() {
  createCanvas(600, 600);
  for (let i = 0; i < 800; i++) {
    stars[i] = new Star();
  }
}


Danach erstellen wir die zweite Standardfunktion - draw (). Der Unterschied besteht darin, dass sie in einer Endlosschleife arbeitet. Wir malen den Hintergrund durch background () in Schwarz, danach zählen wir die Länge des Sternen-Arrays (800) durch die for-Schleife und rufen unter Bezugnahme auf jeden Stern im Array die Funktion (in diesem Fall die Methode) update () und show () auf.

function draw() {
  background("black");
  for (let i = 0; i < stars.length; i++) {
    stars[i].update();
    stars[i].show();
  }
}


Das war's, wir sind geflogen, aber es gibt zwei Probleme. Der erste - da der Referenzpunkt der Koordinaten standardmäßig nicht in der Mitte, sondern in der oberen linken Ecke liegt, sehen wir nur den Teil der Sterne, der keinen anfänglich negativen Koordinatenwert hat. Der Rest bleibt außer Sicht.



Die Lösung dieses Problems ist einfach genug - Sie müssen nur die Mitte durch den Übersetzungsbefehl (Breite / 2, Höhe / 2) verschieben.



Das zweite Problem ist, dass die Sterne ihre Größe nicht ändern, wenn sie sich dem Betrachter nähern. Auf einer unbewussten Ebene haben wir das Gefühl, dass etwas nicht stimmt.

Um dieses Problem zu lösen, kehren wir zu unserer Ellipse zurück, erstellen eine weitere Variable r, die ebenfalls von z abhängt und den dritten und vierten Parameter der Ellipse bestimmt, die für den horizontalen und vertikalen Durchmesser verantwortlich sind. Der endgültige Code lautet wie folgt:

let stars = [];
let speed = 25;

function setup() {
  createCanvas(600, 600);
  for (let i = 0; i < 800; i++) {
    stars[i] = new Star();
  }
}

function draw() {
  background("black");
  translate(width / 2, height / 2);
  for (let i = 0; i < stars.length; i++) {
    stars[i].update();
    stars[i].show();
  }
}

function Star() {
  this.x = random(-width, width);
  this.y = random(-height, height);
  this.z = random(width);
 
  this.update = function() {
    this.z = this.z - speed;
    if (this.z < 1) {
      this.x = random(-width, width);
      this.y = random(-height, height);
      this.z = width;
    }
  };

  this.show = function() {
    fill("white");
    var sx = map(this.x / this.z, 0, 1, 0, width);
    var sy = map(this.y / this.z, 0, 1, 0, height);
    var r = map(this.z, 0, width, 10, 0);
    ellipse(sx, sy, r, r);
  };
}
