ブレーンストーミング：タスクを別の角度から見る方法

転置によるブレーンストーミング


時々私は行き詰まり、別の角度からタスクについて考える方法を探す必要があります。マトリックスまたはテーブルの形式で表示できるタスクがあります。それらの構造は次のようになります。

2 A2 B2 C2 D2 E2
３ A3 B3 C3 D3 E3
4 A4 B4 C4 D4 E4

使用しているセルは、列と行に配置されています。簡単なゲームの例を見てみましょう：

攻撃 守る 特別
戦士 剣 鎧 スラム
魔道士 火の玉 反射する 氷結
泥棒 短剣 かわす 武装解除

行は文字クラスです：戦士、魔道士、泥棒。

列はアクションのタイプです：攻撃、防御、特別なアクション。

マトリックスには、文字の各タイプのアクションの各タイプを処理するためのすべてのコードが含まれています。

コードはどのように見えますか？通常、このような構造は次のようなモジュールに編成されています。

  1. Fighter 剣のストライキを処理するためのコードが含まれ、鎧と特別な強力な打撃によるダメージが軽減されます。
  2. Mage 火の玉、ダメージの反射、凍結による特殊攻撃を処理するためのコードが含まれます。
  3. Thief 短剣攻撃を処理するコードを含み、ダッジダメージと特別な武装解除攻撃を回避します。

行列を転置すると便利な場合があります。別の軸に配置できます：

分解ブレーンストーミング


分解と呼ばれる別の手法を使用します。

代数では、分解操作は5x²+ 8x-21の形式の多項式を（x + 3）・（5x-7）に変換します。方程式5x²+ 8x-21 = 0を解くには、まず（x + 3）・（5x-7）= 0に展開します。次に、x + 3 = 0 または 5x-7 = 0 と言えます。複雑なタスクをいくつかの簡単なタスクに変えます。

後方ブレーンストーミング


前の2つのパートでは、マトリックスに整理するために、タスクに時々アプローチする方法について説明しました。時々これは役に立たない場合があり、それから私はタスクを反対方向に見ようとします。たとえば、手続き型マップの生成を見てみましょう。多くの場合、私はノイズ関数から始め、次にオクターブを追加し、パラメーターを調整し、レイヤーを追加します。これは、特定の特性を持つカードが必要なためです。


パラメータを使った実験から始めることは十分に可能ですが、パラメータのスペースは非常に大きく、自分の要件に最も適したパラメータを見つけることができるかどうかはわかりません。したがって、少し実験した後、反対の順序で考えを止めて始めます。必要なものを説明できれば、これはパラメーターを見つけるのに役立ちます。

この動機で代数学を勉強しました。5x²+ 8x-21 = 0の形式の方程式がある場合、xは何になりますか？私が代数を知らなかったとき、私はこの方程式を解き、xの異なる値を代入しようとしました、最初にランダムにそれらを選択し、ソリューションに近づいたと感じたときに調整します。代数は別の方向に進むためのツールを提供します。答えを推測する代わりに、彼女は私に装置（分解、または2次方程式、または根の反復探索のニュートン法）を提供します。これを意識的に使用して、x値（-3または7/5）を検索できます。

単体テストは別の例です。テストする例のリストを作成することをお勧めします。たとえば、六角形のグリッドの場合、条件をチェックする必要があると思うかもしれませんadd(Hex(1, 2), Hex(3, 4)) == Hex(4, 6)。次に、ゼロを確認する必要があることを覚えていますadd(Hex(0, 1), Hex(7, 9)) == Hex(7, 10)。次に、負の値をチェックする必要があると考えることができますadd(Hex(-3, 4) + Hex(7, -8)) == Hex(4, -4)。まあ、素晴らしい、いくつかの単体テストがあります。
