Pythonを使用してぼやけた写真を取り除く方法

一連の大量のショットを撮るとき、それらのいくつかはぼやけています。大手自動車会社も同じ問題に直面しました。車の検査中の写真の一部は不鮮明で、販売に悪影響を及ぼす可能性があります。

低品質の画像は直接利益を減らします。

  • アプリケーションは、アルゴリズムレベルでファジー写真をどのように認識しますか？
  • RGB画像の鮮明度を測定する方法は？



問題の定式化


私は大手自動車会社のアナリストとして働いています。車を検査するとき、会社の検査官は特別なアプリケーションを介して多くの写真を撮り、すぐにデータベースに送信します。一部の画像はぼやけており、販売には不向きです。

ここから問題が発生します。「アルゴリズムレベルでファジー画像を認識する方法は？」

車のさまざまな要素の1200枚の写真のサンプルに基づくアルゴリズムを開発しました。サンプルの特徴は、ラベル付けされていないことです。鮮明な画像とそうでない画像を特定するのは困難です。

「教師とともに」MLモデルを学習することは、ソリューションには適用できないことがわかります。

仕事の過程で私はツールを使用しました：

  • Python。ライブラリ：numpy、matplotlib、cv2;
  • Jupyterノートブック。

この記事では、私が直面した問題の解決策について説明します。

問題を解決するためのアプローチの説明


ステージ1.境界を定義する


クリアと呼べる写真はありますか？
オブジェクトの境界がはっきりしているもの。ぼやけたショットでは、オブジェクトの境界がぼやけます。

画像内のオブジェクトの境界を決定する方法は？

最大の色の違いが見られる境界。

画像の鮮明さを判断するには、まず写真のオブジェクトの境界を判断し、次にそれらのサイズ、厚さ、数などを評価する必要があることがわかりました。

写真は、0〜255の数値の3次元配列で構成されています：（幅、高さ、3色）。
ディープニューラルネットワークの作成と同様にフィルターを適用して境界を定義しました。3次元配列に行列（各色）を乗算します。

    │ 1 -1 │
    │ 1 -1 │

色の違いにより、結果の配列は高いモジュラス数を生成します。
したがって、垂直境界と水平境界を定義します。算術平均は、写真の一般的な境界を示します。

ステージ2.明確にするための境界の分析


境界が定義されます。

ぼやけた画像の境界とクリアの境界を区別する方法は？

さまざまなオプションを検討したところ、次のアプローチが見つかりました。

  1. 元の写真の境界を定義します（ステップ1で説明）。
  2. 元の画像をぼかします。
  3. ぼやけた画像の境界を定義します（ステップ1で説明）。
  4. 段落1と段落2の算術平均の比率を検討します。
  5. 結果の係数は、画像の明瞭さを特徴付けます。

ロジックは単純です。鮮明な写真の場合、境界の変化はファジーの場合よりも顕著に発生するため、係数が高くなります。

アルゴリズムのPython実装


問題を直接解決するために、次のライブラリを使用します。

import numpy as np
import matplotlib.pyplot as plt
import cv2

境界を決定するためのパラメーターとして、行列定義関数を定義します。

def edges(n, orient):
    edges = np.ones((2*n, 2*n, 3))
    
    if orient == 'vert':
        for i in range(0, 2*n):
            edges[i][n: 2*n] *= -1
    elif orient == 'horiz':
        edges[n: 2*n] *= -1
    
    return edges

パラメータnの下で、境界推定に含めるピクセル数を指定します。マトリックスの方向は、水平または垂直です。

その他の機能は、ディープニューラルネットワーク層に似ています。

# Apply one filter defined by parameters W and single slice
def conv_single_step(a_slice_prev, W):
    s = W * a_slice_prev
    Z = np.sum(s)
    Z = np.abs(Z)
    
    return Z
   
# Full edge filter
def conv_forward(A_prev, W, hparameters):
    m = len(A_prev)
    (f, f, n_C) = W.shape
    stride = hparameters['stride']
    pad = hparameters['pad']
    
    Z = list()
    flag = 0
    z_max = hparameters['z_max']
    
    if len(z_max) == 0:
        z_max = list()
        flag = 1
    
    for i in range(m):
        
        (x0, x1, x2) = A_prev[i].shape
        A_prev_pad = A_prev[i][ 
                            int(x0 / 4) : int(x0 * 3 / 4), 
                            int(x1 / 4) : int(x1 * 3 / 4), 
                            :]
        
        (n_H_prev, n_W_prev, n_C_prev) = A_prev_pad.shape
        n_H = int((n_H_prev - f + 2*pad) / stride) + 1
        n_W = int((n_W_prev - f + 2*pad) / stride) + 1
        z = np.zeros((n_H, n_W))
        
        a_prev_pad = A_prev_pad
        
        for h in range(n_H):
            vert_start = h * stride
            vert_end = h * stride + f
            
            for w in range(n_W):
                horiz_start = w * stride
                horiz_end = w * stride + f
                
               
                a_slice_prev = a_prev_pad[vert_start: vert_end, horiz_start: horiz_end, :]

                weights = W[:, :, :]
                z[h, w] = conv_single_step(a_slice_prev, weights)
        
        if flag == 1:
            z_max.append(np.max(z))
        Z.append(z / z_max[i])
        
    cache = (A_prev, W, hparameters)
    
    return Z, z_max, cache

# pooling
def pool_forward(A_prev, hparameters, mode = 'max'):
    m = len(A_prev)
    f = hparameters['f']
    stride = hparameters['stride']
    
    A = list()
    
    for i in range(m):
        (n_H_prev, n_W_prev) = A_prev[i].shape
        
        n_H = int(1 + (n_H_prev - f) / stride)
        n_W = int(1 + (n_W_prev - f) / stride)
        
        a = np.zeros((n_H, n_W))
        
        for h in range(n_H):
            vert_start = h * stride
            vert_end = h * stride + f
            
            for w in range(n_W):
                horiz_start = w * stride
                horiz_end = w * stride + f
                
                a_prev_slice = A_prev[i][vert_start: vert_end, horiz_start: horiz_end]

                if mode == 'max':
                    a[h, w] = np.max(a_prev_slice)
                elif mode == 'avg':
                    a[h, w] = np.mean(a_prev_slice)
                        
        A.append(a)

    cache = (A_prev, hparameters)
    
    return A, cache

conv_single_step-境界を明らかにする行列による画像の色の1つの乗算。
conv_forward-写真全体の境界の完全な定義。
pool_forward-結果の配列のサイズを減らします。

それとは別に、conv_forward関数の行の値に注意します。

分析には、画像全体ではなく中央部分のみを使用します。カメラはより頻繁に中心に焦点を合わせます。画像がはっきりしている場合は、中心がはっきりします。

次の関数は、前の関数を使用して画像内のオブジェクトの境界を決定します。

# main layer
def borders(images, filter_size = 1, stride = 1, pool_stride = 2, pool_size = 2, z_max = []):
    Wv = edges(filter_size, 'vert')
    hparameters = {'pad': pad, 'stride': stride, 'pool_stride': pool_stride, 'f': pool_size, 'z_max': z_max}
    Z, z_max_v, _ = conv_forward(images, Wv, hparameters)
    
    print('edge filter applied')
    
    hparameters_pool = {'stride': pool_stride, 'f': pool_size}
    Av, _ = pool_forward(Z, hparameters_pool, mode = 'max')
    
    print('vertical filter applied')
    
    Wh = edges(filter_size, 'horiz')
    hparameters = {'pad': pad, 'stride': stride, 'pool_stride': pool_stride, 'f': pool_size, 'z_max': z_max}
    Z, z_max_h, _ = conv_forward(images, Wh, hparameters)
    
    print('edge filter applied')
    
    hparameters_pool = {'stride': pool_stride, 'f': pool_size}
    Ah, _ = pool_forward(Z, hparameters_pool, mode = 'max')
    
    print('horizontal filter applied')   
    
    return [(Av[i] + Ah[i]) / 2 for i in range(len(Av))], list(map(np.max, zip(z_max_v, z_max_h)))

関数は垂直境界を決定し、次に水平境界を決定し、両方の配列の算術平均を返します。

そして、定義パラメータを発行するための主な機能：

# calculate borders of original and blurred images
def orig_blur(images, filter_size = 1, stride = 3, pool_stride = 2, pool_size = 2, blur = 57):
    z_max = []

    img, z_max = borders(images, 
                         filter_size = filter_size, 
                         stride = stride, 
                         pool_stride = pool_stride, 
                         pool_size = pool_size
                        )
    print('original image borders is calculated')
    
    blurred_img = [cv2.GaussianBlur(x, (blur, blur), 0) for x in images]
    print('images blurred')
    
    blurred, z_max = borders(blurred_img, 
                             filter_size = filter_size, 
                             stride = stride, 
                             pool_stride = pool_stride, 
                             pool_size = pool_size, 
                             z_max = z_max
                            )
    print('blurred image borders is calculated')

    return [np.mean(orig) / np.mean(blurred) for (orig, blurred) in zip(img, blurred)], img, blurred

最初に、元の画像の境界を決定し、次に画像をぼかし、次にぼやけた写真の境界を決定し、最後に、元の画像とぼかしの算術平均境界の比率を検討します。

この関数は、定義係数のリスト、元の画像の境界の配列、ぼやけた境界の配列を返します。

アプローチ機能


  • 画像が鮮明であるほど、境界線の変化が強くなります。つまり、パラメータが高くなります。
  • 異なるニーズには、異なる明快さが必要です。したがって、自分で明瞭度の境界を決定する必要があります。どこかで、十分な鮮明な写真の係数が7を超え、どこかで10を超える場合があります。
  • 係数は写真の明るさに依存します。暗い写真の境界は弱く変化します。つまり、係数は小さくなります。明快さの境界は、照明、つまり標準的な写真を考慮して決定する必要があることがわかります。
