アバターでのSVGグラフィックの使用



アバターの

SVGグラフィックは、プロフィール写真、ユーザーページの写真、またはグループの写真などのアバター画像に使用されます。

HTMLコードは次のとおりです。

<svg role="none" style="height: 36px; width: 36px;">
  <mask id="avatar">
    <circle cx="18" cy="18" fill="white" r="18"></circle>
  </mask>
  <g mask="url(#avatar)">
    <image x="0" y="0" height="100%" preserveAspectRatio="xMidYMid slice" width="100%" xlink:href="avatar.jpg" style="height: 36px; width: 36px;"></image>
    <circle cx="18" cy="18" r="18"></circle>
  </g>
</svg>

これを見たとき、なぜこのアプローチが使われたのか疑問に思いました。私はこの質問にいくつかの答えがあります：

  • アバターの内側のフレームには、半透明の黒色（10％）が必要です。これは、軽いアバターを丸いイメージのように見せるためです。さらに、これらが完全に白い画像であっても。
  • キーワードを使用してHTML要素に<img>内側のシャドウ（box-shadow）を追加することはできませんinset。この問題を解決するためにSVGが使用されます。
  • 画像が丸い形になるように、SVG要素<mask>とが使用され<image>ます。

先ほど述べたように、アバター内のフレームは明るい画像に非常に役立ちます。これは、このアイデアを示すレイアウトです。


アバター

内部フレームは、次のCSSを使用して構成されます。

circle,
rect {
  stroke-width: 2;
  stroke: rgba(0, 0, 0, 0.1);
  fill: none;
}

画像が正方形の場合、図が使用されrectます。

<svg role="none" style="height: 36px; width: 36px;">
  <mask id="avatar">
    <rect cy="18" fill="white" height="36" rx="8" ry="8" width="36" x="0" y="0"></rect>
  </mask>
  <g mask="url(#avatar)">
    <image x="0" y="0" height="100%" preserveAspectRatio="xMidYMid slice" width="100%" xlink:href="avatar.jpg" style="height: 36px; width: 36px;"></image>
    <rect cy="18" fill="white" height="36" rx="8" ry="8" width="36" x="0" y="0"></rect>
  </g>
</svg>

興味深いことに、フィードのメインページでは、内部の半透明フレームを構成するために使用されるタグ<img>と要素を<div>使用してアバターが作成されます。

<div class="avatar-wrapper>
    <img class="avatar" width="40" height="40" src="avatar.jpg" width="40" alt="">
    <div class="avatar-outline"></div>
</div>

このHTMLのスタイルは次のとおりです。

.avatar-wrapper {
    position: relative;
}

.avatar {
    display: block;
    border-radius: 50%;
}

.avatar-outline {
    position: absolute;
    left: 0;
    top: 0;
    width: 100%;
    height: 100%;

}

アバター内のSVGは一部の場所でのみ使用されているため、使用理由はページサイズに関連している<img>と考えられ<div>ます。テープからのアバターでSVGグラフィックが使用された場合、テープをスクロールするときにロードされるデータの量が増加します。

divを使用してマウスオーバーに応答する要素を作成する


通常、要素上でのマウスに伴う効果は、CSSを使用して作成されます。たとえば、特定のボタンの上にマウスを置いたときに、そのボタンを特別なグレーのシェードでペイントする必要がある場合は、次のようにします。

.element:hover {

ただし、Facebookなどの大規模なサイトでは、このアプローチは現実的ではないようです。サイトを探索する過程で、マウスでホバーしたときにのみ表示される要素に注目しました（「ホバー要素」と呼びましょう）。これが主な仕事です。ここに彼のスタイルがあります：

    background-color: var(--hover-overlay);
    transition-property: opacity;
    transition-timing-function: var(--fds-animation-fade-out);
    cursor: pointer;
}

opacityこの要素 のプロパティ値は、JavaScriptによってから0に変更されます1。私はそれを実験し、それが多くのコンポーネントに使用されていることを知りました。以下は、この要素の使用を示す一連のスクリーンショットです。

insetプロパティの使用


プロパティ値の省略されたレコードについて話しているので、要素の上部、右、下部、および左の部分に影響を与えます。このプロパティinsetは次のように使用できます。

.element {
    inset: 4px 0;
    /*   : top: 4px, bottom: 4px, left: 0, right: 0 */
}

プロパティはinset、他のいくつかの要素に関連付けられた上記のホバー要素に対して構成されます。スタイルはHTMLに埋め込まれます。以下のコンポーネントへの適用に気づきました。


inset

プロパティの影響を受ける領域は青で強調表示されます。このマテリアルが記述されている場合、プロパティinset は Firefox 66以降のみをサポートしています。

Dir = "auto"属性とCSSブールプロパティ 


Facebookなどの多言語サイトでは、コンテンツがどうなるかを正確に予測することが難しい場合があります。たとえば、コンポーネントのユーザー名には属性がありますdir=«auto»。つまり、テキストの方向は言語によって異なります。たとえば、英語を使用する場合、テキストは左から右に表示され、アラビア語を使用する場合は、右から左に表示されます。

また、テキストの向きを変えるスタイルが組み込まれている（属性がdir=«auto»足りない感じがする）ことに注意してください。これは次のようになります。

<div dir="auto" style="text-align: start;">محتوى بالعربية</div>

要素にスタイルが追加されていることに注意してくださいtext-align: start。これは論理的なCSSプロパティを使用します。LTRレイアウトのスタイルはのようになりますtext-align: right。
