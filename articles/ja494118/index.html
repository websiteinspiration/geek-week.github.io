1. APIを構成する


最初にインストールする必要があるのは、Telegram API「python-telegram-bot」のラッパーライブラリです。このための標準コマンドは次のとおりです。

pip install python-telegram-bot --upgrade

次に、次のボットイベントの「ハンドラー」を定義して、小さなプログラムのフレームワークを構築します。

  • エラー-エラー。

次に、ライブラリのドキュメントの例と同様に、これらすべてのハンドラーを割り当ててボットを開始するmain関数を定義します。

def get_answer():
    """Start the bot."""
    # Create the Updater and pass it your bot's token.
    # Make sure to set use_context=True to use the new context based callbacks
    # Post version 12 this will no longer be necessary
    updater = Updater("Token", use_context=True)

    # Get the dispatcher to register handlers
    dp = updater.dispatcher

    # on different commands - answer in Telegram
    dp.add_handler(CommandHandler("start", start))
    dp.add_handler(CommandHandler("help", help))

    # on noncommand i.e message - echo the message on Telegram
    dp.add_handler(MessageHandler(Filters.text, message))

    # log all errors
    dp.add_error_handler(error)

    # Start the Bot
    updater.start_polling()

    # Run the bot until you press Ctrl-C or the process receives SIGINT,
    # SIGTERM or SIGABRT. This should be used most of the time, since
    # start_polling() is non-blocking and will stop the bot gracefully.
    updater.idle()


if __name__ == "__main__":
    get_answer()

ボットを起動する方法には2つのメカニズムがあることに注意してください。

  • 標準ポーリング-新しいイベント用の標準テレグラムAPIツールを使用したボットの定期的なポーリング（updater.start_polling()）;
  • Webhook-ボットからのイベントが到着するエンドポイントでサーバーを起動します。これにはHTTPSが必要です。

すでにお気づきのように、簡単にするために標準のポーリングを使用しています。

3.メッセージを処理して応答を生成します


質問に答える必要がある最初の事柄はナレッジベースです。あなたができる最も簡単なことは、Key-Value値の形式で単純なjsonファイルを作成することです。ここで、Keyは提案された質問のテキストで、Valueは質問への回答です。ナレッジベースの例：

{
  "      ?": "  —  .     -     ,     ,     ,        .    ,      ,   .        ,     .",
  "   ?": "  :\n     \n    \n    \n     \n\n         ,    .",
  "  ?": " :\n- (    , , )\n- (  )",
  }

質問に答えるためのアルゴリズムは次のとおりです。

  1. ユーザーから質問のテキストを取得します。
  2. ユーザーのテキスト内のすべての単語を連語化します。
  3. 結果のテキストとナレッジベースからの見出し語化されたすべての質問を明確に比較しません（レーベンシュタイン距離）。
  4. 知識ベースから最も「類似した」質問を選択します。
  5. 選択した質問への回答をユーザーに送信します。

私たちの計画を実装するには、ライブラリが必要です：fuzzywuzzy（ファジー比較の場合）およびpymorphy2（見出し語化の場合）。

新しいファイルを作成し、サウンドアルゴリズムを実装します。

import json
from fuzzywuzzy import fuzz
import pymorphy2

#   
morph = pymorphy2.MorphAnalyzer()
#  
with open("faq.json") as json_file:
    faq = json.load(json_file)


def classify_question(text):
    #  
    text = ' '.join(morph.parse(word)[0].normal_form for word in text.split())
    questions = list(faq.keys())
    scores = list()
    #      
    for question in questions:
        #    
        norm_question = ' '.join(morph.parse(word)[0].normal_form for word in question.split())
        #       
        scores.append(fuzz.token_sort_ratio(norm_question.lower(), text.lower()))
    # 
    answer = faq[questions[scores.index(max(scores))]]

    return answer

メッセージハンドラを記述する前に、対応の履歴をtsvファイルに保存する関数を記述します。

def dump_data(user, question, answer):
    username = user.username
    full_name = user.full_name
    id = user.id

    str = """{username}\t{full_name}\t{id}\t{question}\t{answer}\n""".format(username=username,
                                                                 full_name=full_name,
                                                                 id=id,
                                                                 question=question,
                                                                 answer=answer)

    with open("/data/dump.tsv", "a") as myfile:
        myfile.write(str)

ここで、メッセージテキストメッセージハンドラーメッセージに記述したメソッドを使用します。

def message(update, context):
    """Answer the user message."""
    # 
    answer = classify_question(update.message.text)
    #  
    dump_data(update.message.from_user, update.message.text, answer)
    # 
    update.message.reply_text(answer)

ほら、テレグラムに行って文章を楽しんでください。



4. Dockerを構成してアプリケーションをデプロイする


古典が言ったように、「実行すると、実行するのは美しくなります。」、すべてを人として行うために、Docker Composeを使用してコンテナー化を構成します。

これには、次のものが必要です。

  1. Dockerfileの作成-コンテナのイメージとエントリポイントを定義します。
  2. docker-compose.ymlを作成-単一のDockerfileを使用して多くのコンテナーを起動します（この場合は必要ありませんが、サービスが多い場合は便利です）。
  3. boot.shを作成します（スクリプトは起動を直接担当します）。

したがって、Dockerfileの内容：

#
FROM python:3.6.6-slim

#  
WORKDIR /home/alex/covid-bot

#  requirements.txt
COPY requirements.txt ./

# Install required libs
RUN pip install --upgrade pip -r requirements.txt; exit 0

#      
COPY data data

#   
COPY app.py faq.json reply_generator.py boot.sh  ./

#   
RUN apt-get clean && rm -rf /var/lib/apt/lists/* /tmp/* /var/tmp/*

# 
RUN chmod +x boot.sh

#  
ENTRYPOINT ["./boot.sh"]

docker-compose.ymlの内容：

# docker-compose
version: '2'
#  
services:
  bot:
    restart: unless-stopped
    image: covid19_rus_bot:latest
    container_name: covid19_rus_bot
    #    boot.sh
    environment:
      - SERVICE_TYPE=covid19_rus_bot
    # volume      
    volumes: 
        - ./data:/data

boot.shの内容：

#!/bin/bash
if [ -n $SERVICE_TYPE ]
then
  if [ $SERVICE_TYPE == "covid19_rus_bot" ]
  then
    exec python app.py
    exit
  fi
else
  echo -e "SERVICE_TYPE not set\n"
fi

これで準備が整いました。これをすべて開始するには、プロジェクトフォルダで次のコマンドを実行する必要があります。

sudo docker build -t covid19_rus_bot:latest .
sudo docker-compose up

以上でボットの準備は完了です。
