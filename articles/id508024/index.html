XGBoost mendistribusikan pembelajaran dan peramalan bersamaan dengan Apache Spark

Halo, Habr! Sudah pada akhir Juli, Otus meluncurkan kursus baru, "Industrial ML on Big Data . " Secara tradisional, untuk mengantisipasi dimulainya kursus baru, kami telah menyiapkan untuk Anda terjemahan materi yang bermanfaat.







Informasi Umum



Dalam meningkatkan (dari ansambel model pembelajaran mesin), algoritma menerapkan proses berurutan (sebagai lawan bagging, di mana ia diparalelkan), yang menghasilkan algoritma pembelajaran yang lemah dan menggabungkannya dengan yang kuat (seperti dalam semua metode ensembel). Dalam meningkatkan pada setiap iterasi proses, model mencoba untuk secara adaptif memperbaiki kesalahan dari iterasi sebelumnya, berbeda dengan mengantongi, di mana algoritma pembelajaran yang lemah dilatih secara independen.







Salah satu algoritma peningkatan, peningkatan gradien, menggunakan gradient descent untuk meminimalkan fungsi kerugian secara langsung dalam model sekuensial ini (tidak seperti algoritma AdaBoost, di mana pelatihan terjadi dengan mengubah bobot instance pelatihan).



Algoritma pembelajaran yang lemah dibuat selama peningkatan gradien selama pelatihan biasanya diimplementasikan sebagai pohon keputusan. Yang paling tidak efektif dalam meningkatkan gradien adalah proses berurutan menciptakan pohon, karena dalam hal ini hanya satu pohon yang dibuat pada satu waktu.



Untuk mengatasi keterbatasan ini, Tianji Chen dan Carlos Gestrin mengusulkan peningkatan algoritma peningkatan gradien yang disebut XGBoost, yang merupakan singkatan dari Extreme Gradient Boosting atau Extreme Gradient Boosting. Ini adalah semacam peningkatan gradien pada steroid, yang digunakan terutama untuk klasifikasi, tetapi juga kadang-kadang untuk regresi dan peringkat.



Dibandingkan dengan peningkatan gradien standar, metode baru ini secara signifikan meningkatkan produktivitas karena hiperparameter, dukungan GPU, validasi silang, dan regularisasi algoritma. Secara umum, model ini lebih efisien, belajar lebih cepat, dan kurang rentan terhadap pelatihan ulang.



Baru-baru ini, XGBoost telah mendapatkan popularitas besar dan memenangkan banyak kompetisi pembelajaran mesin di Kaggle. Diyakini bahwa ia memiliki kekuatan dan akurasi komputasi yang hebat.







XGBoost dan Apache Spark



Selama alur kerja standar dalam ML, sistem seperti Spark digunakan untuk membuat pipa pembelajaran mesin, tempat Anda memproses dan membersihkan data, dan kemudian hasilnya diteruskan ke fase pembelajaran mesin, sering menggunakan Spark MLlib jika Anda sudah menggunakan Spark.



Dalam konteks artikel ini, penting bahwa XGBoost memiliki paralelisasi proses pembangunan pohon, yang memungkinkan pelatihan yang didistribusikan dan peramalan antar node. Artinya, jika saya, sebagai pengguna Apache Spark MLlib, dapat menggunakannya untuk memperluas kemungkinan mempelajari XGBoost dan bekerja di produksi, maka, pada kenyataannya, saya dapat menikmati kinerja tinggi XGBoost dan mekanisme Spark yang kuat untuk fitur-fitur teknik dan membangun saluran pipa ML.



Temui XGBoost4J-Spark, sebuah proyek yang mengintegrasikan XGBoost dan Apache Spark, menambahkan XGBoost ke kerangka kerja Apache Spark MLlib.



XGBoost4J-Spark memungkinkan untuk membangun saluran pipa MLlib yang memproses data sebelum melatih model XGBoost, melatihnya dan dapat digunakan untuk perkiraan paralel pada produksi. Menggunakan perpustakaan ini, setiap pekerja XGBoost berubah menjadi tugas Spark, sedangkan dataset pelatihan dari memori Spark dikirim ke pekerja XGBoost yang secara tidak terlihat ada di seniman Spark.







Untuk mulai menulis aplikasi pembelajaran mesin di XGBoost4J-Spark, Anda harus menambahkan ketergantungan yang sesuai: 



<dependency>
    <groupId>ml.dmlc</groupId>
    <artifactId>xgboost4j-spark</artifactId>
    <version>0.90</version>
</dependency>


Persiapan data (contoh dengan iris)



Seperti disebutkan sebelumnya, XGBoost4J-Spark memungkinkan Anda untuk "menyesuaikan" data di bawah antarmuka XGBoost.



Segera setelah kita mempertimbangkan dataset Iris Flowers di DataFrame, kita perlu:



  • Konversi kolom dari String ke Double;
  • Gabungkan kolom fitur ke dalam vektor sehingga data cocok dengan antarmuka kerangka kerja pembelajaran mesin Spark.


import org.apache.spark.ml.feature.StringIndexer
import org.apache.spark.ml.feature.VectorAssembler
val stringIndexer = new StringIndexer().
  setInputCol("class").
  setOutputCol("classIndex").
  fit(irisDF)
val labelTransformed = stringIndexer.transform(irisDF).drop("class")
val vectorAssembler = new VectorAssembler().
  setInputCols(Array("sepal length", "sepal width", "petal length", "petal width")).
  setOutputCol("features")
val xgbInput = vectorAssembler.transform(labelTransformed).select("features", "classIndex")


Dalam DataFrame di atas, hasilnya akan menjadi dua kolom, "fitur" : vektor yang mewakili tanda-tanda iris dan "classIndex" : label tipe Double . DataFrame seperti itu dapat dengan aman dimasukkan ke mesin pembelajaran XGBoost4J-Spark.



Pelatihan yang didistribusikan



import ml.dmlc.xgboost4j.scala.spark.XGBoostClassifier

val xgbClassifier = new XGBoostClassifier().
      setFeaturesCol("features").
      setLabelCol("classIndex").
      setObjective("multi:softmax")
      setMaxDepth(2).
      setNumClass(3).
      setNumRound(100).
      setNumWorkers(10).


Daftar lengkap opsi XGBoost dapat ditemukan di sini . Perhatikan bahwa di XGBoost4J-Spark Anda juga dapat menggunakan camelase, seperti pada contoh di atas.

Sekarang kita bisa membuat transformator dengan melatih classifier XGBoost pada input DataFrame. Sebagai hasil dari proses pembelajaran, kami mendapatkan model yang dapat digunakan untuk membuat prediksi. 



val xgbClassificationModel = xgbClassifier.fit(xgbInput)


Peramalan paralel



XGBoost4j-Spark mendukung prediksi batch dan prediksi spot.



Untuk peramalan bets, model mengambil DataFrame dengan kolom yang berisi vektor fitur, membuat prakiraan untuk setiap vektor fitur dan menampilkan DataFrame baru dengan hasilnya. Dalam proses ini, XGBoost4J-Spark meluncurkan tugas Spark dengan pekerja XGBoost untuk setiap bagian input DataFrame untuk prediksi batch paralel. 



val predictionsDf = xgbClassificationModel.transform(inputDF)
predictionsDf.show()
