.NET Core 3.1 Preview 2の紹介

.NET Core 3.1 Preview 2の紹介。.NET Core 3.1は、.NET Core 3.0の2つの大きな新しいリリースである、BlazorとWindowsデスクトップの主要な機能強化に焦点を当てた小さなリリースになります。これは、12月2019年の予想最終ロールアウトの日付との長期サポート（LTS）リリースとなります

あなたがすることができ  、.NETのコア3.1のプレビュー2をダウンロードする  Windowsの、MacOSの、およびLinux用。

  • .NET Core 3.1 Preview 2 SDKおよびランタイム
  • Dockerイメージ

ASP.NET Core  とEF Coreも利用できるようになりました。

Visual Studio 16.4 Preview 3およびVisual Studio for Mac 8.4 Preview 3も利用できるようになりました。これらは、.NET Core 3.1 Preview 2を使用するために必要な更新です。VisualStudio 16.4には.NET Core 3.1が含まれているため、単純なVisual Studio更新で両方のエディションがインストールされます。

詳細：

  • .NET Core 3.1リリースノート
  • GitHubリリースノート



機能強化


このリリースでの最大の改善点は、C ++ / CLI（AKA "managed C ++"）のサポートです。C ++ / CLIはWindowsでのみ使用できます。C ++ / CLIの変更は、主にVisual Studioで行われます。C ++ / CLIを使用するには、「C ++でのデスクトップ開発」ワークロードと「C ++ / CLIサポート」コンポーネントを設定する必要があります。この選択されたコンポーネント（最後に表示されたコンポーネント）を下の画像で確認できます。



このコンポーネントは、使用できるいくつかのパターンを追加します。

  • CLRクラスライブラリ（.NET Core）
  • CLR空のプロジェクト（.NET Core）

それらが見つからない場合は、[新しいプロジェクト]ダイアログボックスで見つけてください。

結論


.NET Core 3.1の主な目的は、.NET Core 3.0で導入した機能とスクリプトを改善することです。.NET Core 3.1は  、少なくとも3年間サポートされる長期サポート（LTS）リリースです。

.NET Core 3.1 Preview 2をインストールしてテストし、レビューを残してください。このバージョンは、本番環境での使用は推奨されていません。

突然見逃した場合は、.NET Core 3.0の発表を読んでください  。
