短所.container


コンテナの最も明らかなマイナス点は、マークアップに追加のブロックを作成することです。これにより、要素の入れ子のチェーンが増加し、コードの可読性が低下します。

また、コンテナを作成すると、クラス名に問題が発生する可能性があります（場合によっては、コンテナ内でラップするために追加のクラスを考え出す必要があります）。これはすべてコードを増やすだけです。

上記では、最も単純な例を検討しました。しかし、サイトに15以上のセクションがある場合はどうなりますか？これらは15の追加ブロックです。また、セクションをコンテナ内で2つの部分に分割して、それらをflex要素にする必要がある場合はどうでしょうか。divラッパーを作成する必要があります。これは、コンテナーごとにもう1ブロックです。セクションごとにすでに+2の追加ブロック！上で述べたように、これらのdivはマークアップでネストする「ゴミ」です。しかし、CSSの可能性により、CSSなしで実行できます。

.containerとメディアクエリを操作する


さほど昔のWeb開発ではありませんでしたが、コンテナーを適切に操作する方法をすでに見つけたので、その結果、コンテナーを完全に取り除くことができました。私が理解しているように、.containerはブートストラップで「生まれ」、今では絶対にどこでも使用されています。ブートストラップグリッドに

基づいて、次のコードを使用してCSSメディアクエリでの作業を大幅に簡略化できます： CSS：



.container {
  padding: 0 15px;
}

@media (min-width: 575.98px){
  .container {
    max-width: 540px;
    margin: auto;
    padding: 0;
  }
}

@media (min-width: 767.98px){
  .container {
    max-width: 720px;	
  }
}

@media (min-width: 991.98px) {
  .container {
    max-width: 960px;
  }
}

@media (min-width: 1199.98px){
  .container {
    max-width: 1140px;
  }
}

彼は何をしているの？つまり、コンテンツを常に中央に保ち、さまざまな画面でサイズを変更します。

  1. 最大576pxの画面では、コンテンツは引き伸ばされますが、15pxのサイドマージンは保持されます。
  2. 576pxから768pxの画面では、コンテンツは中央にあり、その幅は540pxです。
  3. 768pxから992pxの画面では、中​​央のコンテンツとその幅は720pxです。
  4. 992pxから1200pxまでの画面では、中​​央のコンテンツとその幅は960pxです。
  5. 1200pxからの画面では、中​​央のコンテンツは幅が1140pxです。

このようなグリッドシステムには明確な利点があります。

  1. 4つの明確なメディアクエリがあります。大画面用、中画面用、小画面用、タブレット用です。
  2. 320pxから576pxまでの小さなギャップが1つだけあります。これは、「手動」のように調整する必要があります-コンテンツの幅を1px減らします。

コンテナでこのメディアクエリの設定を使用すると、非常に便利です。あなたがそれを使用していない場合は、試してみてください、そしてあなたは間違いなく同意します。結局のところ、Twitterはばかげているわけではなく、そのような画面サイズのグリッドを考え出しただけではありません。

最後に、私たちは最も興味深いものになります。では、どのように.containerを置き換えるのですか？まず、何をする必要があるかを考えてみましょう。次の2つが必要です。

  1. そのため、コンテンツは固定幅（1180pxなど）であり、横に散らばらないようにします。
  2. これにより、セクション全体の色または背景を設定できます。

何がコンテンツをセクションの中央にプッシュし、背景はどこにでも残ることができますか？答えは簡単でした。セクションのパディングを設定しました—パディング。しかし、パディングは簡単ではありません。しかし、例として、最初に単純なパディングを設定してみましょう。

1180pxのコンテンツ幅が必要だとすると、1920-1180 = 740/2 = 370px-セクションに横マージンができます。見ます：

HTML：

<section>
  <div class="content">
    content
  </div>
</section>

CSS：

section {
  background-color: pink;
  height: 1000px;
  padding: 0 370px;
}



うん。画面が縮小すると、インデントがコンテンツを圧縮します。良くない。動的にインデントすることは可能ですか？多分！そしてCSSだけのリソース！

これを行うには、calc（）関数を使用します。現在、css関数calc（）は96.5％でサポートされています。これはflexboxよりも1％だけ少ないため、安全に使用できます。動的インデントの場合、1つの数学演算を実行する必要があります。

すぐに例を見てみましょう：



私はそれが好きです！追加のブロックがない場合、このセクションの動作は.containerの場合とまったく同じです。画面が縮小すると、パディングが減少し、コンテンツの幅は固定されたままになります。そして、これはCSSプロパティの1つです：

CSS：

section {
  padding: 0 calc(50% - 590px);
}

@media (min-width: 575.98px){

    padding: 0 calc(50% - 270px);
  }

@media (min-width: 767.98px){

    padding: 0 calc(50% - 360px);
  }
}

@media (min-width: 991.98px) {
  section {
    padding: 0 calc(50% - 480px);
  }
}

@media (min-width: 1199.98px){
  section {
    padding: 0 calc(50% - 590px);
  }
}

結果：



さまざまな画面サイズの合計：

  1. 320pxから576pxまでの自動コンテンツ幅と15pxの固定マージン。
  2. 幅576px〜768px 540pxのダイナミックインデント。
  3. 幅が768ピクセルから992ピクセル、幅が720ピクセルで動的なインデント。
  4. 幅992px〜1200px 960pxのダイナミックインデント。
  5. 幅1200pxから1180pxまで、動的インデント。

これはdiv.containerとまったく同じです！余分なブロックなし。

それで、要約し始めます：

Pros .container：、
  • コンテンツは中央に固定され、希望の幅になります。
  • セクション全体の背景を設定できます。

短所 .container：

  • 追加のdivです。
  • 場合によっては、別のdivを作成する必要があります。
  • コードは肥大化し、読みにくくなっています。
  • 場合によっては、セクションの背景、およびラッパーブロックの他のスタイルを設定する必要があります。
  • ラッパーブロックのクラスを考え出す必要があります。

パディングの長所（.containerと比較して）：

  • コンテンツは中央に固定され、希望の幅になります。
  • セクション全体の背景を設定できます。
  • 追加のブロック、ラッパーはありません。
  • コードはより簡潔で読みやすくなっています。
  • 必要なすべてのスタイルは、セクションにのみ設定されます。

パディングの短所：

  • セクションでpadding-topとpadding-bottomを別々のプロパティとして設定する必要があります。

それは：
section {
  padding: 50px 0;
}

しかし、このように：

section {
  padding-top: 50px;
  padding-bottom: 50px;
}

そうしないと、動的インデントが上書きされます。

推奨事項


動的インデントを使用する方法が好きです。通常の「コンテナ」レイアウトの代わりに試してみてください。コードの純度がどのように向上したかがすぐにわかります。この方法について疑問がある場合は、コメントに記入してください-議論します！

％ではなくvwを使用することをお勧めします。そのため、幅は親からではなく、ブラウザーのプロモーター領域の幅から考慮されます。したがって、より信頼性が高くなります。VWサポート-96％こちら。

CSS：

section {
  padding: 0 calc(50vw - 590px);
}

メディアクエリを使用して、これらのスタイルのテンプレートを作成します。次に例を示します。

section, header, footer {
  padding: 0 15px;
}

  section, header, footer {

結論


動的インデントを使用するメソッドは、div.containerと同じ機能を実行しますが、すべての欠点が完全にありません。おそらくこの方法にはいくつかの落とし穴がありますが、私はまだそれらに会っていません。これまでのところ、すべてがコンテナの場合とまったく同じように機能します。

新しいことを恐れないでください。この方法の使用を開始するだけで、すぐに少し安心します。CSSを最大限に使用してください！
