Pharmakokinetische Modellierung in Julia: Praktische Verwendung von DiffEquations.jl und Optim.jl

Ein pharmakokinetisches Modell ist häufig eine vereinfachte mathematische Beschreibung, wie sich die Konzentration (Menge) einer Testsubstanz in einem biologischen System im Laufe der Zeit ändert. Vereinfacht, weil der Körper ein komplexer Mechanismus ist, bei dem Absorption, Verteilung, Metabolismus und Ausscheidung von Substanzen ( ADME)) können auf verschiedene Weise auftreten, abhängig von der Substanz selbst und vom aktuellen Zustand des Körpers und nicht selten mit nicht offensichtlichen Rückkopplungen. Daher ist es praktisch unmöglich und in den meisten Fällen nicht ratsam, alle möglichen Faktoren zu reflektieren und zu beschreiben, die auf die Kinetik eines Stoffes einwirken. Trotzdem ist eine Darstellung möglich, bei der eine Substanz in begrenzten Bereichen des Körpers verteilt ist, die als Kameras bezeichnet werden, und der Übergang einer Substanz von einer Kammer in eine andere durch eine bestimmte Gleichung beschrieben wird.







Die pharmakokinetische Modellierung kann als Sonderfall der nichtlinearen Modellierung betrachtet werden. Zu diesem Zweck werden häufig kommerzielle Pakete verwendet, z. B. Phoenix WinNonlin, NONMEM, Monolix usw., mit denen Sie Schätzungen von Modellparametern unterschiedlicher (beliebiger) Komplexität erhalten können. Für die Computerumgebung R Project (nlme, nls, saemix) und Julia (CurveFit.jl, LsqFit.jl, NLreg.jl) stehen auch Pakete zur Verfügung, mit denen die Parameter pharmakokinetischer Modelle bestimmt werden können. Die Einschränkungen der Pakete R Project und Julia hängen damit zusammen, dass der Forscher eine nichtlineare Funktion in expliziter Form angeben muss, was nicht immer möglich ist. Darüber hinaus verfügen nicht alle dieser Pakete über ausreichende Funktionen, um ein Modell mit gemischten Effekten (zufällige und feste Effekte) zu bewerten.







Pumas.jl ist ein wenig auseinander, als Trotz der ernsthaft deklarierten Funktionalität hat das Paket (möglicherweise vorerst) keine breite Verbreitung erhalten, wahrscheinlich wird dies (weite Verbreitung) durch eine undeutliche Nutzungslizenz verhindert. Der Zweck dieser Arbeit: eine Beschreibung der wichtigsten Ansätze zur pharmakokinetischen Modellierung, die Konstruktion eines pharmakokinetischen Modells in Form eines Differentialgleichungssystems, die Auswahl von Parametern des pharmakokinetischen Modells.

Mit einer Kombination aus DiffEquations.jl und Optim.jl können Sie ein Modell visuell erstellen und eine Analyse durchführen. Gleichzeitig ist es möglich, sowohl individuelle als auch einfache Populationsmodelle zu erstellen.







Vollständiger Code:







using DifferentialEquations,  Plots, Optim, LossFunctions, Random, Distributions, DiffEqParamEstim

#Function
function pkf!(du, u, p, t)
  Kₐ = p[1]
  K₁₂= p[2]
  K₂₁= p[3]
  Kₑ = p[4]
  Aᵍ = u[1]
  C₁ = u[2]
  C₂ = u[3]
  Vf = p[5]
  du[1] = - Kₐ  * Aᵍ
  du[2] =   Kₐ  * Aᵍ / Vf   - Kₑ * C₁  - K₁₂ * C₁  + K₂₁ * C₂
  du[3] =   K₁₂ * C₁ - K₂₁ * C₂
end

#Start value
u0    = [3.0, 0.0, 0.0]
#Parameters
p     = [0.8, 0.35, 0.20, 0.15, 0.8]
#Time span
tspan = (0.0, 50.0)
#ODE problem
prob  = ODEProblem(pkf!, u0, tspan, p)
#Solution
sol   = solve(prob)
#Plots
#plot(sol)
#plot(sol,vars=(1))
plot(sol,vars=(2))
#plot!(sol,vars=(3))

#x = float.(append!(collect(0.25:0.25:3.75), collect(4:2:40)))
x = [0.25, 0.5, 1.0, 2.0, 2.5, 3.0, 3.5, 4.0, 4.5, 5.0,  8.0, 12.0, 24.0, 36.0, 48.0, 60.0]
y = hcat(sol.(x)...)'[:,2] .* exp.(rand(Normal(), length(x)) ./ 10)
scatter!(x,y)

function loss_function(sol, x, y)
   tot_loss = 0.0
   if any((s.retcode != :Success for s in sol))
     tot_loss = Inf
   else
     tot_loss = sum(value(L2DistLoss(), hcat(sol.(x)...)'[:,2], y))
     #tot_loss = sum(value(LogitDistLoss(), hcat(sol.(x)...)'[:,2], y))
     #tot_loss = sum(value(HuberLoss(mean(y)), hcat(sol.(x)...)'[:,2], y))
   end
   tot_loss
end

cost_function = build_loss_objective(prob, Tsit5(), f ->  loss_function(f, x, y))

result = optimize(cost_function, [0.4, 0.1, 0.1, 0.1, 1.0], NelderMead(), Optim.Options(allow_f_increases = true, iterations = 5000, time_limit = 60.0))

prob   = ODEProblem(pkf!, u0, tspan, result.minimizer)
sol    = solve(prob)
plot!(sol,vars=(2))
