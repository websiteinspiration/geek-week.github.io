ExpressとMySQLで独自のCRUD APIを作成する：パート1

みなさん、こんにちは。「フルスタックJavaScript開発者」コースの開始を見越して、フリーランスのライターが書いた小さな資料を共有したいと思います。



Expressは、ルーティング、ミドルウェイト、テンプレートシステムをサポートする最も人気のあるWebフレームワークの1つです。 ExpressはNode.jsでWebアプリケーションを構築するための事実上のフレームワークであるため、2020年にExpressなしでNode.jsをバックアップするのは、かなり奇妙なアクティビティです。
今日は、MySQLデータベースを使用して簡単なCRUD APIを作成してみましょう。経験豊富なNode.jsリーダーは、Node.jsが通常一緒に使用されるMongoDBの場所を尋ねますが、この記事は、Node.jsに少し触れて、新しいエコシステムへの移行を軽減したいPHP開発者向けです。 MySQLを使用します。単純なTodoアプリケーションのAPIを作成します。誰かが自分のTodoにアタッチしたい場合（Reactで記述されているかどうかは関係ありません）は、少し後ろにアタッチして遊んでください。

このアプリケーションでは、Todoを作成、受信、更新、削除することができます。一般的に、CRUD関数の最小セット（create read update delete）があります。

最後に、アプリケーションとReact上のアプリケーションを組み合わせて、すべてをテストします。

Node.jsアプリケーションの作成

お気に入りのディレクトリに移動し、ターミナルを起動してアプリケーションを作成します。ちなみに、Node.jsが既にインストールされていることは、おそらくタイトルから明らかです。まだダウンロードしていない場合は、ここからのリンクからダウンロードすることはできません。

    $ mkdir CRUD_API
    $ cd CRUD_API

次に、npm initを使用してアプリケーションを初期化する必要があります。なぜなら 私たちのアプリケーションは、テストおよびトレーニング用のものです。細かい部分に時間を費やさないように、すべてのデフォルトデータを入力するコマンドを使用します。

 npm init -y

その場合は、手でpackage.jsonデータを必要なもの（githubリポジトリ、タグ、作成者など）に変更できます。

次に、アプリケーションが機能するために必要なすべてのパッケージを配置する必要があります。

  npm i express mysql body-parser --save


後で開発をgithubに送信する場合は.gitignore、最も重いnode_modulesをドラッグしないように隠しファイルを作成する必要があります。私たちは書く：

    touch .gitignore 
    code .gitignore

そこで入力しますnode_modules。後でこのファイルに、リモートサーバーで後で表示するフォルダとファイルの名前を入力できます。

基本的なExpressセットアップ


プロジェクトのルートフォルダにファイルを作成しますserver.js。

    const express = require("express");
    const bodyParser = require("body-parser");
    
    const app = express();
    
    //     json
    app.use(bodyParser.json());
    
    //    : application/x-www-form-urlencoded
    app.use(bodyParser.urlencoded({ extended: true }));
    
    //   response - request
    app.get("/", (req, res) => {
      res.json({ message: "    " });
    });
    
    //  ,   
    app.listen(3001, () => {
      console.log("   3001 ");
    });

これでサーバーを起動できます：

node server.js

APIを作成するにはExpressが必要です。body -parserパッケージはリクエストを解析しreq.body、ルーティングに役立つものを作成するのに役立ちます。

これで、サーバーは次のことを実行できます。

  • app.use（）を使用して、ボディパーサーをミッドレバーにレンダリングするExpressアプリを作成します
  • アプリケーションをテストするだけの簡単なgetがあります
  • すべての着信変更についてポート3001をリッスンします

最初は、アプリケーションが動作し、データベースを実行できます。

MySQLデータテーブルを作成する

この記事の読者が、MySQLとMySQLWorkBranchを自分でダウンロードしてインストールしても問題ないことを本当に願っています。次に、自分で一番好きな名前のスキーマ（DB）を作成し（私の場合はTODO）、次に稲妻リクエストを選択して次のコマンドをコピー/入力します。

    CREATE TABLE IF NOT EXISTS `todo` (
        id int(11) NOT NULL PRIMARY KEY AUTO_INCREMENT,
        text varchar(255) NOT NULL
      ) ENGINE=InnoDB DEFAULT CHARSET=utf8;

私たちのデータベースでは、すべてが最小限になります。IDとケース自体のテキストのみです。ただし、ケースの緊急度などを記載できる列を追加していただくことで、問題にならないようお願いいたします。

構成とデータベースとの接続

データベースへの接続を確立して構成しましょう。ルートフォルダーに直接新しいアプリフォルダーを作成し、そこにMySQLWorkBranchの設定に応じてほぼ同じ内容のファイルdb.config.jsをフォルダーに作成しますconfig。私の場合、ファイルは次のようになります。

  module.exports = {
    HOST: "localhost",
    USER: "pavel",
    PASSWORD: "",
    DB: "TODO"
  };

その後、アプリ/モードモデルを説明するフォルダーを作成します。ここでモデルを作成します。

  const mysql = require("mysql");
  const dbConfig = require("../config/db.config.js");
  
  //      
  const connection = mysql.createConnection({
    host: dbConfig.HOST,
    user: dbConfig.USER,
    password: dbConfig.PASSWORD,
    database: dbConfig.DB
  });
  
  //      
  connection.connect(err => {
    if (err) throw error;
    console.log("    ");
  });

  module.exports = connection;
