DataGrip 2019.2：接続管理、データ検索、ナビゲーションでのフィルタリング

こんにちは！DataGripで4か月に行ったことについて話します。他のIDEでデータベースサポートを使用している場合は、この投稿も参考になります。



サービスツールボックス


IntelliJプラットフォームIDEには、サービスという新しいツールウィンドウがあります。DataGripでは、接続はここに表示されます。



各ツリーノードは、サービス、データソース、接続のいずれであっても、個別のタブで開くことができます。ツールバーの[ 新しいタブに表示 ]をクリックします。または、ドラッグアンドドロップを使用します。接続で緑の円が点灯している場合は、アクティブであることを意味します。コンテキストメニューから接続を閉じることができます。

クエリ結果は、対応する接続​​に添付されます。「以前と同じように」ビューに戻す場合は、歯車のアイコンをクリックして、表示設定でサービスツリーをオフにします。



リクエストタイマー


「サービス」ウィンドウで、要求にかかる時間を確認する機能が追加されました。クロックは、それが実行されているコンソールの右側に向かって進んでいます。

データ検索


保存場所がわからない場合でも、データを探します。データソース、ソースのグループ、または逆にいくつかのテーブルのみを強調表示し、Ctrl / Cmd + Alt + Shift + Fを押します。または、コンテキストメニューから[ 全文検索 ]を選択します。



ダイアログボックスで、検索対象を入力します。左側はデータソースのリストです。ウィンドウの下部には、DataGripが検索のために実行するクエリが表示されます。

システムオブジェクト


データベースにはシステムディレクトリがあります-メタデータはそこに保存されます：テーブル、プロシージャ、組み込み関数に関する情報。

これらのディレクトリのオブジェクトはクエリでよく使用されるため、オートコンプリートで表示し、正しく強調表示する必要があります。IDEは、これらが不明なオブジェクトであると考えるべきではありません。

以前は、これが機能するためには、システム回路をエクスプローラーに追加する必要がありました。これは、DataGripがイントロスペクトされたオブジェクトのみを認識していたためです。しかし、システム回路は変化せず、毎回それらに関する情報を受け取ることは意味がありません。さらに、それらは常に車掌に必要であるとは限らず、そこでの作業計画を妨害します。

図の前にある稲妻は、エクスプローラに追加していなくても、DataGripが常にそのオブジェクトに関する情報を持っていることを意味します。



ここではいくつかの系統図の例は以下のとおりです。

SQL Serverの：INFORMATION_SCHEMA

DB2：SYSCAT、SYSFUN、SYSIBM、SYSIBMADM、SYSPROC、SYSPUBLIC、SYSSTAT、SYSTOOLS

まだ


  • DataGrip 2019.2は、JetBrains Runtime 11、OpenJDK 11の認定されていないフォークを使用しています。
  • [Cassandra]次のタイプの列を編集できるようになりました：set、list、map、tuple、udt、inet、uuid、timeuuid。
  • IS NULLおよびIS NOT NULLオートコンプリートリストの新しいアイテム。NULLと比較する前に、個別に入力する必要がありました。
  • タブを使用した閉じ括弧/引用の外側へのジャンプの設定は、デフォルトで有効になっています。Tabキーを押すと、カーソルが引用符と括弧のブロックから外側に移動します。
  • 選択範囲を引用符または中括弧で囲んで設定すると、デフォルトでオンになります。選択したコードフラグメントに引用符または角かっこを入力すると、フラグメントは引用符または角かっこ内に表示されます。
  • エイリアスの導入アクションがリファクタリングメニューの一部になりました。
  • PostgreSQL 12との連携を設定します。詳細については、DBE-8384を参照してください。
  • 読み取り専用モードにロジックを追加しました。以前は、この設定には一度に2つのモードが含まれていました。IDEレベル（DataGripはデータを変更するリクエストを分析します）とドライバーレベル（実装されている場合）です。IDEでは、必要に応じて、警告後にリクエストを実行する機能を追加しました。しかし以前は、読み取り専用モードでは、そのような要求がドライバレベルでまだDBE-8145でブロックされていました。今ブロックしません。
