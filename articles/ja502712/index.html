アドレスLEDストリップはどのように機能しますか？


おそらく、この「どのように機能するか」という質問は、多くの人にとってばかげているようです。答えはほぼ明白です。アドレス可能なLEDストリップは、直列に接続された複数の「スマートLED」で構成されています。これは、テープデバイスを見るだけで確認できます。フレキシブルケーブルにはんだ付けされた個々のマイクロサーキットを見ることができ、接続を見ることができます。 1本のワイヤのみがピクセルカラーデータを送信します。どう？スマートLEDとは何ですか？

次に、WS2812Bに基づくLEDストリップで使用されるデータ転送プロトコルについて説明します。さらに、FPGAチップでほぼ独自の「LEDストリップチップ」を作成します。

したがって、テープは単一のデータ信号を介してシリアル伝送を使用します。

ビット0は、短い正のインパルスと、インパルスの約2倍の幅のポーズとして送信されます。ビット単位は、広い正のインパルスと短いポーズとして送信され



ます。50マイクロ秒を超える送信がない場合、テープは元の状態に戻り、最初からピクセルを受け入れる準備ができています。

シーケンスの24ビットごとに、3つのRGBカラーの3バイトです。そして実際には、シーケンスはGRBになります。 G7の上位ビットが最初に来ます。

最初の24ビットのシーケンスは1ピクセルで、テープ内の最初のLEDを受け取ります。最初のLEDが飽和するまで、それは次のLEDにデータを転送しません。最初のLEDが24x RGBビットの部分を受信すると、次のLEDへの送信を開始します。基本的に、一連のLEDは、水で満たされた水差しのカスケードとして連続的に想像でき



ます。

したがって、転送プロトコルは整理されていると思います。

そのような「スマートLED」を自分で設計することは可能ですか？もちろん、これは実用的な意味がほとんどありませんが、独学と視野の拡大にとって、これは興味深い課題です。プログラミング言語Verilog HDLでチップのロジックを記述してみましょう。もちろん、これは実際のチップ設計ではなく、制限があります。最も重要な制限の1つ-マイクロ回路に外部クロックが必要になります。実際のスマートLEDには、そのようなジェネレータも存在しますが、すでにチップに統合されています。

次のようにVerilogモジュールを起動してみましょう。

module WS2812B(
	input wire clk,
	input wire in,
	output wire out,
	output reg r,
	output reg g,
	output reg b
);

clkのクロック周波数、「スマートLED」の入出力信号、そしてもちろん、出力信号r、g、bを介して、実際の外部LEDを赤、緑、青で制御します。

入力信号を2ビットのシフトレジスタにキャプチャし、キャプチャされたこれらのビットの現在の状態から、信号の立ち上がりエッジの開始を次のように判断できます。

reg [1:0]r_in = 0;
always @( posedge clk )
	r_in <= { r_in[0],in };

wire in_pos_edge; assign in_pos_edge = (r_in==2'b01);

さらに、新しい送信を開始する前に制御コントローラーが一時停止したときに、テープのリセットのステータスを確認することが重要です。

localparam reset_level = 3000;
reg [15:0]reset_counter = 0;
always @( posedge clk )
	if( r_in[0] )
		reset_counter <= 0;
	else
	if( reset_counter<reset_level )
		reset_counter <= reset_counter+1;

wire reset; assign reset = (reset_counter==reset_level);

さらに、ポジティブエッジin_pos_edgeから、新しいビットを修正する瞬間を得るために、一時停止に耐える必要があります。

localparam fix_level   = 50;
reg [7:0]bit_length_cnt;
always @( posedge clk )
	if( in_pos_edge )
		bit_length_cnt <= 0;
	else
	if( bit_length_cnt<(fix_level+1) && !pass )
		bit_length_cnt <= bit_length_cnt + 1;

wire bit_fix; assign bit_fix = (bit_length_cnt==fix_level);

チップで既に受信したビット数は、次のように考慮されます。

reg pass = 0;
reg [5:0]bits_captured = 0;

always @( posedge clk )
	if( reset )
		bits_captured <= 1'b0;
	else
	if( ~pass && bit_fix )
		bits_captured <= bits_captured+1'b1;

ここでは、入力ストリームの出力へのリダイレクトを決定する重要なパス信号が導入されています。24xピクセルビットを受け入れた後、パス信号は1に設定されます。

always @( posedge clk )
	if( reset )
		pass <= 1'b0;
	else
	if( bits_captured==23 && bit_fix )
		pass <= 1'b1;
		
reg pass_final;
always @( posedge clk )
	if( reset )
		pass_final <= 1'b0;
	else
	if( r_in!=2'b11 )
		pass_final <= pass;
		
assign out = pass_final ? in : 1'b0;

pass_final信号が1の場合、入力出力は出力出力に多重化されます。

もちろん、受け取ったピクセルの24ビットが累積されるシフトレジスタが必要です。

reg [23:0]shift_rgb;
always @( posedge clk )
	if( bit_fix )
		shift_rgb <= { in, shift_rgb[23:1] };

reg [23:0]fix_rgb;
always @( posedge clk )
	if( bits_captured==23 && bit_fix )
		fix_rgb <= { in, shift_rgb[23:1] };

24ビットすべてを受信すると、最後の24ビットレジスタにもコピーされます。

今問題は小さいままです。受信したRGBバイトに従って実際の外部LEDに輝度を送信するには、PWM（緯度パルス変調）信号を実装する必要があります。

wire [7:0]wgreen; assign wgreen = { fix_rgb[0 ], fix_rgb[1 ], fix_rgb[2 ], fix_rgb[3 ], fix_rgb[4 ], fix_rgb[5 ], fix_rgb[6 ], fix_rgb[7 ] };
wire [7:0]wred;   assign wred   = { fix_rgb[8 ], fix_rgb[9 ], fix_rgb[10], fix_rgb[11], fix_rgb[12], fix_rgb[13], fix_rgb[14], fix_rgb[15] };
wire [7:0]wblue;  assign wblue  = { fix_rgb[16], fix_rgb[17], fix_rgb[18], fix_rgb[19], fix_rgb[20], fix_rgb[21], fix_rgb[22], fix_rgb[23] };

reg [7:0]pwm_cnt;

always @( posedge clk )
begin
	pwm_cnt <= pwm_cnt+1;
	r <= pwm_cnt<wred;
	g <= pwm_cnt<wgreen;
	b <= pwm_cnt<wblue;
end

それだけのようです。
