LiveJournal記事のLDA +視覚化

LDA（ディリクレの潜在的な配置）が「ライブジャーナル」の資料で強調するトピックが興味深いものになると、彼らが言うように、関心があります-問題ありません。

手始めに、指のLDAについて少し説明しますが、数学的な詳細については説明しません（興味のある人は誰でも-読み取り）。したがって、LDA-トピックをモデリングするための最も一般的なアルゴリズムの1つです。各ドキュメント（記事、本、またはその他のテキストデータソース）はトピックの混合であり、各トピックは単語の混合です。


ウィキペディアから取得した画像

したがって、LDAのタスクは、ドキュメントのコレクションからトピックを形成する単語のグループを見つけることです。次に、トピックに基づいて、テキストをクラスター化したり、キーワードを強調表示したりできます。

1800についての記事は、それらのすべてがjsonl形式に変換して、LifeJournalウェブサイトから受信された

stop_words=[]

for i in range(0,len(stop_words)):

次に、すべての単語を通常の形式にします。これには、pimorphを介してインストールできるpymorphy2ライブラリを使用します。

morph = pymorphy2.MorphAnalyzer()
for i in range(0,len(texts)):
    for j in range(0,len(texts[i])):
        texts[i][j] =  morph.parse(texts[i][j])[0].normal_form

はい、単語の形式に関する情報は失われますが、このコンテキストでは、単語の相互の互換性に関心があります。これで前処理が完了しましたが、完全ではありませんが、LDAアルゴリズムがどのように機能するかを確認するには十分です。

さらに、上記のポイントは原則として省略できますが、私の意見では、結果はより適切であり、しきい値はどのくらいになるかは、たとえば、ドキュメントの平均長とその数に依存する関数を作成することができます：

counter = collections.Counter()
for t in texts:
    for r in t:
        counter[r]+=1
limit = len(texts)/5
too_common = [w for w in counter if counter[w] > limit]
too_common=set(too_common)
texts = [[word for word in document if word not in too_common] for document in texts]

モデルのトレーニングに直接進みましょう。これには、たくさんのクールなバンを含むgensimライブラリをインストールする必要があります。まず、すべての単語をエンコードする必要があります。Dictionary関数がそれを実行してから、単語を同等の数値に置き換えます。LDAを呼び出すためのコメントアウトされたオプションは長くなります。これは、各ドキュメントの後に更新されるため、設定を試して適切なオプションを選択できるためです。

texts=preposition_text_for_lda(my_r)
dictionary = gensim.corpora.Dictionary(texts)
corpus = [dictionary.doc2bow(text) for text in texts]
#lda = gensim.models.ldamodel.LdaModel(corpus=corpus, id2word=dictionary, num_topics=10, update_every=1, chunksize=1, passes=1)
lda = gensim.models.ldamodel.LdaModel(corpus=corpus, id2word=dictionary, num_topics=10)

プログラムの作業後、コマンドを使用してトピックを表示できます

lda.show_topic(i,topn=30)

ここで、iはトピック番号、topnは表示されるトピック内の単語数です。
テーマを視覚化するための小さなボーナスです。これには、wordcloudライブラリをインストールする必要があります（同様のユーティリティもmatplotlibにあります）。このコードはテーマを視覚化し、現在のフォルダーに保存します。

from wordcloud import WordCloud, STOPWORDS
for i in range(0,10):
    a=lda.show_topic(i,topn=30)
    wordcloud = WordCloud(
                          relative_scaling = 1.0,
                          stopwords = too_common # set or space-separated string
                          ).generate_from_frequencies(dict(a))
