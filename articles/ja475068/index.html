すべてのLANトラフィックを速度制限なしでVPNにラップします


前回の記事では、我々は見て単一のホストからのすべてのインターネットトラフィックを匿名化する方法で。次に、ローカルエリアネットワーク全体をVPNでラップして、セキュリティのレベルを上げましょう。同時に、まだ設定されていないデバイスからインターネットにアクセスする危険を取り除き、プロバイダーのアドレスをこのデバイスに関連付けます。

この目的のために、ルーターで許可されている場合は、ゲートウェイにVPNクライアントを構成するだけで済みます。しかし、このようなソリューションは、インターネットの速度の低下、ルーターの負荷の増加という形で結果をもたらします。さらに、一部のクライアントは、VPNから切断した場合、すべてのトラフィックをメイン接続経由ですぐに送信します。主要なVPNプロバイダーでさえ、サーバーに100％のアップタイムを提供できないことを忘れないでください。

だから私たちの目標は何ですか：

  • すべての発信トラフィックをVPN経由で渡す
  • VPN-


トラフィックを高速で暗号化できる強力なルーターが必要です。彼はVPNゲートウェイとして機能します。このタスクを思いついたAliExpressの素晴らしいミニPCを見つけました：クアッドコアIntel Celeron、AES-CBC、AES-XTS、AES-GCM、AES-ICMのネイティブサポート、および最大4つのRJ-45ポート。そしてデフォルトで、pfSenseがそれらにインストールされました。私たちは彼女と一緒に働きます。

ISPが特別な接続設定を必要とする場合は、さらに2つのルーターを使用して、インターネットとローカルネットワークへのアクセスを共有し、それらの間にVPNゲートウェイを置くことができます。別のケースでは、プロバイダーのワイヤーをVPNゲートウェイに直接接続し、その背後にローカルネットワークを備えたホームルーターを配置できます。pfSenseでのインターネット接続の初期設定は、この記事の範囲を超えています。

カスタマイズ


この記事では、インターネットが最初のポートに接続され、PCまたはホームネットワークが2番目のポートに接続され、VPNを構成する前に、インターネットにアクセスできたと想定しています。

さらなる問題を回避するために、お気に入りのVPNプロバイダーにログインして、pfSenseのセットアップ手順を見つけましょう。プロバイダーがpfSenseでの手動構成の手順を提供していない場合、私のお気に入りのプロバイダーからこれを使用できます：www.expressvpn.com/support/vpn-setup/pfsense-with-expressvpn-openvpn-主なポイントは変更されません。上記の写真付きの記事では、新しく購入したルーターをpfSenseで完全に構成する方法について説明しています。

新しいVPNを設定するための短いチェックリストを次に示します。

  • システム-証明書。マネージャー-CA。VPN CA証明書を追加する
  • システム-証明書。マネージャー-証明書。VPNサーバー証明書を追加する
  • VPN — OpenVPN — Clients. VPN-
  • Interfaces — Assignment.
  • System — Routing. .
  • Firewall — NAT. NAT
  • Firewall — Rules — LAN. Gateway
  • System — Routing. VPN-a Monitor IP, VPN-

VPNは、ステータス-OpenVPNで再起動されます。ステータス-パッケージログ-OpenVPNでログを表示します。

この段階で、VPN経由でインターネットにアクセスできることを停止して確認します。VPNから切断すると、アクセスが完全に消えます。インターネットがない場合は、どこかで間違いがありました。VPNログを確認し、設定をもう一度確認します。VPNの接続を解除した後、トラフィックがメインゲートウェイを通過し始めた場合は、ファイアウォール-ルール-LANで妨害されていることを意味します。

興味深い部分です。プロバイダーが毎秒20メガビットを発行し、その後夜間に発行する場合-この段階では、VPNオームによって完全に閉じられたローカルネットワークをすでに受信しており、可能な最高速度で動作します。しかし、チャンネルの幅が広い場合はどうでしょうか？

スケーラブル


上記の指示に従って、異なるサーバー用にさらに2、3のVPNクライアントをセットアップしました。 CAおよびサーバー証明書を追加する必要はありません。すでに追加されているものを選択します。また、ファイアウォール-ルール-LANのステップは実行しません。後で実行します。必要なクライアント数は、各サーバーを介した速度測定の結果によって経験的に確立されます。

次に、[システム]-[ルーティング]-[ゲートウェイグループ]に移動します。追加をクリックします。グループ名に覚えやすい名前を入力します。



次に、Gateway Priorityテーブルに注意してください。ゲートウェイグループは次のように機能します。レベルごとのフェイルオーバー、レベル内でのバランス。 Tier列は、このゲートウェイが使用されるレベルを示します。最も簡単なオプションは、最初のレベルですべてのアクティブなVPNゲートウェイを指定することです。遅いインターネットのオプションは、2つのクライアントを作成し、それらを第1レベルと第2レベルに配置することですが、この場合はフォールトトレランスのみになります。

以下のトリガーレベルを見つけます。これは、ゲートウェイがグループから一時的に除外される状況です。 Member Down以外のオプションを使用すると、パケットが完全にドロップするよりも少し前に、ゲートウェイへのパケットの送信を停止できます。パケット損失のしきい値を超えたり、高いpingを送信したりします。損失とpingのしきい値は、システム-ルーティング-ゲートウェイで各ゲートウェイに個別に設定されます。

レベルごとにゲートウェイを配置するための便利なオプションを選択したら、[保存]をクリックします。

ゲートウェイの新しいグループにトラフィックを転送する時が来ました。ファイアウォール-ルール-LANに移動し、前に作成したリダイレクトルールを開き、ゲートウェイのリストに移動して、このリストに作成したグループを確認します。それを選択し、ルールを保存して変更を適用します。以上で、すべての新しい接続はグループ内の新しいVPNクライアントを経由します。

テスト時間：api.ipify.meを開き、キャッシュを無効にしてキープアライブを行い、ページを再読み込みします。ネットワーク上の唯一のユーザーである場合は、ページを更新するたびに、自宅のIPアドレスとは異なる新しいIPアドレスが表示されます。同じアドレスが表示される場合は、Ctrl + F5（ポピーではCommand + Shift + R）でページを完全に更新するか、新しいプライベートタブを開きます。それでも問題が解決しない場合は、グループの設定に誤りがあるか、ファイアウォールのルールでゲートウェイを変更していないことを意味します。
