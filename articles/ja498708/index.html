コードからのCoreDataモデル。または「.XCDataModelを使用しない方法」（パート1）

なんらかの理由で、Xcode GUIを使用してcoreDataモデルを作成するプロセスが気に入らない場合、またはアプリケーションで.XCDataModelファイルを削除して少し速く実行したいので、プログラマーに夢中なら、catへようこそ。


簡単な背景


半年ほど前に自分のiOS用アプリケーションを開発してきましたが、最近まで、データストレージの分野で問題なく開発プロセスは順調に進んでいました。


しかし、最近、coreDataモデルは私の観点からみて下品なサイズに成長しました。私は盲目であるため、大量の半ビジュアルデータを扱うのは物理的に難しいため、coreDataに関連するすべてのものをコードに転送することにしました。


coreDataを使用した「非視覚的」な作業に関する資料はほとんどないため、私の結論と開発の半分はPlaygroundsでのクレイジーな実験に基づいており、不足している情報をGoogleにクレイジーに試みています。


一般的に、行こう！


クラスから始めましょう。


coreDataを使用した標準的な作業で、すべてのエンティティとそれらの間の関係をモデルを使用して別のファイルで説明する場合、この場合、最初にそれらのエンティティを説明するクラスが必要になります（操作内容を理解しやすくなり、Xcodeが役立ちます、すでに作成されているクラスの名前を要求します）。


ちなみに、以下では、各クラスの最後に奇妙なポストスクリプト「MO」-管理対象オブジェクトを使用します。これは、他の、おそらく類似したクラスと混同しないでください。ただし、coreDataとは関係ありません。


以下の状況について説明します。


  • 会社があります（CompanyMOクラスで記述）。
  • 従業員がいます（WorkerMOクラスで説明）。
  • 各従業員は1つの会社でのみ働くことができます（またはまったく働くことができません）。
  • 各企業には、任意の数の従業員を含めることができます（従業員をまったく含まないことも含みます）。

@objc(CompanyMO)
public class CompanyMO: NSManagedObject {
@NSManaged public var companyName: String
@NSManaged public var workers: NSSet
}

@objc(WorkerMO)
public class WorkerMO: NSManagedObject {
@NSManaged public var firstname: String
@NSManaged public var lastname: String
@NSManaged public var company: CompanyMO?
}
