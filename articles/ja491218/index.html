PlatformIOでのPVS-Studioの統合


最近、PlatformIO組み込みシステム開発環境で、PVS-Studioのサポートが登場しました。この記事では、例としてオープンプロジェクトを使用して、静的アナライザーでコードをテストする方法を学びます。

PlatformIOとは何ですか？


PlatformIOは、クロスプラットフォームのマイクロコントローラープログラミングツールです。PlatformIOのコアはコマンドラインツールですが、Visual Studio Codeのプラグインとして使用することをお勧めします。最新のチップとそれらに基づくマザーボードが多数サポートされています。適切なアセンブリシステムを自動的にダウンロードすることができ、サイトにはプラグイン電子コンポーネントを管理するためのライブラリの大規模なコレクションがあります。PVS-Studioを含むいくつかの静的コードアナライザーのサポートがあります。

プロジェクトをインポート


実例として、Arduino MegaボードのArduPod ヘキサポッド制御プログラムを見てみましょう。


適切なボードの新しいプロジェクトを作成し、ソースコードをコピーします。


フォルダー/ arduino / AP_Utils / examples /には、ヘキサポッドを構成して実行するためのプログラムの例がいくつかあります。ここでは、servo_test.inoを使用します。原則として、Arduino用のプログラムは、スケッチの形でINO形式で作成されますが、この場合は完全に適しているわけではありません。それから正しい.cppファイルを作成するには、通常、ファイル名拡張子を変更し、#include <Arduino.h>ヘッダーを先頭に追加し、関数とグローバル変数がアクセスする前に宣言されていることを確認します。


アセンブリプロセス中に、必要なサードパーティライブラリの不足に関するエラーが発生する場合があります。ただし、PlatformIOはリポジトリでそれらを見つけるのに役立ちます。

pio lib install "Adafruit PWM Servo Driver Library"

アナライザーの構成とテストの開始


アナライザーを構成するには、platformio.ini構成を次のように編集する必要があります。

[env:megaatmega2560]
platform = atmelavr
board = megaatmega2560
framework = arduino
check_tool = pvs-studio
check_flags =
  pvs-studio:
    --analysis-mode=4 ; General analysis mode. Set to 32 for MISRA
    --exclude-path=/.pio/libdeps ; Ignore dependency libraries

check_tool パラメーターは、使用するコードアナライザーを示し、その構成はcheck_flagsパラメーターで行われます。詳細な手順については、公式ウェブサイトのドキュメントをご覧ください。https：//docs.platformio.org/en/latest/plus/check-tools/pvs-studio.html

最後に、ターミナルでコマンドを使用してプロジェクトチェックを実行できます。最初のテストの前に、環境自体が現在のアナライザー配布をダウンロードします。

pio check

ヘキサポッドプログラムのチェック結果


今回の記事の目的は、PVS-StudioとPlatformIOの統合を示すことであり、アナライザーの診断機能を示すことではありません。ただし、プロジェクトが検証されたら、見つかったいくつかのエラーを検討して、プロジェクトが正常に分析されたことを示します。

V519「-」演算子の左側と右側に同一の部分式があります：pow（t、2）-pow（t、2）。 AP_Utils.cpp 176

pointLeg* AP_Utils::traceLeg(uint8_t leg, float phi, float z,
  int resolution, uint8_t shape) {
  ....
  if(shape == ELLIPTIC) {
    ....
    float v = sqrt(pow(phi - legs[leg].phi, 2) + pow(z - legs[leg].z, 2));
    float u = sqrt(pow(phi - phi0, 2) + pow(z - z0, 2));
    float t = sqrt(pow(phi0 - legs[leg].phi, 2) + pow(z0 - legs[leg].z, 2));
    theta = acos((pow(t, 2) - pow(t, 2) - pow(v, 2))/(-2.0*t*u));
    ....
  }
  ....
}

2つの同一の式が互いに減算されます。この違いの数学的意味が何であるかは不明です。おそらく、プログラマーは単純に式を減らしませんでした。または、タイプミスが許可されている可能性があり、これらのtのいずれかの代わりに別の引数が必要です。

V550奇妙な正確な比較：値！=-1.定義された精度の比較を使用するほうがよいでしょう：fabs（A-B）> Epsilon。AP_Utils.cpp 574

float AP_Utils::sr04_average(uint8_t trig, uint8_t echo,
  int unit, int samples, int time) {
  ....
  float average, pause, value;
  ....
  for(int i=0; i<samples; i++) {
    value = sr04(trig, echo, unit);
    if(value != -1) { // <=
      total += value;
      delay(pause);
    } else {
      i--;
    }
  }
  average = total/samples;
  ....
  return average;
}

警告は、浮動小数点数の比較が不正確であることを示します。有限のビット数で実数を正確に表現することは不可能であるため、小数の差を特定の精度インデックスと比較することにより、小数の同等性を確立する方が安全です。たとえば、次のようなもの：

bool is_equal(double x, double y) {
  return std::fabs(x - y) < 0.001f;
}

非整数の数値を直接比較する唯一の安全なオプションは、定数を変数に割り当ててから、それらの値をこれらの定数と比較することです。この場合、特に-1が割り当てられていない任意の場所の変数値の値。これは、呼び出されたAP_Utils :: sr04メソッドの動作方法であり、チェックされた値を返します。

float AP_Utils::sr04(uint8_t trig, uint8_t echo, int unit) {
  ....
  float duration, distance;
  ....
  duration = pulseIn(echo, HIGH);
  distance = (346.3*duration*0.000001*unit)/2; // <=
  
  if((distance >= 0.02*unit) && (distance <= 4*unit)) {
    ....
    return(distance);
  } else {
    ....
    return 0;
  }
}

ご覧のとおり、値はいくつかの計算の結果として書き込まれています。割り当て-1はどこにも見られませんが、AP_Utils :: sr04は0を返す可能性があり、これは結果との比較が行われないことを示唆しています。
