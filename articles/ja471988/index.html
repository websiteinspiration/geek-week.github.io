ルールに従って（一部の）セマンティクスを追加できる場合は、文法はさらに優れています。特に、私が開発しているPythonアナライザーでは、CPythonの現在のAST実装を使い続けたいので、各代替からASTノードを返す必要があります。

多くの文法は、規則にアクションを追加できる規則を使用します-通常、{中括弧}内のコードのブロック。より正確には、それらは選択肢に結びついています。このブロックのコードは、他のコンパイラと同じ言語で記述されています。たとえば、C言語で記述され、代替要素の要素を参照する機能が追加されています。元のpgenPythonではこの機能を追加しませんでしたが、新しいプロジェクトでは実装したいと考えています。


これが、この一連の投稿の一部として開発している単純化されたパーサージェネレーターでの実行方法です。


アクションの構文は通常、次のとおりです。


rule: item item item { action 1 } | item item { action 2 }

これにより文法が冗長になるため、パーサージェネレーターは通常、複数行のルールを許可します。次に例を示します。


rule: item item item { action 1 }
    | item item { action 2}

これにより、パーサーが少し複雑になりますが、読みやすさのために重要なので、そのようなレコードをサポートします。
