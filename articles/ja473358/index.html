コードアプローチとしてのインフラストラクチャを使用してNexus Sonatypeをインストールして構成する

Sonatype Nexusは、開発者がJava（Maven）依存関係、Docker、Python、Ruby、NPM、Bowerイメージ、RPMパッケージ、gitlfs、Apt、Go、Nugetをプロキシ、保存、管理し、ソフトウェアを配布できる統合プラットフォームです。セキュリティ。


Sonatype Nexusが必要なのはなぜですか？


  • プライベートアーティファクトを保存するため。
  • インターネットからダウンロードされたアーティファクトをキャッシュする場合。

Sonatype Nexusベースパッケージでサポートされるアーティファクト：


  • Java、Maven（jar）
  • Docker
  • Python（pip）
  • ルビー（宝石）
  • NPM
  • バウアー
  • ヤム（rpm）
  • gitlfs
  • 生
  • Apt（deb）
  • 行く
  • ヌゲ

コミュニティがサポートするアーティファクト：


  • 作曲
  • コナン
  • CPAN
  • ELPA
  • 兜
  • P2
  • R

https://github.com/ansible-ThoTeam/nexus3-ossを使用してSonatype Nexusをインストールします


必要条件


  • インターネットでのansibleの使用についてお読みください。
  • pip install ansibleプレイブックが起動するワークステーションにansible をインストールします。
  • プレイブックが起動するワークステーションにgeerlingguy.javaをインストールしますansible-galaxy install geerlingguy.java。
  • プレイブックが起動するワークステーションにgeerlingguy.apacheをインストールしますansible-galaxy install geerlingguy.apache。
  • gitリポジトリのクローンを作成する git clone https://github.com/ansible-ThoTeam/nexus3-oss.git
  • CentOS 7, Ubuntu Xenial (16.04) and Bionic (18.04), Debian Jessie and Stretch
  • jmespath playbook. sudo pip install -r requirements.txt
  • playbook ( ) nexus.yml
  • nexus ansible-playbook -i host nexus.yml
