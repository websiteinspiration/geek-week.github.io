変数を使用してファイルの名前を変更する-Visual Studio Code


私たちは皆IDEに甘やかされています。もっと贅沢したい！ファイル名の変更と変数の複数置換を行うための無料のVisual Studio Code拡張機能の紹介。

リファクタリングのための多くの拡張機能と組み込み機能があります。主な機能は、名前の変更（変数、クラス、メソッド、ファイルなど）です。私たちは皆そのような機会に精通しており、彼らは毎日私たちの気分を高めています！

私たちはルールを愛し、スタイルガイドに従います。内容に基づいてファイルに名前を付けます。私の考えでは、app-component.tsファイルにはAppComponentクラスが存在するはずです。AppComponentクラスの名前をRootComponentに変更し、次のステップはファイルの名前をroot-component.tsに変更することです...たとえば、Angularコンポーネントの場合、途中でさらに3つのファイル、CSSセレクター、インスタンス名の名前を変更する必要があります。これに直面しましたか？最も単純な名前変更操作は、多くの場合、私たちからの信じられないほどの努力が必要です。

そのようなコードを見たことがありますか？

// src/models/animal.ts
class Animal {
  run(): void;
  jump(): void;
  catchMouse(): void;
  meow(): void;
}

時間の経過とともに、クラスが進化することに同意しますか？これはOOPやSOLIDについてではなく（プロジェクト内に他の動物は存在しません）、すばらしいクラスですが、名前は現実を反映していません。このクラスのメソッドは、これが猫であると叫びます！実際、クラスに良い名前を付けるチャンスを与える必要があります。しかし、すべてのインスタンス変数についてはどうでしょう、何百ものconst animal = new Cat（）;？そして、私たちがすでにキーワードanimalを持つモジュール全体を持っている場合。



一般に、適切な命名のトピックは常に関連しています。しかし、まだ自動化されたルーチンはあまりありません。

楽しむために、node.jsでファイルの行を最も簡単に置き換えるだけでなく、ファイル名を変更するための小さなcliを書きました（@ justerest / multi-replace）そして、私はこの松葉杖の使いやすさに非常に驚いています！ 1つのコマンドでAngularコンポーネントのコレクション全体の名前を変更できるため、コンポーネントやモジュールの名前を間違えることを恐れません。



しばらくして、このツールが回路図の優れた代替品として機能することも発見しました。最も類似したファイル（コンポーネント）をコピーして名前を変更するだけです！それがきっかけで、私のお気に入りのVisual Studio Codeエディターのマルチ置換拡張機能を作成するようになりました。複数置換

コードやファイルの種類を分析せずに、プリミティブのString.ptototype.replace（）で実行します。実装は非常にシンプルです-最小限のコード、最大の利益。フォルダ内のファイルとテキストの名前をフォルダとともに変更し、変更されたファイルを新しいフォルダにコピーすることができます。

私は、ハブロフスク市民からの名前の変更に関する批判、提案、または代替ソリューションを聞くように自分自身を促進し、主張したいと思います。

感謝！
