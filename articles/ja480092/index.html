問題の定式化


このタスクでは、特定の数のトラックにインデックスを付け、サンプルを元にオリジナルの楽曲をデータベースで検索する方法を学ぶ必要があります。この場合、サンプルはノイズが多く、不良マイクに録音されている可能性があります。周波数が異なる可能性があります。ほとんどのコードはすでに参加者向けに作成されており、トラックから指紋を削除する指紋機能を実装するだけで済みます。

音を録音する方法


どのトラックもアナログ的な性質を持つ機械的な波であることは明らかです。物理学の波には、周波数と振幅の2つの特性があります。音波に適用される場合、簡単にするために、振幅は音量とピッチの周波数であると想定できますが、実際には、同じ音では人には高音が大きく聞こえます。

つまり、物理学の観点から、各構成は連続関数によって記述されます。つまり、任意の小さな曲の断片には無限の量の情報が含まれます（ただし、これが何らかのポストパンクである場合、おそらくトラックに少しの情報があります）もっと少なく）。このため、アナログ信号を保存することができず、デジタル化を処理する必要があります。アナログ信号をデジタル化する主な方法は、このセクションで説明するパルスコード変調です。PCMは、離散化、量子化、コーディングの3つの段階で構成されます。それぞれに何が起こるかを簡単に分析しましょう。

サンプリング


したがって、時間の振幅関数があります。誰かが質問をしている場合、どこに頻度があるかは、関数グラフの曲線の後ろに隠されています。彼女はまだ見えていませんが、後で引き出します。アナログ信号について話しているので、関数は連続的であり、可能な引数のセット全体（ゼロからトラックの終わりまでの任意の実数）で定義されます。つまり、私たちはいつでも関数の値を知っており、多くの瞬間があります。それほど多くは必要ないことは明らかなので、個別のサブセットを使用します。これを行うには、一定の短い間隔で信号値を保存します。これは耳で違いが聞こえないように十分小さくする必要がありますが、あまり保存しないように十分大きい必要があります。これも望ましくないためです。

実際、デジタル化する場合、設定されるのは間隔ではなく、「サンプリング周波数」と呼ばれる周波数です。タスクに応じて、サンプリング周波数は、電話の8 kHzから業務用オーディオ機器の数千kHzまで可能です。通常、レコーディングスタジオの外で聴く音楽は、44.1 kHzまたは48 kHzの周波数で保存されます。

量子化


離散化のおかげで、連続関数グラフの代わりに一連のポイントが得られましたが、それでも処理できず、サウンドをさらに損なう必要があります。振幅対時間の初期関数は、連続振幅と連続時間を比較しました。時間の経過とともにそれを理解しましたが、現在の値は実際の数値のセット全体に散らばっているため、問題なく保存することができないため、振幅を計算する必要があります。たとえば、その中には、確かに、丸めずに保存できない不合理なものがあります。

量子化は、振幅を事前に選択されたセットからの値に丸めるプロセスです。もちろん、振幅の数は2の累乗にする必要があります。通常のオーディオトラックでは、16ビットの量子化が使用されます。つまり、振幅の数は65 536（2〜16度）になります。プロのサウンドレコーディングはより正確に実行できますが、耳で16ビットの量子化と24ビットの量子化を区別​​できる人はほとんどいません。したがって、2の累乗を取り、一連の整数振幅を取り、それらを量子化レベルと呼びます。次に、信号は65 536レベルを超えて量子化されていると言えます（信頼できるように聞こえますよね？）。各振幅はいずれかのレベルに丸められます。これにより、最終的にその値を16ビットで保存できます。また、そのような録音は、アナログの連続音と区別がつきません。

コーディング


コーディング段階では、前のステップの結果を理解可能な形式で保存します。これより前のすべてのアクションは通常、専用の機器によって実行されますが、振幅が存在するコンピューター上のファイルまたはメモリ内の配列が必要です。したがって、この段階で、機器の信号は数値の配列に変換されます。これをPCM（パルスコード変調）と呼びます。そのインデックスは条件付き時間（サンプリング後の間隔インデックス）であり、その振幅は量子化ステージで整数に丸められて格納されます。

目標は、トラックにマッピングでき、変更の影響を受けず、コンポジションの周波数と振幅を持つハッシュの作成方法を学ぶことです。それらは非常に異なる場合があります。小さなノイズ、すべての周波数のシフト、別の曲の並行再生などです。また、データベースには、互いに区別する必要のある多くの類似したトラックを同時に含めることができることも考慮する必要があります。または、おそらくすべてのトラックが異なり、問題はどちらがより適切であるかを確立することではなく、どれが適切でないかを理解することです。一般に、創造性には一定の範囲があります。

説明されている主流の実装論文。このアルゴリズムを複数の行で実装できることは特に素晴らしいことです。たとえば、元の記事では、周波数を6つの間隔に分割し、それぞれの最大振幅を見つけ、6つすべての平均を取り、平均よりも高いビンを保存しますが、他の多くの実装が可能です。

var freqBins = [...]int16{40, 80, 120, 180, 300}

func getKeyPoints(frame []freq_domain) int {
	highScores := make([]float64, len(freqBins))
	recordPoints := make([]uint, len(freqBins))

	for bin := freqBins[0]; bin < freqBins[len(freqBins)-1]; bin++ {
		magnitude := frame[bin]

		binIdx := 0
		for freqBins[binIdx] < bin {
			binIdx++
		}

		if magnitude > highScores[binIdx] {
			highScores[binIdx] = magnitude
			recordPoints[binIdx] = (uint)(bin)
		}
	}

	return hash(recordPoints)
}

上記の関数は、フィンガープリントアルゴリズムを実装しています。最後に、周波数の配列（またはビン）が `hash（）`関数に渡されます。これにより、いくつかの数値の配列が1つの数値に変換されます。これは適切な方法で行うことができます。md5を使用することもできます（これは悪い考えですが）。
