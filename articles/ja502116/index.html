PVS-Studioバージョン7.07機能の概要


この記事の目的は、PVS-Studioスタティックアナライザーの機能の概要を示すことです。これを行う最も簡単で視覚的な方法は、例を使用してツールの操作を示すことです。Visual Studio用のプラグインを使用したプロジェクトの検証が表示され、Linuxオペレーティングシステムでアナライザーが起動し、分析結果がSonarQubeにインポートされます。

一般情報


PVS-Studioは、C、C ++、C＃、およびJavaで記述された静的コードアナライザーです。PVS-Studioは、コードチェックの広い範囲を実行しますが、タイプミスや失敗したコピー・貼り付けの結果発見で最も強力である（例：1、2、3、4を）。静的分析は、欠陥を検出するための最も古く、最も信頼性の高い方法の1つであるコードレビューを補完するものです。コードの共同レビュー中に一部のエラーを見つけるのは困難です（たとえば、同じタイプミス）。例えば：

int trans_rest(transcoder_settings *trans)
{
  ....
  for(i=0; i<16; i++);
  {
    trans->eq.eq.preamp[i]   = 0.0;
    for(j=0; j<32; j++)
    {
      trans->eq.eq.boost[i][j] = 0.0;
    }
  }
}

気づいていない人のために、最初ののあとのセミコロンに注意してください。または：

private class ObjectArrayComparer : IEqualityComparer<object[]>
{
  public bool Equals(object[] x, object[] y)
  {
    ....
    for (int i = 0; i < x.Length; i++)
    {
      if (!object.Equals(x[0], y[0])) 
      {
        return false;
      }
    }
    return true;
  }
  ....
}

同じ要素は常に比較されます。そのようなエラーの検索に時間を無駄にしないためには、静的アナライザーを導入するだけの価値があります。このリンクからPVS-Studioをダウンロードして試すことができます。

Visual Studioのプラグイン


PVS-Studioには、Visual Studio 2010-2019用のプラグインがあります。テストに直接進む前に、プラグインの機能を示します。「何がすべてなのか」という精神で疑問を投げかけるポイントをいくつか見ていきましょう。



[メッセージの抑制]アイテムに注目します。これにより、抑制されたアナライザの警告を操作するためのウィンドウが開きます。次に、このアイテムについて説明します。実際、PVS-Studioは警告を抑制できるため、大規模なプロジェクトでもアナライザーの使用を簡単に開始できます。これにより、静的アナライザーを開発プロセスにすばやく簡単に組み込むことができます。

プロジェクトが小さい場合、アナライザーの実装は非常に簡単です。ただし、プロジェクトに「履歴あり」の場合、結果の処理とフィルタリングに誰が関与するか、これに費やす必要がある時間など、疑問が潜んでいる可能性があります。私はそれらを払拭しようとします。

この場合のアナライザーの理想的な使い方は次のとおりです。これをプロジェクトで実行すると、一連のアナライザーメッセージが表示されます。次に、警告抑制メカニズムを使用して、これらのメッセージを表示しないようにPVS-Studioに指示します。なぜならプロジェクトは長い間開発されてきたため、アナライザーレポートには、ほぼ間違いなくいくつかの重大な欠陥に関連する警告はほとんどありません。さらに、開発プロセスでは、変更されたコードまたは新しいコードに関連するエラーのみが表示されます。抑制されたこれらの警告は、今やあなたの技術的義務であり、徐々に戻っていくことができます。警告抑制メカニズムについては、ここで詳しく説明します。



次に、「CWEコードを出力ウィンドウに表示する」に移動します。PVS-Studioは、静的アプリケーションセキュリティテスト（静的アプリケーションセキュリティテスト、SAST）のためのツールです。PVS-Studioアラートは、Common Weakness Enumeration（CWE）に従って分類できます。CWEによるPVS-Studioアラートの分類、CWEが必要な理由、それが何であるかなどについては、次のリンクを参照することをお勧めします。

  • Common Weakness Enumerationに基づくPVS-Studio警告分類。
  • PVS-Studio：セキュリティの欠陥を検索します。

「オプション」タブに移りましょう。


「検出可能なエラー（C、C ++）」セクションでは、アナライザーメッセージタイプの表示を制御できます。なんらかの理由でこのプロジェクトに関連しない診断メッセージは、非表示または無効にすることができます。たとえば、MISRA診断は、組み込み開発者以外の誰も関心を寄せそうにありません。したがって、デフォルトでは無効になっています。アナライザーを最初に使用することを決めたユーザーは、MISRA警告の数を怖がらせ、昏迷に陥ることがあります。また、意図的にアナライザーを起動してプロジェクトをMISRA規格に準拠しているかどうかを確認したユーザーは、設定でオンにすることができます。


ここで、パス/マスクを入力して、一部のファイルまたはフォルダーを分析から除外できます。パスによるマスク（Path Mask）またはファイル名によるマスク（FileName Mask）のいずれかを選択できます。サードパーティのライブラリ、自動生成されたファイルなどを分析から除外する場合に必要です。除外マスクを設定すると、それらに対応するファイルからのメッセージがPVS-Studio出力ウィンドウから消え、次のスキャンに含まれなくなります。したがって、マスクを介してファイルとディレクトリを除外すると、プロジェクト全体の全体的な分析時間を大幅に短縮できます。

詳細はこちら。


含まれているテキストに従って、アナライザーの出力をフィルタリングすることもできます。たとえば、アナライザーレポートでは、テキストにmy_super_function関数について言及する警告がありますが、これらの警告はすべてfalseです。次に、このフィールドにmy_super_functionという単語を追加できます。この単語を含むすべての警告は除外されます。まあ、念のため、ここにドキュメントの対応するセクションへのリンクを残しておきます。

注意。 PVS-Studioでは、コードまたはマクロで誤った警告をマークする他の方法があります。これらについては、偽アラートの抑制セクションのドキュメントで説明されています。

設定の概要が完了したら、プロジェクトのチェックに進みましょう。お気づきかもしれませんが、この記事にはC ++プロジェクトに関連する設定がいくつかありましたが、C＃とJavaに固有の設定はありませんでした。 C ++よりもC ++の設定の方が多いため、これらの簡単なレビューを検討することにしました。ただし、ストーリーの変更については、Visual Studioのプラグインを使用してC ++ではなくC＃プロジェクトを確認し、アナライザーレポートがどのように表示されるかを確認してみましょう。

当然のことながら、プロジェクトの最初のチェック後、誤検知の割合を減らすために設定に戻る必要があります。誤検知は避けられませんが、許容値に設定することにより、その割合を減らすことができます。 「EFLコアライブラリの例を使用したPVS-Studioアナライザーの特性、誤検知の10〜15％」の記事を参照してください。

PVS-Studio Java


C ++プロジェクトの設定を確認し、C＃でプロジェクトをテストしましたが、Javaはどういうわけか記事から外れました。これを修正し、PVS-Studio Javaアナライザーの概要を伝える必要があります。アナライザーには、プロジェクトに統合するいくつかの異なる方法があります。ビルドシステムに応じて、MavenまたはGradleのプラグインを使用できます。アナライザーコアを直接使用することもできます。 Gradle、Maven、またはカーネルプラグインを直接使用すると、Sonar QubeまたはCI（Jenkins）を簡単に操作できます。これらのプラグインを有効にしている場合、Javaプロジェクトはビルドプロセス中に静的分析によって直接チェックされ、その結果、アセンブルされたプロジェクトだけでなく、PVS-Studioアナライザーからのログも受け取ります。

PVS-Studio Javaアナライザーは、IntelliJ IDEAのプラグインとしても使用できます。この場合、プロジェクト構造の分析はこのIDEを使用して実行され、プラグインはアナライザーを操作するための便利なグラフィカルインターフェイスを提供します。


PVS-Studio Javaの起動方法 に関するセクションで詳細を読むことをお勧めします。

また、Javaプロジェクトをチェックすることにより、アナライザーメッセージを抑制して、大規模なプロジェクトへの統合をすばやく簡単に行うことができます。

すべてのアナライザーメッセージを非表示にすることもできます。

Gradleでアナライザーの警告を抑制するには、次のコマンドを実行します。

./gradlew pvsSuppress "-Ppvsstudio.report=/path/to/report.json"
  "-Ppvsstudio.output=/path/to/suppress_base.json"

Mavenでアナライザーの警告を抑制するには、次のコマンドを実行します。
mvn pvsstudio:pvsSuppress "-Dpvsstudio.report=/path/to/report.json"
  "-Dpvsstudio.output=/path/to/suppress_base.json"

抑制メカニズムの詳細については、こちらをご覧ください。
