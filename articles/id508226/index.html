Android dan kamera 3D. Pengenalan wajah dengan perlindungan Fraud

Halo! Nama saya Vladimir Shalkov, saya seorang pengembang Android di Surf.



Belum lama ini, kami perlu menerapkan sistem pengenalan wajah di Android dengan perlindungan penipuan. Pada artikel ini, saya akan berbagi aspek implementasi yang paling menarik dengan contoh kode dan tautan. Saya yakin Anda akan menemukan sesuatu yang baru dan menarik untuk diri Anda sendiri, jadi duduk, mulai.







Sistem pengenalan wajah kini menjadi semakin populer: jumlah perangkat dengan fitur buka kunci wajah semakin meningkat, serta jumlah alat untuk pengembang.



Apple menggunakan FaceID dalam produk-produknya , selain itu mereka juga menangani pengembang dan menambahkan API untuk mengakses fungsi ini. FaceID dianggap cukup aman dan dapat digunakan untuk membuka kunci aplikasi perbankan. Android SDK, hingga saat ini, tidak memiliki solusi yang sudah jadi. Meskipun produsen perangkat menambahkan kemampuan untuk membuka kunci perangkat dengan wajah mereka di firmware mereka, pengembang tidak dapat menggunakan fungsionalitas dalam aplikasi, dan keamanan dari metode membuka kunci ini sangat diinginkan.



Baru-baru ini, kelasFingerprintManager , yang digunakan untuk membuka kunci aplikasi menggunakan sidik jari, telah diberi API 28 dan lebih tinggi, dan pengembang didorong untuk menggunakan BiometricPrompt . Kelas ini berisi logika yang berkaitan dengan biometrik, termasuk identifikasi orang. Namun, menggunakannya di setiap smartphone tidak akan berfungsi, karena menurut informasi dari Google , perangkat harus memiliki peringkat keamanan yang tinggi.



Beberapa perangkat tidak memiliki pemindai sidik jari bawaan, mereka menolaknya karena tingkat perlindungan yang tinggi terhadap penipuan dalam pengenalan wajah dan semuanya berkat sensor ToF (Time-of-flight) yang menghadap ke depan . Dengan menggunakannya, Anda dapat membuat peta kedalaman, sehingga meningkatkan ketahanan sistem terhadap keretakan.



Persyaratan



Aplikasi yang kami implementasikan, berdasarkan fungsinya, adalah sistem kontrol akses, di mana seseorang adalah cara mengidentifikasi seseorang. Dengan menggunakan algoritma khusus, afiliasi seseorang dengan orang nyata diperiksa. Anda dapat menambahkan pengguna baru ke database langsung dari perangkat dengan mengambil gambar dan menentukan nama. Jika perlu untuk menentukan keberadaan seseorang dalam database, maka pencarian dilakukan menggunakan foto yang diambil secara real time dari perangkat. Algoritma menentukan kesamaan dengan orang-orang dari database, jika ini ditemukan, informasi tentang orang ini dikeluarkan.



Tujuan utama kami adalah untuk memastikan tingkat keamanan maksimum: perlu untuk meminimalkan kemungkinan melewati sistem pengenalan wajah, misalnya, menggunakan foto yang dibawa ke jendela bidik. Untuk melakukan ini, kami memutuskan untuk menggunakan kamera Intel RealSense 3D (model D435i), yang memiliki sensor ToF bawaan, berkat itu Anda bisa mendapatkan semua data yang diperlukan untuk membangun peta kedalaman.







Sebagai perangkat yang berfungsi, kami perlu menggunakan tablet dengan diagonal layar besar, yang tidak memiliki baterai bawaan dan memerlukan koneksi konstan ke listrik.



Keterbatasan lain yang tidak kalah pentingnya adalah pekerjaan offline. Karena itu, kami tidak dapat menggunakan layanan cloud untuk pengenalan wajah. Selain itu, tidak masuk akal untuk menulis algoritma pengenalan wajah dari awal, dengan mempertimbangkan batasan waktu akun dan biaya tenaga kerja. Muncul pertanyaan: mengapa menciptakan kembali roda, jika sudah ada solusi yang sudah jadi? Berdasarkan hal tersebut di atas, kami memutuskan untuk menggunakan perpustakaan 3DiVi Face SDK .



Mendapatkan Gambar dari Kamera Intel RealSense



Pada tahap pertama implementasi, perlu untuk mendapatkan dua gambar dari kamera 3D: satu warna, yang kedua dengan peta kedalaman. Kemudian mereka akan digunakan oleh perpustakaan Face SDK untuk perhitungan lebih lanjut.



Untuk mulai bekerja dengan kamera Intel RealSense di proyek Android, Anda perlu menambahkan ketergantungan RealSense SDK untuk Android OS : ini adalah pembungkus di atas pustaka C ++ resmi. Dalam sampel resmi Anda dapat menemukan cara menginisialisasi dan menampilkan gambar dari kamera, kami tidak akan berhenti di situ, semuanya cukup sederhana di sana. Mari kita langsung menuju kode akuisisi gambar: 



private val pipeline = Pipeline()
private val streamingHandler = Handler()

private var streamRunnable: Runnable = object : Runnable {
    override fun run() {
        try {
            FrameReleaser().use { fr ->
                val frames = pipeline.waitForFrames(1000).releaseWith(fr)

                val orgFrameSet = frames.releaseWith(fr)
                val processedFrameSet = frames.applyFilter(align).releaseWith(fr)

                val orgFrame: Frame = orgFrameSet.first(StreamType.COLOR, StreamFormat.RGB8).releaseWith(fr)
                //    
                val videoFrame: VideoFrame = orgFrame.`as`(Extension.VIDEO_FRAME)

                val processedDepth: Frame = processedFrameSet.first(StreamType.DEPTH, StreamFormat.Z16).releaseWith(fr)
                //    
                val depthFrame: DepthFrame = processedDepth.`as`(Extension.DEPTH_FRAME)

                upload(orgFrame) //     
            }
            streamingHandler.post(this)
        } catch (e: Exception) {
            Logger.d("Streaming, error: " + e.message)
        }
    }
}
streamingHandler.post(streamRunnable) //


Menggunakan FrameReleaser () kita mendapatkan frame individual dari aliran video yang bertipe Frame. Anda dapat menerapkan berbagai filter ke bingkai melalui applyFilter ().



Untuk mendapatkan bingkai dari format yang diinginkan, bingkai harus dikonversi ke jenis yang sesuai. Dalam kasus kami, yang pertama dengan tipe VideoFrame, yang kedua adalah DepthFrame.



Jika kami ingin menampilkan gambar pada layar perangkat, maka untuk ini ada metode unggah (), dalam parameter yang ditunjukkan jenis bingkai yang ditampilkan di layar, kami memiliki bingkai dari kamera berwarna.



Konversi bingkai menjadi gambar



Langkah selanjutnya adalah mendapatkan gambar dalam format yang kita butuhkan dari VideoFrame dan DepthFrame. Kami akan menggunakan gambar-gambar ini untuk menentukan apakah orang yang ada dalam gambar itu milik orang asli dan menambahkan informasi ke basis data.



Format gambar:



  • Warna, dengan ekstensi .bmp yang diperoleh dari VideoFrame
  • Dengan peta kedalaman, memiliki ekstensi .tiff dan diperoleh dari DepthFrame


Untuk mengimplementasikan konversi, kita membutuhkan pustaka visi komputer sumber terbuka OpenCV. Semua pekerjaan terdiri dalam membentuk objek Mat dan mengubahnya ke format yang diinginkan: 



fun videoFrameToMat(videoFrame: VideoFrame): Mat {
    val colorMat = Mat(videoFrame.height, videoFrame.width, CvType.CV_8UC3)
    val returnBuff = ByteArray(videoFrame.dataSize)
    videoFrame.getData(returnBuff)
    colorMat.put(0, 0, returnBuff)

    val colorMatNew = Mat()
    Imgproc.cvtColor(colorMat, colorMatNew, Imgproc.COLOR_RGB2BGR)
    return colorMatNew
}


Untuk menyimpan gambar warna, perlu untuk membentuk matriks dengan tipe CvType.CV_8UC3, kemudian mengonversinya menjadi BRG, sehingga warna memiliki rona normal.

Menggunakan metode Imgcodecs.imwrite, simpan ke perangkat: 



fun VideoFrame.saveToFile(path: String): Boolean {
    val colorMat = videoFrameToMat(this)
    return Imgcodecs.imwrite(path + COLOR_IMAGE_FORMAT, colorMat)
}


Hal yang sama perlu dilakukan untuk DepthFrame dengan satu-satunya perbedaan bahwa matriks harus dari tipe CvType.CV_16UC1, karena gambar akan dibangun dari bingkai yang berisi data dari sensor kedalaman: 



fun depthFrameToMat(depthFrame: DepthFrame): Mat {
    val depthMat = Mat(depthFrame.height, depthFrame.width, CvType.CV_16UC1)
    val size = (depthMat.total() * depthMat.elemSize()).toInt()
    val returnBuff = ByteArray(size)
    depthFrame.getData(returnBuff)
    val shorts = ShortArray(size / 2)
    ByteBuffer.wrap(returnBuff).order(ByteOrder.LITTLE_ENDIAN).asShortBuffer().get(shorts)
    depthMat.put(0, 0, shorts)
    return depthMat
}


Menyimpan gambar dengan peta kedalaman: 



fun DepthFrame.saveToFile(path: String): Boolean {
    val depthMat = depthFrameToMat(this)
    return Imgcodecs.imwrite(path + DEPTH_IMAGE_FORMAT, depthMat)
}


Bekerja dengan Face SDK



Face SDK memiliki sejumlah besar komponen perangkat lunak, tetapi kebanyakan dari mereka tidak kita perlukan. Perpustakaan, seperti RealSense SDK, ditulis dalam C ++ dan memiliki pembungkus agar nyaman bekerja di bawah Android. Face SDK tidak gratis, tetapi jika Anda seorang pengembang, Anda akan diberikan lisensi tes.



Sebagian besar komponen perpustakaan dikonfigurasikan menggunakan file konfigurasi XML. Bergantung pada konfigurasi, algoritma ini atau itu akan diterapkan.

Untuk memulai, Anda perlu membuat turunan dari kelas FacerecService, ini digunakan untuk menginisialisasi komponen lain, jalur ke DLL, file konfigurasi dan lisensi dilewatkan dalam parameter.



Selanjutnya, menggunakan layanan ini, Anda perlu membuat objek kelas FacerecService.Config dan Capturer: 



private val service: FacerecService = FacerecService.createService(
                dllPath,
                confDirPath,
                onlineLicenseDir
        )
private val confManual: FacerecService.Config = service.Config("manual_capturer.xml")
private val capturerManual: Capturer = service.createCapturer(confManual)


Kelas Capturer digunakan untuk pengenalan wajah. Konfigurasi manual_capturer.xml berarti bahwa kita akan menggunakan algoritma dari pustaka OpenCV - ini adalah detektor wajah frontal Viola-Jones; Fitur Haar digunakan untuk pengenalan. Perpustakaan menyediakan satu set file XML siap pakai dengan konfigurasi yang berbeda dalam hal kualitas pengenalan dan runtime. Metode yang lebih lambat memiliki kinerja pengenalan yang lebih baik. Jika kita perlu mengenali wajah dalam profil, maka kita harus menggunakan file konfigurasi XML lain - common_lprofile_capturer.xml. Ada banyak konfigurasi, Anda dapat membaca lebih lanjut tentang mereka di dokumentasi. Dalam kasus kami, itu perlu untuk menggunakan konfigurasi common_capturer4_singleface.xml - ini adalah konfigurasi dengan ambang batas kualitas yang lebih rendah sehingga tidak lebih dari satu orang akan selalu dikembalikan.



Untuk menemukan wajah dalam gambar, metode capturerSingleFace.capture () digunakan, di mana array byte gambar yang berisi wajah orang tersebut ditransfer: 



fun createRawSample(imagePath: String): RawSample? {
    val imageColorFile = File(imagePath)
    val originalColorByteArray = ImageUtil.readImage(imageColorFile)
    return capturerSingleFace.capture(originalColorByteArray).getOrNull(0)
}


Objek RawSample menyimpan informasi tentang wajah yang ditemukan dan berisi serangkaian metode yang berbeda, misalnya, jika Anda memanggil getLandmarks (), Anda bisa mendapatkan titik antropometrik wajah.



Afiliasi seseorang dengan orang sungguhan



Untuk menentukan apakah orang sungguhan ada di dalam bingkai, dan bukan foto yang terpasang pada kamera pendeteksi wajah, perpustakaan SDK Wajah menyediakan modul DepthLivenessEstimator, itu mengembalikan enum dengan salah satu dari empat nilai:



  • NOT_ENOUGH_DATA - Terlalu banyak nilai yang hilang pada peta kedalaman
  • NYATA - orang yang diamati adalah milik orang yang hidup.
  • PALSU - Orang yang diamati adalah sebuah foto.
  • NOT_COMPUTED - perhitungan gagal


Inisialisasi modul: 



val depthLivenessEstimator: DepthLivenessEstimator = service.createDepthLivenessEstimator(
           "depth_liveness_estimator_cnn.xml"
   )


Penentuan kepemilikan seseorang kepada orang sungguhan: 



fun getLivenessState(
            rgbPath: String,
            depthPath: String
    ): DepthLivenessEstimator.Liveness {
        
val imageColorFile = File(rgbPath + COLOR_IMAGE_FORMAT)
        val originalColorByteArray = readImage(imageColorFile)

        val originalRawSimple = capturerSingleFace.capture(originalColorByteArray).getOrNull(0)
        val originalRawImage = RawImage(
                SCREEN_RESOLUTION_WIDTH,

        )

        val originalDepthPtr = Natives().readDepthMap(depthPath + DEPTH_IMAGE_FORMAT)

//  
        val hFov = 69.4f 
        val vFov = 42.5f 

        val depthMapRaw = DepthMapRaw()

Metode getLivenessState () sebagai parameter mendapatkan tautan ke gambar: warna dan dengan peta kedalaman. Kami membentuk objek RawImage dari yang berwarna. Kelas ini menyediakan data gambar mentah dan informasi pemangkasan opsional. DepthMapRaw dibentuk dari peta kedalaman - peta kedalaman yang terdaftar sesuai dengan gambar warna asli. Ini harus dilakukan untuk menggunakan metode estimLiveness (originalRawSimple, depthMapRaw), yang akan mengembalikan kita enum dengan informasi apakah orang sungguhan ada di dalam frame.



Perlu memperhatikan pembentukan objek DepthMapRaw. Salah satu variabel disebut depth_data_ptr - ini adalah pointer ke data kedalaman, tetapi seperti yang Anda tahu di Jawa tidak ada pointer. Untuk mendapatkan pointer, gunakan fungsi JNI, yang mengambil argumen tautan ke gambar dengan peta kedalaman: 



extern "C" JNIEXPORT jlong JNICALL Java_ru_face_detect_Natives_readDepthMap
(JNIEnv *env, jobject obj, jstring jfilename)
{
    const char * buf = env->GetStringUTFChars(jfilename, NULL);
    std::string filename = buf;
    env->ReleaseStringUTFChars(jfilename, buf);

    cv::Mat depth_map = cv::imread(filename, -1);
    unsigned char * data = new unsigned char[depth_map.rows * depth_map.cols * depth_map.elemSize()];
    memcpy(data, depth_map.data, depth_map.rows * depth_map.cols * depth_map.elemSize());

    return (jlong) data;
}


Untuk memanggil kode yang ditulis dalam C di Kotlin, Anda perlu membuat kelas jenis ini: 



class Natives {

    init {
        System.loadLibrary("native-lib")
    }

    external fun readDepthMap(fileName: String): Long
}


Nama file .cpp dilewatkan ke System.loadLibrary (), di mana metode readDepthMap () terkandung, dalam kasus kami ini adalah native-lib.cpp. Juga diperlukan untuk menginstal pengubah eksternal, yang berarti bahwa metode ini tidak diterapkan di Kotlin.



Identifikasi wajah



Fungsi yang sama pentingnya adalah identifikasi orang yang ditemukan dalam bingkai. Face SDK memungkinkan Anda menerapkan ini menggunakan modul Recognizer. Inisialisasi: 



val recognizer: Recognizer = service.createRecognizer(
        "method8v7_recognizer.xml",
        true,
        true,
        true
)


Kami menggunakan file konfigurasi method8v7_recognizer.xml, yang memiliki tingkat pengakuan tertinggi, tetapi kualitas pengakuan lebih rendah daripada metode 6v7 dan 7v7.



Sebelum mengidentifikasi seseorang, perlu membuat daftar orang, dengan menggunakan yang kami akan menemukan korespondensi sesuai dengan foto sampel. Untuk mengimplementasikan, Anda perlu membuat Vektor dari objek Templat: 



var templates = Vector<Template>()
val rawSample = createRawSample(imageUrl)
val template = recognizer.processing(rawSample)
templates.add(template)


Untuk membuat Templat, digunakan metode acceptizer.processing (); RawSample diteruskan sebagai parameter. Setelah daftar dengan templat wajah dibuat, perlu menambahkannya ke Recognizer dan menyimpan TemplatesIndex yang dihasilkan, yang diperlukan untuk pencarian cepat di database besar: 



val templatesIndex = recognizer.createIndex(templates, SEARCH_THREAD_COUNT)


Pada tahap ini, kami membentuk objek Recognizer yang berisi semua informasi yang diperlukan untuk melakukan identifikasi: 



fun detectFaceSearchResult(rgbPath: String): Recognizer.SearchResult {
    val rawSample = createRawSample(rgbPath + COLOR_IMAGE_FORMAT)
    val template = recognizer.processing(rawSample)

    val searchResult = recognizer.search(
            template,
            templateIndex,
            searchResultCount,
            Recognizer.SearchAccelerationType.SEARCH_ACCELERATION_1
    ).firstElement()
    return searchResult
}


Fungsi recognitionizer.search () akan mengembalikan hasil di mana kita bisa mendapatkan indeks elemen yang ditemukan, mencocokkannya dengan daftar orang dari database dan mengidentifikasi orang tersebut. Selain itu, kita dapat mengetahui nilai kesamaan, bilangan real dari 0 hingga 1. Informasi ini disediakan di kelas Recognizer.MatchResult, variabel lingkup: 



val detectResult = detectFaceSearchResult(rgbPath)
//    -    0  1.
val scoreResult = detectResult.matchResult.score


Kesimpulan



Tidak ada keraguan bahwa di masa depan sistem seperti itu akan digunakan di mana-mana: ketika mendekati pintu masuk, pintu akan otomatis terbuka, dan mesin kopi di kantor akan memilih tingkat penggilingan favorit Anda.



Di Android SDK, API secara bertahap ditambahkan yang memungkinkan pengembang untuk bekerja dengan sistem identifikasi wajah, tetapi sekarang semuanya berada pada tahap awal pengembangan. Dan jika kita berbicara tentang sistem kontrol akses menggunakan tablet Android, perpustakaan Face SDK dan kamera Intel RealSense 3D, saya ingin mencatat lebih banyak fleksibilitas dan ekstensibilitas. Tidak ada pengikatan pada perangkat, kamera dapat dihubungkan ke smartphone modern apa pun. Anda dapat memperluas jangkauan kamera 3D yang didukung, serta menghubungkan beberapa bagian ke satu perangkat. Dimungkinkan untuk mengadaptasi aplikasi tertulis untuk Android Things, dan gunakan di rumah pintar Anda. Jika Anda melihat kemampuan pustaka SDK Wajah, maka dengan bantuannya kami dapat menambahkan identifikasi wajah dalam aliran video berkelanjutan, menentukan jenis kelamin, usia, dan emosi. Kemungkinan-kemungkinan ini memberikan ruang untuk banyak eksperimen. Dan kita dapat mengatakan dari pengalaman kita sendiri: jangan takut dengan eksperimen dan tantang dirimu sendiri!
