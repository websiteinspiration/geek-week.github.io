Web 3.0 Waveに乗る

開発者のChristophe Verdotが最近受講したオンラインコース「Mastering Web 3.0 with Waves」について話します。



あなたについて少し教えてください。このコースに興味をお持ちですか？

私はフリーランサーを中心に、約15年間ウェブ開発に携わってきました。

ある銀行グループから委託された発展途上国向けの長期登録Webアプリケーションを開発しているときに、ブロックチェーン認証をそれに統合するという課題に直面しました。当時、私はすでに暗号技術に興味を持っていましたが、主に投資家として、ブロックチェーン認証についてはあまり知りませんでした。

その結果、この機能は実装されませんでしたが、組織や銀行がそのようなアプリケーションのソリューションに興味を持っていると考え、問題を調査し始め、すぐに署名チェーンプロジェクトを開始しました。

ベータ版を開発しました。これはメインネットワークで既に利用可能です。当時、[Wavesプログラミング言語] Rideはまだありませんでした。私はJSONが埋め込まれた翻訳トランザクションを使用して、最も簡単な方法ですべてを行いました。しかし、主な目標は、ライドの開始後に、より高度な機能を追加することでした。そして、これが私がコースに参加した主な理由です。プロジェクトの開発の次の段階は、分散アプリケーション（dApp）の作成に関係していました。

コースのどの部分があなたにとって最もシンプルに見え、どの部分が最も難しかったですか？

最も単純なことは、すべてのタスクに十分な時間をかけたことです。このコースの意味は、何かを学ぶことであり、互いに競争することではありません。説明は非常にアクセスしやすく、イラストはシンプルですが包括的なものでした。さまざまなトピックを視覚化して理解するのに役立ちました。

課題を完了するとき、私たちは独立して考え、時には自分で何かを勉強するように強いられました。これは、何かを学び、教室で話し合ったアイデアを理解するための最良の方法です。

タスクを完了しながらコードを書き始めるまで、何度か理論的な部分を完全には理解していませんでした。 「コピー/貼り付け」を行うことは許可されていませんでした。すべてのコードを自分で作成する必要があり、これによりすべてをよりよく理解することができました。

最も難しいのは、いくつかの答えがある課題の質問が必ずしも明確ではなかったということです。私の英語は完璧ではなく、質問はネイティブスピーカーではない人によって書かれたので、時々誤解がありました。

おそらく、コースのオラクルとNFTの部分をより詳細にすることができます。ただし、いずれにしても、コースの主な目的は開発者の興味を引くことです。そして、そのすべての側面を完全に理解するために、もちろん、実験と実践にある程度の時間を費やす必要があります。

コース全体で取り組んだソリューション「クーポンバザール」について詳しく教えてください。コード例も見ることができますか？

はい、私たちは「クーポンバザール」に取り組みました。これは、人々がクーポンを売買して、より安い価格で商品やサービスを購入する権利を彼らに与える市場の場所です。各クーポンはデジタルアセットで表され、サプライヤからの特別割引が含まれます。



いくつかのアプリケーションコンポーネントを開発する必要がありました。まず、仕入先の登録とクーポンの管理システムを構築する必要がありました。次に、検証機能とユーザーがクーポンを検索する機能が必要でした。



コース中に、投票システムやサプライヤーの確認とブラックリスト登録を可能にする機能など、いくつかの新しい機能も追加しました。

最初に、スマートアセット、スマートアカウント、dAppアカウントの違いと、検証機能の基本的な操作方法を検討しました。検証機能を使用すると、アカウントのデフォルトの動作を変更できます。デフォルトでは、トランザクションの署名を検証しますが、検証機能を使用すると、他の「ルール」を設定できます。

{-# STDLIB_VERSION 3 #-}
{-# CONTENT_TYPE DAPP #-}
{-# SCRIPT_TYPE ACCOUNT #-}
letownerPublicKey = base58'H8ndsHjBha6oJBQQx33zqbP5wi8sQP7hwgjzWUv3q95M'
@Verifier(tx)
funcverify() = {
    matchtx {
        cases: SetScriptTransaction=>sigVerify(tx.bodyBytes, tx.proofs[0], ownerPublicKey)
        cased: DataTransaction=>true
        case_ =>false
    }
}

次に、クーポンの追加を開始しました。私たちはdAppの最も重要な機能の1つを使用しました。これにより、任意のタイプのデータをキーと値のペアの形式でブロックチェーンに書き込むことができます-データトランザクション。これを、ブロックチェーンの外部からdAppの呼び出された関数を呼び出すために使用される新しいトランザクションinvokeScriptと組み合わせました。

このコースで使用したデータトランザクションのタイプは、市場にクーポンを追加することでした。

このデータをaddItem関数で処理し、購入関数とその他のオプションを開発するために、ユーザーがブロックチェーンの外部から呼び出すことができる呼び出された関数を使用しました。その結果、資金の送金の開始、dAppデータウェアハウスでのデータの書き込みや更新など、さまざまなタスクを実行できます。

次に、addItem関数で使用される呼び出された関数の例を示します。

@Callable(i)
funcaddItem(title: String, price: Int, data: String) = {
    letsupplierAddress = toBase58String(i.caller.bytes)
    letitem = getKeyItem(supplierAddress, title)
    if( price <= 0) thenthrow("purchase amount cannot be less than item price")
    elseif( getValueItemSupplier(item) !=NONE ) thenthrow("an item is already exist")
    else{
        WriteSet([
           DataEntry(getKeyItemSupplier(item), supplierAddress),
           DataEntry(getKeyItemPrice(item), price),
           DataEntry(getKeyItemData(item), data)
        ])
    }
}

その後、特定の製品の宣伝または削除に投票できる投票システムを開発しました。投票プロセスへの外部からの影響を防ぐために、彼女は「コミット-リビール」スキームを使用しています。

コミット段階は、ハッシュ関数とソルトを使用して暗号化された投票を収集するために使用されます。

「公開」フェーズは、暗号化された投票を収集し、それらのハッシュを比較するために使用されます。

以下は、ここで使用される呼び出された関数の例です。

@Callable(i)
funcvoteCommit(item: String, hash: String) = {
    letuser = toBase58String(i.caller.bytes)
    letcommits = getValueCommitsCount(item)
    letstatus = getValueItemStatus(item)
    if( commits >=VOTERS) thenthrow("reached max num of voters")
    elseif(getValueCommit(item, user) !=NONE) thenthrow("user has already participated")
    elseif(getKeyItemSupplier(item) ==NONE) thenthrow("item does not exist")
    elseif(status !=NONE && status !=VOTING) thenthrow("voting is not possible")
    else{
        WriteSet([
           DataEntry(getKeyCommit(item, user), hash),
           DataEntry(getKeyCommitsCount(item), commits +1),
           DataEntry(getKeyItemStatus(item),if(commits ==VOTERS) thenREVEAL elseVOTING)
        ])
    }
}

コースから他に何を学びましたか？

このコースには、トークン化トークンと非変更可能トークン（NFT）も含まれています。これらのトークンは、一意であり、したがって互換性がないものを表します。

最後のレッスンは神託についてでした。ブロックチェーンは外部からデータを受信できないため、このデータを送信するためのオラクルが必要です。

私たちの市場では、オラクルは検証する必要があり、必要に応じて、たとえば、販売されたクーポンを受け入れなかったサプライヤーをブラックリストに登録しました。

次に例を示します。

funcgetExtValueItemWhiteListStatus(item:String) = {
    item +"_verifier_status"
}
 
letverifier = "3Mx9qgMyMhHt7WUZr6PsaXNfmydxMG7YMxv"
letVERIFIED = "verified"
letBLACKLISTED = "blacklist"
@Callable(i)
funcsetstatus(supplier: String, status: String) = {
    letaccount = toBase58String(i.caller.bytes)
    if( account !=verifier ) thenthrow("only oracle verifier are able to manage whitelist")
    elseif( status !=VERIFIED && status !=BLACKLISTED) thenthrow("wrong status")
    else{
        WriteSet([
           DataEntry(getExtValueItemWhiteListStatus(supplier), status)
        ])
    }
}

あなたにとって何が一番役に立ちましたか？

最も有用な部分は割り当てです。そのおかげで、講義の資料がより明確になり、得られたばかりの知識が試行錯誤によって強化されました。IDE、エクスプローラ、オラクルでの実践的な作業は非常に役に立ちました。

あなたが実際に学んだことをどのように使うつもりですか？

最初から、このコースが私のプロジェクトを次のレベルに引き上げるのに役立つと期待していました。アイデアは今sign-web.appコードを書くことでしたライド中。既存のバージョンにはすでに文書認証の機能がありますが、RIDEのおかげで大幅に改善できます。新しいバージョンはより柔軟で理解しやすくなり、メールの認証、複数の当事者間の合意など、より多くの機能を備えています。

コースはまた、思考の糧を提供し、私は多くの新しいアイデアを思いつきました。将来的にはその成果が必ず現れると思います。
