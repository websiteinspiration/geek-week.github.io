PeerJSのブラウザーでのビデオ通話。ファストスタート

Habrのすべての読者に挨拶します。今年私は、教師と生徒のWebサイトでビデオコミュニケーションを介して直接電話をかけるための1つのトレーニングポータル用のビデオコミュニケーションモジュールを作成する機会がありました。このような初期の課題を解決する必要はありませんでした。少し検索したところ、FlashとWebRTCの 2つの方法があることがわかりました。純粋な形式のWebRTCは複雑であることが判明しました。ビデオ通信のタスクは単純ではないため、一般的には当然です。しかしその後、WebRTCのラッパーであるPeerJSに出会いました。この記事では、ブラウザダイヤラをすばやく整理する方法について説明します。

この例を繰り返すには、httpsプロトコルを使用してテストページにアクセスする必要があります（ページはカメラとマイクへのアクセスを要求し、安全なプロトコルがないとブラウザはエラーを表示するだけなので）

起動レイアウトは次のようになります。

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8" />
    <title>Peer</title>
   <script src="https://unpkg.com/peerjs@1.0.0/dist/peerjs.min.js"></script>
</head>
<body>
	<p><h3> ID: </h3><span id=myid ></span></p>
	<input id=otherPeerId type=text placeholder="otherPeerId" > <button onclick="callToNode(document.getElementById('otherPeerId').value)"></button>
	
	<br>
	<video id=myVideo muted="muted" width="400px" height="auto" ></video>
	<div id=callinfo ></div>
	<video id=remVideo width="400px" height="auto" ></video>
</body>

ヘッドセクションでは、PeerJSをリモートで接続します。スクリプトをダウンロードしてローカルに接続することもできます。

input id = otherPeerId-私たちが呼び出すもののごちそうを入力するように設計されています（インデックスまたは電話番号として使用できます）。

2つのビデオタグは、自分のビデオと対話者のビデオをそれぞれ表示するように設計されています。

次に、WebRTCテクノロジーと、呼び出しの方法について少し説明します。WebRTCはサーバーを介さずにクライアントからクライアントに直接呼び出しを行うため、最初のステップで2つのブラウザーがお互いを見つける必要があります。このため、従来のWebRTCにはシグナルサーバー、つまり1つのブラウザーに別のブラウザーのパラメーターを通知するサーバーが必要です。WebRTCでは、このようなサーバーを自分で編成する必要があります。ただし、PeerJS開発者は独自のシグナルサーバーを提供します。必要なのは、peerIDを潜在的な対話者（つまり、PeerJSシステムで受信した一意のインデックス）に渡すことだけです。草案では、次のように整理しました。

  1. ページを読み込んだ後、Peerオブジェクトが作成されます
  2. そのpeerIDはmysqlデータベースに書き込まれます
  3. [通話]ボタンを押すと、対話者のpeerIDがデータベースから引き出され、接続を確立するために使用されます

現在のテスト例では、対話フィールドのpeerIDをテキストフィールドotherPeerIdに入力します。

そこで、コード

1の記述を開始します。メインオブジェクトピアを作成します

var peer = new Peer();

2.ごちそうの初めに、私たちは切望されたpeerIDを受け取ります。これは、パートナーに連絡できるようにパートナーに転送する必要があります

peer.on('open', function(peerID) {
			document.getElementById('myid').innerHTML=peerID;			
		});

3.電話を受けるために、callイベントのハンドラーを切断します。

var peercall;
peer.on('call', function(call) {
		  // Answer the call, providing our mediaStream
			peercall=call;
			document.getElementById('callinfo').innerHTML="  <button onclick='callanswer()' ></button><button onclick='callcancel()' ></button>";
		});

着信コールでは、コールオブジェクトを取得し、それをpeercallグローバル変数に保存します。また、情報ブロックには、着信に関する通知と2つのボタンが表示されます：承諾と拒否

4. 承諾ボタンの関数を記述します

function callanswer() {
	navigator.mediaDevices.getUserMedia ({ audio: true, video: true }).then(function(mediaStream) {
	var video = document.getElementById('myVideo');		    				  
	 peercall.answer(mediaStream); //        
	//peercall.on ('close', onCallClose); //  - 
	 video.srcObject = mediaStream; //      (  )
	document.getElementById('callinfo').innerHTML=" ... <button onclick='callclose()' > </button>"; //,   ,    
	video.onloadedmetadata = function(e) {// ,   
		video.play();
	  };
	  setTimeout(function() {
                 //       
	        document.getElementById('remVideo').srcObject = peercall.remoteStream; 
		document.getElementById('remVideo').onloadedmetadata= function(e) {
//      
				document.getElementById('remVideo').play();
		  };
	},1500);			  				  
				  
}).catch(function(err) { console.log(err.name + ": " + err.message); });
}

navigator.mediaDevices.getUserMedia-カメラとマイクへのアクセスをリクエストします。このメソッドに渡されるオブジェクトのデータ{audio：true、video：true}では、カメラまたはマイクへのアクセスのみを適宜要求できます。さらにコメントがコードに直接追加されました。

setTimeoutが実験的に追加されました。パートナーのビデオは再生を開始しませんでしたが、タイムアウトで動作しました。

5. 通話ボタンによるダイヤル機能

function callToNode(peerId) { //
	  navigator.mediaDevices.getUserMedia ({ audio: true, video: true }).then(function(mediaStream) {
	  var video = document.getElementById('myVideo');				  
	  peercall = peer.call(peerId,mediaStream); //,  peerId-    mediaStream
	  peercall.on('stream', function (stream) { // ,  
		  setTimeout(function() {
		  document.getElementById('remVideo').srcObject = peercall.remoteStream;
		  document.getElementById('remVideo').onloadedmetadata= function(e) {
			document.getElementById('remVideo').play();
		  };
	      },1500);	
     });
     //  peercall.on('close', onCallClose);				  
	video.srcObject = mediaStream;
	video.onloadedmetadata = function(e) {
		video.play();
	 };
   }).catch(function(err) { console.log(err.name + ": " + err.message); });
}

前の段落と同様に、メディアストリームをリクエストします。呼び出しオブジェクトを返すピアオブジェクトの呼び出し関数を呼び出した後、それをpeercallに保存します。私たちは、私たちが答えたものを見つけると、対応するビデオオブジェクト内の着信ストリームを置くためにストリームイベントを処理する

...すべてのですが、

両方の発信者が背後にある場合はNAT-番目のコールが通過しません。（なぜですか？habr.com/en/company/yandex/blog/419951をお読みください）
この障害を克服するために、ピアオブジェクトを作成するときにTURNサーバーを指定する必要があります（それを取得する場所の問題は、最も簡単ではないことが判明しました。自分で上げる：Ubuntu 16.04のVPSコマンドによるインストール
apt install coturn
）

その後、ごちそうの作成は次のようになります。

var callOptions={'iceServers': [
		{url: 'stun:95.xxx.xx.x9:3479',		
		username: "user",
		credential: "xxxxxxxxxx"},
		{ url: "turn:95.xxx.xx.x9:3478",		
		username: "user",
		credential: "xxxxxxxx"}]
	}; 
peer= new Peer({config: callOptions});
