DotNext 2019モスクワプログラムの概要：誰が何を教えてくれますか？



新しい会議に興味があるかどうかわからないことがあります。Webサイトを開いてから、40のレポートがあり、目が大きくなります。さて、あなたは40ページの説明を注意深く通過する必要がありますか？また、サイトに「ワークショップ」セクションがある場合、それらについて何を知る必要がありますか？

次のモスクワDotNext（11月6〜7日）について、ハブラの住民を支援するために、レポートをいくつかのブロックに分割して1つのテキストを作成しました。これはより速く理解できます。

同時に、彼らは同じ話者の過去のいくつかのレポートのビデオ録画を参照しました。彼らが話者のようであるかどうかを事前に理解し、役立つものを学ぶことができます。

ワークショップ


まず、このDotNextが前のDotNextとどのように異なるかに注意してみましょう。レポートに加えて、2つのワークショップもあります（個別のチケットは必要ありません。これは会議の一部です）。つまり、2時間を費やした後、理論だけでなく実際に何かを学ぶことが可能になります。参加したい場合は、ラップトップを持参してください。

彼らは、ワークショップに加えて、レポートを作成するスピーカーによって主導されます：

Dmitry Soshnikov（シュワール）興味深い機械学習（これは彼のレポートに特化しています）、および関数型プログラミングです。したがって、彼のワークショップでは、これらの両方のトピックにすぐに慣れることができます。彼は「F＃機械学習のコーディング道場」と呼ばれています。

そしてMaxim Arshinov（マルシノフ）「題材モデルの輝きと貧困」と題する講演を行うワークショップの名前には「モデル」という言葉もある。これは「貧血モデルから5ステップの豊富なモデル」と呼ばれ、リファクタリングの助けを借りて、手の動きが簡単なCRUDスタイルのアプリケーションがDDDスタイルに変わる具体的な例を示します。


基調講演


多くのレポートは、狭い特定のトピックを深く掘り下げることを提案しています。しかし、その日を開閉する基調講演では、それは異なります。それらは、「高く座る-私は遠くを見る」という原則に基づいて構築されており、グローバルなことについて考えるよう促します。

たとえば、JetBrainsのRiderチームのリーダーであるCyril Skryganは 、Platform Warsのパフォーマンスでカンファレンスを開きます。彼の仕事により、必然的にコードだけでなく、これらの非常に「プラットフォーム戦争」も目にします。大規模なIT企業とそのプロジェクトは業界でどのように相互作用し、これは私たちにとって何を意味しますか？

CyrilはDotNextで繰り返し演奏しており（優れた聴衆のレビュー付き）、プラットフォームの変化が彼の以前のレポートのトピックに反映されています。 2015年に戻って、彼のレポートReSharper vs. Roslyn "そして、私たちがIDEshkaを書いたなら...そしてすでにそこに響きました..."。そして、彼らは同じIDEを作成し、Cyrilは彼のレポート「Rider：A JetBrainsからの新しいクロスプラットフォーム.NET IDE」でまだあまり知られていないプロジェクトを発表しました。



そして会議は、基調講演「作曲の力」をスコット・ヴァラシンが締めくくります。これは関数型プログラミングに直接関係しますが、F＃の熟練した専門家である必要はありません。これは「全員向け」の一般的なレポートです。

最近、スコットのハーバードにインタビューしました-このテキストから彼についてもっと知ることができます。そして、彼がどのように話しているのかを確認したい場合（同時にオブジェクト指向のものから機能的な世界を調べてください）-最初の知人にとって、NDC Oslo会議からのレポート「F＃for C＃プログラマー」の録音は完璧です。



基調講演は「グローバル」に特化しているため、他のレポートとは異なり、開発に関するものではない場合もあります。このDotNextには、科学の有名なポピュラライザと、疑似科学Alexander Panchinとの戦闘機があり、バイオテクノロジーに関するSF作家の予測と実際に私たちが実際に目にするものを比較します。

アレクサンダーはまだDotNextに到着していませんでしたが、春に彼は別のテーマですでに私たちの他の会議で話しました：「神のゲーム。科学は限界を超えましたか？」そして、その報告が会議のトップ10に入って以来、開発者は彼のスピーチを本当に気に入っています。


トレンド


まだ生産にユビキタスではないが、増加しており、人気に向かっている技術についての報告があります。今はまだそれらを必要としない可能性もありますが、それらについて知ることはすでに役に立ちます-時間が来ると、武装します。

たとえば、ML.NETと機械学習については、ロシア語のDmitry Soshnikovによる「.NET開発者はディープラーニングについて知りたかったが質問することを恐れていた」と、英語を話す「C＃開発者向けの機械学習：ML.NETの紹介」がJeff Prozaizによって同時に報告されます。。そして、私たちはこれらのスピーカーの両方にもインタビューしました、あなたは比較することができます：Prosease（WintellectでのJeffrey Richterとの協力についてだけでなく）、Soshnikov（ITへの子供たちの紹介についてだけでなく）。

多くのDotNext視聴者はDmitryをよく知っています。たとえば、彼は彼のレポート「人工知能と.NET開発者のためのニューラルネットワーク」（DotNext 2017モスクワ）ですでにMLのトピックを取り上げました。 ML.NETの登場により、そのレポートの内容は少し関連性が低くなりましたが、それでもスピーカーのスキルを評価できます。


リリースC＃8.0と.NET Core 3.0は約1か月前に行われました。つまり、すべてのユーザーがそれらを試すことはできませんでした。時間がなかったすべての人のために、Raffaele Rialdi（DotNetLiguriaの社長およびDeveloper SecurityカテゴリーのMVP）が これを行うのに役立ちます。 彼は春にサンクトペテルブルクのDotNextに飛んで来ました、あなたは彼の報告の記録を見ることができます。トピックはまったく異なります（「Linux上のRaspberry PIと.NET Core」）が、英語がイタリア語のアクセントであなたに適しているかどうかを理解します。




また、.NET FoundationエグゼクティブディレクターのJon Galloway が、.NET Core 3.0の新機能と新機能について説明します。検討されているものの中には、デスクトップシステム（WPFおよびWindowsフォーム）、Blazor、1つの実行可能ファイル内のプログラム、言語関数などのサポートがあります。 ジョンは春のDotNext 2019 Piterにも参加していました。そのとき、彼はオープンソースについて話しました。もちろん、.NET Foundationの人、当然の質問です。




おそらくすでにGraphQLという単語に出会ったことでしょう。しかし、このテクノロジーは昨日登場しなかったものの、静かなせせらぎで新しい支持者を征服し続けているため、トレンドに起因している可能性があります。若い戦闘機は、ChilliCreamの共同創設者Michael Staibによって教えられ、RESTとGraphQLを比較し、ASP.NET CoreでHot Chocolateを使用してGraphQL APIを構築する方法を示します。


3年前、Dockerは大きなトレンドでしたが、DotNextではこの名前も鳴りました。誰も驚かないでしょうが、Kubernetesは彼の後の業界標準にもなりました。特に、Visual Studio 2019の新機能を使用して、既存のソリューションをこの世界に転送する方法を説明します。これは、ASP.NETカテゴリの9回のMicrosoft MVPステータスホルダーであるMarco De Sanctis が表示します。


ビジュアルプログラミングのアイデア自体は新しいものではありませんが、ごく少数の場合にのみ適用できると見なされています。vvvvプロジェクトのJoregとElias Holzerはこれに同意しません。彼らは、ビジュアルプログラミングが以前よりもはるかに広く使用される新しい時代が近づいていると信じており、レポートは vvvv の機能を詳細に示します。


理論


今では、最近その名前が聞かれるようになった新しい問題についてではなく、建築の問題など、より永遠の問題についてです。原則として、作曲に関するScott Vlashinの基調講演はこのブロックにも当てはまります。そして、ここに他に何があるかです：

スピーカーとしてのマキシムについて、私はこれに注意したいと思います。彼の昨年のレポート「Instant Design」は、YouTubeでいいね！を収集するだけでなく、Habrのテキストバージョンも作成するほど需要が高かったことがわかりましたそして彼女は多くの注目を集めました。

そしてアレクセイマーソン（別名m_a_d）は、.NETスタックで高品質の複雑なアプリケーションを作成する方法を説明します。AlexはASP.NETボイラープレートフレームワーク（ABP）を使用します。アレクセイはリアルタイムで、一連のバン（監査ログ、プラグイン、SignalR、ローカリゼーションなど）を備えた実用的なWebポータルを作成し、これらのバンの実装方法を簡単に説明します。
