Comment créer un serveur PostgreSQL sur Google Cloud Platform SQL

Une traduction de l'article a été préparée avant le début du cours PostgreSQL .



introduction


Dans cet article, je vais vous présenter GCP SQL et montrer comment créer un serveur PostgreSQL dans ce service.

Comment créer PostgreSQL dans GCP SQL


Commençons par créer un serveur PostgreSQL dans GCP SQL.

Avant de commencer, connectez-vous à votre compte GCP. Pour créer un serveur de base de données PostgreSQL -> Sélectionnez SQL dans le panneau Services Google.



Pour créer une instance de SQL -> Cliquez sur CRÉER UNE INSTANCE.



Sélectionnez la base de données dont vous avez besoin → Cliquez sur PostgreSQL.



Remplissez les détails de configuration:

ID d'instance: - Nommez votre instance,

Définissez un mot de passe - Mot de passe,

Choisissez la région la plus proche de vous - Région,

Sélectionnez la version du serveur de base de données - Version de la base de données.

Cliquez sur Afficher les options de configuration pour configurer des informations supplémentaires.



Développer les détails de configuration



Cliquez sur Connectivité → Par défaut, l'instance n'est pas accessible à partir de réseaux externes; Pour accorder l'accès à une instance, autorisez une adresse IP ou un partage spécifique. Cliquez sur + Ajouter un réseau.



En entrant 0.0.0.0/0 dans le réseau, vous autorisez toutes les IP IPv4 et vous pouvez accéder à l'instance de n'importe où.



Ensuite, nous attendons que les paramètres soient enregistrés.



Pour communiquer avec des ressources supplémentaires et l'API → activez IP privée et sélectionnez le réseau associé (le réseau VPC que vous avez créé ou par défaut).



Nous attendons à nouveau la sauvegarde.



Vous pouvez également vérifier les connexions de service privées en cliquant sur VPC → Homologation VPC



Sélectionnez le type de machines et de stockage: nombre de cœurs et mémoire.



Si vous avez l'intention d'utiliser l'instance à des fins de test, sélectionnez une seule zone.



Si vous disposez d'un environnement de production, sélectionnez Haute disponibilité, mais cela augmentera le coût.



Cliquez sur Ajouter un élément pour ajouter les indicateurs de base de données.



Sélectionnez le drapeau dont vous avez besoin.


.
GCP mettra à jour la base de données tous les quelques mois, nous sommes donc invités à définir une date et une heure à l'avance pour redémarrer votre base de données, ce qui ne prend pas beaucoup de temps. Alors, choisissez une date et une heure si vous avez une base de données de production.



Spécifiez la clé et la valeur pour plus de clarté dans l'onglet Étiquettes.



Cliquez sur Créer pour créer la base de données.



GCP crée le serveur PostgreSQL:



succès! Le serveur PostgreSQL a été créé avec succès.



Vous verrez des informations et un aperçu de la nouvelle création de PostgreSQL.



En cliquant sur Connexions, vous pouvez voir les détails de la connexion et également voir que la base de données a été autorisée à communiquer avec GCP App Engine.



Pour accéder au serveur PostgreSQL créé, nous devons créer un utilisateur dans l'onglet UTILISATEURS.



Vous pouvez également créer une base de données sous l'onglet BASES DE DONNÉES.



Vous pouvez gérer les sauvegardes dans BACKUPS.
