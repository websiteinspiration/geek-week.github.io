GitHub：OSINT用の新しいオープンソースライブラリ


情報セキュリティの世界では、個人情報を取得するためのオープンソースの調査について、ディフェンダー（たとえば、境界を監視し、オープンな脆弱性を特定する）であるか、侵入者（エントリポイントの検索、境界監査など）であるかについてよく尋ねられます。これらのソースの1つであるhunter.ioについて検討します。hunter.ioは、そのための新しいライブラリを作成することによって最適化しました。詳細と猫の下のgithubへのリンク。

機密情報（サブドメインや電子メールアドレスなど）を見つける方法はたくさんありますが、パブリックドメインに表示されたことのあるアドレスを見つけるためのツールもあります。すべての方法が異なります。

  • プレゼンテーション方法（例：クラウドベース、オンプレミス、ハイブリッド）、
  • アクセス（Webビュー、APIなど）、
  • 費用（有料、無料）。

hunter.ioサービスは、ドメインの連絡先情報を取得できる企業のメールアドレスを見つけるためのツールです。そのような情報は、たとえばhaveibeenpwned.comで、侵害が見つかった電子メールをさらにチェックするために必要です。

Hunter.ioは、5つの主要な方法を使用して、6700万のオープンソースから必要なデータを処理および抽出できます。

  1. ドメイン検索：ドメインまたは組織名でメールを検索
  2. Email Finder：名前と会社で個々の従業員を検索します
  3. Email Verifier：メールの状態と関連性の確認
  4. メール数：1つのドメインまたは1つの会社のメール数を確認できます
  5. アカウント情報：個人アカウントを管理します（例：リクエストのバランス）

検索プロセスを自動化してGo（Golang）プログラミング言語で開発された既存のサービスに含めるために、gohunterライブラリを作成しました。そのプロトタイプはpyhunterライブラリ（https://github.com/VonStruddle/PyHunter）でした。 Python 3では、hunter.io APIを使用します。

ここではGo（Golang）を選択した理由については説明しません。Habréには多くの記事があります。これを公理として考えてください。

最初は、pyhunterスクリプト（https://github.com/n4xh4ck5/pyhunterを使用して、hunter.ioでコマンドラインから情報を取得しました）ドメインのみでメールを検索します。ここで作成したライブラリは、ドメインだけでなく組織の名前も探しており、関連性とソースへのポイントについてメールをチェックしています。

したがって、ライブラリは、hunter.ioサービスで使用可能なメソッドのクライアントです。このようなメソッドは、hunter.io HTTP API（バージョン2またはv2）を呼び出し、JSON形式で応答を受信します。これは、必要な情報を検索および選択するのに便利です。感謝しないでください。
