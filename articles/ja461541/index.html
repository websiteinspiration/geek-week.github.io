Reactは、ユーザーインターフェイスを開発するためのオープンソースのJavaScriptライブラリです。

1. Reactを使い始める


1.1 Reactを追加する方法


Reactをサイトに追加するには、主に2つの方法があります。

  1. <script />タグの使用
  2. create-react-appの使用

1.2 addメソッドの選択


方法の選択はニーズに依存します。既存のページに小さな対話機能を追加したいだけの場合、またはReactを試してみたい場合は、最初の接続方法を使用します。本格的なReactアプリケーションをビルドする場合は、create-react-appを使用します。

1.2.1 <script />タグを使用してReactを追加する


ステップ1ページのヘッドコンテナに3つのタグを追加します。

1.2.2 create-react-appによるReactの追加


Reactの開発に使用されるツールはNode.jsに基づいているため、最初に行う必要があるのは、npmを使用するためにNodeをインストールすることです。

create-react-appパッケージは、複雑なReactパッケージを作成および管理するための標準的な方法であり、開発者に完全なツールセットを提供します。使用作成反応するアプリを新しいアプリケーションに反応作成します。

nodejsをインストールした後、create-react-appをインストールするには：

// npx
npx create-react-app my-app

// npm
npm init react-app my-app

// yarn
yarn create react-app my-app

選択は、Reactアプリケーションを作成するために使用するツール（npx、npm、yarn）です。

新しいアプリケーションを作成すると、CLIはYarnを使用して依存関係をインストールします（可能な場合）。Yarnがインストールされているがnpmを使用する場合は、createコマンドに--use-npmを追加できます。例えば：

npx create-react-app my-app --use-npm

これらのコマンドを実行すると、現在のフォルダー内にmy-appというディレクトリが作成されます。このディレクトリ内に、初期プロジェクト構造が生成され、必要な依存関係がインストールされます。

cd my-app

コードに変更を加えると、ページは自動的にリロードされます。コンソールにビルドエラーと警告が表示されます。localhost：3000を開いて、アプリケーションをブラウザーで表示します。

2.1 Reactオブジェクト


Reactは、Reactライブラリへのエントリポイントです。スクリプトタグを使用してReactを接続すると、グローバルReactオブジェクトでトップレベルのAPIを使用できるようになります。ES6とcreate-react-appを使用する場合は、Reactオブジェクトをインポートする必要があります。

import React from 'react';

ES5とcreate-react-appを使用する場合、次のようにReactオブジェクトをインポートできます。

var React = require('react');

この記事では、ES6構文を使用しており、ES6構文も使用すると想定しています。それ以外の場合は、ドキュメントの関連セクション「React without ES6」を読むことをお勧めします。

2.2 React element


React要素はReactアプリケーションの最小単位であり、ページに表示したいものを説明するオブジェクトです。 React要素は不変ではありません。つまり、React要素の状態は作成後に変更できません。

React要素を作成するには、createElement（）または
createFactory（）メソッドを使用する必要があります。後者は廃止されたと見なされており、その使用は推奨されていないため、ここでは詳しく説明しません。 JSXを使用する場合、これらのメソッドを呼び出す必要はありません。

Reactは、複製のためのメソッドを提供します-cloneElement、チェック-isValidElement（）およびデータ構造の操作props.childrenはReact.Children（React.Childrenメソッドのファミリー）です。

React要素を表示するには、ReactDomオブジェクトのRenderメソッドを呼び出す必要があります。 React要素を表示（レンダリング）した後は、その子孫や属性を変更することはできません。唯一の解決策は、ReactDom.render（）を再度呼び出すことです。実際には、このメソッドを再度呼び出すことはお勧めしません。したがって、インタラクティブな要素の場合は、Reactコンポーネントを使用する必要があります。

React要素を使用すると、特定の瞬間におけるインターフェースの外観（時間の経過に伴う変化など）を簡単に説明できます。さらに、このアプローチでは、クラス全体のエラーを取り除くことができます。JavaScriptインライン式は

React要素で許可されます。 インライン式の使用例：



const name = 'Michael';
var nameUser = () => "Franklin";
const element = <h1>, {name}</h1>;
const element_2 = <h1>, {nameUser()}</h1>;

ReactDOM.render(element, document.getElementById("root"));
ReactDOM.render(element_2, document.getElementById("root_2"));

//Output
, Michael
, Franklin

JSXを使用していない場合は、createElementメソッドの引数の1つとしてそれらを渡すことにより、インライン式を使用できます。

createElementメソッドでインライン式を使用する例：

var element = React.createElement("h1", null, ", ", name);
var element_2 = React.createElement("h1", null, ", ", nameUser());

2.2.1 React.createElement


React要素を作成するには、createElementメソッドを使用します。

React.createElement(
  type,
  [props],
  [...children]
);

  • type-タグ名、Reactコンポーネント、またはReactフラグメントを含むJavaSacript文字列。
  • props- プロップ（Reactオブジェクトのインスタンスプロパティ）
  • children-タイプの子要素。

React.createElementを使用して、JSXを使用せずに要素を作成します。 

element = React.createElement(Message, {
  message: "Hello, world"
}, React.createElement("p", null, "Some child element"));

JSXを使用する場合、React要素を作成するための特別なメソッドは必要ありません。

JSXを使用してReact要素を作成する：

element = <Message message="Hello, world">
  <p>Some child element</p>
</Message>;

ご覧のとおり、どちらの例でも同じオブジェクトが作成されます。したがって、上記の例から、各JSX要素はReact.createElement（）を呼び出すための構文上の砂糖であると結論付けることができます。

あなたは自分でそれをチェックすることができます。このオンラインコンパイラBabelの使用。

2.2.2 React.cloneElement


React.cloneElement-React要素のクローンを作成できるメソッド。

React.cloneElement(
  element,
  [props],
  [...children]
)

渡されたReact要素を開始点として使用して、新しいReact要素を返します。引数は、最初のパラメーターが新しいタグの名前ではなく、既存のReact要素であることを除いて、React.createElement（）と同じです。結果の要素には、元の要素の小道具が含まれ、新しい小道具がサーフェスにマージされます。新しい子供は既存の子供を置き換えます。ソース要素からのキーと参照が保存されます。

例：

var a = <p>Some React element</p>;
var b = React.cloneElement(a);

ReactDOM.render(b, document.getElementById("root"));
//Output 
Some React element

2.2.3 React.isValidElement


React.isValidElement（オブジェクト）-オブジェクトがReact要素であることを確認します。オブジェクトがReact要素の場合はtrueを返し、React要素でない場合はfalseを返します。

const A = <p>Some text</p>;
const B = {};

if (React.isValidElement(A))
  console.info("A is React Element");

if (React.isValidElement(B))
  console.info("B is React Element");
else
  console.error("B is not a React Element");

// Output
A is React Element
B is not a React Element

2.2.4 React.Children


React.Childrenは、不透明なデータ構造this.props.childrenを操作するための関数を提供します。より単純な場合、props.childrenは、コンポーネントを呼び出すときに開始タグと終了タグの間に挿入するものです。

簡単な例：

function Chill(props) {
        return <h1>Hello {props.children}</h1>
}

ReactDom.render(<Chill>Mike</Chill>, element); // Hello Mike

このように、props.childrenはReactコンポーネントをより柔軟で再利用可能にするのに役立ちます。「共通ブロック」を表し、「事前に子を知らない」コンポーネントには、props.childrenを使用できます。つまり、上記の例では、Chillコンポーネントは複数回使用できるテンプレートであり、結果は異なります。もちろん、このような単純化された例は、propsの使用に非常によく似ていますが、props.childrenを使用すると、より複雑なデータ構造を構築できます。

props.childrenを操作するメソッドを簡単に見てみましょう。

  • React.Children.map-配列の各要素に対して指定された関数を呼び出した結果を持つ新しい配列を作成します。
  • React.Children.forEach-配列内の要素ごとに指定された関数を1回実行します。
  • React.Children.count — children;
  • React.Children.only — children (React ), ;
  • React.Children.toArray — children ,

React.Children.map

React.Children.map(children, function[(thisArg)])

Array.prototype.map（）に似ています。配列の各要素に対して指定された関数を呼び出した結果で新しい配列を作成します。子に含まれる各直接の子孫に対して関数を呼び出し、それらを順番にthisArgに渡します。子が配列の場合は渡され、配列内の子ごとに関数が呼び出されます。子がnullまたは未定義の場合、このメソッドは配列ではなくnullまたは未定義を返します。子がReact.Fragmentの場合、それは全体の子孫と見なされ、内部の要素は渡されません。 mapメソッドは、要素ごとに渡されたコールバック関数を出現順に1回呼び出し、その呼び出しの結果から新しい配列を作成します。

React.Children.forEach

React.Children.forEach(children, function[(thisArg)])

Array.prototype.forEach（）に似ています。配列内の要素ごとに指定された関数を1回実行します。React.Children.mapとは異なり、関数呼び出しの結果から配列を返しません。ubdefinedを返します。

例：

var users = ["Mike", "Bill", "Scott"];

function Chill(props) {
    return <span>{React.Children.map(props.children, (thisArg) => <p>Hello {thisArg}</p>)}</span>;
}

render (<Chill>{users}</Chill>, element); // Hello Mike, Hello Bill, Hello Scott

あなたはこことここでさらに多くの例を見つけます。

2.3 Reactコンポーネント


Reactコンポーネント-React要素を返す関数、またはReact要素を返すrender（）メソッドを実装するJavaScriptクラス。したがって、JavaScript関数を使用して宣言されたコンポーネントは一般に「機能コンポーネント」と呼ばれ、JavaScriptクラスを使用して宣言されたコンポーネントは一般に「クラスコンポーネント」と呼ばれます。

JavaScript関数として宣言されたコンポーネントは「ステート」を含むことができないため、ステートレスコンポーネント、つまりステートレスコンポーネントとも呼ばれ、JavaScriptクラスとして宣言されたコンポーネントは「ステート」を含むことができるため、ステートフルコンポーネント、つまりコンポーネントと呼ばれます。状態あり。 *実際には、フックを使用して機能コンポーネントから同様の機能を実現できます。

React.ComponentまたはReact.PureComponentを サブクラス化することにより、クラスReactコンポーネントを宣言できます。後者は、shouldComponentUpdate（）メソッドを実装している点のみが異なります（コンポーネントのライフサイクルを参照）。

機能的なReactコンポーネントは、React.memoでラップできるJavaScript関数です。React.memoはReact.PureComponentに似ていますが、機能コンポーネントのみを対象としています。

プロップは、コンポーネントに渡される単なる引数です。ReactはJSX属性を検出すると、それらを1つのオブジェクトに収集してコンポーネントに渡します。このオブジェクトは「小道具」（props）と呼ばれます。

機能コンポーネントでの小道具の使用例：

function SayHi(props) {
    return <h1>, {props.name}</h1>;
}

class SayHi_2 extends React.Component {
    render() {
        return <h1>, {this.props.name}</h1>;
    }
}

ReactDOM.render(<SayHi name="Mark" />, document.getElementById("root"));
ReactDOM.render(<SayHi_2 name="Bob" />, document.getElementById("root_2"));

//Output
, Mark
, Bob

覚えてね！カスタムコンポーネントのタイプの名前は大文字で始める必要があります!!!
Reactコンポーネントは、それらのプロップに関連して純粋な関数として動作する必要があります。

コンポーネントは、それが機能的であるかクラスであるかに関わらず、その小道具に何かを書くべきではありません。

2.3.1 React.Component


React.Componentは、Reactコンポーネントの基本クラスです。Reactコンポーネントを作成するには、React.Componentからプロパティとメソッドを継承します。

class myComponent extends React.Component {
  render() {
    return <h1>Hello, world</h1>;
  }
}

各コンポーネントには、コンポーネントがマウント、更新、およびアンマウントされるときに自然に呼び出されるライフサイクルメソッドがあります。このメソッドをオーバーライドすると、プロセスの特定の段階でコードを実行できます。（2.3.5ライフサイクルを参照）。

2.3.2 React.Fragment


ご覧のとおり、React要素とコンポーネントは1つのノード（1つのhtmlタグ）のみを返す必要があります。複数のタグをレンダリングする必要がある場合は、1つのコンテナでラップする必要があります。そうしないと、エラーが発生します。しかし、追加のコンテナーを作成したくない場合はどうすればよいでしょうか。回答：React.Fragment。

React.Fragmentコンポーネントを使用すると、追加のDOM要素を作成せずにrender（）メソッドで複数の要素を返すことができます。

render() {
    return (
       <React.Fragment>
            <p>Some text</p>
            <h2>Tittle</h2>
      </React.Fragment>
    );
}
もちろん、Jsxを使用していない場合、コードは次のようになります。 

React.createElement(React.Fragment, null, React.createElement("p", null, "Some text"), React.createElement("h2", null, "Tittle"));

React v16.2.0はReact.Fragmentの新しい構文シュガーを導入しました。このバージョンから、React.Fragmentでラップされる空のhtmlタグを簡単に記述できます。

render() {
    return (
       <>
            <p>Some text</p>
            <h2>Tittle</h2>
      </>
    );
}

2.3.3州


コンポーネントの状態（State）は、クラスReactコンポーネントのプライベートプロパティ（状態）です。 setState（）メソッドを使用して状態を更新すると、コンポーネントが繰り返しレンダリングされます。これにより、（setState（）を呼び出さずに）状態が直接変更された場合、コンポーネントは再度レンダリングされないことに注意してください。

カウンターボタンを使用した最も簡単な例を見てみましょう。この例では、ボタンがクリックされると、incrementCountメソッドが呼び出され、setStateを使用してコンポーネントの状態を更新し、コンポーネントを再度レンダリングして、新しいカウンター値を表示します。

class Counter extends React.Component {
    constructor(props) {
        super(props);

        this.state = { count: 0 };
        this.incrementCount = this.incrementCount.bind(this);
    }
    incrementCount() {
        this.setState(state => ({ count: state.count + 1 }));
    }

    render() {
        return <div>
            <p>Count = {this.state.count}</p>
            <button onClick={this.incrementCount}>Add</button>
        </div>
    }
}
ReactDOM.render(<Counter />, document.getElementById("root"));

コードを分解してみましょう。

super（）キーワードは、親コンストラクターを呼び出す関数として使用されます。コンストラクターの本体でthisキーワードを最初に呼び出す前に呼び出す必要があります。したがって、コンポーネントの初期状態を初期化する前に、super（）を呼び出す必要があります。クラスコンポーネントは常に、props引数を使用して基本コンストラクターを呼び出す必要があります。小道具を使わなくても。

メソッドの場合、Function.prototype.bind（）から継承されたbind（）メソッドを呼び出す必要があります。これは、呼び出されたときに、指定された値を実行コンテキストとして設定する新しい関数を作成します。これは、JavaScriptのクラスメソッドがデフォルトでコンテキストバインドされていないためです。したがって、コンテキストにバインドしないと、関数が呼び出された時点でこれは未定義になります。パブリッククラスフィールドの構文を使用して、bind（）を呼び出さないようにする方法があります。

incrementCount = () => {
        this.setState(state => ({ count: state.count + 1 }));
}

onClickキーワードを使用すると、イベントが発生するたびにincrementCountメソッドを呼び出すイベントハンドラーが登録されます。 Reactイベント処理機能については、次の章で説明します。

setState（）は、現在の状態値を使用して更新します。したがって、新しい状態のオブジェクトの代わりに、現在の状態を引数として受け取る関数が渡されます。実際には、setState呼び出しは非同期であるため、this.stateが誤った値を表示する場合があります。これを理解するために、setStateメソッドの構文を見てみましょう。

setState(updater[, callback])

setState（）は、要求を行うfetch（）関数に似ています。 setState（）メソッドは、常にコンポーネントをすぐに更新するわけではありません。彼は次回まで更新をグループ化または遅延できます。これにより、setState（）を呼び出した直後にthis.stateを読み取ることがトラップになる可能性があります。 updater関数の最初の引数は次のとおりです。

(state[, props]) => stateChange

ここでの状態は、変更時のコンポーネントの状態への参照です。関数によって受信された状態と小道具の両方が更新されることが保証されています。setState（）の2番目のパラメーターは、setStateが実行され、コンポーネントが再レンダリングされた後に実行される追加のコールバックです。次の状態が現在の状態に依存しない場合は、最初の引数として、新しい状態のオブジェクトを渡すだけです。

ステータスの更新がマージされます！

状態に複数の値が格納されている場合、1つの値を変更しても他の値には影響しません。

constructor(props) {
    super(props);
    
this.state = { value_1: true, value_2: false };
  }

両方のフィールドは、個別のsetState（）呼び出しを使用して個別に更新できます。

changeValue_1() {
    this.setState(state => ({ value_1: !state.value_1 }));
}
changeValue_2() {
    this.setState(state => ({ value_2: !state.value_2 }));
}

状態は表面的に結合するため、changeValue_1（）を呼び出すとvalue_2はそのまま残りますが、value_1は完全に置き換えられます。

状態はコンポーネント自体でのみ使用可能であり、他のユーザーからは隠されています。つまり、this.stateはプライベートプロパティです。

コンポーネントは、その状態を子コンポーネントの小道具としてツリーに渡すことができます。

アップデートさせる （）

component.forceUpdate(callback)

render（）メソッドが他のデータに依存している場合は、forceUpdate（）を呼び出して、再レンダリングする必要があることをReactに伝えることができます。

forceUpdate（）を呼び出すと、コンポーネントでrender（）メソッドが実行され、shouldComponentUpdate（）がスキップされます。これにより、各子コンポーネントのshouldComponentUpdate（）を含む、子コンポーネントの通常のライフサイクルメソッドが呼び出されます。 Reactは、マークアップが変更された場合にのみ、DOMを更新し続けます。

この例では、ボタンを押すと、handleClickが呼び出されてコンポーネントが更新され、TittleでgetSnapshotBeforeUpdateが呼び出されて、コンポーネントが表示される前にbutton.clickプロパティの値が変更されます。その結果、ボタンを押すたびに、タイトルのキャプションが変わります。

class Clicker extends React.Component {
    constructor(props) {
        super(props);
    }
    handleClick = () => {
        this.forceUpdate();
    }
    render() {
        return <React.Fragment>
            <Tittle></Tittle>
            <button onClick={this.handleClick}>Click me</button>
        </React.Fragment>;
    }
}
class Tittle extends React.Component {
    constructor(props) {
        super(props);

        this.button = { click: false };
    }
    getSnapshotBeforeUpdate() {
        this.button.click = (this.button.click) ? false : true;
    }
    render() {
        if(this.button.click)
            return <h1>Button enabled</h1>;
        else
            return <h1>Button disabled</h1>;
    }
}
ReactDom.render(<Clicker />, document.getElementById("root"));

もちろん、handleClickでクリッカーの状態を変更し、Tittleで小道具として導くことで、これを簡単に実装できます。しかし、この例はforceUpdate（）の動作をよく示しています。コンポーネントの更新を強制する必要がある場合、forceUpdate（）はsetState（）の代替と言えます。このメソッドは2つのケースで使用できます。コンポーネントの状態を実装したくないが、強制的な更新メソッドが必要な場合、またはコンポーネントの更新時にshouldComponentUpdate（）をスキップする場合です。

getDerivedStateFromProps（）getDerivedStateFromProps（）

メソッドについては、すでに上記で説明しています。これは、初期マウント時とコンポーネントの更新時の両方で呼び出されるためです。マウント時に呼び出される理由は明らかではありません。開発者が考えた主なタスクは、オブジェクトを返して状態またはnullを更新し、何も更新しないことです。

shouldComponentUpdate（）

shouldComponentUpdate(nextProps, nextState)

shouldComponentUpdateは、状態の変化と小道具に基づく次のレンダリングの必要性を示します。デフォルトでは、状態が変化すると再レンダリングが行われます。ほとんどの場合、この動作に依存する必要があります。ただし、たとえば、特定の小道具または状態が変更された場合にのみコンポーネントを更新する必要が生じる特定の場合があり、それ以外の場合は必要ありません。 this.propsをnextPropsと、this.stateをnextStateと比較し、falseを返してReactの更新をスキップできます。

falseを返しても、状態が変化したときに子コンポーネントが再レンダリングされるのを妨げないことに注意してください。

デフォルト値はtrueです。このメソッドは、最初のレンダリング時、またはforceUpdate（）の使用時に呼び出されません。

ReactDom.render（）への追加の呼び出しはアプリケーションの速度に悪影響を与える可能性があるため、このメソッドはパフォーマンスを向上させるためにのみ必要です。レンダリングを「防止」するshouldComponentUpdate（）の機能に依存しないでください。バグが発生する可能性があります。

前述のように（2.3 Reactコンポーネントを参照）、React.ComponentではなくReact.PureComponentをサブクラス化することによって宣言されたクラスReactコンポーネントは、デフォルトでshouldComponentUpdate（）メソッドを実装します。

React.PureComponent基本クラスのshouldComponentUpdate（）メソッドは、オブジェクトの浅い比較のみを行います。それらに複雑なデータ構造が含まれている場合、これはより深い差異（つまり、構造の表面に表現されない差異）の誤動作につながる可能性があります。単純な小道具と状態を使用する予定の場合にのみPureComponentクラスを継承するか、ネストされたデータ構造が変更されたことがわかっている場合はforceUpdate（）を使用します。

shouldComponentUpdate（）の直後に更新する場合、render（）が呼び出されます。その原理については上記で説明しました。

getSnapshotBeforeUpdate（）

getSnapshotBeforeUpdate(prevProps, prevState)

getSnapshotBeforeUpdate（）は、「コミット」フェーズの直前（たとえば、DOMに追加する前）に呼び出されます。これにより、コンポーネントは、変更前にDOMから情報（たとえば、スクロール位置）を取得できます。このライフサイクルメソッドによって返される値は、componentDidUpdate（）パラメータとして渡されます。

このメソッドはrender（）の後に呼び出されますが、レンダリング前のDOMの外観である「スナップショット」を引き続き取得できます。

スナップショットの値（またはnull）は、このメソッドから返される必要があります。

componentDidUpdate（）

componentDidUpdate(prevProps, prevState, snapshot)

componentDidUpdate（）は、DOMを更新した直後に呼び出されます。 DOMを必要とするアクションの実行に適しています。また、現在の小道具と以前の小道具を比較した結果に基づいて実行されるネットワーク要求の実行にも適しています。小道具が変更されていない場合、新しいリクエストは必要ない場合があります。

componentDidUpdate（）でsetState（）を呼び出すことはできますが、無限再帰が発生しないように、条件でラップする必要があります。

getSnapshotBeforeUpdate（）ライフサイクルメソッドが実装されているまれなケースでは、その結果は3番目のスナップショットパラメータとしてcomponentDidUpdate（）に渡されます。

shouldComponentUpdate（）がfalseを返す場合、componentDidUpdate（）は呼び出されません。

2.3.6参照


refは、renderメソッドで作成されたDOMノードまたはReact要素にアクセスする機会を提供します。通常、親コンポーネントは小道具を介してのみ子と対話できますが、通常のデータストリームをバイパスして子要素を強制的に変更する必要がある場合があります。Refは小さな抜け穴のようなもので、通常の方法では必要な機能を実現できない場合に、Reactセマンティクスをバイパスできます。このため、宣言的な方法で問題を解決できる場合は、refの使用を避けてください。

参照の使用が正当化される状況：

  • フォーカスコントロール、テキスト選択、またはメディア再生。
  • 命令的な呼び出しアニメーション。
  • サードパーティのDOMライブラリとの統合。

RefはReact.createRef（）を使用して作成され、ref属性を介してReact要素にアタッチされます。 HTML要素でref属性を使用すると、コンストラクターでReact.createRef（）を使用して作成されたrefの現在のプロパティが、対応するDOM要素を取得します。

ref属性をクラスコンポーネントで使用すると、refオブジェクトの現在のプロパティは、マウントされたコンポーネントのインスタンスを取得します。

ref属性は、インスタンス化されていないため、機能コンポーネントでは使用できません。これを行うには、useRefフックを使用します。簡単に言えば、refはDOM要素またはマウントされたコンポーネントのインスタンスへの参照です。

例：

class RefExample extends React.Component {
    constructor(props) {
        super(props);

        this.ref = React.createRef();
    }
    handleClick = () => {
        var color = Math.floor(Math.random() * 16777215).toString(16);
        this.ref.current.style.color = `#${color}`;
    }
    render() {
        return( 
            <>
                <h1 ref={this.ref}>Hello world</h1>
                <button onClick={this.handleClick}>Click me</button>
            </>);
    }
}

この例では、this.refはホーム要素h1-Hello worldへの参照です。ボタンが押されると、refを介してh1へのリンクを使用してhandleClick関数が呼び出され、テキストの色がランダムに変更されます。

Refsの詳細については、こちらをご覧ください。

3.1リストとキー


Reactの開発時に著者が追求した目標の1つは、組み込みインターフェースを再利用することでした。したがって、コンポーネントはインターフェースのテンプレートに似ている場合があります。さまざまな人が作成したコンポーネントは、一緒に動作するはずです。このパターンは、コンポジション（en。コンポジション）と呼ばれます。

そのような概念の1つは、リストの使用です。たとえば、メッセンジャーのユーザーの友達リストまたはオンラインストアのショッピングリストに共通のテンプレートを複数定義して、そのようなパーソナライズされたリストをユーザーごとに表示します。

簡単な例を見てみましょう。サイドバーに表示したいユーザーの友達のリストがあります。これは次のようにして行うことができます。

var users = ["Mike", "Bill", "Scott"];

function UsersList(props) {
    const userlist = props.children.map((users) => <p>{users}</p>);
    
    return <div>{userlist}</div>;
}

render(<UsersList>{users}</UsersList>, element);

ライフサイクルの2.3.5 章でrender（）メソッドを調べると、このメソッドがReact要素またはコンポーネントだけでなく、配列、フラグメント、ポータル、文字列、数値、ブール値、またはnullも返すことができることがわかりました。このようにして、React要素で構成されるユーザーの配列を返すことができます。これはもちろん便利ですが、コンポーネントから配列を返すと、次の警告が表示されます。
警告：リスト内の各子には、一意の「キー」プロップが必要です。

リスト内の各子には、一意の「キー」プロパティが必要です。

キーは、要素のリストを作成するときに指定する必要がある特別な文字列属性です。 DOMが時間とともに変化する場合、Reactのパフォーマンスを最適化するために必要です。 DOMを変更する際のReactの理解を深めるには、記事を読んでください。比較アルゴリズム。キーは、単一のアイテムリスト内で一意である必要があります。 Keyを使用してコンポーネントを書き換えると、次のようになります。

function UsersList(props) {
    const userlist = props.children.map((users, index) => <p key={index}>{users}</p>);
    
    return <div>{userlist}</div>;
}

実際には、上記の例のように、Key属性に要素インデックスを使用することは、要素の順序がパフォーマンスに悪影響を及ぼし、コンポーネントの状態に問題を引き起こす可能性があるために順序が変更される可能性がある場合は、不適切です。したがって、データのIDを使用することをお勧めします。キーはDOM要素の属性として表示されず、React要素のプロパティです。要素.__ reactInternalInstance $ 235y8cxv0e8を使用してコンソールで要素オブジェクトを呼び出すことでこれを確認できます（$の後に別の番号が表示される場合があります）。element .__ reactInternalInstance $ 235y8cxv0e8-実験的なプロパティ、開発時にそれに依存しないでください。

3.2エラー処理


大規模なアプリケーションを開発する場合、設計エラー、論理エラー、構文エラーなど、さまざまなエラーを検出して修正することが困難な場合があります。これを行うために、Reactには、開発者がそのようなエラーを見つけて修正するのに役立つように設計されたさまざまなツールがあります。

UIコードのどこかでJavaScriptエラーが発生しても、アプリケーション全体の操作が中断されることはありません。これを行うために、子コンポーネントのツリーの任意の場所でJavaScriptエラーをキャッチし、エラーログに保存して、クラッシュしたコンポーネントツリーの代わりに予備のUIを表示する特別なメソッドが追加されました。レンダリング、ライフサイクルメソッド、およびそれらの下にあるコンポーネントツリーのコンストラクターでエラーをキャッチします。このようなメソッドを実装するオブジェクトは、一般に「ヒューズ」と呼ばれます。ヒューズの詳細については、エラー境界のドキュメントの特別なセクションをご覧ください。これらは次のように機能します。ヒューズの子要素でエラーが発生した場合、特別なメソッドを呼び出して、失敗したインターフェースではなくフォールバックをレンダリングします。

動的型付けを備えたJavaScript言語。このタイプのタイピングは、プログラマーの作業を大幅に容易にしますが、複雑なエラーを引き起こす可能性があります。FlowやTypeScriptなどの静的型付けツールを使用すると、コードを実行する前にこれらのエラーのほとんどをキャッチできます。さらに、自動追加やその他の機能を追加して、開発プロセスを大幅に改善します。 Reactアプリケーションを作成するときに静的型付けを使用するには、静的型検査のドキュメントの該当するセクションをお読みください。

ただし、FlowやTypeScriptなどの拡張機能を使用しない場合でも、Reactには組み込みの型チェック機能があります。これはPropTypesによる型チェックです。コンポーネントの小道具で転送されたデータ型が正しいかどうかを確認するには、propTypes特殊プロパティを使用する必要があります。

class PropTypesExample extends React.Component {
    render() {
        return <h1>, {this.props.name}</h1>;
    }
}
  
PropTypesExample.propTypes = {
    name: PropTypes.string
};
ReactDom.render(<PropTypesExample name={5} />, element);

3.2.2 componentDidCatch


componentDidCatch(error, info)

このメソッドは、getDerivedStateFromErrorと同様に、エラーの処理を担当し、getDerivedStateFromErrorの直後に呼び出されます。getDerivedStateFromErrorの目的が状態を更新するための値を返すことである場合、componentDidCatchの目的はエラーログです。つまり、componentDidCatchで、キャッチされたエラーに関する情報をログに記録するコードを記述する必要があります。

このメソッドは、次の2つのパラメーターを受け取ります 。error-
エラーがインターセプトされた
情報-エラーが発生したコンポーネントに関する情報を含むcomponentStackキーを持つオブジェクト。

class App extends React.Component {
    constructor(props) {
        super(props);

        this.state = { error: false };

    static getDerivedStateFromError(error) {
        //  !!!

        return {error: true};
    }
    componentDidCatch(error, info) {
        console.info(info.componentStack); //       
        console.error(error); //     console   !!!

        logComponentStackToMyService(info.componentStack); //   
    }
    render() {
        if(this.state.error) { //  
            return <h1>-   .</h1>; //   UI
        }
        //  
            return <Welcome>{this.props.children}</Welcome>; //    
    }
}
