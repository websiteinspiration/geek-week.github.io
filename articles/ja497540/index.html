ビジネスをクラウドに移行する際の6つの重要な質問



強制休暇のため、ITインフラストラクチャを開発している大企業でも、スタッフのリモート作業を整理することは難しく、中小企業には必要なサービスを展開するための十分なリソースがありません。もう1つの問題は情報セキュリティに関連しています。従業員の自宅のコンピューターから内部ネットワークへのアクセスを開くことは、特別なエンタープライズクラスの製品を使用しないと危険です。仮想サーバーをレンタルすることは、資本支出を必要とせず、安全な境界外で一時的なソリューションを作成することができます。短い記事では、自己分離条件でVDSを使用するためのいくつかの典型的なシナリオを検討します。記事はオリエンテーションであり、トピックのみを掘り下げる人に向けられていることに注意してください。

1. VDSを使用してVPNを構成する必要がありますか？


仮想プライベートネットワークは、従業員がインターネット経由で企業の内部リソースに安全にアクセスするために不可欠です。VPNサーバーは、ルーターまたはセキュリティで保護された境界内で上げることができますが、自己分離の状態では、同時に接続されているリモートユーザーの数が増加します。つまり、生産的なルーターまたは専用のコンピューターが必要になります。既存のもの（たとえば、メールサーバーやWebサーバー）を使用するのは危険です。VPNは既に多くの企業で機能していますが、VPNがまだ機能していない場合や、ルーターがすべてのリモート接続に対応できるほどランダムでない場合は、外部仮想サーバーを注文することで費用を節約し、セットアップを簡素化できます。

2. VDSでVPNサービスを整理する方法は？


まず、VDSを注文する必要があります。中小企業は独自のVPNを作成するために強力な構成を必要としません-GNU / Linuxのエントリーレベルのサーバーで十分です。コンピューティングリソースが不足している場合は、常に増やすことができます。 VPNサーバーへのクライアント接続を整理するためのプロトコルとソフトウェアを選択する必要があります。多くのオプションがありますが、Ubuntu LinuxとSoftEtherを使用することをお勧めします-このオープンクロスプラットフォームサーバーとVPNクライアントは設定が簡単で、さまざまなプロトコルをサポートし、強力な暗号化を提供します。サーバーを構成した後でも、最も興味深いことは残ります。クライアントアカウントと、従業員の自宅のコンピューターからのリモート接続のセットアップです。従業員にオフィスLANへのアクセスを提供するには、暗号化されたトンネルを介してサーバーをLANルーターに接続する必要があり、ここでSoftEtherが再び役立ちます。

3.なぜ独自のビデオ会議サービス（VKS）が必要なのですか？


電子メールとインスタントメッセンジャーだけでは、ビジネス上の問題や通信教育のオフィスでの日常的なコミュニケーションに取って代わるには不十分です。 udalenkaへの移行に伴い、中小企業や教育機関は、音声およびビデオ形式の電話会議を組織するための公的に利用可能なサービスを積極的に開発し始めました。最近のズームに関するスキャンダルはこの考えの悪さを明らかにしました：それは市場のリーダーでさえプライバシーについて十分に気にしていないことが判明しました。

独自の会議サービスを作成できますが、オフィスに配備することが常に推奨されるとは限りません。これには生産的なコンピュータと、最も重要なことには、高帯域幅のインターネット接続が必要になります。経験がなければ、会社のスペシャリストはリソース要件を誤って計算し、弱すぎたり、強すぎたり、高価な構成を注文したりすることがあり、ビジネスセンターで借りているエリアのチャネルを拡張することが常に可能であるとは限りません。さらに、インターネットから安全な境界内でアクセス可能なVKSサービスを起動することは、情報セキュリティの観点からは最善のアイデアではありません。

仮想サーバーは問題を解決するのに理想的です。毎月のサブスクリプションのみが必要ですが、計算能力は好きなように増減できます。さらに、グループチャット、ヘルプデスク、ドキュメントストレージ、ソースリポジトリ、およびVDSでのチームワークとホームトレーニングのためのその他の関連する一時的なサービスを備えた安全なメッセンジャーを簡単に展開できます。仮想サーバー上で実行されているアプリケーションが仮想サーバーを必要としない場合は、仮想サーバーをオフィスネットワークに接続する必要はありません。必要なデータをコピーするだけです。

OpenMeetingsの高レベルのセキュリティ、およびプラットフォームをカスタマイズして、人気のあるCMS、トレーニングシステム、オフィスのIPテレフォニーと統合する機能に注目する価値があります。ソリューションの欠点は、その利点の結果です。オープンソースソフトウェアを構成することは非常に困難です。同様の機能を持つ別のオープンソース製品はBigBlueButtonです。小規模チームは、たとえば国内のTrueConf Server FreeやVideoMostなどの商用ビデオ会議サーバーのシェアウェアバージョンを選択できます。後者はまた、大規模な組織に適しています：による自己分離モードには、開発者が可能に 3ヶ月間無料で1000人のユーザのためのバージョンを使用して。

次の段階では、ドキュメントを調べ、リソースの必要性を計算し、VDSを注文する必要があります。通常、ビデオ会議サーバーの展開には、十分なRAMとストレージを備えたGNU / LinuxまたはWindowsでのミッドレンジ構成が必要です。もちろん、すべては解決するタスクに依存しますが、VDSを使用すると実験を行うことができます。リソースを追加したり、不要なリソースを放棄したりしても、遅すぎることはありません。最後に、最も興味深い部分は残ります。ビデオ会議サーバーと関連ソフトウェアを構成し、ユーザーアカウントを作成し、必要に応じてクライアントプログラムをインストールします。

5.安全でない家庭用コンピュータを交換するには？


会社に仮想プライベートネットワークがあっても、安全なリモート作業に関するすべての問題が解決されるわけではありません。通常の状況では、内部リソースへのアクセスが制限された多くの人がVPNに接続することはありません。オフィス全体が自宅で働いているとき-これは完全に異なるスポーツです。従業員のパーソナルコンピュータはマルウェアに感染している可能性があり、家庭ではそれらを使用しており、マシンの構成は企業の要件を満たさないことがよくあります。
すべての人にラップトップを提供することは不採算であり、新しい手間のかかるクラウドデスクトップ仮想化ソリューションも高価ですが、解決策があります-Windowsのリモートデスクトップサービス（RDS）。それらを仮想マシンにデプロイすることは素晴らしいアイデアです。すべての従業員が標準のアプリケーションセットを使用して作業し、単一ノードからLANサービスへのアクセスを制御することがはるかに容易になります。ウイルス対策ソフトウェアと一緒に仮想サーバーをレンタルして、ライセンス購入を節約することもできます。カスペルスキーのアンチウイルス保護機能がWindowsのどの構成でも利用できるとしましょう。

6.仮想サーバーでRDSを構成するにはどうすればよいですか？


まず、コンピューティングリソースの必要性に焦点を当てて、VDSを注文する必要があります。どちらの場合も個別ですが、RDSには強力な構成が必要です。少なくとも4つのコア、同時に作業するユーザーごとに1ギガバイトのメモリ、システム用に約4 GB、およびかなり大量のストレージが必要です。チャネルの帯域幅は、ユーザーあたり250 Kbpsの必要性に基づいて計算する必要があります。

Windows Serverでは、標準で、同時に2つまでのRDPセッションを作成でき、コンピューター管理のみが可能です。本格的なリモートデスクトップサービスを構成するには、サーバーの役割とコンポーネントを追加し、ライセンスサーバーをアクティブ化するか、外部のサーバーを使用して、別途購入するクライアントアクセスライセンス（CAL）をインストールする必要があります。 Windows Serverの強力なVDSとターミナルライセンスをレンタルするとコストが高くなりますが、「鉄」サーバーを購入する方が有利です。さらに、ライセンスを合法的に支払わないオプションもあります。120日以内に、RDSを試用モードで使用できます。

Windows Server 2012以降、RDSを使用するには、マシンをActive Directory（AD）ドメインに入力することが望ましいです。これは多くの場合省略できますが、VPNを介してオフィスLANに配置されたドメインに実際のIPを持つ別の仮想サーバーを接続することは難しくありません。さらに、ユーザーは仮想デスクトップから企業の内部リソースにアクセスする必要があります。生活を楽にするために、クライアントの仮想マシンでサービスを利用するプロバイダーに連絡する必要があります。特に、RuVDSからRDS CALを購入した場合、弊社のテクニカルサポートが独自のライセンスサーバーにインストールし、クライアントの仮想マシンでリモートデスクトップサービスを構成します。

RDSを使用すると、ITスペシャリストが従業員の自宅のコンピュータのソフトウェア構成を一般的な企業の分母にすることで頭痛の種から解放され、ユーザーワークステーションのリモート管理が大幅に簡素化されます。
