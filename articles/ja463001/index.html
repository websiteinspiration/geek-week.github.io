コードとしてのインフラストラクチャにPulumiと汎用プログラミング言語を使用すると、スキルと知識が得られる、抽象化によりコード内の定型文を排除できる、IDEやリンターなどのチームが使い慣れたツールを使用できるなど、多くの利点があります。これらのソフトウェアエンジニアリングツールはすべて、生産性を向上させるだけでなく、コードの品質も向上させます。したがって、汎用のプログラミング言語を使用すると、ソフトウェア開発- テストで別の重要なプラクティスを実装できるのは当然です。

この記事では、Pulumiが「インフラストラクチャーをコードとして」のテストにどのように役立つかを見ていきます。



インフラストラクチャをテストする理由


詳細に入る前に、「インフラストラクチャをテストする必要があるのはなぜですか？」という質問をする価値があります。これには多くの理由があり、そのうちのいくつかを以下に示します。

  • プログラムの個々の関数またはロジックの一部を単体テストする
  • 特定の制限に準拠しているかどうか、インフラストラクチャの望ましい状態を確認します。
  • ストレージバケットの暗号化の欠如や安全でない、インターネットから仮想マシンへのオープンアクセスなどの一般的なエラーの検出。
  • インフラストラクチャプロビジョニングの確認。
  • 「プログラムされた」インフラストラクチャ内で実行されているアプリケーションのロジックのランタイムテストを実行して、プロビジョニング後の状態をチェックします。
  • , . Polumi . .


Pulumiプログラムは、JavaScript、Python、TypeScript、Goなどの汎用プログラミング言語で作成されます。したがって、テストフレームワークを含むツールやライブラリを含め、これらの言語のすべての機能を利用できます。 Pulumiはマルチクラウドです。つまり、任意のクラウドプロバイダーをテストに使用できます。

（この記事では、多言語とマルチクラウドであるにもかかわらず、JavaScriptとMochaを使用してAWSに焦点を合わせていunittestます。Python、Goテストフレームワーク、またはその他の好きなテストフレームワークを使用できます。もちろん、PulumiはAzure、Google Cloud、Kubernetesとうまく連携します。）

これまで見てきたように、インフラストラクチャコードをテストする必要がある理由はいくつかあります。それらの1つは通常の単体テストです。コードには関数が含まれている可能性があるため（CIDRの計算、名前、タグなどの動的な計算など）-あなたはおそらくそれらをテストしたいでしょう。これは、お気に入りのプログラミング言語でアプリケーションの定期的な単体テストを作成するのと同じです。
少し複雑になったら、プログラムがリソースを割り当てる方法を確認できます。説明のために、単純なEC2サーバーを作成する必要があり、次のことを確認したいとします。

  • インスタンスにはタグがありNameます。
  • インスタンスはインラインスクリプトuserDataを使用しないでください。AMI（イメージ）を使用する必要があります。
  • インターネット上でSSHを開いてはいけません。

この例は、aws-js-webserverの例に基づいて記述されています。

index.js：


"use strict";
 
let aws = require("@pulumi/aws");
 
let group = new aws.ec2.SecurityGroup("web-secgrp", {
    ingress: [
        { protocol: "tcp", fromPort: 22, toPort: 22, cidrBlocks: ["0.0.0.0/0"] },
        { protocol: "tcp", fromPort: 80, toPort: 80, cidrBlocks: ["0.0.0.0/0"] },
    ],
});
 
let userData =
`#!/bin/bash
echo "Hello, World!" > index.html
nohup python -m SimpleHTTPServer 80 &`;
 
let server = new aws.ec2.Instance("web-server-www", {
    instanceType: "t2.micro",
    securityGroups: [ group.name ], // reference the group object above
    ami: "ami-c55673a0"             // AMI for us-east-2 (Ohio),
    userData: userData              // start a simple web server
});
 
exports.group = group;
exports.server = server;
exports.publicIp = server.publicIp;
exports.publicHostName = server.publicDns;

これは基本的なPulumiプログラムです。EC2セキュリティグループとインスタンスを割り当てるだけです。ただし、ここでは、上記の3つのルールすべてに違反していることに注意してください。テストを書いてみましょう！

 // check 1:    Name.
        it("must have a name tag", function(done) {
            pulumi.all([server.urn, server.tags]).apply(([urn, tags]) => {
                if (!tags || !tags["Name"]) {
                    done(new Error(`Missing a name tag on server ${urn}`));
                } else {
                    done();
                }
            });
        });

通常のテストのように見えますが、注目に値するいくつかの機能があります。

  • , ”plan“ ( ”preview"). , , . , . — . , .
  • Pulumi- «», , apply, . then .
  • エラーメッセージにリソースのURNを表示するためにいくつかのプロパティを使用するため、関数pulumi.allを使用してそれらを組み合わせる必要があります。
  • 最後に、これらの値は非同期で計算されるため、コールバックdoneまたはプロミスリターンでMochaの組み込み非同期機能を使用する必要があります。

すべてを構成した後、単純なJavaScript値として入力データにアクセスできます。プロパティtagsはマップ（連想配列）であるため、（1）falseではなく、（2）がのキーであることを確認しますName。とてもシンプルで何でもチェックできるようになりました！

次に、2番目のチェックを記述します。これはさらに簡単です：

 // check 2:    inline- userData.
        it("must not use userData (use an AMI instead)", function(done) {
            pulumi.all([server.urn, server.userData]).apply(([urn, userData]) => {
                if (userData) {
                    done(new Error(`Illegal use of userData on server ${urn}`));
                } else {
                    done();
                }
            });
        });


最後に、3番目のテストを記述します。セキュリティグループに関連付けられたログインルール（多くの場合あり）と、これらのルールのCIDR範囲（多くの場合）を探しているため、これは少し複雑になります。しかし、管理しました：

    // check 3:    SSH,   .
        it("must not open port 22 (SSH) to the Internet", function(done) {
            pulumi.all([ group.urn, group.ingress ]).apply(([ urn, ingress ]) => {
                if (ingress.find(rule =>
                        rule.fromPort == 22 && rule.cidrBlocks.find(block =>
                            block === "0.0.0.0/0"))) {
                    done(new Error(`Illegal SSH port 22 open to the Internet (CIDR 0.0.0.0/0) on group ${urn}`));
                } else {
                    done();
                }
            });
        });

それで全部です。今度はテストを実行しましょう！
