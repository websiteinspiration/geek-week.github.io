.NET Core vs Node.js.



Der Zweck dieses Artikels besteht nicht darin, die beste der beiden Plattformen auszuwählen, sondern nach den Stärken und Schwächen jeder dieser Plattformen zu suchen. Beide Technologien sind in der Welt der Webentwicklung gut etabliert. Jeder von ihnen hat seine Fans, die gute Ergebnisse erzielt und ihre Erfolge im Internet geteilt haben. Das Netzwerk verfügt über zahlreiche Bewertungen zu den Vorteilen, die Entwickler beim Wechsel zu .Net Core oder Node.js erhalten haben. Versuchen wir es herauszufinden.

Ein bisschen Kontext


Ich arbeite in einem Unternehmen , das sich seit 2015 mit kundenspezifischer Entwicklung beschäftigt. Zuerst haben wir über .NET geschrieben, dann zu .NET Core gewechselt und dann den Stack schrittweise auf Node.JS geändert. Der Hauptgrund war die Geschwindigkeit der Entwicklung.

Allgemeine Information






.NET Core .NET Core ist ein Open-Source-Allzweck-Framework zum Erstellen plattformübergreifender Anwendungen. Ermöglicht das Erstellen von Anwendungen für verschiedene Betriebssysteme Windows, MacOS und Linux. Unterstützt x64-, x86-, ARM32- und ARM64-Prozessoren. Die erste offizielle Veröffentlichung wurde am 27. Juni 2016 veröffentlicht.

Node.js

Laut der offiziellen Website ist Node.js eine auf V8 basierende Softwareplattform. V8 wiederum ist Googles Open-Source-JavaScript-Engine (in C ++ geschrieben), die JavaScript in Maschinencode übersetzt. Mit Node.js können Sie Anwendungen unter macOS, Linux, SmartOS, FreeBSD, Microsoft Windows, AIX und Android erstellen. Die erste Veröffentlichung war im Jahr 2009.

Das Stackoverflow-System zum Wissensaustausch hat im vergangenen Jahr eine Bewertung der Beliebtheit von Frameworks veröffentlicht, die auf einer Umfrage unter mehr als 90.000 Entwicklern basiert. Die Umfrageergebnisse lauten wie folgt:


Node.js ist laut dieser Umfrage beliebter . Die Ergebnisse können sich jedoch ändern, wenn .Net 5.0 veröffentlicht wird, wodurch der zweite und der dritte Platz im Wesentlichen zusammengeführt werden, nachdem eine Gesamtzahl von Stimmen erhalten wurde, die derzeit mit Node.js vergleichbar ist.

Programmiersprache




Net Core

Meistens wird C # verwendet - eine objektorientierte Programmiersprache, die von Microsoft in den Jahren 1998-2001 entwickelt wurde. Durch strikte Eingabe können Sie Fehler in der Kompilierungsphase abfangen, eine schnellere Rückkopplungsschleife erhalten und Anwendungen mit weniger Fehlern schneller entwickeln. Mit dem Schlüsselwort "dynamic" können Sie die dynamische Eingabe emulieren.

Node.js

JavaScript ist eine interpretierte, objektorientierte Sprache mit dynamischer Typisierung. Es hat auch eine Reihe von Funktionen, die funktionalen Sprachen eigen sind.

Die Standardsprache für JavaScript ist ECMAScript. JavaScript ist einfach zu verwenden, aber im Gegensatz zu C # müssen viele Fehler zur Laufzeit behoben werden. Um diesen Nachteil zu beseitigen, können Sie TypeScript verwenden, das 2012 von Microsoft eingeführt wurde. TypeScript ist ein JavaScript-Add-On, das abwärtskompatibel mit JavaScript ist und in letzteres kompiliert wurde. TypeScript bietet jedoch auch nicht die gesamte Leistung der stark typisierten objektorientierten Sprache, die C # bietet. TypeScript ist untrennbar mit JavaScript verbunden, das eine strukturelle Typisierung aufweist. Daher kann es die Arbeit nur mit Typen imitieren, die C # ähneln.

Ein Beispiel für einen Fehler, der in der Montagephase nicht auftritt:

interface A {
    x: number;
}
 
let a: A = {x: 5}
 
let b: {x: number | string} = a; 
b.x = "unsound";
 
a.x.toFixed(0); // Not a function error occurs.

Das Problem mit diesem Code ist, dass der Typ von "ax" aufgrund von Zuweisungen eine Zeichenfolge ist und der Fehler nur zur Laufzeit auftritt.

Community verteilt Tools über NuGet- Pakete. Derzeit gibt es 205.000 Pakete. Für die Entwicklung sind Visual Studio und Rider am gebräuchlichsten, die die Anwendungsentwicklung vereinfachen und beschleunigen.

Node.js Das

Toolkit ist NPM mit 1.298.879 Paketen. Eine offizielle Entwicklungsumgebung wurde nicht angekündigt, aber die häufigste Verwendung von Visual Studio Code mit Plugins für die Entwicklung für Node.js.
Im Allgemeinen sieht es so aus, als hätte Node.js eine größere Auswahl an Tools und alternativen Paketen.

Skalierbarkeit




Betrachten wir nur eine Skalierbarkeitsoption - horizontal, d.h. Hinzufügen von Servern zur Bewältigung steigender Workloads.

Die Net Core-

Plattform bietet die Möglichkeit, Multithread-Anwendungen zu schreiben, sodass das Herunterladen eines separaten Servers etwas einfacher ist als bei Node.js. Für Bereitstellungs-, Skalierungs- und Ausfallsicherheitsdienste hat Microsoft das Azure Service Fabric SDK veröffentlicht. Sie können dieses SDK auf Ihren Servern in Azure oder AWS verwenden. Die Dokumentation finden Sie hier .

Node.js Eine

einzelne Instanz verwendet nur einen Thread. Um alle Serverkerne zu laden, müssen Sie das Modul „ Cluster “ verwendenOder externe Prozesskontrolle. Für die Skalierung über mehrere Server benötigen Sie einen Load Balancer (AWS ELB / NGINX). Es gibt auch Tools, mit denen Dienste und Datenverkehr verwaltet werden können, z. B. Istio.

Im Allgemeinen wird das Problem der Skalierung auf der Grundlage beider Technologien gelöst, die alle dafür erforderlichen Werkzeuge bereitstellen.

Performance




Die Leistung hängt von vielen Parametern ab, einschließlich der ausgewählten Technologien, auf deren Grundlage die Anwendung erstellt wird. Ausgewählte Szenarien spielen ebenfalls eine Rolle, deren Ergebnisse verglichen werden. Daher besteht die Aufgabe eines stärkeren Vergleichs der Leistung darin, die Bedingungen der Experimente, unter denen Messungen durchgeführt werden, auszugleichen. Ich habe die realitätsnahen Konfigurationen unter den verfügbaren Benchmarks ausgewählt , um ihre Ergebnisse zu vergleichen.

In allen folgenden Tests sind die Parameter gleich und haben die folgenden Werte: OS - Linux, DB - Postgres, Typ ORM - FULL (d. H. Kein Micro-ROM und kein direkter Zugriff auf DB).

Abfrage einer einzelnen Datenbank

In diesem Test ruft jede Abfrage eine Zeile aus der Tabelle ab, dann wird diese Zeile in JSON serialisiert und eine Antwort gesendet. Die Anzahl der gleichzeitigen Verbindungen liegt zwischen 16 und 512.



Mehrere Abfragen in der Datenbank

Bei diesem Test führt jede Abfrage mehrere Abfragen in der Datenbank aus und jede von ihnen ruft viele Zeilen ab. In diesem Test sind 512 kompetitive Verbindungen enthalten.



Aktualisieren von Daten in der Datenbank

Jede Abfrage ruft viele Zeilen aus der Datenbank ab, konvertiert sie in Objekte, ändert Objekte und speichert die Änderungen einzeln in der Datenbank. Anschließend werden die aktualisierten Objekte serialisiert und in der Antwort gesendet. In diesem Test sind 512 kompetitive Verbindungen enthalten.



Hello World-Test

Die Anfrage wird im Klartext „Hello, World“ beantwortet.



Laut Benchmarks ist Net Core schneller als Node.js, aber Sie sollten immer das spezifische Szenario und die verwendeten Technologien berücksichtigen.

Fazit


Zusammenfassende Bewertungstabelle für die oben diskutierten Elemente.



Es ist noch einmal erwähnenswert, dass es kein universelles Werkzeug für alle Gelegenheiten gibt. Wenn Sie ein Anfänger sind und schnell Anwendungen mit einer Benutzeroberfläche und einem Servercode erstellen möchten, ist Node.js eine gute Wahl. Es reicht aus, sich mit JavaScript zu befassen, und es kann eine einzige Sprache für die Server- und Clientseite werden.

Komplexe, umfangreiche Aufgaben lassen sich am besten mit .Net Core lösen. Dies reduziert in den meisten Fällen die Anzahl der externen Abhängigkeiten, was letztendlich die Unterstützung vereinfacht. Node.js hat eine gute Leistung, aber wenn Sie etwas mehr benötigen und Ihre Anwendung extremen Belastungen ausgesetzt ist, ist dies eine Gelegenheit, über die Verwendung von .Net Core nachzudenken.
