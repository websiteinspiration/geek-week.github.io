MicrosoftとIBM：Javaサポートの主な変更点

Javaコミュニティにとって予期せぬことに、MicrosoftはOracle Code OneカンファレンスでのDeveloper Advocateの代理としてIBMを上回りました。

かつて、IBMはおそらくJavaコミュニティのエンタープライズセグメントにおける支配的な勢力でした。それにもかかわらず、Oracle Code One 2019では、すべてがMicrosoftとIBMの対立において深刻な変化が待ち受けていることを示していました。

IBMは常にJava開発に多額の投資をしてきましたが、Microsoftは心配していませんでした。しかし、IBMがJavaテーブルを離れようとしているようで、Microsoftはこの場所に立つ準備をしています。

MicrosoftとIBM：役割の逆転


IBMがEclipse環境を作成しました。 IBMはFortune 500クライアントをWebSphereに「フック」しました。これにより、サーバー側Javaが広く採用されるようになりました。 Oracleが10年前にSun Microsystemsを買収したとき、IBMはSunのテクノロジーに大きな賭けをしました。 Oracleが興味深い提案をしていなかったら、IBMはおそらくJava言語を習得していたでしょう。

しかし、IBMの関心は時間とともに低下し、Webコンテンツ管理（WCM）ツール、サーバー、およびポータル管理ツールの奇妙なアップデートを提供することで、IBMはWebSphereユーザーを無視しました。現在でも、WCMエディターは複数のブラウザータブをサポートしていません。 [戻る]ボタンをクリックすると、サーバー管理パネルのWebインターフェースが壊れ、ポータル構成ツールが非常に古くなっています。 IBMが今年初めにWebSphereビジネスをHCL Industriesに売却したとき、業界の誰も驚かなかった。

IBMが長い間WebSphereとサーバー側のJavaを放棄していたことをユーザーは感じました。代わりに、Big BlueはAI、クラウドテクノロジー、およびその論争の的となることに焦点を当てていますWatsonブランドの各種ツール。もちろん、IBMがカンファレンスをJavaOneと呼んだときのようにOracle Code One 2019でそれほど代表を表明したわけではなく、これはMicrosoftとは対照的です。

Microsoft対IBM：Java開発者に向かう途中の人もいれば、放棄する人もいます


Microsoft .NETプラットフォームは常にJava EEと直接競合しており、Bill Gatesの旗の下にあるツールはどれほど妥当であるかに関わらず、Javaコミュニティから批判されることが予想されていました。 Java開発者の頭の中での評判にもかかわらず、過去18か月間、Microsoftはコミュニティを満足させるためにあらゆることを行ってきました。

マイクロソフトは2018年6月にAdoptOpenJDKをスポンサーしました。最近では、マイクロソフトがjClarityを購入し、JavaチャンピオンのMartijn VerburgとBen Evansがそれぞれマイクロソフトに参加しました。そして、独自の人気のある強力なバージョン管理システム-Team Foundation Serverにもかかわらず、Microsoft はGitHubに75億ドルを費やしました、多くのApacheプロジェクトと他のオープンソースJavaプロジェクトをホストする分散バージョン管理システム。

Microsoftサーバーは、Java EEアプリケーション向けではありませんでした。ただし、Dockerで実行されるクラウドネイティブJavaアプリケーションは、Microsoft Azureに簡単にデプロイできます。コンテナー化は、以前はアクセスできなかったエンタープライズJavaアプリケーションの市場をMicrosoftに開放しました。

Oracle Code One 2019は、マイクロソフトとIBMの対立のターニングポイントとして記憶されます。今年、マイクロソフトは真剣にJavaコミュニティに突破口を開き、部外者を重要なプレーヤーに変えました。

これを理解したのは、Javaチャンピオンの Code One での Kirk Pepperdine のパフォーマンスを見たときです。、常に独立しており、現在はマイクロソフトのチーフエンジニアであることを認識していました。Oracleの元Java EEエバンジェリストであるReza Rahmanも、BoFセッション（「Birds of a Feather」）でMicrosoftの代理を務めました。

MicrosoftがJava市場の企業セグメントに深刻な賭けをした一方で、IBMが他の方向に急落したことは明らかです。
