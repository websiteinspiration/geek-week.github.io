PythonとBalabolkaを使用した最も簡単な英語の単語シミュレーター

外国語一般、特に英語を勉強する方法はたくさんあります。しかし、方法が何であれ、単語を学ぶことは依然として必要です。

これらの目的のために、学習する単語を選択できる非常に多くの異なるシミュレータがあります。ただし、それらの機能が十分でない場合があります。

これらのシミュレーターの1つで、私はすでに十分な言葉を学びました。そして、一つの問題に直面しました。シミュレーターはロシア語の単語を提供します。それに応答して、この単語を英語で書く必要があり、アナウンサー（実際には音声合成装置）が英語版を発声します。私はすべての単語を完璧に書きますが、英語のテキストを見ると、この単語を教えたことを覚えているだけで、それが何を意味するのか覚えていません。つまり、言葉の認識が足りないことに気づきました。

選択したシミュレーターのもう1つの欠点は、空の位置が入力用に提供され、そこに文字を入力する必要があることです。また、単語内の文字数のヒントが常にあり、これはスポーツではありません。

これらのことを踏まえて、勉強した単語を繰り返すシミュレーターを作りたいと思いました。このシミュレータは、リスニングをトレーニングするために、翻訳の方向が英語-ロシア語で、英語のテキストを音声で操作する必要があります。

問題を解決する手順は次のとおりです。

  • 私は、必要なすべての単語を含む「word」と「translate」の2つの列からcsvファイルを準備しました。
  • ファイルとクリップボードの両方からテキストを音声で操作することを目的とした、よく知られた無料プログラムtts（text-to-speach）であるBalabolkaのコンソールバージョンをダウンロードしました。
  • 私は、Windows 10オペレーティングシステムで動作するコードをJupiter Notebookに書きました。

私はすぐにBalabolkaを使うようになりませんでした。最初はpyttsx3ライブラリを使用したかったのですが、pyttsx3.init（）パッケージを初期化するときに多くのエラーがクラッシュし、pyttsx3.init（ 'dummy'）を開始したときに音が出ませんでした。私はこのパッケージで相互の誤解を打破することに成功しなかったので、他のオプションを探す必要があり、私はバラボルカで成功しました。

私が得たコードはこれです：

import subprocess, clipboard
import pandas as pd

標準スタート。パンダがすべてを知っているなら、サブプロセスとクリップボードライブラリは私には馴染みのないものでした。サブプロセスパッケージは、コマンドラインをシミュレートして、Balabolkaのコンソールバージョンにアクセスします。クリップボードパッケージを使用すると、テキストをクリップボードにコピーし、そこから音声合成装置でテキストを読み取ることができます。

words = pd.read_csv ('C:/.../Python_Scripts/words.csv', delimiter=';')
words.head()

用意されたcsvファイルを言葉と翻訳で読みました。

words.shape

そして、私は常に変数の値、それらに何がどのように書き込まれるかを制御します。

words1 = words.sample(frac=1)
words1.head()

カードのようにファイル内の単語を混ぜて、単語のシーケンスを覚えないようにします。

words10 = words1.iloc[0:100]

場合によっては、単語ファイル全体を調べるのに十分な時間がないことがあり、ここで繰り返し単語の数を制限できます。

for i in range (len (words10)):
    path = r'C:\...\balcon\balcon.exe'
    flag = ' -n Slt -s -3 -c'
    clipboard.copy (words10['word'].iloc[i])
    subprocess.Popen(path + flag, creationflags=0, close_fds = True)
    print ("\033[1m {}\033[0m" .format(words10['word'].iloc[i]))
    tr = input ()
    print ("\033[1m {}\033[0m" .format(words10['translate'].iloc[i]))
    res = words10['translate'].iloc[i] == tr
    if res == True:
        print("\033[1m\033[36m {}\033[0m" .format(''))
    else:
        print("\033[1m\033[31m {}\033[0m" .format(''))
    i = +1

プログラム自体のテキスト。

パス変数では、Balabolkaのコンソールバージョンであるbalcon.exeファイルを呼び出します。

コンソールには、フラグ変数に書き込まれた独自の呼び出しパラメーターがあります。

具体的には、ここで私は使用します：

  • free voice -n Slt（音声シンセサイザーOlga Yakovlevaの開発者からダウンロードしました。私は必要な発音の明瞭さで英語の音声をピックアップしました。実際、音声は他のものでもかまいません）
  • 発音速度-s -3（このオプションは、発音速度-3を標準よりも遅くすることを意味します）
  • クリップボードからの読み取り-c（Balabolkaはファイルとクリップボードの両方からテキストを読み取ることができます。単語を含む何千ものファイルを作成するのは大変な作業なので、私はクリップボードを選択しました）。

プログラムの結果は次のようになります。



最初、テキストを強調表示しない長い出力の「ソーセージ」は完全に読めないように見えました（最初に英語の単語、次に私が入力した回答、次に正しい翻訳の出力、次に元のバージョンと入力したものを比較した結果、1単語あたり合計4行）。これにより、エラーの分析が非常に困難になりました。ANSIコードを使用して、後で間違いを犯した単語をすぐに見つけられるようにしなければなりませんでした。

この最も単純なシミュレータは、オリジナルのふりをしません。より高度なプログラムが多数作成されていますが、私自身のローカルの問題を解決するために、この結果は私に適しています。最初から最後まで制御できます。スクリプトは変更することができ、英語の単語のスペルを削除し、声だけを残して、聴くことができます。また、翻訳の方向を変更し、英語の単語のスペルをトレーニングすることもできます。これは、元のシミュレーターでは十分ではなく、この全体の話が始まりました。

この単純なスクリプトが、強力なプログラミングスキルを持たない人々が独自のシミュレーターを作成したり、このコードに基づいて何か他のものを実装したりするのに役立つとうれしいです。
