TelegramチャンネルからYouTubeプレイリストに何千ものクリップを追加する

YouTubeの視聴機能を備えたテレビを購入し、クリップが毎日配置される多くのTelegramチャンネルに登録したことで、携帯電話とテレビの間の不要な動きを取り除き、最新の最新コンテンツを視聴したいと思いました。



アイデアは簡単です：

  • 指定：最新のクリップを毎日アップロードする1つ以上のTelegramコンテンツメーカーチャンネル。
  • タスク：これらのすべてのクリップを使用してYouTubeでプレイリストを作成し、自動的に新しいクリップをキャッチします。
  • ツール：Google Apps Script（GAS）、YouTube API



電報チャネル履歴のエクスポート


YouTubeクリップで投稿を投稿する適切なテレグラムチャネルを探しています（たとえば、@ fresh_farshを使用）。

右上隅に3つのドット-「チャンネル履歴のエクスポート」があります。


リンクのリストの場合、必要なのはテキストだけなので、すべてのチェックマークを削除し、最大アップロード期間、つまり「最初のメッセージから現在の日付まで」を設定します。


ChatExportフォルダーは、いくつかの番号付きメッセージファイル（2、3、4など）の内部にアップロードされます。Html 、ファイルの数は、チャネルコンテンツの量によって異なります。


Googleスプレッドシートでチャネルを解析する


Googleスプレッドシートに追加するYouTubeリンクのリストを作成する必要があります。このサイトを選びました。

最上行に正規表現を入力します。

">(https:\/\/<a href="about:blank">www.youtube.com\/.*</a>?)<\/a>|">(https:\/\/youtu.be\/.*?)<\/a>

以下では、すべてのメッセージ（2、3、4）.htmlのテキストを続けて挿入します。右側のペインに結果が表示されます。

ボタンをクリックして結果をアップロードします。


「完全一致をエクスポートされたデータに含める」を押して、不要な一致がアンロードされないようにします。CSVを選択し、アンロードします。


Googleスプレッドシートを作成します。CSV

をコピーし、不要な行を削除して、すべてのリンクをyoutu.beに持ってきます

つまり、「アクセスできない/削除された」動画の割合を考慮すると、1日に約180のクリップを送信できます。

//     Telegram
function doPost(e) {
  //     JSON
  var inp = JSON.parse(e.postData.contents)
  //   .   ,  channel_post,      message
  if (typeof inp.channel_post === 'undefined'){
    var inpTxt = inp.message.text
  }else{
    var inpTxt = inp.channel_post.text
  }
  //   
  var y = /(?<=youtube.com\/watch\?v=)(.*?)$|(?<=youtu.be\/)(.*?)$/.exec(inpTxt)[0]
  try {
    YouTube.PlaylistItems.insert({
      snippet: {
        playlistId: idCHannel, //

          kind: "youtube#video",
          videoId: y // 
        }
      }
    }, "snippet")
      } catch (e) {

      }
}

ブロックをバイパスする方法

多くのプロバイダーでは、ブラウザーを介したテレグラムの操作に関連するすべてがブロックされています。したがって、https：//api.telegram.org/を呼び出すと、「サイトにアクセスできません」と表示されます。

ここではVPNが役立ちます。私はHotspot Shield Free VPN Proxy- Google Chromeの無制限VPN拡張機能を使用しました。

そのため、ボットはスクリプトに接続されており、ボットに何かを書き込むと、doPost関数がこれをキャッチして処理します。

JSONメッセージの入力構造はこちらです。

原則として、これらの機能の一部は一部の人々にとって十分です-私たちは多くのチャンネルをサブスクライブし、すべてを1つのプレイリストに解析してから、すべてのチャンネルからボットに新しい投稿を送信します。それらはすぐにプレイリストに追加されます。

自動

化プロセスを自動化するには、ここでプライバシーポリシーを読んだ後、管理者がチャネルにボットを追加します。

さて、チャンネルに新しい投稿があると、スクリプトは

JSON.parse(e.postData.contents).channel_post.text

eは入力パラメーターです。

ボットからそれぞれ送信する場合

JSON.parse(e.postData.contents).message.text

自由裁量で、マニュアルを使用して、クリップ、リプレイ、セルへのログインなどの存在のチェックを短縮できます。

GASは最初の開始時にすぐには処理されませんが、約10〜30秒後に処理されるため、ボットへの最初のメッセージはすぐには送信されません。
