ランダムアルゴリズム

この記事では、乱数を生成するための最も多様な自転車アルゴリズムを紹介します。

どの記事について


結果の品質と実行速度が異なる疑似乱数を生成するアルゴリズムについて。この記事は、プログラムで高性能の数値生成を実現したい人や、マイクロコントローラーやZXスペクトラムやMSXなどの古いプラットフォーム用のソフトウェアの開発者に役立つでしょう。

C ++ rand


初心者のC ++プログラマがランダムを取得することについて最初に学ぶのは、0とRAND_MAXの間の乱数を生成するrand関数です。RAND_MAX定数はstdlib.hファイルに記述されており、32'767ですが、これは、たとえばLinuxの場合とは異なる場合があります（コメントを参照）。コンパイラのrand（）が32'767（0x7FFF）以内の数値を生成し、大きなサイズの乱数を取得したい場合は、この問題の解決策として以下のコードを検討できます。

int64_t A = rand();
A <<= 15; //   15,   7FFF  15 
A |= rand();
A <<= 15;
A |= rand();
A <<= 15;
A |= rand();
A <<= 3;
A |= rand() & 0b111; //  3


古いCでのrand関数の実装は単純で、次のようになりました。

static unsigned long int next = 1;

int rand()
{
  next = next * 1103515245 + 12345;
  return (unsigned int)(next / 65536) % 32768;
}

この実装では数値の分布があまりよくなかったため、C ++で改善されました。また、標準C ++ライブラリには、乱数を取得するための追加の方法が用意されています。

C ++ 11 STLランダム


このさまざまなランダム性はC ++ 11で登場し、次のクラスセットで構成されています：minstd_rand、mt19937、ranlux、knuth_bおよびそれらのさまざまなバリエーション。

プログラムが起動するたびに乱数のシーケンスが繰り返されるのを防ぐために、疑似ランダムジェネレーターの「粒度」を現在の時間の形式で設定します。また、一部のレトロな（そしてゲームだけではない）ゲームの場合は、キーボード/ジョイスティックからのキーストロークの間隔を設定します。ランダムライブラリは、std :: random_deviceを使用して時間（NULL）よりも細かくすることを推奨しますが、Windows上のMinGWコンパイラの場合、関数は実際には期待どおりに機能しません。まだ…

PRNG-疑似乱数ジェネレータ


この名前は、線形合同法の同義語と考えることができます。PRNGアルゴリズムはCのrand実装に似ており、定数のみが異なります。

unsigned PRNG()
{
  static unsigned seed = 1; //     0
  seed = (seed * 73129 + 95121) % 100000;
  return seed;
}

PRNGアルゴリズムは多くの言語で高速かつ簡単に実装できますが、期間はそれほど長くありません。

Xorshift


アルゴリズムには多くのバリエーションがあり、周期と使用されるレジスタによって互いに異なります。XorShiftの詳細と種類は、WikipediaまたはHabréで確認できます。128度で2のシーケンスを持つオプションの1つを指定します。

struct seed_t
{
  unsigned x = 1; //     
  unsigned y = 123;
  unsigned z = 456;
  unsigned w = 768;
};

unsigned XorShift128()
{
  static seed_t s;
  unsigned t = s.x^(s.x<<11);
  s.x = s.y;
  s.y = s.z;
  s.z = s.w;
  s.w = (s.w^(s.w>>19)) ^ (t^(t>>8));
  return s.w;
}

このジェネレータは、除算と乗算の演算がまったくないという点で非常に優れています。これは、アセンブラの除算/乗算の命令がないプロセッサやマイクロコントローラ（PIC16、Z80、6502）で役立ちます。

z26エミュレーターの8ビットランダム


Z26は、古いAtari2600プレフィックスのエミュレーターです。このコードでは、1バイトのレジスターを操作するためにランダムに方向付けられています。

// P2_sreg - static uint8_t
void P2_Read_Random()
{
  P2_sreg =
    (((((P2_sreg & 0x80) >> 7) ^
       ((P2_sreg & 0x20) >> 5)) ^
      (((P2_sreg & 0x10) >> 4) ^
       ((P2_sreg & 0x08) >> 3))) ^ 1) |
        (P2_sreg << 1);
  DataBus = P2_sreg;
}

DOOMのランダムハウスジェネレーター


ゲームDoomのソースコードには、m_random.c （コードを参照）と呼ばれる興味深いファイルがあります。これは、「テーブル」のランダム性の関数を記述します。つまり、ビットシフトの数式や魔法はまったくありません。

この関数の動作を明確に示す、よりコンパクトなコードを示します。

const uint8_t random_map[] =
{
  4,  1,   63, 3,
  64, 22,  54, 2,
  0,  52,  75, 34,
  89, 100, 23, 84
};

uint8_t get_random()
{
  static uint8_t index = 0;
  index = (index + 1) & 0xF; // 0xF,      random_map
  return random_map[index];
}

RDRAND


最近のプロセッサの中には、RDRANDという単一のアセンブラー命令で乱数を生成できるものがあります。この関数をC ++で使用するには、アセンブラー挿入を使用して必要な命令を手動で書き込むか、GCCでimmintrin.hファイルを接続して、_rdrandXX_step関数のバリエーションを選択します。XXはレジスターのビット数を意味し、16、32、または64にすることができます。

#include <immintrin.h>

unsigned val;
_rdrand32_step(&val);

コンパイルエラーが表示される場合は、-mrdrndフラグを有効にしていないか、コンパイラ/プロセッサがこのツールをサポートしていないことを意味します。最速の乱数ジェネレーターかもしれませんが、暗号の強度について疑問があるので、考えてみてください。

エンディング


STLランダムライブラリのstd :: minstd_randクラスは、通常のrand（）よりも高速に動作し、minstdの期間の長さを特に気にしない場合は、代替となる可能性があります。これらの関数がWindowsとUnixでどのように機能するかは異なる場合があります。
