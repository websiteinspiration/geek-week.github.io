当社のPVS-Studioは、C、C ++、C＃およびJavaで記述されたプログラムのソースコードのエラーおよび潜在的な脆弱性を特定する静的コードアナライザーを開発しています。そして、私たちが若い専門家を引き付けることに興味があるのは誰にも秘密ではありません。プログラミングの分野で専門的な経験を積みたいと思っていて、まだこの分野の知識が不足している学生がいることがよくあります。さらに、新しく到着したスペシャリストも専門家としての成長を支援する必要があります。以上のことを踏まえ、プログラミング講座を作成し実施することとした。このアイデアは少なくとも3つの目標を追求しました。若い専門家やインターンに、コードをより正確に、より効率的かつ安全に書く方法を示すこと。C ++をオフィスの外で普及させ、その言語が学習の観点から有望であり、さらにそれで作業することを実証します。興味深い雇用者として当社を普及させるため。私たちの目標を決め、アイデアに触発されて、私たちは夢を現実のものにし始めました。

C ++ 20と近い将来


最新のC ++ 20標準の差し迫ったリリースについての話では、講義サイクルを終えることができませんでした。主な重点は、概念（Concepts）、範囲のライブラリ（Ranges）、コルーチン（Coroutines）、モジュール（Modules）、コントラクト（Contracts）、3者間比較演算子 <=>（宇宙船演算子）、機能をテストするためのマクロ（機能テスト）に置かれましたマクロ）、属性（Attributes）、std :: format、std :: jthreadなどなど...


そして、すべての組織的な問題が解決されたとき、そのアイデアを実現するときがきました。

実装


もちろん、この段階を「期待-現実」と呼びます。通常発生するように、計画段階で考えられるすべてのものが、実装段階で100万回以上変化するためです。当然、私たちはこの事実を認識しており、準備ができていました（いいえ）何かがうまくいかないこと。私たちの主な問題は、借りていた建物を積極的に修理していた下からの隣人であり、私たちのオフィスでは、講義の記録に影響を与えざるを得なかったドリルパンチングタイム（コメントなし）がありました。これに関して、コースの講義のほぼ半分が再度書き直されたか、ビデオの一部が撮影されました。アンドレイの場合、彼の講義は約1時間続き、会社が下からインストール作業を行わないのに十分な時間があるため、書き直す必要がなかった場合、フィリップの講義に苦しむ必要がありました。すべてのプログラマーは生き残り、すべての労働者が働くことなくそんなに待つことができるわけではありません。おそらくこの状況が最も予期されなかったものであり、そうでなければ深刻な問題はありませんでした。
