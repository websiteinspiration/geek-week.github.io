Huawei Dorado 5000V6のレビューとテスト


6か月前、私はすでにファーウェイのストレージライン（ドラドV6）の更新について話し始めていました。実際、公式発表の前に何とか彼らと知り合うことができましたが、その当時、私は彼らに自分の手で触れる機会がなかったのは当然です。私の記事「Huawei Dorado V6：Sichuan Heat」では、アーキテクチャの観点から見ると、その時点で最も興味深く、古いDorado 8000および18000 V6モデルに焦点を当てました。最後に、私は私たちの研究室で5000V6システムをテストし、これらのシステムの技術的な側面についてさらに話す機会を得ました。


一方で、これは5000V6システムのレビューとテストであり、もう一方では前の記事の論理的な続きです。過去6か月間で、システムのさまざまなコンポーネント、作業のロジック、および実装された機能についての詳細が表示されました。

しかし、要点を理解しましょう。このシステムはより身近なものである-デュアルコントローラーシステムなので、その姉妹の利点のいくつかを失っています。

  1. フロントエンドとバックエンドが共有されていない。

  2. また、5000V6および6000V6システムでは、2つのKunpeng 920プロセッサーのみが各コントローラーにインストールされ、1つは3000V6にインストールされています。
  3. 5000V6と6000V6に、再構築プロセスを高速化するように設計されたKunpengチップを備えた8000V6と18000V6と同じシェルフが提供され、これらのシェルフが100Gb RDMAを介して接続されている場合、より新しいモデルの3000V6にはSASシェルフしか使用できません。

  4. 5000V6および6000V6は、36個のPALM SSDまたは25 x 2.5インチの通常のSAS SSD（注文時にシステムを構成するときに選択）を取り付けるように設計されており、最新の3000V6は25個のSAS SSDのみをサポートします。
  5. 3000V6は、コントローラに最大3つのホットスワップ可能なインターフェイスカードの取り付けをサポートし、5000V6および6000V6は6つのカードをサポートします。

前の記事と同様に、エンドツーエンドのNVMeに関する脚注があります。RoCEv2上のNVMeとTCP / IP上のNVMeのサポートは近い将来に計画されています。

もちろん、アーキテクチャ上、デュアルコントローラシステムも異なります。


OceanStor Dorado V6は、以下のテクノロジーを備えたアクティブ-アクティブアーキテクチャを使用しています。

  • 負荷分散アルゴリズム：各コントローラーが受け取った読み取り要求と書き込み要求のバランスをとります。 
  • グローバルキャッシュ：LUNが所有者を持たないようにします。各コントローラーは、受信した読み取り要求と書き込み要求を処理し、コントローラー間の負荷分散を提供します。
  • RAID 2.0+：ストレージプール内のすべてのディスクにデータを均等に分散し、ディスクの負荷を分散します。

RAID 2.0+


SSDにデータが均一に保存されていない場合、負荷の高いSSDがシステムのボトルネックになる可能性があります。OceanStor Dorado V6はRAID 2.0+を使用して各SSDのすべてのLUNにデータを均等に分散し、ドライブ間の負荷を分散します。OceanStor Dorado V6は、RAID 2.0+を次のように実装します。

  • 複数のSSDがストレージプールを形成
  • 各SSDは固定サイズのチャンク（通常、チャンクごとに4 MB）に分割され、論理スペース管理を簡素化します
  • 異なるSSDのチャンクは、ユーザー定義のRAIDポリシーに基づいてチャンクグループを形成します
  • チャンクグループは、ボリュームの最小単位である「グレイン」（通常は8 KB）に分割されます。


ROWフルストライプ書き込み


SSDのフラッシュメモリチップは、限られた回数だけ消去できます。従来のRAID書き換えモード（write-to-place）では、SSDのホットデータは継続的に上書きされ、フラッシュチップはすぐに消耗します。OceanStor Dorado V6は、新しいデータと古いデータの両方に完全な双方向書き込み（ROW）記録を使用します。すべてのフラッシュチップが消去される回数のバランスを取りながら、レコードごとに新しいフラッシュチップを割り当てます。これにより、コントローラー自体のプロセッサー負荷と書き込みプロセス中のSSDの読み取り/書き込み負荷が大幅に軽減され、さまざまなRAIDレベルでのシステムパフォーマンスが向上します。


エンドツーエンドのI / O優先順位付け


特定のI / Oタイプの一貫したレイテンシを確保するために、OceanStor Dorado V6コントローラーは、タイプに応じて各I / O操作に優先順位を付けます。これにより、システムはCPUとその他のリソースを計画して優先順位を付けることができ、I / O優先順位に基づいてレイテンシが保証されます。特に、いくつかのI / O SSDを受け取った後、それらの優先順位を確認し、主に優先順位の高い操作を処理します。

OceanStor Dorado V6は、I / O操作を次の5つのタイプに分類し、降順で優先順位を付けて、最適な内部および外部I / O応答を提供します。

  • 読み取り/書き込み操作
  • 高度なI / O機能
  • 再請求
  • キャッシュをフラッシュする
  • ガベージコレクション


各ドライブでは、入出力操作に優先順位を付けることに加えて、OceanStor Dorado V6は、現在の書き込みおよび消去操作を中断するために優先度の高い読み取り要求も許可します。この場合、ディスクの読み取り遅延（キャッシュにデータがなかった場合）がホストの読み取り遅延に直接影響します。通常、SSDフラッシュドライブは、読み取り、書き込み、および消去の3つの操作を実行します。消去遅延は5ミリ秒から15ミリ秒、書き込み遅延は2ミリ秒から4ミリ秒、読み取り遅延は数十マイクロ秒から100マイクロ秒です。フラッシュチップが書き​​込みまたは消去操作を実行する場合、読み取り操作は現在の操作が完了するまで待機する必要があります。これにより、読み取り遅延が大幅に増加します。

スマートディスクエンクロージャー


前回の記事で書いたように、新しいディスクシェルフには独自のプロセッサとRAMが搭載されています。これにより、ディスク障害からの回復などのタスクをコントローラーから転送できます。これにより、ディスク障害が原因でデータが回復した場合のコントローラーの負荷が大幅に軽減されます。次の図は、例としてRAID 6（21 + 2）を使用した単一のディスクシェルフ内のデータ回復プロセスを示しています。ドライブD1に障害がある場合、コントローラーはD2-D21とPを読み取ってから、D1を再カウントする必要があります。合計で、21個のデータブロックがディスクから読み取られます。読み取りおよび復元操作は、大きなプロセッサリソースを消費します。


Smart Disk Enclosureシェルフは復元要求を受信し、データを読み取ってローカルでパリティデータを計算します。次に、パリティデータをコントローラーに転送するだけです。次の図は、コントローラーとディスクシェルフの間で転送する必要があるのは4つのパリティデータブロックだけであることを示しています。これにより、ネットワーク帯域幅が5分の1に削減され、回復操作のパフォーマンスが節約されます。


その結果、この負荷をシェルフ内のプロセッサーに転送すると、コントローラーの負荷を再構築タスクから10％未満に減らすことができます。

OceanStor Dorado V6は、ユーザーデータの特性に基づいて適応型重複排除と圧縮を自動的に実行し、効率を最大化します。重複排除と圧縮の適応プロセスは次のとおりです。

OceanStor Dorado V6は、データを圧縮する前に、独自の前処理アルゴリズムを使用して、データ形式に基づいてデータブロックで圧縮するのが難しい部分を特定します。 

OceanStorはデータを並べ替え、2つの部分に分割します。

  • 圧縮が難しい部分については、特別に開発されたHuawei圧縮アルゴリズムを使用して圧縮されます。
  • 他の部分では、システムは一般的な圧縮アルゴリズムを使用します

一部のストレージベンダーは、既に自社のデータ圧縮を使用しています。ただし、通常は4Kまたは1Kブロックを使用してデータを圧縮します。Huaweiはさらに進んで、バイトアライメントを使用しています。イラストははっきりしていると思います。

効果的な容量保証


新しいV6ストレージシステムを使用すると、圧縮と重複排除を備えた効率的なデータストレージに安全に依存できます。一般に、市場の多くのベンダーがすでに同様のプログラムを提供しています。これにより、より効率的なストレージにより、取得したデータストレージシステムの有用なボリュームに、より多くのデータを確実に保存できます。

SSDの各テラバイトのコストは依然として非常に高いため、圧縮および重複排除テクノロジーは非常に有用であり、多くのタイプのデータに対して高い効率を示します。データがビデオ、オーディオ、画像、科学データ、PDF、XML、または暗号化されたデータである場合、重複排除は効果的ではありません。そうでない場合、保証は機能します。最大数のドライブが取り付けられたシステムを購入した場合でも、プログラムに含まれるドライブとともに、拡張シェルフも受け取ります。
