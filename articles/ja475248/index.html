クロスブラウザーアプリケーションを作成する際のポリフィルの使用

最近、おかしな話が起こりました。私はWebプロジェクトを作成し、従業員が私の組織で使用する既存のアプリケーションの機能を拡張しました。すべてが順調に見えました。感謝の手紙を楽しみにして、プロジェクトが開始されたことを嬉しく思います。

最初のリリースから数日後、私は本当に手紙を受け取り始めました。しかし、彼らには感謝は見られませんでした。マネージャー、人事担当者、および私のプログラムを使用しようとしたすべての人が私に手紙を書きました。彼らのすべては彼らのアプリケーションが正しく動作しないと述べました。



何か問題でもありますか？しかし、実際には、プロジェクトを作成したときにChromeでテストしました。しかし、このプロジェクトのユーザーは常にFirefoxとIEを使用しています。私のアプリケーションでの作業も例外ではありませんでした。結局、数日前に開始されたプロジェクトが完成しなければならなかったのは本当に悲しいことでした。

実際のところ、ここでポリフィルが役立ちました。

ポリフィル


ポリフィル（ポリフィルまたはポリフィラー）は、開発者が標準のブラウザ機能の中で期待することを実装するコード（または特定のプラグイン）です。ポリフィルは、いわば、ブラウザAPIの不規則性を「滑らかにする」ことを可能にします。

Web環境では、ポリフィルは通常JavaScriptコードで表されます。このコードは、レガシーブラウザーに、これらのブラウザーがサポートしていない最新の機能を装備するために使用されます。

たとえば、ポリフィルを使用すると、Microsoft Internet Explorer 7のHTML Canvas要素の機能をシミュレートできます。これには、Silverlightプラグインが使用されます。ポリフィルを使用してrem、CSS の測定単位、またはテキストシャドウ属性をサポートできます。、またはその他。開発者がブラウザーの組み込み機能に注意を払わずにポリフィルのみを使用しない理由は、ブラウザーの標準機能がより優れた機能と高いパフォーマンスを提供するためです。さまざまなAPIの独自のブラウザ実装は、ポリフィルよりも多くの機能を備えており、高速に動作します。

ポリフィルは、異なるブラウザーが同じ機能を異なる方法で実装するという事実に関連する問題を解決するために使用される場合があります。このようなポリフィルは、非標準の機能を使用して一部のブラウザと対話し、他のJavaScriptプログラムが標準を満たす特定のメカニズムにアクセスできるようにします。今日ポリフィルを使用するこのような理由は、以前ほど重要ではなくなっていることに注意してください。ポリフィルは、IE6、Netscape、NNavの時代に特に流行し、各ブラウザはJavaScript機能を他のブラウザとは異なる方法で実装していました。

2つの理由により、一部のブラウザーで表示されるページにデータがありません。

  1. すべてのブラウザーがサポートしているわけではないプロミスとコールバックを使用しました。たとえば、そのようなブラウザにはIEやEdgeが含まれます。
  2. 配列メソッドを使用してflat()、既存の配列から新しい「フラット」配列を作成しました。このメソッドは、一部のブラウザではサポートされていません。その中には、前のケースと同様に、IEとEdgeがあります。

polyポリフィルの用途


Bluebird ライブラリを使用して、プロミスとコールバックの問題を修正します。これは、promiseに関連するメカニズムの本格的なJS実装です。Bluebirdライブラリの最も興味深い機能は、他のNodeモジュールを「オーバークラス化」して処理し、非同期で操作できるようにすることです。この種の処理は、コールバックを使用するコードに適用できます。

適切なCDNリソースを使用して、Bluebirdライブラリをページにダウンロードします。これを行うには、ファイルヘッダーindex.html（要素head）に以下を配置します。

<script src="https://cdnjs.cloudflare.com/ajax/libs/bluebird/3.7.0/bluebird.min.js"></script>

配列メソッドに関する問題を修正するにはflat()、script.js次のコードをファイルの先頭に追加します。

Object.defineProperty(Array.prototype, 'flat',
    {
        value: function (depth) {
            depth = 1;
            return this.reduce(
                function (flat, toFlatten) {
                    return flat.concat((Array.isArray(toFlatten) && (depth > 1)) ? toFlatten.flat(depth - 1) : toFlatten);
                }, []
            );
        },
        configurable: true
});

これで、アプリケーションはすべてのブラウザーで期待どおりに動作するはずです。たとえば、ここでは、Microsoft Edgeでの外観を示します。
