Windows Terminal Preview v0.9

Die Veröffentlichung von Version 0.9 von Windows Terminal hat stattgefunden. Dies ist die neueste Version des Terminals, die vor der Veröffentlichung von Version 1 neue Funktionen enthalten wird. Sie können Windows Terminal aus dem Microsoft Store oder von der Release-Seite auf GitHub herunterladen . Schauen wir uns die Update-Details genauer an!



Kommandozeilenargumente


Der Alias wt unterstützt jetzt Befehlszeilenargumente. Jetzt können Sie das Terminal mit neuen Registerkarten und Bedienfeldern starten, die nach Belieben getrennt sind, mit Profilen, die Sie mögen, beginnend mit Verzeichnissen, die Sie mögen! Die Möglichkeiten sind endlos! Hier einige Beispiele:

wt -d.
Öffnet ein Terminal mit einem Standardprofil im aktuellen Arbeitsverzeichnis.

Gewicht -d.; new-tab -d C: \ pwsh.exe
Öffnet ein Terminal mit zwei Registerkarten. Im ersten - dem Standardprofil, beginnend mit dem aktuellen Arbeitsverzeichnis. Das zweite ist das Standardprofil mit pwsh.exe als "Befehlszeile" (anstelle des Standardprofils "Befehlszeile"), das im Verzeichnis C: \ beginnt.

wt -p "Windows PowerShell" -d.; split-pane -V wsl.exe
Öffnet ein Terminal mit zwei vertikal getrennten Bedienfeldern. Das Profil mit dem Namen "Windows Terminal" wird im oberen Bereich gestartet, und das Standardprofil wird im unteren Bereich verwendet, wobei wsl.exe als "Befehlszeile" verwendet wird (anstelle des Standardprofils "Befehlszeile").

wt -d C: \ Benutzer \ Zimt \ GitHub \ WindowsTerminal; split-pane -p "Eingabeaufforderung"; split-pane -p "Ubuntu" -d \\ wsl $ \ Ubuntu \ home \ cinnak -H
Siehe unten.

PowerShell Autodiscover


Wenn Sie ein großer Fan von PowerShell Core sind , haben wir gute Neuigkeiten für Sie. Windows Terminal erkennt jetzt jede Version von PowerShell und erstellt automatisch ein neues Profil für Sie. Die PowerShell-Version, die unserer Meinung nach die beste ist (von der größten Versionsnummer über die GA-Version bis zur Version mit dem besten Bundle), wird als "PowerShell" bezeichnet und belegt den ursprünglichen PowerShell Core-Steckplatz in der Dropdown-Liste.



Bestätigen Sie das Schließen aller Registerkarten


Sind Sie derjenige, der nicht jedes Mal aufgefordert werden möchte, alle Registerkarten zu schließen? Wenn Sie mit Ja geantwortet haben, ist diese neue Funktion definitiv für Sie! Es wurde ein neuer globaler Parameter erstellt, mit dem Sie das Bestätigungsdialogfeld "Alle Registerkarten schließen" jederzeit ausblenden können. Setzen Sie dazu den Parameter "verifyCloseAllTabs" oben in der Datei profile.json auf false , und dieses Popup wird nie wieder angezeigt ! Vielen Dank an @ rstat1 für den Beitrag dieses neuen Parameters.
