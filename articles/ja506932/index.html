Cobaltサイバーグループの探索：ハッカーが使用する新しいツールと攻撃



PT Expert Security Centerの スペシャリストは、2016年から Cobaltグループの活動を監視しています。今日、グループは世界中の金融機関を攻撃しています。1 年前のこれらの攻撃による被害は10億ルーブルを超えました。過去4年間で、このグループに関連する攻撃のいくつかのレポートを公開しています。

昨年、Cobaltグループは、more_eggs JavaScriptバックドアと組み合わせて主要なCobIntツールとCOM-DLLドロッパーを変更しただけでなく、攻撃の初期段階で保護を回避するために新しい配信方法と新しい手法を使用しました。戦術とツールを更新することは、このグループが世界中の研究者によって長い間標的にされてきたという事実に起因する可能性があり、保護装置をバイパスするために一歩先を行く必要があります。

2019年には、平均して月に3回の攻撃が行われました。私たちの新しい記事-サイバーグループの新しい戦術とツールの分析。調査の完全版はこちらから入手できます。

フィッシングサイト欧州中央銀行


2019年8月の終わりに、情報セキュリティの脅威を調査しながら、CobIntを使用した攻撃を記録しました。攻撃はヨーロッパの金融機関を狙ったとされています。それがどれほど成功したかについてのデータはありません。 CobIntのドロッパーとして機能するカスタムNSISインストーラー。そのようなドロッパーの3つのバージョンを見つけました：Chrome、Firefox、Opera用。それぞれの内部には、同じバージョンのCobIntと、対応するブラウザーのインストーラーがありました。起動後、ドロッパーはCobIntを％TEMP％フォルダーに保存し、インストーラーもそれを起動しました。マルウェアを分析した結果、ドロッパーの配布はphbサイトのecb-european [。] Euから発生していることがわかりました。

2.悪意のあるVHD


2019年12月末に、次のCobInt Cobaltブートローダーを発見しました。異常なのは、このブートローダーを含むコンテナでした。郵送されたと思われる仮想ハードディスク（VHD）でした。

VHD形式は、Hyper-V仮想化テクノロジーの作成以来使用されています。 VHDファイルには、通常のHDDにあるすべてのもの（ディスクパーティション、フォルダーとファイルを含むファイルシステム）を含めることができます。 Windows 7以降では、MMCコンソールなどを使用して、VHDを手動で接続できます。 Windows 8以降、ユーザーはVHDをマウスでダブルクリックするだけで接続でき、システムに通常のディスクとして表示されます。

2019年9月、CERT / CCブログに記事が掲載されましたVHDファイルの危険性と、攻撃における潜在的なベクターとしてのその使用。ウィル・ドーマンは、インターネットからダウンロードされたVHDファイルのコンテンツがコンピューターに害を及ぼす可能性があることをユーザーに通知することは、ウイルス対策もWeb上のマークもしないことを示しました。コバルトグループがこの研究の結果を使用した可能性があります。それらのVHDファイルは、VirusTotalに表示された時点でもアンチウイルスによって検出されませんでした。 6か月後、このファイルは1つのアンチウイルスによって検出されます。これは非常に低い指標です。



VHDの内部には2つのCobIntファイルがあります。セキュリティ機能による検出の可能性を減らすために、2つの無効なGoogle証明書がいずれかのファイルの末尾に添付されています。

VHDがマウントされた後、ユーザーはファイルの1つを手動で開始する必要があります。機能的には、2つのファイルは同一です。CobIntの1つを開始すると、メインライブラリがロードされます。メインライブラリはCobIntによってコントロールサーバーからHTMLファイルとしてロードされます。

3. BIFFマクロ


2020年3月に、COM-DLLドロッパーをダウンロードして実行するCobaltグループドキュメントをXLS形式で記録しました。ドキュメントにはかなり古いExcel 4.0マクロ形式が含まれており、アンチウイルスによって実際には検出されませんでした（VirusTotalサービスによると、60のうち1の評決）。



Excel 4.0マクロファイルをVirusTotalに最初にダウンロードしたときのウイルス対策判定の数
このマクロ標準は20年間使用されています。その特殊性は、マクロがVBAプロジェクトに保存されておらず、ページのセルに配置されている一方で、Excel自体でページを非表示にできることです。

つまり、マクロはVBAストリームではなく、BIFFレコード（Binary Interchange File Format）にあります。
Excelでドキュメントを開くと、1ページしか表示されず、VBAプロジェクトにはマクロがありません。ただし、Excelは引き続きマクロの存在を検出し、その実行をブロックします。

数式マネージャーでソースドキュメントを開くと、ドキュメントを開くと、数式の1つが自動的に開始されます。



ドキュメントが開かれたときに開始するマクロ式。

最初の式は、COMBIN、EXECUTE、SYMBOL、CALLコマンドの大規模なチェーンを起動し、最終的にはCOM-DLLドロッパーのロードと起動につながります。コマンドチェーン全体がExcelページのセルの巨大なフィールドに分散しているため、分析が困難です。

4. COM-DLLドロッパーの分析


2020年4月初旬に、新しいバージョンのCOM-DLLドロッパーを発見しました。機能面では、これまでに見たものとはすべて異なります。ただし、more_eggs JavaScriptバックドアの形式のペイロードは同じままです。

COM-DLLドロッパーは、2017年の夏にCobaltアーセナルに登場し、more_eggs暗号化されアーカイブされた形式で含まれているJavaScriptバックドアを配信するためにグループによってまだ使用されています。

ドロッパーの機能：

  • 完全にPureBasicで記述されています。
  • 多数の異なる抗分析手法を使用します。
  • 暗号化およびアーカイブされたJavaScriptローダー、JavaScriptバックドア、more_eggs JavaScriptバックドアを起動するための正当なコマンドライン変換ユーティリティが含まれています。
  • JavaScriptバックドア用の組み込みの難読化ツールと、内部に接続されたJavaScriptローダーがあります。

結論


コバルトグループは、世界中の金融機関を攻撃し続け、新しい戦術を取り入れ、ツールを更新し、防御をより効果的に回避しようとしています。検疫により、多くの信用および金融部門の従業員は、企業の救済の範囲外で「遠くから」働いています。さらに、Higaisaグループなど、多くの攻撃者が攻撃にコロナウイルスのテーマを使用しています。Cobaltグループがこのテーマを将来の攻撃に使用しようとする可能性があります。
