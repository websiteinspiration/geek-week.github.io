Google Cloud Platformでホスティングする無料のテレグラムボット

なぜGCPなのか？


電報をボットに書き込むとき、ボットを常に迅速かつ無料で機能させる方法についての質問に遭遇しました。複数のボットがある場合、HerokuとPythonanywhereのオプションには制限が小さすぎます。そこで、GCPを使用することにしました。プラットフォームは、年間300ドルの無料+これらの資金の使用に対する大幅な割引（最大94％）を提供します。


ボットをホストする方法は？


ステップ1. GCPに登録する


GCP ウェブサイトにアクセスし、[ 無料で開始 ]をクリックします。

データとカードを入力します。ご自身が有料サブスクリプションをアクティブ化しない場合、カードからのお金は引き落とされません。


ステップ2.仮想マシンを作成する


登録後、サービスのメインページが表示されます。[リソース]セクションで[Compute Engine]タブを選択する必要があります。

新しいインスタンスを作成する必要があります。


データベースを仮想マシンにデプロイしない場合は、g1-smallを使用できます。それ以外の場合は、n1-standartをお勧めします。


また、OSを選択する必要があります。私はDebian GNU / Linux 9（stretch）を選びました。


すべてのVMが作成されました。従来の展開には1〜5分かかります。

ステップ3.仮想マシンを構成する


PCからSSH経由で接続するか、プラットフォームを介して対話できます。
これを行うには、SSHをクリックします。


Linuxターミナルが新しいウィンドウで開きます。


それではセットアップに移りましょう。まず、次のコマンドを入力します。

sudo apt-get update

最新のパッケージバージョンに関する情報を更新します。

入力後：

sudo apt-get install python3-setuptools
sudo apt-get install python3-pip

Python自体をインストールする必要はありません。すでに存在しています。

次に、必要なライブラリをすべてインストールする必要があります。微妙なニュアンスがあり、すべてのライブラリを2回インストールする必要があります。

pip3 install ‘name_of_package’

python3コマンドで使用するため 

sudo pip3 install ‘name_of_package’

systemdの場合。このユーティリティは、ボットを起動し、クラッシュした場合に再起動するのに役立ちます。
python3を介してボットを実行する最も簡単な方法ですが、切断するとオフになります。画面は使用できますが、ボットは再起動しません。ポートチェックでcrontabを使用することもできますが、このオプションはsystemdよりも複雑なようです。

ステップ4.ボットをサーバーに入力する


ボットを満たすには2つの方法があります。Gitを使いたくない場合は、ボットを.tarにアーカイブしてサーバーにアップロードするだけです。


その後、次のコマンドで解凍します。

tar -xvf yourfile.tar

これで、ボットはアーカイブの名前が付いたフォルダーに保存されます。

2番目の方法はGitを使用する方法です。使い方を知っている人にこれを説明する価値はないと思います。

コマンドでインストールした後：

sudo apt install git

VMにクローンを作成できます。

その後、systemdのセットアップに移りましょう。これを行うには、彼のディレクトリに移動します。

cd /etc/systemd/system

そして、bot.serviceファイルを作成します。

sudo nano bot.service

開いたウィンドウに次のように入力します。


[Unit]
Description=Telegram bot ' '
After=syslog.target
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/home/  /     
ExecStart=/usr/bin/python3 /home/  /     /bot.py

RestartSec=10
Restart=always

[Install]
WantedBy=multi-user.target

ファイルを閉じて保存します。コマンドCtrl + Xで閉じます。

その後、コマンドを順番に入力します。

sudo systemctl daemon-reload
sudo systemctl enable bot
sudo systemctl start bot
sudo systemctl status bot

すべてがうまくいくと、次のようなものが表示されます。


以上で、ボットは独立して動作します。私の記事がボットのホスティングに役立つことを願っています。

PS


1.ボットにエラーがないか確認する


systemdからボットを起動する前に、ボットを起動して動作をテストします。これを行うには、ボットのあるフォルダーに移動し、Pythonを実行します。

cd <  >
python3 bot.py

2. pyスクリプトを使用してファイルにエンコーディングを追加する


#!/usr/bin/env python
# -*- coding: utf-8 -*-

ファイルの先頭に挿入します。

3. systemdのエラー


ボットにエラーがないかチェックし、問題なく動作したが、systemdで起動したくない場合は、ログを見て、ファイルを調べることでエラーが何であるかを理解できます。

sudo nano /var/log/syslog

このファイルをダウンロードし、メモ帳++を使用してコンピューターに表示できます。

4.ボットの更新


ボットの新しいバージョンを追加またはアップロードする場合は、次のコマンドを入力します。

sudo systemctl stop bot

必要なすべての操作を提供します。次に、次のコマンドを入力して、再び機能するようにします。

sudo systemctl daemon-reload
sudo systemctl start bot
sudo systemctl status bot
