メモリフォレンジック-レベル2




このタスクでは、RAMのイメージが与えられ、マシンの名前を見つけるように求められます。このタイプのタスクを解決するために、ボラティリティフレームワークを使用します。次のコマンドで画像をロードし、画像へのパスをパラメーターとして指定します。まず、画像に関する情報を見つけます。

volatility -f ch2.dmp imageinfo



これにより、Windowsオペレーティングシステムのメモリイメージであることがわかります。つまり、レジストリからマシンの名前を見つけることができます。次に、プロファイルをパラメーターとして指定する必要があります。レジストリのキー値は、次の方法で表示できます。

volatility -f ch2.dmp --profile=Win7SP1x86_23418 printkey -K "ControlSet001\Control\ComputerName"



そして今、私たちはコンピュータの名前を見つけます。

volatility -f ch2.dmp --profile=Win7SP1x86_23418 printkey -K "ControlSet001\Control\ComputerName\ActiveComputerName"



コンピューターの名前が表示されます。引き渡し、ポイントを獲得します。



ログフォレンジック-ウェブ攻撃




このタスクでは、彼らは私たちにWebサーバーログを提供し、攻撃が行われたと言います。攻撃者がどのデータを学んだかを理解する必要があります。ログを開きましょう。



base64エンコードされたデータが注文パラメーターとして表示されます。それらをデコードするコードを書きます。

from re import *
from base64 import *

f = open('ch13.txt')
log = f.read()
f.close()

k = findall("action=membres&order=(.*?) HTTP", log)
[print(str(b64decode(i.replace("%3D", "=")), "utf-8")) for i in k]



少し理解できるように変換します。



したがって、パスワードの最初の文字の最初の2ビットを00、01、10、11と比較し、対応する一時停止を行います-0、2、4、6秒。2番目と3番目の2ビットでも発生します。その後、7番目のビットがチェックされ、0または1の等式に応じて、2秒と4秒の休止が行われます。

したがって、遅延によって、サーバーが返すビットを決定できます。



たとえば、最初のリクエスト間の時間差は6秒です。つまり、これらはビット11などです。このプロセスを自動化します。

from re import *
from base64 import *
import binascii

f = open('ch13.txt')
log = f.read()
f.close()

k = findall("action=membres&order=(.*?) HTTP", log)
dec_k = [str(b64decode(i.replace("%3D", "=")), "utf-8") for i in k]

t = findall("2015:12:1(.*?) \+0200]", log)
tim = [int(i.split(':')[0])*60 + int(i.split(':')[1]) for i in t]
tim = [tim[i+1] - tim[i] for i in range(len(tim)-1)] + [0]

password = ""

for i in range(0, len(tim), 4):
	c = ''
	for sec in tim[i:i+3]:
		if sec == 0:
			c += '00'
		elif sec == 2:
			c += '01'
		elif sec == 4:
			c += '10'
		elif sec == 6:
			c += '11'
	if tim[i+3] == 2:
		c += '0'
	elif tim[i+3] == 4:
		c += '1'
	
	password += chr(int(c,2))
	
print(password)

パスワードをお伝えします。
