Functional FizzBuzz on Scala

FizzBuzz is a well-known puzzle, playfully or not very asked at interviews, there are many options for implementation, even for such a simple game. There are even masterpieces like FizzBuzzEnterpriseEdition .


I bring to your attention another option, not quite Friday, but rather Saturday: FizzBuzz on Scala, functional style.


Task


For numbers from 1 to 100, you need to display


  • Fizz if the number is divisible by 3;
  • Buzz if the number is divisible by 5;
  • FizzBuzz, if the number is divided by 3 and 5;
  • otherwise the number itself.

Decision


A programmer should not so much solve a problem as create a tool to solve it

Let's start with divisibility


def divisibleBy(n: Int, d: Int): Boolean = n % d == 0

divisibleBy(10, 5) // => true

No, this will not suit us - after all, divisibility is not only a property of type numbers Int, we will describe divisibility in a general form, but for one we will make it an infix operator (Here and below, some of the cats library features are used ):


import cats.implicits._
import cats.Eq

implicit class DivisionSyntax[T](val value: T) extends AnyVal {
  def divisibleBy(n: T)(implicit I: Integral[T], ev: Eq[T]): Boolean = {
    import I._
    (value % n) === zero
  }
  def divisibleByInt(n: Int)(implicit I: Integral[T], ev: Eq[T]): Boolean =
    divisibleBy(I.fromInt(n))
}

10 divisibleBy 5 // => true
BigInt(10) divisibleBy BigInt(3) // => false
BigInt(10) divisibleByInt 3 // => false

Here are used:


  • type class " Integral" requiring the type " T" to be able to calculate the remainder of the division and have the value " zero"
  • type class " Eq" requiring the type " T" to be able to compare its elements (the operator " ===" is its syntax)
  • "T" extension methods & value classes, - ( dotty, )
