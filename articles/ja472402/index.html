RTMPビデオ通話ストリーミング


今日、ストリーミングプラットフォームは人気のピークにあります。何百万人もの才能のある人々が定期的に知識を共有しているためです。この市場が成長するにつれて、スマートフォンまたはラップトップのカメラを使用してライブビデオをストリーミングできる多くのアプリケーションがあります。おそらく、そのような機能をアプリケーションに導入することもできます。もしそうなら、私達は喜んで急いでいます-今Voximplantはあなたがこのプロトコルをサポートする任意のCDNにRTMP経由でビデオ通話を転送することを可能にします。

この機能は、新しいことによって可能になるVoxengineのモジュール- StreamingAgent。カットの下で、この統合を構成するための5つのステップを待っています。ようこそ！

ストリーミングは素晴らしいですが、まだベータ版です。このモジュールの奇妙な動作に遭遇した場合は、このことを覚えておいてください。さらに、support@voximplant.comまでお気軽にご連絡ください。

スキーム



モバイル/ WebアプリケーションまたはSIPからのビデオ通話がクラウドに到着し、ストリーミングCDNに転送されます。次に、CDNはオーディオとビデオのストリームをエンドユーザーに送信します-通常のように。
RTMPはH.264のみをサポートしているため、ストリーミング時にこのコーデックを明示的に指定することが非常に重要です。


必要になるだろう


  • TwitchやYoutubeなどのストリーミングサービスのアカウント。
  • Voximplantアカウント。お持ちでない場合、登録はここにあります。
  • Voximplantアプリケーション、スクリプト、ルール、および1人のユーザー。このチュートリアルでは、これらすべてを作成します。
  • ビデオとオーディオをキャプチャするWebクライアント-デモビデオチャットを使用してください。

1.ストリーミング設定


まず、クラウドスクリプトで使用する2つのパラメーターの値を取得する必要があります- ストリーム名/キーとサーバーURL。以下は、例としてTwitchとYouTubeを使用してこれらの値を取得する方法を示しています。

けいれん


アカウントにログインし、右上隅のプロフィール写真をクリックして、[ビデオプロデューサー]を選択します。


[設定]セクションで、[チャネル]を選択し、ページの上部にある[コピー]をクリックして、プライマリストリームキーを取得します。キーは、2要素認証がアクティブになっている場合にのみ使用できることに注意してください。この値をテキストエディターに貼り付けるか、このページを開いたままにして、後で値をコピーします。

サーバーのURLを見つけるには、ページstream.twitch.tv/ingestsに移動し、推奨サーバーの1つを選択して、最後のスラッシュなしでコピーします。例：rtmp：//live-sfo.twitch.tv/app

YouTube


アカウントにログインし、右上隅のアバターをクリックして、YouTube Studio（ベータ版）を選択します。


YouTubeチャンネルがない場合は、チャンネルの作成に関するダイアログが表示されます。[チャネルの作成]をクリックすると、作成されたチャネルのダッシュボードが表示され、24時間以内にストリーミングが可能になるという通知が表示されます。


YouTube Studioで、左側のメニューから[その他の機能]-> [ライブストリーミング]に移動します。新しいページが開き、その下部にエンコーダーセットアップブロックがあります。ここから、サーバーURLとストリーム名/キー値を取得する必要があります（またはページを開いたままにします）。



2. VOXIMPLANT設定


次に、Voximplant側から設定を行う必要があります。まず、アカウントにログインします：https : //manage.voximplant.com/auth。左側のメニューで、[アプリケーション]、[新しいアプリケーション]の順にクリックして、ストリーミングと呼ばれるアプリケーションを作成します。新しいアプリケーションに移動し、「スクリプト」タブに切り替えて、このコードでstartStreamスクリプトを作成します（8行目と9行目の前のステップの値を置き換えることを忘れないでください）：

require(Modules.StreamingAgent);

VoxEngine.addEventListener(AppEvents.CallAlerting, function (e) {
  const streaming = VoxImplant.createStreamingAgent({
    protocol: "RTMP",
    url: "rtmp://live-sfo.twitch.tv/app",
    streamName: "live_********************",
    keyframeInterval: 4
  });

  e.call.sendMediaTo(streaming);

  streaming.addEventListener(StreamingAgentEvents.Connected, function (e) {
    Logger.write("LOG: StreamingAgentEvents.Connected: " + e.reason);
  });
  streaming.addEventListener(StreamingAgentEvents.Disconnected, function (e) {
    Logger.write("LOG: StreamingAgentEvents.Disconnected: " + e.reason);
  });
  streaming.addEventListener(StreamingAgentEvents.ConnectionFailed, function (e) {
    Logger.write("LOG: StreamingAgentEvents.ConnectionFailed: " + e.reason);
  });
  streaming.addEventListener(StreamingAgentEvents.Error, function (e) {
    Logger.write("LOG: StreamingAgentEvents.Error: " + e.reason);
  });
  streaming.addEventListener(StreamingAgentEvents.StreamStarted, function (e) {
    Logger.write("LOG: StreamingAgentEvents.StreamStarted: " + e.reason);
  });
  streaming.addEventListener(StreamingAgentEvents.StreamError, function (e) {
    Logger.write("LOG: StreamingAgentEvents.StreamError: " + e.reason);
  });
  streaming.addEventListener(StreamingAgentEvents.StreamStopped, function (e) {
    Logger.write("LOG: StreamingAgentEvents.StreamStopped: " + e.reason);
  });

  e.call.answer();

  e.call.addEventListener(CallEvents.Disconnected, function (e) {
    Logger.write("LOG: terminating in 6 secs");
    setTimeout(VoxEngine.terminate, 6000);
  });
});

これは、ストリームを開始し、接続、接続失敗、StreamStartedなどのすべての可能な状態を監視する単純なスクリプトです。-スクリプトの詳細は少し低くなります。次に、アプリケーションの「ルーティング」タブに移動し、「新しいルール」をクリックする必要があります。streamRuleという名前を付け、startStreamスクリプトを指定して、デフォルトのマスク（。*）のままにします。


この段階で最後に行うことは、ユーザーを作成することです。[ユーザー]タブに移動し、[ユーザーの作成]をクリックし、名前（たとえば、user1）とパスワードを指定して、[作成]をクリックします。Webクライアントでの認証には、このユーザー名とパスワードのペアが必要です。


Voximplantの構成は完了しましたが、Webクライアントに進む前に、このシナリオでStreamingAgentモジュールがどのように機能するかを理解することを提案します。

3.シナリオ


StreamingAgentモジュールを使用すると、開発者はRTMPストリームを一般的な刺激プラットフォームに起動できます。モジュールの機能を使用するには、スクリプトの最初にモジュールを接続する必要があります。

require(Modules.StreamingAgent);


次に、CallAlertingイベントのハンドラーを追加して、Webクライアントからのビデオ通話を処理します（はい、すぐに対応します）。まず、ハンドラーにcreateStreamingAgentメソッドの呼び出しを追加します。メソッドには次のパラメーターがあり、それらのほとんどは必須で、オプションは個別にマークされています。

  • プロトコル -これまでのところRTMPのみがサポートされています。
  • url-サーバーのURL。手順1を参照してください。
  • streamName-ストリーム名/キー、ステップ1を参照。
  • applicationName（オプション）-liveNameなどのstreamNameの一部。選択したプラットフォームで必要な場合は、このオプションを使用してください。
  • keyframeInterval（オプション、秒）-ストリーム内でキーフレームが生成される頻度。指定しない場合、デフォルト値は2です。

上記のスクリプトリストでは、createStreamingAgentメソッドの呼び出しは次のようになります。

VoxEngine.addEventListener(AppEvents.CallAlerting, function (e) {
  const streaming = VoxImplant.createStreamingAgent({
    protocol: "RTMP",
    url: "rtmp://live-sfo.twitch.tv/app",
    streamName: "live_********************",
    keyframeInterval: 4
  });

ただし、次のようにkeyframeIntervalを省略してapplicationNameを使用できます。

VoxEngine.addEventListener(AppEvents.CallAlerting, function (e) {
  const streaming = VoxImplant.createStreamingAgent({
    protocol: "RTMP",
    url: "rtmp://a.rtmp.youtube.com/",
    applicationName: "live2",
    streamName: "somename"
  });

ストリーミングオブジェクトが作成された後、スクリプトは引き続きハンドラー内の呼び出しを処理します。次のステップは、Webクライアントから作成されたストリーミングオブジェクトに着信ビデオストリームを送ることです。

e.call.sendMediaTo(streaming);

次に、可能なすべてのストリーミングイベントをデバッグに使用します。ハンドラーはVoximplantセッションログに情報を送信するだけです。必要に応じて、このコードを使用することはできません。

イベントを処理した後、スクリプトは着信呼び出しに応答します。

e.call.answer();

最後に、Webクライアントがビデオの送信を停止したときに、ストリーミングの終了を正しく処理する必要があります。正式には、このハンドラーも使用できませんが、セッションはビデオの送信が停止してから60秒後にのみ終了します。この期待には意味がないため、Disconnectedイベントから開始してセッションを終了することをお勧めします。

e.call.addEventListener(CallEvents.Disconnected, function (e) {
    Logger.write("LOG: terminating in 6 secs");
    setTimeout(VoxEngine.terminate, 6000);
  });

スクリプトのロジックを理解したら、構成の最後の部分を開始する準備が整います。

4. Webクライアント


ビデオチャットの デモを複製またはダウンロードしてください。次に、リポジトリフォルダーに移動し、テキストエディター/ IDEでWebApp / JS / app.jsファイルを開きます。5行目でアプリケーションの名前を変更し、ファイルを保存します。


次に、ストリーミング時に使用されるコーデックを明示的に指定する必要があります。多くのプラットフォーム（TwitchやYouTubeを含む）はRTMPを使用しており、RTMPがH.264をサポートしているため、これを指定する必要があります。行262-266を参照してください。


コーデックが指定されていない場合、StreamingAgentEvents.StreamErrorイベントが発生し、エラーテキストが含まれます。
"Video codec mismatch. " + codecName + " granted, but should be H.264"

どのSDKを使用する場合でも、callメソッド呼び出しの場所で、クライアント側のコーデックを指定する必要があります。


5.ストリーミングを開始する


ローカルまたはWebサーバーでWebAppフォルダーを実行する必要があります。ローカルで実行するには、live-serverユーティリティまたはChromeアプリケーション用のWebサーバーを使用できます。Webクライアントを起動するときに、Voximplantアカウントの名前を指定する必要があります。次に例を示します。
http://127.0.0.1:8080/index.html#accname=johngalt

クライアントを起動した後、ステップ2のログインとパスワードのペアを入力します。
