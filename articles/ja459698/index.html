プログラマーが必要とするだけのセッション変数をOracle BI 12cに強制させる方法は？

Oracle BI 12cサーバーには、ユーザーセッションレベルで名前を指定せず、特定のユーザーのBIセッションを承認するときにリポジトリレベルでそれらの番号を明示的に記述せずに、一連のセッション変数を自動的に作成する興味深い機能があります。

物理層から次の表に入るスキームを作成します。

USERID     NAME       VALUE
---------- ---------- ---------------
weblogic   level      11
weblogic   status     FULL-TIME
Tom Kyte   status     PART-TIME
Tom Kyte   level      9
Tom Kyte   flow       cache

セッションレベルの変数を動的に作成できるクエリについて詳しく説明します。システムセッション変数USERを使用します。


SELECT NAME, VALUE
FROM RW_SESSION_VARS
WHERE USERID='VALUEOF(NQ_SESSION.USER)'


新しい変数を作成するのではなく、[行ごとの初期化]チェックボックスをオンにすることに注意してください（[キャッシュを使用]チェックボックスをオフにすることを忘れないでください）。



作成された変数を表示するには、単一のテキストフィールドを持つダッシュボードを作成します。


[b]@{biServer.variables['NQ_SESSION.USER']}[br/]
@{biServer.variables['NQ_SESSION.level']}[br/]
@{biServer.variables['NQ_SESSION.status']}[br/]
@{biServer.variables['NQ_SESSION.flow']}[/b]


weblogicユーザー用に2つの変数が作成および初期化されていることがわかります。


[b]@{biServer.variables['NQ_SESSION.USER']}=>weblogic[br/]
@{biServer.variables['NQ_SESSION.level']}=>11[br/]
@{biServer.variables['NQ_SESSION.status']}=>FULL-TIME[br/]
@{biServer.variables['NQ_SESSION.flow']}=> [/b]


ユーザーTom Kyteが3つの変数を作成して初期化したことがわかります。


[b]@{biServer.variables['NQ_SESSION.USER']}=>Tom Kyte[br/]
@{biServer.variables['NQ_SESSION.level']}=>9[br/]
@{biServer.variables['NQ_SESSION.status']}=>PART-TIME[br/]
@{biServer.variables['NQ_SESSION.flow']}=>cache[/b]


プログラマーが必要とするだけの数のセッション変数を作成する方法についての質問は、トレーニングセンターのOracle BI 12cリポジトリの構築に関するコースの学生から受けました。
