1. Oracle提供第二代

云Oracle的云基础架构提供了工具和体系结构，这些工具和体系结构可使用高级自动化和内置安全性来减轻威胁，从而帮助企业从本地无缝迁移到云，最终支持出色的迁移和成本效益。

专为创新而设计的Oracle云基础架构。这包括行业领先的可扩展性和可用性，集成的管理和控制以及端到端SLA支持的可靠性。 Oracle的云任务扩展为支持新技术，例如人工智能，机器学习，物联网，区块链和以人为本的界面。

Oracle公司客户已经从试验这些技术转移到将其部署到关键任务应用程序，建立新的业务模型并创造新的业务价值。

Oracle云基础架构解决了第一代云解决方案的关键挑战，这些解决方案并非旨在与由需要大量数据的工作负载或应用程序驱动的大型财务系统一起使用。第一代云解决方案建立在十年前的技术之上，在这些技术中，性能，安全性和迁移设置总是很晚。下一代Oracle Cloud Infrastructure专为满足当今企业的需求而设计，可提供更快，更可预测的性能，更好的价格和安全性以及与企业工作负载的更高兼容性。

作为第二代云产品的一部分，Oracle是IaaS，PaaS和SaaS服务的唯一提供商。独立的Oracle数据库服务使用与其他服务相同的高速网络，从而使您能够快速部署关键任务应用程序以支持持续的创新。

2.Oracle Cloud支持您的长期计划

Oracle Cloud Infrastructure是从根本上来说是第二代第二代开放云体系结构，可作为Oracle Cloud的基础。该基础架构经过专门设计，可提供可预测的性能，从系统核心到其前沿的安全性，以及支持关键和密集型工作负载所需的控制。

Oracle Cloud提供了计算，数据仓库，数据库，网络和平台服务所必需的功能和技术，这些功能和技术可提供可靠的业务成果，从而使您可以重新考虑数据中心的需求。

Oracle Cloud Infrastructure在端到端工作负载性能方面胜过 AWS（Andrew Reichman，“ Oracle在性能方面比Amazon Web Services更好，” Oracle，2018年9月18日）。

3.优化绩效的机会

Oracle IaaS提供了业内无与伦比的广泛功能-从第二代平台和针对不带操作系统（裸机）的平台的服务集到针对技术计算集群的远程直接内存访问（RDMA）。这种差异性确保Oracle云基础架构显示出可预测的性能和良好的云客户端隔离。
Oracle的云基础架构也为诸如Oracle自主数据仓库和Oracle自主事务处理之类的尖端创新打开了大门。

独立工作负载在旨在提供低延迟，高可用性，容错和一致性能的基础架构上最有效。这意味着您的数据存储在Oracle对象存储中，并且多个副本会自动复制，从而确保了高可用性和容错能力。 Oracle的自包含自我修复功能可确保您的数据保持“健康”，并且用户始终可以访问最新的已知副本。

4.深度保护

安全性是Oracle云基础架构中的关键设计原则。 Oracle提供了从系统核心到最前沿的保护，例如客户端隔离，数据安全，内部威胁检测及其高度自动化的消除。 Oracle基础架构将计算和网络资源隔离开来，以保护您的个人数据和流量免受其他用户的攻击。它还将您的代码，数据和资源与控制机器分开，从而帮助防止攻击者窃取或操纵云中的数据。

Oracle Cloud通过精细的客户端隔离提供了非常有限的攻击功能。内置防火墙，DDoS攻击检测器和加密的安全级别可主动检测和阻止威胁。客户可以在新边界上建立身份并使用自适应身份验证在用户活动表明较高风险级别时自动添加其他验证。最后，Oracle拥有业界最广泛的云和本地解决方案组合之一。

5.从Oracle优化工作负载

Oracle云基础架构提供了多种独特的功能和工具，可支持有效迁移和启动Oracle数据库和业务应用程序产品组合。移动Oracle应用程序所需的更改最少，从而减少了迁移到云的成本和时间。 Oracle的首要任务是提供最新的设备和技术，以不断提高客户的生产率。

Oracle数据库在存储和竞争对手的性能方面可扩展许多倍，每个实例可达到40 TB的容量和每秒数百万个I / O操作（IOPS）。当需要最大数量的I / O操作来执行最复杂的自治数据库工作负载时，Oracle提供业界最大数量的本地非易失性全闪存快速（NVMe）内存（每个实例高达51.2 TB）。在Oracle Cloud Infrastructure中启动Oracle Real Applications Clusters（Oracle RAC）群集可提供高可用性（HA）数据库，并在几秒钟内完成故障转移，将性能扩展到成千上万的I / O，并基于滚动补丁和更新实现流畅的操作。这项技术为生产数据库应用程序引入了新的云标准。

Oracle Exadata云使客户可以在与整个系统或整个机架相同的系统上管理的裸机计算和其他Oracle云基础架构服务在相同的扩展区域和虚拟云网络中运行Exadata外形尺寸计算机。管理工具包，并且可以通过相同的控制台/ API访问。

Oracle的云基础架构为关键的Oracle应用程序（包括Oracle E-Business Suite和Oracle PeopleSoft，Siebel和JD Edwards）以及自定义和相关数据库提供了一套迁移，配置和管理工具，可帮助客户加快向云的过渡。特别是，对于独立数据库，Oracle的云基础架构可以优化事务分析和事务处理工作负载。它专门设计用于运行ML工作负载，以基于人工智能提供信息，从而允许客户端连接多个数据源并组合关键数据以支持合理的业务决策。 Oracle的云基础架构也是新的高级服务的基础，包括集成，容器，微服务和区块链，并提供增强的可扩展性以支持物联网的部署。

6.为应用程序开发

提供全面支持 Oracle正在快速创新，并提供了一个开放的，基于标准的，完全集成的应用程序开发平台，使您可以低成本地基于API和移动设备创建，部署和管理现代的基于云的应用程序。此外，Oracle通过少量可修改的代码为容器和云开发提供支持。

Oracle AppDev平台提供了一个全面的DevOps环境，用于持续集成（CI）/连续交付（CD），对Java应用程序的深入诊断以及与SaaS和本地软件的简化集成。

7.最佳性价比

Oracle Cloud Infrastructure提供了当今市场上最好的价格/性能。部署在Oracle云基础架构中的工作负载通常需要较少的计算服务器和块存储量，从而降低了优化工作负载性能的成本。

Oracle的无阻塞云基础架构不仅可以帮助最大程度地减少延迟，还可以降低其他网络用户影响客户端环境的风险，并且得到业界最全面，财务安全的端到端SLA的支持。

8. Oracle云超越AWS

Oracle Cloud以最低的成本提供了最佳的性能，使其成为启动关键任务企业应用程序的Oracle Database行业标准的最佳选择。 AWS不提供Oracle RAC，Oracle自主数据仓库或Oracle自主事务处理，并且Amazon Aurora在AWS云中关闭用户，而Oracle数据库则提供并结合了不同制造商的灵活性。

与AWS等竞争性虚拟机相比，Oracle虚拟机和裸机之类的Oracle计算服务器可帮助公司节省大量成本（25-65％）。

与AWS相比，Oracle数据库速度更快，成本降低了多达66％，这使您可以更有效，更经济地基于业务数据获得结果（埃森哲，“企业工作负载满足云计算：了解云中的Oracle数据库” ”， “ 存储评论”，“ Oracle云基础架构计算裸机实例评论”）。



9. Oracle Cloud支持中小企业增长

Oracle Cloud旨在满足包括中小型企业在内的所有企业的利益。这是由于以下事实：小型企业正在变得越来越大，并且经常会在应用程序实施速度上大幅度提高。但是，它们可能仍然面临与大型企业相同的问题。 Oracle Cloud支持所有可能的方案：为开发人员和关键任务工作负载规划和使用传统和现代工具。

10. Oracle和Microsoft提供连接的云系统

甲骨文和微软之间的技术合作伙伴关系可用于相互之间的云交互，从而支持常规和关键任务应用程序。客户现在可以使用Oracle无缝资源（包括Oracle自主数据库）和Microsoft Azure的最佳功能（具有无缝兼容性）迁移到云或创建新的应用程序。

这种战略伙伴关系提供了一组新功能，使联合客户可以更轻松地过渡到云，包括：

  • 跨云连接，
  • 集成云服务
  • 协同支持模型
  • 应用程序部署的灵活性。

客户最大程度地提高性能的主要能力仍然是在Oracle云基础架构中运行Oracle应用程序。
Oracle Cloud提供了一个免费试用版，使您有机会探索具有自主管理，自我保护和自我修复功能的全球首个独立数据库。使用该试用版并发现虚拟机的最高性能和最具成本效益的计算实例。
