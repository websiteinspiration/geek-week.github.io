ユーザーのコホートをGrafanaのグラフの形式で収集する方法[+例のdocker画像]


Grafanaを使用して、Promopultサービスでユーザーのコホートを視覚化するタスクをどのように解決したか。


Promopultは、多数のユーザーを対象とした強力なサービスです。10年以上の運用で、システムの登録数は100万を超えました。同様のサービスに出くわした人は、この一連のユーザーが均一ではないことを知っています。


誰かが登録し、永遠に「眠りに落ちた」。誰かがパスワードを忘れて、6か月でさらに2、3回登録しました。誰かがキャッシャーにお金を持ってくると、誰かが無料のツールを探しに来ました。そして、みんなからいくらか利益を得ることは素晴らしいことです。


私たちのような大規模なデータ配列では、個々のユーザーの行動を分析してミクロな決定をすることは意味がありません。ただし、傾向を把握し、大規模なグループで作業することは可能であり、必要です。私たちが実際にやっていること。

SELECT
  DATE_ADD(CONCAT(s.RegistrationMonth, '-01'), INTERVAL s.MonthsDiff MONTH) AS time_sec,
  SUM(s.Users) AS value,
  s.RegistrationMonth AS metric
FROM (
  ##  ,  
  SELECT 
    COUNT(DISTINCT(b.userId)) AS Users, 
    DATE_FORMAT(b.Date, '%Y-%m') AS BillingMonth,
    DATE_FORMAT(u.RegistrationDate, '%Y-%m') AS RegistrationMonth,
    PERIOD_DIFF(DATE_FORMAT(b.Date, '%Y%m'), DATE_FORMAT(u.RegistrationDate, '%Y%m')) AS MonthsDiff
  FROM billing AS b LEFT JOIN users AS u ON b.userId = u.userId
  WHERE
    u.RegistrationDate BETWEEN '2018-01-01' AND CURRENT_DATE
  GROUP BY 
    BillingMonth, RegistrationMonth, MonthsDiff 
) AS s
GROUP BY 
  time_sec, metric
