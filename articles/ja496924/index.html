Webコンポーネントのルーティング

特に最も人気のある代表者と比較すると、Webコンポーネントがフレームワークに到達しないように見えることがあります。しかし、Webコンポーネントの概念に含まれている標準の実装によって不明瞭にならない最も差し迫った問題の多くは、ライブラリまたは少量のエレガントなコードを使用して固定するのに十分簡単です。これらの機能の1つは、フロントエンドルーティングです。この機能は常に必要なわけではなく、おそらく物議を醸すかもしれませんが、ほとんどのフレームワークでは、この機能が存在し、多くの人が使用しています。Webコンポーネントについてはどうですか？そして、navigoライブラリを使用してルーティングを実装しようとします。

Windowsでコンソールコマンドを適切に機能させるには、このシステムのgitディストリビューションからUnixユーティリティをインストールするか、新しいWSLおよびターミナル機能を使用します。

まず、プロジェクトを作成します。

mkdir testnavigo
cd testnavigo
npm init

それが解放されるまでEnterキーを何度も押してから

、ライブラリを配置します

npm i navigo --save

index.html

を作成してライブラリを接続し、アプリケーションのメインクラスを定義します。ここでは、すべての子コンポーネントのルーティングと制御のロジックをカプセル化します。

Webコンポーネントを使用すると、ブラウザーオブジェクトのツリーでクラスが割り当てられているツリーに要素が表示されたときに、フックを自動的に呼び出すことができます。その中で、ルーティングライブラリを、ライブラリgithubの例から直接コードで初期化します（実際には、モジュールロードでより美しく実行する必要があります）。

<script src="/node_modules/navigo/lib/navigo.min.js"></script>

<my-app></my-app>

<script type="module">

   class MyApp extends HTMLElement {

       connectedCallback() {
           let root = null;
           let useHash = true; // Defaults to: false
           let hash = '#!'; // Defaults to: '#'
           this.router = new Navigo(root, useHash, hash);
       }
    }
   customElements.define('my-app', MyApp);
</script>


ルート処理を追加します。

connectedCallback() {
   let root = null;
   let useHash = true; // Defaults to: false
   let hash = '#!'; // Defaults to: '#'
   this.router = new Navigo(root, useHash, hash);
   router
       .on('page-a', () => {
           this.innerHTML = '';
           this.insertAdjacentHTML('beforeend', '<page-a></page-a>');
       })
       .on('page-b', () => {
           this.innerHTML = '';
           this.insertAdjacentHTML('beforeend', '<page-b></page-b>');
       })
       .on('*', () => {
           this.innerHTML = '';
           this.insertAdjacentHTML('beforeend', '<div>Please click links above</div>');
       })
       .resolve();
}


ルートが変更されるたびに、コンポーネントは新しいコンテンツを追加して内部コンテンツを更新します。これらのタグを新しいページコンポーネントに登録する必要があります。

class PageA extends HTMLElement {
   connectedCallback() {
       this.insertAdjacentHTML('beforeend', '<div>This is a page A</div>');
   }
}
class PageB extends HTMLElement {
   connectedCallback() {
       this.insertAdjacentHTML('beforeend', '<div>This is a page B</div>');
   }
}
customElements.define('page-a', PageA);
customElements.define('page-b', PageB);


ページの最上部に、ナビゲーションを確認するためのリンクを追加します。

<a href="page-a" data-navigo>Page A</a>
<a href="page-b" data-navigo>Page B</a>


data-navigo属性を使用すると、バックエンドにリクエストを送信するのではなく、クリックしてルーターで処理できます。これは、たとえば次のように開始する時間です。

npx http-server


127.0.0.1を 開いてください：ブラウザーで8080またはそれがあなたに書き込んだもので、すべてが機能することを確認してください。



これで、必要に応じてすべてを個別のファイルで配布し、テンプレートを添付してページを動的にロードし、構成へのルートを作成できます。 Webコンポーネントはほとんどの作業をアプリケーションオブジェクトのライフサイクルで行います。フック切断コールバックでrazbindivatを開くイベントだけを忘れてはなりません（）
