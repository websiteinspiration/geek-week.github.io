Huawei TaiShan 2280v2の開梱


arm64アーキテクチャに基づくプロセッサを搭載したサーバーは、私たちの生活に注意深く入っています。この記事では、新しいTaiShan 2280v2サーバーの開梱、インストール、小規模なテストについて説明します。

開梱



サーバーは目立たない箱に入って来ました。ボックスの側面には、Huaweiのロゴと、パッケージおよびパッケージの看板があります。上には、サーバーをボックスから適切に取り外す方法の説明があります。開梱しましょう！



サーバーは帯電防止材料の層で包まれ、フォームの層の間に置かれます。一般に、サーバーの標準パッケージ。


小さな箱には、スライド、2つのボルト、2つのSchuko-C13電源ケーブルが入っています。スライドはシンプルに見えますが、後で説明します。


サーバーの上部には、このサーバーに関する情報と、BMCモジュールおよびBIOSへのアクセスがあります。シリアル番号は1次元バーコードで表され、QRコードにはテクニカルサポートサイトへのリンクが含まれています。

サーバーのカバーを取り外し、内部を確認します。

中は何ですか？



サーバーカバーは、プラスドライバーを使用するときに閉じてロックできる特別なラッチで保持されています。ラッチを開くと、サーバーのカバーがずれます。その後、カバーは問題なく取り外されます。



サーバーは、次の構成で、TaiShan 2280 V2 512G標準構成と呼ばれる既成の構成になります。

  • 2x Kunpeng 920（ARM64アーキテクチャ、64コア、基本周波数2.6 GHz）;
  • 16x DDR4-2933 32GB（合計512 GB）;
  • 12x SAS HDD 1200GB;
  • アイオニスタに基づくバックアップ電源を備えたAvago 3508ハードウェアRAIDコントローラ。
  • 4つの1GEポートを備えた2xネットワークカード。
  • 4つの10GE / 25GE SFP +ポートを備えた2xネットワークカード。
  • 2x電源装置2000ワット;
  • ラックマウント2Uエンクロージャ。
サーバーのマザーボードは、標準のPCI Express 4.0を実装しています。これにより、4基の25GEネットワークカードをフルパワーで使用できます。
送信されたサーバー構成では、16個のRAMスロットが空です。物理的には、Kunpeng 920プロセッサーは最大2 TBのRAMをサポートします。これにより、128 GBのメモリスティックを32個インストールでき、1つのハードウェアプラットフォームでRAMの合計量を4 TBに拡張できます。

プロセッサーには、独自のファンのない取り外し可能なラジエーターがあります。予想に反して、プロセッサはマザーボード（BGA）にはんだ付けされており、障害が発生した場合は、サービスセンターで特別な機器を使用してのみ交換できます。

次に、サーバーを組み立て直して、ラックマウントに移ります。

取り付け



まず、レールをラックに取り付けます。スライドは、サーバーが収まるシンプルな棚です。一方で、このソリューションは非常にシンプルで便利ですが、サーバーをラックから引き出さずにサービスを提供することはできません。


他のサーバーと比較して、TaiShanはフラットなフロントパネルと緑と黒の配色で注目を集めています。また、製造元はサーバーにインストールされている機器のマーキングに敏感であることにも注意してください。各ディスクスライドには、インストールされているディスクに関する必要な情報が示され、ディスクの番号を示すアイコンがVGAポートの下に表示されます。


前面パネルのVGAポートと2つのUSBポートは、背面パネルのメインVGA + 2 USBポートに加えて、メーカーからの素晴らしいボーナスです。背面パネルには、MGMTとマークされたIPMIのポートと、IOIOIとマークされたRJ-45バージョンのCOMポートもあります。

初期設定



初期設定中に、BIOSエントリ設定とIPMI設定が変更されます。Huaweiはセキュリティを促進するため、BIOSとIPMIは通常のadmin / adminパスワードとは異なるパスワードで保護されています。初めてBIOSに入ると、デフォルトのパスワードは強力ではなく、変更する必要があることを警告します。


Huawei BIOSセットアップユーティリティは、SuperMicroサーバーで使用されるAptioセットアップユーティリティとインターフェイスが似ています。ここには、ハイパースレッディングテクノロジ用のスイッチもレガシーモードもありません。


BMCモジュールのWebインターフェースは、予想される2つではなく3つの入力フィールドを提供します。ローカルログインパスワードまたはリモートLDAPサーバーを介した認証を使用して、インターフェースに入ることができます。

IPMIには、サーバー管理のための多くのオプションがあります。

  • RMCP;
  • RMCP +;
  • VNC;
  • KVM;
  • SNMP

デフォルトでは、ipmitoolで使用されるRMCPメソッドは、セキュリティ上の理由から無効になっています。KVMアクセスの場合、iBMCは2つのソリューションを提供します。

  • 「クラシック」Javaアプレット。
  • HTML5コンソール。


ARMプロセッサはエネルギー効率が良いと位置付けられているため、iBMC Webインターフェースのメインページで、このサーバーを使用して節約したエネルギー量だけでなく、大気中に到達しなかった二酸化炭素のキログラムを示すエネルギー効率ブロックを確認できます。
電源装置の印象的な電力にもかかわらず、アイドルモードでは、サーバーは340ワットを消費し、全負荷では440ワットしか消費しません。

使用する


次の重要なステップは、オペレーティングシステムのインストールです。arm64アーキテクチャーには多くの人気のあるLinuxディストリビューションがありますが、最新バージョンのみがサーバーにインストールされ、正しく機能します。以下は、実行に成功したオペレーティングシステムのリストです。

  • Ubuntu 19.10;
  • CentOS 8.1。
  • 単にLinux 9。
記事の準備中に、ロシアの企業Basalt SPOがSimply Linuxオペレーティングシステムの新しいバージョンをリリースしたというニュースが出ました。Simply LinuxはKunpeng 920プロセッサをサポートしていると述べられていますが、このOSの主なアプリケーションはデスクトップですが、サーバーでの動作を確認する機会を逃さず、結果に満足しました。
主な機能であるプロセッサアーキテクチャは、まだすべてのアプリケーションでサポートされているわけではありません。ほとんどのソフトウェアは、ユビキタスなx86_64アーキテクチャに焦点を当てており、arm64に移植されたバージョンは、ほとんどの場合、機能が著しく遅れています。
Huaweiは、CentOSベースの商用LinuxディストリビューションであるEulerOSの使用を推奨しています。これは、このディストリビューションが最初にTaiShanサーバーの機能を完全にサポートするためです。EulerOS- OpenEulerの無料バージョンがあります。
GeekBench 5やPassMark CPU Markなどのよく知られたベンチマークは、まだarm64アーキテクチャでは機能しません。したがって、パフォーマンスを比較するために、プログラムのアンパック、コンパイル、および数πの計算という形で「日常的な」タスクを実行しました。

x86_64の世界の競合他社が、Intel®Xeon®Gold 5218を搭載したデュアルソケットサーバーを採用しました。サーバーの技術的特徴を以下に示します。

すべてのテストは、Ubuntu 19.10オペレーティングシステムで実行されます。テストを実行する前に、すべてのシステムコンポーネントがfull-upgradeコマンドで更新されました。

最初のテストでは、「単一の順位」でのパフォーマンスを比較します。つまり、1つのコアで数πの1億文字を計算します。 UbuntuのAPTリポジトリには、この問題を解決するプログラムpiユーティリティがあります。

テストの次の段階は、LLVMプロジェクトのすべてのプログラムをコンパイルすることによるサーバーの完全な「ウォームアップ」です。コンパイルされたものとしてLLVM monorepo 10.0.0が選択され、build-essentialsパッケージで提供されるgccおよびg ++バージョン9.2.1がコンパイラーです。サーバーをテストしているため、アセンブリを構成するときに-Ofastスイッチを追加します。

cmake -G"Unix Makefiles" ../llvm/ -DCMAKE_C_FLAGS=-Ofast -DCMAKE_CXX_FLAGS=-Ofast -DLLVM_ENABLE_PROJECTS="clang;clang-tools-extra;libcxx;libcxxabi;libunwind;lldb;compiler-rt;lld;polly;debuginfo-tests"

これにより、コンパイル時に最大限の最適化が可能になり、さらにテストサーバーが読み込まれます。コンパイルは、使用可能なすべてのスレッドで並行して実行されます。

コンパイル後、ビデオのトランスコーディングを行うことができます。最も有名なコマンドラインユーティリティであるffmpegには、特別なベンチマークモードがあります。Ffmpegバージョン4.1.4がテストに参加し、高解像度のBig Buck Bunny 3D漫画が入力ファイルとして使用されました。

ffmpeg -i ./bbb_sunflower_2160p_30fps_normal.mp4 -f null - -benchmark

テスト結果のすべての値は、タスクを正常に完了するのにかかった時間です。

x86_64アーキテクチャの代表的なものの主な利点は、インテル®ターボブーストテクノロジーで達成された3.9 GHzの周波数であることは容易に理解できます。arm64アーキテクチャのプロセッサは、周波数ではなく、コアの数を使用します。

予想通り、1つのスレッドでπの数を計算する場合、コアの数は役に立ちません。ただし、大規模なプロジェクトをコンパイルすると、状況が変化します。

結論


物理的な観点から見ると、TaiShan 2280v2サーバーは使いやすさとセキュリティの点で注目に値します。PCI Express 4.0の存在は、この構成の別の利点です。

サーバーを使用する場合、arm64アーキテクチャのソフトウェアに問題がある可能性がありますが、これらの問題は特定のユーザーごとに固有です。
