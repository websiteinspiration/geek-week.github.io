Mis trabajos para Outline 2020

Del 22 al 24 de mayo, se lleva a cabo una votación en línea de demopati 2020. El evento se planeó para mayo de este año, pero se pospuso hasta octubre por todos los motivos conocidos.
Hablaré de mi trabajo


No recuerdo cuándo llamé la atención sobre el concurso de introducción pequeña de 128 bytes, pero cada año trato de participar. La preparación para la competencia se lleva a cabo de antemano, para el año 2020 los planes fueron de trabajo para el ZX Spectrum y Atari XE, pero más tarde los planes cambiaron. Comenzaré con Apple II:

HiRes Walk




Me gusta esta computadora, es atractiva porque tomará mucho esfuerzo resolver problemas simples. Las tareas fueron: reproducción de sonido, operaciones de disco, gráficos, intérprete SWEET-16.
Los gráficos para Apple II son dos modos: LoRes con una resolución de 40x48, 16 colores y Hires con una resolución de 280x192 6 colores.

Estudiar la estructura de la pantalla parecerá extraño, probablemente este formato se deba al hecho de que Wozniak, mientras diseña una computadora, guarda en todos los chips disponibles. Tratar con el modo de texto y LoRes fue fácil para mí, pero el modo HiRes fue un callejón sin salida. Cualquier operación con la pantalla requerirá secciones de código que no permitirán que la introducción se ajuste al tamaño de 128 bytes. Entonces decidí hacer trampa usando los procedimientos disponiblesy prueba tu mano. La idea era simple: dibuja una línea que dejará una marca.

Cualquiera que haya escrito algo similar a un juego de pong conoce el algoritmo. Para la introducción, necesita los datos X1, dx1, dxe / Y1, dy1, tinte. X1, Y1 - la coordenada actual, dx1, dy1 determinan la dirección del movimiento, dxe, tinte - establece el límite de la coordenada. Con este conjunto de datos, puede usar un procedimiento para las coordenadas (X1, Y1), (X2, Y2) o para la primera línea blanca.

Queda por hacer un rastro. La idea surgió por sí sola: escribí un programa, lo ejecuté 16 veces con el depurador y tomé los datos listos: un par (X3, Y3) y (X4, Y4) servirán para dibujar una línea negra. Resulta que los puntos (X1, Y1), (X2, Y2) se mueven a lo largo de una trayectoria dada, y (X3, Y3), (X4, Y4) los siguen, pero con cierto retraso. El trabajo final: transfirió los datos necesarios a la dirección 0, por lo que el tamaño del código se guarda debido al direccionamiento de la página cero.

Los perfeccionistas se quejarán de que la línea no toca la esquina derecha, pero obtendré la excusa preparada. La resolución de la pantalla es de 280 puntos, por lo que el código ha cambiado para manipular datos de 16 bits y la introducción ha crecido de 128 bytes a 256 bytes.

Halfton




Esta es una nueva versión del trabajo anterior , decidí cambiar la entrada de atributos por textura. 47 bytes aumentaron a 128, el resto fue ocupado por la generación y visualización de texturas. Hay un inconveniente: la introducción es lenta, pero no puede saltar más allá de 128 bytes.

Sparx




La idea de una introducción surgió por sí sola, decidí escribir código y ver qué pasa. La introducción es simple: se utiliza un modo de texto estándar con configuraciones adicionales. Atari tiene un modo de gráficos GTIA, se llama GRÁFICOS 9. Por cierto, en BASIC puede usar el mismo modo. Se activa configurando el bit 6 en la ubicación de memoria GRIOR (dirección 623). En este modo, los bytes de mordisco alto y bajo se interpretan como el valor del brillo del color (0-15).
Un ejemplo es el programa (Turbo BASIC).

Pero, además de la codificación, también hay un código de caracteres internos:


escriba TurboBasic
POKE DPEEK(88),1
en la esquina superior izquierda "!"

Esta numeración se utiliza para describir la fuente: 8 bytes para el espacio, los siguientes 8 bytes para "!". En BASIC, la dirección de la fuente se almacena en la celda 756 (CHBAS), debe multiplicar el valor por 256. Además, la dirección 756 es el valor sombra de $ D409.

Las fuentes se formaron para el modo seleccionado: 8 valores $ 00, 7 valores $ 11..8 valores $ FF. Tenemos caracteres con códigos 0-15 y con un brillo de 0-15.

El algoritmo de acción resultó ser simple: copiar línea por línea y disminuir el valor en 1 si se encuentra un código que no es igual a 0.

Smalevi4




El trabajo es una broma. Leí el artículo Serepindipitous Circles en BYTE Magazine, agosto de 1977, vol.2 N8, y decidí probar el código. Y nuevamente, resultó una pequeña introducción de 48 bytes. No quería mostrar el trabajo terminado, así que cambié el código y las opciones de dibujo para que se dibujara una figura similar con un cuadrado en el centro. Total 128 bytes.
