RESTリクエストからのデータに基づくAtlassian Confluenceのテーブル

こんにちは！

この記事では、アトラシアンConfluenceでテーブルを使用してページを作成する方法を説明します。データはRESTリクエストから取得されます。

Jiraのプロジェクトデータを含むテーブルを含むページをConfluenceに作成します。標準のJira REST APIのプロジェクトメソッドを使用して、Jiraからこのデータを取得します。

この記事のビデオはこちらからご覧いただけます。

Confluence用の電源スクリプトをインストールする


Confluence プラグインのパワースクリプトを使用して、Jira REST APIを呼び出します。これは無料のプラグインであるため、このソリューションでは費用はかかりません。

まず、ConfluenceにPower Scripts for Confluenceプラグインをインストールする必要があります。これを行う方法の詳細な説明は、ここにあります。

スクリプトを書く


次に、ギアメニュー項目->アプリの管理-> SILマネージャーに移動します。



次のコードでgetProjects.silファイルを作成します。

struct Project {
    string key;
    string name;
    string projectTypeKey;
}

HttpRequest request;
HttpHeader authHeader = httpBasicAuthHeader("admin", "admin");
request.headers += authHeader;
 
Project [] projects = httpGet("http://host.docker.internal:8080/rest/api/2/project", request);
runnerLog(projects);
return projects;

host.docker.internal：8080 /の アドレスをJiraインスタンスのアドレスに変更します。

スクリプトを実行して、データがJiraから選択されていることを確認します。

同時に、少しリファクタリングします。

スクリプトが行うことは次のとおりです。

  • Jira REST APIの呼び出しを通じてJiraからプロジェクトデータを取得する
  • 受け取ったプロジェクトデータを表形式のビューに変換します
  • 結果を印刷する

これは、コードで次のように表示されます。

Project [] projects = getProjectData();

return tableRows;

次に、プロジェクトデータの取得方法を指定します。

  • リクエストを作成する
  • Jiraからデータを受信するユーザーに関する情報を含むヘッダーをリクエストに作成します
  • 展開パラメーターをクエリに追加します。プロジェクトマネージャーに関するデータを選択する必要がありますが、デフォルトの回答ではそのようなデータはありません。したがって、応答でプロジェクトマネージャーに関するデータを表示することをJiraに指示する必要があります。これには、expandパラメーターが使用されます。
  • 私たちは要求を満たします
  • データを返す

しかし、言葉はコードに変わりました：

function getProjectData() {
    HttpRequest request;
    HttpHeader authHeader = httpBasicAuthHeader("admin", "admin");
    request.headers += authHeader;
    HttpQueryParam param = httpCreateParameter("expand", "description,lead,url,projectKeys");
    request.parameters += param;
    Project[] projects = httpGet("http://host.docker.internal:8080/rest/api/2/project", request);
    return projects;
}

次に、プロジェクトのデータ構造を定義しましょう。

プロジェクトメソッドがJira REST APIから返される答えは次のとおりです（不要なデータを削除して、答えを短くして読みやすくしました）。

[
   {
      "key":"KAN",
      "lead":{
         "name":"admin",
         "displayName":"Alexey Matveev",
      },
      "name":"kanban",
      "projectTypeKey":"software"
   },
   {
      "key":"SCRUM",
      "description":"",
      "lead":{
         "name":"admin",
         "displayName":"Alexey Matveev",
      },
      "name":"scrum",
      "projectTypeKey":"software"
   }
]

ご覧のとおり、key、name、projectTypeKeyフィールドの値は、j​​sonの最初のレベルで定義されています。しかし、リードフィールドでは、値の代わりにjsonが表示されます。そして、このjsonにはすでにnameフィールドとdisplayNameフィールドの値が含まれています。したがって、最初に、リード（リード）フィールドにjsonの構造を作成します。

struct Lead {
    string name;
    string displayName;
}

これで、json（プロジェクト）の第1レベルの構造を作成する準備ができました。

struct Project {
    string key;
    string name;
    string projectTypeKey;
    Lead lead;
}

しかし、問題は、SILテーブルマクロが1レベルのネストを持つjsonでのみ機能することです。そのため、2レベルのネスト（プロジェクト）を持つ構造を1レベルのネスト（フラット構造）を持つ構造に変換する必要があります。ただし、最初にフラット構造（TableRow）を作成します。

struct TableRow {
    string key;
    string name;
    string projectTypeKey;
    string lead;
    string leadDisplayName;
}

次に、Project構造のデータをTableRow構造に変換する関数を記述します。

function convertProjectDataToTableData(Project [] projectData) {
    TableRow [] tableRows;
    for (Project project in projectData) {
        TableRow tableRow;
        tableRow.key = project.key;
        tableRow.name = project.name;
        tableRow.projectTypeKey = project.projectTypeKey;
        tableRow.lead = project.lead.name;
        tableRow.leadDisplayName = project.lead.displayName;
        tableRows = arrayAddElement(tableRows, tableRow);
    } 
    return tableRows;
}
