Docker WebアプリのSSL証明書

この記事では、Docker Webアプリケーション用のSSL証明書を作成する方法について説明します。インターネットのロシア語部分-私は同様の解決策を見つけられませんでした。



カットの下の詳細。

docker v.17.05、docker-compose v.1.21、Ubuntu Server 18、1パイントのクリーンなLet'sEncryptがありました。Dockerに本番環境をデプロイする必要があるわけではありません。しかし、Dockerの収集を開始すると、停止が難しくなります。

そこで、まずは、開発段階での標準設定をお伝えします。一般に443ポートとSSLなし：

docker-compose.yml
version: '2'
services:
    php:
        build: ./php-fpm
        volumes:
            - ./StomUp:/var/www/StomUp
            - ./php-fpm/php.ini:/usr/local/etc/php/php.ini
        depends_on:
            - mysql
        container_name: "StomPHP"
    web:
        image: nginx:latest
        ports:
            - "80:80"
            - "443:443"
        volumes:
            - ./StomUp:/var/www/StomUp
            - ./nginx/main.conf:/etc/nginx/conf.d/default.conf
        depends_on:
            - php
    mysql:
        image: mysql:5.7
        command: mysqld --sql_mode=""
        environment:
            MYSQL_ROOT_PASSWORD: xxx
        ports:
            - "3333:3306"

次に、実際にSSLを実装する必要があります。正直なところ、私は2時間程度でcomゾーンIを勉強しました。提案されているオプションはすべて興味深いものです。しかし、プロジェクトの現在の段階では、SSLのLet'sEnctyptをnginxコンテナにすばやく確実に固定する必要がありました。

まず、サーバーにcertbotをインストールし、 次にドメインのワイルドカード証明書を生成しました
sudo apt-get install certbot



sudo certbot certonly -d stomup.ru -d *.stomup.ru --manual --preferred-challenges dns

certbotを実行すると、DNS設定で指定する必要がある2つのTXTレコードが提供されます。 

_acme-challenge.stomup.ru TXT {CertBot}

Enterキーを押します。

その後、certbotはDNS内のこれらのレコードの可用性をチェックし、証明書を作成します。
証明書を追加したがcertbotがそれを見つけられなかった場合は、5〜10分後にコマンドを再起動してください。

まあ、ここに私たちは90日間のLet'sEncrypt証明書の誇り高い所有者ですが、今それをDockerに送る必要があります。

これを行うには、最もありふれた方法であるdocker-compose.ymlのnginxセクションで、ディレクトリをリンクします。

SSLを使用したDocker-compose.ymlの例
version: '2'
services:
    php:
        build: ./php-fpm
        volumes:
            - ./StomUp:/var/www/StomUp
            - /etc/letsencrypt/live/stomup.ru/:/etc/letsencrypt/live/stomup.ru/
            - ./php-fpm/php.ini:/usr/local/etc/php/php.ini
        depends_on:
            - mysql
        container_name: "StomPHP"
    web:
        image: nginx:latest
        ports:
            - "80:80"
            - "443:443"
        volumes:
            - ./StomUp:/var/www/StomUp
            - /etc/letsencrypt/:/etc/letsencrypt/
            - ./nginx/main.conf:/etc/nginx/conf.d/default.conf
        depends_on:
            - php
    mysql:
        image: mysql:5.7
        command: mysqld --sql_mode=""
        environment:
            MYSQL_ROOT_PASSWORD: xxx
        ports:
            - "3333:3306"

ここで、一般的にポート443およびSSLで動作するようにnginx構成を変更する必要があります。

SSLを使用したMain.confの構成例
#
server {
	listen 443 ssl http2;
	listen [::]:443 ssl http2;

	server_name *.stomup.ru stomup.ru;
	set $base /var/www/StomUp;
	root $base/public;

	# SSL
	ssl_certificate /etc/letsencrypt/live/stomup.ru/fullchain.pem;
	ssl_certificate_key /etc/letsencrypt/live/stomup.ru/privkey.pem;
	ssl_trusted_certificate /etc/letsencrypt/live/stomup.ru/chain.pem;

      client_max_body_size 5M;

      location / {
          # try to serve file directly, fallback to index.php
          try_files $uri /index.php$is_args$args;
      }

      location ~ ^/index\.php(/|$) {
          #fastcgi_pass unix:/var/run/php7.2-fpm.sock;
          fastcgi_pass php:9000;
          fastcgi_split_path_info ^(.+\.php)(/.*)$;
          include fastcgi_params;
          fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
          fastcgi_param DOCUMENT_ROOT $realpath_root;
          fastcgi_buffer_size 128k;
          fastcgi_buffers 4 256k;
          fastcgi_busy_buffers_size 256k;
          internal;
      }

      location ~ \.php$ {
          return 404;
      }

      error_log /var/log/nginx/project_error.log;
      access_log /var/log/nginx/project_access.log;
}


# HTTP redirect
server {
	listen 80;
	listen [::]:80;

	server_name *.stomup.ru stomup.ru;

	location / {
		return 301 https://stomup.ru$request_uri;
	}
}


実際にこれらの操作の後-Docker-composeを使用してディレクトリに移動し、docker-compose up -dと書き込みます。また、SSLの機能を確認します。すべてが離陸するはずです。

主なことは、Let'sEnctypt証明書が90日間発行され、コマンドを使用して証明書を更新してから、コマンドを使用しsudo certbot renewてプロジェクトを再起動することを忘れないことです 。または、このシーケンスをcrontabに追加します。 私の意見では、これはSSLをDocker Webアプリに接続する最も簡単な方法です。 PSテキストに記載されているすべてのスクリプトが最終的なものではないことを考慮してください。現在、プロジェクトは深いDevの段階にあるため、構成を批判しないようにお願いします。構成は何度も変更されます。docker-compose restart
