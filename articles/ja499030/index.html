20分でisic上のGrafanaのMySQLパフォーマンスを監視する

こんにちは、親愛なるハブリャニンと、このおいしい健康的なモニタリングのための別のレシピ！

私の名前はミハイルアンドリアノフです。私はEmarsysでテクニカルディレクターを7年間務めています。

私たちはクライアントとデータベースを使用することが多いため、クライアントと私たちが平和に暮らし、作業できるように、それらを頻繁に監視する必要があります。

そして、MySQLの世界では非常に一般的な監視システムをデプロイするためのレシピを共有するというアイデアが私に生まれました。

この美しさの概観は次のとおりです。



すべてのダッシュボードは既にレシピに含まれており、彫刻する必要はありません。実際、すべてを正しく構成し、シックなグラフとテーブルの世界に真っ先に突入する必要があるだけです。

したがって、必要な準備のために：

Mysql> 5.6（それは可能であり、より若いですが、クエリで何かを修正する必要があります）
Grafana> 6.7.1
Ubuntu / MS Windows


第一段階


ログを保存するためのデータベースを設定し

、サービスベースの基礎として、便利で簡単に準備できるmy2Collector をイタリアのMeSql料理Meoのマスターから含めました

mysql --user=root -pXXX < my2.sql

*私は最愛のUbuntu 16のコマンドを使用します。

ユーザーにmy2管理者権限を与えることはできません-後で横に表示される可能性があります。

データベースがダンプからなくなったときは、データベース内のデータを更新する手順を理解することをお勧めします。特に、手順

 `collect_stats`()

リクエストの頻度を5分に増やすことをお勧めします。したがって、データベースの状態に関するより関連性の高いデータが得られます。

これで統計収集エンジンが実行されたので、このソースをGrafanaに接続することに進むことができます。

ステージ2


Grafanaをログでデータベースに接続します。



すべて標準です。MySQLデータソースを 次に

選択... URL、ユーザー、パスワード ** ログを収集し、読み取りアクセスperformance.schemaとmy2データベースへの書き込みのみを制限するために使用するユーザーを確認できます。グラファナ したがって、Grafanaへのアクセスを失った場合でも、ベースが危険にさらされることを回避できます。







ステージ3


ダッシュボードの

構成我慢強くならないように、私は故意に構成するための最も速くて最も便利な方法を選びました。

もう一度、メオに向けます。

彼のダッシュボードは2MySQL Simple Dashboardと呼ばれています。Grafanaでダッシュボード

を設定する最も便利な方法は、もちろんJSON設定オブジェクトをインポートすることです。

この簡単な手順を完了するには：

  1. JSON自体をダウンロードする
  2. Dashaboards->管理のダッシュボードの管理に入りましょう
  3. [インポート]を選択して、ダウンロードしたファイルからオブジェクトを読み込みます。





これで、非常に効率的なデータ視覚化を備えた既製の美しいパネルが完成しました。


Grafanaの柔軟性により、各パネルを簡単に変更することができます。2MySQLシンプルダッシュボードの組み込み変数は、たとえば、別のパネルまたはダッシュボードでデータベースの特定のユーザーを追跡することに最も関心のあるユーザーとプロセスにパネルをすぐに調整するのに役立ちます。



次のステップは、作成されたダッシュボードのアラートを設定することです。このプロセスは省略し、説明したパスに沿って進むことにしたすべての人の裁量に任せます。すべてのアラートは、特定の各パネルのレベルで構成されます。



約束どおり、展開は非常に迅速に行われ、タンバリンと一緒に踊ることはありません。特にalerts'ovに関して、経験豊富なhabrovtsevから私のレシピへの追加に喜んでいます。
