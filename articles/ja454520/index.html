クリーンなJavaScriptコードを書くためのガイドライン

コード自体とその記述方法に関心があり、動作するプログラムの作成だけに関心がない場合、これはコードをクリーンにすることを意味します。プロの開発者は、コンピューターだけでなく、将来このコードに出会った自分や他のプログラマーのためにコードを作成します。あなたが書いたコードは、コンピューターの腸内で永遠に消えることはありません。それは生き続け、変化し、そしてもしそれが不十分に書かれているなら、あなたがそれを書いた後にそれを編集しなければならないだろう誰かを非常に動揺させることができます。あなたはこの「誰か」になる可能性があります。 これらのアイデアに基づいて、クリーンなコードは、それ自体を説明するような方法で記述されたコードとして定義できます。このコードは人々が簡単に理解でき、簡単に変更または拡張できます。

4.条件付き構成


ブール変数に名前を付けないようにして、名前が否定されるようにします。ブール値を返す関数についても同様です。このようなエンティティを条件付き構文で使用すると、コードを読み取ることが難しくなります。

不完全に：

  // 
}

if (!isUserNotBlocked(user)) {
  // 
}

良い：

function isUserBlocked(user) {
  // 
}

if (isUserBlocked(user)) {
  // 
}

条件付き構文には短い形式を使用します。おそらくこの推奨事項はささいなことのように思えるかもしれませんが、言及する価値があります。あなたは、変数の値はならないことを確信しているということだけ論理変数のため、イベントでは、このアプローチを使用しますundefinedかnull。

}

  // - ...
}

良い：

  // - ...
}

if (!isValid) {
  // - ...

6.してはいけないこと


コードをきれいにしたい人は、それを繰り返さないようにしてください。ポイントは、同じコードを書かなければならない状況を避けることです。さらに、コードベースで決して実行されない未使用の関数やプログラムフラグメントを残す必要はありません。

さまざまな理由で重複したコードが表示される場合があります。たとえば、プロジェクトにわずかに異なる機能のペアがある場合があります。これらの違いの性質や時間の不足により、プログラマーは、たとえば、ほぼ同じコードを含む2つの独立した関数を作成する必要があります。このような状況で、重複コードを取り除くことは、違いを抽象化し、より高いレベルでそれらを操作することです。

次に、未使用のコードについて説明します。これはコードベースに存在するコードですが、まったく何もしません。これは、たとえば、開発の特定の段階で、プログラムの特定のフラグメントにもはや意味がないと判断された場合に発生します。このようなコードフラグメントを取り除くには、コードベースを注意深く確認して削除する必要があります。そのようなコードが不要になったと判断された時点で、そのようなコードを取り除くのが最も簡単です。後で、それが何に使用されたかを忘れることができます。これは彼との戦いを非常に複雑にします。

不要なコードとの戦いを延期すると、プログラムは次の図のようになります。
