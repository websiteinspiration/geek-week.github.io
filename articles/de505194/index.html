Windows Defender SmartScreen schadet unabhängigen Entwicklern

Angenommen, Sie sind ein unabhängiger Entwickler und haben Ihre Anwendung veröffentlicht. Um die Installation zu vereinfachen, erstellen Sie ein Installationsprogramm und beginnen mit der Verteilung. Einige mutige Experimentatoren laden herunter, starten den Installer - sehen aber eine strenge Warnung:


Windows SmartScreen blockiert den Start der ausführbaren Datei

In der aktuellen Umgebung blockiert Windows aktiv den Start von Binärdateien über die SmartScreen-Funktion.

Aber was ist ein Smartscreen?


SmartScreen sammelt Installationsdaten von allen Windows-Benutzern, um die „Reputation“ der Anwendung zu bestimmen. Wenn das Programm keinen guten Ruf hat, erhalten Sie eine solche Warnung. Die meisten Benutzer in dieser Situation löschen .exe, da es sich um ein Schadprogramm handelt. Sie können SmartScreen jedoch umgehen, indem Sie auf die Schaltfläche "Details" und dann auf "Trotzdem ausführen" klicken.

Digital signierter Schläger


Aber wie baut man einen guten Ruf auf? Zunächst sollte Microsoft in der Lage sein, Informationen darüber zu sammeln, wer die Anwendung veröffentlicht hat, und dies erfolgt mithilfe eines Codesignaturzertifikats. Die offensichtlichste Schlussfolgerung ist, dass nicht signierte Anwendungen immer SmartScreen aufrufen. Eine heimtückischere Schlussfolgerung ist, dass der Erwerb eines Codesignaturzertifikats für einen einzelnen Entwickler einen hohen Aufwand darstellt. Derzeit gibt es kein Äquivalent zu Let's Encrypt zum Signieren eines Codes. Sie müssen daher ein Zertifikat von seriösen Zertifizierungsstellen kaufen. Die Preisspanne ist breit, aber ein einjähriges Zertifikat kostet normalerweise etwa 100 US-Dollar.


SSL.com bietet eines der „billigsten“ Codesignaturzertifikate auf dem Markt zum Preis von 129 USD pro Jahr.

Angenommen, Sie haben an einem Haken gepickt, sich eine überschätzte Anzahl von Primzahlen gekauft, die mit einem Computer multipliziert wurden, Ihren Code signiert und Ihre Anwendung erneut veröffentlicht. Jetzt beginnen Benutzer mit der Installation Ihrer Anwendung, oder? Falsch.

Einen Ruf aufzubauen ist ein Trick 22


Selbst mit einer digitalen Signatur funktioniert SmartScreen weiterhin. Schließlich sind Sie ein unbekannter neuer Publisher, und der SmartScreen-Prozess zur Reputationsbildung ist eine absolute Black Box. Hier ist also der Trick 22: Um einen guten Ruf zu schaffen, müssen Mitarbeiter Ihre Software installieren - und Microsoft sammelt Daten mit Installationsstatistiken. Damit Benutzer Ihre Software installieren können, müssen Sie eine Meldung entfernen, die eindeutig darauf hinweist, dass Ihr Code ihre Computer beschädigt.

Weiter schlimmer.

Wenn Sie versuchen, Ihre Software mit dem WinGet-Paketmanager zu veröffentlichen (kurz gesagt: Wenn die Beta-Version die Beta-Version verlässt, ersetzt dieser aus AppGet kopierte Manager wahrscheinlich den Windows Store), erhalten Sie die folgende Meldung von Microsoft:


Microsoft WinGet betrachtet das Starten von SmartScreen als "Fehler" und Ihre Software wird als "bösartig" eingestuft.

Laut Microsoft wird Ihre Anwendung als bösartig angesehen, wenn SmartScreen gestartet wird. Sie empfehlen, die Datei zur Überprüfung über einen speziellen Link zu senden. Wenn Sie diesem Vorgang folgen, erhalten Sie die folgende Antwort:

Und dann noch schlimmer.

Angenommen, Sie werden schließlich von den Algorithmen als zuverlässiger Herausgeber anerkannt. Ihr Zertifikat läuft ab und Sie erneuern es. Eine häufige Sache im Internet. In der Welt der Windows-Software wird Ihr Ruf jedoch zunichte gemacht. SmartScreen erkennt keine Zertifikataktualisierungen, betrachtet Sie als neuen Herausgeber und muss diesen schmerzhaften Prozess erneut durchlaufen.

Dies kann durch den Kauf eines Zertifikats mit einer längeren Gültigkeitsdauer gemildert werden. Dann werden die Kosten für die meisten unabhängigen Entwickler zu einer unüberwindlichen Barriere.

EV-Codesignaturzertifikate


Für unabhängige Entwickler ist dies das Ende des Weges. Für Verlage gibt es eine andere Lösung: ein erweitertes Verifizierungszertifikat für die Codesignatur (EV). Wenn Sie den Code mit einem von ihnen signieren, umgehen Sie SmartScreen automatisch. Es ist nur Unternehmen vorbehalten und bietet, wie der Name schon sagt, eine zusätzliche Überprüfung der Informationen. Es wird von Zertifizierungsstellen durchgeführt.

Die beliebte Zertifizierungsstelle Digicert.com verkauft solche Zertifikate für satte 699 US-Dollar pro Jahr.

Ein unabhängiger Entwickler muss die Schwierigkeiten bei der Gründung einer juristischen Person durchstehen und diese Zertifikate zu unverschämten Preisen anbieten. Für die meisten ist dies natürlich nicht realistisch.

"Entwickler, Entwickler, Entwickler!"




"Entwickler, Entwickler, Entwickler!" Rief Steve Ballmer von der Bühne. Eine der legendären Reden des Microsoft-CEO. Diese berüchtigten Worte wurden bereits 2006 gesprochen. Vierzehn Jahre später wurde Microsoft bei Sathya Nadella dafür gelobt, dass das Unternehmen offener als je zuvor war. Ironischerweise stellt sich heraus, dass Microsoft seine Windows-Umgebung gegenüber seinen Lieblingsentwicklern äußerst feindlich eingestellt hat. Um diese schreckliche Situation zu beheben, müssen Sie SmartScreen oder die Funktionsweise der Codesignaturzertifikate ändern.
