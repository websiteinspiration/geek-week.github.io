インターネット上で情報を送信する最大単位が1500バイトになったため



イーサネットは至る所にあり、何万ものメーカーがそのサポートを備えた機器を製造しています。ただし、これらのデバイスのほとんどすべてに1つの共通番号-MTUがあります。

$ ip l
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 state UNKNOWN
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
2: enp5s0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 state UP 
    link/ether xx:xx:xx:xx:xx:xx brd ff:ff:ff:ff:ff:ff

MTU（Maximum Transmission Unit）は、個々のデータパケットの最大サイズを定義します。一般的なケースでは、LAN上のデバイスとメッセージを交換する場合、MTUのサイズは約1500バイトであり、インターネット全体もほぼ1500バイトのサイズで機能します。ただし、これは、これらの通信技術がより大きなパケットを送信できないことを意味しません。

たとえば、802.11（WiFiとして広く知られている）では、MTUは2304 bであり、ネットワークでFDDIを使用している場合、MTUは4352 bです。イーサネット自体には、「巨大なフレーム」という概念があります。MTUに最大9000 bpsのサイズを割り当てることができる場合（このモードのNIC、スイッチ、ルーターをサポート）。

ただし、インターネットでは、これは特に必要ではありません。メインのインターネットトランクは主にイーサネット接続であるため、他のデバイスでのパケットの断片化を回避するために、事実上の非公式の最大パケットサイズは1500 Bに設定されています。

1500という数値自体は奇妙です。たとえば、コンピュータの世界の定数は2の累乗に基づくと予想されます。それで、1500 Bはどこから来たのですか、なぜそれらをまだ使用しているのですか？

マジックナンバー


イーサネットの世界への最初の大きな進歩は、標準の10BASE-2（シン）と10BASE-5（シック）という形で起こりました。これらの数字は、単一のネットワークセグメントが何百メートルをカバーできるかを示しています。

当時は多くの競合するプロトコルがあり、鉄には独自の制限があったため、フォーマットの作成者は、パケットバッファのメモリ要件がマジックナンバー1500の出現に役割を果たしたことを認めています。

振り返ると、より大きな最大値がただし、最善の解決策は、初期段階でNIC（ネットワークコントローラー）のコストを増加させた場合、イーサネットがそれほど広がらないようにすることです。

ただし、これだけではありません。では仕事「イーサネット：ローカルコンピュータネットワークでの分散パケットスイッチング」1980。ネットワークで大きなパケットパケットを使用する効率の最も早い分析の1つ。当時、これはイーサネットネットワークにとって特に重要でした。これは、すべてのシステムを1本の同軸ケーブルで接続できるか、1つのパケットを一度に1つのセグメントのすべてのノードに送信できるハブで構成できるためです。

セグメントでメッセージを送信するときに遅延が大きくなりすぎないように（場合によってはかなりビジーになる）、同時にパケット数をあまり増やしない数を選択する必要がありました。

どうやら、当時のエンジニアは、1500 B（約12000ビット）を最も「安全な」オプションとして選択していました。

それ以来、他のさまざまなメッセージングシステムが出現したり、姿を消したりしましたが、その中でイーサネットは、1500 B以来最低のMTU値でした。ネットワークの最小MTU値を超えると、パケットの断片化が発生するか、PMTUDに従事することになります[最大パケットサイズを検索する選択されたパス]。どちらのオプションにも固有の問題がありました。主要なOSメーカーがMTU値をさらに低くする場合があります。
