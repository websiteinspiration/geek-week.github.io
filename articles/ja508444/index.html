OpenNebulaでのLXDのインストールと構成

Linux Administratorコースの開始に先立って、記事の翻訳が準備されました。









この記事では、LXDでのOpenNebulaのクラウドベースのオーケストレーションシステムの基本について説明します。最初に、シンプルなMiniONEツールを使用して、OpenNebula機能を試してみたいユーザーのために、物理マシンまたは仮想マシンにシングルノードクラウド環境を作成し、次に、GUIとCLIを使用して通常の作業セッションを確認し、最後に、分散した本番環境の作成に役立つリンクをいくつか示します。 。



MiniONEによる自動導入



MiniONEは、物理ホストまたは仮想マシンをシングルノードクラウドとして構成して、シンプルだが完全に機能するテストケースを迅速に展開するツールです。



LXDを評価するには、MiniONEには、Ubuntu 18.04または18.10のフレッシュインストールと、少なくとも2GBのRAMと20GBの空きディスク領域、および特権ルートアクセスを備えた専用の仮想マシンまたは物理ホストが必要です。たとえば、MiniONEを使用すると、Amazon仮想マシンでLXD / OpenNebula環境を簡単に構築できます。推奨される最小サイズは最も可能性が高いt2.medium



です。シンプルに25 GBのディスクスペースを与え、WebUIが実行されているTCPポート9869にアクセスします。



専用システムでMiniONEスクリプトを実行します。 



wget https://github.com/OpenNebula/minione/releases/download/v5.8.0/minione
chmod u+x minione
sudo minione --lxd


GUIを理解する



MiniONEでの手順が完了すると、次のようなWebインターフェイスへの接続手順が表示されます。 



### Report
OpenNebula 5.8 was installed
Sunstone (the webui) is runninng on:
  http://192.168.100.101:9869/
Use following to login:
  user: oneadmin
  password: o6ARsMAdGe


MiniONEの完了後、最初に行うことはoneadminとしてログインし、Sunstoneの管理ビューを確認することです。このビューには、通常のユーザー向けの他のインターフェースよりもはるかに多くのオプションがあります。ダウンロードしたすべてのクラウドリソースを確認します。 で管理者ビュー、あなたはOpenNebulaで何かを行うことができますが、エンドユーザーにこれらの同じ機会を与える必要はありません！Cloud Viewに切り替えて、OpenNebulaでエンドユーザーに表示される内容を確認します。Cloud View インターフェース















エンドユーザーを対象としているため、はるかに単純です。+ボタンをクリックして、新しい仮想マシンを作成します。利用可能な唯一のテンプレートを選択し、[作成]ボタンをクリックします。[ 作成 ]ボタンをクリックすると、ダッシュボードが表示され、実行中のすべての仮想マシンを確認できます。







コントロールパネルを開くには、仮想マシンをクリックします。そこで、VNCからそれにアクセスし、その状態を保存し、再起動するなどできます。







コンソールアイコンをクリックすると、VNCを介してユーザーとしてrootとしてコンソールにアクセスできます。oneadmin







ロールを使用すると、ユーザーが表示および実行できる内容を構成できます。クラウドユーザー。



CLIに会う



OpenNebulaはoneadminとして起動し、メインの管理者はこのユーザーに代わってコマンドを実行する必要があるため、最初に行う必要があるのはoneadminに切り替えることです。 



su - oneadmin




oneadmin アカウントから、読み込まれたリソースを確認できます

。1つの視覚化ノードがあります。 



onehost list


作成されたCentOSイメージ 



oneimage list


登録された仮想マシンテンプレート： 



onetemplate list


テンプレートの構成をさらに詳しく見ることができます。 



onetemplate show 0


MarketPlaceにアクセスする



OpenNebulaにはプレインストールされたMarketPlaceが付属しており、そこから多くの既製の画像（アプリ）を取得できます。パブリックLXDイメージサーバーの1つであるイメージとの統合もあります







。Alpine、Centos、DebianなどのさまざまなLinuxディストリビューションと、WordPressやGitLabなどのアプリケーションがプリインストールされたサービスイメージがあります。







公開LXD画像サーバーのコンテナがあります。たとえば、完全に新しいUbuntuを見つけることができます。







そしてそれをリポジトリにダウンロードします。







これで、[テンプレート]-> [VM]セクションに移動すると、インスタンス化できます。



製造時の取り付けアセンブリ



クラウドを本番環境にデプロイする場合は、ドキュメントをご覧ください。フロントエンドと複数のLXDノードを含む環境をデプロイするには、次の手順が必要です。



  • OpenNebulaフロントエンドをインストールします。
  • lxd-nodeパッケージを仮想化ノードにインストールします。
  • フロントエンドとノードを接続します。
  • すべてが機能していることを確認します。


これで、MarketPlaceのアプリケーションを使用して仮想マシンを展開できます。
