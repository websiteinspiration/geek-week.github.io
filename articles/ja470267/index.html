認定温度監視

当社では、品質管理システム（QMS）を使用しています。ISO：9001認証の一部として、サーバールームや倉庫など、一部の部屋の温度と湿度の読み取り値を収集するように注文されました。

なぜなら すべての官僚的手続きには紙が必要なので、データ収集は検証済みのデバイスで実行する必要があります。

この点で、最も普通の湿度計は絞首刑にされ、数年間人々は毎日日記に証言を書きました。同意する、不快。


この手順を自動化するために、温度と湿度を収集するためのデバイスの検索が開始されました。

目標は何とかしてデータを収集し、それをすべてZabbixに転送することでした。 （Zabbix、QMSには保管場所は必要ありませんが、温度監視用に別のシステムを作成したくありませんでした）。

原則として、Equicom PING3やnetpingなど、このための既製のキットはたくさんありましたが、そのような例はまだたくさんあります。arduinoやraspeberryでシステムを構築することもできます
が、これらすべてのオプションは適合しませんでした。それらのすべては認証、または計量学的検証を持たないため、公式の測定機器とは見なせず、ISO 9001の合格には適していません。

その結果、イーサネット経由でデータを送信できるデバイスが見つかり、測定器のレジスターに含まれていました。それは「Ivit-M.E」であることがわかりました。ノボシビルスクで開発され、そこでも生産されています。注文しました（ちなみに、注文には長い時間がかかります。制作時間は約1か月、検証、およびモスクワへの配送です）。

これはすべてこの形式で提供されます。それを手に取ると、これはロシアの生産品であることがすぐにわかります:-)

このデバイスのマイナス点は、SNMP経由でデータを送信する方法がわからない、Modbusプロトコル経由で機能することです。

原則として、Modbusを介して読み取ることができる多くのSCADAシステムとOPCサーバーがありますが、それらのほとんどは有料であるか、独自のデータベースに収集されています。そこから、zabbixにさらに送信するためにデータを取り出す方法も明確ではありません。

そのため、センサーに直接アクセスするための小さなpythonスクリプトが作成されました。
レジスター内のデータは浮動小数点数の形式で格納され、この形式でそれらにアクセスすると取得されます

READ_INPUT_REGISTERS (16855, 57976)

センサーを構成するためのconfig-ivit.exeアプリケーションがありますが、そこ

には多くの設定はありません。ネットワーク、メールを設定できます。



コンフィギュレータでは、レジスタは16進法で指定されますが、Pythonスクリプトでは、10進法でレジスタを指定する必要があります。

ieee-754標準に従って、データを通常の読み取り可能な形式にします。

スクリプトから受け取ったデータをzabbixに転送するには、2つの方法があります。

最初の方法：pythonおよびzabbiksエージェントがインストールされているノードでuserparametrを使用します。たとえば、zabbixサーバー自体を使用していました。

import modbus_tk.defines as cst
import modbus_tk.modbus_tcp as modbus_tcp
import sys

def check_reg(reg,ip):
    reg = int(reg)
    master = modbus_tcp.TcpMaster(host=ip, port=int(502))
    master.set_timeout(1.0)
    getDI = master.execute(1,cst.READ_INPUT_REGISTERS, reg, 2)
    Num1 = str(bin(getDI[0]))[2:]
    while len(Num1) < 16:
        Num1 = '0' + Num1

    Num2 = str(bin(getDI[1]))[2:]
    while len(Num2) < 16:
        Num2 = '0' + Num2

    res = Num1 + Num2
    znak = int(res[0], 2)
    znak2 = (0 - 1) ** znak
    e = int(res[1:9], 2) - 127
    exp = 2 ** e
    m = 1 + (int(res[9:], 2) / float(2 ** 23))
    F = znak2 * exp * m
    print(round(F, 1))

check_reg(sys.argv[1],sys.argv[2])

2番目の方法：zabbix_senderを介して値を転送します。この方法は、エージェントを持たない別のネットワークノードを作成できるため、多くのデバイスのテンプレートを作成できるので便利です。

import modbus_tk.defines as cst
import modbus_tk.modbus_tcp as modbus_tcp
import sys, os

def check_reg(reg,ip):
    reg = int(reg)
    master = modbus_tcp.TcpMaster(host=ip, port=int(502))
    master.set_timeout(1.0)
    getDI = master.execute(1,cst.READ_INPUT_REGISTERS, reg, 2)
    Num1 = str(bin(getDI[0]))[2:]
    while len(Num1) < 16:
        Num1 = '0' + Num1

    Num2 = str(bin(getDI[1]))[2:]
    while len(Num2) < 16:
        Num2 = '0' + Num2

    res = Num1 + Num2
    znak = int(res[0], 2)
    znak2 = (0 - 1) ** znak
    e = int(res[1:9], 2) - 127
    exp = 2 ** e
    m = 1 + (int(res[9:], 2) / float(2 ** 23))
    F = znak2 * exp * m
    return round(F, 1)

os.system('zabbix_sender -z 127.0.0.1 -s '+ sys.argv[2]+' -k Humidity -o '+ str(check_reg(22,sys.argv[1])))
os.system('zabbix_sender -z 127.0.0.1 -s '+sys.argv[2]+' -k Temper -o '+ str(check_reg(34,sys.argv[1])))
os.system('zabbix_sender -z 127.0.0.1 -s '+sys.argv[2]+' -k Rosa -o '+ str(check_reg(36,sys.argv[1])))

あとは、このスクリプトを実行して、デバイスのIPアドレスとホストのホスト名のみを指定してクラウンで実行するだけです。
