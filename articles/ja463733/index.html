メッシュVS WiFi：ワイヤレスに何を選択するか？



まだマンションに住んでいた時、ルーターから離れた部屋で低速の問題に直面しました。実際、多くの人にとって、ルーターは廊下にあり、プロバイダーが光学機器またはUTPを引き出し、そこに標準デバイスが設置されていました。また、所有者がルーターを自分のものに変更する場合にも適しています。通常、プロバイダーから提供される典型的なデバイスが最も予算が多いか、単純なモデルです。あなたは彼らに高いパフォーマンスを期待すべきではありません-それはうまくいきます。しかし、2.4 GHzおよび5 GHzでの動作をサポートする無線モジュールを備えた、ギガビットポートを備えたルーターをインストールしました。そして、アパート内、特に遠くの部屋でのインターネット接続の速度は完全に憂鬱なものでした。これは、一部はノイズの多い2.4 GHz帯域によるもので、一部は鉄筋コンクリート構造物を通過するときの信号の消光と多重反射によるものです。その後、デバイスを追加してネットワークを拡張することにしました。疑問が生じました：Wi-Fiネットワークまたはメッシュシステム？私はそれを理解し、テストを行い、私の経験を共有することにしました。ようこそ。

Wi-Fiとメッシュの理論


Wi-Fi経由でネットワークに接続してYouTube動画を見る一般ユーザーにとって、どちらのシステムを使用しても違いはありません。しかし、通常のWi-Fiカバレッジを整理するという観点から見ると、これらのシステムは根本的に異なり、それぞれに長所と短所があります。Wi-Fiシステムから始めましょう。

Wi-Fiシステム




これは、独立して動作できる通常のルーターのネットワークです。このようなシステムでは、1つのマスタールータが割り当てられ、残りはスレーブになります。同時に、ルーター間の遷移はクライアントからは見えないままであり、ルーター自体の観点からは、クライアントは1つのセルから別のセルに混在します。このようなシステムは、セルラー通信と比較できます。これは、ルーターとトランスレーターの単一のローカルネットワークが形成されるためです。このシステムの利点は明白です。ネットワークは徐々に拡張でき、必要に応じて新しいデバイスを追加できます。さらに、このテクノロジーをサポートする安価なルーターを購入するだけで十分です。マイナスが1つありますが、重要なのは、イーサネットケーブルと電源を各ルーターに接続する必要があることです。つまり、すでに修理を行い、UTPケーブルを敷設していない場合は、可能であればベースボードに沿って引き伸ばすか、別のシステムを検討する必要があります。

Mesh




これは、特別な機器のネットワークであり、複数のデバイスのネットワークも形成し、継続的なWi-Fi信号カバレッジを作成します。これらのポイントは通常デュアルバンドなので、2.4 GHzネットワークと5 GHzネットワークの両方で作業できます。大きなプラスは、新しい各デバイスを接続するためにケーブルを引く必要がないことです。それらは別個のトランスミッターを介して通信し、独自のネットワークを作成し、データがそれを介して送信されます。その後、このデータは通常のWi-Fiアダプターに転送され、ユーザーに到達します。利点は明白です。追加のワイヤは必要ありません。新しいポイントのアダプタをソケットに差し込み、メインルータに接続して使用するだけです。しかし、欠点もあります。たとえば、価格。メインルータのコストは、従来のルータのコストの数倍です。また、追加のアダプターのコストも重要です。しかし、修理をやり直したり、ケーブルを引っ張ったり、ワイヤーについて考える必要はありません。




私はすでに鉄筋コンクリートのアパートから自分の家に引っ越しており、無線ネットワークの速度低下の問題にも直面しています。以前は、空気のノイズが隣接するWi-Fiルーターの影響を大きく受けていた場合（そして結局のところ、誰もが電力を最大にして、隣人を「消滅」させて速度を上げるよう努めています）、距離とオーバーラップが影響し始めています。 45平方メートルのアパートの代わりに、200平方メートルの2階建ての家に引っ越しました。家の中での生活について多くのことを話すことができ、隣人のWi-Fiポイントがたまにスマートフォンのメニューに表示されるだけで、ワイヤレスネットワークが検出されないという事実でさえ、すでに多くのことが述べられています。いずれにせよ、私は家の地理的な中心にルーターを配置しようとしましたが、2.4 GHzの周波数でどこにでも通信を提供しますが、サイトではカバレッジはすでに不十分です。しかし、ルーターから最も離れた部屋にあるラップトップでホームサーバーから映画を見ると、フェージングが発生することがあります。 5 GHzネットワークは壁や天井がいくつかあるため不安定であり、ラップトップは安定性が高くデータ転送速度が遅い2.4 GHzネットワークに切り替えることを好むことがわかりました。 「もっとスピードが必要です！」-ジェレミー・クラークソンが言うように。そして、私はワイヤレス通信を拡張し、高速化する方法を探しに行きました。額の中で2つのシステムを比較することにしました。KeeneticのWi-FiシステムとZyxelのMeshシステムです。そして、私はワイヤレス通信を拡張し、高速化する方法を探していました。 2つのシステムを正面から比較することにしました。KeeneticのWi-FiシステムとZyxelのMeshシステムです。そして、私はワイヤレス通信を拡張し、高速化する方法を探していました。 2つのシステムを正面から比較することにしました。KeeneticのWi-FiシステムとZyxelのMeshシステムです。



Keenetic側からは、Keenetic GigaルーターとKeenetic Vivaルーターが参加しました。それらの1つはネットワークの主催者で、2番目はスレーブポイントでした。どちらのルーターにも、ギガビットイーサネットとデュアルバンド無線モジュールがあります。さらに、USBポートと非常に幅広いファームウェア設定があります。テスト時には、利用可能な最新のファームウェアがインストールされており、リードはKeenetic Gigaでした。それらは、ギガビットの有線イーサネットケーブルを介して相互接続されていました。



Zyxel側には、Multy XとMulti miniで構成されるメッシュシステムがあります。最古のポイントであるMulty Xはインターネットに接続され、「最年少」のMulti miniが家の隅に設置されました。主要なポイントはネットワークに接続され、追加機能はネットワークを無線および有線で分散することでした。つまり、追加の接続ポイントは、Wi-Fiモジュールを備えていないがイーサネットポートを備えている機器のワイヤレスアダプターとしても機能します。

機能性




製造業者は、ワイヤレスネットワークを備えたデバイスの異常に広い範囲についてプレスリリースでしばしば述べています。ただし、壁、反射面、電波干渉のない空き領域で機能します。実際には、スマートフォンで1.5〜2ダースのワイヤレスネットワークが見えるアパートで、速度の低下とパケット損失に直面しました。したがって、ノイズの少ない5 GHz帯域を使用する方が効率的です。

簡単にするために、ヘッドユニットをWi-Fiおよびメッシュシステムルーターと呼びます。各ルーターは単なるワイヤレスデバイスにすることができます。しかし、ルータがネットワークへのアクセスを提供できるデバイスの数と速度はどのくらいかと思います。最初の質問について、状況は次のとおりです。サポートされるデバイスの数は、Wi-Fiモジュールによって異なります。 Zyxel Multy XおよびMulty miniの場合、各バンド（2.4 + 5 GHz）で64 + 64デバイスになります。つまり、2つのポイントがある場合、2.4 GHzの128デバイスと5 GHzの128デバイスを接続できます。
メッシュネットワークの作成は、可能な限りシンプルで簡単に行われます。仕事のためには、スマートフォンを用意し、そこにZyxel Multiアプリケーションをインストールするだけで十分です。また、iOやAndroidデバイスを使用しているかどうかは関係ありません。インストールウィザードのプロンプトに従って、ネットワークが作成され、後続のすべてのデバイスが接続されます。驚いたことに、ネットワークを最初に作成するには、地理位置情報を有効にし、インターネットに接続する必要があります。したがって、少なくとも、スマートフォンからネットワークにアクセスできる必要があります。

コンピューターとスマートフォンの両方からルーターを構成できます。また、ローカルネットワークのWebベースのインターフェイスを介して簡単に実装できる場合は、スマートフォン用の特別なアプリケーションがあり、将来的にはトレントダウンロードやUSB経由で接続されたドライブ上のファイルへのアクセスなどの追加機能を使用できるようになります。 Keeneticには優れた機能-KeenDNSがあり、灰色のIPアドレスがあれば、外部ネットワークから公開サービスのWebサービスに接続できます。つまり、NATのルーターのインターフェイスに接続でき、NATのDVRまたはWebサーバーのインターフェイスに接続できます。ただし、この資料はまだネットワークに関するものであるため、Wi-Fiネットワークの編成も非常に単純であることに注意してください。主要なルーターがマスターデバイスになり、スレーブアダプターのモードが他のルーターでオンになります。同時に、スレーブルーターはVLANを作成し、単一のアドレス空間で機能し、各ワイヤレスアダプターの電力を10％の増分で設定できます。したがって、ネットワークを何度も拡張できます。ただし、Wi-Fiネットワークを構成するには、すべてのルーターがイーサネットを使用して接続されている必要があります。

テスト方法


クライアント側のワイヤレスネットワークは変わらないため、ネットワークの技術構成の観点から見ると、根本的に異なるため、ユーザー向けの手法が選択されました。テストは、Zyxel Multy X + Multiy miniおよびKeenetic Giga + Keenetic Vivaデバイスに対して個別に行われました。プロバイダーの影響を避けるため、ローカルネットワークでは、ヘッドユニットの前にサーバーが設置されました。そして、ユーザーデバイス上で、クライアントが編成されました。その結果、トポロジは次のようになりました：サーバーをリードするルーターアクセスポイントクライアント。

すべてのテストは、継続的なデータ転送をエミュレートするIperfユーティリティを使用して実行されました。毎回、1、10、100スレッドのテストが行​​われました。これにより、さまざまな負荷でのワイヤレスネットワークのパフォーマンスを評価できます。 Youtubeでのビデオの視聴などのシングルスレッドデータ転送と、急流のダウンロード作業などのマルチスレッドの両方をエミュレートしました。テストは、2.4および5 GHzネットワーク経由で接続されている場合に個別に実行されました。

さらに、Zyxel MultyおよびZyxel miniデバイスはアダプターとして機能できるため、イーサネットインターフェイスを介して1000 Mbpsの速度でユーザーのコンピューターに接続され、3つの速度テストも行われました。同様のテストで、Keenetic VivoルーターはWi-Fiアダプターとして参加し、パッチコードによってラップトップに接続されました。

すべてのデータが表にリストされ、速度グラフがプロットされます。

写真は、2.4 GHzの周波数のすべてのデバイスが5 GHzの周波数よりも効率が悪いことを示しています。そしてこれは、隣接する干渉ネットワークからのノイズがなかったという事実にもかかわらず、2.4 GHz周波数にノイズがあった場合、結果は著しく悪化するためです。それにもかかわらず、5 GHzの周波数でのデータ転送速度は2.4 GHzのほぼ2倍であることがはっきりとわかります。また、同時ロードのストリーム数にもある程度の影響があることがわかります。つまり、ストリーム数が増えると、データチャネルの使用密度が高くなりますが、その差はそれほど大きくありません。

Keeneticルーターがリピーターとして機能したときに、伝送速度が2つに分割されていることがはっきりとわかるので、Wi-Fiネットワークのカバレッジを拡張するだけでなく、大量の情報を高速で転送する場合は、これを考慮する必要があります。

最後のテストでは、Zyxel Multy XとZyxel Multy miniがリモートデバイスを配線するためのアダプターとして機能しました（ベースのZyxel Multy Xと受信デバイス間の接続はワイヤレスでした）が、特にマルチスレッドデータ転送でのMulty Xの利点を示しました。これは、Zyxel Multy Xのより多くのアンテナに影響しました。ZyxelMulty miniの6個に対して、9個です。

結論


したがって、2.4 GHzの周波数の無負荷の空気があったとしても、大量の情報を十分に速く転送する必要がある場合は、5 GHzに切り替えるのが理にかなっています。同時に、2.4 GHzの周波数であっても、ルーターをリピーターとして使用して、FullHD品質で映画を視聴することは十分に可能です。ただし、通常のビットレートの4Kの映画はすでに途切れ始めているため、ルーターと再生デバイスは5 GHzの周波数で動作するはずです。ワイヤレスアダプターとして2つのZyxel Multy XまたはZyxel Multi X + Multy miniのセットを使用すると、最高速度が達成されます。

そして今価格について。 Keenetic Giga + Keenetic Vivaルーターのテスト済みペアは、14,800ルーブルです。 Zyxel Multy X + Multy mini-21 900ルーブルのセット。



Zyxelのメッシュシステムは、余分なワイヤーを敷設することなく、非常にまともな速度で広いカバレッジを提供できます。これは特に、修理が既に行われているが、追加のツイストペアケーブルが敷設されていない場合に当てはまります。さらに、このようなネットワークの構成は、スマートフォン上のアプリケーションを使用して、可能な限り簡単です。メッシュネットワークは6つのデバイスで構成でき、スタートポロジとツリートポロジの両方を持つことができることを追加する必要があります。つまり、エンドデバイスは、インターネットに接続されているスタートルータから非常に離れている場合があります。



同時に、KeeneticルーターをベースにしたWi-Fiシステムははるかに機能的であり、安価なネットワークを提供します。ただし、これにはケーブル接続が必要です。ルーター間の距離は最大100メートルで、ギガビットの有線接続を介した伝送が原因で速度が低下することはありません。同時に、そのようなネットワークには6台を超えるデバイスが存在する可能性があり、移動中のWi-Fiデバイスのローミングはシームレスになります。

したがって、誰もが自分で何を選択するかを決定します。機能性とネットワークワイヤーを敷設する必要性、またはいくつかの大きなお金のためにワイヤレスネットワークを拡張するシンプルさ。
