善行を有名にすることはできません

シリーズ「期待と現実」から、または同志M.F. キンティリアーナ：「害を及ぼすのは簡単で、助けるのは難しい」とは、私たちにとって新しい意味を獲得しました。

COVID19。おそらく誰もがこのウイルスについて聞いたことがあるでしょう。今、私は自分の個人的な態度を説明したり、これについての陰謀論を議論したりすることを想定していません。私個人的には、これは病気の愛する人との実際のケースです。

私たちのチームは、コビッドと対面して、この悪との戦いに貢献することを決定しました。


位置データとBluetoothを使用して、covidで危険な領域の位置を追跡できるモバイルアプリケーションを開発しました。

アナログとは異なり、私たちはすべてを匿名かつ自発的に行いたかったのです。登録を使用せずに、電話番号と電子メールをバインドします。

すべての可能な当局にデータを送信することなく。

アイデアは単純でした-誰もが自分の健康状態を送信し、それを自発的に他の参加者と共有できるようにしました。これにより、ユーザーはマップ上で感染したルートを確認し、病気の人と横断した後に通知を受け取ることができます。そこにその地域の感染指数と、私たちの意見では、他の多くの有用なアイデアが統合されています。

バックグラウンドでの場所の作業から始まり、接続なしのBluetoothを介したデータの交換まで、多くの技術的な問題がありました。

Bluetoothは多くの質問を引き起こす可能性があり、私たちの意見では最も興味深いので、私はあなたとトリッキーな解決策を共有します。多分誰かがプロジェクトで重宝するでしょう。

ビーコンビーコンおよびビーコンスキャナーとしてBluetoothを同時に使用しました。

私たちの仕事は、近くの病人との接触を見つけることです。私たちには匿名のアプリケーションがあり、各ユーザーは匿名化されたIDしか持っていないため、Bluetooth経由でブロードキャストすることにしました。

Androidでは、このためにフォアグラウンドサービスBluetoothMonitorServiceを作成しました。このサービスは、Bluetooth経由でIDをブロードキャストし、近くのエリアをスキャンして他のIDを探します。通常のサービスでは、オペレーティングシステムはアプリケーションを機能させません。残念ながら、トレイには常に通知が表示されます。

この段階での2番目の重要なポイントには、許可が必要です。

Manifest.permission.ACCESS_COARSE_LOCATION.

そのため、場所をリクエストする必要があります。この許可がない場合、空のリストが送信されます。

そしてもちろん：

android.permission.BLUETOOTH
android.permission.BLUETOOTH_ADMIN

それでは始めましょう。

まず、ブロードキャストを開始します。

private void StartAdvertisingLegacy(long timeoutInMillis, int id) {

        byte[] bytesPID = ByteBuffer.allocate(charLength).putInt(id).array();

        data = new AdvertiseData.Builder()
                .setIncludeDeviceName(false)
                .setIncludeTxPowerLevel(false)
                .addServiceUuid(pUuid)
                .addManufacturerData(9, bytesPID)
                .build();

        try {
            Log.d(TAG, "Start advertising");
            advertiser = (advertiser != null) ? advertiser : BluetoothAdapter.getDefaultAdapter().getBluetoothLeAdvertiser();
            advertiser.startAdvertising(settings, data, advertisingCallback);
        } catch (Exception e) {
            Log.d(TAG, "Failed to start advertising legacy: ${e.message}");
        }
    }

ご覧のとおり、addManufacturerDataに配置したユーザーIDです。

パケットサイズは31バイトに制限されています。名前の表示と電力レベルの情報を無効にして、ペイロードで十分になるようにします。

addServiceUuidは、128ビットの一意の属性識別子です。アプリケーションに固有のUUID。これにより、スキャン中にデバイスをフィルタリングできます。

その後、私たちは答えを聞きます：

 AdvertiseCallback advertisingCallback = new AdvertiseCallback() {
        @Override
        public void onStartSuccess(AdvertiseSettings settingsInEffect) {
            super.onStartSuccess(settingsInEffect);
            Log.i(TAG, "Advertising onStartSuccess");
            Log.i(TAG, settingsInEffect.toString());
            isAdvertising = true;
        }

        @Override
        public void onStartFailure(int errorCode) {
            super.onStartFailure(errorCode);

            String reason;

            switch (errorCode) {
                case ADVERTISE_FAILED_ALREADY_STARTED:
                    reason = "ADVERTISE_FAILED_ALREADY_STARTED";
                    isAdvertising = true;
                    break;

                case ADVERTISE_FAILED_FEATURE_UNSUPPORTED:
                    reason = "ADVERTISE_FAILED_FEATURE_UNSUPPORTED";
                    isAdvertising = false;
                    break;

                case ADVERTISE_FAILED_INTERNAL_ERROR:
                    reason = "ADVERTISE_FAILED_INTERNAL_ERROR";
                    isAdvertising = false;
                    break;
                case ADVERTISE_FAILED_TOO_MANY_ADVERTISERS:
                    reason = "ADVERTISE_FAILED_TOO_MANY_ADVERTISERS";
                    isAdvertising = false;
                    break;
                case ADVERTISE_FAILED_DATA_TOO_LARGE:
                    reason = "ADVERTISE_FAILED_DATA_TOO_LARGE";
                    isAdvertising = false;
                    charLength--;
                    break;

                default:
                    reason = "UNDOCUMENTED";
            }

            Log.e(TAG, "Advertising onStartFailure: " + errorCode + " " + reason);
        }
    };

ServiceUUIDはブロードキャスト時と同じである必要があります。そうでない場合、情報はまったく受信されません。これはアプリケーションに固有です。https://www.uuidgenerator.netで生成できます。

便利なポーリング間隔はサービス自体に設定する必要があります。30秒ごとに設定しています。これにより、バッテリーの電力を節約できます。

アプリケーションは大食いではありませんでした。場所を考慮に入れても、それは消費リストの最後にあります。

数か月の開発後、iOSおよびAndroidバージョンのアプリケーションが登場しました。 GoogleとAppleにレビューのために送りました。

そしてここで楽しみが始まりました。解決策を開発することがわかりました-これが最も簡単です。難しさは店での配置から始まりました。

Googleでは、公式の州紙を提供することで成功しました。彼らがアプリケーションに反対していないことを体。ロシアでそれを得ることはそれ自体が偉業ですが、それは別の話です。
Appleは長い間静かでした。我々は再び書いて、彼らに国家の許可を送った。臓器、しかし答えはありませんでした。

そして、ある晴れた日、ベル。アップルから電話が来ました！私は英語を忘れるほど心配でした。 Appleは非常に丁寧で、5分間のコミュニケーションの後、彼らは翻訳者を見つけると言った。

しばらくして、彼らは再び私に電話をかけ、すでにアクセントをつけてロシア語で彼らはアプリケーションについて尋ね始めました。

彼らは彼らの医療部門が私たちのアプリケーションに興味を持っていて、彼らの承認なしには私たちはテストに合格しないだろうと言った。私たちが彼らの質問に答えたら、彼らは協力する準備ができています。その後しばらくの間、私たちは彼らのすべての質問に忠実に答え、解決策と待望の協力を待っていました。

しかし、時間を無駄にしないために、人々に群れとして強制するのではなく、選択肢と自発的なツールを提供する代替手段と機会を示すために、申請について当局に伝えることにしました。
