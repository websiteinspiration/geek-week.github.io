ServerLessをすべての人に


それでも、ServerLessコンピューティングについては多くのことが言われています。しかし、以前はAzure FunctionsとLogic Appsだけがこのソースに包まれていた場合、Azure SQL Database ServerLessでもフックされました（発表を見て、ServerLess SQLが面白いように聞こえたので非常に驚きました）。

一方、ServerlessはAzureだけでなく、コンテナー内でローカルに起動することもできます。したがって、サーバーレスはより理解しやすく、魔法のようなものではなくなります（エンジニアは通常、魔法が好きではありません。彼らはそれがどのように機能するかを理解していないためです...機能しない場合は、魔法を修正する方法）。

SQL ServerLess


SQL Serverless の本質は、消費されるリソースの上限と下限を指定でき、下限と上限の間に消費したものに加えて支払うことです。データベースが使用されていない場合にSQLを一時停止できる（コンピューティングリソースの代金を払わない）ことができるマイナーな機能（今は6時間一時停止できる）なので、まだ検討していません。そのため、データベースへの単一の要求が数時間の間まったく発生しないようにします。これは非常に奇妙な状況です。少なくとも監視があり、ベースが（約束どおり）30〜60秒で開始することも重要です。

Windows Presentation Platform


私が言及したいセッションの1つは、Windowsプレゼンテーションプラットフォーム（WPP）セッションです。正直、うっかりWindows Presentation Foundation（WPF）だと思っていたのですが、思った以上に広いので驚きました。

2009年に戻ってWPFが登場したばかりのとき、このトピックは人気がありました。WPFアプリケーションでのWinFormsコントロールのホスティング、またはその逆-WinFormsにWPFコントロールを挿入しようとすることです。アプリケーションの巨大な部分を取り、新しいフレームワークにフリップすることがしばしば不可能だったからです。最初のリリースでは、そのような機能は宣伝されていませんでしたが、下位互換性を取り除かなければならないことが明らかになると、積極的に宣伝し始めました。

そのため、2019では、WPF（XAMLアイランド）でU WPコントロールをホストする方法と、WinUI 3.0のUIコンポーネントとオペレーティングシステムを分離する（つまり、コントロール/コンパイラ/ランタイムをnugetパッケージとして提供し、Windows Updateを待たない）アイデアは素晴らしいですが、UWPの最初のバージョンでそれを行わなければなりませんでした。当時、Entity Frameworkチームは、Windows Updateへの依存をやめるために、.net FrameworkからEFを削除してきました。そして、.Net Coreになるものは開発中でした。



そして最も重要なのは、Windows 10の最新バージョンだけでなく、かなり古いものでも利用できるようになることです。


大ホールはセッション用に予約されましたが、ホールには十分な空きスペースがありました（Windowsの開発は約10年間埋められています）が、それはポイントではありません。ホールには非常に少数の若者しかいませんでした（Asp.Net Core、特に機械学習のセッションと比較しています）。これらは私の主観的な観察に過ぎず、私はWindowsの葬式の開発を扱っていません。
