ホームオートメーションシステム（「スマートホーム」）で音声アラートを作成する簡単な方法

Linphonec SIPコンソールクライアントをテストした後、ホームオートメーションシステム用の携帯電話に音声アラートを実装する簡単で迅速な方法の可能性を確認しました。

この方法は多くのDIYホームオートメーションシステムに適していると思います。独自のバージョンを提供します。

私はMajorDoMoスマートホームオートメーションシステムをRaspberry Piシングルボードコンピューター（Raspbian（Debian 9）を実行）に

インストールしています。追加の機器やVoIPサーバーを使用せずにMajorDoMoからスマートフォンに通知する機能を作成します。

アラートの簡単な原則：重大な事故（誰もが自分の事故リストを決定できる）が発生した場合、オートメーションシステムはコンソールのVoIPクライアントを管理し、SIPプロトコルを介して所有者を呼び出します。応答後、以前に記録されたサウンドファイルを事故メッセージとともに再生します。

まず、ネットワーク内で無料のインバウンド/アウトバウンドを使用するSIPプロバイダーからの2つのアカウントが必要です。たとえば、Zadarma.comクラウドテレフォニーサービスを見てみましょう。

最初のステップ：個人アカウントとSIPアカウントにアクセスしながら、プロバイダーに登録します。複数のメールボックスに登録できますが、アカウントを補充して追加の番号を取得できます。いずれの場合も、2つの番号（アカウント）とそれにパスワードが必要です。

スマートフォン（Androd OSで動作します）には、市場からモバイルアプリケーションをインストールします。SIPクライアントなら何でもかまいません。便宜上、初期設定を最小限に抑えるために、Zadarma呼び出しには専用のアプリケーションを使用できますが、より機能的にすることができます。
インストール後、番号とパスワードを入力して登録を行い、スマートフォンの設定は完了です。したがって、1つの番号を使用しました。

Raspberryホームオートメーションサーバーでは、Linphonec SIPターミナルクライアントが必要です。 Raspbianリポジトリからインストールする場合、ALSAサウンドシステムでは正常に動作しないかなり古いバージョン3.6.1がインストールされますが、音声を直接使用する必要がないので、簡単な方法を試してみましょう。

前の記事で説明したように、Debian 9のグラフィカルインターフェイスなしでLinphoneパッケージの最新バージョン（3.12.0）をインストール：DIY呼び出しボタン。 Raspberry Pi、MajorDoMo、Freeswitch、Linphonec。そこでは、USBサウンドカードのインストールとRaspberry Piのサウンド設定に関する情報を見つけることができます。

sudo apt-get install linphone-nogtk

コンソールクライアントユーティリティlinphonecおよびlinphonecshのインストール-バックグラウンドで実行されているlinphonecを管理するためのコンソールプログラムが/ usr / binディレクトリにインストールされます。

register sip:xxxxxx@sip.zadarma.com sip.zadarma.com PASSWORD

音声メッセージを録音します（例：「システムの再起動が行われ、ステータスは現在稼働中です」：

arecord -D plughw:1,0 -f cd /home/pi/alarm_restart.wav

Linphonecをデーモンモードで起動して制御するには、Linphonecshユーティリティを使用します。最初に、起動時にlinphonecsh -initクライアントの初期化が必要です。このコマンドを実行すると、構成ファイルをロードせずにコンソールクライアントが起動します。

起動時に設定が読み込まれるように、次のフラグを付けて実行します。

 linphonecsh init -c /home/pi/.linphonerc

これで、ターミナルからコンソールクライアントを管理できます。

Linphonecshコマンドリスト
Usage:
linphonecsh <action> [arguments]
where action is one of
	init		: spawn a linphonec daemon (first step to make other actions)
			followed by the arguments sent to linphonec
	generic		: sends a generic command to the running linphonec daemon
			followed by the generic command surrounded by quotes,
			 for example "call sip:joe@example.net"
	register	: register; arguments are 
			--host <host>
			--username <username>
			--password <password>
	unregister	: unregister
	dial		: dial <sip uri or number>
	status		: can be 'status register', 'status autoanswer' or 'status hook'
	soundcard	: can be 'soundcard capture', 'soundcard playback', 'soundcard ring',
			 followed by an optional number representing the index of the soundcard,
			 in which case the soundcard is set instead of just read.
	exit		: make the linphonec daemon to exit.



次のコマンドを試してみましょう。

linphonecsh generic 'soundcard use files'

linphonecsh generic 'play alarm.wav'

linphonecsh dial sip:@sip.zadarma.com

汎用はLinphonecプログラム自体のコマンドです。XXXXXXは、スマートフォンにインストールされているSIP番号です。
