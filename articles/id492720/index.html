diskussion: layanan file proyek

Programmer dengan GitHub / GitLab yang mereka miliki hanya dapat iri - pengembangan bersama sangat nyaman.


Apa, misalnya, yang dilakukan oleh para perancang di bidang konstruksi (jaringan teknik) yang, sebagai hasil dari kerja keras mereka, juga melihat "kumpulan file"? Sebagai aturan, ini adalah "cara lama": file di "server", flash drive, mail atau file hosting, dan diskusi - di pertemuan, melalui telepon dan email. Jika kontraktor masih terlibat, maka pada akhir pekerjaan semuanya perlahan-lahan masuk ke dalam kekacauan: surat-surat penting dilewati, permintaan tidak tercapai, dll. Oleh karena itu, kegagalan untuk memenuhi tenggat waktu karena pekerjaan yang tidak terkoordinasi adalah hal yang umum, dan tampaknya semua orang telah sepakat dengan hal ini.


Tentang solusi untuk masalah ini - di bawah jalan pintas




Bagaimana jika Anda menggabungkan server file, sistem manajemen tugas, dan korespondensi di satu tempat? Apalagi tugas yang diatur langsung ke file dan folder. Jadi kami lakukan dalam layanan diskusi kami.

Hanya tugas, daftar pengguna dan korespondensi yang disimpan dalam layanan, dan Yandex.Disk digunakan untuk menyimpan file. Karena baru-baru ini Yandex belum menyambut penggunaan Yandex.Disk kecuali untuk kebutuhan pribadi, ia direncanakan untuk menambahkan metode penyimpanan lain, misalnya, Google Drive atau server SFTP / FTPS kontraktor umum.


Alih-alih sebuah kesimpulan


Saat ini ada produk di pasar yang memecahkan masalah yang sama, seperti Oracle ConjectPM dan Autodesk BIM 360 Document Managementment, yang pada kenyataannya tidak semua kontraktor umum mampu. Layanan kami berada pada tahap MVP dan kami akan berterima kasih atas tips tentang apa yang dapat ditingkatkan dan ditambahkan.
