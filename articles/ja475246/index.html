Python非同期プログラミング：概要

プログラムの実行について話すとき、「非同期実行」とは、プログラムが特定のプロセスの完了を待たずに、それとは独立して動作し続ける状況を意味します。非同期プログラミングの例として、非同期で動作するログファイルに書き込むユーティリティを使用できます。このようなユーティリティは失敗する可能性がありますが（たとえば、空きディスク領域の不足が原因で）、ほとんどの場合、正しく機能し、さまざまなプログラムで使用できます。彼らは彼女に電話をかけ、彼女に記録用のデータを渡し、その後彼らは自分のことを続けることができます。



特定のプログラムを作成するときに非同期メカニズムを使用すると、このプログラムは、そのようなメカニズムを使用しない場合よりも高速に実行されます。同時に、ロギング用のユーティリティなど、非同期で実行する予定があるものは、緊急事態を考慮して作成する必要があります。たとえば、ログ用のユーティリティは、ディスク容量が足りなくなった場合に、ログを停止するだけで、エラーでメインプログラムを「クラッシュ」させることはありません。

非同期コードの実行には、通常、別のスレッドでのこのようなコードの操作が含まれます。これは、シングルコアプロセッサを搭載したシステムの場合です。マルチコアプロセッサを搭載したシステムでは、このようなコードは別のコアを使用するプロセスによって実行される場合があります。ある時点でのシングルコアプロセッサは、1つの命令しか読み取って実行できません。本を読むようなものです。 2冊の本を同時に読むことはできません。

あなたが本を読んでいて、他の誰かがあなたに別の本を与えているなら、あなたはこの2冊目の本を持ってそれを読み始めることができます。しかし、最初のものは延期される必要があります。マルチスレッドのコード実行は、同じ原理で構成されています。そして、あなたのコピーのいくつかが一度に複数の本を読むとしたら、それはマルチプロセッサシステムのしくみに似ています。

シングルコアプロセッサ上で、異なる計算能力を必要とするタスク間を切り替えるのが非常に速い場合（たとえば、特定の計算とディスクからのデータの読み取りの間）、シングルプロセッサコアが同時にいくつかのことを行うように感じる場合があります。または、一度にブラウザで複数のサイトを開こうとすると、これが発生します。ブラウザが個別のストリームを使用して各ページをロードする場合、これらのページが一度に1つずつロードする場合よりも、すべてがはるかに速く行われます。ページの読み込みはそれほど難しいタスクではありません。システムのリソースを最大限に使用しないため、そのような複数のタスクを同時に起動することは非常に効果的な方法です。

Python非同期プログラミング


当初、Pythonはジェネレーターベースのコルーチンを使用して非同期プログラミングタスクを解決していました。次に、Python 3.4では、非同期プログラミングメカニズムを実装するモジュールが登場しましたasyncio（その名前はと表記されることもありますasync IO）。Python 3.5では、async / awaitコンストラクトが導入されました。

Pythonで非同期開発を行うには、いくつかの概念に対処する必要があります。これらはコルーチンとタスクです。

コルーチン


通常、コルーチンは非同期関数です。コルーチンは、コルーチン関数から返されるオブジェクトにすることもできます。

関数を宣言するときに非同期であることが示されている場合は、キーワードを使用して呼び出すことができますawait。

await say_after(1, ‘hello’)

このような構造は、プログラムがawait-expressionに遭遇するまで実行され、その後、関数を呼び出し、呼び出された関数の作業が完了するまでその実行を一時停止することを意味します。その後、他のコルーチンも開始できるようになります。

プログラムを一時停止するとは、制御がイベントループに戻ることを意味します。モジュールを使用する場合asyncio、イベントループはすべての非同期タスクを実行し、I / Oを実行し、サブプロセスを実行します。ほとんどの場合、タスクはcorutinを実行するために使用されます。

タスク


タスクを使用すると、イベントループでコルーチンを実行できます。これにより、いくつかのコルーチンの実行制御が簡素化されます。コルーチンとタスクを使用する例を次に示します。構成を使用して宣言されたエンティティasync defはコルーチンであることに注意してください。この例は、公式の Python ドキュメントから抜粋したものです。

import asyncio
import time

async def say_after(delay, what):
    await asyncio.sleep(delay)
    print(what)

async def main():
    task1 = asyncio.create_task(
        say_after(1, 'hello'))

    task2 = asyncio.create_task(
        say_after(2, 'world'))

    print(f"started at {time.strftime('%X')}")

    #     (  
    #  2 .)
    await task1
    await task2

    print(f"finished at {time.strftime('%X')}")
asyncio.run(main())

関数say_after()にはプレフィックスがありますasync;その結果、コルーチンがあります。この例から少し外れると、この関数は次のように呼び出すことができると言えます。

    await say_after(1, 'hello')
    await say_after(2, 'world')

ただし、このアプローチでは、コルーチンが順次呼び出され、完了するまでに約3秒かかります。私たちの例では、それらは競争的に立ち上げられます。それらのそれぞれに対してタスクが使用されます。結果として、プログラム全体の実行時間は約2秒になります。このようなプログラムを実行するにはmain()、キーワードで関数を宣言するだけでは不十分であることに注意してくださいasync。このような状況では、モジュールを使用する必要がありますasyncio。

サンプルコードを実行すると、次のようなテキストが画面に表示されます。

started at 20:19:39
hello
world
finished at 20:19:41

最初と最後の行のタイムスタンプは2秒異なることに注意してください。この例をcorutinの順次呼び出しで実行すると、タイムスタンプの差はすでに3秒になります。

例


この例では、一連の数値の10個の要素の合計を計算するために必要な操作の数が決定されます。シーケンスの最後から計算が実行されます。再帰関数は、数値10を取得することから開始し、次に数値9および8で自分自身を呼び出し、返されるものを合計します。これは、計算が完了するまで続きます。その結果、たとえば、1から10までの数列の合計が55であることがわかります。さらに、この関数は非常に効率が悪いため、ここでは構造を使用していますtime.sleep(0.1)。

これが機能コードです：

import time

def fib(n):
    global count
    count=count+1
    time.sleep(0.1)
    if n > 1:
        return fib(n-1) + fib(n-2)
    return n

start=time.time()
global count
count = 0
result = fib(10)
print(result,count)
print(time.time()-start)

非同期メカニズムを使用してこのコードを書き換え、asyncio.gather2つのタスクを実行し、それらが完了するのを待機する責任がある設計をここに適用するとどうなりますか？

import asyncio,time

async def fib(n):
    global count
    count=count+1
    time.sleep(0.1)
    event_loop = asyncio.get_event_loop()
    if n > 1:
        task1 = asyncio.create_task(fib(n-1))
        task2 = asyncio.create_task(fib(n-2))
        await asyncio.gather(task1,task2)
        return task1.result()+task2.result() 
    return n

実際には、この例では、すべてが1つのスレッドで実行されているので、少しでも遅くなる前のものよりも動作し、通話create_task、gatherなど、システムに追加の負荷を作成します。ただし、この例の目標は、複数のタスクで競争し、それらが完了するのを待つ能力を示すことです。

概要


タスクとコルチンの使用が非常に役立つ状況があります。たとえば、プログラムに入出力操作と計算操作が混在している場合や、同じプログラムで異なる計算が実行されている場合は、コードを競合ではなく競合で実行すると、これらの問題を解決できます順次モード。これにより、プログラムが特定のアクションを実行するために必要な時間が短縮されます。ただし、これにより、たとえば、計算を同時に実行することはできません。マルチプロセッシングは、このような計算を整理するために使用されます。これは別の大きなトピックです。
