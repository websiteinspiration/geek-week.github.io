Red Hat OpenShift v3でのAppDynamicsの使用


最近、多くの組織がRedHat OpenShift v3などの「サービスとしてのプラットフォーム」（PaaS）を使用してアプリケーションをモノリスからマイクロサービスに移行することを望んでいます。AppDynamicsは、こうしたプロバイダーとのファーストクラスの統合を提供するために多大な投資を行っています。



AppDynamicsは、Source-to-Image（S2I）手法を使用して、エージェントをRedHat OpenShift v3と統合します。S2Iは、再生可能なDockerイメージを構築するためのツールです。アプリケーションソースをDockerイメージに挿入し、新しいDockerイメージを構築することで、すぐに実行できるイメージを作成します。ベースイメージ（ビルダー）とビルドされたソースを含む新しいイメージは、docker launchコマンドで使用する準備ができています。S2Iは、以前にロードされた依存関係、以前に作成されたアーティファクトなどを再利用する増分アセンブリをサポートしています。

処理する


RedHat OpenShiftでAppDynamicsを使用するための完全なプロセス


ステップ1：RedHatはすでに提供されています


手順2と3を完了するには、次のGitHubリポジトリでS2Iスクリプトを使用し、JBoss WildflyおよびEAPサーバー用の拡張ビルダーイメージを作成する方法を説明します。リンクをたどる
特定の例を見て、アプリケーションテンプレートを使用してリンクをたどります。

前提条件：
  • OCがインストールされていることを確認してください（リンク）
  • stiがインストールされていることを確認してください（リンク）
  • dockerhubアカウントを持っていることを確認してください（リンク）

ステップ2：AppDynamics Builderイメージを作成する


 $ git clone https://github.com/Appdynamics/sti-wildfly.git
$ cd sti-wildfly
$ make build VERSION=eap6.4

ステップ3：アプリケーションイメージを作成する


 $ s2i build  -e “APPDYNAMICS_APPLICATION_NAME=os3-ticketmonster,APPDYNAMICS_TIER_NAME=os3-ticketmonster-tier,APPDYNAMICS_ACCOUNT_NAME=customer1_xxxxxxxxxxxxxxxxxxf,APPDYNAMICS_ACCOUNT_ACCESS_KEY=xxxxxxxxxxxxxxxxxxxxx,APPDYNAMICS_CONTROLLER_HOST=xxxx.saas.appdynamics.com,APPDYNAMICS_CONTROLLER_PORT=443,APPDYNAMICS_CONTROLLER_SSL_ENABLED=true” https://github.com/jim-minter/ose3-ticket-monster appdynamics/sti-wildfly-eap64-centos7:latest pranta/appd-eap-ticketmonster
$ docker tag openshift-ticket-monster pranta/openshift-ticket-monster:latest
$ docker push pranta/openshift-ticket-monster

ステップ4：OpenShiftでアプリケーションをデプロイする


$ oc login 10.0.32.128:8443
$ oc new-project wildfly
$ oc project wildfly
$ oc new-app –docker-image=pranta/appd-eap-ticketmonster:latest –name=ticketmonster-demo



これで、コントローラーに入り、アプリケーションパネルでticketmonsterアプリケーションを確認できます。
