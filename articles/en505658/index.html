Creating the effect of fast flight through space in 10 minutes on p5.js



Recently, under the inspiration from The Coding Train channel, I decided to participate in one of the 10-minute challenges in which I had to create the illusion of flying through space at high speed.

To implement the project, I chose the well-known to me p5.js - a library for JavaScript, designed to create art in an algorithmic way. Why was it impossible to use standard graphics packages from Adobe?

Firstly, doing this with code is beautiful.
Secondly, the code allows on the fly, changing just a few parameters, to get a different result, the speed of experiments and iterations increases significantly.
And thirdly, this code is then easily integrated into any web project in the form of a script in JS.

Well, to infinity and beyond ...


To start a project, the easiest way is to go to the online editor at editor.p5js.org.

The first thing we should do is create stars. It is clear that we need a large number of stars, although it is technically not difficult to find space in outer space where you can not see stars at all with the naked eye, but our idea of ​​this effect was created using science fiction films. I think you can neglect accuracy for the sake of effectiveness. So. We need to make a random star generator.

First, create an array at the very top that will store our stars:

let stars = [];


Let's create a new function called Star (). It should have four key parameters: position along x, y and variable z, through which we will change everything else. To create a random occurrence of the initial points, we can use the random () function, and as the parameters set the boundaries of our screen in width and height.

The random () function works with 0, 1, or 2 arguments. Without arguments - returns a random number from 0 to 1, with one - from 0 to the argument, with two - from the first to the second.


Since we are going to build a “factory” for the production of stars, let's immediately introduce the this keyword, which will access and access information from the object from which we will call this function (or rather, the method).

function Star() {
  this.x = random(-width, width);
  this.y = random(-height, height);
  this.z = width;


Experienced people may have a question - are we going to make real 3d? But in fact it is much easier to make pseudo-3d, as in old-school games. The variable z is needed here for other purposes. She will be responsible only for the illusion of the third dimension.

Done, although the stars are not yet visible, but we can already create them. It is time to give them movement. To do this, we expand our Star () function with a new method (nested function) update (), which is responsible for an endless cycle of changes.

We need to determine the speed of movement through the stars. To do this, create a variable speed and make it equal to 25. The logic is as follows: we change the parameter z, which will affect the parameters x and y, so that changing one parameter we will change everything. As soon as this parameter reaches a value of less than unity, we roll back the parameters again to random initial values, starting a new cycle of motion of the star on the screen.

    this.z = this.z - speed;
    if (this.z < 1) {
      this.x = random(-width, width);
      this.y = random(-height, height);
      this.z = width;

  };


Well, stars are created and even moving, it is time to see them. To do this, we will use the show () function, also creating a call to the object through this and painting our stars white.

this.show = function() {
    fill("white");
}


Now we need to create a shape for our star, so as to have a variety of trajectories and size. We will do this through the same variable z, which is initially equal to the width of the screen (width), but decreases by 25 with each frame until it becomes less than 1, after which the cycle repeats.

To get a beautiful movement of stars from the center to the edge, let's see how the position should change, namely the variables x and y. Each star should appear on an imaginary ray and go along it to the edge of the screen.

Imagine that the starting point is the coordinate 200,200. We need to make both of these indicators gradually change downward

Since z is always equal to the width of the screen (for example, 600), and x and y range from -600 to 600, the variables x and y at the start will always be less than or equal to z. This suggests that we can translate the values ​​into easier-to-manage values ​​from 0 to 1 by simply dividing x or y by z.

Next, we can make a standard mapping function - to equate numbers from 0 to 1 in proportion to the width or height of the screen. And it is precisely these values ​​that are already passed as parameters to our ball star: ellipse ()

this.show = function() {

    var sx = map(this.x / this.z, 0, 1, 0, width);
    var sy = map(this.y / this.z, 0, 1, 0, height);

  1. x and y are negative
  2. x and y are positive

Imagine that width = 600 is our cosmos. x = -300, y = -300. In this case, the starting point will be -300/600 = -0.5, after 4 frames, when z decreases by 4 * 25 = 100, the coordinate will already be -300 / 500 = -0.6, i.e., the coordinates will be LINEAR away from the center of the screen (coordinates 0,0) up and to the left, until z becomes less than 1.

In case they are positive, everything will happen exactly the same, only the movement will be down and to the right.

In case they differ, the option is down and left (for positive y and negative x) or up and right (for negative y and positive x).

The ellipse function takes four parameters: position along x, position along y, diameter along x and diameter along y, on the basis of which the ellipse is built.

Well, it's time to light the stars and create space.

Under starts and speed, we create a standard function that creates the environment (it works once) - setup () Inside we create our canvas - createCanvas (600,600) and start our factory for creating stars through a for loop, for a start we will limit ourselves to 800.

function setup() {
  createCanvas(600, 600);
  for (let i = 0; i < 800; i++) {
    stars[i] = new Star();
  }
}


After that, we create the second standard function - draw (), its difference is that it works in an infinite loop. We paint the background through background () in black, after which we count the length of the stars (800) array through the for loop and, referring to each star in the array, we call the function (in this case, the method) update () and show ()

function draw() {
  background("black");
  for (let i = 0; i < stars.length; i++) {
    stars[i].update();
    stars[i].show();
  }
}


That's it, we flew, but there are two problems. The first one - since the reference point of coordinates by default is not in the center, but in the upper left corner, we see only that part of the stars that do not have an initially negative coordinate value. The rest are left out of sight.



To solve this problem is easy enough - you just need to move the center through the translate command (width / 2, height / 2).



The second problem is that the stars do not change their size when approaching the observer. On a subconscious level, we feel that something is not right.

In order to solve this problem, let's go back to our ellipse, create another variable r, which will also depend on z and will determine the third and fourth parameters of the ellipse, which are responsible for the horizontal and vertical diameters. The final code is as follows:

let stars = [];
let speed = 25;

function setup() {
  createCanvas(600, 600);
  for (let i = 0; i < 800; i++) {
    stars[i] = new Star();
  }
}

function draw() {
  background("black");
  translate(width / 2, height / 2);
  for (let i = 0; i < stars.length; i++) {
    stars[i].update();
    stars[i].show();
  }
}

function Star() {
  this.x = random(-width, width);
  this.y = random(-height, height);
  this.z = random(width);
 
  this.update = function() {
    this.z = this.z - speed;
    if (this.z < 1) {
      this.x = random(-width, width);
      this.y = random(-height, height);
      this.z = width;
    }
  };

  this.show = function() {
    fill("white");
    var sx = map(this.x / this.z, 0, 1, 0, width);
    var sy = map(this.y / this.z, 0, 1, 0, height);
    var r = map(this.z, 0, width, 10, 0);
    ellipse(sx, sy, r, r);
  };
}


That's all, it's time to fly.



If you are interested in looking at different options and how the parameters affect the final picture, I recommend playing with the values ​​of the speed variables, taking other color variations for the background and for the stars, increasing or decreasing their number.
