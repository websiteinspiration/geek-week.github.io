WEB 3.0-発射物への2番目のアプローチ



まず、少し歴史。

Web 1.0は、所有者がサイトに投稿したコンテンツにアクセスするためのネットワークです。静的なHTMLページ、情報への読み取り専用アクセス、主な喜びは、このサイトや他のサイトのページにつながるハイパーリンクです。典型的なサイト形式は情報リソースです。オフラインコンテンツをネットワークに転送する時代：本のデジタル化、画像のスキャン（デジタルカメラはまだ珍しかった）。

Web 2.0-人々を結びつけるソーシャルネットワーク。インターネット空間に真っ向から飛び込んだユーザーは、Webページに直接コンテンツを作成します。インタラクティブな動的サイト、コンテンツのタグ付け、Webシンジケーション、マッシュアップ技術、AJAX、Webサービス。情報リソースは、ソーシャルネットワーク、ブログホスティング、ウィキに道を譲ります。オンラインコンテンツ生成の時代。

「web 1.0」という用語は、「web 2.0」が登場して初めて、古いインターネットを指すようになったことは明らかです。そしてほとんどすぐに、将来のバージョン3.0について話し合いが始まりました。この将来にはいくつかのビジョンオプションがあり、それらすべてはもちろん、Web 2.0の欠点と制限の克服に関連していました。

Netscape.comのCEOであるJason Kalakanisは、主にユーザーが作成したコンテンツの質の悪さに懸念を抱いており、インターネットの未来は「高品質のコンテンツを作成する」「才能のある人々」のためであると提案しました（Web 3.0、「公式」の定義、2007年）。自分には理にかなっているが、どのサイトでどのように、どこでそれを行うのかについては説明しなかった。まあ、Facebookではない。

「ウェブ2.0」という用語の作者Tim O'Reillyは、このような信頼できない仲介者はネットワークに情報を投稿するために必要ではないと合理的に示唆した人として。技術デバイスはデータをインターネットに提供することもできます。また、同じ技術デバイスはWebベースのストレージから直接データを読み取ることができます。実際、Tim O'Reillyは、Web 3.0を「Internet of Things」に関連付けることを提案しました。

World Wide Webの創設者の1人であるTim Berners-Leeは、インターネットの将来のバージョンで、セマンティックWebに関する彼の長年の（1998年の）夢の実現を見ました。そして、彼の用語の解釈が勝ちました-最近まで「ウェブ3.0」と発音した人のほとんどは、最近ではセマンティックウェブ、つまりサイトのページのコンテンツがコンピューターにとって意味があり、機械で読み取り可能なネットワークを念頭に置いていました。 2010年から2012年頃のどこかで、オントログ化について多くの話があり、セマンティックプロジェクトはバッチで生まれましたが、その結果は誰にでも知られています-私たちはまだインターネットバージョン2.0を使用しています。実際、Schema.orgのセマンティックマークアップスキームとインターネットモンスターGoogle、Microsoft、Facebook、LinkedInのナレッジグラフのみが完全に生き残った。

デジタル革新の強力な新しい波は、セマンティックWebの失敗を隠すのに役立ちました。マスコミと一般の人々の関心は、ビッグデータ、モノのインターネット、ディープトレーニング、ドローン、追加の現実、そしてもちろんブロックチェーンに切り替わりました。リストの最初のものが主にオフラインテクノロジーである場合、ブロックチェーンは本質的にネットワークプロジェクトです。 2017年から2018年の人気のピークで、彼は新しいインターネットの役割さえ主張しました（このアイデアは、イーサリアムの創設者の1人であるジョセフルビンによって繰り返し表現されました）。

しかし時が経ち、「ブロックチェーン」という言葉はもはや未来への突破口とは関係がなく、むしろ不当な希望と結びついていました。そして、ブランド名を変更するという考えが自然に生まれました：ブロックチェーンを自給自足のプロジェクトとして語るのではなく、すべてを新しくて明るいものを具現化するテクノロジーのスタックにそれを含めましょう。この「新しい」のすぐそこに「web 3.0」の名前が見つかりました（新しいわけではありません）。そして、名前のこの非新規性をどういうわけか正当化するために、「ライト」スタックにセマンティックネットワークを含める必要がありました。

したがって、今のトレンドはブロックチェーンではなく、ブロックチェーン、機械学習、セマンティックWeb、モノのインターネットなど、いくつかの基本的なテクノロジーで構成されるWeb 3.0分散型インターネットインフラストラクチャです。過去1年間に登場したWeb 3.0の新たな生まれ変わりに捧げられた多くのテキストでは、その各コンポーネントについて詳しく調べることができますが、不運なことに、これらのテクノロジーを全体に統合する方法、ニューラルネットワークにモノのインターネットが必要な理由、そしてブロックチェーンセマンティックウェブ？ほとんどのチームは単にブロックチェーンに従事し続けます（おそらく、手掛かりを克服できる、または単に投資を解決できるクリプトを作成することを望んでいます）が、新しい看板「web 3.0」の下にあります。しかし、少なくとも将来についての何かであり、不当な希望についてではありません。

しかし、すべてがそれほど悲しいわけではありません。ここで、上記の質問に簡単に答えてみましょう。

なぜセマンティックネットワークブロックチェーンなのか？もちろん、ここでは、ブロックチェーン自体（暗号リンクされたブロックのチェーン）についてではなく、ピアツーピアネットワークでの暗号化方法に基づくユーザー識別、コンセンサス検証、およびコンテンツ保護を提供するテクノロジーについて説明する必要があります。したがって、そのようなネットワークとしてのセマンティックグラフは、レコードとユーザーの暗号化されたIDを備えた信頼性の高い分散ストレージを受け取ります。これは、無料ホスティングでのページのセマンティックマークアップではありません。

なぜ条件付きブロックチェーンのセマンティクスか？オントロジー、それは一般的にサブジェクトエリアとレベルへのコンテンツの配布についてです。これは、ピアツーピアネットワーク上でスローされるセマンティックWeb、またはよりシンプルな場合はネットワークデータを単一のセマンティックグラフに編成することにより、ネットワークの自然なクラスタリング、つまり水平スケーリングを保証することを意味します。グラフのレベル編成により、意味的に独立したデータの処理を並列化できます。これはデータアーキテクチャであり、すべてを無差別にブロックに落としてすべてのノードに格納するのではありません。

モノのインターネットのセマンティクスとブロックチェーンはなぜですか？ブロックチェーンを使用すると、すべてが取るに足らないように見えます。これは、暗号化キーによってアクター（IoTセンサーを含む）を識別するための組み込みシステムを備えた信頼性の高いストレージとして必要です。また、セマンティクスは、一方ではデータストリームをサブジェクトクラスターごとに分離することを可能にします。つまり、ノードのアンロードを保証し、他方では、IoTデバイスから送信されるデータを意味のあるものにするため、アプリケーションに依存しません。アプリケーションAPIのドキュメントのリクエストを忘れることがあります。

そして、機械学習とセマンティックネットワークの交差による相互利益が何であるかを見つけることはまだ残っていますか？まあ、ここではすべてが非常に簡単です。セマンティックグラフにない場合、検証済みの構造化されたセマンティックデータの巨大な配列を単一の形式で見つけることができるので、ニューロンのトレーニングに必要ですか？一方、ニューラルグリッドよりも優れているグラフでは、有用な、または有害な異常の存在を分析して、たとえば、新しい概念、類義語、スパムを特定しますか？

そして、そのようなウェブ3.0が必要です。Jason Kalakanis氏はこう言います。これは、才能のある人々が高品質のコンテンツを作成するためのツールになると言いました。Tim Berners-Leeは喜ぶでしょう：セマンティクスルール。そして、ティム・オライリーも正しいでしょう。web3.0は、「インターネットと現実の世界の相互作用」、つまり「ネットワークに入る」という言葉を忘れたときに、オンラインとオフラインの境界線を消去することについてです。
