Spring Testing Framework


Springフレームワークの主なアイデアの1つは、開発者がシンプルで疎結合のクラス、プログラミングインターフェイスを作成して、ソフトウェアの信頼性と拡張性を高めることです。 Springフレームワークは、ユニットテストと統合テストを簡略化するツールを提供します（実際に、実際にインターフェイスをプログラミングする場合、Springアプリケーションの機能をテストする必要はありません）。言い換えると、アプリケーションは、JUnitまたはTestNGテストシステムを使用して、オブジェクトに基づいてテストできる必要があります（単にnew演算子を使用して作成され、Springや他のコンテナーは使用されません）。

Springフレームワークには、アプリケーションの単体テストまたは統合テスト用に設計されたいくつかのパッケージが含まれています。ユニットテストでは、いくつかのシミュレーションオブジェクト（環境、PropertySource、JNDI、サーブレット、リアクティブテストユーティリティのServerHttpRequestおよびServerHttpResponse）を使用して、コードの分離テストを実行できます。

Springフレームワークで最も一般的に使用されるテスト機能の1つは、統合テストです。その主なタスク：
  • テスト実行間のSpring IoCコンテナーキャッシングの管理
  • トランザクション管理;
  • テストオブジェクトインスタンスの依存性注入
  • Spring専用に設計された基本クラスの作成。

Springフレームワークは、アプリケーションコンテキストテスト（ApplicationContext）に統合することで簡単にテストできます。Springテストモジュールは、ApplicationContextをプログラムで使用したり、注釈を使用したりするいくつかの方法を提供します。

  • 抽象BootstrapWith。TestContext Springフレームワークのブートストラップを構成するためのクラスレベルのアノテーション。
  • @ContextConfigurationアノテーション。統合テストのためにApplicationContextをロードして構成する方法を定義するクラスレベルのメタデータを定義します。これは、ApplicationContextがすべてのコンポーネント定義をロードする場所であるため、クラスの必須アノテーションです。
  • @WebAppConfigurationを抽象化します。統合テストのためにロードされたアプリケーションコンテキストがWebApplicationContextであることを示すクラスレベルの注釈。
  • アノテーション@ActiveProfile。統合テストのApplicationContextをロードするときにアクティブにする必要があるコンポーネント定義プロファイルを示すクラスレベルの注釈。
  • @TestPropertySourceを抽象化します。統合テスト用にロードされたApplicationContextの環境で設定されたPropertySourceオブジェクトに追加されるプロパティファイルと埋め込みプロパティの場所を指定するように設計されたクラスレベルの注釈。
  • @DirtiesContextを抽象化します。使用されたApplicationContextがテスト中に「ダーティ」であり（たとえば、単一のコンポーネントの状態を変更することによって変更または損傷された）、閉じる必要があることを示します。

これらはすべての注釈からはほど遠いです;他にもたくさんあります（@TestExecutionListeners、 コミット、 ロールバック、@ BeforeTransaction、@ AfterTransaction、 SQL、 SQLグループ SQL構成 時限、 繰り返す、@ IfProfileValueなど）。

ご覧のとおり、Springフレームワークには多くのテストオプションがあります。通常、@ RunWithアノテーションは常に使用され、テストフレームワークのすべての要素をリンクします。例えば：

@Transactional

        }

Spring Boot Testing Framework


Spring Bootは、Spring Testing Frameworkの機能を利用して、古いものを拡張し、新しい注釈と機能を追加して、開発者のテストを容易にします。

Spring Bootテストのすべての機能の使用を開始するには、テスト（スコープテスト）スコープを持つアプリケーションにspring-boot-starter-test依存関係を追加するだけです。 Spring Initializrサービスはすでにこの依存関係を追加しています。

spring-boot-starter-test依存関係は、Spring Bootテスト機能と非常に一貫性のあるいくつかのテストフレームワークを使用する機能を提供します：JUnit、AssertJ、Hamcrest、Mockito、JSONassert、およびJsonPath。もちろん、Spring Boot Testモジュールで正常に動作する他のテストフレームワークもあります。対応する依存関係のみを手動で指定する必要があります。

Spring Bootは@SpringBootTestアノテーションを提供して、Springアプリケーションのテストを簡素化します。通常、Springアプリケーションをテストするときは、特定のアプリケーションの機能をテストするためにいくつかの注釈を追加する必要がありますが、Spring Bootでは行いません。ただし、テストの場合は、@ RunWith注釈（SpringRunner.class）を指定する必要があります。これが行われない場合、新しいSpring Bootテストアノテーションは無視されます。@SpringBootTestアノテーションには、RANDOM_PORTやDEFINED_PORTなどのWebアプリケーションのテストに役立つオプションがあります。

次のコードスニペットは、Spring Bootテストフレームワークです。

@RunWith(SpringRunner.class)
@SpringBootTest
public class MyTests {

        @Test
        public void exampleTest() {
                 ...
        }
}

Webアプリケーションのエンドポイントテスト


Spring Bootはエンドポイントをテストするための便利な方法を提供します：MockMvcクラスと呼ばれるシミュレーション環境：

@RunWith(SpringRunner.class)
@SpringBootTest
@AutoConfigureMockMvc
public class MockMvcToDoTests {

        @Autowired
        private MockMvc mvc;

        @Test
        public void toDoTest() throws Exception {
              this.mvc
              .perform(get("/todo"))
              .andExpect(status().isOk())
              .andExpect(content()
                   .contentType(MediaType.APPLICATION_JSON_UTF8));
        }
}

TestRestTemplateクラスを使用することもできます。

@RunWith(SpringRunner.class)
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
public class ToDoSimpleRestTemplateTests {

      @Autowired
      private TestRestTemplate restTemplate;

      @Test
      public void toDoTest() {
            String body = this.restTemplate.getForObject("/todo", String.class);
            assertThat(body).contains("Read a Book");
      }
}

このコードは、サーバー全体を起動し、TestRestTemplateクラスのインスタンスを使用して/ todoエンドポイントにアクセスするテストを示しています。ここでは、Stringオブジェクトが返されることを前提としています（これはJSONの戻りをテストする最良の方法ではありません。心配する必要はありません。TestRestTemplateクラスを正しく使用する方法を示します）。

コンポーネントの模倣


Spring Bootテストモジュールは、ApplicationContextのコンポーネントのMockitoシミュレーションオブジェクトを記述する@MockBeanアノテーションを提供します。つまり、このアノテーションを追加して、新しいSpringコンポーネントの模倣を作成したり、既存の定義を置き換えることができます。これらすべてがApplicationContext内で発生することを覚えておいてください。

@RunWith(SpringRunner.class)
@SpringBootTest
public class ToDoSimpleMockBeanTests {

        @MockBean
        private ToDoRepository repository;

        @Test
        public void toDoTest() {
              given(this.repository.findById("my-id"))
                    .Return(new ToDo("Read a Book"));
              assertThat(
                    this.repository.findById("my-id").getDescription())
                    .isEqualTo("Read a Book");
        }
}

スプリングブートテストスライス


Spring Bootの最も重要な機能の1つは、特定のインフラストラクチャを必要とせずにテストを実行することです。Spring Bootテストモジュールには、サーバーやDBMSを使用せずにアプリケーションの特定の部分をテストするように設計された、いわゆるスライスが含まれています。

@JsonTestアノテーション


Spring Bootテストモジュールには@JsonTestアノテーションがあり、JSONオブジェクトのシリアライズ/デシリアライズを簡素化し、すべてが正しく機能するかどうかを確認します。@JsonTestは、クラスパスにあるライブラリ（Jackson、GSON、またはJSONB）に応じて、サポートされているJSON表示ツールを自動的に構成します。

@RunWith(SpringRunner.class)
@JsonTest
public class ToDoJsonTests {

      @Autowired
      private JacksonTester<ToDo> json;

      @Test
      public void toDoSerializeTest() throws Exception {
            ToDo toDo = new ToDo("Read a Book");

            assertThat(this.json.write(toDo))
            .isEqualToJson("todo.json");
            assertThat(this.json.write(toDo))
            .hasJsonPathStringValue("@.description");

            assertThat(this.json.write(toDo))
            .extractingJsonPathStringValue("@.description")
            .isEqualTo("Read a Book");
      }

      @Test
      public void toDoDeserializeTest() throws Exception {
            String content = "{\"description\":\"Read a Book\",\"completed\":
                  true }";
            assertThat(this.json.parse(content))
                  .isEqualTo(new ToDo("Read a Book", true));
            assertThat(
               this.json.parseObject(content).getDescription())
            .isEqualTo("Read a Book");
      }
}

本格的なサーバーを使用せずにコントローラーをテストするには、Spring Bootの提案されたアノテーション@WebMvcTestを使用できます。これにより、Spring MVCインフラストラクチャが自動的に構成され、表示されるコンポーネントのリストが次のように制限されます。 コントローラ、 コントローラAdvice、@ JsonComponent、Converter、GenericConverter、Filter、WebMvcConfigurerおよびHandlerMethodArgumentResolver; これにより、コントローラーが期待どおりに動作するかどうかを確認できます。

としてマークされているものを理解することが重要です成分 このアノテーションを使用するとコンポーネントは表示されませんが、必要に応じて@MockBeanアノテーションを使用できます。

@RunWith(SpringRunner.class)
@WebMvcTest(ToDoController.class)
public class ToDoWebMvcTest {

      @Autowired
      private MockMvc mvc;

      @MockBean
      private ToDoRepository toDoRepository;

      @Test
      public void toDoControllerTest() throws Exception {
            given(this.toDoRepository.findById("my-id"))
                  .Return(new ToDo("Do Homework", true));
                  this.mvc.perform(get("/todo/my-id").accept(MediaType.APPLICATION_
                     JSON_UTF8))
                        .andExpect(status().isOk()).andExpect(content().
                           string("{\"id\":\"my-id\",\"description\":\"Do Homework\",
                           \"completed\":true}"));
      }
}

@WebFluxTestアノテーション


リアクティブコントローラーの場合、Spring Bootは@WebFluxTestアノテーションを提供します。このアノテーションは、Spring WebFluxモジュールインフラストラクチャを自動的に構成し、検索のみを行いますコントローラ、 コントローラAdvice、@ JsonComponent、Converter、GenericConverterおよびWebFluxConfigurer。

としてマークされているものを理解することが重要です成分 このアノテーションを使用するとコンポーネントは表示されませんが、必要に応じて@MockBeanアノテーションを使用できます。

@DataJpaTestアノテーション


JPAアプリケーションをテストするために、Spring Bootテストモジュールは@DataJpaTestアノテーションを提供し、RAMにある組み込みデータベースを自動的に構成します。彼女は探していますエンティティ コンポーネントをロードしません 成分。さらに、JPA EntityManagerクラスとよく似た、テスト指向のヘルパークラスTestEntityManagerを提供します。

@RunWith(SpringRunner.class)
@DataJpaTest
public class TodoDataJpaTests {
  
      @Autowired
      private TestEntityManager entityManager;

      @Autowired
      private ToDoRepository repository;

      @Test
      public void toDoDataTest() throws Exception {
            this.entityManager.persist(new ToDo("Read a Book"));
            Iterable<ToDo> toDos = this.repository.
                   findByDescriptionContains("Read a Book");
            assertThat(toDos.iterator().next()).toString().contains("Read a Book");
      }
}

@JdbcTestアノテーション


この注釈は、@ DataJpaTestによく似ています。唯一の違いは、JDBCテストでのみ実行されることです。注釈付きのものをすべてスキップしながら、RAMにある組み込みDBMSとJdbcTemplateクラスを自動的に構成します。成分 クラス。

@RunWith(SpringRunner.class)
@JdbcTest
@Transactional(propagation = Propagation.NOT_SUPPORTED)
public class TodoJdbcTests {

      @Autowired
      private NamedParameterJdbcTemplate jdbcTemplate;

      private CommonRepository<ToDo> repository;

        @Test
        public void toDoJdbcTest() {
              ToDo toDo = new ToDo("Read a Book");

            this.repository = new ToDoRepository(jdbcTemplate);
            this.repository.save(toDo);

            ToDo result = this.repository.findById(toDo.getId());
            assertThat(result.getId()).isEqualTo(toDo.getId());
        }
}


@DataMongoTestアノテーション


MongoDBアプリケーションをテストするために、Spring Bootテストモジュールは@DataMongoTestアノテーションを提供します。可能な場合は、組み込みのメモリベースのMongoサーバーを自動的に構成します。そうでない場合は、spring.data.mongodbの必要なプロパティを追加する必要があります。また、MongoTemplateクラスを調整し、@ Documentアノテーションを探します。部品成分 スキップされます。

@RunWith(SpringRunner.class)
@DataMongoTest
public class TodoMongoTests {

      @Autowired
      private MongoTemplate mongoTemplate;

      @Test
      public void toDoMongoTest() {
          ToDo toDo = new ToDo("Read a Book");
          this.mongoTemplate.save(toDo);

          ToDo result = this.mongoTemplate.findById(toDo.getId(),ToDo.class);
          assertThat(result.getId()).isEqualTo(toDo.getId());
      }
}

外部MongoDBサーバー（非ビルド、インメモリー）が必要な場合は、excludeAutoConfiguration = EmbeddedMongoAutoConfiguration.classパラメーターを@DataMongoTestアノテーションに追加します。

@RunWith(SpringRunner.class)
@DataMongoTest(excludeAutoConfiguration = EmbeddedMongoAutoConfiguration.class)
public class ToDoMongoTests {
        // ...
}

@RestClientTestアノテーション


もう1つの重要なアノテーションは、RESTクライアントをテストするために設計された@RestClientTestです。このアノテーションは、Jackson、GSON、JSONBをサポートする設定を自動的に行い、RestTemplateBuilderクラスをセットアップして、MockRestServiceServerのサポートを追加します。

@RunWith(SpringRunner.class)
@RestClientTest(ToDoService.class)
public class ToDoRestClientTests {

      @Autowired
      private ToDoService service;

      @Autowired
      private MockRestServiceServer server;

      @Test
      public void toDoRestClientTest()
                  throws Exception {
             String content = "{\"description\":\"Read a Book\",\"completed\":
                   true }";
            this.server.expect(requestTo("/todo/my-id"))
             .andRespond(withSuccess(content,MediaType.APPLICATION_JSON_UTF8));
            ToDo result = this.service.findById("my-id");
            assertThat(result).isNotNull();
            assertThat(result.getDescription()).contains("Read a Book");
      }
}
