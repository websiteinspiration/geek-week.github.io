Free Software Foundationは、共同開発ツールをサポートし、以前に開発されたフリーソフトウェアホスティングの倫理基準を満たす新しいコードホスティングを作成する計画を発表しました。新しいプラットフォームは、既存のサバンナホスティングに加えて作成され、サポートは継続されます。新しいプラットフォームを作成する目的は、オープンソースソフトウェアを開発するためのインフラストラクチャの問題を解決することです。現在、多くの無料プロジェクトは、コードを公開せず、独自仕様のソフトウェアの使用を強制しないプラットフォームに依存しています。このプラットフォームは2020年に稼働を開始する予定であり、個々の企業の利益に結び付けられていない独立したコミュニティによって開発されたコードに取り組むためのすでに作成された無料のソリューションに基づいて構築されます。最も可能性の高い入札者はPagureプラットフォームと見なされ、Fedora Linux開発者によって開発されました。

FOSSライセンス：選択するライセンスとその理由




Ars Technicaは、プロジェクトにFOSSライセンスを選択する問題の詳細な分析を公開し、どのライセンスが存在するか、どのように異なるか、なぜプロジェクトのライセンスの選択が非常に重要であるかを説明しています。無料ライセンスとオープンライセンスの違いを理解していない場合、「お金を節約する」と「著作権」を混同し、さまざまなバージョンとプレフィックスの「これらすべて」のGPL、MPL、CDDL、BSD、Apacheライセンス、MIT、CC0、WTFPLと混同します記事は間違いなくあなたを助けます。

欧州委員会は安全上の理由から無料のシグナルメッセンジャーを選択しました




The Vergeは、欧州委員会（EUの最高行政機関）が従業員に無料のSignal暗号化メッセンジャーに切り替えて通信セキュリティを向上させることを推奨したと報告しています。Politicoは、今月初めに委員会の内部サイトに「外部連絡先の推奨アプリケーションとして信号が選択されました」というメッセージが表示されたと付け加えました。ただし、すべての通信でSignalが使用されるわけではありません。暗号化された電子メールは、未分類の重要な情報に引き続き使用され、プライバシークラスでドキュメントを転送するために特別なツールが引き続き使用されます。

Manjaro Linux 19.0ディストリビューションリリース




OpenNETによると、Marcharo Linux 19.0ディストリビューションのGNU / Linuxリリースは、Arch Linuxに基づいて構築されていますが、初心者を対象としています。 Manjaroには、よりシンプルなグラフィカルインストーラー、自動ハードウェア検出およびドライバーインストールのサポートがあります。配布は、グラフィカル環境KDE、GNOME、Xfceを含むライブアセンブリの形式で提供されます。 Manjaroは、Gitのイメージで設計された独自のBoxItツールキットを使用して、リポジトリを管理します。独自のリポジトリに加えて、AURリポジトリ（Arch User Repository）を使用するためのサポートがあります。バージョン19.0では、Linux 5.4カーネルが導入され、Xfce 4.14の更新バージョン（Matchaの新しいテーマを含む）、GNOME 3.34、KDEプラズマ5.17、KDEアプリ19.12.2が追加されました。 GNOMEは、さまざまなテーマのデスクトップモードスイッチを提供します。バージョン9まで。3 Pamacパッケージマネージャーが更新され、デフォルトでスナップ形式とフラットパック形式の自己完結型パッケージのサポートが含まれています。これは、新しいBauhアプリケーション管理インターフェースを介してインストールできます。

Open Cybersecurity Alliance




ZDNetは、サイバーセキュリティ関連のプログラム間でデータとコマンドを交換するために設計されたフレームワークであるOpenDXL Ontologyの登場を発表しました。サイバーセキュリティツール間の断片化を埋めるために設計された新しいフレームワークは、オープンソースコミュニティに導入されました。 OpenDXL Ontologyは、IBM、Crowdstrike、McAfeeなどのサイバーセキュリティプロバイダーのコンソーシアムであるOpen Cyber​​security Alliance（OCA）によって開発されました。 OCAによると、OpenDXLオントロジーは「一般的なメッセージングシステムを介してサイバーセキュリティツールを接続するための最初のオープンソース言語」です。 OpenDXLオントロジーは、ツールとサイバーセキュリティシステムの間で共通の言語を作成することを目的としており、相互に作用するときに最も効果的な製品間のカスタム統合の必要性を排除します。エンドシステム、ファイアウォールなどの間で行われますが、断片化とベンダー固有のアーキテクチャの影響を受けます。

プログラマーとミュージシャンがアルゴリズム的に可能なすべてのメロディーを生成し、それらをパブリックドメインにした




Habrで終了しましょう。真実は無料のオープンソースソフトウェアにも直接当てはまりませんが、著作権とコピーレフトは同じで、アートのみです。ソフトウェアの弁護士であるダミアンリエルとミュージシャンのノアルービンの2人の愛好家は、盗用された音楽の申し立てによる著作権侵害の訴訟の問題を根本的に解決しようとしました。彼らが開発したソフトウェアアルゴリズム（Creative Commons Attribution 4.0ライセンスでGitHubから入手可能）を使用してすべての音楽を作成し、「1オクターブに含まれるすべての可能なメロディーを生成し、保存し、このアーカイブを著作権で保護し、パブリックドメインにした。将来これらのメロディーが知的財産権に該当しないようにするためです。」生成されたすべての曲は、インターネットアーカイブに1.2 TBのMIDI形式で公開されます。ダミアンリエルは、TEDでこのイニシアチブについても話しました。
