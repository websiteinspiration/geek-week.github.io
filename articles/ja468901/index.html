React Native-アプリケーションと批評

ほとんどの場合、この言語を選択すると、2つのプラットフォーム用の1つのアプリケーションの開発に2つのアプリケーションの開発より半分の時間がかかることが予想されます。しかし、結局のところ、外見上の輝きとマーケティングに隠された困難のために、開発にはそれ以上ではないにしても多くの時間がかかることがわかりました。React Nativeでの作業の最後の数か月で発生した同様の問題のいくつかについて説明します。


React Nativeは、Javascriptをモバイル開発に適応させます。これは、彼がプロジェクトをビルドするためにいくつかのコレクターを使用するという事実によって達成されます-Metro BundlerはJSコードを解釈し、ターゲットシステムのリソースとコレクターを表します。私たちの場合、それはAndroidのグラドルでした。理論的には、React Nativeアプリは非常に単純に実行する必要があります。 react-native run-androidコマンドはMetro Bundlerを有効にし、接続されているすべてのAndroidデバイスとエミュレーター用のアプリケーションを構築します。


実際には、この段階でも困難があることがわかりました。私たちのプロジェクトには常に「JSバンドルをダウンロードできません」というエラーが表示されました。これは、バンドラーがコードをネイティブコードに変換できなかったことを意味します。後で判明したように、起動しなかったためです。 StackOverflowは問題を確認し、react-native startコマンドを使用して別のスレッドでバンドラーを実行することを提案しました。これにより、package.jsonが変更された場合にのみ、バンドラーを再起動できます。これは、手順によって開発が大幅に遅くなることはないためです。


Package.jsonは、アプリケーションの一連の外部モジュールを含むファイルです。 npmjs.comには、React Native用のさまざまなライブラリが多数あり、機能を拡張して開発を簡素化しています。多くのライブラリ（Firebaseなど）はネイティブ関数を使用するため、ネイティブコードに直接関連付ける必要があります。これを行うには、react-native link <library-name>コマンドを使用します。これにより、これらの関係がネイティブコードと構成されます。
