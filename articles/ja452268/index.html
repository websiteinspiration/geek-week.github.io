C＃WPFアナログWindow.ShowDialog（）またはDispatcherFrameを処理する

問題の定式化


1つのアプリケーションの開発の一環として、次のスキームを実装する必要がありました。


  1. 非同期メソッドはデータを要求します
  2. ユーザーがキーボードからデータを入力する
  3. メソッドは、関数の実行結果として入力結果を受け取り、同じ場所から続行します

追加要件：追加のウィンドウを作成しないでください。


それは単純に見えるでしょうか？それは本当にシンプルであることがわかりました。しかし、まず最初に。


決定


インターネットを検索せずに正面からそれを行う最初の試みは、メインストリームの妨害につながり、それゆえ、良くありませんでした。そして、私は記事に出くわしたので、ShowDialogを使用しようとしていました。著者は、WPFでShowDialogがどのように作成されるかを見ました。まさに必要なもの！


彼の記事では、ShowDialogメソッドの独自の実装を作成することを提案しています


[DllImport("user32")]
internal static extern bool EnableWindow(IntPtr hwnd, bool bEnable);

public void ShowModal()
{
    IntPtr handle = (new WindowInteropHelper(Application.Current.MainWindow)).Handle;
    EnableWindow(handle, false);

    DispatcherFrame frame = new DispatcherFrame();

    this.Closed += delegate
    {
        EnableWindow(handle, true);
        frame.Continue = false;
    };

    Show();
    Dispatcher.PushFrame(frame);
}
