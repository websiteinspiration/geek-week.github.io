入力文字列データをクリアする


ユーザーが入力したデータを消去するタスクは、ほとんどすべてのプログラムに関連しています。多くの場合、この入力処理は文字を大文字または小文字に変換することになります。正規表現を使用してデータをクリアできる場合があります。ただし、タスクが複雑な場合は、より効果的な方法で解決できます。例えば-これ：

user_input = "This\nstring has\tsome whitespaces...\r\n"

character_map = {
 ord('\n') : ' ',
 ord('\t') : ' ',
 ord('\r') : None
}
user_input.translate(character_map)  # This string has some whitespaces... "

ここでは、どのように空白見ることができます"\n"し、"\t"通常のスペースに置き換えられ、シンボルは次のように"\r"完全にラインから削除されます。これは簡単な例ですが、パッケージunicodedataとその関数を使用して大きな文字の再割り当てテーブルを作成することで拡張できますcombining()。このアプローチにより、必要のないものすべてを行から削除できます。

反復可能なオブジェクトの開始をスキップ


ご存知のように、コメントのある行など、不必要な行から始まるファイルで作業する必要がある場合があります。これらの行をスキップするために、もう一度、可能性に訴えることができますitertools：

string_from_file = """
// Author: ...
// License: ...
//
// Date: ...

Actual content...
"""

import itertools

for line in itertools.dropwhile(lambda line: line.startswith("//"), string_from_file.split("\n")):
 print(line)

このコードは、ファイルの先頭にあるコメントブロックの後の行のみを返します。この方法は、反復可能オブジェクトの先頭にある要素（この場合は行）だけを破棄する必要があるが、正確な数がわからない場合に役立ちます。

名前付き引数のみをサポートする関数（kwargs）


特定の関数を使用する場合、名前付きの引数のみが渡されるようにするには、次のようにします。

def test(*, a, b):
 pass

test("value for a", "value for b")  # TypeError: test() takes 0 positional arguments...
test(a="value", b="value 2")  #   - ...

これは、コードのわかりやすさを向上させるために役立ちます。ご覧のとおり*、名前付き引数のリストの前に引数を使用することで、タスクが簡単に解決されます。ここでは非常に明白ですが、位置引数を使用することもできます-引数の前に配置する場合*。

withステートメントをサポートするオブジェクトの作成


たとえば、ファイルを開く方法や、場合によっては、オペレーターを使用してロックを設定する方法を誰もが知っていますwith。しかし、ロック制御メカニズムを独立して実装することは可能ですか？はい、それはかなり現実的です。実行コンテキスト管理プロトコルは、メソッド__enter__とを使用して実装されます__exit__。

class Connection:
 def __init__(self):
  ...

 def __enter__(self):
  #  ...

 def __exit__(self, type, value, traceback):
  #  ...

with Connection() as c:
 # __enter__() executes
 ...
 # conn.__exit__() executes

これは、Pythonでコンテキストマネージャーの機能を実装する最も一般的な方法ですが、同じことをより簡単に行うことができます。

from contextlib import contextmanager

@contextmanager
def tag(name):
 print(f"<{name}>")
 yield
 print(f"</{name}>")

with tag("h1"):
 print("This is Title.")

ここでは、コンテキスト管理プロトコルは、デコレータを使用して実装されていcontextmanagerます。関数のtag前の部分（before yield）は、ブロックに入るときに実行されますwith。次に、このブロックが実行され、その後、残りの関数が実行されtagます。

__slots__でメモリを節約する


特定のクラスのインスタンスを本当に大量に作成するプログラムを作成したことがある場合、そのようなプログラムが予期せず大量のメモリを必要とすることに気付くでしょう。これは、Pythonが辞書を使用してクラスインスタンスの属性を表すためです。これはパフォーマンスに良い影響を与えますが、メモリ消費の点では非効率的です。ただし、通常、この機能は問題を引き起こしません。ただし、同様の状況でメモリ不足に直面している場合は、属性を使用してみてください__slots__。

class Person:
 __slots__ = ["first_name", "last_name", "phone"]
 def __init__(self, first_name, last_name, phone):
  self.first_name = first_name
  self.last_name = last_name
  self.phone = phone

ここで、属性を宣言すると__slots__、Pythonは固定サイズの小さな配列を使用して、辞書ではなく属性を格納します。これにより、クラスの各インスタンスに必要なメモリの量が大幅に削減されます。属性の使用に__slots__はいくつかの欠点があります。したがって、それを使用すると、新しい属性を宣言できません。使用できるのは、で使用できる属性に限定され__slots__ます。さらに、属性__slots__を持つクラスは多重継承を使用できません。

CPUとメモリの制限


プログラムを最適化したり、プロセッサの使用方法を改善したりする代わりに、使用可能なリソースに厳密な制限を設定するだけでよい場合は、適切なライブラリを使用できます。

import signal

import os

 print("CPU exceeded...")

def set_max_runtime(seconds):
 #   signal

モジュールからインポートできるものとできないものの制御


一部の言語には、変数、メソッド、およびインターフェースのモジュールからの非常に明確なエクスポートメカニズムがあります。たとえば、名前が大文字で始まるエンティティのみがGolangにエクスポートされます。Pythonでは、すべてがエクスポートされます。ただし、属性が使用されるまでのみ__all__：

 pass

__all__ = ["bar"]

上記の例では、関数のみがエクスポートされbarます。また、属性を__all__ 空のままにすると、モジュールから何もエクスポートされません。そのようなモジュールから何かをインポートしようとすると、エラーが生成されAttributeErrorます。

比較演算子の作成を簡略化する


多くの比較演算子があります。例えば- 、__lt__、__le__、。__gt__ __ge__特定のクラスへの実装の見通しを好む人はほとんどいません。この退屈な作業を簡略化する方法はありますか？はい、できます-デコレータの助けを借りてfunctools.total_ordering：

from functools import total_ordering

@total_ordering
class Number:
 def __init__(self, value):
  self.value = value

 def __lt__(self, other):
  return self.value < other.value

 def __eq__(self, other):
  return self.value == other.value

print(Number(20) > Number(3))
print(Number(1) < Number(5))
print(Number(15) >= Number(15))
print(Number(10) <= Number(2))

functools.total_orderingここで は、デコレータを使用して、クラスインスタンスの順序を実装するプロセスを簡略化しています。その動作を保証するために必要なのは、比較演算子__lt__とを宣言することだけ__eq__です。これは、デコレーターが残りの比較演算子を構成するために必要な最小値です。
