クリーンなSILコード

SILは、Atlassian JiraとConfluenceで作業を自動化するためのプログラミング言語です。SILの詳細については、こちらをご覧ください。

私はSILで記述されたスクリプトを使用することが多いので、SILコードを「クリーン」にする方法について私の考えを共有したいと思います。

この記事では、最初にSILコードを記述する際のガイドとなるルールを作成し、次にコードの例を示し、これらのルールを使用してリファクタリングを行います。

ルール


  1. 名前の構造は大文字で始まります。
  2. AddElementメソッドを使用して配列要素を追加します。
  3. ユーザー定義ルーチンを使用して、コードを論理ブロックに分割し、コード内のコメントを取り除きます。
  4. 再利用可能なコードをライブラリーに組み込みます（組み込み）。
  5. 残りの変数宣言の前に構造を宣言します。
  6. 構造と変数に意味のある名前を付けます。この場合、構造と変数にコメントを追加する必要はありません。
  7. Google Javaスタイルガイドに従って変数と関数に名前を付けます。

コードを確認することは、コードが何をしているかをすばやく判断することは非常に困難です。コード内のコメントもあまり役に立ちません。上記のルールに従ってコードを変更してみましょう。

最初に構造を見てください。以下はその一例です。

struct body {
    storage storage;
}

「ストレージストレージ」はわかりにくいように見えます。コードが次のようになっていると、より明確になります。

struct body {
    Storage storage;
}

これで、構造体にStorageタイプのstorageという名前でフィールドを定義していることがわかります。このようなコードを作成するには、各構造の最初の文字を大文字にする必要があります。

私の意見では、そのような名前は構造について追加のコメントを必要としません。また、コードの最初で構造体を宣言します。

コードを見ると、コードで何が起こっているのかすぐに理解できません。まず、コードを論理ブロックに分割してみましょう。

getNewConfluencePageContent();
createHttpRequest();
createConfluencePage();
addCommentToJiraIssue();

つまり、最初にConfluenceの新しいページのコンテンツを取得し、次にhttpリクエストを作成し、Confluenceにページを作成し、JiraリクエストにConfluenceのページが作成されるというコメントを書き込みます。これらの関数を実装しましょう：

function getNewConfluencePageContent() {
    CreateConfluencePageRequest data;
    data.type = "page";
    data.title = "Page for issue " + key + "  " + summary + ".";
    data.space.key = project;
    data.body.storage.value = "<p> Author:"+userFullName(currentUser()) + " description: "  + description + "</p>";
    data.body.storage.representation = "storage";
    return toJson(data);
}

function createHttpRequest() {
    HttpRequest request;
    HttpHeader header = httpCreateHeader("Content-Type", "application/json");
    HttpHeader authHeader = httpBasicAuthHeader("admin", "admin");
    request.headers = addElement(request.headers, header);
    request.headers += addElement(request.headers, authHeader);
    logPrint("WARN", request);
    return request;
}

function createConfluencePage(string pageJson) {
    HttpRequest request = createHttpRequest();
    CreateConfluencePageResponse result = httpPost("http://192.168.54.203:8090/rest/api/content/", request, pageJson);
    logPrint("ERROR", "Last error message: " + httpGetErrorMessage());
    logPrint("WARN", result);
    return result;
}

function addCommentToJiraIssue(string resultStatus) {
    string COMMENT = "Page created in Confluence " + updated + " by " + userFullName(currentUser()) + " Status : " +resultStatus + "."; 
    addComment(key, currentUser(), COMMENT);
}

string pageJson = getNewConfluencePageContent();
CreateConfluencePageResponse result = createConfluencePage(pageJson);
addCommentToJiraIssue(result.status);
return result.status;

関数の論理ブロックを削除しました（ルール2）。これで、次のようなコードを簡単に作成できます。

string pageJson = getNewConfluencePageContent();
CreateConfluencePageResponse result = createConfluencePage(pageJson);
addCommentToJiraIssue(result.status);
return result.status;

コードには4つの論理ブロックが含まれています。最初に新しいページのJSONを取得してから、Confluenceでページを作成し、リクエストにコメントを追加して、ページ作成ステータスをConfluenceに返します。これらの4行は、スクリプトが何をするかのトップレベルのビューを提供します。Confluenceのページのコンテンツを理解したい場合は、いつでもgetNewConfluencePageContentに移動して、この関数のコンテンツを確認できます。この関数には、ページのコンテンツを作成するためのコードのみが含まれているため、関数コードを学習している間、他の機能に邪魔されることはありません。

1つのスクリプトだけでなくConfluenceでページを作成する必要があると想定できます。したがって、confluence_helper.inclファイルを作成して、Confluenceでページを作成するために必要なすべての関数をこのファイルに配置します。

struct Space {
    string key;
}
struct Storage {
    string value;
    string representation;
}
struct Body {
    Storage storage;
}
struct CreateConfluencePageRequest {
    string type;
    string title;
    Space space;
    Body body;
}
struct CreateConfluencePageResponse {
    string status;
}

function createHttpRequest() {
    HttpRequest request;
    HttpHeader header = httpCreateHeader("Content-Type", "application/json");
    HttpHeader authHeader = httpBasicAuthHeader("admin", "admin");
    request.headers = addElement(request.headers, header);
    request.headers += addElement(request.headers, authHeader);
    logPrint("INFO", request);
    return request;
}

function createConfluencePage(string confluenceUrl, string pageJson) {
    HttpRequest request = createHttpRequest();
    CreateConfluencePageResponse result = httpPost(confluenceUrl, request, pageJson);
    logPrint("ERROR", "Last error message: " + httpGetErrorMessage());
    logPrint("INFO", result);
    return result;
}

createConfluencePage関数に別のconfluenceUrlパラメーターを追加して、より一般的なものにしました。したがって、ルール4に従いました。

メインスクリプトは次のようになります。

include "confluence_helper.incl";

function getNewConfluencePageContent() {

    string comment = "Page created in Confluence " + updated + " by " + userFullName(currentUser()) + " Status : " +resultStatus + ".";

const string CONFLUENCE_URL = "http://192.168.54.203:8090/rest/api/content/";

最初の行には、Confluenceのページ作成機能を使用するためのconfluence_helper.inclファイルを含めました。

Confluenceアドレスを定数変数に保存し、変数名をGoogle Java表記スタイル（ルール7）に従って変更しました。

これはリファクタリングで止められるように思えます。リファクタリングの結果、Confluenceに再利用可能なページ作成機能が追加され、コードがより読みやすく、サポートされるようになりました。
