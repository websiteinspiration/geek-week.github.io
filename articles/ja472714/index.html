フロントランナーが仕事を探すためにどこに落ちないか：電報、スラック、そして

彼らが採用担当者と話すボットやメール用のあらゆる種類のスパムフィルターについて話している資料はすでにありました。数週間前、私も転職することにしました。私は一つの仕事をしました-特に輝かないために。



そして、私は2つの理由で仕事を注意深く検索したいと思いました。

  1. 現在の雇用主とまだ完了していないプロジェクトに対する責任があるため、提案を検討していることを宣伝したくありませんでした。
  2. たくさんのスパムを捕まえて、自分が誰を探しているのかわからない企業のインタビューに行きたくありません。

すべてがどのように始まったか。さて、最初に履歴書をHeheに投稿しました（きちんと匿名でもかまいません）が、10分後に飛んで



いきました。検索プロセスを自分で制御することに決め、そこから履歴書を解体しました。

彼は、これを実行できるプラットフォーム、電報チャネルとチャット、Slackコミュニティを自分で集めました。そして、エキゾチックな愛好家のために-Skype。

インタビューを受けている最中。そして私は選択を共有します：

プラットフォーム：


ジンニ -私は彼女について多くのことを聞きましたが、主にウクライナの聴衆がいて、私は何も賢明なものを得られませんでした（SimilarWeb統計で判断します）。

Arenaは匿名で検索できる新しいプラットフォームです。欠員が来て、「興味がある」または「興味がない」をクリックします。最初の場合、会社はあなたの連絡先を見て、償却します。私はいくつかの欠員、すべてのルールを得ました。1つのように。

電報チャット：


pro js-とりわけ、空席を見つけることができるチャット。

Javscript Jobs（@javascript_jobs）-多くの話がありますが、欠員もいます。ほとんどの場合、それらは@js_ruで複製されます。

電報チャネル：


JavaScript Jobs（@jsdevjob）は、Proglibコミュニティチャネルであり、賢明なポジションがありますが、非常にまれに投稿します。

JavaScriptジョブ（@js_ru）は、JSジョブの最大のチャネルです。それらが厳しく管理されていて、どこにでも確実にプラグがあるのはクールです。異なるフレームワーク用のチャネルがあります：Node JS Jobs（@nodejs_jobs）、React JS（@react_js）、Angular JS（@angular_ru）。

Web Freelance（@webfrl）-ある種のプロジェクト作業またはリモートサイトに固執できるチャネル。

TProger（@tproger_official）は、典型的なプログラマーのチャネルです。ここでは、記事や同様のあらゆるものに加えて、欠員もいます。

フロントエンド開発者ジョブ（@front_end_jobs）は非常に小さなチャネルですが、興味深い位置がここに出てくることがあります。

プログラマー向けの欠員（@jobGeeks）-すべての行で、＃frontendタグを使用して賢明なものを取得できます。

React JSは国際的なコミュニティですが、空席のある支部もあります。
