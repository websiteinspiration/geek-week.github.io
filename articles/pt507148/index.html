Envie links mágicos usando o Node.js

A tradução do artigo foi preparado antes do lançamento do desenvolvedor Node.js curso .


O método mais popular de entrar no aplicativo é fornecer um nome de usuário e senha, mas os usuários podem encontrar certas desvantagens nisso. Embora existam muitos gerenciadores de senhas, usá-los leva tempo. Não muito, mas ainda. Alguns aplicativos promovem uma política de senhas complexa e, portanto, decepcionante, portanto, criar uma nova senha pode ser complicado. Um sistema sem senha pode ser uma boa alternativa à autenticação convencional por login e senha. É usado pelo Slack e pelo FramerX e parece enviar um link mágico para o usuário para habilitar essa autenticação no aplicativo. Um sistema sem senha funciona da seguinte maneira:

  1. O usuário visita o aplicativo, onde há um formulário para inserir um endereço de email;
  2. O usuário digita um endereço de email e pressiona o botão de confirmação;
  3. Um link mágico para entrar no sistema é enviado ao email do usuário;
  4. O usuário clica no link e é redirecionado para o aplicativo em que ele já está conectado.


Depois disso, o link mágico não é mais válido.
Abaixo, veremos alguns pontos-chave para implementar a autorização sem senha no Node.js.

Vamos criar um aplicativo expresso

Primeiro, precisamos criar um aplicativo expresso simples e adicionar alguns pontos de extremidade. O código abaixo é responsável por configurar um servidor expresso com auxiliares para analisar solicitações recebidas , fazer login e adicionar cors . Precisamos criar dois diretórios aos quais adicionaremos manipuladores posteriormente. Será /logine /account. O primeiro manipulador será responsável por enviar links mágicos aos usuários e o segundo - por sua autenticação.

Configurar nodemailer


Usaremos o nodemailer para enviar emails, mas primeiro você precisa configurá-lo. Vamos precisar de vários componentes: um endereço de email, uma senha de email, um host de servidor SMTP e uma porta de servidor SMTP. Também precisaremos de um modelo de carta, que enviaremos aos usuários com um link mágico. Um modelo pode ser apenas uma string ou uma página HTML.



Geração de token


Para isso, usaremos o pacote jsonwebtoken , que como um todo fará tudo por nós. Nosso token consistirá em duas propriedades: endereço de email e prazo de validade.

Autenticação de usuário


Quando um usuário faz logon no sistema, precisamos autorizar suas solicitações ao nosso servidor. Para fazer isso, faça o seguinte:
  1. Verifique se há um cabeçalho de autorização na solicitação;
  2. Verifique se o cabeçalho da autorização é bearer token;
  3. Tente decodificar esse token;
  4. Verifique se o token possui todos os campos obrigatórios. No nosso caso: emaile expiration.

Aqui está a configuração básica para o login sem senha no topo de um fluxo de autorização normal. A maior vantagem que vejo nessa abordagem é que, se os usuários puderem acessar um produto com mais rapidez e facilidade, sua experiência com o uso do produto melhorará.

Por outro lado, mesmo que o login no sistema possa ser mais fácil do que com um login / senha regular, o retorno a ele pode ser mais difícil. Cada vez que um usuário faz login, ele precisa aguardar um email, verificar sua caixa de entrada e clicar no link. Esse processo também pode ser desagradável; portanto, se uma sessão tiver curtos períodos de tempo limite ou se for esperado que os usuários façam logon no sistema com frequência, a autorização sem senha poderá não ser a melhor opção.
