aircrack-ngと一致するWi-Fiパスワード

この記事は、教育と研究を目的としてのみ作成されています。ネットワークと法律の取り扱いに関するルールを遵守し、情報セキュリティについて常に覚えておくことをお勧めします。

前書き


1990年代初頭、Wi-Fiが登場した直後、Wired Fiquivalent Privacyアルゴリズムが作成されました。これは、Wi-Fiネットワークの機密性を確保するためのものでした。ただし、WEPは簡単に解読できる効果のないセキュリティアルゴリズムであることが証明されています。

現在、ほとんどのWi-Fiアクセスポイントで使用されている新しいWi-Fi Protected Access II保護アルゴリズムが置き換えられました。WPA2は暗号化アルゴリズムであるAESを使用していますが、これは解読が非常に困難です。

そして、脆弱性はどこにありますか？


WPA2の欠点は、ユーザーがいわゆる4ウェイハンドシェイク（4ウェイハンドシェイク）中に接続すると、暗号化されたパスワードが送信されることです。ハンドシェイクをキャッチすると、暗号化されたパスワードがわかり、復号化するだけで済みます。この目的のために、私たちはaircrack-ngを使用します。

では、ハッキングする方法は？


ステップ1.インターフェースを定義する


まず、必要なネットワークインターフェイスを見つける必要があります。これには、次のコマンドを入力します。

$ ifconfig

私たちは答えを得ます：

eth0      no wireless extensions.

wlan0     IEEE 802.11abgn  ESSID:off/any
Mode:Managed  Access Point: Not-Associated   Tx-Power=15 dBm
Retry short limit:7   RTS thr:off   Fragment thr:off
Encryption key:off
Power Management:off

lo        no wireless extensions

私の場合、インターフェースは3つしかなく、そのうち2つにはワイヤレス拡張がありません。したがって、関心があるのはwlan0だけです。

手順2.ネットワークアダプターを監視モードにする


ネットワークアダプターを監視モードにすると、近くにある無線トラフィックを確認できます。これを行うには、次のコマンドを入力します。

$ airmon-ng start wlan0

airmon-ngがインターフェースの名前を変更したことに注意してください（私はmon0と呼び始めましたが、まだ確認する必要があります）。

ステップ3.トラフィックを傍受する


ネットワークアダプターが監視モードになったので、airodump-ngコマンドを使用して、通過するトラフィックをキャプチャできます。入る：

$ airodump-ng mon0



表示されているアクセスポイントはすべて画面の上部に表示され、クライアントは画面の下部に表示されます。

ステップ4.特定のアクセスポイントにインターセプトを集中させます。


次のステップは、アクセスポイントの1つとそのチャネルに注力することです。BSSIDとアクセスポイントのチャネル番号に興味があります。これらはクラックします。別のターミナルを開いて入力してみましょう：

$ airodump-ng --bssid 08:86:30:74:22:76 -c 6 -w WPAcrack mon0



  • 08：86：30：74：22：76 BSSIDアクセスポイント
  • -c Wi-Fiアクセスポイントが機能する6チャネル
  • ハンドシェイクが書き込まれるWPAcrackファイル
  • 監視モードのmon0ネットワークアダプター

上のスクリーンショットでわかるように、チャネル6でBelkin276 ESSIDを使用して1つのアクセスポイントからデータをキャプチャすることに集中しています。ターミナルを開いたままにします！

ステップ5.ハンドシェイクを取得する


暗号化されたパスワードを取得するには、クライアントに認証（Wi-Fiに接続）する必要があります。すでに認証されている場合は、認証を解除して（切断）、システムが自動的に再認証（接続）し、その結果、暗号化されたパスワードを取得できます。

つまり、接続しているユーザーを切断して、再度接続する必要があります。これを行うには、別のターミナルを開いて次のように入力します。

$ aireplay-ng --deauth 100 -a 08:86:30:74:22:76 mon0



  • 認証を解除するユーザー数100
  • 08：86：30：74：22：76 BSSIDアクセスポイント
  • mon0ネットワークアダプター

ここで、再接続すると、前のステップで残したウィンドウがハンドシェイクをキャッチします。ターミナルairodump-ngに戻って見てみましょう。



右側の最上行にairodump-ngが印刷されていることに注意してください：「Handshake WPA」。つまり、暗号化されたパスワードの取得に成功しました！これが成功への第一歩です！

ステップ6.パスワードを選択する


WPAcrackファイルに暗号化されたパスワードがあるので、パスワードの推測を開始できます。しかし、そのためには、使用したいパスワードのリストが必要です。このようなリストは、Googleで5分で見つかります。また、aircrack-ng：BackTrack darkcOdeに含まれているデフォルトのパスワードリストを使用します。

新しいターミナルを開き、次のように入力します。

$ aircrack-ng WPAcrack-01.cap -w /pentest/passwords/wordlists/darkc0de



  • ハンドシェイクを書き込んだWPAcrack-01.capファイル（-01.capの末尾にあるairodump-ng属性）
  • / pentest / passwords / wordlist / darkc0deパスワードリストへの絶対パス

どのくらい時間がかかりますか？


このプロセスには時間がかかる場合があります。それはすべてパスワードリストの長さに依存し、数分から数日待つことができます。私のデュアルコアIntelプロセッサーでは、aircrack-ngは毎秒800強のパスワードを取得します。

パスワードが見つかると、画面に表示されます。パスワードの推測が成功するかどうかは、リストによって異なります。あるリストからパスワードが見つからない場合は、絶望せずに別のリストを試してください。

使い方のヒント


  1. このタイプの攻撃は、リストからのパスワード推測には効果的ですが、ランダムな推測にはほとんど役に立ちません。すべての時間です。Wi-Fiがラテン文字と数字の平均的なパスワードで保護されている場合、ランダムな選択には数年かかります。
  2. パスワードリストを選択するときは、地理的な要素を考慮してください。たとえば、ロシアのパスワードリストに従って、パリのレストランで選択をしても意味がありません。
  3. 自宅のWi-Fiをハッキングする場合は、被害者の個人データ（名前、姓、生年月日、犬の名前など）を見つけて、このデータからパスワードの追加リストを生成してください。
  4. 握手をした後は、aireplay-ngの操作をオフにしてください（一般のユーザーを困らせないでください）。
