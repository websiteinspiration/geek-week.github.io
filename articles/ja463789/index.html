関連するタイプ、where句、添え字など...

記事「The Power of Generics in Swift。パート1」では、ジェネリック関数、ジェネリック型、型制限について説明しました。初心者の場合は、最初の部分を読んで理解を深めることをお勧めします。

プロトコルを定義するとき、1つ以上の関連する型を定義の一部として宣言すると便利な場合があります。関連付けられたタイプは、プロトコルの一部として使用されるタイプのスタブ名を指定します。この関連タイプに使用される実際のタイプは、プロトコルが使用されるまで指定されません。関連する型は、キーワードを使用して宣言されassociatedtypeます。

最初のパートで 作成したスタックのプロトコルを定義できます。

protocol Stackable {
    associatedtype Element
    mutating func push(element: Element)
    mutating func pop() -> Element?
    func peek() throws -> Element
    func isEmpty() -> Bool
    func count() -> Int

}

プロトコルStackableは、スタックが提供する必要のある機能を定義します。

プロトコルに準拠するすべてのタイプは、Stackable格納する値のタイプを指定できる必要があります。正しいタイプの要素のみがスタックに追加されることを確認する必要があり、その添え字によって返される要素のタイプを明確にする必要があります。

プロトコルに従ってスタックを変更しましょう：

enum StackError: Error {
    case Empty(message: String)
}

protocol Stackable {
    associatedtype Element
    mutating func push(element: Element)
    mutating func pop() -> Element?
    func peek() throws -> Element
    func isEmpty() -> Bool
    func count() -> Int
    subscript(i: Int) -> Element { get }
}

public struct Stack<T>: Stackable {
    public typealias Element = T
    
    var array: [T] = []
    
    init(capacity: Int) {
        array.reserveCapacity(capacity)
    }
    
    public mutating func push(element: T) {
        array.append(element)
    }
    
    public mutating func pop() -> T? {
        return array.popLast()
    }
    
    public func peek() throws -> T {
        guard !isEmpty(), let lastElement = array.last else {
            throw StackError.Empty(message: "Array is empty")
        }
        return lastElement
    }
    
    func isEmpty() -> Bool {
       return array.isEmpty
    }
    
    func count() -> Int {
        return array.count
    }
}

extension Stack: Collection {
    public func makeIterator() -> AnyIterator<T> {
        var curr = self
        return AnyIterator { curr.pop() }
    }
    
    public subscript(i: Int) -> T {
        return array[i]
    }
    
    public var startIndex: Int {
        return array.startIndex
    }
    
    public var endIndex: Int {
        return array.endIndex
    }
    
    public func index(after i: Int) -> Int {
        return array.index(after: i)
    }
}

extension Stack: CustomStringConvertible {
    public var description: String {
        let header = "***Stack Operations start*** "
        let footer = " ***Stack Operation end***"
        let elements = array.map{ "\($0)" }.joined(separator: "\n")
        return header + elements + footer
    }
}

stack.push(element: 1)

関連するタイプに制約を追加します。


プロトコルの関連するタイプに制限を追加して、関連するタイプがこれらの制限に確実に準拠するようにすることができます。
プロトコルを変更しましょうStackable。

protocol Stackable {
    associatedtype Element: Equatable
    mutating func push(element: Element)
    mutating func pop() -> Element?
    func peek() throws -> Element
    func isEmpty() -> Bool
    func count() -> Int
    subscript(i: Int) -> Element { get }
}

これで、スタック要素のタイプが一致するはずです。一致Equatableしない場合、コンパイル時エラーが発生します。

Generic- where


関連するタイプの場合、要件を定義すると便利です。要件は、一般的なwhere句で記述されます。Generic句をwhere使用すると、関連する型が特定のプロトコルに準拠すること、または特定の型パラメーターと関連する型が同じであることを要求できます。一般的な条件whereはkeyword whereで始まり、その後に関連するタイプの制限、またはタイプと関連するタイプ間の等価関係が続きます。ジェネリック条件whereは、型または関数の本体の左中括弧の直前に記述されます。

func allItemsMatch<C1: Container, C2: Container>
    (_ someContainer: C1, _ anotherContainer: C2) -> Bool
    where C1.Item == C2.Item, C1.Item: Equatable {
}

ジェネリック条件を持つ拡張機能


where拡張の一部として 一般的な条件を使用できます。次の例では、Stackメソッドを追加することにより、前の例の一般的な構造を拡張していisTop (_ :)ます。

extension Stack where Element: Equatable {
    func isTop(_ item: Element) -> Bool {
        guard let topItem = items.last else {
            return false
        }
        return topItem == item
    }
}

拡張機能はisTop (_ :)、スタック上のアイテムがEquatableをサポートする場合にのみメソッドを追加します。whereプロトコル拡張で一般的な条件を使用することもできます。あなたは、whereカンマで区切っ、条件にいくつかの要件を追加することができます。

ジェネリック型のエイリアス：


タイプエイリアスには、共通のパラメーターを設定できます。エイリアスのままです（つまり、新しいタイプは導入されません）。

typealias StringDictionary<Value> = Dictionary<String, Value>
var d1 = StringDictionary<Int>()
var d2: Dictionary<String, Int> = d1 // okay: d1 and d2 have the same type, Dictionary<String, Int>
typealias DictionaryOfStrings<T : Hashable> = Dictionary<T, String>
typealias IntFunction<T> = (T) -> Int
typealias Vec3<T> = (T, T, T)
typealias BackwardTriple<T1,T2,T3> = (T3, T2, T1)

このメカニズムでは、型パラメーターに対する追加の制限は使用できません。
そのようなコードは機能しません：

typealias ComparableArray<T where T : Comparable> = Array<T>

一般的な上付き


下付き文字は一般的なメカニズムを使用し、一般的な条件を含めることができますwhere。の後にタイプ名を山括弧で囲み、添え字の本文の左中括弧の直前にsubscript一般的な条件を記述whereします。

extension Container {
    subscript<Indices: Sequence>(indices: Indices) -> [Item]
        where Indices.Iterator.Element == Int {
            var result = [Item]()
            for index in indices {
                result.append(self[index])
            }
            return result
    }
}
