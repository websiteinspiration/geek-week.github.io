.NET Coreで開発しますか？Ubuntuの郭、私は作成しました


写真提供：Kevin Horvat
私のキャリアの12年間、私は.NETを使用しており、Windowsおよび独自開発ツールに強く関わってきました。しかし、Microsoftのおかげで、.NET Coreはすべてを変更し、今では、ほぼすべての.NETで.NET用に開発できます。ポイントは小さいです-プロジェクトをCoreにドラッグしてください。少し前に、私はこの質問を決定し、Ubuntuへの完全な移行のためのトラクターを始めました。

結果が本当に気に入りました-すべてがうまくいき、開発が簡単で、DockerとKubernetesによって移動プロセスがはるかに簡単になりました。しかし、OS、bash、および一部のツール（同じDockerなど）のインストールオプションの混乱に関する知識不足のため、初期セットアップには1日以上かかりました。つまり、プロセスはかなり長く、場合によっては混乱を招きます。

私自身とLinuxの開発を試みる予定のある人々のために時間を節約するために、すべてをスクリプトの形式で設計しました。あなたは純粋なUbuntuでそれらを実行することができ、カモメを座って飲んでいる間、それらはすべてを設定します。また、必要に応じて、ニーズに合わせて簡単に仕上げることができます。

これがあなたに役立つと思われる場合は、Catへようこそ。

スクリプトはGithubのリポジトリで入手できます。bashの最初の紹介はそれらを読むのに十分であり、それらは豊富に参照されています。そして、洗練された人は、それらの中で最適でない瞬間を見つける可能性が最も高いです（それらが見つかった場合は、知らせてください、私はあなたにとても感謝します）。

スクリプトが元の形式で使用されるよりも特定のニーズに合わせて「調整」されることが多いと想定すると、すべての微妙な点（たとえば、sudoモードで現在のユーザーの下からコマンドを実行する方法）にもリンクが提供されます。

最後のセットは、5つのファイル（3つのスクリプトとkubernetes用の2つの構成ファイル）のみで構成されています。

1_opinionated.sh


申し訳ありませんが、最初のスクリプトは「仕上げ」またはパスの主要な候補です。

まず、彼はkubernetesをさらに起動するためにハイパーバイザーをインストールします。私はVirtualboxを選択しましたが、KVMで実行することも可能で、通常はハイパーバイザーがありません。各オプションには独自のニュアンスがあるため、最終的な選択はあなた次第です。

スクリプトは、ロシア語のサポートもインストールします（この記事を書けるようにするため）。

起動後、残りのツールをインストールする前にVirtualboxが自動的に構成できるように、再起動する必要があります。

2_setup.sh


最大かつ最も有用なスクリプト。彼は次のツールをインストールします。

  • ギット
  • .NET Core 3.1 SDK
  • Nodejs
  • Docker Community Edition、Docker更新用のリポジトリーを追加
  • VirtualboxまたはKVMがインストールされていることを確認し、minikubeをインストールします
  • Visual Studio Codeと、フロントエンドとバックエンドの両方を開発するためのいくつかの拡張機能をインストールします：Gitlens、TSLint、Prettier、Stylelint、C＃、Dockerツール、Kubernetesツール、Kubernetesサポート。

3_configure.sh


インストールされているツールのセットアップを実行します。つまり：

  • Gitのユーザー名とメールをリクエストする
  • 反対側は、VS CodeをGitのデフォルトエディターとしてインストールすることを提案しています
  • オプションでlibsecretを使用してGitパスワードを暗号化された形式で保存することを提案します
  • 常にsudoを使用せずにDockerを操作するために必要なdockerグループに現在のユーザーを追加します。
  • Minikubeが起動し、ダッシュボードをインストールして、Web UIを介してクラスターにアクセスします
  • ダッシュボードにアクセスするためにminikubeで管理ユーザーを作成します。これを行うには、使用minikube_admin_user.yamlとminikube_role_binding.yamlファイルをリポジトリから。
  • ダッシュボードにアクセスするためのトークンを取得する手順を表示します。

dockerアクセス​​設定を適用するには、ログアウトしてdockerサービスを再起動する必要があります。または単にOSを再起動します。

それは、実際には、すべてです。このスクリプトが、Ubuntuと.NET Coreの開発にすぐに慣れたいという人に役立つことを願っています。
