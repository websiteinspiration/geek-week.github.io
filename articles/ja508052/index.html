SnortまたはSuricata。パート3：オフィスネットワークを保護する

では前の記事、私たちは、Ubuntuの18.04 LTSにはSuricataの安定したバージョンを実行する方法について話しました。1つのノードでIDSを構成し、フリールールセットを接続するのはかなり簡単です。今日は、最も一般的なタイプの攻撃である仮想サーバーにインストールされたSuricataを使用して企業ネットワークを保護する方法について説明します。これには、2つのコアを持つLinux上のVDSが必要です。RAMの容量は負荷に依存します。2GBで十分です。より深刻なタスクでは4または6もかかる場合があります。さらに、実験の可能性がある仮想マシン：最小限の構成で開始し、必要に応じてリソースを増やすことができます。







写真：ロイター

仮想マシンへのIDSの削除は、主にテストのために必要になる場合があります。このようなソリューションを扱ったことがない場合は、急いで物理ハードウェアを注文し、ネットワークアーキテクチャを変更しないでください。コンピューティングのニーズを判断するには、システムを安全に実行し、追加コストなしで実行することをお勧めします。すべての企業トラフィックが単一の外部ノードを通過する必要があることを理解することが重要です。ローカルネットワーク（または複数のネットワーク）をIDS SuricataがインストールされたVDSに接続するには、強力な暗号化を提供する、設定が簡単なクロスプラットフォームVPNサーバーであるSoftEtherを使用できます。オフィスのインターネット接続には実際のIPがない可能性があるため、VPSにアップグレードすることをお勧めします。 Ubuntuリポジトリには既製のパッケージはありません。ソフトウェアをダウンロードする必要があります。プロジェクトサイト、またはLaunchpadサービスの外部リポジトリから（信頼できる場合）： 



sudo add-apt-repository ppa:paskal-07/softethervpn
sudo apt-get update


次のコマンドを使用して、利用可能なパッケージのリストを表示できます。 



apt-cache search softether






softether-vpnserver（テスト構成のサーバーはVDSで実行されています）と、それを構成するためのsoftether-vpncmd-コマンドラインユーティリティが必要です。 



sudo apt-get install softether-vpnserver softether-vpncmd


サーバーを構成するには、特別なコマンドラインユーティリティを使用します。 



sudo vpncmd






設定については詳しく説明しません。手順は非常に単純で、多数の出版物に詳しく説明されており、記事のトピックには直接適用されません。つまり、vpncmdを起動した後、項目1を選択してサーバー管理コンソールに移動する必要があります。これを行うには、ハブの名前を入力する代わりに、名前localhostを入力してEnterキーを押します。管理者パスワードは、serverpasswordsetコマンドを使用してコンソールで設定され、仮想DEFAULTハブ（hubdeleteコマンド）が削除され、Suricata_VPNという名前で新しいものが作成され、そのパスワードも設定されます（hubcreateコマンド）。次に、ハブのSuricata_VPNコマンドを使用して新しいハブの管理コンソールに移動し、groupcreateコマンドとusercreateコマンドを使用してグループとユーザーを作成します。ユーザーパスワードは、userpasswordsetを使用して設定します。



SoftEtherは、SecureNATとローカルブリッジの2つのトラフィック転送モードをサポートしています。 1つ目は、独自のNATとDHCPを使用して仮想プライベートネットワークを構築するための独自のテクノロジーです。 SecureNATはTUN / TAPを必要とせず、Netfilterまたは別のファイアウォールを設定する必要もありません。ルーティングはシステムのコアに影響を与えません。使用されるハイパーバイザーに関係なく、すべてのプロセスは仮想化され、どのVPS / VDSでも機能します。これにより、SoftEther仮想ハブを物理ネットワークアダプターまたはTAPデバイスに接続するローカルブリッジモードと比較して、プロセッサーの負荷が増加し、速度が低下します。



この場合のセットアップは、Netfilterを使用してカーネルレベルでルーティングが行われるため、複雑です。私たちのVDSはHyper-V上に構築されているため、最後のステップでローカルブリッジを作成し、bridgecreate Suricate_VPN -device：suricate_vpn -tap：yesコマンドを使用してTAPデバイスをアクティブ化します。ハブ管理コンソールを終了すると、まだIPが割り当てられていない新しいネットワークインターフェイスがシステムに表示されます。 



ifconfig






次に、インターフェース間のパケットルーティングを有効にする必要があります（IP転送）。 



sudo nano /etc/sysctl.conf


次の行のコメントを外します。 



net.ipv4.ip_forward = 1


変更をファイルに保存し、エディターを終了して、次のコマンドを使用して変更を適用します。 



sudo sysctl -p


次に、仮想ネットワークのダミーIP（たとえば、10.0.10.0 / 24）を使用してサブネットを定義し、インターフェースにアドレスを割り当てる必要があります。 



sudo ifconfig tap_suricata_vp 10.0.10.1/24


次に、Netfilterルールを詳しく説明する必要があります。



1.必要に応じて、受信パケットへの着信パケットを許可します（独自のプロトコルSoftEtherはHTTPSとポート443を使用します） 



sudo iptables -A INPUT -p tcp -m tcp --dport 443 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 992 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 1194 -j ACCEPT
sudo iptables -A INPUT -p udp -m udp --dport 1194 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 5555 -j ACCEPT


2. 10.0.10.0/24サブネットからメインIPサーバーへのNATを構成する 



sudo iptables -t nat -A POSTROUTING -s 10.0.10.0/24 -j SNAT --to-source 45.132.17.140


3.サブネット10.0.10.0/24からのパケットの通過を許可します 



sudo iptables -A FORWARD -s 10.0.10.0/24 -j ACCEPT


4.すでに確立されている接続にパケットを通過させる 



sudo iptables -A FORWARD -p all -m state --state ESTABLISHED,RELATED -j ACCEPT


初期化スクリプトを使用してシステムを再起動したときのプロセスの自動化は、宿題として読者に任されます。



IPクライアントを自動的に発行する場合は、ローカルブリッジに何らかのDHCPサービスをインストールする必要もあります。これでサーバーのセットアップが完了し、クライアントに移動できます。SoftEtherは多くのプロトコルをサポートしており、その使用はLAN機器の機能に依存します。 



netstat -ap |grep vpnserver






テストルーターはUbuntuでも動作するため、独自のプロトコルを使用するために、外部リポジトリからsoftether-vpnclientおよびsoftether-vpncmdパッケージをインストールします。クライアントを起動する必要があります： 



sudo vpnclient start


設定するには、vpncmdユーティリティを使用して、vpnclientが実行されているマシンとしてlocalhostを選択します。すべてのコマンドはコンソールで実行されます。仮想インターフェース（NicCreate）とアカウント（AccountCreate）を作成する必要があります。



場合によっては、AccountAnonymousSet、AccountPasswordSet、AccountCertSet、およびAccountSecureCertSetコマンドを使用して認証方法を指定する必要があります。 DHCPを使用しないため、仮想アダプターのアドレスは手動で設定されます。



さらに、ip forward（/etc/sysctl.confファイルのパラメーターnet.ipv4.ip_forward = 1）を有効にし、静的ルートを構成する必要があります。必要に応じて、Suricataを備えたVDSで、ローカルネットワークにインストールされているサービスを使用するようにポート転送を構成できます。これに関して、ネットワークの結合は完全であると考えることができます。

Suricataをカスタマイズする



では前の記事、私たちは、モードを動作する2つのIDSの話：NFQUEUEキュー（NFQモード）を介して、ゼロコピー（AF_PACKETモード）を介して。2つ目は2つのインターフェースを必要としますが、高速性が異なります-使用します。このパラメーターは、デフォルトで/ etc / default / suricataに設定されています。/etc/suricata/suricata.yamlのvarsセクションを編集して、仮想サブネットをホームサブネットとして設定する必要もあります。







IDSを再起動するには、次のコマンドを使用します。 



systemctl restart suricata

攻撃をシミュレートします



外部IDSサービスの戦闘での使用には、いくつかのシナリオが考えられます



。DDoS攻撃に対する保護（主な目的）



分析パッケージは、インターネットインターフェイスを見ているシステムに送信する必要があるため、企業ネットワーク内でこのようなオプションを実装することは困難です。 IDSがそれらをブロックした場合でも、偽のトラフィックがデータチャネルを配置する可能性があります。これを回避するには、ローカルネットワーク上のすべてのトラフィックとすべての外部トラフィックをスキップできる、かなり効率的なインターネット接続を備えたVPSを注文する必要があります。これは多くの場合、オフィスチャネルを拡大するよりも簡単で安価です。代わりに、DDoSに対する保護のための専門サービスについて言及する価値があります。それらのサービスのコストは仮想サーバーのコストに匹敵しますが、面倒な設定は必要ありませんが、欠点があります-お金のためにクライアントはDDoSからの保護のみを受け取りますが、独自のIDSは好きなように構成できます。



他のタイプの外部攻撃に対する保護



Suricataは、インターネット（メールサーバー、Webサーバー、Webアプリケーションなど）からアクセスできる企業ネットワークサービスのさまざまな脆弱性を悪用する試みに対処できます。通常、このために、IDSは境界デバイスの後のLAN内にインストールされますが、テイクアウトは存在する権利を持っています。



内部侵入者に対する保護



システム管理者の最善の努力にもかかわらず、企業ネットワーク上のコンピュータがマルウェアに感染する可能性があります。さらに、フーリガンがLANに現れ、特定の違法な操作を実行しようとする場合があります。 Suricataはこのような試みをブロックするのに役立ちますが、内部ネットワークを保護し、トラフィックを1つのポートにミラ​​ーリングできる管理されたスイッチと一緒に使用するために、境界の内側にインストールすることをお勧めします。この場合の外部IDSも役に立たないわけではありません。少なくとも、LANに存在するマルウェアによる外部サーバーへの接続試行をキャッチすることができます。



最初に、VPSを攻撃する別のテストを作成し、ローカルネットワークルーターでデフォルトの構成でApacheを起動します。その後、IDSサーバーから80番目のポートを転送します。次に、攻撃ノードからのDDoS攻撃をシミュレートします。これを行うには、GitHubからダウンロードし、攻撃ノードで小さなxerxesプログラムをコンパイルして実行します（gccパッケージのインストールが必要な場合があります）： 



git clone https://github.com/Soldie/xerxes-DDos-zanyarjamal-C.git
cd xerxes-DDos-zanyarjamal-C/
gcc xerxes.c -o xerxes 
./xerxes 45.132.17.140 80
