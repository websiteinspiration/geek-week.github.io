高性能コンピューティングの3つの条件


人間のゲノムの計算は、高性能コンピューティング（HPC、高性能コンピューティング）の分野における唯一のタスクではありません。科学者は物理的なフィールド、エンジニア-航空機、投資家-経済モデルの詳細を計算し、すべて一緒にビッグデータを分析し、ニューラルネットワークを構築し、他の多くの複雑な計算を行います。

HPCの3つの条件は、驚異的な処理能力、非常に大容量で高速なストレージ、高いネットワーク帯域幅です。したがって、LDC計算を実行する標準的な方法は、自社のデータセンター（オンプレミス）またはクラウド内のプロバイダーです。

ただし、すべての企業が独自のデータセンターを持っているわけではなく、所有者がリソース効率の点で商用データセンターに負けていることがよくあります（ハードウェアとソフトウェアの購入と更新、有能な人材の支払いなどに資本的支出が必要です）。 。反対に、クラウドプロバイダーは、従量課金制の運用コストモデルに従ってITリソースを提供します。家賃はご利用時のみ課金されます。計算が完了すると、アカウントからサーバーを削除できるため、IT予算を節約できます。ただし、データをプロバイダーに転送することについて法的または企業の禁止がある場合、クラウドでのHPCコンピューティングは利用できません。

プライベートマルチクラウドストレージ


Private MultiCloud Storageのアーキテクチャは、データ自体を企業のサイトに、またはコロケーションサービスを使用してデータセンターの独立した安全なコンパートメントに物理的に残すことにより、クラウドサービスへのアクセスを提供するように設計されています。本質的に、これはデータ中心の分散コンピューティングモデルであり、クラウドサーバーはプライベートクラウドのリモートストレージシステムと連携します。したがって、同じローカルデータウェアハウスを使用して、最大のプロバイダーのクラウドサービス（AWS、MS Azure、Google Cloud Platformなど）で作業できます。

KingstonはSupercomputing-2019でのPMCS実装の例を示し、DC1000M SSDに基づく高性能データストレージシステム（DCS）のサンプルを発表しました。クラウドベースのスタートアップの1つは、主要なクラウドプロバイダーとのソフトウェア定義ストレージおよび専用通信チャネル用のStorOne S1管理ソフトウェアです。 。

プライベートストレージを備えたクラウドコンピューティングの作業モデルとしてのPMCSは、AT＆TおよびEquinixインフラストラクチャでサポートされているデータセンター間のネットワーク接続が発達した北米市場向けに設計されていることに注意してください。したがって、Equinix Cloud Exchangeの任意のノードのコロケーション上のストレージシステムとAWSクラウド間のpingは1ミリ秒未満です（ソース：ITProToday）。

展示会で示されたPMCSアーキテクチャのデモでは、DC1000M NVMeストレージシステムが同じ場所に配置され、仮想マシンがAWS、MS Azure、およびGoogle Cloud Platformクラウドで相互にpingしました。クライアントサーバーアプリケーションは、データセンター内のキングストンストレージシステムおよびHP DL380サーバーとリモートで連携し、通信チャネルのインフラストラクチャを通じて、エクイニクスは上記の主要プロバイダーのクラウドプラットフォームにアクセスしました。



Supercomputing-2019でのプライベートマルチクラウドストレージのプレゼンテーションからのスライド。出典：キングストン

プライベートマルチクラウドストレージのアーキテクチャを管理するための同様のソフトウェアが、さまざまな企業から提供されています。このアーキテクチャの用語は、プライベートマルチクラウドストレージまたはプライベートストレージクラウドのように異なって聞こえる場合もあります。

KingstonのSSDのコーポレートガバナンスマネージャーであるKeat Shimmentiは、次のように述べています。「これらのHPCアプリケーションでは、プロセッサーのパフォーマンスとI / O速度のはるかに大きな一致が必要です。」キングストンのソリューションがコンピューティングのブレークスルーをどのように支援し、世界で最も極端なコンピューティング環境とアプリケーションに必要なパフォーマンスを提供するかについてお話しできることを誇りに思います。」

DC1000Mを駆動し、それに基づくストレージシステムの例


DC1000M U.2 NVMe SSDは、キングストンがデータセンター向けに設計したもので、人工知能（AI）や機械学習（ML）のアプリケーションなど、データとHPCの集中的な作業用に特別に設計されています。



3.84TB DC1000M U.2 NVMeドライブ。出典：Kingston

DC1000M U.2ドライブは、Silicon Motion SM2270コントローラー（PCIe 3.0およびNVMe 3.0）によって制御される96レイヤーのIntel 3D NANDメモリに基づいています。 Silicon Motion SM2270は、PCIe 3.0 x8インターフェイス、デュアル32ビットDRAMデータバス、および3つのARM Cortex R5デュアルプロセッサーを備えた16チャネルNVMe企業コントローラーです。

さまざまなボリュームのDC1000Mがリリース用に提供されています：0.96〜7.68 TB（最も一般的なものは3.84および7.68 TBと見なされます）。ドライブのパフォーマンスは80万IOPSと推定されます。



10x DC1000M U.2 NVMe 7.68 TBのストレージシステム。出典：Kingston

HPCアプリケーションのストレージシステムの例として、KingstonはSupercomputing-2019で、それぞれが7.68 TBの容量の10個のDC1000M U.2 NVMeドライブを備えたラックマウントソリューションを発表しました。ストレージシステムは、AICの1UフォームファクタープラットフォームであるSB122A-PHに基づいています。プロセッサー：2x Intel Xeon CPU E5-2660、Kingston DRAM 128 GB（8x16 GB）DDR4-2400（部品番号：KSM24RS4 / 16HAI）。 OSはUbuntu 18.04.3 LTS、Linuxカーネルver 5.0.0-31がインストールされています。 gfio v3.13テスト（フレキシブルI / Oテスター）は、23.8 Gb /秒のスループットで580万IOPSの読み取りパフォーマンスを示しました。

プロバイダーで高性能コンピューティングを実行するが、オンプレミスでデータを物理的に保存するタスクは、ロシアの企業にも関係します。国内ビジネスでよくあるもう1つのケースは、外国のクラウドサービスを使用する場合に、データをロシア連邦に配置する必要がある場合です。私たちは、長年のキングストンパートナーとしてのSelectelクラウドプロバイダーに代わって、これらの状況についてコメントするよう求められました。

「ロシアでは、同様のアーキテクチャを構築できます。さらに、ロシア語でのサービスと顧客の会計に関するすべてのレポートドキュメントを使用できます。企業がオンプレミスのストレージシステムを使用して高性能コンピューティングを実行する必要がある場合、Selectelは、FPGA、GPUなどのさまざまなタイプのプロセッサを備えたレンタルサーバーを提供しますまたはマルチコアCPU。さらに、パートナーを通じてお客様のオフィスとデータセンター間の専用光チャネルの敷設を手配します。 -クライアントは、特別なアクセスモードでマシンルームのコロケーションにストレージを配置し、当社のサーバーとグローバルプロバイダーAWS、MS Azure、Google Cloudのクラウドの両方でアプリケーションを実行することもできます。もちろん、後者の場合の信号遅延は、クライアントのストレージが米国にある場合よりも大きくなりますが、ブロードバンドマルチクラウド接続が提供されます。」
