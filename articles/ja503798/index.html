STM32の農場気象ステーション（Bluepill、STM32IDE、HAL）

こんにちは、ハブル！

この記事では、隔離中の小さな記事-STM32（ブルーピルボード）に基づく気象ステーションについてお話します。



このデバイスは、気象ステーションを実際に組み立てる必要性からではなく、何かを収集したいという欲求から外れることになることを前もって明らかにしておきます。そして、ウェザーステーションの組み立て後、そのわずかな機能を拡張し、大幅にやり直したいというたまらない欲望が現れました。

センサー端子の電圧を読み取ると、次のように判断できます。温度と湿度、電圧値とデータシートの表を関連付けます。湿度から始めましょう：





グラフは、湿度と電圧の関係が線形であることを示しています。つまり、ADC値を直接湿度値に変換できます。指定しましょう：

ADC_hum-ADCの値。
ハム-湿度値（％）;
V_hum-センサーの電圧値。
V_hum = 3.3 V（ADCの基準電圧）* ADC_hum / 4095（ADCのビット数）。
湿度1％は0.3 / 10 = 0.03 Vに対応し、次に
hum = V_hum / 0.03 = ADC_hum * 0.027

計算が決定したので、コードに移りましょう。 PA4（ADC1）に接続された湿度センサー。

float hum;
uint16_t ADC_hum;

HAL_ADC_Start(&hadc1);                              //  
HAL_ADC_PollForConversion(&hadc1,100);              //  
ADC_hum = HAL_ADC_GetValue(&hadc1);                 //    
HAL_ADC_Stop(&hadc1);                               //  

hum = ADC_hum;
hum *= 0.027;

ここで、サーミスタ（抵抗が温度によって変化する抵抗器）について考えます。 ADCを使用すると、要素の電圧を測定できますが、その抵抗は測定できません。サーミスタを使用するには、サーミスタの出力（白線）とアースの間に10kΩの抵抗を追加する必要があります。結果として、我々は（サーミスタ内部がVCCに接続されている）は、分圧器を得る：



今我々は最初のキルヒホッフのルールを使用して、サーミスタの抵抗を測定することができる-ノードの電流の和はもちろんGET学校の物理学を覚える0である：

Untc / RNTC = U1 / R1

RA5では、10k抵抗の両端の電圧を測定します。サーミスタの場合、その抵抗は温度変化に比例して変化しません。ドキュメントには、-40〜+125の温度でのトランジスタ抵抗の巨大な表が含まれています。テーブル全体をマイクロコントローラーに転送するのもまた別の作業ですが、10度から30度（21の値）の範囲を取った。

マイクロコントローラーが毎回抵抗値を計算する必要がないように、Mathcadでプログラムを作成しました。結果を配列に変換しました：

int out_temp[21] ={2200, 2255,	2315, 2377, 2433, 2497, 2558, 2618, 2680, 2741, 2801, 2862, 2923, 2983, 3041, 3102, 3166, 3215, 3283, 3336, 3396};
