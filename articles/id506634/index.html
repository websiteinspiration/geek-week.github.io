Module Federation di Webpack 5, sebuah plug-in untuk bertukar modul antara aplikasi Javascript, deskripsi dan contoh

Di versi kelima pembangun Webpack, satu set plugin untuk pertukaran modul antara aplikasi Javascript telah muncul.

Artikel ini adalah deskripsi singkat dan contoh berdasarkan dua aplikasi kecil yang dibangun pada kerangka kerja ReactJS.

Plugin Module Federation memungkinkan aplikasi untuk mengekspor satu atau lebih modul ke file JS yang terpisah. Cara yang bagus untuk membangun aplikasi microfrontend. Aplikasi pihak ketiga dapat mengimpor modul yang sudah jadi, misalnya, komponen reaksi. Selain itu, impor dependensi Webpack mengambil alih. Perbedaan dari NPM adalah mengimpor ke runtime.



Aplikasi sumber dalam konfigurasi webpack secara eksplisit menunjukkan:

  • modul untuk ekspor
  • ekspor nama file mis. export.js
  • dependensi yang tidak perlu diekspor mis. export.js

Dalam HTML aplikasi penerima, file JS dari aplikasi sumber diimpor sebagai skrip JS biasa: 

<script src="http://source-app.com/export.js">

Dan di konfigurasi webpack, modul mana yang diambil dari file.

Ini adalah tampilannya


Aplikasi sumber harus mendaftarkan ekspor dalam pengaturan webpack.config.js.
Dalam pengaturan plugin, tunjukkan nama wadah yang diekspor, modul mana yang akan dimasukkan di dalamnya dan bagaimana file akan dipanggil.

contoh konfigurasi 
const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
 
module.exports = {
 ...
 plugins: [
   new ModuleFederationPlugin({
     name: 'home',
     library: { type: 'var', name: 'home' },
     filename: 'export.js',
     exposes: {
       ProductCarousel: './src/ProductCarousel'
     },
     shared: ['react', 'react-dom', '@material-ui/core', '@material-ui/icons']
   }),
 ]
};

Aplikasi Penerima

  • Dalam javascript.html mengimpor file aplikasi js sumber

    contoh 
    <!DOCTYPE html>
<html lang="en">
 <head>
   <script src="http://source-app.com/export.js"></script>

  • Di webpack.config.js, dalam pengaturan plugin, tentukan nama wadah yang Anda butuhkan, plugin akan menemukannya di file JS

    contoh konfigurasi 
    const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
 
module.exports = {
...
 plugins: [
   new ModuleFederationPlugin({
     remotes: {home: 'home' } // 
   }),
 ]
};

  • Di BEJ, impor modul dari wadah yang diinginkan, seolah-olah itu adalah modul lokal biasa

    contoh kode 
    import React from 'react';
import ProductCarousel from 'home/ProductCarousel';
 
function App() {
 return (
     <ProductCarousel />
 );
}
 
export default App;
