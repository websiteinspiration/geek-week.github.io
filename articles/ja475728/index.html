次のメンバーは…？-シーケンスのn番目の項、生成関数、およびZ変換の式を探しています

コードを使用してファイルをダウンロードし、データをにすることができ、私のブログのオリジナルのポスト

言語ヴォルフラム言語の4つの非常にエキサイティングな機能があります：FindSequenceFunction、RSolve、DifferenceRootReduceとFindFormula。この記事では、それらの機能について説明し、それらに密接に関連する関数について説明します。線形再帰パラメーターFindLinearRecurrence（線形再帰方程式の係数）の検索、関数の生成、GeneratingFunctionおよびZ変換ZTransformです。

最初の関数 FindSequenceFunctionは、n番目のメンバーの式を一連の数値で検索します。これ以上何も必要ありません。

Hold @ FindSequenceFunction[{1, 1, 2, 3, 5, 8, 13}, n]

n]



2番目の関数 -RSolve-は、さまざまなタイプの再帰方程式を解きます。要素は次のようになりますa[f[n]]、 a[f[f[n]]]、 a[f[f[...f[n]...]]]ここで、fの形式は次のとおりです。n+ A（算術差分方程式）、B * n-幾何またはq差分方程式）、B * n + a（算術幾何関数差分方程式）、B * n ^ d（べき乗幾何関数）差分方程式）、（A * n + B）/（C * n + D）（線形分数関数差分方程式）。

RSolve[
	{
		a[n + 3]==2 * a[n],
		a[1]==α,
		a[2]==β,
		a[3]==γ
	},
	a, n
]



RSolve[
	{
		v[n]==(2 * Pi * v[n - 2]) / n,
		v[2]==Pi,
		v[3]==(4 * Pi) / 3
	},
	v @ n, n
]

		y[0]==0,

		y[2]==4Sqrt[Pi]

シーケンスの漸化式を見つける方法は？


シーケンス内の共通用語の検索方法は、多くの場合、漸化式を選択する必要があるという事実に基づいています。

これは次のように機能します：フォームのシーケンスのn番目のメンバーを探しましょうf[n]=∑i=1ka[i]f[n−i]。シーケンスの最初のメンバーを見てみましょう：

sequence = {1, 0, 1, 2, 5, 12, 29, 70, 169, 408, 985, 2378, 5741, 13860, 33461}



次の形式でn番目の用語の式を見つけてみましょう f[n]=∑i=11a[i]f[n−i]=a[1]f[n−1]：

seauenseEq1 = MovingMap[
	Function[
		Dot[Part[#, 1;;1], {a @ 1}]==Part[#, -1]
	],
	sequence, 1
]



Hold @ Solve @ seauenseEq1



ご覧のとおり、解決策はありません。

さて、フォームで検索してみましょうf[n]=∑i=12a[i]f[n−i]=a[1]f[n−1]+a[2]f[n−2]：

seauenseEq2 = MovingMap[
	Function[
		Dot[Part[#, 1;;2], {a @ 1, a @ 2}]==Part[#, -1]
	],
	sequence, 2
]



Hold @ Solve @ seauenseEq2



ご覧のとおり、判明しました。したがって、n番目の項は次の形式になります。f[n]=f[n−1]+2f[n−2]。

実際FindLinearRecurrence、先ほどと同じように、線形再帰を検索できる組み込み関数があります。

Hold @ FindLinearRecurrence @ sequence



関数を使用してLinearRecurrence、シーケンスを拡張できます。

LinearRecurrence[{2, 1}, sequence[[1;;2]], 50]



または、シーケンスを拡張し、差分方程式を生成し、n番目の項の一般式を見つける関数を作成して、すべてを1行に結合します。

sequenseExtension[list_, n_] := Module[
	{lr, eq},
	lr = FindLinearRecurrence @ list;
	eq = Flatten[
		{
			a[k]==Total[
					Table[
						a[k + -i] * Part[lr, i],
						{i, 1, Length @ lr}
					]
				],
			Table[a[i], list[[i]]], {i, 1, Length @ lr}]
		}
	];
	<|
		"" -> eq,
		"" -> FullSimplify[a[k] /. Part[RSolve[eq, a, k], 1]],
		"" -> LinearRecurrence[lr, Part[list, Span[1, Length[lr]]], n]
	|>
];

Hold @ sequenseExtension[{1, 1, 2, 3, 5}, 20]



Hold @ sequenseExtension[{1, 2, 2, 1, 1, 2, 2, 1}, 20]



Hold @ sequenseExtension[
{1, 0, -1, 0, 2, 0, -2, 0, 3, 0, -3, 0, 4, 0, -4},
25
]



シーケンスのn番目のメンバーの式を見つける方法は？


Z変換


Z変換は、次の形式の一連の計算から構成されます。 ∑n=0∞f(n)z−n 離散関数から f(n)。この変換により、シーケンスを関数のイメージの方程式に設定するための漸化式を削減できます。f(n)これは、微分方程式を代数方程式に変換するラプラス変換に似ています。

仕組みは次のとおりです。

Grid[
	Transpose[
		Function[
			{
				#,
				Map[TraditionalForm, Map[FullSimplify, ZTransform[#, n, z]]]
			}
		][
			{
				f[n - 2],
				f[n - 1],
				f @ n,
				f[n + 1],
				f[n + 2]
			}
		]
	],
	Background -> White, Dividers -> All
]



例を見てみましょう。たとえば、よく知られているフィボナッチ数列を考えてみましょう。

fibonacciEq = f[n]==f[n - 1] + f[n - 2];

initialConditions = {f[1] -> 1, f[2] -> 1};

以下のような形式で書き直す必要があることは明らかです。 f(−1) Z変換を適用した後。

fibonacciEq = f[n + 2]==f[n + 1] + f[n];

initialConditions = {f[0] -> 1, f[1] -> 1};

Z変換を実行します。

fibonacciEqZTransformed = ReplaceAll[fibonacciEq, pattern:f[__] :> ZTransform[pattern, n, z]]



関数f-ZTransform [f [n]、n、z]の画像の方程式を解きます。

fZTransformed = ReplaceAll[
	ZTransform[f @ n, n, z],
	Part[Solve[fibonacciEqZTransformed, ZTransform[f @ n, n, z]], 1]
]



逆Z変換を実行し、同時に初期条件を代入します（最後の式でnをn-1に置き換えて、シーケンスに正しいインデックスが付けられるようにします（最初から、ゼロ項ではありません）。

ReplaceAll[InverseZTransform[fZTransformed /. initialConditions, z, n],
	n -> (n - 1)
]



もちろん、独自のRSolve対応を作成することにより、これを自動化できます。

myRSolve[eq_, initials_, f_, n_] := Module[
	{z, initialsInner, eqZTransformed, fZTransformed},
	initialsInner = ReplaceAll[initials, f[x_] :> f[x - 1]];
	eqZTransformed = ReplaceAll[eq, pattern:f[__] :> ZTransform[pattern, n, z]];
	fZTransformed = ReplaceAll[ZTransform[f @ n, n, z],
		Part[Solve[eqZTransformed, ZTransform[f @ n, n, z]], 1]
	];
	FullSimplify[
		InverseZTransform[fZTransformed /. initialsInner, z, n] /. n -> (n - 1)
	]
];

myRSolve[
	{
		f[n + 2]==(2 * f[n + 1]) + -(5 * f[n])
	},
	{f[1] -> 20, f[2] -> 0},
	f, n
]



RSolve[
	{
		f[n + 2]==(2 * f[n + 1]) + -(5 * f[n]),
		f[1]==20,
		f[2]==0
	},
	f, n
]



しかし、もちろん、RSolveには、さまざまな離散方程式を解くための多くの可能性が含まれています。

RSolve[a[n]==(n * a[n]) + n, a, n],
RSolve[
	{
		a[n + 1]==(2 * a[n]) + (3 * a[n]) + 4,
		a[0]==0
	},
	a, n
],
RSolve[
	y[n + 1 * 3]==(2 * y[n + 1 * 6]) + n * 2,
	y, n
]







関数の生成


シーケンス関数の生成 a(n) これはそのような機能です G(x)、Taylorシリーズ（またはより広義にはLauren）の展開は次の形式を持っています- G(x)=∑i=0∞a(n)xn。言い換えると、系列内の関数の展開におけるxのべき乗の係数がシーケンスを決定します。

発言機能G(x)=11−x シーケンス1、1、1、1、...の生成関数です。

Series[1 / (1 + -x), {x, 0, 10}]



機能 G(x)=11−x−x2 フィボナッチ数列1、1、2、3、5、8、13、...の生成関数です。

Series[(1 * 1) + (-x) + -(x * 2),
	{x, 0, 10}
]



一種の生成関数もあります-指数関数生成関数、 a(n) の形をしています- G(x)=∑i=0∞a(n)n!xn。

たとえば、シーケンス1、1、1、1 ...と1、1、2、3、5、8、13、...の場合、指数生成関数は次のようになります。ex そして 15e−2x1+5(e5x−1)：

ReplaceAll[Normal[Series[E ^ x, {x, 0, 10}]],
	Power[x, n_] :> ((x ^ n) * Factorial[n])
]



ReplaceAll[
	Normal[
		FullSimplify[
			Series[
				Plus[E,
					(-(2 * x * 1)) + 5 * ((E * 5 * x) - 1) * 5
				],
				{x, 0, 10}
			]
		]
	],
	Power[x, n_] :> ((x ^ n) * Factorial[n])
]



Wolfram言語の生成関数は2つの関数にあります- GeneratingFunctionとFindGeneratingFunction（指数関数ExponentialGeneratingFunction）：

GeneratingFunction[-(m * Factorial[n]), {n, m}, {x, y}]



TraditionalForm[
	FullSimplify[
		ExponentialGeneratingFunction[-(n * Factorial[n - 1] * Factorial[2 * n]), n, x]
	]
]



生成関数を使用してシーケンスの共通メンバーを見つける方法はたくさんあります。これについて詳しく説明することはしません。たとえば、genfunc.ruのウェブサイトには良い理論が載っています。

メソッドの1つはZ変換に似ています。

generatingFEq = ReplaceAll[
	f[n + 2]==f[n + 1] + f[n],
	pattern:f[__] :> GeneratingFunction[pattern, n, z]
],
generatingF = ReplaceAll[
	GeneratingFunction[f @ n, n, z],
	Part[Solve[generatingFEq, GeneratingFunction[f @ n, n, z]], 1]
],
nthTerm = SeriesCoefficient[generatingF, {z, 0, n}],
FullSimplify[
	ReplaceAll[ReplaceAll[nthTerm, {f[0] -> 1, f[1] -> 1}],
		n -> (n - 1)
	],
	GreaterEqual[n, 1]
]

潜在的な式を検索する


最後FindFormulaに、与えられた一連の数値に基づいて、それらを表すことができる式を作成する関数に注目します。依存関係を受け入れることができます。さまざまなクラスの関数から多くを選択できます。

data = Table[
  {
    x,
    Sin[2 * x] + Cos[x] + RandomVariate[NormalDistribution[0, 0.2]]
  },
  {x, RandomReal[{-10, 10}, 1000]}
];

ListPlot[data, Background -> White, ImageSize -> 600]



formulas = FindFormula[data, x]



ご覧のように、Wolfram言語は、 "ノイズの多い"データが作成された関数に非常に近い関数、つまり-Sin [2x] + Cos [x]を選択しました。

Plot[formulas,
	{x, -10, 10},
	PlotStyle -> AbsoluteThickness[3],
	Prolog -> {AbsolutePointSize[5], Gray, Point @ data},
	Background -> White, ImageSize -> 800, PlotLegends -> "Expressions"
]



より多くの依存関係、たとえば10を構築できます。

formulas = FindFormula[data, x, 10]



Plot[formulas,
	{x, -10, 10},
	PlotStyle -> AbsoluteThickness[3],
	Prolog -> {AbsolutePointSize[5], LightGray, Point @ data},
	Background -> White, ImageSize -> 800, PlotLegends -> "Expressions"
]
