-ASCII


非ASCII文字はできるだけ使用しないでください。エンコードはUTF-8にする必要があります。
ユーザーに表示するために文字列をハードコーディングする必要はありません（英語も含む）ので、非ASCII文字はほとんどありません。ただし、場合によっては、そのような単語をコードに含めることが許可されます。たとえば、コードがデータファイルを解析する場合（英語以外のエンコーディング）、国別の区切り文字をコードに含めることができます。より一般的なケースでは、単体テストコードに国別文字列が含まれる場合があります。これらの場合、UTF-8エンコーディングを使用する必要があります。ほとんどのユーティリティ（ASCIIだけでなく）でも認識されます。

特に読みやすさを向上させる場合は、16進数も有効です。たとえば、"\ xEF \ xBB \ xBF"またはu8 "\ uFEFF"-Unicodeでは長さがゼロの不可分なスペースであり、正しいUTF-8テキストでは表示されないはずです。\ uXXXXなどのリテラルがUTF-8でエンコードされるように

関数の宣言と定義


戻り値の型、関数の名前、およびそのパラメーターを1行に配置してください（すべてが当てはまる場合）。関数呼び出しの引数と同じように、パラメーターのリストを長すぎる行に分割します。

浮動小数点数


浮動小数点数は常に小数点とその両側にある数値でなければなりません（指数表記の場合でも）。このアプローチにより可読性が向上します。すべての浮動小数点数は同じ形式であり、整数と混同しないでください。また、指数表記の文字E eは 16進数には使用できません。指数表記の数値は整数ではないことに注意してください。

float f = 1.f;
long double ld = -.5L;
double d = 1248e6;


float f = 1.0f;
float f2 = 1;   //

関数呼び出し


関数呼び出し全体を1行で記述するか、引数を新しい行に配置します。また、インデントは最初の引数または4つのスペースのいずれかにすることができます。行数を最小限に抑え、各行にいくつかの引数を配置します。

関数呼び出し形式：

bool result = DoSomething(argument1, argument2, argument3);

引数が1行に収まらない場合は、それらを複数の行に分割し、後続の各行を最初の引数に揃えます。括弧と引数の間にスペースを入れないでください：

bool result = DoSomething(averyveryveryverylongargument1,
                          argument2, argument3);


4つのスペースのインデントを使用して、複数の行に引数を配置できます。
if (...) {
  ...
  ...
  if (...) {
    bool result = DoSomething(
        argument1, argument2,  //  4 
        argument3, argument4);
    ...
  }

1行に複数の引数を配置して、関数呼び出しごとの行数を減らしてください（読みやすさが損なわれない場合）。一部の人々は、行ごとに1つの引数に厳密にフォーマットすると読みやすくなり、引数の編集が容易になると考えています。ただし、主にコードリーダー（編集ではない）に焦点を当てているため、読みやすさを向上させるための多くのアプローチを提供しています。

同じ行のいくつかの引数が（式の複雑さまたは複雑さのために）読みやすさを低下させる場合は、引数の「会話」変数を作成してみてください。

int my_heuristic = scores[x] * y + bases[x];
bool result = DoSomething(my_heuristic, x, y, z);

または、複雑な引数を別の行に配置して、説明コメントを追加します。

bool result = DoSomething(scores[x] * y + bases[x],  //  
                          x, y, z);

関数呼び出しにまだ別の行に配置することが望ましいという引数がある場合は、それを配置します。ソリューションは、コードの読みやすさの向上に基づいている必要があります。

引数は構造を形成することがあります。この場合、必要な構造に従って引数をフォーマットします。

//     3x3
my_widget.Transform(x1, x2, x3,
                    y1, y2, y3,
                    z1, z2, z3);

条件


ブラケットの内側にスペースを挿入しないようにしてください。場所場合とそうでない別の行に。

条件をフォーマットするには2つの方法があります。1つは角括弧と条件の間にスペースを入れ、もう1つは入れません。

スペースなしの推奨オプション。別のオプションも有効ですが、一貫性があります。既存のコードを変更する場合は、すでにコードに含まれている形式を使用してください。新しいコードを作成する場合は、同じディレクトリにあるファイルのような形式を使用するか、プロジェクト形式を使用します。不明な場合は、スペースを追加しないでください。

if (condition) {  //    
  ...  //  2 
} else if (...) {  // 'else'      
  ...
} else {
  ...
}

スペース形式を使用する場合：

if ( condition ) {  //   
  ...  //  2 
} else {  // 'else'      
  ...
}

いずれの場合も、ifと開始ブラケットの間にスペースが必要であることに注意してください。閉じ括弧と中括弧（ある場合）の間にもスペースが必要です。

if(condition) {   //  -    'if'
if (condition){   //  -    {
if(condition){    //  


if (condition) {  //   -     'if'   {

読みやすさが向上する場合は、短い条件を1行で記述できます。このオプションは、行が短く、条件にelseセクションが含まれていない場合にのみ使用してください。

if (x == kFoo) return new Foo();
if (x == kBar) return new Bar();

else セクションがある場合は、短縮バージョンを使用しないでください。

//  -    ,   'else'
if (x) DoThis();
else DoThat();

通常、ブレースは短い状態では必要ありませんが、許容されます。複雑な条件やコードも中括弧で読むとよくなります。かっこを付けて使用する必要がある場合がよくあります。

if (condition)
  DoSomething();  //  2 

if (condition) {
  DoSomething();  //  2 
}

また、条件の一部で中括弧を使用している場合は、中括弧も一緒に発行します。

//  -    'if',  'else' - 
if (condition) {
  foo;
} else
  bar;

//  -    'else',  'if' - 
if (condition)
  foo;
else {
  bar;
}



//  -     'if'   'else'
if (condition) {
  foo;
} else {
  bar;
}

ループとスイッチ


switch構文では、ブロックに括弧を使用できます。オプション間の重要な移行について説明します。ブラケットは、単一の式ループではオプションです。空のループは、括弧で囲まれた空の本体を使用するか、続行する必要があります。スイッチ

のケースブロックには、中括弧を付けても付けなくてもかまいません。括弧を使用する場合は、以下に説明する形式を使用してください。 switchのデフォルトセクションに切り替えることをお勧めします。列挙型を使用する場合、これは必要ありません。すべての値が処理されない場合、コンパイラーは警告を発行することがあります。デフォルトのセクションを実行しない場合は、エラーとして構成してください。例えば：



switch (var) {
  case 0: {  //  2 
    ...      //  4 
    break;
  }
  case 1: {
    ...
    break;
  }
  default: {
    assert(false);
  }
}

あるラベルから次のラベルへの遷移は、マクロABSL_FALLTHROUGH_INTENDEDでマークする必要があります。（absl / base / macros.hで定義されています）。
場所ABSL_FALLTHROUGH_INTENDED; 遷移が発生するポイントで。このルールの例外は、コードのない連続したラベルです。この場合、何もマークする必要はありません。

switch (x) {
  case 41:  //  
  case 43:
    if (dont_be_picky) {
      //    ( )    
      ABSL_FALLTHROUGH_INTENDED;
    } else {
      CloseButNoCigar();
      break;
    }
  case 42:
    DoSomethingSpecial();
    ABSL_FALLTHROUGH_INTENDED;
  default:
    DoSomethingGeneric();
    break;
}

ブラケットは、単一操作ループではオプションです。

for (int i = 0; i < kSomeNumber; ++i)
  printf("I love you\n");

for (int i = 0; i < kSomeNumber; ++i) {
  printf("I take it back\n");
}

空のループは、括弧のペアとして、または括弧なしで継続するようにスタイルを設定する必要があります。単一のセミコロンは使用しないでください。

while (condition) {
  //    false
}
for (int i = 0; i < kSomeNumber; ++i) {}  // .      -   
while (condition) continue;  //  - continue     


while (condition);  //  -     do/while

ポインターとリンク


「。」の周り および「->」はスペースを入れません。逆参照演算子またはキャプチャー演算子にはスペースを入れないでください。

以下は、ポインターとリンクを含む式の適切なフォーマットの例です。

x = *p;
p = &x;
x = r.y;
x = r->y;

注意：

  • 「。」および「->」はスペースなしで使用されます。
  • *または＆演算子はスペースで区切られません。

変数または引数を宣言する場合、タイプと名前の両方に「*」を配置できます。

// ,   *, &
char *c;
const std::string &str;

// ,   *, &
char* c;
const std::string& str;

コードファイルでは単一のスタイルを使用するようにしてください。既存のファイルを変更する場合は、使用されている書式を使用してください。

1つの式で複数の変数を宣言することができます。ただし、ポインタまたはリンクを含む複数の宣言を使用しないでください。これは誤解される可能性があります。

//  - 
int x, y;


int x, *y;  //  -      &  *
char * c;  //  -     *
const std::string & str;  //  -     &

たとえば、ここでAND演算子をラップすると、行の最後に配置されます。

if (this_one_thing > this_other_thing &&
    a_third_thing == a_fourth_thing &&
    yet_another && last_one) {
  ...
}

&&とAND演算子が行を完成させる ように、コードは（例に従って）分割されていることに注意してください。このスタイルは、Googleコードでより頻繁に使用されますが、行頭の演算子の位置も許容されます。また、ブラケットを追加して読みやすくすることもできます。演算子を句読点（&&や〜など）の形式で使用することは、単語andおよびcomplの形式で演算子を使用するよりも好ましいことに注意してください。

クラスのフォーマット


セクションを次の順序で配置します：public、protected、private。インデントは1つのスペースです。

クラスの基本的な形式を以下に示します（コメントを除き、クラスの説明に関するコメントを参照してください）。

class MyClass : public OtherClass {
 public:      //  1 
  MyClass();  //  2-  
  explicit MyClass(int var);
  ~MyClass() {}

  void SomeFunction();
  void SomeFunctionThatDoesNothing() {
  }

  void set_some_var(int var) { some_var_ = var; }
  int some_var() const { return some_var_; }

 private:
  bool SomeInternalFunction();

  int some_var_;
  int some_other_var_;
};

備考：

  • 基本クラスの名前は、継承されたクラスの名前と同じ行に記述されます（もちろん、80文字の制限を考慮しています）。
  • キーワードpublic：、protected 、:、およびprivate：は、1つのスペースでインデントする必要があります。
  • これらの各キーワードの前には空白行を置く必要があります（最初の言及を除く）。また、少人数のクラスでは、空白行を省略できます。
  • これらのキーワードの後に​​空白行を追加しないでください。
  • パブリックセクションが最初になり、その後ろが保護され、最後にプライベートセクションになります。
  • これらの各セクションの宣言の作成については、宣言手順を参照してください。

名前空間のフォーマット


名前空間のコンテンツはインデントされています。

  ...
}

namespace {

  // .   ,   
  void foo() {
    ...
  }

}  // namespace

ネストされた名前空間を宣言するときは、各宣言を別々の行に配置します。

縦の内訳


垂直分割を最小限に抑えます。

これは原則よりも原則です。特別な必要がない限り、空白行を追加しないでください。特に、関数の間に1行以下の空行を入れてください。空行で関数を開始しないでください。空行で関数を終了せず、空行を少なくしてください。コードブロックの空の行は、小説の段落のように機能する必要があります。

基本原則：1つの画面に収まるコードが多いほど、実行のシーケンスを理解して追跡するのが簡単になります。空の文字列は、このシーケンスを視覚的に分離するためにのみ使用してください。

空白行に関するいくつかの役立つメモ：

  • 関数の最初または最後に空の行があっても、読みやすさは向上しません。
  • if-elseブロックチェーンの空の行は読みやすさを向上させることができます。
  • コメント行の前の空行は通常、コードを読みやすくするために役立ちます。通常、新しいコメントには、古い考えの完成と新しいアイデアの始まりが含まれます。そして、空の線は明らかにそれをほのめかしています。
