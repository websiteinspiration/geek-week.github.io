TypeScriptのクリーンなコードの原則に従って資料を見つけ、それを読んだので、私はその翻訳を取り上げることにしました。ここでは、この翻訳の抜粋をいくつかお伝えしたいと思います。純粋なTypeScriptコードのいくつかの瞬間はJavaScriptと同じ原則を繰り返すため、ここでは説明しません。興味深いのであれば、JSの翻訳はすでにハブ（@ BoryaMogila）で公開されています。元のソースでそれらに慣れることができます。



はじめに、これらのクリーンなコードの原則を理解しましょう。しかし、残念ながら、クリーンなコードの明確な定義を与えることはほとんどありません。たとえば、美術館に来たときに写真を見て醜さを考えると、人によって異なりますが、別の人がやって来て素晴らしいと言います。はい、私たちは世界のいくつかの明確で一般的な特徴を持っています。そこでは、どちらかが美しいと言えますが、確かにこれをすべて定義することはできません。したがって、ここでは、この美しさのいくつかの小さな基準にすぎません。これらは花崗岩に刻まれたルールではありません。これらは単なる推奨事項です。

function fibonacci(n: number): number[] {
  if (n === 1) return [0];
  if (n === 2) return [0, 1];

  const items: number[] = [0, 1];
  while (items.length < n) {
    items.push(items[items.length - 2] + items[items.length - 1]);
  }

  return items;
}

function print(n: number) {
  fibonacci(n).forEach(fib => console.log(fib));
}

// Print first 10 Fibonacci numbers.
print(10);

// Generates an infinite stream of Fibonacci numbers.
// The generator doesn't keep the array of all numbers.
function* fibonacci(): IterableIterator<number> {
  let [a, b] = [0, 1];

  while (true) {
    yield a;
    [a, b] = [b, a + b];
  }
}

function print(n: number) {
  let i = 0;
  for (const fib of fibonacci()) {
    if (i++ === n) break;  
    console.log(fib);
  }  
}

// Print first 10 Fibonacci numbers.
print(10);

プライベート/セキュアフィールドでオブジェクトを作成する


TypeScriptはpublic （デフォルトで）、protectedおよびprivateクラスプロパティにアクセスする手段をサポートします。


不完全に：


class Circle {
  radius: number;

  constructor(radius: number) {
    this.radius = radius;
  }

  perimeter() {
    return 2 * Math.PI * this.radius;
  }

  surface() {
    return Math.PI * this.radius * this.radius;
  }
}

良い：


class Circle {
  constructor(private readonly radius: number) {
  }

  perimeter() {
    return 2 * Math.PI * this.radius;
  }

  surface() {
    return Math.PI * this.radius * this.radius;
  }
}
