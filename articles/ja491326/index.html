データ分類方法の要約

データサイエンスを研究しているとき、私はデータ分析で使用される基本的なテクニックの要約を自分で編集することにしました。これは、メソッドの名前を反映し、本質を簡単に説明し、迅速な適用のためのPythonコードを提供します。私は自分用に大要を準備していましたが、面接前、コンテスト、または新しいプロジェクトを始めるときなど、誰かにも役立つかもしれないと思いました。一般的にこれらすべての方法に精通しているが、メモリ内で更新する必要がある対象者向けに設計されています。カットの下の記事。

#imports
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.naive_bayes import GaussianNB

#model fit
X, y = load_iris(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.5, random_state=0)
gnb = GaussianNB()
y_pred = gnb.fit(X_train, y_train).predict(X_test)

#result
print("Number of mislabeled points out of a total %d points : %d"
            % (X_test.shape[0], (y_test != y_pred).sum()))


- 最近傍の方法。他の観測との類似度に従って各観測を分類します。アルゴリズムはノンパラメトリック（分布関数などのデータに制限はありません）であり、遅延トレーニングを使用します（事前トレーニング済みモデルは使用されず、分類中に使用可能なすべてのデータが使用されます）。




長所 -結果の解釈が容易で、説明変数の数が少ないタスクに適しています。弱点 -他の方法と比較して精度が低い。多数の説明変数と大量のサンプルを使用するため、かなりの計算能力が必要です。

- 決定木。データをツリー構造の形で特定の条件に従ってサブサンプルに分割します。数学的には、クラスへの分割は、クラスを可能な限り正確に決定するすべての条件が見つかるまで、つまり、各クラスに別のクラスの代表が存在しないときに行われます。実際には、限られた数の特性とレイヤーが使用され、常に2つのブランチがあります。




強み -複雑なプロセスをシミュレートし、簡単に解釈することが可能です。マルチクラス分類が可能です。弱点 -多くのレイヤーを作成すると、モデルを簡単に再トレーニングできます。エミッションは精度に影響を与える可能性があります。これらの問題の解決策は、低いレベルをトリミングすることです。

#imports
import numpy as np
import matplotlib.pyplot as plt
from sklearn import ensemble
from sklearn import datasets
from sklearn.utils import shuffle
from sklearn.metrics import mean_squared_error

#model fit
boston = datasets.load_boston()
X, y = shuffle(boston.data, boston.target, random_state=13)
X = X.astype(np.float32)
offset = int(X.shape[0] * 0.9)
X_train, y_train = X[:offset], y[:offset]
X_test, y_test = X[offset:], y[offset:]
params = {'n_estimators': 500, 'max_depth': 4, 'min_samples_split': 2,
          'learning_rate': 0.01, 'loss': 'ls'}
clf = ensemble.GradientBoostingRegressor(**params)
clf.fit(X_train, y_train)

#result
mse = mean_squared_error(y_test, clf.predict(X_test))
print("MSE: %.4f" % mse)
