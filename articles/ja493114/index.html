データ収集と処理プロセスの複雑さを考えると、ETLプロセスの開発と変更が高速であることを考えると、すでに最終段階にあるデータの品質を監視する必要があります。通常、ClickhouseまたはPostgreSQLデータベースがあります。このようなメトリックは、プロセスがどれだけ速く実行されるかを示します。

SELECT server, 
       avg(updated - close_time) 
FROM   trades 
WHERE  close_time > subtractHours(Now(), 2) 
GROUP  BY server

これらはデータの重複を見つけるのに役立ちます（Clickhouseには一意の制約はありません）。

SELECT SUM(count) FROM (
   SELECT
      COUNT(*) AS count
   FROM trades
   GROUP BY order_id
   HAVING count > 1
)

データの品質を監視するのに役立つ大量のクエリ（多くは既に使用しています）を考え出すことができます。ソーステーブルと宛先テーブルの行数の比較、テーブルへの最後の挿入時間、2つのクエリの内容の比較などです。
