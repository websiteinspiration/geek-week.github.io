1. JavaScript等価検証機能について説明する


困難： *


JavaScriptには、値の等価性をチェックするための2つの演算子があります。1つ目は、いわゆる厳密な等価演算子です。2つ目は、厳密でない等価演算子です。これは、チェック済み数量のタイプを変換するために使用できます。

  • 厳密な等価演算子（===）は、型変換を実行せずに値が等しいかどうかをチェックします。
  • 非厳密な等価演算子（==）は、値が等しいかどうかをチェックし、それらを共通の型に変換します。

var a = "42";
var b = 42;

a == b;         // true
a === b;        // false

JavaScriptでさまざまな等式チェッカーを使用するためのガイドラインを次に示します。

  • 比較された値のいずれかが値になる可能性がある場合、trueまたはfalse演算子を回避しようとし==ます。演算子を使用します===。
  • 演算子を使用してください===：あなたは、以下の値を使用している場合した場合に0、«»または[]（空の配列）。
  • それ以外の場合はすべて、演算子を安全に使用できます==。さらに、これは安全であるだけでなく、コードを単純化して可読性を向上させるのにも役立ちます。

9.「ジェネレータ」とは何ですか？


困難： ***


ジェネレーターは、そこから「終了」したり、必要に応じて「開始」したりできる関数です。それらのコンテキスト（変数バインディング）は、それらへの「エントリ」のセッション間で維持されます。ジェネレータはキーワードを使用して宣言されfunction*ます。このような関数は、最初に呼び出されたときにコードを実行せず、特別なオブジェクトであるジェネレーターを返します。これにより、その実行を制御できます。ジェネレーターによって発行された次の値を取得するには、そのメソッドを呼び出す必要がありますnext()。このためyield、値を返すキーワードが見つかるまで関数コードが実行されます。

ジェネレータ関数は何度でも呼び出すことができます。新しいジェネレータが戻るたびに。ただし、各発電機は1回だけバイパスできます。

function* makeRangeIterator(start = 0, end = Infinity, step = 1) {
    let iterationCount = 0;
    for (let i = start; i < end; i += step) {
        iterationCount++;
        yield i;
    }
    return iterationCount;
}

function deepFreeze(object) {
    let propNames = Object.getOwnPropertyNames(object);
    for (let name of propNames) {
        let value = object[name];

    }
    return Object.freeze(object);
}
