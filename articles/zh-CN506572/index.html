接下来创建一个产品卡组件cart_single，但是在创建它之前，让我们看看htmlix是如何工作的：

因此，当从地址“ /”或“ / category /：idCategory”加载页面时，服务器会向我们发送两个组件的html代码，更准确地说是三个，另一个“菜单”，但是我们尚未使用它，我们对购物车组件使用html代码，类别，在此代码的基础上htmlix创建了组件的模板，但是cart_single组件的模板未在这些地址处发送给我们，因为他不在那里。我们如何“加载”？为此，我们在应用程序描述中创建cart_single组件，并将其放置在特殊的“ fetchComponents”对象中，并在“ stateSettings.templatePath”设置中，指定用于加载该组件模板的地址，以及所有其他将出现的模板。因此，应用程序将首先创建服务器与html一起发送的组件，然后将第二个请求发送到stateSettings.templatePath之后，它将创建位于fetchComponents对象中的其余组件。

因此，如果我们始终可以从地址“ /”或“ / category /：idCategory”访问应用程序，那么我们会这样做，但是如果我们首先从地址“ / cart /：idCart”加载应用程序怎么办？该应用程序将给出一个错误，即它无法为carts数组找到模板，因为现在它不在该地址，因此我们现在需要将carts放入“ fetchComponents”对象中，相反，从那里获取single_cart，因为它应该首先与Categories组件一起初始化，我该怎么办？

为了解决这个问题，我们将使用HTMLixRouter（），在创建应用程序实例之前，在加载页面之后，它会检查现在的url，并将其与我们将要指示的url进行比较，然后将第一次加载时没有模板的URL放入其中，放入“ fetchComponents”对象中，然后初始化应用程序，因此我们将避免创建多个用于描述应用程序的选项，以用于不同的url和相应的代码重复。

因此，现在我们有了三个地址“ /”，“ / category /：idCategory”和“ / cart /：idCart”，这三个主要组成部分是carts，category和cart_single。在头两个地址处，应首先初始化部件carts和category，在地址“ / cart /：idCart”中，应初始化category和cart_single。
