前回の記事では、単一のリポジトリー用にSergeを構成する方法について説明しました。私たちの会社では、翻訳が必要なリポジトリが数十あるので、ローカライズ用に別のサーバーを割り当てました。そのファイル構造と環境は、前回の記事で説明したものとまったく同じです。各リポジトリは、独自のSergeのインスタンスを使用します。コマンドを手動で実行しないようにするため、各インスタンスにはクラウンがあり、リポジトリから新しい行を受信し、新しい翻訳を受信し、解析し、新しい行をSmartcatに送信し、新しい翻訳をGitlabに送信します。

1つのリポジトリに2組の言語


最も単純なケースから始めましょう。リポジトリに複数のリソースファイルセットがあるとします。たとえば、クライアント文字列とアプリケーションAPIは同じリポジトリに保存されますが、別のディレクトリに保存されます。クライアントは20の言語に翻訳され、APIは6に翻訳されます。

タスク：各ディレクトリへの翻訳の独立した配信を整理します。
ソリューション：

  1. Smartcatで2つのプロジェクトを設定します（6言語と20言語）。
  2. ローカリゼーションサーバーに2つのプロジェクトを設定します。
  3. 最初のプロジェクトで、project1.cfgファイルに$ unmerged_branch_mask = '^（translateAPI-）'という行を追加します。＃このマスクに一致するマージされていないブランチを処理します。ここで、「translateAPI-」はブランチ名のプレフィックスです。プレフィックスは、このブランチがAPIディレクトリでの翻訳が必要であることをSergeに示します。
  4. project1.serge.tmpl source_dir API.
  5. , project2.cfg our $unmerged_branch_mask = '^(translateCLIENT-)'; # process unmerged branches matching this mask, “translateCLIENT” . Serge, Client.
  6. project2.serge.tmpl source_dir CLIENT.

接頭辞は、1つのリポジトリに設定されたすべてのプロジェクト間で一意である必要があることに注意してください。

合計すると、Smartcatに2つのプロジェクトがあり、ローカリゼーションサーバーに2つの対応するプロジェクトがあります。どちらのプロジェクトも、Gitlabの同じリポジトリを調べますが、ディレクトリは異なります。Sergeはブランチ接頭辞を使用して、翻訳のために送信する必要がある行を理解します。差分を計算するために、同じbase-translateブランチが使用されます。

ローカリゼーションSwagger


当社では、ドキュメントを含むすべての製品がローカライズされています。今、私たちはswaggerからのドキュメントの自動生成を導入しており、それをローカライズする必要に直面しています。
タスク：最小限の労力でswaggerをローカライズします。

解決策：myproject.tmpl.sergeファイルで、データオブジェクトをパーサーオブジェクトに追加し、値を抽出して変換のために送信する必要があるフィールドをその中にリストします。

parser {
            plugin                  parse_json
            data
            {
             path_matches          \/(summary|description)$
            }
        }

同様のタスク：ファイルからテキストを翻訳する必要がありますが、すべてではなく、正当なものだけを翻訳する必要があります。その他のテキストは、マーケティングチームによって提供されます。構造を複雑にせず、法的テキスト用の追加ファイルを作成しないようにするために、すべての法的行のキーに接頭辞「legal」を付けました。

parser {
            plugin                  parse_json
             data
            {
             path_matches          ^\/legal\..*
            }
        }

法的翻訳の機微


別の興味深いケース。法的文書があり、その条件は国によって異なります。しかし、それでも、これは1つのアプリケーションであり、リソースファイルは同じディレクトリにあります。

目的：複数のドキュメントを翻訳する1つのプロジェクトのフレームワークで、各ドキュメントを1つの特定の言語に翻訳する必要があります。

行われたこと：

  1. 国ごとに適切なディレクトリが作成され、その中に、その国に関連する英語のソースファイルが置かれました。
  2. source_dir変数のパスは、リソースファイルのある共有ディレクトリに指定されます。
  3. すべてのサブディレクトリでリソースファイルの検索を有効にします。source_process_subdirsYES
  4. 呼び出されたプラグインのリストに新しいプラグインを追加します。これにより、特定の各リソースファイルを目的の言語に送信できます。ガイドとして、それが存在するディレクトリの名前を使用します。

callback_plugins {
            :feature_branch {
                plugin              feature_branch
 
                data {
                    master_job      job.base-translate
                }
            }        
            :limit_languages
            {
                plugin              limit_languages
                data
                {
                    # all rules are processed top to bottom; each rule can add or remove languages
                    # so the most priority rules are placed at the bottom
                    if
                    {
                        # by default, don't localize
                        file_matches                 .
                        then
                        {
                            exclude_all_languages    YES
                        }
                    }
 
                    if
                    {
                        file_matches                 de-au\/
 
                        then
                        {
                            include_languages        de-AT
                        }
                    }
 
                    if
                    {
                        file_matches                 li-LI\/
 
                        then
                        {
                            include_languages        li
                        }
                    }
 
                    if
                    {
                        file_matches                 pt\/
 
                        then
                        {
                            include_languages        pt-BR
                        }
                    }
 
                    if
                    {
                        file_matches                 zh-Hans\/
 
                        then
                        {
                            include_languages        zh-Hans
                        }
                    }
 
                    # and so on..
                }
            }
