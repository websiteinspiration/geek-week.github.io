ネイティブvs クロスプラットフォーム：ビデオ監視プロトコルのビジネス効果



IPカメラに基づくセキュリティシステムは、導入以来多くの新しい利点を市場にもたらしましたが、開発は常にスムーズに進んでいるわけではありません。数十年の間、ビデオ監視設計者は機器の互換性の問題に直面しています。

この問題を解決するには、高速PTZカメラ、ズームとズームレンズを備えたデバイス、マルチプレクサー、ネットワークビデオレコーダーなど、さまざまなメーカーの製品を1つのシステム内で組み合わせるには、単一の国際プロトコルが必要でした。

ただし、これまでのところ、ビデオ機器メーカーのネイティブプロトコルは依然として関連しています。 video98％のカメラタイプをクラウドに接続できるIvideon Bridgeデバイスでも、ネイティブプロトコルで作業するときに特別な機能を提供します。

これがなぜ起こったのか、ネイティブプロトコルの利点は何か、Dahuaテクノロジーとの統合の例についてさらに説明します。

シングルスタンダード




歴史的に、多くのメーカーのクラス最高のソリューションを組み合わせて、最も効率的なシステムを作成するには、大量の統合作業が必要でした。

2008年の機器の非互換性の問題を解決するために、Open Network Video Interface Forum標準が開発されました。ONVIFにより、設計者と設置者は、ビデオシステムのすべてのコンポーネントのデバッグに必要な時間を短縮できました。

システムインテグレーターとエンドユーザーは、システムを拡張するとき、または個々のコンポーネントを部分的に交換するときに、メーカーを自由に選択できるため、ONVIFで費用を節約できました。

ビデオ機器のすべての主要メーカーからのONVIFのサポートにもかかわらず、ほとんどすべての主要企業は、すべてのカメラおよびDVRメーカーに固有のネイティブプロトコルを残しています。

Dahua Techでは、多くのデバイスがonvifと独自のDahuaプライベートプロトコルの両方をサポートしています。Dahuaは、独自の機器に基づいて複雑なセキュリティシステムを構築するために使用しています。

ネイティブプロトコル




制限がないことは、ネイティブ開発の利点です。組み込み機能のメーカーは、彼が主要な機能であると考える「機能」に焦点を当てており、自分のハードウェアのすべての機能をサポートしています。

その結果、ネイティブプロトコルは、ハードウェアリソースの使用効率を最大化するため、製造元にデバイスのパフォーマンスとセキュリティへの信頼を与えます。

これは必ずしも良いことではありません。Aliexpressの膨大な数のカメラが、トラフィックを全世界に「公開」する「漏出性」の高いオープンプロトコルに対応していることは、この証拠です。 Dahua Technologyレベルのメーカーは、システムのセキュリティを長期間テストする余裕があるため、状況は異なります。

ネイティブIPカメラプロトコルを使用すると、ONVIFでは実現できないレベルの統合を実現できます。たとえば、ONVIF互換のカメラをNVRに接続する場合、デバイスを見つけて追加し、リアルタイムで動作を確認する必要があります。カメラがネイティブプロトコルに従って「通信」する場合、その検出とネットワークへの接続は自動的に行われます。

他社製のカメラでレコーダーを使用すると、画質が低下することがあります。同じメーカーのデバイスにネイティブプロトコルを使用する場合、最大800メートルのケーブルを介して信号を送信する場合でも、この問題は原則として発生しません（拡張Power over Ethernetテクノロジを使用）。

このテクノロジーは、Dahua Technologyによって作成および導入されました。EPoE（Power over Ethernet）テクノロジーは、従来のイーサネットとPOE（どちらもネットワークポート間の100メートルに制限されています）の制限を克服し、PoEデバイス、イーサネットエクステンダー、または追加のネットワークスイッチの必要性を排除します。

2D-PAM3コーディング変調を使用することにより、新しいテクノロジーは、長距離にわたって電力、ビデオ、オーディオ、および制御信号を提供します。DahuaのePoEは、より柔軟で信頼性の高いビデオ監視システムであり、設置と配線を節約できます。

Dahuaテクノロジーとの統合




2014年に、Ivideon は、世界のセキュリティシステム市場で2番目に大きなシェアを所有する、世界の大手ビデオ機器メーカーの1つであるDahuaとの提携を開始しました。現在、Dahua は、 a＆s Security 50の売上高が最大の企業のランキングで2番目です。 当社の緊密な相互作用により、合計で数千のモデルのネットワークカメラやDVRを含む多くの機器プラットフォームの統合が可能になりました。 2017年に、Dahua HDCVI DVRを使用して標準および高精細アナログカメラをクラウドに接続できるソリューションを開発しました。





また、DVR、PC、または追加のソフトウェアを使用せずに、地理的な場所に関係なく、Dahuaカメラをいくつでもクラウドに接続する簡単なメカニズムを提供することもできました。

2019年、私たちはDIPP（Dahua Integration Partner Program）の枠組みの戦略的パートナーになりました。これは、ビデオ分析ソリューションを含む統合された統合ソリューションの共同開発を目的とした技術協力プログラムです。 DIPPは、共同製品の優先設計と技術サポートを意味します。

新しい製品を作成するすべての段階でDahuaをサポートすることで、さまざまなソリューションのネイティブプロトコルと対話できるようになりました。昨年の最も興味深いガジェットの1つ-Ivideon Bridgeこれにより、Dahuaカメラとの互換性を「ネイティブ」デバイスのレベルで達成することができました。

「橋」が導くところ



ブリッジは、小さなWi-Fiルーターのサイズのガジェットです。このボックスでは、最大16台のカメラをIvideonクラウドに接続できます。つまり、ローカルシステムのユーザーは、インストールされている機器を交換することなく、クラウドサービスにアクセスできます。Ivideon Bridgeに接続されたDVRを介して、クラウドにアナログカメラを追加することもできます。

今日のデバイスのコストは6,000ルーブルです。価格/チャネル比の点で、BridgeはIvideonクラウドに接続するための最も収益性の高い方法になっています。比較のために：クラウドにアクセスできるカメラを購入すると、1つのチャネルのコストは5,500ルーブルになります。

Ivideon Bridgeは単なるDVRではなく、クラウドを介したリモート管理を大幅に簡素化するプラグアンドプレイデバイスです。

ブリッジの興味深い機能の1つは、Dahuaネイティブプロトコルの完全なサポートです。その結果、BridgeはCCTVシステムの有効性に直接影響を与える機能で強化されました。

ローカルデータ記録


Edge Storageは、ネイティブプロトコルを使用してBridge経由で接続されているすべてのDahuaカメラとDVRで使用できます。Edgeでは、内部メモリカードまたはNASにビデオを直接記録できます。Edge Storageは、次の柔軟な記録ツールを提供します。

  • ネットワークリソースとドライブの節約。
  • データストレージの完全な分散化。
  • 帯域幅使用率の最適化。
  • 接続が切断された場合にバックアップアーカイブバックアップを作成する。
  • クラウドアーカイブへの保存：より低い料金プランを設定するだけで十分です。たとえば、クラウド内の8台のカメラの最低年間コストは、わずか1,600ルーブル/月または19,200ルーブル/年になります。

ネイティブプロトコルを使用してのみ利用可能なEdgeモードは、一方では突然の切断のビジネスのリスクを軽減し、他方では高いトラフィックコストを節約するハイブリッド録音ソリューションです。

OSDとバックライトの設定




Ivideon Bridgeでは、画像に任意のテキスト、日付、時刻をオーバーレイする設定（オンスクリーンディスプレイ、OSD）へのアクセスが開かれています。

テキストラベルと日付をドラッグすると、非表示のグリッドに「固定」されます。このグリッドはカメラごとに異なり、ラベルが配置されている画像の部分に応じて、オーバーレイテキストの実際の位置は異なる方法で計算できます。

テキストオーバーレイまたは日付を無効にすると、設定が保存され、オンにすると復元されます。

特定のカメラで利用可能な設定の構成は、モデルとファームウェアのバージョンによって異なります。

モーションディテクターの操作パラメーター




このシステムでは、任意の検知ゾーンの設置を含め、モーション検知器のパラメーターを非常に敏感に変更できます。

ビデオストリームの設定を変更する




ビデオストリームとオーディオストリームのパラメーターを設定すると、インターネットチャネルの負荷を軽減できます。多くの値を「トリミング」してトラフィックを節約できます。

マイクのセットアップ




ビデオストリームの場合と同様に、マイク設定は感度スケールへのアクセスを提供します。これにより、騒がしい室内でのデバイスの使用を最適化できます。
