モバイルOSおよびブラウザーのinputmode属性の機能

こんにちは、ハブル！クリスチャン・オリフの記事「inputmodeについて知りたいことすべて」の翻訳を紹介します。

翻訳者のメモ：記事の最初のタイトル「inputmodeについて知りたかったすべて」は、2番目がより正確に本質を伝えるため、「モバイルOSとブラウザーのinputmode属性の機能」に変更されました。

グローバルinputmode属性は、ユーザーがinputまたはtextarea要素を選択したときに表示するオプションを画面キーボード付きのデバイスのブラウザーに通知します。

<input type="text" inputmode="" />
<textarea inputmode="" />

フォームのタイプを変更するのとは異なり、inputmodeは、ブラウザーが入力を解釈する方法を変更しません-どのキーボードで表示するかをブラウザーに指示します。

inputmode属性には長い歴史がありますが、つい最近、主要なモバイルブラウザー（iOSのSafariおよびAndroidのChrome）に採用されました。以前は2012年にAndroid版Firefoxに実装されていましたが、その後数か月後に削除されました（ただし、フラグを通じて引き続き使用できます）。ほぼ6年後、Chrome for Androidはこのプロパティを作成しました。iOS12.2の最新リリースでは、Safariもこのプロパティをサポートしています。

ブラウザ属性のサポートはこちらで確認できます。数字はブラウザのバージョンを示します。

一番下の行に入る前に、WHATCの生活標準がinputmodeのドキュメントを提供しているのに対し、W3C 5.2仕様にはそれが含まれていないことを覚えておいてください。WHATWGは属性を文書化し、ブラウザーはそのサポートに取り組んでいるため、WHATWGの技術的特性を標準として検討します。
Inputmodeはいくつかの値を取ります。それらを1つずつ見ていきましょう。

＃無し


<input type="text" inputmode="none" />

キーボードは必要ないので、それから始めましょう。inputmode =” none”を使用すると、Android for Chromeでキーボードが表示されません。iOS 12.2はデフォルトで英数字キーボードを表示するため、この場合の特性noneはiOSの場合は少しリセットされる可能性があります。

#Numeric


<input type="text" inputmode="numeric" />

これはおそらく最も一般的なinputmode値の1つです。たとえば、ピンコード、郵便番号、クレジットカード番号など、数字のみを必要とする入力フィールドに最適です。 type =” text”で数値を使用すると、入力をtype =” number”に切り替えるよりも意味があります。inputmode=” numeric”は、maxlength、minlength、およびpattern属性で使用できるため、さまざまな状況でより用途が広くなります。

テンキーを表示するためにtype =“ tel”を外国語で使用しているサイトをよく見ました。これは回避策と見なされますが、意味的には正しくありません。

inputmodeはpattern属性をサポートしていることに注意してください。同じ効果を得るために、入力にpattern =” \ d *”を追加できます。Androidは（iOSとは異なり）ユーザーがキーボードで文字を使用してアクセスすることを許可しないため、有効なデータの入力を誤って妨げる可能性があるため、入力が数字のみの入力を許可することが確実な場合にこれを使用します。

#Tel


<input type="text" inputmode="tel" />

標準の英数字キーパッドを使用して電話番号を入力するのは難しい場合があります。電話のキーパッドでは、数字が付いた各ボタン（1と0を除く）は、同じキーにある3つの文字（たとえば、2はA、B、Cを表します）を表します。英数字キーパッドはこれらの文字を参照しないため、文字を含む電話番号（たとえば、1-800-COLLECT）をデコードするには追加のリソースが必要です。

inputmodeをtelに切り替えて使用すると、標準の電話のキーパッドが表示されます。これには、0〜9の番号用のボタン、ポンド記号、アスタリスクが含まれます。さらに、アルファベットのニーモニック文字（ABCなど）を使用します。

＃小数


<input type="text" inputmode="decimal" />

10進数に切り替えるinputmodeフィールドは、iOSの小さな変更につながります。この場合、キーボードはtelの場合と同じに見えますが、+ *＃記号は小数点に置き換えられます。Androidは数字キーパッドのみを使用します。

＃Eメール


<input type="text" inputmode="email" />

メールアドレスの入力が必要なフォームに入力したため、@記号にアクセスするためだけにキーボードを切り替える必要がありました。これは致命的ではありませんが、決して便利ではありません。これがメールの価値が役立つ場所です。@記号と小数点が追加されます。

このようなキーボードは、Twitterでユーザー名を入力する必要があるユーザーに役立ちます。@は、このソーシャルネットワークでユーザーを識別するための主要な文字であるためです。使用するもう1つの理由は、電子メールをチェックするための独自のスクリプトがあり、ブラウザーに組み込まれている検証機能を使用したくない場合です。

#URL


<input type="text" inputmode="url" />

URL値を使用すると、1つのボタンを押すだけでドメイン名拡張子（たとえば、.com）を追加するのが最も簡単になります。文字（。）や（/）など、Webアドレスに一般的に使用されるボタンもあります。キーボードに表示される拡張子がユーザーエントリに追加されます。

このようなキーボードは、入力フィールドがドメイン名（site.com）と完全修飾ドメインアドレス（https://site.com）を受け入れることをユーザーに示すのに役立ちます。
入力フィールドで入力した文字の検証が必要な場合は、type =” url”を使用します。

＃探す


<input type="text" inputmode="search" />

この値は、どちらもReturnの代わりに、iOSでは青色のGoボタン、Androidでは緑色のEnterを表示します。名前から推測したように、検索は検索フォームに使用され、クエリを生成するためのキーを提供します。

緑の矢印の代わりにiOSではEnterの代わりに検索を表示し、Androidでは虫眼鏡アイコンを表示する場合は、type =” search”を使用します。
