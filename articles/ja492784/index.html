DMCAとは何ですか？


デジタルミレニアム著作権法は、デジタル時代の著作権を強制する法律です。彼は1998年に米国で受け取られ、それぞれ米国の管轄区域でのみ行動します。

DMCAは2つの部分で構成されます。1つ目は「回避に対する保護」、暗号化を回避する責任を確立すること、2つ目は「追跡と通知」により、違反者との裁判に時間とお金を費やすことなく、著作権を侵害するコンテンツをネットワークからすばやく削除することです。映画、アルバム、ゲーム、またはその他の著作権オブジェクトの違法コピーがサイトで見つかった場合、著作権所有者は削除通知サービスプロバイダーの正式な代表者を送り、その中で彼の主張を述べ、とりわけ彼の意図の誠実さまたは「誠意」を確認しますコンテンツが悪意を持って「スパイラル」であると。このような通知を受け取ったサービスプロバイダーは、バックアップコピーを保持しながらコンテンツを削除するか、検索結果から非表示にします。


モデルのウェブカメラは、DMCA保護ロゴをブロードキャストに追加して、海賊版サイトからの録音を簡単に削除できるようにします。

ただし、削除されたコンテンツの所有者はAPの違反について異議申し立て通知を提出する権利を持っているため、これで話が終わることはありません。今回、貧弱なサービスプロバイダーは、削除されたファイルを10日以内に復元し、ポップコーンを補充し、紛争当事者が法廷に行くか、月桂樹で休むのを待たなければなりません。
DMCAは米国のすべてのホスティングプロバイダーに必須であり、実際にはリクエストが毎日数百件届く可能性があり、最も不合理なものであってもすべて満たす必要があるため、多くの場合頭痛の種となります。ただし、サービスプロバイダーは、海賊版ファイルまたはリンクのコンテンツに対する責任から解放されます。そのタスクは、コンテンツが削除通知によって削除され、作者に要求の事実を報告することを確認することのみです。

VPNサーバー上のDMCA侵入者を見つける


米国でサーバーをレンタルする場合、IP Echelonからの自動苦情は大きな問題になる可能性があります。

実際の事例：企業リソースにアクセスするためのVPNサーバー。 VPNサーバーはインターネットへのアクセスにも使用されます。つまり、すべてのトラフィックをルーティングします。

企業規則では、動作中のVPNを介した海賊版コンテンツのダウンロードを厳しく禁止しています。しかし、安定して月に1回、誰かがVPNに接続する前に急流のクライアントをオフにするのを忘れ、その結果、ホスティング事業者はDMCAの乱用を受け取り、すぐに満足する必要があります。
同時に、サーバー自体には海賊版コンテンツはなく、従業員の自宅のコンピューターにあります。その結果、トレントクライアントをすべてのVPNユーザーにオフにする要求を含む通知を送信する必要がありました。

海賊版コンテンツを配布するユーザーを正確に特定するタスクがありました。 IP Echelonの苦情には、トレントクライアントが接続された正確な時刻とポート番号が含まれているため、すべてのVPNクライアント接続を時間参照で記録し、これらのログを使用して侵入者を見つけることができます。

iptablesルールを追加：

iptables -t NAT -A POSTROUTING -s 10.0.0.0/24 -o eth0 -m conntrack --ctstate NEW -j LOG

10.0.0.0/24は、VPNクライアントのIPアドレスの内部範囲です。

このようなルールは、作成されたすべてのNAT変換をシステムログ、つまり、内部IPアドレスを参照するVPNクライアントの外部接続に保存します。

カーネルログエントリは次のようになります。

kernel: IN= OUT=enp0s4 SRC=10.0.0.3 DST=1.1.1.1 LEN=61 TOS=0x00 PREC=0x00 TTL=64 ID=26834 PROTO=UDP SPT=10821 DPT=53 LEN=41
kernel: IN= OUT=enp0s4 SRC=10.0.0.3 DST=140.82.114.26 LEN=64 TOS=0x00 PREC=0x00 TTL=64 ID=0 DF PROTO=TCP SPT=64837 DPT=443 WINDOW=65535 RES=0x00 SYN URGP=0

内部アドレスを持つVPNクライアントが10.0.0.31つのDNSクエリを完了し、アドレスへのHTTPS（ポート443）に接続していることがわかります140.82.114.26。さて、虐待を受け取ったので、正確な時間までに侵入者の正確な住所を見つけて、彼にのみ通知することができます。

DMCAに従うべきか


DMCAは米国以外の国には適用されません。その他の地域では、知的財産権の主張は、現地法またはWIPO（世界知的所有権機関）によって管理されています。後者は、論争の的であるコンテンツの削除よりも、所有者からのドメイン名全体の削除に関する論争に巻き込まれることが多い。

したがって、会社とホスティングプロバイダーが米国に所在しない場合は、DMCAに準拠する必要はありません。

ただし、一部の国内のホスティング業者は、顧客にそのような苦情への対応を求め、2回目の苦情を受け取ったときにクライアントをブロックすることさえあります。誰もが苦情を送信し、クライアントを麻痺させる可能性があるため、この動作は顧客にとって危険な場合があります。
