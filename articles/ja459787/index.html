C ++、型が定義されているかどうか：必要なオブジェクトの予備宣言

前回、SFINAEを使用して、型に定義があるかどうかを確認しました。これをif constexprユニバーサルラムダ式と組み合わせて  使用し、コードが型を定義されている場合に使用できるようにしながら、コンパイラーがそれを受け入れ（そして破棄）しましたタイプが定義されていない場合。

ただし、このアプリケーションにはいくつかの問題があります。

  • あなたが書く必要があるたびに  struct。
  • タイプが存在しない場合、名前を割り当てると、このタイプは、タイプを指定するネームスペースではなく、現在のネームスペースに入力されます。
  • struct非修飾名で使用する必要があります  。これを使用して、現在のネームスペースにインポートしていないタイプをチェックすることはできません。

3つの問題すべてを1つの解決策で修正できます。まず、目的の名前空間で型を宣言します。



// awesome.h
namespace awesome
{
  //     
  struct special { ... };
}

//  

namespace awesome
{
  //   ,   
  struct special;
}

これを行った後はstruct、構造が宣言されているため、記述する必要はありません。でcall_if_defined すべてがすでに発表されているため、テンプレートタイプのパラメータとして使用しても、  新しい宣言は作成されません。そして、彼女が宣言されたので、あなたは彼女の非修飾名、完全な名前空間名、またはその間のあらゆるものを通して彼女にアクセスできます。また、型エイリアスまたは依存型を介して（残念ながら、それらは間にありません）。

namespace app
{
  void foo()
  {
    call_if_defined<awesome::special>([&](auto* p)
    {
       //      "awesome::special"
       // .     "special"
       //     .
       using special = std::decay_t<decltype(*p)>;

       //      "special"  
       //   "awesome::special".
       special::do_something();
    });
  }
}

このシリーズの記事を最初からフォローしている人は、メソッドcall_if_definedが以前に作成したバージョンと完全に一致していないことに気づいたかもしれません。新しいバージョンはいくつかの型パラメーターをサポートし、すべての型が定義されている場合にのみラムダを呼び出します。

詳しく見てみましょう：

template<typename... T, typename TLambda>
void call_if_defined(TLambda&& lambda)
{
  if constexpr ((... && is_complete_type_v<T>)) {
    lambda(static_cast<T*>(nullptr)...);
  }
}

if constexpr（（...））の二重括弧は奇妙に見えますが、必須です。演算子には外部括弧が必要  if constexprであり、畳み込み式には内部括弧が必要です  。畳み込み式は次のように拡張されます

  if constexpr (
    (is_complete_type_v<T1> &&
     is_complete_type_v<T2> &&
     ...
     is_complete_type_v<Tn>))

ラムダ呼び出しは、パラメーターパッケージ拡張を使用し  ます。

    lambda(static_cast<T*>(nullptr)...);

それはに拡大します

    lambda(static_cast<T1*>(nullptr),
           static_cast<T2*>(nullptr),
           ...,
           static_cast<Tn*>(nullptr));

それはどこ  static_cast<T*>(nullptr) 種類ごとに一回繰り返されます。

前に述べたように、すべての型が定義されている場合は、この関数を使用してラムダを呼び出すことができます。

void foo(Source const& source)
{
  call_if_defined<special, magic>(
    [&](auto* p1, auto* p2)
    {
      using special = std::decay_t<decltype(*p1)>;
      using magic = std::decay_t<decltype(*p2)>;

      auto s = source.try_get<special>();
      if (s) magic::add_magic(s);
    });
}

C ++ 20では、次のように記述できます。

void foo(Source const& source)
{
  call_if_defined<special, magic>(
    [&]<typename special, typename magic>
    (special*, magic*)
    {
      auto s = source.try_get<special>();
      if (s) magic::add_magic(s);
    });
}

これにより、テンプレートのタイプに名前を付けることができるため、での作業中にテンプレートを再抽出する必要がなくなりstd::decay_tます。
