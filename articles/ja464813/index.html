Punto Switcher、Birmanレイアウトをautohotkeyのスクリプトに置き換える

この記事では、Punto Switcherを完全に置き換えることは提案していません（このプログラムについてはSergey Moskalevに感謝します）。ただし、機能が非常に限定されています。私は長い間Punto Switcherプログラムを使用してきましたが、自動切り替えの目的ではなく、左右のシフトのスイッチとして使用しています。ドス以来、私はそのような切り替えに慣れてきました。Windows（特にバージョン7）では、ウィンドウに表示される言語を考える必要はありませんが、左Shiftキーを押すだけで、メインのロシア語を入力する準備が整います。右手にマウスを置いている間、左はすでに左Shiftキーを押すだけで言語を切り替えたりロシア語を設定したりできます。このため、どこにでも集中する必要がなく、すべてが自動的に行われます。したがって、左シフトで言語をロシア語に、右シフトで英語に切り替えると、常に便利でした。

時が経つにつれ、ユーザーに紹介するのに別の言語が必要になり、Birmanレイアウトやその他のレイアウトのサポートも求められます。私は自分自身を評価することができたどんな不便さ、そしてそれが私がこれを行う方法を理解しなければならなかった理由です。バーマンレイアウトのみを使用する場合、ホットキーがある場合は右側のAltを使用すると、それらはこのレイアウトと重なり、何も機能しません。また、各デスクトップショートカットで「Ctrl + Alt +キー」へのショートカットを切断できるため、このようなホットキーはたくさんあります。ただし、バーマンのレイアウトのように作成され、適切なAltがなければ、キーを使用する場合、これらの言語のヒープを切り替える問題は依然としてそうです。

つまり、autohotkeyでスクリプトを使用するよりも優れた方法は思い付きませんでした。このスクリプトを使用すると、必要な多くのことを実行でき、他の人が簡単に変更できるからです。したがって、この記事はHabrの一部の読者にとって有用であり、私が悔い改めたときに専門家が私のコードを修正することができますが、これはautohotkeyでスクリプトを書いた最初の経験です。

完璧な解決策はなく、常に何かを犠牲にしなければなりません。この場合、右のキーRAltとRWinはめったに使用しないので、それらの機能を犠牲にする必要がありました。 RAltは他の言語でも使用できるため、RCtrlを使用しています。一部のプログラムには独自のキーボードステートのトリッキーインターセプターがあるため、RAltキーとRWinキーを押したときの反応を完全に「殺す」ために、KeyTweakプログラムを使用して、適切なALtを「WWW Stop」に、適切なWinを「My Computer」に設定する必要がありました。マルチメディアキーボードのこれらのキーは、ほとんどの人が使用することはないため、私の目的に最適です。また、誤ってクリックしても、Windowsの機能には影響しません。

私のバーマンレイアウトはほぼ完全に繰り返されています（グリブナ記号が追加されています）。ただし、それはRWinキー上にあり、おそらくこれはRAltのオリジナルほど便利ではありません。しかし、これは私の主なツールではなく、RAltキーの主なツールです。

Birmanレイアウトからシンボルを印刷するには、RWinキーとその上にこのシンボルが茶色で描かれているキーを押す必要があります。シンボルが明るい色で描画されている場合は、Shiftキーを押したままにするとシンボルが印刷されます。例：RWin + Shift +は、文字を出力します。

CapsLockキーは左Ctrlとして使用されます。私が盲目的にタイプすると、LCtrlに到達するのが非常に非常に不便であり、RCtrlに到達することは完全に不可能です。 CapsLockが必要になることはめったにないので、LAlt + CapsLockに切り替えます。

いくつかの切り替えレイアウトがあります（可能なWindowsを除いて：RWin + SpaceまたはRAlt + RShift）：LShift-ロシア語に切り替え、RShift-英語に切り替え、RAlt-EnとRuレイアウトのみを切り替え、LAlt + Space-他のレイアウトを切り替え、 EnでもRuでもない。 CapsLockおよびこれらのレイアウトの切り替えはすばやく行われます。つまり、CapsLockはボタンを押した直後にボタンを押しました。これは、複雑なキーの組み合わせを押すことによる問題を排除するために行われます。 Shift言語の切り替えは、Punto Switcherよりも速く応答します。これにより、タイピングの高速化に満足するだけです。 RAltに従ってレイアウトを切り替えた後、別の言語（ロシア語や英語ではない）で単語を入力する必要がある場合、LAlt +スペースですばやく切り替えて目的の単語を入力すると、RAltを1回クリックするだけで、その言語を取得できますその前でした言語（RuまたはEn）が記憶されるため。これまでのところ、LAlt + Spaceによる言語は記憶されていませんが、言語が多く、多くの場合3言語だけでなく4.5言語などが必要な場合は、スクリプトを変更してそのような関数を作成する必要がある場合があります。

LAltキーを使用すると、通貨記号、ウクライナ語のアポストロフィ、アクセント記号、長いスペース、複雑なスペースを入力できます。 LAltがメニューで使用され、すべての文字キーを再割り当てしないことが重要であることを理解していますが、ここではスクリプトでいくつかのデジタルキーを使用し、LAlt + 1,2,3,4には触れなかったため、標準の計算機で使用するのに便利です。

主な作業キーはRAltです。このスクリプトとこのキーを使用して、カーソルをテキストの上に簡単に移動し、目的の文字を簡単に削除または挿入できます。右手を動かさずに、ブラインドセットのメインポジションから離れます。長い間プログラミングをしなければならない人は、ロシア語の文字が記号[]または<>に取って代わったという問題を知っています。記事を書いているときにこれらの文字を入力する必要がある場合は、言語を切り替えることなく行うことはできません。このスクリプトでは、RAltキーを使用してこの問題を解決しました。例：RAlt + b-文字を出力します]; RAlt + 2-@記号を印刷します。 RAlt + Yu-a>記号を印刷します。

キーの組み合わせRAlt + Tab +、などは、テキストカーソルを移動せず、マウスカーソルを移動します。 Tabを使用すると、このスクリプトはマウスが突然壊れた場合などに、何らかの方法でマウスを置き換えることができます。私にとって、この機能は組み込みのWindowsの特別な機能よりも少し便利です。

まあ、基本的なロシア語はあるが、ウクライナ語、ベラルーシ語の記号を入力する必要があり、モルダビア語のキリル文字のӂのようなほとんど使用されていない文字でも、RAlt +キーを使用して目的の文字を入力する機会があります。

Autohotkeyを使用するユーザーにとって、このアーカイブから「Keybord Assistent 1_0.ahk」ファイルを確認する必要があることは明らかです。キーを再割り当てする方法すら知らない一般ユーザー（KeyTweakプログラムを使用しているとしましょう）は、管理者権限で実行し、以下を実行する必要があります。

  1. Keybord Assistent 1_0.rarアーカイブを解凍します
  2. 「RAlt RWin.regの再割り当て」ファイルを実行します
  3. コンピュータを再起動します
  4. 「Keybord Assistent 1_0.exe」ファイルを実行します

プログラムはトレイに配置され（約1時間）、プログラム管理メニューが右ボタンで使用可能になり、ヘルプを表示することができます（上図）。 「停止」モードでは、すべてのキーがゲームまたは一部のプログラムで必要になる場合があるため、すべての標準キーの機能（動作）が返されます（RAltとRWinも再割り当てされます）。

プログラムが気に入らなかった場合は、「Return as RAlt RWin.reg」ファイルを実行して、コンピュータを再起動します。

上級ユーザーの場合、スクリプト「Keybord Assistent 1_0.ahk」を編集して実行することができます（もちろん、自動ホットキーがインストールされている場合）。すべての機能の変更は、Keybord Assistent 1_0.xlsxファイルでも修正できます。HelpKeybAs.jpgという新しいファイルを作成して、スクリーンショットを撮ります。
