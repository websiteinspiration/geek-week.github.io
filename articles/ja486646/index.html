jQueryからSvelteに移行する

みなさん、こんにちは。

私はバックエンド開発者であり、フロントエンドタスクをできる限り解決します。つまり、jQueryでは2015年に機能し、現在は機能しています。しかし、Vue and Reactがあれば、これはもはやキャンパスではありません。特別な方法への愛情から、私は何百万人もの開発者によってテストされていないAngular / React / Vueをマスターすることに決めました、私はSvelteを試すことにしました。

教科書からいくつかの練習をした後、私は練習に移ることに決めました。このために、私は正常に完了したテストタスクの1つを取りました。

課題では、タスクのリストを確認する必要があり、このリストのタスクの1つであるCRUDは必要ありません。

クライアント部分はSPAとして実装され、DOMでのすべての作業はjQueryを介して行われます。これは、jQueryをSvelteで置き換える優れた候補です。

以下では、最初の障害について、そしてもちろんそれらを克服する方法について説明します。
Svelteチュートリアルは非常にアクセスしやすく、直感的ですが、Svelteを任意のプロジェクトに埋め込む方法はあまり明確ではありません。SvelteはjQueryのようなライブラリーではないため、コンパイラーです。つまり、Svelteディレクティブを使用して記述されたコードは、ネイティブJSにコンパイルする必要があります。

別の障害は、

$

Svelteでは、これは予約文字であるため、Svelteによってコンパイルされるコードでこの文字を使用すると、エラーが発生します。 


[!] (plugin svelte) ValidationError: $ is an illegal variable name

`$`の使用


もちろん、問題が発生する可能性があります：jQueryをSvelteに完全に変更した場合の `$`はどのようなものですか？
はい、質問は合理的ですが、考えれば問題が明らかになります。象全体を食べる必要はなく、経験豊富な人々は象を部分的に食べます。 jQueryの使用からSvelteの使用への移行に関しては、これは一連のアトミックリファクタリングであり、進行状況は常に表示され、「システム」は全体として常に動作状態になります。

計画は次のとおりです。部分を書き直し、テストし、修復し、次の部分を書き直し、その後、痕跡のない象全体を食べるまで続けます。

`$`の問題は、Svelteがコンパイルするコードでのみ発生します。SPAロジックはすべて「business-process.js」ファイルに移動しました。この問題を解決する必要はありませんが、「business-process.js」のすべてのコードを実行する必要がありますApp.svelteおよびその他の関連アプリケーションコンポーネント。

JSコードでSvelteディレクティブをコンパイルする方法


Svelteブログで説明されて いるように、空のプロジェクトを別のフォルダーに送り込みます（途中でNode.jsをインストールします）。次に、ダウンロードしたファイルをプロジェクトのディレクトリに転送します。

クライアント（ユーザーのブラウザー）がダウンロードするファイルがある場所に「パブリック」からファイルを転送します。「パブリック/アセット/サイト」を持っています。「global.css」からのスタイルは必要ありません。このファイルは転送しませんでした。

ファイル「public / index.html」、サーバーがビューのテンプレートを取得する場所に転送：「view / Site / index.html」。

「src」のファイルがフロントエンドのソースに追加されます-「src / frontend」。

フロントエンドのソースを説明するファイル：

  • rollup.config.js
  • package-lock.json
  • package.json

私はルートに移動し、そこでIDEがそれらを取得してすべて（インストールされた依存関係）を処理しました。
次に、JSのSvelteディレクティブのコンパイル構成を変更する必要があります。

rollup.config.jsを開き、ファイルパスを変更します。


export default {
	input: 'src/frontend/main.js',
	output: {
		sourcemap: true,
		format: 'iife',
		name: 'app',
		file: 'public/assets/Site/bundle.js'
	},
	plugins: [
		svelte({
			dev: !production,
			css: css => {
				css.write('public/assets/Site/bundle.css');
			}
		}),

これが、Svelteを既存のプロジェクトに最初に統合する際の問題です。

依存関係をインストールし、オンザフライでコンパイルするためにフロントエンドのソースコードの変更の追跡を開始します。

npm install
npm run dev
