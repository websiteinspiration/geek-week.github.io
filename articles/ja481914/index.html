Spring Boot vs Spring MVC vs Spring-彼らはどのように比較しますか？




Spring、Spring Boot、Spring MVC、どこでも「春」という言葉があります。これらの各ツールをいつどこで使用できるかを見ていきましょう。


この記事では、Spring、Spring MVC、Spring Bootのレビューで、どのような問題が解決され、どこに最適に適用されるかを説明します。あなたが学ぶ最も重要な事実は、Spring、Spring MVC、およびSpring Bootが同じ場所で競争しないことです。彼らはさまざまな問題を解決し、それらを非常にうまく解決します。

このSpring Frameworkが解決する主な問題は何ですか？


長く考えてください。Spring Frameworkによって解決される問題は何ですか？

どうしてそれが重要ですか？なぜなら、DIまたはIOCが正しく使用されると、疎結合アプリケーションを開発できるからです。また、疎結合アプリケーションは単体テストで簡単にテストできます。

簡単な例を見てみましょう。

依存性注入のない例


以下の例を検討してください。WelcomeControllerは、WelcomeServiceに依存してウェルカムメッセージを受信します。WelcomeServiceのインスタンスを取得するために彼は何をしますか？

WelcomeService service = new WelcomeService();

この行は、このサービスのインスタンスを作成します。そして、これは彼らが強く結びついていることを意味します。たとえば、WelcomeControllerの単体テストでWelcomeServiceのモックを作成した場合、WelcomeControllerでモックを使用するにはどうすればよいですか？簡単ではありません！

@RestController
public class WelcomeController {   private WelcomeService service = new WelcomeService();   @RequestMapping("/welcome")
   public String welcome() {
       return service.retrieveWelcomeMessage();
   }
}

単純な依存性注入の例


依存性注入により、世界ははるかに簡単に見えます。Spring Frameworkにハードワークを任せます。2つの単純な注釈を使用します。成分 そして 自動配線。

  • 使用する 成分Spring Framework：やあ、これは管理する必要のあるビンです。
  • 使用する 自動配線Spring Frameworkと言います。ねえ、この特定の型に合ったものを見つけてプラグインします。

以下の例では、SpringフレームワークはWelcomeServiceのBeanを作成し、WelcomeControllerに接続します。

単体テストでは、SpringフレームワークにWelcomeServiceモックをWelcomeControllerに接続するように要求できます。（Spring Bootは@MockBeanアノテーションを使用してこれを行うことで物事を簡単にします。しかし、それはまた別の話です！）

@Component
public class WelcomeService {
  //Bla Bla Bla
}@RestController
public class WelcomeController {  @Autowired
  private WelcomeService service;  @RequestMapping("/welcome")
  public String welcome() {
      return service.retrieveWelcomeMessage();
  }
}

Spring Frameworkは他に何を解決しますか？


問題1：コードの重複/汚染


Spring Frameworkは依存関係の注入で停止しますか？番号。これは、複数のSpringモジュールによる依存性注入の基本概念に基づいています。

  • 春のJDBC
  • 春のMVC
  • 春aop
  • 春のオーム
  • 春のjms
  • 春テスト

Spring JMSとSpring JDBCについて少し説明します。

これらのモジュールは新しい機能をもたらしますか？番号。これらすべてをJ2EEまたはJava EEで実行できます。それで、彼らは何をもたらしますか？それらは単純な抽象化をもたらします。これらの抽象化の目的は次のとおりです。

  • ボイラープレートコードの削減/重複の削減
  • 切断の促進/テスト容易性の拡張

たとえば、従来のJDBCまたはJMSと比較して、JDBCTemplateまたはJMSTemplateを使用するために必要なコードがはるかに少なくて済みます。

問題2：他のフレームワークとの適切な統合


Spring Frameworkの優れた点は、すでに解決されている問題を解決しようとしていないことです。彼がしていることは、他のフレームワークとの優れた統合を提供することであり、それが今度は優れたソリューションを提供します。

  • ORMの休止状態
  • オブジェクトマッピング用のiBatis
  • ユニットテスト用のJUnitとMockito

Spring Boot Starterプロジェクトオプション


Spring Boot Starter Webからわかるように、スタータープロジェクトは、特定の種類のアプリケーションの開発をすばやく開始するのに役立ちます。

  • spring-boot-starter-web-services：SOAP Webサービス
  • spring-boot-starter-web：WebおよびRESTfulアプリケーション
  • spring-boot-starter-test：単体テストと統合テスト
  • spring-boot-starter-jdbc：従来のJDBC
  • spring-boot-starter-hateoas：HATEOAS機能をサービスに追加します
  • spring-boot-starter-security：Spring Securityを使用した認証と承認
  • spring-boot-starter-data-jpa：Hibernateを使用したSpring Data JPA
  • spring-boot-starter-cache：Spring Frameworkのキャッシングサポートを有効にする
  • spring-boot-starter-data-rest：Spring Data RESTを使用した単純なRESTサービスの提供

Spring Bootの他の目標


技術的なことにはいくつかのスターターもあります。

  • spring-boot-starter-actuator：すぐにアプリケーションを監視および追跡するなどの高度な機能を使用するには
  • spring-boot-starter-undertow、spring-boot-starter-jetty、spring-boot-starter-tomcat：特定の組み込みサーブレットコンテナーを選択するには
  • spring-boot-starter-logging：logbackを使用したロギング
  • spring-boot-starter-log4j2：Log4j2を使用したロギング

Spring Bootは、より短い期間でアプリケーションを作成することを目的としています。

  • アクチュエータ：高度なアプリケーションの監視と追跡を可能にします。
  • 組み込みサーバー統合：サーバーはアプリケーションに統合されているため、サーバーに別のアプリケーションサーバーをインストールする必要があります
