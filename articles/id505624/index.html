Bagaimana cara memproses bingkai data dengan miliaran catatan dalam hitungan detik?

Analisis Data Besar Python sedang dalam era kebangkitan. Itu dimulai dengan perpustakaan NumPy. Perpustakaan ini, omong-omong, adalah salah satu komponen alat yang akan dibahas dalam materi ini. Pada tahun 2006, topik pemrosesan data besar mendapatkan momentum, proses ini dipercepat dengan munculnya Hadoop. Kemudian muncul panda perpustakaan dengan struktur data DataFrame-nya, yang biasanya disebut hanya "data frame". Pada 2014, big data menjadi arus utama, di tahun yang sama, platform Apache Spark muncul. Pada 2018, perpustakaan Dask dan alat-alat lain untuk menganalisis data dengan Python dirilis.



Setiap bulan saya menemukan alat baru untuk analisis data dengan Python yang saya benar-benar ingin pelajari. Menghabiskan satu atau dua jam untuk mempelajarinya, dalam jangka panjang, Anda dapat menghemat banyak waktu. Selain itu, penting untuk memantau perkembangan baru di bidang teknologi yang menarik minat Anda. Mungkin Anda berpikir artikel ini tentang perpustakaan Dask . Tapi ini tidak benar. Hari ini saya akan memberi tahu Anda tentang perpustakaan Python yang baru ditemukan, yang perlu diketahui oleh mereka yang terlibat dalam analisis data.

Perpustakaan Vaex Python



Vaex adalah pustaka Python berkinerja tinggi yang dirancang untuk mengatur pemrosesan data frame yang "malas" (mirip dengan panda data frame), yang dirancang untuk visualisasi dan penelitian set data tabular yang besar. Perpustakaan ini mampu menghitung indikator statistik utama untuk data yang sedang dipelajari. Pada saat yang sama, memproses satu miliar rekaman bisa menjadi sekitar satu detik. Perpustakaan mendukung banyak alat visualisasi, yang membantu untuk mengeksplorasi data besar secara interaktif.

Membandingkan Vaex dan Dask



Pustaka Vaex tidak seperti Dask, tetapi struktur data yang digunakan di dalamnya mirip dengan entitas DataFrame yang digunakan dalam Dask. Mereka dibangun berdasarkan panda dataframe. Ini berarti bahwa Dask mewarisi dari panda pembatasan tertentu untuk bekerja dengan bingkai data. Misalnya, persyaratan bahwa data yang akan diproses harus dimuat sepenuhnya ke dalam memori. Dalam kasus Vaex, ini tidak benar.

Vaex tidak membuat salinan bingkai data, karena itu, dengan menggunakan perpustakaan ini Anda dapat memproses bingkai data skala besar pada komputer dengan jumlah RAM yang kecil.

Baik Vaex dan Dask menggunakan mesin pengolah data yang malas. Perbedaan utama di antara mereka adalah bahwa Vaex menghitung nilai bidang saat dibutuhkan, dan dalam Dask Anda harus secara eksplisit menggunakan fungsi perhitungan nilai.

Untuk melepaskan potensi penuh dari Vaex, data harus disajikan dalam format HDF5 atau Apache Arrow.

Instal Vaex


Menginstal Vaex semudah paket Python lainnya:

pip install vaex

Eksperimen dengan Vaex


Buat bingkai data panda yang berisi 1 juta baris dan 1000 kolom. Ini akan memungkinkan kita untuk mendapatkan file data yang besar.

import vaex
import pandas as pd
import numpy as np
n_rows = 1000000
n_cols = 1000
df = pd.DataFrame(np.random.randint(0, 100, size=(n_rows, n_cols)), columns=['col%d' % i for i in range(n_cols)])
df.head()

Informasi tentang penggunaan memori

Simpan kerangka data ini ke disk untuk kemudian membacanya menggunakan Vaex.

file_path = 'big_file.csv'
df.to_csv(file_path, index=False)

Jika Anda langsung membaca file CSV ini menggunakan Vaex, kami tidak akan mendapatkan banyak manfaat darinya. Kecepatan baca akan serupa dengan kecepatan yang disediakan oleh panda. Di laptop saya, baik Vaex maupun panda membaca data ini dalam waktu sekitar 85 detik.

Untuk melihat kemampuan Vaex, data CSV harus dikonversi ke format HDF5 (Hierarchical Data Format versi 5). Vaex memiliki fungsi untuk melakukan konversi seperti itu, yang memungkinkan Anda untuk bekerja dengan data yang volumenya melebihi jumlah RAM yang tersedia. Ini dicapai karena fakta bahwa sumber data dibagi menjadi beberapa bagian.

Jika tidak memungkinkan, karena kekurangan memori, untuk membuka file yang cukup besar menggunakan panda, file ini dapat dikonversi ke format HDF5 dan diproses menggunakan Vaex.

dv = vaex.from_csv(file_path, convert=True, chunk_size=5_000_000)

Fungsi di atas secara otomatis membuat file HDF5 dan menyimpannya ke disk.

Periksa jenis variabel dv:

type(dv)
# 
vaex.hdf5.dataset.Hdf5MemoryMapped

Sekarang mari kita baca dataset 7,5 GB menggunakan Vaex. Kami tidak harus membacanya sekarang - faktanya itu sudah diwakili oleh variabel dv. Tetapi kami masih menjalankan perintah berikut, melakukan ini untuk mengetahui kecepatan operasi ini:

dv = vaex.open('big_file.csv.hdf5')

Vaex membutuhkan waktu kurang dari satu detik untuk menyelesaikan perintah ini. Tapi Vaex, pada kenyataannya, karena penggunaan mekanisme pemrosesan data "malas", file ini tidak membaca.

Sekarang mari kita membuat Vaex membaca file dengan menghitung jumlah nilai dalam kolom col1:

suma = dv.col1.sum()
suma

Tetapi tim ini benar-benar mengejutkan saya. Juga diperlukan kurang dari satu detik untuk menghitung jumlah Vaex. Bagaimana ini mungkin? Dan mungkin ini karena pemetaan memori.

Visualisasi data


Vaex juga menunjukkan kecepatan yang baik pada visualisasi data. Perpustakaan memiliki fungsi khusus: plot1d, plot2ddan plot2d_contour.

dv.plot1d(dv.col2, figsize=(14, 7))

Kolom virtual


Vaex, saat menambahkan kolom baru ke dataset, membuat kolom virtual. Dia tidak mengambil memori. Nilai yang disimpan di dalamnya dihitung, seperti kata mereka, "on the fly."

dv['col1_plus_col2'] = dv.col1 + dv.col2
dv['col1_plus_col2']

Penyaringan data yang efektif


Vaex tidak membuat salinan bingkai data saat memfilter data. Ini berkontribusi pada penggunaan memori yang lebih efisien.

dvv = dv[dv.col1 > 90]

Agregasi data


Vaex mengumpulkan data sedikit berbeda dari panda. Tetapi yang paling penting di sini adalah bahwa dalam Vaex operasi ini sangat cepat.

Buat kolom baru yang berisi hasil perbandingan dv.col1 >= 50:

dv['col1_50'] = dv.col1 >= 50

Vaex menggabungkan operasi pengelompokan dan pengumpulan data dalam satu tim. Perintah berikut mengelompokkan data menurut kolom col1_50dan menghitung jumlah kolom col3:

dv_group = dv.groupby(dv['col1_50'], agg=vaex.agg.sum(dv['col3']))
