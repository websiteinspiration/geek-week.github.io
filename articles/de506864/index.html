Kommende Linux 5.8-Version: Millionen Zeilen neuer Code und 14.000 Änderungen


In der Regel ist die Veröffentlichung einer neuen Version des Linux-Kernels eine alltägliche Routine, die nur für Benutzer begeistert ist, die erwarten, dass eine bestimmte Funktion angezeigt wird. In Erwartung der Veröffentlichung von Linux 5.8 ist die Situation jedoch etwas anders. In den letzten Wochen wurden im Netzwerk zahlreiche Veröffentlichungen zur neuen Version des Kernels mit einer Beschreibung seiner Funktionen veröffentlicht.

Die neue Version gilt trotz der Tatsache, dass sie noch nicht veröffentlicht wurde, als Version mit der maximalen Anzahl von Änderungen in der Geschichte von Linux. Über 1 Million neue Zeilen sind im Release-Code enthalten. Höchstwahrscheinlich wird es möglich sein, das neue Produkt in den nächsten Wochen zu evaluieren - Linus Torvalds wird RC1 veröffentlichen, wenn nichts Außergewöhnliches passiert.

Besprechen Sie in der Zwischenzeit die Liste der Änderungen .

Daher ist der neue Kern für die Arbeit mit den neuesten Prozessoren von Intel und AMD, einer Reihe anderer Chips und Module sowie der ARM-Architektur optimiert. Netzwerkkomponenten wurden hinzugefügt, neue Open Source AMD Radeon-Grafiktreiber erschienen. Eine stabile Version dieses Zweigs wird übrigens gegen Ende des Sommers - Frühherbst - erwartet. Dann sollten Ubuntu 20.10 und Fedora 33 mit Unterstützung für den neuen Kernel erscheinen.

Der neue Kern kann Flash-Arrays basierend auf MLC als SLC konfigurieren. Microsoft exFAT Treiber wurde entwickelt, SMB3, EXT4 und Btrfs sind optimiert. Außerdem wurde DAX-Unterstützung für den direkten Zugriff auf nichtflüchtigen Speicher hinzugefügt.

Gute Nachrichten für Netzwerkadministratoren, Ingenieure und alle, die an der Bereitstellung der Hardware-Infrastruktur beteiligt sind! Der neue Zweig unterstützt das Testen von Ethernet-Kabeln auf Softwareebene . Damit diese Funktion funktioniert, ist jedoch auch Hardware-Unterstützung erforderlich, damit nichts rein Neues „wächst“.

Es ist bekannt, dass ungefähr 40% der Änderungen im neuen Kernel Treiber betreffen, 16% - um den Code für verschiedene Prozessorarchitekturen zu aktualisieren, 10% der Änderungen beziehen sich auf den Netzwerkstapel, 3% - auf Dateisysteme.

Die Gesamtzahl der Innovationen überstieg 14.000 und betraf etwa 20% der Dateien im Repository. Die Patchgröße 5.8-rc1 beträgt 61 MB.
