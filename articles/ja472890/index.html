開発者向けのDeepPavlov：＃1 NLPツールとチャットボットの作成

みなさん、こんにちは！自然言語処理（または単にNLP）に関連する実用的な問題の解決と、ニューラルシステムおよびディープラーニング研究所のチームによって開発されたオープンソースライブラリDeepPavlovを使用したインタラクティブエージェント（チャットボット）の作成に特化した一連の記事を公開しますMIPT。サイクルの主な目的は、DeepPavlovについて幅広い開発者のサークルと知り合い、機械学習と数学の博士号に関する深い知識がなくても、適用されたNLP問題を解決する方法を示すことです。

NLPタスクには、テキストの調性の決定、名前付きエンティティの解析、相手がボットに求めているものの決定（ピザの注文やヘルプ情報の取得など）が含まれます。NLPのタスクとメソッドの詳細については、こちらをご覧ください。

この記事では、事前にトレーニングされたNLPモデルを使用してREST Northを実行し、追加の構成やトレーニングをしなくてもすぐに使用できるようにする方法を説明します。

DeepPavlovをインストールする


以下に、Linux向けの手順を示します。Windowsについては、ドキュメントをご覧ください。

  • 現在サポートされているバージョンのPythonで仮想環境を作成してアクティブ化します。

    virtualelnv env -p python3.7
source env/bin/activate
  • DeepPavlovを仮想環境にインストールします。

    pip install deeppavlov

以下の記事で、DeepPavlovの内部にあるものについて詳しく説明します。今のところは、次のことだけを知っていれば十分です。

  • DeepPavlovのモデルは、その構成の名前で識別されます。
  • モデルを実行するには、そのコンポーネントをDeepPavlovサーバーからダウンロードする必要があります。
  • また、モデルを実行するには、使用するPythonライブラリをインストールする必要があります。

最初に発表するモデルは、多言語の名前付きエンティティ認識（NER）です。モデルは、テキストワードが属する名前付きエンティティのタイプ（固有名詞、地理名、通貨名など）に従ってテキストワードを分類します。NERの最新バージョンの構成の名前：

ner_ontonotes_bert_mult

RESTサーバーをモデルで起動します。

  1. アクティブな仮想環境の構成に示されているモデルの依存関係をインストールします。

    python -m deeppavlov install ner_ontonotes_bert_mult
  2. DeepPavlovサーバーからシリアル化されたモデルコンポーネントをダウンロードします。

    python -m deeppavlov download ner_ontonotes_bert_mult

    シリアル化されたコンポーネントは、デフォルトで配置されているDeepPavlovホームディレクトリにダウンロードされます。 
    ~/.deeppavlov
    ダウンロード時には、すでにダウンロードされているコンポーネントのハッシュが、サーバーにあるコンポーネントのハッシュと照合されます。一致した場合、ダウンロードはスキップされ、既存のファイルが使用されます。ダウンロードされたコンポーネントのサイズは、平均で0.5〜8 Gbであり、解凍後に20 Gbに達する場合もあります。
  3. RESTサーバーをモデルで起動します。

    python -m deeppavlov riseapi ner_ontonotes_bert_mult -p 5005

このコマンドの結果として、RESTサーバーがホストマシンの5005ポート（デフォルトポートは5000）上のモデルで起動されます。

API DeepPavlov


各DeepPavlovモデルには、少なくとも1つの入力引数があります。 REST APIでは、引数に名前が付けられ、その名前は着信ディクショナリのキーです。ほとんどの場合、引数は処理する必要のあるテキストです。引数とモデルが返す値については、DeepPavlovドキュメントのMODELSセクションで詳しく知ることができます。

この例では、2つの行のリストが引数xに渡され、それぞれに個別のマークアップが指定されています。 DeepPavlovでは、すべてのモデルが、独立して処理される値のリスト（バッチ）を入力として受け入れます。

「バッチ」という用語は、機械学習の分野を指し、アルゴリズムまたはニューラルネットワークによって同時に処理される独立した入力値のパッケージを指します。このアプローチにより、入力に個別に転送される同じ値と比較して、バッチの1つの要素のモデル処理時間を（多くの場合、大幅に）削減できます。ただし、処理結果はすべての要素を処理した後にのみ発行されます。したがって、入力バッチを作成するときは、モデルの速度と、個々の要素ごとに必要な処理時間を考慮する必要があります。

DeepPavlovモデルの引数がいくつかある場合、それらのそれぞれに独自の値のバッチがあり、出力ではモデルは常に1つの回答バッチを提供します。送信バッチは、同じインデックスを持つ受信バッチの要素を処理した結果です。

上記の例では、モデルの結果は、各行のトークン（単語と句読点）への分解と、それが表す名前付きエンティティ（組織の名前、通貨）に対するトークンの分類でした。現時点では、ner_ontonotes_bert_multモデルは 18種類の名前付きエンティティを認識できます。詳細な説明はこちらにあります。

DeepPavlovの他の標準モデル


DeepPavlovのNERに加えて、以下の既成のモデルが執筆時点で利用可能です。

テキスト質問回答


本文への質問への回答は、この本文の一部です。モデル構成：squad_ru_bert_infer

リクエストの例：

{
  "context_raw": [
    "DeepPavlov   .",
    " -      15 ."
  ],
  "question_raw": [
    "  DeepPavlov?",
    "     ?"
  ]
}

結果：

[
  [" ", 27, 31042.484375],
  ["15 ", 39, 1049.598876953125]
]

侮辱の検出


テキストの宛先である人に対する侮辱の存在の識別（執筆時、英語のみ）。モデル構成：insults_kaggle_conv_bert

リクエストの例：


{
  "x": [
    "Money talks, bullshit walks.",
    "You are not the brightest one."
  ]
}

結果：

[
  ["Not Insult"],
  ["Insult"]
]

感情分析


テキストの調性の分類（ポジティブ、ニュートラル、ネガティブ）。モデル構成：rusentiment_elmo_twitter_cnn

リクエストの例：

{
  "x": [
    "   DeepPavlov.",

    "    ."
  ]
}

結果：

[
  ["positive"],
  ["neutral"],
  ["negative"]
]

言い換え検出


2つの異なるテキストが同じ意味を持つかどうかを判断します。モデル構成：stand_paraphraser_ru

{
  "text_a": [
    "   ,  .",

  ],

    " ,     ,   .",
    "       ."
  ]

結果：

[

結論


この記事では、DeepPavlov APIと、追加設定なしで提供されるライブラリーのテキスト処理機能のいくつかについて説明しました。どのNLPタスクでも、タスクのサブジェクトドメイン（ドメイン）に対応するデータセットでモデルをトレーニングすると、最良の結果が得られることに注意してください。さらに、原則として、さらに多くのモデルをすべての状況でトレーニングすることはできません。

次の記事では、追加のライブラリ設定について検討し、DockerからDeepPavlovを起動します。その後、モデルの指導に移ります。また、DeepPavlovにはフォーラムがあることを忘れないでください。ライブラリとモデルについて質問してください。ご清聴ありがとうございました！
