Waves RIDE dAppをビルド、デプロイ、テストする方法

こんにちは！この記事では、Wavesノードで通常のdAppを作成して実行する方法を示します。必要なツール、方法、開発例を検討してください。



dAppsと通常のアプリケーションの開発スキームはほとんど同じです。

  • コードを書く
  • 自動テストの作成
  • アプリケーションを起動します
  • テスト中

ツール


1. dockerノードとWaves Explorer

を起動するにはノードを起動しない場合は、この手順をスキップできます。結局のところ、テストと実験的なネットワークがあります。ただし、ノードをデプロイしないと、テストプロセスが遅れる可能性があります。

  • テストトークンのある新しいアカウントが常に必要になります。テストネットワーククレーンは、10分ごとに10波を変換します。
  • テストネットワークのブロックの平均時間は1分、ノードでは15秒です。これは、トランザクションで複数の確認が必要な場合に特に顕著です。
  • パブリックテストノードでは、積極的なキャッシュが可能です。
  • また、メンテナンスのため一時的にご利用いただけない場合があります。

さらに私はあなたがあなたのノードで働いていると考えます。

2. Surfboardコマンドラインツール

  • https://nodejs.org/en/download/で、ppa、homebrew、またはexeを使用してNode.jsをダウンロードしてインストールします。
  • 既存のノードでテストを実行できるツールであるSurfboardをインストールします。

npm install -g @waves/surfboard

ウォレットデモアプリ


このセクションは、RIDE言語のガイドではありません。むしろ、ブロックチェーンで何が起こっているかをよりよく理解するためにデプロイしてテストするアプリケーションを見てください。

シンプルなWallet-demoアプリを考えてみましょう。誰もがトークンをdAppアドレスに送信できます。あなたはあなたのWAVESだけを引き出すことができます。InvokeScriptTransactionを介して2つの@Callable関数を使用できます。

  • deposit()WAVESで添付の支払いが必要です
  • withdraw(amount: Int)トークンを返す

dAppライフサイクル全体を通じて、構造（アドレス→金額）がサポートされます。

状況を完全に理解するためのコードは次のとおりです。 

# In this example multiple accounts can deposit their funds and safely take them back. No one can interfere with this.
# An inner state is maintained as mapping `address=>waves`.
{-# STDLIB_VERSION 3 #-}
{-# CONTENT_TYPE DAPP #-}
{-# SCRIPT_TYPE ACCOUNT #-}
@Callable(i)
func deposit() = {
 let pmt = extract(i.payment)
 if (isDefined(pmt.assetId))
    then throw("works with waves only")
    else {
     let currentKey = toBase58String(i.caller.bytes)
     let currentAmount = match getInteger(this, currentKey) {
       case a:Int => a
       case _ => 0
     }
     let newAmount = currentAmount + pmt.amount
     WriteSet([DataEntry(currentKey, newAmount)]) 
   }
 }
@Callable(i)
func withdraw(amount: Int) = {
 let currentKey = toBase58String(i.caller.bytes)
 let currentAmount = match getInteger(this, currentKey) {
   case a:Int => a
   case _ => 0
 }
 let newAmount = currentAmount - amount
 if (amount < 0)
   then throw("Can't withdraw negative amount")
   else if (newAmount < 0)
     then throw("Not enough balance")
     else ScriptResult(
       WriteSet([DataEntry(currentKey, newAmount)]),
       TransferSet([ScriptTransfer(i.caller, amount, unit)])
      )
 }
@Verifier(tx)
func verify() = false

「wallet.ride」のテストケース


テストファイルを 見てみましょう。JavaScriptのMochaフレームワークを搭載。「前」関数と3つのテストがあります。

  • 「以前」は、MassTransferTransactionを介していくつかのアカウントに資金を提供し、スクリプトをコンパイルしてブロックチェーンに展開します。
  • 「預金可能」は、InvokeScriptTransactionをネットワークに送信し、2つのアカウントのそれぞれに対してdeposit（）関数をアクティブにします。
  • 「預けられた以上の引き出しはできない」というテストは、他人のトークンを盗むことはできないことを示しています。
  • デポジットにより、引き出しが正しく処理されていることを確認できます。
