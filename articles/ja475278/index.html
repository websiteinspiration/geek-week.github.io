このような「役割」の配分は、5年前にはよい考えでした。しかし、そのとき私には多くのことが良い考えに思えました。

これらのクラスで違反者を分類するために、CVE / CVSS ConfImpact（「機密性への影響」）、IntegImpact（「アクセシビリティへの影響」）、AvailImpact（「整合性への影響」）から取得します。

ベクトルでは、影響は次の値を取ります。リソースへの影響なし（N）、リソースへの部分的な影響（P）、リソースに対する完全な制御（C）。
クラス 機密性への影響 可用性への影響 整合性への影響
スカウト N P / c N
駆逐艦 P / c N / p N
インベーダー N / P / C N / P / C P / c

結論として、クラスはシステムに異なる影響を与えることを示します。少なくとも、すべてが「スカウト」であり、何よりも「インベーダー」です。
したがって、SIEMから一連の攻撃を取得すると、侵入者を分類できます。そしてこれについては、違反者の目標についていくつかの結論を出すことができます。
一連の攻撃を通過すると、侵入者はスキルとクラスを増やすことはできますが、減らすことはできません。低→中→高とスカウト→駆逐艦→侵略者。

分類は、侵入者の意図を間接的に予測する際に使用されます。
しかし、違反者を分類すると、彼らが何を求めているのかを推測することができます。
おそらく、分類は卒業証書の中で最もよく機能する場所であると言えるでしょう。

SIEMデータベースで攻撃が格納されている場所を見つけ、この情報をグラフデータベースに転送するパーサーを作成する必要がありました。

ベースの塗りつぶし方法：SIEMが攻撃を検出してCVEを取得し、ベースに新しい頂点を追加するか、すでにそのような頂点がある場合は、頂点間の遷移の数を1単位増やします。攻撃が続く場合は、頂点/遷移を追加します。

このアプローチには、次のような利点があります。

  • 自己記入、つまり 攻撃者自身が行うため、新しい攻撃グラフを作成するために追加の手順は必要ありません。
  • 冗長性が低い、つまり グラフには、攻撃者が使用する攻撃グラフのみが含まれています。

このアプローチの主な欠点は、攻撃者が以前は使用されていなかった攻撃を使用した場合、攻撃者の行動を予測することができないことです。

この方法は、他の方法でモデル化された攻撃グラフをデータベースに追加し、遷移を「0」に設定することで改善できます。
