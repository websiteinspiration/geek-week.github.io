最新のDコンパイラ

LDC開発チームは昨年、LLVMを使用するバージョン1.19.0のDコンパイラで終了し、コアチームは、DMDリファレンスコンパイラのバージョン2.090.0で新年を迎えました。そして、あなたが聞いたことがないなら、それほど前にGCCに基づいたDコンパイラ、GDCに関する重要なニュースが現れました。化粧！

LDC 1.19.0


このリリースには、カトリッククリスマス直後のリリース時点での最新バージョンであるフロントエンド2.089.1のバージョンDへのLDCコンパイラのアップデートが含まれています。アセンブリされたアセンブリはLLVM 9.01に基づいています。

このリリースのハイライトには、Androidに対する同情があります。既製のDRuntime / PhobosライブラリがすべてのAndroidターゲットで利用可能になりました。このリリースは、支援プログラムとインターフェースの集まりであるAdam RuppeのAndroid Project Dと組み合わせて使用​​でき、LDCを使用したAndroid Dでの開発用に、現在は完成間近のベータ版です。

Windowsユーザーは、提供されているMinGWベースのリンクライブラリが更新されていることに気付くでしょう。現在、これらはパッケージの.defファイルから派生していますMinGW-w64 7.0.0。これらのライブラリを使用すると、Windows SDKをインストールしなくても、Windowsシステムライブラリを使用できます。

DMD 2.090.0


DMDの最新バージョンが1月7日に発表されました。彼女は10の大きな変更と71のバグをクローズし、48人の共著者からの好意で提供されました。

このリリースでは、遅延パラメーターでさらに多くのことが可能になりました。Dは長い間レイジーオプションをサポートしてきました：
lazyパラメーターの引数は、関数が呼び出されるまで評価されません。引数は、/が関数内で評価されるときにのみ処理されます。
したがって、遅延引数は0回以上実行できます。
内部的には、デリゲートとして実装されています。これで、以前は禁止されていたパラメーターアドレスを取得することで、ベースデリゲートに到達できます。

import std.stdio;

void chillax(lazy int x)
{
    auto dg = &x;
    assert(dg() == 10);
    writeln(x);
}

void main()
{
    chillax(2 * 5);
}

また、このリリースでは、ファイナライズ（通常はガベージコレクターによって開始されるオブジェクトの非決定的破棄）とクラスまたは構造デストラクタの内部からの通常の破棄（オブジェクトの確定的破棄）を区別する必要がある人が使用していたイディオムDが廃止されます。

GCの現在の状態では、ファイナライズ中のすべてのGC操作は禁止されています。ただし、Dは個別のファイナライザとデストラクタを提供していません。これだけある〜、両方の役割を果たすという事実にもかかわらず、これはデストラクタと呼ばれます。これにより、GCと非GCの両方のアロケーター用に設計された型のデストラクタを実装することが困難になる場合があります。クリーンアップ中にGCを呼び出すと、InvalidMemoryOperationErrorエラーが発生する可能性があります。これは、前述の回避策が必要であることを意味します。

これで、静的GC関数であるcore.memory.GC.inFinalizerを呼び出して、デストラクタで状態を取得できます。現在のスレッドがファイナライズ中の場合はtrueを返します。その場合、GCでの操作を含むアクションを実行する必要はありません。（私は私のGCシリーズの次の記事を書く前にこのような何かを期待していました）。

GDC


Jena Bucklau、Johannes Pfau、そして長年にわたってサポートし貢献してくれたすべてのボランティアの努力のおかげで、GDC は 2018年末にGCC 9に承認され、昨年5月にリリースされたGCC 9.1キットの一部として利用可能になりました。GCC 9.2は昨年8月にリリースされました。このバージョンのGDCは、フロントエンドDのバージョン2.076を実装しています。自分でビルドするか、通常GCC 9.xを入手するのと同じ場所からインストールできます。
