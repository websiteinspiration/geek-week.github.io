.NET Core 3.1がリリースされました

.NET Core 3.1のリリースをお知らせいたします。これは本当に小さなアップデートで、主に数か月前に公開した.NET Core 3.0のバージョンの問題を修正することを目的としてい  ます。.NET Core 3.1で最も重要なことは、長期サポートリリース  （LTS）で  あり、このバージョンは少なくとも3年間サポートされます。過去と同様に、LTSの新しいリリースを急いでリリースしたくありませんでした。さらに2か月（.NET Core 3.0のリリース後）で、既に安定したバージョンと比較して、改善の正しいセットを選択して実装することができました。.NET Core 3.1は、想像力を駆使できるあらゆる分野で使用できるようになりました。

あなたはできる  .NETのコア3.1をダウンロード Windowsの、MacOSの、およびLinux用：

  • .NET Core 3.1 SDKおよびランタイム
  • Dockerコンテナイメージ
  • スナップインストーラー

ASP.NET Core  と  EF Core  も公開されました。

Visual Studio 2019 16.4  も利用可能になり、.NET Core 3.1が含まれています。これは、Visual Studioで.NET Core 3.1を使用するために必要なアップデートです。Visual Studio 2019ユーザーの場合、.NET Core 3.1を個別にダウンロードする代わりに、Visual Studioを16.4にアップグレードすることをお勧めします。

Visual Studio for Macは.NET Core 3.1もサポートおよびインクルードします。VisualStudio for Mac 8.4プレビューをお試しください。

リリースノート：

  • .NET Core 3.1リリースノート
  • .NET Core 3.1の問題を説明するGitHubページ
  • GitHubリリース



.NET Core 3.1での変更は、主に.NET Core 3.0の2つの大きな新リリースであるBlazor  と  Windows Desktopに焦点を当てました  。これらには、Windows指向の開発者から頻繁に要求されたC ++ / CLIサポートが含まれています。

新しい.NET Core 3.1製品の概要に進む前に、新しい.NET Core 3.0の機能を確認することをお勧めします  。

プラットフォームサポート


.NET Core 3.1は、次のOSでサポートされています。

  • アルパイン：3.10以上
  • Debian：9以上
  • Ubuntu：16.04以降
  • Fedora：29以上
  • RHEL：6以上
  • openSUSE：15以上
  • SUSE Enterprise Linux（SLES）：12 SP2以降
  • macOS：10.13以降
  • Windowsクライアント：7、8.1、10（1607以降）
  • Windowsサーバー：2012 R2以降

注：WindowsフォームおよびWPFアプリケーションがサポートされ、Windowsでのみ実行されます。

チップサポート：

  • Windows、macOS、およびLinux上のx64
  • Windows上のx86
  • WindowsおよびLinux上のARM32
  • Linux上のARM64（カーネル4.14以降）

注：.NET Core 3.1 ARM64デプロイメントはLinuxカーネルバージョン4.14以降を使用することに注意してください。たとえば、Ubuntu 18.04はこの要件を満たしていますが、16.04は満たしていません。

Windowsフォームコントロールの削除


次のWindowsフォームコントロールが.NET Core 3.1から削除されました。

  • DataGrid
  • ツールバー
  • コンテキストメニュー
  • メニュー
  • メインメニュー
  • メニュー項目

これらのコントロールは、2005年に.NET Framework 2.0でより強力なコントロールに置き換えられました。それらはVisual Studio Designer Toolboxでデフォルトで長年使用できません。その結果、これらのコントロールを削除し、新しいコントロールに焦点を合わせることにしました。

以下の代替品をお勧めします。
古い要素（API） 推奨代替品 その他の関連するリモートAPI
DataGrid DataGridView DataGridCell, DataGridRow, DataGridTableCollection, DataGridColumnCollection, DataGridTableStyle, DataGridColumnStyle, DataGridLineStyle, DataGridParentRowsLabel, DataGridParentRowsLabelStyle, DataGridBoolColumn, DataGridTextBox, GridColumnStylesCollection, GridTableStylesCollection, HitTestType
ToolBar ToolStrip ToolBarAppearance
ToolBarButton ToolStripButton ToolBarButtonClickEventArgs, ToolBarButtonClickEventHandler, ToolBarButtonStyle, ToolBarTextAlign
ContextMenu ContextMenuStrip
Menu ToolStripDropDown, ToolstripDropDownMenu MenuItemCollection
MainMenu MenuStrip
MenuItem ToolstripMenuItem
はい、これは転機です。アプリケーションで削除したコントロールを使用すると、アセンブリが壊れることに気づくでしょう。また、.NET Core 3.0アプリケーションを最新バージョンの.NET Core Windowsフォームデザイナで開いた場合、これらのコントロールを使用するとエラーが発生します。

アプリケーションを.NET Core 3.1にアップグレードし、別のコントロールに切り替えることをお勧めします。コントロールの置換は単純なプロセスであり、基本的に「検索と置換」です。

まず、.NET Core 3.0をリリースする前に、これらの変更を行う必要がありました。変更が遅すぎるのを避けようとしていますが、これが起こったことを非常に残念に思っています。

Windowsフォームデザイナーのプロジェクトの実装をさらに進めると、これらのコントロールは最新のアプリケーションの作成とは関係がなく、.NET Core Windowsフォームの一部であってはならないことに気付きました。また、それらをサポートするには多くのリソースが必要になることにも気づきました。

私たちの目標は、高いDPI、可用性、および信頼性を提供するためにWindowsフォームを引き続き改善することであり、この遅い変更は非常に重要でした。

C ++ / CLI


Visual Studio 2019 16.4の.NET Core 3.0+で使用できるC ++ / CLIコンポーネント（別名「マネージC ++」）の作成のサポートを追加しました。C ++ / CLIを使用するには、「C ++を使用したデスクトップ開発」ワークロードと「C ++ / CLIサポート」コンポーネントをインストールする必要があります。

このコンポーネントは、使用できるいくつかのパターンを追加します。

  • CLRクラスライブラリ（.NET Core）
  • CLR空のプロジェクト（.NET Core）

見つからない場合は、[新しいプロジェクト]ダイアログボックスを検索してください。

C ++ / CLIはWindowsでのみ使用できます。.NET Framework向けに設計されたC ++ / CLIコンポーネントを.NET Coreとともに使用することはできません。

結論として


できるだけ早く.NET Core 3.1に切り替えることをお勧めします。これは、（主に3.0による）優れたリリースであり、.NET Coreの多くの側面に改善をもたらします。これは、少なくとも3年間サポートされる長期サポートリリース（LTS）でもあります。

ライフサイクルの更新：

  • .NET Core 3.0のサポートは2020年3月3日に終了します。
  • .NET Core 2.2のサポートは2019年12月23日に終了します。
  • .NET Core 2.1は2021年8月までサポートされます（これもLTSリリースです）。
