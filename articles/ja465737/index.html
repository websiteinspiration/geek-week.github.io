スンダラマふるいは、与えられた範囲の自然数3≤n≤Nですべての素数を見つけ、成分を「ふるいにかける」。一般性を失うことなく、Nの値は奇数と見なすことができます。 Nが偶数の場合、Nは複合であることが保証され、上限の値を1つ減らすことにより、ふるいの範囲から除外できます。

アルゴリズムは次の事実に基づいています。奇数の複合数nは、1より大きい2つの自然な奇数の積として表すことができます：

（1）n =（2 * i + 1）*（2 * j + 1）、

奇数nを2 * m + 1の形式で記述し、（1）に代入して、mの式を取得します：

（2）m = 2 * i * j + i + j。

この変換は、スンダラムのふるいのアイデアに直接つながります。

指定された間隔から複合数をフィルターで除外するには、インデックスmの要素が数値2 * m + 1を表す配列を使用する必要があります。変数iとjの値の列挙を整理したら、式（2）と対応する式でインデックスmを計算します配列要素はマーク「複合番号」を設定します。列挙が完了すると、範囲内のすべての合成数にマークが付けられ、マークがない場合は素数を選択できます。

以下は、説明されたメソッドを実装する非常に単純なC＃Sundaramクラスのリストです。 

using System;
using System.Collections;

namespace CSh_Sundaram
{
    public class Sundaram
    {
        public double dt;          //   ()
        private long t1;           //  
        private long t2;           //  
        private uint limit;        //    
        private int arrlength;     //  
        private BitArray Prim;     //   
        private int counter;

        public Sundaram(uint _limit)
        {
            limit = _limit;
            if (limit % 2 == 0) limit -= 1;
            arrlength = (int)(limit / 2);
            Prim = new BitArray(arrlength);
            t1 = DateTime.Now.Ticks;
            Sieve();                 // 
            t2 = DateTime.Now.Ticks;
            dt = (double)(t2 - t1) / 10000000D;
            counter = -1;
        }

        public uint NextPrime
        {
            get {
                while (counter < arrlength - 1) {
                    counter++;
                    if (!Prim[counter])
                        return (uint)(2 * counter + 3);
                }
                return 0;
            }
        }

        private void Sieve() {
            int imax = ((int)Math.Sqrt(limit) - 1) / 2;
            for (int i = 1; i <= imax; i++) {
                int jmax = (arrlength - i) / (2 * i + 1);
                for (int j = i; j <= jmax; j++) {
                    Prim[2 * i * j + i + j - 1] = true;
                }
            }
        }
    }
}

Sundaram型のオブジェクトを作成する際のパラメータとして、ふるい分け範囲の上限を示します。ふるいにかけるために、クラスはBitArray型の配列を使用します。これにより、実行時間が長くなりますが、uint型の32ビット範囲全体をカバーできます。クラスコンストラクターからSieve（）メソッドにアクセスすると、ふるい分けが実行されます。完了後、dtフィールドにはSieve実行時間が秒単位で含まれます。

NextPrimeプロパティにアクセスすると、値3から順番に昇順で素数が返されます。範囲内のすべての単純なものが使い果たされると、値0が返されます。
