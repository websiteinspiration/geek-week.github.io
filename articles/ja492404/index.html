Flutter + Arduino nano 33 BLEセンス=非常にシンプルなBLEセンサー

この記事では、非常にシンプルなbluetooth気象ステーション（なしの場合:)）を作成し、それをFlutterでモバイルアプリケーションを作成する方法を説明します。



最初に、センサー自体を検討してください


繰り返しますが、Arduino nano 33 BLEセンスボードが必要です。

ボードはnrf52840上に構築されています。arduinoのボードマネージャーからインストールします。



必要なライブラリをすぐにインストールします。

  • Arduino ble
  • Arduino HTS221
  • Arduino LPS22HB

これらのライブラリは、ボード自体にすでにはんだ付けされているセンサーに必要です。

理論を少し説明してから、実際の実装を検討します


主なアイデアは、プラグインデバイスを作成することではなく、必要なすべての情報が含まれたブロードキャストメッセージを実装することでした。

通常のBluetoothタグモードが使用されましたが、ManufacturerDataが変更されました。このパッケージは、すべてのアドバタイズパッケージで送信できます。

アドバタイズパッケージの合計サイズは31バイトです。これには、必要なすべての情報（デバイス名、システムデータ、ユーザーデータ）が含まれます。純粋な形式では、ManufacturerDataのユーザーは約20バイトです。これは気象観測所のデータを送信するのに十分です。
低消費電力でデータを送信するこの方法の利点は、デバイスとの常時接続を維持する必要がないため、ブロードキャストされます。このようなメッセージは、受信範囲内の無制限の数の受信者をキャッチできます。
ManufacturerDataは、広告の開始前にインストールされます。

そして今実用的な部分


arduinoコードでは、BLEパーツの作業のタイプを示し、開始ManufacturerDataを設定します。
また、便宜上、デバイスの名前を指定しているので、アプリケーションで見つけやすくなっています。

BLE.setLocalName("nrf52840.ru");
BLE.setConnectable(false);
byte data[8] = { 0x01, 0x02, 0x03, 0x04, 0x05, 0x06, 0x07, 0x08};
BLE.setManufacturerData(data, 8);
// start advertising
BLE.advertise();

初期データは、配列を満たすためのバイトのセットとして提供されます。

次に、メインコードで広告を停止し、ラジオを停止し、必要なデータを測定し、ManufacturerDataに実際のデータを入力してから、ブロードキャストを開始します。この操作は2秒ごとに実行されます。

BLE.stopAdvertise();
// read all the sensor values
---------------------
       
---------------------
byte data[8] = { 0x00, 0x01, t1, t2, h1, h2, p1, p2}; // t -  (2 ), h -  (2 ), p -  (2 )
BLE.setManufacturerData(data, 8);
BLE.advertise();
// wait 2 second to print again
delay(2000);

これでセンサーでの作業は完了です。センサーは100ミリ秒ごとにブロードキャストし、2秒ごとにデータを現在のものに更新します。その結果、非常にシンプルな天気コードと実装センサーができました。

次に、モバイルアプリケーションを検討します。


私はすぐに予約します。私はモバイルアプリケーションの開発者ではありません。

仕事では、FlutterプラグインでVSCodeを使用しました。この環境は、思ったようにAndroid Studioよりもシンプルに見えます。BLEを使用するために、デバイスの接続を大幅に簡略化するFlutter_blueライブラリが使用されました。

アプリケーションのロジックも非常に単純です。当社のセンサーは通常のビーコンモードでブロードキャストするため、次の2つのアクションを実行するだけで済みます。

  1. ブロードキャストをスキャン-指定された名前のデバイスを検索し、
  2. ManufacturerDataを解析して、データを画面に表示します。

これがどのように行われるか見てみましょう。

アプリケーションの起動後、タイマーが定期的に起動され、Bluetoothデバイスを10秒ごとに2秒間スキャンします。より頻繁に、そしてより長くそれはスキャンすることは意味をなさず、バッテリー消費は増加し、そしてセンサーは一般的に100ms毎にブロードキャストします。

DeviceScanner() {
   _subscribeToScanEvents();
   _timer = new Timer.periodic(const Duration(seconds: 10), startScan);
 }

void startScan(Timer timer) {
   FlutterBlue.instance.startScan(timeout: Duration(seconds: 2));
}

次に、スキャン結果の分析を開始し、指定した名前のデバイスがスキャンリストに存在するかどうかを確認します。存在する場合は、そのパッケージを解析して結果をユーザーに表示します。


  void _subscribeToScanEvents() {
    FlutterBlue.instance.scanResults.listen((scanResults) {
      for (ScanResult scanResult in scanResults) {
        if (scanResult.device.name.toString() == "nrf52840.ru") {
          final int rssi = scanResult.rssi;
          final String name = scanResult.device.name;
          final String mac = scanResult.device.id.toString();
          final double temp = scanResult.advertisementData.manufacturerData[256]
                  [0] +
              scanResult.advertisementData.manufacturerData[256][1] * 0.01;
          final double humm = scanResult.advertisementData.manufacturerData[256]
                  [2] +
              scanResult.advertisementData.manufacturerData[256][3] * 0.01;
          final double press =
              scanResult.advertisementData.manufacturerData[256][4] +
                  scanResult.advertisementData.manufacturerData[256][5] * 0.01;
          final SensorData sensorData = new SensorData(
              name: name,
              rssi: rssi,
              mac: mac,
              temperature: temp,
              humidity: humm,
              pressure: press);
          _streamController.add(sensorData);
          print(
              'Manufacturer data ${scanResult.advertisementData.manufacturerData}');
          FlutterBlue.instance.stopScan();
        }

        print(
            '${scanResult.device.name} found! mac: ${scanResult.device.id} rssi: ${scanResult.rssi}');
      }
    });
  }

少しニュアンス。FlutterのBluetoothライブラリは奇妙に思えるかもしれません。データはint配列の形式で受信され、Arduinoではバイトからパッケージを形成するため、センサーにパッケージを作成して分析を簡素化しました。25.85度の温度は、25と85の2つの値に分割され、別々のバイト値として送信され、同じ方法で収集されます。

最終結果はそのようなアプリケーションです。
