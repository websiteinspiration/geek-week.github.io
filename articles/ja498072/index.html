Atari 65XEのラスター効果


2020年1月に、フォーエバー2020デモパーティに参加する準備を始めました。コンボテーマはROBOから発表されたので、Atari 65XEの簡単な作品を書くことにしました。イントロの準備ができたら、ビットマップ効果を追加することにしました。残念ながら、フォーエバーのキャンセルにより、イントロの一部が公開されました。

AtariプロセッサはMOS 6502-8ビットレジスタであり、ハードウェアにアクセスするためのポートがありません。コンピュータの別のコンポーネントはANTICです。これは画像-色、ビデオモード、シンボルを実装します。ANTICにアクセスするためのハードウェアセルがあり、その目的はマッピングで説明されていますAtari

セルのすべてのアドレスと宛先をリストするのではなく、効果自体のアイデアについて説明します。便宜上、マッピングのアドレスにはニーモニックが付いていることに注意してください。覚えておくと便利です。


画面とは？テキストモードでは、シンボルとフレームのある長方形が表示されます。

便宜上、0〜559の値を書き込むことで長方形をオフにできます。ご覧のように、ラスター効果の余地があります。

次に、図面を画面への出力と同期させる必要があります。2つの方法があります。

RTCLOK      equ $0012
      lda RTCLOK+2
waits
      cmp RTCLOK+2
       beq waits

ここで、RTCLOCKは、コンピューターの電源がオンになったときに開始するリアルタイムクロックです。最初に$ 14、次に$ 13と$ 12を増やします。同期の考え方は簡単です：セル$ 14の値を現在の値と比較します。以前の値が一致しない場合、割り込みが発生しています。

しかし、私の効果では、メソッドは適合しないため、フレームで生成されたラインの垂直カウンターであるVCOUNT（$ D40B）を使用する方が便利です。フレームの開始を待つのは：

ws
	lda $D40B ; VCOUNT
	bne ws

アイデア自体は、COLBKレジスタ（$ D01A）-フレームの色に色の値を書き込むことです。
レジスタに書き込まれる値は、タイプ* 16 +輝度として計算される、カラータイプと輝度の2つの部分に分かれています。こことここで詳しく説明されています。

ただし、各ラインの色を設定する場合、同期が必要です。WSYNC（D40A）に値を書き込むと、プロセッサが停止し、次のラインが開始するまで待機します。同じ方法がDisplay List割り込みでも使用されます。

エフェクト自体は、フレームの待機、カラー値の生成、および説明されたレジスタへの書き込みに分けられます。簡単にするために、ゼロページアドレスを使用しました。 xasmアセンブラを使用してコンパイルできます。



	org $4000
	
start

wait_frame
ws
	lda $D40B ; VCOUNT
	bne ws
       LDY #0
       STY 559
zz
	LDA $CD
	CLC
	ADC #5
	STA $CD
	BCC BR1
	INC $CE
BR1
	ADC $CB
	STA $CB
	BCC BR2
	INC $CC
BR2
	LDA $CC
	STA $D40A ; WSYNC
	STA $D01A
	INY
	BNE zz
	BEQ wait_frame
	run start

それは54バイトでした。54-6（プログラムヘッダー）= 48バイト。緑の明るさを試すなど、実験する時間があります

	lda $CC
;*  
	and #$0f
	ORA #$C0
;*  
	sta $D40A ; wsync
	sta $D01A
