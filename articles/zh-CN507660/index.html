通过UptimeRobot进行监控

任何站点都需要监视，没有任何资源所有者不希望停机。有趣的是，许多开发人员没有监视他们的宠物项目，而且自由职业者几乎从不理会可访问性检查，尽管这通常比连接某些分析要困难得多，并且可以省去很多麻烦。



用于监视站点的服务有十多种，其中大多数是完全付费的或免费版本已大大减少。除了它们之外，还有UptimeRobot，它免费提供了巨大的功能，并允许您根据需要灵活配置付费订阅。



在大多数情况下，UptimeRobot将HTTP请求发送到指定的地址并通过响应标头确定可用性（2xx-一切正常，4xx / 5xx-它从其他点触发重复的请求持续30秒，如果答案再次为否，则计入停机时间）。

API



UptimeRobot以XML / JSON / JSON-P提供了一个简单的API，该API完全复制了Web版本的功能。简而言之，该方法和参数的详细说明位于API页面上，您可以阅读统计信息和日志，创建/删除/编辑/重置监视器以及管理用户。



有一个有趣的指南，使用UptimeRobot，Go，Python 上的客户端以及对其他语言，面板和自定义状态页面的大量解释，在Kubernetes上实现了正常运行时间监控。



心跳



要监视不能直接访问的服务器，可以在付费版本中使用心跳监视器。创建此链接时，将生成一个唯一链接，UptimeRobot服务将在该链接上侦听来自远程服务器的定期传入请求。如果请求停止到达，则计入停机时间。
