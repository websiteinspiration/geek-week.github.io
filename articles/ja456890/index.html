stm32のIRリモートコントロール



こんにちは。

stm32マイクロコントローラーを使用して、さまざまな家庭用リモコンから赤外線信号を読み取り、デコードして送信するためのライブラリーの説明。arduinoのIRremote ライブラリに基づいており、stm32に適合しています。

ライブラリは、信号の受信と送信の両方に1つのタイマーを使用します。レシーバーは任意のピン（GPIO_Input）に接続され、トランスミッターはPWM生成（PWM）モードで動作するタイマーのチャネルの1つに接続されます。この例では、タイマーNo. 4の最初のチャネル、PB6（送信機）とピンPB5（受信機）が使用されています。

タイマーは信号を受信するために割り込みモードで動作します-50μsごとに入力ピンのステータスをチェックし、送信中にPWM生成モードに切り替えて信号を送信し、割り込みモードに戻ります。

設定はIRremote.hファイルにあります

extern TIM_HandleTypeDef htim4;

//     -   50  (     72)
#define MYPRESCALER 71 //   1
#define MYPERIOD 49    // 50 

//     -    
#define MYSYSCLOCK 72000000

//     recive_IR
#define RECIV_PIN (HAL_GPIO_ReadPin(recive_IR_GPIO_Port, recive_IR_Pin))

別のタイマーを構成する場合は、対応する構造名-htim4を指定する必要があり、IRremote.およびirSend.ファイルでも同じことを行います。私はこの経済すべてを定義するのが面倒だった。キューバで別のタイマーを選択するときは、チャネルと内部クロックソースのみを指定する必要があります



。残りはプログラムによって設定されます。別のチャネル番号を選択した場合は、irSend.sファイルでもその名前を変更する必要があります。

残りの設定では、すべてが明確であると思います。システム周波数（72 MHzの例）に基づいて、プリスプリッターとオーバーフローの値は、50μsごとの割り込みに置き換えられます。ピンの周波数と読み取り値を以下に示します。

以下は、関連するプロトコルを定義する定義です...

////////////////////////////////////   ////////////////////////////////////////
#define DECODE_RC5           1 //     RC5   0
#define SEND_RC5             1 //       RC5   0

#define DECODE_RC6           1
#define SEND_RC6             1
...

不要なプロトコルを無効にすると、プログラムのサイズが小さくなります。原則として、未使用のプロトコルと対応するファイル（ファイルには固有の名前が付いています）の機能を通常は切り取ることができます。

プログラムは非常にシンプルです。my_decode（＆results）関数は受信した信号をデコードし、ボタンコード、プロトコルタイプ、パケット長を表示します...



他にリモートはありません。

デコードされた信号を送信するには、対応する名前の関数を使用します...

sendSAMSUNG(0x707048b7, 32); 
my_enableIRIn();

my_enableIRIn（）関数が必要です。これはPWMを無効にし、タイマーを受信モードにします。同じ関数が初期化にも使用されます（無限ループの前）。この機能のために、あなたはあなた自身のシグナルを捕らえることができません-これは解決することができます、しかし私はこれに意味がないと思います。

不明な信号を受信するだけでなく、送信する必要がある場合は、「生データ出力」の行のコメントを外して

ライブラリは、マイクロ秒の遅延にDWTカウンターを使用します。私の知る限り、すべてのstm32がそれを持っているわけではなく、どこにでも同じように構成されているとは限りません。ストーンにDWTがない場合は、irSend.sファイルの最後にあるcustom_delay_usec（unsigned long us）関数で置き換えるものを考え出す必要があります。設定は最初です。
