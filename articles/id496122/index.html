Merawat pekerjaan rumah Anda: cadangkan laptop atau desktop Anda menggunakan Agen Veeam gratis

Dalam kehidupan sehari-hari, saya, seperti banyak dari Anda, pembaca yang budiman, bekerja di kantor hampir sepanjang waktu, dan itu sangat cocok untuk saya. Tetapi hari ini situasinya sedemikian rupa sehingga saya dan kolega saya di berbagai belahan dunia, dan, saya pikir, mayoritas orang membaca artikel ini, kita semua bekerja dari rumah.

Dan di sini akan sangat tidak pada tempatnya mengingat perlunya membuat cadangan komputer rumah atau laptop. Karena ketika lingkungan kerja berubah, cara kerja biasa dengan data juga berubah. Semuanya jelas dengan pekerjaan di kantor: akses langsung ke sumber daya yang diperlukan, backup terpusat, jika ada, Anda selalu dapat menghubungi helpdesk. Dengan bekerja dari rumah, banyak hal yang perlu Anda sediakan untuk diri sendiri, termasuk cadangan.

Saya percaya bahwa banyak dari Anda sudah menggunakan Veeam Agent untuk Microsoft Windows atau Veeam Agent untuk Linux untuk ini. Jika belum, maka mungkin Anda akan tertarik dengan ulasan singkat hari ini. Jadi selamat datang di kucing.



Pada 2014, ketika versi pertama produk kami, yang sekarang dikenal sebagai Veeam Agent untuk Microsoft Windows , dirilis , itu disebut Veeam Endpoint Backup Free dan memiliki satu-satunya edisi non-komersial.

Tentu saja, solusinya telah banyak berubah sejak saat itu, dan versi komersial telah keluar, di mana ada banyak fitur canggih - manajemen terpusat, driver CBT, dukungan cluster, pemrosesan disk paralel, dan banyak lagi. Veeam Agent untuk Linux juga berkembang, yang memungkinkan Anda untuk membuat cadangan mesin fisik dan mengembalikan data dari cadangan. Tetapi sekarang, ketika berada di halaman 2020, versi baru Veam Agent untuk Microsoft Windows dan Veeam Agent untuk Linuxada juga edisi gratis yang dirancang untuk komputer individu.

Jadi, jika Anda perlu membuat cadangan desktop rumah, laptop, atau bahkan tablet yang berjalan pada Windows, gunakan kesehatan Anda.

Veeam Agent untuk Microsoft Windows: OS yang Didukung


Versi OS 64-bit dan 32-bit (jika berlaku) didukung:

  • Microsoft Windows Server 2019
  • Microsoft Windows Server 2016
  • Saluran Semi-Tahunan Microsoft Windows Server (termasuk 1909)
  • Microsoft Windows Server 2012 R2
  • Microsoft Windows Server 2012
  • Microsoft Windows Server 2008 R2 SP1
  • Microsoft Windows 10 (dari versi 1607 hingga 1909)
  • Microsoft Windows 8.1
  • Microsoft Windows 7 SP1

Catatan:

  • Untuk sistem operasi server, edisi Small Business Server dan Server Essentials didukung.
  • Versi Server Core untuk sistem server dalam skenario penyebaran yang berdiri sendiri tidak didukung.
  • Didukung Windows Embedded / Windows IoT OS (kecuali build khusus).
  • Microsoft Failover Clusters didukung ketika Veeam Agent untuk Microsoft Windows menjalankan Veeam Backup & Replication (tidak didukung dalam skenario penyebaran yang berdiri sendiri).
  • Microsoft Windows 10 Education didukung dimulai dengan build 10586.

Persyaratan sistem yang tersisa dijelaskan secara rinci dalam dokumentasi (dalam bahasa Inggris).

Cadangkan pekerjaan di Veeam Agent untuk Microsoft Windows


Untuk cadangan, konfigurasikan tugas cadangan dengan memulai panduan Pekerjaan Pencadangan Baru . Di dalamnya, pilih jenis cadangan:

  • Seluruh komputer - seluruh mesin.
  • Pencadangan tingkat volume - pencadangan volume yang dipilih.
  • Cadangan level file - cadangan file tertentu.

Veeam Agent untuk Microsoft Windows dapat menggunakan berbagai jenis sistem penyimpanan sebagai lokasi penyimpanan cadangan:

  • Penyimpanan lokal - drive eksternal (USB, Firewire atau eSata). Hal utama di sini, seperti yang Anda tahu, adalah tidak membuat cadangan diri Anda ke disk bawaan pada mesin.
  • Folder bersama - Bagikan SMB pada server file atau NAS.
  • Veeam backup repository - Veeam repository (lokal).
  • Repositori Veeam Cloud Connect adalah repositori berbasis cloud yang disediakan oleh penyedia layanan Veeam.
  • Microsoft OneDrive

Kemudian kami mengatur pengaturan khusus untuk jenis penyimpanan yang dipilih.

Penting! Saat bekerja dengan drive USB yang dapat dilepas, Anda harus ingat bahwa jika drive tersebut diformat untuk FAT32, maka drive tersebut tidak mendukung penyimpanan file yang lebih besar dari 4 GB. Jadi tidak disarankan menggunakan disk seperti target penyimpanan cadangan.

Selanjutnya, konfigurasikan jadwal pada langkah Jadwal . Di sini Anda dapat menentukan apakah akan membuat cadangan secara teratur pada waktu-waktu tertentu atau melakukannya ketika peristiwa yang dipilih terjadi:

  • Kunci - saat Anda mengunci mobil
  • Log off - ketika log out untuk rekening giro
  • Ketika target cadangan terhubung - ketika penyimpanan target cadangan terhubung

Jika opsi ketiga ditentukan, Anda juga dapat menginstruksikan Veeam Agent untuk menyelesaikan pemutusan (Eject) penyimpanan target di akhir cadangan sehingga cadangan offline. Ini akan mengurangi risiko terpapar salinan cadangan virus enkripsi, jika terdeteksi.

Veeam Agent untuk Linux


Veeam Agent untuk Linux dirancang untuk melindungi data mesin yang berjalan di Linux, dan solusi ini juga memiliki edisi gratis. Kernel Linux 2.6.32 dan lebih tinggi didukung.

Versi distribusi yang didukung adalah 64-bit dan 32-bit:

  • Debian 8.0 - 10.3
  • Ubuntu 14.04, 16.04, 18.04, 19.10
  • RHEL 6.0 - 8.1
  • CentOS 6.0 - 8.1
  • Oracle Linux 6 - 8.1 (RHCK)
  • Oracle Linux 6 (dimulai dengan UEK R1) - Oracle Linux 7 (hingga UEK R5 U2)
  • SLES 11 SP4, 12 SP1 - 15 SP1
  • SLES untuk SAP 11 SP4, 12 SP1 - 15 SP1
  • Fedora 30 - 31
  • openSUSE Leap 15.1
  • openSUSE Tumbleweed

Catatan:

  • Fedora didukung hingga dan termasuk kernel versi 5.4.14.
  • openSUSE Tumbleweed didukung hingga dan termasuk kernel versi 5.1.7.
  • Kernel Linux 2.6.32-754.6.3 tidak didukung pada CentOS / RHEL dan Oracle Linux (RHCK).

Untuk informasi lebih lanjut, lihat dokumentasi (dalam bahasa Inggris).

Jenis cadangan yang sama didukung seperti di Veeam Agent untuk Microsoft Windows, termasuk cadangan tingkat file. Pengaturan cadangan dapat diatur menggunakan langkah-langkah di wisaya atau dari baris perintah.

Informasi lebih lanjut tentang masing-masing solusi dapat ditemukan di buku petunjuk untuk Windows (dalam bahasa Inggris) atau untuk Linux (juga dalam bahasa Inggris).

Tentang batasan edisi gratis


Untuk Veeam Agent untuk Linux (dibandingkan dengan Server edisi komersial):

  1. Hanya 1 pekerjaan cadangan yang dapat dibuat.
  2. Repositori Veeam Cloud Connect tidak didukung sebagai repositori cadangan target.
  3. Skrip pra-beku dan pasca pencairan tidak didukung untuk pekerjaan pencadangan.
  4. Cadangan dengan mempertimbangkan karya DBMS Oracle, MySQL, PostgreSQL tidak didukung.

Rincian lebih lanjut ada di tabel di sini . Dan tentang apa yang terjadi "di balik layar", bagaimana teknisi kami bekerja untuk memberikan dukungan untuk "kebun binatang" ini dan kompatibilitas solusi Veeam, baca artikel oleh Sergey Shtepa, seorang pengembang di sebuah blog di Habré.

Pembatasan untuk Agen untuk Windows tercantum secara rinci dalam tabel yang sama . Yang utama, kami perhatikan ini:

  1. Cadangan berbasis aplikasi tidak didukung.
  2. Repositori Veeam Cloud Connect tidak didukung sebagai repositori cadangan target.
  3. Microsoft OneDrive tidak didukung sebagai penyimpanan cadangan untuk mesin yang menjalankan OS server Windows.
  4. Microsoft OneDrive hanya dapat digunakan untuk cadangan status sistem. (Untuk informasi lebih lanjut tentang tipe cadangan, lihat di sini .)

Saya harap solusi Veeam Agent bermanfaat bagi semua orang yang bekerja dari kantor pusat. Jika rumah Anda memiliki infrastruktur yang lebih luas dan Anda memerlukan manajemen cadangan terpusat, Anda dapat menggunakan edisi gratis Veeam Backup & Replication Community Edition - ulasan dalam bahasa Rusia ada di sini .
