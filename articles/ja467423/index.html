Xamarin.Forms-アプリケーションでのアイコンフォントの便利な使用法


問題の定式化


Xamarin.Formsアプリケーションでアイコンを表示するために、png、svg、ttfフォントなどのさまざまな形式の画像を使用できます。ほとんどの場合、Googleマテリアルアイコンなどのアイコン付きフォントは、標準アイコンを追加するのに便利です。アイコン付きのフォントのサイズは約200Kで、通常、アプリケーションのサイズを節約するよりも使いやすさが重要です。アイコンはどの画面解像度でも見栄えがよく、白黒になります。

アイコンを使用するための既製のnugetパッケージがあります。長い間アイコン化を使用しました（nuget-www.nuget.org/packages/Xam.Plugin.Iconize ; git-github.com/jsmarcus/Iconize）10個を超えるフォントを接続し、IconButton、IconImage、IconLabelなどの新しいコントロールを追加できます。しかし、ここでは既製のライブラリに対する通常の議論があります。追加の機能、追加のファイルサイズ、動作、バグなどに完全には満足していません。したがって、ある時点で、完成したライブラリを破棄し、クラスとフォントのペアからの単純な自転車に置き換えることにしました。

ttfフォントのアイコンは、xamlで次のように使用されます（iOS）。

<Label Text="&#xe5d2;" FontFamily="MaterialIcons-Regular"/>

Cで：

var label = new Label { Text ="\ue5d2", FontFamily = "MaterialIcons-Regular" };

発生する問題をリストします。

1. XAMLおよびCでアイコンコードを書き込むための異なる形式＃（ "＆＃xe5d2;" / "\ ue5d2"）

2.アイコンコードはアイコン自体に関連付けられていません。アイコン化

3のように、「arrow_back」のようなものを使用したいと思います。アプリケーションがAndroidとiOSで実行される場合は、フォント名を異なる方法で記述する必要があります-Androidでは「MaterialIcons-Regular.ttf＃MaterialIcons-Regular」、iOSでは「MaterialIcons-Regular」

ここで説明するソリューションには、LabelコントロールのTextChangedイベントが必要です。この問題をリストに追加します

。4. LabelコントロールにTextChangedイベントがありません。これは、ラベルテキストの変更を追跡し、xamlでバインディングを使用するために必要です。

「Reactions to Xamarin.Forms Events」（https://docs.microsoft.com/ru-ru/xamarin/xamarin-forms/app-fundamentals/behaviors/）のロシア語翻訳では、ソリューションのいわゆる動作を使用し

ます。これを行うには、 GoogleMaterialFontBehaviorクラス：

public class GoogleMaterialFontBehavior: BehaviorBase<CustomLabel>
{
}

私たちの行動はラベルテキストの変更に対応するため、ラベルも確定する必要があります。

using System;
using Xamarin.Forms;

namespace MyApp.UserControls
{
    public class CustomLabel : Label
    {
        public event EventHandler<EventArgs> TextChanged;

        public static readonly new BindableProperty TextProperty =
            BindableProperty.Create(
                propertyName: nameof(Text),
                returnType: typeof(string),
                declaringType: typeof(CustomLabel),
                defaultValue: "", 
                defaultBindingMode: BindingMode.OneWay, 
                propertyChanged: TextChangedHandler);

        public new string Text
        {
            get => (string)GetValue(TextProperty);
            set
            {
                base.Text = value;
                SetValue(TextProperty, value);
            }
        }

        private static void TextChangedHandler(BindableObject bindable, object oldValue, object newValue)
        {
            var control = bindable as CustomLabel;
            control.TextChanged?.Invoke(control, new EventArgs());
        }
    }
}

GoogleMaterialFontBehaviorクラスでは、次のものが必要です。

  • OnAttachedTo / OnDetachingFromメソッドをオーバーライドする
  • テキスト変更ハンドラーを追加：HandleTextChanged
  • フォント名fontFamilyを追加
  • 文字コードのディレクトリを追加しますiconCodeDict

namespace MyApp.Behaviors
{
    public class GoogleMaterialFontBehavior: BehaviorBase<CustomLabel>
    {
        protected override void OnAttachedTo(CustomLabel bindable)
        {
            HandleTextChanged(bindable, null);
            bindable.TextChanged += HandleTextChanged;
            base.OnAttachedTo(bindable);
        }

        protected override void OnDetachingFrom(CustomLabel bindable)
        {
            bindable.TextChanged -= HandleTextChanged;
            base.OnDetachingFrom(bindable);
        }

        private void HandleTextChanged(object sender, EventArgs e)
        {
            var label = (CustomLabel)sender;

            if (label?.Text?.Length >= 2
                && iconCodeDict.TryGetValue(label.Text, out var icon))
            {
                label.FontFamily = fontFamily;
                label.Text = icon.ToString();
            }
        }

        //

        private static readonly string fontFamily = Device.RuntimePlatform == Device.Android ? "MaterialIcons-Regular.ttf#MaterialIcons-Regular" : "MaterialIcons-Regular";

        private static readonly Dictionary<string, char> iconCodeDict = new Dictionary<string, char>
        {
            {"3d_rotation", '\ue84d'},
            {"ac_unit", '\ueb3b'},
            {"access_alarm", '\ue190'},
        …
        }
    }
}

そして今、iconCodeDictリファレンス自体について。フォント文字の名前とコードのペアが含まれます。Googleマテリアルアイコンの場合、このデータは別のファイルのgitリポジトリにあります：github.com/google/material-design-icons/blob/master/iconfont/codepoints

使用例


XAML：

ページの上部に2つの名前空間を追加します。

xmlns:uc="clr-namespace:MyApp.UserControls"
xmlns:b="clr-namespace:MyApp.Behaviors"

そしてアイコンを表示します：

<uc:CustomLabel Text="arrow_back" FontSize="30">
    <Label.Behaviors>
        <b:GoogleMaterialFontBehavior />
    </Label.Behaviors>
</uc:CustomLabel>

C＃：

var label = new CustomLabel();
label.Behaviors.Add (new GoogleMaterialFontBehavior());
label.Text = "arrow_back";
