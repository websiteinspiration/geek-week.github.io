Aperçu du terminal Windows v0.9

La sortie de la version 0.9 de Windows Terminal a eu lieu. Il s'agit de la dernière version du Terminal, qui inclura de nouvelles fonctionnalités avant la sortie de la v1. Vous pouvez télécharger Windows Terminal depuis le Microsoft Store ou depuis la page de publication sur GitHub . Examinons de plus près les détails de la mise à jour!



Arguments de ligne de commande


L'alias wt prend désormais en charge les arguments de ligne de commande. Vous pouvez maintenant lancer le Terminal avec de nouveaux onglets et panneaux, séparés comme vous le souhaitez, avec des profils que vous aimez, en commençant par les répertoires que vous aimez! Les possibilités sont infinies! Voici quelques exemples:

wt -d.
Ouvre un terminal avec un profil par défaut dans le répertoire de travail actuel.

wt -d.; new-tab -d C: \ pwsh.exe
Ouvre un terminal avec deux onglets. Dans le premier - le profil par défaut, à partir du répertoire de travail actuel. Le second est le profil par défaut avec pwsh.exe en tant que «ligne de commande» (au lieu du profil par défaut «ligne de commande») commençant dans le répertoire C: \.

wt -p "Windows PowerShell" -d.; volet partagé -V wsl.exe
Ouvre un terminal avec deux panneaux séparés verticalement. Le profil avec le nom «Windows Terminal» est lancé sur le panneau supérieur et le profil par défaut est utilisé sur le panneau inférieur, en utilisant wsl.exe comme «ligne de commande» (au lieu du profil par défaut «ligne de commande»).

wt -d C: \ Users \ cinnamon \ GitHub \ WindowsTerminal; split-pane -p «Invite de commandes»; split-pane -p "Ubuntu" -d \\ wsl $ \ Ubuntu \ home \ cinnak -H

Découverte automatique PowerShell


Si vous êtes un grand fan de PowerShell Core , nous avons de bonnes nouvelles pour vous. Windows Terminal détecte désormais toute version de PowerShell et crée automatiquement un nouveau profil pour vous. La version PowerShell, qui, à notre avis, est la meilleure (à partir du plus grand numéro de version, jusqu'à la version GA, jusqu'à la version avec le meilleur bundle), sera appelée «PowerShell» et occupera l'emplacement d'origine PowerShell Core dans la liste déroulante.



Confirmer la fermeture de tous les onglets


Êtes-vous celui qui ne souhaite pas à chaque fois être invité à fermer tous les onglets? Si vous avez répondu oui, alors cette nouvelle fonctionnalité est définitivement pour vous! Un nouveau paramètre global a été créé qui vous permet de toujours masquer la boîte de dialogue de confirmation «Fermer tous les onglets». Pour ce faire, définissez le paramètre «confirmCloseAllTabs» sur false en haut du fichier profiles.json, et vous ne verrez plus jamais ce pop-up! Merci @ rstat1 pour la contribution de ce nouveau paramètre.
