CSSメディア表現が最大幅を超えています



最近、私たちは完全に再設計されたすっきりとしたデザインとたくさんの素晴らしい新しいイラストとアニメーションを備えた新しいマビロフトのウェブサイトを公開しました。

私たちのデザイナーがサイトを考え、描いて素晴らしい仕事をした後、私たち開発者は、コードを書いている大きな2K画面からそれを使って古いiPhone 4まで、各デバイスに対してレイアウトを実装し、きれいにして完全に保つ必要がありました。 3.5インチディスプレイ。

数週間後、ウェブサイトはほぼ準備が整いました。私はそれを調べて、すべてが正常に機能していることを確認しました。 iPadでメインページのレイアウトが正しく表示されないことに気づいたので、CSSルールをすばやく変更してエラーを修正しましたが、これにより、小さな画面のすべてのラップトップでレイアウトを表示する際に問題が発生しました


この問題の解決策を探したところ、JavaScriptを使用せずに、CSSメディア式のみを使用して両方のレイアウトを完璧にできることがわかりました。

実際、その前は基本的にCSSメディア式を使用して、レイアウトをmax-widthとmin-widthでレスポンシブにしました。

今、私はCSSメディア式が単なる最大幅ではなく、多くの異なる値であることを発見しました。そして、それらのいくつかは日常生活で本当に役立つことができます。

それで、私が話しているこれらのメディア表現は何ですか？聞いたことがあるかもしれません。しかし、私はそれらのいくつかを強調表示しました、それらの使用は非常に役立つことができます

ビューポートサイズ



はい、私は幅、高さ、最小幅、最小高さ、最大幅、最大高さについて話しています。

彼らは特別な紹介が必要ですか？簡単に説明します。
これらのメディア関数は、さまざまな画面サイズにさまざまなスタイルを設定するために使用されます。応答性の高いシステムの開発に非常に役立ちます。幅と高さの

関数はスコープの正確なサイズのスタイルのみを設定できるため、max-およびmin-プレフィックスを使用する可能性が高くなります。たとえば、次のコードでは、ビューポートの高さが320ピクセルを超える場合にのみスタイルが適用されます。

/*   ,      320  */
body {
  background-color: #FFD23F; /*   */
}
 
@media screen and (min-height: 320px) {
  body {
    background-color: #EE4266; /*   */
  }
}

ただし、これらの関数を組み合わせてさまざまなサイズを処理することもできます

/*   ,      320  600  */
body {
  background-color: #FFD23F; /*   */
}
 
@media screen and (min-width: 320px) and (max-width: 600px) {
  body {
    background-color: #EE4266; /*   */
  }
}

次の例は、max-width関数を使用して、スコープの幅に応じて背景色を変更する方法を示しています

body {
  background-color: #0EAD69;
}
 
@media screen and (max-width: 1600px) {
  body {
    background-color: #3BCEAC;
  }
}
 
@media screen and (max-width: 1280px) {
  body {
    background-color: #FFD23F;
  }
}
 
@media screen and (max-width: 960px) {
  body {
    background-color: #EE4266;
  }
}
 
@media screen and (max-width: 600px) {
  body {
    background-color: #540D6E;
  }
}



画面の向き



向きは、ディスプレイの向きに応じて変更できる優れたメディア機能です。これには2つの意味があります。縦（縦）と横（横/アルバム）です。

しかし、ブラウザはポートレートとランドスケープをどのように考慮していますか？ポートレートの値は、スコープの高さが幅よりも大きくなるたびに変更をトリガーします。同様に、幅が高さより大きい場合、ビューポートは「ランドスケープ」と見なされます。

たとえば、このコードでは次の結果が得られます。

/*    ,     */
body {
  background-color: #FFD23F; /*   */
}
 
@media screen and (orientation: landscape) {
  body {
    background-color: #EE4266; /*   */
  }
}



アスペクト比



このメディア関数は方向関数に似ていますが、より正確です。必要な特定のアスペクト比のルールを設定できます。たとえば、画面が16/9と18/9の電話に異なるレイアウトを設定できます。

この関数には、さまざまなアスペクト比の画面範囲を処理するための接頭辞maxおよびminを付けることもできます。

ガイダンスとポインター




したがって、これらのメディア機能は非常に役立つので、できるだけ簡単に説明しようと思います。ホバーとポインターの

両方のメディア機能は、サイトのメイン入力メカニズムの定義に関連しています。たとえば、マウス。スマートフォンを使用している場合は指。すべての入力メカニズムを定義するには、any-hoverおよびany-pointer関数を使用できます。 今、彼らはどうですか？ホバー（および任意のホバー） 関数を使用すると、ブラウザーは、メインの入力メカニズムが要素（マウスカーソルなど）にホバーできるかどうかを確認できます。 この関数を使用して、主な入力メカニズムを決定できます。






  • ホバー：ホバー、要素の上にホバーできます
  • ホバー：なし、要素の上にホバーできない、または入力のホバーメカニズムがない

いつ使用する価値がありますか？たとえば、モバイルデバイスでホバーアニメーションを処理する場合、入力メカニズム（指）は要素の上にホバーできません。ポインター（および任意のポインター）

関数を使用すると、ブラウザーは、メインの入力メカニズムにポインター（たとえば、マウス）があるかどうか、ある場合はその正確さを確認できます。 この関数を使用して、主な入力メカニズムを決定できます。


  • ポインター：粗い、ポインターの制限された精度を含む
  • ポインター：細かい、正確なポインターを含む
  • ポインター：なし、ポインターを含まない

これは、たとえば、ポインターが不正確なデバイスでクリック領域を増やす場合に役立ちます

反転色



反転色機能は、システムカラーが同じ設定であるときにいくつかのスタイルを適用する必要がある場合に非常に役立ちます。

システムカラー反転を有効にする理由 これは通常、読みやすさを向上させたい場合に非常に役立ちます。

また、フォントを増やし、色が反転したときに太さを減らすこともお勧めします。これがまさにメディア表現でできることです。

この関数には2つの値しかありません。
  • 反転：色が反転している場合はスタイルを適用します
  • none：デフォルトのスタイルを適用します


ここに例があります！色が反転すると、テキストサイズが大きくなります

.text {
  font-size: 24px;
}
 
@media screen and (inverted-colors: inverted) {
  .text {
    font-size: 36px;
  }
}



この機能は現在、Safari（macOSおよびiOS）でのみサポートされています。サポートされているブラウザの常に最新のリストは、こちらにあります。

好みの配色



これは私のお気に入りの機能の1つであり、徐々に普及していくことを期待しています。

この機能により、開発者は、配色が変更された場合に要素に異なるデザインオプションを設定できます。 WindowsとmacOSの両方で、ユーザーはシステムの配色を明るい色から暗い色に、またはその逆に変更できます。 iOS 13にもこの機能が導入されており、Androidユーザー向けに、この機能はAndroid Qの登場により利用可能になる予定です。

ブラウザーのサポートもまともです。 Google Chrome、Mozilla Firefox、Safariの最新バージョンは現在この機能をサポートしています。モバイルデバイスでは、Operaを除くすべての主要なブラウザーでもサポートされています。サポートされているブラウザの常に最新のリストは、こちらにあります。

この関数の値は非常に直感的です。
  • light：ユーザーが明るいテーマを好む場合はスタイルを適用します
  • dark：ユーザーが暗いテーマを好む場合はスタイルを適用します
  • no-preference：デフォルトのスタイルを適用します

以下の例では、この機能がいかにすばらしいかがわかります。

.appbar {
  background-color: #EE4266;
}
 
.fab {
  background-color: #424242;
}
 
@media screen and (prefers-color-scheme: dark) {
  body {
    background-color: #424242;
  }
 
  .appbar {
    background-color: #212121;
  }
 
  h2 {
    color: #fff;
  }
 
  .fab {
    background-color: #EE4266;
  }
}

アニメーションを減らす



それも非常に重要です。個人的には、アニメーションとトランジションが大好きです。適切なアニメーションは、サイトの使いやすさを本当に向上させると思います。しかし、誰もがアニメーションを好むわけではなく、さらに重要なことに、一部の人々は乗り物酔いとめまいを引き起こす可能性のある前庭障害を持っている可能性があります。私はそのような現象を非常によく説明する有用な記事を見つけました。

一般に、最も重要なデスクトップおよびモバイルオペレーティングシステムには、要素の移動性を低下させることでこれらの問題を解決できるユーザー補助オプションがあります。

この関数には2つの意味があります。
  • reduce：ユーザーがアニメーションや遷移を見たくない場合にスタイルを適用します。この値は通常、それらを無効にするために使用されます。
  • no-preference：通常のスタイルを適用します


以下の例でこの機能を確認してください。
.pulse {
  animation: pulse 2s infinite;
}
 
@media screen and (prefers-reduced-motion: reduce) {
  .pulse {
    animation: none;
  }
}



ユーザーが要素の移動性を減らすことを選択すると、リップルアニメーションは停止します。

この機能のブラウザーの互換性は十分です。現時点では、EdgeとIEのみがサポートしていません。サポートされているブラウザの常に最新のリストは、こちらにあります。
