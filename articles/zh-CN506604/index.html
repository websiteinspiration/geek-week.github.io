并发和效率：Python与FSM

我承认，但我不了解Python。只是因为我不使用它。尽管如此，在研究它的发展以及试图解释神秘的缩写GIL时，我去了一篇文章，描述了Python依赖于CPU的并行功能的并行版本的“莫名其妙的魔术现象”。希望重新检查该测试，并将其与可视化组件编程（自动）-VKP（a）环境中的状态机模型（简称为有限状态机或FSM）形式的等效实现进行比较。

显然任何该程序在某种程度上取决于CPU。另一方面，如果这不仅是汇编程序，则通过使用一种或另一种高级语言进行测试，我们将更有可能检查其隐藏的软件层。因此，考虑到Python，谈论其解释器对CPU的依赖会更正确。甚至可以说，Python程序的速度取决于解释器的版本，并且具有特定的“神秘主义”。

同时，在某些情况下，可能几乎不依赖CPU（我们将看到这一点）。我们正在谈论的是其计算模型与典型处理器体系结构不同的语言。Python的计算模型与之相对应，稍后将讨论的计算自动机模型具有不同的体系结构，这将决定其测试的细节。在“自动机代码”的测试过程中，将揭示速度以及神秘事物是否会发生。

清单1和2分别显示了从本文中借用的Python代码的串行和并行版本。在我们的示例中，它用固定工作时间的行进行了补充，并且顺序测试包含在“包装器”中，该包装器允许您设置重复次数（由于我熟悉Python中的装饰器）。

清单3.实现一个C ++测试 

class FsmCount
{
public:
    FsmCount(int n) { nCount = n; nState = 0; };
    void ExecOneStep();
    int nState;
protected:
    int nCount;
    int x1();
    void y1();
};

#include "FsmCount.h"

int FsmCount::x1() { return nCount > 0; }
void FsmCount::y1() { nCount--; }
void FsmCount::ExecOneStep()
{
    switch (nState) {
    case 0:
        if (x1())
            y1();
        else nState = 1;
        break;
    }
}

#include <QCoreApplication>
#include <QTime>
#include "FsmCount.h"

int main(int argc, char *argv[])
{
    QCoreApplication a(argc, argv);

    QTime time;
    FsmCount *pFsmCount{nullptr};

    pFsmCount = new FsmCount(100000000);
    time.start();
    while (!pFsmCount->nState)
        pFsmCount->ExecOneStep();
    double dTime = QString::number(time.elapsed()).toDouble();
    printf("%f", dTime);
    if (pFsmCount) delete pFsmCount;

    return a.exec();
}

void ThCount::run()

            int n = pQCount->pVarCount->GetDataSrc();
            if (n>0) {
                pQCount->time.start();

                n--;
                while (bIfExecuteStep) { bIfExecuteStep = false; }

            pQCount->pVarCount->SetDataSrc(pQCount, 0.0);
            while (n<=0) {
                n = int(pQCount->pVarCount->GetDataSrc());
            }
        }
    }
    bIfStop = true;        //    
}
