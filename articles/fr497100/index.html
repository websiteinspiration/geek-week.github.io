Lancer Jupyter en orbite LXD

Avez-vous déjà dû expérimenter avec du code ou des utilitaires système sous Linux de manière à ne pas secouer le système de base et à tout enlever en cas d'erreur de code qui devrait s'exécuter avec les privilèges root?


Mais qu'en est-il du fait, par exemple, qu'il est nécessaire de tester ou d'exécuter tout un cluster de divers microservices sur une seule machine? Cent ou même mille?


Avec des machines virtuelles gérées par un hyperviseur, ces tâches peuvent et vont réussir, mais à quel prix? Par exemple, un conteneur dans LXD basé sur la distribution Alpine Linux consomme peu de 7.60MBRAM et où il occupe la partition racine après le démarrage 9.5MB!Ça vous plaît, Elon Musk?Je recommande de se familiariser avec les capacités de base de LXD - systèmes de conteneurs sous Linux


Une fois qu'il est généralement devenu clair ce que sont les conteneurs LXD, allons-y et réfléchissons à ce qui se passerait s'il y avait un tel processeur de plate-forme où il serait sûr d'exécuter du code pour l'hôte, de générer des graphiques, de lier dynamiquement (interactivement) l'interface utilisateur. widgets avec votre code, ajoutez du code avec du texte avec blackjack ...mise en page? Quelque chose comme un blog interactif?Wow ... je le veux! Vouloir! :)


Regardez le chat où nous lancerons le JupyterLab dans le conteneur - la prochaine génération de l'interface utilisateur au lieu du obsolète Jupyter Notebook, et installons également des modules Python tels que NumPy , Pandas , Matplotlib , IPyWidgets qui vous permettront de faire tout ce qui précède et de tout enregistrer dans un fichier spécial - IPython-laptop .

from ipywidgets import interactive
import matplotlib.pyplot as plt
import numpy as np

def f(m, b):
    plt.figure(2)
    x = np.linspace(-10, 10, num=1000)
    plt.plot(x, m * x + b)
    plt.ylim(-5, 5)
    plt.show()

interactive_plot = interactive(f, m=(-2.0, 2.0), b=(-3, 3, 0.5))
output = interactive_plot.children[-1]
output.layout.height = '350px'
