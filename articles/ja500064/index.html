エレクトロンとウェビックスの友達を作る方法

前書き


一日のいい時間！ElectronとWebixの組み合わせを使用してデスクトップJavaScriptアプリケーションを作成した個人的な経験をお伝えします。このようなバンドルを使用すると、特にWebixフレームワークで実行できるレイアウトやその他のWeb作業に時間を費やすことなく、インターフェースのレイアウトプロセスを高速化できます。

ツール


それでは、始めましょう。次のツールが必要です。

  1. コードを直接記述するエディター。ここから取得できるビジュアルスタジオコード（VSC）を使用します。
  2. Node.jsサーバー。ここから取得できます。それをダウンロードしてインストールしてください。
  3. Webix Frameworkは無料バージョンです（Webix StandardはGNU GPLv3ライセンスに基づく無料のUIライブラリです）。これはwebix.com/get-webix-gplから入手できます。ダウンロードするには、上記のリンクにアクセスして、メール、名前、姓を入力し、3つのチェックマークを付け、[送信]をクリックすると、ダウンロードリンクがメールに送信されます。

一般的に、これで十分です。

ワークスペースの編成


次に、ワークスペースの編成に移ります。最初に行う必要があるのは、これなしでは実行できないいくつかのフォルダーとファイルを作成することです（図5）。

  • 「libs」フォルダー。ここでは「Webix」サイトからダウンロードしたファイルを配置します。
  • 「.vscode」フォルダー。ここでは、アプリケーションを実行するjsonファイルをVSCに配置します。
  • ファイル "main.js"、Electronアプリケーションを実行するメインファイル。
  • 「index.html」ファイルには、コンテンツとマークアップが含まれます。

  • 「launch.json」ファイル、「。vscode」フォルダー内の「F5」ボタンを使用して「VSC」環境でコードを実行するために必要なファイル。
  • ファイル「style.css」。このファイルでは、ウィンドウが適切に見えるように、いくつかのスタイルを登録する必要があります。

。15-Webixのレイアウトを使用したアプリケーションの作業ウィンドウ

いくつかのチップを追加します。


すべてが機能しているようです。でも、私たちのデザインには合わない、不要なウィンドウフレームを削除したいと思います。カスタムソリューションを使用します。「main.js」ファイルを開いてフレームパラメータを追加する理由：BrowserWindowの作成時にfalseを指定すると、このフラグは標準のメニューとウィンドウの境界線を削除します。次のようになります。

const win = new BrowserWindow({
    width: 800,
    height: 600,
    frame:false,
    webPreferences: {
      nodeIntegration: true
    }
  })

「F5」をクリックして、フレームウィンドウなしで結果を確認します（図16）。


図。16-Webixのレイアウトを使用したアプリケーションの作業ウィンドウ

図。17-ビューへのスタイルの追加：label要素

次に、この目的のために特別に作成されたファイルにこのスタイルを登録する必要があります。「style.css」ファイルを開き、次のスタイルを作成します。

.head_win {
    -webkit-app-region: drag;
}

その後、「F5」アプリケーションを起動し、タイトルでウィンドウをドラッグしてみます。すべてが機能するはずです。

最後に、アプリケーションは「-、+、x」ウィンドウのタイトルボタンのクリックに応答します。renderer.jsファイルの最後に次のコードを追加する理由：

//  
$$("close-bt").attachEvent("onItemClick", () => {
    const window = remote.getCurrentWindow();
    window.close();
})

//  
$$("min-bt").attachEvent("onItemClick", () => {
    const window = remote.getCurrentWindow();
    window.minimize();
})

//  
$$("max-bt").attachEvent("onItemClick", () => {
    const window = remote.getCurrentWindow();
    if (!window.isMaximized()) {
        window.maximize();
    } else {
        window.unmaximize();
    }
})

このコードでは、「$$（“ max-bt”）」という形式のエントリは、「id」による「Webix」要素へのアクセスを意味します。したがって、ヘッダーボタンでこれらのIDをファイル "renderer.js"で指定する必要があります。これは、図18のようになります。

   { "view": "label", "label": "Elcetron +Webix its cool!", css:"head_win" },
                    { "label": "-", "view": "button", "height": 38, "width": 40, id:"min-bt" },
                    { "label": "+", "view": "button", "height": 38, "width": 40, id:"max-bt" },
                    { "label": "x", "view": "button", "height": 38, "width": 40, id:"close-bt" }


図。18-ビューにスタイルを追加する：label要素
