言語構文


ChaiScriptは、構文がC ++とJSに非常に似ています。まず、ほとんどのスクリプト言語と同様に、動的に型付けされますが、JavaScriptとは異なり、厳密に分類されます（none 1 + "2"）。組み込みのガベージコレクターもあり、言語は完全に解釈可能で、バイトコードにコンパイルせずにコードを1行ずつ実行できます。例外（さらに、ジョイント、スクリプト内とC ++の両方で例外をキャッチできる）、ラムダ関数、演算子のオーバーロードをサポートしています。スペースの影響を受けないため、セミコロンを介して単一行に記述したり、式を新しい行で区切ってPythonスタイルで記述したりできます。

プリミティブタイプ


ChaiScriptはデフォルトで整数変数をint、realとしてdoubleとして保存し、文字列をstd :: stringとして保存します。これは主に、呼び出しコードとの互換性を確保するために行われます。言語には数値のサフィックスもあるので、変数のタイプを明示的に示すことができます。

/* 
  chaiscript    js
   ,  var / auto
`;`      
*/
var myInt = 1 // int
var myLongLong = 1ll // long long int
var myFloating = 3.3 // double
var myBoolean = false // bool
var myString = "hello world!\n" // std::string

変数のタイプを変更してもうまくいかない場合があり、これらのタイプには独自の `=`演算子を定義する必要があります。そうしないと、例外をスローする（これについては後で説明します）か、次のように丸めの犠牲になるリスクがあります。 

var integer = 3
integer = 5.433
print(integer) //  5    double    int!
integer = true //   -   `=`  (int, bool)

ただし、値を割り当てずに変数を宣言することができます。その場合、値が割り当てられるまで、変数には一種の未定義が含まれます。

実行コンテキスト


ChaiScriptにはローカルコンテキストとグローバルコンテキストがあります。コードは上から下に1行ずつ実行されますが、関数から取り出して後で呼び出すこともできます（ただし、先に呼び出すことはできません！）。関数または条件/ループの内部で宣言された変数は、デフォルトでは外部から見えませんが、global代わりに識別子を使用してこの動作を変更できますvar。グローバル変数は通常の変数とは異なります。1つはローカルコンテキストの外部で表示され、もう1つは再宣言できます（再宣言時に値が設定されていない場合、値は変わりません）。

//     chaiscript
def foo(x)
{
    global G = 2
    print(x)
}

foo(0) //  foo(x), G = 2
print(G)  //  2
global G = 3 //  G = 3,   global -  !

ちなみに、変数があり、その変数に値が割り当てられているかどうかを確認する必要is_var_undefがある場合は、変数が未定義の場合にtrueを返す組み込み関数を使用します。

文字列補間


メソッドが定義されている基本オブジェクトまたはユーザーオブジェクトは、to_string()構文を使用して文字列に配置できます${object}。これにより、不要な文字列の連結が回避され、一般的に見栄えがよくなります。

var x = 3
var y = 4

//  sum of 3 + 4 = 7
print("sum of ${x} + ${y} = ${x + y}")

Vector、Map、MapPairおよびすべてのプリミティブもこの機能をサポートしています。ベクトル出力形式[o1, o2, ...]、地図どのように[<key1, val1>, <key2, val2>, ...]してMapPair： <key, val>。

機能とそのニュアンス


ChaiScript関数は他のすべてのオブジェクトです。キャプチャして変数に割り当て、他の関数にネストして、引数として渡すことができます。また、それらに対して入力値のタイプを指定することができます（これは動的に型付けされた言語に欠けているものです）。これには、関数パラメーターを宣言する前にタイプを指定する必要があります。呼び出されたときに、パラメーターを指定されたパラメーターに変換できる場合、変換はC ++規則に従って行われます。それ以外の場合は、例外がスローされます。

def adder(int x, int y)
{
    return x + y
}

def adder(bool x, bool y)
{
   return x || y
}

adder(1, 2) // ,  3
adder(1.22, -3.7) // ,  1 + (-3) = 2
adder(true, true) // ,  true 
adder(true, 3) // ,    adder(bool, int)

言語の関数は、呼び出し条件（コールガード）を設定することもできます。尊重されない場合は例外がスローされ、それ以外の場合は呼び出しが行われます。また、関数の最後にreturnステートメントがない場合、最後の式が返されることにも注意してください。小さなルーチンに非常に便利です：

def div(x, y) : y != 0 { x / y } //  `y`    -    `x`  `y`

print(div(2, 0.5)) //  4.0
print(div(2, 0)) // , `y`  0!

クラスとDynamic_Object


ChaiScriptにはOOPの基本機能があります。これは、複雑なオブジェクトを操作する必要がある場合に間違いなくプラスです。言語には、Dynamic_Objectという特殊なタイプがあります。実際、クラスと名前空間のすべてのインスタンスは、プロパティが事前定義されたDynamic_Objectです。動的オブジェクトを使用すると、スクリプトの実行中にフィールドを追加して、それらにアクセスできます。

var obj = Dynamic_Object();
obj.x = 3;
obj.f = fun(arg) { print(this.x + arg); } //  obj   f (     `x`
obj.f(-3); //  0

クラスは非常に簡単に定義されます。フィールド、メソッド、コンストラクターを設定できます。興味深い点から、特別な関数を介してset_explicit(object, value)、クラス宣言の後の新しいメソッドまたは属性の追加を禁止することにより、オブジェクトのフィールドを「修正」できます（これは通常、コンストラクターで行われます）。

class Widget
{
    var id; //  id
    def Widget() { this.id= 0 } //   
    def Widget(id) { this.id = id } //   1 
    def get_id() { id } //  
}
var w = Widget(10)
print(w.get_id()) //  10 (w.id)
print(w.get_id) //   10,       
set_explicit(w, true) //   
w.x = 3 //      Widget   x

重要な点-実際には、クラスメソッドは、最初の引数が明示的に指定された型を持つクラスのオブジェクトである関数です。したがって、次のコードは既存のクラスにメソッドを追加することと同じです。

def set_id(Widget w, id) { w.id = id }

w.set_id(9) // w.id = 9
set_id(w, 9) //  , w.id = 9

C＃に精通している人なら誰でも、拡張メソッドのように見えるものを置き換えることができ、真実に近いでしょう。したがって、この言語では、組み込みクラス（たとえば、stringやint）でも新しい機能を追加できます。著者は、演算子をオーバーロードするトリッキーな方法も提供しています。これを行うには、次の例のように、演算子記号をチルド（ `）で囲む必要があります。

//   +     Widget
def `+`(Widget w1, Widget w2)
{
    print("merging two widgets!")
}

var widget1 = Widget()
var widget2 = Widget()
widget1 + widget2 //     

//        :
var plus = `+`
print(plus(1, 7)) //  8

名前空間


ChaiScriptの名前空間について言えば、これらは本質的に常にグローバルなコンテキストにあるクラスであることを覚えておく価値があります。関数を使用してそれらを作成しnamespace(name)、必要な関数とクラスを追加できます。デフォルトでは、この言語のライブラリはありませんが、拡張機能を使用してインストールできます。これについては、後で説明します。一般に、名前空間の初期化は次のようになります。

namespace("math") //    math

//  
math.square = fun(x) { x * x }
math.hypot_squared= fun(x, y) { math.square(x) + math.square(y) }

print(math.square(4)) //  16
print(math.hypot_squared(3, 4)) //  25

ラムダ式とその他の機能


ChaiScriptのラムダ式は、C ++でわかっている式に似ています。それらにはfunキーワードが使用され、キャプチャする変数を明示的に指定する必要もありますが、常に参照によりこれを行います。言語には、関数パラメーターに値をバインドできるbind関数もあります。

var func_object = fun(x) { x * x }
func_object(9) //  81

var name = "John"
var greet = fun[name]() { "Hello, " + name } 
print(greet()) //  Hello, John
name = "Bob"
print(greet()) //  Hello, Bob

var message = bind(fun(msg, name) { msg + " from " + name }, _, "ChaiScript");
print(message("Hello")) //  Hello from ChaiScript

例外


スクリプトの実行中に例外が発生する場合があります。ChaiScript自体（ここで説明します）とC ++の両方で傍受できます。構文はプラスとまったく同じです。数値または文字列を破棄することもできます。

try {
    eval(x + 1) // x  
} catch (e) {
    print("Error during evaluation"))
}

//   C++   ChaiScript
//   Vector -   std::vector,    std::exception     
try {
    var vec = [1, 2]
    var val = vec[3] //    
} catch (e) {
    print("index out of range: " + e.what()); // e.what    ChaiScript
}

//  atch   guard     ,    `:`
try {
  throw(5.2)
} catch(e) : is_type(e, "int") {
  print("Int: ${e}"); //   `e`  int
} catch(e) : is_type(e, "double") {
    print("Double: ${e}"); //  `e`  double
}

良い方法では、例外のクラスを定義してスローする必要があります。2番目のセクションでは、C ++でインターセプトする方法について説明します。インタープリター例外の場合、ChaiScriptはeval_error、bad_boxed_castなどの例外をスローします。

C ++での実装


取り付け


ChaiScriptは、C ++テンプレートベースのヘッダーのみのライブラリです。したがって、インストールの場合は、クローンリポジトリを作成するか、このフォルダのすべてのファイルをプロジェクトに配置するだけです。IDEによって、これはすべて異なる方法で行われ、長い間フォーラムで詳細に説明されてきたので、ライブラリを接続でき、include：を含むコードが#include <chaiscript/chaiscript.hpp>コンパイルされたと想定します。

C ++コードの呼び出しとスクリプトの読み込み


ChaiScriptを使用した最小のサンプルコードは次のようになります。 std ::文字列を取り、変更された文字列を返す単純な関数をC ++で定義し、それを呼び出すためにChaiScriptオブジェクトにその関数へのリンクを追加します。コンパイルにはかなりの時間がかかる可能性がありますが、これは主に、コンパイラー用に多数のテンプレートをインスタンス化することが容易ではないためです。

#include <string>
#include <chaiscript/chaiscript.hpp>

std::string greet_name(const std::string& name) 
{
    return "hello, " + name;
}

int main() 
{
    chaiscript::ChaiScript chai; //  chaiscript
    chai.add(chaiscript::fun(&greet_name), "greet"); //    greet

    //  eval      
    chai.eval(R"(
        print(greet("John"));
    )");
}

成功して、関数の結果を見たと思います。ちょっとしたニュアンスに注意したいのですが、ChaiScriptオブジェクトを静的として宣言すると、不快なランタイムエラーが発生します。これは、言語がデフォルトでマルチスレッドをサポートし、そのデストラクタでアクセスされるローカルフロー変数を格納するためです。ただし、静的インスタンスのデストラクタが呼び出される前に破棄されるため、アクセス違反またはセグメンテーションフォールトエラーが発生します。githubの問題に基づくと、最も簡単な解決策は#define CHAISCRIPT_NO_THREADS、コンパイラー設定を単に置くか、ライブラリー・ファイルを組み込む前にマルチスレッドを無効にすることです。私の理解では、このエラーを修正することはできませんでした。

次に、C ++とChaiScriptの間の相互作用がどのように行われるかを詳細に分析します。ライブラリーfunには、関数、ファンクター、またはクラス変数へのポインターを受け取り、状態を格納する特別なオブジェクトを返すことができる特別なテンプレート関数が定義されています。例として、ウィジェットクラスをC ++コードで定義して、さまざまな方法でChaiScriptに関連付けてみましょう。

class Widget
{
    int Id;
public:
    Widget(int id) : Id(id) { }
    int GetId() const { return this->Id; }
};

std::string ToString(const Widget& w)
{
    return "widget #" + std::to_string(w.GetId());
}

int main()
{
    chaiscript::ChaiScript chai;
    Widget w(2); //  Widget  C++ 

    chai.add(chaiscript::fun([&w] { return w; }), "get_widget"); //        
    chai.add(chaiscript::fun(ToString), "to_string"); //  
    chai.add(chaiscript::fun(&Widget::GetId), "get_id"); //  

    //    ,   Widget    GetId,    to_string,   
    chai.eval(R"(
        var w = get_widget()
        print(w.get_id) //  2
        print(w) //  widget #2
        )");
}

ご覧のように、ChaiScriptはまったく知らないC ++クラスと完全に落ち着いて動作し、それらのメソッドを呼び出すことができます。コードのどこかで間違いを犯した場合、ほとんどの場合、スクリプトはソート例外をスローしますがerror in function dispatch、これは重要ではありません。ただし、関数をインポートできるだけでなく、ライブラリを使用してスクリプトに変数を追加する方法を見てみましょう。これを行うには、タスクを少し難しく選択します-std ::ベクトル<ウィジェット>をインポートします。関数chaiscript::varとメソッドはこれで私たちを助けますadd_global。またData、ウィジェットにパブリックフィールドを追加して、クラスフィールドをインポートする方法を確認します。

class Widget
{
    int Id;
public:
    int Data = 0;
    Widget(int id) noexcept : Id(id) { }
    int GetId() const { return this->Id; }
};

std::string ToString(const Widget& w)
{
    return "widget #" + std::to_string(w.GetId()) +
        " with data: " + std::to_string(w.Data);

int main()
{
    chaiscript::ChaiScript chai;

    std::vector<Widget> W; //    Widget
    W.emplace_back(1);
    W.emplace_back(2);
    W.emplace_back(3);

    chai.add(chaiscript::fund(ToString), "to_string");

    chai.add(chaiscript::fun(&Widget::Data), "data"); //     
    
    //     ChaiScript
    chai.add_global(chaiscript::var(std::ref(W)), "widgets"); //     std::ref
    chai.add(chaiscript::fun(&std::vector<Widget>::size), "size"); //  
   
    // .       
    using IndexFuncType = Widget& (std::vector<Widget>::*)(const size_t);
    chai.add(chaiscript::fun(IndexFuncType(&std::vector<Widget>::operator[])), "[]");

    chai.eval(R"(
        for(var i = 0; i < vec.size; ++i)
        {
            vec[i].data = i * 2;
            print(vec[i])
        }
    )");
}

上記のコードが表示されますwidget #1 with data: 0、 widget #2 with data: 2、 widget #3 with data: 4。ChaiScriptのクラスフィールドへのポインターを追加し、フィールドがプリミティブ型であることが判明したので、その値を変更します。またstd::vector、それを使用するためにいくつかのメソッドが追加されましたoperator[]。STLに精通しているstd::vector人は、2つのインデックス作成方法があることを知っています。これが、オーバーロードされた関数の場合は、そのタイプを明示的に示す必要がある理由です。そうしないと、あいまいさが生じ、コンパイラーがエラーをスローします。

ライブラリには、オブジェクトを追加するためのメソッドがいくつかありますが、それらはすべてほぼ同じなので、詳細に検討する意味がわかりません。小さなヒントとして、以下のコードを提供します。

chai.add(chaiscript::var(x), "x"); // x   ChaiScript

chai.add(chaiscript::var(std::ref(x), "x"); //  ,    C++  ChaiScript

auto shared_x = std::make_shared<int>(5);
chai.add(chaiscript::var(shared_x), "x"); // shared_ptr      C++  ChaiScript

chai.add(chaiscript::const_var(x), "x"); //    ChaiScript   

chai.add_global_const(chaiscript::const_var(x), "x"); // global const . ,  x  

chai.add_global(chaiscript::var(x), "x"); // global , .  x  

chai.set_global(chaiscript::var(x), "x"); //   global ,    const

STLコンテナーの使用


プリミティブ型を 含むSTLコンテナーをChaiScriptに渡す場合は、テンプレートコンテナーのインスタンス化をスクリプトに追加して、各型のメソッドをインポートしないようにすることができます。

using MyVector = std::vector<std::pair<int, std::string>>;

MyVector V;
V.emplace_back(1, "John");
V.emplace_back(3, "Bob");

//    - vector  pair
chai.add(chaiscript::bootstrap::standard_library::vector_type<MyVector>("MyVec"));
chai.add(chaiscript::bootstrap::standard_library::pair_type<MyVector::value_type>("MyVecData"));

chai.add(chaiscript::var(std::ref(V)), "vec");
chai.eval(R"(
    for(var i = 0; i < vec.size; ++i)
    {
      print(to_string(vec[i].first) + " " + vec[i].second)
    }
  )");

内部では、いくつかのChaiScript関数が呼び出され、それら自体が必要なメソッドを追加します。一般に、クラスがSTLコンテナーを使用した同様の操作をサポートしている場合は、この方法で追加することもできます。cの場合std::vector<Widget>、残念ながら、これは不可能です。ChaiScriptでは、vector_typeウィジェットにはなかったelementのパラメーターなしのコンストラクターが必要なためです。

ChaiScript内のC ++クラス


おそらくタスクの一部として、ChaiScriptでオブジェクトを変更するだけでなく、スクリプトでオブジェクトを作成する必要があります。まあ、それはかなり可能です。たとえば、Widgetクラスをもう一度取り上げて、そこからWindowWidgetクラスを継承し、スクリプトに両方を作成する機能を追加し、継承したクラスを基本クラスに変換します。

class Widget
{
    int Id;
public:
    Widget(int id) : Id(id) { }
    int GetId() const { return this->Id; }
};

class WindowWidget : public Widget
{
    std::pair<int, int> Size;
public:
    WindowWidget(int id, int width, int height)
        : Widget(id), Size(width, height) { }

    int GetWidth()  const { return this->Size.first; }
    int GetHeight() const { return this->Size.second; }
};

int main()
{
    chaiscript::ChaiScript chai;

    //   Widget   
    chai.add(chaiscript::user_type<Widget>(), "Widget");
    chai.add(chaiscript::constructor<Widget(int)>(), "Widget");

    //   WindowWidget   
    chai.add(chaiscript::user_type<WindowWidget>(), "WindowWidget");
    chai.add(chaiscript::constructor<WindowWidget(int, int, int)>(), "WindowWidget");

    // ,  Widget -    WindowWidget
    chai.add(chaiscript::base_class<Widget, WindowWidget>());

    //   Widget  WindowWidget
    chai.add(chaiscript::fun(&Widget::GetId), "get_id");

    chai.add(chaiscript::fun(&WindowWidget::GetWidth), "width");
    chai.add(chaiscript::fun(&WindowWidget::GetHeight), "height");

    //  WindowWidget    
    chai.eval(R"(
        var window = WindowWidget(1, 800, 600)
        print("${window.width} * ${window.height}")
        print("widget.id is ${window.get_id}")
    )");
}

ポリモーフィズムは、情報を提供する型のC ++とまったく同じ方法でChaiScriptで機能します。なんらかの理由で、継承されたメソッドへのポインタを追加するときに多義性がある場合（おそらく、クラスは複数の基本メソッドから一度に継承される可能性があります）、インデックス演算子を使用した上記の例で行ったように、それを目的のクラスに明示的に持ってきstd::vector<Widget>ます。

インスタンスをメソッドにバインドして型を変換する


シングルトンオブジェクトの場合、メソッドまたはフィールドと一緒にそれらへのリンクのキャプチャを使用すると便利です。この場合、ChaiScriptでは、このオブジェクトに言及せずにアクセスできる関数またはグローバル変数を取得します。

Widget w(3);
w.Data = 4444;

//  Widget w
chai.add(chaiscript::fun(&Widget::GetId, &w), "widget_id");
chai.add(chaiscript::fun(&Widget::Data, &w), "widget_data");

chai.eval(R"(
    print(widget_id)
    print(widget_data)
)");

また、より多くの「ライブラリ」クラスをC ++からChaiScriptにエクスポートする場合（vec3、complex、matrixなど）、あるタイプから別のタイプへの暗黙的な変換の可能性がしばしば必要になります。ChaiScriptでは、この問題はtype_conversionオブジェクトにスクリプトを追加することで解決されます。たとえば、Complexクラスと、追加時にintおよびdoubleをそれに変換する実装を考えてみます。

class Complex
{
public:
    float Re, Im;
    Complex(float re, float im = 0.0f) : Re(re), Im(im) { }
};

int main()
{
    chaiscript::ChaiScript chai;

    //  Complex,   re, im,    `=`
    chai.add(chaiscript::user_type<Complex>(), "Complex");
    chai.add(chaiscript::bootstrap::standard_library::assignable_type<Complex>("Complex"));
    chai.add(chaiscript::constructor<Complex(float, float)>(), "Complex");
    chai.add(chaiscript::fun(&Complex::Re), "re");
    chai.add(chaiscript::fun(&Complex::Im), "im");

    //     double  int  Complex
    chai.add(chaiscript::type_conversion<int, Complex>());
    chai.add(chaiscript::type_conversion<double, Complex>());

    //     `+`   
    chai.eval(R"(
        def `+`(Complex c, x)
        {
            var res = Complex(0, 0)
            res.re = c.re + x.re
            res.im = c.im + x.im
            return res
        }

        var c = Complex(1, 2)
        c = c + 3
        print("${c.re} + ${c.im}i")
    )"); // : `4 + 2i`
}

したがって、C ++自体で変換関数を作成する必要はありません。変換関数をChaiScriptにエクスポートするだけです。変換を追加して、スクリプトコード自体に新しい機能を記述できます。2つの型の変換が重要な場合は、ラムダを引数として関数に渡すことができますtype_conversion。キャスト時に呼び出されます。

同様の原則を使用して、VectorまたはMap ChaiScriptをカスタムタイプに変換します。このために、vector_conversionとされているライブラリで定義されましたmap_conversion。

ChaiScriptの戻り値の解凍


メソッドevalとeval_file、最後に実行された式の値をオブジェクトとして返しますBoxed_Value。展開してC ++コードで結果を使用するには、戻り値の型を明示的に指定するか、関数を使用しますboxed_cast<T>。タイプ間の変換が存在する場合は実行され、それ以外の場合は例外が発生しbad_boxed_castます。

//      
double d = chai.eval<double>("5.3 + 2.1");

//     Boxed_Value,    
auto v = chai.eval("5.3 + 2.1");
double d = chai.boxed_cast<double>(v);

ChaiScript内のすべてのオブジェクトはshared_ptrを使用して格納されるため、オブジェクトをポインタとして取得して、さらに操作することができます。これを行うには、戻り値を変換するときにshared_ptrタイプを明示的に指定します。

auto x = chai.eval<std::shared_ptr<double>>("var x = 3.2");

主なことは、逆参照されたshared_ptrの値への参照を保持しないことです。そうしないと、スクリプトの自動ガベージコレクション中に変数が削除された後にアクセス違反が発生するリスクがあります。

変数と同様に、ChaiScriptオブジェクトの状態をキャプチャするパックファンクタの形式でChaiScriptから関数を取得できます。たとえば、すでに実装されているComplexクラスの機能を使用し、それを使用してプログラム実行段階で関数を呼び出します。

auto printComplex = chai.eval<std::function<void(Complex)>>(R"(
    fun(Complex c) { print("${c.re} + ${c.im}i"); }
)"); //  ,   ,      C++
printComplex(Complex(2, 3)); //  chaiscript,  `2 + 3i`

ChaiScript例外キャッチ


作成者は、自分で生成した例外に加えて、3種類の例外をキャッチすることを推奨しています。これはeval_error実行時エラー用でありbad_boxed_cast、戻り値が誤ってアンパックされたとき、およびstd::exceptionその他すべてに対して呼び出されます。独自の例外をスローする予定の場合は、C ++型への自動変換を構成できます。

class MyException : public std::exception
{
public:
    int Data;
    MyException(int data) : std::exception("MyException"), Data(data) { }
};

int main()
{
    chaiscript::ChaiScript chai;
    
    //      chaiscript
    chai.add(chaiscript::user_type<MyException>(), "MyException");
    chai.add(chaiscript::constructor<MyException(int)>(), "MyException");

    try
    {
        //         
        chai.eval("throw(MyException(11111))",
            chaiscript::exception_specification<MyException, std::exception>());
    }
    catch (MyException& e)
    {
        std::cerr << e.Data; //   `11111`
    }
    catch (chaiscript::exception::eval_error& e)
    {
        std::cerr << e.pretty_print();
    }
    catch(std::exception& e)
    {
        std::cerr << e.what();
    }
}

上記の例は、C ++でほとんどの例外をキャッチする方法を示しています。方法に加えてpretty_print、eval_errorそのようなコールスタック、ファイル名、エラーの詳細として有用なデータの多くはまだあるが、私たちはあまりこの記事では、このクラスに入ることはありません。

ChaiScriptライブラリ


残念ながら、デフォルトでは、ChaiScriptはライブラリに関して追加の機能を提供していません。たとえば、数学関数、ハッシュテーブル、ほとんどのアルゴリズムが不足しています。それらの一部を公式のChaiScript Extrasリポジトリからモジュールライブラリの形式でダウンロードし、スクリプトにインポートできます。たとえば、数学ライブラリとacos（x）関数を使用します。

#include <chaiscript/chaiscript.hpp>
#include <chaiscript/extras/math.hpp>

int main()
{
    chaiscript::ChaiScript chai;
    //  
    auto mathlib = chaiscript::extras::math::bootstrap();
    chai.add(mathlib);

    std::cout << chai.eval<double>("acos(0.5)"); // ~1.047
}

言語用のライブラリを作成してインポートすることもできます。これは非常に簡単に行われるので、オープンソースの数学やリポジトリ内の他のソースについて理解することをお勧めします。原則として、C ++との統合の一環として、ほぼすべてを検討したので、これでセクションは完了できると思います。
