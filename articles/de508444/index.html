Installieren und Konfigurieren von LXD auf OpenNebula

Vor dem Start des Linux-Administratorkurses wurde eine Übersetzung des Artikels erstellt .









In diesem Artikel finden Sie eine schrittweise Anleitung zu den Grundlagen des Cloud-basierten OpenNebula-Orchestrierungssystems auf LXD. Zuerst verwenden wir das einfache MiniONE-Tool, um eine Cloud-Umgebung mit einem Knoten in einer physischen oder virtuellen Maschine für Benutzer zu erstellen, die die OpenNebula-Funktionalität ausprobieren möchten. Anschließend sehen wir uns eine reguläre Arbeitssitzung mit einer GUI und einer CLI an und geben schließlich einige Links an, die beim Erstellen einer verteilten Produktionsumgebung hilfreich sind .



Automatisierte Bereitstellung mit MiniONE



MiniONE ist ein Tool, das einen physischen Host oder eine virtuelle Maschine als Einzelknoten-Cloud konfiguriert, um schnell einen einfachen, aber voll funktionsfähigen Testfall bereitzustellen.



Für die Evaluierung von LXD benötigt MiniONE eine dedizierte virtuelle Maschine oder einen physischen Host mit einer Neuinstallation von Ubuntu 18.04 oder 18.10 mit mindestens 2 GB RAM und 20 GB freiem Speicherplatz sowie privilegiertem Root-Zugriff. Mit MiniONE ist es beispielsweise einfach, eine LXD / OpenNebula-Umgebung auf einer virtuellen Amazon-Maschine zu erstellen. Die empfohlene Mindestgröße ist am wahrscheinlichsten t2.medium



. Geben Sie ihr einfach 25 GB Speicherplatz und Zugriff auf den TCP-Port 9869, auf dem die WebUI ausgeführt wird.



Führen Sie das MiniONE-Skript auf dem dedizierten System aus: 



wget https://github.com/OpenNebula/minione/releases/download/v5.8.0/minione
chmod u+x minione
sudo minione --lxd


Die GUI kennenlernen



Nach dem Schritt mit MiniONE erhalten Sie Anweisungen zum Herstellen einer Verbindung mit der Weboberfläche wie folgt: 



### Report
OpenNebula 5.8 was installed
Sunstone (the webui) is runninng on:
  http://192.168.100.101:9869/
Use following to login:
  user: oneadmin
  password: o6ARsMAdGe


Nachdem Sie die Arbeit von MiniONE abgeschlossen haben, melden wir uns zunächst als oneadmin an und sehen uns die Admin-Ansicht in Sunstone an , in der es für normale Benutzer viel mehr Optionen als für andere Benutzeroberflächen gibt. Sehen Sie sich alle heruntergeladenen Cloud-Ressourcen an.







In der Administratoransicht können Sie mit OpenNebula alles tun, aber Sie müssen Endbenutzern nicht dieselben Möglichkeiten bieten! Wechseln Sie zur Cloud-Ansicht und sehen Sie, was Endbenutzer in OpenNebula sehen. Cloud View-







Oberflächeviel einfacher, weil es sich an Endbenutzer richtet. Erstellen Sie eine neue virtuelle Maschine, indem Sie auf die Schaltfläche + klicken. Wählen Sie die einzige verfügbare Vorlage aus und klicken Sie auf die Schaltfläche Erstellen. Nachdem Sie auf die Schaltfläche Erstellen geklickt haben, gelangen Sie zum Dashboard, in dem Sie alle ausgeführten virtuellen Maschinen sehen können.







Um das Control Panel zu öffnen, können Sie auf Ihre virtuelle Maschine klicken. Dort können Sie von VNC aus darauf zugreifen, seinen Status speichern, neu starten usw.:







Durch Klicken auf das Konsolensymbol gelangen Sie über VNC als Root als Benutzer zur Konsole.







Mit der Rolle oneadmin können Sie konfigurieren, was sie sehen und tun können Ihre Cloud-Benutzer.



Treffen Sie die CLI



OpenNebula wird als oneadmin gestartet, und der Hauptadministrator muss Befehle für diesen Benutzer ausführen. Sie müssen also zunächst zu oneadmin wechseln : 



su - oneadmin




Auf dem oneadmin- Konto können Sie die geladenen Ressourcen sehen:

Wir haben einen Visualisierungsknoten: 



onehost list


CentOS-Image erstellt 



oneimage list


Registrierte Vorlage für virtuelle Maschinen: 



onetemplate list


Sie können die Konfiguration der Vorlage detaillierter sehen. 



onetemplate show 0


Greifen Sie auf MarketPlace zu



OpenNebula wird mit vorinstalliertem MarketPlace geliefert, von dem Sie viele vorgefertigte Bilder (Apps) erhalten können. Es gibt auch eine Integration mit Images , einem der öffentlichen LXD-Image-Server:







Es gibt verschiedene Linux-Distributionen: Alpine, Centos, Debian usw. sowie einige Service-Images mit vorinstallierten Anwendungen wie WordPress oder GitLab.







Es gibt Container vom öffentlichen LXD-Image-Server. Sie können zum Beispiel ein völlig frisches Ubuntu finden.







Und laden Sie es in das Repository herunter.







Wenn Sie jetzt zum Abschnitt Vorlage -> VMs gehen , können Sie ihn instanziieren.



Installation Montage in der Produktion



Wenn Sie die Cloud in der Produktion bereitstellen möchten, können Sie die Dokumentation lesen . Das Bereitstellen einer Umgebung mit einem Frontend und mehreren LXD-Knoten erfordert die folgenden Schritte:



  • Installieren Sie das OpenNebula- Frontend .
  • Installieren Sie das lxd-node- Paket in Virtualization Nodes .
  • Verbinden Sie das Frontend und die Knoten .
  • Stellen Sie sicher, dass alles funktioniert .


Jetzt können Sie virtuelle Maschinen mithilfe von Anwendungen von MarketPlace bereitstellen.
