Swift 5.2-新機能

Swift 5.2の最初のベータ版がXcode 11.4ベータ版に登場しました。コードのサイズと使用メモリのサイズの縮小に加えて、言語が変更され、エラーをすばやく検出できる新しい診断システムが導入されました。

関数としてのKeyPath式の使用


次の構造を定義します。

struct User {
    let name: String
    let age: Int
    let bestFriend: String?

    var canVote: Bool {
        age >= 18
    }
}

構造体のインスタンスをいくつか作成して、配列に入れましょう。

let eric = User(name: "Eric Effiong", age: 18, bestFriend: "Otis Milburn")
let maeve = User(name: "Maeve Wiley", age: 19, bestFriend: nil)
let otis = User(name: "Otis Milburn", age: 17, bestFriend: "Eric Effiong")
let users = [eric, maeve, otis]

ここで注意：すべてのユーザーの名前の配列を取得する必要がある場合は、次のように実行できます。

let userNames = users.map(\.name)
print(userNames)

以前は、クロージャーを使用する必要がありました。

let oldUserNames = users.map { $0.name }

同じ新しい方法で、投票できるすべてのユーザーを取得できます。

let voters = users.filter(\.canVote)

そして、ここには親友を持つすべての人がいます：

let bestFriends = users.compactMap(\.bestFriend)

ユーザー定義型の値


lowerBoundプロパティとupperBoundプロパティを使用し てDice構造を作成し、callAsFunction関数を追加します。したがって、サイコロの値を取得するたびに、ランダムな値が取得されます。

struct Dice {
    var lowerBound: Int
    var upperBound: Int

    func callAsFunction() -> Int {
        (lowerBound...upperBound).randomElement()!
    }
}

let d6 = Dice(lowerBound: 1, upperBound: 6)
let roll1 = d6()
print(roll1)

ここでは、1から6までの乱数を取得します。これは、callAsFunction（）への直接呼び出しと完全に同じです。次のように同じことを行うことができます。

let d12 = Dice(lowerBound: 1, upperBound: 12)
let roll2 = d12.callAsFunction()
print(roll2)

Swiftは、callAsFunction（）の定義方法に基づいて、正しい呼び出しを自動的に選択します。たとえば、いくつかのパラメータを追加したり、戻り値を変更したり、必要に応じてメソッドを変更としてマークしたりできます。

ここでは、実行したステップ数をキャプチャし、完了したステップ数が10,000に達したかどうかを通知するStepCounter構造を作成します。

struct StepCounter {
    var steps = 0

    mutating func callAsFunction(count: Int) -> Bool {
        steps += count
        print(steps)
        return steps > 10_000
    }
}

var steps = StepCounter()
let targetReached = steps(count: 10)

callAsFunction（）はスローと再スローもサポートし、通常のオーバーロードと同様に、いくつかのcallAsFunction（）メソッドを定義できます。

下付き文字はデフォルトの引数を宣言できます


タイプに添え字を追加するときは、デフォルトの引数を使用できます。たとえば、ユニット内の将校をリストするカスタム添え字を持つPoliceForce構造がある場合、配列が境界の外で読み取られた場合にそれを返すデフォルトパラメータを追加できます。

struct PoliceForce {
    var officers: [String]

    subscript(index: Int, default default: String = "Unknown") -> String {
        if index >= 0 && index < officers.count {
            return officers[index]
        } else {
            return `default`
        }
    }
}

let force = PoliceForce(officers: ["Amy", "Jake", "Rosa", "Terry"])
print(force[0])
print(force[5])

インデックス5の配列要素がないため、ここでは出力に「Amy」が表示され、次に「Unknown」が表示されます。

デフォルトのデフォルトを記述したので、次のようにカスタム値を設定できます。

print(force[-1, default: "The Vulture"])

新しく改善された診断


Swift 5.2では、Xcode開発エラーメッセージの品質と精度を向上させることを目的とした新しい診断アーキテクチャが導入されています。これは、Swiftが誤検出エラーメッセージを頻繁に生成するSwiftUIコードを操作する場合に特に顕著です。

たとえば、次のコードについて考えてみます。

struct ContentView: View {
    @State private var name = 0

    var body: some View {
        VStack {
            Text("What is your name?")
            TextField("Name", text: $name)
                .frame(maxWidth: 300)
        }
    }
}

ここでは、ビューTextFieldを整数に関連付けようとしています 状態間違っているプロパティ。この場合、Swift 5.1は、フレーム（）修飾子「Int」が「CGFloat？」に変換できないことを示すエラーを生成します。しかし、Swift 5.2以降は$ nameのバインディングのエラーを正しく認識します：タイプ 'Binding'の値を予期される引数タイプ 'Binding'に変換できません。
