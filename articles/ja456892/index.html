Webアプリケーションの脆弱性を探す方法：8つの一般的なスキャナーの比較

Webアプリケーションスキャナーは、今日のソフトウェアのかなり人気のあるカテゴリです。有料スキャナーと無料スキャナーがあります。それぞれに、検出可能な独自のパラメーターと脆弱性のセットがあります。OWASPトップ10（Open Web Application Security Project）で公開されたものだけに限定されているものもあれば、ブラックボックステストでさらに進んでいるものもあります。



この投稿では、8つの一般的なスキャナーを収集し、それらをより詳細に調査して試してみました。2つのプラットフォーム（.NETやPHP）の独立したポイントは、研修の対象として選択した：premium.pgabank.comとphp.testsparker.com。

OWASP ZAP


名前が示すように、序論で述べたOWASP組織がOWASP ZAPのリリースを担当しています。これは、侵入テストとWebアプリケーションの脆弱性を見つけるための無料のツールです。

OWASP ZAPの主な機能：

  • 中間者プロキシ
  • 従来のスパイダーとAJAXスパイダー
  • 自動スキャナー
  • パッシブスキャナー
  • 強制閲覧
  • ファザー

追加機能
  • Dynamic SSL certificates
  • Smartcard and Client Digital Certificates support
  • Web sockets support
  • Support for a wide range of scripting languages
  • Plug-n-Hack support
  • Authentication and session support
  • Powerful REST based API
  • Automatic updating option
  • Integrated and growing marketplace of add-ons

プログラムのインターフェースはロシア語に翻訳されており、一部のユーザーにとって便利です。OWASP ZAPワークスペースは、いくつかのウィンドウで構成されています。下部には、現在のタスクとその実装のプロセスを示すタブがあり、左側にはサイトのツリーがあり、要求と応答ウィンドウの右側にさらに表示できます。



マーケットプレイスを使用すると、スキャナーの機能を少し拡張できます。



プログラムの各コンポーネントには、多くのカスタマイズ可能なパラメーターがあります。たとえば、アクティブスキャンの着信ベクターの構成、動的SSL証明書の生成、HTTPセッション識別子の追加などを行うことができます。

すべてのスキャン結果をレポートにエクスポートできます（* .pdf、* .html、* .xml、* .jsonがサポートされています）。レポートでは、脆弱性、検出されたベクトル、および脆弱性を「閉じる」方法について詳しく説明します。



一般的に、OWASP ZAPを使用するのが好きでした。Pentest Webアプリケーションに必要なすべてのツール、シンプルで直感的なインターフェイス、クイックワンクリックスキャンがあります。同時に、より詳細なスキャンのための柔軟で詳細な設定は、脆弱性をさらに手動で検索するための出発点として役立ちます。以下では、OWASP ZAPと多くの共通点を持つBurp Suite Proスキャナーについても説明します。見つかった脆弱性の数と質によって、最初に調査したスキャナーは非常に良い結果を示しました。仕事での使用をお勧めします。

ワピティ


別の優れたコンソールスキャナー。W9scanと同じように、1つのコマンドで起動する準備ができていますが、スキャン設定はさらに異なります。



Wapitiは次の脆弱性を検索します。

  • ファイルの開示（ローカルおよびリモートのinclude / require、fopen、readfile ...）
  • データベースインジェクション（PHP / JSP / ASP SQLインジェクションとXPathインジェクション）
  • XSS（クロスサイトスクリプティング）インジェクション（反映および永続）
  • コマンド実行の検出（eval（）、system（）、passtru（）...）
  • CRLFインジェクション（HTTP応答分割、セッション修正...）
  • XXE（XML外部エンティティ）インジェクション
  • SSRF（サーバー側リクエストフォージェリ）
  • Use of know potentially dangerous files
  • Weak .htaccess configurations that can be bypassed
  • Presence of backup files giving sensitive information
  • Shellshock

上記すべてに加えて、プロキシ（HTTP、HTTPsおよびSOCKS5）、さまざまな認証方法（Basic、Digest、Kerberos、NTLM）、SSL証明書のサポート、さまざまなHTTPヘッダーまたはユーザーエージェント設定を追加する機能のサポートがあります。

サイトphp.testsparker.comをスキャンしたところ、ブラインドSQLインジェクション、クロスサイトスクリプティング、コマンド実行の脆弱性が見つかりました。でpremium.bgabank.com、ワピチは、他のスキャナと比較して、ショーは、このような優れた結果ではない：だけでクロスサイトスクリプティングが検出されました。



スキャナーの結果に基づいて、見つかった脆弱性のカテゴリーと数、それらの説明、リクエスト、カールのコマンド、見つかったセキュリティホールを閉じる方法のヒントを含むHTMLレポートも生成されます。

もちろん 、WapitiはOWASP ZAPのレベルには達していません。それでも、ディレクトリやサブドメインを検索したり、サービスバージョンを決定したりはしませんでしたが、W9scanよりもうまく機能しました。

パロス


別のGUI Web脆弱性スキャナー。デフォルトでは、Kali Linuxディストリビューションに含まれており、ローカルにインストールされます。分析用のサイトを追加するための組み込みプロキシ、サイトを分析してクエリマップを作成できる組み込みWebスパイダーがあります。

ユーザーの個人アカウントをスキャンするには、Parosプロキシ経由でトラフィックのリダイレクトを有効にしてブラウザにログインする必要があります。スキャナーは、スキャンプロセス中に承認されたCookieを使用します。活動レポートはHTMLにエクスポートできます。これはroot / paros / session / LatestScannedReport.htmファイルに保存され、その後上書きされます。以前のスキャンの結果を保存する場合は、次のスキャンを開始する前に、既存のファイルのコピーを作成する必要があります。

主な機能（OWASP TOP 10 2017に注目）：

  • A1：インジェクション-SQLinjection、SQLinjection Fingerprint（SQLinjが存在する可能性がある場所）
  • A6：セキュリティの誤った構成-ディレクトリの参照、ISSのデフォルトファイル、Tomcatソースファイルの公開、IBM WebSphereのデフォルトファイル、およびソースコードなどを含むその他の標準または廃止されたファイル（廃止されたファイル）。
  • A7：XSS

追加機能：

  • パスワードフォームの有効なオートコンプリートを検索します。さらに、入力フィールドの属性がtype = "password"の場合、誤検知が発生します。
  • CRLF注入
  • 安全なページブラウザーキャッシュ（重要な情報を含むブラウザーでのページのキャッシュ）
  • ユーザーの保護された領域をスキャンする機能（個人アカウント）
  • ローカルネットワーク上のWebアプリケーションをスキャンする機能



脆弱性の各タイプの最終レポートには、より詳細な情報とそれを修正する方法に関するいくつかの推奨事項があります。
