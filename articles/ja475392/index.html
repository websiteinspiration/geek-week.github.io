python -m pipとは何ですか？


はじめに、python -m pipは、pythonステートメントに指定したバージョンのPythonを使用してpipを実行します。したがって、/usr/bin/python3.7 -m pipにあるインタープリターに対してpipを実行することを意味します/usr/bin/python3.7。フラグがどのように機能するかわからない場合は、フラグに関するドキュメントを読むことができます-m（ちなみに、非常に便利です）。

より詳細な情報がなければ、答えはわかりません。最初に、PATHにあるものを理解する必要があります。つまり、/usr/bin最初に来るか、/usr/local/bin（ちなみに、Pythonをインストールする最も一般的な場所は、通常、/usr/local/ 最初です）。そのため、Python 3.7と3.8をインストールした場所と、それらが異なるディレクトリであることを覚えており、PATHで最初に何が発生したかがわかります。両方を手動でインストールしたとします。おそらく、Python 3.7.3が既にシステムにプリインストールされており、Python 3.7.5をインストールしたとします。この場合、Pythonの両方のバージョンがにインストールされ/usr/local/binます。現在、どのピップが取り付けられているか教えていただけますか？

あなたは答えを知りません。各バージョンがいつインストールされたかがわからず、pipの最新バージョンがに書き込まれ/usr/local/bin/pipたことはわかっているが、pipコマンドで使用されるインタープリターがわからない場合。 「私は常に最新バージョンをインストールします。つまり、3.7.5よりも新しいので、Python 3.8.0が最後にインストールされることを意味します。」さて、しかし、Python 3.7が出たときに何が起こるか.6？あなたのpipは、Python 3.8ではなくPython 3.7から使用され

。Windowsを使用している場合は、python -m pipを使用する追加のインセンティブがあります。これは、pipが自身を更新できるようにするためです。主に、pip install --upgrade pipと記述したときに、pip.exeが実行中と見なされるためです。この時点で、Windowsはpip.exeの再インストールを許可しません。ただし、python-m pip install --upgrade pipを実行すると、pip.exeではなくpython.exeが起動されるため、この問題を回避できます。
