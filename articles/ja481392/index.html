暗号通貨、それはどのようなものですか？


情報はWikipedia

Cryptocurrency（デジタル通貨の一種）から取得されます。内部決済単位の会計は、分散型決済システム（内部または外部の管理者またはその類似者は存在しません）によって提供され、完全自動モードで機能します。

暗号通貨自体には特別な資料や電子形式はありません。これは、決済単位のデータの量を示す数値であり、データ転送プロトコルの情報パケットの対応する位置に書き込まれ、多くの場合、他のトランザクションに関する他の情報と同様に暗号化されていませんシステムアドレス。

つまり、cryptocurrencyは次のように機能します。

  1. 残高のあるネットワークユーザーがトランザクションを別の匿名ユーザーに送信します。
  2. トランザクションはネットワークに入ります。
  3. 鉱山労働者（またはブロックを作成するユーザー）は、条件付きの特別なアルゴリズムによって生成された問題を解決します（たとえば、ハッシュに以前の10から30のゼロが存在する、またはその逆）。
  4. そのおかげで、条件がハッシュで満たされると、マイナーは新しいブロックの作成とトランザクションについて通知するメッセージをネットワークに送信します。
  5. 受信者は資金を受け取り、鉱山労働者はタスクの準備中に計算された手数料の形で報酬の額を受け取ります。

開発前の注意事項を検討


作業証明を使用した二重ハッシュ検証システムを使用します。ソフトウェアおよびネットワークアプリケーションレベルでハッシュをチェックすることは理にかなっています。これは、サーバーの負荷を最適化するプロセスによるものです。なぜなら、ハッシュをチェックするためにサーバーに何度も行くとき、そのような負荷が現れるからです。したがって、私は次の方法を実行することを提案します。

  1. ソフトウェアマイナーで、ローカルでハッシュをチェックして条件を満たすようにします。
  2. ソフトウェアマイナーの条件が必要な条件と一致する場合は、アプリケーションネットワークレベル（サーバーレベル）でチェックを行います。

これは、私の意見では、負荷を分散するための最良の方法です。サーバーへの要求を何度もスパムするよりも、コンピューターのローカルで数回確認する方が簡単です。

        public function login ($walletId, $password) // 
	{
		if ($this->db) //   
		{

			$hash = crc32 ($walletId) . $walletId . hash_hmac ('sha512', str_rot13 ($password), 'FLC'); //     .
			$hash = hash_hmac ('sha512', $hash, 'FLC2'); //     .

			$newUser = $this->db->prepare ("
					SELECT
						*
					FROM
						`wallets`
					WHERE
						walletId = :walletId
					AND
						loginHash = :hash
					LIMIT
						0, 1
				"); //    .

			$newUser->bindParam (':walletId', $walletId, PDO::PARAM_STR); //    
			$newUser->bindParam (':hash', $hash, PDO::PARAM_STR); //    

			$newUser->execute (); //  .
			$object = $newUser->fetch_object (); //   ,    
			if ($object AND hash_equals ($object->hash, $hash)) //   ,   .
			{
				$loginString = new stdClass; //    stdClass
				$loginString->hash = $hash; //       .
				$loginString->wallet = $walletId;

				return ['token' => serialize ($loginString)]; //      ,   .
			}
			else
				return false; //   / .
		}
		else
			return false; //   
	}
