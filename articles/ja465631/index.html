Cisco 200-125 CCNA v3.0のトレーニング。27日目。ACLの概要。パート2

もう1つ言及し忘れたことは、ACLは許可/拒否の原則に基づいてトラフィックをフィルタリングするだけでなく、さらに多くの機能を実行することです。たとえば、ACLはVPNトラフィックを暗号化するために使用されますが、CCNA試験に合格するには、トラフィックのフィルタリングにそれがどのように使用されるかを知るだけで済みます。問題番号1に戻ります。



リストされているACLを使用して、R2出口インターフェースで経理部門と営業部門のトラフィックをブロックできることがわかりました。



このリストの形式を気にする必要はありません。これは、ACLの本質を理解するための例としてのみ必要です。 Packet Tracerを使い始めるとすぐに、正しい形式に到達します。

問題2は次のとおりです。サーバーは、管理部門のホスト以外のすべてのホストと通信できます。つまり、サーバーコンピュータは、販売および経理部門のコンピュータにはアクセスできますが、管理部門のコンピュータにはアクセスできません。これは、サーバーITスタッフが管理部門の責任者のコンピューターにリモートアクセスできないようにする必要があることを意味します。障害が発生した場合は、彼のオフィスに来て、その場で問題を修正してください。サーバールームがネットワーク経由で管理部門と通信できなかった理由がわからないため、このタスクは実用的な意味を持たないことに注意してください。この場合は、ケーススタディのみを検討します。

この問題を解決するには、最初にトラフィックの伝送パスを決定する必要があります。サーバーからのデータは、ルーターR1の入力インターフェースG0 / 1に送られ、出力インターフェースG0 / 0を介して管理部門に送信されます。



拒否条件192.168.1.192/27を入力インターフェイスG0 / 1に適用すると、覚えているように、標準ACLがトラフィックソースの近くに配置され、販売部門と経理部門を含むすべてのトラフィックがブロックされます。



管理部門宛てのトラフィックのみをブロックする必要があるため、出力インターフェイスG0 / 0にACLを適用する必要があります。この問題は、ACLを宛先に近づけることによってのみ解決できます。同時に、経理ネットワークと営業部門からのトラフィックは管理部門に自由に到達する必要があるため、リストの最後の行はPermit anyコマンドとなり、前の条件で指定されたトラフィック以外のトラフィックを許可します。

問題3に進みましょう。営業部門のラップトップ3は、営業部門のローカルネットワークにあるデバイス以外のデバイスにはアクセスできません。訓練生がこのコンピューターで働いていて、LANを超えてはならないものとします。
この場合、ルータR2の入力インターフェイスG0 / 1にACLを適用する必要があります。このコンピューターにIPアドレス192.168.1.3/25を割り当てる場合、拒否条件192.168.1.3/25が満たされている必要があり、他のIPアドレスからのトラフィックはブロックされないようにする必要があるため、リストの最後の行は[許可]になります。



同時に、トラフィックの禁止はLaptop2には影響しません。

次はタスク番号4です。サーバーネットワークにアクセスできるのは財務部門のコンピューターPC0だけで、管理部門はアクセスできません。



覚えていれば、タスク＃1のACLはルーターR2のS0 / 1/0インターフェイス上のすべての発信トラフィックをブロックしますが、タスク＃4はトラフィックがPC0にのみ送信されるようにする必要があるため、例外を作成する必要があると述べています。

私たちが現在解決しているすべてのタスクは、オフィスネットワークのACLを設定するときの実際の状況で役立つはずです。便宜上、クラシック形式の録音を使用しましたが、録音を修正できるように、すべての行を手動で紙に書くか、コンピューターで印刷することをお勧めします。私たちの場合、タスク1の条件に従って、クラシックACLがコンパイルされます。タイプがPermit <PC0のIPアドレス>のPC0に例外を追加する場合は、この行をリストの4番目の行、Permit Anyの後にのみ配置できます。ただし、このコンピューターのアドレスは拒否192.168.1.128/26条件検証アドレス範囲にあるため、この条件が満たされた直後にそのトラフィックはブロックされ、ルーターはこのIPアドレスからのトラフィックを許可する4番目の回線に到達しません。
したがって、タスク1のACLを完全にやり直す必要があります。最初の行を削除して、PC0トラフィックを許可するPermit 192.168.1.130/26行に置き換え、すべての経理および営業部門のトラフィックを禁止する行を再入力します。



したがって、1行目には特定のアドレスのコマンドがあり、2行目にはこのアドレスが配置されているネットワーク全体に共通のコマンドがあります。最新タイプのACLを使用する場合、最初のコマンドとしてPermit行192.168.1.130/26を配置することで、ACLを簡単に変更できます。クラシックACLがある場合は、それを完全に削除してから、正しい順序でコマンドを再入力する必要があります。

問題＃4の解決策は、タスク＃1のACLの上部に許可行192.168.1.130/26を配置することです。これは、この場合に限り、PC0トラフィックがルーターR2の出力インターフェイスから支障なく出るためです。 PC1のトラフィックは完全にブロックされます。これは、そのIPアドレスがリストの2行目に含まれている禁止の対象であるためです。

次に、Packet Tracerに進み、必要な設定を行います。以前の簡略化されたスキームは理解するのが少し難しいため、すべてのデバイスのIPアドレスをすでに構成しました。さらに、2つのルーター間にRIPを構成しました。特定のネットワークトポロジでは、4つのサブネットのすべてのデバイス間の通信が制限なしに可能です。ただし、ACLを適用するとすぐに、トラフィックのフィルタリングが開始されます。

財務部門PC1から始めて、サーバールームにあるServer0に属するIPアドレス192.168.1.194にpingを送信してみます。ご覧のとおり、pingは問題なく成功しています。管理部門のLaptop0のラップトップにも正常にpingを送信しています。最初のパケットはARPにより破棄され、残りの3つは自由にpingされます。



トラフィックフィルタリングを整理するために、R2ルーターの設定に進み、グローバル構成モードをアクティブにして、モダンな外観のACLのリストを作成します。従来のACL 10もあります。最初のリストを作成するには、紙に書いたのと同じリスト名を指定する必要があるコマンドを入力します。ipaccess-list standard ACL Secure_Ma_And_Se。その後、システムは可能なパラメータのヒントを提供します。拒否、終了、いいえ、許可、またはコメントを選択し、1〜2147483647のシーケンス番号を入力することもできます。そうしないと、システムによって自動的に割り当てられます。



したがって、この番号は入力しませんが、この許可は特定のPC0デバイスに対して有効であるため、すぐにpermit host 192.168.1.130コマンドに進みます。ワイルドカードマスクを使用することもできます。次に、その方法を示します。

次に、deny 192.168.1.128コマンドを入力します。 / 26があるため、後方マスクを使用して、コマンドにそれを追加します：拒否192.168.1.128 0.0.0.63。したがって、192.168.1.128 / 26のネットワークトラフィックを拒否します。

同様に、次のネットワークからのトラフィックをブロックします。拒否192.168.1.0 0.0.0.127。他のすべてのトラフィックは許可されるため、permit anyコマンドを入力します。次に、このリストをインターフェイスに適用する必要があるため、int s0 / 1/0コマンドを使用します。次に、ip access-group Secure_Ma_And_Seと入力します。システムは、インターフェイスを選択するヒントを提供します-着信パケット用と発信パケット用です。 ACLを出力インターフェイスに適用する必要があるため、ip access-group Secure_Ma_And_Se outコマンドを使用します。

PC0コマンドラインに移動して、Server0に属するIPアドレス192.168.1.194にpingを送信します。 PC0トラフィックに特別なACL条件を使用したため、pingは成功しました。 PC1から同じことを行うと、他のアカウンティングIPアドレスからのトラフィックがサーバーアクセスに対してブロックされるため、システムは「宛先ホストを利用できません」というエラーを出します。

ルーターのR2 CLIにログインし、show ip address-listsコマンドを入力すると、財務部門ネットワークのトラフィックがどのようにルーティングされたかを確認できます。これは、許可に従ってpingがスキップされた回数と、禁止に従ってブロックされた回数を示しています。



いつでもルーターの設定にアクセスして、アクセスリストを確認できます。これにより、タスクNo.1とNo.4の条件を満たす。もう1つお見せしましょう。何か修正したい場合は、R2設定のグローバル構成モードに入り、ip access-list standard Secure_Ma_And_Seコマンドを入力してから、コマンド「host 192.168.1.130 is not allowed」-no permit host 192.168.1.130を入力します。



アクセスリストをもう一度見ると、10行目が消え、20、30、40行しか残っていないことがわかります。このようにして、ルーターの設定でACLアクセスリストを編集できますが、それがクラシックな形式でない場合に限られます。

次に、3番目のACLに移りましょう。これは、R2ルーターにも関係しているためです。 Laptop3からのトラフィックは営業部門のネットワークから出てはならないということです。この場合、Laptop2は問題なく財務部門のコンピュータと通信する必要があります。これを確認するには、このラップトップからIPアドレス192.168.1.130にpingを実行し、すべてが機能することを確認します。

次に、Laptop3のコマンドラインに移動し、アドレス192.168.1.130にpingを送信します。 pingは成功しましたが、タスクの条件により、Laptop3は販売部門の同じネットワーク上にあるLaptop2とのみ通信できるため、Pingは必要ありません。これを行うには、従来の方法を使用して別のACLを作成します。

R2設定に戻り、permit host 192.168.1.130コマンドを使用して、削除されたレコード10を回復しようとします。このエントリがリストの最後の50番に表示されていることがわかります。ただし、特定のホストの解決を示す行がリストの最後にあり、すべてのネットワークトラフィックを禁止する行がリストの一番上にあるため、アクセスはいずれにしても機能しません。 PC0コンピューターからmanagement0ラップトップLaptop0にpingを実行しようとすると、ACLに50の許可レコードがあるにもかかわらず、「宛先ホストが利用できません」というメッセージが表示されます。

したがって、既存のACLを編集する場合は、R2モードでno permit host 192.168.1.130コマンド（config-std-nacl）を入力し、リストから行50が消えていることを確認して、10 permit host 192.168.1.130コマンドを入力する必要があります。リストが元の形式になり、このエントリが最初の行になったことがわかります。シーケンス番号は、リストを任意の形式で編集するのに役立ちます。そのため、最新のACLフォームは、従来のものよりもはるかに便利です。



次に、ACL 10の従来の形式がどのように機能するかを示します。従来のリストを使用するには、access – list 10？コマンドを入力し、プロンプトに従って、目的のアクション（拒否、許可、またはコメント）を選択する必要があります。次に、行access – list 10 deny hostを入力し、その後にコマンドaccess – list 10 deny 192.168.1.3を入力して、後方マスクを追加します。ホストがあるため、直接サブネットマスクは255.255.255.255で、戻り値は0.0.0.0です。その結果、ホストトラフィックをブロックするには、access – list 10 deny 192.168.1.3 0.0.0.0コマンドを入力する必要があります。その後、権限を指定する必要があります。そのために、コマンドaccess – list 10 permit anyを入力します。このリストはルーターR2のG0 / 1インターフェースに適用する必要があるため、コマンドは常にg0 / 1、ip access-group 10 inに入力します。使用するリストがクラシックかモダンかに関係なく、このリストをインターフェイスに適用することは、同じコマンドで行われます。

設定が正しいことを確認するには、Laptop3コマンドラインターミナルに移動し、IPアドレス192.168.1.130にpingを送信しようとします。ご覧のように、システムは宛先ホストが利用できないことを報告します。

show ip access-listsコマンドとshow access-listsを使用してリストを確認できることを思い出してください。ルータR1に関連する別の問題を解決する必要があります。これを行うには、このルータのCLIに移動し、グローバルコンフィギュレーションモードを開始して、ip access-list standard Secure_Ma_From_Seコマンドを入力します。ネットワーク192.168.1.192/27があるため、そのサブネットマスクは255.255.255.224になります。つまり、リバースマスクは0.0.0.31であり、deny 192.168.1.192 0.0.0.31コマンドを入力する必要があります。他のすべてのトラフィックが許可されているため、リストはpermit anyコマンドで終わります。 ACLをルータの出力インターフェイスに適用するには、ip access-group Secure_Ma_From_Se outコマンドを使用します。



次に、サーバーのServer0コマンドラインターミナルに移動し、IPアドレス192.168.1.226でManagement0 Laptop0にpingを送信します。試行は失敗しましたが、アドレス192.168.1.130にpingを送信すると、接続は問題なく確立されます。つまり、サーバーコンピューターが管理部門との通信を禁止し、他の部門の他のすべてのデバイスとの通信は許可されました。したがって、4つの問題すべてを正常に解決しました。

別のことをお見せしましょう。 R2ルーターの設定に進みます。ここには、クラシックとモダンの2種類のACLがあります。 ACL 10、標準IPアクセスリスト10を編集するとします。これは、クラシック形式では2つのエントリ10と20で構成されています。doshow runコマンドを使用すると、最初に4つのエントリの最新のアクセスリストがあり、一般的な見出しSecure_Ma_And_Seおよびその下の番号は、同じアクセスリスト10の名前が繰り返されている、クラシックフォームの2つのACL 10エントリです。



たとえば、deny host 192.168.1.3エントリを削除して別のネットワークからデバイスのエントリを入力するなど、いくつかの変更を行う場合、このエントリに対してのみdeleteコマンドを使用する必要があります。noaccess-list 10 deny host 192.168.1.3。しかし、このコマンドを入力するとすぐにACL 10のすべてのエントリが完全に消えてしまうため、ACLの従来の外観は編集に非常に不便です。最新の録音方法は無料で編集できるため、使用するのにはるかに便利です。

このビデオチュートリアルの内容を学ぶために、もう一度確認して、プロンプトを表示せずに自分で問題を解決することをお勧めします。 ACLはCCNAコースの重要なトピックであり、ワイルドカードマスクを作成するなどして、多くの人が困惑しています。マスク変換の概念を理解するだけで十分であり、すべてがはるかに簡単になるでしょう。 CCNAコースのトピックを理解する上で最も重要なことは、実践のみが特定のシスコの概念を理解するのに役立つため、実践的なトレーニングであることを忘れないでください。練習は私のチームのコピー＆ペーストではなく、自分のやり方で問題を解決することです。自分自身に質問してください。ここから条件を適用する場所までのトラフィックの流れをブロックするために何をする必要があるかなどを確認し、それらに答えてみてください。
