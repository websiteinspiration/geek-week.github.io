スプリングアクチュエータとは？

アプリケーションを開発して本番環境にデプロイしたら、そのパフォーマンスを監視することが非常に重要です。これは、アプリケーションの障害がビジネスに直接影響する銀行システムなどの重要なアプリケーションに特に当てはまります。

従来、Spring Actuatorの前は、アプリケーションの状態をテストするためのコードを記述する必要がありましたが、Spring Actuatorではコードを記述する必要がありませんでした。Spring Actuatorは、アプリケーションの監視に役立ついくつかのエンドポイントを提供します。

Spring ActuatorをMavenまたはGradleプロジェクトに追加する方法

メイベン

<dependencies>
  <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-actuator</artifactId>
   </dependency>
</dependencies>

Gradle

dependencies {
  compile("org.springframework.boot:spring-boot-starter-actuator")
}

デフォルトでは、を除くすべてのエンドポイントが有効になっていshutdownます。エンドポイントを有効にするには、ファイルで次のプロパティを使用しますapplication.properties。

management.endpoint.<code><</code>id<code>></code>.enabled

翻訳者の注：デフォルトでは、すべてのエンドポイントへのアクセスはJMXを介してのみ行われ、HTTPを介したすべてのエンドポイントへのアクセスはありません（以下を参照）。

エンドポイントを有効にするにはshutdown、ファイルに次のエントリを作成する必要がありますapplication.properties。

management.endpoint.shutdown.enabled=true

management.endpoint.info.enabled=true

プリセットインジケーター

スプリングブートアクチュエーターには、アプリケーションのさまざまな部分のヘルスをテストするために、多くの自動構成された「ヘルスインジケーター」（HeathIndicators）があります。例えば、DiskspaceHealthIndicatorディスク容量情報を提供します。MongoDBを使用MongoHealthIndicatorする場合は、Mongoデータベースをチェックし（サーバーが実行中かどうかに関係なく）、関連情報を表示します。デフォルトでは、アプリケーションの最終ステータスHealthAggregatorによって、それぞれが提供するステータスのリストを並べ替えるだけが決まりますHealthIndicator。ソートされたリストの最初のステータスは、アプリケーションの最終ステータスとして使用されます。

すべてのプリセットインジケーター

の無効化上記の「ヘルスインジケーター」はデフォルトで有効になっていますが、次のプロパティを使用して無効にすることができます。

management.health.defaults.enabled=false

別のインジケーターを無効にする別

の方法として、HealthIndicatorたとえば以下に示すように、別のインジケーターを無効にして、ディスク容量チェックを無効にすることができます。

management.health.diskspace.enabled=false

注：識別子HealthIndicatorは、サフィックスのないBean名になりますHealthIndicator。

例：

DiskSpaceHealthIndicator	diskspace
MongoHealthIndicator		mongo
CassandraHealthIndicator	cassandra
DataSourceHealthIndicator	datasource

など...

インジケーターの記述（HealthIndicator） Spring Boot Actuatorによって提供される

組み込みのものに加えて、HealthIndicator独自のステータスインジケーターを作成できます。これを行うには、以下に示すように、インターフェースを実装するクラスを作成し、HealthIndicatorそのメソッドhealth()を実装しHealthて、関連情報を含む回答として返す必要があります。

import org.springframework.boot.actuate.health.Health;
import org.springframework.boot.actuate.health.HealthIndicator;
import org.springframework.stereotype.Component;

@Component
public class CustomHealthIndicator implements HealthIndicator {

 @Override
 public Health health() {
  int errorCode = 0; 
  // In the above line,I am simple assigning zero,but you can call Health check related code like below commented line and that method can return the appropriate code.
  // int errorCode = performHealthCheck();
  if (errorCode != 0) {
   return Health.down().withDetail("Error Code", errorCode).build();
  }
  return Health.up().build();
 }

}
