言語を選択：ORMの選択

Goは、その利点を含め、最も単純なプログラミング言語の1つとして有名になりました。コードの読み書きが容易になり、ほとんどの場合、コードベースのサポートが単純になります。Goのいくつかの利点について説明します。そのおかげで、SimbirSoftは、Webサービスとマイクロサービス（SOA）の両方のさまざまなアーキテクチャを備えた高負荷のプロジェクトの数でGoを使用しました。

応用


このシステム言語は、ほとんどのネットワークタスク、すべてのマイクロサービス、マクロサービス、Webサービス、およびシステムツールに適しています。多くのDevOpsエキスパートがこのツールを愛するのは、理由がないわけではありません。Goは機械学習での使用を見出し、一部の開発者はそれを使用してWebサイトを作成しています。私たちの実務では、保険会社などの内部使用のために個別のCRMとソフトウェア製品を開発したときに、私たちはGoを選びました。標準のGo言語リポジトリには、モバイル開発用のパッケージもあります。このパッケージは引き続きサポートされていますが、より適切なツールがあります。

Go：開発と認識


Googleでは、2007年にGo言語の開発が始まりました。それ以前は、Java、Python、C ++を使用していましたが、特定の欠点（特にメモリ消費量が多い）または使用が制限されていました。それからグーグルは彼らのニーズのために新しい言語を開発し始めました。

言語の要件は何ですか：

  • 1つの実行可能ファイル（依存関係なし）
  • 高速コンパイル
  • 高速ランタイム（少なくともJavaおよびC ++と同等）
  • リソースリークを最小限に抑えるガベージコレクター
  • マルチプロセッサシステムの最適化
  • 静的型付け
  • ミニマリズム
  • 構文の軽さ
  • マルチパラダイムネス（命令的、機能的、オブジェクト指向）

Goを使用しない場合は、作成の歴史を簡単に思い出してください。Googleは2007年に言語仕様を考え始めました。リリース1.0は2012年に、2015年にはリリース1.5がリリースされましたが、すでに1行のC ++コードがありません。その瞬間から、言語は自動的にコンパイルされるため、Goでの個々のタスクの開発を加速することができました。

コミュニティで新しい言語を認識する平均年齢は約10歳ですが、現在Goは最も人気のある言語の1つです。TIOBEインデックスによると、2019年6月に、Goは世界での人気度で15位に上昇し、年間3つのポジションを追加しました。その成功の理由には、次のものがあります。

  • 強力な標準ライブラリ
  • クールな標準チューニング
  • 最も必要な機能は「ボックス」に表示され、ライブラリはオープンソースに提供されます
  • とてもフレンドリーなコミュニティ

GoのORM


Goの弱点の1つはORMであると考えられています。おそらくこれは、言語が若いという事実によるものです-その中で、多くの必要なツールが開発され始めたばかりです。特に、database / sqlパッケージを使用してデータベースを操作できます。パッケージの要件は何ですか？

  • すべてのSQLデータベースの一般パッケージ
  • ポータブル：データベースを操作する分野の機能は含まれていません。理想的には、ドライバーを変更した場合、すべてが正しく機能するはずです。
  • Goタイプを標準のデータベースタイプに変換する（特定のタイプを処理できるようにする必要があります）
  • 接続プールが含まれています
  • ゴルーチンのスレッドセーフ

データベース/ SQLを使用したサンプルSELECTクエリ


defer rows.Close()
for rows.Next() {
	var message string
	err := rows.Scan(&message)
	if err != nil {
		log.Panic(err)
	}
	fmt.Println(message)
}
if err := rows.Err(); err != nil {
	log.Panic(err)
}

例からわかるように、多くのテンプレートアクションを毎回実行する必要があります。例えば：

  • 行オブジェクトを閉じる：行を延期します。Close（）
  • エラーのチェックerr：= rows.Err（）; err！= nil since 行がなぜなのか明確ではありません。Next（）がfalseを返しました

以上のことから、ORMなど、より普遍的でシンプルなものを使用したいと考えています。

ORM実装のタイプ


開発者がGoを操作するためのORMの選択に直面した場合、それらはしばしばgithubの評価によって導かれます。私たちが自分でORMを選択したとき、Beegoは最高の評価（18,000の「星」）を獲得しました。ただし、実際にはORMはフレームワークの一部であるため、評価は完全に客観的ではありません（ORMのみを選択した場合）。また、指導者の間では、2013年から開発を続けているGormがトランザクションをサポートし、他にも多くの利点があります。



ただし、Gormでの経験が示したように、Gormを使用すると、特にテスト時に特定の問題が発生する可能性があります。たとえば、データの競合や外部キーの削除などです。Gormでこのようなエラーが発生する原因は次のとおりです。

  • マスターですぐにコミット
  • タグがありません
  • 予測不可能なSQLビルダー

現時点では、私たちの作業では、インターフェイス{}を使用しないいくつかのORMを好みます。これらはKallax、Reform、Sqlboilerです。通常、移行にはGolangパッケージを使用し、GoでORMベンチマークを使用してorm-b​​enchリポジトリを使用します。同時に、ベンチマークのみを追跡するべきではありません。多くの場合、速度はコードとその機能の品質ほど重要ではありません。

要約する


開発者と企業にとってのGoの利点には、開発の生産性の向上、少ないサーバーを使用できること、新しい開発者の構文の容易さと迅速なトレーニング、多数のツールとフレンドリーなコミュニティが含まれます。それは若い言語であるという事実にもかかわらず、それは常に成長し、発展しています。Go機能は、ネットワークタスクからマイクロサービス、ブロックチェーンまで、さまざまなプロジェクトに最適な言語です。
