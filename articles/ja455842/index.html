単一リンクリスト


リンクリストは、基本的なデータ構造の 1つです。その各要素（またはノード）は、実際には、格納されたデータと隣接する要素へのリンクで構成されています。単一リンクリストには、構造内の次の要素へのリンクのみが格納され、二重リンクリストには、次および前へのリンクが保持されます。このようなデータの編成により、それらを任意のメモリ領域に配置できます（たとえば、配列、すべての要素を次々にメモリに配置する必要がある場合とは異なります）。

もちろん、リストについてはもっと多くのことを言う必要があります。リストは循環する（つまり、最後の要素が最初の要素へのリンクを格納する）か、または循環しない（つまり、最後の要素へのリンクがない）場合があります。リストを入力できます。同じタイプのデータが含まれているかどうか。等々。

コードを書いてみてください。たとえば、どういうわけかリストノードを想像できます。

final class Node<T> {
    
    let payload: T
    var nextNode: Node?
    
    init(payload: T, nextNode: Node? = nil) {
        self.payload = payload
        self.nextNode = nextNode
    }
    
}

「ジェネリック」は、フィールドにpayload任意のタイプのペイロードを格納できるタイプです。

リスト自体は、最初のノードによって完全に識別されます。

final class SinglyLinkedList<T> {
    
    var firstNode: Node<T>?
    
    init(firstNode: Node<T>? = nil) {
        self.firstNode = firstNode
    }
    
}

リストが空の可能性があるため、最初のノードはオプションとして宣言されています。

理論的には、もちろん、クラスでは、挿入、削除、ノードへのアクセスなど、必要なすべてのメソッドを実装する必要がありますが、別の機会に実装します。同時に、struct（Appleが私たちの例で積極的に推奨している）使用がより良い選択であるかどうかを確認し、おそらく「コピーオンライト」アプローチを思い出してください。

最初の方法。サイクル


今日私たちがここにいるビジネスに取り掛かる時が来ました！そしてそれに対処する最も効果的な方法は2つの方法です。最初は、最初のノードから最後のノードまでの単純なループです。

サイクルは3つの変数で動作します。これらの変数には、開始前に前のノード、現在のノード、次のノードの値が割り当てられています。 （この時点では、前のノードの値は当然空です。）サイクルは、次のノードが空でないことを確認することから始まり、空の場合は、サイクルの本体が実行されます。ループで魔法は発生しません。現在のノードでは、次の要素を参照するフィールドに前の要素へのリンクが割り当てられます（最初の反復で、リンクの値はそれぞれリセットされ、これは最後のノードの状態に対応します）。さらに、前のノード、現在のノード、次のノードに対応する変数には、新しい値が割り当てられます。ループを終了した後、現在の（つまり、一般的に最後の反復可能）ノードには、次のノードへの新しいリンク値が割り当てられます。サイクルからの脱出はその瞬間に発生しますリストの最後のノードが現在になるとき。

つまり、言うまでもなく、これらはすべて奇妙で理解しにくいように聞こえるので、コードを見たほうがよいでしょう。

extension SinglyLinkedList {
    
    func reverse() {
        var previousNode: Node<T>? = nil
        var currentNode = firstNode
        var nextNode = firstNode?.nextNode
        
        while nextNode != nil {
            currentNode?.nextNode = previousNode
            previousNode = currentNode
            currentNode = nextNode
            nextNode = currentNode?.nextNode
        }
        currentNode?.nextNode = previousNode
        
        firstNode = currentNode
    }
    
}

2番目の方法。再帰


同じUターンを実行する2番目の既知の方法は、再帰に基づいています。これを実装するために、リストの最初のノードを取り、「新しい」最初のノード（以前の最後のノード）を返す関数を宣言します。

この関数内では、空になるまで（つまり、リストの最後に到達するまで）後続の各ノードで何度も呼び出されるため、パラメーターと戻り値はオプションです。したがって、関数の本体では、関数が呼び出されたノードが空かどうか、そしてこのノードに次のものが含まれているかどうかを確認する必要があります。そうでない場合、関数は引数に渡されたものを返します。

実際、アルゴリズム全体を正直に言葉で表現しようとしたのですが、結果がわかりにくいので、結局ほとんど全部消してしまいました。フローチャートを描き、アルゴリズムのステップを正式に説明するために-この場合も、Swiftコードを読んで理解しようとするだけの方が私とあなたにとって便利なので、この場合も意味がないと思います。

extension SinglyLinkedList {
    
    func reverseRecursively() {
        func reverse(_ node: Node<T>?) -> Node<T>? {
            guard let head = node else { return node }
            if head.nextNode == nil { return head }
            
            let reversedHead = reverse(head.nextNode)
            head.nextNode?.nextNode = head
            head.nextNode = nil
            
            return reversedHead
        }
        
        firstNode = reverse(firstNode)
    }
    
}

呼び出し自体の便宜上、アルゴリズム自体は実際のリストのタイプのメソッドによって「ラップ」されます。

調査結果


逆転メソッドを注意深く見ると（または呼び出しカウントで実験を行うと）、最初のケースのループと2番目のケースの内部（再帰）メソッドの呼び出しは、リストのノード数（この場合は9）の1倍少ないことがわかります。時間）。また、最初のケースではループの周囲で何が起こるか（同じ割り当てシーケンス）、2番目のケースでは最初の非再帰的なメソッド呼び出しにも注意を払うことができます。どちらの場合も、10個のノードのリストに対して「円」が正確に10回繰り返されることがわかります。したがって、両方のアルゴリズム-O（n）の線形複雑度があります。

一般的に言えば、この2つのアルゴリズムは、この問題の解決に最も効果的であると考えられています。計算の複雑さに関しては、低い値のアルゴリズムを考え出すことはできません。何らかの方法で、各ノードに「アクセス」して、リンク内に保存されている値に新しい値を割り当てる必要があります。

言及する価値のある別の機能は、「割り当てられたメモリの複雑さ」です。両方の提案されたアルゴリズムは、固定数の新しい変数を作成します（最初のケースでは3つ、2番目のケースでは1つ）。これは、割り当てられたメモリの量が入力データの量的特性に依存せず、定数関数-O（1）によって記述されることを意味します。

しかし、実際には、2番目のケースではそうではありません。再帰の危険性は、スタック上の再帰呼び出しごとに追加のメモリが割り当てられます。この場合、再帰の深さは入力データの量に対応します。
