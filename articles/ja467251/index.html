今日公開する翻訳の第1部である資料の著者は、なぜ1959年に登場した言語であるCOBOLが依然として非常に広まっているのかという疑問に対する答えを見つけるでしょう。

なぜCOBOLはまだ生きているのですか？


この質問への伝統的な答えは深く皮肉です。組織は怠惰、無能、そして愚かさです。彼らは安さを追い求めており、ソフトウェアシステムを最新のものに書き換えることに投資する傾向はありません。一般に、このような多数の組織の作業がCOBOLに依存している理由は、慣性と近視眼力の組み合わせにあると考えられます。そしてもちろん、これにはいくつかの真実があります。紛らわしいコードを大量に書き直すのは大変な作業です。高いです。それは複雑です。また、既存のソフトウェアがうまく機能しているように見える場合、このソフトウェアを更新するプロジェクトに投資する意欲は特にありません。

これはすべてそうです。しかし、私がIRSで働いていたとき、COBOL退役軍人はJavaでコードを書き直そうとしたことについて話し、Javaが計算を正しく実行できないことがわかりました。

それは私には非常に奇妙に聞こえました。奇妙なことに、私はすぐに警戒心を抱きました。「主よ、それは、IRSが50年にわたって皆への納税を四捨五入してきたことを意味します!!!」 COBOLがIRSに必要な数学的計算に関してJavaを回避できるとは信じられませんでした。結局、彼らは人々を宇宙に送り出しませんでした。

夏にCOBOLを学ぶことの興味深い副作用の1つは、次のことを理解し始めたことです。重要なのは、Javaが数学的な計算を正しく実行できないことではありません。重要なのは、Javaが計算を正しく行う方法です。そして、Javaで計算がどのように実行されるか、およびCOBOLで同じことがどのように実行されるかを理解すると、多くの組織がコンピューターのレガシーを取り除くことが非常に困難であると感じる理由を理解し始めます。

どの「i」を点線化する必要がありますか？


COBOLの話から少し離れて、データのバイナリ表現が事実上の標準になる前に、コンピュータがどのように情報を格納したかについて話します（ただし、 z / OS インターフェースの使用方法に関する資料。これは特別なものです）。私たちの問題を検討する上で、この方向でメインテーマから逸脱することは有益だと思います。上記の資料では、バイナリスイッチを使用して数値を2進数、3進数、10進数で格納したり、負の数を格納したりするさまざまな方法について説明しました。私が十分に注意しなかった唯一のことは、10進数がどのように格納されるかでした。

独自の2進数コンピューターを設計している場合は、2進数システムを使用することを決定することから始めます。ポイントの左側のビットは整数を表します-1、2、4、8。そして右側のビット-小数-1 / 2、1 / 4、1 / 8 ...


2.75バイナリ表現

ここでの問題は、小数点自体を格納する方法を理解することです（実際、私は「バイナリポイント」と言う必要があります-結局、2進数について話しているからです）。これは「コンピュータの錬金術」のようなものではないので、浮動小数点数と固定小数点数について私が何を話しているかはよくわかります。浮動小数点数では、2進小数点はどこにでも配置できます（つまり、「浮動」できます）。ポジションポイントは指数として保存されます。ポイントを移動する機能は、そのような機会がない場合に利用できるよりも広い範囲の数値を格納することを可能にします。小数点を数値の一番後ろに移動し、すべてのビットを選択して、非常に大きな数値を表す整数値を格納できます。ポイントを数値の前に移動して、非常に小さな値を表すことができます。しかし、この自由は正確さを犠牲にしています。前の例の2.75のバイナリ表現をもう一度見てみましょう。 4から8への遷移は、4分の1から1/8への遷移よりもはるかに多くなります。以下に示すように例を書き直すと、これを想像する方が簡単かもしれません。


私は目で数字の間隔を選びました-私の考えを実証するためだけです。

数字の違いは自分で簡単に計算できます。たとえば、1/16と1/32の間の距離は0.03125ですが、1/2と1/4の間の距離はすでに0.25です。

どうしてそれが重要ですか？整数のバイナリ表現の場合、これは問題ではありません。バイナリレコードの隣接する数値間の距離は、正確さを失うことなく、適切なビットの組み合わせで埋めることによって簡単に補正できます。しかし、小数の表現の場合、それはそれほど単純ではありません。隣接する数字の間の「穴」を「埋める」としようとすると、何かがこれらの穴に「落ちる」（実際には落ちる）可能性があります。これは、バイナリ形式では分数の正確な表現を取得することができないという事実につながります。

これは、数値0.1（1/10）の古典的な例で示されています。この数値をバイナリ形式で表す方法は？ 2 -1は1/2、つまり0.5です。これは、やりすぎ。 1/16は0.0635です。これは少なすぎます。 1/16 + 1/32は既により近いですが（0.09375）、1/16 + 1/32 + 1/64はすでに必要以上です（0.109375）。

これらの考慮事項を無期限に続けることができると信じるなら-あなたは正しい- それがそうである方法です。

ここで、自分に次のように言うことができます。「数値1を保存するのと同じ方法で0.1を保存しないのはなぜですか？数値1は問題なく保存できます。整数を格納するのと同じ方法で、小数点を削除して数値を格納しましょう。」

これは、バイナリ/小数点を所定の場所に固定する必要があることを除いて、この問題の優れた解決策です。それ以外の場合、10.00001と100000.1はまったく同じに見えます。しかし、ポイントが固定されていて、たとえば、2桁が数値の小数部分に割り当てられている場合、100000000を10.00に丸めることができ、100000.1は100000.10になります。

固定小数点数を「発明」しただけです。

固定小数点数を使用してさまざまな値を表現することで、私たちはそれを理解しました。簡単です。固定小数点数を使用して、他のいくつかの問題の解決を促進することは可能ですか？ここで私たちの親友について思い出してみましょう-2進数の10進数について（Binary Coded Decimal、BCD）。ちなみに、これらの数値は、ほとんどの科学計算およびグラフ計算機で使用されています。明らかなこれらのデバイスから、彼らは正しい計算結果を期待しています。

固定小数点数は、数の間の「穴」が一定であるという事実と、十分なスペースがない数を想像する必要がある場合にのみ丸めが発生するため、より正確であると見なされます。しかし、浮動小数点数を使用する場合、同じ量のメモリを使用して非常に大きな数と非常に小さな数を表すことができます。確かに、彼らの助けを借りて、アクセス可能な範囲内のすべての数値を正確に表すことは不可能であり、「穴」を埋めるために丸めに頼らざるを得ません。

COBOLは、デフォルトで固定小数点数が使用される言語として作成されました。しかし、これは数学計算を実行するためにCOBOLが現代の言語よりも優れていることを意味しますか？ 0.1 + 0.2の値を計算した結果のような問題に遭遇した場合、前の質問には「はい」と答えるべきだと思われるかもしれません。しかし、それは退屈でしょう。それでは次に進みましょう。

いわゆるMuller's Recurrence関係を使用してCOBOLを実験します。Jean-Michel Mullerはフランスの科学者で、情報技術の分野で大きな科学的発見をした可能性があります。彼は数学を使ってコンピュータの正しい操作を壊す方法を見つけました。彼は信頼性と正確さの問題を研究していると彼が言うだろうと私は確信しているが、彼はコンピューターを「破壊する」数学的問題を作成している。これらのタスクの1つは、その繰り返し式です。次のようになります。

以下は、浮動小数点数と固定小数点数を使用してミュラーの再帰関係の結果を計算する小さなPythonスクリプトです。

from decimal import Decimal
def rec(y, z):
 return 108 - ((815-1500/z)/y)
 
def floatpt(N):
 x = [4, 4.25]
 for i in range(2, N+1):
  x.append(rec(x[i-1], x[i-2]))
 return x
 
def fixedpt(N):
 x = [Decimal(4), Decimal(17)/Decimal(4)]
 for i in range(2, N+1):
  x.append(rec(x[i-1], x[i-2]))
 return x
N = 20 
flt = floatpt(N)
fxd = fixedpt(N)
for i in range(N):
 print str(i) + ' | '+str(flt[i])+' | '+str(fxd[i])

おそらく、誰もがそのような大規模な再帰的な計算を実行することはありそうもないと考えています。しかし、これがまさに、パトリオットのミサイル制御システムが時間を誤って計算したとき、28人の死に至った 1991年の災害の原因でした。浮動小数点の計算が誤って多くの害を及ぼすことが判明しました。ここに素晴らしいものがありますおそらく、高性能コンピューティングは、間違った答えを得るためのより速い方法です。ここで説明されている問題に関する詳細情報を取得し、他の例を確認したい場合は、この作業を読んでください。

問題は、コンピューターに搭載されているRAMの容量が無限ではないことです。したがって、無限小数（または2進数）の位置を無限に格納することは不可能です。固定小数点の計算は、浮動小数点の計算よりも、使用する形式よりも多くの数値が必要になる可能性が低いと確信できる場合は、浮動小数点の計算よりも正確になります。数値がこの形式に適合しない場合、四捨五入されます。固定小数点計算も浮動​​小数点計算も、ミュラーの再帰関係が示す問題から保護されないことに注意してください。結果として、それと他の両方は誤った結果を与えます。問題はこれがいつ起こるかです。たとえば、Pythonスクリプトの反復数を20から22に増やすと、最終的な数は、固定小数点での計算で得られる0.728107になります。 23反復？ -501.7081261。 24？ 105.8598187。

さまざまな言語で、この問題はさまざまな方法で現れます。COBOLのように、パラメータが厳密に設定されている数値を操作できるものもあります。たとえば、Pythonには、コンピューターに十分なメモリがある場合に構成できるデフォルト値があります。getcontext().prec = 60デフォルトで行われるように、ピリオドの後に28桁ではなく60桁を使用することをPython 10進モジュールに通知することにより、プログラムに行を追加すると、プログラムはエラーなしでMuellerの反復関係の40回の反復を完了することができます。
