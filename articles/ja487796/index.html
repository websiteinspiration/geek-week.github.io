Eclipse開発環境がJava開発で最も人気のある環境の1つであることは間違いありません。多くのプラグインがEclipseで使用できるという事実は、Eclipseを優れたツールにします。たとえば、GitHub、SVN、CVSなどからコードをダウンロードするためのプラグインなど、さまざまなタイプのタスクや他の必要なツールとの統合のための数百のEclipseプラグインがあります。

Javaでの開発、Pythonでの開発、SpringでのJavaアプリケーションの作成、Eclipse自体でのAndroidアプリケーション用の特別なプラグインがあります。適切なプラグインをインストールするだけで生産性を上げることができました。

たとえば、EGitプラグインをインストールすると、Eclipseから直接Gitリポジトリを管理できます。他のGitクライアントは必要ありません。同様に、Spring Toolsを追加することで、SpringおよびSpring-Bootでのアプリケーションの開発を容易にすることができます。

私は、Eclipseを完全に使用しなかった多くのJava開発者に会いました。多くのJava開発者は、他のJavaプロジェクトと同様にSpringでアプリケーションを作成します。これは基本的には優れていますが、最適ではありません。 Springツールを使用してSpringでアプリケーションを開発することにより、生産性を向上させることができます。

1. EGit、EclipseのGit統合


今日、これはおそらくJava開発者にとって最も必要なプラグインです。GitHubからコードをダウンロードして、EclipseのGit統合を提供します。このプラグインにより、履歴クエリの検索と実行も迅速かつ多目的に行えます。つまり、EGitはJava開発に欠かせないものです。



2. Springツール（別名Spring IDEまたはSpring Tool Suite）


そこ春は、最も人気のJavaフレームワークであることは疑いあり、そして春のツールは、それが簡単にEclipseで春と春のブートにプロジェクトを作成することができます。STSを使用すると、単純な統合start.spring.ioを使用して、Spring Bootでプロジェクトをすばやく作成できます。

また、Spring Java-Configを使用したアプリケーション開発、高度なコード補完、コンテンツアシスト、検証、Springアプリケーションのクイックフィックスサポートもサポートしています。クラウドでのデバッグを含め、Cloud FoundryのIDEを統合できるため、Springを使用したマイクロサービスの開発に最適です。



3. Maven Integration for Eclipse（M2E）


M2EプラグインまたはMaven Integration for Eclipseプラグインは、Java開発に必要なもう1つの人気のあるEclipseプラグインです。Eclipseの包括的なMaven統合を提供します。

M2Eを使用して、シンプルおよびマルチモジュールの両方のMavenプロジェクトを管理し、Eclipseインターフェースを介してMavenビルドを実行し、Mavenリポジトリーと対話することができます。

さらに、一部のプラグインは、使用しているEclipseのバージョンに依存します。Juno、Lunaなどには、個別のプラグインがあります。



4. サブクリップ


これは、SVNからコードをダウンロードし、Eclipse自体からSVNに関連するすべての操作を実行できるようにする別のバージョン管理プラグインです。これは、Subversion開発者によって設計および保守されています。プラグインは、Subversionの最新の機能とリリースに従って常に更新されています。SVNでJavaプロジェクトに取り組んでいる場合は、このプラグインが必要です。



5. Eclipseカラーテーマ


すべての開発者は異なる好みを持っています。古い醜いカラーテーマEclipseを好む開発者もいれば、VimやIntelliJ IDEAなどの暗いテーマを好む開発者もいます。
Eclipse Color Themeを使用すると、副作用なしに便利にカラーテーマを切り替えることができます。色のテーマを変更したり、いくつかのプログラミング言語で一度に作業したりしたい場合は、このプラグインを使用すると便利です。



6. JBossツール


JBoss Toolsは、JBoss、およびHibernate、JBoss AS / WildFly、CDI、OpenShift、Apache Camel、Red Hat JBoss Fuse、Docker、JSF、（X）などの関連テクノロジーのサポートを含む、一連のEclipseプラグインの包括的なプロジェクトです。 HTML、Maven、その他。



7. Eclipse用のTestNG


JUnitがEclipseに付属していることがわかっていても、TestNGを使用してJavaプロジェクトのユニットテストを作成する場合、このEclipseプラグインが役立ちます。EclipseからTestNGテストを実行できます。セット、グループ、または個別のメソッドを実行できます。バグレポートは別のタブにあり、失敗したテストにすばやくジャンプできます。

プラグインには、テストを作成するためのいくつかのテンプレートも含まれています。



8. Eclipse（ADT）用のAndroid開発ツール


Android開発ツール（ADT）は、Androidアプリケーションを開発するための強力な統合環境を提供するように設計されたEclipse IDEのプラグインです。

ADTは、新しいAndroidプロジェクトの迅速な作成、UIアプリケーションの作成、Android SDKツールを使用したアプリケーションエラーの修正、アプリケーションパッケージ（APK）の署名とその配布のエクスポートを可能にすることで、Eclipseの機能を拡張します。



9. EclEmma


EclEmmaは、Eclipse用の無料のJavaコードカバレッジツールであり、Eclipse Public Licenseの下で利用できます。Eclipseプロダクション環境でコードカバレッジ分析を直接提供します。適切に使用すると、迅速な開発/テストサイクルが高速化され、コードの品質が向上します。



10. JRebel for Eclipse


JRebelは、アプリケーションをその場でデプロイしてから変更されたクラスやその他のリソースを開発者が再ロードできるようにするツールです。これは、Java開発に典型的な再構成、再起動、および再配備のサイクルをスキップします。JRebelを使用すると、開発者はより多くの作業を同時に行うことができ、コードの作成中にスプレーする必要がなくなります。

JRebelは、ほとんどの既存のエンタープライズJavaスタックをサポートし、既存の開発環境に簡単にインストールできます。
