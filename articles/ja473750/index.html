警告：ビデオ内のプロジェクトは、ステレオビジョンと「頭の回転」に関する記事のチュートリアルに従って実行できるサンプルにすぎません。 xboxリモコン付きのタンクは含まれていません。

明快さにもかかわらず、ビデオでのプロジェクトの作者の貧弱な物語とリンクの可用性は、それがすべてどのように機能するかを理解するためにすぐに簡単ではありません。同様の何かをより便利な形式で収集したい場合は、読むことをお勧めします。

*すぐに予約します。YouTubeの動画に写っている人は私に慣れていません。秘密のデータを送信していません。彼のプロジェクトの現在の状態はわかりません。

したがって、私たちは2つのことに関心があります。

  • ヘルメットの電話でステレオ画像を取得する方法;
  • 頭の回転でサーボを制御する方法。

ビデオで使用されている概念は、要約すると次のようになります。

  • 2ラズベリーpiは、webrtcサービスを介してカメラからネットワークにビデオストリームを送信します。
  • 電話（ヘルメット内）は、電話の2つの同一のアプリケーション（フロートアプリ）でストリームを受け入れます。
  • 同時に、電話はラズベリーに接続されたサーバーを制御します。

すべてが簡単です。しかし、ご存じのように、ディアブロは詳細と不便さ、すなわち：

  • 2つのラズベリーを実行し、2つのカメラの設定を監視する必要があります。パワーラズベリー* 2。
  • フロートアプリは常に電話でスライドします。画面上の画像を揃える必要があります。
  • ...

したがって、私たちはステレオピに転送します。幸いにも、ロシアのストアに表示されます（この投稿の後に消えないことを願っています）：



ステレオピは、現在積極的に市場を征服している同国の開発です。

その魅力は名前に由来しています-2つのCSIラズベリーpiカメラを同時に接続できます。同時に、これらすべてが1つのラズベリーpi計算モジュールに基づいて機能します。残念ながら、モジュール自体はキットに含まれていません。自分で購入する必要があります。

Stereopiについては、Habréに関する記事があります。

それから、2つのビデオストリームとGPIOによるサーバーの制御が必要です。
Stereopiのベースとして、Raspberry Pi Compute Module 3+を使用します。

電話からラズベリーのサーボを制御するには、ラズベリーで実行されるコードと電話でのアクションが必要です。

しかし、最初に、サーボを決定します。 YouTubeビデオは、gpioラズベリーに直接接続されたサーボを使用しています。サーボは低電力であるため、おそらく2つのサーボをgpioラズベリーに掛けることができます。これらのトリックは、sg-90サーバーで簡単に実行できます。彼らは栄養を要求していませんが、特に負荷にはあまり適していません。原則として、ステレオピからの2台のカメラでサスペンションを保持するのに十分なはずです。ジンバル自体は、「pan-tilt」という検索で同じaliexpressで購入できます。ただし、これらのサーボには重大なマイナスもあります。「恐怖で震えます」。 YouTubeビデオの作者が観察するのはこの効果です。なぜこれが起こっているのか、それをどうするのかはここでは考慮しません。

コードがどこから来たのか、他に何が修正できるのかを理解するために、コメントがテキストに残されています。
コードの一般的な意味は次のとおりです。

  • 開始時、サーボは中間位置に設定されています。
  • サーボの信号線が掛かるピンが3本（gpio）あります。この例では、2つのピン（2つのサーボのサスペンション）です。
  • gpioは、1000〜2000のPWM範囲の信号を適用することによって制御されます。
  • jsonによって解析される電話からラインが到着し（他のことを実行できます）、値xおよびyがそこから取得されます。次に、これらの値は、サーボを回転させるためにPWM値に変換されます。

*問題は、xがyのように0から360（電話がその軸を中心に回転する）の値を取ることです。そして、これらの値は、1000から2000までの値を取るPWMにバインドする必要があります。コードは、pitchPW = key1 * 5 + 500という式を使用します。500は最小PWMサーボ値です（ただし、コードでは1000と想定しています）。また、5を掛けることは条件付きです。x = 360では、PWM値が最大値の何倍も高いため、この点を改善する必要があります。サーボは、損傷を避けるために最大回転角度を超えないように保護されていますが、これはあまり好ましくありません。
