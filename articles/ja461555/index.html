Googleスプレッドシートの視覚化

Excelは当初から、プログラマー以外のユーザーやセミプログラマー（私のような）にとって最も人気があり、用途の広い計算自動化ツールになっています。かつて、私はVBAで多くのことを書いたり、大学での学生コース執筆のためのスクリプトを作成したり、論文の燃焼中の固体燃料の熱分布の強力な反復計算を書き留めたりして、20分間コンプを麻痺させました。今日、クラウドとWebソリューションの時代に、このニッチはGoogleスプレッドシートに引き継がれています。

最近、視覚化を操作するためにGoogleテーブルの簡単な調査を行いました。研究の過程で、いくつかの興味深いアイデアと1つの脚本が生まれました。カットの下で、私の研究の結果。


私は建設管理と計画の分野で働いており、建設現場では、Ixelよりも複雑なものが根付いていないことがほとんどです。したがって、Googleのテーブルには、ビルダーの心（またはおそらくは心）を獲得するあらゆる機会があります。スプレッドシートで既に標準となっているいくつかの方法は、前任者からGoogleによって採用されているため、一見するとisxelとの違いはわずかです。同時に、いくつかの機能は新しい形式で実装されており、私の意見ではそのような成功した実装の1つは配列数式です。彼らもイクセルにいましたが、私はそれらを見て、「どこにそれらが必要かわからないことを神に感謝します」と言いたかった。グーグルでは、彼らと一緒に仕事をするのは素晴らしいことであり、彼らはまだ多くのタスクで彼らの有用性を証明すると思います。それらを使用して建物の視覚化を操作する方法を示します。

これをすべてビデオ形式で示す方が簡単なので、小さなアドオンを作成しました。


そして、ここでは、パラメーター参照を使用して、視覚化とさまざまな表記法をリンクできるようにするスクリプトコードを示します。

/**
 *        
 * @param {"A5:D20"} arr    
 * @param {"A21:D25"} sprav  ()
 * @param {""} param    
 * @customfunction
 */
function (arr, sprav, param) {
  if(typeof sprav=="object"&&sprav.length!=undefined) {
    if(typeof arr=="object"&&arr.length!=undefined) {
      for (var i = 0; i<arr.length; i++){
        for (var j= 0; j<arr[i].length; j++){
          if (arr[i][j] != ""){
            var r = sprav.map(function(value){return value[0]}).indexOf(arr[i][j]);
            var c = sprav[0].indexOf(param);
            if (r!=-1 && c!=-1) {arr[i][j] = sprav[r][c]}
          }
        }
      }
    }  
  }
  return  arr;
}

おそらく、コードの最適化は標準に達していないかもしれませんが、私はまだPROプログラマーではありませんが、うまくいきます。

もちろん、話題になっている人たちは、これはすべてゴミであると言っています。長い間、BIMはすでに存在しています。はいあります。ただし、BIMを実装するよりも、SketchUpでモデルにテーブルをすばやく固定します（Googleもそうです）。建材メーカーやデザイナーからビルダーやメンテナンスにいたるまですべてがBIMで機能するわけではありませんが、BIMの作業には時間がかかります。そして私たちの国では、これはすぐには起こりません。

最後にGoogleについて。製品の品質からすると、アナログよりはるかに優れており、その表面グーグルによって銀河全体が放出されます。しかし、これほどまでに考え抜かれた作業ロジック、高品質のクロスデバイス、スクリプトを使用して機能を拡張する機能に出会ったことはありません。しかし、これは私の意見にすぎず、私が間違っている場合は、あなたの意見を共有してください。
