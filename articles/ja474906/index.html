Xamarin.Forms-SkiaSharpによる装飾的なQRCodeマッピング



バーコードを出力/読み取るための人気のあるZXingライブラリがあります。それは多くの異なるフォーマットを出力して読むことができます：QRCode、Aztec、その他、2ダース以上。コードを読み取るための準備ができているコントロールZXingScannerViewがあります。この機能をアプリケーションに追加するには、最小限のコードが必要です。

ただし、このライブラリの助けを借りて、コードは白黒で標準的に表示されます。たとえば、色付き、丸みを帯びた要素で装飾したり、他の方法で装飾したりするなど、カスタム外観のコードを描写する必要がある場合を考えてみてください（そのため、読みやすくなります）。

そして、創造性の本当の自由は、あなたが自分でコードを描いた場合にのみ開きます-そして、すべてが完全にあなたの手に渡ります。これは、QRコードの例で行うことです。

完成したアプリケーションはこちらです。

この記事では、記事から引用したアルゴリズムを実装しています。QRコード生成アルゴリズム、SkiaSharpライブラリを使用した描画に触れ、装飾的な要素を備えたQRコードジェネレーターの適用例を示します。

実装の説明


空のXamarin.Formsアプリケーションを作成することから始めましょう。 "モバイルアプリ（Xamarin.Forms）"などのVisual Studioで新しいプロジェクトを作成し、[空白]テンプレートを選択します。 「プラットフォーム」セクションでいずれかを選択すると、SkiaSharpはAndroidやiOSを含む多くのプラットフォームで動作します。

.Androidおよび.iOSプロジェクトは変更されていません。クロスプラットフォームの実装があります。

クロスプラットフォームプロジェクトでは、
2つの背景画像が追加されます。1つはページ全体用、もう1つはQRコード用です。ビルドアクションの「組み込みリソース」があることに注意してください。
出力はMainPage.xamlファイルとMainPage.xaml.csファイルにあります
。xamlのリソースの画像を使用するためのImageResourceExtensionクラスです。

すべてのロジックはQRCodeEncoderプロジェクトにあります。クラスに分かれています：

  • エンコーダー-データ文字列を1と0のシーケンスにエンコードします
  • レンダラー-結果のデータ配列を正しく配置し、オーバーヘッド情報を追加して、PNGファイルをストリームに返します

QRコード生成アルゴリズムには、番号付きの参考書が多数あります。それらすべての意味については、「QRコード生成アルゴリズム」の記事を参照してください（上記を参照）。

漢字エンコーディング（文字用）はアプリケーションに実装されていません。

スキアサープ


SkiaSharpは、.NET用のクロスプラットフォーム2D描画ライブラリです。これは、GoogleのSkia Graphics Libraryに基づいています。NuGetパッケージとして利用可能：

nuget install SkiaSharp

Microsoftドキュメント：Xamarin.FormsのSkiaSharp

すべての描画は、RendererクラスのDrawメソッドで行われます。たとえば、一部の要素は丸く、一部は正方形です。

canvas.DrawCircle()
canvas.DrawRect()

一部は強調表示されています：

var paint1 = new SKPaint { IsAntialias = true, Style = SKPaintStyle.Fill, Color = SKColors.DeepSkyBlue };
var paint2 = new SKPaint { IsAntialias = true, Style = SKPaintStyle.Fill, Color = SKColors.Red };
var paint3 = new SKPaint { IsAntialias = true, Style = SKPaintStyle.Fill, Color = SKColors.Gold };

また、隣接する要素間の遷移をスムーズにする方法を示します。
