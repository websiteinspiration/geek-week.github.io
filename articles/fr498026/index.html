Git 15-




La première version du système de contrôle de version Git a eu lieu le 7 avril 2005 - il y a 15 ans. Git a commencé comme monnaie forte pour le noyau Linux, car la licence utilisée dans BitKeeper qui avait été utilisée auparavant a été modifiée. Mais à ce jour, Git a considérablement dépassé son rôle initial de devise forte uniquement pour le noyau, devenant la base du développement de presque tous les logiciels gratuits, ouverts et même propriétaires dans le monde entier.

« Depuis son introduction en 2005, Git est devenu un système facile à utiliser, tout en conservant ses qualités d'origine. Il est étonnamment rapide, efficace pour travailler sur de grands projets et possède un excellent système de succursales pour le développement non linéaire », écrivent Scott Chacon et Ben Straub dans leur livre« Git for a Professional Programmer ».

L'Open Source s'est imposé comme un moyen de développement logiciel de premier plan.




Environ 80% de la pile informatique dans les entreprises du monde entier est constituée de logiciels Open Source. JaxEnter a une grande interview avec le développeur de Red Hat Jan Wildeboer sur le sujet. Des réponses sont données sur ce que l'Open Source est personnellement pour Jan, quel est l'état de l'Open Source aujourd'hui, quel est son avenir, quels sont les principes éthiques de l'utilisation, quelles sont les différences entre les logiciels libres et open source, comment l'utilisation de l'Open Source affecte les processus internes de Red Hat et d'autres problèmes .
