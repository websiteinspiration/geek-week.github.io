インセット


insets という用語は、Android Lollipopの時代にはいつでもステータスバーの作業領域を常に使用しようとしたため、Android開発者の間で恐怖を引き起こしています。たとえば、StackOverflowに関するこの古い質問には、非常に多くのビューがあります。

インセットは、ナビゲーションバーやステータスバーなど、画面のどのコンポーネントがシステムインターフェイスと交差するかを示します。交差は単にコンテンツの上に表示することを意味するかもしれませんが、ここではシステムのジェスチャーに関する情報も取得できます。インセットを使用して、たとえば、viewエッジから離れるように移動することで、競合を解決しようとすることができます。

Androidでは、インセットはクラスで表されますWindowInsets、およびAndroidXクラスWindowInsetsCompat内。Android Q以降、アプリケーション画面の作成に使用できる5種類のインセットがあります。どちらのタイプをinset使用するかは状況によって異なりますので、各タイプを個別に見てみましょう。

安定したインセット


メソッド：getStableInsets（）

安定したインセットは、Androidで利用可能な最後のタイプのインセットです。これらはジェスチャーの管理には特に重要ではありませんが、それらについて話すことも価値があると私は判断しました。

安定したインセットは、システムウィンドウのインセットを指しますが、システムインターフェースをアプリケーションの上に表示できる場所を示しており、原則として表示される場所ではありません。安定したインセットは主に、システムインターフェースがその表示をオンまたはオフにできるように構成されている場合に使用されます。たとえば、リーンバックモードや没入型モード（ゲーム、写真やビデオプレーヤーの表示など）を使用する場合などです。

インセットの処理


さまざまなタイプのインセットについて理解を深めていただければ幸いです。それでは、アプリケーションでそれらをどのように使用できるかを見てみましょう。WindowInsets

にアクセスするための主なメソッドは、setOnApplyWindowInsetsListenerメソッドです。ナビゲーションバーの背後に表示されないようにパディングを追加する例を見てみましょう。view

ViewCompat.setOnApplyWindowInsetsListener(view) { v, insets ->
    v.updatePadding(bottom = insets.systemWindowInsets.bottom)
    // Return the insets so that they keep going down the view hierarchy
    insets
}

ここではview、システムウィンドウinsetのボトムインデントの値にボトムインデントを調整するだけです。

注：ViewGroupでこれを行う場合は、おそらくインストールする必要があります android:clipToPadding="false"。これは、すべてのタイプのクリップ描画がデフォルトでインデントされているためです。この属性は一般的にRecyclerViewで使用されます。これについては、次の記事で詳しく説明します。

リスニング機能がべき等であることを確認してください。同じインセットで複数回呼び出された場合、結果は毎回同じになります。非べき等関数の例を以下に示します。
