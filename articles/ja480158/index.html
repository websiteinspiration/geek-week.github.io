Java、Kotlin RestControllerを調理できますか？

彼の人生のほとんどすべてのJavaプログラマーがRestControllerを作成しましたが、彼がそれを正しく行うかどうか疑問に思う人はほとんどいません。あなたが経験豊富なプログラマーであっても、私が答えようとする質問があるかもしれません。この記事では、Spring Bootバージョン1.5や2.0、Quarkusなどのフレームワークについて説明します。最近登場した、Red HatのSpring Bootのライバルです。




問題


JavaとSpring Boot 1.5で長い間プログラムされました。しかし、新しいプロジェクトを書く必要がありました：

  1. 1600行を統合するためのjsonがあります。一部のクラスには100フィールドがあります
  2. KotlinとQuarkusを試してみたかった。
  3. アノテーションなしで、ロンボクマジックを使用せずに、kotlinデータクラスを操作できるRESTコントローラを記述します。データクラスを小さくしたい

おそらく、kotlinデータクラスは不変クラスまたは不変クラスであると推測しました。コンストラクターにすべてのフィールドが含まれるクラス。私はそのような概念を大いに信じています。クラスを作成した後は、クラスを変更することはできません。セッターはありません。Dockerイメージは世界では変更できないため、コントローラーに分類される日付クラスは変更できません。

問題を解決するための可能な方法、より正確には、現代のプロジェクトでコントローラーをどのように作成できるかを見てみましょう。

標準オプション。手動でスプリングまたはスプリングブート1.5


Restコントローラーはずっと前に登場しました。次の例は、すべてのチュートリアルにあるJavaでの典型的な使用例です。

そしてPOJO（プレーンな古いjavaオブジェクト）を作成します

問題：

不変にできますか？

2番目のオプションは、救いとしてのスプリングまたはスプリングブート1.5 +アノテーションです。


したがって、不変のクラスが必要であることに気付き、Jacksonを使用していることがわかりました。それからanatationationが救助に来ます。

@Getter
public class Fruit {
    private String name;
    private String description;

    @JsonCreator
    public Fruit(@JsonProperty("name") String name, @JsonProperty("description")String description) {
        this.name = name;
        this.description = description;
    }
}

現在、sping boot 1.5に関するプロジェクトでは、これらの注釈は文字通りすべてに満ちています。
また、人気のjsonschema2pojoジェネレーターを使用すると、さらに多くのアノテーションが生成されます。正直なところ、私は好きではありません。

そこでコピーしてみてください：

{
	"description": "description",
	"name": "name"  
}

次のような出力が表示されます（目を細めたり、めくったりできます）。

@JsonInclude(JsonInclude.Include.NON_NULL)
@JsonPropertyOrder({
        "description",
        "name"
})
public class Example {

    @JsonProperty("description")
    private String description;
    @JsonProperty("name")
    private String name;

    @JsonProperty("description")
    public String getDescription() {
        return description;
    }

    @JsonProperty("description")
    public void setDescription(String description) {
        this.description = description;
    }

    @JsonProperty("name")
    public String getName() {
        return name;
    }

    @JsonProperty("name")
    public void setName(String name) {
        this.name = name;
    }
}

短所：アノテーションはクラスを大きく膨らませます。アマチュアにとっては非常に冗長です。

4番目のオプションスプリングブート2.0


@Getter
public class Fruit {
    private String name;
    private String description;
    public Fruit( String name, String description) {
        this.name = name;
        this.description = description;
    }
}

驚いたことに、Spring Boot 2.0はこのような不変のクラスで静かに動作します。そして彼の双子の兄弟のコトリンデータクラスも

data class Fruit(
 val name : String,
 val description : String)

ランタイムのjavaはコンストラクターのパラメーター名を認識していないように見えますが、何らかの理由で、spring boot2はすでにデータクラスの操作方法を認識しています。そこで、spring-boot-starter-parentを見てください。Kotlinのサポートがそこに追加されています。

<plugin>
             <groupId>org.jetbrains.kotlin</groupId>
...
              <configuration>
                        <javaParameters>true</javaParameters>  
              </configuration>
</plugin>
<plugin>
              <artifactId>maven-compiler-plugin</artifactId>
              <configuration>
                        <parameters>true</parameters>
              </configuration>
</plugin>

解読します。ためには、実行時に失われることがないよう、クラスのコンストラクタのパラメータ名のため、コンパイラは合格しなければならないのjavac -parametersフラグを。そして春ブーツ2.0は、これを行います。
