科学および工学計算におけるRPAの適用

前書き


学校では、知識を統合するために、多くの同様の例を解くように求められました。私たちはいつもイライラしていました：何がそんなに貴重なのですか？数式の2つまたは3つの値を代入して、答えを取得します。思考の流れはどこにありますか？現実は学校よりも厳しいことがわかった。

現在、私はITアナリストとして働いています。 IT分野に参加する前は、ヒートエンジニア、CNCプログラマーとして働き、研究プロジェクトに参加していました。

私自身の経験から、エンジニアと科学者は作業時間の95％をそのような「同様の行動」に費やしていると確信しました。方程式の計算、チェック、結果の記録、仕様のコピー。プロジェクトごと、実験ごと、毎日。

これが私の過去の作品の例です。

2019年まで、熱真空成形用のモックアップを作りました。そのようなモデルが加熱されたプラスチックで包まれている場合、このモデルの形状を正確に繰り返す製品が得られます。ここに技術の説明。

レイアウトの生産サイクルでは、高度に専門化されたアプリケーションのセット全体が必要です。

  • 3Dモデリング用のAutodesk Inventor;
  • ワークピースサイズをアップロードするためのExcel。
  • レイアウトのコストを計算するためのExcel;
  • NC制御プログラムをコンパイルするためのHSMモジュール。
  • プログラムファイルを管理するためのコンピュータファイルシステム。
  • CNCマシンを制御するためのMach3環境。

手動で環境から環境にデータを転送する必要があり、これらはテーブル全体と値の配列です。プロセスは遅く、エラーが頻繁に発生しました。

これに先立ち、光ファイバー（リンク）の設計・製造に携わりました。多くの研究、設計、計算がありました。熱工学と照明工学の計算（Ansys、Dialux）に特化した環境に加えて、費用対効果の計算に加えて、モデルと図面のAutocadとInventorです。そして、同じ困難があります。あるアプリケーションの計算結果を、次の計算のために別のアプリケーションにドラッグする必要があります。そして、最適な解決策を探すために数回。

エンジニアの時間と科学者の時間は非常に高価な時間です。これは給与についてではありません。エンジニアの計算の背後には、チームによる大きなプロジェクトがあります。科学者の研究の背後には、業界全体の展望があります。しかし、多くの場合、高度な資格を持つスペシャリストは、概念を開発し、モデル化し、結果を解釈し、論争を引き起こし、同僚とブレインストーミングするのではなく、プログラム間で価値を「愚かに」中断します。

今日のビジネス環境の特徴はスピードです。市場は常に押し上げています。 2014年には、レイアウトの作成に2〜3週間かかりました。 2018年-3日、そしてそれはすでに長すぎるように見えました。今、設計者は同時に複数のソリューションを発行する必要があり、以前は1つのオプションしかなかった。

そしてもう一つ-投資とリスク。プロジェクトを「フック」するには、企業は顧客との契約を締結する前に、このプロジェクトのコストの最大6％を概念開発に投資する必要があります。これらの資金はなくなります：

  • 研究用;
  • コンセプチュアルデザイン;
  • 人件費の評価;
  • スケッチ等の作成

同社はそれらをポケットから取り出しますが、これは独自のリスクです。コンセプトに注意を払うには専門家の時間がかかり、日常業務で忙しい。

IT企業の仕事のツールに慣れた後、私はビジネスプロセスの自動化からどのような実践がエンジニアに役立つか興味を持った。そのため、ビジネスは長い間、ルーチン処理にプロセスロボティクス（RPA）を使用してきました。

RPAメーカーは、この自動化ツールの次の利点を主張しています。

  1. 普遍性（ロボットは任意のアプリケーション、任意のデータソースで動作できます）;
  2. 開発の容易さ（プログラミングと管理の深い能力は必要ありません）;
  3. 開発速度（既製のアルゴリズムは従来のプログラミングよりも時間がかかりません）;
  4. 日常業務からの従業員の実際の荷降ろし。

これらの基準に基づいて、工学/科学計算でRPAを使用する効果を確認します。

説明の例


簡単な例を考えます。貨物で固定された片持ち梁があります。

エンジニアの視点と科学者の視点からこの問題を見てみましょう。

ケース「エンジニア」：2 mの長さに固定された片持ち梁があり、3倍の安全マージンで500 kgの荷重を保持する必要があります。ビームは長方形のパイプでできています。 GOSTカタログに従ってビーム断面を選択する必要があります。

ケース「科学者」：荷重の質量、断面、および梁の長さがこの梁の支持力にどのように影響するかを調べます。回帰方程式を導き出します。

どちらの場合も、ビームの質量に比例してビームに作用する重力が考慮されます。

最初のケースである「エンジニア」について詳しく説明します。ケース「科学者」も同様の方法で実装されます。

技術的には、この例は非常に単純です。そして、対象の専門家はそれを計算機で簡単に計算することができます。私たちの別の目標は、タスクが大規模になったときにRPAソリューションがどのように役立つかを示すことです。

簡略化すると、パイプセクションは、角を丸めず、溶接を考慮しない理想的な長方形になります。

エンジニアの仕事


ケース「エンジニア」の一般的なスキームは次のとおりです。

  1. Excelシートには、GOSTに基づくパイプの品揃えのテーブルがあります。
  2. この表の各エントリについて、Autodesk Inventorで3Dモデルを作成する必要があります。
  3. 次に、Inventorの応力解析環境で強度計算を実行し、計算結果をhtmlにアップロードします。
  4. 結果のファイルで「最大ミーゼス応力」という値を見つけます。
  5. 安全係数（材料の降伏強度と最大フォンミーゼス応力の比）が3未満の場合、計算を停止します。

適切なセクションのビームは3倍の安全マージンを提供し、他のオプションの中で重量が最小になると信じています。



全体として、私たちのタスクでは、スペシャリストが3つのアプリケーションを処理します（上の図を参照）。実際の環境では、アプリケーションの数はさらに多くなる可能性があります。

GOST 8645-68「長方形鋼管」には、300のエントリが含まれています。デモタスクでは、リストを短くします。各サイズファミリから1つの位置をとります。合計19レコードから、1つを選択する必要があります。



モデルを作成して強度計算を行うInventorモデリング環境には、完成した材料のライブラリが含まれています。次のライブラリからビームの材料を受け入れます：

材料-鋼
密度7.85 g / cu。 CM;
降伏強さ207 MPa;
引張強度345 MPa;
ヤング率210 GPa;
せん断弾性率80.7692 GPa。

これは、読み込まれた梁の3次元モデルがどのように見えるかです。



そして、これが強度計算の結果です。システムは、ビームの脆弱な領域を赤で着色します。これらの場所では、緊張は最大です。左側のスケールは、ビーム材料の最大応力値を示しています。


 

作業の一部をロボットに転送します


作業スキームは次のように変更されます。



ロボットをAutomation Anywhere Community Edition（以降、AA）環境で組み立てます。評価基準を見て、主観的な印象を説明しましょう。

汎用性


RPAソリューション（特に商用ソリューション）は、ビジネスプロセスを自動化し、オフィスの従業員の作業を自動化する手段として永続的に位置付けられています。例やトレーニングコースでは、ERP、ECM、Webとの相互作用を分析します。すべてが非常に「オフィス」です。

最初は、AAがAutodesk Inventorからインターフェイスとデータを取得できるかどうかについて疑問がありました。しかし、すべてが実際に機能しました。各要素、各コントロールが決定され、記録されました。パラメータテーブルのあるサービスフォームでも、ロボットは、マウスの方向から目的のセルに簡単にアクセスできました。

次は、強度計算スタジオの立ち上げによるテストでした。また、問題ありません。この段階では、システムが計算の完了を期待している場合、アクション間の一時停止を慎重に処理する必要がありました。

Webからサマリーデータを取得してExcelに貼り付ける作業はスムーズに進みました。
このタスクの一環として、普遍性が確認されています。他のRPAベンダーの説明から判断すると、汎用性はこのソフトウェアカテゴリの真の共通プロパティです。

簡単に学べる


開発には数晩かかりました：コース、ケーススタディ-これだけです。多くのRPAベンダーには無料のトレーニングがあります。唯一の障壁：環境インターフェースとAAコースは英語のみです。

開発スピード


私たちは夕方に「エンジニアタスク」のアルゴリズムを開発してデバッグしました。アクションのシーケンスは、ちょうど44の命令に適合します。以下は、完成したロボットとのAutomation Anywhereインターフェースの一部です。低コード/コードなしの概念-プログラムする必要はありませんでした。彼らはオペレーションレコーダーを使用するか、コマンドライブラリからドラッグアンドドロップしました。次に、プロパティウィンドウで設定を構成します。



定期的な荷降ろし


ロボットは1つのレコードを処理するのに1分20秒かかります。ロボットなしで1つのレコードの処理に費やした時間とほぼ同じです。

何十、何百というレコードについて話していると、その人は必然的に疲れ、気が散り始めます。専門家が突然他の仕事を引き受けるかもしれません。人の場合、「タスクにA分かかる場合、そのようなN個のタスクはA * N分で完了することができます」という形式の比率は機能しません。常に時間がかかります。

この例では、ロボットは最大のセクションから始めて、レコードを順番に反復します。大規模な配列では、これは簡単な方法ではありません。加速の場合、ニュートン法や半除算などの逐次近似を実装できます。

科学者の仕事


科学者のタスクは、いくつかの数値実験を行って、ビームの断面積、長さ、負荷の質量に応じてビームの支持力が変化する法則を決定することです。見つかった法則は、回帰方程式の形式で定式化されます。

回帰方程式を正確にするために、科学者は大量のデータを処理する必要があります。

この例では、入力変数の配列が割り当てられます。

  • パイププロファイルの高さ;
  • 幅;
  • 壁の厚さ;
  • ビーム長;
  • 貨物の質量。

各変数の少なくとも3つの値の計算を行う必要がある場合、合計で243回の繰り返しになります。1回の反復に2分の期間があるため、合計時間はすでに8時間、つまり1営業日です。より完全な調査のために、3つの値ではなく、10以上の値を取る必要があります。

調査により、モデルに追加の要素を含める必要があることが明らかになります。たとえば、さまざまなグレードの鋼を「駆動」します。計算量は数十倍、数百倍に増えます。

実際のタスクでは、ロボットは科学者を数日解放することができ、専門家はこれを使って出版物を準備します。これは科学者の活動の主要な指標です。

概要


エンジニアの「製品」は、実際に機能するデバイス、デザインです。計算のロボット化は、プロジェクトのより深い研究（より多くの計算、より多くのモード、より多くのオプション）によるリスクを減らします。

科学者の「産物」は、方程式、規則性、またはその他の簡潔な説明です。そして、それが正確であればあるほど、分析に含まれるデータが多くなります。 RPAソリューションは、モデルの情報「食品」を形作るのに役立ちます。

例を要約します。

橋梁モデル、エンジンモデル、暖房システムモデルなど、あらゆるモデルが計算モデルの役割を果たすことができます。スペシャリストは、モデルのすべてのコンポーネントが互いに適切に相互作用していること、およびモデルが「外部」の主要な変数パラメーターのセットを提供していることが必要です。

コンピューティング環境の役割は、専門家が作業で使用するすべてのアプリケーションによって実行されます。Ansys、Autocad、Solidworks、FlowVision、Dialux、PowerMill、Archicad。または、独自の設計、たとえば、製造工場でファンを選択するためのプログラム（Systemair機器選択プログラムを参照）。

データソースとして、ウェブサイト、データベース、Excelワークシート、txtファイルを検討します。
作業の最終結果-レポート-は、自動的に生成されたテキストを含むWord文書、Excelグラフ、スクリーンショットのセット、または電子メールの送信です。

RPAは、エンジニアリング分析が適用できる場所であればどこでも適用できます。ここにいくつかの領域があります：

  • 強度計算と変形;
  • 水力およびガス力学;
  • 熱交換;
  • 電磁気;
  • 学際的分析;
  • ジェネレーティブデザイン;
  • NC制御プログラム（入れ子など）。
  • 医学的および生物学的研究;
  • フィードバックのあるシステムまたは非定常システムの計算（最終結果をソースデータに転送して計算を繰り返す必要がある場合）。

今日、RPAソリューションはビジネスで積極的に使用され、プロセスを自動化してデータを処理します。オフィスの従業員、エンジニア、科学者の日常には多くの共通点があります。私たちは、ロボットが工学および科学活動に適していることを示しました。

感想をまとめます。

  1. 汎用性-はい、RPAは汎用的なツールです。
  2. 習得が簡単-はい、シンプルで手頃な価格ですが、言語が必要です。
  3. 開発の速度-はい、アルゴリズムは高速になります。特に、レコーダーを使って作業する場合はそうです。
  4. ルーチンからのアンロード-はい、大規模なタスクで本当に役立ちます。
