前書き


ここでは、型注釈を使用して記述されたコードがどのように見えるかの古典的な例を見つけることができます。

これが通常のコードです：

def greeting(name):
    return 'Hello ' + name

注釈が追加されるコードは次のとおりです。

def greeting(name: str) -> str:
    return 'Hello ' + name

型注釈付きのコードを作成するためのテンプレートは次のようになります。

def function(variable: input_type) -> return_type:
    pass

一見すると、コードに型注釈を適用することは単純で簡単に見えます。しかし、開発者コミュニティには、型注釈とは何かを正確に理解する上で、依然としてかなりの不確実性があります。さらに、それらを正しく呼び出す方法、つまり「注釈」または「ヒント」、およびコードベースでの使用による利点にはあいまいささえあります。

このトピックを調査し始め、型注釈を使用する必要があるかどうかを考えたとき、私は完全に混乱しました。その結果、私はいつもと同じように理解することにしました。私はこの問題を深く研究し、この資料の形で私の研究を発表することにしました。これは、私と同じように、Pythonで型注釈を扱いたい人に役立つと思います。

コンピュータはどのようにプログラムを実行しますか？


Python開発者が型注釈で何を達成しようとしているのかを理解するために、Pythonコードより数レベル下のコンピューターシステムのメカニズムについて話しましょう。これのおかげで、一般的な用語で、コンピューターとプログラミング言語がどのように機能するかをよりよく理解することができます。

プログラミング言語は、中核として、中央処理装置（CPU）を使用してデータを処理し、処理する必要のあるデータと処理の結果得られたデータをメモリに格納できるようにするツールです。


簡略化されたコンピュータ回路

プロセッサは、本質的に、非常に「愚か」なものです。彼はデータで印象的なアクションを実行できますが、電気信号のセットに要約される機械命令しか理解していません。機械語は、ゼロと1で構成されるものとして表すことができます。
これらのゼロとプロセッサが理解できるものを準備するには、コードを高水準言語から低水準言語に変換する必要があります。ここにコンパイラとインタープリタが入ります。

言語がコンパイルまたは実行可能である場合（Pythonコードはインタープリターによって実行されます）、この言語のコードは、低レベルのコンピューターコンポーネント、つまりハードウェア用の命令を含む低レベルのマシンコードに変わります。

一部のプログラミング言語で記述されたコードを、マシンが理解できるコードに変換する方法はいくつかあります。プログラムコードでファイルを作成し、コンパイラーを使用してそれをマシンコードに変換するか（これはC ++、Go、Rust、およびその他のいくつかの言語で機能します）、またはコードをマシンコマンドに変換するインタープリターを使用してコードを直接実行します。このようにして、インタプリタの助けを借りて、Pythonのプログラムが、PHPやRubyなどの他の「スクリプト」言語で起動されます。

型システムの紹介


昔、遠い遠い銀河では、人々の数学的な計算を行ったが、手動で、彼らは数字や数式の要素を持つ「型」を比較した場合、彼らはこれらの要素についての証明を導出する際に表示される論理エラーの数を減らすことができることに気づきました。

当初、コンピュータサイエンスは本質的に大量の手動計算の実行に限定されていたため、これらの古い原則の一部はこれらの計算に移されました。型システムは、さまざまな変数または要素に適切な型を割り当てることにより、プログラムのエラーの数を減らすために使用されるツールになっています。

静的型付け言語と動的型付け言語のデータ型の比較


静的型付け言語と動的型付け言語のデータ型の違いは何ですか？

静的型付けを使用する場合、事前に型を宣言する必要があります。たとえば、Javaで作業している場合、プログラムは次のようになります。

public class CreatingVariables {
  public static void main(String[] args) {
    int x, y, age, height;
    double seconds, rainfall;

    x = 10;
    y = 400;
    age = 39;
    height = 63;

    seconds = 4.71;

    rainfall = 23;

    double rate = calculateRainfallRate(seconds, rainfall);
  
  }
private static double calculateRainfallRate(double seconds, double rainfall) {
  return rainfall/seconds;
}

プログラムの始まりに注意してください。そこではいくつかの変数が宣言されており、その隣にこれらの変数のタイプの表示があります：

int x, y, age, height;
double seconds, rainfall;

さらに、型は関数の宣言時と引数の宣言時の両方で指定されます。これらの型宣言がないと、プログラムをコンパイルできません。Javaプログラムを作成するときは、最初から、これらまたはそれらのエンティティーがどのタイプを持つかを計画する必要があります。その結果、コンパイラーは、そのようなプログラムのコードを処理するときに、マシンコードを生成するプロセスで何をチェックする必要があるかを正確に把握します。
