はタイプセーフな言語です。タイプを操作するときはいつでも、明示的に指定する必要があります。たとえば、複数の型で機能する関数が必要です。 SwiftにはタイプAnyとがありますAnyObjectが、常に使用する必要はなく、慎重に使用する必要があります。を使用するAnyとAnyObject、コンパイル時に型の不一致を追跡することが不可能になるため、コードが脆弱になります。これはジェネリックが助けになるところです。

ジェネリックコードを使用すると、コンパイル時に型の安全性を確保しながら、特定の制限を満たす任意の型で機能する再利用可能な関数とデータ型を作成できます。このアプローチにより、重複を回避し、その機能を明確な抽象的な方法で表現するコードを記述できます。たとえば、のようなタイプArrayでSet、Dictionaryジェネリックを使用してアイテムを格納します。

整数値と文字列で構成される配列を作成する必要があるとしましょう。この問題を解決するために、2つの関数を作成します。

let intArray = [1, 2, 3, 4]
let stringArray = [a, b, c, d]
func printInts(array: [Int]) {
  print(intArray.map { $0 })
}
func printStrings(array: [String]) {
  print(stringArray.map { $0 })
}

次に、float型の要素の配列またはユーザーオブジェクトの配列を出力する必要があります。上記の関数を見ると、型の違いだけが使用されていることがわかります。したがって、コードを複製する代わりに、再利用するための汎用関数を作成できます。

ジェネリック関数


ジェネリック関数は、任意のユニバーサルタイプパラメータで機能しますT。型名はそれがどうあるべきかについては何も言いませんが、それが何であるかにかかわらず、両方の配列は型でなければならないことを示しています。代わりに使用するタイプ自体は、関数が呼び出されるたびに決定されprint(_:)ます。

func print<T>(array: [T]) {
  print(array.map { $0 })
}

ジェネリック型


これらは、配列や辞書と同様に、任意のタイプで機能するカスタムクラス、構造、および列挙です。

スタックを作ろう

import Foundation

enum StackError: Error {
    case Empty(message: String)
}

public struct Stack {
    var array: [Int] = []
    
    init(capacity: Int) {
        array.reserveCapacity(capacity)
    }
    
    public mutating func push(element: Int) {
        array.append(element)
    }
    
    public mutating func pop() -> Int? {
        return array.popLast()
    }
    
    public func peek() throws -> Int {
        guard !isEmpty(), let lastElement = array.last else {
            throw StackError.Empty(message: "Array is empty")
        }
        return lastElement
    }
    
    func isEmpty() -> Bool {
        return array.isEmpty
    }   
}

extension Stack: CustomStringConvertible {
    public var description: String {
        let elements = array.map{ "\($0)" }.joined(separator: "\n")
        return elements
    }
}

var stack = Stack(capacity: 10)
stack.push(element: 1)
stack.push(element: 2)
print(stack)
stack.pop()
stack.pop()

stack.push(element: 5)

stack.push(element: 3)
stack.push(element: 4)
print(stack)

ジェネリック型の制限


ジェネリックはどのタイプでもかまいませんので、それを使って多くを行うことはできません。ジェネリック関数またはジェネリック型で使用できる型に制約を適用すると便利な場合があります。タイプ制限は、タイプパラメータが特定のプロトコルまたはプロトコル構成と一致する必要があることを示します。

たとえばDictionary、Swiftの型は、辞書のキーとして使用できる型に制限を課します。辞書には、特定のキーの値がすでに含まれているかどうかを確認できるように、キーをハッシュする必要があります。

func someFunction<T: SomeClass, U: SomeProtocol>(someT: T, someU: U) {
       // function body goes here
}

実際、タイプTのスタックを作成しましたが、ここではタイプが一致しないため、2つのスタックを比較することはできませんEquatable。を使用するには、これを変更する必要がありますStack<T: Equatable>。

ジェネリックはどのように機能しますか？例を見てみましょう。

func min<T: Comparable>(_ x: T, _ y: T) -> T {
       return y < x ? y : x
}

コンパイラーには、関数コードを作成するために必要な2つのものが欠けています。

  • T型の変数のサイズ。
  • 関数<の特定のオーバーロードのアドレス。実行時に呼び出す必要があります。

コンパイラーがジェネリック型の値を検出すると、その値をコンテナーに配置します。このコンテナには、値を格納するための固定サイズがあります。値が大きすぎる場合、Swiftはそれをヒープに割り当て、そのリンクをコンテナーに格納します。

コンパイラーは、各総称パラメーターごとに1つ以上の監視テーブルのリストも保持します。値用の監視テーブル1つと、タイプ制限プロトコルごとの監視テーブル1つです。ウィットネステーブルは、実行時に関数呼び出しを目的の実装に動的に送信するために使用されます。
