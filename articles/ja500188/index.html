A-FrameとTone.jsを使用してオーディオビジュアルVRエクスペリエンスを作成する

こんにちは、ハブル！Sean Sullivanによる記事「AフレームとTone.jsを使用したWeb上のVRオーディオ/ビジュアルエクスペリエンスの作成」の翻訳を紹介します。



Aフレームは、ウェブ上で仮想現実を作成するためのフレームワークです。リンクだけで、VRヘルメットやVR対応のスマートフォンを持っている人なら誰でも3D空間に没頭できます。Tone.jsは、サウンドを作成するためのJavaScriptライブラリです。それらを組み合わせるとどうなるか見てみましょう。

まず、Aフレームを使用した非常にシンプルな環境を作成します。基本的なHTMLのみを使用して、3Dスペース全体を作成できます。これには、aframe-environment-componentが必要です。以下は、私たちの目的のための基本的なマークアップです。

<!DOCTYPE html>
<html>
<head>
  <title>Basic Scene with Environment - A-Frame</title>
  <meta name="description" content="Basic Scene with Environment - A-Frame">
  <script src="https://aframe.io/releases/1.0.4/aframe.min.js">  </script>
  <script src="https://unpkg.com/aframe-environment-component@1.1.0/dist/aframe-environment-component.min.js"></script>
</head>
<body>
  <a-scene environment="preset: starry">
    <a-camera>
      <a-entity cursor="fuse: true; fuseTimeout: 500"
            position="0 0 -1"
            geometry="primitive: ring; radiusInner: 0.02; radiusOuter: 0.03"
            material="color: black; shader: flat">
      </a-entity>    
    </a-camera>
  </a-scene>
</body>
</html>

要素に注意してください：
<a-entity cursor>
それは私たちの細胞の中にあります。少し後、彼は私たちのシンセサイザーと通信できるようになります。ただし、開始する前に、プロジェクトが正しく読み込まれていることを確認する必要があります。

ページを開くと、3次元の空、星、地球上のグリッドが表示されます。これらはすべて、環境を定義したときにaframe-environment-componentによって作成されました。

<a-scene environment=”preset: starry”>

必要に応じて、環境を変更できます。別のテンプレートを追加するだけです。この記事の執筆時点では、選択できる環境テンプレートは16種類あります。



私たちのシンセサイザーが宇宙にあるのが好きです。宇宙はクールです。私たちの環境を惑星の表面のようにしましょう。

最初に、グリッドを削除し、地球のテクスチャを追加して、次のように変更します。

<a-scene environment="preset: starry">

に

<a-scene environment="preset: starry; grid: none; groundTexture: walkernoise">

ここでページを起動すると、地球がまだ暗すぎて地球上の何も見えないことがわかります。これを修正するには、シーンに光源を追加します。

<a-entity light="type: ambient; color: #CCC"></a-entity>

コンポーネントの作成


フレームは、entity-component-systemに基づいて構築されています。コンポーネントを作成して、シーン内のエンティティに追加できます。コンポーネントのsynth.js

ファイルを作成しましょう。

AFRAME.registerComponent('synth', {
  schema: {
    //   .
  },

  init: function () {
    //     .
  },

  update: function () {
    //     .
  },

  remove: function () {
    //       .
  },
  tick: function (time, timeDelta) {
    //     ()  .
  }
});

ご覧のように、ライフサイクルメソッドはAフレームに組み込まれているため、WebVRプロジェクトに対話機能を簡単に追加できます。コンポーネントベースの準備ができたので、Tone.jsを使用してシンセサイザを作成するプロセスを見てみましょう。

Tone.js


Tone.js-ブラウザーでインタラクティブな音楽を作成するためのフレームワークは、Web Audio APIのラッパーです。tone.jsを使用してシンセサイザを作成するのは簡単です。次の行を書くだけです。

var synth = new Tone.Synth().toMaster()

ただし、さらにカスタマイズを簡略化するために、発振器を作成していくつかのパラメータを追加します。

const synth = new Tone.Synth({
  volume: -15, // -15dB
  oscillator: {
    type: 'triangle' //   -  ""
  },
  envelope: {
    attack: 0.05, //  -  
    release: 2 //   - 
  }
}).toMaster()

このコードを、synth.jsファイルのコンポーネントの上に直接追加します。これでシンセサイザができましたが、コンポーネントにアクセスする方法をコンポーネントに提供する必要があります。カメラに追加した<a-entityカーソル>を覚えていますか？このカーソルには、パラメータfuse = "true"があります。これにより、カーソルがエンティティとどのように相互作用するかを追跡できます。ヒューズのコンポーネントにEventListenerを追加します。ライフサイクルのinitメソッドでEventListener

を作成し、Tone.jsを起動するトリガーと呼ばれる新しいメソッドを作成します。

...
init: function () {
  //  EventListener     fuse
  this.el.addEventListener('fusing', this.trigger.bind(this))
},
//  ,  tone.js
trigger: function () {
  //tone.js ,     
  synth.triggerAttackRelease(this.data.note, this.data.duration)
},
...

シーンにシンセサイザコンポーネントを追加します。


コンポーネントを作成しました。Aフレームシーンに追加します。

まず、Tone.jsとシンセサイザーコンポーネントをマークアップに追加しました。ファイルを接続する順番に注意してください-synth.jsはTone.jsの後にロードされます。

...
<script src="https://unpkg.com/tone@13.8.25/build/Tone.js"></script>
<script src="synth.js"></script>
</head>
...

また、コンポーネントを接続するいくつかのエンティティも必要です。シーンで使用する標準のAフレーム形状をいくつか追加します。

<a-scene>
...
<a-box synth="note: E4" position="-1 0.5 -3" rotation="0 45 0" color="#4CC3D9"></a-box>
<a-sphere synth="note: C4" position="0 1.25 -5" radius="1.25" color="#EF2D5E"></a-sphere>
<a-cylinder synth="note: G4" position="1 0.75 -3" radius="0.5" height="1.5" color="#FFC65D"></a-cylinder>
...

シンセ 属性に注意してください。これが作成したコンポーネントです。「シンセ」は私たちが登録した名前です
AFRAME.registerComponent（ 'synth'、{}）
また、コンポーネント図で「ノート」を宣言しました。「duration」プロパティもあります。これを使用して、ノートの長さを変更できます。例えば：
synth = "注：E4; 時間：8n»
標準の1/4ではなく、ノート全体の1/8を演奏します。

これで、ブラウザーでシーンを開くと、フィギュアが表示されます。フィギュアにカーソルを合わせると、シンセサイザーコンポーネントのノートが再生されます。



Oculus Goコントローラーの使用


これで、シーンは次のように機能します-カーソルは画面の中央に固定されます。VRヘルメットでは、これは「ビジュアル」コントロールと呼ばれます。頭を回すと、カーソルがユーザーの移動方向に移動します。これは絶対に通常の経験であり、多くのプロジェクトでうまく機能します。しかし、VRコントローラを使用してシンセサイザを制御したい場合はどうでしょうか。腕を動かして音楽のサウンドを楽しくするため、シーンを変更してOculus Goコントローラーを使用します。

まず、シーンにいくつかのエンティティを追加します-コントローラとレイキャスター。

...
<a-entity oculus-go-controls>
<a-entity laser-controls raycaster="far: 200; interval: 100"></a-entity>
...

ここには、Oculus Goを制御する独自のエンティティと、100ミリ秒ごとに実行されるレイキャスター用のエンティティがあります。

次に、シンセサイザーコンポーネントを変更してOculusを制御します。これを行うには、コンポーネントに応じてレイキャスターを追加します。

AFRAME.registerComponent('synth', {
  dependencies: ['raycaster'],
...

次に、initメソッドで、EventListenerを変更します。イベントを追跡する必要があります。
レイキャスター交差


init: function () {
  this.el.addEventListener('raycaster-intersection', this.trigger.bind(this))
},
...

Oculus Goのシーンの最初にコントローラーが表示されます。また、数字にカーソルを合わせると、レーザーコントロールがシンセサイザーによるノートの再生を開始します。
