Transférer des fichiers à l'aide de tuyaux et d'autres petites choses sur Delphi

D'une certaine manière, ils m'ont confié la tâche d'écrire plusieurs unités dans Delphi qui donnent accès aux fonctionnalités Windows suivantes:

  1. Mémoire partagée .
  2. Pipes .

Il était impossible d'utiliser des composants tiers. La mémoire partagée était censée être utilisée pour trier des fichiers texte volumineux - plusieurs dizaines, voire des centaines de mégaoctets, sur lesquels le TStringList est tombé d' EOutOfMemory . Pipes - pour transférer des fichiers d'une application à une autre.

Réécrire le code d'exemples de la documentation Microsoft de C vers Delphi n'était même pas la moitié de la bataille, mais à 10%. L'essentiel était de tout lier ensemble, d'ajouter des fonctions supplémentaires si nécessaire et de tout faire fonctionner, et pas seulement d'exister en tant que code source. En conséquence, un monstre si laid est né qu'il ne devait rien montrer à personne, mais le regarder était effrayant. Cependant, peut-être que quelqu'un trouvera quelque chose d'utile dans ses entrailles (et si?), J'ai donc décidé de le poster ici. Pour reproche général. Le code source complet de cette abomination est disponible sur GitHub . Le texte de l'article donnera les noms des fichiers où vous pourrez voir l'un ou l'autre morceau de l'implémentation.

La première tâche consiste à trier le fichier texte - a été décidé comme ceci:

  1. Ouvrez le fichier ( CreateFile ).
  2. Nous utilisons CreateFileMapping et MapViewOfFile (common \ MemoryMappedFileUnit.pas).
  3. Nous déterminons l'encodage du fichier. Ici, je me suis limité, en fait, à seulement deux encodages: UTF-16 et ACP - un encodage à un octet avec une page de codes par défaut. Dans Windows russophone, Windows-1251 est généralement utilisé .
  4. Nous divisons le texte en lignes. Autrement dit, nous recherchons des marqueurs de fin de ligne et, par conséquent, nous considérons le texte d'un marqueur à l'autre comme une seule ligne.
  5. Triez le tableau de chaînes résultant (SortingTest \ MergeSorterUnit.pas).
  6. Nous écrivons le tableau trié de chaînes dans le fichier.
  7. PROFIT!

Le client souhaitait vraiment faire du tri multithread, donc Merge Sort a été pris comme algorithme de tri . (Merci au gourou qui a créé l'implémentation de travail et l'a publiée sur Stackoverflow .)

La deuxième tâche - transférer le fichier d'une application à une autre - s'est avérée beaucoup plus difficile. Obtenir l'implémentation de canaux sur Delphi à partir d'exemples Microsoft est une chose, mais en faire un transfert de fichier de travail en est une autre. En tant que programme serveur, Microsoft propose plusieurs options:

  1. Serveur de tuyaux multithread .
  2. Serveur de tuyaux utilisant des E / S superposées nommées .
  3. Named Pipe Server Using Completion Routines .

Comme il s'est avéré à la suite de deux semaines de tourments, la mise en œuvre du mécanisme de transfert de fichiers basé sur la première version du serveur ( Multithreaded Pipe Server ) est trop compliquée. Il est plus raisonnable d'utiliser la deuxième option ( Named Pipe Server Using Overlapped I / O ), avec elle l'implémentation est beaucoup plus simple (cependant, le client voulait vraiment la première version du serveur pour une raison quelconque ). Mais je devais encore me tourmenter pour me souvenir de tout cela. En conséquence, un émetteur de fichier basé sur des tuyaux est né: TCommunicationEndpointBase (CommunicationTest \ SimpleCommunicationUnit.pas).
