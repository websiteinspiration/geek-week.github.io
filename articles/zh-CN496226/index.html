TeamViewer如何存储密码


TeamViewer是一种流行的程序，用于远程访问桌面。因此，查看密码存储方式非常有趣。简而言之，密码在Windows注册表中被加密。为了进行加密，使用了AES-128-CBC算法和秘密密钥0602000000a40000525341313100040000。

这种保存密码的方法和相关的特权提升已于2020年2月7日正式注册为漏洞CVE-2019-18988（适用于TeamViewer的所有版本，直到07/14/1965年为止）。

揭露此漏洞的新手安全专家偶然发现了该漏洞。他说他为客户端工作，在备份期间注意到TeamViewer注册表项，称为OptionsPasswordAES和SecurityPasswordAES。

随后，他对这些键是什么以及TeamViewer一般如何存储密码感兴趣。他将它们导入到虚拟机中的新系统中，并启动了BulletPassView扫描程序，该程序收集系统中的密码。扫描程序以明文形式给TeamViewer密码。


扫描仪BulletPassView

然后，使用Cheat Engine程序（用于在Windows下破解游戏），专家在RAM中搜索此密码-并再次以纯文本形式找到它。后来发现，此漏洞已经在两年前作为 CVE-2018-14333修复。

然后是时候检查密钥从TeamViewer客户端存储器中的位置了：从服务器还是从本地主机。原来，没有网络流量，但是内存中仍然有密码。使用IDA Pro， API Monitor， procdump和 Frida对TeamViewer二进制文件进行逆向工程花费了几周的时间，但没有给出任何结果，尽管此人一路掌握了一些新工具，所以这个过程不能被称为无用。

在搜索信息期间，事实证明，已经有很多人想知道如何在资源中找到Unity游戏的AES密钥。事实证明，这是一个非常简单的过程，使用调试器就足够了。六个小时后，他发现了一段负责AES加密的TeamViewer代码：

此代码从寄存器中取出几个字节，ServerPasswordAES并使用XOR'a规定值esi和所获得的密钥对它们进行解密eax。结果是关键0602000000a400005253413100040000，IV的值相等0100010067244F436E6762F25EA8D704。

在TeamViewer 14中，新的脚本引擎允许您通过NT AUTHORITY\SYSTEM使用用户权限读取注册表中的TeamViewer密码来增加特权。

嗯，您还可以通过使用搜索参数[filetype：reg]来搜索[SecurityPasswordAES]，[OptionsPasswordAES]，[SecurityPasswordExported]或[PermanentPassword]，从而解密其他一些密码。

TeamViewer已在最新版本中关闭了此漏洞。由于该程序现在可以加密密码，因此他们还没有弄清楚。
