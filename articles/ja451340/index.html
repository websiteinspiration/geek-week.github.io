GitHub Package Registryが提供

GitHub Package Registry（パブリックまたはプライベートパッケージをソースの近くで簡単に公開できるようにするパッケージ管理サービス）がリリースされました。

GitHubパッケージレジストリはGitHubと完全に統合されているため、リポジトリの場合と同じ検索、表示、制御ツールを使用してパッケージを検索および公開できます。同じユーザーとチームの権限を使用して、コードとパッケージを共同で管理することもできます。 GitHub Package Registryは、グローバルなGitHub CDNをサポートし、高速で信頼性の高いダウンロードを提供します。さらに、JavaScript（npm）、Java（Maven）、Ruby（RubyGems）、. NET（NuGet）、Docker（およびその他の進行中の）など、おなじみのパッケージ管理ツールもサポートしています。

GitHub Package Registryを制限付きベータ版で今すぐ試すことができます。オープンソースではいつでも無料で使用できます。詳細な価格情報は間もなく発表されます。

コードを含むパッケージ


パッケージに依存するプロジェクトに取り組んでいるときは、パッケージを信頼し、コードを理解し、パッケージを作成したコミュニティとコミュニケーションすることが重要です。また、組織内では、使用が承認されているものをすばやく見つけることができるはずです。GitHub Package Registryを使用すると、同じ使い慣れたGitHubインターフェースを使用して、GitHub上の任意の場所にあるパブリックパッケージや、組織やリポジトリのプライベートパッケージを簡単に検索できます。



GitHubパッケージレジストリは通常のパッケージ管理クライアントと互換性があるため、選択したツールを使用してパッケージを公開できます。リポジトリがより複雑な場合は、さまざまなタイプのいくつかのパッケージを公開できます。また、WebhookまたはGitHubアクションを使用すると、公開および公開後のワークフローを完全にカスタマイズできます。

オープンソースパッケージを公開していますか？ほとんどのオープンソースプロジェクトにはGitHubに独自のコードがあるため、コミュニティでテストするためにパッケージのプレビューバージョンを公開し、特定のバージョンを選択したオープンレジストリに簡単に追加できます。

統合された認証と権限




現在コードとパッケージに異なるシステムを使用している場合、異なる資格情報とユーザー権限のセットを維持する必要があります。これで、両方に1つの資格情報セットを使用し、同じツールを使用してアクセス許可を管理できます。GitHubパッケージはリポジトリに関連付けられた可視性と権限を継承し、組織は個別のパッケージレジストリを維持したり、異なるシステムの権限をミラーリングする必要がなくなりました。

パッケージの関係者




GitHubでホストされているパッケージには、詳細情報とダウンロード統計情報、およびその履歴全体が含まれているため、パッケージに関するすべてを正確に把握できます。これにより、プロジェクトの依存関係として適切なパッケージを見つけて使用しやすくなり、必要なものだけが含まれているという確信が高まります。公開するパッケージに関する詳細情報を受け取ることにより、他の人やリポジトリがそれらをどのように使用するかを正確に理解できます。
