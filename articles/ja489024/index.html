テーブルデータのフィルタリングと並べ替え


テーブルから始めましょう-私は多くの場合、それらを使用して多くのデータを処理しました。Webixライブラリのテーブルには、テーブルウィジェットとツリーテーブルウィジェットのヘッダーに直接設定される多数の組み込みフィルターがあります。テーブルウィジェットでは、2つのオプションを使用します。単純なテキストフィルター（textFilter）と、ドロップダウンリストにオプションのセットを持つフィルター（selectFilter）です。テーブルでは、列ごとに1つのフィルターを追加できます。タイトルと年の2つについてこれを行います。配列と文字列の代わりにそれらにヘッダーを設定します-ヘッダーとフィルターに合うように。配列の2番目の要素は、コンテンツプロパティとフィルター名を持つオブジェクトです。
テーブルウィジェットのコードはtable.jsファイルにあり、[ダッシュボード]タブにレンダリングされます。

文字が"textFilter"データに入力されると、それらはサブストリングによるマッチングによってフィルタリングされます。オプションを選択した場合"selectFilter"-選択した値による。

columns:[
    { id:"rank", header:"", width:50, css:"rank"},
    { id:"title", header:["Film title", { content:"textFilter"}], fillspace:true },
    { id:"year",  header:["Released", {content:"selectFilter" }], width:100 },
    { id:"votes", header:"Votes", width:100 },
    { id:"rating", header:"Rating", width:100 },
    { header:"", template:"<span class='webix_icon wxi-close'></span>", width:35}
]

APIの並べ替えとフィルタリング


アイテムのフィルタリングと並べ替えの既製のソリューションにはテーブルしかありません。ただし、一般に、すべてのウィジェットは、適切なAPIメソッド（フィルターとソート）を通じてこれらの機能をサポートします。リストウィジェットでAPIを使用したフィルタリングと並べ替えのデモを行います。

リストウィジェットのコードは、users_module.jsファイルにあり、[ユーザー]タブに表示されます。

濾過。[ ユーザー ]タブの[ 新しいユーザーを追加]ボタンの後に、リストから名前のフィルターとして使用するテキストウィジェットをインストールします。

cols:[
    { 
        view:"button", id:"btn_add_person", 
        value:"Add new person", width:150, css:"webix_primary", 
        click:addPerson
    },
    { 
        view:"text", id:"list_input" 
    },
]

次に、script.jsファイルを開き、要素のフィルタリングを担当するロジックを追加します。

$$("list_input").attachEvent("onTimedKeyPress",function(){
    var value = this.getValue().toLowerCase();
    $$("user_list").filter(function(obj){
        return obj.name.toLowerCase().indexOf(value) !== -1;
    })
});

要素はこの原則に従ってフィルタリングされます。
  • attachEventメソッドを使用して、onTimedKeyPressイベントにハンドラーを追加します。
  • onTimedKeyPressイベントは、テキストフィールドに文字を入力するとトリガーされますが、すべてのキーストロークでフィルターがアクティブにならないように少し遅れます。
  • 次に、入力したテキストを取得し、filterメソッドを使用してフィルタリングを開始します。リストウィジェットで一致を検索します。

APIによる結果のフィルタリング：



並べ替え。“Sort asc”とボタンをクリックすると、リストウィジェット要素がソートされます“Sort desc”。

[ ユーザー ]タブでボタンを作成するには、テキストボックスの後に、クリックイベントハンドラーを備えた2つのボタンウィジェットを追加します。

cols:[
    { 
        view:"button", id:"btn_add_person", 
        value:"Add new person", width:150, css:"webix_primary", 
        click:addPerson
    },
    { 
        view:"text", id:"list_input" 
    },
    { view:"button", id:"btn_asc", width:150, value:"Sort asc", css:"webix_primary", 
        click:()=>{
            $$("user_list").sort("#name#","asc")
     }},
    { view:"button", id:"btn_desc", width:150, value:"Sort desc", css:"webix_primary", 
        click:()=>{
            $$("user_list").sort("#name#","desc")
    }},
]

クリックハンドラー内で、sortメソッドはパラメーターを受け入れます。データの並べ替えの基準となるフィールドの名前と、並べ替えの方向“asc”（昇順）-昇順、および“desc”（降順）-降順です。デフォルトでは、データは行と見なされ、それに応じてソートされます。

ツリーテーブルデータのグループ化


任意のパラメータに従ってデータをグループ化する必要がある状況を考えます。
例として、TreeTableウィジェットを使用してグループ化を調べます。

TreeTableウィジェットのコードは、products_module.jsファイルにあり、「Poducts」タブにレンダリングされます。

この記事の内容：モジュール、グラフ、ツリーテーブル、およびデータの操作。CRUD、ツリーテーブルで階層データを使用しました。この問題を解決するために、線形配列になるように変更しました。階層を削除して、フィールドをその階層から各レコードに転送しました“category”。

[
    {"id": "1.1",   "title": "Standard Ticket",  "price": 21, "category":"Cinema", "rank":1.1},
    {"id": "2.1",   "title": "Cola",  "price": 10, "category":"Cafe", "rank":2.1},
    {"id": "3.1",   "title": "Flowers",  "price": 10, "category":"Other", "rank":3.1}
]

テーブル内のデータをグループ化するには、2つの方法があります。

  • それらをロードするとすぐに;
  • グループおよびグループ解除メソッドを動的に使用します。

これらのメソッドのパラメーターは同じであり、データが到着したらすぐにグループ化する必要があるため、最初のオプションを使用します。
TreeTableウィジェットの構成で、schemeプロパティを追加します。このプロパティは、さまざまな状況でデータが処理されるスキームを決定します。shemeのハンドラーの中には、データをグループ化する必要がある$ groupメソッドがあります。

const products = {
    editable:true,
    view:"treetable",
    scrollX:false,
    columns:[
        { id:"rank", header:"", width:50 },
        { id:"title", header:"Title", fillspace:true, template:"{common.treetable()} #title#"},
        { id:"price", header:"Price", width:200, editor:"text" }
    ],
    select:"row",
    url:"data/products.js",
    scheme:{
        $group:{
            by:"category",
            map:{
                title:["category"]
            }
        },
        $sort:{ by:"value", dir:"asc" }
    }
}

$グループハンドラー内では、2つのパラメーターが使用されます。
  • 必要なパラメータによってデータがグループ化されます。以下はフィールドの1つ（「カテゴリ」）です。
  • マップオブジェクト-ここでは、作成されたグループのデータフィールドについて説明します。グループ化すると、指定したパラメーターに従ってソースデータが分割され、それらの「親レコード」が作成されます。マップを通じて、これらのレコードに新しいフィールドを追加できます。テーブルのデータが正しく表示されるように、「タイトル」フィールドを追加します。グループ化が発生するパラメーターの値を渡します。

さらに、グループ化されたデータをアルファベット順に並べ替える$ sort関数を設定しました。

コンポーネント同期


このタスクでは、チャートウィジェットとリストウィジェットを使用します。これらのコードはusers_module.jsファイルにあり、[ユーザー]タブに表示されます

チャートウィジェットとリストウィジェットは同じデータ、つまりJSON配列を使用します。これらのコンポーネントは、1つのコンポーネント（ソース）でのすべてのデータ変更が別のコンポーネントに送信されるようにリンクできます。これを行うには、syncメソッドを使用します。sync

メソッドを使用すると、1つのコンポーネントからデータをコピーして、別のコンポーネントに転送できます。同時に、追加、削除などの主要コンポーネントの変更はすぐに別のコンポーネントに反映されます。 まず、チャートウィジェット-チャート-データリンクを削除します。



{
    view:"chart",
    id:"chart",
    type:"bar",
    value:"#age#",
    label:"#age#",
    xAxis:{
        template:"#name#",
        title:"Age"
    }
}

次に、script.jsファイルで、syncメソッドを使用して、ChartウィジェットをListウィジェットと同期させます。

$$("chart").sync($$("user_list"));

addPerson 関数では、追加はリストウィジェットにのみ残されます。

let addPerson = () => {
    let obj = {
        name:"Some name",
        age:Math.floor(Math.random() * 80) + 10, 
        country:"Some country"
    }
    $$("user_list").add(obj);
};

これで、リストにエントリを追加および削除すると、グラフが変更されます。リストウィジェットでの並べ替えとフィルタリングは、チャートのデータに影響します。
