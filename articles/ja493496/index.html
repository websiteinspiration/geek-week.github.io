私たちは、次のようにエラーを再現します-私たちは、ある「お問い合わせ」ページ上にある別のルートから切り替わります、私たちは、ページ「ホーム」に行くと、すぐに「会社概要」ページに再び戻り、出来上がり「...実行できません...」 。

実際のところ、サーバーはリクエストに即座に応答せず、非同期を使用して、必要なタスクと並行してリクエストを再生します。しかし、About Usページにすばやく戻る場合、Homeコンポーネントはマウント解除されています。つまり、このコンポーネントの状態はリセットされますが、非同期リクエストはsetMovieを起動しますが、そこには存在せず、エラーがスローされます。最善の解決策は、コンポーネントをアンマウントするときに状態更新の使用を禁止することです：

/src/Pages/HomePage.js

import React, { useState, useEffect } from 'react';

import { MOVIE_DB_GET } from '../config';

const HomePage = () => {
  // movie - react-;
  // setMovie -   react-
  const [ movie, setMovie ] = useState(null);

  useEffect(() => {
    let cleanupFunction = false;
    const fetchData = async () => {
      try {
        const response = await fetch(MOVIE_DB_GET);
        const result = await response.json();
        console.log(result, 'result')

        //     ,    
        if(!cleanupFunction) setMovie(result);
      } catch (e) {
        console.error(e.message)
      }
    };

    fetchData();

    //   useEffect
    return () => cleanupFunction = true;
  }, []);

  // descriptionMovie -  ,  view,         movie

  return (
    <main>
      <h2> </h2>
      <p> </p>

      {
        movie ? descriptionMovie() : false
      }
    </main>
  )
};

export default HomePage;

  • 構成の相互作用を試行/キャッチし、非同期/非同期を待つ
