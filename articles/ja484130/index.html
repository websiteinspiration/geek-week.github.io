リソースの概要


このチュートリアルでは、適切なURIとHTTPメソッドを使用して3つのサービスを提供する学生用のリソースを作成します。

  • すべての学生のリストを取得する- @GetMapping（ "/学生"）
  • 特定の学生に関する情報を取得する- @GetMapping（ "/ student / {id}"）
  • 生徒情報を削除- @DeleteMapping（ "/ student / {id}"）
  • 新しい生徒のエントリを作成する- @PostMapping（ "/ student"）
  • 学生情報の更新- @PutMapping（ "/ student / {id}"）

外国為替サービス


Forexサービス（FS、Forex Service）はサービスプロバイダーです。さまざまな通貨の為替レートを提供します。彼が外国為替取引所と通信し、通貨間の交換の現在の価値を提供すると仮定しましょう。リクエストとレスポンスの例を以下に示します。

GET to http://localhost:8000/currency-exchange/from/EUR/to/INR


{
  id: 10002,
  from: "EUR",
  to: "INR",
  conversionMultiple: 75,
  port: 8000,
}

上記の要求に対する応答は、ユーロからINRへの為替レートです。ConversionMultipleの答えは75です。
ポートフィールドについては、少し後で説明します。

通貨換算サービス


通貨換算サービス（CCS）は、多くの通貨を別の通貨に換算できます。現在の為替レートを取得するためにForexサービスを使用しています。CCSはサービスの利用者です。リクエストとレスポンスの例を以下に示します。

GET to http://localhost:8100/currency-converter/from/EUR/to/INR/quantity/10000

{
  id: 10002,
  from: "EUR",
  to: "INR",
  conversionMultiple: 75,
  quantity: 10000,
  totalCalculatedAmount: 750000,
  port: 8000,
}

上記のリクエストにより、インドルピーで10,000ユーロの価値を決定できます。
TotalCalculatedAmountは750,000インドルピーです。次の図は、CCSとFSの関係を示しています。

モノリシックアプリケーションとは何ですか？


プロジェクトで働いたことはありますか...

  • 数か月に1回生産（生産に投入）
  • これは、幅広い機能を備えています。
  • 50人以上のチームが働いている
  • デバッグの問題が大きな問題である場合
  • 新しいテクノロジーと新しいプロセスの導入がほとんど不可能である場合

これらは、モノリシックアプリケーションの典型的な特性です。

通常、モノリシックアプリケーションは巨大で、10万行を超えるコードです。場合によっては、100万行を超えるコードです。

モノリスの特徴は次のとおりです。

  • 大きなアプリケーションサイズ
  • 長いリリースサイクル
  • 大きなチーム

典型的な問題は次のとおりです。

  • スケーラビリティの問題
  • 新技術の採用
  • 新しいプロセス-アジャイル？
  • テストの自動化が難しい
  • 現在の開発慣行に適応するのは難しい
  • プロジェクトの急速な成長に対応するのは困難です。

マイクロサービス


マイクロサービスアーキテクチャは、モノリシックアーキテクチャによるスケーラビリティとイノベーションの課題に対処するために進化しました。マイクロサービスには、いくつかの定義が提案されています。
連携して動作する小規模なスタンドアロンサービス-Sam Newman
小さなサービスのセットの形式での個別のアプリケーションの開発。それぞれが独自のプロセスで機能し、軽量のメカニズム（多くの場合、HTTPリソースAPI）と対話します。これらのサービスはビジネスチャンスに基づいて構築されており、完全に自動化された展開メカニズムを使用して個別に展開できます。これらのサービスの集中管理には最低限のレベルがあり、さまざまなプログラミング言語で記述し、さまざまなストレージテクノロジーを使用できます-James Lewis（James Lewis）とMartin Fowler（Martin Fowler）

マイクロサービスの単一の承認された定義はありませんが、いくつかの重要な特性があります。

  • REST-RESTfulリソースを中心に構築されています。サービス間の通信は、イベントまたはHTTPプロトコルに基づくことができます。
  • 厳選された小さな展開可能なブロック-限られたコンテキスト
  • クラウド機能-動的スケーリング

マイクロサービスアーキテクチャはどのようなものですか？

マイクロサービスアーキテクチャには、適切に設計された小さなメッセージングコンポーネントがいくつか含まれています。
