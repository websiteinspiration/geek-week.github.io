GTK3でcairoを使ってグラフを描く


float型の配列が入力として機能します。プログラムは、チャートの表示、ストレッチ、スクロールを整理します。

書き込みスタイルはC（クラスあり（gtkmmなし））です。抽象化が流れるため、完璧ではないことがわかりました。特に、コールバック関数はカプセル化を低下させます。変数のかなりの部分をパブリックセクションに移動する必要があります。
基本的に、コールバック関数は、クラスの残りの関数（graphic_parametersと呼びます）とともにファイルに配置できます。 GTKでは、ウィジェットのタイプごとに独自の信号があり、その一部は継承されます。たとえば、GtkEventBoxは常にコンテンツのサイズを取得するため、GtkEventBoxには「button-press-event」信号がありますが、ウィジェットのサイズ変更に応答するために必要な「configure-event」はありません。コンテンツのサイズは手動で設定されます。 GtkFrameコンテナを使用できます。

cairo_surface_t  *surface = cairo_image_surface_create(CAIRO_FORMAT_ARGB32, width, height);
cairo_t  *cr = cairo_create(surface);

cairo_tでは、行、ラベルが作成され、cairo_stroke関数によって表示されます。プロファイリングの際、cairo_strokeは多くのプロセッサ時間を必要とすることがわかったため、できるだけ使用しないでください
。cairo_move_to、cairo_line_toなどの関数の実行時間は非常に短くなります。cairo_strokeの後、cairo_tの内容はクリアされ、cairo_stroke（cr）を再度呼び出しても何も出力されません。
cairo_stroke_preserveを使用してコンテンツとcairo_save / cairo_restoreを保存できますが、私はそれらを使用しませんでした。

寸法が変更された場合（マウスを使用してconfigure_event_cb信号で拡大）、描画ごとにcairo_surface_tとcairo_tを削除して再作成する必要があります。スケジュールを巻き戻す場合、再作成する必要はありません

    cairo_set_source_rgb(cr,0.8,0.8,0.8);
    cairo_paint(cr);

次に、cairo_surface_tが画像に変換されます

void gtk_image_set_from_surface (GtkImage *image, cairo_surface_t *surface);

接頭辞を削除しました。たとえば、scrolledwindowのタイプはGtkScrolledWindowです。
添付ファイルの順序が簡単に表示されるimage-> scrolledwindow-> viewport-> box-> eventbox->（Frame）scrolledwindow-
> viewportコンテナを削除すると、グラフは増加するだけで減少はしません。ボックスはスクロールを追加します。3つあることがわかりますが、そのうち2つは使用されておらず、必要なコンテナを初期化するためにのみ必要です。1つの子ウィジェットが収まるコンテナウィジェットでは、gtk_container_add関数を使用して挿入します。g_object_setは、追加のプロパティを設定します
。特に、scrolledwindowウィジェットのスクロールバーがないことを設定します。GValueを介してプロパティを設定することもできます

スクロールメカニズム：セグメント全体が100で除算され、スケジュールの変更はコールバック関数で計算されます。100は、gtk_adjustment_new（0,0,110,1,5,10）の数値から100 = 110-10として取得されます。

さらに、パラメーター化について。

テキストをパラメーター化するには、pangoライブラリを使用してラベルをパラメーター化します。これにより、特定のフォントおよびそのトポグラフィックサイズのテキストのサイズをピクセル単位で計算し、それをcairoレイヤーにエクスポートできます。

PangoLayout* get_width_height_of_text(char *text, char *font, float size, float *w, float *h)
{
    GdkScreen *screen = gdk_screen_get_default();
    PangoContext *context = gdk_pango_context_get_for_screen (screen);
    PangoLayout *layout = pango_layout_new (context);
    if(g_utf8_validate(text,-1,0))
    {
        pango_layout_set_text(layout,text,-1);
        PangoFontDescription *desc=pango_font_description_new();
        pango_font_description_set_family(desc,font);
        pango_font_description_set_size(desc,size*1024);
        pango_layout_set_font_description (layout, desc);
        int width=0,height=0;
        pango_layout_get_size(layout, &width, &height);
        *w=(float) width/1024;
        *h=(float) height/1024;
        pango_font_description_free(desc);
    }
    else
    {
        printf("      UTF8\n");
    }
    return layout;
}

ご覧のように、pangoはサイズを独自の単位でカウントします。
