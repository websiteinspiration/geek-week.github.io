古いBIOSとLinux OSを搭載したコンピューターでシステムドライブとしてNVME SSDを使用する



適切に構成されていれば、古いシステムでもNVME SSDから起動できます。オペレーティングシステム（OS）がNVME SSDで動作することが想定されています。 OSでNVMEドライバーを使用できる場合、SSDはロード後にOSに表示され、使用できるため、OSのロードを検討します。 Linux用の追加のソフトウェア（ソフトウェア）は必要ありません。 BSDファミリおよびその他のUnixのオペレーティングシステムでは、この方法が適している可能性が最も高いです。

ドライブから起動するには、起動プログラム（BIOS）、BIOSまたはEFI（UEFI）にこのデバイスのドライバーが含まれている必要があります。NVME SSDはBIOSと比較してかなり新しいデバイスであり、古いマザーボードのファームウェアのファームウェアにはそのようなドライバーはありません。EFIでは、NVME SSDをサポートしていなくても、適切なコードを追加できます。これにより、このデバイスを完全に使用できるようになります。オペレーティングシステムをインストールしてロードできます。いわゆる OSをロードすることで「レガシーBIOS」を実行できる可能性はほとんどありません。ただし、これは回避できます。

実行する方法


私はopenSUSE Leap 15.1を使用しました。他のLinuxの場合、アクションはほぼ同じです。

1.オペレーティングシステムをインストールするコンピューターを準備します。
PCI-E 1.0で十分なバージョンに関係なく、無料のPCI-E 4xコネクタを備えたPCまたはサーバーが必要です。もちろん、PCI-Eバージョンが新しいほど、速度は速くなります。まあ、実際には、アダプターM.2を備えたNVME SSD-PCI-E 4x。
また、300 MB以上の容量を持つ何らかの種類のドライブが必要です。これは、BIOSから認識でき、OSをロードできます。 IDE、SATA、SCSIを介して接続されたハードドライブにすることができます。 SAS。またはUSBフラッシュドライブ、またはメモリカード。フロッピーディスクには収まりません。 CD-ROMが動作しないため、書き換えが必要です。 DVD-RAM-わからない。これを「レガシーBIOSドライブ」と呼びましょう。

2. Linuxをダウンロードしてインストールします（光ディスクまたはブート可能なフラッシュドライブなどから）。

3.ディスクをマークアップするとき、利用可能なドライブにOSを配布します
。サイズが8 MBの「レガシーBIOSドライブ」の先頭にGRUBブートローダー用のパーティションを作成します。ここではopenSUSE機能-別のセクションでGRUBを使用していることに注意してください。 openSUSEでは、デフォルトでBTRFSファイルシステム（FS）が使用されます。 GRUBがBTRFS FSのあるパーティションに配置されている場合、システムは起動しません。したがって、別のセクションが使用されます。ロードされていれば、GRUBを別の場所に配置できます。
3.2。 GRUBを使用してパーティションを作成した後、システムフォルダーの一部（「ルート」）、つまり「/ boot /」を使用して、サイズが300 MBのパーティションを作成します。
3.3。残りのシステム-システムフォルダーの残り、スワップパーティション、ユーザーパーティション「/ home /」（作成する場合）は、NVME SSDに配置できます。

インストール後、システムは/ブート/からファイルをダウンロードするGRUBをロードし、その後NVME SSDが使用可能になり、システムはNVME SSDからブートします。
実際、私はかなりの加速を得ました。

「レガシーBIOSドライブ」の容量要件：GRUBパーティションの場合は8 MBがデフォルトで、/ boot /の場合は約200 MBです。余裕を持って300MBを取りました。カーネルを更新するとき（および新しいカーネルをインストールするとき）、Linuxは/ブート/パーティションに新しいファイルを補充しますが、不要なカーネルを削除すると、スペースが解放されます。これら2つのセクション以外の「レガシーBIOSドライブ」の残りのスペースは、自由に使用できます。

速度制限：
PCI-E 3.0 4x約3900 MB / s
PCI-E 2.0 4x 2000 MB / s
PCI-E 1.0 4x 1000 MB / s
実際にPCI-E 3.0 4xを搭載したストレージドライブは、約3500 MB / sの速度に達します。
達成可能な速度は次のようになると想定でき
ます。PCI-E3.0 4x約3500 MB /秒
PCI-E 2.0 4x約1800 MB /秒
PCI-E 1.0 4x約900 MB /秒

これはSATA 600 MB /秒よりも高速です。 SATA 600 MB /秒で達成可能な速度は約550 MB /秒です。
同時に、古いマザーボードでは、オンボードコントローラーのSATA速度が600 MB /秒ではなく、300 MB /秒または150 MB /秒である場合があります。ここでは、オンボードコントローラー=チップセットのサウスブリッジに統合されたSATAコントローラー。

NCQはNVME SSDでも機能しますが、古いオンボードコントローラーではこれが当てはまらない場合があります。

PCI-E 4xの計算を行いましたが、一部のドライブにはPCI-E 2xバスがあります。これはPCI-E 3.0には十分ですが、古いPCI-E規格（2.0および1.0）の場合、このようなNVME SSDは使用しない方がよいでしょう。また、メモリチップの形のバッファを備えたドライブは、それがない場合よりも高速になります。

オンボードSATAコントローラーを完全に放棄したい場合は、2つのSATA 600ポート（内部または外部）を提供するAsmedia ASM 106xコントローラー（1061など）を使用することをお勧めします。（ファームウェアの更新後）非常にうまく機能し、AHCIモードではNCQをサポートします。PCI-E 2.0 1xバス経由で接続。

その最大速度：
PCI-E 2.0 1x 500 MB / s
PCI-E 1.0 1x 250 MB / s
達成可能な速度は：
PCI-E 2.0 1x 460 MB / s
PCI-E 1.0 1x 280 MB / s

これは1つのSATAに十分ですSSDまたは2つのHDD。

欠点


1. SMARTパラメータは NVME SSDで読み込まれません。製造元、シリアル番号などの一般的なデータのみがあります。多分古いマザーボード（mp）が原因です。私の非人道的な実験では、nForce4チップセットで見つけた最も古いmpを使用しました。

2. TRIMは機能するはずですが、チェックする必要があります。

結論


他の可能性があります：PCI-E 4xまたは8xコネクタ付きのSASコントローラを購入します（16xまたは32xはありますか？）。ただし、安価な場合はSAS 600をサポートしますが、SATAはわずか300であり、高価なものは上記で提案した方法よりも高価で遅くなります。
