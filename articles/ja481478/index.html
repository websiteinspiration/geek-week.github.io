STM32 + CMSIS + STM32CubeIDE

ここでは、STMマイクロコントローラー用の「ネイティブ」IDE-STM32CubeIDEを使用して、CMSISで最小限のプロジェクトを作成する方法について説明します。

おそらくSTM32CubeIDEには多くの欠点がありますが、私の意見では、プロプライエタリやフリーなど、いくつかの利点があります。少なくとも、これまでにこれを行ったことがない場合は、この開発環境に注意を払う必要があります。



ARM Cortex-M0コアを備えたSTM32F072マイクロコントローラーが、ARM Cortex-M3のより馴染みのあるSTM32F103のファームウェアのオブジェクトとして選択され、コアが修正されました。プロセスは同じです。

必要なリソースはすべてst.comからダウンロードできます。必要なものは次のとおりです。

  • IDE自体、私はWindowsバージョンを使用していますが、MacとLinuxのバージョンも利用できます

  • ARM Cortex-M0用のCMSISライブラリ。これはSTM32CubeFxアーカイブにあり、カーネルバージョンで分類されています。アーカイブ自体には、CMSISに加えて、周辺機器の操作の例からUSBドライバーまで、さまざまなリソースが含まれています。実際、このアーカイブは、STM32Cubeを使用してプロジェクトを作成する場合に使用されます

  • データシートとリファレンスマニュアルは傷つかない


ワークスペースフォルダーをインストールして開始し、選択したら、プロジェクトの作成を開始できます。現在STM32CubeIDEバージョン1.1.0であるため、さまざまな設定の場所はこれに基づく必要があります。

新しいプロジェクトの作成-File / New / STM32Project。考えた後、マイクロコントローラーの選択ウィンドウが表示されます。私の場合は、LQFP64パッケージのSTM32F072RBです。目的の行を選択してクリックします。さらに、プロジェクト名、場所、C / C ++プログラミング言語、実行可能ファイル/静的ライブラリ、およびプロジェクトがCubeMXを使用して生成されるか口ひげを使用して生成されるかを選択することが提案されています。この場合、キューブの生成は必要ないため、プロジェクトタイプEmptyが仕上げです。




左側の[ プロジェクトエクスプローラー]ウィンドウに、プロジェクトツリーが表示されましたが、注文どおり、まったく空ではありません。原則として、生成されたフォルダー構造があなたに合う場合は、そこにCMSISライブラリーからファイルを追加して作業できますが、ここでは、プロジェクト構造を私の美意識と調和させる方法を示します。これにより、リンカースクリプト以外のすべてが削除されます。つまり、拡張子が.ldのファイル-便利です。



フォルダとファイルを使用したすべての操作は、プロジェクト名を右クリックすることで、エクスプローラとIDEの両方で実行できます。たとえば、右ボタン->新規->フォルダのようになります。IDEの外部でプロジェクト構造が変更された場合は、プロジェクトを更新するだけです。右ボタン-> [更新]をクリックします。

プロジェクト構造の私のバージョンは次のようになります。

  • スタートアップ -ここでは、リンカースクリプトが保存されます。生成されたプロジェクトから残っているスクリプトと、CMSISから取得したスタートアップファイルです。
  • CMSIS \ srcおよびCMSIS \ inc-ここに、ソース、scrフォルダーに拡張子.cが付いたファイル、およびCMSISライブラリーに関連するincフォルダーに拡張子.hが付いたヘッダーファイルがあります。
  • Core \ srcおよびCore \ inc-プロジェクト自体はここに配置されます。まず、main.cとmain.hをそこに配置します


次に、CMSISライブラリファイルをプロジェクトに転送する必要があります。ライブラリは、コアファイルと周辺機器ファイルで構成されています。カーネルファイルは、core_またはcmsis_で始まります。これらのファイルは、このカーネルを使用するすべてのマイクロコントローラーに共通です。周辺機器ファイルの名前にはマイクロコントローラーstm32の名前が含まれており、特定の製造元（この場合はSTM）に固有です。

解凍された形式では、アーカイブにはSTM32Cube_FW_F0_V1.11.0フォルダーが含まれ、すべてのパスはこのフォルダーを基準にして示されます。合計は、あなたがコピーする必要があります

でCMSIS \ INCを：

  • ドライバー\ CMSIS \インクルード\ cmsis_compiler.h
  • ドライバー\ CMSIS \インクルード\ cmsis_gcc.h
  • ドライバー\ CMSIS \インクルード\ cmsis_version.h
  • ドライバー\ CMSIS \インクルード\ core_cm0.h
  • ドライバー\ CMSIS \デバイス\ ST \ STM32F0xx \インクルード\ stmf0xx.h
  • ドライバー\ CMSIS \デバイス\ ST \ STM32F0xx \インクルード\ stm32f072xb.h
  • ドライバー\ CMSIS \デバイス\ ST \ STM32F0xx \インクルード\ system_stm32f0xx.h

でCMSIS \ SRC：

  • ドライバー\ CMSIS \デバイス\ ST \ STM32F0xx \ソース\テンプレート\ system_stm32f0xx.c

起動時：

  • ドライバー\ CMSIS \デバイス\ ST \ STM32F0xx \ソース\テンプレート\ gcc \ startup_stm32f072xb.s

これは、完成したフォームでプロジェクトがどのように見えるかです。


プロジェクトフォルダでの操作が行われたため、設定で表示する必要があります。

プロジェクト名の右ボタン->プロパティ-> C / C ++ビルド->設定->ツール設定-> MCU GCCリンカー->一般 -ここで、[ 参照... ]ボタンを使用してリンカースクリプトの新しい場所を指定する必要があります


また、プロジェクトへのパスを指定する必要があるファイル

のプロパティを持っている- >付属-一般の> C / C ++
]> [ソースの場所を-プロパティが持っている-一般の> C / C ++

に付属のパスフォルダ株式会社、およびソースの場所は、論理-フォルダだったSRCが、その場合、これらのフォルダーはプロジェクトツリーに個別に追加されます。ツリーを視覚的に乱雑にしないために、ソースの場所で、ルートフォルダーCore、CMSIS、およびStartupを指定できます。



プロジェクトをコンパイルするには、stm32f0xx.hファイルでマイクロコントローラーの名前を含む行のコメントを外し、もちろんmain.cに main関数を追加する必要があります。



実際にはすべて。エラーのないコンパイルで、1.5キロバイトものRAMメモリがRAMになり、すぐにスタックとヒープが記憶されます。プロジェクトの作成プロセスでは、どこにも言及されていません。スタックとヒープのサイズは、拡張子が.ldのリンカースクリプトファイルに示されています。これらは、プロジェクトの要件に応じて変更でき、変更する必要があります。これらの値は、ファイルの先頭に、サイズが16進数のラベル_Min_Heap_Size / _Min_Stack_Sizeの形式で配置されています。



例として、従来のLED点滅の小さなプロジェクトを挙げます。

STM32F072B-DISCOデバッグボードでLEDが点滅し、48 MHzの周波数で内部HSI48ジェネレーターからクロッキングが実行され、SysTickタイマーが遅延ソースとして使用され、1 msの周期で割り込みを生成します。これにより、正確な遅延時間がカウントされます。LEDは、プッシュプル出力用に構成されたポートCのピン6に接続されています。

この情報が誰かに役立つことを願っています。かつて、STM32プログラミング資料が豊富であるにもかかわらず、今では明白に見えることを実現するために、多くのマニュアルをシャベルで削らなければなりませんでした。
