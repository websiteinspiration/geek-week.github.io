Perl 5.30がリリースされました

昨日5月23日、Perl 5.30.0がリリースされました。このニュースは、Perlの主要開発者の1人であるSawyer Xによってperl.perl5.porters Usenetグループで発表されました。


約11か月前にリリースされた以前の安定版リリース5.28.0と比較して、約620,000行のコードが変更され、変更は1300ファイルに影響し、58人の作成者が開発に参加しました。ソースコード自体の変更（.pm、.t、.c、および.hファイルのみ）は、約510,000行および750ファイルと推定されます。


次のブランチ5.31の開発はオープンです。次の安定版リリースは2020年5月に予定されています。


主な変更点：


  • Perl API C関数sv_utf8_downgradeおよびsv_utf8_decodeは、実験的なものとは見なされなくなりました。
  • たとえば、「（？<= Foo？）」や「（？<！Ba {1,9} r）」などの可変長後読み式の実験的サポートを実装しました（以前はエラーが発生していました）
  • "{m、n}"正規表現のブロック内のサイズ指定子（ "n"）の最大値が32767から65534に増えました。
  • Unicode 12.1のサポート。
  • Unicodeプロパティ値の仕様に制限付きのワイルドカードサポートが追加されました（正しく変換する方法がわかりません）。たとえば、式「qr！\ P {nv = /（？X）\ A [0-5] \ z /}！」タイ語またはベンガル語の数字を含め、0〜5の数字を定義するすべてのUnicode文字を選択できます。
  • qr '\ N {name}'のサポートが実装されました（正規表現内の名前付き文字は一重引用符に制限されます。以前の正規表現はエラーを引き起こしていました）。
  • これで、ロケール（-Accflags = '-DUSE_THREAD_SAFE_LOCALE'）でスレッドセーフな操作のみを使用してPerlをコンパイルできます。
  • 「-Dv」フラグ（拡張デバッグ出力）と「-Dr」フラグ（正規表現デバッグ）の組み合わせにより、すべての可能な正規表現デバッグモードが含まれるようになりました。
