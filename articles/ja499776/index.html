前書き


MSH言語はMUMPS言語の概念に基づいて構築されています。 MUMPSは、前世紀に開発されたあまり知られていない言語です。しかし、それはまだ情報アプリケーションで使用されています。この言語に関する情報はインターネット上にあります。この言語の実用的な実装とそれをサポートするプログラマーのコミュニティがあります。 MUMPSは米国とロシアで開発されています。また、中南米、ドイツ、オーストラリア、中国でも使用されています。一般的に、この言語の生きている概念。 MUMPSと会うとき、その古風な性質が印象的です。この開発は、その利点、シンプルさ、一貫性、データ編成を維持しながら、欠点を排除するように設計されています。
プログラムの構成。

翻訳単位はMSH言語モジュールです。モジュールは、言語バージョンを識別する標準の8バイトから始まります。行の先頭には、ラベルまたはスペースがあります。ラベルは「：」で終わり、任意の数のスペースで残りのコマンドから分離されます。行はコマンドで構成されています。チーム終了の合図は「;」の記号です。コマンドと引数はスペースで区切られています。文字の大文字小文字は関係ありません。コマンドは、任意のレジスタの文字で書き込むことができます。さらに、多くのチームには省略形があります。コマンドには、その実行の条件があります。存在する場合、記号「？」はスペースなしでコマンドの後に続きますこのコマンドの実行条件。コマンドの実行条件が0でない場合、コマンドが実行されます。条件内ではスペースを使用できません。スペースを使用しないと、コマンドと引数の間の区切り文字として処理されます。

例：

SET？[5]> 5 Val [1] = 25; //正しい
SETですか？[2]> 5 Val [1] = 25; //構文エラー
SET？（[1,2]> 5）Val [1] = 25; //正しく、内側の（）スペースは許可されます。

引数は記号「、」で区切られます。引数の内部では、スペースは特殊文字ではなく、どこにでも含めることができます。通常、コマンドは任意の数の引数を持つことができます。モジュール内のラベルはモジュール内でローカライズされており、CASEコマンド内のラベルを除いて、モジュール内で一意である必要があります。このコマンド内にローカライズされたCASEコマンドのラベルは、このコマンド内でのみ一意である必要があり、このコマンドの外側とネストされたCASEコマンドの両方でラベルを複製できます。

ランタイム


実行時に、アプリケーションには1つ以上のタスクがあります。すべてのタスクは並行して実行されます。各タスクでは、プログラムが順次実行されます。常に、タスクで実行されるプログラムコードは1つだけです。タスクは最後に実行中のプログラムの終了で終了します。メインタスクは言語ランタイムによって起動されます。残りのジョブはJobコマンドによって生成されます。

データ管理


MUMPS言語に精通している人にとって、MSHでのデータの構成は非常に明確です。 MSHにはデータの説明がありません。データ宣言はありません。データはツリーとして保存でき、ツリーノードへのアクセスはオプションの名前とインデックスによって実行されます。インデックスは角括弧[]で囲まれています。名前は彼らの前にあります。

例：

SET Pr [4,5、 "rt"] = Is ["ty ^ 578"];
ここでPrとIsは木の名前です。4,5、「rt」および「ty ^ 578」はノードインデックスです。

ツリーには任意の数のレベルを含めることができ、したがって、インデックスには対応する数のフィールドがコンマで区切られます。インデックスフィールドには、基本タイプの任意の値を指定できます。 MSHの基本的なタイプは数値と文字列です。記録されたノードのみがツリーに直接格納されます。名前、インデックスフィールド、およびインデックス自体を式にすることができます。計算後、名前は識別子のみとなります。

また、データは連続配列として格納でき、配列要素へのアクセスはオプションのシンボル名「$」とインデックスで構成されます。インデックスは整数です。最小の配列インデックスは1です。名前とインデックスは式にすることができます。計算後、名前は識別子のみになり、インデックスは整数になります。配列は任意の順序で埋めることができますが、書き込んだ場合$ 1000 MC、その後、MCの配列の1000個の要素が作成されます。未定義の要素には値は含まれませんが、存在します。配列内の要素の数は、この配列のゼロ要素を参照することで確認できます。

例：mc $ 0
この要素に新しい配列の長さを書き込むことにより、配列のサイズを変更できます。ただし、一般的には、配列は自動的に拡張されるため、これは必要ありません。

ツリーノードと配列要素には、基本タイプのデータが含まれています。これらは文字列または数値です。プログラマデータの格納方法は関係ありません。データ型の格納とその操作は、MSH言語実装の責任です。

データのローカリゼーション


MSHデータはグローバルとローカルに分けられます。名前のタイプが異なります。グローバルデータは長期メモリに保存され、アプリケーションの寿命に依存しません。それらが作成されると、それらはアプリケーションによって変更でき、アプリケーションがKILLコマンドでそれらを破棄するまで存在します。すべてのグローバルデータの名前には接頭辞「^」が付いています。

グローバルデータへのアクセスは、さまざまなタスクから同時に可能です。したがって、グローバルにアクセスするときは、同期が必要です。このような同期は常に自動です。グローバル記述子には常に同期プリミティブがあり、グローバルへのアクセスを制御します。さらに、グローバルの読み取りは読み取りによってブロックされ、グローバルへの書き込みは書き込みによってブロックされます。追加のグローバル同期は必要ありません。グローバル配列へのアクセスも同期されます。

例：

^ gl [87,9] -グローバルツリーノードへのアクセス。

^ glar $ 45-グローバル配列の要素へのアクセス。

ローカルデータは、アプリケーションの実行中にのみ存在します。次回のアプリケーションの起動では、前回の起動のローカルデータにアクセスできません。

ローカルデータのスコープは、そのタイプによって異なります。データのローカリゼーションには3つのタイプがあります。

1.ローカルプログラムデータ。これらはプログラム内にローカライズされ、プログラムが起動されてから完了するまで存在します。プログラムがサブルーチンを呼び出すと、新しいサブルーチンデータが作成され、サブルーチン内のローカルプログラムデータは表示されません。プログラムに戻ると、ローカルプログラムデータが再び使用可能になります。ローカルプログラムデータには名前がありません。

例：

[7,9] -プログラム内にローカライズされたツリーのノードへのアクセス。

$ 5-プログラム内にローカライズされた配列要素へのアクセス。

例外が1つあります。A $プログラムに渡されるパラメーターの配列もプログラム内でローカライズされます。

2.ローカルアプリケーションデータ。それらは、アプリケーションのすべてのタスクで表示されます。どのタスクからでも連絡できます。これらは、タスクによってアプリケーションで作成された瞬間から、アプリケーションが完了するまで、またはKILLチームによって破棄されるまで存在します。そのようなデータの名前には、「％」という接頭辞が付きます。これらの変数は異なるタスクで同時に使用できるため、グローバルと同じように同期されます。

例：

％dapp [87.9] -アプリケーション内にローカライズされたツリーのノードへのアクセス。

％dapp $ 45-アプリケーション内にローカライズされた配列要素へのアクセス。

3.ローカルジョブデータ。これらはタスク内にローカライズされ、タスクプログラムで作成された瞬間から、KILLチームによってタスクが完了するか破棄されるまで存在します。このようなデータには名前が必要で、接頭辞「^」および「％」は含まれません。例外は、プログラムパラメータA $の配列で、プログラム内にローカライズされます。

例：

djob [87,9] -タスク内にローカライズされたツリーのノードへのアクセス
djob $ 45-ジョブ内にローカライズされた配列の要素へのアクセス。
変数へのアクセスは、ここにリストされているタイプのみを持つことができます。

定数チーム


CONSTANT コマンドを使用して、定数に名前を付けることができます。これらは放送時間名です。ブロードキャストはそれらをそれらの値に置き換えます。実行時には、これらの名前は存在しません。したがって、CONSTANTコマンドで名前に式を割り当てることはできません。値は正確に定数でなければなりません。定数の名前は、引用符なしでプログラムで指定された定数の値と一致しないように選択する必要があります。

例：
定数ioInOut = "/ini/par.ini>,maxIs=25;
ioInOutおよびmaxIs定数には値が割り当てられます。さらにプログラムでは、これらの値の代わりにこれらの名前を使用できます。定数
チームこれには2つの形式があります。この場合、等式の右側はありません。このチームの意味は異なります。名前は、定数を含むモジュールの名前です。このモジュールの定数は現在のモジュールにエクスポートされます。定数をインポートするモジュールには、インポートに関する追加の説明は含まれていません。インポートモジュールには、定数とプログラムの両方のみを含めることができます。
例：
CONSTANT sysCnsNet、usrCnsByx;
sysCnsNetおよびusrCnsByxは、定数を含むモジュール名です。
どちらの形式も、単一のCONSTANTコマンドの引数として使用できます。

XECUTEチーム


XECUTE コマンドはかなりエキゾチックなチームですが、それはMUMPS言語で存在します。他のプログラミング言語では、彼女はJavaScriptでのみ私に会いました。そこではevalと呼ばれます。このコマンドを実行すると、このコマンドの引数の式が計算され、この式が文字列に変換されてMSHコマンドとして解釈され、実行されます。

例：
XECUTE "SET $ 1 = 89;";
その結果、変数$ 1は値89を受け取ります。
これは原始的な例です。この形式では、このコマンドを使用することはほとんど意味がありません。XECUTEコマンドの引数は式であり、プログラムの実行時にさまざまなMSHコマンドを生成できます。
コマンドは、コマンドが配置されているプログラムのコンテキストで実行されます。ローカルプログラムデータを含むすべてのプログラムリソースを利用できます。

COPYおよびMOVEコマンド


COPYコマンドがあると似MERGEの MUMPSのコマンド。これらのコマンドは、すべての子孫と共にソースノードをレシーバーノードにコピーします。これらのコマンドの引数は、記号「=」で区切られた2つのフィールドで構成されています。この記号の右側はソースノード、左側はレシーバーです。ノードとして省略リンクを使用できます。この場合、ツリー全体が使用されます。これらのコマンドは、子孫だけでなくノード自体もコピーします。COPYコマンドを実行するデータのマージ。ソースデータはクリーニングせずにレシーバーにコピーされます。ソースは変更されません。MOVEチーム実際にデータを移動します。以前は、レシーバーがクリーンアップされ、次にソースノードのすべての子孫がコピーされ、ソースノードはそのすべての子孫とともに削除されました。

例：
//ノードus [5,6]はノード[1]にコピーされます
COPY [1] = us [5,6];
// usツリー全体がノードに移動されます[8]
MOVE [8] = us [];
これらのコマンドは、アレイのコピーにも使用できます。この場合、ソースとレシーバーとして使用できるのは短縮リンクのみです。
プログラムの引数がarg配列にコピーされます。
COPY arg $ = A $;
MOVEコマンドを使用して移動できます。
MOVE a1 $ = b1 $;
任意のアレイをコピーして移動できます。

リソース同期


複数のタスクを実行する場合、共有リソースにアクセスする必要があります。同期はロックコマンドによって実行されます。同期コマンド自体は何もブロックしません。これは、共有リソースへのアクセスを区別できるようにする一連の合意です。ロックコマンドがジョブ間で共有されていない場合、アクセス同期は行われません。同期はロック名に基づいて構築されます。これらの名前はアプリケーション内でローカライズされ、すべてのタスクで同じです。ロックでは、多くのリーダーと一度に1人のライターのみの概念が受け入れられます。したがって、読み取りロックと書き込みロックがあります。LockR

チーム名前の読み取りをブロックします。彼女の議論にリストされた名前は読書によってブロックされます。任意の数のタスクが同じ名前をブロックできますが、これらの名前のいずれかをブロックしようとしたタスクは、これらの名前がす​​べてロック解除されるまで待機します。書き込み名をキャプチャした後は、書き込みロックが名前を解放するまで、読み取りロックコマンドを実行できません。ロックできない場合、コマンドは名前が解放されるのを待ちます。

例：

これらの名前は読み取りによってブロックされます。もう一つの課題はまた、彼らはロックを解除するのを待たずにこれらの名前をロックすることができます。LockW

コマンドレコードで名前をブロックします。彼女の議論にリストされた名前はレコードによってブロックされます。引数にリストされた名前がロックコマンドによってすでにブロックされている場合、このコマンドはこれらの名前の解放を待ちます。
例：

これらの名前はレコードによってロックされます。LockUn
コマンドは、この名前のロックを解除します。数回読むことで名前がブロックされている場合は、同じ回数ロックを解除する必要があります。
例：

標準関数には、タイムアウトのあるこれらのコマンドに類似しています。

SETコマンドの短縮形


SET コマンドは省略形です。この場合、等式の左側は存在せず、式で最後に言及された変数がその役割を果たします。
例：
SET $ 1 = 2、$ 1 + 3;
変数$ 1は5と等しくなります。
式に複数の変数がある場合、結果は最後の変数に割り当てられます。

例：
SET $ 1 = 1、$ 2 = 2、$ 3 = 3、$ 1 + $ 2 + $ 3; $ 3
変数は1 + 2 + 3 = 6になります。このフォームは、最初の例と同様に、非常に単純なケースでのみ使用するのに適しています。2番目の例は、この形式のSETコマンドの機能の例としてのみ提供されています。

チームをブロックする


ブロックコマンドは、コマンドのブロックを形成し、同時にブロックの見出しとして機能します。ブロック内にコマンドが1つしかない場合でも、各ブロックコマンドには独自のENDコマンドが必要です。

IFチーム


IF コマンドは、このコマンドの実行条件が真の場合に実行されるブロックを形成します。このコマンドには引数はありません。このブロックにはELSEコマンドが含まれる場合があります。ELSEコマンドには引数がありません。IFブロックの外では、これらのコマンドには意味がなく、適用できません。ある場合はELSEコマンド内のIFブロックの条件IFコマンドが満たされ、後ろにコマンドのみIF実行されているコマンドの次まで、ELSEコマンドが。ELSEチーム実行条件が含まれている可能性があります。この条件が真の場合、このコマンドの後にあるコマンドのみが、次のELSEコマンドまたはENDコマンドまで実行されます。IFブロックには、実行条件のないELSEコマンドを1つだけ含めることができ、ELSEコマンドの中で最後でなければなりません。

例：
IF？[6] <0;
SET y [1] = 1;
ELSE？[6] <5;
SET y [1] = 2;
ELSE？[6] <10;
SET y [1] = 3;
そうしないと SET y [1] = 4;
終わり
このコマンドの実行条件が存在しない場合、このブロックはどのような場合でも実行されます。なぜこれが役立つのか想像するのは難しいですが。

ケースチーム


このコマンドのセマンティクスは、他のMSHチームのセマンティクスとは多少異なります。その中でコマンドを実行するための条件はそうではありません。CASEコマンドの実行の条件では、式は、制御が転送されるラベルを計算する必要があります。このコマンドには引数はありません。

このコマンドの各ラベルは、このラベルから次のラベルへのブロックを形成します。制御がラベルに移されると、次のラベルまでコマンドのみが実行され、その後、現在のCASEブロックが終了します。このコマンドは、CスイッチコマンドよりもPascal表記に近いものです。ラベルの名前を計算した結果、現在のCASEブロックに欠落している名前が見つかった場合、CASEコマンドの後で最初のラベルの前にあるコマンドが実行されます。

イベント処理


イベント処理は、高水準言語には通常含まれていない強力なプログラミングメカニズムです。しかし、ライブラリとオペレーティングシステムにおけるその存在は、その重要性を物語っています。ビジュアルコンポーネントのライブラリは、このメカニズムに基づいて構築されています。言語のイベント処理の存在は、言語の機能を拡張します。新しいイベント駆動型プログラミングスタイルを開きます。ちなみに、アセンブラーのような尊敬されているプログラミング言語はそのような機能を持っています。したがって、このメカニズムはMSH言語に追加されました。

このメカニズムの基本はイベントです。アプリケーション内でローカライズされています。したがって、これらの名前はアプリケーション全体で一意である必要があります。イベント名がCONSTANTコマンドで宣言された名前と一致する場合の場合、イベント名は宣言された定数の値に置き換えられます。注意してください。イベントに名前を付けるときは、なんらかの命名戦略に従うことをお勧めします。たとえば、evuで始まるイベント名を割り当てます。名前の長さは、UTF8エンコーディングを考慮して、18バイトを超えてはなりません。この制限は、現在の言語実装にのみ関連しています。

1つのタスクで作成されたイベントは表示され、任意のタスクで処理できます。イベントハンドラは任意の数にすることができ、さまざまなタスクで使用できます。
イベントが発生すると、このイベントのハンドラーがあるかどうかがチェックされ、ハンドラーがない場合はイベントが削除されます。ハンドラーがある場合、それらは順次実行されます。イベントには、システムとユーザーがあります。システムイベントはシステムによって生成されます。カスタムイベントは、EVENTTRAPコマンドによって生成されます。プログラムを呼び出すときのように、イベントはハンドラーにパラメーターを渡します。処理後、イベントハンドラーは削除されません。イベントを削除するには、EVENTDELETEコマンドを使用します。イベントは、EVENTCALLおよびEVENTWAITコマンドで処理できます。

オペレーション


MSHでの操作は特別な役割を果たします。データ型の変換を制御するのは彼らです。操作に応じて、オペランドは目的のデータ型にキャストされます。データの結果のタイプは、操作のタイプに一意に対応します。 Cのような言語の+演算のように、文字列と数値演算はオーバーラップしません。 MSHの演算のタイプは、オペランドのタイプに依存せず、すべてが正反対です。 MSHには操作の優先順位はありません。これはMUMPSの歴史的な遺産です。

例：
SET $ 1 = 2 + 3 * 4;
$ 1は14ではなく20になります。
結果が14になるには、括弧が使用されます。
セット$ 1 = 2 +（3 * 4）;
文字列を結合する操作として、記号「_」を使用します。
操作の優先順位の欠如は珍しいですが、非常に便利です。優先順位の自然さへの言及は、操作が合計以上の場合、非常に疑わしくなります。優先順位がないことを一度学んだので、優先順位を痛々しく思い出す必要はありません。一般に、これは習慣の問題です。

最新のプログラミング言語でのオブジェクトの存在は良い形です。一般的な場合、オブジェクトは2つの部分で構成されます。宣言的な説明の一部と実装の一部。 MUMPSシステムでは、変数には型宣言の宣言部分はありません。クラスは基本的にユーザーデータ型です。 MUMPSの原則に違反しないようにするために、MSHにはクラスの説明の宣言部分がありません。結局のところ、それなしで完璧に実行できます。実装の一部のみがクラスに残されています。クラスの実装は、標準モジュールで完全に表すことができます。クラスのみが、オブジェクトのプロパティの説明に関連する追加の規則を導入する必要がありました。オブジェクトはツリー内にのみ存在できます。オブジェクトのプロパティを格納する場所がないため、オブジェクトを配列に配置すると失敗します。オブジェクトにプロパティがない場合でも、試すことができます。どんなオブジェクトなのか。
パブリックプロパティを説明するには、このプロパティの値を返す関数が必要です。プロパティ名が必要であり、クラス名を持つモジュール内になければなりません。プロパティに書き込むプログラム。このプログラムには、プロパティ名の前に「。」が付いています。そして、プロパティ削除プログラム。このプログラムには、プロパティ名の前に「..」が付いています。

モジュール内の関数名は、読み取り用のパブリックプロパティに対応している場合があります。この場合、そのような関数の戻り値は、パブリックプロパティの値として呼び出し側プログラムに渡されます。

オブジェクトの継承


MSHクラスは多重継承をサポートします。PARENTコマンドは、このクラスのすべての祖先を設定します。さらに、コマンド引数におけるこのクラスの祖先の名前の順序は、継承の優先順位を決定します。クラスが後で言及されるほど、優先順位は低くなります。

例：
PARENT USER、BOX;
クラスはUSERおよびBOXの祖先から継承します。USER祖先の優先度が高くなります。優先順位は何ですか。オブジェクトにアクセスすると、パブリックプロパティまたはクラスメソッドがクラス自体で検索されます。見つからない場合は、優先度が最も高い祖先で検索され、次にこのクラスの祖先で検索されます。
