Cisco 200-125 CCNA v3.0のトレーニング。50日目。EIGRPの構成

今日は引き続きICND2コースのトピックのセクション2.6を学習し、EIGRPプロトコルの構成と検証を検討します。 EIGRPの設定は非常に簡単です。 RIPやOSPFなどの他のルーティングプロトコルと同様に、ルーターのグローバル構成モードに入り、router eigrp <＃>コマンドを入力します。＃は自律ASシステムの番号です。



この番号はすべてのデバイスで同じである必要があります。たとえば、5つのルーターがあり、それらすべてがEIGRPを使用する場合、それらは同じ自律システム番号を持つ必要があります。 OSPFでは、これはプロセスIDまたはプロセス番号であり、EIGRPでは、これは自律システム番号です。

OSPFでは、プロセスIDの近接性を確立するために、異なるルーターが一致しない場合があります。 EIGRPでは、すべてのネイバーのAS番号は必ず一致する必要があります。一致しない場合、ネイバーフッドは確立されません。 EIGRPプロトコルを有効にする方法は2つあります。リバースマスクを指定することも、ワイルドカードマスクを指定することもありません。

最初のケースでは、タイプ10.0.0.0のクラスIPアドレスがnetworkコマンドで指定されています。これは、IPアドレス10の最初のオクテットを持つすべてのインターフェイスがEIGRPルーティングに参加することを意味します。つまり、この場合、ネットワーク10.0.0.0のすべてのクラスAアドレスが含まれます。リバースマスクを指定せずに正確なサブネットタイプ10.1.1.10を入力しても、プロトコルはそれを10.0.0.0の形式のIPアドレスに変換します。したがって、いずれの場合でも、システムは指定されたサブネットのアドレスを受け入れますが、それはクラスアドレスと見なされ、IPアドレスの最初のオクテットの値に応じて、クラスA、B、またはCのネットワーク全体で機能します。

EIGRPを10.1.12.0/24サブネットで実行する場合は、network 10.1.12.0 0.0.0.255という形式の逆マスクコマンドを使用する必要があります。したがって、クラスアドレッシングネットワークでは、EIGRPはリバースマスクなしで機能し、クラスレスサブネットではワイルドカードマスクの使用が必須です。

Packet Tracerに移動して、前のビデオチュートリアルのネットワークトポロジを使用してみましょう。この例では、FDとRDの概念を理解しました。



このネットワークをプログラムで構成し、どのように機能するかを確認します。 5台のR1-R5ルーターがあります。 Packet TracerがGigabitEthernetインターフェイスを備えたルーターを使用しているという事実にもかかわらず、このスキームが前述のトポロジーに一致するように、ネットワーク帯域幅と遅延を手動で変更しました。 10.1.1.0/24ネットワークの代わりに、アドレス10.1.1.1/32が割り当てられたR5ルーターに仮想ループバックインターフェイスを接続しました。



R1ルーターの設定から始めましょう。ここにはまだEIGRPを含めていませんが、ルータにIPアドレスを割り当てました。 config tコマンドを使用して、グローバル構成モードに入り、router eigrp <autonomous system number>コマンドを入力してプロトコルをオンにします。これは、1から65535の範囲である必要があります。番号1を選択し、Enterキーを押します。さらに、先に述べたように、2つの方法を使用できます。

ネットワークとネットワークIPアドレスをダイヤルできます。ネットワーク10.1.12.0/24、10.1.13.0/24、および10.1.14.0/24は、R1ルーターに接続されています。これらはすべて「10番目」のネットワーク上にあるため、1つの共通ネットワーク10.0.0.0コマンドを使用できます。 Enterキーを押すと、EIGRPが3つのインターフェイスすべてで起動されます。これを確認するには、do show ip eigrp interfacesと入力します。プロトコルが2つのギガビットイーサネットインターフェイスとR4ルーターが接続されている1つのシリアルインターフェイスで実行されていることがわかります。



確認のために再度do show ip eigrp interfacesコマンドを入力すると、EIGRPが実際にすべてのポートで機能していることを確認できます。

R2に移動して、config tコマンドとrouter eigrp 1コマンドを使用してプロトコルを実行しましょう。今回は、ネットワーク全体に対してコマンドを使用するのではなく、リバースマスクを使用します。これを行うには、network 10.1.12.0 0.0.0.255コマンドを入力します。設定を確認するには、do show ip eigrp interfacesコマンドを使用します。入力されたコマンドのパラメーターに対応するのはこのインターフェイスのみであるため、EIGRPはGig0 / 0インターフェイスでのみ起動されることがわかります。



この場合、リバースマスクは、IPアドレスの最初の3オクテットが10.1.12に等しいすべてのネットワークでEIGRPモードが有効であることを意味します。同じパラメーターのネットワークがいくつかのインターフェースに接続されている場合、このインターフェースは、このプロトコルが実行されているポートのリストに追加されます。

R5ルーター、つまり10.1.1.0/24ネットワークに注目しましょう。これはルーティングテーブルの最初の行です。括弧内の最初の数字はアドミニストレーティブディスタンスです。これはEIGRPプロトコルの場合は90です。文字Dは、このルートに関するデータがEIGRPプロトコルによって提供されることを意味し、括弧内の2番目の数値（26112に等しい）は、ルートR2-R5のメトリックです。前のスキームに戻ると、ここではメトリック値が28416であることがわかります。そのため、この不一致の理由を確認する必要があります。

R5の設定でコマンドshow interface loopback 0を入力します。その理由は、ループバックインターフェイスを使用したためです。回路のR5遅延を見ると、10μsであり、ルーターの設定では、DLY遅延が5000マイクロ秒であるという情報を取得しています。この値を変更できるかどうか見てみましょう。グローバルコンフィギュレーションモードR5に入り、interface loopback 0およびdelayコマンドを入力します。システムは、遅延値を1〜16777215の範囲で、数十マイクロ秒単位で割り当てることができるというヒントを与えます。 10で10μsの遅延値は1に対応するので、delay 1コマンドを入力します。インターフェースパラメーターをもう一度確認すると、システムがこの値を受け入れなかったことがわかります。R2設定でネットワーク設定を更新する場合でも、これを実行したくありません。
ただし、R5ルーターの物理パラメーターを考慮して前のスキームのメトリックを再計算すると、R2からネットワーク10.1.1.0/24までのルートの実行可能な距離は26112になります。コマンドを入力して、R1ルーターのパラメーターの同様の値を見てみましょうIPルートを表示します。ご覧のとおり、ネットワーク10.1.1.0/24に対して再計算が実行され、メトリック値は28416ではなく26368になりました。

この再計算は、前のビデオチュートリアルのスキームを基に、パケットトレーサーの機能を考慮に入れて確認できます。パケットトレーサーは、他のインターフェイスの物理パラメーター、特に異なる遅延を使用します。これらの帯域幅と待ち時間の値を使用して独自のネットワークトポロジを作成し、そのパラメーターを計算してみてください。あなたの練習では、そのような計算を実行する必要はなく、その方法を知っているだけです。前回のビデオで説明したロードバランサーを使用する場合は、遅延を変更する方法を知る必要があるためです。 EIGRPを微調整するには、帯域幅を変更することはお勧めしません。遅延値を変更するだけで十分です。
したがって、帯域幅と遅延の値を変更して、EIGRPメトリックの値を変更できます。これが宿題になります。いつものように、このために私たちのサイトからダウンロードして、Packet Tracerで両方のネットワークトポロジを使用できます。スキームに戻りましょう。
