パート1. REST APIの作成：Webサーバーの基本


Webサーバーは、REST APIの最も重要なコンポーネントの1つです。この投稿では、いくつかの初期ディレクトリとファイルを作成してREST APIプロジェクトを開始します。次に、Webサーバーモジュールを作成してプラグインし、Webサーバーが正しく起動およびシャットダウンするようにします。

このプロジェクトのコードは、必要に応じて調整および構築できる共通のディレクトリ構造を使用して編成されます。

cd ~
mkdir hr_app
cd hr_app/
touch index.js
mkdir config
touch config/web-server.js
mkdir controllers
mkdir db_apis
mkdir services
touch services/web-server.js

index.jsファイルは、アプリケーションの「メイン」ファイルと見なすことができます。これは、アプリケーションへのエントリポイントになります。このファイルと、configおよびservicesディレクトリのweb-server.jsファイルにコードを追加します。

次のコードをファイルHome> hr_app> config> web-server.jsに貼り付けます。

module.exports = {
  port: process.env.HTTP_PORT || 3000
};

Node.jsでは、プロセスオブジェクトには、ユーザー環境を含むenvプロパティがあります。これを使用して、ポート値をHTTP_PORT環境変数の値に設定します。この環境変数が定義されていない場合、デフォルト値は3000です。

次のコードをファイルに貼り付けますHome> hr_app> services> web-server.js

const http = require('http');
const express = require('express');
const webServerConfig = require('../config/web-server.js');
 
let httpServer;
 
function initialize() {
  return new Promise((resolve, reject) => {
    const app = express();
    httpServer = http.createServer(app);
 
    app.get('/', (req, res) => {
      res.end('Hello World!');
    });
 
    httpServer.listen(webServerConfig.port)
      .on('listening', () => {
        console.log(`Web server listening on localhost:${webServerConfig.port}`);
 
        resolve();
      })
      .on('error', err => {
        reject(err);
      });
  });
}
 
module.exports.initialize = initialize;

1〜3行目：複数のモジュールが必要です。 httpモジュールはNode.jsに含まれていますが、Expressモジュールはnpmを介してインストールする必要があります。

7〜27行目： initializeという名前の関数が宣言されています。関数はすぐにpromiseを返します。これは、Webサーバーが正常に起動したかどうかに応じて許可または拒否されます。

9〜10行目：新しいエクスプレスアプリケーションが作成され（実際には単なる関数です）、httpモジュールを介してhttpサーバーを作成するために使用されます。

12-14行目：アプリケーションのgetメソッドを使用して、ルートパス（/）にあるGETリクエストのハンドラーを追加します。コールバック関数は、そのような要求を受信すると呼び出され、「res」（res）パラメーターを使用して「Hello World！」応答を送信しますお客様へ。

16〜24行目：サーバーのリスニングメソッドを使用して、指定されたポートにバインドし、着信要求のリスニングを開始します。

28行目：モジュールを外部で使用できるようにエクスポートする

async function startup() {

Webサーバーモジュールを接続し、非同期起動関数を決定して呼び出します。 Webサーバーの初期化関数はpromiseを返すため、async / awaitで使用して、try-catchブロックでラップできます。初期化関数が成功した場合、Webサーバーは機能します。そうしないと、例外がキャッチされて処理されます。 npmを初期化してExpressをインストールするだけで、アプリケーションを起動できます。ターミナルでhr_appディレクトリから次のコマンドを実行します。

npm init -y
npm install express -s
node .

npm initコマンドは、npmがマニフェストファイルとして使用するpackage.jsonファイルを作成するために使用されます（-yフラグはデフォルトオプションを受け入れます）。npm installコマンドを使用して、expressをインストールします（-sフラグは、expressをpackage.jsonの依存関係リストに追加します）。Npmは、インストールしたモジュールをnode_modulesディレクトリに格納します。また、package.lock.jsonというファイルを作成して、開発チームに同一のツリーを提供します。localhost：3000で待機して

制御されたシャットダウン


シャットダウン中にCtrl + Cを押すと、これがどのように発生したかを制御できなくなります。シャットダウンプロセスを制御するには、Webサーバーを明示的に閉じ、Node.jsを終了する必要があります。

次のコードをファイルに貼り付けますHome> hr_app> services> web-server.js

// *** previous code above this line ***
 
function close() {
  return new Promise((resolve, reject) => {
    httpServer.close((err) => {
      if (err) {
        reject(err);
        return;
      }
 
      resolve();
    });
  });
}
 
module.exports.close = close;

close関数は、Webサーバーが正常に閉じられたときに解決されるpromiseを返します。 httpServer.closeメソッドは、新しい接続の確立を停止しますが、すでに開いている接続を強制的に閉じることはありません。開いている接続の数とその接続によっては、コールバックが機能するまで少し待つ必要がある場合があります。このモジュールではこれを行いませんが、カスタムコードまたはhttp-shutdownなどのnpmモジュールを使用して、開いている接続を強制的に閉じることができます。

次のコードをファイルHome> hr_app> index.jsに貼り付けます。

// *** previous code above this line ***
 
async function shutdown(e) {
  let err = e;
    
  console.log('Shutting down');
 
  try {
    console.log('Closing web server module');
 
    await webServer.close();
  } catch (e) {
    console.log('Encountered error', e);
 
    err = err || e;
  }
 
  console.log('Exiting process');
 
  if (err) {
    process.exit(1); // Non-zero failure code
  } else {
    process.exit(0);
  }
}
 
process.on('SIGTERM', () => {
  console.log('Received SIGTERM');
 
  shutdown();
});
 
process.on('SIGINT', () => {
  console.log('Received SIGINT');
 
  shutdown();
});
 
process.on('uncaughtException', err => {
  console.log('Uncaught exception');
  console.error(err);
 
  shutdown(err);
});

SIGINTおよびSIGTERMイベントは、たとえばctrl + cが押されたときに、プロセスにシャットダウンしてそれをシャットダウンできるシグナルに関連しています。uncaughtExceptionイベントは、JavaScriptエラーがスローされてもキャッチされず、try-catchステートメントを使用して処理されたときに発生します。アプリケーションをもう一度起動して閉じてください。ターミナルに「シャットダウン」メッセージが表示されると、すべてが正しく機能することがわかります。

サーバーへの登録


Webサーバーモジュールを完成させるもう1つ、HTTPロギングがあります。このタイプのログインに使用できるさまざまなモジュールがありますが、morganは単純なモジュールの1つです。npmを使ってmorganをインストールしましょう。

npm install morgan -s

次に、エクスプレスが必要な行（行2）の下のservices / web-server.jsに次の行を追加します。

const morgan = require('morgan');

これで、すべてのリクエストがapp.useを使用して処理されるミドルウェアとして、モーガン機能を有効にできます。 「hello world」というメッセージを表示するapp.getを呼び出す前にこの行を追加します。

// Combines logging info from request and response
    app.use(morgan('combined'));
 
    // *** app.get call below this line ***

app.useは、HTTP要求および応答とやり取りできるミドルウェア関数のパイプラインを作成することに注意してください。ミドルウェア機能は有効化された順に実行されます。アプリケーションを再起動し、ターミナルをインストールして、ブラウザーとブラウザーを同時に表示できるようにします。ページをリロードするたびに、ターミナルに新しいログエントリが表示されます。デフォルトでは、morganはログ情報をSTDOUT（端末に表示されます）に渡します。
