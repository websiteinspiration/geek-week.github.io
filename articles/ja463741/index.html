Firefox（既に修正済み）とChromeでは、Alt-Svcヘッダーでイントラネットポートをスキャンできます

更新：現在のFirefox 68では、脆弱性はすでに修正されています（おかげでドゥカット）


ボストン大学のTrishita TiwariとAri Trachtenbergは、ユーザーの内部ネットワークまたはローカルホスト上のホストのポートをスキャンするための新しい攻撃方法を示す論文を発表しました （CVE-2019-11728）。攻撃は、RFC-7838 HTTP代替サービス（2016年に承認）で導入されたAlt-Svc HTTPヘッダーを使用して実行されます。


Alt-Svc HTTPヘッダーを使用すると、リクエストを新しいホストとポートにリダイレクトできます。これは、負荷のバランスを取るために使用されます。 例えば：


Alt-Svc: http/1.1="other.example.com:443";ma=200;persist=1


HTTP / 1.1を使用して要求されたページを受信するために、ポート443でother.example.orgホストに接続するようブラウザーに指示します。 「ma」パラメータは、リダイレクトの最大期間を設定します。 HTTP / 1.1に加えて、HTTP / 2-over-TLS（h2）、HTTP / 2-overプレーンテキスト（h2c）、SPDY（spdy）、およびUDPを使用するQUIC（quic）プロトコルがサポートされています。



攻撃ホストは、繰り返されるリクエスト間の遅延を記号として使用して、内部ネットワークアドレスとネットワークポートを順次検索できます。リダイレクトされたリソースが利用できない場合、ブラウザは応答としてRSTパケットを即座に受信し、すぐに代替サービスを利用不可としてマークし、リクエストで指定されたリダイレクトライフタイムをリセットします。ネットワークポートが開いている場合、接続が完了するまでに時間がかかり（適切なパケット交換で接続を確立する試みが行われます）、ブラウザはすぐに応答しません。


これにより、ブラウザが接続に対して安全ではないと見なしているポート（Firefox、Chrome）への接続、たとえばメールやDNSサービスのポートへの接続に対する制限をバイパスすることもできます。
