同僚がReactに投票していることを知ったとき、私は驚きませんでした。ほとんどの調査参加者は、Reactを最高のJavaScriptライブラリの1つと見なしました。私たちの開発者は多くのプロジェクトでReactを使用しています。これにより、このツールの長所を明確に見ることができました。Reactを使用する人は誰でも、次の機能の組み合わせを利用できます。

アプリケーションの概要


私たちのアプリケーションは非常にシンプルに構成されています。これは、オンラインの学校活動を管理するのに役立つ音楽教師プログラムです。このプロジェクトは、教師が基本的なビジネスを行うのに役立ち、組織の問題を解決しやすくします。

このプロジェクトを作成する上での主な困難は、教師向けに設計されたコントロールパネルの開発でした。それは彼らが学生の活動を管理し、彼らの達成を追跡することを可能にします。Reduxライブラリエコシステムをアプリケーションのベースとして使用することで、この問題を解決しました。つまり、教師向けに設計されたアプリケーションのセクションを作成しました。このセクションを使用して、生徒の成績に関する情報を受け取り、新しい音楽レッスンをデモンストレーションし、コミュニケーションを取り、ゲームと何かを比較し、フィードバックを残しました。

次に、これらの各プロジェクトについて説明します。Reactから始めましょう。

1.反応する


JavaScriptの世界では、Reactは間違いなくリーダーです。このライブラリは、リアクティブプログラミングのアイデアを使用しており、フロントエンド開発と独自のコンセプトの多くを紹介しています。

Webプロジェクトの開発でReactを柔軟に使用するには、多くの追加ツールを学ぶ必要があります。たとえば、ここでは、Reactと組み合わせて使用​​できるライブラリによって提供されるツールの完全なリストではありません。これらは、Redux、MobX、Fluxy、Fluxible、RefluxJSです。React開発では、jQuery AJAX、Superagent、Axios、およびFetch APIを使用することもできます。

▍競合モード


今年の10月24日からのツイートです。これは、Reactでの、競合モード（コンカレントモード）をサポートするための実験的機能の紹介について報告しています。React開発者は常にこのモードを改善しています。ここでは、競合モードでのReact Conf 2019のパフォーマンスへのリンクや、React.lazyコンポーネント内での（で作成された）遅延コンポーネントのレンダリングなど、他のReactの革新的な機能へのリンクを見つけることができますReact.Suspense。これらのテクノロジーはどちらも、メインスレッドをブロックせずにレンダリングすることで、Reactアプリケーションの応答性を高めます。これにより、Reactは、ユーザーの操作に対するアプリケーションの応答の形成など、優先度の高いタスクの処理を遅延させません。

▍サスペンスおよびその他のテクノロジー


このテクノロジーReact.Suspenseは、Reactアプリケーションでの非同期データロードの処理を改善します。このテクノロジーについて簡単に説明すると、特定の条件が満たされるのを待つと同時に、アプリケーション全体の動作を中断させないようにコンポーネントを待機させることができると言えます。

React 16.8で導入されたもう1つの革新はフックです。Hooks Reactを使用すると、開発者はReactの最も重要な機能を使用できるため、クラスに基づくコンポーネントの使用を省略できます。これらの機能の中には、コンポーネントの状態の管理と、そのライフサイクルのメソッドの操作があります。 Reactにはいくつかの組み込みフックが含まれていますが、プログラマーは独自のフックを作成できます。

Reactアプリケーションは、インターフェースを形成するためのアプリケーションロジックとHTMLマークアップを含むコンポーネントで構成されています。コンポーネント間の相互作用を改善するために、開発者はFluxまたは同様のJavaScriptライブラリを使用できます。

Reactプログラミングは、コンポーネントの状態やプロパティなどの概念を使用します。それらは関連するオブジェクトによって表されます。それらを使用することで、コンポーネント内のデータストレージとコンポーネント間のデータ交換を整理できます。たとえば、コンポーネントによって実装されたプログラムロジックからアプリケーションインターフェイスにデータを転送したり、親コンポーネントから子コンポーネントにデータを転送したりします。

▍反応生態系要素


Reactライブラリーの周りには、さまざまなサポートツールやライブラリーに代表されるエコシステム全体が登場しています。このエコシステムのコンポーネントの一部を次に示します。

  • ReactライブラリーとReactルーター自体は、アプリケーションでルートを管理するためのツールです。
  • DOMで動作するように設計されたreact-domパッケージ。
  • FirefoxおよびChrome用のReact Developer Tools。
  • JSXは、JavaScriptコードでHTML要素を記述できるマークアップ言語です。
  • create-react-appコマンドラインツール。テンプレートReactプロジェクトを作成するために設計されています。
  • さまざまなサポートライブラリ。たとえば、アプリケーションの状態を制御するために使用されるReduxライブラリや、サーバーAPIとデータを交換するために使用されるAxiosライブラリなどです。

Reactについての話を締めくくると、このライブラリーは、高度なWebプロジェクトを作成することを計画している人にとって最良の選択であると言えるでしょう。

2.角度


AngularConnect 2019カンファレンスで、Angular開発チームは、Angular 9がこのフレームワークの開発のターニングポイントになることを可能にするステートメントを発表しました。特に、Angular Ivyコンパイラーをすべてのアプリケーションで使用できる標準ツールにする予定です。このテクノロジーの主な利点は、そのアプリケーションにより、開発プロセスの高速化、アプリケーションのサイズの縮小、生産性と信頼性の向上を可能にすることです。

Modern Angularは、フロントエンド開発用の高度なモジュール式フレームワークです。以前は、Angularをページに接続するには、適切なタグをHTMLコードに追加するだけで十分でしたが、開発者は必要なAngularモジュールをプロジェクトにインポートできました。

Angularはその柔軟性で知られています。これが、Angular 1.xバージョンがまだ関連している理由です。ただし、コンポーネントベースのアーキテクチャに向けて大幅に変更されたMVCフレームワークアーキテクチャのため、今日多くの開発者がAngular 2+を使用しています。

Angularを使いたい人は誰でも、このフレームワークを習得する際にいくつかの困難に直面する必要があります。したがって、Angularベースのアプリケーションを作成するには、TypeScriptを使用することがほぼ必須です。これはAngularでの作業を複雑にしますが、この状況には利点があります。特に、高度な型制御によりアプリケーションの信頼性が向上し、プログラマーに追加の開発ツールが提供されます。
