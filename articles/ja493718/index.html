ディスカッション：使用したことがなく、現在使用している標準のUNIXユーティリティ

1週間前、UNIXパイプラインの開発者であり、「コンポーネント指向プログラミング」のコンセプトの作成者であるDouglas McIlroyは、広く使用されていない興味深い珍しいUNIXプログラムについて話しました。この出版物は、ハッカーニュースに関する活発な議論を開始しました。私たちは最も興味深いものを集めており、あなたが議論に参加していただければ幸いです。

テキストを操作する


UNIXのようなオペレーティングシステムには、テキストをフォーマットするための標準的なツールセットがあります。typoユーティリティを使用すると、タイプミスやハパクのドキュメントを表示できます-資料に一度だけ現れる単語。興味深いことに、このプログラムはタイプミスの検索に辞書を使用しません。ファイル内の情報のみに依存し、トライグラム（3文字のシーケンス）の頻度分析を実行します。この場合、必要なすべてのカウンターは 26x26x26配列に格納されます。 Douglas McIlroy氏によれば、このメモリ容量は、いくつかのシングルバイトカウンターにはほとんど不十分です。したがって、それらを保存するために、それらは対数形式で書かれました。

今日、タイプミスはより近代的で正確な辞書ベースのスペルチェッカーに置き換えられています。しかし、ツールがまだ覚えている-数年前、愛好家は、導入に行った事がタイプミスの実装を。リポジトリはまだ更新中です。

80年代のもう1つのドキュメントツールは、Lorinda CherryのWriter's WorkbenchとBell LabsのNina McDonaldです。品詞とドキュメントスタイルを決定し、トートロジーと過度に複雑な文を検索するためのツールが含まれていました。ユーティリティは学生の補助として開発され、かつては使用されていましたアメリカのコロラド州立大学の学生。しかし、90年代の初めには、ライターズワークベンチはバージョン7 Unixに含まれていなかったため、忘れられていました。ただし、このツールは模倣者（たとえばIBM PCのGrammatik）の道を歩み続けました。

UNIXには、数式での作業を簡略化するための標準ツールもあります。数式eqnを処理するための言語プリプロセッサがあります。数式を表示するには、開発者が簡単な単語や記号でそれを記述するだけで十分であることは注目に値します。キーワードを使用すると、数学記号を垂直および水平にシフトしたり、サイズやその他のパラメータを変更したりできます。この行をユーティリティに渡すと、次のようになります。

sum from { k = 1 } to N { k sup 2 }

次の式が出力で生成されます。

∑k=1Nk2


1980年代と1990年代には、eqn は ITプロフェッショナルがソフトウェアマニュアルを作成するのに役立ちました。しかし、その後、Habrでも使用するLaTeXシステムに置き換えられました。しかし、eqnはこのクラスの最初のツールであり、UNIXライクなOSの残りの部分です。

ファイルを操作する


テーマ別スレッドで、ハッカーニュースの居住者は、ファイルの操作にほとんど使用されないユーティリティをいくつか指摘しました。それらの1つはそれらを比較するcommでし た。これは、スクリプトで作業するためにシャープにされたdiffの単純化された類似物です。リチャード・ストールマン自身がデビッド・マッケンジーと一緒に書いたものです。 プログラムの出力は3つの列で構成されます。最初の列には、最初のファイルに固有の値、2番目の列には2番目のファイルに固有の値が含まれています。 3列目には一般的な値が含まれます。 commが正しく機能するためには、比較されるドキュメントが字句的にソートされている必要があります。したがって、サイトの居住者の1人は、次の形式でユーティリティを使用することを提案しました。



comm <(sort fileA.txt) <(sort fileB.txt)

Commは、単語のスペルを確認するのに役立ちます。それらを参照辞書文書と比較するだけで十分です。ファイルの並べ替えの必要性に関連する微妙な点を考えると、ストールマンとマッケンジーは、このユーザーケース専用にユーティリティを作成したと考えられています。

また、HNに関するディスカッション参加者は、ペーストオペレーターの機能を指摘しましたが、これは彼には明らかではありませんでした。出力時にデータストリームを交互にしたり、1つのストリームを2つの列に分割したりできます。

$ paste <( echo -e 'foo\nbar' ) <( echo -e 'baz\nqux' )
foo     baz
bar     qux
$ echo -e 'foo\nbar\nbaz\nqux' | paste - -
foo     bar
baz     qux
