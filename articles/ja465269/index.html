DHCPサーバーに戻って、PC2が配置されている2番目のネットワークのプールを整理しましょう。このネットワークをNET2と呼びます。これを行うには、ip dhcp pool NET2およびnetwork 192.168.3.0 255.255.255.0コマンドを一貫して入力します。次に、両方のネットワークに共通のDNSサーバーのIPアドレス（192.168.1.2）を入力し、デフォルトルーターを割り当てます。ネットワーク3.0に関しては、IPアドレス192.168.3.2の2番目のルーターRouter1をデフォルトルーターとして使用しています。

また、2つのルーター間のネットワークを構成する必要があるため、Router1の設定に進み、対応するIPアドレス192.168.2.2および192.168.3.2をf0 / 0およびf0 / 1インターフェースに割り当てます。
ここで、PC2のIP設定に移動してDHCPモードをオンにすると、このコンピューターはルーター1を要求します。ルーターはDHCPサーバーではないため、システムは要求が失敗したことを示すメッセージを表示し、APIPAサービスが使用されます。これは、ネットワークにDHCPサーバーがない場合にIPアドレスをデバイスに自動的に割り当てます。



この場合、APIPAはランダムなIPアドレス169.254.133.157をコンピューターに割り当てました。これは正常であり、なぜこれが起こったのかはわかっています。ただし、IPアドレスの要求を受け取ったルーター1がネットワーク経由でDHCPサーバーに送信する必要があります。つまり、このルーターがパッシブDHCPリレーサーバーとして機能する必要があります。設定する前に、最初のDHCPサーバールーターに戻り、RIPルーティング機能を有効にします。



次に、2番目のルーターであるルーター1に移動し、同様の設定を行って、これらのデバイスの両方がRIP経由で通信できるようにします。



ここで、外部インターフェイスf0 / 1を構成する必要があります。これには、補助アドレスヘルパーアドレスを指定します。 helper-addressコマンドは、ブロードキャストDHCP要求を指定されたアドレス（この場合は192.168.2.1）に転送するために使用されます。



これで、PC2からのDHCP要求は自動的にDHCPサーバーに転送されます。 DHCPを再度有効にしてみましょう。理由は明確ではありませんが、コンピューターが再度APIPAを使用しています。DHCPサーバーからの要求は応答されませんでした。

問題に対処してみましょう。これを行うには、Router1の設定に移動し、ping 192.168.2.1コマンドを入力します。 pingが通過しないため、show ip int briefコマンドを使用してDHCPサーバーのインターフェイスをすばやく確認しましたが、エラーが見つかりました。はい、そのような知識のある試験に合格しません！したがって、皆さん、そのような間違いをしないように練習する必要があります。今、それを修正します。

DHCPサーバーの設定で重要なコマンドを使用するのを忘れていたので、今度はf0 / 1インターフェースに導入します。ipadd 192.168.2.1 255.255.255.0。これは、私が修正した厄介な間違いです。 PC2設定に戻り、DHCPを有効にします。また何ですか？リクエストは再び失敗します！

Router1の設定をもう一度確認しています。ルーティングテーブルを使用すると、すべてが正常で、必要なエントリがすべて存在しますが、問題は何ですか。セカンダリアドレスも正しく割り当てられます。ええと、何も、結局理由を見つけます。

DHCPサーバーの設定に戻り、show ip routeコマンドを発行します。プロトコルバージョンを指定しなかったため、ルートが存在しないことがわかりました。私はこのエラーを修正しており、これですべてが機能するはずです。

PC2に戻り、DHCPを再びオンにします。これですべて正常になり、コンピュータはDHCPサーバーによって割り当てられたIPアドレス192.168.3.6を受け取ります。不注意な為、間違えて見つけてしまいましたので、ご容赦ください。



ここで、3番目のコンピューターのブラウザーに移動し、www.nwking.orgと入力すると、、リクエストは信頼できるネームサーバーサーバーに送信され、コンピュータがこのリクエストをウェブサーバー192.168.1.3に送信する必要があるというメモとともに返されます。その後、PC2は指定されたアドレスにリクエストを送信し、リクエストされたHTMLファイルを受信します。

これは、ヘルパーアドレスを使用するパッシブDHCPリレーサーバーの動作方法です。ご覧のとおり、すべてが非常に単純であり、トピックを理解することは特別な困難をもたらすものではありません。 DNSとDHCPの基本をマスターすることは、コンピューターをネットワークに接続するために非常に重要です。
