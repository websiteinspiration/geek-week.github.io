主要なディストリビューションにsdl2をインストールする

こんにちは、ハブル。最近、sdl2 に関する新しいシリーズの記事を見つけましたが、インストール手順はありませんでした。複雑なことはなさそうですが、興味があれば、私たちを待っているかもしれない熊手に沿って一緒に歩きましょう。macOS 10.14.5のxcode10、Ubuntu 18.04およびVisual Studio 2019 Windows 10のコード::ブロックをインストールします。もちろん、clionを使用することもできます。すべてのシステムで1つのアイデアですが、何らかの理由で私は彼らの製品が好きではありません。個人的なことではなく、コーディングするだけです。

確認のため、次のコードを使用します。

#include <iostream>

#include <SDL2/SDL.h>

const int WIDTH = 800, HEIGHT = 600;

int main( int argc, char *argv[] )
{
    SDL_Init( SDL_INIT_EVERYTHING );

    SDL_Window *window = SDL_CreateWindow( "Hello SDL World", SDL_WINDOWPOS_UNDEFINED, SDL_WINDOWPOS_UNDEFINED, WIDTH, HEIGHT, SDL_WINDOW_ALLOW_HIGHDPI );
    
    // Check that the window was successfully created
    if ( NULL == window )
    {
        // In the case that the window could not be made...
        std::cout << "Could not create window: " << SDL_GetError( ) << std::endl;
        return 1;
    }
    
    SDL_Event windowEvent;
    
    while ( true )
    {
        if ( SDL_PollEvent( &windowEvent ) )
        {
            if ( SDL_QUIT == windowEvent.type )
            {
                break;
            }
        }
    }
    
    SDL_DestroyWindow( window );
    SDL_Quit( );
    
    return EXIT_SUCCESS;
}

インストールvsスタジオ。ライブラリをダウンロードします。中身を解凍し、好きな場所に配置します。私はめったにウィンドウを使用しないので、他のすべてのライブラリがどこに保存されているのかわかりません。

必要に応じて、空のcppファイルまたはコンソールアプリケーションを使用してプロジェクトを作成します。
「プロジェクト」->「プロパティ」->「C / C ++」->「インクルードファイルの追加ディレクトリ」に移動します。includeを指定して、解凍したディレクトリへのパスを追加します。私の場合、\\ Mac \ Home \ Downloads \ SDL2-2.0.9 \ include、仮想マシンを持っています。



その後、「リンカー」->「一般」->「追加ライブラリディレクトリ」セクションで、SDL2.dllへのパスを指定する必要があります（例：\\ Mac \ Home \ Downloads \ SDL2-2.0.9 \ lib \ x64）。

あまり残っていません。「リンカー」->「入力」->「追加の依存関係」、使用する必要があるもの、SDL2.lib、SDL2main.lib、SDL2test.libを追加します。その後、プロジェクトを組み立てる必要があります。アセンブルしたら、SDL2.dllファイルをlib \ x64またはlib \ x32ディレクトリからコピーし、実行可能ファイルの横に配置します。次に、コードを試してください。別のポイント：なぜなら 「インクルードファイルの追加ディレクトリ」で、接続されたヘッダーへのパスをすぐに示しました。
