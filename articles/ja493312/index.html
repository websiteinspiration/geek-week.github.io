React Nativeでのジオトラッキング

モバイルアプリケーションは従業員の「ワークプレイス」として機能できますが、地理座標やその他のデータの転送が必要になる場合があります。 iOSとAndroidのクロスプラットフォームアプリケーション開発では、このタスクにFlutterやReact Nativeなどのフレームワークがよく使用されます。この記事では、この例を使用して、React Nativeでジオロケーションを操作する機能について説明します。



React Nativeの信頼性、スピード、成熟度は、開発者から批判されることもあり、約3年前には、そのような疑念は正当化されたと言えます。しかし、この間、テクノロジーはより高度になり、React Nativeは幅広いアプリケーションを作成するための実用的なツールとして認識されてきました。特に、React Nativeは、地理座標を使用するアプリケーション（さまざまなトラッカー、宅配便のアプリケーションなど）に適しています。

また、店舗では、オンラインストアのReact Nativeアプリケーション、レストランメニュー、ニュースフィード、配信アプリケーション、ソーシャルネットワーク用の顧客で開発されています。そのため、膨大な数のReact Nativeは、何も考えずに常に使用しています。簡単な例は、Facebook、Instagram、Skype、Uber Eats、Wallmartです。予測によると、特にWindowsプラットフォームのサポートを実装した後、Microsoftはこのリストを補充し、モバイルアプリケーションをReact Nativeに転送できます。Qt、macOS、「組み込み」weechatアプリケーションのReact Nativeサポートがありますが、これらのプラットフォームのサポートとドキュメントはまだ不十分です。

React Nativeは、ほとんどすべてをネイティブアプリケーションと同じように実行できます。たとえば、Java / KotlinまたはObjC / Swiftアプリケーションが行うように、デバイスセンサーから情報を受信するか、フォアグラウンドサービスまたはバックグラウンドフェッチを使用してバックグラウンドでデータを送信します。唯一の違いは、React Nativeでは、ネイティブ部分とJavaScriptの間の相互作用のためのインターフェースを実装する必要があるということです。日常のタスクでは、これらのインターフェースはすでに実装されていることがよくあります。これは、Javascriptの場所と操作をバックグラウンドで追跡する場合にも当てはまります。独立したオープンソースライブラリの形式のソリューションがすでに存在します。コミュニティが独立してJavascriptを前進させ、プログラマーも企業もオープンソースソリューションを積極的に使用するようになりました。同じ数のライブラリ、フレームワークを誇​​る言語コミュニティはほとんどありません。質の高いユーティリティ。それでも、React Nativeでは、ネイティブ開発に存在するいくつかの機能がまだ制限されています。これは覚えておく必要があります。

React Nativeを使用する


React Native上のクロスプラットフォームアプリケーションの開発速度を保証するいくつかの重要な要素を検討してください。

共通コードベース


ロジックを作成すると、さまざまなプラットフォームのレイアウト画面とコンポーネントが機能し、可能な限り同じように見えます（たとえば、Androidでは、通常のシャドウではなく、高揚感の概念が使用されます）。古いバージョンの開発者にとって、実質的に永遠の頭痛はありません。たとえば、Android 4.4でもサポートを追加するのは簡単ですが、実際には、このタスクの実装は追加のネイティブモジュールの使用にも依存します。

最適なワークフロー


多くの場合、Reactは、新機能の開発をネイティブ言語よりも数倍速くする最適なワークフローを提供します。これは、レイアウト、コンポーネントアプローチ、依存関係のインストール、Reactの一般的なアーキテクチャソリューションに適用されます。

ホットリロード


開発者は、変更を確認するために毎回アプリケーションを再構築する必要はありません。Hot Reloadのおかげで、アプリケーションを現在の状態に更新するのに数秒しかかかりません。さらに、バージョン0.61では、Live Reload（現在はFast Refresh）がさらに思い浮かびました。編集時のタイプセットの変更は、コンポーネントの現在の状態をリセットすることなく、「オンザフライ」でプルされます。

アプリケーションを作成する場合、いくつかの重要な領域で特別な注意が必要です

。1）Javascript

React Nativeは、Javascriptを使用することを目的としており、ロジックとインターフェースをすばやく記述できます。通常の方法で使用できるWebおよびNode.jsの世界からの多くの既製のモジュールがあります（当然、それらがDOMまたはサーバー側に接続されていない場合）。 JavaScriptは高速ですが、ネイティブ言語に劣る場合があります。マルチスレッドはありません。 React Nativeでは、ネイティブAPIに直接アクセスすることはできませんが、この制限は間もなく解除される可能性があります。 JavaScriptは主に、ネイティブ部分で実行およびレンダリングする必要があることを記述します。

2）ブリッジ

反応ネイティブライブラリによって提供される特別なインターフェイスを介してJavascript呼び出しを受け入れるバス。呼び出しは通常のシリアル化されたメッセージです。各コンポーネント（単純なビュー、入力フィールド、またはシステムダイアログの呼び出し）ごとに、Javascriptで記述されたインターフェースがあり、ロジックはネイティブ部分に実装されています。バスは最も遅い部分で、非同期で、Javascriptとネイティブの間でデータを転送するのに時間がかかります（おそらくFacebookが完全に取り除くことができます）。多くの場合、React Native上のアプリケーションのパフォーマンスの問題は、ブリッジの操作によって正確に引き起こされますが、それを正しく使用すれば（データの転送を必要最小限に抑えて）、React Nativeアプリケーションのパフォーマンスに大きな違いはありません。

3）ネイティブ部分

React Native上のアプリケーションは、最初は通常のネイティブアプリケーションです。特徴は、React JSですべてのロジックを記述できるように、必要なすべてのライブラリがすでにそのライブラリに接続されていることです。ここで、JSで使用する（または使用しない）ために独自のネイティブロジックを記述し、通常のネイティブライブラリを追加できます（たとえば、KitKat以前でTLS v1.2をサポートするため）。既製のネイティブアプリケーションでは、React Nativeのサポートを追加できます。

React Nativeでの地理追跡の操作


プロジェクトの1つでは、特にクライアントの従業員がバックグラウンドで地理座標を確実に転送できるように、「モバイルワークプレイス」を開発する必要がありました。

React Nativeには、地理位置情報を操作するためのツールがいくつかあります。たとえば、これはReact Native @ react-native-community / geolocation communityのパッケージですが、ほとんどのユースケースに対応していますが、バックグラウンドでは機能しません。

タスクを実装するために、@ mauron85 / react-native-background-geolocationというスピーキング名のライブラリを含む、いくつかのソリューションを使用しました。当初、このライブラリはCordovaのプラグインでしたが、後で作者はベースを分離し、React Native用に個別のラッパーを作成しました。また、transistorsoftからの同名の有料ライブラリもありますが、プロジェクトでの作業の時点では、そのドキュメントでしか獲得できませんでした。

ライブラリにより、位置を追跡するための柔軟な設定を行うことができました。たとえば、システムサービスに問い合わせる頻度、バックグラウンドで正確に動作する方法、位置が静止していると見なされる半径。これらの座標をサーバーに送信するための設定と方法は、「そのまま」使用できますが、要求をスパムしないで、受信した座標をリポジトリに集約し、大きな間隔で送信することにしました。同時に、ライブラリにより、アプリケーションをforegroundActivityとして起動できるようになり、多くのAndroidデバイスでバックグラウンドのアプリケーションは追加の設定でのみ動作するため（Huawei、MIUI、Samsung最近）。

ここでは、通知、アイコン（ネイティブアセットから）の設定を行うこともできます。権限についてはどうですか？Androidの場合、パッケージをインストールした後、何もする必要はありません。ライブラリがシステムの地理位置情報へのアクセス要求を引き継ぎます。ただし、XcodeのiOSで通常の操作を行うには、プロジェクトの[署名と機能]タブでバックグラウンド処理と場所の更新を有効にする必要もあります。



同じメソッドのon（ 'location'）イベントで座標を保存および送信するためのメインメソッドを実装します。アクティブなアプリケーションと最小化された状態の両方で機能します。

BackgroundGeolocation.on('location', location => {
    BackgroundGeolocation.startTask(async taskKey => {
     //       ,
     //     iOS-
      const batteryLevel = this.isSimulator
        ? 100
        : (await DeviceInfo.getBatteryLevel()) * 100;
      const updateDelta = location.time - this.lastUpdateAt;

     //      
      this.props.locationUpdate(location);
      this.props.locationAddTrackerData({
        altitude: location.altitude,
        batteryChargeLevel: batteryLevel,
        latitude: location.latitude,
        longitude: location.longitude,
        course: location.bearing,
        accuracy: location.accuracy,
        speed: location.speed,
        timestamp: Number(String(location.time).substr(0, 10)),
        isAlertOn: false,
      });

     //  ?
      if (updateDelta / 1000 > config.sendInterval) {
        this.syncDataWithInterval();
        this.lastUpdateAt = location.time;
      }

     //    AsyncStorage,    
      Storage.setLastKnownLocation(location);
      BackgroundGeolocation.endTask(taskKey);  //   !
    });
});


ここでは、react-native-device-infoライブラリーを使用して、カスタム形式で座標データを転送し、さらにバッテリー測定値にデータを保存するストアからアクションを呼び出します。これらすべては、アプリケーションがアクティブまたはバックグラウンドで動作している場合、AndroidとiOSで同じように機能します。

さらに、現在の場所を取得する方法があります（ただし、これはサービスの実行中にのみ機能します）。アプリケーションの状態のアクティブからバックグラウンドへの遷移をサブスクライブし、アプリケーションのシステム設定に切り替えます。ただし、概して、最も必要なものはこれらの例に示されています。
