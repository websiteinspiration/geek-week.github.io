ADAM-6200 I / Oモジュール


リモートI / Oモジュールは、さまざまなタイプの周辺機器と通信するように設計されています。これは、産業システムの構築に不可欠な要素です。それらは、他のデバイスから信号を受信し、デバイスに制御信号を送信して、MODBUS TCP、MQTT、HTTPなどを介してSCADA中央制御システムと統合することができます。

業界で使用されていた前世代のI / Oデバイスは、通常RS-232 / 485シリアルインターフェイスを使用して接続されていたため、このようなシステムのスケーラビリティは困難でした。ADAM-6200シリーズのモジュールは、イーサネット経由で接続する機能と、2つのイーサネットポート用の組み込みスイッチを備えているため、デバイスをチェーンで直列に接続できます。

ADAM-6200 モジュールは、信号伝送に加えて、プログラマブルロジックコントローラーの役割を果たし、簡単な自動化タスクを解決できます。条件付きグラフィックロジックの言語のサポートにより、プログラミング言語の知識がなくてもプログラミングできます。

この記事では、デバイスの特性、障害に対する保護の範囲と追加機能について説明します。

条件付きロジックを使用したプログラミング


多くの計算能力を必要としない単純な自動化タスクは、グラフィック条件ロジックを使用してADAMコントローラーで直接プログラムできます。たとえば、事故が発生した場合はサイレンを作動させ、センサーが発生した場合はバルブを閉じます。 GCL環境ではプログラミングのスキルはほとんど必要なく、マウスカーソルを使用して本格的な自動化スクリプトを作成できます。Adam / Apax .NETユーティリティ



プログラムでのGCLスクリプトの開発。条件としては、デジタルまたはアナログ入力の状態、ディスクリートシグナルカウンター、内部タイマーなどが使用できます。ルールは相互に参照し、より複雑な条件を形成できます。ドキュメントでGCL開発環境の詳細をご覧ください。


条件付きロジック開発環境

シリアル接続（デイジーチェーン）


ADAM-6200 シリーズのデバイスには、ボード上に2つのイーサネットポートがあり、デイジーチェーンで直列に接続できます。これにより、ネットワークトポロジが大幅に簡素化され、ネットワークのリモートセクションを構築するときに追加のスイッチが不要になります。



RS-232 / 485インターフェースを使用した接続とは対照的に、イーサネット接続を使用すると、コントローラーを任意のIPネットワークに簡単に統合できるだけでなく、VPNトンネルを使用してインターネット経由でリモートオブジェクトを接続できます。追加のインターフェースコンバーターを使用する必要はありません。

パワーブレーク保護（自動バイパス）


自動バイパス機能は、チェーンの中間デバイスで電源が失われると自動的にアクティブになります。このモードでは、ケーブルが直接接続されているかのように、デバイスはパッシブ接続として機能します。この場合、制限はUTP接続に適用されます。したがって、パッシブ接続の最大長は100メートルであるため、2つの隣接ポイント間のケーブル長の合計を考慮し、両側で50メートルを超えないようにすることが重要です。この機能のバッテリー寿命は最大4日間です。


自動バイパスのおかげで、中間デバイスの電源が切られていても通信は切断されません

結論


ユニバーサルADAM-6200 I / Oモジュールは幅広いプロトコルサポートを備えており、Modbus TCP、MQTT、HTTP RESTを備えており、従来のSCADAシステムや最新のソフトウェア製品に簡単に統合できます。

GCL言語でのプログラムのサポートにより、追加のデバイスを使用せずに簡単な自動化タスクを解決できます。ピアツーピアの相互運用性により、中間データ処理サーバーを使用せずに、デバイス間で直接データを交換できます。

チェーン内のデバイスを順番に接続する機能により、追加の機器なしでネットワークの大きなセグメントを簡単に構築できます。また、自動バイパス機能は、チェーン内のデバイスの障害によるラインブレークから保護します。
