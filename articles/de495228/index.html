WebStorm 2020.1: Verbesserungen der Benutzeroberfläche, Vuex-Unterstützung und Starten von Prettier beim Speichern von Dateien

Hallo alle zusammen! Wir freuen uns, Ihnen dieses Jahr das erste große WebStorm-Update vorstellen zu können. In der neuen Version finden Sie viele neue Funktionen und lang erwartete Verbesserungen, einschließlich der Vuex-Unterstützung und einer neuen Option zum Starten von Prettier beim Speichern von Dateien.



Laden Sie die 30-Tage-Testversion von WebStorm 2020.1 online oder über die Toolbox-App herunter . Die Vollversion kann von Inhabern eines aktiven Abonnements für WebStorm oder All Products Pack sowie von kostenlosen Studenten und Entwicklern von Open Source-Projekten verwendet werden.

Schauen wir uns nun die wichtigsten Verbesserungen genauer an.

Neue Schriftart für die Arbeit mit Code


Im Laufe des vergangenen Jahres haben wir eine spezielle Schriftart für die Arbeit mit Code entwickelt, mit der unsere Benutzer bequem programmieren und die Belastung der Augen minimieren können. Das Ergebnis unserer Bemühungen ist JetBrains Mono , eine neue Open Source-Schriftart. Ab Version 2020.1 ist JetBrains Mono in WebStorm und unseren anderen IDEs standardmäßig ausgewählt. Die Möglichkeit, eine andere Schriftart einzuschließen, blieb natürlich ebenfalls erhalten.



Nützlichere schnelle Dokumentation.


Mit WebStorm 2020.1 können Sie schnell die relevanten Informationen zu einem Symbol finden, da jetzt eine schnelle Dokumentation angezeigt wird, wenn Sie mit der Maus über dieses Symbol fahren, und nicht nur, wenn Sie F1 drücken . Wenn in Ihrem Code ein Problem vorliegt und WebStorm bei der Lösung helfen kann, werden Sie darüber informiert, indem eine verfügbare Schnellkorrektur direkt im Popup-Fenster der Dokumentation angezeigt wird.



Wenn Sie mit JavaScript- und TypeScript-Code arbeiten, werden in der Schnelldokumentation detaillierte Informationen zum Typ und zur Sichtbarkeit eines Zeichens sowie zur Definition dieses Zeichens angezeigt.



Zen-Modus für fokussierte Arbeit


Mit dem neuen Zen- Modus können Sie sich ganz auf Ihren Code konzentrieren. Durch die Kombination der beiden Modi Ablenkungsfrei und Vollbild können Sie beide Modi schnell einschalten und vorübergehend etwas isolieren, das Ihre Aufmerksamkeit ablenken kann.



Um den neuen Modus auszuprobieren, gehen Sie zu Ansicht | Aussehen | Rufen Sie den Zen-Modus über das WebStorm-Hauptmenü auf oder verwenden Sie das Popup-Fenster Switch ( Strg + `| Ansichtsmodus | Zen-Modus aufrufen ).

Unterstützung für Vuex und Vue Composition API


Vue.js wird immer beliebter, daher arbeiten wir aktiv daran, WebStorm zur nützlichsten IDE für die Arbeit mit diesem Framework zu machen. Dieses Mal haben wir zwei wesentliche Verbesserungen hinzugefügt.

Wenn Sie mit der Vuex-Bibliothek arbeiten, werden beim Bearbeiten von Vue-Komponenten zunächst Optionen für die Code-Vervollständigung für Vue-Speichersymbole und -module angezeigt. Mit WebStorm können Sie auch schnell Getter, Mutationen und Aktionen definieren.



Zweitens haben wir beschlossen, Unterstützung für WebStorm 2020.1 hinzuzufügen, da Sie die in der kommenden Version Vue 3 verfügbare Composition-API bereits verwenden können.



Ausführen von Prettier beim Speichern von Dateien


Dank der neuen Option Beim Speichern für Dateien ausführen können Sie eine schönere Formatierung auf alle Dateien anwenden, die in den WebStorm-Einstellungen angegeben und im aktuellen Projekt bearbeitet wurden, während Sie diese Dateien speichern. Sie müssen den Datei-Watcher nicht mehr konfigurieren oder ein Plugin eines Drittanbieters verwenden.



Hilfe bei Grammatik- und Stilfehlern


Ab Version 2020.1 wird WebStorm mit Grazie geliefert, unserem Tool zur Überprüfung von Rechtschreibung, Grammatik und Textstil. Auf diese Weise können Sie Grammatikfehler vermeiden, wenn Sie Kommentare, Commits und verschiedene Sprachkonstrukte hinzufügen.



Standardmäßig ist Grazie nur für die englische Sprache aktiviert und überprüft nicht alle Dateitypen auf mögliche Grammatikfehler. Sie können weitere Sprachen hinzufügen und vordefinierte Einstellungen unter Einstellungen / Einstellungen | ändern Editor | Korrekturlesen (gehen Sie zum Abschnitt Grammatik, wenn Sie den Scanbereich konfigurieren möchten, lesen Sie die vorhandenen Regeln und fügen Sie Ausnahmen hinzu).

Neue Absichtsaktionen und Inspektionen


Wie immer haben wir einige neue Absichtsaktionen und Schnellkorrekturen hinzugefügt, um Ihnen Zeit beim Arbeiten mit JavaScript- und TypeScript-Code zu sparen. Beispielsweise konvertiert eine der neuen Absichtsaktionen, die durch Drücken von Alt + Eingabetaste verfügbar sind , vorhandenen Code schnell in optionales Verketten und / oder Null-Zusammenführen.



Weitere Informationen zu den neuen Absichtsaktionen und Schnellkorrekturen finden Sie im Release- Blog-Beitrag (in englischer Sprache).

Unterstützung für die neuesten Versionen gängiger Technologien


In WebStorm 2020.1 finden Sie integrierte Unterstützung für TypeScript 3.8-Funktionen, einschließlich Nur-Typ-Importe / -Exporte, private Felder und Top-Level-Funktionen await, mit denen Sie effizienter arbeiten können. Darüber hinaus haben wir die Arbeit mit Angular 9-Projekten erleichtert, indem wir das neue Metadatenformat vollständig unterstützt haben.

Bequemere Neuausrichtung von Commits


Mit der Aktion Interaktives Wiederherstellen von hier , die auf der Registerkarte " Protokolle" des Git- Fensters verfügbar ist , können Sie frühere Commits bearbeiten, zusammenführen und löschen. Dies hilft dabei, den Commit-Verlauf linearer und verständlicher zu gestalten.



In WebStorm 2020.1 finden Sie eine verbesserte, interaktivere Version des Dialogfelds " Wiederherstellen von Commits ", das geöffnet wird, wenn die Aktion " Interaktives Wiederherstellen von hier" aufgerufen wird . Das aktualisierte Dialogfeld zeigt Aktionen an, die auf jedes Commit angewendet werden können. Außerdem werden detaillierte Informationen zu jedem Commit angezeigt, und Sie können den Unterschied erkennen und angewendete Änderungen schnell verwerfen.

Verwenden von WebStorm zum schnellen Bearbeiten von Dateien


Dank des neuen LightEdit- Modus können Sie Dateien in einem separaten Texteditorfenster öffnen, ohne ein gesamtes Projekt erstellen oder laden zu müssen. Der Modus funktioniert, wenn WebStorm noch nicht ausgeführt wird. Wenn WebStorm ausgeführt wird, wird die Datei wie gewohnt darin geöffnet.

Um den neuen Modus in Aktion zu testen, klicken Sie mit der rechten Maustaste auf die Datei, die Sie bearbeiten möchten, und wählen Sie WebStorm aus der Liste der vorgeschlagenen Programme aus. Alternativ können Sie den Befehlszeilen-Launcher auch wie hier beschrieben konfigurieren und die Datei über die Befehlszeile öffnen.
