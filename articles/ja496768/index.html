RGBとHEXを忘れる

CSSで色を表す方法はいくつかあります。その1つがHSLシステムです。この記事では、レイアウトデザイナーにどのような可能性が開かれるかを紹介します。

HSLとは？


HSL形式の名前は、最初の文字の色相（色相）、彩度（彩度）、および明度（明度または明度）の組み合わせから派生しています。

色相はカラーホイールのカラー値で、度単位で設定されます。0°は赤、120°は緑、240°は青に対応します。色相の値は、0〜359の範囲で変化します。



彩度は、0％〜100％のパーセントで測定される色の強度です。同じ明るさのグレーから色までの距離を決定します。

明るさは色の明るさを表し、0％から100％の割合で示されます。小さな値は色を暗くし、高い値は明るくします。極端な値0％と100％は白黒に対応します。



HSLの利点（）


HSLの主な利点は、互いに独立して色特性を指定できることです。これはあなたにとって素晴らしい機会を開きます。色を簡単に明るくしたり、暗くしたり、彩度を上げたり、変色させたりすることができます。そしてその陰を維持しながら。またはその逆-彩度や明るさを変更せずに色合いを変更します。多くの場合、これらの機能を使用するためにプリプロセッサが使用されますが、プリプロセッサなしで実行することもできます。

いくつかの例


同じ色相の派生色


最も一般的なタスクの1つ：1つの基本的なシェードを使用するが、そのいくつかのバリエーションで構成されるコンポーネントを作成する。私の例では、これは典型的なアラートで、テキスト、背景、ストロークの色は同じ色相と彩度ですが、明るさが異なります。

.success {  
  border-color:     hsl(120, 50%, 40%);
  color:            hsl(120, 50%, 20%);
  background-color: hsl(120, 50%, 90%);
}


どれほど美しく澄んでいますか？これが花の「家族」の1つであることはすぐにわかります。他の形式では、次のようになります。

/* rgb */
.alert {
  border-color:     rgb(51, 153, 51);
  color:            rgb(25, 77, 25);
  background-color: rgb(217, 242, 217);
}

/* hex */
.alert {
  border-color:     #339933;
  color:            #194d19;
  background-color: #d9f2d9;
}

あなたのことはわかりませんが、このようなコードを見て、これらの色が何らかの形で接続されているかどうかを判断するのは難しいです。

しかし、これらはささいなことです。CSSカスタムプロパティを使用すると、真のパワーが明らかになります。

彼らが言うように、あなたの手を見てください。色相や変数など、色のコンポーネントを作成する場合、基本的な色相のみを変更することで、コンポーネントのバリエーションを無限に簡単に作成できます。そして、すべての派生物はそれから反発されます。

.alert {
  border-color:     hsl(var(--hue), 50%, 40%);
  color:            hsl(var(--hue), 50%, 20%);
  background-color: hsl(var(--hue), 50%, 90%);
}



漂白


この例では、オフボタンの彩度が低くなっています。それはまだ微妙な青みがかっていますが、ほとんど灰色です。また、ホバーすると、ボタンは色相と彩度を保持しますが、暗くなります。



この問題を解決するには、3つのパラメーターの1つだけを変更して、導関数を取得しますが、色は似ています。rgbまたはhexを使用する場合、それほど簡単ではありません。

最初の例と同様に、コンポーネントの他のすべての色特性に影響を与えることなく、色相を簡単に変更できます。


色の継承


最もクールなテクニックの1つ：1つのシェードを別のシェードから作成します。これは、calc（）で単純に加算することで実現されます。

:root{
  --hue: 120;
}

header {
  background-color: hsl(var(--hue), 30%, 20%)
}

header button {
  background-color: hsl(calc(var(--hue) + 130), 80%, 50%)
}

この例では、ボタンの色相はコンテナの色相に依存します（カラーホイールでは+ 130°）。


したがって、簡単な方法で、サイトに本格的な柔軟なカラーパレットを作成し、基本的なシェードを1つだけ設定してそこから開始できます。

また、前の例で示したように、すべてのコンポーネントは彩度と明るさを保持します。



未来


CSSカラーモジュールレベル4 仕様では、同じ名前の形式で色を指定するlab（）およびlch（）という新しい関数が説明されています。LCHはhslと同じ柔軟性を持っていますが、最新のデバイスを視野に入れて、いくつかの改善をもたらします。詳しくは、「CSSのLCHカラー：何を、なぜ、そしてどのように？」

合計


私の意見では、hsl形式は他の形式と比較して素晴らしい柔軟性を持っています。変数と組み合わせると、プリプロセッサからネイティブCSSまで、色を操作するためのすべての可能性を実際にもたらします。
