Konferensi FOSDEM 2020

Kode WireGuard Akan Masuk ke Linux




Setelah beberapa tahun pengembangan, WireGuard, menyebut ZDNet sebagai "pendekatan revolusioner" dalam membangun VPN, akhirnya dijadwalkan untuk dimasukkan dalam kernel Linux dan diharapkan akan dirilis pada April 2020.

Linus Torvalds sendiri dianggap sebagai salah satu penggemar terbesar WireGuard, katanya: â€œ Bisakah aku sekali lagi mengakui cintaku untuk proyek ini dan berharap itu akan segera beku? Mungkin kodenya tidak sempurna, tapi saya cepat membacanya dan, dibandingkan dengan OpenVPN dan IPSec, ini adalah karya seni â€(sebagai perbandingan, basis kode WireGuard memiliki 4.000 baris kode, dan OpenVPN memiliki 100.000 baris).

Terlepas dari kesederhanaannya, WireGuard menyertakan teknologi kriptografi canggih seperti kerangka kerja Noise protocol, Curve25519, ChaCha20, Poly1305, BLAKE2, SipHash24, dan HKD. Juga, keamanan proyek telah terbukti secara akademis.

Canonical Memberikan Opsi Tingkat Lanjut untuk Penyedia Peralatan Bersertifikat




Dimulai dengan versi LTS dari Ubuntu 20.04, pemasangan dan pengoperasian sistem akan berbeda pada perangkat bersertifikat Canonical. Pengembang Ubuntu sedang bekerja untuk memeriksa perangkat yang disertifikasi dalam sistem selama boot GRUB menggunakan modul SMBIOS menggunakan string pengenal perangkat. Menginstal Ubuntu pada perangkat keras bersertifikat akan memungkinkan, misalnya, dukungan di luar kotak untuk versi kernel yang lebih baru. Jadi, khususnya, Linux 5.5 akan tersedia (sebelumnya diumumkan untuk 04/20, tetapi kemudian ditinggalkan) dan mungkin 5.6. Selain itu, perilaku ini berlaku tidak hanya untuk instalasi awal, tetapi juga untuk operasi selanjutnya, pemeriksaan serupa akan dilakukan ketika menggunakan APT. Misalnya, pendekatan ini akan bermanfaat bagi pemilik komputer Dell.

Proyek TFC Menawarkan Perpesanan yang Dilindungi Paranoid




Proyek TFC (Obrolan Tinfoil) mengusulkan prototipe perangkat lunak dan perangkat keras pengiriman pesan â€œyang dilindungi paranoidâ€ yang memungkinkan Anda untuk menjaga kerahasiaan korespondensi bahkan ketika perangkat akhir dikompromikan. Kode proyek dapat diaudit, ditulis dalam Python di bawah lisensi GPLv3, dan skema akses besi untuk FDL.

Utusan umum yang menggunakan enkripsi ujung ke ujung melindungi terhadap intersepsi lalu lintas perantara, tetapi tidak melindungi terhadap masalah di sisi klien, misalnya, dari kompromi sistem jika ada kerentanan di dalamnya.

Skema sisi klien yang diusulkan menggunakan tiga komputer - gateway untuk terhubung ke jaringan melalui Tor, komputer untuk enkripsi, dan komputer untuk dekripsi. Ini, bersama dengan teknologi enkripsi yang diterapkan, secara teoritis harus secara signifikan meningkatkan keamanan sistem.

Pengadilan mendukung pengembang, yang berbicara membela GPL




Pengadilan Banding California memutuskan dalam perkara hukum antara Open Source Security Inc., proyek Grsecurity yang sedang berkembang, dan Bruce Perens, salah satu penulis definisi Open Source, salah satu pendiri OSI, pencipta BusyBox, dan salah satu pemimpin pertama dari proyek Debian.

Inti dari proses tersebut adalah bahwa Bruce di blognya mengkritik pembatasan akses ke karya Grsecurity dan memperingatkan agar tidak membeli versi berbayar karena kemungkinan pelanggaran lisensi GPLv2, dan perusahaan menuduhnya menerbitkan pernyataan palsu dan menggunakan posisinya di masyarakat untuk membahayakan bisnis perusahaan. .

Pengadilan menolak banding tersebut, dan menemukan bahwa posting blog Perence adalah pendapat pribadi berdasarkan fakta yang diketahui. Dengan demikian, putusan pengadilan yang lebih rendah dikonfirmasi, di mana semua klaim terhadap Bruce ditolak, dan perusahaan diperintahkan untuk memulihkan biaya hukum $ 259 ribu.

Namun, persidangan tidak secara langsung membahas masalah kemungkinan pelanggaran GPL, dan ini, mungkin, akan menjadi yang paling menarik.

Startup Menarik Investasi $ 40 Juta untuk Menyederhanakan Akses ke Proyek Open Source Cloud




Semakin populernya perangkat lunak Open Source sangat penting dalam evolusi sektor TI perusahaan. Tetapi ada sisi lain - kompleksitas dan biaya untuk mempelajari dan mengadaptasi perangkat lunak semacam itu dengan kebutuhan perusahaan.

Aiven, startup dari Finlandia, baru saja membangun platform untuk memfasilitasi tugas-tugas seperti itu dan baru-baru ini mengumumkan kenaikan $ 40 juta.

Perusahaan ini memberikan solusi berdasarkan 8 proyek Open Source yang berbeda - Apache Kafka, PostgreSQL, MySQL, Elasticsearch, Cassandra, Redis, InfluxDB dan Grafana - yang mencakup berbagai fungsi dari pekerjaan dasar dengan data untuk mencari dan memproses informasi dalam jumlah besar.

"Meningkatnya penggunaan infrastruktur Open Source dan penggunaan layanan cloud publik adalah salah satu tren paling menakjubkan dan kuat dalam teknologi perusahaan dan Aiven memungkinkan untuk mengambil keuntungan dari infrastruktur Open Source untuk pelanggan dari semua ukuran, â€kata Eric Liu, mitra Aiven di IVP, pemain terkemuka di pasar korporat perangkat lunak yang mendukung proyek-proyek terkenal seperti Slack, Dropbox, dan GitHub.

Open Source Industrial Platform Kontrol IOT Open Source




Alliander, operator sistem terdistribusi Belanda, telah meluncurkan Open Smart Grid Platform (OSGP), platform IIoT yang dapat diskalakan. Ini memungkinkan Anda untuk mengumpulkan data dengan aman dan mengelola perangkat pintar di jaringan. Secara khusus, ini dapat digunakan dengan cara-cara berikut:

  1. Pengguna atau operator terhubung ke aplikasi web untuk memantau atau mengontrol perangkat.
  2. Aplikasi terhubung ke OSGP melalui layanan web, dibagi dengan fungsionalitas, seperti "penerangan jalan", "sensor pintar", "kualitas catu daya". Pengembang pihak ketiga dapat menggunakan layanan web untuk mengembangkan atau mengintegrasikan aplikasi mereka.
  3. Platform bekerja dengan permintaan aplikasi melalui protokol yang terbuka dan aman.

Platform ditulis dalam Java, kode tersedia di GitHub di bawah lisensi Apache-2.0.

Kernel Linux menyelesaikan masalah 2038




Pada 19 Januari 2038, Selasa pukul 03:14:07 UTC, masalah serius diperkirakan terjadi karena penggunaan nilai 32-bit untuk menyimpan waktu UNIX. Dan ini bukan masalah Y2K yang membengkak. Tanggal akan diatur ulang, semua sistem UNIX 32-bit akan kembali ke masa lalu, pada awal 1970.

Tapi sekarang kamu bisa tidur dengan tenang. Pengembang Linux di kernel versi 5.6 baru memperbaiki masalah ini delapan belas tahun sebelum kemungkinan kiamat sementara. Pengembang Linux telah mengatasi masalah ini selama beberapa tahun. Selain itu, tambalan untuk solusinya akan porting ke beberapa versi sebelumnya dari kernel Linux - 5.4 dan 5.5.

Benar, ada peringatan - aplikasi pengguna harus dimodifikasi seperlunya untuk menggunakan libc versi baru. Dan inti baru juga harus didukung oleh mereka. Dan ini dapat menimbulkan rasa sakit bagi pengguna perangkat 32-bit yang tidak didukung, dan bahkan lebih lagi bagi pengguna program sumber tertutup.

Kernel Linux akan dapat memecahkan masalah kunci bersama




Kunci terpisah terjadi ketika instruksi atom beroperasi pada data dari beberapa lokasi cache. Karena sifat atomnya, dalam hal ini, kunci bus global diperlukan, yang menyebabkan masalah dengan kinerja seluruh sistem dan sulitnya menggunakan Linux dalam sistem "waktu nyata yang sulit".

Secara default, pada prosesor yang didukung, Linux akan menampilkan pesan dalam dmesg ketika kunci split terjadi. Dan ketika Anda menentukan opsi kernel split_lock_detect = fatal, sinyal SIGBUS juga akan dikirim ke aplikasi masalah, yang akan memungkinkannya untuk mengakhiri atau memprosesnya.

Fitur ini diharapkan akan dimasukkan dalam versi 5.7.
