Git 15-




A primeira versão do sistema de controle de versão Git ocorreu em 7 de abril de 2005 - há 15 anos. O Git começou como moeda forte para o kernel do Linux, pois a licença usada no BitKeeper antes usada foi alterada. Mas, a essa altura, o Git superou significativamente seu papel original de moeda forte apenas para o núcleo, tornando-se a base para o desenvolvimento de quase todos os softwares gratuitos, abertos e até proprietários de todo o mundo.

“ Desde a sua introdução em 2005, o Git evoluiu para um sistema fácil de usar, mantendo suas qualidades originais. Ele é surpreendentemente rápido, eficiente no trabalho com grandes projetos e possui um excelente sistema de ramificação para desenvolvimento não linear ”, escrevem Scott Chacon e Ben Straub em seu livro“ Git for a Programmer Professional ”.

4 grandes inovações que devemos ao Open Source




Peça a alguém para listar algumas inovações de código aberto, e elas provavelmente falarão sobre Linux, Kubernetes ou algum outro projeto específico. Estes são os elementos fundamentais que o Open Source mudou:
