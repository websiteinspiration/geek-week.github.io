Open Data Hubプロジェクト-Red Hat OpenShiftに基づくオープン機械学習プラットフォーム

未来が到来し、人工知能と機械学習テクノロジーは、お気に入りのショップ、運送会社、さらにはトルコで育つ農場でも既に成功裏に使用されています。



そして何かが存在する場合、それについてインターネット上ですでに存在しています...オープンプロジェクト！Open Data Hubが新しいテクノロジーのスケーリングにどのように役立ち、それらの実装の困難を回避するかをご覧ください。

人工知能（AI）と機械学習（ML）のすべての利点について、組織はこれらのテクノロジーのスケーリングに困難を伴うことがよくあります。これの主な問題は、原則として次のとおりです。

  • 情報交換と協力 -不必要な努力なしに情報を交換したり、高速反復のモードで協力することは実際には不可能です。

これらの問題は未解決のままであり、貴重なデータ処理および分析の専門家の速度、効率、生産性に悪影響を及ぼします。これは彼らの欲求不満、仕事への失望につながり、結果としてAI / MLに関するビジネスの期待が失われます。

これらの問題を解決する責任は、クラウドのようなデータアナリストを提供する必要があるITプロフェッショナルにあります。さらに開発された場合、選択の自由を与え、便利で簡単にアクセスできるプラットフォームが必要です。同時に、それは高速で、簡単に再構成可能で、オンデマンドでスケーラブルで、障害に耐性があります。オープンソーステクノロジーに基づいてこのようなプラットフォームを構築すると、ベンダーに依存しなくなり、コスト管理の面で長期的な戦略的優位性を維持できます。

数年前、アプリケーション開発でも同様のことが起こり、マイクロサービス、ハイブリッドクラウド環境、IT自動化、アジャイルプロセスが登場しました。これらすべてに対処するために、ITプロフェッショナルはコンテナー、Kubernetes、オープンハイブリッドクラウドの使用を開始しました。

現在、この経験はアルの挑戦に答えるために適用されています。したがって、ITプロフェッショナルは、コンテナーに基づくプラットフォームを作成し、アジャイルプロセスの一部としてAI / MLサービスを作成し、イノベーションを加速し、ハイブリッドクラウドに注目して構築することができます。



ソフトウェアとハ​​ードウェアのMLソリューション（NVIDIA、H2O.ai、Starburst、PerceptiLabsなど）の急成長するエコシステムを持つハイブリッドクラウド用のコンテナーKubernetesプラットフォームであるRed Hat OpenShiftを使用して、このようなプラットフォームの構築を開始します。BMW Group、ExxonMobilなどのRed Hatの顧客の一部は、このプラットフォームとそのエコシステムに基づいて、コンテナ化されたMLツールチェーンとDevOpsプロセスをすでに導入しており、MLアーキテクチャを商用運用に持ち込み、データアナリストの作業をスピードアップしています。

Open Data Hubプロジェクトを立ち上げたもう1つの理由は、いくつかのオープンソースプロジェクトに基づくアーキテクチャの例を示し、OpenShiftプラットフォームに基づくMLソリューションのライフサイクル全体を実装する方法を示すことです。

データハブプロジェクトを開く


これは、対応する開発コミュニティのフレームワーク内で開発され、OpenShiftプラットフォームでコンテナーとKubernetesを使用してAI / MLタスクを解決するときに、初期データの読み込みと変換からモデルの形成、トレーニング、メンテナンスまでの操作の完全なサイクルを実装するオープンソースプロジェクトです。このプロジェクトは、OpenShiftおよびTensorflow、JupyterHub、Sparkなどの関連するオープンソースツールに基づくサービスソリューションとしてオープンAI / MLを構築する方法の例であるリファレンス実装と見なすことができます。 Red Hat自体がこのプロジェクトを使用してAI / MLサービスを提供していることに注意することが重要です。さらに、OpenShiftは、NVIDIA、Seldon、Starbust、その他のベンダーの主要なソフトウェアおよびハードウェアMLソリューションと統合されます。独自の機械学習システムの構築と立ち上げを容易にします。



Open Data Hubプロジェクトは、次のカテゴリのユーザーとユースケースに焦点を当てています。

  • セルフサービス機能を備えたクラウドのタイプ別に整理された、MLプロジェクトを実装するためのソリューションを必要とするデータアナリスト。
  • 幅広い最新のオープンソースAI / MLツールおよびプラットフォームから最大限の選択を必要とするデータアナリスト。
  • モデルのトレーニング時にデータソースにアクセスする必要があるデータアナリスト。
  • コンピューティングリソース（CPU、GPU、メモリ）へのアクセスを必要とするデータアナリスト。
  • 日付は、同僚と協力して作業結果を共有し、フィードバックを受け取り、高速反復法を使用して改善を導入する機会を必要とするアナリストです。
  • MLモデルと作業結果を本番環境に導入するために、開発者（および開発チーム）と対話したいデータアナリスト。

Open Data Hubプロジェクトは、多数のオープンソースツールを組み合わせて、完全なAI / ML操作を実装します。 Jupyter Notebookは、ここではデータ分析の主要な作業ツールとして使用されています。このツールキットは現在、データ処理および分析の専門家に人気があり、Open Data Hubでは、組み込みのJupyterHubを使用してJupyter Notebookワークスペースを簡単に作成および管理できます。ノートブックJupyterの作成とインポートに加えて、Open Data Hubプロジェクトには、AIライブラリの形式で既製のノートブックが多数含まれています。

このライブラリは、ラピッドプロトタイピングを簡素化するオープンソースの機械学習コンポーネントとサンプルスクリプトソリューションのコレクションです。 JupyterHubはOpenShift RBACアクセスモデルと統合されているため、既存のOpenShiftアカウントを使用してシングルサインオンを実装できます。さらに、JupyterHubはspawnerと呼ばれる便利なユーザーインターフェイスを提供します。これにより、ユーザーは選択したJupyter Notebookのコンピューティングリソース（プロセッサコア、メモリ、GPU）の量を簡単に構成できます。

データアナリストがラップトップを作成してセットアップした後、OpenShiftの一部であるKubernetesスケジューラーが残りを処理します。ユーザーは実験を実行し、作業の結果を保存して共有することしかできません。さらに、上級ユーザーは、JupyterノートブックからOpenShift CLIシェルに直接アクセスして、JobなどのKubernetesプリミティブや、TektonやKnativeなどのOpenShift機能を有効にすることができます。または、「OpenShift Webコンソール」と呼ばれる便利なOpenShift GUIを使用できます。





次のステップに進むと、Open Data Hubはデータパイプラインを管理する機能を提供します。これには、S3互換オブジェクトデータウェアハウスとして提供されるCephオブジェクトが使用されます。 Apache Sparkは、外部ソースまたはCeph S3組み込みストレージからデータをストリーミングし、予備的なデータ変換を実行することもできます。 Apache Kafkaは、データパイプラインの高度な管理を提供します（データの変換、分析、および格納の操作だけでなく、複数のダウンロードを実行できます）。

したがって、データアナリストはデータにアクセスしてモデルを構築しました。現在、彼は同僚やアプリケーション開発者と結果を共有し、サービスの原則に関する彼のモデルを彼らに提供したいと思っています。これを行うには、出力サーバーが必要です。OpenData Hubにはそのようなサーバーがあり、Seldonと呼ばれ、モデルをRESTfulサービスとして公開できます。

ある時点で、Seldonサーバーにはそのようなモデルがいくつかあり、それらがどのように使用されるかを監視する必要があります。これを行うために、オープンデータハブは、広く使用されているオープンソース監視ツールであるプロメテウスとグラファナに基づいて、関連するメトリックのコレクションとレポートエンジンを提供しています。その結果、特に本番環境でのAIモデルの使用を監視するためのフィードバックが得られます。



このように、Open Data Hubは、アクセスとデータ準備からモデルのトレーニングと産業運用まで、AI / ML運用サイクル全体を通じてクラウドのようなアプローチを提供します。

すべてを一緒に入れて


今問題は、OpenShift管理者のためにこれをどのように整理するかです。そして、Open Data Hubプロジェクト用の特別なKubernetesオペレーターが登場します。



このオペレーターは、JupyterHub、Ceph、Spark、Kafka、Seldon、Prometheus、Grafanaなどのツールのデプロイメントを含む、Open Data Hubプロジェクトのインストール、構成、およびライフサイクルを管理します。 Open Data Hubプロジェクトは、OpenShift Webコンソールのコミュニティオペレーターセクションにあります。したがって、OpenShift管理者は、対応するOpenShiftプロジェクトが「Open Data Hubプロジェクト」として分類されるように指定できます。これは一度だけ行われます。その後、OpenShift Webコンソールを介してデータアナリストがプロジェクトスペースに入ると、対応するKubernetesオペレーターがインストールされており、自分のプロジェクトで使用できることがわかります。次に、ワンクリックでOpen Data Hubプロジェクトのインスタンスを作成し、すぐに上記のツールにアクセスします。そして、これらすべてを高可用性とフォールトトレランスモードで構成できます。



自分の手でOpen Data Hubプロジェクトを試す場合は、インストール手順と入門チュートリアルから始めます。 Open Data Hubアーキテクチャの技術的な詳細はここにあり、プロジェクト開発計画はここにあります。将来的には、Kubeflowとの追加の統合を実装し、データ規制とセキュリティに関する多くの問題を解決し、DroolsとOptaplannerのルールに基づいてシステムとの統合を組織する予定です。コミュニティページで、意見を表明してOpen Data Hubプロジェクトのメンバーになることができます。

要約すると、スケーリングに関する深刻な問題により、組織は人工知能と機械学習の可能性を完全には実現できません。Red Hat OpenShiftは、ソフトウェア業界で同様の問題を解決するために長い間首尾よく使用されてきました。オープンソース開発コミュニティの一部として実装されたOpen Data Hubプロジェクトは、OpenShiftハイブリッドクラウドに基づく完全なAI / ML操作サイクルを整理するためのリファレンスアーキテクチャを提供します。このプロジェクトには明確で思慮深い開発計画があり、OpenShiftプラットフォーム上でオープンAIソリューションを開発するためのアクティブで実りあるコミュニティを作成することに真剣に取り組んでいます。
