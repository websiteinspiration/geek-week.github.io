自分でやるSkype

良い一日。

Linux（Mint、Cinnamon shell）での最後のSkype更新（バージョン8.51.0.92）の後、メッセージ入力フィールドの言語レイアウトを切り替えると、不快なバグが発生しました。Alt+ Shiftを押すと、入力フィールドからフォーカスがウィンドウの上部パネルに移動します（メニュー）。

数日間のアクティブな使用のために、それは信じられないほど激怒しました。Ubuntuを使用している同僚は、バグもそこに現れると述べました。私はなんとか問題を解決しました（松葉杖）、私はそれをコミュニティと共有することにしました、誰かが助けてくれることを願っています。

ソースを注意深く分析する時間がなかったので、このパネルをメニューから完全に削除することにしました。まったく必要ありません。ほとんどすべての機能が以下に複製されており、3ポイントメニューから設定にアクセスできます。

1. npmパッケージマネージャーをインストールします（インストールされていない場合）。

sudo apt install npm

2.パッケージをasarに入れます

sudo npm install -g asar

3.パックされたSkypeファイルがあるディレクトリに移動します

cd /usr/share/skypeforlinux/resources/

4.必要なapp.asarを相対appディレクトリに解凍します

sudo asar extract app.asar app

5. Skypeをオフにし

ます。6。元のapp.asarファイルの名前を変更します（Electronがasarファイルを見つけられない場合、相対アプリディレクトリから解凍されたソースを使用します）。

sudo mv /usr/share/skypeforlinux/resources/app.asar /usr/share/skypeforlinux/resources/app.asar.backup

7. Skypeのパフォーマンスを確認する：オンとオフを切り替えるだけです。

8.ファイル/usr/share/skypeforlinux/resources/app/WindowBase.jsをお気に入りのテキストエディターのsudoで開き、BrowserWindowオブジェクトを作成した後、次の行をコンストラクターに追加します。

this._instance.setMenuBarVisibility(false);

パラグラフ8の関数に関するこれの説明：win.setMenuBarVisibility（表示）

もちろん、解決策は最も美しくはありませんが、（次の更新まで）機能します。

ご清聴ありがとうございました。
