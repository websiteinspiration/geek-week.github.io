Python 3.9的新功能

回顾最新Python迭代中包含的最佳功能。



时机已到，发布新版本的Python是不可避免的。它现在处于beta（3.9.0b3）中，但是很快我们将看到Python 3.9的完整版本。

其中的一些最新功能非常有趣，发布后看到它们的使用将令人惊讶。我们将考虑以下内容：

  • 字典组合运算符
  • 类型提示
  • 两种新的字符串方法
  • 新的Python解析器非常酷

首先让我们看一下新功能以及我们将如何使用它们。

字典组合


语法的新功能，也是我最喜欢的功能之一。如果我们有两个字典 a并且b需要合并，那么我们现在使用联合运算符。

我们有合并运算符“ |”：

a = {1: 'a', 2: 'b', 3: 'c'}
b = {4: 'd', 5: 'e'}
c = a | b
print(c)

[Out]: {1: 'a', 2: 'b', 3: 'c', 4: 'd', 5: 'e'}

还有更新语句“ |=”，它将更新源词典：

a = {1: 'a', 2: 'b', 3: 'c'}
b = {4: 'd', 5: 'e'}
a |= b
print(a)

[Out]: {1: 'a', 2: 'b', 3: 'c', 4: 'd', 5: 'e'}

如果我们的字典有一个公共键，那么将使用第二个字典中的一个键值对：

a = {1: 'a', 2: 'b', 3: 'c', 6: 'in both'}
b = {4: 'd', 5: 'e', 6: 'but different'}
print(a | b)

[Out]: {1: 'a', 2: 'b', 3: 'c', 6: 'but different', 4: 'd', 5: 'e'}

使用迭代更新字典


“ |=” 运算符的另一个有趣行为是能够使用迭代对象（例如，列表或生成器）使用新的键值对更新字典。

a = {'a': 'one', 'b': 'two'}
b = ((i, i**2) for i in range(3))
a |= b
print(a)

[Out]: {'a': 'one', 'b': 'two', 0: 0, 1: 1, 2: 4}

如果我们尝试使用标准的联合运算符“ |” 重复相同的操作TypeError，则会得到，因为它将只允许类型之间的联接dict。
