一般的なネットワークセキュリティエラーの回避



9月中旬には、それが知られるようになった 1つのロシアの通信事業者のネットワークで運用-検索対策（SORM）システムの操作に関する情報が含まれていたデータのほとんど2TBの漏洩、について。リークは、正しく構成されていないrsyncバックアップユーティリティが原因でした。このようなエラーは、大企業の問題の一般的な原因です。この記事では、最も一般的な7つのネットワークセキュリティエラーを分析します。これらのエラーを検出して排除する方法について説明します。

ネットワーク内での攻撃の開発が成功する一般的な理由は、通信チャネルまたはデータ処理およびストレージシステムの構成におけるエラー、およびIS規制への違反です。これにより、使用されている保護具の効果が低下し、攻撃者が侵入して攻撃を仕掛ける可能性が高くなります。インシデントを調査してトラフィックを分析するプロジェクトの期間中、PT Expert Security Centerのチームは、情報システムの構成における典型的なエラーと企業のIS規制への違反を定期的に発見します。これらのエラーが何であるか見てみましょう。

7つの一般的なネットワークセキュリティエラー


私たちの実践が示すように、10の組織のうち9つでは、その規模や活動範囲に関係なく、次のエラーが最も頻繁に発生します。

  1. 資格情報をネットワーク経由でクリアテキストで転送します。
  2. 暗号化されていないメールメッセージ。
  3. リモートアクセスのためのユーティリティの使用。
  4. ブロードキャストプロトコルLLMNRおよびNetBiosの使用。
  5. ネットワーク構成エラー。
  6. TOR、VPNトンネル、およびネットワークアクティビティを非表示にするその他のツール。
  7. システムの悪用（暗号通貨マイナー、急流）

エラーの理由は、情報セキュリティへの注意が不十分である、組織内のISおよびIT規制の欠如または違反、システム構成のエラー、および大規模な企業ネットワークでの正しい構成の制御が難しいためです。

次に、各エラーについて説明し、それらがどのような結果をもたらす可能性があるか、どのようにしてそれらを特定できるかを示し、それらの解消についての推奨事項を示します。

ネットワークを介した資格情報の送信をクリアする


ユーザー資格情報がプレーンテキストで送信されるネットワークプロトコルの使用はまだあります。これらはHTTP、暗号化のないメールプロトコル、LDAPおよびTelnetです。私たちの調査によると、重要な情報をオープン形式でネットワークリソースに保存しているのは、セキュリティ分析を実施した組織の44％です。ネットワークが侵害された場合、攻撃者はパッシブモードで資格情報を傍受し、インフラストラクチャ内での存在を統合して、特権を増加させることができます。

次によくある間違いは、組織のサーバーから外部メールサーバーへの途中でオープンメールプロトコルを使用することです。これにより、ネットワーク内で安全な形式で送信された手紙が、後でインターネットを介してオープンな形式で送信される可能性があります。その結果、攻撃者は（たとえば、インターネットプロバイダーを介して）外部ネットワークトラフィックにアクセスでき、手紙から情報を自由に受け取ることができます。

外部ネットワークに送信される安全でない送信メールを検索するために、SMTPプロトコル、送信元および宛先アドレスを使用する PT NADフィルターを使用しました。暗号化された接続を除外するために、STARTTLSコマンドを使用してフィルターを追加しました。その結果、添付ファイル付きの手紙が見つかり、クリアテキストで送信されました。

エラーの考えられる解決策：

  1. メールを送信するときにTLSを強制するようにサーバーを構成します（ただし、この場合、暗号化をサポートしていないサーバーにレターが配信されない場合があります）。
  2. 暗号化およびデジタル署名されたメッセージを送信するための標準であるS / MIMEの使用を設定します。メールクライアントとS / MIME証明書のセットアップが必要です。詳細はこちら。
  3. PGPを適用します。PGPを強制的に使用すると、平文での文字の送信も除外されますが、これにはクライアントでの追加の構成と受信者への公開鍵の転送が必要です。このオプションは、特殊なケースでの使用により適しています。

リモートアクセスユーティリティの使用


従業員は、TeamViewer、Ammyy Admin、RMSなどのリモートアクセスツール（RAT）をよく使用します。これが内部ISポリシーで許可されている場合、攻撃者が同じツールを使用する場合、不正な使用と正当な使用を区別することは困難です。

接続性はTeamViewerを通じてトラフィック分析システムを使用して検出できます。私たちのケースでは、そのようなネットワークセッションが2つ見つかりました。組織がリモートコントロールユーティリティの使用を禁止している場合は、情報セキュリティの専門家が調査を行い、活動の原因を特定する必要があります。

RATユースケースを検出するためのもう1つのメカニズムは、事前定義されたルールです。彼らの助けを借りたビデオで、リモート管理ユーティリティを使用している事実を発見しました。

LLMNR NetBios


組織のネットワーク設定に関するもう1つの問題は、スプーフィングされたLLMNRおよびNetBiosプロトコルの使用です。これらのプロトコルにより、ローカルL2ネットワークセグメントのブロードキャスト要求を通じて、DNSサーバーを使用せずに隣接するコンピューターの名前を解決できます。これらのプロトコルは、DNSが利用できない場合にも自動的に使用されます。会社の内部ネットワークに侵入する侵入者の場合、彼は「中間者」攻撃（Eng。Man in the middle、MITM）を実行することができます。攻撃者はブロードキャスト要求に応答し、攻撃者が制御するサーバーに被害者の要求をリダイレクトできます。この攻撃を実行すると、認証データを傍受できるようになります。

私たちは、識別しようとしましたPT NADの「アプリケーションプロトコル」ウィジェットを使用したこれらのプロトコルの使用。通常のプロトコルに加えて、LLMNRおよびNBNSプロトコルが使用されていることがわかりました。それらをフィルターに追加すると、このプロトコルを使用してリクエストを送信したすべてのクライアントも見つかりました。

トラフィック非表示ツールには、VPN、Tor、暗号化プロキシ、およびその他の暗号化トンネルがあります。それらの無許可で無制限の使用は、組織内の保護装置の有効性の低下につながる可能性があります。トンネリングされたトラフィックで送信されるコンテンツに対する制御の喪失。攻撃者は、従業員の自宅のコンピュータが危険にさらされている場合、暗号化されたトンネルを組織の内部ネットワークに侵入させることができます。

これらのツールの使用を識別するには、次のフィルターが適しています。Torはその下にマスクされているため、Torネットワークノードの現在のアドレスを含むtor-relaysレピュテーションリストと、TLSプロトコルを使用するフィルターに基づいています。 「疑わしい」セッションで使用されるTLS証明書は自動的に生成され、Torネットワーク接続を示します。

VPNやその他のトンネルを検出するには、PPTP（ポイントツーポイントプロトコル）キーワードフィルターを使用できます。SOCKS5トラフィックを検出するには、プロトコルで既知のフィルターを使用します。したがって、外部ホストとSOCKS5を介した多くの接続を使用したVPNセッションが見つかりました。

システムの誤用


システムの誤用には、暗号通貨マイナー、Bittorentクライアント、およびオンラインゲームの使用が含まれます。これはセキュリティに直接的な脅威をもたらすわけではありませんが、コンピューティングシステムと情報転送チャネルへの負荷を増大させ、マルウェアをインストールするリスクも伴います。

マイナーレピュテーションリストは、よく知られているマイニングプールのアドレスや、さまざまな暗号通貨のブロックチェーンノードを含むマイナーの識別に役立ちます。その結果、クリプトマイナーの操作を示す多数のDNSクエリが表示されます。クリプトマイナーの操作の別の指標は、機能したルールである可能性があります。

Bittorentとオンラインゲームを使用すると、より簡単です。急流のトラフィックを検索するには、Bittorentプロトコルを使用してフィルターを使用し、オンラインゲームには、人気のオンラインゲームのサーバーを使用します。これは、雇用者が望む方法ではなく、労働時間を使用する従業員を計算するのに役立ちます。

対策は上記の段落とほとんど同じです。

  1. ローカルユーザーの権限を区別します。
  2. ソフトウェアのホワイトリストポリシー。
  3. ウイルス対策とそのデータベースを更新します。

要約する


ほとんどの企業では、大規模な企業ネットワークを正しく設定し、IS構成、ポリシー、規制に準拠していないという問題に気づきます。これは、ネットワークの絶え間ない成長とネットワーク内の変化、およびルールとポリシー自体の変化によるものです。多くのエラーを回避するための一般的なガイドラインは次のとおりです。

  1. オープンプロトコルの使用を最小限に抑えます。
  2. ネットワークアクセスの差別化を制御します。
  3. ユーザー権限を区別します。

同時に、組織内のネットワークアクティビティを監視し、構成エラーと悪意のあるアクティビティの両方をタイムリーに検出できるツールがすでに市場に出ています。そのようなシステムの1つがPT Network Attack Discoveryです。上級スペシャリストのAlexey Lednev
