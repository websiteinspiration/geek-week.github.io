9 pengumuman utama konferensi Microsoft Build 2020

Konferensi dua hari kami, Microsoft Build 2020, yang menyiarkan lagu utama yang kami selenggarakan di Habré , telah berakhir. Pada artikel ini kita akan berbicara tentang pengumuman yang paling ambisius dan menarik dari acara masa lalu. Lihat di bawah kucing!

Dan kebaruan mana yang paling Anda sukai? Bagikan komentar Anda.



Windows Terminal 1.0 Release


Kami sangat senang mengumumkan rilis Terminal Windows 1.0! Terminal Windows telah berjalan jauh sejak pengumuman di Microsoft Build 2019 . Seperti biasa, Anda dapat mengunduh Terminal Windows dari Microsoft Store atau dari halaman rilis di GitHub . Terminal Windows akan memiliki pembaruan bulanan mulai Juli 2020.

Kami mengatakan lebih rinci tentang terminal baru kemarin di ini artikel tentang Habr.



Menciptakan salah satu dari 5 superkomputer paling kuat di dunia


Superkomputer Microsoft yang baru  dikembangkan  berdasarkan cloud Azure dalam  kolaborasi  dan khusus untuk  OpenAI , sebuah organisasi penelitian yang didirikan oleh Elon Musk dan Sam Altman. Kapasitasnya lebih dari 285.000 inti prosesor (inti CPU), 10.000 prosesor grafis (GPU), dan kecepatan koneksi jaringan adalah 400 gigabit per detik untuk setiap server. Ini dirancang khusus untuk melatih generasi berikutnya dari model AI terdistribusi besar yang dapat melakukan berbagai fungsi yang lebih luas dan, karenanya, membutuhkan sumber daya komputasi yang jauh lebih banyak.

Baca detailnya di situs berita kami (dalam bahasa Rusia).



Preview Package Manager untuk Windows


Manajer paket dirancang untuk membantu Anda menghemat waktu. Bahkan, itu adalah seperangkat alat perangkat lunak yang membantu mengotomatiskan proses mengunduh perangkat lunak ke komputer Anda. Anda menunjukkan aplikasi mana yang ingin Anda instal, dan ia mencari versi terbaru (atau yang Anda tentukan) dan menginstalnya di komputer.

Hampir setiap pengembang ingin memiliki pengelola paket asli di Windows. Hari ini akhirnya tiba. Salah satu berita kerennya adalah open source. Sekarang ini sangat sederhana, misalnya, menginstal terminal winget, kemudian menginstal winget PowerShell, dan setelah winget menginstal powertoys.

Baca lebih lanjut tentang Pengelola Paket Windows di blog kami .



Pembaruan Kerangka Kerja Cairan


Platform Kolaborasi Web Interaktif Fluid Framework,  diperkenalkan  setahun lalu di Build 2019,  menjadi  perangkat lunak sumber terbuka. Dan juga, segera fungsi Fluid Framework akan tersedia sebagian tidak hanya untuk pengembang, tetapi juga untuk pengguna akhir: komponen dan lingkungan kerjanya akan muncul di Office.com dan di versi web Outlook.

Integrasi Fluid Framework pertama di Microsoft 365 akan memungkinkan beberapa pengguna untuk secara bersamaan membuat perubahan pada dokumen dan membuat komponen yang terhubung yang dapat dibagi secara instan dan sangat mudah dalam aplikasi. Misalnya, tabel, bagan, dan daftar tugas dapat ditempelkan ke versi web Outlook sehingga data penjualan, tugas proyek, dan laporan penelitian Anda terus diperbarui dengan perubahan terbaru. Di Office.com, Anda bisa membuat dan mengelola ruang kerja Fluid Framework, termasuk dalam aliran aktivitas dokumen Anda, dalam daftar rekomendasi, di @ sebutkan, atau mencarinya di Office.com. Dan karena komponen Fluid Framework memiliki bobot rendah, pengeditan diterapkan secara instan.

Lingkungan jaringan Fluid dapat secara instan mengubah aplikasi Anda menjadi alat kolaborasi. Ini mencakup struktur data tersinkronisasi latensi rendah dan layanan switching untuk menghubungkan titik akhir. Jika Anda mengganti struktur data statis dengan struktur data Fluid, aplikasi Anda akan langsung mulai mendukung kolaborasi waktu nyata.

Melepaskan potensi penuh Fluid Framework adalah mungkin ketika menciptakan komunitas pengembang multikultural, terbuka, dan aktif. Untuk alasan ini, Microsoft akan mengubah Fluid Framework menjadi lingkungan sumber terbuka, yang memungkinkan pengembang untuk memanfaatkan Fluid Framework inti dalam aplikasi mereka sendiri. Kami merilis dokumentasi dan alat tambahan, dan juga mengundang pengembang untuk bekerja dengan Microsoft dalam pembuatan dan rilis Fluid Framework.

Microsoft Edge: Perbaikan untuk Pengembang


Microsoft Edge memperkenalkan serangkaian fitur baru untuk meningkatkan alur kerja bagi konsumen, pengembang, dan pelanggan komersial.

Pertama-tama, ini adalah cara pencarian yang lebih cepat dan lebih kontekstual menggunakan bilah samping. Cukup pilih kata atau frasa, klik kanan dan pilih "cari di bilah sisi" di menu konteks. Hasilnya akan ditampilkan di panel di sisi kanan jendela browser, sehingga pengguna tidak kehilangan konteks konten asli.

Untuk pengembang, kami telah berkontribusi pada proyek Chromium, termasuk peningkatan aksesibilitas, tulisan tangan, gulir, dan pelokalan. Versi pratinjau Webview 2.0 juga tersedia untuk pengembangan dengan .NET dan UWP. Dan berkat peningkatan baru pada Progressive Web App (PWA) (dalam pratinjau), aplikasi ini terasa lebih asli, dengan antarmuka pengguna yang lebih alami untuk berbagi, pemberitahuan, dan banyak lagi.

Luncurkan Reuni Proyek - Cara Baru Mengembangkan Aplikasi Windows


Pengguna menghabiskan total lebih dari 4 triliun menit sebulan di Windows 10 - itu 75% lebih dari tahun lalu. Memperkenalkan Reuni Proyek untuk membantu pengembang menciptakan lebih banyak inovasi untuk 1 miliar perangkat Windows 10 aktif per bulan. Inisiatif ini bertujuan untuk menyediakan integrasi yang mudah antara antarmuka pemrograman Win32 dan Universal Windows Platform, menyediakan platform pengembangan tunggal dengan dukungan kompatibilitas mundur.
