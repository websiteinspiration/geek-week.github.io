NutrientBot、またはフィットネストレーナーからパンを取りたい方法

みなさん、こんにちは！すべての開発者はかつて自分自身のために何かをしたいという欲望に直面し、製品の重要な価値さえ重要でなくなっても、主なことはそれ自体が機能することです。バズ。  

ですから、5月だけの休暇​​として、コーディングの知識と他の重要なプロジェクトを1つのプロジェクトにまとめたかったのです。私の考えでは、私が人生を捧げ、それについて語りたい面白いサービスであることがわかりました。 

私は栄養士でもプロのスポーツ選手でもありません。もちろん、彼らが疑いなく私を信頼してくれるふりはしません。しかし、かつて、適切な栄養のトピックに非常に興味があり、記事を勉強し、訪問したホールでクールなトレーナーとコミュニケーションを取り始め、そのようなことに気づきました：人々は栄養計画を立てるためにかなりのお金を費やしています。

そして、私は決定しました。しかし、目的とパラメータに応じて、人の製品をグラムで自動的に計算するようなシステムを作成します。

私はバックエンドデベロッパーなので、フロントに出くわすことはほとんどありません。そのことを考え、ボットを作ることにしました。メッセンジャーの選択は、いくつかの要因によって影響を受けました：支払いシステムの需要と可用性。ソーシャルネットワークVkontakteが登場しました。

計算には、ハリス・ベネディクトの公式を使用しました。（ボットのユーザーが入力する）そのような入力パラメーターを考慮に入れました：

  • 重量
  • 高さ
  • 目標（体重の減少/体重の増加/維持）
  • 身体活動（すべての情報が利用可能であり、ハリス・ベネディクトの公式による計算の標準です）
  • 乳糖耐性
  • 床

製品にタンパク質、脂肪、炭水化物、Kcal、GI（グリセミックインデックス）、および乳糖含有量のフィールドがある製品データベースを作成しました。一般的には、途中でリストとコメントを付けたいと思います。

function _vkApi_call($method, $params = array()) { 
  $params['access_token'] = //    ,        ; 
  $params['v'] = VK_API_VERSION; 
  $url = VK_API_ENDPOINT.$method.'?'.http_build_query($params); 
  $curl = curl_init($url); 
  curl_setopt($curl, CURLOPT_RETURNTRANSFER, true); 
  $json = curl_exec($curl); 
  curl_close($curl); 
  $response = json_decode($json, true); 
  return $response['error']['error_msg']; 
  $err=$response['error']['error_msg'];
}

VKには支払いシステムがあるため、VKペイによる支払いを想定しました（VKサーバーがそのようなリクエストをWebhookに送信しますが、パブリックがSTOREとしてリストされており、VKペイを通じて支払い可能な製品がある場合のみ）。メッセージに添付できる翻訳。

ボットに何かを書き込むすべてのユーザーをデータベースに保存すると、支払いに応じてボットとのすべての通信が行われます。支払いがなかった場合、ボットは最初に支払う必要があると言い、支払いが行われた場合は次に何をすべきかを教えてくれます。テストのために、計画するためにボットにメッセージを入力できます。

支払い後、ボットは計算に必要なパラメーターを尋ねてきます。特定のユーザーのデータベースにすべて保存し、すべての情報を受け取った後、考えて答えます。作成した関数を使用して、ユーザーのuser_idを示すメッセージを送信します。
