PeerJSを使用したピアツーピアメッセージの送信

読者の皆様、こんにちは。では前の記事、私が使用したブラウザでシンプルなダイヤラにする方法について話しましたPeerJSを。そして今日、私は2人のユーザー間でメッセージを遅滞なく直接交換する方法を検討する予定です。

誰も気にしない？プレーヤー間でデータをすばやく交換する必要があるオンラインゲームを開発している場合は、ダイレクトメッセージングがおそらく必要です。

マークアップと初期化


2人のユーザー間の 単純なチャットの例を使用して、このテクノロジーがどのように機能するかを示し、コードをゲームデータの交換に適合させる方法についても説明します。

ピアオブジェクトの初期マークアップと初期化から始めましょう

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8" />
    <title>PeerJS  </title>
     <script src="https://unpkg.com/peerjs@1.0.0/dist/peerjs.min.js"></script>
</head>
<body>
	<h3> ID: <span id=myid ></span></h3>
	<input id=otherPeerId type=text placeholder="otherPeerId" ><button onclick="connectToNode(document.getElementById('otherPeerId').value)"></button>
	<div id=messages style="width:400px;height:60vh; background:#ADD8E6;margin:5px;">
	</div><br>
	<textarea id=mess style="width:400px;height:15vh" ></textarea><br>
	<button onclick="sendMess(document.getElementById('mess'))"></button>
	<script>
		var messList=[];		
		function addMess(mess) {
			messList.push(mess);
			document.getElementById('messages').innerHTML=messList.join("");
		}
		var peer=new Peer(); // peer		
		var conn; //,  
		peer.on('open', function(peerID) {
			document.getElementById('myid').innerHTML=peerID;			
		});		
	</script>
</body>

見出しでは、PeerJSを接続します。myidおよびotherPeerIdインデックスを持つ要素が、呼び出しの記事で果たす役割は何ですか？messList

配列はメッセージフィードを格納します。addMess関数は、この配列に要素を追加し、その内容を対応コンテナに出力します。 次は、ピアオブジェクトの初期化です。これは、前回の記事でも説明されています。 接続について少し説明します。接続を確立するには、ある参加者が別の参加者のpeerIDを知っていて、彼との接続を開始し、2番目の参加者がこの接続を受信する必要があります。





接続を確立する


peer.on('connection', function(c) { // ...
	conn=c;
	initConn();
});
function connectToNode(partnerPeer) { // ...
	conn = peer.connect(partnerPeer);
        conn.partnerPeer=partnerPeer;
	initConn();
}

ピア オブジェクトの「接続」イベントは、着信接続で発生します。機能接続オブジェクトピアが接続を確立します。どちらの場合も、接続オブジェクトを変数connに保存します。現在のトレーニングの例に関連する他のアクションは同じです（ただし、戦闘プロジェクトには違いがあるかもしれません）ので、私はinitConnを別の関数に入れました。

function initConn() {
	conn.on ('open', function () { // 
		  addMess("<div><h4> </h4></div>");
		  conn.on ('data', function (data) { // 
			 addMess("<div><b>: </b>"+data+"</div>");
		  });
	});
	conn.on('close',function() {addMess('----------- -------------');});
}

ここでは、2つのハンドラーをハングします。接続を開いて閉じるためです。接続を開くためのハンドラーで、データを受信するハンドラーを追加します。これにより、着信メッセージがダイアログコンテナーに追加されます。送信

ボタンを押してメッセージを送信する関数を実装するだけです。

  1. 彼のフィードにメッセージを追加します
  2. メッセージをパートナーに送信します（接続オブジェクトの送信メソッド）
  3. メッセージ入力フィールドをクリアします

    function sendMess(elem) {
	addMess("<div><b>: </b>"+elem.value+"</div>");
	conn.send(elem.value);
	elem.value="";
}

ゲームデータを転送するための適応


平文ではなく、ゲーム中に交換する必要があるデータを同じ方法で送信するために何をする必要がありますか？特別なことは何もありません。 JSには、オブジェクトを文字列に、またはその逆に変換するJSON.stringifyおよびJSON.parseメソッドがあります。データオブジェクトをラップし、送信する前にオブジェクトを文字列（JSON.stringify）に変換し、受信したデータを受信時にオブジェクト（JSON.parse）に変換します

//
gameObject={x:2,y:5,...}
conn.send(JSON.stringify(gameObject));

//
 conn.on ('data', function (data) { // 
       gameObject=JSON.parse(data);
 });

通常、ゲームオブジェクトやテキストメッセージを送信するために大量のデータは必要ありません。ただし、ページ全体のコンテナ全体（HTMLコードの束）のコンテンツを転送する場合は、大きな接続が変更されない場合があることに注意してください。

個人的な経験から言うと、この方法では10 KB（1万文字まで）を超えるメッセージを転送しないでください。そのようなメッセージを一時ファイルに書き込み、このファイルからコードを読み取るようにパートナーにコマンドを送信することをお勧めします（要点はわかったと思います）。

そうでなければ、これで止めることができます...

接続の切断


はい、これは起こっています。これの理由は不安定なインターネットです。勝ったことはほとんどないが、接続が切断され、すべての進捗状況が失われることはありませんか？これを回避するために、切断された接続を発生させるコードを追加しましょう。このため、イベント「close」を処理します。このイベントは次の場合に発生します。

  1. 接続は意図的に閉じられました
  2. インターネットの質が悪いために接続が失われたか、パートナーがタブを閉じた

    conn.on('close',function() {
    setTimeout(function() { 
        if(conn.partnerPeer) {
              var pp=conn.partnerPeer;
              conn = peer.connect(conn.partnerPeer);
              conn.partnerPeer=pp;
             initConn();
         }
	else conn=null;
    }
    ,2000);
    addMess('----------- -------------');
});

ここでは、接続が切断されてから2秒の遅延があるため、単に新しい接続を確立しようとしています。

connオブジェクトのpartnerPeerは、最初に接続を確立したパートナーにのみ存在します。つまり、接続が切断されたときに、接続の2つの側のうちの1つだけが接続の復元を開始します。
