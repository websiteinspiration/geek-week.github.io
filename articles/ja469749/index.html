TabキーでUXを改善する

アプリケーションを開発するとき、フロントエンドベンダーは、ユーザーがブラウザーによって提供されるキーボード機能をどのように使用するかに注意を払うことはほとんどありません。私も例外ではありませんが、ある日、「Tab」キーと「Shift + Tab」キーを押して、UXと移行に関するタスクを与えられました。

タスクの本質は透明でクリーンです。インターフェースがあり、そのレイアウトは下に表示されています。概念的には、1つのページに2つの異なるフォームを含めることができ、「タブで実行すると1つのフォームから別のフォームに移動しない」という要件がありました。



ブラウザがフォームのフォーカスを「自然に」ブロックできれば、すべてうまくいきます。下の図に例を示します。オレンジ色の境界線は現在の要素を示し、前の要素は灰色で示されています。

あなたが見ることができるように、「ネイティブ」動作が要件を満たしていません。では、この問題を解決しましょう。ソリューションは複雑ではないので、検討してください。

最後の要素（「Tab」を使用）または最初の要素（「Shift + Tab」を使用）から「tabindex」またはデフォルトでフォーカスをサポートすることから、フォーカスを「飛び出す」ことができない「ゲート」がある場合に理想的です。つまり、本質は単純です。「ゲート」は非表示の「入力要素」であり、「onFocus」イベント中に「イベント」イベントを引数として受け取り、元の要素にフォーカスを返します。下の図。



「event」オブジェクトの「relatedTarget」プロパティを使用すると、前の要素を取得できます。以下のソリューションの視覚化。

そして、これがコード自体です。「ES6 +」構文がないことに注意してください。これは、Babelのような異なる「トランスパイラー」を接続せずに、異なるブラウザーでコードをサポートするためです。

function getGateInput(handleTabOut) {
    var input = document.createElement("input");
    // not visibiliy:hidden or display:none as need to focus on this element
    var hiddingStyle =
      "opacity: 0;cursor: none;position: absolute;top: -10px;left: -10px;";

    input.setAttribute("style", hiddingStyle);
    input.addEventListener("focus", handleTabOut);

    return input;
  }

ここでは複雑なことは何もありません。「入力」が作成され、スタイルが「ゲート」を「隠す」ように設定されます。ここでは、「display：none」は使用されていません。ブラウザがそのような要素に「タブ」をフォーカスしていないためです。この動作のため、要素を透明にしてブラウザウィンドウの外に移動する必要があります。

function getTabOutHandler(element, GATES) {
    return function(event) {
      var relatedTarget = event.relatedTarget || event.fromElement;
      var target = event.target;
      var gatesTrapped = target === GATES[0] || target === GATES[1];

      if (gatesTrapped && isChild(relatedTarget, element)) {
        event.preventDefault();
        relatedTarget.focus();
      }
    };
  }

フォーカスを前のアイテムに戻すには、getTabOutHandlerを使用します。これがHOCです。最初の引数はコンテナー（「ゲート」を設定したもの）で、2番目の引数はgetGateInputを使用して作成した「ゲート」の配列を想定しています。この関数は、上記の原則に従って動作するイベントハンドラーを返します。

コンテナにフォーカスを移すために、「ゲート」を開閉する必要があります。これを行うには、「tabindex」属性を設定します。（-1-タブでフォーカスしない、0-ストリームに従ってフォーカスする）

function moveGates(open, GATES) {
    GATES[0].setAttribute("tabindex", open ? -1 : 0);
    GATES[1].setAttribute("tabindex", open ? -1 : 0);
  }

ゲートを制御するには、Tab（コード9）を押すのを「リッスン」するハンドラーをインストールし、フォーカスされた要素（activeElement）がコンテナー内にある場合は「ゲート」を閉じ、それ以外の場合は開きます。

window.addEventListener("keydown", function(event) {
      if (event.keyCode === 9) {
        if (isChild(document.activeElement, element)) {
          moveGates(false, GATES);
        } else {
          moveGates(true, GATES);
        }
      }
    });
