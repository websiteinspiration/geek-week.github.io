Unity UIのカスタム形状ボタン

最近、プロジェクトで非標準フォームのボタンを使用する必要がありました。しかし、この一見ささいな仕事は私にいくつかの困難を引き起こしました。

問題は、標準のUIボタンが透明な領域でもクリックを処理することです。この場合、これは完全に望ましくありません。同時に、Buttonコンポーネント自体には、ボタンのどの領域がホバー/クリックを処理できるかを担当する（インスペクターに表示される）パラメーターがありません。


しばらくインターネットを検索し、文書を喫煙した後、次のことが、このかなり単純な解決策で思い付きました。

（同様の例がUnityのドキュメントにあることを予約しますが、現時点ではそのコードはすでに古いものとしてマークされているため、更新して少し変更することにしました）

using UnityEngine;
using UnityEngine.UI;

public class ButtonClickMask : MonoBehaviour 
{
    [Range(0f, 1f)] //1
    public float AlphaLevel = 1f; //2
    private Image bt; //3
   
    void Start()
    {
        bt = gameObject.GetComponent<Image>();
        bt.alphaHitTestMinimumThreshold = AlphaLevel; //4
    }
}

  1. インスペクターに0から1までの値を持つ便利なスライダーを表示する属性。
  2. クリックを処理するためにカーソルがホバーするテクスチャの部分に必要な最小アルファチャネル値。
  3. ボタンの画像コンポーネント（ボタンではなく、ボタンで作業する必要があります）。
  4. alphaHitTestMinimumThresholdパラメーターは、クリックを処理できるように、テクスチャーの一部の透明度の最小レベルとまったく同じである必要があります。

これで、スクリプトの準備ができました。次に、ボタンのテクスチャを調整します。コードが機能してエラーが発生しないようにするには、インポート設定テクスチャで読み取り/書き込み機能を有効にする必要があります。[適用]をクリックして変更を受け入れることを忘れないでください。



次に、目的のボタンでスクリプトを終了し、アルファレベルを目的の値（私の場合は1）に設定します



。... ...起動して、透明な領域でのクリックがボタンで処理されなくなったことを喜ばしく思います。
