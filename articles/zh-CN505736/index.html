运行，PHPUnit，运行：我如何优化测试执行时间

从其中一个Web项目的工作开始，我们就力争通过测试实现高水平的代码覆盖，并且在开发的最初阶段，我并未考虑优化其执行速度。结果，随着项目的增长，测试范围的不断扩大以及团队的增长，测试执行时间从几秒钟增加到几十分钟。快速测试与整个应用程序的性能一样重要。


我是如何应对的？最终发生了什么？

该应用程序正在PHP 7.2和Symfony 4.2中开发。要编写测试，我们使用PHPUnit。大多数测试是针对REST API的集成测试。

所有测试耗时约20分钟。这导致以下问题：


  • 在测试期间，自动CI流程开始受阻，导致免费的CI赛跑者精疲力尽，
  • 开发人员被迫在虚拟队列中排队，等待CI管道完成，
  • 在所有测试期间，您可以在当地通过新闻制作一杯咖啡和一片叶子，这只会激起人们对忽略其发布的渴望。

#!/bin/sh

php_no_xdebug() {
  temporaryPath="$(mktemp -t php.XXXXXX).ini"

  # Using awk to ensure that files ending without newlines do not lead to configuration error
  php -i | grep "\.ini" | grep -o -e '\(/[a-z0-9._-]\+\)\+\.ini' | grep -v xdebug | xargs awk 'FNR==1{print ""}1' | grep -v xdebug >"$temporaryPath"

  php -n -c "$temporaryPath" "$@"
  rm -f "$temporaryPath"
}

php_no_xdebug $@
