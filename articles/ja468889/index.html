スクリプトコールバック関数

場合によっては、COMコンポーネントがクライアントに通知を送信するか、コールバック関数を呼び出す必要があります。スキームは単純です。コンポーネントはインターフェースを公開し、クライアントはインターフェースから継承されたオブジェクトを作成してコンポーネントに渡し、コンポーネントはインターフェースの関数を呼び出し、それによってクライアント側の関数を呼び出します。


Visual BasicまたはVisual Basic for Applicatonsの場合、任意のインターフェイスから継承したクラスを記述できますが、VBScriptスクリプトファイルではこれは不可能です。


ここでインターフェースは私たちを助けるために急いでいますIDispatch。このインターフェイスを使用すると、強力なコンポーネントが謙虚にクライアントの控えめな役割を引き受け、小さなスクリプトが実際のオートメーションサーバーになります。


FreeBASICプログラミング言語でコンポーネントを開発します。


スクリプトファイルのクラス


スクリプトファイルでクラスを宣言して使用できます。このようなクラスは、暗黙的にインターフェースから継承され、IDispatch実際のCOMクラスです。


クラスを宣言し、そのインスタンスを後でコンポーネントに渡します。


Class CallBack

    Function CallBack(Param)
        '   
        WScript.Echo Param
        CallBack = 0
    End Function

End Class

コンポーネントはクラスのインスタンスを受け取りCallBack、関数CallBackを呼び出して、パラメータにテキストを含む文字列を渡します。


' 
Dim Component
Set Component = CreateObject("BatchedFiles.TestCOMServer")

'    ,     
Dim objCallBack
Set objCallBack = New CallBack

'      
Component.SetCallBack objCallBack, ""

'      
result = Component.InvokeCallBack()
WScript.Echo result

Set objCallBack = Nothing
Set Component = Nothing
