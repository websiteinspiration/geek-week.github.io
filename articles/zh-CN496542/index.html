欧洲的互联网流量增长了一半半。骨干提供商记录负载记录

自三月以来一直在进行有关欧洲人大规模自我隔离增加了各级互联网基础设施负担的讨论，但是不同的来源提供了不同的数据。有人说负载增加了很多倍，而另一些人说大约增加了20％。但是，无论如何，对于阿姆斯特丹的TIER-1集线器来说，它处于中间位置：根据AMS-IX的统计数据，平均流量负载增加了约50％，从4.0 TB / s增加到了6.0 TB / s。


早在3月中旬，YouTube宣布将降低英国和瑞士用户的视频质量，此后，整个欧洲和整个世界范围内。其他视频托管和流媒体服务，主要是Netflix和Twitch，也开始采取相同的措施。

但是，尽管每个人都提到负载急剧增加，但没有一个单一的来源明确指出哪些数据流存在问题。

如果我们看一下AMS-IX的统计数据，AMS-IX是在阿姆斯特丹设有主要枢纽的最大的欧盟骨干网提供商之一，情况就开始变得清晰起来。

首先，值得注意的是，去年年底用户之间信道消耗增加的趋势开始形成，这符合4G网络发展模式，并进一步过渡到5G。实际上，隔离措施导致这样的事实，即仅在两三年后，提供商和电信运营商才预期出现这种负载。以下是AMS-IX图表，该图表反映了过去一年中提供商节点上的负载动态：


这些是与AMS-IX网络连接的所有连接和数据中心的统计信息，也就是说，这些数据非常相关，显示了整个欧洲的负载动态。

如果您仔细看一下上面的图片，您会看到一个更早的论点的证实，即不仅冠状病毒应归咎于互联网拥挤：2019年10月至11月间，渠道消费增长的动态变得明显，当时甚至在中国都没有发现这种病毒。此外，从10月到11月的一个月中，流量增长了〜15％或〜0.8 Tb / s，从〜4.2 Tb / s增至5 Tb / s。

现在，在该渠道的每日消费图表上，您将不会感到惊讶。负载的增长与白天的时间一致，其峰值在接近午夜的时间下降，在深夜里急剧下降到几乎为零的值：


值得注意的是，在当前自我隔离的情况下，一周中的一天不再影响欧洲用户的频道消费。在几乎相同的每周负荷计划中，只有星期二被淘汰-在这一天，人们坐在网上的时间比其他日子多一点。再过一会，负载高峰持续到上周日：


而且，实际上，是AMS-IX网络上的每月负载图：


一些“专家”将网络负载的增加归因于人们向远程工作的过渡，但这并不完全正确。每个使用过Zoom或其他VoIP的人都知道，在视频会议模式下，信道上的负载是多么微不足道：Skype，Zoom或其他应用程序从来没有设定自己以高比特率提供FullHD图像的目标。他们的任务是纯粹的功利主义-给人一个看和听对话者的机会，现代频道上没有任何高质量和高负荷的话题。相反，PornHub通过其订户共享产生的流量比欧洲大陆上所有远程工作者的总流量还多。

更为现实的情况是，主要负载来自开头提到的YouTube和Netflix文章，可以在工作日结束后垂直上升至6 Tb / s的图表中清楚地看到。负载持续到午夜-大部分时间关闭“电影”和“串行”并进入睡眠状态。

一般而言，网络必须应对增加的负载，而当前的情况只会鼓励提供商升级“最后一英里”的骨干网和基础设施，因为在欧盟，ADSL和xADSL宽带仍然很流行，到2020年几乎是蛮荒的，而3 -4G无法应付。

您可能会认为，现在不仅恒定，而且峰值负载也给通信质量带来压力：历史上，欧洲网络第一次遇到这种流量，并且在某个时间点的负载波动在黄金时段高达2 Tb / s，从6稳定到8峰值Tb / s。

但是实际上，这种情况对于提供商来说是相当熟悉的，我们所有的问题更多地在于数据总量，而不是波动。

考虑到该地区的频道消费平均每年增长20％至26％，现在欧盟的高峰波动与五年前该大陆所有稳定的互联网流量相当，但这种“麻烦”负载几乎总是存在。这是来自DE-CIX的图表，DE-CIX是来自法兰克福的又一主要欧盟干线，是欧洲大陆和阿姆斯特丹之间最大的两个枢纽之一：


如您所见，DE-CIX网络在2015年的峰值负载约为4 Tb / s，而平均而言仅为2 Tb / s。如果以线性方式推断情况，那么从逻辑上讲，平均负载为6 Tb / s，当前峰值应为10-12 Tb / s。一切都应运而生：流媒体服务的发展，4G和互联网对每个家庭的渗透。但是那没有发生。不管信道上稳定负载的大小如何，在DE-CIX的所有五年观测中，峰值负载均为+ 2-Tb / s。为什么会发生？很难明确回答，这是骨干网专家的问题。
