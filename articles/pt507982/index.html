Depurar contêiner PHP usando Xdebug e PhpStorm

Uma tradução do artigo foi preparada antes do início do curso Backend PHP Developer .









Instrução do Docker # 9: xdebug



Vou criar uma página php muito simples e executá-la com xdebug e PhpStorm.



Os arquivos de origem podem ser encontrados aqui:

github.com/ikknd/docker-study na pasta receita-09



1. Crie um "Dockerfile" na pasta "docker": 



FROM php:7.2-fpm

#Install xdebug
RUN pecl install xdebug-2.6.1 && docker-php-ext-enable xdebug

CMD ["php-fpm"]


Execute este comando na pasta janela de encaixe para criar a imagem: 



docker build -t php-xdebug-custom -f Dockerfile .


2. Crie o arquivo docker-compose.yml na pasta docker: 



version: "3.7"

services:

  web:
    image: nginx:1.17
    ports:
      - 80:80
    volumes:
      - /var/www/docker-study.loc/recipe-09/php:/var/www/myapp
      - /var/www/docker-study.loc/recipe-09/docker/site.conf:/etc/nginx/conf.d/site.conf
    depends_on:
      - php

  php:
    image: php-xdebug-custom
    volumes:
      - /var/www/docker-study.loc/recipe-09/php:/var/www/myapp
      - /var/www/docker-study.loc/recipe-09/docker/php.ini:/usr/local/etc/php/php.ini


Aqui eu uso a imagem personalizada php-xdebug em vez de php: 7.2-fpm „



3. Faça as seguintes configurações no arquivo php.ini: 



[xdebug]
zend_extension=xdebug.so
xdebug.profiler_enable=1
xdebug.remote_enable=1
xdebug.remote_handler=dbgp
xdebug.remote_mode=req
xdebug.remote_host=host.docker.internal
xdebug.remote_port=9000
xdebug.remote_autostart=1
xdebug.remote_connect_back=1
xdebug.idekey=PHPSTORM


4. Configure o servidor no PhpStorm:



Arquivo -> Configurações -> Idiomas e estruturas -> PHP -> Servidores

Adicione um novo servidor usando o ícone + e configure-o conforme mostrado na captura de tela a seguir:







Certifique-se de selecionar “Usar mapeamento de caminho” ( "Use mapeamentos de caminho") e mapeou a pasta php para /var/www/myapp



.



5. Configure o depurador remoto do PHP no PhpStorm:



Execute -> Editar configurações -> PHP Remote Debug



Adicione uma nova configuração e atribua valores a ela, como na captura de tela a seguir:







6. Selecione a configuração de depuração no painel de depuração do PhpStorm







7. Vá /var/www/docker-study.loc/recipe-09/docker/



e execute: 



docker-compose up -d


Se eu digitar myapp.loc/



o navegador agora, verei os resultados do arquivo index.php



.



Posso definir um ponto de interrupção, começar a ouvir as conexões no painel de depuração do PhpStorm e recarregar a página.



Tenha uma boa depuração!
