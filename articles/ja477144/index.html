CSSコードを1行追加してHTMLをレスポンシブにする方法

こんにちは、ハブル！Per Harald Borgenによる「CSSの1行を追加してHTMLをレスポンシブにする方法」という記事の翻訳を紹介します。



この記事では、CSSグリッドを使用して、画面またはブラウザーウィンドウの幅に応じて列数を変更するクールな画像グリッドを作成する方法について説明します。

最もクールなのは、応答性を追加するために、1行のCSSコードを記述する必要があることです。

つまり、醜いクラス名（col-sm-4、col-md-8など）でHTMLを乱雑にしたり、各画面のサイズに対応するメディアクエリを作成したりする必要はありません。

また、無料のCSSグリッドコースも作成しました。クリックしてコースへの完全なアクセスを取得します。



はじめましょう！

カスタマイズ


これは、最初のグリッドがどのように見えるかです：



HTMLコード：

<div class="container">  
  <div>1</div>  
  <div>2</div>  
  <div>3</div>  
  <div>4</div>  
  <div>5</div>  
  <div>6</div>  
</div>

CSSコード：

.container {  
    display: grid;  
    grid-template-columns: 100px 100px 100px;  
    grid-template-rows: 50px 50px;  
}

注：この例では、CSSグリッドとは関係がないため、ここでは考慮しないいくつかの基本的なスタイルがあります。

列をレスポンシブにすることから始めましょう。

基本的な応答性


CSSグリッドにはfrと書かれた値があり、コンテナーをパーツに分割できます。パーツの数はいくつでもかまいません。

grid-template-columnsの値を1fr 1fr 1frに変更します。

.container {  
    display: grid;  
    grid-template-columns: 1fr 1fr 1fr;  
    grid-template-rows: 50px 50px;  
}

これで、グリッドはコンテナーの幅全体を3つの部分に分割します。各列は独自の部分（ユニット）をとります。結果は次のとおりです。grid -template-columnsの



値を1fr 2fr 1frに変更すると、2番目の列は他の2倍の幅になり、合計幅は4ユニットになります（2番目の列は2つを占め、残りは1つずつです）。見た目：つまり、fr値を使用すると、列の幅を簡単に変更できます。





応答性の向上


ただし、このグリッドには常に固定数の列があるため、上記の例では必要な応答性が得られません。グリッドがコンテナーの幅に応じて列の数を変更するようにします。これを実現するには、3つの新しい概念を学ぶ必要があります。

repeat（）

まず、repeat（）関数から始めましょう。これは、行と列のパラメーターを指定するためのより便利な方法です。

.container {  
    display: grid;  
    grid-template-columns: repeat(3, 100px);  
    grid-template-rows: repeat(2, 50px);  
}

言い換えると、繰り返し機能（3、100px）は、100px 100px 100pxを書き込むことと同じです。最初のパラメーターは必要な列または行の数を示し、2番目のパラメーターはその幅を示します。開始時と同じレイアウトを取得します



。auto-fit

次に、auto-fit関数について考えます。指定した数の列を自動調整関数で置き換えます。

.container {  
    display: grid;  
    grid-gap: 5px;  
    grid-template-columns: repeat(auto-fit, 100px);
    grid-template-rows: repeat(2, 100px);  
}

これにより、次のようになります。



グリッドは、コンテナーの幅に応じて列数を変更します。

実際、この関数は単にコンテナーに100ピクセル幅の列を配置しようとしています。

ただし、すべての列の幅が厳密にたとえば100pxになるようにエンコードすると、列の全幅が表示されないため、希望する柔軟性が得られません。上の図からわかるように、グリッドは右側に空のスペースを残しています。

minmax（）

これを修正する必要がある最後の関数はminmax（）と呼ばれます。 100pxをminmax（100px、1fr）に置き換えるだけです。以下は最新のCSSコードです。

.container {  
    display: grid;  
    grid-gap: 5px;  
    grid-template-columns: repeat(auto-fit, minmax(100px, 1fr));
    grid-template-rows: repeat(2, 100px);  
}

CSSコードを1行記述することで、すべての応答性が追加されることに注意してください。

これは次のようになります。



ご覧のとおり、これは問題なく機能します。minmax（）関数は、min以上でmax以下のサイズの範囲を定義します。

したがって、列のサイズは常に少なくとも100ピクセルになります。ただし、より多くの使用可能なスペースがある場合、グリッドはそれを各列に均等に配分します。

画像を追加する


最後のステップは画像を追加することです。これはCSSグリッドとは関係ありませんが、このステップを見てみましょう。

まず、各グリッド要素内に画像タグを追加します。

<div><img src="img/forest.jpg"/></div>

画像を要素のサイズに合わせるには、要素自体と同じ幅と高さに設定し、object-fit：coverを使用します。次に、画像はその画像に提供された領域全体をカバーし、ブラウザは必要に応じて画像をトリミングします。

.container > div > img {  
    width: 100%;  
    height: 100%;  
    object-fit: cover;  
}
