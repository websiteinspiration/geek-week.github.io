火星のプロモーションコード

作成者：Maxim Pedchenko
制限時間 1秒
メモリ制限 64 MB
入る 標準入力またはinput.txt
結論 標準出力またはoutput.txt
火星でのYandex.Taxiの打ち上げから6か月が経過しました。火星の新年までに、Yandex.Taxiは火星人にプロモーションコードを提供することを決定しました。しかし、火星のサーバーは地球のルールに従って動作しないため、火星人は地球のプロモーションコードを使用できないことがわかりました。したがって、Yandex.Taxiは特別な火星のプロモーションコードを考え出しました。

火星のプロモーションコードは次のように生成されます。

  1. 暗号化キーが生成されます。
  2. 暗号化キーは、ランダムな長さのサブストリングに分割されます。
  3. すべての部分文字列のうち、長さLの部分文字列が選択されます。
  4. 選択した部分文字列が混合され、連結されます。
  5. 連結の結果はプロモーションコードです。

目的：
入力したプロモーションコードが有効かどうかを確認する必要があります。プロモーションコードが有効な場合は、「はい」を印刷する必要があります。無効な場合、「NO」。

決定


暗号化キーのすべての可能なパーティションを長さLのサブストリングに検討し、可能なパーティションからこのプロモーションコードを作成できるかどうかを確認する必要があります。

ヒントは条件ノートに隠されています：
プロモーションコードの長さは[6、30]の範囲です。
キーの長さは[6、30]の範囲です。

小さな制限は、効果的なソリューションが不要であることを示します。つまり、最適化の検索に時間を費やす必要がないことを意味します。問題を正面から解決することをお勧めします。

この状況は、製品のバックエンド開発の典型です。多くの場合、最適なアルゴリズムに数週間費やすことができる状況がありますが、制限を比較検討する場合、最適なソリューションではなく迅速なソリューションを使用する方がよいことが明らかになります。

したがって、プロモーションコードのある行を長さLのサブストリングSのシーケンスと見なします。各サブストリングSについて、暗号化キーからそれに等しいすべてのサブストリングS kを見つけます。各S kについて、その使用を覚えて次の部分文字列Sに進み、アルゴリズムを繰り返します。

部分文字列Sについて、未使用のS kを見つけることができない場合、別の分割オプションを試す必要があります。他にオプションがない場合、プロモーションコードは無効です。

各Sの少なくとも1つのイベントがS kを検出できた場合、プロモーションコードが有効であることを意味します。

火星輸送システムの最適化

著者：Dmitry PolishchukおよびAnton Todua
制限時間 3秒
メモリ制限 64 MB

2058年でした。最初の入植者のコロニーはすでに火星に上陸し、そこに生息し始め、Yandex.Taxiはシャトル駅のシステムを配備し始めました。

通常の操作では、シャトルステーションはエネルギーネットワークからの一定の電力供給を必要とします。ステーションに電力を供給するには、ステーション自体の内部にウラン原子力発電機を構築するか、別の（すでに電力が供給されている）シャトルステーションにケーブルを敷設する必要があります。異なるシャトルステーション内に発電機を構築するコストは異なる場合があります。シャトルステーション間のケーブル配線もコストが変動し、常に可能とは限りません。ケーブル接続は双方向です。

決定


まず、美しい説明から無向加重グラフに移動する価値があります。シャトルステーションの場所にピークがあり、ケーブルがエッジになり、ケーブルを構築するコストが対応するエッジの重みに変わります。しかし、問題は残っています-ステーション（ピーク）自体の内部にウラン発電機を構築するコストをどのように考慮するか？この質問に対する答えは、問題の本質です。

別の頂点-エネルギー源があり、対応するステーション（上）にウラン発生器を構築するコストに等しい重みで、この頂点からグラフの各頂点にエッジが描画されているとします。このような仮定により、接続されたグラフが得られます。グラフをツリーにして、エッジの重みの合計が可能な限り最小になるようにする必要があります。これは、グラフ内の最小全域木を見つける問題にすぎません。PrimaやKraskalなどの既知のアルゴリズムを使用して、最小のスパニングツリーを構築できます。

    bool Emplace(std::size_t v1, std::size_t v2) {
        while (v1 != v2) {
            if (v2 < v1) {
                std::swap(v1, v2);
            }

            auto& next_v2 = equals_[v2];
            if (next_v2 == v2) {
                next_v2 = v1;
                return true;
            }

            v2 = next_v2;
        }
        return false;
    }

private:
    std::vector<size_t> equals_;
};

決定


これは、慎重な実装と注意のためのタスクです。この決定には2つの主要なポイントがあります。
