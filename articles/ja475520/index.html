0pxからautoへの高さプロパティのCSS遷移

こんにちは、ハブル！

css transition（transition）プロパティをheightから0pxに作成する別の方法を共有したいと思いますauto。

WebコンポーネントTreeViewおよびDataGridを開発するときにこの問題に直面しました。 TreeViewでは、ノードのスムーズな展開/折りたたみ、およびDataGrid-追加コンテンツを含む行を作成することにしました。インターネットを読んだ後、私はいくつかの実装方法を見つけました。主なものは-プロパティmax-heightとJavaScriptです。 JavaScriptの実装は除外されました-トランジションとアニメーションをサポートするCSSがあります。残りmax-height、特にドロップダウンメニューの例では、すべてが機能します。

TreeViewでは、各ノードのネストが無制限であるため、コンテンツの最大の高さをすぐに決定することはできません。max-height非常に大きく設定すると、遷移アニメーションに問題が発生します。また、子ノードを展開すると、親の高さが増加し、重なる場合がありmax-heightます。好むとmax-height好まざるとにかかわらず、そうではありません。DataGridにも同じ問題があります-追加のコンテンツが連続して存在する可能性があります。必要height:auto！

それでは、プロパティheightfromから0pxへの遷移の実装を始めましょうauto。簡単な例を考えてみましょう。

させてください：

  • elBlock: HTMLDivElement -展開/折りたたむブロック;
  • elToggle: HTMLButtonElement -ボタンスイッチのステータス。

ブロックのcssクラスを定義します。ここで、コンテンツのトリミングと遷移自体を設定します。

.block {
    overflow: hidden;
    transition: height 500ms ease;
}

私たちは、イベントハンドラを記述するonClickためにelToggle：

elToggle.addEventListener("click", () => {
    if (elBlock.style.height === "0px") {
        elBlock.style.height = `${ elBlock.scrollHeight }px`
    } else {
        elBlock.style.height = `${ elBlock.scrollHeight }px`;
        window.getComputedStyle(elBlock, null).getPropertyValue("height");
        elBlock.style.height = "0";
    }
});

height:auto移行後に 払い戻しを追加する必要があります。

elBlock.addEventListener("transitionend", () => {
    if (elBlock.style.height !== "0px") {
        elBlock.style.height = "auto"
    }
});

さて、これですべてです。ブロックの展開/折りたたみは正常に機能し、コンテンツのサイズに依存しません。

このアプローチの短所は注目に値します。

  • JavaScriptを使用して、CSSのみを希望します。
  • トランジション中、コンテンツ（その高さscrollHeight）は変化する可能性があり、その完了後、returnの場合auto、ブロックの高さは一方向または別の方向に劇的に変化します。この影響を回避するには、変更を追跡し、を変更する必要がscrollHeightありますheight。練習が示すように、通常、展開/折りたたみの遷移にはそれぞれ0.5秒かかります。この間、ユーザーは内部を変更する時間がない可能性があります。たとえば、TreeViewの場合、子ノードを展開します。
