テトリスのランダム化アルゴリズムの歴史


1985年、アレクセイパジトノフとヴァディムゲラシモフはテトリスをリリースしました。この魅力的で中毒性の高いゲームでは、プレイヤーはランダムに出現するピースを接続する必要がありました。それ以来、150を超えるライセンス版のテトリスがリリースされました。ゲームモード、ルール、実装が異なるため、それらはすべてわずかに（または非常に）異なってプレイされました。テトリスランダマイザーは、ランダムに選択された形状を返す関数です。長年にわたり、形状を選択するためのルールは進化し、ゲームプレイとランダム性自体に影響を与えてきました。これらのアルゴリズムの一部はリバースエンジニアリングされ、文書化されています。私が重要だと考えるランダマイザーのリストをまとめました。テトリスの内部構造が長年にわたってどのように変化したかを記事で示します。

テトリス（1985年頃）


テトリスの最初のオリジナルバージョンには、偏りのないランダマイザーがありました。次のフィギュアの選択に影響を与えるものはなく、単に選択されてプレーヤーに表示されました。

ディスプレイスメントなしのランダマイザーを使用すると、プレーヤーが1つのフィギュアのシーケンス（「洪水」、フラッドと呼ばれる）または特定のフィギュアのないシーケンス（「干ばつ」、干ばつと呼ばれる）を受け取る状況が発生します。テトリスの異なるバージョンの設計者がこの問題をわずかに解決しようとした方法を確認します。

偏見のないランダマイザーはプレイヤーにとって最も複雑なパズルを作成しますが、それは不安定であり、無敵のシーケンスにつながる可能性があります（PDF）。ただし、コンピューターには真の乱数ジェネレーターがないため、実際のゲームではこれは起こりません。疑似乱数ジェネレーター（PRNG）は真のランダム性をシミュレートしようとしますが、連続して70千のZ形状を生成できるプロパティはありません。

テトリス、任天堂（1989）


4年後、NES用の非常に人気のあるバージョンのテトリスがリリースされました。

数値のフラッド（繰り返し）の数を減らすために、ランダマイザーに履歴チェックが追加されました。この簡単なチェックでは、次のことが行われました。

  1. フィギュアを選ぶ
  2. 図が前のものと一致するかどうかを確認し、
  3. もしそうなら、アルゴリズムは新しい形状を選択しましたが、一度だけ、
  4. そして結果がどうであれ、フィギュアはプレイヤーに与えられました。

連続して1ピースを取得する可能性は減少していましたが、ゲームが2つの交互ピースを発行することを妨げるものはありませんでした。さらに、このバージョンでは、30桁を超える干ばつが頻繁に発生しました。干ばつはどのタイプのテトラミノでも発生する可能性がありますが、このゲームでのスコアリングには私が重要な数字であり、その大きな干ばつが最終スコアに大きく影響する可能性があります。

function* historyRandomizer() {

がNESのためのテトリスが偏りなくランダムに比較アルゴリズムを改善し、その中の干ばつは依然として頻繁にありました。テトリスの時点で：グランドマスター（TGM）をマスターするために、実際には同じシステムを使用しましたが、長い歴史と多数のスローがあります。

4つのピースと4つのスローで物語を暗記する


function* historyRandomizer() {
  const pieces = ['I', 'J', 'L', 'O', 'S', 'T', 'Z'];

  // First piece special conditions
  let piece = ['I', 'J', 'L', 'T'][Math.floor(Math.random() * 4)];
  yield piece;

  let history = ['S', 'Z', 'S', piece];

  while (true) {
    for (let roll = 0; roll < 4; ++roll) {
      piece = pieces[Math.floor(Math.random() * 7)];
      if (history.includes(piece) === false) break;
    }
    history.shift();
    history.push(piece);
    yield piece;
  }
}

Tetris Worlds et on（2001）



テトリスワールドは、ランダムジェネレーターに質量を導入しました。彼は現在、テトリスワールドの後のゲームのほとんどの公式バージョンで、公式のランダマイザーになり、今日まで使用されています。

履歴ベースのランダマイザーは洪水を取り除く（または少なくとも洪水を最小限にする）のに役立ちましたが、干ばつを止めることはしませんでした。特定の条件下では、致命的な数字のシーケンスを取得する可能性がまだありました。

ランダムジェネレーター（Random Generator）は、「バッグ」（バッグ）の新しいシステムを使用してこれらの問題を解決します。このシステムでは、数字のリストが「バッグ」に入れられ、その後、「バッグ」が空になるまで、数字が次々にランダムに抽出されます。それが空になると、ピースはそこに戻り、プロセスが繰り返されます。ランダムジェネレーターには、サイズ7（7バッグ）の「バッグ」があります。つまり、7つのテトラミノのそれぞれで満たされた「バッグ」です。他のタイプの「バッグ」も可能です。たとえば、各タイプのテトラミノの2つのピースが配置される14バッグです。

「バッグ」、2桁の洪水と4つの数字（の「蛇」の歴史の欠如に起因する、など）は、その接合部で発生する可能性があります。つまり、ある意味で、これはNES向けの従来のテトリスと比較すると、一歩後退しています。

ピースは7バッグから安定して落下し、それにより予測が容易になります。あなたが「バッグ」のどの部分にいるのか、そして必要な数字がいつ来るのかを理解するのは簡単です。このランダム性ジェネレータの予測可能性により、ゲームは実際に無限にプレイできます。一般に、これは非常に愚かなシステムであり、それが一般に公式のランダマイザーになった方法は不明です。

テトリス：グランドマスター3-テロ本能（2005）


TGM3は、ランダム性を生成するという考え方を大幅に進歩させました。これは他のバージョンにはないユニークなシステムです。

バッグやストーリーの代わりに、TGM3はシェイプのプールを使用します。当初は各種類5枚、計35枚。フィギュアが引っ張られると、プールから削除されませんが、干ばつが最大のフィギュア（長期間削除されていないもの）に置き換えられます。徐々に、プールは、最終的に引き出されるまで、この数値でますます満たされます。これにより、「バッグ」システムの問題と、履歴のあるシステムが解決されます。彼女は、両方のタイプのランダム化から最善を尽くしています。

35体のプールと6投


function* tgm3Randomizer() {
  let pieces = ['I', 'J', 'L', 'O', 'S', 'T', 'Z'];
  let order = [];

  // Create 35 pool.
  let pool = pieces.concat(pieces, pieces, pieces, pieces);

  // First piece special conditions
  const firstPiece = ['I', 'J', 'L', 'T'][Math.floor(Math.random() * 4)];
  yield firstPiece;

  let history = ['S', 'Z', 'S', firstPiece];

  while (true) {
    let roll;
    let i;
    let piece;

    // Roll For piece
    for (roll = 0; roll < 6; ++roll) {
      i = Math.floor(Math.random() * 35);
      piece = pool[i];
      if (history.includes(piece) === false || roll === 5) {
        break;
      }
      if (order.length) pool[i] = order[0];
    }

    // Update piece order
    if (order.includes(piece)) {
      order.splice(order.indexOf(piece), 1);
    }
    order.push(piece);

    pool[i] = order[0];

    // Update history
    history.shift();
    history[3] = piece;

    yield piece;
  }
}
