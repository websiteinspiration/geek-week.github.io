GridmiAPI-RESTとRPCを操作するためのシンプルで柔軟なAndroidライブラリ

こんにちは、カブロバイトです！

Androidプラットフォーム用のネットワークアプリケーションを開発しているとき、一方のアプリケーションともう一方のアプリケーションは互いに似ていることを理解しています。この考えで、大多数の開発者は別の考えを開始します-「明らかに自分のタスクは新しいものではないので、既製のソリューションを使用しないのはなぜですか？」この瞬間から、現在の問題を解決するための適切なライブラリの検索が始まります。しばらくして-利益。ライブラリーが見つかり、現在の問題の解決策の実装を作成することだけが残っていたため、すべてが良いように見えました。そして、ここで問題が始まります。これらはスレッド制御に関連付けられており、Javaはそのコンパクトさで有名ではありませんでした。すべての「魔法」は開発者が直接実装する必要があります。時々私はこれに肯定的な側面だけを見つける、結局のところ、各プロジェクトのロジックは互いに根本的に異なります。常にストリームからストリームに切り替える必要がありますが、これはあまり便利ではなく、前述のように面倒です。この問題の解決策は以前に実装されましたが、希望するほどコンパクトではありません。 HTTPを操作するためのライブラリが助けになります-GridmiAPI。これは最もコンパクトなソリューションであり、データをリアルタイムで操作できます。

作業プロセス


すべてのライブラリの作業は、初期化から始まります。初期化は一度だけ必要です。初期化とは、次の構造を指します。

GridmiAPI.init("https://habr.com/API/", 8000, JSONObject.class);

  1. リクエスト処理エンドポイントアドレス
  2. サーバー応答タイムアウト
  3. サーバー応答をキャストするクラス

それで全部です！ライブラリは完全に使用できる状態です。

最初のサーバー要求


サーバーへの最初のリクエストを完了するには、数行のコードと必要なデータを記述する必要があります。信じないで？見る：

GridmiAPI.onRequest(this, new GridmiAPI.Request("profile/get"), new GridmiAPI.Handler.OUT() {
    
    @Override
    protected void onSuccess(GridmiAPI.Response response) {
        Log.d("TagGridmiAPI", "result = " + ((JSONObject) response.getData()).toString());
    }

    @Override
    protected void onFailed(Exception exception) {
        Log.d("TagGridmiAPI", "exception = " + exception.getMessage());
    }
    
}).start();

おめでとう！これがあなたの「Hello world！」です このライブラリを使用します。GridmiAPI.Handler.OUTクラスの2つの書き換えられたメソッドを見てみましょう。まず、これらの2つのメソッドはメインスレッドで呼び出されるため、リクエストが完了すると、アクティビティにアクセスしてメインスレッドを開始しても意味がありません。たとえばJSONを使用して、TextViewのテキストを変更するだけです。すべてが簡単です。

サーバーにファイルを送信


多くの場合、サーバーにファイルを送信する必要があります。また頭痛？ありえない。GridmiAPIライブラリを使用すると、頭痛なくファイルを簡単に送信できます。上記の例よりもわずかに多くの行がありますが、それでもコンパクトです。ファイルを送信する方法は次のとおりです。

@Override
protected void onActivityResult(int requestCode, int resultCode, @Nullable Intent data) {
    
    super.onActivityResult(requestCode, resultCode, data);
    if (data == null || data.getData() == null) return;

    try {

        //    
        GridmiAPI.Request request = new GridmiAPI.Request("POST", "photo/add");

        //     
        GridmiAPI.Multipart multipart = new GridmiAPI.Multipart(getContentResolver());
        multipart.appendData("photo", data.getData());
        
        //   
        request.setBody(multipart);
        
        //  
        GridmiAPI.onRequest(this, request, new GridmiAPI.Handler.OUT() {

            @Override
            protected void onSuccess(GridmiAPI.Response response) {
                try {
                    
                    //     
                    boolean result = ((JSONObject) response.getData()).getBoolean("result");
                    
                    // 
                    Toast.makeText(MainActivity.this, result ? "!" : "!", Toast.LENGTH_LONG).show();
                    
                } catch (Exception exception) {
                    this.onFailed(exception);
                }
            }

            @Override
            protected void onFailed(Exception exception) {
                //    
                Toast.makeText(MainActivity.this, exception.getMessage(), Toast.LENGTH_LONG).show();
            }

        }).start();

    } catch (Exception exception) {
        Toast.makeText(this, exception.getMessage(), Toast.LENGTH_LONG).show();
    }

}
