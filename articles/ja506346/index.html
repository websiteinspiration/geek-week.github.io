メールを送信する関数の例を次に示します。
public function email($emails, $body, $subject)
{
    if (empty($emails)) {
        return false;
    }

    $prefix = getenv('NATS_PREFIX');
    $encoder = new JSONEncoder();
    $options = new ConnectionOptions([
        'token' => getenv('NATS_TOKEN')
    ]);
    $client = new EncodedConnection($options, $encoder);

    try {
        $client->connect();
        $client->publish(
            $prefix . 'email',
            [
                'Emails' => $emails,
                'HtmlBody' => $body,
                'TextBody' => strip_tags($body),
                'Subject' => $subject,
                'Sender' => 'info@' . getenv('DOMAIN_NAME')
            ]
        );
    } catch (\Exception $exception) {
        return false;
    }

    return true;
}

ストリーミング組織


すべてが許可で簡単であれば、請求は自分で台無しにすることができます。このようなタスクは、平均的な統計開発者によく見られます。しかし、ストリーミングでは、一見したようにすべてが明確ではありません。なにが問題ですか？

  1. 私たちは彼に対処したことがありません。教師用のサーバー+クライアントが必要でした
  2. 画像をミラーリングする必要があります。縦表示（ダンス動画の特徴）
  3. キャンプに参加する参加者数は正確にはわかりませんでした。また、（既製のストリーミングソリューションの）どこでも（今すぐ）\ \ \トラフィックのアカウントを購入することはできませんでした。それはすべての利益を飲み込むでしょう
  4. 少なくとも参照者によるコンテンツの盗難に対するiframe保護が必要でした

ソリューションとして、larixブロードキャスターアプリ-> nginx + rtmp-> cdn->クライアントを選択しました。

検索中に、モバイルデバイスからRTMPをブロードキャストするためのLarix Broadcasterアプリケーションが最も信頼性が高く、安定していることがわかりました。別の小さな仮想サーバーで、nginx + rtmpをレイズして、さらにCDNにストリーミングしました。

このスキームは最も安いです。既製のストリーミングプラットフォームのサービスを使用するよりもはるかに安価です。プレミアム\ゴールド\スーパーアカウントを1か月\年間購入する必要はありません。使用したトラフィックに対してのみ支払います。これは私たちの状況では完璧なソリューションでした。
