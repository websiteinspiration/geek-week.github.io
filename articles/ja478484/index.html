パートI. RaspberryへのRapid SCADAのインストール



1. Rapid Scada Webサイトのフォームに記入して、配布パッケージを受け取り、Linuxの最新バージョンをダウンロードします。

2.ダウンロードしたファイルを解凍し、「scada」フォルダをデバイスの/ optディレクトリにコピーします。

3. daemonsフォルダから/etc/init.dディレクトリに3つのスクリプトを配置します

。4. 3つのアプリケーションフォルダへのフルアクセスを付与します。

sudo chmod -R ugo+rwx /opt/scada/ScadaWeb/config
sudo chmod -R ugo+rwx /opt/scada/ScadaWeb/log
sudo chmod -R ugo+rwx /opt/scada/ScadaWeb/storage

⠀5。スクリプトを実行可能にします。

sudo chmod +x /opt/scada/make_executable.sh
sudo /opt/scada/make_executable.sh

⠀6。リポジトリを追加：

sudo apt install apt-transport-https dirmngr gnupg ca-certificates
sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys 3FA7E0328081BFF6A14DA29AA6A19B38D3D831EF
echo "deb https://download.mono-project.com/repo/debian stable-stretch main" | sudo tee /etc/apt/sources.list.d/mono-official-stable.list
sudo apt update

⠀7。Mono .NET Frameworkをインストールします。

sudo apt-get install mono-complete

⠀8。Apache HTTPサーバーをインストールします。

sudo apt-get install apache2

⠀9。追加モジュールをインストールします。

sudo apt-get install libapache2-mod-mono mono-apache-server4

⠀10。Webアプリケーションへのリンクを作成します。

sudo ln -s /opt/scada/ScadaWeb /var/www/html/scada

⠀11。「apache」フォルダにダウンロードされたアーカイブから、scada.confファイルを/ etc / apache2 / sites-availableディレクトリにコピーします

sudo a2ensite scada.conf

⠀12。このパスsudo nano /etc/apache2/apache2.confをたどり、ファイルの最後に以下を追加します。

<Directory /var/www/html/scada/>
  <FilesMatch "\.(xml|log|bak)$">
    Require all denied
  </FilesMatch>
</Directory>

⠀13。スクリプトを実行します。

sudo /opt/scada/svc_install.sh

⠀14。ラズベリーを再起動します。

sudo reboot

⠀15。ウェブサイトを開きます：

http://IP- /scada

⠀16。開いたウィンドウで、ログイン「admin」とパスワード「12345」を入力します。

それでは始めましょう：

調査結果


強力な組み込み産業用コンピュータの登場により、プログラマブルロジックコントローラーの機能を拡張および補完できます。それらに同様のSCADAシステムをインストールすると、小規模な製造または技術プロセスのタスクをカバーできます。ユーザー数が多い、またはセキュリティ要件が高まる大規模なタスクの場合、ほとんどの場合、本格的なサーバー、オートメーションキャビネット、使い慣れたPLCをインストールする必要があります。ただし、小規模な工業用建物、ボイラーハウス、ポンプ室、スマートホームなどの中小規模のオートメーションのポイントでは、このようなソリューションが適切と思われます。私たちの計算によると、このようなデバイスは、最大500のデータ入出力ポイントのタスクに適しています。

さまざまなグラフィックエディタでの描画の経験があり、自分でグラフィックディスプレイの要素を作成する必要があるという事実に混乱しない場合は、Rapid SCADA for Raspberryのオプションが非常に最適です。既製のソリューションとしての機能はオープンソースであるため多少制限されますが、小規模な工業用建物のタスクをカバーすることもできます。したがって、視覚化テンプレートを自分で準備する場合は、このソリューションを使用して、プロジェクトの一部ではなくすべてを統合することが可能です。

したがって、Raspberryでのこのようなソリューションの有用性と、プロジェクトがLinuxのオープンソースSCADAシステムでどれだけ置き換えられるかを理解するために、合理的な質問が生じます。どのSCADAシステムを最も頻繁に使用しますか？
