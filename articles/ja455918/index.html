admin-ajax.phpの重大な脆弱性

先週、私は非常に不愉快な事実に遭遇しました。私のサイトを訪問したところ、私が知らないリソースにリダイレクトされていることがわかりました。アンチウイルス博士はこれを非常に誓います Web

サイトはWordPressの最新バージョン5.1で実行されます。

エンジン、プラグイン、テーマのすべての送信アップデートは予定どおりにインストールされます。公式リポジトリのプラグインのみ、テーマも。

サイトのバックアップはすぐにダウンロードされ、ウイルス対策ソフトウェア（Dr. Web、Kaspersky、AI-BOLIT）によってスキャンされましたが、結果は得られず、すべて問題ありません。

テーマファイルと一部のプラグインは手動でチェックされましたが、やはり結果はありませんでした。
siteurlパラメータのwp_optionsテーブルのデータベースダンプを確認すると、他の誰かのURLが隠されていました。実際、それへのリダイレクトがありました。

これは、原則に従って発生しました。ページをロードすると、「siteurl」パラメーターがすべて

<script type='text/javascript' src=' URL  '></script>

同時に、次のスクリプトが読み込まれました。

他人のURLがデータベースにどのように侵入したかは謎のままです。 URLを正しいURLに変更すると、すべてが再び機能しましたが、翌日、確認すると、Dr。 Webはリダイレクトされたページを誓います。このパラメーターは、データベースで再度変更されました。

その後、サイトへの新しいアクセスログとエラーログがダウンロードされました。エラーはありませんでしたが、サイトへの非常に興味深いリクエストがアクセスログで見つかりました。

/wp-admin/admin-ajax.php?action=fs_set_db_option&option_name=siteurl&option_value= URL

設定をもう一度修正してすべてが機能することを確認した後、サイトでこの要求を繰り返し実行しようとしましたが、何も機能しませんでした。データベースの設定は変更されていません。

サイトは登録のために開かれており、ユーザーは「サブスクライバー」の役割を取得し、管理部分へのアクセスは完全に閉じられていることに注意してください。

新しいユーザーを登録してログインしようとしましたが、このサイトへの要求が機能した後、データベースの設定が変更されました。

その結果、サブスクライバーの役割があり、管理パネルへのアクセスが閉じている場合でも、サイトでユーザー登録が開いていると、この要求は機能することがわかります。

すべてのプラグインをオフにしてデフォルトのテーマを設定した後、別のサイトで確認されました-結果は同じです。

登録をオフにして不審なユーザーを削除する方法を除いて、戦う方法-私はまだ解決策を見つけていません。

PS：リクエストをグーグル、感染したサイトが見つかった、注意してください。
