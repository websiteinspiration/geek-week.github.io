疑似ランダムシーケンスの並べ替えによる調査と変換

C＃およびqbasicのアルゴリズムとExcel互換のテーブルが作成され、疑似ランダムシーケンスのランダム性を調べ、非ランダムシーケンスまたは低電力シーケンスを決定できることが証明されました。

グラフィカルシェル：5万を超える研究に対応するExcelテーブル。 2種類の要素：
1.数列の研究;
2.一連の数字0と1の研究。




数列研究：表は、例えば、より少ない/多い、偶数/奇数などのバイナリ機能を定義します。
Excel互換テーブルのグラフィカルシェルは、数式を使用します。


行の一致数は、数式N = log（1-C）/ log（1-P）で計算されます
。Nはステップ、Pは確率、Cは確率の信頼性です。

分布ステップ番号：
C = P = 0.5; N = 1 = log0.5 / log0.5 = log（1-1 / 2）/ log（1-1 / 2）= 1
at C = 0.25; P = 0.5; N = 2 = log0.75 / log0.5 = log（1-1 / 4）/ log（1-1 / 2）= 2など

ランダムなものの繰り返しを最小限に抑えるための最大10 ^ 6の整数。

順列は、2つの列QとRをソートすることによって行われます。
列Qが先行し、列Rがスレーブです。

結果：列Rの順列と新しいシーケンス。

統合されたPRNGに基づくPRNG研究は、アルゴリズムの正常性を示しています。
500セルの順列の前：500セル

の順列の後：

チェックは、小/大および偶数/奇数の兆候を比較して、良好な分布を示しています。

この表では、標準のPRNGを使用せずに、三角関数の小数桁を使用して三角関数のPRNGを調べています。
'rndsin.bas
OPEN "rndsin.txt" FOR OUTPUT AS #1
c = 0: a = SIN(TIMER) * 100 + 200
PRINT #1, "a= ", a

FOR k = 1 TO 10 ^ 3 + a * 10 ^ 3: NEXT

FOR i = 1 TO 100
    FOR j = 1 TO a
        x = SIN(TIMER) * 1000 + 2000
        b = COS(x): c = c + b
        LOCATE 1, 1: PRINT j
    NEXT

    d = (ABS(c)) - INT(ABS(c))
    PRINT #1, d
    FOR k = 1 TO 10000 + a * b * c * 10 ^ 2: NEXT
NEXT

500セルの順列の前：不十分

明らかに、分布が貧弱であり、値の頻度とばらつきが明らかになり、符号が小さい/大きい、偶数/奇数であることがわかります。

500セルの再配置後：通常の


目的：組み込みのPRNGを除外します。

並べ替え方法：元のシーケンスが並べ替えられます。反転または同じ方法で反転された同じシーケンスは、並べ替えのランダムとして受け入れられます。

たとえば、Excelでは、シーケンスの列の2つのコピーが距離を置いて作成され、行の先頭行1 ... 55000が左側の1つの列に構築され、2つの列が最大から最小に並べ替えられ、元のデータを反転させます。

次に、シーケンスの2つの列が並べてマップされ、並べ替えられます。先頭の列は逆の列で、スレーブの列は最初の列です。

500セルの順列前：不十分500セル

の順列後：正常


結果：組み込みのPRNGなしでシーケンスが正常になりました。

結論：真のランダム性は人にとって不自然であり、人やコンピュータがランダムシーケンスとして受け入れる低電力または偽のシーケンスを合成することが可能です。

シーケンスはプログラミング言語とExcel互換テーブルで実際に合成できます。

ランダム性を克服する問題は、グラフを含むExcelスプレッドシートでランダム性を正常または偽として認識することで解決されます。
