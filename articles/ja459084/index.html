Google Home Hubについて、または130ユーロでフォトフレームを購入した方法

こんにちは、ハブル。

「スマートホーム」のコンセプトと別の画面からの使用は、私にとって長い間興味深く思えてきました。かつて私はOpenHABとRaspberry Piでプロジェクトを開始しました-非常に高度なことですが、多数の構成を構成する必要性はすぐに飽きてきました。そして、偶然にも、Google Nest Hub（旧称Google Home Hub）が販売されているのを見て、これがまさに必要なものだと判断しました-タッチスクリーンによるカスタマイズと音声制御の可能性を備えた、Googleの1つのデバイスですべてを実現しました。



今では、スマートホーム用の「コントロールセンター」を1つにすることが可能になると思っていました。つまり、私は笑った。しかし、それでも、それがどのように機能するか見てみましょう。

Google


Googleのデバイスが「スマートスピーカー」から進化したことは明らかです。そのため、マーケターが考えたように、ユーザーは「ねgoogle google、寝室の照明をつけて」などのフレーズを非常に喜んで言うでしょう。私の意見では、スイッチを押す方が簡単ですが、まあ、マーケティング担当者はよく知っています。しかし、私たちは画面を備えたデバイスを持っており、すべてがはるかに優れているはずです。

そして、それはすべてです。メイン画面に表示するには、2つのオプションしかありません。これは、フォトフレームまたはフルスクリーンの時計です。最初のケースでは、現在の天気が画面の下隅に細かく細かく表示されます。時計の場合、天候すら推定できません。すべてが、他に何も愚かではありません。ウィジェット、カスタムインターフェイス要素？いいえ、聞いていません。 「スマートウェザーステーション」Netatmoを200〜300ドルで購入しても、この画面に部屋の温度を表示することはできません（「天気を表示して」というコマンドを言うと天気が表示されますが、ウィジェットとして画面に常に表示することはできません。 ）実際には、この画面では何も設定できません。

スマートホーム管理はどうですか？上から下にスワイプしても、メニューでは引き続きデバイスを管理できます。 Googleのマーケティング担当者によると、これはおそらく便利です。



しかし、一般的に、元の計画によれば、すべてはおそらく音声によって制御されるべきです。Google Homeには興味深い機能があります。たとえば、IPカメラを備えたドアベルとの統合（カメラからのビューは通話中に画面に表示されます）やNestサーモスタットとの統合などですが、これらすべてのカスタマイズ性は、どういうわけか多くのことを望んでいません。エコシステム全体が「ブロンド」向けに明確に設計されています。「一般」という言葉から自分用にカスタマイズすることは不可能です。公平ではありますが、Google Homeでサポートされているデバイスのリストは非常に多く、音声コントロールのみに制限すると、原則として機能します。

ちなみに消費電力について。Google Nest Hubは主電源から約2.5Wを消費しますが、原則としてそれほど多くはなく、デバイスをオンにしておくことに問題はありません。写真のパワーメーターのスクリーンショット。



API


次に知って興味深いのは、デバイスを接続するためのなんらかのAPIの存在でした。悲しいかな、簡単なことは何も見つかりませんでした。Google Homeでサポートされているデバイスのリストは長いですが、一部のデバイスまたはAPIを追加しても機能しませんでした。多分私
はひどく見ていました、誰かが知っているなら、コメントに書いてください。

もちろん、回避策があります。たとえば、Google Homeの音声コマンドでLEDを点滅させるには、次のことを行う必要があります。

  • IFTTTオンラインサービスに登録して、Googleアシスタントのソースリストに追加します。
  • サーバーを上げ（外部から見えるようにします）、IFTTTのトリガーとしてWebhookとサーバーパラメーターを指定します。
  • サーバーに必要なAPIのハンドラーを追加し、そこでLEDをオン/オフにします。

一般に、電球をつけるのにどれだけのプログラマーとソフトウェアサービスを使用する必要があるかという冗談は、残念ながら現実になりましたが、かつてはばかげていたように思えました。電球を使って実験を繰り返したい場合は、https：//www.instructables.com/id/Google-Home-Raspberry-Pi-Power-Strip/で手順を確認できます。しかし、私の意見では、これは完全に巨大で意味のないオーバーヘッドです。GoogleHomeの設定でチームのローカルサーバーをすぐに指定できないのはなぜか、明確ではありません。お使いのデバイスのGoogle Homeですでにサポートされているデバイスの1つをエミュレートする場合、おそらく回避策があります。誰かが試した場合は、コメントに書き込んでください。

ちなみに、興味深いのは、Google Hubがネットワーク上に表示され、一部のパラメーターはブラウザーで直接表示できることです。たとえば、デバイスパラメーターはリンクに表示されます。http://192.168.1.102:8008/ssdp/device-desc.xml。詳細はこちらまたはこちらで読むことができますが、これは純粋に理論上の関心事ですが、そこに役立つコマンドはありません。SSHアクセス、カスタムファームウェア、または公式API？いいえ、Googleはこれについて聞いていません。

別の可能性は、かなり限定的ではありますが、すでに上で述べたPyChromecastライブラリの使用です。これを使用すると、Google Nest Hubに独自の画像、音声、動画を表示できます。

結論


すべての機能を理解した後、最初に考えたのは、Googleテクノロジーのこの奇跡を店に戻すことでした-音声コントロールは私にとってそれほど重要ではなく、他のすべては実際には思い通りに機能しません。しかし、それをテーブルの上に置いて、フォトフレームがうまく機能するように決めました。たとえば、「タイマーを10分に設定して」などの音声コマンドも、ソーセージを調理する場合などに役立つことがあります。出かける前に便利な「今日の雨の可能性は？」などの簡単な質問にも答えられます。一般的に、ワイヤレススピーカーの改良版として、おそらく悪くはありません。しかし、次のバージョンでは改善される可能性はありますが、それでも「スマートホームハブ」とは言えません。

一方、インターネットでは、Google Nest Hubに関する熱狂的なレビューがたくさん見つかるので、誰かに役立つかもしれません。おそらく、Appleエコシステムと同様に、嫌いな人もいれば、畏敬の念を抱いている人もいるでしょう。

そしてもちろん、多分私は考慮に入れなかったか、何かを逃しました。Nest Hubを使用する他の方法がある場合は、コメントに書き込んでください。
