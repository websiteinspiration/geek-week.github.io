OpenGLのWBOIT：ソートなしの透過性

2013年にJCGTで説明されている手法（リンク）の「加重混合順序非依存透明度」（以下WBOIT）について説明します。

複数の透明なオブジェクトが画面に表示される場合、ピクセルの色はどちらが観察者に近いかに依存します。この場合のよく知られた色混合式は次のとおりです。

C=Cnearα+Cfar(1−α)(1)


彼女にとって、フラグメントの順序は重要です。ニアフラグメントの色とその不透明度はC nearとαで示され、その後ろにあるすべてのフラグメントの結果の色はC farと指定されます。不透明度は0から1までの値を取るプロパティで、0はオブジェクトが非常に透明で表示されないことを意味し、1は非常に不透明なので背後に何も表示されないことを意味します。

この式を使用するには、まずフラグメントを深度でソートする必要があります。これにどれほどの頭痛が伴うか想像してみてください！通常、並べ替えはフレームごとに行う必要があります。オブジェクトを並べ替える場合、複雑な形状の一部のオブジェクトを断片にカットし、カットパーツの深さで並べ替える必要があります（特に、交差するサーフェスの場合は、必ず行う必要があります）。フラグメントを並べ替えると、シェーダーで並べ替えが行われます。このアプローチは「順序に依存しない透明度」（OIT）と呼ばれ、ビデオカードのメモリに保存されているリンクリストを使用します。このリストに割り当てる必要があるメモリの量を予測することは、ほとんど非現実的です。また、十分なメモリがない場合、アーティファクトが画面に表示されます。

ステージ上に配置される半透明のオブジェクトの数と、それらの相対的な位置を制御できる人にとっては幸運です。しかし、CADを実行すると、ユーザーが望むだけの透明なオブジェクトができ、それらはランダムに配置されます。

これで、一部の人々の生活を簡素化したいという願望を理解し、並べ替えを必要としない色を混合するための式を考え出しました。そのような公式は、私が最初に言及した記事にあります。そこにはいくつかの公式がありますが、著者の意見（そして私の意見でも）で最も良いのはこれです。

C=∑i=1nCiαi∑i=1nαi(1−∏i=1n(1−αi))+C0∏i=1n(1−αi)(2)




スクリーンショットには、深さ4層にある半透明の三角形のグループがあります。左側では、WBOITテクニックを使用してレンダリングされています。右側は、式（1）を使用して取得された画像で、フラグメントの配置の順序を考慮した、古典的な色のブレンドです。次に、それをCODB（クラシック次数依存ブレンディング）と呼びます。

透明なオブジェクトのレンダリングを開始する前に、すべての不透明なオブジェクトをレンダリングする必要があります。その後、透明オブジェクトは深度テストでレンダリングされますが、深度バッファーへの書き込みは行われません（これは次のように行われます：glEnable(GL_DEPTH_TEST); glDepthMask(GL_FALSE);）つまり、これはいくつかの画面座標（x、y）があるポイントで発生します。不透明度よりも近い透明フラグメントは、既に描画された透明フラグメントと比較して深度にどのように配置されているかに関係なく、深度テストに合格しています。不透明、深度テストに合格しないため、破棄されます。式（2）の

C 0は不透明なフラグメントの色で、その上に透明なフラグメントが描画され、その中にn個のピース​​があり、インデックス1〜nで示されます。 C iが、i番目の透明断片の色であるα iはその不透明度です。

よく見ると、式（2）は式（1）に少し似ています。これが近くの C だと想像すると、C 0は Cであり遠い、そしてあるα、これは第1式、一から一であろう。実際、これは透明なフラグメントの色の加重平均である（質量の中心が同じ式で力学決定される）は、最も近い断片Cの色のために通過する周辺。 C 0は、この加重平均を計算したすべてのフラグメントの背後にある不透明なフラグメントの色で、C farを通過します。つまり、すべての透明なフラグメントを1つの「平均化された」フラグメントに置き換え、色を混合するための標準的な公式である公式（1）を適用しました。元の記事の著者が私たちに提供するこのαの狡猾な式は何ですか？

深度ベースのWBOIT


深度によるソートの欠如を何らかの形で補うために、記事の著者は式（2）に深度を追加するためのいくつかのオプションを考え出しました。これにより、実装がより困難になり、結果が予測不可能になり、特定の3次元シーンの特性に依存するようになります。私はこのトピックを掘り下げなかったので、気にしない人-私は記事を読むことを提案します。

WBOITは、古典的なソートの透過性ではできないことができる場合があると主張されています。たとえば、煙を2つの粒子のみを使用して粒子のシステムとして描画します-濃い煙と薄い煙を使用します。ある粒子が別の粒子を通過するとき、分類を伴う古典的な色混合は醜い結果をもたらします-光からの煙の色は急激に暗くなります。この記事によると、深度に敏感なWBOITはスムーズな移行を可能にし、より信頼できるように見えます。毛皮と髪を細いチューブの形でモデリングする場合も同様です。

ApplyTextures（）関数では、ウィンドウ全体に長方形を描くだけです。フラグメントシェーダーは、現在の画面座標をテクスチャ座標として、現在のサンプル番号（gl_SampleID）をマルチサンプルテクスチャのサンプル番号として使用して、作成したすべてのテクスチャのデータを要求します。シェーダーでgl_SampleID変数を使用すると、フラグメントシェーダーがサンプルごとに1回呼び出されると、モードが自動的にオンになります（通常の条件では、ピクセル全体に対して1回呼び出され、プリミティブ内にあるすべてのサンプルに結果が書き込まれます）。

フラグメントシェーダー：

#version 450 core
out vec4 outColor;

layout (location = 0) uniform  sampler2DMS colorTextureNT;
layout (location = 1) uniform  sampler2DMS colorTexture;
layout (location = 2) uniform  sampler2DMS alphaTexture;

void main() {
    ivec2 upos = ivec2(gl_FragCoord.xy);
    vec4 cc = texelFetch(colorTexture, upos, gl_SampleID);
    vec3 sumOfColors = cc.rgb;
    float sumOfWeights = cc.a;

    vec3  colorNT = texelFetch(colorTextureNT, upos, gl_SampleID).rgb;

    if (sumOfWeights == 0)
    { outColor = vec4(colorNT, 1.0); return; }

    float alpha = 1 - texelFetch(alphaTexture, upos, gl_SampleID).r;
    colorNT = sumOfColors / sumOfWeights * alpha +
              colorNT * (1 - alpha);
    outColor = vec4(colorNT, 1.0);
}
