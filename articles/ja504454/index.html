Node.jsのベストプラクティス



Node.jsベストプラクティス リポジトリには133人のプログラマーが貢献しており、GitHubには4万を超えるスターがいます。ほぼ毎日更新されます。これは、誰もがNode.jsの世界で起こっていることに遅れずについていくことができると同時に、最良の実用的な開発方法を学ぶことができる素晴らしいリソースです。

以下に、その対象となる開発領域の一部を示します。

  • エラー処理。
  • コードスタイル
  • 高品質のコードをテストおよび保証します。
  • 生産での出力のためのプロジェクトの準備。

HTML5ボイラープレート



HTML5ボイラープレートは、高速で信頼性が高く、柔軟なWebアプリケーションおよびサイトを作成するためのプロフェッショナルなフロントエンドテンプレートです。

このプロジェクトは、長年にわたる反復的な開発の結果であり、Webプログラマーコミュニティから大量の知識を蓄積してきました。彼はそれを使用する人に特定の開発哲学を課さず、特定のフレームワークの使用を規定していません。このプロジェクトを使用すると、誰もが自分に合ったコードを設計できます。

テンプレートに表示される機能は次のとおりです。

  • Normalize.cssのサポート。
  • CDNによるjQueryサポート。
  • Apacheサーバーの設定。
  • 便利なヘルパーCSSクラス。
  • 標準の印刷スタイル、パフォーマンス最適化メカニズムなど。

このテンプレートのどの機能を使用する予定であるか、およびどのように使用する予定であるかに応じて、特定の資料をプロジェクトにコピーする必要がある場合があります。このような資料を使用することは、開発者が事前に構成された基本テンプレートを持っていることを意味します。これにより、Webプロジェクトの開発がスピードアップします。

現実の世界



RealWorld リポジトリのおかげで、不必要な困難なしに、新しいフレームワークの基本概念とイデオロギーを習得できます。

通常、何か新しいことを学ぶとき、プログラマーはドキュメントを読んで、CodePenのような何かで練習から引き裂かれた例を実行し、これらの例を解析し、それを収集し、コマンドラインツールをインストールします...一般的に、それは多くの時間と労力を費やします。さらに、プログラマーが学習しようとしているツールの本質を十分に理解していなかったという事実により、状況は複雑になる可能性があります。

このリポジトリを使用すると、クライアント開発（React、Angular 2、その他多数）でサポートされている任意のテクノロジ、（Node.jsやDjangoなどの）サーバープラットフォームを選択し、それらを相互に統合して、実際のサンプルアプリケーションを見ることができます。

サンプルの実装では、Webプロジェクトの開発に必要なテクノロジーのスタック全体を使用するため、それらのパーツを完全に組み合わせることができませんが、さまざまなテクノロジーを使用しても、同等の機能と同様のUX仕様を示します。

このリポジトリからの一般的なテクノロジー統合の例をいくつか示します。

  • 角度+ ngrx + nx
  • ClojureScript +リフレーム
  • React / MobX
  • 行く+ジン
  • NestJS + TypeORM / Prisma

これは、RealWorldリポジトリにあるもののほんの一部です。あなたはそれを探求することに興味があると思います。

ちなみに、ウェブ開発用に設計された例だけではありません。NativeScriptに基づくものは、モバイルデバイス用に設計されたプロジェクトを作成する人がここで見つけることができます。

リポジトリは、React Native、Jasonette、Swift、C＃およびXamarin、Kotlin / Android、Onymos、Quasarフレームワーク、Swift PerfectおよびFlutterに焦点を当てたサンプルの準備に取り組んでいます。

ストーリーブック



Storybook リポジトリには、ユーザーインターフェースコンポーネントを作成するために設計された、同じ名前の開発環境のコードがあります。この環境では、コンポーネントライブラリの操作、さまざまな状態のコンポーネントの調査、インタラクティブモードでのコンポーネントの開発とテストを行うことができます。

Storybook環境はアプリケーションの外部で実行されます。これにより、インターフェイスコンポーネントを分離して開発できるため、コンポーネントの再利用性が向上し、テスト性が向上し、開発速度が向上します。特定のアプリケーションに固有の依存関係を気にすることなく、Storybookのコンポーネントを非常に迅速に作成できます。

Storybookには、この環境をすばやく学習するためのコマンドラインツールとコードサンプルが含まれています。

フロントエンドのチェックリスト



フロントエンドチェックリスト リポジトリでは、本番環境でリリースする前にプロジェクトに含める必要があるもの、またはプロジェクトが実際の作業の準備ができていると見なす前にチェックする必要があるものの包括的なリストを見つけることができます。

このリポジトリは、Web開発クライアント側を対象としています。生産性、セキュリティ、SEOなどの問題に対処します。このリポジトリに提示されている要件は、ほとんどのプロジェクトで満たされているはずです。ただし、これらの要件の一部は、特定の条件下では満たされない場合があります。いくつかは他より重要ではないかもしれません。推奨事項には、追加のドキュメントを読んだり、さまざまなツールを使用したり、いくつかの困難な問題の説明を含むビデオを見たりすることが含まれます。
