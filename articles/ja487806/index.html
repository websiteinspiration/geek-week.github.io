小さなDeno APIを作成する

この投稿では、Denoを使用して小さなAPIを作成するプロセスについて説明します。Denoは、Node.jsの作成者Ryan Dahlが開発した最新のJavascriptおよびTypescriptランチャーです。



私たちの目標：

  • ユーザーデータを処理するAPIを開発する
  • GET、POST、PUT、およびDELETEメソッドを使用する機能を提供する
  • ローカルJSONファイルにユーザーデータを保存して更新する
  • フレームワークを使用して開発をスピードアップする

インストールする必要があるのはDenoだけです。DenoはそのままでTypescriptをサポートしています。この例では、Denoバージョン0.22を使用しましたが、このコードは将来のバージョンでは機能しない可能性があります。
インストールされているDenoのバージョンは、ターミナルのdeno versionコマンドで確認できます。

フレームワークの選択


Node.jsには多くの優れたフレームワークがあります。最も人気のあるものはExpressです。Express'a- Koaの最新バージョンもあります。残念ながら、DenoはNode.jsライブラリをサポートしておらず、選択肢ははるかに少なくなっていますが、Koa- Oakに基づくDenoのフレームワークがあります。小さなプロジェクトで使用します。Koaを使用したことがない場合でも、心配はいりません。Expressによく似ています。

アプリケーションのエントリポイントの作成


index.ts

import { Application } from "https://deno.land/x/oak/mod.ts";
import { APP_HOST, APP_PORT } from "./config.ts";
import router from "./routing.ts";
import notFound from "./handlers/notFound.ts";
import errorMiddleware from "./middlewares/error.ts";

const app = new Application();

app.use(errorMiddleware);
app.use(router.routes());
app.use(router.allowedMethods());
app.use(notFound);

console.log(`Listening on ${APP_PORT}...`);

await app.listen(`${APP_HOST}:${APP_PORT}`);

最初の行では、主要なDenoチップの1つを使用します-モジュールをインターネットから直接インポートします。その後、異常なことはありません。アプリケーションを作成し、ミドルウェア、ルートを追加し、最後にサーバーを起動します。すべてはExpressまたはKoaを使用する場合と同じです。

ルートファイルの作成


routing.ts

import { Router } from "https://deno.land/x/oak/mod.ts";

import getUsers from "./handlers/getUsers.ts";
import getUserDetails from "./handlers/getUserDetails.ts";
import createUser from "./handlers/createUser.ts";
import updateUser from "./handlers/updateUser.ts";
import deleteUser from "./handlers/deleteUser.ts";

const router = new Router();

router
  .get("/users", getUsers)
  .get("/users/:id", getUserDetails)
  .post("/users", createUser)
  .put("/users/:id", updateUser)
  .delete("/users/:id", deleteUser);

export default router;

繰り返しますが、異常なことは何もありません。ルーターを作成し、それにいくつかのルートを追加しました。Expressアプリケーションからコードをコピーしたように見えますよね？

ルートのイベント処理


ハンドラー/ getUsers.ts

import { getUsers } from "../services/users.ts";

export default async ({ response }) => {
  response.body = await getUsers();
};

すべてのユーザーを返します。コアを使用したことがない場合は、説明します。応答オブジェクトはExpressのresに類似しています。Express のresオブジェクトには、応答の送信に使用されるjsonやsendのようないくつかのメソッドがあります。OakとKoaでは、返したい値をresponse.bodyプロパティに設定する必要があります。

ハンドラー/ getUserDetails.ts

import { getUser } from "../services/users.ts";

export default async ({ params, response }) => {
  const userId = params.id;

  if (!userId) {
    response.status = 400;
    response.body = { msg: "Invalid user id" };
    return;
  }

  const foundUser = await getUser(userId);
  if (!foundUser) {
    response.status = 404;
    response.body = { msg: `User with ID ${userId} not found` };
    return;
  }

  response.body = foundUser;
};

ここでもすべてが簡単です。ハンドラーは、目的のIDを持つユーザーを返します。

ハンドラー/ createUser.ts

import { createUser } from "../services/users.ts";

export default async ({ request, response }) => {
  if (!request.hasBody) {
    response.status = 400;
    response.body = { msg: "Invalid user data" };
    return;
  }

  const {
    value: { name, role, jiraAdmin }
  } = await request.body();

  if (!name || !role) {
    response.status = 422;
    response.body = { msg: "Incorrect user data. Name and role are required" };
    return;
  }

  const userId = await createUser({ name, role, jiraAdmin });

  response.body = { msg: "User created", userId };
};

このハンドラーは、ユーザーの作成を担当します。

ハンドラー/ updateUser.ts

import { updateUser } from "../services/users.ts";

export default async ({ params, request, response }) => {
  const userId = params.id;

  if (!userId) {
    response.status = 400;
    response.body = { msg: "Invalid user id" };
    return;
  }

  if (!request.hasBody) {
    response.status = 400;
    response.body = { msg: "Invalid user data" };
    return;
  }

  const {
    value: { name, role, jiraAdmin }
  } = await request.body();

  await updateUser(userId, { name, role, jiraAdmin });

  response.body = { msg: "User updated" };
};

ハンドラーは、指定されたIDのユーザーが存在するかどうかを確認し、ユーザーデータを更新します。

ハンドラー/deleteUser.ts

import { deleteUser, getUser } from "../services/users.ts";

export default async ({ params, response }) => {
  const userId = params.id;

  if (!userId) {
    response.status = 400;
    response.body = { msg: "Invalid user id" };
    return;
  }

  const foundUser = await getUser(userId);
  if (!foundUser) {
    response.status = 404;
    response.body = { msg: `User with ID ${userId} not found` };
    return;
  }

  await deleteUser(userId);
  response.body = { msg: "User deleted" };

ユーザーの削除を担当します。

サービスの追加


ユーザーデータを処理するサービスを作成する前に、2つの小さな補助サービスを作成する必要があります。

services / createId.ts

import { v4 as uuid } from "https://deno.land/std/uuid/mod.ts";

export default () => uuid.generate();

新しいユーザーはそれぞれ一意のIDを受け取ります。Deno標準ライブラリのuuidモジュールを使用して、乱数を生成します。

サービス/db.ts

import { DB_PATH } from "../config.ts";
import { User } from "../models/user.ts";

export const fetchData = async (): Promise<User[]> => {
  const data = await Deno.readFile(DB_PATH);

  const decoder = new TextDecoder();
  const decodedData = decoder.decode(data);

  return JSON.parse(decodedData);
};

export const persistData = async (data): Promise<void> => {
  const encoder = new TextEncoder();
  await Deno.writeFile(DB_PATH, encoder.encode(JSON.stringify(data)));
};

このサービスは、ユーザーデータが保存されるJSONファイルとのやり取りに役立ちます。
すべてのユーザーを取得するには、ファイルの内容を読み取ります。readFile関数は、Uint8Array型のオブジェクトを返します。これは、JSONファイルに入力する前にString型に変換する必要があります。

そして最後に、ユーザーデータを操作するためのメインサービスです。

services / users.ts

import { fetchData, persistData } from "./db.ts";
import { User } from "../models/user.ts";
import createId from "../services/createId.ts";

type UserData = Pick<User, "name" | "role" | "jiraAdmin">;

export const getUsers = async (): Promise<User[]> => {
  const users = await fetchData();

  // sort by name
  return users.sort((a, b) => a.name.localeCompare(b.name));
};

export const getUser = async (userId: string): Promise<User | undefined> => {
  const users = await fetchData();

  return users.find(({ id }) => id === userId);
};

export const createUser = async (userData: UserData): Promise<string> => {
  const users = await fetchData();

  const newUser: User = {
    id: createId(),
    name: String(userData.name),
    role: String(userData.role),
    jiraAdmin: "jiraAdmin" in userData ? Boolean(userData.jiraAdmin) : false,
    added: new Date()
  };

  await persistData([...users, newUser]);

  return newUser.id;
};

export const updateUser = async (
  userId: string,
  userData: UserData
): Promise<void> => {
  const user = await getUser(userId);

  if (!user) {
    throw new Error("User not found");
  }

  const updatedUser = {
    ...user,
    name: userData.name !== undefined ? String(userData.name) : user.name,
    role: userData.role !== undefined ? String(userData.role) : user.role,
    jiraAdmin:
      userData.jiraAdmin !== undefined
        ? Boolean(userData.jiraAdmin)
        : user.jiraAdmin
  };

  const users = await fetchData();
  const filteredUsers = users.filter(user => user.id !== userId);

  persistData([...filteredUsers, updatedUser]);
};

export const deleteUser = async (userId: string): Promise<void> => {
  const users = await getUsers();
  const filteredUsers = users.filter(user => user.id !== userId);

  persistData(filteredUsers);
};

ここにはたくさんのコードがありますが、これは純粋なTypescriptです。

エラー処理


ユーザーデータの処理中にサービスエラーが発生した場合に何が起きるのでしょうか？プログラム全体がクラッシュする可能性があります。このシナリオを回避するには、各ハンドラーでtry / catch構文を使用できます。しかし、より洗練されたソリューションがあります-各ルートの前にミドルウェアを追加し、発生する可能性のある予期しないエラーを防止します。

ミドルウェア/error.ts

export default async ({ response }, next) => {
  try {
    await next();
  } catch (err) {
    response.status = 500;
    response.body = { msg: err.message };
  }
};

要約する


どのツールを使用しましたか？

  1. ファイルの読み取り/書き込み用のDenoグローバルオブジェクト
  2. 一意のIDを作成するためのDeno標準ライブラリのuuid
  3. oak -Koaに触発されたNode.jsのサードパーティフレームワーク
  4. JavaScriptに含まれる純粋なTypescript、TextEncodeまたはJSONオブジェクト

Node.jsとの違いは何ですか？


  • Typescriptやts-nodeのような他のツール用のコンパイラをインストールして設定する必要はありません。deno index.tsコマンドでプログラムを実行するだけです。
  • 事前インストールの必要なしに、すべてのサードパーティモジュールをコードに直接含める
  • package.jsonとpackage-lock.jsonの欠如
  • プログラムのルートディレクトリにnode_modulesがない。ダウンロードされたすべてのファイルはグローバルキャッシュにあります
