ラズベリーのホームシアター

たまたま私は映画をオンラインで見る問題を単純化することにしました。標準のスマートTVオプションは私には適していません。ラップトップに接続するために常につながっているHDMIケーブルは私を困らせました。Raspberry Pi 4の購入で見た私の問題の解決策は、TVの後ろに注意深く隠すことができます。

私はアリで既製のキットを注文し、待っている間に私は方法と何をすべきかを読みました。私に適した2つのソリューションがありました。

  • ラズベリーをミニコンピューターに変えて、通常はラップトップで行うように、ブラウザーからストリーミングビデオを見る
  • Kodiプレーヤーをインストールし、ラズベリーを一種のセットトップボックスとして使用します。

実装中に、私は両方のオプションを組み合わせることにしました。Kodiプレーヤーを自動実行するように構成し、必要に応じてRaspbianシステムを終了します。そして、主な問題が隠されたのはそのような解決策でしたが、すべてが順調でした。

私がしたように、私は段階的に着手します。

ステップ1：シングルボードコンピューターの適切なバージョンの選択

私の場合、私は売り手が持っていたものの一番上のバージョンを取った：

  • -
  • 5

  • HDMI/microHDMI 1,5

必要に応じて、よりシンプルで安価なキットを入手することもできますが、「予備」をとる方法を決定しました。

ステップ2：オペレーティングシステムをインストールします。

Raspderryの最初のリリースについて詳しく説明し、公式Webサイトの写真にも示しています。

私は少し異なるパスを取り、balenaEtcherユーティリティを使用してRaspbianシステムイメージをSDカードに解凍しました。誰がこの方法を選択するかは、こちらからダウンロードできます。ユーティリティ自体は管理において原始的です。ファームウェア用のUSBフラッシュドライブを挿入し、balenaEtcherを実行し、システムイメージへのパスを指定して、[フラッシュ]をクリックします。



Raspbianイメージは公式サイトからダウンロードできます。

「Raspbian Buster with desktop and Recommended software」のバージョンを使用しました。これは、「すぐに使える」と呼ばれるほぼ完全なソリューションです。内部には、デスクトップ環境とプリインストールされたソフトウェアがあります。このバージョンでは、デフォルトの環境はPIXEL（Pi改良Xwindows環境、軽量）であり、本質的にLXDE環境の改良版です。

最初の起動時に、Raspbianは構成に関する質問（国、言語、タイムゾーンの選択/パスワードの設定/ Wi-Fi接続/最新のアップデートの受信）を行い、再起動を提案します。私たちはそれらについて詳しく説明しません。それらは直感的で、公式ウェブサイトの上のリンクにある写真で説明されています。

ステップ3：セットアップ。

モニターの解像度を調整する必要がある場合があります。これは次の方法で行うことができます：スタート/設定/画面構成。画面レイアウトエディターウィンドウが開きます。ここで、[構成]、[画面]、[HDMI]、[解像度]の順に進み、リストから目的の解像度を選択して、緑色のチェックマークをクリックします。

レイアウト言語を切り替える機能を設定する必要があります。これは次のように行われます：

  1. タスクバーを右クリックして、「パネル要素の追加/削除」を選択します。
  2. パネル設定ウィンドウが開きますので、「追加」をクリックします。
  3. 開いたリストで「キーボードレイアウトハンドラ」を見つけ、「追加」を選択してクリックします。
  4. 前のメニューに戻ります。「キーボードレイアウトハンドラー」という項目が表示されます。「パラメーター」を選択してクリックします。
  5. 「システム設定を保存」のチェックを外します
  6. アクティブになった「キーボードレイアウト」ウィンドウで、「追加」をクリックして必要なレイアウトを選択します。
  7. 少し下に「grp：shift_caps_toggle」と表示されます。これは、レイアウトを変更するためのホットキーの組み合わせです。ここをクリックして、以下のリストから選択することにより、このキーの組み合わせを通常の便利なものに変更できます。
  8. [既存の設定をリセットしない]チェックボックスをオンにします。
  9. 十字をクリックしてキーボードハンドラウィンドウを閉じます。ここでは閉じるボタンは機能せず、パネル設定ウィンドウを閉じます。



私がインストールしなければならなかった追加のソフトウェアのうち、オンボード仮想キーボードは、私の物理的なキーボードを使用することは私の計画ではありません。



これを行うには2つの方法があります。

  • ターミナルに行き、次のように入力します。 
    sudo apt-get install onboard
  • スタート/設定/ソフトウェアの追加/削除を検索でオンボードに入力し、ドロップダウンリストで「シンプルなスクリーンキーボード」を見つけて、この項目の前にチェックを入れ、「適用」をクリックします。



スタート/アクセシビリティ/オンボードでキーボードを探す必要があります。

これでセットアッププロセスは完了です。

ステップ4：Kodi Playerをインストールします。

プレーヤー自体をインストールする前に、GPUとCPU間のメモリ割り当ての設定、使用メモリ領域の拡張、4k 60fpsモードの接続などの操作を行うことは価値があります。これは少なくとも3つの方法で実行できますが、私の考えではこれが最も簡単です。

  • 端末を起動して、次のように入力します。 
    sudo raspi-config
  • 構成設定ウィンドウ（Raspberry Piソフトウェア構成ツール（raspi-config））が開きます。いくつかの項目を含める必要があります：
    7詳細オプション/ A1ファイルシステムの拡張-フラッシュカード全体のファイル領域の使用を拡張
    7詳細オプション/ A3メモリ分割160から640（デフォルトでは64）-これは、GPUにGPUに割り当てられたメモリの量です。最低でも160で十分です。黒い画面がある場合、またはビデオの再生時に速度が低下する場合は、GPU
    7 Advancedオプション/ AA Pi 4 により多くのメモリを割り当てる必要があります。ビデオ出力-4k 60fpsを許可



次に、ラズベリーは再起動を要求します。これは実行する価値があります。その後、ターミナルを再度開き、コマンドを使用してシステムアップデートを取得します（少し前にシステムをインストールした場合に必要です）。

sudo apt-get update
sudo apt-get dist-upgrade

次に、次のコマンドでKodiプレーヤーをインストールします。

sudo apt-get install kodi

このコマンドの後、システムは続行するかどうかを尋ねてきます。もちろん、「y」と答え、インストールが完了するのを待ちます。次に、システムを再起動します。

これらのすべての操作の後、Kodiプレーヤーを開始できます（開始/オーディオおよびビデオ/ Kodi）。

ステップ5：Kodi Autorunを構成します。

ラズベリーをオンにすると、PIXEL環境が起動し、通常のアプリケーションとしてKodiを起動できます。ただし、システムの起動時にKodiを起動する必要があります。必要に応じて、PIXEL環境に入ることができます。これを行うには、ラズベリーの起動時に自動的に起動するようにアプリケーションを構成する必要があります。再びターミナルに戻り、次のように書きます。

sudo nano ~/.config/lxsession/LXDE-pi/autostart

空の自動実行構成ファイルが開きます。次の行を入力してください：

@kodi

保存して閉じるCtrl + x / y / Enter。ここで、ターミナルはそのようなディレクトリがないことを誓い始めるかもしれません。次に、最初にこのディレクトリを作成する必要があり、3つのオプションがあります。

  • デスクトップで何かを構成してみてください。たとえば、壁紙を変更します（デスクトップ/ RMB /デスクトップ設定/画像）。その後、フォルダは自動的に作成されます。
  • ターミナルでmkdir〜/ .config / lxsession / LXDE-piと入力します
  • ファイルマネージャーを開き、Ctrl + Hキーを押して非表示のファイルとフォルダーを表示し、.config / lxsession /に移動して、フォルダーLXDE-piを作成します。

ラズベリーを再起動すると、PIXELの代わりにKodiインターフェイスが表示されます。さて、Kodiから抜け出そうとすると、PIXEL環境に入る必要があり、ここに主な問題があり、しばらく時間がかかりました。 PIXELの代わりに、Openbox環境の黒い画面が表示されますが、これはまったく予期したものではありません。この問題の解決策は簡単です。自動実行ファイルに変更を加えたら、1つ前のステップに戻りましょう。 1行で@kodiは不十分です。PIXEL環境を起動する必要があります。したがって、このファイルには次のように記述する必要があります。

@lxpanel --profile LXDE-pi
@pcmanfm --desktop --profile LXDE-pi
@kodi

これで、すべてが当初の予定どおりに機能するはずです。Kodi自体を構成するだけですが、ここでは全員が自分で構成します。インターネットには、これを行う方法に関する多くの記事があります。

ステップ6：Kodiの設定に関する簡単なヒント。

必要になる可能性が最も高い設定についてのみ説明します。

言語の変更：設定/インターフェース/地域/言語
解像度の調整：設定/システム/画面/解像度
キーボードレイアウトの追加：設定/システム/ Enter /キーボードレイアウト

リポジトリからのアドオンのインストール：

  • アドオンリポジトリの追加：アドオン/アドオンブラウザ/ ZIPファイルからインストール。たとえば、ここからリポジトリをダウンロードできます。
  • リポジトリからのインストール：アドオン/アドオンブラウザ/リポジトリからインストール

このための便利な機能は、電話からのKodiの制御です。

  • [設定] / [サービス] / [管理]に移動し、 2つの項目を
    有効にします。HTTP経由のリモート制御を
    許可します他のシステムでのアプリケーション管理を許可します
  • Kodiに接続するためのデータをすぐに記録します（電話のアプリケーションで必要になります）：ポート、ユーザー名、パスワード。
  • お好きなアプリケーションをアプリケーションストアからダウンロードしてください。最も一般的なのは公式のKoreアプリですが、Sibuの方が好きでした。
  • アプリケーションの設定で、以前に記録したポート、ユーザー名、パスワードを指定します。
  • 便利な「リモコン」をお楽しみください。

IP TVを見たい場合は、PVR IPTVシンプルクライアントを使用する必要があるかもしれません。デフォルトでは、Kodiリポジトリにはありませんが、「Simple IPTV PVR for Kodi」というリクエストでソフトウェアの追加と削除でアクセスできます。
