PVS-StudioアナライザーによるEmbyソースコード分析


Embyは、PlexやKodiとともに、かなり有名なメディアサーバーです。この記事では、PVS-Studioスタティックアナライザーを使用してソースコードを検証します。Embyの公式Webサイトには、「ReSharperで構築」という小さなチェックマークがあります。さて、それはさらに興味深いです。

PVS-Studioコードアナライザー


PVS-Studioは、Windows、Linux、およびmacOSの64ビットシステムで動作します。彼は、C、C ++、C＃、Javaで書かれたプログラムのソースコードでエラーを探す方法を知っています。

監査済みプロジェクト


Embyはメディアサーバーであり、そのソースコードはGitHubにあります。これにより、ブロードキャストを整理し、任意のデバイスからメディアコンテンツ（ビデオ、オーディオ、写真）にアクセスできるようになります。Embyの公式Webサイトからスクイーズ：

  • Embyは、メディアをその場で自動的に変換し、任意のデバイスで再生できます。
  • 広範なペアレンタルコントロールにより、コンテンツへのアクセスを簡単に制御できます。これは、小さな子供を持つ家族に関連します。
  • Embyはメディアをシンプルでエレガントなプレゼンテーションに整理します。彼らは決して同じに見えることはありません。
  • クラウドでの同期によるデータのストリーミング。
  • メディアファイルを友人や家族と簡単に共有できます。
  • そして、はるかに。

結論


あらゆる意味で、開発者はEmbyを作成する素晴らしい仕事をしました（プロジェクトには215 539行のコード、4.6％-コメントがあります）。彼らは本当に素晴らしいです。ただし、アナライザーは、高-113、中-213、および低-112 のレベルの警告を発行することも試みました。それらのいくつかは誤検知でしたが、ほとんどのエラーは互いに類似しているため、記事には表示されませんでした。

たとえば、警告V3022のみ（式は常にfalse / trueです）は106個のスコアを付けました。もちろん、それらのどの部分が誤検知であったかをチェックし、残りを記事に追加できますが、これは非常に退屈なテキストになり、誰も読めなくなります。

開発中の静的アナライザーの有用性をなんとか実証できたと思います。もちろん、1回限りのチェックでは不十分であり、静的アナライザーを継続的に使用する必要があります。詳細については、Godot：On Static Use of Static Code Analyzersの記事をご覧ください。
