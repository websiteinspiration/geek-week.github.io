テレグラムブロッキング、ヘッツナーサブネット/ 16、ILVの経験

私は世界中に駐在員事務所を持つかなり大きなコンサルティング会社で働いています。ドイツの本社...たまたま、20年ほど前にロシアを離れた今まで、私はILVとブロッキングについて読んで共感しただけでした。

以下の情報は私が理解しているようにニュースではありませんが、コミュニティの関連情報、コミュニケーション経験（応答時間）、およびテンプレートの応答自体と共有したかったのです。

特に、ドイツの大手プロバイダーであるHetznerからいくつかのサーバーをレンタルしています。最近、ブロック178.63.0.0/16で小さなサブネット/ 29（8つのIPアドレス）を受け取った。利用開始の翌日、ロシアからサンクトペテルブルクとMSCのオフィスにこのネットワークとの接続がないという電話が届き始めました。ILVロックがすぐに発見されました。

ホスティングプロバイダーのブロッキング/ 16（つまり65536アドレス）は確かに強力です。グーグルは彼らにとってこれは制限ではないことを示していますが。同時に、電報はロシアからの知人と連携します...意味は、おそらく、誰も知りません。ILVの回答は少なくともスペルが正しく記述されていて、誰かが現代で書いたものでした。彼の脳は爆発せず、キーボードは溶けませんでした。ロシアでホストするという厚い提案は特に良い印象を残しました。

ただし、プロバイダーに連絡しました。答えは次のとおりです。

ヘッツナーの答え
We already tried to contact RKN about the issue with the IP rang 178.63.x.x, but without success.
You can order another IPv4 subnet via robot and write into the comment section that you need a subnet from a different range than 178.63.x.x.
