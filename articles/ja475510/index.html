数十行の増分バックアップ

バックアップで最も重要なことは何ですか？そうです、再現性。それでは、膝の上--link-destとrsyncオプションで自転車を作ってみましょう。私たちのバイクには、gitのような複雑なRESTICデータ構造も、重複のようなバックエンドの束もありません。しかし、ストレス下でも彼の作品を記憶から復元することができます。


このオプションを--link-dest使用すると、以前のバージョンのバックアップを指定できます。前回のファイルが変更されていない場合、rsyncによってハードリンクが配置されます。


つまり、rsync --link-dest=/var/backups/www.1 remote:/var/www /var/backups/www.0変更されたファイルのみをリモートサーバーから/var/backups/www.0フォルダーにコピーし、/ var / backups / www.1の残りのファイルへのハードリンクを配置します。


さて、問題は小さくなります。rsyncバックアップを1つ後ろにシフトし、新しいバックアップ用のスペースを解放するコードで呼び出しをラップし、/var/backups/www.0必要に応じて最後のコピーを削除します/var/backups/www.9。


# 
find /var/www/backups/ -maxdepth 1 -type d -name '*.[0-9]'| sort -rn| while read dir 
do
    #    
    this=`expr match "$dir" '.*\([0-9]\)'`;  
    #   1,  ,      10 
    let next=($this+1)%$10; 
    basedirname=${dir%.[0-9]}
    if [ $next -eq 0 ] ; then
         rm -rf $dir
    else
         mv $dir $basedirname.$next
    fi
done

このコードはに名前/var/backups/www.1を変更し/var/backups/www.2、に/var/backups/www.0名前を変更し/var/backups/www.1ます。


実行するだけでrsync --link-dest=/var/backups/www.1 remote:/var/www /var/backups/www.0、好みにオプションを追加できます。オプションは非常に--deleteファイルの最後のコピーを削除します（rsyncデフォルトではありません）、オプションでは-C、フォルダは無視され.svn、.gitアーティファクトpatchや一時ファイルのいくつかの一般的なタイプを。


一緒：


#!/bin/bash
FROM=$1 # 
TO=$2 # 
LINKTO=--link-dest=$TO/`basename $FROM`.1 #  
OPTS="-Ca --delete" #    rsync
NUMBER_OF_BACKUPS=10 #     
#    ,      
# dir.1, dir.2, dir.3     dir.9
find $TO -maxdepth 1 -type d -name '*.[0-9]'| sort -rn| while read dir
do
    this=`expr match "$dir" '.*\([0-9]\)'`; 
    let next=($this+1)%$NUMBER_OF_BACKUPS;
    basedirname=${dir%.[0-9]}
    if [ $next -eq 0 ] ; then
         rm -rf $dir
    else
         mv $dir $basedirname.$next
    fi
done
#  ,  rsync
rsync $OPTS $LINKTO $FROM/ $TO/`basename $FROM.0`

エラー処理（を参照set -e）と通知は括弧の後ろに残り、コピー数はカスタマイズ可能にすることができます。
