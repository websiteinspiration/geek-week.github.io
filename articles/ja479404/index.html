サンタクロースのスタッフ



毎年12月31日、私はサンタクロースのコスチュームを着ており、妻はスノーメイデンの役割をしていて、おめでとうと友達に街を走り回っています。もうすぐ新年が来るので、彼の準備がすべて整っているかどうかを確認し、古くなったスタッフを地下室から取り出しました。驚いたことに、地下室で1年間横になった後、彼は今でも適切に働いていました。彼のバッテリーはまだなんとか充電されています！スタッフはあまり明るくありませんが、光っています。もちろん充電する必要があります。1つのLEDが機能しなくなったようですが、これは修正可能です-修正します。

2012年にこのスタッフを作り、それ以来私は忠実に奉仕してくれました。物事は非常にシンプルですが、非常に効果的で、特に子供たちは喜んでいます。彼はほとんどジェダイの剣に似ています-スタッフだけで、本物のサンタクロースにとってはこれは必要なものです。私はここで彼についてハブレで書くことを決めました-突然誰かがアイデアに触発され、新年までにこれを自分のために作るでしょう、まだ時間があります。

スタッフを作るには、次のものが必要です。

  • 1.3メートルのプラスチック製の水道管。
  • 装飾用の少し光沢のある自己接着剤。
  • ネットワークケーブルUTP-5またはその他の適切なもの;
  • 青色LED、少なくとも24個;
  • 電池;
  • 電源スイッチ;
  • プログラム可能なものがある小さなマイクロコントローラーまたはFPGAボード;
  • 大きなボールクリスマスツリーのおもちゃ。半分に分割できるため、ボードとバッテリーの両方をその中に隠すことができます。

必要なパーツはこの写真に表示されています



。スタッフのスタッフは非常にシンプルです。特に書くことは何もありません。白いプラスチック製の水道管を取り、同じ距離200 mmで複数の穴を開けます。私は6つの穴を得ました。

UTP-5ケーブルをいくつかのツイストペアに分解します。 6ペアのワイヤが必要です。それらを穴に打ち込み、パイプの一端から出力して、スタッフのマジックボールを配置します。ワイヤーのペアがパイプの穴に入る場所では、それらをきれいにして、互いに一定の距離をおいてパイプの周りにしっかりと巻き付けます。はんだ付けします。結果の「バス」と平行に4つの青いLEDをスタッフの周りの円ではんだ



付けします。パイプの上部に、より大きな穴を開けて、そこにあるコントロールボードの電源ボタンを貼り付ける必要があります。



火星探査車のスカーフを制御盤として持っています。これがアルテラMAX IIのFPGAです。このボードはもともと低電力のおもちゃのエンジンを接続することを目的としていたため、ボード上のチップ出力は10個にまとめられ、より多くの電流を提供します。ボード上にそのような結合されたピンは6つしかありません-これらはボードf0、f1、f2、f3、f4、f5の「信号」です。したがって、FPGAピンを焼くことを恐れずに、これらの調査結果から直接LEDに電力を供給することができます。また、ボード自体には8つの黄色のLEDがあります-点灯することもできます。譜表上部のボールが半透明の場合は、視覚効果も追加されます。

たとえばスタッフでArduinoを使用する場合、マイクロコントローラーの1つのピンから10〜15mAを超えないようにする方が良いため、トランジスタキーで小さなスカーフをはんだ付けして多くのLEDを制御する必要があるかもしれませんが、LEDにはさらに多くのものが必要です。

ボードは制御プログラムでフラッシュする必要があります。スタッフのボタンを押すと、ボードがオンになり、すぐにスタッフのライトが点灯します。では、誰がどんなファンタジーを持っているのか。最初に1行目から6行目まですべてのLEDを順に点灯し、次に6行目から1行目までのターンをオフにします。

Verilog HDLコードはおそらく私が何を意味するかをよりよく説明します：

module effect(
	input wire clk,
	input wire mclk,
	output wire [7:0]leds,
	output wire f0,
	output wire f1,
	output wire f2,
	output wire f3,
	output wire f4,
	output wire f5
);

reg [3:0]cnt;
reg [5:0]bits;

initial 
begin
 bits= 6'h00;
 cnt = 4'h0;
end

always @(posedge clk)
	if(cnt==11)
		cnt<=0;
	else
		cnt <= cnt + 1'b1;
	
always @*
begin
	case(cnt)
	4'h0: bits = 6'b000001;
	4'h1: bits = 6'b000011;
	4'h2: bits = 6'b000111;
	4'h3: bits = 6'b001111;
	4'h4: bits = 6'b011111;
	4'h5: bits = 6'b111111;
	4'h6: bits = 6'b011111;
	4'h7: bits = 6'b001111;
	4'h8: bits = 6'b000111;
	4'h9: bits = 6'b000011;
	4'ha: bits = 6'b000001;
	default:
			bits = 6'b000000;
	endcase
end

assign f0 = bits[0] & mclk;
assign f1 = bits[1] & mclk;
assign f2 = bits[2] & mclk;
assign f3 = bits[3] & mclk;
assign f4 = bits[4] & mclk;
assign f5 = bits[5] & mclk;

assign leds = {bits[5],bits[5],bits[4],bits[3],bits[2],bits[1],bits[0],bits[0]};

endmodule

これは最上位モジュールではなく、effect.vモジュールです。最上位モジュールから、clk〜40Hzの2つのクロック周波数がここに供給されます-この周波数が効果の速度を決定し、2番目の周波数mclk〜320Hz、8倍高い。 mclk信号はさらに出力を再度変調するため、出力が常にオンになるとは限らないため、LEDがそれほど明るくならず、チップが熱くなりません。

この記事を覚える必要があるかどうかわかりません。ただし、念のため、上記のVerilogコードはこのテストベンチを使用して完全にシミュレーションできることを書きます。

`timescale 1ms / 1us

module tb;

reg clock = 1'b0;
always #10
	clock = ~clock;

reg [3:0]counter=0;
always @(posedge clock)
	counter <= counter+1;
	
wire [7:0]wleds;
wire wf0,wf1,wf2,wf3,wf4,wf5;

effect inst(
	.clk( counter[2] ),
	.mclk( clock ),
	.leds( wleds ),
	.f0( wf0 ),
	.f1( wf1 ),
	.f2( wf2 ),
	.f3( wf3 ),
	.f4( wf4 ),
	.f5( wf5 )
);

initial
begin
	$dumpfile("out.vcd");
	$dumpvars(0,tb);
 
	#10000;
	$finish();
end

endmodule

void setup()
{
  for (int i = start_pin; i< end_pin; i++ ) 
    pinMode(i, OUTPUT);
}

void loop()
{
	for (int i = start_pin; i< end_pin; i++ )
	{
		digitalWrite(i, HIGH);
		delay(250);
	}
	for (int i = start_pin; i< end_pin; i++ )

		digitalWrite(i, LOW);
