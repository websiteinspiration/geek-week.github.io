DevOps-それは何で、なぜ、そしてどれだけ需要があるのですか？



数年前、新しい専門のDevOpsエンジニアがITに登場しました。彼女はすぐに最も人気があり、市場で人気の1つになりました。しかし、パラドックスは、DevOpsの人気の一部は、そのような専門家を雇っている会社がしばしば他の専門家の代表とそれらを混同するという事実に起因するということです。 
 
この記事は、DevOpsの専門職のニュアンス、現在の市場の状況、および展望を分析するためのものです。この困難な問題は、Dmitry BurkovskyのGeekUniversityのオンライン大学のGeekBrainsのDevOps学部長の助けを借りて解決しました。


では、正確にはDevOpsとは何でしょうか


この用語自体は、開発オペレーションを表しています。これは、製品やサービスを準備するときに中規模または大規模な会社で仕事を整理するためのアプローチとして、それほど専門的ではありません。実際には、1つの会社のさまざまな部門が準備プロセスに参加しており、それらのアクションは常に十分に調整されているわけではありません。 
 
したがって、たとえば、開発者は、リリースされたプログラムまたはサービスを使用するユーザーにどのような問題が発生するかを常に知るわけではありません。テクニカルサポート-すべてを完全に把握していますが、ソフトウェアの「内部」に何があるかを知らない場合があります。そして、開発プロセスの調整、プロセスの自動化、透明性の向上を支援するDevOpsエンジニアが登場します。 
 
DevOpsコンセプトは、人、プロセス、ツールを1つにまとめます。 
 

DevOpsエンジニアは何を知って、何ができるはずですか？


DevOpsコンセプトの最も有名な支持者の1人によると、専門職の代表であるJoe Sanchezは、コンセプト自体のニュアンスを十分に理解し、WindowsシステムとLinuxシステムの両方の管理の経験があり、異なる言語で書かれたプログラムコードを理解し、Chef、Puppet、Ansibleを実行する必要があります。コードを解析するには、いくつかのプログラミング言語を知る必要があるだけでなく、開発の経験もあることは明らかです。また、既製のソフトウェア製品やサービスをテストした経験も非常に望ましいです。 
 
しかし、理想的には、このレベルの経験と知識は、IT分野のすべての代表とはかけ離れています。以下は、優れたDevOpsに必要な最小限の知識と経験のセットです。

  • GNU / Linux OS、Windows。
  • 少なくとも1つのプログラミング言語（Python、Go、Ruby）。
  • シェルスクリプト言語は、Linuxではbash、Windowsではpowershellです。
  • バージョン管理システム-Git。
  • 構成管理システム（Ansible、Puppet、Chef）。
  • コンテナーオーケストレーション用のプラットフォームの少なくとも1つ（Kubernetes、Docker Swarm、Apache Mesos、Amazon EC2 Container Service、Microsoft Azure Container Service）。
  • Terraformを使用してクラウドプロバイダー（例：AWS、GCP、Azureなど）と連携して、アプリケーションがクラウドにどのようにデプロイされているかを把握する機能。
  • CI / CDパイプライン（Jenkins、GitLab）、ELKスタック、監視システム（Zabbix、Prometheus）を構成する機能。

そして、ここにDevOpsスペシャリストがHaber Careerで最も頻繁に指摘するスキルのリストがあります。


 
さらに、DevOpsスペシャリストは、ビジネスのニーズと要件を理解し、開発プロセスにおけるその役割を理解し、顧客の関心を考慮してプロセスを構築できる必要があります。 


そして、入場閾値についてはどうですか？


知識と経験のリストは上記で無駄に提示されました。誰がDevOpsスペシャリストになることができるかを理解しやすくなりました。この職業に就く最も簡単な方法は、他のIT専門分野の代表、特にシステム管理者と開発者向けであることがわかりました。そして、それと他の人々は、不足している量の経験と知識をすばやく蓄積することができます。彼らはすでに必要なセットの半分、多くの場合半分以上を持っています。
 
そして、優れたDevOpsエンジニアはテスターから来ています。彼らは、ソフトウェアとハ​​ードウェアの欠点と欠点を認識しながら、それがどのように機能するかを知っています。 5分のDevOpsがなくても、プログラミング言語とプログラムの記述方法を知っているテスターと言えます。
 
しかし、開発やシステム管理のどちらも扱ったことがない技術者以外の専門家にとっては難しいでしょう。もちろん、不可能はありませんが、初心者は自分の強みを適切に評価する必要があります。必要な「手荷物」を受け取るまでには、かなりの時間がかかります。 


DevOpsはどこで機能しますか？


アプリケーションの開発とハードウェアの管理に直接または間接的に関連している大企業。DevOpsエンジニアの最大の不足は、エンドユーザーに多数のサービスを提供している企業にあります。これらは、銀行、通信事業者、最大のインターネットプロバイダーなどです。DevOpsエンジニアを積極的に採用している企業には、Google、Facebook、Amazon、Adobeがあります。
 
DevOpsと小規模ビジネスの新興企業を紹介しますが、これらの企業の多くにとって、DevOpsエンジニアを招待することは、実際の必要性よりもファッションへの賛辞です。もちろん例外はありますが、多くはありません。中小企業はむしろ、「スイス人、死神、そして男」、つまり多くの分野で働くことができる人を必要としています。良いサービスステーションは、これらすべてを処理できます。事実、中小企業は重要な仕事のスピードであり、中規模および大企業では作業プロセスの最適化が重要です。
