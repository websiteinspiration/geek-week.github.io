Sentry - Tracking-Java-Ausnahme in Java



Standardmäßig überwachen Java-Entwickler Fehler, Ausnahmen durch Protokolle. Es gibt aber auch einen anderen Weg, nämlich das Senden einer Ausnahme an Sentry.







Sentry ist ein Tool zum Überwachen von Ausnahmen und Fehlern in Ihren Anwendungen.







Vorteile der Verwendung von Sentry:







  • Seien Sie nicht nervös, wenn Sie Anwendungen auf einem Kampfserver platzieren.
  • Finden Sie schnell die Ursachen für Probleme
  • Beheben Sie Fehler, bevor Tester, Supportkollegen, Benutzer, ein PM oder ein Direktor Sie darüber informieren
  • Identifizieren Sie Probleme, die für den Rest des Teams unsichtbar sind und die das Leben der Benutzer schädigen, und verringern Sie die Wirksamkeit Ihres Produkts.
  • frei
  • leicht in das Projekt zu integrieren,
  • Fängt Fehler sowohl im Browser des Benutzers als auch auf Ihrem Server ab.
  • Wenn dieselbe Ausnahme in ELK mehrmals auftritt, werden sie als separate Datensätze gespeichert und belegen Speicherplatz und RAM. Wenn in Sentry dieselbe Ausnahme mehrmals auftritt, wird das Feld EREIGNISSE vergrößert, wodurch Speicherplatz und RAM gespart werden.

Hauptmerkmale:







  • Die Fehlerliste wird in Echtzeit aktualisiert.
  • Wenn der Fehler als behoben markiert wurde und erneut angezeigt wurde, wird er erneut erstellt und in einem separaten Thread berücksichtigt.
