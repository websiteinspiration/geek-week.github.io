Appiumとは何ですか？


モバイルテストについて一言。自宅にコンピュータやラップトップを置くことはできませんが、私たちはモバイルデバイスなしで私たちの生活を認識していません。モバイルアプリケーションのテストは...テスト...に大きなニッチを取ります...うーん...バターはバターですが、そうです。このニッチは非常に複雑です。

独自のグラフィカルインターフェイス、画面解像度などを備えた、オペレーティングシステムを備えたデバイスが非常に多くあります。このリストは無限大だと思います。そのため、モバイルデバイスは多種多様であるため、テスターの負担を軽減し、数百のデバイスの1つの機能のテストを「本当の意味での地獄」に変えないために、アプリケーションのテストを自動化するとよいでしょう（下の画像を参照）。



モバイルデバイス用の自動テストを作成するための多くのツールがあります。私の前に設定されたタスクを解決するために、私はAppiumを選択しました。これは、AndroidとiOSに同時に適しており、「みんなのお気に入り」のJavaを使用してセルフテストを記述できるためです。

Appiumは、AndroidとiOSの両方のアプリケーションを自動化するのに役立つ無料のオープンソースのクロスプラットフォームツールです。AppiumはSelenium WebDriverと同じアプローチを採用しています。これは、クライアントからJSON HTTPリクエストを受信し、それが動作するプラットフォームに応じてそれらを変換します。



必要最小限


それでは、Appiumと連携するために何が必要ですか？

Androidのテストを作成するには、次の要件が提案されます。

  • Java（バージョン> = 7）
  • Android SDK API（バージョン> = 17）
  • Android仮想デバイス（AVD）/実際のデバイス
  • Intel Hardware Accelerated Execution Manager

Androidデバイスエミュレーターを実際のデバイスの速度に近い速度で動作させるには、Intel Hardware Accelerated Execution Managerをインストールする必要があります。これにより、アプリケーションの起動とデバッグにかかる​​時間を短縮できます。
