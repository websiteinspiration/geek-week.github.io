Googleフォームをエクスポート+ REST API（Python）経由でGoogleスクリプトをダウンロード



2つのGoogleフォームがあり、それぞれに75の質問があり、これらのフォームを積極的に編集した5人のビジネスユーザーと、フォームをJSONにエクスポートするGoogleスクリプトがありました。毎回手で実行するのは難しいことではありませんが、作業を自動化し始めたら、この趣味に最後まで行きます。

公式ドキュメントでは、悪魔が脚を折るので、猫の下で、Pythonを使用したREST APIを介したリモートダウンロードとGoogle Apps Scriptの起動を詳しく見ていきます。

前書き


Doctor Nearでは、シナリオを記述するためにGoogleフォームが使用されるチャットボット用のプラットフォームを開発しています。したがって、ノード（フォームポイント）とそれらへのメタデータ（ノード間の遷移、ノードのタイプ、名前）を含むボタンをクリックするだけでフォームからJSONを取得したいと思います。欲求は単純に思えますが、Googleはこの機能をサポートしていないため、この「エクスポーター」を自分の手で組み立てる必要があります。それを作成する手順を見てみましょう。

/**
 * Returns the form metadata object for the given Form object.
 * @param form: Form
 * @returns (Object) object of form metadata.
 */
function getFormMetadata(form) {
  return {
    "title": form.getTitle(),
    "id": form.getId(),
    "description": form.getDescription(),
    "publishedUrl": form.getPublishedUrl(),
    "editorEmails": form.getEditors().map(function(user) { return user.getEmail() }),
    "count": form.getItems().length,
    "confirmationMessage": form.getConfirmationMessage(),
    "customClosedFormMessage": form.getCustomClosedFormMessage()
  };
}

/**
 * Returns an Object for a given Item.
 * @param item: Item
 * @returns (Object) object for the given item.
 */
function itemToObject(item) {
  var data = {};

  data.type = item.getType().toString();

  // Downcast items to access type-specific properties

  var itemTypeConstructorName = snakeCaseToCamelCase("AS_" + item.getType().toString() + "_ITEM");
  var typedItem = item[itemTypeConstructorName]();

  // Keys with a prefix of "get" have "get" stripped

  var getKeysRaw = Object.keys(typedItem).filter(function(s) {return s.indexOf("get") == 0});

  getKeysRaw.map(function(getKey) {
    var propName = getKey[3].toLowerCase() + getKey.substr(4);

    // Image data, choices, and type come in the form of objects / enums
    if (["image", "choices", "type", "alignment"].indexOf(propName) != -1) {return};

    // Skip feedback-related keys
    if ("getFeedbackForIncorrect".equals(getKey) || "getFeedbackForCorrect".equals(getKey)
      || "getGeneralFeedback".equals(getKey)) {return};

    var propValue = typedItem[getKey]();

    data[propName] = propValue;
  });

  // Bool keys are included as-is

  var boolKeys = Object.keys(typedItem).filter(function(s) {
    return (s.indexOf("is") == 0) || (s.indexOf("has") == 0) || (s.indexOf("includes") == 0);
  });

  boolKeys.map(function(boolKey) {
    var propName = boolKey;
    var propValue = typedItem[boolKey]();
    data[propName] = propValue;
  });

  // Handle image data and list choices

  switch (item.getType()) {
    case FormApp.ItemType.LIST:
    case FormApp.ItemType.CHECKBOX:
      data.choices = typedItem.getChoices().map(function(choice) {
        return choice.getValue()
      });
    case FormApp.ItemType.MULTIPLE_CHOICE:
      data.choices = typedItem.getChoices().map(function(choice) {
        gotoPage = choice.getGotoPage()
        if (gotoPage == null)
            return choice.getValue()
        else
            return {
                "value": choice.getValue(),
                "gotoPage":choice.getGotoPage().getId()
        };
      });
      break;

    case FormApp.ItemType.IMAGE:
      data.alignment = typedItem.getAlignment().toString();

      if (item.getType() == FormApp.ItemType.VIDEO) {
        return;
      }

      var imageBlob = typedItem.getImage();

      data.imageBlob = {
        "dataAsString": "", //imageBlob.getDataAsString(), - BLOB too big
        "name": imageBlob.getName(),
        "isGoogleType": imageBlob.isGoogleType()
      };

      break;

    case FormApp.ItemType.PAGE_BREAK:
      data.pageNavigationType = typedItem.getPageNavigationType().toString();
      break;

    default:
      break;
  }

  // Have to do this because for some reason Google Scripts API doesn't have a
  // native VIDEO type
  if (item.getType().toString() === "VIDEO") {
    data.alignment = typedItem.getAlignment().toString();
  }

  return data;
}

/**
 * Converts a SNAKE_CASE string to a camelCase string.
 * @param s: string in snake_case
 * @returns (string) the camelCase version of that string
 */
function snakeCaseToCamelCase(s) {
  return s.toLowerCase().replace(/(\_\w)/g, function(m) {return m[1].toUpperCase();});
}


コードで何が起こるか：

  • getFormMetadata関数-フォームのメタデータを含むJSONを返します
  • itemToObject関数-form.itemオブジェクトを必須フィールドを使用してJSONに変換します
  • sendEmail関数-JSONファイルを指定されたメールにテキストで送信します
  • メイン関数-結果のJSONを返します
  • メイン関数のform_url変数は、Googleフォームのアドレスです

ステップ4. Python REST API


REST API を使用してスクリプトを自動化する番です。言語としてPythonが使用されました。

Apps Script APIログイン


コードはプロジェクトにアクセスできる必要があるため、最初の非常に重要な手順はApps Script APIへのログインです。ページ2を開く->「APIおよびサービス」->「資格情報」->「資格情報の作成」->「OAuthクライアントID」->「その他のタイプ」。私たちは識別子と呼び、それに行きます。 [認証情報]タブで、[JSONファイルのダウンロード]を選択します。これにより、コードからGoogleのプロジェクトにアクセスするためのキーファイルがダウンロードされます。このファイルを信任状フォルダーに配置します。

次に、このプロジェクトの一部としてAPI（この場合はApps Script API）を使用する権限を与える必要があります。これを行うには、「APIとサービス」->「ライブラリ」->検索に「Apps Script API」と入力し、「有効にする」をクリックします。

Googleとやり取りするアプリケーションには、起動時にユーザーが与えなければならない一連の権限があります。このコピーは、特定のスクリプトで使用される関数に依存しており、スクリプト編集ウィンドウの「ファイル」->「プロジェクトのプロパティ」->「スコープ」の1ページに移動すると確認できます。これらの権限は、今後のコードでの参照用に保持する必要があります。

この場合、ログイン関数は次のようになります。

import pickle
import os.path

from googleapiclient.discovery import build
from google_auth_oauthlib.flow import InstalledAppFlow
from google.auth.transport.requests import Request

def login(config):
    try:
        creds = None
        # The file token.pickle stores the user's access and refresh tokens, and is
        # created automatically when the authorization flow completes for the first
        # time.
        token_file = config['credentials_path'] + config['token_file']
        credentials_file = config['credentials_path'] + config['credentials_file']
        if os.path.exists(token_file):
            with open(token_file, 'rb') as token:
                creds = pickle.load(token)
        # If there are no (valid) credentials available, let the user log in.
        if not creds or not creds.valid:
            if creds and creds.expired and creds.refresh_token:
                creds.refresh(Request())
            else:
                flow = InstalledAppFlow.from_client_secrets_file(credentials_file, config['SCOPES'])
                creds = flow.run_local_server(port=0)
            # Save the credentials for the next run
            with open(token_file, 'wb') as token:
                pickle.dump(creds, token)

        service = build('script', 'v1', credentials=creds)
        pprint('Login successful')
        return service

    except Exception as e:
        pprint(f'Login failure: {e}')
        return None

このコードブロックは、Google App Scriptを使い始めるための標準的な手順です。
認証トークンを使用し、ログインして新しいトークンを作成するか、既存のトークンを使用します。

便宜上、次の形式のJSON構成ファイルが作成されました。

{
    "SCOPES": ["https://www.googleapis.com/auth/forms",
        "https://www.googleapis.com/auth/script.send_mail"],
    "credentials_path": "credentials/",
    "credentials_file": "google_test_project.json",
    "token_file": "token.pickle"
}

重要：トークンは、特定の範囲の権限を持つ認証用に作成されます。つまり、権限の範囲を変更する場合は、ログイン時にトークンを削除し、新しいトークンを作成する必要があります。

リモート更新スクリプトコード


次に、スクリプトコードをリモートで更新し、このコードを実行して結果を取得する方法を学びます。実際、Googleエディターで実行するコードに加えて、起動権、デプロイメント設定などを含むマニフェストファイルもあります。その構造の詳細については、こちらをご覧ください。
スクリプト用にGoogleが作成したデフォルトのマニフェストファイルを確認するには、[表示]-> [マニフェストファイルを表示]のスクリプトエディターに移動します。マニフェストは、このスクリプトに関連するファイルのリストに表示されます。

マニフェストに関するスピーチには理由がありました。リモートスクリプトの更新には、ファイル（* .gs）とマニフェスト（appscript.json）の両方のコードをダウンロードする必要があります。

まず、デプロイする.gsファイルのコードを読み取ります。

    with open('export-google-form.gs', 'r') as f:
        sample_code = f.read()

次に、自動生成されたマニフェストをコピーし、目的に合わせて少し変更します。ドキュメントはマニフェストファイルの構造を非常に包括的に説明しているため、この点については詳しく説明しません。スクリプトを機能させるには、デフォルトのマニフェストに「executionApi」セクションを追加する必要があります。これは、APIを介してリモートでスクリプトを実行するために必要です。このセクションでは、それを実行する能力を持つ人々の輪を示します。ID「ANYONE」に対応する、承認を通過したすべてのユーザーに起動を許可しました。

{
    "timeZone": "America/New_York",
    "exceptionLogging": "STACKDRIVER",
    "executionApi": {
        "access": "ANYONE"
    }
}
'''.strip()

  • ソース：ファイルコード

スクリプトの実行


スクリプト起動 リクエストには、次の構造を持つscriptIDと本文が含まれています。

  • function：実行する関数の名前
  • parameters：（オプション）関数に渡されるプリミティブ型のパラメーターのセット（文字列、配列...）
  • sessionState：（オプション）はAndroidアプリケーションにのみ必要です
  • devMode：（オプション）ユーザーがスクリプトの所有者であり、Apps Script APIを使用してデプロイされたバージョンよりも最新のバージョンが起動される場合はTrue。（デフォルト-False）

スクリプト内のGoogleフォームのURLをステッチしないようにするために、我々は通過しますform_urlに主な機能の引数として。

注意。スクリプトをテストしたとき、main関数は何も受け入れなかったため、.gsファイルのコードの最初の行を次のように変更します。

function main(form_url) {

  var form = FormApp.openByUrl(form_url);
.......

アプリケーションはAndroid用ではなく、スクリプトの所有者であるため、最終的には本文は次のようになります。

body = {
        "function": "main",
        "devMode": True,
        "parameters": form_url
    }

スクリプトを実行し、結果をresp変数に書き込みます。

resp = service.scripts().run(scriptId=script_id, body=body).execute()

便利なJSON形式でファイルにrespを保存します。

import json

with open('habr_auto.json', 'w', encoding='utf-8') as f:
    json.dump(resp['response']['result'], f, ensure_ascii=False, indent=4)

注意。script.run（）リクエストがソケットを介して結果を待機しているため、実行時間がタイムアウトを超えると、次のタイプのエラーが発生します。

socket.timeout: The read operation timed out

この動作を回避するには、プログラムの開始時に、ソケットのオープン時間に制限を設定することをお勧めします。これは、スクリプトの実行が完了するのを待つだけで十分です。私の場合、120秒で十分です。

import socket

socket.setdefaulttimeout(120)

from pprint import pprint
import json
import sys

from googleapiclient import errors

from google_habr_login import login

MANIFEST = '''
{

    "exceptionLogging": "STACKDRIVER",
    "executionApi": {
        "access": "ANYONE"
    }
}
'''.strip()


def update_project(service, script_id, script_file_name):
    # Read from file code we want to deploy
    with open(script_file_name, 'r') as f:
        sample_code = f.read()

    # Upload two files to the project
    request = {
        'files': [{
            'name': 'hello',
            'type': 'SERVER_JS',
            'source': sample_code
        }, {
            'name': 'appsscript',
            'type': 'JSON',
            'source': MANIFEST
        }
        ]
    }

    # Update files in the project
    service.projects().updateContent(
        body=request,
        scriptId=script_id
    ).execute()

    pprint('Project was successfully updated')


def main():

    try:

        args = sys.argv
        if len(args) != 4:
            raise TypeError('Wrong number of arguments. Three argument required: <config_file_name>, <script_id> and ' 
                            '<script_file_name>')
        config_file_name = args[1]
        script_id = args[2]
        script_file_name = args[3]

        with open(config_file_name, "r") as f:
            config = json.load(f)

        service = login(config)

        update_project(service, script_id, script_file_name)

    except (errors.HttpError, ) as error:
        # The API encountered a problem.
        pprint(error.content.decode('utf-8'))


if __name__ == '__main__':
    main()


export_form.py
from pprint import pprint
import socket
import json
import sys

from googleapiclient import errors

from google_habr_login import login


socket.setdefaulttimeout(120)


# Get JSON, which is returned by script
def get_json(service, file_name, script_id, form_url):
    pprint('Exporting form...')
    body = {
        "function": "main",
        "devMode": True,
        "parameters": form_url
    }
    # Get JSON from script
    resp = service.scripts().run(scriptId=script_id, body=body).execute()

    # Write out JSON to file
    with open(file_name, 'w', encoding='utf-8') as f:
        json.dump(resp['response']['result'], f, ensure_ascii=False, indent=4)

    pprint('Form was successfully exported')


def main():

    try:

        args = sys.argv
        if len(args) != 5:
            raise TypeError('Wrong number of arguments. Four arguments required:  <config_file_name>, '
                            '<result_file_name>, <script_id> and <google_form_url>')
        config_file_name = args[1]
        file_name = args[2]
        script_id = args[3]
        form_url = args[4]

        with open(config_file_name, "r") as f:
            config = json.load(f)

        service = login(config)

        get_json(service, file_name, script_id, form_url)

    except (errors.HttpError, ) as error:
        # The API encountered a problem.
        pprint(error.content.decode('utf-8'))


if __name__ == '__main__':
    main()


まず、認証情報フォルダーにGoogleアクセスキーを含むJSONファイルを配置し、スクリプトと同じディレクトリに構成JSONを配置する必要があります。

次に、スクリプトをリモートで更新する場合は、ターミナルコールで次のようにします。

python update_script.py <config_file_name> <script_id> <script_file_name>

この場合：

  • config_file_name-構成JSONファイルの名前
  • script_id-スクリプトID
  • script_file_name -Googleにアップロードされる.gsファイルの名前

スクリプトを実行するには、次を呼び出します。

python export_form.py <config_file_name> <result_file_name> <script_id> <google_form_url>

この場合：

  • config_file_name-構成JSONファイルの名前
  • result_file_name-フォームがアンロードされるJSONファイルの名前
  • script_id-スクリプトID
  • google_form_url -GoogleフォームのURL
