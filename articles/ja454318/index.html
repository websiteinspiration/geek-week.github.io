過去の響き。データを音に変換する歴史家のためのガイド

過去を見るのに疲れました。歴史的な遺物の外観を再現するためのガイドはたくさんありますが、これは創造的な行為であることを忘れがちです。おそらく、私たちは画面にあまりにも執着しており、外観を重視しすぎています。代わりに、過去のことを聞いてみましょう。

archaeoacousticsと音景観上の豊富な文献は、場所のサウンドを再現できます、それがあったように（例えば、参照のセントポール大聖堂仮想または古代オスティアのジェフVeitchはの仕事を）。しかし、私はデータ自体を「ボイシング」することに興味があります。これらのアルゴリズムを歴史科学で使用できるように、データを音の形で表す構文を定義したいと思います。ドラッカーは言った「データ」は実際には与えられたものではなく、キャプチャされ、変換されたもの、つまり「キャプタ」であるという有名なフレーズ。データを発声するとき、文字通り過去を現在に再現します。したがって、このデータの仮定と変換が前面に出ます。結果として得られる音は、新しい方法で歴史の現代の層を聞くことができる「変形パフォーマンス」です。

過去の意味を聞きたいのですが、不可能だと思います。しかし、楽器を聞くと、ミュージシャンを物理的に想像できます。エコーと共鳴によって、私は物理的な空間を区別することができます。低音を感じ、リズムで動くことができます。音楽は私の体、すべての想像力を覆います。以前に聞いた音、音楽、音色とのつながりは、私と過去との具体的な関係のシステムである深い時間的体験を生み出します。視覚？私たちは過去を視覚的に表現してきたので、これらの文法は芸術的な表現力とパフォーマンスの側面をほとんど失っています。

このレッスンでは、履歴データからノイズを発生させる方法を学習します。このノイズの重要性は、まあ...あなた次第です。ポイントの一部は、データを再び見慣れないものにすることです。それらを翻訳、再コーディング、復元することで、目視検査中に見えないままだったデータ要素が見え始めます。この変形は、例えば社会の変形についてのマーク・サンプルや「物質の抵抗」についてのベサニー・ヌウィスキーによってなされた議論と一致しています。スコアリングは、データからCaptへ、社会科学からアートへ、グリッチから美学へと私たちを導きます。それがどのように見えるか見てみましょう。

目標と目的


このチュートリアルでは、データからサウンドまたは音楽を生成する3つの方法について説明します。

まず、Jonathan Middletonが開発した無料のオープンミュージカルアルゴリズムを使用します。その中で、主要な問題と用語に精通します。次に、小さなPythonライブラリを使用して、データを88キーのキーボードに「変換」し、作業に創造性をもたらします。最後に、データをSonic Piのリアルタイムサウンドおよび音楽処理プログラムにアップロードします。このプログラムについては、多くのチュートリアルとリファレンスリソースが公開されています。

サウンドを操作することで、単純な視覚化から本当に効果的な環境にどのように移動するかがわかります。

空白を埋める


人は音を聞いて、沈黙の瞬間を期待で満たします。mp3がMIDIに変換され、mp3に戻されるビデオを考えてみます。音楽は「フラット化」されるため、すべてのオーディオ情報が1つのツールで再生されます（効果は、Webページを.txtとして保存し、Wordで開いてから、.html形式で再度保存するのと同じです）。すべてのサウンド（ボーカルを含む）は対応する音価に変換され、mp3に戻されます。

これはノイズですが、要点をキャッチできます。


何が起きてる？この歌があなたに知られていれば、おそらくあなたは本当の「言葉」を理解したでしょう。しかし、歌には言葉がありません！これまで聞いたことがない場合は、意味のない不協和音のように聞こえます（詳細については、Andy Bayoのウェブサイト）。この効果は、幻聴と呼ばれることもあります。この例は、厳密に言えば、どのようなデータ表現でどのように聞こえるか、何がそうでないかを示しています。私たちは私たち自身の期待で空白を埋めます。

これは物語にとって何を意味しますか？データを声に出して、音のパターンや奇妙な爆発を聞き始めた場合、音楽に対する文化的な期待（特定の状況で聞いた音楽の同様の断片の記憶）によって解釈が色付けされます。これは過去のすべてのアイデアに当てはまると思いますが、スコアリングは標準的な方法とはかなり異なるため、この自己認識は過去の（データに関する）重要なパターンを特定または表現するのに役立ちます。

データをスコアリングするための3つのツールを検討し、ツールの選択が結果にどのように影響するか、および別のツールでデータを再考してこの問題を解決する方法に注意してください。結局のところ、スコアリングは視覚化よりも客観的ではないため、研究者は自分の選択を正当化し、この選択を透明性と再現性のあるものにする準備ができている必要があります。（スコアリングやアルゴリズムで生成された音楽は新しいものだと誰も考えないように、私は興味のある読者にHedges、1978を紹介しています）。

各セクションには概念的な概要が含まれ、その後に考古学的または歴史的なデータを使用したウォークスルーが続きます。

MIDITime


MIDITimeは、Reveal News（旧称Center for Investigative Journalism）によって開発されたPythonパッケージです。Github上のリポジトリ。 MIDITimeは、時系列（つまり、時間をかけて収集された一連の観測）を処理するように特別に設計されています。

ミュージカルアルゴリズムには多かれ少なかれ直感的なインターフェイスがありますが、ここでの利点はオープンソースです。さらに重要なことに、以前のツールでは履歴時間を考慮に入れてデータを考慮することができません。 MIDITimeを使用すると、この要素に関する情報をクラスター化できます。主題モデルが

適用された歴史的な日記があるとします。結果の出力には、文字列の形式の日記エントリが含まれる場合があり、列には各トピックの貢献率が含まれます。この場合、価値観を聞くことは、グラフの形では伝えることができない日記からの思考のパターンを理解するのに役立ちます。公聴会では、チャートに表示されていないサージや繰り返しの音楽パターンがすぐにわかります。

MIDITimeの設定


1つのpipコマンドによるインストール：

$ pip install miditime

ポピー用;

$ sudo pip install miditime

Linuxの下で; 

> python pip install miditime

Windows（手順が機能しない場合は、このユーティリティを試して Pipをインストールできます）。

練習


スクリプトの例を考えてみましょう。テキストエディターを開き、次のコードをコピーして貼り付けます。

#!/usr/bin/python

from miditime.miditime import MIDITime
# NOTE: this import works at least as of v1.1.3; for older versions or forks of miditime, you may need to use
# from miditime.MIDITime import MIDITime

# Instantiate the class with a tempo (120bpm is the default) and an output file destination.
mymidi = MIDITime(120, 'myfile.mid')

# Create a list of notes. Each note is a list: [time, pitch, attack, duration]
midinotes = [
    [0, 60, 200, 3],  #At 0 beats (the start), Middle C with attack 200, for 3 beats
    [10, 61, 200, 4]  #At 10 beats (12 seconds from start), C#5 with attack 200, for 4 beats
]

# Add a track with those notes
mymidi.add_track(midinotes)

# Output the .mid file
mymidi.save_midi()

スクリプトをとして保存しますmusic1.py。ターミナルまたはコマンドラインで、次のコマンドを実行します。

$ python music1.py

新しいファイルがディレクトリに作成されますmyfile.mid。QuicktimeまたはWindows Media Playerを使用してリスニング用に開くことができます（GarageBandまたはLMMSにツールを追加します）。

Music1.pymiditimeをインポートします（スクリプトを実行する前にインストールすることを忘れないでください:) pip install miditime。その後、ペースを設定します。すべてのノートは個別にリストされ、最初の数字は再生の開始時間、高さ（つまり、ノート自体！）、ノートが演奏される強さ（リズム）の強さ（アタック）、およびその持続時間です。次に、ノートがトラックに記録され、トラック自体がファイルに記録されますmyfile.mid。

スクリプトで遊んで、さらにメモを追加します。ここでは、曲「Baa Baa Black Sheep」のメモです。

D、D、A、A、B、B、B、B、A
Baa、Baa、black、sheep、have、you、any、wool？

ところで。ABC Notationと呼ばれる、音楽を説明するための特別なテキストファイル形式があります。これはこの記事の範囲を超えていますが、たとえばスプレッドシートでスコアリング用のスクリプトを記述して、ABC表記でメモの値を比較できます（これまでにExcelでIF-THEN構文を使用したことがある場合は、これを行う方法を考えています）。次に、このようなサイトを通じて、 ABC表記は.midファイルに変換されます。

自分のデータをアップロードする


このファイルには、Macroscopeの John Adams日記テーマモデルのサンプルが含まれています。最も強い信号のみがここに残され、列の値は小数点以下2桁に丸められました。このデータをPythonスクリプトに挿入するには、特別な方法でフォーマットする必要があります。最も難しいのは日付フィールドです。

このチュートリアルでは、変数名と残りをサンプルスクリプトから変更しないでおきましょう。例は地震データを処理するように設計されています。したがって、ここで「大きさ」は「トピックの貢献」として表すことができます。 

my_data = [
    {'event_date'：<datetime object>、 'magnitude'：3.4}、
    {'event_date'：<datetime object>、 'magnitude'：3.2}、
    {'event_date'：<datetime object>、 'magnitude'：3.6}、
    {'event_date'：<datetime object>、 'magnitude'：3.0}、
    {'event_date'：<datetime object>、 'magnitude'：5.6}、
    {'event_date'：<datetime object>、 'magnitude'：4.0}
]

正規表現を使用してデータをフォーマットすることができ、スプレッドシートをさらに簡単にフォーマットできます。テーマの貢献度の値を持つ要素を新しいシートにコピーし、列を左右に残します。以下の例では、それを列Dに入れてから、残りの部分に入力します。

あ B C D E
1 {'event_date'：日時 （1753、6、8） 、「大きさ」： 0.0024499630 }、
2
３

次に、不変要素をコピーして貼り付け、列全体に入力します。日付のあるアイテムは、形式（年、月、日）である必要があります。テーブルに入力した後、それをコピーしてテキストエディタに貼り付け、配列の一部にすることができます。my_data次に例を示します。

my_data = [
{'event_date': datetime(1753,6,8), 'magnitude':0.0024499630},
{'event_date': datetime(1753,6,9), 'magnitude':0.0035766320},
{'event_date': datetime(1753,6,10), 'magnitude':0.0022171550},
{'event_date': datetime(1753,6,11), 'magnitude':0.0033220150},
{'event_date': datetime(1753,6,12), 'magnitude':0.0046445900},
{'event_date': datetime(1753,6,13), 'magnitude':0.0035766320},
{'event_date': datetime(1753,6,14), 'magnitude':0.0042241550}
]

最後の行の終わりにコンマがないことに注意してください。

Miditime自体のページの例を使用すると、最終的なスクリプトは次のようになります（以下のコードフラグメントはコメントによって中断されていますが、テキストエディターで単一のファイルとして一緒に挿入する必要があります）。

from miditime.MIDITime import MIDITime
from datetime import datetime
import random

mymidi = MIDITime(108, 'johnadams1.mid', 3, 4, 1)

MIDITimeの後の値は、次のように設定されMIDITime(108, 'johnadams1.mid', 3, 4, 1)ます。

  • 1分あたりのビート数（108）、
  • 出力ファイル（ 'johnadams1.mid'）、
  • 歴史上の1年を表す音楽の秒数（暦年あたり3秒、50年の日記エントリは50×3秒、つまり2分半のメロディーにスケーリングされます）、
  • 音楽の基本オクターブ（中央のCは通常C5として表されるため、ここでは4が基準より1オクターブ低い）
  • 高さを比較するためのオクターブの数。

次に、データを配列にロードしてスクリプトに渡しますmy_data。

my_data = [
{'event_date': datetime(1753,6,8), 'magnitude':0.0024499630},
{'event_date': datetime(1753,6,9), 'magnitude':0.0035766320},

...ここでは、すべてのデータを挿入し、最後の行の末尾にあるカンマを削除することを忘れないでください。event_dateデータの後に、最後の角かっこを別の行に挿入します。

{'event_date': datetime(1753,6,14), 'magnitude':0.0042241550}
]

次にタイミングを挿入します：

my_data_epoched = [{'days_since_epoch': mymidi.days_since_epoch(d['event_date']), 'magnitude': d['magnitude']} for d in my_data]

my_data_timed = [{'beat': mymidi.beat(d['days_since_epoch']), 'magnitude': d['magnitude']} for d in my_data_epoched]

start_time = my_data_timed[0]['beat']

このコードは、異なる日記エントリ間のタイミングを設定します。日記のエントリが時間的に互いに近い場合、対応するメモも近くなります。最後に、データと高さの比較方法を決定します。初期値は0.01（つまり1％）から0.99（99％）の範囲のパーセンテージで示されるためscale_pct、0と1の間に設定します。パーセンテージがない場合は、最低値と最高値を使用します。したがって、次のコードを挿入します。

def mag_to_pitch_tuned(magnitude):
    scale_pct = mymidi.linear_scale_pct(0, 1, magnitude)
    # Pick a range of notes. This allows you to play in a key.
    c_major = ['C', 'C#', 'D', 'D#', 'E', 'E#', 'F', 'F#', 'G', 'G#', 'A', 'A#', 'B', 'B#']

    #Find the note that matches your data point
    note = mymidi.scale_to_note(scale_pct, c_major)

    #Translate that note to a MIDI pitch
    midi_pitch = mymidi.note_to_midi_pitch(note)

    return midi_pitch

note_list = []

for d in my_data_timed:
    note_list.append([
        d['beat'] - start_time,
        mag_to_pitch_tuned(d['magnitude']),
        random.randint(0,200),  # attack
        random.randint(1,4)  # duration, in beats
    ])

ソニックパイ


GarageBandまたは他の音楽エディターで独自のMIDIを処理するということは、単純なダビングからミュージカルアートへの移行を意味します。この記事の最後のセクションは、Sonic Piの使用に関する完全なガイドではなく、音楽の形式でのリアルタイムエンコーディングとデータの再生を可能にする環境に精通しています（リアルタイム再生によるエンコーディングの例については、ビデオを参照してください）。プログラムに組み込まれたチュートリアルは、コンピューターを楽器として使用する方法を示します（組み込みのエディターにRubyコードを入力すると、インタープリターがすぐに結果を再生します）。

なぜこれが必要なのですか？このガイドから理解できるように、データを読むときに、データを音声に変換する方法について決定を始めます。これらの決定は、重要なデータに関する暗黙的または明示的な決定を反映しています。必要に応じて、「客観性」の連続があります。一方、有声歴史データ、他方-過去のアイデアは、よくできた公開講座と同じくらい刺激的で個人的なものです。サウンドを使用すると、ドキュメントに格納されているデータを実際に聞くことができます。これは一種の公的な歴史です。私たちのデータの音楽パフォーマンス...想像してみてください！

ここでは、csvとして格納されている値のリストにすぎない、データをインポートするためのコードスニペットを提案します。ジョージワシントン大学の司書であるLaura Vrubelに感謝します。彼のライブラリ操作の実験をgist.github.comにアップロードしてくれました。このサンプルには2つのトピック

があります（イエズス会関係から生成された主題モデル）。1行目のヘッダーは「topic1」と「topic2」です。

練習


インターフェースと機能に慣れるまで、組み込みのSonic Piチュートリアルに従ってください（これらのチュートリアルはすべてここにまとめられています。SonicPiの作成者であるSam Aaron へのインタビューも聞くことができます）。次に、新しいバッファー（エディターウィンドウ）で、次のコードをコピーします（ここでも、1つのスクリプトで個別のフラグメントを収集する必要があります）。

require 'csv'
data = CSV.parse(File.read("/path/to/your/directory/data.csv"), {:headers => true, :header_converters => :symbol})
use_bpm 100

path/to/your/directory/ これがコンピュータ上のデータの実際の場所で あることを忘れないでください。ファイルの名前が実際にであることを確認するdata.csvか、コードのこの行を編集してください。

次に、このデータを楽曲にロードします。

#this bit of code will run only once, unless you comment out the line with
#'live_loop', and also comment out the final 'end' at the bottom
# of this code block
#'commenting out' means removing the # sign.

# live_loop :jesuit do
data.each do |line|
  topic1 = line[:topic1].to_f
  topic2 = line[:topic2].to_f

  use_synth :piano
  play topic1*100, attack: rand(0.5), decay: rand(1), amp: rand(0.25)
  use_synth :piano
  play topic2*100, attack: rand(0.5), decay: rand(1), amp: rand(0.25)
  sleep (0.5)
end

最初の数行はデータ列をロードします。次に、使用するサウンドのサンプル（ピアノ）を指定し、指定された基準に従って最初のトピック（topic1）を再生するように指定します。ノートの強さ（アタック）には、0.5未満のランダムな値が選択されます。崩壊の場合-1未満のランダムな値。振幅の場合、0.25未満のランダムな値。

100倍した行を見る（*100）？データ値（10進数）を取り、それを整数に変換します。このフラグメントでは、番号は直接音符と同じです。最低音が88で最高音が1の場合、このアプローチは少し問題があります。実際にはここではピッチを表示しません！この場合、ミュージカルアルゴリズムを使用してピッチを表示し、これらの値をSonic Piに返すことができます。さらに、このコードは多かれ少なかれ標準のRubyであるため、データを正規化する通常の方法を使用して、1から88の範囲で値を線形比較することができます。手始めに、Sonic Piを使用した気象データの発声に関するSteve Lloydの研究を見るとよいでしょう。

そして、ここで最後に注意する必要があるのは、「rand」（ランダム）の値により、ダイナミクスに関して音楽に少し「人間らしさ」を加えることです。「topic2」についても同様です。

Sonic Piがサポートするリズム（1分あたりの拍数）、ループ、サンプル、およびその他の効果を指定することもできます。コードの場所は再生に影響します。たとえば、上記のデータブロックの前に配置すると、最初に再生されます。たとえばuse_bpm 100、次の行を挿入した場合：

#intro bit
sleep 2
sample :ambi_choir, attack: 2, sustain: 4, rate: 0.25, release: 1
sleep 6

...少し音楽的な紹介が表示されます。プログラムは2秒間待機し、サンプル「ambi_choir」を再生してから、さらに6秒間待機してからデータの再生を開始します。メロディ全体に不吉なドラムを少し追加したい場合は、このビットを次のように（自分のデータの前に）配置します。

#bit that keeps going throughout the music
live_loop :boom do
  with_fx :reverb, room: 0.5 do
    sample :bd_boom, rate: 1, amp: 1
  end
  sleep 2
end

コードは非常に単純です。特定の速度でのリバーブのサウンドエフェクトを含むループサンプル 'bd_boom'です。サイクル間の休止は2秒です。

「リアルタイムコーディング」とは、コードを変更しながら、変更を再生できることを意味します。聞いたことが気に入らないのですか？コードをすぐに変更してください！

Sonic Piの探索は、このワークショップから始めることができます。セミナーへの参加に関するローラ・ヴルベルのレポートも参照してください。この領域での彼女の仕事と彼女の同僚の仕事についても説明しています。

太陽の下で新しいものはありません


繰り返しますが、アルゴリズムアプローチにより、私たちは科学の最前線にいると考える必要はありません。1978年、18世紀の「ミュージカルサイコロゲーム」に関する科学記事が発表されました。サイコロのロールが、以前に書かれた曲の組み換えを決定しました。Robin Newman は、Sonic Pi用にこれらのゲームのいくつかを研究およびコーディングしました。ニューマンは楽譜については、Markdown + Pandocとして記述できるツールを使用し、ノートへの変換にはLilypondを使用します。The Programming Historianブログのすべてのトピックには長い歴史があります！

結論


発言すると、私たちのデータは多くの場合、私たちのパフォーマンスの解釈ほどストーリーを反映していないことがわかります。これは、データを音に変換するために必要な斬新で芸術的な性質によるものです。しかし、これは音の解釈を従来の視覚化と大きく区別します。たぶん、生成された音が「音楽」のレベルに上がることはありません。しかし、彼らが過去に対する私たちの理解を変え、他の人々に影響を与えるのを助けるなら、その努力はそれだけの価値があります。Trevor Owensが言うように、「サウンドは、既知のものの正当化ではなく、新しいものの発見です。」
