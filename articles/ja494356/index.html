ワイヤレスネットワークを見つける方法


このパートは、アクセスポイント（AP）で動作する管理対象ネットワークの簡単な紹介です。各アクセスポイントは、毎秒約10個のいわゆるビーコンフレームを送信します。これらのパッケージには、次の情報が含まれています。

  • ネットワーク名（ESSID）
  • 暗号化が使用されているか（およびどのような暗号化が使用されているか。ただし、この情報は、アクセスポイントが報告するためだけではない可能性があることに注意してください）。
  • サポートされているデータレート（MBit）。
  • ネットワークが接続されているチャネル。


この情報は、特にこのネットワークに接続するツールに表示されます。マップでのネットワークのスキャンを許可したiwlist <interface> scan場合、およびairodump-ngを実行した場合に表示されます。

各アクセスポイントには、一意のMACアドレス（48ビット、6組の16進数）があります。これは次のようになります：00：01：23：4A：BC：DE。各ネットワークデバイスにはこのようなアドレスがあり、ネットワークデバイスは互いに助け合って通信します。つまり、それは一種のユニークな名前です。MACアドレスは一意であり、同じMACアドレスを持つ2つのデバイスはありません。

ネットワーク接続


ワイヤレスネットワークに接続するには、いくつかのオプションがあります。ほとんどの場合、オープンシステム認証が使用されます。（オプション：認証の詳細については、こちらをご覧ください。）

オープンシステム認証：

  1. アクセスポイント認証を要求します。
  2. アクセスポイントが応答します。OK、認証されました。
  3. アクセスポイントの関連付けを要求します。
  4. アクセスポイントが応答します。OK、接続されました。


これは最も単純なケースですが、アクセス権がない場合に問題が発生します。

  • WPA / WPA2を使用しており、APOL認証が必要です。アクセスポイントは2番目のステップで拒否します。
  • アクセスポイントには、許可されたクライアント（MACアドレス）のリストがあり、他のユーザーが接続することはできません。これはMACフィルタリングと呼ばれます。
  • Shared Key Authentication, WEP-, . (. « ?» )




最初にすることは、潜在的なターゲットを見つけることです。 aircrack-ngパッケージにはこのためのairodump -ng がありますが、たとえばKismetなどの他のプログラムを使用できます。

ネットワークを検索する前に、カードをいわゆる「監視モード」に転送する必要があります。監視モードは、コンピュータがネットワークパケットをリッスンできるようにする特別なモードです。このモードでは、注射を行うこともできます。次回は注射についてお話します。

ネットワークカードを監視モードにするには、airmon-ngを使用します。

airmon-ng start wlan0


したがって、別のインターフェースを作成し、それに「mon」を追加します。したがって、wlan0はwlan0monになります。ネットワークカードが実際に監視モードになっているかどうかを確認するには、それiwconfigを実行して自分で確認します。

初期化ベクトルスニッフィング


チャネル間のホッピングのため、ターゲットネットワークからのすべてのパケットをインターセプトすることはありません。したがって、後でハッキングに使用できるように、1つのチャネルのみをリッスンし、さらにすべてのデータをディスクに書き込む必要があります。

airodump-ng -c 11 --bssid 00:01:02:03:04:05 -w dump wlan0mon


パラメータを使用し-てチャネルを選択します。その後のパラメータ-wは、ディスクに書き込まれるネットワークダンプのプレフィックスです。このフラグ–bssidは、アクセスポイントのMACアドレスとともに、パケットの受信を1つのアクセスポイントに制限します。このフラグ–bssidは、airodump-ngの新しいバージョンでのみ使用できます。

WEPをハッキングする前に、40,000〜85,000の異なる初期化ベクトル（初期化ベクトル、IV）が必要です。各データパケットには、初期化ベクトルが含まれています。それらは再利用できるため、ベクトルの数は通常、キャプチャされたパケットの数よりもわずかに少なくなります。
したがって、（IVからの）40kから85kのデータパケットを傍受するのを待つ必要があります。ネットワークがビジーでない場合は、非常に長い時間がかかります。アクティブな攻撃（またはリプレイ攻撃）を使用して、このプロセスを高速化できます。次のパートでそれらについて話します。

1つまたは複数のファイルに保存されているインターセプトされた初期化ベクターが既に十分にある場合は、WEPキーを解読することができます。

aircrack-ng -b 00:01:02:03:04:05 dump-01.cap


フラグの後のMACアドレス-bは、ターゲットのBSSIDでありdump-01.cap、キャプチャされたパケットを含むファイルです。複数のファイルを使用できますdump*.cap。たとえば、コマンドにすべての名前を追加するか、ワイルドカードを使用します。マニュアルからaircrack-ng

パラメータ、出力、および使用法の詳細を学ぶことができます。 キーをクラックするために必要な初期化ベクトルの数に制限はありません。これは、一部のベクトルは他のベクトルよりも弱く、重要な情報を失うためです。通常、これらの初期化ベクトルはより強力なものと混合されます。したがって、運が良ければ、わずか20,000の初期化ベクトルでキーを解読できます。しかし、これはしばしば十分ではありません、

aircrack-ngは長期間（エラーが多い場合は1週間以上）機能し、キーが解読できないことを通知します。初期化ベクトルが多いほど、ハッキングが速くなり、通常は数分または数秒で実行されます。ハッキングには、40,000〜85,000のベクターで十分であることが経験上示されています。

特別なアルゴリズムを使用して弱い初期化ベクトルを除外する、より高度なアクセスポイントがあります。その結果、アクセスポイントからNを超えるベクトルを取得できなくなります。または、キーをクラックするために数百万のベクトル（たとえば、500〜700万）が必要になります。あなたはそのような場合に何をすべきかフォーラムで読むことができます。

ARPリプレイ


パケットインジェクションが機能することがわかったので、初期化ベクトルのインターセプトを大幅に加速する何かを実行できます。ARPリクエストのアタックインジェクションです。

本旨


簡単に言うと、ARPはブロードキャスト要求をIPアドレスに送信することで機能し、このIPアドレスを持つデバイスは応答を返します。WEPはリプレイから保護しないため、パケットをスニッフィングして、有効な間は何度でも送信できます。したがって、トラフィックを作成する（および初期化ベクトルを取得する）ために、アクセスポイントに送信されたARP要求を傍受して再生する必要があるだけです。

怠惰な方法


最初にairodump-ngでウィンドウを開き、トラフィックをスニッフィングします（上記を参照）。Aireplay-ngとairodump-ngは同時に動作できます。クライアントがターゲットネットワークに表示されるのを待ち、攻撃を開始します。

aireplay-ng --arpreplay -b 00:01:02:03:04:05 -h 00:04:05:06:07:08 wlan0mon


-bターゲットBSSID、-h接続されたクライアントのMACアドレスを指します。

ここで、ARPパケットを待つ必要があります。通常、数分待つ必要があります（または記事をさらに読みます）。
