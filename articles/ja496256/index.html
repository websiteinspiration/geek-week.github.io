例：ニューラルネットワークを使用したコンクリート強度のモデリング


土木工学の分野では、建築材料の有効性を正確に推定することが非常に重要です。これらの評価は、建物、橋、道路の建設における材料の使用を管理する安全規則を作成するために必要です。

特に興味深いのは、具体的な強度の評価です。コンクリートはほとんどすべての構造で使用されます。コンクリートは、複合体で相互作用する膨大な数の成分で構成されるため、パフォーマンス特性は大きく異なります。結果として、完成品の強さを正確に表現することは困難です。出発材料の組成を考慮に入れて、コンクリートの強度を確実に決定できるモデルは、建設現場により高いレベルの安全性を提供できます。

ステップ1.データ収集


この分析では、I-Cheng YehからUCI Machine Learning Repository（http://archive.ics.uci.edu/ml）に提供された具体的な圧縮強度データを使用します。Ai-Cheng Yeはニューラルネットワークを使用してこのデータをモデル化することに成功したので、Rでニューラルネットワークの単純なモデルを適用して、彼の研究を再現しようとします。

サイトから判断すると、このデータセットには、コンクリートミックスの構成に使用されるコンポーネントを説明する8つの特性を持つコンクリートのさまざまなブランドに関する1030のレコードが含まれています。これらの特性は、最終的な圧縮強度に影響を与えると考えられています。これらには、セメント、水、さまざまな添加剤、完成品に使用された砕石や砂などの大小の骨材の量（立方メートルあたりのキログラム）、および硬化時間（日数）が含まれます。

この例を実行するには、concrete.csvファイルをダウンロードして、R作業ディレクトリに保存します。

ステップ2.調査とデータ準備


通常どおり、read.csv（）関数を使用してデータをRオブジェクトにロードすることで分析を開始し、結果が期待される構造に対応していることを確認します。

> concrete <- read.csv("concrete.csv")
> str(concrete)
'data.frame':        1030 obs. of 9 variables:
$ cement       : num 141 169 250 266 155 ...
$ slag            : num 212 42.2 0 114 183.4 ...
$ ash             : num 0 124.3 95.7 0 0 ...
$ water          : num 204 158 187 228 193 ...
$ superplastic : num 0 10.8 5.5 0 9.1 0 0 6.4 0 9 ...
$ coarseagg    : num 972 1081 957 932 1047 ...
$ fineagg        : num 748 796 861 670 697 ...
$ age             : int 28 14 28 28 28 90 7 56 28 28 ...
$ strength      : num 29.9 23.5 29.2 45.9 18.3 ...

データフレームの9つの変数は、8つの特性と1つの期待される結果に対応していますが、問題があることが明らかになりました。ニューラルネットワークは、入力データが0を中心とする狭い範囲にスケーリングされている場合に最適に機能し、ここでは0から1000を超える範囲の値が表示されます。

通常、この問題の解決策は、正規化または標準化関数を使用してデータをスケーリングすることです。データ分布が釣鐘型の曲線（正規分布、第2章を参照）に対応している場合は、組み込みのscale（）関数を使用した標準化を使用することは理にかなっています。データ分布が均一に近いか、または通常と非常に異なる場合は、0から1の範囲への正規化の方が適している可能性があります。この場合、後者のオプションを使用します。

第3章では、独自のnormalize（）関数を作成しました。

> normalize <- function(x) {
       return((x - min(x)) / (max(x) — min(x)))
}

このコードを実行した後、lapply（）関数を使用して、選択したデータフレームのすべての列にnormalize（）関数を適用できます。正規化が機能し

> concrete_norm <- as.data.frame(lapply(concrete, normalize))

ていることを確認するには、strength属性の最小値と最大値がそれぞれ0と1かどうかを確認します。

> summary(concrete_norm$strength)
       Min.     1st Qu.         Median     Mean      3rd Qu.      Max.
   0.0000     0.2664         0.4001  0.4172      0.5457   1.0000

比較のため：この属性の初期の最小値と最大値はそれぞれ2.33と82.60でした：

> summary(concrete$strength)
     Min.       1st Qu.     Median       Mean      3rd Qu.       Max.
    2.33         23.71       34.44      35.82        46.14      82.60

モデルをトレーニングする前にデータに適用された変換はすべて、逆の順序で適用して、属性を元の単位に戻す必要があります。スケーリングを容易にするために、ソースデータ、または少なくともソースデータの統計の概要を保存することをお勧めします。

元の記事でYeが説明したシナリオに従って、データをすべての例の75％を含むトレーニングセットと、25％で構成されるテストセットに分割します。使用されるCSVファイルはランダムな順序で並べ替えられているため、2つの部分にしか分割できません。 トレーニングデータセットを使用してニューラルネットワークを構築し、テストデータセットを使用して、モデルが将来の結果に対してどれだけ一般化できるかを評価します。ニューラルネットワークは簡単に再トレーニングの状態になるため、この手順は非常に重要です。

> concrete_train <- concrete_norm[1:773, ]
> concrete_test <- concrete_norm[774:1030, ]



ステップ3.モデルをデータでトレーニングする


具体的な製造に使用される成分と完成品の強度との関係をシミュレートするために、多層直接分配ニューラルネットワークを構築します。 Stefan FritschとFrauke Guentherによって開発されたニューラルネットパッケージは、そのようなネットワークの標準的で使いやすい実装を提供します。このパッケージには、ネットワークトポロジを構築するための関数も含まれています。ニューラルネットの実装は、ニューラルネットワークに関する追加情報を取得するための優れた方法ですが、これは実際の作業にも使用できないという意味ではありません。すぐにわかるように、かなり強力なツールです。

R , , . nnet R, , , . , . — RSNNS, , , .

ニューラルネットパッケージはベースRに含まれていないため、install.packages（ "neuralnet"）と入力してインストールし、library（neuralnet）コマンドを使用してダウンロードする必要があります。パッケージのニューラルネット（）関数は、次の構文を使用して数値予測でニューラルネットワークをトレーニングするために使用できます。

ニューラルネットワークの構文


ニューラルネットパッケージの神経網（）関数の使用

モデルの構築：

m <- neuralnet(target ~ predictors, data = mydata,
                       hidden = 1, act.fct = "logistic")

•target-mydataデータフレームのトレーニングの結果として構築されるモデル。

•予測子-予測に使用されるmydataデータフレームから特性を決定するR式。

•データ-ターゲットと予測子が属するデータフレーム。

•hidden-非表示層のニューロンの数（デフォルトは1）。注：いくつかの非表示層を説明するために、整数のベクトルが使用されます。たとえば、c（2、2）です。

•act.fct-アクティベーション関数：「ロジスティック」または「tanh」。注：他の微分可能な関数も使用できます。

この関数は、予測に使用できるニューラルネットワークオブジェクトを返します。

予測：

p <-計算（m、テスト）

•m-ニューラルネット（）関数を使用してトレーニングされたモデル。

•test-分類器の構築に使用されるトレーニングデータと同じ特性を持つテストデータを含むデータフレーム。

この関数は、2つのコンポーネントで構成されるリストを返します。$ニューロン（ニューロンは各ネットワークレイヤーに格納されます）と$ net.result（このモデルを使用して予測された値が格納されます）。

例：



concrete_model <- neuralnet(strength ~ cement + slag + ash,
      data = concrete, hidden = c(5, 5), act.fct = "tanh")
model_results <- compute(concrete_model, concrete_data)
strength_predictions <- model_results$net.result

最初に、デフォルトパラメータを使用した最も単純なマルチレベル直接配信ネットワークをトレーニングします。これには、非表示ノードが1つだけあります。

> concrete_model <- neuralnet(strength ~ cement + slag
         + ash + water + superplastic + coarseagg + fineagg + age,
         data = concrete_train)

そして、図に示すように。7.11、関数plot（）を使用してネットワークトポロジを視覚化し、結果のモデルオブジェクトに渡すことができます。

> plot(concrete_model)


この単純なモデルでは、8つのフィーチャのそれぞれに1つの入力ノードがあり、次に1つの非表示ノードと1つの出力ノードがあり、コンクリート強度の予測を提供します。この図には、各接続の重みと、番号1でマークされたノードに示されるオフセット値も示されています。オフセット値は、ほぼ線形方程式のシフトのように、指定したノードの値を上下にシフトできる数値定数です。

1つの非表示ノードを持つニューラルネットワークは、第6章で説明した線形回帰モデルの「いとこ」と見なすことができます。入力ノードと非表示ノード間の重みはベータ係数に似ており、オフセットの重みはシフトのようです。

図の下部に、トレーニングステップの数とエラーの大きさが表示されます。合計平均二乗誤差（二乗誤差の合計、SSE）は、予想どおり、予測値と実際の値の差の二乗の合計です。SSEが小さいほど、モデルはトレーニングデータにより正確に対応します。これは、これらのデータの有効性を示しますが、モデルが未知のデータでどのように機能するかについてはほとんど説明しません。

ステップ4.モデルの有効性を評価する


ネットワークトポロジ図は、ニューラルネットワークのブラックボックスを垣間見ることができますが、モデルが将来のデータとどの程度一致するかについてはあまり情報を提供しません。テストデータセットの予測を生成するには、compute（）

> model_results <- compute(concrete_model, concrete_test[1:8])

関数を使用できます。compute（）関数の動作は、これまでに使用したpredict（）関数とは少し異なります。これは、2つのコンポーネントで構成されるリストを返します。ニューロンがネットワークレイヤーごとに格納される$ニューロンと、予測値が格納される$ net.resultです。必要なのは$ net.resultです。

> predicted_strength <- model_results$net.result

分類ではなく数値予測のタスクがあるため、不整合のマトリックスを使用してモデルの精度を検証することはできません。コンクリート強度の予測値と真値の間の相関を測定します。予測値と実際の値が強く相関している場合、おそらく、モデルはコンクリートの強度を決定するのに役立ちます。

2つの数値ベクトル間の相関関係を取得するには、cor（）関数を使用することを思い出してください。

> cor(predicted_strength, concrete_test$strength)
                    [,1]
[1,] 0.8064655576

あなたの結果が私たちのものと異なっていても心配しないでください。ニューラルネットワークはランダムな重みで動作を開始するため、本に示されている予測はモデルによって異なる場合があります。結果を正確に一致させたい場合は、ニューラルネットワークの構築を開始する前にset.seed（12345）コマンドを試してください。

相関が1に近い場合、これは2つの変数間の強い線形関係を示しています。したがって、約0.806の相関は、かなり強い関係を示しています。これは、モデルが単一の非表示ノードでも非常にうまく機能することを意味します。隠しノードを1つだけ使用した場合、モデルの効率を改善できる可能性があります。これを試みます。

ステップ5.モデルの効率を改善する


より複雑なトポロジを持つネットワークは、より複雑な概念を研究できるため、非表示ノードの数を5に増やすとどうなるかを見てみましょう。以前と同様に、neuralnet（）関数を使用しますが、hidden = 5パラメーターを追加します。

> concrete_model2 <- neuralnet(strength ~ cement + slag +
                                               ash + water + superplastic +
                                               coarseagg + fineagg + age,
                                               data = concrete_train, hidden = 5)

ネットワーク図を再構築すると（図7.12）、接続数が急激に増加します。これは効率にどのように影響しましたか？

> plot(concrete_model2)

結果のエラー（再びSSEとして測定）は、以前のモデルの5.08から1.63に減少したことに注意してください。さらに、トレーニングステージの数が4882から86,849に増加しました。これは、モデルがいかに複雑であるかを考えると、当然のことです。ネットワークが複雑になるほど、最適な重みを見つけるために必要な反復が多くなります。

同じステップを適用して予測値と実際の値を比較すると、約0.92の相関が得られます。これは、1つの非表示ノードがあるネットワークの以前の結果である0.80と比べてはるかに優れています。

> model_results2 <- compute(concrete_model2, concrete_test[1:8])
> predicted_strength2 <- model_results2$net.result
> cor(predicted_strength2, concrete_test$strength)
                  [,1]
[1,] 0.9244533426


大幅な改善にもかかわらず、モデルの有効性をさらに高めることができます。特に、追加の隠れ層を導入し、ネットワークアクティベーション機能を変更することが可能です。これらの変更を行うことにより、シンプルなディープニューラルネットワークを構築するための基礎を築いています。

活性化機能の選択は、ディープラーニングにとって非常に重要です。特定の学習タスクに最適な関数は、通常、実験的に発見され、機械学習研究者のコミュニティで広く使用されています。

最近、蒸留機能、または整流器と呼ばれる活性化機能は、画像認識などの複雑なタスクでのその成功した応用のために非常に人気があります。整流器が活性化関数として使用されるニューラルネットワークノードは、整流線形ユニット（ReLU）と呼ばれます。図に示すように。 7.13では、整流器タイプのアクティブ化関数は、xが0以上の場合はxを返し、それ以外の場合は0を返すように記述されています。この関数の重要性は、一方では非線形であり、もう一方では単純な数学的特性を備えているため、計算が安価で勾配降下に非常に効率的です。残念ながら、x = 0の場合、整流器の導関数は定義されていません。したがって、整流器をニューラルネット（）関数と組み合わせて使用​​することはできません。

代わりに、softplusまたはSmoothReLUと呼ばれる平滑化されたReLU近似を使用できます。これは、log（1 + ex）として定義されるアクティブ化関数です。図に示すように。7.13、softplus関数は、x値が0未満の場合はゼロに近く、xが0より大きい場合はxにほぼ等しくなります。


Rでsoftplus（）関数を定義するには、次のコードを使用します。

> softplus <- function(x) { log(1 + exp(x)) }

このようなアクティブ化関数は、act.fctパラメーターを使用して入力ニューラルネット（）に提供できます。さらに、5つのノードで構成される2番目の非表示レイヤーを追加し、非表示パラメーターに整数ベクトルc（5、5）の値を割り当てます。その結果、2層のネットワークが得られ、各層には5つのノードがあり、それらすべてがsoftplusアクティベーション機能を使用します。

> set.seed(12345)
> concrete_model3 <- neuralnet(strength ~ cement + slag +
                                               ash + water + superplastic +
                                               coarseagg + fineagg + age,
                                               data = concrete_train,
                                               hidden = c(5, 5),
                                               act.fct = softplus)

以前と同様に、ネットワークを視覚化できます（図7.14）。

> plot(concrete_model3)


コンクリートの予測強度と実際の強度の相関は、次のように計算できます。

> model_results3 <- compute(concrete_model3, concrete_test[1:8])
> predicted_strength3 <- model_results3$net.result
> cor(predicted_strength3, concrete_test$strength)
                  [,1]
[1,] 0.9348395359

予測された強度と実際の強度の間の相関は0.935で、これはこれまでに得られた最良の指標です。興味深いことに、元の出版物では、イェは0.885の相関を報告しました。つまり、比較的少ない労力で、同等の結果を得ることができ、この分野の専門家の結果をも上回ることができました。確かに、あなたがたの結果は1998年に発表され、ニューラルネットワークの分野で20年以上にわたる追加の研究を先導してくれました。

モデルをトレーニングする前にデータを正規化したので、もう1つの重要な詳細を考慮する必要があります。予測も0から1までの正規化された間隔内にあります。たとえば、次のコードは、初期データセットの具体的な強度値を対応する予測とラインごとに比較するデータフレームを示しています。

> strengths <- data.frame(
      actual = concrete$strength[774:1030],
      pred = predicted_strength3
   )
> head(strengths, n = 3)
      actual        pred
774 30.14 0.2860639091
775 44.40 0.4777304648
776 24.50 0.2840964250


相関関係を調べると、正規化または異常化されたデータの選択が、計算されたパフォーマンス統計に影響を与えないことが わかります。以前と同様に、相関関係は0.935です。スケールの選択は非常に重要です。 これを念頭に置いて、ミニマックス正規化の逆を実行し、正規化された予測を元のスケールに変換できるunnormalize（）関数を作成できます。

> cor(strengths$pred, strengths$actual)
[1] 0.9348395359





> unnormalize <- function(x) {
     return((x * (max(concrete$strength)) -
           min(concrete$strength)) + min(concrete$strength))
   }

私たちが書いたunnormalize（）関数を予測に適用した後、新しい予測のスケールがコンクリート強度の初期値に類似していることが明らかになります。これにより、絶対誤差の意味のある値を計算できます。さらに、異常な強度値と初期強度値の間の相関は変化しません：

> strengths$pred_new <- unnormalize(strengths$pred)
> strengths$error <- strengths$pred_new — strengths$actual
> head(strengths, n = 3)
           actual                pred             pred_new                    error
774          30.14         0.2860639091               23.62887889      -6.511121108
775          44.40         0.4777304648               39.46053639      -4.939463608
776          24.50         0.2840964250               23.46636470      -1.033635298

サポートベクターマシン（SVM）メソッドは、属性の例と値を記述する多次元空間にプロットされたデータポイント間の境界を形成する表面として表すことができます。 SVMの目的は、平らな境界線を作成することです。つまり、空間の両側に同種のグループが形成されるように空間を分割する超平面です。したがって、SVMトレーニングは、第3章で説明したインスタンスに基づく最近隣トレーニングと第6章で説明した線形回帰モデリングの両方の側面を組み合わせたものです。これは、SVMが非常に複雑な関係をモデル化できる非常に強力な組み合わせです。

SVMの基礎となる基本的な数学が何十年も存在しているという事実にもかかわらず、これらの方法はMLに適用され始めて以来、これらの方法への関心が大幅に高まっています。これらの方法の人気は、複雑な学習問題の解決における有名なサクセスストーリーの後、およびRを含む多くのプログラミング言語でサポートされているライブラリで授与および実装されたSVMアルゴリズムの開発の後に増加しました。その後、SVMメソッドは幅広い聴衆に受け入れられました。そうでなければ、SVMの実装に必要な複雑な計算を適用することはおそらく不可能でしょう。良いニュースは、数学はおそらく複雑ですが、基本的な概念は理解できるということです。

SVMメソッドは、分類や数値予測など、ほぼすべてのタイプのトレーニングタスクを使用するように適合させることができます。このアルゴリズムの主要な成功の多くは、パターン認識に関連しています。これらの方法の最もよく知られているアプリケーションは次のとおりです。

  • 文書で使用される言語の決定、トピックによる文書の分類などのテキスト分類。
