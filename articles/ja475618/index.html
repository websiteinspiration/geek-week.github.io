今日は、2番目のモジュールであるStorage APIの使用方法について説明します。Storage APIには、ブロックチェーンのスマートコントラクトの永続ストレージを追加、削除、変更できる5つの関連APIがあります。

以下は、これら5つのAPIの簡単な説明です

1. Storage APIの使い方


GetContextおよびGetReadOnlyContext


GetContextおよびGetReadOnlyContextは、現在のスマートコントラクトが実行されるコンテキストを取得します。戻り値は、スマートコントラクトの現在のハッシュの逆です。名前が示すように、GetReadOnlyContextは読み取り専用モードのコンテキストを取ります。以下の例では、戻り値は右上隅に表示される契約ハッシュの逆です。



プット


Put 関数は、ブロックチェーンにデータを辞書の形で格納します。示されているように、Putは3つのパラメーターを取ります。GetContextは、現在実行中のスマートコントラクトのコンテキストを取得します。keyは、データを保存するために必要なキー値であり、valueは、保存する必要があるデータの値です。キー値がすでにリポジトリにある場合、関数は対応する値を更新することに注意してください。

取得する


Get 関数は、キー値を使用して現在のブロックチェーンのデータを読み取ります。以下の例では、右側のパラメーターパネルにキー値を入力して関数を実行し、ブロックチェーン内のキー値に対応するデータを読み取ることができます。



削除する


Delete 関数は、キー値を使用してブロックチェーン内のデータを削除します。以下の例では、キー値を入力して、右側のパラメーターパネルで関数を実行し、ブロックチェーンのキー値に対応するデータを削除できます。



2.サンプルストレージAPIコード


以下のコードは、GetContext、Get、Put、Delete、GetReadOnlyContextの5つのAPIの使用の詳細な例を示しています。SmartXで APIデータを実行してみてください。

from ontology.interop.System.Storage import GetContext, Get, Put, Delete, GetReadOnlyContext
from ontology.interop.System.Runtime import Notify

def Main(operation,args):
    if operation == 'get_sc':
        return get_sc()
    if operation == 'get_read_only_sc':
        return get_read_only_sc()
    if operation == 'get_data':
        key=args[0]
        return get_data(key)
    if operation == 'save_data':
        key=args[0]
        value=args[1]
        return save_data(key, value)
    if operation == 'delete_data':
        key=args[0]
        return delete_data(key)
    return False

def get_sc():
    return GetContext()
    
def get_read_only_sc():
    return GetReadOnlyContext()

def get_data(key):
    sc=GetContext() 
    data=Get(sc,key)
    return data
    
def save_data(key, value):
    sc=GetContext() 
    Put(sc,key,value)
    
def delete_data(key):
    sc=GetContext() 
    Delete(sc,key)
