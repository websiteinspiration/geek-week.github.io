この記事は、Spring Boot Microservicesシリーズの一部です。

  • パート1.マイクロサービスアーキテクチャの概要

このシリーズの記事では、マイクロサービスの概念を理解し、Spring BootとSpring Cloudを使用してマイクロサービスを作成する方法を学びます。

このガイドは、マイクロサービスアーキテクチャの基本を学ぶのに役立ちます。また、Spring Bootを使用した基本的なマイクロサービスの実装についても見ていきます。

Spring Initializrを利用して、さまざまなプロジェクトを作成できます。

ユーレカをオンにする


SpringBootMicroserviceEurekaNamingServerApplicationで EurekaServerを有効にします。

@SpringBootApplication
@EnableEurekaServer
public class SpringBootMicroserviceEurekaNamingServerApplicatio

Eurekaサーバーのアプリケーション名とポートを構成します。

/spring-boot-microservice-eureka-naming-server/src/main/resources/application.properties


spring.application.name=netflix-eureka-naming-server
server.port=8761
eureka.client.register-with-eureka=false
eureka.client.fetch-registry=false

ユーレカネームサーバーの起動


SpringBootMicroserviceEurekaNamingServerApplicationをJavaアプリケーションとして 起動します。

実行中のEurekaサーバーのステータスは、
http://localhost:8761

まだ1つのインスタンスがEurekaに接続されていないことがわかります。



FSおよびCCSマイクロサービスをユーレカに接続する


両方のマイクロサービスに対して次の変更を行います。

  • 次の依存関係をpom.xmlに追加します。

    <dependency>
	<groupId>org.springframework.cloud</groupId>
	<artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
</dependency>

  • application.propertiesでEureka URLを構成します。
    eureka.client.service-url.default-zone=http://localhost:8761/eureka

CCSおよびFSのすべてのインスタンスを再起動します。CCSおよびFSマイクロサービスがEurekaネームサーバーに登録されていることがわかります。これはカッコいい！

動作中のユーレカ


現在、次のサービスがあります。

  • 8100の通貨換算マイクロサービス（CCS）
  • 8000および8001上のForexマイクロサービスの2つのインスタンス
  • Eurekaサーバーが実行中です

これで、CCSリクエストが、Eureka経由のリボンを使用してForexマイクロサービスの2つのインスタンス間で分散されることがわかります。

リクエスト1


GET to http://localhost:8100/currency-converter-feign/from/EUR/to/INR/quantity/10000

  port: 8000,

2つの回答のポート番号が異なることに気付くでしょう。

演習：8002でForexサービスの別のインスタンスを実行します。負荷が自動的にリダイレクトされることがわかります。
