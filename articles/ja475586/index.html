AudioContext APIによるコンピュータのサウンドフィンガープリント



インターネット上でのユーザーアクティビティを追跡する企業は、ユーザーの知らないうちに各人を確実に識別する必要があります。ブラウザを介したフィンガープリントは完全に適合します。Webページがキャンバスを介してグラフィックフラグメントを描画するように要求したり、応答パラメータを測定してボリュームがゼロのサウンド信号を生成したりしても、誰も気付かないでしょう。

このメソッドは、Torを除くすべてのブラウザでデフォルトで機能します。ユーザー権限は必要ありません。

総追跡


最近、NYタイムズのジャーナリスト、カシミールヒルは、Siftと呼ばれる特定のあまり知られていない会社が400ページの書類を蓄積していたことを発見しました。数年間の買い物リスト、Airbnbのホストへのすべてのメッセージ、携帯電話のCoinbaseウォレット起動ログ、IPアドレス、iPhoneピザの注文などがあります。同様の収集が複数の採点会社によって行われています。各ユーザーの「信頼度」をコンパイルする場合、最大16,000の要素が考慮されます。 Siftトラッカーは、34,000のサイトとモバイルアプリケーションにインストールされます。

Cookieとスクリプトの追跡は常に適切に機能しないか、クライアントで無効になるため、ユーザーの追跡はフィンガープリントによって補完されます。これは、ブラウザー/システムの一意の「フィンガープリント」を取得するための一連のメソッドです。インストールされているフォント、プラグイン、画面解像度、およびその他のパラメータのリスト全体で、一意のIDを取得するのに十分な情報が得られます。キャンバスを介したフィンガープリントは適切に機能します。

指紋テスト

最近、パラメータのセットに加えて、もう1つ追加されました。AudioContextAPIを介した音の指紋です。

2016年には、この識別方法はExpedia、Hotels.comなどの数百のサイトで既に使用されていました。

AudioContext APIによるフィンガープリント


アクションのアルゴリズムは同じです。ブラウザがタスクを実行し、実行の結果を記録して一意のハッシュ（指紋）を計算します。この場合のみ、データはオーディオスタックから抽出されます。 Canvas APIの代わりに、AudioContext APIにアクセスしています。これは、すべての最新のブラウザーがサポートするWeb Audio APIです。

ブラウザは低周波数のオーディオ信号を生成します。これは、デバイスにインストールされているサウンド設定と機器を考慮して処理されます。この場合、音声は記録または再生されません。スピーカーとマイクは関与していません。

このフィンガープリンティング方式の利点は、ブラウザーに依存しないため、ChromeからFirefox、Operaなどに切り替えた後でもユーザーを追跡できることです。

AudioContext APIによる指紋テスト指紋



を取得する方法、ステップバイステップガイド：

  1. 最初に、頻度値を格納する配列を作成する必要があります。

     let freq_data = [];

  2. 次に、AudioContextオブジェクトとさまざまなノードが作成され、AudioContextオブジェクトの組み込みメソッドを使用して信号を生成し、情報を収集します。

     // Create nodes
const ctx = new AudioContext(); // AudioContext Object
const oscillator = ctx.createOscillator(); // OscillatorNode
const analyser = ctx.createAnalyser(); // AnalyserNode
const gain = ctx.createGain(); // GainNode
const scriptProcessor = ctx.createScriptProcessor(4096, 1, 1); // ScriptProcessorNode

  3. ボリュームをオフにして、ノードを相互に接続します。

     // Disable volume
gain.gain.value = 0;
// Connect oscillator output (OscillatorNode) to analyser input
oscillator.connect(analyser);
// Connect analyser output (AnalyserNode) to scriptProcessor input
analyser.connect(scriptProcessor);
// Connect scriptProcessor output (ScriptProcessorNode) to gain input
scriptProcessor.connect(gain);
// Connect gain output (GainNode) to AudioContext destination
gain.connect(ctx.destination);

  4. を使用してScriptProcessorNode、サウンド処理中に周波数データを収集する関数を作成します。
    • 関数は、Float32Arrayデータ値の数（頻度）に等しい長さの型付き配列を作成しAnalyserNode、それに値を入力します。
    • 次に、これらの値を以前に作成した配列（freq_data）にコピーして、出力に簡単に書き込むことができるようにします。
    • ノードをオフにして、結果を表示します。

       scriptProcessor.onaudioprocess = function(bins) {
// The number of (frequency) data values
bins = new Float32Array(analyser.frequencyBinCount);
// Fill the Float32Array array of these based on values
analyser.getFloatFrequencyData(bins);
// Copy frequency data to 'freq_data' array
for (var i = 0; i < bins.length; i = i + 1) {
freq_data.push(bins[i]);
}
// Disconnect the nodes from each other
analyser.disconnect();
scriptProcessor.disconnect();
gain.disconnect();
// Log output of frequency data
console.log(freq_data);
};

  5. 音色を再現し始め、機能に応じて音を発生させて処理します。

     // Start playing tone
oscillator.start(0);
