DbToolを使用して.NETアプリケーションにデータベースをシードする（コア）

DbTool-データベースデータをさまざまな形式にエクスポートするためのコマンドラインユーティリティと、Korzh.DbUtilsオープンソースライブラリをご紹介します。これらを使用すると、.NET（コア）アプリケーションでのデータベースの初期「播種」を大幅に簡略化できます。


このツールキットを使用すると、次のことができます。


  1. ローカルデータベースのデータを、プロジェクトに簡単に接続できる特定のテキスト形式（XML、JSON）のファイルに保存します。
  2. 保存されたファイルを使用して、最初の起動時にアプリケーション自体のデータベースを作成します。

以下では、なぜこれがすべて必要なのか、これらのツールをインストールして構成する方法、およびそれらを使用するための詳細なシナリオについて説明します。



なぜDbToolを使ったのですか


最初のタスクは、.NET（コア）アプリケーションで便利なデータベースフィルメカニズムを作成することでした。仕様に起因する活動の私達の種類（部品開発）、我々は、多くの場合、1つまたは当社の製品の別の特徴を示し、小さなサンプルアプリケーションを作成する必要があります。このようなデモプロジェクトは特定のテストデータベースで動作するはずなので、アプリケーションの最初の起動時にこのデータベースを自動的に作成してデータを投入することをお勧めします。

    using (var context = scope.ServiceProvider.GetService<AppDbContext>()) {
        if (context.Database.EnsureCreated()) { //run only if database was not created previously
            Korzh.DbUtils.DbInitializer.Create(options => {
                options.UseSqlServer(Configuration.GetConnectionString("DefaultConnection")); //set the connection string for our database
                options.UseFileFolderPacker(System.IO.Path.Combine(env.ContentRootPath, "App_Data", "SeedData")); //set the folder where to get the seeding data
            })
            .Seed();
        }
    }
