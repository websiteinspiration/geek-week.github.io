CP / M：ゲイリーキルドールの興亡

技術の歴史は驚きと鋭い変化に満ちています。勝利と敗北、裏切りと忠実さ。時々、成功するために、あなたはちょうど適切な時に適切な場所にいなければなりませんでした。または、フレンドリーなアドバイスを受けます。



今日私たちは、彼が彼の人生でもう少し運が良ければ、ビル・ゲイツのレベルの有名人である多億万長者である可能性のある男について話します。この男の名前はゲイリーキルドールでした。



あなたはこの名前を知らないかもしれませんが、CP / Mオペレーティングシステムというその発想を見つけたり、使用したりする可能性が高いです。最近のPCユーザーにはほとんど知られていませんが、このOSは、ZX Spectrum（およびそのCISクローン）、Robotron、および「プロプライエタリ」なCommodore 128（オンボードコプロセッサとしてのZilog Z80のおかげ）など、数十の異なるマシンで広く使用されていました。その年にCP / Mを実行しているコンピュータを使用した人の多くは、それがどこから来たのかさえ考えていませんでした。







早い時期



キルドールのキャリアはカリフォルニア州モントレーで始まりました。彼は海軍大学院で数学を教えた。正確な科学に対する彼の好みは、彼をプログラミングに駆り立てました。奇妙なことに、そのような活動の急激な変化の理由は広告にありました。 Kildallは、Intel 4004ベースの4ビット「マイクロコンピュータ」の販売の発表に衝撃を受けました。わずか25ドルで、デバイスを購入しました。マシンの可能性は小さく、最初はマイクロプロセッサ4004が計算機で使用されるはずでしたが、キルドールがコンピュータに没頭してプロセッサの作成者と会うことさえできました。



すでに1972年に、彼はコンピューターサイエンスの博士論文を擁護し、インテルのコンサルタントとして就職しました。質の高い学術教育、複雑なタスクへの愛情、そして起業家精神により、キルドールは可能な限り早く驚くべき結果を達成することができました。



目撃者によると、彼はコンピューターへの情熱的な愛好家で、かつて彼の息子にロゴのプログラミング方法を教えることを決心しました。手元に適切な通訳がいなかったため、キルドールが自分で作成しました。



1970年代初頭、KildallはPL / Mプログラミング言語、Programming Language for Microcomputersを開発しました。それは当時非常に人気のあったPL / 1に基づいていました。プログラムの「高レベル」コードは、IBM-360のメインフレームに最初に記述され、その後、目的のプロセッサのマシンコードに変換されると想定されていました。 PL / Mの後のバージョンはすでにIntelの最初のコンピュータであるIntellec-4 ROMに含まれていました。







Intelとの長年の協力の間に彼に思いついたキルダルのもう1つの注目すべきアイデアは、Intel 8008（Intellec-8）を備えたマイクロコンピューターの「キット」に、モニター、キーボード、パンチリーダーだけでなく、Shugartフロッピーディスクドライブも含めることでした。大学の友人と一緒に、1973年にキルダルはドライブコントローラーを開発し、うまく適用しました。ちなみに、あるバージョンによれば、「フロッピードライブ」という単語の出現は、その機知に富んだキルドールのおかげです。



デザインと運命CP / M



1970年代には、コンピュータ用のソフトウェアが個別に開発されました。そのようなアプローチは、進取的で野心的なプログラマーを悩ませざるを得なかったと推測するのは簡単です。 Intellec-8はいくつかの技術的な調整のおかげで操作がはるかに便利になりましたが、それを本格的なマシンに変えるには、オペレーティングシステムが必要でした。基本は以前の開発でした-PL / M



Kildallは、かつて作成されたソフトウェアのクロスプラットフォームおよび広範な使用を最前線に置きました。そして彼は誰とも競争しようとはしませんでした。その年、マイクロプロセッサは主にテクノロジーを制御するために使用され、その上に本格的なコンピュータを構築するという考えは新しくて興味深いように見えました。 Intelの立場は非常に奇妙に思われる：彼らは喜んでPL / MをKildallから買った、そして彼らはCP / Mに完全に無関心のままであった。おそらく、Intellec-8が十分な人気を得て、購入に利益をもたらすとは期待していなかったからでしょう。 Intelは独自の方法でマイクロプロセッサの生産市場をしっかりと占領し、KildallはOSの開発を継続して独自に販売することを決定しました。



キルドールの同時代の人たちは、彼の将来の運命を決定づけた要因の1つは年齢だったと書いています。平均して、彼はPC時代の創始者として現在私たちが知っている多くの人よりも10〜15歳年上でした。彼は彼の世代からノックアウトされましたが、「若者」の中に志を同じくする人々を見つけることができませんでした。



1976年に妻のドロシーと一緒に、銀河系デジタル研究会社を設立しました。そのような有名な名前にもかかわらず、当初はスタッフは2人だけで構成され、「本社」はカリフォルニア州のキルダルズハウスにありました。 1977年、同社はDigital Research Inc.に社名変更されました。 （DRI）。





ゲイリー・キルドールと妻ドロシー



大成功でした。 CP / Mのコピーはホットケーキよりも早く分岐し、Kildallの会社は独占でさえ非難されました。 1970年代の終わりまでに、最大の市場プレーヤーのマイクロコンピューターは、マイトとメインでCP / Mを使用していました。その中には、IMSAI 8080、ノーススター、オズボーンが含まれていました。同時に、道路の交通量データを収集したビルゲイツとポールアレンのTraf-O-Dataの小さな会社も、CP / Mを積極的に使用したことは注目に値します。一部のレポートによると、1980年までに、CP / Mの25万部以上が販売されました。







IMSAIは個別に言及する価値があります。当時、この会社はほぼ独占的にAltairのクローンを市場に供給していました。 1977年に、彼らはキルダルからCP / Mライセンスを購入し、これがさらに市場での流通に貢献しました。 Kildallは必然的に繁栄し、金持ちになり、そのオペレーティングシステムを開発しました。そのため、最初にプリエンプティブマルチタスクを導入したのはCP / Mで、専用のBIOSが登場しました。 Kildallが開発したアルゴリズムと原則の1つまたは複数が今日まで使用されています。





Altair 8800のCP / M



一方、マイクロコンピュータの時代はすでに終わりに近づいています。それらは徐々にパソコンに置き換わり始めました。 1976年にスティーブウォズニアックなどがリリースしたApple Iから始まりました。わずか3年で、PC市場は10億ドルに「膨らみ」、IBMがついに登場しました。







米国で最大の技術企業の経営陣は非常によく理解していました。市場は若者に属しており、勝つための唯一の方法は、ユーザーに全体的、迅速、便利な何かを与えることです。多くの影響力のあるIBMの人々と短い間足を踏み入れていたGatesは、Gary KildallのOSに注意を払うことを勧めました。しかし経営陣IBMとMicrosoftは秘密保持契約に署名し、Microsoftの従業員は会話でIBMについて言及する権利すらなかったとしています。



記事の冒頭で書きました。適切なタイミングで適切な場所にいるだけの価値がある場合もあります。 CP / Mの「死」とMS-DOSの昇格を間接的に説明する伝説の1つをお伝えします。確かにそうだとは言えません。それぞれの当事者が独自の方法でストーリーを解釈するため、反対側が勝った場合に世界がどうなるかについて推測するよりも、現実を受け入れる方が簡単です。



伝説によると、ゲイツはキルダルを呼び、陰謀のささやきの中で、彼は非常に重要なゲストを期待する必要があると近い将来に言った。非常に重要な講演のために。 IBMが彼に世紀の申し出をするつもりであると直接言ったが、彼はできなかった。 NDAでは、エンドカスタマーについて言及することさえできませんでした。ゲイツは、洗練されたキルダルがそれが誰であるかを理解すると信じていました。しかし、どうやら、ゲイリーは懐疑的な同僚の言葉に反応しました。したがって、IBMのゲストが家の敷居に現れた日、彼はすべての事柄を妻に任せて、静かに私用飛行機で飛んだ。



ドロシーとDRI法務部は訪問者に会い、CP / Mをできる限り提示しました。どういうわけか、彼らはIBMに印象づけなかったか、少なくともIBMに興味を示しませんでした。ストーリーはキルダルの妻と彼の従業員も不愉快な秘密保持契約に署名しなければならず、契約自体は失敗に終わりました。確かに、当時のキルドール自身は16ビットバージョンのCP / Mを持っていなかったため、IBMを完全に満足させることはできませんでした。



これが、Microsoftの最高の時が来た場所です。ビルゲイツは肩をすくめて、この機会をつかみました。はい、彼の会社にはまだ当時独自のOSがありませんでした。しかし、これはそのような大当たりを失う価値があることをまったく意味しませんでした。ビルのゲイツは、彼自身の功績もあり、母親のつながりもあり、最初のIBMパーソナルコンピュータ用のオペレーティングシステムを供給する契約を結んだ。



ゲイツ氏は次に、シアトルコンピュータに問い合わせ、キルダルOSクローンをリリースした。同社の創設者であるティムパターソンは、CP / M-86-DOSの彼自身のバージョンを開発しました。彼が後で述べたように、元のCP \ Mをコピーするには、公式ドキュメントからAPI呼び出しをコピーするだけで十分だったので、彼は違法なものをコミットしませんでした。システムには多くのバグと欠陥があったため、内部の俗語ではQ-DOS（Quick and Dirty Operating System）と呼ばれていました。システムはPC-DOSで「名前が変更され」、IBMから提供されました。将来、それに基づいて、すべての読者に知られているMS-DOSが作成されました。この記事で、CP / Mとビルゲイツに関する陰謀の複雑さについて詳しく読むことができます。



ゲイリーは激怒した。彼自身の不注意のために、彼はパソコンの世界のオリンパスに登る素晴らしい機会を逃しました。神でさえ彼を祝福し、オリンパスで-彼からOSの16ビットバージョンを待って正直に購入する代わりに、Microsoftは直接の競争相手に頼りました。



IBMは、著作権をめぐる熱狂的なスキャンダルを見越して、賢明な行動をとった。どちらのOSもIBM PCで動作するようにライセンスされています：PC-DOSとCP / M、そして勝者は市場を選択することでした。







Kildallの次の大流行は、IBMがコンピューターの売り込みを始めたときでした。広告に示されている価格によると、PC-DOSは40ドルで販売され、公式ディストリビューターのCP / M Kildallはすでに240ドルでした。噂によると、価格の6倍の違いは、キルダルがその運命的な会議に現れなかった方法を「思い出す」ための意図的な試みでした。



DRIでは、停滞が始まっています。しばらくの間、会社は標準に達していなくてもとどまろうとし、Microsoftの背後にある2つの訴訟を避けました。 Kildallは彼自身のグラフィカルインターフェイスであるGEMも開発しましたが、Windowsのリリースの前に、Microsoftは彼に開発を完全に販売する機会を与えませんでした。







Kildallの人生は、IBMとの契約に失敗した後、劇的に変化しました。次の10年間で、システムは並行して進化し、相互に互換性がありました。そのため、DR-DOS（1987年以降の新しい名前CP / M）が引き続き販売されました。しかし、1991年にWindows 3.1が登場しましたが、これはKildallのDR-DOSとの互換性がなくなりました。わずか1年で、売上高は桁違いに減少しました。 1991年、キルダルはノベルの会社を売却し、マイクロソフトの傘下での反撃を期待した。しかし、この試みは失敗しました。NovellDOSはMS-DOSおよびWindowsと競合できなくなりました。



キルドールの人生には、新しい仲間-アルコールがあります。彼はプロの連絡先をすべて失い、人気のテレビ番組「コンピュータークロニクルズ」の共催者としての出演をやめました。



キルダルの人生の最後の数ヶ月についてはあまり知られていない。彼はまだ裕福な男ではなかった。彼はスポーツカー、不動産の大規模なコレクションを所有していましたが、彼が長年住んでいたテクノロジーへの関心と情熱はなくなりました。

1994年7月、52歳のとき、彼はバイカーバーでの戦闘中に不思議な状況で亡くなりました。2016年、ゲイリーの子供たちの許可を得て、回想録の一部が出版されました。



あとがき



歴史は皆を彼らの場所に置きました。誰かが不明で忘れられています。誰かは億万長者であり、慈善家であり、世界で最も切望されている秘密のサンタです。



Gary Kildallは、今日私たちがコンピュータを知っている方法でコンピュータを作るために多くのことをしました。このブログでお話ししたいたくさんの人の一人。



たとえ市場での戦争に負けたとしても、才能のある人々のメリットを軽視することはできません。そして、勝者が彼らの失敗について非難されることは決してありません。
