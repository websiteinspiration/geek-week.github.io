PowerShell 7.0の紹介

本日、PowerShell 7.0のパブリック（GA）バージョンのリリースを発表し  ます！まず、コード、テスト、ドキュメント、および問題に関するフィードバックを共有することにより、このリリースを可能にしてくれた多くの貢献者に感謝します  。PowerShell 7は、あなたの助けなしには不可能でした。



PowerShell 7とは何ですか？


初心者にとって、PowerShell 7はPowerShellの最新のメジャーアップデートであり、構造化データ（JSON、CSV、XMLなど）を操作するために最適化されたクロスプラットフォーム（Windows、Linux、およびmacOS）自動化ツールと構成インフラストラクチャ、REST APIです。およびオブジェクトモデル。 PowerShellには、コマンドラインシェル、オブジェクト指向のスクリプト言語、およびスクリプト/コマンドレットを実行してモジュールを管理するための一連のツールが含まれています。

3年前、PowerShellの最新バージョンとしてPowerShell Core 6を発表しました。 .NET Coreに基づいて、PowerShell CoreはWindows、macOSおよびLinuxのクロスプラットフォームサポート、SSHベースのPowerShell Remoting、大幅に改善されたRESTおよびJSONサポート、公式のDockerコンテナーなどを導入しました。さらに、これはPowerShellの最初のリリースであり、オープンソースライセンス（MIT）の下で作成されました。これにより、PowerShellの愛好家と初心者の両方が、ソースコード、テスト、およびドキュメントに直接貢献することができます。

PowerShell Coreの3つの成功したリリースの後、PowerShellの継続的な開発の次の章であるPowerShell 7を見ることに興奮しています。 PowerShell 7では、通常の多くの新しいコマンドレット/ APIとバグ修正に加えてには、次のような多くの新機能が導入されています。

  • ForEach-Object-Parallelによるパイプラインの並列化
  • 新しいオペレーター：
    • 三項演算子：a？b：c
    • パイプラインチェーンオペレーター：|| と&&
    • ヌル合体ステートメント：?? および?? =
  • 簡略化された動的なエラー表示と、エラーをより簡単に調査するためのGet-Errorコマンドレット
  • ユーザーが暗黙のWindows PowerShellセッションでモジュールをインポートできる互換性レベル
  • 自動更新通知
  • PowerShell 7から直接DSCリソースを呼び出す機能（試験的）

機能と修正のより完全なリストについては、こちらを参照して  ください。

PowerShell Core 6.xから7.0へのアップグレードは、.NET Core 2.xから3.1へのアップグレードも意味します。 .NET Core 3.1は、多くの.NET Framework API（特にWindowsの場合）を返し、既存のWindows PowerShellモジュールとの大幅な下位互換性を提供します。これには、Out-GridViewやShow-CommandなどのGUI機能を必要とするWindows上の多くのモジュール、およびWindowsの一部として出荷される多くのロール管理モジュールが含まれます。詳細については、PowerShell 7で動作する最新の最新のモジュールを使用する方法を示すモジュール互換性チャートをご覧ください。

モジュールの互換性の問題のために過去にPowerShell Core 6.xを使用できなかった場合、これは、Coreプロジェクトの立ち上げ以降にすでに提供されている機能の一部を利用している状況である可能性があります。

涼しい！PowerShell 7の入手方法は？


はじめに、Windows、  macOS、または  Linuxのインストールドキュメントを確認してください  。OSのバージョンと優先するパッケージ形式に応じて、いくつかのインストール方法があります。

すでに何をすべきかがわかっていて、バイナリパッケージ（MSI、ZIP、RPMなど）を探しているだけの場合は、GitHubの最新バージョンのタグにアクセスしてください。

または、多数のDockerイメージのいずれかを使用できます。それらの使用の詳細については、PowerShell-Dockerリポジトリを確認してください。

PowerShell 7はどのオペレーティングシステムをサポートしていますか？


PowerShell 7は、次の  x64 オペレーティングシステムをサポートしてい  ます。

  • Windows 7、8.1、および10
  • Windows Server 2008 R2、2012、2012 R2、2016、2019
  • macOS 10.13以降
  • Red Hat Enterprise Linux（RHEL）/ CentOS 7以降
  • Fedora 29以降
  • Debian 9以降
  • Ubuntu 16.04以降
  • openSUSE 15以降
  • Alpine Linux 3.8以降

また、ARM32およびARM64バージョンのDebianとUbuntu、およびARM64 Alpine Linuxもサポートしています。

公式にはサポートされていませんが、コミュニティはArchおよびKali Linuxのパッケージも提供しています。

このリストにないプラットフォームのサポートが必要な場合は、配布リクエストをGitHubに送信してください（ただし、最終的には.NET Core 3.1がサポートされていることに限定されます）。

PowerShell、「コア」はどうなったのですか。


.NET 5を搭載した.NET と同様に、PowerShell 7はWindows PowerShellとの最大の下位互換性への道の終わりを示すものと信じています。この点で、PowerShell 7以降が将来的にサポートされる唯一のPowerShellであると信じています。

6.x / 7.xとWindows PowerShellを区別するために、PowerShell 7は引き続きCoreエディションで機能しますが、全体的には今後「PowerShell 7」と呼ばれるようになります。

PowerShell 7を既にサポートしているマイクロソフト製品はどれですか？


PowerShell Core 6.xで既にサポートされているすべてのモジュールは、PowerShell 7でもサポートされています。

  • Azure PowerShell（Az。*）
  • Active Directory
  • Windows 10およびWindows Serverのモジュールの多く（Get-Module -ListAvailableで確認）

Windowsでは、-UseWindowsPowerShellパラメーターをImport-Moduleに追加して、互換性のないモジュールを使用するユーザーのためのPowerShell 7への移行を簡素化しました。このオプションは、ローカルのWindows PowerShellプロセスを使用してこのモジュールに含まれるすべてのコマンドレットを暗黙的に実行するプロキシモジュールをPowerShell 7に作成します。この機能の詳細については、Import-Moduleのドキュメントをご覧ください。

まだ互換性がないモジュールについては、Microsoft Graph、Office 365などを含むPowerShell 7の組み込みサポートを追加するために、多くの同僚と協力しています。

Azure Cloud ShellはPowerShell 7を使用するように既に更新されていますが、.NET Core SDK DockerコンテナーイメージやAzureの機能はまもなく更新されます。

次は何ですか？


すでにPowerShell 7.1に取り組んでおり、7.0に含まれていない新機能や修正が満載された、近い将来の最初のプレビューを期待できます。7.1に対する現在の調査と要望の概要を説明した、より詳細なロードマップブログをお楽しみに。
