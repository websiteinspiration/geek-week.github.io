ANSI Cでの非推奨の関数宣言による未定義の動作


ANSI C標準は、関数プロトタイプの概念を定義します。これは、入力パラメーターのタイプを示す関数宣言のサブセットです。プロトタイプは、一般的な関数宣言が持つ欠点を排除するために導入されました。


したがって、関数プロトタイプの括弧内にパラメーター型のリストを指定することは必須です。そうでない場合、そのような式はコンパイラーによって廃止された関数宣言として認識され、この記事で説明するあいまいな状況につながる可能性があります。


古いプロトタイプ


関数宣言は、関数の戻り値の型とその識別子を指定されたスコープに導入します。すべての関数宣言をプロトタイプと見なすことはできませんが、入力パラメーターの型のリストを持っているものだけに注意してください。


したがって、以下のコードの最初の式は宣言ですが、関数プロトタイプではありません。次の式は、パラメーターの型を指定しているため、プロトタイプと見なすことができます。


/* #1 (   "foo") */
void foo();

/* #2 (  "bar") */
void bar(int count, const char *word);
