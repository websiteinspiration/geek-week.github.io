MegaFonでビッグデータはどのように機能し、そこに到達する方法を教えてください。

MegaFonは、モバイル通信を提供する単なる通信会社ではなく、顧客の生活のエコシステムを形成する製品を作成するデジタル企業です。「独自のカード」、「独自のキャッシュバック」、「MegaFon.TV」、「MegaFon.Music」などその他。 MegaFon Big Data Analytics部門は、各クライアントのニーズに合わせてオファーをパーソナライズします。


2019年春のデータフェスト会議でのMegaFonビッグデータアナリストのスピーチ

メガフォンのデータサイエンティストは、テレコムサービス市場の成長が鈍化する中、同社の優先事項の1つである加入者ベースを維持するという問題を解決しています。たとえば、数年前、ビッグデータに基づいて、新しい「Turn on」料金表が開発されました。それはデジタルユーザーの真の興味に基づいて構築されています：話す、メッセンジャーでチャットする、音楽を聴く、ソーシャルネットワークでチャットする、ビデオを見る。関税の名前は関心に応じて埋めることに対応しており、使い慣れたアプリケーションを無制限に使用するには、消費されたトラフィックの計算は必要ありません。エコシステムを形成するときの私たちの仕事は、各クライアントに個別の提案をすることです。

ビッグデータは、小売に関連するタスクも解決します。たとえば、機械学習モデルを利用して、効率の悪いサロンをどこに移動し、どこに新しいサロンを開くかを理解しています。この方向では、地理データを操作することが役立ちます。

ビッグデータ分析は、ネットワークインフラストラクチャの開発に関連するタスクでも使用されます。タワーおよびタワーからのトラフィックの分析を使用して、最適なカバレッジを決定し、建設の有望な場所を予測します。

どのテクノロジーが使用されていますか？

私たちが扱うデータの量は、数百万のサブスクライバーとその数十億の毎日の記録です。ビッグデータは、Oracle、MySQL、MongoDBなどのデータベースだけではありません。ビッグデータは、それらを操作するためのあらゆる種類のソフトウェアです。ビッグデータを操作するには、Hadoopの仕組みを理解し、Spark、Hive、HDFSを操作する機能を理解する必要があります。多くの場合、私たちのところにやってきたデータアナリストは、これまでにこれらのツールを使用していません。この場合、十分でないスキルを教えます。

ビッグデータを扱うスキルは経験を積んでいるため、MegaFonは、必要なすべてのツールを学び、会社の実際のタスクに適用する準備ができている有能なアナリストに興味を持っています。

MegaFonのビッグデータの専門家はどのようにしてモデルを開発しますか？

MegaFonのビッグデータエキスパートは、アナリスト（データ専門家）とエンジニアに分かれています。アナリストは仮説をテストし、機械学習モデルを構築します。エンジニアは、アナリストが店先を収集し、ETLプロセスを最適化し、実稼働環境でモデルを設定する責任があります。

モデルの開発は次のとおりです。まず、HadoopまたはOracleで必要なデータを収集します。次に、モデルは、大量のメモリとCPUコアを備えた専用サーバーでトレーニングされます。ニューラルネットワークのトレーニングには、GPUを備えたサーバーを使用します。


MegaFon、2019年のオフィスのBigDataCamp

モデル開発の主な言語はPythonです。 Pythonでのデータ処理には、通常、標準ライブラリPandas、NamPy、Scikit-learnが必要です。 Hadoopでの計算にはPySparkとHiveを使用し、モデリングにはライブラリScikit-learn、Xgboost、LightGBM、PyTorchなどを使用しています。リストはタスクによって異なります。なぜpythonその主な利点は、生産性の単純さです。すぐに共通インフラストラクチャに統合されるソリューションを作成できます。必要なライブラリがPythonにない場合がありますが、他の言語にあります。たとえば、RにはPythonにはない統計ライブラリがあります。

Hadoopを誰も知らない場合はどうなりますか？

Hadoopスキルは望ましいですが、私たちのチームに参加するための前提条件ではありません。すべての企業がMegaFonが持っているデータの量を持っているわけではなく、その結果、候補者は以前の勤務先でHadoopを使用する機会がありませんでした。

Hadoopクラスターを操作するための基本的なコマンドを習得することはそれほど難しくありませんが、より複雑なタスクになると、ビッグデータアルゴリズム、MapReduce、クエリ最適化メソッドを深く理解する必要があります。たとえば、HadoopエコシステムにはHiveなどの製品があります。 SQLのようなクエリを記述して、Hadoop上で実行できます。もともとはFacebookによって開発されました。ただし、SQLで記述しているにもかかわらず、これはリレーショナルデータベースの操作ではないことを覚えておく必要があります。ここでは単純なクエリを記述できますが、効率、つまり速度とクラスターリソースの使用を最小限に抑えるために、MapReduceを使用したクエリ最適化のニュアンスを理解する必要があります。

チームにどのように入力しますか？

インターンへのインタビューは、経験豊富な専門家へのインタビューとは異なります。インターンを検索するとき、採用担当者は簡単な電話インタビューを行います。その結果、候補者が私たちのタスクに興味があるかどうか、および候補者が現在持っている知識と経験のレベルが明らかになります。候補者がPythonでプログラミングできるかどうか、機械学習の基本的なライブラリを知っているかどうか、ビッグデータの分析に関連する教育上の問題を解決した経験があるかどうか、数学モデルを以前に構築したことがあるかどうか、使用したアルゴリズムがあるかどうかは重要です。

電話インタビューの結果に基づいて、5〜10人の候補者を選択し、同時に2〜3時間オフィスに来て、チームのメンバーを知り、技術的なタスクを解決します。これは、通信業界にできるだけ近いものです。加入者を分類するためのモデルを構築する必要があります。次に、結果を比較し、最終面接に最適なものを招待して、個々の作業スケジュール、タスク、およびその他の条件について話し合います。

インターンシップは3ヶ月続きます。インターンは実際の業務に従事しています。ほとんどの場合、タスクは既に形式化されており、実行する必要があることを明確に理解しています。そうでない場合でも、いつでもメンターに頼ることができます。

ビジネスタスクに加えて、インターンは定期的にオフラインおよびオンラインのトレーニングを受けます。私たちは、ニュープロラボ、ビッグデータチーム、オタクブレイン、データジムなどと連携しており、エキスパートはCourseraにアクセスできます。

練習でわかるように、3か月で、引き続き共同作業を行うかどうかを理解できます。研修生が良い結果を示した場合、私たちは彼をジュニアデータサイエンティストの立場に導き、さらに成長します。

メガフォンのオフィスで働く

ワークフローは次のように構成されています。最初に、マネージャーは顧客担当者と要件について話し合います。原則として、機械学習とデータ分析を使用してビジネスプロセスを改善することを話します。たとえば、小売り向けのスマートフォンの販売を最適化できます。次に、マネージャー、チームリーダー、アナリストが共同で開発の条件と段階について話し合います。アレンジメントはJiraで記録され、Confluenceも実行されます。これは私たちの内部Wikiです。もちろん、Gitlabを使用しています。

今年は、データサイエンスプロジェクトのすべての主要な段階でコードレビュープロセスを導入し、すでに結果を確認しています。多くの人のコード品質が大幅に向上しました。開発プロセスを改善するためのさらなる計画は、データセットを含むプロジェクト全体のバージョン管理を可能にするDVC（Data Version Control）ツールの導入です。

プロジェクトの期間は、数か月から6か月です。アナリストは、要件の形式化からモデルのターゲットイベントの決定まで、プロジェクトのすべての段階に関与し、生産における結果の安定性の監視で終わります。

私たちは非常に結果重視であり、MegaFonにどのようなメリットをもたらすことができるかを明確に理解しない限り、開発に着手することはありません。
モデルを構築したら、その作業の結果に基づいてテストキャンペーンを開始します。成功した場合は、数百万人のMegaFonサブスクライバーにソリューションを展開します。将来的には、対象セグメントの正確性や完全性などのモデル指標の観点から結果を分析するだけでなく、ビジネス指標の分析にも真剣に取り組みます。当社のビジネスアナリストがこれを支援します。
