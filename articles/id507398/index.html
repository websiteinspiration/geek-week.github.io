Mencari APT30: saat kami melihat aktivitas baru grup setelah beberapa tahun diam







Cybergroup APT30 telah dikenal cukup lama - pada tahun 2015, kolega kami dari FireEye menggambarkan aktivitasnya. Anggota kelompok ini biasanya menyerang agen pemerintah di Asia Selatan dan Tenggara (di India, Thailand, Malaysia, dan negara-negara lain) dengan tujuan spionase dunia maya, dan alat mereka telah dikembangkan setidaknya sejak 2005.



Dalam beberapa tahun terakhir, APT30 tidak terlalu aktif, tetapi pada musim semi 2020 kami dapat mendeteksi jejak malware baru yang dikembangkan oleh peretas. Studi lengkap diterbitkan dengan referensi , dan dalam artikel ini kami akan menyentuh pada poin utamanya.



pengantar



Pada 8 April 2020, para pakar kami dari PT Expert Security Center menemukan aktivitas kelompok penjahat cyber yang terkenal. Sumber daya yang populer untuk analisis malware dinamis memicu tanda tangan jaringan yang menggambarkan aktivitas APT30, yang tidak pernah didengar sejak lama. Ini adalah titik awal penelitian kami.



Ternyata, peretas tidak hanya terus mendukung alat lama mereka yang terkenal satu dekade yang lalu, tetapi juga mematuhi pendekatan mereka untuk mengatur sumber daya jaringan.



BACKSPACE dan NETEAGLE backdoors



Pada 25 Agustus 2019, sebuah file (MD5: f4f8f64fd66a62fc456da00dd25def0d) dengan nama AGENDA.scr dari Malaysia diunggah ke VirusTotal . Ini adalah file PE yang dapat dieksekusi berbasis x86 yang dikemas oleh UPX. Sampel memiliki ikon dokumen kantor untuk menipu pengguna, dan sumber dayanya berisi dua objek yang lebih terenkripsi.







File pertama (MD5: 634e79070ba21e1e8f08aba995c98112) ditulis ke direktori dengan templat Microsoft Office - % APPDATA% \ Microsoft \ Windows \ Templates \ AGENDA.docx - dan mulai. Ini adalah dokumen kantor yang berisi rencana rapat di salah satu departemen di pemerintah Malaysia. Tujuan dari dokumen ini adalah untuk menarik perhatian pengguna.







File kedua (MD5: 56725556d1ac8a58525ae91b6b02cf2c) ditempatkan di direktori startup% APPDATA% \ Microsoft \ Windows \ Start Menu \ Program \ Startup \ WINWORD.EXE . Pada saat pembuatan, file tidak mulai (penyerang berharap itu berfungsi, misalnya, setelah me-reboot sistem agar tidak menarik perhatian). Ini adalah pintu belakang keluarga NETEAGLE, yang modifikasinya dibahas secara rinci dalam laporan kolega kami dari FireEye. Sangat mengherankan bahwa garis NetEagle yang khas dari sampel tahun 2015 (yang berfungsi sebagai nama keluarga malware) sekarang digantikan oleh JokerPlay .

Menurut tanggal kompilasi, mereka dibuat pada 21 Oktober 2019, dan salah satunya juga diunggah ke VirusTotal dari Malaysia, dan hanya pada Mei 2020. Malware ini milik keluarga BACKSPACE, modifikasi yang juga dipertimbangkan dalam laporan para ahli dari FireEye. Kami memberikan garis yang didekripsi untuk setiap sampel bersama dengan algoritma decoding.



Backdoor RHttpCtrl



Merupakan file PE yang dapat dieksekusi berbasis x86. Pertama, malware mencoba untuk mengekstrak nilai kunci acak dari cabang registri HKCU \ Software \ HttpDiv. Jika ini gagal, menggunakan fungsi WinAPI, GetSystemTimeAsFileTime, waktu sistem akan diperoleh - dan kemudian digunakan sebagai pembuat seed dari nomor acak. Nomor acak yang dihasilkan akan disimpan dalam registri dan digunakan nanti.



Menggunakan permintaan GET ke alamat hxxp: //www.kabadefender.com/plugins/r.exe, malware menerima dan menyimpan arsip WinRAR yang sah (komponen CLI-nya, 4fdfe014bed72317fa40e4a425350288). Kemudian ia menciptakan sidik jari sistem, mengumpulkan nama komputer, alamat IP dan versi sistem, dan mengirimkan permintaan POST ke alamat tersebut hxxp://www.kabadefender.com/clntsignin.php



.



Backdoor RCtrl



Ini adalah file PE yang dapat dieksekusi berbasis x86 yang dikembangkan menggunakan perpustakaan MFC dan dikemas menggunakan UPX.



Menggunakan XOR byte tunggal dengan angka 0x23, malware mendekripsi alamat server penyerang utama: 103.233.10 \ .152. Beralih kepadanya melalui TCP pada port 4433, memeriksa keberhasilan koneksi. Jika koneksi gagal, itu menggunakan data tambahan untuk mendapatkan alamat server.



Data tambahan adalah alamat hxxp://www.gordeneyes.com/infos/p



dan hxxp://www.techmicrost.com/infos/p



disandikan dalam XOR bita tunggal dengan angka 0x25. Setelah mendekripsi alamat, malware mencoba menghubungkannya secara bergantian melalui permintaan GET. Respons server mengharapkan 8 byte: alamat dan port IP server.







Mendapatkan alamat server penyerang: '0xAC 0xF7 0xC5 0xBD' ‚Üí '172 247 197 189', '0xBB 0x01 0x00 0x00' ‚Üí 0x1BB ‚Üí 443



Setelah menerima alamat IP yang valid dari server penyerang, malware menghubungkan kembali ke server dan berharap untuk menerima string Jo * Po * Hello darinya. String ini dikodekan dalam tubuh malware dengan XOR byte tunggal dengan angka 0x24. Trik aneh: sebagai aturan, trojan sendiri yang memulai pertukaran data.



Jika saluran diterima, sidik jari sistem dibuat: versi sistem, alamat IP, informasi tentang merek dan frekuensi prosesor, dan jumlah ruang disk. Data yang dikumpulkan dienkripsi dengan algoritma unik berdasarkan pergeseran siklik bitwise dan XOR (yaitu, pergeseran kiri sebesar 4 + 3 = 7 bit dan XOR dengan 0x23) dan dikirim ke server penyerang.



Kemudian aliran terpisah dibuat, yang mengirimkan buffer data yang sama ke server setiap 30 detik. Selanjutnya, kontrol menerima fungsi pemrosesan perintah yang mendekripsi data yang diterima (algoritma dekripsi terdiri dari tindakan yang berlawanan dengan algoritma enkripsi yang dipertimbangkan) dan mengambil nomor perintah.



Kesimpulan



Ini bukan untuk mengatakan bahwa perangkat lunak yang dibuat oleh para penyerang dibedakan oleh kode program kelas tinggi, beberapa trik untuk bersembunyi di sistem atau deteksi bypass. Tampaknya tidak ada tujuan yang berubah: mungkin mereka tidak memerlukan persenjataan berbahaya yang lebih canggih. Berbicara tentang toolkit baru, kami mencatat ketidaklengkapannya. Mungkin kelompok ini sedang menguji perangkat lunak baru dalam pertempuran, mengidentifikasi kelemahan. Jadi di masa depan kita harus mengharapkan tampilan backdoors yang ditingkatkan, RHttpCtrl dan RCtrl, mungkin menggunakan teknik untuk menyembunyikan atau mempersulit analisis.
