RxJSを使用したサーバーへのファイルのインタラクティブなアップロード



RxJSの基本についての前回の記事を書いてから久しぶりです。コメントでは、実際に役立つかもしれないより複雑な例を示すように求められました。そこで、理論を少し薄めることにしました。今日は、ファイルのアップロードについてお話します。

私たちは何をしますか？

  • ユーザーがファイルを選択してサーバーにアップロードできる小さなページを作成します
  • 進行状況バーを追加して、ファイルのアップロードの進行状況を表示します。
  • キャンセルボタンをクリックしてダウンロードをキャンセルする機能を追加する

この記事を理解するには、RxJSの基本的な知識が必要です。何で観察可能、事業者だけでなく、HOO事業者は、

猫を尻尾まで引っ張らず、すぐに仕事に取り掛かります！

トレーニング


まず、ファイルのダウンロード要求を受け入れることができるサーバーが必要です。どのサーバーでもこれに適している可能性があります。記事では、 node.jsをexpressおよびmulterと組み合わせて使用します。

const express = require("express");
const multer  = require("multer");

const app = express();
const upload = multer({ dest:"files" });

app.post("/upload", upload.single("file"), function (req, res) {
    const { file } = req;

    if (file) {
        res.send("File uploaded successfully");
    } else {
        res.error("Error");
    }

});

app.listen(3000);

これで、ページにはユーザーが操作する4つの要素があります。

  • アップロードボタン、クリックするとアップロードが始まります
  • ダウンロードをキャンセルするキャンセルボタン
  • 開始幅が0のプログレスバー。アップロードプロセス中に、その幅を変更します

すべてが流れです


選択するファイルをブラウザに指示するには、ユーザーは「ファイルを選択」ボタンをクリックする必要があります。その後、オペレーティングシステムのダイアログボックスが開き、フォルダツリーが表示されます。ファイルを選択すると、ブラウザはそのファイルに関する必要な情報をすべてダウンロードします。

ユーザーがファイルを選択したことをどのように理解できますか？これには「変更」イベントがあります。イベントがトリガーされた後、ファイルデータが書き込まれる入力のファイル配列に移動できます。

どのように変更イベントをリッスンしますか？addEventListenerメソッドを使用して操作できます。ただし、RxJSを使用して、イベントをストリームとして表すことができます。

fromEvent(input, 'change').pipe(
    //    
    map(() => input.files[0])
).subscribe({
    next: data => console.log(file)
});

ファイルをサーバーにアップロードするアップロード機能を追加します。今のところ、彼女の体を空のままにしておきます：

function upload(file) {
  console.log(file);
}

アップロード関数は、uploadBtnボタンをクリックした後に呼び出す必要があります。

fromEvent(uploadBtn, 'click').subscribe({
  next: () => upload(input.files[0])
});

ストリームのマージ


これで、コードはaddEventListenerを使用して記述するものと変わりません。はい、動作しますが、そのままにしておくと、RxJSが提供する利点が失われます。

私たちにできること ファイルをアップロードする一連の手順について説明します。

  • ファイル選択
  • 「uploadBtn」をクリックします
  • input.filesからファイルを抽出する
  • ファイルのアップロード

次に、このシーケンスをコードに転送します。しかし、入力とuploadBtnストリームを組み合わせる方法は？switchMapオペレーターはこれを支援し、1つのスレッドを別のスレッドに投影できるようにします。

fromEvent(input, 'change').pipe(
    switchMap(() => fromEvent(uploadBtn, 'click')),
    map(() => input.files[0])
).subscribe({
    next: file => upload(file)
});

このコードは、上記で説明した一連の命令に非常に似ています。ユーザーがファイルを選択すると、switchMapがトリガーされ、uploadBtnにサブスクライブします。しかし、その後は何も起こりません。

switchMapでは、fromEvent（uploadBtn、 'click'）によって生成された値のみを外部ストリームに渡すことができます。ファイルのアップロードを開始するには、2番目の命令、つまり、uploadBtnをクリックする必要があります。その後、配列からファイルを抽出するmapメソッドを実行し、uploadメソッドは既にサブスクライブで呼び出されます。

ここで最も興味深いのは、命令のシーケンスが壊れていないことです。アップロード機能が機能するためには、前に「変更」イベントが発生する必要があります。

しかし、それでも1つの問題が残っていました。ユーザーはファイルを選択して、選択を解除できます。そして、ファイルをアップロードしようとすると、未定義のアップロード関数が渡されます。この状況を回避するには、チェックを追加する必要があります。

fromEvent(input, 'change').pipe(
    switchMap(() => fromEvent(uploadBtn, 'click')),
    map(() => input.files[0]),
    filter(file => !!file)
).subscribe({
    next: file => upload(file)
});

xhrでの作業


最も難しい-アンロードプロセスを実装する時が来ました。xhrを使用した例でそれを示します。これは、現時点では、fetchがアップロードの進行状況を追跡する方法を知らないためです。
アンロードは、axiosやjQuery.ajaxなどの他のライブラリを使用して実装できます。


サーバー側でmulterを使用するため、フォーム内でファイルを転送する必要があります（multerはこの形式のデータのみを受け入れます）。このために、createFormData関数を作成しました。

function createFormData(file) {
    const form = new FormData();
    //       file
    form.append('file', file);
    return form;
}

fromEvent(input, 'change').pipe(
    switchMap(() => fromEvent(uploadBtn, 'click')),
    map(() => input.files[0]),
    filter(file => !!file),
    map(file => createFormData(file))
).subscribe({
    next: data => upload(data)
});

XMLHttpRequestを通じてフォームをアンロードします。このオブジェクトのインスタンスを作成し、そのオブジェクトに対してunloadメソッドとonerrorメソッドを定義する必要があります。1つ目はアップロードが完了したときに発生し、2つ目はエラーが発生したときに発生します。

function upload(data) {
    const xhr = new XMLHttpRequest();

    //        
    xhr.onload = () => console.log('success');

    //    
    xhr.onerror = e => console.error(e);

    //  
    xhr.open('POST', '/upload', true);

    //  
    xhr.send(data);
}

これで実用的な例があります。しかし、これにはいくつかの欠点があります。

  • ダウンロードをキャンセルする方法はありません
  • uploadBtnボタンをn回クリックすると、1つのファイルをアップロードするためのn個の並列接続があります

これは、アップロード機能がストリームから機能しないためです。彼女は一人暮らしをしています。修正する必要があります。関数にObservableを返させましょう。次に、ファイルのアップロードを制御できます。

function upload(data) {
    return new Observable(observer => {
        const xhr = new XMLHttpRequest();

        //    ,      
        //   
        xhr.onload = () => {
            observer.next();
            observer.complete();
        };
        
        xhr.onerror = e => observer.error(e);
        
        xhr.open('POST', '/upload', true);
        xhr.send(data);

        //   -  
        return () => xhr.abort();
    });
}

Observable内で返されるarrow関数に注意してください。このメソッドは、登録解除時に呼び出され、アップロードをキャンセルします。

アップロードの呼び出しをswitchMapに入れます。

fromEvent(input, 'change').pipe(
    switchMap(() => fromEvent(uploadBtn, 'click')),
    map(() => input.files[0]),
    filter(file => !!file),
    map(file => createFormData(file)),
    switchMap(data => upload(data))
).subscribe({
    next: () => console.log('File uploaded')
});

これで、ユーザーがもう一度[アップロード]ボタンをクリックすると、以前のリクエストはキャンセルされますが、新しいリクエストが作成されます。

クリックリクエストをキャンセルする


まだcalcelBtnボタンがあります。リクエストのキャンセルを実装する必要があります。takeUntilオペレーターがここで役立ちます。

takeUntilは「テイクバイ」に変換されます。この演算子は、外部ストリームから値を取得し、チェーンのさらに下にそれらを提供します。内部スレッドが存在し、何も生成しない限り。内部スレッドが値を生成するとすぐに、takeUntilは登録解除メソッドを呼び出し、外部スレッドから登録解除します。

オペレーターを追加する前に、サブスクライブを解除するストリームを決定する必要があります。ファイルのアップロードを完了するだけでよいため、アップロードに関心があります。内部スレッドからの退会：

fromEvent(input, 'change').pipe(
    switchMap(() => fromEvent(uploadBtn, 'click')),
    map(() => input.files[0]),
    filter(file => !!file),
    map(file => createFormData(file)),
    switchMap(data => upload(data).pipe(
        //    upload
        takeUntil(fromEvent(cancelBtn, 'click'))
    ))
).subscribe({
    next: () => console.log('File uploaded')
});

プログレスバー


進行状況バーを追加するために残ります。進行状況を追跡するには、xhr.upload.onprogressメソッドを定義する必要があります。このメソッドは、ProgressEventイベントが発生したときに呼び出されます。イベントオブジェクトには、便利ないくつかのプロパティが含まれています。

  • lengthComputable-trueの場合、ファイルの全サイズがわかります（この場合、常にtrue）。
  • 合計-バイトの総数
  • loaded-送信されたバイト数

アップロード機能を変更します。

function upload(data) {
    return new Observable(observer => {
        const xhr = new XMLHttpRequest();

        xhr.upload.onprogress = e => {
            //  
            const progress = e.loaded / e.total * 100;
            observer.next(progress);
        };

        xhr.onerror = e => observer.error(e);
        xhr.onload = () => observer.complete();

        xhr.open('POST', '/upload', true);
        xhr.send(data);

        return () => xhr.abort();
    });
}

次に、アップロードステータスをアップロードステータスとしてストリームに出力します。ProgressBar要素のスタイルプロパティを変更する関数を記述するだけです。

function setProgressBarWidth(width) {
    progressBar.style.width = `${width}%`;
}

思い出して


完全に機能するアプリケーションを入手しました。いくつかのストロークを追加する必要があります。これで、uploadBtnボタンをクリックすると、以前のアップロードがキャンセルされ、新しいアップロードが開始されます。しかし、キャンセルボタンは既にあります。

ファイルをアップロードするまで（またはアップロードをキャンセルするまで）、uploadBtnボタンが後続のクリックに応答しないようにしたい。何ができますか？

アップロードプロセスが完了するまで、disable属性をハングアップできます。ただし、別のオプションがあります。exhaustMapオペレーターです。このステートメントは、内部スレッドが完了するまで、外部スレッドからの新しい値を無視します。switchMapを排気マップに置き換えます。

exhaustMap(data => upload(data).pipe(
  takeUntil(fromEvent(cancelBtn, 'click'))
))

これで、アプリケーションが完成したと見なすことができます。少しリファクタリングして、最終バージョンを取得します。

import { fromEvent, Observable } from "rxjs";
import { map, switchMap, filter, takeUntil, exhaustMap } from "rxjs/operators";

const input = document.querySelector('#file');
const uploadBtn = document.querySelector('#upload');
const progressBar = document.querySelector('#progress-bar');
const cancelBtn = document.querySelector('#cancel');

const fromUploadBtn = fromEvent(uploadBtn, 'click');
const fromCancelBtn = fromEvent(cancelBtn, 'click');

fromEvent(input, 'change').pipe(
    switchMap(() => fromUploadBtn),
    map(() => input.files[0]),
    filter(file => !!file),
    map(file => createFormData(file)),
    exhaustMap(data => upload(data).pipe(
        takeUntil(fromCancelBtn)
    ))
).subscribe({
    next: width => setProgressBarWidth(width)
});

function setProgressBarWidth(width) {
    progressBar.style.width = `${width}%`;
}

function createFormData(file) {
    const form = new FormData();
    form.append('file', file);
    return form;
}

function upload(data) {
    return new Observable(observer => {
        const xhr = new XMLHttpRequest();

        xhr.upload.onprogress = e => {
            const progress = e.loaded / e.total * 100;
            observer.next(progress);
        };

        xhr.onerror = e => observer.error(e);
        xhr.onload = () => observer.complete();

        xhr.open('POST', '/upload', true);
        xhr.send(data);

        return () => xhr.abort();
    });
}

AngularとHttpClient


Angularを使用している場合は、xhrを直接使用する必要はありません。AngularにはHttpClientサービスがあります。このサービスは、ロード/アンロードの進行状況を追跡できます。これには、次のパラメーターをpostメソッドに渡すだけで十分です。

  • reportProgress：true-アップロード/ダウンロード情報を取得します
  • observe： "events"-ストリームからHttpEventsを受信することを示します

Angularのアップロードメソッドは次のようになります。

export class UploaderService {
  constructor(private http: HttpClient) { }

  public upload(data: FormData): Observable<number> {
    return this.http.post('/upload', data, { reportProgress: true, observe: 'events' })
      .pipe(
        filter(event => event.type === HttpEventType.UploadProgress),
        map(event => event as HttpProgressEvent),
        map(event => event.loaded / event.total * 100)
      );
  }
}

filterステートメントは、アップロードイベントのみを除外します。他のイベントは私たちに興味がありません。さらに、読み込まれたプロパティと合計プロパティにアクセスするために、イベントをHttpProgressEventに持ってきます。割合を考慮します。

HttpClientは、xhrの単なるラッパーであり、ボイラープレートから私たちを救い、HTTPでの作業を容易にします。

結論


RxJSは、開発者にとって非常に強力なツールです。彼の兵器庫には、あらゆる機会のためのオペレーターの巨大なセットがあります。残念ながら、このため、このテクノロジーへの参入のしきい値は非常に高くなっています。そして多くの場合、人々は無意識のうちに自分の「バイク」を書き始めるため、コードの保守が困難になります。

ですから、読者の皆様にはじっと立ち止まらず、実験を恐れないでいただきたいと思います。RxJSについて学びます。突然、10行のコードを1つに変換できるオペレーターに出会います。または、コードをもう少し明確にするのに役立ちます。
