01.抽象クラスの書き方を学ぶ

抽象クラスの作成を始めるとすぐに、これがコードベースにどれほど明確にできるかがわかります。それらは同じメソッドとメソッド名を強制します。多くの人が同じプロジェクトに取り組んでいる場合、誰もが異なる方法を書き始めます。これは逆効果の混乱を引き起こす可能性があります。抽象クラスはこれを防ぐのに役立ちます。

長い例 
import os
from abc import ABCMeta, abstractmethod


class DataProcessor(metaclass=ABCMeta):
    """Base processor to be used for all preparation."""
    def __init__(self, input_directory, output_directory):
        self.input_directory = input_directory
        self.output_directory = output_directory

    @abstractmethod
    def read(self):
        """Read raw data."""

    @abstractmethod
    def process(self):
        """Processes raw data. This step should create the raw dataframe with all the required features. Shouldn't implement statistical or text cleaning."""

    @abstractmethod
    def save(self):
        """Saves processed data."""


class Trainer(metaclass=ABCMeta):
    """Base trainer to be used for all models."""

    def __init__(self, directory):
        self.directory = directory
        self.model_directory = os.path.join(directory, 'models')

    @abstractmethod
    def preprocess(self):
        """This takes the preprocessed data and returns clean data. This is more about statistical or text cleaning."""

    @abstractmethod
    def set_model(self):
        """Define model here."""

    @abstractmethod
    def fit_model(self):
        """This takes the vectorised data and returns a trained model."""

    @abstractmethod
    def generate_metrics(self):
        """Generates metric with trained model and test data."""

    @abstractmethod
    def save_model(self, model_name):
        """This method saves the model in our required format."""


class Predict(metaclass=ABCMeta):
    """Base predictor to be used for all models."""

    def __init__(self, directory):
        self.directory = directory
        self.model_directory = os.path.join(directory, 'models')

    @abstractmethod
    def load_model(self):
        """Load model here."""

    @abstractmethod
    def preprocess(self):
        """This takes the raw data and returns clean data for prediction."""

    @abstractmethod
    def predict(self):
        """This is used for prediction."""


class BaseDB(metaclass=ABCMeta):
    """ Base database class to be used for all DB connectors."""
    @abstractmethod
    def get_connection(self):
        """This creates a new DB connection."""
    @abstractmethod
    def close_connection(self):
        """This closes the DB connection."""



02.疑似ランダムジェネレーターの初期化値を修正します。

実験の再現性は非常に重要であり、初期値は私たちの敵です。それを管理下に置いてください。そうしないと、異なる実験中に異なる値が除算をトレーニングデータとテストデータに変更し、ニューラルネットワークの異なる初期重みを作成します。これは、繰り返しの実験で再現不可能な結果につながります。

def set_seed(args):
    random.seed(args.seed)
    np.random.seed(args.seed)
    torch.manual_seed(args.seed)
    if args.n_gpu > 0:
        torch.cuda.manual_seed_all(args.seed)

オプション2-fastprogress


from fastprogress.fastprogress import master_bar, progress_bar
from time import sleep
mb = master_bar(range(10))
for i in mb:
    for j in progress_bar(range(100), parent=mb):
        sleep(0.01)
        mb.child.comment = f'second bar stat'
    mb.first_bar.comment = f'first bar stat'
    mb.write(f'Finished loop {i}.')

07.
すべての関数が同じように作成されるわけではないため、関数の実行時間を測定します。

コードが機能しても、優れたコードを記述したとは限りません。一部の「ソフト」バグはコードを遅くする可能性があるため、それらを見つけることが重要です。関数の実行時間を制御するには、デコレーターを使用します。

import time


def timing(f):
    """Decorator for timing functions
    Usage:
    @timing
    def function(a):
        pass
    """

    @wraps(f)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = f(*args, **kwargs)
        end = time.time()
        print('function:%r took: %2.2f sec' % (f.__name__,  end - start))
        return result
    return wrapper


08.クラウドサービスにお金を無駄にしないでください。

クラウドリソースを浪費するエンジニアが好きな人はいません。


私たちの実験のいくつかは何時間も数えることができる。計算が終わった直後に全員を追跡し、クラウドインスタンスを強制終了することは困難です。私自身はとても間違っていて、インスタンスを残して人々が数日間不必要に働いているのを見ました。これは、月曜日に必要なものを金曜日に計算し始める場合は特に一般的です。

import os

def run_command(cmd):
    return os.system(cmd)
    
def shutdown(seconds=0, os='linux'):
    """Shutdown system after seconds given. Useful for shutting EC2 to save costs."""
    if os == 'linux':
        run_command('sudo shutdown -h -t sec %s' % seconds)
    elif os == 'windows':
        run_command('shutdown -s -t %s' % seconds)


コード実行の最後にこの関数を呼び出すだけで、ディストリビューションに分類されないようにすることができます。エラーが原因でコードを停止する問題を同時に解決するには、メインコードをtryでラップし、shutdownコマンドをexceptでラップする必要があります。

09.レポートを作成して保存します。

シミュレーションの特定の時点を過ぎると、すべての新しい知識はエラーとメトリックの分析から得られます。自分自身と管理者のために、適切に記述されたレポートの作成と維持に注意してください。

マネージャーはレポートが大好きです！

import json
import os

from sklearn.metrics import (accuracy_score, classification_report,
                             confusion_matrix, f1_score, fbeta_score)

def get_metrics(y, y_pred, beta=2, average_method='macro', y_encoder=None):
    if y_encoder:
        y = y_encoder.inverse_transform(y)
        y_pred = y_encoder.inverse_transform(y_pred)
    return {
        'accuracy': round(accuracy_score(y, y_pred), 4),
        'f1_score_macro': round(f1_score(y, y_pred, average=average_method), 4),
        'fbeta_score_macro': round(fbeta_score(y, y_pred, beta, average=average_method), 4),
        'report': classification_report(y, y_pred, output_dict=True),
        'report_csv': classification_report(y, y_pred, output_dict=False).replace('\n','\r\n')
    }


def save_metrics(metrics: dict, model_directory, file_name):
    path = os.path.join(model_directory, file_name + '_report.txt')
    classification_report_to_csv(metrics['report_csv'], path)
    metrics.pop('report_csv')
    path = os.path.join(model_directory, file_name + '_metrics.json')
    json.dump(metrics, open(path, 'w'), indent=4)


10.良いAPIを書く
悪い結果をもたらすすべての悪いこと。

高品質のデータのクレンジングとモデリングを実行できますが、最終的にはなんらかの混乱を引き起こす可能性があります。人々との私の経験から、多くの人々は理解可能なAPIの記述方法、それらのドキュメントの記述方法、およびサーバーの引き上げ方法を理解していません。私はすぐにこれについて別の投稿を書くつもりですが、今は基本的なレベルで問題に触れます。

以下は、1分あたり最大1000リクエストというそれほど高負荷ではない従来のML / DLデプロイメントの方法論です。

束を満たします-Fastapi + uvicorn
この束には多くの明らかな利点があります。

控えめに言っても、fastapiを使用してAPIを記述します。これは、ベンチマークによって最速のpython APIであるためです。



パフォーマンスの詳細については、こちらをご覧ください。
