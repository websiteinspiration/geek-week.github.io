Kami membuat spektrometer kilau diri dari ... sebuah radiometer

Pada musim semi saya dikarantina dan sedikit waktu muncul untuk menyolder sesuatu yang menarik. Pilihan jatuh pada perangkat ini .

Tentu saja, saya ingin memperluas kemampuan radiometer dan mencari tahu apa isotop radioaktif tertentu menunggu dalam kehidupan saya sehari-hari di Kiev, yang sangat dekat dengan ChEZ.



Dalam artikel ini saya akan memberi tahu bagaimana saya merakit perangkat dan apa yang saya ubah di sirkuit dan firmware.

Hal pertama yang harus dilakukan adalah memesan papan. Dalam materi untuk artikel asli ada file gerber, jadi semuanya sederhana. Pesanan dibuat pada PCBWay dan JLCPCB untuk membandingkan kualitas. Saya tidak bisa merekomendasikan yang pertama: pengiriman memakan waktu 3 bulan, lubang pemasangan di papan kurang dari yang diperlukan. Dari 5 motherboard, 2 ternyata rusak (yang mereka laporkan kepada saya dalam surat). Dengan JLCPCB semuanya menjadi baik dan tidak ada yang perlu dikeluhkan.

Saya memesan komponen pada Mouser dan set kapasitor dan resistor pada Ali (kemalasan mulai memilih semuanya sesuai kapasitas dan memutuskan untuk hanya memesan satu set). Karena SiPM menggunakan MicroFC 60035 - ini adalah bagian perangkat yang paling mahal. Pada saat memesan, harganya $ 70 untuk Mouser. Dengan 30035 yang lebih kecil dan lebih murah, ia memutuskan untuk tidak mengacaukannya, takut akan lebih sulit untuk menyolder dan merakitnya.

Komponen utama kedua perangkat, selain photodetector, adalah kristal kilau. Dan inilah bidang besar untuk modifikasi. Saya tidak dapat menemukan CsI (Tl) yang digunakan oleh penulis ukuran kecil kurang dari $ 90. Karena itu, saya berhenti di NaI (Tl) 10x40mm c ebeya untuk pengiriman 32 dolar. Pencarian kristal seperti itu adalah kegiatan yang mengasyikkan, hal utama di sini adalah jangan terburu-buru. Semua pencarian Google membawa saya ke Eugene dari Ukraina, tetapi dia tidak memiliki kristal transparan untuk spektrometri. Semua yang dia kirim memiliki rona urin kuning yang tidak menyenangkan.

Dan sekarang, semua bagian dan papan datang, Anda dapat mulai menyolder. Hal pertama yang saya putuskan adalah menyolder papan analog. Tidak ada petualangan di sini, hal utama adalah jangan lupa menyolder resistor, tempat yang tidak dilarutkan (baca dengan seksama tips perakitan untuk artikel asli).

Perubahan berikut harus dilakukan pada board sistem: Menurut lembar data LM2733Y, tegangan output tidak tergantung pada tegangan input, jadi tidak ada yang perlu disesuaikan. Kami mengambil rumus R1 = R2 X (VOUT / 1.23 - 1) dari lembar data yang sama dan dari apa yang ditemukan, set R13 = 1.8K, R12 = 12K, R11 = 300K. Outputnya stabil 28.18V (saya mencoba memasok 2.5V, 5V - semuanya stabil di output). Setelah menghubungkan layar, perangkat mulai memberikan jumlah pulsa yang jauh lebih besar daripada yang sebenarnya. Kami berhasil memperbaikinya dengan mengubah sirkuit daya display: kami menghubungkan input dari switch DA6 ke input DA3. Pada output dari DA6, kami mengatur konverter ke 5V (saya punya cj7032 pololu di ujung jari saya) dan sudah memberi makan tampilan dari itu. Dengan koneksi ini, semua gangguan segera hilang.

Saya mengambil nx4024t032 sebagai tampilan: lebih kecil, lebih murah, lebih sedikit konsumsi, dan yang paling penting, itu tersedia di toko lokal. Saya masih berencana untuk mengubah firmware, lebih lanjut tentang itu di bawah ini.

Setelah menyolder, saya mencuci sirkuit dengan isopropil alkohol dalam rendaman ultrasonik. Setelah alkohol, perlu dicuci dengan air suling dalam bak mandi yang sama dan dijemur dalam oven pada suhu sekitar 70-80 derajat.

Sekarang saatnya untuk melakukan yang paling menarik: hubungkan sensor dan lihat apa yang terjadi. MicroFC 60035 hampir disolder dengan sempurna ke sepotong papan tempat memotong roti 3x3: kami membuat lubang sudut dan menyolder sensor dengan pengering rambut. Solder kabel dari belakang papan tempat memotong roti. Ini adalah tampilannya.



Ini adalah bagaimana sekelompok papan dan bagian tanpa kasing terlihat seperti.



Pembaca yang penuh perhatian mungkin memperhatikan bahwa saya mengambil prosesor STM32L152CBT6A - sedikit lebih banyak memori dan tersedia secara lokal.

Kasing ini dibuat di Fusion 360 dan dicetak pada printer 3D. Berikut ini tautan ke proyek tersebut .

Waktunya telah tiba untuk bagian yang paling menarik - perubahan dalam firmware. Tapi kami ingin membuat detektor kilau, dan bukan hanya radiometer. Untuk melakukan ini, kita perlu menggunakan DMA dengan ADC (hanya ada satu ADC dalam prosesor ini, tetapi ada saklar input). Dan kami memiliki dua input: SP dan tegangan baterai. DMA diperlukan untuk mempercepat seluruh proses. Saya juga ingin memperhatikan jumlah siklus pengukuran ADC_SampleTime, dengan 48 atau lebih, saya tidak berhasil. 4 siklus menunjukkan hasil yang paling stabil.

Ubah kode inisialisasi ADC sebagai berikut:

void initADC(void) {
	/* PWR_CTRL and CHG_STAT clock enable */
	RCC_AHBPeriphClockCmd(RCC_AHBPeriph_GPIOB, ENABLE);

	/* UBAT input pin configuration */
	GPIO_InitStructure.GPIO_Pin = GPIO_Pin_12|GPIO_Pin_15;
	GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AN;
	GPIO_InitStructure.GPIO_PuPd = GPIO_PuPd_NOPULL;
	GPIO_InitStructure.GPIO_Speed = GPIO_Speed_40MHz;
	GPIO_Init(GPIOB, &GPIO_InitStructure);

	/*------------------------ DMA1 configuration ------------------------------*/
	/* Enable DMA1 clock */
	RCC_AHBPeriphClockCmd(RCC_AHBPeriph_DMA1, ENABLE);
	/* DMA1 channel1 configuration */
	DMA_DeInit(DMA1_Channel1);
	DMA_InitStructure.DMA_PeripheralBaseAddr = ADC1_DR_ADDRESS;
	DMA_InitStructure.DMA_MemoryBaseAddr = (uint32_t)&ADC_ConvertedValue[0];
	DMA_InitStructure.DMA_DIR = DMA_DIR_PeripheralSRC;
	DMA_InitStructure.DMA_BufferSize = 2;
	DMA_InitStructure.DMA_PeripheralInc = DMA_PeripheralInc_Disable;
	DMA_InitStructure.DMA_MemoryInc = DMA_MemoryInc_Enable;
	DMA_InitStructure.DMA_PeripheralDataSize = DMA_PeripheralDataSize_HalfWord;
	DMA_InitStructure.DMA_MemoryDataSize = DMA_MemoryDataSize_HalfWord;
	DMA_InitStructure.DMA_Mode = DMA_Mode_Circular;
	DMA_InitStructure.DMA_Priority = DMA_Priority_High;
	DMA_InitStructure.DMA_M2M = DMA_M2M_Disable;
	DMA_Init(DMA1_Channel1, &DMA_InitStructure);
	/* Enable DMA1 channel1 */
	DMA_Cmd(DMA1_Channel1, ENABLE);
	/*----------------- ADC1 configuration with DMA enabled --------------------*/
	/* Enable The HSI (16Mhz) */
	RCC_HSICmd(ENABLE);
	/* Check that HSI oscillator is ready */
	while
	(!RCC_GetFlagStatus(RCC_FLAG_HSIRDY));
	/* Enable ADC1 clock */
	RCC_APB2PeriphClockCmd(RCC_APB2Periph_ADC1, ENABLE);
	/* ADC1 Configuration -----------------------------------------------------*/
	ADC_InitStructure.ADC_Resolution = ADC_Resolution_12b;
	ADC_InitStructure.ADC_ScanConvMode = ENABLE;
	ADC_InitStructure.ADC_ContinuousConvMode = DISABLE;
	ADC_InitStructure.ADC_ExternalTrigConvEdge = ADC_ExternalTrigConvEdge_None;
	ADC_InitStructure.ADC_ExternalTrigConv = 0;
	ADC_InitStructure.ADC_DataAlign = ADC_DataAlign_Right;
	ADC_InitStructure.ADC_NbrOfConversion = 2;
	ADC_Init(ADC1, &ADC_InitStructure);
	/* Enable temperature sensor and Vref */
	ADC_TempSensorVrefintCmd(ENABLE);
	//ADC_TempSensorVrefintCmd(DISABLE);
	/* ADC1 regular channel configuration */
	ADC_RegularChannelConfig(ADC1, SP_ADC_CHANNEL, 1, ADC_SampleTime_4Cycles);
	ADC_RegularChannelConfig(ADC1, UBAT_ADC_CHANNEL, 2, ADC_SampleTime_4Cycles);

	/* Enable the request after last transfer for DMA Circular mode */
	ADC_DMARequestAfterLastTransferCmd(ADC1, ENABLE); /* Causes problem.. */
	/* Define delay between ADC1 conversions */
	ADC_DelaySelectionConfig(ADC1, ADC_DelayLength_None);
	/* Enable ADC1 Power Down during Delay */
	ADC_PowerDownCmd(ADC1, ADC_PowerDown_Idle_Delay, ENABLE);
	/* Enable ADC1 DMA */
	ADC_DMACmd(ADC1, ENABLE);
	/* Enable ADC1 */
	ADC_Cmd(ADC1, ENABLE);

	while(!ADC_GetFlagStatus(ADC1, ADC_FLAG_ADONS));

	ADC_SoftwareStartConv(ADC1);
}
