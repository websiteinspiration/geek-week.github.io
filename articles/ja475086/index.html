フロントエンド開発者向けのREST APIタイピング

現在、OpenApiやGraphQLなど、ブラウザーとサーバーの相互作用を説明するために、次のアプローチが広く使用されています。

この記事では、静的に型付けされたREST APIを作成し、フロントエンドチームがデータリクエストを作成するためのコードを記述せずに済むようにし、テストを簡略化して、発生する可能性のあるエラーの数を減らす方法について説明します。



これらのツールは私たちを助けます：

  • 仕様に基づいてAPIを標準に合わせて設計およびモデル化
  • , API
  • API
  • API
  • API API

アプローチの主な考え方は、クライアントとサーバーで同じデータ型を使用すると、チームの開発者がクライアントでエラーを発生させることがより困難になることであり、コード生成を使用してコードを作成、保守する必要がないため、ユニットテストでカバーする必要があります。

REST APIが受け入れる/提供するデータ型でコマンドが誤りを犯した場合、フロントエンドアプリケーションはコンパイルされません。

したがって、クライアントに静的に型付けされたコードがあると、愚かな型関連のエラーを取り除くことができ、コードがAPIの現在のバージョンと完全に互換性があることを確認できます。

静的に型付けされたAPIの上記の利点をすべて得るには、OpenAPI仕様に従って型記述ファイルを生成できるコードジェネレーターを使用する必要があります。Typescriptの場合、これらは* .d.tsファイルです。

私たちのプロジェクトはマイクロサービスアーキテクチャを使用しており、バックエンド全体が.NETで記述されているため、NSwagを使用してフロントエンド/バックエンド用のAPIクライアントを生成しました。このツールを使用すると、OpenAPIドキュメントを生成できます。OpenAPIドキュメントは、クライアントコードの生成に既に使用されています。

建築


一般に、コード生成プロセスはいくつかの段階で構成されています。

  • OpenAPIドキュメントの生成
  • OpenAPIドキュメントを使用したコードの生成

私たちの場合、バックエンドは.Netを使用して書かれており、主要なC＃開発言語がツールを選択する理由です。NSwagと同じツールを使用して、OpenAPIドキュメントとクライアントを生成します。


図1 RESTクライアントコードを生成するためのソリューションのアーキテクチャ

この図は、次のことを示しています。

  • マイクロサービス1 ... N-APIを提供するマイクロサービス
  • NSwag-マイクロサービスAPIコードからOpenAPIドキュメントを生成する（マイクロサービスは、OpenAPIドキュメントを生成するためのツールがある任意の言語で記述できます）
  • NSwag-OpenAPIドキュメントのクライアントAPIコードを生成します（テクノロジースタックにより適したツールを選択できる多くのツールがあります）
  • OpenAPI doc-生成されたOpenAPIドキュメント
  • APIクライアント1 ... N-APIデータを消費するクライアント（NSwag C＃およびTypescriptのジェネレーターをサポートする任意の言語で実装できます）
  • SPA 1 ... N-フロントエンドアプリケーション（この場合はReact）（NSwagは、AngularJS / Angular、React（フェッチ）、JQuery（Promise / Callback）、Aureliaのクライアント生成をサポートしています）

アクションのシーケンスは次のとおりです。

  • APIコントローラに対応するタグ/属性/デコレータをマークします。APIの言語によって異なります
  • OpenAPI APIコードドキュメントの生成
  • OpenAPIドキュメントを使用してクライアントAPIコードを生成する
  • クライアントAPIコードをデータコンシューマーAPIアプリケーションに統合する

ドキュメンテーション


コードを生成できるようにするために、APIコントローラーの受け入れられた値/返された値のタイプを記述する必要があります。この例では、C＃言語が使用され、SwaggerOperation属性を使用して、サーバー上のすべてのアンケートのリストを返すメソッドをマークしました。クライアントコードでは、取得するメソッドです。データはGetAllQuestionnairesと呼ばれます。

[HttpGet, Route("")]
[SwaggerOperation(OperationId = "GetAllQuestionnaires")]
[SuccessResponse(typeof(IEnumerable<QuestionnaireViewModel>))]       
 public IEnumerable<QuestionnaireViewModel> Get()
 {
   var surveys = _questionnaireRepository.GetAll();
   return surveys.Select(QuestionnaireViewModel.ToViewModel).ToArray();
 }

コード生成


また、NSwagを使用してクライアントAPIコードを生成します。入力でOpenAPIドキュメントを受け取り、指定された設定に従ってクライアントAPIコードを生成します。フロントでは、受信したコードの横に、package.jsonを追加して、ローカルのnpmレジスターに送信します。

バックエンドのコードリスト（リスト1を参照）からわかるように、コントローラーメソッドに属性

[SwaggerOperation(OperationId = "GetAllQuestionnaires")]

今回のケースではC＃属性で指定されたOperationIdがクライアントメソッドの名前になります。

調査結果
