コードに意味のある名前を付ける方法

これは、Medium WebサイトのBetter Programmingブログの記事の翻訳です。その中で、Daanプログラマーは単純なルールを共有し、それに従って関数と変数に適切な名前を付けることができます。



開発者として、変数の作成とその名前の作成に多くの時間を費やすことになります。名前はどこにでもあります。ファイル、クラス、メソッド、およびそれらと同じ変数に必要です。

名前の割り当てには非常に多くの時間を費やしているため、品質の高い方法でこれを行うことが非常に重要です。この記事では、いくつかの簡単なルールについて説明します。これらのルールに従って、適切な名前を作成できます。結局、これは全体の芸術です！

あなたの意図を明らかにする名前を使用してください


名前は、コードの目的を示す必要があります。言うより簡単だよね？プログラマの意図について何も言わない名前の変数に遭遇する頻度はどれくらいですか？

ルールはこれです：名前にコメントが必要な場合、意図は伝えられません。

このコードスニペットには、目的が明確ではない名前の変数があります。

<?php
private $s; // Time in seconds

変数$sは何も教えず、期間と関連付けられていません。何をどの単位で測定するかを示す名前を選択することをお勧めします。

以下の名前のいずれかがより適切です。

<?php
private $days_since_creation;
private $elapsed_time_in_seconds;
private $seconds_since_last_modified;

インテント名を使用すると、コードが理解しやすくなり、保守が容易になります。

あなたは正しい名前を選ぶのに時間を費やしますが、将来それを保存します。

この例を考えてみましょう：
<?php
function getList() {
    $list1 = [];
    
    foreach ($this->the_list as $x) {
        if ($x % 2 != 0) {
            $list1[] = $x;
        }
    }
    
    return $list1;
}
function getOddNumbers() {
    $odd_numbers = [];
    
    foreach ($this->numbers as $number) {
        if (isOdd($number)) {
            $odd_numbers[] = $number;
        }
    }
    
    return $odd_numbers;
}

関数の機能を理解するのが難しいのはなぜgetListですか？複雑な式はありません。コードは正しく整列およびフォーマットされ、3つの変数のみが含まれます。

関数を見てみましょうgetOddNumbers。彼女が同じことをしているのを見ましたgetListか？

コードが複雑になったり単純になったりしていないことに注意してください。演算子、変数、ネストレベルの数は維持しましたが、より明確で透過的になりました。

名前の小さな変更-何が起こっているのかを理解するのはすでにはるかに簡単です。

誤報を避ける


コードの目的を歪める誤った関連付けを避けてください。

意図したものとは異なる方法で理解できる単語は使用しないでください。たとえば、productList本当にタイプオブジェクトでない限り、商品グループに名前を付けないでくださいList。そのような名前は誤った結論につながる可能性があります。単純に使用する方が良いproducts。

変数の最悪の名前は、大文字Oと小文字で始まる可能性があります。Lこれらの文字は0と1に非常に類似しているためです。

また、互いにわずかに異なる名前のみを使用します。SomeMethodForEfficientHandlingOfFilesあるファイルとSomeMethodForEfficientStorageOfFiles別のファイルのわずかな違いに気付くまでにどのくらいかかりますか？一見、それらは同一であることに同意します。

有意義な違いを強調する


ナンバリングの使用は、ネーミングの最良の方法ではありません。このような名前は、作者の意図をまったく明らかにしないため、有益ではありません。

次の例を見てみましょう：

<?php
public function duplicateArray($arr1, &$arr2) {
  foreach ($arr1 as $key => $value) {
    $arr2[$key] = $value;
  }
}

このコードスニペットは、名前を変更して$arr1と$arr2にすると読みやすく$sourceなり$destinationます。

発音できる名前を使用する


関数の名前を発音できない場合は、その関数について話すことができず、ばかげて聞こえません。コードの議論はプログラミングの一部であるため、これは実際に重要です。しかし、私たち一人ひとりが発音できない名前を持つ変数に遭遇したに違いありません。

名前の付いた変数があり$xsq、会社とよく話し合っているとします。そして、あなたは同僚と話しています：
-さて、それは元エスキューとは何ですか？
- ものによって？アクセスキューとは？

一部の開発者は、変数の名前を単語として、また一部を略語として発音します。

検索可能な名前を使用する


1文字の名前の問題は、コードで見つけるのが難しいことです。

数値定数についても同様です。それらを変数に置き換えることをお勧めします。たとえば、コードで何かを見つける必要がある場合、通常の数字の8は多くの問題を引き起こす可能性があります。ただし、数値をのように置き換えると、はるかに簡単になりますMAX_BLOCKS_DISPLAYED。

1文字の名前を使用する価値がある唯一の状況は、短いメソッド内のローカル変数の名前を発明することです。

クラスメンバーのプレフィックス


m_接頭辞は使用しないでください。

一部の開発者は、クラスメンバーの前にアンダースコア付きのプレフィックスを使用する癖があります。このようにしないでください。クラスとメソッドは、これらの接頭辞が不要になるように十分コンパクトでなければなりません。

または、IDEを使用して（またはプラグインをインストールして）、スコープに応じて変数を強調表示できます。

コードを森の中のピクニックスポットと考えてください。到着する前よりもきれいにしてください。
