SVGまたはキャンバス？



SVGとキャンバスは、Webページに何かを描画するために使用できるテクノロジーです。したがって、SVGをいつ使用し、canvasをいつ使用するかを比較して理解する必要があります。これらのテクノロジーの本質の表面的な理解でさえ、非常に情報に基づいた選択をすることができます。実際のところ、ここに2つの典型的な状況があります。1つはSVGを優先し、もう1つはキャンバスです。

  • 小さなアイコンを描く必要がありますか？これは確かにSVGの領域です。
  • インタラクティブなブラウザゲームを作成する必要がありますか？ここでは、間違いなく、キャンバス技術が必要です。

本日の翻訳を公開している記事の著者は、この選択の理由をまだ明らかにしていないことを知っていると述べています。しかし、SVGとキャンバスに関する詳細を共有した後、これらの理由が完全に明らかになることを期待しています。

SVG-宣言的に記述されたベクトルグラフィック


ベクター画像をWebページに配置する必要があることがわかっている場合は、SVGが適していることを意味します。通常、ベクター画像は、JPGファイルなどに保存されている類似のラスター画像よりも見栄えがよくなります。さらに、ベクター画像ファイルは通常、ラスター画像ファイルよりも小さいです。

その結果、ロゴの描画にSVGがよく使用されます。このような画像を記述するコードは、HTMLに直接埋め込むことができます。宣言的な指示のセットのように見えます：

<svg viewBox="0 0 100 100">
  <circle cx="50" cy="50" r="50" />
</svg>

柔軟で応答性の高い画像が必要な場合は、SVGが最適です。

Canvas-描画用のJavaScript API


HTMLコードでキャンバス画像を形成するために、ページは要素を配置し<canvas>、その後、JavaScriptを使用して、この要素に「描画」します。言い換えれば、プログラマーはブラウザーで何かを描画する方法を指示します（このアプローチは宣言よりも命令に近いものです）。これは次のようになります。

<canvas id="myCanvas" width="578" height="200"></canvas>
<script>
  var canvas = document.getElementById('myCanvas');
  var context = canvas.getContext('2d');
  var centerX = canvas.width / 2;
  var centerY = canvas.height / 2;
  var radius = 70;

  context.beginPath();
  context.arc(centerX, centerY, radius, 0, 2 * Math.PI, false);
  context.fillStyle = 'green';
  context.fill();
</script>

SVG画像はDOMに保存されます


clickやなどの DOMイベントに精通している場合はmouseDown、SVGの操作時にこれらのイベントも使用できることに注意してください。この点で、SVG要素は<circle>HTML要素とそれほど変わりません<div>。

<svg viewBox="0 0 100 100">
  
  <circle cx="50" cy="50" r="50" />
  
  <script>
    document.querySelector('circle').addEventListener('click', e => {
      e.target.style.fill = "red";
    });
  </script>
  
</svg>

SVGはページのコンテンツをアクセシブルにするのに適しています


canvas要素に代替テキストを割り当てることができます：

<canvas aria-label="Hello ARIA World" role="img"></canvas>

SVG要素についても同じことができます。ただし、SVG画像とその内部メカニズムはDOMに直接格納されるため、SVGは通常、障害を持つ人々がアクセスできるプロジェクトを作成する場合に使用されるテクノロジーと見なされています。

つまり、支援技術が機能する要素を使用してSVG画像を作成し、ユーザーがWebページをナビゲートできるようにすることができます。

画像の一部であるテキストデータは、SVGを使用することが望ましい場合にも当てはまります。 SVGにも特別な要素があります- <text>。それはあなたがテキストの明確な文字を表示することを可能にします、それは支援技術によって知覚されます。テキストを視覚化するためにcanvas要素を使用すると、そのようなテキストはよく表示されず、ぼやけて見えます。

Canvasは、個々のピクセル画像を扱うのに適しています。

CSS画像はCSSを使用して影響を受ける可能性があります


SVG要素はDOMに格納され、これらの要素へのアクセスはJavaScriptコードから取得できると既に述べました。CSSの場合、この話は繰り返されます。

<svg viewBox="0 0 100 100">
  
  <circle cx="50" cy="50" r="50" />
  
  <style>
    circle { fill: blue; }
  </style>
  
</svg>

例では、ブロック<script>と<style>ブロックはブロック内に配置されています<svg>。これは完全に正常です。あなたはSVG要素は、通常のHTMLコードであるという事実を考えるとしかし、あなたは室内ユニットということでしょう<script>と<style>もそれから除去することができます。さらに、必要に応じて、外部スタイルとスクリプトを使用してSVG要素を操作できます。

ここで SVG要素とCSSのプロパティに関する材料の多くは。ここで最も重要なことは、CSSを使用してページ要素で実行できるすべてがSVGに適用されるということです。たとえば、これは疑似クラス:hoverとアニメーション要素を使用しています。

ここだ要素をアニメーションの例<circle>CSSを使用しては。

<svg viewBox="0 0 100 100">
  
  <circle cx="50" cy="50" r="50" />
  
  <style>
    circle { fill: blue; animation: pulse 2s alternate infinite; }
    @keyframes pulse {
      100% {
        r: 30;
      }
    }
  </style>
  
  <script>
    document.querySelector('circle').addEventListener('click', e => {
      e.target.style.fill = "red";
    });
  </script>
  
</svg>
