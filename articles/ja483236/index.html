仮想化の魔法：Proxmox VEの入門コース


今日は、1つの物理サーバー上に異なるオペレーティングシステムを備えた複数の仮想サーバーを迅速かつ公平に展開する方法について説明します。これにより、すべてのシステム管理者が会社のITインフラストラクチャ全体を集中管理して、大量のリソースを節約できます。仮想化を使用すると、物理サーバー機器から可能な限り抽象化し、重要なサービスを保護し、非常に深刻な障害が発生した場合でも作業を簡単に復元できます。

疑いもなく、ほとんどのシステム管理者は仮想環境での作業方法に精通しており、彼らにとってこの記事は発見ではありません。それにもかかわらず、正確な情報が不足しているため、仮想ソリューションの柔軟性と速度を使用しない企業があります。私たちの記事が、物理インフラストラクチャの不便さと欠点を経験するよりも、仮想化を使い始める方がはるかに簡単であることを例によって理解するのに役立つことを願っています。

幸い、仮想化の仕組みを試すのは非常に簡単です。会社が使用するCRMシステムを転送するなど、仮想環境でサーバーを作成する方法を示します。ほとんどすべての物理サーバーを仮想サーバーにすることができますが、最初に基本的な作業方法を習得する必要があります。これについては以下で説明します。

どのように機能しますか


仮想化に関しては、多くの初心者スペシャリストが用語を理解するのは難しいため、いくつかの基本的な概念について説明します。

  • ハイパーバイザー -仮想マシンを作成して管理できるようにする特別なソフトウェア。
  • 仮想マシン（以下、VMと呼びます）は、独自の特性、ドライブ、およびオペレーティングシステムのセットを持つ物理サーバー内の論理サーバーであるシステムです。
  • 仮想化ホストは、ハイパーバイザーが実行されている物理サーバーです。

サーバーが完全な仮想化ホストとして機能するためには、そのプロセッサーがIntel®VTまたはAMD-V™の2つのテクノロジーのいずれかをサポートしている必要があります。どちらのテクノロジーも、最も重要なタスクを実行します-仮想マシンにサーバーハードウェアリソースを提供します。

重要な機能は、仮想マシンのアクションがハードウェアレベルで直接実行されることです。さらに、それらは互いに分離されているため、個別に管理することは非常に簡単です。ハイパーバイザー自体が制御組織の役割を果たし、それらの間でリソース、役割、および優先順位を分散します。ハイパーバイザーは、オペレーティングシステムの正常な動作に必要なハードウェアの一部もエミュレートします。

仮想化の導入により、1つのサーバーの複数の実行中のコピーを持つことが可能になります。このようなコピーに変更を加えるプロセスで重大な障害またはエラーが発生しても、現在のサービスまたはアプリケーションの動作には影響しません。同時に、2つの主要な問題にも対処します。スケーリングと、同じ機器で異なるオペレーティングシステムの「動物園」を維持する機能です。これは、それぞれに個別の機器を購入する必要なく、さまざまなサービスを組み合わせる理想的な機会です。

仮想化により、サービスと展開されたアプリケーションの復元力が強化されます。物理サーバーが故障していて別のサーバーと交換する必要がある場合でも、ストレージメディアが保持されていれば、仮想インフラストラクチャ全体が完全に機能し続けます。この場合、物理サーバーは別のメーカーのものである可能性があります。これは、廃止され、他のモデルに切り替える必要があるサーバーを使用している企業に特に当てはまります。

次に、今日存在する最も人気のあるハイパーバイザーをリストします。

  • VMware ESXi
  • Microsoft Hyper-V
  • Open Virtualization Alliance KVM
  • Oracle VM VirtualBox

それらはすべて非常に普遍的ですが、それぞれに選択の段階で常に考慮すべき特定の機能があります。それは、展開/保守のコストと技術仕様です。VMwareとHyper-Vの商用ライセンスは非常に高価であり、障害が発生した場合、これらのシステムの問題を自分で解決することは非常に困難です。

一方、KVMは、特にProxmox Virtual Environmentと呼ばれる既製のDebian Linuxベースのソリューションの一部として、完全に無料でかなり使いやすいです。このシステムは、仮想インフラストラクチャの世界を初めて理解するときにお勧めします。

Proxmox VEハイパーバイザーを素早くデプロイする方法


ほとんどの場合、インストールでは問題が発生しません。公式サイトから最新バージョンのイメージをダウンロードし、Win32DiskImagerユーティリティ（Linuxではddコマンドを使用）を使用して外部メディアに書き込み、その後、このメディアからサーバーを直接ロードします。専用サーバーをレンタルしているクライアントは、KVMコンソールから直接目的のイメージをマウントする方法と、PXEサーバーを使用する方法の2つの簡単な方法を使用できます。

インストーラーにはグラフィカルインターフェイスがあり、いくつかの質問をします。

  1. インストールを実行するディスクを選択します。[ オプション]セクションでは、追加のマークアップオプションを設定することもできます。

  2. 地域設定を指定します。

  3. ルートスーパーユーザーの認証に使用するパスワードと管理者の電子メールアドレスを指定します。

  4. ネットワーク設定を指定します。FQDNは完全修飾ドメイン名を表します（例：node01.yourcompany.com）。

  5. インストールが完了したら、[再起動]ボタンを使用してサーバーを再起動します。

インストール後の作業


Proxmoxのインストール後に行うべき重要なことがいくつかあります。それぞれについて詳しく説明します。

システムを最新バージョンに更新する


これを行うには、サーバーのコンソールに移動し、有料リポジトリを無効にします（有料サポートを購入したユーザーのみが利用できます）。これを行わないと、aptはパッケージソースの更新時にエラーを報告します。

  1. apt:

    nano /etc/apt/sources.list.d/pve-enterprise.list
  2. . #, :

    #deb https://enterprise.proxmox.com/debian/pve stretch pve-enterprise
  3. Ctrl + X , Y .
  4. :

    apt update && apt -y upgrade


パスワードのブルートフォース（ブルートフォース）による攻撃から保護する 最も人気のあるユーティリティFail2Banをインストールすることをお勧めします。その動作の原理は、攻撃者が不正なユーザー名/パスワードを使用して、指定された時間のログイン試行回数を超えると、攻撃者のIPアドレスがブロックされることです。ブロッキング期間と試行回数は、構成ファイルで指定できます。
実際の経験に基づくと、開いているsshポート22と外部静的IPv4アドレスでサーバーを実行してから1週間以上、パスワードを見つけるために5000回以上の試行がありました。そして、約1500は正常にブロックされたユーティリティに対処します。
インストールを完了するために、小さな指示を与えます：

  1. WebインターフェイスまたはSSHを介してサーバーコンソールを開きます。
  2. パッケージソースの更新：

    apt update
  3. Fail2Banをインストールします。

    apt install fail2ban
  4. 編集のために構成ユーティリティを開きます。

    nano /etc/fail2ban/jail.conf
  5. 個々のサービスごとに、bantime変数（攻撃者がブロックされる秒数）とmaxretry（ログイン/パスワードの試行回数）を変更します。
  6. キーボードショートカットのCtrl + Xを使用してエディタを終了し、ファイルの保存に関するシステムの質問にYと答えます。
  7. サービスを再起動します。

    systemctl restart fail2ban

ユーティリティのステータスを確認できます。たとえば、1つの簡単なコマンドで、SSHパスワードを試行したブロックIPアドレスの統計を削除できます。

仕事の始まり


Proxmoxがインストール直後に新しいマシンを作成する準備ができているという事実に注目したいと思います。ただし、将来のシステム管理が容易になるように、事前に設定することをお勧めします。実践では、ハイパーバイザーと仮想マシンを異なる物理メディアに分離する必要があることを示しています。これを行う方法については、以下で説明します。

ディスクドライブを構成する


次のステップは、仮想マシンのデータとバックアップを保存するために使用できるストレージの構成です。
! . RAID-, . ,
物理サーバーに、ハイパーバイザーがインストールされている2つのディスク- / dev / sdaと、仮想マシンデータの格納に使用される予定の空の/ dev / sdbディスクがあるとします。システムが新しいストレージを認識できるようにするために、最も単純で最も効果的な方法を使用できます-通常のディレクトリとして接続します。ただし、その前に、いくつかの準備アクションを実行する必要があります。例として、ext4ファイルシステムにフォーマットして、任意のサイズの新しいドライブ/ dev / sdbを接続する方法を見てみましょう。

  1. ディスクにマークを付け、新しいパーティションを作成します。

    fdisk /dev/sdb
  2. oまたはgを押します（MBRまたはGPTでディスクにマークを付けます）。
  3. 次に、nキーを押します（新しいパーティションを作成します）。
  4. そして最後にw（変更を保存するため）。
  5. ext4ファイルシステムを作成します。

    mkfs.ext4 /dev/sdb1
  6. パーティションをマウントするディレクトリを作成します。

    mkdir /mnt/storage
  7. 編集のために構成ファイルを開きます。

    nano /etc/fstab
  8. そこに新しい行を追加します。

    /dev/sdb1	/mnt/storage	ext4	defaults	0	0
  9. 変更を加えたら、Ctrl + Xを押して編集者の質問にYと答え、変更を保存します。
  10. すべてが機能することを確認するには、サーバーを再起動します。

    shutdown -r now
  11. 再起動後、マウントされたパーティションを確認します。

    df -H

コマンドの出力には、/ dev / sdb1が/ mnt / storageディレクトリにマウントされていることが示されます。これは、ドライブが準備ができていることを意味します。

新しいストレージをProxmoxに追加


コントロールパネルにログインし、[ データセンター] ➝[ ストレージ] ➝[ 追加] ➝[ ディレクトリ]セクションに移動します。

開いたウィンドウで、次のフィールドに入力します。

  • ID-将来のストレージの名前。
  • ディレクトリ -/ mnt / storage;
  • コンテンツ -すべてのオプションを選択します（各オプションを1つずつクリックします）。


その後、追加ボタンをクリックします。これでセットアップは完了です。

仮想マシンを作成する


仮想マシンを作成するには、次の一連のアクションを実行します。

  1. オペレーティングシステムのバージョンを決定します。
  2. 事前にISOイメージをダウンロードしてください。
  3. [ ストレージ ]メニューで、新しく作成したストレージを選択します。
  4. コンテンツ ➝ ダウンロードをクリックします。
  5. リストからISOイメージを選択し、[ ダウンロード ]ボタンをクリックして選択を確認します。

操作が完了すると、画像が使用可能なもののリストに表示されます。


最初の仮想マシンを作成します。

  1. [ Create VM]をクリックします。
  2. ：私たちは、次のパラメータを入力名 ➝ ISOイメージ ➝ ハードディスクのサイズと種類 ➝ プロセッサの数 ➝ RAMサイズ ➝ ネットワークアダプタ。
  3. 必要なパラメータをすべて選択したら、[ 完了 ]をクリックします。作成されたマシンは、コントロールパネルのメニューに表示されます。
  4. それを選択し、[ 実行 ]をクリックします。
  5. コンソール項目に移動し、通常の物理サーバーと同じ方法でオペレーティングシステムをインストールします。

別のマシンを作成する必要がある場合-上記の操作を繰り返します。すべての準備が整ったら、同時に操作して、いくつかのコンソールウィンドウを開きます。

自動実行を構成する


デフォルトでは、Proxmoxは自動的にマシンを起動しませんが、これは2回クリックするだけで簡単に解決できます。

  1. 必要な車の名前をクリックします。
  2. 選択して[オプション ]タブ➝ ブート時に起動します。
  3. 同名の銘刻の前にダニを入れています。

これで、物理サーバーが再起動すると、VMが自動的に起動します。


上級管理者の場合は、[開始/シャットダウンの順序]セクションで追加の起動パラメーターを指定することもできます。マシンを起動する順序を明示的に指定できます。次のVMの開始前に経過する時間とシャットダウン遅延時間を指定することもできます（オペレーティングシステムに作業を完了する時間がない場合、ハイパーバイザーは一定の秒数後に強制的にシャットダウンします）。

この記事では、Proxmox VEを使い始める方法の基本を概説しました。初心者の専門家が最初のステップを踏み、仮想化を実際に試すのに役立つことを願っています。

Proxmox VEは、システム管理者にとって非常に強力で便利なツールです。主なことは、それが実際にどのように機能するかを実験して理解することを恐れないことです。
