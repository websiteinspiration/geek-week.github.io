こんにちは、％ユーザー名％。 6年間何もしないでブログを公開し、コミュニティに役立つものを提供するためにもう一度試す時が来ました。

ハブのSwarmpitに関する記事がまだないことに非常に驚いています。もちろん、Dockerを研究しているほとんどの人はおそらくそれを止めてから、Kubernetesと呼ばれるエベレスト山に登ろうとすることを理解しています。正確な数はわかりませんが、ほとんどが失われ、Dockerで殺されるか、そのまま使用されると思います。しかし、無駄です！

Docker Swarmは、ほぼすべての人にとってほぼ完璧なシステムです。非常に多くの人がKubernetesを必要としているのではないでしょうか。私はほぼ1日間彼を殺しましたが、あと30名殺すと後で落ち着いて使えるようになることに気付きましたが、私は単純なプログラマーであり、この素晴らしいDevOpsはすべてプログラミングに関するものではありません。そして、Docker Swarmについて聞いたところ、1日で簡単に侵入できました。

誰がDocker Swarmを必要としていますか？すでにDockerエコシステムの優れた機能が盛り込まれています。アプリケーションはマイクロサービスで構成されています。同時に、いくつかのサーバーがあり、フォールトトレランスを確保し、負荷をなんとかして管理したいと考えています。

そして、この記事では、それを構成する方法とそれが何であるかについては説明しません。さらに進んで、すでに構成されているクラスターを操作するための非常に単純なツールを示します。はい、クラスターを構成する必要があります。これは実際にはいくつかのコマンドです（詳細はこちら）：

docker swarm init
docker swarm join
