タイルカードで使用できる手続き型パターン

手続き型生成は、ゲームの多様性を高めるために使用されます。よく知られているプロジェクトには、Minecraft、Enter the Gungeon、およびDescendersがあります。この投稿では、Unity 2017.2で2D関数として表示されたTilemapシステムとRuleTileを使用するときに使用できるいくつかのアルゴリズムについて説明します。

カードを手続き的に作成することで、合格する各ゲームはユニークになります。時間やプレーヤーの現在のレベルなど、さまざまな入力データを使用して、ゲームのアセンブリ後でもコンテンツを動的に変更できます。

この投稿については何ですか？


手続き型の世界を作成する最も一般的な方法のいくつかと、私が作成したいくつかのバリエーションを見ていきます。以下は、記事を読んだ後に作成できるものの例です。3つのアルゴリズムが連携して、TilemapとRuleTileを使用してマップを作成します。


任意のアルゴリズムを使用してマップを生成するプロセスで、intすべての新しいデータを含む配列を取得します。このデータを引き続き変更するか、タイルマップにレンダリングできます。

さらに読む前に、次のことを知っておくとよいでしょう。

  1. タイルとは何か、バイナリ値を使用していないものを区別します。1はタイル、0はタイルなしです。
  2. すべてのカードを、各関数の最後にユーザーに返される2次元整数配列に格納します（レンダリングが実行されるものを除く）。
  3. GetUpperBound（）配列関数を使用して各マップの高さと幅を取得します。これにより、関数が受け取る変数が少なくなり、コードが簡潔になります。
  4. Tilemap座標系は左下から始まり、Mathf.FloorToInt（）を使用して数値を整数に丸めることができるため、私はMathf.FloorToInt（）をよく使用します。
  5. この投稿のすべてのコードはC＃で書かれています。

アレイの生成


GenerateArrayは、int指定されたサイズの新しい配列を作成します。また、配列を埋めるか空にするか（1または0）を指定することもできます。これがコードです：

public static int[,] GenerateArray(int width, int height, bool empty)
{
    int[,] map = new int[width, height];
    for (int x = 0; x < map.GetUpperBound(0); x++)
    {
        for (int y = 0; y < map.GetUpperBound(1); y++)
        {
            if (empty)
            {
                map[x, y] = 0;
            }
            else
            {
                map[x, y] = 1;
            }
        }
    }
    return map;
}

地図のレンダリング


この関数は、タイルマップ上にマップをレンダリングするために使用されます。マップの幅と高さをループし、テストされたポイントの配列の値が1の場合にのみタイルを配置します。

public static void RenderMap(int[,] map, Tilemap tilemap, TileBase tile)
{
    //Clear the map (ensures we dont overlap)
    tilemap.ClearAllTiles(); 
    //Loop through the width of the map
    for (int x = 0; x < map.GetUpperBound(0) ; x++) 
    {
        //Loop through the height of the map
        for (int y = 0; y < map.GetUpperBound(1); y++) 
        {
            // 1 = tile, 0 = no tile
            if (map[x, y] == 1) 
            {
                tilemap.SetTile(new Vector3Int(x, y, 0), tile); 
            }
        }
    }
}

地図の更新


この関数は、マップの更新にのみ使用され、再レンダリングには使用されません。これにより、各タイルとそのタイルデータを再描画することなく、より少ないリソースを使用できます。

public static void UpdateMap(int[,] map, Tilemap tilemap) //Takes in our map and tilemap, setting null tiles where needed
{
    for (int x = 0; x < map.GetUpperBound(0); x++)
    {
        for (int y = 0; y < map.GetUpperBound(1); y++)
        {
            //We are only going to update the map, rather than rendering again
            //This is because it uses less resources to update tiles to null
            //As opposed to re-drawing every single tile (and collision data)
            if (map[x, y] == 0)
            {
                tilemap.SetTile(new Vector3Int(x, y, 0), null);
            }
        }
    }
}

ノイズパーリン


パーリンノイズはさまざまな目的に使用できます。まず、それを使用してマップの最上位レイヤーを作成できます。これを行うには、現在の位置xとシードを使用して新しい点を取得します。

シンプルなソリューション


この生成方法では、レベル生成で最も単純な形式のPerlinノイズを使用します。PerlinノイズのUnity関数を使用して、自分でコードを記述しないようにすることができます。また、Mathf.FloorToInt（）関数を使用して、タイルマップに整数のみを使用します。

public static int[,] PerlinNoise(int[,] map, float seed)
{
    int newPoint;
    //Used to reduced the position of the Perlin point
    float reduction = 0.5f;
    //Create the Perlin
    for (int x = 0; x < map.GetUpperBound(0); x++)
    {
        newPoint = Mathf.FloorToInt((Mathf.PerlinNoise(x, seed) - reduction) * map.GetUpperBound(1));

        //Make sure the noise starts near the halfway point of the height
        newPoint += (map.GetUpperBound(1) / 2); 
        for (int y = newPoint; y >= 0; y--)
        {
            map[x, y] = 1;
        }
    }
    return map;
}

平滑化


この機能を使用して、スムーズにすることもできます。パーリンの高さを修正する間隔を設定し、これらのポイント間でスムージングを実行します。間隔については整数値のリストを考慮する必要があるため、この関数はもう少し複雑になります。

public static int[,] PerlinNoiseSmooth(int[,] map, float seed, int interval)
{
    //Smooth the noise and store it in the int array
    if (interval > 1)
    {
        int newPoint, points;
        //Used to reduced the position of the Perlin point
        float reduction = 0.5f;

        //Used in the smoothing process
        Vector2Int currentPos, lastPos; 
        //The corresponding points of the smoothing. One list for x and one for y
        List<int> noiseX = new List<int>();
        List<int> noiseY = new List<int>();

        //Generate the noise
        for (int x = 0; x < map.GetUpperBound(0); x += interval)
        {
            newPoint = Mathf.FloorToInt((Mathf.PerlinNoise(x, (seed * reduction))) * map.GetUpperBound(1));
            noiseY.Add(newPoint);
            noiseX.Add(x);
        }

        points = noiseY.Count;

この関数の最初の部分では、間隔が1より大きいかどうかを最初にチェックします。もしそうなら、ノイズを生成します。平滑化を適用できるように、間隔を置いて生成を実行します。関数の次の部分は、ポイントを滑らかにすることです。

//Start at 1 so we have a previous position already
            for (int i = 1; i < points; i++) 
            {
                //Get the current position
                currentPos = new Vector2Int(noiseX[i], noiseY[i]);
                //Also get the last position
                lastPos = new Vector2Int(noiseX[i - 1], noiseY[i - 1]);

                Vector2 diff = currentPos - lastPos;

                //Set up what the height change value will be 
                float heightChange = diff.y / interval;
                //Determine the current height

                //Work our way through from the last x to the current x
                for (int x = lastPos.x; x < currentPos.x; x++)
                {
                    for (int y = Mathf.FloorToInt(currHeight); y > 0; y--)
                    {
                        map[x, y] = 1;
                    }
                    currHeight += heightChange;
                }
            }
        }

平滑化は次のように実行されます。

  1. 現在と最後の位置を取得します
  2. 2点間の差を取得します。最も重要な情報は、y軸に沿った差です。
  3. 次に、ポイントに到達するために必要な変更の量を決定します。これは、yの差を間隔変数で割ることによって行われます。
  4. 次に、位置を設定し始め、ゼロまで進みます
  5. y軸で0に到達したら、高さの変更を現在の高さに追加し、次のx位置についてプロセスを繰り返します
  6. 最後の位置と現在の位置の間の各位置で終了すると、次のポイントに移動します

間隔が1未満の場合は、前の関数を使用するだけで、すべての処理が行われます。

ランダムウォークトップ


このアルゴリズムは、コイン投げを実行します。2つの結果のうちの1つを取得できます。結果が「イーグル」の場合は1ブロック上に移動し、結果が「テール」の場合はブロックを下に移動します。これにより、絶えず上下に移動して高さが作成されます。このようなアルゴリズムの唯一の欠点は、非常に目立つブロックネスです。それがどのように機能するかを見てみましょう。

public static int[,] RandomWalkTop(int[,] map, float seed)
{
    //Seed our random
    System.Random rand = new System.Random(seed.GetHashCode()); 

    //Set our starting height
    int lastHeight = Random.Range(0, map.GetUpperBound(1));
        
    //Cycle through our width
    for (int x = 0; x < map.GetUpperBound(0); x++) 
    {
        //Flip a coin
        int nextMove = rand.Next(2);

        //If heads, and we aren't near the bottom, minus some height
        if (nextMove == 0 && lastHeight > 2) 
        {
            lastHeight--;
        }
        //If tails, and we aren't near the top, add some height
        else if (nextMove == 1 && lastHeight < map.GetUpperBound(1) - 2) 
        {
            lastHeight++;
        }

        //Circle through from the lastheight to the bottom
        for (int y = lastHeight; y >= 0; y--) 
        {
            map[x, y] = 1;
        }
    }
    //Return the map
    return map; 
}


ランダムウォークトップとスムージング

この生成では、Perlinノイズ生成に比べて高さが滑らかになります。

このランダムウォークのバリエーションにより、以前のバージョンと比較してよりスムーズな結果が得られます。関数にさらに2つの変数を追加することで実装できます。

  • 最初の変数は、現在の高さを維持するのにかかる時間を決定するために使用されます。これは整数であり、高さが変わるとリセットされます
  • 2番目の変数は関数に入力され、高さの最小セクション幅として使用されます。関数を見るとわかりやすくなります。

これで、何を追加する必要があるかがわかりました。関数を見てみましょう：

public static int[,] RandomWalkTopSmoothed(int[,] map, float seed, int minSectionWidth)
{
    //Seed our random
    System.Random rand = new System.Random(seed.GetHashCode());

    //Determine the start position
    int lastHeight = Random.Range(0, map.GetUpperBound(1));

    //Used to determine which direction to go
    int nextMove = 0;
    //Used to keep track of the current sections width
    int sectionWidth = 0;

    //Work through the array width
    for (int x = 0; x <= map.GetUpperBound(0); x++)
    {
        //Determine the next move
        nextMove = rand.Next(2);

        //Only change the height if we have used the current height more than the minimum required section width
        if (nextMove == 0 && lastHeight > 0 && sectionWidth > minSectionWidth)
        {
            lastHeight--;
            sectionWidth = 0;
        }
        else if (nextMove == 1 && lastHeight < map.GetUpperBound(1) && sectionWidth > minSectionWidth)
        {
            lastHeight++;
            sectionWidth = 0;
        }
        //Increment the section width
        sectionWidth++;

        //Work our way from the height down to 0
        for (int y = lastHeight; y >= 0; y--)
        {
            map[x, y] = 1;
        }
    }

    //Return the modified map
    return map;
}

ノイズパーリン


前のパートでは、Perlinノイズを適用してトップレイヤーを作成する方法について説明しました。幸いなことに、Perlinのノイズを使って洞窟を作成することもできます。これは、モディファイアを掛けた現在位置のパラメータを受け取る新しいパーリンノイズ値を計算することによって実現されます。修飾子は0〜1の値です。修飾子の値が大きいほど、パーリンの生成はより混乱します。次に、この値を整数（0または1）に丸め、それをマップ配列に格納します。これがどのように実装されているかを見てください：

public static int[,] PerlinNoiseCave(int[,] map, float modifier, bool edgesAreWalls)
{
	int newPoint;
	for (int x = 0; x < map.GetUpperBound(0); x++)
	{
		for (int y = 0; y < map.GetUpperBound(1); y++)
		{

			if (edgesAreWalls && (x == 0 || y == 0 || x == map.GetUpperBound(0) - 1 || y == map.GetUpperBound(1) - 1))
			{
				map[x, y] = 1; //Keep the edges as walls
			}
			else
			{
				//Generate a new point using Perlin noise, then round it to a value of either 0 or 1
				newPoint = Mathf.RoundToInt(Mathf.PerlinNoise(x * modifier, y * modifier));
				map[x, y] = newPoint;
			}
		}
	}
	return map;
}

シードの代わりに修飾子を使用します。Perlinの生成結果は、0から0.5までの数を掛けた方がよく見えるからです。値が低いほど、結果はブロック状になります。サンプル結果を見てください。Gifはモディファイア値0.01で始まり、徐々に0.25の値に達します。

public static int[,] RandomWalkCave(int[,] map, float seed,  int requiredFloorPercent)
{
    //Seed our random
    System.Random rand = new System.Random(seed.GetHashCode());

    //Define our start x position
    int floorX = rand.Next(1, map.GetUpperBound(0) - 1);
    //Define our start y position
    int floorY = rand.Next(1, map.GetUpperBound(1) - 1);
    //Determine our required floorAmount
    int reqFloorAmount = ((map.GetUpperBound(1) * map.GetUpperBound(0)) * requiredFloorPercent) / 100; 
    //Used for our while loop, when this reaches our reqFloorAmount we will stop tunneling
    int floorCount = 0;

    //Set our start position to not be a tile (0 = no tile, 1 = tile)
    map[floorX, floorY] = 0;
    //Increase our floor count
    floorCount++;

  2. 削除する床タイルの数を計算します
  3. 開始位置のタイルを削除します
  4. タイルの数に1を加えます。

何が起こっていますか？


まず、幅の値を設定します。幅の値は、マイナスの値からプラスの値になります。このおかげで、必要なサイズが得られます。この場合、値1を使用します。

今度は、値-1、0、1を使用するため、合計幅は3になります。次に、xの初期位置を設定します。これには、マップの幅の中央を使用します。その後、マップの最初の部分にトンネルを敷くことができます。


それでは、マップの残りの部分に入りましょう。

セルラーオートマトン


セルオートマトンは隣接セルを使用して、現在のセルがオン（1）かオフ（0）かを判断します。隣接するセルを決定するための基礎は、ランダムに生成されたセルのグリッドに基づいて作成されます。このソースメッシュは、C＃Random.Next関数を使用して生成します。

セルオートマトンの実装がいくつかあるので、この基本的なグリッドを生成する別の関数を書きました。関数は次のようになります。

public static int[,] GenerateCellularAutomata(int width, int height, float seed, int fillPercent, bool edgesAreWalls)
{
    //Seed our random number generator
    System.Random rand = new System.Random(seed.GetHashCode());

    //Initialise the map
    int[,] map = new int[width, height];

    for (int x = 0; x < map.GetUpperBound(0); x++)
    {
        for (int y = 0; y < map.GetUpperBound(1); y++)
        {
            //If we have the edges set to be walls, ensure the cell is set to on (1)
            if (edgesAreWalls && (x == 0 || x == map.GetUpperBound(0) - 1 || y == 0 || y == map.GetUpperBound(1) - 1))
            {
                map[x, y] = 1;
            }
            else
            {
                //Randomly generate the grid
                map[x, y] = (rand.Next(0, 100) < fillPercent) ? 1 : 0; 
            }
        }
    }
    return map;
}

ムーアの近所


ムーア周辺は、セルオートマトンの初期生成をスムーズにするために使用されます。ムーアの近所は次のようになります。


次のルールが近隣に適用されます：

  • 各方向の隣人をチェックします。
  • ネイバーがアクティブなタイルである場合、周囲のタイルの数に1を加えます。
  • 隣人が非アクティブなタイルである場合、何もしません。
  • セルの周囲のタイルが4つを超える場合は、セルをアクティブにします。
  • セルの周囲のタイルがちょうど4つある場合は、何もしません。
  • 各マップタイルを確認するまで繰り返します。

ムーアの近傍チェック関数は次のとおりです。

static int GetMooreSurroundingTiles(int[,] map, int x, int y, bool edgesAreWalls)
{
    /* Moore Neighbourhood looks like this ('T' is our tile, 'N' is our neighbours)
     * 
     * N N N
     * N T N
     * N N N
     * 
     */

    int tileCount = 0;       
    
    for(int neighbourX = x - 1; neighbourX <= x + 1; neighbourX++)
    {
        for(int neighbourY = y - 1; neighbourY <= y + 1; neighbourY++)
        {
            if (neighbourX >= 0 && neighbourX < map.GetUpperBound(0) && neighbourY >= 0 && neighbourY < map.GetUpperBound(1))
            {
                //We don't want to count the tile we are checking the surroundings of
                if(neighbourX != x || neighbourY != y) 
                {
                    tileCount += map[neighbourX, neighbourY];
                }
            }
        }
    }
    return tileCount;
}

タイルをチェックした後、この情報をスムージング機能で使用します。ここでは、セルオートマトンの生成と同様に、マップのエッジを壁にするかどうかを指定できます。

public static int[,] SmoothMooreCellularAutomata(int[,] map, bool edgesAreWalls, int smoothCount)
{
	for (int i = 0; i < smoothCount; i++)
	{
		for (int x = 0; x < map.GetUpperBound(0); x++)
		{
			for (int y = 0; y < map.GetUpperBound(1); y++)
			{
				int surroundingTiles = GetMooreSurroundingTiles(map, x, y, edgesAreWalls);

				if (edgesAreWalls && (x == 0 || x == (map.GetUpperBound(0) - 1) || y == 0 || y == (map.GetUpperBound(1) - 1)))
				{
                    //Set the edge to be a wall if we have edgesAreWalls to be true
					map[x, y] = 1; 
				}
                //The default moore rule requires more than 4 neighbours
				else if (surroundingTiles > 4)
				{
					map[x, y] = 1;
				}
				else if (surroundingTiles < 4)
				{
					map[x, y] = 0;
				}
			}
		}
	}
    //Return the modified map
    return map;
}

ここで、関数にはfor指定された回数の平滑化を実行するループがあることに注意することが重要です。これにより、より美しいカードが得られます。

フォンノイマン地区は、セルオートマトンを実装するもう1つの一般的な方法です。そのような世代には、ムーア世代よりも単純な近傍を使用します。近所はこのようになります：


次のルールが近隣に適用されます：

  • 対角線を考慮せずに、タイルのすぐ隣をチェックします。
  • セルがアクティブな場合は、数量に1を加えます。
  • セルがアクティブでない場合は、何もしません。

  • セルの隣接セルが2つ未満の場合は、現在のセルを非アクティブにします。
  • 隣接セルが2つしかない場合は、現在のセルを変更しないでください。

次の関数でネイバーをチェックします。

static int GetVNSurroundingTiles(int[,] map, int x, int y, bool edgesAreWalls)
{
	/* von Neumann Neighbourhood looks like this ('T' is our Tile, 'N' is our Neighbour)
	* 
	*   N 
	* N T N
	*   N
	*   
    */
     
	int tileCount = 0;

    //Keep the edges as walls
    if(edgesAreWalls && (x - 1 == 0 || x + 1 == map.GetUpperBound(0) || y - 1 == 0 || y + 1 == map.GetUpperBound(1)))
    {
        tileCount++;
    }

    //Ensure we aren't touching the left side of the map
	if(x - 1 > 0)
	{
		tileCount += map[x - 1, y];
	}

    //Ensure we aren't touching the bottom of the map
	if(y - 1 > 0)
	{
		tileCount += map[x, y - 1];
	}

    //Ensure we aren't touching the right side of the map
	if(x + 1 < map.GetUpperBound(0)) 
	{
		tileCount += map[x + 1, y];
	}

    //Ensure we aren't touching the top of the map
	if(y + 1 < map.GetUpperBound(1)) 
	{
		tileCount += map[x, y + 1];
	}

	return tileCount;
}

隣人の数を受け取ったら、配列の平滑化に進むことができます。前と同様に、for入力に渡された平滑化反復の数を完了するためのループが必要です。

public static int[,] SmoothVNCellularAutomata(int[,] map, bool edgesAreWalls, int smoothCount)
{
	for (int i = 0; i < smoothCount; i++)
	{
		for (int x = 0; x < map.GetUpperBound(0); x++)
		{
			for (int y = 0; y < map.GetUpperBound(1); y++)
			{
				//Get the surrounding tiles
				int surroundingTiles = GetVNSurroundingTiles(map, x, y, edgesAreWalls);

				if (edgesAreWalls && (x == 0 || x == map.GetUpperBound(0) - 1 || y == 0 || y == map.GetUpperBound(1)))
				{
                    //Keep our edges as walls
					map[x, y] = 1; 
				}
                //von Neuemann Neighbourhood requires only 3 or more surrounding tiles to be changed to a tile
				else if (surroundingTiles > 2) 
				{
					map[x, y] = 1;
				}
				else if (surroundingTiles < 2)
				{
					map[x, y] = 0;
				}
			}
		}
	}
    //Return the modified map
    return map;
}
