Rebanadas de hojas de Sprite


A menudo, en juegos 2D, usan una imagen grande para almacenar varias imágenes más pequeñas, por ejemplo, mosaicos en mosaicos en lugar de muchas imágenes pequeñas para cada mosaico. Dicha imagen se llama hoja de sprite y es muy conveniente para el trabajo, ya que no necesitamos cambiar la textura que queremos dibujar, solo indicamos qué parte de la textura usar.

En este tutorial, veremos cómo seleccionar partes de una textura usando SDL_RenderCopy , así como un poco sobre cómo detectar eventos de pulsación de teclas específicos que usaremos para seleccionar qué parte de la textura dibujar. Habrá cuatro círculos coloridos en la hoja de sprites:



En este tutorial, una hoja de sprites consta de muchos sprites del mismo tamaño, en cuyo caso el corte no es difícil. De lo contrario, para sprites de diferentes tamaños, necesitaríamos un archivo de metadatos que contenga información sobre la ubicación de las partes. Para este tutorial usaremos 4 sprites de tamaño 100x100. El código para esta lección se basa en la lección 4, si no tiene un código para escribir, puede obtenerlo de Github .

Selección de parte


Con el SDL, es muy fácil seleccionar la parte de la textura que desea dibujar. En la lección 4, los parámetros restantes de SDL_RenderCopy con un valor NULL indican las coordenadas del rectángulo que determina qué parte de la textura queremos dibujar. Al pasar un valor NULL, indicamos que necesitamos toda la textura, pero podemos agregar fácilmente los parámetros del rectángulo y dibujar solo una parte de la textura. Para hacer esto, realice cambios en la función renderTexture para que pueda ocupar un área rectangular, pero aún así mantenga una versión corta de la sintaxis de la versión anterior por conveniencia.

Cambiar renderTexture


Para no vincular más y más parámetros a nuestra función renderTexture y al mismo tiempo mantener la conveniencia de los valores predeterminados, lo dividiremos en dos funciones. El primero es casi idéntico a la llamada SDL_RenderCopy, pero proporciona un parámetro de región de recorte con un valor nullptr. Esta versión de renderTexture recibirá una ubicación en forma de área rectangular, que podemos configurar nosotros mismos o usar una de nuestras otras funciones especializadas de renderTexture. La nueva función básica de renderizado se está volviendo muy simple.

/**
*  SDL_Texture  SDL_Renderer   .
*     
* @param tex  ,   
* @param ren ,    
* @param dst     
* @param clip     ( )
*                 nullptr   
*/
void renderTexture(SDL_Texture *tex, SDL_Renderer *ren, SDL_Rect dst,
        SDL_Rect *clip = nullptr)
{
        SDL_RenderCopy(ren, tex, clip, &dst);
}

Por conveniencia, escribiremos otra función en la que no necesitemos crear SDL_Rect para la ubicación, sino que solo proporcione x e y y deje que nuestra función de visualización llene el ancho y la altura de la textura. Crearemos una versión sobrecargada de renderTexture que hará esto con algunas configuraciones para manejar el recorte. Agregue un rectángulo de corte como parámetro con el valor predeterminado nullptr y, en caso de que se apruebe, usaremos el ancho y la altura del corte en lugar del ancho y la altura de la textura. Por lo tanto, no estiraremos el sprite pequeño al tamaño de su hoja de sprite potencialmente muy grande cuando se dibuje. Esta función es una modificación de la función renderTexture original y se ve muy similar.

/**
*  SDL_Texture  SDL_Renderer   x, y, 
*          
*    ,       
*          
* @param tex  ,   
* @param ren ,    
* @param x  x,    
* @param y  y,    
* @param clip     ( )
*                 nullptr   
*/
void renderTexture(SDL_Texture *tex, SDL_Renderer *ren, int x, int y,
        SDL_Rect *clip = nullptr)
{
        SDL_Rect dst;
        dst.x = x;
        dst.y = y;
        if (clip != nullptr){
                dst.w = clip->w;
                dst.h = clip->h;
        }
        else {
                SDL_QueryTexture(tex, NULL, NULL, &dst.w, &dst.h);
        }
        renderTexture(tex, ren, dst, clip);
}

Definir rectángulos de recorte


En nuestro caso, es muy fácil calcular rectángulos de recorte utilizando el método muy similar al método de mosaico de la lección 3 , sin embargo, en lugar de ir línea por línea, iremos columna por columna. Por lo tanto, la primera pieza será verde, la segunda será roja, la tercera será azul y la cuarta será amarilla. La idea de computación es la misma que en la lección 3, pero solo en lugar de filas corremos a través de las columnas. Por lo tanto, nuestras coordenadas y se calculan obteniendo el resto al dividir el índice de mosaico por el número de mosaicos (2), y la coordenada x dividiendo el índice por el número de mosaicos. Estas coordenadas x e y son índices x e y, por lo que las traducimos en coordenadas de píxeles reales al multiplicarlas por el ancho y la altura del recorte, que es el mismo para todos los mosaicos (100x100). Finalmente, seleccione la pieza a dibujar, en este caso la primera.

También nos gustaría dibujar nuestras piezas en el centro de la pantalla, por lo que calculamos estas coordenadas xey utilizando el ancho y la altura del mosaico en lugar del ancho y la altura de la textura.

//iW  iH    
//   ,     ,     
int iW = 100, iH = 100;
int x = SCREEN_WIDTH / 2 - iW / 2;
int y = SCREEN_HEIGHT / 2 - iH / 2;

//    
SDL_Rect clips[4];
for (int i = 0; i < 4; ++i){
        clips[i].x = i / 2 * iW;
        clips[i].y = i % 2 * iH;
        clips[i].w = iW;
        clips[i].h = iH;
}
//     
int useClip = 0;

Si, en cambio, usáramos una hoja de sprites más compleja con sprites rotados de diferentes tamaños empaquetados, necesitaríamos almacenar información sobre su ubicación y rotación en algún archivo de metadatos para que podamos encontrar fácilmente las piezas.

Cambiar imágenes según la entrada


Para verificar todas las partes de la imagen que creamos, agregamos el procesamiento de entrada del teclado al bucle de procesamiento de eventos y seleccionamos la parte visualizada usando las teclas 1-4. Para determinar si se ha presionado una tecla, puede verificar si el evento es del tipo SDL_KEYDOWN y, de ser así, podemos averiguar qué tecla se presionó al verificar el código de tecla dentro del evento usando e.key.keysym.sym. En la wiki hay disponible una lista completa de tipos de eventos , códigos clave y otra información sobre SDL_Event .

Cuando se presiona la tecla, necesitamos cambiar el valor de useClip al número de la parte de la imagen que queremos dibujar. Con estos cambios, el bucle de eventos se ve así:

while (SDL_PollEvent(&e)){
        if (e.type == SDL_QUIT)
                quit = true;
        //       
        if (e.type == SDL_KEYDOWN){
                switch (e.key.keysym.sym){
                        case SDLK_1:
                                useClip = 0;
                                *break;
                        case SDLK_2:
                                useClip = 1;
                                *break;
                        case SDLK_3:
                                useClip = 2;
                                *break;
                        case SDLK_4:
                                useClip = 3;
                                *break;
                        case SDLK_ESCAPE:
                                quit = true;
                                *break;
                        default:
                                *break;
                }
        }
}

Dibuja una imagen recortada


¡Lo último que debe hacer es obtener la parte correcta de la imagen en la pantalla! Haremos esto llamando a nuestra versión más conveniente de renderTexture para dibujar parte de la imagen sin escalar y transferir la parte que queremos usar (la que se usa en useClip).

SDL_RenderClear(renderer);
renderTexture(image, renderer, x, y, &clips[useClip]);
SDL_RenderPresent(renderer);
