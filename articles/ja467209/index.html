「自律エージェント」またはオープン暗号プラットフォームObyteでコードを実行



Obyteは、ブロックチェーンの代わりにトランザクションレジストリとして有向非巡回グラフ（DAG、サイクルなしの有向グラフ）を使用するオープンな暗号化プラットフォームであり、すべての参加者がレジストリに無料で平等にアクセスできます。誰にとっても通常のブロックチェーンとは異なり、DAGは、ブロックの拒否とトランザクションの厳密な順序付けにより、マイナーの参加なしに分散レジストリに直接データを送信することを可能にします。ブロックチェーンと比較して、DAGアーキテクチャはマイナー、偽造者、ブロックプロデューサーなどのユーザーの特権クラスを必要としないため、ユーザーとネットワーク上のレジストリーの間に仲介者はなく、誰もがレジストリーにトランザクションを追加するための平等かつ直接的なアクセス権を持っています。

この記事では、Obyte、Autonomous Agentsについて話し、これらすべてをEthereumと比較します。

自律エージェントとは何ですか？ 


これは、ソースコードがレジストリ（DAG）に格納され、他のネットワーク参加者のアドレスと同様に独自のアドレスを持つプログラムです。プログラムは、このアドレスへのコインとデータの送信に応じて実行されるアクションを記述します。イーサリアムのスマートコントラクトに精通している場合は、自律エージェントの意味は非常に近いです。

人生の例は自動販売機です。キーボードで入力された製品番号と金額を入力すると、デバイスはユーザーに製品自体を提供します。 AAと自動販売機の間には1つの重要な違いがあります。自動販売機は所有者によって制御され、ユーザーはデバイスのアクションがユーザーの期待に一致することを望んでいるだけです。自律エージェントは真に自律的です。プログラム全体がDAGに格納され、そのソースコードはすべてのネットワーク参加者に表示され、変更、無効化、または削除することはできません。

たとえば、AAをプログラミングしてトークンを交換できます。プログラミングの場合、これらのタスク用に開発された言語はOscriptです。
Obyteにはスマートコントラクトの概念もありますが、AAやEthereumのスマートコントラクトとは異なります。 AAスマートコントラクトとObyteスマートコントラクトの主な違いは、現実世界の通常のコントラクトと同様に、スマートコントラクトは2人の参加者間で動作し、コントラクトを実行できる条件を示します。簡単に言えば、トランザクションを許可または禁止する一連の条件です。同時に、自律エージェントは独立してプログラムされた参加者であり、相互作用して新しい応答などを開始します。

AAコードの実行は、トリガートランザクションが安定した後に開始されます（つまり、トランザクションにコンセンサスがあり、二重の支出は絶対にありません）。各フルノードは、レジストリのコピーでAAコードを実行し、実行結果をデータベースにのみ記録します。実行結果を他のノードに中継する必要はありません。ネットワークのすべての参加者の状態は同じであり、実行の結果も確定的であり、誰にとっても同じです。レジストリの最終的な状態は、AA実行のルールによってのみ決定され、鉱山労働者、有権者、または他の誰によっても制御されません。

ユーザーの透明性


存在初日からのオーバイトは、人間が理解できる形で人々の間でスマートな契約を結ぶ機会を提供します。透明性とユーザーによる認識の容易さは、これまでも、プラットフォームを作成する上での主要な原則の1つです。

Oscriptはプログラミング言語であり、機械とプログラマーによる認識のために作成されたものであり、一般の人々ではないという事実にもかかわらず、私たちは原則から逸脱しないように努めました。したがって、プログラミングに慣れていない人のために、GUIウォレットはトランザクションが完了する前にAAコードを解釈して、彼が理解できる方法で結果を表示します。



Oscript-スタンドアロンエージェント言語


Cのような言語に精通している開発者は、AAでの作業に問題はありません。

最初に、特徴を明らかにしたいと思います。 

AAはトランザクションのトリガーでのみ機能します。支払いのみ、またはAAデータ転送のいずれかです。たとえば、データ{foo： 'bar'、value：100}を渡すと、AAがinputとして受け取り、これに基づいてアクションが実行されます。

AAがアクセスできるもの


  • DAGの現在の状態を説明する変数
  • トリガートランザクションで受け取った金額
  • トランザクショントリガーからのデータ
  • トリガートランザクションを送信したユーザー
  • 私たちと他のAAの状態変数
  • DAGからのさまざまなパブリックデータへのアクセス（oraclesからのdata_feed、実名証明など）
  • 私たちと他のAAのバランス
  • 資産情報（資産）

Oscriptの機能


Oscript-完全な言語をチューリングしないこと。ただし、その武器には次のようなものがあります。

  • 算術演算
  • 論理演算（および、またはなど）
  • 比較
  • 文字列連結
  • いくつかの数学関数
  • 暗号化機能（ハッシュ作成、署名検証）
  • 分岐（if / else）
  • ループや関数はありません（無限再帰を防ぐため）。詳細については、このリンクのマニュアルを参照してください

リソースの使用を制御し、悪意のあるスクリプトを防ぐために、AAは暗号化機能やデータベースアクセスなど、最大100の労働集約的な操作に制限されています。検証は、エージェントコードの展開時に行われます。

ただし、AAパフォーマンスのコストを計算するための複雑な式はありません。そのような式はまったくなく、コードの実行は無料です。AA自身のようなユーザーは、他のネットワークメンバーと同様に、DAGレジストリにデータを追加することに対してのみ支払います-DAGに追加されるデータのバイトごとに1バイトのObyte通貨。ただし、AAは、そのバランスを、格納するすべての状態変数のサイズ以上のバイト単位で維持する必要があります。

実行中にAAが別のAAの実行をトリガーする場合、最初に最初のAAの結果として生じるトランザクションがレジストリに記録され、その後初めて次のAAが実行を開始します。

自律エージェントの動作


次に、最も単純なピンポンスクリプトを記述します。私たちはAAのお金を送ります、彼は自分にいくらかの金額を残して残りを送り返します。ObyteとEthereumの 2つのプラットフォームで同じ問題を解決し、比較します。イーサリアムでは、同様のAAエンティティは一般にスマートコントラクトと呼ばれます（イーサリアムホワイトペーパーでは自律エージェントを1回だけと呼んでいますが）。

Obbyte


公開にはOscriptエディターを使用

します。提供されているテンプレートをクリアしてください。一から書いていきます。最小AAは、メッセージの配列を持つオブジェクトです。

{
messages: []
}

支払いを送信するので、これを示すプロパティを持つオブジェクトをメッセージに追加する必要があります。

{
messages: [
        {
            app: 'payment',
            payload: {
                asset: 'base',
                outputs: [
                    {address: "{trigger.address}", amount: "{trigger.output[[asset=base]] - 20000}"}
                ]
            }
        }
    ]
}

ここに、支払いの標準的なメタデータが表示されます。これらは、行で「{}」を使用して変更されます。値を転送するには、トリガーオブジェクトを使用します。トリガーオブジェクトは、受け取った支払いに関する情報を格納します。構文について詳しくは、ドキュメントをご覧ください。

このコードをエディターに貼り付け、[Deploy]をクリックします。すべて！
AAが発行され、以下にアドレスが表示されます。次に、testnet GUIウォレットが必要です。インストールして実行します。次に、テストコインを取得するには、「チャット」に移動し、「Bot Store」を選択して、そこに「Faucet Bot」を追加します。チャットが開きます。左側のメニューをクリックし、[アドレスを挿入]を選択します


次に、お金が届いたというメッセージ「支払い：...」を送信して待ちます。次に、メインのコインに移動して、「保留中の確認を含む」という表記が消えるまで数分待つ必要があります。これは、受け取ったコインが確認され、使用できることを意味します。

あとは、「受信」に進み、アドレスAAを挿入し、金額「500000」を入力して、「送信」を押すだけです。次に、トランザクションが安定するまで待つ必要があります（履歴を確認します）。これはトリガーするために必要です。お金が戻ってきたかどうかを確認してみましょう



。AAの探検家に行くと、彼の残高が20,000マイナス私たちに支払う料金を差し引いたことがわかります。Obyteでは、自律エージェント自身が実行するアクションに対して手数料を支払います。

調査結果


上記の比較からわかるように、Obyteは、発行、書き込み、および制御をはるかに簡単に実行できます。たとえば、イーサリアムでは、送信トランザクションのガスを正しく選択できることは常にあります。そのため、非常に長い間確認できます。イーサリアムの利点の中で、迅速な公開と確認を提供することができます（供給されるガスに応じて、約1分）。Obyteでの発行はすぐに行われ、確認時間はネットワークの負荷に依存します。負荷が軽い場合は約10〜15分、負荷が大きい場合は3分未満です。
