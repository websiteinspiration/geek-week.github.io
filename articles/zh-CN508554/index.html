如何在Go程序运行时确定变量的大小

摘要：该笔记讨论了一种通过Go应用程序的组件分析内存消耗的方法。



通常，数据结构存储在程序的内存中，它们在程序运行时会动态更改其大小。这种结构的示例可以是数据高速缓存或程序操作日志，或者是从外部系统接收的数据。在这种情况下，可能会出现以下情况：内存消耗增加，设备功能不足以及特定的泄漏机制不清楚。



剖析Go应用程序的主要方法是从“ net / http / pprof”包连接pprof工具。结果，您可以在运行的程序中获得具有内存分配的表或图形。但是使用此工具需要非常大的开销，并且可能不适用，尤其是在无法使用真实数据运行程序的多个实例的情况下。



在这种情况下，期望通过按需的程序对象来测量存储器消耗，例如以显示系统统计数据或将度量传递到监视系统。但是，用语言手段通常是不可能的。在程序运行时，Go没有用于调整变量大小的工具。



因此，我决定编写一个提供这种机会的小包装。主要工具是反射（包“反射”）我邀请所有对此类概要分析应用程序感兴趣的人继续阅读。







首先，谈谈内置函数。 



unsafe.Sizeof(value)


和 



reflect.TypeOf(value).Size()


这些函数是等效的，通常在Internet上建议使用这些函数来确定变量的大小。但是这些函数不会返回实际变量的大小，而是返回变量容器的大小（以字节为单位）（大约是指针的大小）。例如，对于int64类型的变量，这些函数将返回正确的结果，因为此类型的变量包含实际值，而不是对其的引用。但是对于包含指向实际值的指针的数据类型（例如切片或字符串），这些函数将为该类型的所有变量返回相同的值。该值对应于包含指向变量数据的链接的容器的大小。我将举一个例子说明： 



func main() {
	s1 := "ABC"
	s2 := "ABCDEF"
	arr1 := []int{1, 2}
	arr2 := []int{1, 2, 3, 4, 5, 6}
	fmt.Printf("Var: %s, Size: %v\n", s1, unsafe.Sizeof(s1))
	fmt.Printf("Var: %s, Size: %v\n", s2, unsafe.Sizeof(s2))
	fmt.Printf("Var: %v, Size: %v\n", arr1, reflect.TypeOf(arr1).Size())
	fmt.Printf("Var: %v, Size: %v\n", arr2, reflect.TypeOf(arr2).Size())
}


结果，我们得到： 



Var: ABC, Size: 16
Var: ABCDEF, Size: 16
Var: [1 2], Size: 24
Var: [1 2 3 4 5 6], Size: 24


如您所见，该变量的实际大小未计算。



标准库具有binary.Size（）函数，该函数以字节为单位返回变量的大小，但仅适用于固定大小的类型。也就是说，如果在结构的字段中遇到字符串，切片，关联数组或仅是int，则该功能不适用。但是，我将此功能作为size包的基础，在其中尝试将上述机制的功能扩展到没有固定大小的数据类型。



为了确定对象在程序运行期间的大小，有必要了解其类型以及所有嵌套对象的类型（如果它是结构）。通常需要以树的形式来表示需要分析的最终结构。因此，必须使用递归来确定复杂数据类型的大小。



因此，对任意对象消耗的内存量的计算表示如下：



  • 确定简单（非复合）类型变量大小的算法；
  • 对数组的元素，结构字段，键和关联数组的值进行算法的递归调用;
  • 无限循环的定义；


要确定简单类型变量（而不是数组或结构）的实际大小，可以使用上述反射包中的Size（）函数。此函数对包含实际值的变量正确起作用。对于数组变量，字符串，即包含指向值的链接，您需要浏览这些元素或字段并计算每个元素的值。 为了分析变量的类型和值，反射包将变量打包到一个空接口（interface {}）中。在Go中，空接口可以包含任何对象。此外，Go中的接口由包含两个字段的容器表示：实际值的类型和指向实际值的链接。







正是将分析值映射到空接口（反之亦然）作为方法本身名称（反射）的基础。



为了更好地了解Go中反射的工作原理，我推荐Rob Pike在Go 官方博客上的文章。这篇文章的翻译是在哈布雷。



最后，开发了size包，可以在您的程序中使用它，如下所示： 



package main

import (
	"fmt"

	"github.com/DmitriyVTitov/size"
)

func main() {
	a := struct {
		a int
		b string
		c bool
		d int32
		e []byte
		f [3]int64
	}{
		a: 10,                    // 8 bytes
		b: "Text",                // 4 bytes
		c: true,                  // 1 byte
		d: 25,                    // 4 bytes
		e: []byte{'c', 'd', 'e'}, // 3 bytes
		f: [3]int64{1, 2, 3},     // 24 bytes
	}

	fmt.Println(size.Of(a))
}

// Output: 44


备注：



  • 实际上，计算带有大嵌套的约10 GB的卷结构的大小需要10到20分钟。这是因为反射是一项相当昂贵的操作，需要将每个变量打包到一个空接口中并进行后续分析（请参见上面链接中的文章）。
  • 由于速度相对较低，因此应使用该程序包大致确定变量的大小，因为在实际系统中，在分析大型结构时，实际数据可能会有时间更改。或者，如果适用，在使用互斥量进行计算时提供对数据的独占访问。
  • 该程序不考虑数组，接口和关联数组的“容器”的大小（对于数组和切片，这是24字节；对于映射和接口，这是8字节）。因此，如果您有大量这样的小尺寸元素，那么损失将是巨大的。
