Mac OS、Linux、およびWindows WSL2で複数のJDKを管理する

またあったね。Java Developerコースの開始を期待して、興味深い記事の翻訳を用意しました。





私と同じように、複数のプロジェクト（レガシーとモダンの両方）と同時に作業する必要があり、異なるバージョンのJavaを使用するユーザー向けにある種のライブラリまたはユーティリティを開発している場合は、異なるバージョン間の切り替えの問題を理解できます。 JDK。







現在、開発中のJDKのインストール、管理、切り替えは、多くの開発者がJava 8を使用しているにもかかわらず、多くがJava 11に移行しているため、困難な作業になります。この問題は、さまざまな方法で解決できます。この記事では、それらのいくつかについて検討します。



手動の方法



JDKベンダーのサイトからディストリビューションをダウンロードしてすべて手動でインストールする（または$user/jdks



、そのようなフォルダに解凍する）ことができます。ただし、JAVA_HOME



必要なJDKが存在するように、毎回チェックして更新する必要があります。



これは、bashスクリプト、bash関数などを使用して行うことができます。その場合の問題は何ですか？あなたは単にスクリプト/関数を呼び出すことを忘れて、あなたがそれを見るときだけ何かがうまくいかなかったことに気づくことができますUnsupportedClassVersionError



。



すべてを手動で実行したい場合は、インターネットを調べて解決策を見つけることができます。そして、これは上級ユーザーにとっては良いオプションです。しかし、私はこれにいくつかのユーティリティを使用します。それでは、それらを見てみましょう。



最良の方法



私の考えでは、理想的な解決策は、かなり以前から存在していた2つのオープンソースツールを組み合わせることであり、おそらく少なくとも1つのツールに精通しているでしょう。しかし、あなたが知らないのは、それらを一緒に使用できることです。



  • SDKMAN！
  • jEnv


それらの共有を見てみましょう。



複数のJDKをインストールする





SDKMANを インストールした後！タイプ： 



$ sdk list java


目的のベンダー、バージョンを見つけ、次のように入力します。 



$ sdk install java <id>


4つのJDKをインストールしてみましょう。OracleOpenJDKの最後の2つのバージョンとAdoptOpenJDKの2つのLTSバージョンです。



インストール時には、デフォルトのJDKを選択しないでください。



  1. AdoptOpenJDKビルドOpenJDK 8u252 -LTS
  2. AdoptOpenJDKビルドOpenJDK 11.0.7 -LTS
  3. Oracle OpenJDK 14-最新GA
  4. Oracle OpenJDK 15-早期アクセスビルド


これらの4つのバージョンは、レガシーJava 8プロジェクト、最新のJava 11プロジェクト、およびJava 14とJava 15の革新を試すためのニーズを満たすはず



です。SDKMANを使用してローカルに4つのJDKがインストールされています！このユーティリティはJDKのインストールに非常に便利だと思います。



インストールされているすべてのJDKは、次のディレクトリにあります。 



$ cd /Users/bruno/.sdkman/candidates/java
$ ls
11.0.7.hs-adpt  14.0.1-open     15.ea.20-open   8.0.252.hs-adpt


異なるJDKを頻繁に切り替える必要がない場合は、これで十分です。SDKMANを使用できます！JDKを選択するには： 



$ sdk current java
Using java version 15.ea.19-open
$ sdk default java 15.ea.20-open
Default java version set to 15.ea.20-open


これで停止してSDKMANのみを使用できますが、遅かれ早かれ、手動によるアプローチと同じ方法でJDKを切り替えるのを忘れます。したがって、私はJDKの自動切り替えを好みます。



jEnvで複数のJDKを管理する







SDKMANという事実にもかかわらず！JDKをインストールすると便利ですが、あるプロジェクトから別のプロジェクトに切り替えるときに、異なるJDK間で自動的に切り替えると役に立ちません。ここではjEnvが役立ちます。



SDKMANを使用していくつかのJDKをインストールした後！または他の方法で、jEnvに追加する必要があります。 



$ cd ~/.sdkman/candidates/java
$ ls -1
11.0.7.hs-adpt 
14.0.1-open    
15.ea.20-open
8.0.252.hs-adpt
$ jenv add 15.ea.20-open
openjdk64-15-ea added
15-ea added
15-ea added
...
$ jenv versions
 system
 1.8
 1.8.0.252
 11.0
 11.0.7
 14.0
 14.0.1
* 15-ea (set by /Users/bruno/.jenv/version)
 openjdk64-1.8.0.252
 openjdk64-11.0.7
 openjdk64-14.0.1
 openjdk64-15-ea


jenv add



JDKの残りの3つのバージョンに対して コマンドを繰り返します。



jEnvは、コマンドラインを常に使用するユーザーにとって特に便利です。彼はあなたをさせてくれます：



  1. システムのJavaバージョンをグローバルにインストールします。
  2. 現在のディレクトリ/現在のプロジェクトのJavaバージョンをインストールします。
  3. 現在のシェルのJavaバージョンをインストールします。


jEnvはshimを使用して、JAVA_HOME環境変数を自動的に管理します。したがって、システムを構成した後、jEnvは上記の優先順位と現在のディレクトリに基づいて、JDKバージョンを最適なバージョンに切り替えます。涼しい！他にも便利な機能があります。詳細については、Webサイトを参照してください。



JDK間の自動切り替え



これで、Java 8、11、14、および15-EAがインストールされました。では、どうやってそれらを切り替えるのですか？バージョンをカスタマイズして、スイッチをテストしましょう。



最初に、システムでデフォルトのグローバルJDKを指定する必要があります。これには通常、早期アクセスを使用します。新しいプロジェクトで作業を開始するたびに、次のリリースを自動的に使用します。これは、OpenJDKプロジェクトに報告できる潜在的な問題を特定するのに役立ちます。あなたもやります... 



$ jenv global 15


これで、ユーザーが置かれているすべてのフォルダーについて、実行時にjava -version



OpenJDK 15 EAを受け取ります。



Java 8を使用する必要があるプロジェクトがある場合は、フォルダーに移動して実行します。 



$ jenv local 1.8




その結果、.java-version



次の内容の名前でファイルが作成されます。 



$ cat .java-version
1.8


このファイルは、このフォルダーにいるときに使用するJDKをjEnvに指示します。



最後に、一時的に別のバージョンのJDKに切り替える場合は、を使用しますjenv shell



。上記のすべてと自動切り替えの仕組みを説明できる
