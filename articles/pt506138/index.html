Por que uma lista em uma tupla se comporta estranhamente em Python?

Nas linguagens de programação, sempre me interessei por sua estrutura interna. Como um ou outro operador trabalha? Por que é melhor escrever assim e não o contrário? Tais questões nem sempre ajudam a resolver o problema "aqui e agora", mas a longo prazo elas formam a imagem geral da linguagem de programação. Hoje, quero compartilhar o resultado de um desses mergulhos e responder à pergunta sobre o que acontece ao modificar tuple'a em list' e.

Todos sabemos que o Python tem um tipo de dados list:

a = []
a.append(2)

listÉ apenas uma matriz. Permite adicionar, excluir e modificar itens. Ele também suporta muitos operadores interessantes diferentes. Por exemplo, o operador + = ao qual adicionar itens list. +=altera a lista atual, mas não cria uma nova. Isso é claramente visível aqui:

>>> a = [1,2]
>>> id(a)
4543025032
>>> a += [3,4]
>>> id(a)
4543025032

Python tem outro ótimo tipo de dados: tuple- uma coleção imutável. Não permite adicionar, excluir ou alterar elementos:

>>> a = (1,2)
>>> a[1] = 3
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

Ao usar o operador +=, um novo é criado tuple:

>>> a = (1,2)
>>> id(a)
4536192840
>>> a += (3,4)
>>> id(a)
4542883144

Atenção, pergunta: o que o código a seguir fará?

a = (1,2,[3,4])
a[2] += [4,5]

Opções:

  1. Os itens são adicionados à lista.
  2. A exceção de imutabilidade da tupla será lançada.
  3. Ambos.
  4. Nem um nem o outro.

Escreva sua resposta em um pedaço de papel e vamos fazer uma pequena verificação:

>>> a = (1,2,[3,4])
>>> a[2] += [4,5]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

Bem então! Então nós descobrimos! A resposta correta é 2. Embora, espere um momento:

>>> a
(1, 2, [3, 4, 4, 5])

De fato, a resposta correta é 3. Ou seja, os elementos foram adicionados e a exceção foi lançada - wat ?!


Vamos ver por que isso acontece. E um módulo maravilhoso nos ajudará com isso dis:

import dis

def foo():
    a = (1,2,[3,4])
    a[2] += [4,5]

dis.dis(foo)

O primeiro bloco é responsável por criar tuple'a e armazená-lo em uma variável a. Então a diversão começa:

       21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)

Carregamos o ponteiro na variável ae na constante 2 na pilha .

       27 DUP_TOP_TWO

Duplique-os e coloque-os na pilha na mesma ordem.

       28 BINARY_SUBSCR

Este operador pega o elemento superior da pilha (TOS) e o próximo (TOS1). E escreve um novo elemento no topo da pilha TOS = TOS1[TOS]. Portanto, removemos os dois principais valores da pilha e colocamos nela um link para o segundo elemento tuple'a (nossa matriz).

       29 LOAD_CONST      4 (4)
       32 LOAD_CONST      5 (5)
       35 BUILD_LIST      2

Construímos uma lista dos elementos 4 e 5 e a colocamos no topo da pilha:

       38 INPLACE_ADD

Aplique +=nos dois principais elementos da pilha (Importante! Essas são duas listas! Uma consiste em 4 e 5 e a outra é retirada tuple). Está tudo bem aqui, a instrução é executada sem erros. Como ela +=modifica a lista original, a lista em tuple'e já foi alterada (naquele exato momento).

       39 ROT_THREE
       40 STORE_SUBSCR

Aqui trocamos os três principais elementos da pilha (mora lá tuple, nele o índice da matriz e a nova matriz) e escrevemos a nova matriz tuplepor índice. É aqui que a exceção ocorre!

Bem, é isso! Na verdade, você pode alterar a lista, mas tudo cai sobre o operador =.

Vamos finalmente descobrir como reescrever esse código sem exceção. Como já entendemos, você só precisa remover a entrada tuple. Aqui estão algumas opções:

>>> a = (1,2,[3,4])
>>> b = a[2]
>>> b += [4,5]
>>> a
(1, 2, [3, 4, 4, 5])

>>> a = (1,2,[3,4])
>>> a[2].extend([4,5])
>>> a
(1, 2, [3, 4, 4, 5])
