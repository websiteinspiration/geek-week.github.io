無料のdbForge Searchユーティリティを使用して、MS SQL Serverデータベース内のデータとオブジェクトを検索します

データベース内のデータとオブジェクトを検索する一般的な必要性の説明


データベース内のストアドプロシージャ、テーブル、およびその他のオブジェクトと同様に、データの検索は、C＃開発者や.NET開発全般を含む、かなり関連性の高い問題です。

多くの場合、次のことを見つける必要がある状況が発生する可能性があります。

  1. データベースオブジェクト（テーブル、ビュー、ストアドプロシージャ、関数など）
  2. データ（値とテーブルが配置されている場所）
  3. データベースオブジェクト定義のコードスニペット

有料と無料の両方の多くの既製のソリューションがあります。

最初に、組み込みのDBMSツールを使用してデータベース内のデータとオブジェクトを検索する方法を説明し、次に、無料のdbForge検索ユーティリティを使用してこれを実行する方法を説明します。

組み込みのDBMSツールを使用して検索する


次のスクリプトを使用して、Employeeテーブルがデータベースにあるかどうかを確認できます。

名前でテーブルを検索
select [object_id], [schema_id],
	   schema_name([schema_id]) as [schema_name], 
	   [name], 
	   [type], 
	   [type_desc], 
	   [create_date], 
	   [modify_date]
from sys.all_objects
where [name]='Employee';


結果は次のようになります。 ここに表示されます：





  1. オブジェクトの識別子とオブジェクトが配置されているスキーム
  2. この回路の名前とこのオブジェクトの名前
  3. オブジェクトのタイプとこのタイプのオブジェクトの説明
  4. オブジェクトの作成日時と最終変更日時

「Project」行のすべての出現を検索するには、次のスクリプトを使用できます。

名前の部分文字列ですべてのオブジェクトを検索する
select [object_id], [schema_id],
	   schema_name([schema_id]) as [schema_name], 
	   [name], 
	   [type], 
	   [type_desc], 
	   [create_date], 
	   [modify_date]
from sys.all_objects
where [name] like '%Project%';


結果は次のようになります。結果 からわかるように、ここでサブストリング「Project」には2つのテーブルProjectとProjectSkillだけでなく、いくつかの主キーと外部キーも含まれています。 これらのキーが正確に誰に属しているかを理解するために、parent_object_idフィールドとその名前、およびそれが配置されているスキームを次のように追加します。







親オブジェクトを含む名前の部分文字列ですべてのオブジェクトを検索します
select ao.[object_id], ao.[schema_id],
	   schema_name(ao.[schema_id]) as [schema_name],
	   ao.parent_object_id,
	   p.[schema_id] as [parent_schema_id],
	   schema_name(p.[schema_id]) as [parent_schema_name],
	   p.[name] as [parent_name],
	   ao.[name], 
	   ao.[type], 
	   ao.[type_desc], 
	   ao.[create_date], 
	   ao.[modify_date]
from sys.all_objects as ao
left outer join sys.all_objects as p on ao.[parent_object_id]=p.[object_id]
where ao.[name] like '%Project%';


結果は、親オブジェクトに関する詳細情報を含むテーブルになります。つまり、主キーと外部キーが定義されています 。次のシステムオブジェクトはクエリで使用されます。





  • sys.all_objectsテーブル
  • スカラー関数schema_name

そこで、組み込みのDBMSツールを使用してデータベース内のオブジェクトを検索する方法を見つけました。
次に、文字列検索の例を使用して、データベース内のデータを検索する方法を示します。

すべてのデータベーステーブルの文字列値を見つけるには、次のソリューションを使用できます。このソリューションを簡略化し、たとえば、次のスクリプトを使用して「Ramiro」の値を見つける方法を示します。

すべてのデータベーステーブルで部分文字列によって文字列値を検索します
set nocount on
declare @name varchar(128), @substr nvarchar(4000), @column varchar(128)
set @substr = '%Ramiro%'

declare @sql nvarchar(max);

create table #rslt 
(table_name varchar(128), field_name varchar(128), [value] nvarchar(max))

declare s cursor for select table_name as table_name from information_schema.tables where table_type = 'BASE TABLE' order by table_name
open s
fetch next from s into @name
while @@fetch_status = 0
begin
 declare c cursor for 
	select quotename(column_name) as column_name from information_schema.columns 
	  where data_type in ('text', 'ntext', 'varchar', 'char', 'nvarchar', 'char', 'sysname', 'int', 'tinyint') and table_name  = @name
 set @name = quotename(@name)
 open c
 fetch next from c into @column
 while @@fetch_status = 0
 begin
   --print 'Processing table - ' + @name + ', column - ' + @column

   set @sql='insert into #rslt select ''' + @name + ''' as Table_name, ''' + @column + ''', cast(' + @column + 
	' as nvarchar(max)) from' + @name + ' where cast(' + @column + ' as nvarchar(max)) like ''' + @substr + '''';

	print @sql;

   exec(@sql);

   fetch next from c into @column;
 end
 close c
 deallocate c
 fetch next from s into @name
end
select table_name as [Table Name], field_name as [Field Name], count(*) as [Found Mathes] from #rslt
group by table_name, field_name
order by table_name, field_name

drop table #rslt
close s
deallocate s


実行結果は次のようになります。 ここではテーブル名が表示され、どの列にサブストリング「Ramiro」を含む値が格納されているかが示されます。見つかったテーブルと列のペアに対して見つかったこのサブストリングの入力数。 定義に特定のコードが含まれているオブジェクトを検索するには、次のシステムビューを使用できます。







  1. sys.sql_modules
  2. sys.all_sql_modules
  3. sys.syscomments

たとえば、最後のビューを使用すると、次のスクリプトを使用して、特定のコードが見つかった定義内のすべてのオブジェクトを見つけることができます。

データベースオブジェクト定義でコードスニペットを検索する
select obj.[object_id],
	   obj.[name],
	   obj.[type_desc],
	   sc.[text]
from sys.syscomments as sc
inner join sys.objects obj on sc.[id]=obj.[object_id]
where sc.[text] like '%code snippet%';


オブジェクトの識別子、名前、説明、完全な定義がここに表示されます。

無料のdbForge検索ユーティリティで検索


ただし、既製の優れたツールを使用して検索する方が便利です。そのようなツールの1つがdbForge Searchです。SSMS

ウィンドウでこのユーティリティを呼び出すには、ボタンをクリックします。

次の検索ボックスが表示 されます。上部パネルに注意してください（左から右へ）：





  1. 検索モードを切り替えることができます（DDL（オブジェクト）またはデータを探します）
  2. 私たちが探しているもの（部分文字列）を直接
  3. 大文字と小文字を区別するかどうか、完全に一致する単語を検索するか、出現箇所を検索するか：


  4. オブジェクトタイプごとに結果をグループ化-ボタン
  5. 検索するオブジェクトのタイプを選択します。

  6. 複数のデータベースを指定して、MS SQL Serverのインスタンスを検索および選択することもできます。

これはすべてオブジェクト検索モード、つまりDDLがオンになっている場合です 。データ検索モードでは、オブジェクトタイプの選択のみが変更されます。 つまり、データ自体が格納されているテーブルのみが選択可能になります。 これで、以前と同様に、「Project」サブストリングのすべてのオカレンスが見つかります。オブジェクトの名前： ご覧のとおり、DDLオブジェクトの検索モードが選択されています。探しているのは「プロジェクト」行で、残りはすべてデフォルトでした。 見つかったオブジェクトを強調表示すると、このオブジェクトまたはその親オブジェクト全体の定義コードが下に表示されます。 ボタンをクリックして、見つかったオブジェクトにナビゲーションを移動することもできます。 また、見つかったオブジェクトをタイプ別にグループ化することもできます。



























名前にサブプロジェクト「Project」を含むフィールドを含むテーブルでも表示されることに注意してください。ただし、検索モードを変更できることを思い出してください。完全なコンプライアンス/部分的/大文字と小文字を区別するかどうかを検索します。

これで、すべてのテーブルの「Ramiro」値が見つかりました 。選択したEmployeeテーブルの「Ramiro」サブストリングを含むすべての行の下に表示されていることに注意してください。 また、以前のようにボタンをクリックすることで、見つかったオブジェクトへのナビゲーションを移動することができます。 このように、我々は、データベースに必要なオブジェクトとデータを検索することができます。











結論


データ自体とデータベース内のオブジェクトの両方を検索する方法は、MS SQL Server DBMS自体の組み込みツールと無料のdbForge検索ユーティリティの両方を使用して調べられました。
