Zimbra Collaboration Suite Open-Source Editionで複数のアドレス帳を管理する

Zimbra OSEのグローバルアドレス帳は、各Zimbra OSEメールドメインで利用できる個別のgalsyncシステムアカウントを使用して実装されます。ドメイン内の各ユーザーに関する情報がこのアカウントの連絡先リストに追加され、ドメインユーザーはこのアドレスのリストにアクセスでき、それを使用してユーザーを検索し、メールアドレスを入力するときにオートコンプリートを使用できます。 

グローバルアドレス帳のこの実装により、新しいデータのインポートを簡略化できますが、デフォルトでは、1つのドメイン内でのみ機能します。大企業は多くの場合、部門や子会社ごとに別々のメールドメインを使用しており、多くの場合、複数またはすべての会社ドメインのメールアドレスとユーザーデータを含む共通のグローバルアドレス帳が必要になります。


異なるメールドメインのGalsyncアカウントの連絡先リストを組み合わせるには、同期を設定する必要があります。 msk.company.ruドメインとspb.company.ruドメインを例として使用して、モスクワとサンクトペテルブルクにある会社の支店に共通のアドレス帳を構成します。

デフォルトでは、Zimbra OSEドメインのgalsyncアカウントには、数字と文字のランダムなセットであるサフィックスがあります。これにより、今後の作業が困難になる可能性があるため、最初にgalsync.kf9song4@msk.company.ruアカウントを削除してから、次のコマンドを使用して新しいgalsync @ msk company.ruを作成します。

zmgsautil deleteAccount -a galsync.kf9song4@msk.company.ru
zmgsautil createAccount -a galsync@msk.company.ru -n GAL --domain msk.company.ru -t zimbra -s mail.company.ru -f GAL

Galsyncアカウントは通常のアカウントと同じ方法で削除してはならないという事実に注意を向けます。これにより、Galsyncアカウントなしで残されたドメインにアクセスしようとすると、Zimbra OSEは常にエラーを報告します。

その後、モスクワドメインのgalsyncアカウントに新しいSPBデータソースを追加します。連絡先は_SPB連絡先フォルダーに自動的に追加されます。これは、コマンドzmgsautil addDataSource -a galsync@msk.company.ru -n SPB --domain msk.company.ru -t ldap -f _SPB -p 1dを使用して行われます。 

情報のソースとして、ドメインspb.company.ruを指定し、フィルターを設定して連絡先を検索します。zmprov mds galsync @ msk。 = zimbraDistributionList）（！（zimbraHideInGAL = TRUE）））） 'zimbraGalSyncLdapSearchBase dc = spb、dc = company、dc = ru zimbraGalSyncLdapURL ldap：//192.168.0.15：389。

現在は、zmprov mds galsync@msk.company.ru SPB zimbraDataSourceEnabled TRUEによって追加された連絡先ソースをオンにし、コマンドzmgsautil forceSync -a galsync@msk.company.ru -n SPBを使用して最初の同期を強制するだけです。

グローバルアドレス帳間の連絡先の同期が機能することを確認するには、先ほど作成したzmmailbox -z -m galsync@msk.company.ru gafの_SPBフォルダーの内容を確認します  。すべてが順調である場合は、コマンドzmprov md msk.company.ru zimbraGALModeの両方を入力   し、サンクトペテルブルクドメインから連絡先にダイヤルするときにオートコンプリートがどのように機能するかを確認するだけです。

spb.company.ruドメインのユーザーがmsk.company.ruドメインのユーザーも表示できるようにする必要がある場合は、同じことを行う必要がありますが、ドメインを交換します。

  1. zmgsautil addDataSource -a galsync@spb.company.ru -n MSK --domain spb.company.ru -t ldap -f _MSK -p1d
  2. zmprov mds galsync@spb.company.ru SPB zimbraGalSyncLdapBindDn $zimbra_ldap_userdn zimbraGalSyncLdapBindPassword $zimbra_ldap_password zimbraGalSyncLdapFilter '(|(&(mail=*)(zimbraAccountStatus=active)(!(zimbraHideInGAL=TRUE)))(&(mail=*)(objectClass=zimbraDistributionList)(!(zimbraHideInGAL=TRUE))))' zimbraGalSyncLdapSearchBase dc=msk,dc=company,dc=ru zimbraGalSyncLdapURL ldap://192.168.0.15:389
  3. zmprov mds galsync@spb.company.ru MSK zimbraDataSourceEnabled TRUE
  4. zmgsautil forceSync -a galsync@spb.company.ru -n MSKzmmailbox -z -m galsync@spb.company.ru gaf
  5. zmprov md spb.company.ru zimbraGALMode both

Galsyncアカウントのデータソースの数は無制限です。

管理コンソールで、galsyncアカウントの新しい連絡先ソースを追加できます。ただし、これを行うには、とにかくコマンドラインで作業する必要があります。たとえば、カルーガのブランチからモスクワドメインの同期に連絡先を追加するには、コマンド  zmgsautil addDataSource -a galsync@msk.company.ru -n KLG --domain msk.company.ru -t ldap -f _KLGを使用します。 -p 1dは、KLGデータソースと_KLGと呼ばれるそれらのフォルダーを追加します。



その後、管理コンソールで[設定]-[ドメイン]-[GALの構成]に移動し、開いているグローバルアドレス帳構成ウィザードで、KLGソースを選択し、前の例で使用したものと同様に、必要なすべてのパラメーターを入力します。



Active Directoryを外部連絡先ソースとして接続することもできます。これは、通常のLDAPの場合と同じ方法で行われます。ADデータソースが作成されます-zmgsautil addDataSource -a galsync@msk.company.ru -n AD --domain msk.company.ru -t ldap -f _AD -p 1d、そして検索コンソールとオートコンプリートのデータが管理コンソールに表示されます。
