UnityのMVCとスクリプト可能なオブジェクト。パート2

Cem Ugur Karacamによる一連の記事の続き（パート1はここで読むことができます）。

今回は、インベントリオブジェクトを表示するいくつかのビューと、オブジェクトのパラメータを表示する情報パネルを作成することから始めます。ある種の原始的な目録。次に、インベントリウィンドウに焦点を当て、次のパートで統計ウィンドウを終了します。



私たちの仕事が何で構成されるか見てみましょう。するべき：

  • 主題と在庫に関する情報のための視覚的なコンポーネントを作成します。
  • 在庫のあるアイテムのプレハブを作成します。
  • ビューで在庫を埋める。
  • アイテムビューをクリックして、コンソールに情報を表示します。

これはたくさんあります。まず、画面の解像度を調整します。



Unity UIオブジェクトを使用します。空のキャンバスを作成します（階層ウィンドウで右クリック→UI→キャンバス）。



次に、2つのパネルを作成します-アイテムと在庫に関する情報。
また、背景用のオブジェクトを作成し、それをBGと呼び、それに垂直レイアウトコンポーネントを割り当てます。







これで、背景の子が垂直に配置されます。

BG内にパネルを作成しましょう。



RectTransformコンポーネントでは、垂直レイアウトコンポーネントの制御下で、パネルの高さ-300のみを設定し、幅は自動のままにします。これがBGオブジェクトの設定です





。パネルを複製し、高さを440に設定します。



次に、パネルに正しい名前を付けます。



次のようになります。



InfoPanelオブジェクトでの作業のストーリーは、次のパートに続きます。

在庫アイテムのビューを準備して、プレハブとして保存しましょう。パネル内にボタンを作成します。インベントリ内のアイテムを正しく表示するには、便利なグリッドレイアウトグループコンポーネントをInventoryPanelオブジェクトに追加します。



これで、インベントリパネルの各子オブジェクトが、グリッドレイアウトグループコンポーネントのグリッド設定に従って画面に配置されます。ボタンに戻りましょう。



グリッドをまだ構成していないため、ボタンは正しく機能しません。とりあえず、画像のオブジェクトのボタンを項目のアイコンの子として追加し、画像コンポーネントからレイキャストターゲットフラグを削除して、画像をクリックしてもボタンがクリックされないようにします。



ボタンを数回コピーします。このレッスンでは4つのコピーで十分です。外観は次のとおりです。次に進む



前に、アイテムを表すスクリプトを1つ作成する必要があります。彼はインベントリアイテム自体に関する情報を格納する責任があり、ボタンをクリックするイベントに応答します（ここでは、Debug.Logコマンドをコンソールに出力することに専念します）。

新しいスクリプトを作成し、ItemViewと呼びます。



ItemViewは、データのロード、視覚的な表示、クリック処理、およびデータをロードする前の初期状態を担当します。

using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UI;

public class ItemView : MonoBehaviour
{
    public Button button;

    private ItemData itemData;

初期状態を設定するメソッドを作成しましょう。

using System;
using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UI;

public class ItemView : MonoBehaviour
{
    public Button button;
    public Image itemIcon;

    private ItemData itemData;

    public void InitItem(ItemData item)
    {
        this.itemData = item;

        button.onClick.AddListener(ButtonClicked);
    }

    private void ButtonClicked()
    {
        Debug.Log(itemData.name);
    }
}

私たちのプロジェクトでは、いくつかのアイコンが必要になると思います。Asset Storeで無料のキットを見つけました。それをプロジェクトに追加しましょう。



これで、ItemDataクラス内のアイテムのアイコンを格納する変数を追加できます。

using UnityEngine;

[CreateAssetMenu]
public class ItemData : ScriptableObject
{
    public string itemName;
    public ItemType type;
    public float attack;
    public Sprite icon; 

    public float GetPrice()
    {
        return attack * 40;
    }
}

public enum ItemType
{
    Dagger,
    Axe,
    Hammer,
    Potion
}

これで、ItemViewでビューにアイコンを追加できます。

public void InitItem(ItemData item)
{
  this.itemData = item;
  itemIcon.sprite = itemData.icon;

  button.onClick.AddListener(ButtonClicked);
}

シーンのボタンにスクリプトを追加し、ボタンのあるオブジェクトの名前をItemに変更します。その後、ImageIconオブジェクトをItemIconフィールドに割り当てます。



残りの3つのボタンはグリッドを構成するために必要だったので削除し、プロジェクトウィンドウにドラッグしてItemオブジェクトをプレハブとして保存します。



プレゼンテーションはほぼ完了です。在庫を満たすには4つのアイテムが必要です。前のレッスンで作成したモデルにアイコンを作成して割り当てましょう。



できました。次に、ItemViewControllerスクリプトを作成します。

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class ItemViewController : MonoBehaviour
{
    public Inventory inventoryHolder;
    public Transform inventoryViewParent;
    public GameObject itemViewPrefab;

    private void Start() 
    {
        foreach (var item in inventoryHolder.inventory)
        {
            var itemGO = GameObject.Instantiate(itemViewPrefab, inventoryViewParent);
            itemGO.GetComponent<ItemView>().InitItem(item);
        }
    }
}

したがって、プロジェクトフォルダーに作成されたItemDataを格納する在庫へのリンク、このパネルの子としてビューを作成するためのInventoryPanelへのリンク、およびアイテム自体のビューを含むプレハブへのリンクがあります。

次に、Startメソッドで、アイテムのプレゼンテーションのインスタンスを作成し、InitItemメソッドを呼び出して、プレゼンテーションデータに各在庫アイテムのモデルのデータを入力します。

最後に、シーンに空のオブジェクトを作成し、それにコントローラースクリプトをアタッチして、シーン内の必要なオブジェクトへのリンクを割り当てます。



すべてのアイテムが在庫に割り当てられていることを確認してください。



私たちはかなりのことをしました。それがどのように機能するか見てみましょう。



アイテムは作成されましたが、ボタンからTextオブジェクトを削除するのを忘れていました。Unityでは、ボタンはデフォルトでテキスト付きで作成されます。プロジェクトに保存したプレハブを開き、Textオブジェクトを削除します。次に、プレハブを保存してシーンに戻ります。
