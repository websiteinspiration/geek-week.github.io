9つのWebコンポーネントライブラリ

1. 材料コンポーネントのウェブ




タイトルに「Material」という単語を使用すると、UIライブラリの評価で1位になることが保証されています。Google WebライブラリMaterial-componentsは、Material-UIライブラリのWebコンポーネントのバージョンです。これらのWebコンポーネントは、さまざまなコンテキストや環境に組み込むことができます。


この記事は、電子診察システムを開発しており、ソフトウェアサポートにも関与している EDISON Softwareのサポートにより翻訳されました。


2. ポリマー要素




GoogleのPolymerライブラリを使用すると、他のHTML要素をインポートして使用するのと同じくらい簡単に、標準のHTML要素のように機能する再利用可能なカプセル化されたWebコンポーネントを作成できます。Polymer要素は、100を超える再利用可能なPolymerコンポーネントをスタンドアロンのリポジトリとして含み、そのまま参照して使用できるGitHub組織です。例：

<!-- Import a component -->
<script src="https://unpkg.com/@polymer/paper-checkbox@next/paper-checkbox.js?module" type="module" ></script>
<!-- Use it like any other HTML element -->
<paper-checkbox>Web Components!</paper-checkbox>

3. Vaadin Webコンポーネント




これは、最新のブラウザーでモバイルおよびデスクトップWebアプリケーションのユーザーインターフェイスを作成するための30近くの開発中のオープンソースWebコンポーネントのコレクションを含むかなり新しいライブラリです。

4. ワイヤードエレメント




ワイヤードエレメントは、手書きのセット（7,000個の星）です。そして、これらの要素からワイヤーフレームを作成することができ、それらは2つのレンダリングがまったく同じにならないように描画されます-2つの個別の手描きのシェイプのように。ここで、そしてReactとVueでさえ、それらを試すことができます。

5. エリックス




Elixは、一般的なユーザーインターフェイステンプレート用のコミュニティ主導の再利用可能なカスタムWebコンポーネントのセットです。高品質の標準を確保するために、コンポーネントは、埋め込みHTML要素を品質指標として使用するゴールドスタンダードのWebコンポーネントチェックリストと比較されます。

6. 時間要素


<local-time datetime="2014-04-01T16:30:00-08:00">
  April 1, 2014 4:30pm
</local-time>
--
<local-time datetime="2014-04-01T16:30:00-08:00">
  1 Apr 2014 21:30
</local-time>

このかなり古いライブラリ（150万星）。これは主に、標準のHTML要素のカスタムサブタイプを提供するコンポーネントですtime。タイムスタンプをローカライズされた文字列またはユーザーのブラウザーで自動的に更新される相対テキストとしてフォーマットすることにより、どこでも使用できる独自の拡張機能を作成できます。

7. UI5-webcomponents






このライブラリはSAPによって作成されます。これは、軽量で再利用可能な独立したユーザーインターフェイス要素のコレクションです。UI5の上に構築されていないコンポーネントは、スタンドアロン要素のままです。フレームワークやアプリケーションの上で使用できます。コンポーネントの設計は、SAP Fiori設計ガイドラインに沿っており、Fiori 3設計が含まれていますライブプレイグラウンドとAPIをテストします。

8. パターンフライ




デモを実行：

git clone git@github.com:patternfly/patternfly-elements.git
cd patternfly-elements
npm install # this will take a while due to lerna bootstrap
npm run storybook

PatternFly Elementsは、約20の柔軟で軽量なWebコンポーネントとそれらを作成するためのツールのコレクションです。PatternFly要素は軽量で直感的であり（これは主にWebコンポーネントの標準です）、HTML要素が使用されている場所であればどこでも、React、Vue、Angular、vanilla JSで機能します。

9. Webコンポーネント組織




これはライブラリではありませんが、Polymerの要素や友人の上に構築されたGoogle Webコンポーネント検索ポータルです。ここにリストします。これは、Polymerチームから新しいWebコンポーネントを更新するための便利な方法であり、Webコンポーネントの使用を開始するときに読むための役立つリソースを提供するためです。
