WSLでのphp-fpm + nginxバンドルの構成

私は農家であり、あらゆる利点があるにもかかわらず、あらゆる方法で私のPCでのLinuxへの最終的な移行を回避しています。そのため、ここ数年、Docker for Windowsを使用してプロジェクトのローカルプラットフォームを作成しました。事は超安定であると言っているわけではないが、Laravelサイト上で一つのプロジェクトにまで重大な問題引き起こさなかった奇妙な動作し始めた：いくつかの要求（と常に異なる部分）がエラーで戻りました。スクリプトはキャッシュファイルを開くことができませんでした。

理由を探すのにかなりの時間を費やしたので、この場合、Dockerを放棄し、手動モードのWSLでプラットフォームをデプロイすることにしました。

WSLはほぼ Linuxであるため、-実際には、設定には独自の特性があります。
MySQL、nginx、php7.2がbashからインストールされた後、仮想サーバーがnginxで構成され、これらすべてが機能しなくなりました。nginxログには次のエントリがありました：

 connect() to unix:/var/run/php/php7.2-fpm.sock failed (2: No such file or directory) while connecting to upstream

WSLでは、私の知る限り、そのようなソケットはないため、以下を実行する必要がありました。

  1. /etc/php/7.2/fpm/pool.d/www.confファイルを開いて編集し、listenディレクティブの値を修正します

    ; listen = /run/php/php7.2-fpm.sock ; old value
listen = 127.0.0.1:9000 ; new value
  2. nginx仮想サーバーのセットアップで同じ値を設定します

    location ~ \.php$ {
...
fastcgi_pass   127.0.0.1:9000;
...
}

これは一種の動作するサーバーを与えました。処理されたoooooooooooooooooooooooooooooooからの各リクエストが遅いためです。すでに制限時間内です。ブレーキを取り除くには、それが判明したように、/ etc / nginx / nginx.confファイルに行を追加する必要があります

http {
...
fastcgi_buffering off;
...
}

その後、システムは離陸し、今のところ素晴らしい気分です。
