Appleでのサインインの紹介-Apple Authorization System

こんにちは、ハブル！

この夏は、WWDC 2019の会議で、Appleは独自の導入のAppleアカウントでサインイン認証システムを  、それが作られた必須のソーシャルメディアでのログインを使用してApp Storeですべてのアプリケーションのために。例外は、独自の認証を使用する教育、企業、政府、およびビジネスアプリケーションです。Appleでのサインインについては、Appleが高品質のドキュメントを作成しました。この記事では、CIANの例を使用して、サービスに実装する方法を説明します。



Apple開発者アカウントを構成する


統合作業は、開発者アカウントの設定から始まります。最初に、App IDの[Appleでサインイン]オプションを有効にする必要があります。これを行うには、Apple開発者アカウントの識別子のリストに移動し、必要なアプリIDを選択して、Appleでサインインオプションを有効にします。

次に、サービスIDを構成します。これは、Apple APIでサインインにアクセスするために必要なWebアプリケーションの一意の識別子です。 1つのアプリIDに対して最大5つのサービスIDを作成できます。これを行うには、識別子を作成するためのボタンをクリックし、[サービスID]を選択し、必要なフィールドに入力して、[Appleでサインイン]フィールドの[編集]をクリックします。フォームが開き、正しいプライマリアプリIDを選択し、ウェブドメインを指定して、ログイン成功後のリダイレクトのURLをリストします。入力できるリターンURLは10個のみであることに注意してください。



保存するには、[保存して続行して登録]をクリックします。はい、設定を変更する場合は、3つのボタンすべてをクリックする必要があります。そうしないと、変更は有効になりません。

次に、[サービスID]リストで作成した識別子を選択し、[Appleでサインイン]フィールドの[編集]をもう一度クリックします。開いたウィンドウで、Webアドレスのフィールドの横に2つの新しいボタンが表示されます。

AppleアプリでサインインボタンをiOSアプリに追加します。


CIANは、iOS、Android、Webの3つのプラットフォームで動作します。iOSにはネイティブSDKがあるため、認証は次のようになります



。AppleでのサインインをiOSアプリケーションに追加するには、ASAuthorizationAppleIDButtonボタンを追加して、クリックハンドラーをハングさせます。

let appleIDProvider = ASAuthorizationAppleIDProvider()
let request = appleIDProvider.createRequest()
request.requestedScopes = [.fullName, .email]

let authorizationController = ASAuthorizationController(authorizationRequests: [request])
authorizationController.delegate = self
authorizationController.presentationContextProvider = self
authorizationController.performRequests()

Apple for web and Androidへのサインインを実装します


突然、AndroidとWebの場合、AppleはSDKを提供しないため、どちらの場合もAppleの承認ページを開く必要があり、プロセスは異なり



ます。承認ページのURLは次のようになります。

https://appleid.apple.com/auth/authorize?\
state=abvgd&\
response_type=code&\
client_id=ServiceID&\
scope=email+name&\
response_mode=form_post&\ redirect_uri=https%3A%2F%2Fcian.ru%2Fauth%2Fsome-callback%2F%3Ftype%3Dappleid

そのパラメータを考慮してください：

  • client_id-上で登録されたサービスID。
  • redirect_uri-AppleIDによる認証が成功した後にユーザーがリダイレクトされるURI。Apple Developerを設定するときに、このURIを指定しました。
  • stateは、redirect_uriが呼び出されたときにAppleが返すユーザーのセッションIDであり、送信者を確認できます。ランダム文字列など、このパラメータを自分で生成するためのルールを作成できます。
  • スコープ-このパラメーターは、ユーザーに必要な情報を示します。たとえば、上記の例のように、名前、電子メール、またはその両方。
  • response_type-このパラメーターは、回答が必要な形式を示します。コードまたはid_tokenを指定できます。id_tokenを選択した場合は、それをresponse_modeパラメーターで指定する必要があります。ここで、query、fragment、およびform_postを指定できます。

appleIDによる2要素認証が成功した後、Appleは指定されたredirect_uriを呼び出し、状態とコードパラメータを渡します。

curl -X POST \
'https://www.cian.ru/some-callback/?type=appleid' \
-H 'Content-Type: application/x-www-form-urlencoded' \
--data ' \
state=abvgd&\
code=12345&\
user={"name":{"firstName":"Tanya","lastName":"Sviridova"},"email":"someemail@gmail.com"}'

codeパラメータでは、5分間有効な1回限りのユーザー認証コードが渡されます。状態パラメーターでは、承認フォームの作成時に送信されたセッション識別子、ユーザーパラメーターではユーザーデータ。

データ検索  


すべてのクライアントで、ユーザーデータを保存するには、Appleからaccess_tokenを取得する必要があります。これを行うには、最初にauthorization_codeをリクエストします。

curl -X POST https://appleid.apple.com/auth/token -d '\
client_id=some_client_id&\
code=12345&\
client_secret=jwt_part1.jwt_part2.jwt_part3&\
grant_type=authorization_code'

このリクエストでは： 

  • client_idは、Webアプリケーション用に作成されたServiceIDとiOSアプリケーション用のAppIDを示します。
  • コード-リダイレクト後またはiOSクライアントからの転送後に受け取った
  • grant_typeパラメータで、トークンを取得する目的を渡します：承認（authorization_code）またはトークンの更新（refresh_token）
  • client_secretパラメータ内-アプリケーションの登録中に受信した秘密鍵に基づくJSON Web Token。

PythonでJSON Webトークンを作成できます。

claims = {
	'iss': APPLEID_TEAM_ID,
	'aud': 'https://appleid.apple.com',
	'sub': client_id,
	'iat': current_timestamp,
	'exp': current_timestamp + expire_period,
}

headers = {'kid': 'APPLEID_KEY_ID', 'alg': 'ES256'}

client_secret = jwt.encode(payload=claims, key=secret_key, algorithm='ES256', headers=headers).decode('utf-8')

すべてがうまくいった場合、次のパラメーターが応答に含まれます。
{
  "access_token":"ufhzch",
  "token_type":"Bearer",
  "expires_in":3600,
  "refresh_token":"some_refresh_token",
  "id_token":"some_long_signed_jwt_token"
}

やったー、こちらがaccess_tokenです。それに加えて、必要に応じてaccess_tokenを更新するために使用できるrefresh_tokenが付属しています。 

ユーザー情報はid_tokenフィールドに格納されますが、デコードする必要があります。

public_key = jwt.algorithms.RSAAlgorithm.from_jwk(
    json.dumps(apple_public_key)
)
data = jwt.decode(
    id_token,
    public_key,
    algorithm="RS256",
    verify=True,
    audience=client_id,
)

Apple_public_keyは、リンクから取得できる公開鍵です。 

デコード後、次のようになります。

data = {

メールは、ユーザーがAppleにサインインしてサービスに最初にログインしたときに一度だけ送信されます。次回、ユーザーがアプリケーションをアンバインドした場合にのみ、Appleはこのデータを転送します。Appleからのこの承認は、APIを介してデータを取得できる他のサービスとは異なり、そのようなものを実装することを計画しているという情報は見つかりませんでした。

この回答では、毎回送信されるサブパラメータとメールが必要なので、システムに保存し、クライアントに承認の成功を通知します。利益。
