ペルコナライブで講演者になった経緯（およびアメリカ国境からの興味深い詳細）



Percona Live Open Source Database Conferenceは、DBMSワールドカレンダーの主要なイベントの1つです。いったんすべてがMySQLのフォークの1つの開発から始まりましたが、その後、祖先を大きく上回りました。多くの資料（および訪問者）はまだMySQLのトピックに密接に関連していますが、一般的な情報の背景ははるかに広くなっています。これらはMongoDB、PostgreSQL、およびその他のあまり人気のないDBMSです。今年、ペルコナは私たちのカレンダーで重要なイベントになりました。初めて、このアメリカの会議に参加しました。おそらくすでにご存じのとおり、私たちは現代の監視技術の状態に非常に懸念しています。インフラストラクチャのパラダイムが最大限の柔軟性、マイクロサービス、およびクラスターソリューションに移行するにつれて、関連するサポートツールとアプローチも変化するはずです。それについては、実際、私の報告でした。しかし、最初に、彼らが米国の会議にどうやって到達するか、そして着陸直後に彼らがどのような驚きを期待できるかについてお話したいと思います。

それでは、人々はどのようにして外国の会議に参加するのでしょうか？実際、このプロセスはそれほど複雑ではありません。プログラム委員会に連絡し、レポートのトピックを記述し、技術イベントでの講演経験があるという証拠を添付する必要があります。当然のことながら、会議の地理を考えると、語学力は重要なポイントです。英語を話す聴衆と話す経験を含む非常に望ましい。これらの問題はすべてプログラム委員会と話し合い、あなたの可能性を評価します。

もちろん、法的な問題は個別に解決する必要があります。おかげで、ロシアでビザ書類を発行する理由がいくら難しいのかご自分で理解できます。たとえば、モスクワでは、執筆時点でビジタービザを待っている期間は300日です。首都の居住者は、一般的に、いくつかの隣接する州での事務処理でこれらの困難を回避するために使用されています。しかし、私たちはイルクーツクに拠点を置いているので、私たちに最も近い隣接する州はモンゴルです...停止します。ウランバートル！結局、そこにはアメリカ大使館もあります。そして、率直に言って、あまり人気がないため、あまり忙しくない。飛行機でイルクーツクからウランバートルまでの所要時間は1時間です。タイムゾーンは変わりません-使い慣れたリズムで作業を続けることができます。大使館の入り口から文字通り30分でビザを取得できます。唯一の難点は、領事料金をカーン銀行の支店で現金でしか支払えないことです。したがって、すぐにビザを取得したい場合は、この問題の解決を手伝ってくれる人がいるといいでしょう。

そう。ビザを受け取り、飛行機の座席に乗った。州自体への進入は近づいていた。国境を越えることは常に非常に退屈な出来事でした。私が2010年に初めて到着したとき、ワシントンでのパスポート管理にかかる時間に夢中になりました。いいえ、もちろん、大切な窓への線は常に古典的でした。しかし、まともな時間の間（確かに数年の間）、情報を突破して写真付きのチラシを提供する特別な機械が追加され、すべてがより速くなりました。往復の航空券を使って最後に行ったすべての旅行で、すべての宿泊施設のデータが航空券に反映されました。そして今回は、日付が延期されたチケットがあり、それに関連付けられた往復チケットはありませんでした。そして出来上がり：白い紙に書かれた写真は、取り消し線で表示されました。

役員アプローチ


急に数年前の路線になり、1時間後、ようやくパスポートコントロールができるようになり、とてもリラックスできました。役員は私がなぜ来たのか尋ねた。私は返信しました-ビジネス（販売、ビザb1 / b2のタイプで許可されています）と休息（休暇）です。彼は私が飛んだ飛行機を指定し、私は飛行機のデータベースにはないことを説明しました。本当に眠りたいと思って、なぜそうなのかわからなかったと返事をしました。アメリカの当局者は、なぜ私が出発日を変更したのか、いつ飛行機に戻ったのか疑問に思いました。私は別の時間に飛ぶことに決めたので私は変わったと答えました、そして私が戻って飛ぶとき、私はおよそ答えることしかできません。それから警官は「大丈夫」と言って手を挙げ、私のパスポートを渡した別の人を呼びました。彼は私を追加のチェックに導いた。リマインダー私は飛行機を1時間で持っていると、彼は冷静に答えました。

ああすごい。私は部屋に入った。そこにはさらに40人が座っている。彼は座り、電話に乗っただけで、すぐに警備員が駆け上がって電源を切るように言い、壁を指差しました。それは、私が疲労と睡眠不足から気づかなかった「電話は使えない」という標識の周りのすべてのものであることがわかりました。オフにしたのですが、隣人には時間がありませんでした。時間がない人は、電話を持ち帰るだけです。時々約3時間が経過し、時々誰かが追加で呼ばれた。インタビュー、彼らは私をどこにも呼ばなかった-彼らは私を入れたスタンプが付いたパスポートを発行しただけだった。それが何だった？（c）確かに、欠落したフライトのチケットは、最終的には、受け取った証明書に従って実際に変更されました。



シティオースティンテキサス


そして最後に、私の足の下で、テキサスの土地。テキサスはロシアの人々によく知られている地名ですが、それでも同胞が最も訪問する場所ではありません。以前はカリフォルニアとニューヨークにいましたが、そんなに南に行く必要はありませんでした。そして、Percona Liveでない場合でも、いつ必要になるかは不明です。



オースティン市はテキサス州内の「カリフォルニアエンクレーブ」の少しです。どうやってそうなった？もちろん、政府の投資に加えて、渓谷の急速な成長の最初の基礎は、穏やかな気候と低コストで生活し、ビジネスを行うことでした。しかし、サンフランシスコと郡が文字通り法外なコストの象徴になっている今、新しいスタートアップは新しい場所を探しています。そしてテキサスは良い選択でした。まず、ゼロ所得税。第二に、個々の起業家の粗利に対するゼロ税。多数の大学-したがって、発達した熟練労働市場。アメリカの基準では、生活費はそれほど高くありません。これらすべては、全体として、新しい技術企業の発展に良い影響を与えます。そして-関連するイベントの観客を作成します。



Percona Live自体は、Hayat Regencyで開催されました。今日人気のあるスキームによると、会議は並行して実行されているいくつかのテーマ別スレッドで構成されていました。残念ながら、私たち自身のパフォーマンスのための準備の厳しいモードのため、プログラム全体を完全に評価することはできませんでした。しかし、私が見る機会があったそれらの報告は非常に面白いものでした。Peter Zaitsevの「変化するオープンソースデータベースの状況」と「データが多すぎますか？」イヴ・トルドーから。私たちはそこでアレクセイ・ミロビドフと会いました-彼はまたレポートを作成し、クリックハウスからチーム全体を連れてきました、そして私も私のスピーチで触れました。



報告させてください


そして、実際、主なことについて、私は何について話しましたか？レポートは、時系列データベースモニタリングシステムの新バージョンにどのように選択したかを説明したものです。どういうわけか私たちのパレスチナ人では、そのようなツールの必要が生じたとき、Clickhouseを採用するのがデフォルトであることが起こりました。どうして？「彼はより速いので。」間違いなく速いですか？いくら？たぶん、他のことを試すまで考えない他の長所と短所がありますか？問題の研究へのハードコアなアプローチを決定しました。しかし、特性を取り上げてリストするだけでは退屈であり、率直に言って、あまりよく覚えられていません。そして美しい人に教えられたようにp0b0rchyRoman Poborchiy、話を聞くのはもっと面白いです。したがって、監視エージェントからリアルタイムで毎秒受信する販売データで、テスト済みのすべてのDBMSをどのように駆動するかについて話しました。



イベントの印象を教えてください。


すべてが完璧に整理され、レポートは興味深いものでした。しかし、最も際立っていたのは、DBMSが技術的に現在動いているところです。たとえば、多くの人は長い間、セルフホストソリューションを使用していませんでした。私たちは今のところ、あまり慣れていないため、DBMSのインストール、構成、およびサポートに異常は見られません。そして、クラウドはずっと前に皆を奴隷にしており、条件付きRDSがデフォルトのオプションです。パフォーマンス、セキュリティ、バックアップについて心配する必要があります。すべてが事前に検討されている既成のサービスを利用できる場合は、このために個別の技術スペシャリストを確保してください。

これは非常に興味深く、おそらく、同様の形式でソリューションを提供する準備がまだ整っていない人にとっては驚くべき鐘です。

そして一般的に、これはDBMSだけでなく、サーバーインフラストラクチャ全体にも当てはまります。管理はLinuxコンソールからWebコンソールに移行します。ここでは、特定のクラウドプロバイダーがEKS、ECS、GKEおよびその他の大文字をどのように処理するかを理解するために、適切なサービスを選択して相互に交差させる必要があります。個人データに関する私たちのお気に入りの法律に関連して、ホスティング市場の内部プレーヤーはうまく発展しましたが、これまでのところ、私たちは世界的な技術運動の最先端に少し遅れており、そのようなパラダイムの変化をまだ経験していません。
