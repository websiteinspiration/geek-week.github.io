OpenGear-帯域外管理を備えたコンソールサーバーを使用してビジネスのダウンタイムを削減


シンプルなデバイスはコンソールサーバーのように見えますが、今日は単なる愚かなデバイスではなく、アクティブなネットワーク機器の展開を管理および自動化するためのエコシステムを作成するためのスマートプラットフォームです。結局のところ、彼女には最も重要なものがあります-無限の可能性を提供するクラシックコンソールです。

すべては、安全性とユーザビリティに重点を置いたアクセシビリティの向上を中心に展開されます。 99.999％の可用性を提供したいITシステムを構築している場合、OpenGearソリューションが役立ちます。それらには、一元化されたアクセスと自動化のためのNetOps / DevOpsモジュールを備えたクラシックコンソールがあります。 ソリューションには以下が含まれます。



  • セルラーへのフェイルオーバー-セルラーへのフェイルオーバー-この機能により、メインチャネルが利用できない場合に4G LTEまたは3Gチャネルでの通信をサポートできます。
  • Smart Out-of-Band-メインネットワークとは独立して機能し、問題を自動的に検出して修正します。これにより、コストが削減され、ダウンタイムが最小限になります。
  • ゼロタッチプロビジョニング-機器の導入プロセスを簡素化し、反復的なタスクを自動化して、プロセスへの人間の介入の割合を減らし、それにより、考えられるエラーの数を減らします。
  • 集中管理-どこにいても、アクティブなネットワーク機器に3クリックで簡単にアクセスできます。

興味を持っていただければ幸いです）

Opengearについて


Opengearは急成長している企業であり、そのセクターで最も進んでいます。同社はもともとオーストラリアから発展しましたが、すぐにグローバル企業になり、現在はアジアからアメリカまでのすべての主要な地域に存在しています。開発センターはオーストラリアとシリコンバレーにあります。

Opengearは、教育から小売、銀行まで、さまざまなセクターに適用できるユニバーサルソリューションです。

同社は2004年に設立されました。現在、定期的に更新される多数の新製品があります。

Opengear-コンソールサーバーとして


Opengearは、コンソールサーバーソリューションのクラスに属しています。すべてのアクティブなネットワーク機器は、コンソールポート経由でOpengearに接続します。それは古典的なRS-232と現代のUSBの両方をサポートしています。イーサネット管理ポートを接続することもできます。

サポートされるハードウェア：任意。


ラインナップとセレクション


機能面では、すべての機器は同じです。機器のフォームファクタのみが異なります。Opengearモデルには、コンパクトなインフラストラクチャマネージャーと、1Uに最大96個のコンソールポートを備えた非常に印象的なコンソールサーバーが分散しています。ポートのセットはさまざまで、さまざまな組み合わせがあります。



Opengear-スマートなアウトオブバンドソリューションとして


アウトオブバンド（OOB）管理により、どのような状況であってもネットワークをフォールトトレラントにすることができます。あなたは常に「プランB」を持っています。このテクノロジーを使用すると、リモートサイトにアクセスして故障を診断できるため、ネットワークのダウンタイムを削減できます-MTBFを増やし、MTTRを減らします。

外部通信チャネルのオプション：

  • 2つの通信チャネル：銅線/光ファイバーのプライマリおよびバックアップ。
  • 4G LTE / 3Gモデム（小型デバイスでは2つのSIMカード）;
  • PSTN回線用のV.92アナログモデム。
  • Wi-Fi。

これらすべての通信チャネルは、制御システムへの接続を構築するために使用できます。OpenGearデバイスがどの種類のNATにあるかは関係ありません。データセンターに到達します。

組み込みのTFTP、DHCPサーバー、および大容量のメモリにより、ファームウェアと構成のバックアップをデバイスに直接保存できます。したがって、接続が不十分な場合（たとえば、衛星モデム経由）でも、いつでも接続を復元できます。

次に、Opengearの作業スキーマを見てみましょう。ネットワークとリモートサイトがあり、メインの通信チャネルを介してこのサイトにアクセスするとします。プライマリ通信チャネルに障害が発生した場合、バックアップチャネルにアクセスできます。これにより、リモートサイトへのアクセスを失うことはありません。


Smart OOBの主な利点：

  1. ネットワークが利用できない場合でも、必要なときにネットワーク機器にリモートアクセスできます。
  2. 障害時の状況認識を提供します。
  3. ネットワークのダウンタイムとITインフラストラクチャを最小限に抑える。
  4. 「セルラーへのフェイルオーバー」（F2C）によるネットワークおよびIT障害からの迅速な回復。
  5. デバイスと物理環境を監視することにより、障害につながる前に問題を事前に検出します。
  6. 集中管理プラットフォーム。

Opengear-監視および制御センターとして


Opengearデバイスは、スマートソケットコンセント（PDU）の管理、UPSの状態の監視、さらには環境の監視を行うことができます。たとえば、サーバールームへの侵入を検出し、温度や湿度の上昇に対応します。

外部センサー：

  • 温度と湿度（小さなデバイスに組み込まれています）は、リモートでラック内の特定のポイントの温度を測定できます。
  • 浸透;
  • 振動
  • 洪水;
  • リレーに接続できるサードパーティのセンサー。



Opengearは、シリアルコンソール、イーサネット、またはUSBを介して接続できるPDUおよびUPSの100を超えるメーカーとモデルをサポートしています。APC、Eaton、Server Techなどでサポートされており、完全なリストが公開されています。


機能：

  • UPSバッテリーのステータスの監視とPDUコンセントのロード。
  • 帯域外のコンソールによるホットキー電源管理。
  • 自動電源制御。

組み込みのオープンソースツール：

  • ネットワークUPSツール（NUT）
  • パワーマン
  • IPMIツールキット

OpenGearは、SolarWinds、ZabbixなどのSNMPを介して任意の監視システムに統合できます。Nagiosエージェントはデバイスに統合されており、上位システムに統合できます。リモートサイトのサーバーを監視し、中央システムに情報を送信します。これにより、エージェント用に別のマシンを用意する必要がなくなります。


イベントレスポンスの自動化


おそらく、私にとって最も興味深い機能は自動化です。OpenGearでは、入力信号をチェック（チェック）して、これらの信号に応答（応答）し、この信号の消失に対する反応（解決）を行うことができます。

例えば：

  • コンソール出力を監視でき、カーネルパニックが表示されたら、電源装置をリセットします。
  • 温度を監視し、温度が上昇すると、自動的にサーバーをオフにするコマンドを送信し、次にデータウェアハウスを送信します。温度が回復したら、すべての電源を入れます。したがって、データと機器を保存します。

プラットフォームは柔軟性があり、組み込みアクションをブロックしません。何かをチェックして目的のアクションを実行する独自のスクリプト/プラグインを作成できます。


灯台-3クリックで任意のデバイスへの集中アクセス


灯台は、Opengear機器の中央ハブおよび制御ポータルです。コンソールサーバー自体は、安全なLHVPN（X.509証明書を備えたOpenVPN）を使用して「コールホーム」を作成します。

Lighthouseは、すべてのSmart OOBコンソールサーバーからの情報を集約します。

  • すべてのコンソールサーバーの現在のリスト。
  • コンソールゲートウェイ：検索を一覧表示し、HTML5 WebターミナルまたはSSHを介してコンソールポートに接続します。

Lighthouseは、最大10ノードの地理的冗長性を含むクラスタリングをサポートしています。単一の管理下で100,000を超えるコンソールポートをサポートします。Lighthouseは基本的に仮想マシンであり、目的の仮想化システムで実行されます。Opengear機器だけでなく、サードパーティのソリューション（Ciscoや通常のSSH / Telnetなど）も接続できます。Lighthouseは強力なRESTful APIをサポートしています。


トップユーザーエクスペリエンス


Opengear機器は、SSHとHTML5の両方を介して、ポートごとに50を超える同時セッションをサポートします。これにより、複雑なシステムを同時に復元し、同僚とベンダーの代表をつなぐことができます。HTML5はJavaを使用しないため、ソフトウェアの適切なバージョンを探して、セキュリティの更新に追いつく必要はありません。テキストのコピーと貼り付けは、Webインターフェースで機能します。これにより、作業が大幅にスピードアップします。

保証と信頼性


記事では、このデバイスを使用すると、ネットワークが利用できない場合でもリモートサイトにアクセスできるとしばしば言っていましたが、非常に論理的な質問をすることができます。この装置の製造では、従来のネットワークよりも桁違いに高い信頼性が組み込まれています。これは、コンポーネントパーツをテストし、その要件を増やすことで達成されます。製造元は、少なくとも4年間、装置の保証を提供します。 Opengear機器の信頼性についてはいつまでも語ることができますが、ここに1つ事実があります。 Opengearの機器も展示されていた展示会の1つで、男性がスタンドに近づき、展示されている製品に興味を持った。会話の中で、彼は会社で、12年以上継続的に稼働している設置済み機器Opengearを使用しています。

高いセキュリティ要件


Opengearにはこのような低レベルのアクセス権があるため、セキュリティ要件は可能な限り高くなります。

主なポイントをリストします：

  • 定期的なファームウェアリリース-3か月ごと。使用されているコンポーネントに脆弱性のクロージャがある中間ファームウェア。
  • 組み込みファイアウォール、デフォルトの禁止ポリシー。
  • IPSec、OpenVPN、PPTP、SSH、HTTPS;
  • ファイアウォールの背後での作業、パブリック/プライベートAPN、フェイルオーバー。
  • 各ポートの各ユーザーのアクセス権の厳密な分離（ロールモデル）。
  • LDAPS、TACACS +、RADIUS、2要素認証。
  • PCI DSS 3.0+への準拠。
  • SIEMシステムとの統合。
  • コンソールポートでのすべての出力と入力の監査ログ。
  • イベントアラート、コンソールケーブルを抜くまで。
  • SSHv2のサポートとSSHv1を無効にする機能。
  • ソースコードへのアクセスがあります。
  • 独自のファームウェアイメージ（ファームウェア）およびLinuxカーネルモジュールを作成する機能。

最後の2つの点について詳しく説明します。

ソースコードへのアクセスと独自のファームウェアの作成


おそらく、独自のタスクがあり、そのための特別なプログラムを作成する必要があるか、会社で使用されているソリューションのソースコードを詳しく調べたいと考えていました。名前に「オープン」という言葉が無駄ではないので、オープンギアはこの機会を与えます。

カスタム開発キット（CDK）の手順はここからダウンロードできます。ソースコード自体はFTP経由でダウンロードできます。あなたはそれの隣にあなた自身のプログラムを置くことができ、それはあなた自身のファームウェアに含まれます。

NetOps / DevOps


絶え間ない変化は、現代の企業にとっての課題です。仮想化とデジタル化は、企業におけるサービスのセットアップと提供の従来の方法を変え続けています。IT環境は常に変化しており、速度は数桁増加しています。

Opengearは、あらゆるワークフローとあらゆるアプリケーションに統合されます。NetOpsオートメーションプラットフォームは、Docker、Ansible、Gitなどの実績のあるコンポーネントを中心に設計されています。さまざまなランタイムがサポートされています：Python、Ruby、Perl、bash、x86バイナリ。これにより、リモートサイトを最初から展開できます-必要なものがすべて手元にあります。

NetOpsオートメーションプラットフォームの基礎は次のとおりです。

  • OM2000デバイス、サイトで直接動作します。
  • 灯台、一元化。

展開順序：

  1. DHCPオプションを使用したOGデバイスの自動チューニング（ゼロタッチプロビジョニング）。
  2. 灯台での自動チェックイン。
  3. 特定のポートに接続されているアクティブなネットワーク機器の識別。
  4. GitのAnsible構成を使用したコレクション。
  5. アクティブなネットワーク機器に設定を適用します。


ナレッジベースとドキュメント


ソリューションのドキュメントは開いており、サイトで入手できます。LighhouseのイメージとファームウェアはFTPでダウンロードできます。ナレッジベースは公開されており、検索エンジンを通じて問題を簡単に特定できます。

  • 信頼できる常時アクティブなバックアップ接続を提供します。
