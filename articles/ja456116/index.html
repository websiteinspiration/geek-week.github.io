yEdグラフエディターで美しいデータベーススキーマを構築する方法

さまざまな図やスキームを構築するように設計された、現在人気のあるyEdアプリケーションを使用して、完全な（すべての表記で）データベーススキーマを作成する方法を紹介します。

そのため、まず、ERD（エンティティ関係図）を構築するための要素を備えた標準のコンテナーがあります。それらの名称を図に示します。



少し余談ですが、図にテキストだけを挿入する必要がある場合、つまりブロックなしで、jaiprakashからのコメントからの次の簡単な手順は、この問題をすばやく解決します。 「リンクを選択->ラベルを追加->テキストを入力->ノードに変換。」

したがって、次のスキームを構築するという条件付き問題を提起します。



基本的な要素のフレームワークを作成します。



主キーと外部キーの表記を追加するには、少し工夫する必要があります。目的の要素を選択し、それに新しいラベルを追加します。これをKEYと呼びます。なぜなら本体の下にありますので、クランプしたCtrで選択してください。





内部を選択してその配置（配置）を設定します：左上（内側、左上）、必要に応じて色、距離（エッジからの距離）をゼロに、インセット（ピクセル単位での挿入）を20 2 0 0に設定して、属性フィールドの反対側にします。距離が短い場合、Font Family（フォント）はSegoe UI Symbolを設定します。必要な文字を正確にサポートします。



コピー用のシンボルは、たとえばWindowsシンボルテーブルにあります。
