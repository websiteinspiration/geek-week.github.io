PowerShellの現在のバージョンは、$ PSVersionTable変数で確認できます。更新後は、次のようになります。Get-ChildItem



コマンドレットなどを使用する場合は、通常のパスの代わりにLiteralPathを使用します。 パスの形式は少し異なります。



Get-ChildItem -LiteralPath "\\?\C:\Folder"
Get-ChildItem -LiteralPath "\\?\UNC\ServerName\Share"
Get-ChildItem -LiteralPath "\\?\UNC\192.168.0.10\Share"

使い慣れた形式からLiteralPath形式にパスを変換するのに便利なように、次の関数を使用できます。

Function ConvertTo-LiteralPath {
Param([parameter(Mandatory=$true, Position=0)][String]$Path)
    If ($Path.Substring(0,2) -eq "\\") {Return ("\\?\UNC" + $Path.Remove(0,1))}
    Else {Return "\\?\$Path"}
}

LiteralPath パラメータを指定するときは、ワイルドカード（*、?など）を使用できないことに注意してください。LiteralPath

パラメーターに加えて、PowerShellの更新バージョンでは、Get-ChildItemコマンドレットがDepthパラメーターを受け取りました。これを使用して、再帰検索の入れ子の深さを設定できます。これを数回使用し、満足しました。 今、あなたはあなたのPSスクリプトが迷子になり、遠くのファイルを作成しないことを恐れることはできません。たとえば、このアプローチは、DFSRフォルダー内のファイルに「一時的な」属性をドロップするためのスクリプトを作成するときに非常に役立ちました。しかし、これは別の話です。別の記事で伝えようと思います。
