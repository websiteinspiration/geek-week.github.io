VNAや日中機器の質問へ

「それで、それは可能でしたか？」



極端な記事で、Jack Hansleyは、最近NanoVNAと呼ばれるデバイス、または50 kHz〜900 MHzの範囲で50ドルのコストの小型ポータブルベクトルネットワークアナライザー（VAC）と呼ばれるデバイスについて言及しました。

「50ドル、カール！」

ジャックが書いたように、そのようなデバイスをそのような価格で購入しない理由はなく、私は彼に完全に同意します。常に無線機器に取り組んでいるわけではありませんが、この機器をアリで注文し、届くまで待ってからパラメータを確認しました。ニュアンスはありますが、それを獲得できた幸運な人は通常満足しています。その情報を信じれば、かなり昔（3年前）に特定の日本人によって開発され、ソースコードがGitにレイアウトされ、今年、中国人が突然複製して販売し始めました（もちろん、日本語ではなくデバイスです）。

トピックの人々は何が起こっているのかをすぐに理解しました。残りの部分については、VACの価格は$ 3,000から始まり、R＆S以外では誰も知らないことを説明します。ダイナミックレンジが120dB、周波数が最大1500MHzで比較的安価なVACを176,000ルーブルで購入できます。これは特徴的な数値です。したがって、多くの人は、デバイスが到着するのを待つのをためらうことを理解し、特にすべてのソースコードが開いているので、デバイスがどのように配置され、なぜ機能するのかをすぐに理解します。

私は開発者のgitに行き、回路を見て、少し驚かれ始めます。



マイクロコントローラー、タッチスクリーン、ボタンは省略しています（面白いロッキングチェアはあります）。これはまったく面白くなく、普通のことです。メーターを調べます。

まず、プログラム可能な発電機（VACを想像することは難しい）を観察し、よく知られている（そして安価な）m / s Si5351Aを観察します。しかし、結局のところ、彼女の出力周波数範囲は明らかに900 MHzではありません。私は日付を取りますが、出力周波数は最大200 MHzです。さらに、出力信号は基本的に直接石炭（これはこの接続で冗談です）であり、サインはありません。

では、先に進んで、m / cx SA612AD（明示的なローカルオシレーター）に3つのバランスミキサーを見てみましょう。同じことですが、1つの入力で許容される周波数は最大500 MHz、2番目の入力では最大200 MHzであるため、900 MHzの範囲は明らかに拡大されていません。

そして、何らかの理由で、ミキサーの出力はm / c TLV320AIC3204（オーディオコーデック）の入力に送られますが、MKには速度と解像度の両方で劣らないADCが含まれています。一般に、回路ではなく、継続的な誤解、およびITが主張された周波数範囲をどのように提供できるかは謎のままです。

読者に質問を考え、答えてもらいます。

原則として、すでに十分なデータがありますが、私は（正直に言って）さらに2つのヒントを示します。

  1. 機器のダイナミックレンジが不均一-70 dB（50 kHz-300 MHz）、60 dB（300 m-600 MHz）、50 dB（600 m-900 MHz）、
  2. フォーラムでは、ラジオ猫が範囲を1200に拡大できるかどうかを誰かに尋ねました。彼らは（スペクトログラムを添付した状態で）第7高調波はすでに弱すぎると合理的に回答しました。

まあ、これが正確に周波数の混合を調査中のデバイスに適用する方法である場合でも、特性を得るために、いくつかの周波数を他の周波数から分離し、それらを別々に測定する必要があります。彼は）それは実現可能ですが、基本的なタスクではありません。ただし、回路にはフィルターがありません。整合回路にあるいくつかのRC回路は、このような機能を実行できません。それにもかかわらず、デバイスはどのように機能しますか？

最初は独創的であると思われるアイデアを思いつきました。ミキサーの周波数に小さな離調を導入します。次に、ミキサーの最初の入力でf + 3 * f + 5 * f + ...、2番目の入力でf1（f-Δf）+ 3 * f1 + 5 * f1 + ...、出力で周波数の混合が得られます
（f + 3 * f + 5 * f + ...）*（f1 + 3 * f1 + 5 * f1 + ...）=（f * f1）+（3 * f * f1）+（f * 3 * f1 ）+（3 * f * 3 * f1）+ ...、その後、各製品は合計（f±f1）+（3 * f±f1）+（f±3 * f1）+（3 * f±3 * f1 ）... =（Δf+（2 *f-Δf））+（（2 * f +Δf）+（4 * f-Δf））+（（4 * f +Δf）+（2 * f-Δf））+（ 3 *Δf+ 6 * f-3 *Δf）+ ....次に、透過率が0からΔfまで、カットオフが3 *Δfまでのバンドパスフィルターを配置すると、最初の高調波に依存する成分のみが残り、他のすべては抑制され、安全に測定できます。調整可能なフィルターはコーデックに含まれています、唯一の問題はその技術的な能力です、それらは十分でなければなりません。

アイデアは本当に良いです、それは正弦波の代わりに蛇行を使用することを可能にします、しかしそれは大きな欠陥があります-より高い周波数（3番目と5番目の高調波）をどうするかです。結局、3番目の高調波のビートがフィルターバンドに入るように離調をピックアップすると、1番目の高調波のビートがさらにそこに到達します。私は予備的に3つの可能な解決策を見ました：

  1. Δfが帯域に入るように調整可能な下限を持つフィルターを作成しますが、Δf/ 2およびΔf* 3/2はそうしません。タスクは単純ではありませんが、基本的に十分な設備で解決されます。
  2. 信号の3次高調波を測定するために、パイロットの5次高調波が信号と戦うように離調します。その後、フィルターは単純なままですが、信号の5次高調波をどうするかは明確ではありませんが、パイロットの3次高調波は明らかに到達しません。
  3. 順次測定を行います-最初にΔfを使用して最初の高調波のビートを測定し、Δfを3（2）倍に減らし、5番目と同様に1番目と3番目の和を測定します。他から1つを引いて、必要なすべてのものを取得します。ここで、DSPの分野での私の知識は十分ではなく、そのようなオプションの実際的な実現可能性を評価できません。

このデバイスにどのオプションが実装されているかは、コーデックの機能の慎重な調査とプログラムのテキストの分析から判断できますが、原則としてこれはそれほど興味深いものではありません。可能な解決策が概説されています。個人的には、3番目に行きます。もしそれが根本的に実行可能であれば、後でわかります。
