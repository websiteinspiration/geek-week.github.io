Reactフックルーター



ReactフックをReactルーターに置き換える方法の 無料翻訳。
Peter Ekene Ezeによる投稿。

React Hooksの登場以来、多くの変化がありました。フックによって提供される機会により、ルーティングを含むReactの特定の概念へのアプローチを再考することができました。

この投稿は、React Routerによって軽視されたり軽視されたりすることを意図したものではありません。他の可能性を探り、フックを使用してReactアプリケーションでルーティングを改善する方法を見ていきます。

そのために、React Routerとhooksrouterを実例と比較します。はじめに、React Routerを詳しく見てみましょう。

Reactルーター


Reactルーターは、R​​eactアプリケーションでルートを管理するための一般的な宣言的な方法です。これにより、アプリケーションのすべてのページと画面のルートを手動で構成する際の作業量が削減されます。React Routerは、ルーティングの作成に役立つ3つの主要コンポーネント（Route、Link、およびBrowserRouter）を提供します。

React Routerでのルーティング


3ページのReactアプリケーションを作成しているとしましょう。通常、ルーティングにはReactルーターを使用し、次のように記述します。

import Users from "./components/Users";
import Contact from "./components/Contact";
import About from "./components/About";
function App() {
  return (
    <div>
      <Router>
        <div>
          <Route path="/about" component={About} />
          <Route path="/users" component={Users} />
          <Route path="/contact" component={Contact} />
        </div>
      </Router>
    </div>
  );
}

React Routerパッケージからインポートされた<Route />コンポーネントは、ユーザーがたどるパスと、指定されたパスに沿って表示するコンポーネントの2つのパラメーターを受け入れます。

代替ルーティングとしてのフック


hookrouterモジュールは、事前定義されたルートオブジェクトをチェックして結果を返すuseRoutes（）フックをエクスポートします。ルートオブジェクトでは、ルートをキーとして定義し、それらの値をルートが一致したときに呼び出される関数として定義します。フックのコード例：

import Users from "./components/Users";
import Contact from "./components/Contact";
import About from "./components/About";

const Routes = {
  "/": () => ,
  "/about": () => ,
  "/contact": () => 
};
export default Routes;

ルートを書くこの方法はより魅力的です。どうして？多くの仕事をする必要がなかったからです。React Routerでは、アプリケーション内のすべての個別ルートの<Route />コンポーネントをレンダリングする必要がありました。彼に渡したすべてのパラメータは言うまでもありません。フックを利用すると、ルートをuseRoutes（）フックに渡すだけでルートを使用できます。

import {useRoutes} from 'hookrouter';
import Routes from './router'

function App() {
  const routeResult = useRoutes(Routes)
  return routeResult
}

これにより、React Routerを使用した場合とまったく同じ結果が得られますが、実装がより簡潔で簡単になります。

Reactルーターのナビゲーション


React Routerには、アプリケーションナビゲーションのカスタマイズとインタラクティブルーティングの管理に役立つ<Link />コンポーネントが含まれています。3ページのアプリケーションがあるので、画面に表示して、押されたら次のページに移りましょう。

import { Route, Link, BrowserRouter as Router } from "react-router-dom";
import Users from "./components/Users";
import Contact from "./components/Contact";
import About from "./components/About";

function App() {
  return (
    <div className="App">
      <Router>
        <div>
          <ul>
            <li>
              <Link to="/about">About</Link>
            </li>
            <li>
              <Link to="/users">Users</Link>
            </li>
            <li>
              <Link to="/contact">Contact</Link>
            </li>
          </ul>
          <Route path="/about" component={About} />
          <Route path="/users" component={Users} />
          <Route path="/contact" component={Contact} />
        </div>
      </Router>
    </div>
  );
}

アプリケーションのあるページから別のページに移動するために必要なナビゲーションを作成しました。これがどのように機能するかの良い例です。



ナビゲーションにはフックを使用します


hookrouterモジュールは、HTML <a/>タグの<A/>ラッパーを提供します。これは、reactコンポーネントとして使用でき、ネイティブの<a/>タグと100％互換性があります。唯一の違いは、新しいページを実際にロードする代わりに、ナビゲーションを履歴スタックに移動することです。

const routes = {
  "/user": () => <Users />,
  "/about": () => <About />,
  "/contact": () => <Contact />
};

function App() {
  const routeResult = useRoutes(routes);
  return (
    <div className="App">
      <A href="/user">Users Page</A>
      <A href="/about">About Page</A>
      <A href="/contact">Contacts Page</A>
      {routeResult}
    </div>
  );
}



ソフトウェアナビゲーション


hookrouterモジュールは、navigate（）フックへのアクセスを提供します。このフックにURLを渡すことができ、ユーザーをこのURLにリダイレクトします。navigate（）関数の各呼び出しは、前方ナビゲーションを表します。その結果、ユーザーはブラウザーの戻るボタンをクリックして、前のURLに戻ることができます。これはデフォルトで発生します。

navigate('/user/');

ただし、別の動作が必要な場合は、ブラウザの履歴を置き換えることができます。navigate（）フックは、navigate（url、[replace]、[queryParams]）という3つのパラメーターを受け取ります。2番目のパラメーターは、置換動作を変更するために使用されます。現在の履歴エントリを削除し、新しいエントリに置き換えます。この効果を実現するには、引数をtrueに設定するだけです。

navigate('/user', true);

Reactルータースイッチ


React Routerは、特定のルートが一致しない場合に<Switch />コンポーネントを使用してデフォルトページを表示します。通常、404ページが表示され、選択したルートがアプリケーションで定義されていないことをユーザーに通知します。これを行うには、すべてのルートを<Switch />コンポーネント内にラップし、ファイルへのパスを定義せずに404ページを描画します。

import { Route, Link, BrowserRouter as Router, Switch } from "react-router-dom";
import Users from "./components/Users";
import Contact from "./components/Contact";
import Home from "./components/About";
import NoPageFound from "./components/NoPageFound.js";

function App() {
  return (
    <div className="App">
      <Router>
        <div>
          <ul>
            <li>
              <Link to="/">Home</Link>
            </li>
            <li>
              <Link to="/users">Users</Link>
            </li>
            <li>
              <Link to="/contact">Contact</Link>
            </li>
          </ul>
          <Switch>
            <Route exact path="/" component={Home} />
            <Route path="/users" component={Users} />
            <Route path="/contact" component={Contact} />
            <Route component={NoPageFound} />
          </Switch>
        </div>
      </Router>
    </div>
  );
}

したがって、未定義のルートに切り替えると、React RouterはNoPageFoundコンポーネントを表示します。このようにして、アプリケーションのナビゲート中にユーザーがどこにいて何が起こるかをユーザーに通知できます。

フックの代替スイッチ


すべてのルートを含むルートオブジェクトを定義し、このオブジェクトをuseRoutes（）フックに渡すだけなので、ルートを条件別に表示することが非常に簡単になります。選択されたルートが定義されていないときに、レンダリング用のデフォルトのNoPageFoundコンポーネントを定義する場合、次のように結果ファイルとともにレンダリング用にこのファイルを渡すだけで済みます。

import { useRoutes, A } from "hookrouter";
import routes from "./router";
import NoPageFound from "./components/NoPageFound";
function App() {
  const routeResult = useRoutes(routes);
  return (
    <div className="App">
      <A href="/user">Users Page</A> <br />
      <A href="/about">About Page</A>
      <br />
      <A href="/contact">Contacts Page</A> <br />
      {routeResult || <NoPageFound />}
    </div>
  );
}



React Routerのコンポーネントを使用してデフォルトでページをレンダリングする場合と比較すると、コンポーネントは少しすっきりとして読みやすくなっています。

React Routerリダイレクト


リダイレクトは、ユーザーをあるルートから別のルートに動的に誘導するときに発生します。たとえば、認証中にユーザーが正常にログインした場合、ユーザーをルート「/ login」からルート「/ dashboard」にリダイレクトします。

React Routerでは、履歴オブジェクトまたは<Redirect />コンポーネントを使用するなど、いくつかの方法でこれを行うことができます。たとえば、ログインフォームがある場合、ブラウザの履歴オブジェクトを使用して、ログイン時にユーザーを「/ダッシュボード」ルートにリダイレクトできます。

import React from 'react'
class Login extends React.Component {
  loginUser = () => {
  // if (user is logged in successfully)
    this.props.history.push('/dashboard')
  }
  render() {
    return (
      <form>
        <input type="name" />
        <input type="email" />
        <button onClick={this.loginUser}>Login</button>
      </form>
    )
  }
}
export default Login

フックリダイレクト


hookrouterモジュールは、ソースルートとターゲットルートをパラメーターとして取ることができるuseRedirect（）フックを提供します。

useRedirect('/user', '/dashboard');

これにより、「/ユーザー」パスが見つかると、ユーザーは自動的に「/ダッシュボード」ルートにリダイレクトされます。たとえば、ユーザーを表示せずに、ユーザーを/ダッシュボードに自動的にリダイレクトする場合は、次のように記述します。

import {useRoutes, useRedirect} from 'hookrouter';
import dashboard from "./components/Dashboard";
const routes = {
    '/home': () => <Users />,
    '/dashboard': () => <Dashboard />
};
const Users = () => {
    useRedirect('/user', '/dashboard');
    const routeResult = useRoutes(routes);
    return routeResult
}



useRedirect（）フックがナビゲーション履歴の置き換えを引き起こすことは注目に値します。その結果、ナビゲーション履歴には1つのエントリのみが存在します。これは、検討中の最後のコードフラグメントの場合のように、「/ユーザー」から「/」ダッシュボードへのリダイレクトが発生した場合、ルート「/ユーザー」は閲覧履歴に表示されないことを意味します。「/ダッシュボード」ルートのみを使用します。

React RouterでのURLパラメーターの処理


URLパラメータは、動的URLに基​​づいてコンポーネントを表示するのに役立ちます。これはネストされたルートでも同様に機能しますが、この場合、ルートは変更されずに更新されます。

たとえば、アプリケーションに異なるユーザーがいる場合、「user / user1 /」や「users / user2 /」などの個別のルートでユーザーを個別に識別することは理にかなっています。これを行うには、URLパラメータを使用する必要があります。Reactルーターでは、<Route />コンポーネントのパスプロパティにコロンで始まるパラメーター（idなど）を渡すだけです。

<Route path="users/:id" component={Users} />

フックを使用してURLパラメーターを処理する


hookrouterとReact RouterでのURLパラメーターの処理に実質的な違いはありません。設計は同じです（つまり、コロンとパラメーター名を使用して、URLパラメーターをターゲットルートに渡すことができます）。

ただし、フックルーターの動作には違いがあります。すべてのURLパラメータを読み取り、オブジェクトに配置します。これは、routesオブジェクトで定義したキーを使用して行われます。次に、すべての名前付きパラメーターが、結合されたオブジェクトとしてルートの結果関数にリダイレクトされます。

const routes = {
  '/user/:id': ({id}) => <User userId={id} />
}

オブジェクトの分解を使用して、propsオブジェクトからidプロパティを取得し、それをコンポーネントに適用します。したがって、React Routerとまったく同じ結果が得られます。

結論


Reactアプリケーションでの代替ルーティング方法を検討しました。Reactルーターは優れたツールですが、Reactにフックが登場したことで、ルーティングの仕組みなど、多くの変更が加えられました。このフックベースのモジュールは、小規模なプロジェクトでルートを処理するためのより柔軟でクリーンな方法を提供します。

投稿で説明されている主な機能に加えて、hookrouterには次のようないくつかの追加機能があります。

  • コンポーネントの遅延読み込み
  • ネストされたルート
  • ルートへの追加データの転送
  • 航海迎撃
  • サーバーレンダリングサポート
