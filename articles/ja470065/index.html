ボットは、どれだけ上手にプレイしても、ライブプレーヤーの代わりにはなりません。ボットがうまく機能しない場合-これは興味深いことではありません。強ければ、それは侮辱的で、また面白くありません。バランスを保つのは困難です（特に、プレイヤーごとに異なるため）。長い間、私はネットワークゲームを実装するつもりでしたが、すべては自分のサーバーを維持する必要性に依存していました。幸いなことに、決定は予想外の角度から来ました。

Ed van Zonは、私が長い間連絡を取り合っている人物です。偶然にも、開発者（Jeff MallettとMark Lefler）がプロジェクトの開発に突然興味を失ったとき、彼がジリオンの落下バナーをキャッチしたのは彼でした。したがって、これらのすべてのゲームは彼の力によってサイトに公開されています。それでも、彼とChristian Freelingは自分のサイトのサポートと開発に取り組んでいます（もちろん、ボードゲームについても）：


しかし、これはすべてことわざでした。物語は、このサイトに穴（または、むしろ穴ですが、これはポイントではありません）があり、ライブプレーヤーが通信でプレイできるという事実から始まります。むしろ、Javaアプレットのテクノロジーが時代遅れであると見なされるまでは可能でした。最近、単なる人間による使用が困難になったため、エドはより近代的な解決策を考えています。そして、私は自分のプロジェクトと一緒に姿を現しました。


非常に短いトレーニングツアーの後、私の側では、Edがわずか2か月間、Dagazエンジンで30個の新しいゲームを入手し、サイトに投稿しました。私はあることを思い出してみましょうMITライセンス（私のような）、あらゆる方法で、このようなアクションをサポートしています。

SessionManager.prototype.save = function() {
  if (_.isUndefined(this.current) || _.isUndefined(this.current.board)) return null;
  var states = [];
  var board  = this.current.board;
  while (board.parent !== null) {
      states.push(board);
      board = board.parent;
  }
  var r = "(";
  while (states.length > 0) {
      var board = states.pop();
      r = r + ";" + Dagaz.Model.playerToString(board.parent.player);
      r = r + "[" + Dagaz.Model.moveToString(board.move) + "]";
  }
  r = r + ")";
  return r;
}


SessionManager.prototype.load = function(sgf) {
  var res = Dagaz.Model.parseSgf(sgf);
  this.states = [];
  delete this.current;
  var board = Dagaz.Model.getInitBoard();
  this.addState(Dagaz.Model.createMove(), board);
  for (var i = 0; i < res.length; i++) {
       var p = res[i].name;
       if (p != Dagaz.Model.playerToString(board.player)) return false;
       if (res[i].arg.length != 1) return false;
       var move = this.locateMove(board, res[i].arg[0]);
       if (move === null) return false;
       board = board.apply(move);
       this.addState(move, board);
  }
  this.controller.setBoard(board);
  return true;
