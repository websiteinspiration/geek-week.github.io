セキュリティインシデントの検出


今回のケースでelasticsearchを使用する主なタスクは、情報セキュリティインシデントのみを収集することです。ログ転送の少なくともいくつかのモードをサポートしていれば、あらゆる保護手段から情報セキュリティインシデントを収集できます。標準的なものは、syslogまたはファイルへのscp保存です。

ログ転送を構成する場所だけでなく、セキュリティ機能の標準的な例を示すことができます。

  1. NGFWの任意の手段（チェックポイント、フォーティネット）;
  2. 脆弱性スキャナー（PTスキャナー、OpenVas）。
  3. Webアプリケーションファイアウォール（PT AF）;
  4. Netflowアナライザー（Flowmon、Cisco StealthWatch）;
  5. ADサーバー。

Logstashでログ転送と構成ファイルを設定した後、さまざまなセキュリティツールからのインシデントと関連付けて比較できます。これを行うには、特定のデバイスに関連するすべてのインシデントを格納するインデックスを使用すると便利です。つまり、1つのインデックスは1つのデバイスに対するすべてのインシデントです。このようなディストリビューションを実装するには2つの方法があります。

最初のオプションは、Logstash構成を構成することです。これを行うには、特定のフィールドのログを別のタイプの別のユニットに複製する必要があります。そして、後でこのタイプを使用します。この例では、ログはCheck Point IPSファイアウォールブレードを使用して複製されます。

filter {
    if [product] == "SmartDefense" {
        clone {
	    clones => ["CloneSmartDefense"]
	    add_field => {"system" => "checkpoint"}
	}
    }
}

このようなイベントを別のインデックスに保存するには、たとえば、宛先IP攻撃シグネチャなどのログフィールドに応じて。同様の構成を使用できます。

output {
    if [type] == "CloneSmartDefense"{
    {
         elasticsearch {
    	 hosts => [",<IP_address_elasticsearch>:9200"]
    	 index => "smartdefense-%{dst}"
    	 user => "admin"
    	 password => "password"
  	 }
    }
}

このようにして、IPアドレスやマシンのドメイン名などによって、すべてのインシデントをインデックスに保存できます。この場合、シグネチャのIPアドレスにあるインデックス"smartdefense-％{dst}"に保存します。

ただし、製品ごとにログのフィールドが異なるため、混乱や不要なメモリ消費が発生します。そして、ここでLogstashの構成設定でフィールドを以前に考えられたフィールドに置き換えることが慎重に必要になります。

ELKスタックが必要な最も明白で理解可能な理由は、ログの保存と視覚化です。以前の記事では、Logstashを使用してさまざまなデバイスからファイルをログに記録する方法を示しました。ログがElasticsearchに送信されたら、以前の記事でも触れたダッシュボードをセットアップして、視覚化により必要な情報と統計を提供できます。

例：

  1. 最も重要なイベントを含む脅威防止イベントダッシュボード。ここでは、検出されたIPSシグネチャを地理的にどこから取得したかを反映できます。

この場合、ダッシュボードを数秒ごとに更新するように構成すると、イベントをリアルタイムで監視するためのかなり便利なシステムを取得できます。これを使用して、別の画面にダッシュボードを配置すると、ISインシデントにすばやく対応できます。

インシデントの優先順位付け


大規模なインフラストラクチャの状況では、インシデントの数がチャートから外れる可能性があり、スペシャリストがすべてのインシデントを時間どおりに管理することができなくなります。この場合、最初に、大きな脅威となるインシデントを特定する必要があります。したがって、システムは、インフラストラクチャに関連する危険性に従ってインシデントに優先順位を付ける必要があります。イベントデータのメールまたは電報にアラートを設定することをお勧めします。優先順位付けは、視覚化を設定することにより、通常のKibanaツールを使用して実装できます。ただし、通知の方が難しいです。デフォルトでは、この機能は基本バージョンのElasticsearchには含まれていません。有料バージョンにのみ含まれています。したがって、有料版を購入するか、または再度、電子メールまたは電報でリアルタイムで専門家に通知するプロセスを自分で記述します。
