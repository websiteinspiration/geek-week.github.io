Splunkのアプリケーションインターフェイスローカリゼーションメカニズムの分析


この記事では、Splunkアプリケーションのインターフェース（検索アプリケーションの標準要素を含む）をローカライズするための主要なメカニズムであるgettextの国際化とローカリゼーション（i18n）について考察します。

翻訳の機会：

  • Splunkインターフェース
  • ダッシュボード（パネルのタイトルと入力フィールド）;
  • 現在のロケールに基づく静的リソース（画像、CSSなど）の選択（例：logo-en_RU.gifまたはlogo-en_GB.gif）。

ローカリゼーションを行うには、いくつかの手順を実行する必要があります。

  1. Splunkで新しいロケールを作成します（必要な場合）。
  2. 翻訳辞書ファイルを生成します。
  3. JavaScriptを使用して、翻訳されたフィールドをロードします。

また、OpenVASの脆弱性の例を使用して検索結果をさらに翻訳します。これは、翻訳システム自体には適用されませんが、全体像をよく補完します。

Splunkの現在の問題に関心のある方は、テレグラム（@SplunkRU）のチャンネルにご参加ください。

それでは、練習に移りましょう。

Splunkで新しいロケールを作成する


デフォルトではSplunkでen-USロケールを使用することを思い出させてください。これはブラウザー行で確認できます：MYSERVER：8000 / en-US / app / search（ヒント： en-USを手動でen-GBに変更すると、時刻と日付はより身近な形で表示されます）。

まず、ロシアの新しいロケールを作成します。既存のロケールをコピーします。

sudo cp /opt/splunk/lib/python2.7/site-packages/splunk/appserver/mrsparkle/locale/en_GB /opt/splunk/lib/python2.7/site-packages/splunk/appserver/mrsparkle/locale/ru_RU -ru

Splunkの再起動：

sudo /opt/splunk/bin/splunk restart

そしてチェック：

MYSERVER:8000/ru-RU/app/launcher/home

翻訳ファイルの作成


次に、翻訳ファイルの作成に直接進みます。

既存のアプリケーションを翻訳できます。新しく作成したアプリケーションに対してこれを行います。

sudo /opt/splunk/bin/splunk extract i18n -app testapp

その後/opt/splunk/etc/apps/testapp/locale/messages.pot、アプリケーションで使用されるすべてのフィールドを含むファイルがあります。新しいアプリケーションがあります。要素は1つだけです-アプリケーションの名前へのリンク。poeditor.com

JSスクリプト


次のステップでは、トークンを対応するフィールドロケールに置き換えるJSスクリプトを記述します。すぐにいくつかの言語への翻訳の可能性を築きます：ロシア語とドイツ語。

スクリプトが読み込まれるフォルダを作成します。

sudo mkdir /opt/splunk/etc/apps/testapp/appserver
sudo mkdir /opt/splunk/etc/apps/testapp/appserver/static

スクリプト自体は/opt/splunk/etc/apps/testapp/appserver/static/dashTranslate.jsに記述します。

require([
    'jquery',
    'underscore',
    'splunk.i18n',
    'splunkjs/mvc',
    'splunkjs/mvc/simplexml/ready!'
  ], function ($, _, i18n, mvc) {      
      var defaultTokens = mvc.Components.get("default");
      var envTokenModel = mvc.Components.getInstance("env");
      
      	   if (envTokenModel.get("locale") != "ru-RU" || envTokenModel.get("locale") != "de-DE") {
    	      defaultTokens.set("form.t_locale", "ru-RU"); //       ,      (),        ( , ,       lookup ,       ,       $env::locale$ )
    } else {
      defaultTokens.set("form.t_locale", envTokenModel.get("locale")); // set locale token
    }
      
//      ,     
      if (i18n._("openvasTitle") == "openvasTitle") {
        defaultTokens.set("form.t_openvasTitle", "OpenVAS Events"); // default value
      } else {
        defaultTokens.set("form.t_openvasTitle", i18n._("openvasTitle")); // translated value
      }
      
      //  .           ,      
      defaultTokens.set("form.t_signature", i18n._("signature"));
      defaultTokens.set("form.t_description", i18n._("description"));
      defaultTokens.set("form.t_count", i18n._("count"));
  
    });

ダッシュボードを作る


新しいダッシュボードを作成して追加します（最後にレイジーのダッシュボードソース）：

  • 入力テキスト-名前：カウント、トークン：t_count
  • 入力テキスト-名前：openvasTitle、トークン：t_openvasTitle
  • 入力テキスト-名前：署名、トークン：t_signature
  • 入力テキスト-名前：説明、トークン：t_description

フォーム。*トークンは入力なしでは機能しないため、ダッシュボードの完全な状態をURLに転送できません。無邪気な目をしないように、それぞれに「depends = "$ nothing $"」を追加します。

パネル、統計表-タイトル：$ t_openvasTitle $

index=openvas
| stats count by "NVT Name" Summary
| rename "NVT Name" AS $form.t_signature$ Summary AS $form.t_description$ count AS $form.t_count$

パネル、円グラフ-タイトル：$ t_openvasTitle $

index=openvas
| stats count by "NVT Name"
| rename "NVT Name" AS $form.t_signature$ count AS $form.t_count$

制限事項


注意すべき2つの制限があります。

  1. ダッシュボードのヘッダーは翻訳できません。
  2. init \ setでトークンの初期値を設定することは不可能です。翻訳スクリプトの後に実行されます（JSを後でダウンロードできますが、HTMLに変換する必要があります）。

検索結果の翻訳


エンドユーザーにテキストデータを表示するには、ルックアップを使用して、任意の数の言語に翻訳することが提案されています。この例では、グローバルトークンSplunk $ env :: local $を使用しています。脆弱性識別子は、翻訳を接続するためのハッシュです。また、変換が空の場合、初期値が使用されることにも注意してください。ダッシュボードに新しい静的テーブルを追加します。

index="openvas"
| eval signature=replace(signature, "[\n\r]", " ")
| eval description=replace(description, "[\n\r]", " ")
| eval hash=md5(signature.description)
| stats count by signature description hash
| lookup OpenVAS_translate hash AS hash OUTPUT signature_$env:locale$ AS signature_$env:locale$ description_$env:locale$ AS description_$env:locale$
| eval signature=coalesce('signature_$env:locale$', signature) 
| eval description=coalesce('description_$env:locale$', description)
| fields - signature_$env:locale$ description_$env:locale$ count hash
| rename signature AS $form.t_signature$  description AS $form.t_description$

OpenVAS_translate.csvファイルは次のようになります（ルックアップを操作するには、ルックアップエディターアプリケーションを使用することをお勧めします）。 したがって、翻訳がある場合、現在のロケールに基づいて作成されます。 ru-RU： de-DE：
