I / Oストリームを作成する


ChatRoom.swiftの 内容を次のように置き換えます。

import UIKit
class ChatRoom: NSObject {
  //1
  var inputStream: InputStream!
  var outputStream: OutputStream!

  //2
  var username = ""

  //3
  let maxReadLength = 4096
}

ここでは、ChatRoomクラスを定義し、必要なプロパティを宣言します。

  1. 最初に、入力/出力ストリームを定義します。それらをペアとして使用すると、アプリケーションとチャットサーバーの間にソケット接続を作成できます。もちろん、出力ストリームを使用してメッセージを送信し、入力ストリームを使用して受信します。
  2. 次に、ユーザー名を定義します。
  3. 最後に、変数maxReadLengthを定義します。これは、単一メッセージの最大長を制限します。

次にChatRoomViewController.swiftファイルに移動して、次の行をそのプロパティのリストに追加します。

let chatRoom = ChatRoom()

クラスの基本構造を作成したので、計画されている最初のタスク（アプリケーションとサーバー間の接続を開く）を実行します。

接続を開く


ChatRoom.swiftに戻り、このメソッドをプロパティ定義に追加します。

func setupNetworkCommunication() {
  // 1
  var readStream: Unmanaged<CFReadStream>?
  var writeStream: Unmanaged<CFWriteStream>?

  // 2
  CFStreamCreatePairWithSocketToHost(kCFAllocatorDefault,
                                     "localhost" as CFString,
                                     80,
                                     &readStream,
                                     &writeStream)
}

ここでは、次のことを行います。

  1. 最初に、自動メモリ管理を使用せずにソケットストリームの2つの変数を定義します
  2. 次に、同じ変数を使用して、ホストとポート番号にバインドされたストリームを直接作成します。

この関数には4つの引数があります。1つ目は、スレッドを初期化するときに使用するメモリアロケータのタイプです。スレッドの動作を変更する場合に備えて、他の可能なオプションがありますが、kCFAllocatorDefaultを使用する必要があります。

翻訳者メモ
CFStreamCreatePairWithSocketToHost : NULL kCFAllocatorDefault. kCFAllocatorDefault , NULL. !

次に、ホスト名を設定します。この例では、ローカルサーバーに接続しています。サーバーが別の場所にある場合は、そのIPアドレスを設定できます。

次に、サーバーがリッスンしているポート番号。

最後に、I / Oストリームへのポインタを渡し、関数がそれらを初期化して、作成したストリームに接続できるようにします。

初期化されたフローができたので、setupNetworkCommunication（）メソッドの最後にこれらの行を追加して、フローへのリンクを保存できます。

inputStream = readStream!.takeRetainedValue()
outputStream = writeStream!.takeRetainedValue()

アンマネージオブジェクトに適用されるようにtakeRetainedValue（） を使用すると、そのオブジェクトへの参照を維持できると同時に、将来のメモリリークを回避できます。これで、スレッドをどこでも使用できます。

次に、アプリケーションがネットワークイベントを正しく処理できるように、これらのスレッドを実行ループに追加する必要があります。これを行うには、setupNetworkCommunication（）の最後に次の2行を追加します。

inputStream.schedule(in: .current, forMode: .common)
outputStream.schedule(in: .current, forMode: .common)

ついに出航の時間です！はじめに、これをsetupNetworkCommunication（）メソッドの最後に追加します。

inputStream.open()
outputStream.open()

これで、クライアントアプリケーションとサーバーアプリケーションの間の接続が開かれました。

アプリケーションをコンパイルして実行することはできますが、現時点ではクライアントサーバー接続で何も実行していないため、変更はまだ表示されません。

チャットプロトコル


TCPレイヤーを使用する利点の1つは、通信用に独自の「プロトコル」を定義できることです。

HTTPを使用する場合は、GET、PUT、PATCHというこれらの異なる単語を使用する必要があります。 URLを作成し、適切なヘッダーを使用する必要があります。

非常にシンプルですが、まったく無理がないため、重要なプロジェクトではこの方法を使用しないでください。

これで、サーバーが何を期待しているかがわかり、クラスにメソッドを書き込むことができますChatRoom、ユーザーがチャットに接続できるようにします。唯一の引数は、ユーザーのニックネームです。ChatRoom.swift

内にこのメソッドを追加します。

func joinChat(username: String) {
  //1
  let data = "iam:\(username)".data(using: .utf8)!
  
  //2
  self.username = username

  //3
  _ = data.withUnsafeBytes {
    guard let pointer = $0.baseAddress?.assumingMemoryBound(to: UInt8.self) else {
      print("Error joining chat")
      return
    }
    //4
    outputStream.write(pointer, maxLength: data.count)
  }
}

  1. まず、独自の「プロトコル」を使用してメッセージを形成します
  2. 後で参照できるように名前を保存します。
  3. withUnsafeBytes（_ :)は、クロージャ内の安全でないポインタを操作する便利な方法を提供します。
  4. 最後に、メッセージを出力ストリームに送信します。これは予想以上に複雑に見えるかもしれませんが、書き込み（_：maxLength :)は前のステップで作成された安全でないポインタを使用します。

これでメソッドの準備ができました。ChatRoomViewController.swiftを開き、このメソッドの呼び出しをviewWillAppear（_ :)の最後に追加します。

chatRoom.joinChat(username: username)

受信メッセージへの応答


サーバーは、あなたを含むチャットに参加している全員にクライアント参加メッセージを送信します。幸いなことに、このアプリケーションには、ChatRoomViewControllerのメッセージテーブルのセルの形式で着信メッセージを表示するためのすべてのものがすでにあります。

必要なのは、inputStreamを使用してこれらのメッセージを「キャッチ」し、それらをMessageクラスのインスタンスに変換して、表示するためにテーブルに渡すことだけです。

着信メッセージに応答できるようにするには、ChatRoomがStreamDelegateプロトコルに準拠する必要があります。 これを行うには、ChatRoom.swiftファイルの下部にこの拡張機能を追加します。



extension ChatRoom: StreamDelegate {
}

次に、inputStreamのデリゲートになるユーザーを宣言します。

スケジュールを呼び出す直前に、setupNetworkCommunication（）メソッドに次の行を追加します（inMode :)：

inputStream.delegate = self

次に、ストリーム（_：ハンドル:)メソッドの実装を拡張機能に追加します。

func stream(_ aStream: Stream, handle eventCode: Stream.Event) {
    switch eventCode {
    case .hasBytesAvailable:
      print("new message received")
    case .endEncountered:
      print("The end of the stream has been reached.")
    case .errorOccurred:
      print("error occurred")
    case .hasSpaceAvailable:
      print("has space available")
    default:
      print("some other event...")
    }
}

着信メッセージを処理します


これで、着信メッセージの処理を開始する準備ができました。関心のあるイベントは.hasBytesAvailableで、着信メッセージが到着したことを示します。

これらのメッセージを処理するメソッドを記述します。新しく追加されたメソッドの下に、次のように記述します。

private func readAvailableBytes(stream: InputStream) {
  //1
  let buffer = UnsafeMutablePointer<UInt8>.allocate(capacity: maxReadLength)

  //2
  while stream.hasBytesAvailable {
    //3
    let numberOfBytesRead = inputStream.read(buffer, maxLength: maxReadLength)

    //4
    if numberOfBytesRead < 0, let error = stream.streamError {
      print(error)
      break
    }

    // Construct the Message object
  }
}

  1. 着信バイトを読み取るバッファを設定します。
  2. ループでスピンしますが、入力ストリームには読み取るものがあります。
  3. read（_：maxLength :)を呼び出します。これは、ストリームからバイトを読み取り、それらをバッファーに入れます。
  4. 呼び出しが負の値を返した場合、エラーを返し、ループを終了します。

着信ストリームにデータが入ったらすぐにこのメソッドを呼び出す必要があるため、ストリーム（_：handle :)メソッド内のswitchステートメントに移動し、.hasBytesAvailableスイッチを見つけて、printステートメントの直後にこのメソッドを呼び出します。

readAvailableBytes(stream: aStream as! InputStream)

この場所には、受信したデータのバッファが用意されています！

ただし、このバッファをメッセージテーブルのコンテンツに変換する必要があります。

このメソッドをreadAvailableBytes（stream :)に配置します。

private func processedMessageString(buffer: UnsafeMutablePointer<UInt8>,
                                    length: Int) -> Message? {
  //1
  guard 
    let stringArray = String(
      bytesNoCopy: buffer,
      length: length,
      encoding: .utf8,
      freeWhenDone: true)?.components(separatedBy: ":"),
    let name = stringArray.first,
    let message = stringArray.last 
    else {
      return nil
  }
  //2
  let messageSender: MessageSender = 
    (name == self.username) ? .ourself : .someoneElse
  //3
  return Message(message: message, messageSender: messageSender, username: name)
}

まず、このメソッドに渡すバッファとサイズを使用して文字列を初期化します。

テキストはUTF-8になり、最後にバッファーを解放し、メッセージを記号「：」で分割して、送信者の名前とメッセージ自体を区切ります。

ここで、メッセージが別の参加者からのものかどうかを分析しています。製品では、一意のトークンのようなものを作成できます。これはデモには十分です。

    processedMessageString(buffer: buffer, length: numberOfBytesRead) {
  // Notify interested parties

ChatRoomDelegateプロトコルを作成する


したがって、新しいメッセージについてChatRoomViewController.swiftに通知する必要がありますが、それへのリンクはありません。強力なチャットルームリンクが含まれているため、強力なリンクサイクルの罠に陥ることがあります。

これは、デリゲートプロトコルを作成するのに最適な場所です。ChatRoomは、新しい投稿について誰が知る必要があるかを気にしません。ChatRoom.swift

の上部に、新しいプロトコル定義を追加します。

protocol ChatRoomDelegate: class {
  func received(message: Message)
}

次にChatRoomクラス内に、デリゲートとなるストアへの弱いリンクを追加します。

weak var delegate: ChatRoomDelegate?

次にreadAvailableBytes（stream :)メソッドを追加し、if-letコンストラクト内のメソッドの最後のコメントの下に次の行を追加します。

delegate?.received(message: message)

ChatRoomViewController.swiftに 戻り、次のクラス拡張を追加します。これにより、MessageInputDelegateの直後にChatRoomDelegateプロトコルへの準拠が保証されます。

extension ChatRoomViewController: ChatRoomDelegate {
  func received(message: Message) {
    insertNewMessageCell(message)
  }
}

元のプロジェクトにはすでに必要なものが含まれているため、insertNewMessageCell（_ :)はメッセージを受け入れ、テーブルビューに正しいセルを表示します。super.viewWillAppear（）を呼び出した直後に

これをviewWillAppear（_ :)に追加して、ビューコントローラーをデリゲートとして割り当てます。

私たちは自分をきれいにする


積極的にファイルを読み書きするアプリケーションを開発したことがある場合は、優れた開発者が作業を終えたときにファイルを閉じることを知っておく必要があります。実際には、ソケットを介した接続はファイル記述子によって提供されます。つまり、他のファイルと同様に、作業の最後にファイルを閉じる必要があります。

これを行うには、send（message :)を定義した後、ChatRoom.swiftに次のメソッドを追加します。

func stopChatSession() {
  inputStream.close()
  outputStream.close()
}

ご想像のとおり、このメソッドはスレッドを閉じ、メッセージを送受信できなくなります。さらに、スレッドは以前に配置した実行ループから削除されます。

次にChatRoomViewController.swiftに戻り、viewWillDisappear（_ :)で同じことを行います。

chatRoom.stopChatSession()

結論


ソケットを使用したネットワークの基本をマスターしたので、知識を深めることができます。

UDPソケット


このアプリケーションは、宛先へのパケットの配信を保証するTCPを使用したネットワーク通信の例です。

ただし、UDPソケットは使用できます。このタイプの接続は、意図した目的へのパッケージの配信を保証するものではありませんが、はるかに高速です。

これは特にゲームで役立ちます。ラグを経験したことがありますか？これは、接続が不良で、多くのUDPパケットが失われたことを意味します。

WebSocket


アプリケーションにおけるHTTPのもう1つの代替手段は、Webソケットと呼ばれるテクノロジーです。

通常のTCPソケットとは異なり、WebソケットはHTTPを使用して通信を確立します。彼らの助けを借りて、通常のソケットと同じように、ブラウザのように快適さとセキュリティを実現できます。
