Pythonistes, arrêtez d'utiliser les crochets pour obtenir les valeurs d'un dictionnaire

Bonjour, Habr! Je vous présente la traduction de l'article «Arrêtez d'utiliser la notation entre crochets pour obtenir la valeur d'un dictionnaire en Python» de Jonathan Hsu.

Aller au-delà de la méthode traditionnelle et protéger votre code


Un dictionnaire est une collection non triée de clés et de valeurs. Cela signifie que:

  • Chaque élément du dictionnaire se compose d'un terme (clé) et de sa définition (signification).
  • Les clés doivent être uniques pour ce dictionnaire - pas de répétitions.
  • Les clés du dictionnaire n'ont pas la disposition explicite des éléments de la liste.

Pour créer un dictionnaire, utilisez des accolades, à l'intérieur desquelles écrivez les paires clé: valeur séparées par des virgules. Comme ça:

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}

Une façon traditionnelle (et peu sûre) d'obtenir un sens à partir d'un dictionnaire.

Pour un accès normal à la valeur d'un élément à partir d'un dictionnaire, des crochets sont utilisés. Lorsque vous utilisez une telle entrée, la clé est indiquée entre crochets après le nom du dictionnaire:

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author['username']) #  Friday1719
print(author['middle_initial']) #   KeyError: 'middle_initial'

Notez qu'essayer d'accéder à une valeur par une clé qui n'est pas dans le dictionnaire lèvera une exception KeyError. Cela peut créer de graves problèmes, en particulier lorsqu'il s'agit de données de travail imprévisibles.

Bien sûr, vous pouvez utiliser la construction try / except ou utiliser l'instruction if. Mais une telle attention accrue à une opération aussi simple ne fait qu'encombrer le code.

author = {}
try:
   print(author['username'])
except KeyError as e:
   print(e) # 'username'

if 'username' in author:
   print(author['username'])

Si vous avez une formation en développement JavaScript, vous souhaiterez peut-être obtenir la valeur du dictionnaire en utilisant la notation en pointillés. Mais en Python, cela ne fonctionnera pas.

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author.username)
# AttributeError: 'dict' object has no attribute 'username'

Nous utilisons la méthode get ()


Lorsque vous devez obtenir une valeur à partir d'un dictionnaire, le moyen le plus sûr consiste à utiliser la méthode get (). Cette méthode a deux paramètres:

  • Premier (obligatoire): le nom de la clé par laquelle nous voulons obtenir la valeur. Ce nom peut être une chaîne ou un nom de variable si notre clé peut changer pendant le programme.
  • Deuxième (facultatif): la valeur qui sera utilisée si notre clé n'apparaît pas soudainement dans le dictionnaire.

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author.get('username')) #   Friday1719
print(author.get('middle_initial', “Monday1917”)) #  Monday1917

Si la clé existe dans le dictionnaire, la méthode get () fonctionne exactement de la même manière que l'accès à la clé entre crochets. Mais dans le cas où il n'y a pas une telle clé dans le dictionnaire, la méthode get () renverra la valeur par défaut, vous évitant d'avoir à gérer l'exception.

La valeur par défaut peut être n'importe quel objet valide dans ce contexte. N'oubliez pas que ce paramètre est facultatif. Par conséquent, si vous ne le spécifiez pas explicitement, lorsque vous essayez d'accéder à une clé qui n'existe pas dans le dictionnaire, la méthode get () renvoie un objet None.

Nous utilisons la méthode setdefault ()


Parfois, vous devrez non seulement recevoir en toute sécurité des données du dictionnaire, mais également ajouter en toute sécurité de nouvelles données au dictionnaire. Pour cela, les dictionnaires ont une méthode setdefault (). Elle a les mêmes paramètres que la méthode get (), mais contrairement à cette dernière, lors de l'accès au dictionnaire à l'aide d'une clé inexistante, elle renverra non seulement la valeur par défaut, mais créera également un nouvel élément dans le dictionnaire avec cette clé et la valeur transmise. Si lors de l'accès au dictionnaire à l'aide de la méthode setdefault (), la clé transmise est déjà dans le dictionnaire, cette méthode laissera le dictionnaire inchangé.

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}

print (author.setdefault ('username')) # affichera vendredi1719
print (author.setdefault ('middle_initial', "lundi")) # affichera lundi et créera
# dans le dictionnaire un élément avec la clé 'middle_initial' et la valeur de cette clé "lundi ”

Dans l'exemple ci-dessus, nous voyons que le comportement de la méthode setdefault () n'est pas différent du comportement de la méthode get () ou de l'utilisation de crochets lors de l'accès au dictionnaire en utilisant la clé qui y est. S'il n'y a pas une telle clé dans le dictionnaire, la méthode setdefault () renverra non seulement la valeur de son deuxième argument au programme (comme la méthode get ()), mais créera également un élément dans le dictionnaire avec la clé et la valeur qui lui seront transmises. Il s'agit du comportement de la méthode setdefault () qui le distingue de la méthode get ().

Maintenant, si vous suivez l'exemple ci-dessus et affichez les éléments du dictionnaire,on obtient alors le résultat suivant:

print(author)
{
  "first_name": "Robinson",
  "last_name": "Crusoe",
  "username": "Friday1719"
  "middle_initial": "Monday"
}

L'utilisation des méthodes get () et setdefault () est une technique de première classe pour accéder aux valeurs du dictionnaire. Il ne faut que du temps pour abandonner les vieilles habitudes et commencer à utiliser cette technique dans la pratique.

Si vous avez seulement besoin d'obtenir la valeur du dictionnaire, votre aide est la méthode get ().

Si vous devez ajouter en toute sécurité une nouvelle valeur au dictionnaire, appelez la méthode setdefault ().
