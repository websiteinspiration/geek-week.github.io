- Pourquoi est-ce nécessaire?

Le cas le plus courant est que la base de données comptables se trouve dans l'entreprise et non sur l'ordinateur personnel du comptable. Si un comptable tombe malade ou qu'il lui arrive autre chose, il sera difficile de tout obtenir. Les cas les plus rares sont ceux où vous devez avoir votre environnement de travail sur votre téléphone depuis n'importe où dans le monde, ou lorsque vous utilisez souvent les ordinateurs d'autres personnes.

Il est également utile pour créer des serveurs de jeux, négocier en bourse, des torrents, divers processus répétitifs, des programmes pour automatiser les réseaux sociaux (et comme les tricheurs, ce qui est déjà là), les utilitaires de référencement, préparer la comptabilité blanche, voyager (surtout lorsque vous ne voulez pas emmenez votre ordinateur à la maison quelque part dans une ville voisine pendant un mois), etc. Le plus souvent, VPS est utilisé pour la sécurité de la banque cliente de l'entreprise.

- Comment installer la première fois?

La logique derrière le levage d'un serveur de bureau distant consiste à déployer une machine virtuelle (VPS), puis à installer une distribution Windows qui prend en charge la connectivité à distance. Le premier est fait à partir du compte personnel d'hébergement VPS (cela prend de quatre à sept minutes avec SLA pendant 15 minutes), le second pourrait vous faire bien apprendre le matériel, mais dans notre cas, Windows Server est sur le marché, et vous pouvez commander une machine tout de suite avec un système d'exploitation étendu système. L'accès à la connexion se fera directement dans votre compte. Il ne reste plus qu'à se connecter.

- Quelle est la configuration système requise pour Windows Server?

La configuration minimale est de 2 Go de RAM, 20 Go sur le disque dur. Pour Win Server 2003, un cœur suffit; pour Win Server 2016 ou 2019, deux sont déjà nécessaires. Il est préférable d'ajouter un peu plus d'espace sur le disque dur, les deux systèmes d'exploitation seront serrés à 20 Go - déjà S2012, par exemple, ne pourra pas être mis à niveau. Certains utilisateurs installent Win sur VPS avec 500 Mo de RAM. Nous recommandons un tarif de 579 roubles par mois (cela n'inclut pas les remises pour Habr, vous pouvez composer 15% du code promo ci-dessous et 20% supplémentaires lorsque vous payez pour l'année), où il y a un cœur, 2 Go de RAM et 20 Go de disque dur. À des tarifs moins élevés, il est préférable de mettre Windows Server Core ou un autre système d'exploitation.

- Est-il possible de configurer des sauvegardes planifiées?

Oui, vous pouvez prendre une copie de la machine VPS, par exemple, une fois par jour. Configurez via votre compte. Une copie est effectuée vers un stockage indépendant dans le même centre de données où votre VPS est déployé. Si vous souhaitez copier vers un autre centre de données, vous devrez le configurer manuellement.

- Et si Windows de l'autre côté cesse de répondre?

Si quelque chose arrive au système d'exploitation ou au service d'accès au terminal, vous ne pourrez pas vous connecter via RDP, car cela nécessite un système d'exploitation fonctionnel, des pilotes réseau, une connexion réseau fonctionnelle et RDP activé. Hors de la boîte, nous avons tout. Mais si Win "tombe en panne", il sera alors possible d'utiliser un autre mode spécial: nous avons la possibilité de diffuser de la vidéo directement à partir du niveau de l'hyperviseur et de transmettre des frappes et des clics via l'hyperviseur. Ce mode est assez exigeant en ressources, il y a donc une limite à son utilisation. Mais en cas de problème, vous pouvez l'utiliser pour redémarrer en mode sans échec. Si le problème est plus grave, vous pouvez soit contacter le support, soit simplement réinstaller Windows en quelques clics.

Le mode d'urgence peut avoir une réponse assez longue aux actions, mais visuellement, il semble assez habituel. C'est tellement courant que nous avons des utilisateurs qui l'utilisent régulièrement. Mais nous n'avons absolument pas besoin d'être utilisés dans la vie quotidienne en dehors d'une urgence, car il existe des moyens plus corrects et confortables, pour Windows - bien sûr, c'est RDP.

- Et la licence OS?

Habituellement, l'hébergement VPS vous donne un serveur virtuel et ne regarde pas ce que vous en faites (uniquement pour le téléchargement par les ressources et la nature du trafic). Autrement dit, en théorie, vous pouvez y mettre n'importe quel logiciel, y compris les logiciels piratés. Mais en pratique, il est plus pratique de créer une machine avec un système d'exploitation en utilisant l'un des modèles d'hébergement. Dans notre cas, une licence pour Windows Server (ou Windows Server Core sans interface graphique) est déjà incluse dans le plan tarifaire, car nous avons tous l'hébergement sur Hyper-V et est entièrement sous licence. Certaines autres sociétés d'hébergement ouvrent en ce moment une nouvelle "star" avec le coût mensuel de location d'une licence.

- Puis-je apporter ma propre image de système d'exploitation?

Oui, vous pouvez et il y a deux options. Vous pouvez créer un conteneur VHDX standard pour Hyper-V et le transférer vers le support. Dans ce cas, il peut y avoir des difficultés lors de la configuration d'une connexion réseau - peut-être que dans votre image finale, il n'y aura pas de pilote pour une carte réseau appropriée. Vous devrez vous connecter et l'installer. Néanmoins, c'est un bon moyen de tout transférer, par exemple, d'un vieil ordinateur portable à un ordinateur en ligne. Ou vous pouvez transférer la prise en charge vers l'image ISO, qui est montée sur un serveur virtuel, et à partir de laquelle vous, en utilisant le mode d'urgence via votre compte personnel, pouvez effectuer l'installation vous-même.

Ces services sont fournis uniquement sur des plans tarifaires de 1 000 roubles par mois.

- Quel stockage est utilisé?

Tous les disques sont des étagères SSD dans les serveurs, mais sur les tarifs avec les disques durs, leur vitesse est limitée aux vitesses des disques durs de broche. Duplication - RAID 5. Il reste plusieurs RAID 6, mais il y en a déjà pas mal. Si l'un des disques tombe en panne, la reconstruction du raid commence. Si une panne de courant se produit à ce stade, les données seront perdues. Dans ce cas, nous avons deux alimentations distinctes avec des chemins de câbles séparés vers chaque serveur physique et une sauvegarde technique de la machine entière (sans accès sélectif à un stockage séparé).
