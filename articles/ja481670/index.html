AWS CAFアクションプランは、クラウドテクノロジースタックへの移行の準備に役立ちます。旅は、CAFの6つの側面を検討するリーダーシップチームから始まります。各側面は、既存のスキルとプロセスのギャップを見つけるワークフローを作成するために使用され、それらはすでに入力として記録されています。この入力は、組織がクラウドテクノロジースタックに移行したときに変更を推進するAWS CAFアクションプランを作成するための基盤です。

AWSクラウド導入フレームワークのアクションプランの概要


アクションプランは、AWSクラウド導入フレームワーク（AWS CAF）の重要な部分です。アクションプランを開発するプロセスは、クラウドテクノロジーの採用によって引き起こされるタスクと課題の特定に役立ちます。作成されたアクションプランは、組織にプロアクティブなソリューションを提供し、クラウドインフラストラクチャに移行する際の間違いを防ぐのに役立ちます。

AWSクラウド導入フレームワークとその側面


クラウドインフラストラクチャへの移行が成功すると、どの組織スキルを引き上げる必要があり、どのスキルを学ぶ必要があるかが決まります。 AWS CAFは、ギャップに対処するための効果的なクラウド導入計画を作成することを条件に、組織をサポートおよび指導します。組織に共通する6つの主要な領域（ビジネスコンポーネント、人、管理、プラットフォーム、セキュリティ、運用）に関するガイダンスを提供します。各側面は、特定の対象者と役割を考慮して編成されています。

ビジネスの側面の一般的な役割：ビジネスマネージャー、財務マネージャー、予算マネージャー、戦略的利害関係者。

人事面の一般的な役割：人事、サービスマネージャー、人事マネージャー。

管理面の一般的な役割：マネージングディレクター、部門長、プロジェクトマネージャー、システムアーキテクト、ビジネスインテリジェンス、投資マネージャー。

プラットフォームの一般的な役割：最高技術責任者、ITマネージャー、ソリューションアーキテクト。

セキュリティ面の一般的な役割：情報セキュリティ担当ディレクター、情報セキュリティマネージャー、情報セキュリティアナリスト。

運用面の一般的な役割：エグゼクティブITマネージャー、ITサポートマネージャー。

たとえば、ビジネスの側面は、ビジネスマネージャー、財務マネージャー、予算マネージャー、および戦略的利害関係者が、クラウドテクノロジーの採用の結果として、組織における役割の特定の側面がどのように変化するかを理解するのに役立ちます。

作成するアクションプランは、6つの側面に基づいています。


AWS CAFの各側面には、機能的に関与する1つ以上の利害関係者に属する機能が含まれています。それぞれの側面を検討するとき、関係者がクラウドインフラストラクチャの実装を成功させるためのスキルとプロセスをどのように改善するかを決定し始めます。これは4つの段階で発生します。

  • 組織内の誰がクラウドテクノロジーの採用に重要であるかを決定します。
  • 利害関係者のためのクラウドテクノロジーの実装を遅くしたり妨げたりする可能性がある問題や問題を特定します。
  • これらの問題に対処するために改善する必要があるスキルまたはプロセスを特定します。
  • スキルまたはプロセスの特定されたギャップに対処するための行動計画を作成します。
