DAT-どのようなプロトコルであり、誰が使用するか

このP2Pプロトコルの動作原理と、それに基づいて構築されたプロジェクトについて話しています。


/アンスプラッシュ/ Alina Grubnyak

データとは


Dat（分散アーカイブトランスポート）は、分散ネットワーク（ピアツーピア）内でデータとメッセージを交換するためのオープンプロトコルです。バージョンコントロール機能が組み込まれており、データセットの変更を監視できます。 Datは、エンジニアMax Ogdenによって開発されました。彼は、Code for Americaプロジェクトの一部として2013年にこのプロトコルを導入しました。 その参加者は、地方自治体のサービスの向上に貢献するサービスを開発しています。 Datは、国家間で市民の個人データを転送するためのツールとして作成されました。組織。しかしその後、プロジェクトの焦点は科学情報に移った。 Datの周りに大規模なコミュニティが形成されました（GitHubで 7000の星）



）プロトコルとそれに基づいて構築されたアプリケーションは、非営利団体Dat Foundationによって推進されています。Code for Science＆Societyオープンソース財団であるMozilla と、P2Pネットワーク開発者であるWireline によってサポートされています。

彼はどのように働きますか


ファイルをDatネットワークにアップロードするには、そのURLを指定する必要があります。次に例を示します。

dat://778f8d955175c92e4ced5e4f5563f69bfec0c86cc6f670352c457943666fe639/dat_intro.gif

アドレスリンクを見つけるために、DatクライアントはマルチキャストDNSを使用します。参加者の1人が「聞いて」情報を共有することを期待して、ピアはローカルネットワークに要求をブロードキャストします。顧客はインターネット上のサーバーにもアクセスできます。メインサーバーはdiscovery1.datprotocol.comにあります。-それは利用できない場合は、そのミラーを参照することができdiscovery2.datprotocol.com。

ピアが別のピアのIPアドレスとポート番号を学習すると、それらはTCP接続を確立します。送信されるすべてのデータは暗号化されます。これには、XSalsa20ストリーム暗号化システムが使用されます。アルゴリズムは、20サイクルのハッシュ関数を使用します。変換操作は、AESに関係する操作を連想させます。

Datネットワークのデータは個別のフラグメント（チャンク）で送信され、そのサイズは異なる場合があります。システムでは、Datファイルに新しいフラグメントを追加できますが、既存のフラグメントを変更または削除することはできません。開発者によると、このアプローチでは、ドキュメントの変更履歴全体を保存できます。システムは、接続が不安定な環境で自由に機能できます。


/ Unsplash / Sven Brandsma

現在、Dat Foundationのメンバーは、大量のデータを処理できるようにプロトコルを改善しています。特に、彼らは何百万ものファイルに対処するためにファイルシステム（Hyperdriveと呼ばれる）を作り直し、新しいピア検索エンジン（Hyperswarm）を導入することを計画しています。

誰が使う


例としては、オープンなScienceFairプロジェクトがあります。これは、科学文献を閲覧および検索するためのデスクトップアプリケーションです。このプラットフォームでは、科学者と研究者は個人的なメモ、雑誌、またはそれらからの抜粋を操作できます。科学文献のコンテンツを表示するために、ScienceFairはレンズリーダーを使用します。これはJATS XML形式のレンダリングを担当します。

Datはもともと科学情報を交換するためのプロトコルとして考えられていましたが、コミュニティはそれを使用してWebサイト、チャット、その他のアプリケーションを作成しています。

最近の1つの例は、ビーカー P2Pブラウザーです。 、Dat開発チームと共同で開発されました。その目的は、ユーザーが「ブラウザーで直接」Webサイトをホストできるようにすることです。Beakerの作者はHashbaseクラウドサービスを立ち上げました。これは、ローカルコピーが利用できないdatサイトへの常時アクセスを提供します。

プロジェクトは完全にオープンで、そのソースはGitHubにあります。ブラウザの機能を自分で評価する場合、Linuxで実行するには、libtool、m4、autoconfをインストールする必要があります。

sudo apt-get install libtool m4 make g++ autoconf # debian/ubuntu
sudo dnf install libtool m4 make gcc-c++ libXScrnSaver # fedora

十分な実行後：

git clone https://github.com/beakerbrowser/beaker.git
cd beaker
npm install
npm run rebuild # see https://github.com/electron/electron/issues/5851
npm start

その他のサンプルアプリケーションは、プロジェクトのWebサイトにあります。

アナログ


もちろん、最近活発に開発されているP2PプロトコルはDatだけではありません。別の方法は、IPFS（InterPlanetary File System）プロジェクトです。これは、ピアツーピアの分散ファイルシステムです。他の分散型ネットワークとの違いは、ブロックで機能することです。ファイルの一部と他のブロックへのリンクの両方を含めることができます。これらのブロックから、ディレクトリまたはファイルを形成する、一般化されたツリーのような有向グラフが形成されます。 IPFSは分散ハッシュテーブルで動作し、分散型ブロック交換をサポートします。 IPFSには障害点がなく、ホストは相互に信頼する必要はありません。ファイルシステムへのアクセスは、FUSEまたはHTTP を使用して取得できます。

ソリューションはすでにNeocitiesホスティングとマーケットプレイスを使用していますOpenBazaar。IPFSやDatなどのプロトコルの開発者は、彼らのプロジェクトがインターネットユーザーにデータのより多くの制御を与えることを望んでいます。
