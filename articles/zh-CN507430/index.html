在Postgresql中DIY生产日历







您好，我叫Victor，是Gems Development的一名开发人员。我想告诉我们如何在Postgresql中实现生产日历的创建和完成。



工作流程取决于国家和国际法定假日。在公司应用程序中，计算日期的任务通常会考虑工作日和周末。例如，应按时提供“签发施工许可证”，“签发调试许可证”等服务。



在分析任务之后，我们得出的结论是，在日历中存储周末和假日就足够了，因为它们很难计算并且可以根据每年的生产日历进行更改。



使用日历的规则：



  • 工作日的定义取决于日期是否在周末和节假日清单中。
  • 如果与转账相关的工作日不工作，则该工作日也将记录在表格中。
  • 如果休息日是工作日，则相应地，该表中的记录也将被删除。


为了演示，我们将创建一个单独的方案并在会话中设置一个变量，以便每次我们都不在请求中指定方案： 



create schema calendar;
set search_path=calendar;


为了支持这种计算，在数据库中创建了一个表： 



CREATE TABLE d_holidays
(
  holiday_date date,
  comment character varying(1000),
  CONSTRAINT d_holidays_pk PRIMARY KEY (holiday_date)
);


还有一个索引，可让您加快表中所需日期的搜索： 

CREATE UNIQUE INDEX d_holidays_pk_idx
  ON d_holidays
  USING btree
  (holiday_date);


为了创建数据，我们使用来自Postgresql的标准函数，该函数返回一个集合。让我们通过用足够大的一组生成数据预填充表来检查索引的操作：从01/01/2017到12/31/2025的所有天。 



insert into d_holidays(holiday_date)
select generate_series(to_date('01.01.2017','dd.mm.yyyy'),
            to_date('31.12.2025','dd.mm.yyyy'),
            '1 day');


我们获得了3287条记录，对应于从指定日期开始的天数。确保在Postgresql配置中启用参数“ autovacuum = on”很重要。



让我们看一下查询计划： 



explain(analyze)
select comment from d_holidays where holiday_date='09.09.2020';

假期的计算将在一年之内进行。检查从指定年份的1月1日到12月31日的所有日期后，选择休日并将其写在表中。为了实现该算法，我们使用过程语言plpgsql和匿名块的构造。



我们将准备一个将日期定为1月1日和12月31日的设计。 



do
$$
declare
    calc_year integer:=2020; -- 
    begin_date date; --    
    end_date date; --    
    dow integer; --  . -1,...,-6,-0
begin
    --   
    if calc_year between 1988 and 2099 then
        begin_date :=to_date('01.01.'||calc_year, 'dd.mm.yyyy');
        end_date :=to_date('31.12.'||calc_year, 'dd.mm.yyyy');
    end if;
end$$;


为了计算星期几，我们使用提取函数，该函数确定星期几（考虑到星期日为0）。为了确定星期几的名称，我们使用简单的案例构造。



我们得到以下脚本： 



do
$$
declare
    calc_year integer:=2020; -- 
    begin_date date; --    
    end_date date; --    
    dow_value integer; --  . -1,...,-6,-0
begin
    --   
    if calc_year between 1988 and 2099 then
        begin_date :=to_date('01.01.'||calc_year, 'dd.mm.yyyy');
        end_date :=to_date('31.12.'||calc_year, 'dd.mm.yyyy');
        
        while begin_date<=end_date
        loop
            raise info '%',to_char(begin_date,'dd.mm.yyyy');
            begin_date:=begin_date+interval '1 day';
            dow_value:=extract(dow from begin_date);
            if dow_value in(0,6) then
                insert into d_holidays(holiday_date,comment)
                        values(begin_date,
                            case when dow_value=6 then '' else '' end);
            end if;
        end loop;
    end if;
end$$;

放假可能会变成工作日。在这种情况下，算法如下（例如，选择了2018，因为2020年没有工作日休息）： 



delete from d_holidays where holiday_date=to_date('29.12.2018','dd.mm.yyyy'); 
delete from d_holidays where holiday_date=to_date('09.06.2018','dd.mm.yyyy');


注意设计： 



on conflict(holiday_date) do update set comment=EXCLUDED.comment;


这项创新出现在Postgresql 9.5版中，使您可以有效地处理插入记录时的冲突。在我们的示例中，如果假日已经是假日，那么为了避免在添加具有现有日期的记录时与主键发生冲突，我们将更新日期描述而不创建重复项。您可以在文档中阅读有关此语法的更多信息：
