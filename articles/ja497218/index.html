並列プログラミングの基礎


4.1。並列データ処理


仕事


データのコレクションがあります。各データ項目で同じ操作を実行する必要があります。この操作は計算上制限されており、時間がかかる場合があります。

決定


Parallelタイプには、このタスク専用に設計されたForEachメソッドが含まれています。次の例では、行列のコレクションを取得し、これらの行列を回転させます。

void RotateMatrices(IEnumerable<Matrix> matrices, float degrees)
{
   Parallel.ForEach(matrices, matrix => matrix.Rotate(degrees));
}

サイクルを途中で中止する必要がある場合があります（たとえば、無効な値が検出された場合）。次の例では、各行列を逆にしますが、無効な行列が見つかると、ループが中断されます。

void InvertMatrices(IEnumerable<Matrix> matrices)
{
   Parallel.ForEach(matrices, (matrix, state) =>
   {
      if (!matrix.IsInvertible)
        state.Stop();
      else
        matrix.Invert();
   });
}

このコードはParallelLoopState.Stopを使用してループを停止し、ループ本体への以降の呼び出しを防止します。サイクルは並列であるため、現在のサイクルに続く要素の呼び出しなど、サイクルの本体に対する他の呼び出しが既に行われている可能性があることに注意してください。上記のコード例では、3番目の行列が可逆的でない場合、サイクルが中断され、新しい行列は処理されませんが、他の行列がすでに処理されていることがわかる場合があります（たとえば、4番目と5番目）。

より一般的な状況は、並列ループをキャンセルする必要がある場合に発生します。これは、サイクルを停止することと同じではありません。サイクルは内側から停止し、外側でキャンセルされます。たとえば、次の例のように、キャンセルボタンはCancellationTokenSourceをキャンセルし、並列ループをキャンセルできます。

void RotateMatrices(IEnumerable<Matrix> matrices, float degrees,
      CancellationToken token)
{
   Parallel.ForEach(matrices,
         new ParallelOptions { CancellationToken = token },
         matrix => matrix.Rotate(degrees));
}

各並列タスクは異なるスレッドで実行できるため、すべてのジョイント状態を保護する必要があることに注意してください。次の例では、各行列を反転し、反転できなかった行列の数をカウントします。

// :     .
//      
//    .
int InvertMatrices(IEnumerable<Matrix> matrices)
{
  object mutex = new object();
  int nonInvertibleCount = 0;
  Parallel.ForEach(matrices, matrix =>
  {
     if (matrix.IsInvertible)
    {
       matrix.Invert();
    }
    else
    {
       lock (mutex)
      {
         ++nonInvertibleCount;
      }
    }
  });
  return nonInvertibleCount;
}

説明


Parallel.ForEachメソッドは、一連の値の並列処理を提供します。同様のParallel LINQ（PLINQ）ソリューションは、LINQのような構文でほぼ同じ機能を提供します。ParallelとPLINQの違いの1つは、PLINQはコンピューター上のすべてのコアを使用できると想定しているのに対し、Parallelは変化するプロセッサの状態に動的に応答できることです。

Parallel.ForEachは、並列foreachループを実装します。並列forループを実行する必要がある場合、ParallelクラスはParallel.Forメソッドもサポートします。Parallel.Forメソッドは、単一のインデックスを受け取る複数のデータ配列を操作する場合に特に役立ちます。

4.2。並列集計


仕事


並列処理の最後に結果を集計する必要があります（集計の例は、値の合計または平均の計算です）。

決定


集約をサポートするために、Parallelクラスはローカル値の概念、つまり並列ループ内にローカルに存在する変数を使用します。これは、ループ本体が同期を必要とせずに、値に直接アクセスできることを意味します。ループがすべてのローカル結果を集計する準備ができると、localFinallyデリゲートを使用してこれを行います。localFinallyデリゲートは、結果を保存するために変数へのアクセスを同期する必要がないことに注意してください。並列合計の例：

// :     .
//      
//    .
int ParallelSum(IEnumerable<int> values)
{
  object mutex = new object();
  int result = 0;
  Parallel.ForEach(source: values,
        localInit: () => 0,
        body: (item, state, localValue) => localValue + item,
        localFinally: localValue =>
       {
          lock (mutex)
             result += localValue;
       });
  return result;
}

Parallel LINQは、Parallelクラスよりも包括的な集計サポートを提供します。

int ParallelSum(IEnumerable<int> values)
{
   return values.AsParallel().Sum();
}

PLINQには多くの一般的な演算子（Sumなど）のサポートが組み込まれているため、これは安価なトリックでした。PLINQは、Aggregate演算子を使用した一般的な集計サポートも提供します。

int ParallelSum(IEnumerable<int> values)
{
  return values.AsParallel().Aggregate(
        seed: 0,
        func: (sum, item) => sum + item
  );
}

説明


Parallelクラスを既に使用している場合は、その集計サポートを使用する必要があります。他の場合では、PLINQサポートは通常より表現力があり、コードは短くなります。

4.3。並列呼び出し


仕事


並行して呼び出す必要のあるメソッドのセットがあります。これらのメソッドは（ほとんど）互いに独立しています。

決定


Parallelクラスには、このようなシナリオ用に設計された単純なInvokeメソッドが含まれています。次の例では、配列が2つに分割され、2つの半分が個別に処理されます。

void ProcessArray(double[] array)
{
   Parallel.Invoke(
         () => ProcessPartialArray(array, 0, array.Length / 2),
         () => ProcessPartialArray(array, array.Length / 2, array.Length)
   );
}

void ProcessPartialArray(double[] array, int begin, int end)
{
   // ,   ...
}

呼び出しの数が実行前に不明な場合は、Parallel.Invokeメソッドにデリゲートの配列を渡すこともできます。

void DoAction20Times(Action action)
{
   Action[] actions = Enumerable.Repeat(action, 20).ToArray();
   Parallel.Invoke(actions);
}

Parallel.Invokeは、Parallelクラスの他のメソッドと同様に、キャンセルをサポートしています。

void DoAction20Times(Action action, CancellationToken token)
{
   Action[] actions = Enumerable.Repeat(action, 20).ToArray();
   Parallel.Invoke(new ParallelOptions { CancellationToken = token },
        actions);
}

説明


Parallel.Invokeメソッドは、単純な並列呼び出しの優れたソリューションです。私は、各入力データ要素のアクションをアクティブにする必要がある状況（これにはParallel.ForEachを使用することをお勧めします）、または各アクションが出力を生成する場合（代わりに並列LINQを使用する必要があります）にはあまり適していません。
