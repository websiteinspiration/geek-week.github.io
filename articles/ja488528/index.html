AndroidプラットフォームでのPKCS＃11暗号トークンメカニズムの使用

AndroidプラットフォームでPKCS＃1 1 暗号化メカニズムを使用する時が来ました。 Androidにはハードウェアトークンがないと誰かが言うかもしれません。しかし、そうであれば、これは一時的な現象にすぎません。しかし今日では、ソフトウェアトークンを置くか、クラウドトークンを使用できます。cryptoarmpkcs-AユーティリティはTrow / Tkスクリプト言語でAndrowishを使用するAndroidプラットフォーム用に開発されているため、トークンの接続にはTclPKCS11バージョン1.0.1パッケージが使用されます。

同時に、パッケージの処理は必要ありませんでした。Cで書かれたパッケージライブラリをビルドするスクリプトがプロジェクトに追加されただけです。、Androidおよびライブラリ配布自体の場合。さまざまなプラットフォーム用のソフトウェアls11sw2016とクラウドls11cloudトークンのライブラリーを同じプロジェクトに追加することが決定されました。

残りは簡単です。作業トークンを選択するためのボタンを2番目のウィンドウに追加して、以下を取得します。

  • ソフトウェアトークンls11sw2016;
  • クラウドトークンls11cloud;
  • 別のPKCS11＃11トークン。



デフォルトでは、ソフトウェアトークンが接続されています。それが以前に作成されていない場合は、初期化することが提案されます。トークンの初期化は、ラベルの割り当てとカスタムPINコードの設定で構成されることを思い出して



ください。ソフトウェアトークンは、PKCS＃11 v.2.40およびTK-26の推奨に従って設計されていることに注意してください。サポートされている暗号化メカニズムを含むトークンに関する完全な情報を取得するには、メインメニュー（[機能]）の[トークンとその暗号化について]ボタンをクリックします。今日、ロシアの暗号化をサポートする単一のハードウェアトークンにこの機能はありません。



問題が発生します：-トークンの個人証明書をインストールする方法。 2つのオプションがあります。最初のオプションは、PKCS＃12コンテナーを介して証明書自体とそのキーペアをインポートすることです（「Work with PKCS12 / PFX」ボタン）：



2番目のオプションには、証明書のリクエスト（PKCS＃10）の作成が含まれます。このオプション（「Request for Certificate」機能）は、トークン上で直接キーペアを生成：



フラッシュドライブの要求を保存し、証明書のドキュメントとともに証明書センターにアクセスできます。証明書をテスト目的または企業内で使用する場合、CAFL63ユーティリティを使用して証明書を発行することもできます。受信した証明書は、[リクエストの表示/証明書]タブのトークンにインポートされます。



これで、[ ドキュメントに署名]タブを使用して、トークンに保存された証明書でドキュメントに署名できます。

PS保存ジャーナル


パッケージをビルドするには、AndroWish SDKのボーンユーティリティを使用します。パッケージのアセンブリが正常に完了すると、[インストールと実行]ボタンが表示されます。その後、ログウィンドウをクリックすると、adb logcat（Android Debug Bridgeロギングツール）の出力が表示されます。この画面の欠点は、特に「V」（詳細）フラグがオンになっていると、エラーメッセージを見つけるのが難しいことです。そのため、「保存」ボタンが追加され、ログをファイル/tmp/logBone.txtに保存します（ここではLinuxについて話しています）。



このボタンの登場により、アプリケーションのデバッグプロセスが大幅に簡素化されました。

ボタンを追加するには、次のコードを2591行目以降のbonesファイルに追加するだけです。

	    ttk::button $l.frame.clear -text "Clear" -width 6 \
		-command [list adb_logcat_clear $l.text]
# 
	    ttk::button $l.frame.save -text "Save" -width 6 \
		-command [list adb_logcat_save $l.text]

	    ttk::button $l.frame.run -text "Run" -width 6 \
		-command [list adb_logcat_run $l]

保存コマンドadb_logcat_saveの形式は次のとおりです。

proc adb_logcat_save {text} {
    set tt [$text get 1.0 end]
    set file "/tmp/logBone.txt"
    file delete -force $file
  #   
    set fd [open $file w]
    chan configure $fd -translation binary
    puts -nonewline $fd $tt
    close $fd
}
