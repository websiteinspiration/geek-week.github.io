マネージドKubernetesの価格比較（2020）

注意ペレフ。：アメリカのDevOpsエンジニアであるSid Palasは、Google Cloudの最近の発表を情報ガイドとして使用し、世界の主要なクラウドプロバイダーのマネージドKubernetesサービス（異なる構成）のコストを比較しました。彼の作品のもう1つの利点は、対応するJupyter Notebookが公開されたことです。これにより、（Pythonに関する最小限の知識で）ニーズに合わせて計算を調整できます。

TL、DR：AzureとDigital Oceanは、コントロールプレーンに使用されるコンピューティングリソースに課金されないため、多くの小さなクラスターをデプロイするのに適しています。GKEは、少数の大きなクラスターを実行するのに最適です。さらに、スポット/混雑/低優先度ノードを使用するか、同じノードを長期間使用するように「サブスクライブ」することで、コストを大幅に削減できます（これはすべてのプラットフォームに適用されます）。


クラスターサイズ（ワーカー数）

一般情報


GKEでの各クラスターの1時間あたり10セントの課金の開始に関する最近のGoogle Cloudの発表により、管理対象のKubernetesの領域での主なオファーの価格を分析するように促されました。


この発表はいくつかを大きく混乱させました...

記事の主な特徴は次のとおりです：

  • Google Kubernetes Engine（GCP）- 価格計算機 ;
  • Elastic Kubernetes Service（AWS）- 価格計算機 ;
  • Azure Kubernetes Service（Azure）- 価格計算ツール;
  • Kubernetes on Digital Ocean- 料金ページ。

費用の内訳


これらの各プラットフォームでKubernetesを使用する場合の総コストは、次のコンポーネントで構成されています。

  • クラスター管理料金;
  • 負荷分散（Ingressの場合）;
  • ワーカーのコンピューティングリソース（vCPUとメモリ）。
  • 下りトラフィック
  • 永久保管
  • ロードバランサーによるデータの処理。

さらに、クラウドプロバイダーは、クライアントがプリエンプティブル、スポット、または低優先度のノードを希望/使用できる場合、または同じノードを1〜3年間使用することに同意した場合、大幅な割引を提供します。

コストはサービスプロバイダーを比較および評価するための適切な基準ですが、他の要因も考慮する必要があることを強調する価値があります。

  • 稼働時間（サービスレベル契約）;
  • 周囲のクラウドエコシステム。
  • K8sの利用可能なバージョン。
  • ドキュメント/ツールキットの品質。

ただし、これらの要因はこの記事/調査の範囲を超えています。A 2月StackRoxはポストブログの詳細EX、AKS、およびGKEのための非価格要素を。

メモ帳ジュピター


最も収益性の高いソリューションの検索を簡素化するために、plotly + ipywidgetsを使用してJupyterノートブックを開発しました。これにより、さまざまなクラスターサイズとサービスセットに対するプロバイダーのオファーを比較できます。

バインダーでノートブックのライブバージョンを練習できます：mybinder.orgのmanaged-kubernetes-price-exploration.ipynb 計算または元の価格が間違っているかどうかを知らせてください（これは、GitHubのissueまたはpullリクエストを使用して行うことができます- ここにリポジトリがあります） 。

で究極のKubernetes費用ガイド：デジタルオーシャン対Azureの対GCP対AWS2018年に公開された参照クラスターは、100個のvCPUコアと400 GBのメモリで使用されました。比較のために、私の計算によれば、これらの各プラットフォーム（オンデマンドインスタンス用）の同様のクラスターには、次の金額がかかります。

  • AKS：51465 USD /年
  • EKS：43138 USD /年
  • GKE：30870 USD /年
  • DO：36131米ドル/年
