AxiosまたはFetch：2019で何を使用するか？

Axiosは有名なJavaScriptライブラリです。これは、ブラウザーおよびNode.js用のPromiseベースのHTTPクライアントです。あなたが過去数年にJavaScriptプログラマーとして働いていたなら、あなたは間違いなくこのライブラリーを使用しました。2019年10月、Axiosパッケージはnpmから2,500万回ダウンロードされました。Axiosの未来は雲一つないようです。しかし、もし私がAxiosが死んだプロジェクトだと言ったらどうでしょう。これが、Reddit に関する 1つの議論の正体です。つまり、次のことを話しているのです。

  • Axios GitHubリポジトリの開発者アクティビティは非常に低いレベルです。
  • 問題とPRは無視されます。
  • 開発チームは黙っています。

状況は、広く議論されている脆弱性によって悪化します。この脆弱性は2017年に報告されました。プロジェクトの作者は2年以上彼女を無視しました。

Axiosライブラリが普及したとき、ブラウザーにはpromiseベースのHTTPクライアントを実装するAPIがありませんでした。標準のXML HTTPリクエスト（XHR）インターフェイスは不便であり、操作が困難でした。このライブラリーによって生活が楽になったため、開発者はAxiosを喜んで受け入れました。 2015年にFetch APIがリリースされました。2019年もAxiosを使用しているのはなぜですか？これら2つのテクノロジーを比較してみましょう。





テンプレートコード量


▍フェッチ


fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(response => response.json())
  .then(json => console.log(json))
// {
//   "userId": 1,
//   "id": 1,
//   "title": "delectus aut autem",
//   "completed": false
// }

▍アクシオス


axios.get("https://jsonplaceholder.typicode.com/todos/1")
  .then(response => console.log("response", response.data))
// {
//   "userId": 1,
//   "id": 1,
//   "title": "delectus aut autem",
//   "completed": false
// }

Fetchを使用する場合、2つのプロミスに対処する必要があります。ただし、Axiosを使用するdata場合は、応答オブジェクトのプロパティでJSON結果に直接アクセスできます。Body mixin

メソッドは、応答ストリームを受け取り、それを完全に読み取ります。リクエストの本文テキストを解析したJSON結果によって解決されるpromiseを返します。 POSTリクエストを処理する場合、Fetchでさらに多くのボイラープレートコードを使用する必要があります。json()



▍フェッチ


fetch("https://jsonplaceholder.typicode.com/posts", {
  method: "POST",
  body: JSON.stringify({
    title: "Title of post",
    body: "Post Body"
  })
})
  .then(res => {
    if (!response.ok) throw Error(response.statusText);
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.log(error));

▍アクシオス


axios
  .post("https://jsonplaceholder.typicode.com/posts", {
    title: "Title of post",
    body: "Body of post"
  })
  .then(response => console.log(response.data))
  .catch(error => console.log(error));

Axiosを使用すると、大量のボイラープレートコードを記述する必要がなくなり、コードがよりクリーンで明確になります。

エラー処理


▍フェッチ


fetch("https://jsonplaceholder.typicode.com/todos/100000")
  .then(response => {
    if (!response.ok) throw Error(response.statusText);
    return response.json();
  })
  .then(data => console.log("data", data))
  .catch(error => {
    console.log("error", error);
  });
// error Error: Not Found

▍アクシオス


axios
  .get("https://jsonplaceholder.typicode.com/todos/100000")
  .then(response => {
    console.log("response", response);
  })
  .catch(error => {
    console.log("error", error);
  });
// error Error: Not Found

Axiosライブラリはネットワークエラーを報告しますが、Fetch APIは報告しません。Fetchを使用する場合は、常にプロパティを確認する必要がありますresponse.ok。この問題の解決策を簡略化するために、このエラーのチェックは別の関数として発行できます。

const checkForError = response => {
  if (!response.ok) throw Error(response.statusText);
  return response.json();
};
fetch("https://jsonplaceholder.typicode.com/todos/100000")
  .then(checkForError)
  .then(data => console.log("data", data))
  .catch(error => {
    console.log("error", error);
  });

不足している機能


Axiosはデータアップロードの進行状況を監視できます。Fetchはこれをサポートしていません。これは、ユーザーが写真やビデオファイルをサーバーにアップロードできるようにするアプリケーションを開発している人のためのテクノロジを選択する際の決定的な要素になる場合があります。

const config = {
  onUploadProgress: event => console.log(event.loaded)
};
axios.put("/api", data, config);

代替ライブラリ


AxiosとFetch APIに代わる方法をいくつか示します。

  • Kyは、小型でスマートなHTTPベースのクライアントwindow.fetchです。
  • Superagentは、に基づく小さなプログレッシブHTTPベースのクライアントライブラリですXMLHttpRequest。

概要


2019では何を使用すればよいですか？それは多くに依存します。たとえば、サーバーへの素材のアップロードの進行状況を追跡する必要がある場合は、AxiosまたはSuperagentが最適です。Fetchの制限を処理できる場合は、このAPIを使用することをお勧めします。Fetchコードを少し改善するには、Kyのようなラッパーライブラリを試してください。
