私はあなたの注意を提示します-現実的なページめくりの効果を作成するためのかなり面白いトリックを実装するための可能なオプションの1つ。Githubの



デモとドキュメント 大学とDelphiで、同じような効果を長期間実装しました。かなりの時間を費やしましたが、それは結構価値のあるものでした。さて-自己分離の時代に、JSにPCやモバイルデバイス用に同様の何かを実装することが面白くなりました。 率直に言って、この効果が実際に適用できるかどうかはわかりませんが、ほとんどの場合、アニメーションを使用せずに画像を変更するだけでよいと思います。しかし、たとえばレストランのサイトでメニューを公開するために使用することはかなり可能です（ただし、最も重要なことは、次のリンクで複製されます！）






すべてがTypescriptで書かれています。サードパーティのライブラリは使用されませんでした。依存関係はありません。

主要なライブラリ機能


  • 単純な画像、キャンバスでのレンダリング、およびhtmlブロックの両方で機能します-CSS変換を使用します
  • かなり柔軟な設定システムとシンプルなAPI
  • モバイルデバイスをサポート
  • ポートレートモードとランドスケープモード間の自動方向変更

コードはES6 +にのみ目を向けて作成されており、モジュール式システムもES6です。ブラウザのサポートはcaniuse.comに基づいて平均90％です。

取り付け


インストールはnpmから可能です：

npm install page-flip

または、リポジトリから収集したファイルをダウンロードします。

基本的なライブラリ初期化オプションは次のようになります。

<div id="book">
    <div class="my-page">
        Page one
    </div>
    <div class="my-page">
        Page two
    </div>
    <div class="my-page">
        Page three
    </div>
    <div class="my-page">
        Page four
    </div>
</div>

import {PageFlip} from 'page-flip';
const pageFlip = new PageFlip(document.getElementById('book'),
    {
        width: 400, // required parameter - base page width
        height: 600  // required parameter - base page height
    }
);

pageFlip.loadFromHTML(document.querySelectorAll('.my-page'));

計算


最初に話をするのは数学モデルです。原則として、すべての計算はかなり簡単ですが、かなりの時間がかかりました。

解決する必要がある主なタスクは、ページめくりの変換角度を決定することです。次の画像を見てみましょう。



ポイント「A」は、本の現在のタッチポイントを示します。このポイントの位置に基づいて、さらに計算を実行する必要があります。

角度を決定するには-2つの値を計算する必要があります-ポイントAから本の上部と右の境界までの距離。下の画像では、それらはそれぞれTとLで示されています。



Gは角度の対角です。ピタゴラスの定理によって計算できます。

その結果、画像の回転を計算するには、次の式を使用できます：角度=-2 * acos（L / G）、そして最も重要なのは、この場合の変換点がページの左上隅であることを忘れないでください。

角度を計算した後-最も時間がかかる部分が残ります-これはページの可視性を計算しています。残すために明らかに必要なもの、そして残りの部分-それぞれトリミングするために。

最初に、あなたは本の境界線でめくられるページの交点を見つける必要があります。図では、ポイントBとCで示されています。



これは、最も単純で最も複雑でない方法で、額で行いました。彼は線の方程式を2点で作成し、それらの交点を探しました。

すべての交点が見つかったので、可視領域の頂点を決定します。これらの点で、ページをトリミングしてフリップします。



基本的に、ここでのすべての計算は2つのことに帰着します。

  • 変換角度の計算
  • ページの可視性の計算

シェーディングはすでに以前に行われた計算に基づいています。

次に、実装中に直面しなければならなかったいくつかのポイントに移りましょう。

一般的なアルゴリズムは非常に単純で、ページめくりとクロッピングです。

キャンバスと単純な画像の場合、すべてが非常に単純です。計算を実行した後、移動、回転、クリップなどの2Dキャンバスコンテキストメソッドが使用されます。
HTMLブロックを使用すると、もう少し複雑になります。そして、回転に問題がない場合-css変換のおかげで、トリミングするとすべてが少し悪くなります。

その結果、最も簡単な方法は、Figureのclip-pathおよびcssプロパティを使用することでした-ポリゴン。ただし、トリミングのためにポリゴンの頂点を設定する前に、「グローバル」キャンバスからのポイントの座標を、html要素を基準にしてローカル座標に変換する必要があります。これは、回転行列を逆に適用して、要素の位置を基準にしてシフトすることで解決されます。

別の問題は、本の拡大縮小と自動配置でした。作成時に渡される構成オブジェクトを使用してこれを解決しようとしました。しかし、結局のところ、かなりの数のパラメーターがあり、それはあまり便利ではなく、明白ではありませんでした。

私は最初にアセンブリにWebpackを使用しましたが、最終的にはrollup.jsを試すことにし、最終的なコードに非常にうれしく思いました。Webpackは、オンザフライで数倍速くアセンブリを処理するため、まだ残っています。また、開発中の方が便利です。
