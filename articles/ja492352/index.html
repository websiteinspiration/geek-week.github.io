Gnuplot 5.0。DIY 4軸スパイダープロット


記事のデータ視覚化に取り組むとき、すべてに正のラベルが付いた4つの軸が必要になりました。


他の記事のグラフと同様に、私はgnuplotを使用することにしました。まずは例が多い公式サイトを見てみました。必要な例を見つけたときはとても嬉しかったです（ファイルを使ってあまり作業しないので、美しくなります）。

コードをすばやくコピーして実行します。エラーが発生します。わかります。私は古いgnuplot（Version 5.0 patchlevel 3 last modified 2016-02-21）を持っていることがわかりましたが、持っていません。

彼はgnuplotの柔軟性を知っていたため、ワールドワイドウェブを探索し始め、stackoverflow（Gnuplotを使用したスパイダープロットの2つのx軸とGnuplotでスパイダープロットを作成する方法？）とgithub（gnuplot-radarchart）の適切な例に出くわしました。彼らは出発点になりました。

さらに、コマンドを使用した私の操作は次のようになりました：

0）境界線をオフにします
unset border

1）4つのゼロラインを作成します-メイン2つと追加2つ：
set xzeroaxis
set yzeroaxis
set x2zeroaxis
set y2zeroaxis

ドキュメントの ゼロ行についてのいくつかの言葉。この操作により、軸を画像の中心に縮小できます。そして、それらに肯定的な目盛りを表示するために追加のものが必要です。

2）軸の目盛りの表示を構成します。
max = 1.5 #  
min = -max
set xtics  axis  0,.5,max in scale 0.5,0.25 mirror norotate  autojustify offset 0.35
set ytics  axis .5,.5,max in scale 0.5,0.25 mirror norotate  autojustify            
set x2tics axis .5,.5,max in scale 0.5,0.25 mirror norotate  autojustify            
set y2tics axis .5,.5,max in scale 0.5,0.25 mirror norotate  autojustify

設定の軸にティックをもう少し。
axis-目盛りが配置される場所、軸または（ボーダー-ボーダー）。右向きの
軸の場合。最初の数字は原点、2番目はステップ、3番目はカウントの終わりです。最初のfrom 、および残りのfromの場合、ゼロが原点で干渉しないようにします。x0,.5,max00.5

座標の中心にゼロを混ぜる。
すべての間隔は次の*ticsように構成されています0,.5,max 		せずoffset 0.35にxtics

この設定scale 0.5,0.25 mirror により、軸にセリフが追加されます。数字をいじると、サイズが変化します。
またmax, min、グラフの軸の境界を制御する変数も紹介します。
ティックの設定の詳細については、Xticsセクションのドキュメントを参照してください。

3）軸の範囲を構成します。
set xrange  [ min : max ] 
set yrange  [ min : max ]
set x2range [ max : min ]
set y2range [ max : min ]

2軸はmin増加からカウントを開始し、2軸maxは減少から開始することに注意してください。
詳細については、Xrangeセクションを参照してください。

4）軸に名前を付け、適切に配置します。
set label "H_1" at  0,       max center offset char  2, 0
set label "H_2" at  max+0.1, 0   center offset char -1, 1
set label "H_3" at  0,       min center offset char -2, 0
set label "H_4" at  min,     0   center offset char  0, 1

5）入力データを作成します
。各列には2つの列があります。行番号-軸番号、ループを閉じるための5番目の行。奇数列-座標xさえ、 - y。すべての点が軸上にあるという事実により、ペアの1つ(x, y)は常にゼロに等しくなります。
4つの軸すべてが正であるという事実にもかかわらず、一部のデータは主軸xとに配置されているため、人為的に負の半分に移動されyます。
 0  1  0     1.21
 1  0  1.21  0   
 0 -1  0    -1.06
-1  0 -1.19  0   
 0  1  0     1.21 #

完全なコード
#!/usr/bin/gnuplot -persist
#   cp1251      eps

set encoding cp1251
set terminal postscript eps enhanced monochrome size 5cm,5cm
set output "./img/eps/fig2.eps"

unset border
set key at -2, 1.5 font 'LiberationSerif, 23' 
set key left top samplen 4.5

set xzeroaxis
set yzeroaxis
set x2zeroaxis
set y2zeroaxis

max = 1.5
min = -max

set xtics  axis  0,.5,max in scale 0.5,0.25 mirror norotate  autojustify offset 0.35 font 'LiberationSerif, 20
set ytics  axis .5,.5,max in scale 0.5,0.25 mirror norotate  autojustify             font 'LiberationSerif, 20
set x2tics axis .5,.5,max in scale 0.5,0.25 mirror norotate  autojustify             font 'LiberationSerif, 20
set y2tics axis .5,.5,max in scale 0.5,0.25 mirror norotate  autojustify             font 'LiberationSerif, 20

set xrange  [ min : max ]
set yrange  [ min : max ]
set x2range [ max : min ]
set y2range [ max : min ]

set label "H_1" at  0,       max center offset char  2, 0   font 'LiberationSerif, 23'
set label "H_2" at  max+0.1, 0   center offset char -1, 1   font 'LiberationSerif, 23'
set label "H_3" at  0,       min center offset char -2, 0   font 'LiberationSerif, 23'
set label "H_4" at  min,     0   center offset char  0, 1   font 'LiberationSerif, 23'

set style line 1 linetype 1 pointtype 7 linewidth 3 linecolor black
set style line 2 linetype 2 pointtype 7 linewidth 3 linecolor black

plot 'data.csv' using  1:2 title "1" w lp ls 1 ,\
     'data.csv' using  3:4 title "2" w lp ls 2
