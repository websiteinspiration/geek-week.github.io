1から10までの乱数を選択する方法

1から10までの均一に分布した乱数、つまり1から10までの整数を生成する必要があると想像してください。しかし、コイン、コンピューター、放射性物質、または（疑似）乱数の他の同様の情報源にアクセスできない場合は、人との部屋しかない。

この部屋に8500人強の学生がいると仮定します。

最も簡単なことは、誰かに尋ねることです：「ねえ、1から10までの乱数を選んでください！」。男性は「セブン！」と答えます。いいね！今、あなたは数を持っています。ただし、均等に分散されているかどうか疑問に思われるようになります。

それで、あなたはもう数人に尋ねることに決めました。あなたは彼らに尋ね続け、彼らの答えを数え、端数を四捨五入し、1から10の範囲が0を含むと考える人を無視し続けます。結局、あなたは分布がまったく均一ではないことに気づき始めます：

library(tidyverse)

probabilities <-
  read_csv("https://git.io/fjoZ2") %>%
  count(outcome = round(pick_a_random_number_from_1_10)) %>%
  filter(!is.na(outcome),
         outcome != 0) %>%
  mutate(p = n / sum(n))

「人間のRNG」の分布を均一な分布に変換する関数が見つかればいいのに...

ここの直感は比較的単純です。分布質量を10％を超える場所から取得し、10％未満の場所に移動するだけです。チャートのすべての列が同じレベルになるように：



理論的には、そのような関数が存在する必要があります。実際、（順列のために）多くの異なる関数がなければなりません。極端なケースでは、各列を無限に小さなブロックに「カット」して、任意の形状の分布（レゴブロックなど）を構築できます。

もちろん、そのような極端な例は少し面倒です。理想的には、初期分布をできる限り維持することをお勧めします（つまり、断片化と移動をできるだけ少なくします）。

そのような関数を見つける方法は？


さて、上記の説明は線形計画法によく似ています。ウィキペディアから：

線形計画法（LP、線形最適化とも呼ばれます）は、線形関係によって要件が表される数学的モデルで最高の結果を達成する方法です...標準形式は、線形計画問題を記述する通常の最も直感的な形式です。3つの部分で構成されています。

  • 最大化される線形関数
  • 次のフォームの問題の制限
  • 非負の変数

同様に、再配布の問題を定式化できます。

問題の提示


変数のセットがあります (xi,j、それぞれが整数から再分布される確率の一部をエンコードします i （1から10）を整数に j（1から10まで）。したがって、(x7,1=0.2の場合、回答の20％を7から1に転送する必要があります。

variables <-
  crossing(from = probabilities$outcome,
           to   = probabilities$outcome) %>%
  mutate(name = glue::glue("x({from},{to})"),
         ix = row_number())

これらの変数を制限して、すべての再分布確率の合計が10％になるようにします。つまり、みんなのためにj 1から10：

x1,j+x2,j+… x10,j=0.1


これらの制限をRの配列のリストとして表すことができます。後で、それらを行列にバインドします。

fill_array <- function(indices,
                       weights,
                       dimensions = c(1, max(variables$ix))) {
  init <- array(0, dim = dimensions)

  if (length(weights) == 1) {
    weights <- rep_len(1, length(indices))
  }

  reduce2(indices, weights, function(a, i, v) {
    a[1, i] <- v
    a
  }, .init = init)
}

constrain_uniform_output <-
  probabilities %>%
  pmap(function(outcome, p, ...) {
    x <-
      variables %>%
      filter(to == outcome) %>%
      left_join(probabilities, by = c("from" = "outcome"))

    fill_array(x$ix, x$p)
  })

また、初期分布からの確率の質量全体が保持されることを確認する必要もあります。だからみんなのためにj 1から10の範囲：

x1,j+x2,j+… x10,j=0.1


one_hot <- partial(fill_array, weights = 1)

constrain_original_conserved <-
  probabilities %>%
  pmap(function(outcome, p, ...) {
    variables %>%
      filter(from == outcome) %>%
      pull(ix) %>%
      one_hot()
  })

すでに述べたように、元のディストリビューションの保存を最大化したいと考えています。これが私たちの目的です：

maximise(x1,1+x2,2+… x10,10)


maximise_original_distribution_reuse <-
  probabilities %>%
  pmap(function(outcome, p, ...) {
    variables %>%
      filter(from == outcome,
             to == outcome) %>%
      pull(ix) %>%
      one_hot()
  })

objective <- do.call(rbind, maximise_original_distribution_reuse) %>% colSums()


次に、問題をソルバー、たとえばlpSolveR のパッケージに転送し、作成した制約を1つの行列に結合します。

# Make results reproducible...
set.seed(23756434)

solved <- lpSolve::lp(
  direction    = "max",
  objective.in = objective,
  const.mat    = do.call(rbind, c(constrain_original_conserved, constrain_uniform_output)),
  const.dir    = c(rep_len("==", length(constrain_original_conserved)),
                   rep_len("==", length(constrain_uniform_output))),
  const.rhs    = c(rep_len(1, length(constrain_original_conserved)),
                   rep_len(1 / nrow(probabilities), length(constrain_uniform_output)))
)

balanced_probabilities <-
  variables %>%
  mutate(p = solved$solution) %>%
  left_join(probabilities,
            by = c("from" = "outcome"),
            suffix = c("_redistributed", "_original"))

次の再配布が返されます。

このグラフは、誰かが8を乱数として呼び出す場合の約8％の場合、答えを1つの単位として受け取る必要があることを示しています。残りの92％のケースでは、8人のままです。

均等に分布した乱数（0から1）のジェネレーターにアクセスできれば、問題を解決するのは非常に簡単です。しかし、私たちの部屋は人でいっぱいです。幸いなことに、いくつかの小さな不正確な点で妥協する準備ができている場合は、2回以上尋ねることなく、人からかなり良いRNGを作成できます。

元の分布に戻ると、数値ごとに次の確率があり、必要に応じて確率を再割り当てするために使用できます。

たとえば、誰かが乱数として8を与えた場合、この8がユニットになるかどうか（確率8％）を決定する必要があります。乱数を他の人に尋ねると、8.5％の確率で「2」と答えます。したがって、この2番目の数値が2の場合、一様に分布した乱数として1を返す必要があることがわかります。

このロジックをすべての数値に拡張すると、次のアルゴリズムが得られます。

  • 人に乱数を尋ね、 n1。
  • n1=1,2,3,4,6,9, または 10：
    • あなたの乱数 n1
  • もし n1=5：
    • 別の人に乱数を尋ねます（n2）
    • もし n2=5 （12.2％）：
      • あなたの乱数2
    • もし n2=10 （1.9％）：
      • あなたの乱数4
    • それ以外の場合、乱数は5です。
  • もし n1=7：
    • 別の人に乱数を尋ねます（n2）
    • もし n2=2 または 5 （20.7％）：
      • あなたの乱数1
    • もし n2=8 または 9 （16.2％）：
      • あなたの乱数は9です
    • もし n2=7 （28.1％）：
      • あなたの乱数は10です
    • それ以外の場合、乱数は7です。
  • もし n1=8：
    • 別の人に乱数を尋ねます（n2）
    • もし n2=2 （8.5％）：
      • あなたの乱数1
    • それ以外の場合、乱数は8です。
このアルゴリズムを使用すると、人々のグループを使用して、1から10までの均等に分散された乱数のジェネレーターに近いものを取得できます。
