畳み込みニューラルネットワークを使用した感情認識


感情を認識することは、科学者にとって常に刺激的な課題でした。最近、私はこのテクノロジーの可能性を理解するために、実験的なSERプロジェクト（音声感情認識）に取り組んでいます。このため、Githubで最も人気のあるリポジトリを選択し、それらを私のプロジェクトのベースにしました。

プロジェクトの理解を始める前に、SERがどのようなボトルネックになっているのかを思い出しておくとよいでしょう。

主な障害


  • 感情は主観的であり、人々さえそれを異なって解釈する。「感情」の概念そのものを定義することは困難です。
  • オーディオへのコメントは難しいです。どうにかして、すべての単語、文、またはコミュニケーション全体を全体としてマークする必要がありますか？認識に使用する一連の感情は何ですか？
  • データの収集も簡単ではありません。映画やニュースから多くの音声データを収集できます。ただし、ニュースは中立である必要があり、俳優の感情が再生されるため、どちらのソースも「偏見」があります。オーディオデータの「客観的な」ソースを見つけることは困難です。
  • マークアップデータには、多くの人的および時間的なリソースが必要です。画像にフレームを描画するのとは異なり、オーディオの録音全体を聞いて分析し、コメントを提供するには、特別な訓練を受けた担当者が必要です。そして、評価は主観的であるため、これらのコメントは他の多くの人々に評価されなければなりません。


プロジェクトの説明


畳み込みニューラルネットワークを使用して、オーディオ録音の感情を認識します。そして、はい、リポジトリの所有者はソースを参照していませんでした。

データの説明


RAVDESSおよびSAVEEリポジトリで使用された2つのデータセットがあり、モデルでRAVDESSを適合させました。RAVDESSコンテキストには、音声と歌の2種類のデータがあります。

データセットRAVDESS（感情的なスピーチと歌のライアソン視聴覚データベース）：

  • 12人の俳優と12人の女優が演説で彼らのスピーチと歌を録音しました。
  • 俳優＃18には録音された曲はありません。
  • 「歌」のデータには、嫌悪感（嫌悪感）、中立感（中立感）、驚き（驚き）の感情はありません。

def noise(data):
   """
     
   """
   #     : https://docs.scipy.org/doc/numpy-1.13.0/reference/routines.random.html
   noise_amp = 0.005*np.random.uniform()*np.amax(data)
   data = data.astype('float64') + noise_amp * np.random.normal(size=data.shape[0])
   return data
