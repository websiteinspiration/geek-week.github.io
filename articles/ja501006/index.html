この記事では、I2Cバス上のHALライブラリを使用してLCDディスプレイをSTM32マイクロコントローラーに接続した経験についてお話します。



ディスプレイ1602と2004を接続します。どちらにも、PCF8574Tチップをベースにしたはんだ付けされたI2Cアダプターがあります。デバッグボードはNucleo767ZIで、開発環境はSTM32CubeIDE 1.3.0です。

I2Cバスの動作原理については詳しく説明しません。こことここをご覧になることをお勧めします。

アダプターはRXをTXに、TXをRXに接続します。アダプターのジャンパーは3.3Vです。



PCF8574Tチップとディスプレイでの作業を詳しく見てみましょう。以下は、ディスプレイを備えたモジュールの概略図です

マイクロ回路のどのピンがディスプレイに接続されているか、およびそれらが何を担当しているのかを検討します。

  • マイクロサーキットの端子P0は、表示データ（1）または表示操作命令（0）の受信を担当するディスプレイのRS端子に接続されています。
  • ピンP1はR \ Wに接続されています。1-ディスプレイにデータを書き込み、0-読み取り。
  • 端子P2はCSに接続されています-状態の変化が読み取られている端子です。
  • 結論P3-バックライト制御。
  • 結論P4-P7は、データをディスプレイに送信するために使用されます。

複数のデバイスを1つのI2Cバスに同時に接続できます。特定のデバイスにアクセスできるようにするために、それらのそれぞれに独自のアドレスがあります。最初に確認します。アダプターボードの接点A1、A2、A3がシールされていない場合、アドレスはおそらく0x27ですが、確認することをお勧めします。これを行うには、I2Cバスに接続されているすべてのデバイスのアドレスを表示する小さな関数を記述します。

void I2C_Scan ()
{
	//  ,  
        HAL_StatusTypeDef res;                          
	//    
        char info[] = "Scanning I2C bus...\r\n";      
        //    UART	
        HAL_UART_Transmit(&huart5, (uint8_t*)info, strlen(info), HAL_MAX_DELAY);  
	/* &huart5 -   UART
	 * (uint8_t*)info -     
	 * strlen(info) -   
	 * HAL_MAX_DELAY - 
	 */
        //    
	for(uint16_t i = 0; i < 128; i++)              
	{
            // ,      i  
            res = HAL_I2C_IsDeviceReady(&hi2c1, i << 1, 1, HAL_MAX_DELAY);                   
	    //  , 
            if(res == HAL_OK)                              
	    {
	    	char msg[64];
	    	//   i,   ,     
                // 16 :
	    	snprintf(msg, sizeof(msg), "0x%02X", i);
	    	//    
	    	HAL_UART_Transmit(&huart5, (uint8_t*)msg, strlen(msg), HAL_MAX_DELAY);
	    	//    
	    	HAL_UART_Transmit(&huart5, (uint8_t*)"\r\n", 2, HAL_MAX_DELAY);
	    }
	    else HAL_UART_Transmit(&huart5, (uint8_t*)".", 1, HAL_MAX_DELAY);
	}
	HAL_UART_Transmit(&huart5, (uint8_t*)"\r\n", 2, HAL_MAX_DELAY);
}

この関数は、0から127までのすべてのアドレスをポーリングし、このアドレスから応答を受信した場合、アドレス番号を16進形式でUARTに送信します。

理事会とのコミュニケーションには、シロアリプログラムを使用しています。デフォルトでは、マイクロコントローラーのUART速度は115200に設定されています。シロアリでも同じに設定する必要があります。プログラムの本体で関数を呼び出し、ボードをフラッシュし、シロアリをマイクロコントローラーに接続します。



ドットは、応答が受信されなかったすべてのアドレスを示します。ジャンパーA0をはんだ付けしたので、ディスプレイのアドレスは0x26です。次に、2番目のディスプレイを最初のディスプレイと平行に接続し、プログラムが何を与えるかを確認します。



0x26（ディスプレイ1602）と0x27（ディスプレイ2004）の2つのアドレスがあります。次に、ディスプレイの操作方法について説明します。マイクロコントローラがアドレスバイトを送信し、バスに接続されているすべてのデバイスが独自のアドレスバイトでそれをチェックします。一致した場合、モジュールはマイクロコントローラーとの通信を開始します。まず、表示を構成する必要があります。文字カウントの開始位置と方向、カーソルの動作などです。その後、すでに表示用の情報をディスプレイに送信することができます。特徴は、情報の送信に4ビットしか使用できないことです。データは2つの部分に分割する必要があります。データは上位ビット（4〜7）に格納され、下位ビットは、バックライトがオンになるか（3ビット）、データが出力されるか、または表示設定（RS出力、0ビット）、および2ビットを示すために使用されます。読み取りが行われる変化によって、すなわちe 1バイトのデータを送信するには、4バイトを送信する必要があります-最初のバイトには4ビットの情報が含まれ、2番目のビットは状態1、2番目のバイトは1番目の繰り返し、2番目のビットのみが状態0になります。3番目と4番目のバイトは類似していますが、それらのみが含まれますデータの後半。少しわかりづらいようですが、例を示します。

void I2C_send(uint8_t data, uint8_t flags)
{
	HAL_StatusTypeDef res;
	 //     
        for(;;) {
                // ,      lcd_addr                                                                                        
	        res = HAL_I2C_IsDeviceReady(&hi2c1, LCD_ADDR, 1, HAL_MAX_DELAY);               
	         //  ,     
                if(res == HAL_OK) break;                                                  
	    }
        //    1111 0000      0  3,    4  7
	uint8_t up = data & 0xF0;   
        //   ,  data   4                	
        uint8_t lo = (data << 4) & 0xF0;          
                                          
	uint8_t data_arr[4];
         // 4-7   ,  0-3   
	data_arr[0] = up|flags|BACKLIGHT|PIN_EN; 
         //  ,       0  
	data_arr[1] = up|flags|BACKLIGHT;         
	data_arr[2] = lo|flags|BACKLIGHT|PIN_EN;
	data_arr[3] = lo|flags|BACKLIGHT;

	HAL_I2C_Master_Transmit(&hi2c1, LCD_ADDR, data_arr, sizeof(data_arr), HAL_MAX_DELAY);
	HAL_Delay(LCD_DELAY_MS);
}

順番に見ていきましょう。最初に、表示アドレスを格納する変数と、データとともに毎回送信する必要がある設定のビットがあります。送信機能では、まず、記録されたアドレスにモジュールがあるかどうかを確認します。 HAL_OKメッセージを受信した場合、送信用のバイトを形成し始めます。送信するバイトの先頭で、2つの部分に分割し、両方を上位ビットに書き込む必要があります。ディスプレイに文字「s」を表示させたいとすると、バイナリシステムでは1110011（計算機）論理演算を使用して＆0 = 01110000を変数に書き込みます。最上位ビットのみを書き込みます。下位ビットは最初に4文字左にシフトされ、変数lo = 00110000に書き込まれます。次に、表示される文字に関する情報を含む4バイトの配列を形成します。次に、構成ビット（0〜3ビット）を既存のバイトに割り当てます。その後、HAL_I2C_Master_Transmit（）関数を使用して、アドレスバイトと4バイトの情報をディスプレイに送信します。

ただし、最初は表示設定を行う必要があるため、急いでプログラムをダウンロードしないでください。このサイトには、表示をカスタマイズするためのコマンドを含む優れた翻訳済みの表があります。ドキュメントで確認したところ、自分に最適な次の設定になりました。

  I2C_send(0b00110000,0);   // 8  
  I2C_send(0b00000010,0);   //     
  I2C_send(0b00001100,0);   //   ,  
  I2C_send(0b00000001,0);   //

これらのコマンドは、無限ループの開始前に配置されるため、設定は作業を開始する前に1回送信されます（arduinkiのvoidセットアップのように）。I2C_send関数では、バイトに加えて、表示設定またはデータのどちらを送信するかを指定する必要があります。関数の2番目の引数が0の場合は設定、1の場合はデータ。

そして最後のタッチ-文字ごとに期限を送信する関数が必要です。ここではすべてが非常に単純です。

void LCD_SendString(char *str)
{
    // *char    
    //    
	while(*str) 
        {                                   
                //    
		I2C_send((uint8_t)(*str), 1); 
                //     1               
                str++;                                     
        }
}

これらの関数をすべてまとめたら、次のように書くことができます。

  LCD_SendString("  Hello");
  I2C_send(0b11000000,0);   //  
  LCD_SendString("    Habr");
