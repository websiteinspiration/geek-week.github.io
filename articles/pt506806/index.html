MVP para Android - os benefícios de usar o Moxy como uma biblioteca auxiliar

Este artigo descreve os benefícios do uso do Moxy como uma biblioteca auxiliar ao usar o MVP para um aplicativo Android.


Importante: o MVP não é comparado com outras abordagens arquiteturais, como MVVM, MVI, etc. O artigo descreve por que, se o MVP é escolhido como a arquitetura da parte da interface do usuário do aplicativo, é melhor não usar a implementação nativa do MVP, mas usar o Moxy.


A biblioteca Moxy permite evitar o código padrão para manipular fragmentos e atividades do ciclo de vida e trabalhar com o View como se estivesse sempre ativo.
Além disso, o View é entendido como a implementação do View como um fragmento ou atividade.
O interator é entendido como a essência da lógica de negócios, ou seja, uma classe que está em um nível de abstração mais baixo que o Presenter.


Benefícios comuns de Moxy


  • Suporte ativo e desenvolvimento da biblioteca.
  • O suporte ao recurso Kotlin, como val presenter, por moxyPresenter {component.myPresenter} e presenterScope para corotina.
  • Restaurar automaticamente o estado de exibição.
  • Ligação automática com o ciclo de vida (e, portanto, a ausência de vazamentos da Activiti e outros encantos semelhantes).
  • As visualizações são acessadas através de um viewState não anulável. Não há risco de perda de qualquer comando de exibição.
