みんな読書を楽しむ

フラクタルは美しいです。それらは非常に複雑なパターンに従って整列し、いかなる倍率でも歪みなく保存されます！この記事では、L-Systemsと呼ばれるツールとPython用のTurtleモジュールを使用して段階的な描画を実行することで、数種類のフラクタルを簡単に描画する方法について説明します。

この記事では、技術的な詳細についてはあまり詳しく説明しません。代わりに、簡単な紹介に制限し、アニメーションの例と、そのような例を生成できるコードを示します。理論をスキップしてアニメーションだけを見たい場合は、例に直接進んでください。さらに、最後に、コードの記述と基本的な数学の両方で、探索したいリソースがいくつかあります。

フラクタルとは？

はじめに、フラクタルの「緩い」定義を与えましょう。原則として、フラクタルは、増加の度合いに関係なく同じ特性を示す幾何学的図形です。

この定義は完全ではないため、Math World Webサイトからのより正確な定義を次に示します。
フラクタルは、任意のスケールで（形式的な意味で）自己相似性を示すオブジェクトまたは量です。オブジェクトは異なるスケールで同一の構造を示しませんが、同じ「タイプ」の構造はフラクタルのすべてのレベルで出現する必要があります。この場合、グラフは対数スケールの座標系でプロットされます。ここで、マグニチュードとスケールは軸に沿ってカウントされ、グラフは、フラクタルの次元を反映する勾配を持つ直線になります。- 数学の世界

Pythonを使用してフラクタルを描画する方法は？

フラクタルの深い性質は再帰の概念によって決定されるため、通常、フラクタルのレンダリングは複雑です。グラフとその描画について言えば、通常、それらはピクセルまたはベクトルによって形成されると考えられますが、ピクセルまたはベクトルの数は常に制限されており、フラクタルは、定義により、無限に再帰的です。したがって、フラクタルを座標グリッドに適用しようとすると、ある時点で停止する必要があるため、この場合は「反復」について話します。各反復でフラクタルはより複雑になり、ある時点で、その2つの反復を互いに区別することができなくなります（このような瞬間は、ピクセルのサイズに匹敵するレベルで変化が発生したときに発生します）。ここで停止することは論理的ですが、原則として、フラクタルの形状が速く現れ、さらに早く停止することができます。

そのような2つの例は、3回の反復後に構造が明らかに現れるコッホの正方形の島と、8回の反復で完全な構造を構築するのに十分なカーター・ヘイトウェイドラゴンです。必要な反復回数は、使用している特定のフラクタルに大きく依存します。

もちろん、プロット用のPythonライブラリは多数ありますが、その中で最も人気のあるのはMatplotlibですが、通常、これらは、統計の描画とよく知られたグラフの描画用に設計されています。特に、Matplotlibにはフラクタルを構築できる低レベルの構成がいくつか含まれていますが、今回はTurtleと呼ばれる標準ライブラリの不当に知られていないモジュールに焦点を当てます。

タートルモジュール

でPythonドキュメント「カメのグラフィックスは、プログラミングをしている子供たちの最初の知り合いにとって人気のあるツールです。彼は1966年にWallyFörzegとSeymour Papertによって開発されたオリジナルのロゴプログラミング言語の一部でした。

つまり、タートルはデフォルトで3つのコマンドを認識します。

  • クロールフォワード
  • 左角度に回転
  • 直角に回転

注：他のコマンドは標準ライブラリで提供されていますが、ここではこれら3つだけを使用します。

また、次のこともできます。

  • レコードをミュート
  • 記録を有効にする

これらの特性は単純すぎて、フラクタルのような複雑なグラフに描画することはできず、それらだけに依存していますが、この小さな命令セットのみを使用する別のツールを使用します。私はLシステムについて話している。

Lシステム

Lシステムは、再帰構造（フラクタルなど）を文字列として表現し、そのような文字列を複数回書き換える方法です。ここでも、正式な定義を示します。
リンデンマイヤーシステム（Lシステムとも呼ばれます）は、1から2の次元を持つフラクタルを生成するために使用できるライン書き換えメカニズムです-Math World

Lシステムとは何かを理解したら、再帰的な構造を作成できますが、最初に、これに必要なコンポーネントを理解しましょう。各Lシステムには以下が含まれます。

  • : , L-.
  • : .
  • : , .

コンピュータサイエンスのファンへの注意：コンピュータサイエンスを深く学んだ場合、上記のすべてが何かを思い出させるかもしれません。実際、正式な文法は非常によく定義されています。重要な違いは、文法とは異なり、ここでは各反復で1つだけではなく、できるだけ多くのルールが適用されることです。したがって、Lシステムは文脈自由文法のサブセットです。

Turtleを使用してグラフを作成し、Lシステムを使用して何をプロットするかを表すとすると、それらの間の関係を作成する必要があります。

Turtleには上記のチームしかないため、それぞれにシンボルを割り当てます。アルファベットはこれらの文字で構成されます。

  • F：クロールフォワード
  • +：右折
  • -： 左折してください

これを機能させるには、フラクタルごとに角度を指定する必要があります。これは、亀が右または左に曲がる角度になります。簡単にするために、1つのコーナーのみを提供することに同意し、これを念頭に置いてLシステムを作成します。
文字列を作成するための公理と指示はフラクタルにのみ依存しますが、フラクタルはこれらの3つの文字だけで表現できるように記述する必要があります。したがって、制限があるため、1行のフラクタルしか作成できません。つまり、この方法ではCantorセットのようなものは取得できません。しかし、これは単なる簡略化です。なぜなら、記録せずに前進するための他の2つのコマンドと、後進するための同じコマンドを常に入力できるからです。

それでは、例を見てみましょう！

コード

上記のすべての例は、同じコードを使用して取得されたものであり、それらに取り組むときに、色、反転、オフセットを操作するだけでなく、ベクトル形式。ここでは、最も単純なバージョンのみを示します。
このバージョンは、フラクタルを白黒で表示し、エクスポート機能を備えていません

import turtle

def create_l_system(iters, axiom, rules):
    start_string = axiom
    if iters == 0:
        return axiom
    end_string = ""
    for _ in range(iters):
        end_string = "".join(rules[i] if i in rules else i for i in start_string)
        start_string = end_string

    return end_string


def draw_l_system(t, instructions, angle, distance):
    for cmd in instructions:
        if cmd == 'F':
            t.forward(distance)
        elif cmd == '+':
            t.right(angle)
        elif cmd == '-':
            t.left(angle)


def main(iterations, axiom, rules, angle, length=8, size=2, y_offset=0,
        x_offset=0, offset_angle=0, width=450, height=450):

    inst = create_l_system(iterations, axiom, rules)

    t = turtle.Turtle()
    wn = turtle.Screen()
    wn.setup(width, height)

    t.up()
    t.backward(-x_offset)
    t.left(90)
    t.backward(-y_offset)
    t.left(offset_angle)
    t.down()
    t.speed(0)
    t.pensize(size)
    draw_l_system(t, inst, angle, length)
    t.hideturtle()

    wn.exitonclick()

次に、L-システムを生成する必要があります。これは、亀のための一連の指示になります。create_l_system反復数、公理、および構築ルールを受け取る関数を定義します。これは公理から始まり、補助変数を使用しますend_string。反復が0の場合、一部のフラクタルはゼロ反復でも適用できるため、公理を返します。この場合、ルールは辞書の形式であると想定されているため、各キーは一意であり、記号を表し、値は何を置き換えなければならないか、何を置き換える必要があるかを示します。したがって、各文字のすべての置換を組み合わせ、最終的に次の反復のための文字列を取得します。

def draw_l_system(t, instructions, angle, distance):
    for cmd in instructions:
        if cmd == 'F':
            t.forward(distance)
        elif cmd == '+':
            t.right(angle)
        elif cmd == '-':
            t.left(angle)

次にdraw_l_system、タートルを受け入れるもの、一連の指示（Lシステムの出力）、左または右に曲がる角度、および個々の線の長さを決定します。これはelif、以前に定義された各チームの単純な構造で構成されています。

機能について最後に、レッツ・トークのmainL-システムの生成に必要なすべてのパラメータを取り、など、y_offset、x_offset、offset_angle、widthとheight。最初の3つはカメの変位を表します。必要な方法でキャンバス上にグラフを配置するだけです。

この関数は、最初に一連の指示を生成し、それらをinstに保存します。次に、タートルと画面を初期化し、タートルを特定のポイントに配置し、指示に従ってグラフを描画し、クリックが閉じるのを待ちます。

特別な考慮事項

上で述べたように、多くの制限がここに残されています。まず、タートルにレンダリングせずに移動する機能を提供しませんでした。これには別の文字が必要になります。また、後退したり、以前のポジションを思い出したりするためのシンボルもありません。これらは、上記のすべてのフラクタルに必要なわけではありませんが、他のフラクタル（たとえば、フラクタルツリー）には必要です。
