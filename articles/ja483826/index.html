Voアナログ出力を使用したCO2センサーモデルMH-Z19Bの接続

室内の空気の質に興味があり、それを制御および管理したいすべての人を歓迎します。

私も最近この問題に興味を持ち、CO2センサーMH-Z19 Bを自分の手で感じることにしました。MH-Z19のダークサイドなど、多くの記事に触発されました。



ただし、どこにも（手順でも）Voピンを使用する方法が見つかりませんでした...

私はこれに注意を引きました：記事赤外線CO2センサーMH-Z19の概要で、この点が言及されています：

  • Vo-出力電圧3.3V、最大10mA

「二酸化炭素センサーCO2 MH-Z19B」などの一部のオンラインストアでは、同様の情報を提供しています。

  • Vo-出力電圧3.3V、最大10mA

（カーボンコピーとして）

ただし、指示（ここ）はもう少し示しています。

  • Vo アナログ出力（0.4〜2 V）または（0〜2.5V）

さらに掘り下げる場合は、指示（ここ）でさらに詳しく説明します。

  • Vo（Pin2）- アナログ出力（0.4〜2 V）（0〜3V 範囲はカスタマイズ可能）

私はすぐにググって…この方法について何も見つけられなかった！Stirlitz氏は、奇妙なことに、この出力をESP8266のアナログ入力（Arduinoの場合はこれにも関連します）にすばやく接続し、研究を開始しました。

調査結果：

  1. アナログ出力の電圧は、0.4xxから2.002vまで変動します。
  2. アナログ出力は、デジタルppm信号の電圧を繰り返します。
  3. UARTコマンドがppm 2k、5k、10kの範囲を変更すると、それに応じてVo変換範囲が変更されます。
  4. Vo = 2.001vの最大値は、最大値をppm（2/5 / 10k）で設定する必要がある変換範囲であるため、範囲によって変化しません。
  5. Voの最小値はレンジ（？）によって異なり、レンジ自体は400ppmから始まります。
  6. ADCコントローラーの実際の値を選択すると、電圧とppmレベルの間で適切な一致を得ることができます。2kまでの範囲では、デジタルUART信号とアナログVo変換の間の誤差は数ppm単位を超えません。
  7. まずUSB-TTLコンバーターを使用してセンサーを構成するか、Arduino +（自動キャリブレーションを無効にして400〜2000ppmを「ホーム」範囲に切り替える）を使用してセンサーを構成してから、アナログ信号のように操作します。

変換方法：

[...]
const word cADC00v   =   3;  // ADC = 3   .  0.0v
const word cADC04v2k = 235;  // ADC = 235 .  400ppm @2k
const word cADC04v5k = 174;  // ADC = 174 .  400ppm @5k
const word cADC04v1k = 153;  // ADC = 153 .  400ppm @10k
const word cADC20v   = 646;  // ADC = 646 6.  2.0v  

word ADC = analogRead(A0);            //  10  ADC 0...1023

long  Vin = map(ADC, cADC00v, cADC20v, 0, 2001);     //  0..3,3v  ,   2,001v


long ppm_2k_04_2  = map(ADC, cADC04v2k,cADC20v, 400, 2000); //  Vo  ppm   2k
long ppm_5k_04_2  = map(ADC, cADC04v5k,cADC20v, 400, 5000); //  Vo  ppm   5k
long ppm_10k_04_2 = map(ADC, cADC04v1k,cADC20v, 400,10000); //  Vo  ppm   10k
[...]

「魔法」の数は実験的に取り上げられました。それはより速くそしてより正確に起こった。数学的に非常に正確に計算しようとすると、実際に大きなエラーが発生しました。

なぜ機能するのですか（0.4 ... 2.0vで、0 ... 2.5vではありません）わかりません。センサーは数日前に購入され、生産は2019年9月26日です。 5k ppmまでの範囲がケースにエンボス加工されています。

なぜ必要なのですか？ （「私は間違っていたので...」への回答）

まあ...はんだごてを手に持つのが好きな人がいますが、彼らはコントローラーと友達ではありません。彼らにとって、アナログ出力は少なくともファンをオン/オフにし、最大で-排気/吸気/復熱器の速度のスムーズな制御です。

具体的には、私にとっては、隣の部屋で2年間換気システムが機能しており、自由にプログラム可能な産業用コントローラーで作成しました。なぜならコントローラは産業用であり、プログラミングシステムは特定のものです。独自のUART交換プロトコルを記述できるかどうかはわかりません。はい、ModBusで正常に動作するRS484シリアルポートがありますが、このポートはディスパッチシステムによって占有されています。

ただし、0〜10Vに設定できる無料のアナログ入力があります。そのため、約30ドルしかないため、インテリジェントな省エネのために換気システムをアップグレードできます（必要なときにのみ換気し、必要な分だけ換気します）。
