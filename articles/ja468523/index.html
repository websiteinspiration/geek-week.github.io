ロケットからロボットへ、そしてPythonはそれと何をしなければならないか。GeekBrains Alumni Story


今日は、Andrey VukolovによるITへの移行の歴史を公開しています。宇宙への子供の頃の熱狂はかつて彼をMSTUでロケット科学に導いた。厳しい現実は夢を忘れさせましたが、すべてがさらに面白くなりました。C ++とPythonを研究することで、ロボット制御システムのロジックをプログラミングすることもできます。

開始


私は子供の頃から宇宙について絶賛していました。そのため、放課後、どこへ行ったらいいのか迷うことなく、モスクワ州立工科大学に入学しました。バウマン、ロケット推進部へ。ただし、コース自体のブランチ（宇宙ロケットの粉末または液体エンジン）を選択する必要はまったくありませんでした。2001年に、特別教員委員会が応募者のターゲットグループを配布しました。 「火薬の樽の中に入った」。

当時、「ロケットブーム」は計画にのみ存在し、エンジニアはわずかな給与を受け取り、特別な閉鎖型設計局や研究機関で働いていましたが、実際にはキャリアと専門家の成長の見通しはありませんでした。それでも、ロシアのパウダーロケットは純粋に軍事製品です。

現在、この方向性は求められていますが、すでに私の研究中に、ロケット科学では、自分の主導による活動はほとんど不可能であることに気付きました。実際、これは兵役です。たとえば、ロケット業界で働いている場合、この活動は厳しく規制されているため、私自身でもソフトウェアを独自に開発する機会を完全に奪われます。

すべてのソフトウェア製品は、特別注文およびプライバシー委員会（現在はFSTECの一部門）の承認を得て、独占的に開発されています。そこの開発者は、文字通りすべてのコード行を登録してライセンスを取得する必要があります。すべてのソフトウェアは、最初はタスクレベルで秘密です。これは、ロケットの学生を訓練するために現在使用されているソフトウェアが、1990年代に最大で開発された理由を部分的に説明しています。

私が卒業するまでに、メカニズム理論学科で働くことができ、C ++で技術プロセスのトレーニングシミュレータの開発を始めたので、比較のための例があり、すべての長所と短所を比較検討することができました。選択は明白でした、そして、私は徐々にITとロボット工学に向かってドリフトし始めました。応用力学では、ロケット科学よりもはるかに楽しいものでした。未解決の問題が多く、オープンな環境があり、開発業界が存在せず、シミュレーションソフトウェアの緊急の必要性がありました。ロボット工学では、一般的なソフトウェアの不安定なアーキテクチャと、ファジーロジックやAIの基本を含む複雑なアルゴリズムを繰り返し実装する必要性。そのため、実験データを処理するための最初のプログラムの後、（卒業プロジェクトを除いて）ミサイルに戻ることはほとんどありませんでした。

その結果、モスクワ近郊の航空宇宙産業向けの複合構造プラントでの卒業証書を取得する前に、わずか4か月間専門分野で働く機会がありました。トレーニングの最後には、仕事を探す必要すらありませんでした。すぐにロボット工学部で応用力学を教えるようになりました。

教育からプログラミングまで



IFTOMM世界会議で学生-研究グループのメンバー（私は右側）

と一緒に、MSTUのプロボテクニカルファカルティで10年間働き、メカニズム理論のコースを教えました。彼は科学論文（記事の最後を参照）を発表し、徐々に力学からCADおよびロボット工学へと移行しました。そして最後に彼は教えるのをやめることにしました。この決定の理由を最も鮮明に説明するために、私が数十年にわたって、私が読んだ研究のコースはコンマによって変わっていないと言います。出版物から判断すると、応用力学は非常に成功しました。

さらに、仕事はますます官僚に似ていました-レポート、プログラム、標準、そして大量の紙。そのような状況では、教える喜びはこの喜びの受け取りに関する報告に取って代わられ、これは開業スペシャリストにとっては不愉快なことではありません。

そして、ついに私はこのようなロボット工学に行きました：2007-2009年に、A。ゴロビン教授とN.ウムノフ教授と一緒に、私たちは最初の科学論文の準備を始めました。そこで、ストロボスコープ写真に従ってオブジェクトの軌跡を決定するためにアルゴリズムを使用する必要がありました。このトピックから-マシンビジョン、OpenCV、およびロボットオペレーティングシステムへの1つのステップ（当時、私はそのようなスケールについてさえ考えていませんでした）。その後、ようやく応用力学とロボット工学の研究に力を入れ、開発は補助的な活動になりました。

しかし、ロボット工学の新しい仕事を見つけるには、プログラミングの知識を向上させ、補足する必要がありました。結局のところ、毎年の大学のコース（ObjectPascalとBorland VCL in C ++）を除いて、ITを勉強しませんでした。開発の理論的な側面は数学に依存していました。

最初に、母国の研究所でフルタイムコースのオプションを検討しました。確かに、そのような研究を部署での仕事と組み合わせることは、スケジュールが破れていたり、自分のスケジュールの外で頻繁に仕事（交換など）があったため、実際には不可能であることはすぐに明らかになりました。そのため、有料コースをリモートで終了するというアイデアが徐々に思い付きました。私はバウマンカにあるTechnopark Mail.ruトレーニングセンターの教授の推薦でGeekBrainsに参加し、Pythonプログラマーコースに参加しました。

コースは問題を引き起こしませんでした、一つの問題-絶えず結合されなければなりませんでした：部門での仕事、科学的な仕事とイベントで。家の外のほとんどの社会的つながりを（幸いにも一時的に）犠牲にしなければならないほどの十分な時間はありませんでした。

彼はこのような負荷に対処しました：彼は道路上の問題を解決しました。数多くの出張で開発されたこのスキルは、非常に有用であることがわかりました。それがないと、すべての宿題を完了することさえできなかったためです（そして、瞑想の代わりにもなります...）ノートパソコン、スマートフォン、スマートフォン用のワイヤレスキーボードを使用して、外出先でプログラミングする方法を学びました。

私はDell Latitude 3470ラップトップを持っていますが、対角線5.5インチ以上のスマートフォンであれば、Logitech K 810 BTキーボードとペアにできます。一般に、私はLogitech製品をすべての人に推奨します。これは非常に信頼性が高く、非常に過酷な使用条件に耐えます（これは決して広告ではありません）。


Logicool K810

Python キーボードは、優れたエディタを使用して、このような作業を非常に行うことができます。別のプログラミングハック：デスクトップまたはランタイムへのリモート接続を使用します。自宅のコンピューターでDjangoを実行している安全なWebサーバーを使用して、いくつかのタスクを完了しました。彼は列車から同時に働き、ソフトウェアはPyDroid、DroidEdit、Maximaを使用しました。

なぜPythonなのか？


しばらくの間、PHPをシステムスクリプト言語として使用しようとしました。 Pythonは最初は独立していて、少しずつ「自分のために」勉強しました。モジュールレベルでPythonとC ++の間に効果的な接続が存在することを知った後、彼は真剣に研究することにしました。1つの言語でデータを準備するために最適化されたアルゴリズムと手順を分離することは興味深いようです。

最も単純な例：RISCプロセッサを搭載した組み込みマシンに、C ++で実装された非標準の強力なドライブ制御システムがあります。管理は、たとえば、ネットワークを介したサブシステム間の通信をサポートする外部のマシン依存APIを介して行われます。高レベルでは、ドライブ操作アルゴリズムはデバッグされないか、一定ではありません（作業プロセスに応じて、さまざまなアルゴリズムのロードが必要です）。

そのようなシステムを取得する最良の方法の1つは、クロスプラットフォームのインタープリターで使用されるPythonクラスのセットの基礎として、マシン固有のAPI C ++サブシステムを使用することです。したがって、トップレベルの開発者は、組み込みマシンとそのOSの機能を考慮する必要はなく、低レベルAPIの「ラッパー」として機能するPythonクラスを使用するだけです。

C ++とPythonのバインディングを一から学習する必要がありました。すぐに、高レベルのオブジェクト指向機能は低レベルよりもはるかに重要であることが理解されました。このため、Pythonレベルでクラスを選択し、C / C ++でグローバルデータを共有することを選択して、APIを設計および実装する方法を完全に変更する必要がありました。コード生成に慣れる：たとえば、ROSフレームワーク自体がPythonで名前とオブジェクトを生成するため、言語の違いを考慮に入れる必要があります。

現在の作業：Pythonとロボット制御システムのロジック


現在、MSTUの科学および教育センター「Robotics」でPythonおよびC ++のプログラマーとして働いています。政府部門から委託された研究プロジェクトとソフトウェアツールを実装します。組み込みのビジョンシステムと、システムに依存しない高度な自動制御アルゴリズムを備えたマニピュレーターを開発しています。

今、私はPythonでロボット制御システムの高水準ロジックをプログラミングしています。この言語は、C ++、アセンブラー、およびGoで書かれた高度に最適化されたモジュールを相互接続します。

ロボット制御アルゴリズムのプログラミングでは、2つの大きなアルゴリズムグループが使用されます。それらの最初のものは、低レベルで機器に直接実装されます-これは、ドライブコントローラー、通信ラインコンセントレーター、オペレーターとの相互作用のサブシステムのファームウェアです。

ここでのアルゴリズムは、制御された実行速度と信頼性がロボット全体のパフォーマンスを超えるように設計されています。システム全体のセキュリティは低レベルの制御ソフトウェアに依存するため、後者が必要です。

アルゴリズムの2番目のグループは、ロボット全体の動作を決定します。これらは高レベルのプログラムであり、その開発ではアルゴリズムの実装の理解可能性と実装速度に重点が置かれ、多くの場合非常に複雑です。さらに、ロボットの高レベルのソフトウェアは、試運転とテストのプロセスで非常に頻繁に変更されます。そのような開発には、解釈可能な汎用言語が不可欠です。

そのような仕事に必要な知識


C ++テンプレート言語とPythonのオブジェクト指向機能について学習する必要があります。ほとんど必須のスキルは、APIを設計および文書化する能力です。 Boost :: Pythonなどの専用ライブラリの機能を研究することは不必要ではありません。低レベルのソフトウェアで作業する人は、マルチスレッド（およびカーネルレベル）、Linux / UNIX / QNXシステムコールを処理する必要があります。ロボット工学の構築原理の理解を深めるには、Robotic Operating Systemフレームワークに慣れると非常に役立ちます。

私は、アセットに少なくとも1つのコンパイル言語と1つのインタープリター型プログラミング言語を組み込んで、開発し、要求に応えようとしています。これは、高度に専門化された（読み取り：異常な）アルゴリズムを開発し、それらをコンパイル言語で実装することが常に必要とされるエンジニアリングで働くための勝利戦略です。このようなソフトウェアのデータを準備する作業は、インタープリター言語を使用して解決する方がはるかに優れています。最初に、私のセットにはC ++、Pascal、BASICが含まれ、後にPHPとBASHが追加されました。

学生の学習に開発ツールがどのように役立つか


現在の専門能力開発の主な計画は、教授法を開発およびテストするために、教育学における専門ソフトウェア開発ツールの使用のための科学的基盤をもたらすことを試みることです。

2016年、私は開発ツール（プログラミング言語、IDE、ドキュメントジェネレーター、バージョン管理システム）を高等教育の教育実践に導入する大規模な実験を開始しました。これで、定性的に要約できる結果を得ることができました。

たとえば、教育プロセスに資料のバージョン管理を導入すると、学生が共同プロジェクトで共同作業するという条件の下でのみ、学生の作業の質が大幅に向上します。専門のソフトウェア開発ツールを使用して技術分野を教える方法の開発は、現在、MSTUの補足教育プログラムの学生、申請者、学生で構成される私の研究グループの力によって積極的に行われています。

ちなみに、私は教育実習を辞めませんでした。MSTUの高等研究所のためにLinuxデバイスと管理に関する著者の詳細な対面コースを開発し、私自身が教えています。
