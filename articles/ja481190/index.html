8位：「ビット演算の危険性」


V1046 bool 'and' int 'types in the operations in the operation'＆= 'の安全でない使用法です。GSLMultiRootFinder.h 175

int AddFunction(const ROOT::Math::IMultiGenFunction & func) {
  ROOT::Math::IMultiGenFunction * f = func.Clone();
  if (!f) return 0;
  fFunctions.push_back(f);
  return fFunctions.size();
}

template<class FuncIterator>
bool SetFunctionList( FuncIterator begin, FuncIterator end) {
  bool ret = true;
  for (FuncIterator itr = begin; itr != end; ++itr) {
    const ROOT::Math::IMultiGenFunction * f = *itr;
    ret &= AddFunction(*f);
  }
  return ret;
}

コードに基づいて、SetFunctionList関数はイテレータのリストをバイパスすることが期待されていました。そして、それらの少なくとも1つが無効な場合、戻り値はfalseになり、それ以外の場合はtrueになります。

ただし、実際には、SetFunctionList関数は有効なイテレータであってもfalseを返すことがあります。状況を見てみましょう。AddFunction関数は、fFunctionsリスト内の有効なイテレータの数を返します。それら。ゼロ以外のイテレータを追加すると、このリストのサイズは1、2、3、4などのように順次増加します。ここで、コード内のエラーが発生し始めます。

ret &= AddFunction(*f);

なぜなら この関数はboolではなくint型の結果を返すため、偶数の '＆='演算は値falseを返します。結局のところ、偶数の最下位ビットは常にゼロになります。したがって、このような明らかでないエラーは、有効な引数であってもSetFunctionsList関数の結果を台無しにします。 例のコードを注意深く読んだ場合（そして、コードを注意深く読んだことでしょうか？）、これがROOTプロジェクトのコードであることに気付くでしょう。もちろん、それをテストしました：「ROOTコード分析-科学研究データの分析のためのフレームワーク」

static int
EatWhitespace (FILE * InFile)
  /* ----------------------------------------------------------------------- **
   * Scan past whitespace (see ctype(3C)) and return the first non-whitespace
   * character, or newline, or EOF.
   *
   *  Input:  InFile  - Input source.
   *
   *  Output: The next non-whitespace character in the input stream.
   *
   *  Notes:  Because the config files use a line-oriented grammar, we
   *          explicitly exclude the newline character from the list of
   *          whitespace characters.
   *        - Note that both EOF (-1) and the nul character ('\0') are
   *          considered end-of-file markers.
   *
   * ----------------------------------------------------------------------- **
   */
{
    int c;

    for (c = getc (InFile); isspace (c) && ('\n' != c); c = getc (InFile))
        ;
    return (c);
}                               /* EatWhitespace */
