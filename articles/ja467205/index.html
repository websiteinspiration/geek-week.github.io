他のアプリにデータを送るアプリ（エコシステムアプリ）を開発

こんにちは、みんな！

これは、プロジェクトの開発に関する2番目の記事です。前の記事を読んだことがない人のために：ある場所（Googleテーブル）から別の場所（ホーム）にデータを自動的にエクスポートする方法についてです。

本日は、サードパーティのアプリケーションが私のアプリケーションから送信されたデータを受信できるように、ライブラリをどのように作成（および設計）したかをお伝えします。猫の下で興味を持っている皆さんにお願いします。

パート1.問題


プロジェクトについて簡単に説明します。スマートフォンに接続するデバイスがあります。内部にニューラルネットワークがデバイスからのデータを認識し、結果を提供する私のアプリケーションがあります。そして、これだけの結果を得たい他のアプリケーションがあります。

結果にはいくつかのタイプがあります。デバイスからのベアデータ、デバイスからの処理済みデータ、デバイスのステータスに関する情報、データへのアプリケーションのアクセス権に関する情報（たとえば、ユーザーがデバイスのデータへのアクセス権を取り消した場合）。なんとかしてこの結果を他のアプリケーションに転送する必要があります。

私のコードで突然何かが間違って説明された場合-これは私のライブラリのドキュメントです。

パート2.ソリューションの計画


素晴らしいメカニズムがあります-ブロードキャスト。つまり、あるアプリケーションから他のアプリケーションへのメッセージです。すぐに全員に送信することも、特定の相手に送信することもできます。

このビジネスを送受信するには、次のものが必要です。

  1. 何とか渡されたオブジェクトからJSONを作成する
  2. ブロードキャストを送信
  3. 別のアプリケーションでブロードキャストを受け入れる
  4. JSONから送信されたオブジェクトを復元する

一般に、オブジェクトからJSONを作成することは常に正しいとは限りません。Parcelableと呼ばれるものをBroadcast経由、またはSerializableで送信できます。SerializableはJavaの標準的なもので、Parcelableも同様ですが、モバイルデバイス用に最適化されています。

私のオブジェクトはかなり小さいです。さらに、JSONを取得する必要があることがよくあります。私自身がサーバーに生データを送信するようにサードパーティのアプリケーションを作成しました。そのため、「渡されたオブジェクトからJSONを作成する」を選択しました。多分それから私は私の心を変えます。

パート3.ソリューションのソーイング


パラグラフ1とパラグラフ4。JSONで、次に戻る


ここではすべてが簡単です。私たちのニーズに非常に適したGsonライブラリがあります。

物事をクールにするには、toString（）メソッドをオーバーライドします。そうですね、fromString（）を実行してオブジェクトを取得してください。

class SecureData(val eeg1: Double?, val eeg2: Double?, date: Date) {
    override fun toString(): String {
        val gson = Gson()
        return gson.toJson(this)
    }

    companion object {
        fun fromString(model: String): SecureData {
            val gson = Gson()
            return gson.fromJson(model, SecureData::class.java)
        }
    }
}

ポイント2.オブジェクトを送る


そのようなコードの例を次に示します。

val intent = Intent()
intent.action = BroadcastUtils.BROADCAST_GESTURE
intent.putExtra(BroadcastUtils.EXTRA_GESTRE, it.toString())
sendBroadcast(intent)

ここでは、インテントを作成し、そのアクションを設定し、オブジェクトを配置して、ブロードキャストとして送信します。
BroadcastUtils.BROADCAST_GESTURE-これは、別のアプリケーションで着信ブロードキャストをフィルタリングするための小さな処理です（処理する必要があるかどうかに関係なく）。

特定のアプリケーションにメッセージを送信するには、さらに以下を指定する必要があります。

 intent.component = ComponentName(
         PermissionsFetcher.REFACE_APP,
         "${PermissionsFetcher.REFACE_APP}.receivers.ActionsReceiver"
 )

PermissionsFetcher.REFACE_APPはアプリケーションのAPPLICATION_IDで、$ {PermissionsFetcher.REFACE_APP} .receivers.ActionsReceiverはレシーバーへのパスです。

class GesturesReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent) {
        Timber.i("Received gesture")

        val action = BroadcastUtils.reparseGestureIntent(intent)
