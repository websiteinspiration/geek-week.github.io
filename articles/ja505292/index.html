DATA VAULTデータショーケース

以前の記事では、DATA VAULTの基本を紹介し、DATA VAULTをより適切な状態分析に拡張し、BUSINESS DATA VAULTを作成しました。 3回目の記事でシリーズを完了する時が来ました。

前回の出版物で発表したように、この記事ではBIのトピック、またはBIのデータソースとしてのDATA VAULTの準備に焦点を当てます。ファクトテーブルとディメンションを作成し、それによってスタースキーマを作成する方法を検討します。

DATA VAULTでデータマートを作成するトピックについて英語の資料を勉強し始めたとき、プロセスが十分に複雑であると感じました。記事は印象的なボリュームを持っているので、Data Vault 2.0方法論に現れた公式の変更への参照があり、これらの公式の重要性が示されています。

しかし、翻訳を深く掘り下げた結果、このプロセスはそれほど複雑ではないことが明らかになりました。しかし、おそらくあなたは別の意見を持っています。

では、要点を説明しましょう。

DATA VAULTのディメンションテーブルとファクトテーブル


最も理解しにくい情報：

  • 測定テーブルは、ハブとその衛星の情報に基づいて構築されています。
  • ファクトテーブルは、リンクとその衛星の情報に基づいて構築されています。

そして、これはDATA VAULTの基本についての記事を読んだ後に明らかです。ハブは、ビジネスオブジェクトの一意のキー、時間に関連付けられたビジネスオブジェクトの属性の状態のサテライト、トランザクションをサポートするリンクに関連付けられたサテライトにこれらのトランザクションの数値特性を格納します。

この理論では、原則として、終了します。

しかし、それでも、私の意見では、DATA VAULT方法論に関する記事にあるいくつかの概念に注意する必要があります。

  • 生データマート-生データのウィンドウ。
  • 情報マート-情報ストアフロント。

「Raw Data Marts」という用語は、かなり単純なJOINを実行することにより、DATA VAULTデータに基づいて構築されたストアフロントを指します。 「Raw Data Marts」アプローチでは、分析に適した情報を使用して、倉庫プロジェクトを柔軟かつ迅速に拡張できます。このアプローチは、ストアフロントに配置される前の複雑なデータ変換の実装やビジネスルールの実装を意味するものではありませんが、「Raw Data Marts」データはビジネスユーザーに理解可能であり、BIツールなどを使用したさらなる変換の基礎として機能する必要があります。

「情報マート」の概念はData Vault 2.0の方法論に登場し、「データマート」の古い概念に取って代わりました。この変更は、データを情報に変換するものとしてレポートを作成するためのデータモデルを実装するタスクの認識によるものです。「情報マート」スキームは、そもそも意思決定に適した情報をビジネスに提供する必要があります。

十分に冗長な定義は、2つの単純な事実を反映しています。

  1. 「Raw Data Marts」のディスプレイケースは、基本（HUBS、LINKS、SATELLITES;
  2. Information Martsストアフロントは、BUSINESS VAULT要素（PIT、BRIDGE）を使用して構築されています。

従業員に関する情報の格納例を見ると、現在（今日有効）の従業員の電話番号を表示するストアフロントは、「Raw Data Marts」タイプのストアフロントであると言えます。このようなショーケースを作成するには、従業員のビジネスキーと、衛星アップロード日付（MAX（SatLoadDate））の属性で使用されるMAX（）関数を使用します。属性変更の履歴をストアフロントに保存する必要がある場合-これを使用する場合、電話が最新の日付であると理解する必要があります。ビジネスキーのコンパイルとサテライトへのロードの日付が主キーとしてこのテーブルに追加され、有効期限の有効期限フィールドの日付も追加されます。

電話番号、住所、名前など、ハブに含まれるいくつかの衛星の各属性の関連情報を格納するショーケースを作成することは、関連性のすべての日付を簡単に取得できるアクセスを通じて、PITテーブルを使用することを意味します。このタイプのショーケースは「インフォメーションマート」と呼ばれています。

どちらのアプローチも、測定と事実の両方に関連しています。

複数のリンクとハブに関する情報を格納するウィンドウ表示を作成するには、BRIDGEテーブルへのアクセスを使用できます。

この記事でDATA VAULTコンセプトのシリーズを完了します。私が共有した情報がプロジェクトの実装に役立つことを願っています。
