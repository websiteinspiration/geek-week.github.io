CSSフレームワークを比較するためのツール

こんにちは、私はアレクサンダーです。私は約1年間、積極的にWeb開発を研究し、行っています。それはすべて、お気に入りのゲームのクライアントの削除とメンターの検索から始まりました。彼はWeb開発を改善し、CSS Comparatorプロジェクトを作成しました。このプロジェクトの本質は、CSSフレームワークをサイズやその他の人気のある最新のパラメーターで比較することです。プロジェクトを作成するとき、商業目的は追求されません。この記事は、主に他のWeb開発者からフィードバックを受け取ることを目的としています。

これは現時点ですでに3番目のプロジェクトであり、初心者と経験豊富なWeb開発者の両方にとって興味深く、役立つので、私はそれについて記事を書くことにしました。

それはすべて、私の同僚がサイズのCSSフレームワークを探していたという事実から始まりました。Googleが提供する最初の答えは次のとおりです。



つまり、アイデアが思いついた-サイズやその他のパラメーターの点で人気のあるCSSフレームワークを比較する製品を作成することです。プロジェクトに取り組んでいるのは2人だけです。私は、フロントエンドWeb開発者として、そしてGolangでのスクレイピングに携わっていた私の同僚です。プロジェクトの最初のプロトタイプの外観はかなり単純化されていました。



一般的な代替案とは異なり、このプロジェクトには、アクセシビリティ（依存関係）、依存関係なし（JavaScript、jQueryなどに依存しない）、Gitterチャットおよびスタックオーバーフローへのリンクのフィルターがあります。

その後、プロジェクトは完全に私に渡されました。 2019年7月上旬に、作業が開始されました。最初の要望は、製品をより表現力豊かな外観にすること、そしてもちろんサイズによるフィルタリングとソートを追加することでした。必要な結果を反映するには、対応する入力フィールドに必要なサイズ（バイト単位）を指定するだけです



。サイズテーブルも個別に利用できます。



サイズによるフィルタリングと同様に、Stack Overflowでの質問の数とGitHubでの星の数によるフィルタリング。結局のところ、すべてのWeb開発者が適切な情報を探すために多くの時間を費やすことが知られています。この点で、関連するリンクとフォーラムへのリンク（ある場合）を追加することにしました。これらのソリューションは、必要なすべてのリンクが手元にあるため、開発を大幅に促進およびスピードアップするのに役立ちます。そして、フィルタリングは、ユーザーの間でより人気のあるフレームワークを見つけるのに役立ちます。

もちろん、スマートフォンの時代には、フレームワークの適応性に関するデータを追加せずに、マルチセレクトでそれをフィルタリングすることは不可能でした



。一般的に、フレームワークのカードに投稿された情報は、次のリストで概略的に表すことができます。

  • フレームワーク名
  • 会社の開発者
  • GitHubスター
  • 可用性のアクセシビリティ
  • JavaScript
  • Stack Overflow
  • Gitter

です。クライアント側では、HTML5、CSS3（Flexboxを使用したレイアウト）、およびサードパーティのライブラリとフレームワークを使用しない純粋なJavaScriptが使用されます。 GitHubスター、およびフレームワークのすべてのサイズは、Golangスクレーパーを使用してオープンAPIから取得され、cronジョブは毎週更新されます。



スクレーパー自体は非常に簡単に機能します。上記のjsonから取得したGitHubの星の数に関するデータ（独自のフレームワークごとに）。これは、参照によりBootstrapの例にあります。また、サイズは次のように計算されます。CSSのdevバージョンとprodバージョンがロードされ、次にprodバージョンがgzip形式でアーカイブされます。

したがって、クライアントで何が起こっているかをさらに説明します。

フィルター処理は、3つのパラメーターを取るonChecked関数を使用してチェックボックスによって実行されます。

$要素-チェックボックス
名自体-フィルター名
フィルター-フィルター自体。

例としてアクセシビリティフィルタリングを使用すると、コード全体は次のようになります。

const FILTER_ACCESSIBILITY = "accessibility";

function matchAll(framework) {
    return true;
}

function matchAccessibility(framework) {
    return framework.accessibility;
}

const matchStateMap = {
    [FILTER_ACCESSIBILITY]: matchAll,
};

function match(frameworks) {
    const result = frameworks
        .filter(matchStateMap[FILTER_ACCESSIBILITY]);
        // other filtration

    return result;
}

function $(id) {
    return document.getElementById(id);
}

function render(list) {
    const views = new Array(list.length);

    for (let i = 0; i < list.length; i++) {
        const item = list[i];

        views[i] = `<h2><a href="${item.siteUrl}" target="_blank">${item.name} v${item.version}</a></h2>`;
    }

    $("demo").innerHTML = views.join("");
}

function onChecked($element, name, filter) {
    $element.addEventListener("click", function () {
        if ($element.checked) {
            matchStateMap[name] = filter;
        } else {
            matchStateMap[name] = matchAll;
        }

        render(match(frameworks));
    });
}

const $accessibilityCheckbox = $("js-checkbox-accessibility");
onChecked($accessibilityCheckbox, FILTER_ACCESSIBILITY, matchAccessibility);

ソートは同様の機能で実行されます

let compare = null;
const matchStateMap = {
    [FILTER_ACCESSIBILITY]: matchAll,
    // ...
};

function match(frameworks) {
    const result = frameworks
        .filter(matchStateMap[FILTER_ACCESSIBILITY]);
        // other filtration

    // sort O(N *ln(N)), so better first match, than sort
    if (compare !== null) {
        result.sort(compare);
    }

    return result;
}

function sortByStars(a, b) {
    // DESC
    return b.repository.stars - a.repository.stars;
}

function onSort($element, sort) {
    $element.addEventListener("click", function () {
        compare = sort;

        render(match(frameworks));
    });
}

const $moreStars = $("js-sort-by-stars");
onSort($moreStars, sortByStars);
