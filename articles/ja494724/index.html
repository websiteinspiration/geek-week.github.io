ADAM-6700：ユニバーサルLinuxオブジェクト通信モジュール



ADAMシリーズのモジュールは、当社で最も有名な製品です。 1992年に発売されて、それらは常に進化し、古典的な認識可能なデザインを維持しています。今日、さまざまな問題を解決するためにいくつかのクラスのデバイスを組み合わせた強力な多機能デバイスです。

ADAM-6700は、Linux RT（リアルタイム）を実行するCortex-A8プロセッサに基づく新しいシリーズのデバイスです。通信モジュールの機能と、オブジェクト、プログラマブルロジックコントローラー（PLC）、データを収集および処理するためのデバイス、デジタルおよびアナログインターフェイス用のIPゲートウェイを組み合わせます。

組み込みのNode-REDサーバーがあり、Webインターフェースを介して構成できます。オープンプラットフォームのおかげで、モジュールでクライアントアプリケーションをCで実行し、作業のロジックをプログラムで実装できます。



仕様書


ADAM-6700 シリーズの新しいデバイスは、ARMアーキテクチャに基づいており、ローカル変換、ストレージ、およびデータ処理を可能にする生産的なシステムです。リアルタイム操作をサポートするバージョンのLinux OSを実行しているため、タイミングにとって重要な操作に使用できます。

  • オペレーティングシステム：Linux RT
  • CPU：Cortex-A8 32ビット1 GHz
  • RAM：512 MB
  • LANポート：2つの独立
  • USBスレーブ：1（設定用のコンソールポート）
  • USBホスト：1（デバイスの接続用）
  • 入力/出力ポート
  • デジタル入力：5（モデルADAM-6717）12（モデルADAM-6750）
  • デジタル出力：4（モデルADAM-6717）12（モデルADAM-6750）
  • アナログ入力：8（ADAM-6717モデル）



カスタマイズ


デバイスの初期構成は、ADAM / Apax.NETユーティリティを使用して行うことができます。ADAM/ Apax.NETユーティリティでは、ネットワークの動作モードとIPアドレスを設定できます。または、Webインターフェイスを介して行うこともできます。Webインターフェイスでは、Advantech独自のシェルとNode-REDの2つのデバイス構成オプションを使用できます。開始ページで、オプションの1つを選択する必要があります。


シェルを選択するための

Node-Red シェルWebインターフェースは、ビジュアルプログラミングとデータフロー制御のための一般的なオープンソース環境です。ADAM-6700シリーズのデバイスには組み込みのNode-REDサーバーがあり、プログラミングはWebブラウザーから直接、または構成をインポートすることによって実行できます。


Node-RED Webインターフェイス

デジタルおよびアナログI / Oインターフェースは、C言語のAPIを介して使用でき、独自のプロトコルを実装して顧客の機器と統合するために使用できます。

マイクロUSBによる設定



設定を簡単にするため、ADAM-6700はマイクロUSB経由でスレーブデバイスとしてコンピューターに接続できます。この場合、モジュールはUSBポートを介して給電されます。これは、端子を介して電源を接続する必要がないため、セットアップを大幅に簡素化します。つまり、ツールや電源やドライバーなどの追加機器なしでエンジニアが実行できます。ホストシステムでは、デバイスは仮想COMポートとして定義されます。

USBホストインターフェイス


ADAM-670 0には、ホストモードでデバイスを接続するための完全なUSB Aポートがあります。外部ドライブ、USBスティック、およびADAM 600W WiFiモジュールなどの拡張モジュールを接続できます。必要に応じて、システムのカーネルは、USBデバイス用のドライバーをサポートするようにコンパイルできます。

WiFiモジュールADAM-600W


ADAM-600W拡張モジュールは、ADAM-6700シリーズにWiFiサポートを追加します。接続はUSBを介して行われ、追加のソフトウェアをインストールする必要はありません。ADAM-600Wモジュールを接続すると、WebインターフェイスでWiFi接続設定メニューが使用できるようになります。

WiFi拡張モジュールを備えたADAM-6700は、イーサネット経由で接続された他のデバイスをサーバーに接続するためのゲートウェイとして機能できます。

結論


ADAM-6700 シリーズのユニバーサルデバイスを使用して、さまざまな自動化タスク、データ収集、周辺機器管理を解決できます。マルチタスキングオペレーティングシステムは、複雑なアプリケーションの作成において開発者を制限しません。オープンNode-REDプラットフォームのサポートとWebインターフェースを介した構成機能により、エントリーのしきい値が削減され、開発者は使い慣れた環境で作業できます。
