Verwenden Sie die if-else-Anweisung nicht mehr





Sie haben sich unzählige Tutorials mit If-else-Anweisungen angesehen. Sie lesen wahrscheinlich auch Programmierbücher, in denen die Verwendung von If-else als eigentliche Verzweigungsmethode empfohlen wird.



Vielleicht ist dies sogar Ihr Standardmodus, um If-else zu verwenden. Aber lassen Sie uns dies jetzt hinter uns bringen, indem wir If-else durch Statusobjekte ersetzen.



Beachten Sie, dass dieser Ansatz verwendet werden kann, wenn Sie eine Klasse mit Methoden schreiben, deren Implementierungen abhängig vom aktuellen Status geändert werden müssen. Sie würden einen anderen Ansatz wählen, wenn Sie sich nicht mit dem sich ändernden Zustand des Objekts befassen würden.

Ich habe nach besten Kräften eine Tabelle gezeichnet, in der die verschiedenen Bedingungen aufgeführt sind, unter denen sich die Reservierung befinden kann.







Das Refactoring der Verzweigungslogik aus unserem Code erfolgt in drei Schritten:



  1. Erstellen Sie eine abstrakte Basiszustandsklasse
  2. Implementieren Sie jeden Status als separate Klasse, die vom Basisstatus erbt
  3. Lassen Sie eine Klasse Booking



    eine private oder interne Methode haben, die eine Basiszustandsklasse als Parameter verwendet


Implementierung



Zunächst benötigen wir eine Basiszustandsklasse, die alle Zustände erben.







Beachten Sie, dass diese Klasse auch zwei Methoden hat, Akzeptieren und Abbrechen - obwohl sie hier als intern markiert sind.



Darüber hinaus verfügt der Basisstatus über eine „spezielle“ Methode EnterState(Booking booking)



. Sie wird aufgerufen, wenn einer Reservierung ein neuer Status zugewiesen wird.



Zweitens erstellen wir separate Klassen für jeden Status, den wir darstellen möchten.







Beachten Sie, wie jede Klasse den in beschriebenen Status darstellt Darüber hinaus erlaubt CanceledState nicht, dass unsere Buchung in einen neuen Status verschoben wird. Diese Klasse ist im Geiste der Nullobjektvorlage sehr ähnlich.



Schließlich die Buchungsklasse selbst.







Mal sehen, delegiert die Buchungsklasse nur die Berechtigung zum Akzeptieren und Ablehnen?



Dies ermöglicht es uns, den größten Teil der bedingten Logik zu eliminieren und es jedem Staat zu ermöglichen, sich nur auf das zu konzentrieren, was für sich selbst wichtig ist - der aktuelle Zustand hat auch die Möglichkeit, die Reservierung in einen neuen Zustand zu übertragen.



Wie man mit neuen Bedingungen umgeht



Wenn eine neue Funktion normalerweise mithilfe einer bedingten Prüfung implementiert wurde, können Sie jetzt einfach eine neue Statusklasse erstellen.



Es ist sehr einfach. Sie müssen sich nicht mehr mit der sperrigen if-else-Anweisung befassen.



So speichern Sie ein Statusobjekt in einer Datenbank



Das Statusobjekt spielt beim Speichern des Objekts keine Rolle, z. B. in einer SQL- oder NoSQL-Datenbank. Es ist nur wichtig, den Status des Objekts zu kennen und zu wissen, wie es in der Spalte angezeigt werden soll.

Sie können den Status mit einem lesbaren Namen, einer Aufzählung oder einer Ganzzahl abgleichen. All dies ist praktisch für Sie, solange Sie eine Möglichkeit haben, den gespeicherten Wert wieder in ein Statusobjekt umzuwandeln.
