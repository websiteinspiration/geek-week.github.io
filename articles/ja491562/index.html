Let's Encrypt証明書の大量失効




OpenNETは、非営利団体Let's Encrypt証明機関が、コミュニティの管理下にあり、証明書をすべての人に無料で提供しており、以前に発行された多くのTLS / SSL証明書の失効について警告していると書いています。3月4日、1億1,600万のアクティブな証明書のうち300万を少し超える数が失効しました（2.6％）。」証明書要求が一度に複数のドメイン名をカバーする場合、エラーが発生します。各ドメイン名にはCAAレコードの検証が必要です。エラーの本質は、再チェック時に、すべてのドメインを検証するのではなく、リストから1つのドメインのみが再チェックされたことです（リクエストにN個のドメインがある場合、N個の異なるチェックではなく、1つのドメインがN回チェックされました）。残りのドメインでは、再検証は実行されず、決定を行う際に、最初の検証のデータが使用されました（つまり、最長30日前のデータが使用されました）。その結果、最初の検証から30日以内に、CAAレコードが変更され、Let's Encryptが有効な証明機関のリストから削除された場合でも、Let's Encryptは証明書を発行できました。

OSIメーリングリストからのEric Raymondの削除とオープンライセンスの倫理上の問題

APT 2.0パッケージマネージャーリリース




OpenNETは、Debianプロジェクトによって開発されたAdvanced Package Tool（APT）のバージョン2.0のリリースを発表しました。Debianとその派生ディストリビューション（Ubuntuなど）に加えて、APTは、PCLinuxOSやALT Linuxなどのrpmパッケージマネージャーに基づく一部のディストリビューションでも使用されています。新しいリリースはまもなくDebian UnstableブランチとUbuntuパッケージベースに統合されます。いくつかの革新：

  1. パッケージ名を受け入れるコマンドでのテンプレートのサポート;
  2. 引数として渡される文字列で指定された依存関係を満たすために、「satisfy」コマンドを追加しました。
  3. システム全体を更新せずに他のブランチからパッケージを追加する。たとえば、テストまたは不安定から安定したパッケージにインストールすることが可能になった。
  4. dpkgロックが解放されるのを待っています（失敗した場合は、ロックファイルを保持しているプロセスの名前とpidが表示されます）。
