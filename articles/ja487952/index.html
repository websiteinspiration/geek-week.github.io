Pythonのスコープについて知りたいが、質問するのが恥ずかしかったすべて

コース"Python Developer"で新しいスレッドが始まることを見越して、Pythonのスコープについて話すことにしました。これから何が起こったのですか？-以下の資料をお読みください。



今日は、有能で読みやすく美しいコードを書くために理解し、覚えておかなければならない重要な理論的基礎について話します。変数のスコープについて説明します。この記事は、初心者だけでなく、他の言語からPythonに来てそのメカニズムを理解したい経験豊富なプログラマーにも役立ちます。

スコープは、プログラムのどの部分で特定の変数を操作できるか、および変数がそこから「非表示」になるかを決定します。必要な値と変数のみを使用する方法と、言語インタープリターの動作を理解することは非常に重要です。また、変数を使用するアクションのスコープによって課される制限を回避する方法についても説明します。Pythonでは、3つのスコープがあります。

  • 地元
  • グローバル
  • 非ローカル

最後の非ローカルスコープはPython 3で追加されました。

通常、私たちは関数に精通しているときのスコープについて話しています。それらの例を使用して、変数のスコープの動作を検討します。

ローカルスコープ


some_list要素 ごとにリスト要素を表示する関数を考えてみましょう：

def print_list(some_list):
    for element in some_list:
        print(element)

ここにelementありsome_list、関数の内部でのみ表示され、その操作中に関数内で割り当てられた値を使用して関数の外部で使用できないローカル変数です。つまり、プログラムの本体でを呼び出すとprint(element)、エラーが発生します。

NameError: name 'element' is not defined


次のことを行います。

def print_list(some_list):
    for element in some_list:
        print(element) 

element = 'q'
print_list([1, 2, 3])
print(element)


そして私達は得る：

1
2
3
q

ここで、element関数内の変数と関数外の同じ名前の変数は2つの異なる変数であり、それらの値は交差せず、交換されません。それらは同じと呼ばれますが、メモリ内の異なるオブジェクトを参照します。さらに、関数内のelementという名前の変数は、関数が実行されている限り存続します。ただし、ローカル変数とグローバル変数に同じ名前を付けるように注意してください。次に、その理由を示します。

def print_list(some_list):
    for element in sudden_list:
        print(element) 

sudden_list = [0, 0, 0]
print_list([1, 2, 3])

結果：

0
0
0

インタープリターがエラーを示していないことに注意してください。そして、すべて sudden_listがグローバルスコープ内にあるため、つまり、関数print_listの内部からアクセスできます。内部からは、外部で何が起こっているのかを確認できます。このような作業のメカニズムのため、関数内のローカル変数には、グローバルスコープで変数と呼ぶものとは異なる名前を付けてください。

ここで定数について話すことが重要です。変数と呼ばれるPythonインタープリターに違いはないため、上記のコードは次のように書き直すとよいでしょう。

SUDDEN_LIST = [0, 0, 0]

def print_list(some_list):
    for element in SUDDEN_LIST:
        print(element) 

print_list([1, 2, 3])

これですべてが整いました。問題は、Pythonでは定数を変更してはならないオブジェクトとして厳密に定義することはできないということです。したがって、名前が大文字で書かれている変数の値を使用する方法は、あなたの良心にのみ残ります。別の質問は、この方法で記録された変数は、変数がどこにも変更されないことをコードを読んだ人に明らかにすることです。または、少なくともそうすべきではありません。

グローバルスコープ


Pythonには、global関数内からグローバル変数の値を変更できるキーワードがあります。変数名の前に記述され、関数内でグローバルと見なされます。例からわかるように、変数の値はcandy増加しており、関数への引数として渡さないことに注意してくださいget_candy()。

candy = 5

def get_candy():
    global candy 
    candy += 1
    print('  {} .'.format(candy))
    
get_candy()
get_candy()
print(candy)


その結果、以下が得られます。


	  6 .
  7 .
7


ただし、関数内からグローバル変数の値を変更することはベストプラクティスではなく、コードの読みやすさに貢献しないため、変更しないことをお勧めします。関数内で発生することが小さいほど、グローバルスコープに依存します。

ライフハック：変数の命名の影響を受けないようにするために、メインプログラムコードを関数main()に入れることができます。その後、この関数内で宣言されるすべての変数はローカルのままであり、グローバルスコープを損なわず、エラーの可能性が高くなります。

非ローカルスコープ


このコンセプトは、キーワードとともにPython 3に登場しましたnonlocal。彼の文章の論理は、とほぼ同じですglobal。ただし、nonlocal特徴があります。Nonlocalネストされた関数では特定の変数がローカルではないが、一般的な意味ではグローバルではないことをインタープリターに理解させたい場合に、ネストされた関数で最もよく使用されます。

def get_candy():
    candy = 5
    def increment_candy(): 
        nonlocal candy
        candy += 1
        return candy
    return increment_candy
    
result = get_candy()()
print(' {} .'.format(result))

結果：

 6 .

自分で決めるのがどれほど便利か。あなたはここでより多くの例を見つけることができます。

結論として、いくつかのルールを策定できます。

  1. 関数内からは、関数の内部と外部の両方で定義された可視変数です。内部で定義された変数はローカルで、外部でグローバルです。
  2. 関数の外では、それらの内部で定義された変数は表示されません。
  3. , global.
  4. nonlocal , , .
