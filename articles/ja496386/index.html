DBの相互作用


次のステップは、データベースと対話するためのエンティティーです。このために、Springデータのjpa依存関係を接続します。

<dependency>
    <groupId>org.springframework.boot</groupId>

そして、src / main / java / main / modelディレクトリにPOJOクラスTodo を作成し、注釈を付け@Entityます。

フィールドを宣言すると、ID、名前、説明、日付が含まれます。

特別な注意を払ってsetDate()、文字列の入力とそれからjava.util.Dateへの変換でそれを行いました。さらにatStartOfDay().atZone(ZoneId.of("UTC")、日付フィールドの注釈にも注意を払っています。

package main.model;

import javax.persistence.*;
import java.time.LocalDate;
import java.time.ZoneId;
import java.util.Date;

@Entity
public class Todo {

    @Id
    @GeneratedValue(strategy = GenerationType.AUTO)
    private int id;
    private String name;
    private String description;
    @Temporal(TemporalType.DATE)
    private Date date;

    //getters and setters …
   
    public void setDate(String date) {
        this.date = Date.from(LocalDate.parse(date).atStartOfDay().atZone(ZoneId.of("UTC")).toInstant());
    }
}

@GetMapping("/todo-list/{id}")
public ResponseEntity<Todo> get(@PathVariable int id){
    Optional<Todo> todoOptional = todoRepository.findById(id);

@Controller
public class DefaultController {
    @Autowired
    TodoRepository todoRepository;
    @RequestMapping("/")
    public String index(Model model){
        Iterable<Todo>  todoIterable = todoRepository.findAll();
        ArrayList<Todo> todoList = new ArrayList<>();
        for(Todo todo : todoIterable){
            todoList.add(todo);
        }
        model.addAttribute("todoList", todoList);
        return "index";
    }
}

PUTリクエスト


TodoController で、入力マップ<String、String>にput()注釈を付けたメソッドと、注釈@PutMappingと、@RequestParamResponseEntityにラップされたTodoの出力に値から抽出されたintを作成します。また、リポジトリにはupdate（）メソッドがないため、次のようにすべてが行われます

。Todoは、todoRepositoryを介してデータベースから抽出されます=>新しいパラメータが割り当てられますTodo =>リポジトリを通じてデータベースに保存されます=>フロントエンド応答が送信されます

@PutMapping(value = "todo-list/{id}")
public ResponseEntity<Todo> put(@RequestParam Map<String, String> mapParam, @PathVariable int id){
    Optional<Todo> todoOptional = todoRepository.findById(id);
    if(todoOptional.isEmpty()){
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(null);
    }
    todoOptional.get().setName(mapParam.get("name"));
    todoOptional.get().setDescription(mapParam.get("description"));
    todoOptional.get().setDate(mapParam.get("date"));
    todoRepository.save(todoOptional.get());
    return ResponseEntity.ok(todoOptional.get());
}

[変更]ボタンをクリックします=>要素からTodoデータが収集されます=>ケースを変更するためにフォームが編集されます（フォームの名前が変更され、ボタンが入力値のTodoデータで置き換えられます）=>フォームが開かれます=>変更用のデータが入力されます=>フォームの[変更]ボタンをクリックします=>データ収集=> PUTリクエストが生成されます（パス、データ）=>変更されたTodoオブジェクトから応答を受け取りますが、同じIDでフロントエンドは必要な処理を行います。トド。
