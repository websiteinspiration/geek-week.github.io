TypeScriptの開発者は、classキーワードを介して実行される「厳密な契約」を処理したと思います。さらに、クラスは抽象的である必要はありません。

Find：extendsセクションでは式を使用できます。詳細。質問する場合-2つのクラスから継承できるかどうかは、正式な答えはノーです。ただし、機能をエクスポートする場合は-はい。

class One {
    one = "__one__";
    getOne(): string {
        return "one";
    }
}

class Two {
    two = "__two__";
    getTwo(): string {
        return "two";
    }
}

//  ,    :   IDE (  )    .
class BothTogether extends mix(One, Two) {
    //                     ^   ,    extends  
    info(): string {
        return "BothTogether: " + this.getOne() + " and " + this.getTwo() + ", one: " + this.one + ", two: " + this.two;
        //                             ^   IDE   ^ 
    }
}

type FaceType<T> = {
    [K in keyof T]: T[K];
};

type Constructor<T> = {
    // prototype: T & {[key: string]: any};
    new(): T;
};

// TODO:    ,   .      
function mix<O, T, Mix = O & T>(o: Constructor<O>, t: Constructor<T>): FaceType<Mix> & Constructor<Mix> {
    function MixinClass(...args: any) {
        o.apply(this, args);
        t.apply(this, args);
    }
    const ignoreNamesFilter = (name: string) => ["constructor"].indexOf(name) === -1;
    [o, t].forEach(baseCtor => {
        Object.getOwnPropertyNames(baseCtor.prototype).filter(ignoreNamesFilter).forEach(name => {
            MixinClass.prototype[name] = baseCtor.prototype[name];
        });
    });
    return MixinClass as any;
}

const bt = new BothTogether();
window.console.log(bt.info()); // >> BothTogether: one and two, one: __one__, two: __two__

感嘆符-無制限の演算子と修飾子



strictNullChecksおよびstrictPropertyInitializationコンパイル設定を使用しない場合、
おそらく近くに渡される感嘆符に関する知識...主な目的に加えて、さらに2つの役割が割り当てられます。

検索：非nullアサーション演算子としての感嘆符

この演算子を使用すると、nullをチェックせずにnullにすることができる構造体フィールドにアクセスできます。説明付きの例：

 //     --strictNullChecks
 type OptType = {
     maybe?: {
         data: string;
     };
 };
 // ...
 function process(optType: OptType) {
     completeOptFields(optType);
     //   ,   completeOptFields    .
     window.console.log(optType.maybe!.data);
     //                              ^ -    ,    null
     //                                    !,    tsc: Object is possibly 'undefined'
 }

 function completeOptFields(optType: OptType) {
     if (!optType.maybe) {
         optType.maybe = {
             data: "some default info"
         };
     }
 }

合計では、この作業は、あなたが...私たちは確信している場合は、コード内でヌルために、不要なチェックを削除することができます

検索と感嘆符：修飾子明確な代入アサーション

strictPropertyInitializationコンパイルオプションをオンにしてこの修飾子は、どこかのコードでは、私たちは、後にクラスのプロパティを初期化することを可能にします。説明付きの例：

//     --strictPropertyInitialization
class Field {
    foo!: number;
    // ^
    // Notice this '!' modifier.
    // This is the "definite assignment assertion"

    constructor() {
        this.initialize();
    }

    initialize() {
        this.foo = 0;
        //       ^  
    }
}

しかし、感嘆符に関するこのすべての小さな計算は、ユーモアの瞬間なしでは意味がありません。

質問：次の式がコンパイルされると思いますか？

//     --strictNullChecks
type OptType = {

        data: string;

};
function process(optType: OptType) {
    if (!!!optType.maybe!!!) {
        window.console.log("Just for fun");
    }
    window.console.log(optType.maybe!!!!.data);

タイプ



複雑なタイプを書く人は皆、多くの興味深いことを発見します。運が良かったです。

検索：サブタイプは、メインタイプのフィールドの名前で参照できます。

type Person = {
    id: string;
    name: string;
    address: {
        city: string;
        street: string;
        house: string;
    }
};

type Address = Person["address"];

自分で型を書くとき、この宣言アプローチはほとんど意味がありません。しかし、型は外部ライブラリからのものですが、サブタイプはそうではありません。

サブタイプのトリックは、コードを読みやすくするためにも使用できます。クラスが継承するジェネリック型を持つ基本クラスがあるとします。以下の例は、言われたことを示しています。

class BaseDialog<In, Out> {

    show(params: In): Out {/**  .   return ... */ }
}

//  -
class PersonDialogOld extends BaseDialog<Person[], string> {/**   */}

//  
class PersonDialog extends BaseDialog<Person[], Person["id"]> {/**   */}

検索：TypeScript型システムの助けを借りて、目的の機能をカバーする生成型の組み合わせセットを実現できます。言うのは難しいと思います。私はこの処方について長い間考えていました。最も有名なビルダーテンプレートの例を紹介します。この設計パターンを使用してオブジェクトを構築する必要があると想像してください。

class SimpleBuilder {

    private constructor() {}

    static create(): SimpleBuilder {
        return new SimpleBuilder();
    }

    firstName(firstName: string): this {
        return this;
    }

    lastName(lastName: string): this {
        return this;
    }

    middleName(midleName: string): this {
        return this;
    }

    build(): string {
        return "what you needs";
    }
}

const builder = SimpleBuilder.create();
//     .
const result = builder.firstName("F").lastName("L").middleName("M").build();

とりあえず、冗長なcreateメソッド、プライベートコンストラクター、および一般にtsでのこのテンプレートの使用については見ていないでください。コールチェーンに焦点を当てます。呼び出されたメソッドは厳密に1回使用する必要があるという考え方です。また、IDEもこれを認識している必要があります。つまり、ビルダーインスタンスでメソッドを呼び出した後、このメソッドを使用可能なメソッドのリストから除外する必要があります。このような機能を実現するには、NarrowCallside型が役立ちます。

type ExcludeMethod<T, K extends keyof T> = Pick<T, Exclude<keyof T, K>>;

type NarrowCallside<T> = {
    [P in keyof T]: T[P] extends (...args: any) => T ?
        ReturnType<T[P]> extends T ?
        (...args: Parameters<T[P]>) => NarrowCallside<ExcludeMethod<T, P>>
        : T[P]
        : T[P];
};

class SimpleBuilder {

    private constructor() {}

    static create(): NarrowCallside<SimpleBuilder> {
        return new SimpleBuilder();
    }

    firstName(firstName: string): this {
        return this;
    }

    lastName(lastName: string): this {
        return this;
    }

    middleName(midleName: string): this {
        return this;
    }

    build(): string {
        return "what you needs";
    }
}

const builder = SimpleBuilder.create();
const result = builder.firstName("F")
//                    ^ -   
                .lastName("L")
//              ^ -   lastName, middleName  build
            .middleName("M")
//          ^ -   middleName  build
        .build();
//      ^ -    build

検索：TypeScript型システムを使用して、厳密な順序を指定することにより、呼び出しのシーケンスを制御できます。以下の例では、DirectCallsideタイプを使用して、これを示しています。

type FilterKeys<T> = ({[P in keyof T]: T[P] extends (...args: any) => any ? ReturnType<T[P]> extends never ? never : P : never })[keyof T];
type FilterMethods<T> = Pick<T, FilterKeys<T>>;

type BaseDirectCallside<T, Direct extends any[]> = FilterMethods<{
    [Key in keyof T]: T[Key] extends ((...args: any) => T) ?
        ((..._: Direct) => any) extends ((_: infer First, ..._1: infer Next) => any) ?
        First extends Key ?
        (...args: Parameters<T[Key]>) => BaseDirectCallside<T, Next>
    : never
    : never
    : T[Key]
}>;

type DirectCallside<T, P extends Array<keyof T>> = BaseDirectCallside<T, P>;

class StrongBuilder {

    private constructor() {}

    static create(): DirectCallside<StrongBuilder, ["firstName", "lastName", "middleName"]> {
        return new StrongBuilder() as any;
    }

    firstName(firstName: string): this {
        return this;
    }

    lastName(lastName: string): this {
        return this;
    }

    middleName(midleName: string): this {
        return this;
    }

    build(): string {
        return "what you needs";
    }
}

const sBuilder = StrongBuilder.create();
const sResult = sBuilder.firstName("F")
//                      ^ -   firstName  build
                    .lastName("L")
//                  ^ -   lastName  build
                .middleName("M")
//              ^ -   middleName  build
            .build();
//          ^ -   build
