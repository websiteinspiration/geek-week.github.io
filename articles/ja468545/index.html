ボット


ボットの歴史から始めましょう。 1966年に、心理療法士を装ったElizaボットが登場しました。彼とコミュニケーションをとることができ、生きている人が答えると信じている人もいます。 1995年に、ALICEボットが登場しました。 -アリスと混同しないでください。ボットは実在の人物になりすますことができました。今日まで、オープンソースにあり、最終決定されています。残念ながら、ALICEはチューリングテストに合格していませんが、誤解を招くことはありません。

2006年、IBMはボットに巨大なナレッジベースと高度なインテリジェンスを配置しました。これが、IBMワトソンが生まれた方法です。これは、英語のスピーチを処理していくつかの事実を伝えることができる巨大なコンピューティングクラスタです。

2016年、マイクロソフトは実験を行いました。彼女は、ボット作成テイを彼女がツイッターで立ち上げたこと。そこでボットは、ライブのサブスクライバーがボットとどのように対話したかに基づいてマイクロブログを学びました。その結果、テイは人種差別主義者と誤認主義者になりました。現在、これは閉鎖されたアカウントです。道徳：ツイッターで子供たちをさせないで、彼は悪い人を教えることができます。

しかし、これらはすべてあなた自身の利益のために通信することができないボットです。 2015年、「有用な」がTelegramに登場しました。ボットは他のプログラムに存在していましたが、テレグラムはスプラッシュを作りました。情報を提供し、コンテンツを生成し、パブリックを管理する便利なボットを作成することが可能でした-可能性は大きく、APIはシンプルです。ボットは画像、ボタン、ツールチップを追加しました-インタラクションインターフェイスが表示されました。

徐々に、アイデアはほぼすべてのインスタントメッセンジャーに広がりました：Facebook、Viber、VKontakte、WhatsAppおよびその他のアプリケーション。今やボットはトレンドであり、至る所にあります。すべてのプラットフォーム用のAPIをすぐに作成できるサービスがあります。

音声アシスタント


開発はボットと並行して行われましたが、アシスタントの時代が後に来たと想定します。

2011年8月9日、Siriが登場しました。当初、それは独立したプロジェクトで、Appleが興味深いものを見つけたので、私はそれを購入しました。これは、OSに組み込まれた最も古い人気の音声アシスタントです。 1年後、GoogleはGoogle Nowの音声アシスタントをオペレーティングシステムに統合することで、すぐにAppleに追いつきました。

2年後、MicrosoftはMicrosoft Cortanaをリリースしました。しかし、その理由は明らかではありません-音声アシスタントのモバイル市場はすでに見落としているようです。同社は、さまざまなデバイスの市場ですでに闘争があったときに、音声アシスタントをデスクトップシステムに統合しようとしました。その年の少し後にAmazon Alexaが出ました。。



アシスタントが進化しました。音声を操作する方法を知っているソフトウェアシステムに加えて、スピーカーがアシスタントと共に登場しました。統計によると、2019年の初めには、米国の3人に1人の家族にスマートコラムがあります。これはあなたが投資できる巨大な市場です。

しかし問題があります-外国人のアシスタントはロシア語が下手です。アシスタントは英語で投獄され、よく理解していますが、ロシア語でコミュニケーションをとると、翻訳が困難になります。言語は異なり、自然言語処理への異なるアプローチが必要です。

アリス


アリスは2017年10月10日にオープンベータ版でリリースされました。それはロシア語のために投獄されており、これはその大きな利点です。アリスは英語を理解できますが、もっと悪いです。

アリスの使命は、ロシア語を話すユーザーを助けることです。

Yandexは大企業であり、なんとか話すことができるすべてのアプリケーションにAliceを組み込む余裕があります。

アシスタントの開発の2年間で、彼女は多くのサービスに統合され、新しいスキルを追加しました。彼女は、音楽を再生し、写真を認識し、Yandexで情報を検索し、スマートホームで作業する方法を知っています。

なぜそれほど人気が​​あるのですか？


手が忙しいときに便利です。私は夕食を作っていて、音楽をつけたいです。長い間、蛇口に行き、手を洗い、乾かし、アプリケーションを開いて、目的のトラックを見つけます。音声コマンドをすばやく簡単に入力できます。

怠惰。私は格子縞の毛布の下のソファーに横になっていて、どこかに行ってスピーカーをオンにしたくありません。あなたが怠惰な場合は、完全に。

大きな市場は子供向けのアプリです。幼児はまだ読み書き、印刷の方法を知りませんが、話し、話し言葉を理解します。したがって、子供たちはアリスを崇拝し、彼女とコミュニケーションするのが大好きです。両親も幸せです-子供とどうするかを探す必要はありません。興味深いことに、アリスはよく訓練されたニューラルネットワークのおかげで子供を理解しています。

可用性。視覚障害者は音声アシスタントを快適に操作できます。インターフェイスが表示されていないときは、音声を聞いてコマンドを出すことができます。

声が速くなります。開発者ではなく平均的な人が1分あたり平均30ワードを印刷し、120と言います。1分あたり、4倍の情報が音声で送信されます。

未来。幻想的な映画と未来の予測は、未来は音声インターフェースにあることを示唆しています。脚本家は、音声制御が、画像がそれほど重要ではないインターフェースと対話するための主な方法になると考えています。

統計によると、毎月3,500万人がアリサを使用しています。ちなみにベラルーシの人口は9,475,600人です。つまり、約3.5ベラルーシでは毎月アリスを使用しています。

音声アシスタントが市場を征服します。予測によると、2021年までに約2倍に成長すると予測されています。今日の人気は止まることはありませんが、成長し続けます。ますます多くの開発者が、この分野に投資する必要があることを認識しています。

import ical from 'node-ical';

function parseCalendar(str) {
    return new Promise((resolve, reject) => {
        ical.parseICS(str, function(err, data) { 
            if (err) {
                reject(err);
            }
            resolve(data);
        });
    });
}

スクリプト作業


優れたスキルには、興味深いユースケースが必要です。読む価値のある1冊の本を勧めます-「音声ユーザーインターフェイスの設計：会話体験の原則」。音声インターフェース用のスクリプトを作成し、ユーザーの注意を引き付けることは素晴らしいことです。本は英語で見ませんでしたが、とても簡単に読めます。



スキルの開発を開始する最初のことは挨拶です。

「何を言うべきかを考えている間、礼儀正しくしてください！時間を節約できます。」

スキルがオンになったら、ユーザーを最初の1秒からどうにかホールドする必要があります。そのためには、スキルの使用方法を説明する必要があります。ユーザーがスキルを起動し、沈黙があったとします。スキルがまったく機能するかどうかはどうやってわかりますか？画面上のボタンなど、ユーザーに指示を与えます。

簡単な対話


簡単な対話の兆候。このリストはIvan Golubevが考案したもので、私はこの表現が本当に好きです。

  • Lパーソナル。
  • E stestvenny。
  • D柔軟です。
  • ontekstny によって。
  • そして、うんざり。
  • ブリーフQを。

パーソナルとは、ボットにキャラクターが必要であることを意味します。アリスと話すと、彼女がキャラクターであることを理解できます-開発者がこれを処理します。したがって、有機性のためのボットには「個性」が必要です。少なくとも1つの声でフレーズを伝えるには、同じ言葉の構造を使用します。これは、ユーザーを維持するのに役立ちます。

ナチュラル。ユーザーのリクエストが単純であれば、答えは同じになるはずです。ボットとの通信中に、ユーザーは次に何をすべきかを理解する必要があります。

フレキシブル。何のために準備をしなさい。ロシア語には多くの同義語があります。ユーザーは、列から注意をそらして会話を対話者に転送し、列に戻ることができます。このすべてを処理することは困難です。しかし、ボットをうまく作成したい場合は、ボットを作成する必要があります。一部の割合の非認識が引き続き存在することに注意してください。これに備えてください-オプションを提供してください。

コンテキスト  -ボットは、理想的には、以前に何が起こったかを覚えておく必要があります。その後、会話は活発になります。

-アリス、今日の天気は？
-今日、この地区では+11から+20まで曇りのち晴れです。
- そして明日？
-地区の明日+14から+27まで、曇りで晴れ。

ボットがコンテキストを保存する方法を知らないと想像してください。それでは、リクエストは「明日」とはどういう意味ですか？アリスのようにコンテキストを維持する方法を知っている場合は、以前の結果を使用してスキルの答えを改善できます。

クリエイティブに -これにより、スキルがトップディレクトリに存在する可能性が高くなります。ユーザーは創造的なスキルを使うことにもっと興味を持っています。

要約する。音声操作が便利です。それは他のすべてに取って代わるわけではありませんが、私はそれが発展して拡大すると信じています。おそらくすぐにフロントエンドは声と密接に連携するでしょう。

所定の位置にとどまるだけの速度で実行する必要がありますが、どこかに到達するには、少なくとも2倍の速度で実行する必要があります。
