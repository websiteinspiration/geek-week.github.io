MitogenでAnsibleをスピードアップ

Ansibleは、最も人気のある構成管理システムの 1つになりました。2015年にRed Hatが購入された後、プロジェクト参加者の数は数千人を超え、Ansibleはおそらく最も使用された展開およびオーケストレーションシステムでした。その幅広い用途は非常に印象的です。


Ansibleは、リモートホストへのSSH接続を介して機能します。SSHセッションを開き、ログインを行い、ネットワーク経由でPythonコードをコピーして、別の一時ファイルに書き込みます。その後、このファイルをリモートマシンで実行します。この一連の操作全体は非常に長くて面倒なので、最適化するにはさまざまな方法があります。


これらの方法の1つは、SSHパイプラインです。これにより、毎回新しいセッションを開くのではなく、1つのSSHセッションを使用して命令を実行できるため、時間を大幅に節約できます。（リモートマシンのファイルでrequirettysudo の設定を無効にすることを忘れないでください/etc/sudoers）


Ansibleを「オーバークロック」する新しい方法は、Mitogenと呼ばれるpythonライブラリを使用することです。誰かが聞いたことがない場合は、その機能について簡単に説明します。これにより、リモートマシンでのPythonコードの高速実行が可能になり、Ansibleは1つの使用例にすぎません。 MitogenはリモートマシンでUNIXパイプを使用し、Pythonコードを圧縮したzlibに渡し、pickleを使用してシリアル化します。これにより、実行速度が上がり、トラフィックが節約されます。詳細な説明に興味がある場合は、その仕組みのページで説明を読むことをお勧めします。しかし、今日は、Ansibleを使用したライブラリの作業のみに焦点を当てます。


特定の状況でMitogenを使用すると、Ansibleコードが数回高速化され、トラフィック消費が大幅に削減されます。最も一般的な使用例を確認して、これがどのように役立つかを見てみましょう。


何よりも私がAnsibleを使用するのは、リモートマシンでの構成ファイルの作成、パッケージのインストール、リモートマシンとの間のファイルのコピーです。多分あなたは他の例を持っています-コメントに書いてください。

AnsibleのMitogenの設定は非常に簡単です
。Mitogenライブラリをインストールします。


pip install mitogen

これで、同等の2つの方法があります。ansible.cfg構成ファイルでオプションを構成する方法と、必要な環境変数を設定する方法です。


インストールされているMitogenへのパスがであるとします/usr/lib/python2.7/site-packages/ansible_mitogen/plugins/strategy。次に：


export ANSIBLE_STRATEGY_PLUGINS=/usr/lib/python2.7/site-packages/ansible_mitogen/plugins/strategy
export ANSIBLE_STRATEGY=mitogen_linear

または


[defaults]
strategy = mitogen_linear
strategy_plugins = /usr/lib/python2.7/site-packages/ansible_mitogen/plugins/strategy

Mitogenの有無にかかわらず、virtualenvにAnsibleをインストールします。


virtualenv mitogen_ansible
./mitogen_ansible/bin/pip install ansible==2.7.10 mitogen
virtualenv pure_ansible
./pure_ansible/bin/pip install ansible==2.7.10
