デコードソフトウェアには「小さな」複雑さが1つあります。これはLinuxで書かれています。おそらくWindowsでGnu Radioプロジェクトをコンパイルする秘密の方法があるかもしれませんが、私は、非常に多くのエラーを除いて、失敗したことはありません。 Linuxでは、すべてがシンプルでそのまま使用できますが、ほとんどの読者にはWindowsがインストールされていると思います。そのため、別の方法で回避します。信号をHDSDRに書き込み、デコーダーを実行してWindows 10のUbuntuから信号を処理します。幸い、10ka がこの機能を提供しています。

ステップ1。 gr-iridiumのコンパイル

まず、UbuntuをApp Storeにダウンロードし、そこにGNU Radioをインストールする必要があります。

実際にgr-iridiumデコーダーをコンパイルすることは特に難しくありません：

git clone https://github.com/muccc/gr-iridium.git
cd gr-iridium
mkdir build
cd build
cmake ..
make
sudo make install
sudo ldconfig

ステップ2信号の記録

ここではすべてが簡単です。HDSDRを開いて、信号の多い領域を選択し、IQ記録を行います。イリジウムの信号は常に送信されるのではなく、「パケット」で送信されるため、少し待たなければならない場合があります。欠点は1つだけです。2MHzの帯域幅で1分間という大量の録音には約500 MBかかります。

「通常の」Linuxではこれは必要ないことをもう一度繰り返します。中間レコードのないSDRレシーバーでgr-iridiumをすぐに実行できますが、これはWindowsでは機能しませんでした-lsusbは接続されたデバイスを表示しません。

ステップ3処理

HDSDRでの記録を停止し、フォルダ/ mnt / c / Users / XXX / Documents / HDSDRでUbuntuに切り替えて、次のコマンドを入力します。

iridium-extractor -c 1619XXXXXX -r 2000000 -f sc16 --offline HDSDR_20200112_110653Z_1619XXXkHz_RF.wav | grep "A:OK"

ここで、HDSDR_20200112_110653Z_1619XXXkkHz_RF.wavは記録したファイル、1619XXXXXXは中央記録周波数、2,000,000は記録したwavファイルの帯域幅です。すべてが正しく行われている場合は、次のように表示され



ます。パッケージが表示されない場合、さらに進んでも意味がありません。エラーを調べる必要があります。レコードが表示されている場合は、コマンドを再度繰り返し、結果をファイルに保存し、iridium-parser.pyを使用してファイルを処理します。

iridium-extractor -c 1619XXXXXX -r 2000000 -f sc16 --offline HDSDR_20200112_110653Z_1619XXXkHz_RF.wav | grep "A:OK" > output.bits

python iridium-parser.py output.bits > output.parsed

これで、保存されたoutput.parsedファイルから情報を抽出できます。そこから、テキストや音声メッセージなどのさまざまなデータを「取得」できます。法的な理由により、ここではこれを行いません。誰でもプロジェクトページの詳細を読むことができます。たとえばイリジウムでは、テキストまたは音声データだけでなく、GSMデータ、つまりイリジウム端末は、通常のカバレッジがない場所でGSM通信サービスを提供できます。ここでの衛星と端末は、おそらく標準のGSMパケットの「延長コード」としてのみ機能します。
