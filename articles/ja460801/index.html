現代のJavaScript開発者向けの27の素晴らしいVS Codeツール

Visual Studio Code（VS CodeおよびVSCodeとも呼ばれます）は、デスクトップアプリケーションの形式で作成された、費用対効果の高い強力なクロスプラットフォームコードエディターです。本日の翻訳を公開している資料の著者によると、VS CodeはTypeScriptやChrome デバッガーなどの多くの開発ツールをサポートしています。これに加えて、信じられないほど多くのオープンソース拡張機能が書き込まれているという事実により、VS Codeは非常に人気があり、多くの人に愛されています。



JavaScriptプログラマーの武器に新しい何かを追加したい場合-資料の作成者は、ここで役立つ何かが見つかることを期待しています。ここで説明する27のツールのすべてが、JS開発のみを目的としているわけではありません。しかし、それらはすべて、JavaScriptで書く人によって正常に適用できます。

1.プロジェクトスニペット


プロジェクトスニペット拡張、すべての時間の私のお気に入りは、に基づいており、ユーザースニペット VSコードに組み込まれたツール。 

この標準のVSコード機能に慣れていない場合は、ユーザーがいわゆるスニペットの形式で独自のコードフラグメントを設計できることを知っておいてください。これは、プロジェクトでそれらを再利用するために行われます。再利用とはどういう意味ですか？

例を考えてみましょう。同じ定型コードを頻繁に作成する必要があると仮定します。例えば-これ：

import { useReducer } from 'react'

const initialState = {
  //
}

const reducer = (state, action) => {
  switch (action.type) {
    default:
      return state
  }
}

const useSomeHook = () => {
  const [state, dispatch] = useReducer(reducer, initialState)
  return {
    ...state,
  }
}

export default useSomeHook

同様のコードをスニペットの形式で発行で​​きます。その結果、キーボードからすべて入力する（またはどこかからコピーする）代わりに、いわゆるプレフィックスを入力するだけで済みます。その後、このプレフィックスに対応するコードがエディターに表示されます。

VS Codeでメニューに移動すると、File > Preferences > User Snippetsオプションでコマンドを選択して新しいグローバルスニペットファイルを作成できますNew Global Snippets File。

たとえば、TypeScriptを使用するReactプロジェクトの新しいスニペットファイルを作成するには、コマンドNew Global Snippets Fileを選択してファイル名をに設定しますtypescriptreact.json。エディターでは、新しく作成された.jsonファイルが開き、スニペットを記述できます。

上記のコードのフラグメントに基づいてスニペットを作成します。このファイルについてtypescriptreact.json 次のフォームに持ってくる必要があります：

{
  "const initialState = {}; const reducer = (state, action)": {
    "prefix": "rsr",
    "body": [
      "const initialState = {",
      "  //$1",
      "}",
      "",
      "const reducer = (state, action) => {",
      "  switch (action.type) {",
      "    default:",
      "      return state",
      "  }",
      "}"
    ]
  }
}

ファイルにスニペットの説明が含まれるようになったので、拡張子.tsxを付けて新しいTypeScriptファイルを作成し、その中にプレフィックスを入力できますrsr。これにより、スニペットの説明を含むポップアップが表示されます。をクリックTabすると、スニペットをファイルに挿入する手順が完了します。次のコードで表されます。

const initialState = {
  //
}

const reducer = (state, action) => {
  switch (action.type) {
    default:
      return state
  }
}

グローバルスニペットを使用する際の不便な点は、開発中のすべてのプロジェクトで利用できることです（ただし、場合によっては、汎用スニペットの場合、これは逆に便利です）。ただし、一部のプロジェクトは特別な方法で構成できます。その結果、そのようなプロジェクトで使用されるものがすべてのプロジェクトで必要ない場合、グローバルスニペットファイルは、再利用可能なコードのフラグメントの処理の問題に対する最良の解決策ではない可能性があります。

プロジェクトが特別なフォルダー構造を使用するとします。このプロジェクトの作業を簡略化するためtypescriptreact.jsonに、次のコードに似たものをファイルに用意することができます。

{
  "import Link from components/common/Link": {
    "prefix": "gcl",
    "body": "import Link from 'components/common/Link'"
  },
  "border test": {
    "prefix": "b1",
    "body": "border: '1px solid red',"
  },
  "border test2": {
    "prefix": "b2",
    "body": "border: '1px solid green',"
  },
  "border test3": {
    "prefix": "b3",
    "body": "border: '1px solid magenta',"
  }
}

最初のスニペットに注意してください。特定のフォルダとファイル構造を持つプロジェクトに適しているかもしれません。しかし、コンポーネントLinkが同様のパスに沿って配置されている別のプロジェクトで作業している場合はどうcomponents/Linkでしょうか。

他のスニペットを見てみましょう。それらは単一引用符を使用します。それらを使用すると、のようなコードに構文が表示されますborder: '1px solid red'。

このような構造はJavaScriptで完全に使用できます。しかし、スタイル付きコンポーネントライブラリをプロジェクトのスタイルを設定するツールとして使用するとどうなるでしょうか。この場合、そのような構文は適していません。確かに、styled-componentsは通常のCSS構造を使用します！

厳密に言うと、Project Snippets拡張機能がここで役立ちます。

この拡張機能により、プロジェクトまたはワークスペースレベルで動作するスニペットを作成できます。そのようなスニペットは、他のプロジェクト用に作成されたスニペットと競合せず、それらの操作を妨害しません。とても快適です。
