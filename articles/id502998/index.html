Kami membuat plugin layanan pemerintah berfungsi untuk semua pengguna di lingkungan terminal


Selama lebih dari satu tahun sekarang, semua karyawan kami hanya bekerja dalam aplikasi yang diterbitkan, dan kami memusatkan semua ini melalui Parallels RAS. Kami juga memiliki mekanisme otomatis untuk menerbitkan EDS: jika pengguna yang berwenang meluncurkan situs, katakanlah bea cukai, maka EDS perusahaan akan dituliskan di HKCU-nya terlebih dahulu dan plug-in yang diperlukan diluncurkan. Ini bekerja sangat baik dengan KONTUR, VLSI, Crypto-PRO, tetapi plug-in dari layanan publik (IFCPlugin) harus diselesaikan dengan file, tetapi ada pertanyaan untuk pengembang ...


Ketika saya menerima aplikasi untuk menginstal plug-in lain yang bertuliskan EDS, saya tidak mengharapkan sesuatu yang rumit. Untuk bekerja dengan EDS, kami telah mengalokasikan host RDS terpisah tempat beberapa plug-in telah berdiri dan semuanya bekerja dengan sempurna. Saya mengunduh plugin layanan publik dari situs resmi dan tanpa basa-basi memulai instalasi dalam konteks mesin dari shell istimewa:


msiexec -i c:\pathtofile\mypackage.msi ALLUSERS=1

Sepintas, instalasi berhasil. Dalam daftar program yang diinstal pada host ini, "Plugin untuk pengguna sistem e-government" muncul, di bawah akun saya, saya dapat masuk menggunakan tanda tangan digital. Tetapi untuk pengguna lain, plugin tidak berfungsi, seolah-olah itu belum diinstal.
