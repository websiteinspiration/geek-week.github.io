Snort或Suricata。第2部分：Suricata的安装和初始设置

据统计，网络流量每年增长约50％。这导致设备负载增加，尤其是提高了IDS / IPS的性能要求。您可以购买昂贵的专用硬件，但是有一个更便宜的选择-引入一种开源系统。许多新手管理员发现很难安装和配置免费IPS。对于Suricata，这不是完全正确的-您可以安装它，并在几分钟内开始使用一组免费规则来抵制典型的攻击。


Snort或Suricata。第1部分：选择免费的IDS / IPS保护企业网络

为什么需要另一个开放式IPS？


自九十年代末以来，很长一段时间以来，一直被认为是标准的Snort系统已经被开发出来，因此它最初是单线程的。多年来，所有现代芯片都已出现在其中，例如对IPv6的支持，分析应用程序级协议或通用数据访问模块的可能性。

基本引擎Snort 2.X已学会使用多核，但仍保持单线程运行，因此无法最佳利用现代硬件平台的优势。

该问题在系统的第三个版本中得以解决，但是准备了这么长时间，以至于从头编写的Suricata设法出现在市场上。在2009年，他们开始精确地开发它，作为Snort的多线程替代产品，后者具有IPS功能。该代码是根据GPLv2许可证分发的，但是该项目的财务合作伙伴可以访问引擎的封闭版本。系统的第一个版本中出现了一些可伸缩性问题，但是很快就解决了。

为什么选择Suricata？


Suricata具有多个模块（如Snort）：捕获，捕获，解码，检测和输出。默认情况下，捕获的流量会先经过一个流解码，尽管这会给系统带来更多负担。如有必要，可以在设置中划分线程并在处理器之间分配-Suricata针对特定硬件进行了很好的优化，尽管对于初学者而言，这不再是HOWTO级别。还值得注意的是，Suricata具有基于HTP库的高级HTTP检查工具。它们也可以用于记录流量而无需检测。该系统还支持IPv6解码，包括IPv4-in-IPv6，IPv6-in-IPv6隧道等。

可以使用不同的接口来拦截流量（NFQueue，IPFRing，LibPcap，IPFW，AF_PACKET，PF_RING），并且在Unix套接字模式下，可以自动分析另一个嗅探器捕获的PCAP文件。此外，Suricata的模块化体系结构使连接新元素变得容易，以捕获，解码，分析和处理网络数据包。还需要注意的是，在Suricata中，流量是通过操作系统的标准过滤器阻止的。 GNU / Linux上有两个IPS选项可用：通过NFQUEUE队列（NFQ模式）和通过零拷贝（AF_PACKET模式）。在第一种情况下，进入iptables的数据包被发送到NFQUEUE队列，在该队列中可以在用户级别进行处理。 Suricata根据自己的规则运行它，并做出以下三个判定之一：NF_ACCEPT，NF_DROP和NF_REPEAT。不需要前两个说明，最后一个说明允许您标记数据包并将其发送到当前iptables表的开头。 AF_PACKET模式的特点是速度较高，但对系统施加了许多限制：它必须具有两个网络接口并用作网关。阻塞的数据包根本不会转发到第二个接口。

Suricata的一个重要功能是可以使用Snort开发的功能。管理员尤其可以访问Sourcefire VRT和OpenSource Emerging Threats规则集以及商业Emerging Threats Pro。可以使用流行的后端来分析统一输出，并且还支持输出到PCAP和Syslog。系统设置和规则存储在YAML文件中，这些文件易于阅读并且可以自动处理。 Suricata引擎可识别许多协议，因此无需在规则中绑定端口号。另外，在Suricata规则中积极实践了流位的概念。为了跟踪响应，会话变量用于创建和应用各种计数器和标志。许多IDS将不同的TCP连接视为单独的实体，并且可能看不到它们之间的连接，这表明攻击开始了。 Suricata尝试查看整体情况，并且在许多情况下会识别出分布在不同连接上的恶意流量。您可以长期谈论其优势，我们最好继续进行安装和配置。

如何安装？


我们将在运行Ubuntu 18.04 LTS的虚拟服务器上安装Suricata。所有命令必须以root（根）身份运行。最安全的选择是使用普通用户的权限通过SSH连接到服务器，然后使用sudo实用程序增加特权。首先，您需要安装我们需要的软件包：

sudo apt -y install libpcre3 libpcre3-dev build-essential autoconf automake libtool libpcap-dev libnet1-dev libyaml-0-2 libyaml-dev zlib1g zlib1g-dev libmagic-dev libcap-ng-dev libjansson-dev pkg-config libnetfilter-queue-dev geoip-bin geoip-database geoipupdate apt-transport-https

我们连接一个外部存储库：

sudo add-apt-repository ppa:oisf/suricata-stable
sudo apt-get update

安装Suricata的最新稳定版本：

sudo apt-get install suricata

如有必要，请编辑配置文件名称，将默认的eth0替换为服务器外部接口的实际名称。默认设置存储在文件/ etc / default / suricata中，用户设置存储在/etc/suricata/suricata.yaml中。设置IDS在很大程度上仅限于编辑此配置文件。它有很多参数，其名称和目的与Snort的参数一致。但是，语法完全不同，但是与Snort配置相比，该文件更易于阅读，并且注释良好。

sudo nano /etc/default/suricata



和

sudo nano /etc/suricata/suricata.yaml



注意！在开始之前值得检查vars部分中变量的值。

要完成配置，您将需要安装suricata-update来更新和下载规则。要做到这一点很简单：

sudo apt install python-pip
sudo pip install pyyaml
sudo pip install <a href="https://github.com/OISF/suricata-update/archive/master.zip">https://github.com/OISF/suricata-update/archive/master.zip</a>
sudo pip install --pre --upgrade suricata-update

接下来，我们需要运行suricata-update命令来设置“新兴威胁开放”规则集：

sudo suricata-update



要查看规则源列表，请运行以下命令：

sudo suricata-update list-sources



更新规则的来源：

sudo suricata-update update-sources



我们重新访问更新的资源：

sudo suricata-update list-sources

如有必要，您可以启用可用的免费资源：

sudo suricata-update enable-source ptresearch/attackdetection
sudo suricata-update enable-source oisf/trafficid
sudo suricata-update enable-source sslbl/ssl-fp-blacklist

之后，您需要再次更新规则：

sudo suricata-update

到此为止，可以认为Ubuntu 18.04 LTS中Suricata的安装和初始配置已完成。最有趣的部分开始：在下一篇文章中，我们将通过VPN将虚拟服务器连接到办公室网络，并开始分析所有传入和传出流量。我们将特别注意阻止DDoS攻击，恶意软件活动以及尝试利用可从公共网络访问的服务中的漏洞。为了清楚起见，将对最常见类型的攻击进行建模。
