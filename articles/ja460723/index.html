NVIDIA Jetson Nano：テストと第一印象

こんにちは、ハブル。

比較的最近、この2019年に、NVIDIA は、Raspberry Piフォームファクターと互換性のある、シングルボードコンピューターを発表しました。



それが発売された後、それがどのように機能し、何ができるのかを見るのが面白くなりました。標準のベンチマークを使用することはそれほど面白くないので、独自のベンチマークを考え出します。すべてのテストについて、ソースコードはテキストで示されています。何が起こったかに興味がある人のために、カットの下で続けました。

ハードウェア


手始めに、NVIDIA Webサイトの技術仕様：



興味深い点から、ここにいくつかの点があります。

1つ目は、それぞれ128コアのGPUで、ボード上でCUDA（サポートされ、そのままインストールされる）やTensorflowなどのGPU指向のタスクを実行できます。メインプロセッサは4コアであり、以下に示すように、非常に優れています。 CPUとGPU間で共有される4GBのメモリ。

2つ目は、Raspberry Piとの互換性です。このボードには、さまざまなインターフェイス（I2C、SPIなど）を備えた40ピンコネクタがあり、Raspberry Piと互換性のあるカメラコネクタもあります。多数の既存のアクセサリ（画面、モーターコントロールボードなど）が機能すると想定できます（Jetson Nanoのサイズがまだ異なるため、延長ケーブルを使用する必要がある場合があります）。

第3に、ボードには2つのビデオ出力、ギガビットイーサネットとUSB 3.0があります。 Jetson Nanoは全体として、プロトタイプよりも少しだけ機能的です。 5V電源、マイクロUSB経由、またはビットコインのマイニングに推奨される別のコネクタ経由で取得可能リソースを大量に消費するタスク。 Raspberry Piの場合と同様に、ソフトウェアはSDカードからロードされ、そのイメージは事前に記録されている必要があります。一般的に、イデオロギー上、このボードは、NVIDIAで考案されたRaspberry Piに非常に似ています。しかし、ボード上にWiFiはありません。確かなマイナスがあります。USB-WiFiモジュールを使用する必要があるでしょう。

よく見ると、デバイスは構造的に2つのモジュールで構成されていることがわかります。JetsonNanoモジュール自体と、コネクタ付きの下部ボードで、接続はコネクタを介して行われます。



それら。ボードを切り離して個別に使用できるため、組み込みソリューションに便利です。

価格といえば。米国でのJetson Nanoの元の価格は99ドル、地元の店で値上げされたヨーロッパでの価格は約130ユーロです（割引が適用される場合は、おそらくもっと安く見つけることができます）。ロシアでのナノの費用はいくらか不明です。

上記のように、ダウンロードとインストールはRaspberry Piと大差ありません。EtcherまたはWin32DiskImagerを介してSDカードにイメージをロードし、Linuxにアクセスして、必要なライブラリを配置します。優れたステップバイステップガイドはこちらです。使用しました。すぐにテストに移りましょう。Nanoでさまざまなプログラムを実行して、それらがどのように機能するかを見てみましょう。比較のために、私は3台のコンピューターを使用しました。私の仕事用ラップトップ（Core I7-6500U 2.5GHz）、Raspberry Pi 3B +、およびJetson Nanoです。

計算のために、単純なものから始めましょう。ただし、プロセッサー時間を必要とします。たとえば、数Piを計算します。私は簡単なPythonプログラムを使ってstackoverflow。

スポイラーの下のソースコード
import time

# Source: https://stackoverflow.com/questions/9004789/1000-digits-of-pi-in-python

def make_pi():
    q, r, t, k, m, x = 1, 0, 1, 1, 3, 3
    for j in range(10000):
        if 4 * q + r - t < m * t:
            yield m
            q, r, t, k, m, x = 10*q, 10*(r-m*t), t, k, (10*(3*q+r))//t - 10*m, x
        else:
            q, r, t, k, m, x = q*k, (2*q+r)*x, t*x, k+1, (q*(7*k+2)+r*x)//(t*x), x+2

t1 = time.time()

pi_array = []
for i in make_pi():
    pi_array.append(str(i))

pi_array = pi_array[:1] + ['.'] + pi_array[1:]
pi_array_str = "".join(pi_array)

print("PI:", pi_array_str)
print("dT:", time.time() - t1)


予想通り、プログラムは速く動作しません。 Jetson Nanoの結果：0.8c。



Raspberry Pi 3B +は、3.06cという大幅に長い時間を示しました。 「模範的な」ラップトップは0.27秒でタスクを完了しました。一般的に、GPUを使用しなくても、Nanoのメインプロセッサはそのフォームファクターに非常に適しています。 Raspberry Pi 4でご確認いただける方はおりません。

確かに、Pythonはそのような計算に最適ではないというコメントを書きたいという人もいます。時間を比較することが重要であり、最小化する必要はないということをもう一度繰り返します。 Pi数をはるかに速く計算するプログラムがあることは明らかです。

ピクダ

GPUを使用したより興味深い計算に移りましょう。もちろん（ボードはNVIDIAからのものです）、CUDAを使用します。PyCUDAライブラリはインストール中にいくつかのシャーマニズムを必要とし、それはcuda.hを見つけませんでした、コマンド「sudo env」PATH = $ PATH「pip install pycuda」の使用は助けとなりました、おそらく別の方法があります（より多くのオプションがdevtalk.nvidia.comフォーラムで議論されました）。

テストでは、単純なプログラムSimpleSpeedTest for PyCUDAを使用しました。このプログラムは、ループ内の正弦をカウントするだけで、何の役にも立ちませんが、評価することは可能であり、コードは単純で明確です。

スポイラーの下のソースコード
# SimpleSpeedTest.py
# https://wiki.tiker.net/PyCuda/Examples/SimpleSpeedTest

import pycuda.driver as drv
import pycuda.autoinit
from pycuda.compiler import SourceModule
import numpy
import time


blocks = 64
block_size = 128
nbr_values = blocks * block_size
n_iter = 100000

print("Calculating %d iterations" % (n_iter))
print()

#####################
# SourceModule SECTION

# create two timers so we can speed-test each approach
start = drv.Event()
end = drv.Event()

mod = SourceModule("""__global__ void gpusin(float *dest, float *a, int n_iter)
                      {
                          const int i = blockDim.x*blockIdx.x + threadIdx.x;
                          for(int n = 0; n < n_iter; n++) {
                            a[i] = sin(a[i]);
                          }
                          dest[i] = a[i];
                      }""")

gpusin = mod.get_function("gpusin")

# create an array of 1s
a = numpy.ones(nbr_values).astype(numpy.float32)
# create a destination array that will receive the result
dest = numpy.zeros_like(a)

start.record() # start timing
gpusin(drv.Out(dest), drv.In(a), numpy.int32(n_iter), grid=(blocks,1), block=(block_size,1,1) )
end.record() # end timing
# calculate the run length
end.synchronize()
secs = start.time_till(end)*1e-3
print("PyCUDA time and first three results:")
print("%fs, %s" % (secs, str(dest[:3])))
print()

#############
# CPU SECTION
# use numpy the calculate the result on the CPU for reference

a = numpy.ones(nbr_values).astype(numpy.float32)
t1 = time.time()

for i in range(n_iter):
    a = numpy.sin(a)

print("CPU time and first three results:")
print("%fs, %s" % (time.time() - t1, str(a[:3])))


ご覧のとおり、計算はCUDAを介してGPUを使用し、numpyを介してCPUを使用して行われます。

結果：
Jetson nano-0.67c GPU、13.3c CPU。
Raspberry Pi 3B +-41.85c CPU、GPU-データなし、RPi上のCUDAが機能しません。
ノートブック-0.05s GPU、3.08c CPU。

すべてがかなり期待されています。GPUでの計算はCPU（まだ128コア）での計算よりもはるかに高速であり、Raspberry Piは大幅に遅延します。もちろん、オオカミに餌をいくら与えても、ゾウはまだ Jetson Nanoのカードよりもはるかに速いラップトップビデオカードを持っています-はるかに多くの処理コアがある可能性があります。

結論


ご覧のとおり、NVIDIAボードは非常に興味深く、非常に生産的であることがわかりました。それはラズベリーパイよりもわずかに大きく、より高価であり、誰かがB必要があるかどうかについて、コンパクトなサイズでlshayaコンピューティングパワーを、それはそれだけの価値があります。もちろん、これは必ずしも必要ではありません。たとえば、narodmonに温度を送信するには、Raspberry Pi Zeroで十分で、複数のマージンがあります。したがって、Jetson NanoはRaspberryとクローンを置き換えることを主張していませんが、要求の厳しいタスクにとっては非常に興味深いものです（ドローンやモバイルロボットだけでなく、たとえば、顔認識付きのドアベル用のカメラにもなります）。
