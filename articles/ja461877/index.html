JavaとKotlin for Android：開発者の意見

2018 年のスタックオーバーフロー調査では、回答者の45.3％がJavaを最も人気のあるプログラミング言語と呼び、Kotlinは開発者の4.5％だけを好みました。ただし、Google は 2019年5月、優先プログラミング言語として Kotlin を発表しました。同社はC ++とJavaの放棄を求めていませんでしたが、プログラマー間の論争はさらに激しくなりました。

Netologyは、開発者から、使用するプログラミング言語と、JavaとKotlinの将来についてどのように考えているかを学びました。

「メインスレッドはJavaのままで、Kotlinは迅速な開発に使用されます」



ケリーITソリューションのリーディングプログラマー、ドミトリーカチャロフ氏が

知っている言語最初はウェブ開発言語を勉強していましたが、ある時点でモバイル開発もしたいと思っていました。私はAndroidの開発を研究し始め、必要最低限​​のJava言語を習得して、最初のアプリケーションを作成し、電話で実行し、市場で公開できるようにしました。これは、Kotlinが登場したばかりで、幅広い開発者には知られていませんでした。

どちらを使用します。現在、エンタープライズアプリケーションの開発では、Javaを使用しています。多くのライブラリが彼のために利用可能であり、既製のモジュールがあります。さらに、Android Studioはそれをサポートしているので、文字通り1文字を書くことができ、コードは自動的に作成されます。とても快適です。

私の実践では、他の言語の知識を必要とするようなものはありませんでした。多くのライブラリがあり、原則としてあらゆるタスクを実行できるため、通常はJavaで十分です。検討する価値がある唯一のものは、Androidプラットフォーム自体の制限です。言語に関係なく、すべての制限は遅かれ早かれプラットフォームの制限に反するからです。私の意見では、タスクを実装する言語に大きな違いはありません。主なことは、特定のプラットフォーム内で機能する優れたソリューションを見つけることです。

なぜJavaを好むのか。その上で、無料のサービス、独自のAPIメソッド、書き込みサイト、サーバー、モバイルアプリケーションを作成できます。これは基本的に、多数の操作を実行してさまざまなアプリケーションを開発できる汎用言語です。彼には多くの開発とライブラリがあります。そしてまた-常に質問に答えたり助けたりする準備ができている大きなコミュニティ。

言語の対立と未来について。 Kotlinは間違いなく興味深い一歩ですが、私の意見では、新しいアプローチやパラダイムはありません。それは、より短い構成でコードを書くのにかかる時間を単に減らします。同時に、それはまだ同じJavaに基づいています。

近い将来、Javaは依然として最も人気があります。メインスレッドはJavaのままで、Kotlinは単純なアプリケーションのラピッドプロトタイピングまたはプロトタイピングの条件での迅速な開発に使用されます。これは非常に挑発的な視点ですが。KotlinはJavaと互換性があるため、何かが根本的に変わるとは言えません。

「初心者でAndroidを使いたい場合は、Kotlinを学んでください。それ以外の場合は、Javaを選択します」



Artyom Pervushov、ライセンスされたAndroid ATCスペシャリスト、NetologyのAndroid Developerコースの著者、

Javaを好む理由まず、Kotlinが何であるかを理解する必要があります。 Kotlinは構文糖に関するものです。こんなに砂糖が必要ですか？おそらくそうではないでしょう。私はすでにJavaコードを読んでいて、間違いを犯さないことに（ほとんど）慣れています。プログラミングの初心者は明らかにより複雑なJavaを選択する必要がありますか？おそらくどちらでもない。

それでもあなたの時間を投資することの利点を考慮する必要があります。 JavaはAndroidとデスクトップだけの話ではありません。その上には他にも膨大な数のプロジェクトがあります。 Androidから他の場所に移動したい場合は、言語を知っていると便利です。

言語の未来について。多分将来的にはコトリンが世界を引き継ぐか、そうでないかもしれません。「世界の捕獲」であるGo、Python、Ruby、JS、<代わりに必要なもの>は、これらのテクノロジーのファンから言われているよりもはるかに一般的ではありません。したがって、私のアドバイスは次のとおりです。もしあなたが初心者であり、あなたの運命をAndroidと結びつけることに固く決心しているなら、Kotlinを学んでください。それ以外の場合は、Javaに焦点を当てます。

「Kotlinの開発者は、Javaフレームワークとライブラリを使用して、より理解しやすく単純なコードを書くことができます。」



Android開発者で、NetologyコースのエキスパートであるSemyon Pilunts氏。

どの言語を知っているか。私のプログラミングの知識は私の研究所で起こりました、私はパスカルから始めました、私はベーシック、Qベーシックも学びました。私はすでに学年の終わりにCを勉強し始めました。そこで止まらず、他の言語を勉強し始めました-Javascript、VB 6.0、MS SQL、Java。彼はすぐにJavaが私に最も近いことに気付いたので、それに集中することにしました。

数年後、偶然、私はJavaを使用してAndroid用のモバイルアプリケーションを作成し始めました。 Android自体でJavascript、HTML、SQLを使用する必要がある場合がありました。他の言語を知ることは常に役立ちます。

どちらを使用します。現在、JavaとKotlinの両方で働いています。これまでのところ、Javaを完全に放棄することはできませんが、徐々に進んでいます。

実際には、タスクを実行する前に、タスクを評価し、アプローチとその方法を選択する必要があります。ここで、どの言語を使用するのが適切かが明らかになります。この場合、対応できない状況はありません。同時に、Kotlinの開発者はJavaフレームワークとライブラリを使用して、より理解しやすく単純なコードを書くことができます。

Kotlinが好む理由 Kotlinの使用は、Javaよりも簡単で便利です。その中で、プログラムコードは平均40％短くなっています。また、プログラムの実行中に発生する可能性のあるエラーを防ぐこともできます。コードがよりシンプルで理解しやすいものになると、エラーの作成が難しくなり、検出が容易になるため、エラーの数は急速に減少します。開発とテストに費やす時間を短縮できます。

言語の未来について。近い将来、KotlinはJavaをほぼ完全に置き換えることができると思います。すでにAndroid市場を50％カバーしており、登場からわずか3年。しかし、それでも、JavaはそれがなければコードをKotlinに転送するのが難しいでしょう。ここで、KotlinはJavaと完全に互換性があることに注意してください。つまり、アプリケーションをすぐに完全に書き直す必要はまったくありません。多くの場合、これを徐々に行う方が便利です。

そして、Androidアプリケーションの開発だけがKotlinの方向性ではありません。JavaScriptをサポートし、プログラマーはフロントエンドを転送できます。Kotlinはサーバーアプリケーションの開発に最適で、既存のJavaテクノロジーベースのスタックとの完全な互換性を維持しながら、短くて表現力のあるコードを記述できます。また、開発者がGradleファイルを作成できるように、作成者はGradleサポートを追加しました。積極的にはKotlin Nativeの開発であり、これを使用してiOSアプリケーションを作成できます。

「Kotlinは一般的にJavaよりも優れていませんが、Androidの世界では優れています」



AIMSの共同創設者で、NetologyのKotlin Developerコースの著者であるIlnaz Gilyazovは、

彼が知っている言語を教えています。最初は、必要に応じていくつかの言語を学びました：アセンブラー、C / C ++、次にJava、JS、PHP、Python。私は情報セキュリティの分野で働いていたので、1つの言語で止まることはできません。

どちらを使用します。今では、お客様のタスクや好みに応じてツールを選択しています。バックエンドの場合-Java / Kotlin、多くの統合なしの軽量のものの場合-Go / NodeJS、フロントエンドの場合-JS / TypeScript +フレームワーク、システムの場合-C ++ / Go。つまり、個人的な好みではなく、言語とそのエコシステムのタスクと機能から始めます。

Kotlinが好む理由多くのコードを必要としないシンプルで便利なツールを開発者に提供したいという、新しい見方によるKotlin言語の利点。これらは構文糖とコルーチンです。 Kotlinを使用すると、より少ないコードを記述できます-喜ばれます。

Androidでは、Javaの代わりにKotlinを選択するのは明らかです。バックエンドの世界では、Kotlinの立場ははるかに弱くなっていますが、彼はそこにいます。 Spring Framework（多くの場合、バックエンドの標準です）には、Javaでは使用できないKotlin用のチップが含まれています。これはすでに多くのことを言っています。大きく複雑なバックエンドでは、一部の場所でKotlinを使用していますが、Javaで作業しています。特定のマイクロサービスが多数の統合を必要としないマイクロサービスアーキテクチャの場合、Kotlinはすばらしいと感じています。

Kotlinは一般的にJavaよりも優れていませんが、Androidの世界では優れています。これも議論の余地はありますが、すべて評価に使用するパラメーターによって異なります。

言語の対立と未来について。 KotlinとJavaの開発者間の対立は、一般的な合意がなく、両方の言語を使用するプロジェクトで時々見られます。そして、唯一の正しい言語を信じる狂信者がいます。それ以外の場合は、統合の問題に粗さが残っていますが、すべてが多かれ少なかれスムーズです。 Androidのプロになるには、両方の言語を知っている必要があります。とにかく、常に新しい言語、ライブラリ、フレームワーク、アイデア、アプローチを学ぶ必要があるという事実に慣れる価値はあります。

現在、AndroidにおけるJavaの将来は不透明です。すべてのAndroidプロジェクトをKotlinに転送したところ、AndroidのJavaは7番目のレベルでスタックし、8番目からいくつかのチップが出ました。また、GoogleがOracleとの紛争を解決し、新しいバージョンのサポートを導入する予定があるのか​​、それとも現在の立場に留まるのかは依然として不明です。したがって、AndroidでのJavaの学習、使用、サポートへの投資は、少なくとも中期的にはリスクが高いようです。

一方、Kotlinは多かれ少なかれ予測可能に開発を行い、言語のすべての最新機能を使用できるラッパーを提供し、Android向けの開発に特化した多くの快適さと機能を追加します。これは、開発者の満足度に影響を与える重要な要素です。

近い将来、KotlinがJavaに取って代わるとは思わない。本来、これを指向していたため、Javaと共存できます。既存の人気のあるJavaライブラリのほとんどがKotlinで書き直されることを期待する価値はありません。しかし、Kotlin（Kotlin <-> Java統合のニュアンス、たとえばリフレクションとすべての落とし穴を理解している）で記述し、Javaで記述されたライブラリを使用できます。

「現在、JavaのみまたはKotlinのみの知識を持つ開発者を探している人はいません。普遍的なスペシャリストである必要があります」



RoadARのAndroid開発者、Dmitry Gordin氏

の対決について。宣伝を繁殖させず、個人的な感情に基づいていないために、主要な市場プレーヤーを見てみましょう。まず、GoogleはKotlinをAndroidの主要な開発言語として認識しています。第二に、これはトップ企業のHH.ruの求人がどのように見えるかです。

  • Tinkoff-「重要なプラス：Kotlinでのアプリケーション開発の経験」;
  • Yandex-"ようこそ：Kotlinでの開発経験";
  • Redmadrobot-「望ましい：Kotlin開発経験」;
  • ロシアのポストさえ-「私たちはあなたから次の知識と能力を期待しています：Android SDKテクノロジースタック、Kotlin / Javaの知識。」

言語の選択について。両方の言語を知っている必要があります。Kotlinを知り、Javaを知らないことは不可能です。退屈なリーグは、障害を見つけて、Javaに関係のないKotlinネイティブが存在することに気付く場合があります。ただし、ここではAndroid開発、つまりDalvik / ARTランタイムについて説明します。したがって、Kotlinの開発者を探していると書いているすべての雇用主は、実際には両方の言語を探しています。

私の意見では、最良の方法はJavaに慣れてからKotlinにスムーズに切り替え始めることです。Kotlinは、Javaからあなたが知っているすべてを逆さまにするわけではありません。最小限の変更で新しい言語に切り替え、「Javaスタイル」で記述し、コードを徐々にコード化し、それに砂糖とKotlin固有のチップを追加できます。
