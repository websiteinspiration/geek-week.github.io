産業オートメーションの安全性を向上させる方法


Intel Security CorporationのIoT Security Solutions GroupのチーフアーキテクトであるSven Schreckerはかなり有名な人物であるため、業界の情報セキュリティに関する彼の最近の記事の1つを翻訳することにしました。読書を楽しむ！

産業オートメーションの歴史は私にとって非常にエキサイティングなようです。新しいテクノロジーは、産業時代から生産プロセスをピックアップし、それらを情報化時代へと飛躍させました。生産性がすでにピークに達しているように思われたとき、インターネットは生産効率を想像を絶するレベルに上げるのに役立ちました。 

残念なことに、産業用オートメーションは、データ主導でインターネットに接続された現代の世界に突入し、デジタルセキュリティを押しのけました。 

仮想空間の匿名のハッカーが産業オートメーションシステムに侵入できるデジタル時代へようこそ。これらのシステムのセキュリティを確保しながら、同時にビジネスのニーズを満たす方法は？ 

運用技術（OT）チームには、高い復元力と可用性が依然として必要です。情報技術（IT）チームは相互接続され、安全で、規制に準拠する必要があります。これらのチームには両方とも、新しい専門家が関与する必要があります。つまり、ビジネス上の決定ごとにリアルタイムでデータを収集、交換、分析する必要のあるデータアナリストです。この記事では、最新の産業オートメーションシステムのセキュリティ状況、その改善の技術的および組織的な問題、および産業用制御システム（ICS）と制御およびデータ収集システム（SCADA）にセキュリティと信頼を実装するための動的モデルについて説明します。

データへの攻撃


データの収集と分析は、現代の競争の武器です。それらに基づいて、ビジネスの各領域のプロセスを最適化する分析的アイデアが生成されます。メーカーは、効率を10〜20％向上させるために、数億ドルを投資することがよくあります。消費者の使用と需要の分析、調達、サプライチェーンの最適化、生産プロセス、生産計画などの結果として得られたデータ（およびそれらからの結論）により、効率が向上します。このデータを注意深く変更することにより、攻撃者は気づかない会社のプロセスを最適化できなくなります。最も簡単なデータ操作でさえ、常に均衡の危機に瀕しているビジネスに害を及ぼす可能性があります。

セキュリティの神話と誤解


多くの神話と誤解が産業オートメーション安全システムの開発を妨げました。最も一般的なものは次のとおりです。

主な要件


セキュリティシステムの統合モデルは、産業オートメーションシステムの信頼性の高い相互接続を確立および保証します。このモデルには3つの基本的な要件があります。

  • 安全なデバイス（情報セキュリティ機能が組み込まれています）
  • 安全な通信（デバイス間の通信プロトコル）
  • セキュリティの監視と管理（イベントとインシデントへの対応）

保護されたデバイス


信頼の連鎖の作成は、デバイス認証から始まります。 IPアドレスやMACアドレスを使用するなど、デバイスIDを確認する以前のアプローチは信頼できません。IPアドレスは定期的に変更され、ハッカーによって非常に簡単に改ざんされる可能性がありますが、MACアドレスは簡単にリセットできます。したがって、デバイス認証は、ハードウェア内のプロセッサの物理レベルで開始する必要があります。信頼できるテクノロジを使用して、セキュリティを強化できます。たとえば、組み込みのセキュリティコプロセッサは、不正アクセスから保護されたハードウェアコンテナに暗号化キーを格納するように設計された特別なマイクロプロセッサです。

これにより、チップ自体が、ブートプロセス、オペレーティングシステム、仮想マシン、またはアプリケーションの信頼レベルの測定などの暗号化操作を実行できます。このプロセスの鍵は、コード、データ構造、構成、情報、またはメモリにロードできるあらゆるものの正確な測定です。測定中、暗号化ハッシュは安全なハッシュアルゴリズムを使用して計算されます。整合性をチェックし、コード、構成、またはデータの変更（または損傷）を検出できます。

ディスクにあるソフトウェアにも同様のスキームが適用され、メモリにロードされて実行される前に改ざんされていないかどうかが確認されます。信頼チェーンは、データが暗号化されてクラウドに転送されている場合でも、ダウンロードプロセス中やシステム全体を含め、ソフトウェアスタック全体を通じて整列し、チェックされ続けます。信頼できるデバイス、データ、およびプログラムを入手することは、特にマシン間通信の優位性を考えると、産業オートメーションでは重要です。たとえば、信頼できるデバイスは、信頼できる産業用監視センサーによって受信されたデータに電子的に署名できます。ハッカーがデータを操作すると、データ署名は不正確になり、監視システムによってマークが付けられます。この場合、データの信頼できない部分と、変更が発生したマシンまたはセンサーがインストールされます。

安全な接続


トランザクション用の信頼できるスペースは、許可されたビジネスコミュニケーションを可能にする領域です。デバイスは、各ゾーン内のデータの信頼性と整合性を確保する必要があります。信頼できるゾーン間の通信を可能にする2つのセキュリティ技術：ユーザーがオフィスからクラウドにデータを安全に集約、フィルタリング、交換できるインテリジェントセキュリティゲートウェイ。どこでも安全かつ確実にアプリケーションデータを実行できる信頼できるランタイム。

インテリジェントゲートウェイ：過去と未来をつなぐ


レガシーシステムが産業オートメーションで非常に一般的である理由があります。それらは機能します。それらのいくつかは何十年もの間改善されてきました。新しいクラスのスマートゲートウェイ（わずか2インチx 2インチ）を使用して、次世代のスマートインフラストラクチャに接続することにより、レガシーシステムを拡張します。これらのゲートウェイは、レガシーシステム、生産エリア、および外界を物理的に分離し、産業オートメーションシステムの攻撃面を制限します。

このようなゲートウェイは古いデバイスを変更せずに保護できるため、産業環境で一貫したレベルのセキュリティを作成するための魅力的な初期セキュリティソリューションになります。他の安全なデバイスと同様に、セキュリティゲートウェイは安全に起動し、ネットワークで認証し、その背後にあるデバイスに代わって任意の数のセキュリティおよび通信タスクを実行する必要があります。整合性計算のチェック、証明書の検証、暗号化の適用、および信頼できる通信チャネルの確立により、信頼ゾーンをリンクする準備ができます。ゲートウェイには、それらが接続されている本番システムを管理するためのプロトコルも含まれる場合があり、これらのシステムの寿命が延びる可能性があります。機器配置の信頼できる領域を実際に訪問することなく、修理と更新を実行できます。

信頼できるランタイムは、悪意のあるコードの実行を防ぐことでセキュリティを強化します。仮想化および暗号化テクノロジーを使用して、アプリケーションとデータ用の安全なコンテナーを作成します。承認されたデバイスのみがこのようなコンテナで動作します。これらの環境は安全で信頼できる領域であり、データを不正アクセスから保護し、データやアプリケーションを第三者から見えなくして、機密情報の漏洩の可能性を防ぎます。

不明なベンダーによって管理されている仮想マシンであっても、信頼できるランタイムはデータを認証してデジタル署名し、その後そのデータの整合性を検証できます。たとえば、保存および処理する産業オートメーションシステムからの生産データクラウドサービスプロバイダーは、なりすましや変更から保護されます。

セキュリティの監視と管理


古いITのことわざは、あなたがコントロールできないものはコントロールできないということです。分散型産業オートメーションシステムの効果的な監視には、エンタープライズデバイスの集中管理と、すべてのデバイスのイベント情報を監視、収集、分析する機能が必要です。これは、システム全体の横断的な状況認識を提供します。

エンタープライズセキュリティ管理コンソール


管理コンソールにより、ITスタッフは高度に開発されたグローバルな分散環境を管理および監視できます。管理コンソールから、デバイスのソフトウェアがリモートで制御および更新され、デバイスのセキュリティポリシーが決定および調整されます。たとえば、信頼できるアプリケーション、データ、通信、およびデバイスで実行できるその他の機能のリストを定義するリモートデバイスのホワイトリストポリシーの管理。

企業のコーポレートガバナンスコンソールは、情報保護およびイベント監視（SIEMシステム）やその他のセキュリティモジュールのソリューションと密接に統合する必要があります。高レベルの統合により、複雑さが大幅に軽減され、状況認識が向上し、管理の時間とコストが削減されます。さらに、エンタープライズ管理コンソールと使用するSIEMのスケーラビリティが重要になってきています。

セキュリティ情報とイベント監視


SIEMシステムは、産業オートメーションシステム内のすべての管理対象デバイスからセキュリティイベントを収集、統合、比較、評価、および優先順位付けします。SIEMを使用すると、すべてのイベントの状況的および状況的な認識を維持し、基本的な傾向と異常を特定できます。行動特性は、作業の正常なパターンと異常なパターンを区別し、ポリシーを改善して、誤検知を最小限に抑えるのに役立ちます。SIEMデータはフォレンジックにとっても重要であり、セキュリティインシデントやデバイス障害をよりよく理解することができます。

生態系の創造


現代の産業オートメーションシステムの分散した相互接続された性質を考えると、完全な安全性を達成するには、一度に多くのベクトルで作業する必要があります。業界および重要インフラストラクチャ（OEM）の機器メーカーは、相互運用性を確保し、オープンスタンダードを設定し、アプリケーションプログラミングインターフェイスを定義するために、企業のセキュリティプロバイダーと積極的にコンソーシアムを形成しています。新しいシステムと産業用制御デバイスはゼロから構築され、後方互換性と前方互換性を提供するセキュリティテクノロジーを使用して開発されます。
