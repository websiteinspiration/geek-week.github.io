Pythonユニットテスト：クイックスタート

記事の翻訳は、Python QAエンジニアコースの学生を対象に特別に作成されました。



ユニットコードテストは、ソフトウェア開発ライフサイクルの不可欠な部分です。単体テストは、回帰テストを実施するための基礎も形成します。つまり、ユニットテストは、新しい機能が追加された場合や既存の機能が変更された場合に、システムがシナリオに従って動作することを保証します。

この記事では、1つのクラスでの単体テストの基本的な考え方を示します。実際には、多くのテストケースを作成し、それらをテストスイートに追加して、すべてを一緒に実行する必要があります。テストケースの管理については、次の記事で説明します。

今日は、バックエンドのテストに焦点を当てます。つまり、開発者は仕様（Calculator.pyなど）に従ってプロジェクトを実装し、開発したコードが実際にそれらに一致することを確認する（たとえば、を使用するTestCalculator.py）ことがタスクです。

Calculatorクラスを作成して、加算、減算、乗算、除算などの基本的な計算機能を実行するとします。

このためのコードは次のCalculator.pyとおりです（）：

#A simple calculator
class Calculator:
  #empty constructor
  def __init__(self):
    pass
  #add method - given two numbers, return the addition
  def add(self, x1, x2):
    return x1 + x2
  #multiply method - given two numbers, return the 
  #multiplication of the two
  def multiply(self, x1, x2):
    return x1 * x2
  #subtract method - given two numbers, return the value
  #of first value minus the second
  def subtract(self, x1, x2):
    return x1 - x2
  #divide method - given two numbers, return the value
  #of first value divided by the second
  def divide(self, x1, x2):
    if x2 != 0:
      return x1/x2

次に、単体テストを実行して、上記のクラスの機能が計画どおりに機能することを理解します。

Pythonは通常、パッケージに付属していますunittest。システムにない場合は、pipを使用してインストールします。

：ユニットテストは、以下の構造を有し、



setUp()そしてtearDown()これらは、ユニットテストフレームワーク（それらがたunittest.TestCaseクラスで定義されている）に付属している標準的な方法であります- 。テストケースに応じて、デフォルトでこれら2つのメソッドをオーバーライドする場合とオーバーライドしない場合があります。

テストケースのコードを見てみましょう。こちらがファイルですTestCalculator.py。

import unittest
from Calculator import Calculator
#Test cases to test Calulator methods
#You always create  a child class derived from unittest.TestCase
class TestCalculator(unittest.TestCase):
  #setUp method is overridden from the parent class TestCase
  def setUp(self):
    self.calculator = Calculator()
  #Each test method starts with the keyword test_
  def test_add(self):
    self.assertEqual(self.calculator.add(4,7), 11)
  def test_subtract(self):
    self.assertEqual(self.calculator.subtract(10,5), 5)
  def test_multiply(self):
    self.assertEqual(self.calculator.multiply(3,7), 21)
  def test_divide(self):
    self.assertEqual(self.calculator.divide(10,2), 5)
# Executing the tests in the above test case class
if __name__ == "__main__":
  unittest.main()

これは必須ではありませんが、原則として、接頭辞Test（この場合はTestCalculator）を使用してテストクラスを呼び出します。このクラスの重要な要件は、スーパークラスの存在ですunittest.TestCase。

このテストケースが実行されるたびに、setUp（）メソッドが最初に実行されます。この場合、クラスCalculatorのオブジェクトを作成し、それをクラスの属性として保存するだけです。親クラスには他にもいくつかのデフォルトメソッドがありますが、これについては後で説明します。

ここtest_xxxでは、計算機クラスの各メソッドをテストするメソッドを作成するだけです。すべてのテストメソッドはプレフィックスで始まることに注意してくださいtest_。これは、ユニットテストフレームワークを使用するPythonに、これらがテストメソッドであることを伝えます。

各テストメソッドで、組み込みメソッドを使用してassertEqual、計算機メソッドが期待値を返すかどうかを確認しました。戻り値が期待値と等しい場合、テストは成功し、それ以外の場合は失敗します。後で説明する

組み込みメソッドは多数ありますassert。

上記のコードの最後の行は、テストケースを開始するだけTestCalculatorです。クラス内で定義された各テストメソッドを実行し、結果を返します。

python TestCalculator.py -v
