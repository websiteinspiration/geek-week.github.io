Introduction to Traefik 2.0

Traefik is an open source reverse proxy server that provides easy operation with microservices and / or just containers with your applications.







A reverse proxy server (reverse proxy, reverse proxy) is used to relay requests from an external network to any servers / services on the internal network (for example, a web server, database or file storage) and allows:







  • ensure the concealment of the structure of the internal network and details about the services located in it;
  • perform load balancing between instances of the same service or servers with the same tasks;
  • provide an encrypted (HTTPS) connection between the client and any service, in which case an SSL session is created between the client and the proxy, and an unencrypted HTTP connection is established between the proxy and the service on the internal network; if the service supports HTTPS, an encrypted connection can also be established on the internal network;
  • organize access control to services (client authentication), as well as set up a firewall (firewall).


The article will describe the use of Traefik in Docker as a reverse proxy for other Docker containers, as well as non-containerized services.
