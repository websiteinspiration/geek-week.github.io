1.新規/削除演算子の標準形式


C ++はいくつかの演算子オプションをサポートしていますnew/delete。それらは基本的な標準、追加の標準およびカスタムに分けることができます。このセクションとセクション2では標準フォームについて説明し、カスタムフォームについてはセクション3で説明します。

1.1。基本的な標準フォーム


new/deleteオブジェクトおよびタイプの配列を作成および削除するときに使用される 演算子の主な標準形式はT次のとおりです。

new T(/*   */)
new T[/*   */]
delete ptr;
delete[] ptr;

彼らの仕事は次のように説明できます。演算子が呼び出されるとnew、メモリが最初にオブジェクトに割り当てられます。選択が成功すると、コンストラクターが呼び出されます。コンストラクタが例外をスローすると、割り当てられたメモリが解放されます。演算子が呼び出されると、deleteすべてが逆の順序で行われます。まず、デストラクタが呼び出され、次にメモリが解放されます。デストラクタは例外をスローすべきではありません。

オペレーターがnew[]オブジェクトの配列を作成するために使用され、メモリは最初に配列全体に割り当てられます。選択が成功すると、デフォルトのコンストラクター（または初期化子がある場合は別のコンストラクター）が、ゼロから始まる配列の各要素に対して呼び出されます。コンストラクターが例外をスローした場合、配列の作成されたすべての要素について、コンストラクター呼び出しの逆の順序でデストラクターが呼び出され、割り当てられたメモリが解放されます。配列を削除するには、演算子を呼び出す必要があります。配列のdelete[]すべての要素について、コンストラクタの逆の順序でデストラクタが呼び出され、割り当てられたメモリが解放されます。

注意！正しい形式の演算子を呼び出す必要がありますdelete単一のオブジェクトまたは配列が削除されるかどうかに応じて。このルールは厳密に遵守する必要があります。遵守しないと、未定義の動作が発生する可能性があります。つまり、メモリリーク、クラッシュなど、何でも発生する可能性があります。詳細については、[Meyers1]を参照してください。

上記の説明では、1つの明確化が必要です。いわゆる自明な型（組み込み型、Cスタイルの構造体）の場合、デフォルトのコンストラクターは呼び出されない場合があり、デストラクターは何もしません。

標準のメモリ割り当て関数は、要求を満たすことができない場合、型例外をスローしますstd::bad_alloc。ただし、この例外はキャッチできます。これには、関数呼び出しを使用してグローバルインターセプターをインストールする必要がありますset_new_handler()。詳細については、[Meyers1]を参照してください。

任意の形式の演算子deletenullポインタに安全に適用します。

演算子を使用して配列を作成する場合、new[]サイズをゼロに設定できます。

どちらの形式の演算子でもnew、中括弧で初期化子を使用できます。

new int{42}
new int[8]{1,2,3,4}

1.2。追加の標準フォーム


ヘッダーファイルを接続すると、<new>さらに4つの標準演算子フォームが使用可能になりますnew。

new(ptr) T(/*  */);
new(ptr) T[/*   */];
new(std::nothrow) T(/*   */);
new(std::nothrow) T[/*   */];

それらの最初の2つは、「new非割り当て配置」と呼ばれますnew。引数ptrは、インスタンスまたは配列を保持するのに十分な大きさのメモリ領域へのポインタです。また、メモリ領域には適切な配置が必要です。このバージョンの演算子newはメモリを割り当てず、コンストラクターへの呼び出しのみを提供します。したがって、このオプションを使用すると、メモリ割り当てのフェーズとオブジェクトの初期化を分離できます。この機能は、標準のコンテナで積極的に使用されています。deleteこの方法で作成されたオブジェクトの演算子は、もちろん呼び出すことはできません。オブジェクトを削除するには、デストラクタを直接呼び出し、メモリの割り当て方法に応じた方法でメモリを解放する必要があります。

2番目の2つのオプションは、例外をスローしない演算子new（nothrow new）と呼ばれ、要求を満たすことができない場合に返さnullptrれますが、型例外はスローしませんstd::bad_alloc。オブジェクトの削除は、メインオペレーターを使用して行われますdelete。これらのオプションは廃止されたと見なされており、使用は推奨されていません。

1.3。メモリ割り当てと空き関数


標準形式のオペレーターnew/deleteは、以下の割り振りおよび割り振り解除機能を使用します。

void* operator new(std::size_t size);
void operator delete(void* ptr);
void* operator new[](std::size_t size);
void operator delete[](void* ptr);
void* operator new(std::size_t size, void* ptr);
void* operator new[](std::size_t size, void* ptr);
void* operator new(std::size_t size, const std::nothrow_t& nth);
void* operator new[](std::size_t size, const std::nothrow_t& nth);

これらの関数は、グローバル名前空間で定義されています。ホストステートメントのメモリ割り当て関数newは何もせず、単に戻りptrます。

C ++ 17は、メモリ割り当ておよび割り当て解除関数の追加の形式をサポートし、整列を示しました。それらの一部を次に示します。

void* operator new (std::size_t size, std::align_val_t al);
void* operator new[](std::size_t size, std::align_val_t al);

これらのフォームは、ユーザーが直接アクセスすることはできません。整列要件を超えるオブジェクトに対してコンパイラーが使用するため、ユーザーがフォームを__STDCPP_DEFAULT_NEW_ALIGNMENT__誤って非表示にしないことが主な問題です（セクション2.2.1を参照）。C ++ 11では、ユーザータイプの配置を明示的に設定できるようになったことを思い出してください。

struct alignas(32) X { /* ... */ };

2. new / delete演算子の標準形式のオーバーロード


標準形式の演算子のオーバーロードは、標準のnew/deleteシグネチャと一致するシグネチャを持つメモリを割り当てたり解放したりするためのユーザー定義関数を定義することです。これらの関数は、グローバル名前空間またはクラスで定義できますが、グローバル以外の名前空間では定義できません。標準ホストステートメントのメモリ割り当て関数newは、グローバルネームスペースでは定義できません。このような定義の後、対応する演算子new/deleteは標準の演算子ではなくそれらを使用します。

2.1。グローバル名前空間でのオーバーロード


たとえば、グローバル名前空間のモジュールで、ユーザー定義関数が定義されているとします。

void* operator new(std::size_t size)
{
// ...
}

void operator delete(void* ptr)
{
// ...
}

この場合、実際new/deleteには、モジュール全体の任意のクラス（標準クラスを含む）のすべての演算子呼び出しにメモリを割り当てて解放するための標準関数の置き換え（置換）があります。これは完全な混乱につながる可能性があります。説明されている置換メカニズムは、この場合にのみ実装される特別なメカニズムであり、一般的なC ++メカニズムではないことに注意してください。この場合、メモリの割り当てと解放のためのユーザー関数を実装すると、対応する標準関数を呼び出すことができなくなり、それらは完全に非表示になり（演算子::は役に立ちません）、それらを呼び出そうとすると、ユーザー関数への再帰呼び出しが発生します。

グローバル名前空間定義関数

void* operator new(std::size_t size, const std::nothrow_t& nth)
{
// ...
}

標準のものも置き換えられますが、例外をスローしない演算子newはほとんど使用されないため、潜在的な問題は少なくなります。ただし、標準フォームも使用できません。

配列の関数についても同じ状況です。グローバル名前空間で

2.2。クラスのオーバーロード


new/deleteクラス内の 演算子のオーバーロードには、上記の欠点はありません。オーバーロードは、演算子の呼び出しのコンテキストに関係なく、対応するクラスのインスタンスを作成および削除する場合にのみ有効ですnew/delete。演算子を使用してメモリを割り当てたり解放したりするためのユーザー定義関数を実装::する場合、対応する標準関数にアクセスできます。例を考えてみましょう。

class X
{
// ...
public:
    void* operator new(std::size_t size)
    {
        std::cout << "X new\n";
        return ::operator new(size);
    }

    void operator delete(void* ptr)
    {
        std::cout << "X delete\n";
        ::operator delete(ptr);
    }

    void* operator new[](std::size_t size)
    {
        std::cout << "X new[]\n";
        return ::operator new[](size);
    }

    void operator delete[](void* ptr)
    {
        std::cout << "X delete[]\n";
        ::operator delete[](ptr);
    }
};

この例では、トレースは単純に標準操作に追加されています。今、用語new X()でnew X[N]は、これらの関数を使用してメモリの割り当てと解放を行います。

これらの関数は正式には静的であり、として宣言できますstatic。しかし、基本的にはそれらはインスタンスであり、関数呼び出しoperator new()を使用すると、インスタンスの作成が開始され、関数の呼び出しによってoperator delete()その削除が完了します。これらの関数が他のタスクで呼び出されることはありません。さらに、以下に示すように、関数operator delete()は基本的に仮想です。したがって、なしで宣言する方が正しいでしょうstatic。

2.2.1。新規/削除演算子の標準形式へのアクセス


演算子new/deleteは、追加のスコープ解決演算子と一緒に使用できます::new(p) X()。この場合、operator new()クラスで定義された関数は無視され、対応する標準が使用されます。同様に、演算子を使用できますdelete。

2.2.2。新規/削除演算子の他の形式を非表示にする


クラスでX例外をスローするかスローしないかを使用しようとするとnew、エラーが発生します。実際、この関数operator new(std::size_t size)は他のフォームを隠しますoperator new()。この問題は2つの方法で解決できます。最初に、適切なオプションをクラスに追加する必要があります（これらのオプションは、標準関数の操作を委任するだけです）。2番目のnew例では、スコープ解決演算子とともに演算子を使用する必要があります::new(p) X()。

2.2.3。標準コンテナ


Xたとえばstd::vector<X>、標準のコンテナに インスタンスを配置しようとすると、関数がメモリの割り当てと解放に使用されていないことがわかります。実際のところ、すべての標準コンテナには、メモリの割り当てと解放のための独自のメカニズム（コンテナのテンプレートパラメータである特別なアロケータクラス）があり、配置演算子を使用して要素を初期化していますnew。

2.2.4。継承


メモリの割り当てと解放のための関数は継承されます。これらの関数が基本クラスで定義されているが、派生クラスでは定義されていない場合、演算子は派生クラスに対してオーバーロードnew/deleteされ、基本クラスで定義および割り当てられている関数は、メモリの割り当てと解放に使用されます。

次に、各クラスが演算子をオーバーロードするポリモーフィックなクラス階層について考えnew/deleteます。次に、派生クラスのインスタンスdeleteを、基本クラスへのポインタを介して演算子を使用して削除します。基本クラスのデストラクタが仮想の場合、標準では、この派生クラスのデストラクタが呼び出されることが保証されています。この場合operator delete()、この派生クラスに定義された関数呼び出しも保証されます。したがって、関数はoperator delete()実際には仮想です。

2.2.5。演算子の削除（）関数の代替形式


クラスでは（特に継承が使用される場合）、メモリを解放する関数の代替形式を使用すると便利な場合があります。

void operator delete(void* p, std::size_t size);
void operator delete[](void* p, std::size_t size);

パラメータsizeは、要素のサイズを設定します（配列のバージョンでも）。このフォームでは、特定の派生クラスに応じて、さまざまな関数を使用してメモリを割り当てたり解放したりできます。

3.ユーザーオペレーターの新規/削除


C ++は、new次の形式のカスタム演算子形式をサポートできます。

new(/*  */) T(/*   */)
new(/*  */) T[/*   */]

これらのフォームをサポートするには、メモリの割り当てと解放に適切な関数を決定する必要があります。

void* operator new(std::size_t size, /* .  */);
void* operator new[](std::size_t size, /* .  */);
void operator delete(void* p, /* .  */);
void operator delete[](void* p, /* .  */);

メモリ割り当て関数の追加パラメータのリストは空であってはならず、1つvoid*またはconst std::nothrow_t&で構成されていてはいけません。追加パラメータでのリストoperator new()とoperator delete()一致している必要があります。オペレーターに渡される引数newは、メモリー割り当て関数の追加パラメーターに対応している必要があります。カスタム関数operator delete()は、オプションのサイズパラメータを持つフォームにすることもできます。

これらの関数は、グローバル名前空間またはクラスで定義できますが、グローバル以外の名前空間では定義できません。それらがグローバル名前空間で定義されている場合、それらは置き換えられませんが、メモリの割り当てと解放の標準関数をオーバーロードするため、それらの使用は予測可能で安全であり、標準関数は常に利用可能です。これらがクラスで定義されている場合、標準フォームは非表示になりますが、標準フォームへのアクセスは演算子を使用して取得できます::。これについては、セクション2.2で詳しく説明しています。

ユーザー定義の演算子フォームnewは、newユーザー定義の配置と呼ばれますnew。newセクション1.2で説明されている標準の（非割り当て）配置演算子と混同しないでください。

対応する演算子フォームdeleteが存在しません。ユーザー定義演算子を使用して作成されたオブジェクトnewを削除するには、2つの方法があります。ユーザー定義関数operator new()がメモリ割り当て操作を標準のメモリ割り当て関数に委任する場合、標準の演算子を使用できますdelete。そうでない場合は、明示的にデストラクタを呼び出してから、ユーザー定義関数を呼び出す必要がありますoperator delete()。コンパイラーがユーザー定義関数を呼び出すのoperator delete()は、ユーザー定義演算子の操作中にnewコンストラクターが例外をスローした場合のみです。

次に例を示します（グローバルスコープ内）。

void* operator new(std::size_t size, int a, const char* b)
{
    std::cout << "new " << a << " + " << b << "\n";
    return ::operator new(size);
}
void operator delete(void* p, int a, const char* b)
{
    std::cout << "delete " << a << " + " << b << "\n";
    ::operator delete(p);
}

class X {/* ... */};
X* p = new(42, "meow") X(); // : new 42 + meow
delete p; //   ::operator delete()
