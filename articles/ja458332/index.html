この記事はかなり古いですが、関連性を失っていません。非同期/待機になると、通常、それへのリンクが表示されます。私はロシア語への翻訳を見つけることができませんでした、私は流暢でない人を助けることにしました。



非同期プログラミングは長い間、マゾヒズムへの渇望を持つ最も経験豊富な開発者の王国でした-非線形の実行フローでのコールバックからのコールバックについて考える十分な自由時間、傾向、精神的能力を持っていた開発者。 Microsoft .NET Framework 4.5の登場により、C＃とVisual Basicによってすべてが非同期になり、単なる人間が非同期メソッドを同期メソッドと同じくらい簡単に記述できるようになりました。コールバックは不要になりました。ある同期コンテキストから別の同期コンテキストへの明示的なマーシャリングコードは不要です。実行結果や例外がどのように移動するかを心配する必要はもうありません。非同期コードの開発の便宜のために、プログラミング言語の手段を歪めるトリックの必要はありません。つまり、トラブルや頭痛がなくなります。

public static Task<int> SumAsync(Task<int> a, Task<int> b, Task<int> c)
{
  return (a.Status == TaskStatus.RanToCompletion &&
          b.Status == TaskStatus.RanToCompletion &&
          c.Status == TaskStatus.RanToCompletion) ?
    Task.FromResult(Sum(a.Result, b.Result, c.Result)) :
    SumAsyncInternal(a, b, c);
}

private static async Task<int> SumAsyncInternal(Task<int> a, Task<int> b, Task<int> c)
{
  await Task.WhenAll((Task)a, b, c).ConfigureAwait(false);
  return Sum(a.Result, b.Result, c.Result);
}
