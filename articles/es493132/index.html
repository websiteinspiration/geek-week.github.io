Alrededor de data.table

Esta nota ser√° interesante para aquellos que usan la biblioteca de procesamiento de datos de tablas para R - data.table y, tal vez, se alegrar√°n de ver la flexibilidad de su aplicaci√≥n con varios ejemplos.

Inspirado en un buen ejemplo de un colega , y con la esperanza de que ya haya le√≠do su art√≠culo, sugiero profundizar en la optimizaci√≥n del c√≥digo y el rendimiento basado en data.table .

Introducci√≥n: ¬øde d√≥nde viene data.table?


Es mejor comenzar a familiarizarse con la biblioteca un poco desde lejos, es decir, a partir de las estructuras de datos a partir de las cuales se puede obtener el objeto data.table (en adelante, DT).

Formaci√≥n


El c√≥digo
## arrays ---------

arrmatr <- array(1:20, c(4,5))

class(arrmatr)

typeof(arrmatr)

is.array(arrmatr)

is.matrix(arrmatr)


Una de estas estructuras es una matriz ( ? Base :: array ). Como en otros idiomas, las matrices son multidimensionales aqu√≠. Sin embargo, es interesante que, por ejemplo, una matriz bidimensional comience a heredar propiedades de la clase de matriz (? Base :: matriz ), y una matriz unidimensional, que tambi√©n es importante, no hereda del vector ( ? Base :: vector ).

Se debe entender que el tipo de datos contenidos en una instalaci√≥n se debe comprobar la funci√≥n de base :: el typeof , que devuelve la descripci√≥n interna del tipo de acuerdo con las partes internas R - protocolo de lenguaje com√∫n asociado con el primer nacido el C .

Otro comando para determinar la clase de un objeto es base :: class, devuelve en el caso de vectores un tipo de vector (difiere del nombre interno, pero tambi√©n le permite comprender el tipo de datos).

Lista


Desde una matriz bidimensional, es una matriz, puede ir a la lista ( ? Base :: list ).

El c√≥digo
## lists ------------------

mylist <- as.list(arrmatr)

is.vector(mylist)

is.list(mylist)


En este caso, suceden varias cosas a la vez:

  • La segunda dimensi√≥n de la matriz colapsa, es decir, obtenemos una lista y un vector.
  • La lista hereda as√≠ de estas clases. Debe tenerse en cuenta que un solo valor (escalar) de la celda de la matriz matriz corresponder√° al elemento de la lista.

Debido al hecho de que la lista tambi√©n es un vector, se le pueden aplicar algunas funciones para vectores.

Marco de datos


Desde una lista, matriz o vector, puede ir al marco de datos ( ? Base :: data.frame ).

El c√≥digo
## data.frames ------------

df <- as.data.frame(arrmatr)
df2 <- as.data.frame(mylist)

is.list(df)

df$V6 <- df$V1 + df$V2


Lo interesante es que el marco de datos hereda de la lista. Las columnas del marco de datos son celdas de lista. Esto ser√° importante en el futuro cuando usemos las funciones aplicadas a las listas.

tabla de datos


Puede obtener DT ( ? Data.table :: data.table ) de un marco de datos , lista, vector o matriz. Por ejemplo, as√≠ (en su lugar).

El c√≥digo
## data.tables -----------------------
library(data.table)

data.table::setDT(df)

is.list(df)

is.data.frame(df)

is.data.table(df)


Es √∫til que, como el marco de datos, DT herede las propiedades de la lista.

DT y memoria


A diferencia de todos los dem√°s objetos en la base R, los DT se pasan por referencia. Si necesita copiar en una nueva √°rea de memoria, necesita la funci√≥n data.table :: copy o debe hacer una selecci√≥n del objeto antiguo.

El c√≥digo
df2 <- df

df[V1 == 1, V2 := 999]

data.table::fsetdiff(df, df2)

df2 <- data.table::copy(df)

df[V1 == 2, V2 := 999]

data.table::fsetdiff(df, df2)


En esta introducci√≥n llega a su fin. DT es una continuaci√≥n del desarrollo de estructuras de datos en R, que ocurre principalmente debido a la expansi√≥n y aceleraci√≥n de operaciones realizadas en objetos de la clase de marco de datos. En este caso, se preserva la herencia de otras primitivas.

Algunos ejemplos de uso de propiedades data.table


Como una lista ...


Iterar sobre las filas de un marco de datos o DT no es una buena idea, ya que el c√≥digo de bucle en el lenguaje R es mucho m√°s lento que C , y es bastante posible recorrer las columnas en columnas que generalmente son mucho menos. Al recorrer las columnas, recuerde que cada columna es un elemento de la lista, que generalmente contiene un vector. Y las operaciones en vectores est√°n bien vectorizadas en las funciones b√°sicas del lenguaje. Tambi√©n puede usar los operadores de selecci√≥n espec√≠ficos para listas y vectores: `[[`, `$` .

El c√≥digo

Vectorizaci√≥n


Si es necesario pasar por las l√≠neas de un DT grande, escribir una funci√≥n con vectorizaci√≥n es la mejor soluci√≥n. Pero si no lo hace el trabajo, se debe recordar que el ciclo dentro del DT todav√≠a ciclo m√°s r√°pido en el R , porque se ejecuta en el C .

Estad√≠sticas, aprendizaje autom√°tico, etc. dentro de DT


Puede usar funciones lambda, pero a veces es mejor crearlas por separado, registrar toda la canalizaci√≥n de an√°lisis de datos y luego funcionan dentro del DT. El ejemplo se enriquece con todas las caracter√≠sticas anteriores, adem√°s de varias cosas √∫tiles del arsenal de DT (como acceder al propio DT dentro del DT por referencia, a veces no insertado secuencialmente, pero para ser).

El c√≥digo
# function

rm(lm_preds)

lm_preds <- function(
	sd, by, n
)
{
	
	if(
		n < 100 | 
		!by[['a']] %in% head(letters, 4)
	   )
	{
		
		res <-
			list(
				low = NA
				, mean = NA
				, high = NA
				, coefs = NA
			)
		
	} else {

		lmm <- 
			lm(
				d ~ c + b
				, data = sd
			)
		
		preds <- 
			stats::predict.lm(
				lmm
				, sd
				, interval = "prediction"
				)
		
		res <-
			list(
				low = preds[, 2]
				, mean = preds[, 1]
				, high = preds[, 3]
				, coefs = coefficients(lmm)
			)
	}

	res
	
}

res5 <- 
	dt %>%
	.[e < 0] %>%
	.[.[, .I[b > 0]]] %>%
	.[, `:=` (
		low = as.numeric(lm_preds(.SD, .BY, .N)[[1]])
		, mean = as.numeric(lm_preds(.SD, .BY, .N)[[2]])
		, high = as.numeric(lm_preds(.SD, .BY, .N)[[3]])
		, coef_c = as.numeric(lm_preds(.SD, .BY, .N)[[4]][1])
		, coef_b = as.numeric(lm_preds(.SD, .BY, .N)[[4]][2])
		, coef_int = as.numeric(lm_preds(.SD, .BY, .N)[[4]][3])
	)
	, a
	] %>%
	.[!is.na(mean), -'e', with = F]


# plot

plo <- 
	res5 %>%
	ggplot +

			, ymax = high
			, fill = a
		)
		, size = 0.1
		, alpha = 0.1
	) +
	geom_point(
		aes(
			x = c * coef_c + b * coef_b + coef_int

		)
		, size = 1
	) +

		aes(

		)
		, size = 1
		, color = 'black'
	) +
	theme_minimal()
