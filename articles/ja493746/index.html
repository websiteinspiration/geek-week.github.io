スマートハウス。開発と実装の経験

この記事では、2部屋のアパートでのスマートホームの開発と実装の経験を確認します。ここでは、スマートホームの設計と開発へのアプローチに関する著者の見解と、4年間成功しているターンキーソリューションを紹介します。


家をスマートにしたい人へようこそ。

スマートホームとは？私たちは公式の定義から離れて、消費者の視点からそれを見ていきます。著者が初めてスマートホームのアイデアを友人と共有したとき、彼は言った、「自分でスイッチに行って、自分で電気をつけることはできませんか？」彼自身のやり方では、彼は正しいです-著者は怠惰です（冗談）。スマートホームのアイデアは、彼自身が私たちのために操作を実行して、照明、電源を制御すると同時に、リソースを節約し、安全を確保するという事実にあります。したがって、スマートホームは、家に住む人々の生活の質を向上させることを目的としたソフトウェアとハ​​ードウェアのソリューションの複合体です。はい、そうです。スマートホームは、大きくて高価なおもちゃではなく、あなたの人生をより快適にするものと考えるべきです。この概念では、記事の著者はスマートホームの音声制御を不要であると認識し、スマートホームが実現すべきこと、そして将来的には、典型的な人間の行動も予測します。

行動パターン


スマートホームについて考えるとき、どこから始めますか？テクノロジーの選択により、中央コントローラーはありますか？いいえ、またいいえ。まず第一に、あなた自身に質問をしてください-なぜ私はスマートホームが必要なのですか？彼は私のために何をしますか？記事の著者によると、スマートホームが提供できる機能には4つのグループがあります。

  • 行動パターン-電化製品と照明の使用に関して、アパートでの生活の典型的な行動をサポートします。
  • 快適な環境-昼、夜、ゲスト、ロマンチックな夜などの条件に応じて快適さを生み出します。
  • 情報サポート-天気、交通渋滞、友人の誕生日などの現在のイベントなど、アパートに住んでいる人のためのさまざまなヒント
  • 安全な家-緊急防止、電化製品の迅速なシャットダウン、セキュリティ機能。

これらの機能のグループはすべて慎重に検討し、アパートに住むすべての人と合意する必要があります。その後、スマートホームシステムの設計について話すことができます。記事の著者のスマートホーム機能の例を使用して、そのようなデザインの例を示しましょう。

それでは、行動パターンから始めましょう。それらを見るために、あなたはあなた自身とあなたの家族をしばらくの間見守る必要があります。観察のフレームワークでは、私たちは記録します。人は何時に起きますか、彼は常に同じ時間または一定の間隔で食事をとりますか？アパートの周りのルート、目を覚まして家を離れた瞬間から、家に到着した瞬間から終わりの時間までのアパートに沿ったルート。その結果、私たちにとって重要な写真が得られます。人が使用する光源の種類、快適な生活に必要なもの-浴室に暖かい床を含めること、柔らかな照明、夜間の薄暗い照明など。このようなパターンは、家族ごとに見つける必要があります間に何かが入ると、他の人の快適さが損なわれることはありません。夫が毎日妻より早く起きるとします。

妻を起こさないようにするには、天井の照明の代わりに弱い緑色のライトを5分間オンにしてからオフにします（著者の経験から、弱い緑色のライトは他の人の睡眠を妨げません）。その結果、週末の朝と週の朝の2つのパターンが実現しました。これらのパターンの一部として、スマートホーム自体が適切なタイミングで照明をオンにし、ラジオがオフになり、さまざまな色のシーリングライトを使用して、仕事に行くまでの残り時間を通知します。パターン自体は、Googleカレンダーからイベントが始まる前の朝の手続きに必要な時間の計算に基づいて、スマートホームによって起動されます。このために、今後のイベントに関するデータを分析する特別なインテリジェントな目覚まし時計が開発されました。

快適な環境を作るために、夜に台所に行くときに快適に感じる照明を観察する必要があります。毎日、または週末にのみ朝は暖かい床をオンにしますか。テレビを見るとき、どのような照明を設定しますか？そのような観察の結果、著者は部屋に緑色のバックライトを実現し、公共の場では、オーバーヘッドライトはモーションセンサーのコマンドによってオン/オフされる公称の10％でした。ゲストが到着すると、バックライトの色が滑らかに変化し、テレビの暖炉でビデオを「ねじれ」、オーディオコラムから心地よい音楽を再生できます。そして、これらすべては、コントロールパネルまたは電報からの1つのコマンドで行います。この機能グループの枠組みの中で、著者は、ナイトモードに加えて、テレビの電源を入れながら明るい白いバックライトをオンにすることを実現しました。バックライトの色が滑らかに変化するゲストモード。

情報サポートはオプションですが、楽しいです。結局のところ、仕事に行くとき、外で雨が降っているので、人間の声で思いやりのあるスマートホームは傘を取る必要があることを思い出させます。そして、夕方、家に帰ると、同じスマートホームに自動的に含まれる長い勤務日の後、落ち着いた音楽を聞くのはいつも楽しいです。

そして最後に、安全な家。スマートホームの基本的な機能は、人々が家を出るときに照明と電化製品をオフにすることです。しかし、両親が仕事中に何かをすることができる好奇心旺盛な学生が自宅にいるとしたらどうでしょう。スマートホームはすべてのソケットをオフにするため、問題を引き起こす可能性はほとんどありません。そして、両親が来ると、システムがこれを認識し、アパート全体の電源を入れます。記事の著者は、彼が電気コンロをオフにするのを忘れて仕事に行ったときに事件がありました。そして、スマートハウスが自動的にすべての電化製品をオフにしたという事実のためだけに、不幸は起こりませんでした。

建築


すべての願いが定式化されたら、スマートホームのアーキテクチャの開発を開始できます。ここでは、システムが管理する部屋の規模を考慮することが重要です。電気および制御ラインのトポロジーを事前に決定する必要があります。理想的なオプションはスター型トポロジーで、各消費者が独自の電源ラインを持ち、すべての制御電子回路が、パネルスイッチが設置されているクラシックパネルの隣の1つのパネルに組み込まれています。より経済的なオプションは、ロシアのビルダーが構築するように「共通バス」配線が使用され、制御リレーがデバイスの正面の回路に直接接続されている場合です。この方法の欠点は、引き続く天井、タイル、または壁紙で閉じることができるため、その後のリレーのメンテナンスが複雑になることです。現在まで、電気機器とともに取り付けボックスに直接取り付けることができるリレーがありますが、ロシアの都市の新しい建物に大量に設置されるリレーよりも深いものがあります。

スマートホームのアーキテクチャを定義する次の重要なポイントは、制御通信です。管理通信には次の2つのタイプがあります。

  • 各スイッチまたはリレーへの個別の通信回線、またはMegaD-2561（https://ab-log.ru/）などの1つの通信回線によって制御されるリレーグループ。利点-信頼性、欠点-追加の通信回線に高コスト
  • ラジコン。各リレーは、無線で送信されるコマンドによって制御されます。この方法の利点には、設置が簡単で、追加の制御ラインが不要なことが含まれます。不利な点は無線信号であり、外部干渉のために減衰する可能性があり、コマンドがリレーに到達しない可能性があります。このようなリレーの例として、記事の作成者が使用しているNooliteシステム（http://noolite.by/）を引用できます。

経済的考察に基づいて、スマートホームの記事の著者は無線チャネルを選択しました。システムの柔軟性を高めるために、各「消費者」には独自の電線が敷設されていました。

スマートホームのアーキテクチャで最も重要な決定は、分散型または集中型の制御方式です。それぞれを詳細に分析します
分散型制御スキームでは、すべてのデバイスが相互接続され、システムに単一の意思決定センターはありません。ここでは、「それぞれと」の関係が実現されています。実際、これは通常の照明制御回路と同じです。ただし、スイッチとリレー自体が少しの「知能」を受け取り、たとえば、タイムアウトによってライトをオフにしたり、単一のスイッチから、リレー。このようなスキームでは、スマートホームコントローラーの存在が許可されます。これは、光源の状態に関する通知も受け取り、組み込みロジックに従って、制御コマンドをリレーに送信できます。



示されている図では、ボタン（ボタン）からのコマンドは、それに関連付けられているリレーに直接送信されますが、このコマンドはコントローラーに送信され、コントローラーは押すことを通知されます。同時に、リレーは、その状態を変更するときに、これについてもコントローラーに通知します。コントローラが故障した場合でも、照明は機能しますが、追加の機能はないため、このようなスキームは完全に信頼できます。一方、このような方式は柔軟性が低くなります。たとえば、1つのスイッチの動作ロジックを異なるリレーで変更することは、たとえば時刻によっては困難になるためです。このようなタスクには、集中方式が適しています。



この図から、すべてのコマンドがスマートホームコントローラーに送信され、すべての制御もこのコントローラーによって実行されることがわかります。このようなスキームの主な欠点は、単一障害点です。コントローラーが故障すると、システム全体が機能しなくなりますが、このソリューションの重要な利点は、デバイス管理の柔軟性が非常に高く、居住者の好みに応じて作業のロジックを完全に変更できることです。コントローラの故障の可能性が低いという事実に基づいて（システムの4年間の運用中、メモリカードによる重大な故障は1つしかなかった）、著者はスマートホーム用の集中型アーキテクチャスキームを選択しました。

実装


この記事の目的ではないハードウェアソリューションに加えて、スマートホームを実装するには、ソフトウェアプラットフォームを選択する必要があります。現在、市場にはフリーソフトウェアに基づく多くのソリューションがあります。著者によると、以下のプロジェクトは検討に値します（著者が直接作業したシステムを含む）：

  • Majordomo-システムはPHPで記述されています。プログラミングには、オブジェクト、そのプロパティ、およびメソッドの概念が使用されます。デバイスの大規模なセットをサポートし、大規模なロシアのコミュニティがあります。これはクイックスタートには適していますが、MySQL DBMSと従来のWebアプリケーションのアーキテクチャをこのシステムで使用することが大きな欠点です。このシステムは、著者がスマートホームの最初のバージョンで使用したものです。PHPまたはGoogle Blocklyをロジックプログラミングツールとして使用できます。

スマートホーム自体は現在、Rapsberry PI 3とRapsberrianオペレーティングシステムに基づいて実装されています。すでに述べたように、ioBrokerシステムは制御システムとして使用されます。照明と電力管理はNooliteテクノロジーに基づいています。ハードウェアアドオンとして、GSMモジュールが使用されます。GSMモジュールは、インターネットがない場合の緊急通知に使用されます。また、信頼できる電話からのSMSを介した追加のシステム制御チャネルとしても使用されます。

デバイス間の相互作用のロジックは、JavaScript、Google Blockly、Node-redスクリプトに基づいて実装されています。

JavaScriptはNodeJSの通常のプログラムです。たとえば、チャイムのチャイムは、次のように著者のスマートホームに実装されます。

schedule('0 0,8-23 * * *',() =>
	{
	 let currentTime = new Date();
	 let hour = currentTime.getHours();
	 if(hour > 12)
		hour -= 12;
	 let file = hour.toString() + 'h.mp3';
            playSound('audio/' + file); 
           });

ここで、ioBrokerコードは、1時間ごとに8：00-23：00の間隔で関数を呼び出します。この関数は、対応する時間のチャイムのチャイムが記録されているファイルの名前を生成し、外部デバイスで必要なオーディオファイルを再生します。

Google Blocklyを使用すると、JavaScriptの詳細に触れなくても、ビジュアルプログラミング言語でコードを記述できます。たとえば、モーションセンサーによる廊下への光の包含は、このように実装されています。太陽が沈むと、ライトは100％で、夜間モードで、10％の明るさで点灯します。

Node-redを使用すると、簡単な制御シナリオを実装できます。たとえば、バスルームでは、シンプルなコードを使用してモーションセンサーからの光を制御できます。



モーションセンサーの状態が変化した瞬間に、Node-redスクリプトブロックが自動的に実行されます。モーションセンサーが「オフにする」コマンドを送信すると、スクリプトは5分の遅延を与え、その後ライトをオフにします。 5分以内にモーションセンサーが「電源を入れる」コマンドを送信すると、遅延はリセットされ、ライトは点灯したままになります。 Node-redブロックの実装の詳細については、この記事の範囲を超えています。

したがって、生活の質を改善するシステムとしてのスマートホームへのアプローチのおかげで、プロジェクトの作成者は、多大な経済的コストを経験することなく、スマートホームの夢を実現することができました。2部屋のアパートのプロジェクトの開始時に、費用は約50,000ルーブル（811ドル）に達しました。
