cPanelとPlesk Obsidianの違い

どのコントロールパネルが最適かを判断するには、以下に説明するいくつかの要因を考慮する必要があります。

ユーザーインターフェース


Plesk


Plesk Obsidianには、JavaScriptで開発されたUI / UXインターフェースがあります。ダッシュボードは、WordPress管理ダッシュボードに似ています。

主な機能：

  • クリーンで整頓されたユーザーインターフェイス
  • 初心者にやさしい
  • すべての機能とツールを簡単に見つけることができます。


Cpanel


cPanel Linuxでは、各アイテムはパーティション化されています。ファイル、データベース、ドメイン、電子メール、メトリック、セキュリティ、ソフトウェア、およびアプリケーションはすべて、コントロールパネルのさまざまな部分です。

主な機能：

  • グラフィカル・ユーザー・インターフェース
  • 同様の機能がグループ化されています。
  • ツールバーをカスタマイズできます


オペレーティング・システム


PleskはLinuxおよびWindowsサーバーと互換性があります。cPanelはLinuxサーバーでのみ機能します。

Plesk


  • Debian
  • Ubuntu
  • CentOS
  • Cloudlinux
  • Amazon Linux
  • Virtuozzo Linux
  • Windowsサーバー2016

Cpanel


  • CentOS
  • Cloudlinux
  • Red Hat Enterprise Linux
  • Amazon Linux

安全性


セキュリティは、すべてのWebユーザーにとって大きな関心事です。選択するプラットフォームは完全に保護されている必要があり、これらのコントロールパネルは両方とも、セキュリティを確保するためにできることはすべて実行します。

Plesk


Pleskコントロールパネルは、Immunity 360、Fail2ban侵入防止、LDAP / Active Directoryとの統合などの多くのセキュリティ拡張機能をサポートしています（拡張機能は無料バージョンと有料バージョンで利用できます）。

主な機能：

  • 24時間のセキュリティ修正
  • DNSとDNSSECを構成してドメインを保護する機能
  • SSLを暗号化またはDigicertしましょう
  • 専門家のサポートなしで技術的な問題を解決する自動処理および回復の機能。
  • Plesk Advisorを使用してワンクリックですべてのサイトのSSLおよびHTTP / 2。

Cpanel


cPanelが提供する最も有名なセキュリティ拡張機能には、SSL証明書の自動インストール、IPアドレスの拒否、パスワードで保護されたディレクトリなどがあります。

主な機能：

  • 各IPまたは地域と国全体のブラックリストまたはホワイトリスト
  • サーバーネットワーク全体のサイトにSSL証明書を展開する
  • すべての送信メッセージの暗号化
  • クライアントの多要素認証
  • サードパーティによる簡単な統合

バックアップ


どちらのプラットフォームも優れたサポートとセキュリティを提供しますが、いつバックアップする必要があるかはわかりません。PleskとcPanelのバックアップサポートの違いを見てみましょう。

Plesk


2種類のバックアップを提供します。 

  1. データが最後に更新された時刻に関係なく、すべてのデータを含む完全バックアップ。
  2. 前回のバックアップ以降に変更されたデータのみを含む増分バックアップ。

追加のバックアップを使用すると、バックアップ操作時間と、バックアップファイルによって占有されるディスク領域が最小限になります。このバックアップは、Plesk内部ストレージまたは外部FTPストレージに保存できます。

主な機能：

  • バックアップをスケジュールできます。
  • サーバー全体をバックアップできます
  • 個々のユーザーアカウント/サブスクリプションをバックアップする利点をカスタマイズできます。

最近の機能：最近、パスワードを使用してバックアップ内の機密データを保護できます。

Cpanel


3種類のバックアップを提供します。 

  1. 圧縮。すべての情報が圧縮形式で保存され、ディスク容量を節約します。 
  2. 非圧縮。より多くのディスク領域が必要ですが、圧縮よりも高速に動作します。 
  3. リンクを使用してディスク領域を節約する増分バックアップ。

主な機能：

  • 毎日、毎週、毎月の予備
  • 計画および保存する機能
  • デフォルトでは、システムはバックアップファイルをローカルの/ backup /ディレクトリに保存します

注： cPanel＆WHMには、システムバックアップファイルを復元するためのインターフェースが含まれていません。

機能とツール


Plesk


  • ワンクリックでワードプレスを保護
  • 200,000を超えるDockerイメージでのLinux Dockerサポート
  • Let's EncryptはWebホスティングのコントロールパネルから始まります
  • Git互換ですが、git拡張機能を使用する必要があります

Cpanel


  • サイトのワンクリックでHTTPSリダイレクト
  • セキュリティのためのImunify360
  • 新しいcPanelアカウントのデフォルトのSpamBox

注： PleskはNginxおよびApache Webサーバをサポートしていますが、cPanelは現在Apache Webサーバのみをサポートしています。

サポート


Plesk


  • Pleskサポートを24時間年中無休で無料で呼び出すことができます
  • あなたが彼らのクライアントでないなら、あなたは彼らの公式フォーラムまたはソーシャルネットワークで質問をすることができます。

Cpanel


  • カスタマーサポートはチケットを通じて実施されます（中央標準時：月曜から金曜の06:00から18:00、土曜から日曜の6:00から16:00）
  • 緊急援助を得るには、事件ごとに65ドルを支払う必要があります。

PleskとcPanelのどちらが良いですか？


選択したWebホスティングコントロールパネルは、サーバーとのやり取りで重要な役割を果たします。コントロールパネルは機能的なツールであるため、プロジェクトを快適に管理する必要があります。

  • cPanelは使いやすく、ほとんどのインターネットを制御するLinuxベースのサーバーで動作するため、非常に人気があります。
  • 複数のサーバを使用しているときに同じコントロールパネルを引き続き使用する場合は、Pleskが最適です。
  • Pleskパネルには、10〜30のWebサイト用に個別のライセンスがあり、サイトの数が少ないほど、最終的にそれらを管理するためのコストが低くなります。
  • ただし、サイトの数が多い場合は、cPanelの方が手頃な財務オプションになる可能性があります。
  • 初心者でWebサイトから始める場合は、cPanelではなくPleskをお勧めします。
