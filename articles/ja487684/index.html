Vue.jsでのより高速なアプリケーションの開発

JavaScriptは、最新のWebアプリケーションの魂です。これは、フロントエンド開発の主要な要素です。 Webプロジェクトインターフェイスを作成するためのさまざまなJavaScriptフレームワークがあります。 Vue.jsはこれらのフレームワークの1つであり、かなり人気のあるソリューションに起因している可能性があります。

Vue.jsは、ユーザーインターフェイスを作成するためのプログレッシブフレームワークです。そのコアライブラリは、主にインターフェイスの可視部分を作成することを目的としています。他のライブラリは、必要に応じてVueベースのプロジェクトに簡単に統合できます。さらに、Vue.jsの助けを借りて、最新のツールと補助ライブラリを使用して、複雑な単一ページのアプリケーションを作成できます。



この記事では、特定のタスクに関するメモを処理するように設計された単純なVue.jsアプリケーションを作成するプロセスについて説明します。これがプロジェクトのフロントエンドリポジトリです。これが彼のバックエンドのリポジトリです。その過程で、Vue.jsと補助ツールのいくつかの強力な機能を分析します。

ルーティング


ルーティングは、最新のWebアプリケーションの優れた機能の1つです。ルーターは、ライブラリを使用してVue.jsアプリケーションに統合できますvue-router。これはVue.jsプロジェクトの公式ルーターです。その機能の中で、次のことに注意してください。

  • ネストされたルート/ビュー。
  • ルータのモジュール構成。
  • ルーティングパラメータ、リクエスト、テンプレートへのアクセス。
  • Vue.jsの機能に基づく表現の遷移のアニメーション。
  • 便利なナビゲーションコントロール。
  • アクティブリンクの自動スタイル設定のサポート。
  • HTML5-API履歴のサポート、URLハッシュを使用する機能、IE9互換モードに自動的に切り替えます。
  • カスタマイズ可能なページスクロール動作。

アプリケーションにルーティングを実装するには、フォルダー内routerにファイルを作成しますindex.js。次のコードを追加します。

import Vue from "vue";
import VueRouter from "vue-router";
import DashboardLayout from "../layout/DashboardLayout.vue";

Vue.use(VueRouter);

const routes = [
  {
    path: "/home",
    component: DashboardLayout,
    children: [
      {
        path: "/notes",
        name: "Notes",
        component: () =>
          import(/* webpackChunkName: "home" */ "../views/Home.vue")
      }
    ]
  },
  {
    path: "/",
    redirect: { name: "Notes" }
  }
];

const router = new VueRouter({
  mode: "history",
  base: process.env.BASE_URL,
  routes
});

export default router;

routesアプリケーションでサポートされているルートの説明を含む オブジェクトを考えます。ネストされたルートがここで使用されます。

オブジェクトにchildrenは、コントロールパネル（ファイルDashboardLayout.vue）を表すアプリケーションページに表示されるネストされたルートが含まれます。このページのテンプレートは次のとおりです。

<template>
  <span>
    <nav-bar />
    <div class="container is-fluid body-content">
      <router-view :key="$router.path" />
    </div>
  </span>
</template>

このコードで最も重要なのはタグrouter-viewです。表示されたルートに対応するすべてのコンポーネントを含むコンテナの役割を果たします。

コンポーネントの基本


コンポーネントは、Vue.jsアプリケーションのコアコンポーネントです。これにより、モジュール式の開発アプローチを使用する機会が得られます。つまり、DOMページをいくつかの小さなフラグメントに分割し、さまざまなページで再利用できます。

コンポーネントを設計する場合、コンポーネントをスケーラブルにして再利用に適したものにするために、考慮すべきいくつかの重要な点があります。

  1. プロジェクトからコンポーネントとして選択できる別の機能を特定します。
  2. 主な機能に対応しない機能でコンポーネントをオーバーロードしないでください。
  3. 独自の操作を保証するために使用されるコードのみを含めます。たとえば、これは、年、ユーザーの性別など、特定のコンポーネントの標準データバインディングの操作を保証するコードです。
  4. たとえば、特定のAPIなど、コンポーネントの外部メカニズムを使用するコードをコンポーネントに追加しないでください。

コンポーネントの相互作用


どのWebアプリケーションでも、データフローの適切な編成は非常に重要です。これにより、アプリケーションデータを効果的に操作および管理できます。

コンポーネントアプローチを使用する場合、マークアップとアプリケーションコードを小さな部分に分離するとき、開発者はさまざまなコンポーネントで使用されるデータを転送して処理する方法を尋ねます。この質問に対する答えは、コンポーネントの相互作用の編成です。

Vue.jsプロジェクトのコンポーネントの相互作用は、次のメカニズムを使用して整理できます。

  1. プロパティ（プロパティ）は、親コンポーネントから子コンポーネントにデータを転送するときに使用されます。
  2. $ emit（）メソッドは、子コンポーネントから親コンポーネントにデータを転送するときに使用されます。
  3. グローバルイベントバス（EventBus）は、深くネストされたコンポーネント構造が使用される場合、またはパブリッシャー/サブスクライバーモデルに従ってコンポーネント間の交換を編成する必要がある場合に使用されます。

Vue.jsでのコンポーネントの相互作用の概念を理解するために、2つのコンポーネントをプロジェクトに追加します。

  • Addシステムに新しいタスクを追加し、既存のタスクを編集するために使用されるコンポーネント。
  • NoteViewer1つのタスクに関する情報を表示するように設計されたコンポーネント。

<script>
// @ is an alias to /src
// import NoteEditor from "@/components/NoteEditor.vue";
import NoteEditor from "@/components/Add.vue";
import NoteViewer from "@/components/NoteViewer.vue";
export default {
  name: "Home",
  components: {
    // NoteEditor,
    NoteEditor,
    NoteViewer
  },
  data() {
    return {
      enableAddNote: false,
      data: []
    };
  },
  mounted() {
    this.getNotes();
  },
  methods: {
    enableAdd() {
      this.enableAddNote = true;
    },
    disableAdd() {
      this.enableAddNote = false;
    },
    async getNotes() {
      this.data = [];
      const data = await this.$http.get("notes/getall");
      data.forEach(note => {
        this.data.push({
          content: note,
          viewMode: true
        });
      });
    },
    async addNote(note) {
      await this.$http.post("notes/create", note.content);
      this.disableAdd();
      await this.getNotes();
    },
    editNote(note) {
      note.viewMode = false;
    },
    async updateNote(note) {
      await this.$http.put(`notes/${note.content.id}`, note.content);
      note.viewMode = true;
      await this.getNotes();
    },
    cancelUpdate(note) {
      note.viewMode = true;
    },
    markCompletedConfirm(note) {
      this.$buefy.dialog.confirm({
        title: "Mark Completed",
        message: "Would you really like to mark the note completed?",
        type: "is-warning",
        hasIcon: true,
        onConfirm: async () => await this.markCompleted(note)
      });
    },
    async markCompleted(note) {
      note.content.isCompleted = true;
      await this.$http.put(`notes/${note.content.id}`, note.content);
      await this.getNotes();
    },
    deleteNoteConfirm(note) {
      this.$buefy.dialog.confirm({
        title: "Delete note",
        message: "Would you really like to delete the note?",
        type: "is-danger",
        hasIcon: true,
        onConfirm: async () => await this.deleteNote(note)
      });
    },
    async deleteNote(note) {
      await this.$http.delete(`notes/${note.content.id}`);
      await this.getNotes();
    }
  }
};
</script>

Axiosライブラリの使用


Axiosは、さまざまな外部サービスとの対話を整理するためのPromiseベースのライブラリです。

多くの機能があり、安全な作業に焦点を当てています。AxiosがXSRF攻撃に対する保護、リクエストとレスポンスのインターセプター、リクエストとレスポンスのデータを変換する手段、リクエストのキャンセルなどをサポートしているという事実について話しています。

Axiosライブラリをアプリケーションに接続し、使用するたびにインポートする必要がないように構成します。フォルダにaxiosファイルを作成しますindex.js。

import axios from "axios";

const apiHost = process.env.VUE_APP_API_HOST || "/";

let baseURL = "api";

if (apiHost) {
  baseURL = `${apiHost}api`;
}
export default axios.create({ baseURL: baseURL });

main.js外部APIと対話するように設計され た応答インターセプターをファイルに追加します。インターセプターを使用して、アプリケーションに転送されるデータを準備し、エラーを処理します。

import HTTP from "./axios";

//   
HTTP.interceptors.response.use(
  response => {
    if (response.data instanceof Blob) {
      return response.data;
    }
    return response.data.data || {};
  },
  error => {
    if (error.response) {
      Vue.prototype.$buefy.toast.open({
        message: error.response.data.message || "Something went wrong",
        type: "is-danger"
      });
    } else {
      Vue.prototype.$buefy.toast.open({
        message: "Unable to connect to server",
        type: "is-danger"
      });
    }
    return Promise.reject(error);
  }
);

最適化


何百ものコンポーネントとビューを含むアプリケーションのサイズが大きくなったと想像してください。

JavaScriptコードのすべてが一度にブラウザーにダウンロードされるため、これはアプリケーションのロード時間に影響します。アプリケーションのロードを最適化するには、いくつかの質問に答える必要があります。

  1. 現在使用されていないコンポーネントとビューが読み込まれていないことを確認するにはどうすればよいですか？
  2. ダウンロードした資料のサイズを小さくするには？
  3. アプリケーションの読み込み時間を改善するには？

これらの質問に対する回答として、次のことを提案できます。アプリケーションの基本構造をすぐにロードし、コンポーネントとビューが必要なときにロードします。これを行うには、Webpackの機能を使用し、ルーター設定に次の変更を加えます。

{
path: "/notes",
name: "Notes",
component: () =>
import(/* webpackChunkName: "home" */ "../views/Home.vue")
}
//   /* webpackChunkName: "home" */

これにより[view].[hash].js、ユーザーがこのルートにアクセスしたときにレイジーモードで読み込まれるアプリケーション（ビュー）マテリアルを使用して、特定のルートに個別のフラグメントを作成できます。

プロジェクトをDockerコンテナーにパックしてデプロイする


これで、アプリケーションは正常に機能します。つまり、コンテナー化するときです。次のファイルをプロジェクトに追加しますDockerfile。

# build stage
FROM node:lts-alpine as build-stage
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
ARG VUE_APP_API_HOST
ENV VUE_APP_API_HOST $VUE_APP_API_HOST
RUN npm run build

# production stage
FROM nginx:stable-alpine as production-stage
COPY --from=build-stage /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

本番環境でアプリケーションを使用する場合、Nginxなどの強力なHTTPサーバーの背後に配置します。これにより、アプリケーションがハッキングやその他の攻撃から保護されます。

Axiosのセットアップ時に宣言したホスト情報を含む環境変数を覚えていますか？彼女が来た：

const apiHost = process.env.VUE_APP_API_HOST || "/";

これはブラウザアプリケーションであるため、アセンブリ時にこの変数を設定してアプリケーションに渡す必要があります。--build-arg画像を組み立てるときにオプションを使用してこれを行うのは非常に簡単です：

sudo docker build --build-arg VUE_APP_API_HOST=<Scheme>://<ServiceHost>:<ServicePort>/ -f Dockerfile -t vue-app-image .

置き換える必要のあるもの<Scheme>、<ServiceHost>および<ServicePort>プロジェクトにとって意味のある値に注意してください。

アプリケーションコンテナがアセンブルされたら、起動できます。

sudo docker run -d -p 8080:80 — name vue-app vue-app-image
