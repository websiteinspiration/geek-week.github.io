ovirt-engineをインストールする


エンジンの最小要件は、2コア/ 4 GiBのRAM / 25 GiBのストレージです。推奨-4コア/ 16 GiBのRAM / 50 GiBのストレージ。管理されたクラスターの外部にある専用の物理マシンまたは仮想マシンでエンジンを実行する場合は、スタンドアロンマネージャーオプションを使用します。インストールでは、たとえば、別のESXi *上の仮想マシンを取り上げましょう。デプロイメント自動化ツールを使用するか、事前に準備されたテンプレートからの複製またはキックスタートのインストールが便利です。

*注：本番システムの場合、これは悪い考えです。マネージャーは遠慮なく働き、ボトルネックになります。この場合、オプションのセルフホストエンジンを検討することをお勧めします。

必要に応じて、スタンドアロンをセルフホストに変換する手順は、ドキュメントに詳細に記載されています。特に、Hosted Engineをサポートするホストに再インストールコマンドを与える必要があります。

VMの最小構成でCentOS 7をインストールしてから、システムを更新して再起動します。

$ sudo yum update -y && sudo reboot

仮想マシンの場合、ゲストエージェントをインストールすると便利です。

$ sudo yum install open-vm-tools

VMware ESXiホスト、またはoVirtの場合：

$ sudo yum install ovirt-guest-agent

リポジトリを接続し、マネージャーをインストールします。

$ sudo yum install https://resources.ovirt.org/pub/yum-repo/ovirt-release43.rpm
$ sudo yum install ovirt-engine

標準設定：

$ sudo engine-setup

ほとんどの場合、選択されているデフォルト設定で十分ですが、自動で使用するには、次のキーを使用して構成を開始できます。

$ sudo engine-setup --accept-defaults

これで、ovirt.lab.example.comで新しいエンジンに接続できます。ここではまだ空なので、ハイパーバイザーのインストールに移りましょう。

oVirtへのノードの追加


それは非常に簡単です：

計算→ホスト→新規→...

ウィザードでは、必須フィールドは名前（表示名、たとえば、kvm03）、ホスト名（FQDN、たとえば、kvm03.lab.example.com）および認証セクション- ルートユーザー（不変） -パスワードまたはSSH公開鍵。

クリックした後[OK]をボタンを、あなたはメッセージが表示されますあなたは、このホストの電源管理を設定していません」。続行してもよろしいですか？」。これは正常です。ホスト接続が成功した後で、電源管理について検討します。ただし、ホストがインストールされているマシンが管理（IPMI、iLO、DRACなど）をサポートしていない場合は、オフにすることをお勧めします：コンピュート→クラスター→デフォルト→編集→フェンシングプロイシー→フェンシングを有効化、チェックを外します。

oVirtリポジトリーがホストに接続されていなかった場合、インストールは失敗しますが、問題はありません。追加する必要があります。次に、[インストール]-> [再インストール]をクリックします。

ホスト接続には5〜10分しかかかりません。

ネットワークインターフェイスのセットアップ


我々はフォールトトレラントシステムを構築するように、ネットワーク接続もホスト→タブ→計算で行われた冗長接続、提供しなければならないHOSTセットアップホストのネットワーク-ネットワークインターフェイス→を。

ネットワーク機器の機能とアーキテクチャアプローチに応じて、オプションが可能です。トップオブラックスイッチのスタックに接続して、スイッチが故障してもネットワークの可用性が中断されないようにするのが最適です。集約されたLACPチャネルの例を考えます。集約チャネルを構成するには、マウスを使用して2番目の未使用のアダプターを「取り」、1番目に「取り」ます。 [ 新しいボンドの作成]ウィンドウが開きます。LACP（モード4、動的リンク集約、802.3ad）がデフォルトで選択されています。スイッチ側では、通常のLACPグループ構成が実行されます。スイッチスタックを構築できない場合は、アクティブバックアップモード（モード1）を使用できます。次の記事でVLAN設定を検討します。詳細については、計画と前提条件ガイドのドキュメントでネットワーク構成の推奨事項を説明します。

ISOイメージストレージ


OSをインストールするには、インストールファイルが必要になります。ほとんどの場合、ISOイメージの形式で入手できます。組み込みパスを使用できますが、oVirtで画像を操作するために、NFSサーバーをターゲットにできる特別なタイプのストレージ（ISO）が開発されます。追加：

ストレージ→ドメイン→新しいドメイン、
ドメイン機能→ISO、
エクスポートパス-例：mynfs01.example.com:/exports/ovirt-iso（接続時、フォルダーは空である必要があり、マネージャーはそれに書き込むことができるはずです）、
名前-例：mynfs01-iso。

画像を保存するために、マネージャーは構造
/ exports / ovirt-iso / <some UUID> / images / 11111111-1111-1111-1111-111111111111 /を作成します

NFSサーバーにISOイメージが既にある場合は、スペースを節約するためにファイルをコピーする代わりに、それらをこのフォルダーにリンクすると便利です。
