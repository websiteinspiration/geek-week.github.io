スマートホームを構築し、夢中にならない方法



それ自体が家の中の秩序を管理するスマート家具は、ほとんどすべての未来の写真にとってなくてはならないものです。実際、自動調節の気候、自動オン/オフライト、家電製品の音声制御-これらすべてを今すぐ構成できます。しかし、それは少しの経験、テクノロジーの分野での基本的な知識、そして時にはプログラミングだけでなく、ファンタジーの海全体を必要とします。私の場合、ファンタジーだけで十分になるようにしましたが、まず最初に...

5年ほど前に「スマートホーム」のアイデアに興味を持った。最初は最も単純なシステムを作成しました。彼女は廊下と浴室の光をモーションセンサーで制御し、フードを湿度センサーで制御し、気象ステーションも制御していました。自尊心のあるDIYボックスはすべて、気象ステーションを作らなければなりませんでした。

まず、廊下とバスルームの照明を自動でオンにする制御リレーをアパートに設置しました。それはこのように見えました：1つのセンサーは廊下に、2番目はバスルームにありました。

誰かがバスルームに入ると、廊下センサーが彼の動きを検知し、廊下とバスルームのライトをすぐにオンにしました。同時に、誰もバスルームに入らなかった場合、これはバスルーム内にあるセンサーによってすでに修正されています。 15秒後、ライトが消えました。人がバスルームに入ると、廊下のライトが1分後に消えました。 

バスルームの「白い友達」に座っているときに誰かがあまりにも多くのことを考えていた（私はバスルームを組み合わせていた）場合にも、そのようなケースを考えました。このため、バスルームのライトは2つのグループに分けられました。 1つはバスルームのセンサーが動きの検出を停止した3分後にオフになり、もう1つは5分後にオフになりました。そのため、5分間以上、バスルームで明かりの下で動かずにいることはできませんでした。非常に規律があります。ただし、いつでも手を動かして、本質について考え続けることができます。

湿度センサーはバスルームでも機能し、湿度が50％を超えると自動的にフードが作動しました。部屋が45％の湿度に換気されるとすぐに、フードはオフになりました。 

経営陣は、Arduinoプラットフォームを通過しました。  メーカーの


ウェブサイトから撮影した写真

ほとんどすぐに、このプラットフォームは完全にスマートホームを作成することではないことが明らかになりました。 Arduinoを使用することの主な欠点は、プラットフォームがネットワークなしでは機能し、ネットワークなしでは真に統一されたエコシステムを取得できないことでした。もちろん、Arduinoをやり直してネットワークサポートを追加することはできますが、なぜですか？私はより単純なパスを選択し、このプラットフォームを別のプラットフォームに変更しました。 

Arduinoで十分に遊んだ後、家をESP-8266ボードに再接続しました。実際、これは同じArduinoですが、Wi-Fi +の方がサイズがコンパクトです。このモジュールは、スマートホーム用のガジェットのメーカーに今も人気があります。  インターネット


から撮影した写真

並行して、私はスマートホームをさらにスマートにしようとしました。たとえば、24時間床暖房やエアコンが常時オンになっている問題を解決します。このため、Beok Chinese WiFiサーモスタットを購入しました。彼らはリモートで床暖房をオフにすることを許可しましたが、電話の特別なアプリケーションを通じてそれを行わなければなりませんでした。 

Broadlink RM Pro赤外線信号エミュレーターを使用して、エアコンのリモートコントロールの問題を解決しました。何も複雑ではありません：エアコンのコントロールパネルからエミュレーターに信号を記録し（リモコンで制御できる技術はあります）、電話でアプリケーションのボタンを押すと、エミュレーターは以前に記録した信号を再生します。エアコンの場合は、電源を入れたり切ったり、運転モードを設定したり、その他のパラメーターをリモートで設定する機会がありました。  

Livoloスイッチもインストールされています。彼らの助けを借りて、私はまた、無線で照明をオンまたはオフにすることができました。 



マイナス点としては、制御のために、別のアプリケーションを再度インストールする必要があり、フィードバックはありませんでした。つまり、従来のスイッチを使用して誰かが手動でオンまたはオフにした場合、ライトがオンであることがわかりませんでした。 

SonoffやTuyaなどのさまざまな制御されたWiFiリレー、さらには別のアプリケーションを必要とするアパートをロックするための高価なDanalockも家に登場しました。ほとんどすべてのこれらの小さなもの（ダナロックを除く）は、中国のサイトで購入しました。 

最初の比較的深刻な購入の1つはTionブリーザーでした。自動CO2制御により、彼は多かれ少なかれ対応しましたが、冬の暖房の温度は常に手動で調整する必要がありました。そして再び-管理するには、別のアプリケーションをインストールする必要がありました。 


写真はメーカーのウェブサイトから取得したものです。

そのときに試したすべてのセンサーとコントローラーは、覚えていません。私のスマートフォンには、それらを管理するためのアプリケーションが詰まっています。それは動物園全体であり、常に従わなければなりませんでした。これらのアプリケーションの管理を、HomeBridge / MajorDomoなどのあらゆる種類のアグリゲーターを介して組み合わせようとしました。しかし、誰もが重大な欠点を示しました。

  • 不親切なインターフェース、時にはひどいインターフェース
  • 使用されているすべてのアプリケーションのサポートの欠如
  • 複雑な接続

このような大量のセンサー、コントローラー、その他の制御システムを集中管理するためのアプリケーションの検索は、成功に至っていません。次に、「スマート」デバイスの1つであるTionブリーザーを個別に思いついた。室温に応じて暖房温度を自動制御するスクリプトを書きました。実際のところ、換気システムには暖房温度の自動調整機能がありませんでした。部屋はとても暑いか、とても寒かった。中立を達成する方法はありませんでした。ここでは、書かれたスクリプトとブリーザーの助けを借りて、この問題は解決されました。 

ブリーザーのスクリプトで成功したため、スマートホームを管理するための独自のアプリケーションを作成するようになりました。主な目標は、スマートデバイスの統合、マルチレベルの自動化条件、および家の中のすべてのデバイスを管理する機能を備えたプログラムを作成することでした。 

私は約1年間、バックエンドとフロントエンドのアプリケーション開発に携わっていました。 

サーバー側はNodeJSで書かれています。 NodeJSを選択したのは、市場に出ているほとんどすべてのデバイスに実装されたプロトコルが存在するコミュニティが開発されたためです。クライアント部分はAngular（Ionic）で記述され、Android / iOSで実行されます。一般的に、クラシックなクライアント/サーバーアーキテクチャ。

メモについて：アプリケーションの作業の過程で、デバイスドライバーを作成する際の不純物の使用に関する技術的な洞察を得ました。分からないかもしれませんが、初歩的な人もいますが、本当に呼吸しやすくなりました。

次のようなものになるまで、デバイスドライバを何度も書き直しました。

デバイスの1つのサンプルコード 
import {XiaomiSubdeviceV2} from '../xiaomi.subdevice.v2';
import {load_power} from '../capabilities/load_power';
import {power_plug} from '../capabilities/power_plug';
import {PowerPurpose} from '../../base/PowerPurpose';
import {Relay} from '../../base/types/Relay';
import {HomeKitAccessory} from '../../hap/HomeKitAccessory';
import {Lightbulb2Accessory} from '../../hap/Lightbulb2Accessory';

export class LumiPlug extends XiaomiSubdeviceV2.with(Relay, power_plug, load_power, PowerPurpose,
  HomeKitAccessory, Lightbulb2Accessory,
  Yandex, YandexLightOrSwitch) {

  onCreate() {
    super.onCreate();
    this.model = 'Mi Smart Plug';
    this.class_name = 'lumi.plug';
    this.driver_name = 'Mi Smart Plug';
    this.driver_type = 3;
    this.parent_class_name = 'lumi.gateway';
  }

  getIcon() {
    return 'socket';
  }
}


要するに、さまざまなデバイスが豊富であるにもかかわらず、それらはすべてほぼ同じことを行い、ほぼ同じ情報を提供するということです。したがって、デバイスのすべての機能は、最終的なドライバーを構成する個別の不純物に配置されました。たとえば、アプリケーションは、オン/オフ機能を持つ多くのデバイスをサポートしています。それは別の不純物で作られ、すべてのデバイスで同じように使用されます。小学校、ワトソン！

これにより、新しいデバイスのドライバーは非常に迅速かつ簡単に作成されました。すべてが標準化されており、受信した情報をさらに保存することを心配する必要はありません。完全に新しいプロトコル（まだ用意していませんでした）では、既存のものに基づいた不純物も書き込まれています。彼らはすでにデバイス情報を受信し、それをチェーンのさらに下流に送信しています。このアプローチにより、コードの量を数十倍に減らすことができました（最初は、各ドライバーは同様のドライバーのコピーでした）。

それで、私は徐々に地獄のすべての輪を通り抜けて、バッキングとフロントを終えました。アプリケーションがかなり許容できる外観を取得したとき、私は考えました。開発を一般に公開してみませんか？プロジェクトのパートナーとアプリケーションを思い付くためのアシスタントが見つかりました。 

まず最初に、アプリケーションの設計を覚えておく必要がありました。これを行うには、プロのデザイナーに依頼する必要がありました。 3ヶ月から4ヶ月かかると単純に信じていましたが、最終的にはプロセスが遅れました。アプリケーションの構造は元のソースからそれほど変わっていないという事実にもかかわらず、文字通りすべてをやり直す必要がありました。

並行して、私だけではなく、プロジェクトパートナーのチームと一緒に、スマートホーム用の最も人気のあるデバイスを購入し、これらのガジェットをサポートしていない場合はアプリケーションを追加しました。しかし間もなく、すべてのスマートデバイスに十分なお金がないことが明らかになり、市場の既存のプレーヤーと話し合い、機器の無料テストサンプルに無料で同意することを決定しました。私たちは否定されませんでした、そしてWirenboardとMiMiSmartは最初の深刻なサプライヤーになりました。

それで、私はみんなと一緒に、あらゆるプラットフォームに搭載されたクラシックなクライアント/サーバーアーキテクチャと便利でモダンなデザインを備えたスマートホームオートメーション用の新しいアプリケーションを作成しました。BARYに会いましょう*。

*名前はバリアリバソフの名前からではなく、バリモアの執事-あなたの個人的な「賢い執事」によるアーサーC.ドイルの本「バスカヴィルの猟犬」の性格から来ました。

アーキテクチャとセキュリティ


クライアント/サーバー交換はAESテクノロジーを使用して暗号化され、サーバーは自動化された部屋の中に直接配置されます。私の意見では、これはシステムをサードパーティの不要な介入から可能な限り保護します。

白いIPアドレスがない場合は、クラウドに接続できます。これは、コマンドを解読する機能なしに、仲介者として機能します。キーはサーバー上にあります。 

入手先


サーバー側は既存のほとんどすべてのプラットフォームで実行できます-NodeJSのおかげです。最も一般的なプラットフォームでは、すべての作業を自動的に行うスクリプトを用意しています。

Debian Stretchに基づくRaspberry Piの場合：

wget -qO- "http://bary.io/install?target=pi" | sudo bash

targetパラメーターはターゲットプラットフォームを担当し、次の値を持つことができます。
Raspberry Pi（Debian Stretch）
 パイ
Raspberry Pi（Debian Buster）
 pi_buster
ティンカーボード（Debian Stretch）
 tb
Wiren Board（Debian Stretch）
 wb

誰かが別のプラットフォームにインストールしたい場合は、私たちに書いてください。スクリプトを更新します。何か問題が発生した場合は、書いてください。私たちは本当にフィードバックが必要です。 

このアプリケーションは、Google PlayおよびApp Storeで無料で入手できます。おそらく年末までに、アプリケーションは有料になります。

結論


なぜこの記事を書いたのですか？主な目標はあなたからのフィードバックを得ることです。

現在、プロジェクトは急速に発展しており、私たちのチーム全体が、市場で入手可能なサポートされている機器のリストを最大限に活用しようとしています。プロジェクトに取り組んでいるのは私だけではありませんが、家のスマートソリューションの独立した設置に携わったすべての人の希望を考慮し、問題を解決する最も便利なアプリケーションを作成するというタスクは変わりませんでした。

統合の可能性について対話することができ、パートナーシップに関心のある企業からの機器サポートをできるだけ早く実装する準備ができています。既製のアプリケーションを入手でき、ソフトウェアの開発に時間を費やす必要はありません。そして、あらゆる味と色に対応する幅広いデバイスをサポートしています。すべては順調です。
