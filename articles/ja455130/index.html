継続的展開を組織化するための3つの一般的なツール（継続的展開）



継続的展開は、ソフトウェア開発における特殊なアプローチであり、ソフトウェアのさまざまな機能を迅速、安全、効率的に実装するために使用されます。

主なアイデアは、開発者が完成した製品をユーザーにすばやく提供できる信頼できる自動化プロセスを作成することです。同時に、本番環境には一定の変更が加えられます。これは継続的デリバリーパイプライン（CDパイプライン）と呼ばれます。

Skillboxの推奨事項：実践コース「モバイルデベロッパーPRO」。

私たちはあなたを思い出させる： 「Habr」のすべての読者のために-万ルーブルの割引を「Habr」プロモーションコードを使用して、もちろんすべてのSkillbox登録時。



フローを制御するために、さまざまなツールを使用できます。その中には、有償と完全無料の両方があります。この記事では、すべてのプログラマーに役立つ、開発者の間で最も人気のある3つのソリューションについて説明します。

ジェンキンス


完全にスタンドアロンのオープンソース自動化サーバー。ソフトウェアの組み立て、テスト、配布、展開に関連するあらゆる種類のタスクを自動化するには、彼と協力する価値があります。

PCの最小要件：
  • 256 MBのRAM、1 GBのファイルスペース。

最適な：
  • 1 GBのRAM、50 GBのハードディスク容量。

作業には、追加のソフトウェア-Javaランタイム環境（JRE）バージョン8も必要です。

Linuxへのインストール

まず、Jenkinsリポジトリをシステムに追加する必要があります。cd

/ tmp && wget -q -O-pkg.jenkins.io/debian-stable/jenkins.io.key | sudo apt-key add-echo 'deb pkg.jenkins.io/debian-stable binary /' | sudo tee -a /etc/apt/sources.list.d/je

Jenkinsのインストール：

sudo apt install jenkins

その後、Jenkinsはデフォルトのポート8080でシステム上で使用可能になります。

機能を確認するには、ブラウザーでlocalhost：8080 アドレスを開きます。次に、システムはroot権限を持つユーザーの初期パスワードの入力を要求します。このパスワードは、/ var / lib / jenkins / secrets / initialAdminPasswordファイルにあります。

結論として


これらは3つのツールにすぎませんが、実際には他にもたくさんあります。選択するのは難しいため、追加の側面に注意を払う必要があります。

ツールのオープンソースコードは、それが何であるかを理解し、さらに新しい機能をより速く追加することを可能にします。しかし、何かがうまくいかない場合は、自分自身とコミュニティの助けだけに頼る必要があります。有料ツールは、時には重要なサポートを提供します。

安全が最も重要な場合は、ローカルツールを使用する必要があります。そうでない場合は、SaaSソリューションを選択することをお勧めします。

そして最後に：継続的な展開の本当に効果的なプロセスを確実にするために、基準を策定する必要があります。その具体性により、利用可能なツールの選択範囲を狭めることが可能になります。
