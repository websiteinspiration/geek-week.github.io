次の例を考えてみましょう。要素の最大幅はを超えないようにする必要があります500px。

.element {
    width: min(50%, 500px);
}


ビューポートの幅が1150px（1000pxを超える）の場合、要素の幅は500pxになります。

ブラウザは、50%およびから最小値を選択する必要があります500px。ビューポートの幅は選択結果に影響します。ビューポートの幅の50％がより大きい場合500px、この値は無視され、関数はを返し500pxます。

それ以外の場合、ビューポートの幅の50％がより小さい500px場合、結果の値は要素の幅を調整するために使用されます。表示領域のどの幅で、幅の50％が超えないと500px思いますか？これは明らかに1000pxです。この幅では、関数に渡される値は両方とも同じになります。ビューポートの幅が小さい場合1000px、それから要素の幅を調整するために使用されます。


ビューポートの幅が1000px未満の場合、この幅の50％が要素の幅を調整するために使用されます。

クランプ（）関数


この機能でclamp()は、下限と上限を設定することにより、特定の値の変動範囲を制限できます。最小値、計算された（推奨）値、最大値の3つのパラメーターを取ります。計算された値が関数に渡された最小値と最大値によって制限された制限を超えない場合、その値を返します。

ここに例があります。

.element {
    width: clamp(200px, 50%, 1000px);
}

ここでは、要素の幅を調整し200pxます1000px。これは次第に小さくなってはいけません。の値が50%これらの制限を超えない場合は、この値が使用されます。これがどのように見えるかです。


ビューポートの幅の50％が200px〜1000pxの範囲にある場合、この値は要素の幅を設定するために使用されます。この例では、575pxです。

この例を見てみましょう。

  • 要素の幅が小さくなることはありません200px。
  • 関数の2番目のパラメーターはとして設定され50%、ビューポートの幅が400px次第に大きくなる場合にのみ使用され2000pxます。
  • アイテムの幅はを超えません1000px。

その結果、この関数clamp()では、送信される計算値が変化する可能性のある範囲を指定できると言えます。

▍数式


関数を使用するときにclamp()数式を渡すことができるので、関数を使用する必要がありませんcalc()。この仕様は、各引数でclamp()完全な数式を使用できることを示しています。したがって、ネストされた関数は必要ありませんcalc()。さらに、最終的な値にいくつかの制限を適用する必要がある場合は、関数に3つ以上の引数を渡すことができます。

次の例を見てください。

.type {
  /*  font-size ,   12px  100px */

}

▍ヘッダーのフォントサイズ



柔軟なヘッダーのフォントサイズ調整

関数を使用するための優れたスクリプトclamp()は、柔軟なヘッダーのフォントサイズ調整です。ヘッダーの最小フォントサイズ16pxと最大-が必要であると想像してください50px。関数を使用clamp()することで、フォントサイズをこれらの値内に収めることができます。

.title {
    font-size: clamp(16px, 5vw, 50px);
}

関数clamp()は、上記の問題の完全な解決策です。フォントサイズを設定して、表示領域のさまざまなサイズでテキストにアクセスしやすく、読みやすくすることができます。たとえば、ここで関数を使用すると、min()小さな表示領域に表示されるページのフォントサイズを制御できなくなります。

.title {
    font-size: min(3vw, 24px); /*  ,     */
}

以下はこのアイデアの説明です。


min（）関数を使用すると、小さな画面ではテキストが小さすぎるように見えます。

モバイルデバイスの画面では、フォントサイズが小さすぎます。したがって、フォントサイズを設定する機能だけを使用することはお勧めしませんmin()。もちろん、フォントサイズはメディアクエリを使用して再構成できますが、CSS比較関数を使用する意味は完全に失われます。

すでに述べたように、関数min()は関数内で使用できますmax()。これにより、機能の特徴を再現することができますclamp()。

.title {
    font-size: max(16px, min(10vw, 50px));
}

ここで例を試すことができます。Twitterで

の議論の結果10vw、計算された値としてフォントサイズを使用することはまったく正しくないことがわかりました。つまり、ユーザーがブラウザでズームインすると、アクセシビリティの問題が発生するということです。

したがって、そうすることをお勧めします。

.title {
    font-size: clamp(16px, (1rem + 5vw), 50px);
}

ここでは、式は計算されたフォントサイズとして指定されます(1rem + 5vw)。これは問題を解決します。

min min（）、max（）、clamp（）をサポートしないブラウザーの冗長メカニズム


他の新しいCSS機能の場合と同様に、関数の適用min()、max()およびclamp()、バックアップメカニズムに注意する必要があります。これらのメカニズムを作成するには、次のいずれかの方法を使用できます。

1.バックアップメカニズムの手動設定


ここで、手動設定とは、幅広いブラウザのサポートを享受するプロパティをコードに追加することと理解されます。このプロパティは、CSS比較関数を使用するプロパティの前に配置する必要があります。これは次のようになります。

.hero {
    padding: 4rem 1rem;
    padding: clamp(2rem, 10vmax, 10rem) 1rem;
}

ここでサポートするブラウザーはclamp()、最初の式を無視します。そして、この機能をサポートしないものは、インデントを調整する最初の方法を使用します。

2. CSSディレクティブの使用 サポート


@supportsさまざまなテクノロジーに対するブラウザーのサポートをテストするために設計され たディレクティブを使用して、ブラウザーがCSS比較関数を処理できるかどうかを確認できます。最初の段落で説明した手動のソリューションではなく、このソリューションを使用することを好みます。実際、比較関数をサポートするブラウザはすべて、ディレクティブもサポートする必要があり@supportsます。

.hero {
    /*     ,   min() */
    padding: 4rem 1rem;
}

@supports (width: min(10px, 5vw)) {
   /*   ,   min() */
  .hero {
    padding: clamp(2rem, 10vmax, 10rem) 1rem;
  }
}

contentコンテンツの利用について


CSS比較関数は、それらを使用してより柔軟なWebページを作成する新しい方法を提供しますが、注意が必要です。たとえば、関数を単独で使用しmin()てプロパティを設定するだけでfont-sizeは不十分です。実際のところ、モバイルデバイスでは、この機能のみを使用してサイズが設定されているフォントが小さすぎる可能性があります。特に重要なページコンテンツを設定する場合は、最小値と最大値の設定を制限することを忘れないでください。そうしないと、プロジェクトでの作業のユーザーエクスペリエンスが低下する可能性があります。
