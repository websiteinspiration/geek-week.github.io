面接時に提示する必要があるコード

今日の午前中に、面接でどのコードを書く必要があるか、どのスキルを実証すべきかについて非常に興味深い投稿がありました。テストタスクはこれでした。

TypeScriptで、指定された数値の配列に対して、範囲のテキスト文字列を表示する関数を記述します。

getRanges([0, 1, 2, 3, 4, 7, 8, 10]); // 0-4,7-8,10
getRanges([4, 7, 10]);                // 4,7,10
getRanges([2, 3, 8, 9]);              // 2-3,8-9

残念ながら、この投稿は利用できなくなったため、ここで示した解決策は復元しません。しかし、このソリューションは非常に美しく、締め切りまでに管理職に少し苦しめられた平均的なプログラマにとっては非常に読みにくいものでした。これは、ほとんど1行の書き込み専用のPerlスタイルのスクリプトでした。

そして問題は、このコードを評価する方法ですか？一方では、彼は候補者の高い技術レベルを示しています。一方、まったく異なるレベルのトレーニングを持つプログラマーによるチーム開発プロセスの誤解も示しています。

約10分で決定をスケッチしました。次のようになります。

class Interval {
    start: number;
    stop: number

    constructor(start: number, stop: number) {
        this.start = start;
        this.stop = stop;
    }

    toString(arr: number[]) {
        let text: string;
        
        text = arr[this.start].toString();
        if (this.start < this.stop) {
            text += '-' + arr[this.stop].toString();
        }

        return text;
    }
}

function getRanges(arr: number[]) {
    // find start-stop intervals in the sequence
    let intervals : Interval[] = [];

    let start = 0;
    for (let i = 1; i < arr.length; i++) {
        if (arr[i] - arr[i - 1] > 1) {
            intervals.push(new Interval(start, i - 1));
            start = i;
        }
    }
    intervals.push(new Interval(start, arr.length - 1));

    // convert intervals to the string
    let out : String = "";
    for (let i = 0; i < intervals.length; i++) {
        out += intervals[i].toString(arr);

        if (i < intervals.length - 1) {
            out += ',';
        }
    }

    console.log(out);
}

getRanges([0, 1, 2, 3, 4, 7, 8, 10]); // 0-4,7-8,10
getRanges([4, 7, 10]);                // 4,7,10
getRanges([2, 3, 8, 9]);              // 2-3,8-9

ここでのアルゴリズムはこれです：

  • 最初のステップでは、数値の配列から間隔を選択します
  • 2番目のステップでは、間隔を文字列に変換します

しかし、それから私は自分の決定を見ました。そして、それも完璧に見えたわけではありません。やや冗長で、追加のエンティティ「Interval」が導入されています。しかし、それでも、この決定の方が読みやすく、理解しやすいと確信しています。

しかし、ここに質問があります。オリンピアードプログラミングファンの観点から、この決定はインタビューでどのように評価されますか？

一般に、私はこの議論を続けたいと思います。
