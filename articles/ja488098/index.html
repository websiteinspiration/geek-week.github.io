テンプレートからDjangoプロジェクトを作成する方法

また会いましたね！特に「PythonでのWeb開発者」コースの学生のために、興味深い翻訳をもう1つ用意しました。




Djangoプロジェクトテンプレートは、デフォルトのDjangoプロジェクト形式が要件を満たさなくなったときに発生する問題を解決する自然な方法です。このガイドでは、今日、テンプレートから独自のプロジェクトを作成する方法を学びます。

Djangoのプロジェクトテンプレートとは何ですか？


Djangoのあまり知られていない機能の1つは、テンプレート、つまりカスタムディレクトリ構造からプロジェクトを作成する機能です。

これは、デフォルトのプロジェクト形式では不十分な場合（そして、ソリューションを本番環境にデプロイしたい場合は間違いなく十分ではありません）、または同じ構成を何度も何度も使用するようになる場合に便利です。 Django上の多くのプロジェクト。

Djangoプロジェクトテンプレートは、数分で自分で作成できる事前定義されたカスタムディレクトリ構造を持つプロジェクトにすぎません。
人気のあるDjangoテンプレートの例は、django-cookiecutterです。Django cookiecutterこれには、本番環境への自己完結型デプロイメントの準備がすでに整っている幅広い機能と構成があります。これは、Djangoでほとんどすべての新しいプロジェクトを作成するために使用できるパスです。

ただし、特に初心者にとっては、django-cookiecutterが冗長になることがあります。そのため、使用する前に、基本を理解するのに役立つ簡単なDjangoテンプレートを作成してください。

次のセクションでは、独自のテンプレートを作成する方法を学習します。

独自のDjangoプロジェクトテンプレートを作成する方法


開始するには、選択したフォルダーにPython仮想環境を作成し、Djangoをインストールします（これらは4つの個別のコマンドであることに注意してください）。

mkdir project-name && cd $_
python3 -m venv venv
source venv/bin/activate
pip install django

次に、以下を使用してDjangoプロジェクトを作成します。 

django-admin startproject project_name .

名前に注意してください。正確にproject_nameである必要があります。その後、エディターでプロジェクトを開きます。

（この例では、フォルダーとプロジェクト名にproject_nameを使用したことに注意してください）。

次に、プロジェクト内の各ファイルを確認し、すべてのproject_nameを{{project_name}}に置き換える必要があります（単一行のsedコマンドで問題ありません）。

変更する必要があるファイル：

  • manage.py
  • settings.py
  • wsgi.py
  • asgi.py

ご想像のとおり、{{project_name}}変数はプレースホルダーです。たとえば、manage.pyでは、次のものを置き換える必要があります。

# manage.py

def main():
    os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'project_name.settings')
    try:
# omit

これについて：

# manage.py

def main():
    os.environ.setdefault('DJANGO_SETTINGS_MODULE', '{{ project_name }}.settings')
    try:
# omit

でwsgi.pyあなたはこれを変更します。

os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'project_name.settings')

次へ：

os.environ.setdefault('DJANGO_SETTINGS_MODULE', '{{ project_name }}.settings')

等。

さらに、開発、テスト、本番用にDjango設定を個別のファイルに解析して、必要に応じて、またはさらに優れた構成を作成できます。ここに例があります。

次の手順に進む前に、必ずプロジェクトテンプレートから仮想環境を削除してください。

独自のテンプレートからDjangoプロジェクトを作成する方法


これでプロジェクトテンプレートが作成されたので、それを使用できます。

これを行うには、別のフォルダーに新しいPython仮想環境を作成し、Djangoをインストールします。

mkdir new-django-project && cd $_
python3 -m venv venv
source venv/bin/activate
pip install django

django-admin startprojectを 実行するだけでなく、テンプレートからインストールする--templateフラグを渡します。〜/ project-nameホームディレクトリにテンプレートを作成したとすると、次のようになります。

django-admin startproject --template ~/project-name new_django_project .

このコマンドは、テンプレートから新しいDjangoプロジェクトを作成します。

（この例では、外部フォルダーとしてnew-django-projectを使用し、プロジェクト名としてnew_django_projectを使用したことに注意してください）。

たとえば、GitHubのリポジトリからリモートテンプレートを使用できるかどうか疑問に思うかもしれません。はい、もちろん！

リモートテンプレートからDjangoプロジェクトを作成する方法


startproject の--templateフラグは、URLをリモートソースとして受け入れることができます。したがって、リモートテンプレートからDjangoプロジェクトを作成できます。次に例を示します。

django-admin startproject --template https://github.com/username/repo/archive/master.zip new_django_project .
