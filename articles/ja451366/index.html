コードを少なくする

フレームワークのバージョン3の最近のリリース後、Hype on Svelteは、公式チャットとロシア語電報チャネルの両方で非常に増加したコミュニティと膨大な数の質問によって証明されるように、まだ場所があります。ますます多くの開発者がこっそりまたは真剣にこのテクノロジーを見て、新しいプロジェクトでのそのアプリケーションについて考えています。これらの開発者や、コードをできるだけ効率的に作成するトピックに関心のある他のすべての人のために、フレームワークの作成者およびイデオロギーであるRich Harrisは、Svelteが開発者が最新のリアクティブWebアプリケーションを作成する労力を最小限に抑えるのにどのように役立つかについての記事を公開しました。


コードにバグがある可能性があります。したがって、アプリケーションで発生する可能性のあるバグの数は、作成するコードの量とともに増加します。


明らかに、私たちが書かなければならないコードが多いほど、時間がかかります。まもなく、最適化、アプリケーションへの新機能の追加などの必要なことや、ラップトップで持ち上げるのではなく屋外で歩くには十分ではなくなります。


実際、アプリケーションコードベースの増加に伴って、プロジェクトの開発時間とバグの数は、線形ではなく2次の依存関係でも増加することが広く知られています。これは、10行のプルリクエストに簡単に注意を向けるときの潜在意識の動作を説明できます。そのレベルの注意は、100行を超えるコードにはめったに行きません。そして、コードが長くなりすぎて1つの画面に収まらなくなるとすぐに、コードを理解するために必要な認知作業が大幅に増加します。私たちは、リファクタリングしてコメントを追加することで状況を修正しようとしています。これは悪循環です。


私たちは皆少し夢中ですよね？アプリケーションのパフォーマンス、バンドルのサイズ、および少なくとも何らかの方法で測定できるその他すべてを監視しますが、記述するコードの量に注意を払うことはほとんどありません。


コードの可読性は重要です


もちろん、読みやすさを損なうことなくコードをできるだけコンパクトにするために、トリッキーなトリックを使用すべきだと言っているのではありません。また、このように通常の読み取り可能なコードを変換するのに役立つため、コード行を減らすこと自体が目的であると言っているわけではありません...


for (let i = 0; i <= 100; i += 1) {
    if (i % 2 === 0) {
        console.log(`${i} — `);
    }
}

...難消化性のものに：


for (let i = 0; i <= 100; i += 1) if (i % 2 === 0) console.log(`${i} — `);

自然な方法でより少ないコードを記述できるようにする言語とパターンを優先する方が良いことをあなたに納得させたいだけです。
