クラスインスタンスに指定された名前の属性がない場合は、同じ名前のクラス属性にアクセスしようとします。

>>> class A:
...     x = 2
...
>>> A.x
2
>>> A().x
2

インスタンスは、クラスが持たない属性を持つことも、異なる値を持つ属性を持つこともできます。

>>> class A:
...     x = 2
...     def __init__(self):
...         self.x = 3
...         self.y = 4
...
>>> A().x
3
>>> A.x
2
>>> A().y
4
>>> A.y
AttributeError: type object 'A' has no attribute 'y'

インスタンスに属性がないかのように動作させたい場合は、クラスには属性がありますが、このインスタンスからのアクセスを禁止するカスタムハンドルを作成する必要があります。

class ClassOnlyDescriptor:
    def __init__(self, value):
        self._value = value
        self._name = None  # see __set_name__

    def __get__(self, instance, owner):
        if instance is not None:
            raise AttributeError(
                f'{instance} has no attribute {self._name}'
            )

        return self._value

    def __set_name__(self, owner, name):
        self._name = name


class_only = ClassOnlyDescriptor


class A:
    x = class_only(2)


print(A.x)  # 2
A().x       # raises AttributeError

クラス本体で宣言された関数は、このクラスのスコープからアクセスできません。これは、このスコープがクラスの作成中にのみ存在するためです。

>>> class A:
...     x = 2
...     def f():
...         print(x)
...     f()
...
[...]
NameError: name 'x' is not defined

通常、これは問題ではありません。メソッドは、メソッドになるためにクラス内で宣言され、後で呼び出されます。

>>> class A:
...     x = 2
...     def f(self):
...         print(self.x)
...
>>>
>>>
>>> A().f()
2

驚いたことに、理解力についても同じことが言えます。それらには独自のスコープがあり、クラスのスコープにアクセスすることもできません。これは、ジェネレーター内包の観点からは非常に論理的です。クラス内のコードは、クラスが既に作成されているときに実行されます。

>>> class A:
...     x = 2
...     y = [x for _ in range(5)]
...
[...]
NameError: name 'x' is not defined

ただし、内包表記にはへのアクセス権がありませんself。これへのアクセスを提供する唯一の方法xは、別のスコープを追加することです（ええ、愚かなソリューション）：

>>> class A:
...     x = 2
...     y = (lambda x=x: [x for _ in range(5)])()
...
>>> A.y
[2, 2, 2, 2, 2]


PythonはNone同等Noneなので、次のNone使用を確認できるように見えるかもしれません==。

ES_TAILS = ('s', 'x', 'z', 'ch', 'sh')


def make_plural(word, exceptions=None):
    if exceptions == None:  # ← ← ←
        exceptions = {}

    if word in exceptions:
        return exceptions[word]
    elif any(word.endswith(t) for t in ES_TAILS):
        return word + 'es'
    elif word.endswith('y'):
        return word[0:-1] + 'ies'
    else:
        return word + 's'

exceptions = dict(
    mouse='mice',
)

print(make_plural('python'))
print(make_plural('bash'))
print(make_plural('ruby'))
print(make_plural('mouse', exceptions=exceptions))

しかし、それは間違いです。はい、None等しいNoneですが、それだけではありません。カスタムオブジェクトも同じにすることができますNone：

>>> class A:
...     def __eq__(self, other):
...             return True
...
>>> A() == None
True
>>> A() is None
False

Pythonの浮動小数点数はNaN値を持つことができます。たとえば、そのような数はで取得できますmath.nan。nanあなた自身を含む何にも等しくない：

>>> math.nan == math.nan
False

さらに、NaNオブジェクトは一意ではなく、さまざまなソースからいくつかの異なるNaNオブジェクトを持つことができます。

>>> float('nan')
nan
>>> float('nan') is float('nan')
False

つまり、通常、NaNを辞書キーとして使用することはできません。

>>> d = {}
>>> d[float('nan')] = 1
>>> d[float('nan')] = 2
>>> d
{nan: 1, nan: 2}


typingジェネレータのタイプを定義できます。さらに、生成されるタイプ、ジェネレータに渡されるタイプ、およびを使用して返されるタイプを指定できますreturn。たとえば、Generator[int, None, bool]整数を生成し、ブール値を返し、はサポートしていませんg.send()。

しかし、例はもっと複雑です。chain_while他のジェネレータからのデータを、それらの1つが関数に従って停止信号である値を返すまでプロキシしますcondition。

from typing import Generator, Callable, Iterable, TypeVar

Y = TypeVar('Y')
S = TypeVar('S')
R = TypeVar('R')


def chain_while(
    iterables: Iterable[Generator[Y, S, R]],
    condition: Callable[[R], bool],
) -> Generator[Y, S, None]:
    for it in iterables:
        result = yield from it
        if not condition(result):
            break


def r(x: int) -> Generator[int, None, bool]:
    yield from range(x)
    return x % 2 == 1


print(list(chain_while(
    [
        r(5),
        r(4),
        r(3),
    ],
    lambda x: x is True,
)))


ファクトリメソッドの注釈の設定は、見かけほど簡単ではありません。次のようなものを使いたいだけです：

class A:
    @classmethod
    def create(cls) -> 'A':
        return cls()

しかし、それは間違っているでしょう。トリックはそれがあることでcreateはない返しA、それを返し、clsあるA いずれかの子孫の一つ。コードを見てください：

class A:
    @classmethod
    def create(cls) -> 'A':
        return cls()


class B(A):
    @classmethod
    def create(cls) -> 'B':
        return super().create()

mypyチェックの結果はエラーerror: Incompatible return value type (got "A", expected "B")です。繰り返しになりますが、問題はそれがsuper().create()returnとして注釈されていることですAが、この場合はreturn Bです。

これはcls、TypeVar次のように注釈を付けると修正できます。

AType = TypeVar('AType')
BType = TypeVar('BType')


class A:
    @classmethod
    def create(cls: Type[AType]) -> AType:
        return cls()


class B(A):
    @classmethod
    def create(cls: Type[BType]) -> BType:
        return super().create()

今、createクラスのインスタンスを返しますcls。ただし、これらの注釈は曖昧すぎて、clsサブタイプとは何かに関する情報が失われていますA。

AType = TypeVar('AType')


class A:
    DATA = 42
    @classmethod
    def create(cls: Type[AType]) -> AType:
        print(cls.DATA)
        return cls()

エラーが発生します"Type[AType]" has no attribute "DATA"。

これを修正するには、明示的にATypeサブタイプとして定義する必要がありますA。このためTypeVar、引数とともに使用されますbound。

AType = TypeVar('AType', bound='A')
BType = TypeVar('BType', bound='B')


class A:
    DATA = 42
    @classmethod
    def create(cls: Type[AType]) -> AType:
        print(cls.DATA)
        return cls()


class B(A):
    @classmethod
    def create(cls: Type[BType]) -> BType:
        return super().create()
