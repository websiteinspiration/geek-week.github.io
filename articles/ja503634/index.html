1.オブジェクトの正確なタイプを判別する


我々はすべて知っているように、JavaScriptで6つのプリミティブデータ型があります（Boolean、Number、String、Null、Undefined、Symbol）、および1つのオブジェクトタイプ- Object。異なるオブジェクト値のタイプを区別する方法を知っていますか？オブジェクトは配列または関数にすることができ、値のコレクションMapまたはその他のものにすることができます。オブジェクトの正確なタイプを見つけるために何をする必要がありますか？

この質問への答えを探して前に、の違いについて考えてみましょうObject.prototype.toString.call(arg)とString(arg)。

これらの式の使用は、渡されたパラメーターを文字列に変換することを目的としています。ただし、動作は異なります。

呼び出されると、String(arg)システムはarg.toString()またはを呼び出そうとしますarg.valueOf()。その結果、argプロトタイプ内またはプロトタイプ内argこれらのメソッドは上書きされ、呼び出されObject.prototype.toString.call(arg)、String(arg)異なる結果が得られます。

例を考えてみましょう。

const _toString = Object.prototype.toString
var obj = {}
obj.toString()  // [object Object]
_toString.call(obj) // [object Object]

この場合、通話中obj.toString()とObject.prototype.toString.call(obj)同じ結果になります。

    return _toString.call(value).slice(8, -1)

2.関数の結果をキャッシュする


長い計算を実行する次のような関数があるとします。

function computed(str) {    
    console.log('2000s have passed')
    return 'a result'
}

このような関数を作成することで、返された結果をキャッシュする予定です。この関数が次回呼び出されたときに、以前と同じパラメーターを渡した場合、「重い」関数コードは実行されません。代わりに、キャッシュされた結果が余分な時間なしに返されます。どうやってするの？

たとえば、目的関数のラッパー関数を記述できます。このような関数には名前を付けることができますcached。この関数は、目的関数を引数として受け入れ、キャッシュ機能を備えた新しい関数を返します。関数でcachedは、エンティティObjectまたはを使用して、ターゲット関数への以前の呼び出しの結果をキャッシュできますMap。この関数のコードは次のとおりです。

function cached(fn){
  //     ,      
  const cache = Object.create(null);

  //      
  return function cachedFn (str) {

    //       -   
    if ( !cache[str] ) {
        let result = fn(str);

        //      
        cache[str] = result;
    }

    return cache[str]
  }
}

3. hello-world形式の文字列をhelloWorld形式の文字列に変換します


複数のプログラマーが同じプロジェクトで一緒に作業する場合、コードの統一されたスタイルに注意を払うことが非常に重要です。たとえば、誰かがある形式helloWorldで複合識別子を記録し、ある形式で誰かを記録することができますhello-world。この領域に秩序をもたらすために、ビューhello-worldラインをビューラインに変換する関数を作成できますhelloWorld。

const camelizeRE = /-(\w)/g
const camelize = cached((str) => {
  return str.replace(camelizeRE, (_, c) => c ? c.toUpperCase() : '')
})
camelize('hello-world')
// "helloWorld"

4. JavaScriptコードが実行される環境を決定する


現在、ブラウザーの急速な発展により、JavaScriptコードはさまざまな環境で実行できます。プロジェクトをさまざまな環境に適合させるには、プログラムが実行される場所を特定できる必要があります。

const inBrowser = typeof window !== 'undefined'
const inWeex = typeof WXEnvironment !== 'undefined' && !!WXEnvironment.platform
const weexPlatform = inWeex && WXEnvironment.platform.toLowerCase()
const UA = inBrowser && window.navigator.userAgent.toLowerCase()
const isIE = UA && /msie|trident/.test(UA)
const isIE9 = UA && UA.indexOf('msie 9.0') > 0
const isEdge = UA && UA.indexOf('edge/') > 0
const isAndroid = (UA && UA.indexOf('android') > 0) || (weexPlatform === 'android')
const isIOS = (UA && /iphone|ipad|ipod|ios/.test(UA)) || (weexPlatform === 'ios')
const isChrome = UA && /chrome\/\d+/.test(UA) && !isEdge
const isPhantomJS = UA && /phantomjs/.test(UA)
const isFF = UA && UA.match(/firefox\/(\d+)/)

5.組み込み関数とユーザー定義関数の区別


JavaScriptには2種類の機能があることが知られています。最初のタイプは組み込み、または「ネイティブ」関数とも呼ばれます。このような機能は、コードが実行される環境によって提供されます。2番目のタイプは、いわゆる「ユーザー関数」、つまりプログラマー自身が作成するものです。これらの関数は、文字列に変換するとさまざまな結果が返されることを考慮して、区別できます。

Array.isArray.toString() // "function isArray() { [native code] }"
function fn(){} 
fn.toString() // "function fn(){}"

このコードをコンソールで実験します。


toString()ネイティブ関数 メソッドは常に次の形式の構造を返します。

function fnName() { [native code] }

これを知っていれば、ネイティブ関数とユーザー定義関数を区別できる関数を作成できます。

function isNative (Ctor){
  return typeof Ctor === 'function' && /native code/.test(Ctor.toString())
}

→  これは、そのような関数があるVueコードベースの場所です。
