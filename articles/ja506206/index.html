ICQ New：ボット育種ガイド



メッセンジャーに入るたびに、ボットに最も多様な形で出会います。天気について話したり、ハンバーガーを遊んだり、ミームを投げたりします。きっと多くの人の間で思い浮かんだ考えは、「自分でボットを作ってもいいのか？」残念ながら、そのような考えは、ボットを作成する方法についての誤解についてしばしば破られています。おそらく、このためには、クールなITスペシャリストになり、何百万ものテクノロジーを理解する必要がありますか？あんまり。そして今日、私たちはボットの作成がシンプルで理解できるプロセスであることを示しようとします。メッセンジャーから必要なデータを取得してからコードを記述してサーバーで起動するまでのボット作成の全サイクルを分析してみましょう。

しばらく前に、ボットプラットフォームはICQで大幅に更新されました。彼女はよりフレンドリーで、理解しやすく、快適になりました。開発者のPythonライブラリを使用して、最初のボットを作成します。

初めにすること


まず、ICQに登録する必要があります。これは、携帯電話、コンピューターのアプリケーションから、またはWebバージョンのブラウザーから直接実行できます。

登録後、システムに独自のボットを確立できます。

  1. 見つけるには メタボット ICQで。
  2. 彼にチームを書きなさい/newbot。
  3. （で終わる必要があり、あなたの新しいボットの名前を送信ボット）。

このメタボットがボットのデータを送信した後：

  • botId：一意のボット番号。
  • nick：検索するボットの名前。
  • token：サーバーへのネットワーク要求に使用されるトークン。

これでボットが作成されました。彼は検索で見つけることができます、彼へのメッセージを書いてください。次に、ボットをアクティブにして応答する必要があります。

ページhttps://icq.com/botapi/#/には、APIメソッドの完全な説明があります。これらはサーバーとの対話に使用されます。それらのいくつかをさらに詳しく見てみましょう。

基本的な方法は/events/getです。新しいボットイベントを受信するために使用されます。たとえば、誰かがボットを作成した場合、このイベントはプロンプトが表示されたときに表示され/events/getます。ボットにメッセージを書いて、対応するイベントの外観を確認しましょう。これは、たとえばブラウザーで行うことができます。これを行うには、https：//api.icq.net/bot/v1/events/get？token = Your token＆pollTime = 1＆lastEventId = 0にアクセスします。パラメータpollTimeリクエストがサーバーによって保持される時間の長さを管理します。たとえば、値60を入力すると、サーバーはボットのイベントを60秒間待機し、この間にイベントがない場合、サーバーは空のイベント配列を返します。lastEventId最後に処理されたイベントを担当します。つまり、送信された値よりも小さい値を持つイベントはクリップされます。

書いています！


例として、単純な数学の知識をテストするゲームボットを作成します。彼は次のように機能します。人がボットにメッセージを書き込み、自己紹介し、プレイを申し出ます。ユーザーが同意すると、ボットは簡単な数学的例と4つのオプションから選択できます。ユーザーが選択した答えでボタンをクリックすると、ボットが正解します。

したがって、ボットは最初にサーバーからイベントを受信する必要があります。これを行うには、すべてのライブラリをインポートすることに加えて、クラスオブジェクトを作成しBotて更新の受信を開始する必要があります。

import logging
from bot.bot import Bot


logging.basicConfig(format='%(asctime)s %(levelname)s %(message)s',
                    datefmt='%Y.%m.%d %I:%M:%S %p', level=logging.DEBUG)

TOKEN = "" #your token here

def main():
    bot = Bot(token=TOKEN)
    bot.start_polling()

    main()

ここで、ロギングライブラリをインポートして実行します。また、ICQボットのライブラリをインポートし、必要なパラメータを設定しtoken、クラスオブジェクトBotを作成して、サーバーから更新の受信を開始します（bot.start_polling()）。

これで、ボットはイベントを受信できますが、処理できません。メッセージに応答できるようにボットを補足します。これを行うには、必要なインポートを追加します。

import json
from bot.handler import MessageHandler

jsonボタンを担当するメッセージの部分をフォーマットするために必要。MessageHandlerハンドラーコードに新しいメッセージを追加する必要があります。

次に、応答のメッセージを作成する関数が必要です。

def startup(bot, event):
    default_markup = [
        [{"text": "!", "callbackData": "start"}]
        ]

    first_message_text = "!  -  . ?"
    bot.send_text(chat_id=event.from_chat,
         text=first_message_text,
         inline_keyboard_markup=json.dumps(default_markup))

作成されたボットの入力と新しいメッセージのイベントを受け取ります。default_markup-ボタンの行列（ボタン文字列の配列で構成される「リスト」オブジェクト。この例では、これが唯一のスタートボタンです！）。first_message_text-文字列。テキストはユーザーに送信されます。このメソッドsend_textは以下を受け入れます。

  • chat_id （処理中のイベントのフィールド）-メッセージの受信元のチャットの識別子
  • 送信するメッセージのテキスト
  • そしてinline_keyboard_markup-私たちのボタン。

メッセージを受信したときのこの関数の呼び出しについて説明します。このため、ライブラリには特別なデザインがあります。

bot.dispatcher.add_handler(MessageHandler(callback=startup))

これにより、新しいメッセージを受信したときにstartup()、必要なパラメーターを持つ関数を自動的に呼び出すことができます。このハンドラは関数に配置する必要がありますmain()。ボットがメッセージを受信したときに開始メッセージを送信できるようなコードを取得しました。

メッセージで送信したボタンをさらに詳しく検討します。

[ [{«text»: «!», «callbackData»: «start»}]]

ボタンに書かれたテキストに加えて、メッセージにはフィールドがありますcallbackData。ボタンがクリックされると、サーバーはボタンクリックイベントを生成します。その中で、彼はこのフィールドを、押されたボタンの識別子として渡します。つまり、サーバーから受信したこの行により、ユーザーがどのボタンをクリックしたかを判断できます。したがって、さらにボタンクリック処理ロジックをボットに追加する必要があります。これを行うには、必要なメソッドを再度インポートする必要があります。

from bot.handler import BotButtonCommandHandler
from bot.filter import Filter

1行目はボタンをクリックするためのイベントハンドラー、2行目はどのボタンが押されたかを理解するのに役立つフィルターです。ボタンのクリックに対する応答のロジックについても説明する必要があります。

def start(bot, event):
    question = ''
    operands = ["+", "-", "*"]
    operations_count = randrange(3)+2
    for i in range(operations_count):
        question += str(randrange(9))
        question += choice(operands)
    question = question[:-1]
    answer = eval(question)
    buttons = [
        [{'text': str(answer), 'callbackData': "right"},
         {'text': str(answer+1), 'callbackData': "wrong"}],
        [{'text': str(answer-1), 'callbackData': "wrong"},
         {'text': str(answer+2), 'callbackData': "wrong"}]]
    shuffle(buttons[0])
    shuffle(buttons[1])
    shuffle(buttons)
    bot.answer_callback_query(
        query_id=event.data['queryId'],
        text=' ')
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text=question,
        inline_keyboard_markup=json.dumps(buttons))

メッセージからの類推により、関数は同じ引数を入力として受け入れます。次に、例と回答付きのボタンの配列が生成されます。正解は1つです。これに基づいて、答えは異なりcallbackDataます。最後に、必ずanswer_callback_queryパラメーターを指定してメソッドを呼び出す必要があります。

  • query_id -ボタンを押すイベントの一意の番号、
  • およびtext-ボタンをクリックしたときにツールチップに表示されるテキスト。

このメソッドを呼び出すことにより、ボットがボタンのクリックを受け入れたことをユーザーに示します。それ以外の場合、ダウンロードインジケータがしばらくボタンに表示されます。メソッドを使用send_textして、質問と回答のボタンでメッセージ自体を送信する必要があります。

正解と不正解を処理する関数をすぐに追加してみましょう。

def right(bot, event):
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text='!')
    start(bot, event)


def wrong(bot, event):
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text=':(')
    start(bot, event)

これらの関数は、正解と不正解をクリックすると呼び出されます。そして、正しい答えのメッセージの後に、ユーザーは新しい質問を受け取ります。

あとは、ボタンクリックハンドラーを追加するだけです。

    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=start, filters=Filter.callback_data(«start»)))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=wrong, filters=Filter.callback_data(«wrong»)))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=right, filters=Filter.callback_data(«right»)))

これら3つのハンドラーのそれぞれには、フィルターcallbackData（例filters=Filter.callback_data(«start»)：）と呼び出す関数が含まれています。

これでボットのプログラミングが完了しました。結果のコードは次のようになります。

import logging
import json
from bot.bot import Bot
from bot.handler import BotButtonCommandHandler
from bot.handler import MessageHandler
from bot.filter import Filter
from random import randrange, choice, random, shuffle

logging.basicConfig(format='%(asctime)s %(levelname)s %(message)s',
        datefmt='%Y.%m.%d %I:%M:%S %p', level=logging.DEBUG)

TOKEN = "" #your token here


def startup(bot, event):
    default_markup = [
        [{"text": "!", "callbackData": "start"}]
        ]

    first_message_text = "!  -  . ?"
    bot.send_text(chat_id=event.from_chat,
        text=first_message_text,
        inline_keyboard_markup=json.dumps(default_markup))


def start(bot, event):
    question = ''
    operands = ["+", "-", "*"]
    operations_count = randrange(3)+2
    for i in range(operations_count):
        question += str(randrange(9))
        question += choice(operands)
    question = question[:-1]
    answer = eval(question)
    buttons = [
        [{'text': str(answer), 'callbackData': "right"},
         {'text': str(answer+1), 'callbackData': "wrong"}],
        [{'text': str(answer-1), 'callbackData': "wrong"},
         {'text': str(answer+2), 'callbackData': "wrong"}]]
    shuffle(buttons[0])
    shuffle(buttons[1])
    shuffle(buttons)
    bot.answer_callback_query(
        query_id=event.data['queryId'],
        text=' ')
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text=question,
        inline_keyboard_markup=json.dumps(buttons))


def right(bot, event):
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text='!')
    start(bot, event)


def wrong(bot, event):
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text=':(')
    start(bot, event)


def main():
    bot = Bot(token=TOKEN)
    bot.dispatcher.add_handler(MessageHandler(callback=startup))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=start, filters=Filter.callback_data("start")))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=wrong, filters=Filter.callback_data("wrong")))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=right, filters=Filter.callback_data("right")))
    bot.start_polling()


if __name__ == '__main__':
    main()

関数が質問を生成するために必要なインポートが含まれていることに注意します。この関数や他の関数の代わりに、独自のロジックを書くことができます:)

どこでどのように実行しますか？


リモートサーバーでボットを起動するのが最も便利です。プログラムをインストールするための十分なアクセス権を持つ仮想サーバーを提供する多くのサービスがあります。一部のサービスは無料の試用期間を提供します。Mail.ruなので、Mail.ru Cloud Solutionsの例を使用して分析します。簡単な登録プロセス、サーバーへの迅速なアクセス、そして無料の試用期間があり、銀行カードやその他の問題はありません。

  1. https://mcs.mail.ru/でアカウントを登録します。
  2. 仮想マシンのインスタンスを作成します。すべての設定は変更されずに残され、十分です。
  3. サービスは、アクセスキーのダウンロードを提供します。ダウンロード。
  4. 次に、仮想マシンに移動します。ssh -i ./xxx.pem user@ip

    ここで./xxx.pem、ダウンロードしたキーへのパスです。
  5. その後、必要なプログラムとパッケージをインストールする必要があります。

    $ sudo curl https://raw.githubusercontent.com/dvershinin/apt-get-centos/master/apt-get.sh -o /usr/local/bin/apt-get
$ chmod 0755 /usr/local/bin/apt-get
$ sudo apt-get install python3
$ curl https://bootstrap.pypa.io/get-pip.py -o get-pip.py
$ python3 get-pip.py
$ pip3 install --upgrade mailru-im-bot

    ここで、ボットが機能するために必要なすべてのライブラリとPythonをインストールしました。
  6. ボットをサーバーにコピーして実行するために残っています。スクリプトのあるフォルダーから、次を実行します。scp -i ./xxx.pem ./path/to_bot.py user@ip:~

    • ./xxx.pem -キーへのパス。
    • ./path/to_bot.py -ボットを含むファイルへのパス。
    • user@ip:~ -ボットをコピーするログイン、アドレス、およびパス。

    その後、ボットを含むファイルはリモートサーバーのホームディレクトリに配置されます。
  7. サーバーに戻って実行します。python3 ~/file_name.pyここfile_name.pyで、ボットを含むファイルの名前です。
