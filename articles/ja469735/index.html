任意の正の数からn次の根を計算するアルゴリズム



面白いパズルに出くわしました。確かに、数字aと正の整数nが与えられています。ライブラリを使用せずに、数値のn番目の根を計算します。

入力データ：数値aは実数で、負ではなく、1000を超えず、小数点以下6桁の精度で指定されます。数値nは自然な値で、10を超えません。
出力：プログラムは単一の数値を出力する必要があります：小数点以下5桁の精度での問題への答え。

当然、ドラフトで鉛筆でそれを解決し、それをエディターで描画してコンパイルしてみるのは興味深いことでした。グーグルせずに、ヒント、さらにはライブラリの使用。これを初めて解決する場合は、まず通常の平方根を見つけるプログラムを書いてみてください。タスクが難しいと思われる場合は、ほぼ同じように解決しますが、より簡単です。そうすればあなたの恐れはなくなり、ある種の大まかな理解が生まれます。

まず、ライブラリ関数を使用せずに平方根を計算する方法の例を示します。逐次反復アルゴリズム。多数の場合でも、非常に速く収束します。

/* Calculating the square root by iterations */
 
#include <stdio.h>
 
int main(void) {
    double num = 570.15;
    double root = num / 2;
    double eps = 0.01;
    int iter = 0;
    while( root - num / root > eps ){
        iter++;
        root = 0.5 * (root + num / root);
        printf("Iteration: %d : root = %f\n", iter, root);
    }
    printf("root = %f", root);
    return 0;
}

アルゴリズムの対数の複雑さ？または別ですか？ :)

これで、タスクの複雑なバージョンに進むことができます。この場合、ソリューションはより一般化されます。

#include <stdio.h>
 
double mabs(double x){ return (x < 0)? -x : x; }
 
int main(void) {
    double num = 8;
    int rootDegree = 3;
 
    printf(",     = %f\n", num);
    printf("  n = %d\n", rootDegree);
   
    double eps = 0.00001;             // 
    double root = num / rootDegree;   //  
    double rn = num;                  //   
    int countiter = 0;                // 
    while(mabs(root - rn) >= eps){
        rn = num;
        for(int i = 1; i < rootDegree; i++){
            rn = rn / root;
        }
        root = 0.5 * ( rn + root);
        countiter++;
    }
   
    printf("root = %f\n", root);
    printf("  = %i\n", countiter);
   
    return 0;
}

このソリューションでは、比較的良い初期近似のアイデアを使用します。次に、逐次除算は、n次の根の2番目の近似です。次に、2つの現在のものを平均することによって、新しい近似が考慮されます。一貫して、アルゴリズムは所定のエラーで目的のルートに収束します。これは、単純な反復法に少し似ています。

これは、ひざの上で書かれた最初の実用的なアルゴリズムです。それでも、加速の複雑さと可能性について考える必要があります。ところで、あなたの意見では、このアルゴリズムの加速のどの機能を実装できますか？

「100年前にすべてがライブラリに実装されているのに、なぜこれを行うのか？」という疑問があると思います。
