CloudFlareを使用したカスタム動的DNS

序文


個人的なニーズのために、彼はVSphereを提起しました。その上で、私は仮想ルーターとUbuntuサーバーをメディアサーバーとたくさんのグッズとしてひねり、このサーバーはインターネットからアクセスできるはずです。しかし問題は、私のプロバイダーが静物をお金で提供していることです。あなたはいつでもより有用なアプリケーションを見つけることができます。したがって、一連のddclient + cloudflareを使用しました。

ddclientが機能しなくなるまで、すべてが順調でした。少し掘り下げてみると、松葉杖や自転車の時期が来ていることに気づきました。問題の発見に時間がかかり始めました。結局、すべてがうまく機能する小さな悪魔に変わりましたが、もう必要ありません。
誰も気にしない-猫へようこそ。

使用するツールと「それ」の仕組み


したがって、私がcloudflareサイトで最初に学んだことは、APIについて知る必要があるすべてのことです。そして、私はすでにPythonですべてを実装するために座っていました（Pythonを知った後、いくつかの単純なタスクのために、またはプロトタイプをすばやく作成する必要があるときに、それをより頻繁に使用します）。
一般に、ラッパーpython-cloudflareがベースとして使用されていました。

DNSを更新するための1つの例を取り上げ、構成ファイルの使用と、ゾーン内のいくつかのAレコードを更新する機能を追加しました。当然、無制限の数のゾーンです。

ロジックは次のとおりです。

  1. スクリプトは構成ファイルからゾーンのリストを受け取り、ループでそれらを通過させます
  2. 各ゾーンで、スクリプトはタイプAまたはAAAAの各DNSレコードをループし、レコードでパブリックIPをチェックします
  3. IPが異なる場合は変更し、そうでない場合はループの繰り返しをスキップして次へ進みます
  4. 設定で指定された時間に眠りに落ちる

インストールとセットアップ


おそらく.debパッケージを作成することは可能ですが、私はそれが得意ではなく、それほど複雑ではありません。
プロセスについては、リポジトリページの README.mdで詳しく説明しました。

しかし念のために、私はロシア語で一般的な用語で説明します：

  1. python3とpython3-pipがインストールされていることを確認します。インストールされていない場合は、インストールします（Windowsでは、python3-pipはPythonとともにインストールされます）。
  2. リポジトリを複製またはダウンロードする
  3. 必要な依存関係をインストールします。

    python3 -m pip install -r requirements.txt


  4. Linux のインストールスクリプトを実行します。

    chmod +x install.sh
sudo ./install.sh

    Windowsの場合：windows_install.bat

  5. Linux の構成ファイルを編集します。

    sudoedit /etc/zen-cf-ddns.conf

    Windowsの場合：

    スクリプトがインストールされたフォルダーのzen-cf-ddns.confファイルを開きます。

    これは通常のJSONファイルです。設定によって複雑になることはありません。具体的には、2つの異なるゾーンの例として説明されています。

インストーラーの背後に隠されているものは何ですか？


Linuxのinstall.sh：

  1. ユーザーは、ホームディレクトリとログイン機能を作成せずに、デーモンを起動するために作成されます。

    sudo useradd -r -s /bin/false zen-cf-ddns

  2. ログファイルは/ var / log /に作成されます
  3. 新しく作成したユーザーのログファイルの所有者を作成します
  4. ファイルはその場所にコピーされます（/ etc内の構成、/ usr / bin内の実行可能ファイル、/ lib / systemd / system内のサービスファイル）
  5. サービスがアクティブ化されました

Windowsのwindows_install.bat：

  1. 実行可能ファイルと構成ファイルをユーザー指定のフォルダーにコピーする
  2. システムの起動時にスクリプトを実行するタスクをスケジューラに作成します
    schtasks /create /tn "CloudFlare Update IP" /tr "%newLocation%" /sc onstart

設定を変更した後、スクリプトを再起動する必要があります。Linuxでは、すべてがシンプルで使い慣れています。

sudo service zen-cf-ddns start
sudo service zen-cf-ddns stop
sudo service zen-cf-ddns restart
sudo service zen-cf-ddns status

Windowsの場合、pythonwプロセスを終了してスクリプトを再実行する必要があります（CでWindowsでサービスを作成するのは非常に面倒です）。

taskkill /im pythonw.exe

このインストールと構成が完了したら、あなたの健康状態を使用してください。
