この記事では、優れたFreeacsプロジェクトのテストサーバーを完全に機能する状態にインストールするプロセスを段階的に説明し、mikrotikを操作するための実用的な方法を示します。パラメーターによる構成、スクリプトの実行、更新、追加モジュールのインストールなどです。

この記事の目的は、ひどい熊手と松葉杖を使ったネットワークデバイスの管理を、自己作成のスクリプト、Dude、Ansibleなどの形式で放棄することを同僚に奨励することです。そして、この点で、広場で花火と大喜びを引き起こします。

1. freeacsをインストールする


Centos7にインストールします。デバイスは多くのデータを送信し、ACSはデータベースを積極的に使用しているため、リソースに貪欲になることはありません。快適な作業のために、2つのCPUコア、4 GBのRAM、および16 GBのssd raid10高速ストレージを割り当てます。私はProxmox VE lxcコンテナーにfreeacsをインストールします。あなたはあなたにとって便利なツールで作業できます。
ACSを使用するマシンで正しい時刻を設定することを忘れないでください。

システムはテスト用であるため、スマートではなく、提供された親切なインストールスクリプトをそのまま使用します。

wget https://raw.githubusercontent.com/freeacs/freeacs/master/scripts/install_centos.sh
chmod +x install_centos.sh
./ install_centos.sh

スクリプトが完了するとすぐに、admin / freeacs資格情報を使用して、IPマシンを介してWebインターフェースにすぐにアクセスできます。これは、このようなすてきなミニマルなインターフェースです。




2. freeacの初期設定


ACSの基本的な制御ユニットは、ユニットまたはCPE（顧客宅内機器）です。そして最も重要なのは、ユニットを制御するために必要なのは、そのユニットタイプ、つまり カスタマイズ可能なユニットパラメータとそのソフトウェアのセットを定義する機器モデル。しかし、新しいユニットタイプを取得する方法はわかりませんが、ディスカバリモードをオンにして、ユニットについてそれを確認することをお勧めします。

本番環境では、このような体制を使用することは絶対に不可能ですが、エンジンを始動してシステムの機能を確認する必要があります。すべての基本設定は/ opt / freeacs- *に保存されます。したがって、開く

 vi /opt/freeacs-tr069/config/application-config.conf

我々は気づく 

discovery.mode = false

に変更 

discovery.mode = true

さらに、nginxとmysqlが動作する最大ファイルサイズを増やしたいと考えています。mysqlの場合、/ etc / my.cnfに行を追加します

max_allowed_packet=32M

、およびnginxの場合は、/ etc / nginx / nginx.confに追加します 

client_max_body_size 32m;

httpセクションに。それ以外の場合は、1M以下のファームウェアで作業できます。

再起動し、デバイスを使用する準備が整いました。

そして、デバイス（CPE）の役割では、hAP kid ACAP liteを使用します。

テスト接続の前に、CPEを手動で最小稼働構成に構成して、将来構成するパラメーターが空にならないようにすることをお勧めします。ルーターの場合、ether1でdhcpクライアントを最小限に有効にして、tr-069clientパッケージをインストールし、パスワードを設定できます。

プロビジョニングブロックで、[すべて読み取り]ボタンをクリックします。ブロック内-赤い碑文。CPE（現在）値の列が右側に表示されます。システム設定で、ProvisioningModeがREADALLに変更されました。



そして... System.X_FREEACS-COM.IM.Messageのメッセージ以外は何も起こりませんKick failed at...。

TR-069クライアントを再起動するか、ルーターをリロードし、右側の明るい灰色の長方形にパラメーターが表示されるまでブラウザーページを更新し続けます。
古いものを飲み込みたい場合、このモードはマニュアルで10.2検査モードとして説明されています。それ

IPアドレスの変更、インターフェースの有効化/無効化、コメントのあるファイアウォールルール（そうでなければ完全な混乱）、Wi-Fiなどを簡単に変更できます。

4.2配信パラメーター


次に、「自然な」方法で、パラメーターをtr-069経由でルーターに配信してみましょう。最初の犠牲者はDevice.DeviceInfo.X_MIKROTIK_SystemIdentityです。ユニットパラメータAllにあります。どうやら-それは設定されていません。これは、どのユニットもそれ自体がアイデンティティを持つことができることを意味しています。これで十分です。
作成列でDawを突いて、名前をMr.Whiteに設定し、Update parametersボタンを突いてください。あなたがすでに推測した次に何が起こるか。本社との次の通信セッションで、ルーターはIDを変更する必要があります。



しかし、これでは十分ではありません。 Identityなどのパラメーターは、目的のユニットを検索するときに常に手元にあると便利です。パラメータ名を挿入して、そこにdaws Display（D）とSearchable（S）を配置します。パラメータキーがRWSDに変わります（名前とキーはユニットタイプの最高レベルに設定されています）



これで、値が一般的な検索リストに表示されるだけでなく、Support > Search > Advanced form



開始プロビジョニングでの検索やIDの確認にも使用できるようになります。ミスター・ホワイトこんにちは！これで、tr-069clientの実行中に自分でIDを変更することはできません

4.4ソフトウェアの更新


Mikrotikファームウェアはモジュール式であるため、これは非常に重要なポイントですが、モジュールを追加してもデバイスファームウェアの一般的なバージョンは変更されません。 ACSは正常であり、慣れていません。
これを迅速かつダーティなスタイルで実行し、NTPモジュールを一般的なファームウェアにすぐにプッシュしますが、デバイスでバージョンが更新されるとすぐに、同じ方法で別のモジュールを追加できなくなります。
本番環境では、このようなトリックを適用しない方がよいでしょう。ユニットタイプのオプションモジュールは、スクリプトでのみインストールしてください。

したがって、最初に行う必要があるのは、必要なバージョンとアーキテクチャのソフトウェアパッケージを準備し、アクセス可能なWebサーバーに配置することです。テストでは、Mr.Whiteが到達できる人は誰でもアクセスできます。本番環境では、必要なソフトウェアの自動更新ミラーを組み立てることをお勧めします。
重要！更新には必ずtr-069clientを含むパッケージを含めることを忘れないでください！

結局のところ、パケットへのパスの長さは非常に重要です！私がhttp://192.168.0.237/routeros/stable/mipsbe/routeros-mipsbe-6.45.6.npkそのようなものを使おうとしたとき、mikrotikはリソースへの循環接続に陥り、TR-069にTRANSFERCOMPLETEを繰り返すログを与えました。そして、何が悪いのかを解明するために、ある程度の神経細胞を費やしました。したがって、今のところ、それをルートに配置して確認します。

したがって、httpを介して3つのnpkファイルを使用できるはずです。それが判明した

http://192.168.0.241/routeros-mipsbe-6.45.6.npk
http://192.168.0.241/routeros/stable/mipsbe/ntp-6.45.6-mipsbe.npk
http://192.168.0.241/routeros/stable/mipsbe/tr069-client-6.45.6-mipsbe.npk

次に、microticsに供給するFileType = "1 Firmware Upgrade Image"のxml-fileで発行する必要があります。名前

</upgrade>

5。結論


私が書き始めたとき、最初にip-phoneの接続を説明し、彼の例を使用して、tr-069が簡単かつ簡単に機能するときのクールさを説明したいと思いました。しかし、その後、資料を調べて掘り下げていくと、Mikrotikを接続した人にとって、独立した研究のために怖い電話はないと思いました。

原則として、テストしたFreeacsはすでに本番環境で使用できますが、そのためにはセキュリティ、SSL、Mikrotikをリセット後に自動構成するように構成する必要があり、ユニットタイプの正しい追加をデバッグし、Webサービスとフュージョンシェルの作業を分解する必要があります。試して、発明して、続編を書いてください！
