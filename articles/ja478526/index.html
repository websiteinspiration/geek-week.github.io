Windows ServerとMikrotik上の一連のOpenVPNとLinuxへの移行

良い

すべての企業は、遅かれ早かれ、突然、リモートアクセスを必要とします。
事実上、すべてのITスペシャリストは、企業のネットワークへのリモートアクセスを整理する必要に直面しています。
私も、多くの人がそうであるように、「昨日」という切手でカバーする必要があります。すべての長所と短所を分析し、大量の情報をシャベルで掘り下げて理論的に少し掘り下げたので、インストールを続行することにしました。

セキュリティ上の理由から、私は次の実装でOpenVPNを選択しました。仮想マシンはWindows Server 2012を実行しているサーバーにインストールされ、その上にWindows Server 2012があり、その上に証明書を発行して署名したOpenVPNサーバーがありました。

便宜上、これを「認証サーバー」と呼びます。さらに、サーバー証明書を取得してMikrotikにプッシュし、ルーターでMikrotikがアカウントとプロファイルを使用してOpenVPNを生成しました。また、証明書サーバーを使用してクライアント証明書を発行しました。

もちろん、実装はひどいもので、当時はそうした経験は十分ではありませんでしたが、セキュリティの点では悪い決断ではありませんでした。

このバンドルはしばらくの間機能し、私は新しい導入を与えられました：Mikrotikとの接続を維持しながら認証サーバーをLinuxに転送します-クライアントは苦しむべきではありません。

そのときのLinuxの知識は、RDPを介してWindowsサーバーに接続するためのターミナルとして使用されていたグラフィカルインターフェイスを備えたUbuntu 16.04LTSで終わりました。つまり、sudo apt-get -f install -yであり、センチメートル以上ではありません。

LinuxファミリのどのOSが私の組織にとってより安定していて有望であるかという問題を調査した結果、私はCentOS 7 Minimalに落ち着きました。

まず、理論を少し掘り下げて、一般的にどのように機能し、どのように機能するかを理解することにしました。チャンネルwww.youtube.com/channel/UCKdRgZWgy42YxoFcTJ30LTAでビデオチュートリアルを見ました（一般に広告ではなく、最初に私を手に入れました）。心地よい声の女の子が、選択したOSでの作業の基本を紹介してくれました。

まず、自分のコンピューターでHyper-Vを起動し、CentOS 7 Minimalをインストールしました。インストール中に、管理者ユーザーを作成し、rootのsshを完全に閉じました。端末の白黒の世界に飛び込んだ、美しいマルチカラースクリーンに別れを告げます。

ソフトウェアのインストールプロセスを説明しても意味がないと思います。プロセス中に発生し、小さなスクリプトを書く必要があった問題（猫の下にあります。各ユーティリティの説明はインターネット上にありますが、私がすべてのときにそれをしました、このスクリプトはまだありませんでした、すべてが初めて、タッチしてランダムに行われました）。

スクリプトでは、サーバーに最低限必要なユーティリティのインストールの自動化、Selinuxの無効化、Epelリポジトリの接続、OpenVPNのインストールなどを試みました。以下はスクリプト自体です。簡単ですが、使用することもできます。分解はしませんが、必要に応じて返信してください。

スクリプトを使用した後、すでに構成されているOpenVPNサーバーが表示され、緑色の目でウィンクします。

証明書を失効させようとしたときに問題が発生しました。単語からはまったく機能しませんでした。index.txtファイルをUNIX形式に変換する必要がありましたが、すぐには実行しませんでした。 dos2unixユーティリティを使用しました。

現在、証明書は取り消されましたが、Mikrotikは証明書が取り消されたことを知らず、何らかの形で通知する必要があったため、問題なく機能し続けました。

手順を読み、Alexander ERIに相談した後（ありがとうございました！）、認証サーバーで単純なApache httpサーバーを取得し、失効した証明書のファイルを公開しました。 1つのIPから公開されたファイルを除いて、完全にアクセスを閉じました。

Mikrotikターミナルの/システム/証明書/ CRLタブで、公開されたcrl.pemへのパスを示しました。ここで、MikrotikはhttpとCRLタブの絶対アドレスのみを受け入れることを明確にする必要があります。次のようになります。127.0.0.1/ crl / 1.crl
少なくともRouterOSのバージョン6.4.2.xではすべてが機能しましたが、クライアント構成を手動で作成する必要があり、これは私にとって不運であり、多くの不便を引き起こしました。1週間で約50クライアントの構成を作成する必要があったとき、私はこのプロセスを高速化することを決定し、そのためにインターネット上で見つかった他の誰かのスクリプトを使用しました。

スクリプトは次のように機能します。起動後、「クライアント名」を指定し、「パスワードを設定するかどうか」の質問に答え、その後、証明書と設定が統合された既成の構成ファイル「client.ovpn」を取得します。それを使用するには、/ etc / openvpnが必要です。パスを自分のものに置き換える必要があるコメンティラインに署名します。また、クライアントが設定されたファイルを作成して、スクリプトが構成プロセス中にそれらを置き換えるようにすることも必要です。

#!/bin/bash
function newClient () {
	echo ""
	echo "Tell me a name for the client."
	echo "Use one word only, no special characters."

	until [[ "$CLIENT" =~ ^[a-zA-Z0-9_]+$ ]]; do
		read -rp "Client name: " -e CLIENT
	done

	echo ""
	echo "Do you want to protect the configuration file with a password?"
	echo "(e.g. encrypt the private key with a password)"
	echo "   1) Add a passwordless client"
	echo "   2) Use a password for the client"

	until [[ "$PASS" =~ ^[1-2]$ ]]; do
		read -rp "Select an option [1-2]: " -e -i 1 PASS
	done

	#cd /etc/openvpn/easy-rsa/ || return
	case $PASS in
		1)
		sudo /home/admin/EasyRSA-3.0.3/easyrsa build-client-full "$CLIENT" nopass
		;;
		2)
		echo "You will be asked for the client password below"
			./easyrsa build-client-full "$CLIENT"
		;;
	esac
# Generates the custom client.ovpn
	cp /etc/openvpn/client-template.txt "$home/home/admin/IT/Temp/$CLIENT.ovpn" 
#       .
#,     
	{
		echo "<ca>"
		cat "/etc/openvpn/pki/ca.crt" #   
		echo "</ca>"

		echo "<cert>"
		awk '/BEGIN/,/END/' "/etc/openvpn/pki/issued/$CLIENT.crt" #   # 
		echo "</cert>"

		echo "<key>"
		cat "/etc/openvpn/pki/private/$CLIENT.key" #    
		echo "</key>"

} >> "$home/home/admin/IT/Temp/$CLIENT.ovpn" #,     #

	echo ""
	echo "Client $CLIENT added, the configuration file is available at $home/admin/IT/OVPN/Temp/$CLIENT.ovpn."
	echo "Download the .ovpn file and import it in your OpenVPN client."
exit 0;
}
