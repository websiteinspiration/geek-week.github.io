エミュレーションコモドール65


いつどこで、いつ、どのコンピュータで未知のコンピュータについて学ぶのかは、決してわかりません。

今回は簡単になりました。

私はMOS 6502ベースのコンピューターのリストを探し始め、1990-1991年にCommodore Business Machinesで作成されたプロトタイプコンピューターである65CE02ベースのC64Dx（またはC65）に言及しました。これはCommodore 64の改良版であり、Amigaに近い多数の高度な機能を提供しながら、古いコンピューターとの下位互換性があると想定されていました。発売モデル数はさほど多くはありませんが、C65購入時に発表される価格表は印象的です。

MEGA65-C64 / C65と互換性のあるコンピューター-についても読みました。それだけです。知り合いになりました。

最初の目標は、「Hello、world！」を表示するプログラムを作成することでした。フォーラムのトピックを読んで解決策が提案されました：

*=$0FFE
.byte $00, $10
  ldx #$0
cycle  lda hworld,x
       cmp #0
       beq exit
       sta $0800,x
; sta $D800,x
       inx
       jmp cycle
exit   rts
hworld .text 'hello world!',0



問題が発生しましたか？エンコーディングだけが異なります。



したがって、テキストは次のように変更されました。

hworld .byte 8,5,$0C,$0C,$0F,$20,$17,$0F,$12,$0C,4,$21,0



次に、Commodore 64 Programmers Reference Guideのドキュメントと、多くの便利な機能を見つけた別のアーカイブを見つけました。c64-programmers_reference_guide-03-programming_graphics.pdf

ドックを読むと、単一の効果のアイデアが示唆されました。例えば

	lda #$62
	sta $D018

ビデオメモリを$ 1800に配置し、文字データを$ 800に配置します。読み込みc65manual.txtを、私は色の情報が見つかりました：

$ D100、$ D200、D300の$ - R、G、ベッドとカラー値を（0-15）
$ D800- $ DBFFF -カラーテキストの属性（またはむしろ、すべての色がアドレスに格納されています$ 1F800-$ 1FFFF）。

コードのいくつかのサンプルで、既製の文字セットとシンプルな色を入手しました。 DMAgicチップを使用してメモリの2つのセクションを転送したかっただけです。ドック

を読み始めました。 コントローラは4つのモードで動作します： COPY- SWAP メモリのセクションのコピー -FILL セクションのデータ交換-メモリセクションへのデータの書き込み






MIX-ソースとレシーバーでブール型のMintermミックスを実行します（私を殺す、私は書く方法がわかりません）

。便宜上、DMATテーブルが使用され、レジスタにデータを書き込むと操作が実行されます。

		lda #0	;the bank (0-15) where the list is
