Pythonの* argsおよび** kwargsとは何ですか？

機能は命です。真実？Pythonの学習を始めたばかりの場合、それが最初のプログラミング言語であるか、別の言語からPythonに移行したかは関係ありません。関数宣言のパラメーターの数は、関数が呼び出されたときに渡される引数の数に対応していることはすでにわかっています。 これらは基本です。これは、人々が自分の周りの世界を理解するのに役立つものです。しかし、「パラメーターの数は引数の数と等しい」というステートメントは、初心者の頭に時限爆弾を配置します。これは、神秘的な構造を見た後、または関数宣言で発砲します。 バッジで馬鹿にされないようにしてください。ここには古風なものは何もありません。一般的に、これらのデザインがあなたに馴染みのないものである場合は、対処することをお勧めします。

位置および名前付き引数


*argsおよび を処理するには**kwargs、位置引数と名前付き引数の概念を習得する必要があります。

最初に、それらがどのように異なるかについて話しましょう。最も単純な関数では、引数とパラメーターの位置をマッピングするだけです。引数1はパラメーター1に対応し、引数2はパラメーター2に対応します。

def printThese(a,b,c):
   print(a, "is stored in a")
   print(b, "is stored in b")
   print(c, "is stored in c")
printThese(1,2,3)
"""
1 is stored in a
2 is stored in b
3 is stored in c
"""

関数を呼び出すには、3つの引数すべてが必要です。少なくとも1つをスキップすると、エラーメッセージが表示されます。

def printThese(a,b,c):
   print(a, "is stored in a")
   print(b, "is stored in b")
   print(c, "is stored in c")
printThese(1,2)
"""
TypeError: printThese() missing 1 required positional argument: 'c'
"""

関数を宣言するときにデフォルト値をパラメーターに割り当てると、関数を呼び出すときに対応する引数を指定する必要がなくなります。パラメータはオプションになります。

def printThese(a,b,c=None):
   print(a, "is stored in a")
   print(b, "is stored in b")
   print(c, "is stored in c")
printThese(1,2)
"""
1 is stored in a
2 is stored in b
None is stored in c
"""

オプションのパラメーターは、それらの名前を使用して関数を呼び出すときにも指定できます。

次の例では、3つのパラメーターをデフォルト値に設定しNone、関数を呼び出すときに使用される引数の順序に注意せずに、名前を使用してそれらを割り当てる方法を確認します。

def printThese(a=None,b=None,c=None):
   print(a, "is stored in a")
   print(b, "is stored in b")
   print(c, "is stored in c")
printThese(c=3, a=1)
"""
1 is stored in a
None is stored in b
3 is stored in c
"""

アスタリスク演算子


演算子は*、人間では乗算の演算と最もよく関連付けられますが、Pythonでは別の意味があります。

この演算子を使用すると、特定の要素が格納されているオブジェクトを「アンパック」できます。次に例を示します。

a = [1,2,3]
b = [*a,4,5,6]
print(b) # [1,2,3,4,5,6]

ここでは、リストの内容が取得されa、解凍されてリストに配置されますb。

* argsおよび** kwargsの使用方法


つまり、Pythonの「アスタリスク」演算子は、構成要素をオブジェクトから「引き出す」ことができることを知っています。また、2種類のパラメーター関数があることも知っています。すでにご自分で考えられている可能性もありますが、念のため申し上げます。つまり、*argsこれは「arguments」（引数）の省略形であり、** kwargsは「keyword arguments」（名前付き引数）の省略形です。

これらの各構成要素は、対応する型の引数をアンパックするために使用され、可変長の引数のリストを使用して関数を呼び出すことができます。たとえば、学生がテストで入力した結果を表示できる関数を作成します。

def printScores(student, *scores):
   print(f"Student Name: {student}")
   for score in scores:
      print(score)
printScores("Jonathan",100, 95, 88, 92, 99)
"""
Student Name: Jonathan
100
95
88
92
99
"""

関数を宣言するときに構成を使用しませんでした*args。代わりに、私は-を持ってい*scoresます。ここにエラーがありますか？ここに間違いはありません。実際、「args」は引数を表すために使用される文字のセットにすぎません。ここで最も重要なのは演算子*です。そして、それ以降は特別な役割を果たしません。使用を通じて*、関数が呼び出されたときに関数に渡されたものに基づいて、位置引数のリストを作成しました。

で理解した後は*args、**kwargs問題を理解することはできなくなります。名前も、重要ではありません。主なものは2文字**です。それらのおかげで、関数が呼び出されたときに関数に渡される名前付き引数を含む辞書が作成されます。

def printPetNames(owner, **pets):
   print(f"Owner Name: {owner}")
   for pet,name in pets.items():
      print(f"{pet}: {name}")
printPetNames("Jonathan", dog="Brock", fish=["Larry", "Curly", "Moe"], turtle="Shelldon")
"""
Owner Name: Jonathan
dog: Brock
fish: ['Larry', 'Curly', 'Moe']
turtle: Shelldon
"""

概要


ここでは、関数の操作時に発生する一般的な問題を回避し、知識を広げるのに役立ついくつかのヒントを示します。

  • 従来の設計*argsを使用**kwargs して、位置引数と名前付き引数をキャプチャします。
  • 構造は**kwargまで配置できません*args。これを行うと、エラーメッセージが表示されます。
  • **kwargs値が**kwarg引数として渡される予定であるが、この値のキー名が名前付きパラメーターの名前と一致する場合は、名前付きパラメーターとの間の競合に注意してください。
  • 演算子*は、関数宣言だけでなく、呼び出されたときにも使用できます。
