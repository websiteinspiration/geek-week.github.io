まず、基本を思い出してみましょう。

スタック


スタックはLIFOの原則を実装します（たとえば、後入れ先出し、「後入れ先出し」）。

スタックには2つの主な操作があります。

  • push-スタックにアイテムを置きます
  • pop-スタックから要素を取得する

スタックは通常、配列に実装されます（リンクリストでも可能です）。スタックとその実装について詳しくは、こちらをご覧ください。

順番


キューは、FIFO要素（先着順-「先着順」）にアクセスできるデータ構造です。

キューのインターフェースには、次の2つの主要なメソッドがあります。

  • push-キューの最後にアイテムを置きます
  • pop-キューの先頭から要素を取得します

通常のキューについて詳しくは、こちらをご覧ください。

通常、キューの実装には2つの基本的なアプローチを検討します。

  1. アレイ上
  2. リンクされたリスト

なぜスタックはラインよりも冷たいです


データ構造が次の一連の操作をサポートする必要があるとします。

  • 最小サポート（分）
  • 最大サポート（最大）
  • GCDサポート（英語の最大公約数-最大公約数）
  • LCMサポート（最小公倍数）

これらの操作はすべて結合的です（セミグループを形成します）が、いずれも逆要素を持ちません（グループを形成しません）。

スタックの場合、次の操作のいずれかを非常に単純にサポートできます。その上にカップルを格納するだけです。

（eleメートルeんt、operat私oんResあなたlt）

ここで、ペアの2番目の要素は、スタックのすべての要素に適用された操作の結果です。

以下は、Pythonでの最小スタックサポート実装の例です。

class Stack:
    def __init__(self):
        self.stack = []

    def __bool__(self):
        return bool(self.stack)

    def push(self, elem):
        if self.stack:
            self.stack.append((elem, min(elem, self.stack[-1][1])))
        else:
            self.stack.append((elem, elem))

    def pop(self):
        return self.stack.pop()[0]

    def get_min(self):
        if not self.stack:
            return math.inf
        return self.stack[-1][1]

次に、キューで同じトリックを実行する方法について考えますか？アレイ上に編成された従来のキューを試す場合、何かがうまくいくとは考えられません。これは、最小演算には逆演算がない（たとえば、加算演算には減算演算がある）ためです。ご想像のとおり、2つのスタックでのキューのそれほど古典的でない実装について説明します。

2つのスタックキュー


満たさなければならない主な条件は、すべての操作が償却済みO（1）に対して実行されなければならないことです。

s1とs2の2つのスタックを取ります。

プッシュ操作は常にs1スタックで行われます。

pop操作は次のように編成されます。s2スタックが空の場合、popとpushを連続して呼び出すことにより、すべての要素をs1からs2に転送します。これで、s2スタックには逆の順序で要素が含まれています（最上位の要素が今度は最初の要素です）。

s2が空でない場合、その要素を愚かに取り出します。s2が再び空になったらすぐに、同じ操作を繰り返します。

Pythonコード：

class Queue:
    def __init__(self):
        self.s1 = Stack()
        self.s2 = Stack()

    def push(self, elem):
        self.s1.push(elem)

    def pop(self):
        if not self.s2:
            while self.s1:
                self.s2.push(self.s1.pop())
        return self.s2.pop()

    def get_min(self):
        return min(self.s1.get_min(), self.s2.get_min())

労働時間


プッシュ操作：アイテムを愚かにs1スタックにプッシュします。稼働時間O（1）。

ポップ操作：要素ごとに、スタックからスタックへの転送は1回のみであるため、償却済み作業時間はO（1）になります。

操作get_min：s1およびs2スタックの最小値は既知であるため、最小値から最小値を取得します。稼働時間O（1）。
