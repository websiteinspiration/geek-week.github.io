Episode 4 - Finding Elements



In this episode we will look at how it works findElement



. First of all, we need to be able to find elements before we can interact with them.



Finding an item is easy, right?



Well, yes, if people developed their applications with an eye on primarily testability, pushing everything else into the background. Unfortunately, we live in a world where most of the time people think about quality, and about testing as something secondary.



Anyway ... let's see how we can look for them.



Search Types



This part is quite simple, Selenium offers the same search methods that you can find when interacting with the DOM via JavaScript. If we can search for items using document.querySelectorAll(aQuery);



then they will be searchable through find_element



or find_elements



. They are available from WebDriver



or WebElement



. If you do element.find_element(...)



, this is equivalent element.querySelectorAll(aQuery)



, which sets the starting point or root to search on this element.



The following is an example of use find_element



: 



from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Firefox()
driver.get("https://www.theautomatedtester.co.uk")
driver.find_element(By.ID, "someID")
driver.find_element(By.CSS_SELECTOR, "#someID")
driver.find_elements(By.CSS_SELECTOR, "#someID")
driver.find_element(By.NAME, "someName")
driver.find_elements(By.NAME, "someName")
driver.find_element(By.CLASS_NAME, "someName")
driver.find_elements(By.CLASS_NAME, "someName")
driver.find_element(By.TAG_NAME, "someName")
driver.find_elements(By.TAG_NAME, "someName")


We can also search for items on a page by XPATH .



We can do the following

Episode 4 - Click



In this episode, we‚Äôll look at what happens when you click. For the sake of simplicity, we will only consider work performed using element.click()



.



You have an element that you want to click on, what now?



So, you have found the element , and you need to click on it. Selenium will take the following steps before it completes the click.



Checking if another element is on the page



When we found the element , the browser returned a representation of this element. All subsequent calls to this element should be checked for the fact that the element is still on the page. If this is not the case, you will get an exception StaleElementReferenceException



.



Checking the visibility of an element



Once we have verified that the element is still on the page, we need to make sure that the element is displayed. This is important since we do not expect the user to click on an item that is not visible. Since the user will never click there, we must make sure that Selenium will not do this either. If it does not appear, you will get an exception ElementNotVisibleException



.



Scrolling to an item



Next, we need to make sure that we can scroll to the item. For the correct launch of all events, we need an element in the viewing area . If the item is not in the viewport after trying to scroll to it, the browser will throw an exception ElementNotInteractableException



.



Check if an item is interactive



Now the browser will check whether the element that will receive clicks in the area we are talking about is the same element that we passed. If it turns out that this is not the right element, the browser will throw an exception ElementClickInterceptedException



.



Turning off some events



Now we are ready to start some events. We have a special case for <



option>



to make sure that this is the parent element that receives the events.



Then we calculate the center of the element and start sending events. You can read what events we send in the WebDriver specification .



What about navigation?



We need to tell the browser to follow the navigation that occurs as a result of the click. If a click causes navigation, we apply the same logic as if we were navigating ourselves .
