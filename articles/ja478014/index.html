DataGrip 2019.3：MongoDB、デバッガー、トレンディな進行状況バー、多くの新しい検査

こんにちは！いつものように、年に3回、私たちが取り組んだことについて話します。この記事は、DataGrip 2019.3の新機能、したがって、データベースをサポートするすべてのJetBrains IDE の新機能に関するものです。



MongoDBサポート


これは、Habréに関する私たちの最初の出版物であり、彼らは尋ねないコメントで、「そして、Mongaのサポートはいつ計画されていますか？」

会いましょう！このバージョンでの動作：

イントロスペクション：コレクションとフィールドはデータベースエクスプローラーに表示されます。各コレクションの最初の10個のドキュメントを分析して、そのフィールドについてのアイデアを持っています。

この数を増やす場合は、データソースのプロパティの[ 詳細設定 ]タブにあるJDBCパラメータfetch_documents_for_metainfoで指定します。データの表示：コレクションを表形式でページごとに参照します。値で並べ替え、フィルターを使用できます。ツリー表示







：特にMongoDBでは、データのツリービューを実行しました。オンにするには、歯車をクリックし、次に[...として表示 ]をクリックします。木 別のそのようなビューはPostgreSQL配列で機能し、JSONおよびXMLフィールドをツリーのように見せることを計画しています。



クエリコンソール：現在、コードの記述はサポートされていませんが、クエリは機能します。クエリを実行するには、キャリッジをその上に置き、Ctrl / Cmd + Enterまたはツールバーの[ 実行 ]ボタンを押します。

JavaScript対応のIDEで作業している場合、リクエストは強調表示されます。後でDataGripにバックライトを追加します。



Oracle用PL / SQLデバッガ


待望のOracle用デバッガの最初のバージョン。現在、彼はプロシージャ、関数、パッケージ、トリガー、匿名ブロックを使用しています。

オブジェクトはデバッグ用にコンパイルする必要があることに注意してください。これを行うには、コンテキストメニューを使用します。再コンパイル。[デバッグあり]



チェックボックスをオンにします。プロシージャがすでにデバッグ用にコンパイルされている場合は、アイコンに緑のバグがあります。 コンソールでデバッガを起動するには、ツールバーの[ デバッグ ]ボタンをクリックします。 デバッガーの詳細は、ドキュメント（英語）にあります。ご意見・ご感想・ご提案はこちらまでお願いします。









ソースコードを操作する


手続きコードの比較


2つの手順を比較するには、それらを選択してCtrl / Cmd + Dを押します。以前は署名のみが比較されていましたが、現在はソースです。



3つのパネルの融合


オブジェクトのソースコードを更新すると、この間にデータベースでコードが変更された場合、DataGripは警告を出します。競合を解決するために、3つのパネルを使用してマージを提供します。左側にはバージョン、右側にはデータベースからの結果、中央にはマージの結果が表示されます。



他の誰かのコードを書き換えないようにするには、データソースのプロパティで古いDDL設定を編集する場合に警告が有効になっていることを確認してください。

コードの作成に役立つ


オブジェクトのソースコードでは、コードの記述に役立ちます。たとえば、アスタリスクを開くことができます。



PostgreSQLの美しいソース


pretty-printed 設定が無効になっている場合、PostgreSQLはソースに多数の冗長なブラケット



を提供します。デフォルトで有効にすることにしました。この例では、括弧が残っていません:)



ただし、PostgreSQLがドキュメントに書き込む内容に注意してください。

きれいに印刷されたコードは読みやすくなりますが、デフォルトの形式はPostgreSQLの将来のバージョンでよりよく理解される可能性があります。したがって、ダンプにきれいに出力されたコードを使用しないことをお勧めします。

きれいに印刷されたフォーマットは読みやすくなりますが、デフォルトのフォーマットはPostgreSQLの将来のバージョンで同じように解釈される可能性が高くなります。かなり印刷された出力をダンプの目的で使用しないでください。
