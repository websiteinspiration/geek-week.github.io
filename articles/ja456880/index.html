Math.RoundがTry .NETのブラウザーから印刷ウィンドウを開くのはなぜですか

今日、インタラクティブな例をドキュメントのC＃に埋め込むために設計されたツールであるTry .NETで発見された、おかしな非論理的なバグに世間の注目が集まりました。未解決の問題は、リンクの Github で確認できます。次の

コードを実行すると（Math.Roundメソッドを呼び出すとき）、期待した結果ではなく、ブラウザーから印刷ウィンドウが突然開きます。

using System;
public class Example
{
    public static void Main()
    {
        var x = Math.Round(11.1, MidpointRounding.AwayFromZero);
    }
}




ユーザーはエラートレースとブレークポイントを使用して、この動作の原因とされる理由を発見しました。これはmono.jsライブラリに隠されていました。



答えは簡単です。どうやら、誰かがJSで独自のprint（）関数を使用したかった（またはそれをconsole.logと混同した）が、スコープ内に何もなかったので、プログラムはwindow.print（）を呼び出し、これは現在の印刷ウィンドウを実際に開くはずであるdocument- windowはブラウザのメインストリームのグローバルオブジェクトであるため。

このフォールバックはすぐには気づかれませんでしたが、ユーザーの1人はこのエラーが昨年11月に修正されたと主張しています。

この種のエラーが発生する状況を回避するために、create-react-appプロジェクトは、「紛らわしい」ブラウザーグローバル変数のリストを保持しています。

handleClick() { //   `event`
  this.setState({
  	text: event.target.value //    `event`- !
  });
}
