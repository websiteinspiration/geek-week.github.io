Apache Sparkを使用したXGBoost分散学習と同時予測

こんにちは、ハブル！すでに7月の終わりに、Otusは新しいコース「ビッグデータの産業用ML」を開始します。伝統的に、新しいコースの開始を見越して、有用な資料の翻訳を用意しました。







一般情報



（機械学習モデルのアンサンブルからの）ブースティングでは、アルゴリズムは逐次プロセスを実装し（並列化されるバギングとは対照的に）、弱い学習アルゴリズムを生成し、それらを（すべてのアンサンブルメソッドのように）強力なアルゴリズムと組み合わせます。プロセスの各反復でのブースティングでは、モデルは、弱学習アルゴリズムが個別にトレーニングされるバギングとは異なり、前の反復のエラーを適応的に修正しようとします。







ブースティングアルゴリズムの1つである勾配ブースティングは、勾配降下を使用して、これらの順次モデルで損失関数を直接最小化します（トレーニングインスタンスの重みを変更することでトレーニングが発生するAdaBoostアルゴリズムとは異なります）。



トレーニング中の勾配ブースティング中に作成された弱い学習アルゴリズムは、通常、決定木として実装されます。グラディエントブースティングで最も効果がないのは、ツリーを作成する順次プロセスです。この場合、一度に作成されるツリーは1つだけです。



この制限を回避するために、Tianji ChenとCarlos GestrinはXGBoostと呼ばれる勾配ブースティングアルゴリズムの改善を提案しました。これは、Extreme Gradient BoostingまたはExtreme Gradient Boostingの略です。これはステロイドの一種の勾配ブースティングであり、主に分類に使用されますが、回帰とランキングにも使用されます。



標準のグラディエントブースティングと比較して、この新しい方法では、ハイパーパラメーター、GPUサポート、アルゴリズムの相互検証と正則化により、生産性が大幅に向上します。一般に、モデルはより効率的で、学習が速く、再トレーニングする傾向がありません。



最近、XGBoostは人気を博し、Kaggleで多くの機械学習コンテストに勝ちました。それは素晴らしい計算能力と精度を持っていると信じられています。







XGBoostとApache Spark



標準ワークフローでは、MLはSparkなどのシステムを使用してデータを前処理してクリーンアップする機械学習パイプラインを作成し、結果は機械学習フェーズに渡されます。すでにSparkを使用している場合は、Spark MLlibを使用することがよくあります。



この記事のコンテキストでは、XGBoostがツリー構築プロセスの並列化を備えていることが重要です。これにより、ノード間の分散トレーニングと予測が可能になります。つまり、Apache Spark MLlibのユーザーとしてそれを使用してXGBoostを学習し、プロダクションで作業する可能性を広げることができれば、実際には、XGBoostの高性能と、エンジニアリング機能とMLパイプラインの構築のためのSparkの強力なメカニズムを楽しむことができます。



XGBoostとApache Sparkを統合し、XGBoostをApache Spark MLlibフレームワークに追加するプロジェクトであるXGBoost4J-Sparkをご覧ください。



XGBoost4J-Sparkは、XGBoostモデルをトレーニングする前にデータを前処理し、トレーニングし、本番環境での並列予測に使用できるMLlibパイプラインを構築することを可能にします。このライブラリを使用すると、各XGBoostワーカーはSparkタスクになり、Sparkメモリからのトレーニングデータセットは、Sparkアーティストに見えないように存在するXGBoostワーカーに送信されます。







XGBoost4J-Sparkで機械学習アプリケーションの作成を開始するには、まず適切な依存関係を追加する必要があります。 



<dependency>
    <groupId>ml.dmlc</groupId>
    <artifactId>xgboost4j-spark</artifactId>
    <version>0.90</version>
</dependency>


データの準備（アイリスの例）



前述のように、XGBoost4J-Sparkを使用すると、XGBoostインターフェースでデータを「適合」できます。



DataFrameのIris Flowersデータセットを検討したらすぐに、次のものが必要になります。



  • 列をStringからDoubleに変換します。
  • データがSparkの機械学習フレームワークインターフェースと一致するように、特徴列をベクトルに結合します。


import org.apache.spark.ml.feature.StringIndexer
import org.apache.spark.ml.feature.VectorAssembler
val stringIndexer = new StringIndexer().
  setInputCol("class").
  setOutputCol("classIndex").
  fit(irisDF)
val labelTransformed = stringIndexer.transform(irisDF).drop("class")
val vectorAssembler = new VectorAssembler().
  setInputCols(Array("sepal length", "sepal width", "petal length", "petal width")).
  setOutputCol("features")
val xgbInput = vectorAssembler.transform(labelTransformed).select("features", "classIndex")


上記のDataFrameでは、結果は2つの列「features」：アイリスの符号を表すベクトルと「classIndex」：Double型のラベルになります。このようなDataFrameは、XGBoost4J-Spark学習エンジンに安全に供給することができます。



分散トレーニング



import ml.dmlc.xgboost4j.scala.spark.XGBoostClassifier

val xgbClassifier = new XGBoostClassifier().
      setFeaturesCol("features").
      setLabelCol("classIndex").
      setObjective("multi:softmax")
      setMaxDepth(2).
      setNumClass(3).
      setNumRound(100).
      setNumWorkers(10).


XGBoostオプションの完全なリストはここにあります。XGBoost4J-Sparkでは、上記の例のようにcamelaseも使用できることに注意してください。

早期停止によってサポートされているnum_early_stopping_roundsとmaximize_evaluation_metricsのパラメータ。



これで、入力DataFrameでXGBoost分類器をトレーニングしてトランスフォーマーを作成できます。学習プロセスの結果として、予測を行うために使用できるモデルが得られます。 



val xgbClassificationModel = xgbClassifier.fit(xgbInput)


並行予測



XGBoost4j-Sparkは、バッチ予測とスポット予測をサポートしています。



バッチ予測の場合、モデルは、特徴ベクトルを含む列を持つDataFrameを取得し、各特徴ベクトルの予測を行い、結果を含む新しいDataFrameを表示します。このプロセスでは、XGBoost4J-Sparkは、並列バッチ予測の入力DataFrameの各部分に対してXGBoostワーカーを使用してSparkタスクを起動します。 



val predictionsDf = xgbClassificationModel.transform(inputDF)
predictionsDf.show()

ポイント予測の場合、モデルは1つのベクトルを取ります。 



val features = xgbInput.head().getAs[Vector]("features")
val result = xgbClassificationModel.predict(features)


XGBoostを使用したスポット予測はオーバーヘッドが高いため推奨されません。単一の予測に匹敵するためです。
