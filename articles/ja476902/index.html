バリモア、ボクシムプラントの話題は何ですか？実装されたWebソケット、サー


WebSocketは、クライアントとサードパーティのサービス間のリアルタイムでの全二重（双方向）通信の進歩的な標準です。 Webソケットは、追加のHTTPリクエストなしで継続的なデータ交換を整理するために使用されます。

そして、これらすべてがVoximplantで可能になったことをお知らせします。これは、新しいVoxEngineモジュール（-surprise - WebSocket）のおかげです。これからは、Webソケットを最大限に活用して、テキストと音声を送信できるようになります。簡単に言うと、アプリケーションをアップグレードする別のツールがあります。

この記事では、発信WebSocket接続を作成し、それを介してオーディオストリームを転送し、Google Cloud Speech-to-Text APIを使用してテキストに変換する方法を学びます。
Voximplantには、ASRモジュールの制御下でリアルタイムで音声をテキストに変換する機能が組み込まれていることに注意してください。このモジュールは、Google、Yandex、Tinkoffの関数を使用します。詳細はこちらをご覧ください。

現在の記事では、音声をテキストに変換するためにサードパーティのサービスを使用したり、Voximplantアカウントからではなく彼のアカウントからお金を使いたい場合について説明しています。

私たちのモジュールは2つの作業形式を提供します：
  • 発信接続を作成します。
  • 着信接続を受信し、そのためのWebソケットを作成します。

発信


アウトバウンド接続を作成するときに最初に行う必要があるのは、VoxEngineスクリプトを実行することです。次に、VoxEngine.createWebSocketメソッドを呼び出します。これにより、WebSocketオブジェクトが作成されます。このメソッドは2つのパラメーターを取ります。「wss：// + domain + path」形式のURLとプロトコル（オプション）。コードでは次のようになります。

VoxEngine.addEventListener(AppEvents.CallAlerting, function(e) {
    const webSocket = VoxEngine.createWebSocket( /*url*/ "wss://your_link/");
    //      
});

すべてがうまくいった場合、call.sendMediaToメソッドがオーディオストリームをソケットに送信し、WebSocket.sendがデコードされたオーディオストリームをJSON形式で送信します。その結果、リクエストを処理するサービスからメッセージを受け取ります。接続を閉じるに

はWebSocket.closeメソッドが必要です。クライアント側とサーバー側の両方で接続を閉じることができることに注意してください。

受信トレイ


着信接続を可能にするために、あなたはを通してそれについて通知しなければならないVoxEngine.allowWebSocketConnections、ともに加入AppEvents.WebSocketのイベント。次に、着信接続を受け入れて、WebSocket：event.WebSocketオブジェクトを取得できます。以下のコードを参照してください。

VoxEngine.allowWebSocketConnections();

VoxEngine.addEventListener(AppEvents.WebSocket, function(e) {
    //      
});

インバウンドWebソケットを作成するには、accessSecureURLコントロールが必要です。AppEvents.Startedイベントまたはセッションを開始したHTTPリクエストへの応答から取得できます。URLで「https」を「wss」に変更する必要があることに注意してください。

残りの手順は、発信接続スキームで示した手順と同じです。

必要になるだろう


アプリケーションにWebSocketテクノロジーと音声認識を実装するには、次のものが必要です。
  • Voximplantアカウント。お持ちでない場合は、こちらから登録してください。
  • Voximplantアプリケーション、スクリプト、ルール、および1人のユーザー。このすべてがこのチュートリアルで作成されます。
  • Speech-to-Text APIに接続されたクラウドクライアントライブラリを使用したシンプルなバックエンド（node.jsでサーバーを起動します）。
  • 電話をかけるためのWebクライアント（phone.voximplant.comの Web電話を使用します）。

1. VOXIMPLANT設定


開始するには、manage.voximplant.com / authでアカウントにログインします。左側のメニューで[アプリケーション]をクリックし、新しいアプリケーションを作成して、websocketという名前を付けます。アプリケーションに移動し、[スクリプト]タブに切り替えてスクリプトを作成し、次のコードを貼り付けます。

require(Modules.WebSocket);

VoxEngine.addEventListener(AppEvents.CallAlerting, function(e) {
    call = e.call;
    call.answer();

    const webSocket = VoxEngine.createWebSocket( /*url*/ "wss://your_ngrok_link/");

    webSocket.addEventListener(WebSocketEvents.ERROR, function(e) {
        Logger.write("LOG OUTGOING: WebSocketEvents.ERROR");
        call.sendMessage("LOG OUTGOING: WebSocketEvents.ERROR");
    });
    webSocket.addEventListener(WebSocketEvents.CLOSE, function(e) {
        Logger.write("LOG OUTGOING: WebSocketEvents.CLOSE: " + e.reason);
        call.sendMessage("LOG OUTGOING: WebSocketEvents.CLOSE: " + e.reason);
    });
    webSocket.addEventListener(WebSocketEvents.OPEN, function(e) {
        Logger.write("LOG OUTGOING: WebSocketEvents.OPEN");
        Logger.write(JSON.stringify(e))
        call.sendMessage("LOG OUTGOING: WebSocketEvents.OPEN");
    });
    webSocket.addEventListener(WebSocketEvents.MESSAGE, function(e) {
        Logger.write("LOG OUTGOING: WebSocketEvents.MESSAGE: " + e.text);
        call.sendMessage("LOG OUTGOING: WebSocketEvents.MESSAGE: " + e.text);
        if (e.text == "Hi there, I am a WebSocket server") {
            call.sendMediaTo(webSocket, {
                encoding: WebSocketAudioEncoding.ULAW,
                "tag": "MyAudioStream",
                "customParameters": {
                    "param1": "12345"
                }
            });
        }
    });

    call.addEventListener(CallEvents.Disconnected, function(e) {
        Logger.write("LOG OUTGOING: terminating in 1 sec");
        webSocket.close();
        setTimeout(VoxEngine.terminate, 1000);
    });
});

このVoxEngineスクリプトは、オーディオストリームをWebSocketに送信し、そのイベント（ERROR、CLOSE、OPEN、MESSAGE）も追跡します。スクリプトの詳細については、後で詳しく説明します。

次に、「ルーティング」タブに移動し、「新しいルール」をクリックして、socketRuleと呼びます。これで、スクリプトを選択するだけで、マスクはデフォルト（。*）のままになります。


この段階で最後に行う必要があるのは、ユーザーを作成することです。「ユーザー」タブに切り替え、「ユーザーの作成」をクリックし、名前（例えば、socketUser）とパスワードを指定して、「作成」をクリックします。最後のステップでWebクライアントで認証を行うには、このユーザー名とパスワードのペアが必要です。


構成は完了しましたが、サーバーの作成を開始する前に、シナリオでWebSocketモジュールがどのように機能するかを見てみましょう。

2.スクリプトの詳細


WebSocketモジュールを使用すると、開発者は安定した接続を開き、それを介してデータを送信できます。このモジュールを使用するには、スクリプトの最初に接続する必要があります。

require(Modules.WebSocket);

createWebSocketメソッドを使用して、URLとプロトコルを決定します（オプション）。WebSocketのURLを取得する方法については、次のセクションで学習できます。

const webSocket = VoxEngine.createWebSocket( /*url*/ "wss://your_ngrok_link/");

WebSocketオブジェクトを作成した後、ハンドラー内で呼び出しを管理し続けます。つまり、call.sendMediaToメソッドを使用して、メディアをWebSocketオブジェクトに送信します。

ここでは、優先するエンコード形式、タグ、およびいくつかのカスタムパラメータを設定できます。エンコーディングを設定しない場合、デフォルトでPCM8が使用されます。

このメソッドは、正常な接続メッセージが受信されたときに呼び出されます。このシナリオでは、呼び出しコードは次のようになります。

call.sendMediaTo(webSocket, {
    encoding: WebSocketAudioEncoding.ULAW,
    "tag": "MyAudioStream",
    "customParameters": {
        "param1": "12345"
    }
});

コードに表示される他のすべてのWebSocketイベントはデバッグ用であり、Voximplantセッションのログに情報を送信します。必要に応じて、それらを削除できます。

最後に、正しいデータ転送完了ハンドラを追加します。私たちの場合、Voximplantセッションは、確立された通話の終了（切断）の1秒後に作業を終了します。

call.addEventListener(CallEvents.Disconnected, function(e) {
    Logger.write("LOG OUTGOING: terminating in 1 sec");
    webSocket.close();
    setTimeout(VoxEngine.terminate, 1000);
});

スクリプトのロジックが明確になったので、次の例の非常に重要な部分に移りましょう。

3.バックエンド


まず、コンピューターにNode.js.がインストールされていることを確認します。Node.jsのメインサイトからダウンロードできます。次に、ターミナルウィンドウで次のコマンドを1つずつ実行して、作業環境をセットアップします。

npm install express
npm install ws
npm install @google-cloud/speech

そして、これが完了したら、空のJSファイルを作成し、そこに次のコードを配置します（コードのニュアンスは以下で強調表示されます）。

const app = require('express')();
const http = require('http').createServer(app);
const WebSocket = require('ws');
const fs = require('fs');

const wss = new WebSocket.Server({
    server: http
});

//     Google Cloud
const speech = require('@google-cloud/speech');

//   
const client = new speech.SpeechClient();

const config = {
    encoding: 'MULAW',
    sampleRateHertz: 8000,
    languageCode: 'ru-RU',
};

const request = {
    config,
    interimResults: true,
};

let audioInput = [];
let recognizeStream = null;
process.env["NODE_TLS_REJECT_UNAUTHORIZED"] = 0;

app.get('/', function(req, res) {
    res.send('<h1>Hello world</h1>');
});

wss.on('connection', (ws) => {
    //  ,      
    var wstream = fs.createWriteStream('myBinaryFile');
    //   audioInput
    audioInput = [];
    //   
    recognizeStream = client
        .streamingRecognize(request)
        .on('error', err => {
            ws.close();
        })
        .on('data', data => {
            ws.send(data.results[0].alternatives[0].transcript)
            process.stdout.write(
                data.results[0] && data.results[0].alternatives[0] ?
                `Transcription: ${data.results[0].alternatives[0].transcript}\n` :
                `\n\nError occurred, press Ctrl+C\n`
            )
        });

    ws.on('close', (message) => {
        console.log('The time limit for speech recognition has been reached. Please disconnect and call again.');
        wstream.end();
    })
    //  ,     message
    ws.on('message', (message) => {
        //     base64  recognizeStream
        try {
            let data = JSON.parse(message)
            if (data.event == "media") {
                let b64data = data.media.payload;
                let buff = new Buffer.from(b64data, 'base64');
                recognizeStream.write(buff);
                wstream.write(buff);
            }
        } catch (err) {
            console.log(message)
        }
    });
    //      
    ws.send('Hi there, I am a WebSocket server');
});

http.listen(3000, function() {
    console.log('listening on *:3000');
});

サーバーが構成されたので、音声認識を実行するのに役立ちます。ngrokを使用してlocalhost 3000にトンネルを作成することにより、ソリューションをローカルでテストします。

これを行うには、次の手順に従います。
  1. Webサイトの指示に従ってngrokをインストールします。
  2. ngrokのauthtokenを指定して、クライアントをこのアカウントにバインドします。
  3. 次のコマンドnode your_file_name.jsを実行して、サーバーをlocalhost：3000で起動します。
  4. コンピューターのngrokフォルダーに移動し、コマンド./ngrok http 3000を実行して、機能しているローカルサーバーとパブリックURLの間にトンネルを作成します。

生成されたパブリックURLに注意してください。スクリプトではプレフィックス 'wss'を使用してWebSocket URLとして使用します。



4.音声認識


バックエンドのコードにGoogle Cloudに関連する行が含まれていることにお気づきでしょう。

ライブラリ自体は次のようにインポートされます。

const speech = require('@google-cloud/speech');

次に、音声認識リクエストの処理方法を指定する必要があります。これを行うには、生成された構成でエンコーディング、sampleRateHertz、およびlanguageCodeを選択します。

const config = {
    encoding: 'MULAW',
    sampleRateHertz: 8000,
    languageCode: 'en-US',
};

次に、転送されたデータをファイルに保存できる書き込みストリームを作成します。

var wstream = fs.createWriteStream('myBinaryFile');

すべてが設定されたら、メッセージを解析し、base64形式のオーディオデータをcogniseStreamに入れる必要があります。

let data = JSON.parse(message)
if (data.event == "media") {
    b64data = data.media.payload;
    let buff = new Buffer.from(b64data, 'base64');
    recognizeStream.write(buff);
    wstream.write(buff);
}

この直後に、認識要求が開始され、この要求の処理が開始されます。

recognizeStream = client
    .streamingRecognize(request)
    .on('data', data => {
        ws.send(data.results[0].alternatives[0].transcript)
    });

最後に、サービスアカウントの認証情報を提供して、Googleライブラリをそのバックエンドに接続します。これを行うには、Google認証ページに移動し、そこに記載されているすべての手順に従います。次に、コマンドと同じワークスペース（同じ「ターミナル」タブ内）でエクスポートコマンドを実行します。node your_file_name.js:

export GOOGLE_APPLICATION_CREDENTIALS="/home/user/Downloads/[FILE_NAME].json"

ソケットを起動する


phone.voximplant.comを 開き、フォームに入力して、[サインイン]をクリックします。ユーザー名とパスワードは、手順1で作成したユーザーを指します。


認証に成功したら、[通話]をクリックして通話を開始します。Speech-to-Textクラウドは、スピーチをリアルタイムでテキストに変換し、このテキストをターミナルウィンドウで確認できます。
