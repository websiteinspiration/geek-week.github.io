レスポンシブWebデザインを5分で学ぶ

こんにちは、ハブル！Perによる「レスポンシブWebデザインを5分で学ぶ」という記事の翻訳を紹介します。

この記事では、5分でレスポンシブデザインの多くのトリックを紹介します。これは明らかに適切な研究には十分ではありませんが、ここでは以下の最も重要な方法の概要を説明します。

  • CSS相対単位
  • メディアクエリ
  • フレックスボックス
  • レスポンシブタイポグラフィ

CSS相対単位


レスポンシブWebデザインは、相対CSSユニットに基づいています。これらは、いくつかの外部値から値を取得する単位です。これは、たとえば、ブラウザウィンドウの幅に対して画像の幅を比例的に拡大縮小できるため便利です。

最も一般的な相対単位：

  • ％
  • 全角
  • レム
  • vw
  • vh

このセクションでは、最初に測定の単位としてパーセンテージ％を使用し、最後のセクションで単位remを確認します。

以下のような非常にシンプルなサイトがあるとします。



そのHTMLコードは次のとおりです。

<body>
    <h1>Welcome to my website</h1>
    <image src="path/to/img.png" class="myImg">
</body>

下のGIFからわかるように、デフォルトの画像の幅は固定されています



。画像は特に反応が遅いため、幅を70％に変更します。次のように書くだけです。

.myImg {
    width: 70%;
}

これにより、画像の幅が、タグに書き込まれた元の要素の幅の70％に設定されます。タグは画面の幅全体をカバーするため、画像は常にブラウザウィンドウ自体の70％になります。

です。レスポンシブなイメージを作成するのはとても簡単です。

メディアクエリを使用してサイトのモバイルバージョンを改善する


レスポンシブレイアウトには1つの問題があります。非常に小さな画面では奇妙に見えます。携帯電話で表示した場合、70％の幅は減少するはずです。自分の目で確かめてください。



この場合に画像をより見やすくするには、このタスクはメディアクエリ専用です。たとえば、画面の幅に応じて、さまざまなスタイルを適用できます。画面の幅が768px未満の場合は、別のスタイルを使用する必要があると規定できます。

CSSでメディアクエリを作成する方法は次のとおりです。

@media (max-width: 768px) {
    .myImage {
        width: 100%
    }
}

このCSSブロックは、画面の幅が768ピクセル未満の場合にのみ適用されます。

結果は次の



とおりです。ご覧のとおり、ページにはブレークポイントがあり、画像が広くなっています。ブラウザウィンドウの幅が768pxの場合、画像の幅は70％から100％に変わります。

Flexboxを使用したナビゲーションメニュー


次はFlexboxです。Flexboxに慣れるまで、応答性を理解することはできません。Flexboxが数年前に導入されたとき、このモジュールにより要素を軸に沿って配置することがはるかに簡単になるため、レスポンシブデザインが変更されました。

Flexboxを使用するには、ヘッダーの上にナビゲーションメニューを追加して、サイトをもう少し複雑にします。これのHTMLは次のとおりです。

<nav>
    <a href="#">Home</a>
    <a href="#">About me</a>
    <a href="#">Contact</a>
</nav>

デフォルトでは、次のようになります。



メニュー項目は左にシフトしていますが、これは必要なものではありません。ページの幅全体に均等に配置する必要があります。

これを行うには、コンテナーをフレックスボックスに変換してから、justify-contentマジックプロパティを適用します。

nav {
    display: flex;
    justify-content: space-around;
}

それを理解しましょう。ディスプレイ：flexはフレキシブルボックスに変わります。Justify-content：space-aroundは、フレキシブルコンテナー内の要素の周囲にスペースが必要であることをブラウザーに通知します。したがって、ブラウザは3つの要素の周囲に残っているすべてのスペースを均等に分散します。

これがどのように見えるかです。また、ご覧のとおり、サイトは適切に拡張されています。



レスポンシブタイポグラフィ：rem


最後のステップは、タイポグラフィをレスポンシブにすることです。ご覧のとおり、画面幅が768px未満の場合、ナビゲーションメニューとタイトルを少し小さくしたいと思います（参照ポイント、覚えていますか？）。

これを行う1つの方法は、たとえば次のように、メディアクエリのフォントサイズを小さくすることです。

@media (max-width: 768px) {
    nav {
        font-size: 14px;
    }
    h1 {
        font-size: 28px;
    }
}

それは完璧とは程遠いことが判明しました。アプリケーションには、いくつかの制御点と多くの要素（h2、h3、段落など）を含めることができます。その結果、すべての異なるブレークポイントのすべての要素を追跡する必要があります。これにより混乱が生じることは容易に想像できます。

ただし、ほとんどの場合、それらは異なる制御点で互いにほぼ同じように関係します。たとえば、h1は常に段落ではありません。

1つの要素を設定し、残りのフォントサイズをその要素に対して相対的に拡大縮小する方法がある場合はどうなりますか？

レムを入力してください！

次に、メディアクエリのrem値を変更する方法を示します。

@media (max-width: 768px) {

それと同じように、すべてのフォントサイズにブレークポイントがあります。ページが768ピクセルのマークを横切るときにフォントサイズがどのように変化するかに注意してください。
