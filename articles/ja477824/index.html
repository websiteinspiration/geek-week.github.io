リソースのスケーリング


サイトは通常、データベース、Webサーバー、キャッシングシステムなどのさまざまなモジュールで構成されています。これらの各モジュールには、さまざまなタイプと量のリソースが必要です。事前にストレステストを使用して消費されたリソースの量を分析し、ディスクI / Oの速度、プロセッサ時間、メモリ、およびサイトのインターネットストリップの幅を評価する必要があります。

ストレステストは、システムのボトルネックを特定し、事前にスケールするのに役立ちます。したがって、たとえば、アクションの実行中にハードドライブのスペースを増やしたり、サイトの帯域を拡張したり、仮想サーバーのRAMを増やしたりすることで、サーバーの能力を向上させることができます。アクション後、すべてを元に戻すことができます。これは、テクニカルサポートに連絡することなく個人アカウントで行われ、数分かかりますが、事前に、サイトでの最小限の顧客アクティビティの時間中に行うことをお勧めします。

事前にDDoS攻撃から身を守る


営業日のサイトは、顧客フローの増加だけでなく、DDoS攻撃のために減少しています。トラフィックをフィッシングリソースにリダイレクトしようとする攻撃者によって組織されます。 

DDoS攻撃は日々より巧妙になっています。ハッカーはさまざまなアプローチを使用し、DDoS攻撃とアプリケーションの脆弱性に対する攻撃に同時に頼っています。ほとんどの場合、攻撃にはサイトをハッキングする試みが伴います。

ここでは、事前に準備し、攻撃から保護されたIPアドレスをサーバーに接続することも重要です。UltraVDSでは、攻撃後ではなくサーバーを保護しますが、最大1.5 Tbpsの24時間体制の攻撃に耐えます。サーバーをDDoS攻撃から保護するために、十分に広い帯域幅でインターネットチャネルに接続されている一連のフィルターが使用されます。フィルターは通過するトラフィックを順次分析し、異常と非標準のネットワークアクティビティを明らかにします。非標準トラフィックの分析パターンには、分散型ボットネットを使用して実装されたものを含む、現在知られているすべての攻撃方法が含まれます。

安全なアドレスを仮想サーバーに接続するには、事前にプロバイダーのサポートサービスにリクエストを送信する必要があります。

サイトの読み込みを高速化


キャンペーンの期間中は、サーバーの負荷が増加し、サイトでは写真や製品カードが長時間ロードされ始めます。また、ページの読み込みは、さまざまなフレームワーク、JSライブラリ、CSSモジュールなどによって重み付けされます。競合他社よりも優れたオファーの場合でも、潜在的なクライアントは、サイトからの応答を受信せずにページを離れることができます。ページの読み込み速度を確認するには、Google DevToolsを使用することをお勧めします。

コンテンツ配信ネットワーク（CDN）は、ページの読み込みを高速化するのに役立ちます。 CDNは地理的に分散したネットワークであり、キャッシングノードで構成されます-存在のポイントは、世界中に配置できます。サイトに入るとき、クライアントは静的コンテンツをサーバーからではなく、CDNネットワークに入り、それに近いものから受信します。サーバーとクライアント間のルートを減らすことにより、サイト上のデータの読み込みが速くなります。

Windows Server Core 2019にVDSがある場合は、CDNネットワークを自分で構成できます。これには、Active Directory、DFS、IIS、WinAcme、RSATなどのオペレーティングシステムに組み込まれたツールを使用します。また、既製のソリューションを使用することもできます。たとえば、CloudFareのCDNを使用すると、問題をはるかに迅速かつ安価に解決できます。さらに、このシステムには追加機能があります。DNS、HTML、CSS、JS圧縮、多数の存在ポイント。
