画像アップロードの最適化

こんにちは、ハブル！「ウェブサイトでの画像の読み込みを最適化する方法」という記事の翻訳を紹介します。

サイトに多くの美しい画像がある場合、これは素晴らしいことですが、ロード時間が大幅に低下する可能性があり、その結果、SEOが低下し、多くのユーザーがロードする前にサイトを閉じます。メインページのスライダーを表示するためだけに、数メガバイトの画像がダウンロードされることがよくあります。

低速のモバイルインターネットを介してこのようなサイトにアクセスすると、時間がかかる可能性があります。この状況では、ほとんどのユーザーは、ダウンロードする前であってもサイトを閉じるだけです。これをテストする良い方法は、Chrome Devtoolsを通じてインターネット速度を制限する機能を利用することです。

問題


問題は、コンテンツ自体が既に読み込まれていて、画像がまだ読み込まれていることがよくあることです。その結果、ユーザーは画像の読み込みが遅い空きスペースを確認できます。これは明らかにあなたが望むものではありません。

以下の例では、4.8MBの背景画像を使用したシンプルなサイトを作成しました。ご覧のとおり、DOMは1.14秒で起動しました。実際、ユーザーには1.14秒後にコンテンツが表示されます。これは、3Gインターネットにかなり適しています。ただし、背景画像は27.32秒で読み込まれます。この間、ユーザーは既にサイトを離れている可能性があります。 ユーザーエクスペリエンスだけでなく、これも影響を受けます。2010年、Googleはダウンロード速度がランキングアルゴリズムの要素の1つであることを発表しました。今後、この要素はますます重要になると思います。





決定


この問題を解決するにはどうすればよいですか？まず、最初に、さまざまなツールを使用して背景画像を圧縮します。たとえば、TinyPNG、ImageOptimizer、JPEGminiなどです。これは簡単な勝利であり、起動時間を約10秒に短縮します。そして、それは決定の大きなステップのように見えますが、10秒はまだ多すぎます。

次のステップは、元のイメージをロードする前にスタブイメージをダウンロードすることです。スタブは元の画像のバージョンですが、解像度が低くなっています。このような画像を作成するときは、解像度を7372x4392ピクセルから20x11に下げます。その結果、画像サイズが4.8MBから900バイトに減少しました。



これにより、ダウンロード速度が10秒から550ミリ秒に低下します。しかし、今、背景として、低解像度の記録画像があります。これは、ページが最初の数秒間読み込まれる間は優れたソリューションですが、それでもユーザーに元の画像を高解像度で表示したいと考えています。

これを行うには、最初に低解像度の画像をダウンロードし、次にバックグラウンドで非同期に高解像度のオリジナルをロードする必要があります。高解像度のバージョンがロードされると、スタブイメージの代わりに配置できます。このために、以前は以下のコードを使用しました。

(() => {
  'use strict';
  // Page is loaded
  const objects = document.getElementsByClassName('asyncImage');
  Array.from(objects).map((item) => {
    // Start loading image
    const img = new Image();
    img.src = item.dataset.src;
    // Once image is loaded replace the src of the HTML element
    img.onload = () => {
      item.classList.remove('asyncImage');
      return item.nodeName === 'IMG' ? 
        item.src = item.dataset.src :        
        item.style.backgroundImage = `url(${item.dataset.src})`;
    };
  });
})();


クラスを持つ要素のためにこのコードルックスasyncImage、それはすべての属性で指定した画像をロードした後、data-srcできるだけ早く画像がロードされて、そして、それは置き換えsrc要素のために<img>、そして残りの要素のCSSを経由して、背景画像を設定します。

<div class="asyncImage" data-src="/images/background.jpg">
...
</div>

または

<img class="asyncImage" src="/images/background-min.jpg" data-src="/images/background.jpg" alt="Beautiful landscape sunrise">


スクリプトは画像のロード後に要素からクラスを削除するため、必要に応じてCSSを使用して美しいアニメーションを作成できます。たとえば、イーズインアウトアニメーションでは、画像が置き換えられたときにスタブが非表示になります。

結論


では、何を達成したのでしょうか？ユーザーエクスペリエンスを改善し、サイトの読み込みを高速化し、高速インターネットがなくてもユーザーがアクセスしやすくなり、Googleでのランキングが向上する可能性があります。これらは、そのような小さな変更に対する大きな改善です。 ご覧のように、スタブを570ミリ秒間ダウンロードします。ロードするとすぐに、元の画像の低解像度バージョンが表示され、元の画像が読み込まれるとすぐに、低解像度バージョンが置き換えられます。こちらで実際の例をご覧ください。







画像の遅延読み込み


さらに進んで画像の読み込みをさらに改善したい場合は、遅延画像読み込みが便利です。

遅延読み込みは、画像が表示領域の端に近づくまで、ページの表示領域外の画像が読み込まれない手法です。

その利点は、非常に多くの場合、画像のごく一部のみがページの表示領域に表示され、残りはユーザーがスクロールするときにダウンロードできるため、初期ロード中にロードされるデータが少ないことです。交差オブザーバーは、これを実装するのに最適です。
