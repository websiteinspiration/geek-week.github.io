Linuxカーネルセキュリティカード

Linuxカーネル保護は、非常に複雑な分野です。これには、複雑に相互接続された多数の概念が含まれており、そのグラフィック表現があると便利です。したがって、私はLinuxカーネルプロテクターマップを開発しました。ここに伝説があります：



したがって、マップには次のオブジェクトが含まれています。

  • 脆弱性クラス
  • 攻撃を行うための彼らの操作のための技術;
  • エラー検出メカニズム;
  • 保護技術。

一方、コア保護テクノロジーは異種混合です。一部はLinuxバニラカーネルの一部ですが、その他はさまざまな理由で個別に出荷されます（たとえば、商用セキュリティ機能があります）。ハードウェアサポートを必要とするカーネル保護メカニズムがあります。 

したがって、Linuxカーネルのセキュリティトピックは非常に広範囲であり、開発されたマップはその調査と分析に役立ちます。マップ上のオブジェクトを接続する各線は、相互の影響を示します。その本質は、ドキュメントで明確にする必要があります。この原理は、一般的な地図の一部に示されています。図1は、STACKLEAKテクノロジーのセキュリティ機能を示しています。



図1. STACKLEAKテクノロジーのセキュリティ機能

PAX_MEMORY_STACKLEAKは、次のタイプの脆弱性の悪用を防ぐ商用Linuxカーネル保護テクノロジーです。スタックのオーバーフローの深さ、初期化されていない変数の使用、ユーザースペースへの情報漏えい。マップには、これらのクラスの脆弱性（CWE、Common Weakness Enumeration）の識別子が含まれています。図に示されているように、STACKLEAK

テクノロジーがLinuxバニラカーネルに導入されました。カーネルをテストするときに、初期化されていない変数の使用とユーザー空間への情報漏えいを検出できるKMSANデバッグメカニズムもあります。

このカードは、コアの攻撃範囲を削減する問題に対応していないことに注意してください。実際、実行可能ファイルのサイズを縮小するほとんどすべての機能を無効にすると、攻撃の境界も縮小されます。そしてこのプロジェクトでは、主な焦点は脆弱性の悪用に対するLinuxカーネルの自己防衛を提供するセキュリティツールです。

マップは常に進化しており、少なくともカーネルのリリースごとに更新する必要があります。したがって、ソースをテキスト形式にして、バージョン管理システムで維持すると便利です。さらに、接続の交差の最小数でオブジェクトを手動で配置する必要がないように、マップを作成するためにDOT言語が選択され、次のコマンドを使用して、GraphVizパッケージによってスキームが自動的に生成されました。

Linux Kernel Security Cardは、GPL v3.0でライセンスされたオープンソースプロジェクトです。リポジトリはGitHubにあります。このプロジェクトは次の場合に非常に役立ちます。

  • カーネルセキュリティの現在の状態を監視し、
  • GNU / Linuxに基づくIPの脅威モデルの開発、
  • 適切な治療法の選択。
