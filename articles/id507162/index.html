Game Code Command & Conquer: bug dari tahun 90an. Volume Satu


Perusahaan Amerika Electronic Arts Inc (EA) telah memposting kode sumber terbuka untuk game Command & Conquer: Tiberian Dawn dan Command & Conquer: Red Alert. Kode ini akan membantu komunitas game mengembangkan mod dan peta, membuat unit khusus, dan menyesuaikan logika gameplay. Kita semua memiliki peluang unik untuk terjun ke dalam sejarah pembangunan, yang sangat berbeda dari yang modern. Kemudian tidak ada situs StackOverflow, editor kode yang mudah digunakan dan kompiler yang kuat. Dan kemudian tidak ada analisa statis, dan hal pertama yang akan ditemui komunitas adalah ratusan kesalahan dalam kode. Tetapi tim PVS-Studio akan membantu dengan ini, menunjukkan tempat kesalahan ini.

pengantar


Command & Conquer - serangkaian game komputer dalam genre strategi waktu-nyata. Game pertama dalam seri ini dirilis pada 1995. Electronic Arts mengakuisisi studio pengembangan untuk game ini hanya pada tahun 1998.

Sejak itu, beberapa game dan banyak mod telah dirilis. Kode sumber untuk game ini diterbitkan dengan rilis koleksi Remastered Command & Conquer .

Untuk menemukan kesalahan dalam kode, analisa PVS-Studio digunakan . Ini adalah alat untuk mendeteksi kesalahan dan kerentanan potensial dalam kode sumber program yang ditulis dalam C, C ++, C # dan Java.

Karena banyaknya masalah yang ditemukan dalam kode, semua contoh kesalahan akan diberikan dalam siklus dua artikel.

V610 Perilaku tidak terdefinisi. Periksa operator shift '<<'. Operan kanan ('(32 - bits_to_shift)' = [1..32]) lebih besar dari atau sama dengan panjang dalam bit dari operan kiri yang dipromosikan. MP.CPP 659

#define UNITSIZE 32

void XMP_Shift_Right_Bits(digit * number, int bits, int precision)
{
  ....
  int digits_to_shift = bits / UNITSIZE;
  int bits_to_shift = bits % UNITSIZE;

  int index;
  for (index = digits_to_shift; index < (precision-1); index++) {
    *number = (*(number + digits_to_shift) >> bits_to_shift) |
      (*(number + (digits_to_shift + 1)) << (UNITSIZE - bits_to_shift));
    number++;
  }
  ....
}

Penganalisa mendeteksi situasi di mana angka 32-bit bergeser ke kanan dengan jumlah bit yang lebih besar berpotensi terjadi. Begini caranya:

int bits_to_shift = bits % UNITSIZE;

The UNITSIZE konstan memiliki nilai 32 :

int bits_to_shift = bits % 32;

Jadi, nilai variabel bits_to_shift akan menjadi nol untuk semua nilai bit yang merupakan kelipatan dari 32 .

Oleh karena itu, dalam kode ini:

.... << (UNITSIZE - bits_to_shift) ....

pergeseran 32-bit akan terjadi jika 0 dikurangi dari konstanta 32 . Daftar semua peringatan dari PVS-Studio tentang perubahan dengan perilaku yang tidak terdefinisi:
