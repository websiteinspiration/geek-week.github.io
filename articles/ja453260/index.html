アクティブDPI

アクティブDPI-他のネットワークデバイスと同様に、通常の方法でプロバイダーのネットワークに接続されたDPI。プロバイダーはルーティングを構成して、DPIがユーザーからブロックされたIPアドレスまたはドメインへのトラフィックを受信し、DPIが既にトラフィックを許可またはブロックすることを決定しているようにします。アクティブDPIは送信トラフィックと受信トラフィックの両方をチェックできますが、プロバイダーがDPIのみを使用してサイトをレジストリからブロックする場合、ほとんどの場合、送信トラフィックのみをスキャンするように構成されます。

トラフィックのブロック効率だけでなく、DPIの負荷も接続のタイプに依存するため、すべてのトラフィックをチェックするのではなく、特定のものだけをチェックする可能性があります。

DPI


データ転送速度を低下させないためには、「通常の」パッシブDPIを使用する必要があります。何かブロック？リソース、デフォルト構成は次のようになります。

  • ポート80のみのHTTPフィルター
  • ポート443のみのHTTPS
  • 6881-6889ポートでのみBitTorrent

しかし、問題が発生します。ユーザーを失わないようにリソースが別のポートを使用している場合、各パッケージを確認する必要があります。たとえば、次のように指定できます。

  • HTTPはポート80および8080で動作します
  • ポート443および8443のHTTPS
  • 他のバンドのBitTorrent

このため、「アクティブ」DPIに切り替えるか、追加のDNSサーバーを使用してブロッキングを使用する必要があります。

DNSブロッキング


リソースへのアクセスをブロックする1つの方法は、ローカルDNSサーバーを使用してDNSクエリを傍受し、必要なリソースではなく「スタブ」のIPアドレスをユーザーに返すことです。ただし、アドレスのなりすましを防ぐことができるため、これは保証された結果を提供しません。

オプション1：ホストファイルの編集（デスクトップ用）

ホストファイルは、常に使用できるオペレーティングシステムの不可欠な部分です。リソースにアクセスするには、ユーザーは次のことを行う必要があります。

  1. 必要なリソースのIPアドレスを見つける
  2. 次の場所にある編集用のホストファイルを開きます（管理者権限が必要です）。
    • Linux：/ etc / hosts
    • Windows：％WinDir％\ System32 \ drivers \ etc \ hosts
  3. 次の形式で行を追加します：<IPアドレス> <リソース名>
  4. 変更内容を保存

この方法の利点は、その複雑さと管理者権限を持つための要件です。

オプション2：DoH（DNS over HTTPS）またはDoT（DNS over TLS）

これらの方法は、暗号化を使用してDNSクエリをスプーフィングから保護しますが、すべてのアプリケーションで実装がサポートされているわけではありません。ユーザーがMozilla Firefoxバージョン66のDoHを簡単に構成できることを考慮してください。

  1. about： Firefoxの設定に移動します
  2. ユーザーがすべてのリスクを負うことを確認する
  3. network.trr.modeパラメータの値を次のように変更します。
    • 0-TRRを無効にする
    • 1-自動選択
    • 2-デフォルトでDoHを有効にする
  4. DNSサーバーを選択してnetwork.trr.uriパラメーターを変更します。
    • Cloudflare DNS：mozilla.cloudflare-dns.com/dns-query
    • Google DNS：dns.google.com/experimental
  5. network.trr.boostrapAddressパラメータを次のように変更します。
    • Cloudflare DNSが選択されている場合：1.1.1.1
    • Google DNSを選択した場合：8.8.8.8
  6. network.security.esni.enabledパラメータの値をtrueに変更します
  7. Cloudflareを使用して設定を確認する

この方法はより複雑ですが、ユーザーの管理者権限は必要ありません。この記事で説明されていないDNSクエリを保護する方法は他にもたくさんあります。
