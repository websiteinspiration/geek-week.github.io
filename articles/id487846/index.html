RFC (Request For Comments) untuk Meningkatkan API Ember Registry


Chris Cricho ( @chriskrycho ) mengusulkan menambahkan API berorientasi objek baru untuk semua antarmuka registri; Deklarasikan API saat ini menggunakan microsyntax berbasis string dan memperkenalkan properti capabilitiesuntuk secara aman mendukung mesin pengenalan yang ada .


RFC: Enhanced Ember Registry API menjelaskan masalah saat ini dengan microsyntax dan kurangnya dukungan TypeScript. RFC merekomendasikan menambahkan Pengidentifikasi API baru yang akan berisi bidang name, typedan, secara opsional, namespacealih-alih microsyntax saat ini. Contohnya:


sebagai gantinya


getOwner(this).lookup('service:session');

akan


getOwner(this).lookup({ type: 'service', name: 'session' })

melainkan


class Example {
  @service('global@session')
  session;
}

akan


class Example {
  @service({ namespace: 'global', name: 'session' })
  session;
}

Jika Anda ingin tahu lebih banyak atau menambahkan ulasan Anda sendiri, lihat RFC di GitHub !
