日常生活「錬金術師」


サーバーのパフォーマンスの比較は複雑なトピックであり、「正しく」比較を行う方法について多くの論争を引き起こしています。この記事では、プロセッサーがサービスに入る前にプロセッサーがどのように進むかについて説明します。

最新のサーバーは多くのコンポーネントで構成されており、各コンポーネントには独自の独自のアプローチが必要です。サーバープロセッサも例外ではありません。これについては後で説明します。

手に新しい


通常、ベンダーは定規を使用してすぐに新しいプロセッサをリリースしますが、私たちが何を望んでいるかははっきりとわかっています。そしてそれが起こりました：Intelは最近、コードネームCascade Lake Refreshのアップデートをリリースしました。ラインナップには14個のプロセッサが含まれていますが、関心があるのはそのうち3つだけです。

  • インテル®Xeon®ゴールド6240R;
  • インテル®Xeon®ゴールド5218R;
  • インテル®Xeon®シルバー4214R。


これらのモデルへの関心を正当化するのは簡単です：私たちは彼らの前任者をうまく利用しました：

  • インテル®Xeon®シルバー4114;
  • インテル®Xeon®シルバー4214;
  • インテル®Xeon®ゴールド6140;
  • インテル®Xeon®ゴールド6240;
  • インテル®Xeon®ゴールド5218。

場合によっては、ベンダーは名前を明かすことなくまったく新しいものをリリースし、機密としてマークされたそのようなコンポーネントのエンジニアリングサンプルをパートナーに送信します。


まず、使用可能なマザーボードとの互換性についてプロセッサがテストされます。これは、マザーボードが未知のプロセッサで正しく動作しない可能性があるためです。幸いなことに、ベンダーはマザーボードメーカーと緊密に連携しているため、事前にBIOSアップデートをリリースして、新しいプロセッサのマイクロコードのサポートを追加できます。プロセッサーの処理能力の増加により放熱が増加する傾向があるので、テスト中に熱モードを注意深くチェックします。

物理レベルでのパフォーマンスを確認した後、理論データに基づいて比較表を作成し、パフォーマンスの比較に進みます。

これで、パフォーマンスの実用的なテストを開始できます。

"重砲"


最初は、有名なSPEC CPU 2017を使用してパフォーマンスの違いを示しましたが、SPEC CPUテストは、パフォーマンス測定の分野における事実上の標準です。プレゼンテーションのほとんどのプロセッサメーカーは、このテストスイートを使用して得られた結果を参照しています。

すべてのSPEC CPUテストは、特定の「テスト対象」のソースコードからコンパイルされます。さらに、Intel C ++コンパイラ（icc）やAMD最適化C / C ++コンパイラ（AOCC）などの最適化コンパイラの使用は禁止されていませんが、推奨されます。このアプローチにより、プロセッサと関連ツールのすべての機能を使用して、異なるアーキテクチャのプロセッサを比較できます。

SPEC組織では、テスト結果を特別なページに投稿できます。「ゴールド」の2ソケットサーバーに興味がありました。リクエストによると、次のことがわかりました。

  • Xeon®Gold 6240Rプロセッサで38件の結果。
  • Xeon®Gold 6240の結果は55件です。
  • Xeon®Gold 6140の結果は116件です。

各テストの中央値は以下の表に示されています：

プロセッサーの理論上のデータを分析すると、ベンチマークでの指標の成長を期待して予測していると言えます。

SPECに加えて、他のより理解しやすいベンチマークを使用します。

ミニマルなアプローチ


パフォーマンスを測定するためのそれほど人気のないプログラムはありませんが、標準になっていないかもしれませんが、いくつかの名声を獲得しています。それはPassMarkパフォーマンステストとGeekBenchについてです。

GeekBench（公開時点ではバージョン5が該当）は、プロセッサーのパフォーマンス測定を専門としており、独自の結果を確認して他のユーザーと比較できるオープンな評価表を備えています。 SPEC CPUとは異なり、GeekBenchは整数および浮動小数点での計算に加えて、特別なプロセッサ命令を含む暗号計算を実行します。

x86_64アーキテクチャに基づく3つの一般的なオペレーティングシステムファミリがサポートされています。Windows、Linux、MacOS Xです。GeekBenchはARMアーキテクチャのデバイスでも動作しますが、AndroidとiOSでのみ動作するため、このアーキテクチャのサーバーで動作する可能性はありません。

テスト結果は、基準に対するポイントで提供されます。 GeekBench 5では、Intel Core i3-8100がベンチマークとして選択され、その結果は1000ポイントでした。プロセッサーが2000ポイントを獲得した場合、それは標準の2倍の強力であることを意味します。

GeekBenchの代わりにPassMark PerformanceTestを選択しました。 PerformanceTestはWindowsにのみ適合しているという事実にもかかわらず、より幅広いプロファイルを備えており、CPUだけでなく、システム内のRAM、ディスク、およびグラフィックスアクセラレータ（存在する場合）もテストします。

GeekBenchとPerformanceTestの主な違いは、実行可能ファイルとしての配布です。このようなテストは、一般的な最適化のみが可能で、新しいプロセッサ機能をサポートできないクローズドソースソフトウェアに似ています。

「実際の」タスク


一部の専門家は、パフォーマンスのベンチマークについて懐疑的です。 「合成！重要ではありません！」彼らは言う、そして彼ら自身のやり方で正しい。 IT企業の専門家は、独自のシステムのコンテキストで排他的に新しいプロセッサを検討します。同時に、プロセッサの処理能力だけでなく、RAMやペリフェラルとの相互作用も必要になります。

IT分野で使用されるソフトウェア製品の種類は非常に多く、それぞれに多くの設定があります。膨大な数のテストの準備には受け入れられないほど長い時間がかかり、経験豊富な専門家がテストの構成に対する不満を表明することもできます。
Selectel LABプロジェクトの枠組みの中で、スペシャリストにテスト用の新しい機器が提供されます。これにより、スペシャリストはあらゆるタスクに対して独自の独立したテストを実施できます。
プロセッサと北部全体の両方のパフォーマンスを評価できるいくつかのタスクを選択しました。ここにリストがあります：

  • 積極的な最適化によるLLVMプロジェクトのコンパイル。
  • πの小数点以下100万桁の計算。
  • ffmpegでビデオを再コーディングします。

LLVMプロジェクトは多くの複雑なプログラムで構成されているため、すべてのプロジェクトプログラムのコンパイルプロセスは、強力なサーバー上でも完全に並列化されます。テストでは、ストレージとしてSSDを使用し、12個のDDR4-2666 RAMダイを使用します。
テスト Xeon®Gold 6240 Xeon®Gold 6240R
数πの計算 3分20.559秒 3分16.246秒
ffmpegによるビデオのトランスコーディング 0分46.646秒 0分46.634秒
LLVMプロジェクトを積極的な最適化でコンパイルする 15分35.933秒 15分0.370秒
実用的なテストは、以前に実施された「合成」テストの結果を確認します。LLVMプロジェクトプログラムのコンパイルは並行して実行されるため、コアの数が増えると、予想どおりにプロセスが加速します。π数の計算では1つのスレッドのみを使用します。このテストでは、最大プロセッサー周波数の増加に伴う速度のわずかな増加を確認できます。

ビデオのトランスコーディングはそれほど簡単ではないタスクであり、ffmpegアプリケーション自体の裁量で並行して計算されます。プロセッサ間のこのようなわずかな違いは、ソフトウェア機能によるものです。

結論の代わりに


進歩は止まらず、カスケードレイクラインの更新は非常に予期された出来事でした。私たちのテストでは、コアの数の増加、L3キャッシュの増加、およびクロック周波数の増加が全体的なプロセッサのパフォーマンスに有利に影響し、「合成」と実際の条件に近いタスクの両方に適切に対処できることがわかりました。

自分のタスクで新しいプロセッサーを試すことができます。サーバーはコントロールパネルで注文できます。
