RDPセッションでWindowsロック画面をバイパスする方法

先日、セキュリティ研究者がMicrosoft Windowsリモートデスクトッププロトコル（RDP）の新しい脆弱性の詳細を明らかにしました。



脆弱性CVE-2019-9510では、クライアント側の攻撃者がリモートデスクトップセッションのロック画面をバイパスすることができます。

Carnegie Mellon University Software Development InstituteのJoe Tammarielloがこの脆弱性を発見しました。この脆弱性を悪用するには、RDP認証にネットワークレベル認証（NLA）が必要です。ちなみに、最近マイクロソフト自身が脆弱性BlueKeep RDP（CVE-2019-0708）から保護することを推奨したのはNLAでした。

CERT / CCのアナリストであるWill Dormannが確認したように、クライアントがサーバーにすでに接続されているが、ログイン画面がロックされているときにネットワーク異常により一時的なRDP切断が発生した場合、「再接続後、RDPセッションは以前の状態に復元されます（ウィンドウがロック解除された状態で）、リモートシステムがどのように放置されていても。」

「Windows 10 1803およびWindows Server 2019以降、セッションのNLAベースのRDP処理が変更され、セッションのブロックに関する予期しない動作が発生する可能性があります」とドーマン氏は記事で説明しています。

「Duo Security MFAなど、Windowsログイン画面と統合する2要素認証システムでも、このメカニズムを回避できます。組織が使用するログインバナーもバイパスされます。」

つまり、攻撃者はターゲットシステムのネットワーク接続を中断する必要があるだけなので、この脆弱性の悪用は非常に簡単です。
ただし、攻撃者はそのようなターゲットシステムへの物理的なアクセス（つまり、画面がロックされたアクティブセッション）を必要とするため、スクリプト自体は非常に限られた数のケースにしか近づきません。

Tammarielloは4月19日にこの脆弱性をマイクロソフトに通知しましたが、同社は「この動作はWindowsのマイクロソフトセキュリティサービシング基準に準拠していません」と回答しました。つまり、技術大手は近い将​​来に問題を修正する予定はありません。

ただし、ユーザーは、リモートシステムではなくローカルシステムをブロックし、単にロックするのではなくリモートデスクトップセッションを切断することにより、この脆弱性の悪用から身を守ることができます。
