Verilogキャッシュの実装

この記事では、Verilogでの最も単純なRAM実装について説明します。

コードの解析に進む前に、Verilogの基本的な構文を学ぶことをお勧めします。

ここにトレーニング資料があります。

羊


ステップ1：対応する入力/出力信号でモジュールを宣言する


module ram (
	input [word_size - 1:0] data,
	input [word_size - 1:0] addr,	
	input wr,	
	input clk,	
	
	output response,
	output [word_size - 1:0] out
);		

parameter word_size = 32;

  • データ -書き込みにデータ。
  • addr -RAM内のメモリへのアドレス。
  • wr-ステータス（読み取り/書き込み）。
  • clk-クロックサイクルシステム。
  • 応答 -RAMの準備（1-RAMが読み取り/書き込み要求を処理した場合、0-それ以外の場合）。
  • out -RAMから読み取られたデータ。

この実装は、32ビットアーキテクチャのアルテラマックス10 FPGAに統合されたため、データとアドレスのサイズ（word_size）は32ビットです。

ステップ2：モジュール内でレジスターを宣言する


データを格納するための配列宣言：

parameter size = 1<<32;
reg [word_size-1:0] ram [size-1:0];

alwaysブロックでの変更を追跡するために、以前の入力パラメーターを保存する必要もあります。

reg [word_size-1:0] data_reg;	
reg [word_size-1:0] addr_reg;
reg wr_reg;

そして、alwaysブロックでの計算後に出力信号を更新するための最後の2つのレジスタ：

reg [word_size-1:0] out_reg;
reg response_reg;

レジスタを初期化します。

initial
begin	
	response_reg = 1;
	data_reg = 0;
	addr_reg = 0;
	wr_reg = 0;
end

ステップ3：ブロックの常時ロジックを実装する


always @(negedge clk)
begin		
	if ((data != data_reg) || (addr%size != addr_reg)|| (wr != wr_reg))
	begin
		response_reg = 0;
		data_reg = data;
		addr_reg = addr%size;
		wr_reg = wr;
	end
	else
	begin
		if (response_reg == 0)
		begin
			if (wr)
				ram[addr] = data;
			else
				out_reg = ram[addr];
				
			response_reg = 1;
		end
	end
end

常にブロックは、negedjeによってトリガーされます。クロックが1から0に移動する瞬間。これは、RAMをキャッシュと正しく同期させるために行われます。そうしないと、RAMにレディステータスを1から0にリセットする時間がなく、次のクロックで、RAMが要求を正常に処理したとキャッシュが判断する場合がありますが、これは根本的に間違っています。

常にブロックアルゴリズムのロジックは次のとおりです。データが更新された場合は、準備状態を0にリセットし、データを書き込み/読み取り、書き込み/読み取りが完了した場合は、準備状態を1に更新します。

最後に、次のコードセクションを追加します。

assign out = out_reg;
assign response = response_reg;

私たちのモジュールの出力信号のタイプはワイヤーです。このタイプの信号を変更する唯一の方法は、alwaysブロック内で禁止されている長期割り当てです。このため、alwaysブロックはレジスタを使用し、レジスタは後で出力信号に割り当てられます。

直接マッピングキャッシュ


直接マッピングキャッシュは、最も単純なタイプのキャッシュの1つです。この実装では、キャッシュはn個の要素で構成され、RAMは条件付きでn個のブロックに分割され、キャッシュのi番目の要素は、i = k％nの条件を満たすRAMのそのようなすべてのk番目の要素に対応します。

下の図は、サイズ4のキャッシュとサイズ16のRAMを示しています。



各キャッシュ要素には、次の情報が含まれています。

  • 有効性ビット -キャッシュ内の情報が関連しているかどうか。
  • tagは、この要素が配置されているRAM内のブロック番号です。
  • データ -私たちが読み書きする情報。

読み取りが要求されると、キャッシュは入力アドレスをタグとインデックスの2つの部分に分割します。インデックスのサイズはlog（n）です。nはキャッシュのサイズです。

ステップ1：対応する入力/出力信号でモジュールを宣言する


module direct_mapping_cache 
(	
	input [word_size-1:0] data,	
	input [word_size-1:0] addr,
	input wr,	
	input clk,	
	
	output response,	
	output is_missrate,	
	output [word_size-1:0] out
);	
	parameter word_size = 32;

キャッシュモジュールの宣言は、新しい出力信号is_missrateを除いて、RAMと同じです。この出力には、最後の読み取り要求がミスレートであったかどうかに関する情報が格納されます。

ステップ2：レジスタとRAMを宣言する


レジスタを宣言する前に、キャッシュとインデックスのサイズを決定します。

parameter size = 64;
parameter index_size = 6;

次に、書き込みと読み取りを行うデータが格納される配列を宣言します。

reg [word_size-1:0] data_array [size-1:0];

また、各アイテムの有効性ビットとタグをキャッシュに保存する必要があります。

reg validity_array [size-1:0];		
reg [word_size-index_size-1:0] tag_array [size-1:0];
reg [index_size-1:0] index_array [size-1:0];

入力アドレスが分割されるレジスター：

reg [word_size-index_size-1:0] tag;
reg [index_size-1:0] index;

前のクロックの入力値を格納するレジスター（入力データの変化を追跡するため）：

reg [word_size-1:0] data_reg;
reg [word_size-1:0] addr_reg;
reg wr_reg;

alwaysブロックでの計算後に出力信号を更新するためのレジスタ：

reg response_reg;
reg is_missrate_reg;
reg [word_size-1:0] out_reg;

RAMの入力値：

reg [word_size-1:0] ram_data;
reg [word_size-1:0] ram_addr;
reg ram_wr;

RAMの出力値：

wire ram_response;
wire [word_size-1:0] ram_out;

RAMモジュールを宣言し、入力信号と出力信号を接続する：

ram ram(
	.data(ram_data),
	.addr(ram_addr),			
	.wr(ram_wr),
	.clk(clk),			
	.response(ram_response),
	.out(ram_out));

レジスタの初期化：

initial
integer i
initial
begin		
	data_reg = 0;
	addr_reg = 0;
	wr_reg = 0;
	for (i = 0; i < size; i=i+1)
	begin
		data_array[i] = 0;
		tag_array[i] = 0;
		validity_array[i] = 0;			
	end
end

ステップ3：ブロックの常時ロジックを実装する


まず、クロックごとに2つの状態があります。入力データが変更されているか、変更されていません。これに基づいて、次の条件があります。

always @(posedge clk)
begin
	if (data_reg != data || addr_reg != addr || wr_reg != wr)
	begin
	end				
		// 1:   
	else
	begin
		// 2:    
	end
end

ブロック1.入力データが変更された場合、最初に行うのは準備状態を0にリセットすることです。

response_reg = 0;

次に、前のクロックの入力値を保存したレジスターを更新します：

data_reg = data;
addr_reg = addr;
wr_reg = wr;

入力アドレスをタグとインデックスに分割します。

tag = addr >> index_size;
index = addr;

タグを計算するには、ビット単位の右シフトを使用します。インデックスについては、単純に割り当てるだけで十分です。アドレスの余分なビットは考慮されません。

次のステップは、書き込みと読み取りのどちらかを選択することです。

if (wr)
begin
	// 
	data_array[index] = data;
	tag_array[index] = tag;			
	validity_array[index] = 1;

	ram_data = data;
	ram_addr = addr;
	ram_wr = wr;
end
else
begin
	// 
	if ((validity_array[index]) && (tag == tag_array[index]))
	begin
		//   
		is_missrate_reg = 0;

		out_reg = data_array[index];
		response_reg = 1;
	end
	else
	begin
		//    
		is_missrate_reg = 1;

		ram_data = data;
		ram_addr = addr;
		ram_wr = wr;			
	end
end

記録の場合、最初にキャッシュ内のデータを変更してから、RAMの入力データを更新します。読み取りの場合、この要素がキャッシュに存在するかどうかを確認し、存在する場合はout_regに書き込みます。それ以外の場合は、RAMにアクセスします。

ブロック2.前のクロックが実行されてからデータが変更されていない場合、次のコードがあります。

if ((ram_response) && (!response_reg))
begin
	if (wr == 0)
	begin
		validity_array [index] = 1;
		data_array [index] = ram_out;
		tag_array[index] = tag;
		out_reg = ram_out;
	end
	response_reg = 1;
end

ここで、RAMが呼び出しを完了するまで待機し（呼び出しがなかった場合、ram_responseは1）、読み取りコマンドがあった場合はデータを更新し、キャッシュの準備を1に設定します。

最後に、出力値を更新します。

assign out = out_reg;
assign is_missrate = is_missrate_reg;
assign response = response_reg;
