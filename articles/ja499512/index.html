最低でもAHK。バインダー

考え


ねえ。私は初心者のC＃.NET開発者です（2年目の初心者として）。いろいろな電卓などを書くのにうんざりしていたので、「Windowsには何が足りないのか」と自問しました。そして、私がたどり着いた答えは、「すべて」という考えを与えてくれました。そこで、C＃、. NET、WPFについて多くを学ぶことができるプロジェクトであるバインダーがありました。これにより、私は非常に気に入った優れたソフトウェアを入手して、みんなに見せることにしました。

プログラム自体について


バインダーは、意外にもバインド用です。これらのバインドは、任意のキーの組み合わせ（ボタン+ 2修飾子（CTRL、Shift、ALt、Win）、およびそれらのスティックスクリプト）に対して構成できます。スクリプトは、C＃にできるだけ類似するように内部言語で書かれています。
原理はAutoHotKeyと同じです。プログラムにはDLLが付属しており、そこにゆっくりとさまざまな関数を追加します（現在49個あります）。私の意見では、バインダー言語は、ほとんど能力はありませんが、すでにインタープリター型プログラミング言語と見なすことができます。 if-elseコンストラクト、whileおよびrepeatループ、関数（非同期も）、breakおよびreturnステートメント、int、double、bool、stringデータ型がすでに存在します。

プログラムでは、バインダーはPC全体でグローバルに機能し、AHKから取得したキーシミュレーションは、ほとんどすべてのアプリケーションでキーストロークをシミュレートできます。

バインダーで解決したかった最初のタスクは、デスクトップアイコンを表示/非表示にすることでした。したがって、最初に追加したスクリプトはShowHideDesktopIcons（ブールショー）です。しかし、ここで問題が発生します。1つのボタンで非表示と表示の両方を行う方法は？バインダーに「バインドスイッチ」パラメーターを追加することでそれを決定しました。これにより、2つのスクリプトを1つのバインドに一度にハングさせ、順番に実行することができます。既にバインダーで、オートクリッカー、またはバインド、Ctrl + Shift + Cなどを選択して、選択したテキストをクリップボードに追加し、置き換えることはできません。または、バインドをオフにしてPCを再起動することができます。最も有用なものの1つは、「Super Alt + F4」バインドです。これは、アクティブウィンドウプロセスを受け取って強制終了します。プログラムは変数をサポートしています。SetVar（）を使用して任意の値を保存し、GetVar（）で取得できます。スクリプトを実行するために作られた属性を持つ今日、1つ目はBlockで、メインスレッドが完了するまでバインドのキーストロークをブロックしていました。近い将来、名前空間を追加して変数を1つのスクリプトから別のスクリプトに転送する予定です。

便利な追加機能は、キーボードとマウスからのすべてのアクションを記録し、すぐにバインダーのスクリプトに変換するRecord関数です。

ヘルプウィンドウでは、使用可能なすべての関数を見つけることができ、すぐにスクリプトの属性が表示されます。

すべてのバインドは、失われないようにファイルに保存できます。

Windowsでのプログラムの起動を有効にする設定では、開くデフォルトのバインドファイルを構成できます。



便利なことに注意したいのですが、入れ子構造を使用している場合（たとえば、while内の場合）、[保存]ボタンをクリックすると、スクリプトが美しくフォーマットされます。

構文


バインダーでは、すべてのプログラマーが使い慣れた構文は、C＃に似ています。

たとえば、MsgBox関数の呼び出し（他の関数と同様）は次のようになります。

MsgBox(" ", "    ");

この行を実行すると、結果は次のようになります



。MsgBox（）はパラメーターを受け入れ、ヘルプウィンドウで各関数のパラメーターを確認できます。

役立つように、すべての関数は戻り値のタイプでソートされます



。C++ / C＃で書き込んだ人は誰でも、これがどのように機能するかをすぐに理解できます。ただし、最初のタイプを除きます。
例えば：

MsgBox(MsgBox("Hello"));

2回実行すると、「Hello」というテキストが付いたメッセージが表示されます。

この行から、もう1つの重要な詳細を理解できます。一部の関数は、他の関数によってパラメータに転送できます。

組み込み関数に加えて、プログラムにはさまざまなデザインがあります。私は1つのチームとみなされる全体の構造を持っているので、それらのそれぞれの最後に置く必要があることに注意してください。後で削除します。

構成を繰り返す


repeatはプログラム内で最も単純なループであり、括弧内にInt値がなければならず、指定された回数だけアクションを繰り返すだけです。次に例を示します。

repeat(SumInt(2,3))
{
    MsgBox("");
};

「こんにちは」というメッセージを5回表示します。

2つの変更点があります

。1）非同期リピート（）。別のスレッドがその実行に割り当てられます。開始直後、それ以降の関数が実行されます。

2）allasync repeat（）は、指定された回数だけコードの実行をすぐに開始します。次に例を示します。

allasync repeat(5)
{
    MsgBox("Hello");
};

同時に5つのメッセージが表示されます。



構築中


ながら-文字通り英語から。-条件が真の間。括弧内の条件が真である限り、それ自体の中で機能を実行します。この条件はブール型である必要があります。

ループの実行は、breakステートメントを使用して強制的に中止できます。例：

SetVar("i", 0);
while(true)
{
    SetVar("i", SumInt(GetVar("i"), 1));
    MsgBox(GetVar("i"));
    if(MoreThan(GetVar("i"), 10))
    {
        break;
    };
    
};
MsgBox("");

1から11までの番号のメッセージを出力し、終了します。

whileには、別のスレッドで単に実行するasync whileの変更もあり、メインコードの実行を継続できます。

機能


バインダーでは、関数を宣言して使用し、パラメーターをそれらに渡すことができます

func f1(text)
{
MsgBox(ReplaceString(text, "\", " habr "));
};
f1(GetDesktopPath());

私の出力：



関数は値を返すこともできます。そのためには、returnステートメントを使用する必要があります。

func f1(text)
{
    return MsgBox(ReplaceString(text, "\", " habr "));
};
MsgBox(f1(GetDesktopPath()));

これで同じ結果になり、メッセージのみが2回表示されます。
