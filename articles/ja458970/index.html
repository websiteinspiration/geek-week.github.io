UIViewPropertyAnimatorを使用してカスタムアニメーションを作成する

アニメーションの作成は素晴らしいです。これらは、iOSヒューマンインターフェイスガイドラインの重要な部分です。アニメーションは、重要なことにユーザーの注意を引き付けたり、アプリケーションの退屈さを軽減したりします。

iOSでアニメーションを実装する方法はいくつかあります。おそらく最も一般的な方法は、UIView.animate（withDuration：animations :)を使用することです。CABasicAnimationを使用して画像レイヤーをアニメーション化できます。また、UIKitはあなたが使用してコントローラを表示するには、configureカスタムアニメーションすることができますUIViewControllerTransitioningDelegateを。

この記事では、ビューをアニメーション化するもう1つのエキサイティングな方法について説明します-UIViewPropertyAnimator。このクラスは、前のUIView.animat e よりもはるかに多くの管理機能を提供します。これを使用して、一時的でインタラクティブな中断されたアニメーションを作成します。さらに、アニメーターを素早く変更することが可能です。

UIViewPropertyAnimatorの紹介


UIViewPropertyAnimatorはiOS 10で導入されました。オブジェクト指向の方法でアニメーションを作成できます。UIViewPropertyAnimatorを使用して作成されたアニメーションの例を見てみましょう。



これは、UIViewを使用したときの状態です。

UIView.animate(withDuration: 0.3) {
    view.frame = view.frame.offsetBy(dx: 100, dy: 0)
}

そして、UIViewPropertyAnimatorでそれを行う方法は次のとおりです。

let animator = UIViewPropertyAnimator(duration:0.3, curve: .linear) {
     view.frame = view.frame.offsetBy(dx:100, dy:0)
 }
 animator.startAnimation()

インタラクティブで中断されたアニメーション


従来の「指でスワイプしてロックを解除する」ジェスチャーを覚えていますか？または、「画面上で指を下から上に動かす」というジェスチャーでコントロールセンターを開きますか？これらはインタラクティブなアニメーションの素晴らしい例です。画像を指で動かしてから離すと、画像が元の位置に戻ります。さらに、アニメーション中に画像をキャッチし、指で動かし続けることができます。

UIViewアニメーションは、アニメーションの完了率を制御する簡単な方法を提供しません。ループの途中でアニメーションを一時停止して、中断後に実行を続けることはできません。

この場合、UIViewPropertyAnimatorについて説明します。次に、簡単な手順で、完全にインタラクティブな、中断されたアニメーションと反転アニメーションを簡単に作成する方法を見ていきます。

展開と折りたたみアニメーションの実装


アプリケーションを起動すると、都市のリストが表示されます。ただし、ユーザーはセルの形でオブジェクトを操作できません。ここで、ユーザーがセルの1つをクリックしたときに、各都市の情報を表示する必要があります。アプリケーションの最終バージョンを見てください。実際に開発する必要があるのは次のとおりです。


アニメーションは良さそうですね。ただし、ここでは特別なことは何もありません。UIViewPropertyAnimatorの基本的なロジックにすぎません。このタイプのアニメーションを実装する方法を見てみましょう。collectionViewメソッド（_：didSelectItemAt）を作成し、次のコードをViewControllerファイルの最後に追加します。

func collectionView(_ collectionView: UICollectionView, didSelectItemAt indexPath: IndexPath) {
    let selectedCell = collectionView.cellForItem(at: indexPath)! as! CityCollectionViewCell
    selectedCell.toggle()
}

次に、toggleメソッドを実装する必要があります。CityCollectionViewCell.swiftに切り替えて、このメソッドを実装してみましょう。

まず、ファイルの先頭、CityCollectionViewCellクラスの宣言の直前にState列挙を追加します。このリストにより、セルの状態を追跡できます。

private enum State {
    case expanded
    case collapsed
    
    var change: State {
        switch self {
        case .expanded: return .collapsed
        case .collapsed: return .expanded
        }
    }
}

CityCollectionViewCell クラスのアニメーションを制御するいくつかのプロパティを追加します。

private var initialFrame: CGRect?
private var state: State = .collapsed
private lazy var animator: UIViewPropertyAnimator = {
    return UIViewPropertyAnimator(duration: 0.3, curve: .easeInOut)
}()

initialFrame 変数は、アニメーションが実行されるまでセルフレームを格納するために使用されます。状態は、セルが展開されているか折りたたまれているかを追跡するために使用されます。そして、アニメーター変数はアニメーションを制御するために使用されます。

次に、toggleメソッドを追加し、それをcloseメソッドから呼び出します。次に例を示します。

@IBAction func close(_ sender: Any) {
    toggle()
}
 
func toggle() {
    switch state {
    case .expanded:
        collapse()
    case .collapsed:
        expand()
    }
}

次に、expand（）とcollapse（）という 2つのメソッドを追加します。今後も継続して実施していきます。まず、展開された（）メソッドから始めます。

private func expand() {
    guard let collectionView = self.collectionView, let index = self.index else { return }
    animator.addAnimations {
        self.initialFrame = self.frame
        
        self.descriptionLabel.alpha = 1
        self.closeButton.alpha = 1
        
        self.layer.cornerRadius = 0
        self.frame = CGRect(x: collectionView.contentOffset.x, y:0, width: collectionView.frame.width, height: collectionView.frame.height)
        
        if let leftCell = collectionView.cellForItem(at: IndexPath(row: index - 1, section: 0)) {
            leftCell.center.x -= 50
        }
        
        if let rightCell = collectionView.cellForItem(at: IndexPath(row: index + 1, section: 0)) {
            rightCell.center.x += 50
        }
        
        self.layoutIfNeeded()
    }
    
    animator.addCompletion { position in
        switch position {
        case .end:
            self.state = self.state.change
            collectionView.isScrollEnabled = false
            collectionView.allowsSelection = false
        default:
            ()
        }
    }
    animator.startAnimation()
}

  2. , animator.addAnimations.
  3. , .
  4. - descriptionLabel closeButton, .
  5. . .

  7. 次に、animator.addComplete（）メソッドを呼び出して、コレクション画像の相互作用を無効にします。これにより、セルの展開中にユーザーがスクロールできないようにします。セルの現在の状態も変更します。セルの状態を変更することが重要です。その後、アニメーションが終了します。

次に、畳み込みアニメーションを追加します。つまり、セルを以前の状態に復元するだけです。

private func collapse() {
    guard let collectionView = self.collectionView, let index = self.index else { return }
    
    animator.addAnimations {
        self.descriptionLabel.alpha = 0
        self.closeButton.alpha = 0
        
        self.layer.cornerRadius = self.cornerRadius
        self.frame = self.initialFrame!
        
        if let leftCell = collectionView.cellForItem(at: IndexPath(row: index - 1, section: 0)) {
            leftCell.center.x += 50
        }
        
        if let rightCell = collectionView.cellForItem(at: IndexPath(row: index + 1, section: 0)) {
            rightCell.center.x -= 50
        }
        
        self.layoutIfNeeded()
    }
    
    animator.addCompletion { position in
        switch position {
        case .end:
            self.state = self.state.change
            collectionView.isScrollEnabled = true
            collectionView.allowsSelection = true
        default:
            ()
        }
    }
    animator.startAnimation()
}

ジェスチャー処理を追加する


UIView.animate を使用して同じ結果を達成すると主張できます。UIViewPropertyAnimatorを使用する意味は何ですか？

さて、アニメーションをインタラクティブにする時が来ました。追加UIPanGestureRecognizerと呼ばれる新しいプロパティpopupOffsetをセルが移動することができますどのくらい追跡します。これらの変数をCityCollectionViewCellクラスで宣言してみましょう。

private let popupOffset: CGFloat = (UIScreen.main.bounds.height - cellSize.height)/2.0
private lazy var panRecognizer: UIPanGestureRecognizer = {
    let recognizer = UIPanGestureRecognizer()
    recognizer.addTarget(self, action: #selector(popupViewPanned(recognizer:)))
    return recognizer
}()

次に、次のメソッドを追加して、スワイプ定義を登録します。

override func awakeFromNib() {
    self.addGestureRecognizer(panRecognizer)
}

次に、スワイプジェスチャを追跡するためにpopupViewPannedメソッドを追加する必要があります。次のコードをCityCollectionViewCellに貼り付けます。

@objc func popupViewPanned(recognizer: UIPanGestureRecognizer) {
    switch recognizer.state {
    case .began:
        toggle()
        animator.pauseAnimation()
        
    case .changed:
        let translation = recognizer.translation(in: collectionView)
        var fraction = -translation.y / popupOffset
        if state == .expanded { fraction *= -1 }
        animator.fractionComplete = fraction
        
    case .ended:
        animator.continueAnimation(withTimingParameters: nil, durationFactor: 0)
        
    default:
        ()
    }
}

3つの状態があります。ジェスチャーの開始時に、トグルメソッドを使用してアニメーターを初期化し、すぐに一時停止します。ユーザーがセルをドラッグしている間に、fractionCompleteマルチプライヤのプロパティを設定してアニメーションを更新します。これがアニメーターの主な魔法であり、制御することができます。最後に、ユーザーが指を離すと、continueAnimationアニメーターメソッドが呼び出されてアニメーションが続行されます。次に、セルがターゲット位置に移動します。

アプリケーションを起動した後、セルを上にドラッグして展開できます。次に、展開したセルを下にドラッグして折りたたみます。

これでアニメーションはかなり良く見えますが、途中でアニメーションを中断することはできません。したがって、アニメーションを完全にインタラクティブにするには、別の機能を追加する必要があります-中断。ユーザーは通常どおり展開/折りたたみアニメーションを開始できますが、アニメーションサイクル中にユーザーがセルをクリックした直後にアニメーションを一時停止する必要があります。

これを行うには、アニメーションの進行状況を保存し、この値を考慮してアニメーションの完了率を計算します。

まず、CityCollectionViewCellで新しいプロパティを宣言します。

private var animationProgress: CGFloat = 0

次に、popupViewPannedメソッドの.beganブロックを次のコード行で更新して、進行状況を記憶します。

animationProgress = animator.fractionComplete

.changed ブロックでは、完了率を正しく計算するために、次のコード行を更新する必要があります。

animator.fractionComplete = fraction + animationProgress

これで、アプリケーションをテストする準備ができました。プロジェクトを実行し、何が起こるかを確認します。私の指示に従ってすべてのアクションが正しく実行されると、アニメーションは次のようになります。


アニメーション逆


現在の実装の欠陥を見つけることができます。セルを少しドラッグしてから元の位置に戻すと、指を離してもセルは拡大し続けます。この問題を修正して、インタラクティブアニメーションをさらに改善しましょう。以下で説明するように、
ブロック更新の.endメソッドpopupViewPannedを実行します。

let velocity = recognizer.velocity(in: self)
let shouldComplete = velocity.y > 0
 
if velocity.y == 0 {
    animator.continueAnimation(withTimingParameters: nil, durationFactor: 0)
    break
}
 
switch state {
case .expanded:
    if !shouldComplete && !animator.isReversed { animator.isReversed = !animator.isReversed }
    if shouldComplete && animator.isReversed { animator.isReversed = !animator.isReversed }
case .collapsed:
    if shouldComplete && !animator.isReversed { animator.isReversed = !animator.isReversed }
    if !shouldComplete && animator.isReversed { animator.isReversed = !animator.isReversed }
}
animator.continueAnimation(withTimingParameters: nil, durationFactor: 0)

次に、ジェスチャーの速度を考慮して、アニメーションを反転する必要があるかどうかを判断します。

そして最後に、別のコード行を.changedブロックに挿入します。このコードをanimator.fractionComplete計算の右側に置きます。

if animator.isReversed { fraction *= -1 }

アプリケーションをもう一度実行してみましょう。これですべてが確実に機能するはずです。


パンジェスチャーを修正


したがって、UIViewPropertyAnimatorを使用してアニメーションの実装を完了しました。ただし、1つの不快な間違いがあります。アプリケーションのテスト中に彼女に会った可能性があります。問題は、セルを水平方向にスクロールできないことです。セルを左右にスワイプしてみましょう。問題に直面しています。

主な理由は、作成したUIPanGestureRecognizerに関連しています。また、スワイプジェスチャを取得し、組み込みのジェスチャレコグナイザUICollectionViewと競合します。

ユーザーは引き続きセルの上部/下部またはセル間のスペースをスクロールして都市をスクロールできますが、そのような悪いユーザーインターフェイスはまだ好きではありません。修正しましょう。

競合を解決するには、gestRecognizerShouldBegin（_ :)というデリゲートメソッドを実装する必要があります。このメソッドは、ジェスチャレコグナイザがタッチを解釈し続けるかどうかを制御します。メソッドでfalseを返すと、ジェスチャー認識機能はタッチを無視します。だから私たちがやろうとしていることは、私たち自身のパノラマ認識ツールに水平移動を無視する能力を与えることです。

これを行うには、パン認識機能のデリゲートを設定します。次のコード行を挿入しpanRecognizerの初期化（あなたが右の前にコードを置くことができ、戻り認識：

recognizer.delegate = self

次に、gestRecognizerShouldBegin（_ :)メソッドを次のように実装します。

override func gestureRecognizerShouldBegin(_ gestureRecognizer: UIGestureRecognizer) -> Bool {
    return abs((panRecognizer.velocity(in: panRecognizer.view)).y) > abs((panRecognizer.velocity(in: panRecognizer.view)).x)
}

縦方向の速度が横方向の速度より大きい場合、開閉します。

うわー！アプリケーションをもう一度テストしてみましょう。セルを左/右にスワイプすると、都市のリストを移動できます。


ボーナス：カスタム同期機能


このチュートリアルを終了する前に、タイミング関数について説明しましょう。作成したアニメーションにカスタム同期関数を実装するように開発者から要求されたケースをまだ覚えていますか？

通常、UIView.animationをCABasicAnimationに変更するか、CATransactionでラップする必要があります。スルーUIViewPropertyAnimator簡単にカスタムタイミング機能を実装することができます。タイミング関数（またはイージング関数）の

下では、アニメーション化されている特定のプロパティの変化率に影響を与えるアニメーション速度関数を指します。現在サポートされているタイプは、easeInOut、easeIn、easeOut、linearの4つです。

次のように、アニメータの初期化をこのタイミング関数に置き換えます（独自のベジェ3次曲線を描いてみてください）。

private lazy var animator: UIViewPropertyAnimator = {
    let cubicTiming = UICubicTimingParameters(controlPoint1: CGPoint(x: 0.17, y: 0.67), controlPoint2: CGPoint(x: 0.76, y: 1.0))
    
    return UIViewPropertyAnimator(duration: 0.3, timingParameters: cubicTiming)
}()

または、3次同期パラメーターを使用する代わりに、春の同期を使用することもできます。次に例を示します。

    let springTiming = UISpringTimingParameters(mass: 1.0,
                                                stiffness: 2.0,
                                                damping: 0.2,
                                                initialVelocity: .zero)
