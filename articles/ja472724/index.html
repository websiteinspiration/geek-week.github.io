Skydiveの概要


Skydiveは、リアルタイムのオープンソースネットワークトポロジーおよびプロトコルアナライザーです。ネットワークインフラストラクチャで何が起こっているかを理解するための網羅的な方法を提供することを目的としています。


kubernetesのオブジェクト（ポッド、インターフェース）のスクリーンショット


公式プロジェクトページ：https : //github.com/skydive-project/skydive
Kubernetesは以下になります。
次に、より簡単な例を見てみましょう。


実例として、3台のサーバーにEtcdクラスターをインストールします。このために、銀河の役割frank6866.etcdを使用します。


ansible-galaxy install frank6866.etcd

以下のYaml


- hosts: etcd
  become: yes
  roles:
      - frank6866.etcd

以下のAnsible.cfgファイル


[defaults]
inventory = inventory
remote_user = user
host_key_checking = False
stdout_callback=debug

インベントリファイル


frank6866-etcd-1 ansible_ssh_host=172.26.9.78 etcd_public_ip=172.26.9.78
frank6866-etcd-2 ansible_ssh_host=172.26.9.79 etcd_public_ip=172.26.9.79
frank6866-etcd-3 ansible_ssh_host=172.26.9.80 etcd_public_ip=172.26.9.80

[cluster1]
frank6866-etcd-[1:3]

[etcd:children]
cluster1

[etcd:vars]
etcd_tls_enabled='false'

インストールを開始します


 ansible-playbook -i hosts.multiple etcd-cluster.yaml

skydiveリポジトリをダウンロードした後


git clone https://github.com/skydive-project/skydive.git

contrib / ansible / inventoryフォルダーに移動します


cd contrib/ansible/inventory

hosts.multipleファイルのIPを編集する


[analyzers]
IP   

[agents]
 IP etcd 

#      skydive-flow-matrix 
# For skydive-flow-matrix add skydive_extra_config:
[agents:vars]
skydive_extra_config={'agent.topology.probes': ['socketinfo',]}

skydiveエージェントとアナライザーのインストールを開始します


ansible-playbook -i inventory/hosts.multiple playbook.yml.sample

私たちがコンピューターから入ってIP :8082
