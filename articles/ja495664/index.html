STM32F407で3Dグラフィックスを表示する

ご存知のように、古いSTMにはまともな周波数とRAMボリュームがあります。そうだとすれば、そのようなコントローラで3Dグラフィックスを実行してみませんか？はい、簡単なことはありません！

デモ画像

3Dグラフィックスを表示するために、STM32F407に基づくSTM32F4Discoveryボードに320x240解像度のディスプレイを接続しました。いいえ、FSMCではありません-このボードには必要な接点があります。ただし、私たちの実験では、通常のポートで十分です。
ディスプレイは次のように接続されています。
  • CS-> E12
  • RST-> E2
  • RS-> E15
  • WR-> E14
  • RD-> E13
  • D0-> E4
  • D1-> E5
  • D2-> E6
  • D3-> E7
  • D4-> E8
  • D5-> E9
  • D6-> E10
  • D7-> E11


リポジトリには、3つの異なる表示オプション（IL9325、SPFD5408、HX8347D）のクラスがあります。


外観

3Dグラフィックスを描画するために、私は非常に昔に、最初のバージョンのOpenGLライブラリーのフラグメントの類似点を作成しました。これは私がSTM32に取り付けたものです。このライブラリには、ピクセルごとに6バイトが必要です（カラーごとに2バイト、Zバッファーごとに4バイト）。このバージョンのSTM32F407のメモリは、160x120で十分です。それでは、画像を縦横2倍に伸ばします。
このレッスンでは、次のことができます
。1）2の累乗の倍数のサイズのテクスチャテクスチャ。
2）Zクリッピング;
3）顔の内側の頂点の色の補間。
4）距離からの減衰調整による8つの光源の照明計算。

ライブラリを使用する手順は次のとおりです。
まず、ライブラリを初期化する必要があります（画面にメモリを割り当てます）。次に、OpenGLと同様に、射影行列とビューポートを構成する必要があります。
 const int32_t WIDTH=160;
 const int32_t HEIGHT=120;
 const float VISIBLE_ANGLE=60;
 const float NEAR_PLANE=1;
 const float FAR_PLANE=1000;	
 float aspect=static_cast<float>(WIDTH)/static_cast<float>(HEIGHT);
 		
 cSGL.Init(WIDTH,HEIGHT);
 cSGL.Perspective(VISIBLE_ANGLE,aspect,NEAR_PLANE,FAR_PLANE);
 cSGL.SetViewport(0,0,WIDTH,HEIGHT);


実際には、OpenGLとほぼ同じ方法で描画できます（同様のコマンドの場合）。

たとえば、光源を指定できます。
 cSGL.MatrixMode(CSGL::SGL_MATRIX_MODELVIEW);
 cSGL.LoadIdentity();
 float l0_position[]={0,0,0};
 float l0_ambient[]={0.1,0.1,0.1};
 float l0_diffuse[]={0.7,0.7,0.7};
 float l0_specular[]={1,1,1};
 float l0_shininess[]={1};
 cSGL.Lightfv(CSGL::SGL_LIGHT0,CSGL::SGL_POSITION,l0_position);
 cSGL.Lightfv(CSGL::SGL_LIGHT0,CSGL::SGL_AMBIENT,l0_ambient);
 cSGL.Lightfv(CSGL::SGL_LIGHT0,CSGL::SGL_DIFFUSE,l0_diffuse);
 cSGL.Lightfv(CSGL::SGL_LIGHT0,CSGL::SGL_SPECULAR,l0_specular);
 cSGL.Lightfv(CSGL::SGL_LIGHT0,CSGL::SGL_SHININESS,l0_shininess);


表面の材質を指定できます：
 float m0_ambient[]={0.1,0.1,0.1};
 float m0_diffuse[]={0.5,0.5,0.5};
 float m0_specular[]={0.5,0.5,0.5};
 float m0_emission[]={0.1,0.1,0.1};
 cSGL.Materialfv(CSGL::SGL_AMBIENT,m0_ambient);
 cSGL.Materialfv(CSGL::SGL_DIFFUSE,m0_diffuse);
 cSGL.Materialfv(CSGL::SGL_SPECULAR,m0_specular); 
 cSGL.Materialfv(CSGL::SGL_EMISSION,m0_emission);


ライティング計算を有効にすることができますcSGL.Enable（CSGL :: SGL_LIGHTING）;そして特定の光源cSGL.Enable（CSGL :: SGL_LIGHT0）;
フロントとリアの照明は同等に考慮されています。さらに、私はバックエッジをカットしませんでした（めったに必要としません）。しかし、あなたがそれをいくつかの些細なことにしたいなら。

光源を作成する必要はありません。 cSGL.Color3fを使用して、自分で顔の色を設定することは可能です。色（およびマテリアルの色パラメーターも）は、正規化[0..1]で指定されます。

テクスチャーはcSGL.BindTextureコマンドで接続されます。ライブラリは常にテクスチャリングを使用していることに注意してください！この場合、テクスチャの色の配列は4バイトのシーケンス（r、g、b、alpha）として定義され、ここでr、g、b、alphaは0〜255の数値です。厳密に言うと、チャネルR、G、レンダリング時にglColor3fによって指定されたB（または光源から計算されたB）。テクスチャデータの配列は、顔が表示されている間、維持する必要があります。

顔の結論は次のとおりです。
  
  cSGL.Begin();
   cSGL.TexCoordf(0,0);	
   cSGL.Vertex3f(x6,y6,z6);
   cSGL.TexCoordf(0,1);	 
   cSGL.Vertex3f(x4,y4,z4);
   cSGL.TexCoordf(1,0);	 
   cSGL.Vertex3f(x2,y2,z2);
  cSGL.End();


cSGL.Beginは面の描画を有効にし、cSGL.Endはこのモードを終了します。これらのコマンドの内部には、ポイントのパラメーター（通常の正規化されたテクスチャ座標、ポイントの正規化された色）とポイントレンダリングcSGL.Vertex3fを設定するコマンドが配置されています。begin / end内のポイントの数は任意であり、全体として表示される凸多角形を構成します。

テクスチャリング、投影、モデリングは、OpenGLと同様に、コマンドで選択された3つのマトリックスによって制御されます。
MatrixMode(SGL_MATRIX_TEXTURE);
MatrixMode(SGL_MATRIX_PROJECTION);
MatrixMode(SGL_MATRIX_MODELVIEW);

これらの行列を使用した操作は、コマンドによって実行されます
LoadIdentity();
cSGL.Rotatef(angle,0,0,1);
cSGL.Translatef(-0.5,-0.5,0);


描画では、ライブラリはCGLScreenColorポイントクラスを使用します。これはパフォーマンスには影響しませんが、異なるディスプレイやアーキテクチャ間でライブラリを簡単に転送できます。
FPSとは何ですか？ 1つの面のない特定の八面体（私はそれを撮った）の場合、実際には、描画手順の継続時間を測定すると、約150〜200 FPSになります。しかし、画像を2倍にしてバッファーポートを介してディスプレイに転送すると、FPSが約10〜15に大幅に減少します（はい、今のところ最適化については考えていません）。
ライブラリがほとんどの場所で最適化されていないという事実に注意を向けます。彼女の仕事は、近代化の可能性を備えた理解可能な3D描画ツールを作成することでした。最適化することで、STM32へのグラフィック出力をもう一度5〜10倍高速化できると思います（たとえば、FSMCを使用して、描画を最適化し、データをディスプレイに転送します）。

なぜそのようなライブラリが役立つのでしょうか？ええと、たとえば、ユーザーへの指示を視覚的な形式で表示します。たとえば、単純な3Dオブジェクトの向きです。
