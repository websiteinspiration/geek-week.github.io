SelectelのHPEサーバー


今日、Selectelブログのゲスト投稿-Hewlett Packard Enterprise（HPE）のテクニカルコンサルタントであるAlexey Pavlovが、Selectelサービスの使用経験について語ります。私たちは彼に床を与えます。

サービスの品質を確認する最良の方法は、自分で使用することです。当社の顧客は、リソースの一部をプロバイダーと一緒にデータセンターに配置するオプションをますます検討しています。慣れ親しんだ実績のあるプラットフォームを扱いたいという顧客の理解したい欲求を、より便利なセルフサービスポータル形式で。

Selectelは最近発売し、顧客のための新しいHPEサーバサービスを。そしてここで問題が発生します：どのサーバーがよりアクセス可能ですか？あなたのオフィス/データセンターまたはプロバイダーのどちらに立っていますか？

従来のアプローチとプロバイダーからの機器レンタルのモデルを比較することによって顧客が解決する問題を思い出してください。

  1. どのくらいの速さで予算に同意し、ブレインチャイルドのパイロット打ち上げのための機器の構成を注文できますか？
  2. 設備の場所を見つける場所。サーバーをテーブルの下に置かないのですか？
  3. ハードウェアスタックの複雑さが増しています。すべてを把握するために、一日のどこで余分な時間を見つけることができますか？

そのような質問や類似の質問に対する回答は長い間あります。サービスプロバイダーに助けを求めることです。以前は、Selectelに機器レンタルサービスを注文することはできませんでしたが、ここではすべてを詳細にテストして説明することもできました。

すべてのアプリケーションは、興味のあるサービスを選択できるポータルから作成されます。


既製のサーバー構成を選択できます。多くのオプションがあります。以前は、このようなモデルは「固定」構成と呼ばれていました。これらは、必要なものが正確にわかっている場合に選択され、動作中に構成を変更する必要はありません。サーバーは事前に組み立てられ、データセンターにマウントされています。


場所、定規、または事前定義されたタグで検索すると便利です。十分な既製の構成がない場合は、目的のモデルを組み立てることができます。

任意の構成の各サーバーは、注文のために個別に組み立てられます。このサイトには、互換性のあるすべてのコンポーネントを備えたサーバーの構築に役立つコンフィギュレーターがあります。創造性の余地があります！契約に基づき、任意の構成のサーバーが最大5営業日で顧客にリースされます。

私の場合、注文は金曜日の夜に行われ、土曜日の8:00にサーバーコンソールにアクセスしました。


多くのお客様は、SAP HANA、MS SQL、Oracle、その他の産業用ソフトウェアで利用できる認定済みオプションの幅広い選択肢など、さまざまな理由でHPEラインのサーバーでの作業に慣れています。現在、これらのサーバーはSelectelポートフォリオに含まれています。


このようなアプリケーションを効果的に展開するには、会社に十分なリソースが必要です。お客様に連絡する際、ソフトウェアとサーバーを個別にではなく、ソリューション全体を設計します。お客様とともに、ソフトウェアおよびハードウェアの製造元によって開発されたリファレンスアーキテクチャと構成について説明し、完全な構成、スケールと寸法、すべての仕様と展開情報を完成させます。

HPEは、これらのリファレンスソリューションを、サービスプロバイダーがお客様のプロジェクトへのさらなる展開のための既製のテスト済みテンプレートとして使用できる多くの異なる構成をカバーするプログラムの一部として開発します。

ベンチマーク


HPEサーバー選択アプローチの利点の1つは、さまざまなベンチマークについてテストすることです。したがって、データベースボリューム、ユーザー数、パフォーマンスなど、事前に定義された負荷の構成を選択できます。

HPE DL380 Gen10サーバーには、すべてのサーバー間で4つのTPC-H（Transaction Processing Councilアドホック/意思決定支援ベンチマーク）レコードがあります。


証明書データウェアハウスのファストトラックの結果


このような証明書を使用すると、テストの一部として特定の構成でシステムのパフォーマンスを評価し、それを次のプロジェクトで予想される特性と大まかに比較できます。

興味深い：バージョン2016以降のMicrosoft SQL Server製品はクラウド製品として開発され、1日あたり数十億のクエリを実行する20以上のサイトのAzureサービスでテストされました。これが、このようなシステムをプロバイダーのデータセンターで実行する必要があるもう1つの理由です。

「また、22のグローバルデータセンターにまたがってAzureで最初にデプロイおよびテストされた機能の大部分を備え、1日あたり数十億のリクエストがある「クラウドファースト」として生まれたのは、おそらく世界で唯一のリレーショナルデータベースでしょう。顧客テスト済みで、戦闘準備が整っています。」（マイクロソフト、Joseph Sirosh）

HPEポートフォリオには、さまざまな産業用プラットフォームでテストされた専用サーバーソリューションが含まれています。たとえば、インフラストラクチャの「ビルディング基盤」として使用できるHPE DL380 Gen10サーバー。彼らはSQL 2017のテストで優れた結果を示し、2018年9月のQphH（1時間あたりのクエリパフォーマンス）の最低コスト：3000 GBでQphHあたり0.46 USDです。

データベースを操作する


DL380 Gen10サーバーには、SQLを操作するための興味深い機能がありますか？

HPE DL380 Gen10は、ビット単位のメモリアクセスを提供する永続メモリテクノロジーをサポートし、レイテンシを削減してトランザクション数を41％に増やします。このような構成のテスト結果は公開されています。


NVDIMMテクノロジーにより、254キューのSASおよびSATAとは異なり、64kの膨大な数のI / Oキューで作業できます。もう1つの重要な利点は、SSDの3〜8分の1の低レイテンシです。

同様のテスト結果が、OracleおよびMicrosoft Exchangeシステムで利用可能です。



NVDIMMテクノロジーに加えて、HPE機器のSelectelでテストできるIntel Optaneデバイスは、データベースアクセラレーションツールの武器として積極的に使用されています。予備的なテスト結果がSelectelブログで公開されました。

機能とテクノロジー


HPE Proliant Gen10サーバーには、他のサーバーよりも際立つ独自のテクノロジーがいくつかあります。

まず、セキュリティ。HPEは、ランタイムファームウェア検証を導入しました。これは、サーバーにインストールする前に、サーバーがその元のファームウェア署名を確認できるようにするテクノロジーです。これにより、なりすましやルートキット（マルウェア）のインストールを回避できます。

プロセッサーのタイプ


HPE ProLiant Gen10には、5つのCPUカテゴリが用意されています。

  • ERP、メモリ分析、OLAP、仮想化、コンテナ用のプラチナ（8100、8200シリーズ）。
  • OLTP、アナリティクス、AI、Hadoop / SPARK、Java、VDI、HPC、仮想化およびコンテナー向けのゴールド（6100 / 5100、6200 / 5200シリーズ）。
  • SMBロード、Webフロントエンド、ネットワークおよびストレージアプリケーション用のシルバー（4100、4200シリーズ）。
  • SMBロード用のブロンズ（3100、3200シリーズ）。

これに加えて、すべてのお客様にとって、Gen10サーバーには多くの革新があります。



ワークロードマッチング -特定のタイプの負荷（SQLなど）に対するすべてのサーバーパラメーターの自動調整。測定結果は、通常の設定と比較して最大9％のパフォーマンスの違いを示しています。これは、サーバーを最大限に活用したいお客様には非常に適しています。

ジッタースムージング-Turbo Boostをオンにした後、設定されたプロセッサー周波数をスプリアスピークなしで保持します。最小限の遅延でより高い周波数で作業する必要があるお客様に最適です。

コアブースティング -プロセッサー周波数を上げてオーバーヘッドを削減できます。Oracleなど、コアの数によってライセンスされるソフトウェアを使用するお客様に最適です。このテクノロジーにより、使用するコアの数を減らすことができますが、周波数は高くなります。

Selectelコントロールパネル


私はなんとかSelectelの注文パネルを使用して作業できました。非常に快適な感じで、ナビゲーションはシンプルで、どこに何が配置されているかは明らかです。

サーバーからのすべてのトラフィックを制御し、IPアドレスを割り当てることができます。



さまざまなOSをインストールできます。インストールは自動的に開始されます。



インストール後、KVMコンソールに移動し、サーバーのすぐ隣にいるかのように通常モードで作業を続けます。



アナリストによると、すべての中小企業の半分以上がインフラストラクチャの少なくとも一部をサービスプロバイダーに転送しています。大規模な顧客には、サービスプロバイダーとの連携を担当する部門全体があります。

Selectelを使用すると、ビジネス上の問題の解決が簡単になり、いくつかの理由があります。

  1. 資金調達の困難に関する問題を解決する（機器の購入と独自のインフラストラクチャの構築のため）。
  2. インフラストラクチャの準備が整っており、市場への製品の発売を加速しています。
  3. 資格のある専門家の支援が常に手元にあります。
  4. インフラストラクチャの拡張、シンプルな構成から始めて、システムをビジネスのニーズに柔軟に適応させることは簡単です。
  5. 最新のテクノロジーは、あらゆるアプリケーションとあらゆる負荷に対して事前テスト済みの構成でテストするために利用できます。
  6. HPEのテスト済みで実績のあるエンタープライズソリューションは、さまざまな構成で利用できます。
