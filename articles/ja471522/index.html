アスコジア。プラグアンドプレイの自動プロビジョニングの仕組み

Askozia PBXを開発するとき、私たちは電話を自動的にセットアップし、独自の方法で解決するという課題に直面しました。

プラグアンドプレイ（PnP）を自動プロビジョニングするこのテクノロジーは、Yealink、Snom、Fanvilなどの多くのメーカーでサポートされています。

自動電話設定の主な利点：

  • 初期設定が容易 -各デバイスのWebインターフェースにアクセスする必要はありません。デバイスのMACアドレスと自動調整サーバー上のアカウントの対応を示すだけで十分です。
  • サポートを簡素化 -デバイスの設定を変更する必要がある場合、それは本当に簡単になります。サーバーの設定を再度制御します
  • 設定をスターコード「* 911 * <SIP_ACC>」のセットに減らすことが可能です -場合によっては、この機能は単に価格が設定されていません。すべてのオフィスワーカーがIP電話を構成できるわけではありませんが、番号の組み合わせをダイヤルするのは簡単な作業です。

自動プロビジョニングプラグアンドプレイの仕組みについて説明します。記事の最後に、PnPサーバーの機能を実装する小さなPHPスクリプトのソースへのリンク。

ケーススタディ


顧客の1人にテレフォニーを導入するときに、興味深い事例がありました。主な問題は、クライアントが別の都市にいることでした。同時に、要件の1つは、電話で小包を開梱してネットワークに接続した後、電話がすぐに機能することでした。

問題は比較的簡単に解決されました。クライアントはいくつかのIPアドレスを予約してくれました。事務所に機器を設置し、箱に詰めて宅配便で発送しました。

自動デバイス構成を使用すると、タスクがはるかに簡単になります。

PnPサーバーの機能


  • アドレス ' 224.0.1.75:5060 'に送信されたリクエストをリッスンします
  • 起動すると、Webサーバー（busybox httpd）が起動します
  • 簡単な電話構成を作成できます
  • YealinkにNOTIFYを送信して再起動できます

PnPサーバーを使用すると、「ワンタイムリンク」を使用できます。


リンクでファイルを指定するとします：

http://172.16.32.153:84/0015657322ff.cfg

ファイルへの通常の直接リンク。明らかに、これは安全ではありません。電話のMACアドレスとサーバーアドレスがわかっているので、ログインとパスワードを使用して設定を取得することができます。

PnPサーバーを使用する場合、SUBSCRIBEリクエストごとに一意のリンクを指定できます。

http://172.16.32.153:84/?mac=0015657322ff&hash=0a67f5290

ハッシュを計算するための式の例：

hash = md5(MAC + DATE + PID)

PIDは、PnPサーバーのプロセスIDです。rootだけが認識できます。
そのようなハッシュを選択することはほとんど不可能です。

無効なリンクを介して申し立てが行われた場合、害虫を禁止します。

Yealink NOTIFYを再起動すると、権限がありません


はい、はい、許可なし。
この可能性から、ファームウェアの現在のバージョンからデバイスを閉じることができませんでした。

コマンドを実行するだけ

php -f MikoServerPnP.php socket_client_notify <IP_PBX> <PORT_SIP_PBX> <IP_PHONE> <PORT_PHONE>

そして、電話が再起動します。このようなコマンドをcronでぶら下げることにより、恐ろしい効果を得ることができます。もちろん、これは電話のIPアドレスとSIPポートがわかっている場合に可能です。

NOTIFYを送信するPHP関数の例
    public static function socket_client_notify($ip_pbx, $port_pbx, $ip_phone, $port_phone):void {
        $phone_user = 'autoprovision_user';
        $sock = socket_create(AF_INET, SOCK_DGRAM, SOL_UDP);
        $msg =  "NOTIFY sip:{$phone_user}@{$ip_phone}:{$port_phone};ob SIP/2.0\r\n".
            "Via: SIP/2.0/UDP {$ip_pbx}:{$port_pbx};branch=z9hG4bK12fd4e5c;rport\r\n".
            "Max-Forwards: 70\r\n".
            "From: \"asterisk\" <sip:asterisk@{$ip_pbx}>;tag=as54cd2be9\r\n".
            "To: <sip:{$phone_user}@{$ip_phone}:{$port_phone};ob>\r\n".
            "Contact: <sip:asterisk@{$ip_pbx}:{$port_pbx}>\r\n".
            "Call-ID: 4afab6ce2bff0be11a4af41064340242@{$ip_pbx}:{$port_pbx}\r\n".
            "CSeq: 102 NOTIFY\r\n".
            "User-Agent: mikopbx\r\n".
            "Allow: INVITE, ACK, CANCEL, OPTIONS, BYE, REFER, SUBSCRIBE, NOTIFY, INFO, PUBLISH, MESSAGE\r\n".
            "Supported: replaces, timer\r\n".
            "Subscription-State: terminated\r\n".
            "Event: check-sync;reboot=true\r\n".
            "Content-Length: 0\r\n\n";
        $len = strlen($msg);
        socket_sendto($sock, $msg, $len, 0, $ip_phone, $port_phone);
        socket_close($sock);
    }
