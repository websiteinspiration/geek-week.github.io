Creating a prototype SPA online store on htmlix.js

This article will describe the step-by-step creation of the frontend for the prototype SPA online store. The server part is written in node.js using express, the template engine is twig, the database is NeDB, the formidable module is used to process form data, the client part is written in htmlix.

For those who are not familiar with Htmlix, read the basics of building applications here .

The source data for this lesson can be downloaded here .

This is a ready-made example, we just delete everything that is there from the file /static/js/front.js and insert it instead:

Next, create a product card component cart_single , but before you create it, let's see how htmlix works:

So, when loading a page from the address "/" or "/ category /: idCategory", the server sends us the html code of two components, more precisely three, another “menu”, but we don’t use it yet, we use html code for the carts components and categories, based on this code htmlix creates templates for components, but the template for the cart_single component is not transmitted to us at these addresses, because he's not there. How do we “load it”? To do this, we create the cart_single component in the application description and place it in the special “fetchComponents” object, and in the `stateSettings.templatePath` settings, specify the address where to load the templates for this component, and all the others that will appear. Thus, the application will first create the components that the server sent along with html,and then after sending the second request to stateSettings.templatePath, it will create the remaining components located in the fetchComponents object.

So we would do it if we would always have access to applications from the addresses, "/" or "/ category /: idCategory", but what if we then first load the application from the address "/ cart /: idCart"? The application will give an error that it cannot find a template for the carts array, since now it will not be at this address and we now need to put carts in the “fetchComponents” object, and on the contrary, get single_cart from there, because it should be initialized first together with the categories component, what should I do?

To solve this problem, we will use HTMLixRouter (), before creating an application instance, after loading the page, it checks what url address is now and compares it with those that we will indicate to it, and then it puts those for which there is no template in the first load, into the “fetchComponents” object, after which it initializes the application, thus we will avoid creating several options for describing the application for different urls and, accordingly, code duplication.

So now we have three addresses "/", "/ category /: idCategory" and "/ cart /: idCart", the three main components are carts, categories and cart_single. At the first two addresses, the components carts and categories should be initialized first, and at the address "/ cart /: idCart", categories and cart_single should be initialized.
