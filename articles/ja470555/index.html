研修


2回2回起きないように、前述のデュエットの2日間のトレーニングについて説明します。会議直後の10月27日と28日に、エフゲニーボリソフとキリルトルカチョフは、「スプリングブートマスタリング+スプリングクラウドマスタリング」という2つのトレーニングを実施します。

本当に理解したい場合：

  • マイクロサービスを作成するときにSpring Bootが正確に行うこと
  • すべてをカスタマイズする方法;
  • Tomcatがスレッド不足にならないように非同期リクエストの問題を解決する方法。
  • ;
  • ;
  • , ;

次に、Spring Boot Masteringが最適です。

解決策として、マイクロサービスの記述方法についての理解を深めることができます。また、マイクロサービスには新しい機会だけでなく、新しい問題もあるので、解決の責任がどこにもないため、Spring Cloud Masteringの後半では、マイクロサービスアーキテクチャに関連する問題を掘り下げることができます。それらを正しく回避できるようにする必要があります。これにより、Spring Cloudも可能になります（もちろん、すべてではありません）。 Spring Cloudによって非常に促進されているマイクロサービスアーキテクチャに切り替えるときに、どのような問題が待ち受けているかを理解したい場合は、それらに対処し、この動的に発展するスタックに遅れずについていくことができれば、Spring Cloudマスタリングが最適です。

会議の反対側、より正確にはその開始前に、10月23日と24日にトレーニング「効果的なのは、Javaアプリケーションのプロファイリング」、となっている著者とホストアレクセイRagozin。Alexeyは、さまざまな業界で高負荷の分散システムの開発に10年以上の経験があり、ロシアおよび外国のソフトウェア会議で積極的に講演し、オープンソースの貢献者です。

アレクセイのリーダーシップの下で、プロファイラーを仕事で効果的に使用するための重要な知識とスキルを得ることができます。プロファイラーの使用を必要とするタスクに直面することは、最も一般的なケースではないことを多くの人が理解していると思います。このため、多くの開発者は通常、自分の経験からこのツールを効果的に所有する方法を学ぶ機会がありません。アレクセイは、トレーニングの枠組みの中で彼の経験と知識を伝達します。そして、重大な状況では、これは問題を解決するための時間を劇的に減らすことができます。
