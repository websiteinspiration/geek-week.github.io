最も人気のあるデータ構造

データ構造とは何ですか？
簡単に言えば、データ構造は、データが特定のレイアウト（形式、またはメモリでの編成方法）で格納されるコンテナーです。この「レイアウト」により、一部の操作ではデータ構造が有効になり、他の操作では無効になります。あなたの目標は、問題の問題に最も最適なデータ構造を選択できるように、データ構造を理解することです。

なぜデータ構造が必要なのですか？
データ構造はデータを体系的に格納するために使用され、データはコンピューターサイエンスの最も重要な要素であるため、データ構造の真の価値は明白です。
解決する問題が何であれ、従業員の給与、株価、買い物リスト、または単純な電話帳でさえ、何らかの方法でデータを処理する必要があります。
さまざまなシナリオに基づいて、データは特定の形式で保存する必要があります。データをさまざまな形式で格納する必要性をカバーするいくつかのデータ構造があります。

スタック
私たちは皆、ほとんどすべてのアプリケーションに存在する有名なオプションの取り消し（キャンセル）に精通しています。これがどのように機能するのか疑問に思ったことはありますか？このメカニズムの本質は、最後のアクションが最初に表示されるように、作業の以前の状態（特定の数によって制限される）をメモリに保存することです。これは配列だけではできません。そこで役立つのがスタックです。

スタックの実際の例は、垂直に配置された本のスタックです。真ん中のどこかで本を入手するには、その上にある本をすべて削除する必要があります。LIFO（後入れ先出し）メソッドの仕組みは次のとおりです。
これは、3つのデータ要素（1、2、3）を含むスタックの画像です。3が一番上にあり、最初に削除されます。



スタックの基本的な操作：
プッシュ-他の要素の上に要素を挿入します。
プル（ポップ）-スタックから削除した後、一番上の要素を返します。
空の？ （IsEmpty）-スタックが空の場合、true（true）を返します。
Top（Top）-スタックから削除せずに一番上の要素を返します。

独立した調査の場合：

リンクリスト
リンクリストは、一見すると配列に似ていると思われるもう1つの重要な線形データ構造ですが、メモリ割り当て、内部構造、および基本的な挿入および削除操作の実行方法が異なります。

リンクリストはノードのチェーンのようなもので、各ノードにはデータなどの情報とチェーン内の次のノードへのポインターが含まれます。リンクされたリストの最初の要素を指すヘッダーポインターがあり、リストが空の場合、それは単にゼロまたは何も指しません。

リンクリストは、ファイルシステム、ハッシュテーブル、および隣接リストの実装に使用されます。

  • 単一リンクリスト（単方向）
  • 二重リンクリスト（双方向）


リンクリストの基本操作
InsertAtEnd- リンクリストの最後にアイテムを挿入します。
InsertB Start（InsertAtHead）-リンクリストの先頭にアイテムを挿入します。
削除-リンクされたリストからこのアイテムを削除します。
DeleteBeginning（DeleteAtHead）-リンクリストの最初の要素を削除します。
検索-リンクされたリストから見つかったアイテムを返します。
空の？ （IsEmpty）-リンクリストが空の場合、true（true）を返します。

独立した調査の場合：

3.リンクリストの最後からN番目のノードを返します。
4.リンクリストから重複を削除します。

グラフ
グラフは、ネットワークの形で相互に接続されているノードのコレクションです。ノードは頂点とも呼ばれます。ペア（x、y）はエッジと呼ばれ、頂点xが頂点yに接続されていることを示します。エッジには重量/コストを含めることができ、上部のxからyに移動するのにかかるコストを示します。



グラフのタイプ：
  • 無向グラフ
  • 有向グラフ


プログラミング言語では、グラフは2つの形式で表されます。

  • 隣接リスト

ツリー
ツリーは、頂点（ノード）とそれらを接続するエッジで構成される階層データ構造です。ツリーはグラフのように見えますが、ツリーとグラフを区別する重要な点は、ループがツリー内に存在できないことです。木は破れたグラフです。

ツリーは、人工知能や複雑なアルゴリズムで広く使用されており、問題解決アルゴリズムの効率的なストレージメカニズムを提供します。

8.マップの色分け：有名な4色の定理は、マップ領域を常に正しく色付けできるため、2つの隣接する領域に同じ色が割り当てられないようにして、4つ以下の異なる色を使用すると述べています。このモデルでは、色付きの領域がノードであり、隣接はグラフのエッジです。

ハッシュ表
ハッシュは、オブジェクトを一意に識別し、「キー」と呼ばれる事前に計算された一意のインデックスによって各オブジェクトを格納するために使用されるプロセスです。したがって、オブジェクトはキーと値のペアの形式で格納され、そのような要素のコレクションは辞書と呼ばれます。各オブジェクトは、このキーを使用して見つけることができます。

さまざまなハッシュベースのデータ構造がありますが、ハッシュテーブルは最も一般的に使用されるデータ構造です。ハッシュテーブルは、キーを持つ要素にアクセスする必要がある場合に使用され、キーの有用な値を決定できます。

ハッシュテーブルは通常、配列を使用して実装されます。

データ構造のハッシュのパフォーマンスは、次の3つの要因に依存します。
  • ハッシュ関数
  • ハッシュテーブルサイズ
  • 衝突処理方法


ハッシュが配列にどのように表示されるかを示します。この配列のインデックスは、ハッシュ関数を使用して計算されます。



独立した研究の場合：
1.アレイで対称ペアを見つけます。
2.完全な移動経路に従います。
3.配列が別の配列のサブセットであるかどうかを確認します。
4.指定された配列が互いに素であるかどうかを確認します。

以下は、.Netのハッシュテーブル使用コードの例です。

static void Main(string[] args)
        {

4.チェスをプレイするためのプログラムを作成するときは、以前に評価された位置を追跡し、必要に応じて再び必要なときに戻ってくることができるようにすることが非常に重要です。
