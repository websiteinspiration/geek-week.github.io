Bagaimana saya menemukan kesalahan perangkat keras pada tablet Lenovo MiiX 2 10 saya



Artikel ini membahas masalah dengan daya / baterai untuk tablet jenis ini:
selama pengoperasian normal tablet, peristiwa mulai terjadi

  1. Selama operasi, indikator baterai berhenti berubah
  2. Saat Anda mengunci layar pada gorden di sudut kanan bawah, indikator baterai menghilang
  3. Di pengelola tugas, perangkat "Baterai" tidak lagi ditampilkan
  4. Seiring waktu, tablet "tiba-tiba" mati. Setelah menghubungkan pengisi daya, LED merah berkedip pada tombol daya, menunjukkan bahwa baterai benar-benar kosong
  5. Setelah mengisi dan memuat, tablet terus gagal

Ternyata, alasannya adalah masalah perangkat keras dengan model tablet Lenovo ini.

Jadi, saya membeli tablet Lenovo MiiX 2 10. Untuk sementara semuanya baik-baik saja, itu berfungsi, OS diperbarui, dll. Kemudian kegagalan yang dijelaskan dalam daftar di atas dimulai. Mereka terjadi setiap 5-20 hari. Memperbarui / mengganti driver tidak membantu. Karena tablet menggunakan Windows (8.1), jawaban sederhana muncul: Windows dan M $ yang harus disalahkan, tentu saja. Dan ya, setelah menginstal ulang (memulihkan) Windows, tablet bekerja dengan baik. Untuk sementara saya pikir masalah ini telah teratasi. Tetapi menginstal ulang Windows seminggu sekali entah bagaimana mengganggu, dan saya memberikannya untuk perbaikan garansi untuk layanan resmi Lenovo. Setelah beberapa minggu saya mendapatkannya kembali. Lokakarya (layanan) memberi tahu saya bahwa mereka mengganti motherboard + layar + baterai. Adapun layar, sepertinya itu benar - layar sentuh sangat membosankan akhir-akhir ini sebelum perbaikan, dan setelah - itu menjadi lebih baik. Namun, crash terus berlanjut,dan garansi berakhir.

Saya terus menggunakan tablet - hal yang baik, mengganti dengan yang lain terlalu mahal. Dan di sini, setelah penginstalan ulang Windows berikutnya, tablet "diperbaiki" - tidak lagi aneh dengan makanan. Dengan bersukacita, saya akan menulis sebuah artikel tentang seberapa pintar saya bagaimana saya menyelesaikan tugas yang sulit ini, di luar layanan resmi. Tetapi dari pengalaman saya sendiri, saya tahu bahwa sangat mudah untuk terbang dalam masalah seperti itu, dan karena itu saya memulihkan semua kondisi sebelumnya di mana ada masalah dengan makanan. Dan kegagalan itu kembali ...

Ya, kadang-kadang mereka kembali


Ada beberapa pertanyaan:

  1. Mengapa gangguan hilang?
  2. Mengapa mereka kembali?
  3. Apa yang terjadi / disensor / di sini?

Biasanya, dalam kasus seperti itu, saya ingat lagu Four Neblondinka â€œ Ada apa? Apa yang terjadi? ".

Ok, mulailah mencari tahu.

Menggunakan metode ilmiah, bertindak dengan metode pengucilan (pengurangan ("deduksi," sebagai penerjemah yang diterjemahkan secara mental)) mulai membandingkan dua keadaan: "ketika itu tidak berhasil" versus "ketika itu bekerja". Pencarian sederhana menghasilkan penemuan yang luar biasa: bekerja dengan titik akses baru, tidak bekerja - dengan yang lama. Penelitiannya panjang, dan berhasil memperbarui peralatan selama percobaan.

Secara umum, bagaimana ini terjadi ?! Yah itu terjadi.

Poinnya berbeda: yang lama di {802.11n, 2.4 GHz}, yang baru di {802.11ac, 2.4 + 5 GHz}.

Tablet ini mendukung {802.11n, 2,4 + 5 GHz}.

Studi lebih lanjut memberi: tablet bekerja dengan baik pada 5 GHz, gagal pada 2,4 GHz, terlepas dari titik akses.

Untuk verifikasi, saya membeli kartu jaringan Wi-Fi USB di {802.11ac, 2,4 + 5 GHz}. Bagaimanapun, tablet bekerja dengan baik dengannya.

Lagi: bagaimana ini terjadi ?! Yah itu terjadi.

Menyadari apa yang sedang terjadi, saya mulai menggali lebih jauh. Kemudian saya perhatikan bahwa jika terjadi kegagalan pada "Windows Device Manager", " Perangkat HID " tertentuÂ»Ikon peringatan muncul. Dari nama perangkat, tidak ada yang jelas. Tetapi ada trik "peretas": di "Properti Perangkat" pada tab "Informasi", Anda dapat melihat beberapa baris yang menggambarkannya. Saya tertarik pada baris "Path to the instance device" yang berisi "ITE8568". Setelah mencari, saya menemukan bahwa ITE8568 sangat mirip dengan pengontrol daya (mungkin) untuk tablet dan laptop berdaya rendah, ITE IT8568 (Power IC Chip Chipset).

Alasan kegagalan


Saat menggunakan tablet dengan jaringan Wi-Fi melalui adaptor bawaan pada frekuensi 2,4 GHz, chip daya "ITE IT8568" mogok dan macet.

Apa yang harus dilakukan selanjutnya?


Anda dapat mengatasi gangguan tanpa menggunakan jaringan Wi-Fi sama sekali, atau menggunakan adaptor Wi-Fi eksternal dengan menghubungkannya melalui USB. Tetapi bekerja dengan tablet menjadi terlalu tidak nyaman. Dengan USB Tipe A, Anda dapat menggunakan adaptor Wi-Fi kecil, tetapi tablet hanya memiliki microUSB, dan saat ini hanya memberikan hingga 0,5 A, yang tidak cukup untuk 802.11ac (atau modem seluler 4G).

Anda dapat menggunakan adaptor Wi-Fi eksternal hanya pada 5 GHz, tetapi kemudian

  1. Rentang Wi-Fi berkurang
  2. Tidak dapat menggunakan jalur akses 2,4 GHz
  3. Tidak mungkin untuk terhubung dengan banyak ponsel cerdas dan modem seluler yang beroperasi pada frekuensi 2,4 GHz tanpa 5 GHz, di antaranya ada banyak, termasuk Saya sudah.

Karena itu, saya lebih suka mengembalikan kesepakatan.

Hubungi Lenovo


Saya membeli tablet Lenovo MiiX 2 10 yang tidak pernah berfungsi dengan baik, tetapi macet. Saya menemukan penyebab kegagalan, dan ini adalah kesalahan perangkat keras. Saya mengerti bahwa semua orang memiliki kesalahan, sejauh ini (saya sejauh ini) saya tidak benar-benar bersumpah. Tetapi Anda harus memperbaikinya. Saya mengusulkan untuk mendiskusikan kemungkinan solusi untuk masalah ini. Salah satunya adalah mengembalikan ke kondisi semula sebanyak mungkin: Saya memberi Anda tablet yang gagal, Anda mengembalikan uang yang saya bayarkan.

Ini dapat dilengkapi dengan klaim untuk kompensasi untuk kerusakan non-uang, biaya untuk penelitian, kerugian dari kegagalan dan hal-hal lainnya - Saya harap para pembaca Habr akan memberi tahu saya apa yang bisa diperoleh. Karena jika hal itu terjadi dan sedang terjadi di Rusia, maka Anda mungkin tidak harus mengandalkan kompensasi yang besar.

Mungkin untuk menyelesaikan masalah dengan menggantinya dengan model yang lebih baru, tetapi saya harus meninggalkan Windows 10 karena perubahan dalam kebijakan perusahaan Microsoft mengenai data pengguna (sebut saja itu), dan itu tidak jauh dari akhir dukungan untuk Windows 8.1. T.ch. Saya harus beralih ke Linux (dan sedikit ke FreeBSD).
