Unity + Git =フレンドシップ：パート1紳士のセット


gitのバージョン管理システムは、長い間、開発の世界で事実上の標準になっていますが、ほとんどのUnity開発者にとって、gitで効果的に使用できないUnityの機能に関連する多くの問題があることは明らかです。

一般的な問題のリストは次のとおりです。

  1. 不要なファイルがリポジトリに入る、またはその逆が正しいファイルを取得しない
  2. 多くの大きなファイルはリポジトリのサイズを膨らませる
  3. yaml Unityファイルのマージに関する問題
  4. ファイル自体またはメタのみがリポジトリに追加されます
  5. プロジェクトに空のフォルダがあります
  6. バージョンとビルドの自動番号付けの複雑さ
  7. 複数のプロジェクト間でコードを使用することの不便さ

連載の記事で、gitとUnityの併用に関連するこれらの問題の解決策について読むことができます。

この記事では、最初の3つの問題の解決策について説明します。

それぞれの問題を解決するための方法を段階的に説明してみましょう。

プロジェクトのリポジトリを作成した後に最初に行う必要があるのは、例外を構成することです。これについて詳しく説明することはしませんが、良い例を示します。

唯一の注意は、最後にいくつかの例外を追加する必要があることです
！*。Dll-プラグインまたはサードパーティのアセットを使用する場合、dllをリポジトリに保存する必要があり、Windowsではgitはデフォルトでdllを無視します。
！*。obj-この形式のモデルを使用する場合、Windowsでは、objファイルはデフォルトで無視できます

filter=lfs diff=lfs merge=lfs-これらは、gitがこれらのタイプのファイルにlfsを使用するようにする魔法の言葉です。 -textは、ファイルがバイナリであり、マージする必要がないことを意味します。

プロジェクトで他の大きなバイナリファイルを使用している場合は、後でファイルストレージのタイプを変更する（lfsに転送する、またはそこから削除する）ことにより、ファイルをここに追加することは非常に困難になります。

次のステップは、複雑なマージで状況を少し改善することです。
UnityにはUnityYAMLMergeユーティリティが含まれています。これにより、yamlファイルを効率的にマージできます。ファイルに.gitattributes数行を追加します。

*.cs diff=csharp text
*.cginc text
*.shader text

*.mat merge=unityyamlmerge
*.anim merge=unityyamlmerge
*.unity merge=unityyamlmerge
*.prefab merge=unityyamlmerge
*.physicsMaterial2D merge=unityyamlmerge
*.physicsMaterial merge=unityyamlmerge
*.asset merge=unityyamlmerge
*.meta merge=unityyamlmerge
*.controller merge=unityyamlmerge

私たちが何をしたかを説明しましょう：
.csファイルの場合、テキストはC＃コードであると提案されました。
cgincおよびシェーダーファイルの場合も、
ほとんどのUnity yamlファイルのテキスト表現を選択し、カスタムマージドライバーunityyamlmergeを選択しました。

また、次のコードを任意の.gitconfigに追加します。ほとんどの場合、.git / configパスにあるローカルファイルに追加します。リポジトリルート：

[merge "unityyamlmerge"]
	name = Unity SmartMerge (UnityYamlMerge)
	driver = \"{    Unity}/Editor/Data/Tools/UnityYAMLMerge.exe\" merge -h -p --force --fallback none %O %B %A %A
	recursive = binary

-pフラグは、競合が完全に解決できなかった場合でもUnityYamlMergeにファイルの内容を強制的に変更させ、手動での解決を大幅に簡略化します。たとえば、標準のgitマージメカニズムを使用して、同じシーンが変更された2つのブランチをマージすると、多くの変更が見られます。カスタムドライバーを使用する場合、同じコンポーネントの同じフィールドが変更されていても、競合は正確に1行になります。

便宜上、Unityでプロジェクトを初めて開いたときにunityyamlmergeが自動的にインストールされるようにする小さなスクリプトを作成しました。Assetsフォルダーの任意の場所に配置できます（システムにgitをインストールし、PATH変数に登録する必要があります。つまり、gitコマンドでアクセスできます）。

このクラス
#if UNITY_EDITOR
using UnityEngine;
using UnityEditor;
using System;

namespace GitIntegration
{
    [InitializeOnLoad]
    public class SmartMergeRegistrator
    {
        const string SmartMergeRegistratorEditorPrefsKey = "smart_merge_installed";
        const int Version = 1;
        static string VersionKey = $"{Version}_{Application.unityVersion}";

        public static string ExecuteGitWithParams(string param)
        {
            var processInfo = new System.Diagnostics.ProcessStartInfo("git");

            processInfo.UseShellExecute = false;
            processInfo.WorkingDirectory = Environment.CurrentDirectory;
            processInfo.RedirectStandardOutput = true;
            processInfo.RedirectStandardError = true;
            processInfo.CreateNoWindow = true;

            var process = new System.Diagnostics.Process();
            process.StartInfo = processInfo;
            process.StartInfo.FileName = "git";
            process.StartInfo.Arguments = param;
            process.Start();
            process.WaitForExit();

            if (process.ExitCode != 0)
                throw new Exception(process.StandardError.ReadLine());

            return process.StandardOutput.ReadLine();
        }

        [MenuItem("Tools/Git/SmartMerge registration")]
        static void SmartMergeRegister()
        {
            try
            {
                var UnityYAMLMergePath = EditorApplication.applicationContentsPath + "/Tools" + "/UnityYAMLMerge.exe";
                ExecuteGitWithParams("config merge.unityyamlmerge.name \"Unity SmartMerge (UnityYamlMerge)\"");
                ExecuteGitWithParams($"config merge.unityyamlmerge.driver \"\\\"{UnityYAMLMergePath}\\\" merge -h -p --force --fallback none %O %B %A %A\"");
                ExecuteGitWithParams("config merge.unityyamlmerge.recursive binary");
                EditorPrefs.SetString(SmartMergeRegistratorEditorPrefsKey, VersionKey);
                Debug.Log($"Succesfuly registered UnityYAMLMerge with path {UnityYAMLMergePath}");
            }
            catch (Exception e)
            {
                Debug.Log($"Fail to register UnityYAMLMerge with error: {e}");
            }
        }

        //Unity calls the static constructor when the engine opens
        static SmartMergeRegistrator()
        {
            var instaledVersionKey = EditorPrefs.GetString(SmartMergeRegistratorEditorPrefsKey);
            if (instaledVersionKey != VersionKey)
                SmartMergeRegister();
        }
    }
}
#endif
