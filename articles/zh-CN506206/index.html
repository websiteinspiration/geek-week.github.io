ICQ新：机器人繁殖指南



每次进入Messenger，我们都会遇到机器人，它们的表现形式各不相同。有些人谈论天气，有些人玩汉堡，而另一些人则表示模因。当然，你们中的许多人突然冒出一个念头：“但是我可以制造自己的机器人吗？”不幸的是，这样的想法常常被误解成对如何制造机器人的误解。为此，您可能需要成为一名出色的IT专家并了解数百万种技术？并不是的。今天，我们将尝试证明创建机器人程序是一个简单且可理解的过程。让我们分析创建机器人的整个周期，从从Messenger获取必要的数据到编写代码并将其在服务器上启动。

前一段时间，该机器人平台在ICQ中进行了重大更新。她变得更加友好，可理解和舒适。使用开发人员的Python库，我们将创建我们的第一个bot。

第一件事


首先，您需要在ICQ中注册。可以通过用于手机，计算机的应用程序来完成此操作，也可以直接从Web版本的浏览器中完成。

注册后，您可以开始在系统中建立自己的机器人：

  1. 寻找 元机器人 在ICQ上。
  2. 给他写一个团队/newbot。
  3. 为您的新机器人命名（应该以bot结尾）。

之后，该元机器人将发送您的机器人数据：

  • botId：唯一的漫游器号；
  • nick：要搜索的机器人名称；
  • token：用于向服务器发送网络请求的令牌。

就是这样，机器人已创建。在搜索中可以找到他，给他写一条消息。现在，我们需要使机器人活动并做出响应。

在https://icq.com/botapi/#/页面上，有API方法的完整说明。它们用于与服务器交互。让我们更详细地研究其中的一些。

基本方法是/events/get。它用于接收新的机器人事件。例如，如果有人编写了一个漫游器，则在提示时将给出此事件/events/get。让我们向机器人发送一条消息，并检查相应事件的外观。例如，这可以在浏览器中完成。为此，请访问https://api.icq.net/bot/v1/events/get?token=您的令牌＆pollTime = 1＆lastEventId = 0。参数pollTime负责服务器保留请求的时间长度。例如，如果输入值60，则服务器将等待bot事件60秒，如果在此期间没有事件，则服务器将返回空事件数组。lastEventId负责最后处理的事件。换句话说，值小于传输值的事件将被裁剪。

我们在写！


举例来说，我们将编写一个游戏机器人，以测试简单的数学知识。他将像这样工作：一个人向机器人写一条消息，自我介绍并建议玩游戏。在用户同意后，机器人会生成一个简单的数学示例和4个选项供您选择。当用户单击带有所选答案的按钮时，漫游器会回答是对还是错。

因此，机器人必须首先从服务器接收事件。为此，除了导入所有库之外，我们还需要创建一个类对象Bot并开始接收更新：

import logging
from bot.bot import Bot


logging.basicConfig(format='%(asctime)s %(levelname)s %(message)s',
                    datefmt='%Y.%m.%d %I:%M:%S %p', level=logging.DEBUG)

TOKEN = "" #your token here

def main():
    bot = Bot(token=TOKEN)
    bot.start_polling()


if __name__ == '__main__':
    main()

在这里，我们导入日志记录库并运行它。我们还导入了ICQ机器人库，设置了必要的参数token，创建了一个类对象，Bot并开始从服务器接收更新（bot.start_polling()）。

现在，我们的机器人可以接收事件，但是无法处理它们。我们对机器人进行了补充，以便它可以响应消息。为此，添加必要的导入：

import json
from bot.handler import MessageHandler

json需要格式化负责按钮的消息部分。MessageHandler需要向处理程序代码中添加新消息。

接下来，我们需要一个函数来为响应编写一条消息：

def startup(bot, event):
    default_markup = [
        [{"text": "!", "callbackData": "start"}]
        ]

    first_message_text = "!  -  . ?"
    bot.send_text(chat_id=event.from_chat,
         text=first_message_text,
         inline_keyboard_markup=json.dumps(default_markup))

它接收创建的机器人的输入和新消息的事件。default_markup-按钮矩阵（“列表”对象，由按钮字符串数组组成。在我们的情况下，这是唯一的“ 开始”按钮！）。first_message_text-一个字符串，其文本将发送给用户。该方法send_text接受：

  • chat_id （正在处理的事件的字段）-接收消息的聊天的标识符，
  • 要发送的消息文字
  • 和inline_keyboard_markup-我们的按钮。

现在剩下的是描述接收到消息时对此函数的调用。为此，该库具有特殊的设计：

bot.dispatcher.add_handler(MessageHandler(callback=startup))

它使您可以在收到新消息时自动startup()使用必要的参数调用函数。该处理程序必须放在函数中main()。我们得到了这样的代码，当您的漫游器收到任何消息时，它可以让您发送启动消息：

import logging
import json
from bot.bot import Bot
from bot.handler import MessageHandler

logging.basicConfig(format='%(asctime)s %(levelname)s %(message)s',
                    datefmt='%Y.%m.%d %I:%M:%S %p', level=logging.DEBUG)

TOKEN = «» #your token here

def startup(bot, event):
    default_markup = [
        [{«text»: «!», «callbackData»: «start»}]
        ]
    first_message_text = «!  —  . ?»
    bot.send_text(chat_id=event.from_chat,
        text=first_message_text,
        inline_keyboard_markup=json.dumps(default_markup))

def main():
    bot = Bot(token=TOKEN)
    bot.dispatcher.add_handler(MessageHandler(callback=startup))

更详细地考虑我们在消息中发送的按钮：

[ [{«text»: «!», «callbackData»: «start»}]]

除了写在按钮上的文字外，消息还有一个字段callbackData。单击按钮后，服务器将生成按钮单击事件。在其中，他将该字段作为按下按钮的标识符传递。也就是说，通过从服务器收到的这一行，我们可以判断用户单击了哪个按钮。因此，我们还需要将按钮单击处理逻辑添加到机器人。为此，我们将再次需要导入必要的方法：

from bot.handler import BotButtonCommandHandler
from bot.filter import Filter

第一行是按钮单击事件处理程序，第二行是有助于理解按下哪个按钮的过滤器。我们还需要描述按钮单击响应的逻辑：

def start(bot, event):
    question = ''
    operands = ["+", "-", "*"]
    operations_count = randrange(3)+2
    for i in range(operations_count):
        question += str(randrange(9))
        question += choice(operands)
    question = question[:-1]
    answer = eval(question)
    buttons = [
        [{'text': str(answer), 'callbackData': "right"},
         {'text': str(answer+1), 'callbackData': "wrong"}],
        [{'text': str(answer-1), 'callbackData': "wrong"},
         {'text': str(answer+2), 'callbackData': "wrong"}]]
    shuffle(buttons[0])
    shuffle(buttons[1])
    shuffle(buttons)
    bot.answer_callback_query(
        query_id=event.data['queryId'],
        text=' ')
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text=question,
        inline_keyboard_markup=json.dumps(buttons))

与消息类似，函数接受与输入相同的参数。然后生成示例和带有答案的按钮数组。答案之一将是正确的。在此基础上，答案将有所不同callbackData。最后，您必须answer_callback_query使用参数来绝对调用该方法：

  • query_id -按下按钮时的唯一编号，
  • 和text-将在工具提示中显示，当你点击按钮上的文本。

通过调用此方法，我们将向用户显示该机器人已接受单击按钮。否则，下载指示器将在按钮上显示一段时间。使用该方法，send_text您需要发送带有问题和答案按钮的消息本身。

让我们立即添加用于处理正确和错误答案的函数：

def right(bot, event):
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text='!')
    start(bot, event)


def wrong(bot, event):
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text=':(')
    start(bot, event)

当您单击正确和不正确的答案时，将调用这些功能。在带有正确答案的消息之后，用户将收到一个新问题。

剩下的就是添加按钮单击处理程序：

    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=start, filters=Filter.callback_data(«start»)))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=wrong, filters=Filter.callback_data(«wrong»)))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=right, filters=Filter.callback_data(«right»)))

这三个处理程序中的每一个都包含一个过滤器callbackData（例如filters=Filter.callback_data(«start»)）和一个要调用的函数。

这样就完成了机器人程序的编程。产生的代码如下所示：

import logging
import json
from bot.bot import Bot
from bot.handler import BotButtonCommandHandler
from bot.handler import MessageHandler
from bot.filter import Filter
from random import randrange, choice, random, shuffle

logging.basicConfig(format='%(asctime)s %(levelname)s %(message)s',
        datefmt='%Y.%m.%d %I:%M:%S %p', level=logging.DEBUG)

TOKEN = "" #your token here


def startup(bot, event):
    default_markup = [
        [{"text": "!", "callbackData": "start"}]
        ]

    first_message_text = "!  -  . ?"
    bot.send_text(chat_id=event.from_chat,
        text=first_message_text,
        inline_keyboard_markup=json.dumps(default_markup))


def start(bot, event):
    question = ''
    operands = ["+", "-", "*"]
    operations_count = randrange(3)+2
    for i in range(operations_count):
        question += str(randrange(9))
        question += choice(operands)
    question = question[:-1]
    answer = eval(question)
    buttons = [
        [{'text': str(answer), 'callbackData': "right"},
         {'text': str(answer+1), 'callbackData': "wrong"}],
        [{'text': str(answer-1), 'callbackData': "wrong"},
         {'text': str(answer+2), 'callbackData': "wrong"}]]
    shuffle(buttons[0])
    shuffle(buttons[1])
    shuffle(buttons)
    bot.answer_callback_query(
        query_id=event.data['queryId'],
        text=' ')
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text=question,
        inline_keyboard_markup=json.dumps(buttons))


def right(bot, event):
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text='!')
    start(bot, event)


def wrong(bot, event):
    bot.send_text(chat_id=event.data['message']['chat']['chatId'],
        text=':(')
    start(bot, event)


def main():
    bot = Bot(token=TOKEN)
    bot.dispatcher.add_handler(MessageHandler(callback=startup))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=start, filters=Filter.callback_data("start")))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=wrong, filters=Filter.callback_data("wrong")))
    bot.dispatcher.add_handler(BotButtonCommandHandler(
        callback=right, filters=Filter.callback_data("right")))
    bot.start_polling()


if __name__ == '__main__':
    main()

在哪里以及如何运行？


在远程服务器上启动机器人最方便。有许多服务为虚拟服务器提供了足够的访问权来安装程序。某些服务提供免费试用期。由于我们是Mail.ru，因此我们将使用Mail.ru云解决方案示例对其进行分析。注册过程很简单，可以快速访问您的服务器，并且免费试用，而无需银行卡或其他麻烦。

  1. 我们在https://mcs.mail.ru/上注册一个帐户。
  2. 创建虚拟机的实例。所有设置保持不变，就足够了。
  3. 该服务将提供下载访问密钥。下载。
  4. 现在我们进入虚拟机：下载密钥的路径ssh -i ./xxx.pem user@ip

    在哪里./xxx.pem。
  5. 之后，我们需要安装必要的程序和软件包：

    $ sudo curl https://raw.githubusercontent.com/dvershinin/apt-get-centos/master/apt-get.sh -o /usr/local/bin/apt-get
$ chmod 0755 /usr/local/bin/apt-get
$ sudo apt-get install python3
$ curl https://bootstrap.pypa.io/get-pip.py -o get-pip.py
$ python3 get-pip.py
$ pip3 install --upgrade mailru-im-bot

    在这里，我们安装了Python和所有必要的库以使我们的机器人能够正常工作。
  6. 仍然需要将Bot复制到服务器并运行它。在带有脚本的文件夹中，执行：scp -i ./xxx.pem ./path/to_bot.py user@ip:~

    • ./xxx.pem -密钥的路径；
    • ./path/to_bot.py -机器人文件的路径；
    • user@ip:~ -将机器人复制到的登录名，地址和路径。

    之后，带有bot的文件将位于远程服务器上的主目录中。
  7. 我们返回到服务器并运行：，python3 ~/file_name.py其中file_name.py带有bot的文件的名称在哪里。
