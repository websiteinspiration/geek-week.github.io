PHPのホワイトスペース難読化ツール

JavaScriptでの難読化に関する記事（つまり、記事の第3部）を読んだ後、PHPの同様の難読化ツールに興味を持ちました。

記事で説明されている難読化ツールにより、JavaScriptコードは（一見すると）タブとスペースのコードで構成される不可視になりました。

一番下の行は、ソースシステムの各シンボルをバイナリシステムに変換されたこのシンボルのコードとして表すことです。次に、1をスペースに、0をタブに置き換えます（もちろん、その逆も可能です）。

このアルゴリズムをPHPに実装すると、JavaScriptのように、すべてが簡単になります。

難読化コード：

function obfuscate($code)
{
    $len = strlen($code);
    $obf = '';
   
    for ($i = 0; $i < $len; $i++) {
        //        i- 
        $bin = decbin(ord($code[$i]));
        //      ,      0
        $bin = ($binLen = strlen($bin) > 7)
                 ? $bin
                 : implode('', array_fill(0, 8 - strlen($bin), '0')) . $bin;
        //    1  ,  0        ,     
        $obf .= str_replace(array('1', '0'), array(chr(9), chr(32)), $bin);
    }
   
    return $obf;
}

ここではすべてが正常であるようです（詳細に触れない場合）。次に、難読化されたコードを呼び出す例を示します。

Deobfuscator（「実行者」）

function include_o($file)
{
    $file = trim($file);
    //    
    if ( empty($file)
     ||  !is_readable($file)
    ) {
        throw new Exception("Filename is empty or file isn't readable");
    }
   
    $string = file_get_contents($file);
    $len    = strlen($string);
    $out    = '';
   
    for ($i = 0; $i < $len; $i++) {
        /*
         *  8      ,
         *              
         */
        $out .= chr(bindec(str_replace(array(chr(9), chr(32)), array('1', '0'), substr($string, $i, 8))));
        $i += 7;
    }
   
    if (!empty($out)) {
        eval($out);
    }
}

関数によって返された値は「混合」されるようです（私が正しく理解していれば、無効-どのようにその一部になりますか？）。しかし、これは詳細に入る場合です。

結論

それはかなり許容できることが判明したように私には思われますが、このタイプはまだ機能する（実際に使用される暗黙の）難読化機能を要求することはできません（私は知っています、平凡なことが言われています）。それはむしろ面白くするために行われました、少なくとも私は結果を見て興味がありました。
