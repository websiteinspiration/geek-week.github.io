WinFormsアプリケーションの外観をカスタマイズする一般的な方法（FAQ.Netの例を使用）

前書き


私のFAQ.Netアプリケーション（Windowsノートプログラム）に関する最初の記事を投稿した後、ノートプログラムをさらに開発したい最初のユーザーが現れました。

ユーザーの関心を高めるために、色やフォントを調整して魅力を高めるというアイデアが生まれました。

アプリケーションの外観をカスタマイズするための要件


  1. 個々のコンポーネントのプロパティをXMLファイルに保存およびロードする
  2. ユーザーは外観をカスタマイズするための簡単なツールを持っている必要があります
  3. カスタムコンポーネントのリストは非常に単純に（1行のコードで）展開する必要があります
  4. プロパティの値を変更すると、ユーザーはフォーム上の変更をすぐに確認できます
  5. すべての変更を元に戻すことができるキャンセルボタンがあるはずです

問題を解決するにはそれが必要です


  1. PropertyGridExコンポーネントを使用して、コンポーネントのプロパティを表示および変更する
  2. プロパティをXMLに格納するのに便利なCustomDesignControlクラスを作成する
  3. 外観を変更するMainFormフォームを作成します
  4. MainFormの外観をカスタマイズするAppSettingsFormフォームを作成します
  5. XMLでの作業を容易にするために、SettingsXmlクラスを作成します

見た目の変化をキャンセルする機能


外観の変更の廃止（要件の第5項）を実装するために、メモリ内に作成されるコンポーネントのプロパティを複製し、表示されるコンポーネントを置き換える一般的な方法を見つけることにしました。

オブジェクトの複製：

public static class ControlExtensions
  {
    public static T Clone<T>(T controlToClone)
        where T : Control
    {
      PropertyInfo[] controlProperties = typeof(T).GetProperties(BindingFlags.Public | BindingFlags.Instance);

      T instance = Activator.CreateInstance<T>();

      RichTextBox rtb = new RichTextBox();
      foreach (PropertyInfo propInfo in controlProperties)
      {
        if (propInfo.Name == "Parent")
          continue;
        if (propInfo.CanWrite)
        {
          if (propInfo.Name != "WindowTarget")
          {
              propInfo.SetValue(instance, propInfo.GetValue(controlToClone, null), null);
          }
        }
      }
      return instance;
    }

コードをコピーし、保存する必要のない不要なプロパティを削除して、色とフォントを残しました。
コードの動作を確認した後、このメソッドの欠点がすぐに明らかになりました。

  1. コントロールをControlクラスにキャストすると、プロパティの半分が失われる可能性があります（たとえば、DataGridViewコンポーネントはControlクラスから継承されますが、独自の追加のプロパティセットがあります）。
  2. コンポーネントを複製するとき、一部のプロパティ（たとえば、DataGridViewセル内のフォントの色）は複製されませんが、複製されたコンポーネントへのリンクが作成されます（これは、プロパティが内部プロパティで使用され、何らかの方法で別の方法で処理する必要があるためです）。

プロパティのクローンを正確に実現するために、彼は2番目の点で実験を開始しました。
これを行うには、new演算子を使用して必要なタイプのオブジェクトを作成します。

DataGridView dgvClone = new DataGridView();

役に立たなかったため、コンポーネントは変更を元に戻すことができませんでした。

次に、new演算子を使用してプロパティ自体を作成し、SetValueメソッドを使用して値を設定しました。

propInfo.SetValue(_controls[i], propInfo.GetValue(_controlsBeforeChange[i], null), null);

そして、この方法は結果を与えませんでした。プロパティ値が単一のコンポーネントで変更されると、複製されたコンポーネントで自動的に更新されます。

もう他の方法を探す必要はなく、作業中のドキュメントを保存したり、再起動をキャンセルしたりできるようにアプリケーションを再起動するだけで、変更を元に戻すことができました。これにより、オブジェクトのクローンを作成する必要がないため、アプリケーションのメモリを大幅に節約できました。

外観をカスタマイズしたアプリケーションの例


念のため、アプリケーションを再起動せずに外観の変更をキャンセルする機能を誰もが確認して変更できるように、コメントで問題についてお知らせできる既製のコードを用意しました。
[自動テスト]ボタンをクリックすると、DataGridViewコンポーネントの背景が灰色になり、セルのフォントが小さくなることを確認する必要があります。
