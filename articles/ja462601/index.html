AWSでのWindowsサーバーのバックアップ

AWSにはWindowsサーバーがあり、タスクはバックアップを構成することです。スナップショットを使用できますが、データの整合性に問題があります。また、毎週と毎月のスナップショットを保持したいのですが、スナップショットのライフサイクルはこれを提供していません。新しいAWSバックアップサービスも、スナップショットを完了する方法をまだ知らないか、方法がわかりません。ええと、私はこれらすべてが私の参加なしで可能な限り機能することを望んでいます。

必要なタスクを達成するために

  1. AWSを実行しているWindows Server 2008 R2以降
  2. SSMエージェントバージョン2.2.58.0以降
  3. AWS Tools for Windows PowerShell 3.3.48.0以降
  4. AWSシステムマネージャー
  5. わたし
  6. SNS
  7. ラムダ

次に、このロールを目的のサーバーに割り当てます。

すべてのssmがこれらのサーバーを「管理」できるようになりました。

次に、サーバーにAWSVssComponentsを配置する必要があります。これを行うには、[コマンドの実行]を選択して[コマンドの実行]をクリックし、AWS-ConfigureAWSPackageを探します。

コマンドパラメータで、[インストール]、[名前]-[AwsVssComponents]、最新バージョンを選択します。
ターゲットでは、バックアップするシステムを選択します。

実行をクリックします。

完了したら、SSMコンソールからバックアップを作成できます。

[コマンドの実行]を選択し、AWSEC2-CreateVssSnapshotを探します。サーバーをターゲットにインストールします。除外するブートボリューム、コピーのみ、ライターなしのオプションを選択します。

実行をクリックします。スナップショットを作成する必要があります。

バックアップ通知の場合は、SNSトピックを作成します。そして、それを購読してください。メール通知を利用しています。

プロセスを自動化するために、SSMメンテナンスウィンドウを使用します。

[メンテナンスウィンドウの作成]をクリックします。名前を入力し、スケジュールを好きなように入力します。

作成されたメンテナンスウィンドウに移動し、RUNコマンドの登録タスクを追加します。パラメータを入力します。タグでは、バックアップのタイプを記述します（TAGキー= SnapshotType、値=）。 3つの可能なパラメーターがあります：日、週、月、それに応じて3つのメンテナンスウィンドウ。 [SNS通知を有効にする]を設定し、SNSとトピックの役割を示します。

すべてのスナップショットがスケジュールに従って作成されます。

そして、私たちはこの方針を新しい役割に絞っています。

lambdaに移動して、新しいPython関数を作成します。

import datetime
import sys
import boto3


def get_volume_snapshots(client, volume_id, SnapshotType):
        args = {
                "Filters": [
                        { "Name": "volume-id", "Values": [volume_id] },
                        { "Name": "status", "Values": ["completed"] },
                        { "Name": "tag-key", "Values": ["SnapshotType"]},
                        { "Name": "tag-value", "Values": [SnapshotType]},
                ],
                "OwnerIds": ["self"]

        }
        snapshots = []
        while True:
                resp = client.describe_snapshots(**args)
                snapshots += resp.get("Snapshots", [])
                if "NextToken" in resp:
                        args["NextToken"] = resp["NextToken"]
                else:
                        break

        return snapshots

  
def delete_snapshot(client, snapshot_id):
        wait_period = 5
        retries = 5
        while True:
                try:
                        client.delete_snapshot(SnapshotId=snapshot_id)
                        return True
                except Exception as ex:
                        # As the list of snapshot is eventually consistent old snapshots might appear in listed snapshots
                        if getattr(ex, "response", {}).get("Error", {}).get("Code", "") == "'InvalidSnapshot.NotFound":
                                return False
                        # Throttling might occur when deleting snapshots too fast
                        if "throttling" in ex.message.lower():
                                retries -= 1
                                if retries == 0:
                                        raise ex
                                time.sleep(wait_period)
                                wait_period = min(wait_period + 10 , 30)
                                continue
                        raise ex

def lambda_handler(event, context):
        retentions = {"Day": 5, "Week": 3, "Month": 2}
        client = boto3.client("ec2")
        vols = client.describe_volumes()
        snapshots_deleted = []
        for vol in vols['Volumes']:
                vol_id = vol['VolumeId']
                for SnapshotType, retention_count in retentions.items():

                        snapshots_for_volume = sorted(get_volume_snapshots(client, vol_id, SnapshotType), key=lambda s: s["StartTime"], reverse=True)
                        snapshots_to_delete = []
                        if retention_count > 0:
                                snapshots_to_delete = [b["SnapshotId"] for b in snapshots_for_volume[retention_count:]]
                        for snapshot_id in snapshots_to_delete:
                                if delete_snapshot(client, snapshot_id):
                                        snapshots_deleted.append(snapshot_id)
        return {
                "DeletedSnapshots": snapshots_deleted
        }

上記で作成したロールを使用します。トリガーとして、CloudWatchイベントを使用します。
この関数は、すべてのボリュームを通過し、SnapshotTypeタグで完了したスナップショットのすべてのボリュームを検索し、スナップショットの保持期間が長いすべてのスナップショットを削除します。最新のスナップショットを毎日5つ、毎週3つ、毎月2つ取得しています。
