追加のマネージャー設定


便宜上、追加のパッケージを提供します。

$ sudo yum install bash-completion vim

自動補完を有効にするには、bash-completionコマンドをbashに切り替える必要があります。


追加のDNS名の追加


これは、代替名（CNAME、エイリアス、またはドメインサフィックスのない短い名前）でマネージャーに接続する必要がある場合に必要です。セキュリティ上の理由から、マネージャーは許可された名前のリストを使用した接続のみを許可します。

構成ファイルを作成します。

$ sudo vim /etc/ovirt-engine/engine.conf.d/99-custom-sso-setup.conf

次のコンテンツ：

SSO_ALTERNATE_ENGINE_FQDNS="ovirt.example.com some.alias.example.com ovirt"

そしてマネージャーを再起動します：

$ sudo systemctl restart ovirt-engine

電源管理のセットアップ


たとえば、エンジンがホストからの応答を長期間受信できない場合に、マシンのハードウェアリセットなどを実行できます。Fence Agentを介して実装されます。

コンピューティング->ホスト-> ホスト -編集->電源管理、次に「電源管理を有効にする」を有効にして、エージェントを追加します-「フェンスエージェントの追加」-> +。

タイプ（たとえば、iLO5の場合はilo4を指定する必要があります）、ipmiインターフェイスの名前/アドレス、およびユーザー名/パスワードを示します。別のユーザー（oVirt-PMなど）を作成し、iLOの場合はそのユーザーに特権を与えることをお勧めします。

  • ログインする
  • リモートコンソール
  • 仮想電源とリセット
  • 仮想メディア
  • iLO設定の構成
  • ユーザーアカウントの管理

理由は問わないでください。経験的に選択されています。コンソールフェンシングエージェントには、より少ない権限のセットが必要です。

アクセスコントロールリストを設定する場合、エージェントはエンジンではなく「隣接」ホスト（いわゆる電源管理プロキシ）で実行されることに注意してください。つまり、クラスターにノードが1つしかない場合、電源管理は機能しません。


SSLセットアップ


完全な公式の手順は、ドキュメントの付録D：oVirtとSSL-oVirt EngineのSSL / TLS証明書の置き換えにあります。

証明書は、当社のCAまたは外部の商用証明機関のいずれかからのものです。

重要な注意：証明書はマネージャーに接続するように設計されており、エンジンとノード間の相互作用に影響を与えません-エンジンが発行した自己署名証明書を使用します。

要件：

  • ルートCAへのチェーン全体（最初の従属発行者から最後のルートまで）を含むPEM形式の発行CAの証明書。
  • 発行CAによって発行されたApache証明書（CA証明書チェーン全体でも補足）
  • Apacheの秘密鍵、パスワードなし。

公開証明機関がsubca.example.comと呼ばれるCentOSを実行していて、要求、キー、および証明書が/ etc / pki / tls /ディレクトリにあるとします。

バックアップを実行し、一時ディレクトリを作成します。

$ sudo cp /etc/pki/ovirt-engine/keys/apache.key.nopass /etc/pki/ovirt-engine/keys/apache.key.nopass.`date +%F`
$ sudo cp /etc/pki/ovirt-engine/certs/apache.cer /etc/pki/ovirt-engine/certs/apache.cer.`date +%F`
$ sudo mkdir /opt/certs
$ sudo chown mgmt.mgmt /opt/certs

証明書をダウンロードし、ワークステーションから実行するか、別の便利な方法で転送します。

[myuser@mydesktop] $ scp -3 causer@subca.example.com:/etc/pki/tls/cachain.pem mgmt@ovirt.example.com:/opt/certs
[myuser@mydesktop] $ scp -3 causer@subca.example.com:/etc/pki/tls/private/ovirt.key mgmt@ovirt.example.com:/opt/certs
[myuser@mydesktop] $ scp -3 causer@subca.example.com/etc/pki/tls/certs/ovirt.crt mgmt@ovirt.example.com:/opt/certs

その結果、3つのファイルすべてが表示されます。

$ ls /opt/certs
cachain.pem  ovirt.crt  ovirt.key

証明書のインストール


ファイルをコピーして信頼リストを更新します。

$ sudo cp /opt/certs/cachain.pem /etc/pki/ca-trust/source/anchors
$ sudo update-ca-trust
$ sudo rm /etc/pki/ovirt-engine/apache-ca.pem
$ sudo cp /opt/certs/cachain.pem /etc/pki/ovirt-engine/apache-ca.pem
$ sudo cp /opt/certs/ovirt03.key /etc/pki/ovirt-engine/keys/apache.key.nopass
$ sudo cp /opt/certs/ovirt03.crt /etc/pki/ovirt-engine/certs/apache.cer
$ sudo systemctl restart httpd.service

構成ファイルを追加/更新します。

$ sudo vim /etc/ovirt-engine/engine.conf.d/99-custom-truststore.conf

ENGINE_HTTPS_PKI_TRUST_STORE="/etc/pki/java/cacerts"
ENGINE_HTTPS_PKI_TRUST_STORE_PASSWORD=""

$ sudo vim /etc/ovirt-engine/ovirt-websocket-proxy.conf.d/10-setup.conf

SSL_CERTIFICATE=/etc/pki/ovirt-engine/certs/apache.cer
SSL_KEY=/etc/pki/ovirt-engine/keys/apache.key.nopass

$ sudo vim /etc/ovirt-imageio-proxy/ovirt-imageio-proxy.conf

# Key file for SSL connections
ssl_key_file = /etc/pki/ovirt-engine/keys/apache.key.nopass
# Certificate file for SSL connections
ssl_cert_file = /etc/pki/ovirt-engine/certs/apache.cer

次に、影響を受けるすべてのサービスを再起動します。

$ sudo systemctl restart ovirt-provider-ovn.service
$ sudo systemctl restart ovirt-imageio-proxy
$ sudo systemctl restart ovirt-websocket-proxy
$ sudo systemctl restart ovirt-engine.service

できた！マネージャーに接続し、接続が署名付きSSL証明書によって保護されていることを確認します。


アーカイブ


彼女なしで！このセクションでは、マネージャのアーカイブについて説明します。VMのアーカイブは別の問題です。アーカイブコピーを1日に1回作成し、たとえば、ISOイメージを配置したのと同じシステム（mynfs01.example.com:/exports/ovirt-backup）のNFSに配置します。Engineが実行されているのと同じマシンにアーカイブを保存することはお勧めしません。

autofsをインストールして有効にします。

$ sudo yum install autofs
$ sudo systemctl enable autofs
$ sudo systemctl start autofs

スクリプトを作成します。

$ sudo vim /etc/cron.daily/make.oVirt.backup.sh

次のコンテンツ：

#!/bin/bash

datetime=`date +"%F.%R"`
backupdir="/net/mynfs01.example.com/exports/ovirt-backup"
filename="$backupdir/`hostname --short`.`date +"%F.%R"`"
engine-backup --mode=backup --scope=all --file=$filename.data --log=$filename.log
#uncomment next line for autodelete files older 30 days 
#find $backupdir -type f -mtime +30 -exec rm -f {} \;

ファイルを実行可能にします。

$ sudo chmod a+x /etc/cron.daily/make.oVirt.backup.sh

これで毎晩、マネージャー設定のアーカイブを受け取ります。


ホスト管理インターフェース


Cockpitは、Linuxシステム用の最新の管理インターフェースです。この場合、それはESXi Webインターフェイスに近い役割を果たします。


図。3-パネルの外観。

インストールは非常に簡単です。コックピットパッケージとcockpit-ovirt-dashboardプラグインが必要です。

$ sudo yum install cockpit cockpit-ovirt-dashboard -y

コックピットをオンにする：

$ sudo systemctl enable --now cockpit.socket

ファイアウォールのセットアップ：

sudo firewall-cmd --add-service=cockpit
sudo firewall-cmd --add-service=cockpit --permanent

これで、ホストに接続できます：https：// [ホストIPまたはFQDN]：9090


VLAN


ドキュメントで ネットワークの詳細をご覧ください。多くの可能性がありますが、ここでは仮想ネットワークの接続について説明します。

他のサブネットを接続するには、最初にそれらを構成で説明する必要があります。ネットワーク->ネットワーク->新規、ここでは名前のみが必須フィールドです。マシンがこのネットワークを使用できるようにするVMネットワークチェックボックスがオンになっていて、タグを有効にするには、[ VLANタグ付けを有効にする]を有効にし、VLAN番号を入力して[OK]をクリックします。

次に、コンピュート->ホスト-> kvmNN->ネットワークインターフェース->ホストネットワークのセットアップホストに移動する必要があります。追加されたネットワークを[割り当てられていない論理ネットワーク]の右側から[割り当てられている論理ネットワーク]の左側にドラッグします

図。 5-ネットワークを追加した後。

複数のネットワークをホストに一括で接続するには、ネットワークの作成時にネットワークにラベルを割り当て、ラベルでネットワークを追加すると便利です。

ネットワークが作成された後、ネットワークがクラ​​スターのすべてのノードに追加されるまで、ホストは非稼働状態になります。この動作は、新しいネットワークを作成するときに、[クラスタ]タブの[すべて必要]フラグが原因で発生します。クラスターのすべてのノードでネットワークが不要な場合は、この症状を無効にできます。ホストに追加されたネットワークは、[不要]セクションの右側にあり、特定のホストに接続するかどうかを選択できます。

HPE固有


ほとんどすべてのメーカーが自社製品の使いやすさを向上させるツールを持っています。HPEの例を使用すると、AMS（エージェントレス管理サービス、iLO5のamsd、iLO4のhp-ams）やSSA（スマートストレージアドミニストレーター、ディスクコントローラーの操作）などが役立ちます。

HPEリポジトリを接続
HPE:

$ sudo rpm --import https://downloads.linux.hpe.com/SDR/hpePublicKey2048_key1.pub
$ sudo vim /etc/yum.repos.d/mcp.repo

:

[mcp]
name=Management Component Pack
baseurl=http://downloads.linux.hpe.com/repo/mcp/centos/$releasever/$basearch/current/
enabled=1
gpgkey=file:///etc/pki/rpm-gpg/GPG-KEY-mcp

[spp]
name=Service Pack for ProLiant
baseurl=http://downloads.linux.hpe.com/SDR/repo/spp/RHEL/$releasever/$basearch/current/
enabled=1
gpgkey=file:///etc/pki/rpm-gpg/GPG-KEY-mcp

():

$ sudo yum --disablerepo="*" --enablerepo="mcp" list available
$ yum info amsd

$ sudo yum install amsd ssacli
$ sudo systemctl start amsd
