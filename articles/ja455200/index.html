JavaScriptでの動的インポート

昨年の初めに、JavaScriptへの動的インポートの可能性を導入することが提案されました。誰も知らない場合は、JavaScriptでネイティブにサポートされているのは静的インポートのみで、これには理由がありますが、これは別の議論のトピックです。この提案は実装され、今後のECMAScript更新のリストに含まれています。バージョン63のGoogle Chromeはすでにこの機能をサポートしています。それが何であるか、そしてそれがどこで役に立つかを少しお話します。


動的インポートは静的インポートとほぼ同じ方法で使用されますが、いくつかの重要な違いがあります。


構文的に動的なインポートは、関数呼び出しに似ています：import（ 'path / to / module.js'）;


import（ 'path / to / file.js'）コマンドはPromiseを返します。Promiseは、モジュール自体がプルアップされて直接インストールされた後、そのすべての依存関係とともに、満たされた状態になります。これは、次のように書くことができることを意味します。


import('path/to/module.js')
  .then(module => {
    module.loadPageInto(main);
  })
  .catch(err => {
    main.textContent = err.message;
  });

または次のようにも：


const module = await import('path/to/module.js');

重要な注意：動的インポートは構文であり、import（）関数の呼び出しのように見えますが、関数ではありません。Function.prototypeから継承されていないため、callまたはapplyを介して呼び出すことはできません。
