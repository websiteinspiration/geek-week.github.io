Kubernetesディストリビューションのトップ5



Kubernetes（K8s）は、過去5年間でソフトウェアの操作方法を劇的に変えた強力なプラットフォームです。それは需要が高いため、一部の企業はサービスとしてKubernetesを提供し始めました。 Cloud4Yは、最も興味深いK8sディストリビューションについて知ることを提案しています。

プラットフォームには、製品に多くの革新をもたらす強力なオープンソースコミュニティがあります。ここからいくつかの現在の欠点が出てきます：インストールの難しさ、セットアップの難しさ。多くの点で、KubernetesはLinuxカーネルと似ています。ソフトウェアを実行するための環境を提供しますが、管理ツールは提供しません。

このため、K8sに含まれていないインストーラー、コントロールパネル、アドオンを含むKubernetesディストリビューションが登場しました。ディストリビューションでは、Kubernetesの内部を掘り下げることなく、ネットワークサービスやコンテナランタイムなどのコンポーネントを変更することもできます。最も人気のあるK8ディストリビューションのいくつかと、それらが通常のKubernetesよりも優れていることを確認することをお勧めします。

Docker Kubernetesサービス（DKS）


DockerとK8sの関係は、共生と呼ぶことができます。 Kubernetesは通常、Dockerコンテナーを整理するために使用され、Kubernetes DockerクラスターはDocker Kubernetes Service（DKS）を使用してデプロイおよび管理されます。

DKSは、YAML構成ファイル、Helmダイアグラム、およびDocker Composeを使用して、異なるKubernetes環境間の整合性を作成する責任があります。これにより、開発者はアプリケーションをテストハードウェアクラスターとハイブリッドクラウドの本番クラスターの両方に安全にデプロイできます。 DKSは、役割ベースのアクセス制御（RBAC）、個人用コンテナー、TLSを使用した暗号化ネットワーク、およびマルチテナント環境も追加します。

DKSはDocker Enterpriseの一部ですおよびDocker Desktop Enterprise。すぐに使えるDKSは、セキュリティと暗号化を提供し、同じクラスター内のDocker Swarmと互換的に機能します。

オープンシフト


Red HatのOpenShiftは、KubersのOrigin Community Distribution（略してOKD）に基づくオープンソースプラットフォームであり、それ自体がK8に基づいています。OKDは、新しいWebコンソール、より強力なコマンドライン、マルチテナントサポート、および高可用性クラスターデプロイメントツールを追加することにより、Kubernetesを拡張します。OpenShiftはAnsibleを使用してデプロイメントプロセスを自動化し、ベアサーバーまたはクラウドサービスにインストールできます。

OpenShiftはOKDを受け入れ、それをIstioサービスグリッド、モニタリング用のPrometheus、ロギング用のElastic Stackなどの他のプロジェクトと一緒にパックします。Source-to-Image（S2I）フレームワークが含まれており、再現可能なコンテナを使用してソースコードをパッケージ化してデプロイします。これにより、開発者は独自のイメージを作成せずにコードを実行できます。OpenShiftは、自動更新、オペレーター、Quayコンテナーレジストリなどの機能をTectonicから受け取ります。

OpenShiftには、どこにどのように展開したいかに応じて、いくつかの種類があります。

  • OpenShift Online：完全に管理されたオープンソースのクラウド製品で、迅速なアプリケーションの展開が可能です。
  • OpenShift Dedicated：完全に管理されたプライベートクラウド。AWSでホストされているOpenShiftクラスター。
  • OpenShift Container Platform：独自のインフラストラクチャにデプロイするOpenShiftエンタープライズクラスター。

Kubernetes、OKD、およびOpenShiftの詳細な比較は、OpenShift Webサイトで確認できます。

牧場主


Rancherは、オープンソースパッケージで商用K8sディストリビューションの機能を提供します。 Rancherの人気の理由は、ベアメタル（Rancher Kubernetes Engineを使用）にクラスターを簡単に作成できること、およびAKSやGKEなどのホスティングプロバイダーにあることにあります。 Rancherをデプロイするには、ホストにDockerをインストールし、YAMLでクラスター構成を定義して、rke upを実行するだけです。 Rancherは、Kops、KubeSpray、またはOpenShiftを含む他のディストリビューションによって作成された既存のKubernetesクラスターをインポートすることもできます。

RKEは完全にコンテナベースであり、必要に応じてクラスターを簡単に変更またはスケーリングできます。Rancherは、役割ベースのアクセス制御（RBAC）機能、さまざまな認証システムとの統合、自動バックアップ、自動証明書管理、および多くのプラグインも追加します。

Rancherは、Kubernetesをリメイクするという目標を設定するのではなく、デプロイと使用を容易にします。

Apache Mesosphere Kubernetes Engine


Mesosphere Kubernetes Engine（MKE）は、2018年末に発表された比較的新しいプロジェクトです。公式のK8sランタイムをApache Mesosphereに追加します。重要な機能の1つは、マルチKubernetes高密度メディア（高密度マルチKubernetes、HDMK）の作成です。これにより、複数の分離されたKubernetesクラスターを同じノードに配置できます。MKEは、自己修復クラスターコンポーネント、ワンクリックのスケーリングと更新、Calicoネットワーキング、コンポーネント間の暗号化通信も提供します。
MKEはDC / OS上で実行されるため、既存のDC / OSクラスターへの展開と管理が容易になります。

ピボタルコンテナサービス


Pivotal Container Service（PKS）は、企業や通信事業者でK8を使用するための特別なソリューションです。これにより、このフレームワークのクラスターの展開と操作が大幅に簡素化されます。 PKSは、vSphereを介してデータセンターとGoogle Cloud Platformにデプロイできます。このサービスは最近、Cloud Native Computing FoundationのKubernetes Software Conformance Certification Programによって認定されました。

PKSは、基本クラスターに安定性と信頼性を提供する分散システムとアプリケーションを展開および管理するための一連のツールであるBOSHを使用します。 BOSHは、Kubernetesプロセスを監視および管理するだけでなく、ホストも監視し、仮想マシンレベルでのフェイルオーバーセキュリティ、リカバリ、スケーラビリティを提供します。

BOSHに加えて、PKSにはHarbourと呼ばれるプライベートレジストリサービスも含まれており、コンテナーイメージとHelmダイアグラムの両方、脆弱性スキャン、アクセス制御をサポートしています。PKSは、暗号化されたネットワーク用のNSX-T、システムの監視と管理用のvRealize Operations Manager、メトリックの収集用のWavefrontなどの多くのVMwareツールとも統合されます。Pivotalは、アドオンと統合をインストールするためのプラットフォームも提供します。

PKSは3つのレベルで構成されています。

  1. 独自のクラスターをデプロイおよび管理するための必須のPKS。
  2. BOSH、Harbour、およびその他のツールを使用した自動展開用のEnterprise PKS。
  3. フルマネージドのホスト型クラスター用のCloud PKS。
