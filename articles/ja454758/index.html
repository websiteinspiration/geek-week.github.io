Windows Defenderを安く陽気に回避する：Mimikatzを難読化する


みなさん、こんにちは。今日は、Windows 10でmimikatzを実行するオプションについて検討します。Mimikatzは、Windows資格情報エディターの機能を実装し、システムにログインしているユーザーの認証データをオープンフォームで抽出できるツールです。

ペンテスト中は、ユーザーのパスワードをダンプするものがあると便利ですが、Windowsに組み込まれている標準のWindows Defenderも問題になり、私たちの壮大な計画に干渉する可能性があります。

この方法は、署名によってバイナリの静的分析を行う他のウイルス対策製品（Virustotal BEFOREおよびAFTERも同様に考える）にも適していることに注意します。
そのため、この方法はEDRソリューションに役立つ可能性は低いですが、Windows Defenderを簡単に回避できます。

以前は、ファイル内の単語をmimikatzからmimidogzに変更し、メタデータとバナーの数行を削除することで回避できました。今ではもっと難しくなっていますが、それでも可能です。

このすべてのアクションのアイデアについて、私はippsecというニックネームの人に感謝を表します。

この記事では、以下を使用します。

  • Windows Defenderが有効になっているWindows 10（データベースが更新されている）
  • ミミカッツ
  • ビジュアルスタジオ
  • HxD（16進エディター）

mimikatzを被害者のコンピュータにコピーすると、そのような警告が表示されることが予想されます。



次に、一連の操作を実行して、Defenderがここで脅威を見ないようにします。

まず、mimikatzという単語を見つけて置き換えます。たとえばmimikatzをthuntに置き換えます（何にでも置き換えられます）。MIMIKATZはTHUNTに置き換えます。こんな感じです。


次に、Visual Studioでmimikatz \ mimikatz \ mimikatz.rcファイルを編集し（置き換え後はthunt.rcになりました）、mimikatzとgentilkiwiを何かに置き換えます。また、mimikatz.icoを他のアイコンに置き換えることも忘れないでください。[ソリューションの再構築]（またはソリューションの再構築）をクリックして、最新バージョンのmimikatzを入手してください。被害者をコンピュータにコピーします、ii ...アラート。Defenderが何のために働くのか調べてみましょう。最も簡単な方法は、アンチウイルスの最初の操作の前に、異なるサイズのバイナリをコピーすることです。

開始するには、半分をコピーしてWindows 10マシンにコピーします。

head –c 600000 mimikatz.exe > hunt.exe

Defenderは沈黙していて、悪くはありません。実験して、最初の操作を見つけます。それは私にはこのように見えました：

hexエディターでhunt.exeを開いて、Defenderが何のために働くか見てみましょう。目を引くのは、KiwiAndRegistryTools文字列です。



ランダムなキャップで遊んでみましょう-それはKiWIAnDReGiSTrYToOlSになり、保存してコピーします。沈黙、それは私たちが推測したことを意味します。コード内のこれらの行のすべての出現を見つけ、プロジェクトを置き換えて再構築します。確認するには、head -c 936300 mimikatz.exe> hunt.exeを実行します。前回ディフェンダーが働いていたが、今はそうではない。次へ。


そのようなトリッキーな方法で、hunt.exeに行を追加することでトリガーワードが発見されました-wdigest.dll、isBase64InterceptOutput、isBase64InterceptInput、multirdp、logonPasswords、credman。ランダムなキャップでそれらを変えて、Defenderが彼らへの悪態をつくのを確実に止めました。
しかし、Defenderはそんなに簡単なことではないと考え、インポートされ、大文字と小文字を区別する関数に取り組みました。これらは、netapi32.dllライブラリから呼び出される関数です。

  • I_NetServerAuthenticate2
  • I_NetServerReqChallenge
  • I_NetServerTrustPasswordsGet

netapi32.dll（C：\ windows \ system32 \ netapi32.dll）を見ると、各関数に番号が割り当てられていることがわかります。



関数呼び出しを
windows.netapi32.I_NetServerTrustPasswordsGet（args）の形式
から
windows.netapi32 [62]（args）
に変更します。これを行うには、mimikatz \ lib \ x64 \ netapi32.min.libを置き換える必要があります。netapi32.defファイルを作成し、そこに次の行を書き込みます。

LIBRARY netapi32.dll
EXPORTS
  I_NetServerAuthenticate2 @ 59
  I_NetServerReqChallenge @ 65
  I_NetServerTrustPasswordsGet @ 62

コマンドを保存して実行します（念のため、元のnetapi32.min.libをバックアップすることを忘れないでください）。

lib /DEF:netapi32.def /OUT:netapi32.min.lib

もう一度、プロジェクトを再構築し、取得したものをコピーします。Defenderは沈黙しています。結果のmimikatzを管理者権限で実行します。



成功。したがって、mimikatzが起動し、Windows Defenderが機能しませんでした。パスワード、外観、ハッシュが発行されます。 待っている

：

現実：

* Username : thunt
* Domain : DESKTOP-JJRBJJA
* Password : (null)

生活の状況は、実験室の状況とは多少異なります。パスワードを表示するには、レジストリを操作する必要がある場合があります。たとえば、UseLogonCredentialキーを有効にするか作成します（HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Control \ SecurityProviders \ WDigest）。しかし、これには問題があるかもしれません。再起動すると、キーを元に戻すことができます。

Windows 10の最新バージョンのいずれかで実行している場合、プレーンテキストのパスワードの代わりに次のように表示されると、さらに悪化する可能性があります。

* Password : _TBAL_{68EDDCF5-0AEB-4C28-A770-AF5302ECA3C9}

これは、自動再起動サインオン（ARSO）の後継であるTBALに関するすべてです。TBALが要求されると、lsasrvはアカウントがローカルアカウントかMSアカウントかをチェックし、これに基づいてmsv1_0またはcloudAPを使用して、ユーザーセッションを再開するために必要なすべてのものを保存します。次に、自動ログオンメカニズムがハードコードされたパスワード_TBAL_ {68EDDCF5-0AEB-4C28-A770-AF5302ECA3C9}に設定されます。

それにもかかわらず、研究室ではユーザーのパスワードを受け取り、戦闘状況では少なくともハッシュを取得できます。
