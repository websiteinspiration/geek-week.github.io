モノリシックなUnixライクなOSの開発-Cライブラリ（2）

では前の記事、私たちは、Hello Worldのカーネルを実行する方法を学び、文字列を操作するための機能のカップルを書きました。次に、Cライブラリを拡張して、kprintfおよびその他の必要な関数を実装できるようにします。行こう！

目次


  1. ビルドシステム（make、gcc、gas）。初期ブート（マルチブート）。（qemu）を起動します。Cライブラリ（strcpy、memcpy、strext）。
  2. Cライブラリ（sprintf、strcpy、strcmp、strtok、va_list ...）カーネルモードおよびユーザーアプリケーションモードでのライブラリのビルド。
  3. カーネルシステムログ。ビデオメモリ 端末への出力（kprintf、kpanic、kassert）。
  4. 動的メモリ、ヒープ（kmalloc、kfree）。
  5. メモリと割り込み処理の構成（GDT、IDT、PIC、syscall）。例外
  6. 仮想メモリ（ページディレクトリとページテーブル）。
  7. 処理する。スケジューラ。マルチタスク。システムコール（kill、exit、ps）。
  8. カーネル（initrd）、elf、およびその内部のファイルシステム。システムコール（exec）。
  9. キャラクターデバイスドライバー。システムコール（ioctl、fopen、fread、fwrite）。Cライブラリ（fopen、fclose、fprintf、fscanf）。
  10. カーネルの完全なプログラムとしてのシェル。
  11. ユーザー保護モード（ring3）。タスクステータスセグメント（tss）。

図書館C


最初に、明示的な次元を持つ型を実装する必要があります。

1つのプラットフォームで収集を行うため、定義と実装が正しくなるまで。それは普遍的ではありませんが、それが読みやすくシンプルな理由です。普遍的なソリューションをサポートするのは非常に難しいので、普遍的ではない何かをする方が良い場合もありますが、現在の要件を満たしているというアプローチをとっています。

typedef unsigned char u8;
typedef unsigned short u16;
typedef unsigned int u32;
typedef unsigned char u_char;
typedef unsigned short u_short;
typedef unsigned int u_int;
typedef unsigned int u_long;

ブール型を導入しても問題ありません。

#pragma once

/* types */
typedef int bool;

#define true 1
#define false 0
#define null 0

また、バイトを処理するためのいくつかのマクロは私たちを傷つけません。

typedef unsigned long size_t;

#define HIGH_WORD(addr) ((addr & 0xffff0000) >> 16)
#define LOW_WORD(addr) ((addr & 0xffff))
#define LOW_BYTE(addr) ((addr & 0x00ff))
#define HIGH_BYTE(addr) ((addr & 0xff00) >> 8)

可変数の引数を処理するには、次のマクロが必要です。このアプローチは、関数がC言語の呼び出し規則に従っている場合にのみ機能します。この規則では、関数の引数は最後からスタックを介して渡されます。

typedef size_t* va_list;

#define va_start(l, a) (l = (void*)((size_t)&a) + sizeof(a))
#define va_end(l) (l = (void*)0)
#define va_arg(l, s) (*(s*)(l++))

もちろん、逆の方向に進むこともできます。独自の関数を定義する代わりに、組み込みライブラリを使用して、LD_PRELOADを介してカーネルにアクセスする関数を置き換えます。しかし、私はプロセスを完全に制御したいので、このチュートリアルでOSを書き始める人のためのアイデアとして、このオプションは残しておきましょう。

さらに、ビデオチュートリアルでは、次のライブラリ関数の実装について検討します。実装は最適性と完全性を主張するものではありませんが、単純さと読みやすさを主張しています。 strtok_rと呼ばれるstrtok関数のスレッドセーフな実装を使用していることに注意してください。そして、前回のレッスンでstrinv関数とstrext関数を自分で考えました。 C言語に精通していれば、以下にリストされているほとんどすべての関数に精通していると思います。

extern char* strcpy(char* s1, const char* s2);
extern char* strncpy(char* s1, const char* s2, u_int n);
extern void* memcpy(void* buf1, const void* buf2, u_int bytes);
extern void* memset(void* buf1, u8 value, u_int bytes);
extern int strcmp(const char* s1, const char* s2);

ルーチンは廃止されます。ボイラープレートコードの終わり。次のレッスンはもっと複雑で興味深いものになります。プロジェクトを後援したい場合は、ライブラリー関数の最適な実装を提供できます。
