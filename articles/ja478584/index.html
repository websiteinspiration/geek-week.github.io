JVMの内部、パート2-クラスファイルの構造

みなさん、こんにちは！この記事の翻訳は、特にJava開発者コースの受講生を対象に作成されました。



Java仮想マシンが内部でどのように機能するかについては、引き続き説明します。では前回の記事（英語オリジナル）、私たちは、クラスのロード・サブシステムを検討しました。この記事では、クラスファイルの構造について説明します。

すでにご存じのとおり、Javaプログラミング言語で記述されたすべてのソースコードは、Java開発キットに含まれているコンパイラを使用して、まずバイトコードにコンパイルされjavacます。バイトコードは、特別なクラスファイルのバイナリファイルに格納されます。次に、これらのクラスファイルは、クラスローダー（ClassLoader）によって動的に（必要に応じて）メモリに読み込まれます。


図-Javaソースコードのコンパイル

拡張子が付いた各ファイルは、.java少なくとも1つのファイルにコンパイルされます.class。ソースコードで定義されたクラス、インターフェース、モジュールごとに、1つの.classファイルが作成されます。これは、インターフェースとネストされたクラスにも適用されます。

注-簡単にするために、拡張子が.class「クラスファイル」のファイルを呼びます。

簡単なプログラムを書いてみましょう。

public class ClassOne{
    public static void main(String[] args){
        System.out.println("Hello world");
    }
    static class StaticNestedClass{
    }
}

class ClassTwo{
}

interface InterfaceOne{
}

javacこのファイルを 実行すると、次のファイルが生成されます。

ClassOne$StaticNestedClass.class
ClassOne.class
ClassTwo.class
InterfaceOne.class

ご覧のとおり、クラスとインターフェイスごとに個別のクラスファイルが作成されます。

クラスファイル内には何がありますか？


クラスファイルはバイナリ形式です。その中の情報は通常、連続する情報の間にインデントなしで書き込まれ、すべてがバイト境界に揃えられます。すべての16ビット値と32ビット値は、2つまたは4つの連続する8ビットバイトを使用して書き込まれます。

クラスファイルには、次の情報が含まれています。

マジックナンバー、サイン。各クラスファイルの最初の4バイトは常に0xCAFEBABEです。これらの4バイトは、Javaクラスファイルを識別します。

ファイルのバージョン。次の4バイトには、ファイルのメジャーバージョンとマイナーバージョンが含まれます。これらの数値を合わせて、クラスファイル形式のバージョンを決定します。クラスファイルにメジャーメジャーバージョンのMとマイナーmがある場合、このバージョンをMmと指定します。

各JVMには、サポートされているクラスファイルのバージョンに制限があります。たとえば、Java 11は45から55、Java 12は45から56のメジャーバージョンをサポートしてい

ます。定数プール。 ClassFile構造とそのサブ構造にある文字列定数、クラス名、インターフェース、フィールド、メソッド、およびその他の定数を表す構造の表。各定数プール要素は、定数のタイプを定義する1バイトのタグで始まります。定数のタイプに応じて、後続のバイトは、即時定数値またはプール内の別の要素への参照になります。

アクセスフラグ。クラスがパブリックまたはプライベートのインターフェイスであり、最終クラスであるかどうかを示すフラグのリスト。などの各種フラグ、ACC_PUBLIC、ACC_FINAL、ACC_INTERFACE、ACC_ENUMetc. Java仮想マシン仕様が記述されています。

このクラス。定数プールのエントリにリンクします。

スーパークラス。定数プールのエントリにリンクします。

インターフェースクラスによって実装されるインターフェースの数。

フィールドの数。クラスまたはインターフェースのフィールド数。

田畑。フィールド数の後に、可変長の構造の表が続きます。各フィールドに1つ、フィールドのタイプと名前の説明（定数のプールを参照）。

メソッドの数。クラスまたはインターフェース内のメソッドの数。この数には、クラスで明示的に定義されたメソッドのみが含まれ、スーパークラスから継承されたメソッドは含まれません。

方法次はメソッド自体です。各メソッドには、次の情報が含まれています：メソッド記述子（戻り値の型と引数リスト）、メソッドのローカル変数に必要なワード数、メソッドオペランドスタックに必要なスタックワードの最大数、メソッドがキャッチする例外テーブル、メソッドバイトコード、およびテーブル行番号。

属性の数。このクラス、インターフェース、またはモジュールの属性の数。

の属性属性の数の後に、各属性を説明するテーブルまたは可変長構造が続きます。たとえば、常に「SourceFile」属性があります。これには、クラスファイルがコンパイルされたソースファイルの名前が含まれます。

クラスファイルは直接人間が読める形式ではありませんが、JDKには内容を便利なフォーマットで表示するjavap。

以下に示すような簡単なJavaプログラムを書いてみましょう。

package bytecode;
import java.io.Serializable;

public class HelloWorld implements Serializable, Cloneable {

    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
