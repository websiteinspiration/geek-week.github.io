Express.js上のReactアプリのサーバー側レンダリング

この記事を書くために、Reactアプリケーション用のサーバーサイドレンダリングを作成する方法について、多かれ少なかれ完全なマニュアルが欠けていることに気付きました。

この問題に遭遇したとき、Next.jsフレームワークを使用するか、Express.jsを使用するかのどちらかを行うための2つのオプションがありました。

Next.jsの調査に約100時間を費やし、既製の大規模OTTプラットフォーム用に取得しましたが、問題が多すぎて拒否しました（これについては再度記事を書きます）。小規模なExpress.jsを選択することもできました。私が伝えたいことについて。

この記事のデモの完全なコードはこちらです。

最初のタスクから始めましょう。

当時私達は持っていました：

  • 以下に示すアーキテクチャと機能を備えた、すでに生産されている完全に既製のアプリケーション。
  • ストアのImmutable.jsで使用
  • 中古Reduxサガ
  • 動的インポートのためのReact.lazyおよびReact.suspense
  • ルーティング用のReact-router 4
  • アイコンに使用されるカスタムフォント

タスク：

  • ソリューションは、レンダリングされたHTMLをデータとともに完全にレンダリングする必要があります
  • このソリューションは、既存のコードへの影響を最小限にする必要があります。つまり、アーキテクチャは最小限の変更で維持する必要があります。
  • プロジェクトに既にあるライブラリを引き続き使用する
  • SEOメタデータはページとともにサーバーから取得する必要があります
  • 画像、スタイル、フォントをサーバーにアップロードしてクライアントに送信し、その後のダウンロードは不要
  • クライアント側の動的インポートのサポート
  • サーバーでプリロードするため、およびクライアント側でナビゲートするときに、コードの重複を避けます

タスクを決定しました。これを行う方法を理解しましょう。

反応のドキュメントから、SSRに対してrenderToString（）およびhydrate（）メソッドを使用できることがわかりますが、次に何をすべきか？

renderToString-アプリケーションのサーバーでHTMLを生成するために使用されます。
水和物 -クライアントとサーバーでのユニバーサルレンダリングに使用されます。

データの読み込み


サーバー側でデータをダウンロードするには、redux-connectライブラリーを使用します。これにより、最初のレンダーを呼び出す前に必要なデータをダウンロードできます。これが必要です。これを行うには、hoc asyncConnectを使用します。サーバー側では、データをロードし、ルーティング時には、componentDidMountとして機能します。

@asyncConnect([
  {
    key: 'usersFromServer',
    promise: async ({ store: { dispatch } }) => {
      await dispatch(getUsersData());

      return Promise.resolve();
    },
  },
])

サーバー側にreduxストアを作成する必要があります。すべて通常どおりです。server.jsファイルで作成するだけです。

また、サーバー側では、redux-connectの loadOnServerメソッドを使用して、データのプリロードを待機します。

renderToStringを使用して、データを含むアプリケーションのHTMLを取得します。

チェックインしたデータには、getState（）を使用してアクセスでき、<script />タグを介してグローバルウィンドウオブジェクトに追加できます。その後、クライアントでデータを取得し、ゲートに設定します。

それはすべてこのように見えます

app.get('*', (req, res) => {
  const url = req.originalUrl || req.url;
  const history = createMemoryHistory({
    initialEntries: [url],
  });
  const store = configureStore(initialState, history);
  const location = parseUrl(url);
  const helpers = {};
  const indexFile = path.resolve('./build/main.html');

  store.runSaga(sagas).toPromise().then(() => {
    return loadOnServer({ store, location, routes, helpers })
      .then(() => {
        const context = {};

        if (context.url) {
          req.header('Location', context.url);
          return res.send(302)
        }

        const css = new Set(); // CSS for all rendered React components
        const insertCss = (...styles) => styles.forEach(style => css.add(style._getCss()));

        const dynamicRoutes = [...routes];
        dynamicRoutes[0].routes = [...dynamicRoutes[0].routes, ...StaticRoutesConfig];

        const appContent = ReactDOMServer.renderToString(
          <StyleContext.Provider value={{ insertCss }}>
            <Provider store={store} key="provider">
              <StaticRouter location={location} context={context}>
                <ReduxAsyncConnect routes={dynamicRoutes} helpers={helpers}/>
              </StaticRouter>
            </Provider>
          </StyleContext.Provider>
        );

        const helmet = Helmet.renderStatic();

        fs.readFile(indexFile, 'utf8', (err, data) => {
          if (err) {
            console.log('Something went wrong:', err);
            return res.status(500).send('Oops, better luck next time!');
          }
          data = data.replace('__STYLES__', [...css].join(''));
          data = data.replace('__LOADER__', '');
          data = data.replace('<div id=app></div>', `<div id=app>${appContent}</div>`);
          data = data.replace('<div id="app"></div>', `<div id="app">${appContent}</div>`);
          data = data.replace('<title></title>', helmet.title.toString());
          data = data.replace('<meta name="description" content=""/>', helmet.meta.toString());
          data = data.replace('<script>__INITIAL_DATA__</script>', `<script>window.__INITIAL_DATA__ = ${JSON.stringify(store.getState())};</script>`);

          return res.send(data);
        });
      });
    store.close();
  });
});

App.js ファイル

const initialState = !process.env.IS_SERVER ? window.__INITIAL_DATA__ : {};

const history = process.env.IS_SERVER
  ? createMemoryHistory({
    initialEntries: ['/'],
  })
  : createBrowserHistory();

const store = configureStore(initialState, history);
if (!process.env.IS_SERVER) {
  window.store = store;
}

const insertCss = (...styles) => {
  // eslint-disable-next-line no-underscore-dangle
  const removeCss = styles.map(style => style._insertCss());
  return () => removeCss.forEach(dispose => dispose());
};

export const browserRender = () => {
  const dynamicRoutes = [...routes];
  dynamicRoutes[0].routes = [...dynamicRoutes[0].routes, ...StaticRoutesConfig];

  hydrate(
    <StyleContext.Provider value={{ insertCss }}>
      <Provider key="provider" store={store} >
        <ConnectedRouter history={history}>
          <ReduxAsyncConnect helpers={{}} routes={dynamicRoutes} />
        </ConnectedRouter>
      </Provider>
    </StyleContext.Provider>,
    document.getElementById('app'),
  );
};

代わりに、すでに述べたように、説明したルートの配列があり、それらをアプリケーションに追加するには、react-router-configを使用する必要があります：

App / index.js

import React, { Component } from 'react';
import PropTypes from 'prop-types';
import { connect } from 'react-redux';
import { Helmet } from 'react-helmet';
import { renderRoutes } from 'react-router-config';

import { getRouterLocation } from './selectors/router';


@connect(state => ({
  location: getRouterLocation(state),
}), null)
export default class App extends Component {
  static propTypes = {
    location: PropTypes.shape().isRequired,
    route: PropTypes.shape().isRequired,
  };

  render() {
    const { route } = this.props;

    return (
      <div>
        {renderRoutes(route.routes)}
      </div>
    );
  }
}

サーバーにカルネ、スタイル、フォントをダウンロードする


スタイルについては、isomorphic-style-loaderが使用されました。これは、通常のslyle-loaderは、ターゲットを備えたWebpackでは機能しないためです。「node」;

彼はすべてのスタイルをDOMに追加するため、既製の美しいページがサーバーから取得されます。別のファイルにスタイルを保存して、ブラウザでキャッシュできるようにすることができます。

サーバーで画像を表示してフォントをダウンロードするには、webpackローダーbase64-inline-loaderを使用しました。

{
    test: /\.(jpe?g|png|ttf|eot|otf|svg|woff(2)?)(\?[a-z0-9=&.]+)?$/,
    use: 'base64-inline-loader?limit=1000&name=[name].[ext]',
  },

これは、画像および使用したすべてのタイプのフォントに含まれていました。その結果、base64サーバーからコードを受け取りました。このコードは、フォントと画像を含むページを、後で読み込むことなく表示しました。

クライアントビルドでは、通常のurl-loaderとfile-loaderが使用されました。

{
        test: /\.(eot|svg|otf|ttf|woff|woff2)$/,
        use: 'file-loader',
      },
      {
        test: /\.(mp4|webm|png|gif)$/,
        use: {
          loader: 'url-loader',
          options: {
            limit: 10000,
          },
        },
      },

はい、読み込まれたページのサイズが大きくなりましたが、ダウンロード速度の点ではユーザーにはそれほど気になりませんでした。

サーバーでの動的インポート


React.lazyとReact.suspenseを 使用するダイナミックローダーをインポートして表示するReact.js では、SSRでは機能しません。同じことを行うreact-loadable

を使用しました。

import Loadable from 'react-loadable';
import Loader from './Loader';

const LoadableComponent = Loadable({
  loader: () => import('./my-component'),
  loading: Loader,
});

export default class App extends React.Component {
  render() {
    return <LoadableComponent/>;
  }
}

クライアントでは、このコードはローダーを表示し、サーバーでは、モジュールをロードするために、次のコードを追加する必要があります。

Loadable.preloadAll().then(() => {
  app.listen(PORT, () => {
    console.log(` Server is listening on port ${PORT}`);
  });
});

Loadable.preloadAll（）-モジュールがロードされたことを示すpromiseを返します。
