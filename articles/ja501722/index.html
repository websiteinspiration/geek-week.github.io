コメット-高速REST API用のPHPフレームワーク

2年間、Swaggerベースのコードジェネレーターを使用してGoでマイクロサービスを作成しました。非常にコンパクトで非常に高速なソリューションであることがわかりました。

私は現在PHPを使用しているので、言語エコシステムの同じ問題を解決する手段を見つけることにしました。私はLaravelとSymfonyを知っており、大好きですが、それらをプロジェクトにドラッグしたくありませんでした。プロジェクトとパフォーマンスを開始するために急なカーブを払わなければならないバッテリーが多すぎます。

その結果、Cometが登場しました-SlimPHPとWorkermanのチームの経験を使用して高速APIを開発するための、最新のPHPベースのフレームワーク。通常の仮想マシンで数万のRPSと1ミリ秒未満のレイテンシ！

根拠のないように、Cometおよびその他の一般的なフレームワークのテスト結果を示します。より正直に比較するために、ORMのようなすべての重いモジュールは、すべてのテスト参加者のアセンブリから削除されました。

最初のケースは、「Hello、World！」ウェルカムテキストをプレーンテキストで返すメソッドに対する数千の競合wrkリクエストの形での高負荷の模倣です：



最小遅延を見積もるために、単一のクライアントが1つのリクエストを同じリクエストに順次送信するオプションを使用しましたエンドポイント：



結果からわかるように、Cometアーキテクチャの機能を使用すると、従来のフレームワークよりも遅延が最小で、リクエストの10倍のストリームを処理できます。

コメットでの実際の開発で何が発生するかを理解するためのコードを見てみましょう：

use Comet\Comet;

require_once __DIR__ . '/vendor/autoload.php';

$app = new Comet();

$app->get('/hello', function ($request, $response) {
    $response
        ->getBody()
        ->write("Hello, Comet!");      
    return $response;
});

$app->run();

すべてが非常に透過的です。ルーターとクロージャーを使用すると、NodeJS / Express開発者が慣れているものに似たコンパクトなコードが提供されます。

私はすべてのコードをGitHubに投稿しており、フレームワークの機能を拡張する予定です
