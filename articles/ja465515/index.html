ACLとは何ですか？最初のビデオレッスンからコースを学習した場合は、さまざまなネットワークデバイス間の通信をどのように編成したかを思い出してください。



また、デバイスとネットワーク間の通信スキルを習得することを目的として、さまざまなプロトコルでの静的ルーティングについても調査しました。これでトレーニングステージに到達しました。トラフィック制御を確実にするために注意を払う必要があります。つまり、「悪意のある人」または権限のないユーザーがネットワークに侵入するのを防ぎます。たとえば、これは、この図に示されているSALES販売部門の人々に関係している可能性があります。ここでは、ACCOUNTS財務部門、MANAGEMENT管理部門、サーバーのSERVER ROOMも示しています。
したがって、100人の従業員が営業部門で働くことができ、ネットワーク経由でサーバーにアクセスできるようにする必要はありません。 Laptop2コンピューターで働くセールスマネージャーは例外で、サーバールームにアクセスできる場合があります。 Laptop3で働く新入社員はこのアクセス権を持っていてはなりません。つまり、自分のコンピューターからのトラフィックがルーターR2に到達した場合、彼はドロップされます。

ACLの役割は、指定されたフィルタリングパラメータに従ってトラフィックをフィルタリングすることです。これらには、送信元IPアドレス、宛先IPアドレス、プロトコル、ポート数、およびその他のパラメーターが含まれています。これにより、トラフィックを識別し、それを使用していくつかのアクションを実行できます。

したがって、ACLはOSIモデルの第3レベルのフィルタリングメカニズムです。つまり、このメカニズムはルーターで使用されます。フィルタリングの主な基準は、データストリームの識別です。たとえば、Laptop3コンピューターを使用している人がサーバーにアクセスできないようにブロックする場合は、まず彼のトラフィックを特定する必要があります。このトラフィックは、ネットワークデバイスの対応するインターフェイスを介してLaptop-Switch2-R2-R1-Switch1-Server1の方向に移動しますが、ルーターのG0 / 0インターフェイスはそれとは関係ありません。



トラフィックを特定するには、そのパスを特定する必要があります。これで、フィルターをインストールする場所を正確に決定できます。フィルター自体について心配する必要はありません。フィルターについては次のレッスンで説明します。ここでは、フィルターを適用するインターフェイスの原則を理解する必要があります。

ルーターを見ると、トラフィックが移動するたびに、データストリームが入るインターフェイスと、このストリームが流れるインターフェイスがあることがわかります。

実際、3つのインターフェイスがあります。入力インターフェイス、出力インターフェイス、およびルーター自体のインターフェイスです。フィルタリングは入力または出力インターフェースにのみ適用できることを覚えておいてください。



ACL操作の原理は、招待されたゲストのリストに名前が記載されているゲストだけがアクセスできるイベントへの入場に似ています。 ACLは、トラフィックの識別に役立つ修飾パラメータのリストです。たとえば、このリストは、IPアドレス192.168.1.10からのトラフィックはすべて許可され、他のすべてのアドレスからのトラフィックは禁止されることを示しています。すでに述べたように、このリストは入力インターフェースと出力インターフェースの両方に適用できます。

ACLには、標準と詳細の2種類があります。標準ACLには、1から99または1300から1999の識別子があります。これらは、番号が増加しても相互に利点がないリストの名前です。番号に加えて、ACLには独自の名前を割り当てることができます。拡張ACLには、100〜199または2000〜2699の番号が付けられ、名前が付けられている場合もあります。

標準ACLでは、分類はトラフィックソースのIPアドレスに基づいています。したがって、そのようなリストを使用する場合、任意のソースに送信されるトラフィックを制限することはできず、一部のデバイスからのトラフィックのみをブロックできます。

Advanced ACLは、送信元IPアドレス、宛先IPアドレス、使用されるプロトコル、およびポート番号によってトラフィックを分類します。たとえば、FTPトラフィックのみ、またはHTTPトラフィックのみをブロックできます。今日は標準ACLについて見ていきます。次のビデオチュートリアルでは、拡張リストを取り上げます。

私が言ったように、ACLは条件のリストです。このリストをルータのインバウンドまたはアウトバウンドインターフェイスに適用すると、ルータはこのリストを使用してトラフィックをチェックし、リストに記載されている条件を満たす場合、このトラフィックを許可するかブロックするかを決定します。複雑なことは何もありませんが、多くの場合、ルーターの入力インターフェイスと出力インターフェイスを特定するのは困難です。着信インターフェイスについて話すとき、これは着信トラフィックのみがこのポートで監視され、ルーターは発信トラフィックに制限を適用しないことを意味します。同様に、出力インターフェイスに関しては、すべてのルールが発信トラフィックにのみ適用され、このポートの着信トラフィックは制限なしで受け入れられます。たとえば、ルーターに2つのポートがある場合：f0 / 0およびf0 / 1次に、ACLはf0 / 0インターフェースに入るトラフィック、またはf0 / 1インターフェースから来るトラフィックにのみ適用されます。 f0 / 1インターフェースに入る、またはf0 / 0ポートから来るトラフィックは、リストの影響を受けません。



したがって、インターフェイスの着信方向または発信方向と混同しないでください。特定のトラフィックの移動方向によって異なります。したがって、ルータがトラフィックをACL条件に準拠しているかどうかをチェックした後は、トラフィックをスキップするか拒否するという2つの解決策しか取ることができません。たとえば、アドレス180.160.1.30宛てのトラフィックを許可し、アドレス192.168.1.10宛てのトラフィックを拒否できます。各リストには多くの条件を含めることができますが、これらの各条件は許可または拒否する必要があります。

まず、ルーターはトラフィックが最初の条件と一致するかどうかをチェックし、一致しない場合は2番目の条件と一致します。トラフィックが3番目の条件に一致する場合、ルーターはチェックを停止し、リストの他の条件と比較しません。彼は「許可」アクションを実行し、トラフィックの次の部分のチェックに進みます。

パケットにルールを設定しておらず、トラフィックがどの条件にも該当せずにリストのすべての行を通過した場合、各ACLはデフォルトでdeny anyコマンドで終了するため、パケットは破棄されます。ルールに該当しない。この条件は、リストに少なくとも1つのルールがある場合に有効になります。それ以外の場合は適用されません。ただし、最初の行にエントリ192.168.1.30の拒否が含まれ、リストに条件が含まれなくなった場合は、最後にpermit anyコマンドが必要です。つまり、ルールで禁止されているトラフィック以外はすべて許可されます。 ACLを構成するときのエラーを回避するために、これを考慮する必要があります。

ACLを生成するための基本的なルールを覚えておいてください。標準ACLをできるだけ宛先（つまり、トラフィックの受信者）の近くに配置し、拡張ACLをソース（トラフィックの送信者）のできるだけ近くに配置します。これらはシスコの推奨事項ですが、実際には、トラフィックソースの近くに標準ACLを配置する方が合理的である場合があります。ただし、試験のACL配置ルールに関する質問がある場合は、シスコの推奨事項に従って、明確に回答してください。標準-宛先に近い、高度な-送信元に近い。

次に、標準ACLの構文を見てみましょう。グローバルルータコンフィギュレーションモードのコマンド構文には、クラシック構文とモダン構文の2つのタイプがあります。



古典的なタイプのコマンドは、access-list <ACL番号> <disable / allow> <criteria>です。 <ACL番号>を1〜99に指定すると、デバイスはそれが標準ACLであることを自動的に認識し、100〜199の場合は拡張されます。今日のレッスンでは標準リストを検討しているため、1〜99の任意の数を使用できます。次に、パラメーターがトラフィックを許可または無効にするという次の基準に一致する場合に適用する必要があるアクションを示します。この基準は現代の構文でも使用されるため、後で検討します。

最新タイプのコマンドは、グローバル構成モードRx（config）でも使用され、次のようになります：ip access-list standard <ACL number / name>。ここでは、1から99までの数字、またはACLの名前（ACL_Networkingなど）を使用できます。このコマンドは、システムをすぐにRx標準モードサブコマンドモード（config-std-nacl）にします。この場合、<prohibit / allow> <criteria>を入力する必要があります。モダンなタイプのチームには、クラシックよりも多くの利点があります。

クラシックリストで、「access-list 10 deny ______」と入力した場合、別の基準で同じ種類の次のコマンドを入力すると、そのようなコマンドが100個表示されます。入力したコマンドを変更するには、access-list全体を削除する必要があります。このリストの単一のコマンドを編集する方法がないため、コマンドno access-list 10を含むリスト10。

最新の構文では、コマンドは2行に分割され、最初の行にはリスト番号が含まれています。アクセスリストの標準10拒否________、アクセスリストの標準20拒否________などがある場合、他の基準を使用してそれらの間に中間リストを挿入できます。たとえば、アクセスリストの標準15拒否________などです。

または、アクセスリストの標準20行を削除して、アクセスリストの標準10行とアクセスリストの標準30行の間に他のパラメータを付けて再入力するだけで、最新のACL構文を編集するさまざまな方法があります。

ACLのコンパイルには細心の注意が必要です。ご存知のとおり、リストは上から下に読み取られます。特定のホストのトラフィックの許可がある行を上に配置すると、その下に、このホストが属するネットワーク全体のトラフィック禁止の行を配置できます。両方の条件がチェックされます-特定のホストへのトラフィックが渡され、他のすべてのホストのトラフィックがチェックされますこのネットワークはブロックされています。したがって、特定のレコードは常にリストの一番上に配置し、一般的なレコードは一番下に配置してください。

したがって、クラシックまたはモダンACLを作成した後、それを適用する必要があります。これを行うには、特定のインターフェイスの設定に移動します。たとえば、interface <type and slot>コマンドを使用してf0 / 0、インターフェイスサブコマンドモードに移動し、ip access-group <ACL番号/名前> <in / out>コマンドを入力します。違いに注意してください。リストをコンパイルするときにはaccess-listが使用され、使用されるときにはaccess-groupが使用されます。リストを適用するインターフェイスを決定する必要があります-着信トラフィックのインターフェイスまたは発信トラフィックのインターフェイス。リストにNetworkingなどの名前が付いている場合、このインターフェイスのlist applicationコマンドで同じ名前が繰り返されます。
