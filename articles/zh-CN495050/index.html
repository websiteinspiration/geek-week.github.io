计划和功能编程研讨会2020的申请开放



2020年科学实践会议计划和功能编程研讨会将打开报告申请。


Scheme and Functional Programming Workshop是一个年度活动，是一次科学实践会议，汇集了从美学或技术角度上满足Scheme Algorithmic语言中规定的基本原理的编程语言的专家：保持通用性的极简主义和提供灵活设计的精心设计。

预计演讲者将发表有关先进科学成果的文章，以及有关工程和教育领域的实际成就的报告，以及有关考虑过早阶段的想法或重新考虑旧方法的详细建议。欢迎和鼓励与Scheme家族中任何语言有关的作品：从RnRS的纯子集到“方案主题的变体”，例如Racket，Lisp的其他方言，包括Closure，Emacs Lisp，Common Lisp和其他功能语言支持闭包和/或宏（或在语言扩展工具（例如库）中具有这种支持）。这样的示例可以是Dylan，ECMAScript，Hop，Lua，Scala，Rust等。与实现示例的语法的各个元素相比，作品的整体优雅程度和与您声明的相关性将是更重要的评估标准。

提要格式


最终的纸张版本必须使用acmart格式和acmlarge子格式。它们必须为PDF，并允许以US Letter格式在纸上打印。 Microsoft Word / LaTeX模板可在此处下载。

此格式与公认的ACM会议格式（包括ICFP，在同一时间同时发生）一致。建议对行编号使用“审阅”选项，这样会使审阅更加容易。

会议欢迎所有可能的报告类型，包括成熟的科学文章，实用报告和闪电演讲。文章和报告限于14页，但欢迎使用较小的尺寸。突击报告限制为192个字。每个文章或报告都有25分钟的时段，包括问题。闪电战报告分配了两个五分钟的时间段，一个用于报告，另一个用于问题。

大小限制不适用于链接和应用程序。对应用程序的大小没有限制，但应在没有限制的情况下阅读材料，并且在考虑应用程序时，可能不会考虑它们。

欢迎在公开许可下发布与所述材料相关的源代码，以使审阅者可以验证所提出的指控。

会议论文集将以《密歇根大学技术报告》的格式出版，并上载至arXiv.org

。本科学实践会议的出版物并不代表排他性，并为以后出版更完整提供了机会。或在其他科学会议或科学期刊上发表的工作的详细版本。
