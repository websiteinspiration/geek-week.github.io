DIY Exergaming secara terpisah

Setiap orang memiliki isolasi sendiri. Dalam kasus saya, masalah utama dengan cepat menjadi menjaga tubuh dalam kondisi fisik yang baik. Bekerja di kantor dan berbelanja hidup-hidup, ternyata, menjaga tubuh dalam kondisi baik. Saya beruntung mendapatkan sepeda olahraga termurah sebelum krisis, sebelum terjual habis di mana-mana, jadi saya pikir masalahnya sudah selesai - tetapi tidak ada. Ketika mengayuh menjadi kebutuhan sehari-hari yang monoton, masalah selanjutnya adalah kebosanan. Dan dari segala sesuatu yang membosankan, adalah sifat manusia untuk menjauh. Bagaimana membuatnya lebih menarik?Mungkin bergabung dengan game? Kenapa tidak? Saya sudah lama berpikir betapa hebatnya jika di dalam mainan terbang kecepatan pesawat Anda bergantung pada kecepatan mengayuh, dan sekarang saatnya untuk menerapkan gagasan itu. Untuk ini, ternyata, mereka telah datang dengan kataâ€” "exergaming".

Pikiran pertama adalah menemukan joystick siap pakai dengan roda untuk kontrol kecepatan, dan entah bagaimana meretasnya. Untungnya, joystick, yang tidak akan disayangkan, tidak ada di tangan, dan saya memutuskan untuk membuatnya sendiri. Di laci meja adalah Arduino Micro, membeli beberapa tahun yang lalu untuk proyek lain, dan tidak pernah digunakan. Micro memiliki port USB, dan komputer mengenalinya sebagai keyboard. Dan di mana keyboard berada, tidak jauh dari perangkat HID (Human Interface Device) lainnya. Perpustakaan untuk joystick ditemukan dengan cepat dan cukup nyaman (github.com/MHeironimus/ArduinoJoystickLibrary ). Tambahkan beberapa tombol dan joystick dari gambar SparkFun - selesai! Rangkaian ini benar-benar sepele - Arduino, tombol, dua variabel resistensi di joystick.

Pertanyaan selanjutnya adalah bagaimana cara mendapatkan kecepatan mengayuh dari simulator. Hampir semua simulator modern memiliki layar dan sensor, tetapi saya tidak ingin memilih simulator (dan pada saat yang sama kehilangan garansi). Selain itu, saya ingin semua orang dapat mengulangi seluruh proses tanpa masalah. Untuk memulai, saya menggunakan tombol sederhana di papan tempat memotong roti untuk memverifikasi bahwa Arduino berfungsi. Saya memeriksanya, tetapi terbawa sehingga pada akhir tes tombolnya benar-benar berantakan. Saatnya memikirkan sensor yang serius. Sensor magnetik dan optik mencapai final, dan saya memilih yang magnetik, dengan asumsi (benar) bahwa akan ada lebih sedikit masalah dengan itu.

Ada banyak model sensor magnetik yang berbeda berdasarkan efek Hall, saya memilih Diodes AH3572 karena tidak peduli dengan polaritas, hanya perlu 3,3 volt dengan Arduino, relatif murah dan tersedia dalam dua jenis kasing, yang nyaman untuk papan tempat memotong roti, dan kemudian untuk Pemasangan PCB. Pertanyaan yang paling menarik adalah - seberapa jauh sensor akan menyala. Di sini saya beruntung, di tempat sampah Ibu Pertiwi ada magnet Cina yang cukup kuat dalam bentuk kubus dengan sisi sekitar satu sentimeter. Dengan magnet ini, sensor bekerja andal pada jarak hingga 3 sentimeter. Magnet pada pedal, sensor dengan selotip pada case - berfungsi.







Satu-satunya kesulitan adalah bahwa sensor, yang dipicu sekali per revolusi, tidak menyediakan data siap pakai yang cocok untuk digunakan sebagai nilai kecepatan. Dari satu klik pada revolusi, Anda perlu entah bagaimana mendapatkan nilai yang relatif mulus, tetapi pada saat yang sama, sehingga merespons dengan cepat terhadap perubahan kecepatan, dan terutama pada pemberhentian. Masalahnya dipecahkan dalam sekitar 20 baris kode.

//    
void interruptHandler() {
  unsigned long now = millis();
  if (lastInterrupted + DEBOUNCE_MS < now) {
    lastInterrupted = now;
    throttleBuffer += THROTTLE_BOOST;
    if (throttleBuffer > MAX_AXIS) {
      throttleBuffer = MAX_AXIS;
    }
  }
}

//       
void calculateThrottle() {
  if (throttleBuffer < RELEASE_PER_TICK) {
    throttle += throttleBuffer;
    throttleBuffer = 0;
  } else {
    throttle += RELEASE_PER_TICK;
    throttleBuffer -= RELEASE_PER_TICK;
  }

  if (throttle > MAX_AXIS) {
    throttle = MAX_AXIS;
  }
  
  if (throttle > DECAY_PER_TICK) {
    throttle -= DECAY_PER_TICK;
  } else {
    throttle = 0;
  }
}

Hingga titik ini, semuanya mudah, kesulitan lebih lanjut dimulai. Secara mengejutkan sulit menemukan mainan yang tepat. Dalam beberapa hari saya mencoba banyak pilihan. Pertama-tama, saya membutuhkannya untuk bekerja secara normal pada laptop kantor yang relatif lemah. Pada titik ini, semua pejuang terbang modern yang cantik keluar. Selebaran lama tidak dimulai, atau sangat membosankan. Game pertama yang cocok adalah Sturmovik IL-2, tetapi kemudian saya menyadari bahwa simulator serius tidak cocok di sini. Pertama, ada terlalu sedikit kontrol, dan joystick tidak cukup akurat. Kedua, di sana Anda perlu terlalu banyak perhatian dan berpikir dengan kepala Anda dalam proses - jangan lupa, kami tidak bermain mainan di sini, tetapi mengayuh dengan kuat, pertama-tama. Yah dan yang paling penting, itu semua sangat menyedihkan siapa pun yang melihat skema warnanya akan mengerti. Itu perlu sesuatu jenis arcade, sederhana,terang dan terang.

Pada akhirnya, saya memilih Sky Rogue ( store.steampowered.com/app/381020/Sky_Rogue ). Menariknya, promotor ruang angkasa terkenal Scott Manley memiliki ulasannya ( youtu.be/bRCsB5QoxEM ). Sederhana, tetapi cerah, bergaya untuk grafis 8-bit, persyaratan minimal, medan yang dihasilkan secara algoritmik (tidak berulang), aksi penyegaran, senjata tanpa akhir - apa lagi yang diperlukan untuk gerakan tanpa henti selama setengah jam? Sky Rogue lebih cocok untuk gamepad, tetapi, untungnya, Steam dalam mode layar penuh memungkinkan Anda untuk menetapkan kembali kontrol joystick untuk meniru gamepad.



Sekarang sentuhan terakhir tetap - korps. Bermain dengan papan tempat memotong roti di tangan, ternyata, adalah mungkin - tetapi agak merepotkan. Autodesk Fusion 360 di satu tangan, printer 3D di tangan lain, pergi! Tetapi pertama-tama Anda harus berpikir sedikit. Kontroler harus dikontrol dengan satu tangan, yang lain harus memegang simulator. Ini akan memalukan isolasi diri untuk dibawa pergi dengan mainan dan mengacau dengan simulator. Bentuknya harus simetris sehingga Anda bisa menggunakannya dengan tangan apa pun. Ya, diinginkan agar mudah mencetak dan memasang - dalam isolasi sendiri, tidak semua orang akan menemukan tiga sekrup M2 identik dengan panjang yang diinginkan.



Desain gambar dua bagian direkatkan. Pasti ada lem instan di mana-mana. Untuk penyelarasan kedua bagian dengan cepat, sampai lem mengering, pin pemandu digunakan, yaitu potongan material 5mm untuk printer. Tombol dicetak secara terpisah dan dimasukkan ke dalam rumahan selama perakitan. Ia tidak memiliki sumbu, berguling-guling di sekitar permukaan papan sirkuit cetak, dan ditekan ke bawah oleh bagian bawah casing. Tentu saja, saya agak terbawa dengan pemilihan bentuk yang paling nyaman. Versi pertama, dibuat persis dengan ukuran komponen, terlalu kecil untuk dipegang dengan nyaman.

Sekitar setengah biaya - Arduino Micro, komponen mahal kedua - joystick. Dengan produksi massal akan mungkin untuk membuat tiga kali lebih murah, tetapi saya tidak punya rencana global seperti itu (jika Anda tertarik, tulis).

Tetapi yang paling penting adalah sejak saya membuat controller saya, selama setengah jam di simulator mereka terbang sepenuhnya tanpa disadari. Jaga kesehatan Anda dan tetap sehat!
