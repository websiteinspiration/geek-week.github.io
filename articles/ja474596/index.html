TSD上のアプリケーションと1Cとの通信：HTTPサービスによるエンタープライズ8.3。パート5（メニュー、コンパニオンオブジェクト）


各アプリケーションにはグローバル変数があります。たとえば、そのうちの1つはデータベース名です。これがコトリンウェイの観点からどれだけ正しいかはわかりません。アプリケーションのグローバル変数を設定するために、以下を実行します。新しいプロジェクトを作成しますNavigation Drawer Activity。その中で、メニューを操作する方法を学び、設定を使用してページを作成します。MainActivityの横に、新しいクラスを作成します

class MyApp: Application() {
    companion object {
        var baseUrl = "http://192.168.0.1/unf/hs/inntsd/v1/"
        const val dbName = "mws_db"
        var user1C = "tsd"
        var pass1C = "123123"
    }
}

ではAndroidManifest.xml、タグ、<application行を追加します。android:name=".MyApp"

AndroidManifest.xml
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.domain.myapp">
    <application
        android:name=".MyApp"
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="@string/app_name"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        <activity
            android:name=".MainActivity"
            android:label="@string/app_name"
            android:theme="@style/AppTheme.NoActionBar">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />

                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>

</manifest>


すべて。これで、アプリケーションのどの部分からでも、変数にアクセスして変数を取得MyApp.

し、2番目の部分に進むことができます。メニューで遊んでみましょう。各フラグメントに独自のメニューがあることをすぐに決定しましょう。プロジェクトでは、次のようになります。

解像度


新しいメニューを作成します。New - Menu resource file次のテキストをファイルに入れます。 orderInCategoryメニューの位置を示します。少ないほど高い。10、20、30を思い出してくださいBASIC 。つまり、要素の間に別のアイテムを追加する機会が常にあります。-！>


<?xml version="1.0" encoding="utf-8"?>
<menu xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto">
    <item
        android:id="@+id/gallery_clear"
        android:orderInCategory="10" 
        android:title=""
        app:showAsAction="never" />
    <item
        android:id="@+id/gallery_send"
        android:orderInCategory="20"
        android:title=""
        app:showAsAction="never" />
</menu>

メインアクティビティには独自のメニューがないと考えています。したがって、関数を大胆に削除します

override fun onCreateOptionsMenu(menu: Menu): Boolean {
        // Inflate the menu; this adds items to the action bar if it is present.
        menuInflater.inflate(R.menu.main, menu)
        return true
    }

そして、ファイルでui/gallery/GalleryFragment2つの関数を再定義します。

override fun onCreate(savedInstanceState: Bundle?) {
        setHasOptionsMenu(true) //,      .
        super.onCreate(savedInstanceState)
    }

    override fun onCreateOptionsMenu(menu: Menu, inflater: MenuInflater) {

        inflater.inflate(R.menu.menu_gallery, menu) //  . 

        super.onCreateOptionsMenu(menu, inflater)
    }

そして最後に、関数を追加します 

override fun onOptionsItemSelected(item: MenuItem): Boolean {
        when (item.getItemId()) {
            R.id.gallery_clear -> Log.d("MenuDebug", " ")
            R.id.gallery_send -> Log.d("MenuDebug", " ")
        }
        return super.onOptionsItemSelected(item)
    }

それで全部です。各フラグメントの類推により、独自のメニューを追加できます。
