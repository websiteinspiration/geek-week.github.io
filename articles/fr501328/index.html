Présentation des espaces de code Visual Studio: développement cloud, où que vous soyez

Alors que nous mettons tous en œuvre des pratiques telles que la distance sociale et le travail à distance, les équipes de développement sont devenues plus réparties. Notre propre équipe a fait face à certains défis qui font partie de cette transition, ce qui nous a incités à intensifier nos efforts pour aider les développeurs à rester productifs de n'importe où . Tout d'abord, nous avons simplifié la collaboration à distance en permettant aux utilisateurs de rejoindre des sessions Live Share à partir de leur navigateur . Aujourd'hui, nous sommes heureux de partager avec vous des nouvelles, cette fois sur Visual Studio Online, qui vous aideront à travailler de manière productive, où que vous soyez.



Présentation des espaces de codes Visual Studio


En novembre, nous avons annoncé la sortie de Public Preview Visual Studio Online. Au cours des derniers mois, nous avons reçu un grand nombre de critiques et avons bien travaillé sur l'outil. L'un des commentaires les plus fréquemment cités que nous avons vus est que la possibilité de créer des environnements de développement basés sur le cloud a un effet positif sur plus de facteurs que nous ne le pensions auparavant. En particulier, nos utilisateurs estiment que:

  • Le temps nécessaire pour se connecter à de nouveaux projets (ou de nouvelles tâches dans un projet existant) est beaucoup plus court
  • Les performances de leur environnement de développement basé sur le cloud sont à bien des égards meilleures que dans l'environnement local.
  • La possibilité de basculer facilement entre Visual Studio Code, un éditeur de navigateur ou Visual Studio IDE (en aperçu privé) leur donne la possibilité de travailler à partir de presque n'importe quel appareil.
  • Travailler avec des collègues est facile, car Live Share peut être utilisé pour le développement collaboratif, l'édition et le débogage.

Nous avons appris que les développeurs considèrent Visual Studio Online comme plus qu'un simple «éditeur de navigateur». Ils disent que "les capacités de cet environnement de développement basé sur le cloud en font l'endroit où vous voulez écrire tout votre code."

Pour mieux correspondre à cette humeur et à la vraie valeur du service, nous renommons Visual Studio Online en espaces de codes Visual Studio. Voulez-vous obtenir une expérience cool sur votre grand projet? Faites-le dans l'espace de code. Avez-vous besoin de créer rapidement un prototype d'une nouvelle fonction ou d'effectuer des tâches à court terme (par exemple, travailler avec des demandes de tirage)? Créez un espace de code. Votre espace de code est simplement l'espace de code le plus productif.

Les prix sont désormais plus bas


Nous pensons que Visual Studio Online Codespaces offre la meilleure expérience de développement à distance du marché. Nous souhaitons que le plus de personnes possible aient accès aux espaces de codes et les utilisent, en particulier compte tenu des problèmes auxquels nous sommes tous confrontés actuellement. Pour en faire une réalité, nous avons réduit les prix.

Comme toujours, nous continuerons à vous facturer uniquement pour le temps et les ressources que vous utilisez. Pendant que vous êtes actif dans l'espace de code, il utilise des ressources informatiques Azure. De nouveaux prix pour chaque espace de code actif sont disponibles sur notre page de tarification Azure ou notre calculateur de prix .

Seule la puissance dont vous avez besoin


Enfin, nous annonçons également une nouvelle fonctionnalité aujourd'hui. Bien que les développeurs aiment les performances qu'ils voient dans nos espaces de code cloud, nous avons également découvert qu'il existe de nombreux scénarios qui ne nécessitent pas autant de puissance ou où les développeurs souhaitent exécuter plusieurs petits espaces de code (espaces de code) afin de pouvoir travailler en mode multitâche. .

Pour mieux implémenter ces scénarios et aider les utilisateurs à essayer le développement à distance pour la première fois, nous présentons le type d'instance de base, qui est livré avec deux cœurs virtuels, quatre Go de RAM et 64 Go de stockage SSD. Nous avons fait beaucoup de tests avec cette configuration, et l'expérience est tout aussi merveilleuse que sur nos types de Standard et Premium, mais à moindre coût. Les types d'instances de base sont disponibles dès maintenant.

Développement à distance. Mieux que jamais


Au cours des derniers mois, nous avons ajouté d'autres fonctionnalités intéressantes au service! Mon préféré est le support pour la création de vos propres fichiers ou images Docker . À l'aide de cette fonctionnalité, Codespace peut être adapté à vos spécifications, soit en pointant sur une image à partir d'un registre tel que Docker Hub, soit en plaçant votre propre Dockerfile dans votre référentiel, auquel vous vous connecterez lorsque vous démarrez les espaces de code Visual Studio.

Nous avons également récemment publié une fonctionnalité petite mais importante qui permet aux utilisateurs de redimensionner les instances dans leurs environnements. Cela signifie que vous pouvez passer la plupart de votre temps dans un environnement à faible consommation d'énergie et à moindre coût, puis «augmenter la puissance» uniquement lorsque vous en avez besoin - sans perdre l'état ou avoir à créer un nouvel environnement. Cela prend même en charge le type d'instance de base.

Le dernier ajout récent qui m'a fait plaisir a été l'énorme amélioration que nous avons apportée aux capacités des environnements auto-hébergés. Avec les environnements autonomes, vous pouvez enregistrer n'importe quel ordinateur dans les espaces de codes Visual Studio et vous y connecter à partir de VS Code ou d'un navigateur. De plus, il est 100% gratuit. Mon collègue Allison a décrit tous les détails. - N'oubliez pas de lire son post pour plus d'informations.
