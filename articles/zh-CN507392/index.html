在Linux和Windows操作系统下都可以使用用于开发自定义应用程序并将其加载到模块中的机制。在本文中，我们将详细研究如何使用SIMCom无线解决方案提供的SDK中的示例来编译用户应用程序并将其加载到模块中。

序幕



通常，蜂窝通信模块仅用于数据传输，语音呼叫，SMS等。所有这些都是通过从外部控制微控制器发送的AT命令来完成的。但是有一类模块，可让您执行从外部加载的自定义代码。在某些情况下，这大大降低了设备的总体预算，使您可以在板上放置一个更简单（以及预算）的微控制器，也可以完全放弃它。随着由Android或Linux控制的LTE模块及其强大资源的出现，您可以解决流行处理器可用的任何问题。本文将讨论由Linux管理的SIM7600E-H。我们将研究如何下载和运行可执行应用程序。



在许多方面，该材料基于文档“ SIM7600 Open Linux development quide”，但其中的一些补充（尤其是俄语版本）将非常有用。本文将帮助刚开始学习该模块的人员下载演示应用程序，并提供进行进一步工作所需的技能。



简要介绍谁是SIM7600E-H



SIM7600E-H是基于高通公司的ARM Cortex-A7 1.3GHz处理器的模块，内部装有Linux操作系统（内核3.18.20），能够与欧洲（包括俄罗斯）频率范围2G / 3G / Cat.4支持的LTE，最大下载速度高达150Mbps，上传速度高达50Mbps。丰富的外围设备，工业温度范围以及集成的GPS / GLONASS导航功能满足了M2M主题对现代模块化解决方案的所有要求。



系统总览



SIM7600E-H模块基于Linux操作系统（内核3.18.20）。反过来，文件系统基于未分类的块映像文件系统（UBIFS）日记文件系统。



该文件系统的重要功能包括：



  • 与分区配合使用，允许您创建，删除或调整分区大小；
  • 在整个介质体积上提供记录对齐；
  • 与坏块一起工作；
  • 使紧急停电或其他故障期间数据丢失的可能性降到最低；
  • 记录。


该描述是从这里进行的，在同一地方更详细地描述了这种文件系统。



那些。这种类型的文件系统非常适合模块的苛刻工作条件和可能的电源问题。但这并不意味着不稳定的电源条件将是模块预期的工作模式，这仅表示设备具有更大的生存能力。

解压并编译SDK



在访问完外壳程序并可以开始使用模块的命令行之后，我们将尝试编译第一个应用程序以加载到模块中。



许多人可能对此有困难！因为 该模块运行在Linux操作系统上，以避免在Windows下编译代码时发生冲突-最好在本机环境-Linux中进行编译。



我们将不讨论在没有Linux的情况下以及如何将其安装在您的计算机上时，如何将其安装在虚拟计算机上。我们将使用VirtualBox，安装Ubuntu版本20.04（在撰写本文时为当前版本），并且已经在其下开始使用编译器，SDK等。



我们切换到Linux环境，然后解压缩从分发服务器收到的存档。 



simcom@VirtualBox:~/Desktop/OpenLinux$ sudo tar -xzf MDM9x07_OL_2U_22_V1.12_191227.tar.gz


转到sim_open_sdk目录并添加环境： 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ cd sim_open_sdk
simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ source sim_crosscompile/sim-crosscompile-env-init


我们保留在同一文件夹中，然后在其中执行后续命令。

如果尚未安装libncurses5-dev库，请安装它：

Python（如果尚未安装）： 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get install python -y

simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get install gcc
