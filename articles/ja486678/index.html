ASP.NET CoreのQuartz

前書き


このテーマについては多くの記事とある種のチュートリアルがあることは知っていますが、公式ドキュメントについても話していませんが、最新のプロジェクトで作業しているときに、あまり言及されていない非常に興味深い問題に遭遇しました。今日は、ASP.NET CoreプロジェクトでDependency InjectionとQuartzを使用する際の問題について説明します。

問題が発生する可能性があるとは思わなかったことから始まり、すぐにさまざまなアプローチを試してみたことをお伝えします。サービスに含まれるすべてのクラスを追加し、DIを介してそれらを使用しました。後で判明しました）、HostedServiceを追加しようとしましたが、機能しませんでした（最後に、Quartzの操作に関する役立つ記事への適切なリンクをいくつか添付します）。私はすでにトリガーに問題があると思っていました-どちらも。この短い記事では、同じ問題を抱えていた可能性のある人々を助けようと努め、私の解決策が彼らの将来の仕事に役立つことを願っています。紹介の最後に、コメントでテクノロジーに精通している人が私が提案したものを改善するのに役立つアドバイスを提供してくれたらとても感謝していることを付け加えたいと思います。

例となるIEmailSenderインタフェースでは、 、手紙をメールで送信するための1つのメソッドを作成します。

public interface IEmailSender
    {
        Task SendEmailAsync(string email, string subject, string message);
    }

次に、このインターフェースを実装するクラスについて説明します。

public class EmailSender : IEmailSender
    {
        
        public Task SendEmailAsync(string email, string subject, string message)
		{
			var from = "****@gmail.com";
			var pass = "****";
            SmtpClient client = new SmtpClient("smtp.gmail.com", 587);
			client.DeliveryMethod = SmtpDeliveryMethod.Network;
			client.UseDefaultCredentials = false;
			client.Credentials = new System.Net.NetworkCredential(from, pass);
			client.EnableSsl = true;
			var mail = new MailMessage(from, email);
			mail.Subject = subject;
			mail.Body = message;
			mail.IsBodyHtml = true;
			return client.SendMailAsync(mail);
		}
    }

次に、クラスDataJob.cs、DataScheduler.cs、JobFactory.csについて説明します。DataJobクラスはIJobインターフェイスを実装します。

 public class DataJob : IJob
    {
        private readonly IServiceScopeFactory serviceScopeFactory;

        public DataJob(IServiceScopeFactory serviceScopeFactory)
        {
            this.serviceScopeFactory = serviceScopeFactory;
        }

        public async Task Execute(IJobExecutionContext context)
        {
            using (var scope = serviceScopeFactory.CreateScope())
            {
                var emailsender = scope.ServiceProvider.GetService<IEmailSender>();
                
                 await emailsender.SendEmailAsync("example@gmail.com","example","hello")
            }
        }
    }

IServiceScopeFactory型のフィールドが表示されるので、サービスをスタートアップから直接取得します。問題を解決し、先に進み、クォーツ自体のShedulerでジョブとトリガーを追加するDataSchedulerクラスを説明するのに役立つのは、このアプローチでした。

public static class DataScheduler
    {
        
        public static async void Start(IServiceProvider serviceProvider)
        {
            IScheduler scheduler = await StdSchedulerFactory.GetDefaultScheduler();
            scheduler.JobFactory = serviceProvider.GetService<JobFactory>();
            await scheduler.Start();

            IJobDetail jobDetail = JobBuilder.Create<DataJob>().Build();
            ITrigger trigger = TriggerBuilder.Create()
                .WithIdentity("MailingTrigger", "default")
                .StartNow()
                .WithSimpleSchedule(x => x
                .WithIntervalInMinutes(1)
                .RepeatForever())
                .Build();

            await scheduler.ScheduleJob(jobDetail, trigger);
        }

そして今、IJobFactoryインターフェースを実装するJobFactoryクラス：

 public class JobFactory : IJobFactory
    {
        protected readonly IServiceScopeFactory serviceScopeFactory;

        
        public JobFactory(IServiceScopeFactory serviceScopeFactory)
        {
            this.serviceScopeFactory = serviceScopeFactory;
        }

        public IJob NewJob(TriggerFiredBundle bundle, IScheduler scheduler)
        {
            using (var scope = serviceScopeFactory.CreateScope())
            {
                var job = scope.ServiceProvider.GetService(bundle.JobDetail.JobType) as IJob;
                return job;
            }
            
        }

        public void ReturnJob(IJob job)
        {
           //Do something if need
        }
    }

ご覧のとおり、実際にはすべての依存関係をserviceScopeFactoryから直接取得しています。すべてがほぼ準備ができており、Programクラスを変更する必要があります。

public class Program
    {
        public static void Main(string[] args)
        {
            var host = BuildWebHost(args);
            using (var scope = host.Services.CreateScope())
            {
                var serviceProvider = scope.ServiceProvider;
                try
                {
                    DataScheduler.Start(serviceProvider);
                }
                catch (Exception)
                {
                    throw;
                }
            }
            host.Run();
        }

        public static IWebHost BuildWebHost(string[] args) =>
           WebHost.CreateDefaultBuilder(args)
               .UseStartup<Startup>()
               .Build();
    }

また、ConfigureServicesメソッドのStartupに以下を追加します。

   services.AddTransient<JobFactory>();
   services.AddScoped<DataJob>();
   services.AddScoped<IEmailSender,EmailSender>();

できました。アプリケーションを起動すると、毎分実行されるタスクが作成されます。値はDataScheduler.Startで変更できます（秒、時間、またはCRONを使用して指定することもできます）。このアプローチの新しいタスクごとに、IJobを実装する新しいクラスを作成し、新しいDataSchedulerタスクを登録する必要があります。新しいタスク用に別のスケジューラクラスを作成することもできます。
