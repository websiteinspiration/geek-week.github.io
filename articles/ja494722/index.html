Windows、Linux、およびmacOS用のMicrosoft PowerShell 7.1の最初のプレビューバージョン

マイクロソフトは、PowerShell 7.1の新しいプレビューバージョンをリリースしました。これは、Windows、Linux、およびmacOSの作業とスクリプト言語を自動化するためのツールです。このバージョンには、PowerShell 7.0にはなかった機能が含まれています。 PowerShell Core 6.xプラットフォームが先行するPowerShell 7.0のパブリックバージョンは、今年3月の初めにリリースされました。PowerShell 7.1プレビューには、PowerShellチームが取り組んでいる新しいモジュールとツールが含まれています。ただし、これがすべてPowerShell 7.1の通常のリリースに当てはまる保証はありません。これは、.NET 5プラットフォームのリリースから1〜2週間後にリリースされる必要があります。これは、PowerShell 7.0以降、Microsoftが新しいバージョンのPowerShellのリリースを調整するためです。 .NETの新しいバージョンのリリーススケジュール。





Microsoftは、PowerShellGet 3.0のプレビューバージョンに取り組んでいます。これは、使いやすさの向上と、PackageManagementとNugetの依存関係の排除を目的としています。さらに、このバージョンはPowerShellスクリプト言語からC＃に切り替わります。最初のプレビューバージョンはすぐに表示され、PowerShellギャラリーで公開されます。

さらに、新しいバージョンのシークレット管理モジュールがPowerShellギャラリーで公開され、PowerShell 5.1およびPowerShell 7がサポートされます。Microsoftは、PowerShell 7.1にこのモジュールを含める可能性を検討しています。

先週、会社はリリースしました異種混合クラウド環境で複雑なPowerShellスクリプトを認証するために使用される秘密と資格情報を管理するために使用できるモジュールの2番目のプレビューバージョン。

このモジュールは、ユーザーがCredential Manager（Windowsの資格情報マネージャー）Windowsなどのある種の安全なデータストレージシステムを使用して機密データをローカルに保存できるようにする一連のコマンドレットを提供します。このモジュールでは、拡張機能を使用して、リモートリポジトリにある認証情報を操作できます。たとえば、Azure Key Vaultなどです。

これまでのところ、このモジュールはWindowsのみをサポートしていますが、Microsoftは次のプレビューで、GNOMEキーリングサービスを介してLinuxサポートを追加する予定です。後でmacOSキーチェーンのサポートを追加する予定です。

さらに、Microsoftは、PowerShell拡張機能のリアルタイムVisual Studioコードリンクに使用されるPSScriptAnalyzer静的コードアナライザーの一部を書き換えます。

Jupyter NotebooksでのPowerShellサポートの作業は継続されます。これは.NET Interactiveプロジェクトの一部です。

マイクロソフトは、PowerShell 7のインストールとアップグレードを簡素化するためのアプローチを模索しています。しかし、現時点では、WindowsにPowerShell 7を含める予定はありません。

さらに、PowerShellの一般的なシステムシェルからのコードフラグメントのコピーと貼り付けを簡略化するシェルの改良作業も進行中です。

ユーザーエクスペリエンスの向上について話す場合、現在は、さまざまな入力補完テクノロジを使用して、ユーザーがコマンドをより速く入力できるようにする方法の問題を調査しています。さらに、コマンドを入力するときにユーザーにコンテキスト支援を提供するタスク、またはシステムの特定の機能の完全なヘルプが考慮されます。これらのメカニズムは有益なはずですが、同時にユーザーの注意をできるだけそらす必要はありません。

最後に、開発者はPowerShellでより多くの色を使用したいと考えています。これにより、大量の出力の読み取りが容易になり、ユーザーに視覚的な手掛かりが与えられます。 PowerShell 7.1には、文字列の色付けのサポートが含まれる場合があります。

さらに、開発者は、展開されたシステムのサイズを縮小する動きの一環として、ユーティリティ、管理、およびセキュリティモジュールを分離する予定です。

一般に、PowerShell 7.1のリリースに近づいているMicrosoftは、この環境でのユーザビリティの向上を目指していることに注意してください。
