Why Blazor, and not standard JavaScript?

In simple words, Blazor technology allows you to create web applications whose client side consists of HTML markup and C # code instead of JavaScript.

Blazor can access the DOM to redraw the page without refreshing the page The
web application on Blazor consists of components. Components are designed to display visual elements such as download indicators or file upload forms. The markup of the elements is stored in a file with the extension .razor, and all the logic in ordinary C # classes with the name {ComponentName} .razor.cs.

The components in Blazor are divided into smart and stupid.

Stupid components are independent of the entire application, they can be plugged into other components, and they can even use other stupid components.

Smart components are disposable, they have to maintain their state and very rarely they are used in other components.

Most often you will have to write smart components.

In Blazor, you can call JavaScript functions using C # code, and vice versa, you can call .Net methods using JavaScript.

Yes, if you have a deep knowledge of JavaScript, are guided in JavaScript frameworks and you just got acquainted with the world of .Net, then it‚Äôs better not to go into Blazor, the technology is relatively new and when you first get to know it, you can get scared. The goal of the article is for people who know how to use C #, but for some reason who don‚Äôt have JavaScript, talk about its promising counterpart from Microsoft, in which you can afford to use all the features of C #.
