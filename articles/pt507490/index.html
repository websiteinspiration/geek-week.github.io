Trabalhando com código: três ferramentas abertas

Vamos falar sobre alguns editores de texto e um utilitário para projetar código na documentação.



Mais ferramentas de desenvolvedor: APIs e elementos da interface do usuário

Micro



Este é um editor de texto terminal, aprimorado pelos sistemas modernos. O código fonte está no GitHub e é distribuído sob a licença MIT . Os desenvolvedores dizem que o produto é o sucessor ideológico do editor Nano para sistemas operacionais semelhantes ao Unix.



Entre as micro funções interessantes, podemos destacar o suporte para o mouse (arrastar, clicar duas vezes, clicar tripple) e vários cursores, vinculação automática do código (verificação de erros) e destaque de sintaxe para mais de 90 idiomas . Em um tópico temático do Hacker News, os moradores observam que o utilitário é semelhante ao editor de texto sublime proprietárioe possui muitas de suas funções: edição em várias linhas, gerenciamento de plugins - até as teclas de atalho são semelhantes.



A ferramenta apareceu há cerca de três anos, portanto ainda são encontrados erros em seu trabalho . Embora um dos residentes de HN tenha observado que recentemente a microestabilidade cresceu. De qualquer forma, a ferramenta possui uma comunidade (mais de 14 mil estrelas no GitHub) e há uma comunicação ativa no chat do Gitter .





Átomo



É um editor de texto de plataforma cruzada e uma ferramenta de desenvolvimento fornecida pelo GitHub. Ele é construído em HTML, JavaScript, CSS, Node.js e usa a estrutura Electron .



Ele oferece configurações abrangentes para a interface e o texto - você pode alterar os estilos, as combinações de teclas e até a aparência do editor, se souber JavaScript. Quase todas as configurações estão em arquivos CSON (formato JSON, mas para objetos CoffeeScript). Como o GitHub está desenvolvendo o Atom, ele possui uma extensão que permite acessar o sistema de controle de versão diretamente de um editor de texto - gerenciar ramificações, executar comandos de confirmação , push e pull . Há também um pacote Teletype- Usando-o, o código pode ser escrito por vários desenvolvedores ao mesmo tempo.





Captura de tela: atom.io / Trabalhando com Git no Atom



Atom é uma ferramenta relativamente jovem, portanto existem problemas de desempenho, no entanto, os autores estão envolvidos na otimização .





Carbono



A ferramenta permite criar belas capturas de tela de código para apresentações, artigos ou documentação. Basta copiar uma parte do programa no campo do site e salvar a imagem (se desejar, você pode postar a foto diretamente no Twitter). O engenheiro Brian Dennis, da FOSSA , um sistema de gerenciamento de aplicativos de código aberto para toda a empresa, está envolvido no desenvolvimento do Carbon .


Usando o Carbon, você pode escolher uma cor de fundo e ajustar o realce da sintaxe - são suportados mais de 30 idiomas e estruturas: de C e Python a Vue e R. Aqui está um exemplo de uma imagem que pode ser gerada em um site (o código é um “esboço” oferecido pelo Carbon):

Uma comunidade bastante grande se formou no local em torno do instrumento - mais de 24 mil estrelas . Seus membros estão desenvolvendo extensões personalizadas. Por exemplo, existem plugins para integrar o Carbon aos editores de texto Emacs , Vim e o já mencionado Atom.
