ARM64アーキテクチャとの互換性のためのUnity Androidプロジェクトの更新

Googleは、開発者にARM64アーキテクチャとの互換性のために、これらの要件を満たさない場合、2019年8月1日までにアプリケーションを更新するよう招待しました


UnityでARM64サポートを見つけて有効にする


これを有効にするには、ファイル-ビルド設定を開き、次にAndroidプラットフォームのプレーヤー設定を開きます。開いたインスペクタで、[その他の設定]タブの[構成]セクションを確認します。



注意！ Scripting BackendオプションをIL2CPPに設定する必要があります。このオプションを有効にした後、Android NDKがインストールされていない場合は、インストールする必要があります。

最新の安定したバージョンをダウンロードし、そのパスを指定します-そして、何らかの理由でUnityのNDKの最新バージョンである "おっと"が適合しませんでした！



私のように、20番の最新の安定版をダウンロードするのに時間を無駄にしないでください。このパッケージのサイズは約2Gbです。r16b（64ビット）バージョンのダウンロードを開始します。

この特定の必須バージョンのWindows用NDK Android r16b（64ビット）は、
こちらからダウンロードできます。

もちろん、それを開梱して、信頼できるアクセス可能な場所に置き、ビルドを組み立てるときに、この場所へのパスを示します。または、このパスをUnityのメイン設定にすぐに登録します。NDKへのパスを指定するためのフィールドがあり、メニューは[外部コンポーネント（外部ツール）]と呼ばれます。

その結果、ターゲットアーキテクチャオプションでARM64フラグがアクティブになるはずです。これらの操作の前は、フラグは非アクティブでした。



プロジェクトでVuforiaを使用している場合は、更新する必要もあります。


Vuforiaを使用するプロジェクト-以前のバージョンの64xはサポートしていませんでした。ただし、バージョン8.1以降、Vuforiaは64ビットをサポートしています。

Unityから直接更新するのが最適です。これを行うには、ウィンドウメニューに移動します-Vuforia構成（Ctrl-Shift-V）-インスペクターで一番上を確認します-更新が利用可能でVuforiaのバージョンが最新でない場合、ダウンロードリンクが利用可能です-実行可能な更新ファイルをダウンロードできます。

更新を確認する2つ目の方法は、[ヘルプ]メニュー-[Vuforia Engine]-[更新の確認]を開くことです。また、アップデートがある場合は、ダウンロードで提供されます。

ダウンロード後、解凍します。内部には、UnitySetup-Vuforia-AR-Support-for-Editor-2018.4-2019.1.exeとほぼ同じ名前の実行可能ファイルがあるはずです。。エディターのバージョンは異なる場合があります。

次に、この更新を実行するときの正しい手順に注意してください。

  1. ライセンス契約に同意する
  2. Unityエディターが開いている場合、それを閉じるように提案されます
  3. Unityエディターへのパスを選択し、[更新]をクリックします

さらに、Unity Hubがインストールされている場合、パスは「C：\ Program Files \ Unity \ Hub \ Editor \ 2018.4.3f1 \」のようになります。

パスに注意してください。すべてがうまくいきます。Unity.exeが直接あるフォルダへのパスを指定すると、「Unity.exeが見つかりません」というエラーが表示されました。正しいパスは、エディターのバージョンの名前にのみ示されています！

別の熊手がここにあるかもしれません。おそらくあなた自身がVuforiaウェブサイトでVuforia SDK Engine 8.3へのリンクを見つけるでしょう：developer.vuforia.com/downloads。ただし、SDKソースを含むzipアーカイブはここからダウンロードされ、Unityに自動的にインストールすることはできません。このアーカイブを使用しないでください！エディターのリンクからのみ更新ファイルをダウンロードします。

Windowsがインストールされている場合、すべての推奨事項はVuforiaの更新に適用されます。Macを使用している場合は、Unityエディターでフォルダーへのパスを書き込むことを除いて、いくつかの違いがあります。そしてもちろん、プラットフォームのパッケージのバージョンを正しく選択してください。
