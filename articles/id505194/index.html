Windows Defender SmartScreen membahayakan pengembang independen

Misalkan Anda adalah pengembang independen dan telah merilis aplikasi Anda. Untuk menyederhanakan instalasi, Anda membuat installer dan mulai mendistribusikannya. Beberapa unduhan pelaku eksperimen yang berani, meluncurkan penginstal - tetapi melihat peringatan ketat:


Windows SmartScreen memblokir peluncuran file yang dapat dieksekusi

Memang, di lingkungan saat ini, Windows secara aktif memblokir peluncuran file biner melalui fungsi SmartScreen.

Tapi apa itu smartscreen?


SmartScreen mengumpulkan data instalasi dari semua pengguna Windows untuk menentukan "reputasi" aplikasi. Jika program tidak memiliki reputasi yang baik, Anda menerima peringatan seperti itu. Sebagian besar pengguna dalam situasi ini akan menghapus .exe, menganggapnya sebagai program jahat, tetapi Anda dapat memotong SmartScreen dengan mengklik tombol "Detail" dan kemudian "Run Anyway".

Racket Bertanda Tangan Digital


Tapi bagaimana cara membangun reputasi? Pertama-tama, Microsoft harus dapat mengumpulkan informasi tentang siapa yang menerbitkan aplikasi, dan ini dilakukan dengan menggunakan sertifikat penandatanganan kode. Kesimpulan yang paling jelas adalah bahwa aplikasi yang tidak ditandatangani akan selalu memanggil SmartScreen. Kesimpulan yang lebih berbahaya adalah bahwa memperoleh sertifikat penandatanganan kode merupakan biaya besar bagi pengembang individu. Saat ini tidak ada yang setara dengan Let's Encrypt untuk menandatangani kode, jadi Anda harus membeli sertifikat dari otoritas sertifikasi terkemuka. Kisaran harga sangat lebar, tetapi sertifikat satu tahun biasanya berharga sekitar $ 100.


SSL.com menawarkan salah satu sertifikat penandatanganan kode "termurah" di pasar, dengan harga $ 129 per tahun.

Misalkan Anda mematuki sebuah kail, membeli sendiri sejumlah besar bilangan prima yang dikalikan dengan komputer, menandatangani kode Anda, dan menerbitkan kembali aplikasi Anda. Sekarang pengguna akan mulai menginstal aplikasi Anda, bukan? Salah.

Membangun reputasi adalah tipuan 22


Bahkan dengan tanda tangan digital, SmartScreen akan tetap berfungsi. Bagaimanapun, Anda adalah penerbit baru yang tidak dikenal, dan proses "membangun reputasi" SmartScreen adalah kotak hitam mutlak. Jadi, inilah trik 22: untuk membuat reputasi, Anda memerlukan orang untuk menginstal perangkat lunak Anda - dan Microsoft akan mengumpulkan data dengan statistik pemasangan. Tetapi untuk membuat pengguna menginstal perangkat lunak Anda, Anda perlu menyingkirkan pesan yang dengan jelas menunjukkan bahwa kode Anda akan merusak komputer mereka.

Lebih buruk lagi.

Jika Anda mencoba untuk mempublikasikan perangkat lunak Anda menggunakan manajer paket WinGet (singkatnya: ketika meninggalkan versi beta, manajer ini disalin dari AppGet mungkin akan menggantikan Windows Store), Anda akan menerima pesan berikut dari Microsoft:


Microsoft WinGet menganggap meluncurkan SmartScreen sebagai "kesalahan" dan perangkat lunak Anda diklasifikasikan sebagai "jahat."

Menurut Microsoft, aplikasi Anda dianggap berbahaya jika meluncurkan SmartScreen. Mereka merekomendasikan pengiriman file untuk ditinjau oleh tautan khusus. Jika Anda mengikuti proses ini, Anda akan mendapatkan jawaban berikut:

Intinya: aplikasi Anda bersih, tetapi kami tidak akan melakukan apa pun dengan kesalahan SmartScreen. Tuan cloud bot kami akan memutuskan kapan Anda bisa dipercaya.

Dan lebih buruk lagi.

Katakanlah Anda akhirnya disetujui oleh algoritme sebagai penerbit yang andal. Sertifikat Anda akan kedaluwarsa dan Anda akan memperbaruinya. Suatu hal yang umum di Internet. Tetapi di dunia perangkat lunak Windows, reputasi Anda akan dibatalkan. SmartScreen tidak mengenali pembaruan sertifikat, itu menganggap Anda sebagai penerbit baru, dan Anda harus melalui proses yang menyakitkan ini lagi.

Ini dapat dikurangi dengan membeli sertifikat dengan masa berlaku lebih lama, tetapi kemudian biaya menjadi penghalang yang tidak dapat diatasi untuk sebagian besar pengembang independen.

Sertifikat Penandatanganan Kode EV


Untuk pengembang independen, ini adalah ujung jalan. Untuk penerbit, ada solusi lain: sertifikat verifikasi lanjutan untuk penandatanganan kode (EV). Dengan menandatangani kode menggunakan salah satunya, Anda secara otomatis memotong SmartScreen. Ini disediakan hanya untuk perusahaan dan, seperti namanya, menyediakan verifikasi informasi tambahan. Ini dilakukan oleh pusat sertifikasi.

Otoritas sertifikasi populer Digicert.com menjual sertifikat semacam itu dengan harga $ 699 per tahun.

Pengembang independen harus melalui kesulitan membentuk badan hukum dan membayar dengan harga yang sangat tinggi untuk sertifikat ini. Tentu saja, bagi kebanyakan orang ini tidak realistis.

"Pengembang, pengembang, pengembang!"




"Pengembang, pengembang, pengembang!" Teriak Steve Ballmer dari panggung. Salah satu pidato legendaris dari CEO Microsoft. Kata-kata terkenal ini diucapkan kembali pada tahun 2006. Empat belas tahun kemudian, Microsoft di Sathya Nadella dipuji karena membuat perusahaan lebih terbuka dari sebelumnya. Namun ironisnya, ternyata Microsoft telah membuat lingkungan Windows-nya sangat memusuhi pengembang favoritnya. Untuk memperbaiki situasi yang mengerikan ini, Anda perlu mengubah SmartScreen atau cara kerja sertifikat penandatanganan kode.
