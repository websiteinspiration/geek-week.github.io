Intersection Observer APIを使用した遅延読み込み

こんにちは！

この記事では、Intersection Observer APIを紹介します。これは、ページ上の画像やその他の要素を遅延ロードするための強力な新しいツールであり、最新のすべてのブラウザーでサポートされています。このツールを使用して、DOMコンテンツの事前ロードまたは遅延ロードを定義することにより、要素の可視性を制御できます。

Webの進化に伴い、ブラウザーがサポートする新機能やAPIは増え続けていますが、開発者にとってはDOMレンダリングという面倒な点があります。新しいアプリケーションの開発、コンポーネントの作成、HTML構造の検討、CSSによるスタイル要素の検討、レンダリングプロセス、ページの一般的な外観、および慎重な処理と影響を必要とするその他の重要な事項についても検討しますアプリケーションのレンダリング。

これは、要素の作成方法、CSSの編成方法、プロジェクトで使用されるライブラリとフレームワークなどに影響します。このため、開発者にとってレンダリングは非常に重要です。

開発者コミュニティーでReactライブラリーが人気の理由の1つは、仮想DOMを使用していることです。本質的に、これは実際のDOMの仮想表現であり、必要に応じて制御できます。すべての更新後、仮想DOMは元のDOMに必要な変更を決定し、これらの変更を「シームレス」な方法で行います。 1つの要素を変更するときにページ全体を更新する必要はありません。これは、他のライブラリやフレームワークに比べてReactの最も重要な利点の1つです。 DOMレンダリングはコストがかかる（パフォーマンスの点で）タスクであり、Reactは問題なく問題なく実行します。

一部のライブラリとAPIを使用すると、アプリケーションの要素のレンダリングを改善できます。それらの1つであるIntersection Observer APIについて理解します。しかし、最初に、「遅延」画像のロードとは何かを理解しましょう。

画像の遅延読み込みはどのように機能しますか？


遅延読み込みは、ページコンテンツをレンダリングする手法の1つです。 「遅延」ロードの概念は、ページを完全にロードするのではなく、必要なパーツのみをロードし、ユーザーが必要とするまで残りのパーツのロードを延期（遅延）することです。

「遅延」ダウンロードの例は、おそらく聞いたことがあるか、または遭遇したことですが、ページの無限スクロールです。これは次のように機能します。ユーザーがページにアクセスして最初のコンテンツが読み込まれ、ユーザーがページを最後までスクロールした後にのみ、コンテンツの次の部分が読み込まれます。この場合、すべてのコンテンツを一度にダウンロードする必要がないため、アプリケーションが大幅に簡素化されます。

「遅延」ダウンロードが「ユーザー生成」コンテンツのみをダウンロードするのに役立つ場合、反対の手法は何と呼ばれますか？以前に遅延読み込みについて聞いたことがありますが、その逆は聞いていません。 「遅延」ロードの反対は「エネルギー」ロード（他のオプションはせっかちで動きが速い）です-すべてのコンテンツがすぐにレンダリングされます。

画像の遅延読み込みにHTML属性があることを知っていますか？それらを使用するには、loading =“ lazy”を“ img”または“ iframe”要素に追加するだけです。



この手法を使用すると、多くの利点が得られます。必要なリソースのみをロードすると、メモリの消費量が削減され、最終的にはページのロード時間が短縮されます。また、「要求されていない」コードのレンダリングを防止します。これにより、ネットワーク帯域幅が低い状態で動作するようにアプリケーションが最適化されます。ただし、いくつかの欠点があります。これは、検索エンジンによるページのランキングに影響を与える可能性があります。また、適切なライブラリまたはAPIを選択するには、しばらく時間がかかります。

交差オブザーバーAPI


JavaScriptは急速に開発が進んでいるため、ほぼ毎年新しいAPIを入手しています。新しいAPIの主な目的は、すばらしいアプリケーションとページの作成を支援することです。

Intersection Observerは、DOM要素の可視性と位置を監視できるAPIです。非同期で動作し、要素の可視性をスムーズに（いつの間にか）変更し、DOMのコンテンツの事前のロードまたは遅延したロードを決定できることが重要です。



数年前、要素の配置と追跡を行うためのAPIまたはその他のツールがありませんでした。クレイジーで巨大な関数を作成し、より多くの問題を引き起こす高価な（パフォーマンスの観点から）メソッドを作成する必要がありました。 Intersection Observer APIを使用すると、次のようなすばらしいことができます。

  • エンドレスページスクロール-テーブル、リスト、グリッドコンテナーなどのエンドレススクロールを作成します。それはそれほど簡単ではありませんでした。
  • 画像-「遅延」ダウンロードを使用して、現時点でユーザーが必要とするコンテンツのみを表示できるときに、一度にすべての画像をアップロードするのはなぜですか？
  • 要素の監視-ページ上の要素を監視できます。たとえば、広告の閉鎖に関するメッセージを受け取る場合があります。

ここからが始まりです：

let myFirstObserver = new IntersectionObserver(callback, options)

IntersectionObserverコンストラクタを初期化し、2つのパラメータを渡します。最初のパラメータはコールバック関数であり、ビューポートとの要素の交差（つまり、要素がビューポートの境界を超えた場合）または別の要素との交差に応答して呼び出されます。オプションパラメータは、コールバック関数を起動するための条件を制御するオブジェクトです。このオブジェクトには次のフィールドがあります。

  • root-ターゲット要素（監視している要素の）のビューポートとして機能する親要素
  • rootMargin-ルート要素の周囲の外側マージン（このフィールドの値はCSSのマージン値と同様です）。これらの値は、交差を確定する前にルートの両側で変更できます。
  • しきい値-ターゲット要素とルート要素との共通部分の割合を決定する数値または数値の配列。ここで、コールバック関数が起動されます。



Intersection Observer APIの使い方を見てみましょう。

let options = {
    root: document.querySelector('.scroll-list'),
    rootMargin: '5px',
    threshold: 0.5
}

let myFirstObserver = new IntersectionObserver(elements => { elements.forEach(element => console.log('element', element)) }, options)

要素の「監視」を開始するには、ターゲットを特定し、「observe」メソッドを使用してオブザーバーを呼び出し、ターゲットをパラメーターとして渡す必要があります。

let myDivTarget = document.querySelector('.header')

observer.observe(myDivTarget)

観測者が観測するすべての要素は長方形であることを覚えておいてください。不規則な形状の要素は、最小の長方形に収まります。

ブラウザのサポートはどうですか？Intersection Observer APIは、Chrome、Edge、Firefox、Operaなどのすべての最新ブラウザーでサポートされています。



たとえば、多くの情報を含むテーブルなど、大量のデータを処理する場合は、Intersection Observer APIが非常に役立ちます。

このAPIの利点の1つは、そのような問題を解決するためにライブラリを使用する必要がなくなることです。

結論


Intersection Observer APIと、遅延読み込み、DOM要素の位置の監視などへの使用について調査しました。ブラウザーは、主に接続速度が遅いユーザー向けにページのレンダリング時間を短縮し、プロジェクトのサイズを縮小して関連コンテンツのみを表示するAPIをますますサポートしています。
