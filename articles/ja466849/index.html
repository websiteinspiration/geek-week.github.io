noexcept-ctcheckまたはコンパイラーがnoexceptコードを作成するのに役立ついくつかの単純なマクロ

C ++で開発する場合、例外が発生してはならないコードを時々書く必要があります。たとえば、ネイティブタイプの例外のないスワップを記述する必要がある場合や、クラスにnoexcept moveステートメントを定義する必要がある場合、または重要なデストラクタを手動で実装する場合などです。


C ++ 11では、noexcept修飾子が言語に追加されました。これにより、開発者はnoexceptでマークされた関数（またはメソッド）からの例外をスローできないことを理解できます。したがって、そのようなマークが付いた関数は、例外が発生してはならないコンテキストで安全に使用できます。


たとえば、次のタイプと関数があるとします。


class first_resource {...};
class second_resource {...};

void release(first_resource & r) noexcept;
void close(second_resource & r);

そしてそこに特定のクラスであるresources_owner型のオブジェクトを所有しているfirst_resourceとsecond_resource：


class resources_owner {
   first_resource first_resource_;
   second_resource second_resource_;
   ...
};

次に、デストラクタresources_ownerを次のように記述できます。


resources_owner::~resources_owner() noexcept {
   //  release()   ,    .
   release(first_resource_);

   //    close()   , 
   //   try-catch.
   try{ close(second_resource_); } catch(...) {}
}

               wg.invoke_after_write_notificator_if_exists( ec );
