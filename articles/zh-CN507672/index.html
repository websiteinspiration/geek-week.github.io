使用PowerShell修改和删除Azure VM

使用PowerShell，工程师和IT管理员不仅可以在内部部署中使用云基础架构（尤其是在Azure中），还可以成功地自动执行各种任务。在某些情况下，通过PowerShell进行操作比通过Azure门户更加方便，快捷。凭借其跨平台功能，PowerShell可用于任何操作系统。



无论您使用Ubuntu，Red Hat还是Windows，都没关系-PowerShell将帮助控制云资源。例如，使用Azure PowerShell模块，可以设置虚拟机的任何属性。



在本文中，我们将研究如何使用PowerShell调整Azure云中VM的大小，以及如何删除VM及其关联的对象。







重要！不要忘记用消毒剂擦拭手准备工作：



  • 您将需要Azure PowerShell模块 -您可以使用命令从PowerShell库下载它Install-Module Az



    。
  • 您需要通过运行命令在虚拟机正在旋转的Azure云中进行身份验证Connect-AzAccount



    。


首先，创建一个调整Azure VM大小的脚本。打开VS Code，然后保存一个名为Resize-AzVirtualMachine.ps1的新PowerShell脚本-在执行示例时，我们将向其中添加代码。



我们要求可用的VM大小



在更改VM的大小之前，您需要确定Azure云中的虚拟机通常可接受的大小。为此，请运行命令Get-AzVMSize



。



因此，对于dev资源组中的devvm01虚拟机，我们要求各种允许的大小： 



Get-AzVMSize -ResourceGroupName dev -VMName devvm01


（当然，在实际任务中，您将为这些参数指定值而不是ResourceGroupName = dev和VMName = devvm01）。



该命令将返回如下列表：







这些都是可以为此虚拟机设置的所有可能的大小。



调整尺寸



例如，我们将调整大小为新大小Standard_B1ls-在上面的列表中位于第一位。（当然，在实际任务中，您可以选择所需的任何大小。）



  1. 首先，使用命令，Get-AzVM



    获取有关对象（虚拟机）的信息，并将其保存在变量中$virtualMachine



    ： 



    $virtualMachine = Get-AzVM -ResourceGroupName dev -VMName devvm01
  2. 然后，我们从该对象获取一个属性.HardwareProfile.VmSize



    并设置所需的新值： 



    $virtualMachine.HardwareProfile.VmSize = "Standard_B1ls"
  3. 现在只需运行更新命令VM- Update-AzVm



    ： 



    Update-AzVM -VM devvm01 -ResourceGroupName dev
  4. 我们确保一切顺利-为此，我们再次请求有关对象的信息并查看属性$virtualMachine.HardwareProfile



    ： 



    $virtualMachine = Get-AzVM -ResourceGroupName dev -VMName devvm01
$virtualMachine.HardwareProfile


如果我们在那里看到Standard_B1ls，那么一切都井井有条，机器的大小已更改。您可以通过使用阵列一次调整多个VM的大小来进一步取得成功。



在Azure中删除VM怎么办？



删除后，并非一切看起来都像看起来那样简单明了。毕竟，您需要删除与该计算机关联的许多资源，包括：



  • 存储诊断存储容器
  • 网络接口
  • 公用IP地址
  • 存储其状态的系统磁盘和Blob
  • 数据盘


因此，我们将创建一个函数并将其命名Remove-AzrVirtualMachine



-它不仅会删除Azure VM，还会删除所有上述内容。



我们遵循标准路径，首先我们使用command获得对象（VM）Get-AzVm



。例如，让它成为机器WINSRV19从资源组MyTestVMs。



我们将此对象及其所有属性保存在变量中$vm



： 



$vm = Get-AzVm -Name WINSRV19 -ResourceGroupName MyTestVMs


删除带有启动诊断文件的容器



在Azure中创建VM时，还会邀请用户创建一个用于存储启动诊断程序的容器（启动诊断程序容器），这样，如果启动存在问题，则可以进行一些故障排除。但是，删除虚拟机后，此容器将保留其现在毫无目的的存在。纠正这种情况。



  1. 首先，找出此容器属于哪个存储帐户-为此，我们需要storageUri



    在DiagnosticsProfile



    VM 对象的内容中找到该属性。为此，我使用以下正则表达式： 



    $diagSa = [regex]::match($vm.DiagnosticsProfile.bootDiagnostics.storageUri, '^http[s]?://(.+?)\\.').groups[1].value
  2. 现在我们需要找出容器的名称，为此，我们需要使用以下命令获取VM ID Get-AzResource



    ： 



    
if ($vm.Name.Length -gt 9) {
    $i = 9
} else {
    $i = $vm.Name.Length - 1
}
 
$azResourceParams = @{
    'ResourceName' = WINSRV
    'ResourceType' = 'Microsoft.Compute/virtualMachines'
    'ResourceGroupName' = MyTestVMs
}
 
$vmResource = Get-AzResource @azResourceParams
$vmId = $vmResource.Properties.VmId
$diagContainerName = ('bootdiagnostics-{0}-{1}' -f $vm.Name.ToLower().Substring(0, $i), $vmId)
  3. 接下来，我们获得容器所属的资源组的名称： 



    $diagSaRg = (Get-AzStorageAccount | where { $_.StorageAccountName -eq $diagSa }).ResourceGroupName
  4. 现在我们有了使用命令删除容器所需的一切Remove-AzStorageContainer



    ： 



    $saParams = @{
    'ResourceGroupName' = $diagSaRg
    'Name' = $diagSa
}
 
Get-AzStorageAccount @saParams | Get-AzStorageContainer | where { $_.Name-eq $diagContainerName } | Remove-AzStorageContainer -Force


删除虚拟机



现在我们将删除虚拟机本身，因为我们已经$vm



为相应的对象创建了一个变量。好吧，运行命令Remove-AzVm



： 



$null = $vm | Remove-AzVM -Force


删除网络接口和公用IP地址



我们的VM具有一个（或什至几个）网络接口（NIC）-删除它们是不必要的，请遍历NetworkInterfaces



我们的VM对象的属性并删除NIC命令Remove-AzNetworkInterface



。如果存在多个网络接口，则使用循环。同时，对于每个NIC，我们检查属性IpConfiguration



以查看接口是否具有公共IP地址。如果找到了，请使用命令将其删除Remove-AzPublicIpAddress



。



这是一个这样的代码示例，其中我们遍历所有NIC，删除它们，检查是否有公共IP。如果存在，则我们解析该属性PublicIpAddress



，按ID获取相应资源的名称并将其删除： 




foreach($nicUri in $vm.NetworkProfile.NetworkInterfaces.Id) {
    $nic = Get-AzNetworkInterface -ResourceGroupName $vm.ResourceGroupName -Name $nicUri.Split('/')[-1]
    Remove-AzNetworkInterface -Name $nic.Name -ResourceGroupName $vm.ResourceGroupName -Force

    foreach($ipConfig in $nic.IpConfigurations) {
        if($ipConfig.PublicIpAddress -ne $null) {
            Remove-AzPublicIpAddress -ResourceGroupName $vm.ResourceGroupName -Name $ipConfig.PublicIpAddress.Id.Split('/')[-1] -Force
        }
    }
}


删除系统驱动器



OS磁盘是一个Blob，要删除该磁盘有一个命令Remove-AzStorageBlob



-但是在执行它之前，您需要为其参数设置所需的值。为此，尤其需要获取包含系统磁盘的存储容器的名称，然后将其与相应的存储帐户一起转移到此命令。 



$osDiskUri = $vm.StorageProfile.OSDisk.Vhd.Uri
$osDiskContainerName = $osDiskUri.Split('/')[-2]
$osDiskStorageAcct = Get-AzStorageAccount | where { $_.StorageAccountName -eq $osDiskUri.Split('/')[2].Split('.')[0] }
$osDiskStorageAcct | Remove-AzStorageBlob -Container $osDiskContainerName -Blob $osDiskUri.Split('/')[-1]


删除Blob系统磁盘状态



为此，您可能已经猜到了，我们将存储该磁盘的存储容器作为容器，并假设blob在末尾包含status



，我们将适当的参数传递给delete命令Remove-AzStorageBlob



： 



$osDiskStorageAcct | Get-AzStorageBlob -Container $osDiskContainerName -Blob "$($vm.Name)*.status" | Remove-AzStorageBlob


最后，删除数据光盘



我们的VM可能具有连接了数据的磁盘。如果没有必要，我们也将其删除。首先，我们解析StorageProfile



虚拟机并找到属性Uri



。如果有多个磁盘，我们通过来组织一个循环URI



。对于每个URI，我们将使用找到对应的存储帐户Get-AzStorageAccount



。然后，我们将解析存储URI，以获得所需的Blob名称，并将其Remove-AzStorageBlob



与您的帐户一起传递给delete命令。在代码中的外观如下： 



if ($vm.DataDiskNames.Count -gt 0) {
    foreach ($uri in $vm.StorageProfile.DataDisks.Vhd.Uri) {
        $dataDiskStorageAcct = Get-AzStorageAccount -Name $uri.Split('/')[2].Split('.')[0]
        $dataDiskStorageAcct | Remove-AzStorageBlob -Container $uri.Split('/')[-2] -Blob $uri.Split('/')[-1]
    }
}
