Mengapa daftar tuple berperilaku aneh dengan Python?

Dalam bahasa pemrograman, saya selalu tertarik dengan struktur internal mereka. Bagaimana cara kerja satu atau operator lain? Mengapa lebih baik menulis seperti ini dan bukan sebaliknya? Pertanyaan seperti itu tidak selalu membantu menyelesaikan masalah "di sini dan sekarang", tetapi dalam jangka panjang mereka membentuk gambaran keseluruhan dari bahasa pemrograman. Hari ini saya ingin membagikan hasil dari salah satu penyelaman ini dan menjawab pertanyaan tentang apa yang terjadi ketika memodifikasi tuple'a in list' e.

Kita semua tahu bahwa Python memiliki tipe data list:

a = []
a.append(2)

listHanya sebuah array. Ini memungkinkan Anda untuk menambah, menghapus, dan memodifikasi item. Ini juga mendukung berbagai operator menarik. Misalnya, operator + = untuk menambahkan item list. +=mengubah daftar saat ini, tetapi tidak membuat yang baru. Ini terlihat jelas di sini:

>>> a = [1,2]
>>> id(a)
4543025032
>>> a += [3,4]
>>> id(a)
4543025032

Python memiliki tipe data hebat lainnya: tuple- koleksi yang tidak dapat diubah. Itu tidak mengizinkan penambahan, penghapusan atau perubahan elemen:

>>> a = (1,2)
>>> a[1] = 3
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

Saat menggunakan operator +=, yang baru dibuat tuple:

>>> a = (1,2)
>>> id(a)
4536192840
>>> a += (3,4)
>>> id(a)
4542883144

Perhatian, pertanyaannya adalah: apa yang akan dilakukan kode berikut?

a = (1,2,[3,4])
a[2] += [4,5]

Pilihan:

  1. Item ditambahkan ke daftar.
  2. Pengecualian tuple immutability akan dibuang.
  3. Kedua.
  4. Tidak satu atau yang lain.

Tulis jawaban Anda di selembar kertas dan mari kita periksa:

>>> a = (1,2,[3,4])
>>> a[2] += [4,5]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

Baiklah kalau begitu! Jadi kami menemukan jawabannya! Jawaban yang benar adalah 2. Meskipun, tunggu sebentar:

>>> a
(1, 2, [3, 4, 4, 5])

Bahkan, jawaban yang benar adalah 3. Yaitu, unsur-unsur ditambahkan, dan pengecualian keluar - wat ?!


Mari kita lihat mengapa ini terjadi. Dan modul yang luar biasa akan membantu kita dengan ini dis:

import dis

def foo():
    a = (1,2,[3,4])
    a[2] += [4,5]

dis.dis(foo)

Blok pertama bertanggung jawab untuk membangun tuple'dan menyimpannya dalam variabel a. Kemudian kesenangan dimulai:

       21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)

Kami memuat pointer ke variabel adan 2 konstan ke stack .

       27 DUP_TOP_TWO

Gandakan dan letakkan di tumpukan dengan urutan yang sama.

       28 BINARY_SUBSCR

Operator ini mengambil elemen teratas dari tumpukan (TOS) dan yang berikutnya (TOS1). Dan menulis elemen baru ke atas tumpukan TOS = TOS1[TOS]. Jadi kami menghapus dua nilai teratas dari tumpukan dan memasukkan tautan ke elemen kedua tuple'a (array kami) di dalamnya .

       29 LOAD_CONST      4 (4)
       32 LOAD_CONST      5 (5)
       35 BUILD_LIST      2

Kami membuat daftar elemen 4 dan 5 dan meletakkannya di atas tumpukan:

       38 INPLACE_ADD

Berlaku +=untuk dua elemen teratas tumpukan (Penting! Ini adalah dua daftar! Satu terdiri dari 4 dan 5, dan yang lainnya diambil dari tuple). Semuanya baik-baik saja di sini, instruksi dijalankan tanpa kesalahan. Karena ia +=memodifikasi daftar asli, daftar di tuple'e telah berubah (pada saat itu).

       39 ROT_THREE
       40 STORE_SUBSCR

Di sini kita menukar tiga elemen teratas dari tumpukan (tinggal di sana tuple, di dalamnya indeks array dan array baru) dan menulis array baru tupledengan indeks. Di sinilah pengecualian terjadi!

Yah, itu dia! Bahkan, daftar dapat diubah, tetapi semuanya ada pada operator =.

Akhirnya mari kita cari tahu cara menulis ulang kode ini tanpa kecuali. Seperti yang sudah kami pahami, Anda hanya perlu menghapus entri tuple. Berikut ini beberapa opsi:

>>> a = (1,2,[3,4])
>>> b = a[2]
>>> b += [4,5]
>>> a
(1, 2, [3, 4, 4, 5])

>>> a = (1,2,[3,4])
>>> a[2].extend([4,5])
>>> a
(1, 2, [3, 4, 4, 5])
