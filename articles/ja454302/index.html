QVDファイル-内容

QlikViewと彼の弟QlikSenseは、国内および海外で非常に人気のある素晴らしいBIツールです。多くの場合、これらのシステムは作業の「中間」結果（「ダッシュボード」を視覚化するデータ）をいわゆる「QVDファイル」に保存します。多くの場合、QVDファイルは、Qlikに基づいて構築されたマルチステージETLプロセスの主ストレージとして使用されます。そして、いくつか（たとえば、私-会社でデータエンジニアリングの問題を扱っています）に質問があります-QlikView / QlikSenseなしでこのデータを使用することは可能ですか？または他の-そしてそこに何があり、それは正しく「カウント」されていますか？


QVDはQlikView / QlikSense用に最適化されたファイル形式です（これらのアプリケーションによる情報の書き込みからこの形式のファイルへの読み取りは、他の形式のファイルよりもはるかに高速です）。このファイルの構造は文書化されておらず、「所有権の薄暗い」で覆われています。このようなファイルを操作できるアプリケーションはほとんどありません（読み取り、さらには書き込み）。このシリーズの記事では、私の経験と得られた実践的な知識を共有します。QVDがどのように機能するかを知っているので、QVDを直接読み書きできます。


この情報に関心を持つユーザー：まず、QlikView / QlikSenseを使用するユーザー、および（私のような）QVDファイルに保存されているデータを使用したいユーザー。そしてもちろん、好奇心旺盛な皆さんに。


このシリーズで書かれていることはすべて、私の個人的な経験に基づいています。もちろん、これは「ドキュメント」や「保証」ではありません（ファイルが私が説明したものとまったく同じであること、またはそれが永遠に続くこと） ）私がすべてのケースを理解したことを保証することもできません-私が記述していないものを含むファイルがある可能性があります（そのようなオプションに出会わなかった場合に限ります）。ただし、さまざまなバージョンのQlikView / QlikSenseを使用しているさまざまなシステムのさまざまなユーザーが作成した大量（数百）のファイルセットで情報がチェックされることに注意してください。
