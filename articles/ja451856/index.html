私はopenmeetingsを試し始めました。テスト済みで安定版のリリースである他のユーザーのアマチュアとして、私は最新の安定版リリース4.0.8をインストールしました（このプロセスはここでは考慮しません）。彼がフラッシュしていることを除いて、すべてが順調です。まあ、もしそうなら、彼はクロムで働くことを拒否しました、それはフォックスに行きました...しかし、これは段落4と矛盾します、なぜなら 誰もがFFを使用しているわけではなく、誰もがそれを好きというわけではありません。フラッシュなしでバージョン5.0.0-M1が発表されたのを見たとき、私はすでに動揺する時間がありました！これがすべての始まりです。すぐにすべてを開始するのではなく、約2週間、1日1〜2時間、完全な起動にかかったとすぐに言います。

そして、私はubuntu 18.0.4-LTSを装着しました。

  • JRE 8
  • クレントメディアサーバー

JRE8から始めましょう。デフォルトでは、リポジトリから11が置かれるため、リポジトリに追加し、必要なバージョンのインストールを開始します。

sudo add-apt-repository ppa:webupd8team/java
sudo apt-get update
sudo apt-get install oracle-java8-installer

インストール後、実行するJavaのデフォルトバージョンを指定する必要があります。

sudo apt-get install oracle-java8-set-default

バージョンを確認する

java -version

発行する必要があります

java version "1.8.0_201"
Java(TM) SE Runtime Environment (build 1.8.0_201-b09)
Java HotSpot(TM) 64-Bit Server VM (build 25.201-b09, mixed mode)

今はホームディレクトリを設定するために残っています。

cat >> /etc/environment <<EOL
JAVA_HOME=/usr/lib/jvm/java-8-oracle
JRE_HOME=/usr/lib/jvm/java-8-oracle/jre
EOL

ビデオ/オーディオストリームの通常の操作には、クレントメディアサーバー（KMS）が必要です。インストールオプションはいくつかありますが、私はDockerオプションを使用しました。インターネットには情報が満載なので、Dockerのインストールと構成はこの記事には含まれていません。そして、KMSを起動します

docker run -d --name kms -p 8888:8888 kurento/kurento-media-server:latest

次に、関連コンポーネントのインストールに進み
ます。MySQL -OMにはデータベースが組み込まれていますが、戦闘バージョンでの使用はお勧めしません。私たちはあなたに便利なバージョンを置きます。標準のリポジトリから適切です。

sudo apt-get install mysql

JavaをMySQLに接続するには、コネクターをダウンロードして、/ webapps / openmeetings / WEB-INF / lib /フォルダーに配置する必要があります。MySQLの接続設定は、ファイル/webapps/openmeetings/WEB-INF/classes/META-INF/mysql_persistence.xmlに位置している
ImageMagickの、一般的なボードに必要な文書や画像のデモ- 。また、標準のカブから取ります。

sudo apt-get install imagemagick

GhostScript -PDFで作業したい場合は、PDFなしではできません。リポジトリも標準です。

FFMPEGとソックス異なるフォーマットでビデオ会議を記録する可能性のために- 。バージョンは10.3以降である必要があります。

これで、openmeetings自体をダウンロードする準備が整いました。
https://openmeetings.apache.org/downloads.html
ダウンロードし、必要なフォルダに解凍します。

すべてを実行する準備ができているようです（特に公式の指示に従っている場合）が、そのようなリンクがここにありますhttps：// localhost：5443 / openmeetings / install。 httpsとポート5443に注意を払えば、何も起こらないことがわかります。もちろん、スクリプト./bin/startup.shを実行すると、サーバーが起動します。リンクhttp：// localhost：5080 / openmeetings / installを使用してアクセスおよび設定することもできますしかし、通常の動作のみではありません。現在、すべてのブラウザー、特にクロムはユーザーの安全のために戦っており、https経由でのみカメラとマイクを操作できます。 FFを使用すると、ログインしてカメラを機能させることができますが、これでも単一のブラウザーに関連付けられています。したがって、SSLのインストールと構成に進みます。あなたはお金のために証明書を作ることができます、またはあなたはそれを自分で行うことができます、これからの作業はOMを悪化させません。

OMバージョン5.0.0-M1は、ApacheではなくTomCatに基づいています。 Webサーバー構成は./conf/フォルダーにあります。自己署名証明書を作成してTomCateにインストールする方法についてはすでに説明しました。

さて、httpsが設定されたので、。/ binフォルダーに移動してstatup.shを実行し、サーバーの起動後にWebインストーラーに移動しますhttps：// localhost：5443 / openmeetings / install。ここでは、「コンバーター」セクションを除いて、すべてがシンプルで直感的です。ここで、追加でインストールしたパッケージへのパスを登録する必要があります。

  1. ImageMagickパス/ usr / bin
  2. FFMPEGパス/ usr / bin
  3. SoXパス/ usr / bin
  4. OpenOffice / Jodconverter / usr / lib / libreofficeのパス（Libraを設定しました）

さらに、設定は複雑ではありません。

システムに最初にログインした後は、必ず「管理」→「構成」に移動し、path.ffmpeg項目を見つけて、そこに書き込まれている「/ usr / bin」の値を削除してください。設定を保存します。

ええと、実際にはビデオ会議サーバーは構成されており、動作する準備ができています。

サーバーを再起動した後、実行する必要があります

  1. DBMSデータベース（組み込みのDerbyを使用しない場合）
  2. KMS
  3. スクリプトstatup.sh

手動で実行できますが、自動実行スクリプトを作成できます。
