Azure PowerShell：「基本的に無害」


みなさん、こんにちは。今日、私たちはテストに関する別のマイクロソフトプロジェクトを持っています。記事のタイトルから、今回は開発者が多くの間違いで私たちを喜ばせることができなかったことが推測できます。私たちは、プロジェクトの作者がその名前に腹を立てないことを望みます。結局のところ、少数のエラーは優れていますね。ただし、Azure PowerShellコードで興味深いものが見つかりました。このプロジェクトの機能をよく理解し、PV＃C-analyzer PVS-Studioで見つかったエラーを確認することをお勧めします。

プロジェクトについて


Azure PowerShell- コマンドラインPowerShellから直接リソースAzureを管理できる一連のコマンドレット（cmdlet）。このセットの主な目的は、Azureの学習と開発のクイックスタートを簡素化することです。 Azure PowerShellは、管理者と開発者にMicrosoft Azureアプリケーションの作成、展開、および管理のための強力な機能を提供します。 Azure PowerShellは.NETスタンダード環境で開発され、Windows用のPowerShell 5.1およびすべてのプラットフォーム用のPowerShell 6.x以降でサポートされています。Azure PowerShellのソースコードはGitHubで入手できます。



最近、マイクロソフトのプロジェクトが頻繁に注目されています。私の意見では、これらのプロジェクトの質は通常最高です。もちろん、記事「WinForms：Errors、Holmes」にあるように、例外なく実行することはできません。ただし、今回はすべて正常です。プロジェクトは大規模です。6845ファイルの.csソースコードには、空を除いて約70万行が含まれています（テスト、および信頼性の第3レベルの警告、私は考慮していません）。このような大量のコードのエラーはほとんどありませんでした。同じタイプの状況がかなりあるため、この記事では、最も興味深いポジティブを選択しました。私が通常行うように、エラーはPVS-Studio診断番号でソートされます。

エラーのように見えるコードの断片にも出くわしましたが、PowerShellの開発機能に慣れていないため、実際の問題の存在について明確な結論を出すことができませんでした。読者の間でこの問題の専門家がいて、私を助けてくれることを願っています。以下について。

「デブリーフィング」を始める前に、私はプロジェクトの構造という点でプロジェクトの特徴に注目します。Azure PowerShellソースコードは、70以上のVisual Studioソリューションで構成されています。一部のソリューションには、他のソリューションのプロジェクトが含まれています。このような構造では、分析が少し遅くなります（それほどではありません）。残りの検証では問題は発生しませんでした。便宜上、エラーメッセージの行（角かっこ内）に、このエラーが検出されたソリューションの名前を示します。
