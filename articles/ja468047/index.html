lsFusionでの階層の操作


さまざまなアプリケーションでは、オブジェクトの階層表現を実装する必要があることがよくあります。通常、これはグループを指定することによってそれらを分類するために使用されます。これらのグループは、動的な深さのツリーを形成し、ナビゲーション、データ集約、およびパラメーター設定に使用されます。

この記事では、オープンで無料のlsFusionプラットフォームにこのロジックを実装する方法を示します。

例として、動的な深さの階層を形成する特定のグループにグループ化された商品のロジックを実装する必要がある単純なロジックを考えてみましょう。この場合、商品をツリーの中間ノードに関連付けることができます。

最初に、標準のスキームに従って、Productグループエンティティを編集フォームとリストを持つ単純なフラットクラスとして宣言します。
CLASS Group '';
name '' = DATA ISTRING[50] (Group);

FORM group ''
   OBJECTS g = Group PANEL
   PROPERTIES(g) name
  
   EDIT Group OBJECT g
;

FORM groups ''
   OBJECTS g = Group
   PROPERTIES(g) READONLY name
   PROPERTIES(g) NEWSESSION NEW, EDIT, DELETE
  
   LIST Group OBJECT g
;

NAVIGATOR {
   NEW groups;
}

次に、グループから階層を作成します。これを行うには、親グループへのリンクを含むプロパティを導入します。
parent = DATA Group (Group);
nameParent ' ' (Group g) = name(parent(g));

次に、2つのグループ間の関係を再帰的に決定するプロパティを作成します。
level '' (Group child, Group parent) =
   RECURSION 1l IF child IS Group AND parent = child
        STEP 2l IF parent = parent($parent) MATERIALIZED;

RECURSION 演算子が機能する原理については、この記事では説明しませんが、levelプロパティは「対応する方向ツリーの子と親の間のパスの長さ」の程度まで2を返します。MATERIALIZEDは、プラットフォームがそれを個別のテーブルに格納する必要があることを示します。接続されたノードの各ペアに対して、対応する列にレベル値を持つ個別のレコードがあります。ツリーの構造が変更されると、このテーブルは自動的に再カウントされます。

さらに、構築されたプロパティに基づいて、グループの正規名を計算できます。
canonicalName ' ' (Group group) =
   GROUP CONCAT name(Group parent), ' / ' ORDER DESC level(group, parent) CHARWIDTH 50;

たとえば、上の図のMilkグループの場合、正規名はAll / Groceries / Dairy / Milkになります。CHARWIDTHは、インターフェースを構築するときにこのプロパティに使用する幅（文字数）をプラットフォームに指示するために指定されます。

次に、新しく作成されたプロパティを持つグループを表示および編集するためのフォームを展開します。
EXTEND FORM group
   PROPERTIES(g) nameParent, canonicalName
;

EXTEND FORM groups
   PROPERTIES(g) READONLY nameParent, canonicalName
;

グループロジックが完了したら、エンティティProductを追加します。
CLASS Product '';
name '' = DATA ISTRING[50] (Product);

所属する商品グループへの商品リンクを作成します。
group '' = DATA Group (Product);
canonicalNameGroup '  ' (Product p) = canonicalName(group(p));

最後に、商品を入力するためのフォームを作成します。このフォームには、グループのツリーと商品のリストの2つの要素があります。選択したツリーグループでは、選択したノードの子孫に属する製品のみがリストに表示されます。まず、フォームを宣言し、グループのリストを含むツリーをフォームに追加します。
FORM products ''
   TREE groups g = Group PARENT parent
   PROPERTIES READONLY name(g)
;

TREE コマンドを使用すると、Groupクラスのオブジェクトからツリーが作成されます。その階層は、以前に作成された親プロパティによって決定されます。

フォームをナビゲーターに追加します。
NAVIGATOR {
   NEW products;
}

この例では、商品の入力と編集は個別のダイアログではなく、フォーム自体で直接実行されます。これを行うには、選択したグループを参照して商品を作成するアクションを作成します。
newProduct '' (Group g) {
   NEW p = Product {
       group(p) <- g;
   }
}

次に、以前に作成したフォームに、編集可能な列を含む製品のリストを追加します。
EXTEND FORM products
   OBJECTS p = Product
   PROPERTIES(p) name, canonicalNameGroup
   FILTERS level(group(p), g)
;

商品を追加および削除するためのフォームボタンをスローします。
EXTEND FORM products
   PROPERTIES newProduct(g) DRAW p TOOLBAR, DELETE(p)
;

アクションnewProductは製品のグループに対して定義されているため、製品のリスト（p）とともにツールバーに追加する必要があることを明示的に指定する必要があります。

ツリーが左側に表示され、製品のリストが右側に表示されるようにデザインを構成する必要があります。それらの間に、オブジェクトのサイズを変更できるセパレーターがあります。
DESIGN products {
   OBJECTS {
       NEW pane {
           type = SPLITH;
           fill = 1;
           MOVE BOX(TREE groups);
           MOVE BOX(p);
       }
   }
}

製品とグループの階層が作成された後、多くの場合、任意のレベルでいくつかのパラメーターを設定する必要があります。また、指定される階層レベルが低いほど、その値は高くなります。たとえば、乳製品グループが30に設定され、牛乳グループが20に設定されている場合、最後のグループを選択する必要があります。

この方法でプレミアムオプションを定義するとします。これを行うには、まずグループに適切なプロパティを作成します。
markup ', %' = DATA NUMERIC[10,2] (Group);

目的の値を見つけるには、最後の値を選択してグループ化を使用します。
parentMarkup ' (  ), %' (Group child) =
   GROUP LAST markup(Group parent) ORDER DESC level(child, parent) WHERE markup(parent);

通常の言語に翻訳すると、この式は、（GROUP）最上位グループ（Group parent）の最後の（LAST）マークアップを、そのマークアップの距離（ORDER DESC level（child、parent））との距離の昇順で見つけます。指定された（WHEREマークアップ（親））。ここで、lsFusionが自然言語にどのように対応しているかに注目します。 上記で作成したプロパティをグループツリーの製品フォームに追加します。


EXTEND FORM products
   PROPERTIES (g) markup, parentMarkup READONLY
;

製品に直接プレミアムを設定する必要があり、グループのプレミアムよりも高いと仮定します。これを行うには、最初に製品のプライマリプロパティを作成します。
dataMarkup '  , %' = DATA NUMERIC[10,2] (Product);

次に、商品のプレミアム（指定されている場合）またはグループのプレミアムを返すプロパティを宣言します。
markup ', %' (Product p) = OVERRIDE dataMarkup(p), parentMarkup(group(p));

その後、両方のプロパティをフォームに追加します。
EXTEND FORM products
   PROPERTIES(p) dataMarkup, markup READONLY
;

結論


上記の記事では、製品のロジックを作成し、それらを動的な深さの階層を持つグループに結合し、ユーザーに任意のレベルでプレミアムを設定する機能を提供することができました。これらすべてに約70行のコードが必要でした。サイトの対応するセクション（[プラットフォーム]タブ）で、コードがオンラインでどのように機能するかを試したり、コードを変更したりできます。以下は、適切なフィールドに貼り付ける必要があるすべてのソースコードです。
