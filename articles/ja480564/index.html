フレームワークなしのJavaでのREST API

記事の翻訳は特にコースの学生のために準備されました。 「Java開発者。」



Javaエコシステムには多くのフレームワークとライブラリがあります。 JavaScriptほどではありませんが、すぐに陳腐化することはありません。しかし、フレームワークなしでアプリケーションを作成する方法をすでに忘れていたと思いました。

あなたは、と言うことができる春が標準であり、なぜ車輪の再発明？また、Sparkは便利で便利なRESTフレームワークです。またはLight-rest-4jis。そしてもちろん、あなたは正しいと言います。

しかし、フレームワークとともに、完成した機能に加えて、多くの魔法、学習の難しさ、使用しない可能性が高い追加機能、およびバグが発生します。また、サービスのサードパーティコードが多いほど、エラーが発生する可能性が高くなります。

オープンソースコミュニティは非常に活発であり、フレームワークのエラーはすぐに修正される可能性が高いです。しかし、それでも、本当にフレームワークが必要かどうか考えてほしいと思います。小規模なサービスまたはコンソールアプリケーションを使用している場合は、それなしで実行できる場合があります。

純粋なJavaコードを使用して何を得る（または失う）ことができますか？それについて考えてください：

  • あなたのコードはよりクリーンで理解しやすいものになります（または、あなたが貧しいプログラマなら、おそらく完全に混乱しています）
  • コードをより詳細に制御できるようになり、フレームワークに制限されることはありません（フレームワークが提供する機能のために、より多くのコードを記述する必要があります）。

最初のエンドポイント


Webアプリケーションはクラスに基づいていcom.sun.net.httpserver.HttpServerます。そして、最も単純なエンドポイント/api/helloは次のようになります。

package com.consulner.api;

import java.io.IOException;
import java.io.OutputStream;
import java.net.InetSocketAddress;

import com.sun.net.httpserver.HttpServer;

class Application {

   public static void main(String[] args) throws IOException {
       int serverPort = 8000;
       HttpServer server = HttpServer.create(new InetSocketAddress(serverPort), 0);
       server.createContext("/api/hello", (exchange -> {
           String respText = "Hello!";
           exchange.sendResponseHeaders(200, respText.getBytes().length);
           OutputStream output = exchange.getResponseBody();
           output.write(respText.getBytes());
           output.flush();
           exchange.close();
       }));
       server.setExecutor(null); // creates a default executor
       server.start();
   }
}

Webサーバーはポート8000​​で実行され、Hello！..を返すだけのエンドポイントを提供します。これは、たとえば、curlを使用して確認できます：ステップ2

curl localhost:8000/api/hello

さまざまなHTTPメソッドのサポート


最初のエンドポイントは正常に機能しますが、使用するHTTPメソッドに関係なく、常に同じように応答することに気付くでしょう。

例えば：

curl -X POST localhost:8000/api/hello
curl -X PUT localhost:8000/api/hello

まず最初に、メソッドを区別するためのコードを追加します。次に例を示します。

server.createContext("/api/hello", (exchange -> {
   if ("GET".equals(exchange.getRequestMethod())) {
       String respText = "Hello!";
       exchange.sendResponseHeaders(200, respText.getBytes().length);
       OutputStream output = exchange.getResponseBody();
       output.write(respText.getBytes());
       output.flush();
   } else {
       exchange.sendResponseHeaders(405, -1);// 405 Method Not Allowed
   }
   exchange.close();
}));

クエリを再試行してください。

curl -v -X POST localhost:8000/api/hello

答えは次のようになります：

> Host: localhost:8000
> User-Agent: curl/7.61.0
> Accept: */*
>
< HTTP/1.1 405 Method Not Allowed

リクエストパラメータの解析


クエリパラメータの解析は、独自に実装する必要があるもう1つの「関数」です。

Hello APIがパラメータnameで名前を取得するようにしたいとします。次に例を示します。

curl localhost:8000/api/hello?name=Marcin

Hello Marcin!

次のようにパラメーターを解析できます。

public static Map<String, List<String>> splitQuery(String query) {
   if (query == null || "".equals(query)) {
       return Collections.emptyMap();
   }

   return Pattern.compile("&").splitAsStream(query)
           .map(s -> Arrays.copyOf(s.split("="), 2))
           .collect(groupingBy(s -> decode(s[0]), mapping(s -> decode(s[1]), toList())));
}

以下のように使用します：

Map<String, List<String>> params = splitQuery(exchange.getRequestURI().getRawQuery());
String noNameText = "Anonymous";
String name = params.getOrDefault("name", List.of(noNameText)).stream().findFirst().orElse(noNameText);
String respText = String.format("Hello %s!", name);

安全性


多くの場合、一部のエンドポイントへのアクセスを保護する必要があります。たとえば、これは基本認証を使用して行うことができます。

HttpContextごとに、次に示すようにオーセンティケーターを設定できます。

HttpContext context = server.createContext("/api/hello", (exchange -> {
   //    
}));
context.setAuthenticator(new BasicAuthenticator("myrealm") {
   @Override
   public boolean checkCredentials(String user, String pwd) {
       return user.equals("admin") && pwd.equals("admin");
   }
});

コンストラクターの「myrealm」の値BasicAuthenticatorは、レルムの名前です。レルムは、認証ドメインを分離するために使用できる仮想名です。

これについて詳しくは、RFC 1945を参照してください。

これで、ヘッダーを追加してこの安全なエンドポイントを呼び出すことができますAuthorization。

curl -v localhost:8000/api/hello?name=Marcin -H 'Authorization: Basic YWRtaW46YWRtaW4='

「Basic」の後のテキストは、Base64でエンコードされたテキストですadmin:admin。これは、この例でハードコーディングされた資格情報です。

実際のアプリケーションでの認証では、ヘッダーから認証情報を取得し、データベースに保存されているユーザー名とパスワードと比較する可能性があります。

タイトルを指定しない場合、APIはステータスで応答します

HTTP/1.1 401 Unauthorized

JSON、例外処理など


次に、より複雑な例を示します。

ソフトウェア開発の経験から、私が開発した最も一般的なAPIはJSON交換でした。

新しいユーザーを登録するためのAPIを開発します。それらを格納するために、メモリ内のデータベースを使用します。

単純なドメインオブジェクトを作成しますUser。

@Value
@Builder
public class User {
   String id;
   String login;
   String password;
}

Lombokを使用して、ボイラープレート（コンストラクター、ゲッター）を削除します。

REST APIでは、ログインとパスワードのみを渡したいので、別のオブジェクトを作成しました。

@Value
@Builder
public class NewUser {
   String login;
   String password;
}

オブジェクトUserは、APIハンドラーで使用するサービスで作成されます。serviceメソッドは単にユーザーを保存します。

public String create(NewUser user) {
   return userRepository.create(user);
}

実際のアプリケーションでは、さらに多くのことができます。たとえば、ユーザー登録が成功した後でイベントを送信します。

リポジトリの実装は次のとおりです。

import java.util.Map;
import java.util.UUID;
import java.util.concurrent.ConcurrentHashMap;

import com.consulner.domain.user.NewUser;
import com.consulner.domain.user.User;
import com.consulner.domain.user.UserRepository;

public class InMemoryUserRepository implements UserRepository {

   private static final Map USERS_STORE = new ConcurrentHashMap();

   @Override
   public String create(NewUser newUser) {
       String id = UUID.randomUUID().toString();
       User user = User.builder()
           .id(id)
           .login(newUser.getLogin())
           .password(newUser.getPassword())
           .build();
       USERS_STORE.put(newUser.getLogin(), user);

       return id;
   }
}

最後に、すべてを一緒に接着しますhandle()。

protected void handle(HttpExchange exchange) throws IOException {
    if (!exchange.getRequestMethod().equals("POST")) {
        throw new UnsupportedOperationException();
    }

    RegistrationRequest registerRequest = readRequest(exchange.getRequestBody(), RegistrationRequest.class);

    NewUser user = NewUser.builder()
        .login(registerRequest.getLogin())
        .password(PasswordEncoder.encode(registerRequest.getPassword()))
        .build();

    String userId = userService.create(user);

    exchange.getResponseHeaders().set(Constants.CONTENT_TYPE, Constants.APPLICATION_JSON);
    exchange.sendResponseHeaders(StatusCode.CREATED.getCode(), 0);

    byte[] response = writeResponse(new RegistrationResponse(userId));

    OutputStream responseBody = exchange.getResponseBody();
    responseBody.write(response);
    responseBody.close();
}

ここで、JSONリクエストはオブジェクトに変換されますRegistrationRequest。
