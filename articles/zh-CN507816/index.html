这篇简短的文章将告诉您有关将JDI Light与Selenium Grid集成的信息。

JDI是用于测试自动化的一系列框架。目前，它由用于UI自动化的JDI Light，用于后端自动化的JDI Dark和JDI Mobile组成。该名称代表Just do it：JDI的主要思想是使用它来轻松测试和维护测试，所有成功工作的发现都可以保存和积累。



Selenium Grid是几个Selenium服务器的集群。它旨在组织分布式网络，使您可以在大量计算机上同时运行许多浏览器。有一个专用服务器，称为“集线器”或“交换机”，服务器的其余部分称为“节点”或“节点”。



集成：



我们需要的第一件事是硒网格本身，您可以在硒官方网站（selenium.dev/downloads/）上下载它，也可以在此处查看文档。



接下来，您需要运行selenium grid，为此，在包含下载文件的文件夹中，打开命令行并执行命令。



  1. 在中心模式下运行硒网格： 



    java -jar selenium-server-standalone-3.141.59.jar -role hub -port 4444


    • selenium-server-standalone-3.141.59.jar-硒网格文件
    • 端口4444-集线器将在其上运行的端口，您需要知道如何连接节点并创建RemoteWebDriver。


  2. 将节点连接到集线器： 



    java -Dwebdriver.chrome.driver=chromedriver.exe -jar selenium-server-standalone-3.141.59.jar -role node -hub http://localhost:4444/grid/register -browser "browserName=chrome,maxInstances=3,platform=WINDOWS" -port 5557


您还可以使用json文件配置节点。您可以在selenium项目的官方网站（selenium.dev/documentation/en/grid/）上了解有关Selenium Grid功能和设置的更多信息。



您可以从控制台或通过单击注册集线器的链接来检查集线器是否已启动以及是否已连接节点。在上面介绍的配置中，我们在端口4444上本地创建了它，因此在浏览器中打开链接（localhost：4444 / grid / console）之后，我们将看到以下站点：







硒网格配置已完成，我们将继续使用它。



如果您使用纯Selenium WebDriver，则要获取WebDriver对象，我们将需要创建一个对象 



RemoteWebDriver(URL remoteAddress, Capabilities capabilities)
- remoteAddress – url 
- capabilities –


创建对象时，我们将转到中心并尝试从中获取具有所需浏览器的节点。如果找不到这样的节点，将引发异常。

开箱即用，Selenium WebDriver不提供即时更改浏览器或将所需浏览器存储在设置文件中的功能。为了实现这些机会，开发人员要花费时间和精力。



但是JDI Light的开发人员却照顾了这些功能的实现。使用JDI，可以节省时间和精力，因为要获取WebDriver对象，我们只需将集线器地址添加到JDI设置文件中： 



driver.remote.url=http://localhost:4444/wd/hub


并在设置文件中指定用于运行测试的浏览器： 



driver=chrome


有关设置文件和JDI Light的其他功能的其他信息（jdi-docs.github.io/jdi-light/#jdi-settings）。



集线器将尝试找到合适的节点。如果未找到该节点，则将引发异常： 



java.lang.RuntimeException: java.lang.RuntimeException: Failed to do screenshot. No Drivers run
