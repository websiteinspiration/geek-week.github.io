Kerasを使用したリカレントニューラルネットワーク（RNN）

Tensorflow.orgからの再帰的ニューラルネットワークガイドの翻訳。この資料では、クイックメッシュのためのKeras / Tensorflow 2.0の組み込み機能と、レイヤーやセルをカスタマイズする可能性について説明しています。CuDNNコアの使用のケースと制限も考慮されます。これにより、ニューラルネットワークの学習プロセスを加速できます。



再帰的ニューラルネットワーク（RNN）は、時系列や自然言語などのシリアルデータのモデリングに適したニューラルネットワークのクラスです。

概略的に言えば、RNNレイヤーはループforを使用して時間順のシーケンスを反復し、内部状態で、彼がすでに見たステ​​ップに関するエンコードされた情報を保存します。

Keras RNN APIは、次のことに重点を置いて設計されています。

使いやすさ：組み込みのレイヤー、、tf.keras.layers.RNNによりtf.keras.layers.LSTM、tf.keras.layers.GRU複雑な構成設定を行わなくても、再帰モデルをすばやく構築できます。

簡単なカスタマイズ：RNNセルの独自のレイヤーを定義することもできます（ループの内部）for）カスタム動作を使用して、 `tf.keras.layers.RNN`の共通レイヤー（the` for`ループ自体）で使用します。これにより、最小限のコードで、柔軟な方法でさまざまな研究アイデアのプロトタイプをすばやく作成できます。

単純なモデルを作成する


Kerasには3つの組み込みRNNレイヤーがあります。

  1. tf.keras.layers.SimpleRNN、完全に接続されたRNN。前のタイムステップの出力を次のステップに渡す必要があります。
  2. tf.keras.layers.GRU、記事「統計的機械翻訳のためのRNNコーデックを使用したフレーズの学習」で最初に提案されました
  3. tf.keras.layers.LSTM、記事「長期短期記憶」で最初に提案されました

2015年の初めに、Kerasは最初の再利用可能なオープンソースのPythonおよびLSTMとGRUの実装を導入しました。

以下は、Sequential各整数を64次元ベクトルにネストして整数のシーケンスを処理し、次にレイヤーを使用してベクトルのシーケンスを処理するモデルの例ですLSTM。

model = tf.keras.Sequential()
#   Embedding      1000, 
#     64.
model.add(layers.Embedding(input_dim=1000, output_dim=64))

#   LSTM  128  .
model.add(layers.LSTM(128))

#   Dense  10    softmax.
model.add(layers.Dense(10))

model.summary()

出力とステータス


デフォルトでは、RNNレイヤーの出力には要素ごとに1つのベクトルが含まれます。このベクトルは、入力シーケンス全体に関する情報を含む最後のRNNセルの出力です。この出力の次元(batch_size, units)。ここで、レイヤーコンストラクターに渡されるunits引数に対応しますunits。

RNNレイヤーは、を指定しreturn_sequences=Trueた場合、各要素の出力シーケンス全体（各ステップに1つのベクトル）を返すこともできます。この出力の次元は(batch_size, timesteps, units)です。

model = tf.keras.Sequential()
model.add(layers.Embedding(input_dim=1000, output_dim=64))

#  GRU  3D   (batch_size, timesteps, 256)
model.add(layers.GRU(256, return_sequences=True))

#  SimpleRNN  2D   (batch_size, 128)
model.add(layers.SimpleRNN(128))

model.add(layers.Dense(10))

model.summary()

さらに、RNNレイヤーは最終的な内部状態を返すことができます。

返された状態を後で使用して、RNNの実行を再開したり、別のRNNを初期化したりできます。この設定は通常、エンコーダー-デコーダーモデルでシーケンスごとに使用され、エンコーダーの最終状態がデコーダーの初期状態に使用されます。

RNNレイヤーが内部状態を返すためには、レイヤーの作成時にパラメーターreturn_stateを値Trueに設定します。LSTM2つの状態テンソルがありGRU、1つだけであることに注意してください。

レイヤーの初期状態を調整するには、追加の引数を指定してレイヤーを呼び出すだけinitial_stateです。

次の例のように、寸法はレイヤー要素の寸法と一致する必要があることに注意してください。

encoder_vocab = 1000
decoder_vocab = 2000

encoder_input = layers.Input(shape=(None, ))
encoder_embedded = layers.Embedding(input_dim=encoder_vocab, output_dim=64)(encoder_input)

#       
output, state_h, state_c = layers.LSTM(
    64, return_state=True, name='encoder')(encoder_embedded)
encoder_state = [state_h, state_c]

decoder_input = layers.Input(shape=(None, ))
decoder_embedded = layers.Embedding(input_dim=decoder_vocab, output_dim=64)(decoder_input)

#  2     LSTM    
decoder_output = layers.LSTM(
    64, name='decoder')(decoder_embedded, initial_state=encoder_state)
output = layers.Dense(10)(decoder_output)

model = tf.keras.Model([encoder_input, decoder_input], output)
model.summary()

RNNレイヤーとRNNセル


RNN APIは、組み込みのRNNレイヤーに加えて、セルレベルのAPIも提供します。入力シーケンスのパケット全体を処理するRNNレイヤーとは異なり、RNNセルは1つのタイムステップのみを処理します。

セルは、forRNNレイヤーのサイクル内にあります。レイヤーでセルをラップすると、tf.keras.layers.RNNシーケンスパケットを処理できるレイヤーが得られます。RNN(LSTMCell(10))。

数学的にRNN(LSTMCell(10))は、と同じ結果になりLSTM(10)ます。実際、TF v1.x内でのこのレイヤーの実装は、対応するRNNセルを作成してRNNレイヤーでラップすることだけでした。ただし、埋め込みレイヤーGRUをLSTM使用すると、パフォーマンスを向上させることができるCuDNNを使用できます。

3つの組み込みRNNセルがあり、それぞれが独自のRNN層に対応しています。

  • tf.keras.layers.SimpleRNNCellレイヤーと一致しSimpleRNNます。
  • tf.keras.layers.GRUCellレイヤーと一致しGRUます。
  • tf.keras.layers.LSTMCellレイヤーと一致しLSTMます。

セルを抽象化し、共通のクラスを使用tf.keras.layers.RNNすると、研究用のカスタムRNNアーキテクチャーを非常に簡単に実装できます。

クロスバッチ保存状態


長いシーケンス（場合によっては無限）を処理するときは、クロスバッチステートフルネスパターンを使用できます。

通常、RNNレイヤーの内部状態は、新しいデータパケットごとにリセットされます（つまり、レイヤーを確認する各例は、過去から独立していると見なされます）。レイヤーは、この要素を処理している間だけ状態を維持します。

ただし、非常に長いシーケンスがある場合は、シーケンスをより短いシーケンスに分割し、レイヤーの状態をリセットせずに順番にRNNレイヤーに転送すると便利です。したがって、一度に1つのサブシーケンスしか表示されませんが、レイヤーはシーケンス全体に関する情報を格納できます。

これを行うには、コンストラクタで「stateful = True」を設定します。

シーケンス `s = [t0、t1、... t1546、t1547]`がある場合は、たとえば次のように分割できます。

s1 = [t0, t1, ... t100]
s2 = [t101, ... t201]
...
s16 = [t1501, ... t1547]

その後、次のように処理できます。

lstm_layer = layers.LSTM(64, stateful=True)
for s in sub_sequences:
  output = lstm_layer(s)

状態をきれいにしたい場合は、を使用してくださいlayer.reset_states()。
注：この場合、iこのパッケージの例は、i前のパッケージの例の続きであると想定されています。つまり、すべてのパッケージに同じ数の要素（パッケージサイズ）が含まれています。たとえば、パッケージにが含まれている[sequence_A_from_t0_to_t100, sequence_B_from_t0_to_t100]場合、次のパッケージにはが含まれて[sequence_A_from_t101_to_t200, sequence_B_from_t101_to_t200]いる必要があります。
以下は完全な例です。

paragraph1 = np.random.random((20, 10, 50)).astype(np.float32)
paragraph2 = np.random.random((20, 10, 50)).astype(np.float32)
paragraph3 = np.random.random((20, 10, 50)).astype(np.float32)

lstm_layer = layers.LSTM(64, stateful=True)
output = lstm_layer(paragraph1)
output = lstm_layer(paragraph2)
output = lstm_layer(paragraph3)

# reset_states()      initial_state.
#  initial_state   ,      .
lstm_layer.reset_states()

双方向RNN


時系列以外のシーケンス（テキストなど）の場合、シーケンスを最初から最後まで処理するだけでなく、その逆も処理すると、RNNモデルがよりよく機能することがよくあります。たとえば、文の次の単語を予測するには、単語の前にある単語だけでなく、単語の前後のコンテキストを知っておくと便利です。

Kerasは、そのような双方向RNNを作成するための単純なAPI、ラッパーを提供しますtf.keras.layers.Bidirectional。

model = tf.keras.Sequential()

model.add(layers.Bidirectional(layers.LSTM(64, return_sequences=True), 
                               input_shape=(5, 10)))
model.add(layers.Bidirectional(layers.LSTM(32)))
model.add(layers.Dense(10))

model.summary()

Bidirectional内部 では、転送されたRNNレイヤーgo_backwardsがコピーされ、新しくコピーされたレイヤーのフィールドが反転します。したがって、入力データは逆の順序で処理されます。

` BidirectionalRNN の出力は、デフォルトでは、フォワードレイヤーの出力とリバースレイヤーの出力の合計になります。他のマージ動作が必要な場合。連結するには、 `Bidirectional`ラッパーコンストラクターの` merge_mode`パラメーターを変更します。

TensorFlow 2.0のパフォーマンス最適化とCuDNNコア


TensorFlow 2.0では、グラフィックプロセッサが利用可能な場合、組み込みのLSTM層とGRU層がデフォルトのCuDNNコアでの使用に適しています。この変更により、以前のレイヤーkeras.layers.CuDNNLSTM/CuDNNGRUは古くなり、機能する機器を気にすることなくモデルを構築できます。

CuDNNカーネルはいくつかの前提で構築されているため、組み込みのLSTMまたはGRUレイヤーのデフォルト設定を変更すると、レイヤーはCuDNNカーネルレイヤーを使用できなくなります。例えば。

  • 関数activationをtanh別のものに変更する。
  • 関数recurrent_activationをsigmoid別のものに変更する。
  • 使用量recurrent_dropout> 0。
  • これunrollをTrueに設定すると、LSTM / GRUは内部tf.while_loopを展開されたループに分解しますfor。
  • use_biasFalseに設定します。
  • 入力データが右揃えされていない場合にマスクを使用します（マスクが厳密に整列された正しいデータと一致する場合でも、CuDNNを使用できます。これが最も一般的なケースです）。

可能な場合はCuDNNカーネルを使用する


batch_size = 64
#    MNIST    (batch_size, 28, 28).
#     (28, 28) (   ).
input_dim = 28

units = 64
output_size = 10  #   0  9

#  RNN 
def build_model(allow_cudnn_kernel=True):
  # CuDNN     ,     .
  #   `LSTM(units)`    CuDNN,
  #   RNN(LSTMCell(units))   non-CuDNN .
  if allow_cudnn_kernel:
    #  LSTM      CuDNN.
    lstm_layer = tf.keras.layers.LSTM(units, input_shape=(None, input_dim))
  else:
    #  LSTMCell  RNN    CuDNN.
    lstm_layer = tf.keras.layers.RNN(
        tf.keras.layers.LSTMCell(units),
        input_shape=(None, input_dim))
  model = tf.keras.models.Sequential([
      lstm_layer,
      tf.keras.layers.BatchNormalization(),
      tf.keras.layers.Dense(output_size)]
  )
  return model

リスト/辞書入力、またはネストされた入力を持つRNN


ネストされた構造により、1つのタイムステップにより多くの情報を含めることができます。たとえば、ビデオフレームには、オーディオとビデオの入力が同時に含まれている場合があります。この場合のデータの次元は次のようになります。

[batch, timestep, {\"video\": [height, width, channel], \"audio\": [frequency]}]

別の例では、手書きデータは、現在のペン位置のx座標とy座標、および圧力情報を持つことができます。したがって、データは次のように表すことができます。

[batch, timestep, {\"location\": [x, y], \"pressure\": [force]}]

次のコードは、このような構造化入力で機能するカスタムRNNセルの例を作成します。

ネストされた入力/出力をサポートするユーザーセルを定義する


NestedInput = collections.namedtuple('NestedInput', ['feature1', 'feature2'])
NestedState = collections.namedtuple('NestedState', ['state1', 'state2'])

class NestedCell(tf.keras.layers.Layer):

  def __init__(self, unit_1, unit_2, unit_3, **kwargs):
    self.unit_1 = unit_1
    self.unit_2 = unit_2
    self.unit_3 = unit_3
    self.state_size = NestedState(state1=unit_1, 
                                  state2=tf.TensorShape([unit_2, unit_3]))
    self.output_size = (unit_1, tf.TensorShape([unit_2, unit_3]))
    super(NestedCell, self).__init__(**kwargs)

  def build(self, input_shapes):
    # #  input_shape  2 , [(batch, i1), (batch, i2, i3)]
    input_1 = input_shapes.feature1[1]
    input_2, input_3 = input_shapes.feature2[1:]

    self.kernel_1 = self.add_weight(
        shape=(input_1, self.unit_1), initializer='uniform', name='kernel_1')
    self.kernel_2_3 = self.add_weight(
        shape=(input_2, input_3, self.unit_2, self.unit_3),
        initializer='uniform',
        name='kernel_2_3')

  def call(self, inputs, states):
    #     [(batch, input_1), (batch, input_2, input_3)]
    #     [(batch, unit_1), (batch, unit_2, unit_3)]
    input_1, input_2 = tf.nest.flatten(inputs)
    s1, s2 = states

    output_1 = tf.matmul(input_1, self.kernel_1)
    output_2_3 = tf.einsum('bij,ijkl->bkl', input_2, self.kernel_2_3)
    state_1 = s1 + output_1
    state_2_3 = s2 + output_2_3

    output = [output_1, output_2_3]
    new_states = NestedState(state1=state_1, state2=state_2_3)

    return output, new_states
