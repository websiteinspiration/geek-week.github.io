ブランチを開いて結果をマップ上で視覚化するのに最適な場所を選択する方法

新しいブランチの場所を選択することは責任ある決定です。間違いは、特に資本集約的な産業では、高額になる可能性があります。ほとんどの場合、そのような決定は管理の専門家によって行われます。都市、業界、以前の経験に関する知識に基づいています。

この記事では、分析がそのような意思決定にどのように役立つかについて説明します。人口、不動産価格に関する情報を収集し、インタラクティブな視覚化を行う方法。顧客の数は、支店までの距離、家が建てられた年、および資産の価値によって異なりますか。

地図を作成するコード
#   
import pandas as pd
from folium.plugins import HeatMap
import folium

# 
df = pd.read_csv('people_spb.csv')
filial = pd.read_csv('filial.csv')
competitor = pd.read_csv('competitors.csv')

# 
hmap = folium.Map(location=[59.95, 30.15], zoom_start=11)

#   
people = folium.FeatureGroup(name = ' ')

hm = HeatMap( list(zip(df.lat.values, df.lng.values, df['People'])), 
              min_opacity = .1,
              max_val = df['People'].max(),
              radius = 15, 
              blur = 25, 
              max_zoom = 1                   
              )

people.add_child(hm)

#   
filial_markers = folium.FeatureGroup(name = ' ')

for index, row in filial.iterrows():
    folium.Marker(
        location = [row['lat'], row['lng']],
        popup = row['Name'],
        icon = folium.Icon(color='blue', icon='cloud')
        ).add_to(filial_markers)

#   
competitor_markers = folium.FeatureGroup(name = ' ')

for index, row in competitor.iterrows():
    folium.Marker(
        location = [row['lat'], row['lng']],
        popup = row['Name'],
        icon = folium.Icon(color='red')
        ).add_to(competitor_markers)  
      
#   
hmap.add_child(people)
hmap.add_child(filial_markers)
hmap.add_child(competitor_markers)

#  
folium.LayerControl(collapsed=False).add_to(hmap)
 
#    html 
hmap.save('people_spb.html')


住宅の人口を評価するために、住宅と公共サービス改革のデータを使用しました。このポータルでは、建設年、居住スペース、居住区の数など、各家に関する情報を入手できます。各家の人口推定は、アパートの数と総居住空間に基づいて行われました。アパートごとに平均約3人で、一部の家と市区町村では若干の違いがあります。

上記は、サンクトペテルブルクの人口密度を示すヒートマップです。私たちの内部使用のためのカードには、顧客の密度を持つ別のレイヤーも含まれています。カバレッジが低い場所である白い斑点を探す方が便利です。

お客様の住所


ビジネスの詳細により、データベースにはほとんどすべての顧客の住所が含まれていました。各住所の地理座標（ジオコーディングまたはジオコーディング）を見つけるだけで済みました。座標を取得するために、Python用のジオコーダーパッケージを使用しました。ジオコーディング中に次の問題が発生しました：

  1. 大文字または小文字が混同されているなど、一部のアドレスが正しくありません。この状況では、ジオコーディングにより、クライアントを幼稚園やオフィスビルに「置く」ことができます。そのような場合、座標を200 m以内の最も近い住宅に変更するプロセスを作成する必要がありました。
  2. 異常に多くの顧客がいるポイント：市内中心部、大きな通りの中央、地区の中央。このような座標は、誤って入力された住所で取得され、全体像を歪める可能性があるため、モデリングする前に削除されました

その結果、93％の顧客の家の正確な座標を取得しました。これで、そのようなマップを作成できます。

このようなマップは、仮説をテストするための便利なツールであることがわかりました。たとえば、ビジネスには、一部のタイプの住宅（ソビエトの大規模建物：船、504シリーズ、フルシチョフなど）にはクライアントがいないという仮説がありました。これは完全に真実ではないことがわかりました。はい、そのような家の人口からの顧客の割合は低いです。しかし、市内にはそのような家がたくさんあり、その結果、顧客フローの最大20％を提供するため、それらを考慮する必要があります。

市区の境界


前のセクションの人口および顧客データを市区町村ごとに再配置してマッピングできます。情報ウィンドウとカスタマイズされたカラーリングを追加すると、非常に有益です。ハブにはすでに優れた記事があり、その手順はそのようなカードの作成方法を示しています。





プロパティ値


不動産価格の決定は困難な作業であることが証明されています。最初の段階では、2018年の初めから不動産販売のすべての広告を取得することができました。これは約70万件のレコードです。

各家屋の1平方メートルあたりの費用は、広告の中央値として計算されました。広告のない家屋の20％について、正方形のコストを見積もりました。m。モデルの使用。主な要因は平方メートルあたりの価格です。m 15最寄りの家。さらに、同様の特性を持つ住宅は、建設年、居住者数、プロジェクトのタイプなどの重みが高くなります。テストセットでのモデルの平均誤差は9.5％でした。特にあなたが1つの家でさえ平方メートルのコストをそれと考えるとき。m。床、修理、面積、その他の要因によって大きく異なる場合があります。

自宅から支店までの距離


4つの部門のグラフは、住宅内の顧客の割合の部門までの距離への依存性を示しています。いくつかの支店では、他の要因（住宅の築年数、不動産の価格）の影響を示唆する強い飛躍があります。



在宅年齢

平方あたりの価格。メートル




価格は顧客シェアと相関します。しかし、依存度は顧客の割合と家の年齢の間よりも弱いです。おそらくその理由は、家の築年数が居住者の築年数と相関しているためです。そして、人の年齢は医療サービスの要求の頻度に大きく影響します。

モデルの説明


将来的には、この分析は本格的なモデルに発展し、座標は入力で提供され、新規顧客からの訪問数は出力で取得されます。記事はボリュームが大きいことがわかりましたので、モデルについて簡単に説明します。

結果の解釈を容易にするために、モデルとして線形回帰を選択しました。対象となる変数は、住宅内の顧客の割合、要因：最寄りのオフィスまでの距離の対数、住宅のコスト、住宅が建設された年です。 3つの要因すべてが重要であることが判明し、モデルに入りました。

このようなモデルに新しい座標を代入すると（つまり、距離係数を最も近い分岐に変更すると）、出力でネットワーク全体の新しいクライアント数が得られます。この数から以前の顧客数を差し引くと、正味の効果が得られます。

このような問題の説明は、現在のブランチの場所を考慮して新しい場所が選択されるという点で便利です。つまり、異なる部門間の「共食い」要因を追加で考慮する必要はありません。

街全体の最適点の探索は、500mごとに座標を列挙するだけの簡単な方法で行い、複数の枝を開いたときの効果を計算するために、点を順番に設定しました。

結果


地区の境界を手動でペイントしたウォールマップをなんとか置き換え、便利なインタラクティブマップで何かを読みました。従業員を手動で修正し、数千の住所を地方自治体の地区と照合しません。データを充実させ、市区町村のレベルから各家屋に移動します。

配置のためのいくつかの非常に有望で自明ではない場所を特定することが判明しました。異なるポイントを自動的かつ公平に比較​​するモデルを構築します。

事業ラインを「地域依存型」と「地域非依存型」に分けたところ、興味深い結果が得られました。前者は新しいブランチの一部である必要があり、後者は現在の場所で開発できます。（記事には記載されていません）。
