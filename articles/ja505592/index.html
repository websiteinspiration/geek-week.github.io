Androidを使用したBK0010のプログラミング



物語はBK00010テレグラムチャンネルで起こりました、私は参加者でした。

疑問が生じました：プログラムの書き方？エミュレータはディスクの書き込みをサポートしていないため、エミュレータでアセンブラを使用しても成功しない可能性があります。さらに、スマートフォンの画像は非常に小さいです。

ソリューションが提案されました（スマスロフスキー）、これについて説明します。あなたはTermux、アンドロイド5.0のバージョンが利用可能なダウンロードする必要があり、ここで、termuxアプリ-のgit-debug.apkファイル。プログラムのバージョンは0.83で、キーバーのカスタマイズをサポートしています。

Termuxをインストールして起動した後、キーを再構成しました。手順は次のとおりです。

mkdir $HOME/.termux

echo "extra-keys = [['ESC','/','-','HOME','UP','END','PGUP'],['TAB','CTRL','ALT','LEFT','DOWN','RIGHT','PGDN']]" > $HOME/.termux/termux.properties

termux-reload-settings

より便利になりまし



た必要な フィニッシュをインストールする必要があります。MACRO-11は.BINを作成するためのアセンブラとして使用されます。
pkg install build-essential
pkg install git




mkdir src
cd src
git clone https://gitlab.com/Rhialto/macro11.git

次に、シェルをインストールしました。

pkg install mc

およびテキストエディター

pkg install nano

Makefileを修正する必要 があるsrc \ macro11フォルダで、パラメータを変更します

OBJFORMAT = -DDEFAULT_OBJECTFORMAT_RT11=1

makeと 入力して結果を待ち

、受信したファイルをコピーします

cp macro11 $PREFIX/bin

cp dumpobj $PREFIX/bin

コンパイルするには、ビルドするスクリプトが必要です

cd ~/src 
git clone https://github.com/smaslovski/pdp11dasm.git
cd pdp11dasm/utils
chmod 755 obj2bkb
cp obj2bkb $PREFIX/bin

コンパイルスクリプト、私はcompileを呼び出しました：

#!/data/data/com.termux/files/usr/bin/sh
n=$1
macro11 $n.mac -o $n.obj -l $n.lst
obj2bkb --binary --base=01000 -o $n.bin $n.obj
du -b $n.bin
cp $n.bin /sdcard

（nano compileを作成できます）

スクリプトをコピーする必要があります

chmod 755 compile
cp compile $PREFIX/bin

テストのために、私は簡単なgg.macプログラムを書きました：

.end

起動：コンパイルgg gg.bin
