TimberとNode.JSを使用して、Androidログをインターネット経由で転送します

Mnogabukaf氏は読みませんでした。リモートロギングの非常にシンプルな実装を見ました。クライアント（またはテスターなど）に理解できないエラーがポップアップし、アプリケーションで何が起こっているかをすぐに確認したい場合に役立ちます。以下に興味があります。




パート1.問題


一般に、問題は次のようになりました。テスターがバグを再現しましたが、私は再現しませんでした。ログを確認する必要があり、adbを介して接続することは不可能でした（テスターが別の都市にいてルーターの管理パネルにアクセスできなかったため）、ファイルを投げることはある種のくだらないことでした。

パート2.問題を説明する


そのようなものがあります- 木材。知らない人のために：これは、AndroidクラスLogの標準機能を拡張するライブラリです。ロギング時に、ライブラリは自動的にクラス名とメソッド名をタグとして追加します。しかし、私たちにとってさらに重要なことは、ロギングメソッドをオーバーライドして、内部で何かを行うことができるということです。



計画はこれです。Android側からサーバーに送信メッセージを書き込み、次にメッセージを受信するサーバーを書き込みます。

パート3. Android向けの記述


build.gradleに物事を入れます。1つはDEBUGモード、もう1つはSERVER_LOGGINGです。
ロガー接続は次のようになります（このコードはアプリケーションで呼び出す必要があります）：

private fun plantTimberTree() {
     if (BuildConfig.DEBUG) {
         if (BuildConfig.SERVER_LOGGING) {
             val logsSource = Injection.provideLogsDataSource()
             Timber.plant(ServerLoggingTree(logsSource))
         }

         else {
             Timber.plant(Timber.DebugTree())
         }
     }
}

送信ログはどのように見えますか？ログには、優先度、メッセージ、タグ、日付があります。リモートロギングの場合は、URL、送信先。この全体がCompletableを返します。

interface LogsDataSource {
    fun sendLog(
            priority: Int, 
            tag: String?,
            message: String,
            date: Date,
            url: String
    ): Completable
}

Retrofit'a などを使用して、すべてをサーバーに送信する必要があります。

interface LogsService {

    @POST
    @FormUrlEncoded
    fun sendLog(
            @Url url: String,
            @Field("priority") priority: Int,
            @Field("tag") tag: String?,
            @Field("message") message: String,
            @Field("date") date: Date
    ): Single<BaseResponse>
}

次に、ディスパッチ自体を記述します。 

class LogsRepository(
        val rxSchedulers: RxSchedulers
) : LogsDataSource {
    private val logsService = RestApi.createService(LogsService::class.java)

    override fun sendLog(
            priority: Int, 
            tag: String?,
            message: String,
            date: Date, 
            url: String
    ): Completable {
        val request = logsService.sendLog(
                        url, 
                        priority, 
                        tag, 
                        message,
                        date
                )
                .subscribeOn(rxSchedulers.io)
        return Completable.fromSingle(request)
    }
}

次に、ServerLoggingTreeクラスを見てみましょう。そこで、ロギングメソッドをオーバーライドし、その中のサーバーに送信するメソッドを呼び出します。

class ServerLoggingTree(
        private val logsDataSource: LogsDataSource
) : Timber.DebugTree() {
    companion object {
        private const val LOG_HOST = "abcdef.ddns.net"
        private const val LOG_PORT = 8443
        private const val LOG_PATH = "api/v1/logs.send"
        const val LOG_URL = "https://$LOG_HOST:$LOG_PORT/$LOG_PATH"
    }

    override fun log(
            priority: Int,
            tag: String?,
            message: String,
            t: Throwable?
    ) {
        val disposable = logsDataSource
                .sendLog(priority, tag, message, Date(), LOG_URL)
                .subscribe({}, {
                    Log.e("ServerLoggingTree", "Failed to send log")
                })

        super.log(priority, tag, message, t)
    }
}

これに関して、実際には、Androidの部分は終わりました。

パート4.サーバーの作成


Node.JSで書き込みますが、一般的には何でも可能です。

ロギングには、Bunyanライブラリを使用します。これは、1つ目はファイルへの書き込みをサポートし、2つ目は、美しい色でメッセージを表示できるためです。

ロギングを担当するものを書きましょう。メッセージのタイプごとにファイルがあると書き込みます。

let bunyan = require('bunyan');
let bformat = require('bunyan-format');
let formatOut = bformat({ outputMode: 'short' });
let fs = require('fs');
let path = process.cwd();

module.exports = function(name) {
    return bunyan.createLogger(
    {
        name: name,
        streams: [
            {
                level: 'error',
                stream: getStream('error.json')
            },
            {
                level: 'trace',
                stream: getStream('trace.json')
            },
            {
                level: 'debug',
                stream: getStream('debug.json')
            },
            {
                level: 'info',
                stream: getStream('info.json')
            },
            {
                level: 'warn',
                stream: getStream('warn.json')
            },
            {
                stream: formatOut
            }]
    });
};

function getStream(file) {
    return fs.createWriteStream(path + '/logs/' + file);
}

次に、Androidからログを受信するためのメソッドを記述します。優先順位付けの原因となる定数は、経験的に決定されました。

const logger = require('../../../utils/Bunyan')("logs.send");

const PRIORITY_INFO = "4";
const PRIORITY_WARN = "5";
const PRIORITY_VERBOSE = "2";
const PRIORITY_DEBUG = "3";
const PRIORITY_ERROR = "6";

module.exports = async function(req, res) {
    const priority = req.body.priority;
    const line = getLogLine(req);

    if (priority === PRIORITY_INFO) {
        logger.info(line)
    }

    else if (priority === PRIORITY_WARN) {
        logger.warn(line)
    }

    else if (priority === PRIORITY_VERBOSE) {
        logger.trace(line)
    }

    else if (priority === PRIORITY_DEBUG) {
        logger.debug(line)
    }

    else if (priority === PRIORITY_ERROR) {
        logger.error(line)
    }

    res.send({status: "ok"})
};

function getLogLine(req) {
    const tag = req.body.tag;
    const message = req.body.message;
    const date = req.body.date;

    return tag + ": " + message + ", when: " + date
}

これをapp.jsで記述して、ロギング用の小さなものを接続します。

const express = require('express');
const helmet = require('helmet');
const bodyParser = require('body-parser');
const logs_send = require('./routes/v1/logs/send');
const fs = require('fs');
const https = require('https');
const constants = require('./config');
const app = express();

app.set('view engine', 'jade');
app.use(helmet());
app.use(bodyParser.json());
app.use(bodyParser.urlencoded({ extended: false }));

app.use('/api/v1/logs.send', logs_send);
app.listen(3600);

module.exports = app;

しかし、一般的には2つの方法があります。上記のように（httpsなしで）実行する場合、Androidでは、CLEARTEXT通信を可能にするnetwork_security_configをロールインする必要があります。したがって、良い方法では、これを行う必要があります。ルーターでDDNSを作成し、（letsencryptを介して）このドメインの証明書を取得します。まあ、すでに証明書を使用してサーバーを起動します。
