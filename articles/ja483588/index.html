ILVを使用してサイトをブロックしようとする方法

みなさん、こんにちは！今日私は、ILV、映画のある他の誰かのサイト、そして私のサイトとは関係のない話をしたいと思います。

トラフィックが安定している仮想サーバーを見つけるためのWebサイトがあります。人気のあるリソースにサイトへのリンクが貼られているために、訪問者の数が急増することがありますが、これはめったに発生せず、訪問者がどこから来たのかをほぼ常に確認できます。新年の直前、ちょうどそのような状況が起こりました。サイトの毎日のオーディエンスはほぼ2倍に増え、訪問者がどこから来たのかを調べ始めました。ほとんどの場合、そのような急増の原因は技術的または技術的に近いリソースですが、この場合は異なりました。訪問者は、映画のある海賊版サイトのさまざまなページからサイトにアクセスし、最終的に私のサイトの404ページに到達しました。



当初はこの点を重視していませんでしたが、2日目は状況が再発し問題を整理することにしました。結局のところ、統計システムは私のサイトでクリックを記録しませんでした-その中に映画が含まれているサイトのページを開くと、私のサイトの404ページのフレームが全画面で開きます（https：//domain/404.php）。理由を知る前に、誰かが自分のサイトへのリンク切れを投稿していると確信していたため、切り替えたときにユーザーが404ページに到達した、つまり事故でしたが、今ではそうではないことがわかりましたが、誰かが意図的にインストールしました私のサイトとフレーム。経験上、YandexまたはGoogleのムービー（参照元と共に）を使用してサイトにアクセスすると、訪問者はサイトの実際のコンテンツを見て映画を見ることができますが、参照元がいない場合、訪問者は私のサイトの404ページを閲覧できることがわかりました。

少し考えた結果、故障率を10％から70％に上げることに加えて、この状況は何の害も引き起こさないという結論に達しました。何もせず、さらなるイベントを観察しないことにしました。突然、すべてが自然に修正されます。



しかし、さらに予想外の事態に発展しました。 1日後、特定の映画のあるサイトページを削除するようにリクエストするILVからの電子メールが、info @ poiskvps.ruの電子メールと、abuse @ ispiria.netフィルムのあるサイトホストの電子メールに届きました。



なぜこの手紙が主催者に宛てられたのか、私には質問がありませんでしたが、なぜそのような手紙を受け取ったのかははっきりしませんでした。ここにロジックがあります：ILVの専門家が禁止された映画でサイトを開き、大文字の「ページが見つかりません」の表記を無視しました（したがって禁止された映画はそこにありません）、関心のあるドメインに属さない電子メールをフッターで見た、そして達成感借金は彼に通知を送った。すべてがILV側から明らかです。サイトにアクセスし、電子メールアドレスを確認して通知を送信します。ただし、この常識による戦術はまったく当てはまりません。この状況で電子メールが表示された場合、たとえばinfo @ Google COM



馴染みのある数人のホスティング業者と話をして、私は状況が非標準であるというアドバイスを得ました、そしてできる唯一のことはILVに書き込むことです答えは彼らが間違っていたということです、そして私は映画サイトとは何の関係もありません私がやるべきだ。

これに加えて、404ページを別のアドレス（https：//domain/404a.php）に移動し、前の404番目（フレーム内）を完全に空にしました。

しかし、これでは効果がありませんでした。しばらくすると、別の通知が届きました。サイトの別のページで、映画と禁止されているコンテンツが見つかりました。

この話のほぼ終わりは、映画サイトがすでにブロックされている他のサイトのミラーとして認識され、ブロックされることになる4番目の通知を受け取ったことです。後で知ったように、サイトのミラーを見つけることに関するこのような通知では、ILVからの手紙に加えて、通信省からの手紙が届きましたが、届きませんでした。ここから、省が通知を送信するアドレスを確認していると結論付けることができます。



しばらくすると、サイトがブロックされたため、私のサイトへのトラフィックは通常に戻りました。それから年末年始が来て、私はすでにこの話を忘れていましたが、NG後の最初の就業日に、私は映画のサイトとは関係がないという情報をメモしたILVからの手紙を受け取りました。残念ながら、私が何をする必要があるかという質問に対する答えはありませんでした。



今まで、私はそれが何であったかを知りません：純粋な偶然（しかし、私はそのような偶然をほとんど信じていません）または誰かが私を困らせてサイトをブロックすることに決めました。私が検討していたオプションの1つは、私のサービスを使用している映画のサイトの所有者が仮想サーバーを見つけ、訪問者を感謝の意を込めて "追いつく"ことです。

予防策として、X-Frame-OptionsヘッダーまたはJavaScriptを使用して、サイトがフレームに表示されないようにすることができます。

if (window.parent.frames.length > 0)
{
window.stop();
}
