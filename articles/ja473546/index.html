インターネット上でHTTPサービスを作成する方法に関するマニュアルはたくさんあります。したがって、すぐに実装について説明します。私たちのサービスは3つのURLパターンで構成されています：

  • ディレクトリ -/ v1 /カタログ/ {ディレクトリ} / {アクション}
  • ドキュメント -/ v1 /ドキュメント/ {ドキュメント} / {アクション}
  • サービス -/ v1 / services / {Action}


記事が膨らまないように、参考書「命名法」の例を考えてみましょう。1Cでは、HTTPメソッドの異なるパスを使用できません。そのため、URLパターンは（/ v1 / catalogs / wares / {Action}であり、2つのHTTPメソッドGET、POSTが定義されています。他のパターンでは、GET、POSTも使用します。
  • / v1-今後、いくつかのAPIオプションをサポートできるようになります。URLテンプレートSpravochniki_v2を追加し、それに応じて/ v2で始まるパスを追加できます。
  • /カタログ/陶器/ - カタログすべてのディレクトリを統合します。これは方法の静的な部分です。ウェアは、命名法ディレクトリに準拠しています。1Cは "wares"をURL パラメータに入れます。それは私たち自身の方法を思い付くことに注意すべきです。そして、私たちも自分たちで1Cでそれらを処理する方法を考えます
  • {アクション} -ここでは、1Cから受け取りたいものを渡します。たとえば、getByParamまたはgetAll


レトロフィットに関して、baseUrlは次のようになります"http://192.168.0.1/unf/hs/inntsd/v1/"。

  • 192.168.0.1 -HTTPサービスが公開されているサーバー
  • / unf -1Cデータベース名
  • / hs -HTTPサービスの静的部分
  • / inntsdは、サービス全体のルートURLです。好きなように電話する
