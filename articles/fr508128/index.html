Flutter et applications de bureau

Ce n'est un secret pour personne que l'équipe de développement de Flutter s'efforce de s'assurer que ce cadre permet, à l'aide d'une base de code unique, de créer des applications pour une grande variété de plates-formes. Cela inclut iOS, Android, Windows, Linux, macOS et le Web. De plus, ces applications doivent être compilées dans un format spécifique à chaque plate-forme, et leur apparence ne doit pas être inférieure à l'apparence de jeux bien conçus. Chez Google, le framework Flutter est utilisé dans de nombreux projets: d' Assistant à Stadia , de Cloud Search à Blogger . D'autres sociétés utilisent Flutter, de ByteDance à Grab , de Nubank à MGM Resorts.. Tous bénéficient de la flexibilité et de la productivité du développement qu'offre Flutter.



Beaucoup d'entre vous souhaitent développer les capacités de bureau de Flutter, qui incluent Windows, macOS et Linux. Dans les sondages et sur GitHub, le développement de bureau est l'une des nouvelles fonctionnalités les plus remarquables de Flutter. Dans un avenir proche, nous allons parler davantage de ce que nous faisons. Nous pensons qu'il vaut la peine de commencer par un examen de ce qui a été fait par les différentes équipes responsables des capacités de Flutter. Bien que la prise en charge des applications de bureau en soit encore au stade de la version technique d'essai, un travail sérieux est en cours sur les capacités correspondantes.









Assemblage d'application



Nous avons récemment publié des profils d'application et des modes de construction pour Windows et Linux - en plus de la prise en charge existante de macOS. Par exemple, si vous utilisez la dernière version de Flutter, les applications Flutter peuvent être compilées en exécutables Windows avec la commande flutter build windows



. Pour ce faire, nous utilisons notre compilateur de niveau de production AOT , qui vous permet de créer du code machine pour l'architecture x64, qui peut être exécuté sur les ordinateurs correspondants sans installer Flutter.



Caractéristiques des applications de bureau



Que vous développiez une application représentée par un fichier exécutable autonome ou une application Web, les applications de bureau ont certaines fonctionnalités et fonctionnalités spécifiques. Par exemple, les fenêtres des applications de bureau sont généralement affichées en mode paysage et leur taille peut être modifiée. Les utilisateurs saisissent généralement des données dans des applications à l'aide d'un clavier et d'une souris, plutôt que sur des claviers à l'écran et une saisie tactile. Les contrôles sont optimisés étant donné que les applications de bureau utilisent des valeurs de densité visuelle spéciales lors du placement des contrôles.



Au niveau du framework, nous avons apporté quelques modifications à Flutter qui visent à prendre en charge les applications de bureau.

FFI



L'équipe Dart a fait un excellent travail pour améliorer l'interface de la fonction étrangère (FFI). Cela permet d'accélérer la mise en œuvre de nouvelles fonctionnalités dans Flutter. Par exemple, pour les API basées sur C, la bibliothèque dart:ffi



fournit un mécanisme direct pour organiser la liaison au code natif. Le runtime Dart permet d'appeler des bibliothèques liées dynamiquement et d'allouer de la mémoire sur le tas. Ce mécanisme est basé sur des objets Dart.



Voici un extrait de code ( ici vous pouvez trouver sa version complète), qui est un exemple simple qui montre un appel à l'API Win32 traditionnelle MessageBox()

Pour ce faire, vous pouvez utiliser la méthode lookupFunction()

Mise à jour du modèle de plugin



Flutter est conçu pour avoir un noyau compact. Pour équiper le framework de fonctionnalités supplémentaires, la fonctionnalité correspondante est réalisée sous la forme de plug-ins et de packages qui permettent l'intégration avec les systèmes d'exploitation sur lesquels Flutter fonctionne. Les plugins et packages peuvent être créés non seulement par les développeurs Flutter, mais également par d'autres programmeurs.



Cependant, étant donné que Flutter améliore la prise en charge des plates-formes mobiles et de bureau, ainsi que des applications Web, le développement de plugins pour chaque plate-forme prise en charge par le framework devient de plus en plus difficile. Lors du développement d'un certain plugin, il est très probable qu'il sera nécessaire de combiner les efforts de différents programmeurs, chacun étant bien versé dans une plate-forme.



Ici, une approche peut nous venir en aide, selon laquelle un certain plug-in de base décrit une interface commune qui peut être implémentée indépendamment sur différentes plateformes. En conséquence, comme cela a été décrit dans les derniers documents sur le développement de plug-ins modernes, nous avons récemment implémenté un système dans Flutter qui simplifie la collaboration de divers auteurs sur les plug-ins. Grâce à ce système, il est déjà possible d' indiquer explicitement quelles plateformes sont supportées par un plugin particulier.



Nous avons commencé à utiliser ce modèle en travaillant sur certains plugins de base. Des exemples d'une nouvelle approche de développement de plug-in collaboratif peuvent être trouvés ici .



Veuillez noter que le plugin API pour Windows et Linux n'a pas encore été établi.Par conséquent, bien que nous conseillons aux développeurs de se familiariser avec les nouvelles fonctionnalités de Flutter, nous ne sommes pas encore prêts à publier ces fonctionnalités en production. Nous travaillons également sur l'ajout de balises de plate-forme de bureau à pub.dev.



Exécution de programmes sous Windows: Win32 et UWP



Un aspect intéressant de notre travail lié à Windows est que nous expérimentons ici différentes approches d'architecture. Flutter, sur n'importe quelle plate-forme, est intégré dans une petite application hôte, qui est un conteneur («embedder») utilisant une approche similaire à celle utilisée dans les moteurs de jeux comme Unity. Cette application d'intégration, spécifique à chaque plateforme, nous fournit un point d'entrée au projet. Il coordonne ses actions avec le système d'exploitation, en travaillant avec des outils de rendu, avec la saisie des utilisateurs et avec des outils qui assurent la disponibilité du contenu. Il prend également en charge la boucle d'événements.









Windows nous donne deux façons de créer une telle application. Le premier est un modèle de programmation Win32 mature qui peut être utilisé comme point d'entrée pour des projets Flutter. Cette approche offre le plus haut niveau de compatibilité descendante avec diverses plates-formes, telles que Windows 7. Elle vous permet de créer des fichiers EXE standard que de nombreux programmeurs s'attendent à recevoir à la suite de leur travail sur un projet. Contrairement à Win32, le modèle d'application UWP moderne est recommandé pour Windows 10. Ce modèle offre des fonctionnalités attrayantes pour la création d'applications Flutter pour des appareils spéciaux, tels que la Xbox, et pour Windows 10X , qui devrait bientôt sortir.



Officieusement, nous travaillons avec divers développeurs et recherchons différentes solutions. Nous aimerions travailler plus étroitement avec Microsoft afin d'améliorer la qualité des applications Windows créées avec Flutter. Étant donné que la famille d'appareils Surface comprend désormais des systèmes Android et Windows, nous pensons que Flutter est en mesure d'offrir à Microsoft une plate-forme puissante qui vous permet de créer des applications natives attrayantes pour une grande variété d'appareils.



Expériences d'applications de bureau



Étant donné que les fonctionnalités qui ont été discutées sont encore au stade de la version technique d'essai, l'API et les outils auxiliaires sont toujours instables. Nous avons prévu de nombreuses autres tâches à effectuer avant de transférer la prise en charge des applications de bureau dans la catégorie des fonctionnalités stables. Cela comprend, en outre, l'amélioration de l'accessibilité et de la localisation du contenu.



Si vous souhaitez déjà découvrir de nouvelles opportunités, vous devez utiliser le canal de développement. La prise en charge de Windows et Linux est disponible dans la branche master



où Flutter se développe activement. Le support MacOS est disponible dans la branche dev



. Il existe des fonctionnalités plus stables, qui ne sont toutefois pas recommandées pour une utilisation en production. Vous pouvez basculer entre les canaux à l'aide des commandes flutter channel master



ouflutter channel dev



. Après cela, vous devez exécuter l'une des commandes suivantes pour activer la prise en charge de la plate-forme qui vous intéresse: 



C:\flutter> flutter config --enable-windows-desktop
$ flutter config --enable-macos-desktop
$ flutter config --enable-linux-desktop


Nous avons déjà vu comment de courageux développeurs ont commencé à créer des applications de bureau avec les nouvelles fonctionnalités de Flutter. L'une des premières applications de ce type a été développée pour macOS. Il s'appelle Sharezone . Il s'agit d'un planificateur d'étudiants destiné au marché allemand de l'éducation. Au début, ce projet a été publié sous forme d'application mobile. Et récemment, il est sorti dans la version bureau et dans la version destinée au web.









Sharezone Schulplan est une application pour les étudiants, les enseignants et les parents. Il vous permet de planifier et de suivre les devoirs, les groupes d'étude et les horaires.
