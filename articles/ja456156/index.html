だから学校代数が必要です。

通常、「なぜ数学が必要なのか」という質問です。 「心の体操」のようなものに答えてください。私の意見では、この説明は十分ではありません。人が身体運動をするとき、彼は発達している筋肉群の正確な名前を知っています。しかし、数学についての話はあまりにも抽象的なままです。学校代数で訓練されている具体的な「マインドマッスル」は何ですか。それは、本当の数学とはまったく異なり、すばらしい発見がなされます。いくつかの紛らわしい機能の派生物を検索する機能を提供するものは何ですか？

弱い生徒にプログラミングを教えることで、「なぜ？」という質問に対するより正確な回答が得られました。記事ではそれを伝えたいと思います。


学校では、表現の変換と単純化にかなりの時間が費やされています。例：81x 2 + 126xy + 49y 2は（9x + 7y）2として変換する必要があります。

  • 二乗量

代数学の普通の学校では、ほとんどいつもそのような表現の変換に従事していた。高等数学の大学では、大きな変化はありません。デリバティブ（インテグラルなど）の取り方を教えられ、たくさんのタスクが与えられました。これは役に立ちましたか？私の意見では、はい。そのような演習の結果として：

  1. 式を変換するスキルが完成しました。
  2. 細部へのこだわりが生まれました。
  3. 理想が形成されました-努力できる簡潔な表現。

私の意見では、このような理想、品質、スキルの存在は、開発者の日常業務において非常に役立ちます。実際、表現を単純化するということは、本質的に、意味に影響を与えることなく、理解を容易にするためにその構造を変更することを意味します。これは何かを思い出させますか？

これは実際には、Martin Fowlerによる同じ名前の本からのリファクタリングの定義です。

彼の研究では、著者はそれらを次のように定式化しています。
リファクタリング（n）：観察された動作に影響を与えることなく、操作の理解を容易にし、変更を単純化することを目的とした、ソフトウェアの内部構造の変更。
リファクタリング（動詞）：動作に影響を与えずに一連のリファクタリングを適用して、ソフトウェアの構造を変更します。
この本は、ソースコードで認識される必要がある「式」とそれらの変換のルールを提供します。

簡単な例として、本から「説明変数の紹介」を引用します。

if ( (platform.toUpperCase().indexOf(“MAC”) > -1 ) &&
    (browser.toUpperCase().indexOf(“IE”) > -1 )&&
    wasInitialized() && resize > 0 ) {
    // do something
}

式の一部は変数に書き込む必要があります。変数の名前はその目的を説明しています。

final boolean isMacOS = platform.toUpperCase().indexOf(“MAC”) > -1;
final boolean isIEBrowser = browser.toUpperCase().indexOf(“IE”) > -1;
final boolean isResized = resize > 0;
if(isMacOS && isIEBrowser && wasInitialized() && isResized) {
   // do something
}

二乗和の公式と二乗の差を使用して代数式を簡略化できない人を想像してみてください。

この人はコードをリファクタリングできると思いますか？

もし彼がこの簡潔主義の理想を持っていなければ、彼は他の人にも理解できるコードを書くことができるでしょうか？私の意見では、いいえ。

しかし、誰もが学校で勉強し、少数派はプログラマーになります。表現変換スキルは一般の人に役立ちますか？私はイエスと思う。スキルのみがより抽象的な形で適用されます。状況を評価し、目標に近づくために次のアクションを選択する必要があります。教育学では、この現象は転移（スキル）と呼ばれています。

最も顕著な例は、即席の手段である「集団農場」法による家庭の修理中に発生します。その結果、同じ「チップ」とライフハックが表示されます。その1つがKDPVに描かれています。アイデアの作者は、木片とワイヤーと4本のネジを持っていました。ランプホルダーのテンプレートを思い出して、彼はそれらから自家製のカートリッジを組み立てました。

ドライバーは自動車を運転しているときでも、常に周囲の世界でパターン認識を行い、目標に到達するための適切な操作を行っています。

あなたが亡くなったとき、あなたはそれについて知りません、他の人にとってのみ、それは難しいです。あなたが数学をマスターしなかったときと同じこと...
人が表現の変換を習得できない場合はどうなりますか？時々、私は学校で数学が苦手だった生徒に個別のレッスンをします。原則として、彼らはサイクルのトピックに完全にこだわっています。彼らは「代数」を扱わなければならないほどに、しかしプログラミング言語で。
これは、サイクルを記述するときの主なトリックは、まったく同じ式のグループを変換することです。

この結果を達成する簡単なプログラムは次のようになります。

static void Main(string[] args)
{
    Console.WriteLine("");
    Console.WriteLine(" 1");
    Console.WriteLine(" 2");
    Console.WriteLine(" 3");
    Console.WriteLine(" 4");
    Console.WriteLine(" 5");
    Console.WriteLine(" 6");
    Console.WriteLine(" 7");
    Console.WriteLine("");
}

しかし、この決定は簡潔な理想からはほど遠い。まず、その中のアクションの繰り返しグループを見つけ、それを変換する必要があります。結果は次のソリューションです。

static void Main(string[] args)
{
    Console.WriteLine("");
    for (int i = 1; i <= 7; i++)
    {
        Console.WriteLine(" " + i);
    }
    Console.WriteLine("");
}

一度に数学を習得しなかった場合、そのような変換を実行することはできません。彼は単に適切なスキルを持っていません。そのため、サイクルのトピックは、開発者のトレーニングにおける最初の障害です。

他の領域でも同様の問題が発生します。人が即興ツールの使い方を知らない場合、家庭に精通していることを示すことができません。邪悪な舌は、手が間違った場所から成長していると言うでしょう。路上では、これは状況を正しく評価できず、操縦を選択できないことで明らかになります。時には悲惨な結果をもたらす可能性があります。

調査結果：

  1. 私たちが持っている手段で世界をより良い場所にするためには、学校と大学の数学が必要です。
  2. 勉強していて、学習サイクルに問題がある場合は、原点に戻るようにしてください-学校代数。9年生の問題集を取り上げ、その例を解きます。
