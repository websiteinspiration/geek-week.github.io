Microsoft将不再颁发MCSE / MCSD / MCSA证书

五年来，在萨蒂亚·纳德拉（Satya Nadella）的明智领导下，微软失去了全球技术领导地位。尽管所有的努力都投入到了云计算方面，但Azure无法赶上并更绕过亚马逊的普及平台，而亚马逊是该市场的先驱和永久领导者。由于不想为Azure的开发吸引外部资金，该公司从其他方向大量投资，结果微软将Apple平板电脑和专业台式机，Google移动技术，游戏和AR / VR连续输给了所有人，Salesforce公司系统无法发布广告必应网络和搜索引擎，以及像女人一样体面的人甚至都不记得像样的Yammer社交网络。没有组织，被内部竞争内部吞噬，没有创新能力，最近，她只购买别人的开发成果：GitHub，LinkedIn，Xamarin和其他已经完成且成熟的项目。

从本质上讲，微软正朝着成为文本编辑器和Linux托管服务商的主要制造商迈进。当然，这样的公司不需要合作伙伴和开发人员，因此，在今年2月，它宣布将关闭针对开发人员和工程师管理员的认证计划，即：

  • MCSA：BI报表
  • MCSA：Dynamics 365 for Operations
  • MCSA：SQL 2016 BI开发
  • MCSA：SQL 2016数据库管理员
  • MCSA：SQL 2016数据库开发
  • MCSA：SQL Server 2012/2014
  • MCSA：通用Windows平台
  • MCSA：Web应用程序
  • MCSA：Windows Server 2012
  • MCSA：Windows Server 2016
  • MCSD：App Builder
  • MCSE：商业应用
  • MCSE：核心基础架构
  • MCSE：数据管理与分析
  • MCSE：生产力

从列表中可以看出，Microsoft相信在现代世界中，它们是不需要的：Web开发人员，开发人员和数据库管理员，台式机和平板电脑应用程序的开发人员，本地服务器的管理员。原计划在6月30日之前停止这些程序的认证。
