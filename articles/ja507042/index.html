私はgamedevにしたい：8人のプロからの27件の返信

これは、ゲーム開発者になりたいジョーンズのための資料です。8人のディレクターがアドバイスを提供します。開始する場所、面接での振る舞い、候補者が面接で評価する方法、通常のポートフォリオがない場合の対処方法、高等教育が必要ですか、インターンシップなどがあります。



5月末に開催された、ハイスクールオブビジネスインフォマティクスのハイダースクールオブエコノミクスがLeader-IDプラットフォームを通じて開催した円卓会議 からのヒントやその他の有用な情報のドライスクイーズです。トレーニングプログラムをサポートする一環として、専門家を集め、6月にgamedevにアクセスする方法を尋ねました。 誰がアドバイスをし、経験を共有したか：

空席を探す場所


ウラジミールプラノフ：Facebookでは、「ゲーム開発者」からのリクエストに応じて、空席が配置されている多くの興味深いグループを見つけたり、業界の人々がニュースを共有したり、履歴書を投稿したりできます。また、候補者として登録し、履歴書を作成し、返信し、カバーレターを書き、仕事を探すことができるinGame JobおよびTalents in Gamesプラットフォームもあります。

履歴書にあるべきもの



これは、Values Valueからの指示です。詳細はこちら

Alina the Wise：要約は構造化されている必要があり、短く（1ページ以内）、多彩ではありません：黒いテキスト、白い背景。婚姻状況は雇用主にとって重要かもしれませんが、それを証明する必要はありません。履歴書を含むファイルには、姓名で名前を付けることをお勧めします。

Oleg Dobroshtan：履歴書を特定の必要要員の要件に変更し、必要なスキルとキーワードを示して、雇用主の質問に答えることをお勧めします。以前の3つの仕事の場所を示すのに十分ですが、無関係な経験は破棄することをお勧めします。場所の頻繁な変更は、あなたについて否定的な印象を与える可能性があります。また、4〜5ポイントの責任、2〜3ポイントの成果、特定のプロジェクトでの実際の役割を反映する必要があります。

履歴書とインタビューの作成における間違い


ウラジミール・プラノフ：

  • 長すぎる、無関係な履歴書。
  • タスク、実績、プラットフォームに関する情報が不十分な要約。
  • もう1つの間違いは、申請者がどのプロジェクト（ジャンル、収益化）とどの会社で働くかを見ないことです。
  • 希望の給与を正しく策定すること、安くすること、自分を過大評価しないことは非常に重要です。

Roman Vasiliev：すべてのテストタスクを完了する必要があります。それらは、開発の方向を教えてくれ、少しの経験を与えるからです。そして、これは仕事に対する将来の態度を示しているので、テストの締め切りは失敗しないはずです。面接前に飲んだり、雇用主に腹を立てたり、友達を作ったり、卑猥な誓いをしたりしないでください。

どのような教育が必要ですか？


ウラジミール・プラノフ：ゲーム開発会社に入社するための教育の重要性は、立場によって異なります。プログラマーにとって、スキル、経験、プロジェクトのポートフォリオははるかに重要です。ゲームデザイナーにとっては、獲得できる知識とスキルも不可欠です。

一般に、高等教育を受ける必要はありません。必要なスキルを明確かつ簡潔に提供するため、十分なコースがゲーム開発者により高く評価される場合があります。


ただし、学習は常に必要です。これは視野を広げ、新しい情報を提供します。理論を知り、実践することが重要です。たとえば、ゲームメカニクスを作成および編集するには、数学の優れた能力が必要です。これは、教育プログラムで簡単に教えられていますが、これにより、一部の学生がより詳細に学習する動機になる場合があります。

Roman Vasiliev：多くの指導者は、教育を地殻としてではなく、真剣な研究に時間を費やした申請者の決意と学習能力の証拠として見ています。

教育は知識だけでなく、雇用に必要な連絡先やつながりも提供します。「ゲームプロジェクト管理」プログラムの学生の75％は、学習過程で、または卒業後、ゲーム開発の経験がないと感じています。

経験がない場合の対処法


Mikhail Kuzmin：申請者がgamedevの経験がない場合は、以前に何をしたかに関係なく、彼はおそらく最初のポジションにたどり着くでしょう。フロントエンドおよびバックエンドのプログラマーに加えて、制限はありません。希望するキャリアパスをよく理解すればするほど、それに沿って早く動けるようになり、1〜3年でうまくいきます。会社はこれをインタビューで見つけ、あなたの適性をさらに詳しく調べます。成功するためには、さまざまなジャンルの多くの見せかけが必要です。これは、市場を理解していること、それを伝えることができること、そして最初は少しでも機械的に多くのことを望んでいるという事実に表れています。主な職業：Q＆A、テクニカルサポート、場合によってはジュニアというプレフィックスが付いているすべてのもの（既にどこかで学習したり、関連するポートフォリオを持っている場合）。

TinyBuildは、オープンなSteamプロファイル（または他のプラットフォーム）へのリンクをゲームデザイナーに要求することがよくあります。これは、申請者と彼のジャンルの理解について多くのことを語ることができます。

開始ポジションには必須ではありませんが、英語は重要です。ただし、基本的な用語で対応できる必要があります。
時々、履歴書からほとんど何も必要ありません：手仕事をする動機と準備を示してください。
スターティングポジションは会社によって異なりますが、目が焼けるだけで十分な場合もあります。

プロジェクトが成功しなかった場合でも、ポートフォリオに含めることをお勧めします。これは、申請者が開発プロセスでほとんどの問題に遭遇し、作業を完了できたことを意味します。

ゲーム開発インターンシップ


Alina the Wise：
ITやメディアではインターンシップが必要ですが、ゲーム開発ではそうではありません。これには多くの企業リソースが必要です。
ゲーム開発では、各従業員がポイントタスクを行うか、指導します。そして、会社は一方と他方の両方に支払う必要があり、多くの場合、見返りに会う必要はありません。


Alina the Wise-CEO Value Value

Oleg Dobroshtan：ロシアのゲーム業界は若いため、インターンシップが普及することはまずありません。また、ゲーム開発者が企業のようではないという役割も果たします。これから何年も契約を結んで安定して動作することはできません。

1C Gamesや101XPなど、多くの企業では、最初の役職はインターンシップとして機能します。

候補者を押しのけることができるもの


Sergey Volkov：特定の収益化やジャンルに対して拒否や同情がある場合、これは重要です。求職者が雇用主について不満を言う印象に悪影響を及ぼします。通常、この症状は「6か月ごとに仕事が変わる」という兆候と一致します。また、毒性は微妙なニュアンスで現れる可能性があり、ウダレンカは決してそれを隠しません。さらに、評判は業界全体で急速に分岐しています。

一般的に、従業員の誠実さが重要です。


セルゲイ・ボルコフ-1C Game Studios / 1C EntertainmentのHRディレクター

出版社とのプロジェクトのインタビューまたはディスカッションで


Konstantin Sakhnov：非常に正しい考えは質問することです。パブリッシャーを探す際に、パブリッシャーがプロモーションに割り当てる金額など、詳細を知ることが重要です。そして、仕事を探しているなら、会社のどのプロジェクト管理モデルが受け入れられているか、そしてキャリア開発のためのオプションは何かを尋ねる価値があります。準備ができているなら、会社や出版社について多くを学び、質問を考え出してください-これはあなたを最高の側面から特徴づけます。そして、会社やパートナーがあなたの質問に答えられない場合、これはおそらく悪い兆候です。


Konstantin Sakhnov-JustForward出版社およびゲームプロデューサーMikhail Kuzminの共同所有者

：パブリッシャーは野心と具体的な目標を示す方が良いです。パブリッシャーのポートフォリオを一読する価値はあります。ゲームにパブリッシャーが必要な理由を明確に理解する必要があります。お金を探している、他のプラットフォームに移植している、などです。パブリッシャーのPCでのみインディーゲームを発行することは、もはや興味がありません。パブリッシャーはすべてのプラットフォームでゲームをリリースすることを望んでいますが、携帯電話でも同じようにリリースすることをお勧めします。

提案のヒント（プロジェクトのプレゼンテーション）


Konstantin Sakhnov：初心者の開発者が犯す一般的な間違い-彼らは複雑なコンセプトを考え出そうとしています。野心は良いですが、もっと単純なことをする方が良いです。

ミハイル・クズミン：熱意をもって集められたばかりのチームからの成功事例の例があります。Black Skylandsは、めったに使用されないエンジンで開発された、非常に美しい描画のゲームです。TinyBuildはプロジェクトにサインアップし、資金を提供し、モスクワのオフィスのレンタルを手伝いました。作者はチームに人員を不足させ、ゲームをUnityに転送し（ゲームをさまざまなプラットフォームに移植する方が簡単になるため）、現在彼らはそれを完了しています。

ゲームデザイナーは誰ですか？プロデューサーとはどのように異なりますか？


Konstantin Sakhnov：ゲームデザイナー-デザイナー：頭の中でゲームの機能、メカニズムを考え、後で実装する方法を知っている人。ゲームデザイナーは技術的な割り当てを記述します-メカニックと機能の詳細な説明、そしてそのような説明のおかげで他の従業員はアイデアを実装できます。ゲームデザイナーとプロデューサーの位置が異なる場合、プロデューサーはプロジェクトを「正当化」するだけです。つまり、ゲームにビジョンを与え、チーム全体のモーションベクトルを設定し、ゲームデザイナーがアイデアを具現化します-メカニズムを回転させ、ゲームを面白くする責任があります。

ゲームの仕組みがどのように登場するか


コンスタンティン・サクノフ：私たちが思いついたすべてのものは、意識的にまたは無意識に自然から、または他の著者の作品からコピーしています。この手法は、オープンかつ一貫して使用できます。

たとえば、次のように：将来のプロジェクトのすべての競合他社を確認し、すべてのアイデアを書き出し、他のプラットフォームや古いゲームから何かを変革して持ち込みます。アイデアは、現実の生活や本から引き出すことができ、行動を仮想に移すことができます。

別の方法があります：何かをコピーして要素を削除します。
最も簡単なオプションは、意識と視野を広げ、積極的に周囲を見回すことです。最も重要なこととして、一部のゲームを完全に他のプラットフォームに転送しようとしないでください。

Udevenkaとgamedevのオフィス


Oleg Dobroshtan：ゲーム開発ではリモートで作業できますが、ライブでのコミュニケーションがより便利で迅速なので、コミュニケーションの位置と頻度に依存します。将来は混合モデルになりそうです。オフィスが小さくなり始め、家にいる機会が増えるでしょう。


Oleg Dobroshtan-101xp

Alina the Wiseの開発およびトレーニングディレクター：離れた場所にいるマネージャーにとって、問題を音声で伝える能力は非常に重要です。感情を見ずに人を理解することは非常に難しいためです。プロセスは機能しますが、感情的な経験は数えるのが困難です。問題や問題について話し合うことができるように準備しておくことも必要です。

多くの企業が離れた場所に移動する傾向は明らかです。以前は従業員がリモートで作業することを検討する準備ができていなかった多くの組織は、現在それほど厳格ではありません。

Konstantin Sakhnov：そのようなフォーマットがあります-チームとのまれなミーティングでのリモートワーク。このような例を検討すると、新しい従業員をリモートチームに接続するプロセスを整理することがいかに重要であるかが明らかになります。たとえば、Playrixでは、これですべてがうまくいきます。

ゲーム開発がスーパードリームの場合


Mikhail Kuzmin：ゲーム開発者をロマンチックにしないでください。これは他の仕事と同じです。違いは2つだけです。ゲームを楽しくする必要があります。そうしないとゲームは販売されません。また、開発サイクルは、企業がビジネスのために何かを作成する場合よりもはるかに短くなります。しかし、gamedevはその中で働く人々に喜びをもたらします。

Game Burning Professional


Oleg Dobroshtan：休息または運動は、燃え尽き症候群から助けることができます。「レッツダンス」シリーズのように生活して仕事をしている場合、バーンアウトは大きな問題にはなりません。

チャンス、展望、雇用について


セルゲイ・ボルコフ：特に経験者によると、ゲーム開発者は不親切で、初心者が障害に満ちているように見えることがよくありますが、会社と候補者がお互いを見つけ、協力が相互に有益で楽しいことがわかります。

そして、覚えておいてください、ゲーム開発は長い時間です。ここから脱出するのは難しいです。ドライブ、面白いタスク、ゲームには非常に中毒性があります。他の地域でより高い給与の地位に去った多くの人が戻ってきました-たとえ戻ったとしても、より低い給与と地位にさえ。そして、これは決して退屈することはありません。業界は非常にモバイルであり、トレンドは常に変化しています。

オレグ・ドブロシュタン：Gamedevは創造性についてですが、これには関係なく、やるべきことはたくさんあります。そして、彼らはゲームを撃ち、パターンでさえ作られましたが、それでも創造的に組織されました。クリエーター自身の「マネジメント」の過程を忘れないでください。

ただし、「管理」は価値があります。コミュニケーションを習得し、一部の雇用主がそのようにするのを待たないことが重要です。しかし、ゲーム開発者はあなたを手放せず、彼をあなたに連れて行きます-そしてあなたは幸せになります。

Vyacheslav Utochkin：数年後、あなたが正しい決断をし、並外れた喜びと経済的安定をもたらす活動に従事し始めたと感じたら素晴らしいでしょう。

ウラジミール・プラノフ：ここにゲーム開発者向けの優れたライブラリ-本、コース、記事、ポッドキャスト、ゲーム開発者に関するグループおよび講義-優れたセレクションがあります。

Roman Vasiliev：恐れる必要はありません。試す必要があります。すべての要件が重要なわけではないため、欠員に対応する価値がある場合があります。いずれにしても、これは何もしないよりはましです。
