Banana Pi R64 OpenWrtに最適なルーターかどうか



こんにちは、私の名前はDmitryです。今日は、Banana Pi R64シングルボードコンピューターを確認します。OpenWrtをインストールする方法と、このコンピューター自体のファームウェアをアセンブルする方法を学びます。

だから、Habréにこのコンピュータの概要すでに存在していることがあり、ここでは、しかし、著者はArmbianは、OSとしてそこに使用しています。私はこのOSとDebianを試しましたが、ルーターを作成するためのOpenWrtに勝るOSはないという結論に達しました。理由は次のとおりです。

  1. OpenWrtでは、ネットワークはユーザーの関与なしに自動的に構成されます。DebianとArmbianでは、設定ファイルを使用して、自分でネットワークを設定する必要があります。
  2. OpenWrt Webインターフェースは、ルーター管理用に設計されています。DebianとArmbianにもWebインターフェース（Webminなど）がありますが、一般的なサーバー管理用に設計されています。
  3. DebianとArmbianの利点のうち、より単純なファイルシステムが注目され、そのまま表示されます。OpenWrtには非常に複雑なファイルシステムがあります。複数のマウントポイントで構成され、そのうちのいくつかは読み取り専用属性を持っています。

openwrtのインストール方法


このマニュアルは、公式のBanana PIフォーラム（forum.banana-pi.org/c/Banana-Pi-BPI-R2）から引用しています。

OpenWrtをインストールするには、次のものが必要です。

  1. マイクロSDカード
  2. エンターネットケーブル
  3. USB-UARTアダプター
  4. Win32ディスクイメージャ
  5. TFTPサーバー
  6. パテ
  7. カードのSD画像
  8. デバイスの内部メモリにアップロードするImg画像
  9. このイメージをダウンロードするために必要なプリローダー

手順：

  1. Win32 Diskimagerを使用して、SDカードにSDイメージを塗りつぶします
  2. Usb-Uartアダプターをボード上の対応する接点（txルーター-rxコンバーター、rxルーター-txコンバーター、アースに接続）に接続し、Puttyに移動してComポートに接続します。データ転送速度は115200です。必要なComポートを確認できます。通常、デバイスマネージャはCOM4です。
  3. カードをバナナPIに挿入し、電源を入れます。
  4. U-Bootメニューで、「b。システムロードフラッシュイメージ、次にTFTP経由でフラッシュに書き込む '（TFTPサーバーを有効にする必要があります。イメージファイルはTFTPサーバーと同じフォルダーに配置する必要があります。ネットワークカードの設定で、IPアドレスとメインゲートウェイを指定する必要があります）。
  5. ファームウェアの後、bootmenuコマンドと '7オプションを使用してU-Bootメニューに戻ります。システムロードプリローダーは、TFTP経由でフラッシュに書き込み、プリローダーに入力します（このファイルは、TFTPサーバーと同じフォルダーにある必要があります）。
  6. ルーターを再起動すると、OpenWrtが表示されます。U-bootメニューがSDカードなしで利用できるようになりました。起動時に常に3秒間表示されます。

残念ながら、デバイスにアップロードした画像は非常に古いものです。したがって、たとえば、追加のアプリケーション（パッケージ）をインストールすることはできません。

OpenWrtはどこで入手できますか？


この記事の執筆時点では、Bpi-R64向けのOpenWrtの安定版は存在していません。このデバイスは開発ブランチでサポートされていますが、現時点では互換性は完全ではありません。つまり、initramfsイメージをロードすれば機能します。ただし、このイメージはOpenWrtに慣れるために作成されたもので、RAMにのみ存在し、デバイスを再起動すると、すべての設定とインストールされたアプリケーションが失われます。 sysupgradeイメージは現在機能していません。インストールすると、デバイスは常に再起動します。

したがって、動作するsysupgradeイメージを取得する唯一の方法は、公式のBanana PIフォーラムのパッチを使用して自分でビルドすることです。

2020年4月16日からOpenWrtイメージを既に組み立てています。このイメージ画像。

  2. NTFS-3Gドライバーの詳細はこちら
  3. サンバ
  4. mc、nano、htop、その他のユーティリティ

機能するOpenWrtイメージを構築する方法


組み立てとその準備のプロセスは非常に膨大であるため、要点のみを説明します。

1）ビルドするには、仮想マシンとLinuxイメージが必要です。それらの準備方法と入手場所については、ここで説明します。仮想マシンの設定では、必ずプロセッサのコア数を指定してください（デフォルトでは1）。カーネルアセンブリプロセスは非常に長く、1つのコアではさらに長くなります。

2）その後、Gitからソースをダウンロードする必要があります。その方法はここで説明されています。 make menuconfigコマンドで停止する必要があります。3

）openwrt / target / linux / mediatek / makefileファイルに移動し、カーネル番号を5.4から4.19に変更します。現在、5.4カーネルでOpenWrtイメージをビルドすることはできません。

4）menuconfigでTarget-> Mediatck Arm、Subtarget-> mt7622、Profile-> Banana Pi R64を選択します。必要な追加パッケージを選択することもできます。それから私達は保全で終了します。

5）make defconfig理由はわかりませんが、常にこのコマンドを実行します。

6）kernel_menuconfig -j5を、jがプロセッサのコア数+ 1として計算された後の数にします（4コアプロセッサを使用しています）。

7）30分待機した後、カーネル構成メニューがここに表示されます。次のオプションを設定する必要があります：
「デバイスドライバー」->「メモリーテクノロジーデバイス（MTD）サポート」->「コマンドラインパーティションテーブルの解析」
「デバイスドライバー」- > "Memory Technology Device（MTD）support" "Self-contained MTD device drivers"-> "MTD using block device"
「デバイスドライバー」->「MMC / SD / SDIOカードのサポート」->「MMCブロックデバイスドライバー」

8）このパッチを入手して、ここにコピーします：openwrt / target / linux / mediatek / patch-4.19。（私はこのパッチが常にコンパイルエラーを引き起こしたことに注意したいと思います。それゆえ、私はここからそこにあるmt7622-bananapi-bpi-r64.dtsファイルに行をコピーしました：openwrt / linux / mediatek / files-4.19 / arch / arm64 / boot / dts / mediatek）

9）make -j5

10）コンパイル後、u-bootメニューオプションからフラッシュできるopenwrt-mediatek-mt7622-bpi_bananapi-r64-squashfs-sysupgrade.binファイルがbinフォルダーにあります。2）システムロードLinuxカーネルは、TFTP経由でフラッシュに書き込みます。

結論


バナナパイは非常に物議を醸しているデバイスです。高性能と開発者による醜いサポートが組み合わされています。公式のファームウェアはありませんが、サードパーティの開発者によるサポートはまあまあです。しかし、それでも5ギガビットポート、1ギガバイトのRAM、高速デュアルコアプロセッサを備えた非常に強力なルーターです。つまり、通常のルーターのようなものはありません。Transsmisonをダウンロードして、ルーターがすべてのきしみを消滅させるかのようにパフします。ここで、トランスミッションの操作はデバイスの応答性に影響を与えません。

デバイスがケースなしで販売されていることも注目に値します。したがって、体は自分でそれを行わなければなりません。しかし、3Dプリンターを使用している場合、これは問題ではありません。ここに私のバージョンがあります
