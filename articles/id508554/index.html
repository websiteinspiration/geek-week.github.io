Cara menentukan ukuran variabel saat runtime program Go

Abstrak: catatan ini membahas salah satu metode untuk menganalisis konsumsi memori oleh komponen aplikasi Go.



Seringkali, struktur data disimpan dalam memori program yang mengubah ukurannya secara dinamis saat program berjalan. Contoh struktur seperti itu bisa berupa cache data atau log operasi program, atau data yang diterima dari sistem eksternal. Dalam kasus ini, situasi dapat muncul ketika konsumsi memori meningkat, kemampuan peralatan tidak cukup, dan mekanisme kebocoran spesifik tidak jelas.



Cara utama untuk profil aplikasi Go adalah untuk menghubungkan alat pprof dari paket "net / http / pprof". Sebagai hasilnya, Anda bisa mendapatkan tabel atau grafik dengan alokasi memori dalam program yang sedang berjalan. Tetapi menggunakan alat ini membutuhkan overhead yang sangat besar dan mungkin tidak dapat diterapkan, terutama jika Anda tidak dapat menjalankan banyak contoh program dengan data nyata.



Dalam hal ini, ada keinginan untuk mengukur konsumsi memori oleh objek program sesuai permintaan, misalnya, untuk menampilkan statistik sistem atau mentransfer metrik ke sistem pemantauan. Namun, dengan bahasa berarti ini umumnya tidak mungkin. Go tidak memiliki alat untuk mengukur variabel saat program sedang berjalan.



Karena itu, saya memutuskan untuk menulis paket kecil yang memberikan kesempatan seperti itu. Alat utama adalah refleksi (paket "refleksi") Saya mengundang semua yang tertarik dengan masalah aplikasi pembuatan profil semacam itu untuk membaca lebih lanjut.







Pertama, beberapa kata tentang fungsi bawaan. 



unsafe.Sizeof(value)


dan 



reflect.TypeOf(value).Size()


Fungsi-fungsi ini setara dan sering direkomendasikan di Internet untuk menentukan ukuran variabel. Tetapi fungsi-fungsi ini tidak mengembalikan ukuran variabel aktual, tetapi ukuran dalam byte untuk wadah variabel (kira-kira - ukuran pointer). Misalnya, untuk variabel tipe int64, fungsi-fungsi ini akan mengembalikan hasil yang benar, karena variabel jenis ini berisi nilai aktual, dan bukan referensi untuk itu. Tetapi untuk tipe data yang berisi pointer ke nilai aktual, seperti slice atau string, fungsi-fungsi ini akan mengembalikan nilai yang sama untuk semua variabel dari tipe ini. Nilai ini sesuai dengan ukuran wadah yang berisi tautan ke data variabel. Saya akan mengilustrasikan dengan sebuah contoh: 



func main() {
	s1 := "ABC"
	s2 := "ABCDEF"
	arr1 := []int{1, 2}
	arr2 := []int{1, 2, 3, 4, 5, 6}
	fmt.Printf("Var: %s, Size: %v\n", s1, unsafe.Sizeof(s1))
	fmt.Printf("Var: %s, Size: %v\n", s2, unsafe.Sizeof(s2))
	fmt.Printf("Var: %v, Size: %v\n", arr1, reflect.TypeOf(arr1).Size())
	fmt.Printf("Var: %v, Size: %v\n", arr2, reflect.TypeOf(arr2).Size())
}


Sebagai hasilnya, kita mendapatkan: 



Var: ABC, Size: 16
Var: ABCDEF, Size: 16
Var: [1 2], Size: 24
Var: [1 2 3 4 5 6], Size: 24


Seperti yang Anda lihat, ukuran sebenarnya dari variabel tidak dihitung.



Pustaka standar memiliki fungsi binary.Size () yang mengembalikan ukuran variabel dalam byte, tetapi hanya untuk jenis ukuran tetap. Yaitu, jika string, slice, array asosiatif, atau hanya int ditemukan di bidang struktur Anda , maka fungsinya tidak berlaku. Namun, saya menganggap fungsi ini sebagai dasar paket ukuran , di mana saya mencoba memperluas kemampuan mekanisme di atas ke tipe data tanpa ukuran tetap.



Untuk menentukan ukuran suatu objek selama operasi program, perlu untuk memahami tipenya, bersama dengan jenis semua objek bersarang, jika itu adalah struktur. Struktur akhir yang perlu dianalisis umumnya disajikan dalam bentuk pohon. Oleh karena itu, rekursi harus digunakan untuk menentukan ukuran tipe data yang kompleks.



Dengan demikian, perhitungan jumlah memori yang dikonsumsi untuk objek arbitrer diwakili sebagai berikut:



  • algoritma untuk menentukan ukuran variabel tipe sederhana (non-komposit);
  • panggilan rekursif dari algoritma untuk elemen array, bidang struktur, kunci, dan nilai array asosiatif;
  • definisi loop tak terbatas;


Untuk menentukan ukuran aktual dari variabel tipe sederhana (bukan array atau struktur), Anda dapat menggunakan fungsi Size () dari paket refleksi di atas . Fungsi ini berfungsi dengan benar untuk variabel yang berisi nilai aktual. Untuk variabel yang merupakan array, string, mis. berisi tautan ke nilai yang Anda butuhkan untuk menelusuri elemen atau bidang dan menghitung nilai setiap elemen.



Untuk menganalisis jenis dan nilai variabel , paket refleksi mengemas variabel menjadi antarmuka kosong ( antarmuka {} ). Di Go, antarmuka kosong dapat berisi objek apa pun. Selain itu, antarmuka di Go diwakili oleh wadah yang berisi dua bidang: jenis nilai aktual dan tautan ke nilai aktual.



Itu adalah pemetaan nilai yang dianalisis ke dalam antarmuka kosong dan sebaliknya yang berfungsi sebagai dasar untuk nama metode itu sendiri - refleksi .



Untuk pemahaman yang lebih baik tentang cara kerja refleksi di Go, saya merekomendasikan artikel Rob Pike di blog resmi Go. Terjemahan artikel ini ada di Habré.



Pada akhirnya, paket ukuran dikembangkan , yang dapat digunakan dalam program Anda sebagai berikut: 



package main

import (
	"fmt"

	"github.com/DmitriyVTitov/size"
)

func main() {
	a := struct {
		a int
		b string
		c bool
		d int32
		e []byte
		f [3]int64
	}{
		a: 10,                    // 8 bytes
		b: "Text",                // 4 bytes
		c: true,                  // 1 byte
		d: 25,                    // 4 bytes
		e: []byte{'c', 'd', 'e'}, // 3 bytes
		f: [3]int64{1, 2, 3},     // 24 bytes
	}

	fmt.Println(size.Of(a))
}

// Output: 44


Catatan:



  • Dalam praktiknya, menghitung ukuran struktur volume sekitar 10 GB dengan bersarang besar memakan waktu 10-20 menit. Ini adalah hasil dari fakta bahwa refleksi adalah operasi yang agak mahal, membutuhkan setiap variabel untuk dikemas ke dalam antarmuka kosong dan analisis selanjutnya (lihat artikel di tautan di atas).
  • Sebagai hasil dari kecepatan yang relatif rendah, paket harus digunakan untuk menentukan perkiraan ukuran variabel, karena dalam sistem nyata selama analisis struktur besar data aktual mungkin akan memiliki waktu untuk berubah. Atau berikan akses eksklusif ke data pada saat perhitungan menggunakan mutex, jika berlaku.
  • Program tidak memperhitungkan ukuran "wadah" untuk array, antarmuka dan array asosiatif (ini adalah 24 byte untuk array dan slice, 8 byte untuk peta dan antarmuka). Karena itu, jika Anda memiliki sejumlah besar elemen dengan ukuran kecil, maka kerugiannya akan signifikan.
