1つのモバイルMMORPGをプレイしてきた長年にわたって、リバースエンジニアリングの経験をいくつか積んできたので、シリーズ記事で紹介したいと思います。トピックの例：

  1. サーバーとクライアント間のメッセージ形式を解析します。
  2. リスニングアプリケーションを作成して、ゲームトラフィックを便利な方法で表示します。
  3. 非HTTPプロキシサーバーを使用したトラフィックの傍受とその変更。
  4. 自分の（「海賊版」）サーバーへの最初のステップ。

この記事では、サーバーとクライアント間のメッセージ形式の解析について説明します。猫の下で興味を持ってください。

必要な工具


下記の手順を繰り返すには、次のものが必要です。

  • PC（私はWindows 7/10で実行しましたが、以下のアイテムがそこで利用可能であればMacOSも動作する可能性があります）;
  • パケット分析用のWireshark。
  • テンプレートによってパケットを解析するための010Editor（オプションですが、メッセージ形式をすばやく簡単に説明できます）。
  • ゲームのモバイルデバイス自体。

さらに、オブジェクト、クリーチャーなどのリストとその識別子など、ゲームからすぐに利用できるデータを用意することは非常に望ましいです。これにより、パッケージ内の重要なポイントの検索が大幅に簡素化され、場合によっては、一定のデータストリームで目的のメッセージをフィルター処理できるようになります。

  • 0x08期待値の前に理解できない;

初期値を取得する方法がわかったので、型のテンプレートを作成できます。

struct VLQ {
    local char size = 1;
    while(true) {
        byte obf_byte;
        if ((obf_byte & 0x80) == 0x80) {
            size++;
        } else {
            break;
        }
    }
    FSeek(FTell() - size);
    byte bytes[size];
    local uint64 _ = FromVLQ(bytes, size);
};

そして、バイトの配列を整数値に変換する関数：

uint64 FromVLQ(byte bytes[], char size) {
    local uint64 source = 0;
    local int i = 0;
    local byte x;
    for (i = 0; i < size; i++) {
        x = bytes[i];
        source |= (x & 0x7F) * Pow(2, i * 7); //   <<   , ..     ,  uint32,        uint64
        if ((x & 0x80) != 0x80) {
            break;
        }
    }
    
    return source;
};

struct Packed {
    VLQ marker <bgcolor=0xFFBB00>; //    VLQ!
    local uint size = marker.size; //       ( , )         
    switch (marker._ & 0x7) {
        case 1: double v; size += 8; break; //    
        case 5: float v; size += 4; break;
        default: VLQ v; size += v.size; break;
    }
};

struct PackedVector3 {
    Packed marker <name="Marker">;
    
    Packed x <name="X">;
    Packed y <name="Y">;
    Packed z <name="Z">;
};

これでモーションメッセージテンプレートが大幅に削減されました。

struct MoveEvent {
