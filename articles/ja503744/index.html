遠隔作業条件でのワークステーションインベントリの3つのケース

オフィスに物理的に不在のときにワークステーションを管理することは、従業員をリモート操作モードに移行させた企業にとって新たな課題です。あなたが管理者権限を持っている場合、ユーザーは誰かの悪意の悪用であることが判明したソフトウェアを誤ってインストールする可能性があります。たとえば、Google、SpaceXなどの有名な企業や政府機関でZoomが拒否されたケースは多くの人が知っています。



この記事では、同じズームのワークステーションでの検出を含め、Quest KACEのソフトウェア配布とインベントリのソリューションの使用例を分析します。カットの下の詳細。

ズームを見つけて無効にする方法


Zoomは油断ならないもので、Program Filesにはインストールされません。代わりに、ワークステーションにインストールしたユーザーの個人用フォルダー（** someuser ** \ AppData \ Roaming \ Zoom \ bin \ zoom.exe）に配置されます。レジストリでその痕跡を見つけることもできますが、なぜ複雑なのでしょうか。



会社のポリシーで一部のソフトウェアのインストールが禁止されていて、そのソフトウェアがわかっている場合は、ワークステーションにインストールされているKACEエージェントを介して専用のシェルスクリプトを実行することで、簡単に見つけることができます。KACEには、カスタムインベントリルールのような概念があります。これを使用して、検出された各デバイスに追加のインベントリ情報をバインドできます。たとえば、どこを検索すれば、c：\ Users \ ** someuser ** \ AppData \ Roaming \ Zoom \ bin \ zoom.exeという形式のすべてのディレクトリで、zoom.exeを再帰的に検索できます。

チームリスト 
ShellCommandTextReturn(CMD /q /c for /d %u in (\Users\*) do (if exist %u\AppData\Roaming\Zoom\bin\zoom.exe echo %u))




次のインベントリサイクルの後、Zoomがインストールされた各ワークステーションで、ユーザーフォルダを示すエントリを見つけます。



PDFレポート（txt、csv、html、xls形式も利用可能）を生成して、関心のある同僚に送信するために残っています。レポートは定期的に生成できます。



Zoomの無許可のインストールを識別した後、既知のアンインストールコマンドによって削除することもできます。ちなみに、ディレクトリを検索するだけでなく、同じスクリプトからWindowsレジストリのZoomについて質問することもできます。

ユーザーデバイスの外部IPアドレスの確認


パンデミックのもう1つの一般的なケースは、ユーザーの外部IPアドレスを識別することです。離れた場所にいる従業員の場所のリストやその他の種類のレポートをまとめるのに役立ちます。

チームリスト 
ShellCommandTextReturn(nslookup myip.opendns.com resolver1.opendns.com)


各ワークステーションの結果は次のようになります。最初のアドレスはリゾルバーアドレスで、2番目のアドレスはユーザーアドレスです。



この情報は、レポートの形式で発行し、必要に応じて送信することもできます。スクリプトは、地理的な場所やプロバイダーなど、複雑で要求される場合があります。

Windows KMSライセンス認証ライセンスの有効期限チェック


通常の状況では、社内にオペレーティングシステムをアクティブ化するKMSサーバーがあります。ライセンス認証に期限があり、KMSサーバーに内部ネットワークからのみアクセスできる場合、リモートワークステーションでのWindowsの拡張はできません。非アクティブなワークステーションを識別するために、Windowsには特別なslmgrがあります。

チームリスト 
ShellCommandTextReturn(cscript.exe c:\windows\system32\slmgr.vbs /dli)
