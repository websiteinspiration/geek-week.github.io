音楽ジェネレーター Web Audio API 初心者体験

コンテキストと背景


私は62歳です。約3年前、かなり複雑なシステムを作成することを決心しました。それまでは、1981年に3か月のコンピュータサイエンスが5年目の無線学部（NSTU、以前はNETI）で私たちを悩ませていた以前は、基本的に20行しかありませんでした。専門家は主題を所有し、それで稼ぐ人であると考えられているので、実際、私は初心者です。

一方で、音楽関連のアルゴリズムをほぼ一生涯扱ってきて、開発プロセスとハードウェア、ソフトウェアを見つけ、可能な限りそれに参加しました。 1978年に、彼はミニムーグシンセサイザーを開発および組み立て（私はそれがウラルで最初だと思います）、ソ連でAgat7（9）用のシンセサイザーを備えた最初のソビエトサウンドカードを開発および生産しました-Apple IIのソビエトアナログ、FM合成モジュールを設計Cakewalk（米国）、PGMusic（カナダ）、PowerFX（スウェーデン）と協力している工場の1つ-より頻繁にスキームの下で：「アイデアと実装は私たちからであり、予算は彼らからのものです」、国際的な音楽展MusikMesse（ドイツ）、NAMMに参加（アメリカ）などもちろん、これは私がやったことのほんの一部です。もちろん、それだけではなく、特定の瞬間に集まったチームと一緒に。

さらに、ノボシビルスク音楽院で情報学科（ソビエト連邦初）の組織にも積極的に参加しました。 1983年から2004年まで、彼はまた、音楽音響学、情報学、音響工学を教え、音楽演奏のコンピュータモデリングに関する論文（ISBN 5-9294-0023-7）を準備しました...

はい、それはすべてb / wモニター、フロッピーディスク、256 kB RAM、音楽アセンブリのパーソナルPCから始まりました...ミュージシャンはすべてを指で説明する必要がありました。そしてトロンボーン奏者がテキストのページを埋めることができたなら、彼はクールで、チジク子鹿をプログラムしたチェロ奏者はただの星です。ちなみに、一部の教師はこのトピック全体に積極的に抵抗しましたが、今ではコンピュータを奪おうとしています！

この経験は私のテーマの開発、トレーニング、深化に非常に役立ちました。体育のようなコンピュータサイエンスはすべての学部で必須であり、生徒の絶え間ない流れによって、多くのテスト、心理音響学の実験、パターン認識、理論的なミュージシャンとのコミュニケーション、そして実際には正確なデータや式がないことを理解することができました。少なくとも、音楽機器やプログラムの開発者が必要とするもの。音楽学者は、残念ながら本質的に過去を説明する歴史家です。もし彼らが科学を持っているなら、おそらく、彼らの相談に加わることは、百万人以上の人口を持つ都市の市長よりも、追加であろう。有名なミュージシャンが「ヒットフォーミュラ」と「売上予測」にいくら払うのでしょうか？..

では、そのようなアルゴリズムで機能する答えを得る必要があるとしましょう。たとえば、次のようになります。

a）メロディーが鳴った（まあ、それがすでにMIDIの「音符」で録音されている場合でも）、トニックがどこにあるかを判断する方法は？ -ミュージシャンはそれを簡単に行いますが、明確な答え（アルゴリズム）はあなたに与えません。

b）メロディーがあります。それに合ったコードをどうやって選ぶか？..はい、合唱指揮者はおそらくボーカリストよりも上手に演奏しますが、両方とも、たとえ指示を与えたとしても、あまり明確ではありません。もちろん、彼らの観点からすれば、それは厳格です。エンジニアがそれをアルゴリズムに変換し始めたときだけ、未定義を克服するのが難しいいくつかの問題があります...

c）または、ドラマーは適度に複雑な何かを演奏しますか、それともサックス奏者は即興ですか-サイズ、強い/弱いビート、音楽フレーズの始まり/終わり、表現力豊かな減速/加速を「テイクオフ」する方法？ EDM（エレクトロニックダンスミュージック）についてではありません。ミュージシャン-演奏者は、教師が「私と同じように」行う方法で伝えられる「ルール」を感じ、それを遵守しますが、エンジニアは数字、グラフィックス、プロポーションを必要とします。これはすべて、演奏者または音楽学者-理論家から入手することは不可能です。

しかし、私が答えを見つけたかったのは、そのような質問でした。実際、実際に響く作品には、大まかなモデル、つまりMIDI形式の楽曲、つまり本質的に「電子楽譜」がある場合があります。これらは正確なデータですが、どのように扱うのですか？一般的に、結局のところ私にとって最も興味深いのは、MIDIデータとそのようなアルゴリズムの分析と変換*です。

パフォーマンスモデリング 1）、
モーフィング 2）、
音楽の生成 3）

そして、これらすべてのために、調性、自動調和、自動フレージングの定義は、別個のサブタスクにすぎません...これらのトピックすべてに対して、許容できる回答が得られ、アルゴリズムとプログラムが作成されました。それはCakewalk / Sonarにも入りました。

*小さな復号化：

1）量子化されたMIDIデータの分析、「音楽オブジェクト」（フレーズ、数字）の認識、Velocity、NoteOn、Durationの一部の変更、およびテンポ、ピッチピッチ、エクスプレッション、モジュレーションカーブのインポジションにより、シンセサイザーリソースをより完全に使用表現力を高めます。

2）1つの調和から他の調和へ、あるサイズから別のサイズ（たとえば、4/4から6/8または7/4）に変換することを目的とした、作業全体と個々のパーツの両方の分析。さらに、結果が「食用」になるように-ミュージシャンの意見では、明らかな違反なしに音楽が「正しい」ようになります。

3）「電子スコア」の生成-1）と2）を使用してMIDIデータを再生すると、ミュージシャン、アレンジャーがコンピュータまたはライブ楽器を使用して生成したものと同様のサウンドを出力するシステム。それら。 AIによって生成された抽象的なものではなく、専門家の狭いサークルでのデモンストレーションだけでなく、完全に「人間」であり、使用に適しています。

この長い話では、かなり面白いことがいくつかありました。たとえば、現在の瞬間を見ると、たとえば最初の386 PCの場合、2つの新しいMuscovites“ Chignon”（IL 2715）をそれぞれ290千ルーブルずつ与えなければなりませんでした。または、温室として、私は科学アカデミーのシベリア支部で使用済みのミンスク32をほとんど手に入れました-このモンスターはスマートフォンよりも弱く、おそらくバー+サウナのように60〜80 m 2を占め、電力を消費しているようです...

そこで、従業員がプログラムを作成し（GDRの最初のWin PCで、TMSとAnalog Devicesの信号プロセッサを使用してZ80、MOS 6502、「Electronics-60」で作業しました...）、毎日画面に「krakozyabra」を見て考えました彼ら（プログラマー）全員が「この世のものではない」ということです。楽譜は簡単ではありませんが！そして結局のところ、何人かのミュージシャンは音楽を聴き、音楽を見て、それが好きなら美的喜びさえ得るのです！一般的に、コードを書く人は誰でも天才であり、これは私のものではないように思えましたが、将来が示されるように、私は間違っていました。多くの時間が失われたとは言えませんが、おそらく多くのプロジェクトの運命は完全に異なっていたでしょう...

最初の実務経験


以前の経験から、プログラマにとって最も「難しい」「不愉快な」ことは、GUIとインターフェース（つまり、ある環境から別の環境への切り替えです。たとえば、外部デバイスからのMIDIデータの受信）であることがわかりました。適切な既製のソリューションはありません。

最初に私はAutoPlay Media Studio（Indigo Rose Software）を取った-それはダミーのデザイナーのビットです。最初は、CD、DVDメニュー、autorun.exeをすばやく作成するように設計されていたようです。しかし、それは機能の海があり、（LUA上で）あなたはほとんどすべてを行うことができることがわかりました：アニメーション、oggの再生。したがって、「不快な」作業のほとんど（ちなみに、新参者はそのような瞬間に止まるだけだと思います）は実行できませんでした-すべてが準備できています。

その結果、自宅で2か月後に何時間も自宅で開発し、正式に配信したトレーニング音楽プログラムのアナログを書きました。さらに、別のGUIにより、さらに興味深い機能が追加されました。オーディオとグラフィックのコンテンツを作成し、テストすることは難しくありません。彼はそれをオフィスで示し、プレゼンターの一人にそのようなバージョンを作るのにどれだけの時間が必要か尋ねたところ、答えは6か月でした！ここで、どうやら、私は初めて考えました：多分私はそれを自分で楽しむだけでなく、何かを勉強して真剣なことをしようとするべきでした...そして私は仕事を正しく整理しましたか？実際、彼の答えは正直でした-単に、「彼ら」はコンストラクタを使用せず、すべて自分で書きます。

2009年以降、私は時間の一部をレコーディングスタジオでの作業とビデオ広告の作成に費やしました。ある友人が、1分で100本のクリップを（高価ではなく）すばやく作成できるかどうかを尋ねました（どうやらYouTubeでの宣伝用）。そのような仕事がひどい！（ちなみに、プログラマーがあまり創造的でない何かを3回行わなければならない場合、彼がスクリプトを書くことを私は長い間知っていました）。私はプログラマーに尋ねました、そしてそのようなプログラムがある場合、それ自体が画面上のボタンを「押す」ことができますか？わあ、これはAutoItです。

一般的に、おもしろくするために、次のようなスクリプトを作成しました。

  1. Operaを立ち上げ、テキスト読み上げを専門とする会社（オランダ語のようです）のサイトを開設しました。
  2. このページでRU（ロシア語で演技する声）までスクロールしました。
  3. * .txtファイルをメモ帳で開きました。ここでは、20〜30のジョークがネットワークの段落から事前にコピーされました。
  4. 1つのジョークをコピーして、サイトのフィールドに貼り付けました。
  5. レコードでSoundForgeを起動し、サイトで再生をオンにしました。
  6. 結果を記録してmp3のフォルダーに「シリアル番号+ジョーク自体のテキストの最初の15文字」という名前のフォルダーに保存しました。

それはうまくいき、私は子供の頃幸せでした。その後、私は友達に言った-100の安価なクリップ（良質）を作るのに問題はなく、写真、音楽、見出し、字幕、説明などをピックアップするだけ-すべてを（サイズ、色で）順番に並べ、フォルダーに入れ、テンプレートを作成します+バリエーション（ランダム！）を決定します... 100でなく200のクリップを作成できます（Vegas ProではAutoItを使用すると簡単です）。それらを選択して見ると、適切な量を自分で選択できます...すべてを行う必要はありませんでしたが、彼女の質問をありがとう！

その後、私は徐々にサイト上の何かを修正し、何かを読み、スポーツの興味のために何かを試し始めました。特に、ビデオの作成に必要な簡単なアニメーションのためにJavaScript / CSSでスクリプトを実行すること。そのため、PHP、HTML、jQuery、JavaScript、CSS、MySQLにいくつかの一般的なアイデアと少しの経験が現れました。「クラックオーバー」はほとんどなくなり、私はこの「プログラミング」というこのトピックを恐れることをやめました。彼女と彼女の能力に関する最後のターニングポイントは、私が2つのポイントを理解したときに発生しました。

  • この領域全体は無限の世界であり、強力なプログラマでさえ、それが彼の領域でない場合、常に質問に答えるとは限りません。他の分野と同じように。一般的には-はい、まあ、ニュアンス-狭い専門家を探してください。幸いにもネットワークがあります。
  • 私は開発中に私が持っていた質問の99％を自分で解決し、インターネットで見られる例よりも効率的に解決した。同時に、私は記述のスタイルによって、私のコードがひどいものになる可能性があることを理解しています。

音楽ジェネレーター


私がどのようにWeb Audio APIに遭遇したのか覚えていませんが、この時点ではLLC、IP、またはチームがありませんでしたが、2、3回の実験の後、「音が聞こえた」とき、私は最も真剣に振ったプロジェクトは音楽ジェネレーター（現在はAlexArと呼んでいます）です。実際、Web Audioには必要なものがすべてありました。ジェネレーター、フィルター、エンベロープ、ミキサー、プロセッサー（C ++）、jQuery、JavaScript、CSSにより、GUIのすべての問題を解決できました。実際、Webオーディオでは、複雑なサウンドの処理と合成の両方を行うことができます。

ここで本当の仕事は、25歳以上の若者のように、ほとんど始まりました。 3〜4回やり直して、システムの1つのバージョンを破棄し、新しいバージョンを収集しました。特定のモジュールを設計するアプローチを根本的に変更することがあります。同時に、私は多くの実験を行い、以前に使用したことがない新しいアルゴリズムをテストしました。彼はシステムを起動しなければ起こらなかったいくつかの発見を自分で行いました。特に、心理音響学では、音楽トラックの「類似性」の認識において、どのパラメータが「異なる」音楽を与えるか、そしてそれらの形式上の違いにより、耳によって「同じ」です。ハーモニーの使用、撥弾時の模倣、コード変更時のフレージング、および「スタイルの正確さ」の理解において、いくつかの非常に有用なものが見つかりました。このすべてがオープンで説明されている可能性があります。しかし、人生ではそれがしばしば起こります自分のことを考えるのは簡単（そして便利）です。

はい、Web Audio APIがバインド、合成にVSTiを使用すること、またはその他の準備ができているとすばらしいでしょう。一方で、構造とアルゴリズムとコンテンツの両方で、若者を想起し、発明、最適化するために必要なすべてのシンセサイザーを作成する機会がありました。ほとんど、昔のように、アーティストが絵を描くだけでなく自分で絵を描いたとき...

その結果、いくつかのリベートと変更、近代化（これは優れた学校と頭脳のトレーニングでした）の後、システムは安定して完全に機能するようになりました。ええ、もちろん、それが第三者に転送できるような形ではありません。ほぼ作成者に「従う」だけの自家製車のようであり、壊れた場合、誰も修理することはできません...



通常、完成したコンポジションは4〜5分で、40〜50秒で作成されます。 （Intel Core i5、2.8GHz、12GB RAM）。 「100ダンストラックを作る」などの質問をしたり、昼食に切り替えたり、散歩したりしました。彼は1時間以内に到着し、システムがクラッシュしなかった場合は、結果を確認して、その妥当性の%%を判断しました。次に、パラメーターを変更して、最初からやり直しました。私が「合理的に」行動した場合、つまりスタイルのぼやけにつながらないパラメータを設定します（たとえば、これはEDMでジャズハーモニーが適用された場合に発生します）。その後、90〜95％のトラックがかなりまともなサウンドになり、そうでない場合は60〜70 %%が破棄される可能性があります。

システムは機能しました、あなたは喜ぶことができます、しかしそれはとんでもないけれども本当の問題が始まったようです：「どうしたらそんなに多くの音楽を聴くことができるのですか？そして、もう1つの瞬間の実現：「そして、作者は誰ですか？」 「この件について」の良い友人は、「作者はコンピュータを含む。それはあなたのものではありません」...もしそうなら、それは残念です、私は曲、フレーズ、ハーモニー、フォームを入力し、数百のパラメーターを設定しました。この場合、5ミリ秒が結果に反映され、システムは結果のみを生成しましたが、アルゴリズムに従って私はそれに投資しました...私はネットワークを検索しました-著作権のある同様のケースではそれを理解しませんでした...

その結果、総再生時間が200時間を超える約3,500曲をリリースし、停止または切り替えました。AlexArで作成したアプローチに基づいたオンラインアレンジメントジェネレーターであるSongModelerを起動する予定です。さて、AlexArでの自動音楽生成の例はここにあります。

あまりにうるさいリスナーの場合、私はプログラミング、ハーモニー、メロディー、フォーム（つまりノート）に多くの時間を費やしたと言いたいのですが、もちろん合成とミキシングははるかに優れています。最後に、私はアイデア、アプローチ自体をチェックしました-それは本当です。そして、そのようなプロジェクトにはチームが必要であり、多くの人が学際的なトピックに取り組むことはめったにありません。一方、「あなたの上にもあなたの下にも」誰もいない場合、仕事ははるかに快適になります。しかし、本当の皮肉は、私がチームとリソースを持っているとき、それがあまりにも複雑であると思ったので、私はそのようなものを目指しなかったということです。

AlexArの作業の途中のどこかで、私が以前にプログラミングを開始した場合、10人のプログラマー（時にはスタッフにはるかに多くのプログラマーがいた）が3人に置き換えられる可能性が非常に高いことに気付きました。
