敵AI：Navigation2Dを使用せずにプレイヤーを追跡し、パスAを見つける*

敵がプレイヤーを追いかけなければならないゲームを作成しますか？すべては単純なものから始まります-敵をプレイヤーに走らせます。しかし、それが木の後ろや壁の角にある場合はどうなりますか？さて、今度は敵はかなり愚かに見えます-彼は適所に指でオブジェクトに遭遇します。あまりよくない！

この問題を解決するには、 Godotに組み込まれているNavigation2DまたはAStarノードを使用できます（両方のノードでのGDQuestチュートリアルです）。しかし、Helms of Furyでは、ゲームに最適な別の方法を使用しました。このチュートリアルでそれを共有したいと思います。これは次のようになります。


働き始める


敵をKinematicBody2Dオブジェクトとして作成し、ステートマシンを使用してそれらの状態を制御すると想定します。ステートマシンが何かわからない？ステートマシンとその使用方法を説明するこの記事は気に入っています。Godotでの単純な状態マシンの実装に関する別の記事を次に示します。

ターゲットに向かって走り、途中のどこかで動けなくなってしまう愚かな敵の単純な追跡状態から始めましょう。

# ChaseState.gd

func _init(enemy, params):
  enemy.dir = (enemy.target.position - enemy.position).normalized()

func _physics_process(delta):
  var motion = enemy.dir * enemy.speed
  enemy.move_and_slide(motion)

においの痕跡


状態を改善するために、移動するときにプレーヤーに以前の位置からのトレースを残すように強制します。このおかげで、敵がプレイヤーを見つけられない場合、彼は自分の過去のポジションが見えるかどうかを確認し、プレイヤーを追跡します。これは犬が足跡をとる方法に似ているため、臭い足跡と呼びます。

においのトレースを機能させるには、Timerノードをプレーヤーに追加し、それを自動的に開始してwait_timeを設定できるようにし（0.1sを使用）、次にカウントダウンが終了したときにプレーヤーがにおいを残すようにコードを追加する必要があります。

# Player.gd
extends KinematicBody2D

const scent_scene = preload("res://Player/Scent.tscn")

var scent_trail = []

func _ready():
  $ScentTimer.connect("timeout", self, "add_scent")

func add_scent():
  var scent      = scent_scene.instance()
  scent.player   = player
  scent.position = player.position

  Game.level.effects.add_child(scent)
  scent_trail.push_front(scent)

次に、破棄されたScent.tscn自体を追加する必要があります。これは、匂いが期限切れになるようにタイマーを含む単純なNode2Dシーンです。

# Player.gd
extends KinematicBody2D

const scent_scene = preload("res://Player/Scent.tscn")

var scent_trail = []

func _ready():
  $ScentTimer.connect("timeout", self, "add_scent")

func add_scent():
  var scent      = scent_scene.instance()
  scent.player   = player
  scent.position = player.position

  Game.level.effects.add_child(scent)
  scent_trail.push_front(scent)

デバッグ中に匂いを表示したい場合は、匂いにColorRectノードを追加して非表示にできます。これを行うと、プレーヤーの後に走ると臭いの痕跡があることがわかります。

次に、内部のブラッドハウンドの敵を目覚めさせて、プレイヤーが見えないときにこれらの新しい匂いを追跡するようにします。ただし、そのためには、RayCast2Dノードを敵に追加し、GodotでPhysics Layersを設定して、ビームが衝突できることを認識できるようにする必要があります。

物理層


Godotで物理レイヤーを設定するには、トップメニューの[プロジェクト]、[プロジェクト設定]の順にクリックし、左下の[レイヤー名]セクションに移動して、[2D物理]を選択する必要があります。


それらに適切な名前を付けます。その後、ゲーム内のオブジェクトに移動し、プロパティインスペクターのサイドバーで[衝突]を展開し、[...]をクリックして割り当てます。オブジェクトの場合、レイヤーとして割り当てる必要があります。


物理レイヤーをオブジェクトに割り当てた後、敵のレイキャスト2Dを変更して、敵が移動できないレイヤーをチェックする必要があります（この場合、これは、ソリッド、オブジェクト、クレート、穴、gate_closedです）。

物理層を設定したら、最後のステップはチェイスの状態を変更することです。

# ChaseState.gd

func _init(enemy, params):
  chase_target()

func chase_target():
  var look     = enemy.get_node("RayCast2D")
  look.cast_to = (enemy.target.position - enemy.position)
  look.force_raycast_update()

  # if we can see the target, chase it
  if !look.is_colliding():
    enemy.dir = look.cast_to.normalized()

  # or chase first scent we can see
  else:
    for scent in enemy.target.scent_trail:
      look.cast_to = (scent.position - enemy.position)
      look.force_raycast_update()

      if !look.is_colliding():
        enemy.dir = look.cast_to.normalized()
        break

func _physics_process(delta):
  var motion = enemy.dir * enemy.speed
  enemy.move_and_slide(motion)

さて、敵がチェイス状態に入ると、彼はプレイヤーを再キャストしようとし始め、邪魔にならない場合、彼は彼に従います！途中で何かがある場合、彼は臭いの痕跡に行き、それらの1つが見つかるまでそれらのそれぞれを再キャストしようとします-そして彼を追跡します！


そして今、すべてが機能し、敵はブラッドハウンドに変わりました。システムを改善するために、敵同士の衝突を回避するシステムを実装することは可能ですが、これは別の記事のトピックです。
