未来はここにあります：音声ロボットがどのように機能し、何ができるか



日常業務のロボット化は、人間ではなく単純かつ同時に労働集約的なタスクを解決するためにロボットが使用される場合、非常に活発な傾向です。お客様との電話での会話など、多くのことが自動化されています。Neuro.net社は、ロボットの能力を向上させる機会を提供するテクノロジーの作成に取り組んでいます。

この記事では、開発者が対話者の性別を音声で認識し、対話の重要な要素に取り組む技術とニュアンスについて話します。

最初のケース、次にテクノロジーの内訳




最も興味深い事例の1つは、パートナー企業のコールセンターの従業員を音声ロボットに置き換えた場合です。後者の機能は、配送先住所の明確化などの通常の状況ではなく、一部の顧客が会社のWebサイトにアクセスする可能性が低くなっている理由を見つけるために使用されました。

このテクノロジーは、個々のスクリプトではなく、本格的なニューラルネットワークの使用に基づいていました。通常はロボットを混乱させる問題を解決することができたのはニューラルネットワークでした。まず、「まあ、私はまだわからない、多分そうですが、そうではない」または「はい」のような対談者の答えについて話している。人間に共通する言葉は、ロボットにとって乗り越えられない障害になります。



訓練中に、ロボットは特定のフレーズにどのような意味があるか、そして答えはどうあるべきかを理解し始めました。ロボットは複数の票を獲得しました-男性と女性の両方。主なタスクは、人間の対話者がマシンの機能をテストせずに、ターゲットシナリオに従って対話を行うようにロボットを「人間化」することでした。

以下は何が起こったかの例です。


ロボットは対話者の声を聞いて、クライアントの発言の意味に応じて答えを出します。会話に使用できるスクリプトブランチの総数は1000を超えます。
このロボットの主な目的は、サイト上の会社のクライアントの活動が減少した理由を理解し、すべての人に興味深い提案をすることでした。これは、コールセンターの業務を自動化するという同社の最初の試みの1つでした。

新しいロボットはより完璧です。ロボットが人間と通信する方法のいくつかの例を次に示します。最初、2番目、3番目の例です。

テクノロジーについて


ロボットの動作を可能にする3つの主要な技術的特徴があります。

  • 音声による対話者の性別の認識、
  • 年齢認識
  • 人間の対話者との対話を構築する。



対話者の性別を声で認識する


なぜこれが必要なのですか？当初、この機能はロボットを使用して調査を行うために作成されました。これまでは、多くのポイントを記入していただいた方が調査作業を行っていました。たとえば、対話者のフロア。このパラメータを決定するために、人が彼または彼女のどちらと話すかを尋ねる必要がないことは明らかです。 99％では、すべてが明確です。ロボットも別の問題ですが、音声認識をある程度正確に学ぶために、大がかりな作業をしなければなりませんでした。そして、それは無駄ではありませんでした、現在、テクノロジーは性別に応じてオファーと音声プロンプトを個人化するために使用されています。

重要なポイント-女性の声は普遍的であり、幅広い製品を扱うのに適しています。女性向けの製品では特に重要です。さまざまな研究によると、女性の声はすべての聴衆によってそれぞれ肯定的に知覚されます。この場合、変換はより大きくなります。例外-「男性」の商品を宣伝する場合は、男性の声をお勧めします。

使い方？最初に、プライマリデータ処理が実行されます。これは、20ミリ秒続く音声録音とフラグメントの処理に基づいています。収集されたすべての音声フラグメントは、VAD（Voice Activity Detection）コンポーネントで前処理されます。これは、「穀類と籾殻」、つまり音声とノイズを分離するために必要です。すべてのゴミが削除され、モデルの精度が向上します。

認識には、ケプストラム係数のいわゆる空間、1番目と2番目の違いが使用されます。基本はGMM法-ガウス混合モデルです。

したがって、10〜20 msの間隔で現在のパワースペクトルが計算され、その後、必要な係数を検索して、スペクトルの対数の逆フーリエ変換が適用されます。

私たちのGMMモデルは、男性と女性の音声MODを教えるために個別に構成されており、大人と子供の声を決定するためにも使用されます。もちろん、システムを最初からトレーニングすることはできません。マークアップされた音声録音が必要です。

システムの効率を上げるために、音色モデルの係数が適用されます。

  • ティンブラーの鋭さ。
  • ティンブラーの暖かさ。
  • ティンバールの明るさ。
  • 震えの深さ。
  • ティンバール硬度。
  • 鼓膜の成長。
  • 震えむら。
  • ティンバーリバーブ。

子どもの声を正しく識別するには、ティンバーモデルが必要です。他のモデルでは、子どもの声を女性として受け入れます。さらに、粗い女性の声（たとえば、高齢の喫煙女性）、高い男性の声などを区別する必要があります。ちなみに、人が「こんにちは」と言ってから咳をした場合、音色フィルターを使用しなかった以前のすべてのモデルは、声を男性と定義していました。



システムの主要部分は、多層パーセプトロンMLPに基づくデータ分類モジュールです。男性と女性の声のモデルからのデータ、音色モデルからのデータを送信します。システムの入口で、分類された値の配列を取得し、出力で性別判定の結果を取得します。

ここで説明するテクノロジーは、オンライン（クライアントの最初のフレーズによる）とオフラインの分類モード（会話後）の両方で機能します。性別認識精度は約95％です。重要な点は、オンラインで作業するときの遅延が120〜150ミリ秒を超えないことです。これは、ロボットの人間化にとって非常に重要です。通常、ロボットと人との間の通信の一時停止はミリ秒単位ではなく秒単位ですが、これはもちろん人間の対話者にとっては奇妙に見え、デジタルシステムが通信していることはすぐにわかります。

計画には、テキストを使用した作業、より正確にはエンディングが含まれています。対談者が「できる」と言った場合、間違いなく、これは女性です。近い将来、この技術は完成し、認識システムに実装される予定です。

対談者の年齢の決定


なぜこれが必要なのですか？まず、未成年者に様々な商品やサービスを提供しないため。さらに、年齢を特定することは、年齢カテゴリ別にオファーをパーソナライズするために役立ちます。

使い方？前のケースとまったく同じテクノロジーが使用されます。システムの精度は約90％です。



ダイアログの作成


そして今、私たちは最も興味深い-ダイアログを構築する原則に進みます。

なぜこれが必要なのですか？人を適切に置き換えるために、ロボットは対話の線形シナリオと非線形シナリオの両方で作業できなければなりません。前者の場合はアンケート、後者の場合はコールセンターの加入者、会社のテクニカルサポートラインなどと連携します。

どのように機能しますか？ NLUエンジンを使用します。その基本は、ASRシステムから受信したテキストのセマンティック分析です。また、エンティティ（インテント）やインテント（インテンション）などの認識フローは、会話の流れを構築するロジックで使用されるものと区別されます。

これは、テクノロジーの仕組みの例です。

音声認識システム（ASR）から受け取ったテキスト：
「一般的に、私はあなたの提案に興味がありますが、もっと安くしてほしいです。そして、今は少し忙しいので、明日6時にかけ直してもらえませんか。」

NLUエンジンで満たされたオブジェクト： インテント

：
確認= true
異議=高額な
質問= null
コールバック= true
wrong_time = true

エンティティ：
日付= 01/02/2019（呼び出し日が01/01/2019であると仮定）
時間= 18：00
量= 6

充填原理この例のオブジェクトは次のとおりです。イン

テント（意図）：

  • 「あなたの提案に興味があります」というテキストは、「true」の値を持つ意図「確認」に翻訳されています。
  • 「もっと安くしてほしい」という文言は、「高価」という価値観を持つ「異議」の意図に翻訳されました。
  • 「今はちょっと忙しい」というテキストは、値が「true」のインテント「wrong_time」に翻訳されました。

インテントとエンティティのリスト全体に対して、特定の値が割り当てられ、会話フローを構築するために使用されます。

次に、NLUエンジンシステムでサポートされている作業アルゴリズムについて説明します。2つのレベルがあります。

最初のレベル-約600〜1000レコードの比較的小さなデータサンプルで機能します。ここではMLアルゴリズムが使用されます。認識精度：90-95％。

第2レベル-プロジェクトへの移行と、100万件を超えるレコードを含む大量のデータサンプルの蓄積後に、その移行が実行されます。DLアルゴリズムはすでにここで使用されています。認識精度：95-98％。

ソリューションは2つのサブシステムで機能します。

  • テキストデータの分類と分類のサブシステム、
  • 対話形成サブシステム。

両方のサブシステムは並行して動作します。分類および分類システムの入り口で、音声フレーズから認識されたサブスクライバーのテキストが送信されます。出力では、決定により、入力されたパラメーターEntityとValueが返され、回答が形成されます。

非線形シナリオを構築するための対話形成サブシステムは、ニューラルネットワーク上に構築されています。システムの入り口で、音声フレーズから認識された加入者のテキストが送信され、出口で、次の瞬間に何を再生するかが決定されます。

非線形のシナリオは、最初のサポートラインに適しています。ロボットは、誰が、どの特定の製品について、どのような質問をしているかを知りません。ここで、対話のさらなる構成は、クライアントの応答に依存します。

ただし、発信コールの場合、最適なソリューションは線形シナリオです。彼の例は記事の冒頭にありました。線形シナリオのもう1つの変形は、クライアントの回答が問題ではないときに調査を実施することです。これは、専門家によってさらに分析されます。しかし、リストにあるすべての質問をクライアントに案内することが重要です。

結果として、音声ロボットは人に取って代わるものではないことを強調したいと思います。現在、彼らは日常の仕事で優れた仕事をしています-いくつかの質問をし、答えを聞いて/記録して/分析するために人々に電話をしています。したがって、コールセンターとテクニカルサポートのオペレーターは、同じルーチン手順を実行する必要がなくなります。代わりに、彼らは本当に興味深い質問や課題に集中できます。
