グラフォニア持ってきた！ニューラルネットワークが古いゲームの解像度をHDに向上させる方法





この記事は、Mediumに関する私の記事の無料翻訳です。



私は子供の頃、コンピューターで遊ぶのが大好きでした。カセットZSスペクトラムで小さなゲームを見つけたが、90年代のカラフルな発見は本当の発見となった。その後、既存のジャンルの大半が発生しました。少し懐かしさを感じた後、私は自分の若さを思い出してDosboxエミュレーターで古いゲームの1つを実行することを決定し、巨大なピクセルと低解像度に不快に襲われました。大きなピクセルの古いグラフィックには独自の魅力があるかもしれませんが、今では多くの人がこの品質に満足していません。



ゲームの解像度を上げて角度をなくすために、現在さまざまな後処理アルゴリズムとアンチエイリアスアルゴリズムが使用されています（詳細については、こちらを参照してください）zen.yandex.ru/media/id/5c993c6021b68f00b3fe919c/kak-rabotaet-sglaj​​ivanie-v-kompiuternyh-igrah-5c9b3e76d82a083cc9a0f1a7）ですが、アンチエイリアシングアルゴリズムはすべてのピクセルを嫌いで、より頻繁に嫌悪します。



同時に、グラフィックスの品質を向上させることは、ゲーマーにとってしばしば重要です。Heroes 3 HDゲームのテクスチャの再描画により、Ubisoftは2014年に約6か月かかり、このゲームへの関心が高まりました。最近のニュースから-hdグラフィックスの最初のCNCの再リリース。詳しくは、グーグルトレンドでHDグラフィックスの再発行への関心の高まりをご覧いただけます。



すべてのゲームで煩わしく、高品質で再発行するのが理にかなっているわけではありません。テクスチャの再描画は、コストのかかる作業です。しかし、超解像技術を使用してグラフィックの品質を向上させることができます。超解像のアイデアは、利用可能なデータに基づいてニューラルネットワークで欠落しているピクセルを描画することにより、画像の解像度を向上させることです。映画のブレードランナーからの画像強化のミームで探索されたシーンとの関連を想起させる印象的な結果が達成されました





超解像技術は、画像の視覚的な認識を向上させます（例：github.com/tg-bomze/Face-Depixelizer）が、画像に新しい情報をもたらします。また、フィルムの品質を向上させるために使用できます。





ただし、ほとんどのアルゴリズムはリソースを大量に消費するため、ゲームをリアルタイムで改善するスクリプトを作成したいと考えました。



理論のビット



以下のすべてが畳み込みニューラルネットワークに適用されます。これは、画像の処理に使用されるニューラルネットワークのサブタイプです。まず、超解像問題を解くためにニューラルネットワークがどのように機能するかを見てみましょう。タスクは、自動エンコーダの問題の解決策と非常に似ています。出力で同じ画像を取得するには、ネットワーク入力に画像を送信する必要があります。ただし、自動エンコーダは通常、効率的なデータ圧縮の問題を解決するために使用されるため、そのアーキテクチャの機能はボトルネック、つまりボトルネック、つまりニューロンの数が少ないネットワーク層です。そのような層の存在により、残りの部分は情報の効率的なコーディングとデコーディングを学習します。超解像ネットワークをトレーニングするために、高品質画像の解像度を最初に意図的に減らし、ニューラルネットワークの入力に送ります。出力は、高品質で期待されるソース画像です。



超解像タスクは、使用されるネットワークのアーキテクチャを定義します。

通常、入力データと出力データの間には接続（スキップ接続）があり、学習を大幅に加速します。入力データのピクセルサイズが増加し、たたみ込みネットワークの出力に追加されます。したがって、実際には、イメージをほぼ同じにするためにネットワークをトレーニングする必要はありません。ピクセルのサイズを大きくして拡大した画像と実際の画像の違いを描くように彼女を訓練する必要があるだけです。スキップ接続をさまざまなレベルで、異なる数のレイヤーに通すというアイデアは非常に効果的であり、ネットワークの残余ネットワーククラスの出現につながっています。現在、このような接続は、ほとんどすべての一般的なアーキテクチャで使用されています。超解像問題を解決するための最新のアーキテクチャーの優れた概要は、ここにあります。。私の仕事は、超解像問題をリアルタイムで解決するニューラルネットワークを作成することでした。



最初に、4ブロック（通常は16層を超えるブロックが使用されます）の解像度を持つ4倍の解像度のResNetアーキテクチャを選択しました。ここでは、github.com / krasserm / super-resolutionの開発と同じプロジェクトのデータジェネレーターを部分的に使用しました。



一般的なネットワークアーキテクチャを図に示します。各ブロックはX * Y * N画像であり、幅はチャネル数に対応します。遷移-3x3のたたみ込みに対応します（resブロックの後に非線形性のためのReLUのアクティブ化が続く場合）。アップスケーリングステップ-チャネルの平坦化による寸​​法の増加。







このような小さなネットワークを注意深く見ると、最後のブロックが線形であり、いくつかのアップスケーリングステップの存在が正当化されていないことがわかります。ネットワークの品質を損なうことなく、品質を損なうことなく、より少ないフィルターで1つのレイヤーに置き換えることができます。







そのようなネットワークは、超解像タスクにほぼ同じ方法で対応しますが、起動が速く、小さなビデオ解像度をリアルタイムで処理できることがわかりました。



高品質の写真がdata.vision.ee.ethz.ch/cvl/DIV2Kからダウンロードされました。原則として、どの写真でもトレーニングに使用できます。 



import tensorflow as tf
from model import resolve_single
from model. rtsr import rtsr
import numpy as np
from utils import load_image, plot_sample
from data import DIV2K
from tensorflow.keras.optimizers.schedules import PiecewiseConstantDecay
# Model creation
rtsr_model= rtsr(scale=4, num_res_blocks=4)
#Data loading and generator creation
train_loader = DIV2K(scale=4,             
                     downgrade='bicubic',
                     subset='train')      
                     


train_ds = train_loader.dataset(batch_size=16,
                                random_transform=True,
                                repeat_count=None)  
valid_loader = DIV2K(scale=4,  
                     downgrade='bicubic',
                     subset='valid')  
                     

valid_ds = valid_loader.dataset(batch_size=16,           # use batch size of 1 as DIV2K images have different size
                                random_transform=True, # use DIV2K images in original size
                                repeat_count=40)         # 1 epoch
#Saving checkpoints during training
cbcks=tf.keras.callbacks.ModelCheckpoint('callback_x4{epoch:02d}-{loss:.2f}', monitor='loss', verbose=0, save_best_only=False)

learning_rate=PiecewiseConstantDecay(boundaries=[20000], values=[1e-4, 5e-5])
#model compilation before training
rtsr_model.compile(loss='mae', optimizer=tf.keras.optimizers.Adam(learning_rate=learning_rate))
#Training
rtsr_model.fit(train_ds,validation_data=valid_ds, steps_per_epoch=3000, epochs=100, validation_steps=10,callbacks=[cbcks])
#saving weights
rtsr_model.save_weights('weights/ myedsr-lastdiv2_4block-x4_weights.h5')


出力は、単純なアーキテクチャを持つ比較的小さな（2.6 MB）ニューラルネットワークです。同時に、このチェックは事前トレーニング済みの16ブロックネットワークとは微妙な違いを与えます。







左側の画像は元の画像で、右側は16層のedsrを使用して改善され、中央はrtsrを使用しています。2と3の違いは、私の観点からは重要ではありません。

開始するには？



すばやく作業するには、cudaおよびcudnn（https://developer.nvidia.com/cuda-gpus）をサポートするビデオカードと、インストールされているcuda / cudnnライブラリが必要です。Python 3.7とtensorflowをインストールする必要があります（gpuをサポートする2.0以上のバージョン）。これは困難な作業であり、互換性の問題が発生する可能性があります。



最も簡単な方法は、Anacondaディストリビューション（https://www.anaconda.com/products/individual）をインストールしてから、condaを使用してtensorflow-gpuをインストールすることです。 



conda install tensorflow-gpu


競合が原因でうまくいかない場合は、 



conda install cudnn
pip install tensorflow-gpu


うまくいくはずです。



他のライブラリは、pipを使用してインストールできます。 



pip install opencv-python
pip install pywin32
pip install mss


次に、次のコマンドでスクリプトを実行する必要があります。 



python superres.py


キャプチャウィンドウは、wsadおよびijklキー（サイズ変更）を使用して制御されます。q-ウィンドウを閉じる。0-超解像モードをオン/オフにします。番号1と2-画像モード（デフォルトでは2は、ピクセルの2倍の増加によるゲームの増加を使用することを意味します）。
