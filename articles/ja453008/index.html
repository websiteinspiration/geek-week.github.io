信じられないかもしれませんが、Emscriptenの作成者はすべてを考えてきました。1つの大きな機能-そして、この記事で最後に説明する機能です。それらが多すぎてすべてを1つにまとめることができないためです-これは、迅速な開発とテストの基礎です。Wasmをコンパイルしてプロジェクトを構築し、サーバーを一気に起動できます。


emrun --port 7777 --no_browser public/index.html

Now listening at http://localhost:7777/

実行しているhtmlページは、emrun対応ではありません。stdout、stderr、exit（戻りコード）のキャプチャは機能しません。--emrunリンカーフラグを使用してアプリケーションを再コンパイルしてこれを有効にするか、-no_emrun_detectをemrunに渡してこのチェックを非表示にします。

さて、2つのフェローで...上の例はうまくいきますが、次の例はうまくいきます。


<b>// compile as emrun project
emcc lib/strings.c -s WASM=1 --emrun -o public/index.html

// run the emrun server again
emrun --port 7777 --no_browser public/index.html</b>

そのままにしておきます。WebAssemblyを勉強して、聖パトリックの日と翌日を祝いました。次に何が起こるかはよくわかりませんが、2日間モニターの前に座って、WebAsseblyの基本を理解しようとすることに興奮しました。これは2011年以来私が書いた中で最もCのコードであり、すばらしい感じです。WebAssemblyには本当の未来があると思いますが、一部の説教のように、WebAssemblyが完全にネットワークを引き継ぎ、JSを殺すかどうかはわかりません。そして、あなたはどう思いますか？
