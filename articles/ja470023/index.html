なぜ支払う必要があるのですか？


すべての電報ボット開発者の生活の中で、ボットに支払いを実装する必要があるときが来ます。そして、ここには2つの方法があります。1つは、TIN、IP /法人を登録し、電信からの公式支払いを接続するか、Free-Kassa、QIWIなどのサードパーティサービスを使用することです。選択は明白だと思います。この記事では、QIWIを使用します。

最初に、ボットがどのように、何から、何から来たのかをチェックする方法について考えてみましょう。証拠のキャプテンは、送金の全履歴をアカウントに取得する必要があることを教えてくれます。これを、支払いを受け入れるためのアカウントとして使用します。やってみましょう：

import requests
import json

QIWI_TOKEN = ''
QIWI_ACCOUNT = ''

s = requests.Session()
s.headers['authorization'] = 'Bearer ' + QIWI_TOKEN  
parameters = {'rows': '50'}
h = s.get('https://edge.qiwi.com/payment-history/v1/persons/'+ QIWI_ACCOUNT +'/payments', params = parameters)
req = json.loads(h.text)

QIWI_TOKEN 定数では、QIWIを操作するためのAPIキーを記述します。ここで取得できます。そして、中にQIWI_ACCOUNT、私たちはあなたのメインのアカウントの電話番号を記録します。すべてを正しく行った場合、エラーは発生しません。データは、データのリストを含むreq辞書にあります。より正確には（読み取らないでください）、 データディクショナリを含むreq変数。これには、リストを含む番号付きディクショナリのリストが含まれます。



ここでは、電話番号、コメント（理由は後で説明します）、および取引金額を取得します。コードを書き続けるには、何もわからない、またはそれが何であるかさえわからない場合でも、データベースを使用できる必要があります。ここに記事へのリンクがありますSqlite3での作業を非常に明確に説明しています。次に、ボットで使用されるアルゴリズムについて考えてみましょう。

  1. 100000〜999999の乱数を生成します。
  2. 一時的にデータをテーブルに入力します（ユーザーID、電話番号、以前に生成された乱数）
  3. コメント、アカウント、金額はreq辞書で確認します。
  4. 金額、電話、コメントが同意した場合は、支払いをカウントします。
