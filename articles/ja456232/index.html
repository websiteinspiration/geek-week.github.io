ガイド：C＃8.0でのデフォルトメンバーを使用したインターフェースの更新

.NET Core 3.0のC＃8.0以降、インターフェイスのメンバーを作成するときに、その実装を決定できます。最も一般的なシナリオは、すでにリリースされ、無数のクライアントによって使用されているインターフェイスにメンバーを安全に追加することです。

このガイドでは、次の方法について学習します。


  • 実装を伴うメソッドを追加することにより、インターフェースを拡張しても安全です。
  • 柔軟性を高めるためにパラメーター化された実装を作成します。
  • 手動制御の可能性があるより具体的な実装を実装する権利を取得します。



どこから始めますか？


まず、C＃8.0プレビューのコンパイラーを含め、.NET Coreで動作するようにマシンを構成する必要があります。このようなコンパイラは、Visual Studio 2019以降、または新しい  .NET Core 3.0プレビューSDKで利用でき  ます。.NET Core 3.0（プレビュー4）以降では、既定のインターフェイスメンバーを使用できます。

public static void SetLoyaltyThresholds(
    TimeSpan ago, 
    int minimumOrders = 10, 
    decimal percentageDiscount = 0.10m)
{
    length = ago;
    orderCount = minimumOrders;
    discountPercent = percentageDiscount;
}
private static TimeSpan length = new TimeSpan(365 * 2, 0,0,0); // two years
private static int orderCount = 10;
private static decimal discountPercent = 0.10m;

public decimal ComputeLoyaltyDiscount()
{
    DateTime start = DateTime.Now - length;

    if ((DateJoined < start) && (PreviousOrders.Count() > orderCount))
    {
        return discountPercent;
    }
    return 0;
}
