Windowsターミナルアップデート：プレビュー1910

こんにちは、ハブル！Windowsターミナルの次の更新プログラムがリリースされました。新製品には、動的プロファイル、カスケード設定、更新されたUI、新しい起動オプションなどがあります。詳細はカットソー下！

いつものように、ターミナルはMicrosoft Store、Microsoft Store for Business、およびGitHubからダウンロードできます。



動的プロファイル


Windowsターミナルは、PowerShellコアとLinux（WSL）ディストリビューション用のインストール済みWindowsサブシステムを自動的に検出するようになりました。つまり、この更新後に配布キットをインストールすると、それはすぐにprofiles.jsonファイルに追加されます。



注：プロファイルをドロップダウンメニューに表示したくない場合"hidden"は、true profiles.jsonファイルでパラメーターを設定できます。

"hidden": true

カスケード設定


ターミナルの設定モデルが改善されました。これからは、すべてのデフォルト設定を含むdefaults.jsonファイルが付属します。ファイルの内容に慣れるには、Altキーを押しながら、ドロップダウンメニューの[設定]ボタンを押します。開かれたファイルは自動的に生成され、ファイルに加えられた変更は無視されて上書きされます。ただし、profiles.jsonファイルには、ユーザー設定をいくつでも追加できます。設定をリセットしたい場合は、 Scott Hanselman（Scott Hanselman、@ shanselman）がブログに投稿した優れた記事に注目することをお勧めします。

新しいプロファイル、スキーム、キーバインディング、またはグローバルパラメータをprofiles.jsonに追加すると、追加されたパラメータと見なされます。既存のプロファイルと一致するGUIDで新しいプロファイルを作成すると、古いプロファイルが新しいプロファイルに置き換えられます。defaults.jsonファイルに、使用を拒否するキーバインディングが含まれている場合は、このバインディングnull をprofiles.jsonに設定します。

{
"command": null, "keys": ["ctrl+shift+w"]
}

新しい起動オプション


これで、常に全画面で実行するか、画面上の初期位置を設定するようにターミナルを構成できます。ターミナルは、グローバルパラメータを追加することにより、全画面で実行するように構成できます"launchMode"。このパラメーターは、"default"またはのいずれかの値を取ることができます"maximized"。

"launchMode": "maximized"

画面上のターミナルの初期位置を設定する場合は、グローバル"initialPosition"座標とX座標とY座標をグローバルパラメータとして追加する必要があります。たとえば、メイン画面の左上隅からターミナルを起動する場合は、ターミナルをプロファイルに追加します。 jsonレコード：

"initialPosition": "0,0"

注：複数のモニターを使用していて、ターミナルをメインモニターの左または上から開始する場合は、負の座標を使用する必要があります。

更新されたUI


ターミナルのインターフェースはさらに良くなりました。ターミナルで使用WinUI TabViewがバージョン2.2に更新されました。このバージョンでは、色のコントラストが最もよく、ドロップダウンメニューの角が丸くなっています。さらに、多数のタブを開くとすぐに、ボタンを使用してタブをスクロールできるようになりました。



修正されたバグ


  • これで、タブバーをダブルクリックしてウィンドウを全画面で展開できます。
  • 新しい行へのコピーと貼り付けで問題が発生するバグを修正しました。
  • HTMLコピーでクリップボードが開いたままになることはなくなりました。
  • これで、名前が32文字を超えるフォントを使用できます。
  • 2つのタブが同時に起動されても、テキストのゆがみは発生しなくなりました。
  • 一般的な安定性の向上。
