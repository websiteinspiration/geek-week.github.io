Node.jsのPredator-Preyモデル

最近、主にその作成者が死亡したという事実が原因で、ゲームLifeへの言及の急増がネットワークを通過しました。

今こそ、誰もが生物学に興味を持つようになりました。どこにいても、これらの生存スケジュールはどこにでもあります。私は、記憶の箱から興味深いモデルを突然取り出しました。

モデルは生命と似ていますが、火と見なすことができ、無限に瞑想し、永遠を反映することができるのと同じ循環プロセスです。

これは、応用数学からの非常に深刻なモデルであるPredator-victimです（英語圏のPredator-prey）。

このプロセスの本質は、特定の森に鹿の群れ（別のバージョンでは、エッセンスではなくウサギ）が生息し、野生に食い込んで野生に繁殖し、遅かれ早かれ領土全体を埋めることです。

ただし、同じ森にはこれらの鹿を食べる捕食者もいます（オオカミですが、野ウサギの場合は通常キツネ）。マルサス

の法則に従って、この豊富な森で自分自身を見つける捕食者のペアは指数関数的に激増しますが、ある時点で、鹿のリソースが枯渇し始め、オオカミが餓死し、絶滅し始めます。

角を曲がったシカは頭を上げ、出展者の電源を入れ、森を支配し始めますが、新鮮な肉に住んでいるオオカミは、生殖能力の新しい波のために自分自身に力を見いだしています...

これがグラフ（ウィキペディアから抜粋）です。



このプロセスの数学的モデルは、20世紀初頭にロトカとヴォルテッラによって説明され、彼らに敬意を表して名付けられました。

なぜこのモデルが100年間存在していて、それでも関連性があるのですか？

主な理由は2つあります。非常に単純で、プロセスを非常に現実的に説明しています。

モデルには4つのパラメーターしかありません。

  • アルファ）鹿の繁殖率
  • ベータ）オオカミによるシカの食べる速度
  • ガンマ）空腹のオオカミの絶滅率
  • デルタ）オオカミの繁殖率

モデルには最小限の非線形性が含まれており、分析的に考慮されます。適切に選択されたパラメータを使用すると、安定しており（鹿も狼も最後まで絶滅することはありません）、個体群の変動のダイナミクスを現実的に表現します。

100年間、より現実的なものを作成するための多くの試みがありましたが、複雑さが増すと、より高いレベルの非線形システムが生じ、すべてが数値的方法によってのみ解くことができる不可解な積分方程式に依存します。

別の方法があります-このプロセスをゲームとしてプログラムするだけです。

実際、このアプローチはマルチエージェントモデリングと呼ばれ、コースに合格するのに非常に適しています。

テクノロジーの選択


私はプログラムに、作成者のマシンだけでなく、可能な限り多くの聴衆にも視覚化してもらい、最小限の労力でそれ自体がすべてになるようにしたいと思っています。
解決策はプログラムをブラウザで実行することであり、そのため、JavaScriptで記述する必要があります。

さて、テクノロジー動物園を作らないようにするために、その上にサーバーも作成します。

node.jsをインストールするための標準的な手順と必要なすべてのものがgithubに記載されています。

オオカミ


オオカミをモデルに紹介する時が来ました。

オオカミがどのように食べて増殖するか、2つのことを決定する必要があります。

誰かがいるときに狩りをするのは簡単です-隣の檻に鹿がいる場合はそれを食べてください。

鹿がいない場合は、しばらく生存できます。

まず、すべての動きでオオカミがいる可能性があるとしましょう。ただし、2つの動きでそれができなかった場合は、進化の側へと進みます。

より多くのオプションの乗算。

まず、繊細さを取り除き、空きスペースがある場合は常にオオカミを増殖させます。

そして制限を加えてください-空腹のオオカミは繁殖しません。

最初のパンケーキ


シカを少し増やしてオオカミを群衆の中に放り込んでみましょう。



モデルは穏やかに、不安定になりました。オオカミはすべてのシカを即座に刈り取り、すぐに死んでしまいました。

1つの欲求不満、禅はありません。

2回目の試み


何かを変更する必要があります。

オオカミがどれだけ爆発的に繁殖しているかに目を痛めます。

彼らの生活を少し複雑にしましょう-隣接する鹿の細胞に狼よりも鹿がいる場合にのみ繁殖することが可能であるという条件を設定します。

    var preys = spots.map(p => this.GM.get(p)).filter(u => u && u.kind == Wilds.DEER)
    var preds = spots.map(p => this.GM.get(p)).filter(u => u && u.kind == Wilds.WOLF)
    if (preys.length <= preds.length)
      return false

シカの個体数が最大に達したらオオカミを放棄します。

この試みははるかにうまく成功しました。

捕食者と犠牲者のバランスは常に変化しており、シカの個体数は大幅に減少し、現在では最大に近づくことさえありません。



しかし、何かが起こる可能性があり、ほとんどの場合、オオカミが絶滅し、シカは再び茂みを勝ち誇って埋め尽くします。

配備


エピローグとして、展開手順。

それは非常に短いです：

1.シンプルなdockerファイルを書きます：

FROM node:14
ADD https://github.com/tprlab/predator-prey/archive/master.zip /
RUN unzip /master.zip
WORKDIR /predator-prey-master
RUN npm install
EXPOSE 8081
CMD ["node", "server.js"]

2. Dockerビルド。-t predator-prey

3. docker run -p 8081：8081 predator-prey
