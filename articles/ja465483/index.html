骨董品：Windows 3.1およびスタートボタンのない生活

私のコンピューターの現実では、Windows 3.xは長くは続かず、ほんの数年であり、Windows 95の登場後すぐに忘れられました。しかし、90年代前半でさえ、それは主なツールではありませんでした。ほとんどすべてがDOSで行われました。ゲーム、ファイルの操作、テキストの操作、インターネット（まだWebではなく、メールとニュースグループのみ）も「コンソール」ユーティリティを介して行われていました。どうやってそうなった？ Windowsの3番目のバージョンは、四半世紀前の苛酷なテキストエクスペリエンスに対するオプションのアプリでしたか？

20年前のコンピュータで十分に遊んだ後、私は以前のシステムを研究し始めました、そして私はdosゲームを除いて彼らは私にとって暗い森です。今日は、MS-DOS上のグラフィカルシェルではなく、まだオペレーティングシステムではないWindows 3.1に精通している知人への最初のアプローチです。1992年にリリースされたこのバージョンのWindowsには、今日でも明らかな多くのインターフェイス要素がありませんが、最も価値の高いマインスイーパを含む多数の革新が含まれていました。

私は古い鉄の収集家の日記を電報に保管しています。年末までに、もっと適度に古い電話をいくつか研究する予定ですが、それだけではありません。386番目のプロセッサをベースにしたコンピュータを構築する大規模プロジェクトが間もなく始まります。

Windows、Mac、OS / 2およびすべてすべて


グラフィカルユーザーインターフェイスのコンテキストにおける80年代の終わりは、さまざまな、互換性の低いプラットフォームのサラダです。今ではすべてが始まったばかりであり、そのときソフトウェア開発者は進歩を進めるだけでなく、10年以上蓄積されてきた過酷な遺産にも対処する必要がありました。Windowsでは、コンピュータとのユーザーインタラクションの概念は最終的にWindows 95と一緒にのみ定義されたと言えるでしょう。その後、主に表面的な改善と機能の改善がありました。そしてその前に何が起こりましたか？


そんな感じ。いいえ、1984年にリリースされたApple Macintoshのインターフェースを見ると、最新のMac OS Xに関連する要素がすでにかなり目立ちます。


しかし、1985年にリリースされたWindows 1.0は、現代の「ウィンドウ」とはまったく異なります。何故ですか？どちらの会社も情報と開発を（自発的でそれほど多くはないが）交換し、Xerox PARCリサーチセンターの初期の開発にアクセスできたが、要点はMicrosoftがAppleに遅れをとっているように思える。しかし、それだけではありません。 「マック」-それはまだ最高の価格カテゴリのコンピュータでしたが、マイクロソフトは大規模なハードウェアで動作する大規模なソリューションを作成しました。そして、このハードウェアは、控えめに言っても、コンピュータが美しいインターフェイスを描画し、同時にいくつかのプログラムを起動し、同時に何か便利なことをする状況には特に適していませんでした。

グラフィカルシェルを備えたマルチタスクOSのコンセプトを本当に引き立たせるほど強力な80386プロセッサは、Intelの最初の32ビットプロセッサで、1985年にリリースされましたが、穏やかに言えば、大衆にはあまりアクセスできません。 90年代の初めになって初めて、すべてが判明しました。グラフィックシェルの開発経験と鉄の入手可能性の両方です。また、これはプロセッサだけでなく、十分に高い解像度の画像を表示できるグラフィックシステムでもあります。最初はVGA標準（640x480、1987）に準拠し、次にSVGA（800x600、1988年に開始）に準拠しています。

私たちは、この非常に表面的な遠足を歴史に加え、コンピュータビジネスの組織化に関する多くの困難を加えています。これは、GUI要素の著作権に関するAppleおよびMicrosoftの訴訟です。また、Intelは386プロセッサの生産を（以前と同様に）横に移すことをためらっていたため、その結果（1991年まで、AMDがクローンの生産を開始するまで）高額でした。そして、マイクロソフトとIBMの複雑な関係は、「次世代の軸」を一緒に大声で見ていましたが、静かに離婚し始めていました。そして、IBMのオープンPCクローン市場を独自のフォーマットに戻すための独自の試みがあり、そのために互換性のないシステムバス（MCA）、キーボードとマウスのコネクター、およびコンピューター全体（PS / 2）が開発されました。

1990年に発売されたWindows 3.0は時代の変化を示しました。創造的な検索が終わり、金儲けが始まりました。インターフェースが思い浮かびます。フルマルチタスク用に、386プロセッサの拡張保護モードがサポートされています。同時に、8086プロセッサを搭載した初期のPCとそのクローンまでの古いシステムのサポートが維持されましたが、実際の利点は、大容量（少なくとも4メガバイト）のメモリ容量を備えた386および486番目のシステムで顕著でした。1992年にWindows 3.1への更新がリリースされ、最新リリース（3.11）が1993年11月にリリースされました。これが16ビットWindowsの話が終わるところです。同じ1993年に32ビットWindows NTがリリースされ、2年後、「人気のある」Windows 95がリリースされました。

コンピュータが速すぎる場合


実験のために、私はWindows 3.1を選びました。バージョン番号のわずかな変更にもかかわらず、それは元の3.0の大きなアップデートでした。Intel 8086の古いシステムのサポートは廃止され、TrueTypeフォントのサポートが登場しました。RAMの16メガバイトの制限が削除されました。これは、私のテストコンピュータにとって重要です。


これは、私のコレクションにあるものの中から、このシリーズで最も低消費電力のThinkPad 600です。昨年、MS-DOSをどのようにインストールしたかについて詳しく話しましたが、その後Windows 3.1は曲がってインストールされ、実際には使用されませんでした。この省略を修正する時が来ましたが、待ってください、それは同じ1998年のコンピューターです。Pentium II 233 MHzプロセッサを搭載。少し多すぎませんか？

多すぎますが、このコンピューターはWindows 3.1を公式にサポートしています。ハードウェア用のドライバーがあります。また、私は（Pentium MMXプロセッサを搭載した）やや強力でないラップトップを持っていますが、それほど大きな違いはありません。Windows3.1の場合、これはいずれにしても多すぎます。 1992年の時点で、50メガヘルツまたは66メガヘルツの周波数を持つIntel 80486プロセッサがトップコンピュータにインストールされていました。しかし、386番目と286番目はまだ早すぎて帳消しにはなりませんでした。幸いなことに、私のラップトップを適切なパフォーマンスにするには速すぎる方法があります。


このビデオは、非常に役立つオーストラリアのチャンネル、フィルのコンピューターラボからのものです。エンターテインメントに関しては、このチャンネルは他のYouTubeの製品よりも劣っていますが、フィルは古いアイアンのコレクターである同僚に多くの有用な情報を提供しています。上記のビデオは、Setmulユーティリティを使用してプロセッサのパフォーマンスを低下させる方法を示しています。Setmulユーティリティは、DOS用の最新の（愛好家によって2014年から開発された）ユーティリティです。 Intel、VIA、およびAMDの一部のプロセッサの周波数を下げることができます。私の場合、これは機能しませんでしたが、別の方法が機能しました。


Philのビデオには、古い3DBenchベンチマークにおけるさまざまなシステムの「リファレンス」パフォーマンスインジケータが含まれています。最初は160ポイントの結果を得ましたが、setmulユーティリティを使用して一次キャッシュを無効にした後、プロセッサのパフォーマンスはほぼ正確に16倍に低下しました。10.3ポイントの最終結果は、25メガヘルツの周波数を持つ80386プロセッサと完全に一致します。もちろん、これは完全に正直な減速ではありません。いずれにしても、ThinkPad 600にはメモリが多すぎ（32メガバイトがボードにはんだ付けされているため、追加のモジュールを取り外しました）、ハードドライブの代わりにコンパクトフラッシュカードが取り付けられています。しかし、実験には適しています。また、内蔵の光学ドライブを3インチドライブに交換しました。

フロッピー禅


事実は、Windows 3.1がCD-ROMで提供されることは決してなかったことです（UPD、たまにそうなった、コメントを参照） ミスターグリム）：6つの3インチフロッピーディスクと7つの5インチフロッピーディスクから選択できます。これで、別のメディアからWindowsをインストールし、必要なファイルをコピーするだけで済みますが、フロッピーディスクからのインストールは正しく、時代に対応しています。ディストリビューションのソースは伝統的にWinWorldPCウェブサイトのアーカイブでしたが、英語版でのみ利用可能なWin3.1の元のディスクイメージがあります。これはあまり良くありません。最新のOSとは異なり、インストール後にロシア語のサポートを追加しても機能しません。


ビデオのように、第386回の「エミュレーション」モードでは、インストールに8分または10分以上かかります。インストール時間は、フロッピーディスクからデータを読み取る速度によって明らかに制限されます。インストール後、私のラップトップのビデオカードとサウンドカード用のドライバーを追加する価値があります（IBMラップトップの場合はここから入手できます）。ありませんが、亀裂が必要です。そのようなの検索では、私が出会ったことにより、本 StiplerのCyrWin亀裂が言及されている、アレクサンダーとグリゴリーフロロフ。そして、分布が見つかりました。


そして、それが起こったのです。 Windows 3.xロジックでは、「プログラムマネージャー」はグラフィカルシェル、デスクトップ、または別のウィンドウの「スタート」ボタンの祖先の重要な要素であり、閉じるとMS-DOSに戻ります。ウィンドウコントロールボタンは、Macintoshのボタンに似ています。右側ではウィンドウを最小化および最大化でき、左側のボタンではウィンドウを閉じることができるコンテキストメニューを開きます。通常の右クリックメニューはまだ存在しません。 Windowsと共に、メモ帳、ペイントブラシ、電卓など、よく知られた多くの名前を持つ適切なプログラムセットが付属しています。タスクバーがないため、時計は別のアプリケーションです。最小化されたが動作するプログラムがデスクトップに配置されます。


Windows 3.1では、マインスイーパが最初に登場し、「7番目」までのすべてのWindowsの標準的な居住者であり、過去27年間のオフィスワーカーの喜びです。Solitaire Solitaireは、1992年にWindows 3.0の最初のリリースと共に少し早くWindowsに登場しました。マインスイーパがあれば、あなたは生きることができます！はい、そうでもありません。


組み込みのファイルマネージャは、新しいドラッグアンドドロップをサポートしていますが、クリップボードを介した複数のファイルのコピーはサポートしていません。一度に複数のファイルを転送する必要がある場合は、別のウィンドウでパスを入力する必要があり、インターフェイスでフォルダーを選択するオプションすらありません。90年代前半でさえ、ファイルマネージャーはあまりうまく応答しませんでしたが、現代の基準では、それは闇と恐怖です。幸い、Total Commanderがいます：


フォークパネルファイルマネージャーは、開発者のサイトで入手できます。 16ビットシステムのサポートはバージョン6.58で停止しましたが、現在の9.22はWindows 95 / NT以降のすべてのOSで動作します。 Total Commanderをインストールして問題なく動作し、自分のライセンスキーも認識しました。


しかし、Microsoft Wordをインストールする試みは失敗に終わりました。Windows 3.0と3.1（1991）のリリース時間に対応するバージョン2.0を選択しました。まず、インストール中に亀裂の問題があり、起動後にフォントの問題がありました。2番目のWordは、TrueTypeフォントを操作するときに問題が発生します。この場合、WYSIWYGの原則に準拠していません。印刷すると、画面に表示されているものが正確に表示されません。Wordのクラシック6バージョンをインストールする必要があります：1993年にリリースされました（2.0以降の次のリリースで、 UPD によって番号3.4および5がスキップされました：これらのバージョンのWordはDOSでのみ存在します）、TrueTypeをサポートし、基本的に必要なすべてを提供しますテキストを操作します。Word 6.0はとてもクールで、2000年代以前には多くの人が使用していました。


感想


1992年4月、PC MagazineはWindows 3.1を問題のテーマにしました。より有望になることはすでに表紙から明らかです。同じ年にリリースされた32ビットOS / 2 2.0は技術的にはるかに高度であり、副次的であることが判明しました。そのレビューでは、主にWindowsアプリケーションを実行するのが便利かどうかについて説明しています。 1992年に、Windowsは以前はAppleコンピューターでのみ利用可能だったソフトウェア、特にAdobe Photoshopバージョン2.5もリリースしました。



PC Magazineは、すべての変更を詳細に説明するだけでなく、配布ファイルの完全なリストと説明を提供します（Windows 10でこれを試みます）。レビューは興味深い事実を提供します。ハードドライブの読み取りおよび書き込み用の改善されたキャッシュシステムのおかげで、一部のシナリオでは、生産性が大幅に向上します。さて、ソフトウェアの例を見つけるのは難しいでしょう。その更新は同じ利点を与えますが、逆ではありません。 Windows 3.1では、安定性が大幅に向上し、バグを検索するための情報が開発者に提供されます。これは、WindowsからDOSプログラムを実行する場合に特に当てはまります。それ以前は、多くの場合、情報のないエラー「回復不可能なアプリケーションエラー」が発生していました。

Windows 3.1では、オブジェクトのリンクと埋め込みのテクノロジが標準になり、ExcelからWordにテーブルを挿入したり、組み込みのレコーダーアプリケーションから書き込みテキストエディターにオーディオ録音を挿入したりできるようになりました。多くの読者にとって、Windows 3.1が最初のグラフィカルシェルになると仮定すると、PC Magazineは購入する5つの理由を提供します。これは、高品質のフォントと複雑なグラフィックを備えたドキュメントの作成です。一度に複数のアプリケーションを実行します。 DDEおよびOLEテクノロジーを介したアプリケーションの相互作用。ソフトウェア開発者からのWindowsへの明確な関心。そして最後に、「Windowsが気に入らない場合は、簡単に削除するか、無視するだけです。」


これが、Windows 3.xがレトロDOSシステムを完全に補完する理由です。これは、バージョン6.22までのクラシックなハイグレードDOSで動作し、再起動せずにテキストモードに戻ることで、いつでもオフにすることができます。ただし、「製品版」では、完全な32ビットディスクサブシステムと通常のネットワークサポートを備えたワークグループ用の最新バージョンのWindows 3.11を引き続き使用します。 Windows 95の方が便利ですが、386システムの場合はすでに重いため、80486またはPentiumプロセッサをベースにしたコンピュータの方が適しています。 1995年に初めてPC互換のコンピューターを手にしたときは、386プロセッサーでした。すべてにもかかわらず、私はWindows 95を使用しました。それはひどく遅いですが、選択肢は明白でした。より近代的であるほど、より優れています。 2019年のWindows 3.xと95の議論はほとんど意味がありません。しかし、Win3.1との2回目の知り合いの結果から、すべてがそれほど悪いわけではないことに気付きました。これは、時間枠に厳密に準拠したレトロコンピュータを構築したい場合に適したOS（または好きなシェル）です。


これは私が近い将来行うことです。
