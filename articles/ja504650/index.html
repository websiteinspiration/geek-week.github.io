具体的な例に関するAlpine.js

すでにAlpine.jsについて聞いたことがあるかもしれません。そうでない場合、これは「少なくともVue.js」です。「ミレニアル世代のAngular1。」それをあなたが望むもの、あなたが理解する主なものと呼んでください。


なぜ別のフレームワークが必要なのですか？まあ、アルパインはそのニッチによく合います。実際、これは、これらの大規模なフレームワークが不要なサイトの大規模なフレームワークの代替手段です。たとえば、トレンディなWeb開発者である私は、複数ページのサイトを作成するために利用されました。フォームを初級およびその他のインタラクティブにする必要があります。何をしたらいいでしょう？私はjQueryを採用します-友人は理解していません。純粋なJSの記述では、すべてが失敗するわけでもありません。さらに、私はすでに反応、角度、ビューを知っています。データ駆動型アプローチとは何かを知っています。HTMLで送信する前にデータを保存するにはどうすればよいですか？


ここでAlpine.jsが役に立ちます。もちろん、VueまたはReactをねじ込むことができます。しかし、JSXなしでReactを使用する場合、正しい心の中に誰も書くことはありません。また、CDNを介して縮小されたVueの重量は34 kBです（Alpineの場合は8.1 kB）。したがって、選択はアルパインにかかっていることがわかります。


最近、ここHabréでAlpine.js についてのなじみのある記事がすでにありました。サイトがテンプレートタグへの参照をいたるところに切り取っているという事実は別として、記事は非常に優れていることがわかりました。


繰り返しにならないように、ドキュメントは非常に短く（フレームワーク全体が13のディレクティブと6つの魔法のプロパティです）、まもなくロシア語で利用できるようになるため、再掲しません（執筆時点では、翻訳者は作成者の承認を得ていますが、すでにそれは完了しています）私のフォークで読むことができます）。

    return {
      // ...
      inputValue: '',
      addTodo: function () {
        if (!this.inputValue) {
          return;
        }

        this.todos.push({
          id: Date.now(),
          title: this.inputValue,
          completed: false,
        });
        this.inputValue = '';
      },
    };
  }
</script>
