O que os desenvolvedores JavaScript devem saber sobre Curl





Bom dia amigos



Apresento a você a tradução do artigo “O que os desenvolvedores de JavaScript devem saber sobre o Curl”, de Valery Karpov.



O Curl é uma ferramenta de linha de comando popular, frequentemente usada para enviar solicitações HTTP. O Curl suporta um grande número de protocolos; no entanto, como desenvolvedor do Node.js., você provavelmente o usará para enviar solicitações HTTP para a API RESTful.



A documentação do curl é uma lista de 383 sinalizadores de linha de comando, por isso é muito difícil encontrar o que você está procurando. Neste artigo, quero compartilhar com você alguns dos modelos que costumo usar. Por exemplo, o serviço httpbin.org será usado .

Fazer upload de arquivos



O Wget é a ferramenta mais comum para baixar arquivos via linha de comando. Está incluído na maioria das distribuições Linux. No entanto, no OSX não é.



O comando é wget url



semelhante ao comando curl -OL url



. Uma opção -



é uma opção --remote-name



que informa ao curl para salvar o corpo da resposta em um arquivo local. A opção -L



diz ao curl para seguir os redirecionamentos.



Por exemplo, abaixo está uma imagem de Unsplash, seu URL é https://images.unsplash.com/photo-1506812574058-fc75fa93fead



.







Para baixar esta imagem, faça o seguinte: 



$ curl -OL https://images.unsplash.com/photo-1506812574058-fc75fa93fead
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
                                 Dload  Upload   Total   Spent    Left  Speed
100 12.1M  100 12.1M    0     0  3927k      0  0:00:03  0:00:03 --:--:-- 3927k


A opção -O



informa ao curl para usar a linha após a última /



como o nome do arquivo. No exemplo acima, a imagem será salva no diretório atual com o nome photo-1506812574058-fc75fa93fead



. Para determinar o nome do arquivo, use a opção -



(letra minúscula "o"). 



$ curl -o miami-beach.jpg https://images.unsplash.com/photo-1506812574058-fc75fa93fead
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current

Enviando uma solicitação autorizada



O cabeçalho da autorização é usado para incluir dados de autorização na solicitação ao acessar a API RESTful. Para adicionar os dados especificados, você deve usar o sinalizador -H



. Por exemplo, se sua chave de interface for (chave da API) my-secret-token



, você poderá incluí-la na solicitação http da seguinte maneira: 



$ curl -H "Authorization: my-secret-token" https://httpbin.org/get
{
  "args": {}, 
  "headers": {
    "Accept": "*/*", 
    "Authorization": "my-secret-token", 
    "Host": "httpbin.org", 
    "User-Agent": "curl/7.58.0", 
    "X-Amzn-Trace-Id": "Root=1-5ee8e1a5-a3aa30e0765a7980b04ca4a0"
  }, 
  "origin": "69.84.111.39", 
  "url": "https://httpbin.org/get"
}


Note que ele httpbin.org



retorna os cabeçalhos da solicitação de http no corpo da resposta na propriedade headers



.



A ondulação também suporta autorização padrão por meio de um sinalizador -u



. No exemplo a seguir, enviamos uma solicitação com um nome de usuário user



e senha pass



: 



$ curl -i -u "user:pass" https://httpbin.org/basic-auth/user/pass
HTTP/2 200

  "authenticated": true,

Aqui está o que acontece ao enviar o nome de usuário ou senha incorretos: 



$ curl -i -u "user:wrongpass" https://httpbin.org/basic-auth/user/pass

Conclusão



Em resumo, aqui estão as opções de ondulação que acho mais úteis:



  • -X



    define um método de solicitação, por exemplo curl -X POST url



  • -d



    define o corpo da solicitação como uma sequência nas solicitações PUT e POST. Use @



    para extrair dados de um arquivo
  • -H



    define o cabeçalho da solicitação, por exemplo curl -H "Authorization: my-secret-token" url



  • -u



    dados de autenticação para autorização padrão
  • -O



    salva o corpo da solicitação no arquivo
  • -i



    mostra resposta completa, incluindo cabeçalhos


O Curl é uma ferramenta útil para interagir com a API por meio da linha de comando, independentemente de ser uma API de terceiros ou uma API que você está desenvolvendo. Para testes rápidos, curl é melhor que o Axios no Node.js ou o ajuste de consultas no Postman se você estiver familiarizado com a sintaxe.
