Mode operasi server surat



Saat memasang Postfix pada distribusi berbasis Debian, antarmuka pseudo-grafis diluncurkan dengan seperangkat templat untuk instalasi. Template ini ditujukan untuk berbagai mode operasi server surat, yang bergantung pada apakah server surat akan menerima surat, meneruskannya dan berbagai variasi dari kedua mode ini. Tapi ini hanya templat yang mudah diubah setelah menginstal server. Meskipun saya menggunakan Postfix pada Debian sebagai contoh, mode ini bersifat universal untuk semua server email.

Opsi "Situs Internet"





Mode kedua - Situs Internet - adalah templat untuk server surat standar yang akan menerima surat dan mengirimkannya secara langsung, tanpa perantara. Mari kita pergi melalui opsi konfigurasi.



Pertama, kita perlu menentukan nama surat - yaitu, domain untuk alamat surat yang ditunjukkan setelah anjing. Secara default, postfix menggunakan pengguna pada sistem sebagai pengguna server mail. Dengan demikian, jika Anda memiliki pengguna pengguna dalam sistem, maka ia akan dapat menerima dan mengirim email ke user@example.org



Untuk tujuan keamanan, tidak disarankan untuk menggunakan akun root untuk bekerja dengan surat, oleh karena itu kami diminta untuk menunjukkan pengguna kepada siapa semua surat yang tiba di alamat postmaster, root dan pengguna sistem lainnya akan diteruskan. Seringkali, berbagai aplikasi dalam sistem, seperti penjadwal tugas cron, mengirim informasi ke alamat-alamat ini.



Selanjutnya, penginstal meminta kami untuk menentukan domain yang akan menerima email server kami. Di sini, selain alamat domain utama kami, ada juga entri localhost dan berbagai variasi nama host. Ini terutama diperlukan untuk berbagai layanan internal yang mengirim pemberitahuan. Selain itu, sebagai opsi, beberapa layanan mengirim pemberitahuan tertentu ke alamat lengkap server email Anda, misalkan server email Anda telah terdaftar dalam daftar spam, misalnya, ke postmaster@mail.example.org.



Sekarang pertanyaannya adalah tentang pembaruan yang sinkron. Pertanyaan ini lebih terkait dengan layanan surat yang dimuat dan berbicara tentang hal-hal berikut. Jika kami mengaktifkan pembaruan sinkron, server email akan memastikan pesan ditulis ke disk setiap kali pesan dikirim dan diterima. Ini mengurangi kinerja server surat, tetapi jika server tiba-tiba dimatikan saat menerima atau mengirim email, tidak akan ada pesan yang hilang. Tetapi jika kita menggunakan sistem file penjurnalan, seperti ext3 atau ext4, maka sistem kita tidak takut akan shutdown mendadak.



Hal ini disebabkan oleh fakta bahwa dalam sistem file jurnal semua perubahan dalam sistem file dipra-rekam dalam jurnal khusus yang menerapkan perubahan ini dan kemudian menghapusnya dari jurnal.





Jika server tiba-tiba dimatikan pada saat perekaman, maka ketika Anda menyalakannya, sistem file memeriksa log, melihat perubahan yang tidak lengkap dan menulisnya ke disk. Karena itu, jika Anda menggunakan sistem file penjurnalan, silakan klik Tidak.



Sekarang kami ditawari untuk menentukan jaringan-jaringan yang postfix-nya akan bertindak sebagai host relay. Dalam peran ini, server email tidak hanya menerima email ke domainnya, tetapi juga dapat meneruskan pesan ke server email lain. Misalkan Anda dapat mengirim pesan ke Gmail dengan menetapkan bukan smtp.gmail.com tetapi mail.examle.org sebagai server email keluar, maka server email Anda akan mengurus pengiriman pesan Anda ke smtp.gmail.com. Kami akan membicarakan hal ini secara lebih rinci nanti, ketika kami akan membahas templat lainnya. Sementara itu, jika Anda tidak berencana untuk menggunakan server Anda sebagai relay host, Anda hanya dapat meninggalkan localhost di dalam field.



Semuanya sederhana di sini - ukuran maksimum default kotak surat pengguna, dalam byte. Jika 0, tidak ada batasan.



Maka sesuatu yang lebih menarik -alias ubahsuaian . Secara umum, alias di server email digunakan sehingga satu pengguna dapat menerima email di beberapa alamat, tanpa membuat pengguna yang terpisah, kotak surat, dll. Katakan, jika Anda ingin semua pesan ke it@example.org dikirim ke user@example.org, maka alias digunakan untuk ini. Selain itu, menggunakan alias, domain berbeda terikat ke satu pengguna. Alias ​​ini dapat dibuat oleh administrator server surat. Namun, administrator dapat mengizinkan pengguna untuk membuat sesuatu seperti alias untuk memudahkan pengguna mengelola email mereka. Tetapi alias pengguna ini tidak menggantikan seluruh alamat, tetapi ditambahkan sebelum anjing, setelah menggunakan karakter khusus, katakanlah, plus. Sebagai contoh, gmail juga menyediakan fitur ini:



Misalnya, user+work@example.org, atau user+social@example.org. Pengguna dapat menentukan alamat-alamat ini pada berbagai sumber selama pendaftaran, yang akan membuatnya lebih mudah untuk menyortir surat dan mengelola surat-suratnya. Jika Anda tidak membutuhkan fungsi ini, Anda dapat mengosongkan bidang ini.



Selanjutnya, Anda harus memilih apakah server email kami akan berfungsi dengan ipv4, ipv6, atau dengan kedua protokol.

Pada pengaturan ini dengan templat Situs Intenet sudah siap, secara teoritis Anda bisa mulai bekerja.

Opsi "Internet dengan smarthost"





Opsi ketiga di antara mode operasi server surat adalah internet dengan smarthost. Ini adalah templat konfigurasi untuk server yang akan menerima email secara langsung, atau menggunakan utilitas apa pun seperti fetchmail, tetapi server akan mengirim email ke alamat eksternal menggunakan server email lain - relay host, yang juga disebut host pintar. Ini dapat digunakan dalam berbagai kasus, misalnya, jika alamat IP server Anda ada dalam daftar hitam, atau untuk mengirimkan surat perusahaan keluar melalui layanan pencegahan kebocoran, dll.



Secara umum, semua opsi dalam mengatur mode ini juga terlihat sama, dengan pengecualian menentukan server relai. Host cerdas dapat ditunjukkan dalam tanda kurung dan tanpa, tergantung pada apakah Anda ingin mengirim pesan segera atau setelah membuat permintaan MX, yang memungkinkan Anda untuk menyeimbangkan lalu lintas antara beberapa host pintar.



Dalam templat, kami dapat menentukan smarthost default, dan di masa mendatang kami dapat mengonfigurasi berbagai relay tergantung pada domain penerima.

Opsi Sistem Satelit





Mode operasi lain dari server surat adalah sistem satelit. Dalam mode ini, server surat tidak menerima surat, tetapi hanya pengalihan, yaitu, ia bekerja secara eksklusif sebagai host relay. Dari contoh sebelumnya, ini dapat berguna dalam organisasi besar untuk layanan internal, serta untuk memfilter email untuk spam, virus, dll. Dalam hal ini, konfigurasi hampir identik dengan opsi yang disebutkan sebelumnya, tetapi dalam penentuan saya Anda tidak harus meninggalkan domain sehingga tuan rumah mengalihkan semuanya ke server mail yang diperlukan.

Opsi "Hanya lokal"



Dan templat terakhir - Hanya lokal - saat server email kami tidak akan mengirim dan menerima email dari server lain.
