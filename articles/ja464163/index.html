前書き


キーワードthisは、JavaScript言語の最もわかりにくい機能の1つです。Javaから来て、それはOOPの実装を支援するためのものでした。私はしばらくの間Javaで書いていましたが、この間this、コードの特定の場所に何が等しいのか疑問に思ったことがあるかもしれません。JavaScriptでは、そのような疑念が毎日発生する可能性があります-少なくとも、いくつかの単純ではあるが自明ではないルールを学ぶ瞬間まで。

についての誤解 this


このキーワードに関しては、いくつかの一般的な誤解があります。私は要点に達する前にそれらをすぐに払いのけたいです。

これは語彙のコンテキストです。

この印象は初心者の間でしばしば発生します。彼らには思えるthis-これは、プロパティとして、すべての変数がこのスコープに格納されるオブジェクトです。この誤解は、ある特定のケースでは、おおまかに言ってこれがそうであるという事実に起因します。最高レベルの場合は、（ブラウザーの通常のスクリプトの場合）とthis等しくなりwindowます。そして、ご存じのように、トップレベルで宣言されたすべての変数は、プロパティとして使用できますwindow。

一般に、これは真実ではありません。これは簡単に確認できます。

function test(){
  const a = "     ,   ";
  console.log(this.a);
}
test(); //  ,

これは、メソッドが属するオブジェクトです。

繰り返しますが、これは多くの特定の場合に当てはまりますが、常にそうとは限りません。重要なことは、関数がどのように呼び出されるかであり、それがオブジェクトのプロパティであるかどうかではありません。これは非常に単純なロジックからも明らかです。同じ関数が2つのオブジェクトのプロパティであるとします。

const f = function(){
  console.log(this);
}

const object1 = {
  method: f
};

const object2 = {
  method: f
};

それで、これらのオブジェクトのどれが彼女になるでしょうか？

重要！ES6クラスを使用してオブジェクトが作成された場合でも、そのメソッドが常に正しいものを持っているとは限りませんthis。「通常の」言語のクラスに似ていることにだまされないでください。

これはジェダイのテクニックで、一度勉強

すればどこでも使用できるようになりthisます。可能であれば避けてください。それが完全に正当化される唯一のケースはOOPです。他の場所では、使用thisはそれほど悪ではありませんが、原則として、コードを混乱させる過剰です。実際this、これは関数への追加の「マイナス1番目」引数であり、初心者には複雑で自明ではない方法で渡されます。高い確率で、それは通常の議論で置き換えることができ、それは良くなるだけです。
