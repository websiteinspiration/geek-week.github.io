C ++イベントのセマフォ

今日は、同期オブジェクト「イベント」に基づいてセマフォを実装する方法について簡単に説明します。

最初に定義を確認します。

1.同期とは何ですか。なぜ必要なのですか。


明らかに、いくつかの方法で一連のアクションを実行できます。最も単純-直列および並列。特定のアクションの並列実装は、さまざまなスレッドを実行することで実現できます。アイデアは単純です。各スレッドに基本的な（またはそうでない）アクションを割り当て、それらを特定の順序で開始します。一般的に言えば、それらはすべて同時に起動することができます-もちろん、時間内に利益を得るでしょう。これは理解できます。たとえば、10,000ワードを次々と出力することと、たとえば100ワードを同時に出力することです。100倍の時間ゲイン（プラスまたはマイナス、遅延を除くなど）。ただし、元のタスクでは、厳密な一連のアクションが必要になる場合があります。

例えば：

  • ファイルを開く
  • ファイルにテキストを書き込む
  • ファイルを閉じる

温室の例は特別に取られました（ここでは並列処理は必要ないことは明らかですが、すべてを単純に順次実行できます）が、トレーニングタスクとして完全に機能し、最も重要なのは、一貫した実行の必要性がその例ではっきりとわかることです。または、わずかに異なる別の例を次に示します。

  • 乱数の3つのシーケンスを生成する
  • 順番に表示する

ここでは、最初のポイントを3つの異なるスレッドで同時に実行できますが、最後の結論は、最初のポイントを計算した後でのみ、順次実行する必要があります。

一般に、並列処理タスクは大きく異なる可能性があり、スレッドを同期するためにいくつかのツールが必要です。

2.スレッド同期のためのツール


Windows.hは、多くの通常の同期ツール（いわゆる「同期オブジェクト」）を実装しています。主なものには、クリティカル領域、イベント、ミューテックス、セマフォがあります。はい、セマフォの場合、windows.hにはすでに実装があります。 「それでなぜそれをプログラムするのですか？」 - あなたが尋ねる。まあ、まず、それがどのように機能するかについてより良い感じを得るために。そして、第二に、C ++の余分な練習はまだ誰も止めていません:)

イベントを使用することを前提として、それが何であるか、およびそれを使用する方法を説明します。

イベントは、保留中のスレッドに通知するために使用されます。つまり、実際には、これはフローに対する何らかの信号です。トリガーすることも、まだトリガーすることもできません。このオブジェクトの非常に意味から、いくつかの信号状態とそれを調整する機能（リセット/「オン」）があることになります。

したがって、windows.hを接続した後、次のコマンドを使用してイベントを作成できます。

HANDLE CreateEvent
(
	LPSECURITY_ATTRIBUTES lpEventAttributes,	//  
	BOOL bManualReset,				//  : TRUE - 
	BOOL bInitialState,			//  : TRUE - 
	LPCTSTR lpName				//  
);

関数が成功すると、イベントハンドルが返されます。オブジェクトを作成できなかった場合は、NULLが返されます。

イベントのステータスをシグナルに変更するには、次の関数を使用します。

BOOL SetEvent
(
	HANDLE hEvent		//  
);

成功した場合、ゼロ以外の値を返します。

さて、セマフォについて。セマフォは、同時に実行されるスレッドの数を制御するように設計されています。1000のスレッドがあるとしますが、一度に機能できるのは2つだけです。これは、セマフォを使用して行われる調整のタイプです。そして、この同期オブジェクトを操作するためにどの関数が実装されていますか？

イベントと同様にセマフォを作成するには、次のようにします。

HANDLE CreateSemaphore
(
	LPSECURITY_ATTRIBUTES lpSemaphoreAttributes,	//  
	LONG lInitialCount,			//    
	LONG lMaximumCount,			//   
	LPCTSTR lpName				//  
);

成功した場合はセマフォへのポインタを取得し、失敗した場合はNULLを取得します。

セマフォカウンタは常に変化しているため（スレッドが実行中であり、空の場所が表示されます）、セマフォの状態を定期的に変更する必要があります。これは、次の関数を使用して行われます。

BOOL ReleaseSemaphore
(
	HANDLE hSemaphore,		//   
	LONG lReleaseCount, 	//    
	LPLONG lpPreviousCount	//  
);

成功した場合、戻り値はゼロではありません。

関数にも注意を払う価値があります：

DWORD WaitForSingleObject(
  HANDLE hHandle, //   ,    
  DWORD  dwMilliseconds //    
);

戻り値のうち、特に注目するのは2：WAIT_OBJECT_0-オブジェクトの状態がシグナルであることを意味します。WAIT_TIMEOUT-割り当てられた時間内にオブジェクトからの信号状態を待機しませんでした。

3.タスク自体


全体として、私たちの仕事は、通常の関数でアナログを書くことです。タスクはそれほど複雑ではなく、「最初の近似での実装」を行います。主なことは、標準のセマフォの定量的な特性を維持することです。コメント付きのコードはGitHubにあります。

タスク自体は単純なので、記事を特に複雑にすることはしませんが、誰かにとって重宝になるかもしれません:)
