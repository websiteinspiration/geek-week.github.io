AWS EC2サービスと連携



みなさん、こんにちは。来週、クラウドサービスコースのクラスが始まります。オープンレッスン「AWS EC2サービス」について簡単に説明します。10年以上の経験を持つクラウドサービスの教師兼スペシャリストであるEgor Zuevによって実施されました。このセミナーのフレームワークでは、AWS EC2サービスを確認し、シンプルなアプリケーションをエンコードしてデプロイし、データベースを構築し、イメージのレプリケーションとロードバランサーについても検討しました。


AWS EC2サービスの概要


EC2（エラスティックコンピューティングクラウド）は、仮想/物理マシンにオペレーティングシステム（Ubuntu、Debian、Windowsなど）の事前に作成されたイメージを提供することにより、コンピューティングパワーを提供するサービスです。選択可能な

インスタンスのタイプ：



イメージと同様に、AWS EC2はAMI（Amazonマシンイメージ）を使用します。これは、EC2インスタンスで使用される準備済みのオペレーティングシステムのイメージでもあります。最も近い例は、Virtual Boxなどのシステムがすでにインストールされている仮想ディスクです。

マシンを構成してイメージを選択するときは、アクセスを構成する必要があることを知っておくことが重要です。マシンはクラウド内にあるため、私たちと他のユーザーはそれに手を差し伸べる必要があります。これを行うには、セキュリティグループを構成します。これは、EC2インスタンスに適用され、送受信トラフィックのアクセス許可を含む一連のルール（ポリシー）です。

コンソールを操作する


サービスに登録するとすぐに、AWS EC2コンソールに移動します。そこで、EC2と、すべてをデプロイするマシンのタイプを選択できます。この例では、Ubuntu Serverを選択します。







重要なポイント：ステップ2では、t2micro（緑色のステッカーでマークされています）に関心があります。

次の3つのステップでは、デフォルト設定のままにしますが、ステップ6には特に注意が必要です。ここでは、グループの名前（たとえば、backend-security-group）を入力し、すぐにルールを追加して、80番目のポートのみが外部から接続できるようにする必要があります。



7番目のステップで、「起動」をクリックしてインスタンスを起動します。しかしその前に、システムはssh経由で接続するためのキーを作成するように要求します。すべての手順が完了すると、初期化に1〜2分かかります。

データベースに接続します


データベースとして、PostgreSQLを使用することを決定しました。続行するには、次の設定で新しいインスタンスを作成します。

  • [サイズ]列に16 GBを入力します（ステップ番号4）。
  • セキュリティグループ名はdatabase-security-groupと呼ばれます（ステップ番号6）。
  • サブリストからPostgreSQLを選択して、構成にルールを追加します（ステップ番号6）。
ちなみに、ここではIPアドレスフィールドでセキュリティグループを操作できます。ここでは、backend-security-groupについて説明しています。したがって、このグループに属するすべての人がプライベートネットワーク上のデータベースにアクセスできます。ここでは、このプライベートネットワークがどんな魔法であるかを聞くことができます。
