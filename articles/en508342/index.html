What is segmentation?



In a broad sense, ‚Äúsegmentation‚Äù, as follows from the Oxford Dictionary, is a division of something into several parts, or one of such parts. In our case, segmentation is a task in computer vision, which allows you to divide a digital image into different parts (segments) according to which object which pixels belong to. Thus, we get the pixel-by-pixel mask of the object.

Interactive Image Segmentation 



Manual marking made it possible to assemble datasets (for example, ImageNet and Microsoft COCO), which underlie all the modern achievements of computer vision. One of the most valuable and time-consuming types of markup is image segmentation. The difference between interactive segmentation is that it requires user participation throughout the process. In particular, the user can select the necessary objects and correct errors of the algorithm with the help of clicks. Interactive segmentation can be used in various applications and for image editing. For example, in Paint 3D on Windows 10 there is a function for selecting image areas with a click. 



About the problem and existing ways to solve it



In this paper, we consider click-based interactive segmentation: the user selects the desired objects using positive or negative clicks. Positive clicks indicate that the click area refers to the desired object, and negative clicks indicate that it does not.



Most click-based interactive segmentation algorithms contain a backbone network trained on the ImageNet dataset. At the entrance, the backbone network receives the image itself and maps of positive and negative clicks. At the output, the model creates a segmentation mask for the desired object. The main problem with this approach is that it is impossible to adjust the result of the response of the neural network. This problem arises due to the peculiarities of neural networks - in a sense, they are black boxes, and no one can guarantee that the trained model will always have an invariant obvious to people: under a positive click there should always be an object mask, and under a negative one - no . Example: a user clicks on an object, but the neural network for some reason does not select the area under the click. Clicking again - the result does not change,because the neural network gave an incorrect answer to a given click, and this answer cannot be corrected. 



How can I solve this problem? At CVPR 2019, authors from Harvard and Koryo University collaborated on Interactive Image Segmentation via Backpropagating Refinement Schemepresented a technique to improve existing interactive segmentation algorithms. First, user input creates cards with positive and negative clicks. They are transmitted to the convolutional neural network along with the original image, and then everything is standard - the neural network generates a probability map. In this map, for each pixel, the probability that this pixel belongs to the object we need is set. Since the neural network does not guarantee that the predictions will be correct in these clicks, the authors proposed using the backpropagating refinement scheme (BRS), which will result in the correct segmentation mask during the next direct pass.

How is this achieved? In BRS, the following optimization problem is solved: it is necessary to minimize the mean square error (MSE loss) of the prediction only at the points where user clicks are located. To do this, the map of user click distances ( distance maps ) is modified pixel by pixel so that the network output matches them better. In our opinion, the intuition explaining the success of such a solution originates from adversarial attacks methods .(note that the authors in their article did not draw a similar analogy). Thanks to the opening of adversarial attacks, it became known that small changes to the network inputs (usually just an RGB image) can dramatically affect the final result of the prediction. BRS is essentially the same optimization task that is solved for adversarial attacks.







BRS has its drawbacks. Its main drawback is a long working time, since at each iteration a return pass is made throughout the network. 



Proposed solution



Before proceeding to the description of the proposed solution, we turn to a recent article by authors from the Moscow Center for Artificial Intelligence Samsung, which solves the problem of instance segmentation. The presented architecture is based on a pre-trained network on ImageNet. It has: 



  1. Instance selection network. It predicts object masks at given coordinates, adapting to each specific object using the AdaIN operation - Adaptive Instance Normalization. 

  2. The controller network (controller network), which extracts the feature from the backbone network in coordinates (x, y), passes it through fully connected layers, and then, using the received parameters, sets up AdaIN layers in the Instance selection network. 

  3. Relative CoordConv , , (x, y) 





The ‚Äúhead‚Äù of the network takes a click on the input, and then the entire network adjusts to the specified object.



Why did we turn to this article? Most recently, authors from the Moscow Samsung Center for Artificial Intelligence have discovered that the BRS mentioned above can be applied to any part of the network. This discovery, together with rethinking the ideas from the article discussed above, allowed us to create f-BRS - Feature Backpropagating Refinement Scheme - in which any output of the middle layer of the network can act as a target optimization variable. In f-BRS, only a small set of parameters within the network is optimized, namely, scale and bias for signs of a fixed layer. This preserves accuracy and significantly increases the speed of the algorithm. 



The f-BRS method has several possible configurations (f-BRS-A, f-BRS-B, f-BRS-C). They differ in the position of the intermediate layers, which must be optimized. In the experiments, only intermediate layers of the DeepLabV3 + decoder are considered, which allows to reduce the computational cost of the BRS return pass. 



The presented architecture looks like this: 







There are two approaches for modifying intermediate layers - pixel-wise (pixel-wise) and channel-wise (channel-wise). In the pixel-wise approach, having a tensor of 100x100 in width and height and 64 in depth, we get 100x100x64x2 = 12800000 parameters. The larger the number of parameters, the greater the likelihood that the network will retrain. In addition, all values ‚Äã‚Äãof the feature map are optimized independently, which can lead to the problem of ‚Äúlocal retraining‚Äù (see illustration below). In the channel-wise approach, we work with only 64 channels and train much less parameters, which makes the approach resistant to retraining.

Another innovation that is presented in the article is the Zoom-In technique, which improves the quality of interactive segmentation. In previous work on interactive segmentation, crop images were used to speed up the model and preserve fine details in segmentation masks. This allows you to get more accurate masks of small objects, but, on the other hand, the results may become worse if the initial resolution of the object is greater than the resolution of the crop. 



The experiments showed that 1-3 clicks are enough for the neural network to reach 80% of the Jacquard measure (Intersection over Union, IoU) with respect to the ground truth mask. That is, starting from the third click, it is possible to receive crop in accordance with the bounding box of the object mask and apply interactive segmentation only to the desired image area. 



Step-by-step Zoom-In technique can be written as follows: 



  1. First we find the bounding box for the predicted object mask.

  2. Extend the rectangle by 40% on each side. 

  3. We get the crop of the image along the borders of the rectangle. 

  4. We increase the resolution of the resulting fragment of the image (initially there was a picture in low resolution). 

  5. Again we make the prediction of the object mask.

Zoom-In improved the results in all experiments and was used by default with f-BRS. At the same time, Zoom-In can also be used with other models for interactive segmentation.
