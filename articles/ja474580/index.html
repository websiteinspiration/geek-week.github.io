ax可用性テストライブラリの哲学


開発者が常にプロジェクトのアクセシビリティをテストし、「手頃な」コーディングの原則を適用し始めた場合にのみ、インターネットは真に包括的なスペースになります。

自動可用性テストにより、開発時間を大幅に節約できます。これを実行するために特別なスキルは必要ありません。これにより、チームは最も重要なこと、つまり製品開発に集中することができます。残念ながら、ほとんどのテストツールは、開発が完了したサイトとアプリケーション向けに設計されており、これが原因で信頼性の高い結果を得ることは非常に困難です。その結果、製品のリリースが間近に迫っているように思われるときに、期限が切れる可能性があります。

axライブラリは、専門家が使用するすべての最新のブラウザ、ツール、テスト環境と互換性があります。そのおかげで、アクセシビリティテストは、チームが毎日実行する他のテスト（たとえば、単体テスト、統合テスト、ブラウザーテストなど）の一部にすることができます。開発の初期段階でアクセシビリティテストツールを使用すると、時間とリソースを節約できるだけでなく、多くのトラブルも回避できます。

AXマニフェスト


  • axはオープンソースプロジェクトです。
  • 誤検知はありません（バグはカウントされません）。
  • axは、すべての最新のブラウザで動作し、あらゆるツール、プラットフォーム、ライブラリ、環境で動作します。
  • axは主要なデジタルアクセス会社（Deque Systems）によってサポートされています。
  • このライブラリは、自動テスト用の既存の機能/受け入れツールと簡単に統合できます。
  • axは、評価コンテキストに基づいて、実行するルールを自動的に決定します。
  • axは、内部メモリフィクスチャ、静的フィクスチャ、統合テスト、およびiFrame無限ネストをサポートしています。
  • axの構成は非常に柔軟です。

Reactを使い始める



GitHubのreact-axページ

NPMまたはYarnを使用してモジュールをインストールします。

NPM：
npm install --save-dev react-axe

糸： 
yarn add react-axe

モジュールを起動します


エクスポートされた関数を呼び出して、ReactオブジェクトとReactDOMオブジェクトとミリ秒単位の時間遅延を渡します。これは、コンポーネントの各変更と分析の開始時間の間で追跡されます。

const React = require('react');
const ReactDOM = require('react-dom');

if (process.env.NODE_ENV !== 'production') {
 const axe = require('react-axe');
 axe(React, ReactDOM, 1000);
}

フレームワークなしで実行


まず、パッケージをインストールしますaxe-core。
npm install axe-core --save-dev

APIパッケージの構成


ax-core API パッケージには以下が含まれます。
  • axe.js -テストしたサイト（API）に追加するJavaScriptファイル。
  • axe.min.js -上記のファイルの小さいバージョン。

JavaScriptファイルを各iFrameフィクスチャまたはテストシステムに追加します。
<script src="node_modules/axe-core/axe.min.js"></script>

次に、各テストポイントへの呼び出しを追加します。テストポイントが表示されるか、新しいインターフェイス要素が表示されます。
axe.run(function (err, results) {
 if (err) throw err;
   ok(results.violations.length === 0, 'Should be no accessibility issues');
   // complete the async call
   ...
});

（上記のコードのように）開発環境でモジュールを起動します。そうしないと、アプリケーションはテスト中に通常の操作中よりも多くのリソースを使用します。（この例に示すように）envifyを使用できます。 起動後、コンポーネントが更新されるたびに、モジュールはChrome Devtoolsコンソールにアクセシビリティエラーに関する情報を表示し始めます。 したがって、Reactアプリケーションを起動すると、コンソールにすべてのアクセシビリティ問題（ある場合）が表示されます。axでエラーとアクセシビリティの問題を表示する








サポートされているブラウザ


ax-core APIは、以下のブラウザーをサポートしています。
  • Microsoft Edge 40以降。
  • Google Chrome 42以降;
  • Mozilla Firefox 38以降。
  • Apple Safari 7以降;
  • Internet Explorer 9、10、11。

サポートとは、バグを修正し、できる限り定期的にブラウザをテストすることを意味します。現在、プルリクエストごとに、Firefox、Chrome、およびInternet Explorer 11のみが

テストされていますが、JSDOMサポートはまだ制限されています。現在、開発者はJSDOMと互換性のあるすべてのルールを作成しようとしていますが、これが不可能な場合は、ルールをオフにすることをお勧めします。このルールはJSDOMでは機能しないことがわかっていcolor-contrastます。

現在、axは、すべての機能がネイティブにサポートされているか、正しく実装されている環境でのみ機能します。ライブラリは、v0 Shadow DOMの廃止された実装をサポートしていません。
