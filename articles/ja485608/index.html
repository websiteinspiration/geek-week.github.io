Spring Bootによるマイクロサービス。パート4.負荷分散のためのリボンの使用

これは、マイクロサービスアーキテクチャに関するシリーズのパート4です。ここでは、ロードバランシングにリボンを使用することに焦点を当てます。

この記事は、Spring Boot Microservicesシリーズの一部です。

このシリーズの記事では、マイクロサービスの概念を理解し、Spring BootとSpring Cloudを使用してマイクロサービスを作成する方法を学びます。

このガイドは、マイクロサービスアーキテクチャの基本を学ぶのに役立ちます。また、Spring Bootを使用した基本的なマイクロサービスの実装についても見ていきます。

いくつかのマイクロサービスを作成し、それらをEureka（Eureka Naming Server）とリボンネームサーバーを使用して相互に通信させ、クライアント側の負荷を分散します。

このチュートリアルでは、Spring Bootマイクロサービスプロジェクトでロードバランサーとしてリボンを使用する方法について説明します。

マイクロサービスの概要


前の2つの部分では、マイクロサービスを作成し、それらの間の接続を確立しました。

GET to http://localhost:8100/currency-converter-feign/from/EUR/to/INR/quantity/10000

{
  id: 10002,
  from: "EUR",
  to: "INR",
  conversionMultiple: 75,
  quantity: 10000,
  totalCalculatedAmount: 750000,
  port: 8000,
}

上記のサービスを起動すると、リクエストがForexサービスにも送信されていることがわかります。これはカッコいい！

2つのマイクロサービスを作成し、それらの間の接続を確立しました。



ただし、FSのURLはCCSコンポーネントCurrencyExchangeServiceProxyにハードコーディングしました。

@FeignClient(name="forex-service" url="localhost:8000")
public interface CurrencyExchangeServiceProxy {
  @GetMapping("/currency-exchange/from/{from}/to/{to}")
  public CurrencyConversionBean retrieveExchangeValue
    (@PathVariable("from") String from, @PathVariable("to") String to);
}

これは、Forexサービスの新しいインスタンスを起動するときに、それらに負荷を分散する方法がないことを意味します。

この部分では、リボンを使用してクライアント側の負荷分散を有効にします。

必要になるだろう：


  • Maven 3.0+-ビルドツール
  • お気に入りのIDE。Eclipseを使用しています。
  • JDK 1.8以降

コードサンプルを含むReady Mavenプロジェクト


Githubのリポジトリには、すべてのコード例があります。

リボンをオンにする


pom.xmlにリボンの依存関係を追加します。

    <dependency>
      <groupId>org.springframework.cloud</groupId>
      <artifactId>spring-cloud-starter-ribbon</artifactId>
    </dependency>

CurrencyExchangeServiceProxyでRibbonClientを有効にします。

@FeignClient(name="forex-service")
@RibbonClient(name="forex-service")
public interface CurrencyExchangeServiceProxy {

application.propertiesでインスタンスを構成します。

forex-service.ribbon.listOfServers=localhost:8000,localhost:8001

8001で外国為替サービスを開始


前の手順では、インスタンス間で負荷を分散するようにリボンを構成しました。ただし、8001で実行されている単一のForex Serviceインスタンスはありません。

次の図に示すように、起動構成を構成することで開始できます。



動作中のリボン


現在、次のサービスがあります。

  • 8100のマイクロサービス通貨コンバーター（CCS）
  • 8000および8001上のForexマイクロサービスの2つのインスタンス

これで、CCSへの要求がリボンを介してForexマイクロサービスの2つのインスタンス間で分散されることがわかります。

リクエスト1


GET to http://localhost:8100/currency-converter-feign/from/EUR/to/INR/quantity/10000

リクエスト2


GET to http://localhost:8100/currency-converter-feign/from/EUR/to/INR/quantity/10000

2つの回答のポート番号が異なることに気付くでしょう。

概要


2つのマイクロサービスを作成し、それらの間の接続を確立しました。



リボンを使用して、Forexサービスの2つのインスタンス間で負荷を分散します。

ただし、CCSで両方のFSインスタンスのURLをハードコーディングします。つまり、FSの新しいインスタンスが表示されるたびに、CCS構成を変更する必要があります。かっこよくありません。
