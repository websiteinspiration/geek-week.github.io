Google広告データハブとは

マーケティング担当者の典型的な痛みは、ディスプレイ広告の効果を評価することです（それとの相互作用は広告主のWebサイトの外で発生するため）。従来のWeb分析ツールはこれを行うことができず、残りのツールは透過的な評価を提供しません。

この問題を解決するために、Google はAds Data Hubを立ち上げました。これは、メディアインベントリデータを取得してその効果を評価するためのツールです。ディスプレイ広告データをサイトデータおよびCRMと組み合わせることができます。

以下では、Ads Data Hubとは何か、どのように機能するかについて説明します。

Ads Data Hubとは何ですか？


従来のビュースルーコンバージョンがあります。ほとんどのメディアツールは、バナーを見た人の何人のユーザーが注文したかを知らせます。ただし、これらのツールは、バナーの表示から購入までのすべてのユーザーのステップを示しているわけではなく、CRMシステムから購入した注文を考慮に入れていません。

広告主がキャンペーン情報と独自のデータを組み合わせることができるように、広告サービスはGoogleのデータ転送などのアップロードツールを提供しています。以前は、ユーザーIDを含め、クリック、インプレッション、イベントごとに、DoubleClick Campaign Manager（DCM）からGoogle BigQueryに未加工のログデータをアップロードするために使用できました。ただし、2018年のGDPRルール広告主ではなくサイト運営者のサイトとやり取りするため、広告主によるユーザーデータのアップロードを禁止しました。

したがって、Googleは広告主にプライバシー要件に違反することなく、ディスプレイ広告の効果を透過的に評価する機会を与えました。これで、ユーザーIDはDCMおよびDPMからアンロードされませんが、Ads Data Hubでの作業に使用できます。

他に何がAds Data Hubの外観を引き起こしますか？ほとんどの広告主にとって、ディスプレイ広告はテレビや屋外広告の代替手段です。つまり、これはパフォーマンスマーケティングの代わりではなく、別のカバレッジツールです。ただし、同じテレビとは異なり、ディスプレイ広告には確立されたベンチマークはありません。したがって、広告主はディスプレイ広告のおかげで、どのオーディエンスとやり取りしたかを示すツールが必要です。

そしてGoogleはそのようなツールを作りました。近い将来、ディスプレイ広告に投資するほとんどの大規模な広告主は、Ads Data Hubを開発する必要に直面するでしょう。

Google広告Data Hubの仕組み


Ads Data Hubは独立したリポジトリではありません。これは、Google BigQueryでデータを操作するためのツールです。本質的に、Ads Data Hubは2つのBigQueryプロジェクト（独自のプロジェクトとGoogleプロジェクト）をリンクします。



Googleプロジェクトには、キャンペーンマネージャー、ディスプレイ＆ビデオ360、YouTube、Google広告のログデータが保存されます。これは、GDPR規則により外部から取得できない情報と同じです。

ユーザーが所有する2番目のプロジェクトには、Googleアナリティクス、CRM、またはその他のソースからBigQueryにアップロードしたすべてのマーケティングデータ（オンラインとオフライン）が保存されます。Googleチームはプロジェクトにアクセスできません。

Ads Data Hubは、ユーザーレベルでそれらをアップロードすることなく、これら2つのプロジェクトから同時にデータをリクエストできるAPIです。これにより、すべてのクリックとコンバージョンをインプレッションデータに関連付け、特定のキャンペーンがコンバージョンにどのように影響したかを確認できます。

重要：Ads Data Hubによって返されるクエリ結果は、最大50ユーザーまで集計されます。つまり、テーブルの各行には約50人以上のユーザーのデータが含まれている必要があります。特定の1人のユーザーにドリルダウンすることはできません。この制限は、GDPRルールに準拠するために導入されました。

Ads Data Hubを使用して作成できるサンプルレポート


Googleディスプレイネットワークでディスプレイキャンペーンを実施していて、その効果を評価したいとします。Ads Data Hubを使用すると、キャンペーンマネージャーのインプレッション、Googleアナリティクスのセッション、オンライン注文、CRMの購入注文を1つのSQLクエリで組み合わせることができます。

その結果、次の情報を含むレポートが表示されます。

  1. 広告キャンペーンの名前。
  2. 各キャンペーンのインプレッション数。
  3. キャンペーンのリーチは、バナーを見たユーザーの数です。
  4. バナーを見た後にサイトを訪れたユーザーのセッション。さらに、ユーザーは広告内のリンクをクリックする必要はありません。これは、セッションとクリックなしでサイトにアクセスしたユーザーを考慮に入れています。たとえば、広告を見て記憶した後、検索であなたのサイトを見つけました。
  5. 広告のリンクをクリックします。
  6. オンラインコンバージョン。
  7. CRMからの確認済みの注文。



つまり、Ads Data Hubを使用すると、ポストビューコンバージョンを評価できます。ADHなしでこのようなレポートを作成する場合、バナーをクリックしてサイトにアクセスしたユーザーのみが存在します。クリックしなかったがサイトにアクセスしたユーザーについては何も知りません。また、レポートにはCRMからの実際の売上はありません（レポートの黄色の列）。

Ads Data Hubの使用方法


Ads Data Hubを使用するには、サイトとGoogle BigQueryのCRMシステムのデータが必要です。これらのデータを、広告キャンペーンのインプレッションに関するデータと組み合わせます。

また、サイト訪問データとインプレッション情報を組み合わせるための識別子が存在するように、キャンペーンマネージャー（以前はDCM）へのクライアントクライアントID転送を構成する必要があります。これは、たとえばFloodlightを使用して行うことができます。

以前は、情報を交換するために、マーケティング担当者はExcelファイルを作成し、相互に送信していました。時間が経つにつれ、データを1か所に保存し、必要なときに関連する情報を利用する方がよいということがわかってきました。そこで、ドキュメント用のクラウドのコンセプトが登場しました。

もちろん、Excelは現在も今もデータの処理に使用されています。しかし、これらは2つの異なる概念です。1つはユーザーが1つのソースに接続するとき、もう1つは全員がデータのコピーを持つときです。

MarTechデータは、コンピューティングとレポート作成のためのすべてのMarTechサービスがデータのコピーを必要としたときに、同じ経路に沿って歴史的に進化しました。 GAでレポートを作成する場合-データをそこにアップロードし、DCMでレポートを作成する場合-データをBigQueryにアップロードします。 Ads Data Hubは、このパラダイムの概念的な代替品です。

何が問題ですか？これで、マーケティング担当者はGoogle広告からBigQueryにデータをアップロードできるようになり、その後、広告サービスが遡及的に変更されます。その結果、彼のBigQueryプロジェクトでは、データが更新されないと古いデータになります。

2番目の問題は、どのデータをアップロードするかです。ディスプレイ広告に関する情報はたくさんあります。マーケティング担当者が週に1度レポートを作成するには、ギガバイトの情報を彼のプロジェクトにアップロードする必要があります。代わりに、Ads Data Hubは、SQLインターフェースをGoogleの広告サービスのデータに接続することをお勧めします。つまり、GBQプロジェクトがあります。これには、Googleにはないデータが含まれています（たとえば、CRMから）。広告キャンペーンに関するデータを含むレポートが必要になったので、それらをリクエストに関連付けて、最新の情報をここで取得します。プロジェクトに広告データを保存する必要がなくなりました。履歴期間中に何かが変更された場合、それらを毎日更新する必要もありません。レポートの場合、収集したデータはすべて利用できます。
