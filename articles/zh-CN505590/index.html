def top_3_similar_items(X,mindex): # X   ,    -3 ,       'mindex'

                third_best_val = tmp

    return first_best_val,first_best_index,second_best_val,second_best_index,third_best_val,third_best_index #

2. CPU和GPU的比较


本质上，中央处理器（CPU）是任何计算设备的大脑：它执行程序中编写的指令，执行控制，逻辑运算以及输入/输出运算。

的确，现代CPU仍然没有很多内核，并且CPU的基本结构和用途-复杂计算的处理-本质上并没有改变。实际上，CPU最适合解决涉及解析或解释代码中包含的复杂逻辑的问题。

反过来，图形处理器（GPU）具有较小的逻辑内核，但是逻辑内核却要大得多（我们正在谈论算术逻辑设备（ALU），控制元素和高速缓存存储器），这些逻辑内核是为整体并行处理而设计的相同且相对简单的操作。


GPU比典型的CPU具有更多的算术逻辑单元（ALU），因此增强了并行处理简单操作的能力

从CPU的角度来看，运算的想法-有这样的运算，例如斐波那契数列的计算，无法并行化。毕竟，只有在计算了前两个后才能找到下一个数字。这种操作最适合CPU。
继而，使用矩阵容易地执行诸如矩阵加法和乘法的运算，这是因为矩阵单元中的这些运算中的大多数是彼此独立的，本质上是相似的，因此可以并行化。

4.简要介绍CUDA


CUDA是Nvidia创建的并行计算平台和API模型。使用此API，您可以使用支持CUDA图形的处理器进行广泛的计算。这种方法称为GPGPU（GPU上的非专业计算）。这里对它们进行了更详细的描述。

NUMBA

Numba是一个免费的JIT编译器，可使用LLVM将Python和NumPy的子集转换为快速的机器代码，这是使用Python中的llvmlite软件包完成的。该软件包提供了许多用于并行化CPU和GPU的Python代码的选项，而代码本身中的更改很少通常就足够了。查看更多。
使用处理器Nvidia Volta V100 16GB GPU，我使用了Numba库。

建筑细节〜流，块和网格

CUDA使用流，块和网格等抽象组织并行计算。

流：CUDA流是分配给CUDA核心的/当前的指令链（实际上，它只是一个管道）。动态运行时，同一CUDA内核上最多可以存在32个线程（在这种情况下，该管道的所有链接都已填充）。这是具有给定索引的内核执行。每个线程使用其索引来访问数组中的元素，以使整个可用线程集一起处理整个数据集。

块：这是一个线程组。关于块内线程的执行，可以说不多：它们可以顺序执行或并行执行，也可以不执行任何特定顺序。例如，您可以使用_syncthreads()强制线程在内核中的某个点停止并等待直到同一块中的所有其他线程也到达同一点的函数来协调线程。

网格：这是一组块。块之间没有同步。



CUDA：线程，块，网格〜源

但是线程，块和网格到底在哪里执行？以Nvidia的G80 GPU架构为例，计算结果显然分布如下：

网格→GPU：整个网格由单个GPU处理器处理。
块→MP：GPU处理器被组织为多处理器的集合，其中每个多处理器负责处理网格中的一个或多个块。一个块永远不会在多个MP之间分配。
流→PP：每个MP进一步细分为流处理器（PP），每个PP处理一个块中的一个或多个线程。

@cuda.jit('void(float32[:], float32[:], float32[:])') #  Cuda 
def cuda_addition(a,b,c):
    """     ."""
    i = cuda.blockIdx.x * cuda.blockDim.x + cuda.threadIdx.x #     
    if i > c.size:
        return
