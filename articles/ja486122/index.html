135行のコードを持つ子ReactJS

ReactJSに関する記事はすでにたくさんあります。しかし、初心者プログラマーの勉強を始めるためには、その作成の基礎がどこから始まるのかを最初に見つける必要があります。このフレームワークのフロントエンド開発の原則を理解するのに複雑なことは何もないことを示したかったのです。


研究の1つの方法により、多くの複雑なソフトウェアシステムの構造を理解することができました。そのため、研究しているプロジェクトを書き直す必要があるという事実に帰着します。その結果、少なくとも少し動き、そのプロトタイプにいくらか似ている未知の獣を手に入れることができます。

私がバックエンドであることを認める必要があります。JavaScriptでマイクロフレームワークを書く動機が何であったのかさえわかりません。しかし、率直に言って、これはJSを学びたいという願望です。

実際、モジュール/コンポーネントの形式でプロジェクトを作成する動機は十分にあります。バックエンドプログラマーから見ると、データはせいぜいJSONオブジェクトのように見えます。データを目的の構造に形成し、必要に応じて送信してから、必要な処理を実行する必要があります。フロントエンドの最も原始的なバージョンでは、IDで必要なHTML要素を選択し、それらの属性を更新し、テキストノードを変更する必要があります。 JavaScriptフレームワークを使いやすくします。

PHP-Slim-frameworkを書いたら、これはオリジナルとはかけ離れていますが、PHPプロジェクトで本当に役立ちます。今日は、ReactJS開発の起源を紹介した方法についてお話ししたいと思います。私はbots.jsというコードを含む135行のコードで1つのファイルを作成しました。これを接続してReactのようなコンポーネントを作成すると、ブラウザーに何かを表示することもできます。私は彼をReactKidsと呼んだ。

アイデアは、開発中のプロジェクトをコンポーネントに分割し、JavaScriptを使用してコンポーネントを追加し、コンポーネント間に依存関係がないことを確認することです。

ReactKidsのコンポーネントは次のようになります。

function Button(attr) {
	// attribute components default values
	if(!attr.labelButton) attr.labelButton = "Click Me"
	
	return elem(
		"button",
		{
			padding: "0.65rem",
			marginTop: "0.4rem",
			color: "gray",
			border: "1px solid gray",
			borderRadius: "0.5rem",
			background: "#fff",
			fontSize: "large",
			cursor: "pointer",
		},
		{
			id: "btn1",
			click: btn1Click,
		}, 
		attr.labelButton
	)
}

function btn1Click(e) {
	console.log("Clicked!")
	setAttr(Label({labelContent: "i-i-i!!!"}), 0.6)
}

この場合のコンポーネントは、paramsがattrと呼ばれる関数のみです。
ここで、この属性が役立つ理由に注意する必要があります。まあ、最初に、反応に精通している人は、娘のコンポーネントにデータを「下げる」ことが可能であることを知っています。つまり、コンポーネントはコンポーネントを返すコンポーネントを返し、以下、子のないコンポーネントまで続きます。ただし、サーバーからのデータのパッケージとしても使用されます。バックエンド要求は、ほとんどの場合、ユーザーインターフェイスとの対話に関連するイベントを処理する関数から送信されます。

サーバーがJSON（通常はテキスト）を返す場合、それはJSオブジェクトに変換され、どこかで実行される必要があります。これが、子実装のReactおよびattrのパラメーターの目的です。
attrでは、サーバーから受信したJSONオブジェクト全体を詰め込むことができます。または、必要な最高のデータのみを取得し、場合によっては他の必要なデータで補足することもできます。

次に、大人のReactのロジックに従います-関数の最初で、attrオブジェクトを処理し、他のビジネス業務を実行します。その後、elem（）関数を呼び出した結果を返す必要があります。この実装はbots.jsにあります。以下のパラメーターが呼び出しパラメーターに転送されます。

  1. タグの名前。
  2. スタイル付きオブジェクト（JS形式）
  3. タグの属性。
  4. テキスト、別のタグまたはコンポーネント（子）または何も渡されません。

もちろん、子供たちはJSXの欠如に気づきました。実際、それは怠惰なプログラマーによって発明され、単に私たちが書くことを容易にします。その結果、JSXタグは引き続きJavaScriptに変わります。

これがbots.jsでどのように実装されているかを確認する必要があります。フレームワークは、実際にはelem（）とsetAttr（）の3つの関数全体で構成されています。

function elem(elt, style, attr, item) {
	/*element */
	if(elt) {
		//  ,      
		var el = document.createElement(elt);
	} else {
		console.log("elt fail")
		return 
	}
	/* style */
	if(style) {
		if(typeof(style) == "object") {
			for(var itm in style) {
				el.style[itm] = style[itm]
			}
		} else {
			console.log("style is not object type")
		}
	} else {
		console.log("style fail")
	}
	/* attr */
	if(attr) {
		if(typeof(attr) == "object") {
			for(var itm in attr) {
				if(typeof(attr[itm]) == "function") {
					el.addEventListener(itm, attr[itm])
				} else {
					// standart
					el[itm] = attr[itm]
				}
			}
		} else {
			console.log("attr is not object type")
		}
	} else {
		console.log("attr fail (add ID for element)")
	}
	/* item */
	if(item) {
		if(typeof(item) == "string") {
			var text = document.createTextNode(item)
			el.appendChild(text)
		} else if(typeof(item) == "object") {
			if(Array.isArray(item)) {
				if(item.length < 1) {
					console.log("not items in array")
					return
				}
				item.map(function(itm) {
					el.appendChild(itm)
				})
			} else {
				el.appendChild(item)
			}
		} else {
			console.log("text is not string or object type")
		}
	} else {
		console.log("text fail")
	}
	return el
}

関数は、渡されたパラメーターを同じシーケンスで処理します。

  1. ドキュメントツリーにコンポーネントを作成します。
  2. スタイルを追加します。
  3. 属性。
  4. 子要素をテキストまたはその他の要素として追加します。

属性を処理するときに、属性のタイプもチェックします。関数が値として受信された場合、これはイベントであると想定され、盗聴が行われます。したがって、イベントとして示される関数を宣言して実装するだけです。

このイベント処理関数でsetAttr（）を呼び出し、オブジェクト自体に更新されたattrを渡します。 1つだけありますが、attrで作成する各要素にはidを指定する必要があります。それ以外の場合は、setAttrを介して更新されません。 IDで彼女はDOMでそれを見つけます。

setAttr（）に関しては、原則を理解するのに十分（またはほぼ十分）ですが、すべてがReactよりも悪いです。

function setAttr(update, slow) {
	if(slow) {
		var replace = document.getElementById(update.id)
		var opamax = 0.99
		var opaint = 0.01
		var outslow = setInterval(function() {
			opamax = opamax - opaint
			if(opamax <= 0) {
				clearInterval(outslow)
				update.style.opacity = opamax
				replace.parentNode.replaceChild(update, replace)
				var inslow = setInterval(function() {
					opamax = opamax + opaint
					update.style.opacity = opamax
					if(opamax >= 1) {
						clearInterval(inslow)
					}
				}, slow)
			}
			replace.style.opacity = opamax
		}, slow)
	} else {
		var replace = document.getElementById(update.id)
		replace.parentNode.replaceChild(update, replace)
	}
}

ここに見られるように、ドキュメントツリーと別のフェード効果による操作のみなので、少なくとも見た目とコードはhelloworldではなく関数のように見えます。
