F＃4：させる/使う/する

この投稿では、F＃のバインディングについて説明します。特に、Let / Use / Doについて説明します。ここで、バインディングとは何かを自問しているかもしれません。バインディングについてはまだ調べていないので、ここでそれについて説明します。

簡単に言えば、バインディングは識別子を値または関数に関連付けます。

しましょう


名前を値または関数に関連付けるには、letキーワードを使用します。実際には、Letの微妙な異なる使用方法があり、1つはモジュールのトップレベルで宣言され、もう1つはローカルコンテキストを定義します。これらの両方の例を次に示します。

module DemoModule =
 
    let someFunction =
        let a = 1
        let b = 2
        a * b

DemoModule.someFunctionなどの完全修飾名を使用してsomeFunctionにアクセスできますが、ネストされたLet（a、b）バインディングは、Let'sトップレベルでのみ使用できます。通常、Letバインディングを使用して内部モジュールのいくつかの値を宣言すると、より多くのケースが表示されるので、そこに取り組みを集中させましょう（ただし、モジュールレベルでLetを使用できることを知っておくことが重要です）。

それでは、さらにいくつかの例を見てみましょう。

let aString ="this is a string"
let aInt = 12
let aDecimal = 12.444
let aPiFunction () = Math.PI
let aSquareRootFunction (x) = Math.Sqrt(x)
let aFullyTypedSquareRootFunction (x :float) = Math.Sqrt(x)
let a,b = "a","tuple"

Letバインディングを使用して、次のようなさまざまなタイプの複数の値にバインドできることがわかります。

  • 整数
  • 10進数
  • 入力パラメーターなしの関数
  • 入力パラメーターを持つ関数（F＃型推論システムが型を正しく選択する場所）
  • 完全に定義されたパラメーター型を持つ関数
  • タプル（この場合、String * String tuple）

他の場所では、クラスでLetバインディングを確認できますが、このシリーズの次の記事でこれについて詳しく説明します。

Letバインディングの詳細については、MSDNを参照してください。

使用する


Useバインディングは、値を式の結果にバインドするため、Letバインディングと非常に似ています。主な違いは、UseバインディングはIDisposable型で機能するように設計されており、スコープから外れたときに値を自動的に削除することです。これは.NETのusingキーワードとよく似ていますが、F＃Useバインディングが.NETでの使用とまったく同じになるとは思いません。 ）ついに。

テキストフォーマッティングに関する前回の投稿でUse bindingの例をすでに見ましたが、念のため、もう一度もう一度見てみましょう。

use sw = new StreamWriter(@"c:\temp\fprintfFile.txt")
fprintf sw "This is a string line %s\r\n" "cat"
fprintf sw "This is a int line %i" 10
sw.Close()

この例では、Useバインディングにより、StreamWriterメソッドが上記のsw.Close（）を呼び出した後にDispose（）メソッドを呼び出すことが保証されています。

UseはIDisposablesでのみ機能し、



次に示すようにそれを他のもので使用しようとするとコンパイルエラーが発生します。Letを使用して接続されました。

let Write =
    use sw = new StreamWriter(@"c:\temp\fprintfFile.txt")
    sw

Useバインディングの一部であるIDisposableを絶対に渡す必要がある場合は、代わりにコールバックを使用できます。このようなものは機能しますが、そのようなことをした場合は、設計を正しく開発したかどうかを自問します。

let Write callback =
    use sw = new StreamWriter(@"c:\temp\fprintfFile.txt")
    fprintf sw "Write is writing to the StreamWriter"
    callback sw
    sw
 
let callback sw = fprintf sw "sw is the StreamWriter"
let disp = Write callback

行う


doバインディングは、関数や値を定義せずにコードを実行するために使用されます。Bindingは常にUnitを返す必要があります（値なし/ void）。多くの場合、Doバインディングは省略でき、すべてが期待どおりに機能します。

Doバインディングの使用例をいくつか示します。

do printf "doing the do"
//oh oh not a unit
do printf "print a sum %i" 1 + 1
do 1 + 1

上記のコードのスクリーンショットを代わりに表示すると、Unit以外の結果でDoを使用しようとするとコンパイラーが文句を言うことがわかります。



次の2つのオプションがあります。

  • パイプライン演算子を使用して結果を無視する
  • letバインディングを作成する

以下にそれぞれの例を示しました。

let x = 1 + 1
do printf "print a sum %i" x
do (1+1 |> ignore)

しよう！使用する！行う！


それらについてはまだ説明しませんが、Let！使用する！これを行うと、これはいわゆる計算式の一部になります。これはF＃非同期ワークフローで見られる可能性が高く、最終的な記事の1つで取り上げます。十分に理解できれば、独自の「計算式」を作成する方法を説明することもできます。ただし、それらはかなり抽象的な概念であり、かなり複雑なトピックなので、今は彼らのための時間ではありません。
