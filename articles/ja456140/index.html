Codeception 3.0に会う

Codeceptionの新しいメジャーバージョンを実現する時が来ました！長年の進化と継続的な改善を通じて、私たちは多くのことを学びました。そして今日、私たちはあなたに最高のバージョンのCodeceptionを提供します。テストの容易さは重要であると私たちはまだ信じています。テストの読み取り、書き込み、デバッグは簡単でなければなりません。

SymfonyからWordPress、LaravelからMagentoまで、あらゆる種類のフレームワークとCMSをカバーしようとするプロジェクトをサポートすることは困難です。しかし、これらのコミュニティの人々は定期的にパッチを送信しており、私たちの哲学は、テストをフレームワークから分離し、モジュールを通じて同様のソリューションを共有することです-正しい方法。私たちの素晴らしいコミュニティは、このプロジェクトを改善し続けています。

OK、Codeception 3.0はどうですか？

互換性のない変更


PHPUnit 8のサポート


バージョン3.0では、何も壊しませんでした。次のメジャーリリースのために残しました。したがって、サポートされているバージョンのPHPは提供していません。Codeception3.0は、PHP 5.6以降と互換性があり、誰でもテストにアクセスできるはずです。それでも、すべてのメインフレームワークをサポートし、PHPUnit 6およびPHPUnit 7と互換性があります。

ただし、これらのすべてのパーツを格納することは困難であり、バージョン3.0では、PHPUnit 8のサポートを追加したため、重大な変更が発生する可能性があると想定しています。それ以外の場合、アップデートあなたのためにスムーズでなければなりません。更新に問題がある場合は、composer.jsonでPHPUnitのバージョンを6または7に変更します。

"phpunit/phpunit": "^7.0"

互換性を損なうことなくPHPUnit 8をサポート してくれた主な貢献者である@Naktibaldaに感謝します。これは大変な仕事でした。私たちのcodeception / phpunit-wrapperプロジェクトを見ると、その理由が理解できます。

リモートモジュール


私たちは古いモジュールをいくつか捨てることに決めました。それ：

  • AngularJS-Angularでのみ機能しました1
  • ZF1-フレームワークは非推奨です
  • Yii1-フレームワークは非推奨です
  • Silex-フレームワークは廃止予定
  • Facebook-モジュールはサポートされていません
  • XMLRPC-サポートされていないモジュール

それらが必要な場合は、2.5ブランチからコードをコピーして、カスタムヘルパーを作成します。

変更されたデフォルト


マルチセッションテストと$ I-> haveFriendコマンドを使用すると、テストが失敗したことがわかります。FriendメソッドはCodeception \ Actorクラスに含まれていないため、手動で追加する必要があります。あなたにAcceptanceTester（または他のクラスの使用マルチセッションテストがあること）、などがCodeception \ Libの\俳優\共有形質を：

<?php
class AcceptanceTester extends \Codeception\Actor
{
    use _generated\AcceptanceTesterActions;
    use \Codeception\Lib\Actor\Shared\Friend;
}

また、新しいインストールの条件ステートメントを（ユーザーが頻繁に悪用するため）オフにしました。そのため、canSeeメソッドを使用する場合は、それらを有効にする必要があります。

新しい機会


強化されたインタラクティブシェル


この機能は、フレンドリーなCodeceptJSプロジェクトから移植されました。彼女は新しいREPLインターフェイスを追加しました。これで、テストを一時停止して、コンソールでさまざまなコマンドを実行できます。

この機能により、機能テストと受け入れテストの記述方法が完全に変わります。盲目的にさまざまなコマンドを試し、常にテストを再開する代わりに、次のように書くことができます。

<?php
$I->amOnPage('/');
$I->pause();

そして、次々にコマンドを入力し始め、テストを書くために段階的に。成功したチームをテストにコピーすると、最終的には完全に機能するテストが得られます。WebDriverを使用する場合、単一のブラウザーセッションを使用して完全な受け入れテストを作成できます。hoa / consoleに

基づくインタラクティブシェルの以前の実装とは異なり、左、右キーを使用して入力を編集できます。



メソッドの試行と再試行


これらのメソッドは、ブラウザのテストを簡単にするために導入されました。JavaScriptと単一ページのアプリケーションでいっぱいの世界では、個々のイベントに依存することができなくなり、実行するためにアクションを数回繰り返す必要がある場合があります。

これで、再試行*メソッドでこれを行うことができます。これは、テストのステップを安定させるのに役立ちます。

<?php
// use
$I->retryClick('Create');

// instead of just
$I->click('Create');

この機能もCodeceptJSから移植されましたが、実装方法が少し異なります。

しかし、サイトが異なる環境で異なる動作をする必要がある場合はどうでしょうか。たとえば、地域に応じてCookieの受け入れ通知を表示し、この通知が表示されたらCookieを受け入れますか？これは、tryTo *アクションで実行できます。

<?php
if ($I->tryToSeeElement('.alert.cookie')) {
  $I->click('Accept', '.cookie');
}

try *およびretry *メソッドは段階的なデコレーターとして実装され、モジュールメソッドをラップすることで追加のアクションを生成できます。条件付きステートメント（canSee *メソッド）も、段階的なデコレーターとしてリファクタリングされました。そのため、設定で明示的に有効にする必要があります。

出力アーティファクト


失敗したテストには、利用可能なすべてのアーティファクトのリストが含まれるようになりました。REST APIテストの場合、各テストには、CIサーバーでのデバッグを簡略化するための最後の応答の本文が含まれます。



インストールとアップグレード


バージョン3にアップグレードするには、composer.jsonでバージョンを「^ 3.0」に変更するだけです。

"codeception/codeception": "^3.0"
