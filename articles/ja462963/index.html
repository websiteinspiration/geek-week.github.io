ReactでContext APIを使用してグローバルアプリケーションテーマを作成する

こんにちは、これはHabrに関する私の最初の投稿です。興味を持っていただければ幸いです。

それで、ui-kitの作成に直接または間接的に関連する一連の投稿を開始したいと思います。



この投稿の目的：コンポーネントがで実装されているアプリケーションのテーマを制御するソリューションを見つけてReact.jsください。ダークとライトの 2つのグローバルテーマを使用します。

この例では、モジュールcreate-react-contextを使用してコンテキストを作成します。

まず、テーマルート（src /）にテーマコンテキストというフォルダーを作成します。このフォルダの構造は次のようになります。

theme-context/
   ThemeConsumer/
      ThemeConsumer.js
      index.js
    ThemeProvider/
      ThemeProvider.js
      index.js
  constants.js 
  context.js
  index.js

個人的には、私は常にindex.jsファイルから始めます。最初はすべてのインポートとエクスポートを実行し、その後、頭がそれらについて害を及ぼすことはありません。

テーマコンテキスト/index.js

export { ThemeProvider } from './ThemeProvider';
export { ThemeConsumer } from './ThemeConsumer';

テーマコンテキスト/ ThemeConsumer / index.js

export { ThemeConsumer } from './ThemeConsumer';

theme-context / ThemeProvider / index.js

export { ThemeProvider } from './ThemeProvider';

theme-context / context.js

次に、ここからcreateContextモジュールを使用して（しゃれのために申し訳ありません）を使用してコンテキストを作成します。

import createContext from 'create-react-context';

const { Provider, Consumer } = createContext();

export { Provider, Consumer };

我々は、インポートcreateContext、上でそれを破壊ProviderしてConsumer、それらをエクスポートします。

theme-context / constants.js

ここではすべてが簡単です。メインファイルを汚染しないように変数を作成します。

export const themeLight = 'light';
export const themeDark = 'dark';
export const defaultTheme = themeLight;
export const themes = [themeLight, themeDark];

先に述べたように、このアプリケーションには明と暗の2つのテーマがあります。

theme-context / ThemeProvider / ThemeProvider.js

ここでは、プロバイダー（各オブジェクトで使用可能なコンポーネント）について説明しますReact.Context。これにより、消費者は変化するコンテキストに耳を傾け、対応することができます。

この例では、prop は指定されたaのthemeすべてのスレッドに渡さProviderれるものです。

import React from 'react';
import { Provider } from '../context';
import { defaultTheme, themes } from '../constants';

function ThemeProvider({ theme, children }) {
  return <Provider value={theme}>{children}</Provider>;
}

export { ThemeProvider };

theme-context / ThemeConsumer / ThemeConsumer.js

このファイルでは、Consumerを使用します。これは、コンテキストの変更を「待機、待機」するコンポーネントです。このコンポーネントの子は関数です。これは、を使用する場合の必須要件ですConsumer。

この関数は、現在のコンテキストの値を受け取り、React Nodeコンポーネントを返します。

ドキュメントから：引数の値（この場合、{theme => / *コンテキスト値に基づいて何かを視覚化する* /}）は、このコンテキストのthemeツリーProviderで最も近い親のプロップと等しくなります。

import React from 'react';
import { defaultTo } from 'lodash';
import { Consumer } from '../context';
import { defaultTheme, themes } from '../constants';

function ThemeConsumer(props) {
  return <Consumer>{theme => props.children(defaultTo(theme, props.defaultTheme))}</Consumer>;
}

export { ThemeConsumer };

ここでは、次の点に注意する価値があります。
トピックが明示的に選択されなかった場合、コンポーネントのテーマを自動的に選択する必要があります。これには、lodash- のジャンクを使用しdefaultToます。ただし、この機能は他の多くの方法で実現できます。

これで、テーマコンテキストを使用する準備ができました！

それを適用する方法を見てみましょう。アプリケーションのコンテキストをリッスンして応答する単純なコンポーネントを作成してみましょう。

.my-class {
  font-family: sans-serif;
  text-align: center;
  font-size: 30px;
}

.my-class-light {
  color: #39cccc;
}

.my-class-dark {
  color: #001a33;
}


import React from "react";
import ReactDOM from "react-dom";
import cx from "classnames";
import { ThemeConsumer, ThemeProvider } from "./theme-context";

import "./styles.css";

function MyComponent() {
  const renderMyComponent = theme => {
    const myComponentClassName = cx("my-class", {
      "my-class-dark": theme === "dark",
      "my-class-light": theme === "light"
    });
    return (
      <div className={myComponentClassName}>
        <h1>    </h1>
      </div>
    );
  };
  return <ThemeConsumer>{theme => renderMyComponent(theme)}</ThemeConsumer>;
};

function App() {
  return (
      <MyComponent />
  );
}

const rootElement = document.getElementById("root");
ReactDOM.render(
//      theme  dark
<ThemeProvider theme="light">
  <App />
</ThemeProvider>
, rootElement);

そのため<App />、プロバイダーでラップし、アプリケーションのすべてのコンポーネント-コンシューマーがテーマを使用できるようになりました。次にthisを<App />返し、コンポーネントを作成してアプリケーションのテーマを渡します。そしてすでに引数としてトピックを持っています：<MyComponent />

<ThemeConsumer>{theme => renderMyComponent(theme)}</ThemeConsumer>
