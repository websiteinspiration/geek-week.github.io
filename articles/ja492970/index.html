TeamViewerがパスワードを保存する方法



TeamViewerは、デスクトップへのリモートアクセス用の一般的なプログラムです。したがって、パスワードがどのように格納されるかを見るのは非常に興味深いことです。つまり、パスワードはWindowsレジストリで暗号化されます。暗号化には、AES-128-CBCアルゴリズムと秘密鍵0602000000a400005253413100040000が使用されます。

パスワードを保存するこの方法および関連する権限昇格は、2020年2月7日に脆弱性CVE-2019-18988として正式に登録されました（1965年7月14日までのTeamViewerのすべてのバージョンに適用）。

この脆弱性を明らかにした初心者のセキュリティ専門家は、偶然それに遭遇しました。彼は言う、彼はクライアントのために働いて、バックアップ中に呼び出されたTeamViewerのレジストリキー、気づいたことOptionsPasswordAESなどをSecurityPasswordAES。

その後、彼はこれらのキーとは何か、TeamViewerがパスワードを一般的に保存する方法に興味を持ちました。彼はそれらを仮想マシンの新しいシステムにインポートし、システムでパスワードを収集するBulletPassViewスキャナーを起動しました。スキャナーはTeamViewerのパスワードを平文で与えました。


Scanner BulletPassView

次に、Cheat Engineプログラム（Windowsのハッキングゲーム用）を使用して、スペシャリストがRAMでこのパスワードを検索し、プレーンテキストで再び見つけました。その後、この脆弱性は2年前に CVE-2018-14333として既に修正されていることが判明しました。

次に、TeamViewerクライアントのメモリ内のどこからキーが取得されたかを確認します。サーバーまたはローカルホストからです。ネットワークトラフィックはないことが判明しましたが、メモリにはパスワードが残っています。 IDA Pro、 API Monitor、 procdump、 Fridaを使用したTeamViewerバイナリのリバースエンジニアリングには数週間かかりましたが、途中でいくつかの新しいツールを習得したため、何もできませんでした。

情報の検索中に、かなりの数の人がリソースでUnityゲームのAESキーを見つける方法をすでに疑問に思っていることがわかりました。これは非常に単純なプロセスであり、デバッガーを使用するだけで十分であることがわかりました。 6時間後、彼はAES暗号化を担当するTeamViewerコードスニペットを見つけました。

このコードは、レジスタから数バイトを取り、ServerPasswordAESXOR '所定の値esiとによって取得されたキーでそれらを復号化しますeax。結果はキー0602000000a400005253413100040000であり、IVの値は等しい0100010067244F436E6762F25EA8D704です。

TeamViewer 14では、新しいスクリプトエンジンを使用NT AUTHORITY\SYSTEMして、レジストリ内のTeamViewerパスワードをユーザー権限で読み取ることにより、権限を増やすことができます。

まあ、[SecurityPasswordAES]、[OptionsPasswordAES]、[SecurityPasswordExported]、または[PermanentPassword]を検索パラメーター[filetype：reg]でグーグルすることで、他のいくつかのパスワードを復号化することもできます。

TeamViewerはこの脆弱性を最近のバージョンで閉じました。このプログラムは現在パスワードを暗号化しているため、まだそれを理解していません。
