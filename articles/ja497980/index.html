どのくらいの水が漏れていますか？ハスケルのムーンウォークで問題を解決します

興味深いタスクがネットワークを歩き、Twitterでのインタビューで尋ねられました。
プロファイルのさまざまな高さの壁を見ていると想像してください。雨が降っている、どこかに水が残っている、どこかに高低差のために壁の端を越えて流れている。課題は、壁の間に残っている水量を判別することです。


壁のシーケンスを表すには、リストを使用できます。

type Height = Int

walls :: [Height]
walls = [2,5,1,2,3,4,7,7,6]

各壁の上部にある水の量を調べるには、3つのことを知る必要があります。

  1. 現在の壁の高さ
  2. 一番左の壁の高さ
  3. 一番上の右壁の高さ

下の壁が右か左かを調べ、現在の壁の高さを最高から差し引いて、水の量を求めます。例を詳しく見てみましょう



。壁bの水の量を計算する必要があると想像してください。最高の左の壁の高さ- 、である3。最高の右の壁の高さ- sが、ある2。左の壁が高いため、水が右にこぼれるので、右の壁の高さから数えます。したがって、壁bの水の体積は= 高さ c- 高さ b = 2-1 = 1です。

リストの1つのパスでこの問題を解決することは機能しませんが、2つになります。全体は、一番上の左と右の壁の計算にあります。（UPD：roman_kashitsynは、各壁に蓄積された水の量に関する情報を取得せずに、1つでそれがどのように可能かを示しています）

まず、状態効果を使用して、各壁の最も高い左壁を計算します。

type Volume = Int

--|        
themax :: Height -> State Height Height
themax h = modify (max h) *> get

-- |            
highest_left :: [Height] -> [Height]
highest_left walls = evalState (traverse themax walls) 0



次に、各壁の最も高い右の壁を計算する必要がありますが、この場合は、左側ではなく右側からカウントを開始する必要があります。この場合どうなりますか？リストをめくりますか？この記事のタイトルから推測したように、もっと興味深い方法があります。

逆適用ファンクタ


興味深いタイプ は、transformersパッケージにあります（文字通りの比喩的な意味）。これにより、アプリケーションエフェクトを逆の順序で実行できます。

newtype Backwards f a = Backwards { forwards :: f a }

使い方？BackwardsのApplicativeクラスインスタンスを詳しく見てみましょう。

-- |          liftA2  <**>
instance Applicative t => Applicative (Backwards t) where
        pure = Backwards . pure
	Backwards f <*> Backwards x = Backwards ((&) <$> x <*> f)

＆は$引数への関数の同じ適用ですが、引数の順序が異なります。最初に引数、次に関数です。

f $ x = x & f

遅延のため、最初に適用可能な計算チェーンの2番目のコンポーネントを計算し、次に最初のコンポーネントを計算します。つまり、このタイプの名前です。

同じでは変圧器別の興味深いのタイプがあります- リバース：

newtype Reverse f a = Reverse { getReverse :: f a }

それはで効果起動Traversableの逆使用して後方を。

そして今、まさにこのタイプの元の問題に戻ります。

ムーンウォークをマスターする


「ムーンウォーク」（英語のムーンウォークから）、または「後方にスライド」は、前方に歩くかのように足の動きをシミュレートしながら、ダンサーが後方に移動するダンステクニックです。（ウィキペディア）。
そのようなトリックを上げるために、（左から右に進むように）ステートエフェクトと同じ関数を使用します。これを使用して、最も高い左の壁を計算しました。

-- |        
themax :: Height -> State Height Height
themax h = modify (max h) *> get

-- |    ,    :
highest_right :: [Height] -> [Height]
highest_right walls = getReverse $ evalState (traverse themax $ Reverse walls)

これで、蓄積された水の総量を計算するためのすべてが揃いました。最高の左と右の壁から、最も低い壁を選択し、そこから壁の高さを差し引きます-これは蓄積された水の量になります：

walls = [2,5,1,2,3,4,7,7,6]
let hls = highest_left walls = [2,5,5,5,5,5,7,7,7]
let hrs = highest_right walls = [7,7,7,7,7,7,7,7,6]
sum $ zipWith3 (\l x r -> min l r - x) hls walls hrs



したがって、アプリケーションファンクタは、特定の効果やデータ構造から抽象化するのに役立ちました。クエリのグループ化や並べ替えなど、さらに興味深い用途がいくつかあります。
