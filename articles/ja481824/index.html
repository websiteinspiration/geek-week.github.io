オブジェクトプロパティの不要な変更のソースのクイック検索

こんにちは！今日は、デバッガを使用して、自明ではないJavaScript問題を解決する方法を説明します。

JavaScriptでは、オブジェクトは複合データ型であり、その値は参照によって渡されます。つまり、オブジェクトをパラメータとして関数に渡すとき、またはプロパティを変更できる場所ならどこでもです。リンクを格納する変数式、およびドット演算子と代入演算子で構成される命令を使用します。その後、この変数/パラメーターで機能する、または機能する他の命令は、参照によりプロパティの変更を受け取ります。

多くの場合、この動作はユーザーデータを歪め、エラーにつながり、望ましくありません。

このような不要なプロパティ変更の原因を見つけるには長い時間がかかる場合があります。そのため、プログラムはすでに数十万の命令で構成されている場合があります。

簡単な例を見てみましょう。

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Debug property mutation example</title>
    <script>
        const user = {
            firstName: 'Vasilij',
            middleName: 'Alibabaevich',
            lastName: 'Radner',
            aka: 'Alibaba',
            getFullName() {
                return `${this.lastName} ${this.firstName.slice(0, 1)}. ${this.middleName.slice(0, 1)}.`
            }
        };
    </script>
    <script src="object-property-mutation.js" type="application/javascript"></script>
</head>
</head>
<body>
<script>
    Promise.resolve(user).then(user.getFullName.bind(user)).then(console.log);
</script>
</body>
</html>

コンソールにエラーがあり、フルネームの出力がないため、Webプログラムは機能しません。



コンソールの一番上のメッセージを読みます：

Uncaught (in promise) TypeError: Cannot read property 'slice' of undefined

次のようなエラー：（promiseとして）処理されませんでした：スライスプロパティを未定義から読み取ることができません。

リンクをクリックして、エラーの場所に移動します。

getFullName() {
                return `${this.lastName} ${this.firstName.slice(0, 1)}. ${this.middleName.slice(0, 1)}.`
}

誤った表現が 
this.firstName.slice(0, 1)
4つの演算子で構成されます。

  1. 2点演算子
  2. 1つのカンマ区切り
  3. 1つのグループ化演算子-括弧のペア

指示を読んでみましょう。左の式が最初に評価されます

this.firstName

これは、左側にドット演算子、これの1次式、および右側に識別子firstNameで構成されています。この式の結果は未定義になります。次のドット文を実行するとエラーになります。ドット演算子はオブジェクトタイプでのみ機能するため、undefinedからの実行はエラーになります-undefinedからsliceプロパティを取得できません。

このプロパティが未定義に設定されていることが判明しました...

この問題を解決するには、反対のことを試してみましょう。デバッグツールを使用して例外を停止します。エラーの場所からコールスタックを下に移動して、プロパティを変更した命令に移動してみましょう。

ツールを選択して、例外で停止します




スタックには2つの呼び出ししかないことがわかります。前の通話に戻ります。



firstNameプロパティを変更する明示的な指示がないことがわかります。



このコールスタックでは変更は発生しないと結論付けます。

このJavaScriptの動作に初めて出会ったとき、私は前髪から髪の房を調査して引き裂くのに数時間を費やしました...

例外で一時停止ツールを無効にします。

そこで、ここに新しい計画があります。ゲッターとセッターを使用して、ユーザーオブジェクトのfirstNameプロパティを定義します。

デバッガーステートメントとセミコロンステートメントを使用して、デバッグ命令をセッターに追加します。

const user = {
            _firstName: 'Vasilij',
            set firstName(value) {
                debugger;
                this._firstName = value;
            },
            get firstName() {return this._firstName},
            middleName: 'Alibabaevich',
            lastName: 'Radner',
            aka: 'Alibaba',
            getFullName() {
                return `${this.lastName} ${this.firstName.slice(0, 1)}. ${this.middleName.slice(0, 1)}.`;
            }
        };

コールスタックをさらに進むと、firstNameプロパティを変更する命令が見つかります。

新しい値がオブジェクトに書き込まれる前に、デバッガーがセッターで停止しました。



パラメータ値の値が未定義であることがわかります。

これで、呼び出しスタックを使用して、前の呼び出しに簡単に移動できます。



iiiii、勝利ああああああ。例外で停止

するデバッグツールを使用してこの問題を解決するさらに簡単な方法があります。

これが新しい計画です。ユーザーオブジェクトをオブジェクトではなく、例外による停止ツールを使用して、誤った命令に簡単にアクセスできます。

undefinedからプロパティを取得しようとするとエラーが発生することがわかっているためです。

例外停止ツールをオンにし、ユーザー変数を未定義に設定します。

    const user = undefined;

再びfirstNameプロパティが歪んだ場所に立ち寄りました。
