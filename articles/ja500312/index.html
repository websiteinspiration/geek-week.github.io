JavaScriptメモ取りアプリケーションの作成



こんにちは！

今日、名前が示すように、メモを作成して保存するための簡単なアプリケーションを作成します。

アプリケーションの機能は次のとおりです。

  1. メモを作成します。
  2. メモの保管。
  3. メモを削除します。
  4. タスクについてマークします。
  5. タスクが完了した日付に関する情報。
  6. タスクを完了するためのリマインダー。

アプリケーションはJavaScriptで記述されます。

ノートはインデックス付きデータベース（IndexedDB）に保存されます。このライブラリは、IndexedDBでの作業を容易にするために使用されます。このライブラリの開発者によると、「IndexedDBと同じですが、約束があります」。

IndexedDBの基本を理解していることを前提としています。そうでない場合は、続行する前にこの記事を読むことをお勧めします。

メモの保存などの問題を解決するには、LocalStorageで十分だと理解しています。しかし、私はIndexedDBのいくつかの機能を探求したいと思いました。したがって、後者を支持する選択は、認識論的考察のみから行われた。最後に、LocalStorageを使用してデータストレージを実装する同様のアプリケーションへのリンクがあります。

じゃ、行こう。

  2. 後で判明したように、「日付」タイプの「入力」タグを選択することは、最善の解決策ではありませんでした。以下について。
  3. 1つのアプリケーションでは、通知（通知）は通知APIを使用して実装されます。ただし、通知を表示する権限をユーザーに要求し、それをオフにする機能を追加するのは私には奇妙に思えました。これは、最初にメモ（タスク）のアプリケーションについて話すとき、リマインダーが暗黙に示され、次に、それらが迷惑にならないように実装できるためです。繰り返し登場するユーザー、つまり 控えめに。
  4. 最初は、アプリケーションは日付だけでなくリマインダーの時間も示す機能を提供していました。その後、日付は十分だと判断しました。ただし、必要に応じて簡単に追加できます。

リマインダーの日付を格納するオブジェクトの配列には、メモを識別するための「id」と日付を比較するための「date」の2つのフィールドがあります。「日付」フィールドにリマインダーの日付の値を書き込むと、inputDate.valueが「yyyy-mm-dd」の形式でデータを返すため、この値を変換する必要があります。このデータを、以前の形式のデータと比較します。「Dd.mm.yyyy」。したがって、「replace」メソッドと正規表現を使用します。グループ化を使用して、ブロックを反転し、ハイフンをドットに置き換えます。より用途の広い、またはエレガントな解決策があるかもしれません。

リマインダーチェック機能は、現在の日付とユーザーが入力したリマインダーの日付を比較します。

const checkDeadline = async dates => {
    //      ".."
    let today = `${new Date().toLocaleDateString()}`

    //   
    dates.forEach(date => {
        //         
        if (date.date === today) {
            //      "?"  "!"
            document.querySelector(`div[data-id="${date.id}"] .info`).textContent = '!'
        }
    })
}
