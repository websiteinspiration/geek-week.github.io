この同僚は私の仕事仲間であり、関数の宣言でパラメーターが正確に何を意味するのかをすぐには理解していませんでした（そのようなアナウンスが必要になる可能性がある場所に興味がある人のために：暗号で）。

そして、ワークショップの同僚からの起こりうる問題を予測して、私はこの記事をチートシート、つまり2つの質問への回答として入れて作成することにしました。

  1. ここにそのような広告を書く方法は？
  2. そして、それらを正しく読む方法は？

「*」


アスタリスクが1つのケースが最も一般的ですが、ここでは誤解も考えられます。

  • 方法：

    • そう： 

      const byte p;
    • とか、ぐらい： 

      byte const p;

  • または、エントリ「*」の場合：

    • そう： 

      byte *const p;
    • とか、ぐらい： 

      const byte *p;

厳密に言えば、コンパイラーの観点からすると、アスタリスクの存在と式内でのその位置によって、次のような疑問が生じます。エントリの最終的な意味は異なります。上記の場合のconst指定子の存在は、データ型との相対的な位置を考慮に入れても何の違いもありませんが、ポインターが表示されると状況が劇的に変化します。

void p() {
    //   .
    byte * a = new byte{1};
    a++; // .
    a[0]++; // .

    //    .
    const byte * b = new byte{1};
    b++; // .
    //b[0]++; // .

    //    .
    byte *const c = new byte{1};
    //c++; // .
    c[0]++; // .

    //     .
    const byte *const d = new byte{1};
    //d++; // .
    //d[0]++; // .
}

2番目のポインターが表示されると、これはすべてより興味深いように見えます（ここでは、書き込みの効率的な読み取りルールが既にトレースされ始めています）。

ご覧のように、ダブルポインタは最大8つの異なる方法で表すことができ、それぞれがターゲットデータを使用する特別な方法を定義します。

このような式を読み取るための規則は次のとおりです。

  • 式で「=」記号を探し、式を右から左に読み取ります。
  • 変数の名前をスキップします。
  • 以降の会議では、通常のポインタを意味する「*」、または「* const」-定数ポインタのいずれか。
  • したがって、データ型（バイト）が検出されるまで読み取ります。
  • データ型の左側の最後の単語はconstにすることができます。これは、この構造全体が変更できないデータを参照していることを意味します。constがそうでない場合、それは可能です。

この形式の書き込みと読み取りにより、最も洗練された式でも読みやすく、理解しやすくなります）
