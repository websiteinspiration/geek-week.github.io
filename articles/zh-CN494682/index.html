Shortwave-适用于GNU / Linux的现代开源Internet广播播放器




Shortwave是一个有趣的开源无线电播放器，它提供了一个不错的用户界面和对Internet电台的强大监听功能。它使用社区生成的在线广播数据库。Shortwave实际上是流行的GNU / Linux Gradio无线电应用程序的后继产品。它的开发人员Felix加入了GNOME，并中止了Gradio，以Rust编程语言从零开始创建Shortwave。如果您使用Gradio作为首选的网络广播播放器，则还可以将库导入Shortwave。Shortwave最近发布了其第一个稳定版本，并且此后似乎继续发布更新。

PostgreSQL Open Source




Linux.com发布了一篇文章，其中简要概述了PostgreSQL的基本原理。流行和民主是最能描述PostgreSQL开源数据库管理系统的两个词。而且，就像Linux一样，它是“最好的开源”的一个很好的例子。就其流行程度而言，Postgres已经存在了30多年，如今它似乎在本地工作或作为来自全球数十家供应商的托管服务。而且，从民主的角度来看，该软件项目由PostgreSQL全球开发小组管理，该小组是来自世界各地的开发人员和志愿者，有500多名参与者。没有一家公司对软件的设计和开发承担任何责任，支配或控制任何软件，因此，没有依赖任何特定个人或组织的危险。没有人可以“捕获” Postgres。普遍与民主共同创造自由。

关于在Linux上启用exFAT支持的争论




Ars Technica理解了开放源代码的支持者与反对者之间的下一个争执，并写道，当软件巨头微软在8月宣布支持将exFAT文件系统直接包含在Linux内核中时，这在媒体上并没有引起太大关注。但是最近，Paragon软件文件系统供应商注意到，本月，由Microsoft批准，很大程度上由三星创建的exFAT版本合并到了VFS存储库中，以用于下一个Linux内核版本。这并没有令Paragon满意，导致发表了一篇文章，批评exFAT解决该问题的开源解决方案。

根据OpenNET，Cloudflare开发人员讨论了如何在Linux内核中优化磁盘加密性能。结果，为dm-crypt子系统和Crypto API准备了补丁程序，从而使综合测试的读写带宽增加了一倍以上，并且延迟减少了一半。在真实设备上进行测试时，来自加密的开销减少到几乎与不使用数据加密的磁盘一起使用时所观察到的水平。

Apache 21




非营利性Apache软件基金会正在庆祝成立21周年。最初，创建该组织的目的是为Apache http服务器的开发人员提供法律和财务支持，但后来转变为一个中立且独立于各个制造商的平台，以开发使用Apache许可证，一般开发规则，精英原则和统一沟通文化的各种开放项目。 。同时，Apache httpd HTTP服务器，拥有25年历史的Apache OpenOffice Office套件和拥有20年历史的Apache Jakarta，Subversion和Tomcat一起庆祝了25年。由Apache开发的项目数量已超过350（其中45个在孵化器中），涉及的领域包括机器学习，处理大量数据，构建管理，云系统，内容管理，DevOps，IoT，移动应用程序开发，服务器系统和Web框架。超过7600名提交者对开发进行了监督。在过去的21年中，支持该基金会的参与者数量已从21个增加到765个。使用COCOMO 2成本估算模型计算时，从零开始开发包含2亿行代码的300个Apache项目的总成本估计为200亿美元。
