Raspberry Pi 4ヒートテスト

Raspberry Pi 4はより低温になりました！過去4か月間、ファームウェアの更新により、スタンバイ負荷が1/2 W、全負荷でほぼ1 W削減されました。Gareth Halfacryは、MagPiマガジンのボードの熱テストを実施しました。




Raspberry Pi 4モデルB


Raspberry Pi 4がリリースされ、より強力なCPUとGPU、より多くのメモリ、ギガビットイーサネット、USB 3.0のサポートなど、ユーザーをアップグレードするための多くの新機能が提供されました。プロセッサー能力の増加は消費電力の増加を伴い、Raspberry Pi 4はファミリーで最もエネルギーを消費するメンバーです。

新しいRaspberry Piモデルの発売は、物語の始まりにすぎません。ボードの開発は進行中であり、ソフトウェアとファームウェアのアップデートにより、ボードが出荷されてからかなり後に、ボードの各バージョンが改善されます。

Raspberry Pi 4アップデート


Raspberry Pi 4も例外ではありません。そのリリース以来、消費電力を削減するいくつかのアップデートがすでにリリースされています。また、これらのアップデートは、リリース日に購入したか、今日のみ注文したかに関係なく、Raspberry Pi 4に適しています。

この記事では、人工のソフトウェアロード（実際のタスクではありません）を使用して、システムのオンチップ（SoC）を可能な限り最高の温度にできるだけ早くウォームアップすることにより、以降の各更新でRaspberry Pi 4がどのように改善されたかを説明します。

そして、ここに簡単なファームウェアアップデートが達成できるいくつかの奇跡があります。

Raspberry Pi 4ファームウェアアップデートのテスト方法


各ファームウェアが加熱にどの程度対応できるかを確認するために、最悪のケースをシミュレートする、人工的な要求の厳しい高負荷を開発しました。 stress-ngユーティリティは常に4つすべてのCPUコアをロードします。その間、glxgearsユーティリティはGPUに作業をロードします。ユーティリティは次のコマンドでインストールされます：

sudo apt install stress-ng mesa-utils

次のコマンドでCPUをロードできます：

stress-ng --cpu 0 --cpu-method fft

デフォルト設定では、コマンドは終日動作します。キャンセルするには、CTRL + Cを押します。

次のコマンドでGPUを

ロードできます：glxgears –fullscreen

プログラムは、画面全体を埋める歯車の3Dアニメーションを表示します。閉じるには、ALT + F4を押します。

チームの作業の詳細については、こちらをご覧ください：

man stress-ng
man glxgears

テスト中、両方のロードプログラムが10分間一緒に動作しました。その後、Raspberry Pi 4は5分間冷却されました。

サーマルイメージャーからの写真は、スタンバイモードで撮影された後、1つのstress-ngコマンドでロードしてから60秒後に撮影されました。

ベースラインパフォーマンス：Raspberry Pi 3B +


Raspberry Pi 3B +デバイスは広く認識されており、これらのインジケーターに従ってそれを回避するには、試す必要がありました。



Raspberry Pi 4のリリース前は、Raspberry Pi 3 Model B +はシングルボードコンピューターに必須でした。このモデルは、以前のRaspberry Pi 3モデルBの開発中に達成されたすべての利点に加えて、更新されたハードウェアを受け取り、今でも人気のあるデバイスです。

Raspberry Pi 4をテストする前に、動作を確認してみましょう。

エネルギー消費




Raspberry Pi 3B +は、前作に比べて効率的なプロセッサと改善された電源回路図により、消費電力を削減できます。スタンバイモードではわずか1.91ワットで、人工的な負荷では5.77ワットに増加します。

エネルギー消費




Raspberry Pi 4ポートに何も接続しなくても、VLIファームウェアに目に見える改善があります。スタンバイ電力消費は2.62ワットに低下し、負荷がかかると最大7.01ワットに達します。

Raspberry Pi 4 VLI、SDRAMファームウェア




VLI電源管理と同時に使用するように設計された次のファームウェアは、Raspberry Pi 4メモリの動作を変更します-LPDDR4 SDRAM。パフォーマンスに影響を与えることなく、スタンバイモードと負荷がかかった状態の両方で消費電力をさらに削減できます。

エネルギー消費




VLIの更新と同様に、SDRAMの更新により、スタンバイモードと負荷がかかった状態の両方で、消費電力が大幅に減少します。これで、Raspberry Pi 4はスタンバイ状態で2.47ワット、負荷状態で6.79ワットを消費します。これは、元の7.28ワットよりも大幅に改善されています。

ビートをスキップ

Raspberry Pi 4 VLI、SDRAM、クロッキング、およびロードステップファームウェア

Raspberry Pi 4の適切な向きで冷却を改善する



ファームウェアの更新は優れた結果をもたらしますが、Raspberry Pi 4を垂直に回転させるとどうなりますか？

最新のファームウェアは消費電力と発熱を大幅に削減できますが、さらに良い結果を得るには、ボードの向きを変えるという別のトリックがあります。このテストでは、最新のファームウェアを備えたRaspberry Pi 4を垂直にインストールしたため、GPIOインターフェイスが下に、HDMIポートが上にあります。

ビートをスキップ




Raspberry Pi 4を単純に垂直に回転させると、すぐに結果が得られます。スタンバイモードのSoCは、以前よりも2℃低く、ウォームウォームがゆっくりと行われます。負荷がかかっている状態では、ボードはクロックスキップモードに移行せずに実行時間が長くなり、大幅に高速を維持します。

ここではいくつかの要因が作用します。垂直方向は対流を改善し、周囲の空気がより早く熱を放散できるようにし、断熱テーブルからボードの背面を持ち上げると、伝熱面が大幅に増加します。

小節をスキップする時間




この図は、負荷がかかった状態でメジャーのスキップに切り替えるのにかかった時間を示しています。Raspberry Pi 3B +は最悪の結果を示し、わずか19秒でスキップメジャーに切り替わりました。Raspberry Pi 4の後続の各ファームウェアアップデートは、この点をさらに押し進めました。ただし、最も深刻な改善は、ボードの向きを変更することで実現できます。

実際の負荷で確認


合成負荷をそのままにして、自問してみましょう-ボードは実際の負荷にどのように対処しますか？

前述のことから、Raspberry Pi 3B +とRaspberry Pi 4の間で速度に実際の違いがあると結論することは困難です。合成測定は、実際のタスクではめったに見られないエネルギー需要の計算を実行し、さらに、無限に繰り返されます。

Linuxのコンパイル


このテストでは、Raspberry Pi 3B +およびRaspberry Pi 4に、ソースからLinuxカーネルをコンパイルするタスクが与えられます。これは、現実の世界で見られるCPU負荷の良い例であり、前のテストの総合的な負荷よりもはるかに現実的なタスクです。

カーネルのコンパイル：Raspberry Pi 3B +


Raspberry Pi 3B +は非常に早い段階でクロックスキップモードに切り替わり、コンパイラーがCPUの負荷からドライブの負荷に切り替わる短い冷却期間まで1.2 GHzのままです。これにより、ボードは1.4 GHzにすばやく切り替わります。コンパイルは5097秒で終了しました-1時間24分57秒。

カーネルのコンパイル：Raspberry Pi 4モデルB


合成タスクと実際のタスクの違いははっきりとわかります。RaspberryPi 4は、クロックサイクルをスキップするほどの高温にはならず、1.5 GHzで動作します-Raspberry Pi 3 B +の場合を除きます。 、コンパイラの動作の変更により、ボードがスタンバイモードの速度に低下するまでの短い期間。コンパイルは2660秒で終了しました-44分20秒。
