StegoPy-PythonのLSBステガノグラフィーツール

序文


最近、私のフォーラムの人々が、画像、mp3ファイル、ビデオのさまざまなテキストのステガノグラフィーのトピックに興味を持っていることに気づきました。そして、私はイメージ内のテキストのステガノグラフィー用の独自のスクリプトを書くというアイデアを持っていました。ステガノグラフィー用の同様のツールを書いたことは一度もなかったと私は直前に言わなければなりません。

LSBステガノグラフィーに慣れる前に、私は他の方法を考えました。むしろ、自分の方法を考え出しました。しかし、私は私自身の何かを思い付くために強さを持っていなかったと私はそれをGoogleに決めた：

LSBピクセルのRGB色のいずれかの下位ビットは、エンコードされたテキストのビットに変更されているステガノグラフィ方法です。

次に、画像と暗号化を扱うための枕ライブラリについて知りました。私の場合、2番目はDES暗号化を作成するために必要でした。DES暗号化は、その後、画像にビットコード化されます。

コーディング


私は自分の頭脳をGitHubに投稿したので、StegoPyでさらに作業するには、GitHubでプロジェクトを複製する必要があります。

> git clone https://github.com/eBind/StegoPy
> cd StegoPy
> pip3 install -r requirements.txt
> python3 stegopy.py



ただし、インストールに関してはすべて通常どおりです。次に、起動コマンドの構文を見てみましょう。

> python3 stegopy.py -e in.jpg data.txt

このコマンドを使用して、data.txtファイルのテキストをin.jpg画像にエンコードしますが、エンコードする
前に、1から4までのバランスを要求されます。これは、プログラムで最も興味深いものの1つにすぎません。

バランスは、ステガノグラフィに関係する下位ビットの数です。範囲は1〜4です。

したがって、バランスが大きいほど、次のようになります。

  1. コーディングに関与するピクセルが少なくなる
  2. カラーチャネルの顕著な変化

このことから、バランスが小さいほどステガノグラフィの信頼性が高くなり、人間の目には見えなくなると結論付けることができます。

もう1つの重要な要素は、青のチャンネルのみが変化することです。その色合いは、私たちの目には最も目立ちません。

バランスを決定した後、スクリプトは2つの追加ファイルを作成します。

  • out.png-エンコードされたテキストを含む画像
  • key.dat-復号化に必要なキーを持つファイル

> python3 ./stegopy.py -d out.png
