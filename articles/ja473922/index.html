この記事では、会議で議論されるトピックと、データセンターにサービスを提供するネットワーク機器が動作する速度の両方について説明します。さらに、Selectelネットワークインフラストラクチャをさらに開発するための現在のタスクについても説明します。

2020年に新しいジュニパートリオチップセットをポンプするTbit / sの量、またはネットワーク機器ソフトウェアのメジャーバージョンにアップグレードするときに発生する可能性のある「レーキ」について知りたいですか？その後、キャットへようこそ！

「Selectelは、ITのさまざまな分野における能力の中心です。サーバー部分、ソフトウェア部分、クラウドです。しかし、データセンターの最も忘れられない部分の1つは、一般に、これらすべてのサービスの先頭にあるネットワークです...今日はネットワークについて話し、ここにさまざまなネットワーク機器の評判の良いベンダーを集めました... "

エクストリームネットワーク


エクストリームネットワークスに最初に報告書を提出しました。エクストリームネットワークスは過去5年間、さまざまなネットワークビジネス企業の買収および買収を通じて積極的に発展してきました。買収の主な目的は、自社の製品における高度なネットワーク技術の開発と統合ほどの事業の買収ではありません。最近の画期的な買収には、Zebraからのワイヤレスビジネス、AvayaとBrocadeからのネットワークビジネスの購入が含まれます。


エクストリームネットワークスの代表であるPavel Denisovのレポートのトピックは、ボーダールーター、いわゆるボーダールーターに関するものでした。これらは、トランクと周辺ネットワークを接続するデバイスであり、次のパラメーターによって特徴付けられます。

  • ハイパフォーマンス;
  • 大量のバッファメモリ。
  • セキュリティの向上;
  • 情報収集のための追加サービス。

エクストリームネットワークスは、SLXラインでエッジルーターとして2つのモデルを提供しています。

  1. 主力のExtreme Networks SLX 9640モデル。24個の10 GbEポートと12個の100/40 GbEポートを搭載。このデバイスは、Optiscale独自の圧縮テクノロジーにより、最大570万のIPv4ルートをサポートします。
  2. 48個の10 GbEポートと6個の100/40 GbEポートを備えたExtreme Networks SLX 9540の「弟」。その中のルートの数は少ないですが（150万）、この数は境界ルーターの役割には十分です。

興味深い機能は、ルーター内でゲスト仮想マシンを実行する機能です。最近のすべてのルーターが実際には仮想化サーバーにすぎず、その中にトラフィックを処理するためのASICチップをプログラミングする仮想マシンが存在することは明らかです。

エクストリームネットワークスのエンジニアはさらに進んで、専用の10 GbEインターフェースを備えた個別のゲスト仮想マシンをアップグレードできるようにしました。これにより、トラフィックを簡単に分析できるだけでなく、多くのプロセスを自動化して、ネットワーク機器を企業のエコシステムに統合することができます。

Selectelネットワークインフラストラクチャ開発


私たちはまた、脇に立つのではなく、インフラストラクチャの開発について話しました。以前は、データセンターのSelectelネットワークは、モスクワとサンクトペテルブルクの2つの大きなルーティングドメインで構成されていました。各ドメインは個別のルーターによって処理されました。問題が発生した場合、これは地域全体に影響を与える可能性があります。数多くの重要なサービスの成長と出現に伴い、ネットワーク構築への取り組みを見直す必要があると考えました。

私たちはネットワークに取り組んできた数年間、一連のタスクを解決してきました。その1つは専用サーバー（インターネットとローカルエリアネットワーク）です。専用サーバーのネットワークは最もシンプルで、古典的なアーキテクチャーに従って構築されています。ルートルーター-アグリゲーションレベルスイッチ-アクセススイッチ。ただし、重要な機能があります。1つのクライアントのサーバーを簡単に別のラックに配置できます。その結果、多くのラックに「ストレッチ」された多数のVLANが取得されます。

もう1つのタスクは、クラウド内の仮想サーバーのネットワーク操作を確実にすることです。Spine Leafトポロジーを使用します。あるサーバーから別のサーバーへの最小パケット通過時間を保証するため。IPストレージコンセプトのアクティブな使用には、独自の特性があり、特に高速リンクが多数存在することが重要です。同時に、これらのリンクの一部が機能しなくなったときに、ネットワークは状況を適切に認識する必要があります。

さて、おそらく最も重要なタスクは、データセンター間のネットワークです（Data Center Interconnect）ここでは、独自のルールと冗長性を持つ別個のネットワークを使用します。このような多数のネットワークの使用は、すべてのサービスへのアクセスを取得するだけでなく、それらを相互に接続する必要がある顧客のニーズによるものです。たとえば、1人の顧客のクラウドサーバーが専用サーバーのネットワークにアクセスし、物理的には異なるデータセンターに配置されている場合です。


現在、ネットワーク開発には次の優先分野があります。

  • ドメイン障害を減らします。
  • コンポーネントの冗長性。
  • スケーリング。

操作中に、かなりの重要な問題に遭遇することがよくあります。たとえば、一部のベンダーは、スタックをバックアップツールとしてではなく、単一のデバイスのポート数を拡張するものと見なしています。このようなスタックのソフトウェアを更新するときは、スタックのコンポーネントを完全に再起動する必要がありますが、これは私たちには簡単にはできません。

他のベンダーは、スタックをファブリックの観点から見ています。つまり、理論的には、スイッチは互いに独立していて、ソフトウェアを個別に更新できます。これは、マイナーバージョンへの更新が発生した場合にのみ機能します。更新を完了するために機器を再起動する必要があるにもかかわらず、サービスの中断は発生しません。ただし、ソフトウェアのメジャーバージョンへの移行が必要になるとすぐに、問題が発生する可能性があります。アップグレードプロセス中にスタックがスムーズに機能することを保証するものはありません。

この問題の解決策は、いわゆる更新グループの作成です。スイッチをバッチで更新します。しかし、問題は、そのようなスイッチのグループを更新することにより、残りのグループが更新されるまで、スイッチを制御できないことです。これがアップデート後に正しく機能するかどうか-質問は未解決のままです。たとえば、16台のスイッチから本格的なテストベンチを組み立てるには、それぞれ1万ドル以上かかるため、非常に問題があります。

ジュニパーネットワークス


その後、ジュニパーネットワークスが発言しました。ジュニパーネットワークスは1998年以来、サービスプロバイダー向けの革新的なソリューションをリリースしてきましたが、過去10年間、企業顧客とデータセンターのセグメントにより多くの注意が向けられてきました。デバイスとマイクロ回路の開発に加えて、同社の優先事項は独自のJunOSオペレーティングシステムの開発です。これは、製造された機器の全範囲で同じになりました。

まず、機能の均一性や各種機器の制御に注目。管理計画は1セットのコードを使用するため、機能を追加すると、ハードウェアデバイスから仮想ソリューションまで、すべてのプラットフォームにすぐに表示されます。

JunOSは、ルーチン操作を自動化するために、プログラミング言語からさまざまなAPIまで、幅広いツールをサポートしています。Junos Continuityコンセプトにより、メジャーソフトウェアバージョンにアップグレードする必要なく、新しいモジュールをデバイスに導入できます。

Juniper NetworkのシステムエンジニアであるOleg Prokofievは、ルーターのMXシリーズと、ハードウェアソリューションのアーキテクチャで積極的に使用されているJuniper Trioと呼ばれる独自のチップセットを作成する分野の開発について語りました。2018年の場合、これらのチップは1つのチップで0.5 Tbit /秒を超えるトラフィックを処理できます。2020年には、3倍の速度で1.5 Tbit / sに到達できる新しいバージョンをリリースする予定です。 シングルチップ上。

帯域内テレメトリ技術


AristaのシステムエンジニアであるAlexander Bespalov によるプレゼンテーションは、IETFがAristaと共に開発した新しいInband Telemetry / Inband Flow Analyzer（INT / IFA）テクノロジーに特化したものでした。現時点では、技術はまだドラフト段階にあります（現在、2番目のバージョンはすでに存在しています）。

このテクノロジーの意味は、各トランジットデバイスが十分な量のメタデータを含むトランジットパケットに独自のヘッダーを追加することです。

  • デバイス識別子;
  • 着信ポート
  • 発信ポート;
  • 発信キュー;
  • 発信ポートの処分;
  • パケット受信タイムスタンプ。
  • パケット転送タイムスタンプ。

INT / IFAをサポートするデバイスは、独自のINT / IFAドメイン内で動作します。メタデータの追加を開始するデバイスは、イニシエーターノードと呼ばれます。ドメインのトランジットノードはトランジットノードとして指定され、ヘッダーを更新して、情報の一部をトランジットパケットに追加します。パケットからメタデータが削除されるデバイスは、終了ノードです。

Spine Leafトポロジーを使用する場合、Spineノードはトランジットノードとして機能し、Leafノードはそれぞれイニシエーターとターミネーターになります。 SpineとLeafがトランジットの役割を果たす別のオプションがあり、ターミナルサーバーはメタデータの収集プロセスを開始し、メタデータを受け入れてさらに処理することができます。

ヘッダーを追加するプロセスは非常に簡単です。イニシエーター側で、IFA / INTヘッダーが1回追加されます。これは、パケットがネットワークを通過するときに変更されません。このヘッダーの後に、ドメイン内の各中継ノードを通過するときに、収集されたメタデータが追加されます。エンドノードは収集したデータを分析サーバーにエクスポートし、ヘッダーとすべてのメタデータをIFA / INTパッケージから完全に削除します。これにより、パケットがネットワークをどのように通過するかを最も正確に理解し、ネットワークインフラストラクチャの操作性に影響を与えるずっと前に問題を特定することができます。

Huawei


Huawei Key Account ManagerであるAlexander Dorofeevによるプレゼンテーションは、高速データ転送のための新製品とソリューションで非常に満ちていました。サーバー間のトラフィック（いわゆるEast-Westトラフィック）の絶え間ない増加により、より高速な標準への移行が必要になり始めました。特に、Selectelはすでに100 GbEリンクをサポートする機器の使用を開始しています。

Broadcomなどのサードパーティ製品を徐々に放棄し、スイッチ用の独自のネットワークプロセッサを積極的に開発しているHuaweiの半導体技術の開発には、特別な注意を払う必要があります。


イベント中に、ファーウェイは最新の製品の1つをゲストに紹介しました-Huawei Solar 5.0チップを搭載した2 Tbit /秒の帯域幅を備えたF1A-14H24Q​​高密度ルーター。ルータポートの半分は最大1/10/25 GbEのトランシーバをサポートし、残りの半分は10/40/100 GbEをサポートします。

結論の代わりに


レポートのセッションの後、ゲストはベンダーの代表と話し始めました。最初に、サポートされる製品の数に関して非常に重要な質問が多くの人に出されました。 EOL（End-of-Life）期間の終了後も、機器自体が長期間適切に機能し続けることは秘密ではありません。

お客様の観点からすると、バグや脆弱性が発見されるとネットワークに深刻な損傷を与える可能性があり、ベンダーが古いデバイスのソフトウェア部分をサポートしていない場合、それらを使用することは単に安全ではなくなります。したがって、EOLA（サポート終了のお知らせ）の発生後に何を当てにすべきかを明確に理解する必要があります。

一方、ベンダーの立場から見ると、同じ状況が発生します。一方、機器は正常に動作する可能性がありますが、陳腐化しているため、顧客は何らかの方法で更新する必要があるため、EOLの条件は非常に論理的で正しいように見えます。

さらに、ゲストはデバイスでどのような仮想化システムが使用されているかに興味を持っていました。ネットワークデバイス用のソフトウェアの品質に関する意見を表明する人もいます。


議論は非常に有用で有益であることがわかりました。ベンダーの代表者はフィードバックを得ることができ、ゲストはネットワーク機器とテクノロジーの使用に関するより明確な情報を受け取りました。
