Windows 10開発者向けのHyper-V

Hyper-Vはサーバー仮想化テクノロジーとしてよく知られています。ただし、Windows 8以降では、クライアントオペレーティングシステムでも使用できます。 Windows 10では、Hyper-Vを開発者やITプロフェッショナル向けの優れたソリューションにすることで、パフォーマンスを大幅に向上させました。
 
Microsoft Hyper-V（コードネームはViridian）は、ネイティブ（タイプ1）ハイパーバイザーであり、VMware Workstation、VirtualBox、およびその他のタイプ2ハイパーバイザーとは異なり、ハードウェア上で直接実行されます。これはWindows Server 2008で最初にリリースされ、x86-64システムで仮想マシンを実行できるようにします。

Hyper-Vを使用すると、開発者は優れたパフォーマンスでWindows 10開発用の仮想マシンを迅速にオーバークロックできますが、Androidエミュレーター、Windows for Linux 2（WSL2）サブシステム、コンテナーなどのサーバーテクノロジーとして他のいくつかの開発機能でも使用されますDocker。この記事では、Windows 10のHyper-Vが開発者にどのように役立つかを簡単に説明します。

仮想マシンのコレクションをすばやく作成




まず、仮想マシンを作成するためのコア機能の1つから始めましょう。これで、ISOファイルを使用してインストールしてHyper-V仮想マシンを作成できるだけではありません。Hyper-V Quick Create VM Galleryを使用して、新しいWindows 10開発環境またはUbuntu仮想マシンをすばやく作成できます。このツールは、Windows 10とVisual StudioまたはUbuntu 18.04または19.04開発環境で事前構成されたHyper-V仮想マシンをロードします。必要に応じて、Quick Create仮想マシンの独自のイメージを作成し、他の開発者と共有することもできます。
 

Linux 2（WSL 2）用のWindowsサブシステム



 
Linux用のWindowsサブシステムにより、MicrosoftはLinux環境をWindows 10デスクトップに移行し、GNU / Linux環境（ほとんどのツール、ユーティリティ、コマンドラインアプリケーションを含む）を変更なしで直接Windowsで実行できるようになりました。本格的な仮想マシンのコストはかかりません。Windows 10 Insider Previewの最新バージョンでは、MicrosoftはWSL 2プレビューをリリースしました。WSL2はHyper-V仮想化テクノロジを使用して、バックエンドでWSLを分離および仮想化します。これにより、セキュリティ上の利点だけでなく、パフォーマンスも大幅に向上します。Linux 2のWindowsサブシステムについて詳しくは、こちらをご覧ください。
 

仮想マシンとデバイスを共有するための高度なセッションモード



 
過去の困難な部分の1つは、仮想マシンとホストマシン間の相互作用でした。拡張セッションモードが追加されたことで、Microsoftはそれらの間のファイルのコピーと貼り付けを簡素化し、デバイスを共有できるようになりました。含む：

  • サイズ変更可能な高DPI仮想マシンを作成
  • 仮想マシンの統合を改善
  • デバイスの共有を許可します

Windows 10 Hyper-Vでは高度なセッションモードがデフォルトで有効になっています。WindowsServerでは、Hyper-V設定でこの機能を有効にする必要があります。独自のLinux仮想マシンを作成する場合は、必ず最新の仮想化ドライバーをインストールしてください。
 

チェックポイント


仮想化の大きな利点は、仮想マシンの状態を簡単に保存できることです。これにより、特定の時点で前後に移動できます。Hyper-Vでは、この機能は仮想マシンチェックポイントと呼ばれ、以前は仮想マシンスナップショットと呼ばれていました。

Hyper-Vはブレークポイントのタイプを認識しています。

  • 標準チェックポイント：チェックポイントが開始されたときの仮想マシンの状態と仮想マシンのメモリのスナップショットを取得します。
  • 本番チェックポイント：Linux仮想マシンでボリュームシャドウコピーサービスまたはFreeze File Systemを使用して、データ整合性のある仮想マシンバックアップを作成します。仮想マシンのメモリステータスのスナップショットは作成されません。


 
チェックポイントのタイプは、仮想マシンで設定できます。どちらのタイプのチェックポイントにも、長所と短所があります。Hyper-Vの最近のバージョンでは、運用チェックポイントがデフォルトで選択されています。ただし、Hyper-VまたはPowerShellマネージャーの設定を使用して、これを簡単に変更できます。Windows 10を搭載したクライアントコンピューターでは、仮想マシンのメモリ状態も保持されるため、標準のブレークポイントを使用することを好みますが、運用サーバーシステムでは、ブレークポイントを使用することを強くお勧めします。

Windows 10でWindowsおよびLinuxコンテナーを実行する


コンテナーは、現在最も人気のあるテクノロジーの1つであり、Docker Desktop for Windowsでは、コンテナーをWindows 10を搭載したコンピューターで実行できます。デフォルトでは、WindowsはHyper-Vテクノロジーを使用して、コンテナーとホストオペレーティングシステム、いわゆるハイパーコンテナーの間に追加の保護を作成します-V。この機能により、Windows 10で完全なLinux仮想マシンを実行する必要なく、WindowsとLinuxコンテナーをWindowsで並列に実行することもできます。
 

PowerShellダイレクトとHVC


Windows 10を実行している仮想マシンと対話する場合は、Hyper-Vマネージャーとコンソールを使用して、オペレーティングシステムと直接対話できます。ただし、コマンドラインを使用して仮想マシンを管理およびアクセスできる他の2つのオプションもあります。PowerShell Directでは、VMバスを使用して仮想マシンのPowerShellリモート処理セッションを作成できるため、ネットワークは必要ありません。 Linux仮想マシンとHVCツールの場合も同じで、仮想マシンへのSSH接続を直接作成できます。どちらのオプションでも、仮想マシンとの間でファイルをコピーできます。


 
これは、自動化を構成していて、仮想マシンでいくつかのコマンドを実行する必要がある場合に非常に便利です。
 

Windowsサンドボックス



 
Windowsのサンドボックスは、バージョン1903年にリリースされたWindows 10の新機能であるWindowsのサンドボックスは、 Windowsの10サンドボックスを提供するために、Hyper-Vの技術を使用しています。サンドボックスを使用すると、信頼できないソフトウェアを実行できる分離された一時デスクトップ環境を起動できます。サンドボックスは、デモ、開発、テスト、トラブルシューティング、またはマルウェアの操作に最適です。サンドボックスを閉じると、すべてのソフトウェアとそのすべてのファイルおよびステータスが完全に削除されます。これらはWindows 10を備えた仮想マシンであり、その利点はWindows 10に組み込まれているため、既存のOSを使用することで、セキュリティを失うことなく、起動が速く、効率がよく、使いやすいことです。

Windowsサンドボックスエクスペリエンスは、構成ファイルを使用してカスタマイズすることもできます。したがって、追加のソフトウェアとツールをWindowsサンドボックスに追加できます。
