Windows Defender SmartScreenは独立した開発者に害を及ぼす

あなたが独立した開発者であり、アプリケーションをリリースしたとします。インストールを簡略化するには、インストーラーを作成して配布を開始します。一部の勇敢な実験者がダウンロードしてインストーラーを起動しますが、厳密な警告が表示されます。


Windows SmartScreenは実行可能ファイルの起動をブロックします

実際、現在の環境では、WindowsはSmartScreen機能を介してバイナリファイルの起動をアクティブにブロックします。

しかし、スマートスクリーンとは何ですか？


SmartScreenは、すべてのWindowsユーザーからインストールデータを収集して、アプリケーションの「評判」を判断します。プログラムの評判が悪い場合は、そのような警告が表示されます。この状況のほとんどのユーザーは、悪意のあるプログラムと見なして.exeを削除しますが、[詳細]ボタンをクリックしてから[実行]をクリックすると、SmartScreenをバイパスできます。

デジタル署名されたラケット


しかし、評判を築くにはどうすればよいですか？最初に、Microsoftは誰がアプリケーションを公開したかに関する情報を収集できなければなりません。これはコード署名証明書を使用して行われます。最も明白な結論は、署名されていないアプリケーションは常にSmartScreenを呼び出すということです。より陰湿な結論は、コード署名証明書の取得は個々の開発者にとって大きな出費であるということです。現在のところ、コードに署名するためのLet's Encryptに相当するものはないため、信頼できる証明機関から証明書を購入する必要があります。価格の範囲は広いですが、1年間の証明書は通常約100ドルかかります。


SSL.comは、市場で最も安価なコード署名証明書の1つを提供しており、価格は年間129ドルです。

フックをつついて、過大評価された素数をコンピューターで乗算して購入し、コードに署名して、アプリケーションを再公開したとします。これでユーザーはアプリケーションのインストールを開始しますよね？違う。

評判を築くことはトリック22


デジタル署名があっても、SmartScreenは引き続き機能します。結局のところ、あなたは未知の新しいパブリッシャーであり、SmartScreenの「評判構築」プロセスは絶対的なブラックボックスです。したがって、ここに秘訣22があります。評判を作成するには、ソフトウェアをインストールする人が必要です。Microsoftは、インストール統計を使用してデータを収集します。しかし、ユーザーにソフトウェアをインストールさせるには、コードがコンピューターに損傷を与えることを明確に示唆するメッセージを取り除く必要があります。

さらに悪い。

WinGetパッケージマネージャーを使用してソフトウェアを公開しようとすると（簡単に言うと、ベータ版を離れると、AppGetからコピーされたこのマネージャーがWindowsストアを置き換える可能性があります）、Microsoftから次のメッセージが表示されます。


Microsoft WinGetはSmartScreenの起動を「間違い」とみなし、ソフトウェアは「悪意のある」ものとして分類され

ます。Microsoftによると、SmartScreenを起動するアプリケーションは悪意があると見なされます。彼らはレビューのために特別なリンクでファイルを送ることを勧めます。このプロセスに従うと、次の答えが得られます。

基本的に、アプリケーションはクリーンですが、SmartScreenエラーでは何もしません。私たちのクラウドボットの領主は、あなたがいつ信頼できるかを決定します。

そしてさらに悪い。

信頼できる発行元として、アルゴリズムによって最終的に承認されたとしましょう。証明書の有効期限が切れ、更新します。インターネットで一般的なこと。しかし、Windowsソフトウェアの世界では、あなたの評判は無効になります。SmartScreenは証明書の更新を認識せず、新しい発行元と見なされるため、この面倒なプロセスを再度実行する必要があります。

これは、有効期間が長い証明書を購入することで軽減できますが、ほとんどの独立した開発者にとって、コストは乗り越えられない障壁になります。

EVコード署名証明書


独立した開発者にとっては、これで終わりです。パブリッシャーには、コード署名（EV）の高度な検証証明書という別のソリューションがあります。それらの1つを使用してコードに署名すると、SmartScreenは自動的にバイパスされます。これは企業のみに予約されており、その名前が示すように、情報の追加検証を提供します。認証センターによって実施されます。

人気の証明機関Digicert.comは、このような証明書を年間699ドルで販売しています。

独立した開発者は、法人を形成するという困難を経験し、これらの証明書の法外な価格でフォークアウトする必要があります。もちろん、ほとんどの場合、これは現実的ではありません。

「開発者、開発者、開発者！」




「開発者、開発者、開発者！」ステージからスティーブバルマーを叫びました。Microsoft CEOの伝説的なスピーチの1つ。これらの悪名高い言葉は2006年に話されました。14年後、サティヤナデラのマイクロソフトは、これまで以上にオープンになったことで称賛されました。しかし皮肉なことに、マイクロソフトがWindows環境をお気に入りの開発者に対して非常に敵対的にしていることが判明しました。このひどい状況を修正するには、SmartScreenを変更するか、コード署名証明書の仕組みを変更する必要があります。
