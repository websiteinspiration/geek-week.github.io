HYPHAE（GIFA）[ギスタブ]



私は、オーバーラップできない互いに接続された円を成長させているシステムで実験を始めました。これは次のように機能します。最初の円（円）をどこかに配置し、半径と移動方向を設定します。次に、移動方向の最初のノードの周囲に沿って新しいノードを追加しようとします。少し「振動」するように、また角度がわずかにずれるたびに、移動の方向を持つことが重要です。また、新しいノードを追加するたびに、新しいノードの半径が前のノードよりも少し小さいことを確認してください。







ブランチを取得するには、ノードをランダムに選択して、新しいノードをブランチの移動方向に対してほぼ垂直に成長させます。そして、ノードが衝突するか、新しいブランチを取得します。より興味深い結果を得るには、分岐の幅（半径）に比例して偏差角度を作成できます。太い枝は細い枝よりもまっすぐになる傾向があります。そして、明らかにもっと多くのことができます。たとえば、細くなるにつれて成長する「フレキシブル」ブランチの数を調整すると、根本的に異なる結果が得られます。







そしてもう1つの小さな注意：ブランチが既存のブランチから分離した場合、そのブランチの半径を以前のブランチよりもはるかに薄くする必要があります。現実の樹で見ることができます。肝心なことは、質量は枝を交差する前後でほぼ同じでなければならないということです。（私はどこかでこれを読みましたが、どこで見つけることができません）。



木[GITHUB]



枝について話していると、本質的に似ているので、木の小さな実験に言及できることに気づきました。







私はGifと同じ方法で枝を育てようとしたが、新しい枝が他の枝の「端」にのみ現れるようにしたかった。さらに、このシミュレーションでは、衝突などに注意を払いたくありませんでした。結果は実際にはツリーではありませんが、一生懸命見ていなければ、結果は非常に現実的に見えます。



おそらく最も興味深いのは、これらの「ツリー」とは異なり、固定幅の幹から成長した単一の実ツリーがないことです。木は時間とともに高さと幅の両方で成長し、これは多くの要因を持つ非常に動的なプロセスです。このアルゴリズムは実際のツリーとの共通点がほとんどないことがわかりますが、このアルゴリズムのいくつかの単純なルールを使用しても、実際のツリーをシミュレートすることで適切な結果を得ることができます。







木に深みのある効果を与えるために、私は枝が成長するにつれて影を枝に加えます。この錯覚はさらに発展し、片側にますます多くの影が追加されます。



重複する行[ GITHUB ]



生物学の影響を受けたGifaやツリーアルゴリズムとは異なり、次の例はJohn Frensenの絵画に基づいています。概して、フレンセンは、各線が以前に描画された線の輪郭に従う線を描画します。後続の各行はわずかに変更され、興味深いパターンがすぐに表示されます。



コンピューターで図形を描くのはとても簡単です。ラインは、一連の頂点（ポイント）と、それらの間にエッジ（セグメント）で構成されます。三角形を作成するには、3つのエッジを取り、それらを3つの頂点に接続する必要があります。円を描くには、同じことをする必要があります。多くの頂点が必要なだけなので、これが実際には円ではないことを確認することは不可能です。このアプローチについて疑問がある場合は、フォームを描画するためのより複雑で正確な方法があるとすぐに言いますが、これは非常にシンプルで正確です。



つまり、多くの頂点を持つ丸みを帯びた形状を描くことにより、フレンセンと同じ方法でこれらの線を描くことができます。フォームの追跡は少しトリッキーですが、試すことができる方法がいくつかあります。前の行を続けながら、図形が実際にどのように描画されるかを模倣する動作が必要でした。私が思いついたアルゴリズムは、そのようには機能しませんでした。つまり、衝突を回避しませんでしたが、結果は非常に良好に見えました。







輪郭をトレースするには、前の線の隣接エッジの一部を選択し、これらのエッジの平均方向を計算し、この方向に沿って現在の線に頂点を追加します。次に、ランダムな動きをいくつか追加して、フリーハンド描画をシミュレートします。しばらくの間、このアプローチはかなりうまく機能しますが、結果に見られるいくつかの「慣性」があり、フォームの適応が遅すぎます。

各頂点に追加するノイズの量は重要です。このノイズは、システム全体で興味深いフォームを作成するものです。フォームの修正により、プロセスでは、一般的な動きといくつかのランダムな歪みの両方を繰り返そうとするためです。

微分は、私がずっと望んできたアルゴリズムです。ある時点で、解決策が頭に浮かんだだけで、アルゴリズムを機能させることができました。自然界の表面が互いに展開するのと同じように動作するシステムを作成したかったのです。たとえば、くるみやキャベツの層のように（赤キャベツの画像を半分にカットしてグーグルするのにどれだけの時間を費やしたかさえわからない）、腸といくつかの種類の葉や花びら。そしておそらく他の多くのもの。



方法を見つける前に、神経系によって作成されたFloraformだけを見ましたが、Andy LomasのCellular Formsも見る必要があります。後で発見したもう1つの重要なドキュメントはこちら。



最初はコッホフラクタルカーブに基づく方法を試してみました。しかし、フラクタルのように対称的で規則的ではなく、ランダムに成長します。私はそれを終えたことはなく、すぐにあきらめました。しかし、しばらくして（プラスまたはマイナス6か月）、Koch曲線が「成長」している間に新しいノードを導入できることに気付きましたが、これらのノードを相互にシフトさせる必要があります。したがって、ロープ上の真珠のように、動的に位置を調整できます。したがって、他のノード（真珠）から適切な距離を保つことができます。



アルゴリズムは以下の通りです。最初に、何らかの形で接続されたノードのセットを作成します（円または線のフィット）。次に、既存のノードのペアの間に新しいノードをランダムに導入します。各反復で、ノードは位置を最適化しようとします。彼らは近くにいることを「望んで」いますが、2人の隣人には近すぎません。同時に、他のすべてのノードから可能な限り離れている必要があります。衝突の可能性はありませんのでご注意ください。







このシステムの動作に影響を与える可能性のあるいくつかの要因があります。その1つは、新しいノードを導入する頻度です。もう1つは、ノードが互いに回避する最大距離です。そして、最も興味深い（私の意見では）新しいノードを挿入する場所を選択する方法です。上のGIFのようにこれを均等に行うことも、下のGIFのように曲線がより鋭い場所に優先順位を付けることもできます。







これらの例では、違いに気づくのは簡単ではありませんが、大規模なシミュレーションでは違いが顕著になります。下図では、曲率による挿入方式を採用しており、作成には約40時間かかりました。







このアルゴリズムから興味深い結果を得る別の方法は、時間の各ステップに合わせて曲線に色を付けることです。視覚的な結果とテクスチャの結果は著しく異なりますが、同じシステム特性を認識するのは簡単です。
