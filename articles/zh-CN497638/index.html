制作调制解调器：使用OFDM和GNU无线电无线传输数字数据

哈Ha

本文可以看作是文章“我们分析拨号调制解调器的声音 ”的续篇，该文章讨论了调制解调器之间建立通信的方法。今天，我们将更进一步，并在实践中了解如何传输数据，为此，我们将使用OFDM和GNU Radio创建一个简单的调制解调器实现。

从字面意义上讲，我们将通过空中传输数据-扬声器和麦克风足以接收和传输。

对于那些对它如何工作感兴趣的人，继续进行下去。

因此，我们的任务是使最简单的调制解调器能够从A点到B点传输数据。有多种调制数字信号的方法，我们将使用OFDM-一种在现代通信系统中广泛使用的方法。在OFDM中，傅立叶变换数字信号被转换为几个并行子载波，从而确保了传输通道的高速和高效。从数字电视和无线电到LTE，OFDM的使用很多。我们的音频通道在参数方面更窄，更适中，因此速度会降低，但基本上本质上不会改变。严格来说，OFDM 不是用于空中数据传输，尽管对于短距离而言，常规的麦克风和扬声器就足够了。

不幸的是，我没有找到用于在GNU Radio中显示接收到的数据的内置单元，因此我必须使用UDP。UDP Sink块用于输出数据； Python程序用于接收数据：

import socket

UDP_IP = "127.0.0.1"
UDP_PORT = 40868

sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM) # UDP
sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
sock.bind((UDP_IP, UDP_PORT))
sock.settimeout(0.5)

while True:
    try:
        data, addr = sock.recvfrom(128) 
        print("Msg:", data, 'Hex:', ' '.join('{:02x}'.format(x) for x in data))
    except socket.timeout:
        pass
