Sobrepondo uma textura 2D em um objeto 3D usando p5.js (parte 2 - aplicando um padrão a um cubo)



Esta é a segunda parte do projeto em que impomos um padrão 2D dinâmico em um objeto 3D. Você pode começar com a primeira parte ( Parte 1 ) ou simplesmente copiar o código resultante alguns parágrafos abaixo.

Há algum tempo atrás, criei um padrão que foi gerado usando várias linhas de código na biblioteca p5.js.
p5.js é uma biblioteca Javascript criada em 2014. Seu principal objetivo é tornar-se uma janela para o mundo da programação para designers, artistas, professores e outros representantes de profissões criativas. Ele suporta animação e diagramas, traduz-se facilmente para o formato da web.

Agora, vou explicar como fazer a peça final - aplique um padrão a um objeto 3d (neste caso, um cubo).

Para começar a trabalhar do zero, você precisa criar um projeto no editor online editor.p5js.org.Nós continuamos alterando o seguinte código:

Para tornar nossa tela volumosa, precisamos adicionar o parâmetro WEBGL à função que cria nossa tela: createCanvas (400.400, WEBGL). Como o ponto de referência para esse objeto já está no centro, podemos (e devemos!) Remover a conversão (200.200), que executou a correção em uma imagem plana.

Para diversificar o esquema de cores e criar uma imagem mais espetacular, vamos substituir os parâmetros fill () pelas nossas variáveis ​​r, a, c - agora, a cada nova iteração do loop infinito draw (), a cor das elipses mudará um pouco. Além disso, vamos adicionar alguma distância entre os círculos e os ângulos de inclinação, alterando a taxa de alteração das variáveis ​​c e a.

   =  + 0.06
  a = a + 0.3

Como melhoria final, vamos adicionar push () antes do nosso código de padrão e pop () depois. O enquadramento dessa maneira torna o código isolado - ele não será afetado por outros objetos que criaremos.

let r=10
let a=0
let c=20

function setup() {
  background(220);
  createCanvas(400, 400, WEBGL);
  
}

function draw() {
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  fill(r,a,c);
  ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
}



No final, temos algo parecido com isto. É hora de criar um cubo.

Na mesma função draw (), depois de pop (), adicionamos o código da caixa:

  push()
  box(200)
  pop()

Agora a caixa com tamanho 200 é virada para nós apenas de um lado e parece apenas um quadrado. Para estimar o volume, precisamos implantá-lo. Para fazer isso, adicione a quarta variável, top, aos r, a, c ones, angle existentes, com um valor padrão de 0. Para esse parâmetro afetar nosso cubo, após o segundo push (), mas antes de criar a caixa. adicione três funções: rotateX (), rotateY (), rotateZ () com o parâmetro angle para cada um.

Como é lógico que um objeto com um ângulo de rotação de 0 graus não gire, adicionamos uma alteração no valor do ângulo variável a 0,003 (para iniciantes, será suficiente). Como resultado, obtemos o seguinte código:

let r=10
let a=0
let c=20
let angle = 0

function setup() {
  background(220);
  createCanvas(400, 400, WEBGL);
  
}

function draw() {
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  fill(r,a,c);
  ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
  
  push()
  rotateX(angle), 
  rotateY(angle), 
  rotateZ(angle)
  box(200)
  angle = angle + 0.003
  pop()
}

Como nosso plano de fundo é criado apenas uma vez (está configurado) - capturamos o primeiro bug que nosso cubo apaga o plano de fundo quando rotacionado.



Para corrigi-lo, você pode simplesmente transferir o plano de fundo (200) dentro da função draw () .No entanto, agora estamos enfrentando outro problema - nossos círculos começam a desaparecer, como no início, porque agora são substituídos a cada novo ciclo e aparecem apenas por uma fração de segundo de nada.

Para resolver esse problema, precisamos criar uma nova variável (por exemplo, art) sem um valor padrão logo no início, ao lado de outras variáveis ​​de nível superior.

Dentro da função setup (), nós a igualamos à função createGraphics (400.400) e a usamos em outros lugares.
Certifique-se de definir os mesmos parâmetros ao criar o Canvas para manter a dimensionalidade.

Agora, dentro do código do padrão, adicionamos a variável art através do ponto à elipse e suas propriedades (como fill () e outras, se você as criou). 

art.fill(r,a,c);
  art.ellipse (x,y,10,10);

Depois disso, resta impor o padrão no cubo como uma textura. Para fazer isso, adicione a propriedade texture (art) dentro do código da caixa de cubo. A lógica é a seguinte: criamos a essência de uma arte gráfica de objeto isolado, que chama a função de criar um padrão com as propriedades especificadas e, depois disso, atribuímos à arte como parâmetro de propriedade uma textura para o cubo.

let r=10
let a=0
let c=20
let angle = 0
let art

function setup() {
  createCanvas(400, 400, WEBGL);
  art = createGraphics(400,400);
}

function draw() {
  background(220);
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  art.fill(r,a,c);
  art.ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
  
  push()
  texture(art)
  rotateX(angle), 
  rotateY(angle), 
  rotateZ(angle)
  box(200)
  angle = angle + 0.003
  pop()
}



Resta resolver o último problema: nosso padrão começa a aparecer nos cantos do cubo. Para começar do centro, podemos simplesmente adicionar metade da largura e comprimento do cubo ao parâmetro xey para mudar o centro do início do desenho do padrão.

art.ellipse (x+200,y+200,10,10);
