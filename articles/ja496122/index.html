宿題のお手入れ：無料のVeeamエージェントを使用してラップトップまたはデスクトップをバックアップ

平凡な生活の中で、私はあなたの多くと同じように、親愛なる読者はほとんどの場合オフィスで働いており、それは私にぴったりです。しかし今日の状況は、私と世界のさまざまな場所にいる同僚の両方、そしてこの記事を読んでいる人々の大多数が、私たち全員が家で仕事をしているような状況です。

そして、ここでは、自宅のコンピュータまたはラップトップをバックアップする必要性を思い起こすのはかなりおかしいでしょう。作業環境が変わると、データを扱う通常の方法も変わります。オフィスでの作業のすべてが明らかです。必要なリソースへの直接アクセス、一元化されたバックアップ、そしてもしあれば、いつでもヘルプデスクに連絡できます。自宅で仕事をする場合、バックアップを含め、自分で用意する必要がある多くのこと。

皆さんの多くは、すでにMicrosoft Windows用のVeeam AgentまたはLinux用のVeeam Agentを使用していると思います。まだの場合は、おそらく今日の簡単なレビューに興味があるでしょう。猫へようこそ。



2014年に、Microsoft Veeam Agent for Microsoft Windowsとして知られる私たちの製品の最初のバージョンがリリースされたとき、それはVeeam Endpoint Backup Freeと呼ばれ、唯一の非商用版がありました。

もちろん、それ以降、ソリューションは大きく変化し、集中管理、CBTドライバー、クラスターサポート、並列ディスク処理など、多くの高度な機能を備えた商用バージョンが登場しました。 Veeam Agent for Linuxも開発中です。これにより、物理マシンをバックアップし、バックアップからデータを復元できます。しかし今、2020年の近づくと、Veeam Agent for Microsoft WindowsとVeeam Agent for Linuxの新しいバージョン個々のコンピュータ用に設計された無料版もあります。

したがって、ホームデスクトップ、ラップトップ、またはWindowsで実行されるタブレットをバックアップする必要がある場合は、健康状態を利用してください。

  2. Veeam Agent VSS, .

  3. 宛先ストレージのデータは、バックアップファイルに保存されます。

用のVeeam Agentの場合も同様です。

  1. 必要なボリュームのスナップショットが作成されます-これは、特別なVeeamsnapモジュールを使用して行われます。（スナップショットは十分な空き容量のあるボリューム上に作成されます。）

    ボリュームがバックアップされると、Veeam Agentはパーティションテーブルとブートローダーをターゲットの場所にコピーします。
  2. Veeam Agent for Linuxはボリュームからデータを読み取り、圧縮を実行してターゲットの場所にコピーします。
  3. 必要なデータがすべてコピーされた後、スナップショットは削除されます。データはバックアップファイルに保存されます。

  • Microsoft Windows Server 2012 R2
  • Microsoft Windows Server 2012
  • Microsoft Windows Server 2008 R2 SP1
  • Microsoft Windows 10（バージョン1607から1909）
  • Microsoft Windows 8.1
  • Microsoft Windows 7 SP1

ノート：

  • サーバーオペレーティングシステムでは、Small Business ServerおよびServer Essentialsエディションがサポートされています。
  • スタンドアロン展開シナリオのサーバーシステムのServer Coreバージョンはサポートされていません。
  • サポートされているWindows Embedded / Windows IoT OS（カスタムビルドを除く）。
  • Veeam Agent for Microsoft WindowsがVeeam Backup＆Replicationを実行している場合、Microsoftフェイルオーバークラスターがサポートされます（スタンドアロンの展開シナリオではサポートされません）。
  • Microsoft Windows 10 Educationは、ビルド10586以降でサポートされています。

Veeam Agent for Microsoft Windowsのバックアップジョブ


バックアップの場合は、新しいバックアップジョブウィザードを起動してバックアップタスクを構成します。その中で、バックアップのタイプを選択します。

  • コンピュータ全体 -マシン全体。
  • ボリュームレベルのバックアップ -選択したボリュームのバックアップ。
  • ファイルレベルのバックアップ -特定のファイルのバックアップ。

Veeam Agent for Microsoft Windowsは、さまざまなタイプのストレージシステムをバックアップストレージの場所として使用できます。

  • ローカルストレージ -外部ドライブ（USB、FirewireまたはeSata）。ご存知のように、ここでの主なことは、マシンの組み込みディスクに自分自身をバックアップすることではありません。
  • 共有フォルダ -ファイルサーバーまたはNAS上のSMB共有。
  • Veeamバックアップリポジトリ -Veeamリポジトリ（オンプレミス）。
  • Veeam Cloud Connectリポジトリは、Veeamサービスプロバイダーによって提供されるクラウドベースのリポジトリです。
  • Microsoft OneDrive

私の例では、バックアップはUSBドライブに送られます-このため、宛先のステップで、[ ローカルストレージ ]を選択します。



次に、選択したタイプのストレージに固有の設定を行います。

重要！リムーバブルUSBドライブを使用する場合、そのようなドライブがFAT32用にフォーマットされている場合、4 GBを超えるファイルの保存はサポートされないことに注意してください。したがって、このようなディスクをターゲットバックアップストレージとして使用することはお勧めしません。

次に、スケジュールステップでスケジュールを設定します。ここでは、特定の時間に定期的にバックアップするか、選択したイベントが発生したときにバックアップするかを指定できます。

  • ロック -車をロックしたとき
  • ログオフ -現在のアカウントでログアウトしたとき
  • バックアップターゲットが接続されているとき-ターゲットバックアップストレージが接続されているとき

3番目のオプションが指定されている場合、バックアップがオフラインになるように、バックアップの最後にターゲットストレージの切断（イジェクト）を完了するようにVeeam Agentに指示することもできます。これにより、暗号化ウイルスのバックアップコピーが検出された場合に、そのコピーにさらされるリスクが軽減されます。

これを行うには、[ バックアップターゲットが接続されている場合 ] → [バックアップが完了したらリムーバブルストレージを取り出す（CryptoLocker保護） ]オプションを選択します。



Linux用のVeeamエージェント


Veeam Agent for Linuxは、Linuxで実行されているマシンのデータを保護するように設計されており、このソリューションにも無料版があります。Linuxカーネル2.6.32以降がサポートされています。

サポートされているのは、64ビットおよび32ビット（該当する場合）バージョンのディストリビューションです。

  • Debian 8.0-10.3
  • Ubuntu 14.04、16.04、18.04、19.10
  • RHEL 6.0-8.1
  • CentOS 6.0-8.1
  • Oracle Linux 6-8.1（RHCK）
  • Oracle Linux 6（UEK R1以降）-Oracle Linux 7（UEK R5 U2まで）
  • SLES 11 SP4、12 SP1-15 SP1
  • SLES for SAP 11 SP4、12 SP1-15 SP1
  • Fedora 30〜31
  • openSUSE Leap 15.1
  • openSUSEタンブルウィード

ノート：

  • Fedoraはカーネルバージョン5.4.14までサポートされます。
  • openSUSE Tumbleweedは、カーネルバージョン5.1.7までサポートされています。
  • Linuxカーネル2.6.32-754.6.3は、CentOS / RHELおよびOracle Linux（RHCK）ではサポートされていません。

詳細については、ドキュメント（英語）を参照してください。

Veeam Agent for Microsoft Windowsと同じタイプのバックアップがサポートされています（ファイルレベルのバックアップを含む）。バックアップ設定は、ウィザードの手順を使用して、またはコマンドラインから設定できます。

各ソリューションの詳細については、Windows（英語）またはLinux（英語）のユーザーガイドを参照してください。

無料版の制限について


Veeam Agent for Linux（サーバーの商用版と比較）：

  1. 作成できるバックアップジョブは1つだけです。
  2. Veeam Cloud Connectリポジトリは、ターゲットバックアップリポジトリとしてサポートされていません。
  3. 凍結前スクリプトと解凍後スクリプトは、バックアップジョブではサポートされていません。
  4. DBMS Oracle、MySQL、PostgreSQLの作業を考慮したバックアップはサポートしていません。

詳細はこちらの表にあります。そして、「舞台裏」で起こっていること、エンジニアがこの「動物園」とVeeamソリューションの互換性をサポートするためにどのように取り組んでいるかについて、Habréのブログの開発者であるSergey Shtepaの開発者による記事を読んでください。

Agent for Windowsの制限は、同じ表に詳細にリストされています。主なもののうち、私たちはこれに注意します：

  1. アプリケーションベースのバックアップはサポートされていません。
  2. Veeam Cloud Connectリポジトリは、ターゲットバックアップリポジトリとしてサポートされていません。
  3. Microsoft OneDriveは、WindowsサーバーOSを実行しているマシンのバックアップストレージとしてサポートされていません。
  4. Microsoft OneDriveは、システムステータスのバックアップにのみ使用できます。（バックアップタイプの詳細については、ここを参照してください）。

Veeam Agentソリューションがホームオフィスで働くすべての人に役立つことを願っています。家のインフラがより広範囲で、一元的なバックアップ管理が必要な場合は、Veeam Backup＆Replication Community Editionの無料版を使用できます。ロシア語でのレビューはこちらです。
