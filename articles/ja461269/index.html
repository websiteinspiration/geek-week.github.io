シェルショック、別名バッシュドア


Shellshock（Bashdoor）は、GNU Bashプログラムで発見されたソフトウェアの脆弱性で、環境変数の非標準値を受け取ったときに任意のコマンドを実行できるようにします。脆弱性は、番号CVE-2014-6271を受け取りました。

Unixライクなオペレーティングシステムでは、各プログラムに環境変数があります。子プロセスは、親から環境変数のリストを継承します。環境変数に加えて、bashは関数の内部リスト（実行可能スクリプトから呼び出すことができる名前付きスクリプト）も保持します。他の（親）スクリプトからスクリプトを実行する場合、既存の環境変数と関数定義の値をエクスポートすることが可能です。関数定義は、空のブラケット「（）」で始まり、その後に文字列としての関数の定義が続く、特別な形式の新しい環境変数の形式でエンコードしてエクスポートされます。新しいbashインスタンスは、開始時にすべての環境変数をスキャンし、この形式を検出して、内部関数の定義に変換します。この方法では、攻撃者が任意の環境変数をbash起動に送信できる場合、任意のコマンドを実行することが可能になります。

次の例では、テキストVULNを出力します。

env x=’() { : ; }; echo “VULN”’ bash -c “echo text”

「echo text」コマンドを実行すると、環境変数xに含まれる関数定義が読み込まれ、「echo」VULN」コマンドが実行されます。
