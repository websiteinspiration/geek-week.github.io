SAPクラウドプラットフォームを使用してデジタルコアの拡張機能を作成する方法

現在、企業はビジネスに革新的なシナリオを適用する多くの機会を持っています。市場は多くの技術とアイデアで代表されています。ただし、それぞれの使用を開始するには、不当に高い努力が必要です。大量のデータ、ロボットルーチン操作、高度な分析、機械学習などの管理-これらの各用語は、個別の技術スタック、または場合によっては複数の用語です。それらはすべて適合しません。それらに関する情報はインターネット上の多くのソースに散在しており、十分な関連性がない場合がよくあります。責任を持って最先端のデジタル企業規模のソリューションを構築するには、IT管理者とアーキテクトが多くのスキルと膨大な実務経験を持っている必要があります。理論的知識はここでは不十分です。ソフトウェアとサービスの主要サプライヤーによって構築され、サブスクリプションによって提供される既製のクラウドプラットフォームは、この問題の解決に役立ちます。この記事では、SAP Cloud Platform（SCP）を使用してさまざまなビジネス上の問題を迅速かつ効率的に解決するための一連の資料を開きます。

SCPは古典的なPaaSとして始まりましたが、長年にわたってビジネステクノロジープラットフォームに進化しました。これにより、お客様はデジタルソリューションを実装する際に、特定のITテクノロジーの実装よりも管理および生産タスクに集中できます。 SCPには数十のビジネスおよび技術サービスが含まれており、プラットフォーム内にビルディングブロックがあります。クライアントアプリケーションにはこれらのブロックの一部が含まれている場合がありますが、開発者は特定の技術的な実装の複雑さを詳しく調べる必要はありません。

SCPは多くの問題を解決できます。重要な問題の1つは、SAP S / 4HANAまたはSAP ERP、SAP C / 4HANA、SAP SuccessFactors、SAP Aribaなど、企業のデジタルコアのさまざまなアプリケーションの拡張です。このプラットフォームを使用して、クラウドおよびオンプレミスソリューションの機能を追加し、SAPソリューションを相互に統合したり、サードパーティシステムと統合したり、新しいテクノロジーをテストしたり、ビジネス仮説をテストしたりできます。

ビジネスシステムの大規模な実装の標準的な問題の1つは、いわゆる「技術的負債」の出現です（図1）。このプロセスはすべてのベンダーに関連しています。遅かれ早かれ、ビジネスを管理するメインシステムが開発プラットフォームに変わります。現在の開発は過去の開発に基づいており、技術的負債は増加しており、利息の支払いさえ必要です。かつて「ひざまずいて」と書かれていた「時が来れば、すべてを正しくやる」という機能のサポート。しかし、時間は決してありません。さらに、開発プラットフォームとしてのERPのアイデアは、迅速な革新、研究、および実験のアイデアとうまく相関していません。

そのため、当社は、SAP Cloud Platformを、コアビジネスソリューションを拡大するための推奨される並列型プラットフォームと見なしています。マイクロサービスアプリケーションを開発するために設計されたSCPでは、ほとんどすべてのプログラミング言語、サーバーレスコンピューティング、およびコンテナーを使用できます。 SAPアプリケーションの最新バージョンにはイベント駆動型のアーキテクチャがあり、それらをSCP環境に統合することは簡単な作業になりつつあります。

もちろん、SAP Cloud Platformと一般的にはside-by-side拡張は、基本的なSAPアプリケーション（デジタルコアと呼ばれるもの）を変更する唯一の可能な方法ではありません。この道に着手した企業は、新しい技術、ビジネスモデル、製品、および「イノベーション」と呼ばれるすべてを開発する十分な機会を得ています。 SCPは、マルチモーダルスキームに従ってIT組織を変革し、開発分野とビジネスプロセスの両方において、最も広い意味でのアジャイルアプローチのレールに変換するための技術的基盤です（図2）。

SAPはこのアプローチを強く歓迎し、サポートしています。石油生産者であるマーフィーオイルコーポレーションは、2019年のSAPイノベーションアワードの受賞者の1人になりました。プロジェクトチームは、SAP ERPなどの会社のコアシステムの機能を拡張するだけでなく、機械学習やIoTなどの多くの革新的なテクノロジーを統合し、モバイルインターフェイスを開発し、外部サービスをカーネルに統合しました（車両監視用のテレマティクスサービスプロバイダーなど）。 。

もちろん、SAP Cloud Platform内に統合されているすべての異種テクノロジーには、SAP本体キットがあります。アプリケーションにIoTまたはブロックチェーンの詳細を含めたい開発者は、これらの特定のテクノロジーを実装するための複雑なことを掘り下げる必要はありません。図3では、プラットフォームとそれに付属するSDKにより、少量のコードと設定を使用してSAPシステムに接続し、必要な形式でデータを取得し、SAPアイデンティティプロバイダーを使用して認証し、SAP環境でプリンシパルプロパゲーションを使用できることがわかります。 SAP環境に固有のその他の機能を使用します。

SCPの使用に関する世界的な経験は、顧客がその目的について同様の認識を持っていることを示しています。ほとんどの場合、このプラットフォームは標準のSAPアプリケーションを拡張するために使用されます。柔軟なライセンスモデルにより、特定の最新テクノロジーを試し、特定のビジネスでの適用性をテストすることができます。大幅な変更不可の資本コストは発生しません。

最も頻繁に実装されるタスクの1つは、SAPソリューションの統合、または他のメーカーのソリューションとの統合です。SAPサービスの別のグループは、プロセスの統合のためのソリューション専用であり、ほとんどの場合、クラウドソリューションと一緒に購入されます。これらは、クラウド統合、エンタープライズメッセージング、API管理などのサービスです。このグループのサービスを使用すると、あるビジネスシステムから別のビジネスシステムに情報を転送するブロックからマイクロサービスアプリケーションを収集できます。

Data Driven Insights


クラウドプラットフォームの利点の1つは、大規模な設備投資をせずに技術実験を実施できることです。これは、データ処理の分野でのビジネス仮説のテストについて話すときに特に重要です。ここで、SAPにはいくつかのツールがあります。SAPHANA Cloudの柔軟なマルチレベル、マルチ温度ストレージでのデータストレージ、クラウドソリューションData Warehouse Cloudでのビジネスユーザー向けの便利なデータマートの組織、Data Science SAP Data Intelligenceのプラットフォームです。これらのソリューションはすべて、かなりのハードウェアリソースと高い設置労力を必要とします。クラウドでの既製のサービスにより、革新的なビジネスノウハウの開発と試運転に費やす時間を大幅に削減できます。例えば、PwCはSAPデータウェアハウスクラウドを使用して、150か国以上のオフィスにあるさまざまなシステムからのデータを統合して表示します。 DWHは「従来の」データウェアハウスを置き換えるものではありませんが、ビジネスユーザーは世界中のさまざまなソースから統合されたデータに簡単にアクセスできます。組み込みのSAP Analytics Cloudを使用すると、ユーザーは新しいアイデアや仮説をすばやくテストでき、ITサービスをほとんど、またはまったく関与させる必要がありません。
