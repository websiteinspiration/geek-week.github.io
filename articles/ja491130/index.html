覚えておくべきVueの機能

私たちPtahチームは、通常のSPAよりも少し先に行くことを決め、Vueをランディングページデザイナーとして使用しようとしました。そして今、私たちの経験の一部を共有したいと思います。

この記事は、主にVueの使用を開始したばかりで、その機能について詳しく知りたい方を対象としています。ここでは、フレームワークの機能のいくつかについてお話ししたいと思います。これらの機能は、初心者の開発者によって不当に忘れられることが多いものです。

レンダリング機能


コンポーネントテンプレートは、開発者がVueを愛するものの1つです。これらのおかげで、フレームワークのエントリしきい値は低くなっています。テンプレート構文は、論理的で美しいコードを書くのに90％のケースで十分です。しかし、残り10％で、コンパクトなコンポーネントを作成できない場合はどうすればよいでしょうか。レンダー機能が役立ちます。

ドキュメントの例を使用してそれが何であるかを見てみましょう：

Vue.component('anchored-heading', {
 render: function (createElement) {
   return createElement(
     'h' + this.level,   //  
     this.$slots.default //   
   )
 },
 props: {
   level: {
     type: Number,
     required: true
   }
 }
})

anchored-heading コンポーネントはlevelプロパティを受け入れ、titleタグを描画します。したがって、記録

<anchored-heading :level="1">, !</anchored-heading>

に変換されます

<h1>, !</h1>

このコンポーネントが標準テンプレートを使用して記述されている場合、異なるレベルのヘッダーを記述する最大6つのv-if条件が含まれます。

<h1 v-if="level === 1">
 <slot></slot>
</h1>
<h2 v-if="level === 2">
 <slot></slot>
</h2>
<h3 v-if="level === 3">
 <slot></slot>
</h3>
<h4 v-if="level === 4">
 <slot></slot>
</h4>
<h5 v-if="level === 5">
 <slot></slot>
</h5>
<h6 v-if="level === 6">
 <slot></slot>
</h6>

使い方？


renderメソッドは2つの引数を取ります。createElementの最初の引数は、どのVue要素を作成するかを記述する関数です。コミュニティでは、省略することが通例であるのcreateElementを - 1つの文字に時間。2番目の引数はcontextで、コンテキストデータにアクセスします。

createElementは3つの引数を取ります。

  1. 作成するアイテム。これは、HTMLタグだけでなく、コンポーネント名にもなります。この引数は必須です。
  2. データを持つオブジェクト。クラス、スタイル、コンポーネントの入力パラメーター、イベントを処理するメソッドなどのリストが含まれる場合があります。詳細については、ドキュメントを参照してください。オプションの引数。
  3. 子仮想ノード。文字列または配列のいずれかです。上記の例では、これは$ Slots.defaultです。

レンダリング関数は、ほとんどの予期しない状況で役立ちます。たとえば、Ptahでは、一部のデザイナー要素が適切に機能するために、ページ内でスタイルタグを使用する必要があることがよくあります。ただし、Vueはテンプレートコンポーネント内でのこのタグの使用を禁止しています。この制限は、小さなラッパーのおかげで簡単に回避できます。

Vue.component('v-style', {
 render: function (h) {
   return h('style', this.$slots.default)
 }
})

テンプレート内では、スタイルタグの代わりに、v-styleを使用できます。

したがって、Vueテンプレートの標準機能では不十分であると思われるようになったらすぐに、Render関数について考えてみてください。一見すると複雑に見えますが、JSが提供するすべての機能を使用できます。

ミックスイン


重複するコードを持ついくつかの同様のコンポーネントがありますか？ミックスインや不純物は、DRYの原則に準拠するのに役立ちます。ミックスインの機能は、一度に複数のコンポーネントで使用できます。

例を見てみましょう。同様のロジックを持つ2つのコンポーネントがあるとします。

export default  {
  name: 'TextElement',

  data () {
    return {
      elementName: 'Text',
      showEditor: false,
      editor: null
    }
  },
  
  methods: {
    initEditor () {
      this.showEditor = true
      this.editor = new Editor(this.elementName)
    }
  }
}

export default  {
  name: 'ButtonElement',

  data () {
    return {
      elementName: 'Button',
      showEditor: false,
      editor: null
    }
  },
  
  methods: {
    initEditor () {
      this.showEditor = true
      this.editor = new Editor(this.elementName)
    }
  }
}

コンポーネントは異なりますが、ロジックは同じです。これを作成するには、通常のjsファイルを作成する必要があります。コンポーネントの隣のmixinsディレクトリに配置するのが理にかなっています。

// mixin.js
export default  {
  data () {
    return {
      showEditor: false,
      editor: null
    }
  },
  
  methods: {
    initEditor () {
      this.showEditor = true
      this.editor = new Editor(this.elementName)
    }
  }
}

// TextElement.vue
import mixin from './mixins/mixin'

export default  {
  name: 'TextElement',

  mixins: [mixin]  //  

  data () {
    return {
      elementName: 'Text',
    }
  },
}

// ButtonElement.vue
import mixin from './mixins/mixin'

export default  {
  name: 'ButtonElement',

  mixins: [mixin]

  data () {
    return {
      elementName: 'Button'
    }
  }
}

例からわかるように、ほとんどすべてのロジックがミックスインに移行されました。コンポーネント内で不純物を使用すると、すべてのオプションが統合されます。そしてコンポーネントで、あなたが自由に呼び出すことができますinitEditor（）メソッドを逆にし、不純物で、がelementNameコンポーネントからは、ここで使用されます。この場合、データオブジェクトは再帰的にマージされ、コンポーネントのプロパティが優先されます。

したがって、不純物の利点は明白です。これはコードの再利用です。しかし、マイナスがあります。この例は、ほんの数行の合成です。Ptahで使用されているような実際のコンポーネントは、数百行のコードで記述できます。特にコードをコンポーネントに追加することを見落としている場合、このコードがどのように機能するかを記述していない人にとっては、必ずしも明確ではありません。残念ながら、このマイナスを取り除くことはできません。私は2つのことをお勧めします。JSDocでコンポーネントの操作を説明し、不純なプロパティに特別な名前を使用します（たとえば、事前にチームと同意する接頭辞を追加できます）。

提供/注入


このオプションのペアは常に一緒に使用され、親コンポーネントからその子孫の階層全体にデータを転送できます。これらのオプションは主にプラグインを作成するために使用され、公式ドキュメントではアプリケーションでの使用を推奨していません。アプリケーションでは、コンポーネント間の通信はVuex上で非常にうまく構築されています。ただし、この機能には注意が必要です。

使い方？


最初に、親コンポーネントにデータを定義する必要があります。これを子孫に渡します。

// Parent.vue        
export default {
  provide: {
   device: 'is-desktop'
  }
}

次に、転送されたデータを子コンポーネントに埋め込む必要があります。

// Child.vue    
export default {
  inject: ['device'],

  created () {
   console.log(this.device) // => "is-desktop"
  }
}

例からわかるように、すべてが非常に単純です。ただし、1つの重要なマイナスに注意する必要があります-提供/挿入バンドルからのデータは、デフォルトでは無効です！ただし、この欠点はObject.definePropertyを使用して簡単に回避できます。

provide () {
 let device = {}
 Object.defineProperty(device, 'type', { enumerable: true, get: () => this.device })
 return { device }
},

data () {
 return {
   device: 'is-desktop'
 }
}

今すぐ変更this.deviceを子孫にそれを変更します親に。

コンポーネントメタ


コードで使用されるコンポーネントが事前にわからない場合があります。私たちのエディターの例を考えてみましょう。タスクは次のとおりです。FirstScreenの条件セクションで、Text、Logo、Button要素を表示し、SocialIconsをこれらの要素に追加します。

したがって、要素のコンテナとして機能するセクションコンポーネントと、要素自体の4つのコンポーネントがあることは明らかです。構造は、おおよそ次のようになります。

/ sections
 -- FirstScreen.vue
/ elements
 -- Text.vue
 -- Logo.vue
 -- Button.vue
 -- SocialIcons.vue

要素のすべてのコンポーネントを一度にFirstScreen テンプレートに追加し、条件を使用してそれらを切り替えることは、非常に賢明な決定ではありません。そのようなタスクのためのシンプルで素晴らしいツールがあります：

<component :is="%componentName%"/>

：を持つコンポーネント 要素は、コンポーネントの名前を書き込むだけの属性です。そして彼のおかげで、私たちの仕事は簡単に解決されました：

<script>
export default  {
  name: 'FirstScreen',

  data () {
    return {
      elements: [
        'Text',
        'Logo',
        'Button',
      ],
    }
  }
}
</script>

<template>
  <div class="first-screen">
    <component v-for="element in elements" :is="element"/>
  </div>
</template>

elements 配列では、コンポーネントの名前を記述してから、これらのコンポーネントをFirstScreenテンプレート内のループで出力します。ここで、ソーシャルネットワークのアイコンを含む要素をセクションに追加するには、this.elements.push（ 'SocialIcons'）を実行するだけです。
