USB-AmazonでのCECアダプターのレビュー
I use this tiny (13g) adapter as reliable solution for the typical scenario. When PC starts up, the adapter powers on the TV, and switches the TV's input to HDMI1. When PC shuts down, the adapter powers the TV off.

All I had to do to make it work is to install the latest version of libCEC by Pulse-Eight on GitHub, and follow the instruction «Turn on/off TV using USB-CEC Adapter (Windows Batch script)» published in FAQ section on Pulse-Eight web site.

The adapter also works in opposite way. If my PC is in sleep or hibernate state and I turn on my TV, the adapter wakes up the PC after 30 seconds, provided the HDMI1 source is selected on TV when it powers on. The same happens if I switch from watching TV to HDMI1 source (without 30 seconds delay). Don't know how to turn off this 'wake the PC' feature, though. By the way, when I power off the TV, the PC remains powered on.

My configuration:
ADAPTER — firmware v9, libCEC v4.0.3, recognized in Windows Device Manager as «Pulse-Eight USB to HDMI CEC Adapter (rev.2)», and for unknown reason as «HID-compliant mouse».
TV — Samsung SmartTV LT27H390S (PC monitor with built-in TV-tuner), supports CEC v1.4.
PC — Intel Core i5, Windows 10 x64 v1903.

The adapter supports CEC v1.4 (there is no difference between v1.4, v1.4a, and v1.4b in terms of CEC). It can emulate many CEC-types of devices: Recording, Playback, Tuner, Amplifier. It's even possible to emulate several types at once.

Hardware part is based on energy efficient AT90USB162 microcontroller. That's why the power consumption is less than 0.1W, and the box doesn't heat up.

Software part is based on Microsoft Visual C++ 2015 Redistributable Package (x86), and employs libusb-win32 when upgrading the firmware.

For those who experience any problems, here is a short troubleshooting list.
Ensure that the option «Anynet+ (HDMI-CEC)» is on (located in TV Settings => General => External Device Manager).
HDMI cable that connects adapter to TV supports CEC.
For 'power on' feature to work, the adapter may have to be physically connected to HDMI 1 port (not HDMI 2, etc).
Update the firmware and software of your adapter.

I would definitely give this device 5 stars, but it has some considerable drawbacks:
As many users have already complained, there are no clear documentation, such as User Guide or at least step-by-step Quick Start Guide instruction in PDF form.
It seems like the development of libCEC has stopped. No new version has been released from November 2018 to January 2020.
Powering on the TV wakes up the PC if «Allow this device to wake the computer» setting is cleared.
There are some functionality and usability issues with cec-client.exe and cec-tray.exe (both are part of libCEC software), but that's another story…

IRトランシーバーの使用を拒否したため、残っている唯一のオプションは、Wi-FiまたはLANを介して、つまりTCP / IPを介してコマンドを送信することです。

samsungctl.exeユーティリティを使用する手順は簡単です。

"%LOCALAPPDATA%\Programs\Python\Python38\Scripts\samsungctl.exe" --host 192.168.0.102 --port 8002 --method websocket --timeout 3 KEY_VOLDOWN

代わりに192.168.0.102、KEY_VOLDOWNテレビのIPアドレスと必要なコマンドを置き換える必要があります（後ですべてのコマンドのリストを提供します）。

そして別の待ち伏せが待っていた。 Samsungは、KEY_TV、KEY_DTVなどのコマンドのサポートを実装していません。これにより、ネットワークコマンドを使用して信号ソースをTV入力に切り替えることができます。私は再び忍耐強く、回避策を探す必要がありました。

最も明白なオプションは、リモートでの連続したボタン押下をエミュレートすることです：KEY_SOURCE、KEY_LEFT、KEY_ENTER。しかし、現時点で信号ソースがHDMI 1ではなく、たとえばHDMI 2である場合はどうでしょうか。 CECメッセージ1F：85 <Request Source Source>への応答もSamsungによって実装されていないため、このオプションは表示されなくなります。

次に、KEY_CH_LIST + KEY_EXIT + KEY_EXITコマンドの組み合わせを使用してみました。しかし、すべてのチャンネルのリストがしばらく画面の右側を占めていたテレビ番組を見るのは不愉快でした。さらに、何らかの理由で、HDMI1入力へのリバーススイッチが時々ありました。

次の行は、KEY_GUIDE + KEY_EXIT + KEY_EXITコマンドの組み合わせでした。しかし、HDMI1入力に戻すことは不可能であることがわかりました。

その結果、KEY_CHUP + KEY_CHDOWNの組み合わせで解決しました。
最後のコマンドは、原則として使用できません。ただし、以前に視聴したチャンネル番号に戻ることができます。最終的に、TV入力に切り替えるためのスクリプトは次のようになります。

"%LOCALAPPDATA%\Programs\Python\Python38\Scripts\samsungctl.exe" --host 192.168.0.102 --port 8002 --method websocket --timeout 3 KEY_CHUP
TIMEOUT /T 2
"%LOCALAPPDATA%\Programs\Python\Python38\Scripts\samsungctl.exe" --host 192.168.0.102 --port 8002 --method websocket --timeout 3 KEY_CHDOWN

2秒続くチーム間のわずかな遅延により、チームの運用の信頼性が向上します。
このスクリプトのショートカットを作成し、モニターからテレビにすばやく切り替えるためのホットキーを割り当てるだけです。これを行うには、Ctrl + Alt + 0の組み合わせを使用しました。

購入時に、このモデルはSmartTV機能、27インチの対角、フルHD解像度（24インチの対角でも同じ）を備えた市場で唯一のモデルでした。これまで受け続けられてきた多数のレビューから判断すると、メーカーは、この市場セグメントの開発をやめたように思われます。たとえば、Yandex.Marketによると、「この製品は2か月で639人が購入しました」（執筆時点）。 24インチモデルの場合、この数値はさらに高く-802人です。ほとんどのロシアのアパートの控えめなキッチンエリアを考えると、当然のことながら、小型テレビモデルが主に購入されています。
なぜこのアイテムをメリットに帰するのですか？結局のところ、競争の欠如は悪いです。多くのパラメータで異なるモデルを比較する必要がなく、1つだけを選択するという苦痛を経験するためです（絵文字がHabr ルールによって歓迎されないのは残念です）。

LCDに不良ピクセルはない。これらの写真の助けを借りてチェックしました（ディスクに保存、便利）：

電源投入後のクイックスタート（3〜4秒）。高速チャネル切り替え（約2秒）。ケーブルデジタルチャネルのクイックセットアップ。

デスクトップ上でマウスを操作する際の遅延（入力遅延）はありません。FPSのテスト、あまりにも、すべてが順序です。

インドネシア製のコンパクトで人間工学に基づいたIRリモートコントロール（Bluetoothベースではスマートではない）。中国製の2つの完全なマスタングAAAバッテリー。
リモコンの寸法：170x45x25 mm。私は彼のためにロシアの会社WiMAXのカバーRCCWM-50170-Bを買いました。
リモコンに欠けている唯一のことは、たとえリモコンのサイズが少し大きくなったためであっても、テレビとコンピュータ（LGなど）をすばやく切り替えるためのTV / PCボタンです。

プラグが完全に挿入されていない場合は、4ピンプラグ付きのヘッドフォン（たとえば、携帯電話のヘッドセット）を使用できます（そうしないと、ステレオではなくモノラルに聞こえます）。

サウンド遅延（リップシンク）は、TVとHDMI入力で個別に設定できます（SPDIF出力でのみ機能します。HDMI-ARC出力ではほとんど確認していません）。モーションライティングの省エネ機能も同様です。

HDMI入力のアイコンは、現時点で信号があるかどうかを示し、不要な切り替えを回避します。
ソースの選択が[ソース]ボタンで行われた場合、[Enter]ボタンでの選択の確認はオプションです。数秒後、選択した入力が自動的にオンになります。

スムーズなフェードサウンドの美しいターンオフアニメーション。また、テレビの電源を入れたときにもスムーズに音が鳴ります。

ボタンを押す音を消したり、音量を調節したりできます。
