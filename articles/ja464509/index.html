テンソルの概要


テンソルはベクトルと行列の抽象的な形で

、それまではベクトルと行列を主要な構造として使用していました。行列はベクトルのリストであり、ベクトルはスカラー（単数）のリストであることを思い出してください。テンソルは、数値のネストされたリストの抽象的な表現です。ベクトルは1次元のテンソルです。行列は2次元のテンソルであり、多数の次元を持つ構造はn次元テンソルと呼ばれます。それでは、Tensorと呼ぶ基本型を定義して、新しいディープラーニングフレームワークの作成を始めましょう。

import numpy as np

class Tensor (object):

     def __init__(self, data):
          self.data = np.array(data)

     def __add__(self, other):
          return Tensor(self.data + other.data)

     def __repr__(self):
          return str(self.data.__repr__())

     def __str__(self):
          return str(self.data.__str__())

x = Tensor([1,2,3,4,5])
print(x)


    [1 2 3 4 5]

y = x + x
print(y)

    [2 4 6 8 10]

これは、基本的なデータ構造の最初のバージョンです。すべての数値情報をNumPy配列（self.data）に格納し、単一のテンソル演算（追加）をサポートすることに注意してください。追加の操作を追加することはまったく難しくありません。対応する機能を備えた追加の関数をTensorクラスに追加するだけです。

自動勾配計算の概要（autograd）


以前は、手動の逆伝播を実行していました。それを自動化しましょう！

第4章では、デリバティブを紹介しました。それ以来、新しいニューラルネットワークごとにこれらの導関数を手動で計算してきました。これは、ニューラルネットワークを介した逆の動きによって実現されることを思い出してください。最初に、ネットワークの出力での勾配が計算され、次に、この結果を使用して、前のコンポーネントの導関数が計算され、アーキテクチャのすべての重みに対して正しい勾配が決定されます。勾配を計算するためのこのロジックは、テンソルクラスに追加することもできます。以下は私が考えていたものを示しています。

import numpy as np

class Tensor (object):

     def __init__(self, data, creators=None, creation_op=None):
          self.data = np.array(data)
          self.creation_op = creation_op
          self.creators = creators
          self.grad = None

      def backward(self, grad):
          self.grad = grad

          if(self.creation_op == "add"):
               self.creators[0].backward(grad)
               self.creators[1].backward(grad)

      def __add__(self, other):
           return Tensor(self.data + other.data,
                                creators=[self,other],
                                creation_op="add")

      def __repr__(self):
           return str(self.data.__repr__())

      def __str__(self):
           return str(self.data.__str__())

x = Tensor([1,2,3,4,5])
y = Tensor([2,2,2,2,2])

z = x + y
z.backward(Tensor(np.array([1,1,1,1,1])))

この方法は2つの革新をもたらします。まず、各テンソルは2つの新しい属性を受け取ります。 creatorsは、現在のテンソルを作成するために使用されるテンソルのリストです（デフォルトはNoneです）。つまり、テンソルzが他の2つのテンソルxおよびyを追加することによって取得される場合、テンソルzの作成者属性にはテンソルxおよびyが含まれます。 creation_opは、このテンソルの作成プロセスで使用される演算を格納するコンパニオン属性です。つまり、命令z = x + yは、3つのノード（x、y、z）と2つのエッジ（z-> xとz-> y）を持つ計算グラフを作成します。この場合、各エッジはcreation_opからの操作、つまり追加によって署名されます。このグラフは、勾配の再帰的な逆伝播を整理するのに役立ちます。


この実装の最初の革新は、各数学演算中にグラフを自動的に作成することです。 zを取得して別の操作を実行すると、グラフはzを参照する新しい変数で継続されます。

Tensorクラスのこのバージョンの2番目の革新は、グラフを使用して勾配を計算する機能です。 z.backward（）メソッドを呼び出すと、z（追加）テンソルが作成された関数を考慮して、xおよびyの勾配が渡されます。上記の例に示すように、勾配ベクトル（np.array（[1,1,1,1,1]]）をzに渡し、それを親に適用します。おそらく第4章で覚えているように、加算による逆伝播とは、逆伝播を適用することを意味します。この場合、xとyに追加するグラデーションは1つだけなので、zからxとyにコピーします。

print(y.grad)
