錆が主流になる



Rust

Rustは一般的なマルチパラダイムプログラミング言語です。これは、オペレーティングシステムカーネルを開発するための有望な言語と見なされています。特に、セキュリティ、特に並列コード実行のセキュリティに注意が払われます。Rustは、構文の点ではC ++に似ていますが、高いパフォーマンスを維持しながら、メモリをより安全に操作できるように設計されています。


錆（出典）

過去4年間、錆は着実な成長を遂げてきました。2020年はRustが正式に主流になる年だと思います。ところでそれはどういう意味ですか？おそらく、誰もが自分でこの概念を定義しています。たとえば、Rustはさまざまな教育機関で教え始めると思います。これにより、Rustプログラマーの新しい波が生まれます。


2019年のStackOverflow 調査からのお気に入りのプログラミング言語

Rustは、活気に満ちた活気のあるコミュニティが発展する素晴らしい言語としての地位を確立しています。そしてFacebookがRustの巨大な天秤座プロジェクトの助けを借りて作成しているという事実を考えると、この言語が何ができるかすぐにわかります。

新しいプログラミング言語を学ぶ場合は、Rustを学ぶことを強くお勧めします。たとえば、この本を試すことができます。

GraphQLの人気は高まり続けています



これらのGoogleトレンドのGraphQL

は、アプリケーションが成長し、その複雑さが増すにつれて、データ消費に関連するこれらのアプリケーションのニーズが増大します。私はGraphQLの大ファンです。私はこのテクノロジーを何度も使用しています。GraphQLは、従来のREST APIよりもはるかに優れていると思います。

典型的なREST APIでは、複数のクエリを実行してさまざまなソースからのデータの読み込みを整理する必要がありますが、GraphQLでは、1つのリクエストのみを実行することで、必要に応じてさまざまなソースから取得したすべてのデータを取得できます。


GraphQL

GraphQLは、さまざまなサイズのチームによって使用されます。このテクノロジーはさまざまな環境で使用され、さまざまな言語で記述されたアプリケーションで利用できます。モバイルアプリケーション、ウェブサイト、APIを作成するときに使用されます。

プログレッシブWebアプリケーションは考慮されるものになります


プログレッシブWebアプリ（PWA）は、アプリケーション開発への新しいアプローチであり、Webアプリケーションとモバイルアプリケーションのすべての最高の組み合わせに基づいています。


プログレッシブWebアプリケーション（ソース）

世界には、モバイルプラットフォームに特化した開発者よりも多くのWeb開発者がおり、これらのプラットフォーム固有のテクノロジーを使用してアプリケーションを作成しています。大企業がWeb開発者の力でPWAを作成できることに気づいたら、そのようなアプリケーションの爆発的な成長が見られると思います。

しかし、大企業がこのアイデアを受け入れるには、しばらく時間がかかります。これは完全に正常です。 PWAの「プログレッシブ」部分は、主にフロントエンド開発に関連しています。実際には、その作成は主に、Web Workers API（これは標準のブラウザーAPIです）との相互作用の説明を意味します。

Webアプリケーションはどこにも行きません。ほとんどのプログラマーは、単一のクロスプラットフォームPWAを作成することで、簡単に言えば、以前と同じ結果を得ることができると同時に、仕事に費やす時間を減らしてより多くを稼ぐことができるという考えに熱心です。

WebAssemblyテクノロジーがさらに注目される



WebAssemblyオープンソース

からWebAssembly（Wasm）テクノロジについて学ぶことができるものは、次のとおりです。 「WebAssemblyは、C / C ++ / Rustなどの高水準言語向けの移植可能なコンパイルターゲットとして設計されており、クライアントおよびサーバーアプリケーション用にWebに展開できます。」 Wasmは、高度なWebアプリケーションを開発するときにも使用できます。 つまり、Wasmは、JavaScriptの世界とテクノロジーの世界を異なるレベルで接続できるようにするメカニズムです。 ReactアプリケーションでRustで記述された画像処理用のライブラリの使用を検討しているとします。WebAssemblyではこれを行うことができます。



アプリケーションのパフォーマンスは常に非常に重要であり、アプリケーションが処理しなければならないデータの量は常に増加しているため、高レベルのパフォーマンスを維持することは簡単な作業ではありません。ここで、C ++またはRustで記述された低レベルのライブラリが機能します。来年は大企業によるWebAssemblyの実装例が数多く見られると思います。

Reactは最前線をリードし続けます。



Google Trends

Reactのフロントエンド開発ツールに関する情報は、間違いなく最も人気のあるJavaScriptフロントエンド開発ライブラリです。この状況は偶然ではありませんでした。 Reactアプリケーションの作成は簡単で楽しいです。 React開発チームとこのライブラリを中心に開発したコミュニティは、Reactとそのサポートライブラリのさまざまな側面を改善するというすばらしい仕事をしてきました。


React

私はVue、Angular、Reactと連携してきましたが、これらはすべてWeb開発のための素晴らしいツールと言えます。ライブラリまたはフレームワークの目的は、特定の問題を解決することであることを忘れないでください。したがって、さまざまなツールについて言えば、それらがどのように配置されているかではなく、どれだけ効果的に開発者を目標に導くことができるかを考慮に入れる価値があります。「最適なツール」についての推論はまったく不要です。そのような推論の代わりに、開発者は直面する問題の解決に全力を注ぐ必要があります。
