PostgreSQLレシピ：非同期タスクスケジューラ

非同期タスクスケジューラを準備するには、postgres自体とその拡張pg_taskが必要です。（postgresのフォークへのリンクを提供しました。元のリポジトリにまだ詰め込まれていないいくつかの変更を加えたためです。既製のイメージを使用することもできます。）

（オリジナルのPostgreSQLのPL / pgSQLではエラーがキャッチされない例外は、PL / pgSQLで書かれたタスクで発生したときに私のスケジューラが正しく動作しないため、あります。私はこのエラーを説明し、ここで、私のフォークでそれを固定し、ここ。）

をインストールするにはスケジューラは（各）データベースに拡張を作成する必要はありません。代わりに、構成ファイルに追加するだけです

shared_preload_libraries = 'pg_task'

PostgreSQLを再起動した後、スケジューラは、データベースユーザーの代わりにすべてのデータベースにタスクテーブルを作成し、これらのユーザーのデフォルトスキーマに作成します。

特定のデータベースに対してのみスケジューラを実行する場合は、構成ファイルでそれらを指定できます。

pg_task.database = 'database1,database2'

データベースユーザーからではなくスケジューラを実行する場合は、次のように設定することもできます。

pg_task.database = 'database1:user1,database2:user2'

ユーザーのデフォルトスキームにないスケジューラテーブルを作成する必要がある場合は、次のように設定できます。

pg_task_schema.database1 = schema3

スケジューラテーブルに別の名前を付ける必要がある場合は、次のように行うことができます

pg_task_table.database1 = table3

デフォルトでは、スケジューラは1000ミリ秒ごとにタスクをチェックしますが、これは次のように変更できます

pg_task_period.database1 = 100
pg_task_period.database2 = 10

したがって、スケジューラは（まだ作成されていない場合）（必要に応じてスキーム）およびそのような列を持つタスクテーブルを作成します。

id BIGSERIAL NOT NULL PRIMARY KEY, -- ,  
dt TIMESTAMP NOT NULL DEFAULT NOW(), --     (- -   )
start TIMESTAMP, --    
stop TIMESTAMP, --     
queue TEXT NOT NULL DEFAULT 'default', --    (      )
max INT, --        (,      )
pid INT, --  ,   
request TEXT NOT NULL, --  SQL  
response TEXT, --   
state TEXT NOT NULL DEFAULT 'QUEUE', --   (- - ,    , , ...)
timeout INTERVAL, --     
delete BOOLEAN NOT NULL DEFAULT false, --     ,   
repeat INTERVAL, --   
drift BOOLEAN NOT NULL DEFAULT true --

実際、スケジューラはいくつかのバックグラウンドワークフローを開始します。1つは構成ファイルの変更を追跡し、必要に応じて、スケジューラの他のバックグラウンドプロセスを開始/停止します。また、各データベースの1つのバックグラウンドワークフローで、各データベースでスケジュールされたタスクを確認し、必要に応じてタスクを実行します。

たとえば、タスクをできるだけ早く完了させたい場合は、SQLコマンドを実行します

INSERT INTO task (request) VALUES ('SELECT now()')

スケジューラーは、タスクの結果をテキスト形式で結果列に書き込みます。タスクの実行の結果としていくつかの列がある場合、スケジューラーはそれらを列タイプとともにヘッダーに追加します。また、タスクの結果として、複数の行が存在する場合があり、それらのすべてが結果列に追加されます。

たとえば5分後にタスクを完了したい場合は、対応する列に予定時間を書き込みます

INSERT INTO task (dt, request) VALUES (now() + '5 min':INTERVAL, 'SELECT now()')

タスクを特定の時間に完了させたい場合は、それを書き留めます 

INSERT INTO task (dt, request) VALUES ('2019-07-01 00:00:00', 'SELECT now()')

タスクを5分ごとに実行する必要がある場合は、次のように記述します 

INSERT INTO task (repeat, request) VALUES ('5 min', 'SELECT now()')

もしそうなら 

INSERT INTO task (repeat, request, drift) VALUES ('5 min', 'SELECT now()', false)

その後、タスクはタスクの完了後5分間繰り返されます（デフォルトでは、スケジュールされた時間の後ではありません）。

タスクの実行中に例外が発生した場合、例外がインターセプトされ、テキスト形式で結果列に書き込まれ、対応する状態がタスクに割り当てられます。

例えば

INSERT INTO task (request) VALUES ('SELECT 1/0')

タスクキューで2つ以下のタスクを同時に実行する必要がある場合は、コマンドを使用してタスクを作成します 

INSERT INTO task (queue, max, request) VALUES ('queue', 2, 'SELECT now()')

このキューに多くのタスクが蓄積され、それらが2によって同時に実行されたとします。コマンドを使用してタスクを作成すると、 

INSERT INTO task (queue, max, request) VALUES ('queue', 3, 'SELECT now()')

その後、このキュー内の他のすべてのタスクができるだけ早く実行されます。それはまだ優先順位のようなものです
