レイアウト


レイアウトはコンポーネントのコンテナです。レイアウトは、内部のコンポーネントに関係なく、このコンテナを全体として配置できるので便利です。さらに、各シーンには1つのコンポーネントしか含めることができないため、シーンのルートコンポーネントとしてレイアウトが必要であり、シーンに必要なすべてのコンポーネントを配置できます。もちろん、通常、1つのレイアウトでは不十分ですが、1つのレイアウトを別のレイアウトの中に配置することができます。

これに加えて、レイアウトはコンポーネントを整理して配置します。使用されるレイアウトに応じて、子コンポーネントが配置される場合があります。

  • 横一枚ずつ
  • 縦に1つずつ
  • 積み重ね
  • グリッド内

さらに多くのオプションがあります。重要なことは、レイアウトのサイズが変更されると、レイアウトがその子の位置を自動的に更新することです。したがって、ユーザーがアプリケーションウィンドウのサイズを変更した場合でも、一貫したレイアウトを使用できます。

HBox


これは、利用可能な最も簡単なモックアップの1つです。それは単にすべてのオブジェクトを左から右へと次々と横一列に水平に配置するだけです。



FXMLでは、次のようにHBoxを使用できます。

<HBox>
    <Button>1</Button>
    <Button>2</Button>
    <Button>3</Button>
    <Button>4</Button>
</HBox>

Javaでは、次のコードを使用できます。

HBox hbox = new HBox();
Button btn1 = new Button("1");
Button btn2 = new Button("2");
Button btn3 = new Button("3");
Button btn4 = new Button("4");
hbox.getChildren().addAll(btn1, btn2, btn3, btn4);

間隔


これで、要素が次々ときれいに並べられ



ます。ただし、要素は隙間なく次々に配置されるため、このオプションはあまり適していません。幸い、コンポーネント間の間隔はHBacing spacingプロパティを使用して決定できます。

<HBox spacing="10">
     ...
</HBox>

またはJavaでsetSpacing（）を使用：

HBox hbox = new HBox();
hbox.setSpacing(10);

パディング


要素は正しく配置されましたが、要素とHBox自体の間にインデントはありません。HBoxにパディングを追加すると便利な場合



があります。各領域を指定して、上、下、左、右に個別に塗りつぶすことができます。

<HBox>
    <padding>
        <Insets top="10" bottom="10" left="10" right="10"/>
    </padding>
    ...
</HBox>

Javaでも同じことができます。

HBox hbox = new HBox();
hbox.setPadding(new Insets(10, 10, 10, 10));

Vbox


VBoxはHBoxによく似ていますが、内部コンポーネントを次々に水平に表示する代わりに、列に垂直に表示します



。HBoxと同じ方法で、間隔とパディングのプロパティを設定できます。

VBoxコードはHBoxとまったく同じ方法で使用されますが、名前だけが異なります。

<VBox spacing="10">
    <padding>
        <Insets top="10" bottom="10" left="10" right="10"/>
    </padding>
    <Button>1</Button>
    <Button>2</Button>
    <Button>3</Button>
    <Button>4</Button>
</VBox>

そしてJavaでは：

VBox vbox = new VBox();
vbox.setPadding(new Insets(10, 10, 10, 10));
vbox.setSpacing(10);
Button btn1 = new Button("1");
Button btn2 = new Button("2");
Button btn3 = new Button("3");
Button btn4 = new Button("4");
vbox.getChildren().addAll(btn1, btn2, btn3, btn4);

Stackpan


このレイアウトは、コンポーネントを互いの上に配置するのに役立ちます。挿入順序によって、要素の順序が決まります。つまり、最初の要素が一番下にあり、次の要素が一番上にあります。

これは、たとえば、レイアウトに画像を表示し、その上にテキストまたはボタンを表示する場合に便利です。



次の例では、FXMLでStackPaneを使用しています。

<StackPane>
    <ImageView>
        <Image url="/image.jpg"/>
    </ImageView>
    <Button>Click Me!</Button>
</StackPane>

同じJavaの例：

StackPane stackPane = new StackPane();
Image image = new Image(getClass().getResourceAsStream("/image.jpg"));
ImageView imageView = new ImageView(image);
Button btn = new Button("Click Me!");
stackPane.getChildren().addAll(imageView, btn);

要素の整列


スタック上の要素の配置を設定して、それらの配置をより適切に整理できます。

<StackPane alignment="BOTTOM_CENTER">
 ...
</StackPane>

もちろん、Javaでも同じことができます。

StackPane stackPane = new StackPane();
stackPane.setAlignment(Pos.BOTTOM_CENTER);

マージン


要素の配置をさらに詳細に制御したい場合は、スタック上の個々の要素にマージンを設定できます。

<StackPane alignment="BOTTOM_CENTER">
    <ImageView>
        <Image url="/image.jpg"/>
    </ImageView>
    <Button>
        <StackPane.margin>
            <Insets bottom="10"/>
        </StackPane.margin>
        Click Me!
    </Button>
</StackPane>

またはJavaで：

StackPane stackPane = new StackPane();
Button btn = new Button("Click Me!");
stackPane.getChildren().add(btn);
StackPane.setMargin(btn, new Insets(0,0,10,0));

フローパン


フローパネルは、水平（デフォルト）または垂直の2つのモードで機能します。

水平モードでは、HBoxのように、アイテムが水平方向に次々と表示されます。違いは、水平方向のスペースがなくなると、最初の行の下の次の行に転送され、再び続行されることです。したがって、HBoxのように1行だけでなく、多くの行が存在する可能性があります。

垂直モードは非常によく似ていますが、（VBoxのように）要素を上から下に垂直に表示します。スペースがなくなったら、彼は別の列を追加して続行します。

コンテナーのサイズを変更すると、コンポーネントの位置がどのように再計算されるかに注意してください。



このレイアウトの内部の塗りつぶしは、HBoxおよびVBoxの場合と同じ方法で設定できます。ただし、間隔プロパティの使用は少し異なります。要素は複数の行/列に表示される可能性があるため、間隔の単一のプロパティの代わりに、水平および垂直の間隔プロパティを個別に設定する必要があります。水平間隔プロパティにはhgapを使用し、垂直プロパティにはvgapを使用します。

FlowPane flowPane = new FlowPane();
flowPane.setOrientation(Orientation.VERTICAL);
flowPane.setVgap(10);
flowPane.setHgap(10);
flowPane.getChildren().addAll(...);

FXMLでのFlowPaneの例：

<FlowPane hgap="10" vgap="10" orientation="VERTICAL">
  ...
</FlowPane>

タイルパン


このレイアウトはFlowPaneによく似ています。コンポーネントの表示方法はほとんど同じです。引き続き水平モードまたは垂直モードを使用して、vgapおよびhgapを定義できます。

重要な違いの1つは、セルのサイズです。 FlowPaneは、各コンポーネントに必要なスペースのみを割り当てます。一方、TilePaneは、最大の要素に応じて、すべてのセルのサイズを同じにします。したがって、すべてのコントロールは行/列にうまく配置されます。



上の画像では、同じコンポーネントが同じ方法で配置されていますが、違いを簡単に確認できます。

FlowPaneは、余分な間隔なしで、コントロールを次々に配置します
TilePaneは、最大の要素に基づいて同じサイズのセルにコントロールを配置します。

TilePaneの作成は、名前以外はFlowPaneと同じです。

<TilePane vgap="10" hgap="10" orientation="VERTICAL" >

Javaでも同じこと：

TilePane tilePane = new TilePane();
tilePane.setVgap(10);
tilePane.setHgap(10);
tilePane.setOrientation(Orientation.VERTICAL);
