カリーニングラードでのDevFest 2019の初日からの11本のビデオ

9月28〜29日、第5回DevFestがカリーニングラードで開催されました。今日、彼らは彼らのYouTubeチャンネルにスピーカーの無料録音のために投稿しました。



カリーニングラードのDevFest 2019について



1.人工知能と人々の新しい科学。アレクセイシャグラエフ。Yandex Search




2.動いている羽ばたき。エフゲニー・サトゥロフ、サーフ


昨年以来、ヴォロネジスタジオサーフはクロスプラットフォーム開発に積極的に関心を持ち、Flutterに依存してきました。モバイルだけでなくWeb開発者にもフレームワークを検討する価値があるのはなぜですか？Flutterが魅力的であり、どこから始めればよいのでしょうか。



3.感情的知性。ミシャ・モルドビンツェフ、シデニス


DevFestは技術会議のようですが、感情的知性に関するレポートはありますか？Sidenis Mikhail Mordvintsevは、チーム開発の開発と開発者間のコミュニケーションの現在の段階では、合理的な側面だけでなく感情的な背景も考慮することが重要であると確信しています。あなたの感情を識別する方法、それらを管理する方法、他の人の感情を読み、正しく反応する方法を学ぶには？このレポートには、考察、科学、およびホリバーの場所があります。



4.成長する企業におけるコミュニケーションの進化。ドミトリー・マルシェンコ、Badoo


あなたの会社が1つのオフィスを占有し、すべての質問がその場で解決され、誰もコミュニケーションの難しさについて途方に暮れるような時間を逃しませんか？

はい、私はチームの成長に伴い、経営陣だけでなく、従業員も成功した相互作用に新たな障害に直面することを認めなければなりません。

バド出身のドミトリー・マルシェンコは、成長中の企業におけるコミュニケーション手法の進化に関する彼の見解と経験を共有しました。

オオバコの代わりに、チームとの会話に失敗した後、開発者の疲れた頭にレポートを適用する必要があります。



5.制限のないバックエンドアーキテクチャ。Semyon Krivosheev、ZeroHub


開発者はビジネス目標の達成にどのくらいの時間を費やしていますか？リトル。彼らがバグを修正したり、車輪を再発明したりする間、会社のリソースは溶けてしまいます。最初はパターンを1つだけ使用すると、プログラミング言語またはアーキテクチャは優れたソリューションのように見えますが、悲しい結果を招きます。

ZerohubのSemen Krivosheevは、暗号通貨市場での金融プロジェクトの作成における彼の経験を共有しました-これは最も難しいビジネス領域の1つです。彼らの開発では、彼らはバックエンド設計に混合アプローチを使用しています：マイクロサービス、モノリシックMVP、go / java / nodejs /何でも。



6.コレクションアンダーフード。クラスメート、ジョージ・エメリャノフ


標準のSwiftコレクションの分析で誰も驚かないでしょうが、場合によっては、開発者はコレクションと基本的なデータ構造がどのように機能するかを知っているだけでなく、独自のコレクションを作成してコードを改善できる必要があります。

彼のレポートで、ジョージは基本的なコレクションを実装するプロトコル、開発者が日常のタスクでそれらをどのように使用できるか、および配列、辞書、セット、レイジーなどのコレクションがどのように機能するかについて話しました。



7. Angularを使用して本番環境対応のPWAを構築します。ÖnderCeylan、KLM


ÖnderCeylanはオランダに住んでいて、忙しいスケジュールの中でGoogle Developer ExpertとしてProgressive Web Appsを宣伝する時間を見つけています。

PWAの実装方法-ガイドラインで自分で読むことができますが、アプリケーションをさまざまなプラットフォームで完全に機能させる方法は、専門家から直接学ぶ必要があります。


8.フロントエンドコード。Pavel Gerasimov、Wrike


開発プロセスの最も難しい部分は、新しい知識の獲得ではなく、それらの体系化と一般化です。1つの小さなチームでも、このプロセスを開始して推進することが難しい場合があります。

そして、あなたに加えて、コードが数十のコマンドで記述されていると想像してください。さまざまな種類のコードを取り除き、他のチームがクールに、等しく、そして等しくクールに書くように動機を与えるにはどうすればよいですか？「上から」ルールを課すのを避け、「下から」すべての開発者の意見を考慮する方法。

WrikeのPavel Gerasimovが、ナレッジベースを作成し、レビューで同じタイプのコメントを削除して、プロジェクトに参加するための基準を下げる方法についての実際の話を共有しました。



9.非同期の苦痛から観察可能なRedux。Yandex.Direct、Alexey Klester


私たち一人一人は常に非同期リクエストに直面しています。それらがなければ、最新のアプリケーションを作成することは不可能です。

単一のリクエストを管理するのは簡単ですが、関連する複数のリクエストが必要な場合、問題が発生する可能性があります。機能するコードを記述するだけでは不十分です。同僚がこのコードを理解することも必要です。Yandex.Directチームのメンバーも同じ問題を経験し、Reduxで観測可能なライブラリで救済を発見しました。



10. Kotlinマルチプラットフォーム。アレクセイミハイロフ、アイスロック


DevFestについて私たちが気に入らないのは、本番環境に関与したことのないテクノロジに関するレビューレポートです。これは、IcotRockのAlexei Mikhailovが1年間に数十の戦闘プロジェクトでKotlin / Nativeを使用し、チームの成果を共有した経験については間違いありません。

レポートでは、彼はフレームワークのアーキテクチャ上の特徴に触れ、チームが見つけたすべての落とし穴を導きました。賛美だけでなく客観的な批判もあります。「iOS開発者をクロスプラットフォーム側に切り替えるように説得する方法」という質問に対する回答もありました。



11. 2019年のAndroidアニメーション。AlexanderSmirnov、Raiffeisenbank


Androidは長年にわたって、アニメーションを操作するさまざまな方法を蓄積してきました。過去2年間で、アプローチは劇的に変化しましたが、実際にはすべてが優れているわけではありません。

RaiffeiseinのAlexander Smirnovによるレポートでは、複雑なアニメーションの開発と操作が簡単になるように、アニメーションの作成の選択にアプローチする方法を聞きます。アニメーションと制作の経験を共有し、各テクノロジーの詳細を確認してください。

MotionLayoutとCoordinatorLayoutについて詳しく知りたい場合は、このレポートが役立ちます。
