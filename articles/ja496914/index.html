最短の10進プログラム

1984年、スティーブンレヴィのカルトブック「ハッカー：コンピューター革命の英雄」がリリースされました。アマチュアのロシア語の翻訳がありますが、それは理想からほど遠いです。私はそれの不正確さを訂正することを約束し、英語の原文をその隣に置きました（ところで、それは罪なしではありません）が、第2章の後でそれを放棄しました。いずれにせよ、10進法で数値を出力するためのサブルーチン専用のフラグメント（別の記事で読むことができます）に注意を向けたいと思います。そのようなプログラムはどのくらい削減できますか？制限は何ですか？

2018年8月に、ソビエトプロセッサ1801BM1の命令の正確な実行時間を測定するためのプログラムを作成しました（PDP-11命令のセットがあります）。BK 0011Mコンピュータの「グッドアップル」デモで作業する場合、正確な時間（プロセッササイクル）を知る必要がありました。測定結果を10進数で見たかった。これを行うには、独自のサブルーチンを記述する必要がありました。テストの詳細により、システム関数は使用できませんでした。

最初に行ったのは、10の累乗のTEN配列を作成することでした。この手順では、レジスタR0の数値を受け取ります。出力はNUMBERのテキスト文字列です。重要：プロセッサには除算命令はありません！

	MOV #NUMBER,R1	; pointer to output text string
	MOV #TEN,R5
1:	CMP (R5)+,R0 	; skip leading zeros
	BHI 1		; branch if higher, for 16-bit not signed
	MOV -(R5),R3
	BEQ 4		; if less then 10

2:	MOV #47.,R4	; 0 symbol in ASCII codepage - 1
3:	INC R4		; count digits
	SUB R3,R0
	BHIS 3		; branch if higher or same, for 16-bit not signed
	MOVB R4,(R1)+	; print R4

	ADD (R5)+,R0
	MOV (R5),R3
	BNE 2

4:	ADD #48.,R0	; 0 symbol in ASCII codepage
	MOVB R0,(R1)+	; print R0
	CLRB (R1) 	; end of text marker
	RET

TEN:	.WORD 10000.,1000.,100.,10.,0

PDP-11アセンブラーを理解するには、引数が左から右に（最初にソース、次にレシーバー）書き込まれ、条件付きブランチコマンドが文字Bで始まる（ブランチ-ブランチから）ということを覚えておけば十分です。アルゴリズムについては説明しません。これは重要ではなく、ここでは開始点としてのみ示しています。このルーチンのサイズは22ワードです（データは含まれません）。

すべてがうまくいった後、スティーブン・レヴィの本の話を突然思い出しました。ハッカーは、同様のプログラムのサイズを減らすのに苦労し、PDPアーキテクチャーについても同様でした。確かに、彼らはPDP-1を持っていましたが、数年後、彼らはPDP-11を得ました。

のインターネットアーカイブを詳しく調べなければなりませんでした。興味深いものはたくさんあります：ミンスキートロン、マンチングスクエア、その他のいわゆる「ディスプレイハック」（ちなみに、それでも-60年代前半-MITハッカーは、現在使用しているのと同じ意味で「デモ」という用語を使用したことは注目に値しますデモシーンで）。アーカイブには多くのシステムルーチンがありますが、残念ながら、それらの間には10進数はありません。

次に、デバッガーで武装して、BK 0010およびBK 0011Mで使用するMKDOSオペレーティングシステムでこの手順がどのように実装されるかを確認することにしました。奇跡だ！ -よく見ると、サブプログラムが「Hackers」という本のぼやけた説明と非常によく合っていることがわかりました。これが彼女のコードです：

	MOV #10.,R4
	CLR R2
1:	CLR R1
2:	SUB R4,R0 	;    10,    
	BLO 3
	INC R1 		;  -   
	BR 2
3:	ADD R4,R0
	ADD #48.,R0 	; ASCII-  0
	INC R2		;  -   
	MOVB R0,-(SP)
	MOV R1,R0 	;    -   
	BNE 1
	INC R2
	MOVB #32.,-(SP)	; ASCII- 
4:	MOVB (SP)+,R0
	CALL PRINT
	SOB R2,4	;     
	RET

プログラムはスタック上にテキスト文字列を形成し、保存された各文字に対して印刷手順を呼び出します。どうやら、これはまさに「逆の方法で変換し、巧妙なソフトウェアフォーカスの助けを借りて、正しい順序で印刷する」というフレーズの下で、Stephen Levyの本で意図されたとおりです。アルゴリズムの他の機能は、コードのコメントから明確でなければなりません。

サブルーチンのサイズは23ワードですが、これを私のサブルーチンと直接比較することはできません。条件が多すぎます。プログラムをMKDOSから自分の条件、つまりメモリ内でのテキスト文字列の形成に作り直すことにしました。

結局、10を引くという考えだけを残し、残りをゼロから書く方がよいことに気付きました。Sansaraの最適化を数周した後、私は次の結果を得ました。

	MOV #NUMBER,R1	; pointer to output text string
	CLRB -(R1)	; end of text marker
	MOV #10.,R4
1:	MOV #-1.,R5
2:	INC R5		; counter of 10s
	SUB R4,R0
	BHIS 2		; branch if higher or same
	ADD #58.,R0	; #10. + '0' ASCII code
	MOVB R0,-(R1)	; store R0 to text string
	MOV R5,R0	; let's count next how many 10s in number of 10s
	BNE 1
	RET		; returns text string pointer in R1

16ワード、制限に達した（私は思った）、ここにあります-スティーブン・レヴィがとても感情的に書いたニルヴァーナ！

; RETURNS:
; R0 = 0
; R1 -> byte following last digit in converted number
CVBTOD:	MOV	R0,-(SP)	;SAVE THE NUMBER PASSED TO US
	CLR	R0		;SET FOR CRUDE DIVIDE BY 10.
10$:	INC	R0		;BUMP QUOTIENT
	SUB	#10.,(SP)	;REDUCE NUMBER BY 10.

20$:	MOVB	(SP)+,(R1)+	;STORE A CONVERTED DIGIT
	RETURN			;UNWIND THE RECURSION

スタック呼び出しをレジスター呼び出しに置き換えたとしても、ループ内のDEC R0とCALL命令により、DECプログラムは遅くなります。

しかし、それだけではありません。私がこの記事を書き始めたとき、私はプログラムに（MKDOSからの）基本的な命令MOV＃10.、R4があることに気づきました-それは内部ループを加速する以外に意味がありません。彼女を取り除く時が来ました。

さて、私の最適化のアイデアは終わりました。それは刺激的で、ギャンブルでさえも挑戦でした。 60年代前半に学生のハッカーによって提案されたPDP-1のアイデアがDECによって10年、さらには20年後に使用され、2000年代初頭までソビエトのコンピューターBK 0011Mに適用されたことは注目に値します。驚いたことに、2018年には、アルゴリズムを部分的に再発明して最適化することが可能でした。特徴として、多くの人がこれは不可能だと考えていました。

そこで、60年代のハッカーが見つけたホーリーグレイル（Stephen Levyによる）は、PDPの最短の10進プログラムです。それとも...もっと短いのでしょうか？

更新：この日が来ることは知っていましたが、すぐには考えていませんでした:)プログラムをもう1語減らすことができることがわかりました！アイデアはMr_Rmのコメントで提案されました：

	MOV #NUMBER,R1	; pointer to output text string
	CLRB -(R1)	; end of text marker
1:	CLR R5
2:	INC R5		; counter of 10s +1
	SUB #10.,R0
	BHIS 2		; branch if higher or same
	ADD #58.,R0	; #10. + '0' ASCII code
	MOVB R0,-(R1)	; store R0 to text string
	MOV R5,R0	; let's count how many 10s in number of 10s
	SOB R0,1	; subtract 1 and loop if not zero
	RET		; returns text string pointer in R1

14ワード。11の指示。
トリックはこれです。R5サイクルの「間違った」編成では、必要以上に1つ取得されることを覚えていますか。まあ、ましょう！プログラムの最後に削減します。以前のバージョンでは、R5はR0にコピーされ、その後、BNE（ゼロでない場合は分岐）コマンドがR0をチェックし、それがゼロに等しくない場合は、サイクルの最初に進みました。最初にR0を1だけ減らし、次に（ゼロにならなかった場合）最初に進みます。ちょっと待ってください。これは通常のSOB（1を減算して分岐）サイクルコマンドです。確かに、ここでは標準的な方法では使用されませんでした。一度満たすと、サイクルカウンターがほつれます。残念なことのように見えますが、以前のバージョンのサブルーチンを見ると、そこから新しい単語がどのように短縮され、1ワード短縮されているかが明らかになります。
