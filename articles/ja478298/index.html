初心者のみ：Web開発者として知っておくべき10のこと

こんにちは、ハブル！Anuupadhyayによる記事「Web開発者として知っておくべき10の事柄」の翻訳を紹介します。

数千行のコードを記述してWebサイトに変換することは、Web開発者にとって最もクリエイティブで複雑なことの1つです。あなたがこのビジネスに不慣れで、たくさんの美しいウェブサイトを見て、それを試してみることを考えているなら、私たちは目を開き、ウェブ開発者が必要とするいくつかのことについて話す必要があります。ユーザーの注意を引くWebサイトを作成することは、さまざまなプログラミング言語を学ぶだけでなく、DevTools、データ形式、テスト、API、認証など、その他の概念も模索しています。ここでは、Web開発者が習得する必要があるものの一部を示します。

1. HTML / CSS / JS


すべての開発者が最初に学ぶべき重要なスキルは、これらの3つの基本的な構成要素、つまりHTML、CSS、JavaScriptを理解することです。インターフェイスのフロントエンドでHTMLとCSSを使用します。Webブラウザーを右クリックし、ページソースを表示するオプションを選択します。多くのHTMLタグがさまざまな目的で使用されているサイトの構造を確認できます。

CSSは、スタイル、デザイン、レイアウト、およびHTML要素を画面に表示する方法を定義するWebインターフェースでも使用されます。

JavaScriptは現在非常に需要が高く、HTMLページを動的にする責任があります。JavaScriptには、サイトをよりインタラクティブにするために、PHP、Python、ASP.Netなどのさまざまな言語も付属しています。MEANスタックやMERNスタックなどのJavascriptを専門とする場合は、この言語が外部言語と内部言語になるため、この言語を詳しく調べる必要があります。

2. GitとGithub


Gitは、ほとんどの組織で使用されている最も人気のあるバージョン管理システムの1つです。ほとんどの場合、Web開発者としての仕事を得た場合、このバージョン管理に取り組む必要があります。このため、Gitの学習や、クローン作成、リポジトリーへの送信、取得リクエストの作成、ブランチのマージなどの基本的なコマンドを学ぶのに、ある程度の時間を費やす

必要があります。Githubは、Gitリポジトリーを使用してコードをホストできるサービスです。Githubはコラボレーションに使用されます。つまり、開発者が共同でプロジェクトに取り組むことができます。

3.開発ツール


デバッグ、HTML要素の編集、CSSプロパティの編集、デバイスのチェック、JavaScriptエラーの追跡など、開発ツールを使用して多くのことができます。すべての開発者は、さまざまなタブ（要素、コンソール、ネットワークなど）の使用に注意する必要があります。 ）DevToolsで、作業をより簡単かつ迅速に行うため。ブラウザーによっては、Chrome DevTools、Firefox DevTools、または使用している別のブラウザーなど、任意のDevTools を使用できます。人々はChrome DevToolsを使用してWebアプリケーションの開発、テスト、デバッグを行うことを好みますが、これもWebサイトの開発に使用するブラウザーの開発者の選択です。

4. API（アプリケーションプログラミングインターフェイス）


Web開発では、サードパーティのデータを処理するAPIを多く使用する必要があります。これにより、開発者はコードを共有せずに一部の機能を使用できます。さまざまな目的に使用できる素晴らしいGithub APIリポジトリがあり、プロジェクトに多くのアイデアを提供します。パブリックAPIリンクをチェックして調べることができます。

Rest APIの使用、HTTPリクエストメソッド（GET、POST、PUT、PATCH、DELETE）、Rest APIの作成、CRUD操作（作成、読み取り、更新、削除）、別のステータスコード、データ形式（ JSON、HTMLまたはXML）リクエストなどで使用されます。

5.認証


特定のWebサイトのユーザーを追跡するために、ユーザー認証を扱う可能性が高いです。たとえば、ユーザーにログイン、ログアウト、または個人アカウントからのリソースの作成、ユーザーが作成したリソースの追跡、またはログインしていないユーザーの一部のページのブロックを行う機能を提供します。ユーザーアカウントのセキュリティは、認証に大きく依存しています。したがって、Webアプリケーションでこのような機能を使用する方法を理解することが重要です。

ユーザーの認証を実装するには多くの方法があり、どのプログラミング言語またはテクノロジーが使用されているかによって異なります。WebインターフェースでReactを使用し、サーバー側でノードをExpressで使用する場合は、JWT（JSON Web Tokens）を認証に使用できます。Phpを使用する場合は、セッションとCookieを使用する必要があります。GoogleやログインするにはTwitter。したがって、認証の操作にはいくつかのモードがありますが、Web開発では、それを調査して実装することが重要です。

6. MVC（モデル、ビュー、コントローラー）


MVCは、アプリケーションを3つの異なるセクションに分割することで開発者の時間を大幅に節約する設計パターンです。MVCパターンを使用すると、開発が迅速かつ簡単になります。Laravel、Django（MVCに近いMVTに基づく）、Angularなどの多くの高レベルフレームワークは、MVCパターンに基づいています。MVCでは、モデルはデータベースとの対話に関連付けられており、ビューはユーザーが画面に表示するすべてのものを担当し、コントローラーはモデルとビューの間のインターフェースとして機能します。MVCを学ぶことで、プログラミング言語の基本を簡単に理解できるようになります。

7.プログラミング言語


コンピュータは人間の言語を理解しないため、プログラミング言語でシステムと通信する必要があります。ウェブ開発では、プログラミング言語だけでなく、幅広いプログラミング言語を使用して適用できるアルゴリズムも必要です。Web開発者は、PHP、Python、Java、Rubyなどのさまざまな言語を使用し、最近ではそれらと競合するJavaScriptも使用しています。Web開発者は、Webアプリケーションを作成するために少なくとも1つのプログラミング言語を知っている必要があります。

8.検索と問題解決


Web開発者になりたい場合は、問題とその解決策を見つける自然なスキルが必要です。このスキルは、いくつかのプロジェクトを使用して、データ構造とアルゴリズムを使用して、いくつかの問題、パズルなどを解決して開発できますが、選択したものは何でも練習し続ける必要があります。

初心者か経験者かに関係なく、Web開発者の役割では、アプリケーションを作成するときに、情報、構文、またはいくつかの問題の解決策を探す必要があります。おそらくあなたが疑わないいくつかの奇妙なバグに遭遇し、この問題を解決するためにGoogleまたはstackoverflowコミュニティに連絡する必要があります。

9.テストを書く


多くの開発者はこれを時間の無駄だと考えています。小さなアプリケーションの場合、テストの記述は無視できますが、大きなアプリケーションを作成する場合は、ユニットテストやその他のタイプのテストなどのテストを記述することをお勧めします。これにより、プロセスの信頼性が高まり、デバッグが容易になります。最初は時間を浪費しているように感じますが、後で実際に時間を節約できることに気づくでしょう。そのため、アプリケーションを作成するときにテストを作成することをルールにしてください。

10. DevOpsと展開


最後に重要なことですが、ウェブ開発者は、Google Cloud、AWS、Heroku、Netlifyなどのさまざまなプラットフォームでのコードの保守、スケーリング、移行、デプロイに関する知識が必要です。多くのオプションがあるため、これらのプラットフォーム、これらのサービスの仕組み、およびこれらのプラットフォームでのコードのデプロイまたは保守方法について学習するために、ある程度の時間を費やしてください。
