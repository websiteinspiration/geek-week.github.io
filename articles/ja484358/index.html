Rのポートフォリオ管理

オプションポートフォリオを分析および管理するための多くのプログラムがあります。それらは、サイト上の個別の商品またはサービスの形で、取引ターミナルにあります。このようなプログラムにはいくつかの制限があります。ポートフォリオは取引プラットフォームに結び付けられており、クオートは特定のソースからポンプで送られ、パラメーターまたはシナリオはソフトウェアによってのみ提供されます。


私はRを使用してさまざまな市場でポートフォリオを管理する問題を解決しました。オープンソースエンジンは多くの可能性を提供します。ポートフォリオは、DBMSまたはExcelに保存されるか、ターミナルからダウンロードされます（QUIK、TWS、APIを使用する他のもの）。引用はそのソース（端末、データベース、またはWebサイト）から汲み上げられ、ポートフォリオ分析が利用可能です！


概要


市場データとポートフォリオは、クラスOptPortとのオブジェクトに読み込まれますOptMarket。ポートフォリオは、これらのデータに基づいて市場ごとに再評価され、現在の利益と「ギリシャ語」が計算され、そのプロファイルも計算されます。ポートフォリオプロファイル（原資産の価格に対する利益または「ギリシャ語」の依存性）が計算され、クラスオブジェクトに格納されますOptProfile。組み込み関数はプロファイルグラフを描画し、複数のポートフォリオのプロファイルを比較できるようにします。


記事の最後にあるリンク（1）から-QUIKおよびAccessに接続して、Mosbirzhyのオプションのポートフォリオを管理するオプション。


OptMarket市場の相場


OptMarketオプションの株価、原資産に関する情報、現在の日付を保存するには、クラスオブジェクトが必要です。


市場価格がなければ、ポートフォリオは本質的な価値で再評価されます。


#    ( )  Access
brd  = GetBoard_fortsdb(ul)

#    OptMarket
mrkt = OptMarket(ul = 'SiZ9',                     #   
                 ul_price = brd$ul_price,         #   
                 board = as.tbl(brd$board),       #  
                 now   = as.Date('2019-11-12'))   #

新しいポートフォリオのプロファイルは、古いポートフォリオと比較できます。この関数JoinProfiles
はプロファイルデータを組み合わせて、共通のグラフを作成します。


#      
port2 = port2 %>% PortPricing(., mrkt, price_at = 'theor') %>% PortValuation(.)

#   
myprofile2 = OptProfile(optport  = port2, params = c('pl', 'delta','theta', 'vega'), ul_range = 0.05) 

#  
profiles = JoinProfiles(list(myprofile, myprofile2))

PlotProfile(profiles)
