首先，我们需要了解什么element.is_displayed()



应该工作，而无需滚动页面。这很重要，因为我们不想不必要地移动页面。



在这里，我们需要看一下元素本身。我们正在看CSS元素。



首先，让我们看看元素是否属于可访问性树。我们不看可访问性树本身，我们只是检查某些脚本的存在及其对页面上元素布局的影响。我们不会查看可访问性树，因为这对于浏览器来说是非常耗时的操作。



一个适合这种情况的简单方案是，一个元素具有一个元素display: none



，如下所示。 



#idOfElement {
    display: none;
}


这将导致以下事实：什么都不会添加到访问树，并且不会影响元素的排列。



接下来，如果需要，我们需要查看是否可以滚动到该项目。我们将简单地检查这是否可行，因为我们并不是真的想要滚动。这对于已移至页面上其他位置的项目很重要。这意味着我们不能总是依赖查看DOM树。我在2013年的上一篇文章中对此进行了讨论。

然后，我们继续检查项目的大小。对于大小为0的元素，我们说它们是不可见的。我们还检查它们的不透明度。如果不透明度为0，则认为该元素不可见。



完成这些测试之后，我们需要递归地遍历DOM并重复所有测试，直到达到为止documentElement

为什么要递归？



不幸的是，由于CSS，我们并不总是知道DOM处于哪个级别。我们需要检查返回途中的每个节点。这意味着它isDisplayed



可能会有点慢，但是可以很好地估计元素的可见性。



我们不能只问浏览器该元素是否可见吗？



抱歉不行。浏览器，选择要呈现的内容，创建显示列表，然后将它们发送到窗口管理器，以便完成所有艰苦的工作。过去，浏览器从不担心生成显示列表的效率。窗口管理器将尝试使显示列表更有效，然后显示它。精彩！



现在，关于显示列表-他们只是告诉窗口管理器在视口中可视化什么。视口是浏览器中页面的实际可见区域。我们想让isDisplayed()



我们知道是否将显示该项目。



我们可能还会遇到部分隐藏的元素。我们如何知道某个元素是真正可见的还是只有一小部分可见？在这里，我们可以使用一些Web API，例如document.elementFromPoint(x, y)



;。同样，这将仅告诉我们查看区域中的元素，即使在某些情况下，一个元素也可以覆盖另一个元素，但单击会穿过“隐藏的元素”。

第4集-寻找元素



在本集中，我们将研究其工作原理findElement



。首先，我们需要能够找到元素，然后才能与它们进行交互。



找到一件商品很容易，对吧？



好吧，是的，如果人们在开发应用程序时主要着眼于可测试性，则将其他一切推向后台。不幸的是，我们生活在一个世界上，在大多数情况下，人们都会将质量和测试视为次要的。



无论如何...让我们看看如何寻找他们。



搜索类型



这部分非常简单，Selenium提供了与通过JavaScript与DOM交互时可以找到的相同搜索方法。如果我们可以使用搜索项目，document.querySelectorAll(aQuery);



则可以通过find_element



或进行搜索find_elements



。可从WebDriver



或获得WebElement



。如果这样做element.find_element(...)



，则等价于element.querySelectorAll(aQuery)



，它将设置要在此元素上搜索的起点或根。



以下是使用示例find_element



： 



from selenium import webdriver
from selenium.webdriver.common.by import By

driver = webdriver.Firefox()
driver.get("https://www.theautomatedtester.co.uk")
driver.find_element(By.ID, "someID")
driver.find_element(By.CSS_SELECTOR, "#someID")
driver.find_elements(By.CSS_SELECTOR, "#someID")
driver.find_element(By.NAME, "someName")
driver.find_elements(By.NAME, "someName")
driver.find_element(By.CLASS_NAME, "someName")
driver.find_elements(By.CLASS_NAME, "someName")
driver.find_element(By.TAG_NAME, "someName")
driver.find_elements(By.TAG_NAME, "someName")


我们还可以通过XPATH搜索页面上的项目。

第4集-点击



在本集中，我们将介绍您单击时发生的情况。为了简单起见，我们将仅考虑使用进行的工作element.click()



。



您有要单击的元素，现在怎么办？



因此，您已经找到元素，并且需要单击它。 Selenium将在完成点击之前执行以下步骤。



检查页面上是否还有另一个元素



当我们找到元素时，浏览器返回该元素的表示形式。应该检查对该元素的所有后续调用，以确保该元素仍在页面上。如果不是这种情况，您将得到一个例外StaleElementReferenceException



。



检查元素的可见性



一旦我们确认该元素仍在页面上，就需要确保该元素已显示。这很重要，因为我们不希望用户单击不可见的项目。由于用户永远不会单击那里，因此我们必须确保Selenium也不会这样做。如果未出现，您将获得一个例外ElementNotVisibleException



。



滚动到该项目



接下来，我们需要确保可以滚动到该项目。为了正确启动所有事件，我们需要在查看区域中添加一个元素。如果尝试滚动到该项目后该项目不在视口中，则浏览器将抛出异常ElementNotInteractableException



。



检查项目是否互动



现在，浏览器将检查在我们所讨论的区域中将获得点击的元素是否与我们传递的元素相同。如果事实证明这不是正确的元素，则浏览器将抛出异常ElementClickInterceptedException



。



关闭一些事件



现在我们准备开始一些事件。我们有一个特殊情况，<



option>



以确保这是接收事件的父元素。



然后，我们计算元素的中心并开始发送事件。您可以阅读WebDriver规范中发送的事件。



导航呢？



我们需要告诉浏览器遵循单击所导致的导航。如果单击导致导航，我们将采用与导航自己相同的逻辑。
