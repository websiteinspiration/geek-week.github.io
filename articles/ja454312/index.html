AngularフレームワークでjQueryライブラリを使用する方法（本当に必要な場合）

blog.bitsrc.ioで公開されているAditya Modi記事の翻訳を紹介します。この記事では、AngularアプリケーションでのjQueryの使用について説明します。



まれに、私は慎重に何かを取り上げます。ただし、jQueryをAngularに統合するのはそれほど簡単ではないので、今回は注意が必要です。

なぜ、実際にこの記事を書いたのですか？ Angularでの開発にはjQueryが本当に必要な場合がありますが、そのようなJavaScriptフレームワークがこのライブラリに関連付けられることはほとんどありません。 AngularはDOMのレンダリングと管理を担当しているため、jQueryを使用してDOMに変更を加えると、非常に予期しない結果が生じるリスクがあります。

ヒント：Bitを使用すると、Angularコンポーネントをすばやく共有して、アプリケーションで再利用できます。これにより、あなたとあなたのチームは、新しいアプリケーションの作成に費やす時間を短縮できます。試してみる。



状況によっては、フレームワークへの干渉が必要であり、正当化される場合があるので、それは現実的な方法になる可能性があります。これらのケースは私のブログでは取り上げられていませんが、おそらく次の記事でそれらについて話します。次に、jQueryライブラリをAngularに追加する方法を説明します。このチュートリアルでは、VS Codeを使用します。他のコードエディターを選択するか、Windows、Linux、およびmacOS用のVisual Studio Codeをダウンロードできます。

AngularでのjQueryの統合


まず、コマンドを使用してAngularでアプリケーションを作成する必要がありますng new。次に、コマンドを使用してcd、jQueryライブラリがnpmからインストールされるフォルダーに移動します。結果のアプリケーションを呼び出しましょうangularmeetsjquery。

ng new angularmeetsjquery
cd angularmeetsjquery
npm install jquery — save

すばらしいように聞こえますが、jQuery純粋主義者は、npmではなくjQuery CDNを介してダウンロードすることを選択します。 CDNは、AngularアプリケーションにjQueryライブラリを追加するための代替方法ですが、私はnpmメソッドを使用することを好みます。https://jquery.com/download/を見ると、jQuery CDNライブラリと、Google、Microsoft、CDNJS、jsDelivrの他のCDNネットワークが見つかります。 CDNはSRI（Subresource Integrity）メカニズムをサポートしているため、最新の公式バージョンをお勧めします。

jQuery CDNを使用するにはscript、jQuery CDNドメインから直接タグ内のファイルへのリンクを作成します。サブリソース整合性関数を含むコードは次のようになります。 （ご覧のとおり、私はjQueryライブラリバージョン3.3.1を使用しています。）

<script
src=”https://code.jquery.com/jquery-3.3.1.js"
integrity=”sha256–2Kok7MbOyxpgUVvAk/HJ2jigOSYS2auK4Pfzbm7uH60=”
crossorigin=”anonymous”>
</script>

スクリプトは、code.jquery.comリソースからコピーできます。次に、それをファイルindex.htmlの< body> タグの最後に挿入する必要があります。

jQueryライブラリをグローバルにする


jquery.min.jsjQueryモジュールのdisフォルダーにある ファイルは公開されていません（フォルダーの正しい名前はdist。-Ed。です）。jQueryライブラリをグローバルにするには、ファイルに移動し.angular-cli.jsonて、jQueryファイルへのパスへのリンクを入れます。

Angularアプリケーション内のファイルパスへのリンクを転送する場合、ルートフォルダーはsrcです。ただし、jQueryライブラリはディレクトリ内にありnode_modulesます。したがって、ファイルへの正しいパスを決定する必要があります.angular-cli.json。

../node_modules/jquery/dist/jquery.min.js

パスが正しいことを確認するには、ディレクトリに移動する必要がありますnode_modules→jquery→dist→jquery.min.js。ここでは、指定されたパスを確認できます。これは、jQueryライブラリがこのアプリケーションに対してグローバルに追加されたことを意味します。ファイルがsrcフォルダーの外にあるため、ブラウザーのlocalhost：4200ページが自動的にリロードすることを期待しないでください。変更をアプリケーションに表示するには、アプリケーションを再起動する必要があります。

ng serve –open

AngularアプリケーションでのjQueryの使用


AngularフレームワークでjQueryライブラリの使用を開始するには、アプリケーションコンポーネントに移動する必要があります。これら2つのテクノロジーが単純なHTML要素でシームレスに機能する方法を紹介しますbutton。ただし、最初に、ファイル内のすべてのコード行を削除しapp.component.html、HTML要素を追加しますbutton。

<button> Click me </button>

ページが自動的に再読み込みされ、ボタンが表示されます。



ファイルでjQueryの文字を設定する必要がありますapp.component.ts。

declare var $: any;

次に、ライフサイクルフックを挿入しますngOnInit。これを行うには、Angular CoreからOnInitをインポートします。

import { Component, OnInit} from ‘@angular/core’;

を追加しngOnInitます。

export class AppComponent implements OnInit {

DOMの準備状況に関係なく、ngOnInit標準のWebアプリケーション開発で行われるように、メソッド内にjQueryコードを記述できます。で作成されたボタンを確認するには、メソッドにapp.component.htmlイベントbutton.clickを追加しngOnInitます。

export class AppComponent implements OnInit {
ngOnInit(){
$(‘button’).click(function(){
alert(‘Wass up!’);
 });
 }
}

ボタンをクリックすると「Wass Up」が表示されます。

結論


このバンドルを使用するだけでなく、AngularにjQueryを追加することは疑わしい考えです。

はい、一見すると方法は単純に見えます。ただし、この記事の冒頭で述べたように、これら2つのテクノロジの共同使用がいかに魅力的に見えても、そのアイデアはほとんどの場合悪いアイデアです。実際のAngularアプリケーションでは、イベント関連のHTML要素よりもはるかに複雑なものを処理する必要がありますbutton。Angularを使用していて、jScriptをTypeScriptコードに埋め込む機会を探しているすべての人への私のアドバイス：必要でない場合は、しないでください。jQueryとAngularの分野で成功した実験をお祈りします！
