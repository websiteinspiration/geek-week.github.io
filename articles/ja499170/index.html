GraphvizとC / C ++プリプロセッサの相乗効果

この記事では、C / C ++プリプロセッサ（以下、単にプリプロセッサと呼びます）と連携して人気のあるGraphvizグラフツールを使用する方法について説明します。


重要な点は、Graphvizで使用されるドットグラフ言語は、プリプロセッサによる処理に構文的に適しているということです。開発者が考え出したのです。彼らの洞察のおかげで、グラフを説明するときに次の機能を使用できます（Wikipediaのメモリから引用しています）。


  • 対応するダイグラフとトリグラフを同等の文字「＃」と「\」で置き換える。
  • エスケープされた改行文字の削除。
  • 行コメントとブロックコメントを空の行に置き換える（周囲のスペースとタブを削除して）。
  • 任意のファイル（#include）の内容の挿入（包含）。
  • マクロ置換（#define）;
  • 条件付きコンパイル（＃if、＃ifdef、＃elif、＃else、＃endif）;

次に、実際に開かれた機会を示します。例として、MediaStrimer2に関する私の記事のグラフを見てください。下の図のようになります。



グラフは十分に大きいので、「額に」それを記述すると、手動でエラーのない多くの作業が必要になります。よく見ると、一部のフィールドの内容のみが異なる繰り返し要素を分離できます。これは、グラフのノードがm 1、m2、m2_1、m2_2、m3、m4のように見える方法です。これらは、マクロを使用した小さな機械化の使用の最初の候補です。メインのドットファイルのヘッダーファイルを作成します。common.dotと呼びましょう：


//  common.dot       .

#define Q(x) #x         //    ""   .

#define QUOTE(x) Q(x)   //   ""    .

//         mblk_t.
//       . 
#define msg_staff \
   <p> *prev \
  |<n> *next \
  |<c> *cont  \
  |<d> *data \
  | other\nstuff

//        mblk_t.
#define msg(name, ... ) \
name[xlabel=mblk_t label=QUOTE(<h> name | msg_staff) \
];

//        dblk_t.
#define dbuf(name ...) \
name[label=QUOTE(<h> name) xlabel="dblk_t"];

//       queue_t.
//         mblk_t, 
//      msg_staff.
#define queue(name, ...) \
name[ xlabel="queue_t" label=QUOTE(<h> name | \
msg_staff)];

ここで、メイングラフファイルを作成します。my_graph.dotと呼びましょう：


//  my_graph.dot

//      .
#include "common.dot"

digraph queue_demo
{
    rankdir=LR;
    node[shape=Mrecord];

    //     ,
    //  ,       .
    dbuf(d1);
    dbuf(d2);
    dbuf(d2_1);
    dbuf(d2_2);
    dbuf(d3);
    dbuf(d4);

    //  ,      ,
    //     .
    msg(m1);
    msg(m2);
    msg(m2_1);
    msg(m2_2);
    msg(m3);
    msg(m4);

    //     .
    //     q1.
    queue(q1);

    //       .
    m1:d->d1;
    m2:d->d2;
    m2_1:d->d2_1;
    m2_2:d->d2_2;
    m3:d->d3;
    m4:d->d4;

    //     .
    m1:n -> m2:h;
    m1:p -> q1:h;
    m2:n -> m3:h;
    m2:c->m2_1:h;
    m2_1:c->m2_2:h;
    m3:n -> m4:h;

    m2:p -> m1:h;
    m3:p -> m2:h;
    m4:p -> m3:h;

    //      .
    q1:n->m1:h;  
    q1:p->m4:h;
    m4:n -> q1:h[color=blue]; //    .
}

ファイルをプリプロセッサで処理します。


cpp my_graph.dot -o my_graph_res.dot
