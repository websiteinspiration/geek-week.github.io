选择Docker Apache Kafka + Zookeeper映像


首先，您必须为我们的容器选择Apache Kafka映像供应商。每个特定项目的要求在解决方案的安全性和可靠性的必要级别上有所不同，当然，在某些情况下，您必须组装自己的映像，但是对于大多数项目，明智的是选择Docker集线器上可用的映像。对于2020年，有3个主要的热门供应商图像kafka + zookeper：

  • 融合图像 [主观上是一大优点-它们由Apache Kafka本身的开发人员提供，在信息安全方面是最可靠的]
  • 来自bitnami的图像 [此外，与融合图像相比，这些图像需要的配置极少，并且“开箱即用”]
  • wurstmeister [ , github hub.docker, ]

Docker-compose


为了启动和配置映像，使用了docker-compose，这使得运行多容器应用程序变得容易。您不能没有它，因为kafka的最低配置提供了一堆Zookeeper和至少一个代理。

ompose与Docker发行版mac / windows捆绑在一起，因此安装必需的环境取决于从docs.docker.com/engine/install下载和安装发行版。

但是linux用户需要单独安装它。有关docker + compose的安装指南，最好查阅官方的docs.docker.com/compose/install文档。

-d 标志使您可以在“分离”模式下启动，并在需要时关闭控制台，而无需关闭图像。
