認定されたKubernetes Application Developer（CKAD）試験に合格するための私の経験とヒント

最近、私は認定Kubernetes Application Developer（CKAD）試験に合格し、証明書を受け取りました。今日は、認証手続きとその準備についてお話したいと思います。私にとっては、審査官の綿密な監督の下でオンラインで試験に合格した興味深い体験でした。貴重な技術情報はなく、記事は本質的に純粋に物語です。また、私はKubernetesでの作業に大きなバックグラウンドがなく、同僚との合同トレーニングはありませんでした。私は自由な時間に自分で勉強し、トレーニングしました。

私はウェブ開発の分野ではかなり若いですが、DockerとK8の少なくとも基本的な知識がなければ、遠くに行くことはできないことにすぐに気付きました。コースに合格してこの種の試験の準備をすることは、私はコンテナとそのオーケストレーションの世界への良い入り口のように思えました。

それでもKubernetesが複雑すぎて自分に向かないと思われる場合は、カットをお願いします。

それは何ですか？


Cloud Native Computing Foundation（CNCF）によるKubernetes認定には2つのタイプがあります。

  • 認定されたKubernetesアプリケーション開発者（CKAD）-Kubernetesのクラウ​​ドネイティブアプリケーションを設計、作成、構成、および公開する機能のテスト。試験は2時間、19タスク、66％の合格点で終了します。基本プリミティブの非常に表面的な知識が必要です。費用は300ドルです。
  • 認定されたKubernetes管理者（CKA）-Kubernetes管理者の責任を果たすためのスキル、知識、能力をテストします。試験は3時間、24タスク、74％の合格点で終了します。システムの構築と構成に関する深い知識が必要です。費用も300ドルです。

CKADおよびCKA認定プログラムは、標準化されたトレーニングと認定を通じてKubernetesエコシステムを拡大するためにCloud Native Computing Foundationによって開発されました。この基金は、GoogleがLinux Foundationと提携して作成したもので、当初は技術的貢献としてKubernetesに移管され、Microsoft、Apple、Facebook、Cisco、Intel、Red Hatなどの企業によってサポートされています（c）Wiki

In ShortこれらはKubernetesによる「主要組織」の試験であると述べています。もちろん、他社からの認証もあります。

何のために？


これはおそらく、このすべてのベンチャーで最も論争の的になっているポイントです。私は、証明書の必要性についてホリバールを上げたくありません。そのような証明書を持つことが労働市場での私の価値にプラスの影響を与えると信じたいだけです。すべてが主観的です-あなたが仕事に連れて行く決断のターニングポイントとして何が正確に役立つかは決してわかりません。

トレーニング


CKADテストでは、トピックごとに次のように配布される19の質問があります。

  • 13％-コアコンセプト
  • 18％-構成
  • 10％-マルチコンテナポッド
  • 18％-観察可能性
  • 20％-ポッドデザイン
  • 13％-サービスとネットワーキング
  • 8％-州の存続

Udemyプラットフォームには、Mumshad Mannambethと呼ばれる1人のインド人からのシックなコースがあります（リンクは記事の最後にあります）。ほんの少しの料金で本当に非常に高品質の素材。特にクールなのは、コース中にテスト環境で実践的な演習を行うことが提案されているため、コンソールで作業するスキルを伸ばすことです。

コース全体を受講し、すべての実践的な演習を解きました（もちろん、解答をのぞかなくてはなりませんでした）。また、試験の直前に、すべての講義をより速く再検討し、最後の2つの試験（模擬試験）を繰り返しました。安定したペースで約1ヶ月かかりました。この資料は私が自信を持って試験に合格するのに十分で、結果は91％でした。1つのタスクで、どこかで間違いを犯しました（NodePortが機能しませんでした）。解決策はわかっていましたが、ファイルからConfigMap接続を使用して別のタスクを完了するには数分では不十分でした。

試験はいかがですか


試験は、Webカメラがオンで画面が共有されているブラウザで行われます。試験規則は、部屋に部外者を必要としません。すでに国で自己分離体制が導入されていたときに受験したので、妻が部屋に入らないようにしたり、子供が泣かないように静かな時間を見つけることが大切でした。時間の選択はあらゆる好みに利用できるので、私は深い夜を選びました。

最初に、審査官は写真と名前（ラテン語）を含むプライマリIDを表示するように要求します-私は外国のパスポートを持っていました。

試験中は、次のいずれかのリソースで別のブラウザタブを開いたままにできます：https : //kubernetes.io/docs/、https://github.com/kubernetes/またはhttps://kubernetes.io/blog/。私はこのドキュメントを持っていました、それで十分でした。

メインウィンドウには、タスクのテキスト、ターミナル、および審査官とのチャットに加えて、いくつかの重要な名前やコマンドをコピーできるメモ用のウィンドウもあります。

kudr run job1 --image=nginx --restart=OnFailure > job1.yaml
kudr run cronjob1 --image=nginx --restart=OnFailure --schedule="*/1 * * * * " > cronjob1.yaml
