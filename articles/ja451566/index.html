Operation TaskMasters：ロシアとCISのサイバーグループ攻撃組織をどのように公開したか



画像：Unsplash

2018年秋、PT Expert Security Centerの専門家は、機密文書やスパイ活動を盗むことを目的とした犯罪グループの活動を明らかにしました。今日は調査の進捗状況について話し、グループが使用した主な方法とツールについて説明します。

注：リンクは完全な調査レポートを提供します。また、攻撃の兆候を識別するために使用できる侵害の指標も提供します。

グループが攻撃しているのは誰で、いつ発見されたか


このグループは、2018年にPT Expert Security Centerの専門家によって特定されました。犯罪者は、タスクスケジューラでの特定のタスク（タスク）の作成に基づいて、インフラストラクチャのセキュリティを保護する通常とは異なる方法を使用しました。そのため、PT ESCはTaskMastersグループを呼び出しました。

タスクスケジューラを使用すると、タスクで指定された特定の時点でOSコマンドを実行し、ソフトウェアを実行できます。さらに、このグループ化で使用されるAtNowスケジューラを使用すると、ローカルだけでなくネットワーク上のリモートコンピューターでもタスクを実行でき、これらのノードの時間設定に関係なくこれを実行できます。また、このユーティリティをインストールする必要はありません。これらの機能は、攻撃の自動化を簡素化します。

ハッカーはさまざまな国の企業をハッキングし、ロシアとCISにはかなりの数の被害者がいます。攻撃された企業のほとんどは業界に起因している可能性があります。全体として、エネルギー、石油、ガスのセクターや政府機関など、さまざまな業界の30を超える組織の妥協を認識しています。

グループの主な目的は、機密情報を盗むことです。攻撃者は、企業の情報システムに足場を築き、企業の主要サーバー、トップの管理ワークステーション、および重要なビジネスシステムにアクセスしようとします。

インフラストラクチャ内のグループの存在の最も初期の痕跡は2010年にさかのぼります。当時、犯罪者はすでに一部のサーバーとワークステーションを完全に制御していたため、侵入ははるかに早く発生しました。

攻撃者の行動


一般的な攻撃ベクトルはかなり予測可能です。ローカルネットワークに侵入した後、攻撃者はインフラストラクチャを調べ、システムの脆弱性（CVE-2017-0176など）を悪用し、侵害されたノードにダウンロードしてユーティリティのセットを展開します。このセットを使用して、関心のあるファイルを検索、コピー、およびアーカイブしてから、管理サーバーに送信します。

ネットワークをナビゲートするために、犯罪者はAtNowユーティリティを使用してリモートノードでシステムコマンドを実行します。これにより、ソフトウェアを実行し、指定した時間間隔の後にコマンドを実行できます。ノードを管理サーバーに接続するための小さなバックドアを使用してノードを管理します。同時に、Exchangeサーバーなどの外部リソースにアップロードされたWebシェルの形式のバックアップチャネルがあります。



攻撃スキーム

グループは、ドメインに動的DNSインフラストラクチャを使用します。攻撃者は、多数のユーティリティとツールを使用してサイバー攻撃を行い、サプライチェーン攻撃スキームを積極的に使用します。

ネットワークをスキャンしてシステムを危険にさらすために、攻撃者は無料ソフトウェア（NBTScan、PWDump、Mimikatzなど）を使用します。サードパーティツールに加えて、自己開発プログラムも使用されます。
