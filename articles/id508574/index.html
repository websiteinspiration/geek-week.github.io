Menyebarkan dan mengonfigurasi otentikasi simpul-merah pada susunan docker



Menyebarkan node-red ke docker-compose dengan otorisasi diaktifkan dan menggunakan volume buruh pelabuhan.



Buat file docker-compose.yml: 



version: "3.7"

services:
  node-red:
    image: nodered/node-red
    environment:
      - TZ=Europe/Moscow
    ports:
      - "11880:1880" # 11880 -     , 1880 -     node-red  .
    volumes:
      - "node-red:/data" # node-red -    docker   , /data -   .
    restart: always
volumes:
  node-red: #   node-red  .


Kami meluncurkan wadah dalam mode debug (peluncuran pertama harus tanpa kunci ` -d` , untuk melihat kesalahan ketika muncul): 



$ docker-compose up node-red
Creating node-red_node-red_1_3e3e59f5e044 ... done
Attaching to node-red_node-red_1_bca4cb987984
node-red_1_bca4cb987984 |
node-red_1_bca4cb987984 | > node-red-docker@1.0.3 start /usr/src/node-red
node-red_1_bca4cb987984 | > node $NODE_OPTIONS node_modules/node-red/red.js $FLOWS "--userDir" "/data"
...


Kami menghentikan wadah dan menjalankan perintah untuk melihat volume: 



$ docker volume ls
DRIVER              VOLUME NAME
local               node-red_node-red


Lihat informasi terperinci tentang volume: 



$ docker volume inspect node-red_node-red

    {
        "CreatedAt": "2020-05-02T18:37:33Z",

Pergi ke direktori volume. Direktori ini sudah berisi file-file yang dibuat buruh pelabuhan saat menggunakan wadah. 



$ sudo ls /var/lib/docker/volumes/node-red_node-red/_data
lib  package.json  settings.js


Kami tertarik pada file settings.js . Buka dan cari sepotong kode adminAuth . Batalkan komentar itu. 



 // Securing Node-RED
    // -----------------
    // To password protect the Node-RED editor and admin API, the following
    // property can be used. See http://nodered.org/docs/security.html for details.
    adminAuth: {
        type: "credentials",
        users: [{
            username: "admin",
            password: "$2a$08$zZWtXTja0fB1pzD4sHCMyOCMYz2Z6dNbM6tl8sJogENOMcxWV9DN.",
            permissions: "*"
        }]
    },
