前の記事で 述べたように、MotionJobクラスと、Motionクラスから継承された派生クラスは、ロボットの動きの実装を担当します。MotionJob

クラスには、指定されたモーションを起動して実行する public executeメソッドが1つだけあります。

int MotionJob ::実行（モーション* pモーション）
int MotionJob::execute(Motion* pMotion) {
  if(m_Status == NONE && pMotion != NULL) {
     
     m_pMotion = pMotion;
     m_MotionTime = 0;
     m_TotalTime = m_pMotion->totalTime();
     m_Status = RUNING;
  } else if(pMotion != NULL){
    
    if(m_pMotion->m_pNext != NULL) 
      delete m_pMotion->m_pNext;
      
    m_pMotion->m_pNext = pMotion;
     m_Status = STOPING;
  }
  return S_OK;
}


そのロジックはかなり単純です。パラメータとして、移動のプロセスを説明するオブジェクトへのポインタを受け取ります。現時点で実行中に他の動きがなかった場合、指示された動きがアクティブになり、実行状態になります（RUNING）。実行時にすでに何らかの動きがあった場合、それを停止状態（STOPING）に変換し、新しい動きは次の動きとしてm_pNextフィールドに記録されます。

重要：Motionから継承されたすべてのオブジェクトは動的メモリ領域での作成を意味し（newを介して）、移動が完了またはキャンセルされた後にMotionJobクラスによって解放されることに同意します。

TransPoseMotionクラス


TransPoseMotion クラスを使用すると、四肢をある位置から別の位置に移動できます。オブジェクトを作成するとき、ポイントの座標と移動が実行される期間がクラスコンストラクターに転送されます。

トランスポーズモーション::トランスポーズモーション（...）
TransPoseMotion::TransPoseMotion(Vector3D begPose[FOOTS_COUNT],
  Vector3D endPose[FOOTS_COUNT], long maxTime, Motion* pNext) : Motion(maxTime, -1, pNext) {
  for(int i = 0; i < FOOTS_COUNT; i++) {
    m_Pose0[i] = begPose[i];
    m_Offset[i] = endPose[i] - begPose[i];
  }
}


各肢の初期位置と変位ベクトルがわかっているので、単純な線形計算を実行することで、0からmaxTimeまでの時間間隔でgetPoseメソッドで足の新しい位置を簡単に取得できます。

int TransPoseMotion :: getPose（...）
int TransPoseMotion::getPose(long time, Vector3D points[FOOTS_COUNT]) {
  for(int i = 0; i < FOOTS_COUNT; i++)
    linearFunc(1.0, m_Offset[i], m_Pose0[i], 1.0*time/m_MaxTime, points[i]);
}


線形計算は、関数の引数を介してパラメーターが指定された通常の線形方程式である補助関数linearFuncに割り当てられます。

void linearFunc（...）
void linearFunc(float a, Vector3D offset, Vector3D p0, float k,  Vector3D& res) {
  res = a*offset*k + p0;
}

LinearMotionクラス


単純なモーションを実装するには、もう少しコードと注意が必要です。ヘキサポッドのデザインには最適な数の四肢があり、移動時に常に3点の支持点を確保できるため、ロボットの重心の投影は常にそれらによって形成される三角形の内側に配置されます。これにより、移動時のロボットの位置が安定します。したがって、ロボットの直線運動は、同一のステップのセットと見なすことができます。このような各ステップでは、2つのフェーズを区別できます。最初のフェーズでは、1つのトライアドの足がサーフェスに接触し、その端がサーフェスと平行に、移動方向と反対の方向に移動します。この場合、2つ目の足の3つ組は、表面に触れることなく、ロボットの移動方向に湾曲した経路に沿って移動します。ある時点で、それらは極端な位置に達し、2番目のトライアドが表面に接触し始めます。 2番目のフェーズでは、動きのプロセスが繰り返されます。今回のみ、2番目のトライアドがロボットを動かし、最初のトライアドが開始点に戻ります。



このアニメーションは、このページのインターネットで記事を作成しているときに見つかりましたが、レビューしたところ、その元のソースはhabrであり、出版物の著者は尊敬されているconKORDであると確信しました。

直線運動を実現する場合、我々は、表面（オフ脚の高さを設定する必要がheigth移動（の開始（および終了時）で、）ロボットの手足の位置をpose0）、速度ベクトル（速度）、それは時間を要し、完全な1ステップ（MAXTIME）、そしてどのくらいロボットべき移動を実行します（totalTime）。totalTimeに負の値を指定した場合、動きはロボットの充電がなくなるか、動きが強制的に完了するまで無期限に続きます。

ステップ全体を4つの四半期に分割します。

  1. 第1クォーターでは、四肢の最初の3つ組（左前、左後、右中央）がプッシュ動作を実行します。つまり、表面に沿って反対方向に移動します。足の2つ目のトライアド（右前、右後、左中央）は移動方向に移動しますが、立ち上がりの高さは、正弦の関数として、脚の指定された高さ（四分の一の始め）からゼロ（四分の一の終わり）まで変化します。
  2. 第2四半期には、トライアドの方向が変わります。これで、2つ目の足のトライアドがロボットを動かし、最初の足がロボットと同じ方向に動きます。この場合、最初のトライアドの足の高さは、正弦波でゼロから最大値まで変化します。
  3. 第3クォーターでは、足の移動方向が維持され、最初のトライアドの端が下がり、第3クォーターの終わりで水面に到達します。
  4. 最後の四半期では、最初のトライアドがサポートになり、2番目のトライアドが方向を変えて正弦波に沿って上向きに移動し、クォーターの終わりの最大高さに達します。

    rotFunc(2.0, m_Angles, m_Pose0[RIGTH_BACK_FOOT_IDX], k - 0.5,

    int h = (int)m_Heigth*cos(M_PI*(k - 0.5)*2);
    points[LEFT_FRONT_FOOT_IDX].z += h;
    points[LEFT_BACK_FOOT_IDX].z += h;
    points[RIGTH_MIDLE_FOOT_IDX].z += h;
  } else { //   
    rotFunc(2.0, m_Angles, m_Pose0[LEFT_FRONT_FOOT_IDX], k - 1,
      points[LEFT_FRONT_FOOT_IDX]);
    rotFunc(2.0, m_Angles, m_Pose0[LEFT_BACK_FOOT_IDX], k - 1,
      points[LEFT_BACK_FOOT_IDX]);
    rotFunc(2.0, m_Angles, m_Pose0[RIGTH_MIDLE_FOOT_IDX], k - 1,
      points[RIGTH_MIDLE_FOOT_IDX]);
      
    rotFunc(2.0, m_Angles, m_Pose0[LEFT_MIDLE_FOOT_IDX], 1 - k,
      points[LEFT_MIDLE_FOOT_IDX]);
    rotFunc(2.0, m_Angles, m_Pose0[RIGTH_FRONT_FOOT_IDX], 1 - k,
      points[RIGTH_FRONT_FOOT_IDX]);
    rotFunc(2.0, m_Angles, m_Pose0[RIGTH_BACK_FOOT_IDX], 1 - k,
      points[RIGTH_BACK_FOOT_IDX]);

    int h = (int)m_Heigth*sin(M_PI*(k - 0.75)*2);
    points[LEFT_MIDLE_FOOT_IDX].z += h;
    points[RIGTH_FRONT_FOOT_IDX].z += h;
    points[RIGTH_BACK_FOOT_IDX].z += h;
  }
  return S_OK;
}
