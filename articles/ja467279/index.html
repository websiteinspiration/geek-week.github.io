UEFA国別比率の計算


サッカーには2つの人気のあるトーナメントがあります。チャンピオンズリーグとヨーロッパリーグです。その結果に基づいて、いわゆるサッカー協会のレーティングが計算されます。この評価に基づいて、各国からのチームがその後のトーナメントに参加する数がさらに決定されます。

この記事では、この評価を計算するオープンで無料のlsFusionプラットフォームに基づいたアプリケーションを作成します。すべてのデータをPostgreSQLに保存し、フィルタリングおよびソート機能を使用してデータを変更および表示するためのWebインターフェイスを提供し、特別なAPIを使用して一致結果をインポートします。

このアプリケーションを実装するためのすべてのコードは、約300の重要な行で構成されます。

ドメインロジック


情報システムの作成は、ドメインロジックのタスクから始まります。
まず第一に、コードと名前のみを持つ最も単純なディレクトリを選択するのが論理的です。

  • トーナメント。チャンピオンズリーグまたはヨーロッパリーグ。
  • シーズン。2018-2019 / 2017-2018など
  • ラウンド。決勝、準決勝、グループステージなど これはトーナメントの構成と見なすことができますが、この実装では、別のエンティティーとして選びました。
  • 国。このアプリケーションは、サッカー協会として使用されます。たとえば、モナコのクラブはモナコの国にありますが、フランス選手権でプレーします。
  • クラブ。バルセロナ、レアルマドリード、マンチェスターユナイテッドなど

lsFusionは同じタイプのロジックを使用してそれらを宣言するため、対応するロジックを生成するメタコード（またはコードテンプレート）を宣言します。

ディレクトリ宣言メタコード
META defineMasterObject(object, caption, captions, nameLength)
   CLASS ###object caption;
  
   id '' = DATA INTEGER (###object);
   object (INTEGER id) = GROUP AGGR ###object o BY id(o); //    
  
   name '' = DATA ISTRING (###object) CHARWIDTH nameLength;
   object (ISTRING name) = GROUP AGGR ###object o BY name(o); //    
  
   FORM object caption
       OBJECTS o = ###object PANEL
       PROPERTIES(o) id, name
      
       EDIT ###object OBJECT o
   ;
  
   FORM object##s captions
       OBJECTS o = ###object
       PROPERTIES(o) READONLY id, name
       PROPERTIES(o) NEWSESSION NEW, EDIT, DELETE
   ;
  
   FORM dialog###object##s captions
       OBJECTS o = ###object
       PROPERTIES(o) READONLY id, name
      
       LIST ###object OBJECT o
   ;
  
   NAVIGATOR {
       NEW object##s;
   }
END


彼は発表します：

  • 指定された名前のクラス
  • 新しいクラスのコードと名前を持つプロパティ
  • 3つのフォーム：オブジェクトの編集、すべてのオブジェクトのリストを含むフォーム、ナビゲーターに追加されるフォーム、このオブジェクトを選択するためのダイアログ。対話として2番目のフォームを使用できますが、ユーザーは選択時にオブジェクトを変更する機会があり、ユーザー側でエラーが発生する可能性があります。

4つのパラメーターがメタコードに渡されます。

  • 識別子（オブジェクト）。この名前でクラスとフォームが作成されます。###構成は、識別子の最初の文字を結果のコードで大文字にするために使用されます。
  • 単数形の名前。クラスとフォームのタイトルに使用されます。
  • 名前は複数形です。コンボボックスとダイアログに使用されます。
  • 名前の長さ。異なるオブジェクトの名前では、異なる長さが予想されます。これは、インターフェースを構築するときに重要です。

作成したメタコードを使用して、上記の5つのエンティティを追加します。
@defineMasterObject(tournament, '', '', 20);
@defineMasterObject(season, '', '', 5);
@defineMasterObject(round, '', '', 15);
@defineMasterObject(country, '', '', 10);
@defineMasterObject(team, '', '', 20);

データのインポート


残念ながら、すべてのEurocupsをサポートするパブリックで無料のAPIを1つだけ見つけることができました。これがFootball APIです。ただし、問題があります。

  • 2016年まで結果はありません。
  • 2018年までヨーロッパリーグの資格はありません。
  • データに特定のエラーがあります。たとえば、Irtysh Pavlodarはロシアに割り当てられていますが、このクラブはカザフスタンを表しています。また、何らかの理由でエウロパFcはジブラルタルではなくスペインを指します。


データエラーは、以前に作成したフォームを使用して手動で修正できます。ただし、合計係数の計算は過去5年間に基づいているため、残念ながら、APIフットボールデータから合計係数を計算することはできません。コメントの誰かが、必要なデータをどこからでも、前の年の形式でどこから入手できるかを提案した場合、私は非常に感謝します。ただし、2018年の完全なデータがあるため、少なくとも今年の計算の正確さを検証することが可能になります。

必要なAPIはHTTPリクエストの形式で実装されており、パラメーターはURLを介して送信され、特別なアクセスキーがヘッダーに示されています。対応するロジックを宣言します。
host = 'api-football-v1.p.rapidapi.com';
key '  API Football' = DATA STRING () CHARWIDTH 50;

url = 'https://' + host() + '/v2';

headers(TEXT name) = CASE
   WHEN name = 'x-rapidapi-host' THEN host()
   WHEN name = 'x-rapidapi-key' THEN key();

すべてのデータインポートアクションは、以前に作成されたリーグフォームに配置されます。そこで、リーグのリストを含むテーブルのツールバーにアクセスキーを配置します。

つまり、tournamentName（0）の値は「ワールドカップ」になり、tournamentName（1） -「プレミアリーグ」になります。

残念ながら、API Footballにはトーナメントエンティティはありません。すべてのリーグをリンクする唯一の方法は、異なるシーズンの同じトーナメントのリーグに一致する名前を付けることです。これを行うには、インポートで、最初にインポートしたリーグのすべての名前をグループ化し、データベースにない場合は、新しいトーナメントを作成します。
FOR [GROUP SUM 1 BY tournamentName(INTEGER i)](STRING tn) AND NOT tournament(tn) DO NEW t = Tournament {
   name(t) <- tn;
}

FOR leagueId(INTEGER i) AND NOT league(leagueId(i)) DO NEW l = League {
   id(l) <- leagueId(i);
   tournament(l) <- tournament(tournamentName(i));
   season(l) <- season(seasonName(i));

係数計算


UEFAの国別係数の計算に直接進みます。このために、特別にインストールされたUEFAモジュールにすべてのコードを配置することは論理的です。

まず、Football APIはEurocupsだけでなく、すべての試合をインポートするためのインターフェースを提供することに注意してください。そのため、トーナメントの名前に応じてヨーロッパカップの試合を分けます（これには個別のプライマリプロパティを設定する方が適切ですが、プロパティの実装は、残りのロジックを変更せずにいつでも変更できます）。
isCL (Tournament t) = name(t) = 'Champions League';
isEL (Tournament t) = name(t) = 'Europa League';

isUL (Tournament t) = isCL(t) OR isEL(t);
isUL (Match m) = isUL(tournament(m)); 

まず、特定の試合の結果について、各クラブがシーズン中に受け取るポイントを計算してみましょう。
この期間中、各チームは以下を受け取り
ます。勝利の場合は2ポイント。
同点の場合は1ポイント。
1999年以降、これらのポイントは、予選ラウンドで獲得された場合、2つに分割され
ます。つまり、勝利の場合は1ポイント。
引き分けの場合は0.5ポイント。


試合とクラブの関係を決定する補助的なプロパティを作成します。
played (Team t, Match m) = homeTeam(m) = t OR awayTeam(m) = t;
won (Team t, Match m) = (homeTeam(m) = t AND goalsHome(m) > goalsAway(m)) OR (awayTeam(m) = t AND goalsHome(m) < goalsAway(m));
draw (Team t, Match m) = played(t, m) AND goalsHome(m) = goalsAway(m);

各試合で得点がいくつあるかを判断するために、ラウンドの数値タイプのプライマリプロパティを追加します。これは、デフォルトでは1に等しくなります。
dataMatchCoeff = DATA NUMERIC[10,1] (Round);
matchCoeff ' ' (Round r) = OVERRIDE dataMatchCoeff(r), 1.0;

次に、勝利と抽選のポイントを数え、合計します。
wonPoints '  ' (Season s, Team t) =
   GROUP SUM 2 * matchCoeff(round(Match m)) IF won(t, m) AND season(m) = s AND isUL(m);
drawPoints '  ' (Season s, Team t) =
   GROUP SUM 1 * matchCoeff(round(Match m)) IF draw(t, m) AND season(m) = s AND isUL(m);
matchPoints '  ' (Season s, Team t) = wonPoints(s, t) (+) drawPoints(s, t) MATERIALIZED;
