Untruncビデオファイルの「moov atom not found」などのエラーを回避して情報を保存する方法

この記事は、Windows 10 Cameraアプリケーションからビデオファイルを再生できなかったすべての人を対象としています。UPD1。この記事は、カメラ、カムコーダー、レコーダー、および多数のウェブカメラにも対応しています。UPD2。この記事の著者は、記事に示されているバグ（アプリケーションによってファイルが閉じられなかった）を解消するための情報を2年間欠落しています。

記事の作成者は、Windows 10 Cameraアプリケーションが数年間実行されていたときに、「誤って」ラップトップのバッテリーを失い、レポートの資料がゴミ箱に捨てられました...



ビデオデータのサポートが必要な人は誰でも、猫にお願いします。

この記事の一般的な場所は、ffmpegユーティリティを使用することです。このユーティリティの機能について理解することを強くお勧めします。

作業はUbuntu 16.04で行われます。このユーティリティはその中にあります。ただし、新しいバージョンを確認することを忘れないでください。さらに、カードやハードドライブで作業する場合、破損したファイルシステムとファイルを復元するときに追加の問題があります。これに関する専門的な記事を読むことも良いです。ファイルを入手できると信じています。

1.まず、libuiライブラリをUbuntu 16.04にインストールします。原則として
、コマンドラインからの操作に慣れている場合は、アクションをスキップできます。

$ # need pip3 to install meson
$ sudo apt install python3-pip libgtk-3-dev ninja-build
$ # need meson 0.48 or higher to build libui
$ sudo -H pip3 install meson
$ git clone https://github.com/andlabs/libui.git
$ cd libui
$ meson setup build # [options]
$ ninja -C build
$  ./build/meson-out/controlgallery
$ sudo ninja -C build install

2. untruncユーティリティをその指示に従って配置します。

sudo apt-get install libavformat-dev libavcodec-dev libavutil-dev
# get the source code
make
sudo cp untrunc /usr/local/bin
sudo apt-get install libva-dev liblzma-dev libx11-dev libbz2-dev zlib1g-dev yasm pkg-config wget
make FF_VER=3.3.9
sudo cp untrunc /usr/local/bin

必要に応じて、インターフェースを接続します。

make untrunc-gui

。GUIなしでインストールされている場合は、

./untrunc /path/to/working-video.m4v /path/to/broken-video.m4v

実用的な観察と警告。

  • Windowsではこれを行わないでください（可能ですが）、コーデックズーは、特定のファイルタイプのバリエーションを物理的に操作できないことがあります。
  • Ubuntu 16.04のリポジトリにはffmpegユーティリティがあります。必要に応じて、より新しいバージョンを使用できます。
  • ここで使用する「良いビデオ」は、これらの形式のコーデックを使用して同じデバイスで撮影し、10分長くする必要があります。
  • 復元されたファイルの名前はWIN_20200204_17_08_00_Pro.mp4_fixed-s2.mp4で、fixedという単語と、起動が行われた一連のキーがあります。元の不良ファイルよりも大幅に小さくしたり大きくしたりすることはできません。
  • 良いビデオがない場合、少なくとも1分はかかりますが、残念なことに、ほとんどの場合、音声ストリームしか得られません。ああ...ユーティリティは全能ではありません。

このテクノロジーは、ビデオファイルを閉じることができない他の多くのデバイスやアプリケーションで機能します。

貴重な情報の復元に大成功！
