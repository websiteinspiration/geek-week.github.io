レポートのタイトルで「マージナルケース」とはどういう意味ですか。極端な値に到達するまで一連のオプションを反復すると、制限的なケースが生じます。たとえば、n辺のポリゴン。 n = 3の場合、これは三角形、n = 4は四角形、n = 5は五角形などです。nが無限大に近づくと、辺がだんだん大きくなり、多角形の輪郭が円のようになります。したがって、円は通常のポリゴンの制限的なケースです。これは、あるアイデアが限界に達したときに起こります。


C ++の極端なケースとしてPythonについて説明したいと思います。C ++から優れたアイデアをすべて取り入れて、それらを論理的な結論にまでクリーンアップすると、一連のポリゴンが円になるのと同じくらい自然にPythonが完成するはずです。


非中核資産


90年代にPythonに興味を持つようになりました。私が言うように、「コア以外の資産」を取り除くのは私の人生のそのような時期でした。多くのことが私を退屈させ始めました。たとえば、中断。多くのコンピューターボードでジャンパーとの接触があったことを覚えておいてください。また、マニュアルにこれらのジャンパを設定して、ビデオカードがより高い優先度の割り込みを受け取り、ゲームがより速く実行されるようにしますか？そのため、ジャンパーを使用してコンピューターのパフォーマンスの調整をやめたのと同じ頃に、メモリの割り当てと解放にうんざりしmalloc()てfree()いました。 1997年くらいでした。


つまり、私たちがプロセスを研究するとき、私たちは通常、可能なすべてのレバーとボタンを手元に用意するために、プロセスを完全に制御できるように努力します。それから、何人かの人々はまだこの制御の可能性に魅了されています。しかし、私の性格は、管理に慣れ、何が何であるかを理解したらすぐに、自分の能力の一部を放棄し、レバーとボタンをいくつかのマシンに移して、割り込みが割り当てられる機会を探し始めていることです。


したがって、90年代後半には、コンピューターのメモリのどの領域にデータが保存されているのか心配するのではなく、主題領域とタスクモデリングに集中できるプログラミング言語を探していました。有名なスクリプト言語の罪を繰り返さずに、C ++を単純化するにはどうすればよいでしょうか。
