ナビ


これは、コマンドライン操作でチートシートを表示および保存するためのツールです。 UNIX / Linuxステートメントのリストを表示し、必要な引数を提案します。自分でテンプレートを作成するか、ダウンロードすることができます。

$ navi search <cmd>


/ナビのしくみ：公式リポジトリの gif-image

開発者によると、1つのコマンドの結果をクリップボードにコピーして次のコマンドに貼り付ける必要はありません。 Naviはシェルのウィジェットとしても機能します。ただし、これまでのところ、zshおよびbashシェルでのみ使用できます。他人をサポートする問題は、GitHubの特別なスレッドで決定されます-誰でも接続できます。

ナビは多くの類似体を持っている-例えば、仲間、 例えば、  cheat.sh、  tldr、  cmdmenuや チートを。ハッカーニュースレジデントも指摘そのような機能はすでにLinuxに組み込まれています。特に、彼らは〜/ binディレクトリを念頭に置いており、そこには自己作成スクリプトが配置されています。ただし、naviの作成者は、選択肢には明確なUIがなく、必要な引数を自動的に置き換えることができないと述べています。

このツールは、katacodaでオンラインでテストできます。プラットフォームは、Dockerコンテナーを起動し、Naviを開始するためのすべての指示を提供しますが、登録が必要です。

Z


このシェルスクリプトを使用すると、コマンドラインでディレクトリを簡単に移動できます。これを使用すると、cdとls（dir）を使用してディレクトリを切り替えることはできません。このツールは、アクセスしたディレクトリを記憶し、それらに特別な重み係数-frecency（英語の「最近」と「頻度」から）を割り当てます。フォルダを開くと、「最近訪れた」というフラグを受け取りながら、周波数応答が1つ増加します。その後、フォルダの名前をユーティリティに転送するだけで、自動的に開きます。

//    /var/www/staging/wp-content/themes :
$ z themes

//      :
$ z staging themes

ディレクトリの頻度係数の合計が9000を超えると、それらすべてに0.99が乗算されます。その結果、係数が1未満のフォルダーは忘れられます。

今日、ZはGitHubに1万を超える星を持っていますが、その人気にもかかわらず、欠陥があります。Hacker Newsの居住者は、アナログと比較して、ツールの機能が制限されていることに注意してください。たとえば、fasdユーティリティは、ディレクトリ名だけでなくファイル名によっても遷移を実行します。標準ツール$ CDPATH-頻繁にアクセスするフォルダーのリストを使用して、同様の機能を実現できることにも注目してください。ただし、この場合、ディレクトリは手動で入力する必要があります。

コウモリ


UNIXライクなシステムでは、catを使用して、ファイルの内容を順次読み取り、標準出力に送信します。猫には類似物-バットがありますが、構文の強調表示とgitとの統合が追加されています。ユーティリティは、余暇にコウモリに取り組んだドイツの物理学者、David Peterによって開発されました。

/ GitHubのBatリポジトリのスクリーンショット

Batは、C＃、C ++、Erlang、Go、Groovyなどのプログラミング言語の構文を強調表示できます。次のコマンドを書くことで、完全なリストを知ることができます。

$ bat --list-languages

番号が付けられている左側の列には、最後のコミット以降のファイルの変更（追加された行、名前の変更など）も表示されます。 batのもう1つの機能は、自動スワッピングのサポートです。結論をページごとに順番に読み取ることができます。

一般に、ツールは肯定的なレビューを収集しました。彼の周りには、GitHubで15,000を超える星を含む大規模なコミュニティさえ形成されました。しかし、それでもコウモリに感謝しない人がいました。彼らによれば、このようなユーティリティは逆行する - UNIXの概念にいずれかを行いシングとそれを行うまあ（「プログラムは、一つのことを行う必要があり、それがうまくやる」）、したがって、バットの機能は、「冗長」と考えられていたが。バットの責任の下でのタスクは、決定します他の一連のツール-たとえば、テキストファイルを解析するためのawkと、ソースを変更した後にプロジェクトをビルドするためのentr（1）。
