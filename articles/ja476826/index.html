複数の実験：理論と実践

今日の世界では、A / Bテストなしで製品開発を想像することは困難です。製品または新しい機能を正常に起動するには、A / Bを正しく設計し、その結果を計算して解釈する必要があります。3つ以上のグループのテストが必要な場合があります。この記事では、このようなケースについて検討します-複数のテスト：

  • 複数のテストを実施する時期と理由について説明します。
  • テスト結果を計算するための主な方法と、その方法の基礎となる数学的原理を検討します。
  • メソッドのソフトウェア実装の例を示します。これらの例をプロジェクトで使用できます。

それでは始めましょう。



複数の実験：いつ、なぜ


明らかに、実験の技術的な複雑さは実際の必要性によって正当化されるべきです。これは複数のテストにも当てはまります。オーディエンスが3つ以上のサブグループに分かれている場合、実験中に第1種のエラーが発生する確率は指数関数的に増加します。

1–(1–α)n,


どこ n -サブグループの数、 α-所定のレベルの統計的有意性。
したがって、通常のペアテストにサブグループを1つだけ追加すると（n=3）特定の標準レベル α=0.05 第一種のエラーの確率を得る p=0.14目標を大幅に上回っています α=0.05。

研究結果の精度が低下する場合、なぜ複数の実験を行うのですか？いくつかの理由が考えられます。

  1. いくつかの変更と、それらが製品メトリックに及ぼす累積的な影響をテストする必要があります。例としては、サービスページで、相互に異なる位置にある2つの新しい要素をユーザーに示しています。
  2. 変更は相互に依存しており、週ごとの季節性の影響を受けやすいため、1つの期間でのみテストできます。たとえば、広告チャネルを無効にして、チャネル間の共食いの影響を計算します。
  3. 顧客は、できるだけ早く、より安価に回答を得たいと考えています。どちらのオプションを選択するかによって、実験の開発と実装を同時に節約できます。

これらのタスクの1つに直面し、テストの統計的有意性を計算する必要がある場合、複数のテストの修正の必要性を考慮する必要があります。それが何であるか、そしてそれをどのように正しく行うかについて、そして以下で議論されます。

複数の実験：計算機能


基本概念


対処する際は、一般的なケースを検討してください n 仮説 H0i、 i=1,…,n 中央値または平均のペアワイズ等価について mサブグループ。この場合、正しい結果と誤った結果の両方が考えられます。H0i VS H1i それぞれの i=1,…,n仮説。結果を混同行列実験として提示します。

したがって、誤って拒否されました V の R基本的な仮説を拒否しました。

これらのパラメーターに基づいて、複数のテスト中に制御されるエラーの2つの重要な概念を紹介します。FWER そして FDR。

グループ誤り確率FWER （Family-Wise Error Rate）は、第1種のエラーが少なくとも1つ発生する確率であり、次の式で決定されます。

FWER=p(V>0).



FDR （False Discovery Rate）は、主な仮説の偏差の総数に対する第1種のエラーの比率の数学的期待値です。

FDR=E(V/R|R>0).



標準的な製品ケースの例を使用して、これらのエラーを制御する方法を検討してください。

ケースの説明


簡単な例として、3つの分離されたばらばらのユーザーグループに3つのページオプションが表示され、広告の[呼び出し]ボタンをクリックする提案があった実験を考えてみます。計算を簡単にするための基本的なメトリックとして、各グループの合計コール数を使用します。

FWERコントロール


このエラーを制御するには多くの方法がありますが、2つが最も一般的です

。1）同時調整を伴う1ステップの手順p−valueBonferroniメソッドを使用してテストされたすべての仮説。

2）順次、反復調整p−value c Hill法の結果に従って、すべてのステップで意思決定を行います。

1.ボンフェローニ修正


この1ステップの手順により、偽陽性の実験結果の可能性が減少します。この方法の本質は、次の場合に対立仮説を受け入れることです。

p≥α/n,


どこ n-テストされた仮説の数。

このメソッドは、標準ライブラリを使用して非常に簡単に実装できますbootstrapped：

from bootstrapped import bootstrap as bs
from bootstrapped import compare_functions as bs_cmp
from bootstrapped import stats_functions as bs_st


bs_ab_estims = bs.bootstrap_ab(np.array(group_A), np.array(group_B), 
                               bs_st.mean bs_cmp.difference, 
                               num_iterations=5000, alpha=0.05/3, 
                               iteration_batch_size=100, scale_test_by=1, 
                               num_threads=4)

bs_bc_estims = bs.bootstrap_ab(np.array(group_B), np.array(group_C), 
                               bs_st.mean bs_cmp.difference, 
                               num_iterations=5000, alpha=0.05/3, 
                               iteration_batch_size=100, scale_test_by=1, 
                               num_threads=4)

bs_ac_estims = bs.bootstrap_ab(np.array(group_A), np.array(group_C), 
                               bs_st.mean bs_cmp.difference, 
                               num_iterations=5000, alpha=0.05/3, 
                               iteration_batch_size=100, scale_test_by=1, 
                               num_threads=4)

統計的評価の結果が得られたら、グループが異なるかどうかを結論付けることができます。

アプローチの主なマイナス点：サブグループが多いほど、基準の検出力が低くなり、誤った仮説を受け入れる確率が高くなります。たとえば、10個のテストとαi=0.05 得る必要がある pi≤5·10−3違いが大きいと言って。これらの欠点を軽減するために、Hillメソッドを選択できます。

2.ヒル法


これは、下方への順次変更手順です。 p−value。メソッドアルゴリズムの最初のステップでは、実際p−value 昇順で並べ替え：

p1≤···≤pn,


その後、オリジナル α-レベル：

αi′=α/(n–i+1),


その後、状態がチェックされます pi≥αi′ そして、主要な仮説が真であるかどうかを結論付けます H0i。

アルゴリズムのブレークポイント-最初の主要な仮説が受け入れられる瞬間iH0i、およびその後のすべて H0j,j>i。
このメソッドは、プロシージャを使用して実装できますmultipletests() 図書館から statsmodels パラメータ付き method=”holm”：

from bootstrapped import bootstrap as bs
from bootstrapped import stats_functions as bs_st
from scipy.stats import ttest_ind
from statsmodels.sandbox.stats.multicomp import multipletests

bs_a = bs.bootstrap(np.array(group_A), stat_func=bs_st.mean, 
             num_iterations=10000, iteration_batch_size=300, 
             return_distribution=True)


bs_b = bs.bootstrap(np.array(group_B), stat_func=bs_st.mean, 
             num_iterations=10000, iteration_batch_size=300, 
             return_distribution=True)

bs_c = bs.bootstrap(np.array(group_C), stat_func=bs_st.mean, 
             num_iterations=10000, iteration_batch_size=300, 
             return_distribution=True)


stat_ab, p_ab = stats.ttest_ind(pd.DataFrame(bs_a), pd.DataFrame(bs_b))

stat_bc, p_bc = stats.ttest_ind(pd.DataFrame(bs_b), pd.DataFrame(bs_c))

stat_ac, p_ac = stats.ttest_ind(pd.DataFrame(bs_a), pd.DataFrame(bs_c))

print(sorted([p_ab, p_bc, p_ac]))

print("FWER: " + str(multipletests(sorted([p_ab, p_bc, p_ac]), alpha=0.05, 
                     method='holm', is_sorted = True)))

FDRコントロール


コントロール FDR 条件が満たされていることを意味します FDR=E(V/R)<α。どこでFDR≤FWER、つまり、制御で1次エラーが発生する確率 FDR 衰退。

ベンジャミニ・ホッホベルク法


このボトムアップ手順には、順次変更が含まれます。 p−value事前にソートされた昇順：

p1≤···≤pn.


次にソース αレベルは次の式で調整されます。

αi′=i·α/n,


次に、ホルム法と同様に、条件がチェックされます pi≥αi′ そして、主要な仮説が真であるかどうかを結論付けます H0i 以降のすべて H0j,j>i。

ホルム法のようなベンジャミニホッヒベルグ法は、次の手順を使用して実装できます。multipletests()：

print("FDR: " + str(multipletests([p_ab, p_bc, p_ac], alpha=0.05, 
                    method='fdr_bh', is_sorted = False)))
