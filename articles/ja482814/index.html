ツリーの一般的なビュー、実装だけでなく

多くはおそらく一般的なツリーの構造を見つけようとしましたが、検索エンジンはバイナリツリーのみを見つけました...バイナリ検索ツリー、バイナリツリーのトラバース、および他の多くのアルゴリズム。
はい、確かに、一般的なツリーはどこでも使用されていません。バイパスは遅く、ユースケースは小さいです。

それで、私はこの質問をしました、そして今、私はツリーがどのように構築されているかを説明します。したがって、理想的には、一般形式のツリーの構造は3つの変数を格納する必要があります。

  • 長男へのポインター
  • 兄弟へのポインター
  • 保存するデータ 

struct Tnode {
    int key;
    struct Tnode *son;
    struct Tnode *brother;
};
typedef struct Tnode Node;

上へのポインタを宣言します。

Node *tree = NULL;

これはバイナリツリーではなく、各頂点は任意の数の息子を持つことができるため、頂点の入力方法を事前に合意する必要があります。

  • + 2（または+ ssbb 2）-ツリーへの挿入（一般的なツリーの場合、パスは行で指定されます。ここで、rはルート、sは長男への遷移、bは兄弟への遷移です）。

例を挙げましょう： 

+r 1
+ 2
+ 3
+ 3
+s 5
+sb 6
+sb 7

その結果、このツリーはリリースされます： 

1
  2
    5
  3
    6
    7
  3

まず、頂点を追加する関数を作成します。つまり、頂点にメモリを割り当て、この頂点へのポインターを渡します（最初は何にも関係ありません）。

Node *create_tree(int v) {
  Node *Tree = (Node *) malloc(sizeof(Node));
  Tree->key = v;
  //     ,  ,   value
  Tree->son = NULL;
  Tree->brother = NULL;
  return Tree;
}

パス文字列（+ bs ...）を処理する関数も作成する必要があります。ルートから走査を開始するたびに、それが作成されていない場合は、NULLを出力します（何もできません）。ピークがない場合は、作成する必要があります。ツリー作成関数に移動し、ルートへのポインターを取得します。

注：ノード**ツリーはコピーではなく構造を渡します。これにより、Node *ツリー宣言とは異なり、実行できないことを変更できます。
