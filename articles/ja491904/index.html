安全なPythonコードの書き方。食事クシャルダス

ここで、オリジナルの英語版このインタビューのは。

今年、講演者の会社モスクワPython会議++は必要なもの（つまり、どのようにして、プログラム委員会がピックアップしたか）を取り上げました。しかし、業績を研究することに興味がある人は、話者が私たちに関係する問題について何を考えているかははるかに興味深いです。知るには、インサイダー情報または経験豊富な開発者のアドバイスを入手し、会議でコミュニケーションする必要があります。しかし、私はその状況を利用して、講演者のクシャルダスから短いインタビューを受けました。

Kushalのスピーチの際立った特徴は、Pythonコードを解読する「秘密の」方法を定期的に公開しているのとは対照的に、NSAがクラックできないようにコードを記述する方法を示しています。私たちの会議で私は食べました教えてくれます、私はセキュリティについて尋ねたので、安全に、Pythonのコードを開発し、展開する方法。



-私は食べました、あなた自身について、そしてあなたがPythonなどを使った仕事について少し教えてください。

Kushal Das：私はインドに住んでいます。現在はアメリカの非営利団体であるFreedom of the Press Foundationで公益技術者として働いており、SecureDropプロジェクトのサポートを手伝っています。SecureDropはオープンソースの情報プラットフォームです。ここでの主な言語はPythonです。

さらに、Python言語自体を含む他のオープンソースプロジェクトにも関わっています。私はCPython Core開発者の1人であり、Python Software Foundationの理事会のメンバーです。

私は2005年からPythonを使用しており、私のキャリアのほとんどすべてはLinuxだけでなく、この言語にも関係しています。これらは私の興味の2つの主要な分野です。私はTorプロジェクトコアチームのメンバーでもあります。多くの人がすでに知っているように、Torはあなたのプライバシーをオンラインに保つのを助けます。

-Torは素晴らしいプロジェクトです。世界中の開発者が、たとえば地域の法律によりAPIやドキュメントへのアクセスをブロックするために使用しています。プレゼンテーションはセキュリティを中心に構築されるため、いくつか質問があります。

まず、特別なトレーニングを受けていない一般の開発者にとって、セキュリティ自体は複雑すぎるという意見があります。私たちが業界として、私たちが作成するソフトウェアがデフォルトで安全になるように、ツールとエコシステムを提供する必要があると考えられています。

セキュリティの人々を教育したり、ツールを提供したりするのに、どちらが良いと思いますか？

ダスを食べた：両方とも思う。実際、セキュリティは複雑な問題です。ただし、初心者プログラマーが基本的なトレーニングを受けたり、チームで作業したりする場合、彼は最も一般的な問題を回避することを学びます。

ほとんどの場合、ハッキング、データ漏洩、データ盗難などの実際のセキュリティエラーは、構成の誤り、更新の欠如、または誤った、場合によってはフーリガンコードが原因で発生します。

業界の初心者と「昔ながら」の両方を訓練すれば、基本的なミスのほとんどは回避できると思います。さらに、人間の目にアクセスできないものは、新しいツール、新しい自動化、新しいプロセスの助けを借りて明らかにすることができます。

非常に単純な例は、GitHubが現在持っている依存関係チェックです。 Pythonアプリケーションにエラーが含まれているか、独立したモジュールの古いバージョンが使用されており、CVEに記載されている特定の脆弱性がある場合、GitHubは開発者として次のように伝えることができます。ここでアップデートが必要です！」 -そしてボットは自動的にPRを実行してモジュールを更新しようとします。

したがって、トレーニングとツールを組み合わせるのが最適ですが、個人的には、できるだけ多くのトレーニングを希望します。データを入力して間違いをする人です。テクノロジーはそのようなエラーをすべて修正することはできません。

-はい、残念ながらそうです。最新のソフトウェア開発は、Python Package Index、node.js、npmなどのオープンリポジトリに大きく依存しています。現在の開発レベルでは、これは通常、pip install またはpoetry add として発生します。

あなたの意見では、中規模企業の通常のPythonバックエンド開発者がこれらのツールのみを使用する場合、セキュリティリスクはどのくらい高いですか？そのようなリスクは許容できますか、それとも開発者は些細な脆弱性を考慮する必要がありますか？

Kushal Das：これは本当に重要な質問です。サポートされているすべてのリポジトリで、さまざまな言語のモジュールをダウンロードできるため、開発者がこのプロセスを簡単にアクセスできるようにするためです。

たとえば、Pythonの場合、読み込まれた最新のPythonモジュールに関する問題について警告するPythonセキュリティメーリングリストがあります。パッケージの名前が非常によく知られているパッケージに似ていて、開発者が誤って間違ったパッケージを使用している、いわゆるタイプミスキャットのケースがあります。 Python側では、PyPIがそのようなことを処理します。

さらに、pipを使用してパッケージをインストールするときは、実際にそれをチェックして、これが必要なものであることを確認し、ダウンロードのセキュリティをチェックできます。たとえば、PEP 458があります。これは、PyPIパケット署名でダウンロードを保護することをお勧めします。

安全機器の開発作業はまだ進行中です。新しい未署名のモジュールで作業を開始する開発者は、作成者が誰であるか、開発に関与している人の数、それらがホストされている場所、使用されている大規模なプロジェクト、ソースコードに何があるか、問題やエラーに関する不満などがないかを調べることをお勧めします。

これは15〜30分で検索できますが、多くの利点があります。作者以外の人がこのパッケージを使用していない場合は、おそらく使用しないでください。この特定のモジュールが他のパッケージまたはソフトウェアで使用されている場合は、コードに対してより安全である可能性が高くなります。

NPMはデータのセキュリティをチェックし、結果を表示します。たとえば、次のメッセージが表示されます。「インストールしたばかりのパッケージの中に、セキュリティリスクの高いパッケージが2つ、平均的なリスクのパッケージが10個あります。次のコマンドを実行して、それらを修正または更新するか、インストールを拒否できます。これは組み込みツールに似ています。そのようなソリューションについてどう思いますか？また、PyPIについても同様のことを期待できますか？

Eaten Das： NPMを使用したことはありませんが、何かをテストする必要があるときに、人生で1〜2回しか使用しませんでした。したがって、これがどれほど優れているかについて直接コメントすることはできませんが、開発者により直感的なユーザーインターフェイスを提供することのようです。

一般的に、これはすべてユーザーインタラクションのストーリーの一部であり、開発者とデザイナーのタスクは、最終的な開発者、つまりこれらすべてのツールとモジュールをインストールするサードパーティによって実際に表示されるという事実を考慮して、データを提示する最良の方法を特定することです。というのも、情報が多すぎる場合は、クリックせずに読んだりスキップしたりするからです。真実はその中間にあります。

Pythonとpipの開発の将来の計画については、コメントすることはできません。ただし、pipの作成者が計画で考慮できるように、誰かが既に同様の機能のリクエストを送信していることを願っています。現在、サポートチームは以前より少し大きくなっています。約7人がコミットを受け入れる権利を持っています。

-ありがとうございました。時間をかけて、人々がセキュリティについてもっと考え、私たちのエコシステムでより多くのチェックを構築することを願っています。Moscow Python Conf ++での今後のパフォーマンスについて少しお話ししましょう。組み込みの依存関係セキュリティとパッケージングを中心に構築されています。

ここでは台無しにはしませんが、セキュリティの専門家の観点から、あなたの観点から、開発者がアプリケーションのセキュリティをチェックするときに他に何を考慮すべきですか？

依存関係の処理とパッケージを保護する方法についてはすでに説明しました。通常の開発者が他に何をチェックする必要がありますか？何を探すべきですか？

私はダスを食べました：私は主要なポイントの1つがアップデートであるべきだと思います。少なくともこれで注文していることを確認してください。Linux、Mac、Windows、またはiOSであっても、OSを含め、依存するすべてが更新されます。または、少なくともプロセスをモデル化して、他に何が問題になるのかを理解してください。

ほとんどの場合、初心者（およびこれは私にも起こりました）の問題は、ユーザーの入力を盲目的に信頼することです。入力が実際に正しく、これが他の問題を引き起こさないという事実にあまり依存する必要はありません。すべてを再確認する必要があります。

また、何かを開いた場合は、忘れずに閉じてください。。普通のドアを想像してみてください。オフィスに入るときに、後ろのドアを閉めるのを忘れたかどうかを確認します。また、プログラミングでは、ファイル、ソケット、その他の何かを開くとき、または何かにアクセスするときに、完了後にすべてがクリーンでクローズされていることを確認する必要があります。

プログラミングの観点から、これらは多くの人が忘れている2つの主要なポイントだと思います。

-はい、人々が忘れていることがたくさんあります、そして今、私は最後の質問だと思います。あなたの個人的な経験に基づいて、Pythonの中間レベルのバックエンド開発者が最もよく行うセキュリティエラーはどのようなものですか？ユーザー入力？中毒？安全なアプリケーションのパッケージ化？より一般的なものは何ですか？

Eaten Das：私が見た問題は、主に間違った入力によるものでした。

2011年、私はFedoraプロジェクト用のツールを開発していて、一時ファイルを削除するのを忘れていました。この特定のケースでは、これらはダンプであり、新しい環境にそれらが存在しないことで、インフラストラクチャに予期しない問題が発生しました。

これは同じトピックの続きであり、何かを作成したり開いたりした場合にコードが「クリーンアップ」されていることを確認する必要がある場合です。だから私にとっては真剣な教訓でした。 「うん、やったよ！すべてが私のラップトップで動作します！」、それが実際のリリースまたは製品で動作することは事実ではありません。

どういうわけか、私たちはしばしば環境は私たちのローカルの環境と同じになると思いますが、それは決して起こりません。大企業でさえ、クライアント側の実際の環境を判断することが難しい場合がよくあります。それを覚えておくことが重要です。

そしてさらに -コードにコメントを追加し、ドキュメントを作成します。このヒントは、初心者向けだけでなく、経験豊富な開発者向けです。数か月または数年コードを調べなかった場合、なぜ何かを書いたのかを理解するのは非常に困難です。
