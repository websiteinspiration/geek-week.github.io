開発者のIoT識別

数日前、GlobalSign社はIoT開発者ポータルの開発者向けにポータルを開設しました。ここでは、無料で登録（企業の住所でのみ）、統合用のAPIを選択、X.509証明書を無料で取得して、IoT認証システムをテストできます。これまでのところ、GlobalSignは、公開鍵基盤（PKI）に基づくデバイス識別子を使用してIoTデバイスに証明書を発行する唯一の認証局であり続けています。

要点を簡単に説明すると、新しいポータルは集中化されたロケールであり、デバイスアイデンティティの統合に必要なすべてのツールへのデモアクセスが提供されます。統合ガイド、テストツール、製品仕様、コード例、RESTful APIなどです。

誰が必要ですか？

ポータルは、開発段階で識別システムをテストするのに最適です。ここでは、実際にすべてが確実かつ安全に機能することを確認できます。そして、実際の商用システムでのAPIへのアクセスと自動証明書管理にかかるコストを見積もる。

センターのAPI


インターネットに接続されたデバイスは、大量のデータを生成します。たとえば、ここに極端な例があります-無人車両。 Intelによると、このようなマシンはそれぞれ、センサーやカメラなどから1日あたり約4〜4.5テラバイトの情報を生成します。


もちろん、これはデバイスの複雑さの点で「ピラミッドの頂点」の一種です。しかし、巨大な規模と数による、スマートホーム、産業用IoT、医療センサーにおける数十億の単純なガジェットは、トラフィックを生み出します。

明らかに、すべての情報をリモートサーバーに転送して処理することは困難です。たとえば、緊急時には、少量の重要なデータを緊急にセンターに転送する必要があります。ここにAPIが適用されます。したがって、RESTful API 実装の1つでは、インシデントに関するそのような一連の情報は、JSONまたはYAML形式で送信されます。

{

 "owner": "Wojtek Konowal",
 "location": "[47.4925, 19.0513]",
 "averageSpeed": 110,
 "passangers": 1,
 "status": "Service is pending",

}

したがって、IoTデバイスネットワークをいくつかの標準RESTful APIと統合することが望ましいです。

セキュリティはアイデンティティから始まる


IoTネットワークは数百億のデバイスに拡大しますが、その機能の重要な部分は、ネットワーク内のサーバーとルーターの前、およびクラウドベースの管理システムの前での相互の信頼できる識別（M2M）に基づいています。これにより、エンドツーエンドのセキュリティ認証チャネルが形成されます。

認証のおかげで、各デバイスには一意の識別子があり、その信頼性を確認できます。これは、承認後にデータを処理し、このデバイスにリソースを割り当てるために重要です。

多くの場合、認証は匿名で実行する必要があります。したがって、デバイスは、特定のグループに属していることを暗号で証明できますが、同時に一意の識別子を秘密に保ちます。現代の状況では、すべてのプロバイダーが各ユーザーに関する最大の個人情報を収集し、それを交換しようとする場合、それも非常に重要です。 GDPRに従って、匿名性は多くの場合規制要件です。

別のトピックは、実際の匿名性は単にIDを非表示にするよりも複雑であることです。たとえば、今ではApple、Amazon、Googleの請負業者が、携帯情報端末のSiri、Alexa、Google Assistantで収集されたユーザーの会話の録音を聞いていると噂されています。これらの請負業者は、性行為の音声録音を聞いています。、喧嘩、麻薬の購入など-そして、ファイルで聞こえる人の名前と住所を簡単に識別できます。記録の正式な匿名化にもかかわらず。

さらに、最近、科学者は匿名データを実在の人々と関連付ける多くの方法があることを証明しました。たとえば、15の人口統計属性を持つデータは、「マサチューセッツの住民の99.98％によって明らかにされます。」調査によると、複雑なデータセットでは、データを正常に匿名化することはほとんど不可能です。

おそらく将来的には、真の匿名性を維持するために、エニグマ暗号を解読したアランチューリングチームが使用したものと同様の方法を使用する必要がありますが、この事実を明らかにすることはできません。チューリングは統計的な異常を隠そうとし、まれにのみ暗号化を発行しました。ほとんどの場合、英国の暗号学者はナチスが兵士や民間人を破壊するのを黙って見守るだけでした。数学の法律では、これらの殺害を防止することはできませんでした。戦術はうまくいった。ナチスは連合国が秘密の情報を「推測」することが多すぎることに気づかなかった。

真のIoT匿名化の場合、これはデータをランダム化し、ランダムなIDとランダムな外部情報を追加することを意味するため、部外者は傍受したトラフィックの真の所有者を把握できませんでした。しかし、これらの問題は将来解決する必要があります。今日の主なことは、IoTネットワークに少なくとも基本的な保護を提供することです。

IoTのコンテキストでは、識別、認証、および許可により、デバイスが不正アクセスから保護され、データがなりすましから保護されます（電気メーターの場合と同様に、人口はおそらくトラフィックを置き換えようとします）。これらは、現代のテクノロジーが処理できる比較的単純なタスクです。
