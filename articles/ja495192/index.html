完璧なMinecraftサーバー起動スクリプト



作者はゲームがとても好きで、彼自身が「純粋に友達のため」の小さなサーバーの管理者です。アマチュアの間でいつものように、すべてがサーバー上で変調され、これは仕事の不安定さと転落の結果として起こります。著者は、Powershellを自分の道の店の場所よりもよく知っているので、「The Minecraft 2020 Launch Script」を作成することにしました。同じスクリプトが、Ruvdsマーケットプレイスでテンプレートの基礎として機能しました。しかし、すべてのソースはすでに記事に含まれています。さて、順番に、それがすべてどのように起こったか。

チームが必要


代替ロギング


かつていくつかのmodを追加したので、戦争を宣言せずにサーバーがクラッシュしたように見えました。サーバーはlatest.logまたはdebugにエラーを書き込まず、このエラーを書き込んで停止するはずのコンソールが閉じられました。

書きたくない-必要ない。Tshell -Objectコマンドレットを備えたPowershell があります。これは、オブジェクトを取得し、それをファイルとコンソールに同時に表示します。

.\handler.ps1 | Tee-Object .\StandardOutput.txt -Append

このようにして、PowershellはStandardOutputを取得してファイルに書き込みます。Start-Processを使用しようとしないでください。これは、StandardOutputではなくSystem.ComponentModel.Componentを返します。- RedirectStandardOutputを使用すると、コンソールに入ることができなくなります。

起動引数


同じペアの改造を置いて、著者はサーバーにもRAMが不足していることに気づきました。そして、これは起動引数を変更する必要があります。誰もが使用するstart.batで毎回変更するのではなく、このスクリプトを使用してください。

Tee-Objectは、実行可能ファイルが「Right So」と呼ばれる場合にのみStandardOutputを読み取るため、別のスクリプトを作成する必要があります。このスクリプトはminecraft自体を実行します。引数から始めましょう。

将来的に究極の遅延にふけるには、スクリプトがその場で開始引数を収集する必要があります。これを行うには、まず最新バージョンのforgeを探すことから始めます。

$forge = ((Get-ChildItem | Where-Object Name -Like "forge*").Name | Sort-Object -Descending) | Select-Object -last 1

sort-objectの助けを借りて、どれだけ多くのオブジェクトをそこに置いても、常に最大の桁を持つオブジェクトを取得します。究極の怠惰。

次に、サーバーにメモリを割り当てる必要があります。これを行うには、システムメモリの量を取得し、その量を文字列で書き込みます。

$ram = ((Get-CimInstance Win32_PhysicalMemory | Measure-Object -Property capacity -Sum).sum /1gb)
$xmx = "-Xms" + $ram + "G"

自動再起動を修正


作成者は他の人からの.batファイルを見ましたが、サーバーが停止した理由を考慮していませんでした。 modファイルを変更したり何かを削除したりする必要があるだけでも不便ですか？
では、正しい再起動をしましょう。著者は以前、サーバーがシャットダウンしたという事実にもかかわらず、サーバーを再起動する奇妙なスクリプトに遭遇しました。 exitcodeを使用します。 Javaは0を正常終了として使用するため、踊ります。

最初に、サーバーが完了に失敗した場合にサーバーを再起動する関数を作成します。

function Get-MinecraftExitCode {
   
    do {
        
        if ($global:Process.ExitCode -ne 0) {
            Write-Log
            Restart-Minecraft
        }
        else {
            Write-Log
        }
 
    } until ($global:Process.ExitCode -eq 0)
    
}

スクリプトは、/ stopコマンドを使用してサーバー自体のコンソールからサーバーが正常にシャットダウンするまでループに残ります。

私たち全員が自動化することを決めた場合、開始日、完了、および完了の理由を収集するとよいでしょう。

これを行うには、Start-Processの結果を変数に書き込みます。スクリプトでは、次のようになります。

$global:Process = Start-Process -FilePath  "C:\Program Files (x86)\common files\Oracle\Java\javapath_target_*\java.exe" -ArgumentList "$xmx -server -jar $forge nogui" -Wait -NoNewWindow -PassThru

そして、結果をファイルに書き込みます。変数で返されるのは次のとおりです。

$global:Process.StartTime
$global:Process.ExitCode	
$global:Process.ExitTime

Add-Contentを使用してこれをすべてファイルに追加できます。少しとかした後、そのようなスクリプトを取得して、handler.ps1と呼びます。

Add-Content -Value "Start time:" -Path $Logfile 
$global:Process.StartTime
 
Add-Content -Value "Exit code:" -Path $Logfile 
$global:Process.ExitCode | Add-Content $Logfile
    
Add-Content -Value "Exit time:" -Path $Logfile 
$global:Process.ExitTime | Add-Content $Logfile

次に、ハンドラーを起動してスクリプトを実行します。

適切な起動


作成者は、1つのモジュールが任意のパスからさまざまなバージョンのMinecraftを起動し、特定のフォルダーにログを配置できることを望んでいます。

問題は、システム上にいるユーザーがプロセスを開始する必要があることです。これは、デスクトップまたはWinRmを介して実行できます。システムまたは管理者に代わってサーバーを起動しても、ログインしないと、Server.jarはeula.txtを読み取って起動することさえできません。

レジストリに3つのエントリを追加することで、システムへの自動ログインを有効にできます。

New-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name DefaultUserName -Value $Username -ErrorAction SilentlyContinue
New-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name DefaultPassword -Value $Password  -ErrorAction SilentlyContinue
New-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name AutoAdminLogon -Value 1 -ErrorAction SilentlyContinue

安全ではありません。ログインとパスワードはここでは平文で示されているため、サーバーを起動するには、ユーザーレベルまたはさらに狭いグループでアクセス権を持つ個々のユーザーを起動する必要があります。これには標準の管理者を使用しないことを強くお勧めします。

自動入力で分類されます。次に、サーバーの下に新しいタスクを登録する必要があります。Powershellからコマンドを実行するため、次のようになります。

$Trigger = New-ScheduledTaskTrigger -AtLogOn
$User = "ServerAdmin"
$PS = New-ScheduledTaskAction -Execute 'PowerShell.exe" -Argument "Start-Minecraft -Type Forge -LogFile "C:\minecraft\stdout.txt" -MinecraftPath "C:\minecraft\"'
Register-ScheduledTask -TaskName "StartSSMS" -Trigger $Trigger -User $User -Action $PS -RunLevel Highest

モジュールをまとめる


次に、すべてを後で使用できるモジュールにしましょう。既製のスクリプトのすべてのコードがここにあり、インポートして使用します。

モジュールに煩わされたくない場合は、上記のすべてを個別に使用できます。

スタートマインクラフト


最初に、それだけを行うモジュールを作成し、標準出力をリッスンして記録するスクリプトを実行します。

パラメーターブロックで、Minecraftを起動するフォルダーとログを配置する場所を尋ねます。

Set-Location (Split-Path $MyInvocation.MyCommand.Path)
function Start-Minecraft {
    [CmdletBinding()]
    param (
        [Parameter()]
        [ValidateNotNullOrEmpty()]
        [string]
        $LogFile,
 
        [Parameter(Mandatory)]  
        [ValidateSet('Vanilla', 'Forge')]
        [ValidateNotNullOrEmpty()]
        [string]
        $Type,
 
        [Parameter(Mandatory)]
        [ValidateNotNullOrEmpty()]
        [string[]]
        $MinecraftPath
 
    )
    powershell.exe -file .\handler.ps1 -type $type -MinecraftPath $MinecraftPath | Tee-Object $LogFile -Append
}
Export-ModuleMember -Function Start-Minecraft

そして、あなたはこのようにMinecraftを実行する必要があります：

Start-Minecraft -Type Forge -LogFile "C:\minecraft\stdout.txt" -MinecraftPath "C:\minecraft\"

次に、すぐに使えるHandler.ps1に移りましょう。

スクリプトが呼び出されたときにパラメーターを受け入れることができるように、パラメーターブロックも指定する必要があります。Oracle Javaが起動することに注意してください。別のディストリビューションを使用している場合は、実行可能ファイルへのパスを変更する必要があります。

param (
    [Parameter()]
    [ValidateNotNullOrEmpty()]
    [string]$type,
 
    [Parameter()]
    [ValidateNotNullOrEmpty()]
    [string]$MinecraftPath,
 
    [Parameter()]
    [ValidateNotNullOrEmpty()]
    [string]$StandardOutput
)
 
Set-Location $MinecraftPath
 
function Restart-Minecraft {
 
    Write-host "=============== Starting godlike game server ============"
 
    $forge = ((Get-ChildItem | Where-Object Name -Like "forge*").Name | Sort-Object -Descending) | Select-Object -first 1
 
    $ram = ((Get-CimInstance Win32_PhysicalMemory | Measure-Object -Property capacity -Sum).sum /1gb)
    $xmx = "-Xms" + $ram + "G"
    $global:Process = Start-Process -FilePath  "C:\Program Files (x86)\common files\Oracle\Java\javapath_target_*\java.exe" -ArgumentList "$xmx -server -jar $forge nogui" -Wait -NoNewWindow -PassThru
    
}
 
function Write-Log {
    Write-host "Start time:" $global:Process.StartTime
 
    Write-host "Exit code:" $global:Process.ExitCode
    
    Write-host "Exit time:" $global:Process.ExitTime
 
    Write-host "=============== Stopped godlike game server ============="
}
 
function Get-MinecraftExitCode {
   
    do {
        
        if ($global:Process.ExitCode -ne 0) {
            Restart-Minecraft
            Write-Log
        }
        else {
            Write-Log
        }
 
    } until ($global:Process.ExitCode -eq 0)
    
}
 
Get-MinecraftExitCode

Register-minecraft


このスクリプトは実質的にStart-Minecraftを繰り返しますが、新しいタスクを登録するだけです。同じ引数を受け入れます。ユーザー名が指定されていない場合は、現在のユーザー名が使用されます。

function Register-Minecraft {
    [CmdletBinding()]
    param (
        [Parameter()]
        [ValidateNotNullOrEmpty()]
        [string]
        $LogFile,
 
        [Parameter(Mandatory)]  
        [ValidateSet('Vanilla', 'Forge')]
        [ValidateNotNullOrEmpty()]
        [string]$Type,
 
        [Parameter(Mandatory)]
        [ValidateNotNullOrEmpty()]
        [string]$MinecraftPath,
 
        [Parameter(Mandatory)]
        [ValidateNotNullOrEmpty()]
        [string]$User,
 
        [Parameter(Mandatory)]
        [string]$TaskName = $env:USERNAME
    )
 
    $Trigger = New-ScheduledTaskTrigger -AtLogOn
    $arguments = "Start-Minecraft -Type $Type -LogFile $LogFile -MinecraftPath $MinecraftPath"
    $PS = New-ScheduledTaskAction -Execute "PowerShell" -Argument "-noexit -command $arguments"
    Register-ScheduledTask -TaskName $TaskName -Trigger $Trigger -User $User -Action $PS -RunLevel Highest
    
}
 
Export-ModuleMember -Function Register-Minecraft

自動ログオンの登録


パラメーターブロックでは、スクリプトはユーザー名とパスワードのパラメーターを受け入れます。ユーザー名が指定されていない場合は、現在のユーザーの名前が使用されます。

function Set-Autologon {
 
    param (
        [Parameter(
        HelpMessage="Username for autologon")]
        $Username = $env:USERNAME,
 
        [Parameter(Mandatory=$true,
        HelpMessage="User password")]
        [ValidateNotNullOrEmpty()]
        $Password
    )
 
    $i = Get-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon"
 
    if ($null -eq $i) {
        New-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name DefaultUserName -Value $Username
        New-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name DefaultPassword -Value $Password 
        New-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name AutoAdminLogon -Value 1
        Write-Verbose "Set-Autologon will enable user auto logon."
 
    }
    else {
        Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name DefaultUserName -Value $Username
        Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name DefaultPassword -Value $Password
        Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name AutoAdminLogon -Value 1
    }
 
    
    Write-Verbose "Autologon was set successfully."
 
}

このスクリプトを実行すると、次のようになります。

Set-Autologon -Password "PlaintextPassword"

使い方


次に、著者自身がこれをどのように使用するかを検討します。WindowsにMinecraftパブリックサーバーをデプロイする方法。最初から始めましょう。

1.ユーザーを作成する

$pass = Get-Credential
New-LocalUser -Name "MinecraftServer" -Password $pass.Password -AccountNeverExpires -PasswordNeverExpires -UserMayNotChangePassword

2.スクリプトを実行するタスクを登録します。

モジュールを使用して、次のように登録できます。

Register-Minecraft -Type Forge -LogFile "C:\minecraft\stdout.txt" -MinecraftPath "C:\minecraft\" -User "MInecraftServer" -TaskName "MinecraftStarter"

または、標準ツールを使用します。

$Trigger = New-ScheduledTaskTrigger -AtLogOn
$User = "ServerAdmin"
$PS = New-ScheduledTaskAction -Execute 'PowerShell.exe" -Argument "Start-Minecraft -Type Forge -LogFile "C:\minecraft\stdout.txt" -MinecraftPath "C:\minecraft\"'
Register-ScheduledTask -TaskName "StartSSMS" -Trigger $Trigger -User $User -Action $PS -RunLevel Highest

3.システムへの自動ログインをオンにし、マシンを再起動します

Set-Autologon -Username "MinecraftServer" -Password "Qw3"
