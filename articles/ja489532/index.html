ユーザーおよび開発者向けのAndroid 11の新機能



先日、開発者向けのAndroid 11のプレビューバージョンが突然公開されました。グーグルは「開発者、昨年私たちに良いフィードバックを与えたので、今回はこれまでより早くプレビューを提供します」（リリースは第3四半期、つまりその約6か月前に予定されています）と言います。

このため、バージョンは特に未加工であり、すべての今後の機能がそれに当てはまるとは限りません（Google I / Oで魅力的だったものと思われます）。また、リリース前にロールバックできるものもあります。しかし、それでも新機能を理解するのは興味深いことです。

いつものように、9to5GoogleやAndroid Policeなどのサイトはすぐにプレビューバージョンの尾とたてがみをテストし、最終的に一連のテキストをリリースしました。そして、この一連のテキストを読み、ロシア語のHabrの編集を行いました。これは、通常のAndroidユーザーには理解できるようになりましたが、特にモバイル開発者向けの投稿が含まれています。



画面ビデオ録画


この機能はまだAndroid 10のベータ版に含まれていましたが、問題があり、リリースには含まれていませんでした。現在、新しいプレビュー版で新しいバージョンに変更された形で登場しています。Android 11のリリースでは、それが落ちると信じたいです。

基本的に、彼女の名前はすべてを説明しています。Androidには、画面上で発生するすべてのことをビデオ録画するためのサードパーティ製アプリケーションがありますが、ネイティブソリューションで問題はありません。

開発者の状況：おそらく、「ユーザーは再現が難しいバグについて不平を言う」状況では、ユーザーにビデオを要求することがより簡単になるでしょう。


泡


この機能はまだAndroid 10のベータ版に含まれていましたが...はい、ストーリーは画面のビデオのようなものです。彼らは「10」のリリースのために機能を思い付くことができず、次のバージョンに移行しました。

これはFacebookチャットヘッドの類似物です。ダイアログを最小化して、他のアプリケーションの上に画面上に残るようにすることができます。おばあちゃん、なぜあなたはbubblesesを必要とするのですか？これは、Habrを読むのをより便利にすると同時に、私の孫娘であるあなたとやり取りするためです。

ただし、すべてのインスタントメッセンジャーで一度に魔法のように完璧に機能するわけではありません。Androidアプリケーションの開発者には、Bubbles APIがあり、問題は開発者がどのように積極的に使用するかです。おそらく、GoogleはGoogleの無限のメッセンジャーセットすべてを熱心にサポートするでしょうが、残りについてはまだ疑問があります。

開発者にとってこれはどうなっていますか：アプリケーションに個人的なメッセージの機能がある場合は、ここでどのように行動すればよいかを理解してください。ドキュメントはここにあります。


スコープ付きストレージ


この機能はまだ...よくわかりましたね Android 10では、この機能はすでに含まれていますが、オプションになりました（アプリケーション開発者が拒否する可能性があります）が、最終的に完成しました。

スコープ付きストレージの本質は、アプリケーションにそのフォルダーへのアクセスのみが許可され、他のファイルを操作できないことです。もちろん、ファイルマネージャーなどのアプリケーションにはフルアクセスが必要です。そのような例外では、「すべてのファイルアクセス」がスローされます。しかし、一般的には、iOSのアプローチには妥協点があります。それは、ユーザーのセキュリティという名目で機能のアプリケーションを制限することです。

開発者の状況：Android 11をtargetSdkVersionに含める予定の場合、Googleは「スコープ付きストレージで動作するようにアプリケーションを更新する」と厳密に述べ、より詳細なテキストへのリンクを提供します。


ワンタイム許可


プライバシーの強化についての詳細。以前は、アプリケーションが許可を要求すると、アプリケーションの使用時にのみ許可、拒否、許可するオプションがありました。カメラ、マイク、地理位置情報へのアクセスなどの重要な権限のために、「1回のみ許可」という新しいオプションが作成されました。

それは非常に賢明に聞こえます：一部のアプリケーションでカメラを一度だけ使用してそこでアバターを作成したい場合は、このアプリケーションにカメラへの常時アクセスを許可します。

ただし、ここで注目に値します。新しいバージョンの公式スクリーンショットでは、以前の「許可」オプションが欠落しており、バックグラウンドを含めて、永久にアクセスを許可するだけです。



私たちが理解している限り、これは同時にGoogleもルールを変更しているためです。これで、バックグラウンドでジオロケーションに本当にアクセスする必要がある理由をGoogleに正当化した人だけがリクエストできるようになります。さて、以前と同様に、コールへのアクセスが完了しました。

開発者の何が問題になっているのですか？ここでは、許可を求めているかどうか、および変更が影響を与えているかどうかがよくわかります。「アプリケーションにWebViewがある場合、アクセスが異なる可能性がある」など、さまざまなニュアンスがあるため、公式テキストを調べてください。


画面リフレッシュレートインジケーター


これは非常に小さなことですが、その背後に全体像を感じることができます。

リフレッシュレートの高い画面がトレンドになっています。たとえば、OnePlus 7Tには90 Hzの画面があり、Samsung S20には120 Hzの画面があります。これにより、スクロール時に前例のない滑らかさが得られます-美しいように見えます。しかし、問題があります。画面を頻繁に更新すると、より積極的にエネルギーが消費されます。

「24 FPSでビデオを再生する場合、より頻繁に更新するのは意味がないので、画面が常に最大限に機能するとは限らない」という考えがありました。いいですね。しかし、最終的には、たとえば、Pixel 4の購入者は90 Hzのスマートフォンを購入したことに気付き、希望するときに（希望しない場合も含めて）60 Hzに切り替えました。そして、スクリーンがあなたが払ったものを一般的にするときを理解することは困難でした。そして今、開発者のネイティブ設定で、インジケーターをオンにすることができます。



開発者の状況：「このGoogleが画面のリフレッシュレートを下げる、またはこのアプリケーションが遅くなる」と理解できない場合は、少し簡単になります。


5G API


5Gをカバーしているため、世界全体は言うまでもなく、米国でもすべてが悲しくなっています。また、5Gスマートフォンの市場シェアも控えめです。しかし、おそらく、5Gゾーンのどこかに5Gスマートフォンの所有者がいて、Googleがそれらを処理します。私たちはこれらの3人を喜びます。

一般に、新しいBandwith Estimatorと動的なメータリングAPIは、モバイルアプリケーションがユーザーの帯域幅を理解し、トラフィックが制限されて最適な品質でデータを送信できるかどうかを理解できるように設計されています。

開発者にとってそれは何ですか？現在、そしてロシアの現実-ほとんど何もありませんが、あなたの先延ばしが「5つの燃えるタスクの代わりに完全にオプションの何かをしよう」とささやくなら、注意を払います。
