Improving SSL security settings in Zimbra Collaboration Suite Open-Source Edition

The reliability of encryption is one of the most important indicators when using information systems for business, because every day they participate in the transfer of a huge amount of confidential information. A common tool for evaluating the quality of an SSL connection is an independent test from Qualys SSL Labs. Since this test can be run by anyone, it is especially important for SaaS providers that the score in this test is maximum. Not only SaaS providers, but also ordinary enterprises take care of the quality of SSL connections. For them, this test is an excellent opportunity to identify potential vulnerabilities and close all loopholes for cybercriminals in advance.





In Zimbra OSE, two types of SSL certificates are allowed. The first is a self-signed certificate, which is automatically added during installation. This certificate is free and not limited in time of use, which means it is ideal for testing Zimbra OSE or its use exclusively within the internal network. However, when entering the web client, users will see a warning from the browser that this certificate is not reliable, and the test from Qualys SSL Labs will definitely fail your server.



The second is a commercial SSL certificate signed by a certification authority. Such certificates are easily perceived by browsers and are commonly used in the commercial operation of Zimbra OSE. Immediately after the commercial certificate has been correctly installed, Zimbra OSE 8.8.15 shows a grade of A in the test from Qualys SSL Labs. This is a great result, but our goal is to achieve an A + result.











In order to achieve the maximum mark in the test from Qualys SSL Labs when using the Zimbra Collaboration Suite Open-Source Edition, you need to perform a number of steps:



1. Increase the parameters of the Diffie-Hellman protocol



By default, in all Zimbra OSE 8.8.15 components that use OpenSSL, the Diffie-Hellman protocol settings are 2048 bits. In principle, this is more than enough to get an A + score in a test from Qualys SSL Labs. However, if you are upgrading from older versions, the value of the parameters may be lower. Therefore, it is recommended that after the update is completed, run the command zmdhparam set -new 2048, which will increase the Diffie-Hellman protocol parameters to acceptable 2048 bits, and if desired, using the same command, you can increase the parameter value to 3072 or 4096 bits, which on the one hand will increase generation time, however, on the other hand, it will positively affect the security level of the mail server.



2. Inclusion of the recommended list of used ciphers



By default, Zimbra Collaborataion Suite Open-Source Edition supports a wide range of strong and weak ciphers, which encrypt data passing through a secure connection. However, the use of weak ciphers is a serious minus when checking the security of the SSL connection. In order to avoid this, you must configure the list of ciphers used.



To do this, use the command. zmprov mcf zimbraReverseProxySSLCiphers 'ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-AES256-GCM-SHA384:DHE-RSA-AES128-GCM-SHA256:DHE-DSS-AES128-GCM-SHA256:kEDH+AESGCM:ECDHE-RSA-AES128-SHA256:ECDHE-ECDSA-AES128-SHA256:ECDHE-RSA-AES128-SHA:ECDHE-ECDSA-AES128-SHA:ECDHE-RSA-AES256-SHA384:ECDHE-ECDSA-AES256-SHA384:ECDHE-RSA-AES256-SHA:ECDHE-ECDSA-AES256-SHA:DHE-RSA-AES128-SHA256:DHE-RSA-AES128-SHA:DHE-DSS-AES128-SHA256:DHE-RSA-AES256-SHA256:DHE-DSS-AES256-SHA:DHE-RSA-AES256-SHA:AES128-GCM-SHA256:AES256-GCM-SHA384:AES128:AES256:HIGH:!aNULL:!eNULL:!EXPORT:!DES:!MD5:!PSK:!RC4'







A set of recommended ciphers is immediately included in this command, and thanks to it, the command can immediately include reliable ciphers in the list and exclude unreliable ones. Now all that remains is to restart the nodes with reverse proxies using the zmproxyctl restart command. After the reboot, the changes will take effect.



If for some reason this list does not suit you, you can remove a number of weak ciphers from it using the command zmprov mcf +zimbraSSLExcludeCipherSuites



. So, for example, a team zmprov mcf +zimbraSSLExcludeCipherSuites TLS_RSA_WITH_RC4_128_MD5 +zimbraSSLExcludeCipherSuites TLS_RSA_WITH_RC4_128_SHA +zimbraSSLExcludeCipherSuites SSL_RSA_WITH_RC4_128_MD5 +zimbraSSLExcludeCipherSuites SSL_RSA_WITH_RC4_128_SHA +zimbraSSLExcludeCipherSuites TLS_ECDHE_RSA_WITH_RC4_128_SHA



that completely eliminates the use of RC4 ciphers. You can do the same with AES and 3DES ciphers.



3. Enabling HSTS



Enabled mechanisms for enforcing connection encryption and TLS session recovery are also prerequisites for obtaining the highest score in the test from Qualys SSL Labs. To enable them, you must enter the command zmprov mcf +zimbraResponseHeader "Strict-Transport-Security: max-age=31536000"



. This command will add the necessary header to the configuration, and for the new settings to take effect, you will have to restart Zimbra OSE using the zmcontrol restart command .



Already at this stage, the test from Qualys SSL Labs will demonstrate an A + score, however if you want to further improve the security of your server, you can take a number of measures.







For example, you can enable forced encryption of interprocess connections, as well as enable forced encryption when connecting to Zimbra OSE services. To verify interprocess connections, enter the following commands: 



zmlocalconfig -e ldap_starttls_supported=1
zmlocalconfig -e zimbra_require_interprocess_security=1
zmlocalconfig -e ldap_starttls_required=true


To enable forced encryption, you must enter: 



zmprov gs `zmhostname` zimbraReverseProxyMailMode
zmprov ms `zmhostname` zimbraReverseProxyMailMode https

zmprov gs `zmhostname` zimbraMailMode
zmprov ms `zmhostname` zimbraMailMode https

zmprov gs `zmhostname` zimbraReverseProxySSLToUpstreamEnabled
zmprov ms `zmhostname` zimbraReverseProxySSLToUpstreamEnabled TRUE




Thanks to these commands, all connections to proxy servers and mail servers will be encrypted, and all these connections will be proxied.







Thus, following our recommendations, you can not only achieve the highest mark in the test for the security of SSL connections, but also significantly increase the security of the entire Zimbra OSE infrastructure.
