インターネットなしの時刻同期



tcp / ip以外にも、時刻を同期する方法はたくさんあります。それらのいくつかは、通常の電話のみを必要としますが、その他は高価で希少で敏感な電子機器を必要とします。時間同期システムの広範なインフラストラクチャには、展望台、政府機関、ラジオ局、衛星配置などが含まれます。



今日は、インターネットなしの時刻同期がどのように機能するか、および自分の手で「衛星」NTPサーバーを作成する方法について説明します。



短波放送



アメリカ合衆国では、NISTは正確な時刻と周波数を2.5、5、10、15、20 MHzでコロラド州フォートコリンズのWWVHステーションからの電波に送信し、カウアイのWWVHステーションから周波数2.5、5、10、15 MHzで送信します。ハワイの状態。タイムコードは60秒間隔で1 bpsの速度で送信されます。 100 Hzのサブキャリアでパルス幅変調を使用します。



カナダのNational Research Council（NRC）は、オンタリオ州オタワにあるCHUステーションから3.33、7.85、および14.67 MHzで時間と周波数の情報を配信しています。





WWVHブロードキャスト形式



短波ステーションからの信号の伝搬は、通常、電離層の上層からの反射によって発生します。信号伝送は長距離で取得できますが、時間の精度は1ミリ秒のオーダーです。



現在のNTPv4標準には、WWV、WWVH、およびCHU用のオーディオドライバーが含まれています。



長波放送



NISTはまた、コロラド州のボルダーから60 kHzの長い電波で正確な時間と周波数を送信します。長波で正確な時間信号を送信するための他のステーションがあります。

静止運用環境観測衛星



米国では、NISTは静止運用環境衛星（GOES）から約468 MHzで正確な時刻と周波数のデータも送信します。タイムコードは、リモートセンサーのポーリングに使用されるメッセージと交互に表示されます。30秒間隔で送信される60個のBCDニブルで構成されます。タイムコード情報は、地上波サービスに似ています。



全地球測位システム



米国国防総省は、陸上、海上、および空中の正確なナビゲーションにGPSを使用しています。このシステムは、55°の角度で傾斜した12時間の軌道で衛星をグループ化することにより、地球を24時間カバーします。



24衛星の初期コンステレーションは、異種構成の31衛星に拡張されたため、少なくとも6衛星が常に見え、ほとんどの世界で8衛星以上が見えています。



GPSなどのサービスは、他の国で運用または計画されています。ロシアのGLONASSは、衛星の総数が26に達した2010年9月2日から数えると、12年間運用されています。この星座は、地球を完全にカバーするために完全に配備されました。

SRNSによる座標決定の数学



スマートフォンのGPS / Glonassナビゲーターは、無線航法通信システム（SRNS）を使用して、このような精度で位置をどのように決定しますか？計算の原理を理解するためには、中学または高校の物理学科の高学年内の立体測定と代数を思い出す必要があります。



各衛星は正確な時間を受信機に伝えます。衛星は原子時計を持っているので、信頼できます。光速を知っているので、衛星が置かれている表面上の球の半径を決定することは難しくありません。地球と接触している同じ球体が、GPS / Glonass受信機が配置されている円を形成します。



信号が2つの衛星からのものである場合、地球と2つの球の交点がすでに存在しており、円上の2つの点しか得られません。 3番目の衛星の球体は、理想的にはこれら2つのポイントのいずれかに該当し、最終的に受信機の座標を決定します。



原則として、2つの衛星からであっても、間接標識によって、2つのポイントのどちらが真実に近いかを理解でき、最新のナビゲーションソフトウェアアルゴリズムがこのようなタスクに対処できます。では、なぜ4番目の衛星が必要なのでしょうか。





衛星配置を使用した測位。



この理想化された図には、計算の精度が依存する多くのニュアンスがあることに気づくのは簡単です。レシーバーの時間はおそらくエラーの最も明白な原因です。すべてが正しく機能するためには、受信機のGPS / Glonass時刻が衛星時刻と同期している必要があります。これがないと、誤差は10万kmになります。



速度、時間、距離の式S = v * tから、SRNS信号を送信するための基本的な方程式が得られます。衛星までの距離は、光速と衛星と受信機の間の時間差の積に等しい。







S=∗(t−t)









これは主に、すべての同期の後でも、レシーバーtprの時刻が十分な精度でわかっているためです。真の時間とtprの間には常にΔtが存在するため、計算誤差は許容できなくなります。そのため、4番目の衛星が必要になります。



4つの衛星の必要性を明確に数学的に正当化するために、方程式系を構築します。







Ri=(x−Xi)2+(y−Yi)2+(z−Zi)2+Δt









4つの未知数x、y、z、およびΔtを決定するには、観測数が未知数の数以上である必要があります。これは必要条件ですが、十分条件ではありません。正規方程式の行列が縮退していることが判明した場合、方程式系には解がありません。



地上の時計と比較した衛星原子時計の時間拡張による相対性理論と相対論的効果の特別な理論を忘れないでください。







t=t1−(vc)2









衛星が14,000 km / hの速度で軌道上を移動すると仮定すると、約7μs（マイクロ秒）の時間の遅れが生じます。一方、一般相対性理論の相対論的効果が作用します。



問題は、軌道上の衛星は地球からかなり離れた場所にあり、時空の連続体の曲率は地球の質量のために地球の表面よりも小さいことです。一般相対性理論によると、巨大な物体に近い時計のコースは、それから遠い場所にある時計よりも遅く見えるでしょう。







t=t∗1−2GMrc2







  • Gは重力定数です。
  • Mはオブジェクトの質量、この場合は地球です。
  • rは地球の中心から衛星までの距離です。
  • cは光の速度です。


この式による計算では、衛星で45μsの時間の遅れが生じます。合計-7μs+45μs=38μsバランス-SRTとGRの影響。



SRNSを使用した測位の応用問題では、電離層および対流圏の遅延も考慮する必要があります。さらに、46 nsの補正は、0.02 GPS衛星軌道の離心率に関連付けられています。



4つ以上のGPS / GLONASS衛星から同時に信号を受信する機能により、受信機の座標を決定する精度をさらに高めることができます。これは、ナビゲーターが4つの未知数を使用して4つの方程式のシステムを解くという事実により達成されます。Cmk 回数と平均値を取得し、数学的統計の法則に従って最終評価の精度を高めます。



衛星経由でStartum 1 NTPサーバーを構成する方法



高品質のタイムサーバーをセットアップするには、GPSD、NTP、および1PPS出力（毎秒1パルス）のGPSレシーバーのみが必要です。



1. gpsdとntpd、またはgpsdとchronydをインストールします。gpsdバージョンは3.20以上である必要があります

2. PPS対応のGPSレシーバーをシリアルRS232またはUSBポートに接続します。



通常の安価なGPSレシーバーは機能しません。適切なものを見つけるために少し走り回る必要があるかもしれません。



3.デバイスが本当にPPSを発行していることを確認します。これについては、gpsmonユーティリティでポートを確認してください。



4. /etc/conf.d/gpsdファイルを開き、次の行を編集します。



交換する 



GPSD_OPTIONS=""


なるように 



GPSD_OPTIONS="-n"


この変更は、gpsdが起動時にSRNSソースの検索をすぐに開始するために必要です。



5. gpsdを起動または再起動します。 



(1:110)$ sudo /etc/init.d/gpsd start
(1:111)$ sudo /etc/init.d/gpsd restart


systemdを使用したディストリビューションの場合、適切なsystemctlコマンドを使用します。



6. cgpsコマンドのコンソール出力を確認します。



データが衛星からのものであることを確認する必要があります。コンソールには、図のようなものが表示されます。





cgps consoleコマンドの出力。



7. /etc/ntp.confファイルを編集します。 



# GPS Serial data reference (NTP0)
server 127.127.28.0
fudge 127.127.28.0 time1 0.9999 refid GPS

# GPS PPS reference (NTP1)
server 127.127.28.1 prefer
fudge 127.127.28.1 refid PPS


一番上のNTP0レコードは、ほとんどすべてのGPSデバイスで利用可能な世界時ソースを示しています。低いNTP1エントリは、はるかに正確なPPSソースを定義します。



8. ntpdを再起動します。 



(1:112)$ sudo /etc/init.d/ntpd restart


systemdのあるディストリビューションでは、systemctlコマンドを使用します。

$ sudo systemctl restart ntp
