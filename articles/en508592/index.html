Submitting an Authorized Request



The authorization header is used to include authorization data in the request when accessing the RESTful API. To add the specified data, you must use the flag -H



. For example, if your interface key is (API key) my-secret-token



, you can include it in the http request as follows: 



$ curl -H "Authorization: my-secret-token" https://httpbin.org/get
{
  "args": {}, 
  "headers": {
    "Accept": "*/*", 
    "Authorization": "my-secret-token", 
    "Host": "httpbin.org", 
    "User-Agent": "curl/7.58.0", 
    "X-Amzn-Trace-Id": "Root=1-5ee8e1a5-a3aa30e0765a7980b04ca4a0"
  }, 
  "origin": "69.84.111.39", 
  "url": "https://httpbin.org/get"
}

Curl also supports default authorization through a flag -u

Conclusion



In summary, here are the curl options that I find most useful:



  • -X



    defines a request method e.g. curl -X POST url



  • -d



    defines the request body as a string in PUT and POST requests. Use @



    to extract data from a file
  • -H



    defines the request header, for example curl -H "Authorization: my-secret-token" url



  • -u



    authentication data for standard authorization
  • -O



    saves request body to file
  • -i



    shows full answer including headers


Curl is a useful tool for interacting with the API through the command line, regardless of whether it is a third-party API or an API that you are developing. For quick testing, curl is better than Axios in Node.js or query tuning in Postman if you are familiar with their syntax.
