WebアニメーションAPIの紹介

こんにちは！

Web APIは常に進化しています。 ConsoleやCanvasなどの一部は、すべてのブラウザで十分にサポートされており、その他はまだ開発中です。

開発中のAPIの1つは、WebアニメーションAPIまたはWAAPIです。仕様の最初のバージョンが2012年に公開され、API自体がFirefoxおよびChromeブラウザーで2014年に初めて実装されたという事実にもかかわらず、私は最近それについて知りました（以前にそれについて聞いたことがない-約）。

開発者はJavaScriptを使用してCSSアニメーションを操作できます。その構文は、従来のCSSアニメーションの構文と似ていますが、開発者がアニメーションを簡単に作成および変更できるようにするいくつかの機能があります。

このAPIを簡単な例で見てみましょう。

下にあるのは回転する四角で、回転するたびに色が変化します。



CSSは次のようになります。

#cube {
    width: 40px;
    height: 40px;
    margin: 50%;
    background-color: pink;
    animation: rotateCube 1s infinite;
}

@keyframes rotateCube {
    0% {
        transform: rotate(0deg);
    }

    30% {
        background-color: purple;
    }

    100% {
        transform: rotate(180deg);
    }
}

次に、WAAPIでも同じことを行います。

アニメーションを作成する


すべては、CSSの@keyframesディレクティブに含まれているものと同様の情報を含むKeyframesオブジェクトの作成から始まります。

let cubeRotating = [
    {transform: 'rotate(0deg)', backgroundColor: 'pink'},
    {backgroundColor: 'purple', offset: 0.3},
    {transform: 'rotate(180deg)', backgroundColor: 'pink'}
]

主な違いは2つあります。

  • 他のステップにbackgroundColorを追加する必要があります。
  • 各ステップを完了する時間をパーセンテージで決定する必要はありません。

WAAPIは自動的にアニメーションをキーの数で等しい部分に分割するため、この場合、背景色はアニメーションの約半分だけ変化します。

ただし、これを30％発生させたいので、2番目のステップで値0.3のオフセットプロパティを追加します。

覚えておくべき重要なことが1つあります。Keyframesオブジェクトには少なくとも2つのキーが必要です。そうでない場合、NotSupportedErrorがスローされます。

次に、繰り返しの期間と回数に関与するアニメーションプロパティを含むオブジェクトが作成されます。

let cubeTiming = {
    duration: 1000,
    iterations: Infinity
}

アニメーションの継続時間はミリ秒単位で設定されます。

「無限」の代わりに、「無限」というキーワードを使用します。

最後に、アニメーションを実行するために、Element.animateメソッドを使用します。

document.getElementById('cube').animate(
    cubeRotating,
    cubeTiming
)

さらにいくつかの構文オプションがあります。例はここにあります。

しかし、それだけではありません。実際、WAAPIを使用すると、アニメーションの再生を制御できます。

アニメーション再生制御


animateメソッドを呼び出すと、アニメーションがすぐに開始されますが、これが常に必要なわけではありません。したがって、pauseメソッドとplayメソッドを呼び出して、それぞれアニメーションを停止および開始できます。

let cubeAnimation = document.getElementById('cube').animate(
    cubeRotating,
    cubeTiming
)

cubeAnimation.pause()

document.body.onclick = () => cubeAnimation.play()

この例では1つのアニメーションを使用していますが、ページにいくつかの「アニメーション」を追加して、好きなように管理できます。

利用可能なWAAPIメソッドの中には、finish、cancel、reverseメソッドもあります。

アニメーションの再生速度も制御できます。

let cubeAnimation = document.getElementById('cube').animate(
    cubeRotating,
    cubeTiming
)

document.body.onclick = () => cubeAnimation.playbackRate *= 1.5

このコードにより、クリックしたときに正方形がより速く回転します。



これまで、1つのアニメーションを作成して再生を制御する方法を学習してきました。もう1つのWAAPI機能は、すべてのアニメーションに同時にアクセスすることです。

複数のアニメーションを管理する


WAAPIには、作成されたすべてのアニメーションにアクセスできるgetAnimationsメソッドがあります。

ユーザーがprefers-reduced-motionを有効にしている場合、ページ上のすべてのアニメーションを遅くしたいとします（CSSメディア機能のprefers-reduced-motionを使用して、OSが使用するアニメーションまたは動きの量を最小化するようにユーザーが要求したかどうかを判断できます-約）：

const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)')

if(mediaQuery.matches){
    document.getAnimations().forEach(animation => {
        animation.playbackRate *= 0.5
    })
}

上記の例では、メディア関数のprefers-reduced-motionを探し、その値がreduceの場合、ページ上のすべてのアニメーションを取得し、それらの再生速度を半分にします。

これは、WAAPIを非常に便利にするものの1つです。1つのプロパティを変更することで、複数のアニメーションに変更を加えることができます。

依存関係


WAAPIのもう1つの興味深い機能は、あるアニメーションのプロパティが別のアニメーションのプロパティに依存していることを判別できることです。

たとえば、2つの正方形があり、2番目の正方形を最初の正方形の2倍の速度で回転させたい場合、2つの方法でこれを行うことができます。

最初の方法：

let cube1Animation = document.getElementById('cube').animate(
    cubeRotating,
    {
        duration: 1000,
        iterations: Infinity
    }
)

let cube2Animation = document.getElementById('cube2').animate(
    cubeRotating,
    {
        duration: 500,
        iterations: Infinity
    }
)

最初の正方形のアニメーション時間は1秒、1秒は500ミリ秒です。

ただし、このアプローチでは、最初の正方形のアニメーション時間を変更するときに、2番目の正方形にも同じことを行う必要があります。

多くのアニメーションや多数のアニメーションオブジェクトがある場合、どれほど難しくなるか想像してみてください。

私たちの問題を解決する最善の方法は、2番目の正方形の回転の1番目の正方形への依存性を確立することです。

let cube1Animation = document.getElementById('cube').animate(
    cubeRotating,
    {
        duration: 1000,
        iterations: Infinity
    }
)

let cube2Animation = document.getElementById('cube2').animate(
    cubeRotating,
    {
        duration: cube1Animation.effect.timing.duration / 2,
        iterations: Infinity
    }
)

したがって、最初の正方形のアニメーション時間を使用して、2番目の正方形のアニメーション時間を決定します。これで、最初の正方形のアニメーション時間を変更すると、2番目の正方形は常に2倍の速さで回転します。



パフォーマンス


パフォーマンスについて言えば、CSSとWAAPIの使用に大きな違いはありませんでした。しかし、これは私の例が単純なためかもしれません。

JSでアニメーションを作成する他の方法に対するWAAPIの重要な利点の1つは、別のスレッドで実行されるため、メインスレッドがアニメーションを「忘れて」残りのコードを実行できることです。

ブラウザのサポート


WAAPIは現在ドラフトステータスであり、最近のバージョンのFirefoxとChrome、および主要なモバイルブラウザーで部分的にサポートされています。

部分的なサポートとは、ブラウザが再生、一時停止、反転、終了、playbackRateなどのメソッドをサポートすることを意味しますが、getAnimationsはサポートしません。
