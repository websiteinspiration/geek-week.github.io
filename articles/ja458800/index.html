「Federated Learning」という文章のレビューを提供します

統合学習のアイデアは、問題を解決するための有用な情報（たとえば、MRIを使用した癌の診断）を含む多くのデータを、強力なディープラーニングモデルを教えるのに十分な量で取得することが難しいという事実から生まれました。データセットには、モデルのトレーニングに必要な有益な情報に加えて、解決される問題に関連しない他の情報も含まれていますが、誰かに開示すると有害になる可能性があります。

統合学習は、安全な環境にモデルを格納し、データをどこにも移動せずに教えるための手法です。例を考えてみましょう。

def to_indices(input, l=500):
     indices = list()
     for line in input:
          if(len(line) < l):
              line = list(line) + ["<unk>"] * (l - len(line))
              idxs = list()
              for word in line:
                   idxs.append(w2i[word])
              indices.append(idxs)
return indices

モデルを連邦にしましょう


上記では、最も通常のディープラーニングが実行されました。ここでプライバシーを追加します

前のセクションでは、電子メール分析の例を実装しました。すべてのメールを1か所に配置します。これは古き良き作業方法です（まだ世界中で広く使用されています）。まず、連邦教育の環境を模倣します。ここでは、いくつかの異なる手紙のコレクションがあります。

bob = (train_data[0:1000], train_target[0:1000])
alice = (train_data[1000:2000], train_target[1000:2000])
sue = (train_data[2000:], train_target[2000:])

まだ複雑なことは何もありません。これで、以前と同じトレーニング手順を実行できますが、すでに3つの別々のデータセットで実行されています。各反復の後、ボブ、アリス、スーのモデルの値を平均し、結果を評価します。一部の連携学習方法では、各パッケージ（またはパッケージコレクション）の後に結合することに注意してください。私はコードをできるだけシンプルにすることを決めました：

for i in range(3):
     print("Starting Training Round...")
     print("\tStep 1: send the model to Bob")
     bob_model = train(copy.deepcopy(model), bob[0], bob[1], iterations=1)

     print("\n\tStep 2: send the model to Alice")
     alice_model = train(copy.deepcopy(model),
                                  alice[0], alice[1], iterations=1)

     print("\n\tStep 3: Send the model to Sue")
     sue_model = train(copy.deepcopy(model), sue[0], sue[1], iterations=1)

     print("\n\tAverage Everyone's New Models")
     model.weight.data = (bob_model.weight.data + \
                                     alice_model.weight.data + \
                                     sue_model.weight.data)/3

     print("\t% Correct on Test Set: " + \
             str(test(model, test_data, test_target)*100))
     print("\nRepeat!!\n")

フェデレーションモデルのハック


トレーニングデータセットから情報を抽出する方法の簡単な例を見てみましょう。

フェデレーションラーニングには、2人の大きな問題があり、特に解決が難しいのは、各人がトレーニングの例を少数しか持っていない場合です-速度と機密性。誰かがいくつかのトレーニング例しか持っていない場合（または、あなたに送信されたモデルがいくつかの例（トレーニングパッケージ）でトレーニングされている場合）でも、ソースデータについて多くのことを学ぶことができます。 10,000人いる（そして、誰もが非常に少量のデータを持っている）と想像すると、ほとんどの時間はモデルの送受信に費やされ、それほどトレーニングは行われません（特にモデルが非常に大きい場合）。

しかし、自分たちより先を行かないようにしましょう。ユーザーが1つのパッケージの重みを更新した後で何がわかるかを見てみましょう。

import copy

bobs_email = ["my", "computer", "password", "is", "pizza"]

bob_input = np.array([[w2i[x] for x in bobs_email]])
bob_target = np.array([[0]])

model = Embedding(vocab_size=len(vocab), dim=1)
model.weight.data *= 0

bobs_model = train(copy.deepcopy(model),
                              bob_input, bob_target, iterations=1, batch_size=1)

ボブは自分の受信トレイにメールでモデルを作成してトレーニングします。しかし、たまたま、「自分のコンピュータのパスワードはピザです」というテキストを自分に送ることで、パスワードを保存しました。素朴なボブ！変更された重みを確認した後、ボブの手紙の辞書を理解し（そしてその意味を理解し）ます。

for i, v in enumerate(bobs_model.weight.data - model.weight.data):
