Como eliminar pontos cegos com testes visuais

Uma tradução do artigo foi preparada antes do início do curso Python QA Engineer .


Erros visuais são erros na apresentação do aplicativo. Eles aparecem durante toda a vida útil do aplicativo e geralmente aparecem quando o aplicativo é exibido nas telas de dispositivos móveis de vários tamanhos (em laptops, telefones, tablets, relógios).

A pior parte dos erros visuais é que eles não podem ser "capturados" usando testes automatizados convencionais. Isso ocorre porque a maioria das ferramentas de teste automatizadas são comparadas com o DOM (modelo de objeto de documento) para relatar o status do aplicativo.

Por exemplo, o aplicativo desse tweet não mostra o último dígito do preço, no entanto, o preço total está no DOM. No primeiro caso, o preço real é de 1250 dólares da Nova Zelândia, mas é exibido como 125 dólares da Nova Zelândia. Diferença muito grande!



Infelizmente, os testes automatizados informarão que tudo está em ordem e o preço está correto, mesmo que os usuários vejam uma imagem diferente. Hoje falaremos sobre como detectar esses erros usando testes visuais .

O que é teste visual?


Para detectar erros visuais, nossos scripts de automação precisam de "olhos", ou seja, a capacidade de ver o aplicativo como os usuários o vêem, a fim de verificar sua aparência.

As ferramentas de teste visual nos permitem fazer exatamente isso. Eles capturam uma captura de tela do seu aplicativo quando ele toma o estado desejado e, em seguida, fazem uma nova captura de tela sempre que você executa o teste.

As capturas de tela do teste de regressão são comparadas com a captura de tela "correta". Se forem detectadas incompatibilidades significativas, o teste falhará.

Algumas dessas ferramentas usam inteligência artificial para visualizar capturas de tela e parecer mais com pessoas. Eles são muito mais eficientes do que os métodos básicos de comparação de imagens com base em pixels.

Devo começar tudo de novo?


Quando digo aos engenheiros sobre os benefícios dos testes visuais, eles costumam me perguntar se agora terão que "jogar fora" todos os testes existentes e começar de novo. A resposta é não.

O teste visual pode ser integrado ao seu sistema de teste existente. Muitas ferramentas de teste visual suportam todas as principais linguagens de programação, bem como estruturas de automação como Selenium WebDriver, Cypress, Appium e outras.

Apesar de as asserções visuais complementarem as asserções funcionais existentes, a vantagem do teste visual é que ele também realiza muitos testes funcionais .

Por exemplo, no exemplo do Wellington Suite acima, os testes atuais provavelmente incluem afirmações de preços. Obviamente, você pode deixá-los e adicionar testes visuais a eles para garantir que os preços sejam exibidos corretamente. Uma comparação feita por algoritmos de teste visual já garante que o preço esteja correto e seja exibido corretamente. Como resultado, verifica-se que, para testes, você precisa escrever menos código e, quanto menos código, mais fácil é manter.

E se eu não quiser tirar uma captura de tela da tela inteira?


Apesar de uma captura de tela de toda a tela poder ajudá-lo a encontrar os erros que você nem imaginava, essa abordagem do teste visual nem sempre é verdadeira. Às vezes, a tela que você está testando ainda está em desenvolvimento e, portanto, frequentemente muda, ou há dados na tela que não são relevantes para seus testes.

Ignorando escopos


Algumas ferramentas de teste visual permitem ignorar áreas específicas da tela. Esse método é adequado para as áreas que não estão relacionadas ao teste, por exemplo, barras de status, linhas de data, publicidade etc.



Quadros de área


Acontece que não há uma área específica que precise ser ignorada, mas há uma área que precisa ser testada, e aqui você também ajudará nos testes visuais. Anotando uma captura de tela ou definindo o localizador de área no código, você pode verificar apenas uma área limitada.

Tenha cuidado com os limites


A verificação da tela inteira garantirá que nada está faltando nela, pois, de fato, essa verificação inclui as coisas que você talvez não cubra com os testes. Ao restringir o escopo dos testes visuais, atenção especial deve ser dada às áreas entre os limites que você pode ter perdido.

Aconselho expandir a área de teste o máximo possível (por exemplo, ocupar uma seção inteira, em vez de um elemento específico dentro de uma seção) e, se necessário, combinar a afirmação visual com a tradicional.

E os dados dinâmicos que variam de teste para teste?


Você pode pensar que o teste visual não é aplicável onde os dados mudam de teste para teste, mas não é. Usando inteligência artificial, o teste visual pode reconhecer os padrões que compõem o layout da tela e, em seguida, usar um algoritmo de comparação diferente que verificará se o layout permanece inalterado durante a regressão, independentemente dos dados contidos nele.

Esse método é ideal para aplicativos como sites de notícias, redes sociais e qualquer outro onde os dados não sejam estáticos.

O teste visual é aplicável apenas a aplicativos da web?


O teste visual não se limita a aplicativos da web. O teste visual pode ser usado para aplicativos móveis , sistemas de design como Storybook, arquivos PDF e imagens individuais.

O teste de plataforma cruzada também pode ser feito usando grades visuais. Essas grades tornam os testes visuais mais rápidos e inteligentes, permitindo executar testes em muitos dispositivos com diferentes tamanhos de tela, onde os erros estão ocultos!

As grades visuais diferem das normais porque não precisam executar o script inteiro para cada configuração. Em vez disso, a grade visual executa o script em uma configuração, captura o estado do seu aplicativo e duplica esse estado específico em todas as configurações suportadas, economizando tempo e esforço na gravação e execução de testes entre plataformas.

Como começar com o teste visual


Existem muitas ferramentas gratuitas que você pode começar a usar agora. Ao escolher um deles, verifique se a ferramenta oferece a flexibilidade que atende às especificidades de seu teste.
