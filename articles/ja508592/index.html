結論



要約すると、ここに私が最も役立つと思うカールオプションがあります。



  • -X



    リクエストメソッドを定義します。 curl -X POST url



  • -d



    PUTおよびPOSTリクエストのリクエスト本文を文字列として定義します。@



    ファイルからデータを抽出するために使用します
  • -H



    たとえば、リクエストヘッダーを定義します curl -H "Authorization: my-secret-token" url



  • -u



    標準認可の認証データ
  • -O



    リクエストボディをファイルに保存します
  • -i



    ヘッダーを含む完全な回答を表示します


Curlは、サードパーティのAPIであるか、開発中のAPIであるかに関係なく、コマンドラインを介してAPIを操作するための便利なツールです。クイックテストでは、構文に精通している場合、curlはNode.jsのAxiosやPostmanのクエリチューニングよりも優れています。
