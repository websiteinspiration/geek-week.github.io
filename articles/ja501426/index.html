Unison：2つのサーバーでのディレクトリの双方向同期の設定と自動化


Getty Images / iStockphoto

2つのサーバー上のディレクトリを、オンボードのオペレーティングシステムのLinuxファミリーと同期する問題は、専用のツールを使用すると簡単に解決できます。 Unisonでこれを行う方法を見てみましょう。

おそらく、複数のサーバーがあり、これらのサーバー上の特定のディレクトリを同期するタスクに直面しているとします。たとえば、一部のアプリケーションが関連情報にアクセスできるように、これらの各サーバーの/データディレクトリを同期する必要があります。または、あるサーバーから別のサーバーに定期的にデータを同期してバックアップする必要があるだけかもしれません。

Unisonはrsync同期ユーティリティに似ていますが、それとは異なり、双方向同期をサポートしています。つまり、ファイルの2つのコピーを同期し、変更内容に応じて各コピーを更新できます。

サーバーでUnisonを使用する場合は、openssh-serverやsshなどのパッケージをインストールすることが望ましいです。これは、セキュリティ上の理由から、SSHプロトコルを使用して同期を行うのが最適だからです。ただし、ネットワークのセキュリティに自信がある場合（たとえば、サーバーがパスワードなしでSSH認証を使用できる場合）、気にする必要はありません。

Unisonを使用するために他に必要なもの


2つのUbuntuサーバーを使用したプロセス全体を例として示します（両方とも18.04）。Unisonを他のディストリビューションでインストールして使用できますが、インストールコマンドを変更する必要があります。そうしないと、Unisonを標準リポジトリからインストールできます（もちろん、ユーザーにはsudo権限が必要です）。

これらのサーバーを使用します。

  • server1-192.168.1.6
  • server2-192.168.1.19

Unisonのインストール方法


最初に注意することは、Unisonのインストールです。これは両方のサーバーで行う必要があります。両方のサーバーに移動して、次のコマンドを入力します。

sudo apt-get install unison unison-all -y

Unisonのインストールが完了するまで待ち、続行します。

SSHキーを生成してコピーする方法


まず、server1に対してのみSSH鍵を生成します。これを行うには、次のコマンドを使用します。

ssh-keygen -t rsa

パスワードの入力を求められたら、Enterキーを押します。
キーが生成されたら、次のコマンドを使用してそれをserver2にコピーします。

ssh-copy-id 192.168.1.19

キーがコピーされると、ビジネスに直接進むことができます。

Unisonの使い方


テスト目的で、サーバーごとに1つのテストディレクトリを作成します。server1のコマンド：

sudo mkdir -p /data1

以下のためのSERVER2：

sudo mkdir -p /data2

さらに、両方のサーバーで、作成されたディレクトリの所有者の名前を変更する必要があります。変更しないと、Unisonはそこに何も書き込むことができません。所有者として、Unisonコマンドを実行するユーザーを指定します。

sudo chown -R $USER.$USER /data2

server1 についても同じようにします。

sudo chown -R $USER.$USER /data1

/ data1に いくつかのテストファイルを配置します。

touch /data1/{test1,test2,test3}

次のコマンドでディレクトリを同期化します（server1で実行します）。

unison /data1 ssh://192.168.1.19//data2

これらのルートディレクトリを初めて同期しようとしているため、同期に時間がかかる可能性があるという警告が表示されます（ただし、この場合、最小ボリュームのテストファイルを3つしかディレクトリに追加していないため、これは起こりません）。本番環境でのバックアップの場合、最初の起動には実際に時間がかかる可能性があります。

だから次回はお茶を飲む。それまでの間、Enterキーを押してプロセスを開始します。これを行った後、各ファイルの同期を確認するよう求められます。終了したら、「y」を入力して続行します。

3つのテストファイルのみを同期するため、これは非常に迅速に行われ、bashシェルに戻ります。ファイルが同期されていることを確認するには、server2に移動します次のコマンドを入力します。

ls /data2

さらに、システムはtest1、test2およびtest3ファイルが/ data2ディレクトリにあると報告します



。成功しました：Unisonが両方のディレクトリを同期しました。

ユーザーの操作を必要とせずにUnisonを起動する方法


Unisonを起動するたびに、各ディレクトリファイルの同期を常に確認したくないと思います。これは、ディレクトリを多数のファイルと同期するときに特に煩わしいものです。したがって、自動化が必要です。server1に移動して、次のコマンドを入力します。

nano ~/.unison/default.prf

編集のために構成ファイルが開いたら、2行追加します。

auto=true
batch=true

次に、ファイルを保存して閉じます。
今ユニゾンはささいなことであなたを気にしないでしょう。しかし、それだけではありません。

Cronスケジューラーのタスクを作成する方法


どこかで間違いを犯したり、サーバーのディレクトリ構造の変更を考慮に入れなかったりする可能性があるため、手動で同期を開始することはほとんどないと思います。自動化を構成するには、server1の cronジョブを作成する必要があります。
同期を開始するスクリプトを作成してみましょう。

sudo nano /usr/local/bin/unisonsync

このファイルに以下を追加する必要があります。 

#!/bin/bash/
unison /data1 ssh://192.168.1.19//data2

次に、スクリプトに実行権限を与える必要があります。

sudo chmod ugo+x /usr/local/bin/unisonsync

次のコマンドでCronのジョブを作成します。

crontab -e

そのタイプの後：

*/5 * * * * /usr/local/bin/unisonsync &> /dev/null

これで、同期は5分ごとに自動的に開始されます。

Unisonを使用すると、2つのLinuxサーバー上のディレクトリを簡単に同期できることを示しました。これを確認するには、両方のサーバーの対応するテストディレクトリ/ data1および/ data2にファイルを追加します。同期が機能していることを確認したら、運用サーバーで戦闘モードで使用できます。
