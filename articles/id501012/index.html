Bagaimana saya mencari mesin anak untuk blog



Saya banyak menulis di Internet dan saya ingin membuat blog standalon pribadi saya sendiri di mana Anda dapat menulis catatan kecil dan artikel yang tidak cocok untuk situs lain. Masalah utama dari blog pribadi adalah bahwa saya terlalu malas untuk menulis posting . Untuk mencoret-coret beberapa baris di Wordpress apa pun, Anda memerlukan terlalu banyak gerakan. Saya mencoba beberapa mesin yang berbeda dan setiap kali saya merasa bahwa saya tidak ingin menggunakannya, jauh lebih cepat dan lebih mudah untuk menulis di buku catatan seperti Evernote atau Notion. Jadi saya mulai mencari mesin blog yang sempurna yang memenuhi semua kebutuhan saya.

Apa itu editor WYSIWYG yang normal?



(Video dengan suara) Contoh editor WYSIWYG yang ideal di Notion

⇧ Video dengan demonstrasi editor terbaik menurut saya ⇧

Editor pos adalah hal terpenting dalam mesin, karena Anda menghabiskan waktu paling banyak di dalamnya. WYSIWYG (apa yang Anda lihat adalah apa yang Anda dapatkan) adalah jenis editor di mana teks diformat segera, tanpa beralih antara kode sumber dan versi final. Sungguh menyakitkan bagi saya untuk menonton para penggemar lama sekolah secara manual menulis tag HTML, BBcode, beralih antara pratinjau dan kode sumber sepuluh kali, mengunggah gambar secara terpisah ke hosting dan, sebagai akibatnya, menghabiskan banyak waktu menulis teks. Editor dalam semua jenis Wordpress sudah ketinggalan zaman selama beberapa dekade!

Pembaruan
Sejak 2018, Wordpress telah menggunakan editor Gutenberg baru yang keren, yang tidak lebih buruk dari Notion. Pada saat penulisan, saya tidak tahu tentang ini.


Ternyata beberapa orang tahu bahwa ada editor modern yang dapat membuat penurunan harga secara real time, menempelkan gambar dari clipboard. Anda biasanya dapat melupakan tentang menulis kode untuk tata letak teks.

Persyaratan editorial saya:


  • WYSIWYG - semuanya harus dirender segera, yaitu, setelah memasukkan dua kisi ## dari baris baru, teks harus segera berubah menjadi header. Tidak ada pratinjau dan kode sumber.
  • Dukungan penurunan harga - hal-hal dasar seperti tajuk, daftar, penyorotan kode harus berfungsi. Tidak perlu menyeret mouse ke elemen format teks, jika semuanya dapat dilakukan dari keyboard.

Notion — Evernote




Saya suka Notion . Ini adalah alat paling canggih untuk mencatat, memelihara wiki, menjelaskan proyek, pengelola tugas sederhana, kalender, dan banyak lagi. Notion mendukung kerja tim, ia memiliki perpustakaan templat yang sangat besar dan, jika diinginkan, dapat diubah menjadi apa pun, bahkan dalam CRM.

Tetapi hal terpenting dalam Notion adalah editornya. Ini sempurna untuk menulis posting. Video di atas menunjukkan contoh menulis posting sederhana menggunakan markdown markup, menempelkan kode, gambar dari clipboard. Sebagai perbandingan, mesin Aegeus Ilya Birman digunakan.

Sayangnya, Notion tidak cocok untuk blog publik. Ya, Anda dapat menerbitkan semua halaman di Internet sebagai situs biasa, bahkan di sini adalah contoh situspada gagasan. Tetapi Anda tidak dapat mengacaukan domain Anda, berpindah-pindah halaman terlihat aneh, kadang-kadang terbuka dalam bentuk windows, bukan halaman terpisah.

Notion belum memiliki API publik, tetapi sudah ada implementasi klien, misalnya di Go . Penulis klien ini menulis sebuah kruk untuk menarik halaman dari gagasan dan mengunggahnya ke servernya. Tetapi API pribadi dapat berubah kapan saja dan semuanya akan rusak, metode ini melibatkan penghancuran terus-menerus.

Pro / kontra Notion

✓ Editor ideal
✓ Memiliki pemformatan canggih
✓ Dukungan untuk templat halaman
✗ Tidak ada versi yang di-host sendiri
✗ Kesesuaian situs yang buruk
✗Anda tidak dapat menghubungkan domain Anda
✗ Sampah di URL, Anda tidak dapat mengatur sendiri alamat halaman
✗ Navigasi yang tidak nyaman, munculan, bukan halaman individual
✗ Tidak ada komentar

Halaman Jekyll dan Github - generator situs statis




Jekyll adalah generator situs statis yang populer di kalangan programmer. Ini biasanya digunakan bersama dengan Github Pages . Anda mungkin bertemu dengan halaman beranda orang-orang komputer yang dibuat oleh generator ini. Di Hacker News, setiap situs ketiga menggunakannya. Poin utamanya adalah bahwa tulisan ditulis dalam IDE favorit Anda dalam format penurunan harga dan digunakan via git dengan cara yang sama dengan kode, dan HTML sudah dihasilkan dari mereka di server.

Saya juga mencoba untuk menggunakannya hub.zhovner.com

Sekilas, semuanya tampak mudah, tetapi untuk menulis posting Anda perlu membuat file teks, YEAR-MONTH-DAY-title.mdberi nama dengan cara khusus dalam format dan meletakkannya di folder_posts. Gambar juga harus diurutkan secara manual ke dalam folder dan tentukan tautan ke sana. Diasumsikan bahwa programmer sendiri akan menulis snippet, hooks, macro untuk semua ini, postingannya akan secara otomatis dipanggil, dipindahkan, gambar akan diletakkan di folder, dll.

Ada banyak plugin untuk IDE populer seperti VSCode untuk rendering markdown real-time, penyisipan gambar, dll. Bahkan ada seluruh CMS yang dibuat sebagai add-on di sekitar jekyll, berikut adalah beberapa di antaranya: forestry.io , cloudcannon.com , netlifycms.org/docs/jekyll/. Mereka terhubung langsung ke repositori github dan situs tersebut dapat diedit melalui editor WYSIWYG mereka dan secara otomatis disinkronkan melalui git. Kedengarannya indah, tetapi kenyataannya itu bekerja dengan buruk. Tidak ada satu pun editor yang dekat dengan editor referensi Notion.

Setelah beberapa tahun menggunakan Jekyll, saya mendapati diri saya berpikir bahwa saya hanya tidak menyenangkan dan sulit untuk bekerja dengannya. Saya lupa cuplikan untuk membuat posting baru di editor Teks Sublime saya, saya terlalu malas untuk mengunggah situs melalui Git dan dalam kebanyakan kasus saya hanya memilih untuk tidak menulis daripada melalui semua prosedur ini.

Pro / kontra Jekyll

✓ Kontrol penuh atas situs dan desain
✓ Dapat di-host secara independen
✓ Hanya HTML statis
✓Anda dapat menghubungkan layanan komentar apa saja
✗ Sulit untuk mempublikasikan posting baru
✗ Tidak ada editor normal
✗ Kruk pihak ketiga seperti CMS dan plugin untuk IDE tidak berfungsi dengan baik

Ghost - CMS untuk blog




Ghost adalah CMS open source modis yang relatif baru. Situs ini hanya menjelaskan versi SaaS berbayar, tetapi ada gambar Docker siap pakai yang dapat digunakan dalam satu klik.

Ghost memiliki editor keren yang hampir sama dengan Notion, saya sangat senang ketika saya mencobanya: dukungan normal untuk penurunan harga secara real time, gambar disisipkan dari clipboard, cantik! Setelah sedikit selesai, bahkan penyorotan sintaksis muncul di blok kode.

Saya sudah memutuskan bahwa saya menemukan mesin ideal saya, sampai saya mulai berurusan dengan desain tampilan situs. Ternyata tema default tidak menyiratkan pengaturan apa pun, karena kita semua terbiasa melihat di Wordpress yang sama. Mereka tidak memiliki pengaturan, itu biasa untuk menghidupkan / mematikan menu, footer / header. Mengaduk-aduk galeri topik resmi , saya menyadari bahwa ada selusin di antaranya dan tidak ada yang cocok untuk saya. Hanya ada seratus lima puluh tema tentang Theme Forest for Ghost, melawan 12 ribu untuk Wordpress. Dalam dua hari pencarian, saya tidak dapat menemukan topik yang cocok, semuanya terlalu bodoh atau sudah lama ditinggalkan dan tidak kompatibel dengan versi mesin saat ini. Saya tidak mengerti tata letak dan menulis topik saya sendiri bukan bagian dari rencana saya. Karena itu, saya harus menunda Ghost dan melanjutkan pencarian.

Ghost , , . Affinity . , .

Out of the box, Ghost memiliki banyak fitur menarik seperti langganan berbayar untuk konten tertutup, seperti Patreon, dan banyak plugin yang siap pakai untuk integrasi. Banyak perusahaan menggunakan mesin ini untuk blog korporat mereka . Pengembangan sangat aktif, berkomitmen pada Github setiap hari .

Pro / kontra dari Ghost

✓ Editor hebat
✓ Kontrol penuh atas situs dan desain
✓ Anda dapat menjadi tuan rumah sendiri
✓ Anda dapat menghubungkan layanan komentar apa pun
✗ Tema tanpa pengaturan, hanya mengedit sumbernya
✗ Buat dan edit posting hanya melalui panel admin

Teletype.in - layanan minimalis untuk blog




Teletype.in adalah analog Medium dari pengembang Rusia minimalis dengan dukungan markdown markdown. Dalam gaya, ini menyerupai layanan dari pengembang messenger Telegram - telegra.ph , hanya dengan kemampuan untuk sepenuhnya mendaftar dan blog dalam profil Anda. Anda dapat menghubungkan domain Anda.

Saya sangat menyukai ide dan pendekatan pengembang: editor WYSIWYG yang benar, gambar disisipkan dari clipboard, tidak ada panel admin, setidaknya antarmuka, fokus hanya pada konten. Begitulah cara saya membayangkan mesin blog yang sempurna.

Saya menghubungkan domain dan mencoba mentransfer beberapa artikel dari blog saya ke Jekyll di Teletype.in. Dalam beberapa menit penggunaan, beberapa bug signifikan keluar: Saya tidak sengaja merusak tata letak dan mendapat halaman posting yang tidak berfungsi. Saya senang bahwa pengembang berkomunikasi dalam bahasa Rusia dan aktif merespons dalam obrolan dukungan di Telegram, mereka mengkonfirmasi semua bug dan memperbaikinya dalam beberapa hari. Orang-orang berjanji untuk menambahkan highlight sintaks di blok kode dan secara aktif memodifikasi editor. Sejauh ini, ketidaknyamanan kecil tetap di dalamnya dan itu tidak sebagus editor Notion.

Komentar belum sangat fungsional, Anda tidak dapat menulis secara anonim, pintu masuknya tidak melalui semua jejaring sosial. Ada sesuatu untuk dimodifikasi.

Pro / Kontra Teletype.in

✓ Editor yang baik
✓ Anda dapat menghubungkan domain Anda
✓Mudah menambahkan dan mengedit posting
✓ Anda dapat menentukan URL untuk setiap posting
✗ Tidak ada kontrol atas situs, Anda tidak dapat meng-host host
✗ editor kasar, ada bug
✗ ruang lingkup kecil untuk desain profil

WordPress, Aegean dan semua mesin lainnya


Saya mencoba banyak mesin dan kebanyakan terlihat usang selama beberapa dekade. Masalah utama adalah antarmuka admin didesain ulang dan kurangnya editor WYSIWYG. Saya tidak akan menganalisis CMS populer, karena mereka semua menderita masalah yang dijelaskan di atas.

Bentuk komentar yang benar


Saya merasa tidak nyaman jika saya melihat artikel di mana Anda tidak dapat meninggalkan komentar. Lebih buruk lagi, ketika kemungkinan komentar dipagari oleh langganan berbayar atau pendaftaran kompleks. Bentuk komentar yang benar harus memungkinkan setiap pengunjung untuk mengekspresikan diri mereka semudah mungkin tanpa perlu mendaftar di situs.

Disqus - layanan komentar buruk




Saya memiliki situs yang menggunakan layanan komentar Disqus . Setiap komentar ketiga ada tanda Disqus sebagai spam dan menyembunyikannya, tidak peduli bagaimana saya mencoba mematikan filter spam. Karena itu, orang tidak dapat berkomunikasi secara normal. Urutan komentar terus berubah secara acak, beberapa komentar hilang begitu saja. Ini menarik banyak javascript dan melambat. Secara umum, Disqus adalah penyimpangan terburuk, dan saya tidak menyarankan siapa pun untuk menggunakannya.

Remark42 - layanan komentar yang baik



Remark42 - mesin komentar terbuka untuk penyematan di situs mana pun dari Eugeneumputun. Tersedia sebagai gambar Docker untuk ditempatkan di server Anda. Ini mendukung komentar anonim dan login media sosial. Mendukung markdown markdown. Bahkan ada notifikasi komentar baru kepada admin di Telegram. Contoh referensi mesin komentar yang bagus.
