M理論の多次元性を理解する試み

良い一日。この投稿では、M理論の多次元性についての「哲学者プログラマ」の理解を定式化しようとしました。この資料は「大声で考える」ものであり、科学的であるとは主張していません。

前回の記事の準備で出された質問から始めます。隣接または発生率の2次元配列（行列）とは異なる形式でグラフクリークを提示することは可能ですか？最初に頭に浮かぶのは、多次元配列A [i 1、i 2、i 3、...、i n ]です。ここで、i nはグラフの頂点の番号です。 A [i 1、i 2、i 3、...、i n ] = trueは、すべての頂点が互いに隣接していることを意味します。この表現はあまり便利ではなく、グラフ理論の観点からは何も与えません。しかし、配列のインデックスによって、テーブルの行番号または列番号だけでなく、理解することもできます。 i 1、i 2、i 3とする-宇宙で私たちによく知られているコーディネート、i 4-時間。なぜならタイプ（整数）には制限があります。これらの量をサンプリングする必要があります。座標のサンプリング間隔が1μm、時間-1 nsであるとします。

重要なポイントを処理している場合、レコードA [1,3,9,15] = trueは、15 nsでこのポイントが位置x = 1μm、y = 3μm、z = 9μmの原点を基準にしていたことを意味します。 。配列Aの「真の」値がいくつかあるので、（離散性を補正して）軌道を追跡し、速度、加速度を計算できます。配列要素の値がブール値（存在/不在）ではなく実数になった場合、たとえば、示された座標での調査対象の点の質量の変化を追跡することができます。一定の時間として、一連の値はボリュームボディを表すことができます。

ここで、ポイントがその軸を中心に回転できるようにするコンポーネントがあるとします。はい、これは重要なポイントの概念と矛盾しますが、私たちは空想的です。配列の形式はA [i 1、i 2、i 3、i 4、i 5 ]で、i 5は回転角度の値です。基本的に、データを処理する架空のプログラムでは何も変更されていません。

いわば悪化する。さらに測定を追加して、A [i 1、i 2、i 3、...、i 11]。この場合、追加のインデックスの物理的な意味を理解しているかどうかは重要ではありません。ある時点でのポイントの状態を表す配列を取得します。ポイントが文字列またはブレーンの一部であると想定すると、値A [i 1、i 2、i 3、...、i 11 ]は、振動の位相の値と等しく設定できます。

配列Aの多くの値を持つことで、理論的にはいつでも文字列の状態を説明できます。いくつかの配列Aのレコード（構造）を形成したら、サブアトミックレベルに進みます。このレコードには、スピン、電荷、質量などの形で追加のプロパティがあります。原子レベルは多くのサブアトミックレコードなどで構成され、巨視的レベルまで続きます。

多次元性の提案された理解に対する追加と修正は歓迎されます。
