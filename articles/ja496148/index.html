Game of Thronesの例を使用したES6 +の構造設計パターン



こんにちは！

構造設計パターンは、柔軟性と効率を維持するために、オブジェクト間の関係の大規模なシステムを構築するために使用されます。それらのいくつかをゲーム・オブ・スローンズに言及して見てみましょう。

ソフトウェア開発では、最も一般的な問題を解決するために設計パターンが使用されます。これらは、アプリケーションのテストとバグの修正の過程で開発者が長期間にわたって開発したベストプラクティスを表しています。

この記事では、構造パターンについて説明します。これらは、インスタンスと対話する簡単な方法を定義することによってアプリケーションを設計するように設計されています。

最も一般的なパターンは次のとおりです。

  • アダプタ
  • デコレータ
  • ファサード
  • アダプティブ（軽量（要素）、フライウェイト）
  • 代理

アダプタ


アダプターは、あるクラスのインターフェースを別のクラスに変換（転送）できるテンプレートです。これにより、クラスの連携が可能になります。これは、互換性のない構造のために、多くの場合不可能です。

Targaryenが自由に使えるすべての部隊（FlawlessとDragons）と戦うことを決定し、Daenerysがそれぞれと相互作用する方法を探していると想像してください。

以下が必要です。

  • Unsulliedクラス
  • ドラゴンクラス
  • Dragonクラスのburnメソッドをkill共通メソッドに渡すアダプタークラス

class Unsullied {
    constructor(name) {
        this.name = name
        this.kill = this.kill.bind(this)
    }

    kill() {
        console.log(`Unsullied ${this.name} kill`)
    }
}

class Dragon {
    constructor(name) {
        this.name = name
        this.burn = this.burn.bind(this)
    }

    burn() {
        console.log(`Dragon ${this.name} burn`)
    }
}

class DragonAdapter extends Dragon {
    kill() {
        this.burn()
    }
}

(() => {
    const Army = [
        new DragonAdapter('Rhaegal'),
        new Unsullied('Grey worm'),
        new DragonAdapter('Drogon')
    ]
    Army.forEach(soldier => soldier.kill())
})()

使用例


  • 新しいコンポーネントまたはモジュールが既存のアプリケーションで動作する必要がある場合、またはコードの一部がリファクタリングの結果として改善されているが、古い部分と相互作用する必要がある場合。
  • 二次的な問題を解決するためのライブラリを見つけ、それをアプリケーションに統合したい場合。

デコレータ


デコレータは、既存のクラスに動作または機能を動的に追加するように設計されたテンプレートです。これは、サブ分類の代替手段です。

ジョフリー王に出された飲み物が毒かどうかをチェックしたいとします。

以下が必要です。

  • キングクラス
  • ジョフリー王のインスタンス
  • ドリンククラス
  • クラス毒入り飲料
  • 王の命を救うisNotPoisoned関数

function isNotPoisoned(t, n, descriptor) {
    const original = descriptor.value
    if(typeof original === 'function') {
        descriptor.value = function(...args) {
            const [drink] = args
            if(drink.constructor.name === 'poisonedDrink') throw new Error('Someone wants to kill the king')
            return original.apply(this, args)
        }
    }
    return descriptor
}

class PoisonedDrink {
    constructor(name) {
        this.name = name
    }
}

class Drink {
    constructor(name) {
        this.name = name
    }
}

class King {
    constructor(name) {
        this.name = name
    }

    //     
    //      
    @isNotPoisoned
    drink(drink) {
        console.log(`The king drank ${drink}`)
    }
}

(() => {
    const joffrey = new King('Joffrey Baratheon')
    const normalDrink = new Drink('water')
    const poisonedDrink = new Drink('poisoned water')
    joffrey.drink(normalDrink)
    joffrey.drink(poisonedDrink)
})()

ファサード


ファサード-図書館で広く使用されているテンプレート。これは、統一されたシンプルなインターフェースを提供し、そのサブシステムまたはサブクラスの複雑さを隠すために使用されます。

野蛮人との戦いで軍隊を支配したいと想像してください。私たちの軍の実例には、騎兵、兵士、巨人を動かす方法があります。ただし、これらのメソッドを個別に呼び出さなければならず、時間がかかります。どうすれば軍の管理を容易にすることができますか？

以下が必要です。

  • 軍の標本
  • ArmyFacadeクラス

class Horse {
    constructor(name) {
        this.name = name
    }
    attack() {
        console.log(`Infantry ${this.name} attack`)
    }
}

class Soldier {
    constructor(name) {
        this.name = name
    }
    attack() {
        console.log(`Soldier ${this.name} attack`)
    }
}

class Giant {
    constructor(name) {
        this.name = name
    }
    attack() {
        console.log(`Giant ${this.name} attack`)
    }
}

class ArmyFacade {
    constructor() {
        //  ,           
        this.army = [];
        (new Array(10)).fill().forEach((_, i) => this.army.push(new Horse(i + 1)));
        (new Array(10)).fill().forEach((_, i) => this.army.push(new Soldier(i + 1)));
        (new Array(1)).fill().forEach((_, i) => this.army.push(new Giant(i + 1)));
        this.getByType = this.getByType.bind(this)
    }
    getByType(type, occurency) {
        return this.army.filter(el => {
            return el.constructor.name === type && occurency-- > 0
        })
    }
    attack(armyInfo = {}) {
        const keys = Object.keys(armyInfo)
        let subArmy = []
        keys.forEach(soldier => {
            switch(soldier) {
                case 'horse': {
                    subArmy = [...subArmy, ...this.getByType('Horse', armyInfo.horse)]
                    break;
                }
                    case 'soldier': {
                    subArmy = [...subArmy, ...this.getByType('Soldier', armyInfo.soldier)]
                    break;
                }
                    case 'giant': {
                    subArmy = [...subArmy, ...this.getByType('Giant', armyInfo.giant)]
                    break;
                }
            }
        })
        subArmy.forEach(soldier => soldier.attack())
    }
}

(() => {
    const army = new ArmyFacade()
    army.attack({
        horse: 3,
        soldier: 5,
        giant: 1
    })
})()

class WhiteWalker {
    constructor({
        sprite,
        someOtherBigInformation
    }) {
        this.sprite = sprite
        this.someOtherBigInformation = someOtherBigInformation
        this.state = 'alive'
        this.resurrect = this.resurrect.bind(this)
        this.kill = this.kill.bind(this)
        this.getState = this.getState.bind(this)
    }
    kill() {
        this.state = 'dead'
    }
    resurrect() {
        this.state = 'resurrected'
    }
    getState() {
        return this.state
    }
}

const whiteWalker = new WhiteWalker({
    sprite: Date.now()
})

class WhiteWalkerFlyweight {
    constructor(position, name) {
        this.position = position
        this.name = name
        this.whiteWalker = whiteWalker
    }
    getInfo() {
        console.log(`The White Walker ${this.name} whit sprite ${this.whiteWalker.sprite} is ${this.whiteWalker.state}`)
    }
    getFatherInstance() {
        return this.whiteWalker
    }
}

(() => {
    const myArmy = []
    for(let i = 0; i < 10; i++) {
        myArmy.push(new WhiteWalkerFlyweight({
            x: Math.floor(Math.random() * 200),
            y: Math.floor(Math.random() * 200),
        }, i + 1))
    }
    myArmy.forEach(soldier => soldier.getInfo())
    console.log('KILL ALL')
    const [onOffWhiteWalker] = myArmy

プロキシ


プロキシテンプレートは、その名前が示すように、このオブジェクトへのアクセスを制御するためのアドインまたは別のオブジェクトの代用として使用されます。

セルセイ女王が許可なしに100人以上の兵士の募集を禁止する命令を出したと想像してください。これをどのように実装しますか？

以下が必要です。

  • クラス兵士
  • プロセス制御用のArmyProxyクラス
  • 許可のためのCerceiクラスのインスタンス

class Soldier {
    constructor(name) {
        this.name = name
    }
    attack() {
        console.log(`Soldier ${this.name} attack`)
    }
}

class Queen {
    constructor(name) {
        this.name = name
    }
    getConsent(casualNumber) {
        return casualNumber %2 === 0
    }
}

class ArmyProxy {
    constructor() {
        this.army = []
        this.available = 0
        this.queen = new Queen('Cercei')
        this.getQueenConsent = this.getQueenConsent.bind(this)
    }

    getQueenConsent() {
        return this.queen.getConsent(Math.floor(Math.random() * 200))
    }

    enrollSoldier() {
        if(!this.available) {
            const consent = this.getQueenConsent()
            if(!consent) {
                console.error(`The queen ${this.queen.name} deny the consent`)
                return
            }
            this.available = 100
        }
        this.army.push(new Soldier(this.army.length))
        this.available--
    }
}

(() => {
    const myArmy = new ArmyProxy()
    for(let i = 0; i < 1000; i++) {
        myArmy.enrollSoldier()
    }
    console.log(`I have ${myArmy.army.length} soldiers`)
})()

ユースケース


  • 使用したいオブジェクトが遠くにある（深くネストされている）場合、クライアントに影響を与えないようにロジックをプロキシに保存します。
  • 実際の結果を見込んで概算したい場合、計算に時間がかかる
  • ロジックに干渉することなく、オブジェクトのアクセスまたは作成を制御したい場合
