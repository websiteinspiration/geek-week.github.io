SnortまたはSuricata。パート1：企業ネットワークを保護するための無料のIDS / IPSを選択する

むかしむかし、ローカルネットワークを保護するには通常のファイアウォールとウイルス対策プログラムで十分でしたが、最近のハッカーや最近発売されたマルウェアの攻撃に対しては、このようなセットは十分に効果的ではありません。古き良きファイアウォールはパケットヘッダーのみを分析し、一連の正式なルールに従ってそれらをスキップまたはブロックします。彼はパッケージの内容について何も知らないため、攻撃者の外部からの正当なアクションを認識できません。ウイルス対策プログラムは常にマルウェアを検出するわけではないため、管理者は異常なアクティビティを監視し、感染したホストをタイムリーに隔離する必要があります。



会社のITインフラストラクチャを保護する方法はたくさんあります。今日は、高価な機器やソフトウェアのライセンスを購入せずに実装できるオープンソースの侵入検知および警告システムについて説明します。

IDS / IPS分類


IDS（侵入検知システム）-ネットワークまたは別のコンピューターシステムでの不審なアクティビティを記録するように設計されています。彼女はイベントログを保持し、それらに関する情報セキュリティを担当する従業員に通知します。次の要素は、IDSの一部として区別できます。

  • ネットワークトラフィック、さまざまなログなどを表示するためのセンサー 
  • 取得したデータの有害な影響の兆候を検出する分析サブシステム。
  • 一次イベントと分析結果の蓄積のためのストレージ。
  • 管理コンソール。

当初、IDSは場所によって分類されていました。IDSは、個々のノード（ホストベースまたはホスト侵入検知システム-HIDS）の保護または企業ネットワーク全体（ネットワークベースまたはネットワーク侵入検知システム-NIDS）の保護に焦点を当てることができました。いわゆる言及する価値がありますAPIDS（アプリケーションプロトコルベースのIDS）：特定の攻撃を検出するためにアプリケーションレベルのプロトコルの限定されたセットを監視し、ネットワークパケットの詳細な分析は行いません。このような製品は通常プロキシを連想させ、特定のサービスを保護するために使用されます：WebサーバーとWebアプリケーション（たとえば、PHPで書かれた）、データベースサーバーなど。このクラスの代表的なものは、Apache Webサーバーのmod_securityです。

私たちは、幅広い通信プロトコルとディープパケットインスペクションテクノロジーをサポートするユニバーサルNIDSに関心を持っています。彼らは、データリンク層から始めて、通過するすべてのトラフィックを監視し、さまざまなネットワーク攻撃や、情報への不正アクセスの試みを識別します。多くの場合、このようなシステムは分散アーキテクチャによって区別され、さまざまなアクティブなネットワーク機器と対話できます。最新のNIDSの多くはハイブリッドであり、いくつかのアプローチを組み合わせていることに注意してください。構成と設定に応じて、1つのノードまたはネットワーク全体の保護など、さまざまなタスクを解決できます。さらに、アンチウイルスパッケージがワークステーションのIDS機能を引き継ぎ、情報を盗むトロイの木馬の拡散により、これは多機能ファイアウォールになり、疑わしいトラフィックを認識してブロックする問題も解決します。

最初は、IDSはマルウェアのアクション、ポートスキャナーの操作、またはユーザーによる企業のセキュリティポリシーの違反のみを検出できました。特定のイベントが発生したとき、彼らは管理者に通知しましたが、すぐに攻撃を認識するだけでは不十分であることが明らかになりました-攻撃をブロックする必要があります。そのため、IDSはIPS（侵入防止システム）に変わりました-ファイアウォールと相互作用できる侵入防止システム。

IDS開発の歴史


インターネットと企業ネットワークの急速な発展の時代は、前世紀の90年代に始まりましたが、専門家は高度なネットワークセキュリティテクノロジーに少し早く戸惑いました。1986年、ドロシーデニングとピーターノイマンはIDES（侵入検知エキスパートシステム）モデルを公開しました。これは、最新の侵入検知システムの基礎となりました。彼女はエキスパートシステムを使用して、既知のタイプの攻撃、統計的手法、ユーザー/システムプロファイルを特定しました。IDESは、Sunワークステーションで実行され、ネットワークトラフィックとアプリケーションデータをチェックしました。1993年、新世代の侵入検知エキスパートシステムであるNIDES（次世代侵入検知エキスパートシステム）がリリースされました。

1988年のDenningとNeumannの研究に基づいて、P-BESTとLISPを使用するMIDAS（Multics侵入検知およびアラートシステム）エキスパートシステムが登場しました。同時に、統計的手法に基づくヘイスタックシステムが作成されました。別のW＆S統計異常検出器（Wisdom＆Sense）は、1年後にロスアラモス国立研究所で開発されました。業界の発展は速かった。たとえば、1990年に、TIMシステム（時間ベースの帰納的マシン）は、順次ユーザーパターン（一般的なLISP言語）の帰納的トレーニングを使用して異常検出をすでに実装しています。異常検出用のNSM（ネットワークセキュリティモニター）はアクセスマトリックスを比較し、ISOA（情報セキュリティ担当者のアシスタント）はさまざまな検出戦略をサポートしました：統計的方法、プロファイル検証とエキスパートシステム。 AT＆T Bell Labsで作成されたComputerWatchシステムは検証に統計的手法とルールを使用し、カリフォルニア大学の開発者は1991年に最初の分散IDSプロトタイプを受け取りました-DIDS（分散侵入検知システム）もエキスパートシステムでした。

当初、IDSは独自仕様でしたが、すでに1998年には国立研究所でした。バークレーのローレンスは、独自のルール言語を使用してlibpcapデータを分析するオープンソースシステムであるBro（彼女は2018年にZeekに改名）をリリースしました。その年の11月に、libpcapを使用したAPEパケットスニファが登場しました。これは、1か月後にSnortに名前が変更され、後に本格的なIDS / IPSになりました。同時に、独自のソリューションが数多く登場し始めました。

SnortとSuricata


多くの企業が無料のオープンソースIDS / IPSを支持しています。長い間、標準ソリューションはすでにSnortによって検討されていましたが、現在ではSuricataによって置き換えられています。それらの長所と短所をさらに詳しく検討します。 Snortは、署名方式の利点とリアルタイムの異常検出を組み合わせています。 Suricataでは、シグネチャによる攻撃の認識以外の方法も使用できます。このシステムは、Snortプロジェクトから分離し、バージョン1.4以降のIPS機能をサポートする開発者のグループによって作成されました。Snortでは、侵入を防止する機能が後で登場しました。

2つの人気のある製品の主な違いは、SuricataがGPUを使用してIDSモードでコンピューティングできることと、より高度なIPSがあることです。システムはもともとマルチスレッド用に設計されましたが、Snortはシングルスレッド製品です。長い歴史と継承されたコードのため、マルチプロセッサ/マルチコアハードウェアプラットフォームを最適に使用できませんが、通常の汎用コンピューターのSuricataでは、最大10 Gb /秒のトラフィックを処理できます。2つのシステムの類似点と相違点については長い間議論することができますが、Suricataエンジンの方が高速ですが、チャネルが広すぎないことは問題ではありません。

導入オプション


IPSは、システムが制御するネットワークセグメントを監視できるように配置する必要があります。ほとんどの場合、これは専用のコンピューターであり、その1つのインターフェイスは境界デバイスの後に接続され、それらを介して安全でないパブリックネットワーク（インターネット）を「監視」します。別のIPSインターフェイスが保護セグメントの入力に接続されているため、すべてのトラフィックがシステムを通過して分析されます。より複雑なケースでは、いくつかの保護されたセグメントが存在する可能性があります。たとえば、企業ネットワークでは、インターネットからアクセス可能なサービスを備えた非武装地帯（DMZ）が割り当てられることがよくあります。



このようなIPSは、パスワードクラッキング、メールサーバー、Webサーバー、またはスクリプトの脆弱性の悪用によるポートのスキャンやハッキングの試み、および他の種類の外部攻撃を防ぐことができます。ローカルネットワーク上のコンピューターがマルウェアに感染している場合、IDSは外部ボットネットサーバーへの接続を許可しません。内部ネットワークのより深刻な保護には、おそらく、ポートの1つに接続されたIDSインターフェイスのトラフィックをミラーリングできる分散システムと高価なマネージドスイッチによる複雑な構成が必要になります。

多くの場合、企業ネットワークは分散型サービス拒否（DDoS）攻撃の影響を受けます。最近のIDSはそれらを処理できますが、上記の展開オプションはここではほとんど役に立ちません。システムは悪意のあるアクティビティを認識し、偽のトラフィックをブロックしますが、そのためには、パケットは外部インターネット接続を経由してネットワークインターフェイスに到達する必要があります。攻撃の強度によっては、データチャネルが負荷に対応できず、攻撃者の目標が達成されます。そのような場合は、より効率的なインターネット接続が知られている仮想サーバーにIDSを展開することをお勧めします。 VPNを介してVPSをローカルネットワークに接続し、それを介してすべての外部トラフィックのルーティングを構成する必要があります。次に、DDoS攻撃の場合、プロバイダーへの接続を介してパケットを送信する必要はなく、外部ホストでブロックされます。



選択の問題


無料システムの中でリーダーを特定することは非常に困難です。IDS / IPSの選択は、ネットワークトポロジ、必要な保護機能、管理者の個人的な好み、および設定をいじくりたいという彼の願望によって決まります。Snortの歴史は長く、文書化されていますが、Suricataの情報もネットで簡単に見つけることができます。いずれにせよ、システムを習得するには、最終的に見返りを得るいくつかの努力をする必要があります-商用ハードウェアおよびハードウェア-ソフトウェアIDS / IPSは非常に高価であり、常に予算に収まるとは限りません。優れた管理者は常に雇用主を犠牲にしてスキルを向上させるため、費やした時間はそれだけの価値はありません。この状況では、誰もが利益を得ます。次回の記事では、Suricataを展開するためのいくつかのオプションを検討し、実際には、より近代的なシステムを従来のIDS / IPS Snortと比較します。
