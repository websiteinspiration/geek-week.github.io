Googleスプレッドシートからコンテンツを公開するためのシンプルなSlackアプリ

私たちは新しいことに挑戦するのが大好きなので、ITやプログラミングの世界からの興味深い情報へのリンクを同僚と共有することがよくあります。私たちは長年のスラックのユーザーであり、そのようなリンクのために私たちは誰もが自分自身にとって興味深いものを見つけることができる別の教育チャンネルを持っています。しかし、私たちは普通の人なので、仕事の喧噪の中で定期的にこれらのリンクを共有する必要があることを忘れ、チャンネルのアクティビティは消えてしまいます。

事前に収集された情報を自動的に取得し、定期的にリンクを共有できる「ボット」を開発するという興味深い課題に直面しました。私たちはインターネットで既製のソリューションを探していましたが、今ではzapierとslackの統合として表示されています。



しかし、以前はそのような統合を見つけられなかったため、独自の小型自転車を作成することにしました。以下では、それがどのように行われたかを正確に説明します。

問題について考えると、次の解決策が思い浮かび
ました。Googleテーブルから関連する有用なリンクに関する情報を受け取り、それをSlackに送信するJavaScriptコード。cronを使用して、スケジュールの最初のアイテムからスクリプトを呼び出す。順番に始めましょう。

将来的には、Slack APIおよびGoogleスプレッドシートとやり取りするコードを実装するときに、このトークンを目的に使用できるようになります。

次に、Googleスプレッドシートのトークンを取得する必要があります。まず、GoogleスプレッドシートのNode.jsクイックスタートを開きますそして、Google Sheets APIを有効にします。次に、サンプルからコードindex.jsをコピーして実行し、認証プロセスを実行してファイル内の認証情報を取得します。これは、スクリプトでの使用に役立ちます。

スクリプトの実装


今こそ、Google SheetsからSlackにメッセージを公開するスクリプトの開発を始める時です。リンクをクリックすると、完全なスクリプトコードが見つかります。

まず、メッセージをスラックチャネルに送信するコードの一部を分析します。
これは、Slack REST APIを呼び出すことで発生します。

const post = async (text, channel = <CHANNEL ID>) => {
  const uri = `https://slack.com/api/chat.postMessage?token=${SLACK_AUTH_TOKEN}&channel=${channel}&text=${text}`;
  const result = await fetch(encodeURI(uri), {
    headers: {
      'Content-Type': 'application/json',
    },
    method: 'POST',
    body: JSON.stringify({
      channel,
      text,
      as_user: true
    })
  });

  await result.json();
};

Google APIでの承認には、2つの関数（authorize、getNewToken）が使用されます。これらのコードはNode.jsクイックスタートから取得されるため、ここでは詳しく説明しません。

次の興味深いコードブロックは、Googleスプレッドシートから送信する情報を取得することです。これは、次の関数を使用して行われます。

const getMessageAndPost = async (auth, spreadsheetId = <ID  Google >) => {
  const sheets = google.sheets({ version: 'v4', auth });
  sheets.spreadsheets.values.get({
    spreadsheetId,
    range: 'Sheet1!A1:B999',
  }, async (err, res) => {
    if (err) return console.log('The API returned an error: ' + err);
    const rows = res.data.values;
    if (rows.length) {
      // Print columns A and E, which correspond to indices 0 and 4.
      const ix = rows.findIndex(r => !r[1]);
      await post(rows[ix][0]);
      sheets.spreadsheets.values.update({
        spreadsheetId: '18VzilQTEDGXBnaH1f_k-uAfa8Mb470gx32Phir6xQT4',
        range: `Sheet1!B${ix + 1}`,
        valueInputOption: 'RAW',
        requestBody: {
          range: `Sheet1!B${ix + 1}`,
          values: [['x']]
        }

      })
    } else {
      console.log('No data found.');
    }
  });
};

先ほど分解したスクリプトは非常に単純で、チームで数年間そのパフォーマンスを確認しています。このスクリプトをサーバーに展開し、使用するだけです。

このため、私たちの会社では、Digital Oceanで通常のUbuntu VPSを使用しています（最も低電力のVPSでも可能です）。

まず、java.scriptコードを実行するための環境を構成する必要があります。これには、node.jsをインストールする必要があります（インストールガイド）

その後、スケジュールに従ってスクリプトを呼び出すようにcronタスクを構成します。

crontab -e

#### 

24 14 * * 1-5 cd /root/google-sheets-to-slack && node post.js

これらの設定を使用して、/ root /フォルダーからのスクリプトは、月曜日から金曜日の14:24に毎日呼び出されます。

セットアップは終了しましたが、注意深い読者はスクリプトにいくつかの欠陥があることに気づいているはずです。たとえば、スクリプトを呼び出すときにエラーが発生した場合、それは「静かに落ちる」ため、それを知ることはできません。この問題を解決するには、エラー処理とスクリプト呼び出しの結果の出力を同じGoogleスプレッドシートまたはログファイルに追加します。

また、一定の時間にメッセージを送信する代わりに、ランダムな遅延でメッセージを送信できるため、メッセージは「ライブ」ユーザーからのメッセージに近くなります。

ある同僚のもう1つの興味深いアイデアは、金曜日にメッセージが排他的に送信され、より面白くなる2番目のテーブルを作成することでした。

一般的に、改良のアイデアは無数にあり、徐々に実装していきますが、既存の形式では、このスクリプトは非常に使いやすくなっています。
