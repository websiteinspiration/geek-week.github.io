画像を音声に変換-何が聞こえますか？

こんにちはHabr。

このサイトの最近の出版物では、視覚障害者が画像を「見て」、音波を使用して画像を変換できるようにするデバイスについて説明しています。技術的な観点から見ると、その記事では詳細はまったくありませんでした（100万人のアイデアが盗まれた場合はどうなるか）が、コンセプト自体は興味深いように見えました。信号処理の経験があったので、自分で実験することにしました。



その由来、猫の下のファイルの詳細と例。

2Dを1Dに変換


私たちを待っている最初の明らかなタスクは、2次元の「フラット」画像を「1次元」の音波に変換することです。その記事のコメントで示唆されているように、これにはヒルベルト曲線を使用すると便利です。

これは基本的にフラクタルに似ており、画像の解像度を上げてもオブジェクトの相対位置は変化しないという考え方です（オブジェクトが画像の左上隅にあった場合、オブジェクトはそこに残ります）。ヒルベルト曲線の次元が異なると、異なる画像が得られます。N= 5の場合は32x32、N = 6の場合は64x64などです。この曲線に沿って画像を「循環」させると、1次元のオブジェクトである線が得られます。

次の質問は画像のサイズです。私は直感的に大きな画像を撮りたいと思っていますが、大きな「しかし」があります。写真が512x512であっても、262144ピクセルです。各ポイントをオーディオパルスに変換すると、サンプリング周波数44100で6秒というシーケンスが得られますが、これは長すぎるため、たとえばWebカメラを使用して画像をすばやく更新する必要があります。サンプリングレートを高くしても意味がありません。フクロウやコウモリには効果があるかもしれませんが、耳には聞こえない超音波周波数が得られます。その結果、128x128の解像度は、0.37cの長さのインパルスを提供する科学的ポーキングの方法によって選択されました。一方で、リアルタイムでナビゲートするのに十分な速度であり、他方で、耳で信号の形状の変化を捉えるのに十分です。

画像処理


最初のステップは、画像をアップロードしてb / wに変換し、希望のサイズに拡大縮小することです。画像サイズはヒルベルト曲線の次元に依存します。

from PIL import Image
from hilbertcurve.hilbertcurve import HilbertCurve
import numpy as np
from scipy.signal import butter, filtfilt


# Create Hilbert curve
dimension = 7
hilbert = HilbertCurve(dimension, n=2)
print("Hilbert curve dimension:", dimension)  # Maximum distance along curve
print("Max_dist:", hilbert.max_h)  # Maximum distance along curve
print("Max_coord:", hilbert.max_x)  # Maximum coordinate value in any dimension

次のステップは音波を形成することです。もちろん、ここでは、非常に多くのアルゴリズムとノウハウが存在する可能性があります。テストでは、輝度コンポーネントのみを使用しました。もちろん、おそらくもっと良い方法があります。

width, height = img_grayscale.size

sound_data = np.zeros(width*height)
for ii in range(width*height):
    coord_x, coord_y = hilbert_curve.coordinates_from_distance(ii)
    pixel_l = img_data[coord_x][coord_y]

    # Inverse colors (paper-like, white = 0, black = 255)
    pixel_l = 255 - pixel_l

    # Adjust values 0..255 to 0..8192
    ampl = pixel_l*32

    sound_data[ii] = ampl

コードから、すべてが明確であることを願っています。座標（from、distance）関数は、座標（x、y）をヒルベルト曲線上の距離に変換するすべての作業を行い、輝度値Lを反転して色に変換します。

それがすべてではありません。なぜなら画像上に同じ色の大きなブロックが存在する場合があります。これにより、サウンドに「DCコンポーネント」が表示される可能性があります。たとえば、[100,100,100、...]のように、一連の非ゼロ値が長くなります。それらを削除するには、ハイパスフィルター（バターワースフィルター）を50 Hzのカットオフ周波数（ネットワーク周波数との一致はランダム）でアレイに適用します。 scipyライブラリにはフィルターの合成があります。これを使用します。

def butter_highpass(cutoff, fs, order=5):
    nyq = 0.5 * fs
    normal_cutoff = cutoff / nyq
    b, a = butter(order, normal_cutoff, btype='high', analog=False)
    return b, a

def butter_highpass_filter(data, cutoff, fs, order=5):
    b, a = butter_highpass(cutoff, fs, order)
    y = filtfilt(b, a, data)
    return y

# Apply high pass filter to remove dc component

cutoff_hz = 50
sample_rate = 44100
order = 5
wav_data = butter_highpass_filter(sound_data, cutoff_hz, sample_rate, order)

最後のステップは画像を保存することです。なぜなら 1つのパルスの長さは短く、10回繰り返します。たとえば、Webカメラからの実際の繰り返し画像に聴覚的に近くなります。
# Clip data to int16 range
sound_output = np.clip(wav_data, -32000, 32000).astype(np.int16)

# Save
repeat = 10
sound_output_ntimes = np.tile(sound_output, repeat)
wav_name = "ouput.wav"
scipy.io.wavfile.write(wav_name, sample_rate, sound_output_ntimes)

結論


もちろん、このテストは論文ではなく、数時間の自由時間で行われた単なる概念の証明です。しかし、それでも基本的には機能し、違いを耳で感じることはかなり可能です。仮に、おそらく何らかの訓練の後で、そのような音によって宇宙をナビゲートすることを学ぶことが可能かどうかはわかりません。改善や実験のための巨大な分野がありますが、たとえば、ステレオサウンドを使用すると、さまざまな側面からオブジェクトをより適切に分離できます。たとえば、さまざまな周波数で色をエンコードするなど、画像をサウンドに変換する他の方法を実験できます。最後に、これは有望です奥行きを知覚できる3Dカメラの使用（残念ながら、そのようなカメラは利用できません）。ちなみに、単純なOpenCVコードを使用して、上記のアルゴリズムをWebカメラを使用するように変更できます動的な画像を試すことができます。

まあ、いつものように、すべて成功した実験。
