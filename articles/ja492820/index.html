.Net Core API：さまざまなソースからのリクエストでデータを取得する

.Net Coreには組み込みのモデルバインディングメカニズムがあり、コントローラーで入力パラメーターを受け入れるだけでなく、フィールドが入力されたオブジェクトをすぐに受け取ることができます。これにより、モデル検証を使用して、必要なすべてのチェックをそのようなオブジェクトに埋め込むことができます。

APIが機能するために必要なデータは次のとおりです。QueryやBodyからだけでなく、ここからも入手できます。一部のデータはヘッダーから受信する必要があります（私の場合、base64にjsonがありました）。RESTを使用する場合、一部のデータは外部サービスまたはActionRouteから受信する必要があります。Bindingを使用して、そこからデータを取得できます。確かに問題があります。カプセル化を解除せずに、コンストラクターを介してモデルを初期化する場合は、シャーマンする必要があります。

私自身と将来の世代のために、私はバインディングとシャーマニズムを使用するための指示のようなものを書くことにしました。

問題


一般的なコントローラーは次のようになります。

[HttpGet]
public async Task<IActionResult> GetSomeData([FromQuery[IncomeData someData)
{
    var moreData = GetFromHeaderAndDecode("X-Property");
    if (moreData.Id == 0)
    {
        return StatusCode(400, "Nginx doesnt know your id");
    }
    var externalData = GetFromExternalService("http://myservice.com/MoreData");
    if (externalData == null)
    {
        return StatusCode(500, "Cant connect to external service");
    }
    var finalData = new FinalData(someData, moreData, externalData);
    return _myService.Handle(finalData);
}

その結果、次の問題が発生します。

  1. 検証ロジックは、要求オブジェクト、ヘッダーからの要求メソッド、サービスからの要求メソッド、およびコントローラーメソッドによって塗りつぶされます。必要なチェックが確実にそこにあることを確認するには、完全な調査を行う必要があります！
  2. 隣接するコントローラーのメソッドは、まったく同じコードになります。攻撃のプログラミングをコピーして貼り付けます。
  3. 通常、例よりもはるかに多くのチェックがあるため、結果として、唯一の重要な行（ビジネスロジック処理メソッドの呼び出し）がコードのヒープに隠されます。彼を見て、ここで一般的に何が起こっているのかを理解するには、ある程度の努力が必要です。

カスタムバインディング（イージーモード）


問題の一部は、リクエスト処理パイプラインにハンドラーを実装することで解決できます。これを行うには、最初に最終オブジェクトをメソッドにすぐに渡してコントローラーを修正します。ずっとよく見えますよね？

[HttpGet]
public async Task<IActionResult> GetSomeData([FromQuery]FinalData finalData)
{
    return _myService.Handle(finalData);
}

次に、タイプMoreDataのバインダーを作成します。

public class MoreDataBinder : IModelBinder
{
    public Task BindModelAsync(ModelBindingContext bindingContext)
    {
        var moreData = GetFromHeaderAndDecode(bindingContext.HttpContext.Request.Headers);
        if (moreData != null)
        {
            bindingContext.Result = ModelBindingResult.Success(moreData);
        }
        return Task.CompletedTask;
    }
    private MoreData GetFromHeaderAndDecode(IHeaderDictionary headers) { ... }
}

最後に、FinalDataモデルのプロパティにバインダーバインディングを追加して修正します。

public class FinalData
{
    public int SomeDataNumber { get; set; }

    public string SomeDataText { get; set; }

    [ModelBinder(BinderType = typeof(MoreDataBinder))]
    public MoreData MoreData { get; set; }
}

もう良いですが、痔が増えました。特別なハンドラーがあることを知り、すべてのモデルでそれを指定する必要があります。しかし、それは解決可能です。

BinderProviderを作成します。

public class MoreDataBinderProvider : IModelBinderProvider
{
    public IModelBinder GetBinder(ModelBinderProviderContext context)
    {
        var modelType = context.Metadata.UnderlyingOrModelType;
        if (modelType == typeof(MoreData))
        {
            return new BinderTypeModelBinder(typeof(MoreDataBinder));
        }
        return null;
    }
}

スタートアップに登録します。

public void ConfigureServices(IServiceCollection services)
{
    services.AddControllers();
    services.AddMvc(options =>
    {
        options.ModelBinderProviders.Insert(0, new MoreDataBinderProvider());
     });
}

プロバイダーは、キューの順序で各モデルオブジェクトに対して呼び出されます。プロバイダーが目的のタイプを満たしている場合は、目的のバインダーが返されます。そうでない場合は、デフォルトのバインダーが機能します。したがって、MoreDataのタイプを指定するたびに、それが取得されてヘッダーからデコードされるため、モデルの特別な属性を指定する必要はありません。

カスタムバインディング（ハードモード）


これらはすべてすばらしいですが、1つだけ違いがあります。魔法が機能するためには、モデルにセットを持つパブリックプロパティが必要です。しかし、カプセル化はどうですか？リクエストデータをクラウド内のさまざまな場所に転送し、それらがそこで変更されないことを知っている場合はどうなりますか？

問題は、デフォルトのバインダーがないモデルではデフォルトのバインダーが機能しないことです。しかし、何が原因で私たちが自分で書くことができないのでしょうか？
このコードを記述したサービスはRESTを使用せず、パラメーターはQueryとBodyを介してのみ送信され、Get
とPostの2種類のクエリのみが使用されます。したがって、REST APIの場合、処理ロジックは少し異なります。
一般に、コードは変更されないままで、オブジェクトを作成してそのプライベートフィールドを埋めるように、バインダーだけを調整する必要があります。さらに、私は興味のないコメント付きのコードを提供します-猫の下の記事の最後にクラスの完全なリストがあります。

最初に、MoreDataがクラスの唯一のプロパティであるかどうかを確認しましょう。はいの場合は、自分でオブジェクトを作成する必要があります（hello、Activator）。そうでない場合は、JsonConvertが完全に機能し、必要なデータをプロパティに挿入します。

private static bool NeedActivator(IReflect modelType)
{
    var propFlags = BindingFlags.NonPublic | BindingFlags.Public | BindingFlags.Instance;
    var properties = modelType.GetProperties(propFlags);

    return properties.Select(p => p.Name).Distinct().Count() == 1;
}

Bodyを使用したクエリの場合、JsonConvertによるオブジェクトの作成は簡単です。

private static object? GetModelFromBody(ModelBindingContext bindingContext, Type modelType)
{
    using var reader = new StreamReader(bindingContext.HttpContext.Request.Body);
    var jsonString = reader.ReadToEnd();
    var data = JsonConvert.DeserializeObject(jsonString, modelType);
    return data;
}

しかし、クエリを使用する場合は、横にならなければなりませんでした。誰かがもっと美しい解決策を提案できたらうれしいです。

配列を渡すと、同じ名前のいくつかのパラメーターが取得されます。フラット型へのキャストは役立ちますが、シリアライゼーションは配列[]に余分な引用符を付けるため、手動で削除する必要があります。

private static object? GetModelFromQuery(ModelBindingContext bindingContext, Type modelType)
{
    var valuesDictionary = QueryHelpers.ParseQuery(bindingContext.HttpContext.Request.QueryString.Value);
    var jsonDictionary = valuesDictionary.ToDictionary(pair => pair.Key, pair => pair.Value.Count < 2 ? pair.Value.ToString() : $"[{pair.Value}]");

    var jsonStr = JsonConvert.SerializeObject(jsonDictionary).Replace("\"[", "[").Replace("]\"", "]");
    var data = JsonConvert.DeserializeObject(jsonStr, modelType);
    return data;
}

最後に、オブジェクトを作成し、データをそのプライベートプロパティに書き込む必要があります。記事の冒頭で私が話したのは、このシャーマニズムについてでした。私はこの解決策をここで見つけました、それに対して著者に感謝します。

private void ForceSetValue(PropertyInfo propertyInfo, object obj, object value)
{
    var propName = $"<{propertyInfo.Name}>k__BackingField";
    var propFlags = BindingFlags.Instance | BindingFlags.NonPublic;
            
    obj.GetType().GetField(propName, propFlags)?.SetValue(obj, value);
}

そうですね、これらのメソッドを1回の呼び出しで組み合わせます。

public Task BindModelAsync(ModelBindingContext bindingContext)
{
    var moreData = GetFromHeaderAndDecode(bindingContext.HttpContext.Request.Headers);
    if (moreData == null)
    {
        return Task.CompletedTask;
    }

    var modelType = bindingContext.ModelMetadata.UnderlyingOrModelType;
    if (NeedActivator(modelType))
    {
        var data = Activator.CreateInstance(modelType, moreData);
        bindingContext.Result = ModelBindingResult.Success(data);

        return Task.CompletedTask;
    }

    var model = bindingContext.HttpContext.Request.Method == "GET"
                            ? GetModelFromQuery(bindingContext, modelType)
                            : GetModelFromBody(bindingContext, modelType);

    if (model is null)
    {
        throw new Exception("  ");
    }

    var ignoreCase = StringComparison.InvariantCultureIgnoreCase;
    var dataProperty = modelType.GetProperties()
                            .FirstOrDefault(p => p.Name.Equals(typeof(T).Name, ignoreCase));

    if (dataProperty != null)
    {
        ForceSetValue(dataProperty, model, moreData);
    }

    bindingContext.Result = ModelBindingResult.Success(model);
    return Task.CompletedTask;
}

BinderProviderを修正して、必要なプロパティを持つクラスに応答するようにします。

public class MoreDataBinderProvider : IModelBinderProvider
{
    public IModelBinder GetBinder(ModelBinderProviderContext context)
    {
       var modelType = context.Metadata.UnderlyingOrModelType;
       if (HasDataProperty(modelType))
       {
           return new BinderTypeModelBinder(typeof(PrivateDataBinder<MoreData>));
       }
       return null;
    }
    private bool HasDataProperty(IReflect modelType)
    {
        var propFlags = BindingFlags.NonPublic | BindingFlags.Public | BindingFlags.Instance;
        var properties = modelType.GetProperties(propFlags);

        return properties.Select(p => p.Name) .Contains(nameof(MoreData));
    }
}


それで全部です。バインダーは、イージーモードよりも少し複雑であることがわかりましたが、今では、追加の作業なしで、すべてのコントローラーのすべてのメソッドに「外部」プロパティをバインドできます。マイナスの：

  1. プライベートフィールドを持つオブジェクトのコンストラクターは、[JsonConstrustor]属性を指定する必要があります。しかし、これはモデルのロジックに完全に適合し、その知覚を妨げません。
  2. どこかで、ヘッダーからではなくMoreDataを取得する必要がある場合があります。しかし、これは別のクラスを作成することによって扱われます。
  3. チームの他のメンバーは、魔法の存在を認識している必要があります。しかし、ドキュメントは人類を救います。

    {
        /// <summary></summary>
        /// <param name="bindingContext"></param>
        public Task BindModelAsync(ModelBindingContext bindingContext)
        {
            var moreData = GetFromHeaderAndDecode(bindingContext.HttpContext.Request.Headers);
            if (moreData == null)
            {
                return Task.CompletedTask;
            }

            var modelType = bindingContext.ModelMetadata.UnderlyingOrModelType;
            if (NeedActivator(modelType))
            {
                var data = Activator.CreateInstance(modelType, moreData);
                bindingContext.Result = ModelBindingResult.Success(data);

                return Task.CompletedTask;
            }

            var model = bindingContext.HttpContext.Request.Method == "GET"
                            ? GetModelFromQuery(bindingContext, modelType)
                            : GetModelFromBody(bindingContext, modelType);

            if (model is null)
            {
                throw new Exception("  ");
            }

            var ignoreCase = StringComparison.InvariantCultureIgnoreCase;
            var dataProperty = modelType.GetProperties()
                                        .FirstOrDefault(p => p.Name.Equals(typeof(T).Name, ignoreCase));

            if (dataProperty != null)
            {
                ForceSetValue(dataProperty, model, moreData);
            }

            bindingContext.Result = ModelBindingResult.Success(model);

            return Task.CompletedTask;
        }

        private static object? GetModelFromQuery(ModelBindingContext bindingContext,
                                                 Type modelType)
        {
            var valuesDictionary = QueryHelpers.ParseQuery(bindingContext.HttpContext.Request.QueryString.Value);

            var jsonDictionary = valuesDictionary.ToDictionary(pair => pair.Key, pair => pair.Value.Count < 2 ? pair.Value.ToString() : $"[{pair.Value}]");

            var jsonStr = JsonConvert.SerializeObject(jsonDictionary)
                                     .Replace("\"[", "[")
                                     .Replace("]\"", "]");

            var data = JsonConvert.DeserializeObject(jsonStr, modelType);

            return data;
        }

        private static object? GetModelFromBody(ModelBindingContext bindingContext,
                                                Type modelType)
        {
            using var reader = new StreamReader(bindingContext.HttpContext.Request.Body);
            var jsonString = reader.ReadToEnd();
            var data = JsonConvert.DeserializeObject(jsonString, modelType);

            return data;
        }

        private static bool NeedActivator(IReflect modelType)
        {
            var propFlags = BindingFlags.NonPublic | BindingFlags.Public | BindingFlags.Instance;
            var properties = modelType.GetProperties(propFlags);

            return properties.Select(p => p.Name).Distinct().Count() == 1;
        }

        private void ForceSetValue(PropertyInfo propertyInfo, object obj, object value)
        {
            var propName = $"<{propertyInfo.Name}>k__BackingField";
            var propFlags = BindingFlags.Instance | BindingFlags.NonPublic;
            
            obj.GetType().GetField(propName, propFlags)?.SetValue(obj, value);
        }
