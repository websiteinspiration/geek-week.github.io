RobotC言語で超音波距離計とエンコーダーを使用してオブジェクトの幅を決定するアルゴリズムの実装

良い一日！

今日は、超音波距離計とエンコーダーを使用してRobotCプログラミング言語で（VEX EDRプラットフォームを使用して）オブジェクトの幅を決定するアルゴリズムの実装を共有したいと思います。

先史時代


最近、RobotCプログラミング言語を使用して興味深い問題を解決するためのマスタークラスを実施するように勧められました。マスタークラスでは、オブジェクトの幅を決定するタスク、この出版物で共有したいアルゴリズムの実装について検討しました。

エレメンタルベース


実験的なトレーニングの場として、VEXプラットフォームのロボット仮想世界環境、仮想ロボットモデルVEXランチャー、VRCターニングポイント競争要素の仮想モデル（図1）、および最も単純な数式（中心角がわかっている円の弦の長さを決定するため）を使用しました。

仕事


-ロボットに取り付けられた超音波距離計とエンコーダーのみを使用してボールの直径を決定する

理論的根拠


オブジェクトの幅を決定するために、放射パターンの概念を使用しました（図2）。


図2.超音波距離計の感度と指向性パターン

本質的に、図の最端点の接線間の最右端位置から最左端位置に回転するときに形成される角度を決定する必要がありました。
図3.考えてみましょう


。図3を。アルゴリズムの形状

角度の半径とロボットの回転の弧の関係に基づいて、弧の中心角を検索します。最終的な幅の値は、m-円弧の弦に対応します。

入力パラメータ


入力パラメーターとして、私は選択しました：

  • 距離= 100 mm（ロボットが測定を停止する距離。パラメーターは、センサーの視野に入り、測定を妨害する可能性のあるオブジェクトの周囲の他のオブジェクトの存在に基づいて選択されます）
  • d_wheel = 102 mm（ロボットのホイールの直径）
  • d_ball = 102 mm（実際のボールの直径）

エラーがこのタスクの出力パラメーターとして決定されました。これは、メソッドの精度と選択されたパラメーターを示します。

  • エラー= abs（d_ball_exper-d_ball）

プログラム操作アルゴリズム


アルゴリズムの次の段階が特定されました。

  1. オブジェクトへの移動。
  2. オブジェクトの右端のポイントまで回転します。
  3. オブジェクトの左端のポイントまで回転します。
  4. エンコーダから取得した読み取り値を使用した角度の計算。
  5. エラー計算。

アルゴリズムのRobotC実装


まず、仮想ロボットの標準構成を初期化する必要があります。

#pragma config(StandardModel, "VEX Launchbot")

ロボットを右端および/または左端に回転させる機能を実装します。この関数は、ターン後にエンコーダーを返します。これは、次の計算で使用されます。

int turn_to_over_object(int leftSpeed, int rightSpeed, int distance, bool need_object, bool left_side)
{
if(need_object == true)
{
if(left_side == true)
{
while(SensorValue[sonarSensor] > distance)
{
setMultipleMotors(-leftSpeed, leftMotor,frontLeftMotor);
setMultipleMotors(rightSpeed, rightMotor,frontRightMotor);
}
}
else
{
while(SensorValue[sonarSensor] > distance)
{
setMultipleMotors(leftSpeed, leftMotor,frontLeftMotor);
setMultipleMotors(-rightSpeed, rightMotor,frontRightMotor);
}
}
}
else
{
if(left_side == true)
{
while(SensorValue[sonarSensor] < (distance + 1))
{
setMultipleMotors(-leftSpeed, leftMotor,frontLeftMotor);
setMultipleMotors(rightSpeed, rightMotor,frontRightMotor);
}
}
else
{
while(SensorValue[sonarSensor] < (distance + 1))
{
setMultipleMotors(leftSpeed, leftMotor,frontLeftMotor);
setMultipleMotors(-rightSpeed, rightMotor,frontRightMotor);
}
}
}

return getMotorEncoder(leftMotor);
}

関数の引数を詳しく見てみましょう。

  • leftSpeed-左ドライブ速度
  • rightSpeed-正しいドライブ速度
  • rightSpeed-正しいドライブ速度
  • 距離-ロボットが停止したオブジェクトまでの距離
  • need_object-モーションモード：センサーの視野にオブジェクトが必要な場合はTrue、オブジェクトが必要な場合はFalse
  • left_side-ロボットの回転方向。True-左折、False-右折

値need_object = Trueの場合、ロボットがオブジェクトからロボットまでの距離（この例では100 mm）になるまでロボットが回転します。

値need_object = Falseを指定すると、オブジェクトがロボットから距離（距離+ 1）になるまでロボットは回転します。全体としての単位パラメーターは、センサーの視野内にオブジェクトが存在しないことを決定する最終的な精度に影響します（このパラメーターを関数の引数に組み込むことで、プログラムを改善することができます）。

プログラムの本体を書きましょう：

task main()
{
int d_ball = 10.2; //sm
int d_wheel = 10.2;

int dist = 10; //sm
//  
setMotor(armMotor, -50);
wait(1);

while(SensorValue[sonarSensor] > 10)
{
setMultipleMotors(50, leftMotor, rightMotor, frontLeftMotor, frontRightMotor);
}
stopAllMotors();
int enc = 0;
enc = turn_to_over_object(50, 50, dist, false, false);
enc = turn_to_over_object(50, 50, dist, true, true);
resetMotorEncoder(leftMotor);
enc = turn_to_over_object(50, 50, dist, false, true);
stopAllMotors();

float teta = abs(enc) * PI * d_wheel * d_wheel / (dist * 627);
float d_ball_exper = 2 * dist * sin(teta / 2);
	
float error = abs(d_ball_exper - d_ball);
while(1)
{
}
}

最初のwhileループは、ロボットがオブジェクトまでの指定された距離に到達するまで実行されます。次に、右端と左端の位置の間のエンコーダの読み取り値を測定するために、上記で記述した関数を1つずつ呼び出す必要があります（測定する前に、エンコーダをゼロにリセットする必要があることに注意してください）。

角度の式を取得するために、私は次の式に導かれました：
-enc = N * 627、ここで
    Nはホイールの回転数、
    encはエンコーダーの読み取り、
    627はエンコーダーの解像度です=>表現N;
-S = N * l、ここで
    lはホイールの長さ、l = PI * d_wheel * d_wheel => Nを代入し、l => S = enc * PI * d_wheel * d_wheel / 627;
-一方、S = dist * teta、ここで
    Sはロボットが回転した円弧の大きさ、
    tetaは目的の中心角です。
-上記のデータに基づいて、テタ=>テタ= enc * PI * d_wheel * d_wheel /（627 * dist）と     表現し

ます。ボールの目的の直径に等しいコードを見つけるには、次の式を使用します：
d_ball_exper = 2 * dist * sin（teta / 2）

デバッガーで受信した値を確認するには、プログラムの最後で無限のwhileループが必要です。

要約データ


ボールの直径とエラーの次の値は、仮想フィールドで取得されました：

    d_ball_exper = 10.6 cm
    error = 0.4 cm

ご覧のように、取得された幅の測定方法は、使用されるセンサーの精度クラスに許容される小さなエラー値を提供します。エラー値を減らすには、distパラメータの値を選択する（たとえば、ロボットがdistに変わる特定のステップで戻るサイクルを開始する）か、PIDコントローラを使用します。これにより、ブレーキ中にエンコーダの最終値に対するロボットの慣性の影響が軽減されます。
