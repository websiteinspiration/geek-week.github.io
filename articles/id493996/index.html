Infrastruktur pemantauan Lifehacks menggunakan PRTG Network Monitor dari Paessler AG

Kontrol Level Toner


Banyak administrator akan merasa sangat praktis untuk mengontrol level toner dalam kartrid semua printer menggunakan Monitor Jaringan PRTG. Tetapi tidak semua perangkat mendukung pengidentifikasi objek (OID) menggunakan sensor SNMP. Namun, ada opsi untuk memantau level toner menggunakan PRTG.



Beberapa printer tidak mendukung pengenal objek (OID) yang dibangun ke dalam sensor printer PRTG, oleh karena itu, ketika mencoba meminta data, pesan kesalahan SNMP No. 222 muncul. Dalam kasus seperti itu, sensor PRTG SNMP bawaan (tabel khusus) akan membantu menyelesaikan masalah. Dalam instalasi bahasa Inggris, ini ditetapkan sebagai SNMP Custom Table Sensor. Untuk melakukan ini, Anda memerlukan OID dari tabel SNMP. Untuk menampilkan level pengisian, OID "1.3.6.1.2.1.43.11.1" digunakan. Masukkan di langkah pertama pengaturan.



Menggunakan tabel SNMP, PRTG menampilkan level toner saat ini dalam daftar.

Kemudian ubah kolom untuk identifikasi menjadi OID "1.3.6.1.2.1.43.11.1.1.6". Ini mencerminkan warna kartrid. Sekarang atur nama saluran. Ini ditampilkan di sebelah hasil sensor, misalnya, "level pengisian". Maka Anda harus mencocokkan kolom dari saluran sensorik. Untuk ini, OID "1.3.6.1.2.1.43.11.1.1.9" diperkenalkan, yang mencerminkan tingkat pengisian persen saat ini. Untuk mengkonfigurasi unit saluran ini, pilih item "persentase" di daftar turun bawah.

Selanjutnya, PRTG membaca semua sensor pada printer Anda dan menunjukkan kepada Anda sebuah tabel dengan nilai yang tersedia. Periksa semua saluran yang ingin Anda tambahkan dan selesaikan pengaturan dengan mengklik Simpan. Sekarang PRTG menyesuaikan sensor untuk setiap catatan yang disorot dan menetapkan nama sensor berdasarkan informasi yang dibaca. Mulai saat ini, level toner kartrid ditampilkan, dan perubahan direkam terus menerus. Untuk petunjuk langkah demi langkah terperinci dengan ilustrasi, lihat Basis Pengetahuan PRTG .

Ikhtisar persediaan PRTG


Dengan PRTG Network Monitor, struktur bagan perangkat menjadi lebih kompleks selama bertahun-tahun. Oleh karena itu, masuk akal dari waktu ke waktu untuk memeriksa semua dependensi interval dan pewarisan aktif antara host. Selain itu, Anda perlu membuat daftar semua sensor dan saluran di mana pembatasan telah diperkenalkan. Mengetahui cara melakukan ini, Anda dapat dengan cepat membuat ulasan yang relevan.



Sistem pemantauan yang terlalu banyak dari waktu ke waktu membutuhkan inventaris.

Informasi yang diperlukan dapat dibaca tanpa masalah dari PRTG. Untuk melakukan ini, kami sarankan menggunakan alat PrtgAPI - antarmuka C # / PowerShell, yang diadaptasi secara individual ke Monitor Jaringan PRTG. Antarmuka program aplikasi (API) memungkinkan Anda membaca hampir semua informasi tentang sensor dari PRTG, dan kemudian memprosesnya dalam spreadsheet. Untuk mendapatkan semua karakteristik saluran, termasuk pengidentifikasi sensor, perintah berikut dijalankan di API:

Get-Sensor *cpu* | Get-Channel | 
 Export-Csv C:\channelLimits.csv -NoType

Jika Anda ingin memasukkan nama sensor dalam tabel, Anda dapat menambahkannya menggunakan objek Select. Baris perintah mungkin terlihat seperti ini:

$sensors = get-sensor *cpu*
$sensors|foreach {
$s = $_ $s|Get-Channel|select @{l="SensorName";e={$s.Name}},
 SensorId,Name,Id, *limit*
}|Export-Csv C:\channelLimits.csv -NoType

Jika Anda ingin membuat laporan dependensi sensor, objek Get-ObjectProperty akan membantu Anda. Ini memungkinkan Anda untuk menganalisis properti DependencyType dan DependentObjectId objek SensorSettings. Untuk instruksi terperinci, termasuk informasi tambahan tentang PrtgAPI, lihat Pangkalan Pengetahuan Paessler .

Memeriksa koneksi ke sensor MQTT


Banyak perusahaan manufaktur menengah secara bertahap memasukkan mesin mereka ke dalam jaringan. Mereka terhubung di area yang ditentukan secara khusus dan mengirim informasi ke broker MQTT pusat, tempat pemrosesan data selanjutnya dilakukan. Penting bagi perusahaan untuk memantau apakah broker MQTT dapat diakses sepenuhnya. Ada rekomendasi terpisah untuk skor ini.



Sensor Relay MQTT memeriksa apakah Anda dapat terhubung ke broker.

Monitor Jaringan PRTG menawarkan transponder MQTT khusus untuk tujuan ini.. Ia memeriksa apakah mungkin untuk terhubung ke broker, dan apakah broker menerima pesan melalui Publish. Selain itu, sensor memonitor apakah broker mengirimkan pesan ke peserta yang relevan, dan, di satu sisi, mengukur waktu koneksi individu, dan di sisi lain, total waktu yang diperlukan untuk sinyal bolak-balik sejak pesan dikeluarkan sampai diterima kembali.

Menyiapkan MQTT sangat sederhana dan hanya melibatkan beberapa langkah. Sensor BETA Round Trip MQTT ditambahkan ke PRTG. Dalam pengaturan sensor, nama ditetapkan untuk sensor dan bagian MQTT (Topik MQTT) disimpan. Nilai standarnya adalah PRTG / roundtrip /% sensorid, di mana PRTG menggantikan% sensorid dengan pengenal sensor yang sesuai. Untuk memungkinkan sensor mengakses bagian MQTT, simpan kredensial MQTT Anda dalam pengaturan perangkat induk. Setelah memulai sensor, ini menampilkan waktu pesan bolak-balik, keadaan bagian, waktu koneksi penerbit dan waktu koneksi pelanggan di berbagai saluran sensor.

Akses jarak jauh ke server PRTG utama


Monitor Jaringan PRTG digunakan oleh banyak perusahaan untuk memantau infrastruktur TI. Seringkali muncul pertanyaan tentang cara mengakses server PRTG utama di luar jaringan perusahaan. Apa yang harus dipertimbangkan oleh administrator dalam kasus ini, dan tindakan pencegahan apa yang penting untuk mencegah akses tidak sah oleh pihak ketiga, disajikan dalam rekomendasi berikut.

Anda telah mengkonfigurasi PRTG di jaringan internal Anda dengan alamat IP pribadi Anda, misalnya, 192.168.0.100. Semuanya berfungsi dengan baik, tetapi sekarang Anda juga ingin mengakses server PRTG utama melalui Internet dari komputer lain. Anda dapat menggunakan aplikasi seluler PRTG untuk ini. Namun, sebelum ini menjadi mungkin, Anda harus yakin bahwa Anda dapat mengakses server PRTG utama dari luar, termasuk melalui firewall.

Karena keamanan adalah alfa dan omega, Anda harus menjaga ketidakmungkinan orang lain memasuki akun PRTG Anda terlebih dahulu. Untuk melakukan ini, Anda harus mengubah kata sandi default untuk akun administrator jika Anda belum melakukannya. Untuk memeriksa apakah Anda menggunakan kata sandi default, cukup buka "Pengaturan / Pengaturan Akun / Akun Saya, bagian Akun Pengguna" di antarmuka web PRTG.

Selain itu, Anda tidak boleh mengakses instance PRTG dari luar melalui HTTP yang tidak aman. Oleh karena itu, pastikan bahwa program PRTG dikonfigurasi untuk menggunakan HTTPS untuk semua koneksi ke antarmuka web PRTG. Untuk melakukan ini, pilih "Secure HTTPS server sebagai port protokol kontrol transmisi (TCP) untuk permintaan server web masuk" di bagian "Pengaturan / Manajemen Sistem / Antarmuka Web, bagian Server Web". Untuk mengakses instalasi server utama dari luar, Anda harus membuka port yang diperlukan di firewall atau mengarahkan mereka. Untuk melakukan ini, buat aturan NAT untuk port ini. Pengaturan aturan NAT di firewall dapat sangat bervariasi tergantung pada pabrikannya. Karena itu, baca dokumentasi untuk perangkat atau model Anda.

Dalam contoh kami, jika program PRTG berjalan di server dengan alamat IP 192.168.0.100, aturan NAT mungkin terlihat seperti yang ditunjukkan pada ilustrasi. Aturan untuk HTTP adalah opsional, tetapi prosesnya sangat disederhanakan. Jika Anda memasukkan "prtgserver. <Mydomain.tld>" di browser Anda, upaya pertama-tama akan dilakukan untuk mengakses situs web melalui HTTP. Jika Anda tidak ingin membuat aturan NAT untuk HTTP, Anda harus memasukkan prtgserver . <Mydomain.tld> setiap kali .



Ini mungkin terlihat seperti aturan NAT pada firewall untuk menyediakan akses jarak jauh ke PRTG.

Anda juga harus memastikan bahwa Windows Firewall dinonaktifkan pada server PRTG, atau bahwa Anda telah membuat aturan yang sesuai. Sekarang Anda dapat memonitor instalasi PRTG dari jarak jauh melalui aplikasi seluler PRTG atau mengakses antarmuka web PRTG melalui Internet dari klien lain. Untuk informasi lebih lanjut tentang pengaturan akses ke Monitor Jaringan PRTG melalui firewall, termasuk pengaturan lanjutan yang bermanfaat, lihat Bagaimana-caranya .

Anda dapat mempelajari tentang kemampuan Paessler PRTG, mengunduh versi uji coba, mengajukan pertanyaan kepada spesialis Softline (mitra utama Paessler di Rusia dan CIS) di paessler.softline.ru .
