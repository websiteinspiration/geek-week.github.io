CentOS 8にAIDE（Advanced Intrusion Detection Environment）をインストールして使用する方法

Linux Administratorコースの開始を予想して、興味深い資料の翻訳を用意しました。



AIDEは、Linuxベースのオペレーティングシステムの変更を監視する最も一般的なシステムの1つである「Advanced Intrusion Detection Environment」の略です。AIDEは、マルウェア、ウイルスから保護し、不正なアクションを検出するために使用されます。ファイルの整合性と侵入検知をチェックするために、AIDEはファイル情報を含むデータベースを作成し、システムの現在の状態をこのデータベースと比較します。AIDEは、変更されたファイルに集中することにより、インシデント調査時間を短縮するのに役立ちます。

AIDEの機能：

  • ファイルタイプ、inode、uid、gid、権限、リンク数、mtime、ctime、atimeなど、さまざまなファイル属性のサポート。
  • Gzip、SELinux、XAttrs、Posix ACL、およびファイルシステム属性圧縮のサポート。
  • md5、sha1、sha256、sha512、rmd160、crc32などのさまざまなアルゴリズムのサポート
  • メール通知。

この記事では、CentOS 8への侵入を検出するためにAIDEをインストールして使用する方法について説明します。

前提条件


  • 少なくとも2 GBのRAMを備えたCentOS 8を実行しているサーバー。
  • ルートアクセス

入門


最初にシステムをアップグレードすることをお勧めします。これを行うには、次のコマンドを実行します。

dnf update -y

アップグレード後、システムを再起動して変更を有効にします。

AIDEをインストールする


AIDEはデフォルトのCentOS 8リポジトリで利用できます。次のコマンドを実行することで簡単にインストールできます。

dnf install aide -y

インストールが完了したら、次のコマンドを使用してAIDEバージョンを表示できます。

aide --version

次が表示されます。

CONFIG_FILE = "/etc/aide.conf"

利用可能なオプションaideは次のように表示できます。

aide --help

AIDEのインストール後に最初に行う必要があるのは、AIDEを初期化することです。初期化では、サーバーのすべてのファイルとディレクトリのデータベース（スナップショット）を作成します。

データベースを初期化するには、次のコマンドを実行します。

aide --init

チェックを実行


AIDEは新しいデータベースを使用する準備ができました。変更を加えずに最初のAIDEチェックを実行します。

aide --check

このコマンドの実行には、ファイルシステムのサイズとサーバーのRAMの容量に応じて時間がかかります。チェックが完了すると、以下が表示されます。

Start timestamp: 2020-01-16 03:05:07 -0500 (AIDE 0.16)
AIDE found NO differences between database and filesystem. Looks okay!!

上記の出力は、すべてのファイルとディレクトリがAIDEデータベースに対応していることを示しています。

AIDEのテスト


デフォルトでは、AIDEはデフォルトのApacheルートディレクトリを追跡しません/var/www/html.。それを表示するようにAIDEを設定しましょう。これを行うには、ファイルを変更する必要があります/etc/aide.conf。

nano /etc/aide.conf

行の上に"/root/CONTENT_EX"次の行を追加します。

/var/www/html/ CONTENT_EX

次に、次のコマンドを使用aide.txtして、ディレクトリにファイルを作成します/var/www/html/。

echo "Test AIDE" > /var/www/html/aide.txt

次に、AIDEチェックを実行し、作成されたファイルが検出されることを確認します。

aide --check

次が表示されます。

mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz

テストを再度実行して、AIDEが新しいデータベースを使用していることを確認します。

aide --check

検証を自動化


AIDEチェックを毎日実行し、レポートをメールで送信することをお勧めします。このプロセスはcronで自動化できます。

nano /etc/crontab

AIDEチェックを毎日10:15に開始するには、ファイルの最後に次の行を追加します。

15 10 * * * root /usr/sbin/aide --check

AIDEがメールで通知します。次のコマンドでメールを確認できます。

tail -f /var/mail/root

AIDEログは、次のコマンドを使用して表示できます。

tail -f /var/log/aide/aide.log

結論


この記事では、AIDEを使用してファイルの変更を検出し、サーバーへの不正アクセスを検出する方法を学びました。追加の設定については、構成ファイル/etc/aide.confを変更できます。セキュリティ上の理由から、データベースと構成ファイルを読み取り専用メディアに保存することをお勧めします。詳細については、AIDE Docドキュメントを参照してください。
