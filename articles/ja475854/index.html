JDBCプールと効率的なファイル処理：12月3日のサンクトペテルブルクでのJava mitap



12月3日にWrikeのサンクトペテルブルクオフィスで開催されるパフォーマンスについて 、Java mitapをご覧ください  。低レベルについて話しましょう。ただし、Java会議で十分なスペースがない場合があるレベルの開発者にとって重要です。

プログラムで


Alexander Kolesnikov、Wrike-JDBCプールの戦い 

AlexanderはWrikeでバックエンドテクライド/ SREとして働いています。社内のほとんどのバックエンドサービスはJavaで記述されており、主なストレージはリレーショナルデータベース（PostgreSQL）です。フォールトトレランスとデータベースのパフォーマンスは、Wrikeは99.99％のアップタイムサービスをユーザーに提供しています。

レポートでは、アレクサンダーは次の質問に答えます。

  1. JDBCプールの目的は何ですか？
  2. なぜそれほど多くの実装があり、それらの違いは何ですか。 
  3. アプリケーション用に選択するプール。4）どのプール設定に注意する価値がありますか。

Dmitry Govorukhin、GridGain-Java開発者向けの効果的なファイル管理

Dmitry-GridGainのチームリーダー/リード開発者。主な専門分野は、分散システム、そのパフォーマンスの問題、フォールトトレランス、およびスケーリングです。

Dmitryは、ディスク集約型アプリケーションの開発における個人的な経験と、Apache Ignite Native Storeの開発中に収集されたレーキについて話します。
 
  • ワークロード以外にパフォーマンスに影響するもの;
  • ディスクから最大限を引き出すために何をする必要があるか。
  • Java IO APIを使用する際の一般的な間違いと誤解。

12月3日から19:00まで、Wrikeのオフィスでお待ちしています。イベントは無料で、登録が必要です。公式プログラム終了後-無料のコミュニケーション、ネットワーキング、ピザ、ビール。
