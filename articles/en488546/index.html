First of all, we look at the information about the user under whom we work.



The user has the SeImpersonatePrivilege privilege enabled (the "Impersonate the client after authentication" right). As Microsoft says :
Assigning a user the right ‚ÄúImpersonate a client after authentication‚Äù allows programs launched on behalf of this user to impersonate a client. Using this parameter prevents unauthorized servers from impersonating clients connecting to these servers using RPC procedures or named pipes.
The fact is that we can raise our rights to SYSTEM. A full study can be found here . The following privileges are affected by this vulnerability:

  • SeImpersonatePrivilege
  • SeAssignPrimaryPrivilege
  • Setcbprivilege
  • SeBackupPrivilege
  • SeRestorePrivilege
  • SeCreateTokenPrivilege
  • SeLoadDriverPrivilege
  • SeTakeOwnershipPrivilege
  • SeDebugPrivilege
