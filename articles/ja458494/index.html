Vueレンダリング関数


レンダリング関数は、常に私にとってVueには少し変わったもののように見えました。このフレームワークのすべてが、シンプルさとさまざまなエンティティの義務の分離への欲求を強調しています。しかし、レンダリング機能はHTMLとJavaScriptの奇妙な組み合わせであり、多くの場合読みにくくなっています。

たとえば、HTMLマークアップは次のとおりです。

<div class="container">
  <p class="my-awesome-class">Some cool text</p>
</div>

それを形成するには、次の関数が必要です。

render(createElement) {
  return createElement("div", { class: "container" }, [
    createElement("p", { class: "my-awesome-class" }, "Some cool text")
  ])
}

そのような構造は、多くの人がすぐにレンダリング機能から離れることになるのではないかと思います。結局のところ、使いやすさはまさに開発者をVueに引き付けるものです。レンダリング機能の見苦しい外観の背後にある真の美徳を多くの人々が見ないのは残念です。問題は、レンダリング機能と機能コンポーネントが興味深い強力なツールであることです。私は、彼らの能力と真の価値を実証するために、彼らが私が実際の問題を解決するのをどのように助けたかについて話します。検討中のプロジェクトのデモバージョンを隣接するブラウザタブで
