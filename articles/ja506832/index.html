コンパイラーはいくつのプロセッサー命令を使用しますか？

1か月前、私は最近のプロセッサーでサポートされているさまざまな命令の数を数えようとしましたが、アイスレイクでは945と数えました。コメンテーターは興味深い質問を投げかけました：この多様性のどの部分がコンパイラーによって実際に使用されているのですか？たとえば、2016年のあるPepijn de Vosは、彼の/ usr / binのバイナリに含まれるさまざまな命令の数を計算し、411をカウントしました。その時点で存在していたすべてのx86_64命令の約3分の1は、そのOSのどの標準プログラムでも使用されていませんでした。彼の他の奇妙な発見は、x86_64のコードが3番目の命令で構成されていることですmov。 （一般に、プログラムを作成するには、命令だけで十分であることが知られてmovいます。）

「参照コード」としてLLVM / Clangコンパイラーを使用してde Vos研究を開発することにしました。名前のないOSの名前のないバージョンの/ usr / binの内容に比べて、いくつかの利点があります。

  1. これを使用すると便利です。これは、サイズが平均的なLinuxの/ usr / binのすべてのコンテンツに匹敵する1つの巨大なバイナリです。
  2. 異なるISAを比較できます。x86 、ARM、SPARC、MIPS、PowerPCの公式リリースは、releases.llvm.org / download.htmlで入手できます。
  3. 履歴の傾向を追跡できます。公式のバイナリは2003年以降のすべてのリリースで利用できます。
  4. 最後に、コンパイラーの研究では、コンパイラーを実験オブジェクトとして使用するのが論理的です:-)
