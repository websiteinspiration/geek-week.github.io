Zimbra Collaboration Suite Open-Source Editionのメールアーカイブ

将来的に表示できるメールをアーカイブすることは、大企業にとって重要な機能です。さまざまな苦情の処理、調査の実施、およびその他の多くの状況で使用できます。この機能は、悪意のあるユーザーがサービスを使用して違法行為を行った場合にSaaSプロバイダーが自身を保護するのにも役立ちます。

特にこれらの目的のために、Zimbra Archiving and Discoveryプラグインが作成されました。これにより、各メールボックスの送信および受信レター、さらにはドラフトメッセージに保存されたレターをアーカイブできます。ただし、この決定には欠点がないわけではありません。 1つは有料のZimbra Collaboration Suite Network Editionでのみ機能し、2つ目はWebクライアントのフレームワーク内でのみ機能し、デスクトップまたはモバイルのメールクライアントを使用する場合は何もアーカイブしません。これに関して、無料のZImbra Collaboration Suite Open-Source Editionで受信メールと送信メールのアーカイブを実装する方法について説明します。さらに、電子メールクライアントから送信されたレターをアーカイブします。


メールのアーカイブは、組み込みのPostfix BCC機能によって実装されます。これは次のように機能します。システム管理者がメールボックスのアーカイブメールアドレスを設定し、特定の設定を入力すると、受信メールと送信メールのそれぞれがアーカイブメールにコピーされ、後で必要なメールを見つけることができます。メールアーカイブ用に別のドメインを作成することをお勧めします。これにより、将来のアーカイブメールボックスの管理が大幅に簡素化されます。

送信メールのアーカイブ




送信メールのアーカイブを設定しましょう。たとえば、manager @ company.ruアカウントを取得して、アーカイブメールボックスmanager@archive.company.ruを作成します。送信メールをアーカイブするには、Postfix設定にいくつかの変更を加える必要があります。これを行うには、ファイル/opt/zimbra/postfix/conf/main.cfを開いて、その行にsender_bcc_maps = lmdb：/ opt / zimbra / postfix / conf / sender_bccという行を追加します。その後、ファイル/ opt / zimbra / postfix / conf / sender_bccを作成し、アーカイブする予定のメールボックスと、アーカイブされたレターの送信先のメールボックスを追加する必要があります。複数のメールボックスを1つにアーカイブすることが可能です。これは次のように行われます。
manager@company.ru manager@archive.company.ru
sales@company.ru manager@archive.company.ru
support@company.ru support@archive.company.ru



すべてのメールボックスが追加されたら、後はpostmap / opt / zimbra / postfix / conf / sender_bccコマンドを実行し、postfix reloadコマンドを使用してPostfixをリロードするだけです。この例からわかるように、再起動後、アカウントmanager@company.ruおよびsales@company.ruからのすべての送信レターは同じメールボックスmanager@archive.company.ruに送信され、アカウントsupport@company.ruからの送信レターはアーカイブされますメールボックスsupport@archive.company.ru

受信メールのアーカイブ


次に、受信メールの自動アーカイブを設定します。これには同じPostfix BCCを使用できます。送信メールをアーカイブする場合と同様に、ファイル/opt/zimbra/postfix/conf/main.cfを開き、recipient_bcc_maps = lmdb：/ opt / zimbra / postfix / conf / recipient_bccという行を追加する必要があります。その後、ファイル/ opt / zimbra / postfix / conf / recipient_bccを作成し、必要なメールアドレスを前の形式で追加する必要があります。



ボックスを追加した後、postmap / opt / zimbra / postfix / conf / recipient_bccコマンドを実行し、postfix reloadコマンドを使用してPostfixを再起動する必要があります。これで、アカウントmanager@company.ruおよびsales@company.ruからのすべての受信レターはメールボックスmanager@archive.company.ruにアーカイブされ、アカウントsupport@company.ruからの受信レターはメールボックスsupport@archive.companyにコピーされます。 .ru。


受信メッセージのフィルタリングの例

リスト/ opt / zimbra / postfix / conf / sender_bccおよび/ opt / zimbra / postfix / conf / recipient_bccでメールアドレスを追加または削除するたびに、postmapコマンドを再度実行する必要があることに注意してください変更されたリストを表示し、Postfixを再起動します。また、送信者と受信者の名前に基づいてZimbra OSEメールフィルターを使用して、受信メッセージと送信メッセージがフォルダーに分類され、後で必要なメッセージを見つけやすくすることをお勧めします。


送信メッセージのフィルタリングの例

作成されたメールアーカイブで後で手紙を検索するには、組み込みの検索Zimbra OSEを使用できます。また、アーカイブでのメールの保存時間は、アカウントでの保存時間よりもはるかに長いことにも注意してください。これは、より多くの割り当てと、より長い期間の保持ポリシーを設定する必要があることを意味します。別のドメインにアーカイブメールボックスがある場合は、これを行う方がはるかに簡単です。
