新しいコンパイラ


プロジェクトリポジトリの/ mashcブランチで、Mashで記述された新しいバージョンのコンパイラを確認できます（言語の最初のバージョン）。

コンパイラのasmリストにコードジェネレータがあります（スタックVMのアセンブラ用）。
現時点では、Java（JDK 1.8）用のジェネレーターのバージョンを開発しています。

新しいバージョンのコンパイラは、言語の最初のバージョンの機能を完全にサポートし、それを補完します。

新しいOOP


新しいバージョンの言語では、クラスでの作業が部分的にやり直されました。
クラスのメソッドは、クラスの本体とそれ以降の両方で宣言できます。
これでクラスに明示的なコンストラクタinitが追加されました。

コード例：

...
class MyClass:
  private:
    var a, b

  public:
    init(a, b):
      $a ?= a
      $b ?= b
    end

    func Foo():
      return $a + $b   
    end
end

func MyClass::Bar(c):
  return $a + $b + c
end
...

継承が発生した場合、継承された呼び出し（スーパー）を簡単に行う機会があります。

コード例：

...
class MySecondClass(MyClass):
  public:
    var c

    init(a, b, c):
      super(a, b)
      $c ?= c
    end

    func Bar():
      super($c)  
    end
end
...

x ?= new MySecondClass(10, 20, 30)
println( x -> Bar() )     // 60

クラスインスタンスをオーバーライドする動的メソッド：

...
func Polymorph::NewFoo(c):
  return $a + $b + c  
end
...
x -> Foo ?= Polymorph -> NewFoo
x -> Foo(30)    // 60

パッケージ/名前空間


名前空間はクリーンに保つ必要があります！
したがって、言語はこの機会を提供する必要があります。
Mashでは、クラスメソッドが静的であれば、コードのどこからでも安全に呼び出すことができます。

例：

...
class MyPackage:
  func MyFunc(a, b):
    return a + b  
  end
end
...
println( MyPackage -> MyFunc(10, 20) )    // 30

ちなみに、スーパーオペレーターはそのような呼び出しで正しく動作します。

例外


新しいバージョンの言語では、これらはクラスとして扱われます。

try:
  raise new Exception(
    "My raised exception!"
  )
catch E:
  if E is Exception:
    println(E)
  else:
    println("Unknown exception class!")
  end
end

埋め込み


潜在的に-MashはLuaと同様に、そのニッチを埋め込みプログラミング言語として使用できます。

これらの目的でMashを使い始めるために、プロジェクトを自分でビルドする必要さえありません。

Mashにはランタイム環境があります-完全なAPIを備えた動的ライブラリとして構築されたスタックされたVM。

プロジェクトに応じて追加し、数回呼び出すだけです。

言語自体は、組み込み言語として機能する機能を提供します。
同時に、言語およびサードパーティのライブラリと連携した操作性は侵害されません。
そこに書かれたさまざまなフレームワークの能力をフルに活用できる組み込み言語を手に入れました。
