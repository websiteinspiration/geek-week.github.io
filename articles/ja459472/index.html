Heroku + Docker + Spring Boot

以下は、無料のHerokuクラウドサービス仮想マシンでドッキングされたSpring Bootアプリケーションを実行した私の経験に焦点を当てます。このプロバイダーの主な利点の1つは、稼働時間が制限された無料の仮想マシンを作成できることです。これには、登録のみで十分です。支払いの詳細さえ確認する必要はありませんが、確認すると追加のボーナスを受け取ることができます。価格表の詳細については、こちらをご覧ください。私の見解では、無料のリソースに関する彼らの政策にはほとんど類似点がありません。

したがって、Herokuでアプリケーションを作成した後、アプリケーションをコードでデプロイするにはいくつかの方法があります。

  • heroku gitリポジトリでコミット
  • アプリケーションをgithubリポジトリにバインドする
  • Dockerコンテナーの使用

次に、これらのメソッドの最後の方法を検討します。続行するには、次のアプリケーションが必要です

  • Docker
  • Maven、Spring Bootアプリケーションの構築用
  • Heroku CLI

これらの最後のツールは、コマンドラインからクラウドを操作するためのすべての操作を実行する機会を提供します。

まず、Spring Initializrを使用してSpring Bootアプリケーションを作成します。依存関係として、Spring Web Starterを追加します。それで十分でしょう。

アプリケーションのメインクラスが配置されているのと同じパッケージに、RESTコントローラの最も単純なクラスを追加して、アプリケーションが何らかの形で生命の兆候を示すようにします。

@RestController
public class DemoController {
      
      @GetMapping("/check")
      public String check() {
           return "Application is alive";
      }
}

ファイルapplication.properties add

server.port=${PORT:8080}

この設定は、Herokuでコンテナーを実行するために重要です。実際のところ、サービスの内部ネットワークでは、アプリケーションは起動時にいくつかの空きポートで起動され、その数はPORT環境変数を通じて送信されます。さらに、アプリケーションは、起動後最初の60秒以内にこのポートに接続する時間が必要です。そうしないと停止します。pom.xmlファイル

のプラグインセクションで、Spotifyからdockerfile-pluginを追加します。これは、アプリケーションのdockerイメージのアセンブリに役立ちます。

<plugin>
  <groupId>com.spotify</groupId>
  <artifactId>dockerfile-maven-plugin</artifactId>
  <version>1.4.6</version>
  <executions>
    <execution>
       <id>default</id>
       <goals>
         <goal>build</goal>
         <goal>push</goal>
       </goals>
    </execution>
  </executions>
  <configuration>
     <repository>registry.heroku.com/${project.artifactId}/web</repository>
     <tag>latest</tag>
     <buildArgs>
       <JAR_FILE>${project.build.finalName}.jar</JAR_FILE>
     </buildArgs>
  </configuration>
</plugin>

このようなプラグイン構成は、インストールターゲットの一部としてイメージの構築を開始し、デプロイターゲットの一部としてHeroku Dockerリポジトリーにイメージをプッシュします。ちなみに、mavenデプロイを実行できるようにするには、収集したjarファイルのデプロイを無効にする必要があります（どこにもアップロードする必要はありません）。これを行うには、pom.xmlファイルのプロパティセクションでmaven.deploy.skipオプションを使用します。

<properties>
  <java.version>1.8</java.version>
  <maven.deploy.skip>true</maven.deploy.skip>
</properties>

次に、プロジェクトルートフォルダー（pom.xmlの横）にDockerfileを作成します。

FROM openjdk:8-jdk-alpine

ARG JAR_FILE

RUN mkdir -p /apps
COPY ./target/${JAR_FILE} /apps/app.jar
COPY ./entrypoint.sh /apps/entrypoint.sh

RUN chmod +x /apps/entrypoint.sh
CMD ["/apps/entrypoint.sh"]

ご覧のとおり、アセンブルされたjarファイルの名前をここで引数としてアセンブリ（ARG JAR_FILE）に渡します。この引数の値は、Mavenプラグイン設定で設定されます。entrypoint.sh

スクリプトもプロジェクトのルートに配置され、非常に簡単になります。

#!/usr/bin/env sh

/usr/bin/java -XX:+UnlockExperimentalVMOptions -XX:+UseCGroupMemoryLimitForHeap -Xmx256m -Xss512k -XX:MetaspaceSize=100m -jar /apps/app.jar

メモリを制限するJVMパラメータ、特に最初の2つのパラメータに注意してください。特別なメモリ管理モードが含まれています。これは、JavaアプリケーションがDockerコンテナー内で起動されるときに必要です。無料のHerokuマシンで使用可能なメモリは512MBに制限されています。この制限を超えると、アプリケーションが終了します。 Java 9以降を使用する場合は、最初の2つのオプションを1つの-XX：+ UseContainerSupportに置き換える必要があります。詳しくはこちらをご覧ください。また、コンテナのサポートは、Java 8の実験的な機能ではなくなったようです。詳細はこちら。

これらの手順を完了したら、開始してみてください
mvnw clean install

すべてが正しく行われていれば、アプリケーションとそのためのDockerイメージがビルドされます。次のコマンドを使用して、イメージが作成されたかどうかを確認できます
docker images

次に、クラウドをセットアップし、これに前述のHeroku CLIを使用します。もちろん、これらの手順を完了するには、Herokuにアカウントを登録する必要があります。

まず、ログインする必要があります。これを行うには、コマンドを実行します
heroku login
以下の指示に従ってください。

その後、heroku Dockerリポジトリーにログインする必要があります。これを行うには、コマンドを実行します
heroku container:login
これがないと、Dockerイメージをプッシュできません。

次に、コマンドを使用してアプリケーションを作成します
heroku apps:create <app-name>

アプリケーション名は、pom.xmlで指定されているアーティファクトの名前と一致する必要があることに注意してください。おそらくここでは、アプリケーションの名前を選択するのにある程度の時間を費やす必要がありますが、まだ誰も使用していません。

アプリケーションが作成された後、実行します
mvnw clean deploy

そして、アプリケーションがビルドされるのを待ち、Dockerイメージをプッシュします。イメージ名がregistry.heroku.com/<app-name>/webテンプレートと一致し、<app-name>という名前のアプリケーションが作成された場合にのみ、プッシュが可能であることに注意してください。mavenプラグイン設定を見ると、すべてがまったく同じように行われていることがわかります。

イメージをデプロイして起動する最後のステップは、コマンドです
heroku container:release web --app=<app-name>

その後、リンクhttps：// <app-name> .herokuapp.com / checkをたどると、しばらくすると、コントローラーハンドラーによって表示されるテキストが表示されます。

ブラウザで実行中のアプリケーションを開く別の方法は、コマンドを使用することです
heroku open --app=<app-name>

何かが機能しない場合、ログはHeroku Webインターフェースまたはコマンドで表示できます
heroku logs

それで全部です！このガイドがお役に立てば幸いです。
