Magento 2: Kritisches CSS



In diesem Artikel erfahren Sie, wie Sie kritisches CSS in Magento 2 konfigurieren und aktivieren.

Was bist du, kritisches CSS?


Sowohl HTML als auch CSS verlangsamen die Seitenausgabe. Ein HTML-Dokument muss verarbeitet werden, um ein DOM-Modell zu erstellen, ohne das die Seite keinen Inhalt enthält. Das CSS-Dokument muss zuerst geladen und dann verarbeitet werden (CSS-Objektmodell).

Um den Prozess zu beschleunigen, müssen Sie CSS vereinfachen und Medienabfragen verwenden. Kritisches CSS trägt auch zur Leistungsverbesserung bei.

Wie oben erwähnt, werden alle aus externen Dateien geladenen CSS-Stile als Renderblockierung behandelt. Dies bedeutet, dass die Webseite erst angezeigt wird, wenn diese Dateien heruntergeladen wurden. Mit kritischem CSS stellen wir das integrierte „kritische CSS“ bereit und machen das Laden aller anderen Stile asynchron.

Das asynchrone Laden wird durch Hinzufügen einer neuen Vorladeanweisung erreichtzum rel-Attribut

Um das browserübergreifende rel = "Preload" zu unterstützen, verwendet Magento eine Polyfüllung.

Ein wichtiges Merkmal ist, dass das „kritische CSS“ nur den sichtbaren Teil der Seite enthält.



Somit können wir die Zeit der "ersten" Anzeige unserer Seiten erheblich verbessern.

Wie schalte ich es ein?


Speicher - Einstellungen - Konfiguration - Erweitert - Entwickler

Die Registerkarte Erweitert ist nur im Entwicklermodus verfügbar.

Setzen Sie den kritischen CSS-Pfad auf Ja.

Oder verwenden Sie die Befehlszeile: bin / magento config: set dev / css / use_css_critical_path 1

Kritisches CSS muss sich in App / Design befinden / frontend / vendor_name / theme_name / theme_name /web/css/critical.css



Wie generieren?


Der vollständig manuelle Erstellungsprozess ist sehr zeitaufwändig und zeitaufwändig.

Generatoren wie Penthouse oder Critical können verwendet werden, um kritisches CSS für Ihr Thema zu erstellen.

Oder wenden Sie sich an eine ziemlich gute Online-Ressource.



Es ist notwendig, die URL-Adresse in die Ressource in Feld (1) einzufügen und CSS einzufügen, das auf der Site verwendet wird, und auf die Schaltfläche Erstellen in Feld (2) zu klicken.



Als Ergebnis erhalten wir das generierte und minimierte kritische CSS.



Ein bestimmter Teil der manuellen Arbeit muss natürlich:

  • Überprüfen und Anpassen des generierten kritischen CSS
  • Vergessen Sie auch nicht, dass Critical.css in Magento eine für alle Seiten ist. Vergessen Sie daher nicht, die Auflistung, die Produktseite, die Registrierungsseite und das Login zu vergessen
