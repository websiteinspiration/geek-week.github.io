デジタルオーディオとアナログの違いは何ですか？


アナログ（または連続体）信号は、時間の連続関数、つまり それは持っています可能な値の連続セットを持つ連続線（図1）

図。1


デジタル信号は、定義されたデジタル値のシーケンスとして表すことができる信号です。いつでも、1つの明確な最終値しか取ることができません（図2）。

図。2


ダイナミックレンジのアナログ信号は任意の値をとることができます。アナログ信号は、離散化と量子化という 2つのプロセスを使用してデジタルに変換されます。プロセスキューは重要ではありません。

離散化は、特定の時間間隔（通常は等しい）で信号の値を記録（測定）するプロセスです（図3）。

図。３


量子化は、信号の振幅範囲を特定の数のレベルに分割し、サンプリング中に測定された値を最も近いレベルに丸めるプロセスです（図4）。

図。4


離散化は、時間成分の信号を分割します（図5左）。
量子化は、信号を所定の値にします。つまり、信号をそれに最も近いレベルに丸めます（水平方向に、図5の右側）。

図。5


これらの2つのプロセスは、いつでも特定の値でオーディオ信号を記述することができる一種の座標系を作成します。
デジタルは、離散化と量子化が適用される信号です。デジタル化は、A / Dコンバーター（ADC）で行われます。量子化レベルの数が多く、サンプリング周波数が高いほど、デジタル信号はより正確にアナログ信号に対応します（図6）。

図。6


量子化レベルには番号が付けられ、バイナリコードが各レベルに割り当てられます。（図7）

図。7


各量子化レベルに割り当てられるビット数は、ビット深度または量子化深度（eng。ビット深度）と呼ばれます。ビット深度が高いほど、より多くのレベルをバイナリコードで表すことができます（図8）。

通常、8、12、16、および24ビットのビットが使用されます。n = 24の場合、レベルの数はN = 16,777,216であると簡単に計算できます。

n = 1の場合、オーディオ信号はモールス符号になります。「ノック」があるかどうかです。32ビット浮動小数点もあります。従来のコンパクトオーディオCDの容量は16ビットです。ビット深度が低いほど、丸められる値が多くなり、量子化誤差が大きくなります。

量子化誤差は、量子化された信号のアナログからの偏差です。入力値の差X と量子化された値 X′ （X−X′）

量子化エラーが大きいと、オーディオ信号に深刻な歪み（量子化ノイズ）が生じます。

ビット深度が高いほど、量子化エラーが小さくなり、信号対雑音比（SNR）が向上します。逆も同様です。ビット深度が低いと、ノイズが増加します（図9）。

図。9


ビット深度は、信号のダイナミックレンジ、つまり最大値と最小値の比率も決定します。各ビットで、ダイナミックレンジは約6dB（デシベル）増加します（6dBは2倍です。つまり、グリッドが密になり、グラデーションが増加します）。

デジタルオーディオの次の特徴はビットレートです。ビットレートは、単位時間あたりに送信されるデータの量です。ビットレートは通常、ビット/秒（ビット/秒またはbps）で測定されます。ビットレートは、可変、一定、または平均化できます。

下記式（非圧縮データストリームに対してのみ有効）を使用すると、ビットレートを計算することを可能にする：

チャネルのビットレート=サンプルレート*ビット*数

、例えば、次のようにオーディオCDのビットレートを計算することができる
チャネルの44100（サンプルレート）×16（ビット）* 2（数、ステレオ）= 1411200 bps = 1411.2 kbit / s

固定ビットレート（CBR）では、単位時間あたりのデータストリームのボリュームの送信は、送信全体で変化しません。主な利点は、最終的なファイルのサイズをかなり正確に予測できることです。マイナス部分-サイズ/品質の最適な比率ではありません。これは、音楽の一部のオーディオ素材の「密度」が動的に変化するためです。

可変ビットレート（VBR）でエンコードする場合、コーデックは必要な品質に基づいてビットレートを選択します。名前が示すように、ビットレートはエンコードされたオーディオファイルによって異なります。この方法により、出力ファイルの品質/サイズ比が最高になります。マイナス点としては、最終的なファイルの正確なサイズの予測が非常に不十分です。

平均ビットレート（ABR）はVBRの特殊なケースであり、一定ビットレートと可変ビットレートの中間的な位置を占めます。特定のビットレートはユーザーが設定します。プログラムはそれを一定の範囲で変化させますが、特定の平均値を超えません。 

特定のビットレートの場合、VBRの品質は通常ABRよりも高くなります。次に、ABRの品質はCBRよりも高くなります：VBR> ABR> CBR。

ABRは、VBRエンコーディングの利点を必要とするユーザーに適していますが、ファイルサイズは比較的予測可能です。ABRの場合、コーデックは最初のパスではオーディオ素材のどの部分を最大ビットレートでエンコードする必要があるかわからないため、通常は2パスでのエンコードが必要です。

デジタルオーディオ素材を保存するには、3つの方法があります。

  • 非圧縮（生）データ
  • ロスレスデータ
  • 不可逆圧縮データ

非圧縮（RAW）データ形式


バイナリ値のシーケンスのみが含まれています。
オーディオ素材がオーディオCDに保存されるのは、このフォーマットです。圧縮されていないオーディオファイルは、たとえばAudacityで開くことができます。拡張子は.raw、.pcm、.samであるか、拡張子がまったくありません。RAWにはファイルヘッダー（メタデータ）が含まれていません。

非圧縮オーディオストリームを保存する別の形式はWAVです。RAWとは異なり、WAVにはファイルヘッダーが含まれています。

ロスレスオーディオ形式


圧縮の原理はアーカイバ（Winrar、Winzipなど）に似ています。データは、情報を失うことなく何度でも圧縮および解凍できます。

ロスレス圧縮で、情報が本当に手付かずのままであることをどのように証明しますか？これは、破壊的な干渉の方法によって証明できます。2つのオーディオトラックを使用します。最初のトラックでは、元の圧縮されていないwavファイルをインポートします。2番目のトラックでは、同じオーディオファイルを可逆圧縮してインポートします。いずれかのトラックの位相を反転します（ミラーイメージ）。両方のトラックを同時に再生すると、出力信号は無音になります。

これにより、両方のファイルにまったく同じ情報が含まれていることが証明されます（図11）。

非可逆圧縮の場合

重要なのは、情報の損失を回避することではなく、主観的な認識を伴う推測にあります（心理音響学）。たとえば、大人の耳は通常、16kHz以上の周波数を知覚しません。この事実を使用すると、「とにかく誰も違いを聞くことができない」ため、非可逆圧縮コーデックは16kHzを超えるすべての周波数を単純にハードカットできます。

別の例はマスキング効果です。強い振幅と重なる弱い振幅は、より低い品質で再現できます。大きな低周波数では、静かな中間周波数は耳に取り込まれません。たとえば、1dBの音量レベルが80dBのサウンドがある場合、40dBの音量の2kHzサウンドは聞こえなくなります。

これはコーデックを使用します：2kHzサウンドは削除できます。
