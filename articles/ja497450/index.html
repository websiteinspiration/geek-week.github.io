Symfony 4/5のカスタム検証の基本と例

Symfonyでは、私の意見では、エンティティの検証に非常に便利な機能です。特に、検証ルールを構成するためのアノテーション形式の使用には非常に感銘を受けました。ほとんどのタスクで、ターンキーソリューションは標準的なケースをカバーします。しかし、ご存じのとおり、検証はデリケートな問題であり、今回はどのような制限を課さなければならないかは決してわかりません。一方、より柔軟で十分に検討された検証は、常にユーザーエラーの回避に役立ちます。


カットの下では、基本的な検証と検証の例を使用して、制限を記述し、利用可能な2つのフィールドの比較を拡張するのがいかに簡単かをお見せします。この記事は、symfonyでの検証にまだ慣れていない人や、独自のバリデーターを作成する可能性を回避した人にとって興味深いかもしれません。


この記事では、検証の古典的なコンテキストに留意します。特定のエンティティがあり、適切なフォームからデータが入力されますが、検証により、入力されたデータのあらゆる側面に制限が課されます。バリデーターは、内部ロジック、API検証などの制御に制限なしで使用できることに注意してください。


私たちは提案された例が唯一の可能な解決策ではなく、最大の最適性や広範な使用を主張するものではなく、デモンストレーションの目的で提供されていることをすぐに判断します。一方、例の一部は戦闘プロジェクトから取られて簡略化されており、実際に中断することなくカスタム検証を作成する単純さを示すのに役立つことを願っています。


基本的な検証


完全に機能する検証を行うには、2つのクラス、つまりConstraintの継承とConstraintValidatorのみを作成する必要があります。制約は、その名前が示すように、制約を定義および記述し、ConstraintValidatorはそれらを検証します。例として、テキスト形式で保存された「hh：mm」の形式で時間のバリデーターを記述します。で公式ドキュメント、された制約は、制約のパブリックプロパティを記述することを提案しました。じゃあやってみましょう。


namespace App\Custom\Constraints;

use Symfony\Component\Validator\Constraint;

/**
 * @Annotation
 * @Target({"PROPERTY"})
 */
class TextTime extends Constraint
{
    public $message = '  ';
}

ここで、Targetアノテーションは、プロパティまたはクラスのどちらで検証が使用されるかを決定します。関数をオーバーライドして、このパラメーターを設定することもできます。


public function getTargets()
{
    //   self::CLASS_CONSTRAINT
    return self::PROPERTY_CONSTRAINT;
}

namespace App\Custom\Constraints;

use Symfony\Component\Validator\ConstraintValidator;

class TextTimeValidator extends ConstraintValidator
{
    /**
     *    
     *
     * @param mixed $value  
     * @param Constraint $constraint   
     */
    public function validate($value, Constraint $constraint)
    {
        $time = explode(':', $value);

        $hours = (int) $time[0];
        $minutes = (int) $time[1];

        if ($hours >= 24 || $hours < 0)
            $this->fail($constraint);
        else if ((int) $time[1] > 60 || $minutes < 0)
            $this->fail($constraint);
    }

    private function fail(Constraint $constraint) 
    {
        $this->context->buildViolation($constraint->message)
            ->addViolation();
    }
}

namespace App\Custom\Constraints;

use Symfony\Component\Validator\Constraints\AbstractComparison;

/**
 * @Annotation
 * @Target({"PROPERTY"})
 */
class TextTimeInterval extends AbstractComparison
{   
    public $message = '       {{ compared_value }}.';
}


namespace App\Custom\Constraints;

use Symfony\Component\Validator\Constraints\AbstractComparisonValidator;

class TextTimeIntervalValidator extends AbstractComparisonValidator
{
    /**
     *       
     *
     * @param mixed $timeFrom   
     * @param mixed $timeTo   
     *
     * @return  true   , false 
     */
    protected function compareValues($timeFrom, $timeTo)
    {
        $compareResult = true;

        $from = explode(':', $timeFrom);
        $to = explode(':', $timeTo);

        try {
            if ((int) $from[0] > (int) $to[0])
                $compareResult = false;
            else if (((int) $from[0] == (int) $to[0]) && ((int) $from[1] > (int) $to[1])) {
                $compareResult = false;
            }
        } catch (\Exception $exception) {
            $compareResult = false;
        }

        return $compareResult;
    }
}

この場合のみ（可能な実装の1つ）、compareValues関数をオーバーライドします。この関数は、検証の成功時にtrue / falseを返し、validate（）関数によってAbstractComparisonValidatorで処理されます。
