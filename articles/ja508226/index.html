Androidおよび3Dカメラ。不正防止による顔認識

こんにちは！私の名前はウラジミールシャルコフです。私はSurfのAndroid開発者です。



少し前まで、不正防止機能を備えたAndroidに顔認識システムを実装する必要がありました。この記事では、実装の最も興味深い側面をコード例とリンクで共有します。きっとあなたにとって新しい面白いものを見つけられるでしょう。







顔認識システムは、ますます人気が高まっています。顔認証機能を備えたデバイスの数や、開発者向けのツールの数が増えています。



Appleは自社製品でFaceIDを使用し、さらに開発者の面倒を見て、この機能にアクセスするためのAPIを追加しました。 FaceIDは非常に安全であると考えられており、銀行アプリケーションのロックを解除するために使用できます。最近まで、Android SDKには既製のソリューションがありませんでした。デバイスメーカーは、ファームウェアの顔を使ってデバイスのロックを解除する機能を追加しましたが、開発者はアプリケーションの機能を使用できず、ロック解除のこの方法のセキュリティは、まだ十分とは言えませんでした。



最近、クラス指紋を使用してアプリケーションのロックを解除するために使用されたFingerprintManagerには、API 28以降が割り当てられており、開発者はBiometricPromptを使用することをお勧めします。このクラスには、個人の識別など、バイオメトリクスに関連するロジックが含まれています。ただし、Googleからの情報によると、デバイスには高いセキュリティ評価が必要であるため、すべてのスマートフォンで使用することはできません。



一部のデバイスには組み込みの指紋スキャナーがありません。顔認識での不正に対する高度な保護と、すべて正面のToF（飛行時間）センサーのおかげで、スキャナーを拒否しました。これを使用すると、深度マップを作成して、システムの亀裂への耐性を高めることができます。



必要条件



私たちが実装したアプリケーションは、その機能により、アクセス制御システムであり、人は人を識別する方法です。特別なアルゴリズムを使用して、実際の人物に対する人物の所属をチェックします。写真を撮って名前を指定することにより、デバイスから直接データベースに新しいユーザーを追加できます。データベース内の人物の存在を判断する必要がある場合は、デバイスからリアルタイムで撮影された写真を使用して検索が実行されます。アルゴリズムはデータベースから人物との類似性を決定し、これが見つかった場合、この人物に関する情報が発行されます。



私たちの主な目標は、最高レベルのセキュリティを確保することでした。たとえば、ビューファインダーに持ち込まれた写真を使用するなど、顔認識システムをバイパスする可能性を最小限に抑える必要がありました。これを行うために、ToFセンサーが組み込まれているIntel RealSense 3Dカメラ（モデルD435i）を使用することにしました。これにより、深度マップを作成するために必要なすべてのデータを取得できます。







機能するデバイスとして、大画面の対角線を備えたタブレットを使用する必要がありました。これには、バッテリーが内蔵されておらず、主電源への常時接続が必要でした。



もう1つの重要な制限はオフライン作業です。このため、顔認識にクラウドサービスを利用できませんでした。さらに、時間制限と人件費を考慮して、顔認識アルゴリズムを最初から作成するのは不合理です。問題が発生します：既製のソリューションがある場合、なぜ車輪を再発明するのですか？上記に基づいて、3DiVi Face SDKライブラリを使用することを決定しました。



Intel RealSenseカメラからの画像の取得



実装の最初の段階では、3Dカメラから2つの画像を取得する必要がありました。1つはカラー、もう1つは深度マップを使用したカラーです。その後、それらはFace SDKライブラリーでさらに計算するために使用されます。



AndroidプロジェクトでIntel RealSenseカメラの使用を開始するには、RealSense SDK for Android OSの依存関係を追加する必要があります。これは、公式のC ++ライブラリのラッパーです。公式サンプルでは、​​カメラからの画像を初期化して表示する方法を見つけることができます。ここで止めることはしません。すべてが非常に簡単です。画像取得コードに直行しましょう： 



private val pipeline = Pipeline()
private val streamingHandler = Handler()

private var streamRunnable: Runnable = object : Runnable {
    override fun run() {
        try {
            FrameReleaser().use { fr ->
                val frames = pipeline.waitForFrames(1000).releaseWith(fr)

                val orgFrameSet = frames.releaseWith(fr)
                val processedFrameSet = frames.applyFilter(align).releaseWith(fr)

                val orgFrame: Frame = orgFrameSet.first(StreamType.COLOR, StreamFormat.RGB8).releaseWith(fr)
                //    
                val videoFrame: VideoFrame = orgFrame.`as`(Extension.VIDEO_FRAME)

                val processedDepth: Frame = processedFrameSet.first(StreamType.DEPTH, StreamFormat.Z16).releaseWith(fr)
                //    
                val depthFrame: DepthFrame = processedDepth.`as`(Extension.DEPTH_FRAME)

                upload(orgFrame) //     
            }
            streamingHandler.post(this)
        } catch (e: Exception) {
            Logger.d("Streaming, error: " + e.message)
        }
    }
}
streamingHandler.post(streamRunnable) //


FrameReleaser（）を使用して、フレームタイプのビデオストリームから個々のフレームを取得します。applyFilter（）を使用して、フレームにさまざまなフィルターを適用できます。



目的の形式のフレームを取得するには、フレームを適切なタイプに変換する必要があります。私たちの場合、1つ目はVideoFrameタイプで、2つ目はDepthFrameです。



デバイスの画面に画像を表示したい場合は、upload（）メソッドがあります。パラメータには、画面に表示されるフレームのタイプが示され、カラーカメラからのフレームがあります。



フレームを画像に変換



次のステップは、VideoFrameとDepthFrameから必要な形式の画像を取得することです。これらの写真を使用して、画像内の人物が実際の人物に属しているかどうかを判断し、データベースに情報を追加します。



画像フォーマット：



  • 色、VideoFrameから取得した拡張子.bmp
  • 深度マップを使用し、拡張子.tiffを持ち、DepthFrameから取得


変換を実装するには、OpenCVオープンソースコンピュータービジョンライブラリが必要です。すべての作業は、Matオブジェクトを形成し、それを目的の形式に変換することです。 



fun videoFrameToMat(videoFrame: VideoFrame): Mat {
    val colorMat = Mat(videoFrame.height, videoFrame.width, CvType.CV_8UC3)
    val returnBuff = ByteArray(videoFrame.dataSize)
    videoFrame.getData(returnBuff)
    colorMat.put(0, 0, returnBuff)

    val colorMatNew = Mat()
    Imgproc.cvtColor(colorMat, colorMatNew, Imgproc.COLOR_RGB2BGR)
    return colorMatNew
}


カラー画像を保存するには、CvType.CV_8UC3タイプのマトリックスを作成し、それをBRGに変換して、色が通常の色相になるようにする必要があります。

Imgcodecs.imwriteメソッドを使用して、デバイスに保存します。 



fun VideoFrame.saveToFile(path: String): Boolean {
    val colorMat = videoFrameToMat(this)
    return Imgcodecs.imwrite(path + COLOR_IMAGE_FORMAT, colorMat)
}


画像は深度センサーからのデータを含むフレームから構築されるため、マトリックスがCvType.CV_16UC1型である必要があるという唯一の違いを除いて、同じことをDepthFrameに対して行う必要があります。 



fun depthFrameToMat(depthFrame: DepthFrame): Mat {
    val depthMat = Mat(depthFrame.height, depthFrame.width, CvType.CV_16UC1)
    val size = (depthMat.total() * depthMat.elemSize()).toInt()
    val returnBuff = ByteArray(size)
    depthFrame.getData(returnBuff)
    val shorts = ShortArray(size / 2)
    ByteBuffer.wrap(returnBuff).order(ByteOrder.LITTLE_ENDIAN).asShortBuffer().get(shorts)
    depthMat.put(0, 0, shorts)
    return depthMat
}


デプスマップで画像を保存： 



fun DepthFrame.saveToFile(path: String): Boolean {
    val depthMat = depthFrameToMat(this)
    return Imgcodecs.imwrite(path + DEPTH_IMAGE_FORMAT, depthMat)
}


Face SDKの操作



Face SDKには大量のソフトウェアコンポーネントがありますが、それらのほとんどは不要です。 RealSense SDKと同様に、ライブラリはC ++で記述され、Androidでの作業を便利にするラッパーを備えています。 Face SDKは無料ではありませんが、開発者であれば、テストライセンスが与えられます。



ほとんどのライブラリコンポーネントは、XML構成ファイルを使用して構成されます。構成に応じて、これまたはそのアルゴリズムが適用されます。

開始するには、FacerecServiceクラスのインスタンスを作成する必要があります。このインスタンスを使用して他のコンポーネントを初期化し、DLLへのパス、構成ファイル、およびライセンスをパラメーターで渡します。



次に、このサービスを使用して、FacerecService.ConfigおよびCapturerクラスのオブジェクトを作成する必要があります。 



private val service: FacerecService = FacerecService.createService(
                dllPath,
                confDirPath,
                onlineLicenseDir
        )
private val confManual: FacerecService.Config = service.Config("manual_capturer.xml")
private val capturerManual: Capturer = service.createCapturer(confManual)


Capturerクラスは顔認識に使用されます。 manual_capturer.xml構成は、OpenCVライブラリのアルゴリズムを使用することを意味します。これはViola-Jones正面顔検出器です。Haar機能が認識に使用されます。ライブラリは、認識品質と実行時間の点で異なる構成を持つ既製のXMLファイルセットを提供します。遅いメソッドほど、認識パフォーマンスが向上します。プロファイル内の顔を認識する必要がある場合は、別のXML構成ファイルcommon_lprofile_capturer.xmlを使用する必要があります。設定はたくさんあります、それらについてはドキュメントでもっと読むことができます。私たちのケースでは、common_capturer4_singleface.xml構成を使用する必要がありました。これは、品質しきい値が低下した構成であるため、常に1人しか返されません。



画像内の顔を見つけるには、capturerSingleFace.capture（）メソッドを使用します。このメソッドには、人物の顔を含む画像のバイト配列が転送されます。 



fun createRawSample(imagePath: String): RawSample? {
    val imageColorFile = File(imagePath)
    val originalColorByteArray = ImageUtil.readImage(imageColorFile)
    return capturerSingleFace.capture(originalColorByteArray).getOrNull(0)
}


RawSampleオブジェクトは、検出された顔に関する情報を格納し、さまざまなメソッドのセットを含みます。たとえば、getLandmarks（）を呼び出すと、顔の人体測定ポイントを取得できます。



人の実在の人物への所属



顔検出カメラに取り付けられた写真ではなく、実際の人物がフレーム内にいるかどうかを判別するために、Face SDKライブラリーはDepthLivenessEstimatorモジュールを提供し、次の4つの値のいずれかを持つ列挙型を返します。



  • NOT_ENOUGH_DATA-デプスマップの欠損値が多すぎる
  • REAL-観察された人は生きている人に属します。
  • FAKE-観察された人は写真です。
  • NOT_COMPUTED-計算に失敗しました


モジュールの初期化： 



val depthLivenessEstimator: DepthLivenessEstimator = service.createDepthLivenessEstimator(
           "depth_liveness_estimator_cnn.xml"
   )


人の実際の人への帰属の決定： 



fun getLivenessState(
            rgbPath: String,
            depthPath: String
    ): DepthLivenessEstimator.Liveness {
        
val imageColorFile = File(rgbPath + COLOR_IMAGE_FORMAT)
        val originalColorByteArray = readImage(imageColorFile)

        val originalRawSimple = capturerSingleFace.capture(originalColorByteArray).getOrNull(0)
        val originalRawImage = RawImage(
                SCREEN_RESOLUTION_WIDTH,
                SCREEN_RESOLUTION_HEIGHT,
                RawImage.Format.FORMAT_BGR,
                originalColorByteArray
        )

        val originalDepthPtr = Natives().readDepthMap(depthPath + DEPTH_IMAGE_FORMAT)

//  
        val hFov = 69.4f 
        val vFov = 42.5f 

        val depthMapRaw = DepthMapRaw()
        with(depthMapRaw) {
            depth_map_rows = originalRawImage.height
            depth_map_cols = originalRawImage.width
            depth_map_2_image_offset_x = 0f
            depth_map_2_image_offset_y = 0f
            depth_map_2_image_scale_x = 1f
            depth_map_2_image_scale_y = 1f
            horizontal_fov = hFov
            vertical_fov = vFov
            depth_unit_in_millimeters = 1f
            depth_data_ptr = originalDepthPtr
            depth_data_stride_in_bytes = (2 * originalRawImage.width)
        }
        return depthLivenessEstimator.estimateLiveness(originalRawSimple, depthMapRaw)
}


パラメータとしてのgetLivenessState（）メソッドは、画像へのリンクを取得します：色と深度マップ。色付きのものからRawImageオブジェクトを作成しますこのクラスは、生の画像データとオプションのトリミング情報を提供します。 DepthMapRawは、元のカラーイメージに従って登録された深度マップである深度マップから形成されます。これは、estimateLivenessメソッド（originalRawSimple、depthMapRaw）を使用するために実行する必要があります。これにより、実際の人物がフレーム内にいるかどうかを示す列挙が返されます。



DepthMapRawオブジェクトの形成に注意を払う価値があります。変数の1つはdepth_data_ptrと呼ばれます-これは深度データへのポインターですが、Javaで知っているように、ポインターはありません。ポインタを取得するには、JNI関数を使用します、深度マップを使用した画像へのリンクを引数として受け取ります。 



extern "C" JNIEXPORT jlong JNICALL Java_ru_face_detect_Natives_readDepthMap
(JNIEnv *env, jobject obj, jstring jfilename)
{
    const char * buf = env->GetStringUTFChars(jfilename, NULL);
    std::string filename = buf;
    env->ReleaseStringUTFChars(jfilename, buf);

    cv::Mat depth_map = cv::imread(filename, -1);
    unsigned char * data = new unsigned char[depth_map.rows * depth_map.cols * depth_map.elemSize()];
    memcpy(data, depth_map.data, depth_map.rows * depth_map.cols * depth_map.elemSize());

    return (jlong) data;
}


KotlinでCで記述されたコードを呼び出すには、このタイプのクラスを作成する必要があります。 



class Natives {

    init {
        System.loadLibrary("native-lib")
    }

    external fun readDepthMap(fileName: String): Long
}


System.loadLibrary（）では、.cppファイルの名前が渡されます。これには、readDepthMap（）メソッドが含まれます。この例では、native-lib.cppです。また、外部修飾子をインストールする必要があります。つまり、このメソッドはKotlinに実装されていません。



顔識別



同様に重要な機能は、フレームで見つかった人物の識別です。Face SDKでは、Recognizerモジュールを使用してこれを実装できます。初期化： 



val recognizer: Recognizer = service.createRecognizer(
        "method8v7_recognizer.xml",
        true,
        true,
        true
)


認識率が最も高い設定ファイルmethod8v7_recognizer.xmlを使用しますが、認識品質はメソッド6v7および7v7よりも低くなります。



人を特定する前に、サンプルの写真に従って対応を見つけるために、顔のリストを作成する必要があります。実装するには、テンプレートオブジェクトからベクターを作成する必要があります。 



var templates = Vector<Template>()
val rawSample = createRawSample(imageUrl)
val template = recognizer.processing(rawSample)
templates.add(template)


テンプレートを作成するには、recognizer.processing（）メソッドを使用し、RawSampleをパラメーターとして渡します。顔テンプレートを含むリストを作成したら、それをRecognizerに追加して、結果のTemplatesIndexを保存する必要があります。これは、大規模なデータベースでのクイック検索に必要です。 



val templatesIndex = recognizer.createIndex(templates, SEARCH_THREAD_COUNT)


この段階で、識別に必要なすべての情報を含むRecognizerオブジェクトを作成しました。 



fun detectFaceSearchResult(rgbPath: String): Recognizer.SearchResult {
    val rawSample = createRawSample(rgbPath + COLOR_IMAGE_FORMAT)
    val template = recognizer.processing(rawSample)

    val searchResult = recognizer.search(
            template,
            templateIndex,
            searchResultCount,
            Recognizer.SearchAccelerationType.SEARCH_ACCELERATION_1
    ).firstElement()
    return searchResult
}


cognizer.search（）関数は、見つかった要素のインデックスを取得できる結果を返し、データベースの人物のリストと照合して、人物を識別します。さらに、類似性の値、0から1までの実数を確認できます。この情報は、スコープ変数Recognizer.MatchResultクラスで提供されます。 



val detectResult = detectFaceSearchResult(rgbPath)
//    -    0  1.
val scoreResult = detectResult.matchResult.score


結論



将来このようなシステムがどこでも使用されることは間違いありません。入り口に近づくと、ドアが自動的に開き、オフィスのコーヒーマシンが好みの粉砕度を選択します。



Android SDKでは、開発者が顔識別システムを操作できるようにするAPIが徐々に追加されていますが、現在はすべてが開発の初期段階にあります。また、Androidタブレット、Face SDKライブラリ、Intel RealSense 3Dカメラを使用したアクセス制御システムについて説明する場合は、柔軟性と拡張性に注目したいと思います。デバイスへのバインドはなく、カメラは最新のスマートフォンに接続できます。サポートされている3Dカメラの範囲を拡大したり、1つのデバイスに複数の部品を接続したりできます。作成したアプリケーションをAndroid Thingsに適合させることが可能、スマートホームで使用します。Face SDKライブラリの機能を見ると、その助けを借りて、連続したビデオストリームに顔の識別を追加し、性別、年齢、および感情を特定できます。これらの可能性は、多くの実験の範囲を提供します。そして私たちは私たち自身の経験から言うことができます：実験を恐れないで、自分自身に挑戦してください！
