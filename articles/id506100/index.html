Ising Model


Model Ising diperkenalkan untuk memahami sifat feromagnetisme dan memengaruhi studi transisi fase dan fenomena kritis. Feromagnetisme menggambarkan penampilan magnetisasi spontan dalam ferromagnet di bawah suhu tertentu - titik Curie. Pada titik Curie (kisaran suhu sempit), pemesanan terjadi, dalam hal ini, penyelarasan momen magnet, yang memerlukan transisi fase, yaitu, sifat-sifat zat berubah secara tiba-tiba.

Awalnya, model ini diusulkan oleh V. Lenz pada tahun 1920, dan diselidiki oleh Ising untuk kasus satu dimensi, untuk menghormati namanya. Dalam model ini, setiap simpul kisi kristal mengambil angka: +1 atau -1, - bidang "naik" atau "turun". Angka yang cocok dengan simpul disebut spin. Dengan demikian, kisi N berputar dapat dalam 2 N negara. Setiap negara berhubungan dengan energi yang diperoleh dari interaksi berpasangan dari spin atom tetangga:



Dimana J adalah energi interaksi spin tetangga (konstanta interaksi pertukaran adalah sama untuk semua pasangan), s i dan s j adalah spin.

Selain itu, J> 0 menggambarkan perilaku feromagnet, J <0 adalah antiferromagnet.

Metode Monte Carlo


Seperti disebutkan sebelumnya, total 2 N keadaan dimungkinkan, dan untuk jumlah N berputar yang cukup besar, sulit untuk mendapatkan hasil numerik. Sebagai contoh, untuk N = 10, kami memperoleh 2 10 keadaan yang tidak begitu mudah untuk langsung memodelkan, oleh karena itu, pendekatan statistik digunakan untuk pemodelan.

Dalam pendekatan ini, sistem dipertimbangkan dalam keadaan kesetimbangan termodinamika pada suhu tertentu T. Selama pertukaran energi dengan lingkungan, energi akan berubah mendekati keadaan kesetimbangan, dan energi rata-rata dari satu partikel sebanding dengan T. Implementasi perubahan acak konstan di sekitar keadaan kesetimbangan menggunakan metode Monte Carlo dan pemodelan dapat dibagi menjadi beberapa tahap:

  1. Mainkan status α i dari sistem (misalnya, secara acak) pada T tetap;
  2. Untuk mempertimbangkan keadaan-keadaan ini karakteristik termodinamika dekat kesetimbangan (energi E, magnetisasi M);
  3. Rata-rata nilai yang diperoleh

Model Ising dua dimensi dalam C ++


Kami akan menulis program kami sendiri yang mengimplementasikan model Ising dua dimensi menggunakan metode Monte Carlo dan algoritma Metropolis. Kami menerapkan fungsi berikut:

  • Pengguna menetapkan ukuran sisi perapian UKURAN, suhu maksimum yang diizinkan T_MAX.
  • ISING SIZE×SIZE, , (0), , , (1), .
  • T ISING. , W S ( ). , – 0,01℃.
  • – . T=2,269℃ .
  • Perubahan konfigurasi akan terjadi langsung di layar setiap 10 ms (milidetik); dengan setiap perubahan konfigurasi, jumlah langkah Monte Carlo akan meningkat.
  • Pengguna dapat mengubah suhu sebelum setiap perubahan baru dalam konfigurasi kisi spin.
  • Setiap 10 detik (detik), data akan dikeluarkan ke konsol (koefisien penerimaan, energi rata-rata per putaran, rata-rata kuadrat energi per putaran, rata-rata magnetisasi, rata-rata kuadrat magnetisasi).

#include <iostream>
#include <random>
#include <locale>
#include "glut.h"

using namespace std;

#define SIZE 256
#define SIZE_PX 800
#define T_MAX 4
#define CURIE_SCALE 1000. 

double quadSize = SIZE_PX / (double)SIZE;

short lattice[SIZE][SIZE];
double w[5];
double T = T_MAX, M, E;

int ratio = 0;
size_t nmcs = 0;
double ecum = 0, e2cum = 0, mcum = 0, m2cum = 0;

void display()
{
	glClear(GL_COLOR_BUFFER_BIT);
	
	glBegin(GL_QUADS);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			glColor3f(lattice[i][j], lattice[i][j], lattice[i][j]);
			glVertex2d(i, j);
			glVertex2d(i, j + 1);
			glVertex2d(i + 1, j + 1);
			glVertex2d(i + 1, j);
		}
	glEnd();

	glBegin(GL_QUADS);
	{
		glColor3f(1, 0, 0);
		glVertex2d(SIZE, 0);
		glVertex2d(SIZE, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, 0);

		glColor3f(1, 1, 1);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
	}
	glEnd();

	glutSwapBuffers();
	glFlush();
}

void calcW()
{
	double e4 = exp(-4 / T), e8 = e4 * e4;
	w[0] = w[4] = e8;
	w[1] = w[3] = e4;
	w[2] = 0;
}

void init()
{
	M = E = 0;
	static std::default_random_engine generator;
	std::uniform_int_distribution<int> distribution(0, RAND_MAX);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			lattice[i][j] = ((distribution(generator) / (double)RAND_MAX >= 0.5) - 1) * 2 + 1;
			M += lattice[i][j];
		}

	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			E += (i + 1 != SIZE) ? lattice[i][j] * lattice[i + 1][j] : 0;
			E += (j + 1 != SIZE) ? lattice[i][j] * lattice[i][j + 1] : 0;
		}

	calcW();
}

void metropolis()
{
	int x, y, sum;
	for (int i = 0; i < SIZE * SIZE; i++)
	{
		x = rand() % SIZE;
		y = rand() % SIZE;
		sum = lattice[(x - 1 + SIZE) % SIZE][y] +
			lattice[(x + 1 + SIZE) % SIZE][y] +
			lattice[x][(y - 1 + SIZE) % SIZE] +
			lattice[x][(y + 1 + SIZE) % SIZE];

		if (sum * lattice[x][y] <= 0 || (rand() / (double)RAND_MAX) < w[sum / 2 + 2])
		{
			lattice[x][y] = -lattice[x][y];
			::ratio++;
			M += 2 * lattice[x][y];
			E -= 2 * lattice[x][y] * sum;
		}
	}
}

void timer1(int)
{
	display();

	metropolis();

	nmcs++;

	ecum += E;
	e2cum += E * E;
	mcum += M;
	m2cum += M * M;


	glutTimerFunc(10, timer1, 0);
}

void outputData(int)
{
	double norm = 1 / (double)(nmcs * SIZE * SIZE);

	cout << "  = " << ::ratio * norm << endl;
	cout << "    = " << ecum * norm << endl;
	cout << "     = " << e2cum * norm << endl;
	cout << "  = " << mcum * norm << endl;
	cout << "   = " << m2cum * norm << endl << endl;

	glutTimerFunc(10000, outputData, 0);
}

void keyboard(unsigned char key, int x, int y)
{
	switch (key)
	{
	case 'w': case 'W':
	{
		T += 0.01;
		T = (T >= T_MAX) ? T_MAX : T;
		calcW();
		break;
	}
	case 's': case 'S':
	{
		T -= 0.01;
		T = (T <= 0.01) ? 0.01 : T;
		calcW();
		break;
	}
	}
}

void motion(int x, int y)
{
	double k = 1 - y / (double)SIZE_PX;
	T = T_MAX * k;
	T = (T >= T_MAX) ? T_MAX : T;
	T = (T <= 0.01) ? 0.01 : T;
	calcW();
}

int main()
{
	setlocale(LC_ALL, "ru");
	init();
	glutInitDisplayMode(GLUT_SINGLE | GLUT_RGB);
	glutInitWindowSize(SIZE_PX + 100, SIZE_PX);
	glutCreateWindow("ISING");
	glMatrixMode(GL_PROJECTION);
	glLoadIdentity();
	gluOrtho2D(0, SIZE + SIZE / 8, 0, SIZE);
	glutDisplayFunc(display);
	glutKeyboardFunc(keyboard);
	glutMotionFunc(motion);
	glutTimerFunc(1, timer1, 0);
	glutTimerFunc(10000, outputData, 0);

	glutMainLoop();
}

Perhatikan tujuan membuat beberapa fungsi dalam produk perangkat lunak ini:

  • init () - membuat konfigurasi awal;
  • display () - rendering kisi putaran dan skala suhu;
  • calcW () - perhitungan probabilitas transisi;
  • metropolis () - implementasi algoritma Metropolis;
  • timer1 () - memperbarui data setelah setiap perubahan konfigurasi, menambah jumlah langkah Monte Carlo;
  • outputData () - perhitungan rata-rata per putaran, output dari data yang diperlukan;
  • keyboard () dan motion () - kemampuan untuk mengubah suhu menggunakan skala dengan dua cara - masing-masing tombol pada keyboard dan klik mouse;

Layak untuk membuat beberapa kesimpulan:

  • Pada suhu kritis T = 2.269 ℃, transisi fase terjadi dari keadaan tidak teratur menjadi keadaan feromagnetik yang teratur.
  • Ketika suhu berubah ke bawah, karakteristik fisik dan statistik berikut berubah sebagai berikut:
    • Tingkat penerimaan berkurang;
    • Energi rata-rata per putaran berkurang;
    • Kuadrat rata-rata energi per putaran meningkat;
    • Magnetisasi rata-rata berkurang;
    • Kuadrat rata-rata dari magnetisasi meningkat.
