新世代のアレイ（ハイブリッドおよび完全にフラッシュベース）であるDell EMC Unity XTは、パフォーマンスが大幅に向上し、効率が向上し、マルチクラウド環境に新しい機能とサービスが追加されました。

NVMeストレージシステムは、遅延の影響を受けやすいアプリケーションで最高の性能を発揮します。たとえば、大規模なデータベースなどのアプリケーションでは、NVMeは低レイテンシと高ピークデータレートを提供します。レイテンシを減らして並行性を高めると、読み取り/書き込みパフォーマンスが大幅に向上します。 IDCの予測によると、2021年までに、NVMeおよびNVMe-oF（NVMe over Fabric）フラッシュアレイが世界の外部ストレージシステムからの収益の約半分を占めることは偶然ではありません。 

ストレージ効率は、データ圧縮アルゴリズムによって強化されます。 Dell EMC Unity XTでは、データを5分の1に削減できます。もう1つの重要な指標は、システム全体のパフォーマンスです。 Dell EMC Unity XTは85％のシステム容量を使用します。圧縮と重複排除はインラインで実行されます-コントローラレベルで。データは圧縮形式で保存されます。システムは、データのスナップショットを使用した作業も自動化します。

統一された（ブロックとファイル）アクセスを備えた使いやすいUnityフラッシュアレイは、安定した応答時間を提供し、クラウドストレージサービスと統合し、データ移行なしのアップグレードをサポートします。基本構成では、このユニバーサルストレージシステムは30分でインストールされます。

「ダイナミックプール」と呼ばれるデータストレージのテクノロジーにより、静的メモリから動的メモリの増加に切り替えることができ、操作の柔軟性が高く、システムの容量を簡単に増やすことができます。動的プールは容量と予算を節約し、再構築に必要な時間を短縮します。容量とパフォーマンスの構築Dell EMC Unityはデータ移行を必要としません。 

今日の多くの企業は、ローカルインフラストラクチャと組み合わせて、いくつかのパブリッククラウドサービスを使用しています。 Dell EMC Unity XTは、Dell Technologies Cloudのコンポーネントとして機能できます。このストレージはパブリッククラウドで使用でき、データをプライベートクラウドに転送できます。さらに、Dell EMC Unity XTストレージはサービスとして利用できます。これはDell EMC Cloud Storage Servicesの1つです。

ストレージプール


多くのSQL Server専門家は、すべての最新のストレージアレイが、固定のRAIDレベルの保護でディスクをより大きなストレージユニットにグループ化する機能を提供することを知っています。ドライブの個別のRAIDグループは、従来のストレージプールです。 Unity XTハイブリッドシステムは従来のプールのみをサポートしますが、Unity XTオールフラッシュアレイは動的ストレージプールも提供します。動的ストレージプールの場合、RAID保護はディスクエクステント（フルディスクよりも小さいストレージユニット）に適用されます。動的プールは、ディスクプールの管理と拡張において、より高い柔軟性を提供します。 

Dell EMCは、最小限の複雑さで最大のパフォーマンスを実現するストレージプール管理の推奨事項を提供します。たとえば、Unity XTストレージプールの数を最小限にして、複雑さを軽減し、柔軟性を高めることをお勧めします。ただし、追加のストレージプールを設定すると、次のような場合に非常に役立ちます。

  • 異なるI / Oプロファイルで個別のワークロードをサポートします。
  • 特定のパフォーマンスパラメータを達成するためにリソースを割り当てます。
  • マルチテナンシーに個別のリソースを割り当てます。
  • 障害から保護するために小さなドメインを作成する

ストレージボリューム（LUN）


アレイ内のボリューム数を選択するときに、管理と柔軟性の妥協点を見つけるにはどうすればよいですか？ UnityでSQL Serverを最大限に活用するには、データベースファイルごとにボリュームを作成することをお勧めします。実際には、ほとんどの組織は、重要なデータベースが最大限の柔軟性を得て、重要性の低いデータベースファイルが少数の大きなボリュームにグループ化される階層型アプローチを採用しています。データの保護と監視の技術はファイルの分離と配置に依存しているため、すべてのデータベース要件と関連アプリケーションを調査することをお勧めします。

多くの場合、特に仮想環境では、複数のボリュームを管理することが困難です。SQL Serverの仮想化環境は、単一のボリュームで複数の種類のファイルをホストすることが適切である場合の良い例です。作成するボリュームの数を決定するとき、データベースまたはストレージ管理者（またはその両方）は、柔軟性とメンテナンスの容易さの間で適切なバランスを選択する必要があります。

ファイルストレージ


NASサーバーは、Unity XTストレージシステム上のファイルシステムをホストします。ファイルシステムには、SMBまたはNFSプロトコルを使用してアクセスできます。マルチプロトコルファイルシステムのおかげで、両方のプロトコルを同時に使用できます。ホストをSMB、NFS、マルチプロトコルファイルシステム、およびVMware NFSデータストアとVMware仮想ボリュームに接続するために、NASサーバーは仮想インターフェースを使用します。ファイルシステムと仮想インターフェイスは単一のNASサーバー内に分離されているため、複数のNASサーバーをマルチテナンシーに使用できます。ストレージプロセッサに障害が発生すると、NASサーバーは自動的にフェイルオーバーします。関連するファイルシステムもフェイルオーバーします。

SQL Server 2012（11.x）以降のバージョンは、サーバーメッセージブロック（SMB）3.0プロトコルをサポートしています。これにより、ネットワークファイルを共有してストレージに保存できます。スタンドアロンインストールとフェールオーバークラスターの両方で、SMBストレージオプションを使用して、システムデータベース（マスター、モデル、msdb、tempdb）とカスタムデータベースエンジンデータベースをインストールできます。ファイル共有には高可用性ネットワーク共有へのアクセスが必要であるため、SMBストレージの使用は、Always On可用性グループを使用する場合に適したオプションです。

Unity XTストレージを使用してSQL ServerをデプロイするためのSMBファイル共有の作成は、NASサーバー、ファイルシステム、およびSMB共有を作成する必要がある、3つの簡単なプロセスです。Dell EMC Unisphereストレージ管理ソフトウェアには、このプロセスを完了するのに役立つ構成ウィザードが含まれています。ただし、SQL ServerワークロードをSMBファイル共有に配置するときは、SMBファイル共有の使用に必ずしも適用されないいくつかの重要な考慮事項を忘れないでください。マイクロソフトは、インストールとセキュリティの問題のリストを、現在知られている問題とともにまとめました。詳細については、Microsoftのドキュメントの「SMBファイルストレージを使用したSQL Serverのインストール」を参照してください。

薄いクローン


シンクローンは、ボリューム、コンシステントグループ、VMware VMFSデータストレージなどのシンブロックストレージリソースの読み取り/書き込みコピーであり、親リソースとブロックを共有します。シンクローンは、SQL Serverデータベースのコピーをすばやくコンパクトに表現する優れた方法です。これは、従来のSQL Serverツールを使用して実現することはできません。シンクローンがホストに提示された後、ボリュームをオンライン（オンライン）にすることができ、データベースはSQL Serverのデータベース接続方法を使用して接続されます。

シンクローンで更新機能を使用する場合は、シンクローンにあるすべてのデータベースを切断（オフラインで転送）します。これは、更新操作の前に行う必要があります。アップグレードを実行する前にデータベースをオフラインにしないと、SQL Serverでデータの不整合エラーや誤ったデータ結果が発生する可能性があります。

データ削減と高度な重複排除


ストレージシステムのDell EMC Unityファミリーは、豊富で使いやすいデータ削減サービスを提供します。構成されたプライマリストレージリソースだけでなく、これらのリソースのスナップショットやシンクローンでも節約が実現します。スナップショットとシンクローンは、ソースストレージ内のデータ量を削減する設定を継承するため、容量を節約できます。

データ削減機能には、重複排除、圧縮、ゼロブロックの検出が含まれます。これにより、ユーザーオブジェクトや内部使用に使用できるストレージスペースの量が増える可能性があります。 Unity XT 4.3以降の圧縮機能は、Unity XTのデータ削減機能に置き換わります。圧縮は、データセットを格納するために必要な容量の物理的な分布を減らすことができるデータ削減アルゴリズムです。

Unity XTシステムは、データ削減が有効な場合に有効にできる高度な重複排除機能も提供します。高度な重複排除は、Unityデータブロックの少数のコピー（多くの場合、1つのコピーのみ）を保存することにより、ユーザーデータに必要な容量を削減します。重複排除領域は1つのLUNです。ストレージスキームを選択するときは、このことを考慮してください。 LUNが少ないほど、重複排除は向上しますが、LUNが多いほどパフォーマンスが向上します。 

高度な重複排除による容量の節約により、ほとんどの環境で最高のパフォーマンスを実現できますが、Unityアレイプロセッサリソースを使用する必要もあります。 OE 5.0では、高度な重複排除が有効になっている場合、任意のブロック（圧縮または非圧縮）が重複排除されます。詳細については、Dell EMCドキュメント。

Unity SQL Server


SQL Server 2008 Enterprise Editionは、ネイティブデータ圧縮機能を備えた最初のリリースでした。 SQL Server 2008の行とページの圧縮では、内部SQL Serverデータベースのテーブル形式の知識を使用して、データベースオブジェクトが占めるスペースを削減します。スペースを削減すると、ページあたりの行数とバッファープールのページ数を増やすことができます。 NVARCHAR（MAX）などのオフラインデータなど、8kデータページ形式で保存されていないデータでは、行またはページの圧縮方法が使用されないため、MicrosoftはTransact-SQL COMPRESSおよびDECOMPRESS関数を導入しました。 

これらの関数は、従来のデータ圧縮アプローチ（GZIPアルゴリズム）を使用します。これは、圧縮または解凍のために各データセクションに対して呼び出す必要があります。

SQL Serverに固有ではないUnity XT圧縮は、ソフトウェアアルゴリズムを使用してストレージデータを分析および圧縮します。 Unity OE 4.1以降、フラッシュプール内のブロックストレージボリュームとVMFSデータストレージボリュームでUnityデータ圧縮を使用できるようになりました。 Unity OE 4.2以降、フラッシュドライブのストレージプールにあるファイルシステムとNFSデータストアでも圧縮が利用可能になりました。

SQL Serverのデータ圧縮方法の選択は、いくつかの要因に依存します。これらの要素には、データベースコンテンツのタイプ、ストレージとデータベースサーバーの両方で使用可能なCPUリソース、SLAを維持するために必要なI / Oリソースが含まれます。一般に、SQL Serverによって圧縮されたデータの追加のスペース節約が期待できますが、GZIPアルゴリズムを使用してTSQL圧縮機能を使用して圧縮されたデータは、Unity XT圧縮機能からのボリュームの大幅な追加の削減を受けることはほとんどありません。ユニバーサルアルゴリズム。

VMware


VMware vSphereは、仮想化とクラウドのための効率的で安全なプラットフォームです。 vSphereのコアコンポーネントは、VMware vCenter ServerとVMware ESXi Hypervisorです。

vCenter Serverは、統合されたvSphere環境管理プラットフォームです。リソースの導入とプロアクティブなリソース最適化は簡単です。 ESXiは、物理サーバーに直接インストールするオープンソースのハイパーバイザーです。 ESXiは主要なリソースに直接アクセスでき、150 MBの小さなサイズでメモリ要件を最小限に抑えます。さまざまなアプリケーションワークロードに信頼性の高いパフォーマンスを提供し、強力な仮想マシン構成（最大128個の仮想CPU、6 TBのRAM、120個のデバイス）をサポートします。

SQL Serverを最新のハードウェアで効率的に実行するには、SQL Serverオペレーティングシステム（SQLOS）がハードウェア構造を「理解」する必要があります。マルチコアおよびマルチノードの異種メモリアクセスシステム（NUMA）の登場により、コア、論理プロセッサ、物理プロセッサ間の関係を理解することが特に重要になりました。

記憶


最近のほとんどすべてのサーバーは、メインメモリとプロセッサ間の通信に不均一メモリアクセス（NUMA）アーキテクチャを使用しています。 NUMAは、物理プロセッサ間の物理メモリのブロックの分離を実装する共有メモリのハードウェアアーキテクチャです。 NUMAノードは、専用のメモリブロックを備えた1つ以上のCPUソケットです。 

過去10年間、NUMAは広く議論されてきたトピックです。 NUMAの相対的な複雑さは、特に、異なるベンダーの実装によるものです。仮想化環境では、NUMAの複雑さは、ハードウェアからハイパーバイザー、ゲストオペレーティングシステム、そして最後にSQL Serverアプリケーションまで、構成パラメーターとレベルの数によっても決まります。仮想化されたSQL Serverインスタンスを使用するSQL Serverデータベース管理者には、NUMAハードウェアアーキテクチャを十分に理解することが不可欠です。

コアの数が多いサーバーで効率を高めるために、MicrosoftはSoftNUMAを導入しました。SoftNUMAソフトウェアでは、1つのNUMA内で使用可能なCPUリソースをいくつかのSoftNUMAノードに分割できます。VMwareによると、SoftNUMAはVMwareの仮想NUMA（vNUMA）トポロジと互換性があり、ほとんどのワークロードに対してデータベースエンジンのスケーラビリティとパフォーマンスをさらに最適化できます...

SQL ServerによるVMware仮想化の場合は、

  • SQL Serverデータベースエンジンのメモリ不足を検出するための仮想マシンの監視。この問題は、I / Oの増加とパフォーマンスの低下につながります。

  • パフォーマンスを向上させるには、ESXiホストレベルでの過度のメモリ負荷を回避することにより、仮想マシン間のメモリの競合を防止します。
  • NUMA物理メモリのハードウェア割り当てを確認して、NUMA物理境界内の仮想マシンに割り当てることができるメモリの最大量を決定することを検討してください。
  • 適切なパフォーマンスを達成することが主な目的である場合は、割り当てられたメモリと同じメモリをバックアップすることを検討してください。このパラメーター設定は、仮想マシンが物理メモリのみを受け取ることを保証します。

仮想化ストレージ


仮想化環境でストレージを構成するには、ストレージインフラストラクチャの知識が必要です。NUMAの場合と同様に、さまざまなI / Oレベルがどのように機能するかを理解する必要があります。この場合、VM内のアプリケーションから、永続的なストレージメディアでの情報の物理的な読み取りと書き込みまでです。

vSphereは、Unity XTを使用したSQL Serverの実装に役立つアプリケーションを備えた多くのストレージ構成オプションを提供します。 FS VMFSは、Unity XTなどのブロックストレージシステムで最も広く使用されているストレージ方式です。 Unity XTアレイは、vSphereによって論理ドライブ（ボリューム）として表される物理ドライブで構成される下位層です。 Unity XTボリュームは、ESXiハイパーバイザーによってVMFSボリュームとしてフォーマットされます。 VMware管理者は、ゲストオペレーティングシステムに提供される1つ以上の仮想ディスク（VMDK）を作成します。 RDMにより、仮想マシンはVMFSをフォーマットせずに（FCまたはiSCSIを介して）Unity XTブロックストレージに直接アクセスできます。 VMFSボリュームとRDMボリュームは、同じトランザクションスループットを提供できます。 

ESXi用のNFSベースのストレージの場合、Dell EMCでは、汎用NFSファイルシステムではなくVMware NFSを使用することをお勧めします。 NFSデータストアのVMDKを使用してSQL Serverで実行されている仮想マシンは、基盤となるNFSレイヤーを認識していません。ゲストオペレーティングシステムは、仮想マシンをWindows ServerおよびSQL Serverを実行している物理サーバーと見なします。 NFSデータストアのフェイルオーバークラスターインスタンス構成の共有ドライブはサポートされていません。

VMware vSphere Virtual Volumes（VVols）は、物理メモリ（ボリュームやファイルシステムなど）の基本的な表現に関係なく、より詳細な仮想マシン管理を提供します。VVolを使用したアレイベースのレプリケーションは、VVol 2.0（vSphere 6.5）以降でサポートされています。RDMディスクの代わりにVVolディスクを使用して、vSphere 6.7以降、SQLフェイルオーバークラスターインスタンスにディスクリソースを提供し、永続的なSCSI冗長性をサポートできます。

仮想化ネットワーク


仮想世界のネットワークは、物理世界と同じ論理的概念に従いますが、物理的なケーブルやスイッチではなくソフトウェアを使用します。SQL Serverのワークロードに対するネットワーク待機時間の影響は大きく異なります。既存のワークロードまたは適切に実装されたテストシステムのネットワークパフォーマンスメトリックを代表的な期間にわたって監視すると、仮想ネットワークの作成に役立ちます。

SQL ServerでVMware仮想化を使用する場合は、次の点を考慮してください。

  • 標準の仮想スイッチと分散仮想スイッチの両方が、SQL Serverに必要な機能を提供します。
  • 管理トラフィック、vSphere vMotion、ネットワーク接続ストレージを論理的に分離するには、VLANタギングと仮想スイッチポートグループを使用します。
  • vSphere vMotionトラフィックまたはiSCSIトラフィックを含む仮想スイッチで大きなフレームを有効にすることを強くお勧めします。
  • 一般に、ゲストオペレーティングシステムとハードウェアのネットワークガイドラインに従ってください。

 結論 


SQL Serverデータベース環境は、ますます大規模で複雑になってきています。SQL Server 2019では、MicrosoftはSQL Serverのコア機能を改善し、Apache SparkとHDFSを使用したビッグデータワークロードのサポートなどの新しい機能を追加しました。Dell EMCは、Microsoftと共同で、SQL Server環境に必要なインフラストラクチャコンポーネント（サーバー、ストレージ、ネットワーク）を提供し続けます。 

ストレージとデータベースの専門家が共同で共有ストレージプラットフォーム上にSQL Serverのインフラストラクチャソリューションを作成すると、稼働時間が大幅に増加し、総所有コスト（TCO）が低下します。 Dell EMC Unity XTフラッシュアレイは、高パフォーマンスと低レイテンシを必要とするSQL Serverの開発者と管理者向けのミッドレンジソリューションです。 Unity XTオールフラッシュは、すべてのフラッシュドライブで動作するように設計されており、デュアルプロセッサーCPU、デュアルコントローラー構成、およびマルチコア最適化をサポートしています。

組織はますます、SQL Server環境を仮想化しています。仮想化は、アーキテクチャスタックに別の設計層を追加しますが、大きなメリットをもたらします。上記のSQL Server環境で最も一般的に使用されるVMwareの機能とツールをお楽しみください。詳細については、リソースへのリンクもお勧めします。
