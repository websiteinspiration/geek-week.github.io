ResizeObserver-適応性を操作するための新しい強力なツール

こんにちは！

レスポンシブは、Web開発の標準の1つです。画面の解像度は多数あり、この数は常に増加しています。フレンドリーなユーザーインターフェースを維持しながら、可能な限りすべての画面サイズをサポートするよう努めています。この問題の優れた解決策は、メディアクエリ（メディアクエリ）です。しかし、Webコンポーネントについてはどうでしょうか？最新のWeb開発はコンポーネントに基づいており、コンポーネントを応答可能にする方法が必要です。今日は、メディアクエリの場合のように、ビューポート全体（ビューポート）ではなく、特定の要素のサイズの変化を監視（観察）できるResizeObserver APIについてお話します。

ちょっとした歴史


以前は、自由に使用できるメディアクエリしかありませんでした-メディアデバイスの画面のサイズ、タイプ、解像度に基づくCSSソリューション（メディアデバイスとは、コンピューター、電話、またはタブレットを意味します）。メディアクエリは非常に柔軟で使いやすいです。長い間、メディアクエリはCSSでのみ利用可能でしたが、現在はwindow.matchMedia（mediaQueryString）を通じてJSでも利用可能です。これで、ページが表示されているデバイスから確認でき、表示領域のサイズの変化を監視することもできます（ここでは、MediaQueryList.addListener（）メソッドについて説明しています）。

要素クエリ


私たちが欠けていたのは、ビューポート全体だけでなく、単一のDOM要素のサイズを監視する機能でした。開発者はこれについて長年不満を述べてきました。これは、最も期待される機能の1つです。2015年、提案さえ提出されました-コンテナーサイズの要求（コンテナークエリ）：
多くの場合、開発者は、表示領域に関係なく、親コンテナーのサイズを変更するときに要素のスタイルを設定する機能を必要とします。コンテナサイズのリクエストにより、この機会が与えられます。CSSの使用例：
.element：media（min-width：30em）screen {***}
すばらしいように聞こえますが、ブラウザベンダーはこの提案を拒否する正当な理由がありました-循環依存（循環依存）（あるサイズが別のサイズを定義すると、無限ループが発生します（詳細はこちら）。他にどのようなオプションがありますか？window.resize（コールバック）を使用できますが、これは「費用のかかる喜び」です。イベントが発生するたびにコールバックが呼び出され、コンポーネントのサイズが実際に変更されたかどうかを判断するために多くの計算が必要になります...

ResizeObserver APIを使用して要素のサイズ変更を監視する


Chrome のResizeObserver APIをご覧ください：
ResizeObserver APIは、要素のサイズ変更を追跡するためのインターフェースです。これは、要素のwindow.resizeイベントの一種です。

ResizeObserver APIはライブドラフトです。PC用のChrome、Firefox、Safariにはすでに実装されています。モバイルのサポートはそれほど印象的ではありません。AndroidのChromeとSamsung Internetのみです。残念ながら、本格的なポリフィールは存在しません。利用可能なポリフィルにはいくつかの制限があります（たとえば、サイズ変更への応答が遅い、スムーズな移行をサポートしていないなど）。ただし、これによってこのAPIのテストが停止することはありません。じゃあやってみよう！

例：要素のサイズを変更するときにテキストを変更する


次の状況を想像してみてください-要素内のテキストは、要素のサイズに応じて変化するはずです。ResizeObserver APIは、ResizeObserverとResizeObserverEntryの2つのツールを提供します。ResizeObserverはアイテムのサイズを追跡するために使用され、ResizeObserverEntryにはサイズ変更されたアイテムに関する情報が含まれます。
コードは非常に簡単です：

<h1> size </h1>
<h2> boring text </h2>


const ro = new ResizeObserver(entries => {
    for(let entry of entries){
        const width = entry.contentBoxSize
        ? entry.contentBoxSize.inlineSize
        : entry.contentRect.width

        if(entry.target.tagName === 'H1'){
            entry.target.textContent = width < 1000 'small' : 'big'
        }

        if(entry.target.tagName === 'H2' && width < 500){
            entry.target.textContent = `I won't change anymore`
            ro.unobserve(entry.target) //  ,     500px
        }
    }
})

//       
ro.observe(document.querySelector('h1'))
ro.observe(document.querySelector('h2'))

最初に、ResizeObserverオブジェクトを作成し、コールバック関数をパラメーターとして渡します。

const resizeObserver = new ResizeObserver((entries, observer) => {
    for(let entry of entries){
        // 
    }
})

この関数は、ResizeObserverEntriesに含まれる要素の1つがサイズ変更されるたびに呼び出されます。コールバック関数の2番目のパラメーターは、オブザーバー自体です。たとえば、特定の条件が満たされたときに追跡を停止するために使用できます。

コールバックはResizeObserverEntryの配列を取得します。各エントリには、観測された要素（ターゲット）の次元が含まれます。

for(let entry of entries){
    const width = entry.contentBoxSize
    ? entry.contentBoxSize.inlineSize
    : entry.contentRect.width

    if(entry.target.tagName === 'H1'){
        entry.target.textContent = width < 1000 ? 'small' : 'big'
    }
    ...
}

要素のサイズを表す3つのプロパティ（borderBoxSize、contentBoxSize、contentRect）があります。それらは、後で説明する要素のボックスモデルを表します。次に、サポートについて少し説明します。ほとんどのブラウザーはcontentRectをサポートしますが、このプロパティは廃止される予定です。
contentRectはResizeObserverの予備開発段階で登場し、現在の互換性のためにのみ追加されました。おそらく将来的には廃止される予定です。


したがって、bordeBoxSizeおよびcontentBoxSizeと組み合わせてcontentRectを使用することを強くお勧めします。ResizeObserverSizeには、inlineSizeとblockSizeの2つのプロパティが含まれます。これらは、幅と高さとして解釈できます（テキストの水平方向で作業する場合-書き込みモード：水平）。

元素観察


最後に、アイテムの追跡を開始します。これを行うには、ResizeObserver.observe（）を呼び出します。このメソッドは、監視対象アイテムのリストに新しいターゲットを追加します。このリストには、1つまたは複数の要素を一度に追加できます。

// resizeObserver(target, options)
ro.observe(document.querySelector('h1'))
ro.observe(document.querySelector('h2'))

2番目のパラメーターは「オプション」です。これまでのところ、利用可能な唯一のオプションは、ブロックモデルを定義するボックスです。可能な値はcontent-box（デフォルト）、border-box、およびdevice-pixel-content-box（Chromeのみ）です。1つのResizeObserverで定義できるブロックモデルは1つだけです。

監視を停止するには、ResizeObserver.unobserve（target）を使用します。すべての要素の追跡を停止するには、ResizeObserver.disconnect（）を使用します。

オブザーバーはいつ変更について知るのですか？


ターゲット要素のサイズが変更されるたびにコールバックが呼び出されます。仕様はそれについて述べています：

  • 監視は、監視項目がDOMに追加またはDOMから削除されるとトリガーされます。
  • 観測された要素の表示プロパティがnoneに設定されている場合、観測がトリガーされます。
  • 観測は、「非置換」の線要素には機能しません。
  • 観察はCSS変換では機能しません。
  • , , .. , 0,0.
