iOSアプリでのAppleでのサインインの紹介

WWDC 2019で、Appleは再びiOS開発者の平和を妨害しました-新しいユーザー認証システムを導入しましたAppleにサインインします。現在、サードパーティの認証システム（Facebook、Twitterなど）を使用するすべてのiOSアプリケーションは、必ずAppleでサインインを実装する必要があります。私たちは運命を誘惑しないことに決め、この機能を実装するために走りました。正確にこれを行った-カットの下で見つけます。

アップルを通じて認可サービスを書く


私たちの作業では、VIPER + SOAアーキテクチャを使用しているため、別のサービスとしてAppleを通じて承認を行いました。最初に、認証のタイプ（Facebook、VK、Googleなど）を便利に拡張するために、データを列挙型でラップします。

enum AuthToken {
    case apple(code: String, name: String)
}

RxSwift のObservableを使用して結果を渡します。

protocol AuthProviderProtocol {
    var authResult: Observable<AuthToken> { get }

    func login()
    func logout()
}

プロトコルの実装：

import AuthenticationServices
import Foundation
import RxSwift

@available(iOS 13.0, *)
class AppleAuthService: AuthProviderProtocol {

    private let authResultSubject = PublishSubject<AuthToken>()
    var authResult: Observable<AuthToken> {
        return authResultSubject.asObservable()
    }

    func login() {
        let appleIDProvider = ASAuthorizationAppleIDProvider()
        let request = appleIDProvider.createRequest()
        request.requestedScopes = [.fullName, .email]

        let authorizationController = ASAuthorizationController(authorizationRequests: [request])
        authorizationController.delegate = self
        authorizationController.performRequests()
    }

}

@available(iOS 13.0, *)
extension AppleAuthService: ASAuthorizationControllerDelegate {

    func authorizationController(
        controller: ASAuthorizationController,
        didCompleteWithAuthorization authorization: ASAuthorization
    ) {
        guard
            let credential = authorization.credential as? ASAuthorizationAppleIDCredential,
            let tokenData = credential.authorizationCode,
            let token = String(data: tokenData, encoding: .utf8)
        else { return }

        let firstName = credential.fullName?.givenName
        let lastName = credential.fullName?.familyName

        authResultSubject.onNext(.apple(code: token, name: firstName + lastName))
    }

}
