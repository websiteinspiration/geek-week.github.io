Windows VPSの99ルーブルの関税をどうやって達成したか

VDSホスティングのクライアントの場合、オペレーティングシステムの標準イメージに何が付属するか考えたことはありますか？

標準のクライアント仮想マシンを準備する方法を共有し、標準のWindows Server 2019 Coreイメージを作成した方法を99ルーブルの新しいUltralight関税の例で示し、変更点を説明することにしました。


変更のリストはこのイメージに対してのみ有効です。デスクトップバージョンの場合、管理対象サーバーを箱から取り出して0.5ギガバイトに収まるようにするために、それほど多くの変更を行う必要はありません。

変更の完全なリスト


1.含まれるファイアウォールルール：

  • リモートイベントログ管理グループのすべてのルール
  • 仮想マシン監視（DCOM受信）
  • 仮想マシンの監視（エコー要求-ICMPv4受信）

2.ルールが変更された

  • Windowsリモート管理（HTTP受信）

3.削除されたコンポーネント：

  • Windows Defenderアンチウイルス

4.統合サービスが個人アカウントでインストールされます-Hyper-V Server Manager
5.圧縮されたすべてのファイルは、compact.exeによって圧縮されました。
6. oledlg.dllファイルを追加
7. RDPを有効化

更新


インストールプロセスは省略します。それ以上のことは行われません。インストール直後に、アップグレードする必要があります。このプロセスを最も便利にするために、Windows管理センターを使用しています。


これはSconfigでも実行できますが、これは私たちのオプションではありません。それ以外の場合は、左手を使用する必要があります。

管理を有効にする


次に、RSATを介してサーバーを制御できるように、ポートを開く必要があります。

これを行うには、リモートイベントログ管理および仮想マシン監視（DCOM受信）グループのすべてのルールを有効にします。つまり、タスクスケジューラ、イベントビューア、ローカルユーザー、perfmon、サービスのリストなど、ほとんどのRSAT機能が利用できるようになりました。Powershellを使用すると、ルールのグループ全体を含めることができます。これは1つのエレガントなコマンドで実行できます。

Enable-NetFirewallRule -DisplayGroup "Remote Event Log Management"


ファイアウォールにはルールがありますが、Server Coreでのボリュームとデバイスの管理はサポートされていません。

また、パブリックネットワークでWINRM管理を有効にするには、スコープを変更してWindowsリモート管理（HTTP-In）ルールを変更する必要があります。

Set-NetFirewallRule -name WINRM-HTTP-In-TCP-PUBLIC -Profile Any

Windows Defenderをアンインストールする


RAMについて

512メガバイトのRAMに収めるには、犠牲を払わなければなりません。追加のRAMをノックアウトするには、何かを投げる必要があります。そして、Windows Defenderを捨てます。

私たちは、このような操作を販促料金でのみ許可しました。

 

圧縮


私たちの関税はわずか10ギガバイトの空き容量を提供します。すべてのコンポーネントをインストールした後、オペレーティングシステムは9.64 GBを占有し始めますが、この数値はcompact.exeで改善できます。2つのターミナルを開き、1つでディスクのルートに移動して、次のコマンドを入力します。

compact /s /c /i /f /a /exe:lzx


LZXオプションはWindows Server 2016および2019でのみ使用できます。システムファイルはこれらのエディションでのみ圧縮されるため、スペースを節約したい場合、選択は適切ではありません。

2番目では、次のコマンドを入力します。

Compact /Compactos:always

Oledlg.dllを追加


Oledlg.dllは、GUIを使用してWindowsにダイアログボックスを実装するために必要な基本的なOLE関数を含むライブラリです。このファイルは、Server Coreを実際のワークステーションにするために必要です。

それは、とりわけ、外国為替ターミナルを配備して取引することを可能にします。
