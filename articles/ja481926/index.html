AWSソリューション用の新しいVeeam Backupをご覧ください

12月初旬に、Amazon Elastic Compute Cloudクラウドインフラストラクチャ（Amazon EC2）のバックアップとリカバリのための新しいVeeam Backup for AWSソリューションがリリースされました。

これを使用すると、EC2インスタンスのバックアップコピーを作成して、Amazon Simple Storage Service（Amazon S3）クラウドストレージに保存したり、EU2スナップショットのチェーンをネイティブ形式で作成したりできます。

Veeam Backup for AWSは、データ復旧のために次のオプションを提供しています。

  • EC2インスタンス全体を復元する
  • インスタンスボリュームの回復
  • ゲストOSインスタンスのファイルとフォルダーの復元

さらに、ソリューションはVeeam形式でバックアップを作成するため、Veeam Backup＆Replicationを使用してEC2バックアップのコピーをオンプレミスのリポジトリに保存し、クラウド、仮想、オンプレミスのインフラストラクチャ間でデータ移行を実行できます。

そしてもちろん、ユーザーは新しいソリューションの無料版に満足するでしょう。Veeam Backup for AWSの詳細については、Catへようこそ。



主な特徴


Amazon EBSスナップショットを自動的に作成し、Amazon S3クラウドにバックアップを保存するための前述の機能に加えて、ソリューションには次のものが含まれます。

  • バックアップ管理者向けの多要素認証
  • ポリシーベースのデータ保護
  • IAMロール分離のサポート
  • 地域間の構成のサポート
  • 支払いの制御に役立つサービスの運用コストを事前に見積もるための組み込みアルゴリズム。

さて、すでに述べたように、無料のライセンス、BYOL（独自のライセンスを作成）、およびリソースの消費に基づくライセンスがあります。誰でも適切なライセンスを選択できます。

バックアップポリシー


インスタンスを保護するポリシーを作成します。

オブジェクトのタイプごとに異なるポリシーを構成できます。たとえば、Tier 3アプリケーション（最も重要度が低い）を保護するように設計されたポリシー、またはTier 2およびTier 1アプリケーションのポリシーを構成できます。ポリシー設定で、次を指定します。

  • IAMロールを持つアカウント
  • 地域-複数選択できます
  • 保護するもの-すべてのリソース（すべてのリソース）、選択したインスタンス、または（タグ）
  • 除外するリソース
  • スナップショット設定（スナップショットを使用するかどうか、およびストレージの期間はどうあるべきかを含む）
  • バックアップ設定：リポジトリへのパス、ストレージのスケジュールと期間
  • サービスのコストの見積もり（以下について）
  • スケジュールと通知の設定

組み込みのコスト見積もり


Veeam Backup for AWSには、統合されたサービスコストの見積もりが組み込まれています。特定のポリシーに従って、バックアップサービスのコストを即座に計算します。計算には次のメトリックが含まれます。

  • バックアップ費用
  • スナップショットのコスト
  • トラフィックのコスト-リポジトリがインフラストラクチャが機能しているリージョンの外にある場合、これは特に重要です（Amazon AWSは他のリージョンにトラフィックを請求します）
  • 取引コスト
  • 総費用




データはCSVまたはXMLファイルにエクスポートできます。

サポートコンポーネント-ワーカー


トラフィックコストを削減するために、保護されたオブジェクトと同じAWSリージョンで補助コンポーネント（ワーカー）の自動作成を構成できます。ワーカーは、Amazon S3クラウドとの間でデータが転送されたとき、または復旧中にのみ自動的に起動し、操作が完了すると、電源がオフになり、削除されます。



バックアップ


Veeam Backup for AWSは、バックアップオペレーションにネイティブスナップショットを使用します（Amazon EBSスナップショットを参照）。バックアップ中、Veeam Backup for AWSはAWS CLIコマンドを使用して、EC2インスタンスに接続されたEBSボリュームのスナップショットを作成します。次に、バックアップシナリオに応じて、Veeam Backup for AWSはEC2インスタンスのネイティブスナップショットのチェーンまたはイメージレベルのバックアップを作成します。

ネイティブスナップショット


Veeam Backup for AWSは、EC2インスタンスのネイティブスナップショットを次のように作成します。

  1. 最初に、このインスタンスに接続されているEBSボリュームのスナップショットが削除されます。
  2. EBSスナップショットには、作成時にAWSタグが割り当てられます。これらのタグのキーと値には、暗号化されたメタデータが含まれています。Veeam Backup for AWSは、EBSメタデータスナップショットをEC2インスタンスのネイティブスナップショットとして扱います。
  3. EC2インスタンスがバックアップポリシーによって既に処理されている場合、Veeam Backup for AWSはスナップショットチェーン内のリカバリポイントの数をチェックします。ポリシーで指定された制限値を超えると、最も古いポイントが削除されます。注：ストレージと自動削除（保持）のポリシーは、手動で作成されたスナップショットには適用されません（個別に作成されたスナップショットについて説明しています）。ここで説明するように、そのようなスナップショットを削除できます。（「手動」とは、ポリシーのスケジュール外の手動起動を意味し、この方法で作成されたスナップショットの場合、保持は機能します。）

  1. 最初に、このインスタンスに接続されているEBSボリュームのスナップショットが削除されます。
  2. Veeam Backup for AWS EBS . .

Veeam Backup for AWSを使用すると、次のようにデータを回復できます。

  • 元のインスタンスを上書きして元の場所に移動します。このインスタンスのすべてのデータは、バックアップに保存されているデータで上書きされ、インスタンスの構成が保存されます。
  • 新しい場所に、新しいインスタンスを作成します。このシナリオでは、新しい場所または新しい設定で復元することを選択した場合、復元の完了後にインスタンスに適用される構成パラメーターを指定する必要があります。
    • 領域
    • 暗号化設定
    • インスタンス名とタイプ
    • ネットワーク設定：Virtual Private Cloud（VPC）、サブネット、セキュリティグループ



ボリュームの回復



また、スナップショットまたはバックアップから元の場所または新しい場所にEC2インスタンスボリュームを復元することもできます。2番目のケースでは、新しい場所のAWSリージョン、アベイラビリティーゾーン、およびその他のパラメーターを指定する必要があります。

回復プロセスには労働者も含まれます。

プロセス自体は簡単に次のようになります（バックアップからの回復の例を使用）。

  1. Veeam Backup for AWSは、ワーカーを目的のAWSリージョンで実行し、必要な数の空のEBSボリュームを作成して、ワーカーインスタンスにアタッチします。
  2. バックアップからこれらのボリュームにデータを復元します。
  3. EBSボリュームの切り離しを実行し、これらのボリュームが個別に格納されている目的の場所（ソースまたは他のAWSリージョン）に転送します。
  4. 操作の完了時にワーカーインスタンスを削除します。
    注：復元後、ボリュームはEC2インスタンスに自動的にアタッチされないことに注意してください（別のEBSボリュームとして指定された場所に保存されるだけです）。

ファイルの回復


インスタンス全体を復元しなくても、個々のファイルを復元できます。

ファイルレベルでリカバリを開始すると、（ワーカーのパブリックDNS名に基づいた）URLを取得します。これにより、ゲストOS上のファイル構造全体を確認し、必要なファイルを見つけてローカルマシンにアップロードできます。
また、セキュリティのために、証明書とそのフィンガープリントをチェックして、MiTMがないことを確認できます。



Veeam Backup＆Replicationとの統合


Veeam Backup＆Replicationがインフラストラクチャにデプロイされている場合、AWSへのダイレクトリストア機能を使用してそのマシンのAmazon EC2クラウドへのリカバリを構成し、Veeam Backup for AWSでこのクラウドデータを保護できます。
Veeam Backup＆Replicationは、Veeam Backup for AWSが作成するAmazon S3リポジトリもサポートしています。AmazonEC2インスタンスをオンプレミスインフラストラクチャに復元できます。

無料版の特徴


Veeam Backup for AWSの無料バージョンでは、最大10個のEC2インスタンスをバックアップできます。制限なしでバックアップから復元します。
注：t2.mediumの使用をお勧めします。
