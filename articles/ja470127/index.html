音楽を自動的に作成する



プログラミングを学んだ直後、音楽を作曲できるソフトウェアを作りたかった。

数年間、私はVisions of Chaosの音楽を自動的に作曲する原始的な試みをしました。基本的に、単純な数式またはランダムな一連の音符の遺伝的変異が使用されました。最近、TensorFlowとニューラルネットワークを研究してセルラーオートマトンを検索するために適度な成功を収めたため、ニューラルネットワークを使用して音楽を作成することにしました。

使い方


作曲家は、長い短期記憶（LSTM）を備えたニューラルネットワークを教えます。LSTMネットワークは、データシーケンスで次に何が起こるかを予測するのに適しています。LSTMの詳細については、こちらをご覧ください。


LSTMネットワークは、さまざまなノートのシーケンスを受信します（この場合、これらは単一チャネルのMIDIファイルです）。十分なトレーニングの後、彼女は教育用資料に似た音楽を作成する機会を得ます。


LSTMの内部には、威圧的に見えるが、使用してTensorFlowおよび/またはKerasは大幅LSTMの作成と実験を簡素化します。

モデルトレーニング用のソースミュージック


このような単純なLSTMネットワークの場合、ソース構成が単一のMIDIチャンネルで十分です。これに最適なのは、ソロからピアノまでのMIDIファイルです。Classical Piano Midi PageとmfilesでピアノソロのMIDIファイルを見つけ、それらを使用してモデルをトレーニングしました。

異なる作曲家の音楽を別々のフォルダに入れました。このおかげで、ユーザーはバッハを選択し、[作成]ボタンをクリックして、バッハのようになる（うまくいけば）曲を入手できます。

モデルをトレーニングする期間


元の音楽との結果の類似性は、モデルのトレーニングの期間（時代の数）によって異なります。元号の数が少なすぎると、結果の結果の繰り返し音符が多すぎます。時代が多すぎる場合、モデルは再トレーニングされ、元の音楽を単純にコピーします。

しかし、どのように多くの時代を止めるべきかをどうやって知るのですか

簡単な解決策は、500年代のトレーニング実行で50年代ごとにモデルと精度/損失グラフを保存するコールバックを追加することです。このおかげで、トレーニングが完了すると、50年代単位のモデルとグラフが表示され、トレーニングの進行状況がわかります。

class CustomSaver(keras.callbacks.Callback):
	def __init__(self):
		self.epoch = 0	
	# This function is called when the training begins
	def on_train_begin(self, logs={}):
		# Initialize the lists for holding the logs, losses and accuracies
		self.losses = []
		self.acc = []
		self.logs = []
	def on_epoch_end(self, epoch, logs={}):
		# Append the logs, losses and accuracies to the lists
		self.logs.append(logs)
		self.losses.append(logs.get('loss'))
		self.acc.append(logs.get('acc')*100)
		# save model and plt every 50 epochs
		if (epoch+1) % 25 == 0:
			sys.stdout.write("\nAuto-saving model and plot after {} epochs to ".format(epoch+1)+"\n"+modeldirectory + modelfileprefix + "_" + str(epoch+1).zfill(3) + ".model\n"+modeldirectory + modelfileprefix + "_" + str(epoch+1).zfill(3) + ".png\n\n")
			sys.stdout.flush()
			self.model.save(modeldirectory + modelfileprefix + '_' + str(epoch+1).zfill(3) + '.model')
			copyfile(notesfile,modeldirectory + modelfileprefix + '_' + str(epoch+1).zfill(3) + '.notes');
			N = numpy.arange(0, len(self.losses))
			# Plot train loss, train acc, val loss and val acc against epochs passed
			plt.figure()
			plt.subplots_adjust(hspace=0.7)
			plt.subplot(2, 1, 1)
			# plot loss values
			plt.plot(N, self.losses, label = "train_loss")
			plt.title("Loss [Epoch {}]".format(epoch+1))
			plt.xlabel('Epoch')
			plt.ylabel('Loss')
			plt.subplot(2, 1, 2)
			# plot accuracy values
			plt.plot(N, self.acc, label = "train_acc")
			plt.title("Accuracy % [Epoch {}]".format(epoch+1))
			plt.xlabel("Epoch")
			plt.ylabel("Accuracy %")
			plt.savefig(modeldirectory + modelfileprefix + '_' + str(epoch+1).zfill(3) + '.png')
			plt.close()

# get all the notes and chords from the midi files
def get_notes():

	# remove existing data file if it exists
	if os.path.isfile(notesfile):
		os.remove(notesfile)
	
	notes = []

	for file in glob.glob("{}/*.mid".format(mididirectory)):
		midi = converter.parse(file)

		sys.stdout.write("Parsing %s ...\n" % file)
		sys.stdout.flush()

		notes_to_parse = None

		try: # file has instrument parts
			s2 = instrument.partitionByInstrument(midi)
			notes_to_parse = s2.parts[0].recurse() 
		except: # file has notes in a flat structure
			notes_to_parse = midi.flat.notes

		for element in notes_to_parse:
			if isinstance(element, note.Note):
				notes.append(str(element.pitch))
			elif isinstance(element, chord.Chord):
				notes.append('.'.join(str(n) for n in element.normalOrder))

	with open(notesfile,'wb') as filepath:
		pickle.dump(notes, filepath)

	return notes

def prepare_sequences(notes, n_vocab):
	sequence_length = 100

	# get all pitch names
	pitchnames = sorted(set(item for item in notes))

	 # create a dictionary to map pitches to integers
	note_to_int = dict((note, number) for number, note in enumerate(pitchnames))

	network_input = []
	network_output = []

	# create input sequences and the corresponding outputs
	for i in range(0, len(notes) - sequence_length, 1):
		sequence_in = notes[i:i + sequence_length] # needs to take into account if notes in midi file are less than required 100 ( mod ? )
		sequence_out = notes[i + sequence_length]  # needs to take into account if notes in midi file are less than required 100 ( mod ? )
		network_input.append([note_to_int[char] for char in sequence_in])
		network_output.append(note_to_int[sequence_out])

	n_patterns = len(network_input)

	# reshape the input into a format compatible with CuDNNLSTM layers
	network_input = numpy.reshape(network_input, (n_patterns, sequence_length, 1))
	# normalize input
	network_input = network_input / float(n_vocab)

	network_output = np_utils.to_categorical(network_output)

	return (network_input, network_output)

# prepare the sequences used by the neural network
def prepare_sequences(notes, pitchnames, n_vocab):
	# map between notes and integers and back
	note_to_int = dict((note, number) for number, note in enumerate(pitchnames))

	sequence_length = 100
	network_input = []
	output = []
	for i in range(0, len(notes) - sequence_length, 1):
		sequence_in = notes[i:i + sequence_length]
		sequence_out = notes[i + sequence_length]
		network_input.append([note_to_int[char] for char in sequence_in])
		output.append(note_to_int[sequence_out])

	# reshape the input into a format compatible with LSTM layers
	normalized_input = numpy.reshape(network_input, (n_patterns, sequence_length, 1))

	model = keras.models.load_model(modelfile)

# generate notes from the neural network based on a sequence of notes
def generate_notes(model, network_input, pitchnames, n_vocab):
	# pick a random sequence from the input as a starting point for the prediction
	start = numpy.random.randint(0, len(network_input)-1)

	int_to_note = dict((number, note) for number, note in enumerate(pitchnames))

	pattern = network_input[start]
	prediction_output = []

	# generate 500 notes
	for note_index in range(500):
		prediction_input = numpy.reshape(pattern, (1, len(pattern), 1))
		prediction_input = prediction_input / float(n_vocab)

		prediction = model.predict(prediction_input, verbose=0)

		index = numpy.argmax(prediction)
		result = int_to_note[index]
		prediction_output.append(result)

		pattern.append(index)
		pattern = pattern[1:len(pattern)]

		if (note_index + 1) % 50 == 0:
			sys.stdout.write("{} out of 500 notes generated\n".format(note_index+1))
			sys.stdout.flush()

	return prediction_output

			notes_in_chord = pattern.split('.')
			notes = []
			for current_note in notes_in_chord:
				new_note = note.Note(int(current_note))
				new_note.storedInstrument = instrument.Piano()
				notes.append(new_note)
			new_chord = chord.Chord(notes)
			new_chord.offset = offset
			output_notes.append(new_chord)
		# pattern is a note
		else:
			new_note = note.Note(pattern)
			new_note.offset = offset
			new_note.storedInstrument = instrument.Piano()
			output_notes.append(new_note)

		# increase offset each iteration so that notes do not stack
		offset += 0.5

	midi_stream = stream.Stream(output_notes)

	midi_stream.write('midi', fp=midifile)

if __name__ == '__main__':
	generate()

モデルファイルサイズ


カオスのビジョンにニューラルネットワークを含めることの欠点は、ファイルのサイズです。モデルの生成が速い場合は、ボタンを追加して、エンドユーザーが自分でモデルをトレーニングできるようにします。しかし、多くのモデルのトレーニングセッションのいくつかは数日かかる場合があるため、これは特に実用的ではありません。自分ですべてのトレーニングとテストを行い、最良の作業モデルのみを追加する方がよいと私には思えました。また、エンドユーザーがボタンを押すだけで、トレーニングを受けたモデルが楽曲を作成することもできます。

各モデルのサイズは22メガバイトです。現代のインターネットの状況では、これはそれほど多くはありませんが、長年の開発の中で、Visions of Chaosのサイズは徐々に大きくなり、つい最近になって（セルオートマトン検索モデルにより）70 MBから91 MBに突然大きくなりました。そのため、これまでのところ、Visions of Chaosインストーラーに追加したモデルは1つだけです。もっと欲しいユーザーのために、私は別の1 GBモデルへのリンクを投稿しました。また、上記のスクリプトを使用して、midiファイルに基づいて独自のモデルを作成することもできます。
