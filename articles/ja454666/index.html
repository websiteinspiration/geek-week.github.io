こんにちは、ハブル。GNU Radioへの転送

に関する前の部分では、SDRを使用してLoRaプロトコル（低電力デバイスのデータ転送）をデコードできるかどうかについて質問されました。特にLoRa信号自体が異常である、いわゆるチャープスペクトラム拡散変調、または「チャープ変調」が原因で、このトピックは私にとって興味深いように思われました。 それがどのように機能するか、カットの下で続けました。





ちなみに、チャープ変調の文字通りの翻訳は「トゥウィートモジュレーション」のように聞こえますが、完全にシュールに聞こえるので、私は単純な単語「チャープ」を残したほうがいいでしょう（コメントで示唆されているように、これはロシア語では線形周波数変調と呼ばれます）。

LoRa変調


上記のように、LoRaを送信する場合、Semtechが特許を取得した方法で、「チャープ」を使用した変調方式が使用されます。式を使用した実装の詳細が必要な場合は、PDF をsemtech Webサイトまたはここで読むことができますが、非常に失礼な場合は、1つの「チャープ」が1つの周波数の変化であり、ビットストリームは上記の図に示すように、そのような変化でエンコードされます。 LoRaの信号パラメータは、SF（拡散率-実際には1つの「チャープ」の持続時間）と帯域幅-送信帯域幅です。 SFパラメータは、事前定義された値SF7-SF12によって設定されます。ここで、7が最も高速で、12が最も低速なモードです（たとえば、researchgateを使用して「チャーピング」のさまざまな速度を示す図を見ることができます）。

明らかに、「チャープ」の長さが短く、帯域幅が広いほど、伝送速度を上げることができます。これらすべては、おおよそ次の表に関連しています：



距離とノイズ耐性の観点から、ゆっくりと悲しいことに送信することは有利ですが、同時に速度を失います、次に、時間を失います。LoRaルールによれば、デバイスが送信できるのは1以下です他のデバイスに干渉しないようにするための時間の割合。したがって、低電力デバイスに最適な伝送速度を選択することも容易ではありません。

一般原則として、SDRとデコードに移りましょう。

鉄


テストには、LoRa Click RN2483とArduino M0を使用しました。



これは、プロトタイピングにとってかなり便利なフォームファクタです。はんだ付けすることなく、ボードを簡単に相互に交換できます（この形式では、MikroBUSと呼ばれ、さまざまな周辺機器を利用できます）。

本番のふりをしていないドラフトコードがスポイラーの下に追加されました。値「1234」はテストとして渡されます。

rn2483_tx.ino
// RN2483 Modem and LoRa Click test TX. Tested with Arduino M0

int reset = A2;
int rts = 9; // CS
int cts = 3; // INT

// the setup routine runs once when you press reset:
void setup()
{
    Serial1.begin(57600); // Serial port to radio

    // output LED pin
    pinMode(LED_BUILTIN, OUTPUT);
    
    pinMode(cts, INPUT);
    
    pinMode(rts, OUTPUT);
    digitalWrite(rts, HIGH);

    // Reset rn2483
    pinMode(reset, OUTPUT);
    digitalWrite(reset, LOW);
    delay(100);
    digitalWrite(reset, HIGH);

    delay(100);

    sendCommand("sys get ver\r\n");
    sendCommand("sys get hweui\r\n");

    sendCommand("mac pause\r\n");
    sendCommand("radio set mod lora\r\n");
    sendCommand("radio set pwr -3\r\n"); // the transceiver output power, from -3 to 15
    sendCommand("radio set sf sf8\r\n"); // sf7..sf12, sf7 the fastest spreading factor but gives the shortest range
    // sendCommand("mac set dr 0\r\n"); // data rate: 0-4, 4 faster
    sendCommand("radio set freq 869100000\r\n");
    // sendCommand("radio set afcbw 41.7\r\n");
    sendCommand("radio set rxbw 125\r\n");
    // sendCommand("radio set prlen 8\r\n");
    sendCommand("radio set crc on\r\n");
    // sendCommand("radio set iqi off\r\n");
    sendCommand("radio set cr 4/8\r\n");
    // sendCommand("radio set wdt 60000\r\n"); // disable for continuous reception
    // sendCommand("radio set sync 12\r\n");
    sendCommand("radio set bw 125\r\n");
}

void sendCommand(const char *cmd) 
{
    Serial1.print(cmd);
    String incoming = Serial1.readString();
    // SerialUSB.print(cmd); 
    // SerialUSB.println(incoming); 
}

// the loop routine runs over and over again forever:
void loop()
{
    char data[64] = {0};

    //  hexadecimal value representing the data to be transmitted, from 0 to 255 bytes for LoRa modulation and from 0 to 64 bytes for FSK modulation.
    sprintf(data, "radio tx 1234\r\n");
    sendCommand(data);

    if (msg_num > 10000) msg_num=0;

    digitalWrite(LED_BUILTIN, 1);
    delay(400);
    digitalWrite(LED_BUILTIN, 0);
    delay(600);
}

ちなみに、RN2483の最大宣言伝送距離は15 kmまでです。実際には、1階建ての建物が存在する場合、信号は1 kmだけ消え、都市の「丘陵地帯」では100 m以下になる場合があります。

モデムを起動し、デコードに進みます。

解読


GNU Radio自体にはLoRaサポートがないため、サードパーティのコンポーネントを使用する必要があります。それらは2つしかありませんでしたが、残念ながら、どちらの作者も名前に想像力を示さず、まったく同じ名前を付けました-gr-lora（https://github.com/rpp0/gr-lora and https://github.com/ BastilleResearch / gr-lora）。残念ながら、GNU Radioは一度に両方のコンポーネントを持つことができないため、1つのコンポーネントのインストーラーが他のコンポーネントのファイルを上書きします。

rpp0 / gr-lora githubから

デコーダーソースをダウンロードできます。アセンブリは標準であり、問​​題は発生しません。

git clone https://github.com/rpp0/gr-lora.git
cd gr-lora
mkdir build
cd build
cmake ..
make && sudo make install && sudo ldconfig

インストール後、GNUラジオに追加のブロックが表示され、そこからデコーダーを組み立てることは難しくありません。デコーダーのパラメーターとして、送信帯域幅、拡散率、SDRの中心周波数、および受信周波数を指定する必要があります。



GNU Radioのユニットは標準化されているため、RTL-SDRなどの任意の受信機を使用できます。SDRPlayを使用しました。シンプルなPythonプログラムを使用して、データをコンソールに出力しました。

udp_receive.py
import socket

UDP_IP = "127.0.0.1"
UDP_PORT = 40868

sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM) # UDP
sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
sock.bind((UDP_IP, UDP_PORT))
sock.settimeout(0.5)

while True:
    try:
        data, addr = sock.recvfrom(128) # buffer size is 1024 bytes
        print("Msg:", ' '.join('{:02x}'.format(x) for x in data))

    except socket.timeout:
        pass

BastilleResearch / gr-lora

このモジュールは、受信だけでなく送信でも機能するという点で注目に値します。インストールはほぼ同じです。コンポーネントをソースからアセンブルする必要があります。

git clone git://github.com/BastilleResearch/gr-lora.git
cd gr-lora
mkdir build
cd build
cmake ..
make && sudo make install && sudo ldconfig

このデコーダーの接続グラフを図に示します。



ご覧のとおり、さらにブロックがあります。回転子と多相リサンプラーが希望の周波数を選択し、過剰をカットします。復調器は「チャープ」をバイナリコードに変換します（出力は「17 00 3e 00 38 00 2f 00 01 00 39 00 2c 00 30 00 c6 00 18 00 7e 00 d5 00 85 00 e9 00 d8 00 67 00 c4 00”）、そしてデコーダが最終的なパッケージを形成します。

残念ながら、正常に機能しませんでした。デコードは確実に機能しますが、モデムが機能している間はデータが表示されますが、受信したメッセージは送信されたメッセージとは関係ありません。



それでも理由を理解できませんでした。どこかで設定を間違えたか、このデコーダーは自分のエンコーダーとしか互換性がありません。チャネルモデルを使用して、自分で確認できます。

LoRaWAN


ご覧のとおり、ここでは物理層の下位の伝送が考慮されています。上位レベルのLoRaWANプロトコルでは、暗号化、キー、その他のサービスを備えた別の論理層が上に配置されます。エンコーディングの仕組みを確認したい場合は、こちらのオンラインデコーダーをお試しください。

ちなみに、SDRを使用してLoRaWAN信号を受信して​​も、暗号化キー（その1つはプロバイダーのサーバーに格納されている）が存在しない場合でも、パケットの内容は認識されません。

結論


ご覧のとおり、SDRを使用したLoRaのデコードはかなり可能です。もちろん、SDRに基づいて実際のゲートウェイを作成することはほとんどお勧めできません。その感度は、弱い信号を受信するように特別に設計され、より狭帯域のフィルターとLNAを備えた「実際の」モデムの感度よりも劣ります。しかし、テストや研究では、これは非常に興味深い場合があります。
