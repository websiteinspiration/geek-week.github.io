トナーレベル制御


多くの管理者は、PRTGネットワ​​ークモニターを使用してすべてのプリンターのカートリッジのトナーレベルを制御することが非常に実用的であることに気付くでしょう。ただし、すべてのデバイスがSNMPセンサーを使用したオブジェクト識別子（OID）をサポートしているわけではありません。ただし、PRTGを使用してトナーのレベルを監視するオプションがあります。



一部のプリンターは、PRTGプリンターセンサーに組み込まれているオブジェクト識別子（OID）をサポートしていないため、データを要求しようとすると、SNMPエラーメッセージNo. 222が表示されます。このような場合、組み込みのPRTG SNMPセンサー（カスタムテーブル）が問題の解決に役立ちます。英語のインストールでは、SNMPカスタムテーブルセンサーとして指定されています。これを行うには、SNMPテーブルのOIDが必要です。充填レベルを表示するには、OID「1.3.6.1.2.1.43.11.1」を使用します。セットアップの最初のステップで入力してください。



PRTGはSNMPテーブルを使用して、現在のトナーレベルをリストに表示します。

次に、識別用の列をOID「1.3.6.1.2.1.43.11.1.1.6」に変更します。カートリッジの色を反映しています。次に、チャネル名を設定します。センサーの結果の横に表示されます（「充填レベル」など）。次に、感覚チャネルの列を一致させる必要があります。このため、OID「1.3.6.1.2.1.43.11.1.1.9」が導入され、現在の充填レベルがパーセントで反映されます。このチャネルの単位を構成するには、ドロップダウンリストで[パーセンテージ]項目を選択します。

次に、PRTGはプリンターのすべてのセンサーを読み取り、使用可能な値の表を表示します。追加するすべてのチャネルをチェックし、[保存]をクリックして設定を完了します。これで、PRTGは強調表示されたレコードごとにセンサーを調整し、読み取った情報に基づいてセンサーの名前を設定します。この時点から、カートリッジのトナー残量が表示され、変化が継続的に記録されます。イラスト付きの詳細なステップバイステップの説明については、PRTGナレッジベースを参照してください。

PRTGインベントリの概要


PRTG Network Monitorにより、デバイスツリーの構造は年々複雑になっています。したがって、ホスト間のすべての間隔の依存関係とアクティブな継承を確認することは時々理にかなっています。さらに、制限が導入されているすべてのセンサーとチャネルをリストする必要があります。これを行う方法を知っていれば、関連するレビューをすばやく作成できます。



大きくなり過ぎた監視システムには、時々在庫が必要です。

必要な情報は、PRTGから問題なく読み取ることができます。これを行うには、PRTGネットワ​​ークモニターに個別に適合させたPrtgAPIツール-C＃/ PowerShellインターフェイスを使用することをお勧めします。アプリケーションプログラムインターフェイス（API）を使用すると、センサーに関するほとんどすべての情報をPRTGから読み取り、それをスプレッドシートで処理できます。センサー識別子を含むチャネルのすべての特性を取得するには、APIで次のコマンドを実行します。

MQTTセンサーへの接続の確認


多くの中規模製造企業は、自社のマシンを徐々にネットワークに組み込んでいます。それらは特別に定義されたエリアで接続され、中央のMQTTブローカーに情報を送信します。MQTTブローカーでは、後続のデータ処理が行われます。 MQTTブローカーが完全にアクセス可能かどうかを監視することは、会社にとって重要です。このスコアには別の推奨事項があります。



MQTTリレーセンサーは、ブローカーに接続できるかどうかを確認します。

PRTG Network Monitorは、この目的のために特別なMQTTトランスポンダを提供しています。。ブローカーに接続できるかどうか、およびブローカーがパブリッシュを通じてメッセージを受け入れるかどうかを確認します。さらに、センサーは、ブローカーがメッセージを関連する参加者に送信するかどうかを制御し、一方では別の接続の時間を測定し、他方では、メッセージが発行された瞬間から再度受信されるまで、信号が往復するのに必要な合計時間を測定します。

MQTTの設定は非常に簡単で、わずか数ステップで完了します。MQTT Round Trip BETAセンサーがPRTGに追加されます。センサー設定では、センサーに名前が割り当てられ、MQTT（MQTTトピック）セクションが保存されます。デフォルト値はPRTG / roundtrip /％sensoridで、PRTGは％sensoridの代わりに対応するセンサーIDを使用します。センサーがMQTTセクションにアクセスできるようにするには、MQTT資格情報を親デバイスの設定に保存します。センサーを開始すると、メッセージがやり取りされた時間、通過の状態、パブリッシャーの接続時間、サブスクライバーの接続時間をセンサーのさまざまなチャネルで表示します。

メインPRTGサーバーへのリモートアクセス


PRTG Network Monitorは、ITインフラストラクチャを監視するために多くの企業で使用されています。多くの場合、企業ネットワークの外部にあるメインのPRTGサーバーにアクセスする方法に関する問題が発生します。この場合に管理者が考慮しなければならないこと、および第三者による不正アクセスを防ぐために重要な予防策は、次の推奨事項に示されています。

内部ネットワーク上に、192.168.0.100などの個人IPアドレスを使用してPRTGを構成しました。すべてが正常に動作しますが、今度は、別のコンピューターからインターネット経由でメインのPRTGサーバーにアクセスする必要もあります。これにはPRTGモバイルアプリを使用できます。ただし、これが可能になる前に、メインのPRTGサーバーにファイアウォールを介するなど、外部からアクセスできることを確認する必要があります。

セキュリティはアルファとオメガであるため、PRTGアカウントを入力する他の人の不可能性に最初に注意する必要があります。これを行うには、まだ行っていない場合は、管理者アカウントのデフォルトのパスワードを変更する必要があります。デフォルトのパスワードを使用しているかどうかを確認するには、PRTG Webインターフェースの「設定/アカウント設定/マイアカウント、ユーザーアカウントセクション」に移動します。

さらに、安全でないHTTPを介して外部からPRTGインスタンスにアクセスしないでください。したがって、PRTGプログラムが、PRTG Webインターフェースへのすべての接続にHTTPSを使用するように構成されていることを確認してください。これを行うには、「設定/システム管理/ Webインターフェイス、Webサーバーセクション」セクションの「着信HTTPSサーバー要求の伝送制御プロトコル（TCP）ポートとしてセキュアHTTPSサーバー」を選択します。メインサーバーのインストールに外部からアクセスするには、ファイアウォールで必要なポートを開くか、リダイレクトする必要があります。これを行うには、これらのポートのNATルールを作成します。ファイアウォールでのNATルールの設定は、製造元によって大きく異なります。したがって、デバイスまたはモデルのドキュメントをお読みください。

この例では、PRTGプログラムがIPアドレスが192.168.0.100のサーバーで実行されている場合、NATルールは図のようになります。 HTTPのルールはオプションですが、プロセスを大幅に簡素化します。ブラウザに「prtgserver。<Mydomain.tld>」と入力すると、最初にHTTP経由でWebサイトにアクセスしようとします。 HTTPのNATルールを作成したくない場合は、代わりに毎回prtgserver。<Mydomain.tld> を入力する必要があります。



これは、PRTGへのリモートアクセスを提供するファイアウォールのNATルールのように見える場合があります。

また、PRTGサーバーでWindowsファイアウォールが無効になっていること、または適切なルールを作成していることを確認してください。これで、PRTGモバイルアプリケーションを介してPRTGのインストールをリモートで監視したり、他のクライアントからインターネットを介してPRTG Webインターフェイスにアクセスしたりできます。ファイアウォールを介したPRTGネットワ​​ークモニターへのアクセスのセットアップの詳細（役立つ高度な設定を含む）については、「ハウツー」を参照してください。
