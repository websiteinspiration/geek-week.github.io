Schnellbereitstellung von vm ESXi mit Terraform

Hallo allerseits, mein Name ist Ivan und ich bin ein Administrator des alkoholischen Systems (OPS).

Ich möchte Ihnen erklären, wie Sie virtuelle Maschinen unter ESXi ohne vCenter mit Terraform bereitstellen.

Sehr oft müssen Sie virtuelle Maschinen bereitstellen / neu erstellen, um diese oder jene Anwendung zu testen. Wegen der Faulheit dachte ich darüber nach, den Prozess zu automatisieren. Meine Suche führte mich zu einem wunderbaren Hashicorp- Produkt , Terraform .

Ich denke, viele Leute wissen, was Terraform ist, und wer nicht weiß, dass dies eine Anwendung zum Verwalten von Clouds, Infrastrukturen oder Diensten ist, die das Konzept von IasC ( Infrastruktur als Code ) verwendet.

Als Virtualisierungsumgebung verwende ich ESXi. Ziemlich einfach, bequem und zuverlässig.
Ich sehe eine Frage voraus.
Warum Terraform, wenn Sie vCenter Server verwenden können?
Das kannst du natürlich, aber. Erstens ist dies eine zusätzliche Lizenz, zweitens ist dieses Produkt sehr ressourcenintensiv und passt einfach nicht auf meinen Heimserver, und drittens die Möglichkeit, Fähigkeiten zu aktualisieren.

Der Server ist die Intel NUC-Plattform:

CPU: 2 CPUs x Intel(R) Core(TM) i3-4010U CPU @ 1.70GHz
RAM: 8Gb
HDD: 500Gb
ESXi version: ESXi-6.5.0-4564106-standard (VMware, Inc.)

Und so, das Wichtigste zuerst.

Lassen Sie uns zunächst esxi konfigurieren, indem Sie den VNC-Port in den Firewall-Einstellungen öffnen.

Standardmäßig ist die Datei schreibgeschützt. Wir führen folgende Manipulationen durch:

chmod 644 /etc/vmware/firewall/service.xml
chmod +t /etc/vmware/firewall/service.xml
vi /etc/vmware/firewall/service.xml

Fügen Sie am Ende der Datei den folgenden Block hinzu:

<service id="1000">
  <id>packer-vnc</id>
  <rule id="0000">
    <direction>inbound</direction>
    <protocol>tcp</protocol>
    <porttype>dst</porttype>
    <port>
      <begin>5900</begin>
      <end>6000</end>
    </port>
  </rule>
  <enabled>true</enabled>
  <required>true</required>
</service>

Beenden, speichern. Wir ändern die Rechte zurück und starten den Dienst neu:

chmod 444 /etc/vmware/firewall/service.xml
esxcli network firewall refresh

In diesem Fall wird VNC benötigt, um eine Verbindung zur virtuellen Maschine herzustellen und den Pfad zur Kickstart-Datei anzugeben.

Eigentlich vor dem Neustart des Hosts. Danach muss diese Manipulation wiederholt werden.

Außerdem werde ich alle Arbeiten in einer virtuellen Maschine auf demselben Server ausführen.

Als nächstes brauchen wir Packer , ebenfalls ein Produkt von HashiCorp.

Er wird für die automatische Montage des "goldenen" Bildes benötigt. Was wir in Zukunft verwenden werden.

yum install unzip git -y

unzip packer_1.5.5_linux_amd64.zip -d /usr/bin && rm -rf packer_1.5.5_linux_amd64.zip

Beim Packer-Versionsschritt kann ein Fehler auftreten, da ein Paket mit demselben Namen möglicherweise auf RedHat basiert.

which -a packer
/usr/sbin/packer

Zum Lösen können Sie einen Symlink erstellen oder den absoluten Pfad / usr / bin / packer verwenden.

Weitermachen.

Auf einer Gita habe ich alles vorbereitet, was ich brauchte.

git clone https://github.com/letnab/create-and-deploy-esxi.git && cd create-and-deploy-esxi

Im ISO- Ordner müssen Sie die Distribution des Betriebssystems ablegen. In meinem Fall ist es centos 7.

Es ist auch notwendig, die Datei centos-7-base.json zu bearbeiten :

variables:     
iso_urls:  
iso_checksum:

Führen Sie nach allen Änderungen die Assembly aus:

/usr/bin/packer build centos-7-base.json

Wenn alles richtig konfiguriert und spezifiziert ist, sehen Sie ein Bild der automatischen Installation des Betriebssystems.

Terraform installieren:

curl -O https://releases.hashicorp.com/terraform/0.12.24/terraform_0.12.24_linux_amd64.zip
unzip terraform_0.12.24_linux_amd64.zip -d /usr/bin/ && rm -rf terraform_0.12.24_linux_amd64.zip
terraform version
Terraform v0.12.24

Da Terraform keinen Anbieter für ESXi hat, müssen Sie ihn erstellen.

Los geht's:

cd /tmp
curl -O https://dl.google.com/go/go1.14.2.linux-amd64.tar.gz
tar -C /usr/local -xzf go1.14.2.linux-amd64.tar.gz && rm -rf go1.14.2.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin
go version
go version go1.14.2 linux/amd64

Als nächstes sammeln wir den Anbieter:

go get -u -v golang.org/x/crypto/ssh
go get -u -v github.com/hashicorp/terraform
go get -u -v github.com/josenk/terraform-provider-esxi
export GOPATH="$HOME/go"
cd $GOPATH/src/github.com/josenk/terraform-provider-esxi
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-w -extldflags "-static"' -o terraform-provider-esxi_`cat version`
cp terraform-provider-esxi_`cat version` /usr/bin

Wir sind am Ziel. Lassen Sie uns unser Image ausrollen.

Gehen Sie zum Ordner:

cd /root/create-and-deploy-esxi/centos7

Bearbeiten Sie zunächst die Datei variables.tf . Sie müssen eine Verbindung zum ESXi-Server angeben.

Die Datei network_config.cfg enthält die Netzwerkeinstellungen der zukünftigen virtuellen Maschine. Wir ändern unsere Bedürfnisse und betreiben einen Einzeiler:

sed -i -e '2d' -e '3i "network": "'$(gzip < network_config.cfg| base64 | tr -d '\n')'",' metadata.json

Ändern Sie in der Datei main.tf den Pfad zur Eizellendatei in Ihren eigenen, falls abweichend.
