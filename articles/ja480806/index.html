カスタムCSSプロパティが好きな5つの理由

こんにちは、ハブル！Stas Melnikovによる記事「CSSカスタムプロパティが好きな5つの理由」の翻訳を紹介します。

カスタムCSSプロパティの仕様は、私のWeb開発に対する見方を永久に変えました。それが、カスタムCSSプロパティが好きな理由を書きたい理由です。

カスタムCSSプロパティとは何ですか？


つまり、CSSカスタムプロパティは、開発者が作成したプロパティです。ブラウザは、開発者がそれらを決定するまでそれらについて何も知りません。

カスタムプロパティの宣言は2つのダッシュで始まり、その後にプロパティの名前を追加する必要があります。次に、色を追加してプロパティ値を設定します。

たとえば、私は作成しました- ボタン要素に紫色の値を持つプロパティmelnik909BrandColor ：

button {
  --melnik909BrandColor: purple;
}


Var関数


カスタムCSSプロパティの最初の機能はvar関数です。この関数を使用すると、カスタムプロパティの値を取得して任意のCSSプロパティに追加するようにブラウザーに指示できます。

これを行うには、CSSプロパティを定義し、このプロパティの値を追加する必要があります。同じ場所に、カスタムプロパティを引数としてvar関数を設定します。

たとえば、borderプロパティとcolorプロパティに--melnik909BrandColorプロパティを追加しました。

button {
  --melnik909BrandColor: purple;
  color: var(--melnik909BrandColor);
  border: 2px solid var(--melnik909BrandColor);
}

カスタムプロパティはブラウザで直接変更できます


SassやLESSなどのツールを熟知している経験豊富な読者は、次のように言うかもしれません。私はSass変数を知っており、それで十分です。」

注意！Sass変数とLESS変数は、開発者がコードを保守しやすくするためにCSSを整理するのに役立ちます。たとえば、色を保存する変数$ melnik909BrandColorを作成しました。

$melnik909BrandColor: purple;

button {
  color: $melnik909BrandColor;
  border: 2px solid $melnik909BrandColor;
}

コンパイル後、ブラウザに次のコードが表示されます。

button {
  color: purple;
  border: 2px solid purple;
}


この例は、SassおよびLESS変数がブラウザーにないことを示しています。

カスタムCSSプロパティの2番目の機能は、ブラウザーにあることです。そうすれば、その場で値を変更できます。たとえば、ボタン要素の選択状態の—melnik909BrandColorプロパティの値を変更できます。

button {
  --melnik909BrandColor: purple;
  color: var(--melnik909BrandColor);
  border: 2px solid var(--melnik909BrandColor);
}

button:hover {
  --melnik909BrandColor: #27ae60;
}

カスタムCSSプロパティとメディアクエリを一緒に使用できます


カスタムCSSプロパティの3番目の機能は、メディアクエリを使用して値を変更できることです。たとえば、melnik909BrandColorプロパティを定義して、さまざまなデバイスでbody要素のテキストの色を変更します。

body {
  color: var(--melnik909BrandColor);
}

@media (max-width: 768px) {

  body {
    --melnik909BrandColor: purple;
  }
}

@media (min-width: 769px) {

  body {
    --melnik909BrandColor: tomato;
  }
}

Calc関数とCSSカスタムプロパティ


calc 関数を使用する必要がある場合があります。たとえば、行内の要素の数を数える必要がある場合は、この関数を使用します。

.child {
  width: calc(100% / 4);
}

.child {
  width: calc(100% / 3);
}

ただし、このコードを再利用することは難しいため、ここではカスタムCSSプロパティが役立ちます。要素の数を読み取る変数を作成できます。たとえば、前の例に–-itemsNumberプロパティを追加します。

.child {
  width: calc(100% / var(--itemsNumber));
}

そして、私は親要素で値を定義します：

.parent {
  --itemsNumber: 5;
}

/* or */

.parent {
  --itemsNumber: 7;
}

ベクターグラフィックスのカスタマイズ


カスタムCSSプロパティを使用してベクターグラフィックをカスタマイズする方法は2つあります。

最初の方法では、属性fill、stroke、stroke-widthを定義し、カスタムCSSプロパティをそれらの値として追加します。

さらに、カスタムプロパティの値を設定することで、属性値を追加できます。

<svg class="svg-with-attr" viewBox="0 0 55.867 55.867">
  <path 
    stroke="var(--iconStroke)" 
    stroke-width="var(--iconStrokeWidth)" 
    fill="var(--iconFill)" 
    d="...">
</svg>

これで、カスタムプロパティの値を設定して属性値を追加できます。

.svg-with-attr {
  --iconFill: #eeeeee;
  --iconStroke: #000000;
  --iconStrokeWidth: 1px;
}

左側はスタイルなしのデフォルトのアイコンで、右側はスタイル付きのアイコンです。

2番目の方法では、 CSSルールを使用して、stroke、stroke-width、およびfillプロパティのカスタムCSSプロパティを定義します。

body {
  --iconFill: #ffcc00;
  --iconStroke: #000000;
  --iconStrokeWidth: 2px;
}

.svg-with-props {
  stroke: var(--iconStroke);
  stroke-width: var(--iconStrokeWidth);
  fill: var(--iconFill);
}
