Yealink Meeting Serverベースのビデオ会議クラスター

この記事は、ビデオ会議Yealink Meeting Server（YMS）の統合ソリューションに特化した一連の出版物の続きです。

前回の記事「Yealink Meeting Server 2.0-新しいビデオ会議機能」では、ソリューション機能の大幅な進歩について説明しました。

  • YMSに統合された独自の会議録音サービスを追加
  • 新しいタイプのライセンスが登場-非対称会議のコストを最適化できるブロードキャスト
  • Skype for BusinessおよびTeamsと統合

この記事では、YMSをカスケード接続する可能性について検討します。システムをクラスターモードでインストールおよび構成します。

目的


YMSのハードウェアサーバープラットフォームのパフォーマンスにより、最新の高品質のビデオ会議サービスを必要とするほとんどの企業の問題を​​解決できます。1つのハードウェアMCU YMSで最大100のFullHD接続をサポートするソリューションがあります。しかし、それにもかかわらず、クラスターソリューションは需要があり、問題はサーバーのポート容量を拡張する必要があるだけではありません。

カスケードする理由はいくつかあります。

  • 世界中の数百、さらには数千の加入者を単一のVKSインフラストラクチャに統合する必要のある多くの企業があります。負荷分散は、最初のクラスター機能です。
  • VKSの最小のインストールでさえ、このサービスがビジネスプロセスにとって重要である場合、フォールトトレランスと高可用性が必要です。冗長性は、YMSクラスターに基づくフォールトトレラントシステムを構築する2番目の目標です
  • クライアント端末は、さまざまなネットワークだけでなく、世界のさまざまな場所にある場合があります。接続に最適なノードの選択による通信チャネルの最適化は、クラスターソリューションの3番目の切り札です。

取り付け


まず、各クラスターノードの役割を決定する必要があります。YMSソリューションにはこれらの役割が3つあります。

  • manager-master —
  • manager-slave-n —
  • business-n — -,

構成は次の通りである：
（1×マネージャマスタ）+（NX事業）
（1×マネージャマスタ）+（2 + NXマネージャスレーブ）+（NX事業）
したがって、マスタは、少なくとも二つのサーバによって予約されています。

各ノードに、CentOSなどのOSをインストールする必要があります。
YMSが機能するには、最小限のインストールで十分です。

Yealink Meeting Serverの現在のバージョンは、Yealinkの公式パートナー（私たちを含む）を通じて入手できます。

メインサーバー（manager-master）のusr / local /ディレクトリーに、たとえばWinSCPを介してYMSディストリビューションを配置する必要があります。

次に、コンソールを使用して、アーカイブを解凍してインストールを開始する必要があります。

cd /usr/local
tar xvzf YMS_22.0.0.5.tar.gz
cd apollo_install
tar xvzf install.tar.gz
./install.sh

install.shを 実行すると、インストールモードの選択肢が表示されます。

YMSの単一バージョンをインストールするには、[A]
を選択する必要があります。「クラスター」モードでインストールするには、[B]を選択します。



その後、システムは/ usr / local / apollo / data /ディレクトリーに移動し、install.confファイルを編集するように提案します。

このファイルには、ノードにアクセスするためのパラメーターと、ノード間の役割の分散が含まれています。

すべてのサーバーに同じアクセス設定がある場合、グローバル設定で、ルートアクセス用の単一のログインとパスワードを指定します。

[global]
ansible_ssh_user = root
ansible_ssh_pass = 1234567890

資格情報が異なる場合は、ノードごとに個別に指定できます。
例えば：

[manager-master]
ip=111.11.11.101
ansible_ssh_user = admin
ansible_ssh_pass = 0987654321

[manager-slave-1]
ip=111.11.11.102
ansible_ssh_user = root
ansible_ssh_pass = 1234567890

クラスターを構成するには、ノードのIPアドレスとアカウント情報（該当する場合）を役割ごとに割り当てます。

たとえば、クラスター（3 xマネージャー）+（3 xビジネス）は、原則に従って構成されます。

[manager-master]
ip=111.11.11.101

[manager-slave-1]
ip=111.11.11.102

[manager-slave-2]
ip=111.11.11.103

[business-1]
ip=111.11.11.104

[business-2]
ip=111.11.11.105

[business-3]
ip=111.11.11.106

役割が異なるように分散されている場合、不要な行を削除またはコメント化し、欠落している行を追加できます（例：business-4、business-5、business-6など）。install.conf

ファイルへの変更を保存した後、インストールプロセスを再起動する必要があります-install.sh システムは、ネットワーク上の使用可能なノードを自動的に検出し、それらにYMSを展開します。 Webベースのインターフェースを介してYMSクラスターを構成するときは、各サービスのパラメーターに特別な注意を払う必要があります。これにより、1つではなく、クラスターの一部であるいくつかのサーバーで既にアクティブ化できます。 ここでは、システム管理者の裁量により、機能は予約または分散されます。 サービスの設定に役立ちます
