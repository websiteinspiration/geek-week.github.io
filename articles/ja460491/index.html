Arduinoの温度および湿度センサーと送信およびプロット（パート1）

プロジェクトの主な目的は、家から11〜15メートル離れた温室の温度を監視することです。UTPの敷設とそれに力を組織する可能性なし。

温室部分：

  1. プロマイクロを使用してDHT-11センサーで温度/湿度を測定
  2. fs1000aをプロマイクロで送信

ホームパーツ：

  1. Unoでmx-rm-5vの値を受け入れます
  2. イーサネットシールドを使用してサーバーに送信（WIZNET W5100 HR911105A）
  3. mysqlデータベースに書き込む
  4. グラフを描く（jpgraph）



彼は少し乱れた。私にとって、このシーケンスはより複雑だったので、最初に実装することにしました。

必要なパッケージのリストは膨大ではなく、ディストリビューション、つまりnginx、php-gd、php-mysql、mysql-serverによって異なります。jpgraphの場合、ttfフォントが必要になる場合があります。

/**

 */
function writeMessage($message, $exception = null)
{
    $logfile = '/var/www/html/data/arduino.log';
    $datetime = date('d.m.Y H:i:s', time());

    if ($exception !== null) {
        $message .= ': ' . $exception->getFile() . ' (line: ' . $exception->getLine() . ') - ' . $exception->getMessage();
    }

    file_put_contents($logfile, '[' . $datetime . '] ' . $message."\n", FILE_APPEND);
}

テーブル作成
CREATE DATABASE `greenhouse` DEFAULT CHARACTER SET utf8 COLLATE utf8_general_ci;

CREATE TABLE `table` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `sensor` varchar(255) DEFAULT NULL,
  `temperature` decimal(6,3) DEFAULT '0.00',
  `humidity` decimal(6,3) DEFAULT '0.00',
  `created_at` datetime DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 COLLATE=utf8_general_ci;
