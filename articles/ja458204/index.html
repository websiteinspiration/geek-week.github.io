Linuxのストレージパフォーマンスを評価する方法：オープンツールを使用したベンチマーク

前回我々は、プロセッサとメモリの性能を評価するために、オープンソースとツールについて話しました。今日は、Linux上のファイルシステムとストレージシステムのベンチマーク、Interbench、Fio、Hdparm、S、Bonnieについて話しています。


写真-Daniele Levis Pelusi -Unsplash


フィオ


Fio（Flexible I / O Testerの略）は、Linuxファイルシステムのパフォーマンスを評価するためにディスクI / Oストリームを作成します。このユーティリティはWindowsでも実行できます。Cygwinコマンドラインインターフェースをインストールする必要があります。構成ガイドは、GitHubのfioリポジトリにあります。

Fio Author-Jens Aksbo（Jens Axboeによる）、I / O操作のルーティングのためのLinuxおよび開発者ツールblktraceのIOサブシステムを担当。特定の負荷を手動でテストするプログラムを書くのに疲れていたため、彼は fioを作成しました。

ユーティリティはIOPSとシステム帯域幅を計算し、I / Oキューの深さを評価することもできます。このユーティリティは、設定とテスト条件が記述された特別なファイル（.fio拡張子）で動作します。いくつかのテストオプションがあります。たとえば、任意の書き込み、読み取り、再書き込みがあります。ここでの例最初のケースのためのファイルの内容：

[global]
	name=fio-rand-read
	filename=fio-rand-read
	rw=randread
	bs=4K
	direct=0
	numjobs=1
	time_based=1
	runtime=900

現在、fioは大企業で使用されており、SUSE、Nutanix、IBMのユーティリティと連携しています。


Hdparm


ユーティリティは、2005年にカナダの開発者Mark Lordによって作成されました。それはまだ著者によってサポートされており、多くの人気のあるディストリビューションの一部です。hdparmの主な目的は、ドライブ設定を構成することです。ただし、このツールを使用して、読み取り速度の測定など、単純なベンチマークを実行できます。これを行うには、コンソールで次のコマンドを記述します。

$ sudo hdparm -t /dev/sdb

システムは同様の応答を生成します。

Timing buffered disk reads: 242 MB in 3.01 seconds = 80.30 MB/sec

ドライブ設定に関しては、hdparmを使用すると、キャッシュサイズの変更、スリープモードと電源パラメーターの変更、およびSSD上のデータの安全な消去を行うことができます。ただし、ArchLinuxの専門家が警告しているように、システムパラメータを不注意に変更すると、ディスク上のデータにアクセスできなくなり、ドライブに損傷を与えることさえあります。hdparmを使用する前に、マニュアルに慣れることをお勧めします。コンソールでman hdparmコマンドを記述するだけです。


S


これは、I / Oシステムのパフォーマンスを評価するための一連のベンチマークです。ユーティリティの作成者は、イタリアモデナ大学とレッジョエミリアの従業員を含むAlgoDevグループの開発チームでした。 すべてのベンチマークは、ストレージシステムのパフォーマンス（帯域幅、遅延、スケジューラーの作業）を評価する bashスクリプトです。たとえば、throughput-sync.shベンチマークは、読み取りまたは書き込み要求でストレージシステムを「ボンバード」します（この場合、前述のfioユーティリティが使用されます）。このスクリプトのコードは次のとおりです。 別のスクリプト-comm_startup_lat.sh-は、「コールドキャッシュ」（必要なデータがない場合）中にディスクからデータを読み取る際の遅延を測定します。コードも



リポジトリで見つけることができます。

ボニー


1989年に開発されたファイルシステムのパフォーマンスを評価するためのユーティリティ。著者はエンジニアのティム・ブレイ（Tim Bray）でした。ボニーの助けを得て、彼はウォータールー大学の新しいオックスフォード英語辞典プロジェクトに関与するコンピューティングシステムを最適化することを計画しました。 Bonnie は、ランダムな読み取りとディスクへのデータの書き込みを実行します。ユーティリティがCPU秒あたりの処理されたバイト数などのパラメーター、およびプロセッサー使用率のパーセンテージを表示した後。ベンチマークのソースコードはGoogle Codeにあります。 Bonnieに基づいて、ハードディスクをテストするための別のツールセットが構築されています-Bonnie ++



（CではなくC ++で書かれています）。追加のベンチマークツールを提供します。たとえば、zcavはHDDのさまざまなゾーンのパフォーマンスを評価します。Bonnie ++ は、メールサーバーやデータベースサーバーのテストにも適しています。


インターベンチ


ユーティリティは、によって開発されたコン・コリバス、Linuxカーネルとの彼の仕事への彼の貢献のために知られているオーストラリアの麻酔医「公正プロセッサスケジューラ。」 Interbenchは、I / Oスケジューラーとファイルシステム設定の構成に役立ちます。

Interbenchは、インタラクティブタスクを実行するときのCPUスケジューラの動作をエミュレートします。これらのインタラクティブなタスクには、サウンドとビデオの操作、コンピューターゲームの起動、オペレーティングシステムでのダイアログボックスのドラッグアンドドロップなどがあります。

ツールを設定するためのソースコード、例、推奨事項は、GitHubの公式リポジトリにあります。
