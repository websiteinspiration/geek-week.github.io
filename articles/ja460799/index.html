標準ブラウザAPI支払いリクエスト

最近の多くのブラウザには、支払い要求というAPIが組み込まれていることをご存知ですか。このAPIは、請求および連絡先情報の処理をサポートすることを目的とした W3C標準で説明されています。以下は、 developers.google.comの標準の概要です。実装の概要MDN WebサイトでのこのAPIの説明によると、Payment Requestは、ユーザーが好みの支払いシステムとプラットフォームをオンラインストアに関連付けることができる、ブラウザーベースのツールを開発者に提供します。これにより、商品やサービスの支払いの利便性が高まり、このプロセスが高速化および簡略化されます。特に、たとえば、Payment Request APIを使用すると、ユーザーは請求情報と住所を一度だけ入力でき、各サイトに同じ情報を入力することはできません。このAPIをサポートするすべてのサイトでの支払いは、ユーザーにとって同じように見えます。 Payment Request APIには、他にも重要な機能があります。その中でも-障害を持つユーザー向けの支払いツールの可用性に関する問題の解決、異なるユーザーデバイス間の支払いデータの同期、標準化されたエラー処理ツール。



今日公開している翻訳は、Payment Payment APIの実際の使用例です。Payment Request APIは将来的に変更される可能性のある実験的なテクノロジーであることはすぐに注目に値します。しかし、この資料の作者は、それにもかかわらず、彼女は彼女と知り合う価値があると述べています。彼は、このテクノロジーに明るい未来があり、それに加えることができる変更が、それを扱うことの基本原則にあまり影響を与えないことを確信しています。

基礎


Payment Request APIが広く採用されることで、オンライン決済システムの作成と使用に関わるすべての人にメリットがもたらされます。特に、プログラマーとサイト所有者にとって、これは支払い受け入れシステムの作成を容易にすることを意味し、ユーザーにとっては、商品とサービスの支払いプロセスを簡素化してスピードアップすることを意味します。

このAPIをJavaScriptで使用するには、まずオブジェクトを作成する必要がありますPaymentRequest。それを作成するコマンドは次のようになります。

new PaymentRequest(methodData: fn, details: fn, options?);

PaymentRequest2つの必須パラメーターと1つのオプションパラメーターが コンストラクターに渡されます。

  • パラメータmethodDataは、決済サービスプロバイダーに関する情報を含むオブジェクトの配列です。これは、たとえば、サポートされている支払い方法に関する情報です。
  • パラメータdetailsは、特定の支払いに関する情報を含むオブジェクトです。たとえば、支払い総額、税額、送料などです。
  • optionsオプションのパラメーターは、支払いに関する追加情報を含むオブジェクトです。

MDN のドキュメントには、Payment Request APIはHTTPSでのみ使用できるという情報があります。これは、このAPIがどのように機能するかを考えるとかなり明白です。

ユーザーがボタンをクリックした後、BUYオブジェクトの新しいインスタンスを作成しますPaymetnRequest。ここでは、オブジェクトのコンストラクターに渡す必要があるものを返すいくつかの関数を使用します。これらは関数buildSupportedPaymentMethodDataとbuildShoppingCartDetailsです。

コンストラクタの最初の引数は、関数呼び出しによって表されますbuildSupportedPaymentMethodData。サポートされている支払い方法であるオブジェクトの配列を返します。この関数はで宣言されていapp.jsます：

function buildSupportedPaymentMethodData() {
  //    :

コンストラクタの2番目の引数buildShoppingCartDetailsは、購入を完了するために必要な情報を生成する関数呼び出しです。

この情報は、例えば、購入された商品の説明、それらの価値、合計購入額を含み得る。関数コードは次のbuildShoppingCartDetails場所にもありますapp.js。

function buildShoppingCartDetails() {
  return {
    id: 'count-order',

        label: 'Example item',
        amount: {currency: 'USD', value: '1.00'}
      }
    ],
    total: {
      label: 'Total',
      amount: {currency: 'USD', value: count }

}

この関数はオブジェクトの配列ではなくオブジェクトを返すことに注意してください。

どのブラウザがPayment Request APIをサポートしていますか？

ご覧のとおり、このAPIは比較的新しいため、特に幅広いブラウザのサポートはまだありません。良い点は、Chrome、Safari、Firefox、Edgeでサポートされていることです。おそらくこれらのブラウザの少なくとも1つを使用しているため、上記で説明したすべてを完全に体験できます。

ペイメントリクエストAPIのサポートの分野では、ブラウザーがまだやらなければならないことがたくさんあることは明らかです。これは特にモバイルブラウザに当てはまります。したがって、今このAPIを使用する場合、これまでのところモバイルデバイスでの作業は当てになりません。
