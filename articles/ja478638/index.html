db-tree：データベースの検索とナビゲーション


この投稿では、データベース内の行をすばやく見つけてそれらをナビゲートするためのツールについて説明します。あなたがサポートで働いていて、たくさんのデータベースクエリをしなければならないなら、SELECTを書くのに疲れているなら、猫の下でお願いします。

動機


少し前、私は大規模な会計システムの維持を手伝いました。作業の過程で、データベースの情報を検索する必要がありました。一般的なシナリオ：問題のあるユーザーがリクエストN1を呼び出します。診断のために、このアプリケーションに関するいくつかのデータをデータベースに表示する必要があります。私たちは要求を満たします：

SQL SELECT * FROM ORDER WHERE ID = 'N1'

集計はアプリケーションに関連付けられているため、次のリクエストを実行して、集計に関する情報を取得します。 

SQL SELECT * FROM DEVICE WHERE ORDER_ID = 'N1'

次に、ユニットに関連するすべてのアプリケーションを探します。

SQL SELECT * FROM ORDERS WHERE DEVICEID = '92375'

等。Nクエリを実行した後、遅かれ早かれデータに問題が見つかり、対策を講じます。このアプローチの欠点は明らかです。

  • クエリを手動で作成するのは遅く、不便です。特に、データベース構造が複雑で、テーブルが多数ある場合。したがって、トンネル症候群を修正することが可能です。
  • 一意制約または外部キーによって関連する行を検索する必要がある場合は、新しいクエリを作成する必要があります。
  • 通常、データベースツールはデータを表形式で表示します。テーブルに多数の列がある場合、テーブルを水平にスクロールするか、クエリで列を選択する必要があります。この場合も、手作業が必要です。

考え


まず、検索を簡略化する必要があります。このアクションは、最小限のクリックで実行する必要があります。テキストボックスに目的の行を入力して、Enterキーを押すだけです。通常、主キーにはインデックスが付けられるため、主キーまたは一意の制約に含まれるすべての列で値をすぐに検索できます。

次に、ナビゲーションの問題を解決する必要があります。外部キーの関連エントリにすばやくジャンプするにはどうすればよいですか？データベースはファイルシステムと考えることができます。データベースの行がディレクトリで、外部キーによる関連行がシンボリックリンクであり、外部キーではないフィールドが単純なファイルだとします。ファイルシステムドライバーを作成するつもりはありません。それは単なる類推です。したがって、データベース行は階層構造として表すことができ、TreeTableコンポーネントを使用して表示できます。

また、列をTreeTableコンポーネントに追加することもできます。この列には、特定の行の意味のある値が表示されます。この値は、データベースの行フィールドの値を連結することで取得できます。たとえば、注文明細の場合、次の式を作成できます。
ORDER_NAME + ', ' + ORDER_STATUS + ', ' + ORDER_CUSTOMER

最も近い例：JavaのtoString（）メソッド。

実装


プログラミングには何ヶ月もかかりました。最初はC ++とQtを使用しようとしましたが、難しいことがわかりました。C++の世界では、jdbcドライバーに似たものはなく、言語自体ははるかに複雑です。したがって、アプリケーションはJavaで記述されています。



スクリーンショットには、検索フィールド、現在の接続を切り替えるためのコンボボックス、および階層データを表示するTreeTableコンポーネントが表示されています。

探す


テキストボックスに文字列を入力してEnterキーを押すことができます。検索は文字列および数値型の列（VARCHAR、NUMBERなど）でのみ機能するようになりました。日付と時刻の型はまだサポートされていません。デフォルトでは、ツールは主キーに含まれている列の値を検索します。設定では、検索で使用される他のフィールドにチェックマークを付けることができます。

キーナビゲーション


[F]というラベルの付いたノードは、外部キーです。[テーブル]列に、このキーが参照するテーブルの名前が表示されます。ノードを開いたら、関連する行に進みます。複合外部キーもサポートされています。

[U]というラベルの付いたノードは、一意の制約または主キーです。ノードを展開すると、関連する行に移動できます。スクリーンショット



を見てください。検索バーに10248と入力し、ORDERSテーブルに行を見つけました。[U] ORDER_IDノードを開いて、ORDER_DETAILSテーブルに3行を見つけました。次に、各ノードを展開して、ORDER_DETAILSテーブルの行に移動できます。

列文字列


主キー値は、多くの場合、情報を提供しません。前のスクリーンショットでは、値ORDER_ID = 10248、PRODUCT_ID = 11が表示されています。これらの数字からは何もわかりません。どういうわけかそれらを人間化するために、あなたは表現をすることができます：

'Product: ' + PRODUCT_ID.PRODUCT_NAME + ', Price: ' + UNIT_PRICE

入力します。Enterキーを押すと、より意味のある値が表示されます。



技術的な詳細


アプリケーションは、JavaFXのインターフェースであるJavaで記述されています。TreeTableがアイコンの代わりに文字列「[U]」と「[F]」を使用していることに気付くかもしれませんが、これはこの厄介なバグJDK-8190331が原因で行われました。データベースのパスワードは、java-keyringライブラリを使用して安全なリポジトリに保存されます。インストーラーのビルドには、OpenJDK 13と早期アクセスビルドjpackageが使用されます。ビルドコマンドはここにあります。

Oracle、MariaDB、PostgreSQLデータベースがサポートされるようになりました。
