非定型の「ls」またはlinuxoidの楽しみ方

Petersburg LinuxコミュニティSPbLUGの電報チャットで、面白いパズルを投げました。
lsまたはそのエイリアスを使用せずに、できるだけ多くの方法でホームディレクトリ内のファイルを一覧表示します（1つの方法-1つのポイント）

同じタスクが少し後で別のチャットに飛び込んできました。これがその原因です。

1.エコーして印刷する


for i in ~/*  ~/.* ; do echo $i ; done

echoコマンドをprintに置き換えても、まったく同じ結果になります。
実際、サイクルなしで実行できます。それほど美しくはありませんが、タスクの条件に適合します。

echo ~/* ~/.*

2.ツリー


より明白な方法は、ツリーを使用することです。これは、適切なキーを選択した場合、実質的にlsです。

tree -aiL 1 ~

3.見つける


また、明白な解決策ではありません。

find ~ -maxdepth 1 -mindepth 1

4. du


はい、人々はduを忘れませんでした。

du -ad 1 ~

5.タール


私たちは微妙な倒錯を伴う水手順に目を向けます。

tar -cvf /dev/null --no-recursion ~/* ~/.* 2>null

6. 7. PerlおよびPython


問題の状況では、現代のLinuxでは通常システムのすぐに使用できるインタープリターに制限をかけるのを忘れていたので、コックと蛇のブリーダーは離れることができませんでした：
Perl：

perl -e 'use feature "say"; opendir my $dh, "." or die "Could not open . for reading: $!\n"; while (my $thing = readdir $dh) { say $thing; };'

Python：

echo -e "import os\nfor i in os.listdir(os.getenv('HOME')): print(i)" | python

競争外


彼らはCのソースコードも山に配りましたが、コンパイラーはほとんどどこにでも存在しますが、組み込みディストリビューションを除いて、これは完全に混乱していると考えました。;-)

#include <stdio.h>
#include <stdlib.h>
#include <dirent.h>

#define HOME getenv("HOME")

int main(int argc, char const *argv[])
{
    struct dirent *dp;
    DIR *dir = opendir(HOME);
        while ((dp = readdir(dir)) != NULL)
            printf("%s\n", dp->d_name);
    closedir(dir);

    return 0;
}
