Qué hacer con el legado. Contenido con Badoo PHP Meetup # 4

¡Hola a todos!

El 15 de febrero, la próxima reunión de desarrolladores de PHP sobre el tema de Legacy se celebró en la oficina de Badoo. Todo el día escuchamos historias sobre la experiencia de grandes empresas, hablamos y compartimos el dolor.

Resultó considerar el problema desde varios lados:

  • organización de procesos de desarrollo para deshacerse del legado;
  • las tácticas de cortar monolitos heredados en microservicios;
  • Métodos de organización de API que le permiten controlar el crecimiento del código obsoleto;
  • métodos automáticos para detectar código muerto;
  • También intentaron hablar sobre cómo refactorizar el código heredado utilizando enfoques DDD;

Debajo del corte, comparto videos y diapositivas de esta reunión. Por supuesto, muchas cosas valiosas permanecieron al margen y no se incluyeron en el informe, así que venga a nuestro acogedor grupo para conversar sobre la difícil vida de un pahépéšnik, discutir informes o simplemente para pedir consejo :)



"Una victoria indolora sobre el legado"


Anton Zhukov, ManyChat


Una historia fundamental sobre cómo refactorizar el código heredado sin afectar la aplicación, probar la funcionalidad y el rendimiento, así como cambiar sin problemas a la nueva versión en el producto.


Diapositivas

"Un enfoque iterativo contra el legado"


Alexey Korotin, SuperJob


El enfoque SuperJob es una lucha constante contra el código obsoleto. El líder del equipo, Timlid, habló de la API como una forma de aislar el código incorrecto del bueno.


Diapositivas

Código muerto: buscar y desactivar


Danil Mukhametzyanov, Badoo


El problema que resolvimos se ve así: los ingenieros altamente remunerados leen constantemente el código muerto. Danil del equipo de desarrollo del servidor, sobre otros proyectos de los que recientemente hemos hablado . Su historia trata sobre cómo automatizamos la lucha contra Legacy y qué métodos usamos para controlar su apariencia.


Diapositivas


"Las tácticas cortan el monolito PHP"


Pavel Lakosnikov, Avito


Cortar un monolito es una forma segura de encontrar soluciones desactualizadas en código nativo. Pavel contó cómo Avito se deshizo del legado: eliminó diccionarios y otras estadísticas, asignó interfaces, simplificó la jerarquía de herencia y mejoró la cobertura de prueba.


Diapositivas


"Refactorizando el código PHP usando DDD"


Vitaly Chirkov, FunCorp


A pesar del hecho de que el tema de DDD es bastante complicado, y el último informe no es una prueba para todos, Vitaly fue excelente al explicar y literalmente explicar con los dedos cómo reanimar el código obsoleto utilizando patrones de programación orientados a temas.
