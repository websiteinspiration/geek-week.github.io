MicroSPA、または四角いホイールを発明する方法

みなさん、こんにちは。私の名前はアンドレイヤコベンコです。私はデジタルデザインのWeb開発者です。

当社では、サイトフィニティ Webコンテンツ管理システム、または単にCMS を使用して実装された多くのプロジェクトがあります。これを使用する理由は、この記事の前半で説明されています。CMSは通常マルチページアプリケーションです。今日は、フロントエンドフレームワークがサイトフィニティソリューションに追加できるものとその方法について説明します。



叙情的な余談


sitefinityにはAngular.jsが搭載されていますが、ソースコードが閉じているため、他のフレームワークを統合する機能はありません。ただし、これによって他のフレームワークの機能を使用できなくなるわけではありません。

この記事の例は万能薬ではなく、主に情報提供を目的としています。

新しいを検索


Angularは最初のバージョンですが、すでにサイトフィニティに存在しているため、CMSをReactまたはVue.jsと友達にしたいと考えました。

これらのフレームワークはどちらも独自の方法で優れており、アプリケーション開発に対する独自のアプローチがありますが、次の比較は行いません。

可能な解決策をざっと見てみたところ、興味深いプロジェクトが見つかりました。これは、Reactで記述されたページを静的htmlにコンパイルするものです。クライアントサイドでの実行可能コードの使用に加えて、SSR（サーバーサイドレンダー）の魅力をすべて失う必要がないため、このソリューションは適していませんでした。

言葉から行為へ


Sitefinityは、ほとんどのCMSと同様に、コンテンツ（ウィジェット）を表示するためのさまざまな要素を含むページを作成できます。たとえば、既製のウィジェットにVue.jsを埋め込むためのオプションを検討します。

最初の方法（シンプル）


この方法では、Vue.jsをライブラリとしてメインページテンプレートに接続します。

その後、任意のウィジェットで任意の場所にアプリケーションを初期化できます。

シンプルなウィジェットの例：

new Vue({
    el: '#widget-id',
    data: {
        msg: 'Hello world',
    },
})

ただし、これはウィジェットブロック全体がテンプレートとして使用され、Vue.jsがそれを補間しようとすることを意味します。適切な場所にメッセージを表示するには、何かがjsコードを一時停止させる場合に表示される特別な構成を記述する必要があります。レンダリング前、またはユーザーがJavaScriptを無効にしている場合。

この問題の解決策は、ウィジェットを複製する独自のテンプレートを作成することです。コンポーネントを拡張します。

例：

new Vue({
    el: '#widget-id',
    data: {
        msg: 'Hello world',
    },
    template: '<div>{{msg}}</div>',
})

これで、ウィジェットのすべてのhtmlコードが、テンプレートフィールドで説明されているテンプレートに置き換えられます。ウィジェットモデルからVueにデータを転送するには、便利な方法でモデルをJSONに変換し、データフィールドに転送するだけです。

2番目の方法（難しい）


この方法では、vue-loaderの最小構成のWebpackが必要です。また、html-extract-pluginを使用する必要はありません。テンプレートをjsコードに保存できます。

これで、Vueの単一ファイルコンポーネントを使用できます。

単一ファイルコンポーネントの例（Sample.vue）：

<template>
    <div>{{msg}}</div>
</template>

<script>
export default {
    name: 'Sample',
    data: function () {
        return {
            msg: 'Hello world',
        }
    },
}
</script>

単一ファイルコンポーネントには、従来のコンポーネントよりも多くのスタイルオプションがあり、プロジェクトディレクトリ内のファイル数も削減されます。

ウィジェットコンポーネントを作成したら、それを初期化する必要があります。

ウィジェットの初期化の例：

import Vue from 'vue'
import Sample from '../Sample.vue'

Vue({
    render: function (h) {
        return h(Sample)
    }
}).$mount('#widget-id')

ただし、この場合、コードはそれ自体で実行されます。たとえば、セレクターを指定するなど、アプリケーションの初期化を制御する必要がある場合は、ページ上のいつでも呼び出すことができるメソッドでアプリケーションをラップするのが合理的です。

例を拡張してみましょう：

import Vue from 'vue'
import Sample from '../Sample.vue'

function initWidget (selector) {
    Vue({
        render: function (h) {
            return h(Sample)
        }
    }).$mount(selector)
}

initWidget('#widget-id')

ウィジェットモデルデータをVue.jsコンポーネントに転送するために残ります。これは、Vueインスタンスのデータオブジェクトに渡す前にJSONに書き込むことで簡単に実行できます。その後、それらをプロパティとしてコンポーネントに渡すことができます。

例：

import Vue from 'vue'
import Sample from '../Sample.vue'

function initWidget (selector) {
    Vue({
        data: function () {
            return {
                some: 'some value',
            }
        },
        render: function (h) {
            return h(Sample, { props: data })
        }
    }).$mount(selector)
}

initWidget('#widget-id')

小さなレビュー


結論として、提案された方法のそれぞれが独自の、いわゆるアプリケーション環境を持っていると言いたいです。したがって、1つ目の方法は、ボタンやフォームなど、アプリケーションの1つのセクションをすばやく統合するのに適しています。2つ目の方法は、より高度で、アプリケーションの個々の部分と自己完結型ページの両方を書き込むためのより多くの機能を備えています。

ただし、どちらの方法にも1つの大きな欠点があります。フレームワークをこのように使用すると、SEOが依存するページのセクションに対して同じジョブを2回実行する必要があります。

一方、必要に応じて、このアプローチを使用して、任意のフレームワーク、ライブラリを接続したり、独自のDOMマネージャーをJavaScriptまたはjQueryで記述したりできます。

それで全部です。ご清聴ありがとうございました。
