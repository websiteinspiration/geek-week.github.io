Como criar um servidor PostgreSQL no Google Cloud Platform SQL

Uma tradução do artigo foi preparada antes do início do curso PostgreSQL .



Introdução


Neste artigo, apresentarei o GCP SQL e mostrarei como criar um servidor PostgreSQL neste serviço.

Como criar o PostgreSQL no GCP SQL


Vamos começar criando um servidor PostgreSQL no GCP SQL.

Antes de começar, faça login na sua conta do GCP. Para criar um servidor de banco de dados PostgreSQL -> selecione SQL no painel Serviços do Google.



Para criar uma instância do SQL -> Clique em CREATE INSTANCE.



Selecione o banco de dados que você precisa → Clique em PostgreSQL.



Preencha os detalhes da configuração:

ID da instância: - Nomeie sua instância,

defina uma senha - Senha,

Escolha a região mais próxima de você - Região,

selecione a versão do servidor de banco de dados - Versão do banco de dados.

Clique em Mostrar opções de configuração para configurar informações adicionais.



Expand Detalhes da configuração



Clique em Conectividade → Por padrão, a instância não pode ser acessada a partir de redes externas; Para conceder acesso a uma instância, permita um IP ou compartilhamento específico. Clique em + Adicionar rede.



Ao inserir 0.0.0.0/0 na rede, você permitirá todos os IPs IPv4 e poderá acessar a instância de qualquer lugar.



Em seguida, aguardamos que as configurações sejam salvas.



Para se comunicar com recursos adicionais e a API → ative o IP Privado e selecione a rede Associada (a rede VPC que você criou ou o padrão).



Estamos aguardando o salvamento novamente.



Você também pode verificar as conexões de serviço privado clicando em VPC → VPC Peering.



Selecione o tipo de máquina e armazenamento: número de núcleos e memória.



Se você pretende usar a instância para fins de teste, selecione uma única zona.



Se você tiver um ambiente de produção, selecione Alta disponibilidade, mas isso aumentará o custo.



Clique em Adicionar item para adicionar os sinalizadores do banco de dados.



Selecione a bandeira que você precisa.


.
O GCP atualizará o banco de dados a cada poucos meses; portanto, é solicitado que você defina uma data e hora com antecedência para reiniciar o banco de dados, o que não leva muito tempo. Portanto, escolha uma data e hora se você tiver um banco de dados de produção.



Especifique a chave e o valor para maior clareza na guia Etiquetas.



Clique em Criar para criar o banco de dados.



GCP cria servidor PostgreSQL:



sucesso! O servidor PostgreSQL foi criado com sucesso.



Você verá informações e uma visão geral do PostgreSQL recém-criado.



Ao clicar em Conexões, você pode ver os detalhes da conexão e também ver se o banco de dados foi autorizado a se comunicar com o GCP App Engine.



Para acessar o servidor PostgreSQL criado, precisamos criar um usuário na guia USUÁRIOS.



Você também pode criar um banco de dados na guia DATABASES.



Você pode gerenciar backups em BACKUPS.
