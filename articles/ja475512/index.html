MIROはオープンな屋内ロボットプラットフォームです。パート4-ソフトウェアコンポーネント：ARDUINO（AVR）



MIROプラットフォームのソフトウェアコンポーネントを引き続き分解します。より具体的には、AVRのソフトウェアについて検討したいと思います。したがって、この問題には2つの部分を当てます。最初に、ライブラリの一般的な構造を説明し、2番目に、いくつかの主要なクラスメソッドの実装について説明します。

目次：パート1、パート2、パート3、パート4、パート5。

ARDUINOの下のソフトウェアは、自作の最大のソフトウェアであることが判明しました。一般に、ロボットのアクチュエータとセンサーを直接操作するほとんどすべてのロジックは、AVRにあります。そしてこのレベルでは、APIが実装されています-MIROの迅速な開発ソフトウェアライブラリです。

API構造は、対応するリポジトリの wikiセクションで説明されています。これまでのところ、ロシア語のみ。次に、コードをさらに詳しく分析します。意図的にクラスの完全な宣言を行うことはせず、省略記号 "..."でそれを減らし、現時点で重要なものだけを残します。

私たちのソフトウェアモデルでは、各MIROロボットはシャーシと接続されたデバイスのセットで構成されています。設計時には、ロボットシャーシ-常に何らかの種類の車輪付きシャーシ-歩行ロボットまたは他の動作原理を使用するロボットは個別に検討する必要があると想定されていました。

設計者は、シャーシピンの初期設定とロボット構成の初期値を実行します。

Sync（）メソッドは、シャーシとロボットに接続されたすべてのデバイス、ARDUINOスケッチのメインループ（）ループの各ステップに必要な操作を実装します。 MiroクラスのMiro Sync（）メソッドは、シャーシおよびロボットに接続されたすべてのデバイスの対応するSync（）メソッドを呼び出します。

Robotクラスには、ロボットに接続されたデバイスの配列へのポインター、この配列を操作するためのメソッド（新しいデバイスの接続、切断、インデックスと名前による検索）も含まれています。また、Robotクラスには、Chassisクラスのオブジェクトであるシャーシが含まれます。

しかし、より簡単なものから始めましょう-デバイスを使って。ロボット（トロリー）に直接関係しないLED、センサー、アクチュエーターなどのロボットに接続できる各デバイスは、仮想デバイスクラスのすべてのデバイスに共通する後続クラスによって記述されます。

class Device
{
public:
	virtual void Sync();
	virtual void setParam(byte pnum, byte *pvalue);
        virtual void getParam(byte pnum, byte *pvalue);
	virtual byte getPinsCount();
	virtual char* getName();
	virtual byte getParamCount();

protected:	
	byte *pins[2];
};

仮想メソッドsetParam、getParam、getParamCountは、デバイスパラメータの数の割り当て、受信、および決定に関連付けられています。パラメータは任意のプロパティにすることができます：LEDの明るさ、サーボドライブの位置など。各デバイスの後続クラスは、独自の方法でこれらのメソッドを実装します。getName、getPinsCountメソッドの目的は、名前から明らかだと思います。新しく検出されたSyncメソッドは、メインループの反復ごとに定期的に実行する必要がある一部のデバイス操作の非ブロッキングデバイス制御および自動化のための特別なメソッドです。

超音波距離計（上記）のクラスを決定するには、親のメソッドに加えて、メソッドもあります。

  • Init-初期化;

  • _mesCount（getMesCountメソッドで使用される次元の数を格納します）;
  • _max_dist-測定に必要な最大距離*;

次に、Syncメソッドについて説明します。デバイスの状態が_isOn == TRUE（オン）の場合、測定サイクル自体がSyncメソッドで実行され、測定結果が_distフィールドに記録されます。この場合、getDistを呼び出すと、メソッドは_distに記録された値をすぐに返します。測定サイクルはありません。 _isOn == FALSE（オフ）の場合、逆に、測定サイクルはgetDistの呼び出し中にのみ実行され、Syncメソッドでは何も測定されません。プログラマーがロボット全体のSyncメソッドを呼び出し、ロボットがシャシークラス（シャシー）のオブジェクトに接続されているすべてのデバイスと同じ名前のSyncメソッドを呼び出すことが想定されています。

APIのデバイスのうち、MIROが備えているものだけが実装されています。LED、超音波距離計、フォトレジスタンスライトセンサー、サーボドライブ、ラインセンサーです。
