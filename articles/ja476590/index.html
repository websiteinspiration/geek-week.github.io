VueJS + NodeJS + MongoDBアプリケーションをDockerにパックする方法


前回の記事からわかるように、私はさまざまなプロジェクトで作業しました。新しいチームの最初の日は通常同じように進みます。バックエンドは私に座って、アプリケーションをインストールしてデプロイする魔法のアクションを実行します。Dockerは、フロントエンドに不可欠です。バックエンドは多くの場合、幅広いPHP / Java / Python / Cスタックで記述されており、フロントがすべてをインストールしてデプロイするたびにバックエンドの邪魔をする必要はありません。自己テストによってねじ込まれた、透過的なデプロイ、ログを備えたDocker-Jenkinsの束が1か所だけにありました。

dockerに関する詳細な記事がたくさん書かれています。この記事では、VueJS / Vue Routerを使用したシングルページアプリケーションのデプロイメントについて説明します。サーバーパーツはNodeJSを使用したRESTful APIとして、MongoDBはデータベースとして使用されます。Docker Composeは、いくつかのコンテナーアプリケーションを記述および実行するために使用されます。

ドッカーが必要な理由


Dockerを使用すると、アプリケーションのデプロイプロセスを自動化できます。開発者は、自分のマシン上のバージョンの非互換性に対処するために、自分でプログラムをインストールする必要がなくなりました。Dockerをインストールし、1〜2チームをコンソールに送り込むだけで十分です。Linuxでこれを行うのが最も便利です。

入門


DockerとDocker compose をインストールします。

フォルダ構造。

クライアントアプリケーションとサーバーアプリケーション用に2つのフォルダを作成します。拡張子が.ymlのファイルは、アプリケーションコンテナーが定義およびリンクされるDocker Compose構成です。
Docker-compose.yml：

version: "3"
services:
  mongo:
    container_name: mongo
    hostname: mongo
    image: mongo
    ports:
      - "27017:27017"
  server:
    build: server/
    #command: node ./server.js #   CMD  Dockerfile  /server
    ports:
      - "3000:3000"
    links:
      - mongo
  client:
    build: client/
    #command: http-server ./dist #   CMD  Dockerfile  /client
    network_mode: host
    ports:
      - "8089:8089"
    depends_on:
      - server

dockerで3つのサービスを作成します。NodeJS、MongoDB、およびVueのstaticsです。クライアントをサーバーに接続するには、サーバーに依存するが追加されます。リンクmongoは、MongoDBをサーバーAPIにリンクするために使用されます。サーバー、クライアント、mongo-サービス名。

VueJS上のクライアント


/ client フォルダーには、VueJS上のアプリケーションが含まれています。Vue Cliを使用して作成されたアプリケーション。イメージをビルドすると、クライアントアプリケーションは/ distフォルダー内の一連の静的ファイルにビルドされます。Dockerfileは、イメージを構築するための一連のコマンドについて説明しています。

FROM node:10
WORKDIR /client
COPY ./package*.json ./
RUN npm install
RUN npm install -g http-server
COPY . .
RUN npm run build
EXPOSE 8081
CMD ["npm", "test:dev"]

package.jsonは、他のプロジェクトファイルとは別にコピーおよびインストールされることに注意してください。これはパフォーマンスのために行われるため、ビルドが繰り返されるときに/ node_modulesフォルダーの内容がキャッシュされます。各コマンドラインは個別にキャッシュされます。

最後に、コンテナが起動すると、コマンドが実行されnpm run devます。このコマンドは、package.jsonで説明されています。


"scripts": {
	 "test:dev": "http-server dist -p 8081 -c 1 --push-state"
}

/ dist フォルダーからファイルを実行するには、Vueルーターが正しく機能するように、それをグローバルhttp-serverにdev-dependenciesパッケージにインストールspa-http-serverします。–push-stateフラグは、index.htmlにリダイレクトします。1秒の値を持つ-cフラグが追加され、httpサーバーがスクリプトをキャッシュしないようにしています。これはテストケースです。実際のプロジェクトでは、nginxを使用することをお勧めします。

VuexストアapiHost: 'http://localhost:3000'で、NodeJS Apiポートが登録されるフィールドを作成します。クライアント部分は準備ができています。これで、クライアントからバックへのすべての要求がこのURLに送られます。

結論


次に、プロジェクトディレクトリに移動して実行し、docker-compose buildイメージをビルドしてdocker-compose upコンテナを起動します。このコマンドは、サーバー、クライアント、mongoの 3つのコンテナーを生成します。NodeJS上のサーバーの場合は、ユーザーフォルダーに関連付けることでホットリロードを構成できます。また、クライアントは、サーバーとmongoを個別に実行して、ホットリロードでローカルに実行するように開発中です。別のサービスを開始するには、その名前を指定するだけdocker-compose up clientです。pruneリソースを解放するために、コンテナー（コンテナー）、ネットワーク（ネットワーク）、およびイメージ（画像）を削除することを忘れないでください。
