▍ヒープダンプ


ヒープダンプとは何かを理解するには、まずヒープの概念の意味を理解する必要があります。この概念をできるだけ簡単に説明すると、ヒープは、メモリが割り当てられるすべての要素が含まれる場所であることがわかります。これはすべて、ガベージコレクターが不要であると見なされるすべてをヒープから削除するまで、ヒープ上にあります。ヒープダンプは、ヒープの現在の状態のスナップショットです。ダンプには、すべての内部変数とプログラマが宣言した変数が含まれています。これは、ダンプの受信時にヒープに割り当てられたすべてのメモリを表します。

その結果、起動したばかりのサーバーのヒープダンプと、長時間実行されていてメモリがオーバーフローしているサーバーヒープのダンプをどうにか比較できれば、アプリケーションには不要であるが、ガベージコレクターによって削除されていない不審なオブジェクトを特定できます。

会話を続ける前に、ヒープダンプの作成方法について説明しましょう。この問題を解決するために、プログラムでサーバーヒープのダンプを取得できるnpmパッケージheapdumpを使用します。

パッケージをインストールします。

npm i heapdump

このパッケージを使用できるように、サーバーコードにいくつか変更を加えます。

const express = require('express');
const heapdump = require("heapdump");

const app = express();
const port = 3000;

const leaks = [];

app.get('/bloatMyServer', (req, res) => {
  const redundantObj = {
    memory: "leaked",
    joke: "meta"
  };

  [...Array(10000)].map(i => leaks.push(redundantObj));

  res.status(200).send({size: leaks.length})
});

app.get('/heapdump', (req, res) => {
  heapdump.writeSnapshot(`heapDump-${Date.now()}.heapsnapshot`, (err, filename) => {
    console.log("Heap dump of a bloated server written to", filename);

    res.status(200).send({msg: "successfully took a heap dump"})
  });
});

app.listen(port, () => {
  heapdump.writeSnapshot(`heapDumpAtServerStart.heapsnapshot`, (err, filename) => {
    console.log("Heap dump of a fresh server written to", filename);
  });
});

ここでは、このパッケージを使用して、新しく起動したサーバーをダンプします。/heapdumpアクセス時にヒープを作成するように設計されたAPIも作成しました。サーバーが大量のメモリを消費し始めたことに気づいた時点で、このAPIを使用します。

サーバーがKubernetesクラスターで実行されている場合、余分な労力がなければ、サーバーを実行しているメモリーが多すぎるポッドに頼ることはできません。これを行うには、ポート転送を使用できます。さらに、ダンプファイルをダウンロードするために必要なファイルシステムにアクセスできないため、これらのファイルを外部クラウドストレージ（S3など）にアップロードすることをお勧めします。
