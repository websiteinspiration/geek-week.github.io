テンシ


2つのプログラムの実行時間を見積もるツールです。実際、2つのアプリケーションの実行時間を比較することができます。ユーティリティは、2016年に学部生の一環として開発したドイツの学生Johannes Bechbergerによって作成されました。現在、このツールは GNU General Public Licenseの下でライセンスされています。

ヨハネスは、制御された環境でコンピューティングシステムのパフォーマンスを測定するツールを作成したいと考えていました。したがって、temciの主な機能の1つは、テスト環境を構成する機能です。たとえば、次のことができます。CPU周波数マネージャーの設定を変更し、ハイパースレッディングを無効にするL1およびL2キャッシュ、Intelプロセッサなどでターボモードをオフにします。Temciは、ベンチマークに時間、perf_stat、およびgetrusageツールを使用します。

これは、最初のケースでユーティリティがどのように機能するかです。

ベンチマークの結果に基づいて、システムは図、表、およびグラフを含む便利なレポートを生成します。これにより、Tempiを類似のソリューションと区別できます。

temciの欠点のうち、彼の「若さ」が際立っています。このため、すべてのハードウェアおよびソフトウェア構成をサポートしているわけではありません。たとえば、macOSで実行するのは難しく、一部の機能はARMプロセッサを搭載したシステムでは使用できません。著者がプロジェクトを積極的に開発していて、GitHubのスターの数が徐々に増えているため、将来は状況が変わる可能性があります。TemciはHacker Newsのコメントでさえ議論されていません。


uarch-bench


エンジニアのTravis Downs（Travis Downs）が開発した、低レベルのCPU機能のパフォーマンスを評価するためのユーティリティ。最近、彼はパフォーマンスマーターズのブログをGitHub Pagesで実行しており、ベンチマークツールやその他の関連事項について説明しています。一般に、uarch-benchは人気を集め始めたばかりですが、Hacker Newsの居住者がテーマ別スレッドでベンチマークの頼りになるツールとしてすでにかなり頻繁に言及しています。

Uarch-benchを使用すると、メモリパフォーマンス、並列データロードの速度を評価し、YMMレジスタのクリーニングに取り組むことができます。プログラムによって生成されたベンチマーク結果がどのように見えるかは、ページの下部にある公式リポジトリにあります。

temciのようなuarch-bench はIntel Turbo Boost機能を無効にします（負荷がかかると自動的にプロセッサーのクロック速度が上がります）ので、テスト結果は一貫しています。

これまでのところ、プロジェクトは開発の初期段階にあるため、uarch-benchには詳細なドキュメントがありません。たとえば、Ryzenでの実行に問題があることがわかっているなど、作業にバグがある可能性があります。x86アーキテクチャのベンチマークのみもサポートされています。著者は将来さらに機能を追加することを約束し、開発に参加することを勧めます。


リキッド


これは、Intel、AMD、およびARMv8プロセッサを搭載したLinuxマシンのパフォーマンスを評価するためのツールキットです。これは、2017年にドイツ連邦教育研究省の後援の下で作成され、オープンソースに移行されました。

likwidツールの中で、システムによって消費された電力に関するRAPLレジスターからの情報を表示するlikwid-powermeterとlikwid-setFrequenciesを区別できます。これにより、プロセッサー周波数を制御できます。完全なリストはリポジトリにあります。

このツールは、HPCの分野で研究に従事しているエンジニアによって使用されます。たとえば、likwidで動作しますドイツのエアランゲンニュルンベルク大学（RRZE）の地域コンピューティングセンターの専門家チーム。彼女はこのツールセットの開発に積極的に関与しています。

perf-tools


Linuxサーバーのパフォーマンスを分析するこのツールは、 Brendan Gregg によって導入されました。彼は、アプリケーションをリアルタイムでデバッグするための動的トレースフレームワークであるDTraceの開発者の1人です。

perf-toolsは、カーネルサブシステムperf_eventsおよびftraceに基づいています。これらのユーティリティを使用すると、入出力遅延（iosnoop）を分析し、システムコール（unccount、funcslower、funcgraph、functrace）にアクセスするための引数を追跡し、ファイルキャッシュ（cachestat）の「ヒット」に関する統計を収集できます。後者の場合、コマンドは次のようになります。

# ./cachestat -t
Counting cache functions... Output every 1 seconds.
TIME HITS MISSES DIRTIES RATIO BUFFERS_MB CACHE_MB
08:28:57 415 0 0 100.0% 1 191
08:28:58 411 0 0 100.0% 1 191
08:28:59 362 97 0 78.9% 0 8
08:29:00 411 0 0 100.0% 0 9

ツールを中心にかなり大きなコミュニティが形成されました（GitHubの約6,000の星）。また、Netflixなどのperf-toolsを積極的に使用している企業もあります。しかし、このツールは最終化され、修正されています（更新は最近ごくまれにリリースされていますが）。したがって、彼の作業でエラーが発生する可能性があります-著者は、perf-toolsがカーネルパニックを呼び出すことがあると書いています。


llvm-mca


さまざまなCPUでマシンコードが必要とするコンピューティングリソースの量を予測するユーティリティ。これは、評価サイクル当たりの命令（IPCアプリケーションが生成する）とハードウェア負荷。

llvm-mcaは、プログラムの分析、変換、最適化のためのユニバーサルシステムを開発しているLLVMプロジェクトの一部として2018年に発表されました。 llvm-mcaの作者は、インテルIACAソフトウェアパフォーマンス分析ソリューションに触発され、それに代わるものを作ろうとしたことが知られています。ユーザーによると、ツールの出力（マークアップと番号）は実際にはIACAに似ています-例はここにあります。ただし、llvm-mcaはAT＆T構文のみを受け入れるため、コンバーターを使用して作業する必要がある可能性があります。
