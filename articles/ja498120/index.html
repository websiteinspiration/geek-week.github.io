ビルド時間の最適化-パート1

ほとんどすべての開発者が、少なくとも一度はプロジェクトのビルドにかなり長い時間を費やしていました。これにより、生産性が低下し、チーム全体の開発プロセスが遅くなります。ご覧のとおり、プロジェクトのビルド時間を増やすことは、アプリケーションがAppStoreに公開される時間に直接影響し、アプリケーションの新機能をより早くリリースするために重要です。

この記事では、Xcodeでアセンブリのプロファイルを作成し、メトリックを取得する方法を学びます。次の記事では、ボトルネックを解消し、プロジェクトの組み立てを高速化する方法について説明します。また、GithubにあるKickstarter iOSプロジェクトを使用することにも言及する必要があります。それでは始めましょう！

私たちは何を測定していますか？


最初にすべきことは、何を測定して最適化しようとしているのかを判断することです。次の2つのオプションを検討できます。

  • クリーンビルド -プロジェクトをクリーンしてビルドします。多くの場合、クリーンビルドはCIで行われ、新しいプルリクエストを確認してユニットテストを実行します。
  • インクリメンタルビルド -ソースコードに大幅な変更を加えた後、プロジェクトをビルドします。このアセンブリは、新しい機能の開発中に開発者が作成します。

ほとんどの場合、クリーンビルド中の時間を短縮すると、増分ビルドも高速化されます。最良のオプションは、両方のタイプのアセンブリのメトリックを作成し、それらを追跡することです。ほとんどの場合に使用され、開発により大きな影響を与えるため、デバッグ構成を使用してビルド時間を測定します。

組立時間を計算します


ビルド時間を改善する最も効果的な方法は、特定のメトリックに基づいて変更を実装および検証することに基づくアプローチである必要があります。それらと、プロジェクトのビルド時間に関する情報を取得するために使用できるツールについて検討してください。

Xcodeビルド時間レポート


Xcodeを使用してビルド時間データを簡単に取得できます。デフォルトですべてのビルドを追跡し、レポートナビゲーターでプロジェクトのビルド時間を表示できます。



また、アクティビティビューアパネルのXcodeに同様の情報を表示するオプションもあります。次のコマンドを実行することにより、コマンドラインを使用して表示を有効にできます 。プロジェクトのビルド時間は、アセンブリの後に「成功しました」というメッセージとともに表示されます。 これらは、クリーンビルドとインクリメンタルビルドの時間の大まかな考え方を示す2つの基本的なオプションにすぎません。

defaults write com.apple.dt.Xcode ShowBuildOperationDuration YES







Xcodeでのプロジェクトのビルド時間の概要


Xcodeビルド時間の概要-ビルド時間の統計とボトルネックを取得する最初の友達。Product-> Perform Action-> Build With Timing Summaryから実行できます。これで、さまざまなタスクに費やされた時間の大きな内訳が表示されます。



この方法は、アセンブリプロセスで最も時間がかかるタスクを見つけるための良い出発点になるはずです。上のスクリーンショットは、CompileStoryboard、CompileXIB、CompileSwiftSources、およびPhaseScriptExecutionフェーズがビルド時間のほとんどを占めたことを示しています。 Xcodeはタスクの一部を並行して完了することができたため、各コマンドのタスクを完了するために必要な時間よりもはるかに速くアセンブリが完了しました。

私たちは、のビルド時間の概要を取得することができますクリーン使用してビルドをxcodebuildをオプションで- buildWithTimingSummary： それでは、インクリメンタルビルドの同じメトリックを取得してみましょう。インクリメンタルビルド時間は、プロジェクトで変更されるファイルに完全に依存することに注意してください。一貫した結果を得るには、1つのファイルを変更してプロジェクトをコンパイルします。BuckまたはBazelシステムとは異なり、Xcodeはタイムスタンプを使用して、何が変更され、何を再構築する必要があるかを決定します。touchを使用してタイムスタンプを更新できます：

xcodebuild -project 'Kickstarter.xcodeproj' \
-scheme 'Kickstarter-iOS' \
-configuration 'Debug' \
-sdk 'iphonesimulator' \
-showBuildTimingSummary \
clean build | sed -n -e '/Build Timing Summary/,$p'

タイプ検証警告


コンパイル時間がボトルネックになっている場合、XcodeのプロジェクトビルドオプションにOther Swift Flagsを設定することで、より多くの情報を取得できます。フラグが有効になっている場合、Xcodeは、変数のタイプを判別するために100ミリ秒以上かかる関数または式に対して警告を生成します。

  • -Xfrontend -warn-long-function-bodies = 100
  • -Xfrontend -warn-long-expression-type-checking = 100



これで、Swiftコンパイラーに問題があるコードがわかったので、それを改善してみることができます。

コンパイラ診断オプション


Swiftコンパイラーには、アセンブリーのプロファイル作成に使用できる多くの組み込み診断オプションがあります。

  • -driver-time-compilation-ドライバーによって実行されるタスクの高レベルの同期。
  • -Xfrontend -debug-time-compilation — .
  • -Xfrontend -debug-time-function-bodies – .
  • -Xfrontend -debug-time-expression-type-checking – .

-debug-time-compilation フラグを使用して、コンパイル時に最も遅いファイルに関する情報を取得してみましょう。 ご覧のとおり、SettingsNewslettersCellViewModel.swiftをコンパイルするのに25秒かかりました。アセンブリログから、ファイルのコンパイル時間に関する追加情報を取得できます。 これで、型チェックとセマンティック分析が最も時間のかかる作業であることは明らかです。先に進み、型チェックのステップで最も遅い関数と式をリストしてみましょう。

xcodebuild -project 'Kickstarter.xcodeproj' \
-scheme 'Kickstarter-iOS' \
-configuration 'Debug' \
-sdk 'iphonesimulator' \
clean build \
OTHER_SWIFT_FLAGS="-Xfrontend -debug-time-compilation" |
awk '/CompileSwift normal/,/Swift compilation/{print; getline; print; getline; print}' |
grep -Eo "^CompileSwift.+\.swift|\d+\.\d+ seconds" |
sed -e 'N;s/\(.*\)\n\(.*\)/\2 \1/' |
sed -e "s|CompileSwift normal x86_64 $(pwd)/||" |
sort -rn |

目標ビルド時間


ターゲットの組み立て時間を個別に測定し、グラフに表示することをお勧めします。どのターゲットが収集されているか、または並行して収集できるかを理解するのに役立ちます。これを行うには、xcode-build-time-renderingツールを使用できます。RubyGemとして設定しましょう： インストールが完了したら、次のコマンドを実行して、ターゲットのスクリプト作成ビルドフェーズにタイムスタンプログを入力します。 次に、プロジェクトをコンパイルし、次を使用してレポートを作成し ます。すべてのターゲットのアセンブリ：

gem install xcode-build-times



xcode-build-times install



xcode-build-times generate




集約されたメトリック


上記のさまざまな指標を集計することは素晴らしいことです。XCLogParserは、これに役立つ優れたツールです。これは、Xcodeによって生成されたxcactivitylogのログアナライザーであり、プロジェクト内の各モジュールとファイルのビルド時間、警告、エラー、ユニットテストの結果に関する多くの情報を提供します。リポジトリのクローンを作成し、コマンドラインから起動することでインストールできます。 これは、Kickstarter iOSプロジェクト用に作成されたレポートです。

git clone https://github.com/spotify/XCLogParser
rake build
xclogparser parse --project Kickstarter --reporter html




オートメーション


アセンブリ時間のメトリックは、ハードウェアとその使用方法に依存することに注意してください。実験に使用できます。最良のオプションは、プロセスを可能な限り自動化し、専用のCI機器を毎日の指標に使用することです。最終的には、ダッシュボードで監視し、Slackを使用してビルド時間の悪化を通知できます。
