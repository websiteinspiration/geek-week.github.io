テレグラム+ 1C + Webhook + Apache +自己署名証明書

Telegramと1Cの統合については、多くの行が記述されています。しかし、Webhookのインストールと構成に関する完全な説明はどこにもありませんでした。書いてみます。

これにはすべて必要です（または、私が使用したものを言う方が正しいでしょう）。

  1. Apache 2.2.24
  2. OpenSSL（Apacheインストールに含まれています）
  3. 1C（Webサーバーモジュール付き）
  4. 自分のドメイン
  5. Telegramで作成されたボット（非常に簡単なので、その作成については説明しません）

インストールしたすべてのソフトウェアを前提としています。

それでは、証明書を取得することから始めましょう。コマンドラインを開き、次のコードを実行します。

openssl req -newkey rsa:2048 -sha256 -nodes -keyout YOURPRIVATE.key -x509 -days 365 -out YOURPUBLIC.pem -subj "/C=US/ST=New York/L=Brooklyn/O=Example Brooklyn Company/CN=YOURDOMAIN.EXAMPLE"

どこ：

YOURPRIVATE.keyは、証明書の秘密鍵です。これは、

Apache YOURPUBLIC.pem-証明書の公開鍵で使用されます。Webhook

YOURDOMAIN.EXAMPLE- Webhookを使用したドメインのアドレスを登録するときに使用されます。それでもWebhookアドレスと一致する必要があります!!!

このコードを実行すると、鍵ファイルがopensslフォルダーに表示されます（「C：\ Program Files \ Apache Software Foundation \ Apache2.2 \ bin」があります）。

それらをApacheのconfフォルダーにコピーしました。

Apacheの設定に移りましょう。

私はさまざまな方法を見てきました。以下は私にとってうまく

いきました：次の行がhttpd.confに追加されました：

Listen 443

Apacheがポート443を「リッスン」するようにします。

<IfModule ssl_module>ブロックの形式は次のとおりです。

<IfModule ssl_module>
SSLMutex default
SSLSessionCache none
</IfModule>

最後に、証明書へのパスを示す行が追加されます。

SSLEngine On
SSLCertificateFile conf/YOURPUBLIC.pem
SSLCertificateKeyFile conf/YOURPRIVATE.key

そして、行のコメントを外します： 

LoadModule ssl_module modules/mod_ssl.so

構成では、HTTPサービスを作成します。電報に答えて上訴を処理するのは彼です。

データベースを公開してWebhookをアタッチすることは残ります。

公開は通常どおり行われ、チェックマークをHTTPサービスの公開に追加するだけです。



最後のステップとして、1Cを電報に添付します。このために、私は次のコードで単純なhtmlページを使用しました：

フォームで、公開鍵を選択し、httpサービスへの完全パスを入力するだけです。サービスへのフルパスは次のようになることを思い出します：

YourDomain / BaseName / hs / ServiceName / v1

コードをキックしないでください。いくつかはハードコードによって意図的に行われます。これらはすべて、デモンストレーションのみを目的として行われました。
