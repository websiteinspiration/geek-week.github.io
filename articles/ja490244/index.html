基本的な最適化


CでPythonソースコードを書き直す前に、Pythonの基本的な最適化方法を検討してください。

組み込みのデータ構造


setやdictなどのPythonの組み込みデータ構造はCで記述されています。Pythonクラスとして記述された独自のデータ構造よりもはるかに高速に動作します。標準セット、辞書、リスト、タプル以外のデータ構造については、コレクションモジュールのドキュメントで説明しています。

リスト式


標準の方法を使用してリストにアイテムを追加する代わりに、リスト式を使用します。 

# Slow
      mapped = []
      for value in originallist:
          mapped.append(myfunc(value))
      
      # Faster
      mapped = [myfunc(value) in originallist]

ctypes


ctypes モジュールを使用するとsubprocess、CLIから他のプロセスを開始するためにモジュールまたは他の同様のモジュールを使用せずに、PythonからCコードを操作できます。

2つの部分のみがあります。Cコードをコンパイルして品質をロードし、shared objectデータ構造をPythonコードに設定してタイプCにマップします。

この記事では、Pythonコードをlcs.cに接続します。これにより、2つの中で最も長いサブシーケンスが見つかります。文字列のリスト。以下をPythonで機能させたい：

list1 = ['My', 'name', 'is', 'Sam', 'Stevens', '!']
      list2 = ['My', 'name', 'is', 'Alex', 'Stevens', '.']
      
      common = lcs(list1, list2)
      
      print(common)
      # ['My', 'name', 'is', 'Stevens']

1つの問題は、この特定のC関数が文字列のリストを引数の型として受け取り、固定長ではない型を返す関数のシグネチャであることです。この問題は、ポインターと長さを含むシーケンス構造で解決します。

PythonでのCコードのコンパイル


最初に、Cソースコード（lcs.c）をコンパイルしlcs.soてPythonにロードします。

gcc -c -Wall -Werror -fpic -O3 lcs.c -o lcs.o
      gcc -shared -o lcs.so lcs.o

  • - 壁にはすべての警告が表示されます。
  • - Werrorはなりエラーのすべての警告を包みます。
  • - FPICはなりますが、Pythonでこのライブラリを使用する場合、必要となることを、位置に依存しない命令を生成します。
  • - O3は、最適化を最大化します。

次に、結果の共有オブジェクトファイルを使用してPythonコードの記述を開始します。

Pythonの構造


以下は、Cコードで使用される2つのデータ構造です。

struct Sequence
      {
          char **items;
          int length;
      };
      
      struct Cell
      {
          int index;
          int length;
          struct Cell *prev;
      };

そして、これらの構造のPythonへの変換を次に示します。

import ctypes
      class SEQUENCE(ctypes.Structure):
          _fields_ = [('items', ctypes.POINTER(ctypes.c_char_p)),
                      ('length', ctypes.c_int)]
      
      class CELL(ctypes.Structure):
          pass
      
      CELL._fields_ = [('index', ctypes.c_int), ('length', ctypes.c_int),
                       ('prev', ctypes.POINTER(CELL))]

いくつかのメモ：

  • すべての構造は、から継承するクラスですctypes.Structure。
  • 唯一のフィールド_fields_はタプルのリストです。各タプルは（<variable-name>、<ctypes.TYPE>）です。
  • c_char（char）とc_char_p（* char）にctypesも同様のタイプがあります。
  • 渡された各型から型ポインターを作成するctypesものもありPOINTER()ます。
  • のような再帰的な定義がある場合はCELL、最初の宣言を渡して、_fields_後で自分へのリンクを取得するためにフィールドを追加する必要があります。
  • 私はCELLコードでPythonを使用しなかったので、この構造を記述する必要はありませんでしたが、再帰フィールドには興味深いプロパティがあります。

Cでコードを呼び出す


さらに、Pythonの型をCで新しい構造に変換するためのコードが必要でした。これで、新しいC関数を使用してPythonコードを高速化できます。 

def list_to_SEQUENCE(strlist: List[str]) -> SEQUENCE:
          bytelist = [bytes(s, 'utf-8') for s in strlist]
          arr = (ctypes.c_char_p * len(bytelist))()
          arr[:] = bytelist
          return SEQUENCE(arr, len(bytelist))
      
      
      def lcs(s1: List[str], s2: List[str]) -> List[str]:
          seq1 = list_to_SEQUENCE(s1)
          seq2 = list_to_SEQUENCE(s2)
      
          # struct Sequence *lcs(struct Sequence *s1, struct Sequence *s2)
          common = lcsmodule.lcs(ctypes.byref(seq1), ctypes.byref(seq2))[0]
      
          ret = []
      
          for i in range(common.length):
              ret.append(common.items[i].decode('utf-8'))
          lcsmodule.freeSequence(common)
      
          return ret
      
      lcsmodule = ctypes.cdll.LoadLibrary('lcsmodule/lcs.so')
      lcsmodule.lcs.restype = ctypes.POINTER(SEQUENCE)
      
      list1 = ['My', 'name', 'is', 'Sam', 'Stevens', '!']
      list2 = ['My', 'name', 'is', 'Alex', 'Stevens', '.']
      
      common = lcs(list1, list2)
      
      print(common)
      # ['My', 'name', 'is', 'Stevens']

いくつかのメモ：

  • **char （文字列のリスト）Pythonのバイトのリストに直接一致します。
  • シグネチャstruct Sequence * lcs（struct Sequence * s1、struct Sequence * s2）をlcs.c持つ関数があります。戻り値の型を設定するには、を使用します。lcs()lcsmodule.lcs.restype = ctypes.POINTER(SEQUENCE)
  • シーケンス構造への参照を使用して呼び出しを行うには ctypes.byref()、オブジェクトへの「ライトポインター」を返すものを使用します（より高速ctypes.POINTER()）。
  • common.items-これはバイトのリストであり、デコードしretてリストの形式にすることができますstr。
  • lcsmodule.freeSequence（common）は、commonに関連付けられたメモリを解放するだけです。ガーベッジ・コレクター（AFAIK）は自動的に収集しないため、これは重要です。

最適化されたPythonコードは、Cで記述してPythonでラップしたコードです。
