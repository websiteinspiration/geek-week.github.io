私たちは、JavaScriptクラスとしてCookieを使用します

みなさん、こんにちは！今日私は非常に人気のある質問を検討したいと思います：クッキーをどのように扱うか？非常に異なる方法がどこでも提供されていますが、どちらが優れていますか？この質問に対する明確な答えはありませんが、非常に便利な方法を提供したいと思います。興味があれば、カーストへようこそ。

つまり、要点は


get_cookie、set_cookie、およびdelete_cookieメソッドを作成します。

コード
var _cookie = {
        get_cookie:function(cookie_name){
         var r=document.cookie.match('(^|;) ?'+cookie_name+'=([^;]*)(;|$)');
         return r?unescape(r[2]):null
    },
    delete_cookie:function(cookie_name){
        var c=new Date();
        c.setTime(c.getTime()-1);
        document.cookie=cookie_name+="=; expires="+c.toGMTString()
    },
    set_cookie:function(name,value,exp_y,exp_m,exp_d,path,domain,secure){
        var c=name+"="+escape(value);
        if(exp_y){
            var expires=new Date(exp_y,exp_m,exp_d);
            c+="; expires="+expires.toGMTString()
       }
       if(path)c+="; path="+escape(path);
       if(domain)c+="; domain="+escape(domain);
       if(secure)c+="; secure";
       document.cookie=c;
        }
}

さて、これらの方法を確認しており、原則としてこれで終わりです。しかし、これは私たちが目指していることではありません。プロパティc＃のようなことをしたいと思います。

このため、JavaScriptには非常に便利なプロキシクラスがあります。

つまり、プロキシは別のオブジェクトを「ラップアラウンド」し、プロパティの読み取り/書き込みなど、さまざまなアクションを傍受（必要に応じて、独立して処理）できます。

プロキシメソッドを使用するには、2つのラムダ関数を作成します。

コード
var __methods = {
   get:(t,m)=>m in t?t[m]:_cookie.get_cookie(m),
    set:(t,m,v)=>v?_cookie.set_cookie(m,v):_cookie.delete_cookie(m)
}


ここで、Proxyクラス自体を作成します。最初のパラメーターはクラスメソッドのリストであり、_cookieに設定しますが、Cookieを操作するメソッドが必要ない場合は、{}を指定します。2番目のパラメーターは、作成したばかりの__methodsです。

すばらしい、今は単純な行を書くだけです。

var cookie = new Proxy(_cookie,__methods);

フーレイ！__methods .getメソッドが呼び出されるプロパティを呼び出し（get）してクラスを作成し、それに値を割り当てるときに__methods.setを作成しました。

使用例：

コード
var name= prompt("  ","my_name");
var value= parseFloat(prompt("   ( float )","103"));
cookie[name] = value;
if(value<100) cookie[name]+=10;
else cookie[name]-=10;
//  
alert(",  "+name+" = "+value+"   !"+"\n  :\n"+document.cookie);
