GitHub認証情報を使用してAzure DevOpsにログインする

マイクロソフトでは、開発者が高品質のアプリケーションを迅速に作成する機会を提供するという考えに焦点を当てています。これを実現する1つの方法は、ソフトウェア開発ライフサイクルのすべての段階をカバーする幅広い製品とサービスを提供することです。これには、IDEおよびDevOpsツール、クラウドベースのアプリケーションとデータプラットフォーム、オペレーティングシステム、人工知能、IoTソリューションなどが含まれます。それらのすべては、チームや組織で働く個人、および開発者コミュニティのメンバーとして、開発者に集中しています。

GitHubは最大の開発者コミュニティの1つであり、世界中の何百万人もの開発者にとって、GitHubでの識別はデジタルライフの重要な側面となっています。これを認識して、GitHubユーザーがAzure DevOpsやAzure などの開発者向けサービスを使い始めるのに役立つ改善点を発表できることをうれしく思います。


GitHubの認証情報を使用して、Microsoftサービスにアクセスできるようになりました。


これで、開発者は、任意のMicrosoftログインページで既存のGitHubアカウントを使用してMicrosoftオンラインサービスにログインすることができます。GitHub認証情報を使用して、Azure DevOpsやAzureなどのMicrosoftサービスにOAuth経由でログインできるようになりました。

[GitHubでログイン]をクリックすると、アカウントにログインするオプションが表示されます。

GitHubを介してログインし、Microsoftアプリケーションを承認すると、GitHub資格情報に関連付けられた新しいMicrosoftアカウントを受け取ります。このプロセス中に、既存のMicrosoftアカウントがある場合は、それを既存のMicrosoftアカウントにリンクすることもできます。

Azure DevOpsにサインインする


Azure DevOpsは、開発者がアプリケーションを計画、作成、送信するのに役立つ一連のサービスを提供します。また、GitHub認証サポートにより、継続的インテグレーションや継続的デリバリー（Azureパイプライン）などのAzure DevOpsサービスの操作を簡素化することができました。アジャイル計画（Azure Boards）; NuGet、npm、PyPiなどのプライベートパッケージの格納（Azureアーティファクト）。 Azure DevOpsは、個人および最大5人の小さなグループに対して無料です。

GitHubアカウントを使用してAzure DevOpsを開始するには、Azure DevOpsページの[GitHubを使用して無料で開始]をクリックします。



ログインプロセスが完了すると、Azure DevOpsで最後にアクセスした組織に直接移動します。Azure DevOpsを初めて使用する場合は、作成された新しい組織に移動します。

すべてのMicrosoft Online Servicesにアクセスする


Azure DevOpsやAzureなどの開発者サービスにアクセスするだけでなく、GitHubアカウントを使用して、Excel OnlineからXboxまでのすべてのMicrosoftオンラインサービスにアクセスできます。

これらのサービスで認証する場合、[サインインオプション]をクリックした後でGitHubアカウントを選択できます。



お客様のプライバシーへの取り組み。


最初にGitHubアカウントを使用してMicrosoftサービスにアクセスすると、GitHubはあなたのプロファイル情報を使用する許可を求めます。

同意すると、GitHubはGitHubアカウント（パブリックとプライベートの両方）のメールアドレスと、名前などのプロファイル情報を提供します。このデータを使用して、システムにアカウントがあるかどうか、またはアカウントがない場合に新しいアカウントを作成する必要があるかどうかを判断します。 GitHub IDをMicrosoftに接続しても、GitHub上のリポジトリにMicrosoftがアクセスすることはできません。 Azure DevOpsやVisual Studioなどのアプリケーションは、コードを操作する必要がある場合、リポジトリへのアクセスを個別に要求します。そのため、ユーザーは個別に同意する必要があります。

GitHubアカウントはMicrosoftアカウントへのログインに使用されますが、それらはまだ分離されたままです。一方は単にもう一方をログイン方法として使用します。GitHubアカウントに加えた変更（たとえば、パスワードの変更や2要素認証の有効化）は、Microsoftアカウントを変更しません。その逆も同様です。[セキュリティ]タブのアカウント管理ページで、GitHubとMicrosoft ID間の接続を管理できます。

今すぐAzure DevOpsの学習を始めましょう！


Azure DevOpsページに移動し、「GitHubで無料で開始」をクリックして開始します。
