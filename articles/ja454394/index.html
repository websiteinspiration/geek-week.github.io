ミニマリスト4パートMIDIプレーヤー



提案されたプレーヤーはメモリカードを必要としません。最大6,000バイトのMIDIファイルをATtiny85マイクロコントローラーに直接保存します（WAVファイルを再生するこのクラシックなデザインとは異なり、当然ながらメモリカードが必要です）。PWMを使用した減衰による4ウェイ再生は、ソフトウェアで実装されます。音の例はこちらです。

デバイスはスキームに従って作成され



ます。ソフトウェア障害の結果として論理ユニットがPB4の出力に現れた場合、マイクロコントローラーとダイナミックヘッドの間の電解コンデンサーは定数コンポーネントを見逃しません。ヘッドのインダクタンスはPWM周波数を通過しません。デバイスをアンプに接続する場合、PWM信号によるアンプの過負荷を回避するために、ここに示すようにローパスフィルターを追加する必要があります。

MIDIファイルは、次の形式の配列としてファームウェアソースに配置する必要があります。

const uint8_t Tune[] PROGMEM = {
  0x4d, 0x54, 0x68, 0x64, 0x00, 0x00, 0x00, 0x06, 0x00, 0x01, 0x00, 0x01,
  0x03, 0xc0, 0x4d, 0x54, 0x72, 0x6b, 0x00, 0x00, 0x0a, 0x7e, 0x00, 0xff,
  ...
  0x50, 0xb0, 0x5b, 0x00, 0x00, 0xff, 0x2f, 0x00
};

UNIXのようなオペレーティングシステムでファイルをこの形式に変換するための既製のソリューション-xxdユーティリティがあります。MIDIファイルを取得し、このユーティリティを次のように渡します。

xxd -i musicbox.mid

コンソールには次のように表示されます。

unsigned char musicbox_mid[] = {

  ...
  0x50, 0xb0, 0x5b, 0x00, 0x00, 0xff, 0x2f, 0x00

unsigned int musicbox_mid_len = 2708;

2708はバイト単位の長さです。それは6000未満であることが判明しました。クリップボードを介した16進数のシーケンスは、デフォルトの配列ではなく、スケッチに転送されます（覚えておいてください。コンソールでは、Ctrl + Cは不要です）。または、それを残したい場合は、これをすべて行っていません。

タイマーカウンター1は、PLLから64 MHzの周波数で動作します。

  PLLCSR = 1<<PCKE | 1<<PLLE;

このタイマーをPWMモードに転送してDACとして機能させます。デューティサイクルはOCR1Bの値によって異なります。

 TIMSK = 0;                     // Timer interrupts OFF
  TCCR1 = 1<<CS10;               // 1:1 prescale
  GTCCR = 1<<PWM1B | 2<<COM1B0;  // PWM B, clear on match
  OCR1B = 128;
  DDRB = 1<<DDB4;                // Enable PWM output on pin 4

矩形パルスの周波数はOCR1Cの値に依存します。これを255（デフォルト）のままにしておくと、64 MHzの周波数が256で除算され、250 kHzになります。

タイマーカウンター0は割り込みを生成します：

  TCCR0A = 3<<WGM00;             // Fast PWM
  TCCR0B = 1<<WGM02 | 2<<CS00;   // 1/8 prescale
  OCR0A = 19;                    // Divide by 20
  TIMSK = 1<<OCIE0A;             // Enable compare match, disable overflow

16 MHzのクロック周波数は、分周器で8分周され、次にOCR0A値19 + 1で分周され、100 kHzが得られます。プレーヤーは4ボイスで、各ボイスに対して25 kHzが取得されます。中断すると、ISR処理プロシージャが呼び出され（TIMER0_COMPA_vect）、サウンドを計算して出力します。

ウォッチドッグタイマーは、16 msごとに割り込みを生成するように構成されています。これは、ノート周波数を受信するために必要です。

WDTCR = 1<<WDIE | 0<<WDP0;     // Interrupt every 16ms

与えられた形状の振動を得るために、直接デジタル合成が使用されます。 ATtiny85にはハードウェア乗算はないため、矩形パルスを取り、エンベロープの振幅を1または-1で乗算します。振幅は直線的に減少します。特定の瞬間に振幅を計算するには、メーターの読み取り値を直線的に減少させるだけで十分です。

各チャネルには、Freq []-周波数、Acc []-位相バッテリー、Amp []、エンベロープ振幅値の3つの変数が用意されています。 Freq []とAcc []の値が合計されます。上位ビットAcc []は、矩形パルスを取得するために使用されます。 Freq []が多いほど、周波数が高くなります。完成した波形は、エンベロープのアンプ[]で乗算されます。 4つのチャネルはすべて多重化され、アナログ出力に送られます。

プログラムの重要な部分は、発振をアナログ出力に出力するタイマーカウンター0からの割り込みを処理する手順です。この手順は、約95 kHzの周波数で呼び出されます。現在のチャネルcについて、Acc [c]とAmp [c]の値を更新し、現在のノートの値も計算します。結果はOCR1BタイマーカウンターのOCR1B比較レジスターに送信され、ピン4でアナログ信号を取得します。

ISR(TIMER0_COMPA_vect) {
  static uint8_t c;
  signed char Temp, Mask, Env, Note;
  Acc[c] = Acc[c] + Freq[c];  
  Amp[c] = Amp[c] - (Amp[c] != 0);
  Temp = Acc[c] >> 8;
  Temp = Temp & Temp<<1;
  Mask = Temp >> 7;
  Env = Amp[c] >> Volume;
  Note = (Env ^ Mask) + (Mask & 1);
  OCR1B = Note + 128;
  c = (c + 1) & 3;
}

ライン

Acc[c] = Acc[c] + Freq[c];

周波数freq [c]をバッテリーAcc [c]に追加します。Freq [c]が大きいほど、Acc [c]値の変化が速くなります。次にライン

Amp[c] = Amp[c] - (Amp[c] != 0);

特定のチャネルの振幅値を減少させます。フラグメント（Amp [c]！= 0）は、振幅がゼロに達した後、それ以上減少しないようにするために必要です。今行

Temp = Acc[c] >> 8;

Acc [c]の上位9ビットをTempに転送します。そしてライン

Temp = Temp & Temp<<1;

2つの上位ビットが1に等しい場合、この変数の上位ビットは1のままにし、そうでない場合は、上位ビットをゼロに設定します。その結果、オン/オフ比が25/75の矩形パルスになります。以前の構成の1つで、著者は蛇行を適用しました。新しい方法では、高調波がもう少し取得されます。ライン

Mask = Temp >> 7;

最上位の値をバイトの残りのビットに転送します。たとえば、最上位ビットが0の場合、0x00が取得され、1が0xFFの場合。ライン

Env = Amp[c] >> Volume;

Volume = 8であるため、Volume値によって指定されたAmp [c]ビットをEnvに転送します。

Note = (Env ^ Mask) + (Mask & 1);

このすべてが結合します。Mask = 0x00の場合、Noteには値Envが割り当てられます。Mask = 0xFFの場合、NoteにはEnv + 1に追加される値、つまりマイナス記号付きのEnvが割り当てられます。現在、電流波形の正の値から負の値に変化する電流波形が含まれていることに注意してください。ライン

OCR1B = Note + 128;

Noteに128を追加し、結果をOCR1Bに書き込みます。ライン

c = (c + 1) & 3;

対応する割り込みに従って4つのチャネルを出力し、出力で音声を多重化します。

12のノート周波数が配列で与えられます：

unsigned int Scale[] = {
10973, 11626, 12317, 13050, 13826, 14648, 15519, 16442, 17419, 18455, 19552, 20715};

他のオクターブのノート周波数は、2 nで割ることによって得られます。たとえば、10973を2 4で除算して686を取得します。上位ビットAcc [c]は、25000 /（65536/685）= 261.7 Hzの周波数で切り替わります。

2つの変数がサウンドに影響します。ボリューム-7〜9のボリュームと減衰-12〜14の減衰。減衰値が大きいほど、減衰が遅くなります。

最も単純なMIDIインタープリターは、音符、テンポ、および除算係数の値にのみ注意を払い、他のデータを無視します。readIgnore（）ルーチンは、ファイルから受け取った配列内の指定されたバイト数をスキップします。

void readIgnore (int n) {
  Ptr = Ptr + n;
}

readNumber（）ルーチンは、4の精度で指定されたバイト数から数値を読み取ります。

unsigned long readNumber (int n) {
  long result = 0;
  for (int i=0; i<n; i++) result = (result<<8) + pgm_read_byte(&Tune[Ptr++]);
  return result;
}

readVariable（）ルーチンは、MIDI変数の精度で数値を読み取ります。この場合のバイト数は、1〜4にすることができます。

unsigned long readVariable () {
  long result = 0;
  uint8_t b;
  do {
    b = pgm_read_byte(&Tune[Ptr++]);
    result = (result<<7) + (b & 0x7F);
  } while (b & 0x80);
  return result;
}

各バイトから7ビットが取得されます。8バイト目は、別のバイトをさらに読み取る必要がある場合は1、そうでない場合は0です。

インタープリターは、noteOn（）ルーチンを呼び出して、次の使用可能なチャネルでノートを再生します。

void noteOn (uint8_t number) {
  uint8_t octave = number/12;
  uint8_t note = number%12;
  unsigned int freq = Scale[note];
  uint8_t shift = 9-octave;
  Freq[Chan] = freq>>shift;
  Amp[Chan] = 1<<Decay;
  Chan = (Chan + 1) & 3;
}

速度の値は無視しますが、必要に応じて、ノートの初期振幅を設定できます。残りのイベントはスキップします。長さは異なる場合があります。 MIDIファイルでエラーが発生すると、LEDが点灯します。

マイクロコントローラは16 MHzの周波数で動作するので、水晶は必要ありません。組み込みのPLLをそれに応じて構成する必要があります。マイクロコントローラーがArduino互換になるために、Spence Kondeのこの経験が適用されます。ボードメニューで、ATtinyCoreサブメニューを選択し、そこに-ATtiny25 / 45/85。次のメニューで、タイマー1クロック：CPU、BOD無効、ATtiny85、16 MHz（PLL）を選択します。次に、Boot Bootloaderを選択し、プログラムに入力します。プログラマーは、SparkFunのTiny AVR Programmer Boardのように使用されます。
