STM32F7上のOpenCV-Discovery

私はEmboxオペレーティングシステムの開発者の1人です。この記事では、STM32746GボードでOpenCVを実行する方法について説明します。


「STM32ボード上のOpenCV」などの検索エンジンを使用すると、STM32ボードまたは他のマイクロコントローラーでこのライブラリを使用することに関心を持つ人がかなり見つかります。
名前から判断すると、何が必要かを示すビデオがいくつかありますが、通常（私が見たすべてのビデオで）STM32ボードでは、カメラから画像のみが受信され、結果が画面に表示され、画像処理自体も行われました。通常のコンピューター、またはより強力なボード（Raspberry Piなど）。


なんで難しいの？


検索クエリの人気は、OpenCVが最も人気のあるコンピュータービジョンライブラリであるという事実によって説明されます。つまり、より多くの開発者がそれに精通しており、マイクロコントローラーでデスクトップ用のコードを実行できるため、開発プロセスが大幅に簡素化されます。しかし、なぜこの問題を解決するための人気の既製のレシピがないのですか？


小さなボードでOpenCVを使用する際の問題は、次の2つの機能に関連しています。


  • 最小限のモジュールセットでもライブラリをコンパイルすると、非常に大きなコード（数メガバイトの命令）が原因で、同じSTM32F7Discoveryのフラッシュメモリ（OSを考慮しなくても）に収まりません。
  • ライブラリ自体はC ++で記述されています。
    • ポジティブランタイムのサポートが必要（例外など）
    • LibC/Posix, — STL (vector ..)

tl; dr：ステップバイステップの説明


0：次のようなEmboxソースをダウンロードします。


    git clone https://github.com/embox/embox && cd ./embox

1：まず、QSPIフラッシュドライブを「フラッシュ」するブートローダーを作成します。


    make confload-arm/stm32f7cube

ここで、ネットワークを構成する必要があります。TFTP経由でイメージをアップロードします。ボードとホストのIPアドレスを設定するには、conf / rootfs / networkファイルを変更する必要があります。


構成例：


iface eth0 inet static
    address 192.168.2.2
    netmask 255.255.255.0
    gateway 192.168.2.1
    hwaddress aa:bb:cc:dd:ee:02

gateway-イメージがロードされるホストアドレスaddress-ボードのアドレス


その後、ブートローダーを収集します。


    make
