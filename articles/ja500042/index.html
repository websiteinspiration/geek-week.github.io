前書き


FORTはシンプルで自然なコンピューター言語です。それは、高い効率が必要とされるところにかなりの分配を受けました。60年代に、彼は大学からビジネス、そして研究所へと発展する道を歩みました。これは、単純なインタープリターがその機能を拡張し、本格的なプログラミング言語とオペレーティングシステムになることができた方法についての物語です。

この作品は、プログラミング言語の歴史に関する会議のために書かれました-HOPL II。おそらく彼女のプレゼンテーションスタイルのため、彼女は最終的に拒否されました。

FORTは60年代にアメリカで広がりました-他の発達した言語と共に大学、ビジネスと科学で。当時、私は唯一のプログラマーでした。この期間の終わりまで、名前はありませんでした。ここで説明する内容はすべて、メモリ、断片的なドキュメント、および存続するプログラムのリストから復元されます。

FORTは全体としてはほとんどオリジナルではありませんが、コンポーネントのユニークな組み合わせです。私はそれを開発することを可能にしてくれた人々や組織に感謝しています。そして、あなたにそれについて読んでもらうことに興味を持ってくれてありがとう。

FORTはシンプルで自然なコンピューター言語です。今日[1991-約trans。]世界クラスのプログラミング言語と見なされています。彼が産業界、大学または政府の支援なしにそのようなレベルに達したという事実は、その有効性、信頼性および多様性の結果です。フォートは、その有効性が他の言語の人気よりも重要であるときに選択される言語です。これは、制御や通信システムなどの実用的な領域で頻繁に発生します。

多くのFORT組織と多くの中小企業がシステム、アプリケーション、およびドキュメントを提供しています。北米、ヨーロッパ、アジアでは年次会議が開催されます。 ANSI規格草案[ANS 1991]は間もなく発表されます。

フォートに関する本のどれも彼の「味」を完全に説明していません。最高のものはまだ最初だと思います-Leo Brodieの「Starting Forth」[たとえば、こちらから入手できます-https://www.forth.com/starting-forth/-約transl。]。

私たちが議論している古典的なFORTは、プログラマーがアプリケーションに最適な言語を開発するための最小限の十分なサポートを提供します。これは、キーボード、ディスプレイ、コンピューター、ドライブなど、ワークステーションなどの環境を対象としています[当時、ディスクドライブは外付けであることが多かったため、作成者は最小限のコンピューターシステムのコンポーネントを一覧表示しています。 transl。]。

FORTは本質的には文脈自由なテキスト言語です。スペースで区切られた「単語」を組み合わせて、新しい「単語」を作成できます。これらの単語の約150は、以下を提供するシステムを構成します。

         SAO  1958  
        SLAC  1961   
         RSI  1966    ,   , 
     Mohasco  1968  ,  , ,  ,
                    
        NRAO  1971   *,

*刺繍コードは、サブプログラム呼び出しの配列であるプログラムをコンパイルする方法です。https：//en.wikipedia.org/wiki/Threaded_codeを参照してください。ペレフ。

MIT、SAO、1958


最もエキサイティングな時期は、スプートニクが打ち上げられた1957年10月でした。 transl。]。私はMITの2年生で、ハーバード大学のSAO（スミソニアン天体物理天文台、16音節、悪夢）でパートタイムで働いていました。観測所は衛星の光学追跡を担当していました-月を使用した視覚観測[プロジェクトムーンウォッチを意味する-https ://en.wikipedia.org/wiki/Operation_Moonwatch-約。 transl。]および衛星追跡カメラ[Baker-Noonカメラ-https://en.wikipedia.org/wiki/Schmidt_camera#Baker-Nunn-約ペレブ。]。驚いたことに、彼らは学生を雇ってフリーデンのデスクトップ電卓の軌跡を見つけました。 John GaustadがMITのIBM EDPM 704について教えてくれ、彼のFORTRAN IIマニュアルを貸してくれました。その結果、私の最初のプログラムであるエフェメリス4では、仕事がなくなりました。

さて、すでにプログラマーとして、George Veisと最小二乗法を適用して軌道、静止位置のパラメーターを決定し、結果として地球の形状を決定しました。もちろん、この「アルバイト」には少なくとも40時間かかりましたが、はい、私の卒業証書はカバーされました。

John McCarthyは、MITで優れたLISPコースを教えました。このコースは、再帰と素晴らしいプログラミング言語の紹介です。 Wil Badenは、ラムダ計算のLISPは、後置表記Lukasewcleicz [Lukashevichの表記、逆ポーランド表記としても知られている-約transl。]。

潜水艦も関連性があり、奇妙な分析が右から左に行われました。私は彼に感心し、彼のオペレーターを真似ていますが、彼らが最適なセットを構成しているとは思いません。

50年代のプログラミングは、今日よりも厳しいものでした。私のソースコードは2枚のパンチカードを使用しました。車に乗せるには、いじくり回す必要があり、基本的には自分でやりました。組み立てには（Cと同じように）30分かかりましたが、限られたコンピュータ時間は、おそらく3番目のシフトを除いて、1日に1回だけの起動を意味しました。

そして、入力データを含むカードを読み取り、プログラムを制御する簡単なインタープリターを作成しました。彼はコンピューティングも制御していました。軌道の5つのパラメータのそれぞれについて、大気のドリフトと球からの地球の形状の偏差を考慮に入れるための経験式がありました。このようにして、プログラムを再コンパイルすることなく、いくつかの衛星のさまざまな方程式を作成できました。

これらの方程式には、P2（2次多項式）やS（正弦）などの要素が加わりました。ほとんどの場合、36ビットの浮動小数点演算が使用されたため、オーバーヘッドは低くなりました。データスタックは必要ありませんでしたが、おそらく私はそれについて知りませんでした。

そしてFORT通訳の話が始まりました-言葉から

WORD  NUMBER  INTERPRET  ABORT


それらはオペレーター番号であったため、まだ言葉で記録されていません。

INTERPRETは、WORDを使用してスペースで区切られた単語を読み取り、NUMBERを使用して単語を変換します[テキストレコード内の数値-約。変換。]バイナリ形式（この場合、固定小数点付き）。このような自由形式での入力は珍しいものでしたが、より効率的（小さくて高速）で、信頼性が高くなっています。 Fortranスタイルの入力は特定の列にフォーマットされ、タイプミスにより多数の遅延が発生しました。

このインタプリタは、FORTRANでエンコードされたIF ... ELSE IF構文を使用して、各文字を照合しました。すべてのエラー処理は、プログラムの実行を中断するだけでした。次に、今のように、ABORTという単語ユーザーに何をすべきかを尋ねます。入力カードには読み取り順に番号が付けられているため、エラーの場所を見つけることができます。

スタンフォード、SLAC、1961


1961年、私は数学を勉強するためにスタンフォードに来ました。スタンフォードは自身のコンピューターサイエンス部門を設立しましたが、私はコンピューティングにもっと興味を持っていました。彼らが自分のALGOLコンパイラーを（あえて？）書けることに感銘を受けました。それでも、運命は私をバローズB5500コンピュータに連れて行ってくれました。私は再びSLAC（Stanford Linear Accelerator、10音節）でパートタイムで働いて、将来の3キロメートルの電子加速器でビーム保持を最適化するコードを書きました。これは最小二乗法（私が経験した）の位相空間への自然な適用でした。ハル・バトラーは私たちのグループの責任者でした、そして私たちの輸送プログラムは成功しました。

最小二乗法の別のアプリケーションは、1964年にALGOLで書かれたCURVEプログラムでした。それは微分補正を伴う非線形データ選択のための普遍的なプログラムでした。その統計的厳密さは、モデルと実験データの一貫性の詳細な分析を提供しました。

データ形式とモデル方程式が解釈され、計算を容易にするためにスタックが使用されました。CURVEはFORTの印象的な先駆者でした。単純な方程式ではなく、より複雑なモデルで作業する機会を提供するために、彼女は次の言葉を紹介しました。

     +  -  *  NEGATE
     IF  ELSE  THEN  <
     DUP  DROP  SWAP
     :  ;  VARIABLE  !  (
     SIN  ATAN  EXP  LOG


スペルは少し異なりました：

     NEGATE  -    MINUS
       DROP  -    ;
       SWAP  -    .
          !  -    <
   VARIABLE  -    DECLARE
          ;  -    END
     ( ...)  -    COMMENT ...;


6文字の入力ワード（LISPの類推によりATOMと呼ばれる）を定義するために、インタープリターはIF ... ELSE IFを使用しました。DUP、DROP、およびSWAPは機械語命令でした。書き方が違うことに驚きました。単語「：」（コロン）はALGOLのラベル形式から取得され、左から右に解析するために「反転」されました（インタープリターが未定義の単語を検出しないようにするため）。

     ALGOL - LABEL:
     CURVE - : LABEL

実際、コロンは入力行の位置を示していましたが、これは後で解釈されます。解釈は「; "（セミコロン）。コロンの1つのバージョンはDEFINEと呼ばれていました。

レコードオペレーター " ！「（感嘆符）はVARIABLEに関連して現れました。ただし、「@」の読み取りは自動的に行われました。注意してください-コードはコメントを必要とするほど複雑になっています。一度批判された後置条件はここから始まりました：

     ALGOL - IF expression THEN true ELSE false
     CURVE - stack IF true ELSE false THEN

スタックの最上位にゼロ以外の値があった場合、それはブールの真の値（真）として認識されました。THENは独特の結末を提供しました。その欠如はALGOLでいつも私を混乱させました。同様の表現は次のように解釈されました：IFがELSEまたはTHENの入力をスキャンしました。<という言葉は、関係がブール値をスタックに残すという規則を導入しました。真の場合は1、偽の場合は0です。超越関数は、もちろん、ライブラリルーチンへの呼び出しです。

モハスコ、1968


1968年、ニューヨークのアムステルダムにある商業会社Mohasco Industries、Incでプログラマーとして働きました。カーペットや家具などの家庭用品を製造する大企業でした。私はRSIでジェフリーチと協力しましたが、彼は州の北部に移動するように説得しました。私は結婚したばかりで、アムステルダムにはニューヨークとは対照的に、小さな町の素晴らしい雰囲気がありました。

私はCOBOLでコードを書き直して、ビジネスソフトウェアが実際に何であるかを見つけました。Bob Raikoは企業データ処理を担当し、2つの関連プロジェクトを提供してくれました。

彼は、2250のグラフィックディスプレイを備えたIBM 1130ミニコンピューターをレンタルしましたが、その目的は、コンピューターグラフィックスがパターン化されたラグの作成に役立つかどうかを確認することでした。その答えは「色のサポートなしでは解決できない」であり、1130オプションはもう利用できませんでした。

当時、私は最新のミニコンピューターを持っていました：16ビットプロセッサー、8K RAM、ディスクドライブ（私の人生で最初）、キーボード、プリンター、カードリーダーとパンチ、FORTRANコンパイラー。リーダーとパンチは、ディスク障害が発生した場合のバックアップオプションでした。私は再びインタープリターを（FORTRANに戻して）移植し、2250で動作するようにクロスアセンブラーを追加しました。

システムは素晴らしかった。 IBMが静止した平らな絵をほとんど描くことができなかったとき、彼女はアニメーションの3次元画像を描くことができました。これが初めてのリアルタイムグラフィックだったので、私は最初のビデオゲームであるSpacewarのコードを書きました。もともとはALGOLで書かれていたFORTチェスプログラムも書き直しました。

インタプリタを含むファイルはFORTHと呼ばれ、4番目の（次の）世代のソフトウェア、FOURTHを意味していました。ただし、オペレーティングシステムがファイル名を5文字に制限したため、名前を短くする必要がありました。

この2250のプログラミング環境はFORTRANよりもはるかに優れていたため、2250クロスアセンブラを1130コンパイラに拡張しました。

     DO  LOOP  UNTIL
     BLOCK  LOAD  UPDATE  FLUSH
     BASE  CONTEXT  STATE  INTERPRET  DUMP
     CREATE  CODE  ;CODE  CONSTANT  SMUDGE
     @  OVER  AND  OR  NOT  0=  0<


彼らの綴りはまだ異なっていました[FORTの標準から-およそ。翻訳。]：

      LOOP     CONTINUE
     UNTIL -      END
     BLOCK -      GET
      LOAD -      READ
      TYPE -      SEND
 INTERPRET -      QUERY
    CREATE -      ENTER
      CODE -


これは私がこれまでに見つけたセント記号の唯一のアプリケーションでした。

カウンターとループ制限はデータスタックに保存されました。DOとCONTINUEはFORTRANのおかげで生まれました。

BLOCKは、ディスクアクセスを最小限に抑えるためにバッファーの数を制御しました。LOADは、サイズが1024バイトのブロックからソースコードを読み取ります。 1024バイトのサイズは、ディスク上のブロックのサイズに基づいて選択され、成功しました。UPDATEという単語を使用すると、メモリ内のブロックをマークし、後でバッファを解放する必要があるときに（またはFLUSHによって）ディスクにフラッシュすることができます。一種の仮想メモリを実装し、書き込み操作中に暗黙的に呼び出されます。BASE

という言葉8進数、16進数、および10進数を処理できます。CONTEXTという単語は、辞書の最初のヒントであり、編集者の単語を分離するのに役立ちました。STATEという単語は、コンパイルモードと解釈モードを設定します。コンパイル中に、文字の数と単語の最初の3文字がコンパイルされました[識別子にコンパイルされました-約。 transl。]以降の解釈のため。奇妙なことに、単語は特殊文字によって中断される可能性がありますが、これはすぐに見捨てられました。変数、配列、ディスクからの読み取りを区別する必要があるため、読み取り演算子（@）には多くのオプションがありました。DUMPは、デバッグのためにメモリを掘り下げることができました。

しかし、最も重要なのは、辞書ができたことです。インタプリタコードは名前を取り、リンクされたリストでそれを探しました。CREATEという単語は、古典的な語彙エントリを作成しました。

   *       
   *     3  *
   *  ,   
   *

* FORT識別子は次の形式で32ビットにパックされています。1バイト-単語の文字数、残りの3バイト-単語の最初の3文字（ハサー、沈黙！）-約ペレフ。

コードフィールドは重要なイノベーションでした。単語が見つかると同時に、インタプリタオーバーヘッドからの間接遷移は1つだけ残りました。スタンフォード大学のコンパイラ開発者から、単語を区別する際の文字数の意味について学びました。CODE

という単語は、重要なクラスの単語を作成しました。そのパラメーターフィールドには機械命令が含まれていました。したがって、今ではコンピュータの能力の範囲内で任意の単語を作成することが可能でした。 「; CODE」という単語は、新しいクラスの単語に対して実行されるコードを定義し、現在オブジェクトと呼ばれるものを導入しました。 語

SMUDGEは、定義の解釈中に再帰を回避しました。辞書を最新の定義から古い順に表示するため、不要な再帰が一般的でした。

M *は、2つの16ビット数を32ビット積に変換する通常のハードウェア乗算です（もちろん、データスタック内の引数）。その後に* /除算が続き、小数演算を実装します。/ MODは商と剰余の両方を返し、ファイル内のエントリを見つけるのに理想的です。SQRTは、32ビットの引数から16ビットの結果を返します。SIN.COSは、サインとコサインの両方をすぐに返します。これは、ベクトルおよび複素数演算（FFT）に役立ちます。ATANは逆演算であり、象限のあいまいさはありません。EXPとLOGは、基数2で計算されます。
