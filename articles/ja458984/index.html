取引所で取引するための最初のアプリケーションを作成する方法：3つの初期ステップ



現代の交流は非常に技術的であり、ITスペシャリストの注目を集めています（たとえば、トピックに関する私の記事の活発な議論はこれを示しています）。多くはトレーディングロボットの作成のトピックに興味を持っています-誰かが自分で取引所でお金を稼ぎたいと思っています。今日は、そのような最初の製品の作成に取り組む方法について話します。可能なテクノロジースタック、エントリのしきい値を下げること、および可能な損失を最小限に抑える方法について説明します。

免責事項：交換のデバイス、リスクの存在を少なくともおおまかに想像している人のための投稿は、最後のお金のためではなく、一般に常識を持っている実験の機会があります。

可能な技術スタックを探る


他のITプロジェクトと同様に、証券取引所で取引するためのアプリケーションを開発する場合、特定のツールの選択が最終的な目標を決定付けます。トレーディングロボットの開発を開始する前に、システム全体を設計し、その中に含まれるモジュール、モジュールがどのように相互作用するかを理解する必要があります。

ミリ秒ごとが重要な高周波ロボットを作成していますか？取引システムはどの国のどの市場で機能しますか？その中には取引エンジンしかありませんか、それともリスク管理モジュールを追加する必要がありますか？

これらの各ケースで、テクノロジーのセットは異なる場合があります。選択への適切なアプローチは、QuantStartプロファイルリソースの記事で説明されています（もちろん、英語-金融アプリケーションを開発するには、この言語を習得することをお勧めします）。

一般的には、次のようなテクノロジのリストから選択する必要があります。

  • API（これが私の選択したツールです）だけでなく、ほとんどの仲介業者には、C ++および/またはJavaのインターフェースがあります。特定のブローカーのコミュニティの代表者/クライアントがC＃、Python、R、Excel、MatLabで記述した接続用の既製のモジュールを見つけることができる場合があります。しかし、あなた自身でそれらを書く必要があるかもしれません。
  • データ分析には、uBLAS、LAPACK、NAG for C ++などのライブラリがよく使用され、NumPy / SciPyは、MatLabのPythonで人気があります。
  • GPU（FPGA）を使用して高周波ロボットを開発する場合、CUDAフレームワークに精通する必要があります。

組み込みプログラミング言語の助けを借りて、金融分野での開発の詳細を学びます


強力な取引システムの作成は、デフォルトではかなり大規模なプロジェクトです。しかし、原則として、金融分野の開発に精通し、トレーディングロボットを作成する際のさまざまなポイントを研究することが目標である場合、いわゆる組み込みプログラミング言語を使用することは十分に可能です。

取引を手動で完了するだけでなく、組み込みのスクリプト言語を使用して簡単なロボットを作成できる取引端末があります。

ロシアの株式市場で最も人気のある端末の1つはQUIKで、アルゴリズム言語QPILE（QUIK Programmable Interface and Logic Environment）を備えています。これにより、簡単な取引システムを自動化できます。。また、QUIKでは、スクリプト言語LUAのインタープリターが組み込まれています。これはQLUAと呼ばれます。その上のスクリプトは、ソースコードの形式で、およびコンパイル済みバイトコードの形式ですでにターミナルにダウンロードできます。

QPILEコード例：

PROGRAM
FirmCode = "MC0012300000"
CurrentBalance = MONEY_CURRENT_BALANCE(ROWNAME, FirmCode, "EQTV", "SUR")
CurrentLimit = MONEY_CURRENT_LIMIT(ROWNAME, FirmCode, "EQTV", "SUR")
Locked = MONEY_LIMIT_LOCKED(ROWNAME, FirmCode, "EQTV", "SUR")
AvailableMoney = MoneyCurrentBalance + MoneyCurrentLimit – MoneyLocked
If AvailableMoney > 0
Status = " "
Else
Status = " "
SET_ROW_COLOR(ROWNAME, "RGB(255,138,138)", "DEFAULT_COLOR")
End If
END_PROGRAM

どちらの言語にも、かなり発達したユーザーコミュニティがあり、公式フォーラムのスレッドで質問できます。

最も重要なステップ：テストとデバッグ


金融セクター向けのアプリケーションの開発、特に取引所での取引のための手段の開発は、文字通りの意味でのエラーに対してここでお金を払わなければならないという事実によって複雑になります。これは、失敗が気付かれない可能性がある次のWebスタートアップ用のコードを記述しているわけではありません。取引ロボットのコードにエラーがあると、お金が失われます。したがって、デバッグとテストは、このようなソフトウェアを作成する上で最も重要な段階です。

もちろん、すべてのベストプラクティスを適用し、テストを作成し、コードの受け入れを正しく行う必要があります（プロジェクトを単独で開発している場合でも）。そして最も重要なのは、リアルマネーでテストを開始する前に、プログラムを仮想的なもので「ドライブ」することです。

場合によっては、借りた資金での取引（つまりレバレッジ）をテストすることもできますが、最初のロボットを作成するときは、そのような関数をロボットに追加する必要があるかどうかが問題になります。
