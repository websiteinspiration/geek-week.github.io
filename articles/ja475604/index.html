Zabbix 4.4での公式PostgreSQLテンプレートの設定

みなさん、こんにちは。

公式のPostgreSQLテンプレートDBが Zabbixに登場しました。この記事では、Zabbix 4.4で構成します。


注

すべてが英語で問題ない場合は、公式マニュアル

github.com/zabbix/zabbix/tree/master/templates/db/postgresqlに従ってテンプレートをインストールすることをお勧めします。

それでも、私の記事では、このリンクにないニュアンスが考慮されています。

テンプレートの準備


1.ホームディレクトリに移動します。

cd ~

2. gitユーティリティをダウンロードし、GitHubにある公式のZabbixリポジトリのクローンを作成します。

yum -y install git
git clone https://github.com/zabbix/zabbix.git

3. PostgreSQLテンプレートのあるディレクトリに移動します。

cd zabbix/templates/db/postgresql/

Zabbixエージェントのテンプレートの構成


1. PostgreSQLに接続します。

psql -U postgres

2. PostgreSQLサーバーへのアクセス権を持つ読み取り専用ユーザーzbx_monitorを作成します。

PostgreSQLバージョン10以降の場合：

CREATE USER zbx_monitor WITH PASSWORD '<_>' INHERIT; GRANT pg_monitor TO zbx_monitor;

PostgreSQLバージョン9.6以下の場合：

CREATE USER zbx_monitor WITH PASSWORD '<_>';
GRANT SELECT ON pg_stat_database TO zbx_monitor;

--   WAL    superuser.
ALTER USER zbx_monitor WITH SUPERUSER;

3. postgresql /ディレクトリを/ var / lib / zabbix /ディレクトリにコピーします。/ var / lib /にzabbix /ディレクトリがない場合は、作成してください。postgresql /ディレクトリには、PostgreSQLからメトリックを取得するために必要なファイルが含まれています。

cp -r postgresql/ /var/lib/zabbix/

4.次に、template_db_postgresql.confファイルをZabbixエージェント設定ディレクトリ/etc/zabbix/zabbix_agentd.d/にコピーし、Zabbixエージェントを再起動します。

cp template_db_postgresql.conf /etc/zabbix/zabbix_agentd.d/

5. pg_hba.confファイルを編集して、Zabbixへの接続を許可します。pg_hba.confファイルの詳細：https : //www.postgresql.org/docs/current/auth-pg-hba-conf.html。

ファイルを開きます。

vi /var/lib/pgsql/12/data/pg_hba.conf

いずれかの行を追加します（これが必要な理由がわからない場合は、最初の行のみを追加してください）。

host all zbx_monitor 127.0.0.1/32 trust
host all zbx_monitor 0.0.0.0/0 md5
host all zbx_monitor ::0/0 md5
注

PostgreSQLがPGDGリポジトリからインストールされている場合は、pg_isreadyへのパスをzabbixユーザーのPATH環境変数に追加します。

オプションとして：

ln -s /usr/pgsql-12/bin/pg_isready /usr/bin/pg_isready

*-pgsql 12バージョンを使用しているため、pgsql-12の代わりに別のパスを使用します。

これが行われない場合、ステータス：Pingは常にダウン状態になります。

Zabbixフロントエンドでのテンプレートの追加


PostgreSQLからメトリックを削除する必要がある人は、テンプレートを追加する方法を知っていると思います。したがって、そのプロセスを簡単に説明します。

  1. Zabbix;
  2. "Configuration" => "Host";
  3. "Create host" ;
  4. / "Templates" "Add";
  5. «Group» «Templates/Databases», "Template DB PostgreSQL", "Select" "Update";

しばらく待ってから、最後に「監視」=>「最新データ」=>「ホスト」に移動し、PostgreSQLでサーバーを選択します=>「適用」をクリックします。


楽しい！
