マルコフ連鎖のモンテカルロ法（MCMC）。前書き

こんにちは、ハブル！

以前に「余分な単語なしの機械学習」という本を発表したことを思い出してください。現在、この本はすでに販売されています。 MOの初心者にとって、本は確かにデスクトップになることができるという事実にもかかわらず、その中のいくつかのトピックはまだ触れられていませんでした。したがって、Pythonでのそのようなアルゴリズムの実装によるMCMCアルゴリズムの本質に関するSimon Kerstensによる記事の翻訳に関心のあるすべての人に提供しています。

マルコフ連鎖（MCMC）のモンテカルロ法は、特定の（未知の）正規化定数までしか知られていない確率分布からサンプリングするための強力なクラスのメソッドです。
ただし、MCMCについて説明する前に、そのような選択を行う必要がある理由について説明しましょう。答えは次のとおりです。サンプルからサンプル自体に興味があるかもしれません（たとえば、ベイジアン導出法を使用して未知のパラメーターを決定するため）、または確率分布に対して関数の期待値を概算するため（たとえば、統計物理学の状態の分布から熱力学量を計算するため）。場合によっては、確率分布モードのみに関心があります。この場合、数値最適化の手法で取得するため、完全に選択する必要はありません。

最も原始的な確率分布以外の確率分布からのサンプリングは困難な作業であることがわかります。逆変換法は、確率分布からサンプリングするための基本的な手法ですが、累積分布関数を使用する必要があり、これを使用するには、通常は不明な正規化定数を知る必要があります。正規化定数は、原則として数値積分で求めることができますが、次元数が増えるとすぐに実用化できなくなります。偏差サンプリングそれは正規化された分布を必要としませんが、それを効果的に実装するために、私たちにとって関心のある分布について知ることは多くのことを必要とします。さらに、この方法は、次元の呪いにひどく悩まされています。つまり、変数の数が増えると、その効果は急速に減少します。そのため、正規化定数を知る必要なく、ディストリビューションからの代表的なサンプルの受信をインテリジェントに整理する必要があります。

MCMCアルゴリズムは、このために特別に設計されたメソッドのクラスです。彼らはメトロポリスと他のランドマーク記事に戻ります。メトロポリスは彼にちなんで名付けられた最初のMCMCアルゴリズムを開発しました剛体球の2次元システムの平衡状態を計算するように設計されています。実際、研究者たちは、統計物理学に見られる期待値を計算できる普遍的な方法を探していました。
この記事では、MCMCサンプリングの基本について説明します。

マルコフチェーン

なぜサンプリングが必要なのかがわかったところで、MCMCの中心であるマルコフチェーンに移りましょう。マルコフ連鎖とは何ですか？技術的な詳細には触れませんが、マルコフチェーンは特定の状態空間におけるランダムな状態のシーケンスであり、特定の状態を選択する確率はチェーンの現在の状態にのみ依存し、以前の履歴には依存しません。このチェーンにはメモリがありません。特定の条件下では、マルコフチェーンには、特定の数の状態を克服して収束する、一意の定常状態分布があります。マルコフ連鎖におけるこのような状態状態の数の後、不変分布が得られます。

分布からサンプリングするために、MCMCアルゴリズムは、定常分布が次のようなマルコフチェーンを作成およびシミュレーションします;これは、最初の「シード」期間の後、そのようなマルコフ連鎖の状態が原理に従って分配されることを意味します。したがって、からサンプルを取得するには、状態を保存するだけで済みます。

教育目的のために、離散状態空間と離散「時間」の両方を考えてみましょう。マルコフ連鎖を特徴付けるキー量は、遷移演算子である状態である確率を示す時にチェーンが状態にあることを条件とする、時i。
楽しみのために（そしてデモンストレーションとして）、ユニークな定常分布を持つマルコフチェーンをすばやく作成しましょう。グラフのインポートと設定から始めましょう。

PYTHONでMETROPOLIS-HASTINGSアルゴリズムを実装します

さて、Metropolis-Hastingsアルゴリズムがどのように機能するかがわかったところで、その実装に移りましょう。最初に、正規化定数を使用せずに、選択を行う分布の対数確率を確立します。私たちはそれらを知らないと想定されています。次に、標準正規分布を使用します。

def log_prob(x):
     return -0.5 * np.sum(x ** 2)


次に、対称補助分布を選択します。一般に、メトロポリスヘイスティングスアルゴリズムのパフォーマンスは、補助ディストリビューションで選択するディストリビューションに関する既知の情報を含めると改善されます。簡略化されたアプローチは次のようにxなります。現在の状態を使用してからサンプルを選択します。つまり、特定のステップサイズΔを設定し、現在の状態から左または右に移動します。

def proposal(x, stepsize):
    return np.random.uniform(low=x - 0.5 * stepsize, 
                             high=x + 0.5 * stepsize, 
                             size=x.shape)


最後に、提案が受け入れられる確率を計算します。

def p_acc_MH(x_new, x_old, log_prob):
    return min(1, np.exp(log_prob(x_new) - log_prob(x_old)))


次に、これらすべてをまとめて、Metropolis-Hastingsアルゴリズムのサンプリングステージの本当に簡単な実装に組み込みます。

def sample_MH(x_old, log_prob, stepsize):
    x_new = proposal(x_old, stepsize)
    #   ,     :
    #       [0,1]  
    #     
    accept = np.random.random() < p_acc(x_new, x_old, log_prob)
    if accept:
        return accept, x_new
    else:
        return accept, x_old


マルコフ連鎖の次の状態、x_newまたはx_oldに加えて、MCMCステップが採用されたかどうかに関する情報も返します。これにより、サンプル収集のダイナミクスを追跡できます。この実装の結論として、繰り返し呼び出しsample_MHてマルコフ連鎖を構築する関数を記述します。

def build_MH_chain(init, stepsize, n_total, log_prob):

    n_accepted = 0
    chain = [init]

    for _ in range(n_total):
        accept, state = sample_MH(chain[-1], log_prob, stepsize)
        chain.append(state)
        n_accepted += accept
    
    acceptance_rate = n_accepted / float(n_total)
    
    return chain, acceptance_rate

def plot_samples(chain, log_prob, ax, orientation='vertical', normalize=True,
                 xlims=(-5, 5), legend=True):
    from scipy.integrate import quad
    
    ax.hist(chain, bins=50, density=True, label="MCMC samples",
           orientation=orientation)
    #     PDF
    if normalize:
        Z, _ = quad(lambda x: np.exp(log_prob(x)), -np.inf, np.inf)
    else:
        Z = 1.0
    xses = np.linspace(xlims[0], xlims[1], 1000)
    yses = [np.exp(log_prob(x)) / Z for x in xses]
    if orientation == 'horizontal':
        (yses, xses) = (xses, yses)
    ax.plot(xses, yses, label="true distribution")
    if legend:
        ax.legend(frameon=False)
    
fig, ax = plt.subplots()
plot_samples(chain[500:], log_prob, ax)
despine(ax)
ax.set_yticks(())
plt.show()

パラメータstepsizeとはn_total何ですか？最初にステップサイズについて説明します。これは、回路の現在の状態から試行状態をどれだけ削除できるかを決定します。したがって、これはq、マルコフ連鎖が取るランダムなステップの大きさを制御する補助分布パラメーターです。ステップサイズが大きすぎる場合、試行状態は多くの場合、確率値が低い分布の裾に行き着きます。 Metropolis-Hastingsサンプリングメカニズムは、これらのステップのほとんどを破棄します。その結果、受信率が低下し、収束が大幅に遅くなります。自分で見て：

def sample_and_display(init_state, stepsize, n_total, n_burnin, log_prob):
    chain, acceptance_rate = build_MH_chain(init_state, stepsize, n_total, log_prob)
    print("Acceptance rate: {:.3f}".format(acceptance_rate))
    fig, ax = plt.subplots()
    plot_samples([state for state, in chain[n_burnin:]], log_prob, ax)
    despine(ax)
    ax.set_yticks(())
    plt.show()

ステップサイズパラメーターの選択方法に関係なく、マルコフチェーンは最終的に定常分布に収束します。ただし、これには時間がかかる場合があります。マルコフチェーンをシミュレートする時間は、パラメーターによってn_total決定されます。これは、マルコフチェーン（したがって、選択されたサンプル）の状態の数を最終的に決定するだけです。チェーンの収束が遅い場合はn_total、マルコフチェーンが初期状態を「忘れる」時間を持つように、チェーンを増やす必要があります。したがって、ステップサイズを小さくして、パラメーターを増やすことでサンプル数を増やしますn_total。
