みなさん、こんにちは！私の名前はウラジスラフロダンです。現在、OTUSポータルでソフトウェアアーキテクチャと高負荷ソフトウェアアーキテクチャのコースを教えています。現在OTUSには、Spring Frameworkの開発者コースの新しいスレッド用のオープンセットがあります。コースの開始を予測して、私はあなたと共有したい小さな著作物を書くことにしました。




バックグラウンド


Springはそれ自体に多くの「魔法」を含んでおり、自明ではないことをいくつか実行します。これを無知または誤解すると、このフレームワークを使用してアプリケーションを作成するプロセスで発生する可能性のある副作用が発生する可能性があります。

これらの明白でないものの1つは、Java Collection Frameworkインターフェースの注入です。このトピックに関連する熊手を独立して踏み、同僚から次の質問を聞いたので、私はそれを整理し、研究の結果を記事の形式で記録することにしました。これは、すでに働いている人やSpringの初期の開発中に役立つことを期待しています。

シナリオ


映画のヒーローと連携するサービスを見てみましょう。それらの3つがあります：ランボー、ターミネーター、ガンダルフ。それぞれが個別のクラスを表し、その親は共通です-ヒーロー。

public class Hero {
}

@Component
public class Rambo extends Hero {

    @Override
    public String toString() {
        return "Rambo";
    }
}

@Component
public class Terminator extends Hero {

    @Override
    public String toString() {
        return "Terminator";
    }
}


@Component
public class Gandalf extends Hero {
    @Override
    public String toString() {
        return "Gandalf";
    }
}

注射リスト


過激派の英雄と連携するサービスを作成したいとします。おそらく、そのような英雄のリストをそれに注入する必要があります。

問題ない！サービスと構成を作成します。

@Service
@Getter
public class ActionHeroesService {
    @Autowired
    List<Hero> actionHeroes;
}

@Configuration
public class HeroesConfig {

    @Bean
    public List<Hero> action() {
        List<Hero> result = new ArrayList<>();
        result.add(new Terminator());
        result.add(new Rambo());
        return result;
    }
}

すべてが大丈夫ですが、チェックすると、ガンダルフがリストにあることがわかります！

Springは、Listをインジェクトする必要があることに気づき、コンテキスト内にあるBeanを巡回し、ジェネリック型に適合するすべてのBeanを見つけ、それらからListを収集してインジェクトし、リストを無視しました。

Springに私たちがやりたいことをさせるには？

オプション1.松葉杖


問題は、まさにJava Collection Frameworkのインターフェースを挿入しようとしていることにあるため、サービスおよびもちろん構成でListをArrayListに置き換えるだけで、Springが必要なクラスのインスタンスを見つけることができます。その後、すべてが期待どおりに機能します。

@Configuration
public class HeroesConfig {

    @Bean
    public ArrayList<Hero> action() {
        ArrayList<Hero> result = new ArrayList<>();
        result.add(new Terminator());
        result.add(new Rambo());
        return result;
    }
}

@Service
@Getter
public class ActionHeroesService {
    @Autowired
    ArrayList<Hero> actionHeroes;
}

オプション2.より正確


Springの手を縛る別の方法は、リストだけでなく、Qualifierを追加した特別な名前のBeanをサービスに注入するように彼に依頼することです。したがって、私たちは豆を注入することができます。

@Service
@Getter
public class ActionHeroesService {
    @Autowired
    @Qualifier("action")
    List<Hero> actionHeroes;
}

射出マップ


多くの人がリストインジェクションインジェクションのニュアンスを知っているなら、事態は一般的にマップでさらに悪くなります。
映画の主人公と連携するサービスを書いてみましょう。キーが、そして主人公の豆の値が映画の名前を含んでいるマップがそれに注入されます：

@Service
@Getter
public class MainCharactersService {
    @Autowired
    Map<String, Hero> mainCharactersByFilmName;
}

@Configuration
public class HeroesConfig {

    @Bean
    public Map<String, Hero> mainCharactersByFilmName() {
        Map<String, Hero> result = new HashMap<>();

        result.put("rambo", new Rambo());
        result.put("terminator", new Terminator());
        result.put("LOTR", new Gandalf());

        return result;
    }
}

起動時に、ガンダルフのキーはLOTRではなく、ガンダルフであることがわかります。ガンダルフは、書かれたのは映画の名前ではなく、豆の名前であると結論付けることができます。一方、ランボーとターミネーターの場合は幸運でした。メインキャラクターの名前は、映画のタイトル。

実際、キーが文字列であるBeanの値であるMapを注入する必要がある場合、Spring（リストの場合と同様）は提案したMapを単に無視し、コンテキストを通過してすべての適切なBeanを収集しますs、そしてそれらを値として、それらの名前をキーとして持つマップを作成します。

回避策はリストで機能したものと同様です。

オプション1.松葉杖


マップをHashMapに置き換えます。

@Service
@Getter
public class MainCharactersService {
    @Autowired
    HashMap<String, Hero> mainCharactersByFilmName;
}


@Configuration
public class HeroesConfig {

    @Bean
    public HashMap<String, Hero> mainCharactersByFilmName() {
        HashMap<String, Hero> result = new HashMap<>();

        result.put("rambo", new Rambo());
        result.put("terminator", new Terminator());
        result.put("LOTR", new Gandalf());

        return result;
    }
}

オプション2.より正確


修飾子を追加：

@Service
@Getter
public class MainCharactersService {
    @Autowired
    @Qualifier("mainCharactersByFilmName")
    Map<String, Hero> mainCharactersByFilmName;
}
