Power Automate VS Logic应用程序。一般信息

大家好！让我们今天谈论电源自动化和逻辑应用程序。通常，人们不了解这些服务之间的区别以及应该选择哪些服务来解决他们的问题。让我们做对。

Microsoft Power自动化


Microsoft Power Automate是一项云服务，使用户能够创建线程以自动执行耗时的业务任务和流程。此服务适用于Citizen Developers-不是100％开发人员，但参与应用程序开发和流程自动化的用户。

Microsoft Power Automate是Microsoft Power Platform的一部分，Microsoft Power Platform还包括Power Apps，Power BI和Power Virtual Agents等服务。通过该平台，可以轻松地从相关的Office 365服务中获取所有必要的信息，并将其组合到应用程序，数据流，报表以及辅助帮助程序服务中。



创建Power Automate线程基于“ trigger” =>“ action set”的概念。流从特定的触发器开始，例如，可以在SharePoint列表中创建项目，接收邮件通知或HTTP请求。启动后，将开始处理此流程中配置的动作。作为动作，可以使用各种服务的连接器。当前，Microsoft Power Automate支持200多种不同的第三方服务以及Google，Dropbox，Slack，WordPress等巨头的服务，以及各种社交服务：Blogger，Instagram，Twitter，Youtube，Facebook等。当然，除此之外，还可以与Office 365应用程序集成。为了简化Microsoft Power Automate的使用，Microsoft为各种应用程序和事件提供了大量标准模板，我们只需填写必要的参数集即可使用这些模板。此外，用户可以在设计器中自己创建模板，然后将其发布以供其他用户使用。

Microsoft Power Automate的独特功能是：

  1. 大量连接各种第三方服务的连接器。
  2. 支持与Office 365服务集成。
  3. 根据特定触发器启动流的能力-例如集成方案，当收到到Gmail邮箱的信件时，有必要在另一项服务中启动一系列操作，例如，向团队发送消息并在SharePoint列表中创建一个条目。
  4. 具有调试流的能力，以及有关流的每个阶段状态的详细信息。

但是，Microsoft Power Automate是Logic Apps的简化版本。这意味着，当您创建Power Automate流时，将在后台创建Logic Apps流，该流将处理配置的逻辑。简而言之，Power Automate使用Logic Apps引擎来实现线程。

Microsoft Power Automate当前作为Office 365订阅的一部分提供，也可以作为单独的计划提供，由用户或线程购买。



值得注意的是，高级连接器仅在单独购买时才可用。Office 365订阅不提供高级连接器。

逻辑应用


Logic Apps是一项服务，是Azure应用程序服务的一部分。 Azure Logic应用程序是Azure Integration Services平台的一部分，该平台包括访问Azure API的功能。与Power Automate一样，Logic Apps是一项基于云的服务，旨在自动化业务任务和流程。但是，尽管Microsoft Power Automate专注于业务流程，而Logic Apps却更专注于作为集成集成解决方案一部分的业务逻辑块。这样的决定将需要更仔细的管理和控制。 Logic Apps之间的主要区别之一是可以指定检查触发器的频率。 Power Automate没有此设置。
