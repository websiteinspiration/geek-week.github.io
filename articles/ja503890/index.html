ReactおよびReact Nativeパート1でAzure AD B2Cを構成した方法の物語（チュートリアル）


序文


私たちはクロスプラットフォームのソフトウェアを開発しており、「共通の認可システムを作る」という課題に直面していました。

3つのアプリケーションに共通のユーザーデータベースを作成する必要がありましたが、同時に、各アプリケーションには独自の特徴があります。また、共通のマイクロサービスもあり、トークンを1つのソースから生成して検証したいと考えています。なぜなら 私たちはAzureサービスを使用していますが、その選択はAzure AD B2Cにかかっています。このすべてのものをセットアップすると、多くの落とし穴が明らかになりました。ストーリー全体をいくつかのパートに分けます。

  • パート1：Azure AD B2Cでアプリケーションを構築および構成する
  • パート2：Identity Frameworkエクスペリエンスの操作
  • パート3：Reactアプリケーション接続
  • パート4：React Nativeアプリの接続
  • パート5：.NET Core 3でのバックエンドの接続と構成

Azure AD B2Cでアプリケーションを構築および構成する


Azure B2Cを作成するには：

1. Azureポータルで、検索バーに「Azure B2C」と入力します 。2.このチュートリアルのすべての手順を実行します。 次に、ユーザーには2つのオプションがあります。







  1. 基本的なユーザースレッドを使用する
  2. カスタムユーザーストリームを作成します。

カスタムユーザーストリームを使用する場合でも、ベースユーザーストリームにアプリケーションを作成する必要があります


私の経験から-基本的なストリームでは、必要な機能のほとんどを作成できます。

  1. 承認用のカスタムページ。
  2. ユーザーについて収集するデータを選択します
  3. トークンで送信するデータ。

だが

  1. メールを確認するカスタムレターを送信することはできません
  2. 独自の値でフィールドを作成することはできません

基本的なユーザースレッド

カスタムユーザーストリーム


カスタムユーザーフローを作成するには、作成したディレクトリに移動して、2つのアプリケーションを作成する必要があります。

  1. IdentityExperienceFramework
  2. ProxyIdentityExperienceFramework



ここで それらを作成する方法について。

「アプリケーションの登録（暫定版）」の指示通りに全力を尽くしました。

そして、ここで最初の落とし穴がすでに生じており、それは私を愚痴に駆り立てました。

指示のステップの1つは言う：
[詳細設定]セクションで、[アプリケーションをパブリッククライアントとして扱う（[はい]を選択）]を有効にします。

また、ProxyIdentityExperienceFrameworkアプリケーションに次のボタンが表示されます。 ただし、表示されません（このボタンがないと、ちなみに何も機能しません）。それが表示されるようにするためには、あなたがのための指示から2つのステップを実行する必要がIdentityExperienceFrameworkでProxyIdentityExperienceFrameworkすなわち、





1. [ 管理]セクションで、[ APIプロビジョニング]を選択します。

2. [ スコープの追加]を選択し、[ 保存]を選択して、デフォルトのアプリケーション識別子URIを受け入れます。

次に、リポジトリをダウンロードする必要があります。Identity Experience Framework

タブを開き、ルート（シナリオを除く）のいずれかのフォルダーから次の順序でファイルをダウンロードします。
a.TrustFrameworkBase.xml
b.TrustFrameworkExtensions.xml
c.SignUpOrSignin。XML
d.ProfileEdit。XML
e.PasswordReset。XML
