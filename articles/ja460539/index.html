昨年は、お気に入りのフレームワークであるVue.jsを使用して、それを説明し、表現しました。そして、私はまだVueでのエラー処理を理解していないことに気づきました。私は完璧なコードを書いていると言ってこれを説明したいのですが、私たちは皆、本当に物事がどういうものか知っています。過去数日間、私はVueが提供するさまざまなエラー処理方法を試し、発見を共有することにしました。もちろん、このレビューはすべての可能なシナリオを網羅しているわけではありませんが、お役に立てば幸いです。

それでは、Vueアプリケーションでエラーをどのように処理できますか？Vueフレームワークのメインガイドにエラー処理に関するセクションがないことに少し驚いた。



はい、マニュアルに同様のセクションがありますが、非常に短く、その全体の意味は次の引用に適合します。

「コンポーネントのレンダリング中に実行時エラーが発生した場合、それが指定されていれば、グローバル構成関数Vue.config.errorHandlerに渡されます。特にVueとの統合が公式にサポートされているため、このフックをSentryなどのバグ追跡サービスと組み合わせて使用​​すると便利でしょう。

私の意見では、このトピックはドキュメントでもう少しカバーする必要があります（そしてそれを補足するのに役立つと思います）。一般に、Vueでのエラー処理は、次の手段に分類されます。

  • errorHandler;
  • warnHandler;
  • renderError;
  • errorCaptured;
  • window.onerror（このツールはVueに固有のものではありません）。

これらのトリックを詳しく見てみましょう。

ナンバーワンのエラーハンドラー：errorHandler


最初の対策はerrorHandlerです。ご想像のとおり、これはVue.jsアプリケーションの標準エラーハンドラです。次のように割り当てることができます。

Vue.config.errorHandler = function(err, vm, info) {

}

上記の関数宣言では、errは現在のエラーの説明、infoはVueに固有のエラー情報の文字列、vmは現在のVueアプリケーションです。複数のVueアプリケーションが同じWebページで同時に実行できることを覚えておいてください。このエラーハンドラーはそれらすべてに適用されます。次の簡単な例を考えます。

Vue.config.errorHandler = function(err, vm, info) {
  console.log(`Error: ${err.toString()}\nInfo: ${info}`);
}

最初のエラーの場合、このコードはアクションを実行しません。覚えていれば、これはエラーメッセージではなく警告を生成します。

2番目のケースでは、エラーが処理され、次のテキストが表示されます。

Error: ReferenceError: x is not defined
Info: render

最後に、3番目の例では次の結果が得られます。

Error: ReferenceError: x is not defined
Info: v-on handler

前の2つの例では、情報が見出し「情報」の下にある「有用な」ことに注目してください。次のツールがどのように機能するかを確認してみましょう。

3番目のエラーハンドラー：renderError


次に、3番目のエラー処理メソッドであるrenderErrorを示します。前の2つとは異なり、このツールはコンポーネントに依存し、普遍的ではありません。warnHandlerと同様に、このプロダクションハンドラーは無効になっています。

それを使用するには、コンポーネント/アプリケーションに挿入します。以下は、ドキュメントからの変更例です。

const app = new Vue({
  el:'#app',
  renderError (h, err) {
	return h('pre', { style: { color: 'red' }}, err.stack)
  }
})

このエラーハンドラーが最初の例で使用されている場合、最初の例ではエラーではなく警告が表示されるため、何もしません。計算されたプロパティでエラーが発生する2番目の例でこのハンドラをチェックすると、その結果が画面に表示されていることがわかります。これは、以下のcodePenデモで確認できます。

4番目のエラーハンドラー：errorCaptured


最後に、errorCapturedツール（Vueに固有）があります。これは私を混乱させ、正直なところ、まだ少し混乱しています。ドキュメントには次のように書かれています。

「子コンポーネントでエラーが発生すると呼び出されます。このフックは3つの引数を受け取ります。エラー、エラーの原因となったコンポーネントのインスタンス、およびエラーが記録された場所に関する情報を含む文字列です。エラーがさらに伝播するのを防ぐために、フックはfalseを返す場合があります。

私の調査（繰り返しますが、私は強く疑います）によると、このエラーハンドラーは、子コンポーネントのエラーを処理する親コンポーネントでのみ使用する必要があります。私の知る限り、メインのVueインスタンスでは使用できませんが、子を持つコンポーネントでのみ使用できます。

これを確認するために、親と子のコンポーネントのセットを作成しました。

世界中のすべてを管理する単一のツール：window.onerror



最後の（そして最も強力な）オプションは、window.onerrorを使用することです。これは、JavaScriptコードが実行されたときに発生する可能性のあるすべてのグローバルエラーハンドラです。このハンドラーの形式は次のとおりです。

window.onerror = function(message, source, line, column, error) {

}

おそらく、上記のコードで推測できない唯一のことは、スクリプトのURLであるソース引数の意味です。

これが楽しみの始まりです。この関数を定義しても、Vue.config.errorHandlerを使用しない場合は、役に立ちません。 VueはVue.config.errorHandlerを定義することを期待しており、定義しない場合、制限を超えてエラーを伝播しません。たぶん、これは理にかなっています...よくわかりません。私にとっては、あまり意味がありません。さらに奇妙なこと：Vueエラーハンドラー自体にエラーがあるとしましょう。また、window.onerrorハンドラにも到達しません。

これは、対応する例を含むCodePenのデモです。errorHandlerでエラーをコメントアウトしましたが、コメントを削除すると、グローバルエラーハンドラーが機能しないことがわかります。これは、2番目のボタンを押した場合にのみ機能します。
