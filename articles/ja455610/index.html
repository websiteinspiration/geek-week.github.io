過去10年間で最も人気のあったプロセッサの1つがIntel Core i7-2600Kでした。シングルコアプロセッサのパフォーマンスと効率を飛躍的に向上させ、プロセッサ自体もオーバークロックに十分に対応できるため、この設計は革新的でした。次の数世代のIntelプロセッサは、それほど興味深く見えなくなり、ユーザーにアップグレードする理由を与えなかったため、「私は2600Kを使い続けます」というフレーズが、今日でもフォーラムやユビキタスに普及しています。このレビューでは、古いプロセッサーを搭載したボックスからほこりを払い落とし、ベテランを2019年に工場パラメーターとオーバークロックの両方で一連のベンチマークに駆り立て、彼がまだチャンピオンであることを確認しました。

2600Kが世代にとって重要になった理由


椅子に座り、座って2010年の自分を想像してみてください。これは、レガシーCore 2 DuoまたはAthlon IIシステムを見て、アップグレードの時期であることに気付いた年でした。あなたはすでにNehalemのアーキテクチャに精通しており、Core i7-920は十分に加速し、競合他社と同じように機能します。良い時期でしたが、突然Intelは業界のバランスを取り直し、真に革新的な製品を生み出しました。ノスタルジアの響きは今でも聞かれます。




Core i7-2600K：最速のSandy Bridge（最大2700K）

この新製品はサンディブリッジでした。 AnandTechは独占レビューを発表しましたが、その結果は多くの理由で信じられないほどでした。その時の私たちのテストによると、プロセッサは、特に数年前にリリースされたPentium 4サーマルモンスターを考えると、以前に見たすべてのものよりも比較にならないほど高いだけでした。 Intelの32nmプロセスに基づくコアアップグレードは、x86パフォーマンスの最大のターニングポイントであり、それ以来、そのようなブレークスルーは見られません。 AMDがRyzenシリーズで名声を上げるには、さらに8年を要します。インテルは最高の製品の成功をうまく利用して、チャンピオンを獲得しました。

この基本設計では、インテルはイノベーションを惜しみませんでした。重要な要素の1つは、マイクロオペレーションキャッシュでした。これは、再デコードにエネルギーを浪費するのではなく、新たにデコードされた命令が再び必要とされ、すでにデコードされていることを意味します。 IntelとSandy Bridgeの場合、および後にAMDとRyzenの場合、マイクロオペレーションキャッシュを有効にすることは、シングルスレッドパフォーマンスの奇跡でした。 Intelは同時マルチスレッディング（数世代にわたってハイパースレッディングと呼ばれていた）の改善も始めており、コンピューティングスレッドの動的割り当てに徐々に取り組んでいます。

発売時の最高のプロセッサであるCore i7-2600Kのクアッドコア設計は、Ivy Bridge、Haswell、Broadwell、Skylake、Kaby Lakeを含む、次の5世代のIntelアーキテクチャの製品の基礎となりました。 Sandy Bridge以降、Intelはより小さなワークフローに切り替えて低消費電力を利用しましたが、企業はチームのネット帯域幅でこの例外的な飛躍を再現することができませんでした。その後、年間の成長率は1〜7％でした。これは主に、動作バッファー、実行ポート、およびコマンドサポートの増加によるものです。



IntelはSandy Bridgeの画期的な成果を再現できず、コアマイクロアーキテクチャーがx86パフォーマンスの鍵であったため、Core i7-2600K（2つ購入）を購入したユーザーは、長い間、それに乗り続けました。主にパフォーマンスの別の大きなジャンプの期待のためです。そして、何年にもわたって彼らの不満は高まっています。クアッドコアのSandy Bridge Core i7-2600Kがまだ5.0 GHzにオーバークロックされているときに、クロック周波数が4.7 GHzのクアッドコアKaby Lake Core i7-7700Kに投資するのはなぜですか。
（Intelの回答は通常、消費電力と、PCIe 3.0を介したGPUやドライブなどの新機能に関連しています。しかし、一部のユーザーはこれらの説明に満足していません。）

それがCore i7-2600Kが世代を定義した理由です。それは、最初はIntelの喜びに、そしてユーザーが更新したくないときの失望に、最初から有効でした。現在、2019年に、インテルはすでにメインプロセッサの4つのコアを超えていることを理解しています。ユーザーがDDR4に高すぎる場合は、新しいIntelシステムに切り替えるか、AMDパスを選択できます。しかし、Core i7-2600Kが2019のワークロードとゲームをどのように処理するかという問題があります。より正確には、オーバークロックされたCore i7-2600Kはどのように対処しますか？

違いを見つける：Sandy Bridge、Kaby Lake、Coffee Lake


実際、Core i7-2600Kは最速の主流のSandy Bridgeプロセッサではありませんでした。数か月後、インテルはやや「高周波」の2700Kを市場に投入しました。ほぼ同じように機能し、2600Kと同じように高速化されましたが、コストは少し高くなります。このときまでに、パフォーマンスが飛躍的に向上し、アップグレードしたユーザーは、すでに2600Kであり、それを維持していました。

Core i7-2600Kは、ハイパースレッディングテクノロジーを搭載した32 nmクアッドコアプロセッサーで、ベース周波数は3.4 GHz、ターボ周波数は3.8 GHz、公称TDPは95ワットでした。その後、IntelのTDPはまだ現実から離れていませんでした。この記事のテストでは、クロックされていないCPUでのピーク消費電力が88 Wであることがわかりました。プロセッサには、Intel HD 3000グラフィックスが組み込まれており、デフォルトでDDR3-1333メモリをサポートしています。インテルはチップの発売時に317ドルの価格を設定しました。

この記事では、最初に登場したときに購入した2番目のi7-2600Kを使用しました。これは、標準周波数でテストされ、すべてのコアで4.7 GHzにオーバークロックされました。これは平均的なオーバークロックです。これらのチップの最高のものは、日常的なモードで5.0 GHz〜5.1 GHzの周波数で動作します。実際、私は最初のCore i7-2600Kがどのようにしてすべてのコアで5.1 GHz、さらに5.3 GHz（すべてのコアでも）で動作したかをよく覚えています。約2Cの温度で、強力な液体クーラーと720mmラジエーターを使用しました。残念ながら、時間の経過とともにこのチップに損傷を与え、現在では公称の周波数と電圧でもロードできません。したがって、あまり良くはありませんでしたが、オーバークロックされたプロセッサについてのアイデアを提供することができた2番目のチップを使用する必要があります。オーバークロック時には、オーバークロックされたメモリ、DDR3-2400 C11も使用しました。
Core i7-2600Kの発売以来、Windows 7からWindows 10に切り替えたことに注目してください。Corei7-2600KはAVX2命令をサポートしておらず、Windows 10用に作成されたものではないため、これが結果にどのように表示されるかを確認することは特に興味深いでしょう。




Core i7-7700K：HyperThreadingテクノロジーを搭載した最新のIntel Core i7クアッドコアプロセッサー

Intelによってリリースされた、ハイパースレッディングを備えた最速かつ最新の（そして最新の？）クアッドコアプロセッサーは、Kaby LakeファミリーのメンバーであるCore i7-7700Kでした。このプロセッサーは、インテルの改良された14nmプロセステクノロジーに基づいて構築されており、4.2 GHzのベース周波数と4.5 GHzのターボ周波数で動作します。テストでの定格電力91ワットのTDPは、95ワットの消費電力を示しました。 Intel Gen9 HD 630グラフィックスが付属し、標準のDDR4-2400メモリをサポートします。 Intelは339ドルの宣言された価格でチップをリリースしました。
7700Kに加えて、Intelはハイパートレッド機能を備えた最初のオーバークロックデュアルコアプロセッサ、Core i3-7350Kもリリースしました。このレビューの過程で、このようなCore i3をオーバークロックし、工場のパラメーターでそれをCore i7-2600Kと比較して、Intelが古いクアッドコアフラグシップと同様のデュアルコアプロセッサーパフォーマンスを達成できたかどうかの質問に答えようとしました。その結果、i3はシングルスレッドのパフォーマンスで優勢であり、メモリを操作していましたが、いくつかのコアコアがないため、ほとんどのタスクはCore i3にとって非常に困難になりました。




Core i7-9700K：Intel Core i7の最新トップ（現在は8コア）

最新のテストプロセッサはCore i7-9700Kです。現在の世代では、Coffee Lake（現在はi9-9900K）のフラッグシップではありませんが、ハイパートレッドのない8つのコアがあります。コアとスレッドの数が2倍の9900Kとの比較は、特にi9の価格が488ドルの場合は無意味です。対照的に、Core i7-9700Kは、ベース周波数が3.6 GHz、ターボ周波数が4.9 GHzで、「たった」374ドルでまとめて販売されます。そのTDPは95ワットでIntelによって定義されていますが、消費者向けマザーボードでは、チップは全負荷で約125ワットを消費します。 DDR4-2666メモリが標準でサポートされています。



Core i7-2600KはDDR3での動作を強制され、PCIe 3.0ではなくPCIe 2.0をサポートし、NVMeドライブ（このテストには含まれません）で動作するようには設計されていません。オーバークロックされたベテランがCore i7-7700Kにどれほど近いか、そしてCore i7-9700Kのようなものに切り替えたときにどのような成長が見られるかは興味深いでしょう。

Sandy Bridge：コアアーキテクチャ


2019年には、最大8つの高性能コアを備え、IntelまたはAMD GlobalFoundries / TSMCプロセステクノロジーの最新バージョンで作成された約100〜200 mm2のチップについて話します。しかし、32 nm Sandy Bridgeはまったく別の獣でした。 FinFETトランジスタがないため、製造プロセスは依然として「フラット」でした。 High-Kの第2世代が新しいCPUに実装され、以前のより大きな45nmプロセステクノロジーと比較して0.7倍のスケーリングが達成されました。 Core i7-2600Kは最大のクアッドコアチップであり、216 mm2あたり11億6,000万個のトランジスタを搭載していました。比較のために、14 nmの最新のコーヒーレイクプロセッサは、8個のコアと20億個を超えるトランジスターを領域〜170 mm2に備えています。

パフォーマンスが大幅に向上する秘訣は、プロセッサのマイクロアーキテクチャにあります。Sandy Bridgeは、前世代のWestmereプロセッサと比較して同等のクロック速度で優れたパフォーマンスを約束（および提供）し、次の10年間のIntelチップの基本回路も形成しました。サンディブリッジの登場により、多くの主要なイノベーションが小売店で最初に登場し、その後多くの反復が繰り返されて改善され、現在使用されている高いパフォーマンスに徐々に達しています。

現在のレビューでは、2010年にリリースされた2600Kマイクロアーキテクチャに関するAnandtechの初期レポートに大きく依存していました。もちろん、このプロセッサーの最新の外観に基づいていくつかの追加が行われています。

短いレビュー：命令の異常な実行を伴うCPUコア


プロセッサーの設計に不慣れな方のために、エクストラプロセッサープロセッサーの仕組みの概要を簡単に説明します。つまり、カーネルは外部インターフェースと内部インターフェース（フロントエンドとバックエンド）に分割され、データは最初に外部インターフェースに送られます。



外部インターフェイスには、メインメモリからの命令を予測して取得するプリフェッチャーと分岐予測子があります。これは、近い将来（必要になる前に）必要なデータと命令を予測できれば、このデータをカーネルの近くに配置することで時間を節約できるという考え方です。次に、命令がデコーダーに配置されます。デコーダーは、バイトコード命令を一連の「マイクロオペレーション」に変換し、カーネルが処理できるようにします。

単純な命令と複雑な命令にはさまざまなタイプのデコーダがあります。単純なx86命令は単一のマイクロオペレーションに簡単にマッピングされ、より複雑な命令はより多くのオペレーションのためにデコードできます。理想的な状況は、デコード係数が可能な限り低いことですが、これらの演算を並列で実行できる場合（コマンドレベルまたはILPでの並列処理）は、命令をより多くのマイクロ演算に分割できる場合があります。

カーネルにマイクロオペレーションキャッシュがある場合、それはuOpキャッシュでもあり、デコードされた各命令の結果がそこに保存されます。命令がデコードされる前に、カーネルはこの特定の命令が最近デコードされたかどうかをチェックし、成功した場合、エネルギーを消費する再デコードの代わりにキャッシュからの結果を使用します。

現在、マイクロオペレーションは「割り当てキュー」-割り当てキューを配置しています。最新のコアは、命令が単純なサイクルの一部であるかどうか、またはuOps（マイクロオペレーション）を組み合わせてプロセス全体を高速化できるかどうかを決定できます。次に、uOpsがカーネルの「バックエンド」を形成するリオーダーバッファーに送られます。

バックエンドでは、リオーダーバッファーから始めて、各マイクロオペレーションに必要なデータの場所に応じてuOpsを再配置できます。このバッファーは、マイクロオペレーションの移動先（整数オペレーションまたはFP）に応じて名前を変更して配布できます。また、カーネルによっては、完了した命令を削除するメカニズムとしても機能します。再注文後、uOpsバッファーは必要な順序でスケジューラーに送信され、データの準備ができていることを確認し、uOpのスループットを最大化します。

スケジューラーは、必要に応じて（計算を実行するために）uOpsを実行ポートに送信します。一部のカーネルは、すべてのポートに対して単一のスケジューラーを備えていますが、場合によっては、整数/ベクトル演算用のスケジューラーに分割されています。異常な実行を伴うほとんどのカーネルには4〜10個のポートがあり、これらのポートは、命令がカーネルを「通過」するために必要な計算を実行します。実行ポートは、ロードモジュール（キャッシュからロード）、ストレージモジュール（キャッシュに格納）、整数演算モジュール、浮動小数点数演算モジュール、およびベクトル演算、特殊除算モジュール、その他特殊演算用の形式をとることができます。 。実行ポートが機能した後、データは、再利用のためにキャッシュに格納され、メインメモリに配置されます。この時点で、命令は削除キューに送信され、最終的に削除されます。

この概要では、トランザクションバッファー、ストリームバッファー、タグ付けなど、最新のカーネルがキャッシュとデータの取得を容易にするために使用するメカニズムの一部は取り上げていません。一部のメカニズムは、世代ごとに繰り返し改善されますが、通常、「命令」について話すときクロックあたりの「パフォーマンスの指標として、カーネルを介して（フロントエンドとバックエンドを介して）できるだけ多くの命令を「スキップ」するよう努めています。このインジケーターは、プロセッサフ​​ロントエンドのデコード速度、命令の事前選択、並べ替えバッファー、および各クロックサイクルで実行される命令の最大数の削除に伴う実行ポートの最大使用に依存します。

上記に基づいて、読者がSandy Bridgeの打ち上げ中に得られたAnandtechテスト結果をよりよく理解できるようになることを願っています。

Sandy Bridge:


Sandy Bridge CPUアーキテクチャは一見すると進化的に見えますが、Nehalem / Westmere以降に変更されたトランジスタの数の点で革新的です。 Sandy Bridge（およびその後のすべてのマイクロアーキテクチャー）の最も重要な変更は、マイクロ操作キャッシュ（uOpキャッシュ）です。



マイクロオペレーションキャッシュがSandy Bridgeに登場しました。これは、命令をデコードした後にキャッシュします。複雑なアルゴリズムはなく、デコードされた命令は単に保存されます。 Sandy Bridgeプリフェッターが新しい命令を受信すると、その命令は最初にマイクロオペレーションキャッシュで検索され、見つかった場合、パイプラインの残りの部分はキャッシュで動作し、フロントエンドは無効になります。ハードウェアのデコードはx86パイプラインの非常に複雑な部分であり、それをオフにすると、かなりの量のエネルギーが節約されます。

これは直接マッピングキャッシュであり、約1.5 KBのマイクロオペレーションを格納できます。これは、実際には6 KBの命令キャッシュと同等です。マイクロオペレーションキャッシュはL1命令キャッシュに含まれており、ほとんどのアプリケーションのヒット率は80％に達しています。マイクロオペレーションキャッシュの帯域幅は、命令キャッシュと比較してわずかに高く、安定しています。実際のL1命令とデータキャッシュは変更されていませんが、それぞれ32 KBです（合計64 KB L1）。

デコーダーからのすべての命令はこのメカニズムによってキャッシュすることができ、すでに述べたように、その中にはいくつかの特別なアルゴリズムがあります。単純に、すべての命令がキャッシュされます。場所がなくなると、長期間使用されていないデータは削除されます。マイクロオペレーションキャッシュは、Pentium 4ではトレースキャッシュのように見えますが、1つの大きな違いがあります。それは、トレースをキャッシュしないことです。これは、マクロ操作（x86命令）ではなく、マイクロ操作を格納する単なる命令キャッシュです。

Intelは、新しいマイクロオペレーションキャッシュとともに、完全に再設計された分岐予測モジュールも導入しました。新しいBPUは以前のBPUとほぼ同じですが、はるかに正確です。精度の向上は、3つの主要な革新の結果です。



標準分岐予測子は2ビット予測子です。各ブランチは、適切な信頼性（強い/弱い）で受け入れられている/受け入れられていないものとして表にマークされています。Intelは、このバイモーダル予測子によって予測されたほとんどすべてのブランチに「高い」信頼性があることを発見しました。したがって、Sandy Bridgeでは、バイモーダル分岐予測子は、各分岐に1つの信頼ビットではなく、複数の分岐に1つの信頼ビットを使用します。その結果、ブランチ履歴テーブルのビット数は以前と同じになり、より多くのブランチを表すため、将来、より正確な予測が可能になります。

サンディブリッジ：コア近く


マルチコアプロセッサの成長に伴い、コアとメモリ間のデータフローの管理は重要なトピックとなっています。クロスバー（上）、リング（上）、メッシュ（メッシュ）トポロジー、そして後で完全に分離したI / Oチップなど、CPU内でデータを移動するさまざまな方法を見てきました。AnandTechが前述したように、次の10年（2020年以降）の戦いは、核間関係の戦いであり、すでに始まっています。
Sandy Bridgeの特徴は、すべてのコア、メモリ、最後のレベルのキャッシュ、および統合されたグラフィックスを接続するリングバスを使用した、Intelの最初のコンシューマCPUであったことです。これは、現代のCoffee Lakeプロセッサーで見られるデザインと同じです。

リングタイヤ


Nehalem / Westmery Bridgeは、L3キャッシュを共有するチップにグラフィックスプロセッサとビデオトランスコーディングエンジンを追加します。そして、L3への配線を増やす代わりに、Intelはリングバスを導入しました。



アーキテクチャ上、これはNehalem EXとWestmere EXで使用されているのと同じリングバスです。各コア、L3キャッシュ（LLC）の各フラグメント、統合グラフィックスプロセッサ、メディアエンジン、およびシステムエージェント（ノースブリッジの面白い名前）は、リングバスに接続されています。バスは、4つの独立したリングで構成されています。データバス、リクエストバス、確認応答バス、ステータスモニタリングバスです。リングのいずれかへの各呼び出しは、サイクルごとに32バイトのデータを送信できます。コアの数とキャッシュのサイズが増加すると、それに応じてキャッシュのスループットが増加します。

各コアの結果は、L3キャッシュの帯域幅は高性能Westmereプロセッサーと同じ96 GB /秒です。Sandy Bridgeの総スループットは、クアッドコアWestmereの4倍です。これは、コアの数を単純に乗じたものであり、384 GB /秒であるためです。

これは、L3遅延がWestmereの約36クロックサイクルからSandy Bridgeの26-31クロックサイクルに大幅に減少することを意味します（どのコアがどのキャッシュフラグメントにアクセスするかに応じて、いくつかの可変キャッシュ遅延があります）。さらに、Westmereとは異なり、L3キャッシュはコアクロック速度で実行されるようになりました。非コアの概念は依然として存在しますが、インテルはオフコア部分を「システムエージェント」と呼び、L3キャッシュを含まなくなりました。（「アンコア」という用語は現在でも相互接続を説明するために使用されています。）

コア周波数で実行されるL3キャッシュのおかげで、はるかに高速なキャッシュの利点が得られます。欠点は、ターボモードまたはアイドルモードがオンになっているときに、L3がプロセッサコアと共に加速することです。コア周波数が低下したときにGPUがL3を必要とする場合、L3キャッシュは独立しているかのように高速に動作しません。または、システムはカーネルをオーバークロックし、追加の電力を消費する必要があります。

L3キャッシュはフラグメントに分割され、各フラグメントは個別のコアに関連付けられています。Sandy Bridgeには完全にアクセス可能なL3キャッシュがあるため、各コアはキャッシュ全体をアドレス指定できます。各フラグメントには、独自のバスアクセスコントローラーと完全なキャッシュパイプラインがあります。Westmereには1つのキャッシュパイプラインとすべてのコアがリクエストを送信するキューがありましたが、Sandy Bridgeではそれらはキャッシュセグメントに分散されています。リングバスの使用は、新しいアクセスポイントをバスに追加することが、マトリックスのサイズに重大な影響を与えるのをやめたことを意味します。リングの各ユーザーが独自のコントローラーを取得するという事実にもかかわらず、データは常に最短経路をたどります。バス制御はリング全体に分散され、その結果、各モジュールはバスに1クロックバックの空きスロットがあったかどうかを「認識」します。

システムエージェント


何らかの理由で、IntelはSBではアンコアという用語の使用をやめ、Sandy Bridgeではこの部分を「システムエージェント」と呼んでいました。（ここでも、アンコアがインターコネクト、I / O、およびメモリコントローラーに対応するようになりました）。システムエージェントは従来のノースブリッジです。16個のPCIe 2.0レーンがあり、2つのx8レーンに分割できます。再設計されたデュアルチャネルDDR3メモリコントローラーがあり、最終的には約リンフィールドレベルのメモリレイテンシを提供します（クラークデールはメモリコントローラーをCPUからGPUに移動しました）。



システムエージェントには、DMIインターフェイス、ビデオ出力ユニット、およびPCU（電源管理ユニット）もあります。SAクロック周波数はコアの他の部分よりも低く、独立した電源回路を備えています。

サンディブリッジグラフィックス


Westmereと比較したSandy Bridgeのもう1つの重要なパフォーマンス改善は、グラフィックス処理です。プロセッサコアのパフォーマンスは10〜30％向上しますが、Sandy Bridgeのグラフィックスパフォーマンスは、Westmere（Clarkdale / Arrandale）より前のIntel製品の2倍になっています。 45 nmから32 nmへのジャンプにもかかわらず、IPCの大幅な増加により、グラフィックス処理速度が向上しています。

Sandy Bridge GPUは、プロセッサコアと同じ32 nmトランジスタで構築されています。 GPUは、電力と周波数の点で独自の世界にあります。 GPUは、プロセッサに関係なく、オフまたはオンにできます。グラフィックターボは、デスクトッププロセッサとモバイルプロセッサの両方で使用できます。



GPUはSandy Bridgeの世界では平等な市民と見なされており、L3キャッシュへの完全なアクセス権を持っています。グラフィックスドライバーは、L3キャッシュに正確に入るものを制御し、GPUで使用可能なキャッシュの量を制限することもできます。キャッシュに画像データを保存すると、パフォーマンスと消費電力の両方でコストがかかるメインメモリへのアクセスが減少するため、特に重要です。しかし、キャッシュを使用するようにGPUを再構築するのは簡単な作業ではありません。

SNB Graphics（内部的にはGen 6）は、固定機能を備えたハードウェアを広範囲に使用します。アイデアはこれです：固定された関数を使用して記述できるすべてのものが、固定された固定関数によって実装される必要があります。柔軟性は失われますが、利点は、パフォーマンス、パワー、およびマトリックスサイズの縮小です。



シェーダープログラマブルハードウェアは、シェーダー/コア/実行ユニットで構成されます。インテルはこれをEUと呼んでいます。各EUは複数のスレッドからの指示を受け入れることができます。内部ISAは、ほとんどのDirectX 10 API命令と1対1でマッピングされます。これは、CISCのようなアーキテクチャを意味します。 1対1のAPIから指示の表示に移行すると、EUの幅を効果的に広げることでIPCが向上します。

EUには他にも改善点があります。超越数学はEUのハードウェアによって処理され、その生産性は大幅に向上しています。その後、Intelは、正弦波と余弦波の演算がWestmere以前のグラフィックスよりも数桁速いことを発表しました。
以前のIntelグラフィックスアーキテクチャでは、レジスタは「オンザフライ」で再配布されました。スレッドが必要とするレジスターが少ない場合、残りのレジスターを別のスレッドに割り当てることができます。これはマトリックス領域を節約するための優れたアプローチでしたが、パフォーマンスを制限するものであることがわかりました。使用できるレジスタがないため、スレッドを処理できないことがよくありました。 Intelでは、スレッドあたりのレジスタ数を64から80に増やし、次にSandy Bridgeの120に増やしました。レジスタの不足によるダウンタイムのシナリオは大幅に削減されました。



つまり、これらすべての機能強化により、EUの命令スループットが2倍になりました。

ローンチ時には、Sandy Bridge GPUの2つのバージョンがありました。1つは6 EUで、もう1つは12 EUです。すべてのモバイルプロセッサ（起動時）は12 EUを使用しますが、デスクトップSKUはモデルに応じて6または12を使用できます。サンディブリッジは、統合グラフィックスがコンシューマ製品に必須となり、インテルが徐々にGPUに割り当てられるチップ領域の割合を増やし始めたため、インテルにとって正しい方向への一歩でした。同様のレベルの最新（2019）デスクトッププロセッサーは24 EU（Gen 9.5）ですが、将来の10 nmプロセッサーは〜64 EU（Gen11）になります。

Sandy Bridgeメディアエンジン


Sandy Bridge GPUの隣にはメディアプロセッサがあります。 SNBでのマルチメディア処理は、ビデオのデコードとビデオのエンコードという2つの主要コンポーネントで構成されています。

ハードウェアアクセラレーションによるデコードメカニズムが現在の世代に比べて改善されました。現在、ビデオパイプライン全体が、固定機能のモジュールを使用してデコードされています。これは、ビデオデコードのいくつかの段階でEUアレイを使用するSNBの前のIntelの設計とは対照的です。その結果、Intelは、HDビデオを再生すると、SNBプロセッサの消費電力が半分になると主張しています。



ビデオエンコーディングエンジンは、Sandy Bridgeに完全に新しく追加されました。Intelは1080p 30 Mbpsの3分程度のソースビデオを取り、それをiPhone 640 x 360ビデオ形式にトランスコードしました。プロセス全体は14秒かかり、約400フレーム/秒で終了しました。



固定機能のエンコード/デコードの原理は、デスクトップコンピュータやスマートフォンのグラフィック機器でも一般的になりました。当時、Sandy Bridgeはこの基本的なコーディング/デコード構造に3 mm2のマトリックスを使用していました。

新しい、攻撃的なターボ


リンフィールドは、アイドルコアを無効にするときにアクティブプロセッサコアのクロック周波数を動的に増加させるというアイデアを積極的に推進した最初のIntelプロセッサでした。クアッドコアプロセッサ用に95 W TDPがあり、これらの4つのコアのうち3つがアイドル状態の場合、ターボ制限に達するまで1つのアクティブコアのクロック速度を上げることができます。

現在の世代のすべてのプロセッサーでは、プロセッサーはターボをオンにした直後にターボ出力の限界に達すると想定されています。ただし、実際には、プロセッサはすぐには加熱されません。プロセッサが完全な電力消費とゲイン温度を消費しない期間があります。



Sandy Bridgeはこの機能を利用して、PCUがTDPを超えるアクティブコアを短時間（最大25秒）加速できるようにします。 PCUは、ダウンタイム中に利用可能なサーマルバジェットを監視し、プロセッサ負荷が増加したときにそれを使います。プロセッサがアイドル状態でいる時間が長いほど、TDPを超えるマージンが大きくなります。その結果、ワークロードが発生すると、中央プロセッサーは過剰なTDPでターボをオンにし、プロセッサーが加熱すると再び周波数を下げ、最終的にはそのTDPで停止します。 SNBはTDPを超える可能性がありますが、PCUはチップが信頼性の制限を超えることを許可しません。



CPUとGPU Turboの両方が連携して動作できます。 SNBベースのGPUとの関連性が高いワークロードは、CPUコアの低下とGPU周波数の上昇につながる可能性があります。また、CPUに関連するタスクは、GPUの周波数を下げ、CPUの周波数を上げることができます。 Sandy Bridgeは全体として、それ以前に作成されたすべてのものよりもはるかに柔軟なメカニズムであることが判明しました。
