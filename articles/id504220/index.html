Layanan Verifikasi Tajuk HTTP Server

Untuk situs apa pun, penting untuk mengkonfigurasi tajuk HTTP dengan benar. Banyak artikel telah ditulis tentang masalah judul. Di sini kami telah merangkum pelajaran yang dipetik, dokumentasi RFC. Beberapa judul adalah wajib, ada yang usang, ada yang bisa membingungkan dan kontradiktif. Kami melakukan parsing untuk secara otomatis memeriksa header HTTP dari server web . Tidak seperti banyak layanan lain yang hanya menampilkan tajuk, layanan ini memungkinkan Anda untuk:

  1. mengatur nilai header khas;
  2. tambahkan tajuk ubahsuaian Anda sendiri;
  3. menunjukkan versi protokol HTTP: 1.0, 1.1, 2 (memeriksa apakah HTTP / 2 didukung);
  4. tentukan metode permintaan, batas waktu, dan data postdata untuk dikirim ke server;
  5. puzomerka juga memeriksa apakah respons terhadap permintaan If-Modified-Because, If-None-Match benar jika respons server berisi Last-Modified atau ETag.

Kami tidak mengklaim sebagai kebenaran tertinggi. Untuk konten individual dan untuk proyek individual, tentu saja, mungkin ada penyimpangan. Tetapi layanan ini pasti akan memberi tahu Anda apa yang harus Anda perhatikan, mungkin bermanfaat bagi Anda untuk mengedit tajuk Anda. Berikut ini adalah daftar yang diperhatikan oleh layanan verifikasi. Mengapa demikian, baca artikel di Habre .

Tajuk Wajib


  • Tanggal
  • Content-Type menentukan charset untuk konten teks lebih disukai utf-8
  • Kompresi Pengodean Konten untuk Konten Teks

Header usang dan tidak perlu


  • Server dengan versi terperinci dari server web
  • X-power-by
  • X_ASPNET-Versi
  • Kedaluwarsa
  • Pragma
  • P3p
  • Melalui
  • Kompatibel dengan X-UA

Tajuk yang Diinginkan untuk Keamanan


  • X-Content-Type-Options
  • X-XSS-Perlindungan
  • Ketat-transportasi-keamanan
  • Kebijakan rujukan
  • Kebijakan fitur
  • Konten-Keamanan-Kebijakan atau Konten-Keamanan-Kebijakan-Hanya-Laporan untuk menonaktifkan skrip dan gaya inline.

Caching Header


Diperlukan untuk konten statis dengan istilah cache yang panjang dan sangat diinginkan untuk konten dinamis dengan istilah cache pendek.

  • Terakhir diubah
  • ETag
  • Kontrol cache
  • Berbeda
  • Penting agar server merespons dengan benar tajuk: If-Modified-Sejak dan If-None-Match

HTTP / 2


Sekarang server harus mendukung HTTP / 2. Secara default, layanan memeriksa server untuk HTTP / 2. Jika server Anda tidak mendukung HTTP / 2, maka pilih HTTP / 1.1.
