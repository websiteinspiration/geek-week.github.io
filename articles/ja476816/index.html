スマートフォンのドバイモール、または建物のフロアプランをアプリケーションに追加する方法



では前の記事、私はマップとモバイルアプリケーションを作成する方法について話しました。 「膝の上」シリーズの続きでは、フロアプランを実装するためのツールを紹介します。

問題の簡単な説明：モバイルアプリケーションでフロアプランを視覚化し、その上に特定の組織の場所を示すことができます。ユーザーの場所も確認したいのですが、問題は技術面にあります。屋内でデバイスの座標を取得できる機器が必要です。したがって、この側面は記事の範囲外とし、ソフトウェアの部分に焦点を当てます。

以下に、上記の要件を実装できるいくつかのオプションを示します。それはすべて、あなたがどのようなデータを持っているか、そしてアプリケーションが正確に何ができるべきかによって異なります。そして、最も単純なものから始めます。

最初のオプション。データを備えたレディAPI


検討する最初のオプションは、2GISからの既製のウィジェットの使用です。APIの説明はapi.2gis.ruにあります。興味のある建物がすでに2GISで提示されていて、床が建物にすでに描かれている場合に適しています。つまり、データに関しては、すべてがすでに行われています。そして最も重要なのは、このオプションがインターネットでのみ機能するため、オンラインの準備ができていることです。

この場合、床を表示するには、実際には何も必要ありません。WebViewに配置する必要があるWebウィジェットとしての実行の実装。

<WebView
   android:layout_width="match_parent"
   android:layout_height="match_parent"
   android:id="@+id/am_webview"
   android:layout_marginTop="160dp"
/>

これは、ウィジェットを配置するコンテナです。次のように初期化します。

webView = findViewById(R.id.am_webview)
webView.settings.javaScriptEnabled = true
webView.settings.useWideViewPort = true
webView.settings.loadWithOverviewMode = true
//webView.settings.setSupportZoom(true)
//webView.settings.displayZoomControls = true
//webView.settings.builtInZoomControls = true

webView.loadUrl("file:///android_asset/map_api.html")
webView.webViewClient = object : WebViewClient() {
	override fun onPageFinished(view: WebView, url: String) {
		super.onPageFinished(view, url)
		val js = "initMap('13933647002593772');" // The Dubai Mall

		webView.evaluateJavascript(js, ValueCallback {
		})
	}
}

webView自体を設定します。JavaScriptを介してウィジェットとやり取りするため、JavaScriptを確実に有効にしてjavaScriptEnabled = trueにします。必要に応じて、スクロールバーとズームボタンをオンにすることができます（コメント付きのコードを参照）。ただし、うまく機能しないため、お勧めしません。

最も重要なことは、ウィジェットwebView.loadUrl（ "file：///android_asset/map_api.html"）でHTMLをロードし、必要に応じてイベントをサブスクライブすることです。上記の例では、地図を読み込んだ後、map_api.htmlで定義されているinitMapメソッドを呼び出し、床を表示する建物の識別子を渡します。

HTMLはかなり単純なコードです。DG.FloorsWidget.initメソッドが呼び出されます、初期化するデータを含むjsonオブジェクトが渡されます。コンテナーとして、上記のHTMLマークアップでdivを宣言したIDを指定します。幅、高さを調整します。また、initDataでは、complexIdタグで建物を渡し、APIドキュメントにある追加のウィジェット表示パラメーターを渡します。あなたは上の構築に興味があるクリックすると2GISが送信する識別子は、方法によっては、検索クエリに応答して覗くことができます2gis.ru。私の例では、ドバイモールを使用しました。しかし、床を備えた他の建物を示すことを気にする人はいません。

仕上げタッチ。特定の会社に移動するには、showFirmメソッドを呼び出して会社IDを渡します

webView.loadUrl("javascript:showFirm('$firmId')")


とても簡単です。既製の実装例はGithubで見ることができます。

2番目のオプション。絵としてのフロアプラン


データとAPIの準備ができていないオプションが適切でない場合は、以下を使用できます。

この場合、写真、たとえばjpegまたはpngの形式の平面図が必要になります。 「画像に突き刺さる-オブジェクトカードを開く」タイプの対話機能が必要な場合は、ジオコーディングも必要になります。ジオコーディングは個別に実装する必要があります。グローバル座標にスナップする場合は、画像を正しくスケーリングし、コーナーの1つを実際の座標に引っ張って、そこからの変位を計算できるようにする必要があります。この問題については詳しく説明しません。すべてが明確になっていることを願っています。

最も重要なことは、アプリケーションでこの画像を表示することです。このため、TileViewライブラリは最適です。。実際、このライブラリでは、原則として、マップタイルを表示できます。しかし、移動とズーム、指定された位置でのセンタリング、マーカーの表示、座標変換、およびさまざまなイベントをサブスクライブする機能を提供するため、私たちにも適しています。

ライブラリをできるだけ効率的に機能させるには、元の画像をタイルにカットして準備する必要があります。これにはかなり単純な指示があります。指定したページの最後に、4つのタイルセットを作成するスクリプトをお勧めします。取得した結果をアセットに追加し、単純なコードでアクティビティに画像を表示します。

tileView.setSize(floorWidth, floorHeight)
tileView.setShouldRenderWhilePanning(true)

tileView.addDetailLevel(1f, "tiles/floors1/1000/%d_%d.jpg")
tileView.addDetailLevel(0.500f, "tiles/floors1/500/%d_%d.jpg")
tileView.addDetailLevel(0.250f, "tiles/floors1/250/%d_%d.jpg")
tileView.addDetailLevel(0.125f, "tiles/floors1/125/%d_%d.jpg")

tileView.defineBounds(0.0, 0.0, floorWidth.toDouble(), floorHeight.toDouble())
tileView.setScaleLimits(0f, 5f)
tileView.setMinimumScaleMode(ZoomPanLayout.MinimumScaleMode.FIT)

すべて、ディスプレイの準備が整いました。イベントをサブスクライブして、必要なロジックを追加できます。たとえば、マーカーは次のように表示できます。

tileView.setMarkerAnchorPoints(-0.5f, -0.5f)
tileView.addMarker(imageView, x, floorHeight - y, null, null)

完全な例はGithubで入手できます。

3番目のオプション。ベクトルデータ


このオプションは、最も高度で最も難しいものです。これは、ベクターデータ、つまり、ベクターに完全に描かれた床が準備されていることを前提としています。いくつかのタイプのオブジェクトが必要になります。組織、駐車場、フードコート、ステージ、スケートリンクなどのエリア。線形オブジェクト-主に壁、流れ方向。ポイントオブジェクト：入力/出力、エレベーター、ATMなど。

内部の2GISシステムであるフィジーでのフロアプランは次のようになります。



まあ、その視覚化には、前回の記事で説明したベクターエンジンであるmapsforge-vtmが適しています。

アプローチを実証するために、日当たりの良いキプロス島の建物の例を使用して、いくつかのフロアの正方形と線のセットであるテストデータを準備しました。準備のために、建物の元のジオメトリを取得し、単純にするために、ジオメトリのコンポーネントに対応する部分にカットしました。ご存知のように、最も難しいのは高品質のデータの準備です。残りは技術の問題です。床を切り替えるためのボタン、さまざまな正方形や線をレンダリングするための準備済みのスタイル、それらをレンダリングするためのオーバーレイが必要です。ここで

完全なコードを参照してください。FloorData クラスには床のテストジオデータコードが含まれており、FloorsManagerクラスはそれらをレンダリングするように設計されています。 コンストラクターで、正方形と壁のスタイルを定義します。





styles.put(ObjectType.Floor, org.oscim.layers.vector.geometries.Style.builder()
                .fillColor(Color.GRAY)
                .build());

そして、drawFloorメソッドで、マップ上のレイヤーにオブジェクトを追加するロジックを決定します。

public void drawFloor(int floorId) {
	hideFloors();

	indoorLayer = new CustomVectorLayer(this.map);
	List<GeoData> geoObjects = this.floorData.getFloorData(floorId);

	for (GeoData geo: geoObjects) {
		indoorLayer.add(geo.getGeometry(), styles.get(geo.getObjectType()));
	}

	this.map.layers().add(indoorLayer);
	indoorLayer.update();
}

ここではすべてが初歩的です。新しいinteriorLayerレイヤーを作成し、事前に準備されたフロアデータに必要なスタイルを追加して、レイヤーを地図this.map.layers（）に追加します（indoorLayer）を追加します。

フロアを切り替えるためのボタンを追加することは残ります。これを行うには、そこにあるFloorPickerControlに基づいてRecyclerViewあなたが必要なものだけでなく、。時間を無駄にしないようにしましょう。ソースを参照してください。
