Kotlin Basic Androidのヒント


遅延読み込み


遅延読み込みにはいくつかの利点があります。たとえば、次のようなものです。

  • 一部のデータの読み込みが必要になるまで遅延されるため、起動時間を節約できます。これは、Androidの場合に特に当てはまります。ダウンロードタップを起動して見るまでの待ち時間が長いのではなく、ユーザーがアプリケーションのコンテンツをより速く表示できるためです。
  • リソースはリクエストに応じてのみデバイスのメモリに読み込まれるため、メモリを節約できます。携帯電話のリソースは非常に限られているため、これはモバイルプラットフォームにとって特に重要です。

たとえば、ユーザーが主に選択によってガイドされるショッピングアプリケーションを作成している場合は、次のように、実際のショッピングAPIで遅延読み込みを構成できます。

val purchasingApi: PurchasingApi by lazy {
    val retrofit: Retrofit = Retrofit.Builder()
            .baseUrl(API_URL)
            .addConverterFactory(MoshiConverterFactory.create())
            .build()
    retrofit.create(PurchasingApi::class.java)
}

このような遅延ダウンロードを使用した結果、ユーザーがアプリケーションで購入を試みなかった場合、ダウンロードせず、要求されていないリソースを使用することになります。

また、遅延読み込みは、初期化ロジックをカプセル化するための優れた方法です。次に例を示します。

val bounds: RectF by lazy { 
    RectF(0f, 0f, width.toFloat(), height.toFloat()) 
}

最初の呼び出しが行われるとすぐに、ビューの現在の幅と高さに基づいてRectFのインスタンスが作成されるため、個別に作成してこのオブジェクトを設定する必要がなくなります。

カスタムゲッターとセッター


カスタム設定を読み取るために、Kotlin言語は、ユーザーの動作が定義された構造モデルを使用して、フィールドを取得および設定します。Parse SDKなどの特定のフレームワークのモデルを使用する場合、ローカルクラス変数ではないが、たとえばJSONからカスタムメソッドを使用して取得および保存される値を選択します。

ユーザー定義の取得およびインストール方法を使用して、アクセスを簡素化できます。次に例を示します。

@ParseClassName("Book")
class Book : ParseObject() {

    // getString() and put() -  ParseObject
    var name: String
        get() = getString("name")
        set(value) = put("name", value)
    var author: String
        get() = getString("author")
        set(value) = put("author", value)
}

これらの値の取得は、次のような他のモデルでのプロパティアクセス構文の使用と似ています。

val book = api.getBook()
textAuthor.text = book.author

これで、モデルをParseから他のデータソースに転送する必要がある場合、コードを変更する必要はほとんどありません。

ラムダ


ラムダは、ソースファイルのコード行の重複を減らし、関数型プログラミングを使用できるようにします。現在ラムダはAndroidで使用されていますが、Kotlinはそれらを次のレベルに引き上げ、Retrolambdaを処理したり、構築方法を変更したりする必要がないようにします。

たとえば、ファイルをリッスンすると、次のようになります。

button.setOnClickListener { view ->
    startDetailActivity()
}

そして、これは戻り値を処理する方法です：

toolbar.setOnLongClickListener { 
    showContextMenu()
    true
}

Android SDKでは、これらのメソッドのいずれかを実装する必要があることがよくあります。ラムダはこれを強打で対処します。

データクラス


データクラスは自動的にメソッドを追加し、クラスを簡素化するequals()、hashCode()、copy()、とtoString()。モデルで使用するデータを明確にし、ロジックからデータを分離します。

たとえば、次のようなデータクラスがあります。

data class User(val name: String, val age: Int)

それで全部です。彼の仕事にはこれ以上何も必要ありません。Gsonまたは別のJSONタイプライブラリに類似したデータクラスを使用する場合、デフォルト値を使用してデフォルトコンストラクタを作成できます。次に例を示します。

data class User(
    @SerializedName("name") val name: String = "",
    @SerializedName("age") val age: Int = 0
)

データセットのフィルタリング


APIを使用する場合、コレクションを常に処理する必要が生じます。ほとんどの場合、これらはフィルタリングするか、内容を変更する必要があります。

Kotlinのフィルタリングコレクションを使用すると、コードをよりシンプルでわかりやすくすることができます。次のように、コレクションをフィルタリングすることで、結果リストの内容を指定できます。

val users = api.getUsers()
val activeUsersNames = items.filter { 
    it.active
}
adapter.setUsers(activeUsers)

Kotlinの組み込みのコレクションフィルタリングメソッドは、Java 8などの他のプログラミング言語で使用されているメソッドや、Swiftコレクションタイプを操作する場合に非常によく似ています。コレクションをフィルタリングする統一された方法は、要素の必要なリストを取得して表示するために特定の操作を実行する必要性について従業員と通信するときの相互理解を簡素化します。
