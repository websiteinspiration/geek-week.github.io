自動化されたエンジニアのアパート。パート1：問題の説明、鉄の選択

数ヶ月間、私はコンクリートの2部屋の箱の所有者になりました。アパートを取得した後、修理のリクエストは最小限でした-可能であれば、自分の手で最小限のお金で居心地の良さと快適さを実現するためです。その結果、私はデザイナーに快適さを求めなければならず、快適さのコンセプトにはスマートホームの要素が含まれていました。

スマートホームの概要の最初のバージョンは、床下暖房と中央電気接触器の制御にまで至りました。しかし、ご存じのとおり、食欲は食物とともに成長します。現在、私たちのリストは次のポイントに成長しています。

  • 光制御;
  • 空調（床暖房、ラジエーター、エアコン）。
  • カーテン管理;
  • 漏出保護システム管理;
  • 水と電気の消費量の自動計測;
  • マルチルーム統合;
  • TVコントロール;
  • スマートフォンからすべてを制御する機能。
  • ゲストは不快感を経験するべきではありません、照明要素の制御は通常のアパートのようでなければなりません

ご覧のように、ウィッシュリストのリストは大幅に拡大しました。これに関連して、実装にアイロンを選択するという問題が深刻化しました。私は産業オートメーションエンジニアとして、ArduinoやXiaomiが提供するあらゆる種類の「スマート」Wi-Fi搭載デバイスなどのオプションをすぐに却下しました。I / Oモジュールと有線スイッチを備えたプレハブPLCのみ。

数か月にわたってあらゆる種類の情報を調査した結果、PLCをOpenHABまたはHome Assistantスマートホームプラットフォームと組み合わせて使用​​するのが最適であるという結論に達しました。しかし、今やその礎石はPLCの選択です。選択はいくつかのオプションから来ています。

オプション1：Segnetics Matrix + FMR I / Oモジュール


予算内オプション。



長所：

  • シンプルなSMLogix開発環境
  • フォーラムとユーザーコミュニティがあります。

マイナス：

  • 組み込みのWebサーバーがない。
  • MQTTを引き締めることが可能かどうかは不明です。
  • コントローラーは新品です。

オプション2. ARIES PLC210 +入出力モジュールMV210、MU210


予算内オプション。



長所：

  • 統合WEBサーバーを備えたCodesys 3.5開発環境。
  • MQTTのパースペクティブサポート。
  • フォーラムとコミュニティがあります

マイナス：

  • 新しい非ロールコントローラー、側枠が可能です。
  • 製造元への信頼はありません（プロジェクトの実施には否定的な経験があります）。

オプション3. PLCおよびBerghof I / Oモジュール


高予算オプション。



長所：

  • Codesys 3.5開発環境
  • 信頼できるテスト済みコントローラー。
  • EtherCAT I / Oモジュール。

マイナス：

  • 価格（リストされている最も高価なオプション）;
  • MQTTはサポートされていません。

オプション4：WirenBoard 6およびI / Oモジュール


低予算オプション。



長所：

  • 価格（リストされている最も安いオプション）;
  • MQTTを含むさまざまなスマートホームプロトコルのサポートがあります。
  • スマートホームに必要なすべてのモジュールはメーカーから入手できます。
  • モジュール式の実行には、プラストロンを備えた標準キャビネットが必要です。

マイナス：

  • 組み込みWEBサーバーの設計は構成できません。
  • JavaScriptベースのプログラミング言語（新しい環境が私の​​ためにあるため、すべてのアイデアを実装するのは難しいでしょう）。
  • プログラミングと構成に関するドキュメントはほとんどありません。
  • ユーザーコミュニティはありません。

概して、現在の4つのオプションのうち、AriesとWirenBoardのどちらかを選択します。WirenBoardの側には、豊富なプロトコルとスマートホーム向けの主張されたシャープ化があり、Ariesの側では、Codesysでの私の経験と、OpenHABなしのWEBインターフェイスからの制御の保証された実装が保証されています。
