锦标赛排序的主要思想是使用相对较小（与主数组相比）的辅助堆，该堆充当优先级队列。在此堆中，将对较低级别的元素进行比较，结果是较小的元素（在本例中为MIN-HEAP树）上升，并且落入该堆的数组元素部分的当前最小值弹出到根。最小值被转移到附加的“优胜者”数组中，结果堆将比较/移动其余元素-现在树的根中有了新的最小值。请注意，在这样的系统中，下一个最小值比上一个最小值更大-然后可以轻松地组装一个新的有序“赢家”数组，在其中将新的最小值简单地添加到附加数组的末尾。



将最小值转移到附加数组会导致以下事实：主数组下一个元素的空位出现在堆中-结果所有元素都按队列顺序处理。



主要的微妙之处在于，除了“赢家”之外，还有“输家”。如果某些元素已经转移到“优胜者”数组中，那么如果未处理的元素小于这些“优胜者”数组，则在此阶段将它们筛选通过锦标赛树是没有意义的-将这些元素插入“优胜者”数组将太昂贵（在您不能结束它们，但要将它们放在开头，则必须移动已经插入的低点。这些无法放入“获胜者”阵列中的元素将被发送到“失败者”阵列中-当对其余失败者重复所有动作时，它们将通过以下迭代之一遍历锦标赛树。



在互联网上很难找到该算法的实现，但是在YouTube上却找到了清晰，漂​​亮的Ruby实现。在“链接”部分中，仍然提到了Java代码，但是在这里看起来不太可读。

  def bracketize(array)
	
    size = array.size
    pq = PriorityQueue.new

    pq.add(array.shift) until pq.size == MAX_SIZE
    winners = [] #  ""
    losers = [] #  ""

      #  ""  ?

        #      ""

    array = losers + winners
    array.pop while array.size > size
    bracketize(array) #

  def optimal_tourney_sort(array)
    sorted_array = [] #    
    pq = PriorityQueue.new
    array.each { |num| pq.add(num) } #     -
    until pq.heap.empty? #  -  
      sorted_array << pq.heap[0] 
      pq.remove #     
    end
    sorted_array #  
  end

这是一个幼稚的实现，其中，在将每个数组分为“失败者”和“赢家”之后，将这些数组组合在一起，然后针对组合后的数组再次重复所有操作。如果“失败”元素将首先进入组合数组，那么在锦标赛堆中进行筛选会将其与“获胜者”一起排序。









此实现非常简单直观，但是您不能称之为有效。排序的“优胜者”不止一次地进入锦标赛树，这显然是不必要的。我假设这里的时间复杂度是对数二次方O（n log 2 n）。



多路径合并选项



如果通过筛子的有序元素没有反复通过比赛，则算法将明显加速。



在将无序数组分为已排序的“赢家”和未排序的“失败者”之后，整个过程仅针对“失败者”重新进行。在每个新的迭代中，将累积一组具有“获胜者”的数组，并将以这种方式继续进行，直到任何步骤都没有“失败者”。之后，仅合并所有“优胜者”阵列。由于所有这些数组都已排序，因此可以以最小的成本进行合并。









以这种形式，算法可能已经找到了有用的应用程序。例如，如果您必须处理大数据，那么在使用锦标赛堆的过程中，将释放有序数据包，您可以在处理其他所有事情时使用它们进行某些操作。数组



的n个元素中的每个元素仅穿过锦标赛树一次，这在时间上花费O（log n）。在这样的实现中，该算法具有最差/平均时间复杂度O（n log n）。
