2020年のReactフォームの作成

こんにちは、ハブル。Kristofer Selbekkによる記事「2020年のReactでのフォームの作成」の翻訳を紹介します




入力フィールド テキスト領域。ラジオボタンとチェックボックス。ここでは、開発者としてのユーザーとのやり取りの主なポイントをいくつか示します。私たちはそれらをサイトに配置し、ユーザーが記入し、私たちが運が良ければ、完成したフォームは検証エラーなしで私たちに送られます。

フォーム処理は多数のWebアプリケーションに不可欠な部分であり、これはReactが最も得意とすることの1つです。フォームを作成および処理するための大きな自由があります。しかし、これを行うより良い方法はありますか？

これらのすべての例では、電子メールとパスワードのフィールドを持つログインフォームを作成しますが、これらのメソッドはほとんどのタイプのフォームで使用できます。

良い形のルールを覚えてください


これは議論中のトピックとは直接関係ありませんが、フォームを誰にでも理解できるようにすることを忘れないようにしたいと思います。入力フィールドにタグを追加し、入力が無効な場合に適切なaria属性を設定し、コンテンツを意味的に正しく構成します。これにより、ユーザーと開発者の両方がフォームを使いやすくなります。

状態フックを使用したフォームの処理


はじめに、通常はフォームの状態をどのように処理するかを見てみましょう。すべてのフィールドを個別の状態要素として保存し、それらをすべて個別に更新します。これは次のようになります。

function LoginForm() {
  const [email, setEmail] = React.useState('');
  const [password, setPassword] = React.useState('');

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    api.login(email, password);
  };
  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label htmlFor='email'>Email</label>
        <input type='email' id='email' value={email} onChange={(e) => setEmail(e.target.value)} />
      </div>
      <div>
        <label htmlFor='password'>Password</label>
        <input type='password' id='password' value={password} onChange={(e) => setPassword(e.target.value)} />
      </div>
    </form>
  );
}

まず、ユーザー名とパスワードという2つの状態の部分を作成します。これら2つの変数は、対応する入力フィールドに転送され、このフィールドの値が決定されます。フィールド内の何かが変更されるたびに、必ず状態値を更新し、アプリケーションを再描画します。

このフォーム処理はほとんどの場合正常に機能し、シンプルで使いやすく、理解しやすくなっています。ただし、そのような構成を毎回書くのはかなり退屈です。

カスタムフックを作成する


少しリファクタリングして、独自のフックを作成しましょう。これにより、コードが少し改善されます。

const useFormField = (initialValue: string = '') => {
  const [value, setValue] = React.useState(initialValue);
  const onChange = React.useCallback((e: React.ChangeEvent<HTMLInputElement>) => setValue(e.target.value), []);
  return { value, onChange };
};

export function LoginForm() {
  const emailField = useFormField();
  const passwordField = useFormField();

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    api.login(emailField.value, passwordField.value);
  };
  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label htmlFor='email'>Email</label>
        <input type='email' id='email' {...emailField} />
      </div>
      <div>
        <label htmlFor='password'>Password</label>
        <input type='password' id='password' {...passwordField} />
      </div>
    </form>
  );
}

カスタムのuseFormFieldフックを作成して、onChangeイベントハンドラーを作成し、値をフォームの状態に格納します。この場合、すべてのフォームフィールドでフックを使用できます。

多数のフィールドを処理する


このアプローチの短所は、フォームが大きくなるにつれて適切にスケーリングされないことです。ログインのような小さなフォームの場合、これはおそらく良いことですが、ユーザープロファイルフォームを作成するときに、多くの情報を要求する必要がある場合があります。フックを何度も呼び出す必要がありますか？

同様の状況に遭遇するたびに、すべてのフォームを1つの大きなフラグメントに格納するカスタムフックを作成する傾向があります。次のようになります。

export function LoginForm() {
  const { formFields, createChangeHandler } = useFormFields({
    email: '',
    password: '',
  });

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    api.login(formFields.email, formFields.password);
  };
  return (
    <form onSubmit={handleSubmit}>
      <div>
        <label htmlFor='email'>Email</label>
        <input type='email' id='email' value={formFields.email} onChange={createChangeHandler('email')} />
      </div>
      <div>
        <label htmlFor='password'>Password</label>
        <input type='password' id='password' value={formFields.password} onChange={createChangeHandler('password')} />
      </div>
    </form>
  );
}

useFormFieldsフックを使用すると、コンポーネントを複雑にすることなく、フィールドを追加し続けることができます。すべてのフォームの状態に1か所でアクセスでき、コードは簡潔で簡潔に見えます。もちろん、いくつかの状況ではおそらくsetStateを使用する必要がありますが、ほとんどのフォームでは上記の手法で十分です。

代替アプローチ


したがって、状態処理は正常に機能し、ほとんどの場合、これはReactで推奨されるアプローチです。しかし、別の方法があることを知っていましたか？ブラウザはデフォルトでフォームの内部状態を処理することがわかりました。これを使用してコードを簡略化できます！

以下は同じフォームですが、ブラウザがフォームの状態を処理できるようにしています。

export function LoginForm() {
  const handleSubmit = (e: React.FormEvent) => {
	e.preventDefault();
	const formData = new FormData(e.target as HTMLFormElement);
	api.login(formData.get('email'), formData.get('password'));
  };
  return (
	<form onSubmit={handleSubmit}>
  	<div>
    	<label htmlFor="email">Email</label>
    	<input
      	type="email"
      	id="email"
      	name="email"
    	/>
  	</div>
  	<div>
    	<label htmlFor="password">Password</label>
    	<input
      	type="password"
      	id="password"
      	name="password"
    	/>
  	</div>
  	<button>Log in</button>
	</form>
  );
}

簡単ですね。単一のフックが使用されておらず、初期値の設定も、onChangeハンドラーもありません。最良の部分は、コードが以前と同じように機能することですが、どうやって？

handleSubmit関数で少し違うことをしていることに気づいたかもしれません。 FormDataと呼ばれる組み込みのブラウザーAPIを使用します。 FormDataは、入力フィールドから値を取得するための便利な（そして十分にサポートされている）方法です！

submitイベントのtarget属性を介してDOM内のフォームへのリンクを取得し、FormDataクラスの新しいインスタンスを作成します。これで、formData.get（「入力フィールド名」）を呼び出して、name属性ですべてのフィールドを取得できます。

これにより、フォームの状態を明示的に処理する必要がなくなります。デフォルト値が必要な場合（たとえば、データベースまたはローカルストレージから初期フィールド値を変更した場合）、Reactはこのための便利なdefaultValueオプションを提供します。

それぞれのアプローチを使用するタイミング


フォームはほとんどのWebアプリケーションの不可欠な部分であるため、フォームの処理方法を知ることが重要です。そして、Reactはこれを行うための多くの方法を提供します。
厳密な検証を必要としない（またはHTML5フォーム検証コントロールに依存できる）単純なフォームの場合、DOMがデフォルトで提供する組み込みの状態処理を使用することをお勧めします。実行できないことはかなりありますが（たとえば、プログラムで入力値を変更したり、リアルタイムで確認したり）、最も単純なケース（たとえば、上記の検索フィールドやログインフィールドなど）では、おそらく適切だと思いますブラウザAPIを使用した代替アプローチ。

カスタムチェックを実行する場合、または送信する前に一部のフォームデータにアクセスする必要がある場合は、制御されたコンポーネントを使用して状態を明示的に処理する必要があります。通常のuseStateHooksを使用するか、カスタムフックを作成して、コードを少し簡略化できます。

ほとんどの場合、React開発者が制御されたコンポーネントの使用を推奨する（コンポーネントの状態を明示的に処理する）ことは注目に値します。このアプローチにより、将来の柔軟性が高まるためです。私の意見では、開発者はしばしば柔軟性のために単純さを犠牲にしますが、多くの場合、必要はありません。

どの方法を使用する場合でも、Reactフォームの処理は今日ほど簡単ではありませんでした。状況に応じてフォームの状態を明示的に処理しながら、ブラウザに単純なフォームを処理させることができます。上記の方法が役に立ち、コードの行数を減らすことで問題を解決するのに役立つことを願っています。
