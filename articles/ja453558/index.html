機械学習とは何ですか？


それでは、次のタスクから始めましょう-与えられた入力値と出力値。



入力値として値0がある場合は、出力値として32、入力値として8がある場合は、出力値として46.4。入力値が15の場合、出力値は59などになります。

これらの値を見て、質問させてください。入力で38を取得した場合、出力はどうなるかを判断できますか？



100.4と答えたら正解でした。



では、どうすればこの問題を解決できるでしょうか。値をよく見ると、それらが次の式によって関連付けられていることがわかります。



ここで、C-摂氏（入力値）、F-華氏（出力値）。

あなたの脳が今行ったこと-入力値と出力値を比較し、それらの間の共通モデル（接続、依存関係）を見つけました-これが機械学習です。

機械学習アルゴリズムは、入力値と出力値に基づいて、入力値を出力値に変換するための適切なアルゴリズムを見つけます。これは次のように表すことができます



。例を見てみましょう。数式を使用して摂氏を華氏に変換するプログラムを開発したいとしF = C * 1.8 + 32ます。



ソリューションは、従来のソフトウェア開発の観点からアプローチすると、次の関数を使用して任意のプログラミング言語で実装できます。



それで私たちは何を持っていますか？関数はCの入力値を取り、次に明示的なアルゴリズムを使用してFの出力値を計算し、計算された値を返します。



一方、機械学習アプローチでは、入力値と出力値のみがあり、アルゴリズム自体はありません



。機械学習アプローチは、入力値と出力値の間の関係を見つけるためのニューラルネットワークの使用に基づいています。



ニューラルネットワークは、層のスタックと考えることができます。各層は、以前に知られている数学（数式）と内部変数で構成されています。入力値はニューラルネットワークに入り、ニューロンの層のスタックを通過します。レイヤーを通過する際、入力値は数学（所定の数式）とレイヤーの内部変数の値に従って変換され、出力値が生成されます。

ニューラルネットワークが入力値と出力値の間の正しい関係を学習および決定できるようにするには、トレーニングする必要があります。

入力値と出力値を照合する試みを繰り返して、ニューラルネットワークをトレーニングします。



トレーニングのプロセスでは、ネットワークが対応する入力値に対応する出力値を生成することを学習するまで、ニューラルネットワークのレイヤーの内部変数の値の「フィッティング」（選択）が行われます。

後で見るように、ニューラルネットワークをトレーニングし、内部変数の最適な値を選択できるようにするために、数千または数万の反復（トレーニング）が実行されます。



機械学習を理解する簡単なバージョンとして、機械学習アルゴリズムは、正しい入力値が正しい出力値に対応するように内部変数の値を選択する関数と考えることができます。

ニューラルネットワークアーキテクチャには多くの種類があります。ただし、どのアーキテクチャを選択しても、内部の数学（どの計算がどの順序で実行されるか）はトレーニング中に変更されません。数学を変更する代わりに、トレーニング中に内部変数（重みとオフセット）が変更されます。

たとえば、摂氏から華氏に変換するタスクでは、モデルは入力値に特定の数値（重み）を掛け、別の値（オフセット）を追加することから始めます。モデルトレーニングは、実行された乗算と加算の演算を変更せずに、これらの変数に適した値を見つけることです。

しかし、考えるべきクールなことが1つあります。ビデオと以下のテキストに示されている摂氏から華氏への変換の問題を解決した場合は、摂氏から華氏へのこの種の変換の実行方法に関する以前の経験または知識があるため、おそらくそれを解決しました。たとえば、摂氏0度は華氏32度に相当することがわかります。一方、機械学習に基づくシステムには、問題を解決するためのサポート知識がありません。彼らは、以前の知識に基づいていない完全に不在の状態で、そのような問題を解決することを学びます。

CoLab：摂氏を華氏に変換


CoLabソースコードのロシア語版とCoLabソースコードの英語版。

基本：最初のモデルの学習


CoLabへようこそ。ここでは、最初の機械学習モデルをトレーニングします。

提示された資料の簡潔さを維持するよう努め、作業に必要な基本的な概念のみを紹介します。後続のCoLabには、より高度な技術が含まれます。

解決するタスクは、摂氏を華氏に変換することです。変換式は次のとおりです。

f=c×1.8+32



もちろん、直接計算を実行する変換関数をPythonまたはその他のプログラミング言語で記述する方が簡単ですが、この場合は機械学習ではありません:)

代わりに、TensorFlow入力に摂氏の入力（ 0、8、15、22、38）とそれぞれの華氏（32、46、59、72、100）。次に、上記の式にほぼ対応するようにモデルをトレーニングします。

依存関係のインポート


最初にインポートしTensorFlowます。以下、略記しtfます。ログのレベルも設定します-エラーのみ。

次に、NumPyとしてインポートしnpます。Numpyデータを高性能なリストとして表示するのに役立ちます。
from __future__ import absolute_import, division, print_function, unicode_literals
import tensorflow as tf
tf.logging.set_verbosity(tf.logging.ERROR)

import numpy as np

トレーニングデータの準備


前述のように、教師による機械学習手法は、入力データを出力データに変換するためのアルゴリズムの検索に基づいています。このコラボの目標は華氏への摂氏の変換結果を生成できるモデルを作成することですので、我々は二つのリストを作成する- celsius_qとfahrenheit_aどの我々のモデルを訓練するとき、我々が使用しています。

celsius_q    = np.array([-40, -10, 0, 8, 15, 22, 38], dtype=float)
fahrenheit_a = np.array([-40, 14, 32, 46, 59, 72, 100], dtype=float)

for i,c in enumerate(celsius_q):
  print("{}   = {}  ".format(c, fahrenheit_a[i]))

-40.0 = -40.0
-10.0 = 14.0
0.0 = 32.0
8.0 = 46.0
15.0 = 59.0
22.0 = 72.0
38.0 = 100.0

機械学習の用語：

  • プロパティは、モデルの入力値です。この場合、単位の値は摂氏です。
  • ラベルは、モデルが予測する出力値です。この場合、単位値は華氏です。
  • 例は、トレーニングに使用される入出力値のペアです。この場合、それは特定のインデックスからの値celsius_qとそのfahrenheit_a下の値のペアです（例：（22,72））。

モデルを作成する


次に、モデルを作成します。最も単純なモデル、つまり完全に接続されたネットワーク（Dense-net）のモデルを使用します。タスクは非常に簡単なので、ネットワークも単一のニューロンを持つ単一の層で構成されます。

ネットワークを構築する


私たちは、レイヤ名前が付けられますl0（リットルのアヤとゼロ）と初期化することによって、それを作成しtf.keras.layers.Dense、次のパラメータを使用して：

  • input_shape=[1] — — . 1×1 . ( ) , . — , .
  • units=1 — . , . , — — . ( input_shape ).

l0 = tf.keras.layers.Dense(units=1, input_shape=[1])

レイヤーをモデルに変換


レイヤーを定義したら、モデルに変換する必要があります。Sequential-modelは、レイヤーのリストを、それらを適用する必要がある順序（入力値から出力値まで）で引数として受け取ります。

このモデルには1つのレイヤーしかありません- l0。

model = tf.keras.Sequential([l0])

注
予備的な説明やその後の使用ではなく、モデル関数で直接レイヤーを定義することがよくあります。
model = tf.keras.Sequential([
    tf.keras.layers.Dense(units=1, input_shape=[1])
])

損失および最適化関数を含むモデルのコンパイル


トレーニングの前に、モデルをコンパイル（アセンブル）する必要があります。トレーニング用にコンパイルする場合、次のものが必要です。

  • 損失関数 -予測値が望ましい出力値からどれだけ離れているかを測定する方法（測定可能な差は「損失」と呼ばれます）。
  • 最適化関数 -内部変数を調整して損失を減らす方法。


model.compile(loss='mean_squared_error',
              optimizer=tf.keras.optimizers.Adam(0.1))

損失関数と最適化関数は、モデルのトレーニング（model.fit(...)後述）中に使用され、各ポイントで初期計算を実行してから、値を最適化します。

現在の損失を計算するアクションと、モデル内のこれらの値のその後の改善は、トレーニングとまったく同じです（1回の反復）。

トレーニング中、最適化関数は、内部変数の値に対する調整を計算するために使用されます。目標は、モデル内の内部変数の値を調整することであり（これは実際には数学関数です）、摂氏を華氏に変換するための既存の式をできるだけ正確に反映します。

TensorFlowは数値解析を使用してこれらの種類の最適化操作を実行します。この複雑さはすべて私たちの目から隠されているため、このコースでは詳細に触れません。

これらのパラメーターについて知っておくと役立つこと：

この例で使用されている損失関数（標準誤差）と最適化関数（Adam）は、このような単純なモデルの標準ですが、他にも多くのものが利用できます。この段階では、これらの機能がどのように機能するかは問題ではありません。

注意する必要があるのは最適化関数であり、パラメーターは学習率係数（learning rate）です。この例では、0.1。これは、変数の内部値を調整するときに使用されるステップサイズです。値が小さすぎると、モデルをトレーニングするためにトレーニング反復が多すぎます。多すぎる-精度が低下します。学習速度の係数の適切な値を見つけるには、試行錯誤が必要0.01です0.1。通常、（デフォルトでは）からの範囲です。

モデルを訓練する


モデルのトレーニングは、メソッドによって実行されfitます。

トレーニング中、モデルは入力で摂氏を受け取り、内部変数の値（「重み」と呼ばれます）を使用して変換を実行し、華氏に対応する必要のある値を返します。重みの初期値は任意に設定されているため、結果の値は正しい値とは大きく異なります。望ましい結果と実際の結果の差は損失関数を使用して計算され、最適化関数は重みの調整方法を決定します。

この計算、比較、調整のサイクルは、メソッド内で制御されますfit。最初の引数は入力値、2番目の引数は目的の出力値です。引数epochsこのトレーニングサイクルを完了する回数を決定します。引数verboseはロギングのレベルを制御します。

history = model.fit(celsius_q, fahrenheit_a, epochs=500, verbose=False)
print("  ")

次のビデオでは、これがどのように機能するか、完全に接続されたレイヤー（Denseレイヤー）が「内部」でどのように機能するかについて詳しく説明します。

トレーニング統計を表示する


このメソッドfitは、後続の反復ごとの損失の変化に関する情報を含むオブジェクトを返します。このオブジェクトを使用して、適切な損失スケジュールを作成できます。高損失は、モデルによって予測された華氏度が配列の真の値から離れていることを意味しますfahrenheit_a。

可視化に使用しMatplotlibます。ご覧のように、モデルは最初は非常に速く改善され、トレーニングの最後で結果が「ほぼ」完全になるまで、安定したゆっくりとした改善に至ります。

import matplotlib.pyplot as plt
plt.xlabel('Epoch')
plt.ylabel('Loss')
plt.plot(history.history['loss'])



予測にはモデルを使用します。

たとえば、華氏100.0度はいくらですか。以下のコードを実行する前に推測してみてください。

最初の変数の値は〜1.8に近く、2番目の変数は〜32に近いです。これらの値（1.8と32）は、摂氏を華氏に変換する式の直接の値です。

これは実際に式の実際の値に非常に近いです！この点については、以降のビデオで詳しく考察します。そこでは、Denseレイヤーの動作を示しますが、今のところ、単一の入力と出力を持つ1つのニューロンに単純な数学が含まれていることを知っておく価値があります- y = mx + b（線の方程式として）。摂氏を華氏に変換するための式にすぎませんf = 1.8c + 32。

表現が同じであるため、モデルの内部変数の値は、最後に発生した実際の式で示される値に収束するはずです。

追加のニューロン、追加の入力値、および出力値が存在するため、式は少し複雑になりますが、本質は変わりません。

トレーニングプロセス


トレーニングプロセス（メソッドで行われるmodel.fit(...)）は、非常に単純な一連のアクションで構成されます。その結果は、可能な限り元の結果に近い結果を与える内部変数の値である必要があります。勾配降下法と呼ばれる、このような結果が得られる最適化プロセスでは、数値解析を使用して、モデルの内部変数に最も適した値を見つけます。

機械学習を行うために、原則として、これらの詳細を理解する必要はありません。しかし、さらに学習したい人のために：反復による勾配降下はパラメーター値を少し変更し、最良の結果が得られるまで正しい方向に「引き」ます。この場合、「最良の結果」（最良の値）は、その後のパラメーターの変更がモデルの結果を悪化させるだけであることを意味します。各反復でのモデルの良し悪しを測定する関数は「損失関数」と呼ばれ、各「プル」（内部値の調整）の目標は、損失関数の値を減らすことです。

トレーニングプロセスは、「直接分布」ブロックから始まります。このブロックでは、入力パラメーターがニューラルネットワークの入力に移動し、非表示のニューロンに従い、週末に移動します。次に、モデルは入力値と内部変数に内部変換を適用して、応答を予測します。

この例では、入力値は摂氏での温度で、モデルは対応する華氏での値を予測しました。



値が予測されるとすぐに、予測値と正しい値の差が計算されます。この違いは「損失」と呼ばれ、モデルがどの程度うまく機能したかを測定する方法の1つです。損失値は、メソッドを呼び出すときに引数の1つによって決定した損失関数によって計算されmodel.compile(...)ます。

損失値を計算した後、出力値を正しい初期参照値に近づけるために、ニューラルネットワークのすべての層の内部変数（重みと変位）を調整して、損失値を最小化します。



この最適化プロセスは、勾配降下法と呼ばれます。特定の最適化アルゴリズムは、メソッドが呼び出されたときに、各内部変数の新しい値を計算するために使用されますmodel.compile(...)。上記の例では、最適化アルゴリズムを使用しましたAdam。

私たちのネットワークは、単一の完全に接続されたレイヤーで構成されています。しかし、完全に接続されたレイヤーとは何ですか？これを理解するために、2つのニューロンを持つ1つの非表示層と1つのニューロンを持つ1つの出力層の3つの入力パラメーターを持つより複雑なニューラルネットワークを作成してみましょう。



ニューラルネットワークは、それぞれがニューロンと呼ばれるノードで構成される一連のレイヤーとして想像できることを思い出してください。各レベルのニューロンは、後続の各層のニューロンに接続できます。 1つの層の各ニューロンが次の層の他のニューロンに接続されている層のタイプは、完全に接続された（完全に接続された）または高密度の層（Dense-layer）と呼ばれます。



したがって、で完全に接続されたレイヤーを使用する場合keras、このレイヤーのニューロンが前のレイヤーのすべてのニューロンに接続される必要があることを通知します。
