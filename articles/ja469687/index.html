公開された25の最も危険なソフトウェアの脆弱性のリスト



画像：Unsplash

アメリカの組織MITERは、最も危険なソフトウェアの脆弱性の25のリストを公開しています。研究者は、識別子CWE（Common Weakness Enumeration）を使用して、最も危険で一般的なセキュリティ問題の表をまとめました。

上位25件には多数の脆弱性があり、攻撃者はコンピュータプログラムを完全に制御し、それらを無効にして、処理されたデータを盗むことができます。このリストにより、情報セキュリティの専門家はセキュリティの脅威の状況をよりよく理解できますが、ソフトウェア開発者、テスター、およびソフトウェア製品のユーザーにとっても役立ちます。

研究方法


リストをまとめるために、MITREの専門家は公開された脆弱性（CVE、Common Vulnerabilities and Exposures）に関する情報、NVD（National Vulnerability Database）からの情報、およびCVSS（Common Vulnerability Scoring System）の脆弱性評価を使用しました。脆弱性の蔓延と深刻度を評価するために、独自のアルゴリズムが開発されました。

上位3つの危険な脆弱性


レポートによると、最も危険なのは、メモリバッファの境界内での不適切な操作制限と呼ばれる脆弱性でした。エラーは、ソフトウェアが割り当てられたメモリバッファー内で実行されたときに発生しますが、同時に、このバッファーの境界外でデータの読み取りまたは書き込みが可能です。この脆弱性の悪用により、攻撃者は任意のコードを実行し、重要なデータを盗み、ソフトウェアシステムを完全に無効にすることができます。

危険の2番目の場所は、CWE-79コードの脆弱性-Webページ生成中の入力の不適切な中和です。クロスサイトスクリプティング（XSS）は、より一般的な名前です。このタイプの脆弱性は一般的であり、ユーザーがサイトのページに入力するデータを十分に制御できないため、深刻な結果につながる可能性があります。

3位は、無効な入力の検証（CWE-20）の脆弱性です。攻撃者は特別な悪意のある要求を行うことができ、システムはそれに応じて、特権の増加または重要なデータへのアクセスを提供します。

その他の関連するサイバー脅威


現在のサイバー脅威に関する情報源は、MITRE格付けだけではありません。 Positive Technologies は、 2018年の結果に続くWebアプリケーションへの攻撃に関するレポートを公開しました。 Webサイトに対する3つの最も一般的な攻撃には、SQLインジェクション、パストラバーサル、およびクロスサイトスクリプティング（XSS）が含まれていました。

さらに、Positive Technologiesの専門家は、現在のサイバー脅威に関する分析レポートを四半期に1回公開しています。そのため、2019年の第2四半期に、サイバー犯罪者はEximメールサーバーの脆弱性を積極的に悪用しました（CVE-2019-10149）。エラーはさまざまな方法で使用されます。一部の攻撃者はそれを使用して暗号通貨マイニングソフトウェアをダウンロードし、他の攻撃者はメールサーバーにバックドアを配置します。
