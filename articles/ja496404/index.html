Synology NASをゲームサーバーに変える



こんにちは！

したがって、すべての既知の理由により、自宅のモニターの前でより多くの時間を費やす必要があります。
この状況では、過去の出来事を思い出す必要があります。

この記事のタイトルからすでに明らかなように、Synology NASをゲームサーバーとしてセットアップする方法について説明します。

Achtung-記事にはたくさんのスクリーンショットがあります（スクリーンショットはクリック可能です）。

始める前に、必要なツールのリストを以下に示します

。SynologyNAS-ここには何の制限もありません。1万人のプレーヤー用にサーバーを維持する計画がない場合は、誰でもそうするでしょう。

Docker-仕事の原則を理解するのに比喩的に十分な特別なスキルは必要ありません。

LinuxGSM -何についてLinuxGSM読むことができます。オンラインhttps://linuxgsm.com。

現在（2020年4月）、105台のゲームサーバーがLinuxGSMで利用可能です。
リスト全体は、https：//linuxgsm.com/serversにあります。

Steamはゲーム市場です。

LinuxGSMゲームサーバーはSteamCMDと統合されています。つまり、LinuxGSMゲームサーバーはSteamのゲームにのみ使用できます。

Synology NASにDockerをインストールする


この段階では、すべてが簡単です。Synology管理パネルに移動し、「パッケージセンター」でDockerを見つけてインストールします。

パッケージセンター

私たちはこのようなものを始めて見ます（私はすでにこのコンテナをインストールしています）

コンテナ管理

次に、「レジストリ」タブに移動し、「gameservermanagers」検索に移動し、「gameservermanagers / linuxgsm-docker」イメージを選択して、「ダウンロード」ボタンをクリックします。

gameservermanagers / linuxgsm-docker

その後、「画像」タブに移動し、画像の読み込みが完了するのを待って、「起動」ボタンをクリックします。

画像をダウンロード

開いたウィンドウで、[詳細設定]に移動し、[ネットワーク]タブに移動して、[Dockerホストと同じネットワークを使用する]チェックボックスをオンにします。

「コンテナ名」などの他の設定は、弊社の裁量により変更されます。
コンテナ名-ご想像のとおり、これはコンテナの名前です。後で便利になります。私はそれを何とか簡潔に呼ぶことをお勧めします。例えば、それを「テスト」としましょう。

次に、設定が完了するまで「適用」または「次へ」ボタンを数回クリックします。

高度な設定

[コンテナ]タブに移動して、新しいコンテナが起動したことを確認します（起動していない場合は開始します）。
ここで、停止、開始、削除、その他のアクションを実行できます。

コンテナの打ち上げ

LinuxGSM Dockerコンテナーの構成


SSH経由でSynology NASに接続する前に、管理パネルでSSHアクセスを有効にする必要があります。

SSH接続

次に、Synology NASサーバーの内部IPアドレスを使用して、SSH経由で接続する必要があります。

ターミナル（または、Windowsの場合はPuTTYなどの他のアナログ）に移動し、次のコマンドを使用します。

ssh user_name@IP

私の場合はこのようになります

ssh admin@192.168.0.166

Synology NASサーバーのIPアドレス

承認後、ユーザー「root」の下で「test」コンテナ自体（Docker設定の「Container Name」フィールド）に移動するコマンドを実行する必要があります

sudo docker exec -u 0 -it test bash

Dockerに接続する

LinuxGSMをインストールする前に、いくつかの作業を行う必要があります。

rootユーザーのパスワードを設定する

passwd

次に、すべてのパッケージを更新します

apt update && apt upgrade && apt autoremove

プロセスの終了を待っています...

パッケージの更新

次に、必要なユーティリティをインストールします

apt-get install sudo iproute2 netcat nano mc p7zip-rar p7zip-full

「root」の下で、さまざまなアクションを実行することは良い考えではないため、新しいユーザー「test」を追加します。

adduser test

そして、新しいユーザーにsudoを使用させます

usermod -aG sudo test

新しいユーザー「test」に切り替えます

su test

ユーティリティのインストール

LinuxGSMをインストールして構成する


例えば、「カウンターストライク」別名「CS 1.6」を使用してLinuxGSM構成の例で見てみましょうhttps://linuxgsm.com/lgsm/csserver

命令「カウンターストライク」のページに移動しlinuxgsm.com/lgsm/csserver。

「依存関係」タブで、「Ubuntu 64ビット」の下のコードをコピーします。

執筆時点では、このコードは次のようになっています。

sudo dpkg --add-architecture i386; sudo apt update; sudo apt install mailutils postfix curl wget file tar bzip2 gzip unzip bsdmainutils python util-linux ca-certificates binutils bc jq tmux lib32gcc1 libstdc++6 lib32stdc++6 steamcmd

依存関係のインストール

インストールプロセス中に、「スチームライセンス」に同意する必要があります。

Steamライセンス

[インストール]タブに移動し、2番目のステップからコードをコピーします（最初のステップをスキップして、ユーザー「test」は既に存在します）：

インストール

wget -O linuxgsm.sh https://linuxgsm.sh && chmod +x linuxgsm.sh && bash linuxgsm.sh csserver

ダウンロードを待っています：

ダウンロード

そして、インストールを開始します。

./csserver install

すべてが通常どおりに進んだ場合、大事な「Install Complete！」が表示されます。

インストール完了！

...を開始すると、「複数のIPアドレスが見つかりました」というエラーが表示されます。

./csserver start

複数のIPアドレスが見つかりました

次に、使用するIPをサーバーに明示的に通知する必要があります。

私の場合、それは：

192.168.0.166

「場所」としてメッセージに含まれていたパスに移動します。

cd /home/test/lgsm/config-lgsm/csserver

そして、このフォルダにどのファイルがあるかを確認してください：

ls

csserverフォルダー内のファイルのリスト

ファイル「_default.cfg」の内容をファイル「csserver.cfg」にコピーします。

cat _default.cfg >> csserver.cfg

そして「csserver.cfg」ファイルの編集モードに行きます：

nano csserver.cfg

csserver.cfgファイルの編集

行を見つけます：

ip="0.0.0.0"

提案されたIPアドレスを置き換えます。私の場合は「192.168.0.166」です。

次のような結果になります。

ip="192.168.0.166"

キーボードショートカットを固定します。 

Ctr + X

そしてオファー保存後クリック：

Y

ユーザーフォルダー「test」に戻ります。

cd ~

そして再び、サーバーの起動を試みます。これでサーバーは問題なく起動するはずです。

./csserver start

サーバーの起動

より詳細な情報を表示するには、次のコマンドを使用します。

./csserver details

詳細なサーバー情報

重要なパラメータのうち、注目に値するものは次のとおりです。

  • サーバーIP：192.168.0.166:27015
  • インターネットIP：xxx.xx.xxx.xx：27015
  • 構成ファイル：/home/test/serverfiles/cstrike/csserver.cfg

この時点で、ゲームサーバーはローカルネットワークですでに利用可能です。

IP転送の設定


ローカルネットワークでプレイするのは良いですが、インターネット経由で友達とプレイするのが良いです！

ルータがプロバイダーから受信したIPアドレスを転送するために、NATメカニズムを使用します。

また、ほとんどのプロバイダーはクライアントに動的IPアドレスを使用することに注意してください。

利便性と安定性のために、静的IPアドレスを取得することをお勧めします。

私はTP-Link Archer C60ルーターを持っているので、ルーターに実装されているコール転送の設定例を示します。

他のルーターについては、転送設定は同様であると思います。

ここではすべてが簡単です。2つのポートについて、サーバーの外部IPアドレスから内部IPアドレスへの転送を指定する必要があります。

  • 27015
  • 27005


私のルーターの管理パネルでは、このようになります

管理ルーターパネル

ルーターの設定を保存すると、指定したポートの外部IPアドレスでゲームサーバーをネットワークで使用できるようになります。

CS 1.6の例に関する追加設定


たとえば、CS 1.6はいくつかの有用なヒントを提供したいと考えています。

サーバー構成には2つのファイルがあり、

最初のファイルは次のとおりです。

~/lgsm/config-lgsm/csserver/csserver.cfg

2番目はここにあります：

~/serverfiles/cstrike/csserver.cfg

最初のファイルには、IPアドレス、サーバーの初回起動時のマップなどの一般的な設定が含まれています。

2番目のファイルには、Counter-Strikeコンソールを介して実行できるコマンドの設定（rcon_passwordやsv_passwordなど）が含まれています。

2番目のファイルでは、CVar「sv_password」を介してサーバーに接続するためのパスワードを設定し、CVar「rcon_password」を介してサーバー自体のコンソールから管理用のパスワードを設定することをお勧めします。

すべてのCVar変数のリストは 、http：

//txdv.github.io/cstrike-cvarlistにあります。また、「fy_pool_day」などの追加のカードをインストールする必要がある可能性があります。

CS 1.6のすべてのマップは次のとおりです。

~/serverfiles/cstrike/maps

目的のマップを見つけてサーバーに直接アップロードし（アーカイブ内にある場合は解凍します）、拡張子「.bsp」のファイルをファイル「〜/ serverfiles / cstrike / maps」のあるフォルダーに移動し、サーバーを再起動します。

~./csserver restart

ちなみに、利用可能なすべてのサーバーコマンドは次のように表示できます。

~./csserver

合計


結果に満足しています。すべてが迅速に動作し、遅れることはありません。

LinuxGSMには、通知用のTelegramやSlackとの統合など、多くの追加設定がありますが、一部の機能にはまだ改善が必要です。

一般的には、使用をお勧めします！
