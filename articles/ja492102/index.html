サーバーの準備



機能するには、ボットが起動される場所が必要です。.NET Coreは任意のプラットフォームから起動できるため、任意のホスティングがこれに適しています。

Ubuntu 18.04のVPSサーバーの例を検討しました。

ボットが機能するクリーンなサーバーでは、少なくともバージョン2.2の.NET Coreをインストールする必要があります。

次のコマンドを入力します
。MicrosoftキーとWebフィードを登録するコマンド：

> wget -q https://packages.microsoft.com/config/ubuntu/19.04/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb

インストール可能な製品の更新：

> sudo apt-get update
> sudo apt-get install apt-transport-https
> sudo apt-get update

。今、.NETのコア自体をインストールする
ためのランタイムのバージョン、コマンドを入力します。

> sudo apt-get install aspnetcore-runtime-3.1

以下のためのソフトウェア開発キットのバージョン：

> sudo apt-get install dotnet-sdk-3.1

環境をインストールしたら、次のコマンドを使用してその準備状況を確認します。

> dotnet --info

プラグインの作成


チャットボットとしてのVkBotCoreの基本機能には、/ helpと/ everyoneの2つのコマンドのみが含まれます。
機能を向上させるには、プラグインを作成する必要があります。

プラグインは、プラグインとして接続するために必要な基本機能を備えたクラスライブラリです。

新しいプロジェクトを作成してみましょう。vkBotCore.dll



ライブラリをアセンブリからプロジェクトに接続します。using





をvkBotCore.Pluginsに追加し、プラグインからメインクラスを継承します

using vkBotCore.Plugins;

namespace myPlugin
{
    public class MyFirstPlugin : Plugin
    {
    }
}

こんにちは{名前}が応答するようにhelloコマンドを追加します

using vkBotCore.Plugins;
using vkBotCore.Plugins.Attributes;

namespace myPlugin
{
    public class MyFirstPlugin : Plugin
    {
        [Command]
        public void Hello(CommandContext context)
        {
            context.Chat.SendMessage($" {context.Sender.GetMentionLine()}");
        }
    }
}
