Linuxでの最新のテキストレンダリング：パート1

モダンLinuxテキストレンダリングの前半へようこそ。このシリーズの各記事では、文字または文字のシーケンスを視覚化する自己完結型のCプログラムを開発します。これらの各プログラムは、現代のテキストレンダリングに必要だと考える機能を実装します。

最初の部分では、FreeTypeを構成し、コンソールで簡単なシンボルレンダラーを記述します。



これが私たちが書くものです。そして、これがコードです。

システム設定


  • 私のオペレーティングシステム： Ubuntu 18.04.2 LTS (bionic)
  • コンパイラC： clang version 6.0.0-1ubuntu2

FreeTypeをインストールする


Ubuntuでは、FreeTypeとlibpngをインストールする必要があります。

$ sudo apt install libfreetype6 libfreetype6-dev
$ sudo apt install libpng16-16 libpng-dev

  • 私はFreeTypeバージョンを持っていますが、2.8.1-2ubuntu2この記事の執筆時点では最新バージョンですが、FreeType-2.10.1動作します。
  • libpngバージョン (1.6.34-1ubuntu0.18.04.2)

コンソールレンダラー


Cファイルを作成する（main.c私の場合）


#include <stdio.h>

int main() {
  printf("Hello, world\n");
  return 0;
}

$ clang -Wall -Werror -o main main.c
$ ./main
Hello, world

FreeTypeのライブラリをつなぐ


#includeFreeType のインクルードパス（つまり、でファイルを検索するときにコンパイラが通過するディレクトリ）を検索するには、次のコマンドを実行します。

$ pkg-config --cflags freetype2
-I/usr/include/freetype2 -I/usr/include/libpng16

文字列に-I/usr/include/freetype2 -I/usr/include/libpng16は、FreeTypeをCプログラムに接続するために必要なコンパイルフラグが含まれています。

#include <stdio.h>

#include <freetype2/ft2build.h>
#include FT_FREETYPE_H

int main() {
  printf("Hello, world\n");
  return 0;
}

$ clang -I/usr/include/freetype2 \
        -I/usr/include/libpng16  \
        -Wall -Werror            \
        -o main                  \
         main.c
$ ./main
Hello, world

FreeTypeのバージョンを印刷します


内部では、main()FreeTypeを初期化FT_Init_FreeType(&ft)してエラーをチェックします（FreeType関数は、成功した場合0を返します）。

（これから、私が使用するすべての関数はFreeType APIのヘルプから取得されます）。

FT_Library ft;
FT_Error err = FT_Init_FreeType(&ft);
if (err != 0) {
  printf("Failed to initialize FreeType\n");
  exit(EXIT_FAILURE);
}

次に、FT_Library_Versionを使用して、バージョン番号を取得します。

FT_Int major, minor, patch;
FT_Library_Version(ft, &major, &minor, &patch);
printf("FreeType's version is %d.%d.%d\n", major, minor, patch);

最後のコマンドを使用してコンパイルすると、リンカーエラーがポップアップします。

/tmp/main-d41304.o: In function `main':
main.c:(.text+0x14): undefined reference to `FT_Init_FreeType'
main.c:(.text+0x54): undefined reference to `FT_Library_Version'
clang: error: linker command failed with exit code 1 (use -v to see invocation)

修正するには、を追加し-lfreetypeます。

$ clang -I/usr/include/freetype2 \
        -I/usr/include/libpng16  \
        -Wall -Werror            \
        -o main                  \
        -lfreetype               \
         main.c
$ ./main
FreeType's version is 2.8.1

フォントのダウンロード


文字をレンダリングする最初のステップは、フォントファイルをダウンロードすることです。ubuntu monoを使用しています。

フォントフェースコンストラクト、フォントファミリ、および個々のフォントの正確な違いを理解するには、FreeTypeのドキュメントを参照してください。

3番目の引数は面インデックスと呼ばれます。これは、フォント作成者が同じフォントサイズで複数の面を埋め込むことができるように設計されています。各フォントには少なくとも1つの面があるため、値0は常に機能し、最初のオプションを選択します。

 FT_Face face;
err = FT_New_Face(ft, "./UbuntuMono.ttf", 0, &face);
if (err != 0) {
  printf("Failed to load face\n");
  exit(EXIT_FAILURE);
}

顔のピクセルサイズを設定


この命令を使用して、FreeTypeに表示された文字に必要な幅と高さを伝えます。

幅にゼロを渡すと、FreeTypeはこれを「他のものと同じ」、この場合は32pxと解釈します。これは、たとえば幅10px、高さ16pxの文字を表示するために使用できます。

絵文字の場合のように、この操作は固定サイズのフォントでは失敗する場合があります。

err = FT_Set_Pixel_Sizes(face, 0, 32);
if (err != 0) {
  printf("Failed to set pixel size\n");
  exit(EXIT_FAILURE);
}

文字のインデックスを取得する


まず、FreeTypeのドキュメントに戻り、命名規則を確立します。シンボルはグリフと同じではありません。記号はに示されているものcharであり、グリフはこの記号に何らかの形で関連付けられている画像です。charはいくつかのグリフ、つまりアクセントに対応できるため、この関係はかなり複雑です。グリフは多くの文字に対応できます。つまり、合字で、->は単一の画像として表されます。

文字に対応するグリフインデックスを取得するには、を使用しますFT_Get_Char_Index。ご存知のように、これは文字とグリフを1対1で一致させることを含みます。このシリーズの今後の記事では、HarfBuzzライブラリを使用して問題を解決します。

 FT_UInt glyph_index = FT_Get_Char_Index(face, 'a');

顔からのグリフの読み込み


glyph_indexを受け取ったら、対応するグリフを顔からロードできます。

今後の記事では、さまざまなダウンロードフラグと、それらがどのようにヒントやビットマップフォントなどの機能を使用できるようにするかについて詳しく説明します。

FT_Int32 load_flags = FT_LOAD_DEFAULT;
err = FT_Load_Glyph(face, glyph_index, load_flags);
if (err != 0) {
  printf("Failed to load glyph\n");
  exit(EXIT_FAILURE);
}

コンテナ（グリフスロット）にグリフを表示する


これで、最後に、で指定されたコンテナ（スロット）にグリフを表示できますface->glyph。

また、LCD（またはサブピクセル）レンダリングとグレースケールアンチエイリアスを使用できるようにするため、将来的にはレンダリングフラグについても説明します。

FT_Int32 render_flags = FT_RENDER_MODE_NORMAL;
err = FT_Render_Glyph(face->glyph, render_flags);
if (err != 0) {
  printf("Failed to render the glyph\n");
  exit(EXIT_FAILURE);
}

コンソールへの文字出力


レンダリングされたグリフのビットマップ画像は、face->glyph->bitmap.bufferunsigned char値の配列として表示される場所から取得できるため、その値の範囲は0〜255です

。バッファーは1次元配列として返されますが、これは2D画像です。j番目の列のi番目の行にアクセスするには、のように計算column * row_width + rowしbitmap.buffer[i * face->glyph->bitmap.pitch + j]ます。ループで

使用bitmap.widthした配列にアクセスするとbitmap.pitch、ピクセルの各行の長さが等しいbitmap.widthので、バッファの「幅」がであることがわかりbitmap.pitchます。

次のコードでは、すべての行と列が並べ替えられ、ピクセルの明るさに応じて異なる文字が描画されます。

for (size_t i = 0; i < face->glyph->bitmap.rows; i++) {
  for (size_t j = 0; j < face->glyph->bitmap.width; j++) {
    unsigned char pixel_brightness =
        face->glyph->bitmap.buffer[i * face->glyph->bitmap.pitch + j];

    if (pixel_brightness > 169) {
      printf("*");
    } else if (pixel_brightness > 84) {
      printf(".");
    } else {
      printf(" ");
    }
  }
  printf("\n");
}
