Swift 5.2. Resumen de todos los cambios

A finales de marzo, se lanzó Swift 5.2 para Xcode 11.4. Ha mejorado el diagnóstico de errores, el análisis de dependencia y la funcionalidad expandida de SwiftPM. Ya se ha publicado una descripción general de algunos cambios en Habré , en el mismo artículo se considera la evolución del lenguaje en sí con posibles ejemplos de uso.


 

SE-0249 KeyPath como una función


El acceso directo pone keyPath en una función, donde el parámetro de entrada es el objeto mismo, y el resultado es su propiedad.

Veamos un ejemplo. Cree una matriz de modelos simples:

struct Model {
    let isHidden: Bool
}
 
let modelArray = [Model(isHidden: true), Model(isHidden: false)]

Filtre la matriz por la propiedad isHidden. A continuación hay 3 ejemplos con el mismo resultado:

// 1
let funcIsHidden: (Model) -> Bool = \.isHidden
modelArray.filter(funcIsHidden)
 
// 2
modelArray.filter(\.isHidden as (Model) -> Bool)
 
// 3
modelArray.filter(\.isHidden)

La funcionalidad declarada no funciona en el siguiente ejemplo:

//   
let selfFunc: (Model) -> Model = \.self
modelArray.compactMap(selfFunc)
 
// ERROR: Expression type '(Model) -> Model' is ambiguous without more context
modelArray.compactMap(\.self as (Model) -> Model)
 
// ERROR: Key path value type 'Optional<_>' cannot be converted to contextual type '_'
modelArray.compactMap(\.self)

Además, a diferencia de keyPath, el autocompletado no funciona.

Es conveniente usarlo para trabajar con matrices en funciones como filtro, mapa, reducción, clasificación y similares.

Los subíndices SR-6118 pueden contener configuraciones predeterminadas


Todos los parámetros de la función se pueden establecer en el valor predeterminado. Cree una estructura de cuadro que contenga una matriz de elementos y una función de subíndice para acceder a ellos.

struct Box {
    let items: [String]
    
    subscript(_ index: Int = 0) -> String {
        items[index]
    }
}

Ahora, para acceder al primer elemento, puede omitir el índice:

let box = Box(items: ["laptop, , mobile phone"])
let firstItem = box[] // "laptop"

Las funciones locales SR-2189 admiten parámetros predeterminados desde la visibilidad externa


En la práctica diaria, las funciones locales rara vez se utilizan. Puede ser difícil justificar su uso. Sin embargo, admito que en ciertos casos esto puede ser útil.

Por ejemplo, crearemos una función dentro de la cual describiremos una local:

func getXPosition() {
    func calculateWidth(x: CGFloat = minX) -> CGFloat { ... }
    let minX: CGFloat = 0
    ...
}

Como parámetro predeterminado para la función CalculateWidth, podemos usar los valores dentro de la función getXPosition.

SE-0253 Uso de valores como funciones


La funcionalidad, similar a @dynamicCallable, le permite usar un valor como función. Pero es una implementación de la "llamada estática".

En la práctica, todo es extremadamente simple: para llamar a funciones, puede referirse a los valores como métodos.

Crear una estructura de jugador:

struct Player {
    private(set) var isRunning = false
    
    mutating func callAsFunction() {
        isRunning.toggle()
    }
}

Cree una instancia de Player y consúltela como una función:

var player = Player()
print(player.isRunning) // false
player()
print(player.isRunning) // true

Al mismo tiempo, está prohibido lanzar y, por lo tanto, pasar un objeto como una función:

// ERROR: Cannot convert value of type 'Player' to type '() -> Void' in coercion
let playerAsFunc = player as () -> Void

Puede agregar tantos métodos como callAsFunction a una clase, estructura o protocolo:

extension Player {
    func callAsFunction(isRunning: Bool = false) throws { ... }
    func callAsFunction() -> Bool { ... }
}

La aplicación es posible en valores que son funciones matemáticas, expresiones complejas o en casos donde el valor tiene una función dominante. Sin embargo, no debe abusar de esta funcionalidad, ya que puede ser engañosa.

SR-4206 Error solucionado con anulación de función con parámetro genérico


Ahora, anulando la función, no puede cambiar ni agregar restricciones en el tipo genérico. Como ejemplo, cree una clase CarBuilder y herede de ella anulando el método add:

protocol PowerfullEngine { }
 
class CarBuilder {
  func add<T>(engine: T) {}
}
 
class MercedesBuilder: CarBuilder {
    //  
    override func add<T>(engine: T) {}
    // ERROR: Overridden method 'add' has generic signature <T where T : PowerfullEngine> which is incompatible with base method's generic signature <T>; expected generic signature to be <T>
    override func add<T: PowerfullEngine>(engine: T) {}
}

SR-11298 Extender un protocolo sin restricciones de clase hereda esta restricción


En el caso de restricciones en Self, la extensión aplicará la restricción.

Por ejemplo, cree el menú y el protocolo de extensión con una restricción de clase:

protocol Menu {}
 
class SideMenu: Menu {
  var sectionHeight = 0
}
 
extension Menu where Self: SideMenu {
  var menuHeight: Int {
    get { sectionHeight * 20 }
    set { sectionHeight = newValue / 20 }
  }
}

En este caso, el configurador no es mutante, como si el protocolo tuviera una restricción de clase.

SR-11429 Fundición para funciones con etiquetas


Cuando las funciones de conversión para escribir etiquetas ahora se eliminan. Gracias a esto, puede emitir funciones con etiquetas. Anteriormente, esta funcionalidad solo funcionaba en funciones sin etiquetas:

func setX(x: Int) {}
(setX as (Int) -> Void)(5)

Sin embargo, los valores predeterminados no se guardarán:

func setPoint(x: Int, y: Int = 0) {}
(setPoint as (Int, Int) -> Void)(5, 1)

Esto también se aplica a los genéricos, por lo que este código ya no es válido.

typealias Point<T> = T
func set(x: Int) {}
 
//  
(set as Point)(5) 
 
// ERROR: Extraneous argument label 'x:' in call
(set as Point)(x: 5)

SR-11841 La función de filtro (_ :) se llama en el orden esperado en colecciones diferidas


Cree una colección perezosa y llame al método de conteo:

let lazyArray = [0]
    .lazy
    .filter { _ in print("A"); return true }
    .filter { _ in print("B"); return true }
 
//  A B
_ = lazyArray.count

Antes, el resultado de la llamada era lo contrario: B A.

SR-2790 Varios inicializadores de tipo de la familia UnsafePointer / UnsafeBufferPointer ahora emiten una advertencia


La restricción se aplica a cadenas, matrices y parámetros de entrada que no existen fuera de la llamada a la función.

Cree una estructura que tome UnsafePointer como parámetro durante la inicialización:

struct Game {
    let info: UnsafePointer<Int8>
}
 
func createGame() {
    var i: Int8 = 0
    
    // WARNING: Initialization of 'UnsafePointer<Int8>' results in a dangling pointer
    _ = Game(info: .init(&i))
    
    // WARNING: Passing '[Int8]' to parameter, but argument 'character' should be a pointer that outlives the call to 'init(character:)'
    _ = Game(info: [1, 2])
    
    // WARNING: Passing 'String' to parameter, but argument 'character' should be a pointer that outlives the call to 'init(character:)
    _ = Game(info: "")
}
