リモートSSH：ヒントとハック

リモートSSHに関する以前の投稿では、Visual Studio CodeのRemote-SSH拡張機能を使用して Linux仮想マシンを構成し、仮想マシンに接続する方法を説明しました。この投稿では、リモート構成を最大限に活用するために使用できるいくつかのヒントとテクニックについて説明します。



リモートSSHを使用して接続する


Visual Studio Code Remote-SSH拡張機能を使用すると、SSHを使用してリモートマシンまたは仮想マシンに接続でき、すべてVS Code内から接続できます。拡張機能がまだインストールされていない場合は、「拡張機能」（⇧⌘X）タブで「リモートssh」を検索できます。



拡張機能をインストールすると、ステータスバーの左下隅にインジケーターが表示されます。このインジケーターは、VSコード（ローカルまたはリモート）がどのコンテキストで機能するかを示します。インジケーターをクリックして、リモート拡張コマンドのリストを表示します。



SSH構成ファイル


以前のリモートSSHに関する投稿では、1台のマシンにのみ接続し、プロンプトが表示されたら「user @ host」と入力しました。複数のリモートサーバーまたはローカル仮想マシンに定期的に接続する場合は、すべてのユーザー名、アドレス、および追加の構成パラメーターを覚えておく必要なく、より良い接続方法があります。

OpenSSH  は、 さまざまなSSH接続のすべてを保存するための構成ファイルの使用をサポートしています  。 SSH構成ファイルを使用するには、リモートインジケーターをクリックしてリモートコマンドを呼び出し、「構成ファイルを開く」を選択して、「ユーザー/ {yourusername} /。Ssh / config」にあるファイルを選択します。



次にSSH構成ファイルの例を示します。

#      SSH: https://linux.die.net/man/5/ssh_config
Host python-linux-vm
    HostName <vm address>
    User sana
    IdentityFile ~/.ssh/id_python_vm

Host node-vm
    HostName <vm address>
    User sana
    Port 5522
    IdentityFile ~/.ssh/id_node_vm

SSH構成ファイル形式で指定できる 他の多くの構成オプションがあります。このファイルにアドオンが表示されます。IntelliSenseの（⌃Space）をクリックして、構成オプションの詳細を確認できます。

上記で使用したオプション：
ホスト ホストの覚えやすいエイリアス。
ホスト名 サーバーのホスト名（サーバーのIPアドレスを使用できます）。
ユーザー SSH経由でログインするように指定したユーザー。
ポート SSH経由の接続に使用されるポート。デフォルトのポートは22ですが、一意のポートを指定した場合は、ここで構成できます。
IdentityFile 秘密鍵を保存したファイルの場所。
所有しているすべてのホストの情報を追加できます。構成ファイルを保存すると、これらのノードがリモートエクスプローラーに表示され、このノードで開いたフォルダーも表示されます。各ホストまたはフォルダーの横にあるアイコンを選択すると、新しいVSコードウィンドウ（インスタンス）が起動し、そのホストに接続します。以下のスクリーンショットでは、「python-linux-vm」リモートマシンに接続しています。リモートエクスプローラーには、過去に接続したフォルダーと、リモートマシンからリダイレクトされたポートが表示されます。



プロキシコマンド


場合によっては、会社のイントラネットを介して、またはファイアウォールの背後で、デスクトップまたはラップトップコンピュータからリモートコンピュータに接続する必要があります。この場合、中間サーバーまたはジャンプボックスを使用できます。このタイプの構成は、固定されたホストのセットからのみSSH接続を受け入れるように構成された安全なシステムを実行している場合に役立ちます。

Remote-SSH拡張機能でジャンプボックス構成を使用するには、構成パラメーターを使用できますProxyCommand。この構成では、ジャンプボックスとのバックグラウンドSSH接続が開き、プライベートIPアドレスを介してターゲットに接続します。次のように、SSH構成ファイルで

構成パラメーターProxyCommandを設定できます。

# Jump box   IP-
Host jump-box
    HostName 52.179.157.97
    User sana
    IdentityFile ~/.ssh/jumpbox

#     IP-
Host target-box
    HostName <IP address of target>
    User sana
    IdentityFile ~/.ssh/target
    ProxyCommand ssh -q -W %h:%p jump-box

コントロールマスター


2要素認証、パスワードベース、パスフレーズ付きのSSHキーなど、キーベースの認証以外の認証方法を使用してリモートSSHホストに接続する場合、必要な情報を数回入力する必要がある場合があります。

複数のSSH接続を開く代わりに、オプションControlMaster（macOS / Linuxクライアントのみ）を使用して既存の接続を再利用し、パスフレーズを入力する必要がある回数を減らすことができます。

この機能を使用するには、SSH構成ファイルに以下を追加します。

Host *
    ControlMaster auto
    ControlPath ~/.ssh/sockets/%r@%h-%p
    ControlPersist 600

スタンドアロンリモートコンピューター


ファイアウォールによって制限されている場合、または会社が仮想マシンをブロックしていて、仮想マシンがインターネットに接続できない場合、VS CodeはVS Codeサーバーと呼ばれるコンポーネントをリモートマシンにダウンロードする必要があるため、Remote-SSH拡張機能は仮想マシンに接続できません。

ただし、リモート拡張の新しいユーザーパラメータ-SSHを使用して、この問題を解決できるようになりました。このオプションを有効にするremote.SSH.allowLocalServerDownloadと、拡張機能は最初にVS Codeサーバーをクライアントにインストールし、次にSCPを介してサーバーにコピーします。

注意。これは現在実験的な機能ですが、将来のリリースではデフォルトで有効になる予定です。

リモート-SSH Nightly拡張機能


新しいアップデートや実験的な機能が利用可能になり次第、テストする場合は、リモート拡張機能-SSH Nightlyをインストールします（最初に安定したRemote-SSH拡張機能を削除します）。これは拡張機能の夜通しのビルドで、安定版に追加する前に新しい機能と設定を試します。
