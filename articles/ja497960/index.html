Linuxネットワークインストール

最近、異常な状況でCentos 7をインストールしました。

まず、自宅で。つまり、IPMIを備えたサーバーではなく、ローカルコンピューターを扱っていました。

次に、ディスクドライブがないため、通常は起動可能なUSBフラッシュドライブを使用します。この場合、新しいコンピューターはWindowsインストーラーを使用してUSBフラッシュドライブからのみ起動するため、役に立たないことがわかりました。マザーボードのレビューによると、問題は新しいものではありません。

私の処分はネットワークでした。Centos 7をPXEおよびiPXEにインストールする例を示します

それでは始めましょう。

ipxe経由でLinuxをインストールする方法
pxeを使用してLinuxをインストールする方法

PXEによるインストール


イーサネットケーブルをコンピューター1に接続します。コンピューター1には、DHCP、TFTP、コンピューター2があり、OSをインストールする必要があります。

ネットワークアダプターの静的設定をcomputer1に追加します。私のアドレスは192.168.1.50です。TFTPを

ダウンロードしてインストールします。このプログラムでは、DHCPおよびTFTPサーバーを構成し、そこからIPアドレスとインストールファイルをcomputer2に提供します。 ファイアウォールをオフにし、管理者権限でtftpdを実行します。写真と同じ設定にします。tftpdを再起動する必要がある場合があります。 computer2のブートメニューで、ネットワークアダプターを選択します。コンピューター1のtftpdウィンドウに進行状況バーが表示されます。 その後、computer2で、OSインストーラーウィンドウが読み込まれます。













iPXE経由でLinuxをインストールする


イメージipxe.isoをダウンロードします。Rufus、このイメージに基づいて起動可能なフラッシュドライブを作成します。

install.ipxeスクリプトを任意のWebサーバーに配置します。ローカルコンピュータで Webサーバーを起動する方法については、こちらをご覧ください。スクリプトのアドレスは次のようになりますsitename.ru/install.ipxe Centos

7をインストールするためのinstall.ipxeスクリプトの内容

#!ipxe
set base http://mirror.centos.org/centos/7/os/x86_64
prompt -k 0x197e -t 2000 Press F12 to install CentOS... || exit
kernel ${base}/images/pxeboot/vmlinuz initrd=initrd.img repo=${base}
initrd ${base}/images/pxeboot/initrd.img
boot

Centos 7をインストールするためのこのスクリプトと同様に、別のOSをインストールするための独自のスクリプトを準備できます。

Linuxをインストールするコンピューターとイーサネットルーターケーブルを接続します。フラッシュドライブを挿入し、そこから起動します。F12を押すと、ipxeコンソールが表示されます。次のコマンドを使用してIPアドレスを取得し、スクリプトをダウンロードします

iPXE> dhcp
    DHCP (net0 52:54:00:12:34:56).... ok
iPXE> chain http://sitename.ru/install.ipxe

その後、OSインストーラーウィンドウが読み込まれます。
