Apache WebサーバーでのOTP認証

この投稿では、JASとRadiusを使用してApacheでワンタイムパスワード認証を構成する方法について説明します。JAS（JaCarta Authentication Server）は、スタンドアロンの高性能認証サーバーであり、OTPトークンとU2Fトークン、およびモバイルデバイス用のソフトウェアトークンをサポートしています。次のソフトウェアが構成プロセス中に使用されました-JAS 1.x、Linux Debian、Apache2、およびJaCarta WebPassトークン、JaCarta U2Fトークン、イベント別のOTPトークン、ソフトウェアGoogle認証システム、Yandexキーなど。
この問題を解決するには、apache2サーバーにradius認証プラグインを使用できます。Webサーバー上のすべてのアクションが配布のために実行されますLinux Debianストレッチ。

Webサーバーのセットアップ


1. apache2のradiusモジュールをインストールします。
sudo apt install libapache2-mod-auth-radius
sudo a2enmod auth_radius

2.必要なサイトを選択し、その構成で変更を行います。
   ServerName web1.test2.ru
   ServerAdmin webmaster@localhost
   DocumentRoot /var/www/html
#     radius (, ,   )
   AddRadiusAuth 127.0.0.1 P@ssw0rd 5
   AddRadiusCookieValid 60

「ディレクトリ」セクションでは、次のことも示します。
 AuthType Basic
 AuthName "Radius Authentication"
 AuthBasicAuthoritative Off
 AuthBasicProvider radius
 AuthRadiusAuthoritative on
 AuthRadiusActive On
 Require valid-user


3. Webサーバーデーモンを再起動します。
sudo apache2ctl restart

Webサーバーの準備が整いました。
