タスク2〜1で完了


コードは3つの要求に応答して実行され、コンソールに応答を出力して、「完了」すると想定されていました。しかし、問題が発生しました...状況を修正してください。

;(function() {
  let iter = {
    [Symbol.iterator]: function* iterf() {
      let fs = ['/1', '/2', '/3'];
      for (const req in fs) {
        yield fetch(req);
      }
    }
  };
  for (const res of iter) {
    console.log(res);
  }
  console.log('done');
})();

研究課題


ここには何がありますか？これは、反復可能ITERのオブジェクト有する周知Symbol.iteratorの記号定義を介して発電機機能。配列fsは関数の本体で宣言され、その要素はフェッチ関数に入れられて要求を送信し、各関数呼び出しの結果はyieldを介して返されます。フェッチ関数はどのようなリクエストを送信しますか？配列fsのすべての要素は、それぞれ番号1、2、3のリソースへの相対パス（path）です。したがって、次の例のように、location.originと次の番号を連結することにより、完全なURLが取得されます。

GET https://www.example.com/1

次に、各リクエストを結果の出力で順番に実行するために、for-ofを介してiterオブジェクトを反復処理します。つまり、「完了」を出力します。しかし、それはうまくいきません！問題は、フェッチが非同期であり、応答ではなくプロミスを返すことです。したがって、コンソールには次のようなものが表示されます。 実際、タスクはこれらの同じ約束を解決することになります。

Promise {pending}
Promise {pending}
Promise {pending}
done



非同期/待機しています


最初に考えたのは一緒にプレイすることであってもよいPromise.all：それは私たちの与える反復可能なオブジェクト、その後、コンソールへの出力が「完了」されています。しかし、彼は（条件に応じて）要求の順次実行を提供せず、それらをすべて送信して、一般的な解決の前に最後の回答を待ちます。

ここでは最も簡単な解決策は以下のようになり待つ中のため-のボディコンソールに出力する前に次の約束の解像度を待ちます。

for (const res of iter) {
  console.log(await res);
}

以下のためにのawait終了時に表示されるように「完了」の仕事と、あなたは非同期を介してメインの機能を確認する必要があり非同期：

;(async function() {
  let iter = { /* ... */ };
  for (const res of iter) {
    console.log(await res);
  }
  console.log('done');
})();

この場合、問題はすでに（ほぼ）解決されています。

非同期イテレーターとジェネレーター


メイン関数は非同期のままにしますが、awaitの場合、for-of本体よりもこのタスクの方がエレガントです。これは、for-await-ofによる非同期反復の使用です。つまり、

for await (const res of iter) {
  console.log(res);
}

すべてがうまくいきます！しかし、非同期反復に関するこの提案の説明に目を向けると、興味深いのは次のとおりです。

非同期反復可能オブジェクトを反復するfor-of反復ステートメントのバリエーションを紹介します。非同期のfor-ofステートメントは、非同期関数と非同期ジェネレーター関数内でのみ許可されます

つまり、オブジェクトは反復可能であるだけでなく、新しいよく知られたシンボルSymbol.asyncIteratorと「非同期のジェネレーター関数」を介して「asyncIterable」である必要があります。

let iter = {
  [Symbol.asyncIterator]: async function* iterf() {
    let fs = ['/1', '/2', '/3'];
    for (const req in fs) {
      yield await fetch(req);
    }
  }
};

それでは、通常のイテレータとジェネレータでどのように機能しますか？はい、ただ暗黙のうちに、この言語の多くのように。このfor-await-ofはトリッキーです。オブジェクトがiterableのみの場合、非同期で反復するとき、解決を期待してPromiseで要素を（必要に応じて）ラップすることにより、オブジェクトをasyncIterableに「変換」します。彼はアクセル・ラウシュマイヤーの記事でより詳細に話しました。 おそらく、Symbol.asyncIteratorを使用すると、for-await-ofを介して非同期反復のasyncIterableオブジェクトを明示的に作成したため、より正確になります。

、必要に応じて、for-ofの通常のイテレータでオブジェクトを補足する機会を残しながら。JavaScriptでの非同期反復に関する1つの記事で有用かつ十分なものを読みたい場合は、こちらをご覧ください。

非同期for-ofの一部はまだドラフト段階ですが、最新のブラウザー（Edgeを除く）と10.xのNode.jsですでにサポートされています。これが誰かを悩ますなら、あなたはいつでも、例えば反復可能なオブジェクトのための約束の連鎖のためにあなた自身の小さなポリフィルを書くことができます：

const chain = (promises, callback) => new Promise(resolve => function next(it) {
    let i = it.next();
    i.done ? resolve() : i.value.then(res => {
      callback(res);
      next(it);
    });
  }(promises[Symbol.iterator]())
);

;(async function() {
  let iter = { /* iterable */ };
  await chain(iter, console.log);
  console.log('done');
})();

このような方法で、要求の送信と応答の処理を順番に考え出しました。しかし、このタスクにはもう1つの小さな問題があります...

問題3〜ファクター19


数字2019には何回あります！（2019からの階乗）19という数字は発生しますか？答えとともに、JavaScriptソリューションを提供します。

研究課題


問題は表面にあります。サブストリング「19」のすべての出現数を見つけるには、非常に大きな数のレコードが必要です。numberの問題を解決すると、非常にすばやくInfinity（170以降）に遭遇し、何も得られません。さらに、数値を表すための形式float64は15〜17文字の精度しか保証せず、完全なだけでなく正確な数値の記録も取得する必要があります。したがって、主な困難は、この多数を累積するための構造を決定することです。

大きな整数


言語の革新に従うと、タスクは簡単に解決されます。型番号の代わりに、新しい型BigInt（ステージ3）を使用できます。これにより、任意の精度の数値を操作できます。古典的な再帰的階乗関数とString.prototype.splitを介して一致を見つけると、最初のソリューションは次のようになります。

const fn = n => n > 1n ? n * fn(n - 1n) : 1n;
console.log(fn(2019n).toString().split('19').length - 1); // 50

ただし、スタック上の2000の関数呼び出しはすでに危険な場合があります。ソリューションを末尾再帰に導入しても、Tail呼び出しの最適化はまだSafariのみをサポートしています。ここでの階乗問題は、算術サイクルまたはArray.prototype.reduceを介して解決する方が楽しいです。

console.log([...Array(2019)].reduce((p, _, i) => p * BigInt(i + 1), 1n).toString().match(/19/g).length); // 50

これは非常に長い手順のように思えるかもしれません。しかし、この印象は欺瞞的です。見積もると、2000を少し超える乗算を費やす必要があります。クロームのi5-4590 3.30GHzでは、問題は平均4-5ms（！）で解決されます。

計算結果を含む文字列内の一致を見つけるための別のオプションは、グローバル検索フラグ/ 19 / gを使用した正規表現によるString.prototype.matchです。

大きな算数


しかし、このBigInt（およびライブラリ）がまだない場合はどうなりますか？この場合、長い計算を自分で行うことができます。この問題を解決するには、大小を乗算する関数のみを実装するだけで十分です（1から2019までの数値を乗算します）。たとえば、次の行に多数の値と乗算の結果を保持できます。

/**
 * @param {string} big
 * @param {number} int
 * @returns {string}
 */
const mult = (big, int) => {
  let res = '', carry = 0;
  for (let i = big.length - 1; i >= 0; i -= 1) {
    let prod = big[i] * int + carry;
    res = prod % 10 + res;
    carry = prod / 10 | 0;
  }
  return (carry || '') + res;
}
console.log([...Array(2019)].reduce((p, _, i) => mult(p, i + 1), '1').match(/19/g).length); // 50

ここでは、学校で教えられたように、行の最後から最初に列をビット単位で乗算します。しかし、このソリューションではすでに約170msが必要です。

数値レコードの複数の数字を一度に処理することで、アルゴリズムを多少改善できます。これを行うには、関数を変更し、同時に配列に移動します。これにより、毎回行をいじる必要がなくなります。

/**
 * @param {Array<number>} big
 * @param {number} int
 * @param {number} digits
 * @returns {Array<number>}
 */
const mult = (big, int, digits = 1) => {
  let res = [], carry = 0, div = 10 ** digits;
  for (let i = big.length - 1; i >= 0 || carry; i -= 1) {
    let prod = (i < 0 ? 0 : big[i] * int) + carry;
    res.push(prod % div);
    carry = prod / div | 0;
  }
  return res.reverse();
}

ここで、多数のアレイによって表され、各要素そのうち約情報格納桁の使用数レコードからの桁数。たとえば、数字が 3の2016201720182019は、次のように表されます。 結合前に行に変換する場合、先頭のゼロを覚えておく必要があります。係数関数が使用、文字列によって算出階乗を返すMULT関数を計算する数の「大規模な」表現で一度に処理指定した桁数を有します。

'2|016|201|720|182|019' => [2,16,201,720,182,19]
