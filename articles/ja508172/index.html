Azure Functionは、キュー内のオブジェクトの出現などのイベントに応答し、ストレージおよびクラウドオブジェクトと自動的に通信します。新しい写真がストレージに配置されたときに何らかのアクションを実行したい場合、既製のバイナリファイルが対応する関数の入力に供給され、作業のロジックを記述するだけで済みます。

おそらくこれが最も難しい段階です。多くの場合、ボットはNode.jsまたはC＃で記述されているため、これらの言語の場合、ポータルには多くのドキュメント、例、サポートがあります。Pythonの場合、状況は少し悪くなりますが、理解することができます。



以下は、ボットをクラウドに公開するための詳細なガイドです。つまり、ボットの場合、2つのアプリケーションを作成する必要があります。



  • Webアプリケーション-ボットを実行します。

  • アプリケーションをTelegramに接続するオブジェクト。



まず、Azure Directoryで "blenderbot"というアプリケーションを作成し、認証用のパスワードを渡します。  



az ad app create --display-name "blenderbot" --password "%PASSWD%" --available-to-other-tenants


アプリケーションが作成されると、いくつかのappIDが出力されます。







JSON出力からそれを取り出し、変数に格納します。







次のチームでは、WebアプリケーションとTelegramに関連付けられているオブジェクトの両方をデプロイします。







このテクノロジーは、deploymentTemplatesを支援します-1つのコマンドでクラウド内に複数のオブジェクトを一度に作成できます。deploymentTemplatesには、以前にで生成したボットコードが付属していますcookiecutter



。これは便利です。deploymentTemplatesを呼び出すと、必要なものがクラウドにすべて作成されます。



次に、すべてのファイルを含むディレクトリのzipアーカイブを作成しますrequirements.txt



。 



bot.zip


... Azure Deployを呼び出し、すべてをクラウドに配置します。 



az webapp deployment source config-zip


手順は、特に外側から簡単ではありません。しかし、それらは指示でよく説明されています-私がそれをしたのは2回目です。
