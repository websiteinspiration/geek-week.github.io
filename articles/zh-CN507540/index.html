使用htmlix.js中的自定义事件

本文将研究在Htmlix中使用用户事件的基本原理，并创建一个包含四个组件的小型应用程序：用于输入用户名的表格，问候语，用于应用程序的退出按钮以及具有应用程序用户的数组。



可以在此处单击一个现成的示例。



您可以在此处阅读htmlix api 。



在表单中输入用户名后，应用程序将在问候语中显示其名称，然后单击退出按钮，并在数组中为用户创建一个新容器，并将数据保存在localStorage中。重新加载页面后，它将使用localStorage的名称。单击“退出配置文件”按钮将从所有组件中删除数据，并从localStorage中删除变量。



让我们创建四个组件：登录表单-表单，问候语-问候语，注销按钮-注销以及具有所有访问者的数组-users_array，并了解如果不使用用户事件，如何在它们之间创建通信。

在上面的示例中，我们将来自表单组件的“ input”属性数据记录到“ text”变量中。现在，我们需要在Greeting和Logout组件的属性中显示它们，并在users_array数组中创建一个新容器。要了解为什么需要自定义事件，让我们首先尝试以各种方式执行此操作而不使用它们。



1.方法-在表单的click方法本身中，直接转到每个组件并设置text属性： 




        event.preventDefault();
        var text = this.parent.props.input.getProp();
                //  
                this.rootLink.state["greeting"].props.user_name.setProp(text);
                this.rootLink.state["logout"].props.user_name.setProp(text);
                  
                  ///  
                 this.rootLink.state["users_array"].add({user_name: text});


这种方法的缺点很明显，如果我们从其他地方获取该属性的数据怎么办？例如，基于本地存储，它们将自动加载。



2.方法是在stateMethods对象中创建一个方法，并在数据更改时调用它： 




ststeMethods: {

         entryUser: function(text){

                this.state["logout"].props.user_name.setProp(text);
                 this.state["users_array"].add({user_name: text});
       }
}

在这种情况下，我们设法避免了代码的重复，但是问题是，处理组件属性的方法不是在组件内部，而是在外部函数中。如果在设置属性之前我们需要以某种方式为每个组件设置不同的格式，或者我们不需要设置一个属性，而是设置多个属性，那么当组件本身使用其属性时会更好。



3.方法是将使用属性的整个逻辑转移到组件中，并在其中创建用于调用某些动作的方法。

现在已经封装了组件，但是，我们通过其名称访问组件。如果我们以后要更改组件或方法的名称，或者完全删除该组件，或者我们将有许多侦听相同属性的组件该怎么办？否则我们将拥有许多此类属性，因此我们将不得不创建许多此类方法。



为了简化此任务，您可以使用自定义事件。该组件订阅该事件，并在事件发生时对其属性进行某些操作。



现在，使用自定义事件来编辑代码： 



var StateMap = {

   eventEmiters: {	
             //   -  	
             ["emiter-entry-user"]: {prop: ""}, 		
   },
   form: {
	container: "form",
	props: ["input", "click"],
	methods: {
			
		click: function(){
		    event.preventDefault();
		    var text = this.parent.props.input.getProp();

		     //        
		    this.rootLink.eventProps["emiter-entry-user"].setEventProp(text); 	
		    window.localStorage.setItem('user_name', text);							
		}			
	},		
},	
greeting: {
		
	container: "greeting",
	props: [ "user_name", ['listen_entry_user', "emiter-entry-user", "" ] ], //   "emiter-entry-user"
	methods: {
			
		listen_entry_user: function(){
		   //      
		   this.parent.props.user_name.setProp( this.emiter.getEventProp() );
		},	
	},		
},
logout: { 
		
	container: "logout",
	props: [ "user_name", ["listen_entry_user", "emiter-entry-user", "" ]], //   "emiter-entry-user"
	methods: {
			
		listen_entry_user: function(){
		     //       
		     this.parent.props.user_name.setProp( this.emiter.getEventProp() );
		},			
	},	
},
    //      , ..            
users_array: { 
		
	arrayProps: [ ['listen_entry_user', "emiter-entry-user", ""] ], //   "emiter-entry-user"
	   arrayMethods: {
			
		listen_entry_user: function(){
			//       
			this.parent.add( {user_name: this.emiter.getEventProp()} );
		},			
	},		
	container: "user",
	props: [ "user_name",   ],
	methods: {						
		}
   },        	


}
window.onload = function(){
	
		
	var HM = new HTMLixState(StateMap);
	var name = window.localStorage.getItem('user_name');
	
	if(name != null)HM.eventProps["emiter-entry-user"].setEventProp(name); ///  "emiter-entry-user"        	
}


因此，我们摆脱了中间方法“ entryUser”，只是通过将新数据以表单的形式并在页面加载后传递给事件来引发事件。现在，如果我们需要更改组件中的某些内容甚至删除它，则无需更正每个函数中的代码，该代码可以是任何数字。要取消订阅任何事件，您可以简单地删除组件中的侦听器，或使用属性-订户中的disableEvent（）方法暂时将其禁用。
