Djangoコアの概念


Djangoについて少し話しましょう。 DjangoはMVT（モデルビューテンプレート）のアーキテクチャパターンを実装しています。これは、Ruby on RailsとLaravelを実行する一般的なMVC（モデルビューコントローラ）とは少し異なります。

モデル Django のモデルは、データベースのデータスキームを記述します。 Django ORMを使用すると、フィールドやその他のデータ型を個別に記述し、移行を行って開発を簡略化できます。

ビュー Djangoのビューでは、アプリケーションの基本的なロジックとアルゴリズムを設定し、データベースからさまざまなデータを取得するか、それを操作します。ビューは通常、リクエスト\レスポンス関数に基づいています。応答は通常、HTTPリダイレクト、HTTPエラー（404）、MimeTypes、または何らかのパターンです。

テンプレートDjangoのテンプレートは、特別なテンプレート言語Djangoを使用したシンプルなHTMLコードです。 DTL（Djangoテンプレート言語）は、ページのコンテンツを動的に変更できる言語です（たとえば、許可されたユーザーの名前に応じて、ページのユーザー名を変更します）。

設定 Webアプリケーションのすべての設定を含むDjangoの設定ファイル。秘密鍵、テンプレートフォルダー、ミドルウェア（たとえば、他のユーザーがプライベートアルバムを閲覧できないようにする）、データベース接続などが含まれます。

URLルーティング構成ファイルは、AngularやLaravelとほぼ同じです。これにより、ビューがURLリクエストに関連付けられます。

管理ページ Djangoは元々、ニュースサイトの迅速なプロトタイピングと展開のために設計されていたため、管理パネルはデフォルトで含まれています。

class TodoList(models.Model):
	title = models.CharField(max_length=250)
	content = models.TextField(blank=True) # 
	created = models.DateField(default=timezone.now().strftime("%Y-%m-%d")) #  
	due_date = models.DateField(default=timezone.now().strftime("%Y-%m-%d")) #      
	category = models.ForeignKey(Category, default="general",on_delete=models.PROTECT) # foreignkey          
	class Meta: #       
        ordering = ["-created"] #     
	def __str__(self):
    	return self.title
