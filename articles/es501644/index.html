Análisis de rendimiento de animación CSS

¿Qué elegir para animar elementos de páginas web? JavaScript o CSS? Esta pregunta algún día se verá obligada a preguntar a todos los desarrolladores web. Y tal vez, y ni una sola vez.

Los programadores de JavaScript han creado muchas bibliotecas para la animación del navegador. Y parece que todos los que estaban alrededor estaban inclinados a usar estas bibliotecas como una solución preparada para la animación. Pero vamos más despacio. ¿Es correcto? ¿Debo animar los elementos de la página web usando JavaScript? ¿Tal vez pueda confiar en los mecanismos CSS estándar y así lograr animaciones de alta calidad y alto rendimiento?



Como estás leyendo esto, puedo suponer que estás familiarizado con la animación JavaScript. Por lo tanto, propongo explorar el tema de la animación CSS en sus diversas manifestaciones y, además, propongo hablar sobre el rendimiento de dicha animación.

Información general


Las animaciones CSS se basan en algunas propiedades de elementos. Entre ellos, se pueden observar los siguientes:

  • Una propiedad positionque, en particular, puede tomar los valores absolutey relative.
  • Propiedad transform.
  • Propiedad opacity.
  • Propiedades left, right, top, bottom.

Para comenzar, eche un vistazo a un par de proyectos piloto que utilizan varias propiedades CSS para animar elementos.

Si solo mira las páginas de los proyectos, puede decidir qué se implementa en ellos de la misma manera, aunque utilizando diferentes enfoques. Sin embargo, si mide el rendimiento de estos proyectos utilizando las herramientas de desarrollador de Chrome, resulta que son diferentes.

¿Cómo muestra la página el navegador?


Antes de profundizar en la investigación del rendimiento, debemos comprender cómo el navegador muestra la página y qué acciones se toman si se produce algún cambio en la página.

Así es como se ve el proceso de salida de la página.


Salida de página

Considere los eventos que se reflejan en esta figura.

  1. Recalculate Style - Cálculo de estilos para aplicar a elementos.
  2. Layout - Creación de diseños de elementos y su colocación en la página.
  3. Paint- llenar los píxeles de las capas creadas, es decir, crear imágenes ráster para cada capa. La GPU usará estas imágenes para mostrar la página.
  4. Composite Layer- Diseñar capas y mostrarlas en la pantalla. El resultado será la página terminada.

Un evento Composite Layeres justo el momento en que la CPU se comunica con la GPU para crear una animación. Usando propiedades CSS como transformy opacity, podemos transferir la carga de generar animaciones desde la CPU a la GPU.

Animación y GPU


Una GPU es esencialmente una computadora especializada que se utiliza para trabajar con imágenes. Puede procesar eficientemente grandes cantidades de información gráfica.

Como resultado, cuando ocurre un evento Composite Layer, se crea una capa adicional. Este es el mecanismo por el cual puede evitar volver a representar el elemento animado y otros elementos de la interfaz.

Si observa la información detallada sobre las tareas individuales presentadas en el panelPerformance, podemos concluir que se realizan diferentes acciones en el marco de estas tareas. Entonces, en el caso de la animación superior / izquierda, como parte de la tarea, se recalculan los estilos, se organizan las capas, se actualiza el árbol de capas, se crea y presenta el diseño. En el caso de la animación de traducción, la única tarea que resuelve la GPU es mover la capa sin generar nada en el DOM.

Como resultado, debido al uso de potentes funciones de GPU, al usar la animación de traducción, se libera el hilo principal. Esto mejora el rendimiento de la aplicación. El uso de animación superior / izquierda conduce a una gran carga adicional en el hilo principal.

Resumen


Como resultado, podemos concluir que, usando animaciones CSS, vale la pena prestar atención exactamente a qué recursos del sistema están involucrados en su ejecución. Si es posible, vale la pena usar los que realiza la GPU.
