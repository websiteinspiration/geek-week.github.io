Docker：まず始めに。注意。配置する

アプリケーション（ウェブサイトなど）をデプロイするためにサーバー環境をセットアップする必要があった頻度はどれくらいですか？きっと私が望むよりも頻繁に。

最良のケースでは、これらすべてを自動的に実行するスクリプトがありました。最悪の場合、次のようになります。

  • Dデータベースバージョンxxxをインストールする
  • WebサーバーNバージョンxxなどをインストールします。

このように構成された環境管理は、時間の経過とともに非常に多くのリソースを必要とします。構成の少しの変更でも、少なくとも次のことを意味します。

  • すべての開発者がこれらの変更を認識する必要があること
  • これらの変更はすべて、本番環境に安全に追加する必要があります

そのような変更を追跡し、特別なツールなしでそれらを管理することは困難です。何らかの方法で、環境依存関係の構成に問題があります。開発が進むにつれて、これらの問題を見つけて修正することが難しくなります。

上記では、いわゆるベンダーロックインについて説明しました。アプリケーションの開発、特にサーバータイプでは、この現象が大きな問題になります。この記事では、可能な解決策の1つであるDockerについて検討します。アプリケーションに基づいてアプリケーションを作成、デプロイ、実行する方法を学びます。

それは何で、何と一緒に食べますか？


簡単に言うと、DockerはLXCコンテナを抽象化したものです。つまり、Dockerを使用して起動されたプロセスは、自分自身とその子孫のみを認識します。このようなプロセスは、Dockerコンテナーと呼ばれます。

このようなコンテナーに基づいてある種の抽象化を作成できるようにするために、Dockerにイメージが存在します（/ docker image /）。Dockerイメージに基づいて、コンテナーを構成および作成できます。

データベース、ウェブサーバー、その他の重要な要素がプリインストールされた、何千もの既製のDockerイメージがあります。Dockerのもう1つの利点は、必要なリソースのみを使用するため、メモリ消費の非常に経済的なツールであることです。

接近する


インストールについては、詳しく説明 しません。過去数回のリリースのプロセスは、数回のクリック/チームに簡略化されています。

この記事では、サーバーのNode.jsアプリケーションの例を使用して、Dockerアプリケーションのデプロイメントを分析します。これがその原始的なソースコードです：

 // index
const http = require('http');
 
const server = http.createServer(function(req, res) {
	res.write('hello world from Docker');
	res.end();
});
 
server.listen(3000, function() {
	console.log('server in docker container is started on port : 3000');
});

アプリケーションをDockerコンテナーにパッケージ化するには、少なくとも2つの方法があります。

  • コマンドラインインターフェイスツールを使用して、既存のイメージからコンテナを作成して実行します。
  • 完成したサンプルに基づいて独自の画像を作成します。

多くの場合、2番目の方法が使用されます。

はじめに、公式のnode.jsイメージをダウンロードします。

docker pull node

docker pullコマンドは、Dockerイメージをダウンロードします。その後、docker runコマンドを実行できます。これにより、ダウンロードしたイメージに基づいてコンテナーが作成および実行されます。

docker run -it -d --rm -v "$PWD":/app -w=/app -p 80:3000 node node index.js

このコマンドは、index.jsファイルを起動し、3000ポートを80にマップし、作成されたコンテナーのIDを表示します。もういい！しかし、1つのCLIでは、遠くまでは行きません。サーバー用のDockerfileを作成しましょう。

FROM node
 
WORKDIR /app
RUN cp . /app
 
CMD ["node", "index.js"]

このDockerfileは、現在のバージョンの継承元のイメージ、およびコンテナーコマンドのコピー元のディレクトリと、イメージアセンブリが起動されるディレクトリからのファイルコピーコマンドの開始を記述します。最後の行は、作成されたコンテナで実行するコマンドを示しています。

次に、デプロイするこのDockerfileからイメージを構築する必要があります：docker build -t username / helloworld-with-docker：0.1.0。このコマンドは、新しいイメージを作成し、ユーザー名/ helloworld-with-dockerでマークを付け、0.1.0タグを作成します。

コンテナの準備ができました。docker runコマンドで実行できます。このようにして、ベンダーロックイン問題を解決します。アプリケーションの起動は、環境に依存しなくなりました。コードは、Dockerイメージと一緒に配信されます。これらの2つの基準により、Dockerを実行できる任意の場所にアプリケーションをデプロイできます。

配置する


最初の99％は、残りの99％ほどひどくない。

上記のすべての手順を完了すると、展開プロセス自体がテクノロジと開発環境の問題になります。Dockerをデプロイするための2つのオプションを検討します。

  • Dockerイメージの手動デプロイメント。
  • Travis-CIを使用したデプロイメント。

いずれの場合も、イメージを独立した環境（製品のステージングサーバーなど）に配信することを検討します。

手動展開


このオプションは、継続的な統合環境がない場合に適しています。まず、ステージングサーバーからアクセスできる場所にDockerイメージをアップロードする必要があります。私たちの場合は、DockerHubになります。ユーザーごとに、1つのプライベートイメージリポジトリと無制限の数のパブリックリポジトリを無料で提供しています。

ログインしてDockerHubにアクセスします。

docker login -e username@gmail.com -u username -p userpass

そこでイメージを読み込みます：docker push username / helloworld-with-docker：0.1.0。

次に、ステージングサーバーに移動します（Dockerが事前にインストールされている必要があります）。

アプリケーションをサーバーにデプロイするには、コマンドを1つだけ実行する必要があります。

docker run -d --rm -p 80:3000 username/helloworld-with-docker:0.1.0.

そして、それだけです！画像のローカルレジスタを確認してください。目的の結果が見つからない場合は、username / helloworld-with- dockerと入力して、DockerHubレジストリを確認します。この名前の画像は、すでにアップロードされているため、レジスタで見つけることができます。Dockerはそれをダウンロードし、それに基づいてコンテナーを作成し、その中でアプリケーションを起動します。

これで、アプリケーションのバージョンを更新する必要があるたびに、新しいタグを使用してプッシュし、そのたびにサーバーでコンテナーを再起動するだけで済みます。

PS Travis-CIを使用できる場合、この方法はお勧めできません。

Travis-CIでデプロイする


まず、DockerHubデータをTravis-CIに追加します。それらは環境変数に格納されます。

travis encrypt DOCKER_EMAIL=email@gmail.com
travis encrypt DOCKER_USER=username
travis encrypt DOCKER_PASS=password

次に、受け取ったキーを.travis.ymlファイルに追加します。また、今後区別するために、各キーにコメントを追加します。

env:
  global:
    - secure: "UkF2CHX0lUZ...VI/LE=" # DOCKER_EMAIL
    - secure: "Z3fdBNPt5hR...VI/LE=" # DOCKER_USER
    - secure: "F4XbD6WybHC...VI/LE=" # DOCKER_PASS

次に、ログインして画像をダウンロードする必要があります。

after_success:
  - docker login -e $DOCKER_EMAIL -u $DOCKER_USER -p $DOCKER_PASS
  - docker build -f Dockerfile -t username/hello-world-with-travis.
  - docker tag username/hello-world-with-travis 0.1.0
  - docker push username/hello-world-with-travis

また、画像配信はTravis-CIからさまざまな方法で起動できます。

  • 手動;
  • ssh接続経由。
  • オンライン展開サービス（Deploy Bot、deployhq）;
  • AWS CLI;
  • Kubernates;
  • Dockerデプロイメント用のツール。
