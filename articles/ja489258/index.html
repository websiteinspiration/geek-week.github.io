Pythonで非常に長い整数型はどのように実装されていますか？

この記事の翻訳は、Python開発者コースの学生向けに特別に用意されたものです。




Cなどの低水準言語で記述する場合、整数に適切なデータ型と修飾子を選択することを心配しています。各ステップで、それを使用するだけで十分intか、それとも追加する必要があるlongか、さらにはを追加する必要があるかを分析しますlong double。ただし、Pythonでコードを記述する場合、Pythonはinteger任意のサイズの型の数値を処理できるため、これらの「マイナー」なことについて心配する必要はありません。

Cでは、組み込み関数を使用して2 20000を計算しようとするpowlと、出力が得られますinf。

#include <math.h>

int main(void) {
  printf("%Lf\n", powl(2, 20000));
  return 0;

プレゼンテーションと定義


Integer Pythonでは、これは次のように定義されたC構造です。

struct _longobject {
    PyObject_VAR_HEAD
    digit ob_digit[1];
};

PyObject_VAR_HEADマクロで、に展開されますPyVarObject。これは次の構造を持っています：

typedef struct {
    PyObject ob_base;
    Py_ssize_t ob_size; /* Number of items in variable part */
} PyVarObject;

ある他のタイプPyObject_VAR_HEAD：

  • PyBytesObject
  • PyTupleObject
  • PyListObject

これは、タプルやリストなどの整数の長さが可変であることを意味します。これは、Pythonが巨大数の処理をサポートする方法を理解するための最初のステップです。展開すると、マクロ_longobjectは次のように見なされます。

struct _longobject {
    PyObject ob_base;
    Py_ssize_t ob_size; /* Number of items in variable part */
    digit ob_digit[1];
};

構造体PyObjectには参照カウント（ガベージコレクション）に使用されるいくつかのメタフィールドがありますが、これについて話すために、別の記事が必要です。これにob_digit少し焦点を当てるフィールドob_size。

解読 ob_digit


ob_digitタイプの単位長の静的に割り当てられた配列ですdigit (typedef uint32_t)。これは配列であるためob_digit、主に数値へのポインターであり、必要に応じて、malloc関数を使用して任意の長さに増やすことができます。このようにして、Pythonは非常に長い数値を表現および処理できます。

通常、Cなどの低レベル言語では、整数の精度は64ビットに制限されていますが、Pythonは任意精度の整数をサポートしています。Python 3以降、すべての数値はフォームbignumで表示され、使用可能なシステムメモリによってのみ制限されます。

解読 ob_size


ob_sizeにアイテムの数を保存しob_digitます。Pythonは値ob_sizeをオーバーライドしてから、配列に含まれる要素の実際の数を決定して、配列にメモリを割り当てる効率を高めob_digitます。

整数の長い演算


Pythonが任意精度の整数をどのように実装するかが明確になったので、次に、それらを使用してさまざまな数学演算がどのように実行されるかを理解します。

添加


整数は「数値で」格納されます。これは、加算が小学校と同じくらい簡単であることを意味し、Pythonソースコードは、これが加算の実装方法であることを示しています。x_addファイルに名前がある関数は、longobject.c2つの数値を追加します。

...
    for (i = 0; i < size_b; ++i) {
        carry += a->ob_digit[i] + b->ob_digit[i];
        z->ob_digit[i] = carry & PyLong_MASK;
        carry >>= PyLong_SHIFT;
    }
    for (; i < size_a; ++i) {
        carry += a->ob_digit[i];

上記のコードスニペットは、関数から取得されますx_add。ご覧のように、数値で数値を反復処理して数値の加算を実行し、結果を計算してハイフネーションを追加します。

減算


加算が行われるのと同じように、減算も行われます。x_subファイルに名前のある関数は、longobject.cある数値から別の数値を減算します。

...
    for (i = 0; i < size_b; ++i) {
        borrow = a->ob_digit[i] - b->ob_digit[i] - borrow;
        z->ob_digit[i] = borrow & PyLong_MASK;
        borrow >>= PyLong_SHIFT;
        borrow &= 1; /* Keep only one sign bit */
    }
    for (; i < size_a; ++i) {
        borrow = a->ob_digit[i] - borrow;
        z->ob_digit[i] = borrow & PyLong_MASK;

上記のコードスニペットは、関数から取得されますx_sub。その中で、数値がどのように列挙され、減算が実行され、結果が計算され、転送が分配されるかがわかります。確かに、それは加算に非常に似ています。

乗算


また、乗算は小学校の数学のレッスンから学んだのと同じ素朴な方法で実装されますが、あまり効率的ではありません。効率を維持するために、Python はKaratsubaアルゴリズムを実装しています。このアルゴリズムは、2つのn桁の数値をO（n log 2 3）の単純なステップで乗算します。

部門およびその他の業務


整数に対するすべての操作はファイルlongobject.cで定義され、それぞれの作業を見つけて追跡するのは非常に簡単です。注意： それぞれの作業の詳細な理解には時間がかかるため、ポップコーンを事前に用意してください。


Pythonは、-5〜256 の範囲の少数の整数をメモリに事前割り当てします。この割り当ては初期化中に発生します。整数を変更できないため（不変性）、これらの事前割り当てされた数値はシングルトンであり、割り当てられる代わりに直接参照されます。つまり、小さな数値を使用/作成するたびに、Pythonは販売する代わりに、以前に割り当てられた数値への参照を返すだけです。

このような最適化は、マクロIS_SMALL_INTと関数get_small_intc で追跡できますlongobject.c。そのため、Pythonは一般的に使用される整数の計算に多くのスペースと時間を節約します。
