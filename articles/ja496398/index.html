頻度分析：NeoQUEST-2020のオンラインステージの参加者が好むタスクの1つを分析します


本日、ライトアップの課題NeoQUEST-2020を発行します。これについて、参加者は「4タスクメガクール」と「オリンピアードの最高のタスク」と書いています。
見かけは単純ですが、このタスクを完了したのは8人だけです。そして今、あなたは彼のことをもっとよく知ることができます-これで-猫の下でようこそ！


したがって、参加者は音声録音をダウンロードするよう招待されます。オーディオ録音で意味のある単語も観察されないのと同じように、モールス符号はここでも匂いがないことを理解するには、1分間聞くだけで十分です。したがって、私達は拷問を聞いて終わらせ、より詳細に音を見始めます。たとえば、古き良き（そして無料の）Audacityを使用します。音声録音を開き、その構造に注意を払います。キーストロークの間に完全な沈黙があります（誰かが労働条件に恵まれていて、電話や同僚とのチャットが永遠にありません！）。つまり、セクションをフィルタリングしてすべてのキーストロークを抽出することは難しくありません。信号振幅がゼロです。



よし、やろう。これで2つあります。クリック数が100,500の「配列」と、これらの音の中に繰り返しがあることを期待します。そうしないと、「XキーがN回押され、YキーがM回押された」と言えるでしょう。したがって、すべての初期データが分析され、「録音が10分間続くとしたら、そのような一連のステートメントによって何がもたらされるか」という質問に答える必要があります。はい、そうです。今こそ、古典的な周波数分析を明らかにする時です。まあ、私たちはpython、numpy、scikits.audiolabを使用して理論をテストします（サウンドを処理するために、他のパッケージや、同じ成功を収めた言語を選択することもできます）。
環境をセットアップするには、次のコマンドを実行します（Ubuntu 19.10で確実に機能します）。

$ sudo apt-get install -y python python-dev python-pip python-setuptools libsndfile-dev python-numpy
$ pip install --upgrade pip
$ pip install scikits.audiolab


ここで、クリックの音を識別する最も原始的な方法を使用して、コードの数行で仮定のテストを実装します-対応するnumpy配列の値の合計によって：

get list of separated ‘clicks’
data, fs, enc = scikits.audiolab.wavread(filename)
data = data.ravel()
clicks = [list(g) for k, g in groupby(data, lambda x: x != 0) if k]

# calculate number of unique ‘clicks’ and their frequences
freqs = dict() 
cypher_test = list()
for click in clicks:
    value = np.sum(np.array(click), dtype = np.float32)
    cypher_test.append(value)
    if value not in freqs:
        freqs[value] = 1
    else:
        freqs[value] += 1
