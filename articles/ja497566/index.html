ハフマン圧縮アルゴリズム

コースの開始を見越して、「開発者のためのアルゴリズム」では、別の有用な資料の翻訳が用意されています。



ハフマンコーディングは、ファイル圧縮の基本的な考え方を定式化するデータ圧縮アルゴリズムです。この記事では、固定長と可変長のコーディング、一意にデコードされたコード、プレフィックスルール、およびハフマンツリーの構築について説明します。

各文字は0と1のシーケンスとして格納され、8ビットを使用することがわかっています。各文字は同じ固定ビット数を使用して格納するため、これは固定長エンコーディングと呼ばれます。

テキストが与えられたとしましょう。 1つの文字を格納するために必要なスペースの量をどのように減らすことができますか？

基本的な考え方は、可変長コーディングです。テキスト内の一部の文字が他の文字よりも一般的であるという事実（ここを参照）を使用して、より少ないビットで同じ文字シーケンスを表すアルゴリズムを開発できます。可変長をコーディングする場合、このテキストでの出現頻度に応じて、可変数のビットを文字に割り当てます。最終的に、一部の文字は1ビットのみを使用し、他の文字は2ビット、3以上を使用する場合があります。可変長エンコーディングの問題は、シーケンスの後続のデコードのみです。

ビットのシーケンスを知って、それを一意にデコードするにはどうすればよいですか？

「aabacdab」 という文字列を考えてみます。 8文字で、固定長をエンコードする場合、64ビットを格納する必要があります。「a」、「b」、「c」、「d」の文字の頻度は、それぞれ4、2、1、1です。のは、想像してみましょうと「aabacdab」という事実使って、より少ないビットを「」よりも一般的である「B」と「b」をより一般的である「C」と「D」は。我々がコードで開始する「」 0に等しい1ビットを使用して、「B」は、我々は2ビットコード11を割り当て、3ビット100と011我々エンコード使用して「C」をと"D"。

その結果、成功します。

a 0
b 十一
c 100
d 011


したがって、上記のコードを使用して、文字列「aabacdab」を00110100011011（0 | 0 | 11 | 0 | 100 | 011 | 0 | 11）としてエンコードします。ただし、主な問題はデコードです。行00110100011011をデコードしようとすると、次のように表すことができるため、あいまいな結果が得られます。

0|011|0|100|011|0|11    adacdab
0|0|11|0|100|0|11|011   aabacabd
0|011|0|100|0|11|0|11   adacabab

...
など

このあいまいさを回避するには、エンコーディングがプレフィックスルールの概念を満たしていることを確認する必要があります。これは、コードを1つの一意の方法でデコードできることを意味します。プレフィックスルールは、コードが別のプレフィックスではないことを保証します。コードとは、特定の文字を表すために使用されるビットを意味します。上記の例では、0はプレフィックス011であり、プレフィックスルールに違反しています。したがって、コードがプレフィックスルールを満たしている場合、一意にデコードできます（逆も同様）。

上記の例を見てみましょう。今回は「a」、「b」、「c」、「d」の文字を割り当てます プレフィックスルールを満たすコード。

a 0
b 10
c 110
d 111


このエンコードを使用すると、文字列「aabacdab」は00100100011010（0 | 0 | 10 | 0 | 100 | 011 | 0 | 10）としてエンコードされます。しかし00100100011010を独自にデコードして、元の行「aabacdab」に戻すことができます。

ハフマンコーディング


可変長コーディングとプレフィックスルールがわかったところで、ハフマンコーディングについて説明しましょう。

この方法は、二分木の作成に基づいています。その中で、ノードは有限でも内部でもかまいません。最初は、すべてのノードがリーフ（リーフ）と見なされ、シンボル自体とその重み（つまり、発生頻度）を表します。内部ノードには、キャラクターのウェイトが含まれ、2つの子孫ノードを参照します。一般的な合意により、ビット「0」は左側のブランチのフォローを表し、「1」は右側のブランチを表します。完全なツリーには、N個のリーフとN-1個の内部ノードがあります。ハフマンツリーを構築するときは、未使用の文字を破棄して最適な長さのコードを取得することをお勧めします。

プライオリティキューを使用してハフマンツリーを構築します。ハフマンツリーでは、頻度が最も低いノードに最も高いプライオリティが与えられます。構築手順は次のとおりです。

  1. 各文字のリーフノードを作成し、それらを優先度キューに追加します。
  2. 複数のシートを並べて、次の操作を行います。
    • 優先順位が最も高い（頻度が最も低い）2つのノードをキューから削除します。
    • これらの2つのノードが継承される新しい内部ノードを作成します。発生頻度は、これら2つのノードの頻度の合計に等しくなります。
    • 新しいノードを優先度キューに追加します。
  3. 残りのノードだけがルートになり、ツリーの構築が完了します。

「a」、「b」、「c」、「d」、および「e 」という 文字のみで構成されるテキストがあり、それらの出現頻度がそれぞれ15、7、6、6、および5であるとします。以下は、アルゴリズムのステップを反映する図です。











ルートから任意のエンドノードへのパスには、そのエンドノードに関連付けられた文字に対応する最適なプレフィックスコード（ハフマンコードとも呼ばれる）が格納されます。


ハフマンツリー

以下に、C ++およびJavaでのハフマン圧縮アルゴリズムの実装を示します。

#include <iostream>
#include <string>
#include <queue>
#include <unordered_map>
using namespace std;

// A Tree node
struct Node
{
	char ch;
	int freq;
	Node *left, *right;
};

// Function to allocate a new tree node
Node* getNode(char ch, int freq, Node* left, Node* right)
{
	Node* node = new Node();

	node->ch = ch;
	node->freq = freq;
	node->left = left;
	node->right = right;

	return node;
}

// Comparison object to be used to order the heap
struct comp
{
	bool operator()(Node* l, Node* r)
	{
		// highest priority item has lowest frequency
		return l->freq > r->freq;
	}
};

// traverse the Huffman Tree and store Huffman Codes
// in a map.
void encode(Node* root, string str,
			unordered_map<char, string> &huffmanCode)
{
	if (root == nullptr)
		return;

	// found a leaf node
	if (!root->left && !root->right) {
		huffmanCode[root->ch] = str;
	}

	encode(root->left, str + "0", huffmanCode);
	encode(root->right, str + "1", huffmanCode);
}

// traverse the Huffman Tree and decode the encoded string
void decode(Node* root, int &index, string str)
{
	if (root == nullptr) {
		return;
	}

	// found a leaf node
	if (!root->left && !root->right)
	{
		cout << root->ch;
		return;
	}

	index++;

	if (str[index] =='0')
		decode(root->left, index, str);
	else
		decode(root->right, index, str);
}

// Builds Huffman Tree and decode given input text
void buildHuffmanTree(string text)
{
	// count frequency of appearance of each character
	// and store it in a map
	unordered_map<char, int> freq;
	for (char ch: text) {
		freq[ch]++;
	}

	// Create a priority queue to store live nodes of
	// Huffman tree;
	priority_queue<Node*, vector<Node*>, comp> pq;

	// Create a leaf node for each character and add it
	// to the priority queue.
	for (auto pair: freq) {
		pq.push(getNode(pair.first, pair.second, nullptr, nullptr));
	}

	// do till there is more than one node in the queue
	while (pq.size() != 1)
	{
		// Remove the two nodes of highest priority
		// (lowest frequency) from the queue
		Node *left = pq.top(); pq.pop();
		Node *right = pq.top();	pq.pop();

		// Create a new internal node with these two nodes
		// as children and with frequency equal to the sum
		// of the two nodes' frequencies. Add the new node
		// to the priority queue.
		int sum = left->freq + right->freq;
		pq.push(getNode('\0', sum, left, right));
	}

	// root stores pointer to root of Huffman Tree
	Node* root = pq.top();

	// traverse the Huffman Tree and store Huffman Codes
	// in a map. Also prints them
	unordered_map<char, string> huffmanCode;
	encode(root, "", huffmanCode);

	cout << "Huffman Codes are :\n" << '\n';
	for (auto pair: huffmanCode) {
		cout << pair.first << " " << pair.second << '\n';
	}

	cout << "\nOriginal string was :\n" << text << '\n';

	// print encoded string
	string str = "";
	for (char ch: text) {
		str += huffmanCode[ch];
	}

	cout << "\nEncoded string is :\n" << str << '\n';

	// traverse the Huffman Tree again and this time
	// decode the encoded string
	int index = -1;
	cout << "\nDecoded string is: \n";
	while (index < (int)str.size() - 2) {
		decode(root, index, str);
	}
}

// Huffman coding algorithm
int main()
{
	string text = "Huffman coding is a data compression algorithm.";

	buildHuffmanTree(text);

	return 0;
}

import java.util.HashMap;
import java.util.Map;
import java.util.PriorityQueue;

// A Tree node
class Node
{
	char ch;
	int freq;
	Node left = null, right = null;

	Node(char ch, int freq)
	{
		this.ch = ch;
		this.freq = freq;
	}

	public Node(char ch, int freq, Node left, Node right) {
		this.ch = ch;
		this.freq = freq;
		this.left = left;
		this.right = right;
	}
};

class Huffman
{
	// traverse the Huffman Tree and store Huffman Codes
	// in a map.
	public static void encode(Node root, String str,
							  Map<Character, String> huffmanCode)
	{
		if (root == null)
			return;

		// found a leaf node
		if (root.left == null && root.right == null) {
			huffmanCode.put(root.ch, str);
		}


		encode(root.left, str + "0", huffmanCode);
		encode(root.right, str + "1", huffmanCode);
	}

	// traverse the Huffman Tree and decode the encoded string
	public static int decode(Node root, int index, StringBuilder sb)
	{
		if (root == null)
			return index;

		// found a leaf node
		if (root.left == null && root.right == null)
		{
			System.out.print(root.ch);
			return index;
		}

		index++;

		if (sb.charAt(index) == '0')
			index = decode(root.left, index, sb);
		else
			index = decode(root.right, index, sb);

		return index;
	}

	// Builds Huffman Tree and huffmanCode and decode given input text
	public static void buildHuffmanTree(String text)
	{
		// count frequency of appearance of each character
		// and store it in a map
		Map<Character, Integer> freq = new HashMap<>();
		for (int i = 0 ; i < text.length(); i++) {
			if (!freq.containsKey(text.charAt(i))) {
				freq.put(text.charAt(i), 0);
			}
			freq.put(text.charAt(i), freq.get(text.charAt(i)) + 1);
		}

		// Create a priority queue to store live nodes of Huffman tree
		// Notice that highest priority item has lowest frequency
		PriorityQueue<Node> pq = new PriorityQueue<>(
										(l, r) -> l.freq - r.freq);

		// Create a leaf node for each character and add it
		// to the priority queue.
		for (Map.Entry<Character, Integer> entry : freq.entrySet()) {
			pq.add(new Node(entry.getKey(), entry.getValue()));
		}

		// do till there is more than one node in the queue
		while (pq.size() != 1)
		{
			// Remove the two nodes of highest priority
			// (lowest frequency) from the queue
			Node left = pq.poll();
			Node right = pq.poll();

			// Create a new internal node with these two nodes as children 
			// and with frequency equal to the sum of the two nodes
			// frequencies. Add the new node to the priority queue.
			int sum = left.freq + right.freq;
			pq.add(new Node('\0', sum, left, right));
		}

		// root stores pointer to root of Huffman Tree
		Node root = pq.peek();

		// traverse the Huffman tree and store the Huffman codes in a map
		Map<Character, String> huffmanCode = new HashMap<>();
		encode(root, "", huffmanCode);

		// print the Huffman codes
		System.out.println("Huffman Codes are :\n");
		for (Map.Entry<Character, String> entry : huffmanCode.entrySet()) {
			System.out.println(entry.getKey() + " " + entry.getValue());
		}

		System.out.println("\nOriginal string was :\n" + text);

		// print encoded string
		StringBuilder sb = new StringBuilder();
		for (int i = 0 ; i < text.length(); i++) {
			sb.append(huffmanCode.get(text.charAt(i)));
		}

		System.out.println("\nEncoded string is :\n" + sb);

		// traverse the Huffman Tree again and this time
		// decode the encoded string
		int index = -1;
		System.out.println("\nDecoded string is: \n");
		while (index < sb.length() - 2) {
			index = decode(root, index, sb);
		}
	}

	public static void main(String[] args)
	{
		String text = "Huffman coding is a data compression algorithm.";

		buildHuffmanTree(text);
	}
}

注：入力文字列が使用するメモリは47 * 8 = 376ビットであり、エンコードされた文字列は194ビットのみを使用します。データは約48％圧縮されます。上記のC ++プログラムでは、文字列クラスを使用してエンコードされた文字列を格納し、プログラムを読み取り可能にします。

プライオリティキューの効果的なデータ構造にはO（ログ（N））時間を挿入する必要があり、Nリーフの完全なバイナリツリーには2N-1個のノードがあり、ハフマンツリーは完全なバイナリツリーなので、アルゴリズムはO（Nlog（N ））時間、ここでNは文字数です。
