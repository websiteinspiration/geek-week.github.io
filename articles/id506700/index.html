Kami mengonfigurasi Tindakan GitHub untuk Android dengan penerapan berikutnya di PlayMarket

Halo, habrozhiteli! Suatu hari saya mulai belajar Tindakan GitHub untuk Android. Sebelumnya, saya memiliki upaya yang berhasil untuk mengkonfigurasi fungsi ini untuk proyek di Flutter, tetapi tanpa penyebaran, yang penuh dengan informasi dan panduan tentang sumber daya berbahasa Inggris dan sumber daya berbahasa Rusia, tetapi dengan android asli, semuanya tidak begitu membosankan. Dan saya memutuskan untuk menuliskan masalah utama dan solusinya.

Tahap satu: pengaturan untuk penandatanganan otomatis apk yang sudah selesai


Masalah pertama yang saya temui adalah membuat pengaturan universal yang memungkinkan Anda mengumpulkan versi rilis baik lokal maupun global.

Saya menggunakan opsi dengan menggunakan file keystore.propertiesyang memungkinkan kami untuk menambahkan kunci pengembang ke folder proyek tanpa menyinari kata sandi darinya, ini dilakukan seperti ini:

apply plugin: ...

def keystorePropertiesFile = rootProject.file("keystore.properties")
def keystoreProperties = new Properties()
keystoreProperties.load(new FileInputStream(keystorePropertiesFile))

android {
  ...

  signingConfigs {
    release {
        storeFile file("../MyKey.jks")
        storePassword keystoreProperties['RELEASE_STORE_PASSWORD']
        keyAlias keystoreProperties['RELEASE_KEY_ALIAS']
        keyPassword keystoreProperties['RELEASE_KEY_PASSWORD']
    }
    debug {
        storeFile file('../debug.keystore')
    }
  }

  buildTypes {
    release {
        proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        minifyEnabled false
        signingConfig signingConfigs.release
        buildConfigField "String", "PIN_ALIAS", keystoreProperties['PIN_ALIAS']
        buildConfigField "String", "DB_PASS_ALIAS", keystoreProperties['DB_PASS_ALIAS']
    }
    debug {
        minifyEnabled false
        signingConfig signingConfigs.debug
        buildConfigField "String", "PIN_ALIAS", keystoreProperties['PIN_ALIAS']
        buildConfigField "String", "DB_PASS_ALIAS", keystoreProperties['DB_PASS_ALIAS']
    }
  }
}

dependencies {
  ...
}

Dan kemudian muncul masalah, bagaimana memastikan bahwa kita dapat mengambil kunci dari ${{ secrets.MY_KEY }}dan pada saat yang sama memahami lulusannya, jika kita memiliki properti kunci, maka kita ambil dari itu, jika tidak, maka kita ambil dari rahasia? Solusinya ditemukan pada salah satu panduan untuk flutter , di mana mereka menggunakan lingkungan untuk ini (Ngomong-ngomong, ini adalah pendekatan yang keren, agar tidak bersinar dengan kunci pengembang kami), tetapi ini tidak menyelesaikan masalah. Setelah mencoba beberapa opsi dengan memperkenalkan file tambahan, dll., Saya memilih yang paling sederhana: kami juga memperkenalkan beberapa variabel (tergantung pada jumlah yang kami butuhkan), dan memeriksa keberadaan file properti keystore.properties:

def release_store_password
def release_key_password
def release_key_alias
def pin_alias
def db_pass_alias

def keystoreProperties = new Properties()
if (rootProject.file("keystore.properties").exists()) {
    keystoreProperties.load(new FileInputStream(rootProject.file("keystore.properties")))
    release_store_password = keystoreProperties['RELEASE_STORE_PASSWORD']
    release_key_password = keystoreProperties['RELEASE_KEY_PASSWORD']
    release_key_alias = keystoreProperties['RELEASE_KEY_ALIAS']
    pin_alias = keystoreProperties['PIN_ALIAS']
    db_pass_alias = keystoreProperties['DB_PASS_ALIAS']
} else {
    release_store_password = System.env.RELEASE_STORE_PASSWORD
    release_key_password = System.env.RELEASE_KEY_PASSWORD
    release_key_alias = System.env.RELEASE_KEY_ALIAS
    pin_alias = System.env.PIN_ALIAS
    db_pass_alias = System.env.DB_PASS_ALIAS
}

android {
   signingConfigs {
        release {
            storeFile file("../my_key.jks")
            storePassword = release_store_password
            keyAlias = release_key_alias
            keyPassword = release_key_password
        }
    buildType{
       release {
          buildConfigField "String", "PIN_ALIAS", "\"$pin_alias\"" //     
          buildConfigField "String", "DB_PASS_ALIAS", "\"$db_pass_alias\"" //  .
      }
    }
}

Jadi, sekarang kolektor kami dapat mengumpulkan dan segera menandatangani apk kami.

Tahap dua: versi perakitan.


Tidak ada yang melampaui alam, saya ingin mendapatkan semacam opsi yang cukup universal, dengan kompleksitas minimal. Googling, melihat dari dekat berapa banyak pengembang - ada banyak pilihan dan semua orang sesat yang dia bisa. Saya tidak memerlukan pendekatan yang sangat rumit dan saya sudah ingin menggunakannya BUILD_NUMBER, tetapi kemudian saya menemukan parameter y untuk tindakan GitHub: $ {{github.run_number}} .

$ {{github.run_number}}

Karena itu, setelah menimbang semua pro dan kontra, kami memiliki solusi berikut:

def versionPropsFile = rootProject.file('version.properties')
Properties versionProps = new Properties()
versionProps.load(new FileInputStream(versionPropsFile))
def verCode = versionProps['VERSION_CODE'].toInteger()

android {
  defaultConfig {
    versionCode verCode
    versionName "1.1.$verCode"
  }
}

//version.properties 
VERSION_CODE=1

Dalam alur kerja, lakukan ini:

- name: Output version code
        run: echo VERSION_CODE=${{ github.run_number }} > ./version.properties

Tahap Tiga: Menyebarkan


Saat ini, saya menemukan dua solusi siap pakai: Penerbit Gradle Play dan Unggah rilis Android ke Play Store

Opsi pertama dicabut karena suatu alasan: penggunaan howitzer saat menembak burung pipit. Karena itu, saya memilih yang kedua. Tidak ada yang rumit di dalamnya, tetapi ada instruksi rinci: Di sini .


- name: Upload to PlayMarket
        uses: r0adkll/upload-google-play@v1
        with:
          serviceAccountJsonPlainText: ${{ secrets.SERVICE_ACCOUNT_JSON }}
          packageName: com.guralnya.notification_tracker
          releaseFile: app/build/outputs/apk/release/notification_tracker.release.apk
          track: beta
          userFraction: 0.33
          whatsNewDirectory: distribution/whatsnew

Tetapi beberapa poin masih terpikir oleh saya:

  • serviceAccountJson dan serviceAccountJsonPlainText - dengan yang pertama saya masih belum tahu bentuk apa yang perlu dirahasiakan, yang kedua hanya mengambil isi file dan menyimpannya di rahasia kami.
  • releaseFile ‚Äî , , : notification_tracker.release.*.apk, . , (actions/upload-artifact@v2), .
  • whatsNewDirectory ‚Äî . - (en-IN), (ru-RU), , ‚Äî , (uk), , .

Ada situasi di mana karena alasan tertentu Anda mungkin tidak menggunakan, ini adalah saat Anda belum menerbitkan aplikasi (Anda tidak harus bertemu, tetapi dijelaskan dalam satu artikel). Jadi mulailah dengan pertama-tama menerbitkan aplikasi secara manual jika Anda belum melakukannya.

Alur kerja akhir - akan dioptimalkan dan ditingkatkan bersama dengan file hujan es


Android CI.yaml:


name: Android_CI

on:
  push:
    branches:
      - beta_release

jobs:
  build:
    runs-on: ubuntu-latest
    name: Build release-apk and deploy to PlayMarket
    steps:
      - uses: actions/checkout@v2
      - name: set up JDK 1.8
        uses: actions/setup-java@v1
        with:
          java-version: 1.8
      # Without NDK not compile and not normal error message. NDK is required
      - name: Install NDK
        run: echo "y" | sudo ${ANDROID_HOME}/tools/bin/sdkmanager --install "ndk;20.0.5594570" --sdk_root=${ANDROID_SDK_ROOT}
      # Some times is have problems with permissions for ./gradle file. Then uncommit it code
      #    - name: Make gradlew executable
      #      run: chmod +x ./gradlew
      - name: Output version code
        run: echo VERSION_CODE=${{ github.run_number }} > ./version.properties
      - name: Build with Gradle
        run: ./gradlew assemble
        env:
          RELEASE_STORE_PASSWORD: ${{ secrets.RELEASE_STORE_PASSWORD }}
          RELEASE_KEY_PASSWORD: ${{ secrets.RELEASE_KEY_PASSWORD }}
          RELEASE_KEY_ALIAS: ${{ secrets.RELEASE_KEY_ALIAS }}
          PIN_ALIAS: ${{ secrets.PIN_ALIAS }}
          DB_PASS_ALIAS: ${{ secrets.DB_PASS_ALIAS }}
      - name: Upload APK
        uses: actions/upload-artifact@v2
        with:
          name: notification_tracker
          path: app/build/outputs/apk/release/notification_tracker.release.apk
      - name: Upload to PlayMarket
        uses: r0adkll/upload-google-play@v1
        with:
          serviceAccountJsonPlainText: ${{ secrets.SERVICE_ACCOUNT_JSON }}
          packageName: com.guralnya.notification_tracker
          releaseFile: app/build/outputs/apk/release/notification_tracker.release.apk
          track: beta
          userFraction: 0.33
          whatsNewDirectory: distribution/whatsnew

Poin penting


- Kebutuhan akan NDK. Tanpa NDK yang terinstal, Anda tidak akan dapat membangun proyek, setidaknya satu rilis. Orang bisa lama menebak apa masalahnya dan mencari solusi, karena tidak ada pesan kesalahan yang normal. Kadang-kadang Anda dapat menangkap di sini: Task :app:stripDebugDebugSymbols FAILED. Setelah googling dan bereksperimen, ternyata tidak ada NDK. Kami melakukan ini:

- name: Install NDK
        run: echo "y" | sudo ${ANDROID_HOME}/tools/bin/sdkmanager --install "ndk;20.0.5594570" --sdk_root=${ANDROID_SDK_ROOT}

Atau lebih baik tambahkan ke Gradle (terima kasih kepada pengguna Anegin):

}
