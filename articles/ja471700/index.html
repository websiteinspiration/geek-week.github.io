将来の基盤を備えた技術スタックをどのように選択したか

単純なものから始めますが、さらなる成長を犠牲にすることはありません。



ソフトウェアエンジニアとしてのキャリアをスタートさせた後、テクノロジーの起業家になり、1億ドルを超えるシリコンバレーのベンチャーキャピタルを魅了する2つのSaaS企業を設立しました。

問題は、これらの企業が複雑でマルチテナントのソフトウェアシステムを作成するために莫大な投資を必要とし、また、販売と実装の専門家が常に仕事を促進およびサポートする必要があったことです。そのため、2017年に8baseを設立しました。

8baseの中心にあるのは、ソフトウェア開発を加速、改善、および安価にするプラットフォームとツールに対する大きな需要があるという考えです。 8baseは既製の理解可能なソフトウェア環境を提供し、私たちのアーキテクチャは非常に重要です。どうして？技術スタックは、私たちだけでなく、お客様のためにも設計されています。

私たちのチームはSaaS企業での幅広い経験があり、アーキテクチャソリューションの選択において私たちを導く明確な信念があります。 ：ここではそれらのいくつかある

- まず、API（API-最初） -コンピューティングインフラストラクチャ、データおよびビジネス・ロジックは、サーバー側でなければならず、単一の動的APIを使用して表現しました。

- 弾力性-コンピューティングインフラストラクチャは、実行中のアプリケーション用に完全にカスタマイズ可能なサービスである必要があります。

- フロントエンド開発の柔軟性。最近のソフトウェアは科学よりも芸術であり、企業はテクノロジーを通じて自分自身を表現しています。開発者は、さまざまなフォームファクターと組み合わせて、任意のスタイルでユーザーインターフェイスを設計および作成できる必要があります。

これらのアイデアを8baseのコアに入れました。

8baseテクノロジースタック


まず、クラウドコンピューティングの市場リーダーであるため、コンピューティングインフラストラクチャとしてAmazon Web Services（AWS）を選択しました。我々は、多くのAWSサービスを使用します

- AWSラムダサーバレスコンピューティングのために。 8baseを使用して構築されたアプリケーションもLambdaを使用します。サーバーレスは、イベントに応答してコードを実行し、必要なコンピューティングリソースを自動的に管理する比較的新しいコンピューティングモデルです。 Lambdaは、エネルギー会社と同じ方法でコンピューティングのニーズに対応します。つまり、カスタマイズ可能で測定可能なサービスを提供します。これにより、DevOpsエンジニアの必要性が最小限に抑えられ、コストの削減につながります。-AWS Aurora MySQL

データベースおよびMongoDB Atlas。それぞれがサービスとしてのデータベースであり、高速で信頼性が高く、フォールトトレラントで、管理しやすく、スケーラブルです。 8baseは、各開発者ワークスペースに専用のAurora MySQLも提供します。

-オブジェクトを保存するためのサービスAWS S3（Simple Storage Service）。 8baseは、ワークスペースとクライアントのフロントエンドアプリケーションにS3へのアクセスを提供する問題を解決します。これにより、ドキュメント、画像、音声ファイル、ビデオファイルなどを簡単に保存できます。 8baseもFileStackを使用します開発者がクラウドストレージ、ソーシャルアカウント、その他のアカウントを簡単に接続できるようにします。 S3を選択した理由は、S3が安価で高速、信頼性が高く、ストレージボリュームが事実上無制限だからです。

- AWS API Gatewayがされ 8baseエンジンによって使用され、RESTとWebSocketをAPIを作成、公開して確保するためのサービスとして8base上に構築されたクライアントアプリケーションで使用可能なデフォルトです。

8baseは非常に強力なGraphQL APIエンジンを作成しましたプラットフォームの一部。 GraphQLはFacebookによって開発された標準であり、2015年にオープンソースになりました。これは、フロントエンドとバックエンド間の動的接続を提供する単一のエンドポイントです。 GraphQLを使用すると、フロントエンドの開発者は非常に高速に作業でき、バックエンドの開発者への依存を減らします。このAPIテクノロジーは、8baseチームおよび8baseを使用するチームの開発を迅速化するのに役立ちます。あらゆるフロントエンドフレームワーク

をサポートしていますが、8base自体はReactを使用して作成されています。Reactは、Facebookと開発者コミュニティによって維持されているオープンソースのJavaScriptライブラリです。変化の速いデータに最適です。今日、Reactは非常に人気があり、iOSおよびAndroidでネイティブアプリケーションを作成するためのフレームワークであるReact Nativeも提供しています。

最後に、認証にAuth0プラットフォームを選択しました。開発者は、Auth0をクライアントアプリケーションに非常に簡単に統合できると同時に、認証に他のプロバイダーを使用する機会があります。Auth0は、多要素認証や他の多くの強力な機能に加えて、承認のための30を超えるソーシャルサービスおよび企業サービスとの統合を提供します。

データモデルを意識せずにスタートアップ製品を作成する開発者には注意してください。

今日存在する一部のバックエンドソリューションでは、フロントエンド開発者、特にモバイルアプリケーション開発者は、データモデルや構造に関係なく、単純にデータをアップロードできます。一部のアプリケーションでは、これは正常です。しかし、ほとんどのアプリケーションでは-悲しいかな、違います。データへの非構造化アプローチを選択することを忘れないでください。これは正しい選択であり、最も簡単ではありません。それ以外の場合、データは調理されるとスパゲッティのようになります...

あなたが非技術的な創設者であり、開発を助けるために代理店を巻き込む場合、彼らに次の8つの質問をしてください。

代理店は専門的なサービスを提供する傾向があり、できるだけ長く提供することが彼らの利益になります。彼らの目標は常にあなたの目標と一致するとは限りません。これを確認するために、次の質問をすることができます

。1.ソフトウェア開発へのアプローチは何ですか？最初に、設計（設計優先）または最初の設計（設計優先）？
2.最初に得られた結果はどのように見えますか？それは完成した設計、プロトタイプ、またはその両方でしょうか？
3.どのスタートアップ企業を助けましたか、彼らは彼らの市場ニッチを見つけて、財政的に安定しましたか？
4.変更する必要がないように、アプリケーションアーキテクチャをどのように設計しますか？
5.他の開発者はプロジェクトで作業できますか？
6.必要に応じてソフトウェアを拡張できますか？
7.新しいマーケティングデータに基づいてデザインを変更する場合はどうなりますか？
8.製品の発売後、製品をサポートするために何が必要ですか？

技術製品の作成は、終わらない起業家にとっての旅です。何を、誰が、どのようにして、成功と失敗を分けるかについての最初の決定。これらの決定には、製品が何を行うべきかを誰が決定するか、誰が設計および開発するか、およびどの基本技術が使用されるかについての質問に対する回答も含まれます。

これらの質問に対する正しい答えは起業家に勝つチャンスを与えます、さもなければ、決定は時間の損失、追加のコスト、そしておそらく失敗につながります。
