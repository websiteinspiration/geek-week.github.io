2020年に自分より上に成長したいWeb開発者のための32のヒント

2019年が過ぎました。私たちにとって、Web開発者にとって、今年はテストと機会の年でした。周りには学ぶことができるすべてのものがあり、習得を達成しようとすることができる何かがあります。たとえば、React、Vue、Angular、SvelteなどのJavaScriptサイト開発ツールです。しかし、これはもちろん、すべてとはかけ離れています。静的サイトジェネレーター、JAMstack、サーバーレステクノロジー、Gitなどもあります。

Web開発エコシステムの誰もが彼にとって適切なものを見つけることができますが、ここでの問題は、刻々と変化するテクノロジーに追いつくことです。しかし、テクノロジーはテクノロジーに限定されません。すべての開発者が回答を見つける必要のある質問は他にもたくさんあります。どこで誰と仕事をしたいですか？フルスタックの開発者になるべきですか？永久的な仕事が必要ですか、それともフリーランスに近いですか？新しい知識とスキルを開発して習得するにはどうすればよいですか？



この記事の著者は、あなたの注意とその翻訳を今日紹介していますが、彼はすべての知識とすべての経験を、Web開発者に提供される32のヒントに組み込んだと述べています。彼は、これらのヒントが旅を始めたばかりの人や、すでにWeb開発の何かを理解している人に役立つことを期待しています。ここで誰もがあらゆる場面でアドバイスを見つけること、そしてこれらのすべてのヒントが誰にとっても完璧であると言うことは不可能ですが、著者はこの資料の読者が彼の中で刺激を与えたりガイドとして役立つものを見つけられると信じたいと思っています2020年に行動する。

1. Gitの概念を理解しようとする


基本的なGitコマンドを理解することは良いことですが、このシステムの根底にある概念を理解してください。

2.クリーンなコードを書くアイデアを習得します。


クリーンなコードの理念を理解することは、他の人にとって読みやすく、エラーの少ない、より良いコードを作成するのに役立ちます。

3.ポートフォリオサイトを作成する


あなたはあなたの仕事の結果を誇りに思うべきです、それであなたはそれらについて全世界に話し、あなたの将来の顧客と雇用者にあなたを見つける機会を与えるべきです。

4.技術記事を書く


技術的な記事を書くことは多くの利益をもたらします。したがって、人が習得していることを他の人に説明しようとすると、通常、何か新しいことを学ぶことがより簡単になります。さらに、良い記事は、その記事の著者と同じことに興味があるすべての人への寛大な贈り物です。これはコミュニティへの賛辞であり、その仕事は新しいことを学び、それについて話す人々によって使用されます。

5.フリーランスで自分を試す


2020年を転職の年にします。「無料のアーティスト」として自分自身を試してください。契約に求人広告を掲載しているサイトを見て、離れた従業員を探している人に手紙を書いたり、さまざまなイベントに参加したり、興味を持ったりしてください。おそらく現在の雇用主があなたをフリーランサーとして雇うでしょう。

6. JavaScriptの基本を学ぶ


JavaScriptは、誇張することなく「Webエンジン」です。そして、この言語は、今日人気のあるReactやVueなどのフレームワークがなくなっても、おそらくどこにも行きません。JavaScriptの基本を学ぶことに投資してください。

7.すばらしいJavaScriptフレームワークを調べてください。


最新のWebアプリケーションは、多くの場合、Vue、React、Angularなどのフレームワークまたはライブラリに基づいています。Facebook、Google、Airbnbなどの企業がこれらのツールをサポートし、使用しています。その結果、Web開発者はこれらのツールの少なくとも1つを所有する必要があると言えます。しかし結局のところ、それらはすべてJavaScriptに帰着します。

8.シニアデベロッパーポジションに移動してみてください


上級開発者になることの意味を理解する。技術的な知識と「上級」スキルに加えて、人々とコミュニケーションを取り、彼らの感情的な状態を理解する能力などの個人的な資質は非常に重要です。

9.初心者の開発者をサポートする


経験豊富なプログラマ「シニア」の仕事の最も重要な結果は、彼による新しい「シニア」の育成です。ですから、他の人が学ぶのを手助けし、それらを見下ろさないようにしてください。

10.トレーニング資料を作成する


新しいプログラミング言語を学んだ、いくつかの新しいテクノロジーを見つけた、またはツールの新しい興味深い機能を発見した場合は、他の人と知識を共有してください。

11. VS Codeエディターとその拡張機能を試す


主なツールとしてVS Codeエディターをまだ使用していない場合は、ぜひお試しください。これにより、生産性が向上します。VS Codeには、フォーマットツールやコード補完ツールなど、非常に多くの拡張機能が作成されています。

12.不健康な職場を離れる


職場で常に否定性に直面している場合、リーダーシップで共通の言語を見つけることができない場合は、新しい仕事を探してください。時間の経過とともにすべてが落ち着くと予想する場合-人々は通常変化しないことを忘れないでください。才能を無駄にしてはいけないことを忘れないでください。あなたとあなたのスキルに感謝する人を見つけましょう。

13.オープンソースに貢献する


おそらく、オープンソースプロジェクトへの参加は、開発者が考えていたことです。このようなプロジェクトに参加することで、志を同じくする人々のコミュニティを支援し、実際の環境でプログラミングを練習し、履歴書に適切なエントリを作成できます。多くのオープンソースプロジェクトは、これらのプロジェクトの開発を支援できる新しい参加者を歓迎します。

14.すでに知っているものとは異なるプログラミング言語を学ぶ


視野を広げて快適ゾーンを離れるために、完全に新しいプログラミング言語を学ぶことができます。たとえば、Javaで記述する場合は、JavaScript、Rust、またはGolangを試してください。今まで見たことのないものを見つけましょう。

15.サーバーレス機能を拡張する


サーバーレス関数は、単一のタスクを解決することを目的とした小さなコード片であり、管理対象インフラストラクチャに展開されます。たとえば、それらのデプロイには、AWS Lambdaシステムを使用できます。サーバーレステクノロジーは多くのプロジェクトで使用されているため、これらのテクノロジーについて理解する必要があります。

16. GraphQLの概念を探る


RESTful APIは、過去数年間のWeb開発の事実上の標準です。しかし、Facebook GraphQLテクノロジーの出現と普及の後、どのWeb開発者もこのテクノロジーの基礎となる概念を理解するのに害を及ぼすことはないと言えます。

17. JAMstackアプリケーションを作成する


JAMstack手法は、JavaScript、API、およびマークアップを使用するアプリケーション開発アプローチです。JAMstackプロジェクトは強力で軽量で、特殊なバックエンドから独立しています。この手法を使用して構築されたプロジェクトは、組織のサイトの役割と開発者のポートフォリオの役割の両方を果たすことができます。

18. OWASPプロジェクト資料を見てください。


Wikipediaに よると、Open Web Application Security Project（OWASP）はオープンソースのWebアプリケーションセキュリティプロジェクトです。このプロジェクトの一環として公開された資料を研究することで、より安全なWebアプリケーションを作成できます。

19.より多くの時間ペアプログラミングを使う


ペアプログラミングは、開発者間で知識を共有するための最良の方法の1つです。熟練したプログラマでない場合は、より高度な同僚から学ぶことができます。または、十分な経験があれば、初心者を助けることができます。とにかく、これは非常に便利です。特に、当面の作業に集中できるようになるためです。

20.コードレビューに参加する


多くのプロジェクトおよび多くの組織では、コードレビューは、コードがリポジトリに含まれる前に実行される必須の手順です。開発者は、できるだけ頻繁にコードレビューに参加するように努める必要があります。あなたが初心者であれば、これは他の開発者のコ​​ードを分析し、このコードについて彼らに質問することで成長するのに役立ちます。経験豊富なプログラマーであれば、コードをチェックしているプロジェクトの品質を向上させるのに役立ちます。

21.問題そのものではなく、問題解決に特に注意してください。


優れた開発者のサインの1つ（より正確には「シニア」）は、問題自体ではなく、問題の解決に特別な注意を払う能力です。常に問題を解決するためのさまざまな方法を見つけるようにしてください。

22.コードのデバッグ方法を学ぶ


私たちが書くコードは決して完璧なものにはなりません。そこには常にエラーと欠点があります。それらを見つけて修正するには、コードをデバッグする方法を学ぶ必要があります。

23.自分の価値を知る


私は常に、クライアントから受け取るお金、または雇用主が私に支払うお金の価値があることを思い出します。あなたが習得したすべての知識とスキル、プログラミングに費やされたすべての時間、すべてのかけがえのない個人的な資質は、金融問題について話し合うときの交渉に自信を与えるはずです。

24.過ちから学ぶ


誰でも間違いはある。最も成功した開発者でさえ、多くの間違いを犯します。しかし、最も重要なことは、間違いから学ぶことです。これは、専門的および個人的な成長への道です。

25.常にコードをテストする


単体テストや統合テストを書くことは、たいしたことではないように見えるかもしれません。プログラマーは、彼のコードは、いずれにせよ、美しく書かれていてエラーがないと思っているかもしれません。しかし、そう考えると、これ自体は間違いです。ほとんどすべての企業プロジェクトまたはオープンソースプロジェクトには、コードテストが含まれます。テストは、少なくともテストをよりよくマスターするために、個人プロジェクト用に作成する必要があります。

26.デザインパターンを学ぶ


デザインパターンは、アプリケーションアーキテクチャの構成要素です。すべてのプロジェクトでそれらを必要とするわけではありません。しかし、プログラマが少なくとも最も重要なパターンを知ることは害になりません。

27.遠慮なく助けを求めてください


誰かに質問して助けを求めることは何の問題もありません。確かに、質問する前に、自分で問題を解決することを常にお勧めします。ただし、一般的に言えば、足踏み状態にあると感じた場合は、問題のあることを知っている人にアドバイスを求めてください。

28.技術会議に参加する


会議に参加することで、新しいテクノロジーについて学び、自分と同じことに関心のある他の開発者とコミュニケーションを取ることができます。さらに、企業は多くの場合、会議への参加に対して従業員をある程度補償する準備ができており、従業員がこれらのイベントから貴重な知識を持ち込んで同僚と共有することを期待しています。

29.技術会議で話す


カンファレンスで講演して、そこで興味のあることについて話す機会がある場合は、この機会にぜひ参加してください。あなたが内向的である場合でも、それはあなたが自信を築くのに役立ちます。

30.知識とスキルを最新の状態に保つようにしてください。


これは言うより簡単ですが、知識とスキルを最新の状態に保つためには、常に学び、改善する必要があります。記事や本を読んだり、会議に参加したり、ビデオを見たり、ニュースレターを購読したり、トレーニングコースに参加したりできます。ここに、プロとしての健康を維持するためのチャンスがたくさんあります。

31.新しいアイデアを受け入れる


当然のことと考えるべきではない。単一のテクノロジー、単一のフレームワーク、単一のライブラリではありません。業界で成功するためには、プログラマーは絶えず適応する必要があります。そして、新しいアイデアへの開放性は、テクノロジーだけではありません。これは人間関係にも当てはまります。

32.決してあきらめない


誰もが時々困難に直面します-より頻繁に、より少ない頻度で。一般的に、人生は複雑なものです。しかし、あなたは一人ではなく、誰もが何かから始めたことがあることを覚えておいてください。できます！
