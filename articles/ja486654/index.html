Raspberry piのホームアシスタントでのモーションセンサーとアラームおよびビデオ登録接続

Raspberry piでのホームアシスタント（以下、HAと呼びます）を使用した小さな経験と、ビデオ登録機能、モーションセンサー、それに応じて「セキュリティ」操作が発生した場合にメールで画像を受信するホームセキュリティ機能の接続について話したいと思います。

もちろん、理由の1つは、このシナリオを実装して、結局私が「間違った」ことをしたことをフィードバックから理解したいという欲求です。

上記の機能を設定するには、「問題なし」、「短い」、「独立」の3つの方法があります。最初のケースでは、完成したイメージをサイトhttps://ViaMyBox.com/downloadpageからダウンロードしました。Raspberry piの場合、構成済みのフォームですべてがどのように見えるかを確認できます。 2番目のケースでは、ウェブサイトまたはgithubプロジェクトからzipをダウンロードし、ユーティリティ（sudo via-setup.sh）を介してHA Dockerイメージをインストールおよびインストールし、すべてがどのように構成されているかを確認できます。そして最後に、3つ目の方法は、「すべてを自分の方法で」セットアップすることです。プロジェクトのサイトやギザブ、またはこのチュートリアルから、何か役立つものを取得します。記事の最後にあるすべてへのリンク。

すでに機能しているHome Assistant（HA）があるとします。この記事では、yaml構成の形成の側面と、特定のアクション（この場合、「ハウスセキュリティ」の接続とモーションセンサーがトリガーされてから5秒以内に写真の撮影を開始するスクリプト）につながるルールと条件のシーケンスの説明のみを考慮します。その結果、ホームアシスタントは撮影した写真をメールで送信します。私はyamlやHome Assistantの構成の作成の専門家ではありませんが、実際の例に従って、共有したい構成を取得しました。

私たちが参照するすべてのファイルは、記事の最後に配置して、より便利なレビューを行います。この例では、bashまたはpythonでビデオを記録するためのスクリプトの作業はここでは扱いません。ホームアシスタントのみ。しかし、質問がある場合-ウェルカム！

結局のところ、私にとってはすべてがこのように見えます（記事で検討するため、赤で強調表示しています）





。Raspberrypiをプラットフォームとして使用しています。そして、設定ファイルへの私のパスは/ usr / share / hassio / homeassistant /です。構成ファイルへのパスは、私のパスとは異なる場合があります。当時、私はHome Assistant 0.101.3を持っていました。このディレクトリの構成ファイルであるconfiguration.yamlとAutomation.yamlに注目します。

これらのファイルを変更するたびに、変更後に構成にエラーがないか確認する必要があることを覚えておくことが重要です。これは、[ホームアシスタントダッシュボード]-> [構成]-> [サーバーコントロール]-> [構成の確認]タブで行います。そして、検証が成功した場合は、同じ場所で自動化の再読み込みとスクリプトの再読み込みを行います。また、疑問がある場合は、同じタブでサーバー管理を再起動すると、構成が正確に更新されます。

HC-SR501モーションセンサーはRaspberry pi GPIOバスに接続されています。 3本の接触線をGPIOに正しく接続する必要があります。そうすれば、センサーを実際に使用できます。センサーには3つの接点があります。電源gcc（+）、ゼロgnd（-）、制御接点（データ）。 GPIOの説明を読んだ後、次のようにラズベリーに接続しました。 hc-sr501をGPIOコネクタに接続しました：ピン＃2-5.5vvcc;ピン＃26（13番目の行）-データ接点およびピン＃6-センサーのグランド（-）。

写真のソースとしてmjpg-streamerと組み合わせて、usbカメラまたはcsiカメラを使用します。 Raspberry piにmjpg-streamerをインストールして構成する方法については、ここでは考慮しません。インターネットでのクイックインストールの例がたくさん。ただし、このプロジェクトのデータを使用していると言いたいこのモジュールのインストールについても詳しく説明されています。または、写真やビデオを撮るのに便利な方法を使用します。結局のところ、HAに固定するbashスクリプトは、最終的にはあなた次第です。

ホームアシスタントには、ホームセキュリティ機能をアクティブにするための組み込みツールがあります。説明は、www.home-assistant.io / integrations / manualを使用して、configuration.yamlで説明します。

    pending_time: 60
     delay time 40

    pending_time: 0

この設定では、家に帰るまでに60秒（この時間を過ぎるとアラームがオンになります）、家に戻るときに40秒（パスワード1234で）オフになります。何らかの理由で、あなたはいつもあなたが来るよりも長く離れます。）

次に、私たちのconfiguration.yamlでモーションセンサーの操作によって写真をオン/オフにするメカニズムについて説明します（スイッチの詳細-command_lineプラットフォームはここにあります）：

 - platform: command_line
   switches:
      start_stop_motion_rec_timelapse:
        friendly_name: 'Record motion timelapse video'
        command_on: 'curl http://localhost/start_mjpgstrm.php  && curl http://localhost//rec-motion-mjpg.php'
        command_off: 'curl http://localhost/stop_mjpgstrm.php && curl http://localhost/rec-motion-mjpg-stop.php'

ここで、コマンドenable command_onコマンドとcommand_off offコマンドを、カメラの記録を制御するスクリプトにリンクしました。この場合、command_onを実行すると、2つのスクリプトが順次実行されます。これにより、mjpg-streamerが起動し、モーションセンサーの記録がトリガーされます。 bashスクリプトへのアクセスは、Raspberry piの作業サイトのphpファイルを経由します。これを行うには、nginxとWebアクセスを構成し、ブラウザーからhttp：// <Raspberry piのIPアドレス> /start_mjpgstrm.phpの形式でアクセスすると、phpスクリプトが実行されます。このスクリプトは、この状況でmjpg-streamerを実行します。

この記事では、Webアクセス構成については考慮しません。確かにこれはセキュリティではありませんが、私はdockerにHAを使用しているため、分離されたdocker環境と、phpを介してサイトにアクセスすることで確認できるOSの「外の世界」に直面していました。おそらく、DockerやHAの達人による優れたソリューションがたくさんあります。書いて、知っておくと面白いでしょう！
