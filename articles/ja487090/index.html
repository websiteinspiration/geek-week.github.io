新しいレベルのWindows最適化

約2年前、私たちのチームは、ニューラルネットワークテクノロジーを使用してWindowsを最適化するプロジェクトについてHabrに話しました。

過去2年間で多くの変化がありましたが、最も重要なことは変わっていません。この記事では、この2年間に行われた発見と結論を共有したいと思います。



2020年のPCソフトウェアの独立生産の状況に興味がある場合は、catへようこそ。

しかし、メインの物語に進む前に、以下に述べるすべてが独立した会社の視点から記述されていることを明確にする必要があります。独立した会社は、開発の過程で内部リソースを犠牲にしてのみ前進します。

この2年間で何が行われたか




MSストアでのUWP出版物は、最も困難なものの1つになりましたが、プロジェクトで作業するときに最も重要な成果の1つにもなりました。この段階で、CIS市場を超えて、Windowsソフトウェアのグローバル市場を開拓できました。

新機能。

次のステップでは、CPUの優先順位とCPUコアを管理してWindowsを最適化できるようになった後、PCの他のすべての主要コンポーネントを管理して、ニューラルネットワークアルゴリズムを最大の効率で使用するようにアプリケーションを教える必要がありました。
カーネルとCPUの優先順位を管理することですべてが比較的単純な場合、RAMの優先順位とI / Oの優先順位を管理するために、論文を書く時間になるほどWindowsアーキテクチャに飛び込む必要がありました。たとえば、私たちが知っているすべてのソフトウェアから、WPSとProcess Lassoだけが「高い」I / O優先度を設定できます。

まあ、CPU、RAM、ROMの完全な制御がマスターされるとすぐに、それは周辺機器次第でした。したがって、次の開発ステップは電源管理でした。WPSは、静的ではなく動的に電源を管理する唯一のアプリケーションでした。つまり、最大のパフォーマンスを得るために、CPU周波数を常に100％に維持し、コアがパーキングしないようにする必要がないため、最大負荷での一定の動作と消費電力の増加につながります。これで、ニューラルネットワークによるアプリケーションの分析により、同じ最大パフォーマンスを得ることが可能になりましたが、ユーザーが重いソフトウェアで作業しているときや、要求の厳しいゲームをプレイしているときだけで、残りの時間はコンピューターは電気を熱に変換せず、何の利点もありません。

PCの主要な要素を最適化することに加えて、データ転送の最適化が追加され（トラフィック圧縮とVPNでの広告のブロックにより）、ジャンクファイルの自動クリーニングが行われました。

安定性

これまで、アプリケーションアーキテクチャの正しい選択を繰り返し見てきました。システムでのすべての操作はWinAPIを介して実行されるため、操作パラメーターを変更する際の安定性（システムとソフトウェア）の問題はほぼ完全に解決されました。 2年以上のアプリケーションと、それがインストールされた100万台を超えるデバイスについて、システムの安定性またはサードパーティのアプリケーションとの非互換性に関する問題の1つのケースは登録されていません。

UI / UX

過去の主要な作業領域の1つは、UI / UXの改善でした。これにより、新しいユーザーがアプリケーションの主要な機能を習得するためのしきい値を下げることができました。UI / UX開発のトピックに興味がある人は、ネタバレ下のメインインターフェースの一貫した変更を評価できます。

ハードウェアパフォーマンスの評価

特定のPCが与える条件付きオウムの数を言うだけでなく、アプリケーションがインストールされている実際のすべてのPCに関する個々のコンピューターのパフォーマンスの実際のコンプライアンスを評価する機会がありました。このような関数は、3つの問題を一度に解決します。1つ目は、コンピュータのどのコンポーネントが最も古く、交換する必要があるかという問題です。2番目-このコンピューターが他のどのコンピューターよりも一般的に高速/低速である。3番目に、組み込みアルゴリズムはさまざまなアプローチを使用して、強力なPCと弱いPCのパフォーマンスを向上させ、最良の結果を得ることができます。
