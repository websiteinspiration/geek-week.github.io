在没有YouTube API的情况下解析YouTube（包括上传的数据）

介绍


要在YouTube上加载内容数据，他们通常使用Selenium或YouTube API。但是，到处都有缺点。

  1. 硒太慢，无法解析。想象一下，按硒解析一个约1000个剪辑的播放列表。
  2. 当然，如果您拥有某种应用程序或项目，但YouTube API是最好的选择，但是您需要在那里注册应用程序等。在“试用版”中，您需要不断登录才能使用api，并且还有一个快速结束的配额。
  3. 用我们的方法，我想说YouTube生成的非常复杂的数据结构。解析YouTube搜索特别不稳定。

如何在YouTube上加载数据？


有一个令牌，可以在页面的html代码中找到。然后，将来我们会将其用作向YouTube请求的参数，从而为我们提供新的内容。YouTube本身会根据使用此令牌的请求下载内容。

下一步需要其他传出参数。



通过脚本获取令牌


我们为发送给YouTube的请求编写headers参数。除了用户代理，我们还插入了两个附加代理，您将在下面看到它们。

headers = {
    'User-Agent': '      ,      ',
    'x-youtube-client-name': '1',
    'x-youtube-client-version': '2.20200529.02.01'
}

我们使用请求库发出请求。在您要加载的页面上放置一个链接，并添加标题。

token_page = requests.get(url, headers=headers)

解析器无法找到该令牌，因为该令牌已隐藏在标签标签中。为了将其保存到变量中，我编写了一个丑陋的代码：

nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]

通常，这是一个80个字符长的令牌。

要求内容


service = 'https://www.youtube.com/browse_ajax'
params = {
"ctoken": nextDataToken,
"continuation": nextDataToken
}
r = requests.post(service, params=params, headers=headers)

不同类型的已加载数据具有不同的服务链接。我们的适用于播放列表和频道中的视频。

YouTube以json格式发送数据。因此，我们编写了r.json（），但是列表将飞向您，我们需要列表的第二个元素，因此是r.json（）[1]。此外，我们已经有了数据。它仍有待分析。

解析json响应


您会看到字典的长链，但是我们将缩短它们，以便使它们或多或少清晰。

r_jsonResponse = r_json['response']
dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]

在这里，我们获得了一个新令牌，以进一步请求。如果加载的数据结束，则将看不到令牌。
for content in dataContainer["contents"]:
	videoId = content['playlistVideoRenderer']['videoId']

这是通过添加模板部分来提取视频ID的方法，您将获得视频的链接。

要获取以下数据，您必须执行相同的操作-请求令牌，然后解析。

完整的工作代码如下所示：

import requests, json

url = input('url: ')
headers = {
		"User-Agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/81.0.4044.122 Safari/537.36",
		'x-youtube-client-name': '1',
		'x-youtube-client-version': '2.20200429.03.00',
		}
token_page = requests.get(url, headers=headers)
nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]
sleep = False #  ,    
ids = []
while not sleep:
	service = 'https://www.youtube.com/browse_ajax'
	params = {
	"ctoken": nextDataToken,
	"continuation": nextDataToken
	}
	r = requests.post(service, params=params, headers=headers)
	r_json = r.json()[1]
	r_jsonResponse = r_json['response']
	dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
	try: #  
		nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]
	except:
                #  . ,    .    
		sleep = True
	for content in dataContainer["contents"]:
		videoId = content['playlistVideoRenderer']['videoId']
		ids.append(videoId)
print(len(ids))
