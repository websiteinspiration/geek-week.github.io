不適切なテストについてもう少し

ある日、ユーザーが仮想マシンのRAMのパフォーマンスを監視しようとしているコードに誤って目を奪われました。私はこのコードを提供せず（「足布」がある）、最も重要なものだけを残します。だから、猫はスタジオにいます！

#include <sys/time.h>
#include <string.h>
#include <iostream>

#define CNT 1024
#define SIZE (1024*1024)

int main() {
	struct timeval start;
	struct timeval end;
	long millis;
	double gbs;
	char ** buffers;
	buffers = new char*[CNT];
	for (int i=0;i<CNT;i++) {
		buffers[i] = new char[SIZE];
	}
	gettimeofday(&start, NULL);
	for (int i=0;i<CNT;i++) {
		memset(buffers[i], 0, SIZE);
	}
	gettimeofday(&end, NULL);
	millis = (end.tv_sec - start.tv_sec) * 1000 +
		(end.tv_usec - start.tv_usec) / 1000;
	gbs = 1000.0 / millis;
	std::cout << gbs << " GB/s\n";
	for (int i=0;i<CNT;i++) {
		delete buffers[i];
	}
	delete buffers;
	return 0;
}

すべてがシンプルです。メモリを割り当て、そこに1ギガバイトを書き込みます。そして、このテストは何を示していますか？

$ ./memtest
4.06504 GB /秒

約4 GB /秒。

何？！？！

どうやって？！？！？

これはCore i7（最新ではありません）、DDR4、プロセッサはほとんどロードされていません-なぜですか？！？！

いつものように、答えは異常に普通です。

（ところで、malloc関数のように）新しい演算子は実際にはメモリを割り当てません。この呼び出しでは、アロケータはメモリプール内の空きセクションのリストを確認し、ない場合はsbrk（）を呼び出してデータセグメントを拡大し、プログラムに新しく選択したセクションからのアドレスへのリンクを返します。

問題は、選択された領域が完全に仮想であるということです。実メモリページは割り当てられません。

そして、この選択されたセグメントから各ページへの最初のアクセスが発生すると、MMUがページフォールトを「発射」し、その後、アクセスが行われた仮想ページが実際のページに割り当てられます。

したがって、実際には、バスモジュールとRAMモジュールのパフォーマンスではなく、オペレーティングシステムのMMUとVMMのパフォーマンスをテストしています。RAMの実際のパフォーマンスをテストするには、割り当てられたセクションを一度初期化するだけです。たとえば、次のようになります。

#include <sys/time.h>
#include <string.h>
#include <iostream>

#define CNT 1024
#define SIZE (1024*1024)

int main() {
	struct timeval start;
	struct timeval end;
	long millis;
	double gbs;
	char ** buffers;
	buffers = new char*[CNT];
	for (int i=0;i<CNT;i++) {
                // FIXED HERE!!!
		buffers[i] = new char[SIZE](); // Add brackets, &$# !!!
	}
	gettimeofday(&start, NULL);
	for (int i=0;i<CNT;i++) {
		memset(buffers[i], 0, SIZE);
	}
	gettimeofday(&end, NULL);
	millis = (end.tv_sec - start.tv_sec) * 1000 +
		(end.tv_usec - start.tv_usec) / 1000;
	gbs = 1000.0 / millis;
	std::cout << gbs << " GB/s\n";
	for (int i=0;i<CNT;i++) {
		delete buffers[i];
	}
	delete buffers;
	return 0;
}

つまり、割り当てられたバッファをデフォルト値（char 0）で初期化するだけです。

チェック：

$ ./memtest
28.5714 GB / s

もう1つ。

モラル-迅速かつ迅速に機能する大きなバッファが必要な場合は、それらを初期化することを忘れないでください。
