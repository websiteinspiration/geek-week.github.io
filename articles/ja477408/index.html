アンカーパン


AnchorPaneは、興味深い強力なレイアウトです。これにより、レイアウト内のコンポーネントへのアンカーポイント（アンカー）を定義できます。アンカーには4つのタイプがあります。

  • 上
  • 底
  • 左
  • 正しい

各コンポーネントは、アンカーの任意の組み合わせを持つことができます。ゼロから4つすべてまで。

コンポーネントのアンカーとは、レイアウトの特定のエッジ（たとえば、TOP）から特定の距離を維持することを意味します。この距離は、レイアウトのサイズが変更されても維持されます。

例：anchorRight = 10は、コンポーネントがレイアウトの右端から10の距離を保つことを意味します。

コンポーネントをレイアウトの特定のコーナーにスナップするために、反対方向ではない2つのアンカーポイントを指定できます。

バインディングTOP = 10、RIGHT = 10は、コンポーネントがレイアウトの右上隅に両方のエッジから10の距離を置いたままであることを意味します。



上記の例では、ウィンドウのサイズが変更されても、各コンポーネントのサイズは変更されません。ただし、スナップポイントを反対方向に定義すると、ウィンドウのサイズを変更するときにコンポーネントを増減できます。



たとえば、さまざまなバインディングの組み合わせを使用できます。

  • LEFT + RIGHTは水平方向にサイズ変更します
  • TOP + BOTTOMは垂直方向にサイズ変更します
  • 4つのアンカーすべてを指定すると、水平および垂直の両方のコンポーネントのサイズ変更を意味します

FXMLでアンカーポイントを定義するのは簡単です。次の例には4つのアンカーがすべて含まれていますが、必要なアンカーのみを含めることも、含めないこともできます。

<AnchorPane>
    <Button AnchorPane.topAnchor="10"
            AnchorPane.leftAnchor="10"
            AnchorPane.rightAnchor="10"
            AnchorPane.bottomAnchor="10">I am fully anchored!</Button>
</AnchorPane>

次に、Javaでバインディングがどのように実装されるかを見てみましょう。

AnchorPane anchorPane = new AnchorPane();
Button button = new Button("I am fully anchored!");
AnchorPane.setTopAnchor(button, 10d);
AnchorPane.setBottomAnchor(button, 10d);
AnchorPane.setLeftAnchor(button, 10d);
AnchorPane.setRightAnchor(button, 10d);
anchorPane.getChildren().add(button);

グリッドパン


GridPaneは、コンポーネントをテーブルに配置できるレイアウトです。ここでコンポーネントを1つずつ追加するTilePaneとは異なり、ここでは、新しい各コンポーネントを追加するときに、テーブル内のその位置の座標を指定する必要があります。



<GridPane hgap="10" vgap="10">
    <Label GridPane.rowIndex="0" GridPane.columnIndex="0">First</Label>
    ...
</GridPane>

Javaでは、新しいコンポーネントを追加するときに、最初にColumnIndex（x）パラメータを指定し、次にRowIndex（y）を指定します。

    GridPane grid = new GridPane();
    grid.add(new Label("Hello!"), columnIndex, rowIndex);

間隔


デフォルトでは、テーブルセルにはスペースがありません。コンポーネントは間隔を空けずに隣同士にあります。間隔は、行と列に個別に、つまり水平方向と垂直方向に定義できます。
  • hgapは（列間の）水平間隔を設定します
  • vgapは（行間の）垂直間隔を設定します

<GridPane hgap="10" vgap="10">
    ...
</GridPane>

Javaで定義された間隔：

GridPane grid = new GridPane();
grid.setHgap(10);
grid.setVgap(10);

複数のセルの間隔


GridPaneのコンポーネントは、複数の行や列にまたがることができます。rowspanのあるコンポーネントは、元のセルから下まで伸びます。colspanコンポーネントが右に拡張されます。


Javaでは、rowSpanとcolumnSpanを設定する方法が2つあります。テーブルにコンポーネントを追加するときに、これらを直接インストールできます。

grid.add(component, columnIndex, rowIndex, columnSpan, rowSpan);

またはGridPaneを介して：

GridPane.setColumnSpan(component, columnSpan);
GridPane.setRowSpan(component, rowSpan);

サイジング


元の例ではすべてのセルが同じサイズでしたが、これはそうである必要はありません。表のセルサイズは次のように定義されます。
  • 各行の高さは、行の最大要素に等しい
  • 各列の幅は、列の最も広い要素に等しい

列と行の制約


すでに述べたように、デフォルトでは、列と行のサイズはパネル内のコンポーネントに基づいています。幸い、個々の列と行のサイズをより適切に制御することが可能です。

これには、ColumnContstraintsクラスとRowConstraintsクラスが使用されます。

実際には、2つのオプションがあります。個々の行と列に使用可能なスペースの割合を設定するか、適切な幅/高さを設定します。後者の場合、列と行のサイズを変更するときの望ましい動作を決定することもできます。

パーセント


とてもシンプルです。特定の行または列によって占有される利用可能なスペースの割合を設定できます。レイアウトのサイズを変更すると、行と列も変更され、新しいサイズが反映されます。

<GridPane>
    <columnConstraints>
        <ColumnConstraints percentWidth="50" />
        <ColumnConstraints percentWidth="50" />
    </columnConstraints>
    <rowConstraints>
        <RowConstraints percentHeight="50" />
        <RowConstraints percentHeight="50" />
    </rowConstraints>
    ...
</GridPane>

同じJavaの例：

GridPane gridPane = new GridPane();

ColumnConstraints col1 = new ColumnConstraints();
col1.setPercentWidth(50);
ColumnConstraints col2 = new ColumnConstraints();
col2.setPercentWidth(50);
gridPane.getColumnConstraints().addAll(col1, col2);

RowConstraints row1 = new RowConstraints();
row1.setPercentHeight(50);
RowConstraints row2 = new RowConstraints();
row2.setPercentHeight(50);
gridPane.getRowConstraints().addAll(row1, row2);

絶対サイズ


サイズをパーセンテージで指定する代わりに、推奨サイズと最小サイズを決定できます。さらに、レイアウトのサイズが変更されたときの行/列の動作を指定できます。列は使用hgrowプロパティを、行は持ってvgrowのプロパティを。

これらのプロパティには、3つの異なる意味があります。

  • NEVER：レイアウトのサイズが変更されても、決して増減しません。デフォルト値。
  • ALWAYS（ALWAYS）：レイアウトのサイズを変更すると、この値を持つすべての要素が使用可能なスペースを満たすように引き伸ばされるか、縮小されます。
  • SOMETIMES（SOMETIMES）：これらの要素のサイズは、他の要素がない場合にのみ変更されます。


<GridPane>
    <columnConstraints>
        <ColumnConstraints minWidth="50" prefWidth="100" />
        <ColumnConstraints minWidth="50" prefWidth="100" hgrow="SOMETIMES" />
    </columnConstraints>
    <rowConstraints>
        <RowConstraints minHeight="50" prefHeight="100" />
        <RowConstraints minHeight="50" prefHeight="100" vgrow="SOMETIMES" />
    </rowConstraints>
    ...
</GridPane>

同じJavaの例：

 GridPane gridPane = new GridPane();

ColumnConstraints col1 = new ColumnConstraints();
col1.setMinWidth(50);
col1.setPrefWidth(100);
ColumnConstraints col2 = new ColumnConstraints();
col2.setMinWidth(50);
col2.setPrefWidth(100);
col2.setHgrow(Priority.SOMETIMES);
gridPane.getColumnConstraints().addAll(col1, col2);

RowConstraints row1 = new RowConstraints();
row1.setMinHeight(50);
row1.setPrefHeight(100);
RowConstraints row2 = new RowConstraints();
row2.setMinHeight(50);
row2.setPrefHeight(100);
row2.setVgrow(Priority.SOMETIMES);
gridPane.getRowConstraints().addAll(row1, row2);

個々の行と列にmaxHeightおよびmaxWidth パラメーター（最大の高さと線の幅）を指定することもできます。

ボーダーパン


BorderPaneは5つのセクションレイアウトです。

  • 上
  • 下（下）
  • 正しい
  • 左
  • センター



コンポーネントを個々のBorderPaneセクションに割り当てることができます。

<BorderPane>
    <top>
        <Label>TOP</Label>
    </top>
    <bottom>
        <Label>BOTTOM</Label>
    </bottom>
    <left>
        <Label>LEFT</Label>
    </left>
    <right>
        <Label>RIGHT</Label>
    </right>
    <center>
        <Label>CENTER</Label>
    </center>
</BorderPane>

Javaでの同じ例：

Label top = new Label("TOP");
Label bottom = new Label("BOTTOM");
Label left = new Label("LEFT");
Label right = new Label("RIGHT");
Label center = new Label("CENTER");

BorderPane borderPane = new BorderPane();
borderPane.setTop(top);
borderPane.setBottom(bottom);
borderPane.setLeft(left);
borderPane.setRight(right);
borderPane.setCenter(center);
