Angularを学習するとき、コンテンツの投影などの概念をよく見逃しているか、十分な注意を払っていません。これは、柔軟で再利用可能なコンポーネントを作成するための非常に強力なツールです。ただし、そのドキュメントでは、ライフサイクルフックセクションの2つの段落についてのみ言及しています。この脱落を修正してみましょう。



ng-contentを使用したコンテンツの投影


コンテンツプロジェクションは、コンポーネントの外部からHTMLコンテンツをインポートし、特定の場所のコンポーネントテンプレートに貼り付ける方法です。（ドキュメントの無料翻訳）
定義はかなり複雑ですが、実際にはすべてがはるかに単純です。ある種のコンポーネントがあり、その開始タグと終了タグの間にあるものはすべてコンテンツです。

<app-parent>
    <!-- content -->
    I'm content!
    <!-- content -->
</app-parent>

また、Angularでは、elementを使用して、このコンポーネントのテンプレートにHTMLコード（コンテンツ）を埋め込むことができますng-content。

これが必要な理由と、例を使用してどのように機能するかを理解してみましょう。単純なボタンコンポーネントがあるとしましょう。を介してテンプレートに渡すこのボタンのテキストinput property。

// button.component.ts
import {Component, Input} from '@angular/core';

@Component({
  selector: 'app-button',
  template: '<button>{{text}}</button>'
})
export class ButtonComponent {
  @Input() text: string;
}

// app.component.ts
import {Component} from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<app-button [text]="'Button'"></app-button>`,
})
export class AppComponent {
}

よさそうだ。しかし、突然、一部のボタンのテキストにアイコンを追加する必要がありました。すでにアイコンコンポーネントがあります。それをボタンテンプレートに追加し、ディレクティブngIfを添付してinput property、アイコンを動的に表示する別のディレクティブを作成するだけです。

// icon.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-icon',
  template: '☻',
})
export class IconComponent {
}

// button.component.ts
import {Component, Input} from '@angular/core';

@Component({
  selector: 'app-button',
  template: `<button>
               <app-icon *ngIf="showIcon"></app-icon>
               {{text}}
             </button>`,
})
export class ButtonComponent {
  @Input() text: string;
  @Input() showIcon = true;
}

すべてが機能しています。しかし、テキストに対してアイコンの位置を変更する必要がある場合はどうなりますか？または別の新しいアイテムを追加しますか？既存のコードを編集したり、新しいプロパティを追加したりする必要があります。

これはすべてで回避できますng-content。コンテンツのプレースホルダーと見なすことができます。コンポーネントの開始タグと終了タグの間に置いたすべてが表示されます。

// button.component.ts
import {Component} from '@angular/core';

@Component({
  selector: 'app-button',
  template: `<button>
               <ng-content></ng-content>
             </button>`,
})
export class ButtonComponent {
}

// app.component.ts
import {Component} from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<app-button>
               <app-icon></app-icon>
               Button
             </app-button>`,
})
export class AppComponent {
}

select属性はng-contentに対してどのような役割を果たしますか？


一部のコンテンツを残りのコンテンツと比較して特定の場所に配置する必要がある場合があります。この場合selectは、セレクター（.some-class, some-tag, [some-attr]）を受け入れる属性を使用できます。

// button.component.ts
import {Component} from '@angular/core';

@Component({
  selector: 'app-button',
  template: `<button>
               <ng-content></ng-content>
               <div>
                 <ng-content select="app-icon"></ng-content>
               </div>
             </button>`,
})
export class ButtonComponent {
}

ngProjectAsとは何ですか？


selecty 属性ng-contentは、親コンポーネントのネストの最初のレベルにあるタグに対応します。しかし、アイコンコンポーネントをタグでラップしてネストレベルを上げるとどうなるでしょうか。

// app.component.ts
import {Component} from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<app-button>
               <ng-container>
                 <app-icon></app-icon>
               </ng-container>
               Button
             </app-button>`
})
export class AppComponent {}

selectまったく存在しないかのように機能しない ことがわかります。これ<ng-content select="...">は、親のコンテンツのネストの最初のレベルでのみ検索するために発生します。この問題を解決する属性がありますngProjectAs。セレクターを受け取り、その下のDOMノード全体を「マスク」します。

// app.component.ts
import {Component} from '@angular/core';

@Component({
  selector: 'app-root',
  template: `<app-button>
               <ng-container ngProjectAs="app-icon">
                 <app-icon></app-icon>
               </ng-container>
               Button
             </app-button>`
})
export class AppComponent {}

ケース* ngIf + ng-content


別の興味深いケースを見てみましょう。アイコンの非表示/表示ボタンをクリックする必要があるとします。アイコンの表示を担当するボタンコンポーネントクラスにブールプロパティを追加し、ボタンをクリックして変更し、それをハングさせngIfます。

// button.component.ts
import {Component, Input} from '@angular/core';

@Component({
  selector: 'app-button',
  template: `<button (click)="toggleIcon()">
               <ng-content></ng-content>
               <div *ngIf="showIcon">
                 <ng-content select="app-icon"></ng-content>
               </div>
             </button>`,
})
export class ButtonComponent {
  showIcon = true;

  toggleIcon() {
    this.showIcon = !this.showIcon;
  }
}

アイコンは非表示/クリックで表示されます。いいね！しかし、フックOnInitとOnDestroyアイコンコンポーネントのログをいくつか追加してみましょう。ディレクティブ、ということは周知の事実であるngIf条件が変更され、完全に要素を作成/削除、およびOnDestroy/ OnInit適宜たびに働かなければなりません。

// icon.component.ts
import { Component, OnInit, OnDestroy } from '@angular/core';

@Component({
  selector: 'app-icon',
  template: '☻',
})
export class IconComponent implements OnInit, OnDestroy {
  ngOnInit() {
    console.log('app-icon init');
  }

  ngOnDestroy() {
    console.log('app-icon destroy')
  }
}
