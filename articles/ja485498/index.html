Spring Cloud Open Service Broker 3.1


Spring Cloud Open Service Brokerは、Open Service Broker APIを実装するSpring Bootアプリケーションを構築するためのプラットフォームであり、開発者はCloud Foundry、Kubernetes、OpenShiftなどのクラウドプラットフォームで実行されているアプリケーションにサービスを提供できます。バージョン3.0以降、Spring Cloud Open Service Brokerは、コントローラーおよびサービスインターフェイスでReactiveタイプを使用して、Spring WebFluxおよびSpring MVC Webフレームワークをサポートします。

Spring Cloud Open Service Brokerを開始するには、アプリケーションにSpring Bootスターターを含めます。

implementation('org.springframework.cloud:spring-cloud-starter-open-service-broker:3.1.0.RELEASE')

春のデータムーア

Reactive MongoDBを使い始めるには、アプリケーションにSpring Bootスターターを含めます。

implementation('org.springframework.boot:spring-boot-starter-data-mongodb-reactive')

Spring Security 5.2


リアクティブサポートは元々Spring Security 5に含まれており、Spring BootおよびSpring Frameworkとの統合は進化を続けています。

Spring Securityを使用するには、Spring Bootスターターを追加します。

implementation('org.springframework.boot:spring-boot-starter-security')

次に、でセキュリティ構成を定義します@EnableWebCardingSecurity。このコードは、1つのアプリケーションがブローカーのエンドポイント/v2を保護し、/bookstarsサービスインスタンスからのリクエストに応答する方法を示しています。

@Configuration
@EnableWebFluxSecurity
public class SecurityConfiguration {

    @Bean
    public SecurityWebFilterChain securityWebFilterChain(
        ServerHttpSecurity http) {
        return http
            .csrf().disable()
            .httpBasic()
            .and().authorizeExchange()
            .pathMatchers("/bookstores/**").authenticated()
            .pathMatchers("/v2/**").hasAuthority(
                SecurityAuthorities.ADMIN)
            .matchers(EndpointRequest.to("info", "health")).permitAll()
            .matchers(EndpointRequest.toAnyEndpoint()).hasAuthority(
                SecurityAuthorities.ADMIN)
            .and().build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}

次に実装しますReactiveUserDetailsService：

@Service
public class RepositoryUserDetailsService implements
ReactiveUserDetailsService {

    private final UserRepository userRepository;

    public RepositoryUserDetailsService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

最後に、パーミッションエバリュエーターはWebFluxコントローラーではサポートされていませんが、SpEL式でBeanを呼び出し、Authenticationオブジェクトを渡すことで、同様の機能を実現できます。

@GetMapping("/{bookStoreId}")
@PreAuthorize("hasAnyRole('ROLE_FULL_ACCESS','ROLE_READ_ONLY') and
@bookStoreIdEvaluator.canAccessBookstore(authentication, #bookStoreId)")
public Mono<ResponseEntity<BookStoreResource>> getBooks(
	@PathVariable String bookStoreId) {
	return bookStoreService.getBookStore(bookStoreId)
			.flatMap(this::createResponse);
}

この例では、許可を分析して、書店IDが存在するかどうかを判別します。

public boolean canAccessBookstore(Authentication authentication,
    String bookStoreId) {
    return authentication.getAuthorities().stream()
        .filter(authority - > authority.getAuthority()
            .startsWith(BOOK_STORE_ID_PREFIX))
        .map(authority - > {
            String serviceInstanceId = authority.getAuthority()
            .substring(BOOK_STORE_ID_PREFIX.length());
            return serviceInstanceId.equals(bookStoreId);
        })
        .findFirst()
        .orElse(true);
}

春のHATEOAS 1.0


Spring HATEOAS 1.0 GAが最近リリースされ、ビューのリンクとモデリングの反応的なサポートが含まれています。

Spring HATEOASスターターを追加して、Spring Boot自動構成をアクティブにします。Spring WebFlux Reactive Applicationを作成しているので、Spring Webスターターを除外する必要があります。

implementation('org.springframework.boot:spring-boot-starter-hateoas') {
    exclude group: 'org.springframework.boot', module: 'spring-boot-starter-web'
}

WebFluxLinkBuilder ハイパーメディアリソースの構築に は、以下を使用できます。

public Mono < BookResource > toModel(Book book, String bookStoreId) {
    return Mono.just(new BookResource(book))
        .flatMap(bookResource - > linkTo(methodOn(
                BookController.class).getBook(bookStoreId, book.getId()))
            .withSelfRel()
            .toMono()
            .flatMap(link - > Mono.just(bookResource.add(link)))
            .thenReturn(bookResource));
}

次に、このリソースをコントローラーレスポンスの本文で使用できます。

return new BookStoreResourceAssembler().toModel(bookStore)

Spring Framework 5.2


Spring Framework 5は当初、新しいSpring WebFlux Web Frameworkでリアクティブサポートを提供していました。さらに、新しいWebClientおよびWebTestClientには、Spring WebFluxアプリケーションの使用とテストのサポートが含まれています。

それらを使用するには、Spring Bootスターターを追加するだけです。

implementation('org.springframework.boot:spring-boot-starter-webflux')
testImplementation('org.springframework.boot:spring-boot-starter-test') {
    exclude group: 'org.junit.vintage', module: 'junit-vintage-engine'
}
