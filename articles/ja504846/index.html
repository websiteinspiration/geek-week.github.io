Veeam Backup for Microsoft Azureの概要

現代では、安定性と斬新さの両方が楽しいです。ここで、Veeamはそれ自体に忠実です。製品は計画に従ってリリースされ、エンジニアは数週間前にリリースされたVeeam Backup for Microsoft Azureを含む新しいソリューションを開発しています。最後の投稿で、私の同僚はロクスマティマモンその利点と作業の原則についてはすでに説明しました。今日は、新しいソリューションの導入と構成について検討します。



ご記憶のとおり、この製品はMicrosoft Azureクラウドにデプロイされた仮想マシンをバックアップするように設計されています。クラウドインフラストラクチャを安全かつ効率的にバックアップし、必要に応じてそれらを迅速に復元するのに役立ちます。Veeam Backup＆Replicationとの統合がサポートされているため、ハイブリッドインフラストラクチャで作業している組織では、仮想マシンと物理マシンで作業するためのプラットフォーム全体をクラウド内でも独自の容量でも展開できます。

詳しくは猫へようこそ。

したがって、Veeam Backup for Microsoft Azureでは次のことが可能です。

  • バックアップAzure VMのポリシーを構成する
  • Azure VMのネイティブスナップショットを操作する
  • Azure BLOBストレージへのバックアップ
  • 標準のVeeam形式でバックアップを作成する
  • クラウドストレージコストを計算する
  • VM全体または単一のファイルのみを復元する

Azure Marketplace から直接、すばやく簡単にデプロイできます。



バックアップのセキュリティを確保するために、Microsoft AzureサービスアカウントとActive Directory統合が使用されます。

すぐに使える無料のバージョンでは、最大10台の仮想マシンをバックアップできます。（ライセンスについて詳しくは、以前の記事をご覧ください。）

中は何ですか？


主なコンポーネントはコントローラーサーバーであり、Microsoft AzureクラウドにデプロイされたLinuxベースの仮想マシンです。Veeam Backup for Microsoft Azureがプリインストールされています。

コントローラーサーバーはすべての重要な機能を実行します：

  • バックアップインフラストラクチャ管理
  • Microsoft Azure VMのスナップショットの作成、バックアップ、およびリカバリを制御します。
  • スケジュールされたバックアップポリシーのサポート
  • VM、アカウントなどによって保護されたポリシーに関する情報を格納する構成データベースの管理。

特定のクラウドインフラストラクチャオブジェクトを保護するために、バックアップポリシーが作成されます。それらを作成する場合、保護オブジェクトの地域的な所属、サービスレベルアグリーメント（SLA）などに焦点を当てることができます。後でポリシー設定について検討します。

Veeam Backup for Microsoft Azureは、個々のVMを処理するために、補助コンポーネントであるworkerを使用します。また、それらの設定についても少し低く説明します。

Veeam Backup for Microsoft Azureは、Azure仮想マシン（VM構成を含む）の管理対象ドライブと非管理対象ドライブのスナップショットとバックアップの両方の作成をサポートしています。

概略的に、バックアップ中のソリューションの操作は次のように表すことができます。



バックアップポリシーが適用されると、Veeam Backup for Microsoft Azureは次のことを行います。

  1. Microsoft Azureから各VMの構成とVHDを取得します。
  2. これらのディスクのスナップショットとバックアップコピーを作成して保存します。
    • マネージドディスクのスナップショットは、Azure Compute APIを介して、元のVMと同じリソースグループに保存されます。
    • 非管理対象ディスクのスナップショットは、元のVHDが配置されているMicrosoft Azureストアに保存されます。Blobは、Blob Service REST APIを使用してデータをblobに追加します。
    • すべてのVHDバックアップは、Blob Service REST APIを介してblobとしてリポジトリに保存されます。リポジトリにバックアップすると、データの暗号化と圧縮が実行されます。
    • VM構成データは、Veeam Backup for Microsoft Azure構成データベースに保存されます。

リソースグループ、サブスクリプション、その他のMicrosoft Azureオブジェクトを操作するには、このソリューションにサービスアカウントが必要です。これは、特別に設計されたAzure ADアプリケーションで使用されます。このアプリケーションは事前に準備することも、Azureクラウドで自動的に作成して認証する必要があることを示すこともできます。どちらのオプションもユーザーマニュアル（英語）に記載されています。

バックアップインフラストラクチャを構成する


サーバーが構成されました。リポジトリと補助コンポーネントであるworkerを構成します。

  • Microsoft Azureのマシンをネイティブスナップショットのみを使用して保護する場合は、Marlezonバレエの2番目の部分に直接移動して、ワーカーを構成できます。
  • Microsoft Azure BLOBストレージをバックアップストレージとして使用する場合は、Veeam Backup for Microsoft Azureコンソールでバックアップリポジトリエンティティを作成し、対応するアーカイブでコンテナーとフォルダーを指定する必要があります。これらの簡単な手順については、こちらで説明しています。

ワーカーと 同様に、これらは補助的なLinuxマシンであり、プロセスのスケーリングとデータ転送中の負荷の分散に役立ちます。これらのコンポーネントの構成は、コントローラーサーバー（バックアップまたは復元中に自動的に行われます）またはユーザーによって作成されます。

構成は以下を定義します。

  • 並行して作業できるワーカーの数
  • これらのVMが属する仮想ネットワークとセキュリティグループ
  • 彼らはどのようなサイズになります

自動的に作成されたワーカー構成はデフォルトと見なされます。これは1つのAzureリージョンに属し、1〜5 人のワーカーを含むことができます。ここで
説明するように、ユーザーは手動で構成を作成できます。労働者 の仕事について、次のことを覚えておく必要があります。





  • 1つのワーカーは1つのVMからのみデータを処理できます。つまり、バックアップポリシーに30台のマシンがある場合、Veeam Backup for Microsoft Azureは30台のそのようなワーカーを（デフォルト以外の構成を使用して）デプロイできます。
  • デフォルトの構成が関係している場合（言われているように、これは最大5つのワーカーです）、5つのマシンが同時に処理され、残りはインラインで待機します。
  • workerを含む各VMで、コントローラーサーバーは次のサービスをインストールします。

    • ワーカーサービス -このサービスは、Azureクラウドからデータを取得します。
    • ファイルレベルの回復サービス - ファイルレベルの回復中に、このサービスは、バックアップからワーカー VMにデータをマウント（マウント）します。

  • 非アクティブな期間（アイドル時間= 10分）の後、ワーカー VMは自動的にシャットダウンします。

ここの政治家は、バックアップジョブのアナログとして機能します。これらは、ウィザードを使用して構成されます。指定する必要がある手順は次のとおりです。

  • Azure Active Directory
  • 保護されたリソースが配置されている地域
  • ポリシーが適用されるオブジェクト（リソース）のタイプ：
    • 申し込み
    • Resource group —
    • Tags —
    • Individual VMs —

Azure VMバックアップのコストの見積もりに関しては、ポリシー作成および編集ウィザードに組み込まれた計算機によって自動的に実行されます。トラフィック、トランザクション、バックアップ、スナップショットの毎月のコストの計算に基づいています。計算機は、クラウドインフラストラクチャのコストを正当化して計画するのに役立ちます。
そのおかげで、このような設定のポリシーに従ってバックアップを実行すると、バックアップにかかる費用がすぐにわかります。

回復


Microsoft Azureユーザー向けにVeeam Backupに提供されるリカバリオプションは次のとおりです。

  • VM全体を元の場所に復元（元のマシンを上書き）
  • VM全体を新しい場所に復元する
  • ファイルレベルの回復

有用：ファイルレベルでの復元は、マシン全体を持ち上げる必要なくバックアップから必要なデータを取得できるという点で優れています（節約は明らかです）。



Veeam Backup＆Replicationとの統合


Microsoft Azureでは、Veeam Backup＆Replicationによって作成されたバックアップからのリカバリが直接サポートされています。VeeamBackup for Microsoft AzureはVM保護を扱います。

またはその逆：Veeam Backup＆ReplicationはAzure BLOBストレージ（Veeam Backup for Microsoft Azureバックアップ）に基づいたリポジトリで動作できるため、Microsoft Azure VMマシンをバックアップから「地上」インフラストラクチャに復元できます。 Veeam Backup＆Replicationによってすでに保護されています。一般的に、ハイブリッドインフラストラクチャの所有者を満足させるすべてのもの。
