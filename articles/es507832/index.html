No quiero Visual Studio Code: 7 alternativas de código abierto

En esta traducción, hablaremos sobre editores de código alternativos que de ninguna manera son inferiores a VS Code, e incluso con una selección superior de extensiones que lo superan.







Visual Studio Code Editor, también conocido como VS Code, se ejecuta en Linux, Windows y macOS. Es intermedio entre un simple editor de código fuente de texto y un entorno de desarrollo integrado (IDE) que gestiona toda su base de código. La funcionalidad de VS Code se puede ampliar con complementos. Este es un editor de código confiable que puede competir dignamente con competidores propietarios.



VS Code fue creado originalmente como un producto de código abierto. Sin embargo, sus compilaciones terminadas se distribuyen bajo una licencia patentada de Microsoft. Pero no es tan malo.





Usando VS Code como código abierto



Las fuentes de VS Code están disponibles en GitHub . Sin embargo, cuando descarga el ensamblaje terminado , la Licencia de Microsoft entra en juego . Y luego su Código VS de repente deja de ser un producto de código abierto. Resulta que en este caso resuelve el proceso de ensamblaje.



Chris Díaz, el desarrollador del proyecto VS Code, está llevando a cabo un ejemplo similar del navegador Chrome y su proyecto de código abierto Chromium. VS Code está realmente construido sobre la base de soluciones de código abierto. Sin embargo, la compilación oficial de Microsoft está configurada para tener en cuenta características específicas, incluidas la marca registrada, la galería de extensiones, el depurador C # patentado y la telemetría. Pero cuando clona y crea el ensamblaje usted mismo, genera una versión más limpia llamada Code-OSS (OSS significa software de código abierto).



En la práctica, las diferencias entre VS Code y Code-OSS son mínimas. En particular, la telemetría funciona en VS Code: es decir, Microsoft recopila datos sobre el uso del producto. Para ser justos, debe tenerse en cuenta que hoy en día hay muchos productos que recopilan dichos datos.



Si te gusta VS Code, pero quieres considerar las alternativas de código abierto "verdaderas", entonces definitivamente deberías seguir leyendo.



Vscodium











La alternativa más obvia a VS Code es VS Code sin los complementos patentados de Microsoft. El proyecto VSCodium contiene solo componentes gratuitos creados sobre la base de la base del código VS Code. Los desarrolladores de VSCodium han hecho todo lo posible para que podamos deshabilitar la telemetría. Han preparado una compilación limpia del código fuente del Código VS, que puede descargar sin crearlo usted mismo.



Los desarrolladores recuerdan que VS Code incluye algunas herramientas propietarias que pueden no incluirse con código abierto. Si los necesita, hay formas documentadas de resolver estos problemas. Pero si necesita una característica muy específica de la compilación VS Code de Microsoft, debe asegurarse de que funcione en VSCodium.



Además, debe verificar que toda la telemetría esté deshabilitada .



Code-OSS



Si no desea utilizar el ensamblaje VSCodium, puede compilar VS Code usted mismo a partir del código fuente y terminar con lo mismo. El ejecutable se llama Code-OSS, no VSCode. Las condiciones de licencia y las soluciones a los problemas que utilizan características propietarias son las mismas que VSCodium.



Al principio, debe verificar que la telemetría esté deshabilitada.



Átomo











Atom es un editor de texto de código abierto tipo IDE que Microsoft adquirió con GitHub. Al igual que VS Code, puede ampliar el editor Atom con complementos, agregar temas y crear su propia combinación de herramientas adicionales. Este es un producto multiplataforma que tiene integración integrada con GitHub. En resumen, Atom es ideal si las extensiones que necesita ya existen o está listo para escribirlas.



Al igual que VS Code, Atom usa la telemetría de forma predeterminada.. Se puede deshabilitar y, a diferencia del Código VS, no hay restricciones en las extensiones, por lo que no necesita cambiar el flujo de trabajo debido a los requisitos de privacidad. Atom es ciertamente una herramienta útil para desarrolladores. Además: en principio es adecuado para todos los que usan una computadora. Si está buscando un buen editor de texto de uso general, puede probar Atom.



GNOME Builder











GNOME Builder es un editor de código de Linux diseñado específicamente para construir aplicaciones basadas en GNOME. Si está creando aplicaciones para Linux y desea lograr fácilmente la compatibilidad, entonces Builder es la mejor opción. GNOME Builder está disponible en Flathub.org . Cuando lo inicie por primera vez, le pedirá que instale el SDK de GNOME si no lo ha hecho.



Sin embargo, puede usar Builder no solo para desarrollar aplicaciones GNOME. Es compatible con docenas de lenguajes de programación, incluidos Python, Rust, C y C ++, Java, Go, JavaScript, TypeScript, VB.NET, varios lenguajes de marcado, Markdown y muchos otros. Algunos de ellos tienen soporte completo con autocompletar y fragmentos, mientras que otros tienen servicios mínimos, como resaltado de sintaxis y coincidencia automática de paréntesis. Es bueno trabajar con el IDE, ya sea que te consideres un programador serio o solo quieras disfrutar de HTML y CSS.



Geany











Geany es un editor potente, confiable y liviano con características útiles que lo ayudarán a escribir un buen código en Bash, Python, Lua, XML, HTML, LaTeX, etc. Admite 50 lenguajes de programación y lenguajes de programación diferentes, así como lenguajes y archivos de marcado (como .diff y .po). Geany al menos proporciona corchetes coincidentes y resaltado de sintaxis, y generalmente ofrece mucho más.



Mediante el uso de complementos, puede agregar funciones como el navegador del proyecto, el explorador, la depuración, la terminal, etc. Puede continuar expandiendo su funcionalidad y en algún momento se convertirá en un IDE real. Si no puede ejecutar VS Code en su computadora debido a limitaciones de CPU o RAM, Geany puede ser una buena alternativa. Se inicia rápidamente y consume una pequeña cantidad de memoria. Aunque Geany es más lenta que Vim en la terminal, no se ralentiza ni siquiera en la Raspberry Pi.



Soportes











Brackets es un editor de texto e IDE para desarrolladores web. Tiene un poderoso soporte para HTML, CSS, JavaScript, PHP e incluso Python. Brackets tiene un rico ecosistema de extensiones, por lo que con los complementos adecuados le permitirá escribir en casi cualquier idioma moderno.



Hay extensiones para ayudar a analizar, ejecutar scripts e incluso compilar y ejecutar código directamente en la ventana del editor. Brackets tiene una interfaz tradicional que puede ser entendida por casi cualquier persona que vaya más allá del portátil. Si pasa un poco de tiempo agregando las extensiones apropiadas y conociéndolas, los corchetes que usan el autocompletado y un linter lo ayudarán a evitar errores estúpidos de escritura. Y si escribe código, esto acelerará el proceso de prueba y depuración.



Che











Si te gusta adelantarte al resto, entonces deberías probar el Che . Este es un IDE basado en la nube, funciona en el modelo SaaS. Sin embargo, es un proyecto de código abierto completo. Si tiene una copia de Kubernetes, puede transferirla completamente a usted mismo.



El Che no es solo un IDE que vive en la nube. Este es un IDE diseñado para el desarrollo de la nube. No intenta almacenar nada en su sistema de archivos local. Si tiene un servidor Git, puede considerarlo como un sistema de archivos y trabajar en un proyecto directamente desde su repositorio. Por supuesto, también puede descargar cualquier proyecto en el que esté trabajando si desea tener una copia de seguridad local.



Pero su característica principal admirada por los desarrolladores de la nube es que Che es un entorno de desarrollo de código abierto completo centrado en Kubernetes. Si está creando aplicaciones, sitios o contenedores (o una combinación de ellos) para la nube, entonces debería probar el Che.
