JavaScriptで継承されたクラスを初期化する予期しない順序

今日、私はJSコードをリファクタリングする小さなタスクを持っていました、そして私はこの言語の予期しない機能に出会いました。 多くの人に嫌われ 言語は考えず、出くわしませんでした。


加えて、私はロシア語または英語のインターネットで何も見つけることができなかったので、これをそれほど長くはなく、最も面白くはないが有用なメモとして公開することにしました。


従来の意味のない定数を使用しないようにするためfoo/barに、プロジェクトで使用した例を直接示しますが、内部ロジックの束はなく、偽の値があります。同じことを思い出してください。例は非常に総合的なものであることがわかりました。

したがって、クラスがあります。


class BaseTooltip {
    template = 'baseTemplate'
    constructor(content) {
        this.render(content)
    }
    render(content) {
        console.log('render:', content, this.template)
    }
}

const tooltip = new BaseTooltip('content')
// render: content baseTemplate

すべてが論理的です


次に、フィールドが変化する別のタイプのツールチップを作成する必要がありました。 template

    template = 'otherTemplate'

const specialTooltip = new SpecialTooltip('otherContent')

Chrome DevToolsはクラスフィールドを割り当てる方法を知らないため、何が起こっているのかを理解するためにトリックに頼る必要があります。小さなヘルパーを使用して、変数への割り当ての瞬間を記録します


function logAndReturn(value) {
    console.log(`set property=${value}`)
    return value
}

class BaseTooltip {
  template = logAndReturn('baseTemplate')
  constructor(content) {
      console.log(`call constructor with property=${this.template}`)
      this.render(content)
  }
  render(content) {
    console.log(content, this.template)
  }
}

const tooltip = new BaseTooltip('content')
// set property=baseTemplate
// called constructor BaseTooltip with property=baseTemplate
// render: content baseTemplate
