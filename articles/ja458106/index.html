Androidアプリケーション用の新しいUIフレームワークを試す


過去数年間、私は多くのモバイルプロジェクトに参加していたため、Android、ReactNative、Flutterなどのさまざまなテクノロジーを使用する必要がありました。 ReactNativeからクラシックなAndroidに戻すと、複雑な気持ちになりました。 Kotlinへの復帰はうまくいきましたが、React UIフレームワークは本当に見逃しました。ユーザーインターフェースを構成する再利用可能な小さなコンポーネントは優れており、開発の柔軟性とスピードを向上させます。

従来のAndroidに戻ると、View階層をできるだけ均一に保つことを心配する必要がありました。このため、コンポーネントアプローチに真に専念することは困難です。これにより、コピーと貼り付けがより魅力的になり、コードがより複雑になり、サポートが少なくなります。最終的には、UXを改善する可能性のあるユーザーインターフェイスを試してみるのを避けています。

Android UIフレームワークの現在の構造は2008年から存在しており、時間の経過とともに複雑化しており、維持するのは非常に困難です。Jetpack Composeは、最新のコンポーネントの哲学を考慮して、最初から始めることを目指しています。フレームワークは、次の主要な目標を念頭に置いて作成されています。

これにより、分離された再利用可能なコンポーネントを簡単に作成でき、既存の要素を使用して新しい画面を簡単に作成できます。開発者がビューの階層を制御してビューとフラグメントを制御するのではなく、便利なユーザーインターフェイスの作成に集中できるように支援します。

class ComposeActivity : Activity() {

カスタム状態モデル


+state {}単一の値を使用してモデルを作成 する代わりに、@ Modelアノテーションを使用してカスタムモデルを作成することもできます。カウンターアプリケーションを小さなウィジェットに分割し、モデルを更新されてこのモデルの状態を表示する他のウィジェットに渡すことで、カウンターアプリケーションを改善できます。

@Model
class CounterModel {
    var counter: Int = 0
    var header = "Counter demo"

    fun add() { counter++ }

    fun subtract() { counter-- }
}


@ModelCompose Compilerプラグインは、 アノテーションを使用して、モデル内のすべての変数を監視可能にし、ウィジェットを再配置するために使用できるようにします。使用するようにウィジェットを更新しましょうCounterModel：

@Composable
fun CounterHeader(counterModel: CounterModel) {
    Text(text = counterModel.header)
}

@Composable
fun AddSubtractButtons(counterModel: CounterModel) {
    Button(
        text = "Add",
        onClick = { counterModel.add() })

        text = "Subtract",
        onClick = { counterModel.subtract() })
}

@Composable
fun CounterLabel(counterModel: CounterModel) {
