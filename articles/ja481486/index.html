「Stayin 'alive、stayin' alive」：新しいプロトコルにより、Wi-Fiの使用可能範囲が60メートル広がります

私たちのブログでは、さまざまなネットワークプロトコルのデバイスについてよく話します。今日は、Wi-Fiネットワークの機能を拡張するONPCについて話しましょう。猫の下でそれがどのように機能するかについて。


/ Unsplash / ジェドンケリー

プロトコルはどのように登場しましたか


アメリカのブリガムヤング大学のエンジニアは、医学分野の研究プロジェクトに取り組み、実験の参加者の家にセンサーを設置する必要がありました。センサーはWi-Fi経由で監視データを送信し、ルーターの受信端に配置されていました。

IoTデバイスは、長時間ネットワークから「脱落」することがあります。一度、洗濯かごがセンサーの動作を妨げました-信号をブロックしました。したがって、大学の専門家は、デバイスが機能しているか壊れているかを判断し、交換する必要があるメカニズムを必要としていました。

解決策は新しいRF でしたプロトコルONPC-On-Off Noise Power Communication。802.11 Wi-Fi上で実行され、ネットワークノイズの周波数変動に基づいてデバイスのステータスを決定します。新しいアプローチにより、Wi-Fiルーターの標準半径の外側に設置され、67メートル離れた場所にあるガジェットを使用できるようになりました。

彼はどのように働きますか


センサーがデータを送信すると、ワイヤレスネットワークの総ノイズレベルが増加します。逆に、パケットブロードキャストが停止すると、ノイズが減少します。大学のエンジニアは、ネットワークを監視するときにこの機能を考慮に入れているルーター用のソフトウェアコントローラーを開発しました。

システムはStayin 'Aliveという名前で、ONPC機能を制御します。コントローラーは、IoTデバイスから長時間データを受信して​​いないことを認識すると、ルーターをONPCモードに切り替え、空中の特別な識別ノイズを検索します。このノイズは、スマートデバイスの送信機を定期的にオン/オフすることで発生します。

シーケンスは、IoTデバイスがWi-Fiから切断されるたびにブロードキャストされます。コントローラーが無線で対応する識別子を検出すると、デバイスは動作していると判断します-オンラインではありません。それ以外の場合は、故障の可能性について管理者に通知を送信します。

ONPCの機能


開発者によると、これまでのところ、プロトコルの帯域幅は狭く、1秒に1ビットしか送信されません。ただし、彼らは技術に取り組んでいきます。将来的には、このシステムは、スマートデバイスのパフォーマンスを監視するためのメカニズムとしてだけでなく、アプリケーションを見つけるでしょう。

これに基づいて、ガレージのドアセンサー、空気品質モニター、灌漑システム、その他多くの小さなスマートガジェットを構築できます。シングルビットチャネルで、オンとオフを切り替えることができます。

また、ONPCがネットワーク帯域幅を20％削減できる場合があることにも注意しています。CSMA多重アクセス技術は責任がある、ルータモードを変更するときにフレームの送信を一時停止します。速度はWi-Fiアクセスポイントのすぐ近くにあるデバイスでのみ低下します。


ONPCに加えて、長距離でデータを送信するための他のソリューションがあります。最も有望なのは、MACプロトコルLoRaWANです。これは、物事のインターネットのガジェット（カウンターやセンサー）からデータを収集するために使用されます。夏の終わりに、スペインのエンジニアがLoRaWANデバイスの伝送距離の記録を塗り替えました。彼らは766キロの距離にパケットを送りました。放送は山中のスキー場の気球から基地局まで行われた。


/ Unsplash / Webaroo

しかし、LoRaWANプロトコルはWi-Fiよりもはるかに長い距離でデータを送信できますが、使用する周波数が低く、機能するには特定のインフラストラクチャが必要です。

ONPCは、従来のルーターと互換性があります。ブリガムヤング大学のエンジニアチームも将来的には、それらのソリューションはBluetoothなどの他のワイヤレス技術に適用できるようになると指摘しています。確かに、多数のスマートフォンが集中しているショッピングセンターのような強力な「ノイズ汚染」のある環境でプロトコルがどのように現れるかはまだ不明です。
