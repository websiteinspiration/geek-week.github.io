プロジェクトのビルドプロセスの一部としてPurifyCSSを使用したCSSのPurify


PurifyCSSは、オンラインCSSクリーナーとしてよりも、プロジェクトのビルドに使用されるツールとしてより頻繁に使用されます。プロジェクトのドキュメントには、Grunt、Gulp、webpackを使用する際の使用方法が記載されています。たとえば、テンプレートによるファイルの処理：

var content = ['**/src/js/*.js', '**/src/html/*.html'];
var css = ['**/src/css/*.css'];

var options = {
  //  CSS    .
  output: './dist/purified.css'
};

purify(content, css, options);

このアプローチにより、はるかに高い精度が可能になります。分析のために送信されるサイトの資料は、各テンプレート、ページの作成に使用される各フラグメント、およびすべてのJavaScriptファイルを含むリストにすることができます。そのようなリソースのリストを維持するのは容易ではないかもしれませんが、考慮に入れることができるすべてのものを検討することができます。ただし、これは特定のリポジトリにあるページのコンテンツ（データベースに保存されているブログ投稿など）やサードパーティのJavaScriptコードには適用されませんが、場合によっては重要ではないか、おそらく開発者は、そのようなリソースのCSSニーズを何らかの形で考慮することができます。

ではドキュメントPurgeCSSのライバルPurgeCSSには、PurifyCSSで使用されるリソース処理方法に関する警告があります。つまり、PurifyCSSはHTMLやJavaScriptだけでなく、あらゆるタイプのリソースを処理できるということです。作業中のPurifyCSSは、ファイル内のすべての単語を分析し、CSSコード内のセレクターと比較します。各単語はセレクターと見なされます。つまり、多くのセレクタがサイトで使用されていると誤って認識される可能性があります。たとえば、ファイルのテキストコンテンツの通常の文では、特定のCSSセレクタに対応する単語が存在する場合があります。

PurifyCSSはこれについて覚えておく価値があります。このツールは、非常に単純なアルゴリズムを使用して、サイトのマテリアルでCSSセレクターを検索します。これは一方では賢明な考えであり、他方では非常に危険です。

UnusedCSSサービス


このツールが各ページを分析するために実行されるツールの手動チューニングは、可能な限りすべての観点からそれを調べたので、もちろん、退屈なルーチンです。さらに、プロジェクトの開発に応じて、サイトに関する分析情報を新しいデータで更新する必要があるため、このような作業は毎日実行する必要があります。UnusedCSSと呼ばれる1つのオンラインサービスがあります。彼は独立してサイトのページをバイパスし、開発者を単調な作業から解放します。このサービスは、単一のリンクをサイトに送信するのに十分です。

このサービスの有料サブスクリプションにサインアップし、css-tricks.comを分析しました。結果を見た後、私は以前に見たものよりもはるかに正確に見えるように感じたと認めなければなりません。


UnusedCSSでcss-tricks.comを分析します。レポートによると、サイトは読み込まれたCSSコードの93％を使用しています。このサイトのすべてのCSSコードを手動で記述したので、これは私には真実に近いようです。この

サービスでは、CSSコードがクリアされたファイルをダウンロードして、このファイルのコンテンツを管理するためのオプションを提供します。たとえば、これは、開発者がCSSコードに追加したいセレクターまたは追加したくないセレクターを有効または無効にすることです。 UnusedCSSによれば、サイトでは必要ないクラス名が開発者に表示されると想定しますが、開発者はこのクラス名なしでは実行できないことを確実に知っています。その結果、誤って不要と認識されたコードに、必要に応じてマークを付けることができます。 UnusedCSSのその他の機能には、プレフィックスの操作、重複するセレクターの削除などがあります。

UnusedCSSを使用すると、他のツールよりもはるかに正確な結果が得られるという事実がとても気に入りました。ただし、このサービスによって生成されたデータには「ノイズ」が多すぎ、プロジェクトを構築して新しいバージョンを本番環境に置くという定期的に繰り返されるプロセスにUnusedCSSを含める方法もまだわかりません。
