Magento 2のコンテンツセキュリティポリシー

みなさん、こんにちは！

確かにMagento 2.3.5をインストールし、ブラウザコンソールで

[レポートのみ]のようなスクリプトを見つけました。スクリプト '***'は、次のコンテンツセキュリティポリシーディレクティブに違反しているため、拒否されました： "script-src *"。「script-src-elem」は明示的に設定されていないため、「script-src」はフォールバックとして使用されます。



カットの下で何をすべきかを教えます

前書き


バージョン2.3.5以降、Magento_CSPと呼ばれるモジュールがMagentoに登場しました。彼はコンテンツセキュリティポリシーの責任者であり、実際にはContent-Security-Policy-Report-Onlyの代わりにヘッダーContent-Security-Policyを追加します。いつものように、「修正」バージョンを上げるときに、予定どおりに、そしてかなり期待されていました :) コンテンツセキュリティポリシーとは何か、およびHabréで何が食べられるかについては、すでに書きました。 ヘッダーContent-Security-PolicyまたはContent-Security-Policy-Report-Onlyの選択は、構成ファイルvendor / magento / module-csp / etc / config.xmlに依存し、フロント部分は個別に、管理部分は個別に異なります。







<config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:noNamespaceSchemaLocation="urn:magento:module:Magento_Store:etc/config.xsd">
    <default>
        <csp>
            <mode>
                <storefront>
                    <report_only>1</report_only>
                </storefront>
                <admin>
                    <report_only>1</report_only>
                </admin>
            </mode>
        </csp>
    </default>
</config>

「report_uri」ディレクティブの同じ場所にレポートのURLを設定できますが、存在しない場合、Chromeはコンソールが存在しないことをメッセージで容赦なく再定義します。

ポリシーにURLを追加するにはどうすればよいですか？


/ etc / moduleフォルダーのルートに、csp_whitelist.xmlという名前とコンテンツでファイルを作成します。

<?xml version="1.0" encoding="UTF-8"?>
<csp_whitelist xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
               xsi:noNamespaceSchemaLocation="urn:magento:module:Magento_Csp:etc/csp_whitelist.xsd">
    <policies>
        <policy id="POLICY_ID">
            <values>
                <value id="VALUE_ID" type="TYPE" algorithm="ALGORITHM">SOME DOMAIN</value>
            </values>
        </policy>
    </policies>
</csp_whitelist>

ここで、POLICY_IDは次のいずれかです。

  • default-src
  • script-src
  • object-src
  • style-src
  • img-src
  • media-src
  • frame-src
  • font-src
  • connect-src

VALUE_ID-任意の一意の名前TYPE-
タイプは、ドメインまたはハッシュの値を取ることができます
ALGORITHM-ハッシュアルゴリズム（TYPE =ハッシュを使用）、たとえばsha256 Magento自体の統合テストの
