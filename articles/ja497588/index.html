Spring Security-BitBucketを介したOAuth2承認Webアプリケーションの例

この記事では、外部認証サーバー（Bitbucketの例を使用）を使用し、OAuth2プロトコルを使用してSpring Boot上のWebアプリケーションでユーザーを認証する方法を検討します。

何を取得したいですか


たとえば、継続的インテグレーションシステムなどの外部サーバー上のユーザーリソースにアクセスできる安全なWebアプリケーションを開発していて、Bitbucketが外部サーバーとして機能し、ユーザーのユーザー名とパスワードを外部システムから保存したくないとします。つまり、アカウントとリソースにアクセスするには、Bitbucketを介してユーザーを承認する必要があります。さらに、彼がアプリケーションのユーザーであることを確認し、ユーザーがBitbucketから資格情報を開示しないようにします。私はこれを行う方法を疑問に思います-猫へようこそ。

OAuth2


OAuth2は、ユーザー（第三者）にユーザー名とパスワードを与えなくても、ユーザーの保護されたリソースへの制限付きアクセスを第三者に提供できるようにする認証プロトコルです。

OAuth2は4つの役割を定義します：

  • リソース所有者
  • リソースサーバー
  • 認可サーバー
  • クライアント（アプリケーション）

リソース所有者は、クライアントアプリケーションを使用して、リソースサーバーでホストされている自分のアカウントへのアクセスを許可するユーザーです。アカウントへのアプリケーションアクセスは、許可された権限によって制限されます。リソースサーバホストの安全なユーザアカウント。許可サーバーは、リソースの所有者を認証し、アクセストークンを発行します。承認サーバーは同時にリソースサーバーにすることもできます。クライアントアプリケーションは、ユーザーのアカウントとリソースにアクセスする必要があるアプリケーションです。 私たちの場合、クライアントアプリケーションは開発中のアプリケーションであり、Bitbucketは承認サーバーとリソースサーバーの両方になります。








OAuth2は、4種類の承認をサポートしています。承認コード、暗黙的、リソース所有者のパスワード資格情報、およびクライアント資格情報です。私たちはそれらすべてを検討するのではなく、認証コードのタイプに関心があります。承認コードタイプは、ソースコードが公開されておらず、クライアントシークレットコードの機密を保持できるサーバーアプリケーション用に最適化されています。このタイプはリダイレクトに基づいて機能します。つまり、ユーザーは自分のIDを確認し、アプリケーションが自分のアカウントを使用できるようにするために、承認サーバーにリダイレクトされます。

認証コードによる認証プロセスは、次の2つのリクエストのシーケンスで構成されています。

  • 承認リクエスト
  • アクセストークンリクエスト

承認リクエストは、ユーザーの身元を確認するために使用されます。また、ユーザーにアプリケーションの承認をリクエストするためにも使用されます。このリクエストは、次のパラメータを含むGETリクエストです。

  • response_type-値はコードと等しい必要があります
  • client_id-クライアントをOAuth2プロバイダーに登録するときに取得される値
  • redirect_uri-認証後にユーザーがリダイレクトされるURL
  • scope-要求されるアクセスレベルを示すオプションのパラメーター
  • 状態-応答を検証するためにランダムに生成された文字列

リクエスト例：

GET https://server.example.com/authorize?response_type=code&client_id=CLIENT_ID&state=xyz&redirect_uri=REDIRECT_URI

ユーザーが自分のIDを確認し、アプリケーションによるリソースへのアクセスを許可すると、ユーザーエージェントは、リクエストで渡された値を持つ承認コードと状態パラメーターを含む追加のパラメーターコードでクライアントの登録中に指定されたコールバックURLにリダイレクトされます。

コールバックの例：

GET https://client.example.com/cb?code=AUTH_CODE_HERE&state=xyz

アクセスコード要求は、受信した認証コードをユーザーリソースへのアクセスコードに交換するために使用されます。このリクエストは、次のパラメータを含むPOSTリクエストです。

  • grant_type-値はauthorization_codeである必要があります
  • コード-前のステップで取得した認証コード
  • redirect_uri-前のステップで指定されたURLと一致する必要があります
  • client_id-クライアントをOAuth2プロバイダーに登録するときに取得される値
  • client_secret-OAuth2プロバイダーにクライアントを登録するときに取得される値

リクエスト例：

POST https://server.example.com/token
    grant_type=authorization_code&
    code=AUTH_CODE&
    redirect_uri=REDIRECT_URI&
    client_id=CLIENT_ID&
    client_secret=CLIENT_SECRET

サーバーの応答には、アクセスコードとその有効期間が含まれます。

{
    "access_token": "2YotnFZFEjr1zCsicMWpAA",
    "expires_in": 3600,
    "refresh_token": "tGzv3JOkF0XG5Qx2TlKWIA"
}

このプロセス全体はSpring Securityですでに自動化されており、その実装について心配する必要はありません。

Spring Securityを構成する


次に、Spring Securityを設定します。OAuth2を機能させるには、ClientRegistrationオブジェクトを作成する必要があります。ClientRegistrationは、OAuth2プロバイダーに登録されたクライアントに関する情報を格納します。ここでは、前のステップで取得したclient_idとclient_secretが必要です。一般に、そのようなClientRegistrationオブジェクトがいくつか存在する可能性があるため、Spring SecurityはClientRegistrationRepositoryオブジェクトを使用して、それらを保存およびアクセスします。それも作成しましょう。また、承認されたユーザーのみがリクエストを呼び出し、その実装でUserServiceを再定義できることも示しています。

SecurityConfig.java
@Configuration
public class SecurityConfig {

    @Value("${client_id}")
    private String clientId;

    @Value("${client_secret}")
    private String clientSecret;

    @Bean
    public ClientRegistration clientRegistration() {
        return ClientRegistration
                .withRegistrationId("bitbucket")
                .clientId(clientId)
                .clientSecret(clientSecret)
                .userNameAttributeName("username")
                .clientAuthenticationMethod(BASIC)
                .authorizationGrantType(AUTHORIZATION_CODE)
                .userInfoUri("https://api.bitbucket.org/2.0/user")
                .tokenUri("https://bitbucket.org/site/oauth2/access_token")
                .authorizationUri("https://bitbucket.org/site/oauth2/authorize")
                .redirectUriTemplate("{baseUrl}/login/oauth2/code/{registrationId}")
                .build();
    }

    @Bean
    @Autowired
    public MyOAuth2UserService oAuth2userService(UserService userService) {
        return new MyOAuth2UserService(userService);
    }

    @Bean
    @Autowired
    public ClientRegistrationRepository clientRegistrationRepository(List<ClientRegistration> registrations) {
        return new InMemoryClientRegistrationRepository(registrations);
    }

    @Configuration
    @EnableWebSecurity
    public static class AuthConfig extends WebSecurityConfigurerAdapter {

        private final MyOAuth2UserService userService;

        @Autowired
        public AuthConfig(MyOAuth2UserService userService) {
            this.userService = userService;
        }

        @Override
        protected void configure(HttpSecurity http) throws Exception {
            http
                    .authorizeRequests(authorizeRequests -> authorizeRequests
                            .anyRequest().authenticated()
                    )
                    .oauth2Login(oauth2Login -> oauth2Login
                            .userInfoEndpoint(userInfoEndpoint -> userInfoEndpoint.userService(userService))
                    );
        }
    }
}


カスタマイズUserService


Spring Securityは認証プロセスを完全に実装するだけでなく、それをカスタマイズする機能も提供します。例えば、カスタマイズの可能性認証要求、アクセスコード要求、並びにのカスタム後処理の可能なアクセスコード要求に対する応答を。承認が成功すると、Spring SecurityはUserInfo Endpointを使用して、承認サーバーからユーザー属性を取得します。特に、OAuth2UserServiceインターフェースの実装がこれに使用されます。

このサービスの独自の実装を作成します。これにより、認証サーバーでのユーザー認証の後に、彼がアプリケーションのユーザーであるかどうかを確認するか、登録がすべてのユーザーに公開されている場合は登録します。デフォルトでは、Spring SecurityはDefaultOAuth2UserServiceの実装を使用します。彼は私たちの実装の基礎を形成します。

MyOAuth2UserService.java
public class MyOAuth2UserService implements OAuth2UserService<OAuth2UserRequest, OAuth2User> {

    private static final String MISSING_USER_INFO_URI_ERROR_CODE = "missing_user_info_uri";

    private static final String INVALID_USER_INFO_RESPONSE_ERROR_CODE = "invalid_user_info_response";

    private static final String MISSING_USER_NAME_ATTRIBUTE_ERROR_CODE = "missing_user_name_attribute";

    private static final ParameterizedTypeReference<Map<String, Object>> PARAMETERIZED_RESPONSE_TYPE = new ParameterizedTypeReference<Map<String, Object>>() {
    };

    private final UserService userService;

    private final RestOperations restOperations;

    private final Converter<OAuth2UserRequest, RequestEntity<?>> requestEntityConverter = new OAuth2UserRequestEntityConverter();

    public MyOAuth2UserService(UserService userService) {
        this.userService = requireNonNull(userService);
        this.restOperations = createRestTemplate();
    }

    @Override
    public OAuth2User loadUser(OAuth2UserRequest userRequest) throws OAuth2AuthenticationException {
        checkNotNull(userRequest, "userRequest cannot be null");
        if (!StringUtils.hasText(userRequest.getClientRegistration().getProviderDetails().getUserInfoEndpoint().getUri())) {
            OAuth2Error oauth2Error = new OAuth2Error(MISSING_USER_INFO_URI_ERROR_CODE, "Missing required UserInfo Uri in UserInfoEndpoint for Client Registration: " + userRequest.getClientRegistration().getRegistrationId(), null);
            throw new OAuth2AuthenticationException(oauth2Error, oauth2Error.toString());
        }

        String registrationId = userRequest.getClientRegistration().getRegistrationId();
        String userNameAttributeName = userRequest.getClientRegistration().getProviderDetails().getUserInfoEndpoint().getUserNameAttributeName();
        if (!StringUtils.hasText(userNameAttributeName)) {
            OAuth2Error oauth2Error = new OAuth2Error(
                    MISSING_USER_NAME_ATTRIBUTE_ERROR_CODE,
                    "Missing required \"user name\" attribute name in UserInfoEndpoint for Client Registration: " + registrationId, null);
            throw new OAuth2AuthenticationException(oauth2Error, oauth2Error.toString());
        }

        ResponseEntity<Map<String, Object>> response;
        try {
            // OAuth2UserRequestEntityConverter cannot return null values.
            //noinspection ConstantConditions
            response = this.restOperations.exchange(requestEntityConverter.convert(userRequest), PARAMETERIZED_RESPONSE_TYPE);
        } catch (OAuth2AuthorizationException ex) {
            OAuth2Error oauth2Error = ex.getError();
            StringBuilder errorDetails = new StringBuilder();
            errorDetails.append("Error details: [");
            errorDetails.append("UserInfo Uri: ").append(
                    userRequest.getClientRegistration().getProviderDetails().getUserInfoEndpoint().getUri());
            errorDetails.append(", Error Code: ").append(oauth2Error.getErrorCode());
            if (oauth2Error.getDescription() != null) {
                errorDetails.append(", Error Description: ").append(oauth2Error.getDescription());
            }
            errorDetails.append("]");
            oauth2Error = new OAuth2Error(INVALID_USER_INFO_RESPONSE_ERROR_CODE,
                    "An error occurred while attempting to retrieve the UserInfo Resource: " + errorDetails.toString(), null);
            throw new OAuth2AuthenticationException(oauth2Error, oauth2Error.toString(), ex);
        } catch (RestClientException ex) {
            OAuth2Error oauth2Error = new OAuth2Error(INVALID_USER_INFO_RESPONSE_ERROR_CODE,
                    "An error occurred while attempting to retrieve the UserInfo Resource: " + ex.getMessage(), null);
            throw new OAuth2AuthenticationException(oauth2Error, oauth2Error.toString(), ex);
        }

        Map<String, Object> userAttributes = emptyIfNull(response.getBody());

        Set<GrantedAuthority> authorities = new LinkedHashSet<>();
        authorities.add(new OAuth2UserAuthority(userAttributes));

        for (String authority : userRequest.getAccessToken().getScopes()) {
            authorities.add(new SimpleGrantedAuthority("SCOPE_" + authority));
        }

        //     ,   
        //          ,
        //    
        User user = findOrCreate(userAttributes);
        userAttributes.put(MyOAuth2User.ID_ATTR, user.getId());
        return new MyOAuth2User(userNameAttributeName, userAttributes, authorities);
    }

    private User findOrCreate(Map<String, Object> userAttributes) {
        String login = (String) userAttributes.get("username");
        String username = (String) userAttributes.get("display_name");

        Optional<User> userOpt = userService.findByLogin(login);
        if (!userOpt.isPresent()) {
            User user = new User();
            user.setLogin(login);
            user.setName(username);
            return userService.create(user);
        }
        return userOpt.get();
    }

    private RestTemplate createRestTemplate() {
        RestTemplate restTemplate = new RestTemplate();
        restTemplate.setErrorHandler(new OAuth2ErrorResponseErrorHandler());
        return restTemplate;
    }
}


テストのエンドポイント


ここで、先ほど実行した処理の正常性をテストするためのエンドポイントを作成します。エンドポイントは、現在のユーザーを歓迎する1つの要求のみで構成されます。

WelcomeEndpoint.java
@Path("/")
public class WelcomeEndpoint {

    @Autowired
    private UserService userService;

    @GET
    public String welcome() {
        User currentUser = getCurrentUser();
        return String.format("Welcome, %s! (user id: %s, user login: %s)",
                currentUser.getName(), currentUser.getId(), currentUser.getLogin());
    }

    public User getCurrentUser() {
        return userService.findByLogin(getAuthenticatedUser().getName()).orElseThrow(() -> new RuntimeException("No user logged in."));
    }

    private Authentication getAuthentication() {
        return SecurityContextHolder.getContext().getAuthentication();
    }

    private MyOAuth2User getAuthenticatedUser() {
        return (MyOAuth2User) getAuthentication().getPrincipal();
    }
}
