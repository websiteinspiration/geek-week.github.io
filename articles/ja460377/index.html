Liquibaseを使用してSpring Bootアプリケーションでデータベース構造を管理する。パート1

この記事では、Spring BootアプリケーションでLiquibaseユーティリティを使用してリレーショナルデータベースの構造をバージョン管理し、この構造をあるバージョンから別のバージョンに移行する方法について説明します。前半では基本的な例を分析し、後半ではliquibase-mave-pluginを使用して変更をロールバックし、データベース構造を比較してスクリプトを自動的に生成する方法について説明します。

Spring Boot + JPA（Hibernate）で最も単純なアプリケーションを作成することから始めましょう。Spring Initializrがこれを支援します。依存関係から、JPA、MySQL、Webを選択します。Liquibaseもこのステップで接続できますが、理解を深めるために、後で手動で接続します。

アプリケーションの基礎を作成する


アプリケーションにエンティティークラスを追加し、それを操作するためのリポジトリーとRESTコントローラーも追加します。具体的には、作成したエンティティにユーザー情報を格納します。

@Entity
@Table(name = "users")
public class User implements Serializable {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "id")
    private Long id;

    @Column(name = "username", unique = true, nullable = false)
    private String userName;

    @Column(name = "password", nullable = false)
    private String password;

    @Column(name = "first_name")
    private String firstName;

    @Column(name = "last_name")
    private String lastName;

    @Column(name = "email")
    private String email;

    //    ,   
    //    Lombok
}

Spring Dataはリポジトリコードを非常に簡潔にします

public interface UserRepository extends JpaRepository<User, Long> {
}

ユーザーテーブルの内容全体を表示するRESTコントローラ

@RestController
public class UserController {
    
    private UserRepository userRepository;

    @Autowired
    public UserController(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    @GetMapping("/user/all")
    public List<User> allUsers() {
        return userRepository.findAll();
    }
}

application.properties ファイルの設定

spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://localhost:3306/geek_db?createDatabaseIfNotExist=true&allowPublicKeyRetrieval=true&useSSL=false&useUnicode=true&characterEncoding=UTF-8&serverTimezone=UTC
spring.datasource.username=dbuser
spring.datasource.password=dbpassword

spring.jpa.show-sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQL5Dialect

コンピューターがMySQLサーバーを標準ポートで実行していることを前提としています。必要に応じて、接続文字列のサーバーURLと、ユーザー名とパスワードを調整します。createDatabaseIfNotExistパラメータにも注目する価値があります。おかげで、接続時に、サーバー上にない場合はgeek_dbというデータベースが作成されます。

Liquibaseを追加


確かに、Hibernateの設定の1つ、つまり、spring.jpa.hibernate.ddl-autoが欠落していることに気づきました。ほとんどの初心者向けマニュアルでは、更新値が示されています。これにより、Hibernateは、プロジェクトに存在するエンティティークラスに基づいて、サーバー自体でテーブル構造を作成および調整します。この方法は、データスキームが非常に単純な場合やプロジェクトがトレーニング中の場合に使用できますが、多少複雑なスキームでは、Hibernate DDLスクリプトの生成プロセスを制御できないために問題が発生する可能性が高くなります。別の問題は、このアプローチでは、データベース構造でHibernateに加えられた変更を簡単にロールバックできないことです。

Liquibase ユーティリティを使用するのは、上記の問題を解決するためです。幸いなことに、彼女は完全にSpring Bootアプリケーションと統合することができます。それを使用して起動するには、次の手順を実行しなければならない

追加の設定をapplication.propertiesのファイル

spring.jpa.hibernate.ddl-auto=none

これは、Hibernateが回路を変更するアクションを実行しないようにするためです。Liquibaseは今それらを行います。理論的には、ここでは検証値を使用して、テーブル構造の正確さをさらに制御することもできます。pom.xmlに依存関係

を追加する

<dependency>
    <groupId>org.liquibase</groupId>
    <artifactId>liquibase-core</artifactId>
</dependency>

追加後、Springブートはliquibaseという名前の特別なBeanを自動的に作成します。これにより、アプリケーションが起動するたびに、Liquibaseスクリプトに基づいてデータベーススキーマを設定するためのすべてのアクションが実行されます。

次に、必要なテーブルを作成するLiquibaseスクリプト自体を追加する必要があります。 / src / main / resources / db / changelogフォルダーにdb.changelog-master.yamlというファイルを作成し、それに次のコンテンツを追加します

databaseChangeLog:
  - logicalFilePath: db/changelog/db.changelog-lesson1.yaml
  - changeSet:
      id: 1
      author: your_liquibase_username
      changes:
        - createTable:
            tableName: users
            columns:
              - column:
                  name: id
                  type: BIGINT
                  autoIncrement: true
                  constraints:
                    primaryKey: true
                    nullable: false
              - column:
                  name: username
                  type: varchar(50)
                  constraints:
                    unique: true
                    nullable: false
              - column:
                  name: password
                  type: varchar(512)
                  constraints:
                    nullable: false
              - column:
                  name: first_name
                  type: varchar(50)
              - column:
                  name: last_name
                  type: varchar(50)
              - column:
                  name: email
                  type: varchar(50)

このスクリプトの内容を分析してみましょう。まず、1つのchangeSetが含まれています。 ChangeSetは、GitやSVNなどのバージョン管理システムのコミットに類似しています。コミットと同様に、1つのchangeSetの一部として行われた変更は、データベースサーバーにロールバックまたはロールバックできます。各changeSetには、特定のchangeSetがこのデータベースにポンプされているかどうかをLiquibaseが決定する一意の識別子が必要です。

ChangeSetにはテーブルを作成するコマンドが含まれており、テーブルの構造はSQLスクリプトではなくLiquibaseによって記述されます。このおかげで、このファイルはクロスプラットフォームになります。 Liquibaseは、使用するデータベースサーバーに応じてSQLスクリプトを生成します。さらに、このchangeSetをロールバックする必要がある場合、Liquibaseはこのテーブルを削除するスクリプトを自動的に作成できます。 SQLスクリプトを使用した場合、変更をロールバックするスクリプトを手動で作成する必要があります。hangesセクションでは、チームは1つだけです。これは、1つのchangeSetにいくつでもチームを置くことができるという事実にもかかわらず、良い習慣と見なされています。

書かれたコードはプログラムを実行するのに十分ですが、その作業の結果をより明確に見るために、テーブルをデータで満たす別のchangeSetを追加しましょう。

  - changeSet:
      id: 2
      author: your_liquibase_username
      comment: "Create admin user"
      changes:
        - insert:
            tableName: users
            columns:
              - column:
                  name: username
                  value: "admin"
              - column:
                  name: password
                  value: "admin"
              - column:
                  name: email
                  value: "admin@server.com"
        - insert:
            tableName: users
            columns:
              - column:
                  name: username
                  value: "guest"
              - column:
                  name: password
                  value: "guest"
              - column:
                  name: email
                  value: "guest@server.com"
      rollback:
        - delete:
            tableName: users
            where: username in ('admin', 'guest')

この場合、次のように、ロールバック操作用のブロックを手動で書き込む必要があります。Liquibaseは、データの操作時にロールバックSQLを自動的に作成できません。一般に、データベース内のデータの操作はLiquibaseの主要な機能の1つではなく、挿入、削除、または変更という最も単純な操作に限定されます。ちなみに、もっと必要な場合は、ここでRed Gate社のツールを使用できます。

それでは、アプリケーションを実行して、リンクhttp：// localhost：8080 / user / allに移動してみましょう。アプリケーションが起動すると、テーブルに追加された2人のユーザーに関する情報を含むJSON応答が表示されます。また、データベースを初期化するためにLiquibaseが実行するスクリプトを確認できるアプリケーション起動ログを確認することも価値があります。DATABASECHANGELOGテーブルには特に注意を払う必要があります。 Liquibaseがデータベースに加えられた変更のログを保存するのはそのためです。

それは今のところすべてです。しばらくしてから、liquibase-maven-pluginを使用して、データベース構造を比較し、加えられた変更をロールバックしてスクリプトを自動生成することについての続きを公開する予定です。
