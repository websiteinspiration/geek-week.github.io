SwiftのAnyおよびAnyObject。それらの違いは何ですか？

私のプロジェクトでかなり長い間、たとえばJSONデータを処理するときなど、Any型を使用したときは、しかし、2つ目のタイプとしてAnyObjectがあることも知っていました。そして最近、私はこれら2つのタイプの違いについて考えていました。

Appleのドキュメントによると：

  • Any-任意のタイプのインスタンスを表すことができます
  • AnyObject-任意のクラスのインスタンスを表すことができます

少し簡単に言うと、次のようになります。

  • Anyはすべてのタイプに使用されます。
  • AnyObject-クラスタイプに使用

実際にこれら2つのタイプを確認してみましょう。タイプAnyから始めましょう。

これを行うには、Any型の配列を作成して印刷します。

let anyArray: [Any] = ["Macbook", 1, 2]
print(anyArray)
Console: ["Macbook", 1, 2]

ご覧のとおり、Anyを使用すると、さまざまなタイプのデータを同時に処理できます（String、Int）。
ドキュメントによると、この配列の要素（StringおよびInt）は値のタイプである構造体であるため、理論的にはAnyObjectは機能しません。

これを確認するには、AnyObject型の同じ配列を作成します。

let anyObjectArray: [AnyObject] = ["Macbook", 1, 2]

予想通り、コンパイラーは「String / Int」型をAnyObject型に変換できないことについてエラーを表示します

コンパイラエラー
Cannot convert value of type «Int» to expected element type «AnyObject»
Cannot convert value of type «Int» to expected element type «AnyObject»
Cannot convert value of type «String» to expected element type «AnyObject»

しかし、3つのタイプをAnyObjectに持ってきて、結果を出力してみましょう。

let anyObjectArray: [AnyObject] = ["Macbook" as AnyObject, 1 as AnyObject, 2 as AnyObject]
print(anyObjectArray)
Console: [Macbook, 1, 2]

コンパイラエラーがなくなりました。ご覧のとおり、Macbookの文字列は明らかに文字列のように見えますが、SwiftのString型のような通常の引用符はありません。

ループを使用して配列を出力して、実際の型を確認してみましょう。

for item in anyObjectArray {
    if item is String {
        print("\(item)   String")
    } else if item is Int {
        print("\(item)   Int")
    }
}

Console: 
Macbook   String
1   Int
2   Int

文字列は文字列型です。前述したように、Swiftの文字列は構造体であり、クラスのタイプではありません。したがって、それらをAnyObjectとして使用することはできません。

アレイでさらにいくつかの実験を行ってみましょう。Objective ‑ C：NSStringとNSNumberの型をチェックしてみましょう。

for item in anyObjectArray {
    if item is NSString {
        print("\(item)   NSString")
    } else if item is NSNumber {
        print("\(item)   NSNumber")
    }
}

Console: 
Macbook   NSString
1   NSNumber
2   NSNumber

では、なぜこれが起こっているのでしょうか？


Objective ‑ Cとの互換性の一環として、SwiftはCocoaプラットフォームを操作するための便利で効率的な方法を提供します。

Swiftは、一部のObjective ‑ CタイプをSwiftタイプに自動的に変換し、一部のSwiftタイプをObjective ‑ Cタイプに自動的に変換します。Objective ‑ CとSwiftの間で変換できる型は、連結と呼ばれます。

言い換えると、コンパイラーは、「ブリッジ」を自動的に変換および作成すると同時に、アプリケーションのクラッシュを防止することで、これらのタイプを柔軟に処理できるようにあらゆることを行います。

AnyObjectを使用する場合


Appleのドキュメントに記載されているように、AnyObjectを使用して、Classから派生したオブジェクトを操作できますが、共通のルートクラスはありません。

Swift 3では、Objective ‑ Cのid型がSwiftのAny型にマップされ、クラス、列挙、構造、またはその他のSwift型など、任意の型の値を記述します。この変更により、SwiftでObjective ‑ C APIがより柔軟になります。これは、Swiftで定義された値型をObjective ‑ C APIに渡し、Swift型として取得できるため、手動の「ブロック」型が不要になるためです。

したがって、クラスでのみ使用できるようにプロトコルを制限する場合は 、AnyObject を使用することをお勧めします。任意の他のケースでは。

Appleが追加： Anyと AnyObjectは、それらが提供する動作と機能を明示的に必要とする場合にのみ
使用してください。コードで使用することが予想される型については、常に正確であることが最善です。
