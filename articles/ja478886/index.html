インラインコンテンツを含む角度コンポーネント

Angularで開発するときに新しいコンポーネントを作成するときは、何度も再利用できるようにコンポーネントを作成するよう努めます。ソフトウェア開発の場合と同様に、プログラマーは自分のコードをできるだけ再利用可能にするよう努めています。同時に、柔軟性はあるが複雑すぎないコンポーネントが欲しいです。



原則として、コンポーネントの保守とテストを容易にするために、コンポーネントはできるだけ愚かにする必要があります。しかし、同時に、それらはカスタマイズ可能かつ柔軟であるため、何度も再利用でき、視覚的および機能的に類似しているが、まだ何かが異なるコンポーネントのクローンを作成する必要はありません。または、コンポーネントの動作を有効または無効にするパラメーターに基づいて、コンポーネントに多数のパラメーターを追加しないでください。実際には、100ケースに1ケースで使用されます。

乗組員の生命を危険にさらすことなく青い海の底に沈む潜水艦を作成し、青空を飛ぶ前にそれを修正することは簡単な作業ではありません。

角度コンポーネントは、1つの単純なタスクだけを実行する必要があるユーザーインターフェイスの最小単位です。あらゆる場面でコンポーネントを作成することは悪い習慣です。したがって、柔軟性とシンプルさの間の妥協点を見つける必要があります。

ある種のラッパーで構成されるコンポーネントを作成するとします。その中には、ヘッダーとある種のスイッチがあり、その下にコンテンツがあります。このようなもの：



ウィジェットと呼ばれるコンポーネント全体が緑色で概説されています。ヘッダースイッチは赤で強調表示され、コンテンツは青で強調表示されます。 Angularは信じられないほど強力なフレームワークです。図のようなコンポーネントを作成することは、Angularにとって非常に簡単な作業です。

しかし、たとえば、このコンポーネントをより柔軟にして、ウィジェットコンポーネントのパラメーターに応じてコンテンツが変更されるとどうなるでしょうか。

たとえばngIf構造ディレクティブを使用して、ウィジェットパラメータに基づいてコンテンツの外部表現を解決できます。異なるビューを切り替えることができます。提出が多すぎる場合は、別のngSwitch構造ディレクティブを使用できます。

そして、異なるビューがユーザーのアクションに異なる応答をする場合はどうでしょうか？その場合、コンポーネントコードには多数の条件ステートメントも含まれます。しかし、多くの条件付きステートメントがあるコードは、時間の経過とともに維持、バグの修正、または機能の拡張が困難になります...

ウィジェットコンポーネントを作成してコンテンツを受け取ることができますが、同時にフィルターパネルの選択した値をコンテンツ領域のコンポーネントにスローします。また、ウィジェットコンポーネント自体に影響を与えることなく、ウィジェットの親コンポーネントからコンテンツにパラメーターを転送できるようにしたいと考えています。この問題を解決するには、コンポーネントの視覚的表現を「解決」できるすばらしいngTemplateOutletディレクティブを使用できます。追加情報については、ドキュメンテーションを参照できますが、私の見解では、そのすべての機能は示されていません。

この場合、ウィジェットコンポーネントは次のように表すことができます。

// widget.component.ts
…
private widgetContentTemplate: TemplateRef<any>;

//     
@ContentChild(WidgetContentTemplateDirective, { read: TemplateRef })
set WidgetContentTemplate(value: TemplateRef<any>) {
	this.widgetContentTemplate = value;
}

get WidgetContentTemplate(): TemplateRef<any> {
	return this.widgetContentTemplate;
}
…


<!-- widget.component.html -->
…
 <div class="widget__content">
	<!--      ,  
	   context.   ,
	    -->
	<ng-container *ngTemplateOutlet="WidgetContentTemplate || DefaultWidgetContentTemplate;
		context: {
			post: SelectedPost
		}">
	</ng-container>
</div>

<ng-template #DefaultWidgetContentTemplate>
  <div>[    ]</div>
</ng-template>
…

ngTemplateOutletディレクティブは、@ContentChildを使用してコンポーネントで読み取られるTemplateRefオブジェクトを受け入れます。これを行うには、単純なディレクティブを作成します。これが彼女のコードです：

// widget-content-template.directive.ts
import { Directive, TemplateRef } from '@angular/core';

@Directive({
    selector: '[appWidgetContentTemplate]'
})
export class WidgetContentTemplateDirective {
    get Template(): TemplateRef<any> {
        return this.template;
    }

    constructor(private readonly template: TemplateRef<any>) {}
}

ウィジェットスイッチからのパラメーターは、コンテキストオブジェクトを使用してコンテンツに渡すことができます。コンテキストからのパラメーターがコンポーネントによって正しく読み取られるようにするには、親コンポーネント（この例ではAppComponent）で必要です。次のように記述することを忘れないでください。

<!-- app.component.html -->
<app-widget-component Title="  (  )">
    <!-- let-post="post" ,         -->
    <ng-template appWidgetContentTemplate let-post="post">
        <app-post-view [Post]="post"></app-post-view>
    </ng-template>
</app-widget-component>
<br>
<br>
<app-widget-component Title="  (  )">
    <!--  let-post="post"     ,  ShowPostInfo  AppComponent -->
    <ng-template appWidgetContentTemplate let-post="post">
        <app-post-view [Post]="post" [ShowPostInfo]="ShowPostInfo"></app-post-view>
    </ng-template>
</app-widget-component>

したがって、コンテンツを受け入れて表示するウィジェットコンポーネントがありますが、ウィジェットコンポーネント自体はコンテンツについて何も知る必要はありません。スイッチパラメータを転送するだけです。それらを読むかどうかは、内容の問題です。
