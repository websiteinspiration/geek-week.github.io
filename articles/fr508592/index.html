Ce que les développeurs JavaScript doivent savoir sur Curl





Bonjour mes amis!



Je vous présente la traduction de l'article «Ce que les développeurs JavaScript doivent savoir sur Curl» de Valery Karpov.



Curl est un outil de ligne de commande populaire souvent utilisé pour envoyer des requêtes HTTP. Curl prend en charge un grand nombre de protocoles, cependant, en tant que développeur Node.js, vous l'utiliserez très probablement pour envoyer des requêtes http à l'API RESTful.



La documentation de curl est une liste d'indicateurs de ligne de commande 383, il est donc très difficile de trouver ce que vous recherchez. Dans cet article, je souhaite partager avec vous certains des modèles que j'utilise souvent. Par exemple, le service httpbin.org sera utilisé .

Télécharger des fichiers



Wget est l'outil le plus courant pour télécharger des fichiers via la ligne de commande. Il est inclus avec la plupart des distributions Linux. Cependant, sous OSX, ce n'est pas le cas.



La commande est wget url



similaire à la commande curl -OL url



. Une option -



est une option --remote-name



qui indique à curl d'enregistrer le corps de la réponse dans un fichier local. L'option -L



indique à curl de suivre les redirections.



Par exemple, ci-dessous est une image d'Unsplash, son URL est https://images.unsplash.com/photo-1506812574058-fc75fa93fead



.







Pour télécharger cette image, procédez comme suit: 



$ curl -OL https://images.unsplash.com/photo-1506812574058-fc75fa93fead
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
                                 Dload  Upload   Total   Spent    Left  Speed
100 12.1M  100 12.1M    0     0  3927k      0  0:00:03  0:00:03 --:--:-- 3927k


L'option -O



indique à curl d'utiliser la ligne après la dernière /



comme nom de fichier. Dans l'exemple ci-dessus, l'image sera enregistrée dans le répertoire courant avec le nom photo-1506812574058-fc75fa93fead



. Pour déterminer le nom du fichier, utilisez l'option -



(lettre minuscule "o"). 



$ curl -o miami-beach.jpg https://images.unsplash.com/photo-1506812574058-fc75fa93fead
  % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
                                 Dload  Upload   Total   Spent    Left  Speed

$ ls -l miami-beach-jpg

Soumettre une demande autorisée



L'en-tête d'autorisation est utilisé pour inclure les données d'autorisation dans la demande lors de l'accès à l'API RESTful. Pour ajouter les données spécifiées, vous devez utiliser l'indicateur -H



. Par exemple, si votre clé d'interface est (clé API) my-secret-token



, vous pouvez l'inclure dans la demande http comme suit: 



$ curl -H "Authorization: my-secret-token" https://httpbin.org/get
{
  "args": {}, 
  "headers": {
    "Accept": "*/*", 
    "Authorization": "my-secret-token", 
    "Host": "httpbin.org", 
    "User-Agent": "curl/7.58.0", 
    "X-Amzn-Trace-Id": "Root=1-5ee8e1a5-a3aa30e0765a7980b04ca4a0"
  }, 
  "origin": "69.84.111.39", 
  "url": "https://httpbin.org/get"
}


Notez qu'il httpbin.org



renvoie les en-têtes de la demande http dans le corps de réponse de la propriété headers



.



Curl prend également en charge l'autorisation par défaut via un indicateur -u



. Dans l'exemple suivant, nous envoyons une demande avec un nom d'utilisateur user



et un mot de passe pass



: 



$ curl -i -u "user:pass" https://httpbin.org/basic-auth/user/pass
HTTP/2 200 
date: Tue, 16 Jun 2020 15:18:45 GMT
content-type: application/json
content-length: 47
server: istio-envoy
access-control-allow-origin: *
access-control-allow-credentials: true
x-envoy-upstream-service-time: 1

{
  "authenticated": true, 
  "user": "user"
}


Voici ce qui se passe lors de l'envoi d'un nom d'utilisateur ou d'un mot de passe incorrect: 



$ curl -i -u "user:wrongpass" https://httpbin.org/basic-auth/user/pass
HTTP/2 401 
date: Tue, 16 Jun 2020 15:18:51 GMT

www-authenticate: Basic realm="Fake Realm"

access-control-allow-credentials: true
x-envoy-upstream-service-time: 12

Conclusion



En résumé, voici les options de boucles que je trouve les plus utiles:



  • -X



    définit une méthode de demande, par ex. curl -X POST url



  • -d



    définit le corps de la requête comme une chaîne dans les requêtes PUT et POST. Utiliser @



    pour extraire des données d'un fichier
  • -H



    définit l'en-tête de la demande, par exemple curl -H "Authorization: my-secret-token" url



  • -u



    données d'authentification pour l'autorisation standard
  • -O



    enregistre le corps de la demande dans un fichier
  • -i



    affiche la réponse complète, y compris les en-têtes


Curl est un outil utile pour interagir avec l'API via la ligne de commande, qu'il s'agisse d'une API tierce ou d'une API que vous développez. Pour des tests rapides, curl est meilleur qu'Axios dans Node.js ou l'optimisation des requêtes dans Postman si vous connaissez leur syntaxe.
