由于日历客户端格式的桌面客户端（Microsoft Outlook，Mozilla Thunderbird等）和电子邮件Web应用程序以及Android和Apple智能手机与日历服务同步，因此可以方便地以日历事件格式组织小组的课程表，并会在即将发生的事件之前提醒学生和讲师。

选择视频会议系统（网络会议）


您不应该依靠的是：

  • 全球视频通信和协作服务已无法应对指数级增长的负载；
  • 用于视频通信的共享软件解决方案在自由工作时间，可用功能，虚拟房间的数量及其容量方面受到限制。

给大学的IT专家的建议：

  • 在其设施中部署视频通信解决方案，这些解决方案是从释放其他IT系统占用的计算资源中获得的；如有必要，请使用区域云服务提供商的资源；
  • 在发达社区中使用经过验证的开源系统；
  • 考虑可与浏览器，网络摄像头和数字转换器配合使用的可扩展视频通信解决方案。

符合条件的系统数量不多：

  • Jitsi Meet [ https://jitsi.org/jitsi-meet ];
  • Apache OpenMeeting [ https://openmeetings.apache.org ];
  • BigBlueButton [ https://bigbluebutton.org ]。



在功能上，系统是相同的：

  • 可扩展
  • 易于学习和使用；
  • 它们使您可以共享内容，以结合演讲内容和从演讲者的网络摄像头广播视频以创建口头交流渠道的方式进行工作；
  • 允许您录制讲座并在Internet上发布；
  • 如果您需要50多人参加广播，最好创建一个YouTube讲座流。

您可以在会议服务[ https://meet.n-kraft.ru ]中测试解决方案的功能。基于Jitsi Meet创建。



要存储和发布讲义，请使用WebDav存储库：

  • OwnCloud [ https://owncloud.org ];
  • NextCloud [ https://nextcloud.com ]。

讲师工作场所的组织


讲师的典型工作场所是带有内置或外部USB摄像头的PC或笔记本电脑，智能手机的耳机或lantronics[ https://www.plantronics.com/en/ru/product/blackwire-3200 ]或Logitech [ https： //www.logitech.com/en-us/headsets ]。 PC上应安装一套基本的办公软件，我们建议OnlyOffice [ https://www.onlyoffice.com ]。

我们建议使用Logitech C920s [ https://www.logitech.com/ru-ru/product/hd-pro-webcam-c920s ]或Logitech C930 [ https://www.logitech.com/en作为个人网络摄像头-ru / product / c930e-webcam ]。



如果您需要用于演讲厅的网络摄像头，建议您考虑使用Logitech Group [ https://www.logitech.com/ru-ru/product/conferencecam-group ]或Logitech Rally [ https：//www.logitech。 com / ru-ru / product / rally-ultra-hd-conferencecam ]。如果选择了Logitech Rally，则两个系统都将通过PTZ摄像机和Logitech Group扬声器电话或麦克风和扬声器提供出色的音频和视频信号质量。



如果讲师需要电子白板，即手写水工具，我们建议将Intuos Pro Paper Edition系列[ http://www.wacom.ru/products/pen-tablets/intuos-pro-paper-edition.html ] 的Wacom数字化仪与Wacom Inkspace应用程序一起使用（适用于Windows 7 / 8/10，Android和IOS）[https://www.wacom.com/ru-ru/products/apps-services/inkspace ]。此应用程序的功能之一-实时模式，提供了将手写内容从设备广播到PC显示器的功能，可以在视频会话中发布手写内容。

如果需要功能板，但不需要共享它，则最好的解决方案是OpenBoard [ https://www.openboard.ch ]。



如果您需要共享板，我们建议使用白板服务[ https://meet.n-kraft.ru/whiteboard/ ]或在您的资源上部署类似的服务[ https://github.com/cracker0dks/whiteboard ]。



使用URL请求中的参数，您可以创建唯一的协作板，并区分在同一板上同时进行编辑的参与者。

在访问董事会的请求中，您需要指定其标识符以及用户名，例如https://meet.n-kraft.ru/whiteboard/?。whiteboardid = nkraft和用户名 = IvanIvanov

URL参数：

  • whiteboardid-所有连接到具有相同标识符的电路板都可以共享它；
  • 用户名 - 用户名在编辑过程中会显示给其他人。
