Jaringan DOS menggunakan tumpukan mTCP

Beberapa waktu lalu saya diminta membuat instruksi video tentang pengaturan jaringan di sistem operasi DOS. Sayangnya, saya tidak pandai merekam video, jadi saya akan mencoba menuliskan proses ini sedetail mungkin (dan dengan gambar).

Suatu ketika, saya sendiri harus mengatasi masalah ini, karena untuk pengumpulan komputer di rumah saya proses pertukaran file menggunakan server FTP lokal telah dibuat. (Jika ini menarik, saya dapat memberi tahu Anda lebih banyak tentang ini di salah satu artikel berikut)


pengantar


Jadi, mari kita pergi sedikit dari jauh: dari kartu jaringan. Sayangnya, tidak setiap kartu jaringan dapat bekerja di DOS. Dalam sistem lama seperti itu tidak pernah mudah. Saat menulis artikel ini, saya menggunakan Realtek RTL8139 yang sangat umum. Saya juga tahu bahwa Realtec lama di bus ISA dan kartu 3com akan bekerja.


Poin kedua: mengapa saya memilih mTCP? Setelah semua, ada, sehingga untuk berbicara, Microsoft LAN Manager "sengsara", yang ditulis khusus untuk DOS. Ada satu, tetapi pertama-tama ia menyediakan karya hanya utilitasnya sendiri. Program pihak ketiga seperti klien FTP yang sama masih memerlukan driver batch yang terpisah. Dan kedua, 200 kilobyte memori utama memakan (angka yang mengerikan) dan bekerja dengan jaringan yang sedang berjalan menjadi tidak mungkin. mTCP adalah sekumpulan aplikasi jaringan seperti DHCP, TELNET, NETCAT, dll. Untuk pekerjaannya, itu hanya membutuhkan driver batch dan tidak menggantung di memori. Ya, Manajer LAN memiliki keuntungan. Selain TCP, mendukung IPX yang dibutuhkan untuk bermain jaringan. Jadi diperlukan klarifikasi penting: Artikel ini tidak cocok untuk mereka yang ingin menggunakan jaringan untuk permainan.
