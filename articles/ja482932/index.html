IIoT-または、従業員が意図したとおりに脳を使用できるようにする方法

従業員のパフォーマンスを測定し始めた会社に監査人を派遣すると、ほとんどの企業ではその結論は期待外れになります-それは事実です。


これがなぜ起こっているのか疑問に思ったことはありますか？従業員の頭脳が本当にスマートなタスクに取り組む頻度はどれくらいですか？毎日、毎時、分単位、さらには毎時の単純な操作の負担なしに、彼はこれにどのくらいの時間を費やしていますか？ 「自動化」と「デジタル化」という言葉が長く痛ましい時代に、誰もがそれが何であり、なぜそれが必要なのかを理解しているわけではありません。ここでは、IIoT（産業用モノのインターネット）の例でそれについて考察し、話します。

デジタル化、自動化、モノのインターネット、特に産業用インターネットは、日常生活から日常を取り除くように設計されています。ルーチンとは、定期的に繰り返される一連の機械操作を意味します。これはこの記事のタイトルとどのように関係していますか？そして非常に簡単です-あなたの従業員は最終的に、より複雑な問題を解決するために彼らの脳の努力を導くことができるでしょう。そして、これは常に生産の高速化と労働生産性の向上につながります。下記は用例です。

例1-保管条件




与えられた：

  • 冷凍製品を保管するための部屋が多数ある企業N。
  • 冷凍設備の「動物園」全体。
  • 製品の保管条件の違反を回避するための責任は、すべての部屋の監視と空気と環境のパラメーター（温度、湿度、圧力、さまざまなガスの濃度、照明など）の監視を含む歩行者のチームです。

所有者は敷地内のストレージモードを制御する必要があるため、クローラーは毎日、毎日2回、読み取り値をログに書き込んでから、レポートを作成して送信する必要があります。

決定。この場合、制御プロセスを改善するにはいくつかの方法があります。

  1. スマートフォンやコンピューターの画面を介して、自分が管理している敷地内で何が起こっているかを常に把握できます。
  2. 使用を簡単にするために、設定したしきい値に達したときに、自分に便利なチャネル（電子メール、インスタントメッセンジャー）で通知を受信するだけで済みます。

ボーナスソリューション-しかし、電気を節約する方法を学ぶことができます。どうやって？- ものすごく単純！必要な場合にのみ、部屋を暖房または冷房することができます。そして、あなたの従業員の直接参加なしでもこれを行うには。また、従業員をより複雑で知的なタスクに割り当てることができます。

例2-Trust-but-Verify




与えられた：

  • 大きな生産エリアを持つ企業N。
  • 企業内の多数の従業員。

所有者が、従業員が職場を出たり、たとえば特定の部屋に行く頻度を理解することは興味深いですか？ジャーナルに記載されているだけでなく、実際に特定のアクションを実行したかどうかを理解することは興味深いですか？- 絶対そうです。

決定。適切なセンサー、従業員の個人パスカード、企業の機器を使用して、必要なデータが収集され、集計、分析、視覚化のためにプラットフォームに送信されます。ソリューションを使用した結果、所有者は受信したデータの監視とコンパイルに長い時間を費やす必要がありません。ソリューションにより、指示された基準を超える緊急事態の瞬間を追跡し、理解することができます。

決定。ここでの大きな利点は、従来の2G / 3G / 4Gよりもデータ伝送に高い浸透率の通信を使用できることです。ベラルーシ共和国では、これは2017年に900 MHz帯域で発売されたNB-IoT（狭帯域インターネット）です。浸透に加えて、このテクノロジーの疑いのない利点には、低消費電力が含まれます。後者の事実は、印象的なバッテリー寿命を備えたポータブルデバイスを作成することを可能にします。 NB-IoTは、センサーからプラットフォームに小さなパケットを送信するためのあらゆる機能を備えています。その結果、データ伝送のためにワイヤを「引く」必要がなく、電力を供給するために追加のワイヤを「引く」必要もありません。ポータブルデバイスをインストールして、そこから受信したデータの使用を開始するだけです。電池は、AA電池から工業用電池まで、さまざまな方法で使用できます。デバイスの最終的なサイズと、電源の再充電/交換を行わない動作時間は、バッテリーの選択に依存することを覚えておくことが重要です。

例4-リモートで監視する




与えられた：

  • 生計に関するデータを必要とする多数のリモートデバイスを備えたエンタープライズN（たとえば、いくつかの異なる地域の温室や畜舎）。

企業の責任者は、畑の土壌水分、光と温度、または給餌/水の量を確認する必要があり、1日を領土の回り道に費やすことができる労働者の数はゼロに近いです。そしてもちろん、必要なパラメータを制御するために畑や農場に移動することも「まあまあ」の喜びです。

解決策：ここでの切り札は、NB-IoTデバイスが「休止状態」になる機能です。 1時間に1回、12時間に1回、1日に1回など、データを送信するためのシナリオを構成できます。これにより、交換することなく、バッテリー/バッテリーの寿命を延ばすことができます。時間の節約、燃料の節約、関心のあるパラメータやその他の「メリット」に関するオンライン情報の取得は、明らかな利点です。

情報を蓄積し、外部データを追加し、機械学習メカニズムを接続することで、予測を取得して予測的に行動できます。これにより、従業員の「無知」、物忘れ、過失の影響を防止する時間と費用を節約し、作物を保護し、確実に適切な条件で飼育および飼育することができます。

例5-予測




与えられた：

  • さまざまな高価なデバイスが24時間稼働しているエンタープライズN。
  • 各デバイスがアイドル状態の場合、部品を遠方から注文する必要があるため、修理に長い時間がかかります。
  • 単純なものはそれぞれ、企業の経済に財政的な影響を与えます。

決定。この場合、企業のシステムにセンサーを統合して耐摩耗性部品の状態を監視し、ニューラルネットワークをトレーニングしてクリティカルおよびプレクリティカルな状態を防止することで、会社の管理者は継続的な生産プロセスを実現できます。これは、予防保守を実施し、必要な新しい部品とアセンブリを事前に注文すること、およびすでに設置されているデバイスと高価な部品の耐用年数を延ばすことで明らかになります。

あなたは好きなだけ多くの例を与えることができますが、有声の例に基づいていても、すべての解決策が表面に出ていることがわかります-それらを取り上げて実践してください。このようなものを作成しようとしている多くの企業が市場にあります。共通点は1つあります。つまり、ユニバーサルテンプレートソリューションを作成しますが、顧客の問題の本質を理解しようとせず、問題を解決するための最も便利で安価な方法、つまり最初に説明した日常的なプロセスの非常に自動化を提供しません。

ここでは、お客様とのコミュニケーションを構築する必要があるようです。各プロジェクトは、いくつかの段階で構成されてい

ます。1. 顧客とのミーティング



原則として、最初の会議には、会社の痛みとニーズを知っているオーナーまたはマネージャーが必要です。当事者が技術プロセスのみを知っている専門家を最初の会議に送る場合、高い確率で自動化するタスクは所有者の問題を解決しません。これには、希望する目標を達成できず、双方で時間の浪費が伴います。

2. 経済のスケッチ



マネージャーが自動化の目標と目的を定式化した後、自動化が自動化のために機能しないように、プロセスの経済学を定式化することが非常に重要です。

3. 技術専門家との面談



「経済的スケッチ」を経て初めて、「技術者」と出会うことができます。これは、将来のデバイスとクライアントシステムとの相互作用のニュアンスを理解するために行われます。 1回の会議ですべてを規定するのは難しいため、質の高いコミュニケーションを確立することが重要です。より良い結果を得るには、通常、技術専門家との会議を数回行う必要があります。

4. 費用と条件の調整



「大規模セル」でプロジェクトを評価した後、オーナーとの2回目のミーティングが行われ、最終決定のコストとプロジェクトの実施条件が合意されます。つまり、おとぎ話のように、結局はうまくいかないのです。つまり、マネージャ、技術者、およびデバイスの製造元が「途方に暮れている」ことに気づき、すでにこの時点までに多大な労力と時間とお金を費やしていて、結果が得られていない場合です。

5. プロトタイプの



プレファイナルフェーズの開発。原則として、必要なデバイスは、最終バージョンで、または一部、メーカーのデバイス群の中にすでに存在しています。必要に応じて、クライアントに必要なセンサーが不足しているため、ソフトウェアが完成しています。必要な設定を備えたクライアントの個人アカウントがプラットフォーム上に作成されます。
場合によっては、クライアントのすべての要件を満たすために、デバイスをゼロから開発する必要があります。しかし、原則として、タスクはすべて典型的であり、それらを解決するための原則/メカニズムは他のものと根本的に異なるわけではありません。たとえば、新しいプロトコルの動作や、以前に使用されなかったモジュールの機能を調べる必要があります。このようなタスクは、製造会社の従業員の忠誠度を高め、将来的に作成される可能性のある独自の顧客ソリューションの数が大幅に増加します。

6. テスト段階



この段階は、時間が許せば、施設で行うのが最適です。しかし、時間が待たずにクライアントがアプリケーションの考えられる影響をテストしたい場合は、「戦闘条件」でプロトタイプをテストできます。テストの段階でプロトタイプが完成し、必要な変更と修正が行われ、結果が関係者に合う場合は、製品化されます。

もちろん、ここではクライアントを操作する一般的なプロセスについて説明します。これは、それぞれに微妙なニュアンスがあるためです。たとえば、重要な顧客プロセスが影響を受ける場合は、協力協定に署名し、技術サポート問題（SLA）を直ちに規定することで、そのような問題が当事者にとって不愉快な驚きにならないようにします。
