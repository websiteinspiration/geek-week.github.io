最小の違い

7月、Gustoの設計チームは立派なものを提供しました。 3か月間、ロゴ、色、フォントなどの企業資料の更新に全力を尽くしました。しかし、これだけではありませんでした。ライブラリのすべてのコンポーネントを改善する機会があると感じました。重複するコンポーネントを削除し、技術的な負債を取り除くことで、プロジェクトの機能を拡張したいと考えました。その後、障害を持つ人々がプロジェクトにアクセスできるようにすることについて、長年の問題に対処し始めました。 しばらくの間、テーブルのデザインについて心配していました。きちんと見てもらえる瞬間が楽しみでした。





プロジェクトの再設計に真剣に取り組む前の様子は次のとおりです。


再設計前の表

そのようなテーブルを、Figmaで使用されているような設計ドキュメントに配置すると、特別な問題を見つけることができなくなります。そのようなテーブルのすべてが通常読み取られます。しかし、コンポーネントが実際のデータでいっぱいになり始めるとすぐに、テーブルは完全に悪く見え始めます。

実際、Gustoは、他の企業が給与計算を実行するのを支援しています。これらの会社のいくつかは数十人の従業員を抱えています。その結果、私たちはお客様に大量の複雑な情報を常に表示しています。たとえば、これは従業員の給与構造の詳細な構造であり、さまざまなレベルでの税金に関する詳細情報です。これは、テーブルを実装するコンポーネントがお客様にとって非常に重要な役割を果たすことを意味します。これはまた、私たちの設計チームは、大量のデータを出力するときにこのコンポーネントが確実に適切に機能するようにあらゆる努力をする必要があることを意味します。たとえば、次のような状況です。


データが入力されたテーブルの例

ユーザーの行動を調査した結果、クライアントは通常、そのようなテーブルを見て、出力するデータを理解しようとしていることがわかりました。この従業員の給与は正しく計算されていますか？すべての税額は正しいように見えますか？ここにエラーがありますか？残念ながら、UXに関しては、お客様が税金を処理しやすくするためにほとんど何もできません。しかし、長年にわたって私を悩ませてきたユーザーインターフェイスに1つの問題がありました。そして、私はそのソリューションがテーブルの認識を改善するのに役立つことを知っていました。

この問題とその解決策について説明するために、まず、フォントである文字の説明のセットで数字がどのように配置されるかについて説明します。

ほとんどのフォントでは、通常、数値の幅が異なります。その結果、たとえば、数値4は数値1よりも広くなる可能性があります。通常、ここでは問題はありません。数字が単純な文で使用される場合-数字の幅が異なる-これは完全に正常です。通常、1,235.55ドルなどが9,993.24ドルより長くても短くても、心配はありません。


数値の幅は異なり

ますが、特定の数値がテーブルなどのグループで表示される場合、小数点の位置が異なるため、隣の数値が異なっていても、これは実際の悪夢になる可能性があります。正しく配置されます。つまり、テーブルをすばやくスキャンすると、$ 1000.00が突然$ 100.00に非常に似たものになる可能性があります。

次の例を注意深く確認し、必要に応じてモニターに近づけてください。また、ここの数値が正しく揃っていないことに注意してください。


数字の配置が正しく

ありません。これらの数字は同じスタイルで推定されます。 2つの数字の唯一の違いは、それらが異なるグリフを使用することです。そのため、これらは1つのフォントからの数字であり、それらを出力するときに同じ文字間隔が使用されます。

理想的には、すべての数値が正しく配置され、小数点が常に同じ位置を占めるようにして、テーブルの読みやすさを向上させたいと考えています。

良い点は、この問題の解決策がWeb上で長年存在していることです。数値の代替スタイルの使用に関して、多くのフォントに組み込まれているOpenType機能について話しています。この機能は、フォントの説明で、たとえば「表形式の図」、「表形式の図」、「等幅図」と呼ぶことができます。この機能は次のように有効にできます。

.table {
  font-variant-numeric: tabular-nums;
}

Figmaでこの機能を有効にするには、テキストを選択し、画面右側のOpenType設定に移動する必要があります。その下に現在の外観が表示されていますが、将来的にはこのパネルが変更される可能性があります。


Figmaのフォント設定

すべてのフォントがこの機能をサポートしているわけではないことに注意することが重要です。したがって、選択したフォントが数字を表示するための特殊文字をサポートすることを期待しないでください。常にすべてを確認してください。しかし、いずれにしても、結果として得られるものは、適切なフォントを見つけるために費やす努力の価値があります。次の図を見てください。言うまでもなく、このオプションを使用したおかげfont-variant-numeric: tabular-numsで、右側の数字のペアは左側の数字のペアよりもはるかによく見えます。


font-variant-numeric：tabular-numsオプションの使用

ここでは、小数点が常に同じ位置にあることがわかります。1ここで使用されているCentraフォントのシンボルが異なって見え始めていることに気付くでしょう。しかし、古いデータ出力に対する新しいデータ出力方法の利点を実際に確認するには、数値の大きな列を出力する必要がある場合にのみ可能です。特にテーブルでは。


古いテーブルと新しいテーブルの出力オプション。新しいテーブル

と古いテーブルのオプションは非常に異なると思います！以前はそれぞれの番号が独自の生活を送っていましたが、表形式の番号を使用すると、表をすばやく表示しやすくなり、システムのユーザーが表形式のデータを確認しやすくなりました。

突然、新しい方法で設計されたテーブルは、古いテーブルよりもすっきりした見た目になりました。また、古いバージョンと新しいバージョンを比較し始めると、ユーザーインターフェイスのデザインのわずかな変更がプロジェクトの外観に大きな影響を与えることが理解できるようになります。

読者の皆様！プロジェクトのフォントデザインを少し調整するだけで、外観と操作が簡単になる状況に遭遇したことがありますか？
