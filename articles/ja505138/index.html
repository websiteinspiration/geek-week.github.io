アマゾンストリーミング


「CDN」は「コンテンツ配信ネットワーク」の略です。これは地理的に分散されたネットワークインフラストラクチャであり、インターネットユーザーへのコンテンツの配信と配布を最適化できます。 CDNの詳細については、Habréで直接読むことができます。

言い換えれば、あなたの近くのどこかに常に、ブラウザに表示されるすべての写真とビデオを保存してすばやくすばやく提供するサーバーがあります。メインのAmazonサーバーが配置されているバージニア州のサーバーからすべての写真をダウンロードするたびに、サイトの半分が2倍の時間でロードされます。または、まったくロードされません-Webサーバーが負荷から落ちたためです。

オンライン会議のビデオは、CDNを介して配信する必要があります。しかし、数百万ルーブルであっても、自分の手でスタジオまたはハードウェアルームを構築できる場合、世界中で独自のCDNを構築することはできません。

モスクワまたはサンクトペテルブルクでビデオを配信するサーバーを1つだけ構築したいとしても、それは非常に不愉快なことです。このようなシステムの特徴は、最大の可用性と負荷時のフォールトトレランスです。負荷が大きいほど、高い資格を必要とするエンジニアリングの問題が複雑になります。サーバーとハバロフスクの誰かとの間のネットワーク接続が失われた場合はどうなりますか？不明な理由により、ビデオサーバーを備えたコンテナが毎秒5回落下し始めるとどうなりますか？方法をよく知っている人にお金を支払う方が簡単です。

私たちはいくつかの異なるビデオストリーミングソリューションを探索してきましたが、今のところAmazonが提供するものに焦点を当てています。

プロセスは次のとおりです。

  • ハードウェアの結果は、ビデオストリームと事前に記録されたビデオファイルです。
  • RTMPを介してAmazon MediaLiveサーバーにビデオストリームを送信します。
  • ファイルはAmazon MediaConvertサーバーに送信されます。
  • ビデオはストレージにアップロードされます-Amazon S3 ;
  • そして最後に、データはS3から抽出され、CDNサーバー（Amazon CloudFront）に分散されます。

明らかに、すべてのサービスはお金の価値があります。正確なコストは、地域とボリュームによって異なります。当初、これらの金額はそれほど印象的ではありませんが、合計すると、かなりまともです。たとえば、会議が多いほど、保存する必要のあるファイルが多くなります。

残念ながら、CloudFrontにファイルをすぐにアップロードすることはできません。最初にファイルを準備する必要があります。インターネット接続の品質と画面サイズが異なる参加者は、異なる品質のビデオを必要とします。

AWS Elemental MediaConvertアダルト放送レベルの機能を備えた動画トランスコーディングサービスです。ビデオファイルは、WebインターフェイスまたはAPIを介してダウンロードされた後、任意の形式にトランスコードされます。 MediaConvertにファイルをアップロードした場合、エンコードを提供するすべての責任はユーザーではなく、Amazonにあります。費用は公式サイトにあります。



AWS Elemental MediaLiveもほぼ同じですが、ビデオストリーム用です。考えているのは、MediaLiveにあなたが持っている最高の品質でストリーミングすることです。そして、さまざまなデバイスやインターネット速度に適したいくつかのオプションを生成します。



これらのサービスの機能はクールに見えますが、安くはありません。たとえば、トランスコーディングサービスを購入する場合、それらがうまく機能しない可能性があることがわかります。トランスコーダーがクラッシュしてクラウドで再起動すると、時間が無駄になり、会議の参加者には何も表示されないか、プレーヤーでブロードキャストが中断されます。この恐怖が発生しないようにするには、同じビデオストリーム用に複数のトランスコーダーを一度に購入する必要があります。

Amazon Simple Storage Service（Amazon S3）データストレージサービスです。そこで、ほぼ無制限のボリュームでデータを入力できます。一般に、S3はあらゆるもののユニバーサルストレージとして使用できます。ファイルの配布、ビッグデータの分析、IoTデバイスの同期などが可能です。私たちの場合、S3はMediaLiveとMediaConvertがレンダリングするすべての中間ストレージとして必要です。ビデオをCDNに転送できるようにするには、設定でS3の適切なバケットを指定する必要があります。



信頼性に関しては、S3は99.999999999％の信頼性を提供します（ここでは11ナイン）。このような信頼性は銀行にも適していますが、ビデオにはさらに適しています。これはスキーム全体で最も安定した部分であると言えます。この安定性は、 Amazonがメガバイトごとに支払うお金にかかっています。

Amazon CloudFrontはユニバーサルCDNであり、ビデオを配信することもできます。ビデオはロシアではかなり低いレイテンシと高速で動作します。CloudFrontは、S3およびLiveサービスを含むAWSと統合されています。魔法のように見えます。管理パネルでいくつかのパラメータを設定すると、突然、ビデオが世界中に高品質で配信され始めます。

CloudFrontの支払いは使用時に発生します- 会議の参加者が実際にビデオを見た場合にのみお金がかかり始めます。S3とCloudFront間のトラフィックに料金を支払う必要がないことが非常に重要です。Amazonの内部サービス間のトラフィックは完全に無料です。しかし、S3の代わりに他のストレージを使用しようとすると、かなりの費用がかかります。

動画プレーヤーでストリームをキャッチします


CloudFrontにビデオが表示されたら、ブラウザーのフロントエンド（つまり、会議の参加者が見るWebサイト）に組み込まれているビデオプレーヤーを使用してビデオをピックアップできます。
