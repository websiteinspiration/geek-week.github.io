Développer une application web interactive en C # sans aucune connaissance de JavaScript

Je suis développeur .Net chez Turkmen-Tranzit.

Mon PM avait besoin de quelqu'un qui voulait ressentir Blazor. À mon avis, j'ai fait du bon travail dans ce domaine. Au cours de l'étude, j'ai rencontré des problèmes dont les réponses ne sont pas si faciles à trouver sur Internet en russe.



Pourquoi Blazor, et non JavaScript standard?

En termes simples, la technologie Blazor vous permet de créer des applications Web dont le côté client se compose de balises HTML et de code C # au lieu de JavaScript.

Blazor peut accéder au DOM pour redessiner la page sans rafraîchir la page L'
application Web sur Blazor est constituée de composants. Les composants sont conçus pour afficher des éléments visuels tels que des indicateurs de téléchargement ou des formulaires de téléchargement de fichiers. Le balisage des éléments est stocké dans un fichier avec l'extension .razor, et toute la logique dans les classes C # ordinaires avec le nom {ComponentName} .razor.cs.

Les composants de Blazor sont divisés en intelligents et stupides.

Les composants stupides sont indépendants de l'application entière, ils peuvent être connectés à d'autres composants et ils peuvent même utiliser d'autres composants stupides.

Les composants intelligents sont jetables, ils doivent conserver leur état et très rarement ils sont utilisés dans d'autres composants.

Le plus souvent, vous devrez écrire des composants intelligents.

Dans Blazor, vous pouvez appeler des fonctions JavaScript à l'aide du code C # et vice versa, vous pouvez appeler des méthodes .Net à l'aide de JavaScript.

Oui, si vous avez une connaissance approfondie de JavaScript, que vous êtes guidé dans les cadres JavaScript et que vous venez de vous familiariser avec le monde de .Net, alors il vaut mieux ne pas entrer dans Blazor, la technologie est relativement nouvelle et lorsque vous commencez à le connaître, vous pouvez avoir peur. Le but de cet article est pour les personnes qui savent utiliser C #, mais pour une raison quelconque qui n'a pas JavaScript, parlez de son homologue prometteur de Microsoft, dans lequel vous pouvez vous permettre d'utiliser toutes les fonctionnalités de C #.

Par exemple, nous développerons le CRM, avec une liste d'entreprises qui pourraient potentiellement être nos partenaires, avec une recherche automatisée des employés de l'entreprise, une distribution automatisée des messages à tous les clients potentiels, l'enregistrement des actions des utilisateurs et bien plus encore.
