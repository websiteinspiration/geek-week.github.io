JSON.stringify（）の5つのあまり知られていない機能



こんにちは！

この短い記事では、JSON.stringify（）のめったに使用されない機能のいくつかについてお話ししたいと思います。それらはあなたに役立つかもしれません。

JSON.stringify（）は、デバッグ中にオブジェクトまたは通常の文字列をJSON形式の文字列に変換するためによく使用されます。しかし、このメソッドはどのように使用され、toString（）を回避できますか？やってみましょう。

//   user
const user = {
    name: 'Harry Heman',
    age: 29,
    job: 'developer'
}

//    ,    toString(),     
console.log(user.toString()) // [object Object]

[object Object]は、私たちが見たかったものとは正確には違いますね。

なぜtoString（）を使用するのかについては質問しないでください。オブジェクトを文字列に変換する必要があるとします。

JSON.stringify（）を使用してみてください：

console.log(JSON.stringify(user)) // {"name":"Harry Heman","age":29, "job": "developer"}

カール、やった！

ただし、JSON.stringify（）の可能性はこれに限定されません。

1. 2番目の引数として配列を使用する


stringify関数には2番目の引数があるかもしれません。この引数は、コンソールに表示するオブジェクトのキーの配列です。そのようなオブジェクトの配列があるとしましょう：

let users = [
    {
        id: 1,
        login: 'harry',
        password: 'qwerty',
        status: 'admin',
        access: true
    },
    {
        id: 2,
        login: 'alice',
        password: 'qwerty',
        status: 'user'
    },
    {
        id: 3,
        login: 'bob',
        password: 'qwerty',
        status: 'user'
    },
    {
        id: 4,
        login: 'john',
        password: 'qwerty',
        status: 'user'
    },
    {
        id: 5,
        login: 'jane',
        password: 'qwerty',
        status: 'user'
    }
]

...そして、どのユーザーが管理者であるかを確認します（管理者には「アクセス」プロパティがあります）。

console.log（JSON.stringify（users））を使用すると、次のようになります。

[{"id":1,"login":"harry","password":"qwerty","status":"admin", "access": true},{"id":2,"login":"alice","password":"qwerty","status":"user"},{"id":3,"login":"bob","password":"qwerty","status":"user"},{"id":4,"login":"john","password":"qwerty","status":"user"},{"id":5,"login":"jane","password":"qwerty","status":"user"}]

同意する、あまり読めない

次を試してください：

console.log(JSON.stringify(users, ['id', 'access'])) // [{"id":1,"access":true},{"id":2},{"id":3},{"id":4},{"id":5}]

良くなったね。

公平に言うと、この場合はconsole.table（users）を使用できることに注意してください

2. 2番目の引数として関数を使用する


この関数は、各キーと値のペアを2番目の引数として評価し、この関数のロジックに従って文字列化します。undefinedを返す場合、対応するキーと値のペアは表示されません。例：

const result = JSON.stringify(users, (key, value) => {
    if (typeof value === 'string') return undefined

    return value
})

console.log(result) // [{"id":1,"access":true},{"id":2},{"id":3},{"id":4},{"id":5}]

3. 3番目のパラメーターとして数値を使用する


3番目の引数はスペースを担当します。この引数が数値の場合、各行レベルには対応するインデントがあります。

console.log(JSON.stringify(user, null, 2))

以下を取得します。

{
  "name": "Harry Heman",
  "age": 29,
  "job": "developer"
}

4. 3番目の引数として文字列を使用する


3番目のパラメータが文字列の場合、スペースの代わりにこの文字列の文字が使用されます。

console.log(JSON.stringify(user, null, '->'))

以下を取得します。

{
->"name": "Harry Heman",
->"age": 29,
->"job": "developer"
}

5. toJSONメソッド


toJSONメソッドは、任意のオブジェクトのプロパティにすることができます。JSON.stringify（）は、オブジェクト全体を変換せずにこの関数の結果を返します。

const user = {
    first: 'Harry',
    last: 'Heman',
    age: 29,
    toJSON(){
        return {
            full: `${this.first} ${this.last}`
        }
    }
}

console.log(JSON.stringify(user)) // "full": "Harry Heman"
