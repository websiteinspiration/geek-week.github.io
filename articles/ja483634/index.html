CSSの最小と最大の幅/高さのプロパティ

「の翻訳CSSでの最小値と最大幅/高さアフマドShadidによります」。

場合によっては、開発者は要素の幅を親を基準に制限すると同時に、動的なままにする必要があります。このように設定することにより、使用可能なスペースがある場合に拡張が可能な初期サイズになります。たとえば、最小幅のボタンが必要です。このような状況では、最大値と最小値のプロパティを使用すると便利です。

この記事では、幅と高さの最大値と最小値のCSSプロパティに慣れ、それぞれのプロパティとその使用シナリオについて詳しく検討します。

幅のプロパティ


幅に関連するプロパティから議論を始めたいと思います。私たちにはmin-width、max-width,そしてそれぞれが非常に重要であり、特定の状況で役立ちます。

最小幅（最小幅）


目的min-widthは、要素の幅が所定の値を下回らないようにすることです。デフォルト値があるautoまで行くことができ、0。

それを実証するために、基本的な例を見てみましょう。



テキスト付きのボタンがあり、そのサイズは1ワードから数ワードまでさまざまです。このボタンの幅が最小であることを確認するには、単語が1つしかない場合でも使用する必要がありますmin-width。最小幅は100pxに設定されているため、ボタンに「完了」という単語やアイコンのみなどの非常に短いコンテンツが含まれている場合でも、目立つように十分な大きさになります。これは、異なる言語に翻訳されたサイトで作業する場合に特に重要です。

Twitterでのアラビア語の例を考えてみましょう。



上の例では、ボタンに「تم」という単語が含まれています。アラビア語では「Finish」を意味します。内容が短いため、ボタン自体の幅が狭くなりすぎたためmin-width、右側の画像（「後」セクション）に表示されるプロパティに設定して、最小値を設定しました。

最小幅とパディング


このプロパティmin-widthは、コンテンツのサイズの増加に伴ってボタンの幅の増加を許可しますが、padding境界からのインデントを保証するために側面に設定する必要があります。下の図に違いを示します。



最大幅（最大幅）


このプロパティはmax-width、要素の幅が特定の値より大きくなるのを防ぐことを目的としています。デフォルト値はnoneです。

max-widthを使用する一般的で単純な例は、画像に適用することです。以下の例を検討してください。



画像にプロパティを適用すると、max-width: 100%元のサイズが大きくても、プロパティは親要素の境界内にとどまります。画像が最初に親よりも小さい場合、このプロパティはまったく影響を与えません。

最小幅と最大幅の使用




両方のプロパティが要素に適用される場合、どちらが他方の値をオーバーライドしますか？言い換えれば、どのプロパティが優先度が高いのですか？

値がmin-widthより大きい場合max-widthは、要素の幅と見なされます。次の例を考えてみましょう：

HTML

<div class="wrapper">
  <div class="sub"></div>
</div>

CSS

.sub {
  width: 100px;
  min-width: 50%;
  max-width: 100%;
}



初期値widthは100pxで、これに加えてプロパティ値min-widthとが設定されていmax-widthます。その結果、この要素の幅は親ブロックの幅の50％を超えません


高さのプロパティ


最小幅と最大幅のプロパティに加えて、高さにも同じプロパティがあります。

最小の高さ（min-height）


このプロパティはmin-height、アイテムが指定された値を下回らないようにするためのものです。デフォルト値があるautoまで行くことができ、0。

それを示すために、簡単な例を見てみましょう。

テキストコンテンツのセクションがあります。CSSテキストが大量でも少量でも美しく見える必要がある





.sub {
  display: flex;
  align-items: center;
  justify-content: center;
  padding: 1rem;
  min-height: 100px;
  color: #fff;
  background: #3c78dB;
}

最小の高さは100pxに設定され、コンテンツはフレックスボックスを使用して水平および垂直に中央揃えされます。さらにコンテンツがあるとどうなりますか？たとえば、段落全体がありますが、



はい、すでに推測しました。より多くのコンテンツに対応するために、セクションの高さが増加します。これにより、コンテンツの量に対応する柔軟なコンポーネントを作成できます。

最大の高さ（max-height）


このプロパティはmax-height、指定された値を超える要素の高さの増加を防ぐことを目的としています。デフォルト値はnoneです。

次の例でmax-heightは、コンテンツに設定します。しかし、指定された領域より大きいため、オーバーフローが発生し、テキストは親要素の境界を超えます

比例要素のサイズと最大幅/高さ


ビューポートのサイズに応じてスケーリングするプロポーショナルコンテナを作成するために、かつてパディングを使用してハックが提示されました。これで、ビューポートの単位と最大の幅/高さを組み合わせて、同様の動作をシミュレートできます。ミリアム・スザンヌの記事の

資料に基づいて、これがどのように機能するかを説明したいと思います。 644 x 1000ピクセルの画像があります。次のことを行う必要があります。





  • アスペクト比：height/ width（高さを幅で割ったもの）
  • コンテナの幅：固定または動的（100％）
  • heightビューポートの幅にアスペクト比を掛けたものを100％に設定します
  • max-heightコンテナの幅にアスペクト比を掛けた値に等しく設定します
  • max-widthコンテナの幅に等しく設定

CSS

img {
  --ratio: 664 / 1000;
  --container-width: 30em;
  display: block;
  height: calc(100vw * var(--ratio));
  max-height: calc(var(--container-width) * var(--ratio));
  width: 100%;
  max-width: var(--container-width);
}

CSS比較関数


StackOverflowについて開発者が直面する主な問題を調査しているときに、筆者が次の質問をするこの質問に出くわしました。
このようなことをすることは可能ですか？max-width: calc(max(500px, 100% - 80px))または多分これmax-width: max(500px, calc(100% - 80px)))
望ましい動作は、要素の幅を500px以下に保つことです。スコープの幅が要素を収容するのに十分でない場合、幅はでなければなりません100% – 80px。

質問に答えて公開されたソリューションについては、私はそれを適用しようとしましたが、Chrome 79とSafari 13.0.3で動作します。これは記事の範囲を超えていますが、ここでも解決策を提供します：

CSS

.yourselector {
  max-width: calc(100% - 80px);
}
 
/* Update: I changed the max-width from 500px to 580px. Thanks @fvsch */
@media screen and (max-width: 580px) {
  .yourselector {
    max-width: 500px;
  }
}
