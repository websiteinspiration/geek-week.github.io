Task with 104 GIFs



The initial data for the task is an archive containing 104 GIF files with different names. The name of each file has a prefix - part_. The name of the archive is by8_128x128px_2layers .zip. We immediately notice that these are not gifs, but just images. We overtake in another format:









The hardest part of this task is to understand the condition. Based on the file name, we understand that we are dealing with 128x128 images that have 2 layers each, that is, the picture is divided into two disjoint black pixels that, when combined, give the original image, as shown in the illustration:









It turns out that the image is divided into 8 (by8 from the file name) parts. Consequently, the picture is divided into 4 squares of the same size (8/2). This is proved by the fact that each file from the archive has a size of 64x64. Since 104 images are given, and each square is assembled from 8 images, there are 13 (104/8) 128x128 squares in total. Merge layers using a Python script using PIL .
