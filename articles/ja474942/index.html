VMworld 2019、Day II：取り扱い、安全性、およびいくつかの新しい発表

VMworld 2019ヨーロッパ会議からのバルセロナからのライブブロードキャストであるIT-GRADに再び接続しました。総会の2日目の様子を見て、その日の主要なイベントについて話し合う時が来ました。小さなネタバレ：今回は、サンフランシスコとは異なり、ステージから新しいポストの象のお祝いの配布はありませんでした。記事の最後に、同僚からのVM​​world 2019 Europeからの別の短いビデオがあります。




同社のCEOであるPat Gelsingerは、今回は講堂に誇りを持ち、マイクを会社の副社長であるRay O'Farrellに失った。
主にインスピレーションを与える製品の発表と非常に一般的な会社のニュースに焦点を当てた前回のセッションとは異なり、今回は主催者が角を曲がって大画面の新機能のデモに直接行きました。

レイは、セッションの最初の数分で聴衆に見せられたミニフィルムに特に注意を払いました。つまり、テクノロジーが現代人の生活にどのように有機的に統合されているかです。ただし、Balzacを言い換えると、各大規模アプリケーションの背後には何千人もの開発者による膨大な作業があり、コードを発明して作成しただけでなく、あらゆるニュアンスをデバッグして、ユーザーに最高の快適性、速度、利便性を提供しています。ほとんどの場合、アプリケーションを使用している企業に対するユーザーの態度は、ユーザーの良好なエクスペリエンスに依存します。また、多くの企業にとって重要な問題は、古くても信頼できるテクノロジーの使用と新しいテクノロジーの接続との間の妥協点です。この妥協がどのように解決されるかから、市場における会社の将来は異なります。したがって、現時点でのVMwareの主要なタスクの1つは、新しいテクノロジーへの移行をシンプル、柔軟、安全にすることです。そして、この移行が行われた後、これらすべての能力を管理するための信頼できるツールをビジネスに提供します。
もちろん、これは非常に大規模な作業です。大規模なタスクには、大規模なソリューションが必要です。

製品の新機能を実証するために、VMwareはプレゼンテーションを4つの相互接続されたセクターに分割しました。
  • アプリケーションの作成と展開。
  • 接続とセキュリティ;
  • 管理;
  • 経験。

Rayは、ブランドのVMware Tシャツ（Pat Gelsingerが会議の非公式の部分で着用するようなもの）を配布している架空の会社Tanzu Teesを例として挙げました。
Tシャツの生産に加えて、同社は別の問題に直面しています。それは、商品を最も技術的にオンラインで流通させる方法です。


作成と展開


したがって、まず、オンラインストア自体を作成する必要があります。しかし、架空の会社でさえ、VMwareで作成された場合、半分の測定値を受け入れず、すべてを最高レベルで作成する必要があります。
レイの最初のゲストゲストは、VMwareのクラウドコンピューティング技術主任であるJoe Baguliで、聴衆とクラウドネイティブアプリケーションを構築する新しい方法を共有しました。
デモは、Spring Initializr画面から開始されました。 Joeは、基本的な日常の商品配達アプリケーションを作成し、それに主要な依存関係を関連付け、アプリケーションの基本コードを生成して、ロジックを実装する準備をしました。
この方法で作成されたドラフトは、Bitnami（VMware Cloud Marketplace）のカタログにあり、ワンクリックですぐに展開できます。もちろん、いくつかの展開方法があり、レイはそれらの中で最も速い聴衆を示しました。
ただし、別の問題が残っています。Kubernetesを使用して最新のアプリケーションを構築していて、複数の開発チームと複数のクラウドを単一の管理環境で接続したい場合はどうでしょうか。Tanzu Mission Control
はこれをうまく行っています。



上のスクリーンショットでは、Kubernetesクラスターがいくつかのパブリッククラウドに表示されています。Amazon、Microsoft Azure、Google、VMwareで、1つの画面で仮想的に管理できます。



[名前空間]タブでは、すべてのアプリケーションを名前空間で並べ替えて表示できます。ワンクリックで、新しい名前空間（Tanzu Teesを覚えている）を作成し、同時にそれを使用できるクラウドを決定できます。



まったく同じシンプルで理解可能な方法で、必要なアプリケーションポリシーを設定し、選択したすべてのクラウドで自動展開を開始できます。
それでも、多くの企業がvSphereを長年使用しているため、新しいツールへの移行が苦痛を感じることはほとんどありません。 VMwareは、Project Pacificでこの問題を解決する予定です。ProjectPacificは、vSphere内で直接Kubernetesをネイティブに起動し、使い慣れたインターフェイスに新機能を少し追加し、Kubernetesクラスター、仮想マシン、コンテナーを操作することを目的としています。
Joe Baguliによるデモンストレーションは、Kubernetesアプリケーションの将来に関する楽観的な予測と、世界中のビジネスにもたらすメリットの概要で終わりました。

接続とセキュリティ


1つのスライドでBaguliが述べたすべてを要約すると、Rayは次のスピーカーであるVMOSのプリンシパルシステムエンジニア（VMWare）を招待して、Tanzuツールを使用してすべてのアプリケーションコンポーネントを安全かつ簡単に接続する方法について話しました。

議論は、Rayの自明ではない質問から始まりました。構成の単純さと柔軟性を維持しながら、アプリケーションの起こり得るリスクをカバーするセキュリティのレベルをどのように保証するか？

以下のスクリーンショットでは、NSXドメイン内のすべてのアプリケーションの従来のプレゼンテーションが、より理解しやすく便利なものに変換され、レビュー中のTanzu Teesプロジェクトにのみ影響を与えていることがわかります。





感嘆符が付いた絵文字にカーソルを合わせると、システムによって通知されたすべての異常とセキュリティ違反に関する情報を取得できます。

次のデモのスクリーンショットは、NSX Intelligenceのマルチプラットフォーム機能を示しています。



1つの画面には、コンテナー、仮想マシン、およびベアメタルサーバーに関する基本情報が含まれています。



エンドポイントの1つを選択すると、すべてのアクティブなスレッドの詳細な要約を取得できます。



[VM情報]タブでは、アプリケーションのアクティブなプロセスとユーザーの数に関する情報を取得できます。





これは、セキュリティの異常と侵入の試みに関する視覚化された情報です。



しかし、最も強力な新機能の1つは、アプリケーションのパーソナライズされた推奨事項を取得し、必要なものをすぐに適用できることです。



このプロセスでは、シミュレーションモードを開始して、推奨されるセキュリティポリシーを適用する前後の結果を比較できます。結果を比較します。数回クリックするだけで、現在のデータに基づいてアプリケーションのセキュリティが大幅に向上しました。

しかし、それだけではありません。次の質問は、脆弱性と重要なセキュリティイベントを分類し、インフラストラクチャとアプリケーションへの影響を軽減することは可能かということです。

次のデモでは、MarcosがCarbon Blackワークロードツールの機能を示しました。



[トポロジ]タブでは、任意の接続を選択して、アプリケーション内およびグローバルネットワークとの通信に関する情報を取得できます。



違反があった場合は、詳細な情報を入手できます。



ワークロードでは、重大度の評価を持つアプリケーションの脆弱性を調査し、必要に応じてパッチを適用できます。利用可能なパッチがない場合は、拡張侵入防止モードをオンにすることができます。



ワークロードの機能を示すために、マルコスはハッカーの役割を試し、Nmapを使用してApache Tomcatの脆弱性を「発見」しました。



次に、マルコスはWindows Defenderを無効にし、シェルアクセスを取得しようとしました。しかし、強化された侵入防止ポリシーのおかげで、彼は失敗しました。



「攻撃」に関する情報は、ワークロードインターフェースに即座に表示されました。



攻撃が行われたチェーン全体を確認できます。

管理


会議でO'Farrellによって提起された次のトピックは、アプリケーションの管理に関するものです。展開とセキュリティの革新についてはすでに説明しました。アプリケーションを効率的に管理する方法を理解するときが来ました。RayはPurnima Padmanabhan（副社長、製品管理、VMwareのクラウド管理）をステージに招待しました。

では、ITはデータ量の急増とデジタルビジネスの運用ニーズにどのように対応できるでしょうか。複雑なマイクロサービス環境を効果的に管理し、問題をすばやく修正するにはどうすればよいですか？そして、最も重要なのは、どのマイクロサービスグル​​ープが問題を引き起こしているのかをどのように理解するかです。

特にこのために、Wavefrontソリューションが作成されました。これを使用すると、アプリケーションの内部の問題を追跡できます。



問題ゾーンを発見したら、（上記のスクリーンショットのように）混沌としたグラフの山を便利な要約グラフに変えることができます。これにより、正確に何が問題だったかをより明確に確認できます。



Wavefrontインターフェイスを使用すると、仮想インフラストラクチャのあらゆる側面を効率的に管理できます。

もう1つのイノベーションはProject Magnaです。 Magnaは、機械学習に基づいて、インフラストラクチャの任意のコンポーネントに最適な設定セットを決定し、インフラストラクチャの負荷または構成が変化したときに微調整できるようにします。

レイが提起した次の質問-クラウドでのコスト超過を事前に予測して回避する方法は？Tanzu Mission ControlインターフェイスからアクセスできるVMware Cloud Healthは、この問題を解決し、クラウドのコストを透過的に評価できます。予算の上限に近づいたときに通知するように通知を設定できます。

特に、Cloud Healthでは、仮想マシンのステータスをリアルタイムで監視でき、主要なパフォーマンスインジケーターを改善するための具体的な推奨事項を提供します。



運転経験


マルチクラウド環境でのデジタルワークスペースの編成は、Shiki Mittal（製品管理およびVMware戦略ディレクター）によるレポートの主要な議題になりました。彼女はHorizo​​nとWorkspace ONEの新機能について話しました。Horizo​​n 7の新機能
  • 導入の自動化とライフサイクル管理

展開と構成については、更新されたコンソールで必要なパラメーターを指定するだけで十分です。詳細なダッシュボードを使用すると、1つのウィンドウからすべてのサイトを管理できます。



  • 画像とアプリケーションのための単一のリポジトリ

これで、管理者は同じイメージとアプリケーションを使用して、同期の問題なしに世界中のどこにでもHorizo​​n 7を展開できます。

  • Microsoft Azureとの緊密な統合

Microsoft Azureを使用したサイトへのサイトの展開がはるかに簡単になりました。 Microsoft Azure Marketplaceからの仮想マシンのインポートと、構成可能な仮想デスクトップWindowsの使用をサポートしています。

そこで、仮想ワークステーションを操作するためのツールの完全なパッケージを取得しました。それを保護する時が来ました。最終的に、ビジネス側のデバイスの脆弱性は、顧客に深刻な問題を引き起こす可能性があります。特に、これはゼロデイ脆弱性にさらされる可能性のある「若い」アプリケーションに適用されます。

Workspace ONE Intelligenceは、すべてのデバイスとアプリケーションを追跡できる単一のコントロールパネルの更新バージョンです。



彼女はすでに述べたカーボンブラックのワークロードと協力して働いています。ご存じのとおり、Workloadは、これまでにそのようなものに遭遇したことがなくても、疑わしいアクティビティに迅速に対応できます。また、Workspace Oneは、可能な動作を予測してリスクを回避できる機械学習アルゴリズムを採用しています。



会議で提示されたすべてのツールの組み合わせにより、アプリケーションの作成からその展開までのすべての段階で、エンドユーザーのセキュリティを最も簡単かつ最短の方法で確保できます。さらに、VMwareと他の企業との数多くの統合により、vSphereなどの通常のものを放棄せずに、効率を高めてビジネスを利用する機会が企業に与えられます。
その結果、通常のプロセスのコストが削減され、Tanzu Teesのバイヤーは時間通りにシャツを受け取ります。 Pat Gelsingerの満足度を見てください！
