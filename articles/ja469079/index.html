Visual Studio CodeのPython：9月の拡張機能の更新

Visual Studio Codeの9月Python拡張機能拡張が利用可能になったことをお知らせいたします。Python拡張機能  は、Marketplaceからダウンロードするか、Visual Studio Codeの拡張機能ギャラリーから直接インストールでき  ます。Python拡張機能がすでにインストールされている場合は、Visual Studio Codeを再起動して最新のアップデートを入手することもできます。Visual Studio CodeのPythonサポートについて詳しくは、ドキュメントをご覧ください。

これは小さなリリースであり、Python言語サーバーの改善、Jupyter Notebookセルのデバッグ、仮想環境の作成の検出など、35の問題を取り上げました。改善の完全なリストは、 変更ログに記載されています。さて、この記事では、最も興味深い更新を検討します。



Python言語サーバーの改善 


Python言語サーバーにリンティング機能が追加されました。その最新リリースには、新しいリンティングメッセージと多数の追加の一般的な拡張機能が含まれています。

Python言語サーバーによって提供されるLintingメッセージには、未解決のインポート、未定義の変数、関数呼び出しの引数の多さ、不明なキーワード引数、およびクラスではないものからの継承の検出が含まれます。 Linting投稿の完全なリストを確認するには、GitHubのLanguage Serverリポジトリにある  ドキュメント  、   または   Visual StudioのPythonドキュメントの設定ヘルプページをご覧ください。



さらに、＃noqaの一般的なサポートを追加したため、lintingメッセージを無効にできます。コメント#noqaのある行では、診断出力が無効になります。詳細については、ドキュメントを確認してください。 



Jupyter Notebookセルデバッグの機能強化


8月のリリースでは、カスタムコードを入力できるJupyter Notebooksセルをデバッグする機能を追加しました。このリリースでは、この機能が拡張され、必要に応じて非ユーザーコードを入力できるようになりました。この機能を有効にするには、設定ページ（[ファイル]> [設定]> [設定]）を開き、「Data Science：Debug Just My Code」を見つけて、チェックボックスをオフにします。 設定を無効にすると、関数の呼び出しを開始できます。たとえば、非ユーザーコードの動作や、実行時に変数がどのように変化するかを確認できます。







仮想環境の作成の検出


ステータスバーに表示されるPythonインタープリターは、Python拡張機能がPythonコードの実行（たとえば、Pythonコマンド：ターミナルでのPythonファイルの実行を使用）、およびオートコンプリート、構文チェック、リンティング、フォーマットなどの言語サービスの提供に使用する環境を示します。その他 ：新しい仮想環境を作成する際に、このリリースでは、質問はワークスペースのためにそのインタプリタを選択するかどうかを尋ねられます 。これは、ワークスペースの設定に新しい仮想環境からPythonインタプリタへのパスを追加しますので、この環境では使用されますパッケージをインストールし、Python拡張機能を介してコードを実行するとき。
