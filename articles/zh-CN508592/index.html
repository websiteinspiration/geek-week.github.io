JavaScript开发人员应该对Curl了解什么





朋友们，美好的一天！



我向您介绍了Valery Karpov 撰写的文章“ JavaScript开发人员应该了解的Curl”的翻译。



Curl是一种流行的命令行工具，通常用于发送HTTP请求。Curl支持大量协议，但是，作为Node.js开发人员，您很可能会使用它向RESTful API发送http请求。



curl文档包含383个命令行标志的列表，因此很难找到所需的内容。在本文中，我想与您分享一些我经常使用的模板。例如，将使用httpbin.org服务。

提交授权请求



访问RESTful API时，授权标头用于将授权数据包括在请求中。要添加指定的数据，必须使用标志-H



。例如，如果您的接口密钥（API密钥）my-secret-token



，可以将其包括在http请求中，如下所示： 



$ curl -H "Authorization: my-secret-token" https://httpbin.org/get
{
  "args": {}, 
  "headers": {
    "Accept": "*/*", 
    "Authorization": "my-secret-token", 
    "Host": "httpbin.org", 
    "User-Agent": "curl/7.58.0", 
    "X-Amzn-Trace-Id": "Root=1-5ee8e1a5-a3aa30e0765a7980b04ca4a0"
  }, 
  "origin": "69.84.111.39", 
  "url": "https://httpbin.org/get"
}


请注意，它会httpbin.org



在属性的响应正文中返回http请求的标头headers



。



Curl还通过flag支持默认授权-u



。在以下示例中，我们发送带有用户名user



和密码的请求pass



： 



$ curl -i -u "user:pass" https://httpbin.org/basic-auth/user/pass
HTTP/2 200 
date: Tue, 16 Jun 2020 15:18:45 GMT

  "authenticated": true,

}

结论



总之，这是我发现最有用的curl选项：



  • -X



    定义一个请求方法，例如 curl -X POST url



  • -d



    在PUT和POST请求中将请求主体定义为字符串。用于@



    从文件中提取数据
  • -H



    定义请求标头，例如 curl -H "Authorization: my-secret-token" url



  • -u



    标准授权的认证数据
  • -O



    将请求正文保存到文件
  • -i



    显示完整的答案，包括标题


Curl是用于通过命令行与API进行交互的有用工具，无论它是第三方API还是您正在开发的API。为了进行快速测试，如果您熟悉curl的语法，它比Node.js中的Axios或Postman中的查询调优要好。
