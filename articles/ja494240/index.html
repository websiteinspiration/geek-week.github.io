サンドボックスのカスタマイズ：なぜ今必要なのか



サンドボックス画像のカスタマイズの問題は、当初から関連しています。このようなプログラムの最初のバージョンは、情報セキュリティ企業の内部ソリューションでしたが、それでも、孤立したマシンの入念なチューニングが必要でした。そしてそれは、システムの状態を監視するための補助ソフトウェアのインストールだけではありませんでした。

10年前でさえ、環境特性の収集と処理のロジックは悪意のあるサンプルに埋め込まれていました：ユーザー名とコンピューター、ホームドメインまたは企業ドメイン、管理者権限、言語レイアウト、プロセッサコアの数、オペレーティングシステムのバージョン、ウイルス対策ソフトウェアの存在、仮想化の兆候、さらにはアップデートの存在システム。現在までのところ、パラメータの数とそれらを取得する方法は増えています。

サンドボックスバイパスマルウェアテクニック


最初のアプローチはかなり単純でした。マルウェアが名前テストでユーザーをチェックする場合、ジョンと呼びます。レジストリキーSYSTEM \ CurrentControlSet \ Enum \ SCSI \ Disk＆Ven_VMware_＆Prod_VMware_Virtual_Sを使用して仮想環境にいるかどうかを確認する場合、（可能であれば）呼び出しをインターセプトしてキーまたは返された結果を置き換えます。数多くの研究の結果、検出バイパス手法の特定の知識ベースが採用され、その後、環境のセットアップに使用されました。年が経つにつれ、ウイルス作者（だけでなく）がこの知識を仕事に使用するようになり、最終的には次のようになりました。

「これは研究環境ですか？」という質問 「環境は面白いですか？」に置き換えられました。そして、その後のデータ盗難のためにFTPクライアントの痕跡を見つけることに関心はありません。さて、これが特定のソフトウェアを使用する会計担当者のマシンである場合、別のことです。他のアプローチもあります。システムに必要なすべての情報を収集し、それを管理サーバーに送信し、応答でペイロードを受信しない軽量ブートローダーでユーザーを感染させることです。不明なサーバーロジックにより、この被害者は関心がないと判断されました。

つまり、デフォルトで設定されたサンドボックスは効果的なビジネスソリューションではありません。攻撃者はますます検出に対する自己防衛に頼るのではなく、攻撃をさらに発展させるために興味深い標的の探索範囲を狭めることにますます依存しています。

どうやって彼らと戦うのですか？


簡単に言えば、私たちは孤立した環境のユニークなイメージを作ることを提案します。目標は、財務部門、開発中の継続的な統合のためのビルドサーバー、Webサーバー、ドメイン管理者のマシン、そして最後にCEOのステーションなど、攻撃者にとって魅力的なアクセスで、従業員のワークステーションにできるだけ類似した環境を再現することです。

もちろん、そのようなタスクでは、適切なソフトウェアを「搭載」して多数の空白を解放できますが、特定の組織に対応する特定のデータ（ドメイン名、使用するリソースまたはアプリケーション、作業文書の名前と内容）から開始する必要があります-微妙なニュアンスが多いほど、優れています。場合によっては、システムアップデートのバージョンとパッチまで、環境を微調整する必要があります。最も単純なケースでは、イメージは単に「穴がいっぱい」にできます。ただし、ITインフラストラクチャパークが特定のバージョンよりも古い場合、これも考慮に入れられ、特定の数の脆弱性を使用して、無関係な侵入ベクトルの多くを排除できます。

その結果、ウイルス作成者にとって未知であり、実際に関心のある画像が得られるため、標的型攻撃を検出する可能性が大幅に高まります。

「ライブ」コンピュータの動作をエミュレートする必要があるのはなぜですか？


ユーザーアクションのエミュレーションは必須であり、サンドボックスの効果が失われます。場合によっては、アクションがないことでサンドボックス内の存在が検出されます。たとえば、マウスカーソルが長時間その位置を変更しない場合、新しいウィンドウが表示されない、アプリケーションが終了しない、または起動が少なすぎる、新しいファイルが一時ディレクトリに表示されない場合、いいえネットワーク活動。他の状況では、これはマルウェア自体の動作に影響を与えます。たとえば、実行するには、ユーザーがオフィス文書にマクロを含めることに同意する必要がある、またはトロイの木馬が何かに（または何かから）同意する必要がある中間ダイアログボックスを表示する拒否）彼の仕事を続ける。

重要ではないアクションが必要な場合もあります。ユーザーはドキュメントを開き、何かを書き留めるか、パスワードをクリップボードにコピーしてさらに入力する必要があります-まさにこの時点で、スパイトロイの木馬が機能し、重要なデータを傍受して作成者のサーバーに送信します。

これはどのように行われますか？


私たちのソリューションは、生きているユーザーの仕事をエミュレートする新しいアクションで徐々に補充されています。もちろん、複雑な計画されたイベントの事前定義されたシーケンスは、実際のさまざまな用途と決して比較することはできません。私たちは製品のこの側面を研究し改善し続け、その効果を高めています。

上記の関数に加えて、基本的な機能に注目する価値があります。サンドボックスはエージェントレスのクラスに属しています。ほとんどのソリューションでは、仮想マシン内に補助エージェントがあり、システムの状態を管理し、興味深いイベントやアーティファクトをホストサーバーに送受信します。監視の利点とホストマシンとゲストマシン間の相互作用の明確な原則にもかかわらず、このソリューションには重大な欠点があります。それは、エージェントに関連付けられたオブジェクトをマルウェアから隠して保護する必要があることです。イベントプロバイダーがない場合、問題が発生します。仮想マシン内で何が起こっているかについての情報を取得するにはどうすればよいでしょうか。

このために、テクノロジー拡張ページテーブルを使用します（EPT）Intel Corporation。これは、ゲスト物理メモリとホスト物理メモリの間にある中間メモリページです。つまり、これにより次のことが可能になります。

  • ゲストマシンのメモリページの表示を調べます。
  • 興味深いセクションを強調します（たとえば、核機能のアドレスまたはコードを含みます）;
  • 選択したページをマークして、EPTのメモリページへのアクセス権がゲストマシンのページへのアクセス権と一致しないようにします。
  • マークアップされたメモリのセクションへのアピールをキャッチする（この時点でアクセスエラー（#PF）が発生し、結果としてゲストマシンが一時停止されます）
  • 状態を分析し、イベントに関する必要な情報を抽出します。
  • メモリページを正しい状態に再レイアウトします。
  • ゲストマシンを復元します。

発生するすべての監視は、隔離されたマシンの外部で実行されます。内部にあるマルウェアは、観察の事実を検出できません。

サンドボックスでサンプルを実行してその動作を分析することは、複雑な製品のコンポーネントの1つにすぎません。開始後、プロセスメモリで悪意のあるコードがスキャンされ、ネットワークアクティビティが記録されます。これは、5000を超える検出ルールを使用して分析されます。さらに、安全なやり取りを解読することも可能です。

調査中に特定できたすべての侵害の指標（IOC）は、評判リストによってチェックされます。動的分析を受ける前に、サンプルは静的処理のために送信されます。サンプルはいくつかのアンチウイルスで事前にフィルタリングされ、エキスパートセキュリティセンター（PT Expert Security Center）の専門家からの検出ルールで独自のエンジンによってスキャンされます。私たちは、サンプルのメタ情報および埋め込まれたアーチファクトの異常を特定することを含め、包括的な検査を使用します。

PT Sandboxが最も得意とするタスクとその理由は？


PT Sandboxは、いくつかのチームと製品の知識と経験を組み合わせて、標的型攻撃に対抗します。この製品は脅威に対抗するモード（防止）で使用できるという事実にもかかわらず、依然としてITシステムのセキュリティを監視（検出）する手段です。従来のエンドポイント保護ソリューションとの主な違いは、PT Sandboxのタスクは異常に注意を払い、以前は未知であった脅威を登録することです。
