Préparation des données pour régler et former un réseau de neurones pour la détection d'objets.



Nous continuons le sujet de la détection d'objets . Pour que l'algorithme puisse faire la distinction entre les objets ou pouvoir détecter des écarts dans le processus, il doit être formé sur un ensemble de données qui correspond à l'image globale des matériaux vidéo analysés. La préparation d'un ensemble de formation est un processus très long, qui dépend à la fois de la vitesse de formation et de la qualité de l'algorithme. Envisagez l'utilisation de plusieurs outils qui vous aideront à préparer correctement les données.



La première étape de préparation est la compilation d'un premier ensemble de données. En règle générale, plus le volume de données est important et plus la diversité de l'objet analysé est élevée, plus la qualité de l'algorithme est élevée. Si vous avez l'intention d'analyser la vidéo à partir de DVR, il est préférable de compiler un ensemble de données pour prendre des captures d'écran à partir de matériel vidéo existant, car dans ce cas, moins de données et moins de temps de recyclage seront nécessaires. Dans d'autres cas, l'ensemble de données est compilé en tenant compte de l'angle de vue à partir duquel l'objet est capturé, de sa taille, etc., à quel point les objets de l'ensemble d'apprentissage sont similaires aux objets des matériaux qui seront analysés par l'algorithme. L'ensemble de données doit être composé de différentes variantes, à savoir sur lequel un objet et plusieurs seront affichés, par exemple, une personne et un téléphone dans un cadre.



Si la compilation de l'ensemble de données a été effectuée à l'aide d'un appareil photo ou d'un téléphone, les images auront probablement une haute résolution, ce qui affecte négativement le processus d'apprentissage. Le principal problème avec les images haute résolution sera l'erreur «Mémoire insuffisante», même si le paramètre batch_size est petit. Pour réduire la probabilité de cette erreur, il est nécessaire de réduire la taille des images. Le programme IrfanView nous y aidera. Ce programme vous permet de traiter des images par lots, c.-à-d. il vous suffit de spécifier le chemin d'accès aux images source, de définir les paramètres de conversion et de démarrer le processus de conversion. En conséquence, nous obtenons des images «légères» dans le dossier spécifié. Pour ce faire, dans la fenêtre de démarrage du programme, appuyez sur la touche B (mise en page russe "I") ou dans l'onglet Fichier, sélectionnez "Conversion par lots / Renommer",une fenêtre apparaît comme suit:







Ensuite, nous avons besoin de:



  1. sélectionnez le mode de conversion par lots;
  2. sélectionnez le dossier où se trouvent les images source;
  3. configurer les paramètres avancés avancés;
  4. ajouter toutes les images;
  5. sélectionnez un dossier pour le résultat de la conversion;
  6. démarrer la conversion par lots.


Il existe de nombreuses transformations différentes disponibles dans les options avancées, dans cet exemple, nous modifions uniquement la taille de l'image d'origine, réduisons la taille de 10 fois (spécifiez 10% de la valeur d'origine):







Ainsi, si l'image d'origine était de 4160 x 3120, l'image résultante deviendra 416 x 312 pixels.



La formation d'un échantillon de formation est un processus assez fastidieux, que ce soit la formation de captures d'écran à partir de matériel vidéo ou la prise de vue avec un appareil photo et un téléphone. Pour augmenter considérablement le volume de l'ensemble de données, diverses méthodes d'augmentation sont utilisées. Examinons quelques-unes de ces méthodes implémentées dans le langage de programmation Python à l'aide de la bibliothèque Pillow. L'une des méthodes d'augmentation les plus populaires est la «réflexion miroir».



Pour commencer, nous importons les bibliothèques nécessaires: 



from PIL import Image, ImageOps
import os
import numpy


Nous indiquons l'emplacement du répertoire avec les images sources et le chemin du résultat de la conversion: 



path_to_origin = "images/" 
output_folder = "Augmentation_result/"
img_names = os.listdir(path_to_origin)

if not os.path.exists(output_folder):
    os.makedirs(output_folder)


Nous allons écrire une fonction pour convertir l'image en «image miroir»: 



def mirror(path_to_origin, img_name, output_folder):
    im = Image.open(path_to_origin + img_name)
    ImageOps.mirror(im).save(output_folder+"mirror"+img_name, "JPEG")


Une autre méthode d'augmentation populaire est la rotation d'image. Il vaut la peine de considérer avec des objets de taille réelle qu'avec cette méthode d'augmentation, une partie de l'image est perdue. Nous écrivons une fonction pour faire pivoter l'image: 



def rotate(path_to_origin, img_name, output_folder):
    im = Image.open(path_to_origin + img_name)
    im.rotate(-30).save(output_folder+"rotate_"+img_name, "JPEG")


La valeur (-30) est l'angle de rotation de l'image, qui est sélectionné en fonction du type d'objet. Par exemple, une rotation à 180 degrés, s'il y a un téléphone dans l'image, alors c'est une bonne option pour l'augmentation, mais s'il y a une personne dans l'image, alors 180 degrés pour cette image est un mauvais exemple, car il est peu probable que vous voyiez une vidéo dans laquelle les gens marchent à l'envers.



Et enfin, nous allons passer 2 types d'augmentation pour chaque image: 



for img in img_names:
    rotate(path_to_origin, img, output_folder)
    mirror(path_to_origin, img, output_folder)

Il existe de nombreuses méthodes d'augmentation différentes (ajout de bruit, changement de contraste, etc.), tout dépend du type de données traitées. Ainsi, en utilisant deux méthodes d'augmentation, nous avons augmenté la taille des données d'entraînement à trois reprises. La formation de l'ensemble d'apprentissage est terminée, il reste à marquer les objets dans les images.



Pour le balisage, nous utiliserons le programme LabelImage, car à mon avis, il est assez facile à utiliser et multifonctionnel. Ce programme est capable de renvoyer le résultat du balisage dans deux formats (PascalVOC et YOLO). Le premier format est présenté sous la forme d'un fichier XML dans lequel les paramètres de l'image sont indiqués, ainsi que les coordonnées des rectangles qui décrivent l'emplacement des objets dans l'image:







Un tel fichier XML est généré pour chaque image dans laquelle tous les objets marqués sont décrits. Le nom du fichier de balisage est identique au nom de l'image. Le format YOLO est présenté sous la forme d'un fichier texte dans lequel la classe de l'objet et les coordonnées des rectangles sont décrites, décrites par les valeurs x, y, w, h sous forme normalisée:







En plus d'un fichier texte avec balisage pour le format YOLO, un fichier classes.txt est également généré, dans lequel les noms des objets sont indiqués. Les numéros de séquence des noms d'objets dépendent de l'ordre d'apparition lors du marquage des images. Il convient de noter que le fichier classes.txt peut être écrasé lorsque l'application est rouverte, vous devez donc suivre la séquence de saisie des noms dans le programme, sinon une incohérence de coordonnées se produira. Le balisage des objets se produit dans la séquence suivante:







  1. Sélectionnez un répertoire avec des images;
  2. Sélectionnez le répertoire dans lequel les fichiers de balisage seront enregistrés;
  3. Choisissez un format de mise en page (PascalVOC ou YOLO);
  4. Sélectionnez l'outil de sélection d'objet «touche W»;
  5. Sélectionnez un objet dans l'image;
  6. Définissez le nom de l'objet (pendant une opération répétée, vous pouvez sélectionner dans la liste, sous le bouton OK);
  7. Confirmer
  8. Enregistrez «Ctrl + s» (répétez les étapes 4 à 8 pour chaque objet requis);
  9. Passez à l'image suivante.


Une fois terminé, le nombre de fichiers de balisage doit correspondre au nombre d'images portant des noms identiques. Ainsi, nous avons examiné les outils et méthodes qui vous permettent de redimensionner des images, d'augmenter la quantité de données, ainsi que de marquer des objets sur des images qui seront détectées par un réseau de neurones. La formation et la préparation des données est une étape importante et longue dont dépend la réussite de la création de l'algorithme d'apprentissage en profondeur.
