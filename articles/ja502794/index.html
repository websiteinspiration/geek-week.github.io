プログラミングを始めたばかりの多くの人が質問します-他の人のコードをコピーして貼り付けても大丈夫ですか？人が怠惰な生き物であり、プログラマーの時間にお金がかかるとすると、私の意見では答えは明白です-はい。しかし、この原則を馬鹿げた点まで持っている人、考えずに理解せずにコピーする人、そして卒業証書を書く人にも、このエッセイはこのキーボードと記事に捧げられています。そして、はい、このキーボードには、コピーと貼り付けの2つのボタンが含まれています。

次に、digisparkがキーボードになりすまして、ボタンを押してCTRL + C / CTRL + Vを送信する必要があります。

したがって、クリックをキャッチする必要があります。まず、中央のワイヤを+ 5Vに接続し、サイドワイヤを入力用に構成されたピンに接続すると、ボタンが押されたときに、はい、ピンに1が表示されますが、ピンが押されていない場合、干渉をキャッチし、何が何であるかを追跡できません押すこと、そして何という障害か。この問題は驚くほど簡単に解決されます。大きな抵抗（数十オーム）を介してピンをグランドに引き込む必要があります。マイクロコントローラのデータシートを開くと、内部でリフトをオンにできますが、VCC（+電源）までしか確認できません。実際、これは問題ではありません。中央のピンをGND（グラウンド）に接続し、作業のロジックを反転するだけです。

このように想像できます。

ただし、マイクロコントローラ全体を備えているため、3つの方法があります：遅延
バウンスに対処する方法の1つは、一定量のバウンスの後でボタンがまだ押されているかどうかを確認することです。遅延を介して実装した場合、コードはブロックされますが、これを正しく行う方法は、バウンスとarduinoとの戦いに関する数百の記事で確認できます。

それで、ボタンが押されていることに気づきました-次は？CTRL + C / Vが押されていることをコンピューターに伝えます。

これは、digisparkの標準ライブラリを使用して行われます。

結果のコードは次のようになりました：

#include "DigiKeyboard.h" // library for emulate keyboard
#define COPY 2 // copy button pin
#define PASTE 0// paste button pin
#define GND 1// center must be connected to ground
void setup() {
  pinMode(COPY, INPUT); // set pin to input
  digitalWrite(COPY, HIGH);// turn on pullup
  pinMode(PASTE, INPUT);
  digitalWrite(PASTE, HIGH);
  pinMode(GND, OUTPUT); // set pin to output
  digitalWrite(GND, LOW);// and set on pin GND
}


void loop() {
  // if something COPY or PASTE clicked
  if (!digitalRead(COPY) || !digitalRead(PASTE)) {
    delay(30); // wait(debounce)
    if (!digitalRead(COPY)) { // if copy still pressed send CTRL+C
      DigiKeyboard.sendKeyStroke(KEY_C , MOD_CONTROL_LEFT);
      delay(30);
    }
    else if (!digitalRead(PASTE)) {// if paste still pressed send CTRL+V
      DigiKeyboard.sendKeyStroke(KEY_V , MOD_CONTROL_LEFT);
      delay(200);
    }
  }
}
