CSSグリッドの命名

グリッドレイアウトの使用方法を初めて学ぶときは、グリッド上に行番号で要素を配置することから始めます。この場合、特定の行がグリッドのどこに配置されるかを監視する必要があります。また、サイトが右から左に書かれる言語で表示される場合、それらの番号の順序が逆になることに注意してください。

この線システムに基づいて、線やグリッド領域に名前を付けることを可能にする技法が機能します。これらの手法を使用すると、アイテムを番号ではなく名前で配置できます。この記事では、CSSグリッドレイアウトで行と領域に名前を付けるさまざまな方法と、それらが作成するいくつかの興味深い機能について詳しく説明します。

行の命名


グリッド線に名前を付けることから始めましょう。以下の例では、6つの明示的な列トラックと1つの明示的なライントラックを持つグリッドがあります。要素は行番号を使用してグリッドに配置されます。

.grid {
  display: grid;
  grid-gap: 20px;
  grid-template-rows: 20vh ;
  grid-template-columns: 1fr 2fr 1fr 2fr 1fr 2fr;
}

.header {
  grid-row: 1;
  grid-column: 1 / -1;
}

.sidebar {
  grid-row: 2;
  grid-column: 1 / 3;
}

.content {
  grid-row: 2;
  grid-column: 3 / -1;
}

必要に応じて、トラックリストの角括弧内にライン名を指定できます。ここでは、ラインに続くトラックではなく、ラインに名前を付けることを覚えておくことが重要です。線に名前を付けたら、グリッドに要素を配置するときに、番号の代わりにその名前を使用できます。

.grid {
  display: grid;
  grid-gap: 20px;
  grid-template-rows: [header-start] 20vh [header-end] ;
  grid-template-columns: [sidebar-start] 1fr 2fr [sidebar-end] 1fr 2fr 1fr 2fr;
}

.header {
  grid-row: header-start;
  grid-column: 1 / -1;
}

.sidebar {
  grid-row: 2;
  grid-column: sidebar-start / sidebar-end;
}

名前はキーワード以外の何でもかまいませんspan。この記事の後半で説明する理由から、-start先頭行（行または列の行であることは関係ありません）および-end末尾行にはサフィックスを使用して名前を付けることをお勧めします。最後に、それは判明しなければならないmain-startし、main-endかsidebar-startとsidebar-end。

多くの場合、グリッドの1つの部分の終了線と他の部分の開始線は同じですが、線には複数の名前が付いている可能性があるため、これは問題ではありません。角括弧内のスペースを介して追加することにより、複数の名前の行を指定できます。

.grid {
  display: grid;
  grid-gap: 20px;
  grid-template-rows: [header-start] 20vh [header-end] ;
  grid-template-columns: [full-start sidebar-start] 1fr 2fr [sidebar-end main-start] 1fr 2fr 1fr 2fr [main-end full-end];
}

.header {
  grid-row: header-start;
  grid-column: full-start / full-end;
}

.sidebar {
  grid-row: 2;
  grid-column: sidebar-start / sidebar-end;
}

.content {
  grid-row: 2;
  grid-column: main-start / main-end;
}

この例は、個々のグリッド線に名前を付ける必要がないこと、および名前に加えて使用できる線番号が常に利用できることを示しています。

Codepen 1. 行の命名

同じ名前の行。


行に複数の名前を付ける方法を説明しましたが、同じ名前の行が複数ある場合もあります。これは、関数を使用してrepeat()、トラックのリストに名前付きの行を含めると発生します。次の例では、交互に名付け6つのという名前の行を作成col-a-startしてcol-b-start。

.grid {
  display: grid;
  grid-gap: 20px;
  grid-template-columns: repeat(3, [col-a-start] 1fr [col-b-start] 2fr);
}

要素を配置するときに指定した場合、要素col-a-startは最初に一致した行に名前で配置されますcol-a-start（例では、最初のグリッド線になります）。同じ原則に従って、要素を配置するときに指定すると、要素col-b-startは2番目のグリッドラインに配置されます。

後続の行に配置する必要がある場合は、名前の後に番号を追加して、アイテムを同じ名前の行のどの行に配置するかを示します。

.box3 {
  grid-row: 2;
  grid-column: col-a-start 2 / col-b-start 3;
}

Codepen 3. 名前を付ける：名前付き行の位置を再定義する

名前付きリージョン


その前は、名前付きの行のみを考慮していましたが、別の方法があります。グリッド領域に名前を付けることができます。

グリッド領域は、1つ以上のグリッドセルで構成される長方形の領域です。領域は、列と行の開始線と終了線を示す4つのグリッド線で定義されます。


グリッド領域には、プロパティを使用して名前を付けますgrid-template-areas。このプロパティは、ASCIIテーブルの形式でレイアウトを記述し、一連のラインであり、それぞれがグリッドの個別のライントラックを記述します。

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr 2fr) ;
  grid-template-areas: 
"head head head head head head"
"side side main main main main"
"foot foot foot foot foot foot";
}

プロパティ文字列で使用する名前は、プロパティを使用しgrid-template-areasてグリッドの直接の子に割り当てられますgrid-area。このプロパティの値は、命名に使用される場合、いわゆるユーザー識別子であるため、引用符で囲む必要はありません。

.header {
  grid-area: head;
}

.sidebar {
  grid-area: side;
}

.content {
  grid-area: main;
}

.footer {
  grid-area: foot;
}

プロパティを使用してグリッド構造を説明するときにgrid-template-areas領域が複数のセルをカバーするようにするには、行または列に沿ってその識別子を数回再記録する必要があります。作成される領域は長方形である必要があり、LおよびT字型の領域は許可されません。名前ごとに長方形の領域を1つだけ作成することもできます。切断された領域は許可されません。仕様には次のように記載されています。
「このモジュールの将来のバージョンでは、非長方形または切断された領域が許可される可能性があります」
- プロパティgrid-template-areas

グリッドを説明するときは、その完全な表示に注意する必要があります。そうしないと、広告全体が無効として破棄されます。つまり、各グリッドセルを塗りつぶす必要があります。

grid-template-areas: 
"head head head head head head"
"side side main main main main"
"foot foot foot foot foot foot";
}

マークアップ内のいくつかのセルを空のままにしたい場合があるため、仕様では、ポイントシンボル.またはポイントのシーケンスを、スペース....のない空のセルのマーカーとして定義します。

grid-template-areas: 
"head head head head head head"
"side side main main main main"
"....... ....... foot foot foot foot";
}

Codepen 7. リージョン名と同等のライン名

グリッド領域のプロパティの説明


プロパティについて何か知っていると便利ですgrid-area。基本的に、grid-area識別子などと共に使用する場合mainは、エリアの4行すべてをどのように記述するかです。プロパティの有効な値grid-areaも行番号です。

.main {
  grid-area: 2 / 1 / 4 / 3;
}

これはエントリと同じ効果があります。
.main {
  grid-row-start: 2;
  grid-column-start: 1;
  grid-row-end: 4;
  grid-column-end: 3;
}

あなたが尋ねるとき
.main {
  grid-area: main;
}

これは実際には次のことを意味します。
.main {
  grid-area: main / main / main / main ;
}

プロパティgrid-areaは、行番号ではなく、ユーザーIDを使用した場合の動作が少し異なります。プロパティの初期値に行番号を使用する場合grid-area、指定しない終了行番号は自動的にに設定されるautoため、要素を配置する場所を決定するために自動配置が使用されます。

ただし、ユーザーIDを使用して一部の行をスキップすると、次のように設定されます。

3つのライン名が定義されています


.main {
  grid-area: main / main / main ;
}

3つのライン名を指定した場合、基本的には存在しませんgrid-column-end。それは場合はgrid-column-startユーザ識別子で、同じ識別子が設定されていますgrid-column-end。我々はすでに見てきたように、プロパティは、-end地域の端面を使用しますmainので、もし、grid-column-startとgrid-column-end同じ名前が与えられ、コンテンツがこの地域のすべての列に延伸されます。

指定された2つの回線名


.main {
  grid-area: main / main  ;
}

名前を2つだけ指定すると、行と列の開始行が設定されます。場合grid-column-start及び/又はgrid-row-startユーザ識別子であり、次いで、grid-column-endそれはgrid-row-end、それぞれ、同じ値に設定します。

指定された1つの回線名


.main {
  grid-area: main  ;
}

1つの線名を設定することは、grid-areaを領域のメイン名に設定するときに行うこととまったく同じです。この場合、4行すべてがこの値に設定されます。

注：これはとで動作grid-columnしgrid-rowます。

この方法は、実際にはグリッド内の列または行のセットを指定して要素を配置できることを意味します。値だけよう-endプロパティのは、grid-area（省略）の初期値、最終値と同じ値に設定されgrid-column且つ特性も振る舞いますgrid-row。つまり、以下をmain使用して、領域の開始列行と終了列行の間に要素を配置できます。

.main {
  grid-column: main  ;
}

グリッドおよびグリッドテンプレートプロパティの命名と省略形


CSSグリッドには2つの略語があり、1つのコンパクトな構文で多くのグリッドプロパティを設定できます。個人的には、それを理解するのは難しいようです。これについて他の開発者と話し合うと、意見は分かれます。それらのようなものもあれば、個別のプロパティを使用することを好むものもあります。他の略語の場合と同様に、使用しないプロパティの値は元の値にリセットされることに注意してください

グリッドテンプレートの省略形：明示的なグリッドの作成

次に、ライン名を追加することができます。それらは値の各行の最初と最後に配置されます-値の各行はライントラックを表すことに注意してください。文字列の名前は、プロパティで行に名前を付ける場合と同様に、角かっこ内にあるgrid-template-rows必要があり、ライントラックを定義する値の文字列を囲む引用符の外側にある必要があります。

グリッドの略称：明示的および暗黙的なメッシュ


仕様では、暗黙的なグリッドを個別に定義する必要がある場合は、grid代わりに省略プロパティを使用する必要があると想定していますgrid-template。省略形gridは、設定していないすべての暗黙的な値をリセットします。したがって、次のプロパティを設定およびリセットできます

grid-template-columns
grid-template-rows
grid-template-areas
grid-auto-columns
grid-auto-rows
grid-auto-flow

ここでは、暗黙のグリッド定義を定義gridしているため、省略形の使用はそれを使用することと同じになりgrid-templateます。唯一の違いは、プロパティのリセットgrid-auto–*です。省略形gridは、明示的なグリッドを設定して暗黙的なプロパティをリセットするか、暗黙的なグリッドを設定して明示的なプロパティをリセットするために使用できます。両方を同時に行うことはあまり意味がありません

.grid {  
  display: grid;
  grid-gap: 20px;
  grid: 
" head head head head head head" 10vh
[panel-start] "side side main main main main" minmax(200px, auto)

/ [full-start ] 1fr  [panel-start] 2fr 1fr 2fr 1fr [panel-end] 2fr [full-end];

注：CSSグリッド仕様の元の推奨バージョンでは、この省略形はセルgrid-column-gapとの間の間隔もリセットしgrid-row-gapます。ただし、この動作は最終的に変更されました。ブラウザーはこの動作を更新しますが、執筆時点（2017年10月）ではgap、削減が使用されたときにプロパティの値がゼロにリセットされる状況が依然として発生する可能性があるため、後で再定義する必要がある場合があります

どちらのアプローチを使用しますか？


これらすべての方法のうち、特定のタスクに最適な方法はどれですか。厳密なルールはありません。個人的にgrid-template-areasは、テンプレートライブラリで作業するときにコンポーネントに使用するのが好きです。 CSSでコンポーネントの構造を目の前に置くと便利です。grid-template-areasテスト時にさまざまなレイアウトオプションを簡単に試すことができます。メッシュセルのこのような単純な動きのため、コンポーネントセルの視覚的および論理的な順序が乱されないようにすることが重要です。キーボードを使用してサイトを操作し、タブボタンで要素を切り替える訪問者は、マークアップで定義されている順序で要素を選択します。コンテンツを表示するための最も適切な方法を決定したら、すぐにレイアウトを並べ替えることを忘れないでください。この問題の詳細については、CSSグリッドのレイアウトとアクセシビリティに関する記事を読むことをお勧めします。
