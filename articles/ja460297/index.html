WeakRef-ECMAScript標準に追加するための提案

電報チャンネルに小さな投稿を書いた。Habrの読者には興味があるのではないかと思いました。


最近、新しいWeakRef（ステージ3）プロザルに捧げられたV8ブログの記事-「弱い参照とファイナライザ」。


その本質を自分の言葉で例を挙げて説明してみます。たとえば、ブラウザで何らかの画像処理を行っているとします。たとえば、透かしがそれらに適用され（同意する、この例はあまり現実的ではありません）、これらの画像が何らかの方法で使用されます。透かしは、CPUを集中的に消費する関数によって重ね合わされます。画像は繰り返すことができるので、もう一度プロセッサを読み込まないようにするために、透かしを含む画像キャッシュをで作成しMap、画像ファイルの名前をキーにします。しかし、ここでは、一部の画像が使用されない場合に問題が発生しMapます。キーがそれを参照するため（強い参照）、画像はメモリに残ります。したがって、私たちにMap不必要なメモリを使い果たしたわけではありません。そのような状況を何らかの形で特定し、手でキャッシュを消去する必要があります。これはあまり便利ではありません。


これがWeakRefが助けになるところです。WeakRefを使用すると、画像への弱いリンクを作成し、画像自体の代わりにキーで書き込むことができます。


const wr = new WeakRef(image);
cache.set(name, wr);
//   
const ref = cache.get(name);
const image = ref.deref();

この場合、ガベージコレクターは、キャッシュ内のイメージが不要になった状況を個別に判断し、メモリをクリアできます。プロポーザルのMapからキーをクリアするために、追加のFinalizationGroup APIを使用することが提案されています。


興味深い事実。記事の冒頭で、すでに標準に含まれているWeakMapとWeakSetの概要を説明します。WeakMapで使用される関係の最も正式な名前はエフェメロンであることがわかります。
