Functoolsは、高次関数で動作するように設計されたPythonライブラリです。そのような関数は他の関数を取り、関数を返すことができます。これらは、再利用可能なコードを開発者が作成するのに役立ちます。関数は、完全に書き換えることなく使用または拡張できます。Python のfunctoolsモジュールには、説明した効果を実現するためのさまざまなツールが用意されています。たとえば、次のとおりです。

Pythonの部分的な機能


functools モジュールでは、部分関数は最も重要なツールの1つと見なされています。部分関数を使用すると、引数が既に渡されている既存の関数を置き換えることができます。さらに、高品質のドキュメントを追加して、関数の新しいバージョンを作成することもできます。

部分的な引数を渡すことで新しい関数を作成できます。また、新しいオブジェクトの出現につながる関数の引数を凍結することもできます。partialを表すもう1つの方法は、その助けを借りて、デフォルト値を持つ関数を作成できることです。部分的には、固定としてキーワードと位置引数をサポートします。
例を見てみましょう。

部分関数を作成するにはどうすればよいですか？


部分的な関数 を作成するにpartial()は、ライブラリからfunctoolsを使用します。それは次のように書かれています：

partial(func, /, *args, ** kwargs)

したがって、funcを呼び出す部分的な関数を作成し、固定のキーワードと位置引数を渡します。ここでは、通常func関数を呼び出すために必要な引数がいくつか渡されます。残りの引数は* argsおよび** kwargsで渡されます。 以下の関数が2つの数値を加算するとします。



def multiply(x, y):
 
    return x * y

ここで、特定の数値を2倍または3倍にする必要がある場合を考えてみましょう。この場合、次のように新しい関数を定義します。

def multiply(x, y):
        return x * y
 
def doubleNum(x):
       return multiply(x, 2)
 
def tripleNum(x):
       return multiply(x, 3)

もちろん、関数のスクリプトが2〜3しかない場合は、上記のように行う方が論理的です。しかし、そのような関数をさらに100個作成する必要がある場合、同じコードを何度も書き直すのは意味がありません。これは部分的な関数が便利になるところです。それらを使用するには、まずFunctoolsからパーシャルをインポートする必要があります。

from functools import partial
 
def multiply(x, y):
       return x * y
 
doubleNum = partial(multiply, 2)
   tripleNum = partial(multiply, 3)
 
Print(doubleNum(10))
 
Output: 20

例からわかるように、デフォルト値は左側の変数に置き換えられます。doubleNum（10）を呼び出すと、xの代わりに2があり、yの代わりに10があります。この例では、順序は重要ではありませんが、他の用途では重要になる場合があります。変数置換の順序を理解するために、このケースの例を見てみましょう。

from functools import partial
def orderFunc(a,b,c,d):
      return a*4 + b*3 + c*2 + d
 
result = partial(orderFunc,5,6,7)
print(result(8))
 
Output: 60

4で3 を2でorderFunc()乗算し、合計に値を追加 する関数があります。 私たちは、作成した部分的な機能を呼び出すことを今値5、6、7値5、6および7とは、変数に置き換えられます、そしてそれに応じて。変数は、呼び出されたときに送信されるため、8に置き換えられます。結果は（4 * 5 + 6 * 3 + 7 * 2 + 8）= 60です。abcd

この場合、送信される値の順序が重要になります。順序が変わると結果も変わるためです。変数を修正するには、位置引数の代わりにキーワードを使用できます。引数としてキーワードを使用して上記のコードを書き換えましょう。

from functools import partial
def orderFunc(a,b,c,d):
       return a*4 + b*3 + c*2 + d
 
 result = partial(orderFunc,c=5,d=6)
print(result(8,4))
 
Output: 60

ここでは、変数の値5を固定c変数のと6 d。変数の代わりにa、b値8と4が表示されます。その結果、（8 * 4 + 4 * 3 + 5 * 2 + 6）= 60になります。

部分関数はループで決定でき、繰り返し計算に使用できます。例を見てみましょう：

from functools import partial
 
def add(x,y):
      return x + y
 
add_partials = []
for i in range (1, 10):
      function = partial(add, i)
      add_partials.append(function)
      print('Sum of {} and 2 is {}'.format(i,add_partials[i-1](2)))
 
  
Output:
 
Sum of 1 and 2 is 3
Sum of 2 and 2 is 4
Sum of 3 and 2 is 5
Sum of 4 and 2 is 6
Sum of 5 and 2 is 7
Sum of 6 and 2 is 8
Sum of 7 and 2 is 9
Sum of 8 and 2 is 10
Sum of 9 and 2 is 11

この例では、既存の関数を再利用して、特定の範囲の値を2で要約します。ループでpartialを呼び出し、その機能を使用して合計を計算できます。出力の値からわかるように、1から10までのサイクルがあり、この間隔のすべての値は、加算関数を呼び出す部分関数を使用して2に加算されます。

メタデータ


部分関数は独立し ているという事実にもかかわらず、それらは拡張する関数のメモリ（メタデータ）を格納します。

from functools import partial
 
def add(x,y):
      return x + y
 
# create a new function that multiplies by 2
result = partial(add,y=5)
print(result.func)
print(result.keywords)
 
Output:
<function add at 0x7f27b1aab620>
{'y': 5}

最初の呼び出しfuncが関数の名前とメモリ内のアドレスを渡します、とと2回目の呼び出しのキーワードは、関数にキーワードを渡します。したがって、部分的な関数は、拡張可能な関数から受け取ったメタデータを使用して自己文書化と呼ぶことができます。functoolsの 別のツールを使用して関数メタデータを更新できます。機能のメタデータを更新するために使用できるツールです。例でそれを理解しましょう。

update_wrapper partial

Update_wrapper

def multiply(x, y):
 
    """Test string."""
 
    return x * y
 
 result = functools.partial(multiply, y=2)
 
 try:
 
    print ('Function Name:'+result.__name__)
 
except AttributeError:
 
    print('Function Name: __no name__')
 
 print ('Function Doc:'+result.__doc__)
 
 print('Updating wrapper:')
 
functools.update_wrapper(result, multiply)
 
 print ('Function Name:'+result.__name__)
 
print ('Function Doc:'+result.__doc__)
 
Output:
 
Function Name: __no name__
 
Function Doc:partial(func, *args, **keywords) - new function with partial application
 
    of the given arguments and keywords.
 
 
Updating wrapper:
 
Function Name: multiply
Function Doc: Test string.

これで、出力からわかるように、ラッパーを使用する前は、関数に名前またはドキュメントが割り当てられていませんでした。nameとdoc関数をで更新するとすぐにupdate_wrapper、対応する結果が出力に表示されました。
