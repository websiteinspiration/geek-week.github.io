Angular要素を使用したマイクロフロントエンドの作成：初心者向けガイド

過去10年間で、フロントエンド開発ははるかに複雑になりました。純粋なHTML / CSSから、高い対話性、アクセシビリティ、テスト容易性、セキュリティなどのトピックまでです。これらのニーズを満たすために、ほとんどの開発チームはバックエンドチームとフロントエンドチームに分かれています。



これに加えて、アプリケーションの機能は着実に成長しており、ある時点で、1つのコードベースで複数のチームを組み合わせるのは不便になります。
ファッショナブルな用語「マイクロフロントエンド」は、増大するフロントエンドコードを保守しやすい部分に分割するアプローチを意味するようになりました。フロントエンドは、独立したチームによってリリースおよびデプロイされるいくつかの機能またはパーツに分割されます。これにより、テスト容易性、再利用性が向上し、各マイクロインターフェースに異なるテクノロジーを選択できるようになります。


これ以上前戯せずにここで終了します。Angular要素を使用してマイクロフロントエンドの例を作成しましょう。


マイクロフロントエンドの作成


この記事では、旅行予約システムの例を作成します。CLIを使用して2つのAngularプロジェクトをデプロイしましょう：travel-bookingとflight-booking。


カスタムAngular要素を構築して実行するには、いくつかの依存関係が必要です。Flight-Booking内に次の依存関係をインストールします。


ng add @angular/elements
ng add ngx-build-plus

これらの依存関係は、NPMを介してインストールすることもできます。@ angular / elementsは、Angular要素をサポートします。ngx-build-plusは、プロジェクトをビルドするためのAngular CLIの拡張機能です。


注：この問題で説明されているように、/flight - booking/package.jsonでdocument-register-elementモジュールのバージョンを1.8.1にアップグレードする必要がある場合があります。

  ]

  }  ngDoBootstrap() {
    const myCustomElement = createCustomElement(BookingComponent, { injector: this.injector });
    customElements.define('app-flight-booking', myCustomElement);
  }
}
