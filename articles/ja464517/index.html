地理空間データの操作と地図の表示は、多くのビジネスアプリケーションの不可欠な部分です。それは、都市および地域の情報システム、石油およびガス産業への応用、輸送インフラ管理システム、さらには配送サービスなどです。私たちのCUBAプラットフォームでは、そのようなアプリケーションを構築するために、箱から出して提供される基本的な機能に加えて、かなり広範なアドオンとコンポーネントのセットがあります。それらの1つは、チャートとマップです。これにより、グラフの表示に加えて、Googleマップをアプリケーションの視覚的な部分に統合できます。昨年、Googleはマップサービスの利用規約を更新しましたが、これはコストの増加につながり、APIを使用するための支払いプロファイルが必須であるという条件も導入しました。これらの状況により、ほとんどのお客様は代替のカードプロバイダーについて考えるようになり、新しいカードコンポーネントを開発するように勧められました。

 @Geometry
 @MetaProperty(datatype = "GeoPoint", related = {"latitude", "longitude"})
 public Point getLocation() {
   if (getLatitude() == null || getLongitude() == null) {
       return null;
   }
   return GeometryUtils.createPoint(getLongitude(), getLatitude());
 }

 @Geometry
 @MetaProperty(datatype = "GeoPoint")
 public void setLocation(Point point) {
   Point prevValue = getLocation();
   if (point == null) {
       setLatitude(null);
       setLongitude(null);
   } else {
       setLatitude(point.getY());
       setLongitude(point.getX());
   }
   propertyChanged("location", prevValue, point);
 }
