OpenLinux als Teil von SIM7600E-H-Modulen





Der Mechanismus zum Entwickeln und Laden einer benutzerdefinierten Anwendung in das Modul ist sowohl unter Linux- als auch unter Windows-Betriebssystemen verfügbar. In diesem Artikel werden wir detailliert untersuchen, wie die Beispiele aus dem SDK von SIMCom Wireless Solutions zum Kompilieren und Laden der Benutzeranwendung in das Modul verwendet werden.



Bevor ich diesen Artikel schrieb, bat mich ein Freund, weit entfernt von der Entwicklung für Linux, so weit wie möglich mit der Frage der Beschreibung des Entwicklungsprozesses meiner eigenen Anwendung für das SIM7600E-H-Modul fortzufahren. Das Kriterium für die Beurteilung der Verfügbarkeit der Einreichung von Material war der Ausdruck "damit ich es verstehe".



Ich lade Sie ein, sich mit dem Geschehen vertraut zu machen.

Der Artikel wird regelmäßig aktualisiert und aktualisiert.




Auftakt



Typischerweise werden zellulare Kommunikationsmodule nur für Datenübertragung, Sprachanrufe, SMS und dergleichen verwendet. All dies erfolgt über AT-Befehle, die von einem externen Steuerungsmikrocontroller gesendet werden. Es gibt jedoch eine Kategorie von Modulen, mit denen Sie benutzerdefinierten Code ausführen können, der von außen geladen wird. In einigen Fällen wird dadurch das Gesamtbudget des Geräts erheblich reduziert, sodass Sie einen einfacheren (und auch kostengünstigen) Mikrocontroller auf die Platine setzen oder ganz aufgeben können. Mit dem Aufkommen von LTE-Modulen, die von Android oder Linux gesteuert werden, und ihren leistungsstarken Ressourcen können Sie alle Probleme lösen, die gängigen Prozessoren zur Verfügung stehen. Dieser Artikel befasst sich mit SIM7600E-H, das von Linux verwaltet wird. Wir werden uns ansehen, wie eine ausführbare Anwendung heruntergeladen und ausgeführt wird.



In vielerlei Hinsicht basiert das Material auf dem Dokument „SIM7600 Open Linux Development Quide“, aber einige Ergänzungen, insbesondere die russische Version, werden nützlich sein. Der Artikel hilft denjenigen, die gerade erst mit dem Erlernen des Moduls beginnen, zum Download der Demo-Anwendung zu gelangen und die erforderlichen Fähigkeiten für die weitere Arbeit zu vermitteln.



Kurz darüber, wer SIM7600E-H ist



SIM7600E-H ist ein Modul, das auf dem 1,3-GHz-Prozessor ARM Cortex-A7 von Qualcomm basiert und über ein Linux-Betriebssystem (Kernel 3.18.20) verfügt, das mit europäischen (einschließlich russischen) Frequenzbereichen 2G / 3G / arbeiten kann. Cat.4-unterstütztes LTE mit maximalen Download-Geschwindigkeiten von bis zu 150 Mbit / s und Uploads von bis zu 50 Mbit / s. Die reichhaltigen Peripheriegeräte, der industrielle Temperaturbereich und das Vorhandensein einer integrierten GPS / GLONASS-Navigation decken alle Anforderungen an eine moderne modulare Lösung im M2M-Thema ab.



Systemübersicht



Das SIM7600E-H-Modul basiert auf dem Linux-Betriebssystem (Kernel 3.18.20). Das Dateisystem basiert wiederum auf dem journalisierten Dateisystem UBIFS (Unsorted Block Image File System).



Wichtige Funktionen dieses Dateisystems sind:



  • Arbeitet mit Partitionen und ermöglicht das Erstellen, Löschen oder Ändern der Größe.
  • sorgt für die Ausrichtung der Aufzeichnung über das gesamte Volumen des Mediums;
  • arbeitet mit schlechten Blöcken;
  • minimiert die Wahrscheinlichkeit eines Datenverlusts während eines Notstromausfalls oder anderer Ausfälle;
  • Protokollierung.


Die Beschreibung stammt von hier und wird an derselben Stelle ausführlicher über ein solches Dateisystem beschrieben.



Jene. Diese Art von Dateisystem ist ideal für raue Betriebsbedingungen des Moduls und mögliche Stromprobleme. Dies bedeutet jedoch nicht, dass die Bedingungen einer instabilen Stromversorgung die erwartete Betriebsart des Moduls sind. Dies weist nur auf eine bessere Lebensfähigkeit des Geräts hin.

Mittwoch (nicht Wochentag)



SIMCom Wireless Solutions bieten Entwicklern die Möglichkeit, die bekannteste Entwicklungsumgebung für Linux oder Windows auszuwählen. Wenn es sich um eine ausführbare Anwendung auf einem Modul handelt, ist es besser, Windows zu wählen. Es wird schneller und einfacher. Wenn Sie eine komplexe Anwendungsarchitektur und nachfolgende Upgrades wünschen, ist es besser, Linux zu verwenden. Wir brauchen auch Linux, um die ausführbaren Dateien für das spätere Laden in das Modul zu kompilieren. Eine virtuelle Maschine reicht völlig zum Kompilieren aus.



Von dem, was Sie benötigen, steht es nicht öffentlich zum Download zur Verfügung - dem SDK, das Sie bei Ihrem Händler anfordern können.

SDK entpacken und kompilieren



Nachdem wir Zugriff auf die Shell erhalten haben und mit der Befehlszeile des Moduls arbeiten können, werden wir versuchen, unsere erste Anwendung zum Laden in das Modul zu kompilieren.



Viele können Schwierigkeiten damit haben! weil Das Modul läuft unter dem Linux-Betriebssystem, um Kollisionen beim Kompilieren von Code unter Windows zu vermeiden. Am besten kompilieren Sie in der nativen Umgebung - Linux.



Wir werden nicht näher darauf eingehen, wie Sie es in Abwesenheit von Linux und dem Wunsch, es auf Ihrem Computer zu installieren, auf einem virtuellen Computer installieren können. Wir werden VirtualBox verwenden, Ubuntu Version 20.04 (die aktuelle Version zum Zeitpunkt des Schreibens) installieren und bereits darunter mit Compilern, SDKs usw. arbeiten.



Wir wechseln zur Linux-Umgebung und entpacken das vom Distributor erhaltene Archiv. 



simcom@VirtualBox:~/Desktop/OpenLinux$ sudo tar -xzf MDM9x07_OL_2U_22_V1.12_191227.tar.gz


Wechseln Sie in das Verzeichnis sim_open_sdk und fügen Sie die Umgebung hinzu: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ cd sim_open_sdk
simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ source sim_crosscompile/sim-crosscompile-env-init


Wir bleiben im selben Ordner und führen die nachfolgenden Befehle aus, während wir uns darin befinden.

Installieren Sie die libncurses5-dev-Bibliothek, wenn sie nicht installiert wurde: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get update && sudo apt-get install libncurses5-dev -y


Python, falls es nicht ebenfalls installiert wurde: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get install python -y


und gcc: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ sudo apt-get install gcc


Zusammenstellung:



Jetzt müssen wir mehrere Dateien kompilieren und die folgenden Befehle nacheinander ausführen.



Wenn das Kernel-Konfigurationsfenster während der Kompilierung angezeigt wird, wählen Sie einfach Beenden und kehren Sie zur Konsole zurück. Wir müssen den Kernel jetzt nicht konfigurieren.



Wir führen aus: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make


Bootloader kompilieren: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make aboot


Kompilieren Sie den Kernel: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel_menuconfig
simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel


Kompilieren Sie das Root-Dateisystem: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make rootfs


Für Linux-Benutzer ist es wichtig, den Modultreiber zu kompilieren: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make kernel_module


Kompilieren Sie die Demo: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ make demo


Danach werden mehrere neue Dateien im Verzeichnis sim_open_sdk / output angezeigt: 



simcom@VirtualBox:~/Desktop/OpenLinux/sim_open_sdk$ ls output/
appsboot.mbn  boot.img  demo_app  helloworld  system.img

C:\>adb push C:\demo_app /data/
