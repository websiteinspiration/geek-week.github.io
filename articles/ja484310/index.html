初心者の目から見たWebix JavaScriptライブラリ。パート2.フォームを操作する



私は初心者のフロントエンド開発者です。現在、ミンスクのIT企業で勉強とトレーニングをしています。私はWebix JSライブラリを例として使用してweb-uiの基本を学んでいます。ささやかな経験を共有し、この興味深いUIライブラリの小さなチュートリアルとして保存したいと思います。

2番目のタスク


最後の課題では、Webixライブラリを使用してページを作成しました。そして今、私はフォームとのユーザー対話のために次のロジックを設定する必要があります：

  • 読み取りとフォームの検証。
  • 金型洗浄 ;
  • ボタンクリック処理 ;
  • 対話型メッセージと確認ウィンドウ。
  • テーブルにフォームデータを追加します。

記事で使用する主なウィジェットはフォームです。また、ドキュメントで読むこともできます。

ソースリンクをダウンロードできます。

結果のデモアプリケーションはここにあります。

手順1.新しいデータをテーブルに追加します。


コードを読んでプロジェクトを論理ブロックに分割するために、functions.jsファイルを作成し、index.htmlファイルにそのパスを示します。その中で、アプリケーションの動作を担当する関数を設定します。

<body>
    <script src="data.js"></script>
    <script src="functions.js"></script>
    <script src="script.js"></script>
</body>

新しいファイルで、テーブルに新しいデータを追加するaddItem関数を作成します。

const addItem = () =>{                                                       
    var item_data = $$("film_form").getValues();
    $$("film_list").add(item_data);
}

Webixライブラリの要素にアクセスするには、$$メソッドを使用します。このメソッドには要素のIDが渡されます。上記のコードでは、フォームは-構文を使用してアクセスされ$$("film_form")ます。
addItem関数は次のように機能します。フォームからgetValuesメソッドを使用して現在のデータを取得し、item_data変数に書き込みます。このデータを使用して、addメソッドを使用してシートに新しいレコードを作成します。

"button"値 を持つウィジェットをクリックしてこの関数を呼び出す"Add new"には、クリックプロパティを設定します。

margin:10,
cols:[
    { view:"button", id:"btn_add", value:"Add new", css:"webix_primary",
    click:addItem},
    { view:"button", id:"btn_clear", value:"Clear"}
 ]

その結果、フォームのデータをテーブルに追加する必要があります。


ステップ2.フォームの検証。ポップアップメッセージ


新しいデータを追加する前に、入力フィールドからの情報を確認（検証）する必要があります。検証は、データを操作する直前にaddItem関数内で呼び出すvalidate（）メソッドによってトリガーされます。また、検証の成功に関するポップアップメッセージのコードもここに配置します。 Webixの単純なメッセージは、メッセージのテキストをパラメーターとして取るwebix.message（）関数によって作成されます。メッセージは右上隅に表示されます。



const addItem = () =>{        
    if($$("film_form").validate()){                                               
        var item_data = $$("film_form").getValues();
        $$("film_list").add(item_data);    
        webix.message("Validation is successful!");
    }
}

関数はこのように機能します。If条件の内部で、フォームにアクセスし、validate（）メソッドを呼び出します。テストが成功した場合、メソッドはtrueを返し、コードがさらに実行されます。テーブルにデータを追加した後、関数がメッセージ-で呼び出されます"Validation is successful!"。

ポップアップメッセージ：フォームウィジェットでルール


オブジェクトを作成する必要があります。validate（）関数は、すべてのフォームフィールドからの情報を、指定された基準に準拠しているかどうかをチェックする必要があります。 検証が成功するための条件は次のとおりです。


  • タイトルフィールドは空にできません。
  • フィールド年は1970年以上、現在の年を超えてはなりません。
  • Ratingフィールドはゼロであってはなりません。
  • 投票フィールドは1より大きく1,000,000未満でなければなりません。

次に、検証ルールを含むフォームウィジェットのソースコードは次のようになります。

view:"form",
id:"film_form",
width: 350,
elements:[
    { type:"section", template:"EDIT FILMS"},
    { view:"text", name:"title", label:"Title" },
    { view:"text", name:"year", label:"Year" },
    { view:"text", name:"rating", label:"Rating" },
    { view:"text", name:"votes", label:"Votes" }, 
    {
        margin:10, cols:[
            { view:"button", id:"btn_add", value:"Add new", 
            css:"webix_primary", click:addItem},
            { view:"button", id:"btn_clear", value:"Clear" }
        ]
    },
    {}
],
rules:{
    title: webix.rules.isNotEmpty,
    rating(value){
        if(webix.rules.isNumber(value)){
            return true;
        }
    },
    votes(value){
        return value < 1000000 && value >= 1;
    },
    year(value){
        return value>=1970 && value <= new Date().getFullYear();
    }
}

ルールオブジェクトのプロパティname: "…"は、フォーム要素のプロパティの値と一致する必要があります。事前定義されたルール webix.rules.isNotEmptyとwebix.rules.isNumberは、タイトル、評価、および投票のプロパティで設定されます。年には、ユーザールールを定義する関数があります。

ルールは、成功した場合はtrueを返し、エラーの場合はfalseを返します。rulesオブジェクトのすべてのプロパティがtrueを返すと、フォームは検証に合格します。それ以外の場合、検証に失敗したフィールドは赤で強調表示されます。

ステップ3.エラーメッセージ


ルールが設定されました。ここで、入力したデータが指定した基準を満たしていないことをユーザーに通知するエラーメッセージを表示するメカニズムが必要です。これを行うには、フィールドを赤で強調表示し、エラーメッセージを追加します。

フィールドにはinvalidMessageプロパティを設定できます。その値はエラーメッセージを含むテキストです。検証が失敗した場合、このメッセージはフィールドの下に表示されます。

コード：
elements:[
    { type:"section", template:"EDIT FILMS"},
    { view:"text", name:"title", label:"Title", 
    invalidMessage:"Must be filled in"},
    { view:"text", name:"year", label:"Year", 
    invalidMessage:"Should be between 1970 and current" },
    { view:"text", name:"rating", label:"Rating", 
    invalidMessage:"Cannot be empty or 0" },
    { view:"text", name:"votes", label:"Votes", 
    invalidMessage:"Must be less than 100000" }, 
    {
        margin:10, cols:[
            { view:"button", id:"btn_add", value:"Add new", 
            css:"webix_primary", click:addItem },
            { view:"button", id:"btn_clear", value:"Clear" }
        ]
    },
    {}
]

ステップ4.フォームをクリーニングする


フォームをクリーンアップするには、ボタンによって呼び出される関数を作成する必要があります"Clear"。彼女は警告メッセージの出力も担当し、肯定的な応答の場合、彼女は入力されたデータを消去してエラー表示をリセットします。

警告メッセージ：clearForm


関数に名前を付けて、次のコードを記述します。

function clearForm(){
    webix.confirm({
        title:"All data will be lost!",
        text:"Are you sure?"
    }).then( () => {
            $$("film_form").clear();
            $$("film_form").clearValidation();
        }
)};

確認ウィンドウでは、webback.confirm（{...}）コールバックメソッドが使用されます。このメソッドは、テキストとタイトルのプロパティを持つオブジェクトを受け入れます。このメソッドはpromiseを返すので、.thenハンドラーを使用できます。答えが正の場合、.thenハンドラーで2つのメソッドを呼び出します。clear（）-フォームフィールドをクリアするためと、検証エラーラベルを削除するclearValidation（）です。

最後のアクションは、関数呼び出しをボタンにバインドすること"Clear"です。

margin:10, cols:[
    { view:"button", id:"btn_add", value:"Add new", 
    css:"webix_primary", click:addItem},
    { view:"button", id:"btn_clear", value:"Clear",
    click:clearForm}
]
