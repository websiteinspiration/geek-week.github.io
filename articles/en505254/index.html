Snake sugar or write your range in JavaScript

Many people love Python ... Beginners are admired by the absence of semicolons, and advanced ones enjoy real simplicity. Today we will talk about how to implement in JavaScript a semblance of the very simplicity of Python, and specifically the range function .

In Python, the range function can be iterated or, for example, converted to an array - list (range (begin, end)) .

But the question is, is it possible for JavaScript to create something similar and at the same time to make the solution look native and simple?

The first thing that comes to mind is to write a similar class:

function range(from, to, step = 1){
    this.current = from
    this.to = to
    this.step = step

    this.next = () => this.current++ % to
}

Yes, I wrote that next returns the trimmed value, but this is also a problem that needs to be figured out how to solve - throw an error when going to the maximum or do something else?

In general, such a solution will not add convenience at all and it will be easier to write a standard for , for example, you can add the end function and after each iteration check if we have gone abroad, but there is no point in developing this solution.

ES6 introduces a new primitive data type, Symbol. He opens before us the doors to metaprogramming.

To solve our problem with range we will use Symbol.iterator . It makes it possible to create an iterator object using a specific protocol.

In order for an object to become an iterator, you need to define the [Symbol.iterator] () function in it , but since we want to create a common function, not a class, we will have a function that returns an iterator object.

Here's what it looks like:

function range(from, to, step = 1){
    return {
        [Symbol.iterator](){
            return {
                current: from,
                to: to,
                from: from,
                step,
                next(){
                    const it = { done: this.current >= this.to, value: this.current }
                    this.current += this.step
                    return it
                }
            }
        }
    }
}

What's happening? We have a range function , it returns, just the same, our iterator object, in which the [Symbol.iterator] () function returns an object. The most important thing is that this object contains the next function, otherwise the object will not be iterable and an error will appear when trying to use it. The next function should (!) Return an object with the done and value properties , where done (bool) signals the end of the iteration, and value contains the current value.

In principle, everything works, and you can already write something python-like:

    for(let i of range(0, 10, 2))
        console.log(i)

We could stop here ... But you can write a more elegant solution using generator functions:

function range(from, to, step = 1){
    return {
        *[Symbol.iterator](){
            for(let val = from; val < to; val += step){
                yield val;
            }
        }
    }
}

An additional plus will be the fact that we have other opportunities for iterators. For example, you can very easily convert our range to an array:

[...range(0, 10, 2)]
// [ 0, 2, 4, 6, 8 ]
