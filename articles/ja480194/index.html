クラウドソリューションアーキテクチャ。OTUSからの新しいコース

注意！この記事はエンジニアリングではなく、クラウドソリューションの開発とサポートの分野での教育に関心のある読者を対象としています。ほとんどの場合、学習に関心がない場合、この資料はあなたにとって興味がありません。



最近まで、「クラウド」という言葉が使用されたとき、誰もが大気現象について考えていましたが、今では大多数がすでにクラウドストレージと関連しています。現在、最も求められている高額なスペシャリストの一部は、クラウドソリューションアーキテクチャのアジャイル開発とメンテナンスの分野の知識を持つスペシャリストです。



Otusは、実際の組織変革プロジェクトとWell ‑ Architected Frameworkの推奨事項に基づいてクラウドソリューションを開発およびサポートするためのベストプラクティスである「クラウドソリューションアーキテクチャ」コースを開始しました。このコースは、主にアーキテクトと開発者を対象としていますが、次のプロファイルをクラウドネイティブレベルで開発することもできます。

  • IT /ソフトウェアアーキテクト
  • 開発者とDevOpsエンジニア
  • ネットワークおよびシステム管理者
  • 情報セキュリティ専門家
  • マネージャーとチームリーダー

数日前、このコースではオープンレッスンが行われ、学生はクラウドランディングゾーンドメインのアーキテクチャの設計を学び、メインドメインのアーキテクチャテンプレートを調べました。トレーニングの形式を理解し、教師と知り合うために、記録でそれを見ることができます。


12月18日20:00 にオープンデイが開催されます。そこでは、ウラジミールグトロフ先生がクラウドソリューションアーキテクチャコースのすべての質問に答え、コースプログラムの詳細と、卒業後に生徒が習得するスキルと能力について説明します。ウラジミールグトロフ氏-クラウドアーキテクト、ノードクラウドのコンサルタント。彼はスウェーデンのハスクバーナグループのCI / CDチームを率い、高度なエンドツーエンドITソリューションの開発と実装に20年以上の経験があります。クラウドソリューションアーキテクチャ

コースを受講するには、次の知識が必要です。

  • アプリケーションの開発および/またはメンテナンスの経験、できればDevOps Agileの経験
  • 少なくとも1つのクラウドプロバイダー-Azure、GCP、AWSなどの経験

あなたの知識とスキルがトレーニングに十分かどうかを理解するために、入学試験に合格することができます。

このコースは、自社のデータセンターでモノリシックアプリケーションを開発するための従来のウォーターフォールモデルから、マルチクラウド環境（AWS + Azure + GCP）と分散型クラウドネイティブマイクロサービスおよびサーバーレスアプリケーションを使用するアジャイルDevOpsモデルに移行することで、会社の部門を変革するための実際のプロジェクトに基づいています。コース

終了時クラウドソリューションアーキテクチャの開発と進化のためにアジャイルSCRUMプロジェクトを実施する方法を学び、適切に設計されたフレームワークの原則-ビジネスプロセスの最適化、セキュリティ、信頼性、高パフォーマンス、コスト最適化-に適合するクラウドソリューションアーキテクチャ（コードとしてのインフラストラクチャ）を作成できるようになります。また、もちろん、コースの修了証を受け取り、最も成功した学生には、最も人気のあるIT企業の面接への招待状が送られます。
