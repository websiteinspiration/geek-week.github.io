高負荷データベース管理システム向けCisco Hyperflex

Cisco Hyperflexに関する一連の記事を続けます。今回は、高負荷のOracleおよびMicrosoft SQL DBMSの状況におけるCisco Hyperflexの作業を紹介するとともに、取得した指標を競合ソリューションと比較します。

さらに、私たちは引き続き私たちの国の地域でHyperflexの機能を示しています。今回はモスクワとクラスノダールで開催されるソリューションの次のデモンストレーションにぜひお越しください。

モスクワ-5月28日。参照して記録します。
クラスノダール-6月5日。参照して記録します。

最近まで、ハイパーコンバージドソリューションは、特に負荷が高い場合、DBMSにあまり適していませんでした。ただし、UCSファクトリをCisco Hyperflexのハードウェアプラットフォームとして使用したおかげで、10年以上にわたってその信頼性とパフォーマンスが証明されており、この状況は今日変化しています。

もっと知りたい？その後、猫へようこそ。

前書き


現在、ハイパーコンバージドソリューションの編成には2つのアプローチがあります。最初のアプローチは、ソフトウェアとして提供されるソフトウェア定義ソリューションに基づいており、顧客は自分で機器を選択します。 2番目のアプローチは、ソフトウェア、ハードウェア、およびテクニカルサポートを含むターンキーソリューションに基づいています。シスコでは2番目のアプローチを採用し、既製のソリューションをお客様に提供します。これは、この方法でのみ、安定したシステム動作、1つのメーカーによる高品質のテクニカルサポート、および高性能を保証できるためです。
ミッションクリティカルなタスクでの製品の使用を決定する際の重要な要素の1つであるのは、高いシステムパフォーマンスです。

今日、組織は通常、ミッションクリティカルなタスクを従来の3層アーキテクチャソリューション（DSS>ストレージネットワーク>サーバー）に展開しています。同時に、ほとんどの組織は、ITインフラストラクチャの安定性と生産性を低下させずに、ITインフラストラクチャの簡素化とコスト削減に努めています。このため、顧客はハイパーコンバージドソリューションにますます注目しています。

この記事の一部として、独立した研究所ESG（Enterprise Strategy Group）によって実行された最新のテスト（2019年2月）について説明します。テスト中に、高負荷のOracleおよびMS SQL DBMS（OLTPテスト）の作業がエミュレートされました。これは、実際の生産的環境におけるITインフラストラクチャの最も重要なコンポーネントの1つです。

このロードは、3つのソリューション、つまりCisco Hyperflexと、Hyperflexで使用されているのと同じサーバー、つまりCisco UCSサーバーにインストールされた2つのソフトウェア定義ソリューションで実行されました。

テスト構成




キャッシュを使用した構成はソリューション開発者がサポートしていないため、ベンダーAはシステムでキャッシュを使用しません。このため、より大きなストレージディスクが使用されました。

試験方法


OLTPテストは、4つの仮想マシンと3.2 TBの作業データセットを使用して実行されました。各テストの前に、各VMはテストツールを使用して記録されたデータで満たされていました。これにより、テストが「実際の」データを読み取って既存のブロックに書き込むことが保証され、メモリから直接ゼロブロックまたはゼロ値が返されるだけではありません。これは、データがいっぱいでないときに発生するため、アプリケーション環境でのデータの読み書き方法をテストが正確に反映していることを確認することが重要でした。この大きなワーキングセットを完了するには長い時間がかかりましたが、より正確なパフォーマンスデータを取得できるため、時間の生産的な投資であると私たちは考えています。

テストは、HCIベンチツール（Oracle Vdbenchに基づく）と、OracleおよびSQL Serverバックエンドを使用して複雑な重要なOLTPワークロードをエミュレートするように設計されたI / Oプロファイルを使用して行われました。ブロックサイズは、100％ランダムデータアクセス（完全ランダム）でエミュレートされたアプリケーションに従って割り当てられました。

Oracleデータベースのワークロード


1つ目は、Oracle環境をエミュレートするために設計されたOLTPテストでした。Vdbenchを使用して、さまざまな読み取り/書き込み比率でワークロードを作成しました。テストは4つの仮想マシンで実行されました。4時間のテストで、HyperFlexは、わずか4.4ミリ秒の遅延で420,000以上のIOPSを示しました。ソフトウェアソリューションAとBは、それぞれ238,000と251,000 IOPSしか表示できませんでした。




ベンダーBの書き込み遅延が平均26.49ミリ秒で、非常に良好な読み取りレイテンシが2.9ミリ秒であることを除いて、さまざまなシステムのレイテンシレベルはほぼ同じでした。すべてのシステムで圧縮と重複排除がアクティブでした。

Microsoft SQL Serverワークロード


次に、Microsoft SQL Server DBMSをエミュレートするように設計されたOLTPワークロードが調査されました。


このテストの結果、Cisco HyperFlexクラスタは、競合他社AとBの両方を約2倍上回りました。


Cisco HyperFlexでの遅延の結果は、Oracleテストと大差ありません。つまり、4.4 msの良好なレベルでした。同時に、メーカーAとBは、Oracleのテストよりも著しく悪い結果を示しました。競合ソリューションBの唯一の良い点は、2.9 msでの読み取りレイテンシが一貫して低いことです。他のすべての点で、Hyperflexは競合ソリューションより2倍以上優れています。

調査結果


独立したESGラボが実施したテストにより、Cisco Hyperflexソリューションの適切なレベルのパフォーマンスが再び確認されただけでなく、ハイパーコンバージドシステムがミッションクリティカルなタスクで普遍的に使用できることも証明されました。

ハイパーコンバージドシステムは、非クリティカルな負荷により適していると長い間考えられてきました。 ESGは2016年に大企業を対象に調査を実施しました。ハイパーコンバージドよりも伝統的なインフラストラクチャを好む理由を尋ねられました。回答者の54％が理由は生産性であると述べました。

2018年に早送りします。状況は変化しました。ESG調査を繰り返しても、従来のアプローチの方がパフォーマンスの点で優れていると考えている回答者の24％だけがすでに明らかになっています。

テクノロジーの進化により業界で意思決定の基準が変わると、顧客が望むものと得られるものとの間に不一致が生じることがよくあります。何が欠けているのかを確認し、この空白を埋めることができる製造業者は、有利になります。シスコはハイパーコンバージドソリューションを提供し、シンプルさと費用対効果に加え、欠けている重要なワークロードに必要な一貫した高いパフォーマンスを提供します。

シスコはハイパーコンバージドシステムの分野で前進しています。これは、Cisco Hyperflexソリューションの優れた機能だけでなく、市場での存在によっても確認されています。したがって、Gartnerによると、2018年の秋に、シスコはHCI市場のリーダーのグループに当然含まれていました。


すでに、モスクワとクラスノダールのデモにアクセスすることで、Hyperflexが最も複雑で要求の厳しいビジネスタスクに最適なソリューションであることを確認できます。
