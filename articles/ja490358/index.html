現代のプログラマーはアルゴリズムを作成できる必要がありますか？

プログラマーへのインタビューに関する私の最後の記事に対するハブロフスク市民の否定的な反応の後、私はそれを注意深く検討して、プログラミング、プログラマー、そして私自身についての私の考えのいくつかを再考し、修正する必要がありました。そして、どこかからもたらされた傲慢さ、アイドルの例（私はそれをテストしませんでした-デプロイではありません）に加えて、私が最初に望んだことをまったく表現しませんでした。主なことは、問題を解決するコードを書く能力です。


それ以来、開発者との技術面接に参加する義務は私から消えていません。インタビューの形式は同じです。オンラインノートブックのコードで機能しますが、残っているタスクは2つだけで、はるかに簡単になりました。


以下はその1つの例です。


/*   —   ,    : 1    */
function isPrimeNumber(number) {
    /*   ,  true  ,  number     false    */
}

この記事のタイトルの問題は決して修辞的なものではなく、インタビューでこのタスクを中心に展開する情熱にその外観が負っています。


少し入力：JavaScript / TypeScriptで書き込む完全なスタックを探しています。課金システムの開発から、スクリーンショットを描画してクラウドにアップロードするためのデスクトップアプリケーションまで、開発者自身の希望に応じて、タスクの範囲は非常に広くなる可能性があります。したがって、数学の基本的な知識（プラスまたはマイナス、除算-乗算、算術平均の検索、数値の最大-最小）だけでなく、適用された問題を解決する能力も必要です。履歴書によると、候補者は中級以上のレベルで、関連する経験を積んでいます。開発は会社の中核事業ではないので、強力な開発者を引き付けることは難しく、そのような候補者はほとんどいません。
