端から端まで、アプリケーションはシステムパネルを超えています。これにより、アプリケーションのコンテンツが注目され、優れたユーザーエクスペリエンスが作成されます。

実際には、これはアプリケーションの作成者が2つのことを覚えておく必要があることを意味します。

ナビゲーションバー領域の使用


ジェスチャーコントロールをサポートするための最初の最も重要な考慮事項は、ナビゲーションバーの背後の位置を処理することです。ナビゲーションバーのサイズと重要性が減少したため、Android Q +を使用してより魅力的で最新のUXを作成する場合、アプリケーションをナビゲーションバーの背後に配置することを強くお勧めします。

Android Pie以下のデバイスで起動した場合、ナビゲーションバーの後ろの位置はオプションと見なされ、これが理にかなっているかどうかはアプリケーションの作成者に委ねられます。ただし、必要なAPIのほとんどすべてがAPI 21まで下位互換性があります（またはAndroidXが独自に違いを処理します）。そのため、Qバージョンの前にデバイスをサポートするために必要な追加の作業量は最小限です。ジェネレーションQまでのデバイスのユーザーも、没入型体験を得ることができます。必要な作業とテストの最小化のみがオプションと見なされます。

ステータスバー領域の使用


次に、画面の上部にあるステータスバーに注目します。コンテンツまたはレイアウトで必要な場合は、ステータスバーの領域を使用することが理にかなっています。それはどういう意味ですか？ステータスバーが表示されるレイアウトの例は、ワイドスクリーンイメージです。開発者にとって、これはAppBarLayoutのようなものを使用することを意味します。これは画面の上部に配置および固定されています。

実装


エッジツーエッジの使用を実装するには、3つの重要なステップがあります。

  1. 全画面モードで表示するための要求

最初のステップは、アプリケーションをシステムパネルの上（Y軸に沿って）に配置する必要があることをシステムに通知することです。これに使用するAPIは、いくつかのフラグを持つビューのsetSystemUiVisibility（）です。関心のあるフラグは次のようになります。

view.systemUiVisibility = 
    // Tells the system that the window wishes the content to
    // be laid out at the most extreme scenario. See the docs for
    // more information on the specifics
    View.SYSTEM_UI_FLAG_LAYOUT_STABLE or
    // Tells the system that the window wishes the content to
    // be laid out as if the navigation bar was hidden
    View.SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION

その後view、ナビゲーションバーに全画面モードで配置されます。


これで、アプリケーションはナビゲーションバーのフルスクリーンモードで配置されます。

  1. システムパネルの色を変更する

これでアプリケーションが全画面モードになったので、背後にあるコンテンツが見えるようにシステムパネルの色を変更する必要があります。

<!-- values-v29/themes.xml -->
<style name="Theme.MyApp">
    <item name="android:navigationBarColor">
        @android:color/transparent
    </item>

Android Qでは、システムがすべてのナビゲーションモードでシステムパネルのコンテンツの視覚的な保護（時間、アイコン、ドラッグアンドドロップ処理など）をすべて処理するようになりました。これは、もう考える必要がないことを意味します。実際には、これはシステムが2つのことの1つを実行することを意味します：

動的な色調整

システムパネルのコンテンツは、その背後にあるコンテンツに応じて色が変わります。したがって、パネルが何らかの明るいコンテンツの上にある場合、その色は暗い色に変わります。逆に、コンテンツが暗い場合、色は明るい色に変わります。これは、ダイナミックカラーアダプテーションと呼ばれるものです。


Android Qでの動的な色調整。
