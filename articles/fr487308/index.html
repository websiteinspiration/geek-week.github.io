Un des problèmes de Java est sa verbosité et la quantité de code standard nécessaire. C'est une connaissance commune.


Regardons une simple classe Cat en Java. Nous voulons que chaque objet Cat ait les attributs (champs) suivants:


  • Nom
  • Nombre de vies
  • Couleur

Assez simple, non? Voyons maintenant le code en Java. Pour simplifier, rendons notre classe immuable (immuable) - sans setters, nous allons tout configurer dans notre constructeur.


public final class Cat {

    private final String name;
    private final int numberOfLives;
    private final String color;

    public Cat(String name, int numberOfLives, String color) {
        this.name = name;
        this.numberOfLives = numberOfLives;
        this.color = color;
    }

    public String getName() {
        return name;
    }

    public int getNumberOfLives() {
        return numberOfLives;
    }

    public String getColor() {
        return color;
    }
}

IntelliJ IDEA


IntelliJ IDEA Preview feature File → Project Structure.



Pour utiliser les entrées dans IntelliJ IDEA, vous aurez besoin de la version 2020.1 et ultérieure. Depuis le 2/2020, il est disponible en tant que version du programme d'accès anticipé . IDEA a actuellement un support de base pour les enregistrements, mais un support complet devrait être disponible dans la version finale.


Compilation manuelle


Une alternative consiste à assembler manuellement le projet. Ensuite, vous devez fournir les options suivantes pour javac:


javac --release 14 --enable-preview ...

C'est pour la compilation. Au moment de l'exécution, vous fournissez simplement --enable-preview


java --enable-preview ...
