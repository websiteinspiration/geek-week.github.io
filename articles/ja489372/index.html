新しいAzure試験

過去3か月間で、マイクロソフトは認定と試験に対する多くの変更を発表しました。それらのいくつかは、この記事で説明されています。

この記事では、完全に新しい試験に焦点を当てるのではなく、既存の試験の更新バージョンに焦点を当てます。管理者向けのAZ-104、開発者向けのAZ-204、建築家向けのAZ-303およびAZ-304です。

2020年1月（Ignite2018でのロールベース試験の最初の発表から1.5年弱、az-203およびaz-103の登場から1年と1か月後）、マイクロソフトは開発者、管理者、建築家、およびAIエンジニア向けの試験の変更を発表しました（後者は最初の3つほど重要ではありません）。

一部の試験はすでにベータ版（az-204 / az-104）で利用可能であり、アーキテクチャ試験は後で利用可能になります。 2020年8月末までに古い（現在の）試験に合格することができます（期限はCovid-19により延期されました）

。特に、新しい試験の導入後、新しい証明書は導入されません。管理者/開発者/設計者であったため、そのまま残りました。一般に、ゼロによる乗算は行われず、外部オブザーバーにとっても、何も変更されていないかのように、以前は、試験ではなく証明書が表示されていました。

この記事では、試験の変更点について説明します。

コミュニティはめったに試験の変更を楽しんでいません。これが以前の試験にゼロを掛けるたびに（これが完全に真実ではない場合でも、コミュニティはまだこれを認識しています）。しかし、ここでは「すべてがそれほど単純なわけではありません」。

試験の変更に移りましょう。

AZ-203 / AZ-204


公式のスキル測定に基づいて行われた変更の分析。一方では、az-203と比較して、az-204の変更はほとんどありません。一方、az-203はすでに2番目のバージョンです。彼はaz-200beta / az-201betaを置き換えました。2回目の試行から、通常ははるかに簡単になりました。

AZ-204では、Azure Search、Azure Batch、Azure API /モバイルアプリ、Azure Storage Tables、Azure SQL Azureが削除されました。 2019年にコースを実施した経験から、これらのサービス（SQLを除く）を試験に含めることについて多くの不満がありました。数年に渡ってさまざまなプロジェクトでAzureに書き込みを行った経験豊富な開発者は、これらのサービスを見たことはありません。もちろん、これは主観的なものですが、大規模なサンプルではすでに代表的なものです。

管理者向けのAZ-104に移りましょう

AZ-103 / AZ-104


それらがセクション全体で削除されたAz-204とは異なり、再配置は非常に小さかったのですが、Az-104では連続的な再配置がありました。サブスクリプション管理

セクションの代わりに、私たちは今、誇り高いガバナンスを持っています。サブスクリプション、リソースグループ+ ID管理について少し。

隠しテキスト

Storageセクションには、次の変更が含まれています。

  • ストレージからバックアップを削除し（ストレージではないため論理的です）、別のバックアップと監視セクションに移動しました
  • 彼らはデータボックスを削除しましたが、これも正常です 実際に彼と一緒に働く人は非常に少なく、試験で彼について何を尋ねるかは明らかに明確ではありません。
  • CDNを削除しました。また、ストレージではありません。
  • アクティビティログを削除しました。これもストレージ固有のものではなく、別のセクションのバックアップと監視に移動しました

隠しテキスト

仮想マシンセクションはコンピューティングセクションに変換され、仮想マシンだけでなく、マネージドKubernetesクラスター、コンテナーインスタンス、ウェブアプリも含まれるようになりました。

隠しテキスト

同時に、DSCも削除されましたが、最終的にカスタムスクリプト拡張として偽装された可能性があります。もちろん、DSCは興味深いテクノロジーですが、ささいなことであり、Linuxがはるかに大きい世界では、それほど興味深いものではありません（私の個人的な意見では）。

ネットワーキングのセクションは大幅に補足されています。それは明らかに、Azure Application Gateway、Azure Bastion、Azure WANのように見えました。

隠しテキスト




以前は「Azure Resource Utilization」の一部であった別の監視セクションが表示されましたが、率直に言って、本質に対応していません。

隠しテキスト


Identity / Azure Active Directory

非常に多くの点で、Azure Active Directoryのセクションは減量され、変化しました。私はこの決定を支持します。ええと、毎日MFAを構成するわけではありません（通常、テナント全体に対して1回限りの操作ですか？そして、全員がAzure MFAを使用するわけではありません）。広告接続を介した同期は、彼らがそれを削除したことも良いことです。多くのクライアントがそれを使用していますが、通常は1回セットアップしてもう触れません（管理者の99％が24時間体制で紺碧を管理しているにもかかわらず、これを見たことはありません）

全体として、私の主観的な意見では、知識の要件はそれほど変わっていませんが、構造コンテンツがより論理的になりました。

最大の変化に移ります-建築家のための試験。

AZ-300＆AZ301-> AZ-303 / AZ-304


建築の証明書に合格するには、2つの設計および実装試験に合格する必要がありました。この意味では、何も変わっていません。2つの試験があったので、残っています。しかし、試験の構造は大幅に改訂されました。私の意見では、知識と経験のセットは実際には変わっていませんが、試験の構造は根本的に変わっています。結論は、新しいスキルメジャー1と2に基づいて作成されています。

もちろん、実際のアーキテクトが知るのに役立つが、Azure固有ではないか、直接確認することが難しいパーツは削除されました。たとえば、az-301に「要件の収集」というセクションがありました。

隠しテキスト

試験では、スクリプト「ケーススタディ」を使用して、建築家が重要な要件を分離できるかどうかを理解できます。しかし、要件の実際のコレクションについては、ホワイトボードセッション、アンケート、ハッカソン、ワークショップを行う必要があります。テストではこれは行われません。

また、301の試験のうち、役割や人物などの作成に関するセクションを「公開」しました。ソリューションアーキテクトとしての私の経験に基づくと、そのようなことはビジネス分析に有効であり、プロジェクトに何もない場合にのみ、自分で行うことができます。そしてAzureはそれとは何の関係もありません。

隠しテキスト

アーキテクトがSQLコードを記述できるようにする必要がある部分が削除されました。便利ですが、方言はいくつかありますが、一般的に、アーキテクチャはどこにありますか？！アーキテクトは技術リーダープロジェクトの役割を果たすことができますが、この場合、基本的なSQL構文に関する質問はアーキテクトの問題ではありません。しかし、高可用性アーキテクトは、たとえ彼がデータの非常に大きなスペシャリストでなくても、知っておくべきです。

隠しテキスト

SSL / TLSシリーズのセキュリティに関する抽象的な言葉は削除され、Azure Key VaultやAzure AD Managed Identityなどの特定のサービスに置き換えられました。

隠しテキスト

それでも移行セクションがAz-303試験で説明されていて、更新前は簡潔すぎました。

隠しテキスト

しかし、304年に再編成されたのは

ネットワーキングセクションは、より紺碧の機能/サービス固有になりました。実際、私たちは何を知るべきかを教えられました。以前は非常に抽象的でした：ネットワークセキュリティですが、何が含まれているかは明確ではありません。

隠しテキスト

設計データ管理、設計ストレージ戦略、データ保護、アーカイブ戦略（ビジネス継続性から取ったもの）は、適切なデータベースとストレージアカウントを選択するために削減されました。私の意見では、これはコンテンツの変更というよりも再パッケージ化の方が多いと思います。多くのポイントを減らすことができたので、そうでした。

隠しテキスト

Design and Document DataflowセクションはDesign Data Integrationになり、サービスも直接リストされました。「データフロー図の作成」などのチェックされていない質問が削除されたのは素晴らしいことです。繰り返しになりますが、az-301の説明は抽象的な建築家について書かれていたようですが、304では、検証できるものとazureに固有のものだけを書くことにしました。

隠しテキスト

しかし、承認セクション（トピックのリストを見る）は完全に変更されました。以前は、抽象的な「blabla」でした。新しいバージョンでは、Azure ADとAzureの特定のサービスと機能を求めています。新しい問題の一部ですが、彼はガバナンスを開始することもできました。

隠しテキスト

実際、Computeセクションは変更されていませんが、AKSに関する項目が追加されていますが、以前は他のセクションにあったため、何も変更されていないと言えます。

隠しテキスト

統合APIセクションは、「アプリケーションアーキテクチャの設計」セクションに移動しました。Az-300から削除された（最終的にはaz-304に解決された）イベント/メッセージは同じセクションに分類されました。

そこで、「アプリケーションアーキテクチャの設計」の「配置の設計」セクションが移動しました。

隠しテキスト

高可用性は変化していないようで、質問の一部がビジネス継続性に行きました。

隠しテキスト

災害復旧セクションは、バックアップ部分も吸収しました。より論理的になりました。

隠しテキスト

したがって、Az-303 / AZ-304セクションでは、非常に大規模な再配置とリファクタリングが行われたと自信を持って言えます。しかし、私の意見では、検証済みの知識の量は変わっていません。過去の試験の準備をしていた人は、新しい試験に合格するのに問題はないはずです。

4つの試験すべてを見ると、試験の構造（または少なくともスキル測定の構造）に顕著な改善があります。ただし、トレーニングコースを作成した場合は、それらを再設計および再構成する必要があります。
