OpenWrtを実行しているMikrotikルーターにWireGuardを設定します


ほとんどの場合、ルーターをVPNに接続することは難しくありませんが、最適な接続速度を維持しながらネットワーク全体を保護したい場合は、WireGuard VPNトンネルを使用するのが最善のソリューションです。Mikrotik

ルーターは信頼性が高く非常に柔軟なソリューションとしての地位を確立していますが、残念ながらRouterOSでのWireGuardのサポートはまだなく、いつどのバージョンで表示されるかは不明です。最近、それが知られるようになった WireGuard VPNトンネルの開発者が提供していることパッチのセットのLinuxカーネルの彼らのVPNトンネル組織のソフトウェア部分を行います、と私たちは、これがRouterOSの実現に貢献することを願っています。

しかし、現時点では残念ながら、MikrotikルーターでWireGuardを構成するには、ファームウェアを変更する必要があります。

ファームウェアMikrotik、OpenWrtのインストールと構成


まず、OpenWrtがモデルをサポートしていることを確認する必要があります。mikrotik.comで、モデルがそのマーケティング名とイメージに準拠しているかどうかを確認できます。ファームウェアダウンロードセクションの

openwrt.comにアクセスします。

このデバイスには、2つのファイルが必要です。

downloads.openwrt.org/releases/18.06.2/targets/ar71xx/mikrotik/openwrt-18.06.2-ar71xx-mikrotik-rb-nor-flash-16M-initramfs-kernel.bin|elf

downloads.openwrt.org/releases /18.06.2/targets/ar71xx/mikrotik/openwrt-18.06.2-ar71xx-mikrotik-rb-nor-flash-16M-squashfs-sysupgrade.bin


インストールとアップグレードの 両方のファイルをダウンロードする必要があります。



1.ネットワークのセットアップ、PXEサーバーのロードとセットアップ


Tiny PXE Server for Windows最新バージョンを ダウンロードしてください。

別のフォルダに解凍します。config.iniファイルで、パラメーターrfc951 = 1 section [dhcp]を追加します。このパラメーターは、すべてのMikrotikモデルで同じです。



ネットワークの設定に移ります。コンピューターのネットワークインターフェイスの1つに静的IPアドレスを登録する必要があります。


IPアドレス：192.168.1.10
ネットマスク：255.255.255.0


Tiny PXEサーバーを管理者として 起動し、DHCPサーバーフィールドでアドレス192.168.1.10のサーバーを選択します

。Windowsの一部のバージョンでは、このインターフェースはイーサネット接続後にのみ表示される場合があります。ルーターを接続し、すぐにルーターとPCをパッチコードを使用して接続することをお勧めします。



「...」ボタン（右下）をクリックして、Mikrotikのファームウェアファイルをダウンロードしたフォルダーを指定します。

名前が「initramfs-kernel.bin or elf」で終わるファイルを選択してください



2. PXEサーバーからルーターをダウンロードする


PCワイヤとルーターの最初のポート（wan、internet、poe inなど）を接続します。その後、つまようじを取り、「Reset」と書かれた穴に差し込みます。



ルーターの電源を入れ、20秒待ってからつまようじを放します。
次の1分間で、次のメッセージがTiny PXE Serverウィンドウに表示されるはずです。メッセージが表示さ



れれば、正しい方向です。

ネットワークアダプターの設定を復元し、受信アドレスを動的に（DHCPを介して）設定します。

同じパッチコードを使用して、MikrotikルーターのLANポート（この場合は2 ... 5）に接続します。ポート1からポート2に切り替えるだけです。ブラウザでアドレス192.168.1.1を開きます。



OpenWRT管理インターフェースに入り、メニューセクションの「システム->バックアップ/フラッシュファームウェア」に進み



、「新しいファームウェアイメージのフラッシュ」サブセクションで、「ファイルの選択（参照）」ボタンをクリックします。



名前が「-squashfs-sysupgrade.bin」で終わるファイルへのパスを指定します。



その後、「Flash Image」ボタンをクリックします。

次のウィンドウで、[続行]ボタンをクリックします。ルーターへのファームウェアのダウンロードが開始されます。



!!! ファームウェアの処理中にルーターの電源を切断しないでください!!!



ルーターをフラッシュして再起動すると、OpenWRTファームウェアを含むMikrotikが届きます。

考えられる問題と解決策


2019 Mikrotikデバイスの多くは、FLASH-NORメモリチップタイプGD25Q15 / Q16を使用しています。問題は、フラッシュによってデバイスモデルデータが保存されないことです。

「アップロードされた画像ファイルにはサポートされている形式が含まれていません。ご使用のプラットフォームに適した一般的な画像形式を選択してください。」おそらく問題はフラッシュにあります。

これは簡単に確認できます。デバイスのターミナルでモデルIDを確認するコマンドを実行します

root@OpenWrt: cat /tmp/sysinfo/board_name

そして、「不明」という回答を受け取った場合は、「rb-951-2nd」の形式で手動でデバイスモデルを指定する必要があります。デバイスモデル

を取得するには、次のコマンドを実行します。

root@OpenWrt: cat /tmp/sysinfo/model
MikroTik RouterBOARD RB951-2nd

デバイスのモデルを受け取ったら、手動でインストールします。

echo 'rb-951-2nd' > /tmp/sysinfo/board_name

その後、Webインターフェイスまたはsysupgradeコマンドを使用してデバイスをフラッシュできます

WireGuardでVPNサーバーを作成する


WireGuardが構成されたサーバーが既にある場合は、この項目をスキップできます。猫に関するMyVPN.RUN
アプリケーションを使用して、個人用VPNサーバーを構成します。レビューは既に公開しています。

OpenWRTでのWireGuardクライアントの構成


SSH経由でルーターに接続します。

ssh root@192.168.1.1

WireGuardをインストールします。

opkg update
opkg install wireguard

構成を準備します（以下のコードをファイルにコピーし、指定した値を独自のものに置き換えて、ターミナルで実行します）。

あなたがMyVPNを使用する場合は、その後、以下の設定でのみ変更する必要がありWG_SERVサーバーのIP、 - WG_KEY - wireguard設定ファイルとから秘密鍵WG_PUB -公開鍵。

WG_IF="wg0"
WG_SERV="100.0.0.0" # ip  
WG_PORT="51820" #  wireguard
WG_ADDR="10.8.0.2/32" #   wireguard

WG_KEY="xxxxx" #  
WG_PUB="xxxxx" #   

# Configure firewall
uci rename firewall.@zone[0]="lan"
uci rename firewall.@zone[1]="wan"
uci rename firewall.@forwarding[0]="lan_wan"
uci del_list firewall.wan.network="${WG_IF}"
uci add_list firewall.wan.network="${WG_IF}"
uci commit firewall
/etc/init.d/firewall restart

# Configure network
uci -q delete network.${WG_IF}
uci set network.${WG_IF}="interface"
uci set network.${WG_IF}.proto="wireguard"
uci set network.${WG_IF}.private_key="${WG_KEY}"

uci add_list network.${WG_IF}.addresses="${WG_ADDR}"

# Add VPN peers
uci -q delete network.wgserver
uci set network.wgserver="wireguard_${WG_IF}"
uci set network.wgserver.public_key="${WG_PUB}"
uci set network.wgserver.preshared_key=""
uci set network.wgserver.endpoint_host="${WG_SERV}"
uci set network.wgserver.endpoint_port="${WG_PORT}"
uci set network.wgserver.route_allowed_ips="1"
uci set network.wgserver.persistent_keepalive="25"
uci add_list network.wgserver.allowed_ips="0.0.0.0/1"
uci add_list network.wgserver.allowed_ips="128.0.0.0/1"
uci add_list network.wgserver.allowed_ips="::/0"
uci commit network
/etc/init.d/network restart

これでWireGuardのセットアップが完了しました。これで、接続されているすべてのデバイスのすべてのトラフィックがVPN接続によって保護されます。
