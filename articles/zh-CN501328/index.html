Visual Studio代码空间简介：无论您身在何处，都可以进行云开发

当我们都实施社交距离和远程工作之类的实践时，开发团队变得更加分散。我们自己的团队面临着过渡过程中所面临的一些挑战，这促使我们加紧努力，以帮助开发人员随时随地保持生产力。首先，我们通过允许用户从其浏览器加入Live Share会话来简化远程协作。今天，我们很高兴与您分享这次有关Visual Studio Online的新闻，无论您身在何处，它都能帮助您高效地工作。



Visual Studio代码空间简介


11月，我们宣布发布Public Preview Visual Studio Online。在过去的几个月中，我们收到了大量评论，并且在该工具上运行良好。我们看到的最常被引用的评论之一是，创建基于云的开发环境的能力对比我们以前认为的更多的因素具有积极的影响。我们的用户尤其认为：

  • 连接到新项目（或现有项目中的新任务）所需的时间要短得多
  • 他们的基于云的开发环境的性能在许多方面都比本地环境更好。
  • 可以在Visual Studio Code，浏览器编辑器或Visual Studio IDE（在专用预览版中）之间轻松切换的功能使他们有机会在几乎任何设备上工作。
  • 与同事的合作非常容易，因为Live Share可用于协作开发，编辑和调试。

我们了解到，开发人员认为Visual Studio Online不仅仅是“浏览器编辑器”。他们说：“基于云的开发环境的功能使其成为您编写所有代码的地方。”

为了更好地适应这种心情和服务的真正价值，我们将Visual Studio Online重命名为Visual Studio Codespaces。您想在大型项目中获得很棒的经验吗？在代码空间中执行。您是否需要快速创建新功能的原型或执行一些短期任务（例如，处理拉取请求）？创建代码空间。您的代码空间仅仅是最高效的代码空间。

价格现在更低


我们相信Visual Studio Online Codespaces可提供市场上最好的远程开发体验。我们希望有尽可能多的人可以访问并使用Codespace，尤其是考虑到我们现在都面临的问题。为了实现这一目标，我们降低了价格。

与往常一样，我们将继续仅根据您使用的时间和资源向您收费。当您在Codespace中处于活动状态时，它会使用Azure计算资源。在我们的Azure定价页面或价格计算器上可以找到每个活动Codespace的新价格。

只有您需要的力量


最后，我们今天还宣布了一项新功能。尽管开发人员喜欢他们在我们的云代码空间中看到的性能，但我们还了解到，有许多场景要么不需要那么多功能，要么开发人员想要运行几个小的代码空间（代码空间）以使他们可以在多任务模式下工作。

为了更好地实现这些方案，并帮助人们首次尝试远程开发，我们介绍了基本实例类型，该实例类型带有两个虚拟核心，四个GB的RAM和64 GB的SSD存储。我们使用此配置进行了很多测试，并且体验与我们的标准版和高级版一样出色，但是成本更低。基本实例类型现在可用。

远程开发。比以往更好


在过去的几个月中，我们为该服务添加了其他一些很酷的功能！我个人最喜欢的是对创建自己的Docker文件或图像的支持。使用此功能，可以通过指向注册表（例如Docker Hub）中的映像，或将自己的Dockerfile放置在存储库中来适应您的规范，这些存储库将在您启动Visual Studio代码空间时连接到该存储库。

我们最近还发布了一项小巧但重要的功能，允许用户在其环境中调整实例的大小。这意味着您可以将大部分时间花费在功耗更低，成本更低的环境中，然后仅在需要时才“增加功率”，而不会丢失状态或创建新的环境。这甚至支持Basic的实例类型。

最近令我高兴的是，我们对自托管环境的功能进行了巨大改进。在独立环境中，您可以在Visual Studio Codespace中注册任何计算机，然后从VS Code或浏览器连接到该计算机。此外，它是100％免费的。我的同事艾莉森（Allison）描述了所有细节。 -不要忘记阅读她的帖子以获取更多信息。
