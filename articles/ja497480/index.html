サウンドスライド


私が言えること-マイクをオンにして書き込み:)

経験上、最も安価な外部マイクからの録音品質も、ラップトップの内蔵マイクよりも比類ないほど優れていることがわかっています。高品質の機器が必要な場合は、この記事をお勧めします。

録音には、非常にシンプルな録音アプリケーションであるオーディオレコーダーを使用しました。あなたはそれを例えば、ここに取ることができます：

sudo add-apt-repository ppa:audio-recorder/ppa
sudo apt-get update
sudo apt-get install audio-recorder

このステップの主なものは、適切なファイルの命名です。名前は、スライド番号とフラグメント番号で構成する必要があります。フラグメントには奇数番号が付けられています-1、3、5など。したがって、スライドにテキストが画像に表示されている場合、002-1.mp3と002-3.mp3の 2つのファイルが作成されます。

静かな部屋で一度にすべてのビデオを録画した場合、それらを使用して何もできません。複数の段階で録音した場合は、音量レベルを等しくすることをお勧めします。

mp3gain -r *.mp3

何らかの理由でmp3gain ユーティリティは標準リポジトリにありませんが、ここで入手できます。

sudo add-apt-repository ppa:flexiondotorg/audio
sudo apt-get update
sudo apt-get install mp3gain

このすべての後、沈黙のある別のファイルを書き込む必要があります。サイレントクリップにサウンドトラックを追加する必要があります。一方のクリップにサウンドトラックがあり、もう一方のクリップにはない場合、これらのクリップを接着するのは困難です。沈黙はマイクから録音できますが、Audacityエディターでファイルを作成することをお勧めします。ファイルは少なくとも1秒長く（長く）なければならず、silence.mp3を呼び出す必要があります

調理分解ローラー


ここでのすべてはあなたの想像力によってのみ制限されます。Avidemuxエディターを使用してビデオを編集できます。かつては標準のリポジトリにありましたが、その後何らかの理由で切り出されました。これは私たちを止めません：

sudo add-apt-repository ppa:ubuntuhandbook1/avidemux
sudo apt-get update
sudo apt-get install avidemux2.7-qt5

このエディタを海で使用するための手順です。基本的に、すべてが直感的に使用できます。いくつかの条件を観察することが重要です。

まず、映画の解像度は映画の目標解像度と一致する必要があります。これを行うには、「出口ビデオ」で2つのフィルターを使用する必要があります。swsResizeで解像度を変更し、「フィールドを追加」してソビエトの「ナローフォーマット」フィルムのワイドフォーマットを作成します。他のすべてのフィルターはオプションです。たとえば、「ロゴを追加」フィルターを使用して、誰かが議論中のフラグメントのシャリコフ氏の発言が理解できない場合は、「犬のハート」の上にPostgreSQLロゴを追加できます。

次に、すべてのフラグメントで同じフレームレートを使用する必要があります。 1秒あたり25フレームを使用しています。これは、カメラと古いソビエト映画に多くの影響を与えるためです。フレームをカットしたムービーが別の速度で撮影される場合は、「ビデオのリサンプル」フィルターを使用します。

3番目に、すべてのフラグメントは同じコーデックで圧縮され、同じコンテナにパックされる必要があります。したがって、フォーマットのAvidemuxで、ビデオ-「Mpeg4 AVC（x264）」、オーディオ-「AAC（Faac）」、出力フォーマット-「MP4 Muxer」を選択します。

4番目に、カットクリップに正しく名前を付けることが重要です。ファイル名は、スライド番号とフラグメント番号で構成する必要があります。フラグメントには、2から始まる偶数の番号が付けられます。したがって、説明したフレームの場合、チョッピングを含む クリップは002-2.mp4と呼ばれる

必要があります。クリップの準備ができたら、フラグメントを含むディレクトリに転送する必要があります。avidemux設定は、不可解なtbr、tbn、tbcパラメーターを使用したデフォルトのffmpeg設定とは異なります。それらは繁殖には影響しませんが、ローラーを接着することはできません。したがって、再コーディングします。

for f in ???-?.mp4;
do
  ffmpeg -hide_banner -y -i "${f}" -c copy -r 25 -video_track_timescale 12800 ../clips/$f
done

スクリーンセーバーを削除します


ここでもすべてが単純です。ある種の巧妙なスキームの背景で撮影し、受信したビデオをrecディレクトリに入れ、そこからフラグメントを含むディレクトリに転送します。命名規則は引用符区切りの場合と同じで、recodingコマンドは次のとおりです。

ffmpeg -y -i source_file -r 25 -vcodec libx264 -pix_fmt yuv420p -profile:v high -coder 1 -s 1280x720 -ar 44100 -ac 2 ../clips/xxx-x.mp4

ビデオをスピーチで開始する場合は、このフラグメントに000-1.mp4という名前を付けます

静止画からフレームを作ります


静止画像と音声からクリップをマウントする時が来ました。これは次のスクリプトで行われます。

#!/bin/bash

for sound in sound/*.mp3
do
  soundfile=${sound##*/}
  chunk=${soundfile%%.mp3}
  clip=${chunk}.mp4
  pic=slide/${chunk%%-?}.png

  duration=$(soxi -D ${sound} 2>/dev/null)
  echo ${sound} ${pic} ${clip} " - " ${duration}

  ffmpeg -hide_banner -y -loop 1 -i ${pic} -i ${sound} -r 25 -vcodec libx264 -tune stillimage -pix_fmt yuv420p -profile:v high -coder 1 -t ${duration} clips/${clip}
done

最初にサウンドファイルの期間がsoxiユーティリティによって決定され、次に希望する長さのビデオがマウントされることに注意してください。私が見つけたすべての推奨事項はより単純です。- t$ {duration}フラグの代わりに、-shortestフラグが使用されています。実際、ffmpegはmp3の長さを大まかに決定します。インストール中、オーディオトラックの長さはビデオトラックの長さと大きく（1〜2秒）異なる場合があります。ビデオ全体が単一のフレームで構成されているかどうかは問題ではありませんが、このようなムービーを境界で途切れて接着すると、途方もない不快な効果が発生します。

mp3ファイルの期間を確認する別の方法は、mp3infoを使用することです。彼女も誤解され、時々ffmpegはmp3info以上のものを提供し、その逆もあり、時には両方がうそをつきます-私はパターンに気づきませんでした。しかしsoxiは正しく機能しています。

この便利なユーティリティをインストールするには、次の操作を行います。

sudo apt-get install sox libsox-fmt-mp3

スライド間の移行


トランジション-1つのスライドが別のスライドに変わる短いビデオ。このようなビデオを作成するには、スライドをペアで取り、imagemagickを使用して一方を他方に変換します。

#!/bin/bash

BUFFER=$(mktemp -d)

for pic in slide/*.png
do
  if [[ ${prevpic} != "" ]]
  then
    clip=${pic##*/}
    clip=${clip/.png/-0.mp4}
    #
    #  
    #
    ./fade.pl ${prevpic} ${BUFFER} 1280 720 5 direct 0
    ./fade.pl ${pic} ${BUFFER} 1280 720 5 reverse 12
    #
    #   
    #
    ffmpeg -y -hide_banner -i "${BUFFER}/%03d.png" -i sound/silence.mp3 -r 25 -y -acodec aac -vcodec libx264 -pix_fmt yuv420p -profile:v high -coder 1 -shortest clips/${clip}
    rm -f ${BUFFER}/*
  fi
  prevpic=${pic}
done

rmdir ${BUFFER}

何らかの理由で、私はドットでの散乱にスライドをしたかったし、次のスライドは、ドットから組み立てられ、このために、私はと呼ばれるスクリプトを書いたfade.pl持つImageMagickのを、実際のLinuxoidは、特別な効果を作成します誰かが散乱の私の考えを好きならば、ここにスクリプトがあります：

#!/usr/bin/perl

use strict;
use warnings;
use locale;
use utf8;
use open qw(:std :utf8);
use Encode qw(decode);
use I18N::Langinfo qw(langinfo CODESET);

my $codeset = langinfo(CODESET);
@ARGV = map { decode $codeset, $_ } @ARGV;

my ($source, $target, $width, $height, $pixsize, $rev, $file_no) = @ARGV;

my @rects;
$rects[$_] = "0123456789AB" for 0..$width*$height/$pixsize/$pixsize/12 - 1;

for my $i (0..11) {
  substr($_,int(rand(12-$i)),1) = "" for (@rects);
  my $s = $source;
  $s =~ s#^.*/##;
  open(PICTURE,"| convert - -transparent white PNG:- | convert \"$source\" - -composite \"$target/".substr("00".($file_no+$i),-3).".png\"");
  printf PICTURE ("P3\n%d %d\n255\n",$width,$height);
  for my $row (1..$height/$pixsize/3) {
    for my $j (0..2) {
      my $l = "";
      for my $col (1..$width/$pixsize/4) {
        for my $k (0..3) {
          $l .= (index($rects[($row-1)*$width/$pixsize/4+$col-1],sprintf("%1X",$j*4+$k))==-1 xor $rev eq "reverse") ? "0 0 0\n" : "255 255 255\n" for (1..$pixsize);
        }
      }
      print PICTURE ($l) for (1..$pixsize);
    }
  }
  close(PICTURE);
}
