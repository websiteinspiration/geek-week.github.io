タクシーアプリケーションのYandexマップ

位置情報、地図、ルート案内を使用するほとんどのアプリケーションでは、Googleマップを使用します。 Googleマップには豊富な機能セット、優れたドキュメント、視覚的およびプログラミングのアイデアを実装するための優れたマップキットがあるためです。残念ながら、Googleマップはすべての可能なタスクの実装の万能薬ではありません。問題は、会社が十分に理解していない地域や都市などがあり、そのために家の番号や道路全体さえも利用できない場合があることです。

提案依頼


プロンプト要求の最終バージョンに到達する前に、いくつかのメソッドが再確認されました。これは、SuggestSessionを介してプロンプトを受信する最後の最も効果的な方法です。

まず、SearchManagerオブジェクトとのインスタンスのインスタンス作成SuggestSessionのオブジェクトを：

private SearchManager searchManager;
private SuggestSession suggestSession = null;

onCreate（）メソッドでsearchManagerを初期化する

searchManager = SearchFactory.getInstance()
       .createSearchManager(SearchManagerType.COMBINED);

さらに、requestSuggest（String query）メソッドでは、suggestSessionオブジェクトを初期化します。毎回新しいセッションを作成する必要がないことを忘れないでください。

if (suggestSession == null)
   suggestSession = searchManager.createSuggestSession();

会社のヒントを取得するには、SuggestType.BIZパラメーターを使用します。ユーザーの地域のデータを取得するには、BoundigBoxを使用する必要があります。これは、次のように初期化されます。 

private final BoundingBox BOUNDING_BOX = new BoundingBox(
       new Point(66.066191, 76.468435),
       new Point(66.149326, 76.824836));

ここで、最初の点は長方形セクターの左下隅と2番目の右上隅です。

suggestSession.suggest(query, BOUNDING_BOX, new SuggestOptions()
               .setSuggestTypes(SuggestType.BIZ.value)
               .setUserPosition(CENTER),
       new SuggestSession.SuggestListener() {
           @Override
           public void onResponse(@NonNull List<SuggestItem> list) {
           }
           @Override
           public void onError(@NonNull Error error) {
           }
       });

通り、家などを取得する必要がある場合は、SuggestType.BIZパラメータを使用します。

suggestSession.suggest(query, BOUNDING_BOX, new SuggestOptions()
               .setSuggestTypes(SuggestType.GEO.value)
               .setUserPosition(CENTER),
       new SuggestSession.SuggestListener() {
           @Override
           public void onResponse(@NonNull List<SuggestItem> list) {
           }
           @Override
           public void onError(@NonNull Error error) {
           }
       });

また、指摘された問題の1つは、結果として同じ名前の場合に複数の同じ値が発生するという事実でした。そのため、すぐに並べ替えることをお勧めします。

そして今日私が共有したい最後のことはあなたのSuggestItemをリストに表示することです。オブジェクトにはいくつかのプロパティがあるため、最初はすべてが明確に見えました。

getSearchText();
getDisplayText();

私の意見では、これは検索用の行と表示用の行があることを意味しますが、99％の場合、正規表現を使用せずに単純な通りの名前を取得するためにこれらの行が一致します（これは表示に悪影響を与える可能性があるため、理解に必要なデータをトリミングできます：通りの名前、家の番号、アパートなど）、次の関数が使用されました：

suggestItem.getTitle().getText();

奇妙なことに、このオプションは必要な結果をもたらしますが、最初はこのオプションはそれほど明白ではなかったようです。

実際、これはこのアプリケーションで遭遇しなければならなかったほんのいくつかの小さなことです。記事を引き伸ばさないようにするために、ルートを作成し、マーカーを表示しているので、これを2番目の部分に残します。これは、タクシー、回転による地図上での車の動きなどのアプリケーションでも同様に重要です。
