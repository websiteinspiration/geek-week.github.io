ElectronでのMacOSX Home Assistantのマテリアルアイコンの作成

では最初のシリーズ、私はホームアシスタントのスマートホームを醸造しました。その過程で、私はHome Assistantに夢中になり、このシステム用のデスクトップアプリケーションを書くことを考えました。 Home Assitantは素晴らしいWebベースのインターフェースを備えており、Electronでラップして、美しいドックアイコン、ネイティブ通知、クイックアクセス用のメニューバー、その他のzeroconfを取得するのは妥当なようです。使用できるすべてのためのホームアシスタントマテリアルアイコン：ダッシュボード、状態、ボタン、すべて。つまり、マテリアルアイコンのセット全体をElectronアプリケーションに取り込む必要があります。

Electronは、nativeImage APIを使用して画像を処理します。これは、2020年にはベクター形式を受け入れず、PNGおよびJPGのみを使用するように設計されています。アイコンの圧縮アーチファクトは致命的であるため、JPGは適していません。そのため、PNGに転送されるフォーマットを取得するためにどこかに必要です。約15分間、素材アイコンのホームサイトに登った。MacOSXの素材アイコンの既製のパッケージを、さまざまなサイズと異なるdpiでpngにカットしたものを探していた...理由を言うのは難しいが、彼らはそのようなものを公開していない。自主制作する必要があります。

ダークモードのテンプレートメニューアイコンを使用すると、このように見え始め



、目標は達成されました。スクリプトは、すべての画像を目的の形式で抽出します


#!/bin/sh -x

set -e

# Run 'npm install @mdi/svg' beforehead
MDISVG_PATH='node_modules/@mdi/svg/svg/'

#Put inkscape and convert into PATH
PATH=$PATH:/Applications/Inkscape.app/Contents/MacOS/

mkdir -p images
convert -size 32x32 xc:none images/background16@2x.png
convert -size 16x16 xc:none images/background16.png

SIZES="
16,16
32,16@2x
"
FILES=`ls ${MDISVG_PATH} |grep svg`
for SVG in $FILES; do
    BASE=$(basename "$SVG" | sed 's/\.[^\.]*$//')
    for PARAMS in $SIZES; do
        SIZE=$(echo $PARAMS | cut -d, -f1)
        LABEL=$(echo $PARAMS | cut -d, -f2)
        inkscape -w $SIZE -h $SIZE "$MDISVG_PATH/$SVG" \
            -o "images/mdi-${BASE}${LABEL}".png
        convert -verbose \
            -composite images/background${LABEL}.png "images/mdi-${BASE}${LABEL}".png \
            -compose CopyAlpha \
            "images/${BASE}Template${LABEL}".png
    done
done
