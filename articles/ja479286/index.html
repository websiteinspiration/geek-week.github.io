Spring MVC、Spring Data JPA、およびHibernateを使用した単純なWebアプリケーションの作成

こんにちは、ハブル！Nam Ha Minhによるガイド「Spring MVC + Spring Data JPA + Hibernate-CRUD Example」の翻訳をお見せします。

このSpring Javaチュートリアルでは、クライアント情報を管理できる簡単なWebアプリケーションを開発することにより、Spring MVCアプリケーションを設定してSpring Data JPAと連携する方法を学びます。

このチュートリアルが完了すると、Spring MVCおよびSpring Data JPAテクノロジーに基づくJava Webアプリケーションを作成できます。


このガイドで使用するプログラムとテクノロジー：Java 8、Apache Tomcat 9、MySQL Server 5.7、Eclipse IDE 4.7（Oxygen）、Spring Framework 5.1、Hibernate 5.4、Spring Data JPA 2.1.5、およびServlet 3.1。

データベースの作成から始めましょう。

1.データベースの作成


MySQLを使用します。この例では、salesというスキーマにあるcustomerテーブルのデータを操作します。顧客テーブルには4つのフィールドがあります：id、name、emailおよびaddress：


次のMySQLスクリプトを実行して、スキーマとテーブルを作成できます。

CREATE DATABASE `sales`;
CREATE TABLE `customer` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `name` varchar(45) NOT NULL,
  `email` varchar(45) NOT NULL,
  `address` varchar(45) NOT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

このファイルに以下のコードを入力します。

4.モデルクラスの作成


次のように、データベースの顧客Customerテーブルにマップ するクラスを作成します。

package net.codejava.customer;
 
import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;
 
@Entity
public class Customer {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
 
    private String name;
    private String email;
    private String address;
 
    protected Customer() {
    }
 
    protected Customer(String name, String email, String address) {
        this.name = name;
        this.email = email;
        this.address = address;
    }
 
    //       .       IDE,  .
 
}

ご覧のとおり、アノテーションを使用@Entityしてこのクラスを顧客テーブルにマップしています（クラス名はテーブル名と同じです）。すべてのクラスフィールド名は、表のフィールド名と同じです。このフィールドにidは注釈@Idとがあり@GeneratedValue、このフィールドが主キーであり、その値が自動的に生成されることを示します。

5. Spring MVCおよびSpring Data JPAの構成


次に、Spring MVCとSpring Data JPAを構成するJavaコードを記述します。XMLよりも単純なので、Javaベースの構成を使用します。

Spring Dispatcherサーブレットを構成する


アプリケーションでSpring MVCを使用するには、アプリケーションを起動するときに、次のクラスを記述してSpring Dispatcherサーブレットを登録する必要があります。

package net.codejava.config;
 
import javax.servlet.ServletContext;
import javax.servlet.ServletException;
import javax.servlet.ServletRegistration;
 
import org.springframework.web.WebApplicationInitializer;
import org.springframework.web.context.support.AnnotationConfigWebApplicationContext;
import org.springframework.web.servlet.DispatcherServlet;
 
public class WebAppInitializer implements WebApplicationInitializer {
    @Override
    public void onStartup(ServletContext servletContext) throws ServletException {
        AnnotationConfigWebApplicationContext appContext = new AnnotationConfigWebApplicationContext();
        appContext.register(WebMvcConfig.class);
          
        ServletRegistration.Dynamic dispatcher = servletContext.addServlet(
                "SpringDispatcher", new DispatcherServlet(appContext));
        dispatcher.setLoadOnStartup(1);
        dispatcher.addMapping("/");
          
    }
}

onStartup()このクラス のメソッドは、アプリケーションのロード時にサーブレットによって自動的に呼び出されます。Spring Dispatcher Servletは、URL "/"に一致するすべてのリクエストを処理し、WebMvcConfig以下で説明するように、クラスで構成を探します。

Spring MVCを構成する


次のコードを含むWebMvcConfigパッケージに クラスを作成しますnet.codejava.config。

package net.codejava.config;
 
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.view.InternalResourceViewResolver;
 
@Configuration
@ComponentScan("net.codejava ")
public class WebMvcConfig {
    @Bean(name = "viewResolver")
    public InternalResourceViewResolver getViewResolver() {
        InternalResourceViewResolver viewResolver = new InternalResourceViewResolver();
        viewResolver.setPrefix("/WEB-INF/views/");
        viewResolver.setSuffix(".jsp");
        return viewResolver;
    }
}

このクラスには、@ConfigurationSpringに構成ファイルであることを伝える注釈が付けられています。アノテーション@ComponentScanは、パッケージ内の構成クラスを探すようにSpringに指示しますnet.codejava。

このクラスでは、これらのビューのプレフィックスとサフィックスを指定して、ビューを認識するBeanを作成します。したがって、JSPファイルを保管するためのディレクトリーviews内にディレクトリーWebContent/WEB-INFを作成します。

ここで、他のSpring MVC構成を追加できます。

Spring Data JPAの構成


春データJPAで動作するように、我々は2つのBeanコンポーネントを作成する必要がありますEntityManagerFactoryとJpaTransactionManager。したがって、別の構成クラスを作成しますJpaConfig。

package net.codejava.config;
 
import javax.persistence.EntityManagerFactory;
 
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.jpa.repository.config.EnableJpaRepositories;
import org.springframework.orm.jpa.JpaTransactionManager;
import org.springframework.orm.jpa.LocalEntityManagerFactoryBean;
import org.springframework.transaction.annotation.EnableTransactionManagement;
 
@Configuration
@EnableJpaRepositories(basePackages = {"net.codejava.customer"})
@EnableTransactionManagement
public class JpaConfig {
    @Bean
    public LocalEntityManagerFactoryBean entityManagerFactory() {
        LocalEntityManagerFactoryBean factoryBean = new LocalEntityManagerFactoryBean();
        factoryBean.setPersistenceUnitName("SalesDB");
          
        return factoryBean;
    }
      
    @Bean
    public JpaTransactionManager transactionManager(EntityManagerFactory entityManagerFactory) {
        JpaTransactionManager transactionManager = new JpaTransactionManager();
        transactionManager.setEntityManagerFactory(entityManagerFactory);
          
        return transactionManager;
    } 
}

ここでは、2つの重要な注釈を使用します。

  • @EnableJpaRepositories：実行時に適切なコードを挿入するには、指定されたパッケージ（net.codejava）でリポジトリクラスを探す必要があることをSpring Data JPAに通知します。
  • @EnableTransactionManagement：ランタイムトランザクションコードを生成するようにSpring Data JPAに指示します。

このクラスでは、最初のメソッドがEntityManagerFactory永続ユニットを制御するインスタンスを作成しますSalesDB（この名前は上記で示されていますpersistence.xml）。

最後のメソッドは、前のメソッドで作成されたのインスタンスJpaTransactionManagerをEntityManagerFactory作成します。

これは、Spring Data JPAを使用するために必要な最小構成です。

7.サービスクラスの作成


次に、クラスを作成しますCustomerService。

package net.codejava.customer;
 
import java.util.List;
 
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
 
@Service
@Transactional
public class CustomerService {
    @Autowired CustomerRepository repo;
     
    public void save(Customer customer) {
        repo.save(customer);
    }
     
    public List<Customer> listAll() {
        return (List<Customer>) repo.findAll();
    }
     
    public Customer get(Long id) {
        return repo.findById(id).get();
    }
     
    public void delete(Long id) {
        repo.deleteById(id);
    }
     
}

@Transactionalクラスをマーク する注釈に注意してください。つまり、このクラスのすべてのメソッドは、トランザクション管理のためにSpring Data JPAによってインターセプトされます。そしてインターフェースインスタンスCustomerRepositoryはこのクラスに埋め込まれます：

@Autowired CustomerRepository repo;

DAOコードを記述していないので、これは魔法のように見えますが、Spring Data JPAは実行時に自動的に実装を作成します。

ご覧のとおり、このクラスのすべてのメソッドはCRUD操作用です。呼び出し全体をオブジェクトに委譲するだけCustomerRepositoryです。このクラスは冗長に見えるかもしれませんが、ビジネス/サービスレベルをリポジトリ/ DAOレベルから分離する必要があります。

13.顧客で検索


最後に、ユーザーがキーワードを入力して顧客を検索できる検索機能を実装しましょう。検索機能は、名前、電子メール、アドレスの3つのフィールドでキーワードを検索しますCustomerRepository。これらのインターフェースについては、インターフェースで独自のメソッドを記述する必要があります。

package net.codejava.customer;
 
import java.util.List;
 
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.CrudRepository;
import org.springframework.data.repository.query.Param;
 
public interface CustomerRepository extends CrudRepository<Customer, Long> {
     
    @Query(value = "SELECT c FROM Customer c WHERE c.name LIKE '%' || :keyword || '%'"
            + " OR c.email LIKE '%' || :keyword || '%'"
            + " OR c.address LIKE '%' || :keyword || '%'")
    public List<Customer> search(@Param("keyword") String keyword);
}

メソッドsearch()は、単にヘルプで注釈が付けられた抽象的なメソッド@Queryです。検索クエリはJPAクエリです。

次に、メソッドをクラスに追加しCustomerServiceます。

public List<Customer> search(String keyword) {
    return repo.search(keyword);
}

次に、ハンドラメソッドをクラスに追加しますCustomerController。

@RequestMapping("/search")
public ModelAndView search(@RequestParam String keyword) {
    List<Customer> result = customerService.search(keyword);
    ModelAndView mav = new ModelAndView("search");
    mav.addObject("result", result);
 
    return mav;    
}
