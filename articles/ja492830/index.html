SameSite =デフォルトでLax-既にChrome 80で安定（まだ誰もが利用できるわけではありません）

先日、丁寧な同僚（ありがとう、Lena）が奇妙なバグを報告しました-サーバーは通常、ブラウザにCookieを設定しましたが、戻ってきませんでした。すべてが前日に機能しましたが、Cookieが設定されましたが、数秒後に魔法のように消えました（ただし、24時間続くはずです）。それはチームの少数の人々と新しいChrome 80でのみ再現されましたが、Chromeの残りの部分はまったく同じバージョンで、すべてが順調でした。他のブラウザでは、すべてが時計のように機能しました。神秘的な。彼らはそれを理解し始め、しばらくして、ChromeコンソールはCookieを設定する応答ヘッダーの警告に気づきました：
A cookie associated with a cross-site resource at _your_domain_ was set without the `SameSite` attribute. A future release of Chrome will only deliver cookies with cross-site requests if they are set with `SameSite=None` and `Secure`.

彼らはそれが何であるかを研究し始め、徐々にエラーがどのように発生し、なぜ誰もがそれを示さなかったのかが明らかになりました。私たちのサービスは異なるドメインからのリクエストを処理する必要があるため、これSameSiteはまさに私たちのケースです。彼らは追加SameSite=None; Secureし、問題は私たちのために解決されました。

何故ですか？


最初に、HTTP標準に従って、ブラウザは対応するドメインへの要求に対してCookie（または単にCookie）を送信しました。これにより、CSRF攻撃の機会が開かれました。つまり、特定の状況下で、悪意のない人々が、クッキーを使用するだけで、疑いのない善意を装っていくつかのリソースにアクセスできるようになりました。

2016ではSameSite、ページが別のドメインにリクエストを送信した場合にブラウザがCookieを送信するかどうかを制御できる属性が導入されました。リクエストがサードパーティのサイトから実行された場合、開発者はCookieの転送をブロックすることができます。このため、属性SameSiteを明示的にStrictに設定する必要がありました。（Cookieは、リクエストおよびそれらが属するドメインからの移行時にのみ送信されます）、またはLax（Cookieは、ダイレクトリンクによって他のサイトからサイトへの移行時に送信されますが、AJAXリクエストや画像のアップロードなど、他のサイトからのリクエストでは送信されません。 ）不在SameSiteは同等でしたSameSite=None。つまり、デフォルトでは、Cookieはどの要求でも送信されていました。

2019年5月、Google Chromeデベロッパーは、この動作を徐々に変更し、SameSiteがないことをSameSite=Lax（詳細はこちら））つまり、デフォルトでは、ブラウザは、リンクを直接クリックする場合を除いて、現在のページから他のリソースへのリクエストに対するCookieの送信をブロックします。 FirefoxとEdgeの開発者は、時間の経過とともに、この変更をブラウザにも実装することを発表しました。

ユーザーにとって、このブラウザの動作はより安全です。サイトおよびWebサービスの開発者は、他のサイトから要求するときにCookieを引き続き受信する必要がある場合は、Cookie属性を明示的に設定する必要があることに注意する必要がありますSameSite=None, Secure（Secureこのような要求は、さらに、安全なチャネル経由でのみサーバーに送信されるためです）

イノベーションにより現在の標準が変化し、すべてのサイトが迅速に適応できるわけではないため、新しい動作への移行は徐々に発生します。当初は限られた数のChromeベータ版ユーザー向けにオンになっていたが、現在は徐々に増加している。実際の情報はこちらから入手できます。リンクの最新のエントリ：
最終更新日：2020年3月9日

。100％のユーザーではなく、Chrome 80の安定版に新しい動作の適用を開始しました。制御されたロールアウトは、限られた初期母集団に対して行われます。
つまり、ベータ期間は遅れており、Chrome開発者は一般ユーザーに新しい機能を徐々に取り入れ始めています。これは本当に安全なので、原則的には良い方法です。設定を使用し

て以前の動作を復元できます。Chromeのアドレスバーで、と入力chrome://flagsし、ページに移動して、SameSiteをデフォルトのCookieアイテムとして見つけ、それを[ 無効]に設定します。ただし、これは通常のユーザーよりもテストに適しています。同様に、Chrome 80が以前の動作を実行している場合は、同じ設定を[ 有効]に設定することで、新しい機能を強制できます。

したがって、サイトが別のドメインからリクエストするときにCookieを受信する必要がある場合は、次の変更に備えてください（準備ができている人なら誰でもいいです）。すべてに良い！
