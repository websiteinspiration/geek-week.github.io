Visual Studio CodeのPython-2020年1月リリース

Visual Studio Code用のPython拡張機能の2020年1月リリースを発表できることをうれしく思います。拡張機能  はマーケットプレイスからダウンロードするか、Visual Studio Codeの拡張機能ギャラリーから直接インストールでき  ます。Python拡張機能が既にインストールされている場合は、Visual Studio Codeを再起動して最新のアップデートを入手できます。 ドキュメントで、Visual Studio CodeのPythonサポートについて  詳しく読むことができます。  



このリリースでは、次の59の問題を解決しました。 

  • Jupyterノートブックでカーネルを選択する  
  • Jupyter Notebook Editorのパフォーマンスの問題
  • 負荷のかかった端末の環境の自動アクティブ化（Igor Aleksanovに感謝  ！） 
  • 保存中および起動時のctagsリカバリーに関する問題を修正 

興味がある場合は、ジャーナルですべての変更を調べることができます  。 

Jupyter Notebooksでのカーネルピック 




ノートブックエディター  と  インタラクティブウィンドウの 右上隅に、  ラップトップが使用している現在のカーネルとカーネルの状態（スタンバイモード、ビジー状態など）が表示されます。このリリースでは、カーネルを他のPythonカーネルに変更することもできます。現在アクティブなカーネルを変更するには、現在のカーネルをクリックしてVS Codeカーネルセレクターを呼び出し、切り替え先のカーネルをリストから選択します。

Jupyter Notebook Editorのパフォーマンスの向上 


このリリースには、ノートブックエディターとインタラクティブウィンドウの両方で、VS Codeの多くのJupyterパフォーマンスの改善が含まれています。これは、以前のカーネルをキャッシュし、Jupyterの検索を最適化することで達成されました。これらの変更によってもたらされる重要な改善のいくつかは次のとおりです。

  • Jupyterサーバーの最初の起動は高速で、Jupyterサーバーのその後の起動は2倍以上高速です  
  • 新しい空のJupyterラップトップを2倍速く作成する 
  • Jupyter Notebookを開く（特にファイルサイズが大きい場合）の速度が2倍になりました 

注：これらのパフォーマンス計算は当社のテストで測定されたものであり、改善点は異なる場合があります。

負荷のある端末の環境の自動アクティブ化 


ワークスペースで仮想環境またはconda環境が選択されている場合に新しいターミナルを作成すると、Python拡張機能により、この新しいターミナルで選択された環境がアクティブになります。今回のリリースには、Python拡張機能をロードした直後に、すでに開いているターミナルで環境を自動的にアクティブ化する機能が含まれています。 この機能を有効にするには、パラメーター「python.terminal.activateEnvInCurrentTerminal」：trueをsettings.jsonファイルに追加します。次に、拡張機能が読み込まれ、ターミナルがVS Codeで開かれると、選択した環境が自動的にアクティブになります。





保存中および起動時のctagsリカバリーに関する問題を修正 


ctags ツールは、ユーザーのワークスペースキャラクターの作成を担当します。その結果、ドキュメントスキームはファイルシンボルで埋められ、作業領域でこれらのシンボル（たとえば、関数）を簡単に見つけることができます。 

このリリースでは、ctagsに関連するGitHubリポジトリ（GH793）で最も頻繁に言及されるエラーメッセージが修正されています。これで、プロジェクトの.vscodeフォルダーに保存されているタグは、Python拡張機能が読み込まれたときに「python.workspaceSymbols.rebuildOnStart」をtrueに設定して再構築するか、「python.workspaceSymbols.rebuildOnFileSave」をtrueに設定して各保存ファイルを再構築できます。 ctagsサポートの詳細については、ドキュメントをご覧ください  。 





その他の変更 


また、ユーザーから要求されたマイナーな改善とバグ修正を追加しました。これにより、Visual Studio CodeでのPythonのエクスペリエンスが向上します。主な変更点は次のとおりです。

  • ラップトップまたはインタラクティブウィンドウ内でユーザーから情報を受信する機能のサポート。（＃8601）
  • ラップトップのマーケットダウンおよび出力におけるローカルイメージのサポート。（＃7704）
  • インタラクティブウィンドウまたはラップトップ内でのチャートの保存のサポート。（＃7221）
  • Pythonが実行されており、Anacondaが選択されているときに「conda run」を使用する。（＃7696）
  • 名前が使用できない場合は、パスを使用してconda環境をアクティブ化します。（＃3834）
  • QuickPick Run All/Debug All . ( ) ( # 5608 )

私たちは常に新しい機能をテストしています。チームによって発表されていない何かを見つけた場合、おそらくあなたはテストに参加しています！テストに関与しているかどうかを確認するには、Python拡張機能の出力の最初の行を確認できます。A / Bテストを拒否する場合は、ユーザーのsettings.jsonファイルを開き（[表示]> [コマンドパレット]を選択し、[設定：設定を開く（JSON）]を実行）、 " python.experiment.enabled"をfalseに設定します。

必ずPythonの拡張機能をダウンロード上記の改善を試して、今のVisual Studioのコードのために。問題が発生した場合は、GitHub Python VSコードページで報告してください。
