私は写真家であり、自分自身を作業ツールにします

「中国人」は写真家のためのデバイスの分野を含め、非常にクールなことをしています。過去10年間、品質は非常に厳しくなり、彼らは独自のユニークな製品を開発し始めました。これは「著名なメーカー」よりも頭が高いです。しかし、何らかの理由で、ほとんどすべての製品に少なくとも1つはありますが、非常に大きなマイナス記号があります。エレクトロニクスの開発者は、tk fadを正しく入れているようです：「軟膏で飛ぶ-少なくとも200グラム」。

少し前まで、私は一定の光に身を任せることを決め、いくつかの「電柱」yongnuo yn-360を購入しました。スティックは魔法のようであることが判明しましたが、もちろんファットマイナスがありました。リモートコントロール用のAndroidアプリケーションは、実際には機能していないことが判明しました。アプリケーションのマイナーな欠点に加えて-主なものは、一度に複数のデバイスを制御できないことです。このマイナスにより、スティックからスティックに移動して手動でライトを調整する必要があったため、コントロール機能が役に立たなくなりました。

私はプログラミングの経験があったので、数日の間に状況は修正されました。



開発ツールの選択


私はプロの開発者ではありません。プログラミングは私の趣味であり、したがって、テクノロジーを勉強するのに多くの時間を費やす準備ができていません。たまたま私の好きな言語はpythonですが、モバイル開発にはあまり適していません。もちろん、kivyのような素晴らしいプロジェクトはありますが、Python（およびJavaScript）で使用するのに十分に慣れている「バッテリー」は本当に十分ではありません。その結果、私はすでに1つのプロジェクトを実行しているApache Cordovaを使用してJSに選択肢がありました（ちなみに、これは私の仕事に少し関連しています-このアプリケーションを使用して、マーケティング担当者がここで読むために削減したクワッドコプターのDJIスパークを「ポンプ」しました）。コルドバで開発することの利点の中で、これは開発のスピードと容易さ、そして「バッテリー」の存在です。

どのように機能しますか


これらのスティックがスマートフォンでどのように機能するかを推測することはまったく難しいことではありません。それはBluetooth LEです。幸運なことに、私の前に誰かがこの質問に戸惑っていて、数分で、スティックスプロトコルをスパイできるpythonプロジェクトを見つけました。まったく複雑ではないことがわかりました。 BLEで動作するように、コルドバは、プラグイン見つけBLEを、そしてそれが動作するプロトタイプを書くために1夜を正確に取りました。

すべては、Bluetooth LEデバイスのスキャンから始まります。しかし、不必要なものを検索から除外するために（そしてその周りにたくさんありました）、目的のサービスのUUIDを設定してスキャンできます。次に、それに接続して、サービスと特性を取得します。必要なサービスと特性のUUIDは、たとえば、BLEスキャナーアプリケーションを使用して見つけることができます。

これにより、次のような簡単なコードが作成されます。

var devices_dev={};
var devices=[];
var dev_main={};
function start_scan(){
         evothings.ble.startScan(
            function(device)
            {
                //      
                if (! (device.address in devices_dev)){
                    //     
                    devices_dev[device.address]=1;
                    // 
                    evothings.ble.connectToDevice(
                            device,
                    function(dev)
                 {
                        //  ""
                         var service = evothings.ble.getService(dev, 'f000aa60-0451-4000-b000-000000000000');
                        //   
                         var characteristic = evothings.ble.getCharacteristic(service, 'f000aa61-0451-4000-b000-000000000000');
                         //     .      .
                         dev_main[dev.name]=[dev,characteristic];
                         devices.push(dev.name);
                  }
            },
            function(error)
            {
                                console.log(error);
            },
            //   UUID ,   .
            { serviceUUIDs: ['f000aa60-0451-4000-b000-000000000000'] }
            )
}

ちなみに、同じ原理で、他のBLEデバイスに接続できます。私は体重を増やすことができるスマートXiaomi体重計に夢中です。また、レドモンドからスマートケトルをオンにすることもできました（現在の温度を確認することもできます）。

その後は、ライトの色や明るさに関する情報を送信するだけです。これらのスティックにはいくつかのタイプのLEDがあります。コールド5500k、ウォーム3200k、RGBダイオードです。明るさを冷たくまたは暖かく（0〜99％）設定するか、RGBダイオードをオンにできるように作られています。
コールドダイオードまたはウォームダイオードの輝度を設定する必要がある場合は、次のシーケンスを送信します。

var white=0; //  5500  0-99
var yellow= 50; // 3200  0-99
var data= new Uint8Array([0xAE,0xAA,0x01,white,yellow,0x56]);

色付けされている場合：

var RR=255; //   0-255
var GG=0; //    0-255
var BB=; //   0-255
var data=new Uint8Array([0xAE,0xA1,RR,GG,BB,0x56]);

送信関数自体は次のようになります。

evothings.ble.writeCharacteristic(
    dev,
    characteristic,
    data, // Buffer view with data to write
    function()
    {
     console.log('done!');
    },
    function(errorCode)
    {
        console.log('writeCharacteristic error: ' + errorCode);
    });

アプリケーションのさらなる作業の過程で、スティックは迷惑なバグを検出しました。これは検出に長い時間がかかりました。たとえば、「白50％」、次に「赤く点灯」、そして再び「白50％」を送信した場合、最後のコマンドスティックが反応しなくなった。そのような場合は松葉杖を挿入し、このシーケンスの最後のチームにランダムにプラス/マイナス1％を送信する必要がありました。

UIと「チップ」


標準のhtml + js + cssスタックで作業する必要があるため、既製の多くの開発を利用できます。マテリアライズcssがUIに対して選択され、色の選択iro.jsがUIのメインコンポーネントとして選択されました。

アプリケーションをテストする過程で、個別のRGBダイオードの機能、つまりひどいカラーミックスが出てきました。異なる色の少なくとも1％を赤に追加すると、奇妙な色の混乱が発生します。色の選択は通常（公式アプリを含む）、標準の「カラーホイール」によって実装され、「純粋な赤」などを得るのが非常に困難です。カラーダイオードのグループをチャネルごとに制御する必要がありました。

まあ、実際の作業に役立つチップもいくつか作られました。たとえば、スティックの名前を変更します。シリアル番号を頭に置いておく必要はなく、どちらがバックグラウンドに立っているかを知る必要はありません。「バック」、「レフト」などの名前を指定するだけです。さて、ライトを所定のライトにすばやく変更する必要がある場合、いくつかのプリセットがありました。



同僚とのコミュニケーションの過程で、これらのスティックは写真家の間ではなく、ビデオを撮影する人々の間でより人気があることがわかりました。そして、私がそれを管理するためのアプリケーションを作成していると彼から聞いたとき、彼は、1つの機能を実装すると、このアプリケーションは予算スティックを別のメーカーのモデルのアナログに変え、コストが5倍になると書いています。

"アニメーション"


ビデオ撮影では、たとえば、バックグラウンドでのパトカーの「ライト」など、さまざまな照明効果が必要になる場合があります。アプリケーションがライトを「手動」で制御できる場合は、「アニメーション」を実装してみませんか。当初、アイデアは10〜20種類のアニメーションを縫い合わせるために生まれましたが、私はそれらに座るのが面倒なので、さらに2晩過ごした後、シンプルなWebベースのアニメーションエディターを作りました。さらに、最大10チャンネルを編集して、スティック間のアプリケーションでそれらを配布できます。



これらのスティックの唯一の欠点は、より高価な同等のものと比較して、かなり遅いことが判明しました。コマンドの送信から色の切り替えまで、300〜400ミリ秒に達するため、高速でより安定した動作を実現することはできません。
