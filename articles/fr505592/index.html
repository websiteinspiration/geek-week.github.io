Programmation pour BK0010 avec Android



L'histoire s'est produite dans le canal télégramme BK00010, j'étais un participant.

La question s'est posée: comment écrire des programmes? L'émulateur ne prend pas en charge la gravure de disques, il est donc peu probable que l'utilisation de l'assembleur dans l'émulateur réussisse. Pour couronner le tout, l'image sur les smartphones est très petite.

Une solution a été proposée (smaslovski), que je décrirai. Vous devez télécharger Termux, la version pour Android 5.0 est disponible ici , le fichier termux-app-git-debug.apk. La version du programme est 0.83, il prend en charge la personnalisation de la barre de touches.

Après avoir installé et démarré Termux, j'ai reconfiguré les clés, la procédure est la suivante:

mkdir $HOME/.termux

echo "extra-keys = [['ESC','/','-','HOME','UP','END','PGUP'],['TAB','CTRL','ALT','LEFT','DOWN','RIGHT','PGDN']]" > $HOME/.termux/termux.properties

termux-reload-settings

C'est devenu plus pratique



Maintenant, vous devez installer la finition nécessaire , MACRO-11 sera utilisé comme assembleur pour créer .BIN:
pkg install build-essential
pkg install git




mkdir src
cd src
git clone https://gitlab.com/Rhialto/macro11.git

J'ai ensuite installé le shell:

pkg install mc

et éditeur de texte

pkg install nano

dans le dossier src \ macro11 dont vous avez besoin pour corriger le Makefile , modifiez le paramètre

OBJFORMAT = -DDEFAULT_OBJECTFORMAT_RT11=1

tapez make et attendez le résultat,

copiez les fichiers reçus

cp macro11 $PREFIX/bin

cp dumpobj $PREFIX/bin

Pour compiler, vous avez besoin d'un script pour construire

cd ~/src 
git clone https://github.com/smaslovski/pdp11dasm.git
cd pdp11dasm/utils
chmod 755 obj2bkb
cp obj2bkb $PREFIX/bin

script de compilation, j'ai appelé compiler :

#!/data/data/com.termux/files/usr/bin/sh
n=$1
macro11 $n.mac -o $n.obj -l $n.lst
obj2bkb --binary --base=01000 -o $n.bin $n.obj
du -b $n.bin
cp $n.bin /sdcard

(vous pouvez créer une nano-compilation )

Le script doit être copié

chmod 755 compile
cp compile $PREFIX/bin

Pour les tests, j'ai écrit un simple programme gg.mac :



.title proga
.psect code
.=1000

 MOV #40000,R0
M: mov r0,(r0)+
 tst r0
 bpl m
 halt
.end

lancement: compilez gg
le programme gg.bin peut être ouvert dans l'émulateur.

Et, en conclusion, je suis censé écrire quelque chose d'optimiste et de gai, mais je vais enfreindre les règles et donner un exemple de programme que j'ai écrit le soir:

	.title small
	.ASECT
	.=1000

 ;clr r0
 ;clr r1
M:
; inc r0
 add pc,r1
; mov r1,r2
 bic #49152.,r1
; add #40000,r1
 sub r1,40000(r1)
 br M
	.end
