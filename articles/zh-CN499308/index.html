DotNext 2019莫斯科的十大报告

11月初，举行了DotNext 2019莫斯科会议，而我们在等待下一个DotNext会议的时候，是时候查看莫斯科会议的十大报道了。核心，低级详细信息，最高性能，最佳实践，体系结构等等-所有详细信息均已收集供您使用Golodnyj 在猫下。



异步流


演讲者：Stephen Cleary
地点：10
→ 报告的介绍

Stephen是Stack Overflow的前100名用户之一，打开了莫斯科DotNext的前十名报告。该报告从对故事的一点洞察开始，Stephen解释了为什么异步很重要。然后，《 C＃Cookbook中的并发性》一书的作者彻底地安排了C＃异步流，并介绍了JS和Python。



实用内存管理


演讲者：Anatoly Zhmur
地点：9
→ 报告的介绍Anatoly Zhmur 的一份

不寻常的报告以“何时帮助GC而何时不帮助GC”为主要信息，排在第9行。该报告的开头特别令人高兴的是天文学家和历史爱好者。从工程角度来看，Anaatoly完美地适用于使用GC的各种情况。该报告以一系列“秘密GC设置”和“ .NET GC的经验定律”作为结尾。



.NET开发人员想要了解的有关深度学习的所有信息，但又害怕问


演讲者：Dmitry Soshnikov
地点：8位

技术布道者和微软经验丰富的演讲者之一，在每小时的报告中试图揭穿.NET中没有机器学习的神话。在德米特里（Dmitry）的报告中，很大一部分致力于机器学习和神经网络的理论基础。同时，该报告提供了许多使用.NET的实际示例，而不是许多Python所不喜欢的示例。



从ETW到EventPipe：深入了解.NET监视管道，以获取更健康的应用程序


演讲者：克里斯托夫·纳萨尔-苏利埃
广场：7
→ 报告的介绍克里斯托夫第二天

的第一份报告收集了很多积极反馈。在他的演讲中，他展示了如何使用Linux作为.NET Core上的示例，使用CLR事件监视应用程序。



创作的力量


演讲者：Scott Wlaschin
地点：6
→ 报告的介绍

乐高设计师的粉丝将特别喜欢观看Scott Vlaschin的闭幕演讲。Scott是函数式编程的狂热普及者之一，也是Domain Modeling Made Functional的作者。该报告可以被视为会议上最好的报告之一，并建议以几乎任何编程语言向同事查看。



.NET 5：运行时


演讲者：Egor Bogatov
地点：5
→ 报告的介绍

Egor在其报告中对.NET 5的创新进行了分析，对于那些最近在.NET领域中遗漏某些东西的人来说，这是一个很好的报告。听到有关LLVM的一些数学优化也很有趣。



可空引用类型-高级指南


演讲者：Andrey Dyatlov
地点：4
→ 报告的介绍

来自ReSharper Andrey Dyatlov的开发人员之一，有关该语言新功能的使用的有用报告-Nullable引用类型。同时，一位同事讨论了如何在一个已经很庞大的重要项目中快速开始使用一项新功能，以及如何解决不同的地区案例。当然，您可以在报告末尾进行小测验。



.NET中的分布式请求跟踪


演讲者：Egor Grishechko
地点：3
→ 报告的介绍

Egor Grishechko的演讲被列入最受欢迎的三大演讲中。一方面，他的报告是理论性的，另一方面，对于那些首次在.NET中面临分布式查询跟踪的人而言，这是完全实用的。绕开分布式应用程序，微服务这一主题是不可能的，而这仅是当前的现实，因此没有必要。 Yegor简要扼要地介绍了Jaeger，这是一种由Uber编写的用于在分布式系统中收集跟踪标签的系统。而且，如果您打算使用微服务体系结构，那么您将在此报告中找到很多有用的信息。



Platform Wars
演讲者：Cyril Skrygan
地点：2

JetBrains的Cyril Skrygan的开幕主题演讲攀升至报告顶部的第二行。这是当之无愧的，因为Cyril首先对IDE领域的某些产品解决方案进行了非常有趣的历史分析，然后转向平台级别的产品解决方案。该报告很好地阐明了大公司的决策，如果您通过开发人员的眼光看待它，则有助于了解商业世界中正在发生的事情。



神奇的生物技术及其居住地


演讲者：亚历山大·潘钦
地点：1

位生物科学候选人亚历山大·潘钦燃烧弹讲述了恐龙的复活，克隆，人造病毒的产生和人类基因改造。跟亚历山大一起去，跟随正确和错误的科幻小说是非常有趣的。
