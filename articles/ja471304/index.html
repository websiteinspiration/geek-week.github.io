エンパイアERP。面白い会計：総勘定元帳、勘定、残高

この記事では、「血なまぐさい企業」の核心である会計について詳しく説明します。最初に、総勘定元帳、勘定、貸借対照表の調査を行い、それらの固有の特性とアルゴリズムを特定します。Pythonとテスト駆動開発テクノロジーを使用しています。ここではプロトタイピングを行うので、データベースの代わりに、基本的なコンテナーであるリスト、辞書、タプルを使用します。このプロジェクトは、Empire ERPプロジェクトの要件に従って開発されています。


タスク


宇宙...帝国の惑星...地球全体の1つの状態。人口は2週間で2時間働き、2年後には引退します。勘定体系は12のポジションで構成されています。アカウント1〜4はアクティブ、5〜8はアクティブ/パッシブ、9〜12はパッシブです。Horns＆Hooves社。すべてのトランザクションは1つのレポート期間で実行され、期間の初めには残高がありません。

class Balance(list):
    def __init__(self, accounts):
        self.accounts = accounts
        self.suma = Decimal(0.00)
        self.sump = Decimal(0.00)

    def create(self):
        self.suma = Decimal(0.00)
        self.sump = Decimal(0.00)
        for i in self.accounts.range:
            active = self.accounts[i].end if self.accounts[i].end >= 0 else Decimal(0.00)
            passive = -self.accounts[i].end if self.accounts[i].end < 0 else Decimal(0.00)
            self.append((active, passive))
            self.suma += active
            self.sump += passive
