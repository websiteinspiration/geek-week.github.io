来自Docker容器的Webstorm NodeJs解释器

本文介绍了使用位于Docker容器中的NodeJs解释器作为项目部署NodeJs应用程序的过程（默认情况下）。该解决方案很可能仅与Linux OS用户相关，因为最终解释器将取自DockerHub上的官方节点映像。


做什么的？


可以仅在主机上使用docker部署应用程序，无需自己安装解释器，这将使开发人员可以为特定项目使用特定版本的解释器，并且在项目中更改映像版本时，开发人员将无需手动配置，将使用新的解释器实例自动投影。


决断


例如，在ReactJs + Typescipt上创建了一个应用程序，因为如果解释器配置不正确，至少代码突出显示会中断。


1.从完成的模板创建一个应用程序：


$ npx create-react-app reactjs-typescript-custom-nodejs --template typescript

2.编写一个Dockerfile：


FROM node:14
WORKDIR /app

CMD cp /usr/local/bin/node /app/docker/node \
    && npm install \
    && npm run start

CMD, Docker /app/docker/node, /app . .


3. node docker/node .gitignore, :


*

4. :


docker build -t client . && docker run -it --rm --volume=${PWD}:/app -p 3000:3000 --name client client
