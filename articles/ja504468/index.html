Raspberry Piのパフォーマンス：ZRAMを追加してカーネルパラメーターを変更する

数週間前、私はPinebook Proのレビューを公​​開しました。Raspberry Pi 4もARMに基づいているため、前の記事で説明した最適化のいくつかは、それに非常に適しています。これらのトリックを共有し、同じパフォーマンスの向上があるかどうかを確認したいと思います。自宅のサーバールームに

Raspberry Piをインストールした後、RAM不足時に、非常に応答が遅くなり、ハングすることさえあることに気付きました。この問題を解決するために、ZRAMを追加し、カーネルパラメータにいくつかの変更を加えました。

Raspberry PiでのZRAMアクティベーション




ZRAMは、/ dev / zram0（または1、2、3など）という名前のRAMにブロックストレージを作成します。そこで記録されたページは圧縮され、メモリに保存されます。これにより、非常に高速なI / Oが可能になり、圧縮によりメモリが解放されます。

Raspberry Pi 4には、1、2、4、または8 GBのRAMが搭載されています。1 GBモデルを使用するので、モデルに応じて手順を調整します。1 GBのZRAMでは、デフォルトのスワップファイル（遅い！）の使用頻度は低くなります。このようなzram-swapスクリプトを使用して、インストールと自動構成を行いました。

手順は、上記のリンクのリポジトリで提供されています。インストール：

git clone https://github.com/foundObjects/zram-swap.git
cd zram-swap && sudo ./install.sh

設定を編集する場合：

vi /etc/default/zram-swap

さらに、インストールによってZRAMをアクティブ化できますzram-tools。この方法を使用する場合は、必ずファイル内の構成を編集し、/etc/default/zramswap約1 GBのZRAMをインストールしてください。

sudo apt install zram-tools

インストール後、次のコマンドでZRAMリポジトリの統計を表示できます。

sudo cat /proc/swaps
Filename				Type		Size	Used	Priority
/var/swap                               file		102396	0	-2
/dev/zram0                              partition	1185368	265472	5
pi@raspberrypi:~ $

ZRAMをより効果的に使用するためのカーネルパラメータの追加


Raspberry Piが最後の時点でスワップに切り替わったときのシステムの動作を修正します。これにより、フリーズにつながることがよくあります。/etc/sysctl.confファイルに数行を追加して再起動します。

これらの行は、1）メモリーの枯渇を遅らせ、カーネルキャッシュへの圧力を高めます。2）メモリーの枯渇の準備を早めに開始し、事前にスワップを開始します。しかし、それはZRAMを介した圧縮メモリのはるかに効率的なスワップになります！/etc/sysctl.conf

ファイルの最後に追加する行は次のとおりです。

vm.vfs_cache_pressure=500
vm.swappiness=100
vm.dirty_background_ratio=1
vm.dirty_ratio=50

次に、システムを再起動するか、次のコマンドで編集をアクティブにします。

sudo sysctl --system

vm.vfs_cache_pressure = 500は、キャッシュへの圧力を増加させます。これにより、カーネルは、ディレクトリオブジェクトとインデックスのキャッシュに使用されるメモリを再利用する傾向が高まります。長期間、使用するメモリが少なくなります。生産性の急激な低下は、以前のスワップによって無効になります。

最初にZRAMを使用するため、vm.swappiness = 100は、カーネルがメモリのページをどの程度積極的にスワップするかというパラメーターを増やします。

vm.dirty_background_ratio = 1＆vm.dirty_ratio = 50-バックグラウンドプロセスは、1％の制限に達した直後に記録を開始しますが、システムは、dirty_ratioが50％に達するまで同期I / Oを強制しません。

これらの4行（ZRAMで使用する場合）は、必然的にRAMが不足し、私のようにスワップへの切り替えを開始した場合に、パフォーマンスの向上に役立ちます。この事実を知っているだけでなく、ZRAMでのメモリ圧縮を考慮に入れて、このスワップを事前に開始することは3倍優れています。

実際にカーネルに伝えているため、キャッシュへのプレッシャーは助かります。「ねえ、聞いてください。キャッシュに使用する余分なメモリがないので、できるだけ早くそれを取り除き、最も頻繁に使用される重要なデータのみを保持してください。」

キャッシュが削減されていても、インストールされたメモリのほとんどが占有されている場合、カーネルは日和見スワップをはるかに早く開始するため、プロセッサ（圧縮）とスワップI / Oは最後にプルされず、すべてのリソースを一度に使用するのが遅すぎます。ZRAMは圧縮に少しCPUを使用しますが、メモリの量が少ないほとんどのシステムでは、これはZRAMなしのスワップよりもパフォーマンスへの影響がはるかに少ないです。

結果をもう一度見てみましょう。

pi@raspberrypi:~ $ free -h

pi@raspberrypi:~ $ sudo cat /proc/swaps

ZRAMの264448は、ほぼ1ギガバイトの非圧縮データです。すべてがZRAMに送られ、非常に遅いスワップファイルには何も入りませんでした。これらの設定を自分で試してみてください。これらはすべてのRaspberry Piモデルで機能します。私の不適切な冷凍システムは、実用的で安定したものになりました。
