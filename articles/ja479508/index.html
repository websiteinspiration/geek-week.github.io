PostgreSQLアンチパターン：有害なJOINとOR

バッファをもたらす操作を恐れてください...
例として小さなクエリを使用して、PostgreSQLでクエリを最適化するいくつかの普遍的なアプローチを検討してください。それらを使用するかどうかはあなた次第ですが、それらについて知っておく価値はあります。

PGの将来の一部のバージョンでは、状況はスケジューラーの「知恵」によって変わる可能性がありますが、ここでの例のように、9.4 / 9.6の場合はほぼ同じように見えます。

私は非常に現実的な要求をします：

  TRUE

  "" d
INNER JOIN
  "" doc_ex

    t_doc."@" = d.""

  (d."3" = 19091 or d."" = 19091) AND
  d."$" IS NULL AND
  d."" IS NOT TRUE AND
  doc_ex.""[1] IS TRUE AND
  t_doc."" = ''

BitmapOrとUNION


場合によっては、ビットマップヒープスキャンに多額の費用がかかることがあります。たとえば、私たちの状況では、十分なレコードが必要な条件に該当する場合です。プランでBitmapOr演算に変わったOR条件が原因でそれがわかりました。
元のタスクに戻りましょう- いずれかの条件に一致するレコードを見つける必要があります。つまり、両方の条件で59Kのすべてのレコードを検索する必要はありません。1つの条件を解決し、最初の条件で何も見つからなかった場合にのみ2番目の条件に移動する方法があります。このデザインは私たちに役立ちます：
(
  SELECT
    ...
  LIMIT 1
)
UNION ALL
(
  SELECT
    ...
  LIMIT 1
)
LIMIT 1

「外部」のLIMIT 1は、最初のレコードが見つかったときに検索が完了することを保証します。そして、それが最初のブロックに既にある場合、2番目は実行されません（計画では決して実行されません）。

「CASEの下に隠れて」困難な状況


最初のリクエストには非常に不便な瞬間があります-リンクされたテーブル「ドキュメント拡張」を使用してステータスをチェックします。式の残りの条件が真実であるかどうかに関係なく（たとえば、d。「Deleted」はNOT TRUEです）、この接続は常に実行され、「リソースに見合った」価値があります。それらの多かれ少なかれ費やされます-このテーブルのサイズによって異なります。
ただし、関連するレコードの検索が本当に必要な場合にのみ実行されるようにリクエストを変更できます。
SELECT
  ...
FROM
  "" d
WHERE
  ... /*index cond*/ AND
  CASE
    WHEN "$" IS NULL AND "" IS NOT TRUE THEN (
      SELECT
        ""[1] IS TRUE
      FROM
        ""
      WHERE
        "@" = d."@"
    )
  END

リンクテーブルの結果のフィールドは必要ないため、JOINをサブクエリの条件に変換できます。
CASEの「括弧の外」にインデックス付きフィールドを残し、レコードからWHENブロックに単純な条件を追加します。今度は、「重い」クエリはTHENに切り替えたときにのみ実行されます。

私の姓は「合計」です


上記のすべてのメカニズムを使用して、結果のクエリを収集します。
WITH T AS (
  SELECT
    "@"
  FROM
    ""
  WHERE
    "" = ''
)
  (
    SELECT
      TRUE
    FROM
      "" d
    WHERE
      ("3", "") = (19091, (TABLE T)) AND
      CASE
        WHEN "$" IS NULL AND "" IS NOT TRUE THEN (
          SELECT
            ""[1] IS TRUE
          FROM
            ""
          WHERE
            "@" = d."@"
        )
      END
    LIMIT 1
  )
UNION ALL
  (
    SELECT
      TRUE
    FROM
      "" d
    WHERE
      ("", "") = ((TABLE T), 19091) AND
      CASE
        WHEN "$" IS NULL AND "" IS NOT TRUE THEN (
          SELECT
            ""[1] IS TRUE
          FROM
            ""
          WHERE
            "@" = d."@"
        )
      END
    LIMIT 1
  )
LIMIT 1;
