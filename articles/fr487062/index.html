CBS a permis aux utilisateurs de Linux de regarder la série Star Trek, grâce à elle pour cela

En janvier, les fans de science-fiction se sont réjouis de poursuivre la série culte Star Trek . La nouvelle saison s'appelle Star Trek: Picard (la cote IMDB recule: 8,70 ). Mais la joie a été de courte durée. Tout d'abord, pour afficher, vous avez besoin d'un abonnement au service en ligne CBS All Access (en plus des abonnements à Netflix, Disney, Google Play Movies, Hulu, Apple TV + et à d'autres cinémas en ligne que vous regardez). Mais même si vous vous abonnez, ce n'est pas un fait que les films seront diffusés.

Les utilisateurs de Linux étaient indignés de ne voir que des publicités dans la vidéo diffusée sur le site Web de CBS All Access.mais pas un film. Après la publicité, la diffusion se termine et le film lui-même ne peut pas être visionné dans le navigateur. Le blocage se fait via le module cryptographique CDM Widevine, qui doit être intégré dans le navigateur afin de lire du contenu sous protection cryptographique DRM fournie par les titulaires des droits d'auteur. Tous les navigateurs ne sont pas autorisés à installer un tel module, pour cela, il est nécessaire de conclure un accord de licence avec le titulaire des droits d'auteur, et parfois de payer une somme ronde. Pour plus d'informations, consultez l'article "Les navigateurs indépendants ne sont plus compétitifs" .

Lorsque vous essayez de lire une vidéo à partir du site Web CBS All Access, le navigateur Linux affiche uniquement un écran noir:



contrairement à Chrome, le navigateur Firefox pour Linux est initialement sans fonctionnalité DRM, car le module propriétaire viole la licence Open Source. Les utilisateurs peuvent activer manuellement cette fonctionnalité si nécessaire. Cependant, le contenu de CBS All Access n'était pas lu sous Linux dans aucun navigateur: ni dans Firefox, ni dans Chrome, ni même dans Edgium, le nouveau navigateur Microsoft sur le moteur Chromium.

La console Web montre que lors de l'accès au backend CBS Widevine, une erreur HTTP 400 (Bad Request) est renvoyée.



Historiquement, certains sites et services ont intentionnellement bloqué l'accès des agents utilisateurs pour Linux. Mais voici un autre cas, car changer l'agent utilisateur dans Chrome ou Firefox n'a aucun effet. Lorsque vous essayez de lire le contenu, les mêmes erreurs sont renvoyées.

Un système DRM est installé sur le site Web de CBS appelé Widevine, un protocole DRM entièrement multiplateforme créé par Google. Mais d'une manière ou d'une autre, il s'est cassé.



Une enquête plus approfondie a révélé que CBS All Access ne fonctionne pas non plus dans le navigateur Chrome sur Android et le navigateur Safari sur iOS, bien que les applications CBS natives soient disponibles sur les deux plates-formes et qu'il n'y ait aucun problème.

Dans le même temps, les utilisateurs de Linux ont une solution de contournement - CBS All Access est disponible en tant qu'abonnement facultatif à Amazon Prime. Donc, si vous avez besoin d'accéder à du contenu crypté à partir d'un ordinateur Linux, vous pouvez vous connecter à Prime et vous y abonner. Dans ce cas, le contenu est normalement lu via le système Amazon au lieu de CBS Interactive.



Le 31 janvier 2020, l'un des lecteurs d' Ars Technica a proposé une version selon laquelle le problème de lecture de contenu dans les navigateurs Linux pourrait être dû au fait que le paramètre VMP (Verified Media Path) était activé sur le serveur. Ce paramètre de style de démarrage sécurisé UEFI permet d'envoyer du contenu uniquement aux navigateurs avec une infrastructure "authentique" vérifiée.

Les journalistes de la publication ont contacté CBS et ont demandé de transférer des informations sur VMP au service technique. Deux heures plus tard, le problème a été résolu - et le film Star Trek: Picard a été chargé avec succès dans le navigateur Chrome sous Linux. Le navigateur Firefox se bloque toujours avec une erreur.

Le 30 janvier, CBS a posté le premier épisode sur YouTube , il était disponible gratuitement pendant plusieurs jours, mais maintenant l'accès est fermé.

Sur les trackers de torrent russes, les deux premiers épisodes sont distribués à partir de torrents piratés dans SDI Media.

Pendant ce temps, CBS All Access rapporteque le premier épisode de la série a établi un record absolu pour le nombre de flux sur le service, dépassant de 115% le record précédent, qui appartenait également à la série de la série Star Trek, c'était "Star Trek: Discovery" (Star Trek: Discovery).

Grâce à la diffusion de matchs de football et au Grammy Award, CBS a établi en janvier 2020 un record absolu de nombre de nouveaux abonnés, mais la taille totale de l'audience n'est pas indiquée. Les leaders du marché sont Netflix et Hulu avec 61 millions et 29 millions d'abonnés américains. L'audience de Disney + et Apple TV + est estimée respectivement à 23,2 millions et 25 millions, le dernier d'entre eux étant accessible gratuitement pendant un an à tous les acheteurs de nouveaux appareils Apple.
