JavaScriptで使用するアラート（）、確認（）、プロンプト（）のカスタムビューを作成する方法

JavaScriptでの典型的なユーザー対話機能の外観のカスタマイズについて、長い間考えてきました。アラート（）、確認（）、プロンプト（）（以下、モーダルウィンドウ）です。
実際、それらは使用するのに非常に便利ですが、ブラウザによって異なり、外観が非常に見苦しくなります。
最後に、手が届きました。
何が問題ですか？ダイアログを表示する通常の手段（たとえば、ブートストラップ）は、ブラウザーがJavaScriptコードの実行を停止してユーザーが操作する（閉じるボタンをクリックする）のを待機するように調整するという、単なるアラートとして使用することはできません。ブートストラップのモーダルには、別個のイベント処理が必要になります-ボタンをクリックして、モーダルウィンドウを閉じます...
それにもかかわらず、私はすでにゲームのアラートのカスタマイズを使用して、標準メッセージをゲームデザインのスタイルに対応するメッセージに置き換えています。これは、接続エラーメッセージやその他のシステム状況を含め、うまく機能します。ただし、ユーザーが応答を待つ必要がある場合、これは機能しません。

ECMAScript 6（ES6）のPromiseの登場により、すべてが可能になりました！
モーダルウィンドウとコード（アラート（）、確認（）、プロンプト（））のデザインを分離するアプローチを適用しました。ただし、コード内のすべてを非表示にすることができます。そのようなアプローチを引き付けるもの-デザインは異なるプロジェクトで変更できますが、単に異なるページで、または状況に応じて変更できます。
このアプローチの悪い点は、モーダルウィンドウのコードで、さらにはグローバルスコープでも、マークアップ名（ID）を使用する必要があることです。ただし、これは原則のほんの一例であるため、ここでは重点を置きません。

アラートのコードを取得する


それでは、マークアップ（フォントアイコンのブートストラップとFont Awesome）とアラートコード（私はjQueryを使用しています）を解析してみましょう。
    <div id="PromiseAlert" class="modal">
        <div class="modal-dialog" role="document">
            <div class="modal-content">
                <div class="modal-header">
                    <h5 class="modal-title"><i class="fas fa-exclamation-triangle text-warning"></i> <span>The app reports</span></h5>
                    <button type="button" class="close" data-dismiss="modal" aria-label="Close">
                        <span aria-hidden="true">×</span>
                    </button>
                </div>
                <div class="modal-body">
                    <p></p>
                </div>
                <div class="modal-footer">
                    <button type="button" class="btn btn-secondary" data-dismiss="modal">OK</button>
                </div>
            </div>
        </div>
    </div>


    window.alert = (message) => {
        $('#PromiseAlert .modal-body p').html(message);
        var PromiseAlert = $('#PromiseAlert').modal({
            keyboard: false,
            backdrop: 'static'
        }).modal('show');
        return new Promise(function (resolve, reject) {
            PromiseAlert.on('hidden.bs.modal', resolve);
        });
    };

上で述べたように、コードにはグローバル名PromiseAlertとhtmlマークアップクラスが使用されます。コードの最初の行では、アラート関数のパラメーターがメッセージ本文に渡されます。その後、ブートストラップメソッドは、特定のオプションを備えたモーダルウィンドウを表示します（ネイティブアラートに近づけます）。重要！モーダルウィンドウはローカル変数に記憶されます。これは、以下のクロージャーで使用されます。
最後に、アラートPromiseの結果として作成されて返されます。モーダルウィンドウを閉じた結果、このPromiseが実行されたことを解決します。
次に、このアラートの使用方法を見てみましょう。
    $('p a[href="#"]').on('click', async (e) => {
        e.preventDefault();
        await alert('Promise based alert sample');
    });

この例では、段落内の空のリンクをクリックするとメッセージが表示されます。注意を払う！仕様に準拠するには、アラート関数の前にawaitキーワードを付ける必要があります。これは、関数内でasyncキーワードを使用した場合にのみ使用できます。これにより、この場所（ネイティブアラートの場合のようにスクリプトが停止します）でモーダルウィンドウが閉じることを期待できます。
これを行わないとどうなりますか？アプリケーションのロジックに依存します（上図のこのようなアプローチの例）。これがコードの終わりであるか、コードのそれ以上のアクションでページが過負荷にならない場合は、おそらくすべてがうまくいきます！モーダルウィンドウは、ユーザーが閉じるまで垂れ下がります。ただし、モーダルウィンドウがまだある場合、またはページがリロードされる場合、別のページへの移行が発生すると、ユーザーにはモーダルウィンドウが表示されず、ロジックが破壊されます。経験上、さまざまなサーバーエラー（状態）やコードライブラリからのメッセージは、awaitを使用していませんが、新しいアラートで非常にうまく機能していると言えます。

確認のためのアプローチを開発します


さらに進みましょう。間違いなく、confirmはasync / awaitバインディングでのみ使用できます。ユーザーが選択した結果をコードに伝える必要があります。これはプロンプトにも当てはまります。確認してください：
    <div id="PromiseConfirm" class="modal">
        <div class="modal-dialog" role="document">
            <div class="modal-content">
                <div class="modal-header">
                    <h5 class="modal-title"><i class="fas fa-check-circle text-success"></i> <span>Confirm app request</span></h5>
                    <button type="button" class="close" data-dismiss="modal" aria-label="Close">
                        <span aria-hidden="true">×</span>
                    </button>
                </div>
                <div class="modal-body">
                    <p></p>
                </div>
                <div class="modal-footer">
                    <button type="button" class="btn btn-success" data-dismiss="modal">OK</button>
                    <button type="button" class="btn btn-danger" data-dismiss="modal">Cancel</button>
                </div>
            </div>
        </div>
    </div>


    window.confirm = (message) => {
        $('#PromiseConfirm .modal-body p').html(message);
        var PromiseConfirm = $('#PromiseConfirm').modal({
            keyboard: false,
            backdrop: 'static'
        }).modal('show');
        let confirm = false;
        $('#PromiseConfirm .btn-success').on('click', e => {
            confirm = true;
        });
        return new Promise(function (resolve, reject) {
            PromiseConfirm.on('hidden.bs.modal', (e) => {
                resolve(confirm);
            });
        });
    };

違いは1つだけです。ユーザーの選択について通知する必要があります。これは、クロージャー内の別のローカル変数を使用して行われます-確認してください。確認ボタンが押された場合、変数はtrueに設定され、デフォルトではその値はfalseです。モーダルウィンドウのクローズを処理するときに、resolveはこの変数を返します。
使用方法は次のとおりです（async / awaitで必要）。
    $('p a[href="#"]').on('click', async (e) => {
        e.preventDefault();
        if (await confirm('Want to test the Prompt?')) {
            let prmpt = await prompt('Entered value:');
            if (prmpt) await alert(`entered: «${prmpt}»`);
            else await alert('Do not enter a value');
        }
        else await alert('Promise based alert sample');
    });

先へ進む-プロンプトへのアプローチ


上記のロジックは、テストプロンプトでも実装されています。そして、そのマークアップとロジックは次のとおりです。
    <div id="PromisePrompt" class="modal">
        <div class="modal-dialog" role="document">
            <div class="modal-content">
                <div class="modal-header">
                    <h5 class="modal-title"><i class="fas fa-question-circle text-primary"></i> <span>Prompt request</span></h5>
                    <button type="button" class="close" data-dismiss="modal" aria-label="Close">
                        <span aria-hidden="true">×</span>
                    </button>
                </div>
                <div class="modal-body">
                    <div class="form-group">
                        <label for="PromisePromptInput"></label>
                        <input type="text" class="form-control" id="PromisePromptInput">
                    </div>
                </div>
                <div class="modal-footer">
                    <button type="button" class="btn btn-success" data-dismiss="modal">OK</button>
                    <button type="button" class="btn btn-danger" data-dismiss="modal">Cancel</button>
                </div>
            </div>
        </div>
    </div>


    window.prompt = (message) => {
        $('#PromisePrompt .modal-body label').html(message);
        var PromisePrompt = $('#PromisePrompt').modal({
            keyboard: false,
            backdrop: 'static'
        }).modal('show');
        $('#PromisePromptInput').focus();
        let prmpt = null;
        $('#PromisePrompt .btn-success').on('click', e => {
            prmpt = $('#PromisePrompt .modal-body input').val();
        });
        return new Promise(function (resolve, reject) {
            PromisePrompt.on('hidden.bs.modal', (e) => {
                resolve(prmpt);
            });
        });
    };

ロジックと確認の違いはごくわずかです。クロージャーの追加のローカル変数はprmptです。また、論理値ではなく、ユーザーが入力する文字列があります。閉鎖により、その価値は解決します。また、（入力フィールドから）確認ボタンが押されたときにのみ、値が割り当てられます。ところで、ここでは、もう1つのグローバル変数PromisePromptInputを浪費しました。それを使用して、入力フォーカスを設定します（ただし、値を取得する場合と同じ方法または値を取得する方法のいずれかで、単一のアプローチで実行できます）。
