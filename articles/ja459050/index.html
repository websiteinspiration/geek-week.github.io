強力なVuexタイピングモジュール

この記事を書いた動機は、Vue、つまりVuexのタイピングに関する別の記事でした。驚いたことに、私はそこにモジュールの言及を見つけられませんでした。私の意見では、それはその種のVuexの典型的なものの中で最高です。悲しいことに、Habrで、そして実際にRunetで検索しました（実際、英語のソースでは、参照をすぐに見つけることは容易ではありません）。結果は見つかりませんでした。この記事は、使用方法と構成に関する詳細な分析と複数ページのマニュアルではなく、Vue-ninjaという、その仕事を完璧に行うためのツールであるあなたと共有する方法です。

vuex-smart-module


誰がすべての時間がない：Githubに。

ご想像のとおり、このモジュールの主な目的は、タイプを備えたVuexストレージのフルフォーマットカバレッジです。内部およびコンポーネント自体の両方で直接。このモジュールは、Vuexおよびvue-class-componentライブラリの主な貢献者（@ktsn）によって記述されています。

水


率直に言って、私の中にパスを活字体には含めて、ごく最近始まりました そして、デコレータのようなもので、私はこのライブラリを他のアナログと比較することはできません。他のツール（たとえば、vuex-module-decorators）を構成して使用しようとすると、さまざまな問題が発生し、何らかの形で必要なものを理解できませんでした（または、彼らが言うように、それらを調理する方法がわかりませんでした）。私はvuex-smart-moduleで非常に幸運でした。ライブラリは、プロジェクト（およびリポジトリ）をTypescriptに変換しているその瞬間に表示されました。これですべてが正常に機能し、コードは見栄えが良くなりました。

例


私の意見では、ライブラリーには、発生する可能性のあるすべてのケースをカバーする優れたドキュメントがあります（そうでない場合は、テストで他のすべてを見つけることができます）。ただし、記事を少なくとも何らかの方法でコードで薄めるために、接続と使用の基本的な例、いくつかの「ライフ」の例、およびデコレーターとの連携方法（ニュアンスがあります）を示します。

モジュールの作成


モジュールはクラスを使用して作成されます。これは異常に恐ろしいように思えるかもしれませんが、信じてください、あなたはすぐにそれに慣れます。

store / root.ts

//   
import { Getters, Mutations, Actions, Module } from 'vuex-smart-module'

// 
class RootState {
  count = 1
}

// 
//      RootState
class RootGetters extends Getters<RootState> {
  get double() {
    //      `state`
    return this.state.count * 2
  }

  get triple() {
    //       `getters`
    return this.getters.double + this.state.count
  }
}

// 
//     ,     RootState
class RootMutations extends Mutations<RootState> {
  increment(payload: number) {
    //       `state`
    this.state.count += payload
  }
}

// 
//    
//    ,       ,     
class RootActions extends Actions<
  RootState,
  RootGetters,
  RootMutations,
  RootActions
> {
  incrementAsync(payload: { amount: number; interval: number }) {
    //      `state`, `getters`, `commit`  `dispatch` 
    return new Promise(resolve => {
      setTimeout(() => {
        this.commit('increment', payload.amount)
      }, payload.interval)
    })
  }
}

//  
export default new Module({
  state: RootState,
  getters: RootGetters,
  mutations: RootMutations,
  actions: RootActions
})

モジュール


モジュールの接続は、通常のVuexでの接続方法と似ています。これらは、RootStoreのmodulesプロパティで指定する必要があります。

import FooStore from './modules/foo'

/* … */

export default new Module({
  state: RootState,
  getters: RootGetters,
  mutations: RootMutations,
  actions: RootActions,
  modules: {
    FooStore
  }
})

コンポーネント内で使用


storは、グローバルプロパティthis。$ Storeと、Vuexのそれに非常によく似たマッピングの両方で使用できます。

import Vue from 'vue'

//    (      )
// import FooStore from '@/store/modules/foo'
import RootStore from '@/store/root'

export default Vue.extend({
  computed: RootStore.mapGetters(['double']),

  methods: RootStore.mapActions({
    incAsync: 'incrementAsync'
  }),

  created() {
    console.log(this.double)
    this.incAsync(undefined)
  }
})


タイピング


コミットとディスパッチの入力例：
import { categories } from '@/api'

export type Category {
  attributes: {
    hasPrice: boolean;
    icon: string
    lvl: number
    name: string
    slug: string
  };
  id: number
}

export interface IParams {
    city_id: number
}

class AppState {
    categories: Category[] = []
}

/* ... */

class AppMutations extends Mutations<AppState> {
  setCategories(categories: Category[]) {
    this.state.categories = categories
  }
}

class AppActions extends Actions<
  AppState,
  AppGetters,
  AppMutations,
  AppActions
> {
  async getCategories({params}: {params: IParams}): Promise<Category[]> {
    return categories.get({params}).then(
      ({ data }: { data: Category[] }) => {
        this.commit("setCategories", data)
        return data
      }
    )
  }
}


レセプション


デコレーター（vue-property-decorator）を使用して接続する


import { Vue, Component } from "vue-property-decorator"

//    (      )
// import FooStore from '@/store/modules/foo'
import RootStore from "@/store/root"

//  ,   ,      Typescript,     :
const Mappers = Vue.extend({
  computed: {
    ...RootStore.mapGetters(["double"])
  },
  methods: {
    ...RootStore.mapActions({
      incAsync: 'incrementAsync'
    })
  }
});

@Component
export default class MyApp extends Mappers {
  created() {
    console.log(this.double)
    this.incAsync(undefined)
  }
}

ボールトのリセット


ストレージをデフォルト値にリセットする必要がある場合がありますが、これは非常に簡単です。

class FooState {
  /* ... */
}

class FooMutations extends Mutations<FooState> {
  reset () {
      const s = new FooState()
      Object.keys(s).forEach(key => {
        this.state[key] = s[key]
      })
  }
}
