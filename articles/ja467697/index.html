GOPATHとGOROOTが不要になったのは本当ですか？

たまたまGoに慣れ始めたばかりの開発者は、Goプロジェクトの作業ディレクトリを選択するという問題に直面することがよくありました。だから、GolangConf会議のチャットでこの質問も尋ねました。新しいgopherはしばしばGOPATHとGOROOTという言葉でお互いを怖がらせます。ただし、現在のバージョンのGo（1.13）のクイックスタートガイドでは、これら2つの「怖い」単語についてはまったく触れられていません。


その理由を見てみましょう。実験の純粋さのために、新鮮なUbuntuを仮想マシンにデプロイし、Wikiの指示に従って Goをインストールしました。


sudo add-apt-repository ppa:longsleep/golang-backports
sudo apt-get update
sudo apt-get install golang-go

Go 1.13がインストールされ、使用できるようになりました。


$ go version
go version go1.13 linux/amd64
$ which go
/usr/bin/go
$ whereis go
go: /usr/bin/go /usr/lib/go /usr/share/go /usr/share/man/man1/go.1.gz

GOROOT


については2015年GOROOTの記事ですでに完全に記述されており、この情報は依然として関連しています。


奇妙なことに、最後のコマンド（whereis go）によって発行されたディレクトリのリストGOROOTには、実際にはありません。


$ go env GOROOT
/usr/lib/go-1.13

したがって、たとえば、IDEで標準のGoライブラリのファイルへのパスを指定する必要がある場合は、と指定します/usr/lib/go-1.13。おそらく、GOROOT日常生活のユースケースはここで終わります。
