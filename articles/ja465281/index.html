Medtronic 640gポンプによる連続グルコースモニタリング（NMH）


たまたま1年ほど前に、私の娘は1型糖尿病（インスリン依存型）で病気になりました。「ペン」を1か月使用した後、NMGをサポートするMedtronic 640gポンプに切り替えました。猫の下で、ポンプからのデータを使用してどのようにグルコースをモニタリングしたかをお話します。注意、交通。

前書き


このポンプのすべての機能と「長所」と「短所」については説明しませんが、これは記事の範囲を超えています。この情報はインターネットで簡単に見つけることができます。理解に必要な点のみを説明します。

1型糖尿病の主な目標の1つは、最適な血糖値を維持することです。事実は、ブドウ糖レベルがさまざまな要因（食品、身体活動、ホルモン、天候の変化、睡眠、さらには公園内の単純な散歩でさえも、ブドウ糖レベルに大きな影響を与える可能性がある）に対して十分速く変化することです。したがって、グルコースレベルをできるだけ頻繁に、理想的には24時間年中無休で5分ごとに測定することが重要です。従来のグルコメーターでこれを達成することは非常に難しいため、NMGが使用されます。彼らは違う。あなたはここでより詳細にビデオを読んで見ることができます。監視ポンプがあると非常に優れており、多くの問題の解決に役立ちます。ポンプでは、現在のグルコース値を確認できるだけでなく、最近の変化のグラフも確認できます。

すべての測定値が表示されるポンプには、常に娘がいます。つまり、測定値をリモートで確認することはできません。娘はいつも近くにいるわけではなく、学校に通い、内線に通っています。クラス、散歩、一般的に、完全な人生を送ります。妻と私は働いています。常に携帯電話に電話をかけ、1晩に数回（時には数十回）ベッドから出ても、睡眠と神経が強くなることはありません。

入力


ここで は、NMGの実装と動作原理について説明します。私が注目したい唯一のことは、このポンプがZigBeeプロトコルを介して送信機に接続されていることです。合計すると、このポンプで動作する3つのデバイスが見つかりました。

  • Guardian 2 Link トランスミッター自体
  • CareLink USB MMT-7306、いわゆる 「ブラックフラッシュドライブ」
  • Contour NEXT LINK 2.4、グルコメーター

現在、ロシアのこれら3つのデバイスのうち、送信機のみが販売されています。実際、ブドウ糖レベルに関するデータをポンプに転送するのは彼です。 「フラッシュドライブ」は主治医にしか見られませんでした。それを使用して、ポンプからデータを数回彼女のプログラムにダウンロードし、チャートを印刷しました。その後、この「フラッシュドライブ」は私たちにアクセスできなくなりました。彼女は別の都市に与えられ、彼女が戻ったとき誰も知らない。彼らは、この「フラッシュドライブ」はロシア全体で唯一のものであると言っています。この情報がどれだけ本当かはわかりません。ロシアでグルコメーターとフラッシュドライブを購入することは現実的ではありません。私の質問に対して、公式の代表は「彼らはまだロシアで認証を通過していない、それがいつになるかは不明である、我々は待っている」と言っている。長い間調査した結果、イギリスでグルコメーターを購入することができました。そのため、これで私を助けてくれたすべての人に感謝します。

したがって、メーターはポンプに接続でき、USBを備えているため、コンピューターに接続できます。長い検索の末、GitHubで、このメーターを介してポンプからデータを読み取ることができるプロジェクトを見つけました。この機会に、これらのプロジェクトの作者であるLennart Goedhartに感謝の意を表します。彼はプロトコルを研究するという本当に素晴らしい仕事をして、結果を無料で投稿しました。計画を実施するだけです。

問題の定式化


コンピュータでポンプの読み取り値を監視できるようにするには、携帯電話をオンラインで「登録とSMSなしで」実行します。

決定


鉄


最初のバージョンでは、導入する必要のないRaspberry Pi Zero Wが選択されました。彼は小さく、大食いではなく、あなたが必要とするすべてのものを持っています。それに追加購入しました。機器：Waveshare SIM7000C -2.5Gモデム+ GPS、Waveshare 4ポートUSBハブ、Raspberry Pi Zero UPS。これはすべてAliexpressで簡単に購入できます。これらすべてのコンポーネントの選択は、これらすべての寸法が小さく、「パイ」として組み立てられるという事実に基づいていました。食料はパワーバンクを通じて提供されます。テストでは、6000 mAの容量は「軽い」日には十分です。午前8時から午後8時までです。パワーバンク約130グラム。彼はポケットをあまり引っ張りません。

電力、より正確にはRaspberry Pi Zero UPSについて。計画する時、彼が主な栄養源であると思いましたが、私は間違っていました。実際には、1時間続きます。したがって、私にはそれを使用する理由はありません。


最初のバージョン。

運用中、Pi ZeroでUSBを操作することは、まだ多くのことが望まれていません。 Wi-Fiが定期的に落ちた-GPRSはそれほど重要ではありませんが、メーターの凍結はすでに重要です。さらに、USBをリモートで「変形」させることはできませんでした。USBポートには直接電力が供給されます。

結局、私はRaspberry Pi 3を購入しました。選択は、ボード上のWi-Fiの存在と完全なUSBポートの存在によるものでした。


グルコメーターとWaveshare SIM7000Cなしの2番目のバージョン。

サイズを小さくするために、イーサネットコネクタを犠牲にし、メーターを接続するUSB​​ケーブルをはんだ付けする必要がありました。将来的には、メーターの本体を取り除き、ラズベリーに直接はんだ付けする予定です。
ボディの開発と印刷は計画中です。

柔らかい


主な要件は、スマートフォンを含むあらゆるデバイスからのデータへのアクセスです。ラズベリーでサーバーを上げることは意味がありませんでした-モデムを介して通信する場合、すべての計画を台無しにする「灰色」のIPがあります。さらに、エネルギー節約のため、GPRS接続を一定に保つことは私の計画の一部ではありませんでした。幸いにも、私はすでにZabbixを構成していて、Zabbixにデータを転送することにしました。 「美しい」グラフィックとして、グラファナはねじ込まれました。

このプロジェクトは、ラズベリーのソフトウェアの基礎として採用されました。後で判明したように、それは単なる概念でした。一般に、それは非常に効率的であり、現在、私の仕事を実行します。

作業のロジックは次のとおりです。5分ごとにポンプとラズベリーからデータを収集し、zabbix_senderを介してZabbixに転送します。さらに、「SMSによるリモートコントロール」があります。
