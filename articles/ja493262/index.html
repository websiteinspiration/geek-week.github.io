NSManagedObjectContextおよびPersistentStoreCoordinatorを操作する


コアデータスタック階層：



簡単に言うと、エンティティはすべて、上の図では管理対象オブジェクトとして表示されています。管理対象オブジェクトはすべてNSManagedObjectContextに保存されます。コンテキストこれは、SQLにアクセスした後に保存したデータを行キャッシュとして保存するNSPersistentStoreCoordinatorを参照しますが、現在はそうではありません。

ManagedObject（エンティティ）のインスタンスを作成するには、NSPersistentStoreCoordinatorのラッパーであるNSPersistentContainerを参照する必要があります。

var persistentContainer: NSPersistentContainer = {
      let container = NSPersistentContainer(name: "SavingLearn")
      container.loadPersistentStores(completionHandler: { (storeDescription, error) in
          if let error = error as NSError? {
              fatalError("Unresolved error \(error), \(error.userInfo)")
          }
      })
      return container
  }()

次に、PersistentContainerを通じてNSManagedObjectContextにアクセスする必要があります。 

var context: NSManagedObjectContext = {
    return persistentContainer.viewContext
  }()

これで、コンテキストからManageObjectのインスタンスを作成し、それに値を割り当てることができます。

var person = Person(context: context)
person.age = 32

オブジェクトを保存しています


保存するには、単純な関数を使用します。 

func saveContext () {
      if context.hasChanges {
          do {
              try context.save()
          } catch {
            context.rollback()
              let nserror = error as NSError
              fatalError("Unresolved error \(nserror), \(nserror.userInfo)")
          }
      }
  }

上記の方法では、コンテキストの変更をチェックし、変更が存在する場合は保存します。エラーが発生した場合、メソッドが実行されます。

context.rollback()

rollback()
このメソッドは、コンテキストを元の状態に戻し、それに加えられたすべての変更をキャンセルします。
