Membuat toko online prototipe SPA di htmlix.js

Artikel ini akan menjelaskan tentang pembuatan frontend selangkah demi selangkah untuk toko online prototipe SPA. Bagian server ditulis dalam node.js menggunakan express, mesin template adalah ranting, database adalah NeDB, modul tangguh digunakan untuk memproses data formulir, bagian klien ditulis dalam htmlix.

Bagi mereka yang tidak terbiasa dengan Htmlix, baca dasar-dasar membangun aplikasi di sini .

Sumber data untuk pelajaran ini dapat diunduh di sini .

Ini adalah contoh yang sudah jadi, kami hanya menghapus semua yang ada dari file /static/js/front.js dan masukkan :

var State = { };

	window.onload = function(){
			///   HTMLix
			
		var HM = new HTMLixState(State);
	        console.log(HM);
	}

Jadi, kami benar-benar menghapus frontend dari contoh ini, dan kami masih memiliki server aplikasi, mari kita lihat cara kerjanya:

  • Pertama, Anda perlu menginstal semua modul yang tidak tersedia saat ini. Untuk melakukan ini, masukkan di konsol: npm install
  • kemudian masukkan aplikasi simpul di baris perintah
  • lalu pergi ke localhost: 3000

Jadi, dengan mengklik berbagai item menu, kita melihat server yang berfungsi, tanpa bagian frontend, karena kami baru saja menghapusnya.

Sekarang kita akan membuatnya kembali secara bertahap, tetapi pertama-tama kita akan mencari tahu bagaimana server bekerja.

Dengan masuk ke alamat "localhost: 3000 /" kami diberi menu teratas dengan tiga item, halaman utama dengan enam kartu produk, dan daftar kategori di sebelah kiri.

Selanjutnya, buat komponen kartu produk cart_single , tetapi sebelum Anda membuatnya, mari kita lihat cara kerja htmlix:

Jadi, ketika memuat halaman dari alamat "/" atau "/ kategori /: idCategory", server mengirimi kami kode html dari dua komponen, lebih tepatnya tiga, "menu" lain, tetapi kami belum menggunakannya, kami menggunakan kode html untuk komponen gerobak dan kategori, berdasarkan kode ini htmlix membuat templat untuk komponen, tetapi templat untuk komponen cart_single tidak dikirimkan kepada kami di alamat ini, karena dia tidak ada di sana. Bagaimana kita "memuatnya"? Untuk melakukan ini, kita membuat komponen cart_single dalam deskripsi aplikasi dan menempatkannya di objek "fetchComponents" khusus, dan dalam pengaturan `stateSettings.templatePath`, tentukan alamat tempat memuat template untuk komponen ini, dan semua yang lain yang akan muncul. Dengan demikian, aplikasi pertama-tama akan membuat komponen yang dikirim server bersama dengan html,dan kemudian setelah mengirim permintaan kedua ke stateSettings.templatePath, itu akan membuat komponen yang tersisa terletak di objek fetchComponents.

Jadi kita akan melakukannya jika kita selalu memiliki akses ke aplikasi dari alamat "/" atau "/ kategori /: idCategory", tetapi bagaimana jika kita kemudian memuat aplikasi dari alamat "/ cart /: idCart"? Aplikasi akan memberikan kesalahan bahwa ia tidak dapat menemukan templat untuk larik gerobak, karena sekarang tidak akan ada di alamat ini dan kami sekarang perlu meletakkan gerobak di objek "fetchComponents", dan sebaliknya, dapatkan single_cart dari sana, karena harus diinisialisasi dulu bersama-sama dengan komponen kategori, apa yang harus saya lakukan?

Untuk mengatasi masalah ini, kami akan menggunakan HTMLixRouter (), sebelum membuat instance aplikasi, setelah memuat halaman, ia memeriksa alamat url sekarang dan membandingkannya dengan yang akan kami tunjukkan, dan kemudian menempatkannya yang tidak ada template dalam pemuatan pertama, ke dalam objek "fetchComponents", setelah itu menginisialisasi aplikasi, maka kita akan menghindari membuat beberapa opsi untuk menggambarkan aplikasi untuk url yang berbeda dan, karenanya, duplikasi kode.

Jadi sekarang kita memiliki tiga alamat "/", "/ category /: idCategory" dan "/ cart /: idCart", tiga komponen utama kereta, kategori dan cart_single.
