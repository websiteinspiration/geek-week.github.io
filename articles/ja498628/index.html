PowerCLIスクリプト用のロケットアクセラレータを構築する方法 

遅かれ早かれ、VMwareシステム管理者は日常業務を自動化します。コマンドラインから始まり、PowerShellまたはVMware PowerCLIが登場します。

ISEを起動して、なんらかの魔法で動作するモジュールから標準のコマンドレットを使用するよりも、PowerShellを少し習得したとします。何百もの仮想マシンを数え始めると、小さなスケールで役立ったスクリプトが、大きなスケールでは著しく遅くなることがわかります。 

この状況では、2つのツールが役立ちます。

  • PowerShell Runspaces-別のスレッドでプロセスの実行を並列化できるようにするアプローチ。 
  • Get-Viewは、WindowsのGet-WMIObjectに類似した基本的なPowerCLI関数です。このコマンドレットは、それに関連するオブジェクトをドラッグしませんが、単純なデータ型を持つ単純なオブジェクトの形式で情報を受け取ります。多くの場合、それはより速く出てきます。

次に、各ツールについて簡単に説明し、使用例を示します。特定のスクリプトを分析し、1つ目が最もよく機能するとき、2つ目がいつ機能するかを確認します。行こう！

第2段階：Get-View


Get-Viewの有用性を理解するには、コマンドレットの一般的な動作を覚えておいてください。 

コマンドレットは、APIリファレンスブックを調べたりホイールを作り直したりすることなく情報を簡単に取得するために必要です。昔は100行または2行のコードで記述されていましたが、PowerShellでは1つのコマンドを実行できます。この便宜のために私たちはスピードを支払います。コマンドレット自体の内部には魔法はありません。同じスクリプトですが、日当たりの良いインドの熟練した熟練した手によって書かれた、より低いレベルのものです。

次に、Get-Viewと比較するために、Get-VMコマンドレットを使用します。仮想マシンにアクセスして複合オブジェクトを返します。つまり、VMHost、Datastoreなどの他の関連オブジェクトをそれに接続します。  

代わりにGet-Viewを使用しても、返されたオブジェクトに余分なものが含まれることはありません。さらに、必要な情報を厳密に示すことができるため、オブジェクトの出力が容易になります。Windows Server全般、特にHyper-Vでは、Get-WMIObjectコマンドレットは直接の類似物です。考え方はまったく同じです。

Get-Viewは、ポイントフィーチャの日常的な操作には不便です。しかし、それが数千、数万のオブジェクトになると、彼には価格がありません。
詳細については、VMwareブログをご覧ください：Get-Viewの概要

次に、実際のケースですべてを示します。 

VMをアンロードするためのスクリプトを作成します


ある同僚から、スクリプトを最適化するように依頼されました。タスクは通常のルーチンです。cloud.uuidパラメータが重複しているすべてのVMを見つけます（これは、vCloud DirectorでVMを複製するときに可能です）。 

頭に浮かぶ明らかな解決策：

  1. すべてのVMのリストを取得します。
  2. どういうわけかリストを解析します。

元のバージョンはとても単純なスクリプトでした：

function Get-CloudUUID1 {
   #    
   $vms = Get-VM
   $report = @()

   #   ,     2 :    Cloud UUID.
   #     PS-   VM  UUID
   foreach ($vm in $vms)
   {
       $table = "" | select VM,UUID

       $table.VM = $vm.name
       $table.UUID = ($vm | Get-AdvancedSetting -Name cloud.uuid).Value
          
       $report += $table
   }
#   
   $report
}

最初の速度をオンにします：PowerShell Runspaces

このスクリプトで最初に頭に浮かぶのは、ループを直列ではなく並列スレッドで実行し、すべてのデータを1つのオブジェクトに収集してフィルター処理することです。 

しかし、問題があります。PowerCLIでは、vCenterへの多くの独立したセッションを開くことができず、面白いエラーがスローされます。

You have modified the global:DefaultVIServer and global:DefaultVIServers system variables. This is not allowed. Please reset them to $null and reconnect to the vSphere server.

これを解決するには、最初にセッション情報をストリームに渡す必要があります。PowerShellは、少なくとも関数、少なくともScriptBlockにパラメーターとして渡すことができるオブジェクトで機能することを思い出します。$ globalをバイパスするようなオブジェクトとしてセッションを渡しましょう：DefaultVIServers（-NotDefaultキーを使用したConnect-VIServer）：

次に、Runspace Poolsを使用してマルチスレッドを実装します。  

アルゴリズムは次のとおりです。

  1. すべてのVMのリストを取得します。
  2. 並列スレッドでは、cloud.uuidを取得します。
  3. ストリームから1つのオブジェクトにデータを収集します。
  4. CloudUUIDフィールドの値によるグループ化を通じてオブジェクトをフィルター処理します：一意の値の数が1を超えるもので、目的のVMがあります。

結果として、スクリプトを取得します。


function Get-VMCloudUUID {
   param (
       [string[]]
       [ValidateNotNullOrEmpty()]
       $vCenters = @(),
       [int]$MaxThreads,
       [System.Management.Automation.PSCredential]
       [System.Management.Automation.Credential()]
       $Credential
   )

   $ConnectionString = @()

   #     
   foreach ($vCenter in $vCenters)
   {
       try {
           $ConnectionString += Connect-VIServer -Server $vCenter -Credential $Credential -NotDefault -AllLinked -Force -ErrorAction stop -WarningAction SilentlyContinue -ErrorVariable er
       }
       catch {
           if ($er.Message -like "*not part of a linked mode*")
           {
               try {
                   $ConnectionString += Connect-VIServer -Server $vCenter -Credential $Credential -NotDefault -Force -ErrorAction stop -WarningAction SilentlyContinue -ErrorVariable er
               }
               catch {
                   throw $_
               }
              
           }
           else {
               throw $_
           }
       }
   }

   #    
   $Global:AllVMs = Get-VM -Server $ConnectionString

   # !
   $ISS = [system.management.automation.runspaces.initialsessionstate]::CreateDefault()
   $RunspacePool = [runspacefactory]::CreateRunspacePool(1, $MaxThreads, $ISS, $Host)
   $RunspacePool.ApartmentState = "MTA"
   $RunspacePool.Open()
   $Jobs = @()

# ScriptBlock  !)))
#      
   $scriptblock = {
       Param (
       $ConnectionString,
       $VM
       )

       $Data = $VM | Get-AdvancedSetting -Name Cloud.uuid -Server $ConnectionString | Select-Object @{N="VMName";E={$_.Entity.Name}},@{N="CloudUUID";E={$_.Value}},@{N="PowerState";E={$_.Entity.PowerState}}

       return $Data
   }
#  

   foreach($VM in $AllVMs)
   {
       $PowershellThread = [PowerShell]::Create()
#  
       $null = $PowershellThread.AddScript($scriptblock)
#  ,      
       $null = $PowershellThread.AddArgument($ConnectionString)
       $null = $PowershellThread.AddArgument($VM)
       $PowershellThread.RunspacePool = $RunspacePool
       $Handle = $PowershellThread.BeginInvoke()
       $Job = "" | Select-Object Handle, Thread, object
       $Job.Handle = $Handle
       $Job.Thread = $PowershellThread
       $Job.Object = $VM.ToString()
       $Jobs += $Job
   }

#  ,     
#      
   While (@($Jobs | Where-Object {$_.Handle -ne $Null}).count -gt 0)
   {
       $Remaining = "$($($Jobs | Where-Object {$_.Handle.IsCompleted -eq $False}).object)"

       If ($Remaining.Length -gt 60) {
           $Remaining = $Remaining.Substring(0,60) + "..."
       }

       Write-Progress -Activity "Waiting for Jobs - $($MaxThreads - $($RunspacePool.GetAvailableRunspaces())) of $MaxThreads threads running" -PercentComplete (($Jobs.count - $($($Jobs | Where-Object {$_.Handle.IsCompleted -eq $False}).count)) / $Jobs.Count * 100) -Status "$(@($($Jobs | Where-Object {$_.Handle.IsCompleted -eq $False})).count) remaining - $remaining"

       ForEach ($Job in $($Jobs | Where-Object {$_.Handle.IsCompleted -eq $True})){
           $Job.Thread.EndInvoke($Job.Handle)     
           $Job.Thread.Dispose()
           $Job.Thread = $Null
           $Job.Handle = $Null
       }
   }

   $RunspacePool.Close() | Out-Null
   $RunspacePool.Dispose() | Out-Null
}


function Get-CloudUUID2
{
   [CmdletBinding()]
   param(
   [string[]]
   [ValidateNotNullOrEmpty()]
   $vCenters = @(),
   [int]$MaxThreads = 50,
   [System.Management.Automation.PSCredential]
   [System.Management.Automation.Credential()]
   $Credential)

   if(!$Credential)
   {
       $Credential = Get-Credential -Message "Please enter vCenter credentials."
   }

   #   Get-VMCloudUUID,    
   $AllCloudVMs = Get-VMCloudUUID -vCenters $vCenters -MaxThreads $MaxThreads -Credential $Credential
   $Result = $AllCloudVMs | Sort-Object Value | Group-Object -Property CloudUUID | Where-Object -FilterScript {$_.Count -gt 1} | Select-Object -ExpandProperty Group
   $Result
}

ます：GetView何が問題かを見つけます。
最初に明らかなのは、Get-VMコマンドレットの完了に長い時間がかかることです。
2番目：Get-AdvancedOptionsコマンドレットはさらに長く実行されます。
まず、2番目の問題について説明しましょう。 

Get-AdvancedOptionsは個々のVMオブジェクトで便利ですが、多くのオブジェクトを操作する場合は非常に遅くなります。仮想マシンオブジェクト自体から同じ情報を取得できます（Get-VM）。ExtensionDataオブジェクトに埋め込まれているだけです。フィルタリングにより、必要なデータを取得するプロセスを加速します。

手首を振ると、次のようになります。


VM | Get-AdvancedSetting -Name Cloud.uuid -Server $ConnectionString | Select-Object @{N="VMName";E={$_.Entity.Name}},@{N="CloudUUID";E={$_.Value}},@{N="PowerState";E={$_.Entity.PowerState}}

これに変わります：


$VM | Where-Object {($_.ExtensionData.Config.ExtraConfig | Where-Object {$_.key -eq "cloud.uuid"}).Value -ne $null} | Select-Object @{N="VMName";E={$_.Name}},@{N="CloudUUID";E={($_.ExtensionData.Config.ExtraConfig | Where-Object {$_.key -eq "cloud.uuid"}).Value}},@{N="PowerState";E={$_.summary.runtime.powerstate}}

結論はGet-AdvancedOptionsと同じですが、何倍も速く動作します。 

次に、VMを取得します。複雑なオブジェクトを処理するため、すぐには実行されません。論理的な質問が発生します。この場合、VMの名前、その状態、およびトリッキーな属性の値が必要なだけなのに、なぜ追加の情報と巨大なPSObjectが必要なのでしょうか。  

さらに、Get-AdvancedOptionsに直面したブレーキがスクリプトを離れました。セッションの転送時にスクワットを伴うストリームで低速のタスクを並列化する必要がなくなったため、Runspace Poolsの使用は過剰になりました。このツールは優れていますが、この場合には適していません。 

ExtensionDataの出力を見てみましょう。これはGet-Viewオブジェクトにすぎません。 

PowerShellマスターの古くからの手法を見てみましょう。フィルター、並べ替え、およびグループ化を使用する1行です。以前のホラーはすべてエレガントに1行に折りたたまれ、1つのセッションで実行されます。


$AllVMs = Get-View -viewtype VirtualMachine -Property Name,Config.ExtraConfig,summary.runtime.powerstate | Where-Object {($_.Config.ExtraConfig | Where-Object {$_.key -eq "cloud.uuid"}).Value -ne $null} | Select-Object @{N="VMName";E={$_.Name}},@{N="CloudUUID";E={($_.Config.ExtraConfig | Where-Object {$_.key -eq "cloud.uuid"}).Value}},@{N="PowerState";E={$_.summary.runtime.powerstate}} | Sort-Object CloudUUID | Group-Object -Property CloudUUID | Where-Object -FilterScript {$_.Count -gt 1} | Select-Object -ExpandProperty Group

時間を測定します。目的の条件に応じてフィルタリングし、ほぼ1万個のオブジェクトに対して



9秒です。いいね！
