Xamarin.Formsシェル

5月末に、MicrosoftはXamarin.Forms Shellリリースをリリースしました。これは、クロスプラットフォームのモバイルアプリケーションの作成を簡素化し、サイドメニュー、タブ、ナビゲーション、検索などの機能を組み込むことを目的としたシェルです。

Visual Studio 2019で空のXamarin.Formsプロジェクトを作成することから始めましょう。現時点では、シェルは公式に2つのプラットフォームのみをサポートしています。iOSとAndroid、UWPはまだ開発中です。ソリューションのすべてのnugetパッケージをすぐに更新することをお勧めします。



次に、Shellから派生したAppShellクラスを作成します。このために、次の内容でXAMLファイルを一般プロジェクトに追加します

。AppShell.xaml

<?xml version="1.0" encoding="utf-8" ?>
<Shell xmlns="http://xamarin.com/schemas/2014/forms"
       xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
       x:Class="HelloShell.AppShell">

</Shell>

AppShell.xaml.cs

namespace HelloShell
{
    [XamlCompilation(XamlCompilationOptions.Compile)]
    public partial class AppShell : Shell
    {
        public AppShell()
        {
            InitializeComponent();
        }
    }
}

次に、App.xaml.csファイルで、AppShellがMainPageとして機能することを示します。

public App()
{
    InitializeComponent();

    //MainPage = new MainPage();
    MainPage = new AppShell();
}

いくつかのContentPageページ：Page1とPage2。また、画像はテストアプリケーションで使用されるため、Androidの場合はResources => drawableフォルダー、iOSの場合はResourcesフォルダーにプラットフォーム依存のプロジェクトに追加します。

タブ




ルートテンプレートとしてのXamarin.Forms Shellは、下のタブと上のタブ、およびそれらの組み合わせをサポートできます：

AppShell.xaml.cs

<?xml version="1.0" encoding="utf-8" ?>
<Shell xmlns="http://xamarin.com/schemas/2014/forms"
       xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
       xmlns:pages="clr-namespace:HelloShell"
       x:Class="HelloShell.AppShell">

    <TabBar>
        <Tab Title="MainPage" Icon="xamarin.png">
            <ShellContent ContentTemplate="{DataTemplate pages:MainPage}" />
        </Tab>

        <Tab Title="Page1" Icon="xamarin.png">
            <ShellContent Title="Main Page"
                      Icon="xamarin.png"
                      ContentTemplate="{DataTemplate pages:MainPage}" />
            <ShellContent Title="Page2"
                      Icon="xamarin.png"
                      ContentTemplate="{DataTemplate pages:Page2}" />
        </Tab>

        <Tab IsEnabled="False" Title="Page2" Icon="xamarin.png">
            <ShellContent ContentTemplate="{DataTemplate pages:Page2}" />
        </Tab>
    </TabBar>

</Shell>

ご覧のとおり、これは非常に簡単です。もう1つの利点は、ページの効率的な読み込みです。これにより、ユーザーがページにアクセスしたときにのみページを初期化できるため、アプリケーションの起動が大幅に高速化されます。

ナビゲーション


Xamarin.Formsは改善されたURIベースのナビゲーションを提供し、厳密な階層に従うことなくアプリケーションの任意のページに移動し、ナビゲーションスタック内のすべてのページを経由することなく戻ることができます。ナビゲーションを機能させるには、ページを登録する必要があります。Routeプロパティを使用して、FlyoutItem、Tab、ShellContentのXAMLマークアップでこれを行うことができます

<Shell ...>
    <FlyoutItem ...
                Route="page1">
        <Tab ...
             Route="page2">
            <ShellContent ...
                          Route="mainpage" />
            <ShellContent ...
                          Route="page3" />
        </Tab>
        <ShellContent ...
                      Route="page4" />
        <ShellContent ...
                      Route="page5" />
    </FlyoutItem>
    <ShellContent ...
                  Route="about" />                  
    ...
</Shell>

またはコードで

Routing.RegisterRoute("page1", typeof(Page1));

コマンドを使用してナビゲーションが実行されます

await Shell.Current.GoToAsync("//page2");

Xamarin.Formsシェルには、SearchHandlerクラスによって提供される組み込みの検索関数があります。ページに検索関数を追加するには、SearchHandlerから派生したPetSearchHandlerクラスを作成し、OnQueryChangedメソッドとOnItemSelectedメソッドをオーバーライドします。OnQueryChangedメソッドは、ユーザーが検索フィールドにテキストを入力し、2つの引数oldValueとnewValueを受け取ると発生します。oldValueとnewValueには、それぞれ前の検索クエリと新しい検索クエリが含まれています。

SelectedItemメソッドは、ユーザーが検索結果を選択し、オブジェクト（この場合は、Animal）をパラメーターとして受け取ったときに実行されます。

PetSearchHandler.cs

    public class PetSearchHandler : SearchHandler
    {
        protected override void OnQueryChanged(string oldValue, string newValue)
        {
            base.OnQueryChanged(oldValue, newValue);

            if (string.IsNullOrWhiteSpace(newValue))
            {
                ItemsSource = null;
            }
            else
            {
                ItemsSource = PetData.Pets
                    .Where(pet => pet.Name.ToLower().Contains(newValue.ToLower()))
                    .ToList<Animal>();
            }
        }

        protected override async void OnItemSelected(object item)
        {
            base.OnItemSelected(item);
            var pet = item as Animal;
            if (pet is null) return;
            await App.Current.MainPage.DisplayAlert(" ",pet.Name,"ok");
        }
    }

PetSearchHandlerを設定するPetsページを追加します

<?xml version="1.0" encoding="utf-8" ?>
<ContentPage xmlns="http://xamarin.com/schemas/2014/forms"
             ...
             x:Class="HelloShell.Pets">
    <Shell.SearchHandler>
        <controls:PetSearchHandler Placeholder="Enter search term"
                                      ShowsResults="true"
                                      DisplayMemberName="Name" />
    </Shell.SearchHandler>
    <ContentPage.Content>

その結果、ナビゲーションバーに検索フィールドのあるページが表示されます。検索フレーズを入力すると、ペットの名前の簡単なリストが表示されます。



必要に応じて、画像といくつかのテキストラベルをそこに追加することで、リストセルの内容を簡単にカスタマイズできます。

    <Shell.SearchHandler>
        <controls:PetSearchHandler Placeholder="Enter search term"
                                      ShowsResults="true">
            <controls:PetSearchHandler.ItemTemplate>
                <DataTemplate>
                    <StackLayout Padding="10" Orientation="Horizontal">
                        <Image Source="{Binding ImageUrl}"
                               Aspect="AspectFill"
                               HeightRequest="40"
                               WidthRequest="40" />
                        <Label Text="{Binding Name}"
                               FontAttributes="Bold" />
                    </StackLayout>
                </DataTemplate>
            </controls:PetSearchHandler.ItemTemplate>
        </controls:PetSearchHandler>
    </Shell.SearchHandler>
