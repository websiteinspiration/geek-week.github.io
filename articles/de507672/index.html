Ändern und Entfernen von Azure-VMs mithilfe von PowerShell

Mit PowerShell automatisieren Ingenieure und IT-Administratoren erfolgreich verschiedene Aufgaben, wenn sie nicht nur mit lokalen, sondern auch mit Cloud-Infrastrukturen, insbesondere mit Azure, arbeiten. In einigen Fällen ist die Arbeit mit PowerShell wesentlich bequemer und schneller als über das Azure-Portal. Dank seiner plattformübergreifenden Funktionen kann PowerShell für jedes Betriebssystem verwendet werden.



Es spielt keine Rolle, ob Sie mit Ubuntu, Red Hat oder Windows arbeiten - PowerShell hilft bei der Steuerung von Cloud-Ressourcen. Mit dem Azure PowerShell- Modul können Sie beispielsweise beliebige Eigenschaften von virtuellen Maschinen festlegen.



In diesem Artikel wird erläutert, wie Sie mit PowerShell die Größe von VMs in der Azure-Cloud ändern und VMs und die zugehörigen Objekte entfernen können.







Wichtig! Vergessen Sie nicht, Ihre Hände mit einem Desinfektionsmittel abzuwischen , um sich auf die Arbeit vorzubereiten :



  • Sie benötigen das Azure PowerShell-Modul. Sie können es mit dem Befehl aus der PowerShell-Galerie herunterladen Install-Module Az



    .
  • Sie müssen sich in der Azure-Cloud authentifizieren, in der sich die virtuelle Maschine dreht, indem Sie den Befehl ausführen Connect-AzAccount



    .


Erstellen Sie zunächst ein Skript, dessen Größe sich in der Größe von Azure VM ändert. Öffnen Sie VS Code und speichern Sie ein neues PowerShell-Skript mit dem Namen Resize-AzVirtualMachine.ps1 . Im Verlauf des Beispiels werden Code -Teile hinzugefügt .



Wir fordern die verfügbaren VM-Größen an



Bevor Sie die Größe der VM ändern, müssen Sie herausfinden, welche Größen für virtuelle Maschinen in der Azure-Cloud im Allgemeinen akzeptabel sind. Führen Sie dazu den Befehl aus Get-AzVMSize



.



Für die virtuelle Maschine devvm01 aus der Ressourcengruppe dev fordern wir daher alle möglichen zulässigen Größen an: 



Get-AzVMSize -ResourceGroupName dev -VMName devvm01


(Bei realen Aufgaben geben Sie natürlich anstelle von ResourceGroupName = dev und VMName = devvm01 Ihre Werte für diese Parameter an.)



Der Befehl gibt eine Liste wie die folgende zurück:







Dies sind alle möglichen Größen, die für diese virtuelle Maschine festgelegt werden können.



Ändern Sie die Größe des Autos



Zum Beispiel werden wir die Größe auf die neue Größe von Standard_B1ls ändern - sie steht in der obigen Liste an erster Stelle. (Bei tatsächlichen Aufgaben wählen Sie natürlich jede gewünschte Größe.)



  1. Zuerst erhalten Get-AzVM



    wir mit dem Befehl Informationen über unser Objekt (virtuelle Maschine) und speichern sie in einer Variablen $virtualMachine



    : 



    $virtualMachine = Get-AzVM -ResourceGroupName dev -VMName devvm01
  2. Dann nehmen wir eine Eigenschaft von diesem Objekt .HardwareProfile.VmSize



    und setzen den gewünschten neuen Wert: 



    $virtualMachine.HardwareProfile.VmSize = "Standard_B1ls"
  3. Und jetzt führen Sie einfach den Update-Befehl VM - Update-AzVm



    : 



    Update-AzVM -VM devvm01 -ResourceGroupName dev
  4. Wir sorgen dafür, dass alles gut gegangen ist - auch hier fordern wir Informationen zu unserem Objekt an und schauen uns die Immobilie an $virtualMachine.HardwareProfile



    : 



    $virtualMachine = Get-AzVM -ResourceGroupName dev -VMName devvm01
$virtualMachine.HardwareProfile


Wenn wir dort Standard_B1ls sehen , ist alles in Ordnung, die Größe der Maschine wurde geändert. Sie können noch weiter gehen und Erfolg entwickeln, indem Sie die Größe mehrerer VMs gleichzeitig mithilfe eines Arrays ändern.



Was ist mit dem Entfernen von VMs in Azure?



Beim Entfernen ist nicht alles so einfach und unkompliziert, wie es scheint. Schließlich müssen Sie eine Reihe von Ressourcen löschen, die diesem Computer zugeordnet sind, darunter:



  • Speichern Sie Diagnosespeicherbehälter
  • Netzwerk Schnittstellen
  • Öffentliche IP-Adressen
  • Systemdiskette und Blob, auf dem der Status gespeichert ist
  • Datenplatten


Daher erstellen wir eine Funktion und benennen sie Remove-AzrVirtualMachine



- und sie löscht nicht nur die Azure-VM, sondern auch alle oben genannten Funktionen.



Wir folgen dem Standardpfad und erhalten zuerst unser Objekt (VM) mit dem Befehl Get-AzVm



. Angenommen , es handelt sich um ein Auto WINSRV19 der Ressourcengruppe MyTestVMs .



Speichern wir dieses Objekt zusammen mit all seinen Eigenschaften in einer Variablen $vm



: 



$vm = Get-AzVm -Name WINSRV19 -ResourceGroupName MyTestVMs


Löschen Sie den Container mit den Startdiagnosedateien



Beim Erstellen einer VM in Azure wird der Benutzer außerdem aufgefordert, einen Container zum Speichern der Startdiagnose (Startdiagnosecontainer) zu erstellen, damit Sie sich bei Problemen mit dem Start an die Fehlerbehebung wenden können. Wenn die VM entfernt wird, bleibt dieser Container jedoch bestehen, um seine jetzt zwecklose Existenz fortzusetzen. Korrigieren Sie diese Situation.



  1. Stellen Sie zunächst fest, zu welchem ​​Speicherkonto dieser Container gehört. Dazu müssen wir die Eigenschaft storageUri



    im Darm des Objekts DiagnosticsProfile



    unserer VM finden. Dafür verwende ich diesen regulären Ausdruck: 



    $diagSa = [regex]::match($vm.DiagnosticsProfile.bootDiagnostics.storageUri, '^http[s]?://(.+?)\\.').groups[1].value
  2. Jetzt müssen wir den Namen des Containers herausfinden und dafür die VM-ID mit dem folgenden Befehl abrufen Get-AzResource



    : 



    
if ($vm.Name.Length -gt 9) {
    $i = 9
} else {
    $i = $vm.Name.Length - 1
}
 
$azResourceParams = @{
    'ResourceName' = WINSRV
    'ResourceType' = 'Microsoft.Compute/virtualMachines'
    'ResourceGroupName' = MyTestVMs
}
 
$vmResource = Get-AzResource @azResourceParams
$vmId = $vmResource.Properties.VmId
$diagContainerName = ('bootdiagnostics-{0}-{1}' -f $vm.Name.ToLower().Substring(0, $i), $vmId)
  3. Als nächstes erhalten wir den Namen der Ressourcengruppe, zu der der Container gehört: 



    $diagSaRg = (Get-AzStorageAccount | where { $_.StorageAccountName -eq $diagSa }).ResourceGroupName
  4. Und jetzt haben wir alles, was wir brauchen, um den Container mit dem Befehl zu entfernen Remove-AzStorageContainer



    : 



    $saParams = @{
    'ResourceGroupName' = $diagSaRg
    'Name' = $diagSa
}
 
Get-AzStorageAccount @saParams | Get-AzStorageContainer | where { $_.Name-eq $diagContainerName } | Remove-AzStorageContainer -Force


VM löschen



Jetzt löschen wir die virtuelle Maschine selbst, da wir bereits eine Variable $vm



für das entsprechende Objekt erstellt haben. Führen Sie den folgenden Befehl aus Remove-AzVm



: 



$null = $vm | Remove-AzVM -Force


Löschen Sie die Netzwerkschnittstelle und die öffentliche IP-Adresse



Unsere VM verfügt über eine (oder sogar mehrere) Netzwerkschnittstellen (NICs). Um sie als unnötig zu entfernen, gehen Sie die Eigenschaft NetworkInterfaces



unseres VM-Objekts durch und löschen Sie den NIC-Befehl Remove-AzNetworkInterface



. Wenn es mehr als eine Netzwerkschnittstelle gibt, verwenden wir eine Schleife. Gleichzeitig überprüfen wir für jede Netzwerkkarte die Eigenschaft, um festzustellen, IpConfiguration



ob die Schnittstelle eine öffentliche IP-Adresse hat. Wenn dies gefunden wird, löschen Sie es mit einem Befehl Remove-AzPublicIpAddress



.



Hier ist ein Beispiel für einen solchen Code, bei dem wir alle Netzwerkkarten durchlaufen, sie löschen und prüfen, ob eine öffentliche IP vorhanden ist. Wenn dies der PublicIpAddress



Fall ist, analysieren wir die Eigenschaft , ermitteln den Namen der entsprechenden Ressource anhand der ID und löschen sie: 




foreach($nicUri in $vm.NetworkProfile.NetworkInterfaces.Id) {
    $nic = Get-AzNetworkInterface -ResourceGroupName $vm.ResourceGroupName -Name $nicUri.Split('/')[-1]
    Remove-AzNetworkInterface -Name $nic.Name -ResourceGroupName $vm.ResourceGroupName -Force

    foreach($ipConfig in $nic.IpConfigurations) {
        if($ipConfig.PublicIpAddress -ne $null) {
            Remove-AzPublicIpAddress -ResourceGroupName $vm.ResourceGroupName -Name $ipConfig.PublicIpAddress.Id.Split('/')[-1] -Force
        }
    }
}


Löschen Sie das Systemlaufwerk



Die Betriebssystemfestplatte ist ein Blob, für dessen Entfernung ein Befehl vorhanden ist. Remove-AzStorageBlob



Bevor Sie ihn jedoch ausführen, müssen Sie die erforderlichen Werte für die Parameter festlegen. Insbesondere hierfür müssen Sie den Namen des Speichercontainers abrufen, der die Systemfestplatte enthält, und ihn dann zusammen mit dem entsprechenden Speicherkonto an diesen Befehl übertragen. 



$osDiskUri = $vm.StorageProfile.OSDisk.Vhd.Uri
$osDiskContainerName = $osDiskUri.Split('/')[-2]
$osDiskStorageAcct = Get-AzStorageAccount | where { $_.StorageAccountName -eq $osDiskUri.Split('/')[2].Split('.')[0] }
$osDiskStorageAcct | Remove-AzStorageBlob -Container $osDiskContainerName -Blob $osDiskUri.Split('/')[-1]


Löschen Sie den Status der Blob-Systemfestplatte



Dazu nehmen wir, wie Sie wahrscheinlich bereits vermutet haben, den Speichercontainer, in dem diese Festplatte gespeichert ist, und übergeben unter der Annahme, dass der Blob am Ende enthält status



, die entsprechenden Parameter an den Befehl delete Remove-AzStorageBlob



: 



$osDiskStorageAcct | Get-AzStorageBlob -Container $osDiskContainerName -Blob "$($vm.Name)*.status" | Remove-AzStorageBlob


Und schließlich löschen Sie Daten-Discs



An unsere VM können Festplatten mit Daten angehängt sein. Wenn sie nicht notwendig sind, werden wir sie auch entfernen. Analysieren Sie zunächst StorageProfile



unsere VM und suchen Sie die Eigenschaft Uri



. Wenn es mehrere Festplatten gibt, organisieren wir einen Zyklus nach URI



. Für jede URI finden wir das entsprechende gespeicherte Konto mit Get-AzStorageAccount



. Analysieren Sie dann den Speicher-URI, um den benötigten Blob-Namen zu erhalten, und übergeben Sie ihn Remove-AzStorageBlob



zusammen mit Ihrem Konto an den Befehl delete . So würde es im Code aussehen: 



if ($vm.DataDiskNames.Count -gt 0) {
    foreach ($uri in $vm.StorageProfile.DataDisks.Vhd.Uri) {
        $dataDiskStorageAcct = Get-AzStorageAccount -Name $uri.Split('/')[2].Split('.')[0]
        $dataDiskStorageAcct | Remove-AzStorageBlob -Container $uri.Split('/')[-2] -Blob $uri.Split('/')[-1]
    }
}
