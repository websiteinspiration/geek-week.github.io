JavaScriptの末尾再帰の最適化

こんにちは読者。

問題を解決するには、長所と短所がある再帰を使用する必要がある場合があります。スタックオーバーフローの問題が発生しました。
最大再帰深度はJavaScriptエンジンによって制限されます。ネストされた10,000の呼び出しを正確に数えることができます。一部のインタープリターはそれ以上を許可しますが、それらのほとんどでは、100,000の呼び出しは機能の範囲を超えています。呼び出しスタックのオーバーフローを回避するのに役立つ自動最適化（「末尾再帰最適化」）がありますが、まだどこでもサポートされておらず、単純なケースでのみ機能します。

再帰関数の例：
function sum(n) {
  return n === 0 ? 0 : n + sum(n - 1)
}

sum(5) // 1 + 2 + 3 + 4 + 5  = 15
sum(100000) // Error: Maximum call stack size exceeded.


末尾再帰により、コンパイラによる呼び出しの最適化が可能になり、すでにES6標準に含まれていますが、ブラウザのサポートには多くの機能が残されています。

末尾再帰関数の例：
function sum(number, s = 0){
  return number === 0 ? s : sum(number - 1, s + number)
}

しかし、ブラウザのサポートなしでは、同じ問題に直面します-スタックオーバーフロー トランポリンとの併用も可能です。

変更された再帰関数（次のステップ）を受け取り、呼び出しの深さを増やすことなくループで実行するデコレーターを作成します。
function trampoline(fn) {
  return function(...args) {
    let result = fn.apply(fn.context, args)
    while (typeof result === 'function') {
      result = result()
    }

    return result
  }
}


これで、再帰関数はデコレーターによってすぐに実行される関数を返すはずです。このようにして、条件付きで関数の配列を作成し、それらを順番に実行します。しかし、新しい無名関数を返すたびに、このコードは少し遅く動作します。
function sum(number, s = 0) {
  return number === 0 ? s : () => sum(number - 1, s + number)
}


を使用しております：
const trampolineSum = trampoline(sum)
trampolineSum(100000) // 5000050000

これは私の最初の記事です。私は概念を再説明しないようにして、ソースへのリンクを提供しました。トピックは一意ではなく、英語のサイトに長く存在していますが、残念ながらロシア語では見つかりませんでした。
