今日のワイヤレス技術で何が起こっているのか


前の世代のWi-Fiが開発された年（業界で4番目と5番目）に、完全にワイヤレスのオフィス、つまり完全にワイヤレスのオフィススペースという概念が形成されました。しかし、それ以来、大量の水が流れ、Wi-Fiに関するビジネスニーズは質的および量的に変化しました。帯域幅の要件が増加し、遅延の削減が重要になり、さらに、多数のユーザーを接続することが急務となっています。





2020年までに、Wi-Fiネットワーク経由で確実に機能する新しいアプリケーションのランドスケープが形成されました。図は、そのようなアプリケーションが関連する主な領域を示しています。それらのいくつかについて簡単に説明します。

HuaweiがWi-Fi 6の管理を提供する方法


Huaweiは現在、既成のエンドツーエンドのクラウドキャンパスソリューションを推進しています。これは、一方ではクラウドからインフラストラクチャ全体を管理し、もう一方では新しいIoTシナリオを実装するためのプラットフォームとして機能することを目的としています。建物の管理、機器の監視、または、たとえば、医療分野の事例を参照して、患者のバイタルサインを監視する。

Cloud Campusを取り巻くエコシステムの重要な部分は市場です。たとえば、開発者が適切なソフトウェアを作成してエンドデバイスを作成し、Huaweiソリューションと統合した場合、サービスモデルに従って他の顧客が製品を利用できるようにする権利があります。



Wi-Fiネットワークは基本的にビジネスの基盤となるため、これまでの管理方法では不十分です。以前は、管理者はほとんど手動でログを調べて、ネットワークで何が起こっているのかを理解するよう強制されていました。そのような反応的な支援体制は今では十分ではありません。ワイヤレスインフラストラクチャをプロアクティブに監視および管理するためのツールが必要です。これにより、管理者は、何が起こっているかを確実に理解できます。どのレベルのユーザーエクスペリエンスが提供されるか、新しいユーザーが問題なくそれに接続できるか、一部のクライアントを「転送」する必要があるか隣接するアクセスポイント（AP）へ、各状態の個々のネットワークノードの状態など

Wi-Fi 6デバイスに関連して、Huaweiはネットワークで何が起こっているかを予防的かつ詳細に分析および制御するためのすべてのツールを備えています。これらの開発は、主に機械学習アルゴリズムに基づいています。

以前のシリーズのアクセスポイントでは、対応するテレメトリプロトコルをサポートしていないため、これは不可能でした。一般に、これらのデバイスのパフォーマンスでは、最新のアクセスポイントが許可する形式でこの機能を実現できませんでした。

1年半でようやくWi-Fi 6時代が到来します。

まず、Wi-Fi 6は、ワイヤレスネットワーク全体の効率を高めるように設計されています。各ステーションの前に、タイムスロットが順次発行され、20 MHzチャネル全体を占有しました。これに関連して、残りのステーションはトラフィックを送信するまで待機する必要がありました。現在、これらの20 MHzは、最大2 MHzのリソースユニットに結合された小さなサブキャリアにカットされ、最大9つのステーションが1つのタイムスロットで同時にブロードキャストできます。したがって、ネットワークパフォーマンスが大幅に向上します。

第6世代の標準には、以前の256に対して1024-QAMより高い変調方式が追加されたと既に述べました。したがって、エンコードの複雑度は25％増加しました。以前に1文字あたり最大8ビットの情報を送信した場合、現在は10ビット。

空間ストリームの数も増加しています。以前の規格では最大4つでしたが、現在は最大8つであり、Huaweiの古いアクセスポイントでは最大12個です。

さらに、Wi-Fi 6は再び2.4 GHzの周波数範囲を使用します。これにより、Wi-Fi 6をサポートするエンドターミナル用のチップセットを比較的安価に製造でき、本格的なIoTモジュールでも完全に安価なデバイスでも、膨大な数のデバイスを接続センサー。

特に重要なことは、チャネルと周波数の再利用を含め、無線スペクトルをより効率的に使用するための多くのテクノロジが標準で実装されていることです。最初に、基本サービスセット（BSS）のカラーリングについて言及する価値があります。これにより、同じチャネルで動作する他の人のアクセスポイントを無視し、同時に自分自身を "リッスン"できます。

最初のAirEngine 5760-51シリーズのデバイスは、消費者にとって最も手頃な価格であり、最も一般的なシナリオ向けに設計されています。たとえば、小売の場合です。ただし、それらはオフィスのニーズにも適しており、使用される技術スタックとコストの点で普遍的です。

1つは空間ストリームの数です（2.4 GHzで最大4つ、5 GHzで最大8つ）。 2番目の単位は、使用される標準に従って、シンボルの期間とガードインターバルの期間の合計で割った単位です。 Wi-Fi 6では、シンボルの持続時間は4倍になり、最大12.8μsで、ガードインターバルは0.8μsなので、結果として1 / 13.6μsになります。

次に、思い出してください。1024-QAM変調が改善され、1文字あたり最大10ビットをエンコードできるようになりました。合計でビットレートは5/6（FEC）で、4番目の要素です。そして、5番目はサブキャリア（トーン）の数です。

最後に、2.4 GHzと5 GHzの最大パフォーマンスを合計すると、10.75 Gbit / sというすばらしい値が得られます。

無線リソースの分配におけるもう1つの改善は、SmartRadioテクノロジーをもたらしました。以前は、1つのゾーンに複数のアクセスポイントがあった場合、クライアントを再配布するアルゴリズム、新しいAPに接続するAPなどを指定できました。ただし、これらの設定は、Wi-Fiネットワークとの接続および関連付け時に一度だけ適用されました。 AirEngineの場合、負荷分散アルゴリズムは、クライアントが動作し、たとえばアクセスポイント間を移動するときにリアルタイムで適用できます。



アンテナ要素に関する重要なニュアンス：AirEngineモデルでは、垂直偏波と水平偏波の両方を同時に実現します。それぞれが4つのアンテナをサポートし、そのような要素が4つあります。したがって、総数-16アンテナ。



アンテナエレメント自体はパッシブです。したがって、クライアントの方向により多くのエネルギーを集中させるためには、コンパクトなアンテナを使用してより狭いビームを形成する必要があります。ファーウェイは成功しました。結論-無線のカバレッジは、競合するソリューションよりも平均20％多いです。

。Huaweiが新しいアクセスポイントに統合したもう1つのテクノロジーは、ダイナミックターボと呼ばれます。その本質は、オンザフライTDがアプリケーションをクラスごとに認識して分類し（たとえば、リアルタイムビデオ、音声トラフィックなどを送信する）、クライアントをその重要性で区別し、このようにリソースユニットを割り当てることです。高レベルのアプリケーションをできるだけ早くユーザーにとって重要にする。実際、ハードウェアレベルでは、アクセスポイントはDPI（詳細なトラフィック分析）を実行します。



前述したように、Huaweiは現在、そのソリューションでMU-MIMOとOFDMAの同時動作を提供する唯一のベンダーです。 2つの違いは何かについてもう少し詳しく見てみましょう。

どちらのテクノロジーも、マルチユーザーアクセスを提供するように設計されています。ネットワークに多くのユーザーがいる場合、OFDMAを使用すると、周波数リソースを分散できるため、多くのクライアントが一度に情報を送受信できます。ただし、MU-MIMOは最終的に同じことを目的としています。複数のクライアントが部屋の異なるポイントにいる場合、それぞれに一意の空間ストリームを送信できます。明確にするために、周波数リソースは「モスクワ-サンクトペテルブルク」ルートであると想定します。 OFDMAが提案するように：「道路を1つの車線ではなく2つの車線にして、より効率的に使用できるようにしましょう。」 MU-MIMOには別のアプローチがあります。「2番目、3番目の道路を舗装して、交通が独立したルートに沿って進むようにします。」理論的には、一方が他方と矛盾しないため、実際には、2つの方法の組み合わせには特定のアルゴリズムベースが必要です。Huaweiがこのベースを作成したという事実のおかげで、アクセスポイントのスループットは、競合他社が提供できるものと比較してほぼ40％増加しました。





セキュリティに関しては、以前のモデルと同様に、新しいアクセスポイントはDTLSをサポートしています。したがって、以前と同様に、制御CAPWAPトラフィックを暗号化できます。

外部の悪意のある影響からの保護により、すべてが前世代のコントローラーと同じです。ブルートフォース、Weak IV攻撃（弱い初期化ベクトル）など、あらゆる種類の攻撃がリアルタイムで検出されます。DDoSへの応答も構成されています。システムは動的ブラックリストを作成したり、分散ネットワーク攻撃を試みたときに何が起こっているかを管理者に通知したりできます。

AirEngineモデルのソリューションは何ですか？




Wi-Fi 6のコンテキストでのCampusInsight分析プラットフォームは、いくつかの問題を解決します。まず、コントローラーと同等の無線管理に関与します。CampusInsightを使用すると、キャリブレーションをリアルタイムで実行して、チャネルを最適に分配し、信号強度とチャネルの帯域幅を調整し、Wi-Fiネットワークで一般的に行われる処理を制御できます。そのため、CampusInsightはワイヤレスセキュリティ（特に侵入防止と侵入検出）に適用でき、特定のアクセスポイントや1つのSSIDに関連するのではなく、ワイヤレスインフラストラクチャ全体の規模に適用できます。



WLAN Plannerも注目に値します-無線モデリングツールで、壁などの障害物のいくつかを個別に決定できます。出力では、プログラムは簡単なレポートを提供します。これは、とりわけ、部屋をカバーするために必要なアクセスポイントの数を示します。そのような入力に基づいて、機器の仕様、予算などについてより多くの情報に基づいた決定を行うことがはるかに簡単になります。
