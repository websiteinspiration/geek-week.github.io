What are we observing here?

Photos are wrapped in a slideshow class, which is used to align elements horizontally and vertically.

.slideshow has the following styles:

.slideshow {
    width: 100vw;
    height: 100vh;
    display: flex;
    justify-content: center;
    align-items: center;
}

We have 6 photos. Each photo is assigned the class "i_number of photos". Under each photo there is a signature with the class ‚Äúp_number of photo‚Äù.

Img has the following styles:

    position: absolute;
    max-width: 90vw;
    max-height: 80vh;
    opacity: 0;
    animation: slideshow 36s ease-in-out infinite;
}

Animation time depends on the number of photos and the display time of each photo. In this case: 6 photos for 6 seconds each.

Let's move on to the animation:

@keyframes slideshow {
    from {
        opacity: 0;
        filter: grayscale(100%);
    }

    5% {
        opacity: 1;
        filter: grayscale(50%);
    }

    10% {
        opacity: 1;
        filter: grayscale(0);
        transform: scale(1.1);
    }

    15% {
        opacity: 1;
        filter: blur(0);
    }

    20% {
        opacity: 0;
        filter: blur(2px);
        transform: scale(0.9);
    }

    25% {
        opacity: 0;
    }
}

We use only 25% of the animation, but in six steps. 25% is the fourth part, so the photos will overlap slightly. Since we are implementing the ‚Äúfade in-fade out‚Äù effect, i.e. We work with transparency, this is exactly what we need.

With the help of ‚Äúfilter: grayscale‚Äù we saturate the color of the displayed (discolored) photo, with the help of ‚Äútransform: scale‚Äù we scale the photo a little forward when outputting and a little back when replacing, using ‚Äúfilter: blur‚Äù we blur the replaced photo.

Next, we work with individual photos:

    animation-delay: 6s;
    transform: translateX(-100px) rotate(-3deg);
}

.slideshow .i_2 {
    animation-delay: 12s;
    transform: translateX(100px) rotate(3deg);
}

.slideshow .i_3 {
    animation-delay: 18s;
    transform: translateY(-100px) rotate(3deg);
}

.slideshow .i_4 {
    animation-delay: 24s;
    transform: translateY(100px) rotate(-3deg);
}

.slideshow .i_5 {
    animation-delay: 30s;
    transform: rotate(-6deg);
}

.slideshow .i_6 {
    animation-delay: 36s;
    transform: rotate(6deg);
}

We set the animation delay for each photo (6 seconds of the first and +6 seconds of each subsequent). For a change, we shift and rotate each photo using "transform: translate rotate" (the values ‚Äã‚Äãwere selected empirically).

The animation time for the captions corresponds to the animation time for the photo.
