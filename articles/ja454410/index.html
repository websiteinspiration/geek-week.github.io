PHP 7.4の新機能

PHPの新しいバージョンは、マイナーではありますが、言語の構文とそのパフォーマンスの両方について、誇張なしに多くの新しい機能を既に備えています。イノベーションのリストは最終的なものではありませんが、主な変更はすでに行われ、受け入れられています。リリースは2019年12月に予定されています。
 

 
次期バージョンの主な変更点：

  • 型付きクラスのプロパティ
  • パフォーマンス向上のためのプリロード
  • 匿名関数を書くための矢印関数
  • ユニオン演算子にnullを割り当てる（?? =）
  • レガシーメソッドシグネチャの共分散/反分散
  • PHPで拡張機能を開発するための新しい可能性を開く外部関数インターフェース
  • 配列での演算子のアンパック

猫の下でこれらと他の変更についての詳細を読んでください。

免責事項：同僚との私の議論で何度か、Brentの記事「PHP 7.4の新機能」が取り上げられました。最初は翻訳をしたかったのですが、その過程で最新の更新がすべてテキストで示されているわけではなく、不正確な点もあるので、翻訳の代わりにこの記事が表示されました。

アロー関数（RFC）


矢印関数を使用すると、無名関数の短い記録を作成できます。

array_map(function (User $user) {
    return $user->id;
}, $users)

array_map(fn(User $user) => $user->id, $users)

採用された矢印関数の実装のいくつかの機能：

  • parentエリアにアクセスできるため、キーワードを使用する必要はありませんuse。
  • $this 通常の無名関数と同様に利用できます。
  • 矢印関数には、1行しか含めることができません。これは、returnステートメントでもあります。

これらについては、Habrのこの記事を参照してください。

型付きプロパティ（RFC）

 
やったー！クラスプロパティにタイプヒントを設定できるようになりました。これは、PHP 7以降、言語のタイピングを強化するための非常に長い間待ち望まれていた変更です。これで、強いタイピングのためのすべての基本機能が手に入りました。すべてのタイプは、タイピングのために用意されてい除く   voidと、callable.

class Bar
{
    public string $name;
    public ?int $amount;
    public Foo $foo;
}

オブジェクトのプロパティを入力する新しい可能性についてさらに詳しく検討してほしい場合は、コメントに注意してください。それらについては、別の記事を書きます！

null結合演算子（RFC）を割り当てます


そのような長いエントリの代わりに：

$data['date'] = $data['date'] ?? new DateTime();

これで次のように書くことができます：

$data['date'] ??= new DateTime();

配列解凍演算子（RFC）


これで、配列でunpack演算子を使用できます。

$arrayA = [1, 2, 3];
$arrayB = [4, 5];
$result = [0, ...$arrayA, ...$arrayB, 6 ,7];
<i>// [0, 1, 2, 3, 4, 5, 6, 7]</i>

これは非連想配列でのみ機能することに注意してください。

外部関数インターフェース（RFC）


外部関数インターフェース（FFI）を使用すると、PHPコードで直接Cコードを記述できます。これは、PHP拡張が純粋なPHPで記述できることを意味します。

これは複雑なトピックであることに注意してください。これらの機能を正しく使用するには、Cの知識が必要です。

プリロード（RFC）


プリロードは、PHPコアへのすばらしい追加であり、パフォーマンスが大幅に向上するはずです。

たとえば、フレームワークを使用する場合、そのファイルをダウンロードして、リクエストごとに再コンパイルする必要があります。opcacheを使用する場合、これらのファイルが最初にリクエストの処理に参加し、次に変更のチェックが成功するたびに。プリロードにより、サーバーは指定されたPHPファイルを起動時にメモリにロードし、ファイルの変更を追加でチェックすることなく、後続のすべてのリクエストでそれらを常に使用できるようにすることができます。

パフォーマンスの向上は「フリー」ではありません-プリロードされたファイルが変更された場合、サーバーを再起動する必要があります。

レガシーメソッドシグネチャの共分散/逆分散（RFC）


PHPは現在、ほとんど不変のパラメーター型と不変の戻り型を持っています。この変更により、パラメーターのタイプをそのスーパータイプの1つに変更できます。次に、返されたタイプをそのサブタイプで置き換えることができます。したがって、この変更により、置換の原則Barbara Liskovをより厳格に従うことができます。

共変の戻り値の型の使用例：

interface Factory {
    function make(): object;
}
class UserFactory implements Factory {
    function make(): User;
}

そして反変引数：

interface Concatable {
    function concat(Iterator $input);
}
class Collection implements Concatable {
    function concat(iterable $input) {/* . . . */}
}

カスタムオブジェクトのシリアル化（RFC）


2つの新しい魔法のメソッドが利用可能になります：__serializeと__unserialize。このシリアル化メカニズムは、インターフェイスの汎用性とメソッドのSerializable実装 __sleep/__wakeup方法を組み合わせています。それらの違いの詳細はRFCにあります。

連結優先度（RFC）


次のようなものを書いた場合：

echo "sum: " . $a + $b;

PHPはこれを次のように解釈します。

echo ("sum: " . $a) + $b;

PHP 8はこれを異なる方法で解釈します。

echo "sum :" . ($a + $b);

PHP 7.4は、 "。"を含む式を検出すると陳腐化警告を追加します。"+"または "-"の前に、かっこなしで同時に。

__toString（RFC）の例外サポート


以前は、魔法のメソッドから例外をスローすることができませんでした   __toString。この動作の理論的根拠は、オブジェクトの文字列への変換が標準ライブラリの多くの関数で実行され、それらのすべてが例外を正しく「処理」する準備ができているわけではないということです。このRFCの一部として、コードベースでの文字列変換の包括的な監査が行われ、この制限を削除できるようになりました。

リンクリフレクション（RFC）


それらのようなライブラリーは、symfony/var-dumper変数の正確な導出に大きく依存していますReflectionAPI。以前は、リンクリフレクションに対する適切なサポートがなかったため、これらのライブラリはハックに依存してリンクを検出する必要がありました。PHP 7.4は、ReflectionReferenceこの問題を解決するクラスを追加します。

追加されたメソッドmb_str_split（RFC）


この関数はと同じ機能を提供str_splitしますが、マルチバイトエンコーディングで記​​述された文字列用です。

Always Available Extension ext-hash（RFC）


この拡張機能は現在、すべてのPHPインストールで常に利用可能です。

PEARはデフォルトでは有効になっていません（EXTERNALS）


PEARはもはや積極的にサポートされていません。コアチームは、PHP 7.4のデフォルトのインストールからPEARを削除することを決定しました。

パスワードハッシュアルゴリズムレジストリ（RFC）


password_algos登録 されているすべてのパスワードハッシュアルゴリズムのリストを返す新しい関数が追加されました。

弱いリンク（RFC）


弱いリンクを使用すると、オブジェクトへのリンクを保存して、このオブジェクトの破棄を妨げることはありません。たとえば、キャッシュのような構造を実装するのに役立ちます。

数値リテラル区切り文字（RFC）


数値のグループに視覚的な区切り記号がないと、コードの読み取りとデバッグにかかる​​時間が長くなり、意図しないエラーが発生する可能性があります。数値リテラルの下線文字のサポートが追加され、数値のグループを視覚的に分離できるようになりました。

1_000_000_000  // int
6.674_083e-11; // float
299_792_458;   // decimal
0xCAFE_F00D;   // hexadecimal
0b0101_1111;   // binary
0137_041;      // octal

非推奨の短いオープンタグ（RFC）


短い開始タグは<?廃止され、PHP 8で削除されます。短いタグは<?= (echo)影響を受けませんでした。

Left Associative Ternary Operator Declined Obsolete（RFC）


三項演算子には、PHPの奇妙な癖がいくつかあります。このRFCでは、ネストされた3項演算子は廃止されています。

1 ? 2 : 3 ? 4 : 5;   // deprecated
(1 ? 2 : 3) ? 4 : 5; // ok

PHP 8では、このようなレコードはコンパイルレベルのエラーになります。
