WordPress開発環境の別のバージョン（docker、wp-cli）

WordPressのデプロイとwp-cliの使用に関する公式イメージがあります。しかし、それらが一緒に使用されると、WordPressフォルダー全体を共有する必要があり、インストールをカスタマイズするためのより多くのオプションが必要でした。


インストール言語を選択するための招待状ではなく、完成したサイトをブラウザに表示したかったのです。


この記事は、WordPressサイトをデプロイする1つの方法を提供します。


取り付け


git clone https://github.com/eustatos/wordpress-docker-dev-env.git

または、ご希望の場合ssh：


git clone git@github.com:eustatos/wordpress-docker-dev-env.git

次に：


cd wordpress-docker-dev-env
docker-compose up -d
docker-compose exec -T wp-cli sh < install.sh

. localhost:8080 .
localhost:8080/wp-admin.
— admin
— 123


site.env .env.


wp-content, \ .


wp-content:


sudo usermod -a -G www-data $USER


, . , \ - — install.sh.


#!/usr/bin/env bash

wp core download --force --allow-root
wp config create \
  --dbhost=$WORDPRESS_DB_HOST \
  --dbname=$WORDPRESS_DB_NAME \
  --dbuser=$WORDPRESS_DB_USER \
  --dbpass=$WORDPRESS_DB_PASSWORD \
  --allow-root
wp core install \
  --title=$TITLE \
  --url=$URL \
  --admin_user=$ADMIN_USER \
  --admin_password=$ADMIN_PASSWORD \
  --admin_email=$ADMIN_EMAIL \
# add for skip creation default themes
  --skip-themes
# add for skip creation default plugins
  --skip-plugins
  --allow-root

wp core install
