VS Codeでのリモート開発：あなたの人生は決して同じにはなりません

TestMaceでは、Visual Studioコードは最も人気のあるツールの1つです。そして、それが飛躍的に発展していることは、なおさら励みです。あなたの注意は、VS Code Insidersでのみ利用可能な別のおいしい機能に関する記事の翻訳に招待されます。


私は同意します、私は見出しに行き過ぎました、私は感情に圧倒されました。この記事の執筆時点で、VS Code Insidersで入手できるリモート開発の第一印象を共有することにしました。


リモート開発用の拡張機能には、Visual Studio Code Insidersのインストールが必要です。


Visual Studio Codeでのリモート開発の可能性は、コンテナー、リモートコンピューター、またはLinux用のWindowsサブシステム（WSL）を完全な開発環境として使用できることです。同時に、プロセス全体が2つの部分に分かれています。アプリケーションのクライアント部分はローカルコンピューターで実行され、VS Codeサーバーはほとんどどこにでもあります。リモート開発用の拡張キットには、3つの拡張が含まれています。次の3つの記事は、それぞれの記事をさらに詳しく学ぶのに役立ちます。


  • リモート-SSH-SSHを使用したリモートまたは仮想マシン上の任意のディレクトリへのアクセス。
  • リモート-コンテナー-コンテナー内の（またはコンテナーにマウントされた）分離されたツールまたはコンテナー化されたアプリケーションのセットを操作します。
  • リモート-WSL -Linux用のWindowsサブシステム（WSL）でのアプリケーション開発。
    具体的な例を挙げましょう。アプリケーションを任意の言語で開発する必要があるが、コンピューターに言語自体もSDKも必要なツールがない場合を考えてみます。
