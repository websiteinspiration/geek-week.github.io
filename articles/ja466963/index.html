Habrastatistics：Habrがオタクのない生活を送る方法

こんにちは、ハブル。

この記事は、2018年のBest Habrの記事のランキングの論理的な続きです。そして年はまだ終わっていませんが、ご存知のように、夏にはそれぞれルールに変更があり、何か影響があるかどうかが面白くなりました。



統計自体に加えて、記事の更新された評価と、これがどのように機能するかに興味がある人のためのいくつかのソースコードが与えられます。

何が起こったかに興味がある人のために、カットの下で続けました。サイトのセクションのより詳細な分析に興味がある人は、次の部分も見ることができます。

初期データ


この評価は非公式であり、内部データはありません。見やすいように、ブラウザのアドレスバーを見ると、Habréのすべての記事にエンドツーエンドの番号が付けられています。さらに、それはテクノロジーの問題であり、サイクル内のすべての記事を（1つのスレッドで、サーバーをロードしないように一時停止して）連続して読み取るだけです。値自体はPythonの単純なパーサーによって取得され（ソースはここにあります）、ほぼ次のタイプのcsvファイルに保存されています。

2019-08-11T22:36Z,https://habr.com/ru/post/463197/,"Blazor + MVVM = Silverlight , ",votes:11,votesplus:17,votesmin:6,bookmarks:40,views:5300,comments:73
2019-08-11T05:26Z,https://habr.com/ru/news/t/463199/," NASA ",votes:15,votesplus:15,votesmin:0,bookmarks:2,views:1700,comments:7

処理


解析には、Python、Pandas、およびMatplotlibを使用します。統計に興味がない人は、この部分をスキップしてすぐに記事に行くことができます。

最初に、データセットをメモリにロードし、目的の年のデータを選択する必要があります。

import pandas as pd
import datetime
import matplotlib.dates as mdates
from matplotlib.ticker import FormatStrFormatter
from pandas.plotting import register_matplotlib_converters


df = pd.read_csv("habr.csv", sep=',', encoding='utf-8', error_bad_lines=True, quotechar='"', comment='#')
dates = pd.to_datetime(df['datetime'], format='%Y-%m-%dT%H:%MZ')
df['datetime'] = dates
year = 2019
df = df[(df['datetime'] >= pd.Timestamp(datetime.date(year, 1, 1))) & (df['datetime'] < pd.Timestamp(datetime.date(year+1, 1, 1)))]

print(df.shape)

執筆時点で今年は（まだ完成していませんが）12715件の記事が公開されていることがわかりました。比較のため、2018年から15904年の全体について。一般的に、これは1日あたり約43の記事です（そして、これは肯定的な評価がある場合にのみ、ダウンロードされた記事のうち、否定または削除された記事の数です。識別子）。

データセットから必要なフィールドを選択します。指標として、ビューの数、コメント、評価値、および追加されたブックマークの数を使用します。

def to_float(s):
    # "bookmarks:22" => 22.0
    num = ''.join(i for i in s if i.isdigit())
    return float(num)

def to_int(s):
    # "bookmarks:22" => 22
    num = ''.join(i for i in s if i.isdigit())
    return int(num)

def to_date(dt):
    return dt.date() 

date = dates.map(to_date, na_action=None)
views = df["views"].map(to_int, na_action=None)
bookmarks = df["bookmarks"].map(to_int, na_action=None)
votes = df["votes"].map(to_float, na_action=None)
votes_up = df["up"].map(to_float, na_action=None)
votes_down = df["down"].map(to_float, na_action=None)
comments = df["comments"].map(to_int, na_action=None)

df['date'] = date
df['views'] = views
df['votes'] = votes
df['bookmarks'] = bookmarks
df['up'] = votes_up
df['down'] = votes_down

これでデータがデータセットに追加され、使用できるようになりました。データを日ごとにグループ化し、平均値を取ります。

結果は面白いです。ご覧のように、Habrは年間を通じてわずかに「ソーセージ」でした。理由はわかりません。

ご覧のとおり、年間の平均再生回数はわずかに減少しています。これは、新しい記事が検索エンジンによってまだ索引付けされておらず、頻繁に発見されないという事実によって説明できます。しかし、記事ごとの平均評価の低下はより理解しにくいです。読者は、あまり多くの記事を閲覧する時間がないか、評価に注意を払っていないかのどちらかだと感じています。著者の報酬プログラムの観点からは、この傾向は非常に不愉快です。

ちなみに、2018年はそうではなく、スケジュールも多かれ少なかれ均一です。



一般に、リソースの所有者には、考慮すべき点があります。

しかし、悲しいことについては話そうではありません。一般に、Habrは夏の変化を「生き残った」という点で非常に成功し、サイト上の記事の数は減少しなかったと言えます。

結論


評価を作成するときに、興味深いと思われる2つの点に注意を向けました。

まず、結局のところ、トップの60％はgeektimesジャンルの記事です。来年それらが少なくなるかどうか、そしてビール、宇宙、医学などに関する記事なしでHabrがどのように見えるか-私は知りません。読者は間違いなく何かを失うでしょう。様子を見よう。

次に、ブックマークのトップが予想外に高品質であることがわかりました。これは心理的に理解可能であり、読者は評価に注意を払わない場合があり、記事が必要な場合はブックマークに追加します。そして、ここに有用で深刻な記事が最も集中しています。サイトの所有者は、ブックマークの数とインセンティブプログラムの関係を検討する必要があると思います。この場合、Habréでこの特定のカテゴリの記事を増やしたいと考えています。
