MSI / 55-中央店の支店で商品を注文するための古いターミナル



KDPVに表示されているデバイスは、支店から中央店に注文を自動的に送信するためのものでした。これを行うには、事前に注文した商品を入力して、中央店の番号に電話し、音響的に結合されたモデムの原理に従ってデータを送信します。端末がデータを送信する速度は、300ボーと推定されます。それは4つの水銀亜鉛要素（それが可能だった）によって給電され、そのような要素の電圧は1.35 Vであり、バッテリー全体は5.4 Vなので、すべてが5 V PSUで動作しました。スイッチを使用して3つのモードを選択できます。CALC-通常の計算機、OPER-数字やその他の文字を入力できます。SEND-送信しますが、最初は音が出ませんでした。記事を保存して送信できるのは明らかですが、どうやって？あなたが見つけることができる場合著者は音を分析しようとしますこのプログラム、あるいはなんとかして端末をアマチュア通信のデジタルタイプに適合させます。

デバイスは裏側にあり、ダイナミックヘッドとバッテリーコンパートメントが表示されます



。最も重要なことは、ターミナルから音を絞り出す方法です-著者は、同じターミナルをかつて持っていた人から学びました。初期化コードを入力する必要があり、それから記事を入力できます。スイッチをOPERの位置に置くと、Pの文字が表示されます。文字Hが表示されます。001290（これはおそらくパスワードです）を入力し、もう一度ENTを押します。番号0が表示され、部品番号を入力できます。

HまたはPの文字で記事を開始する必要があります（ここでは著者が誤っていました。キーボードにPの文字はなく、Fがあります）。その後、数字が続きます。 ENTキーを押すと、タイプ0004 0451の行が表示されます。後続の各記事では、最初の数が増加し、2番目の数が減少します。これは、それぞれ占有セルと空きセルの数であることを意味します。入力した記事は矢印ボタンでスクロールできますが、作成者はそれらを削除する方法を知りません（つまり、CLRキーが役に立たなかったことを意味します）。各商品の数量の表記方法については、記載していません。

記事を入力したら、スイッチをSEND位置に回してSND / =キーを押す必要があります。 SEND BUSYというメッセージが表示され、送信が開始されます。



4.4秒間、1200 Hzの周波数のトーンが鳴ります。その後、さらに6秒間-1000 Hz。次の2.8秒は変調信号の送信に行き、その後さらに3秒-再び1000 Hzのトーンを送信します。

実際にスペクトルを詳しく見ると、1000 Hzではなく980で、1200-1180ではありません。作成者はWAVファイルを書き留め、上記のプログラム（ここでは「man」）をインストールして、次のように実行しました。

minimodem -r -f msi55_bell103_3.wav -M 980 -S 1180 300

起こりました：

### CARRIER 300 @ 1000.0 Hz ###
 H00  90+ H00  90+ H00  90+ H  3 56  + Ʊ 3 56  +  9  + ƴ56+ H963 5   + 
### NOCARRIER ndata=74 confidence=2.026 ampl=0.147 bps=294.55 (1.8% slow) ###

これは、ベル103変調に似ています。一般的には1070と1270 Hzですが。

しかし、周波数はターミナルで「逃亡」しませんでしたか？作成者は速度が1.8％増加するようにWAVファイルを編集しました。それはほぼ正確に1000と1200であることが判明しました。プログラムの新たな開始：

minimodem -r -f msi55_bell103_4.wav -M 1000 -S 1200 300 -R 8000 -8 --startbits 1 --stopbits 1

そして彼女は答えた：

### CARRIER 300 @ 1000.0 Hz ###
 H00  90+ H00  90+ H00  90+ H  3 56  + Ʊ 3 56  +  9  + ƴ56+ H963 5   + 
### NOCARRIER ndata=74 confidence=2.090 ampl=0.148 bps=299.50 (0.2% slow) ###

どちらの場合も、エラーにもかかわらず、結果には意味的な負荷がかかります。記事H12345678は、信号から「引き出された」ものH 3 56 です。PSUのフィルタリングが不十分である可能性があるため、50 Hzのバックグラウンドが信号に重畳されます。プログラムは、低い値の信頼度（confidence = 2.090）を報告します。これは、歪んだ信号を示します。しかし、現在は、端末がまだ存在していたときに、端末が中央店のコンピューターにデータを送信した方法は少なくとも理解できます。
