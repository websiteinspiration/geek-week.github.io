Rustプログラマの目を通す：第一印象

コースの開始を見越して、「Golang Developer」は興味深い資料の翻訳を準備しました。囲碁の第一印象は？



過去数週間、私は仕事でGoを使用することができました。私は最初に多かれ少なかれ大規模で深刻なプロジェクトでGoを使用しました。その前に、Rustの機能を調査しながら、Goについての多くを読んだり、例や小さなプログラムを練習したりしましたが、実際のプログラミングはまったく別の問題です。

私の印象について聞いてみませんか？私はルストとの比較で行き詰まらないように努めますが、これは私の主な言語なので、避けられません。ラストへの強い偏見を事前に警告する必要がありますが、私は客観的になるために最善を尽くします。

一般的な印象


囲碁プログラミングは素晴らしいです。ライブラリには、必要以上の欠陥がなく、必要なものがすべて揃っていました。学習も楽しい経験でした-それはよく考え抜かれた実用的な言語です。例として、構文を学習すると、他の言語の多くのイディオムがGoに転送されることがわかります。 Goパーツを習得したら、その機能を簡単に予測できます。他の言語でのある程度の経験があれば、特別なGoogleの助けがなくてもGoコードを読んで理解することができました。

C / C ++、Java、Pythonなどを使用するよりもフラストレーションが少なく、生産性がはるかに高いことに気付きました。しかし、Goはまだこの世代の言語の一部として感じられています。彼は彼らから学んできました、そして私はこれがおそらくこの世代の最高の言語だと思います。彼は間違いなくこの世代の一部です。これは、根本的に新しいものよりも段階的な改善である可能性が高いです（これは価値の判断ではないことに注意してください-増分は、ソフトウェア開発の世界ではしばしば有益です）。これの良い例はnilです：RustやSwiftのような言語はnullパラダイムを取り除き、それによってクラス全体のエラーを排除します。Goは危険性を減らします。null値はありません。nilと0の区切り。ただし、基本的な考え方はまだ存在します。また、nullポインターを逆参照することによる広範なランタイムエラーもあります。

開発のしやすさ


Goは非常に簡単に習得できます。これは宣伝のスローガンであることは知っていますが、生産性のレベルにすばやく到達できたことに非常に驚きました。ドキュメント、ツール、および言語自体のおかげで、わずか2日で、有益でコミットに適したコードを書き始めました。

学習に有利ないくつかの要因：

  • 小さくして。多くの言語は小さくしようとしますが、Goは実際には1つです。（これはほとんど良いです、そして私はこれに必要とされた規律に感銘を受けました）。
  • 標準ライブラリは良いです（そしてまた小さいです）。エコシステムでライブラリを見つけて使用するのは非常に簡単です。
  • 他の言語にはない言語がほとんどありません。Goは他の確立された言語から多くのビットを継承し、それらを磨き、きれいに接続します。彼は目新しさを慎重に避けている。


コードルーチン


Goコードは非常に迅速に繰り返されます。繰り返しを減らすためのマクロやジェネリックなどのメカニズムはありません（インターフェイスは抽象化には適していますが、コードの重複を減らすにはあまり適していません）。型を除いて同一の関数を多数蓄積することがよくあります。

エラー処理も繰り返しの一因となります。多くの機能にはif err != nil { return err }、元のコードよりも多くのテンプレートがあります。

コードのルーチンを削減するためのジェネリックまたはマクロの使用は、その読みやすさとのトレードオフとして批判されることがあります。でも囲碁の場合は賛成できません。コードのコピーと貼り付けはすばやく簡単ですが、Goコードのほとんどを無視するか、微妙な違いを探す必要があるため、Goコードを読むのは面倒です。

コヒーレンス


言語開発者およびプログラマーとして、Goは組み込みのものとユーザーが利用できるものとの間で不一致に遭遇することが多いという事実におそらく最も驚いたでしょう。多くの言語のタスクは、可能な限り多くの魔法を払いのけ、ユーザーが組み込み関数を利用できるようにすることです。演算子のオーバーロードは、単純ですが物議を醸す例です。Goにはたくさんの魔法があります！そして、組み込みのことができることを実行できないという壁に非常に簡単にぶつかります。

特に際立ついくつかのポイント：

Goはシンプルでコンパクトな楽しい言語です。いくつかの鋭い角がありますが、ほとんどがうまく設計されています。彼は非常に素早く習得でき、他の言語では知られていない機能を回避します。

Rustと比較すると、Goは完全に異なる言語です。どちらもシステム言語またはCの「置き換え」として大まかに説明できますが、目的やアプリケーション、言語設計スタイル、優先順位が異なります。ガベージコレクションは本当に大きな違いです。 GoにGCがあると、言語がはるかにシンプルで小さくなり、理解しやすくなります。 RustにはGCがないため、非常に高速になり（特に、高帯域幅だけでなく明確な遅延が必要な場合）、Goでは不可能である機能とプログラミングパターンを提供します（少なくともパフォーマンスを犠牲にすることはありません）。

Goは、ランタイムが適切に実装されたコンパイル済み言語です。彼は速いです。Rustもコンパイルされていますが、実行時間ははるかに短くなっています。彼はとても速い。他に制限がないと仮定すると、GoとRustのどちらを使用するかを選択することは、はるかに短い学習曲線とより単純なプログラム（開発の高速化を意味する）と、Rustの側では、より高速で表現力豊かなシステムとの妥協点だと思いますタイプ（プログラムをより安全にし、デバッグとトラブルシューティングを高速化します）。
