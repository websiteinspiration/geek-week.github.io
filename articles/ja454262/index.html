Daskは、誇張もなく、私が出会った中で最も革新的なデータ処理ツールです。パンダとナンピーが好きならでも、RAMに収まらないデータに対応できない場合は、Daskがまさに必要なものです。 DaskはPandasデータフレームとNumpyデータ構造（配列）をサポートしています。 Daskはローカルコンピューターで実行することも、スケーリングして実行し、クラスターで実行することもできます。本質的には、コードを1回だけ記述し、それをローカルマシンで使用するか、それとも最も一般的なPython構文を使用して多くのノードのクラスターにコードをデプロイするかを選択します。機能自体は素晴らしいですが、強調するためにこの記事を書くことにしました。すべてのデータサイエンティスト（少なくともPythonを使用）はDaskを使用する必要があります。私の見解では、Daskの魔法は、コードを最小化することで、すでに利用可能な計算能力を使用してコードを並列化できることです。たとえば、私のラップトップで。並列データ処理を使用すると、プログラムの実行が高速になり、待機時間が短くなるため、分析に残される時間が長くなります。特に、この記事では、オブジェクトについて説明しますdask.delayed データサイエンスタスクのストリームにどのように適合するか。

Daskの紹介


Daskの概要として、完全に控えめで自然な構文の概念を説明するために、いくつかの例を紹介します。このケースで私が提案したい最も重要な結論は、あなたがすでに持っている知識で十分に機能するということです。HadoopやSparkなどの新しいビッグデータツールを習得する必要はありません。

Daskは、RAMのサイズを超えるデータ（データフレーム、バッグ、配列）を格納できる3つの並列コレクションを提供します。これらの各種類のコレクションでは、RAMとハードディスク間でデータをセグメント化してデータを格納し、クラスター内の複数のノードにデータを分散できます。

Dask DataFrameは、Pandasのデータフレームなどの細断データフレームで構成されているため、Pandasクエリ構文の機能のサブセットを使用できます。以下は、2018年のすべてのcsvファイルをダウンロードし、タイムスタンプを使用してフィールドを解析し、Pandasリクエストを起動するコードの例です。

import dask.dataframe as dd

df = dd.read_csv('logs/2018-*.*.csv', parse_dates=['timestamp'])
df.groupby(df.timestamp.dt.hour).value.mean().compute()

Bagでは、メモリに収まらないpythonicオブジェクトのコレクションを保存および処理できます。Dask Bagは、json形式のログおよびドキュメントのコレクションを処理するのに最適です。このコードの例では、2018のすべてのjsonファイルがDask Bagデータ構造にロードされ、各jsonレコードが解析され、ラムダ関数を使用してユーザーデータがフィルター処理されます。

import dask.bag as db
import json

records = db.read_text('data/2018-*-*.json').map(json.loads)
records.filter(lambda d: d['username'] == 'Aneesha').pluck('id').frequencies()
