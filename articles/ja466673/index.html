Cisco 200-125 CCNA v3.0のトレーニング。46日目：OSPFパフォーマンスチェック

今日は、ICND2トピックのセクション2.4で説明されている、OSPFプロトコルの設定とテストに関する前のレッスンのトピックを続けます。プロトコルのテストは非常に重要であるため、このために使用する必要があるコマンドを知る必要があります。このレッスンを始める前に、前のビデオで間違えたことをお詫び申し上げます。 DRとBDRの選択については、異なるルーターのプロセスIDプロセスIDが一致するケースについて述べました。したがって、プロセス番号は一致しませんが、ルーターのOSPFインターフェースの優先順位が一致します。この場合、DRの選択は、ルーターIDルーターの識別子の比較に基づいています。



前の動画を書き直す時間がないので、できるだけ早くコース全体を修了しようと急いでいるので、間違いをもう一度お詫びします。エラーが発生した場合は、通常、ビデオを交換します。ただ、このレッスンが公開される直前の動画を見ていたので、レッスン「45日目」が公開されてから2ヶ月が経過したため、エラーに気づきませんでした。残念ながら、コメントでこのエラーを指摘した人はいません。

したがって、私は繰り返します-DRを選択するとき、最初に、すべてのデバイスのOSPFインターフェイスの優先度がチェックされ、Ciscoのデフォルト値はルーター優先度= 1であり、可能な最大優先度値は255です。インターフェイスの優先度が同じである場合、ルーターはRIDを比較して役割を選択しますルーターIDが大きい方のDR。

5つのOSPF検証コマンドを見てみましょう。これらの各コマンドをチェックすることが正確に何が可能で、どのコマンドでルーティング情報を表示できるかを知っている必要があります。



入力したコマンドに応答してシステムが提供するすべての情報が必要ではない可能性がありますが、主な内容を理解する必要があります。最初のテストコマンドはshow ip protocolです。ルータID、設定されたゾーン番号、ネットワークコマンド、ルート情報ソースなどの重要なデータを表示できます。



それでは、Packet Tracerに移って、前のビデオと同じネットワークトポロジーについて考えてみましょう。 R1ルーターの設定に入り、show ip protocolsコマンドを入力します。



応答として、システムはそのような情報を提供します：ルーティングプロトコル-OSPF 1、ルーターIDルーターID 1.1.2.1、このルーターのゾーン数-1以下は、IPアドレス、リバースマスク、ゾーン番号を持つネットワークルートに関する情報です。



パケットトレーサーを起動したばかりで、ネイバー関係がまだ確立されているため、ルートデータを提供したソースに関する情報には、2つのデバイスのみが含まれています。これは、ルーター3および4のOSPFプロセスレポートから確認できます。システムは、現在、ロードデータベースの更新から完全への移行が進行中であることを示しています。 show ip protocolコマンドをもう一度入力すると、さらに多くのデータが表示されます。これで、ルーターIDを持つ5つのルーターすべてがルーティング情報のソースとして表示されます。



次のテストコマンドはshow ip ospfです。これにより、ルーターの識別子とプロセスの識別子、ゾーンの数、OSPFルーター間で認証が行われるかどうか、および実行されたSPFアルゴリズムの数を確認できます。



R1の設定に入り、このコマンドを入力します。ご覧のとおり、ここでも、前のコマンドを適用した後でも、ルーター識別子1.1.2.1が再び表示され、ルーターがゼロBACKBONEゾーンにあることが示されています。



さらに、このゾーンのルーターに接続されているインターフェイスの数は4であり、実行されるSPFアルゴリズムの数は3であると書かれています。このアルゴリズムは、ネットワークトポロジが変更されるたびに実行され、ルーティングテーブルを更新します。また、このゾーンには認証がないことも示しています。

次のルーティングチェックコマンドはshow ip ospf interfacesです。



このコマンドを使用すると、インターフェイスの物理状態の表示、IPアドレス設定の表示、ネットワークのタイプとルーターIDの表示、ブロードキャストネットワークの専用およびバックアップDRおよびBDRルーターに関する情報の提供、タイマーと隣接情報の構成隣接情報を構成できます。それがPacket Tracerでどのように見えるか見てみましょう。



ループバックインターフェイスに関する情報（アドレス、ゾーン番号、コスト）が表示されます。これは、物理インターフェイスの動作をエミュレートできる仮想インターフェイスです。この場合、このようなインターフェイスを2つ起動しました。ネットワークタイプはループバックです。次に、IPアドレス192.168.1.1のギガビットイーサネットインターフェイスが表示されます。これはブロードキャストネットワークであり、インターフェイスのコストも1です。このデバイスのステータスはDROTHERで、専用のDRルーターはRID 4.4.4.4とIPアドレス192.168.1.4を持っています。バックアップ専用ルーターは3.3.3.3、IPアドレスは192.168.1.3です。

以下は、ハロータイマーの値です。10秒、デッドタイマーの「デッド」タイマーは40秒です。さらに、ネイバーは3つしかなく、そのうち2つは完全な隣接関係が確立されていることが示されています。これらは4.4.4.4と3.3.3.3です。



ルータ2.2.2.2とDROTHER関係が確立されています。 R5ルーターとのシリアル接続はポイントツーポイント接続であり、DRまたはBDRは存在できません。インターフェイスのコストは64で、タイマーに関する情報は前のものと同様です。ポイントツーポイント接続があるため、ルーターが1つのデバイスのみと線形プロトコル近隣関係を確立していることがさらに示されています。

次のテストコマンドshow ip ospf neighborは、システムにこのデバイスのネイバーを表示するように要求します。つまり、ネイバーテーブルを表示します。



これにより、隣接ルーターのRID、その優先度、隣接、ゲートウェイインターフェイスのIPアドレス、デッドタイマーの値（応答しない隣接デバイスが「デッド」と見なされるまでの時間）、およびインターフェイスに関する情報を確認できます。 R1ルーターのCLIコンソールでこのコマンドを入力すると、ルーターの4つのネイバーが表示されます。これらはルーター3、2、4、5です。



各ネイバーについて、その優先度、ネイバー関係の状態、デッドタイマー、IPアドレス、およびインターフェイスのタイプが表示されます。最初の3つのルーターはブロードキャストネットワーク内にあるため、2つはFULL状態で、1つは2WAY状態です。 R5ルーターはポイントツーポイントネットワークの一部であるため、スラッシュの後の情報はありません。スラッシュの後に何も指定されていない場合、これはp2p接続です。ブロードキャストネットワークの場合、スラッシュの後にBDR、DR、またはDROTHERの3つの値のいずれかを指定します。

ルータ3.3.3.3はBDRの役割を果たします。これは、物理インターフェイスGigabitEthernet0 / 0経由でIPアドレス192.168.1.3にアクセスすることで実現できます。シリアルインターフェイスSerial0 / 1/0を介して、アドレス10.1.1.5のルータ5.5.5.5に接続できます。これは、隣人のテーブルがどのように見えるかです。

次のテストコマンドはshow ip ospf databaseです。これにより、OSPFデータベースを表示できます。



このコマンドをPacket Tracerに入力すると、ルーターのLSAレコード、つまりOSPFルーティングデータベースを表示できます。



テーブルの最初にルーターIDとプロセスIDがあり、ゼロゾーンに属するチャネルのリストが表示されます。これはLSAタイプ1のリストです。LSAのタイプはまだ検討していないため、これは単純なステートメントに限定されています。以下は、単一のゼロゾーンにあるすべてのOSPFルーターです。これらの各デバイスには、同じLSDBが必要です。 5つのルーターすべてを含む必要があり、この情報に基づいて、各ルーターはSPFアルゴリズムを開始して独自のルーティングテーブルを形成します。このようなデータは、チャネルステータスデータベースに含まれている必要があります。

そこで、5つのテストチームを調査し、レッスンを完了する前に、もう1つ説明します。



これは部分的に構成されたネットワークのトポロジです。このスキームをこのビデオの下にあるリンクから無料でダウンロードして、プログラムでパケットトレーサーを使用して実際的な問題を解決できます。 OSPFを部分的に設定しましたが、PC0からPC1、PC2、およびPC3にpingを実行する必要があり、PC2とPC3が相互にpingできる条件も作成します。

これを行うには、各ルーターの設定に移動して、これが発生しない理由を確認する必要があります。これらの問題を解決し、pingが指定されたルートに沿っていることを確認したら、このビデオに戻ってコメントと解決策の説明を残してください。
