アーケードリバースエンジニアリング：NBAジャムでマイケルジョーダンを記録する


去年の夏、サニーベールのパーティーに招待されました。ガレージの所有者は、4人のプレーヤー用のアーケードマシンNBA JAMトーナメントエディションを持っていることがわかりました。ゲームは25年以上前（1993年にリリースされました）であるという事実にもかかわらず、特に熱狂的なファンにとって、それをプレイすることは依然として非常に興味深いです。

マイケルジョーダンを含まないシカゴブルズの選手リストに驚いた。情報筋によると[1]、MJは独自のライセンスを取得し、ミッドウェイがNBAと行った契約の一部ではなかった。

マシンの所有者に尋ねたところ、ハッカーがSNES "NBA Jam 2K17"のmodをリリースし、新しいプレイヤーとMJがプレイできることを知りましたが、アーケードバージョンがどのように機能するか分析していませんでした。したがって、私は間違いなく中を見なければなりませんでした。

バックグラウンド


NBAジャムの物語はバスケットボールからではなく、ジャン=クロードヴァンダムから始まります。ユニバーサルソルジャーがリリースされたのとほぼ同時期に、ミッドウェイゲームズは、実際の俳優との類似性を保持する、デジタル化されたフォトリアリスティックな大きなスプライトを操作する技術を開発しました。毎秒60フレームのアニメーション、サイズが100x100ピクセルの前例のないスプライトであり、それぞれに独自の256色のパレットがありました。

同社はこのテクノロジーを使用して、人気のシューティングゲーム「ターミネーター2：ジャッジメントデイ」[2]で大成功を収めましたが、「ユニバーサルソルジャー」のライセンスを取得できませんでした（JCVDの財務状況はミッドウェイでは受け入れられませんでした[3]。）交渉が失敗に終わったとき、ミッドウェイは方向を変え、1991年のカプコンメガヒットファイティングゲーム、ストリートファイターII：ワールドウォリアーの開発を始めました。

4人のチームが編成されました（Ed Booneがコードを書き、John Tobiasがアートとスクリプトに取り組み、John Vogelがグラフィックを描き、Dan Fordenがサウンドエンジニアでした）。1年のハードワーク[4]の後、ミッドウェイは1992年にMortal Kombatを立ち上げました。

ビジュアルスタイルは通常のピクセルアートとは非常に異なり、ゲームデザインは、少なくとも「物議を醸す」ものでした。画面に何リットルもの血が流れ、めちゃくちゃ残酷な押し込みをするゲーム-「致命的」はすぐに世界的なヒットとなり、年間で約10億ドルを稼いだ[5]。


SF2：384×224、4,096色。


MK：32,768色で400×254。

興味深い事実： PCのVGAモード0x13のように、これらのゲームではピクセルは正方形ではありませんでした。Mortal Kombatフレームバッファーのサイズは400×254ですが、4：3 CRT画面比に拡大され、400×300の解像度を提供します[6]。

ミッドウェイTユニット機器


MidwayがMortal Kombatのために開発したハードウェアは非常に優れていることがわかりました。彼は自分にT-Unitという名前を付けられ、他のゲームで再利用できるほど良かったです。

  • モータルコンバット。
  • モータルコンバットII。
  • NBAジャム。
  • NBAジャムトーナメントエディション。
  • Dredd裁判官（釈放されていない）。

T-Unitは2つのボードで構成されています。それらのほとんどはゲームのロジックとグラフィックスを扱います。


NBA JAM TE Editionプロセッサボード（約40x40 cm、つまり15インチ）。


他のボードはそれほど複雑ではありませんが、多くの機能も備えています。オーディオ用に設計されていますが、FM合成を使用した音楽だけでなく、デジタルサウンドも再生できます。

サウンドカードは、電源と背面に取り付けられたグラフィックカードに接続されています。左上にある大きなラジエーターに注意してください。

これらの2つのボードには、200を超えるチップ、抵抗、EPROMが含まれています。シリアル番号に基づいてのみこれをすべて理解することは非常に時間がかかります。しかし、驚くべきことに、90年代以降のデバイスにあるドキュメントが誤って発見されることがあります。そして、NBAジャムの場合、彼女は素晴らしかった。

ミッドウェイTユニットアーキテクチャ


データを探していると、NBAジャムキットに出会いました。このドキュメントの詳細レベルは驚くべきものです[7]。とりわけ、EPROMやチップなど、配線接続の詳細な説明を見つけることができました。


ドキュメントからの情報により、ボードの図を描き、各部品の機能を決定することができました。コンポーネントの検索を支援するために、ボードには右下隅（UA0）を始点とする座標があり、左上隅（UJ26）まで増加します。


メインボードの中心は、周波数50 MHz、EPROMに1メビバイトコード、512 KBのDRAMを備えたTexas Instrument TMS34010（UB21）です[8]。 34010は16ビットバスを備えた32ビットチップで、PIXTやPIXBLTなどの優れたグラフィック命令を備えています[9]。 90年代前半には、このチップはいくつかのハードウェアアクセラレーションカード[10]で使用されており、かなりの量のグラフィック効果を処理できると思いました。驚いたことに、彼はゲームロジックのみを扱い、何も描画しません。

実際、「DMA2」と呼ばれるUE13チップはグラフィックモンスターであることが判明しました。ドキュメントの図によると、32ビットのデータバスと32ビットのアドレスバスが（当時は）印象的であり、ボード上で最大のチップとなっています。この特殊な集積回路（ASIC）は、以下で説明する多くのグラフィック操作が可能です。

すべてのチップ（システムRAM、GFX EPROM、パレットSDRAM、コード、ビデオバンク）は、1つの32ビットアドレス空間にマップされ、同じバスに接続されます。バスプロトコルについての情報が見つからなかったので、もしそれについて何か知っているなら、電子メールに書いてください。

トリックに注意してください。1つのEPROMコンポーネント（青色でマーク）を使用して、別のストレージシステムを作成します（コストを節約します）。これらの512 kb EPROMには、32ビットのアドレスピンと8ビットのデータピンがあります。 16ビットのデータバスを必要とする34010の場合、2つのEPROM（J12およびG12）がアドレスの2つの代替で接続され、1メビバイトのメモリを作成します。同様に、グラフィックリソースはアドレスの4倍の交番で接続され、8メビバイトを含む32ビットストレージシステムで32ビットアドレスを形成します。

この記事では主にグラフィックスパイプラインについて検討しますが、誘惑に抵抗することはできないため、オーディオシステムについて簡単に説明します。


サウンドカードの図は、Motorola 6809（周波数2 MHzのU4）を示しています。Motorola6809は、1つのEPROM（U3）から命令を受け取り、音楽と効果音を制御します。

ヤマハのFM合成チップ2151（3.5 MHz）は、6809から受け取った命令から直接音楽を生成します（音楽はかなり狭い帯域幅を使用します）。

OKI6295（1 MHz）は、ADPCM形式でデジタルオーディオを再生します（たとえば、伝説のTim Boitschakalalaka [11]）。

メインボードでは、デジタル化された音声を格納するためにアドレスのダブルインターリーブを備えた16ビットシステムで、同じ青い512 kバイトEPROM 32a / 8dが使用されますが、8ビット命令の場合、Motorola 6809のデータ/アドレスはインターリーブされません。

フレーム寿命


NBAジャム画面全体は、16ビットパレットで索引付けされます。色は、64 kバイトのパレットにxRGB 1555形式で保存されます。パレットは、512バイトの128ブロック（256 * 16ビット）に分割されます。EPROMに保存されているスプライトは「GFX」とマークされています。各スプライトには、最大256x16ビット色の独自のパレットがあります。スプライトは多くの場合、パレットのブロック全体を使用しますが、複数使用することはありません。CRT信号はRAMDACを使用してモニターに送信されます。RAMDACは、ピクセルごとにビデオDRAMバンクからインデックスを読み取り、パレットでカラー検索を実行します。

NBAジャムビデオの各フレームの寿命は、次のように進行します。

  1. ゲームロジックは、J12 / G12から34010に送信される16ビット命令のストリームで構成されます。
  2. 34010は、プレイヤー入力を読み取り、ゲームステータスを計算してから、画面を描画します。
  3. 34010 ( 34010 J12/G12).
  4. 34010 DMA2, , 8- , , , , .
  5. DMA2 8- GFX ROM J14-G23, 8- 16- . DRAM2 , 8- GFX EPROM 16-
  6. 3-5 , .
  7. 画面の更新に関して、RAMDACはビデオバンクのデータをCRTモニターが理解できる信号に変換します。帯域幅が16ビットインデックスを16ビットRGBに変換するのに十分であるため、パレットは非常に高価で非常に高速なSRAMに保存されます。


興味深い事実： EPROMフラッシュファームウェアは、それほど単純なプロセスではありません。チップに書き込む前に、その内容をすべて完全に消去する必要があります。

これを行うには、チップにUV光を照射する必要があります。まず、EPROMの上部からステッカーをはがしてスキームを開く必要があります。次に、EPROMは、UVランプがある特別な消しゴムデバイスに配置されます。

20分後、EPROMはゼロで埋められ、記録の準備が整います。

MAMEのドキュメント


機材を見つけたので、マイケルジョーダンに書き込むことができるEPROMのセットを理解しました（パレットはコードEPROMに格納され、インデックスはGFX EPROMに格納されています）。ただし、正確な場所や使用されている形式はまだわかりません。

MAMEで見つからないドキュメント。

この素晴らしいエミュレータの仕組みがわからない場合のために、簡単に説明します。MAMEは、ボードの模倣である「ドライバー」のコンセプトに基づいて構築されています。各ドライバは、各チップを（通常は）模倣するコンポーネントで構成されています。Midway T-Unitの場合、次のファイルに関心があります。

mame /含む/ midtunit.h
mame / src / mame /ビデオ/ midtunit.cpp
mame / src / mame /ドライバー/ midtunit.cpp
mame / src / mame / machine / midtunit.cpp
cpu / tms34010 / tms34010.h

元のスプライトをDMA2に転送する過程でそれらを保存できるデバッグ機能もあります（この機能は、MAMEプロジェクトの長い間参加者、Ryan Holtz [13]によって作成されました）。メタデータを含むすべてのファイルがディスクに保存されるように、ゲームをプレイするだけで十分でした。

スプライトは、圧縮なしの16ビットパレットの単純な要素で構成されていることがわかりました。ただし、すべてのスプライトに同じ数の色があるわけではありません。一部のスプライトは4ビットのカラーインデックスで16色のみを使用しますが、他のスプライトは256色を使用し、8ビットのカラーインデックスを必要とします。

パッチ


これでスプライトの場所とフォーマットがわかったので、最小限のリバースエンジニアリングを実行するだけです。私は、EPROMの「コード」と「gfx」の代替を排除するために、Golangで小さなプログラムを作成しました。プログラムの実行中にRAMがどのように見えるかを正確に処理したため、ストライピングを排除することで、ASCIIまたは既知の値を簡単に検索できます。

その後、プレーヤーの特性を簡単に見つけることができます。それらはすべて、16ビットの符号なしビッグエンディアン形式（34010はビッグエンディアンで動作するため、非常に論理的です）で次々に格納されていることがわかりました。プレイヤーの属性を変更するパッチを追加しました。バスケットボールにはあま​​り熱心ではないので、SPEED = 9、3 PTS = 9、DUNKS = 9、PASS = 9、POWER = 9、STEAL = 9、BLOCK = 9、CLTCH = 9と入力しました。

また、新しいスプライトでゲームにパッチを適用するためのコードも書きました。制限は1つだけです。新しいスプライトは、置き換え可能なスプライトと同じサイズにする必要があります。MJの写真として、256色のインデックス付きPNGを作成しました（ここで確認できます）。

最後に、中間形式をインターリーブ形式に変換して個々のEPROMファイルに書き込むコードを追加しました。

ゲームを始める



EPROMの内容にパッチを適用した後、NBAJam診断ツールは、一部のチップの内容が「BAD」としてマークされていることを示しました。EPROMの内容にパッチを当てただけで、CRC形式やその保存場所を探す手間を省いたため、これは予想されていました。

GFX EPROMは赤でマークされています（UG16 / UJ16、UG17 / UJ17、UG18 / UJ18、UG20 / UJ20、UG22 / UJ22およびUG23 / UJ23）。変更した画像が含まれているためです。パレットがあるため、命令（UG12とUJ12）が格納されている2つのEPROMも赤です。

さいわい、ここではCRCは変更されたコンテンツからの保護には使用されず、チップの完全性を検証するためにのみ必要です。ゲームが始まりました。そして獲得！
