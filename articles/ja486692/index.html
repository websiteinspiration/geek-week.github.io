使用したことのないChromeコンソールの機能

Chromeデベロッパーツールコンソールは、おそらく最も広く使用されており、最も便利な専用ブラウザツールの1つです。コンソールはプログラマーに多くの興味深い機能を提供します。ページコードのデバッグ、プロファイリング、監視に役立ちます。 今日翻訳している資料は、Chromeコンソールの機能のうち、それほど広く知られているわけではない機能について説明することを目的としています。





モニター機能


この機能はmonitor、他の機能の監視を整理するために使用されます。特に、特定の関数がいつ呼び出されたか、またどのパラメーターを使用して呼び出されたかを確認できます。例を考えます（そのコードはChromeコンソールに入力されます）：

function traceFunc (arg) { }
monitor(traceFunc)

関数は引数としてtraceFunc関数に渡されmonitorます。これで、関数traceFuncが呼び出されると、これに関する通知が（再び、コンソールで）受信されます。

traceFunc(90)
function traceFunc was called with arguments: 90

アンモニター機能


この機能unmonitorを使用すると、以前にその機能を使用して監視されていた機能の監視を無効にすることができますmonitor。関数の監視を停止するにはtraceFunc、次の構造を使用します。

unmonitor(traceFunc)

この後に呼び出した場合traceFunc、コンソールに通知は表示されません。

traceFunc(90)


監視が無効になっている関数を呼び出しても、この呼び出しに関する情報がコンソールに出力されない

UnmonitorEvents関数


関数をunmonitorEvents使用すると、関数によって開始されたオブジェクトイベントの監視をキャンセルできますmonitorEvents。
