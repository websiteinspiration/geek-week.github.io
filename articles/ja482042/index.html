実際の例でNewtonsoft.Jsonライブラリを操作します。パート2

記事の最初の部分では、動的に変化する構造を持つJSONオブジェクトを解析するメカニズムを検討しました。これらのオブジェクトは、newtonsoft.json.linq名前空間型にキャストされ、C言語構造に変換されました。最初の部分へのコメントでは、大部分が正当化された多くの批判がありました。第2部では、すべてのコメントと提案を考慮に入れます。



次に、データ変換の微調整のためのクラスの準備に焦点を当てますが、最初に、JSON解析自体に戻る必要があります。最初の部分で、newtonsoft.json.linq名前空間のJObjectクラスとJArrayクラスのParse（）メソッドとToObject <T>（）メソッドが使用されたことを思い出してください。

HttpClient httpClient = new HttpClient();
string request = "https://api.exmo.com/v1/trades/?pair=BTC_USD,ETH_USD";
HttpResponseMessage response = 
    (await httpClient.GetAsync(request)).EnsureSuccessStatusCode();
string responseBody = await response.Content.ReadAsStringAsync();

JObject jObject = JObject.Parse(responseBody);
Dictionary<string, List<Order>> dict = 
    jObject.ToObject<Dictionary<string, List<Order>>>();

JsonConvertクラスのnewtonsof.json名前空間に静的メソッドDeserializeObject <>があることに注意してください。これにより、オブジェクトとJSON表記の配列に対応するC＃構造に文字列を直接変換できます。

Dictionary<string, List<Order>> JsonObject = 
    JsonConvert.DeserializeObject<Dictionary<string, List<Order>>>(responseBody);
List<string> Json_Array = JsonConvert.DeserializeObject<List<string>>(responseBody);

将来的には、この特定の方法が記事で使用されるため、newtonsoft.jsonを使用してプログラムに追加する必要があります。

さらに、中間変換の数をさらに減らすことが可能です。Microsoft.AspNet.WebApi.Clientライブラリ（NuGetからも利用可能）をインストールした後、ReadAsAsyncメソッドを使用してストリームから直接データを解析できます。

Dictionary<string, List<Order>> JsonObject = await
    (await httpClient.GetAsync(request)).
    EnsureSuccessStatusCode().Content.
    ReadAsAsync<Dictionary<string, List<Order>>>();

変換のためのクラスの準備


Orderクラスに戻ります。

    class Order
    {
        public int trade_id { get; set; }
        public string type { get; set; }
        public double quantity { get; set; }
        public double price { get; set; }
        public double amount { get; set; }
        public int date { get; set; }

JSON C＃Class Generatorによって提案されたフォーマットに基づいて作成されたことを思い出させてください。このクラスのオブジェクトを操作するときに問題が発生する可能性がある2つの点があります。

1つ目-この形式では、クラスのプロパティがフィールドの命名規則に違反しています。さらに、型Orderのオブジェクトは、その識別子がOrderID（最初の部分の例で発生したtraid_idではなく）と呼ばれることを期待するのが論理的です。 JSON構造要素とクラスプロパティを任意の名前に関連付けるには、プロパティの前にJsonProperty属性を追加する必要があります。

    class Order
    {
        [JsonProperty("trade_id")]
        public int OrderID { get; set; }
        [JsonProperty("type")]
        public string Type { get; set; }
        [JsonProperty("quantity")]
        public double Quantity { get; set; }
        [JsonProperty("price")]
        public double Price { get; set; }
        [JsonProperty("amount")]
        public double Amount { get; set; }
        [JsonProperty("date")]
        public int Date { get; set; }
    }

その結果、「trade_id」要素に対応する値はOrderIDプロパティに、「type」は「type」などに記録されます。JsonProperty属性は、プライベート修飾子または静的修飾子を持つプロパティのシリアル化/逆シリアル化にも必要です-デフォルトでは、このようなプロパティは無視されます。
その結果、通貨ペアBTC_USDとETH_USDのすべてのOrderIDトランザクションのリストをコンパイルするためのコードは、次のようになります。

using HttpClient httpClient = new HttpClient();
string request = "https://api.exmo.com/v1/trades/?pair=BTC_USD,ETH_USD";
string responseBody = await
    (await httpClient.GetAsync(request)).
    EnsureSuccessStatusCode().
    Content.ReadAsStringAsync();

Dictionary<string, List<Order>> PairList = 
    JsonConvert.DeserializeObject<Dictionary<string, List<Order>>>(responseBody);

List<int> IDs = new List<int>();
foreach (var pair in PairList)
    foreach (var order in pair.Value)
        IDs.Add(order.OrderID);

このクラスで作業するときの2番目の問題は、Dateプロパティです。ご覧のとおり、JSON C＃クラスジェネレーターは「日付」要素を素数の整数として定義しました。ただし、クラスのDateプロパティに、日付用に特別に作成された型（DateTime）があれば、はるかに便利です。これを行う方法については、後で説明します。

日付を操作する機能


newtonsof.jsonのドキュメント にある記事の最初のフレーズは、日付の操作に関する説明があり、「JSONのDateTime-難しい」と大まかに翻訳できます。問題は、JSON仕様自体に、日付と時刻を記述するために使用する構文に関する情報が含まれていないことです。

JSON文字列の日付がテキスト形式で提示され、提示形式が「Microsoft」（現在は廃止されていると見なされている）、「JavaScript」（Unix時間）、ISO 8601バリアントのいずれかに対応している場合、すべてが比較的良好です。有効なフォーマットの例：

Dictionary<string, DateTime> d = 
    new Dictionary<string, DateTime> { { "date", DateTime.Now } };
string isoDate = JsonConvert.SerializeObject(d);
            // {"date":"2019-12-19T14:10:31.3708939+03:00"}

JsonSerializerSettings microsoftDateFormatSettings = new JsonSerializerSettings
{
    DateFormatHandling = DateFormatHandling.MicrosoftDateFormat
};
string microsoftDate = JsonConvert.SerializeObject(d, microsoftDateFormatSettings);
            // {"date":"\/Date(1576753831370+0300)\/"}

string javascriptDate = JsonConvert.SerializeObject(d, new JavaScriptDateTimeConverter());
            // {"date":new Date(1576753831370)}

ただし、Exmo交換の場合、すべてが少し複雑になります。交換APIの説明は、日付と時刻がUnix（JavaScript）形式で指定されていることを示しています。そして理論的には、OrderクラスのDateプロパティにJavaScriptDateTimeConverter（）クラスからのフォーマット変換関数を追加すると、DateTime型にキャストされた日付を取得する必要があります。

    class Order
    {
        [JsonProperty("trade_id")]
        public int OrderID { get; set; }
        [JsonProperty("type")]
        public string Type { get; set; }
        [JsonProperty("quantity")]
        public double Quantity { get; set; }
        [JsonProperty("price")]
        public double Price { get; set; }
        [JsonProperty("amount")]
        public double Amount { get; set; }
        [JsonProperty("date", ItemConverterType = typeof(JavaScriptDateTimeConverter))]
        public DateTime Date { get; set; }
    }

ただし、この場合、データをDateTime型の変数に解析しようとすると、記事の最初の部分で例外Newtonsoft.Json.JsonReaderExceptionがすでによく知られています。これは、JavaScriptDateTimeConverterクラスの変換関数が数値データをDateTime型に変換できないために発生します（これは文字列に対してのみ機能します）。



この状況を回避するには、独自のフォーマット変換クラスを作成する方法があります。実際、そのようなクラスはすでに存在し、Newtonsoft.Json.Converters名前空間を事前に接続することで使用できます（このクラスでは、逆関数はDateTimeからJSONに変換されないことに注意してください）。

    class UnixTimeToDatetimeConverter : DateTimeConverterBase
    {
        private static readonly DateTime _epoch = 
            new DateTime(1970, 1, 1, 0, 0, 0, DateTimeKind.Utc);


        public override void WriteJson(JsonWriter writer, object value, 
            JsonSerializer serializer)
        {
            throw new NotImplementedException();
        }

        public override object ReadJson(JsonReader reader, Type objectType, 
            object existingValue, JsonSerializer serializer)
        {
            if (reader.Value == null)
            {
                return null;
            }
            return _epoch.AddSeconds(Convert.ToDouble(reader.Value)).ToLocalTime();
        }
    }

関数をOrderクラスのDateプロパティに接続するだけです。これを行うには、JsonConverter属性を使用します。

    class Order
    {
        [JsonProperty("trade_id")]
        public int OrderID { get; set; }
        [JsonProperty("type")]
        public string Type { get; set; }
        [JsonProperty("quantity")]
        public double Quantity { get; set; }
        [JsonProperty("price")]
        public double Price { get; set; }
        [JsonProperty("amount")]
        public double Amount { get; set; }
        [JsonProperty("date")]
        [JsonConverter(typeof(UnixTimeToDatetimeConverter))]
        public DateTime Date { get; set; }
    }

これで、Dateプロパティの型はDateTimeになり、たとえば、過去10分間の取引のリストを作成できます。

以前は、取引所交換チームと協力していました。このコマンドは、次のフィールドを持つオブジェクトを返します。

public class BTCUSD

    public string quantity { get; set; }

交換コマンドuser_open_ordersは、非常によく似た構造を返します。

public class BTCUSD
{
    public string order_id { get; set; }
    public string created { get; set; }
    public string type { get; set; }
    public string pair { get; set; }
    public string quantity { get; set; }
    public string price { get; set; }
    public string amount { get; set; }
}

したがって、Orderクラスを調整して、tradeコマンドのデータだけでなく、user_open_ordersコマンドのデータも変換できるようにするのは理にかなっています。

違いは、通貨ペアの名前を含む新しいペア要素があり、trade_idがorder_idに置き換えられ（現在は文字列になっています）、日付が作成されて文字列になっていることです。

まず、order_idおよび作成したフィールドを、Orderクラスの対応するOrderIDおよびDateフィールドに保存する機能を追加します。これを行うには、解析のためにフィールド名が置き換えられるOrderDataContractResolverクラスを準備します（System.ReflectionおよびNewtonsoft.Json.Serialization名前空間が必要になります）。

  class OrderDataContractResolver : DefaultContractResolver
    {
        public static readonly OrderDataContractResolver Instance = 
            new OrderDataContractResolver();

        protected override JsonProperty CreateProperty(MemberInfo member, 
            MemberSerialization memberSerialization)
        {
            var property = base.CreateProperty(member, memberSerialization);
            if (property.DeclaringType == typeof(Order))
            {
                if (property.PropertyName.Equals("trade_id", 
                    StringComparison.OrdinalIgnoreCase))
                {
                    property.PropertyName = "order_id";
                }
                if (property.PropertyName.Equals("date", 
                    StringComparison.OrdinalIgnoreCase))
                {
                    property.PropertyName = "created";
                }
            }
            return property;
        }
    }

さらに、このクラスは、次のようにDeserializeObjectメソッドのパラメーターとして指定する必要があります。

Dictionary<string, List<Order>> PairList = 
    JsonConvert.DeserializeObject<Dictionary<string, List<Order>>>(responseBody,
        new JsonSerializerSettings { ContractResolver = 
        OrderDataContractResolver.Instance });

その結果、user_open_ordersコマンドへの応答として取得されるこのようなJSON構造：

{"BTC_USD":[{"order_id":"4722868563","created":"1577349229","type":"sell","pair":"BTC_USD","quantity":"0.002","price":"8362.2","amount":"16.7244"}]}

このようなデータ構造に変換されます。

foreach (var pair in PairList)
