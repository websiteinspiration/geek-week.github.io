Webジオサービス。最新のソリューションの概要



ほとんどすべてのオンライン製品の一部として、マップ、ジオコーディング、ジオポジショニングに関連するサービスの使用を必要とするタスクに対応できます。個人的には、オンライン不動産カタログの作成に10年近く携わっていますが、他の多くのプロジェクトの一環として、さまざまなオンラインジオサービスの特定の機能を使用した経験があります。

この記事では、プログラマーが最も頻繁に直面するジオタスクを調べ、これらの問題の解決策を提供するサービスを簡単にレビューし、これらのサービスを使用した私の経験を共有します。

私はコメンテーターの参加も期待しています-レビューで説明の不正確さ、優れたサービスの欠如、またはあなたがすでに書いたものに追加するものがあることに気付いた場合、コメントに書き込んでください、私は常に将来の手元に優れたツールのリストがあるように、メインテキストにすべてを収集しますすべての機会に。

サイトに地図を表示する


非常に人気のあるタスクは、サイトにオンラインマップを表示することです。たとえば、地図上のどこにオフィスがあるかを正確に示すには、または、広告の作成者が販売しているものを受け取ることができます。

マップを表示するには、2つの主要なタイプのAPIが使用されます。

  • JavaScript API（インタラクティブ）
  • 静的API（静的）

JavaScript API


私が開発したすべてのサービスは主にロシアで動作するため、私が使用していて人気のあるAPIについて説明します。私はbingや他のマッププロバイダーにAPIがあることを知っていますが、それらを使用しませんでした。

ユーザーにランダムな形式で住所を入力させ、この住所を構造化されたデータに変換する必要がある場合があります-国、地域、都市、通り、家の番号の名前を個別に。これは、不動産で機能するすべてのサービスに必要です。重複した住所を特定したり、メールの領収書に記入したりする必要があります。

原則として、上記のリストにあるすべてのジオコーディングサービスは、問題なくこのタスクに対処できます。直接ジオコーディングでは、座標に加えて、見つかったジオオブジェクトに関する構造化された情報も返します。

それとは別に、DaDataの「標準化」サービスに注目する価値があります。ロシアのアドレスには、さまざまな有用な情報が返されます。各アドレスに10 kopecks：
dadata.ru/api/clean/address

さらに、このセクションには、住所に関する追加情報を見つけるタスクも含まれています。特に、指定された住所が都市のどの地区に属するかの決定と、最寄りの地下鉄駅の検索は、住所で広告を検索する場合の重要なタスクです。

市内のエリアの定義


現在、市内のエリアを決定するための2つの解決策を見つけました。どちらもロシアでのみ機能します。

最初で最も明白なのはDaDataです。このサービスについては多くのことが書かれていますが、ロシアの住所を扱うには本当に良いサービスです（そしてこれは広告ではありません）。ご自分でお試しください。

サードパーティのサービスに依存したくない場合は、私が適用し、これまでに適用し続けている別のソリューションがあります。OKATOの住所コードによるエリアの定義です。ロシア国内の各住所には、独自のOKATOコード（11桁）があります。これらのうち、最初の5〜8は、都市によって異なりますが、明確に地域を示しています。

必要な都市については、地区のOKATOコードのデータベースをコンパイルしただけで、次のような配列が得られました。

OKATOコードのサンプル


次の質問は、OKATOアドレスコードを見つける方法です。 DaDataは、Tipおよび標準化サービスの一部としてそれを返します。同じdadatを使用するOKATOコードをアドレスに表示するサイトokato-oktmo.rfがあります。私たちの仕事は、サービスに関係なく、アドレスで独立してOKATOコードを見つけることです。

これを行うには、最新のFIASデータベースをダウンロードして、検索を設定する必要があります。MSSQLにFIASデータベースをインポートする方法の例を含む記事がすでにハブにあります。一般的に、FIASUによる検索を構成し、そこに必要な住所を見つけ、そのOKATOコードを調べ、地区を知っている地区のリストからOKATOコードを調べます-これで問題は解決しました。

60ギガバイトのXMLファイルをデータベースにダウンロードしてスチームバスを使いたくない場合は、FIASデータベースを操作できるdadataサービスよりも安価ですが、機能も劣ります：

近くの地下鉄駅を検索


住所に最も近い地下鉄駅を見つける必要がある場合、特にモスクワの居住者にとって必要な機能。現時点では、このタスクについて2つの解決策を知っています。

Yandexジオコーダー


まず、住所の座標を見つける必要があります。次に、リバースジオコーディングの場合と同様に、これらの座標をリクエストパラメータとして転送し、返されるtoponymの種類kind = metroを転送します。
別のパラメーターは、最寄りの駅を検索する必要があるエリアのサイズを設定します。経験的に、私は都市に応じて0.02から0.05度の値を選択しました（モスクワでは、メトロから2 kmの半径は依然として徒歩距離と見なされ、カザンでは駅間の距離は1.5 kmです）。

また、この方法には、座標の最も近いメトロが実際に必ずしも最も近いとは限らないという問題もあります（たとえば、川の反対側にあるなど）。以下の記事で説明するルートと距離マトリックスを作成するサービスの助けを借りてのみ、アクセシビリティの観点から最も近いメトロを決定できます。最寄りの駅の大まかな決定-Yandexジオコーダーは非常に適しており、1日あたり最大25,000件のリクエストを無料で実行できます。

DaData


はい。DaDataはこのタスクに役立ちます。「プロンプト」または「標準化」サービスの有償の関税の範囲内で、最寄りの3つの地下鉄駅とその距離も各住所に返されます。

座標によるユーザーの位置の特定


このタスクは、リバースジオコーディングのタスクと似ていますが、少し異なります。

ユーザーの座標が正確にわかっている場合は、ユーザーの場所を特定する必要がある場合があります（たとえば、デバイスのGPSセンサーまたはブラウザーのGeolocation APIを使用してユーザーの座標を取得しました）。ここには2つのオプションがあります。リバースジオコーディングは、ユーザーがいる場所の名前を返します。しかし、ユーザーが都市間の高速道路のどこかにいる、または郊外にいる、またはきれいなフィールドにいるだけで、このフィールドの土地の販売に関する広告を見たい場合はどうなりますか？常に逆ジオコーダーがこれに対応するわけではありません。

この場合、私は個人的にこれを行います-私は自分のデータベースに、広告があり、私たちが一般的に作業しているロシアの都市のすべての座標を持っています。そして、ユーザーの座標に応じて、単純な要求で、データベースからユーザーに最も近い都市を特定します。

ORDER BY (|lng - :lng| + |lat - :lat|) ASC LIMIT 1


ルート構築


ある地点から別の地点へのルートを構築する必要がある場合があります。たとえば、ユーザーの場所からオフィスまでのロケーションマップを作成します。YandexとGoogleの有料ソリューションがあります。主な違いは、Yandexがサブスクリプションを年払いで販売していることと、Googleがリクエストごとに請求を行い、その月の月に1回カードからお金を差し引くことです。
Googleにはプレミアムサブスクリプションもあります。これは、いくつかの追加機能（たとえば、静的APIイメージの最大サイズを増やす）と制限のデポジットを提供します。2GISからのソリューションもあります。1つのルートが構築されるという点で最も高価です。

ルートを構築するタスクは、いくつかのサブタスクに分かれています。
  1. ポイントからポイントへのルートを構築することは理解できます
  2. 距離のマトリックス-一連の出発点と到着点の間の移動中の距離と時間のマトリックスを作成できるサービス。たとえば、さまざまな倉庫からさまざまな顧客に商品を配送する必要があるいくつかの宅配便業者に最適なルートを計算する必要がある場合です。または、清掃サービスを提供していて、自宅から今日の各注文にアクセスするためにどのクリーナーがより便利であるかを見つけ、クリーナーを移動するための最適なスキームを作成する必要があります。
  3. Isochrone API-同時に到達できるポイントエリアを中心に構築します。たとえば、「ユーザーの位置から徒歩15分以内にすべてのカフェを検索します」。

プロバイダーが異なれば、これらすべてのタスクに対して異なるソリューションが提供されます。

結論


現代の世界では、解決する必要のあるほぼすべてのタスクの既製のサービスを見つけることができます。または、上記のリストのサービスの組み合わせを使用して解決することができます。唯一の問題は、このソリューションのコスト、それがどれだけあなたにとって有益かということです。

多くのサービスは、その人気が高まるにつれ、かつてないほどの収益化システムに切り替わります-Googleのかつては無料であったGoogleマップが完全に支払われたのと同じように、Yandexは徐々にこれに向かっています。

リストからすべての問題を解決するように見えるオープンなOpenStreetMapsがありますが、そのためには自分でデプロイする必要があり、プロジェクトに加えて、マップのインフラストラクチャ全体を個別に維持します-必要に応じて、全員が自分で決定します。

おそらく、レビューでは、あなたが使用しているいくつかの優れたサービスを示していないか、またはレビューからのサービスを使用していて、すでに書かれているものに追加するものがあります-コメントに書き込んで、テキストに追加します。
