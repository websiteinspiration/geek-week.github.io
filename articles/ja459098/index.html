GitHub Package RegistryはSwiftパッケージをサポートします

5月10日に、GitHub Package Registryの限定ベータテストを開始しました。これは、ソースコードの横にパブリックパッケージまたはプライベートパッケージを簡単に公開できるパッケージ管理サービスです。このサービスは現在、JavaScript（npm）、Java（Maven）、Ruby（RubyGems）、. NET（NuGet）、Dockerイメージなど、おなじみのパッケージ管理ツールをサポートしています。

GitHub Package RegistryにSwiftパッケージのサポートを追加することを発表いたします。 Swiftパッケージを使用すると、ライブラリやソースコードを独自のプロジェクトやSwiftコミュニティと簡単に共有できます。私たちは、Appleのメンバーと協力してこれに取り組みます。



このGitHubブログ記事

GitHubで入手可能な  Swift Package Managerは、Swiftコードをビルド、実行、テスト、およびパッケージ化するための単一のクロスプラットフォームツールです。構成はSwiftで記述されているため、目標設定、製品の宣言、パッケージの依存関係の管理が簡単になります。 Swift Package ManagerとGitHub Package Registryを組み合わせることで、Swiftパッケージの公開と管理が簡単になります。

モバイルデベロッパーにとって、生産性を高めるには、より優れたツールを用意することが重要です。 Swiftエコシステムの開発に伴い、Appleチームと協力してSwift開発者向けの新しいワークフローの作成を支援できることに興奮しています。

GitHub Package Registryのリリース以来、ツールの操作にコミュニティが強く関与してきました。ベータ期間中、私たちはコミュニティからPackage Registryがさまざまなニーズを満たし、それを改善するために何ができるかを学びたいと考えています。GitHub Package Registryをまだ試していない場合は、こちらからベータ版を申請でき  ます。
