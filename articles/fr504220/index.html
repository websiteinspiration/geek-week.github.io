Service de vérification d'en-tête HTTP du serveur

Pour tout site, il est important de configurer correctement les en-têtes HTTP. De nombreux articles ont été écrits sur le thème des titres. Ici, nous avons résumé les leçons apprises, la documentation RFC. Certaines rubriques sont obligatoires, certaines sont obsolètes, certaines peuvent être déroutantes et contradictoires. Nous avons effectué une analyse pour vérifier automatiquement les en-têtes HTTP du serveur Web . Contrairement à de nombreux autres services qui affichent simplement des en-têtes, ce service vous permet de:

  1. définir la valeur des en-têtes typiques;
  2. ajoutez vos propres en-têtes personnalisés;
  3. indiquer la version du protocole HTTP: 1.0, 1.1, 2 (vérifie si HTTP / 2 est pris en charge);
  4. spécifier la méthode de demande, le délai d'expiration et les données post-données à envoyer au serveur;
  5. puzomerka vérifie également si la réponse aux demandes If-Modified-Since, If-None-Match est correcte si la réponse du serveur contient Last-Modified ou ETag.

Nous ne prétendons pas être la vérité ultime. Pour le contenu individuel et pour les projets individuels, bien sûr, il peut y avoir des écarts. Mais ce service vous dira certainement à quoi vous devez faire attention, il peut vous être utile de modifier vos en-têtes. Ce qui suit est une liste de ce à quoi le service de vérification fait attention. Pourquoi donc, lisez les articles sur l'Habre .

En-têtes obligatoires


  • Date
  • Type de contenu spécifiant le jeu de caractères pour le contenu texte, de préférence utf-8
  • Compression de codage de contenu pour le contenu texte

En-têtes obsolètes et inutiles


  • Serveur avec une version détaillée du serveur Web
  • X-power-by
  • Version X_ASPNET
  • Expire
  • Pragma
  • P3p
  • Via
  • Compatible X-UA

En-têtes souhaités pour la sécurité


  • Options de type de contenu X
  • Protection X-XSS
  • Strict-transport-security
  • Referrer-policy
  • Politique de fonctionnalité
  • Content-Security-Policy ou Content-Security-Policy-Report-Only pour désactiver les scripts et les styles en ligne.

Mise en cache des en-têtes


Requis pour le contenu statique à long terme de cache et hautement souhaitable pour le contenu dynamique à court terme de cache.

  • Dernière modification
  • ETag
  • Contrôle du cache
  • Varier
  • Il est important que le serveur réponde correctement aux en-têtes: If-Modified-Since et If-None-Match

HTTP / 2


Maintenant, le serveur doit prendre en charge HTTP / 2. Par défaut, le service vérifie le serveur pour HTTP / 2. Si votre serveur ne prend pas en charge HTTP / 2, sélectionnez HTTP / 1.1.
