使用ObservableObjectが最善の解決策ではない理由


あなたはおそらく疑問に思っています：すでに実用的なソリューションがあるのに、なぜこれを他の方法で実装する必要があるのでしょうか？さて、あなたが私たちのアプリケーションの階層のロジックを見れば、それはあなたに明らかになるでしょう。 MotherViewは、ViewRouterインスタンスを初期化するルートビューです。 MotherViewでは、ContentViewAとContentViewBも初期化し、ViewRouterインスタンスをBindableObjectとして渡します。

ご覧のとおり、初期化されたObservableObjectをすべてのサブビューにダウンストリームで渡す厳密な階層に従う必要があります。これはそれほど重要ではありませんが、多くのビューを持つより複雑なアプリケーションを想像してください。すべてのサブビューやサブビューのすべてのサブビューへの初期化されたObservableルートビューの転送などを常に追跡する必要があります。これは、最終的にかなり退屈な作業になる可能性があります。



要約ObservableObjectすると、より複雑なアプリケーション階層に関しては、cleanを使用すると問題が発生する可能性があります。

代わりに、アプリケーションの起動時にViewRouterを初期化して、アプリケーションの階層に関係なく、すべてのビューをこのインスタンスに直接接続するか、または監視することができます。この場合、ViewRouterインスタンスは、アプリケーションのコードを飛ぶクラウドのように見え、すべてのビューは、ビュー階層の正しい初期化チェーンを気にすることなく自動的にアクセスします。
これは単なる仕事EnvironmentObjectです！

なにEnvironmentObject？


EnvironmentObject初期化後、アプリケーションのすべての表現とデータを交換できるデータモデルです。特に良いのはEnvironmentObject、提供することによって作成されたものObservableObjectであるため、私たちのものViewRouterを使用して作成できますEnvironmentObject！

すぐに我々は我々の宣言と同じくらいViewRouterとしてEnvironmentObject、全ての表現は通常のものと同じようにそれに接続することができObservableObjectますが、アプリケーションの階層ダウン初期化の連鎖を必要とせずに！

すでに述べたように、EnvironmentObject最初にアクセスするときにすでに初期化されているはずです。我々のでMotherView、ルートビューとして、プロパティを見てみましょうCurrentPage ViewRouter、我々はそれを初期化する必要があり、EnvironmentObjectアプリケーションの起動時。次に、currentPageを自動的に変更できますEnvironmentObjectの再レンダリングをContentView要求MotherViewします。

上記のように、アプリケーションを開始すると、すぐにルート表現と呼ばれるため、すぐViewRouterに高品質のインスタンスが提供されます。したがって、次のようにファイル内のシーン関数を更新します。EnvironmentObjectMotherViewEnvironmentObjectSceneDelegage.swift

func scene(_ scene: UIScene, willConnectTo session: UISceneSession, options connectionOptions: UIScene.ConnectionOptions) {

以来viewRouter、私たちのプロパティがされているContentView今、直接的に関連/初期インスタンス観測ViewRouterとしてEnvironmentObject私たちにそれらを初期化するために、我々はもはや必要性をMotherView。だから、更新してみましょうMotherView：

struct MotherView : View {
    
    @EnvironmentObject var viewRouter: ViewRouter
    
    var body: some View { 
        VStack {
            if viewRouter.currentPage == "page1" {
                ContentViewA()
            } else if viewRouter.currentPage == "page2" {
                ContentViewB()
            }
        }
    }
}

これはすばらしいことです。ViewRouter内部で初期化MotherViewしてインスタンスをContentViewに渡す必要がなくなるため、特により複雑な階層の場合に非常に効率的です。
