シスコの最新のルーターパッチについて知っておくべきこと

つい最近まで、ITの巨人がASR 9000システムに重大な脆弱性を発表していたので、カットの下で、バグの本質とパッチの適用方法について説明します。写真-ulleo -PDこの 脆弱性は、64ビットIOS XRを実行するASR 9000シリーズルーターで発見されました。これは、通信会社や携帯電話会社のデータセンター向けのハイエンド機器であり、スロットあたり400 Gbit /秒の容量があり、40 G / 80 Gラインカードをサポートしています。






識別子CVE-2019-1710が割り当てられた脆弱性。彼女は、CVSSスケールで10のうち9.8を獲得しました。

この標準は、バグの危険性を評価するために、Microsoft、Cisco、CERT、IBMなどの企業の情報セキュリティ専門家のグループによって開発されました。

なぜ彼女は危険なのですか


このバグにより、攻撃者は、管理者の仮想マシン上のシステムアプリケーションに不正アクセスする機会を得ることができます。ハッカーはリモートで悪意のあるコードを実行し、DoS攻撃を行うことができます。シスコのエンジニアによると、問題はセカンダリ管理インターフェイス（ルートスイッチプロセッサ上のRMT 1のMGT LAN）が内部管理者アプリケーションから適切に分離されていないことです。攻撃者はそれらの1つに接続することにより、この脆弱性を悪用する可能性があります。

システムに問題があるかどうかを確認するには、sysadmin仮想マシンにログインし、コンソールでshow interfaceコマンドを入力する必要があります。セカンダリインターフェイスが接続されている場合（以下の回答のように）、ルーターは脆弱です。

シスコの専門家によると、脆弱なのはASR 9000プラットフォームのみであり、Cisco IOS-XR 64ビットを実行している他の企業のソリューションは安定しています。同時に、同社はCVE-2019-1710を使用してハッカー攻撃を仕掛ける試みをまだ記録していません。

それを閉じる方法


シスコは、IOS XRバージョン6.5.3および7.0.1の一部としてCVE-2019-1710を修正するパッチを公開しました。このアップデートは、オペレーティングシステムの最新のライセンスを持つすべての組織（および以前に購入した組織）が無料で入手できます。

代替オプションがあります-脆弱性を完全に排除する回避策に頼ることができます。最初に、管理仮想マシンに接続する必要があります。

RP/0/RSP1/CPU0:eXR#admin
Tue Mar 12 22:46:37.110 UTC

root connected from 127.0.0.1 using console on host

次に、Bashを実行して、calvados_bootstrap.cfg構成ファイルを編集します。

sysadmin-vm:0_RSP1:eXR# run bash
Tue Mar 12 22:46:44.224 UTC
bash-4.3# vi /etc/init.d/calvados_bootstrap.cfg

次の2行では、＃記号を削除してファイルを保存する必要があります。

#CTRL_VRF=0
#MGMT_VRF=2

ソリューションに2つのRSPシステムがある場合、それぞれの構成で＃を削除する必要があります。次に、仮想マシンを再起動します。

sysadmin-vm:0_RSP1:eXR# reload admin location 0/RSP1

他に何をパッチしたか


CVE-2019-1710のパッチと並行して、ITの巨人は重要度の低い脆弱性のためにさらに20のパッチをリリースしました。これには、IAPP（アクセスポイント間プロトコル）、およびインターフェイスWLC（ワイヤレスLANコントローラー）とCisco VCS Expresswayの6つのバグが含まれていました。

パッチが適用された製品のリストには、UCS Bシリーズブレードサーバー、Cisco Umbrella、DNAセンター、登録済みエンベロープサービス、ディレクトリコネクタ、Prime Network Registrarなどが含まれます。完全なリストは、公式Webサイトにあります。


写真-Mel Clark -PD

また5月上旬に、会社の開発者は閉鎖しましたASR 9000およびCisco IOS XRの別の脆弱性。マルチキャストルーティングの問題を解決するPIM（Protocol Independent Multicast）機能に関連付けられています。バグ（識別子CVE-2019-1712を受け取った）により、攻撃者はリモートでPIMプロセスを再起動し、DoS攻撃を行うことができます。

さらに、開発者は以前に修正された脆弱性に関する一連の警告を公開しています。情報セキュリティの専門家によると、DNS攻撃にSea Turtleハッカーグループを使用している人もいます。エンジニアは状況を監視し、最新のアップデートを公開することを約束しました。
