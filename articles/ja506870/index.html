オーディオを長時間聞くことの危険性に関する警告を無効にする方法（Android）

おそらく、Androidデバイスから（だけでなく）音楽を聴く多くの人がこの警告に遭遇しました。


この記事では、この警告が発生する理由とタイミング、および警告が発生しないことを確認する方法について説明します。

外部デバイス（ヘッドフォン/スピーカー）を介してオーディオを聞く場合にのみ表示されます。これに出会ったことがない人のために、少し説明します。ヘッドフォンで音楽を聴いていると想像してみてください。突然、音が静かになります。ケースのボタンを使用して音量を上げようとしていますが、うまくいきません。デバイスをポケットから取り出してロックを解除すると、そのような警告が表示されます。彼と合意した後でのみ、音量を上げることができます。

はい、警告は妥当ですが、予測できない瞬間に表示され、時には最も不適切な場合があります。公共交通機関/運転中/冬の路上/汚れた手などの場合です。これらの場合、デバイスを取り外し、ロックを解除し、警告を受け入れ、デバイスを戻すのは不便です。ヘッドフォンではなくスピーカーが接続されている場合、メッセージは完全に適切ではありません。

なぜそれが起こるのですか


この警告は、プラットフォームの作成者の主導ではありません。重要なのは、「安全なリスニング」（安全なリスニング）のためのWHO-ETU標準があることです。欧州およびその他の一部の国では、その実装は必須です。この規格は、難聴のリスクを最小限に抑えながら、音量に応じて音声を聞くことができる時間を規定しています。たとえば、大人の場合、毎週の安全な音の「線量」は1.6 Pa 2時間です。これは、83 dBの音量で20時間聴くことに相当します。


実装


mcc（モバイル国コード）に応じて、セーフリスニングモードをオンまたはオフにできます。これは、リソースの値によって決まりR.bool.config_safe_media_volume_enabledます。

モードがオンの場合、システムは危険な音量（85 dB以上）でのリスニング時間を計算し、その値を定期的に変数に格納しますSettings.Secure.UNSAFE_VOLUME_MUSIC_ACTIVE_MS。値が20時間に達すると、警告が表示されます。警告を受け入れると、値がリセットされ、カウントが再開されます。

このような実装は非常に単純であり、たとえば、ユーザーがこの20時間をどれだけ長く聞いたか、たとえば数日のうちに、または6か月間6〜7分間聞いた可能性があることを考慮に入れていません（標準によると、これは聴覚に対する脅威ではありません。 ）

安全なリスニングロジックは、クラスクラスAudioService.javaに集中しています。、対応するフィールドが表示されます。

// mMusicActiveMs is the cumulative time of music activity since safe volume was disabled.
// When this time reaches UNSAFE_VOLUME_MUSIC_ACTIVE_MS_MAX, the safe media volume is re-enabled
// automatically. mMusicActiveMs is rounded to a multiple of MUSIC_ACTIVE_POLL_PERIOD_MS.
private int mMusicActiveMs;
private static final int UNSAFE_VOLUME_MUSIC_ACTIVE_MS_MAX = (20 * 3600 * 1000); // 20 hours
private static final int MUSIC_ACTIVE_POLL_PERIOD_MS = 60000;  // 1 minute polling interval

このフィールドmMusicActiveMs には、ダイアログの最後の確認以降、ユーザーが安全でないボリュームでリッスンしたミリ秒数が含まれています。初期値は変数から読み込まれますSettings.Secure.UNSAFE_VOLUME_MUSIC_ACTIVE_MS。同じ変数に、新しいmMusicActiveMs値が1分ごとに書き込まれます。

フィールドもありmSafeMediaVolumeState、安全なリスニングシステムの現在の状態が含まれています。

  • DISABLED：オフ
  • ACTIVE：有効になっており、リスニング制限に達しています。つまり、ユーザーが警告に同意するまで音量を上げることはできません。
  • INACTIVE：オン、制限にまだ達していない


// mSafeMediaVolumeState indicates whether the media volume is limited over headphones.
// It is SAFE_MEDIA_VOLUME_NOT_CONFIGURED at boot time until a network service is connected
// or the configure time is elapsed. It is then set to SAFE_MEDIA_VOLUME_ACTIVE or
// SAFE_MEDIA_VOLUME_DISABLED according to country option. If not SAFE_MEDIA_VOLUME_DISABLED, it
// can be set to SAFE_MEDIA_VOLUME_INACTIVE by calling AudioService.disableSafeMediaVolume()
// (when user opts out).
private static final int SAFE_MEDIA_VOLUME_NOT_CONFIGURED = 0;
private static final int SAFE_MEDIA_VOLUME_DISABLED = 1;
private static final int SAFE_MEDIA_VOLUME_INACTIVE = 2;  // confirmed
private static final int SAFE_MEDIA_VOLUME_ACTIVE = 3;  // unconfirmed
private Integer mSafeMediaVolumeState;

制限の超過を確認する方法は次のとおりです。

private void onCheckMusicActive(String caller) {
   synchronized (mSafeMediaVolumeState) {
       if (mSafeMediaVolumeState == SAFE_MEDIA_VOLUME_INACTIVE) {
           int device = getDeviceForStream(AudioSystem.STREAM_MUSIC);

           if ((device & mSafeMediaVolumeDevices) != 0) {
               sendMsg(mAudioHandler,
                   MSG_CHECK_MUSIC_ACTIVE,
                   SENDMSG_REPLACE,
                   0,
                   0,
                   caller,
                   MUSIC_ACTIVE_POLL_PERIOD_MS);
               int index = mStreamStates[AudioSystem.STREAM_MUSIC].getIndex(device);
               if (AudioSystem.isStreamActive(AudioSystem.STREAM_MUSIC, 0) &&
                   (index > safeMediaVolumeIndex(device))) {
                   // Approximate cumulative active music time
                   mMusicActiveMs += MUSIC_ACTIVE_POLL_PERIOD_MS;
                   if (mMusicActiveMs > UNSAFE_VOLUME_MUSIC_ACTIVE_MS_MAX) {
                       setSafeMediaVolumeEnabled(true, caller);
                       mMusicActiveMs = 0;
                   }
                   saveMusicActiveMs();
               }
           }
       }
   }
}
