ラインに従うロボットを作成するすべての段階、またはSTM32ですべての熊手を収集する方法

こんにちは、ハブル！

この出版物では、競技用のロボットを作成する、つまりラインをたどる私の経験について説明します。回路の設計からプリント基板の注文、アルゴリズムやプログラムに至るまで、すべての段階を教えてみます。

ラインに沿ったロボットの競争は、長い間初心者ロボット工学の基本的なテストでした。ほぼすべての地域および国際的なコンテストがこの方向性を含んでいるため、1つのロボットを作成すると、ほとんどどこにでも参加できます。大学の最初の年には、これは最初のLED点滅の後の次のタスクでした。

問題の定式化


開発はすべて技術的な割り当てから始まります。ここでは、Robofinist競技の競技規則がTKとして使用されます。寸法の要件（25x25x25 cm以下）とロボットの重量（1 kg以下）のほか、トラックの形状とその幅1.5 cmに関心があります。トラックの複雑さに応じて、通過するのにかかる最小時間（60秒）のしきい値が設定されます。 ）

タスクは明らかです。ラインを検出し、最初から最後まで最短時間で運転できるロボットが必要です。

コンセプトデザイン


ロボットの中核はSTM32F103C8T6マイクロコントローラーで、QRE1113光学センサーを使用してトラック情報を読み取り、A3906ドライバーを介してエンジンを制御します。明確にするために、センサーにはそれぞれ独自のLEDが割り当てられており、センサーが黒い線を「見る」とすぐに、LEDが点灯します。これにより、デバッグプロセスが大幅に簡略化されます。電源として、電圧3.7 VのLi-poタイプ18650バッテリーと充電チップLTC4054ES5を選択しました。しかし、モーターを正しく動作させるには、少なくとも6 Vが必要です。そのため、LM2577 DC-DCステップアップ電圧コンバーターと、ロジック部に供給するLM1117リニア3.3 Vをインストールしました。



最初の間違いについて話す時間です。マイクロコントローラーの電源ピンの1つのペアを注意深く混同し、起動時に問題を探すために数時間を費やしたため、それらを噛まなければなりませんでした。次に、すべてのセンサーがラインに反応したわけではないという問題が発生しました。私の最初の反応は、マイクロコントローラーを燃やしたか、最後の段階で電源の壊れた脚がまったく不要ではなかったか、接点をはんだ付けしたことでした。

オシロスコープを使用して、10個のセンサーのうち4個が機能せず、センサー自体に問題があることがわかりました。新しいセンサーに交換すると、次の問題が発生しました-3つのインジケーターLED PB3、PB4、PA15が機能しませんでした。すべてのコードを書き直して調べたところ、プッシュプル、10 MHz、クロックが有効になっているすべての場所で、非稼働ポートと稼働ポートの初期化に違いは見つかりませんでした。オシログラムを見て、1つのピンがプラスに引き付けられ、デバッグモードでもまったくリセットされないことに気づき、マイクロコントローラーが焼けていることに気付きました。データシートを閉じると、不思議なことに、これらのピンはJTAGに使用されており、SWDを使用していることに気付きましたが、それが悪化することはないと判断したので、JTAGを無効にする方法を検討し始めました。

結論として、後で何かが焼けたり、他の理由で機能しないと思わないように、すべての段階で正しい栄養を確認してください。

スキームで改善する必要があるもの


私は充電チップを置くと思ったが、それはそれほど重要ではないが、放電を忘れていた。さらに、2020年にタイプCの代わりにミニUSBを置いた。マイクロコントローラーを使用してバッテリーの充電を監視したいが、空きADCピンがなかった。ドライバーを選択するとき、6 Vおよび700 mAエンジンの特性に依存しましたが、その最大電圧が9 Vであることを考慮に入れていませんでしたが、12 Vが必要です。

PCBデザイン


まず、ロボットのPPとパートタイムプラットフォームのサイズと形状を決定することが決定されました。規制によると、寸法は25x25 cmを超えてはなりませんが、中国の工場（PCBWay）でソフトウェアを注文する場合、10x10 cmを超えるボードは大幅に価格が上がることも考慮する価値があります。寸法の選択は明白だと思います。

次に、センサーを対称的に（ボードの下部に）配置し、線幅を考慮してから、モーターやバッテリーなどの最も重い要素を考慮します。 LEDは、ボードの反対側の対応するセンサーと同じ軸に取り付けられます。便宜上、プラットフォームの背面に電源スイッチとバッテリー充電コネクタを取り付けています。残りの要素は中心に近く設定されます。

正方形ではなく、ボードの空のエッジを切り取って、ロボットプラットフォームの完全にまともな画像を取得します。

ユニークな要素が少ないため、ボードは非常にシンプルであることがわかりました。合計2層の最小トラック厚さ0.25 mm。マザーボードをガーバーフォーマットで保存したので、マスクの白色を選んで工場に送りましたが、特に問題はありませんでした。



要素を含むボードの3Dビュー。ライブ写真は最後になります。



ボードで確定する必要があるもの


バッテリーのプラスチックケースに近すぎる要素を取り付けたため、はんだ付けが難しく、はんだ付けがさらに困難でした。プログラミングコネクタSWD、UART、およびエンジンのピン配置に署名するのを忘れたため、回路を調べる必要がありました。

アルゴリズム


アルゴリズムは最も単純なものを実装しました。たとえば、ラインが右側にあると判明した場合、右側のエンジンが減速し始め、ラインが端に近づくほど、エンジンは遅くなり、完全に停止しますが、左側のエンジンは最大速度で動作しますラインに向けようとしています。線が左側にある場合も同様です。したがって、ロボットは安定を試み、ラインを中央のセンサーに戻します。



写真のアルゴリズム










また、他の状況にも備える必要があります。たとえば、すべてのセンサーがラインを検出した場合、これはおそらく交差点を横断しているため、さらに先に進む必要がある可能性が高いです。または、極端なセンサーだけがラインを検出した場合は、フィニッシュラインに到達し、停止する必要があります。

この段階で、あなたはタンバリンを手に入れ、それにぶつかる必要があります。

void RCC_Init(void) {						// Quartz 16 MHz

	RCC->CFGR |= RCC_CFGR_PLLXTPRE;	// PLLXTPRE set divider (/2) = 8 MHz

	RCC->CR |= RCC_CR_HSEON;			// On HSE
	while (!(RCC->CR & RCC_CR_HSERDY)) {
	};

	RCC->CFGR |= (RCC_CFGR_PLLMULL6);     // Setup PLLMULL set multiplier (x6) = 48 MHz

	RCC->CFGR |= RCC_CFGR_PLLSRC;		// PLLSRC set HSE

	RCC->CR |= RCC_CR_PLLON;			// ON PLL
	while (!(RCC->CR & RCC_CR_PLLRDY)) {
	};

	FLASH->ACR &= ~FLASH_ACR_LATENCY;	// Setup FLASH
	FLASH->ACR |= FLASH_ACR_LATENCY_1;

	RCC->CFGR |= RCC_CFGR_HPRE_DIV1; 	// Set not divider (AHB) = 48 MHz
	RCC->CFGR |= RCC_CFGR_PPRE1_DIV2; 	// Set divider (/2) (APB1) = 24 MHz
	RCC->CFGR |= RCC_CFGR_PPRE2_DIV1; 	// Set not divider (APB2) = 48 MHz

	RCC->CFGR &= ~RCC_CFGR_SW; 		// Setup SW, select PLL
	RCC->CFGR |= RCC_CFGR_SW_PLL;

	 DMA1_Channel1->CCR |= DMA_CCR_PSIZE_0;		 		//   16 bit

	 DMA1_Channel1->CCR |= DMA_CCR_CIRC;

	 RCC->CFGR |= RCC_CFGR_ADCPRE_DIV8;
	 RCC->APB2ENR |= RCC_APB2ENR_ADC1EN;

void TIM1_Init(void) {

	/*************** Enable TIM1 (CH1) ***************/
	RCC->APB2ENR |= RCC_APB2ENR_TIM1EN;
	TIM1->CCER = 0;										//  CCER ( )
	TIM1->ARR = 1000; 									//  ,     
	TIM1->PSC = 48 - 1;                					// 
	TIM1->BDTR |= TIM_BDTR_MOE;     					//     

	TIM1->CCR2 = 0; 									//       (  0  TIM1->ARR)
	TIM1->CCMR1 |= TIM_CCMR1_OC2M_1 | TIM_CCMR1_OC2M_2; //     
	TIM1->CCER |= (TIM_CCER_CC2P |TIM_CCER_CC2E); 		//        
														//   -   3
	TIM1->CCR3 = 0; 									//       (  0  TIM1->ARR)
	TIM1->CCMR2 |= TIM_CCMR2_OC3M_1 | TIM_CCMR2_OC3M_2; //     
	TIM1->CCER |= TIM_CCER_CC3NE; 						//        
	TIM1->CCER &= ~TIM_CCER_CC3NP;

	TIM1->CR1 |= TIM_CR1_CEN;
}

void UART3_Send(char chr) {
	while (!(USART3->SR & USART_SR_TC))
		;
	USART3->DR = chr;
}

	uint8_t i = 0;

uint32_t Motor(int32_t which, int32_t speed, int32_t turn) {

	if (which == 0)	//Right motor
	{
		UART3_Send_Number_Float(speed);
		UART3_Send_String("\t");

		if (speed > 0 && speed <= 1000)
		{
			TIM1->CCR3 = speed;
			if (turn == 0) 	// back
			{
				GPIOB->BSRR |= GPIO_BSRR_BS14;
			}
			else			//forward
			{
				GPIOB->BSRR |= GPIO_BSRR_BR14;
			}
		}
		else		//disable motor
		{
			TIM1->CCR3 = 0;
			GPIOB->BSRR |= GPIO_BSRR_BR14;

		}

	}

	else if (which == 1)	//left motor
	{
		UART3_Send_Number_Float(speed);
		UART3_Send_String("\r\n");

		if (speed > 0 && speed <= 1000)
		{
			TIM1->CCR2 = speed;
			if (turn == 1) 	// back
			{
				GPIOA->BSRR |= GPIO_BSRR_BS8;
			}
			else			//forward
			{
				GPIOA->BSRR |= GPIO_BSRR_BR8;
			}
		}
		else		//disable motor
		{
			TIM1->CCR2 = 0;
			GPIOA->BSRR |= GPIO_BSRR_BS8;

		}

	}



	return 0;
}
