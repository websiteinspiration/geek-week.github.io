6. Platform Check Point Maestro yang dapat diukur menjadi lebih terjangkau. Gateway Check Point Baru



Sebelumnya, kami menulis bahwa dengan kedatangan Check Point Maestro , tingkat masuk (dalam istilah moneter) ke platform yang dapat diskalakan telah menurun secara signifikan. Tidak ada lagi kebutuhan untuk solusi sasis. Ambil persis sebanyak yang Anda butuhkan dan tambahkan sesuai kebutuhan tanpa biaya awal yang besar (seperti halnya dengan sasis). Cara ini dilakukan bisa dilihat di sini . Untuk waktu yang lama, hanya beberapa bundel yang tersedia untuk pesanan - 6500, 6800 dan 23800. Maka, tahun ini, Check Point menghadirkan model gateway baru yang lebih kuat - Quantum. Akibatnya, bundel minimal baru dengan satu orkestra (MHO140) dan dua gateway (6200 Plus) turun lebih dari dua kali! Hal ini memungkinkan perusahaan dari hampir semua ukuran untuk menggunakan solusi scalable tanpa biaya mahal di muka. Mari kita lihat model-model baru secara lebih rinci.

Cabang dan kantor kecil (tidak cocok untuk Maestro)


Untuk usaha kecil dan cabang, model baru disajikan - 3600 ( lembar data ) dan 3800 ( lembar data ). Meskipun model-model ini TIDAK BISA digunakan untuk terhubung ke orkestra (diperlukan tautan 10G), saya masih menganggap penting untuk membicarakannya. Dibandingkan dengan model sebelumnya (3100, 3200), produktivitas memiliki lebih dari dua kali lipat, sementara harganya tidak berubah. Usaha kecil dan menengah


Untuk usaha kecil dan menengah, mereka segera merilis 4 model baru: 6200 ( lembar data ), 6400 ( lembar data ), 6600 ( lembar data ) dan 6700 ( lembar data ). Fitur utama dalam gambar di bawah ini:



Semua model dapat digunakan untuk terhubung ke orkestra. Misalnya, jika Anda menggunakan 6200 gateway, Anda dapat meningkatkan "kekuatan" cluster dari 3,6 Gbit / s (2 node) menjadi 93,6 Gbit / s (52 node) dalam mode Pencegahan Ancaman. Untuk gateway 6600, ini akan menjadi angka 7.4 dan 192.4 masing-masing. Angka yang serius.

Bisnis besar


Untuk bisnis besar, dua model baru muncul - 7000 ( lembar data ) dan 16200 ( lembar data ). Karakteristik untuk model 7000 dalam gambar di bawah ini:



Anda dapat meningkatkan daya klaster dari 19 Gbit / detik (2 node) menjadi 494 Gbit / detik (52 node) dalam mode Pencegahan Ancaman.

Fitur-fitur untuk model 16200:



Anda dapat meningkatkan daya klaster dari 30 Gbit / detik (2 node) menjadi 780 Gbit / detik (52 node) dalam mode Pencegahan Ancaman.

Solusi Pusat Data


Dan model yang paling kuat di keluarga, model untuk pusat data - 26000 ( lembar data ) dan 28000 ( lembar data ). Fitur utama pada gambar di bawah ini:



Untuk model 26.000, Anda dapat meningkatkan daya gugus dari 48 Gbit / dt (2 node) menjadi 1248 Gbit / dt (52 node) dalam mode Pencegahan Ancaman.
Untuk model 28000, Anda dapat meningkatkan daya gugus dari 60 Gbit / dt (2 node) menjadi 1560 Gbit / dt (52 node) dalam mode Pencegahan Ancaman. Itu 1,5 Tbps!

Alih-alih sebuah kesimpulan


Saya yakin banyak model dari garis 6000 akan menarik. Berdasarkan pada mereka, Anda dapat segera mulai membangun platform yang dapat diskalakan di sekeliling jaringan. Ini akan lebih menguntungkan daripada klaster klasik, baik dari segi keuangan maupun dari segi teknis. Kami sebelumnya menulis tentang ini . Oleh karena itu, jika Anda hanya berencana memperkenalkan firewall atau sedang mempertimbangkan untuk meningkatkan, kami sangat merekomendasikan untuk melihat ke Check Point Maestro. Dengan tingkat probabilitas yang tinggi, mungkin ini merupakan solusi yang paling optimal.
