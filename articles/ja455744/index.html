視覚的リアリズムを改善した迷路風景生成システム[Jinmo Kimによる記事の翻訳]

こんにちは、ハブル！


この投稿では、Jinmo Kimによる記事「新しい視覚的リアリズムのための没入型仮想現実における迷路地形オーサリングシステム」についてお話します。2019年4月4日に公開されました。記事の全文はここにあります。


システムの簡単な説明


この記事では、迷路の風景を作成するためのオーサリングシステムを提案します。これにより、ユーザーは、シンプルで直感的なシステムでさまざまな複雑なパターンを自動的に生成できます。プログラムで計算された迷路情報を使用して、3次元の迷路風景が迅速かつ効率的に生成されます。


提案されている迷路ランドスケープ生成システムは、3つの主要な機能で構成されています。


  • 迷路の生成の古典的なアルゴリズムを使用して実装された、さまざまなサイズとパターンの迷路グリッドの自動生成機能。
  • 円形迷路生成機能;
  • 画像処理アルゴリズムを使用して、手動スケッチの迷路を3Dオブジェクトに変換する機能。

これら3つの機能からなる提案システムを使用すると、均一から不規則まで、さまざまな迷路を効果的に作成できます。この開発により、同じシステムを介して行われた計算から直接、さまざまな概念の迷路ランドスケープを生成できます。さらに、この作品では、仮想現実への存在感と没入感の効果の改善を分析しています。視覚的リアリズム。また、この作業では、調査を使用して、提案されたオーサリングシステムの適合性を分析しました。

procedure CIRCULAR MAZE GENERATION PROCESS(nr, ne)
    ns ← interval between maze for user road.
    while nr > ns do
        define wall cell by using Bresenham’s circle drawing algorithm.
        nr = nr - ns.
    end while
    nc ← number of drawing circles.
    Arre[ne] ← as many entrance cells as ne are selected from wall cell of the outermost circle.
    for i = nc, 0 do: generate passages in every circle.
        for j = 0, ne do
            select cells with length of ns in entrance cell (Arre[j]).
            rotate selected cells.
            renew Arre[j] by finding a cell that meets the inner circle after rotation.
        end for
    end for
end procedure
