スプライトシートのスライス


多くの場合、2Dゲームでは、1つの大きな画像を使用して、いくつかの小さな画像を格納します。たとえば、タイルごとに多数の小さな画像を用意するのではなく、タイルセットにタイルを格納します。このような画像はスプライトシートと呼ばれ、描画するテクスチャを変更する必要はなく、テクスチャのどの部分を使用するかを示すだけなので、作業に非常に便利です。

このチュートリアルでは、SDL_RenderCopyを使用してテクスチャの一部を選択する方法と、描画するテクスチャの一部を選択するために使用する特定のキーストロークイベントを検出する方法について少し説明します。スプライトシートには4つのカラフルな円があります。



このチュートリアルでは、スプライトシートは同じサイズの多数のスプライトで構成されています。この場合、スライスは難しくありません。そうでなければ、異なるサイズのスプライトの場合、パーツの場所に関する情報を含むメタデータファイルが必要になります。このチュートリアルでは、サイズ100x100の4つのスプライトを使用します。このレッスンのコードはレッスン4に基づいています。まだ書き込むコードがない場合は、Githubから取得できます。

パーツ選定


SDLを使用すると、描くテクスチャの部分を非常に簡単に選択できます。レッスン4では、NULL値を持つ残りのSDL_RenderCopyパラメータは、描画するテクスチャのどの部分を決定する長方形の座標を示します。NULL値を渡す場合、テクスチャ全体が必要であることを示しますが、長方形のパラメータを簡単に追加して、テクスチャの一部のみを描画できます。これを行うには、renderTexture関数に変更を加えて長方形の領域を取得できるようにしますが、便宜上、古いバージョンの構文の短いバージョンを保持します。

renderTextureを変更する


renderTexture関数にパラメーターを追加せず、同時にデフォルト値の利便性を維持するために、2つの関数に分割します。 1つ目はSDL_RenderCopyの呼び出しとほぼ同じですが、nullptr値を持つクリッピング領域パラメーターを提供します。このバージョンのrenderTextureは、長方形の領域の形式で場所を受け取ります。これは、自分で構成するか、他の特殊なrenderTexture関数の1つを使用して設定できます。新しい基本的なレンダリング機能は非常にシンプルになっています。

/**
*  SDL_Texture  SDL_Renderer   .
*     
* @param tex  ,   
* @param ren ,    
* @param dst     
* @param clip     ( )
*                 nullptr   
*/
void renderTexture(SDL_Texture *tex, SDL_Renderer *ren, SDL_Rect dst,
        SDL_Rect *clip = nullptr)
{
        SDL_RenderCopy(ren, tex, clip, &dst);
}

便宜上、ロケーションのSDL_Rectを作成する必要はなく、xとyのみを提供し、表示関数にテクスチャの幅と高さを埋めさせる別の関数を記述します。クリッピングを処理するためのいくつかの設定でこれを実行するrenderTextureのオーバーロードバージョンを作成します。デフォルト値nullptrでパラメーターとしてカット長方形を追加します。カットが渡された場合は、テクスチャの幅と高さの代わりにカットの幅と高さを使用します。したがって、小さいスプライトが描画されるときに、非常に大きい可能性があるスプライトシートのサイズに引き伸ばされることはありません。この関数は、元のrenderTexture関数を変更したもので、非常によく似ています。

/**
*  SDL_Texture  SDL_Renderer   x, y, 
*          
*    ,       
*          
* @param tex  ,   
* @param ren ,    
* @param x  x,    
* @param y  y,    
* @param clip     ( )
*                 nullptr   
*/
void renderTexture(SDL_Texture *tex, SDL_Renderer *ren, int x, int y,
        SDL_Rect *clip = nullptr)
{
        SDL_Rect dst;
        dst.x = x;
        dst.y = y;
        if (clip != nullptr){
                dst.w = clip->w;
                dst.h = clip->h;
        }
        else {
                SDL_QueryTexture(tex, NULL, NULL, &dst.w, &dst.h);
        }
        renderTexture(tex, ren, dst, clip);
}

クリッピング長方形を定義する


今回の場合、レッスン3のタイリング方法とよく似た方法でクリッピング長方形を計算するのは非常に簡単ですが、行ごとではなく列ごとに行います。したがって、最初のピースは緑、2番目のピースは赤、3番目のピースは青、4番目のピースは黄色になります。コンピューティングの考え方はレッスン3と同じですが、行ではなく列を実行します。したがって、y座標は、タイルインデックスをタイルの数（2）で除算するときの余りを取得することによって計算され、x座標は、インデックスをタイルの数で除算することによって計算されます。これらのx座標とy座標はxインデックスとyインデックスであるため、カットアウトの幅と高さを掛けて実際のピクセル座標に変換します。これは、すべてのタイル（100x100）で同じです。最後に、描くピース、この場合は最初のものを選択します。

画面の中央にピースを描画したいので、テクスチャの幅と高さの代わりにタイルの幅と高さを使用して、これらのx座標とy座標を計算します。

//iW  iH    
//   ,     ,     
int iW = 100, iH = 100;
int x = SCREEN_WIDTH / 2 - iW / 2;
int y = SCREEN_HEIGHT / 2 - iH / 2;

//    
SDL_Rect clips[4];
for (int i = 0; i < 4; ++i){
        clips[i].x = i / 2 * iW;
        clips[i].y = i % 2 * iH;
        clips[i].w = iW;
        clips[i].h = iH;
}
//     
int useClip = 0;

代わりに、異なるサイズの回転したスプライトが一緒にパックされた、より複雑なスプライトシートを使用する場合、パーツを簡単に見つけられるように、それらの位置と回転に関する情報をメタデータファイルに保存する必要があります。

入力に基づいて画像を変更する


作成した画像のすべての部分を確認するには、イベント処理ループにキーボード入力処理を追加し、1〜4のキーを使用して表示されている部分を選択します。キーが押されたかどうかを判断するには、イベントのタイプがSDL_KEYDOWNであるかどうかを確認します。そうである場合は、e.key.keysym.symを使用してイベント内のキーコードを確認することで、押されたキーを確認できます。SDL_Eventに関するイベントタイプ、キーコード、およびその他の情報の完全なリストは、wikiで入手できます。 キーが押されたら、useClipの値を描画したい画像の部分の番号に変更する必要があります。これらの変更により、イベントループは次のようになります。



while (SDL_PollEvent(&e)){
        if (e.type == SDL_QUIT)
                quit = true;
        //       
        if (e.type == SDL_KEYDOWN){
                switch (e.key.keysym.sym){
                        case SDLK_1:
                                useClip = 0;
                                *break;
                        case SDLK_2:
                                useClip = 1;
                                *break;
                        case SDLK_3:
                                useClip = 2;
                                *break;
                        case SDLK_4:
                                useClip = 3;
                                *break;
                        case SDLK_ESCAPE:
                                quit = true;
                                *break;
                        default:
                                *break;
                }
        }
}

切り抜いた画像を描く


最後に行うことは、画面上の画像の適切な部分を取得することです！これを行うには、renderTextureのより便利なバージョンを呼び出して、追加のスケーリングなしでイメージの一部を描画し、使用する部分（useClipで使用される部分）を転送します。

SDL_RenderClear(renderer);
renderTexture(image, renderer, x, y, &clips[useClip]);
SDL_RenderPresent(renderer);
