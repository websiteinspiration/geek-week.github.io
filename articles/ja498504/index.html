新年のご挨拶とCOVID-19：ハッカーがニュースをどのように使用するか



サイバー犯罪者は、最新のニュースやイベントを使用して悪意のあるファイルを送信することがよくあります。コロナウイルスのパンデミックに関連して、ガマレドン、SongXY、TA428、ラザロ、コンニ、ウィンティなどの多くのAPTグループがこのトピックをキャンペーンで使用し始めました。そのような活動の最近の例の1つは、韓国のグループHigaisaの攻撃です。

PT Expert Security Centerの専門家が、Higaisaが作成した悪意のあるファイルを検出して分析しました。

例1：偽のWHOレポート


コロナウイルスの蔓延に関する最初の世界保健機関（WHO）の通知が3月上旬に発表されました。数日以内に、Higaisaの参加者は悪意のあるファイルを含む手紙を送り始めました。隠蔽には、PDF形式の正規のWHOレポートファイルが使用されました。

感染は、ファイル20200308-sitrep-48-covid-19.pdf.lnkで始まりました。LNKファイルの

ファイルは、PDFドキュメントのアイコンが付いた.lnkショートカットです。開こうとすると、cmd.exe / cコマンドが次のコマンドラインで実行されます。



findstr.exeを実行すると、LNKファイルの最後にあるBase64ロードが取得され、CertUtil.exe（msioa.exe）を使用してデコードされます。デコード結果はCABアーカイブであり、same％tmp％フォルダーに解凍され、マルウェアのインストールスクリプト、元のWHOレポートファイル（おとりとして）、インストーラーのペイロードなど、いくつかのファイルが含まれています。

例2：新年の挨拶


分析された2番目のサンプルは、新年の挨拶が含まれたRTFファイルです：



お祝いのテキストを含む

ドキュメントこのドキュメントは、Microsoft数式エディターのCVE-2018-0798の脆弱性を利用する人気のあるRoyal Road RTF（または8.t）ビルダーを使用して作成されました。このビルダーは公開されていませんが、TA428、Goblin Panda、IceFog、SongXYなどの中国のAPTグループに広く配布されています。名前8.tは、操作中に悪意のあるドキュメントが暗号化されたペイロードを含む一時フォルダーに8.tという名前のファイルを作成するためです。

この脆弱性を悪用した結果、ファイル％APPDATA％\ microsoft \ word \ startup \ intel.wllが作成されます。これは、次回Microsoft Wordを起動したときに読み込まれるDLLドロッパーです。そのペイロードは、2つのファイルで構成されています：％ALLUSERSPROFILE％\ TotalSecurity \ 360ShellPro.exeと％ALLUSERSPROFILE％\ TotalSecurity \ utils \ FileSmasher.exe。ファイルはxor 0x1Aを使用して暗号化されます。



ドロッパーの主な機能intel.wll（フラグメント）

次に、システムに修正があります。

このファイルは、作者Higaisaの唯一の同様のオブジェクトではありません。 Tencentのアナリストは同じ期間にHappy-new-year-2020.scrおよび2020-New-Year-Wishes-For-You.scrという名前の悪質な実行可能ファイルの配布。この場合、ソースファイルは実行可能であり、餌は挨拶JPGカードの形で存在し、解凍されてデフォルトのビューアで開かれます。



これらの脅威の構造は、CVE-2018-0798の悪用を除いて、RTFドキュメントとほぼ同じです。SCRファイルはドロッパーであり、ペイロードはxor 0x1Aを使用して復号化され、％ALLUSERSPROFILE％のサブフォルダーに解凍されます。

結論


Positive Technologiesのアナリストによる調査では、Higaisaマルウェアの進化が示されました。同時に、使用されるツール（ドロッパー、ローダー）の構造はほとんど変わっていません。検出を複雑にするために、攻撃者はコントロールサーバーのURL、RC4キーのパラメーター、SideLoadingに使用される正当なファイル、HTTP対話用のライブラリなど、特定の詳細を変更します。
