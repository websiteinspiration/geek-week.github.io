18 funciones de GitLab van de código abierto


Abrimos el código fuente de características de las etapas de Planificar, Crear, Verificar, Empaquetar, Liberar, Configurar y Defender.


Sid Sijbrandij pasó algún tiempo estudiando las características de GitLab y llegó a la conclusión de que, de acuerdo con la estrategia para determinar los planes de GitLab dirigidos al comprador , el código fuente de dieciocho características de siete etapas diferentes del ciclo de vida de DevOps debería estar abierto.


Cuando en 2018 presentamos un nuevo modelo de planes GitLab, decidimos que las características se distribuirán entre los cuatro planes en función de quién es el consumidor de esta característica. Las funciones destinadas a un usuario individual entran en el plan Core / Free, las funciones para los gerentes entran en Starter / Bronze, para los directores en Premium / Silver y para la alta gerencia en Ultimate / Gold.
