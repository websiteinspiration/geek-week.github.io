遠隔作業に危険な5つの脆弱性



画像：Unsplash

従業員をリモートモードに移行するとき、IT部門はさまざまなセキュリティエラーを発生させ、部外者に内部インフラストラクチャへのアクセスを提供します。

まず、これらの困難な月が暗号化ウイルスオペレーターや財政的志向のAPTグループにとって「簡単な肉」にならないように、インフラストラクチャで迅速に排除する必要がある脆弱性をリストします。

1. 2月の終わり以来、リモートデスクトッププロトコル（RDP）を使用する利用可能なノードの数は急速に増加しています。私たちの監視によると、このようなノードの平均10％がBlueKeepに対して脆弱です（CVE-2019-0708）。

BlueKeepを使用すると、Windows 7、Windows Server 2008、およびWindows Server 2008 R2のオペレーティングシステムをベースにしたコンピューターをリモートで完全に制御できます（Windows 10にかなり前に切り替えており、安全ですか？）。攻撃の場合は、脆弱なリモートデスクトップサービス（RDS）に特別なRDP要求を送信するだけで十分です。認証は必要ありません。

RDPプロトコルを使用するノードの数が増えると、それらの間の脆弱なマシンの数が増えます（原則として）。たとえば、ウラルでは、開いているノードの数が21％増加し、システムの17％にBlueKeepの脆弱性があります。次に、シベリア（それぞれ21％および16％）、北西（19％および13％）、南（12％および14％）、ヴォルガ（8％および18％）、極東（5％および14％）および中央（4％および11％）連邦地区。

パッチのインストールに加えて、BlueKeepの脆弱性と同様のCVE-2019-1181 / 1182を排除するには、ゲートウェイを介したリモートアクセスを提供する必要があります。 RDP接続の場合、これはVPN-VPNゲートウェイのリモートデスクトップゲートウェイ（RDG）です。職場への直接リモート接続は禁忌です。

2. Windowsの新しいバージョンにも脆弱性があり、攻撃者がリモートデスクトップサービスのエラーを使用して外部ネットワーク上を歩くことができます。これは、多数のBlueKeep-2エキスパートによって命名されたCVE-2019-1181 / 1182です。ネットワーク上のRDGによってリモートアクセスが構成されている場合でも、新しいパッチを確認し、必要に応じてインストールすることをお勧めします。

3.最も危険なセキュリティ問題のランキングでは、Citrixソフトウェアの脆弱性にブロンズを与えます（CVE-2019-19781）、Positive Technologiesの専門家であるMikhail Klyuchnikovによって特定され、更新の遅延とエクスプロイトのために非公式にShitrixと名付けられました。最初の詳細が公開されてから1か月半後、約16,000社に脆弱性が存在しました。エラーは非常に危険であり、インターネットからローカルネットワークに侵入することができます。特に、ランサムウェアウイルスのオペレーターであるRagnarokおよびREvil / Sodinokibiによって使用されます。

4.リモートデスクトッププロトコルCVE-2012-0002（MS11-065）の以前の脆弱性を忘れないでください。これは、依然としてネットワーク境界に見られます。 2012年に発見されたこの欠陥は、MAAPおよび申し立てにおけるMicrosoftのパートナーの1つからのPoCコードの漏洩で記憶されました申し立てによると、GRUの従業員が彼女のためにエクスプロイトを購入しようとしました。

5.最後に、プログラミング言語PHP 7（CVE-2019-11043）の逆シリアル化のメカニズムのエラーに注意する価値があります。また、権限のないユーザーが任意のコードを実行することもできます。FPMが有効になっている危険な状態のnginxサーバー（PHP言語でスクリプトを処理するためのパッケージ）。この欠陥により、NextCloudクラウドストレージユーザーがNextCryに感染しました。

更新とパッチの集中管理システムは、企業システムへのパッチ適用プロセスの自動化に役立ち、セキュリティ分析ツールは、脆弱性がないことを確認するのに役立ちます。

従業員のPCにアップデートをインストールする


サラリーマンが引っ越してきた多くの家庭用パソコンは、ほんの最近ほこりを消してしまったばかりで、情報セキュリティの面で問題があることを思い出してください。理想的な世界では、パソコンへのアクセスを提供するのではなく、実績のある準備された企業システムを強調するのが良いでしょう。しかし、今ではラップトップだけでは十分ではないかもしれません。したがって、攻撃者のエントリーポイントにならないように、リモートでホームPCを更新する大規模なプロセスを組織化する必要があります。

まず、オペレーティングシステム、オフィス製品、ウイルス対策ソフトウェアを更新することが重要です。さらに、サポートされていないバージョンのInternet Explorerなど、古いブラウザを使用することの危険性について従業員に警告する必要があります。ホームコンピューターを更新する前に、リカバリポイントを作成するか、システムのバックアップを作成して、別のWindows 10更新の失敗などの問題が発生した場合にロールバックする必要があります。

パスワードポリシーについては、リモートで接続する場合、非特権アカウントには12文字以上、管理者アカウントには15文字以上のパスワードを使用することをお勧めします。異なる種類の文字（小文字、大文字、特殊文字、数字）を同時に使用し、簡単に推測できるパスワードを除外します。私たちのデータによると、2019年、選択されたすべてのパスワードの48％は、年または月の時間を示す単語と年を示す4桁の数字の組み合わせで構成されていました（2019年9月または英語のキーボードレイアウトCtynz、hm2019）。このようなパスワードは正式にはパスワードポリシーに対応していますが、辞書に従ってほんの数分で選択されます。

一般的に、リモートコントロールツールの飛躍は現在の状況では有害です。1つのプログラムを選択し、ローカルユーザーの権利を区別することをお勧めします。たとえば、Windows AppLockerを使用している一部のリモートコンピューターで、許可されているソフトウェアのリストが登録されている場合は、正しくなります。

また、VPNアクセスの編成に関連して発生する可能性のある問題についても言及する必要があります。 ITスペシャリストは、機器を短時間で再構成し、境界のルールに違反することなく必要なアクセスをすべてのVPNユーザーに提供する時間がない場合があります。その結果、ITスペシャリストは、ビジネスの継続性を確保するために、必要なサブネットへのアクセスを1人の従業員だけでなく、すべてのVPNユーザーに一度に開くという、最も速くて簡単なオプションを選択する必要があります。このアプローチにより、セキュリティが大幅に低下し、外部の攻撃者（侵入できる場合）による攻撃の機会が増えるだけでなく、内部からの攻撃のリスクも大幅に増加します。ネットワークセグメンテーションを維持し、必要な数のVPNプールを割り当てるために、事前にアクションプランを検討することをお勧めします。

ソーシャルエンジニアリングはすでにコロナウイルスのストーリーを最大限に活用しているため、フィッシング攻撃の新しいトピックについて従業員に理解していただくことをお勧めします。 GamaredonやHigaisaなどのAPTグループは、転送、禁止、キャンセル、リモートワークに関連するストーリーを悪用し、従業員の個人メールアドレスを攻撃します。フィッシングメールは未知の攻撃者によって当社に送信されました。犯罪者は資格情報を盗もうとしました。従業員は脅威の重大度を理解し、正当なメールとフィッシングを区別する準備をする必要があります。これを行うには、情報セキュリティとソーシャルエンジニアリングに関する簡単な視覚的なトレーニング資料とメモを配布することをお勧めします。サンドボックスを使用した企業メール内のファイルの動的フィッシングは、フィッシングの症状を特定するのに役立ちます。

電子文書管理システムやERPにも注意を払う必要があります。現在では、以前は内部からしかアクセスできず、脆弱性の分析が行われていなかったビジネスアプリケーションが積極的に公開されています。同時に、分析されたもののセキュリティのレベルは低かった。ミッションクリティカルなアプリケーションに対するWebベースの脅威の悪用を防ぐために、ファイアウォール、アプリケーションレイヤー（Webアプリケーションファイアウォール）を使用することをお勧めします。

今週の可用性と可用性は重要な役割を果たします。多くの企業は、境界の脆弱性を排除し、ISプロセスを微調整する時間がないため、場合によっては、インフラストラクチャに既に陥っている違反者の特定に焦点を合わせる必要があります。そのような場合、それらが適用される場合があります。標的型攻撃をリアルタイムで（トラフィックの保存されたコピーで）検出し、攻撃者が密かに存在する時間を短縮するように設計されたディープネットワークトラフィック分析（NTA）システム。
