原則


「Goスタイル」により、Pascalでメソッドとインターフェースを実装するためのいくつかの原則を理解します。

  • クラス、オブジェクト、継承の独立した概念はありません。
  • このメソッドは、特定のデータ型に対して実装できます。型自体の宣言を変更する必要はありません。
  • インターフェイスは、インターフェイス宣言にリストされているすべてのメソッドが実装されている特定のデータ型と互換性があります。特定のデータ型の広告では、インターフェースを実装する必要はありません。

実装


メソッドとインターフェースを宣言するために、標準キーワードPascal forとが新しいロールで使用されinterfaceます。新しいキーワードは入力されていません。この単語forは、メソッドの受信者の名前とタイプを示すために使用されます（Go用語で）。以下は、TCatフィールドで以前に宣言された型のメソッドの説明の例Nameです。

procedure Greet for c: TCat (const HumanName: string);
  begin
  WriteLn('Meow, ' + HumanName + '! I am ' + c.Name);
  end;

レシーバは、実際にはメソッドの最初の引数です。

インターフェースは通常のPascalエントリであり、その宣言ではword recordがword に置き換えられinterfaceます。このレコードでは、手続き型のフィールド以外のフィールドを宣言することはできません。さらに、非表示フィールドが記録の先頭に追加されSelfます。インターフェイスタイプに変換される特定のタイプのデータへのポインタを格納します。次にインターフェース宣言の例を示します。

type
  IPet = interface
    Greet: procedure (const HumanName: string);
  end;

特定の型をインターフェースに変換するとき、コンパイラーは、インターフェースが必要とするすべてのメソッドの存在とそれらのシグニチャーのマッチングをチェックします。次に、ポインタを設定しSelf、インターフェイスのすべての手続き型フィールドに特定のタイプのメソッドへのポインタを設定します。

Goと比較すると、Pascalの現在のインターフェースの実装には制限があります。インターフェースタイプに変換された特定のデータタイプを動的にクエリすることはできません。したがって、空のインターフェースは意味がありません。おそらく、次の開発ステップはこのギャップを埋めることでしょう。ただし、現在の形式でも、インターフェイスはポリモーフィズムを提供し、それほど重要ではないタスクの多くで役立ちます。そのような問題を1つ検討します。
