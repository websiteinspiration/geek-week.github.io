カードをスワイプするだけ：ニューヨークの地下鉄でのOS / 2の使用方法

ヴィンテージのテクノロジーは、ニューヨーク市の地下鉄の構造物で何十年もの間働いてきました-時には予期しない方法でポップアップします。OS / 2ファンの記事


ニューヨーカーと観光客がタイムズスクエアとしても知られる42 Ulitsaメトロ駅に電話をかけます。冗談の始まりのように聞こえます。実際にはそうではありません。そのうちの1人は、彼がそこに着いてうれしいです。別の状況はひどく迷惑です。最高速度でそこから抜け出す方法を知っています。他はそうではありません-彼は英語を話しません。ニューヨーカーと観光客は別人ですが、現時点では彼らは1人です。どちらも、運送会社のMetropolitan Transportation Authority（MTA）の仕事の気まぐれさと、1990年代初頭の成功した平均的なオペレーティングシステムの前例のない信頼性にさらされています。

2016年の平均的な就業日に、ニューヨークの地下鉄は570万人を輸送しました[比較のために：モスクワの地下鉄の場合、この数字は670万/およそ transl。]。これは1948年以来最高の平均です。平均的なニューヨーカーに尋ねると、彼はおそらく「それだけですか？」信じられないことは理解できます。800万人の永住者が街に住んでいるため、ピーク時や休暇中には、人口が2,000万人に増えることもあります。


ニューヨークの地下鉄の改札口

将来に賭けるのは難しいですが、MTAは実際にそれを行いました。


3月に、Tedium は、オペレーティングシステム用のマイクロカーネルに対するIBMの大きな賭けについて書きました。これには、有名なOS / 2 OSのバリアントが含まれていました。これは、このレートのために会社が被った損失について詳しく説明しています。ただし、OSの成功に対するIBMの信頼により、他の会社も同様の前提を立てました。

ただし、最大の賭けはMTAであるMetropolitan Transit Authorityによって行われました。MTAはトークンを取り除き、すべてがデジタルである時代に移行する方法を考え出す必要がありました。その結果、カルトのMetroCardカードが登場しました。 1993年のリリース以来、ニューヨーカーのウォレットでは、黄色のプラスチックの薄いスライスと目立つ黒いストライプが主なアイテムになっています。

ニューヨークの地下鉄への現在のアクセス方法の歴史は、公共インフラストラクチャーの配置とそれが公共サービスを提供する方法の詳細において興味深いものです。しかしその前に、現在のシステムがどのように登場したかを理解することは役に立ちます。あなたがニューヨークの地下鉄と同じくらい重要なものを構築するとき、それは最終的には意図したとおりに機能するはずだからです。

実際、あなたはたった一度の試みしかありません-そして、どんな間違いでも修理のための何十億もの費用と何百万もの人々の煩わしさになるでしょう。多くの選択肢の中で、最も信頼できるものの1つがIBMの最大の間違いの1つになっています。

で記事 OS / 2程度、多くの興味深い詳細は、マイクロカーネルと他の事について言及したが、この記事では、このOSは、まだその支持者を持っていたという事実が話題に最も関連しています。さて、それなしで。

MTAが最終的にOS / 2を使用することを決定した理由は、地下鉄のいくつかの側面をデジタル化し、1990年代初頭のOSの発売を取り巻く誇大宣伝を反映しています。ただし、会話と開発は数年前に始まりました。これを特に促進するものではなく、MicrosoftとIBMは次世代のOSに取り組んでいました。ゲイツ氏とマイクロソフト社は、現代の話ではMS-DOSでIBMを作ったと考えられていますが、当時、IBMは明らかに異なる考えを持っています。

IBMは失われた利益を追悼する代わりに、明らかに知識の欠如に気づき、次世代のOSを基本的な基礎から開発し始めました。このベンチャーは、ご想像のとおり、MS-DOSの話とほぼ同じ方法でIBMのために終了しました。しかし、1980年代後半のごく短期間で、MTAのディレクターたちは、地下鉄でトークンを拒否して、プリペイドカードに置き換える方法を探していました。利点は明白でした-これにより、運賃を引き上げてトークン支払いを導入することがより簡単になりました。乗客は、片道または往復のいずれかを選択する機会があり、一定期間、無制限のオプションがありました。

この画期的なアップデートを紹介するために、MTAは有名な会社であるIBMを利用しました。それは当時は理にかなっています。


OS / 2バージョン2.1

OS / 2とMTAコンサルタントのニールウォルドハウアーはメールでこう述べています。「OS / 2でキャリアを賭けることができたのは数年前のことでした。」

理由を理解するには、その時間を理解する必要があります。 Waldhauer氏は続けます。「LinuxとWindowsがまだ開発されていなかった時代から開発が続けられています。 OS / 2は、将来的には安全な選択肢のように思えました。」

オプションがないため、MTAが最良のものを選択しました。そして彼は、かなり複雑なシステムの主要なソフトウェアコンポーネントの1つとして数十年間働いていました。

Waldhauerが言うように、彼はさらに生き残ることができます：「MetroCardはシステムでサポートされていますが、OS / 2は引き続き機能します」

非常に興味深い点です。MTAは、さまざまな形態の非接触型決済を支持してMetroCardを放棄しているところです。この移行により、作業効率が向上し、MTAが追加の収入を得るのに役立ちます。

面白そうに聞こえますが、現在のMetroCardシステムの奇妙な機能を研究すれば、問題を簡単に見つけることができます。


私のMetroCard、「Gay Pride Month」の6月版。興味深いことに、1年間しか使用できない標準のMetroCardよりも4か月長く使用できます。

神秘的な磁気ストリップ、そしてそれが人々の生活にどのように影響するか


要するに、トークンからMetroCardへの移行には何年もかかり、まったくスムーズではありませんでした。トークンは2003年に正式に廃止されました。その時までに、MetroCardは市内のすべての駅で受け入れられました-しかし誰もそれを好まなかった。

地下鉄へのアクセスは通常は簡単ですが、カードの配線に関する苦情は至る所にあります。そして、明らかに、多くの問題は、システムのさまざまな部分間の通信の愚かな中断に関連していた。 OS / 2は地下鉄システムのさまざまな部分を大きなメインフレームに接続するために使用されますが、着信コンポーネントの標準は最高ではありませんでした。ニューヨークの駅の改札口は気まぐれさで悪名高いですが、IBMシステムで作業することができました。


ATMもOS / 2に依存していた

コンシューマ市場でのOS / 2の失敗にもかかわらず、それは信じられないほど信頼性が高く、産業および産業システムでのその長寿命を保証しました-そして、ATMはアプリケーションの一例です。 Waldhauer氏は次のように述べています。「MTAで使用されているすべてのOSを思い出すと、メインフレームを除いて、OS / 2はおそらくシステムの最も信頼できる部分であると言えます。」 2019年もニューヨークの地下鉄で使用されています。 IBMは長い時間前にそれを放棄し、（今日は、という会社も、2001年には、それをサポートするソフトウェアに別の会社を許可ノアノハコブネガイは、 OS / 2、の公式にサポートされているバージョン販売しているArcaOSをそのユーザーのほとんどはMTAと同じような状況ではあるが、）。

NYC地下鉄のOS / 2は、指揮者の役割を果たします。人が使うさまざまな部品を、人が使わない部品と組み合わせると役立ちます。 Waldhauer氏は次のように述べています。「ユーザーが操作するシステムにOS / 2アプリケーションはありません。 OS / 2は主に、複雑なメインフレームデータベースと、地下鉄やバスで日常的に使用されている単純なコンピューターとの間のインターフェースとして使用されます。しかし、一般的に、OS / 2コンピュータはシステム全体に分散されています。」

2つのテクノジャイアントの複雑な関係の一部として、90年代前半にリリースされた80年代後半に設計されたOSについて話しています。 MTAは既に決定を下していたため、MTAはこの話のほとんどを無視する必要があり、コースの変更には多額の費用がかかります。

バックエンドと、ニューヨーカーや観光客が遭遇するデバイスの調整は、途方もなく調整されていません。これを想像したい場合は、Waldhauerに話を戻しましょう：「MetroCardがメインフレームデータベースで動作し、ランダムな電子デバイスがそれをすべて結びつけることを開発者が計画したと感じています。」


使用日によると、ニューヨークの地下鉄のトークンは左から右へ：1953-1970; 1970-1980; 1979-1980; 1980-1986; 1986-1995; 1995-2003。

それでは、磁気ストリップについて話しましょう。ブランドに関係なく、MetroCardの下部にある黒いバーは機能するはずです。それが実際にどのように機能するかは、明らかな理由で、秘密です。

「人々はMetroCardをハッキングした」とWaldhauer氏は語った。 「磁気コーディングを検討できる場合、ビットは非常に大きいため、虫眼鏡で見ることができます。磁気ストリップのコーディングは非常に秘密であるため、私はそれを見たことがありません。人々が無料の旅行のために何ができるかは驚くべきことです。」

今日は重要ですか？はい、原則としてそうではありません。 MTAは、ロンドンのオイスターカードと同様に、非接触型決済に切り替える予定であることを明確に述べています。ただし、このプロセスには独自の問題があります。彼らはロンドンのシステムの元首を雇い、MetroCardを完全に取り除くという究極の目標を設定しました。


新たに発売されたOMNYシステムは、今後数年で展開される予定です

将来的には、ディズニーランドでのジェットコースターの今日のラインナップと同じ方法で、人々はニューヨークの地下鉄に入ることができます。このプロセスでは、電話かスマートな時計かに関係なく、改札口を案内するデバイスがインターネットに接続されている必要があります。運が良ければ、MetroCardを備えた新しいシステムができます。しかし、保証はありません。

ニューヨークの地下鉄を作成した実用的および技術的なニーズは、都市のほとんどすべての住民に影響を与えます。ニューヨーカーは新しい支払い方法に切り替えており、それを支払うことができる人はそうします。そして残りは家に座ります。
