TextRadarファジー検索アルゴリズム。基本的なアプローチ（パート1）

ファジー文字列比較とは異なり、比較された両方の文字列が等しい場合、ファジー検索タスクで検索文字列とデータ文字列が強調表示され、2つの文字列の類似度ではなく、データ文字列内の検索文字列の存在度を判断する必要があります。

問題の定式化


データ文字列と検索文字列は、単語（スペースで区切られた文字のグループ）で構成される任意の文字セットとして指定されます。

データの行で、文字の構成と相対位置によって検索行に最も近いフラグメントのセットを見つける必要があります。

検索結果の品質を評価するには、関連性係数を計算します。その値は0から1の範囲内である必要があります。0はデータ文字列に検索文字列の文字が完全に存在しないことに対応し、1は歪んでいない形式のデータ文字列に検索文字列が存在することに対応します。

検索は、行の文字と単語の相互配置を考慮に入れながら、言語の構文と形態を考慮せずに、ソース行の文字ごとの分析によって実行する必要があります。

アルゴリズムの説明


検索はいくつかの段階で実行されます。

偶然のマトリックスの構築


一致行列（M）は2次元の行列で、その列数はデータ文字列の長さに対応し、行数は検索文字列の長さに対応します。一致行列の要素は、スペース（単語区切り文字）を除いて、対応する行の文字が一致するかどうかに応じて0または1の値を取ります。
データ文字列「ABCD EF」と検索文字列「ABC」の一致行列の形式は次のとおりです。



一致行列要素m（i、j）= 1（d（i）= s（j）の場合）、Dはデータ文字列文字の配列、Sは検索文字列文字の配列、iは一致行列Mの列番号M（データ文字列の文字番号） ）、jは一致行列の行番号（検索行の文字数）です。その他の場合、m（i、j）=0。単語区切り文字（この場合、スペース）の一致は考慮されません。つまり、m（i、j）= 0 if d（i）= s（j）= '' 。

推定範囲


最もリソースを消費する操作は次のとおりです。

  • 一致行列の決定M-行列要素の数は、検索文字列の長さとデータ文字列の長さの積として定義されます。L*L;
  • データと検索文字列への射影行列の定義-行列要素の数はMPSの場合：（L+L-1）*L、MPDの場合：（L+L-1）*;
  • すべてのグループのテーブルの形成-グループの数は値L*L/ 2を超えません。
  • グループの結果セットを検索します-サイクルの反復回数は、グループの初期数を超えません。つまり、L*L/ 2です。

したがって、計算の総量は、検索文字列の長さとデータ行の長さの積の倍数になります。



ソースデータのサイズに対する計算量の直線性は、アルゴリズムの実用的なアプリケーションに有利な重要な引数です。

非線形性


線形性は、結果として得られるグループのセットを見つけるための簡略化された手順によるものであることは注目に値します。一般的なケースでは、データ行にグループを配置するためのすべての可能なオプションを考慮し、可能な交差処理オプションを考慮して、サイクルの各反復で1つのグループを選択するのではなく、可能なセットから最適なグループのセットを選択すると、計算量はサイズに関して線形でなくなります。ソースデータ。ソースデータのサイズに対する計算量の非線形依存性は、実際のアプリケーションの可能性を大きく制限します。
