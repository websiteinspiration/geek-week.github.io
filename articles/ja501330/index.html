次のステップは、リクエストを処理できるサービステンプレートを作成することでした。
Poco :: Util :: ServerApplicationからTemplateServerAppクラスを継承し、mainメソッドをオーバーライドする必要があります。

TemplateServerApp 
#pragma once

#include <string>
#include <vector>
#include <Poco/Util/ServerApplication.h>

class TemplateServerApp : public Poco::Util::ServerApplication
{
    protected:
        int main(const std::vector<std::string> &);
};

int TemplateServerApp::main(const vector<string> &)
{
    HTTPServerParams* pParams = new HTTPServerParams;

    pParams->setMaxQueued(100);
    pParams->setMaxThreads(16);

    HTTPServer s(new TemplateRequestHandlerFactory, ServerSocket(8000), pParams);

    s.start();
    cerr << "Server started" << endl;

    waitForTerminationRequest();  // wait for CTRL-C or kill

    cerr << "Shutting down..." << endl;
    s.stop();

    return Application::EXIT_OK;
}


メインメソッドでは、ポート、スレッド数、キューサイズなどのパラメーターを設定する必要があります。そして最も重要なこととして、着信要求ハンドラを指定する必要があります。これは、ファクトリーの作成を通じて行われます。

TemplateRequestHandlerFactory 
class TemplateRequestHandlerFactory : public HTTPRequestHandlerFactory
{
public:
    virtual HTTPRequestHandler* createRequestHandler(const HTTPServerRequest & request)
    {
        return new TemplateServerAppHandler;
    }
};


私の場合、毎回同じハンドラを作成するだけです-TemplateServerAppHandler。ビジネスロジックを配置できるのはここです。

TemplateServerAppHandler 
class TemplateServerAppHandler : public HTTPRequestHandler
{
public:
    void handleRequest(HTTPServerRequest &req, HTTPServerResponse &resp)
    {
        URI uri(req.getURI());
        string method = req.getMethod();

        cerr << "URI: " << uri.toString() << endl;
        cerr << "Method: " << req.getMethod() << endl;

        StringTokenizer tokenizer(uri.getPath(), "/", StringTokenizer::TOK_TRIM);
        HTMLForm form(req,req.stream());

        if(!method.compare("POST"))
        {
            cerr << "POST" << endl;
        }
        else if(!method.compare("PUT"))
        {
            cerr << "PUT" << endl;
        }
        else if(!method.compare("DELETE"))
        {
            cerr << "DELETE" << endl;
        }

        resp.setStatus(HTTPResponse::HTTP_OK);
        resp.setContentType("application/json");
        ostream& out = resp.send();

        out << "{\"hello\":\"heh\"}" << endl;
        out.flush();
    }
};

また、PostgreSQLを操作するためのクラステンプレートも作成しました。テーブルを作成するなど、単純なSQLを実行するために、ExecuteSQL（）メソッドがあります。より複雑なクエリやデータの取得については、GetConnection（）を介して接続を取得し、libpg APIを使用する必要があります。（多分私はこの不正を修正します）。

#include <memory>
#include <mutex>

class Database
