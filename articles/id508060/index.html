Kotlin vs Jawa

Kotlin adalah bahasa pemrograman yang dikembangkan oleh Jetbrains , yang terkenal dengan IntelliJ, WebStorm, PhpStorm, dan plugin ReSharper untuk Visual Studio. Mereka mulai mencari bahasa pemrograman baru yang akan lebih baik daripada Java, tetapi secara fungsional masih kompatibel dengannya. Terinspirasi oleh bahasa Jawa, C #, Scala, dan Groovy, orang-orang Jetbrains membentuk sebuah tim untuk mengembangkan bahasa pemrograman baru. Kotlin dirancang oleh orang-orang yang mengalami semua rasa sakit Jawa.



Jadi, apa itu Kotlin?



Kotlin pertama kali diperkenalkan pada tahun 2011, dan pada Februari 2016 rilis versi 1.0 yang stabil muncul, kemudian 1,1 pada bulan Maret. Bahasa pemrograman open source dikompilasi dalam JVM (Java Virtual Machine), Android dan JavaScript. Dengan demikian, Kotlin dapat digunakan secara bersamaan pada perangkat JVM dan Android (interoperabilitas). Itu juga dapat dijalankan di frontend menggunakan JavaScript. Google secara resmi mengumumkan pada konferensi I / O pada bulan Mei bahwa Kotlin menjadi bahasa yang didukung secara resmi untuk pengembangan Android. Sejak itu, minat terhadap bahasa, aplikasi dan komunitasnya telah tumbuh secara signifikan.

Mediaan Kotlin



Setelah mengunjungi laporan Kotlin lain tentang Droidcon pada 2015 dan 2016 di London dan GDG DevFest 2016 di Amsterdam, tim pengembangan seluler kami memutuskan sudah waktunya untuk melihat bahasa baru. Kami mulai menggunakannya pada Oktober 2016 dan jatuh cinta padanya. Proyek Android baru pertama sudah 100% ditulis di Kotlin. Sejak itu, kami belum kembali ke pengembangan Android di Jawa.



Sekarang, ketika kita melihat kembali pengalaman kita dengan Java, dan melihat bagaimana Kotlin digunakan sekarang, kita dapat menarik kesimpulan berikut:



  • Kode lebih singkat, yaitu, Anda dapat menulis dan mempertahankan lebih sedikit kode;
  • Kode lebih aman karena bahasa membuat Anda memikirkan (tidak) mutabilitas, nullability, dll. saat menulis kode;
  • Di atas juga mempercepat pembangunan. Menjadi lebih mudah bagi Anda untuk melakukan hal-hal kompleks.
  • Dukungan IDE luar biasa (kami menggunakan Android Studio);
  • Komunitas yang besar dan berkembang. Perpustakaan yang ada didukung atau dimigrasi ke Kotlin, Internet penuh dengan posting blog, video, presentasi dan, tentu saja, pertanyaan / jawaban di

    StackOverflow.
  • Kotlin siap diproduksi. Kami menggunakannya dalam produksi serta banyak perusahaan seperti Square, Pinterest, Trello dan Google.


Sekarang Google telah mengumumkan bahwa Kotlin adalah bahasa resmi untuk pengembangan Android, dan Anda praktis tidak punya alasan untuk tidak melihat bahasa ini dan mulai menggunakannya.



Jadi, akankah Anda mulai dengan Kotlin atau pergi ke sana?



Ada banyak tautan yang akan membantu Anda dalam perkembangannya:



  • Situs resmi ;
  • Dokumentasi (manual, panduan, buku, dll.);
  • Pemrograman online ;
  • Pengembangan Android ;
  • Pidato, blog, dan presentasi di YouTube, Medium, dan SlideShare.


Selain itu, plug-in untuk Android Studio memiliki kemampuan untuk dengan cepat mengkonversi file Java terbuka Anda saat ini ke Kotlin. Ini bekerja dengan cukup baik. Anda mungkin perlu mengubah beberapa hal kecil dalam kode, tetapi Anda akan langsung mendapatkan versi Kotlin dari kode Java Anda untuk perbandingan dan belajar. Ada kemungkinan lain yang menakjubkan: ketika Anda bekerja di file di Kotlin dan menempelkan sepotong kode Java ke dalamnya, secara otomatis dikonversi ke kode Kotlin ketika Anda memasukkannya.



Untuk menggunakan Kotlin dalam proyek yang ada atau untuk sepenuhnya mentransfer proyek yang sudah ada ke sana, kami merekomendasikan pendekatan berikut:



  • Gunakan interoperabilitas (Kotlin dan Java dapat bekerja berdampingan);
  • Gunakan plugin untuk mengonversi file dan cuplikan kode.
  • Tulis semua fitur baru di Kotlin;
  • Semua yang Anda edit, konversi dari Jawa ke Kotlin.


Tentang masa depan



Selain mendukung JVM, Android, dan JavaScript, Kotlin bekerja mendukung lebih banyak platform. Dukungan kode mesin adalah langkah besar berikutnya. Misalnya, Anda sudah dapat menjalankan kode pada RaspBerry Pi (dalam versi beta). Jetbrains bekerja untuk menambahkan dukungan untuk iOS, MacOS, dan Windows. Ini berarti bahwa suatu hari nanti Kotlin akan menjadi solusi lintas platform baru untuk aplikasi. Anda dapat mempelajari lebih lanjut tentang pengembangan lebih lanjut di KotlinConf , konferensi mereka sendiri di akhir tahun ini di San Francisco.
