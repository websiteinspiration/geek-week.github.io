ついにHTMLに注目を集めた


ブラウザの発明以来、ほとんどすべてのコントロールのデザインと機能は変わっていないと思われますが、モバイルデバイスへの移行などにより、ブラウザの使用方法は大きく異なりました。ついに気づきました！彼らのプレゼンテーションでは、「HTMLはまだ終わっていません」と、ニコールとグレッグはChromeに間もなく登場するいくつかの革新について話しました。

リサイクルされた<form>要素


最後に、いくつかのフォームの外観を変更します！それらはより近代的になっただけでなく、モバイルデバイスやバリアフリーアクセス用に最適化されています。たとえば、日付を選択するときに値を大きくしました。

CSSも改善されています！


CSSはそのような必要な改良を受け取ります。スタイルの未来についての豊富な講義で、ウナとアダムは12の新しいCSS機能について話しました。Houdiniのような完全に新しい機能はたくさんありますが、本当に興奮したのは、CSSをよりシンプルで効率的にするための改良点です。

セレクター：is（）


セレクターを:is()使用すると、複数の同一のセレクターをグループ化できます。これは例によって最もよく説明されます。

原則として、セレクターの一部を共有する複数の要素に同じスタイルを適用する場合は、毎回セレクター全体を入力する必要があります。

body.theme--light button,
body.theme--light a,
body.theme--light p,
body.theme--light h1 { 
    /* ... */ 
}

を使用:is ()すると、セレクタのさまざまな部分をグループ化できます。

body.theme--light :is(button, a, p, h1) { 
    /* ... */ 
}

すごいね！

論理プロパティ


論理プロパティは、左から右に書くことから、より不可知論的なアプローチへの移行です。左側、右側の概念の代わりに、ブロックモデルの上部と下部は、特性block-start、inline-end、block-end、inline-startを使用します。


図：Una Kravets

最初は理解するのが難しいかもしれませんが、少し考えれば、すべてが理にかなっています。多くの場合、左にインデントするように指定すると、実際には左ではなく組み込みブロックの先頭でインデントする必要があります。新しい論理プロパティを使用して、スタイルはユーザーの言語に合わせて調整されます。まあ、左が左を意味する場合、このオプションも残ります。

適応ダウンロード


私は常に、コンテンツを端末デバイスの機能に合わせて調整するというトピックに興味を持っていました。Eddie Osmani は「アダプティブダウンロード」の概念を導入しました：ユーザーのデバイス/ネットワーク/ブラウザの機能に応じてユーザーに適応します。

これにはいくつかのAPIがあり、ユーザーはデバイスのタイプ、ネットワーク、ブラウザを知ることができます。

たとえば、Network Information APIを使用して、ユーザーが4G経由で接続されているかどうか、およびトラフィックを保存するための設定が有効になっているかどうかを判断できます。

const network = navigator.connection.effectiveType;
// => 4g

const isOnSaveData = navigator.connection.saveData;
// => true

これらの機能のサポートがまだ普及していないことは注目に値します。ただし、それらを使用して、利用可能な追加情報を取得することはできます。
