Funktionelles FizzBuzz auf Scala

FizzBuzz ist ein bekanntes Puzzle, das bei Interviews spielerisch oder wenig gefragt ist. Es gibt viele Optionen für die Implementierung, selbst für ein so einfaches Spiel. Es gibt sogar Meisterwerke wie FizzBuzzEnterpriseEdition .


Ich mache Sie auf eine andere Option aufmerksam, nicht ganz auf Freitag, sondern auf Samstag: FizzBuzz on Scala, funktionaler Stil.


Aufgabe


Für Zahlen von 1 bis 100 müssen Sie anzeigen


  • Fizz, wenn die Zahl durch 3 teilbar ist;
  • Buzz, wenn die Zahl durch 5 teilbar ist;
  • FizzBuzz, wenn die Zahl durch 3 und 5 geteilt wird;
  • sonst die Nummer selbst.

Entscheidung


Ein Programmierer sollte ein Problem weniger lösen als vielmehr ein Werkzeug erstellen, um es zu lösen

Beginnen wir mit der Teilbarkeit


def divisibleBy(n: Int, d: Int): Boolean = n % d == 0

divisibleBy(10, 5) // => true

Nein, das passt nicht zu uns - schließlich ist Teilbarkeit nicht nur eine Eigenschaft von Typennummern Int, wir werden die Teilbarkeit in allgemeiner Form beschreiben, sondern zum einen zu einem Infix-Operator machen (hier und im Folgenden werden einige Funktionen der Katzenbibliothek verwendet ):


import cats.implicits._
import cats.Eq

implicit class DivisionSyntax[T](val value: T) extends AnyVal {
  def divisibleBy(n: T)(implicit I: Integral[T], ev: Eq[T]): Boolean = {
    import I._
    (value % n) === zero
  }
  def divisibleByInt(n: Int)(implicit I: Integral[T], ev: Eq[T]): Boolean =
    divisibleBy(I.fromInt(n))
}

10 divisibleBy 5 // => true
BigInt(10) divisibleBy BigInt(3) // => false
BigInt(10) divisibleByInt 3 // => false

Hier werden verwendet:


  • Typklasse " Integral" erfordert den Typ " T", um den Rest der Division berechnen zu können und den Wert " zero" zu haben.
  • Typklasse " Eq" erfordert, dass der Typ " T" seine Elemente vergleichen kann (der Operator " ===" ist seine Syntax)
  • "T" extension methods & value classes, - ( dotty, )
