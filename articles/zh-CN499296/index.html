演示用英语（结构，短语，问答，语法，技巧）



冲刺结束时的演示（无论是内部的，对客户的还是对大客户的演示）对于非母语的人来说都是一次真实的英语测试，如以下简短演示所示：

  • 口语流利度（流利度）
  • 准确性 -语音中通常有很多错误
  • 语言的自发性（尤其是在询问问题时）
  • 发音
  • 语法水平
  • 财富或贫穷词汇
  • 一个人如何正确地知道如何做句子（俄语和英语句子的结构不同）
  • 有能力利用结构讲话连接词（连接设备：如首先，其次，最后，此外，有什么顾虑，等等）。

与句子的结构怎么办？


在英语中，有一个严格的句子结构-并非总是可以根据需要重新排列单词（在俄语中，这种成功非常成功）。

需要保留：这里我没有考虑“ 倒置” -间接单词顺序，其目的是强调一些信息或将戏剧引入叙事中。例如，我几乎没有意识到我冒着巨大的风险。我再也不会使用该服务！
倒置最常用于文学（书）英语，在通常情况下，您很少听到这样浮夸的句子。虽然，当然知道很有用。

最常用的句子结构是：

1）主语+动词+宾语（主动承诺）
例如，我们添加了新功能。

3d形式的对象+待办事项+动词（被动承诺）
例如，已添加新功能。

2）There + TO BE动词
Eg 应用程序中有一个新功能。

3）+成为动词+调整 +不定式
例如，必须添加一个新功能。

在俄语中，句子结构不是那么僵硬，可以在句子中更容易地重新排列单词，并且在大多数情况下，这在语法上是正确的。

你可以说：

  • 昨天下午5点，我们添加了一项新功能。
  • 昨天下午5点增加了一项新功能。
  • 昨天下午5点增加了一个新功能。

等。

被动很少使用。“功能已添加”，您将不会经常听到。

因此，您不必依赖俄语的句子结构，而先用俄语编写演示文本，然后将其翻译成英语。因此，可能会有更多的错误。您必须立即考虑（或什至更好地）用英语演示示例文本。

现在完美现在，

IT专业人员经常在演示过程中使用“过去简单”。但是，该演示是为什么需要使用Present Perfect的完美示例。 Present Perfect的定义与该演示的目的相吻合：
在演示期间，过去的动作在present中具有可见的结果 = ，在演示期间，您需要演示团队为sprint做的事情。

例如，我们开发了一项新功能。 （活动语音）
已经开发了新功能。 （被动语态）

被动语态

被动语态通常用英语使用，关于俄语不能说。这就是为什么俄语使用者不愿意使用被动语态的原因。

对于演示而言，被动语音非常有用，因为您无需经常说“我/我们”。而且他给出了演讲的形式和客观性，结果更重要，而不是谁来做。自然地，当您需要确切指出执行此任务或任务的人时，您需要主动的声音。但是被动式也不必忽略。

例如，该功能已经过测试。错误已得到修复。

情态动词

可能，可能，可能，应该，必须，必须 -这些情态动词将有助于更清晰地表达思想。

例如，我必须承认...

条件类型1

当然，希望知道并正确使用各种条件语句。但是，这对学生来说是一个非常困难的话题，因为很难记住所有类型，其公式，何时使用哪种类型以及如何正确使用它们。仅考虑条件类型1，因为对于演示而言，它比其他类型更相关。在条件类型1（实际条件）中，虽然我真的想使用future，但因为在俄语中使用了future，所以在if之后将不会设置if。但重要的是要记住以下公式：

如果+现在简单，将来简单

如果您没有问题，我将继续进行议程的下一项。
如果您查看屏幕，则会看到产品的新界面。

文章

IT专家以及大多数讲俄语的学生不喜欢或看不见文章中的实际含义，因此经常在演讲中避免使用它们。

如果您不考虑例外情况（例外情况将另写文章）并尽可能简化所有规则，则可以得出以下生活技巧：

1）如果您可以说一个（一个）或任何一个（任何），那么您可以计算这个名词，这意味着它是唯一可数的包括文章a / an。

例如，给我一个主意（一个主意，任何主意）。我有一个计划（一个）。

2）如果可以放this / that（that，this）意思是一个具体的名词，需要冠词the。

你能解决这个问题吗？（这个问题）您如何看待演示？（该演示）

开始演示的短语


在军械库中使用现成的短语会很有用，这将有助于您在逻辑上清晰地表达自己。此外，观众可以更轻松地跟随演示，而不会失去叙事的线索。

您可以使用以下表达式开始演示：

  • 嗨，大家好！我的名字是 ... 我是（团队名称）的团队负责人。
  • 议程上的问题如下：...
  • 让我们开始吧？
  • 让我们开始吧！
  • 让我们开始滚球。
  • 让我继续第一个功能。/让我们看一下第一个功能。它的功能如下。
  • 首先/开始/对于初学者/首先/首先，我会谈论...

从演示的一部分移至另一部分的短语


在演示过程中，应尽可能清楚，简洁地陈述所有内容。从一个点跳到另一个点不是一个好主意。为了使听众了解何时对一个功能的讨论已经结束而又是对另一个功能的问题，有必要使用以下短语来表达这一点：

  • 让我们继续下一个功能。让我告诉你它是如何工作的。
  • 此外，我们还开发了以下功能。这就是它的工作方式。
  • 如果您没有任何问题/意见，我将继续下一个功能。现在，我将说明它是如何工作的。

如何优雅地完成演示？


观众总是记得任何演讲的开始和结束。因此，不仅要有尊严地开始和呈现所有信息，而且还必须正确地完成演示。

  • 最后但并非最不重要的一点，我想演示以下功能。
  • 让我们回顾一下/让我总结一下主要功能。
  • 我想就是这样。
  • 感谢您的关注。随意问的问题。
  • 我愿意提出建议。
  • 感谢您的评论

如何感谢团队或个人？


  • 特别要感谢克里斯。他负责...
  • 我感谢整个团队的合作。
  • 这要归功于格雷戈里（Gregory），他...

如何处理问答环节？


  • 我准备回答任何问题。
  • 如果您有任何疑问，请随时提问。

如果您被问到一个棘手的问题，却又不知道该如何回答，那就不要说我也不知道，因为这听起来并不合算。至少，有必要说明为什么这个问题很困难并且目前仍未得到回答。

最好说：

  • 这是一个很好的问题！谢谢。我将咨询我的团队，并给您答复。
  • 好问题！目前，我无法为您提供答案，因为它需要进行一些研究。我明天（通过电子邮件等）将答案发送给您。
  • 我现在可以回答你，但这并不准确。我需要仔细检查/验证一些信息，以后再与您联系。

哪些短语动词可以解救？




短语动词经常用在母语人士的语音中，而对于非母语人士则不能这样说。问题是它们很多，要记住所有这些多样性是非常困难的。

以下短语动词特别适合演示。我建议您记住并使用。

  • 挂断电话=挂断电话，挂断电话
  • 给某人回电=回电
  • 移至下一个点=移至下一个点
  • 减速=减速
  • 大声说=大声说
  • 总结=总结
  • 想出=想出（例如问题/答案/解决方案）
  • 此功能将在将来获得回报。=将还清
  • 解决问题=应付/处理问题=解决问题
  • 开始=开始
  • 坚持=坚持=等待

出现技术问题（不仅是技术问题）时的一般用语


奇怪的是，技术（不仅是）问题经常发生。要摆脱困境，请记住以下短语：

  • 对不起，我没听清楚那个名字/日期等等。您能否重复输入最后一个字/名称等？
  • 恐怕连接不好/不好。让我打电话给您回电。
  • 请你放慢脚步好吗？我没有关注您/您讲得太快了。
  • 你能说出来吗？你说得太安静了。
  • 抱歉，我们之前没有遇到过。我们将尽快解决该问题。
  • 我们将优先考虑此问题。
  • 我们将首先解决它。
  • 糟糕，屏幕冻结。让我修复它。
  • 给我一点时间-我将重新加载产品/重新打开功能。
  • 等一下 似乎有一个小故障。
