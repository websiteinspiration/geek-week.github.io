Debuggen Sie den PHP-Container mit Xdebug und PhpStorm

Vor Beginn des Backend PHP Developer- Kurses wurde eine Übersetzung des Artikels erstellt .









Docker-Anweisung Nr. 9: xdebug



Ich werde eine sehr einfache PHP-Seite erstellen und sie mit xdebug und PhpStorm ausführen.



Die Quelldateien finden Sie hier:

github.com/ikknd/docker-study im Ordnerrezept-09



1. Erstellen Sie eine "Docker-Datei" im Ordner "Docker": 



FROM php:7.2-fpm

#Install xdebug
RUN pecl install xdebug-2.6.1 && docker-php-ext-enable xdebug

CMD ["php-fpm"]


Führen Sie diesen Befehl im Docker-Ordner aus, um das Image zu erstellen: 



docker build -t php-xdebug-custom -f Dockerfile .


2. Erstellen Sie die Datei docker-compose.yml im Docker-Ordner: 



version: "3.7"

services:

  web:
    image: nginx:1.17
    ports:
      - 80:80
    volumes:
      - /var/www/docker-study.loc/recipe-09/php:/var/www/myapp
      - /var/www/docker-study.loc/recipe-09/docker/site.conf:/etc/nginx/conf.d/site.conf
    depends_on:
      - php

  php:
    image: php-xdebug-custom
    volumes:
      - /var/www/docker-study.loc/recipe-09/php:/var/www/myapp
      - /var/www/docker-study.loc/recipe-09/docker/php.ini:/usr/local/etc/php/php.ini


Hier verwende ich das benutzerdefinierte Bild php-xdebug anstelle von php: 7.2-fpm „



3. Nehmen Sie die folgenden Einstellungen in der Datei php.ini vor: 



[xdebug]
zend_extension=xdebug.so
xdebug.profiler_enable=1
xdebug.remote_enable=1
xdebug.remote_handler=dbgp
xdebug.remote_mode=req
xdebug.remote_host=host.docker.internal
xdebug.remote_port=9000
xdebug.remote_autostart=1
xdebug.remote_connect_back=1
xdebug.idekey=PHPSTORM


4. Konfigurieren Sie den Server in PhpStorm:



Datei -> Einstellungen -> Sprachen und Frameworks -> PHP -> Server

Fügen Sie mithilfe des Symbols + einen neuen Server hinzu und konfigurieren Sie ihn wie im folgenden Screenshot gezeigt:







Stellen Sie sicher, dass Sie "Pfadzuordnung verwenden" auswählen ( "Pfadzuordnungen verwenden") und ordnete den PHP-Ordner zu /var/www/myapp



.



5. Konfigurieren Sie den PHP-Remote-Debugger in PhpStorm:



Ausführen -> Konfigurationen bearbeiten -> PHP-Remote-Debuggen



Fügen Sie eine neue Konfiguration hinzu und weisen Sie ihr Werte zu, wie im folgenden Screenshot dargestellt:







6. Wählen Sie die Debug-Konfiguration im PhpStorm-Debug-Bereich aus. 7. Gehen Sie zu und führen Sie Folgendes aus:







/var/www/docker-study.loc/recipe-09/docker/ 







docker-compose up -d


Wenn ich myapp.loc/



jetzt den Browser eingebe, werden die Ergebnisse aus der Datei angezeigt index.php



.



Ich kann einen Haltepunkt festlegen, im PhpStorm-Debug-Bereich nach Verbindungen suchen und die Seite neu laden.



Hab ein gutes Debug!
