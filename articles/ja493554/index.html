Windowsターミナルプレビューv0.10

Windowsターミナルv0.10の紹介！いつものように、Microsoft StoreまたはGitHubのリリースページからダウンロードできます。猫の下で、更新の詳細を詳しく見ていきます。



マウス入力


ターミナルは、Linux（WSL）アプリケーション用のWindowsサブシステム、および仮想ターミナル（VT）入力を使用するWindowsアプリケーションでのマウス入力をサポートするようになりました。つまり、tmuxやMidnight Commanderなどのアプリケーションは、ターミナルウィンドウ内のアイテムのクリックを認識します。アプリケーションがマウスモードの場合、VT入力を送信する代わりにシフトを押して選択を行うことができます。



設定を更新


パネルの複製


これで、選択したパネルからプロファイルを複製し、それに焦点を合わせてキーの組み合わせを押すだけで、新しいパネルを開くことができます。これを行うには、profiles.jsonの「keybindings」セクションで、「splitMode」：「duplicate」を「splitPane」に追加する必要があります。「commandline」、「index」、「startingDirectory」、「tabTitle」などの他のパラメーターを使用できます。これらのオプションについて詳しく知りたい場合は、この記事を読むことをお勧めします。

{"keys": ["ctrl+shift+d"], "command": {"action": "splitPane", "split": "auto", "splitMode": "duplicate"}}




エラー訂正


  • ウィンドウのサイズ変更時のテキスト表示が大幅に改善されました。
  • 暗いテーマの境界線は固定されています（白ではなくなりました）。
  • タスクバーが非表示でターミナルが展開されている場合は、画面の下部にカーソルを合わせると自動的に表示されます。
  • Azure Cloud ShellはPowerShellを起動し、マウス入力をサポートできるようになりました。また、優先シェルとしてインストールすることもできます。
  • タッチパッドまたはタッチスクリーンを使用するときのスクロール速度が変更されました。

今後の計画


今後数か月で何が期待できるかがわかるように、計画に関する最新情報を提供したいと思います。現在、v1のリリースに備えてバグ修正に取り組んでいます。Windowsターミナルv1自体は5月にリリースされます。その後、6月に次のアップデートをリリースし、毎月のアップデートサイクルを継続する予定です。私たちのエディションは引き続きMicrosoft StoreとGitHubで入手できます！
