Arduinoとキーボード（完全ガイド）

こんにちは、Habrの住民です。今日、私はArduinoの完全なキーボードガイドを作成することにしました。
注意！記事は主に初心者を対象としています！

多くのプロジェクトでは、ユーザー入力機能を作成する必要があります。多数のボタンを（比較的）実装する必要がある場合、ボタンを個別に接続すると、長くて不合理になります。この目的のために、キーボードを選択することをお勧めしますが、他の選択肢と同様に、意識的にそれに近づく必要があります。キーボードの種類とその操作方法を教えてください。

全体として、このタイプのすべてのモジュールは、マトリックスとアナログの 2つの大きなカテゴリに分類できます。

スケッチライティング


まず、ライブラリを使用してコード内のキーボードを接続できるようにするには、ライブラリをデバイスにインストールする必要があります。これは次のように実行できます。ArduinoIDE自体で、[ スケッチ ] -> [ライブラリの接続 ] -> [ライブラリの管理 ]をクリックするか、キーボードショートカット「Ctrl + Shift + I」を使用します。「keypad.h」（角括弧なし）の検索ドライブで、「インストール」を押します。

インストール後、スケッチの作成に進みます。

#include <Keypad.h> //     

const byte ROWS = 4; //    
const byte COLS = 4; //

{'1','4','7','*'},

byte rowPins[ROWS] = {5, 4, 3, 2}; //     
byte colPins[COLS] = {9, 8, 7, 6}; //     

//    :

接続は非常に簡単ですが、特定のキーボードによって異なる場合があります。この記事では、この中国語アナログキーボードをAli で使用する例を検討します。

スケッチライティング


サードパーティのライブラリを使用する必要はありません。すべてが非常にシンプルで明確です。以下は、押されたボタンを読み取ってその番号を1〜16（左から右、上から下）またはポートモニターに何も押されていない場合は0を表示するコード自体です。

unsigned const short int KEYPAD_PIN = 0;//   

void setup() {
  Serial.begin(9600);
  analogReadResolution(10);//      0...1023
}

void loop() {
  Serial.println(getPressedKeypadButton());//      
}

short int getPressedKeypadButton(){//  keypad`a
//   1  16      0    
  const int OCCURACY = 5;//    
  int sensorValue = analogRead(KEYPAD_PIN);//   
  int keys[] = {1016, 937, 860, 794, 683, 641, 604, 571, 509, 485, 464, 444, 407, 328, 275, 236};//    (0-15,  0=1, 1=2  ..)
  if(sensorValue > -OCCURACY && sensorValue < OCCURACY){return 0;}//   ,    0
  for(short int i = 0; i < 16; i++){//            
    if(sensorValue > (keys[i] - OCCURACY) && sensorValue < (keys[i] + OCCURACY)){
      return i+1;//   , ..     0   ,    0   
    }
  }
}

概要


個人的には、アナログキーボードを使用することをお勧めします。アナログキーボードは、多くの場合、ほとんどの場合より優れており、接続は簡単です。主なことは、製造元から提供された、押されたボタンの値の表に注意を払うことです。存在しない場合は、これらの測定を自分で実行して、ポートモニターのアナログポートから値を表示し、後でコードで使用できます。ただし、選択は常にあなた次第です。標準的な安価なオプションを使用するか、状況によっては大きな利点を得るために過剰に支払うことです。
