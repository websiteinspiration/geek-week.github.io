MS-DOSでC＃でゲームを起動する

MS-DOSでC＃で64ビットのゲームを実行できないことにいつも悩まされていました。今日私はそれを修正しました。



Windows実行可能ファイルは2つの部分で構成されています。

  1. 「このプログラムはDOSモードでは起動できません」と表示するDOS用のプログラム
  2. Windows 実行可能ヘッダー

ある意味で、すべての.exeファイルはDOSプログラムですが、有用なものは何もありません。ですので、ある日、Githubで、それよりもはるかに多くの星に値するプロジェクトを見つけました：

github.com/Baron-von-Riedesel/Dos64-stub

Dos64-stubは、「開始できませんDOSの下で「実行可能ファイルのWindowsセクションをダウンロードし、21世紀に「テレポート」するプロセス。 「テレポーテーション」とは、ページメモリを設定し、プロセッサを64ビットモード（「ロング」）にすることを意味します。

開始するには、Windowsでゲーム「Snake」を使用しましたが、最近、依存関係なしで最大8 kbになりました。



もちろん、通常のWindows APIメソッドはDOSでは利用できないため、ゲームの通信レイヤーを外部と書き換える必要がありました。ここにありますEnvironment.TickCount：

public static unsafe long TickCount64
{
    [MethodImpl(MethodImplOptions.NoInlining)]
    get
    {
        //    BIOS -   1AH.
        // ,         "" .
        //   BIOS -   ,
        //  BIOS    .
        //   DOS   .
        //   0x46C       55.
        uint timerTicks = * (uint *) 0x46C;
        return (long) timerTicks * 55;
    }
}
