春の前の時代


Apache Tapestryは、Webアプリケーションを構築するためのフレームワークの中で真のロングリバーです。その最初のバージョンは2000年にリリースされました;新しいバージョンは現在リリースされ続けています。Tapestryを設計する際、数年前に登場したWebフレームワークであるWebObjectsのアイデアを借りました。Tapestryを使用するアプリケーション（例-モジュールのhelloworld-of web-the tapestry）では、ユーザーインターフェースWebページのモジュラーアーキテクチャとバインディング（バインディング）コンポーネントをそれぞれのJavaクラスに使用します。

アパッチストラット2000年5月に、以前のフレームワークとほぼ同時に登場し、現在に至るまで開発が続けられています。これに基づく例（helloworld-web-strutsモジュール）ではweb.xml、デプロイメント記述子ファイルでクラスフィルターとして表示を確認できますStrutsPrepareAndExecuteFilter。このクラスは、ディスパッチャー要求を提供し、対応するアクション要求（action）を選択します。Apache Strutsは、Tapestryと同様に、MVC設計パターンに基づいています。

2004年3月に、公開された仕様JavaServer Facesとその連続する2つの実装：Eclipse Mojarra（以前の名前-Sun JSFリファレンス実装、JSF RI、Mojarra、Oracle Mojarra）およびApache MyFaces。仕様でサポートされている主なアプローチは、コンポーネントの使用です。両方の例（helloworld-web-jsf-mojarraおよびhelloworld-web-jsf-myfacesモジュール）は、依存関係ライブラリを除いて、完全に同じです。アプリケーションは、JSF仕様の実装バージョン、実装の名前（MojarraまたはMyFaces）、および実装バージョンをWebページに定義して表示します。

2005年6月とイデオロギー的に類似したWicketフレームワークの最初のバージョンのリリースTapestryとJavaServer Faces。コンポーネント指向のアプローチとWebページのHTMLテンプレートをJavaクラスにリンクします。 2007年6月以降、フレームワークはApache Software Foundationに属し、その名前はApache Wicketに変更されました。このフレームワークの人気のピークは2008年から2009年頃に来たもので、その後徐々に関心が低下しました。新しいバージョンが引き続きリリースされます。サンプルアプリケーションはhelloworld-web-wicketモジュールで確認できます。

2005年10月の最初のバージョンに来たのGrailsを、で書かれたWebアプリケーション構築するためのフレームワークJVM -languages のGroovyを。製品の名前が示すように、Ruby on Railsに、で書かれたフレームワークRubyは、その作成に強い影響力を持っていました。また、MVCパターンに基づいています。特徴的な機能は、GSP（Groovy Server Pages）ビューファイルをテンプレートとして使用することです。例（たhelloworld-ウェブのGrailsモジュールは）、作成した構築し、使用して実行することができているGrailsはMavenのプラグイン、用のプラグインMavenを。

春の到来後、2000年代


Vaadinの開発は、別のフレームワークであるMillstone 3を補完するものとして2002年に始まりました。 2006年に作成されたものは、完成した商用製品として組み立てられました。 2009年5月までは、その名前はIT Mill Toolkitでしたが、その直後にVaadinになりました。 2007年の終わりに、以前は独立して実装されていたクライアント部分は、Google Web Toolkit（GWT）に置き換えられました。この例（helloworld-web-vaadinモジュール）では、低レベルの技術的な詳細を隠しながら、すべてのユーザーインターフェイスコンポーネントがプログラムで作成されるJavaクラスファイルが1つしかないことがわかります。

非常に興味深い製品であるGoogle Web Toolkit（GWT）が2006年5月に登場しました。最新バージョンは2年前にリリースされました。サーバー部分とクライアント部分を作成するために、同じJava言語を使用する機会が与えられます。特別なコンパイラが、JavaのクライアントコードをJavaScriptに変換します。この例は、helloworld-web-gwt-client（クライアント部分）、helloworld-web-gwt-server（サーバー部分）、helloworld-web-gwt-shared（クライアント部分とサーバー部分に共通のコード）の3つのモジュールで構成されています。開発中、Mavenの便利なプラグインを使用して、クライアントパーツをモードで起動できます。スーパー開発モード。いわゆるコードサーバーにより、変更されたJavaコードを簡単に再コンパイルできます。

Seamは2007年5月に使用を開始し、2012年に存在しなくなりました。Enterprise JavaBeans（EJB3）とJavaServer Faces（JSF）に基づいていました。JBossによって開発され、当時はRed Hatの一部でした。彼はさまざまな興味深い概念を提案しました（たとえば、対応する注釈があったバイジェクション））フレームワークのサイトはまだ存在しますが、その一部のセクションでは、一部のリンクはもはや関連していません。サンプルアプリケーションはhelloworld-web-seamモジュールにあります。RESTful Webサービス（JAX-RS）

Play Frameworkは2008年5月に登場しました。Scala JVMプログラミング言語で書かれています。ScalaとJavaの両方でそれに基づいてWebアプリケーションを作成できます。Play開発者の特徴は、sbtビルドツールへの取り組みです。このため、例（helloworld-web-play module）私は、これに適切なプラグインを使用して、Mavenでビルドするためにいくつかの努力をしなければなりませんでした。

2010年代、最新の時間


2011年、Rubyで書かれたシナトラの影響下でリリースされた素晴らしいSparkマイクロフレームワークの最初のバージョンがリリースされました。簡潔さ、軽さと構文のミニマリズムはそれに固有です。例（helloworld-web-sparkjavaモジュール）は、完全なアプリケーションを数行だけで作成する方法を示しています。アプリケーションで複雑すぎるものを必要としない場合は、フレームワークの機能で十分な場合があります。2011年には、JVM上で実行されるイベント駆動型フレームワークであるVert.x が登場しました。もともとNode.xと呼ばれていたNode.jsの影響を強く受けたもの

2011年12月は、Dropwizardの存在の最初の時期でした。Dropwizardの作者は、製品をライブラリとフレームワークの間にあるものとして位置付けています。それを構成する3つの主要部分は、Jettyライブラリ（HTTP）です。ジャージー（JAX-RS）およびジャクソン（JSON）。現在も発展を続けており、人気を博しています。例（helloworld-web-dropwizardモジュール）は、DropwizardベースのWebアプリケーションの典型的な構造を示しています。

Ratpack-別のフレームワーク（Sparkを除く）。ライブラリSinatraに触発され、大部分はJVMで記述されたGroovyの言語です。フランクシナトラといわゆるラットパック（「ラットパック」）。フレームワークの最初のバージョンは2013年にリリースされ、新しいバージョンがリリースされ続けています。ライブラリに基づくNetty、高速、ミニマル、使いやすく、拡張性が高い。例はhelloworld-web-ratpackモジュールで見ることができます。
