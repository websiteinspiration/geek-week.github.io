Cuide su tarea: haga una copia de seguridad de su computadora portátil o de escritorio utilizando los agentes gratuitos de Veeam

En la vida cotidiana, yo, como muchos de ustedes, queridos lectores, trabajo en la oficina la mayor parte del tiempo, y me queda perfectamente. Pero hoy la situación es tal que tanto yo como mis colegas en diferentes partes del mundo, y creo que la mayoría de las personas que leen este artículo, todos trabajamos desde casa.

Y aquí estará bastante fuera de lugar recordar la necesidad de hacer una copia de seguridad de una computadora o computadora portátil doméstica. Para cuando cambia el entorno de trabajo, también cambia la forma habitual de trabajar con datos. Todo es obvio con el trabajo en la oficina: acceso directo a los recursos necesarios, copias de seguridad centralizadas, en todo caso, siempre puede contactar al servicio de asistencia. Con el trabajo desde casa, hay muchas cosas que necesita para usted, incluida la copia de seguridad.

Creo que muchos de ustedes ya están utilizando Veeam Agent para Microsoft Windows o Veeam Agent para Linux para esto. Si aún no lo está, quizás le interese la breve reseña de hoy. Bienvenido a gato.



En 2014, cuando se lanzó la primera versión de nuestro producto, ahora conocido como Veeam Agent para Microsoft Windows , se llamaba Veeam Endpoint Backup Free y tenía la única edición no comercial.

Por supuesto, la solución ha cambiado mucho desde entonces, y ha salido una versión comercial, en la que hay muchas características avanzadas: administración centralizada, el controlador CBT, soporte de clúster, procesamiento de disco paralelo y mucho más. Veeam Agent para Linux también se está desarrollando, lo que le permite realizar copias de seguridad de máquinas físicas y restaurar datos de las copias de seguridad. Pero ahora, en el patio de 2020, las nuevas versiones de V eeam Agent para Microsoft Windows y Veeam Agent para LinuxTambién hay una edición gratuita diseñada para computadoras individuales.

Entonces, si necesita hacer una copia de seguridad de su computadora de escritorio, computadora portátil o incluso una tableta que se ejecuta en Windows, use su salud.

Veeam Agent para Microsoft Windows: SO compatible


Se admiten versiones del sistema operativo de 64 bits y 32 bits (cuando corresponda):

  • Microsoft Windows Server 2019
  • Microsoft Windows Server 2016
  • Canal semestral de Microsoft Windows Server (incluido 1909)
  • Microsoft Windows Server 2012 R2
  • Microsoft Windows Server 2012
  • Microsoft Windows Server 2008 R2 SP1
  • Microsoft Windows 10 (de la versión 1607 a 1909)
  • Microsoft Windows 8.1
  • Microsoft Windows 7 SP1

Notas:

  • Para los sistemas operativos de servidor, se admiten las ediciones Small Business Server y Server Essentials.
  • Las versiones Server Core para sistemas de servidor en un escenario de implementación independiente no son compatibles.
  • Compatible con Windows Embedded / Windows IoT OS (excepto compilaciones personalizadas).
  • Los clústeres de conmutación por error de Microsoft son compatibles cuando Veeam Agent para Microsoft Windows ejecuta Veeam Backup & Replication (no se admite en escenarios de implementación independientes).
  • Microsoft Windows 10 Education es compatible a partir de la compilación 10586.

Los requisitos restantes del sistema se describen en detalle en la documentación (en inglés).

Trabajo de respaldo en Veeam Agent para Microsoft Windows


Para el respaldo, configure la tarea de respaldo iniciando el asistente Nuevo trabajo de respaldo . En él, seleccione el tipo de copia de seguridad:

  • Computadora completa : toda la máquina.
  • Copia de seguridad de nivel de volumen : copia de seguridad de volúmenes seleccionados.
  • Copia de seguridad a nivel de archivo : copia de seguridad de archivos específicos.

Veeam Agent para Microsoft Windows puede usar diferentes tipos de sistemas de almacenamiento como una ubicación de almacenamiento de respaldo:

  • Almacenamiento local : unidad externa (USB, Firewire o eSata). Lo principal aquí, como saben, no es hacer una copia de seguridad en el disco incorporado en la máquina.
  • Carpeta compartida : compartir SMB en un servidor de archivos o NAS.
  • Veeam repositorio de respaldo - Veeam repositorio (local).
  • El repositorio Veeam Cloud Connect es un repositorio basado en la nube proporcionado por el proveedor de servicios Veeam.
  • Microsoft OneDrive

En mi ejemplo, la copia de seguridad irá a la Unidad USB; para esto, en el paso Destino , seleccione Almacenamiento local :



Luego establecemos las configuraciones específicas para el tipo de almacenamiento seleccionado.

¡Importante! Cuando trabaje con unidades USB extraíbles, debe tener en cuenta que si dicha unidad está formateada para FAT32, entonces no admite el almacenamiento de archivos de más de 4 GB. Por lo tanto, no se recomienda utilizar discos como el almacenamiento de respaldo de destino.

A continuación, configure la programación en el paso Programar . Aquí puede especificar si desea realizar una copia de seguridad regularmente en determinados momentos o cuando se producen los eventos seleccionados:

  • Lock - cuando bloqueaste el auto
  • Cerrar sesión : cuando se cierra sesión para la cuenta actual
  • Cuando el objetivo de respaldo está conectado : cuando el almacenamiento de respaldo de destino está conectado

Si se especifica la tercera opción, también puede indicar a Veeam Agent que termine de desconectar (expulsar) el almacenamiento de destino al final de la copia de seguridad para que la copia de seguridad esté fuera de línea. Esto reducirá el riesgo de exposición a una copia de seguridad del virus de cifrado, si se detecta uno.

Para hacer esto, seleccione la opción Cuando el objetivo de respaldo esté conectado → Expulsar almacenamiento extraíble una vez que se complete el respaldo (protección de CryptoLocker) :



Veeam Agent para Linux


Veeam Agent para Linux está diseñado para proteger los datos de las máquinas que se ejecutan en Linux, y esta solución también tiene una edición gratuita. El kernel de Linux 2.6.32 y superior es compatible.

Se admiten versiones de distribuciones de 64 bits y 32 bits (cuando corresponda):

  • Debian 8.0 - 10.3
  • Ubuntu 14.04, 16.04, 18.04, 19.10
  • RHEL 6.0 - 8.1
  • CentOS 6.0 - 8.1
  • Oracle Linux 6 - 8.1 (RHCK)
  • Oracle Linux 6 (comenzando con UEK R1) - Oracle Linux 7 (hasta UEK R5 U2)
  • SLES 11 SP4, 12 SP1 - 15 SP1
  • SLES para SAP 11 SP4, 12 SP1 - 15 SP1
  • Fedora 30 - 31
  • openSUSE Leap 15.1
  • openSUSE Tumbleweed

Notas:

  • Fedora es compatible hasta la versión del kernel 5.4.14 inclusive.
  • openSUSE Tumbleweed es compatible hasta e incluyendo la versión del kernel 5.1.7.
  • El kernel de Linux 2.6.32-754.6.3 no es compatible con CentOS / RHEL y Oracle Linux (RHCK).

Para obtener más información, consulte la documentación (en inglés).

Se admiten los mismos tipos de copias de seguridad que en Veeam Agent para Microsoft Windows, incluidas las copias de seguridad a nivel de archivo. La configuración de la copia de seguridad se puede configurar siguiendo los pasos del asistente o desde la línea de comandos.

Puede encontrar más información sobre cada una de las soluciones en la guía del usuario para Windows (en inglés) o para Linux (también en inglés).

Sobre las limitaciones de la edición gratuita


Para Veeam Agent para Linux (en comparación con la edición comercial de Server):

  1. Solo se puede crear 1 trabajo de copia de seguridad.
  2. El repositorio Veeam Cloud Connect no es compatible como el repositorio de respaldo de destino.
  3. Los scripts de precongelamiento y postcongelamiento no son compatibles con el trabajo de copia de seguridad.
  4. La copia de seguridad teniendo en cuenta el trabajo del DBMS Oracle, MySQL, PostgreSQL no es compatible.

Más detalles están en la tabla aquí . Y sobre lo que está sucediendo "detrás de escena", cómo nuestros ingenieros trabajan para brindar apoyo a este "zoológico" y la compatibilidad de la solución Veeam, lea el artículo de Sergey Shtepa, desarrollador en un blog en Habré.

Las restricciones para el Agente para Windows se detallan en la misma tabla . De los principales, notamos aquí que

  1. Las copias de seguridad basadas en aplicaciones no son compatibles.
  2. El repositorio Veeam Cloud Connect no es compatible como el repositorio de respaldo de destino.
  3. Microsoft OneDrive no es compatible como almacenamiento de respaldo para máquinas que ejecutan el sistema operativo del servidor Windows.
  4. Microsoft OneDrive solo se puede usar para copias de seguridad del estado del sistema. (Para obtener más información sobre los tipos de copia de seguridad, consulte aquí ).

Espero que la solución Veeam Agent sea útil para todos los que trabajan desde la oficina en casa. Si su hogar tiene una infraestructura más extensa y necesita una administración de respaldo centralizada, puede usar la edición gratuita de Veeam Backup & Replication Community Edition: su revisión en ruso está aquí .
