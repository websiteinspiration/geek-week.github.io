Loki-Prometheusアプローチを使用したログの収集

敬礼、ハブロバイト！DevOpsの実践とツールコースの新しいセットの開始を見越して、興味深い資料の翻訳を用意しました。




この記事は、Lokiの簡単な紹介です。LokiプロジェクトはGrafanaによってサポートされており、サーバーまたはコンテナーからログを集中的に収集することを目的としています。

ロキのインスピレーションの主な情報源は、ログ管理アプローチを適用するという考えを持つプロメテウスでした：

  • データストレージ用のラベル（ラベル）の使用
  • 少ないリソース消費

プロメテウスの原則に戻り、Kubernetesのコンテキストでの使用例をいくつか示します。

プロメテウスについて一言


Lokiがどのように機能するかを完全に理解するには、一歩下がってPrometheusを少し思い出すことが重要です。

Prometheusの際立った機能の1つは、（エクスポーターを介して）コレクションポイントからメトリックを抽出し、メタデータをラベルの形式で追加してTSDB（時系列データベース、時系列データベース）に格納することです。

なぜ必要なのか


最近、プロメテウスはコンテナとKubernetesの世界で事実上の標準になりました。そのインストールは非常に簡単で、Kubernetesクラスターには最初はプロメテウスのエンドポイントがあります。Prometheusは、特定のタグを保持しながら、コンテナーにデプロイされたアプリケーションからメトリックを取得することもできます。したがって、監視アプリケーションは非常に簡単に実装できます。

残念ながら、ログを管理するためのターンキーソリューションはまだなく、自分で解決策を見つける必要があります。

  • ログを集中管理するためのマネージドクラウドサービス（AWS、Azure、またはGoogle）
  • サービス監視サービス（Datadogなど）としての監視
  • 独自のログ収集サービスを作成します。

3つ目のオプションについては、常にElasticsearchを使用しましたが、常に満足しているわけではありませんでした（特に、設定の重大度と複雑さ）。

Lokiは、次の原則に従って実装を簡素化するように設計されています。

  • 簡単に始められる
  • リソースをほとんど消費しない
  • 特別なメンテナンスなしで独立して作業する
  • バグの調査に役立つPrometheusを補完する

ただし、この単純さは、いくつかの妥協を通じて達成されます。それらの1つは、コンテンツのインデックスを作成することではありません。したがって、テキスト検索はあまり効果的ではなく、リッチでもないため、テキストの内容に関する統計情報を取得できません。しかし、Lokiはgrepと同等になり、Prometheusを補完したいと考えているため、これは欠点ではありません。

ロキローカルテスト


Lokiをローカルでテストする最も簡単な方法は、docker-composeを使用することです。docker-composeファイルはLokiリポジトリにあります。次のコマンドを使用してリポジトリを取得できますgit。

$ git clone https://github.com/grafana/loki.git


次に、運用ディレクトリに移動する必要があります。

$ cd production

その後、Dockerイメージの最新バージョンを取得できます。

$ docker-compose pull

最後に、Lokiスタックが次のコマンドで起動されます。

$ docker-compose up

Kubernetesのデプロイ


LokiコンポーネントをKubernetesにインストールする手順は次のとおりです。

  • サーバークラスタの各マシンにPromtailエージェントをデプロイするためのdaemonSet
  • 配備ロキ
  • そして最後に、グラファナの展開。

幸い、LokiはHelmパッケージとして入手できるため、簡単にデプロイできます。

Helmによるインストール


Helmは既にインストールされているはずです。プロジェクトのGitHubリポジトリからダウンロードできます。アーキテクチャに一致するアーカイブを解凍し、helmをに追加することでインストールされます$PATH。

注： Helmバージョン3.0.0が最近リリースされました。多くの変更があったため、最初に使用する前に少し待つことをお勧めします。


Helmのソースを追加する


最初のステップは、次のコマンドを使用して「loki」リポジトリを追加することです。

$ helm add loki https://grafana.github.io/loki/charts

その後、「loki」という名前のパッケージを検索できます。

$ helm search loki

結果：

loki/loki       0.17.2 v0.4.0 Loki: like Prometheus, but for logs.
loki/loki-stack 0.19.1 v0.4.0 Loki: like Prometheus, but for logs.
loki/fluent-bit 0.0.2  v0.0.1 Uses fluent-bit Loki go plugin for...
loki/promtail   0.13.1 v0.4.0 Responsible for gathering logs and...

これらのパッケージには次の機能があります。

  • loki / lokiパッケージはLokiサーバーにのみ一致
  • loki / fluent-bitパッケージでは、Promtailの代わりにfluent-binを使用してログを収集し、DaemonSetをデプロイできます
  • パッケージloki / promtailには、収集エージェントのログファイルが含まれています
  • loki / loki-stackパッケージを使用すると、LokiをPromtailと一緒にすぐに展開できます。

Lokiのインストール


LokiをKubernetesにデプロイするには、「監視」名前空間で次のコマンドを実行します。

$ helm upgrade --install loki loki/loki-stack --namespace monitoring

ディスクに保存するには、オプションを追加します --set loki.persistence.enabled = true:

$ helm upgrade --install loki loki/loki-stack \
              --namespace monitoring \
              --set loki.persistence.enabled=true

注： Grafanaを同時にデプロイする場合は、パラメーターを追加します--set grafana.enabled = true

Grafanaに接続


Kubernetesの下でGrafanaに接続するには、その下部にトンネルを開く必要があります。以下は、Grafana hearthのポート3000を開くコマンドです。

$ kubectl -n port-forward monitoring svc/loki-grafana 3000:80

もう1つの重要な点は、Grafana管理者パスワードを回復する必要があることです。パスワードは、base64形式のloki-grafanaフィールド.data.admin-userに秘密にされます。

復元するには、次のコマンドを実行する必要があります。

$ kubectl -n monitoring get secret loki-grafana \
 --template '{{index .data "admin-password" | base64decode}}'; echo

このパスワードは、デフォルトの管理者アカウント（admin）で使用します。
