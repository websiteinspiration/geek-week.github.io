Appleのネイティブプレースの使用

数年前、私はSnapchatで同様の機能を備え、かつ狭義のテーマのInstagramのようなアプリケーションになるはずのプロジェクトに取り組んでいました。プロジェクトは、iOSの1つのプラットフォーム用にのみ開発されました。当然、主な機能の開発中に写真を公開する際、クライアントは突然、写真が撮影された場所をマークする機能を追加したいと考えました。ほとんどの場合、多くはすぐにGoogleやFacebookからPlaces APIを思い出しますが、クライアントはこれらのソリューションに特定の制限があるという事実に満足していませんでした。そのため、追加のレビューの結果、Appleの別の代替案が見つかりました-CLPlacemarkは無料で、当時のドキュメントでは、毎日の使用に関する制限についての言及はありませんでした。他のプラットフォームの開発は当初はまったく計画されていなかったため、これは非常に適切なオプションのように思われました。

Appleのドキュメント番組CLPlacemarkがポイントに関する多くの詳細を提供することができ、また、そこにあるという方法でCLGeocoderが容易になり、名前の場所でデータを含む配列を返しCLPlacemark。結局のところ、それはそれほどバラ色ではありません。

この単純なシナリオでは、ジオコーダーは常にCLPlacemark配列を返しますが、問題は、この配列に複数の要素が含まれないことです。その結果、画面全体で、ニューヨーク、ニューゼランド、ニューバランスストアなどのプレースメントの大きなリストが予想されていましたが、入力した内容に必ずしも関連しない要素が1つだけ表示されました。

アレイがどのように形成され、どのように改善できるかを基に研究を始めました。パラメータの選択に影響を与える可能性があるいくつかのことを確認しました。

  1. Appleへのリクエスト元のIPアドレス。VPNがオンになっていると、結果のオブジェクトは既にVPNサーバーの場所に近くなっています。
  2. ユーザーの現在の場所。現在のユーザー座標がリクエストに送信された場合、結果ははるかに正確になります。
  3. デバイスのシステム言語。

さらなる開発コースは、デバイスの現在の場所の使用でした。

import UIKit
import MapKit
import CoreLocation

final class ViewController: UIViewController, CLLocationManagerDelegate {

    private let locationManager = CLLocationManager()
    private let request = MKLocalSearch.Request()
    private var localSearch: MKLocalSearch?
    private var region = MKCoordinateRegion()

    override func viewDidLoad() {
        super.viewDidLoad()
        
        if CLLocationManager.locationServicesEnabled() {
            locationManager.delegate = self
            locationManager.desiredAccuracy = kCLLocationAccuracyBest
            locationManager.requestWhenInUseAuthorization()
            locationManager.startUpdatingLocation()
        }
    }

    func searchPlace(_ place: String) {
        localSearch?.cancel()
        
        request.naturalLanguageQuery = place
        request.region = region
        localSearch = MKLocalSearch(request: request)
        localSearch?.start { [weak self] response, error in
            let mapItems = response.mapItems //     MKMapItem
        }
    }

    // MARK: - CLLocationManagerDelegate
    func locationManager(_ manager: CLLocationManager, didUpdateLocations locations: [CLLocation]) {
        
        guard let lastLocation = locations.last else {
            return
        }
        let span = MKCoordinateSpan(latitudeDelta: 0.5, longitudeDelta: 0.5)
        region = MKCoordinateRegion(center: lastLocation.coordinate, span: span)
    }
}

その後、確かに、引き渡しの優先順位が変更され、主に私の隣にある場所が示されました。

リストの名前をより美しくするだけです。時々、CLPlacemarkの一部のプロパティが同じ名前を持つ場合があるため、結局、あまり見栄えがよくありません：ニューヨーク、ニューヨーク、ニューヨーク。これを行うには、別の構造を作成する必要があります。これにより、リストで美しい名前が形成されます。

import Foundation
import MapKit

struct Placemark {
    
    let location: String
    
    init(item: MKMapItem) {
        
        var locationString: String = ""
        
        if let name = item.name {
            locationString += "\(name)"
        }
        
        if let locality = item.placemark.locality, locality != item.name {
            locationString += ", \(locality)"
        }
        
        if let administrativeArea = item.placemark.administrativeArea,
            administrativeArea != item.placemark.locality {
            locationString += ", \(administrativeArea)"
        }
        
        if let country = item.placemark.country, country != item.name {
            locationString += ", \(country)"
        }
        
        location = locationString
    }
}
