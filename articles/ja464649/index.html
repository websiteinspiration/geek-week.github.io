Zimbra Open-Source Editionのドキュメントコラボレーションシステム

現代のビジネスにおけるドキュメントの共同編集の重要性は、過大評価することは困難です。法務部門の従業員が参加する契約や合意を作成したり、上司の監督下で商用オファーをオンラインで作成したりする機能により、以前は多数の承認に費やされていた数千人もの時間を節約できます。そのため、Zextras Suite 3.0の主な革新の1つはZextras Docsの出現でした。これは、Zimbra Collaboration Suite Open-Source Edition Webクライアントでドキュメントとの本格的なコラボレーションを直接整理できるソリューションです。

現在、Zextras Docsはテキストドキュメント、表、プレゼンテーションとのコラボレーションをサポートしており、膨大な数のファイル形式を処理する方法も知っています。ソリューションのインターフェースはテキストエディターのインターフェースとそれほど変わらないため、企業の従業員はZextras Docsの使用に切り替えるために多くの時間を費やす必要はありません。しかし、最も興味深いのは、いつものように、「内部」です。 Zextras Docsの仕組みと、このドキュメントコラボレーションソリューションが提供できる利点をまとめて見てみましょう。



何よりも、Zextras Docsは、企業内でZimbra OSEとZextras Suiteをすでに使用しているユーザーにアピールします。このソリューションを使用すると、情報システムの数を増やすことなく新しいサービスを本番環境に導入できるため、ITインフラストラクチャの所有コストを増やすことなく実現できます。 Zextras DocsはZimbra OSEバージョン8.8.12以降とのみ互換性があることに注意してください。そのため、古いバージョンのZimbraを引き続き使用している場合は、バージョン8.8.15 LTSにアップグレードすることを強くお勧めします。長期間のサポートにより、このバージョンは最新で安全なまま数年間維持され、現在のすべての追加機能とも互換性があります。

Zextras Docsの利点には、エンタープライズインフラストラクチャへの完全な展開の可能性も含まれます。これにより、ハイブリッドサービスやクラウドサービスを使用する場合によくあるように、第三者への情報の転送が回避されます。そのため、Zextras Docsは、厳格な情報セキュリティポリシーを持つ企業や、企業内で発生するデータフローやプロセスを完全に制御することを好むシステム管理者にとって理想的です。さらに、ローカルに展開されたドキュメントコラボレーションサービスを使用すると、サービスプロバイダーで問題が発生した場合、または高速インターネットアクセスで問題が発生した場合に、クラウドサービスのアクセス不能に関連するリスクを回避できます。

Zextras Docsは、スタンドアロンサーバー、拡張機能、冬の3つの部分で構成されています。これらの3つの部分はそれぞれその役割を果たします。

  • Zextras DocsサーバーはLibreOfficeエンジンであり、Zimbra OSEとのコラボレーションと統合のために強化されています。ユーザーがアクセスするすべてのドキュメントが開かれ、処理され、保存されるのは、Zextras Docsサーバー上です。Ubuntu 16.04、Ubuntu 18.04、またはCentOS 7を実行している専用のコンピューティングノードにインストールする必要があります。ZextrasDocsサービスの負荷が十分に大きい場合は、複数のサーバーを一度に選択できます。

企業でZextras Docsを導入するには、まず1つ以上の物理サーバーまたは仮想サーバーを選択する必要があります。その後、Zextras WebサイトからUbuntu 16.04、Ubuntu 18.04、またはCentOS 7のサーバーアプリケーションディストリビューションをダウンロードし、解凍してインストールする必要があります。インストールの最終段階で、サーバーは、LDAPサーバーのIPアドレスと、新しいサーバーに関するデータをLDAPに入力するために使用されるユーザー名/パスワードのペアを指定するように要求します。インストールが完了すると、各Zextras Docsサーバーが他のすべてのインフラストラクチャノードから見えるようになることに注意してください。

Zextras Docs拡張機能はすでにZextras Suiteに含まれているため、ドキュメントコラボレーションツールにアクセスする必要があるユーザーやグループが使用できるようにするだけで十分です。Zextras Docs Zymeletは、Zimbra管理コンソールからアクティブ化できます。Zextras DocsサーバーをZimbra OSEインフラストラクチャに追加した後、Zimbraプロキシサーバーの構成を更新する必要があることに注意してください。これを行うには、/ opt / zimbra / libexec / zmproxyconfgenファイルをZimbraユーザーとして実行し、zmproxyctl restartコマンドを実行してプロキシサービスを再起動します。
