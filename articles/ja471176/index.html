VSコードでのJupyterノートブックのネイティブ編集

Python拡張機能の10月のリリースで、 Visual Studio CodeでのJupyterノートブックのネイティブ編集のサポートを発表できることを嬉しく思います！これで、.ipynbファイルを直接編集して、VS Codeの全機能を備えたJupyter Notebooksの双方向性を取得できます。

ソースコードを制御したり、複数のファイルを開いたり、IntelliSense、Git統合、マルチファイル管理などの機能を使用したりできます。それらのすべては、データサイエンティストと開発者が効果的にデータを実験して操作するためのまったく新しい方法を提供します。Python拡張機能の最新バージョンをダウンロードし、VS Code内でJupyter Notebookを作成/開くことで、今日それをすべて試すことができます。 初版から



ユーザーがVS Codeのデータ分析の分野で実践するために要求した主な機能の1つは、VS Codeで独自のJupyterノートブックを編集するためのノートブックのようなレイアウトでした。この記事の残りの部分では、これらの新しいVS Code機能について説明します。

仕事の始まり


はじめに、VS CodeでJupyterを使い始める方法を説明します。

  • 既存のJupyter Notebookファイルがない場合は、キーボードショートカットのCTRL + SHIFT + P（Windows）またはCommand + SHIFT + P（macOS）を使用してVS Codeコマンドパレットを開き、コマンドPython：Create Blank New Jupyter Notebookを実行します。
  • Jupyter Notebookファイルが既にある場合は、VS Codeでファイルを開くのと同じくらい簡単です。新しいネイティブJupyterエディターで自動的に開きます。



Jupyter Notebookを開くと、新しいセルを追加したり、セルにコードを書き込んだり、セルを起動したり、ノートブックで他のアクションを実行したりできます。

AIオートフィル


コードを記述すると、IntelliSenseはセル内でスマートなヒントを提供します。さらに、IntelliCode拡張機能をインストールして、現在のコードコンテキストに基づくよりインテリジェントな自動入力の提案を備えたAIベースのIntelliSenseを取得することにより、エディターの機能を大幅に拡張でき  ます。



変数ブラウザ


VSコードを使用するもう1つの利点は、メモ帳ツールバーの[変数]ボタンをクリックして、変数エクスプローラーとグラフビューアーを使用できることです。変数ブラウザーは、ノートブック変数の現在の状態をリアルタイムで監視するのに役立ちます。 データセットを表示し、データをフィルタリングし、グラフをエクスポートすることもできます！データを表示するためにdf.head（）を入力する必要があった時代は終わりました。





リモートJupyterサーバーへの接続


Jupyterノートブックファイルが作成または開かれると、VS CodeはデフォルトでローカルにJupyterサーバーを自動的に作成します。リモートJupyterサーバーを使用する場合も、これは簡単で、VS Codeコマンドパレットから「Specify Jupyter server URI」コマンドを使用して実装し、サーバーURIを入力します。



Pythonコードとしてエクスポート


実験をPythonアクションコードに変換する準備ができたら、上部のツールバーの[変換してPythonファイルとして保存]ボタンをクリックするだけで、Python拡張機能が機能します。次に、このPythonコードを既存のPythonインタラクティブウィンドウで表示し、Python拡張機能を引き続き使用して、コードをさらにプロダクション用に準備できます（たとえば、組み込みデバッガー、リファクタリング、Visual Studio Live Share、Gitソース管理などを使用できます）。



デバッグ中


VS Codeは、前のセクションで説明したPythonコードとしてエクスポート機能を使用したJupyter Notebookのデバッグをサポートしています。インタラクティブなPythonウィンドウにコードが表示されたら、組み込みのVS Codeデバッガーを使用してコードをデバッグできます。次のアップデートでは、Jupyterエディターにセルデバッグを追加するよう取り組んでいるので、ご期待ください。
