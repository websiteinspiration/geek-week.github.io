エレクトロンとウェビックスの友達を作る方法。パート2.ビューでアプリケーションを作成する

前書き


前の記事でWebixフレームワークをElectronに統合し、これらのフレームワークに基づいてシンプルなGUIアプリケーションを作成する問題を検討しました。この記事の目的は、他のJavaScriptフレームワークのElectronおよびWebixの助けを借りて構築されたGUIインターフェースへの統合の問題をさらに発展させることです。 「Webix」に実装されているすべてのGUI要素は、「view」パラメーターによって特徴付けられます。このパラメーターの値に応じて、1つまたは別のGUI要素が表示されます。 「Webix」の作成を可能にする「view」要素のタイプの数は、インターフェースを実装するときのタスクの約90％が重なります。残りの10％は、以前に作成されたコードまたはWebixで明示的にサポートされていないフレームワーク（ライブラリ）を統合する必要がある場合にのみ当てはまります。サードパーティのフレームワーク（ライブラリ）との統合を実装するために、「ビュー」Webix要素を一緒に作成します。

問題の定式化


グラフの動的な再描画で関数「a、b、c」のパラメーターを変更できる「y = a * sin（b）+ c」の形式の関数のグラフを作成するGUIアプリケーション「Electron + Webix」を作成します。

少額予約


「Webix」には、チャートを操作するための標準的な非常に強力なツールがあることをすぐに予約してください。私の目標は、これらのツールを補足することではなく、追加のフレームワークまたはライブラリをElectron + Webixバンドルに統合する方法を示すことです。たまたま、「D3.js」を使用してこのグラフの既製のコードができたので、それをアプリケーションに統合することにしました。

「myview」を作成


次に、新しいタイプ「myview」を定義したファイル「view_chart.js」を作成し、「VSC」で開きます。


図。6-view_chart.jsの編集の開始

次のコードを追加します。

webix.protoUI({
    name: "myview",
    $init: function () {
    },
    redraw: function (a, b, c) {
    },
}, webix.ui.view);

webix.protoUI関数は、標準要素に基づいてカスタムWebix要素を作成するように設計されています。この関数に渡されるパラメーターを検討してください。

  • name-作成されたアイテムの名前。
  • $ initは、要素が作成されたときに起動される関数です。つまり、これはコンストラクタです。
  • 再描画-関数のグラフを再描画する責任を持つあなたと私たちの関数。変数「a、b、c」のパラメーターの値を渡します
  • webix.ui.viewは、要素の親です。

webix.protoUIの詳細については 、こちらをご覧ください。
「ビュー」のフレームワークの準備ができたら、正弦グラフの描画を担当する関数をいくつか作成します。
sin関数の値を計算する関数は次のようになります：
function calcsin(a, b, c) {
    dataset = [];
    for (var i = 1; i < 360; i++) {
        dataset.push({ "x": i, "y": a * Math.sin(b * i) + c });
    };
};


ここではすべてが非常に単純です。数学関数sinにパラメーター値を代入して、結果を配列に保存します。ファイル「view_chart.js」の最後に追加します。
次に、ファイル「view_chart.js」の最後に、D3.jsフレームワークを使用してグラフを描画する関数を追加します。次のようになります。
function drawSin() {
    $(document).ready(function () {
        width = $("#mycontent").width() - margin.left - margin.right
        height = $("#mycontent").height() - margin.top - margin.bottom;
        var xScale = d3.scaleLinear()
            .domain([0, 360]) 
            .range([0, width]); 

        var yScale = d3.scaleLinear()
            .domain([-100, 100]) 
            .range([height, 0]);  

        var line = d3.line()
            .x(function (d, i) { return xScale(d.x); }) 
            .y(function (d) { return yScale(d.y); })
            .curve(d3.curveMonotoneX)

        var svg = d3.select("#mycontent").append("svg")
            .attr("float", "center")
            .attr("class", "webix_chart")
            .attr("width", width + margin.left + margin.right)
            .attr("height", height + margin.top + margin.bottom)
            .append("g")
            .attr("transform", "translate(" + margin.left + "," + margin.top + ")")

        svg.append("g")
            .attr("class", "x axis")
            .attr("transform", "translate(0," + height/2 + ")")
            .call(d3.axisBottom(xScale)); 

        svg.append("g")
            .attr("class", "y axis")
            .call(d3.axisLeft(yScale))

        svg.append("path")

            .datum(dataset) 
            .attr("class", "line") 
            .attr("d", line); 
    })

}

指定された関数の順序は次のとおり

-グラフの値を「X」と「Y」でスケーリングします。
-「X」軸、「Y」軸、およびグラフ自体を描画する「html」要素「svg」を作成します。

ウィンドウの後にグラフを引く


そして最後に、チャートが変化したときにチャートをウィンドウのサイズに合わせる関数を追加します。なぜエディターでファイル「view_chart.js」を開き、次に示すようにウィンドウの変更をキャッチするハンドラーを最後に追加するのですか？

window.onresize = function ( e ) {
  
    $("#mycontent").html("")
    drawSin()
}

この関数は、ウィンドウのサイズが変更されたときに機能します。その機能は、「mycontent」ブロックのコンテンツをクリアし、グラフを再描画すること（「drawSin（）」関数を呼び出すこと）に帰着します。drawSin（）関数が呼び出されるたびに、幅と高さの変数にmycontentブロックのサイズ（高さと幅）に関する現在の情報が割り当てられます。その後、グラフはブロックの寸法に合わせてスケールを変更します。
