いつものように、私たちはさまざまなサブグループで作業し、バグを修正し、テキストを編集し、実装経験を共有し、著者が来られなかった論文を発表しました。したがって、たとえば、最後の瞬間に、Ind / NaNを使用したstd ::形式の作業に気づき、微調整しました。 jthreadインターフェースから驚きを取り除きました。サブグループハッシュプロポーザルのペア、タプル、文字列、配列をドラッグしました。

SG1 Concurrencyサブグループでは、concurrent_unordered_mapについて多くの時間を費やしました。その結果、訪問関数は、渡された値をコンテナーからの削除からのみ、競合する変更から保護しなくなります。

concurrent_unordered_map<int, std::string> conc_map;
conc_map.visit(42, [](std::string& value) { // ,  ....
    //    OK.  OK        value.
    std::cerr << value; 

    //    OK.  OK        value.
    value += "Hello"; 
});
。
このインターフェースを使用すると、実際の同期がユーザーの肩にかかる一方で、いくつかの統計的タスクの最大パフォーマンスを圧迫するために脚を自分で撃つことがより簡単になります。

concurrent_unordered_map<int, std::atomic<int>> conc_map;
conc_map.visit(42, [](std::atomic<int>& value) { // ,  ....
    //  OK
    ++ value; 
});
