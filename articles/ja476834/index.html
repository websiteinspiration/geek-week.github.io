Red Hat OpenShift 4.2：新しいツール

2019年10月、OpenShift 4.2がリリースされました。その全体の本質は、クラウドでの作業の自動化と最適化に向けたコースを継続します。



2019年5月に、次世代のKubernetesプラットフォームであるRed Hat OpenShift 4を導入したことを思い出してください。これは、本番環境でのコンテナー化されたアプリケーションの管理を簡素化するために再設計されました。

このソリューションは、ハイブリッドクラウドでソフトウェアの自動更新とライフサイクル管理機能を備えた自己管理プラットフォームとして作成され、実績のあるRed Hat Enterprise LinuxとRed Hat Enterprise Linux CoreOSに基づいています。バージョン4.2では、開発者にとってプラットフォームをより便利にすることに重点が置かれていました。さらに、OpenShiftのバージョン3からバージョン4への移行手段を提供し、インターネットにアクセスせずに構成のサポートを実装することで、クラスター管理者向けのプラットフォームとアプリケーションの管理作業を簡素化しました。

ここの速度はどこですか？


バージョン4.2では、Kubernetesでの作業が大幅に簡素化され、開発タスク用に最適化された新しいOpenShift管理コンソールモードと、コンテナーの構築、CI / CDパイプラインの整理、サーバーレスシステムの実装のための新しいツールとプラグインが提供されます。これらすべてにより、プログラマーは、Kubernetesの機能に気を取られることなく、アプリケーションコードの作成という主要なタスクに集中することができます。

サーバーレスコンピューティング


当社の他のソリューションであるOpenShift Serverlessは、オンデマンドでレベル0まで簡単に拡大および縮小できるアプリケーションを展開および実行するのに役立ちます。 Knativeプロジェクトに基づいて構築され、Technology Previewバージョンで利用可能なこのソリューションは、対応するKubernetesオペレーターを使用して任意のOpenShift 4クラスターでアクティブ化できます。これにより、OpenShiftでのサーバーレスアプリケーションまたは機能のデプロイに必要なコンポーネントを簡単に開始およびインストールできます。バージョン4.2で登場したOpenShiftコンソールの開発者モードでは、GitからのインポートやDeployan Imageなどの標準的な開発プロセスでサーバーレスオプションを使用できます。つまり、コンソールから直接サーバーレスアプリケーションを作成できます。

CI / CDクラウドコンベヤ


継続的インテグレーションとデリバリー（CI / CD）は、ソフトウェア導入の速度と信頼性を向上させる今日の主要な開発手法です。優れたCI / CDツールキットにより、開発チームはフィードバックプロセスを最適化および自動化できます。これは、アジャイル開発を成功させるために重要です。 OpenShiftでは、従来のJenkinsまたは新しいOpenShift Pipelinesソリューションをそのようなツールとして使用できます。

Jenkinsは今日の事実上の標準ですが、コンテナCI / CDの未来をTektonオープンソースプロジェクトにリンクしています。したがって、OpenShift Pipelinesはこのプロジェクトに基づいて構築され、パイプラインとしてのコード（「パイプラインとしてのコード」）やGitOpsなどのクラウドベースのアプローチをより適切にサポートします。 OpenShift Pipelinesでは、各ステップは独自のコンテナーで実行されるため、リソースはこのステップの実行中にのみ使用されます。これにより、開発者は中央のCI / CDサーバーを使用せずに配信チャネル、プラグイン、およびアクセス制御を完全に制御できます。

OpenShiftパイプラインはまだ開発者プレビュー段階にあり、任意のOpenShift 4クラスターで使用できる適切な演算子として利用できます。Jenkinsは、OpenShiftの第3バージョンと第4バージョンの両方で使用できます。

ハイブリッドクラウドコンテナー管理


OpenShiftの自動インストールと更新により、エクスペリエンスの観点から、ハイブリッドクラウドが正規のクラウドに可能な限り近くなります。4番目のOpenShiftは、以前は主要なパブリッククラウドプラットフォーム、プライベートクラウド環境、仮想化プラットフォーム、およびベアメタルサーバーで利用可能でしたが、バージョン4.2では、このリストにさらに2つの新しいパブリッククラウドプラットフォームが追加されています-Microsoft AzureとGoogle Cloud Platform、およびプライベートOpenStackクラウド。

OpenShift 4.2インストーラーは、さまざまなターゲット環境向けに完成し、孤立した（インターネットに接続されていない）構成で動作するように初めてトレーニングされています。独自の証明書チェーン（CAバンドル）を提供する機能を備えたスタンドアロンインストールと必須のプロキシモードは、規制基準と内部セキュリティプロトコルへのコンプライアンスの確保に役立ちます。サイレントインストールモードでは、常にインターネットにアクセスできないOpenShift Container Platformの最新バージョン、または厳格なイメージテストポリシーのある環境で使用できます。

さらに、Red Hat Enterprise Linuxの軽量バージョンであるRed Hat Enterprise Linux CoreOSを使用して完全なOpenShiftスタックをデプロイすると、インストール開始後1時間未満でクラウドを準備できます。

Red Hat OpenShiftを使用すると、クラウドおよびオンプレミスインフラストラクチャでのコンテナアプリケーションの作成、デプロイ、管理のプロセスを統合できます。シンプルで自動化された高速なインストールにより、OpenShift 4.2がAWS、Azure、OpenStack、およびGCPプラットフォームで利用できるようになり、組織はハイブリッドクラウドでKubernetesプラットフォームを効率的に管理できます。

OpenShift 3からOpenShift 4への簡単な移行


新しいワークロード移行ツールにより、以前のバージョンのプラットフォームからOpenShift 4.2に簡単にアップグレードできます。古いクラスターから新しいクラスターへの負荷の転送は、今やはるかに速く、簡単で、最小限の手動操作で可能です。クラスター管理者は、ソースOpenShift 3.xクラスターを選択し、そのプロジェクト（または名前空間）にマークを付け、対応する永続ボリュームをどうするかを指定するだけで十分です。それらをターゲットOpenShift 4.xクラスターにコピーするか、転送します。この後、管理者が作業を完了するまで、アプリケーションはソースクラスター上で実行され続けます。
