CentOS 8またはRedHat 8にtftpサーバーをインストールする

今日の記事は小さいでしょう。一般に、私の意見では、すべては非常に単純ですが、CentOS8が最近リリースされて以来、そのガイドはほとんどなく、特にCentOSでのtftpについては、インターネットで率直に有害なヒントをたくさん見たので、新しいバージョンがリリースされた瞬間から状況を修正する傾向を尋ねようと思います。それでは、必要な3つの哀れな5チームのセットに取り掛かりましょう。まず、必要なすべてのパッケージをインストールします。

sudo dnf -y install xinetd tftp-server tftp vim

次に、xinetd構成ファイルを作成（または修正）して、対応するポートにアクセスするとtftpサーバーが起動するようにします。これは、標準のviエディター、ほとんどの人に最も馴染みのあるnano、またはデフォルトのviとは異なるvimを介して実行できます。構文の強調表示など、より幅広い機能があります。私は最後の段落を好むので、このエディターは上記のコマンドの最後の単語でした。システムで使用可能なエディタに満足している場合は、vimをインストールできません（ただし、追加の60 MBのディスク領域を使用することに加えて、彼は悪くなることはありません）。

sudo vim /etc/xinetd.d/tftp

ここで、テキスト挿入モード（タイプ：貼り付けて入力）に移動し、編集（キーボードのキーの挿入）を行ってから、以下の構成ファイルを選択し、ターミナルウィンドウにコピーして貼り付けます。設定

テキストを編集するときは、server_args引数に注意してください。行の終わりに、tftpによって提供されるファイルが置かれるディレクトリへのパスが設定されます。このディレクトリを、使用するディレクトリに変更します。また、このディレクトリで動作するようにSeLinuxでtftpサーバーを構成する必要があることもすぐに言いたいのですが、CentOSにはtftpサーバーの「デフォルト」構成はありません。ここでは、SeLinuxの構成については説明しません。そのため、このトピックに関する別の記事を何とか準備する予定です。

# default: off
# description: The tftp server serves files using the trivial file transfer \
#       protocol.  The tftp protocol is often used to boot diskless \
#       workstations, download configuration files to network-aware printers, \
#       and to start the installation process for some operating systems.
service tftp
{
        socket_type             = dgram
        protocol                = udp
        wait                    = yes
        user                    = root
        server                  = /usr/sbin/in.tftpd
        server_args             = -v -s /var/lib/tftpboot
        disable                 = no
        per_source              = 11
        cps                     = 100 2
        flags                   = IPv4
}

次に、Escを押してから、「：wq」と入力します。

サーバーをオンにします。

sudo systemctl start xinetd
sudo systemctl enable xinetd
sudo firewall-cmd --permanent --zone=public --add-service=tftp && sudo firewall-cmd --reload

はい、あなたはしないでくださいする必要があり、起動時にtftpdのハング、これは完全に明白ではないが、。
次に、すべてが機能することを確認する必要があります。私たちはいくつかのファイルを作成してダウンロードしようとします（別のファイルがある場合は、ここで変更します）：

echo passed > ~/test.txt
sudo mv ~/test.txt /var/lib/tftpboot
tftp 127.0.0.1 -c get test.txt
cat test.txt

「passed」という単語がコンソールに表示される場合、サーバーは機能しています。同時にローカルネットワークからアクセスできない場合-firewalldのゾーンを処理するため、パブリックゾーンにアクセスしましたが、誰もがそこからtftpにアクセスする必要はありません。

PSコメントで質問がある場合は、centos 8をfirewalldからiptablesに転送する方法を説明しますが、私見ではそれらの間に基本的な違いはありません。
