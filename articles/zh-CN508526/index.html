来吧，不洁，因为你里面没有真理！



在下面的示例中，条件为true。 



const x = { i: 1, toString: function() { return this.i++; } };
if (x == 1 && x == 2 && x == 3)
    document.write("This will be printed!");


这是由于以下事实而实现的：检查相等性时，操作数被强制转换为相同类型。在这种情况下，将在x上调用重写的toString（）方法，该方法除了返回内容外，还更改对象的状态。我们向自己指出，纯函数很棒，但是出于兴趣的考虑，我们正在尝试从示例中实现该概念。 



Trickster x = new Trickster();
if (x == 1 && x == 2 && x == 3)
    Console.WriteLine("This will be printed!");


如果定义了相应的转换，.NET允许您在可能的情况下覆盖方法和转换。 



class Trickster
{
    private int value = 1;

    public override string ToString() =>
        value++.ToString();

    public static implicit operator int(Trickster trickster) =>
        int.Parse(trickster.ToString());
}


万岁，它有效。但这使Trickster似乎适应int感到困惑。更换 



public static implicit operator int(Trickster trickster) =>
    int.Parse(trickster.ToString());


在 



public static implicit operator double(Trickster trickster) =>
    double.Parse(trickster.ToString());


无论如何，它都能正常工作。现在，Trickster和int被强制翻倍。



实际上，ToString（）的重新定义对于最终结果的实现不是必不可少的，它是为了最大程度地模仿JavaScript中的示例。定义一个强制转换就足够了。当然，不要忘记副作用的重要性。 



class Trickster
{
    private int value = 1;

    public static implicit operator double(Trickster trickster) =>
        trickster.value++;
}


而且，如果您回忆起运算符的重载，则可以实现一个既相等又不相等的对象 



Trickster x = new Trickster();
if (x == 1 && x == 2 && x == 3 && x == new[] { 1, 2, 3 } &&
    x != 1 && x != 2 && x != 3 && x != new[] { 1, 2, 3 })
    Console.WriteLine("This will be printed!");


以这种简单的方式 



class Trickster
{
    public static bool operator ==(Trickster trickster, object o) =>
        true;

    public static bool operator !=(Trickster trickster, object o) =>
        true;
}

与刺猬穿越刺猬



在JavaScript中，您可以对字符串和数字一起执行数学运算。 



var a = "41";
a += 1; // "411"
var b = "41";
b -=- 1; // 42


在C＃中，只有加法才有可能。 



var a = "41" + 1; // 411

// Compilation Error:
// Operator '-' cannot be applied to operands of type 'string' and 'int'
var b = "41" - (-1);


但是语言的爱国者没有任何障碍。通过添加一些魔术，您甚至可以使此代码“正确”运行： 



Trickster a = "41";
Console.WriteLine(a += 1); // 411
Trickster b = "41";
Console.WriteLine(b -=- 1); // 42


您只需要实现一对隐式类型转换并重载该对语句即可。 



class Trickster
{
    private string value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = s };

    public static implicit operator Trickster(int i) =>
        new Trickster { value = i.ToString() };

    public static string operator +(Trickster trickster, int i) =>
        trickster.value + i;

    public static int operator -(Trickster trickster, int i) =>
        int.Parse(trickster.value) - i;

    public override string ToString() =>
        value;
}


您可以替换附加过载 



public static string operator +(Trickster trickster, int i) =>
    trickster.value + i;


隐式转换为字符串 



public static implicit operator string(Trickster trickster) =>	
    trickster.value;


并得到相同的结果。



如果您不担心从减法运算中返回数字并将此数字放入Trickster中，则可以替换 



public static int operator -(Trickster trickster, int i) =>	
    int.Parse(trickster.value) - i;


在 



public static string operator -(Trickster trickster, int i) =>
    (int.Parse(trickster.value) - i).ToString();


并删除了转换为Trickster的int。



杂耍香蕉



在JavaScript中，可以通过以下方式获取字符串“ banana”： 



('b' + 'a' + + 'a' + 'a').toLowerCase(); // "banana"
('b'+'a'++'a'+'a').toLowerCase(); // Uncaught SyntaxError: Invalid left-hand side expression in postfix operation


在此，对第二个“ a”应用一元+会返回NaN，并将其转换为字符串“ NaN”以添加到其余字符串中，结果为“ baNaNa”，并且为了美观起见，所有内容都减小为小写。

        new Trickster { value = s };

        double.TryParse(trickster.value, out double result) ? result : double.NaN;
}

不幸的是，尽管进行了隐式强制转换，编译器仍无法构造字符串string-> Trickster-> double-> string，您必须显式提示它。（如果您考虑一下，这样的转换看起来会很奇怪。）

class Trickster
{

        +trickster.value;

public static implicit operator double(Trickster trickster) =>
