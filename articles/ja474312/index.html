Windows Server CoreへのGUIのインストール

前回の投稿では、標準のクライアント仮想マシンを準備する方法について説明し、Windows Server 2019 Coreの標準イメージを作成する方法を99ルーブルの超軽量ウィンドウvdsで新しい料金の例を使用して示しました。

サポートサービスは、通常のグラフィカルシェルなしでServer 2019 Coreを使用する方法のリクエストを受け取り始めました。Windows Server 2019 Coreの操作方法とGUIのインストール方法を示すことにしました。



稼働中のマシンでこれを繰り返さないでください。デスクトップとしてServer Coreを使用しないでください。RDPを無効にし、情報システムを保護します。セキュリティは「コア」インストールの主な機能です。

次の記事の1つで、Windows Server Coreとのプログラム互換性表を見ていきます。この記事では、シェルのインストール方法について説明します。

サードパーティのシェル




1.最も難しいが、最も経済的な方法


Server Coreでは、箱から出してすぐに馴染みのあるexplorer.exeはありません。私たちの生活をより簡単にするために、explorer ++をダウンロードします。これは、元のエクスプローラーが実行できるすべてのものを置き換えます。エクスプローラー++のみが考慮されましたが、Total Commander、FAR Managerなどを含むほとんどすべてのファイルマネージャーが対応します。

ファイルをダウンロードします。

まず、ファイルをサーバーにダウンロードする必要があります。これは、SMB（共有フォルダー）、Windowsアドミンセンター、およびInvoke-WebRequestを介して実行できます。これは、-UseBasicParsingパラメーターと連動します。

Invoke-WebRequest -UseBasicParsing -Uri 'https://website.com/file.exe' -OutFile C:\Users\Administrator\Downloads\file.exe


ここで、-uriはファイルのURLであり、-OutFileはファイルのダウンロード先のフルパスであり、Powershell

を使用してファイル拡張子とCを示し

ます。サーバーで、新しいフォルダーを作成します。

New-Item -Path 'C:\OurCoolFiles\' -ItemType Directory

共有フォルダを共有します：

New-SmbShare -Path 'C:\OurCoolFiles\' -FullAccess Administrator 
-Name OurCoolShare

PCでは、フォルダはネットワークドライブとして接続されます。


Windowsアドミンセンターで、メニューの項目を選択して新しいフォルダーを作成します。



共有フォルダに移動して送信ボタンを押し、ファイルを選択します。


スケジューラにシェルを追加します。

ログインごとにシェルを手動で起動しない場合は、タスクスケジューラに追加する必要があります。

$A = New-ScheduledTaskAction -Execute "C:\OurCoolFiles\explorer++.exe"
$T = New-ScheduledTaskTrigger -AtLogon
$P = New-ScheduledTaskPrincipal "local\Administrator"
$S = New-ScheduledTaskSettingsSet
$D = New-ScheduledTask -Action $A -Principal $P -Trigger $T -Settings $S
Register-ScheduledTask StartExplorer -InputObject $D

スケジューラーがなくても、CMDを実行できます。

CD C:\OurCoolFiles\Explorer++.exe

方法2.ネイティブエクスプローラーを起動する



GUI

サーバーコアアプリ互換性機能オンデマンド（FOD）は、MMC、Eventvwr、PerfMon、Resmon、Explorer.exe、さらにはPowershell ISEなどのシステムには戻りません。詳細については、MSDNをご覧ください。彼は既存の役割とコンポーネントのセットを拡張しません。

Powershellを起動し、次のコマンドを入力します。

Add-WindowsCapability -Online -Name ServerCore.AppCompatibility~~~~0.0.1.0

次に、サーバーを再起動します。

Restart-Computer



その後、Microsoft Officeを実行することもできますが、システムにアクティブなユーザーがいない場合でも、永久に約200メガバイトのRAMが失われます。
