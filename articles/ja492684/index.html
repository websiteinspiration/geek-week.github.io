簡単な背景


企業では、ユーザーが追加のパスワードを入力せず、Webインターフェイスから管理できるように、ADからのグループによるサイト（httpsを含む）へのアクセスをフィルターする機能を備えたプロキシサーバーを実装する必要がありました。悪いアプリケーションではありませんか？

正解はKerio ControlやUserGateのようなソリューションを購入することですが、いつものようにお金はありませんが、ニーズがあります。

次に、古き良きSquidが私たちを助けに来ますが、再び-Webインターフェイスをどこで入手できますか？SAMS2？道徳的に時代遅れ。ここでpfSenseが役に立ちます。

認証なしでSquid認証とアクセスリストを構成する


ケルベロスの構成に成功したら、それをSquidに固定します。

これを行うには、Services \ Squid Proxy Serverに移動し、メイン設定の一番下まで移動します。そこに[詳細設定]ボタンがあります。

[カスタムオプション（認証前）]フィールドに、次のように入力します。

#
auth_param negotiate program /usr/local/libexec/squid/negotiate_kerberos_auth -s GSS_C_NO_NAME -k /usr/local/etc/squid/squid.keytab -t none
auth_param negotiate children 1000
auth_param negotiate keep_alive on
# 
acl auth proxy_auth REQUIRED
acl nonauth dstdomain "/etc/squid/nonauth.txt" 
# 
http_access allow nonauth 
http_access deny !auth
http_access allow auth

SquidGuardを構成する


Services \ SquidGuard Proxy Filterに移動します。

LDAPオプションでは、Kerberos認証に使用されるアカウントの詳細を次の形式で入力します。

CN=pfsense,OU=service-accounts,DC=domain,DC=local

スペースやラテン文字以外の文字がある場合は、このエントリ全体を一重引用符または二重引用符で囲む必要があります。

'CN=sg,OU=service-accounts,DC=domain,DC=local'
"CN=sg,OU=service-accounts,DC=domain,DC=local"

次に、必ず次のチェックマークを付けてください：



不要なドメイン\ pfsenseを遮断するには ドメインシステム全体が非常に敏感な.LOCAL

次に、Group Aclに移動して、ドメインアクセスグループをバインドします。group_0、group_1などの精神で単純な名前を3つまで使用します。3はホワイトリストへのアクセスのみで、0がすべてです。

グループは次のようにアタッチされます。

ldapusersearch ldap://dc.domain.local:3268/DC=DOMAIN,DC=LOCAL?sAMAccountName?sub?(&(sAMAccountName=%s)(memberOf=CN=group_0%2cOU=squid%2cOU=service-groups%2cDC=DOMAIN%2cDC=LOCAL))

グループを保存し、Timesに移動します。ここでは、常に動作することを意味する1つの間隔を作成しました。次に、Target Categoriesに移動し、好きなようにリストを作成します。リストを作成した後、グループに戻り、選択したグループ内で、場所と場所を指定できないボタンを使用します。 。
