Xcode


上で書いたように、Xcodeテンプレートを使用して、iOS / macOS / tvOS / watchOS /クロスプラットフォームプロジェクトを作成し、ファイルと設定を追加できます。問題は、Appleがこのためのドキュメントを提供していないことです。それがすべてです-いくつかの小さなチュートリアルとさまざまな愛好家の例、私はこのウィキを使用しました。

すぐに
使えるiOSアプリケーションテンプレートは次のとおりです：
/Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/Library/Xcode/Templates/Project Templates/iOS/

MacOSアプリケーションの場合：
/Applications/Xcode.app/Contents/Developer/Library/Xcode/Templates/Project Templates/Mac/

プロジェクトテンプレートは通常の.plistファイルで表されます。おなじみのシングルビューアプリケーションは次のようになります。



各テンプレートには一意の名前（「識別子」）があり、アップルのテンプレートにはcom.apple.dt.unitプレフィックスが付いています。
祖先は現在のテンプレートの親です。テンプレートは複数の継承をサポートします。つまり、シングルビューアプリケーションは、ストーリーボードアプリケーションとコアデータココアタッチアプリケーションからプロパティを継承します。

これは、単一ビューアプリケーション階層がどのように見えるかです。 テンプレートを



作成してみましょう

。ローカルライブラリにカスタムテンプレートを追加することをお勧めします〜/ Library / Developer / Xcode /Templates。Xcodeの更新時にテンプレートが失われないようにします。そのようなディレクトリがない場合は、作成します。

事前定義された依存関係を持つPodfileプロジェクトに追加される小さなテンプレートを用意しました。

準備されたテンプレートコード
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
	<key>Kind</key>
	<string>Xcode.Xcode3.ProjectTemplateUnitKind</string>
	<key>Identifier</key>
	<string>ru.surfstudio.dt.unit.customTemplate</string>
	<key>Ancestors</key>
	<array>
		<string>com.apple.dt.unit.singleViewApplication</string>
	</array>
	<key>Concrete</key>
	<true/>
	<key>Description</key>
	<string></string>
	<key>Definitions</key>
	<dict>
		<key>../Podfile</key>
		<string>platform :ios, '11.0'

pod 'Alamofire'
pod 'Crashlytics'
pod 'Fabric'
			</string>
	</dict>
	<key>Nodes</key>
	<array>
		<string>../Podfile</string>
	</array>
</dict>
</plist>


ここでは、プロジェクトに追加できる変数とファイルを含む定義セクションを作成しました。そこで、Podfileキーを追加しました。キーは、作成または編集しているファイルを示します。キーの後に、このファイルに書き込まれる行を設定します。手動で行を書く代わりに、テンプレートのあるディレクトリからファイルを参照し、そこから取得します。

次に、このコードを取得して、テンプレートディレクトリのCustomTemplate.xctemplateフォルダーにあるTemplateInfo.plistというファイルに入れます。

私たちが得たものを見てみましょう。すべてが計画どおりに進んだ場合、Xcodeでプロジェクトを作成すると（[ファイル]-> [新規]-> [プロジェクト（⇧⌘N）]）、新しいテンプレートセクションが表示され、その中に新しい「CustomTemplate」テンプレートが表示されます。



それを選択してプロジェクトを作成します。作成後、必要に応じてPodfileが追加されます。



テンプレートを汲み上げます

プロジェクトのReadme生成を追加しましょう。それでは、テンプレートを使用してそれを記述せずに、すぐにファイルを使用して追加してみましょう。

したがって、CustomTemplate.xctemplate内にREADME.mdファイルを追加すると、次のようになります。

# ___PROJECTNAME___

Description of my project


テンプレートでノードセクションにファイルを追加できるようにするには、以下を追加する必要があります。

<string>../README.md</string>


また、[定義]セクション内に次を追加します。

<key>../README.md</key>
    <dict>
        <key>Path</key>
        <string>README.md</string>
    </dict>

完全なテンプレートコードは次のようになります。
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
	<key>Kind</key>
	<string>Xcode.Xcode3.ProjectTemplateUnitKind</string>
	<key>Identifier</key>
	<string>ru.surfstudio.dt.unit.customTemplate</string>
	<key>Ancestors</key>
	<array>
		<string>com.apple.dt.unit.singleViewApplication</string>
	</array>
	<key>Concrete</key>
	<true/>
	<key>Description</key>
	<string></string>
	<key>Definitions</key>
	<dict>
		<key>../Podfile</key>
		<string>	
platform :ios, '11.0'

pod 'Alamofire'
pod 'Crashlytics'
pod 'Fabric'
			</string>

		<key>../README.md</key>
            <dict>
                <key>Path</key>
                <string>README.md</string>
            </dict> 
	</dict>
	<key>Nodes</key>
	<array>
		<string>../Podfile</string>
		<string>../README.md</string>
	</array>
</dict>
</plist>

CustomTemplateを使用してプロジェクトをもう一度作成してみましょう。最後に、Readmeがプロジェクトに追加されている



ことがわかります。README.mdで、ヘッダーに「MyNewProject」ではなく___PROJECTNAME___を追加したことに注意してください。___PROJECTNAME___、これは、必要に応じて使用できる事前定義された定数です。

たとえば、コード付きのファイルのヘッダーを生成するには、Xcodeが通常行うように、次のテンプレートを使用します。

//
//  ___FILENAME___
//  ___PACKAGENAME___
//
//  Created by ___FULLUSERNAME___ on ___DATE___.
//___COPYRIGHT___
//

結論


今日はシンプルなプロジェクトテンプレートの作成方法を学びました。これで、PodfileファイルとREADMEファイルをプロジェクトに追加できるテンプレートができました。

私たちの足跡をたどり、プロジェクトの作成に必要な時間を短縮したい場合は、以下をお勧めします

。1.適切なソリューションを選択します。
Xcodeテンプレート以外にも、XcodeGenやベースプロジェクトなどがあります。いずれの場合も、ツールを選択する前に、長所と短所を調べ、適切なものを選択してください。

2.一般的なプロジェクトの作成に必要な手順を詳しく説明してください。プロジェクトはあまり頻繁に開始されないため、いくつかの手順を忘れることがあります。

3.ポイント2からの手順を自動化してみます。
たとえば、それらの一部はXcodeテンプレートに転送する必要があり、一部はプロジェクトの作成後に呼び出されるスクリプトに転送する必要があります。
