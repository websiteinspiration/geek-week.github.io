今日は、クラウドインフラストラクチャへの移行に関するOracleからのホワイトペーパーの翻訳であるクラウドテーマに関心のあるすべての人と共有したいと思います。私たちの同僚と常設のhabroavtorの翻訳をありがとうobieesupport。

RDTEXは、ロシアで最も古いOracleパートナーの1つです（会社間の最初のパートナーシップ契約は1993年に署名されました）。今日、当社はオラクルプラチナパートナーの最高のパートナーステータスを誇っています。RDTEXトレーニングセンターは、ロシアにおけるオラクルの認定トレーニングの主要なセンターの1つです（私たちの読者の1人は、トレーニングセンターのコースに参加したか、または今では遠隔で勉強していると思います）。

私たちの同僚は、「baseodanコミュニティ」の多くの代表者が使用する、伝説的なOracle用語集を作成しました（ネタバレ：次の投稿の1つで、用語集について詳しく説明します）。

ここで、オラクルのクラウドインフラストラクチャをさらに深く掘り下げ、オラクルのクラウドとオフラインデータベースがビジネス革新を加速する方法を学ぶことをお勧めします。オリジナルの「ホワイトペーパー」は、当社のWebサイトからダウンロードすることもできます。





1.オラクルは第2世代の

クラウドを提供オラクルのクラウドインフラストラクチャは、企業がオンプレミスからクラウドにシームレスに移行し、高度な自動化と組み込みのセキュリティを使用して脅威を軽減し、最終的に優れた移行と費用対効果をサポートするのに役立つツールとアーキテクチャを提供します。

革新のために設計されたOracle Cloud Infrastructure。これには、業界をリードするスケーラビリティと可用性、統合された管理と制御、エンドツーエンドのSLAに裏打ちされた信頼性が含まれます。オラクルのクラウドの使命は、人工知能、機械学習、モノのインターネット、ブロックチェーン、人間指向のインターフェースなどの新しいテクノロジーをサポートするように拡張されています。

オラクルの企業顧客は、これらのテクノロジーの実験からミッションクリティカルなアプリケーションへの展開、新しいビジネスモデルの構築、新しいビジネス価値の創出に移行しています。

Oracle Cloud Infrastructureは、大量のデータを必要とするワークロードまたはアプリケーションによって駆動される大規模な金融システムで動作するように設計されていない第1世代のクラウドソリューションの主要な課題に対処します。第一世代のクラウドソリューションは、パフォーマンス、セキュリティ、移行の設定が常に遅れていた10年前のテクノロジーに基づいて構築されました。次世代のOracle Cloud Infrastructureは、今日の企業のニーズを満たすように特別に設計されており、より高速で予測可能なパフォーマンス、より優れた価格とセキュリティ、および企業のワークロードとの互換性の向上を実現します。

オラクルは、その第2世代クラウド製品の一部として、IaaS、PaaS、およびSaaSサービスの唯一のプロバイダーです。また、スタンドアロンのOracleデータベースサービスは、他のサービスと同じ高速ネットワークを使用するため、ミッションクリティカルなアプリケーションを迅速に展開して、継続的なイノベーションをサポートできます。

2.Oracle Cloudは長期計画をサポートします

Oracle Cloud Infrastructureは、Oracle Cloudの基盤として機能する根本的に新しい第2世代のオープンクラウドアーキテクチャです。このインフラストラクチャは、予測可能なパフォーマンス、システムのコアから最先端までのセキュリティ、および重要で集中的なワークロードをサポートするために必要な制御を提供するように特別に設計されています。

Oracle Cloudは、信頼できるビジネス結果を提供するために必要なコンピューティング、データウェアハウジング、データベース、ネットワーク、およびプラットフォームサービスのための機能とテクノロジーを提供し、データセンターのニーズを再考することを可能にします。

Oracle Cloud InfrastructureはAWSをエンドツーエンドのワークロードパフォーマンスで優れています（Andrew Reichman、「Oracle Tests Better Performance in Amazon Web Services」、Oracle、2018年9月18日）。

3.パフォーマンスを最適化する機会

Oracle IaaSは、オペレーティングシステム（ベアメタル）を持たないプラットフォーム向けの第2世代プラットフォームや一連のサービスから、テクニカルコンピューティングクラスター向けのリモートダイレクトメモリアクセス（RDMA）まで、業界で比類のない幅広い機能を提供します。この差別化により、Oracleクラウドインフラストラクチャは、予測可能なパフォーマンスとクラウドクライアントの適切な分離の両方を確実に示します。
Oracleのクラウドインフラストラクチャは、Oracle Autonomous Data WarehouseやOracle Autonomous Transaction Processingなどの最先端のイノベーションへの扉も開きます。

スタンドアロンワークロードは、低レイテンシ、高可用性、フォールトトレランス、および一貫したパフォーマンスを提供するように設計されたインフラストラクチャで最適に機能します。つまり、データはOracleオブジェクトストレージに格納され、複数のコピーが自動的に複製されるため、高可用性とフォールトトレランスが確保されます。オラクルの自己完結型自己修復機能により、データは「正常」のままであり、ユーザーは常に最新の既知のレプリカにアクセスできます。

4.深い保護

セキュリティは、Oracleクラウドインフラストラクチャ内の主要な設計原則です。オラクルは、クライアントの分離、データセキュリティ、内部脅威の検出、高度に自動化された排除など、システムのコアから最先端までの保護を提供します。 Oracle Infrastructureは、コンピューティングリソースとネットワークリソースを分離して、他のユーザーから個人データとトラフィックを保護します。また、コード、データ、リソースを制御マシンから分離し、攻撃者がクラウド内のデータを盗んだり操作したりするのを防ぎます。

Oracle Cloudは、きめ細かいクライアント分離により、非常に限定された攻撃機能を提供します。ファイアウォール、DDoS攻撃ディテクタ、および暗号化が組み込まれたセキュリティレベルは、脅威を予防的に検出して阻止します。顧客は新しい境界でIDを確立し、適応型認証を使用して、ユーザーアクティビティがより高いレベルのリスクを示したときに自動的に追加の検証を追加できます。最後に、オラクルはクラウドおよびオンプレミスソリューションの業界で最も幅広いポートフォリオの1つを持っています。

5. Oracleからのワークロードの最適化

Oracle Cloud Infrastructureは、Oracleのデータベースおよびビジネスアプリケーションのポートフォリオの効率的な移行と起動をサポートするいくつかの独自の機能とツールを提供します。 Oracleアプリケーションの移動には最小限の変更が必要です。これにより、クラウドへの移行のコストと期間が削減されます。オラクルの優先事項は、顧客の生産性を継続的に向上させる最新の機器とテクノロジーを提供することです。

Oracleデータベースは、ストレージと競合他社のパフォーマンスの点で何倍にも拡張され、インスタンスあたり最大40テラバイトの容量と数百万のI / Oオペレーション（IOPS）に達します。最も複雑な自律データベースワークロードを実行するために最大数のI / O操作が必要な場合、Oracleは業界最大のローカル不揮発性オールフラッシュエクスプレス（NVMe）メモリ（インスタンスあたり最大51.2 TB）を提供します。 Oracle Cloud InfrastructureでOracle Real Applications Clusters（Oracle RAC）クラスタを起動すると、数秒でフェイルオーバーし、パフォーマンスを数十万のI / Oにスケーリングし、ローリングパッチと更新に基づいてスムーズなオペレーションを実現する高可用性（HA）データベースを提供します。このテクノロジーは、本番データベースアプリケーションに新しいクラウド標準を導入します。

Oracle Exadata Cloudを使用すると、フルシステムと同じシステムまたはフルラックで管理されるベアメタルコンピューティングやその他のOracleクラウドインフラストラクチャサービスと同じ拡張リージョンおよび仮想クラウドネットワークでExadataフォームファクターマシンを実行できます。管理ツールキットであり、同じコンソール/ APIからアクセスできます。

オラクルのクラウドインフラストラクチャは、Oracle E-Business SuiteやOracle PeopleSoft、Siebel、JD Edwardsなどの主要なOracleアプリケーション向けの一連の移行、プロビジョニング、管理ツール、およびカスタマイズと関連データベースを提供し、顧客のクラウドへの移行を加速します。特に、スタンドアロンデータベースの場合、Oracleのクラウドインフラストラクチャはトランザクション分析とトランザクション処理のワークロードを最適化します。 MLワークロードを実行して人工知能に基づく情報を提供するように特別に設計されています。これにより、クライアントは複数のデータソースに接続し、主要なデータを組み合わせて健全なビジネス上の意思決定をサポートできます。 Oracleのクラウドインフラストラクチャは、統合、コンテナ、マイクロサービス、ブロックチェーンなどの新しい高度なサービスの基盤でもあります。モノのインターネットの展開をサポートするための拡張された拡張性を提供します。

6.アプリケーション開発の完全サポートの使用

オラクルは急速に革新的であり、オープンで標準ベースの完全に統合されたアプリケーション開発プラットフォームを提供します。これにより、APIとモバイルデバイスに基づく最新のクラウドベースのアプリケーションを低コストで作成、展開、および管理できます。さらに、オラクルは、少量の変更可能なコードでコンテナおよびクラウド開発のサポートを提供しています。

Oracle AppDevプラットフォームは、継続的インテグレーション（CI）/継続的デリバリー（CD）、Javaアプリケーションの詳細な診断、SaaSおよびローカルソフトウェアとの統合の簡素化のための包括的なDevOps環境を提供します。

7.最高の価格/性能比

Oracle Cloud Infrastructureは、今日の市場で入手可能な最高の価格/パフォーマンスを提供します。 Oracleクラウドインフラストラクチャにデプロイされたワークロードは、多くの場合、必要な計算サーバーとブロックストレージボリュームが少ないため、ワークロードのパフォーマンスを最適化するコストが削減されます。

オラクルのノンブロッキングクラウドインフラストラクチャアーキテクチャは、レイテンシを最小限に抑えるだけでなく、クライアント環境に影響を与える他のネットワークユーザーのリスクを軽減するだけでなく、業界で最も包括的で経済的に安全なエンドツーエンドのSLAによってサポートされます。

8. Oracle CloudがAWSを上回る

Oracle Cloudは、低コストで最適なパフォーマンスを提供し、ミッションクリティカルなエンタープライズアプリケーション向けのOracle Database業界標準を立ち上げるための最良のオプションです。 AWSはOracle RAC、Oracle Autonomous Data Warehouse、またはOracle Autonomous Transaction Processingを提供していません。AmazonAuroraはAWSクラウドでユーザーを閉じますが、Oracleデータベースは異なるメーカーの柔軟性を提供し、組み合わせます。

Oracle Virtual MachinesやベアメタルなどのOracleコンピューティングサーバーは、AWSなどの競合する仮想マシンと比較して、大幅なコスト削減（25〜65％）を実現します。

OracleデータベースはAWSに比べて高速で、コストが最大66％削減されます。これにより、ビジネスデータに基づく結果をより効率的かつ手頃な価格で取得できます（アクセンチュア、「エンタープライズワークロードがクラウドに対応：クラウド内のOracleデータベースについて」 、」、「Storage Review」、「Oracle Cloud Infrastructure Compute Bare Metal Instances Review」）。



9. Oracle Cloudは中小企業の成長をサポート

Oracle Cloudは、中小企業を含むすべての企業の利益を満たすように設計されています。これは、中小企業が大きくなり、アプリケーションの実装ペースが大幅に増加することが多いためです。ただし、大企業と同じ問題に直面する可能性があります。 Oracle Cloudは、考えられるすべてのシナリオをサポートします。開発者およびミッションクリティカルなワークロード向けの従来のツールと最新のツールを計画して使用します。

10. OracleとMicrosoftは接続されたクラウドシステムを提供します

相互のクラウドと対話するためのOracleとMicrosoftのテクノロジーパートナーシップは、通常のアプリケーションとミッションクリティカルなアプリケーションの両方をサポートしています。顧客はクラウドに移行したり、Oracle Autonomous Databaseを含む最高のOracle Cloudとシームレスな互換性を持つ最高のMicrosoft Azureを使用して新しいアプリケーションを作成したりできます。

この戦略的パートナーシップにより、次のような共同顧客がクラウドに簡単に移行できるようにする新しい一連の機能が提供されます。

  • クラウド間の接続、
  • 統合クラウドサービス
  • 共同サポートモデル
  • アプリケーション展開の柔軟性。

パフォーマンスを最大化するクライアントの主な機能は、引き続きOracleのクラウドインフラストラクチャでOracleアプリケーションを実行することです。
Oracle Cloudには無料の試用版があり、自律管理、自己保護、および自己修復機能を備えた世界初のスタンドアロンデータベースを探索する機会を提供します。この試用版を使用して、仮想マシンの最も高性能でコスト効率の高いコンピューティングインスタンスを発見してください。



試用版を使用するには、oracle.com / cloud / freeにアクセスするか、Oracleパートナーにお問い合わせください。
