時間は待たない：Windows 10タイムラインよりも犯罪者に役立ちます



インシデントとウイルス攻撃を調査してユーザーのコンピューターで発生したイベントを再構築する場合、フォレンジックは多くの場合、さまざまなタイプのタイムラインを使用します。原則として、タイムラインは巨大なテキストファイルまたはテーブルであり、コンピュータ上で発生したイベントに関する情報が時系列で提供されます。

タイムラインを準備するためのメディアの処理（たとえば、Plasoの使用）は、時間のかかるプロセスです。そのため、コンピューターフォレンジックは、Windows 10の次の更新で、Windows 10タイムラインという新しい機能があることを知って非常に満足しました。

「Windowsの新機能について最初に知ったとき、私は思いました。」これで、タイムラインを生成するために時間を無駄にする必要がありません。」しかし、結局のところ、私の喜びは時期尚早でした」とコンピューターフォレンジックグループIBの研究所の専門家であるIgor Mikhailovは言いました。特にHabrリーダーの場合、Igorは、Windows 10タイムライン（新しいタイプのWindows 10アーティファクト）が従来のタイプのタイムラインとどのように関係し、どのような情報が含まれているのかを説明します。

Computer Forensics Group-IB研究所の専門家、Igor Mikhailov による投稿。

既に述べたように、Windows 10の2018年4月の更新では、「タスクビュー」（Windows 10タイムライン、または略してタイムライン）と呼ばれる新機能が登場しました。コンピューターユーザーのアクティビティを表示し、以前に開いたドキュメントやプログラム、以前に表示したビデオや画像、Webサイトにすばやく戻ることができます。Windows 10タイムラインを開くには、次のアイコンをクリックする必要があります。


その後、当日または少し前に開いたプログラム、ドキュメント、サイトのサムネイルが利用可能になります。


研究者たちは、新機能の2つの特徴に注目しました。

  1. 一部のアプリケーションを開くとタイムラインに表示されない
  2. Timeline ( ) Microsoft Cloud

したがって、Windows 10タイムラインの分析を使用して取得できる情報は、分析対象のコンピューターで発生するイベントに関するより完全な情報を含む通常のタイプのタイムラインとは異なります。

Windows 10タイムラインは、フォレンジックユーティリティによって作成されたタイムラインと混同しないでください。たとえば、Autopsy、Belkasoft Evidence Center、AXIOMによって作成されたタイムラインには、Windows 10タイムラインと比較して、調査中のコンピューターのユーザーのさまざまなアクションに関する記録が大幅に含まれています。

Windows 10タイムラインをアクティブ化するには、コンピューターユーザーがオペレーティングシステムの[設定]セクションに移動し、[プライバシー]カテゴリと[アクティビティログ]サブカテゴリを選択して、反対側のボックスをオンにする必要があります。

  • Windowsがこのコンピュータから自分のアクションを収集できるようにする
  • Windowsがこのコンピューターからクラウドにアクションを同期することを許可する

Windows 10タイムラインのユーザーアクティビティに関する情報は、\ Users \％プロファイル名％\ AppData \ Local \ ConnectedDevicesPlatform \ L.％プロファイル名％\のパスに沿って、ActivitiesCache.dbファイルに保存されます。

ActivitiesCache.dbファイル


ActivitiesCache.db ファイルはSQLiteデータベース（バージョン3）です。他のSQLiteデータベースと同様に、2つの補助ファイルActivitiesCache.db-shmとActivitiesCache.db-walが存在するという特徴があります。


削除されたレコードに関する追加情報は、未使用の表スペース、フリーリスト、およびwalファイルに含まれている場合があります。

Windows 10タイムラインの構造


ActivitiesCache.dbには、次のテーブルが含まれています：Activity、Activity_PackageId、ActivityAssetCache、ActivityOperation、AppSettings、ManualSequence、Metadata。研究者にとって最も興味深いのは、テーブルActivity_PackageIdおよびActivity）です。

Activity_PackageIdテーブル


Activity_PackageId テーブルには、実行可能ファイルのパスを含むアプリケーションのエントリ（たとえば... c：\ programdata \ firefly studios \ s tronghold kingdoms \ 2.0.32.1 \ strongholdkingdoms.exe ...）、実行可能ファイルの名前、および有効期限も含まれていますこれらのエントリ。Expiration Time列の値は、エポック時間形式で保存されます。Activity_PackageId

テーブルのエントリは30日間保存され、調査中のハードドライブに既に存在しない実行可能ファイルまたはドキュメントに関する情報が含まれている可能性があります。

Windows 10のタイムラインを学ぶ


ActivitiesCache.db ファイルに含まれるデータを表示する最も簡単な方法は、SQLiteデータベースビューアを使用することです。たとえば、SQLiteユーティリティ用の無料のDBブラウザ。


[データの参照]タブでテーブルを移動すると、特定の調査で興味深い内容とレコード情報を表示できます。

推奨する2番目のユーティリティはWxTCmd（Windows 10 Timelineデータベースパーサー）です。このユーティリティはコマンドラインから実行されます。


その作業の結果として、ActivitiesCache.dbファイルの分析結果を含むcsvファイルが作成されます。

Excelで開いているこのファイルのビュー

私たちが推奨する3番目のユーティリティは、Magnet ForensicsによるAXIOMです。

プログラムがこの成果物を分析するためには、「ケースに含める成果物を選択する」セクションでこれを行うようにプログラムに指示する必要があります。


ファイル分析結果は、Magnet AXIOM Examinerで表示できます。

AXIOMによってActivitiesCache.dbからデータを抽出した結果

このようなファイルの分析に推奨される4番目のユーティリティは、BelkasoftのBelkasoft Evidence Centerです。

[データソースの追加]ウィンドウでデータソース（ハードディスク、論理ドライブ、ディレクトリ、またはファイル）を追加した後、[ システムファイル]セクションで[ Windowsタイムライン]を選択します。



データの抽出が完了すると、Windowsタイムラインカテゴリが[ 概要 ]タブに表示され、ActivitiesCache.dbファイルからのデータ抽出の結果が表示されます。
