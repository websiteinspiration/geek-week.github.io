Bagaimana cara mendapatkan token OpenID / OAuth2 untuk menguji layanan front-end rest?

Ada tugas untuk memuat menguji api web rest depan. Sisanya dilindungi oleh OAuth dengan Hibah Kode Otorisasi. Ini berarti bahwa ada kebutuhan untuk token yang valid untuk Otorisasi: Bearer TOKEN.

Pertanyaannya adalah bagaimana cara mengambilnya? Jadi, lakukan dengan indah dan benar? Di sini saya tidak tahu.



Dalam artikel ini, saya menjelaskan solusi cepat yang saya gunakan dan yang tidak saya sukai. Itu terlihat seperti tongkat penyangga, meskipun saya belum meng-google apa-apa. Karena itu, jika Anda tahu solusi terbaik - pastikan untuk menceritakannya di komentar ... Jadi ...



Ada sistem standar biasa dengan bagian depan web rest dan klien browser Aplikasi Tunggal Halaman di JS. Otentikasi dan Otorisasi - KeyCloak dengan Pemberian Kode Otorisasi + pialang.



Hal ini diperlukan untuk menyediakan pengujian beban reguler dari layanan sandaran depan.



Tugasnya cukup sederhana, jika kita memiliki token yang bisa kita sisipkan ke header dan menggunakan JMeter untuk menghasilkan aliran permintaan yang diperlukan. Di sinilah saya tersandung, browser web menerima token secara sederhana dan alami (KeyCloak JS), tapi saya masih tidak mengerti bagaimana cara mendapatkannya tanpa browser menggunakan permintaan HTTP berurutan dan tanpa mengeksekusi JS ...

Token dalam sistem diperiksa langsung di layanan lainnya, dan tidak aktif API Gateway Tidak mungkin untuk menonaktifkan pemeriksaan karena tidak ada kemungkinan seperti itu. Pengujian saja tanpa token akan gagal.



Selanjutnya, kami berpikir, mengapa tidak menggunakan tes Selenium ujung ke ujung yang kami miliki, tetapi dengan cepat meninggalkannya karena sumber daya yang diperlukan untuk browser yang bekerja secara bersamaan ternyata cukup besar. Untuk minimum 50 utas, kami membutuhkan 50GB + 25 inti. Namun, ini memberi kami gagasan bahwa token dapat diperoleh melalui Selenium, dan kemudian diteruskan ke JMeter untuk digunakan.



Akibatnya, MVP dengan cepat dibuat sesuai dengan skema berikut:



  • Tingkatkan token seumur hidup untuk lingkungan pengujian
  • Nonaktifkan cache dalam sistem untuk mensimulasikan keunikan pengguna.
  • Menggunakan aplikasi Selenium, kami mengikuti prosedur masuk dari grup pengguna uji dan membuang token mereka ke file. Untuk membaca token, kami menggunakan panggilan JS melalui WebDriver - return keykcloak.token;
  • JMeter
  • JMeter






Kami menetapkan token life time menjadi 36 jam. Kami mengambil kode otorisasi sekunder langsung dari database yang sesuai. Waktu masuk per pengguna adalah sekitar 10 detik, yang memberi kami 360 token per jam. Ini memungkinkan Anda untuk dengan mudah mempersiapkannya jika diinginkan.



Sekarang, operator menjalankan selenium pada laptop khusus untuk mengumpulkan token di malam hari, dan di pagi hari file token yang disiapkan siap dan Anda dapat menjalankan JMeter untuk memuatnya.

Skema ini berfungsi, pelanggan senang dengan kesederhanaan dan biaya rendah dari solusi, dan menawarkan untuk mentransfernya ke rel normal dengan beberapa kontainer dan diluncurkan dari Jenkins.



Tapi solusinya menurut saya bengkok. Yah, pasti ada cara untuk melakukannya tanpa selenium! Tulis jika ada yang tahu. Saya tidak bisa google apa pun tentang masalah pengujian OpenID & OAuth2itu dengan Hibah Kode Otorisasi .
