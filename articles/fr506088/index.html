Expliquez-moi comment vous avez compris vous-même les modèles de frappe - ils sont tous flous


Quand j'étais débutant, je pouvais écrire des applications simples en C # et C ++. J'ai joué pendant longtemps avec des programmes de console, je me sentais sur le bureau et à un moment donné, j'ai voulu créer un site. Une grosse surprise m'attendait - pour faire des sites, une de ces choses ne suffit pas. Vous devez également connaître ws, html, css et autres conneries de première ligne. J'ai passé environ une semaine sur ces choses et j'ai réalisé - pas les miennes. Je pouvais écrire une sorte de code javascript, mais il ne contenait pas de types et je ne pouvais pas du tout comprendre comment aborder cela. Il s'agit d'une sorte de programmation de jouets. Eh bien, abandonné à l'enfer.


Plus tard, travaillant sur le troisième, j'ai été transféré au département où ils ont fait le web. J'ai pensé à arrêter, mais ils m'ont expliqué - il y a un script de temps, un script de temps est un tel cookie pour le navigateur.


J'ai accepté, j'ai étudié et maintenant c'est l'un de mes PL préférés. Mais. Dactylographié - ce n'est pas du tout un sysharp. Il s'agit d'un langage avec un système de types fondamentalement différent. Complexe, puissant, mais différent.


Il existe plusieurs paramètres de typification eux-mêmes.


Il existe un typage statique / dynamique. Statique, c'est lorsque les types de données sont connus au stade de la compilation. Le compilateur connaît les types de variables et vérifie sur leur base l'exactitude du programme. Dynamique est lorsque les types sont connus uniquement au moment de l'exécution et que le compilateur ne vérifie rien pendant l'assemblage.
