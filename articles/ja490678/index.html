MonbMQパート5でのRabbitMQの使用

で前の部分、我々は我々のロギングシステムを改善しました。ファンアウト型交換を使用する代わりに、直接型交換を使用したため、メッセージを選択的に受信できました。改善にもかかわらず、システムにはまだ制限があります。たとえば、いくつかの基準に基づいてメッセージを受信することはできません。たとえば、私たちのシステムでは、メッセージの重大度レベルだけでなく、メッセージソースにも基づいてメッセージをリダイレクトすることができます。たとえば、syslog unixツールのように重大度レベル（info / warn / crit ..）だけでなく、ソース（auth / cron / kern ...）にも応じてメッセージをリダイレクトします。これにより、柔軟性を高めることができます。たとえば、「cron」からの重要なメッセージだけを受信でき、「kern」からのすべてのメッセージも受信できます。このようなシステムを実装するには、より複雑なタイプの交換器- トピックに精通する必要があります。

トピック型交換器


トピックタイプエクスチェンジャに送信されるメッセージには、ドットで区切られた単語のセットであるルーティングキーが必要です。言葉は任意ですが、通常、メッセージのいくつかの機能に関連付けられています。有効なルーティングキーの例をいくつか次に示します：stock.usd.nyse、nyse.vmw、quick.orange.rabbit。キーは255バイトに制限されています。

バインディングキーも同様に指定する必要があります。トピックタイプエクスチェンジャーでのメッセージのルーティングは、ダイレクトタイプエクスチェンジャーのルーティングに似ています。メッセージは、ルーティングキーと一致するバインディングキーと共にキューに送信されます。ただし、2つの違いがあります。

  1. *バインドキーの（アスタリスク）は、1つの単語でのみ置換できます
  2. バインドキーの＃（ポンド）は0個以上の単語で置き換えることができます

この例では、動物を表すメッセージを送信します。メッセージルーティングキーは、3つの単語（および2つのドット）で構成されます。最初の単語は速度を表し、2番目の単語は色を表し、3番目の単語は種を表します。

この例では、2つのキューがエクスチェンジャーに接続されています。Q1はバインディングキー"* .orange。*"で、Q2は2つのバインディングキー"*。*。Rabbit"と"lazy。＃"です。

これらのリンクは次のルーティングを意味します：

  • Q1はすべてのオレンジ色の動物に関心があります。
  • Q2 (rabbit) (lazy)

ルーティングキーが「quick.orange.rabbit」のメッセージは、両方のキューで配信されます。キー「lazy.orange.elephant」を持つメッセージも両方に配信されます。

ただし、キーが「quick.orange.fox」のメッセージは最初にのみ届き、キーが「lazy.brown.fox」のメッセージは2番目にしか届きません。「Lazy.pink.rabbit」は、ルーティングキーが両方のバインディングキーと一致するにもかかわらず、2回配信されます。

「Quick.brown.fox」は、ルーティングキーがどのバインディングキーとも一致しないため、どのキューにも入れられません。ルーティングキーの単語数が少ないまたは多いメッセージを送信しようとした場合（たとえば、「オレンジ」または「quick.orange.male.rabbit」）バインドキーよりもメッセージが破棄されます。

バインディングタイプキーとして＃を指定すると、トピックタイプエクスチェンジャーはファンアウトタイプエクスチェンジャーのように動作します。または、バインディングキーで* *または＃を指定せず、単に単語を示す場合も同様です。

コードをまとめる


ロギングシステムにはトピックタイプの交換器を使用します。メッセージルーティングキーは"source.strict。"のようになるという前提から始めましょう。コードは前のパートとほとんど同じです。ここにsend.phpがあります。

try {
   $producer = \MonsterMQ\Client\Producer();

   $producer->connect('127.0.0.1', 5672);
   $producer->logIn('guest', 'guest');

   $producer->newTopicExchange('topic-logs');

   $routingKey = isset($argv[1]) && !empty($argv[1]) ? $argv[1] : 'anonymous.info';

   $message = implode(' ', array_slice($argv, 2));
   $message = empty($message) ? "Hello World!" : $message;

   $producer->publish($message, $routingKey, 'topic-logs');

   echo "\n Sent {$message} \n";
} catch(\Exception $e) {
   var_dump($e);
}

コードworker-1.php

try {
   $consumer = \MonsterMQ\Client\Consumer();

   $consumer->connect('127.0.0.1', 5672);
   $consumer->logIn('guest', 'guest');

   $producer->queue('queue-1')->setExclusive()->declare();

   $consumer->newTopicExchange('topic-logs');

   $bindingKeys = array_slice($argv, 1);
   if (empty($bindingKeys)) {
      file_put_contents('php://stderr', "Usage: $argv[0] [binding_key]\n");
      exit(1);
   }

   foreach ($bindingKeys as $key) {
      $producer->queue('queue-1')->bind('topic-logs', $key);
   }

   $consumer->consume('queue-1');

  echo " \n Waiting for logs. To exit press CTRL+C\n";

   $consumer->wait(function ($message, $channelNumber) use ($consumer){
      echo "\n $message \n";
   });
} catch(\Exception $e) {
   var_dump($e);
}

最初のワーカーによって重要なメッセージのみを受信するには、次を呼び出します

php worker-1.php "*.critical"

2番目のワーカーが使用するキューをバインドするには、2つのバインドキーでエクスチェンジャーを呼び出します。

php worker-2.php "kern.*" "*.critical"

メッセージを送信するには、次のようにします。

php send.php "kern.critical" "A critical kernel error"

これらのプログラムを試してください。
