照顾家庭作业：使用免费的Veeam代理备份笔记本电脑或台式机

在平常的生活中，我像你们中的许多亲爱的读者一样，大部分时间都在办公室工作，这非常适合我。但是今天的情况是如此，我和我在世界不同地区的同事，而且我认为，大多数阅读本文的人都在家里工作。

在这里，回忆起备份家用计算机或笔记本电脑的需求将显得格格不入。对于工作环境发生变化的情况，通常的数据处理方式也会发生变化。在办公室工作，一切都显而易见：直接访问必要的资源，集中式备份（如果有的话），您随时可以与服务台联系。在家工作时，您需要为自己提供很多东西，包括备份。

我相信许多人已经为此使用了Veeam Agent for Microsoft Windows或Veeam Agent for Linux。如果还没有，那么您可能会对今天的简短评论感兴趣。所以欢迎猫。



2014年，当我们发布产品的第一个版本（现在称为Microsoft Windows的Veeam代理）时，它被称为Veeam Endpoint Backup Free，并且是唯一的非商业版本。

当然，此后解决方案发生了很大变化，并推出了商业版本，其中包含许多高级功能-集中管理，CBT驱动程序，集群支持，并行磁盘处理等等。 Veeam Linux代理也正在开发中，它允许您备份物理机并从备份还原数据。但是现在，到了2020年，新版本的V eeam Agent for Microsoft Windows和Veeam Agent for Linux还有一个针对个人计算机的免费版本。

因此，如果您需要备份家用台式机，笔记本电脑甚至Windows上运行的平板电脑，请保持健康。

  • Microsoft Windows Server 2019

  • Microsoft Windows Server半年频道（包括1909年）
  • Microsoft Windows Server 2012 R2
  • Microsoft Windows Server 2012
  • Microsoft Windows Server 2008 R2 SP1
  • Microsoft Windows 10（版本1607至1909）
  • Microsoft Windows 8.1
  • Microsoft Windows 7 SP1

笔记：

  • 对于服务器操作系统，支持Small Business Server和Server Essentials版本。
  • 在独立部署方案中，不支持服务器系统的服务器核心版本。
  • 支持的Windows Embedded / Windows IoT OS（自定义版本除外）。
  • 当Veeam Agent for Microsoft Windows运行Veeam备份和复制时，支持Microsoft故障转移群集（在独立部署方案中不支持）。
  • 从版本10586开始支持Microsoft Windows 10 Education。

其余的系统要求在文档（英文）中有详细描述。

Veeam Agent for Microsoft Windows中的备份作业


对于备份，通过启动“ 新建备份作业”向导来配置备份任务。在其中，选择备份类型：

  • 整个计算机 -整个机器。
  • 卷级备份 -备份所选卷。
  • 文件级备份 -特定文件的备份。

Veeam Microsoft Windows代理可以将不同类型的存储系统用作备份存储位置：

  • 本地存储 -外部驱动器（USB，Firewire或eSata）。如您所知，这里的主要内容不是将自己备份到计算机上的内置磁盘。
  • 共享文件夹 -文件服务器或NAS上的SMB共享。
  • Veeam备份存储库 -Veeam存储库（本地）。
  • Veeam Cloud Connect存储库是Veeam服务提供商提供的基于云的存储库。
  • 微软OneDrive

在我的示例中，备份将转到USB驱动器-为此，请在“ 目标”步骤中选择“ 本地存储 ” ：



然后，设置特定于所选存储类型的设置。

重要！当使用可移动USB驱动器时，请记住，如果将这种驱动器格式化为FAT32，则它不支持存储大于4 GB的文件。因此，不建议将此类磁盘用作目标备份存储。

接下来，在Schedule步骤中配置时间表。在这里，您可以指定是在特定时间定期备份还是在发生选定事件时执行备份：

  • 锁 -锁车时
  • 注销 -注销当前帐户
  • 连接备份目标时- 连接目标备份存储时

如果指示了第三个选项，则还可以指示Veeam代理在备份结束时完成断开（退出）目标存储的连接，以使备份脱机。如果检测到加密病毒的备份副本，这将降低其暴露风险。

Veeam Linux代理


Veeam Linux代理旨在保护Linux上运行的计算机的数据，该解决方案还具有免费版本。支持Linux内核2.6.32和更高版本。

支持64位和32位（如果适用）的发行版：

  • Debian 8.0-10.3
  • Ubuntu 14.04、16.04、18.04、19.10
  • RHEL 6.0-8.1
  • CentOS 6.0-8.1
  • Oracle Linux 6-8.1（RHCK）
  • Oracle Linux 6（从UEK R1开始）-Oracle Linux 7（直至UEK R5 U2）
  • SLES 11 SP4、12 SP1-15 SP1
  • 适用于SAP 11 SP4、12 SP1-15 SP1的SLES
  • Fedora 30-31
  • openSUSE Leap 15.1
  • openSUSE风滚草

笔记：

  • Fedora受内核版本5.4.14及更高版本支持。
  • openSUSE Tumbleweed在内核版本5.1.7之前（包括该版本）受支持。
  • CentOS / RHEL和Oracle Linux（RHCK）不支持Linux内核2.6.32-754.6.3。

关于免费版的限制


对于Linux版Veeam代理（与Server的商业版相比）：

  1. 只能创建1个备份作业。
  2. 不支持将Veeam Cloud Connect存储库用作目标备份存储库。
  3. 备份作业不支持预冻结和解冻后脚本。
  4. 不支持考虑DBMS Oracle，MySQL，PostgreSQL的工作的备份。
