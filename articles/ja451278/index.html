前書き


PyWavelets PyWavelets 1.0.3ライブラリに実装されている離散ウェーブレット変換（DWT）を考えてみましょう。 PyWaveletsは、MITライセンスの下でリリースされた無料のオープンソースソフトウェアです。

コンピューターでデータを処理する場合、連続ウェーブレット変換の離散化バージョンを実行できます。その基本については、前回の記事で説明しました。ただし、任意のステップΔaおよびΔbでウェーブレットのパラメーター（a、b）の離散値を指定するには、多数の計算が必要です。

さらに、結果は、元の信号のサンプル数をはるかに超える過剰な数の係数であり、その再構成には必要ありません。

PyWaveletsライブラリに実装された離散ウェーブレット変換（DWT）は、信号解析とその合成の両方に十分な情報を提供すると同時に、操作の数と必要なメモリの点で経済的です。

フーリエ変換の代わりにウェーブレット変換を使用する場合


フーリエ変換は、周波数スペクトルが静止している場合に非常にうまく機能します。この場合、信号に存在する周波数は時間に依存せず、信号には信号のどこかに存在する周波数xHzが含まれます。信号が不安定であるほど、結果は悪化します。これは問題です。現実の生活で目にする信号のほとんどは、実際には不安定です。

フーリエ変換は、周波数領域では高分解能ですが、時間領域ではゼロ分解能です。これを次の2つの例で示します。

上場
import numpy as np
from scipy import fftpack
from pylab import*
N=100000
dt = 1e-5
xa = np.linspace(0, 1, num=N)
xb = np.linspace(0, 1/4, num=N/4) 
frequencies = [4, 30, 60, 90]
y1a, y1b = np.sin(2*np.pi*frequencies[0]*xa), np.sin(2*np.pi*frequencies[0]*xb)
y2a, y2b = np.sin(2*np.pi*frequencies[1]*xa), np.sin(2*np.pi*frequencies[1]*xb)
y3a, y3b = np.sin(2*np.pi*frequencies[2]*xa), np.sin(2*np.pi*frequencies[2]*xb)
y4a, y4b = np.sin(2*np.pi*frequencies[3]*xa), np.sin(2*np.pi*frequencies[3]*xb)
def spectrum_wavelet(y):    
    Fs = 1 / dt  # sampling rate, Fs = 0,1 MHz 
    n = len(y)  # length of the signal
    k = np.arange(n)
    T = n / Fs
    frq = k / T  # two sides frequency range
    frq = frq[range(n // 2)]  # one side frequency range
    Y = fftpack.fft(y) / n  # fft computing and normalization
    Y = Y[range(n // 2)] / max(Y[range(n // 2)])
    # plotting the data
    subplot(2, 1, 1)
    plot(k/N , y, 'b')    
    ylabel('Amplitude')
    grid()
    # plotting the spectrum
    subplot(2, 1, 2)
    plot(frq[0:140], abs(Y[0:140]), 'r')
    xlabel('Freq')
    plt.ylabel('|Y(freq)|')
    grid() 
y= y1a + y2a + y3a + y4a
spectrum_wavelet(y)
show()


このグラフでは、4つの周波数すべてが、その動作の全時間にわたって信号に存在しています。

このグラフでは、信号は時間的にオーバーラップしていません。サイドローブは、4つの異なる周波数間のギャップが原因です。

まったく同じ4つのピークを含む2つの周波数スペクトルの場合、フーリエ変換では、これらの周波数が信号のどこに存在するかを判別できません。動的周波数スペクトルで信号を分析するための最良のアプローチは、ウェーブレット変換です。

ウェーブレットの主な特性


タイプの選択、したがってウェーブレットのプロパティは、分析のタスクに依存します。たとえば、電力業界での電流の実効値を決定するために、イングリッドドーベシーウェーブレットによってより高い精度が提供されます。ウェーブレットプロパティは、次のリストのpywt.DiscreteContinuousWavelet（）関数を使用して取得できます。

上場

title("  - -%s"%st)
plt.plot(x,phi,linewidth=2, label='level=%s'%i)
legend(loc='best')
grid()
show()

信号拡張モード


フィルターバンク を使用して離散ウェーブレット変換を計算する前に、信号を長くする必要があります。外挿法によっては、信号境界で重大なアーチファクトが発生し、DWT変換が不正確になる可能性があります。

PyWaveletsは、この悪影響を最小限に抑えるために使用できるいくつかの信号外挿手法を提供します。このような方法を説明するには、次のリストを使用します。

近似係数（cA）は、ローパスフィルター（平均化フィルター）DWTの出力を表します。詳細係数（cD）は、ハイパスフィルター（差動フィルター）DWTの出力を表します。

pywt.wavedec（）関数を使用して、より高いレベルの係数をすぐに計算できます。この関数は、入力信号とレベルを入力として取り、1セットの近似係数（n番目のレベル）とnセットの詳細係数（1からn番目のレベル）を返します。5番目のレベルの例を次に示します。

from pywt import wavedec
from pylab import *
from numpy import *

フィルターバンク


コンバージョンレベルに関連するいくつかの問題については、前のセクションで説明しました。ただし、DWTは常に、ハイパスフィルターとローパスフィルターのカスケード形式のフィルターバンクとして実装されます。フィルターバンクは、信号を複数の周波数サブバンドに分割する非常に効果的な方法です。

小規模の最初の段階で、信号の高周波動作を分析します。 2番目の段階では、スケールは2倍に増加し（周波数は2倍に減少）、最大周波数の約半分の動作を分析します。 3番目の段階では、スケール係数は4であり、最大周波数の約1/4の周波数動作を分析します。そして、これは最大レベルの分解に達するまで続きます。

分解の最大レベルはpywt.wavedec（）関数を使用して計算できますが、分解と詳細は次のようになります。

上場

n_level=pywt.dwt_max_level(len(data), 'sym5')

x = linspace (0,  1,  num = 2048)
y = sin (250  *  pi * x**2)
st='sym5'
coeffs = wavedec(y, st, level=7)

grid()

subplot(2, 1, 2)

grid()
legend(loc='best')
show()


我々が得る：

最大分解レベル：7



信号が与えられたsym5ウェーブレットのフィルタ長さよりも短くなると分解が停止します。たとえば、最大1000 Hzの周波数を持つ信号があるとします。最初の段階では、信号を低周波と高周波の部分、つまり0〜500 Hzと500〜1000 Hzに分離します。2番目のステージでは、低周波数部分を取得して、0〜250 Hzと250〜500 Hzの2つの部分に再度分割します。3番目のステージでは、0〜250 Hzの部分を0〜125 Hzの部分と125〜250 Hzの部分に分けました。これは、分解の最大レベルに達するまで続きます。

FFTフーリエとウェーブレットスカログラムを使用したwavファイルの分析


分析には、WebSDRファイルを使用します。 scipy.signalのtriangを使用して削減された信号の分析と、Pythonでの離散フーリエ変換の実装（scipy.fftpackからのfft）を検討してください。シーケンスの長さfftが2nに等しくない場合、高速フーリエ変換（fft）の代わりに、離散フーリエ変換（dft）が実行されます。これがこのチームのしくみです。

次のスキームに従って高速フーリエ変換バッファーを使用します（例として数値データ）

。fftbuffer = np.zeros（15）;ゼロで満たされたバッファを作成します。
fftbuffer [：8] = x [7：]; 信号の終わりをバッファの最初の部分に移動します。fftbuffer [8：] = x [：7] —信号の先頭をバッファの最後の部分に移動します。X = fft（fftbuffer）-信号値で満たされたバッファーのフーリエ変換を検討します。

位相スペクトルを読みやすくするために、位相展開を適用できます。これを行うには、位相特性の計算で行を変更します：pX = np.unwrap（np.angle（X））。

FFT信号フラグメント分析のリスト
import numpy as np
from pylab import *
from scipy import *
import scipy.io.wavfile as wavfile
M=501
hM1=int(np.floor((1+M)/2))
hM2=int(np.floor(M/2))
(fs,x)=wavfile.read('WebSDR.wav')
x1=x[5000:5000+M]*np.hamming(M)
N=511
fftbuffer=np.zeros([N])
fftbuffer[:hM1]=x1[hM2:]
fftbuffer[N-hM2:]=x1[:hM2]
X=fft(fftbuffer)
mX=abs(X)
pX=np.angle(X)
suptitle(" WebSDR")
subplot(3, 1, 1)
st='  (WebSDR.wav)'
plot(x,linewidth=2, label=st)
legend(loc='center')
subplot(3, 1, 2)
st='    '
plot(mX,linewidth=2, label=st)
legend(loc='best')
subplot(3, 1, 3)
st='    '
pX=np.unwrap(np.angle(X))
plot(pX,linewidth=2, label=st)
legend(loc='best') 
show()



比較分析では、matplotlibのtree = pywt.wavedec（signal、 'coif5'）関数を使用して構築できるウェーブレットスカログラムを使用します。

ウェーブレットスカログラムのリスト
from pylab import *
import pywt
import scipy.io.wavfile as wavfile
#     ,     .
def lepow2(x):    
    return int(2 ** floor(log2(x)))
#    MRA.
def scalogram(data):
    bottom = 0
    vmin = min(map(lambda x: min(abs(x)), data))
    vmax = max(map(lambda x: max(abs(x)), data))
    gca().set_autoscale_on(False)
    for row in range(0, len(data)):
        scale = 2.0 ** (row - len(data))
        imshow(
            array([abs(data[row])]),
            interpolation = 'nearest',
            vmin = vmin,
            vmax = vmax,
            extent = [0, 1, bottom, bottom + scale])
        bottom += scale
#  ,   .
rate, signal = wavfile.read('WebSDR.wav')
signal = signal[0:lepow2(len(signal))]
tree = pywt.wavedec(signal, 'coif5')
gray()
scalogram(tree)
show()




したがって、スカログラムは、時間内の周波数の分布の問題に対するより詳細な回答を提供し、高速フーリエ変換は周波数自体に責任があります。そのような単純な例であっても、それはすべてタスクに依存します。
