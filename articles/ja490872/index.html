トリオ-人々のための非同期プログラミング



Pythonには、非同期プログラミングライブラリであるTrioライブラリがあります。
Trioを理解することは、Asyncioで処理できない問題のいくつかを解決できる優れた代替手段であるため、主にAsyncioに取り組む人々にとって興味深いものになります。このレビューでは、トリオとは何か、トリオがどのような機能を提供するかを検討します。

非同期プログラミングを始めたばかりの人のために、非同期と同期とは何かについての簡単な紹介を読むことを提案します。

同期と非同期


では、同期プログラミング、すべての操作が順次実行され、そしてあなただけの前の1を完了した後に新しいタスクを開始することができます。ただし、その「問題点」の1つは、スレッドの1つが非常に長期間タスクに取り組んでいる場合、プログラム全体がフリーズする可能性があることです。計算能力を必要としないが、プロセッサ時間を消費するタスクがあります。これは、別のスレッドに制御を与えることにより、より合理的に使用できます。同期プログラミングでは、現在のタスクを一時停止して、次のギャップを埋める方法はありません。非同期

とは何ですか？？真の非同期と入出力非同期を区別する必要があります。私たちのケースでは、非同期入出力について話しています。グローバル-時間を節約し、生産設備をより効率的に使用するため。非同期では、スレッドの問題領域をバイパスできます。非同期入出力では、現在のスレッドは外部イベントの実行を待たずに、別のスレッドに制御を渡します。したがって、実際には、一度に実行されるスレッドは1つだけです。制御を与えたスレッドはキューに入り、制御がそこに戻るのを待ちます。おそらく、その時までに、予期される外部イベントが発生し、作業を継続することが可能になります。これにより、時間の無駄を最小限に抑えるためにタスクを切り替えることができます。

そして今、私たちは何であるかに戻ることができます非同期。このライブラリイベントループ（イベントループ）の操作には、タスクキューとループ自体が含まれます。サイクルはタスクの実行を制御します。つまり、キューからタスクを引き出し、それに何が起こるかを決定します。たとえば、I / Oタスクを処理している場合があります。つまり、イベントループはタスクを選択して登録し、適切なタイミングで処理を開始します。

コルーチンは、このタスクの制御をイベントループに戻す、つまりキューに戻す特別な関数です。これらのコルーチンは、一連のイベントを通じて正確に起動される必要があります。先物

もあります-タスクの実行の現在の結果が格納されるオブジェクト。これは、タスクがまだ処理されていないか、結果がすでに取得されているという情報である可能性があります。または例外がある可能性があります。

一般に、Asyncioライブラリはよく知られていますが、Trioが終了できるという多くの欠点があります。

トリオ


ライブラリの作者であるNathaniel Smith氏は、Trioを開発する際に、開発者向けの軽量で簡単に使用できるツールを作成しようとしました。

Trioの重要な機能は、Asyncioにはない非同期コンテキスト管理です。これを行うために、著者はトリオでいわゆる「保育園」を作成しました（保育園）-スレッドのグループの原子性（連続性）に責任を持つキャンセルの領域。重要なアイデアは、「ナーサリ」でコルーチンの1つが失敗した場合、「ナーサリ」内のすべてのフローが正常に完了するかキャンセルされるということです。いずれにしても、結果は正しいものになります。そして、すべてのコルーチンが完了したときのみ、関数を終了した後、開発者自身が続行方法を決定します。

つまり、「子供用」を使用すると、エラー処理の継続を防ぐことができます。これにより、すべてが「落下」するか、結果が正しくない結果になる可能性があります。
これは、Asyncioではエラーが発生したにもかかわらず、プロセスが停止しないため、Asyncioで発生する可能性があります。この場合、最初に、開発者はエラーの発生時に正確に何が起こったのかを知りません。次に、処理が続行されます。

例


2つの競合する機能の最も単純な例を考えてみましょう：

Asyncio

import asyncio

async def foo1():
    print('  foo1: ')
    await asyncio.sleep(2)
    print('  foo1: ')

async def foo2():
    print('  foo2: ')
    await asyncio.sleep(1)
    print('  foo2: ')

loop = asyncio.get_event_loop()
bundle = asyncio.wait([
    loop.create_task(foo1()),
    loop.create_task(foo2()),
])
try:
    loop.run_until_complete(bundle)
finally:
    loop.close()

トリオ

import trio

async def foo1():
    print('  foo1: ')
    await trio.sleep(2)
    print('  foo1: ')

async def foo2():
    print('  foo2: ')
    await trio.sleep(1)
    print('  foo2: ')

async def root():
    async with trio.open_nursery() as nursery:
        nursery.start_soon(foo1)
        nursery.start_soon(foo2)

trio.run(root)

Trioでは、エラーが発生した直後に「キャンセル領域」が機能し、エラーを含まない4つのタスクのうち2つが異常終了したことがはっきりとわかります。

Asyncioでは、すべてのタスクが完了し、その後トラックバックが表示されました。

与えられた例では、これは重要ではありませんが、何らかの方法でタスクが相互に依存しており、タスクのセットが原子性のプロパティを持っている必要があると想像してみましょう。この場合、エラーに対するタイムリーな応答がはるかに重要になります。もちろん、await asyncio.wait（tasks、return_when = FIRST_EXCEPTION）を使用できますが、開いているタスクを正しく完了することを覚えておく必要があります。

次に、別の例を示します。

コルーチンが複数の同様のWebサービスに同時にアクセスし、最初に受け取った応答が重要であるとします。

import asyncio
from asyncio import FIRST_COMPLETED
import aiohttp

URL = 'https://yandex.ru/time/sync.json?geo=213'
MAX_CLIENTS = 5

async def foo(session):
    async with session.get(URL) as response:
        content = await response.json()
        return content.get("time")

async def root():
    async with aiohttp.ClientSession() as session:
        tasks = [
            asyncio.ensure_future(foo(session))
            for i in range(1, MAX_CLIENTS + 1)
        ]
        done, pending = await asyncio.wait(tasks, return_when=FIRST_COMPLETED)
        print(done.pop().result())
        for future in pending:
            future.cancel()

ioloop = asyncio.get_event_loop()
try:
    ioloop.run_until_complete(root())
except:
    ioloop.close()

すべてがかなり簡単です。唯一の要件は、完了していないタスクを完了することを覚えておくことです。

Trioでは、同様の操作をクランキングすることは少し難しいですが、「テール」をすぐに非表示にしておくことはほとんど不可能です。

import trio
import asks
URL = 'https://yandex.ru/time/sync.json?geo=213'
MAX_CLIENTS = 5
asks.init('trio')

async def foo(session, send_channel, nursery):
    response = await session.request('GET', url=URL)
    content = response.json()
    async with send_channel:
        send_channel.send_nowait(content.get("time"))
    nursery.cancel_scope.cancel()

async def root():
    send_channel, receive_channel = trio.open_memory_channel(1)
    async with send_channel, receive_channel:
        async with trio.open_nursery() as nursery:
            async with asks.Session() as session:
                for i in range(MAX_CLIENTS):
                    nursery.start_soon(foo, session, send_channel.clone(), nursery)

        async with receive_channel:
            x = await receive_channel.receive()
            print(x)

trio.run(root)

nursery.cancel_scope.cancel（） -完了した最初のコルーチンは、他のすべてのタスクをキャンセルする元に戻す領域の関数を呼び出すため、個別に心配する必要はありません。
確かに、コルーチンの実行結果をその原因となった関数に転送するには、通信チャネルを開始する必要があります。
