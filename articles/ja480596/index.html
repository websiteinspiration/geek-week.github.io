革新的なネットワークオペレーティングシステムArubaOS-CXであなたを待っているもの



テクノロジーの急速な発展とビジネスへの影響にもかかわらず、過去20年間の企業のネットワークインフラストラクチャは、設計とソフトウェアの分野で大きな革新を遂げていません。

いくつかの試みが行われましたが、2000年代の初め以来、ビジネスはITに主に問題を解決してビジネスプロセスのデジタルサブシステムの増加との相互作用を確実にし、結果として増大するネットワークトラフィックの伝送を確実にすることを要求しました。

実際、過去20年間で速度は大幅に向上しています。機能開発を含む、ビジネスの指示。安定したITインフラストラクチャの重要性は、スイッチボックスまたはネットワーク全体のレベルでフォールトトレランスを確保するためのより厳しい要件で表現されました。ネットワークの情報セキュリティの基本要素を含む、スイッチおよび通信チャネルのリソースの使用を最適化することで、貪欲と常識が表現されました。

同時に、ネットワークインフラストラクチャの運用に関してはほとんど変更されていません（誰がエンジニアについて考えるのでしょうか）。そのための主な要件は、スイッチ、通知メカニズム、およびジャーナリングと対話するためのインターフェイスの存在でした。 20年の間、時間は凍結したように思われました。CLIによる従来の取り外し不可能な手動操作、syslog収集、オフライン分析、SNMP経由の通知、およびプログラマビリティとソフトウェアインターフェイスの原則がないため、サードパーティのITシステムとの複雑な統合。

ダンジョンの外を見るとどうなりますか？ネットワークインフラストラクチャの肩にかかっているITの外の世界は、モビリティ、クラウドテクノロジー、モノのインターネット（IoT）の時代が到来し、大きく変化しました。 WEBテクノロジ、プログラミングアプローチ、ユーザーアプリケーションアーキテクチャ、ソフトウェア（IT）サービス間の相互作用、情報ストレージアプローチ（BigData）などが大幅に開発されました。

変化とは、柔軟性と持続可能な成長に対するビジネス要求へのIT業界の対応です。現代の現実に特徴的なビジネスプロセスのデジタル化には、最終的なビジネス指標に何らかの影響を与えるすべての人にとって便利なツールが必要です。

現在、オフィスワーカーツールとは何ですか？これは、最新のテクノロジーを使用して便利なユーザーインターフェイス、高速な機能開発、およびスケーラビリティを実装する特殊なソフトウェア（技術複合体としてのアプリケーションとプラットフォーム）です。

情報ネットワークはすべてのプロセスの主要なトランスポートであり、管理者はビジネスの最後の人々ではありません。彼らはまた、現代の技術で働きたいです！しかし、これ自体が目的ではなく、ネットワークインフラストラクチャを運用/実装するときに、管理者にできるだけ多くの機会を与えることが必要です。したがって、その後の管理のためにオブジェクトを選択するときは、次の特性に注意することが非常に重要です。

プロセス相互作用アーキテクチャ


マイクロサービスのスタイルでモジュール式アーキテクチャを使用し、データベースを介したプロセスの相互作用は、ArubaOS-CXの主要な利点の1つです。
データベースは中心的なコンポーネントです。プロセス間の相互作用を提供し、プロセスの相互依存や他のコンポーネントへの依存を減らすことができます。

このアプローチは、プロセスが直接メッセージを交換し、プロセス間通信と依存関係の複雑なマトリックスを形成する、ネットワークオペレーティングシステムの従来のアーキテクチャとは異なります。 ArubaOS-CXでは、データベースはプロセス間の通信の点で並外れた役割を担っています。プロセスによって生成されたすべての状態とイベントはデータベースに書き込まれ、そこからOSの他のプロセスがそれらを読み取ることができます。

さらに、ベースは、プロセスが「サブスクライブ」して、別の特定のプロセスで発生したイベントに関する情報を即座に受信できる相互作用モデルをサポートします。このアーキテクチャでは、次のシナリオが機能します。たとえば、その動作中、ARPおよびルーティングプロセスはデータベーステーブルのみを更新し、特殊なプロセスはデータベースから集約された情報を受け取り、ASICラインカードをプログラムします。

別の例は、プロセスがフリーズして再起動することです。データベースに保存されているそのステータスに関する情報は、プロセスを復元するとき、および/または最初のプロセスが復元されるまで、別のプロセスの最新の関連データのソースとして使用できます。

このアプローチは、OS内の個別の要素、およびスイッチ全体、そして結果としてネットワークとして、高い信頼性のための肥沃な基盤を作り出します。

オープンソフトウェアインターフェイス


プロセスに関するすべての情報（構成、現在のステータス、統計情報）は構造化された形式でデータベースに保存されるため、APIを介した外部呼び出しを使用してデータへのアクセスを整理するのは非常に簡単です。ArubaOS-CXは完全にプログラム可能であり、ほとんどすべての機能に対してREST APIをサポートしています。ここから、スイッチと対話するより近代的な方法が利用可能になり、サードパーティのITシステムと統合するための幅広い予測可能な機会、試運転からメンテナンスまでのすべての段階での自動化の機会が増えます。

組み込みの監視および分析ツール


今日のシナリオ：IT部門は、CLIやSNMPなどのツールを使用するか、サードパーティの監視、分析、診断システムを使用して、事後のトラブルシューティングとトラブルシューティングを行うことができます。

有能なアプローチにより、トラブルシューティングプロセスは段階に分かれています。問題の性質または症状を考慮して、失敗する可能性があると仮定し、その仮定を確認または否定するために情報を収集します。通常、これはコンソールまたはSSHターミナルによる手動診断です。収集された情報に基づいて、問題を排除するための主要なアクションプランが開発および実装されます。次に、問題が解決したかどうかを確認します。問題が解決しない場合は、次の仮定が進められます。

Network Analytics Engine-ArubaOS-CXに組み込まれたツール-情報の収集とその後の分析のプロセスを自動化し、24時間365日のインテリジェントな監視を提供し、誤動作の原因の発見とインシデントの解決プロセスをスピードアップします。

NAEは、管理者が定義したパラメーター（トラフィック量、スイッチコンポーネントのヘルスインジケーターなど）を継続的に監視します。 NAEは、イン​​ジケーターを測定し、統計を時系列データベースに格納し、グラフの形で視覚化し、グラフを機器のイベントや管理者のアクション（構成の変更など）と関連付けて、履歴トレンドを追跡できます。

NAEを使用すると、管理者はネットワークイベントを監視し、セキュリティの脆弱性、パフォーマンスのボトルネックを特定して、プロアクティブに対処できます。 NAE機能を持つことは、24時間年中無休でトラブルシューティングを行うエンジニアがいるようなものです。収集された一連のデータで異常が検出された場合、NAEは従来のSyslogを通じてアラートを生成したり、APIを使用してケースを開始したり、スイッチでローカルにアクションを実行したり、APIを通じて近隣のスイッチにクエリを実行したりできます。

そのようなスマートな分析のために、NAEはエージェントを使用します。 Pythonプログラミング言語に精通していて、その使用方法を知っている人にとっては、NAE用のエージェントスクリプトを書くのは簡単な作業です。一部の人にとっては、ネットワークインフラストラクチャの運用におけるプログラミングの研究と適用を開始することが、優れた推進力になるでしょう。

スイッチ仮想化テクノロジー


Virtual Switching Extension（VSX）は、ArubaOS-CXオペレーティングシステムを実行するディストリビューション/カーネルレベルスイッチ用の仮想化テクノロジーです。ネットワーク業界では、コアおよび集約レベルでのフォールトトレランスを確保するための仮想化テクノロジーの実装において、2つの主要なアプローチを区別できます。

1つ目は、すべての観点から論理的に統合された仮想スイッチの作成です。2つ目は、コントロールプレーンと管理プレーンに影響を与えずにデータプレーンを仮想化するテクノロジーを使用しています。

アプローチに関係なく、目標は、仮想化に参加しているさまざまなスイッチ（スタック）から取得した物理ポートの集約を通じて分散論理チャネルを作成する機能です。 VSXを実装する際、両方のアプローチの欠点を利用して排除しました。

標準の仮想化メカニズム（VSF、VSS、IRF）またはスタックとは異なり、VSXでは、L3への独立性を維持しながら、OSIモデルの第2レベルで2つのスイッチを単一のデバイスとして機能させることができます。管理の独立性も維持されますが、各スイッチを個別に構成する必要はありません。構成の同期が可能です。管理者はVirtual Switching Extensionを使用して、フォールトトレランスと負荷のバランスを取り、トラフィックに最適なルートを使用する機能を維持しながら、遅い収束と複雑な設定（STP、VRRP）でレガシーバックアッププロトコルを放棄できます。

Aruba CX 8400スイッチシリーズ


Aruba 8400スイッチはコンパクトな8Uシャーシで、10GbE（SFP / SFP +）、40GbE（QSFP +）、40GbE / 100GbE（QSFP + / QSFP28）ポートを備えた8つのインターフェイスモジュールを取り付けることができます。高性能工場は、最大1.2 Tbpsのスロットパフォーマンスを提供し、成長の可能性を秘めています。ハードウェアプラットフォームのアーキテクチャの観点から見たAruba 8400は、エンタープライズネットワークの要件だけでなく、制御サブシステム（1 + 1）、電源（N + N）、冷却（N + 1）およびスイッチングファクトリ（ N + 1）。

Aruba CX 8325スイッチシリーズ
