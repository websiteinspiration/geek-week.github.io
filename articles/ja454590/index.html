Zimbra Collaboration SuiteおよびABQを備えたモバイルコントロール

ポータブルエレクトロニクス、特にスマートフォンやタブレットの急速な発展により、企業の情報セキュリティに多くの新たな課題が生じています。実際、すべてのサイバーセキュリティが安全な境界の作成とその後の保護に基づいていた場合、ほとんどすべての従業員が自分のモバイルデバイスを使用して作業タスクを解決するようになったため、セキュリティ境界の制御は非常に困難になりました。これは、各従業員が電子メールやその他の企業リソースからのユーザー名とパスワードを持っている大企業に特に当てはまります。多くの場合、新しいスマートフォンやタブレットを購入するとき、会社の従業員はその上に資格情報を入力し、しばしば古いデバイスからログアウトするのを忘れます。企業にそのような無責任な従業員が5％しかいなくても、管理者による適切な制御なしでは、モバイルデバイスがメールサーバーにアクセスする状況はすぐに本当の混乱に変わります。



さらに、モバイルデバイスは紛失または盗難に遭ったことが多く、その後、侵害情報の検索や、企業リソースや企業秘密を表すデータへのアクセスに使用されます。原則として、攻撃者が従業員の電子メールにアクセスできる場合、企業のサイバーセキュリティは最も有害です。これにより、住所や連絡先のグローバルリスト、不運な従業員が参加する必要がある会議のスケジュール、および彼の通信にアクセスできます。さらに、企業のメールにアクセスする攻撃者は、信頼できるメールアドレスからフィッシングやマルウェアに感染したメッセージを送信することができます。これらすべてが組み合わさって、攻撃者はサイバー攻撃のほぼ無制限の機会を手に入れます。そして彼らの目標を達成するためのソーシャルエンジニアリングの使用。

セキュリティ境界に含まれるモバイルデバイスを制御するために、ABQテクノロジ、または許可/ブロック/検疫があります。これにより、管理者は、メールサーバーとデータを同期できるモバイルデバイスのリストを制御し、必要に応じて、侵害されたデバイスをブロックし、疑わしいモバイルデバイスを隔離できます。

ただし、Zimbra Collaboration Suite Open-Source Editionの無料版の管理者なら誰でも知っているように、モバイルデバイスとのやり取りは非常に限られています。厳密に言うと、Zimbraの無料バージョンのユーザーは、POP3またはIMAPを介してのみレターを送受信でき、日記、アドレス帳、メモのデータをサーバーと同期する組み込み機能はありません。 Zimbra Collaboration SuiteとABQテクノロジーの無料バージョンには実装されていません。これにより、企業内に閉じた情報境界を作成するすべての試みが自動的に終了します。管理者が自分のサーバーに接続しているデバイスを知らない状況では、情報漏えいが企業で発生する可能性があり、上記のシナリオに従ってサイバー攻撃の可能性が急激に高まります。

Zextras Mobileモジュラー拡張機能は、Zimbra Collaboration Suite Open-Source Editionでこの問題を解決するのに役立ちます。この拡張機能を使用すると、ActiveSyncプロトコルの完全なサポートをZimbraの無料バージョンに追加できます。これにより、モバイルデバイスとメールサーバー間のやり取りの多くの機会が開かれます。その他のさまざまな機能の中で、エクステンションZextras MobileはABQを完全にサポートしています。

正しく設定されていないABQは一部のユーザーがモバイルデバイスのデータをサーバーと同期できない原因となる可能性があるため、セットアップの問題には細心の注意を払って対処する必要があることを直ちに警告します。 ABQの構成は、Zextrasコマンドラインから行います。 ZimbraのABQ動作モードが構成されているのはコマンドライン上であり、デバイスリストも管理されます。

これは次のように実装されます。ユーザーがモバイルデバイスで企業メールにログインすると、サーバーに認証データとデバイスの資格情報を送信します。これにより、ABQの形式で障害に遭遇し、資格情報をスキャンしてそれらと比較します。許可、検疫、ブロックされたデバイスのリストに含まれている。デバイスがリストのいずれにもない場合、ABQはデバイスが動作するモードに従ってそれを入力します。

ZimbraのABQは、3つの動作モードが用意されています

寛容なの：この操作モードでは、ユーザー認証後、モバイルデバイスからの最初の要求時に自動的に同期が実行されます。この操作モードでは、個々のデバイスをブロックすることができますが、他のすべてのユーザーはサーバーとデータを自由に同期できます。

インタラクティブ：この操作モードでは、ユーザー認証の直後に、セキュリティシステムがデバイス識別データを要求し、それらを許可されたデバイスのリストと比較します。デバイスが許可済みとしてリストされている場合、同期は自動的に続行されます。このデバイスが「ホワイトリスト」にない場合は、自動的に隔離されるため、管理者は後でこのデバイスをサーバーと同期させるかブロックするかを決定できます。この場合、ユーザーに通知が送信されます。管理者への通知は、構成可能な期間に1回、定期的に行われます。この場合、新しい通知にはそれぞれ、隔離された新しいデバイスのみが含まれます。

厳格：この操作モードでは、ユーザー認証後、すぐに許可リストにデバイス識別データが存在するかどうかを確認します。そこに表示される場合、同期は自動的に続行されます。デバイスが許可リストにない場合は、すぐにブロックリストに分類され、対応する通知がメールでユーザーに送信されます。

また、必要に応じて、Zimbra管理者はメールサーバーでABQを完全に無効にすることができます。

ABQ動作モードの設定は、次のコマンドを使用して実行されます。
zxsuite config global set attribute abqMode value Permissive
zxsuite config global set attribute abqMode value Interactive
zxsuite config global set attribute abqMode value Strict
zxsuite config global set attribute abqMode value Disabled

コマンドzxsuite config global get attribute abqModeを使用して、ABQの現在の動作モードを確認できます。

インタラクティブモードまたは厳密なABQモードを使用する場合、許可されているデバイスとブロックされているデバイス、および検疫されているデバイスのリストを操作する必要があることがよくあります。 2つのデバイスがサーバーに接続されているとします。1つはiPhone、もう1つは対応する識別データを持つAndroidです。その後、企業のCEOが先日iPhoneを入手してメールで作業することを決めたことが判明し、Androidは通常のマネージャーに属します。マネージャーはセキュリティ上の理由からスマートフォンで作業メールを使用する権利がありません。

インタラクティブモードの場合、それらすべてが隔離され、管理者はそこからiPhoneを許可されたデバイスのリストに、Androidをブロックされたデバイスのリストに転送する必要があります。これを行うには、コマンドzxsuite mobile abq allow iPhoneおよびzxsuite mobile abq block Androidを使用します。その後、CEOは自分のデバイスからのメールを完全に操作できるようになりますが、マネージャーは仕事用のノートパソコンからのみメールを表示する必要があります。

インタラクティブモードを使用している場合、マネージャーがAndroidデバイスにユーザー名とパスワードを正しく入力しても、アカウントにアクセスできませんが、仮想メールボックスに入るので、通知を受け取ります。彼のデバイスは隔離され、彼からのメールは使用できなくなります。



ストリクトモードの場合、すべての新しいデバイスがブロックされ、それらが誰に属しているかが明らかになった後、管理者はzxsuiteモバイルABQ set iPhone Allowedコマンドを使用して許可されたiPhoneデバイスのリストにCEOを追加するだけでよく、マネージャーの電話はそこに残されます。

許容動作モードは企業のセキュリティルールとの互換性が低いですが、許可されたモバイルデバイスのいずれかを引き続きブロックする必要がある場合、たとえばマネージャーがスキャンダルで突然終了した場合は、zxsuite mobile ABQ set Androidコマンドを使用してこれを実行できますブロックされました。

従業員がオフィスガジェットを受け取ってメールを操作する場合、所有者が次に変更したときに、デバイスをABQリストから完全に削除して、サーバーとの同期を許可するかどうかを後で決定できます。これは、zxsuiteモバイルABQのAndroidの削除コマンドを使用して行われます。
