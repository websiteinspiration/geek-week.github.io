データベースのクライアントを作成する例によるGoでのコード生成

この記事では、Golangでのコード生成の問題について検討します。Goに関する記事のコメントで、コードの生成とリフレクションに言及していることが多いことに気付きました。同時に、ハブでのコード生成に関する記事はほとんどありませんが、Goのプロジェクトではかなり使用されています。記事では、コード生成とは何か、アプリケーションの範囲をコード例で説明します。また、反射は無視しません。

コード生成を使用する場合


Habréで話題に良い記事が既に存在しているこことここで、私は繰り返しません。

コード生成は次の場合に使用する必要があります。

  • コードの速度を上げる、つまりリフレクションを置き換える。
  • プログラマーのルーチン（およびそれに関連するエラー）を減らす。
  • 与えられたルールに従ったラッパーの実装。

例から、標準言語の提供に含まれ、数値定数のセットのString（）メソッドを自動的に生成できるStringerライブラリを検討できます。その助けを借りて、変数名の出力を実装できます。ライブラリの例は、上記の記事で詳細に説明されています。最も興味深い例は、パレットからのカラー名の派生でした。そこでコード生成を適用すると、パレットを変更するときにいくつかの場所でコードを変更する必要がなくなります。

より実用的な例として、Mail.ruのeasyjsonライブラリについて触れます。このライブラリーを使用すると、構造体への/からのJSONのmasrshall / unmarshallの実行を高速化できます。ベンチマークでのそれらの実装は、すべての選択肢をバイパスしました。ライブラリを使用するには、easyjsonを呼び出す必要があります。これにより、転送されたファイルで見つかったすべての構造、または// easyjson：jsonにコメントされている構造に対してのみコードが生成されます。例としてユーザー構造を考えます。

type User struct{
    ID int
    Login string
    Email string
    Level int
}

それが含まれているファイルについて、コード生成を実行します。

easyjson -all main.go

その結果、Userのメソッドを取得します。

  • MarshalEasyJSON（w * jwriter.Writer）-構造をJSONバイトの配列に変換します。
  • UnmarshalEasyJSON（l * jlexer.Lexer）-バイトの配列から構造体に変換します。

関数MarshalJSON（）（[]バイト、エラー）およびUnmarshalJSON（データ[]バイト）エラーは、標準のjsonインターフェースとの互換性のために必要です。

Easyjsonコード 

func TestEasyJSON() {
	testJSON := `{"ID":123, "Login":"TestUser", "Email":"user@gmail.com", "Level":12}`
	JSONb := []byte(testJSON)
	fmt.Println(testJSON)
	recvUser := &User{}
	recvUser.UnmarshalJSON(JSONb)
	fmt.Println(recvUser)
	recvUser.Level += 1
	outJSON, _ := recvUser.MarshalJSON()
	fmt.Println(string(outJSON))
}


この関数では、最初にJSONを構造に変換し、1つのレベルを追加して、結果のJSONを出力します。easyjsonを使用したコード生成は、実行時リフレクションを取り除き、コードのパフォーマンスを向上させることを意味します。

コード生成は、gRPCを介して通信するマイクロサービスを作成するために積極的に使用されています。中間言語EDLを使用して、protobuf形式を使用してサービスのメソッドを記述します。サービスの説明の後、protocコンパイラが起動し、目的のプログラミング言語のコードが生成されます。生成されたコードでは、サーバーに実装する必要のあるインターフェースと、通信を編成するためにクライアントで使用されるメソッドを取得します。非常に便利なことに、サービスを単一の形式で記述し、各対話要素が記述されるプログラミング言語用のコードを生成できます。

また、コード生成はフレームワークの開発に使用できます。たとえば、アプリケーション開発者が記述する必要のないコードを実装するが、正しく動作するためには必要です。たとえば、フォームフィールドバリデーターを作成するには、ミドルウェアを自動的に生成し、DBMSのクライアントを動的に生成します。

Goコードジェネレーターの実装


Goでのコード生成のメカニズムがどのように機能するかを実際に調べてみましょう。まず、AST-抽象構文ツリーまたは抽象構文ツリーについて言及する必要があります。詳細については、ウィキペディアにアクセスできます。私たちの目的のために、プログラム全体がグラフの形で構築されていることを理解する必要があります。ここで、頂点はプログラミング言語の演算子でマップされ（マークされ）、葉は対応するオペランドでマッピングされます。

したがって、最初に、パッケージが必要になります：

go / ast
go / parser /
go / token /

コードでファイルを解析し、ツリーをコンパイルするには、次のコマンドを実行します


fset := token.NewFileSet()
node, err := parser.ParseFile(fset, os.Args[1], nil, parser.ParseComments)

ファイル名はコマンドラインの最初の引数から取得する必要があることを示します。また、コメントをツリーに追加するように求めます。

一般に、コード生成を制御するために、ユーザー（他のコードの生成に基づいたコードの開発者）はコメントまたはタグを使用できます（構造フィールドの近くに「json：」「」を記述します）。

例として、データベースを操作するためのコードジェネレータを記述します。コードジェネレーターは、転送されたファイルを調べ、対応するコメントのある構造を探し、データベースの相互作用のために構造（CRUDメソッド）のラッパーを作成します。パラメーターを使用します。

  • dbeコメント：{"table"： "users"}。構造レコードが置かれるテーブルを定義できます。
  • 構造のフィールドのdbeタグ。データベースのフィールド値と属性を配置する列の名前を指定できます：primary_keyおよびnot_null。これらは、テーブルを作成するときに使用されます。また、フィールド名には、列を作成しないように「-」を使用できます。

プロジェクトがまだ戦闘中ではなく、必要なチェックと保護の一部が含まれていないことを事前に予約します。興味があれば開発を続けます。

コードの生成を制御するためのタスクとパラメーターを決定したので、コードの作成を開始できます。

結果のツリーをバイパスし、最初のレベルの各要素を解析します。Goには、BadDecl、GenDecl、およびFuncDeclという解析用の定義済みのタイプがあります。

タイプ説明 
// A BadDecl node is a placeholder for declarations containing
// syntax errors for which no correct declaration nodes can be
// created.
//
BadDecl struct {
    From, To token.Pos // position range of bad declaration
}
// A GenDecl node (generic declaration node) represents an import,
// constant, type or variable declaration. A valid Lparen position
// (Lparen.IsValid()) indicates a parenthesized declaration.
//
// Relationship between Tok value and Specs element type:
//
// token.IMPORT *ImportSpec
// token.CONST *ValueSpec
// token.TYPE *TypeSpec
// token.VAR *ValueSpec
//
GenDecl struct {
    Doc *CommentGroup // associated documentation; or nil
    TokPos token.Pos // position of Tok
    Tok token.Token // IMPORT, CONST, TYPE, VAR
    Lparen token.Pos // position of '(', if any
    Specs []Spec
    Rparen token.Pos // position of ')', if any
}
// A FuncDecl node represents a function declaration.
FuncDecl struct {
    Doc *CommentGroup // associated documentation; or nil
    Recv *FieldList // receiver (methods); or nil (functions)
    Name *Ident // function/method name
    Type *FuncType // function signature: parameters, results, and position of "func" keyword
    Body *BlockStmt // function body; or nil for external (non-Go) function
}


構造に関心があるため、GenDeclを使用します。この段階では、FuncDeclが役立つ場合があります。この場合、関数の定義があり、それらをラップしますが、今では必要ありません。次に、各ノードのSpecs配列を見て、型定義フィールド（* ast.TypeSpec）を操作していることを確認します。これは構造（* ast.StructType）です。構造があると判断したら、コメント// dbeがあることを確認します。完全なツリートラバーサルコードと、使用する構造の定義を以下に示します。

				continue
			}
		} else {
			genParams.TableName = targetStruct.Name
		}

		targetStruct.GenParam = genParams

func generateMethods(reqStruct *StructInfo, out *os.File) {
	for _, spec := range reqStruct.Target.Specs {
		fmt.Fprintln(out, "")
		currType, ok := spec.(*ast.TypeSpec)
		if !ok {
			continue
		}
		currStruct, ok := currType.Type.(*ast.StructType)
		if !ok {
			continue
		}

		fmt.Printf("\tgenerating createTable methods for %s\n", currType.Name.Name)

		curTable := &TableInfo{
			TableName: reqStruct.GenParam.TableName,
			Columns:   make([]*ColInfo, 0, len(currStruct.Fields.List)),
		}

		for _, field := range currStruct.Fields.List {
			if len(field.Names) == 0 {
				continue
			}
			tableCol := &ColInfo{FieldName: field.Names[0].Name}
			var fieldIsPrimKey bool
			var preventThisField bool
			if field.Tag != nil {
				tag := reflect.StructTag(field.Tag.Value[1 : len(field.Tag.Value)-1])
				tagVal := tag.Get("dbe")
				fmt.Println("dbe:", tagVal)
				tagParams := strings.Split(tagVal, ",")
			PARAMSLOOP:
				for _, param := range tagParams {
					switch param {
					case "primary_key":
						if curTable.PrimaryKey == nil {
							fieldIsPrimKey = true
							tableCol.NotNull = true
						} else {
							log.Panicf("Table %s cannot have more then 1 primary key!", currType.Name.Name)
						}
					case "not_null":
						tableCol.NotNull = true
					case "-":
						preventThisField = true
						break PARAMSLOOP
					default:
						tableCol.ColName = param
					}

				}
				if preventThisField {
					continue
				}
			}
			if tableCol.ColName == "" {
				tableCol.ColName = tableCol.FieldName
			}
			if fieldIsPrimKey {
				curTable.PrimaryKey = tableCol
			}
			//Determine field type
			var fieldType string
			switch field.Type.(type) {
			case *ast.Ident:
				fieldType = field.Type.(*ast.Ident).Name
			case *ast.SelectorExpr:
				fieldType = field.Type.(*ast.SelectorExpr).Sel.Name
			}
			//fieldType := field.Type.(*ast.Ident).Name
			fmt.Printf("%s- %s\n", tableCol.FieldName, fieldType)
			//Check for integers
			if strings.Contains(fieldType, "int") {
				tableCol.ColType = "integer"
			} else {
				//Check for other types
				switch fieldType {
				case "string":
					tableCol.ColType = "text"
				case "bool":
					tableCol.ColType = "boolean"
				case "Time":
					tableCol.ColType = "TIMESTAMP"
				default:
					log.Panicf("Field type %s not supported", fieldType)
				}
			}
			tableCol.FieldType = fieldType
			curTable.Columns = append(curTable.Columns, tableCol)
			curTable.StructName = currType.Name.Name

		}
		curTable.generateCreateTable(out)

		fmt.Printf("\tgenerating CRUD methods for %s\n", currType.Name.Name)
		curTable.generateCreate(out)
		curTable.generateQuery(out)
		curTable.generateUpdate(out)
		curTable.generateDelete(out)
	}
}


目的の構造のすべてのフィールドを調べ、各フィールドのタグの解析を開始します。リフレクションを使用して、関心のあるタグを取得します（結局のところ、jsonなど、フィールドに他のタグがある可能性があります）。タグの内容を分析し、フィールドが主キーであるかどうかを決定します（複数の主キーが指定されている場合は、それを呪い、実行を停止します）、フィールドがゼロ以外である必要があります。このフィールドのデータベースを操作して定義する必要がありますか？タグでオーバーライドされた場合の列名。また、構造体フィールドのタイプに基づいてテーブル列のタイプを決定する必要があります。フィールドタイプには有限のセットがあります。基本的なタイプのみを生成し、すべての行をTEXTフィールドタイプに減らしますが、一般的には、列タイプの定義をタグに追加して、より細かく設定できるようにすることができます。一方、事前にデータベースに目的のテーブルを作成したり、作成したテーブルを自動的に修正したりする必要はありません。

構造を解析した後、テーブル作成関数のコードを作成するメソッドと、Create、Query、Update、Delete関数を作成するメソッドを開始します。各関数のSQL式と実行するバインディングを準備します。エラー処理に煩わされることはありませんでした。データベースドライバーからエラーを提供するだけです。コード生成には、text /テンプレートライブラリのテンプレートを使用すると便利です。彼らの助けを借りて、はるかにサポートされた予測可能なコードを入手できます（コードはすぐに表示され、ジェネレーターコードによって汚されません）。

func (tableD *TableInfo) generateCreateTable(out *os.File) error {
	fmt.Fprint(out, "func (in *"+tableD.StructName+") createTable(db *sql.DB) (error) {\n")
	var resSQLq = fmt.Sprintf("\tsqlQ := `CREATE TABLE %s (\n", tableD.TableName)
	for _, col := range tableD.Columns {
		colSQL := col.ColName + " " + col.ColType
		if col.NotNull {
			colSQL += " NOT NULL"
		}
		if col == tableD.PrimaryKey {
			colSQL += " AUTO_INCREMENT"
		}
		colSQL += ",\n"
		resSQLq += colSQL
	}
	if tableD.PrimaryKey != nil {
		resSQLq += fmt.Sprintf("PRIMARY KEY (%s)\n", tableD.PrimaryKey.ColName)
	}
	resSQLq += ")`\n"
	fmt.Fprint(out, resSQLq)
	fmt.Fprint(out, "\t_, err := db.Exec(sqlQ)\n\t\tif err != nil {\n\t\t\treturn err\n\t\t}\n")
	fmt.Fprint(out, "\t return nil\n}\n\n")
	return nil
}

package main

import "database/sql"

func (in *User) createTable(db *sql.DB) error {
	sqlQ := `CREATE TABLE users (
	id integer NOT NULL AUTO_INCREMENT,
	login text NOT NULL,
	Email text,
	Level integer,
	IsActive boolean,
	PRIMARY KEY (id)
	)`
	_, err := db.Exec(sqlQ)
	if err != nil {
		return err
	}
	return nil
}

func (in *User) Create(db *sql.DB) error {
	sqlQ := "INSERT INTO users (`login`,`Email`,`Level`,`IsActive`) VALUES (?,?,?,?);"

	result, err := db.Exec(sqlQ, in.Login, in.Email, in.Level, in.IsActive)
	if err != nil {
		return err
	}
	lastId, err := result.LastInsertId()
	if err != nil {
		return nil
	}
	in.ID = int(lastId)
	return nil
}

func (in *User) Query(db *sql.DB) ([]*User, error) {
	sqlQ := "SELECT * FROM users;"
	rows, err := db.Query(sqlQ)
	results := make([]*User, 0)
	for rows.Next() {
		tempR := &User{}
		err = rows.Scan(&tempR.ID, &tempR.Login, &tempR.Email, &tempR.Level, &tempR.IsActive)
		if err != nil {
			return nil, err
		}
		results = append(results, tempR)
	}
	return results, nil
}

func (in *User) Update(db *sql.DB) error {
	sqlQ := "UPDATE users SET `login`=?,`Email`=?,`Level`=?,`IsActive`=? WHERE id = ?;"

	_, err := db.Exec(sqlQ, in.Login, in.Email, in.Level, in.IsActive, in.ID)
	if err != nil {
		return err
	}
	return nil
}

func (in *User) Delete(db *sql.DB) error {
	sqlQ := "DELETE FROM users WHERE id = ?"
	_, err := db.Exec(sqlQ, in.ID)
	if err != nil {
		return err
	}
	return nil
}


生成されたコードの動作をテストするには、任意のデータでオブジェクトを作成し、そのためのテーブルを作成します（データベースにテーブルが存在する場合、エラーが返されます）。このオブジェクトをテーブルに配置したら、テーブルからすべてのフィールドを読み取り、レベル値を更新して、オブジェクトを削除します。
