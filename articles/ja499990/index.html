ジオコーディング 10分間で25万のアドレスを座標にバインドする方法は？



こんにちは、ハブル！

この記事では、多数のアドレスに関する小さな問題の解決における私の経験を共有したいと思います。ジオコーディングAPIを使用したことがある、またはオンラインツールを使用したことがある場合は、結果を数時間、またはそれ以上待つのは私の苦痛を共有していると思います。

これは複雑な最適化アルゴリズムについてではなく、アドレスのリストを入力として受け取り、結果を含むファイルを返すパケットジオコーディングサービスの使用についてです。これにより、処理時間が数時間から数分に短縮されます。

バックグラウンド


タスクが到着しました-「24千の住所の座標にバインドします。」問題の解決策は2つだけ発生しました。

  1. 大学で使用されていたジオコーディング用のWebアプリケーション。
  2. ジオコーダーのREST APIに基づくスクリプトを記述します。

最初のケースでは、数千のアドレスを処理した後にWebアプリケーションがクラッシュすることがわかりました。同僚間でのデータセットの配布は、すぐに放棄されたアイデアです。

したがって、ジオコーダーのREST APIを使用して独自のスクリプトを記述し、結果を保存する必要があります（これは完全に正当な方法ではなく、サービスの利用規約を読む必要があります）。新しい問題が発生します。これは、アプリケーションでアドレス検索を使用してすぐに結果を取得する場合の1つの問題ですが、タスクが保存して1万を超えるアドレスを処理する場合、スクリプトの作業は大幅に遅延します。 1〜2時間待つことができますが、100万の住所を「膨大な時間」にジオコーディングする必要があるため、別のソリューションを探す必要があります。

ジオロケーションサービスの大規模プロバイダーは、通常のジオコーディングサービスに加えて、1つのリクエストで多数の住所を処理するために、パケットジオコーダー（Batch Geocoder）を提供しています。

バッチジオコーディング


サービスの名前はそれ自体が意味します-サーバーにアップロードするパッケージ（たとえば、表形式のアドレスのリストを含むcsvファイル）があり、それがすべての作業を行います。

プロセスは次のようになります。

  1. サービスがエラーなしでデータセットを受け入れるようにデータセットを準備します。
  2. 作業結果のパラメータの設定（列の選択、セパレータなど）。
  3. ファイルをクラウドにアップロードします。
  4. 処理が完了するのを待っています。
  5. 完成したファイルをダウンロードします。

クラウドコンピューティングの力のおかげで、1時間で自己記述されたスクリプトで行われることは1分で完了します。

次のステップは、パケットジオコーダーの使用に関する最も忠実な条件を持つ会社を選択することです。まず、誰もがそのようなサービスを持っているわけではありませんが、深刻な制限付きでサービスをテストできる人もいます。また、ボリュームが非常に大きい場合は、無料パッケージの制限を超えた場合に備えて、追加のトランザクションのコストに注意する必要があります。

バッチジオコーディングサービスプロバイダーの選択


地理位置情報サービスのグローバル市場では、主導的な地位を占めています。

  • グーグルマップ
  • ここにテクノロジー;
  • MapBox
  • トムトム
  • ESRI。

もちろん、ロシアでかなり強い地位にあるYandex Technologiesを忘れてはいけません。

プロバイダーを選択するための基礎として、以下のパラメーターを採用しました。

  • 1か月あたりのジオコーディングサービスへのリクエスト数は無料です。
  • 1日あたりのトランザクション数の制限。
  • バッチジオコーディングサービスの可用性。
  • 無料プランでパケットジオコーダーを使用する機能。

各企業には独自の収益化モデルがあります。プロジェクトによっては、1つまたは別のモデルが手に入る可能性があり、その逆の場合も大きな制限になります。

グーグルマップ


Googleジオサービスを使い始めるには、まず、アカウントにクレジットカード情報を追加する必要があります。月額200バーチャルドルの制限は、リンクされたカードからの追加トランザクションの支払いになります。この制限内で、さまざまなサービスを使用できますが、各トランザクションは異なる方法で考慮されます。たとえば、1,000のジオコーディングリクエストは5ドルかかりますが、ルート構築サービスは2倍の費用がかかります。詳細はサイトにあります。私たちはジオコーディングサービスにのみ関心があります。

月額200ドルの場合、無料のトランザクション数を計算するのは簡単です-40,000（ジオコーディングサービス）。サービス間にパケットジオコーダーはありません。つまり、独自のスクリプトを記述する必要があり、結果は1秒あたり約1アドレスになります。これは、24,000のアドレスで6時間です。プロセスを高速化するために、Google Cloud APIプラットフォームでスクリプトを実行してみることができますが、私は代替の解決策を探すことにしました。1日あたりのトランザクション数に制限はないため、一度に4万すべてを費やすことができます。

ここにテクノロジー


過去には、Nokia Mapsと、さらに深い過去には、Navteqが毎月250,000件のトランザクションを無料で提供しています。 Googleマップと同様に、この数はすべてのサービスに適用され、それぞれが異なる方法で考慮されます。無料パッケージをご利用の場合、キャッシュカードの添付は不要です。制限を超えた場合、追加の1,000トランザクションごとに$ 1を支払う必要があります。

無料のプランに含まれている個別のサービスとしてパケットジオコーダーを使用することが重要です。その中のトランザクションは、通常のモデルと同じモデルに従って考慮されます。つまり、ファイルの各アドレスは、パケットジオコーダーが1つのトランザクションで認識します。
記事のタイトルから、すべてのトランザクションをジオコーダーで費やして、月に250,000のジオコーディング操作を実行できるため、ここでバッチジオコーダーを使用したことは明らかです。しかし、これが唯一の選択肢ではないため、他の企業が持っているものを検討します。

Mapbox


MapBoxジオコーダーを使用すると、1か月あたり10万回のトランザクションを利用できます。同社は、追加の取引に対する支払いを伴う同じ収益化モデルを遵守しています。「卸売業者」には興味深いオプションしかありません。トランザクションが多いほど、コストは低くなります（もちろん、値下げの制限があります）。たとえば、10万から50万の場合、追加の1000リクエストには0.75ドル、50万から100万-0.60ドルなどの費用がかかります。詳細については、ウェブサイトをご覧ください。残念ながら、バッチジオコーダーは有料アカウントでのみ使用できます。

トムトム


このプラットフォームにより、1日あたり2500トランザクション、1か月あたり約75,000トランザクションを実行できます。テストおよび開発中、1日の制限は競合他社に比べてあまり魅力的ではありませんが、追加のトランザクションに対する支払いが最も柔軟です。追加の1000リクエストには8つの支払いオプションがあり、価格は$ 0.5から$ 0.42に引き下げられます。

サービスの中には、リクエストごとに最大1万のアドレスを処理できるバッチジオコーダーがあります（ただし、1日の制限を考慮する必要があります）。

Yandex Technologies


Yandexの1日のトランザクション制限があり、より忠実な25,000リクエストのモデル。この数に月の日数を掛けると、75万という印象的な数字になります。サイトでは、120ルーブルの範囲のルーブルで、追加の1,000トランザクションの価格を提示しています。11ルーブルまで

サービスとしてのパケットジオコーダーは表示されないため、何らかの最適化を行うと失敗します。

ESRI


1か月あたり100万トランザクションの非常に魅力的な無料プラン。また、各アカウントに50クレジット（約5ドル相当）を請求します。これは地理位置情報サービスの使用に関する最も忠実な計画であることは注目に値します。バッチジオコーディングサービスもありますが、ArcGIS Onlineプラットフォームに企業アカウントがある場合にのみ使用できます。

Pythonサービスガイド


まず、開発者向けのポータルでアカウントを作成し、プロジェクトセクションでREST APIキーを生成する必要があります。

これで、プラットフォームを操作できます。ここでは、パケットジオコーダーの機能の一部のみを説明します。データの読み込み、ステータスの確認、結果の保存です。

それでは、必要なライブラリをインポートすることから始めましょう：

import requests
import json

さらに、エラーが発生していない場合は、クラスを作成します。

class Batch:

    SERVICE_URL = "https://batch.geocoder.ls.hereapi.com/6.2/jobs"
    jobId = None

    def __init__(self, apikey="your_api_key"):
        self.apikey = apikey

つまり、初期化中に、クラスはREST APIの独自のキーを渡す必要があります。
SERVICE_URL変数は、バッチジオコーディングサービスを操作するためのベースURLです。
また、jobIdには、ジオコーダーの現在の作業の識別子が格納されます。

重要な点は、要求に応じた正しいデータ構造です。ファイルには、recIdとsearchTextの2つの必須列が含まれている必要があります。それ以外の場合、サービスはダウンロードエラーに関する情報を含む応答を返します。

以下はデータセットの例です。

ファイルをクラウドにアップロードするための関数：

def start(self, filename, indelim=";", outdelim=";"):
        
        file = open(filename, 'rb')

        params = {
            "action": "run",
            "apiKey": self.apikey,
            "politicalview":"RUS",
            "gen": 9,
            "maxresults": "1",
            "header": "true",
            "indelim": indelim,
            "outdelim": outdelim,
            "outcols": "displayLatitude,displayLongitude,locationLabel,houseNumber,street,district,city,postalCode,county,state,country",
            "outputcombined": "true",
        }

        response = requests.post(self.SERVICE_URL, params=params, data=file)
        self.__stats (response)
        file.close()

すべてが非常に単純で、読み取り用のアドレスのリストを含むファイルを開き、GETリクエストパラメータの辞書を形成します。いくつかのパラメータは説明する価値があります：

  • 「アクション」：「実行」-アドレス処理の開始。
  • “politicalView”: “RUS” – . ( );
  • “gen”: 9 – ( );
  • “maxresults”: 1 – ;
  • “header”: true – ;
  • “indelim”: “;” – , ;
  • “outdelim”: “;” – ;
  • “outcols”: “” – , ;
  • “outcombined”: true – .

次に、要求ライブラリを使用して要求を送信し、統計を表示します。もちろん、関数の最後でファイルを閉じる必要があります。__stats関数は、実行中の作業のIDを含むサーバーの応答を解析し、操作に関する一般的な情報も表示します。

次のステップは、作業のステータスを確認することです。リクエストは同様の方法で形成され、操作IDを転送する必要があるだけです。actionパラメータには値「status」が含まれている必要があります。__stats関数は、ジオコーダーのシャットダウン時間を推定するために、コンソールに完全な統計を表示します。

    def status (self, jobId = None):

        if jobId is not None:
            self.jobId = jobId
        
        statusUrl = self.SERVICE_URL + "/" + self.jobId
        
        params = {
            "action": "status",
            "apiKey": self.apikey,
        }
        
        response = requests.get(statusUrl, params=params)
        self.__stats (response)

最も重要な機能の1つは、結果を保存することです。便宜上、サーバーからのファイルをすぐに解凍することをお勧めします。ファイルを保存する要求は、ステータスをチェックすることと同じです。最後に追加/結果を追加するだけです。

    def result (self, jobId = None):

        if jobId is not None:
            self.jobId = jobId
        
        print("Requesting result data ...")
        
        resultUrl = self.SERVICE_URL + "/" + self.jobId + "/result"
        
        params = {
            "apiKey": self.apikey
        }
        
        response = requests.get(resultUrl, params=params, stream=True)
        
        if (response.ok):    
            zipResult = zipfile.ZipFile(io.BytesIO(response.content))
            zipResult.extractall()
            print("File saved successfully")
        
        else:
            print("Error")
            print(response.text)

サービスの応答を解析するための最後の関数。また、彼女のタスクは、現在のジオコーディングタスクの識別子を保存することです。

    def __stats (self, response):
        if (response.ok):
            parsedXMLResponse = BeautifulSoup(response.text, "lxml")

            self.jobId = parsedXMLResponse.find('requestid').get_text()
            
            for stat in parsedXMLResponse.find('response').findChildren():
                if(len(stat.findChildren()) == 0):
                    print("{name}: {data}".format(name=stat.name, data=stat.get_text()))

        else:
            print(response.text)

完全なバッチクラスの説明


スクリプトを完全に追加しても問題はないと思います。

import requests
import json
import time
import zipfile
import io
from bs4 import BeautifulSoup

class Batch:

    SERVICE_URL = "https://batch.geocoder.ls.hereapi.com/6.2/jobs"
    jobId = None

    def __init__(self, apikey="   REST API "):
        self.apikey = apikey
        
            
    def start(self, filename, indelim=";", outdelim=";"):
        
        file = open(filename, 'rb')

        params = {
            "action": "run",
            "apiKey": self.apikey,
            "politicalview":"RUS",
            "gen": 9,
            "maxresults": "1",
            "header": "true",
            "indelim": indelim,
            "outdelim": outdelim,
            "outcols": "displayLatitude,displayLongitude,locationLabel,houseNumber,street,district,city,postalCode,county,state,country",
            "outputcombined": "true",
        }

        response = requests.post(self.SERVICE_URL, params=params, data=file)
        self.__stats (response)
        file.close()
    

    def status (self, jobId = None):

        if jobId is not None:
            self.jobId = jobId
        
        statusUrl = self.SERVICE_URL + "/" + self.jobId
        
        params = {
            "action": "status",
            "apiKey": self.apikey,
        }
        
        response = requests.get(statusUrl, params=params)
        self.__stats (response)
        

    def result (self, jobId = None):

        if jobId is not None:
            self.jobId = jobId
        
        print("Requesting result data ...")
        
        resultUrl = self.SERVICE_URL + "/" + self.jobId + "/result"
        
        params = {
            "apiKey": self.apikey
        }
        
        response = requests.get(resultUrl, params=params, stream=True)
        
        if (response.ok):    
            zipResult = zipfile.ZipFile(io.BytesIO(response.content))
            zipResult.extractall()
            print("File saved successfully")
        
        else:
            print("Error")
            print(response.text)
    

    
    def __stats (self, response):
        if (response.ok):
            parsedXMLResponse = BeautifulSoup(response.text, "lxml")

            self.jobId = parsedXMLResponse.find('requestid').get_text()
            
            for stat in parsedXMLResponse.find('response').findChildren():
                if(len(stat.findChildren()) == 0):
                    print("{name}: {data}".format(name=stat.name, data=stat.get_text()))

        else:
            print(response.text)
