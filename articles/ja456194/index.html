Goデータ構造のチートシート


一部の企業はオンラインライティングコードにインタビューします。スピードのためにオリンピック問題を解決する必要があります。このような状況では、データ構造の実装の詳細を確認する時間はありません。アイデアをすぐに理解する必要があります。ただし、アルゴリズムとデータ構造のコースでは、疑似コードまたはC ++の例を示します。多くの場合、問題のリファレンスソリューションはC ++で記述されています。インタビューの準備として、貴重な時間の検索を無駄にしないように、STLコンテナのアナログであるライブラリのベビーベッドを作りました。

明白から始めましょう。

動的連続配列


アナログstd::vector。
いくつかのプロセッササイクルの一定時間のインデックスによる要素へのアクセスをサポートします。容量は増減できます。これは組み込みスライスです：

vector := []int{}

便利なことに、基本的な概念が言語に組み込まれています。

スタック


アナログstd::stack。

新しい要素の追加と既存の要素の削除が一方の端から行われる順序付きセット。最後に配置された要素（後入れ先出し-LIFO）は、スタックから最初に削除されます。これも壁に囲まれたスライスです。スニペットはプロジェクト間でコピーされます。

// Push
stack = append(stack, value)


// Pop
// ,  len(stack) > 0
stack, value = a[:len(stack)-1], a[len(stack)-1]

スライス操作では、新しいメモリは割り当てられません。

順番


アナログstd::queueとstd::deque。

キューは、一定の時間内に開始および終了するための取得および追加操作を実装します。最初に配置されたエレメント（先入れ先出し-FIFO）は、最初にキューから削除されます。バッファリングされたチャネルはリングバッファ上のキューであり、リーダーとライターが異なるゴルーチンである場合に使用できます。ただし、標準ライブラリには個別のキュー実装はありません。awesome-goリストは、ライブラリhttps://github.com/gammazero/dequeにアドバイスしています。

import "github.com/gammazero/deque"

進行中の操作：

func (q *Deque) PushBack(elem interface{})
func (q *Deque) PushFront(elem interface{})
func (q *Deque) PopBack() interface{}
func (q *Deque) PopFront() interface{}
func (q *Deque) Back() interface{}
func (q *Deque) Front() interface{}
func (q *Deque) At(i int) interface{}

二重リンクリスト


アナログstd::list。
独自のデータに加えて、次および前のリスト項目へのリンクを含む要素で構成されます。それは標準ライブラリにあります：

import "container/list"

予想どおり、挿入操作（要素の最初、最後、要素の前後）、および削除をサポートしています。

func (l *List) PushBack(v interface{}) *Element
func (l *List) PushFront(v interface{}) *Element
func (l *List) InsertAfter(v interface{}, mark *Element) *Element
func (l *List) InsertBefore(v interface{}, mark *Element) *Element
func (l *List) Remove(e *Element) interface{}

Goはイテレータに特定の構文を提供しません。したがって、次の/前の要素は、任意のノードへのポインターから取得できます。これらのメソッドは、リストに項目を追加/削除した後も、驚くことなく失敗しません。

func (e *Element) Next() *Element
func (e *Element) Prev() *Element

優先キュー


アナログstd::priority_queue
アイテムを任意の順序で追加し、いつでも残りのアイテムの最高の優先順位を取得できます。これは、たとえば、最小全域木を構築するアルゴリズムで使用されます。次のステップで、アルゴリズムは、一端ですでにカバーされている頂点で始まるすべての最短エッジを選択します。

標準ライブラリには、並べ替え可能なコンテナ（実装sort.Interface）を優先キューに変換するアダプタがあります。

import "container/heap"

これは古典的なバイナリヒープです。O（log n）の挿入と削除を実装します。

func Pop(h Interface) interface{}
func Push(h Interface, x interface{})
func Remove(h Interface, i int) interface{}

ハッシュ表


辞書であり、連想配列です。

アナログstd::unordered_map。

Key-Valueを追加し、キーごとに値を削除し、平均してO（1）の要素の存在を確認できます。明らかに地図は言語に組み込まれています：

unorderedMap := make(map[string]int)

make（マップ）の結果はポインターであり、その動作方法は標準のコンテナーとは少し異なります。

//  :
_, ok := unorderedMap["route"]
//  :
delete(unorderedMap, "route")
//  :
n := len(unorderedMap)

std :: unordered_mapとは異なり、「ランタイム/マップ」はプログラマーの面倒を見る-反復中に値を削除しても安全です。

沢山の


アナログstd::unordered_set。
ハッシュテーブルとほとんど同じですが、値を保存しません。
簡単なエントリチェックだけが必要な場合は、組み込みマップを再び使用できます。空の値を指定するだけで、キーのみが重要であることを示すことができます。

var m = make(map[string]struct{})
m["!"] = struct{}{}
_, ok := m["!"] // true

ただし、この実装は複雑な演算子をサポートしていません。ボックスとの違いをマージ、交差するには、サードパーティのライブラリが必要です。星の数で判断すると、最も使用されています：https : //github.com/deckarep/golang-set

import "github.com/deckarep/golang-set"

並べ替え


アナログstd::sortとstd::stable_sort。
消費者の観点から見ると、基本的に異なるタイプは2つしかあり
ません：安定（等しい要素の順序を変更しないでください[[4、0]、[1、2]、[1、1]、[5、6]]-> [[1、 2]、[1、1]、[4、0]、[5、6]]）
および安定していないため、他のフィールドのシーケンスは保証されません。
どちらも標準ライブラリにあります：

func Sort(data Interface)

これは、Hoarクイックソート実装であり、不安定です。ただし、長さが12未満のセクションでは、ヒープのソートが最適化として使用されます。

func Stable(data Interface)

内部にはマージソートがありますが、効率を上げるために、再帰アルゴリズムが20要素未満のブロックに到達すると、挿入ソートが使用されます。

これらは、O（n log n）で機能する古典的なアルゴリズムです。
