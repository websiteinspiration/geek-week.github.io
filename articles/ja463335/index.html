CおよびC ++の組み込みシステムの開発中にPVS-Studio静的アナライザーを使用する

この記事では、組み込みプロジェクトチェックの実行方法とアナライザーレポートの操作方法について学習します。PVS-Studioアナライザーは、組み込みシステム用の多くのコンパイラーをサポートしています。現在のバージョンでは、Window、Linux、およびmacOS用の次​​のコンパイラーの1つを使用してビルドするときにプロジェクトをチェックできます。

  • IAR組み込みワークベンチ
  • Arm用のKeil組み込み開発ツール
  • TI ARMコード生成ツール
  • GNU組み込みツールチェーン

取り付け


インストール手順は、開発中のオペレーティングシステムによって異なります。Linuxでは、リポジトリーまたはインストール・パッケージからアナライザーをインストールできます。

Debianベースのシステムの例：

wget -q -O - https://files.viva64.com/etc/pubkey.txt | sudo apt-key add -

sudo wget -O /etc/apt/sources.list.d/viva64.list \
https://files.viva64.com/etc/viva64.list

sudo apt update
sudo apt install pvs-studio

または

sudo gdebi pvs-studio-VERSION.deb

macOSの場合、インストールと更新はHomebrewから実行できます。

brew install viva64/pvs-studio/pvs-studio
brew upgrade pvs-studio

2番目のオプションは、dmgからパッケージをインストールするか、アーカイブから手動でアンパックすることです。

Windowsでは、インストーラーを使用してインストールを実行します。


サポートされているすべてのシステム用のインストールパッケージをダウンロードできます。必要に応じて、「ダウンロードしてPVS-Studioを試す」ページで試用キーをリクエストできます。

インストール後、ライセンスキーを入力します。さまざまなプラットフォームでこれを行う方法に関する情報は、ドキュメントページの「PVS-Studioライセンスの入力方法と次の手順」に記載されています。

プロジェクトの検証


組み込みシステムのプロジェクト検証は、Windows、Linux、およびmacOSの開発の場合と同じ方法で実行できます。

Linuxの起動オプションについては、「LinuxでC ++プログラムを開発する際のPVS-Studio静的アナライザーの紹介」の記事で説明しています。組み込みでのビルドにはクロスコンパイルが使用され、コンパイラ名は非標準である可能性があるため、プロジェクトの分析時に指定する必要があることに注意してください。これを行うには、–compilerまたは–cコマンドラインオプションを使用します。

pvs-studio-analyzer analyze -c MyCompiler

このパラメーターは、アナライザーが使用するコンパイラーのタイプを判別できない場合、つまり「コンパイル単位が見つかりません」というエラーが発生した場合に使用する必要があります。

クロスコンパイルすると、ターゲットプラットフォームは、一緒にコンパイラで、組立が行われる下で1異なっているので、あなたが使用してターゲットプラットフォームを指定する必要があり--platformパラメータと同様に、プリプロセッサの種類（--preprocessorを）。

サポートされるプラットフォーム：win32、x64、linux32、linux64、macos、arm。

サポートされているプリプロセッサ：gcc、clang、keil。

デフォルトでは、Linuxはlinux64プラットフォームとgccプリプロセッサーを選択します。

Windowsを使用している場合は、コンパイル監視モードでプロジェクトを確認できます。これを行うには、アナライザーと共にインストールされる「CおよびC ++コンパイラー監視UI」ユーティリティを実行します。監視を開始するには、[ ツール ]メニューに移動し、[ ファイルの分析... ] をクリックします。ダイアログが開きます。


「監視の開始」ボタンをクリックして、ビルドを実行します。完了したら、画面の右下隅にあるダイアログボックスの[ 監視の停止 ]ボタンをクリックします。


テスト結果は、「CおよびC ++コンパイラモニタリングUI」ユーティリティのメインウィンドウに表示できます。

コマンドラインから分析を実行することもできます。これを行うには、CLMonitorユーティリティを使用します。次のコマンドで監視を開始できます。

CLMonitor.exe monitor

アセンブリが完了したら、ユーティリティを分析モードで実行する必要があります。

CLMonitor.exe analyze -l "<path>\out.plog"

アナライザーはプロジェクトをチェックし、結果を-lパラメーターで指定されたファイルに保存します。

ドキュメントのセクション「PVS-Studioのコンパイル監視システム」も参照してください。

レポートを操作する


Linuxでレポートを表示するには、アナライザの結果として得られたファイルを、plog-converterユーティリティを使用して、サポートされている形式のいずれかに変換する必要があります。次のコマンドを使用して、ソースコードを表示する機能を備えたHTML形式のレポートを取得できます。

plog-converter -a GA:1,2 -t fullhtml /path/project.log -o /path/report_dir

レポートの変換については、「Linux環境でC ++プログラムを開発する際のPVS-Studio静的アナライザーに関する知識」の記事で詳しく説明しています。

Windowsでは、同様の方法でレポートファイルを変換できるPlogConverterユーティリティもあります。

PlogConverter.exe <path>\out.plog --renderTypes= FullHtml --analyzer=GA:1,2

または

PlogConverter.exe D:\Projct\out.plog -t FullHtml -a GA:1,2

また、plog形式のレポートは、メニューの[ ファイル ] -> [PVS-Studioログを開く... ]からユーティリティ「CおよびC ++コンパイラモニタリングUI」を使用して表示できます。

例：


必要に応じて、このユーティリティから、[ファイル ]メニューを使用して、サポートされている形式のいずれかにレポートをエクスポートできます。

MISRA規格


PVS-Studio は、CWEおよびSEI CERTに従ってアラートを分類します。これは、通常のアプリケーションの静的セキュリティテスト（SAST）に適しています。ただし、組み込みシステムには、特別に開発されたMISRA規格でカバーされている、わずかに異なるセキュリティ要件があります。アナライザーの現在のバージョンは、MISRA CおよびMISRA C ++のルールの一部をサポートしています。診断のリストはここで確認でき、常に拡張されています。

組み込みを目的としていないプロジェクトをチェックするときにMISRAルールを使用することは、多くの場合正当化されません。これは規格の詳細によるものです。ほとんどの場合、コードがもともとMISRA準拠を想定して記述されていなかった場合、チェック中に多くの誤った無用の警告が出されます。したがって、MISRAルールはデフォルトで無効になっています。

LinuxでMISRA分析を有効にするには、-aスイッチを使用して分析を実行し、必要なモードを設定する数値パラメーターを渡す必要があります。このパラメーターはビットフィールドの組み合わせです。

-a [MODE], --analysis-mode [MODE]
        MODE defines the type of warnings:
        1 - 64-bit errors;
        2 - reserved;
        4 - General Analysis;
        8 - Micro-optimizations;
        16 - Customers Specific Requests;
        32 - MISRA.
        Modes can be combined by adding the values
        Default: 4

起動例（GAおよびMISRA診断が含まれています）：

pvs-studio-analyzer analyze -a 36 -o /path/report.log

plog-converter を使用する場合は、MISRA診断をレポートに含める必要があることも示す必要があります。

plog-converter -a MISRA:1,2,3 -m misra ....

Windowsでは、「CおよびC ++コンパイラモニタリングUI」ユーティリティ設定ダイアログを使用できます。


MISRAサポートの詳細については、「PVS-Studio：MISRA CおよびMISRA C ++エンコーディング標準のサポート」の記事を参照してください。
