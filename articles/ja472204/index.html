BH1750センサー




BH1750 16ビット光センサー（光メーター）、I2Cインターフェイス付き。BH1750のフォトダイオードは光の強度を決定し、オペアンプを使用して出力電圧に変換されます。内蔵ADCは16ビットのデジタルデータを提供します。BH1750の内部ロジックは、重要なデジタルデータをルクス（ルクス）で直接出力するため、複雑な計算の必要がありません。

資料によれば、BH1750センサーは可視光に敏感であり、実際には赤外線の影響を受けません。人間の目とほぼ同じスペクトル範囲に反応します。

このセンサーを使用するには、「構成」タブでBH1750を選択します。

BME280センサー




このセンサーは、温度、大気圧、および湿度などの環境パラメーターを測定します。

同様のセンサー-BMP280もあり、湿度を測定する機能がありません。しかし、それははるかに安いです。

センサーは上記のすべてのパラメーターを測定するという事実に加えて、ノイズをフィルターする方法も知っているため、読み取り値のキャリブレーションが組み込まれています。また、他のセンサーとは異なり、このセンサーはこれらすべての測定を迅速に行うだけでなく、より広い範囲で行います。たとえば、多くのセンサーは20％未満の湿度を測定できません。

。BH1750、BME280センサー、ディスプレイ1602で機能するプログラムの例：

#include "mx_init_hw.c"

int main()
{
    char s[64];
    float t=0, p=0, h=0;

    Init_HW();

    // ---------------------
    // The main program loop
    // ---------------------
    while( TRUE )
    {
        delay_ms( 300 );

        t = BME_T / 10.0;
        p = BME_P / 100.0;
        h = BME_H / 10.0;

        sprintf( s, "t=%-7.1fC", t );
        SSD1306_write_string( 1, s );

        sprintf( s, "p=%-7.1fPa", p );
        SSD1306_write_string( 2, s );

        sprintf( s, "h=%-7.1f%% ", h );
        SSD1306_write_string( 3, s );

        sprintf( s, "T=%-7.1fH=%.1f", t, h );
        LCD_write_string( 1, s );

        sprintf( s, "P=%-7.1fL=%u", p, BH_L );
        LCD_write_string( 2, s );
    }
}

CCS811センサー




私たちの周りの空気はガスの混合物であり、主に窒素（約78％）と酸素（約21％）で構成されています。残りの割合は、さまざまな不純物（不活性ガス、二酸化炭素、一酸化炭素、有機揮発性物質（VOC）など）によって占められます。不純物の割合が少ないという事実にもかかわらず、それらの濃度の変化は人間にとって非常に不快で危険なことさえあります。オフィスや住宅の空気質は、ほとんどの場合、CO2とVOCの含有量に関連しています。

有機揮発性物質には、5000以上の化合物が含まれます。それらのほとんどの形成は、どういうわけか人間の生活のプロセスと関連しています。

したがって、密閉された部屋に人間がいるだけでも、空気が「汚染」され、換気が必要になります。空気中のVOCの濃度を推定することは困難です。以前は、CO2センサーを使用してVOC濃度を評価していました。この場合、通常の条件下ではVOCとCO2の濃度が制限されるという事実が考慮されました。 CO2の割合がわかれば、間接的にVOCの濃度を決定できます。空気中の二酸化炭素の含有量が特定の制限に達した場合は、換気をオンにする必要があります。

実際には、VOCとCO2の関係は必ずしも簡単ではありません。たとえば、室内喫煙はVOC含有量を急激に増加させますが、CO2センサーはこれを記録しないため、二酸化炭素濃度はあまり変化しません。家庭用化学薬品、洗浄剤、塗料、ワニスを室内で使用する場合も、二酸化炭素センサーは無力になります。 AMSは、VOC濃度測定の問題に対するソリューションを提供します-CCS811センサー。

CCS811B空気質センサーの機能：



センサーには自動校正アルゴリズムがあります。測定値は数日以内に自動的に設定されます。このため、新しいセンサーには大きな初期エラーがあり、多かれ少なかれ信頼できる結果が得られるまで、少なくとも3〜4日間はワークアウトする



必要があります。ユーザーは、CCS811センサーをオンにした後、毎回「ウォームアップ」する必要があることを覚えておく必要があります。このような「ウォームアップ」の時間は30分を超え、その間にエラーが許容できないほど大きくなることが判明しています

#include "mx_init_hw.c"
#include "dht_22.c"

int main()
{
    int16_t dht_t, dht_h;

    Init_HW();

    dht_init();

    // ---------------------
    // The main program loop
    // ---------------------
    while( TRUE )
    {
        delay_ms( 5000 );
        if( !get_dht_data( &dht_t, &dht_h ) )
            print( "t=%.1f h=%.1f\r\n", (float)dht_t / 10.0, (float)dht_h / 10.0 )
        else
            print( "DHT 22 - ERR\r\n" );
    }
}

CLI


ファームウェアには、コンソールで動作する小さなシェルが含まれています。メインの/ mx_cli.cファイルでコマンドを確認できます。

必要に応じて、独自のコマンドを追加できます。

CLIはNo Echoモードで動作を開始します。つまり、入力した文字は表示されません。入力した文字を表示するには、echo onと入力します。

「sh on」コマンドは、コンフィギュレーターで宣言された変数とセンサーに関連付けられた変数の値を1秒に2回出力します。最初の位置の番号は、コンフィギュレータテーブルの行番号に対応しています。このコマンドは、コンソールに接続されたシステムにデータを転送するために作成されます。

コンソールの忙しさを分析するフラグやミューテックスはありません。そのため、「印刷」と「エコーオン」の演算子を一緒に使用すると、出力情報の混合が発生する可能性があります。両方の方法を同時に使用しないでください。この問題を解決するには、2つの出力チャネルを使用できます。たとえば、コンソールはUART1にあり、ユーザー情報の出力はVCPにあります。またはその逆。

関数を使用：

send_uart1( char *data, int len )
send_usb( char *data, int len )

これらの関数の使用を容易にするために、gbl.hファイルのprintfマクロを変更できます。たとえば、次のとおりです。

#define printf(fmt,argv...){char s[128];sprintf(s,fmt,##argv);send_usb(s,strlen(s));} //   VCP
