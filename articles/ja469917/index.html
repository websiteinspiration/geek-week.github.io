PHP \ PythonのFasttext。最初のステップ

なぜ




長い間、さまざまな種類の機械学習アルゴリズムが人気を博していました。また、技術の進歩を推進している大企業のおかげで、多くのオープンソース製品が登場しています。それらの1つはFasttextで、これについては以下で説明します。

Fasttext-Facebookによる開発。プログラムの主な目的は、テキストの分類です。テキスト分類は次の場合に必要になることがあります。

  1. テキスト情報を「類似性」のグループにまとめる（1つのトピックに関するニュース）
  2. 類似のトピックを持つテキストを1つのグループにグループ化する（自動車に関するニュース）
  3. スパムの可能性がある情報を検索する
  4. クリックベイト検索
  5. ...

実際、多くのオプションがあり、すべてをリストアップしても意味がありません。アイデアは明確でなければなりません。

最初のトレーニング


ライブラリページには、段階的なインストールと最初のトレーニングの説明があります。それらについては詳しく説明しません。

パン
彼らはまた、ここで分類するためのさまざまな言語の既製のモデルを持っています。

ライブラリ設定


トレーニングの問題は、パラメーターの個性です。優れた結果を保証するパラメーターはありません。あなたはインターネット上でパラメーターの例を含むたくさんの（あるいはそうではない）記事を見つけることができます、そしてそれらは満足のいく結果を与えないのであなたには適していないかもしれません。

経験的にのみ、自分に合ったパラメータを選択できます。以下は大幅に結果に影響するもののリストです：

- DIM -寸法はベクトルの大きさを制御します（バター）：それらが大きいほど、より多くの情報を取得できますが、これにはより多くのデータが必要です。ただし、データが多すぎると、トレーニングプロセスが遅くなります。デフォルトは100測定です。 150から始めて、最適な値を選択してください。

- LR -スピードを学びます。パラメータが非常に小さい場合、モデルはテキストに対してより敏感になり、類似のテキストを区別できない可能性がありますが、パラメータが非常に大きい場合、逆に、テキストが類似していると「言う」ことができますが、実際にはそうではありません。 0.1（デフォルトは0.05）から始めます。

- エポック -元号の数。これは、データに応じたパスの数です。もっと-より良い（しかし、悲しいかな、いつもではありません）。これにより、トレーニング時間が長くなります。 150から開始します（デフォルトは5）。

-学習モデル。Facebookから説明を読んでください。それはかなり明確です。

-損失 -比較がどのように行われるか。ここのすべては非常に個人的であり、データに依存します。

小さな余談
テキストの分類やニューラルネットワークの内部メカニズムに関する十分な知識がなくても、非常に機能するモデルを取得できるのは非常にすばらしいことです。

テキストの準備


入力テキストも重要です。テキストが良いほど、モデルからの情報が良くなります。トレーニング用のテキストを準備するための基本的なルール：

  • すべてのタグを削除
  • 小文字にキャスト
  • 句読文字を削除する
  • ハッシュタグ、リンクを削除する
  • ストップワードを除外する
  • 小さな単語を除外する（1、2、3文字。ここでは誰もが彼のデータを決定する）

テキストをモデルに入力してトレーニングすることができると書いている人もいます。このオプションは私には合いませんでした。私は前処理なしでは質の悪いモデルが得られると信じがちです。

分類のためのテキストの準備


ここでも同じ規則が適用されますが、経験から、これらの規則は、見出し語化または詰め込みによって補足できることがわかっています。これらを使用すると、結果が大幅に改善（または悪化）する可能性があります。さらに、すでにクラスターを形成している場合は、クラスター化アルゴリズムもこれらのクラスターに適用する必要があることを忘れないでください。これはスポーツで非常に明白です。モデルはニュースがフットボールからのものであることを理解しています。しかし、スペインのチャンピオンシップとイタリアのチャンピオンシップをモデルで区別することは非常に困難です。

モデルをトレーニングするために、PHP（ほとんどのサイトはその上に書かれているので、それを実行しました）とPython（そのためのライブラリーがあります）の両方を選択できます。しかし、とても面白い瞬間があります。トレーニング時間がかかる場合は、コマンドラインからfasttextを実行してモデルをトレーニングする必要があります。したがって、トレーニング用のコードを何に書いてもかまいません（何に便利か、それを書いてください）。

クラスタリングメカニズムに関しては、少し複雑（または単純）です。自転車が好きな場合（すべてのプロセスを自分で制御し、柔軟な制御メカニズムが必要な場合）phpで書き込みます（サイトがphpの場合）。ライブラリを書きたくなく、言語を選択できる場合は、Pythonを使用することをお勧めします。速度に大きな違いはありませんでした（コードの速度ではなく、コードの作成速度ではありません）。それはあなた次第です。

結論の代わりに


過去数日間のニュースコンテンツのみに基づいて構築されたモデルがあります。その中の言葉のサイズは約40,000です。彼女と遊ぶことができます。ただし、次の点に注意してください。

  • これは普遍的なモデルではありません。彼女はニュースコンテンツのみをトレーニングします。
  • モデルにはデータベースからのすべてのニュースは含まれていませんが、社説のみが含まれています（これはタスクを解決するのに十分です）。これは、モデルが同様のニュースに対して低いパーセンテージを与える可能性があることを意味します。
