Windows Terminal预览版v0.10

Windows Terminal v0.10隆重推出！与往常一样，您可以从Microsoft Store或GitHub的发布页面下载它。在cat下，我们将仔细查看更新详细信息！



鼠标输入


终端现在支持在Linux的Windows子系统（WSL）应用程序以及使用虚拟终端（VT）输入的Windows应用程序中的鼠标输入。这意味着诸如tmux和Midnight Commander之类的应用程序将识别“终端”窗口中对项目的点击！如果应用程序处于鼠标模式，则可以按住shift做出选择，而不发送VT输入。



更新设定


面板重复


现在，您可以通过复制任何选定面板中的配置文件来打开一个新面板，只需将其聚焦并按下组合键即可。要做到这一点，你profiles.json的“按键绑定”部分中，您需要添加“splitMode”：“复制”到“splitPane”。您可以使用其他参数，例如“命令行”，“索引”，“ startingDirectory”或“ tabTitle”。如果您想进一步了解这些选项，建议您阅读本文。

{"keys": ["ctrl+shift+d"], "command": {"action": "splitPane", "split": "auto", "splitMode": "duplicate"}}




错误修正


  • 调整窗口大小时显着改善了文本显示；
  • 深色主题的边框是固定的（不再是白色）。
  • 如果任务栏处于隐藏状态，并且终端已展开，则当您将鼠标悬停在屏幕底部时，它会自动出现；
  • 现在，Azure Cloud Shell可以启动PowerShell并支持鼠标输入，也可以作为首选Shell安装。
  • 使用触摸板或触摸屏时，滚动速度已更改。

未来的计划


我们希望为您提供有关我们计划的最新信息，以便您知道未来几个月的期望。我们目前正在修正错误，以准备发布v1。Windows Terminal v1本身将于5月发布。之后，我们计划在六月发布下一个更新，以继续我们的每月更新周期。我们的版本将继续在Microsoft Store和GitHub上可用！
