理論


Webサーバーは、ソケットを使用して低レベルで機能し、未加工のHTTPリクエストを解析し、ヘッダーを解析し、接続を管理し、負荷を分散するなど、かなり複雑なものです。また、この複雑なモジュールを分離しておくことをお勧めします。これは、Webアプリケーションがすでにモデル、コントローラー、ビューの上位概念で動作しているためです。両方のコンポーネントはWebの作業に必要かつ重要ですが、それらの開発に必要な知識の領域はあまり重複していないため、お互いについてできるだけ少なく知っていることが望ましいです。

ご存知のように、Python Webアプリケーションで記述された互換性のあるサーバーは多数あります（Gunicorn、Bojern、CherryPyですが、多くはありません）。

同様に、これらのWebサーバーのいずれかと連携できる多くのWebフレームワークがあります。

これはどのように起こりますか？これは単純です-標準が必要であり、この標準はWSGIと呼ばれます

ESP8266は工場出荷時のATファームウェアでフラッシュされ、ATコマンドのみを受け入れます。しかし最も重要なことは-ATコマンドの助けを借りて、彼はsosketを北に上げることができます。これはクライアントからhttpリクエストを受信し（WiFi経由で受信）、シリアルポート経由でブロードキャストします。

ESPはシリアルポートで2つの重要なパラメーター-ソケットIDを送信し、クライアントに応答を返すことを示します。ちなみに、ESP8266は最大4つの同時ソケット接続をサポートします。そしてもちろんhttpリクエスト

Python 3 Webサーバーはヘッダーを特別な環境オブジェクトに折りたたんでhttpを解析し、アプリケーションが応答を返すstart_responseメソッドと、すべての必要なヘッダーとhttpリクエストの本文を含む環境ディクショナリ（必要なメソッド）を渡してアプリケーションオブジェクトを生成します。必要な環境ディクショナリキーはWSGI仕様の重要な部分です）

フレームワークが要求を処理して応答を生成した後、start_responseメソッドをプルします。このメソッドでは、http応答の生成に必要なパラメーターを取得し、未加工のhtml文字列を作成し、ESP8266のシリアルポートを介して

、 ESP8266 ATを工場ファームウェアでフラッシュする正しい方法は、優れた記事（有用なURLセクションの2番にあります）で見つかりました

。ATコマンドの場合、ESP8266との通信は次のようになります（通信時にESPloterシェルを使用します）また、WiFiがESP8266で発生していると想定しています。Wi-Fiが起動されていない場合-ESPloterを使用するのも簡単です

  1. コマンドAT + CIPMUX = 1-複数の接続を開く
  2. コマンドAT + CIPSERVER = 1.80（ポート80、ここで開く）

これらのコマンドを入力した後、TCPサーバーをESP8266に上げて、クライアントから要求を受信できるようにしました。

また、2つのコマンドでクライアントに応答を返します

-AT + CIPSENDコマンド=ソケット番号、バイト数
このコマンドが完了すると、端末は指定されたバイト数のメッセージを待ちます。
十分な文字数を入力すると、送信セッションが終了し、新しいコマンド

AT + CIPCLOSE = number を入力する必要がありますソケット。

その後、ブラウザはセッションを終了し、送信されたテキストを表示します。
