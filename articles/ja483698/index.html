LoRaWANが現代のモノのインターネットの構築にどのように役立つか



LoRaWANは、IoTソリューションの分野で急速に普及しているテクノロジーです。同時に、多くの顧客にとって、それはよく理解されておらず、エキゾチックなままです。そのため、周りには多くの神​​話や誤解があります。2018年、LoRaWAN周波数の使用に関する法律の改正がロシアで採択され、ライセンスなしでこのテクノロジーを使用する可能性が拡大しました。実際のビジネス上の問題を解決するために、今がこのテクノロジーの使用を開始するのに最適な時期だと私たちは考えています。

この記事では、LoRaWANの基本原則、独自のネットワークを構築してサードパーティのプロバイダーを使用するためのオプションについて検討し、LoRaWANをサポートする当社の製品についても説明します。

LoRaWANとは


LoRaWANは、長距離にわたって動作する低電力、低電力デバイスの物理およびネットワークデータ伝送層を定義する一連のプロトコルです。省略形のLoRaは長距離、つまりデータ伝送距離が長いことを意味し、WAN（ワイドエリアネットワーク）はプロトコルがネットワーク層も表すことを意味します。

よく知られているワイヤレス標準GSM / 3G / LTE / WiFiとは異なり、LoRaWANはもともと、多数の低電力加入者デバイスに同時にサービスを提供するように設計されていました。したがって、主な重点は、干渉に対する耐性、エネルギー効率、および範囲です。同時に、最大データ転送速度は毎秒数キロビットに制限されています。

セルラーネットワークと同様に、LoRaWANには加入者デバイスと基地局があります。加入者ユニットと基地局との通信範囲は10kmに達することができます。同時に、サブスクライバーデバイスは通常、自律的なバッテリー電源を備えており、ほとんどの場合、省電力モードにあり、サーバーとの短期データ交換のためにときどき起動します。したがって、たとえば、水道メーターは数日ごとに起動し、水の消費量の現在の値をサーバーに送信し、残りの時間はスリープモードになります。このアプローチにより、バッテリーを交換することなく、数年まで動作するデバイスを入手できます。 LoRaWANデバイスのタスクは、必要なデータを基地局から可能な限り迅速に転送/受信し、他のデバイスのために空気を解放することです。そのため、時間に関する厳密なルールがあります空中で占領。デバイスは、ベースステーションからの確認を受信した後にのみデータを送信します。これにより、サーバーからの無線負荷を制御し、データ交換セッションを時間内に均等に分散できます。

LoRa規格は、物理層、周波数範囲433 MHz、ヨーロッパの868 MHz、オーストラリア/アメリカ915 MHz、アジア923 MHzの信号変調について説明しています。ロシアでは、Lo685WAN帯域がLoRaWANに使用されています。

LoRaWANの仕組み


LoRaWANはライセンスのない範囲で動作するため、基地局を含む独自のネットワークの導入が大幅に簡素化されます。この場合、通信事業者に依存する必要はありません。独自のネットワークを展開することに加えて、既存の事業者のネットワークを使用できます。LoRaWANプロバイダーはすでに世界中に存在し、最近ロシアに登場し始めています。たとえば、オペレーターEr-Telecomはすでに多くの都市でLoRaWANネットワークへの接続を提供しています。

ロシアでは、通常LoRaWANは866〜869 MHzの範囲で動作し、1つの加入者ユニットが占有する最大チャネル幅は125 kHzです。これは、ユーザーRuslanによって記録されたスペクトログラムでLoRaWANプロトコルを使用したデータ交換の様子です。ElectricFromUfa SDRを使用するナディルシン。



2018年以降、ロシアは868 MHzの周波数の使用に関する制限を大幅に緩和する法律の改正を採用しています。ロシアのLoRaWANの頻度を管理する新しい法規制の詳細は、この記事に記載されています。

ベースステーション -LoRaWAN標準の用語では、ゲートウェイまたはハブと呼ばれます。設計上、このデバイスは従来のモバイルセルラーネットワークの基地局に似ています。エンドデバイスがそれに接続され、データ送信用のチャネル、電力、およびタイムスロットを選択するための指示に従います。基地局は、ネットワーク全体の状態にアクセスできる中央のソフトウェアサーバーに接続され、周波​​数計画などに従事しています。
通常、LoRaWAN基地局は定常電源に接続されており、安定したインターネットアクセスが可能です。アドバンテックは、100および500サブスクライバーユニットの容量と、イーサネットおよびLTE経由でインターネットに接続する機能を備えたWISE-6610シリーズのLoRaWANベースステーションのいくつかのモデルを提供しています。

サブスクライバーデバイス -通常は自律電力を備えた低電力クライアントデバイス。ほとんどの場合、スリープモード、省エネモードです。データを送受信するためにリモートアプリケーションサーバーと対話します。認証用の暗号化キーをベースステーションとアプリケーションサーバーに保存します。それはいくつかの基地局の範囲にあるかもしれません。基地局から受信した空気に取り組むための規則に厳密に準拠しています。アドバンテックデバイスWISE-4610は、さまざまなアナログおよびデジタルI / OおよびシリアルRS-485 / 232インターフェイスを備えたモジュラーI / O端子です。


お客様は、LoRaWANベースステーションを導入するか、サードパーティオペレーターの既存のネットワークを使用できます

パブリックネットワークLoRaWAN


このアーキテクチャでは、デバイスはサードパーティオペレーターのパブリックネットワークに接続されます。クライアントはサブスクライバーデバイスのみを購入し、プロバイダーと契約を結び、ネットワークにアクセスするためのキーを取得する必要があります。さらに、クライアントはオペレーターのカバレッジに完全に依存しています。
公共のLoRaWANネットワークでは、デバイスが占有できる通信時間に厳格な制限が存在する可能性があるため、より頻繁なデータ交換が必要なアプリケーションでは、このようなネットワークは適さない場合があることに注意してください。
データを送信する前に、デバイスは送信の許可を求め、基地局が確認で応答した場合にのみ、交換が行われます。


サードパーティのLoRaWANネットワークを使用する場合、クライアントは他の誰かの基地局インフラストラクチャを使用し、プロバイダーのカバレッジとそのデータ転送制限に依存します。

このアプローチは、たとえば、必要なインフラストラクチャがすでに存在する都市部のサブスクライバーデバイスを置き換える場合に便利です。たとえば、住宅の建物にセンサーを設置し、少量の送信データを使用して、電気または水道メーターからデータを収集します。このようなデバイスは、数日ごとにデータを送信できます。

プライベートネットワークLoRaWAN


プライベートネットワークを展開する場合、お客様は個別にベースステーションを設置し、カバレッジを計画します。このアプローチは、ネットワークを完全に制御する必要がある場合、またはオペレーターのカバー範囲がない施設で便利です。


プライベートネットワークでは、顧客がインフラストラクチャを完全に制御しますが、

このアーキテクチャでは、顧客は基地局を展開するための機器に一度投資するだけで、サービスやサードパーティのオペレーターに依存しません。このオプションは、遠隔地の農業施設、生産などでネットワークを構築するのに適しています。独自のネットワークにより、既存のインフラストラクチャの拡張、カバレッジ、サブスクライバーデバイスの数、および送信されるデータ量の増加が容易になります。

加入者入出力端子WISE-4610

WISE-4610 シリーズは、さまざまな周辺機器をLoRaWANネットワークに接続するためのモジュラーターミナルです。彼らの助けを借りて、温度計、湿度計、気圧計、加速度計などのデジタルおよびアナログセンサーからデータを収集し、RS-232 / 485インターフェースを介して他のデバイスを制御できます。内蔵の4000mAバッテリーは、最大6か月間独立して動作できます。ソーラーパネルと統合してバッテリーを充電します。内蔵のGPSレシーバーを使用すると、デバイスの場所を正確に特定できるため、アカウンティングとインストールが簡素化されます。デバイスを最初にデータベースに入力しなくてもマウントでき、インストール後、受信した座標に基づいて自動的にオブジェクトに関連付けられます。

プログラミングと構成は通常のUSBインターフェイスを介して実行され、追加のコントローラーやプログラマーを必要としないため、ラップトップを使用して現場で行うことができます。

インターフェースモジュール


外部デバイスをWISE-4610に接続するための一連のインターフェイスは、下から接続するインターフェイスモジュールを使用して実装されます。1つのインターフェイスモジュールを1つのWISE-4610ターミナルに接続できます。これらは、お客様のタスクに応じて、デジタルまたはアナログの入力/出力、またはシリアルインターフェイスになります。インターフェイスピンは、M12ネジ接続の気密コネクタで保護されています。
