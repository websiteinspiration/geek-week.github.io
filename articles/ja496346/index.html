ルートキットハンターのハッシュチェックを毎日のスクリプトに追加します

このすべてのudalenkaにより、以前は半分空だったトレーニング用ラップトップが主な動作マシンになり、なんとかしてより慎重に保護する価値があることがわかりました。一般に、当局が明らかに示唆していること。ラップトップ上に今Ubuntu 20.04ベータがあります。

すべては明確なBIOS設定と完全なディスク暗号化から始まりました。しかし、私はrkhunterユーティリティ（「ルートキット、バックドア、スニファー、エクスプロイトスキャナー」）がファイルレベルで実行できることを理解することにしました。

数年更新されていないソフトウェアに大きな期待はありませんでした。2018年2月のシグネチャを使用してルートキットを検索することは非常に疑わしいとすぐに言っておく必要がありますが、他の操作モードの1つ-ファイルハッシュのチェックに興味を持っていました。誰がそのような経験に興味があり、これまでに何が起こったか-猫へようこそ。

アイデアは単純ですが、変更すべきではないファイルのMD5ハッシュが変更されていないかどうかを確認するために、朝の便利なスクリプトにチェックを追加しましょう。少なくともパッケージの更新が必要になるまで。私の意見では、このように非常に軽量ですが、役に立たないわけではありません。ユーティリティのハッシュを計算およびチェックできる現在のバージョンは、パッケージマネージャーによって簡単に設定されます。

apt search rkhunter
rkhunter/focal,focal,now 1.4.6-8 all [installed]
  rootkit, backdoor, sniffer and exploit scanner

メインのハンター設定は/etc/rkhunter.confにあり、最初に、私の問題を解決するために、これらはそこで設定したパラメーターです：

ENABLE_TESTS=hashes

ハッシュを除けば、まだ何も興味がないからです。他の可能なテストのリストは、コマンドで取得できます。

sudo rkhunter --list test

私はまだMD5-okの衝突は数学に関連していると考えていますが、実際にはそうではありません。私はこの意見を主張しませんが、ほぼ毎日、私は良い害を見ており、これまでのところ、そのような野生の紛争に遭遇したことはありません。だから今のところは設定で十分です。

HASH_CMD=MD5

次に、彼は自分のパッケージマネージャーと、最も重要なことには、ディレクトリ、ファイルハッシュを私に興味のあるものとして示しました。パスは、たとえば/ binおよび/ usr / binになります。ディレクトリに出現したが、計算時に存在しなかったファイルについても、ハンターは警告を出します。

PKGMGR=DPKG
USER_FILEPROP_FILES_DIRS=/bin/*
USER_FILEPROP_FILES_DIRS=/usr/bin/*

ここで重要な点があります。私の設定では、スキャンはパッケージではなくディレクトリに基づいています。おそらく、これはより正確です。パッケージには実行不可能なファイルが多数含まれており、それらを最初に、たとえばファイルを介して除外する必要があるためです。これまで、ディレクトリと例外のリストを増やす方法を選択しました。

これでスキャナーの設定が完了しました。次に、/ var / lib / rkhunter / db / rkhunter.datの正しいハッシュのベースを新しい設定で更新します。これを行うには、コマンドが必要です（データベースに追加するパッケージの名前も使用します（逆の場合））。

sudo rkhunter --propupd

その後、関心のあるすべてのファイルとそのハッシュがrkhunter.datに表示されます。私の場合、これらは2847ファイルで、インデクサーは9分2,117秒かかりました。これは、MD5が3,000の場合、/ bin / lsのmd5sumが0.003の場合を考えると、少し多いですが、実際にはあります。

以上で、チェックを実行する準備ができました。まだ戦闘していないため、一部のマルウェアがハッシュを変更できた可能性はほとんどありません。

sudo rkhunter -c

ここには非常に便利なキーが1つあります。rwo（警告のみを報告）です。これにより、成功した比較がすべてスキップされ、警告のみが出されます。/var/log/rkhunter.logで作業の結果を探す必要があります。チェックの速度については、私のほぼ3000のバイナリでは、インデックス作成時間に相当する10分27,489秒でした。

検証を整理しましたが、パッケージを更新するときのインデックス再作成の問題が残っています。それら。私たちは適切な更新を行いましたが、今では更新されたすべてのファイルをマルウェアとは見なしていません。スキャナーには、タスクを自動化するための別のconfig / etc / default / rkhunterがあります。その中でパラメーターを設定します。

APT_AUTOGEN="yes"

また、/ etc / apt / apt.conf.d / 90rkhunterには、インデックスの再作成を行うスクリプトがあります。


 // Makes sure that rkhunter file properties database is updated after each remove or install only APT_AUTOGEN is enabled
 DPkg::Post-Invoke { "if [ -x /usr/bin/rkhunter ] && grep -qiE '^APT_AUTOGEN=.?(true|yes)' /etc/default/rkhunter; then /usr/share/rkhunter/scripts/rkhupd.sh; fi"; };

実際、誰でも、どんな追加でも受け入れてくれます。お役に立てば幸いです。
