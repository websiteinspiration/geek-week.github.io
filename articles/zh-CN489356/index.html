AWS，Azure和Gitlab上的CI / CD。OTUS开设的新课程

注意！本文不是工程知识，而是针对对CI / CD领域的教育感兴趣的读者。最有可能的是，如果您对学习不感兴趣，那么您将不会对本材料感兴趣。



如果您是负责设置持续集成/持续交付过程的开发人员或管理员，那么OTUS将专门为您开设一系列课程：关于持续开发和交付持续集成和持续交付软件的流行方法的实用密集课程。跨Amazon Web Service，Azure，GitLab和Jenkins平台。

在培训期间，学生将学习如何与三个领先的提供商一起配置应用程序的组装和测试过程以及安装过程，并形成对云提供商的体系结构的了解，并学习代码分析和漏洞搜索的自动化。

培训的结果是，每个学生都将创建毕业作品，该作品将是CI / CD流程的实施，供任何开放源代码项目选择。当然，经过培训后，每位学生将获得所有课程的材料，结业证书，最重要的是，配置应用程序的组装和测试并能够发现漏洞。



当然，此课程并不适合所有人。但是，如果您有经验：

  • 使用git
  • Linux或Windows系统管理
  • 开发或运营
  • 与云提供商合作

那么OTUS在等着你！您可以参加入学考试以确定您是否具有足够的知识来完成有关AWS，Azure和Gitlab的CI / CD课程。

在“适用于AWS，Azure和Gitlab的CI / CD”课程开始的前夕，OTUS举办了开放日活动。老师详细介绍了该课程计划，回答了学生的问题，还描述了学习过程。


同样开放的是，可以免费访问有关开放式网络研讨会，主题为“将Jenkins与K8S结合使用”，该研讨会由AWS，Azure和Gitlab的CI / CD 课程的讲师Boris Nikolaev 主持：


关于“ AWS，Azure和Gitlab上的CI / CD”课程的培训过程以在线网络研讨会的形式进行。在整个培训过程中（持续3个月），学生可以询问经验丰富的老师，他们总是保持联系。练习练习将使用Google Cloud Platform（GCP），Amazon Web Service和Microsoft Azure完成。

该课程计划包含四个主要模块：

  1. 云中的开发（代码）
  2. 组装和测试自动化（持续集成）
  3. 安装自动化（连续交付）
  4. 摘要模块

在课堂上，每个人都将以在线网络研讨会的形式被全面理解，而家庭作业将有助于巩固所获得的知识，根据这些知识，如有必要，您可以从老师那里获得详细的反馈。
