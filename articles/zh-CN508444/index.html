在OpenNebula上安装和配置LXD

在Linux Administrator课程启动之前，准备了本文的翻译。









在本文中，您将找到有关LXD上基于OpenNebula云的业务流程系统基础知识的分步指南。首先，我们将使用简单的MiniONE工具在物理机或虚拟机中为想要试用OpenNebula功能的用户创建单节点云环境，然后查看具有GUI和CLI的常规工作会议，最后提供一些有助于创建分布式生产环境的链接。



MiniONE自动部署



MiniONE是一种工具，可将物理主机或虚拟机配置为单节点云，从而快速部署简单但功能齐全的测试用例。



要评估LXD，MiniONE需要一个专用的虚拟机或物理主机，并重新安装Ubuntu 18.04或18.10，并具有至少2GB的内存和20GB的可用磁盘空间，以及特权根访问权限。例如，MiniONE使在Amazon虚拟机上构建LXD / OpenNebula环境变得容易。最小推荐大小是最可能的t2.medium



。只需给她25 GB的磁盘空间并访问运行WebUI的TCP端口9869。



在专用系统上运行MiniONE脚本： 



wget https://github.com/OpenNebula/minione/releases/download/v5.8.0/minione
chmod u+x minione
sudo minione --lxd

完成MiniONE之后，我们要做的第一件事就是以oneadmin身份登录并查看Sunstone中的Admin View，对于普通用户而言，那里的选项比其他界面要多得多。查看所有下载的云资源。







在Admin View中，您可以使用OpenNebula进行任何操作，但是您不需要为终端用户提供这些相同的机会！切换到Cloud View，查看最终用户在OpenNebula中将看到什么。云视图







界面因为它是针对最终用户的，所以简单得多。通过单击+按钮创建一个新的虚拟机。选择唯一可用的模板，然后单击创建按钮。单击“ 创建”按钮后，将带您进入仪表板，在这里您可以看到所有正在运行的虚拟机。







要打开控制面板，您可以单击虚拟机。在那里，你可以从VNC访问它，保存其状态，重新启动它，等：







通过单击控制台图标，你到了控制台根为通过VNC用户。







使用oneadmin角色，您可以配置他们可以看到和做到您的云用户。



符合CLI



OpenNebula以oneadmin身份启动，并且主要管理员必须代表该用户执行命令，因此您需要做的第一件事就是切换到oneadmin： 



su - oneadmin




从oneadmin帐户中，您可以看到已加载的资源：

我们有一个可视化节点： 



onehost list


创建的CentOS映像 



oneimage list


注册的虚拟机模板： 



onetemplate list


您可以更详细地查看模板的配置。 



onetemplate show 0

生产安装组装



如果要在生产中部署云，可以阅读文档。部署具有前端和多个LXD节点的环境需要执行以下步骤：



  • 安装OpenNebula前端;
  • 在虚拟化节点中安装lxd-node软件包；
  • 连接前端和节点 ;
  • 验证一切正常。


现在，您可以使用MarketPlace中的应用程序部署虚拟机。
