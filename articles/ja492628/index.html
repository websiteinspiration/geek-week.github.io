コロナウイルス患者数の信頼区間（死亡率の計算）

コロナウイルスに関するバイラル出版の人気のある議論-3つのケースの統計を取得するにはどうすればよいですか？そのような小さなサンプルについて結論を出すことはできません！社会科学を研究したすべての人が、母親の母乳を使ってサンプルサイズに関するこの話を吸収しました。そして、これは、私たちが通常扱う状況、つまり選択的統計については正しいです。

亡くなった3人の場合、これらの統計は非常に間接的に関連しています。私がまだ大学で心理学者に数学の方法を教えていた頃、私はいつもこの場所に立ち寄ろうとしました-このコース全体が扱っていることは実際のデータとは関係ありません。無作為標本から一般集団について結論を出す必要がある場合にのみ、問題に。

そして、ここに数字3があります。3つの死者、ある種のベクトルではなく、テーブルやサンプルではありません。事実です。3人の死者が偶然に私たちのところにやってきました。彼らは死んだ。

したがって、死亡率と死亡数によって、症例数を決定する最も簡単な方法の1つを検討します。死亡率がわかっていて、それが1％だとします。この状況では、回復した人の数が297人になると考えるのは論理的で正しいでしょう。しかし、この判断の信頼性はどの程度でしょうか。3人が統計ではないと言って、3人が死亡したことを片付けてもよいでしょうか。



この質問は、負の二項分布とその予言者によって答えられます-ウィキペディア。ギリシャ文字はたくさんありますが、私と同じようにあなたが恐れているなら、どうなるか教えてあげましょう。この分布は、6が5回落ちるようにサイコロを振る必要がある回数の質問に答えます。計算にはRプログラミング言語を使用していますが、ここには信頼区間を評価できる既製の関数があります。

qnbinom(p=c(.025,.975),size=3, prob=0.01)

ここで、pは下から2.5％、上から2.5％で、その間に目的の範囲があります。

その結果、信頼区間は60から717になります。それほど悪くはありません！ 3人の死者は297人が回復したという意味ではなく、60人しかいないと思われます。しかし、多分700のために。 :-(

負の二項分布を信じない非常に疑わしい人のために、私は数値モデリングを提供できます。一般に、数式と分布による計算方法がわからない場合は、モデル化してください！理解できない状況では、モデル、モンテカルロがあなたを待っています

。random_infected関数を作成します、病気や死の状況をシミュレートします。

random_infected <- function(deaths, fatality_rate)
{
  dead = 0
  all = 1
  while (dead < deaths) {
    if (runif(1) < fatality_rate) {
      all = all + 1
      dead = dead + 1
    } else
      all = all + 1
  }
  return(all)
}

この関数は次のことを行います-「n面」の立方体をロールします（均一分布を使用）。 1人がドロップされると、死者の数とすべての数が1つずつ増加します。そうでない場合は、数だけです。この立方体の各ロールは、死ぬか回復することができる病気の人です。 deathsパラメータで指定された死亡数が得られたらすぐに停止し、サイコロを振った回数（allの数）を報告します。架空の立方体に1つが落ちる可能性は死亡率で、この場合はfatality_rateパラメータです。

infected_sizes<-replicate(100000,random_infected(deaths=3,fatality_rate=0.01))

そして、この数を10万回計算してみましょう。私は古いラップトップを持っているので、100万人が数えられるまで待つのをためらいます。

その後、得られた数値の算術平均を計算できます。私は301.2を取得しました-予想される数300に非常に似ています。これは、私たちのデスキューブのロール数の分布がどのように見えるかです。

library(ggplot2)
theme_set(theme_classic())

g <- ggplot(data.frame(infected_sizes=infected_sizes), aes(infected_sizes))
g + geom_density(alpha=0.8,fill="plum")



ここにあります-負の二項分布、愛してください。そのようなデータに基づいて、質問におおよその回答を与えることができます-症例の総数が50（1.2％）未満または1000（0.3％）を超える確率はどれくらいですか。

もちろん、これらは単なる推定値です。それらは正しくない可能性のあるデータに基づいています。コロナウイルスの真の死亡率については知りません。しかし、この死亡率が低ければ低いほど、1人の死亡者により多くの病気が発生し、パンデミックの程度の推定が大きくなります。

このサイコロをすぐに振ります。 Thomas Pueyoによるセンセーショナルな記事で使用された死亡率の計算モデルに対して、少し不満があります。そこで、X日目の3人の死亡、1％の死亡率、および感染から死までの平均時間が17日であるという知識に基づいて、X-17日目に300人が感染したと仮定します。ただし、このような計算は、病気の人の数が毎日同じ場合にのみ有効です。 17日間は厳密な数ではないため、信頼区間とエラーもあります。患者数が急激に増加した場合、X日目に死亡した人の中で、17日前ではなく、16または15日、場合によっては10日前に感染した人がいます。おそらく、17日前に感染した人よりさらに多くの人がいるでしょう。この方法では、症例数の急激な増加の状況では、そのような逆算は疾患の有病率の過大評価につながる可能性があります。一般に、すべてが複雑です。
