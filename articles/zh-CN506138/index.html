为什么元组中的列表在Python中表现异常？

在编程语言中，我一直对它们的内部结构感兴趣。一个或另一个操作员如何工作？为什么这样写更好，而不是这样？这样的问题并不总是能“在这里和现在”解决问题，但从长远来看，它们构成了编程语言的整体图景。今天，我想分享一次潜水的结果，并回答在修改tuple“ a in list” e 时会发生什么的问题。

我们都知道Python具有数据类型list：

a = []
a.append(2)

list只是一个数组。它允许您添加，删除和修改项目。它还支持许多不同的有趣运算符。例如，+ =运算符可将项目添加到list。+=更改当前列表，但不创建新列表。这在这里清晰可见：

>>> a = [1,2]
>>> id(a)
4543025032
>>> a += [3,4]
>>> id(a)
4543025032

Python还有另一种很棒的数据类型：tuple-不可变的集合。它不允许添加，删除或更改元素：

>>> a = (1,2)
>>> a[1] = 3
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

使用运算符时+=，将创建一个新运算符tuple：

>>> a = (1,2)
>>> id(a)
4536192840
>>> a += (3,4)
>>> id(a)
4542883144

注意，问题是：以下代码将做什么？

a = (1,2,[3,4])
a[2] += [4,5]

选项：

  1. 项目将添加到列表中。
  2. 元组不变性异常将被抛出。
  3. 都。
  4. 两者都不是。

在纸上写下您的答案，让我们做一点检查：

>>> a = (1,2,[3,4])
>>> a[2] += [4,5]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

好吧！所以我们想通了！正确答案是2。尽管，请稍等：

>>> a
(1, 2, [3, 4, 4, 5])

实际上，正确答案是3。也就是说，添加了元素，并且出现了异常-wat？！


让我们看看为什么会这样。一个很棒的模块将帮助我们dis：

import dis

def foo():
    a = (1,2,[3,4])
    a[2] += [4,5]

dis.dis(foo)
  2     0 LOAD_CONST      1 (1)
        3 LOAD_CONST      2 (2)
        6 LOAD_CONST      3 (3)
        9 LOAD_CONST      4 (4)
       12 BUILD_LIST      2
       15 BUILD_TUPLE     3
       18 STORE_FAST      0 (a)

  3    21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)
       27 DUP_TOP_TWO
       28 BINARY_SUBSCR
       29 LOAD_CONST      4 (4)
       32 LOAD_CONST      5 (5)
       35 BUILD_LIST      2
       38 INPLACE_ADD
       39 ROT_THREE
       40 STORE_SUBSCR
       41 LOAD_CONST      0 (None)
       44 RETURN_VALUE

第一个块负责构建tuple'a并将其存储在变量中a。然后乐趣就开始了：

       21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)

我们将指向变量a和常量2 的指针加载到堆栈上。

       27 DUP_TOP_TWO

复制它们并将它们以相同顺序放入堆栈中。

       28 BINARY_SUBSCR

该运算符获取堆栈的顶部元素（TOS）和下一个元素（TOS1）。并将一个新元素写入堆栈的顶部TOS = TOS1[TOS]。因此我们从堆栈中删除了前两个值，并在其中放置了指向第二个元素tuple'a（我们的数组）的链接。

       29 LOAD_CONST      4 (4)
       32 LOAD_CONST      5 (5)
       35 BUILD_LIST      2

我们构建元素4和5的列表，并将其放在堆栈的顶部：

       38 INPLACE_ADD

应用+=到堆栈的前两个元素（重要！这是两个列表！一个包含4和5，另一个来自tuple）。一切都很好，指令执行没有错误。由于它+=修改了原始列表，因此tuple'e中的列表已经更改（当时）。

       39 ROT_THREE
       40 STORE_SUBSCR

在这里，我们交换堆栈的前三个元素（tuple位于其中，位于数组的索引和新数组中）并按tuple索引将新数组写入。这是发生异常的地方！

好吧，就是这样！实际上，列表可以更改，但是一切都取决于操作员=。

最后，让我们弄清楚如何无例外地重写此代码。正如我们已经了解的，您只需要删除中的条目tuple。这里有几个选择：

>>> a = (1,2,[3,4])
>>> b = a[2]
>>> b += [4,5]
>>> a
(1, 2, [3, 4, 4, 5])

>>> a = (1,2,[3,4])
>>> a[2].extend([4,5])
>>> a
(1, 2, [3, 4, 4, 5])
