私はいつも色の科学に興味を持っていました。 2014年にはさまざまな会議でCSS Color 4仕様について話し、それより前は2009年に色を選択するためのプログラムを書きました。彼女は非表示のJavaアプレットを使用して、ICCカラープロファイルをサポートし、CMYKで適切に動作しました。私の知る限り、この種の最初のプログラムでした。私はそれをリリースしたことはありませんが、それがこの出版物の出版につながりました。色への興味から、私は夫になった男に連れて行きました。 2019年、CSS Color 5仕様の共同編集者になりました。私の目標は私の提案を具体化することです。

LCHは、誰もがCSSの操作に慣れているRGB / HSL色空間よりもいくつかの利点がある色空間です。実際、私はさらに進んで、LCHをWebでの色の動作を劇的に変えるテクノロジーと呼んでいます。ここに私がそのような発言をすることを可能にする3つの事実があります。

▍1。LCHはsRGBと比較して、約50％多い色にアクセスできます


これは大きな前進です。現在、設定できるCSSカラーはすべてsRGBカラースペースで定義されています。数年前は、プロフェッショナルモニターを除くすべてのモニターの色域がsRGB未満であったため、これで十分でした。しかし、今ではすべてがそうではありません。現在、ほとんどのモニターの色域（つまり、表示できる色の範囲）はP3に近いです。そして、この色空間のボリュームは50％以上ですスペースsRGBの量よりも。CSSは現在、これらの色にまったくアクセスできません。繰り返しますが、現代のモニターが表示できる3分の1の色にはアクセスできません。そして、それはいくつかのマイナーな色についてではありません。モニターが表示できる最も鮮やかな色について話しています。モニターハードウェアはCSS仕様やブラウザー実装よりも速く進化しているため、私たちのサイトは見た目が悪くなっています。

両方の色の明度は50％ですが、実際の明るさはまったく同じではありません。 HSLの「軽さ」の概念の意味は何ですか？

ここでは、少なくとも、同じ色調（色相）と彩度である程度の明るさは理にかなっていると言って、私と議論することができます。つまり、同じ色の中で明度を変更する場合です。そして真実：HSLの明度を上げると、明るい色になり、下げる-暗い色になります。しかし、起こることは必ずしも同じ色ではありません。

LCHカラーはどのように配置されますか？


LCHは、明度、クロマ、色相（明度、色、トーン）の略です。このカラーモデルのカラーコンポーネントは、HSLカラーモデルのコンポーネントと何らかの関係があります。ただし、これらのカラーモデルのコンポーネントには重大な違いがあります。

LCHのピッチ角はHSLトーンと完全には一致しません。つまり-0-これは完全に赤ではありません。この色はマゼンタにかなり近いです。180はターコイズブルーの色ではなく、青緑がかった色で、完全に補色です。


同じ明るさの

色これらの色は、色調は大きく異なりますが、同じ明るさの色のように見えます。

HSL色空間では、彩度は0から100までの適切な範囲の値で表されます。実際、この空間はRGBを極座標に単純に変換した結果です。ただし、LCHでは、彩度成分の値は理論的には無制限です。 LCH標準（Labなど）は、人間が知覚するすべての色を表現できるように設計されています。 P3色空間をサポートしているものでも、これらの色のすべてをモニターに表示できるわけではありません。さらに、モニターの色域に応じた最大表示色数だけでなく、色の違いについても触れています。

例を挙げて説明すると、おそらくこのアイデアは理解しやすいでしょう。簡単にするために、色域がsRGB色空間に完全に一致するモニターがあるとしましょう。比較として、2013年のMacBook Air画面は約60％のsRGBを出力できますが、最近のほとんどのディスプレイは、すでに述べたように150％のsRGBを表示できます。 L = 50およびH = 180（上図のシアンのサンプル）の場合、Cの最大値は35のみです。 L = 50およびH = 0（ピンク）の場合、Cの値はsRGBを超えることなく77に到達できます。 L = 50およびH = 320（紫）の場合、Cの値は最大108になります。

境界線の欠如は恐ろしいものとして認識されますが（人やカラースペースについて）、心配する必要はありません。特定のモニターで表示できない色が設定されている場合、モニターでサポートされている同様の色に減ります。結局、何も新しいものはありません。モニターがsRGBの機能を超える色域を取得するまで、これは、sRGB未満の色域のモニターに表示される通常のCSSカラーで起こったこととまったく同じでした。

質問と答え


私がこの記事を公開した後、彼らは私に質問し始めました。私はここでいくつかの一般的な誤解を明確にすべきだと思います。

▍ブラウザーでのLCHカラーのサポートはまだ実装されていないとのことですが、記事で確認できます。これはどのようにして可能ですか？


この記事で使用されているすべての色は、sRGB色域に含まれます。これは、これまでのところsRGB以外の色を出力することはできないためです。sRGBは色空間であり、色の記述構文ではありません。たとえば、同じ色rgb(255 0 0)をlch(54.292% 106.839 40.853)設定します。

LLCHカラーピッカーはsRGB以外の色をどのように表示しますか？


私のプログラム、LCH Color Pickerはそのような色を表示しません。そして、私が知る限り、インターネットのどこにもそのような色を表示できるプログラムはありません。 LCH Color PickerプロジェクトはWebテクノロジーを使用して実装されています。つまり、標準の制限が他のすべてのWebページに適用されます。非出力カラーをsRGBに変換します。最初は、RGBカラーコンポーネントを0〜100％の範囲に単純に設定しましたが、このPRのおかげで、プログラムはより成功したアルゴリズムを使用するようになりました。つまり、色がsRGBの制限内に収まるまでLCHコンポーネントCの値を減らすことについて話している。そのため、コンポーネントCの値が特定の制限を超えて増加しても、明るい色が得られません。実際には、CSSを使用してそのような色を推定することはまだできません。

Firefox FirefoxがChromeやSafariよりも明るい色を表示することに気付きました。LCHサポートとの接続はありますか？


Firefoxは、CSSカラーをsRGBフレームに制限する仕様の一部を実装していません。代わりに、ブラウザーは生のRGB値を画面に渡すだけです。たとえば、rgb(100% 0% 0%)これはモニターが表示できる最も明るい赤色です。これはスマートなソリューションのように思えるかもしれませんが、ここでは矛盾に直面しています。実際のところ、この方法では、すべての画面で色が異なって表示されるため、色を設定できるのはせいぜい、おおよそだけです。CSSの色を既知の色空間（sRGB）に制限することで、デバイスに依存しなくなります。LCHとLabの色空間も、実際に測定された色に基づいているため、デバイスに依存しません。

theカラー（display-p3 rgb）デザインについて何と言えますか？Safariは2017年からサポートしています。


この記事を公開した後、私はこれについて知らされました。Safariがこの機能に取り組んでいることは長い間知っていましたが、どういうわけか、準備ができた瞬間を逃しました。実際、先月この記事はWebKitでこの構文を説明して公開されました。すごい

デザインcolor(colorspaceid params)はCSS Color 4で説明されている機能です。 CSSのカラーマネージメントに関しては、彼女はスイスのナイフと比較できます。このデザインの素晴らしい点は、ICCカラープロファイルとそこから取得した色を指定できることです。たとえば、Webページで実際のCMYKカラーまたはPantoneカラーを使用する場合に便利です。この設計は、いくつかの事前定義された色空間もサポートします。そのうちの1つはdisplay-p3です。たとえばcolor(display-p3 0 1 0)、P3色空間で最も明るい緑色になります。この設計のサポートをテストするには、このテストを使用できます。サポートがない場合はcolor()、赤色が表示され、可能な場合は明るい緑色になります。

これは素晴らしい機能です（color()この機能がサポートされている場合にプログラムで使用できるように、LCHカラーピッカーを再作成する必要があります）が、この機能が解決したのは最初に述べた問題の1つだけであることに注意してください。つまり、色域からすべての色へのアクセスについて話しています。ただし、このツールはcolor()RGBに基づいているため、RGBに関連するすべての欠点に悩まされています。それは均一な知覚によって区別されず、その助けにより、カラーパラメータを調整することにより、カラーオプション（明るいまたは暗いオプション、多かれ少なかれ明るいなど）を作成することは困難です。

さらに、これは一時的な解決策です。P3に収まらない色を表示できる画面は珍しいので、今ではそれが当てはまります。しかし、ディスプレイをさらに改良した結果、デザインcolor(display-p3 ...)にはRGBカラーと同じ問題が発生することがわかりました。

LCHとLabの色空間はデバイスに依存せず、人間に見えるすべての色を表すことができるため、ハードウェアの進化に関係なく関連性が維持されます。
