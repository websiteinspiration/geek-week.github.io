go-applicationsにデータを保存するためのボックス

CofferGolang と呼ばれる埋め込みKey-Valueデータベースに関する短いメモ。非常に短時間の場合：データベースが停止状態の場合、データはディスク上にあり、データベースが起動すると、データはメモリにコピーされます。読書は記憶から来ます。記録すると、メモリデータが変更され、その変更がログ上のディスクに書き込まれます。保存されるデータの最大サイズは、RAMのサイズによって制限されます。APIを使用すると、データの一貫性を維持しながら、データベースレコードのヘッダーを作成してトランザクションに適用できます。

しかし、最初に、少し叙情的な紹介。むかしむかし、草が青々としたときに、go-applicationのKey-Valueデータベースを埋め込む必要がありました。周りを見回して、さまざまなパッケージにぶつかって、どういうわけか（主観的に）欲しいものが見つからず、外部のリレーショナルデータベースを使用してソリューションを適用しただけでした。素晴らしい作業ソリューション。しかし、彼らが言うように、スプーンが見つかりましたが、土砂は残っていました。まず第一に、私はGoデータベースに書かれたネイティブをそのままネイティブネイティブにしたかったのです。そして、そのようなものがあります。しかし、それらの百万はありません。人生でデータベース、フレームワーク、またはカジュアルゲームを作成しなかったプログラマーは世界ではまれだと考えると、これは驚くべきことです。


さて、ブラックジャックやその他のグッズを使って、自転車を膝に乗せてみてください。同時に、誰もが知っている、または少なくとも推測して、単純なKey-Valueデータベースを作成することさえ、一見しただけでは単純に見えることを知っています。しかし、実際には、すべてがはるかに楽しいです（そしてそれは起こりました）。そして、私はACIDについての好奇心と心配されている取引に打ち勝ちました。真のトランザクションは、財務的な意味でより可能性があります。それから私はフィンテックで忙しかった。

簡単な使用例：

package main

import (
	"fmt"

	"github.com/claygod/coffer"
)

const curDir = "./"

func main() {
	// STEP init
	db, err, wrn := coffer.Db(curDir).Create()
	switch {
	case err != nil:
		fmt.Println("Error:", err)
		return
	case wrn != nil:
		fmt.Println("Warning:", err)
		return
	}
	if !db.Start() {
		fmt.Println("Error: not start")
		return
	}
	defer db.Stop()

	// STEP write
	if rep := db.Write("foo", []byte("bar")); rep.IsCodeError() {
		fmt.Sprintf("Write error: code `%d` msg `%s`", rep.Code, rep.Error)
		return
	}

	// STEP read
	rep := db.Read("foo")
	rep.IsCodeError()
	if rep.IsCodeError() {
		fmt.Sprintf("Read error: code `%v` msg `%v`", rep.Code, rep.Error)
		return
	}
	fmt.Println(string(rep.Data))
}
