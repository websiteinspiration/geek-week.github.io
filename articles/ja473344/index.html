誰かがすぐに結果を確認したい場合、または「ミラー」にモジュールをインストールしたい場合は、ここにリポジトリへのリンクがあります。インストール手順もあります。

情報源として、kudagoサービスが最も適切であることが判明しました。登録は必要ありません。健全な方法。答えの簡潔なデータ。奇妙な報道ではなく、「都市の縁石」や「非ゴム」だけでなく、ロシアの9つの主要都市も含まれています。

開発したモジュールは、modules / modulenameフォルダーに保存する必要があります。（私のような）単純なモジュールを作成するには、このフォルダーに2つのファイルを作成する必要があります。

  • modulename / modulename.jsは、メインモジュールのスクリプトです。
  • modulename / node_helper.jsは、スクリプトによってロードされる「オプションのヘルパー」です。ヘルパーとモジュールスクリプトは、統合ソケットシステムを使用して相互に通信できます。

ティンカー製たMagicMirror開発者からのガイドだけでなく、他の人のソリューションを見て、私はXMLHttpRequestのを使用してkudagoのAPIからエントリを取得するには、スクリプトをアップロードしました。そして、クロスドメインクエリの問題に遭遇しました。私はJavaScriptの専門家ではありません（開発者でさえありません）。「ブラウザでスクリプトを実行する」というリクエストを送信してkudagoからの応答を受け取ることしか理解できなかったので、成功しませんでした。これがlearn.javascriptが私に説明したものです

まあ、本当に私が欲しかったものではありません。JavaScriptの問題を理解したくなかった。結局のところ、手元にあったのは使い慣れた予測可能なPythonでした。リクエストライブラリから情報を取得し、結果をjsonに書き込みます。Pythonのコードをjsから実行する方法の問題を解決するだけです。

node_helper.jsが私にとって便利であることがわかりました！モジュールを開始するときに、python-shellの機能を使用できます。Pythonインタープリターへのパス、スクリプトへのパス、引数とその値を指定するだけです。この形で、クダゴは私に親切な答えを返してくれました。

Pythonスクリプト呼び出しコード
kudagoGetData: function () {
    //call python script for collecting events from KudaGo api
    self = this;
    var options = {
      pythonPath: this.config.pythonPath,
      scriptPath: './modules/MMM-KudaGo',
      mode: 'json',
      args: [
        "--location", this.config.location,
        "--days", this.config.days,
        "--categories", this.config.categories,
        "--tags", this.config.tags,
      ]
    };

    pyshell.PythonShell.run('KudaGo.py', options, function (err) {
      if (err) throw err;
    });
  }


さらに、node_helper.jsだけがjsonファイルからの情報を読み取ることができます（デフォルトでは、ファイルはパスkudago / events.jsonにあります）とkudagoからダウンロードされたデータ。そのファイルから同じ読み取りを実装します。

ファイルから情報を読み取るためのコード
readData: function () {
    //read a file with events
    fs.readFile("./modules/MMM-KudaGo/kudago/events.json", "utf8", (err, data) => {
      if (err) throw err;
      this.sendSocketNotification("DATA", data);
    });
  }


すでに書いたように、MagicMirrorではすべてのモジュールがソケットシステムを介して通信するため、起動時にPythonスクリプトを実行するには、適切なコードをnode_helper.jsに追加します

起動時にモジュールを起動するコード
socketNotificationReceived: function (notification, payload) {
    if (notification === "START") {
      this.config = payload;
      this.kudagoGetData();
      setInterval(() => {
        this.kudagoGetData();
      }, this.config.updateInterval);
    }
  }


this.config から取得されるすべてのパラメーターは、モジュールのメインファイルの事前定義された変数、またはconfig.js（すべてのMagicMirrorモジュールの一般的な構成ファイル）で指定された変数のいずれかです。
