Nginxをまとめる

こんにちは！
私の名前はサーゲイです。私はtinkoff.ruプラットフォームAPIチームでインフラストラクチャエンジニアとして働いています。

この記事では、さまざまなプロジェクトでNginxベースのバランサーを準備する際にチームが直面した問題について説明します。また、それらのほとんどを克服できるツールについても話します。

Nginxは、多機能で積極的に開発されているプロキシサーバーです。多数のモジュールがありますが、完全なリストではありません。各プロジェクトは、バランサーとNginxバージョンの機能（たとえば、http / 2とプロキシgrpcの存在）、およびそのモジュールの構成に特定の要件を課します。

特定のLinuxディストリビューションで動作する、適切なモジュールセットを備えた最新バージョンを確認したいと考えています。私たちの場合、これらはdebおよびrpmベースのシステムです。この記事では、コンテナーオプションは考慮されていません。

バランサーの機能をすばやく変更したいと考えています。そして、ここで問題がすぐに発生します-できるだけ少ないリソースを費やすことによってこれを達成する方法は？そして、有限数の入力パラメーターを設定できるようにプロセスをセットアップし、出力で目的のOSのdeb / rpmパッケージの形式でアーティファクトを取得する方がよいでしょう。

その結果、いくつかの問題が定式化されます。

  • Nginxの最新バージョンのパッケージが常に存在するわけではありません。
  • 適切なモジュールを含むパッケージはありません。
  • パッケージの手動でのコンパイルとアセンブルには多くの時間がかかり、面倒です。
  • 特定のNginxインスタンスの構築方法についての説明はありません。

これらの問題を解決するために、ツールは人間が読める形式の仕様を受け入れ、それに必要な機能を備えたNginxパッケージを収集するように頼みます。

githubのオープンスペースで適切なオプションが見つからなかったため、独自のツールnginx-builderを作成することにしました。

仕様書


私たちのツールでは、コードの形式で仕様記述を作成し、それをGitリポジトリーに入れることができました。これを行うために、そのようなものには通常の形式を選択しました-yamlです。仕様例：

nginx_version: 1.14.1
output_package: deb
modules:
  - module:
      name: nginx-auth-ldap
      git_url: https://github.com/kvspb/nginx-auth-ldap.git
      git_branch: master
      dependencies:
        - libldap2-dev
  - module:
      name: ngx_http_substitutions_filter_module
      git_url: https://github.com/yaoweibin/ngx_http_substitutions_filter_module.git
  - module:
      name: headers-more-nginx-module
      web_url: https://github.com/openresty/headers-more-nginx-module/archive/v0.261.zip
  - module:
      name: nginx-module-vts
      git_url: https://github.com/vozlt/nginx-module-vts.git
      git_tag: v0.1.18
  - module:
      name: ngx_devel_kit
      git_url: https://github.com/simplresty/ngx_devel_kit.git
      git_tag: v0.3.0
  - module:
      name: ngx_cache_purge
      git_url: https://github.com/FRiCKLE/ngx_cache_purge.git
  - module:
      name: ngx_http_dyups_module
      git_url: https://github.com/yzprofile/ngx_http_dyups_module.git
  - module:
      name: nginx-brotli
      git_url: https://github.com/eustas/ngx_brotli.git
      git_tag: v0.1.2
  - module:
      name: nginx_upstream_check_module
      git_url: https://github.com/yaoweibin/nginx_upstream_check_module.git
  - module:
      name: njs
      git_url: https://github.com/nginx/njs.git
      git_tag: 0.2.5
      config_folder_path: nginx

ここでは、必要なモジュールのセットを含むNginxバージョン1.14.2のdebパッケージを見たいことを示しています。モジュールを含むセクションはオプションです。それぞれに設定できます：

  • 題名。
  • 入手できる住所：
    • Gitリポジトリ。ブランチまたはタグを指定することもできます。
    • アーカイブへのWebリンク。
    • アーカイブへのローカルリンク。

一部のモジュールでは、追加の依存関係のインストールが必要です。たとえば、nginx-auth-ldapにはlibldap2-devをインストールする必要があります。必要な依存関係は、モジュールの説明でも指定できます。

環境


私たちのツールでは、パッケージやその他のサポートソフトウェアをコンパイル、ビルドするためのユーティリティがインストールされた環境をすばやく取得できます。ここでは、必要なものがすべて揃ったdockerコンテナーが最適です（ubuntuとcentosのdockerファイルの例がリポジトリにすでにいくつかあります）。

仕様が環境によってコンパイルおよび準備された後、コレクターを実行し、その依存関係をプレインストールします。

pip3 install -r requirements.txt
./main.py build -f [_].yaml -r [_]

ここでのリビジョン番号はオプションであり、アセンブリのバージョン管理に役立ちます。パッケージのメタ情報に記録されているため、サーバーでの更新が簡単です。
ログによって、何が起こっているのかを観察できます。ハイライトの例を次に示します。

builder - INFO - Download scripts for build deb package
builder - INFO - Downloading nginx src...
builder - INFO - --> http://nginx.org/download/nginx-1.14.1.tar.gz
builder - INFO - Downloading 3d-party modules...
builder - INFO - Module nginx-auth-ldap will download by branch
builder - INFO - -- Done: nginx-auth-ldap
builder - INFO - -- Done: ngx_http_substitutions_filter_module

したがって、いくつかのコマンドを実行するだけで、環境と目的のNginxアセンブリが作成され、パッケージはスクリプトの起動元のディレクトリに表示されます。

埋め込み


また、CI / CDプロセスにツールを組み込むこともできます。TeamcityやGitlab CIなど、多くの既存のCIシステムのどれでもこれを支援できます。

その結果、Gitリポジトリの仕様を変更するたびに、アーティファクトのアセンブリが自動的に開始されます。リビジョン番号はビルド起動カウンターに関連付けられています。
もう少し時間をかけた後、ローカルパッケージリポジトリ、Nexus、Artifactoryなどに送信されるようにアーティファクトを構成できます。

さらに、構成yaml-fileをAnsibleまたは自動構成の別のシステムに接続し、そこから、デプロイするパッケージのバージョン番号とタイプを取得することができます。
