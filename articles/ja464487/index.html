選択によりドキュメントを入力するためのインターフェースを構築しています


さまざまなビジネスアプリケーションでは、ドキュメント入力のタスクがよく発生します。通常、ドキュメントは見出しといくつかの行で構成され、それぞれが何らかのオブジェクト（製品など）を参照します。ほとんどの場合、ユーザーが行を追加および削除したり、内容を変更したりできるドキュメントにレコードを入力するために、通常のテーブルが使用されます。

ただし、場合によっては、このようなスキームがユーザーにとって必ずしも便利であるとは限りません。この記事では、製品を選択する手法を使用して、便利なユーザーインターフェイスを構築する方法について説明します。

仕事


原則として、ユーザーのドキュメントを入力する場合、追加できるオブジェクトのセットには制限があります。この場合、ユーザーがそのようなオブジェクトのリストを表示して、それらをドキュメントにすばやく含める（および除外する）ことができるのは当然です。列には、オブジェクトごとに、ドキュメントに含める必要があるかどうかを判断するために必要なデータを表示すると便利です。そして最後に、オブジェクトとともに、その数量を入力する必要があることがよくあります。

ビジネスアプリケーションでよく見られる3種類のドキュメントについて考えてみましょう。

  1. 注文書。このようなドキュメントでは、サプライヤからの注文に利用できるすべての商品のリストをユーザーが表示することが論理的です。列には、現在の残高、特定の期間の売上高、購入および販売のために注文された数量を表示すると便利です。

次に、オープンで無料のlsFusionプラットフォームに基づいて、このロジックを迅速かつ簡単に実装する方法を示します。例として、注文書を入力するためのロジックを作成してみましょう。

まず、標準のCRUDインターフェイスを使用して製品ディレクトリを追加します。
CLASS Product '';
name '' = DATA STRING[50] (Product);

FORM product ''
    OBJECTS p = Product PANEL
    PROPERTIES(p) name
    
    EDIT Product OBJECT p //       
;

FORM products ''
    OBJECTS p = Product
    PROPERTIES(p) READONLY name
    PROPERTIES(p) NEWSESSION NEW, EDIT, DELETE
    
    LIST Product OBJECT p //    ,       
;

NAVIGATOR {
    NEW products;
}

私たちはサプライヤーのコンセプトを作成し、編集の形で彼が使用する製品を選択する機会を与えます：
CLASS Supplier '';
name '' = DATA STRING[50] (Supplier);

in '' = DATA BOOLEAN (Supplier, Product); //  TRUE,        

FORM supplier ''
    OBJECTS s = Supplier PANEL
    PROPERTIES(s) name
    
    OBJECTS p = Product //       
    PROPERTIES in(s, p), name(p) READONLY //      
    
    EDIT Supplier OBJECT s
;

FORM suppliers ''
    OBJECTS s = Supplier 
    PROPERTIES(s) READONLY name
    PROPERTIES(s) NEWSESSION NEW, EDIT, DELETE
    
    LIST Supplier OBJECT s

NAVIGATOR {

次の行で注文ロジックを宣言します。
CLASS Order '';
date '' = DATA DATE (Order);
number '' = DATA INTEGER (Order);

supplier '' = DATA Supplier (Order);
nameSupplier '' (Order o) = name(supplier(o));
  
CLASS OrderDetail ' ';
order '' = DATA Order (OrderDetail) NONULL DELETE;

ラインに製品と数量を追加します。
product '' = DATA Product (OrderDetail);
nameProduct '' (OrderDetail d) = name(product(d));

quantity '-' = DATA NUMERIC[14,3] (OrderDetail);

必要な注文編集フォームの作成に直接進みます。まず、注文自体とその行を追加します。
FORM order ''
    OBJECTS o = Order PANEL
    PROPERTIES(o) date, number, nameSupplier
    
    OBJECTS d = OrderDetail
    PROPERTIES(d) nameProduct, quantity, NEW, DELETE
    FILTERS order(d) = o
    
    EDIT Order OBJECT o
;

したがって、追加および削除を通じて注文明細を処理するための標準インターフェースが得られます。
次に、このフォームに必要な選択機能を追加します。これを行うには、最初にすべての商品のリストを含むオブジェクトをフォーム上に作成します。ここでは、選択したサプライヤーへの注文が許可されている商品のみをフィルターします。
EXTEND FORM order
    OBJECTS p = Product
    PROPERTIES name(p) READONLY
    FILTERS in(supplier(o), p) //      
;

注文ラインと選択が1つのコンテナーのタブとして表示されるようにデザインをセットアップします。
DESIGN order {
    OBJECTS {
        NEW pane { //     
            fill = 1; //     
            type = TABBED; // , 
            MOVE BOX(d); //      
            MOVE BOX(p) { //      
                caption = '';
            }
        }
    }
}

ユーザーが対応する列に数量を表示および入力するための補助プロパティを作成します。
quantity '-' (Order o, Product p) = 
    GROUP SUM quantity(OrderDetail d) BY order(d), product(d);
lastOrderDetail ' ' (Order o, Product p) = 
    GROUP LAST OrderDetail d ORDER d BY order(d), product(d);

最初のプロパティは、このドキュメントのこの注文と製品の数量を考慮します。2番目は最後の行を検索します（ユーザーは1つの製品で複数の行を入力できます）。
次に、ユーザーが選択タブの対応する列に値を入力することを処理するアクションを作成します。
changeQuantity ' -' (Order o, Product p)  { 
    INPUT q = NUMERIC[14,3] DO { //  
        IF lastOrderDetail(o, p) THEN { // ,     
            IF q THEN //  
                quantity(OrderDetail d) <- q IF d = lastOrderDetail(o, p) 
                    WHERE order(d) = o AND product(d) = p; //        
            ELSE //   -  
                DELETE OrderDetail d WHERE order(d) = o AND product(d) == p;   
        } ELSE
            IF q THEN
                NEW d = OrderDetail { //   
                    order(d) <- o;
                    product(d) <- p;
                    quantity(d) <- q;
                }
    }
}

最後に、フォームに列を追加して、ユーザーが値を変更しようとしたときに実行する必要があるアクションを示します。
EXTEND FORM order
    PROPERTIES(o, p) quantity ON CHANGE changeQuantity(o, p)
;

注文のリストを含むフォームを描画してナビゲーターに追加するだけです。

    OBJECTS o = Order
    PROPERTIES(o) READONLY date, number

;

NAVIGATOR {
    NEW orders;
}

1C


異なる1C構成では、選択ロジックにはいくつかの違いがありますが、原理はほぼ同じです。ドキュメント編集フォームには、選択ボタンがあり、製品を選択するダイアログが表示されます



。このダイアログでは、次のように製品を選択できます



。このメカニズムには、少なくとも2つの不便があります。

最初に、製品の数量をドキュメントに追加するには（そして、ほとんどの場合、ユーザーは追加したい数量をすでに知っています）、まず製品をダブルクリックしてラインに追加し、次にラインの数量を変更する必要があります。さらに、ユーザーは、この製品がすでに追加されているかどうか、およびその数量が製品のリストに表示されません。さらに、1つの画面ではなく2つのテーブルを1つの画面に表示する必要があるため、このようなスキームでは追加のスペースが「食われます」。

次に、行がドキュメントに直接追加された後、[選択]ボタンをもう一度クリックすると、選択が「最初から」開始されます。下部のテーブルには行が表示されません。また、再選択したときに、ドキュメントにすでに何が含まれていて、何が含まれていないかは明確ではありません。また、インターフェースを構築する際の重要なルールの1つに違反します。ユーザーが誤っている場合は、戻ってエラーを修正する機会をユーザーに与える必要があります。ここで、ドキュメントへの追加ボタンを押す前と同じデータを持つ商品の選択に戻ることができないことがわかります。

ただし、1Cで上記のスキームを実装することは可能ですが、何らかの理由で、サイトでデモを使用できる一般的な構成では、別のスキームが使用されます。

Microsoft Dynamics 365


解決策として、私は小売を採用しました（その中でのみ、https：//trials.dynamics.comで入手可能な同様の機能を見つけました）。その中で、2つの方法で注文書フォームの選択を呼び出すことができます。



最初のボタンは1Cスキームに従って実装されているため（列ははるかに少ない）、そこで止まることはありません。

2番目のボタンは、数量をすでに列に入力できる製品のリストを含むダイアログを表示します。しかし、これは非常に独特な方法で行われます。

まず、デフォルトでは、製品コードのみが表示されます（名前なしでも）。もちろん、拡張機能を作成してそこに他の列を含めることができることは理解していますが、基本バージョンでこの動作を確認するのは少し奇妙です。

第二に、Microsoftはどうやら設計に夢中だったため、何らかの理由でデータのスクロール、処理のトリック、およびイベントの通常の処理に採点されました。それは次のようになります：



通常のキーボード操作中、現在のシリーズはデータを再読み取りするときに常に切り替えられます。同時に、すべてのイベント処理を待機した後でのみ入力を開始できます。そうしないと、数量がまったく入力されないか、間違ったシリーズに入力されます。実際のユーザーがこれをどのように使用するかは、私にとって謎のままです。どうやら、この機能を特に使用している人はほとんどいません。まあ、またはこれらはデモの制限ですが、動作中はすべてうまくいきます。

結論


個別の選択タブを介した説明されたドキュメント入力スキームは、ユーザーから高く評価されました。

実際には、この概念はさまざまな面で複雑になる可能性があります。たとえば、製品リストの代わりに、個々の特性に対して異なる数量を入力できる記事のリストを表示します。または、ドキュメントの1つで、行に倉庫があり、列にこれらの各倉庫の数量を設定する必要がありました。

そのようなスキームは、たとえば、同じReactで実装でき、ドキュメントの現在の行を状態として使用して、2つのコンポーネントで2つの異なるタブに表示できます。ただし、商品のテーブルには多くのエントリが存在する可能性があり、いわゆる「無限スクロール」を実装する必要があることを忘れないでください。さらに、このリスト内の製品をソートおよびフィルタリングする機能をユーザーに追加することをお勧めします（これは、余分なデータをそこにドラッグしないように、クライアントではなくサーバーで行う必要があります）。これはすべて、開発者にとってかなり重要な作業になります。

lsFusionプラットフォームでは、この機能はデフォルトで実装されており、上記のコードのみが必要です。あなたはそれがどのように機能するかを試すことができ、コードを変更したい場合は、対応するページでオンラインにすることができます。以下は、[プラットフォーム]タブに挿入して[再生]をクリックできるソースコード全体です。

name '' = DATA STRING[50] (Product);

FORM product ''

NAVIGATOR {

CLASS Order '';
date '' = DATA DATE (Order);
number '' = DATA INTEGER (Order);

supplier '' = DATA Supplier (Order);
nameSupplier '' (Order o) = name(supplier(o));
  
CLASS OrderDetail ' ';
order '' = DATA Order (OrderDetail) NONULL DELETE;

product '' = DATA Product (OrderDetail);
nameProduct '' (OrderDetail d) = name(product(d));

quantity '-' = DATA NUMERIC[14,3] (OrderDetail);

FORM order ''
    OBJECTS o = Order PANEL
    PROPERTIES(o) date, number, nameSupplier
    
    OBJECTS d = OrderDetail
    PROPERTIES(d) nameProduct, quantity, NEW, DELETE
    FILTERS order(d) = o
    
    EDIT Order OBJECT o
;

EXTEND FORM order
    OBJECTS p = Product
    PROPERTIES name(p) READONLY
    FILTERS in(supplier(o), p) //      
;

DESIGN order {
    OBJECTS {
        NEW pane { //     
            fill = 1; //     
            type = TABBED; // , 
            MOVE BOX(d); //      
            MOVE BOX(p) { //      
                caption = '';
            }
        }
    }
}

quantity '-' (Order o, Product p) = 
    GROUP SUM quantity(OrderDetail d) BY order(d), product(d);
lastOrderDetail ' ' (Order o, Product p) = 
    GROUP LAST OrderDetail d ORDER d BY order(d), product(d);

changeQuantity ' -' (Order o, Product p)  { 
    INPUT q = NUMERIC[14,3] DO { //  
        IF lastOrderDetail(o, p) THEN { // ,     
            IF q THEN //  
                quantity(OrderDetail d) <- q IF d = lastOrderDetail(o, p) 
                    WHERE order(d) = o AND product(d) = p; //        
            ELSE //   -  
                DELETE OrderDetail d WHERE order(d) = o AND product(d) == p;   
        } ELSE
            IF q THEN
                NEW d = OrderDetail { //   
                    order(d) <- o;
                    product(d) <- p;
                    quantity(d) <- q;
                }
    }
}

EXTEND FORM order
    PROPERTIES(o, p) quantity ON CHANGE changeQuantity(o, p)
;

FORM orders ''
    OBJECTS o = Order
    PROPERTIES(o) READONLY date, number
    PROPERTIES(o) NEWSESSION NEW, EDIT, DELETE
;

NAVIGATOR {
    NEW orders;
}
