Five years Rust

In this mess that is happening in the world right now, it's easy to forget that five years have passed since the release of 1.0 in 2015! Rust has changed a lot over these five years, so we would like to recall the work of all members of the community, starting from the moment the language stabilized.


Recall if someone forgot: Rust is a general-purpose programming language that has the means to build reliable and efficient software. Rust can be used in any field: from the kernel of your operating system to your next web application. This language is fully built by members of an open multifaceted community, mainly volunteers, who generously shared their time and knowledge in order to help make Rust the way it is now.


Major changes since version 1.0


2015


1.2 - Parallel Code Generation: Reducing compilation time has always been a major theme in every release of Rust, and now it‚Äôs hard to imagine that there was once a short period of time when Rust did not have parallel code generation at all.


1.3 - The Rustonomicon: our first edition of the fantastic book "The Rustonomicon", a book that explores the dark side of Rust, Unsafe Rust, and related topics. It has become an excellent source of information for anyone who wants to learn and understand the most difficult aspects of the language.


1.4 - Support for Windows MSVC level 1: moving the platform to support level 1 brought native support for 64-bit Windows with Microsoft Visual C ++ toolkit (MSVC). Prior to 1.4, you had to install MinGW (the GNU environment port for Windows) in order to use and compile your programs in Rust. Windows support has become one of Rust's biggest enhancements in five years. Only recently, Microsoft announced a public pre-release of official support for Rust WinRT API! And finally, it‚Äôs now much easier to build high-quality and cross-platform applications than it has ever been before.
