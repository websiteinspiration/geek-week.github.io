Unityアナライザーがオープンソースに

UnityのVisual Studioツール開発チームは、Unity開発者の生産性を向上させることを使命としています。Visual Studio 2019では、同僚がUnity固有の診断とコード修正のセットであるUnityツールを導入しました。本日、Unity Analyzerがオープンソースになったことをお知らせいたします。カットの下の詳細。



Unityアナライザー


Visual StudioおよびVisual Studio for Macは、当社のコンパイラインフラストラクチャであるRoslynと提携して、素晴らしいC＃プログラミングエクスペリエンスを提供します。 Roslynのお気に入りの機能の1つは、APIを使用するときにプログラムで開発者にプロンプ​​トを表示する機能です。この経験に基づいて、アナライザーはコードテンプレートを検出し、より推奨されるテンプレートに置き換えることを提案する場合があります。

Unity APIの典型的な例は、ゲームオブジェクトのタグの比較です。書いて頂けますか

collision.gameObject.tag == "enemy";

しかし、UnityはCompareTagメソッドを提供しています。これはより効率的であるため、このパターンを検出してより最適化されたメソッドの代わりに使用することを提案するCompareTag診断を実装しました。Mac用のVisual StudioでのWindows、押すだけ（。CTRL +）またはプレス（Alt-Enterキー）上でアクティブにクイックフィックスに、そしてあなたは、変更のプレビューが提供されます。



我々は現在持っているダースのパーサ Unityのためのツールであり、そして、今より多くが作成されています。

デフォルトのエクスペリエンスの改善


Roslynチームは最近アンチジャマーを導入しました。この機能により、Roslynに付属するデフォルトのアナライザーセットをプログラムで抑制することができます。

これはUnity開発者にとって素晴らしいツールであり、Unity開発に関連しないコードを修正するための警告や提案を削除することができます。

典型的な例は、Unity SerializeField属性で装飾されたフィールドです。、Unityインスペクターのフィールドを強調表示します。たとえば、Unityアナライザーがない場合、Visual Studioはシリアル化されたフィールドを読み取り専用にすることを提案しますが、Unityメカニズムがこのフィールドの値を設定することがわかっています。このコードパッチを受け入れた場合、Unityはインスペクターでこのフィールドに対して作成したリンクをすべて削除し、すべてが壊れる可能性があります。サプレッサーを作成することにより、プログラムでこの動作を抑制し、標準のC＃フィールドに対して有効にしたままにすることができます。

すでに利用可能


現在、Unity AnalyzerはTools for Unityの一部として出荷されており、Visual StudioおよびVisual Studio for Macに含まれています。アナライザーはVisual Studioで動作します。これは、警告を抑制しても、Unityエラーリストに表示されることを意味します。将来のリリースでこれを改善するために取り組んでいます。

あなたのライフハックを共有する


Tools for Unityチームには、現在取り組んでいるアナライザー、コード修正、およびサプレッサーのバックログがありますが、Unity開発者のC＃プログラミングエクスペリエンスを向上させる新しいアナライザーを常に探しています。プロジェクトは手助けが簡単です。READMEにアクセスして、新しいアナライザーを提案するか、リポジトリにリクエストを送信してください。
