PythonとBox2Dを使用した単純な空間シミュレーション

こんにちは、ハブル。

この記事は、著者がさまざまな宇宙現象の非常に興味深いシミュレーションを示したModeling the Universeの最近の発行に触発されました。しかし、そこに提示されたコードは初心者には簡単ではありません。ほんの数行のコードを記述して、Box2Dエンジンを使用して物理シミュレーションを行う方法を示します。

間違えるかもしれませんが、これはHabréでのBox2D for Pythonの最初の説明です。このギャップを埋めます。



これがどのように機能するかに興味がある人のために、詳細はカットの下にあります。

Box2Dは、Blizzard Erin Cattoによって作成された無料のクロスプラットフォームライブラリです。ライブラリは2007年に導入され、今日ではほとんどすべてのプラットフォームに移植されています。Pythonにもポートがあり、その説明はやや混乱していますが、この記事の助けを借りて、すべてがより明確になることを願っています。

前書き


pybox2d ライブラリは、Box2D自体、物理モデリング用のクロスプラットフォームライブラリ、およびフレームワークと呼ばれる個別のレンダリングモジュールの2つのコンポーネントで構成されています。作成されたオブジェクトを画面に表示するには、レンダリングが必要です。これは、シミュレーションに十分便利です。 Frameworkクラスはさまざまな出力メソッドを使用できます（詳細はこちら）。pygameを使用します。 pygameライブラリがインストールされている場合は、自動的に「選択」され、他に何もする必要はありません。インストールするには、pip install Box2D pygameコマンドを入力します。

Box2Dを使用した最小の実行プログラムを以下に示します。コードはクロスプラットフォームであり、LinuxとWindowsの両方でどこでも機能します。

from Box2D.examples.framework import Framework
from Box2D import *

class Simulation(Framework):
    def __init__(self):
        super(Simulation, self).__init__()

        # Ground body
        self.world.CreateBody(shapes=b2LoopShape(vertices=[(20, 0), (20, 40), (-20, 40), (-20, 0)]))
        # Dynamic body
        circle = b2FixtureDef(shape=b2CircleShape(radius=2), density=1, friction=1.0, restitution=0.5)
        self.world.CreateBody(type=b2_dynamicBody, position=b2Vec2(0,30), fixtures=circle, linearVelocity=(5, 0))

    def Step(self, settings):
        super(Simulation, self).Step(settings)

if __name__ == "__main__":
    Simulation().run()

ご覧のとおり、すでに言及したフレームワークを継承するシミュレーションクラスを作成しています。次に、CreateBodyメソッドを呼び出して2つのオブジェクトを作成します。 1つ目は、私たちの世界の境界を定義する静的オブジェクトです。 2番目のオブジェクトはb2_dynamicBody型で、他のパラメーター（形状、サイズ、密度、摩擦係数、初速度）はコードから明らかです。Step関数はシミュレーション中に毎回呼び出されます。これは将来使用する予定です。たとえば、UIが必要ない場合は、サーバーのバックエンドを作成します。もちろん、Frameworkクラスは省略できますが、私たちにとっては非常に便利です。

これですべてです。プログラムを実行して、完成したシミュレーションを確認します。



ご覧のとおり、2つのオブジェクトを作成し、それらのパラメーターを指定しました。重力、摩擦、弾性、物体の相互作用など、すべてが「箱から出して」機能します。これに基づいて、「空間」シミュレーションを進めることができます。

「衛星」を打ち上げ


残念ながら、Box2Dにはニュートン重力のサポートが組み込まれていないため、Step関数を追加して自分で追加する必要があります。最初のテストでは、惑星とその周りを回転する衛星の2つの物体を作成します。

全体としてのソースコード：

from Box2D import *
from Box2D.examples.framework import Framework


class Simulation(Framework):
    def __init__(self):
        super(Simulation, self).__init__()

        # Default gravity disable
        self.world.gravity = (0.0, 0.0)
        # Gravity constant
        self.G = 100

        # Planet
        circle = b2FixtureDef(shape=b2CircleShape(radius=5), density=1, friction=0.5, restitution=0.5)
        self.world.CreateBody(type=b2_dynamicBody, position=b2Vec2(0,0), fixtures=circle)

        # Satellite
        circle_small = b2FixtureDef(shape=b2CircleShape(radius=0.2), density=1, friction=0.5, restitution=0.2)
        self.world.CreateBody(type=b2_dynamicBody, position=b2Vec2(0, 10), fixtures=circle_small, linearVelocity=(20, 0))

    def Step(self, settings):
        super(Simulation, self).Step(settings)

        # Simulate the Newton's gravity
        for bi in self.world.bodies:
            for bk in self.world.bodies:
                if bi == bk:
                    continue

                pi, pk = bi.worldCenter, bk.worldCenter
                mi, mk = bi.mass, bk.mass
                delta = pk - pi
                r = delta.length
                if abs(r) < 1.0:
                    r = 1.0

                force = self.G * mi * mk / (r * r)
                delta.Normalize()
                bi.ApplyForce(force * delta, pi, True)

if __name__ == "__main__":
    Simulation().run()

ご覧のとおり、self.world.gravityパラメーターを0に設定して標準重力を「オフ」にします。また、パラメーターGを追加します。これは、Stepメソッドの計算で使用される仮想世界の「重力定数」です。また、衛星と惑星という2つのオブジェクトも作成しました。密度と半径のパラメータに注意することが重要です。これらのパラメータに従って、Box2Dライブラリ自体が計算に使用される質量を計算します。相互作用力を計算するために、ニュートンの重力の法則の通常の「学校」の公式が使用されます。



ここでシミュレーションを開始します。最初の宇宙速度には達していません。衛星はまだ惑星全体を周回していますが、「飛行」と呼ぶのは困難



です。コードの行をlinearVelocity =（28、0）に変更して速度を上げます。



私たちの「衛星」は「惑星」の周りの軌道に首尾よく進入しました！さらに速度を上げると、軌道は楕円形になります。



最後に、「太陽系」に似たものを描き、異なる軌道に異なるサイズの3つの惑星を追加します。

circle_small = b2FixtureDef(shape=b2CircleShape(radius=0.2), density=1, friction=0.5, restitution=0.2)
circle_medium = b2FixtureDef(shape=b2CircleShape(radius=0.3), density=1, friction=1.0, restitution=0.5)
self.world.CreateBody(type=b2_dynamicBody, position=b2Vec2(0, 6), fixtures=circle_small, linearVelocity=(37, 0))
self.world.CreateBody(type=b2_dynamicBody, position=b2Vec2(0, 10), fixtures=circle_small, linearVelocity=(28, 0))
self.world.CreateBody(type=b2_dynamicBody, position=b2Vec2(0, 15), fixtures=circle_medium, linearVelocity=(22, 0))

惑星が「太陽」から離れているほど、革命の期間が長いことがわかります（ケプラーの第3法則）。残念ながら、Box2Dエンジンは画面上にモーショントラックを描画することを許可していないため、ケプラーの第1法則と第2法則を「見る」ことは困難ですが、それらも実装されていることを確認できます。

結論


ご覧のとおり、Box2Dを使用すると、最小限の労力で簡単なシミュレーションを実行できます。もちろん、このエンジンはまだ科学的なエンジンではなくゲームエンジンであるため、ビッグバン中の銀河の衝突や物質の膨張の正しいシミュレーションを期待するべきではありません。しかし、いくつかのパターンは非常に興味深いものです。
