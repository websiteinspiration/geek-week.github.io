ただし、実際にはカメラはファイル内のピクセルデータを回転させません。デジタルカメラのイメージセンサーは、ピクセル情報の連続ストリームとしてラインごとに読み取られるため、電話の実際の位置に関係なく、カメラは常に同じ順序でピクセルデータを保存する方が簡単です。

EXIFメタデータは、もともとJPEG形式の一部ではありませんでした。それらはずっと後に導入され、TIFFフォーマットからアイデアを借りました。下位互換性のために、このメタデータはオプションであり、一部のプログラムはそれを解析する必要がありません。

numpy、scipy、TensorFlow、KerasなどのほとんどのPython画像ライブラリは、共有データセットを扱う真面目な人のための科学的ツールと見なされています。自動画像回転などの消費者レベルの問題は気にしませんが、これは、現代のカメラで撮った世界のほとんどすべての写真に必要です。

つまり、ほとんどすべてのPythonライブラリで画像を処理すると、回転せずに元の画像データが得られます。また、顔または物体検出モデルで写真を自分の側または逆さまにアップロードしようとするとどうなるでしょうか？あなたがそれに悪いデータを与えたので、検出器は作動しません。

問題は初心者と学生のプログラムでのみ発生すると思うかもしれませんが、これはそうではありません！Googleの主力製品であるVision APIのデモバージョンでも、EXIFの向きが正しく処理されません。GoogleVision

ことがはっきりとわかる場合、これは非常に明白な問題です。このデモのように。しかし、すべてが陰湿になるのはここです-通常、あなたはそれを見ません！コンピュータ上のすべての通常のプログラムは、正しい方向で画像を表示しますが、実際にディスクに保存されている方法では表示しません。したがって、モデルが機能していない理由を確認するために画像を表示しようとすると、正しく表示され、モデルが機能していない理由がわかりません。

補正


解決策は、画像をPythonプログラムに読み込むたびに、EXIFオリエンテーションのメタデータを確認し、必要に応じて画像を回転させる必要があることです。それはかなり簡単ですが、インターネット上では、すべての方向で正しく機能するコード例を見つけるのは驚くほど難しいです。

以下は、画像を正しい向きでnumpy配列に読み込むコードです。

import PIL.Image
import PIL.ImageOps
import numpy as np


def exif_transpose(img):
    if not img:
        return img

    exif_orientation_tag = 274

    # Check for EXIF data (only present on some files)
    if hasattr(img, "_getexif") and isinstance(img._getexif(), dict) and exif_orientation_tag in img._getexif():
        exif_data = img._getexif()
        orientation = exif_data[exif_orientation_tag]

        # Handle EXIF Orientation
        if orientation == 1:
            # Normal image - nothing to do!
            pass
        elif orientation == 2:
            # Mirrored left to right
            img = img.transpose(PIL.Image.FLIP_LEFT_RIGHT)
        elif orientation == 3:
            # Rotated 180 degrees
            img = img.rotate(180)
        elif orientation == 4:
            # Mirrored top to bottom
            img = img.rotate(180).transpose(PIL.Image.FLIP_LEFT_RIGHT)
        elif orientation == 5:
            # Mirrored along top-left diagonal
            img = img.rotate(-90, expand=True).transpose(PIL.Image.FLIP_LEFT_RIGHT)
        elif orientation == 6:
            # Rotated 90 degrees
            img = img.rotate(-90, expand=True)
        elif orientation == 7:
            # Mirrored along top-right diagonal
            img = img.rotate(90, expand=True).transpose(PIL.Image.FLIP_LEFT_RIGHT)
        elif orientation == 8:
            # Rotated 270 degrees
            img = img.rotate(90, expand=True)

    return img


def load_image_file(file, mode='RGB'):
    # Load the image with PIL
    img = PIL.Image.open(file)

    if hasattr(PIL.ImageOps, 'exif_transpose'):
        # Very recent versions of PIL can do exit transpose internally
        img = PIL.ImageOps.exif_transpose(img)
    else:
        # Otherwise, do the exif transpose ourselves
        img = exif_transpose(img)

    img = img.convert(mode)

    return np.array(img)

ここから、画像データの配列を、入力で配列が必要な標準のPythonマシンビジョンライブラリに渡すことができます（例：KerasやTensorFlow）。

問題は至る所にあるため、この関数をimage_to_numpyというpipライブラリとして公開しました。次のようにインストールできます。

pip3インストールimage_to_numpy

これは、任意のPythonプログラムで動作し、画像の読み込みを修正します。次に例を示します。

import matplotlib.pyplot as plt
import image_to_numpy
# Load your image file
img = image_to_numpy.load_image_file("my_file.jpg")
# Show it on the screen (or whatever you want to do)
plt.imshow(img)
plt.show()
