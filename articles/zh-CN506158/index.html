useSWR是我最喜欢的React库

在“ React.js Developer”课程开始之前，准备了本文的翻译。


最近几个月，我一直在NextJS上开发应用程序。每星期变得越来越多。该应用程序使用axios进行API调用，并使用unstated -next进行状态管理。有很多API调用，但是我们不希望用户看到很多加载屏幕。因此，我们将axios调用的结果存储在未声明的存储库中。

但是，我们遇到了一个问题。保管库本身变得越来越复杂。页面有时需要多个API调用，而页面又依赖于其他API调用的结果。几天变成了几周，几周变成了几个月，与此同时，我们未声明的文件库变得越来越麻烦。当我们的自制缓存逻辑努力处理意外的极端情况时，我们遇到了奇怪的错误。

然后我们认为路径应该更简单。
他在这里。它甚至由NextJS的创建者Vercel开发。

了解SWR


SWR的名称来自stale-while-revalidate，这是一种缓存方法，现已在前端开发中流行。它允许您立即下载缓存的内容，并立即对其进行更新以在将来处理新内容。在我们的案例中，我们在性能和用户体验之间取得了完美的折衷。

如何使用useSWR


useSWR是由Vercel开发的React钩子库。它允许您从API或其他外部源检索数据，将其保存在缓存中，然后进行呈现。

让我们先来看一个React组件的示例，该组件从JSON服务器接收TODO列表并进行渲染。

import React from "react";
import "./App.css";
const todosEndpoint = "http://localhost:3001/todos";
const TodoApp = () => {
  const [todos, setTodos] = React.useState([]);
  React.useEffect(() => {
    const getData = async () => {
      const response = await fetch(todosEndpoint);
      const data = await response.json();
      setTodos(data);
    };
     getData();
    }, []);
  return (
    <div className="App">
      {todos.map((todo) => (
        <div key={todo.id}>{todo.title}</div>
      ))}
    </div>
  );
};
export default TodoApp;

现在让我们看一下相同的组件，但是使用useSWR重写了它。

import React from "react";
import useSWR from "swr";
const todosEndpoint = "http://localhost:3001/todos";
const getData = async () => {
  const response = await fetch(todosEndpoint);
  return await response.json();
};
const TodoApp = () => {
  const { data: todos } = useSWR(todosEndpoint, getData);
  
return (
    <div>
      {todos && todos.map(todo => (
        <div key={todo.id}>{todo.title}</div>
      )}
    </div>
  );
};
export default TodoApp;

如您所见，它与先前的实现非常相似。

在此示例中，我们使用useSWR（键，访存器，选项）获取我们的TODO列表。缓存中需要useSWR中的密钥。在我们的例子中，我们使用了todosEndpoint。作为访存程序，我们传递了一个异步函数来检索TODO列表。

应当指出，useSWR不知道如何检索数据。您可以使用任何您喜欢的异步数据挖掘策略。您可以使用fetch，axios或GraphQL。当您的函数异步返回数据时，useSWR会很高兴。useSWR

调用返回以下参数：

  • data：使用提取程序获取的关键数据（如果尚未加载，则未定义）；
  • 错误：由提取程序引起的错误；
  • isValidating：一个布尔值，表示是否正在进行请求或重新验证；
  • mutate（data？，shouldRevalidate）：修改缓存数据的功能。

更复杂的例子


让我们看一个更复杂的示例，该示例使用useSWR返回的更多参数。在此示例中，我们将检索TODO列表中的一个元素，并从列表中呈现任务完整状态的复选框。单击复选框后，我们发送一个PUT请求来更新TODO，然后调用mutate来更新useSWR缓存。

import React from "react";
import useSWR from "swr";
import "./App.css";
const todosEndpoint = "http://localhost:3001/todos";
const getTodo = async (id) => {
  const response = await fetch(`${todosEndpoint}/${id}`);
  return await response.json();
};
const updateTodo = async (id, todo) => {
  const response = await fetch(`${todosEndpoint}/${id}`, {
    method: "PUT",
    headers: {
      "Content-type": "application/json; charset=UTF-8",
    },
    body: JSON.stringify(todo),
  });
  return await response.json();
};
const TodoApp = () => {
  const todoId = 1;
  const key = `${todosEndpoint}/${todoId}`;
  const { data: todo, mutate } = useSWR(key, () =>
    getTodo(todoId)
  );
  const toggleCompleted = async () => {
    const newTodo = {
      ...todo,
      completed: !todo.completed,
    };
    await updateTodo(todoId, newTodo);
    mutate(newTodo);
  };
  if (!todo) {
    return <div>Loading...</div>;
  }
  return (
    <div>
      <p>{todo.title}</p>
      <div>
        <input
          type="checkbox"
          id="completed"
          name="completed"
          value="completed"
          checked={todo.completed}
          onChange={toggleCompleted}
        />
        <label htmlFor="completed">Completed</label>
      </div>
    </div>
  );
};

使用mutate是提高Web应用程序可见性能的好方法。我们可以在本地更改数据并更改视图，而不必等到远程数据源更新后。useSWR甚至会在后台重新验证并用最新数据替换它们。

为什么我需要一个图书馆？


您可能会问这个问题。也许您的应用程序中已经具有状态管理机制，但看不到使用第三方库处理诸如数据提取和缓存之类的简单操作的意义。

我会回答你，缓存数据并不容易。随着应用程序的增长，它变得越来越难。您将遇到极端情况，并进行处理，您将获得难以理解和维护的复杂存储库和提供程序。错误会无处不在。
“在计算机科学中，只有两个复杂的问题：缓存无效和创建名称”-Phil Carleton

为什么不依靠自己的数据挖掘解决方案，为什么不依靠React生态系统中最受尊敬的公司之一开发的，经过验证的呢？

useSWR如何帮助我们


在我们的应用程序中将数据提取问题移植到useSWR具有许多优点。

1.他允许我们删除代码

Pull-requests删除不必要的代码，但不添加新代码-我的最爱。应用程序中的代码越少，发生错误的可能性就越小。这很好。随着年龄的增长，我开始越来越喜欢简单。useSWR允许我们完全删除未声明的存储，这简化了我们的应用程序，并使其更易于理解。

2.他简化了新开发人员与useSWR 项目的联系

像大多数开源项目一样，它拥有出色的文档。实施我们自己的解决方案意味着我们需要编写我们自己的文档，并教会项目中的新开发人员如何处理数据提取。现在我们使用了useSWR，我们不需要这样做。

3.他简化了棘手的事情，

我们的应用程序与大多数其他应用程序一样，包含许多API调用。其中一些调用依赖于其他API调用。使用useSWR可以轻松为依赖查询编写钩子。

4.提高了可见性能，

应用程序看起来更快。用户对其进行了评分，并给出了良好的反馈。

5.更新过时的数据

useSWR集中更新过时的数据。这意味着用户始终拥有最新版本的数据，不包括下载时间。

结论


useSWR对我们的应用程序影响很大。该库简化了我们的代码并改善了用户体验。我什至无法想象另一个库会像useSWR一样容易实现并具有许多优点。要了解有关useSWR的更多信息，可以访问GitHub上的站点或存储库。

替代库
