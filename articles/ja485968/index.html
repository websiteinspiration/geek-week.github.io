そして再びロックをバイパスします。RouterOS + BGP + OSPF

良い一日！

新しいことは思いつきませんでした。いくつかの記事を組み合わせて、ロックの便利なバイパスを設定しました。

記事「ロックをバイパスするようにBGPを構成する、バージョン3、VPSなし」に基づいています。

私の「変更」の利点-複数のユーザー/ Mikrotikは同時に1つのVPSを使用できます。

短所-ライセンスを購入するには、いくらかの現金支出が必要です。もちろん、自由に使用できるオプションがありますが、これはあなたの良心とあなた自身の問題です。

自分用のライセンスを購入しました。

私は指ですべてを語ることはしません。 VPSでCHRを構成することに関心がある場合は、IPとゲートウェイを構成し、ループバックブリッジを自分で作成できます。
（aからzへの完全な構成を追加するために投票を追加しました）。

まず、財政的にも地理的にも便利なVPSを選択する必要があります。
VPSが選択され、支払われ、Mikrotik CHRがインストールされます。

RouterOSをインストールし、IPとゲートウェイを構成したので、構成を開始します。

L2TPの構成については説明しません。ハブとGoogleには、MikrotikでのL2TPサーバーのセットアップに関する多くの指示があります。

たとえば、Mikrotik CHR IP 10.100.1.1サーバーでは

、自宅にL2TPクライアントを構成します。
自宅では、ネットワーク10.10.1.0/24としましょう
他のマイクロティクスでは、ホームネットワークは異なるはずです。たとえば、2.0 / 24、3.0 / 24などです。

次に、OSPFを構成する必要があります。
ループバックインターフェイスを作成して、IPプレフィックス32でそれらをハングアップしましょう。たとえば、サーバー上の10.255.255.1/32です。自宅では10.255.255.2/32。等。残りの10.255.255.3-x / 32

サーバー上：

/routing ospf instance
set [ find default=yes ] router-id=10.255.255.1
/routing ospf network
add area=backbone network=10.100.1.0/24

クライアント上：

/routing ospf instance
set [ find default=yes ] router-id=10.255.255.2
/routing ospf network
add area=backbone network=10.10.1.0/24
add area=backbone network=10.100.1.1/32

すべてが正しく行われると、DAoマークの付いたOSPFルートが実行されます。

次に、クライアントMikrotikの1つでVPSを介してすべてのトラフィックを一時的に許可する必要があります。
プロバイダーのゲートウェイを介してVPSへの静的ルートを作成し、デフォルトルートを削除して独自のデフォルトルートを作成します。ゲートウェイとしてのVPS IPアドレスはL2TPです。

次に、記事「BGPの構成」（2-VPSのネットワークから）で述べたように、2つ実行しますが、3つは少し異なります。

ルートを作成する最初のコマンドとフィルターを追加する最後のコマンドを除外します。

/routing bgp instance set default as=<b>64512</b> ignore-as-path-len=yes router-id=<b>81.117.103.94</b>
/routing bgp peer add hold-time=4m in-filter=bgp_in keepalive-time=1m multihop=yes name=antifilter remote-address=163.172.210.8 remote-as=65432 ttl=default

自律システム番号はあなたのものであり、IPはあなたのVPSの静的アドレスです。

VPSでは、プロセッサはホームルーターよりも強力で、メモリはまともです。ルートは目立ったブレーキなしで素早く到着します。

BGPルートが実行されていることを確認し、ホームMikrotikのデフォルトルートを返します。

Mikrotik CHRに戻ります。

/routing ospf instance
set [ find default=yes ] redistribute-bgp=as-type-1 router-id=10.255.255.1

このコマンドは、BGPを介して受信したルートを強制的にOSPFにアドバタイズします。ルートの転送中に、空きメモリの量が32 MB減少しました。

PS私は主に他の目的のためにVPSを使用しました-ネットワーキング。ロックバイパスは素晴らしい追加です。

上記のスキームは、BGPを構成するために2台以上のルーターよりも優れています。
