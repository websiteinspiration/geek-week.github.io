GolangとgRPCでシンプルなマイクロサービスを作成し、Dockerを使用してコンテナー化する方法

こんにちは、ハブル！Mat Evansによる記事「Go、gRPC、Docker」の翻訳を紹介します。

GoとDockerの共有に関する記事はたくさんあります。顧客と相互にやり取りできるコンテナの作成は非常に簡単です。以下は、これが基本レベルでどのように行われるかの小さな例です。

何を作りますか？


非常にシンプルなクライアントとサーバーを作成し、gRPCを使用して相互に対話します。サーバーはDockerコンテナー内に配置されるため、簡単にデプロイできます。

クライアントから文字列を受け取り、逆の文字順序で文字列を返すサービスが必要だとします。たとえば、「猫」を送信し、「最新」の応答を取得します。

.protoファイル


.proto-ファイルには、サービスが実行する操作と、サービスが交換するデータが記述されています。プロジェクトにprotoフォルダーを作成し、その中にreverse.protoファイルを作成します。

syntax = "proto3";

package reverse;

service Reverse {
    rpc Do(Request) returns (Response) {}
}

message Request {
    string message = 1;
}

message Response {
    string message = 1;
}

サーバー上でリモートで呼び出され、データをクライアントに返す関数は、rpcとしてマークされます。相互作用するノード間で情報を交換するために使用されるデータ構造は、メッセージとしてマークされます。各メッセージフィールドにはシーケンス番号を割り当てる必要があります。この場合、関数はクライアントから要求メッセージを受け取り、応答メッセージを返します。.protoファイル
を作成したら、サービスの.goファイルを取得する必要があります。これを行うには、protoフォルダで次のコンソールコマンドを実行します。

$ protoc -I . reverse.proto --go_out=plugins=grpc:.

もちろん、最初にgRPCをビルドする必要があります。
上記のコマンドを実行すると、クライアント、サーバー、およびそれらが交換するメッセージを作成するためのメソッドを含む新しい.goファイルが作成されます。godocを呼び出すと、次のようになります。

$ godoc .

お客様


私たちのクライアントが次のように機能するといいですね：

reverse "this is a test"
tset a si siht

次に、.protoファイルから生成されたデータ構造を使用してgRPCクライアントを作成するコードを示します。

package main

import (
    "context"
    "fmt"
    "os"    
     pb "github.com/matzhouse/go-grpc/proto"
    "google.golang.org/grpc"
    "google.golang.org/grpc/grpclog"
)

func main() {
    opts := []grpc.DialOption{
        grpc.WithInsecure(),
    }
    args := os.Args
    conn, err := grpc.Dial("127.0.0.1:5300", opts...)

    if err != nil {
        grpclog.Fatalf("fail to dial: %v", err)
    }

    defer conn.Close()

    client := pb.NewReverseClient(conn)
    request := &pb.Request{
        Message: args[1],
    }
    response, err := client.Do(context.Background(), request)

    if err != nil {
        grpclog.Fatalf("fail to dial: %v", err)
    }

   fmt.Println(response.Message)
}


サーバ


サーバーは同じ生成された.goファイルを使用します。ただし、これはサーバーインターフェイスを定義するだけですが、ロジックを独自に実装する必要があります。

package main

import (
    "net"
    pb "github.com/matzhouse/go-grpc/proto"
    "golang.org/x/net/context"
    "google.golang.org/grpc"
    "google.golang.org/grpc/grpclog"
)

func main() {
    listener, err := net.Listen("tcp", ":5300")

    if err != nil {
        grpclog.Fatalf("failed to listen: %v", err)
    }

    opts := []grpc.ServerOption{}
    grpcServer := grpc.NewServer(opts...)

    pb.RegisterReverseServer(grpcServer, &server{})
    grpcServer.Serve(listener)
}

type server struct{}

func (s *server) Do(c context.Context, request *pb.Request)
(response *pb.Response, err error) {
    n := 0
    // reate an array of runes to safely reverse a string.
    rune := make([]rune, len(request.Message))

    for _, r := range request.Message {
        rune[n] = r
        n++
    }

    // Reverse using runes.
    rune = rune[0:n]

    for i := 0; i < n/2; i++ {
        rune[i], rune[n-1-i] = rune[n-1-i], rune[i]
    }

    output := string(rune)
    response = &pb.Response{
        Message: output,
    }

    return response, nil    
}


Docker


Dockerとは何か、その目的は 知っていると思います。Dockerfileは次のとおりです。

FROM golang:1.12

ADD . /go/src/github.com/matzhouse/go-grpc/server

RUN go install github.com/matzhouse/go-grpc/server

ENTRYPOINT ["/go/bin/server"]

EXPOSE 5300

Dockerイメージ のアセンブリコードはここに記述されています。行ごとに分析します。

FROM golang:1.12

このコマンドは、以前に作成したイメージ、つまりgolangに基づいてアプリケーションのイメージを作成することを意味します。これは、Goで記述されたプログラムをビルドして実行するための環境がすでに構成されているDockerイメージです。

ADD . /go/src/github.com/matzhouse/go-grpc/server

このコマンドは、アプリケーションのソースコードをGOPATH / srcコンテナーにコピーします。

RUN go install github.com/matzhouse/go-grpc/server

このコマンドは、コンテナーにコピーされたソースからアプリケーションを収集し、GOPATH / binコンテナーフォルダーにインストールします。

ENTRYPOINT ["/go/bin/server"]

このコマンドは、実行可能プログラムとして機能するようにコンテナーを構成します。その中で、アプリケーション実行可能ファイルへのパスと、必要に応じてコマンドライン引数を示します。

EXPOSE 5300

このコマンドを使用して、外部からアクセスできるポートをコンテナーに指示します。

クライアントの立ち上げ


クライアントのコンテナ化には大きなメリットはないので、通常の方法で開始しましょう。

$ go build -o reverse
$ ./reverse "this is a test"
