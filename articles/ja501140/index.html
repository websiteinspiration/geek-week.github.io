iOSでのサーバーデータパーサーのロイヤルティの向上

モバイルアプリケーションを開発する場合、何らかの方法でサーバーデータを内部アプリケーションモデルに解析する必要があります。ほとんどの場合、このデータはJSON形式です。Swift 4以降、JSONを解析するための主なツールは、プロトコルDecodableとオブジェクトを使用することですJSONDecoder。


このアプローチにより、データの解析プロセスが大幅に簡略化され、ボイラープレートコードの数が削減されました。ほとんどの場合、JSONオブジェクトのフィールドと同じように名前が付けられたプロパティを使用してモデルを作成するだけで十分でJSONDecoderあり、残りは自動的に行われます。最小のコード、最大の利益。ただし、このアプローチには1つの欠点があります。それは、パーサーの忠誠度が非常に低いことです。説明します。内部データモデル（Decodableオブジェクト）とJSONに含まれるものの間に不一致がある場合、JSONDecoderエラーをスローし、オブジェクト全体を失います。おそらく、状況によっては、この振る舞いのモデルは、特にそれが例えば金融取引に関しては望ましいです。しかし、多くの場合、解析プロセスをより忠実にすることが有用です。この記事では、私の経験を共有し、この同じ忠誠心を高めるための主な方法についてお話したいと思います。

struct FailableDecodable<Value: Decodable>: Decodable {

    var wrappedValue: Value?

    init(from decoder: Decoder) throws {
        let container = try decoder.singleValueContainer()
        wrappedValue = try? container.decode(Value.self)
    }
}

struct FailableDecodableArray<Value: Decodable>: Decodable {

    var wrappedValue: [Value]

    init(from decoder: Decoder) throws {
        var container = try decoder.unkeyedContainer()
        var elements: [Value] = []
        while !container.isAtEnd {
            if let element = try? container.decode(Value.self) {
                elements.append(element)
            }
        }
        wrappedValue = elements
    }
}

struct Convertible<Value: Decodable & LosslessStringConvertible>: Decodable {
    var wrappedValue: Value
    init(from decoder: Decoder) throws {
        let container = try decoder.singleValueContainer()
        guard let stringValue = try? container.decode(String.self),
            let value = Value(stringValue) else {
                wrappedValue = try container.decode(Value.self)
                return
        }
        wrappedValue = value
    }
}

struct Product: Decodable {
    let id: Int
    let title: String
    let price: Convertible<Double>
    let image: FailableDecodable<Image>?

struct StringConvertible: Decodable {
    var wrappedValue: String
    init(from decoder: Decoder) throws {
        let container = try decoder.singleValueContainer()
        guard let number = try? container.decode(Double.self) else {
            wrappedValue = try container.decode(String.self)
            return
        }
        wrappedValue = "\(number)"
    }
}

上記のすべてを要約するDecodableと、クラスと組み合わせたプロトコルの出現により、JSONDecoderサーバーデータの解析に関連する作業が大幅に簡略化されたことに注意したいと思います。ただし、JSONDecoder現時点では忠誠度が非常に低く、それを改善するには（必要な場合）、少し作業していくつかのラッパーを作成する必要があることに注意してください。JSONDecoder比較的最近になって、キーをスネークケースからキャメルケースに、文字列を日付に変換する方法さえ知らなかったため、これらすべての機能がオブジェクト自体で実現されると思います。これはすべて「そのまま」使用できるようになりました。
