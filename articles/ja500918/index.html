A / B検定の統計的有意性を計算するときに注意すること

Uchi.ruでは、A / Bテストで小さな改善を展開しようとしていますが、今年度は250以上の改善がありました。A/ Bテストは、インターネット製品の通常の開発を想像するのが難しい強力な変更テストツールです。同時に、見かけ上の単純さにもかかわらず、実験の設計段階と結果の集計の両方で、A / Bテスト中に重大なエラーが発生する可能性があります。この記事では、テストの技術的な側面について説明します。テスト期間を決定する方法、要約する方法、テストが予定より早く完了した場合、および複数の仮説を一度にテストする場合に、誤った結果を回避する方法です。


私たち（および多くの人）の典型的なA / Bテストスキームは次のようになります。

  1. 私たちは機能を開発していますが、それをオーディエンス全体に公開する前に、エンゲージメントなどのターゲット指標が改善されることを確認したいと思います。
  2. テストを開始する期間を決定します。
  3. ユーザーをランダムに2つのグループに分けます。
  4. 1つのグループには機能を備えた製品バージョン（実験グループ）、もう1つのグループには古いバージョン（コントロール）を示します。
  5. このプロセスでは、特に失敗したテストを時間内に停止するために、メトリックを監視します。
  6. テストの有効期限が切れたら、実験グループと対照グループのメトリックを比較します。
  7. 実験グループの指標が対照グループよりも統計的に有意に優れている場合は、テストした機能をまったく公開します。統計的有意性がない場合は、否定的な結果でテストを終了します。

細部において、すべては論理的でシンプルに見えます。悪魔はいつものように。

統計的有意性、基準、エラー


A / Bテストにはランダム性の要素があります。グループメトリックは、その機能だけでなく、ユーザーが何にアクセスしたか、またどのように動作するかに依存します。グループの優位性に関する結論を確実に引き出すには、テストで十分な観察結果を収集する必要がありますが、それでも間違いを免れることはできません。それらは2つのタイプによって区別されます：

  • グループ間の差異を修正すると、第1種のミスが発生しますが、実際には存在しません。テキストには同等の用語も含まれます-偽陽性の結果。記事はそのようなエラーに専念しています。
  • 違いの欠如を修正すると、実際にはそうですが、第2種の間違いが発生します。

多数の実験では、第1種のエラーの確率が小さいことが重要です。統計的手法を使用して制御できます。たとえば、各実験で第1種のエラーが発生する確率が5％を超えないようにする必要があります（これは便利な値であり、必要に応じて別の値を使用できます）。次に、0.05の有意水準で実験を行います。

  1. コントロールグループAと実験グループBを使用したA / Bテストがあります。目標は、グループBがグループAといくつかのメトリックで異なることを確認することです。
  2. 帰無統計仮説を立てます。グループAとグループBは違いません。観測された違いはノイズによって説明されます。デフォルトでは、反対が証明されるまで、違いはないと常に考えます。
  3. 厳密な数学的規則-統計的基準、たとえば学生の基準-で仮説をチェックします。
  4. その結果、p値が得られます。これは0〜1の範囲にあり、帰無仮説が真である場合、つまりグループ間に差異がない場合に、グループ間の現在またはそれ以上の極端な差異が見られる確率を意味します。
  5. p値は有意水準0.05と比較されます。大きい場合は、差がないという帰無仮説を受け入れます。それ以外の場合は、グループ間に統計的に有意な差があると考えます。

仮説は、パラメトリックまたはノンパラメトリック基準でテストできます。パラメトリック変数は、確率変数の標本分布のパラメーターに依存し、より強力です（それらは、第2種の誤りを少なくします）が、調査中の確率変数の分布に要件を課します。

最も一般的なパラメトリック検定はスチューデントの検定です。 2つの独立したサンプル（A / Bテストケース）の場合、ウェルチ基準と呼ばれることがあります。この基準は、調査された量が正常に分布している場合に正しく機能します。実際のデータではこの要件はほとんど満たされていないと思われるかもしれませんが、実際には、テストではサンプル自体ではなく、サンプル平均の正規分布が必要です。実際には、これは、テストに多くの観測値（数十から数百）があり、分布に非常に長い裾がない場合に、基準が適用できることを意味します。最初の観測の分布の性質は重要ではありません。読者は、ベルヌーイまたは指数分布から生成されたサンプルでも、スチューデント基準が正しく機能することを個別に確認できます。

ノンパラメトリック基準の中で、マンホイットニー基準が一般的です。サンプルが非常に小さいか、異常値が大きい場合に使用する必要があります（この方法では中央値を比較するため、異常値に耐性があります）。また、基準が正しく機能するためには、サンプルに一致する値がほとんどない必要があります。実際には、ノンパラメトリック基準を適用する必要はありませんでした。テストでは常に生徒基準を使用します。

多重仮説検定の問題


最も明白で最も単純な問題：テストで、コントロールグループに加えていくつかの実験的な問題がある場合、結果を有意水準0.05で合計すると、第1種のエラーの割合が複数増加します。これは、統計的基準を適用するたびに、第1種のエラーの確率が5％になるためです。グループ数でngroups と有意水準 α ある実験グループが偶然勝つ確率は次のとおりです。

P(any false positive)=1−(1−α)ngroups


たとえば、3つの実験グループでは、予想される5％ではなく14.3％が得られます。この問題は、複数の仮説検定に対するBonferroniの修正によって解決されます。重要度レベルを比較（つまり、グループ）の数で除算して操作するだけです。上記の例では、修正を考慮した有意水準は0.05 / 3 = 0.0167であり、第1種のエラーが少なくとも1つ発生する確率は4.9％です。

厳密に言えば、聴衆の異なるメトリックまたはセクションによるグループの比較も、複数のテストの問題の影響を受けます。正式には、すべてのチェックを考慮することは非常に困難です。これは、その数を事前に予測することが難しく、ときには独立していない場合があるためです（特に、スライスではなく、異なるメトリックに関して）。普遍的なレシピはありません。常識に依存し、さまざまなメトリックを使用して多数のスライスをチェックすると、どのテストでも統計的に有意な結果が得られることを覚えておいてください。したがって、たとえば、大都市の新規モバイルユーザーの5日目の定着率が大幅に増加することに注意する必要があります。

のぞき問題


複数の仮説検定の特定のケースは、ピーク問題です。重要なのは、検定中のp値が誤って許容有意水準を下回ることがあるということです。実験を注意深く監視すると、この瞬間を捉えて、統計的有意性を間違える可能性があります。

投稿の冒頭で説明したテストセットアップから離れ、毎日5％の有意水準（またはテスト中に2回以上）で在庫を取ることを決定したとします。要約すると、p値が0.05未満の場合、検定は陽性であり、それ以外の場合は、その継続性がわかります。この戦略では、偽陽性の結果の割合はチェックの数に比例し、最初の1か月で28％に達します。このような大きな違いは直観に反するように思われるため、A / Aテストの方法論に目を向けます。これは、A / Bテストスキームの開発に不可欠です。

初期の試験方法


途中でテストを受けることができるテストオプションがあります。そのうちの2つについてお話しします。重要度は一定（Pocock修正）と、覗き見の数に依存（O'Brien-Fleming修正）です。厳密に言えば、両方の修正について、最大テスト期間とテストの開始から終了までのチェック数を事前に知っておく必要があります。さらに、チェックはほぼ等間隔で（または等量の観測で）行われる必要があります。

ポコック


この方法では、テストの結果を毎日要約しますが、有意性のレベルは（より厳しく）低くなります。たとえば、30回以下のチェックを実行することがわかっている場合、有意水準は0.006に設定する必要があります（モンテカルロ法を使用したピープの数に応じて、つまり経験的に選択されます）。私たちのシミュレーションでは、4％の偽陽性結果が得られます-明らかに、しきい値を増やすことができます。


明らかな素朴さにもかかわらず、一部の大企業はこの特定の方法を使用しています。機密性の高い指標と大量のトラフィックを決定する場合、それは非常にシンプルで信頼性の高い方法です。たとえば、Avitoのデフォルトでは、有意水準は0.005に設定されています。

オブライエンフレミング


この方法では、検証番号によって有意水準が異なります。テストのステップ（またはピープ）の数を事前に決定し、各ステップの有意水準を計算する必要があります。テストの完了を早めにすればするほど、厳しい基準が適用されます。学生統計のしきい値Zstep （最後のステップの値を含む Zlast_step）目的の有意水準に対応することは、検証番号に依存します step_number （1からチェックの総数までの値を取ります total_steps を含みます）そして、経験的に得られた式に従って計算されます：

A / Bテスト電卓


当社の製品の詳細は、どの指標の分布も、テストの対象者（クラス番号など）と時期によって大きく異なることです。したがって、「各グループで100万人のユーザーが入力されたときにテストが終了する」または「解決されたタスクの数が1億個に達したときにテストが終了する」という精神で、テストの終了日のルールを受け入れることはできません。つまり、機能しますが、実際には、このためには、あまりにも多くの要因を考慮する必要があります。

  • どのクラスがテストに入るか;
  • テストは教師または生徒に配布されます。
  • 学年の時間;
  • すべてのユーザーまたは新しいユーザーのみをテストします。

ただし、A / Bテストスキームでは、常に事前に終了日を修正する必要があります。テストの期間を予測するために、内部アプリケーション-A / Bテスト計算機を開発しました。アプリケーションは、過去1年間の選択したセグメントのユーザーのアクティビティに基づいて、選択したメトリックによるX％の上昇を大幅に修正するためにテストを実行する必要がある期間を計算します。複数のテストの修正も自動的に考慮され、テストの早期停止のためにしきい値の有意水準が計算されます。



すべてのメトリックは、テストオブジェクトのレベルで計算されます。指標が解決された問題の数である場合、教師レベルのテストでは、これは生徒が解決した問題の合計になります。生徒の基準を使用するため、すべての可能なスライスについて計算機が必要とする集計を事前に計算できます。テストの開始から毎日、テストの人数を知る必要がありますusers_cnt、メトリックの平均値 metric_mean とその分散 metric_std。コントロールグループの共有を修正するcontrol_group_share実験グループ exp_group_share そしてテストから期待される利得 uplift_expected パーセントで、学生統計の期待値を計算できます ttest_stat_value テストの各日の対応するp値：

ttest_stat_precursor=metric_mean users_cntmetric_stdttest_stat_value=ttest_stat_precursor1control_group_share+1exp_group_share∗uplift_expected/100


次に、毎日のp値を取得するのは簡単です：

pvalue = (1 − scipy.stats.norm.cdf(ttest_stat_value)) * 2

p値と有意水準を把握し、テスト期間中のテストの毎日のすべての修正を考慮に入れて、検出可能な最小上昇を計算できます（英語の文献では、MDE、最小検出可能影響）。その後、逆の問題を解決するのは簡単です。期待される上昇を特定するのに必要な日数を決定することです。

結論


結論として、記事の主なメッセージを思い出してください。

  • メトリックの平均値をグループで比較する場合、おそらく学生基準が適しています。例外は、非常に小さいサンプルサイズ（数十の観測値）または異常なメトリック分布（実際には、私はそのようなものを見たことがない）です。
  • テストに複数のグループがある場合は、複数の仮説テストの修正を使用します。最も単純なBonferroni補正で十分です。
