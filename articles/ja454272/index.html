ティックタイマー


時間に関連するすべての機能は、ハードウェアクロックによって制御されます。これは、定期的に割り込み要求を生成する単純な発振器です。クロック番号がアプリケーションプログラムで意味を持つようにするには、ジェネレーターの周波数がわかっている必要があります。

タイマー割り込み処理


ハードウェアタイマーによって生成された割り込みは、すべての時間関連のRTOS関数を実装する割り込みハンドラー（割り込みサービスルーチン、ISR）で特定の方法で処理する必要があります。Nucleus SEのタイマー割り込みハンドラーの詳細については、以下の記事の1つで説明します。

そして、システムタイマーの精度について簡単に説明する価値があります。

時間関数の精度は、クロックジェネレーターの周波数に直接依存します。たとえば、パルスが10ミリ秒ごとに到着し、アプリケーションタスクが100ミリ秒の遅延を必要とする場合、明らかに10パルスが必要です。ただし、前のインパルスがいつ受信されたかはわかりません。ちょうどまたはほぼ10ミリ秒前に発生した可能性があります。したがって、100ミリ秒の遅延には最大110ミリ秒かかることがあります。

この問題を解決する明白な方法は、ジェネレータの周波数を上げることです。パルスが1ミリ秒間隔で続く場合、100ミリ秒の遅延は100ミリ秒を超えることはありません。このソリューションの欠点は、タイマー割り込みハンドラーのプロセッサー時間が10倍になり、過度にかかることです。システム設計者は、タイマーに必要な精度と使用可能なプロセッサー能力のバランスを見つける必要があります。

システム時刻設定


ほとんどのNucleus SEオブジェクトと同様に、システム時間設定は、ほとんどの場合、nuse_config.hファイルの#defineディレクティブによって制御されます。主なパラメータはNUSE_SYSTEM_TIME_SUPPORTで、システム時間サポートメカニズムをアクティブにします。オブジェクトの数を指定する必要はありません。システム時間がアクティブ化されているかどうかです。 ゼロ以外の値を選択すると、システム時間の主なアクティベーターになります。このパラメーターは、データ構造を定義するときに使用されます。これについては、この記事の後半で詳しく説明します。さらに、ゼロ以外の値はAPI設定をアクティブにします。



APIアクティベーション


Nucleus SEの各API関数（ユーティリティ呼び出し）のnuse_config.hファイルには、アクティブ化する#defineディレクティブがあります。システム時間の場合、これらの文字は次の
とおり です。NUSE_CLOCK_SET
NUSE_CLOCK_RETRIEVE

デフォルトでは、これらの文字はFALSEに設定されているため、すべてのサービス呼び出しが無効になり、それらを実装するコードの組み込みがブロックされます。アプリケーションでシステム時刻を構成するには、必要なAPIサービス呼び出しを選択し、それらをTRUEに設定する必要があります。

以下は、デフォルトのnuse_config.hファイルからのコードのスニペットです。

#define NUSE_SYSTEM_TIME_SUPPORT FALSE   /*    */
#define NUSE_CLOCK_SET        FALSE    /*   */
#define NUSE_CLOCK_RETRIEVE   FALSE   /*    */

システムタイムアクティベーターをオフにしてシステムタイムAPIサービスコールを使用しようとすると、コンパイルエラーが発生します。コードがアクティブ化されていないAPI呼び出しを使用する場合、実装コードがアプリケーションに含まれていないため、レイアウトエラーが発生します。

システム時間ユーティリティの呼び出し


Nucleus RTOSは、システム時間に関連する2つのユーティリティコールをサポートし、次の機能を提供します。

  • システム時刻の値を設定します。Nucleus SEはNUSE_Clock_Set（）関数に実装されています。
  • システム時間の値を取得します。Nucleus SEはNUSE_Clock_Retrieve（）関数に実装されています。

これらの各呼び出しの実装をより詳細に検討してください。

システム時刻を設定および取得するためのサービス呼び出し


システム時間では、特定の値に設定して現在の値を取得する操作のみを実行できます。Nucleus RTOSとNucleus SEは、これらの操作を実装するための2つの基本的なAPI呼び出しを提供します。

システム時間の値の解釈は、それが本質的にカウンターの最後のリセット以降に発生した「クロックサイクル」の数のカウンターであるため、アプリケーションに依存します。この情報を使用するには、発電機の周波数がわかっている必要があります。

時間設定


このAPI関数を呼び出すことにより、どのタスクでもシステム時刻を設定できます。

Nucleus RTOS

サービス呼び出しプロトタイプでシステム時間を設定するための呼び出し：
VOID NU_Set_Clock（UNSIGNED new_value）;

パラメータ：
new_value-システム時刻に割り当てられる

値戻り値：なし。

Nucleus SEでシステム時間を設定するための
呼び出しこのAPI呼び出しは、Nucleus RTOS APIのコア機能をサポートしています。

サービス呼び出しプロトタイプ：
void NUSE_Clock_Set（U32 new_value）;

パラメータ：
new_value-システム時間に割り当てられる

値戻り値：なし

Nucleus SEで時間設定を実装する
コードは非常に単純です。指定された値は、クリティカルセクション内の NUSE_Tick_Clockに書き込まれます。

システム時間を取得する


タスクは、このAPI関数を使用してシステム時刻値を取得できます。

Nucleus RTOS

サービス呼び出しプロトタイプでシステム時間を取得するための呼び出し：
UNSIGNED NU_Retrieve_Clock（VOID）;

パラメータ：なし

戻り値：システム時刻の現在の値

Nucleus SE
サービス呼び出しでシステム時刻を取得するための呼び出しプロトタイプ：
U32 NUSE_Clock_Retrieve（void）;

パラメータ：なし

戻り値：システム時刻の現在の値

Nucleus SEでの受信時刻の実装
コードは非常に単純です。この関数は、クリティカルセクションで取得されたNUSE_Tick_Clock値を返します。
