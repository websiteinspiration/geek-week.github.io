Don't want Visual Studio Code: 7 open source alternatives

In this translation, we‚Äôll tell you about alternative code editors that are not inferior to VS Code, and even outperform it if you select the right extension.







Visual Studio Code Editor, also known as VS Code runs on Linux, Windows and macOS. It is intermediate between a simple text source code editor and an integrated development environment (IDE), which manages your entire code base. VS Code functionality can be expanded with plugins. This is a reliable code editor that can worthily compete with proprietary competitors.



VS Code was originally created as an open source product. However, its finished builds are distributed under a proprietary license from Microsoft. But it is not all that bad.





Using VS Code as open source



VS Code sources are available on GitHub . However, when you download the finished assembly , the Microsoft License comes into play . And then your VS Code suddenly ceases to be an open source product. It turns out that in this case it solves the assembly process.



Chris Diaz, the developer of the VS Code project, is conducting a similar example of the Chrome browser and its open source project Chromium. VS Code is really built on the basis of open source solutions. However, Microsoft‚Äôs official build is configured to take into account specific features, including the trademark, extension gallery, proprietary C # debugger, and telemetry. But when you clone and create the assembly yourself, you generate a cleaner version called Code-OSS (OSS stands for open source software).



In practice, the differences between VS Code and Code-OSS are minimal. In particular, telemetry works in VS Code: that is, Microsoft collects data on the use of the product. In fairness, it should be noted that today there are many products that collect such data.



If you like VS Code, but want to consider the ‚Äútrue‚Äù open source alternatives, then you should definitely read on.



Vscodium











The most obvious alternative to VS Code is VS Code without Microsoft's proprietary add-ons. The VSCodium project contains only free components built on the basis of the VS Code code base. VSCodium developers have done everything possible so that we can disable telemetry. They have prepared a clean build of the VS Code source code, which you can download without creating it yourself.



Developers recall that VS Code includes some proprietary tools that may not ship with open source. If you need them, there are documented ways to solve these problems. But if you need a very specific feature from Microsoft's VS Code build, you need to make sure that it works in VSCodium.



In addition, you need to check that all telemetry is disabled .



Code-OSS



If you do not want to use the VSCodium assembly, you can compile VS Code yourself from the source code and end up with the same thing. The executable is called Code-OSS, not VSCode. Licensing terms and solutions to problems using proprietary features are the same as VSCodium.



At the first start, you need to check that telemetry is disabled.



Atom











Atom is an open-source IDE-like text editor that Microsoft acquired with GitHub. Like VS Code, you can extend the Atom editor with plugins, add themes, and build your own combination of additional tools. This is a cross-platform product that has built-in integration with GitHub. In short, Atom is ideal if the extensions you need already exist or you are ready to write them.



Like VS Code, Atom uses telemetry by default.. It can be disabled, and unlike VS Code, there are no restrictions on extensions, so you do not need to change the workflow due to privacy requirements. Atom is certainly a useful tool for developers. Moreover: it is in principle suitable for everyone who uses a computer. If you're looking for a good general-purpose text editor, you can try Atom.



GNOME Builder











GNOME Builder is a Linux code editor specifically designed to build GNOME-based applications. If you are creating applications for Linux and want to easily achieve compatibility, then Builder is the best option. GNOME Builder is available at Flathub.org . When you first start it, it will prompt you to install the GNOME SDK if you haven‚Äôt.



However, you can use Builder not only to develop GNOME applications. It supports dozens of programming languages, including Python, Rust, C and C ++, Java, Go, JavaScript, TypeScript, VB.NET, several markup languages, Markdown, and many others. Some of them have full support with auto-complete and snippets, while others have minimal amenities, such as syntax highlighting and automatic parenthesis matching. It's nice to work with the IDE, whether you consider yourself a serious programmer or just want to indulge in HTML and CSS.



Geany











Geany is a powerful, reliable and lightweight editor with useful features to help you write good code in Bash, Python, Lua, XML, HTML, LaTeX and so on. It supports 50 different programming and scripting languages, as well as markup languages ‚Äã‚Äãand files (such as .diff and .po). Geany at least provides matching brackets and syntax highlighting, and usually offers much more.



Using plugins, you can add features such as the project browser, explorer, debugging, terminal, and so on. You can continue to expand its functionality and at some point it will turn into a real IDE. If you cannot run VS Code on your computer due to CPU or RAM limitations, Geany may be a good alternative. It starts up quickly and consumes a small amount of memory. Although Geany is slower than Vim in the terminal, it does not slow down even on the Raspberry Pi.



Brackets











Brackets is a text editor and IDE for web developers. It has powerful support for HTML, CSS, JavaScript, PHP, and even Python. Brackets has a rich ecosystem of extensions, so with suitable plugins it will allow you to write in almost any modern language.



There are extensions to help analyze, run scripts, and even compile and execute code directly in the editor window. Brackets has a traditional interface that can be understood by almost anyone who goes beyond the notebook. If you spend a little time adding the appropriate extensions and getting to know them, Brackets using autocomplete and a linter will help you avoid stupid typing errors. And if you write code, this will speed up the testing and debugging process.



Che











If you like to be ahead of the rest, then you should try Che . This is a cloud-based IDE, it works on the SaaS model. Nevertheless, it is a full-fledged open source project. If you have a copy of Kubernetes, you can fully transfer it to yourself.



Che is not just an IDE that lives in the cloud. This is an IDE designed for cloud development. It does not try to store anything on your local file system. If you have a Git server, you can consider it as a file system and work on a project directly from its repository. Of course, you can also download any project you are working on if you want to have a local backup.



But its main feature admired by cloud developers is that Che is a full-fledged open source development environment focused on Kubernetes. If you are creating applications, sites or containers (or a combination of them) for the cloud, then you should try Che.
