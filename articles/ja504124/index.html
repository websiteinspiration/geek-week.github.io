始める前に...


以前にPythonを使用したことがある場合は、コンピューターにAnacondaが既にインストールされている可能性があります。そうでない場合は、Python 3.7以降のAnacondaディストリビューションをこちらからダウンロードできます。

環境設定


Power BIでPyCaret機械学習機能の使用を開始する前に、仮想環境を作成してそこにインストールする必要がありますpycaret。これを行うには、3つの手順を実行する必要があります。

手順1-仮想環境を作成する

Anacondaコマンドプロンプトを開き、次のように入力します。

conda create --name myenv python=3.7

手順2-PyCaretのインストール

Anacondaコマンドプロンプトで次のコマンドを実行します。

pip install pycaret

インストールには15〜20分かかる場合があります。インストール中に問題が発生した場合は、GitHubのページで解決策を理解してください。

手順3-PythonがインストールされているPower BIで示す

作成された仮想環境をPower BIに関連付ける必要があります。これを行うには、Power BI Desktopのグローバル設定を使用します（[ファイル]-> [オプション]-> [グローバル]-> [Pythonスクリプト]）。Anaconda環境はデフォルトでディレクトリに置かれます：

C:\Users\username\AppData\Local\Continuum\anaconda3\envs\myenv



クラスタリングとは何ですか？


クラスタリングは、同様の特性に従ってデータをグループに分割する方法です。このようなグループは、データの調査、パターンの識別、およびデータのサブセットの分析に役立ちます。データのクラスタリングは、基礎となるデータ構造の特定に役立ちます。これは、多くの業界で役立ちます。ビジネスでのクラスタリングのいくつかの一般的な用途は次のとおりです。

  • マーケティングの顧客セグメンテーション。
  • プロモーションと割引に関する消費者行動の分析。
  • 発生時のCOVID-19などのジオクラスターの特定。

クラスタリングタイプ


クラスタリングタスクの主観的な性質を考えると、特定のタイプのタスクを解決するのにより適したさまざまなアルゴリズムがあります。各アルゴリズムには、独自の特性と数学的正当性があり、クラスターの分布の根底にあります。

今日のチュートリアルでは、PyCaretと呼ばれるPythonライブラリを使用したPower BIでのクラスター分析について説明しているため、数学については詳しく説明しません。



今日は、k-means法を使用します。これは、教師なしで最も簡単で最も人気のある教授法の1つです。k-means法の詳細については、こちらをご覧ください。

それでは始めましょう


Anaconda環境をセットアップし、PyCaretをインストールしたので、クラスター分析の基本とビジネスコンテキストを理解しました。次に、ビジネスに取り掛かります。

1.データ収集


最初のステップは、データセットをPower BI Desktopにインポートすることです。Webコネクタを使用してデータをダウンロードできます。（Power BI Desktop→データの取得→Webから）。



csvファイルへのリンク：https : //github.com/pycaret/powerbi-clustering/blob/master/clustering.csv。

2.モデルトレーニング


Power BIのクラスターモデルを学習するには、Power QueryエディターでPythonスクリプトを実行する必要があります（Power Queryエディター→変換→実行pythonスクリプト）。次のコードをスクリプトとして使用します。

from pycaret.clustering import *
dataset = get_clusters(dataset, num_clusters=5, ignore_features=['Country'])



パラメータを使用し てセットの「国」列を無視しましたignore_features。機械学習モデルをよりよくトレーニングするために特定の列を除外する必要がある理由はたくさんあります。

PyCaretを使用すると、不要な列を削除する代わりに非表示にすることができます。将来の分析で必要になる可能性があるためです。たとえば、現時点では「Country」をトレーニングに使用したくなく、この列をに渡しましたignore_features。

PyCaretには、すぐに使える8つの機械学習アルゴリズムがあります。



デフォルトでは、PyCaretは4つのクラスターでk-meansクラスタリングモデルをトレーニングします。ただし、デフォルト値は簡単に変更できます。

  • モデルのタイプを変更するには、のモデルパラメータを使用しますget_clusters()。
  • クラスターの数を変更するには、オプションを使用しますnum_clusters。

たとえば、これは、k-meansクラスタリングを6つのクラスターに行う方法です。

from pycaret.clustering import *
dataset = get_clusters(dataset, model='kmodes', num_clusters=6, ignore_features=['Country'])

クラスタリングの実装


上記では、Power BIでの最も単純なクラスタリングの実装を示しました。この方法では、Power BIでデータセットが更新されるたびに、クラスタリングモデルがトレーニングされます。これは、次の理由で問題になる可能性があります。

  • モデルが新しいデータで再トレーニングされると、クラスターラベルが変更される可能性があります（つまり、以前に一部のデータポイントが最初のクラスターに割り当てられていた場合、再トレーニングされると、それらは2番目のクラスターに割り当てられます）。
  • モデルを再トレーニングするたびに数時間を費やす必要はありません。

Power BIでクラスタリングを何度も再学習する代わりに実装するより効果的な方法は、事前トレーニング済みのモデルを使用してクラスターラベルを作成することです。

初期モデルトレーニング


任意の統合開発環境（IDE）またはノートブックを使用して、モデルをトレーニングできます。この例では、Visual Studio Codeでクラスタリングモデルをトレーニングしました。



次に、トレーニング済みモデルをピクルファイルとして保存し、Power Queryにインポートしてクラスターラベルを生成します。



PyCaretを使用してJupyterノートブックにクラスター分析を実装する方法の詳細については、この2分間のビデオをご覧ください。


事前トレーニング済みモデルの使用


以下のコードを実行して、事前トレーニング済みモデルからタグを生成します。

from pycaret.clustering import *
dataset = predict_model('c:/.../clustering_deployment_20052020, data = dataset)

結果は、以前に観察したものと同じになります。唯一の違いは、事前トレーニング済みモデルを使用する場合、タグは、再トレーニングされたモデルではなく、古いモデルを使用した新しいデータセットに基づいて生成されることです。

Power BIサービスを操作する


.pbixファイルをPower BIサービスにアップロードした後、さらにいくつかの手順に従って、機械学習パイプラインをデータパイプラインにスムーズに統合する必要があります。手順は次のとおりです。

  • データセットのスケジュールされた更新をオンにします。これにより、Pythonスクリプトを使用して更新するデータセットを含むワークブックをスケジュールできます。パーソナルゲートウェイに関する情報も含まれている、スケジュールされた更新の構成セクションをご覧ください。
  • パーソナルゲートウェイのインストール-パーソナルゲートウェイが必要です。パーソナルゲートウェイは、Pythonがインストールされているのと同じディレクトリにインストールする必要があります。Power BIサービスは、Python環境にアクセスできる必要があります。ここでは、Personal Gatewayをインストールして構成する方法について詳しく説明します。
