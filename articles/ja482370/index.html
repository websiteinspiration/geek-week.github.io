nodejs：プロセスマネージャーとES6モジュール

サーバー側のJavaScriptの世界では、私はクリーンでほとんど雑然とした心を持った初心者です。したがって、プロセスマネージャーの存在、特にpm2の存在を知ったとき、私はすぐにそれを使用してnodejs、自己教育の目的でいくつかの簡単なバックエンドサービスを開始しようとしました。JSコードのモジュールをimport（ES6モジュール）で接続する機能に非常に感銘を受けました。これにより、ブラウザーとサーバー側の両方で同じコードを使用できます。ES6モジュールを使用した簡単なサービスを書き留めました。


つまり、アプリケーションのES6バージョンを起動pm2できませんでした。そのようなアプリケーションを起動するにはforever、またはのいずれかを使用することをお勧めしますsystemd。カットの下-より長いテキストが好きな人のための結果に関するレポート。


前書き


このドキュメントでは、プロセスマネージャーとは、実行中のnodejsアプリケーションを監視し、クラッシュが発生した場合に再起動することを主なタスクとするサービスを指します。また、プロセスマネージャは、アプリケーション（プロセッサ、メモリ）によって消費されるリソースに関する情報を収集できます（必須ではありません）。


テストサービス


プロセスマネージャーをテストするために、ES6サービス（github repo）で次のコードを使用しました。


# src/app_es6.mjs
import express from "express";
import mod from "./mod/es6.mjs";

const app = express();
const msg = "Hello World! " + mod.getName();

app.get("/", function (req, res) {
    console.log(msg);
    res.send(msg);
});

app.listen(3000, function () {
    console.log('ES6 app listening on port 3000!');
});

そしてES6モジュールで：


# src/mod/es6.mjs
export default {
    getName: function () {
        return "ES6 module is here.";
    }
}

CommonJSモジュールで実行される同様のサービスは次のようになります。


# src/app_cjs.js
const express = require("express");
const mod = require("./mod/cjs.js");
const app = express();

const msg = "Hello World! " + mod.getName();

app.get("/", function (req, res) {
    console.log(msg);
    res.send(msg);
});

app.listen(3000, function () {
    console.log("CommonJS app listening on port 3000!");
});

CJSモジュール：


# src/mod/cjs.js
module.exports = {
    getName: function () {
        return "CommonJS module is here.";
    }
};

nodejsv12.14.0でプロセスマネージャを使用せずにサービスを開始する：


$  node --experimental-modules ./src/app_es6.mjs    # ES6-service
$  node ./src/app_cjs.js    # CJS-service
