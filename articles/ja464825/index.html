a）ナビゲーションジェスチャー




Android Qでは、Googleはすべてのナビゲーションジェスチャーを、フローティングナビゲーションバー（戻る、ホーム、最近のアプリ）に代わる新しいナビゲーションジェスチャーシステムと組み合わせようとしました。

  • 影響：アプリケーションにジェスチャーが含まれている場合、システムジェスチャーと競合する可能性があります。システムジェスチャーが最初に処理されるため、これはAndroid Qで実行されているすべてのアプリケーションに影響します（ターゲットSDKに関係なく）。
  • 推奨されるアプローチ：フローティングナビゲーションバーが実質的に消えたので、利用可能な新しい画面スペースを最大限に活用して、ユーザーに没入効果を提供することをお勧めします。

没入型効果を作成するには、アプリケーションをステータスバーとナビゲーションバーの背後に表示する必要があります。まず、ステータスバーとナビゲーションバーを透明にします。

<style name="AppTheme">
......
<item name="android:navigationBarColor">@android:color/transparent</item>
<!-- Optional, but recommended for full edge-to-edge rendering -->
<item name="android:statusbarColor">@android:color/transparent</item>
</style>

次に、それActivity/Viewを全画面表示にします。

view.systemUiVisibility =
//Layout as if the navigation bar was hidden
View.SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION
or
//Layout as if the status bar was hidden
View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN
or
//Layout at its most extreme
View.SYSTEM_UI_FLAG_LAYOUT_STABLE



FAB（フローティングアクションボタン）は、フローティングナビゲーションバーで覆われています。わかりやすくするために、フローティングナビゲーションバーが含まれています。

アプリケーションが（透明な）システムコンポーネントの背後に全画面モードで表示されるようになったため、何らかのインタラクティブアプリケーションコンポーネントがシステムUIによってブロックされる可能性があります。この問題を解決するには、インセットを使用します。

インセットは、システムUIと競合しないようにビューをどれだけ移動するかを指示する一連の値です。この情報は、WindowInsetクラスによって提供されます。。 API 20以降、Androidは開発者にシステムウィンドウインセットを提供します。これにより、システムコンポーネント（フローティングナビゲーションバーやステータスバーなど）が占めるスペースがわかります。この情報を使用して、ユーザーがシステムUIではなくインタラクティブビューをタップするようにインタラクティブビューを移動できます。

root.setOnApplyWindowInsetsListener { _, insets ->
            val fabLp = fab.layoutParams as CoordinatorLayout.LayoutParams
            fabLp.bottomMargin = fabOriginalBottomMargin + insets.systemWindowInsetBottom
            fab.layoutParams = fabLp

            insets.consumeSystemWindowInsets()
        }

FABの場合、WindowsInsetsリスナーを設定し、下部ウィンドウインセットを下部インデントとして適用します。

b）ダークテーマ


過去1年間で、多くのアプリケーションが暗いテーマのサポートを開始するのを見てきました。Android Qはこのスイッチをシステム設定に追加します。

ユーザーは3つの方法でダークテーマを有効にできます。

  1. システム設定の新しい項目（設定->表示->テーマ）
  2. クイック設定の新しいアイテム
  3. 製造元によっては、省電力モードでオンにすることができます。

Dark Themeのシステム全体の構成をサポートするには、アプリケーションのデフォルトのテーマがDayNightテーマを継承する必要があります。これにより、アプリケーションのメインテーマと、システムによって制御されるナイトモードフラグが関連付けられます。これは、修飾子フォルダーからリソースを抽出することによって機能しますnight。

<!--App Compat Style-->
<style name="AppTheme" parent="Theme.AppCompat.DayNight">
<!--Material Components-->
<style name="AppTheme" parent="Theme.MaterialComponents.DayNight">

App Compatスタイルの内部を見ると、DayNightテーマが、デフォルトの構成ではAppCompat Light、夜の構成ではAppCompatを指していることがわかります。

<!--values/themes.xml-->
<style name="Theme.AppCompat.DayNight" parent="Theme.AppCompat.Light">
<!--values-night/themes.xml-->
<style name="Theme.AppCompat.DayNight" parent="Theme.AppCompat">

ユーザーがアプリケーション内からテーマを制御できるようにするには、次の4つのモードのいずれかを受け入れるAppCompatDelegate.setDefaultNightMode（）を呼び出します。

  1. MODE_NIGHT_YES -暗いトピックを表示
  2. MODE_NIGHT_NO -明るいトピックを示す
  3. MODE_NIGHT_FOLLOW_SYSTEM -システム設定に従う
  4. MODE_NIGHT_AUTO_BATTERY -デバイスが省電力モードに入ったときに暗いテーマをオンにします。
