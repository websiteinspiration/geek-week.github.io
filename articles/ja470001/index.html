Linuxのヒントとコツ：サーバー、オープン

自分自身、最愛の人、SSH / RDP /それ以外-小さなRTFM /スパー経由で世界中のどこからでもサーバーへのアクセスを提供する必要がある人。


手元にあるデバイスから、VPNやその他の口笛を使わずに行う必要があります。


そして、サーバーがあまり運動しないように。


必要なのは、ノック、まっすぐな腕、5分の作業だけです。


もちろん、すべてがインターネット上にありますが（Habré上でも）、特定の実装に関しては、それは始まります...


Fedora / CentOSの例で練習しますが、それは重要ではありません。


拍車は初心者とこのビジネスのバイソンの両方に合うので、コメントはありますが短くなります。


1.サーバー


  • ノックサーバーを置く：
    yum/dnf install knock-server


  • それを構成します（たとえば、sshで）-/etc/knockd.conf：


    [options]
    UseSyslog
    interface = enp1s0f0
[SSHopen]
    sequence        = 33333,22222,11111
    seq_timeout     = 5
    tcpflags        = syn
    start_command   = iptables -A INPUT -s %IP% -p tcp --dport 22 -j ACCEPT
    cmd_timeout     = 3600
    stop_command    = iptables -D INPUT -s %IP% -p tcp --dport 22 -j ACCEPT
[SSHclose]
    sequence        = 11111,22222,33333
    seq_timeout     = 5
    tcpflags        = syn
    command         = /sbin/iptables -D INPUT -s %IP% -p tcp --dport 22 -j ACCEPT

    「オープニング」部分は、1時間後に自動クローズするように設定されています。あなたは、決して知らない ...


  • / etc / sysconfig / iptables：


    ...
-A INPUT -p tcp -m state --state NEW -m tcp --dport 11111 -j ACCEPT
-A INPUT -p tcp -m state --state NEW -m tcp --dport 22222 -j ACCEPT
-A INPUT -p tcp -m state --state NEW -m tcp --dport 33333 -j ACCEPT
...

  • フォワード：


    service iptables restart
service knockd start

  • 内部で回転している仮想Windows ServerにRDPを追加できます（/etc/knockd.conf;お好みに合わせてインターフェイスに名前を付けます）：


    [RDPopen]
    sequence        = 44444,33333,22222
    seq_timeout     = 5
    tcpflags        = syn
    start_command   = iptables -t nat -A PREROUTING -s %IP% -i enp1s0f0 -p tcp -m tcp --dport 3389 -j DNAT --to-destination 192.168.0.2
    cmd_timeout     = 3600
    stop_command    = iptables -t nat -D PREROUTING -s %IP% -i enp1s0f0 -p tcp -m tcp --dport 3389 -j DNAT --to-destination 192.168.0.2
[RDPclose]
    sequence        = 22222,33333,44444
    seq_timeout     = 5
    tcpflags        = syn
    command         = iptables -t nat -D PREROUTING -s %IP% -i enp1s0f0 -p tcp -m tcp --dport 3389 -j DNAT --to-destination 192.168.0.2

    クライアントからのすべてのキックは、サーバーによってチームによって監視されますiptables -S。
