こんにちは、ハブル！Tommi Virtanenによる「Git for Computer Scientists」の翻訳をお見せします。


GIT Inside Out：はじめに


私自身から： さまざまな人気のあるテクノロジーが内部でどのように配置されているかに関する記事を定期的に読み、この資料に出くわしました。記事は、鈍いテキストのシートよりもはるかによく知覚される単純で理解可能なスキームの存在によって興味深いように見えました。ロシア語に翻訳することにしました。オリジナルから撮影した画像。


誰が興味を持ち、おそらく役に立つかもしれません：Gitを毎日（つまり、最初のソフトウェア開発者でないとしても、毎秒）作業し、その作業のメカニズムをよりよく理解したい人々。


注：記事をよりよく理解するには、そのような獣を有向非巡回グラフ（DAG）として理解する必要があります。


オブジェクトストレージ


Gitオブジェクトリポジトリは、大まかに言って、さまざまなタイプのオブジェクトを含むDAGです。オブジェクトは圧縮形式で格納され、SHA-1ハッシュで識別されます（これは、オブジェクトを表すファイルの内容のハッシュではなく、Gitでのそのプレゼンテーションのハッシュです）。


ブロブ
