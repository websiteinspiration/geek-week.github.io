Zextras Adminを使用したZimbra OSEでの完全なマルチテナンシー

今日、マルチテナンシーは、ITサービスを提供するための最も効果的なモデルの1つです。同じサーバーインフラストラクチャで実行されているが、多くのユーザーや企業がアクセスできるアプリケーションの単一のインスタンスは、ITサービスの提供コストを最小限に抑え、その品質を最大化できます。 Zimbra Collaboration Suite Open-Source Editionアーキテクチャは、もともとマルチテナンシーのアイデアに触発されました。このため、Zimbra OSEの1つのインストールで、ユーザーがお互いに気づくことなく、多数のメールドメインを作成できます。

そのため、Zimbra Collaboration Suite Open-Source Editionは、各企業が独自のドメインでメールを提供する必要があるが、これらの目標に多額の費用をかけたくない企業や持ち株会社のグループに最適です。また、Zimbra Collaboration Suite Open-Source Editionは、企業の電子メールおよびコラボレーションツールへのアクセスを提供するSaaSプロバイダーに適している可能性があります。 Zimbraのオープンソースバージョンのドメイン。言い換えると、Zimbra OSEにはこれらの機能を実装するためのAPIしかありませんが、Web管理コンソールには特別なコンソールコマンドやアイテムはありません。これらの制限を取り除くために、Zextrasは、Zextras Suite Pro拡張セットの一部である特別なZextras Adminアドオンを開発しました。 Zextras Adminが無料のZimbra OSEをSaaSプロバイダーにとって理想的なソリューションに変える方法を見てみましょう。



メインの管理者アカウントに加えて、Zimbra Collaboration Suite Open-Source Editionは他の管理者アカウントの作成をサポートしますが、作成された各管理者は元の管理者と同じ完全な権限を持ちます。組み込み関数を使用して、APIを介して管理者権限をZimbra OSEの任意の1つのドメインに制限することは非常に困難です。その結果、これはSaaSプロバイダーがドメイン管理をクライアントに転送して個別に管理することを妨げる深刻な制限になります。つまり、新しいメールの作成、古いメールボックスの削除、それらのパスワードの作成など、企業メールの管理作業はすべてSaaSプロバイダーが行う必要があります。サービスを提供するコストの明らかな増加に加えて、情報セキュリティに関連する巨大なリスクも生み出します。

Zextras Admin拡張機能はこの問題を解決できます。これにより、Zimbra OSEに管理者権限の差別化機能を追加できます。この拡張機能により、システム管理者は無制限の数の新しい管理者を作成し、必要に応じてその権限を制限できます。たとえば、自分ですべてのクライアントからのリクエストに対応できない場合は、アシスタントのドメインの一部を管理者にすることができます。これにより、顧客からの要求への対応速度が向上し、情報セキュリティが強化され、管理者の作業の質が向上します。

また、ユーザーをいずれかのドメインの管理者にして権限を1つのドメインに制限したり、ドメインのユーザーのパスワードをリセットしたり新しいアカウントを作成したりできるが、従業員のメールボックスの内容にはアクセスできないジュニア管理者を追加したりすることもできます。これにより、提供するメールドメインを独自に管理できるセルフサービスシステムを構築することができます。このオプションは、企業にとって安全で便利なだけでなく、SaaSプロバイダーがサービスを提供するコストを大幅に削減できるようにします。

これはすべて、管理コンソールのいくつかのコマンドを使用して行われることも注目に値します。 mail.company.ruドメインの管理者を作成する例で、これを確認してみましょう。 admin@company.ruをmail.company.ruドメインの管理者にするには、zxsuite admin doAddDelegationSettings admin@company.ru mail.company.ru viewMail trueコマンドを入力するだけ  です。その後、ユーザーadmin@company.ruが自分のドメインの管理者になり、他のユーザーのメールを表示できるようになります。 

メイン管理者を作成することに加えて、コマンドzxsuite admin doAddDelegationSettings manager@company.ru mail.company.ru viewMail falseを使用して、マネージャーの1人をジュニア管理者にし  ます。。後任の管理者は、メインの管理者とは異なり、従業員のメールを表示することはできませんが、メールボックスの作成や削除など、その他の操作は実行できます。これは、メインの管理者が日常の操作を実行する時間がない場合に非常に役立ちます。

また、Zextrasの管理者は、権限を編集する機能を提供します。たとえば、メインの管理者が休暇をとった場合、彼の職務はマネージャーによって一時的に実行されます。マネージャーが従業員のメールを表示できるようにするには、コマンドzxsuite admin doEditDelegationSettings manager@company.ru mail.company.ru viewMail trueを使用するだけで十分  です。その後、プライマリ管理者が休暇から戻ったときに、マネージャをジュニア管理者として再確立することができます。また、zxsuite admin doRemoveDelegationSettings manager@company.ru mail.company.ruコマンドを使用して、ユーザーに管理者権限を付与することもできます  。



上記のすべての機能がZimbra管理Webコンソールで複製されていることも重要です。これにより、コマンドラインの経験がほとんどない従業員でもエンタープライズドメイン管理を利用できるようになります。また、これらの設定のグラフィカルインターフェイスの存在により、ドメインを管理する従業員のトレーニング時間を短縮できます。

ただし、管理権限の委任の複雑さは、Zimbra OSEの唯一の深刻な制限ではありません。さらに、ドメインのメールボックス数に制限を設定する組み込み機能、およびドメインが占有するスペースの制限も、APIを通じてのみ実装されます。このような制限がないと、システム管理者がメールストレージに必要なドライブのボリュームを計画することが難しくなります。また、そのような制限がないことは、関税計画を導入することが不可能であることを意味します。 Zextras Admin拡張機能は、この制限を取り除くことができます。ドメイン制限機能のおかげで、この拡張機能により、メールボックスの数とメールボックスが占めるスペースの両方によって特定のドメインを制限できます。 

mail.company.ruドメインを使用している企業が、50を超えるメールボックスを持つことができない料金を取得し、メールストレージのハードドライブ上で25ギガバイトを超える容量を占めているとします。このドメインを50ユーザーに制限することは論理的です。各ユーザーは512メガバイトのメールボックスを受け取りますが、実際には、このような制限は企業のすべての従業員に適しています。たとえば、サイズが100メガバイトのシンプルなメールボックスでシンプルなマネージャーに十分である場合、常にアクティブに対応している営業スタッフには1ギガバイトでは不十分な場合があります。したがって、会社が管理者に対して1つの制限を導入することは理にかなっており、販売および技術サポートの従業員に対しては料金が異なります。これは、Zimbra OSEで呼び出されるグループに従業員を分割することで実現できます。サービスクラスをクリックし、各グループに適切な制限を設定します。 

これを行うには、メインの管理者がコマンドzxsuite admin setDomainSettings mail.company.ru account_limit 50 domain_account_quota 1gb cos_limits managers：40、sales：10を入力するだけで十分です  。このため、ドメインには50アカウントの制限が導入され、メールボックスの最大サイズは1ギガバイトであり、メールボックスは2つの異なるグループに分離されています。その後、「Managers」グループの40人のユーザーに対して、メールボックスのサイズに人為的な制限を384メガバイトに設定でき、「Salesmen」グループは1ギガバイトの制限を残します。したがって、完全に実装されている場合でも、ドメインmail.company.ruのメールボックスは25ギガバイトを超えることはありません。 



上記の機能はすべてZextras Suite Web管理コンソールにも表示され、ドメイン管理者はトレーニングに多くの時間を費やすことなく、必要な変更を可能な限り迅速かつ便利に行うことができます。

また、SaaSプロバイダーとクライアント間の対話の透過性を最大限に確保するために、Zextras Adminは委任された管理者のすべてのアクションのログを維持します。これはZimbra OSE管理コンソールから直接表示できます。また、毎月1日に、Zextras Adminは、すべての管理者のアクションに関する月次レポートを生成します。これには、失敗したログイン試行やドメイン制限を超える試みの失敗など、必要なすべてのデータが含まれます。 

したがって、Zextras AdminはZimbra Collaboration Suite Open-Source EditionをSaaSプロバイダーに完全に適したソリューションに変えます。ライセンスのコストが非常に低く、セルフサービスの可能性があるマルチテナントアーキテクチャにより、このソリューションにより、ISPはサービスを提供するコストを削減し、ビジネスをより限定的にし、結果として競争力を高めることができます。
