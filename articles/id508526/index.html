Apa intinya?



Dalam JavaScript, dimungkinkan untuk menggunakan operator untuk mendapatkan sisa pembagian ke angka floating point. Ini berfungsi seperti ini: 



3.14 % 5 // 3.14
13.14 % 5 // 3.1400000000000006


Pada saat yang sama, nomor bertipe Number dan disimpan dalam 64 bit sesuai dengan standar IEEE 754. Dalam. NET, tipe ini memiliki saudara kembar System.Double atau hanya ganda. Jika numeric literal berisi floating point, maka defaultnya menjadi double. Niat dapat diekspresikan secara eksplisit dengan menambahkan akhiran d atau D. ke nomor. Kemampuan untuk membagi dua dengan sisanya juga tersedia. Jadi coba. Dan ... Bingo! 



Console.WriteLine(3.14d % 5); // 3,14
Console.WriteLine(13.14d % 5); // 3,1400000000000006

Dalam contoh berikut, kondisinya benar. 



const x = { i: 1, toString: function() { return this.i++; } };

    document.write("This will be printed!");


Ini dicapai karena fakta bahwa ketika memeriksa kesetaraan, operan dilemparkan ke jenis yang sama. Dalam kasus ini, metode toString () yang diganti ditimpa pada x, yang, selain mengembalikan sesuatu, mengubah keadaan objek. Kami mencatat kepada diri kami sendiri bahwa fungsi murni itu bagus, tetapi demi kepentingan kami mencoba menerapkan konsep dari contoh. 



Trickster x = new Trickster();
if (x == 1 && x == 2 && x == 3)
    Console.WriteLine("This will be printed!");


.NET memungkinkan Anda untuk mengganti metode dan gips, jika memungkinkan, jika gips yang sesuai ditentukan. 



class Trickster
{
    private int value = 1;

    public override string ToString() =>
        value++.ToString();

    public static implicit operator int(Trickster trickster) =>
        int.Parse(trickster.ToString());
}


Hore, itu berhasil. Tetapi membingungkan bahwa Trickster tampaknya beradaptasi dengan int. Menggantikan 



public static implicit operator int(Trickster trickster) =>
    int.Parse(trickster.ToString());


di 



public static implicit operator double(Trickster trickster) =>
    double.Parse(trickster.ToString());


Lagipula itu berhasil. Sekarang Trickster dan int digunakan untuk menggandakan.



Sebenarnya, redefinisi ToString () tidak penting untuk pencapaian hasil akhir dan dibuat untuk meniru maksimum contoh dalam JavaScript. Cukup untuk mendefinisikan gips menjadi dua kali lipat. Tentu saja, tidak melupakan pentingnya efek samping. 



class Trickster
{
    private int value = 1;

    public static implicit operator double(Trickster trickster) =>
        trickster.value++;
}

Pegangan elegan



Kode JavaScript ini menghasilkan 3 kali.

  setTimeout(() => console.log(i), 0);


Karena tidak ada fungsi setTimeout di luar kotak di C #, kami mengimplementasikan sendiri analog dan mendapatkan hasil yang persis sama. 



void SetTimeout(Action action, int delay) =>
    Task.Run(async () =>
    {
        await Task.Delay(delay);
        action();
    });

for (int i = 0; i <= 2; ++i)
    SetTimeout(() => Console.WriteLine(i), 0);

Anda hanya perlu menerapkan sepasang konversi tipe implisit dan membebani pasangan pernyataan. 



class Trickster
{
    private string value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = s };

    public static implicit operator Trickster(int i) =>
        new Trickster { value = i.ToString() };

    public static string operator +(Trickster trickster, int i) =>
        trickster.value + i;

    public static int operator -(Trickster trickster, int i) =>
        int.Parse(trickster.value) - i;

    public override string ToString() =>
        value;
}

Pisang juggling



Dalam JavaScript, string "pisang" dapat diperoleh dengan cara berikut: 



('b' + 'a' + + 'a' + 'a').toLowerCase(); // "banana"
('b'+'a'++'a'+'a').toLowerCase(); // Uncaught SyntaxError: Invalid left-hand side expression in postfix operation


Di sini, menerapkan unary + ke 'a' yang kedua mengembalikan NaN, yang dilemparkan ke string 'NaN' sebagai tambahan pada sisa string, dan hasilnya adalah 'baNaNa', dan demi kecantikan semuanya direduksi menjadi huruf kecil.



Di C # kita membuat kelas 



class Trickster
{
    private string value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = s };

    public static double operator +(Trickster trickster) =>
        double.TryParse(trickster.value, out double result) ? result : double.NaN;
}


dan coba 



//  double.NaN.ToString()  "NaN",   " "
Thread.CurrentThread.CurrentCulture = new CultureInfo("en-us");

Sayangnya, terlepas dari casting yang tersirat, kompiler tidak dapat membuat string string -> Trickster -> double -> string, dan Anda harus memberi tahu dia secara eksplisit. (Jika Anda berpikir tentang itu, para pemain seperti itu akan terlihat lebih dari aneh.)



Trickster dapat diimplementasikan secara berbeda: 



class Trickster
{
    private double value;

    public static implicit operator Trickster(string s) =>
        new Trickster { value = double.TryParse(s, out var d) ? d : double.NaN };

    public static double operator +(Trickster trickster) =>
        +trickster.value;
}


Dalam hal ini, Anda bahkan dapat mengganti unary + overload dengan pemeran implisit menjadi dua kali lipat: 



public static implicit operator double(Trickster trickster) =>
    trickster.value;
