Raspberry Piのクラウドオブジェクト検出器を使用したビデオ

プロローグ


現在、ビデオがネットワーク上を歩いています。テスラのオートパイロットが道路をどのように見ているかを示しています。

検出器で強化されたビデオをリアルタイムで放送するために、私の手は長い間かゆみました。



問題は、Raspberryでビデオをブロードキャストしたいということであり、Raspberryでのニューラルネットワーク検出器のパフォーマンスには、まだ多くの要望が残っています。

Intelニューラルコンピュータスティック


私はさまざまな解決策を検討しました。

では前の記事、私はインテルニューラルコンピュータスティックで実験しました。鉄片は強力ですが、独自のネットワーク形式が必要です。

Intelがメインフレームワーク用のコンバーターを提供しているという事実にもかかわらず、ここには多くの落とし穴があります。

たとえば、目的のネットワークの形式に互換性がない場合があり、互換性がある場合、一部のレイヤーがデバイスでサポートされない可能性があります。サポートされている場合、変換プロセス中にエラーが発生し、その結果、奇妙なことが発生します。

一般に、何らかの任意のニューラルネットワークが必要な場合は、NCSでは機能しない可能性があります。したがって、私は最も人気があり、手頃なツールを使用して問題を解決することを試みることにしました。

雲


ローカルのハードウェアソリューションに代わる明白な方法は、クラウドにアクセスすることです。

既製のオプション-目が走ります。

すべてのリーダー：

  • マイクロソフト
  • グーグル
  • アマゾン

...そしてあまり知られていないものの数十。

この多様性の中から選択することは決して簡単ではありません。

そして、私は選ぶのではなく、古き良きOpenCVワークフローをDockerにラップしてクラウドで実行することにしました。

このアプローチの利点は、柔軟性と制御です。ニューラルネットワーク、ホスティング、サーバーを変更できます。一般的に、気まぐれです。

サーバ


ローカルプロトタイプから始めましょう。

従来、私はREST API、OpenCV、およびMobileSSDネットワークにFlaskを使用しています。

Dockerに現在のバージョンをインストールしたところ、OpenCV 4.1.2がモバイルSSD v1_coco_2018_01_28で機能しないことがわかりました。テスト済みの11_06_2017にロールバックする必要がありました。

サービスの開始時に、クラス名とネットワークをロードします。

テンソルの排気を読み取り可能なjsonに変換します。

def build_detection(data, thr, rows, cols):
    ret = []
    for detection in data[0,0,:,:]:
        score = float(detection[2])
        if score > thr:
            cls = int(detection[1])
            a = {"class" : cls, "name" : tf_labels.getLabel(cls),  "score" : score}
            a["x"] = int(detection[3] * cols)
            a["y"] = int(detection[4] * rows)
            a["w"] = int(detection[5] * cols ) - a["x"]
            a["h"] = int(detection[6] * rows) - a["y"]
            ret.append(a)
    return ret

次に、Flaskを介してこの操作をエクスポートします（入力、画像、出力、jsonの検出器の結果）。

別のオプションでは、より多くの作業がサーバーに転送されます。彼は見つかったオブジェクトを丸で囲み、完成した画像を返します。

このオプションは、opencvをサーバーにプルしたくない場合に適しています。

Docker


画像を組み合わせます。

コードはコーミングされてGithubに配置され、Dockerはそこから直接コードを取得します。

プラットフォームとしては、Raspberryと同じDebian Stretchを採用しています。テスト済みの技術スタックはそのままにしておきます。

フラスコ、protobuf、リクエスト、opencv_pythonをインストールし、GithubからモバイルSSD、サーバーコードをダウンロードして、サーバーを起動する必要があります。

FROM python:3.7-stretch

RUN pip3 install flask
RUN pip3 install protobuf
RUN pip3 install requests
RUN pip3 install opencv_python

ADD http://download.tensorflow.org/models/object_detection/ssd_mobilenet_v1_coco_11_06_2017.tar.gz /
RUN tar -xvf /ssd_mobilenet_v1_coco_11_06_2017.tar.gz

ADD https://github.com/tprlab/docker-detect/archive/master.zip /
RUN unzip /master.zip

EXPOSE 80

CMD ["python3", "/docker-detect-master/detect-app/app.py"]

リクエストベースの検出器のための シンプルなクライアント。

Docker Hubに公開する

実験では、ランダムな交差点で撮影されたビデオファイルを使用しました。

検出器を使用すると、毎秒3フレーム以下になり、すべてが非常に遅くなります。
GCloudで強力なマシンを使用すると、1秒あたり4〜5フレームを検出できますが、目との違いはほとんど感知できず、まだ遅いです。



クラウドと交通費はそれとは何の関係もありません;通常のハードウェアでは、検出器はその速度で動作します。

ニューラルコンピュータスティック


私は抵抗できず、ベンチマークをNCSに導けませんでした。

検出器の速度は、0.1秒よりもわずかに遅く、いずれの場合も、弱いマシンの雲より2〜3倍高速でした（つまり、1秒あたり8〜9フレーム）。



結果の違いは、モバイルSSDバージョン2018_01_28がNCSで発売されたという事実によって説明されます。

PSさらに、実験により、I7プロセッサを搭載したかなり強力なデスクトップマシンがわずかに優れた結果を示し、毎秒10フレームをスクイーズできることが判明しました。

集まる


実験はさらに進み、Google Kubernetesの5つのノードに検出器を配置しました。
ポッド自体は弱く、各ポッドは毎秒2フレームを超える処理ができませんでした。
ただし、N個のノードのクラスターを開始してフレームをN個のスレッドに解析すると、十分な数のノード（5）を使用すると、毎秒10フレームの望ましい結果を得ることができます。

def generate():
    while True:
        rc, frame = vs.read()
        if frame is not None:
            future = executor.submit(handle_frame, (frame.copy()))
            Q.append(future)

        keep_polling = len(Q) > 0
        while(keep_polling):            
            top = Q[0]
            if top.done():
                outFrame = top.result()
                Q.popleft()
                if outFrame:
                    yield(b'--frame\r\n' b'Content-Type: image/jpeg\r\n\r\n' + bytearray(outFrame) + b'\r\n')
                keep_polling = len(Q) > 0
            else:
                keep_polling = len(Q) >= M
