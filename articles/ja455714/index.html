IFTTTとDjangoを使用してGoogleフォームをNotionに自動的にエクスポートする

全てにおいて良い日！この記事はNotionを使用するすべての人にとって興味深いものになると思いますが、何らかの理由で完全に移行できませんでした。

先史時代


私は自分のプロジェクトを開発しています。メール入力後のランディングページには、Googleフォームに基づく社会調査へのリンクが掲載されています。回答はGoogleドライブの銘板に記録されます。

問題は、私がNotion ですべてを自分のものにしておくことです。より便利なコーニーです。レビューはほとんどありませんでしたが、コピーと貼り付けによって処理されました。それからそれらのより多くがありました-そして何かを思いつく必要がありました。誰が何が起こったか気にしません-猫へようこそ。

問題


Googleフォームは回答をネームプレートにのみ記録します。つまり、ここには他のレシピはありません。したがって、私は計画を立てました。IFTTTを介してタブレットの更新を聞いて、新しいデータをWebhookに送信し、何らかの方法で処理してNotionにアップロードしましょう。

IFTTTに慣れていない方のために：これは、一連のアクションを作成できるサービスです。たとえば、「電報で投稿された」-「VKontakteにエクスポートします」としましょう。

計画は失敗し始めました：Notionには公式のAPIがありません。しかし、誰かがそれを元に戻し、非公式のAPIを作成しました。



最終的な計画はこれでした：

  • IFTTTでアプレットを作成します。「タブレットに行が追加されました-サーバーに送信してください
  • データを受信し、それをNotionに送信するサーバーを直接作成します

2番目の問題は、IFTTTがGoogleスプレッドシートとの統合に失敗し、アプレットが機能しないことが判明したときに発生しました。



したがって、計画を変更する必要がありました。csv'shkuをGoogleスプレッドシートで取り出し、サーバーで解析して、Notionのすべての新しいものをスローしました。IFTTTは、プロセス全体のトリガーとして使用されます。

パート1. CSVとGoogleスプレッドシート


この部分がおそらく最も簡単です。テーブルを開いて表示します（Cookieを気にする必要がないように）。次に、CSVをエクスポートするためのリンクを取得してコピーします。これを行うには、キーボードのCtrlキーを押してCtrl + Shift + Jを入力する（つまり、開発者コンソールを開く）だけで、[ネットワーク]タブに移動します。次に、[ファイル]-[ダウンロード]-[CSV]をクリックします。リクエストを確認してリンクをコピーします。

パート2.サーバーの作成


Pythonライブラリがあるので、Djangoで記述します。

次に、特に私のテーブルの構造について少し説明します。 Googleスプレッドシートのテーブルとは異なり、Notionのテーブルには「参照」列があります。これは別のテーブルへのリンクです（私の場合、ユーザーが気に入った機能の説明）。残りは一般的に明確です。データだけの列だけです。

Notionに移動します。Ctrl+ Shift + Jに慣れているため、コンソールを開き、Application-Cookiesに移動して、token_v2をコピーし、TOKENと呼びます。次に、プレートのある必要なページに移動し、リンクをコピーします。 NOTIONを呼び出します。 Relationもある場合は、Relationのページに移動し、リンクをコピーして、たとえば、NOTION_FUNCTIONSを呼び出します。

次に、次のコードを記述します（インポート前の概念）。

def index(request):
    if request.method == "POST":
        client = NotionClient(token_v2=TOKEN)

        database = client.get_collection_view(NOTION)
        current_rows = database.default_query().execute()
        database_functions = client.get_collection_view(NOTION_FUNCTIONS)
        current_rows_functions = database_functions.default_query().execute()

次に、このプレートからすべてのデータを調べ、それらがNotionにまだ追加されているかどうかを確認する必要があります。これを行うために、プレートからデータを要求しました。

def checkTimestamp(rows, timestamp):
    for i in range(0, len(rows)):
        row = rows[i]

        if row.name == timestamp:
            return True

    return False

それとは別に、「row.name」について言及する価値があります。注意深い読者はおそらく尋ねるでしょうから、とにかくそれは何ですか？

これは、Notionの列の名前です（記録時刻が格納されます）。どういうわけか、ロシア語の名前を追加できなかったので、すべての名前を英語に変更して追加しました。

checkEmptinessは、nullアイテムが渡されたかどうかをチェックする関数です。私が彼にゼロのフィールドを与えたとき、なんとかしぶしぶ思い込んでノーションが働いたので、書く価値があります。

公式ドキュメントではこれについては見たことがないので、関係の分析に移りましょう。別のデータベースから行へのリンクを作成するには、（この行の）リンクを取得して転送する必要があります。したがって、別のプレートの文字列へのリンクの配列が暗示されている場合は、それらの識別子の配列を取得する必要があります。個人的に関数名でリレーションを追加しました。

def findIds(current_rows, titles):
    print("titles", titles)
    print("current rows", current_rows)
    array = []

    for a in range(0, len(current_rows)):
        if current_rows[a].name in titles:
            array.append(current_rows[a].id)

    print("Ids", array)
    return array

パート4. Heroku


IFTTT側でこのトリガーをいじっていると思っていましたが、これは支払うものではありません。Herokuは、私たちのものをホストするための無料料金を提供します。主なことは、サービスが少なくとも6時間スリープすることです。そして、私たちは彼に毎分ではなく毎時間働くように呼びかけるので、彼は間違いなく寝ます。



次に、次のことを行います。herokuに移動して、新しいプロジェクトを作成します。次に、クライアントをオペレーティングシステムにインストールします。そして、アプリケーションの作成後に表示された指示に従ってすべてを行います。

herokuのすべてをダウンロードしたら、アプレットに移動し、URLを新しいものに編集します。

これで、リストは1時間ごとに更新されます。仮説的に、IFTTTは何らかの長いリクエストがあるというエラーを出すことがありますが、これはそれほど重要ではありません。
