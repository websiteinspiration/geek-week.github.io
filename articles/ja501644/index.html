CSSアニメーションのパフォーマンス分析

Webページの要素をアニメーション化するために何を選択しますか？JavaScriptまたはCSS？この質問はいつかすべてのWeb開発者に尋ねることを余儀なくされます。そして、多分-かつてではなく。

JavaScriptプログラマは、ブラウザアニメーション用の多くのライブラリを作成しています。そして、周りの誰もがアニメーションの既製のソリューションとしてこれらのライブラリを使用する傾向があったようです。しかし、減速しましょう。それが正しいか？JavaScriptを使用してWebページ要素をアニメーション化する必要がありますか？たぶん、あなたは標準のCSSメカニズムに頼ることができ、それによって高品質で高性能なアニメーションを実現できますか？



あなたはこれを読んでいるので、JavaScriptアニメーションに精通していると思います。したがって、CSSアニメーションのトピックをさまざまな形で探索することを提案します。また、そのようなアニメーションのパフォーマンスについて話すことを提案します。

一般情報


CSSアニメーションは、いくつかの要素のプロパティに基づいています。その中で、次のことに注意してください。

  • position特に、値absoluteおよびを取ることができるプロパティrelative。
  • プロパティtransform。
  • プロパティopacity。
  • プロパティleft、right、top、bottom。

まず、さまざまなCSSプロパティを使用して要素をアニメーション化するいくつかのパイロットプロジェクトを見てください。

ここで、プロジェクト識別子とその要素にanimate持つ特性であるposition値に設定されているアニメーションabsolute。素子の特性topとは、変更の対象となっていますleft。


パイロットプロジェクトページ

以下は、このアニメーションを実装するコードです。

#animate {
    position: absolute;
    top: 100px;
    left: 30px;
    width: 100px;
    height: 100px;
    border-radius: 50%;
    background: red;

    animation: move 3s ease infinite;
}

@keyframes move {
    50% {
        top: 200px;
        left: 130px;
    }
}

これは、CSSプロパティを使用して同じアニメーションが実装されたプロジェクトtransformです。

このアニメーションは次のコードで表されます：

#animate {
    position: absolute;
    top: 100px;
    left: 30px;
    width: 100px;
    height: 100px;
    border-radius: 50%;
    background: red;

    animation: move 3s ease infinite;
}

@keyframes move {
    50% {
        transform: translate(100px, 100px);
    }
}

これらのプロジェクトを使用して、CSSアニメーションのパフォーマンスと、それらを実装するためにどのようなシステムリソースが使用されているかについて結論を出しましょう。

プロジェクトのページを見るだけで、異なるアプローチを使用しているとしても、同じものに何が実装されているかを判断できます。ただし、Chrome開発者ツールを使用してこれらのプロジェクトのパフォーマンスを測定すると、異なることがわかります。

ブラウザはどのようにページを表示しますか？


パフォーマンスの調査に入る前に、ブラウザーがページをどのように表示するか、およびページで変更が発生した場合にブラウザーが実行するアクションを理解する必要があります。

これは、ページ出力プロセスがどのように見えるかです。


ページ出力

この図に反映されているイベントを考えます。

  1. Recalculate Style -要素に適用されるスタイルの計算。
  2. Layout -要素のレイアウトとページ上での配置の作成。
  3. Paint-作成されたレイヤーのピクセルを塗りつぶします。つまり、各レイヤーのラスターイメージを作成します。GPUはこれらの画像を使用してページを表示します。
  4. Composite Layer-レイヤーをレイアウトして画面に表示します。結果は完成したページになります。

イベントComposite Layerとは、CPUがGPUと通信してアニメーションを作成するときのことです。transformおよびのようなCSSプロパティを使用opacityして、アニメーションを生成する負担をCPUからGPUに転送できます。

アニメーションとGPU


GPUは基本的に、画像の操作に使用される専用のコンピューターです。大量のグラフィック情報を効率的に処理できます。

その結果、イベントが発生するComposite Layerと、追加のレイヤーが作成されます。これは、アニメーション化された要素やその他のインターフェース要素の再レンダリングを回避できるメカニズムです。
