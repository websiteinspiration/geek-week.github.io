複雑な企業ネットワークでのZextrasチームの信頼できる運用の確保

最後の記事でZextrasチームについて説明しました-Zimbra Collaboration Suite Open-Source Editionに企業のテキストおよびビデオチャットの機能を追加できるソリューション、およびサードパーティのサービスを必要とせず、データを転送せずに多数の参加者とビデオ会議を行う機能側に。この使用シナリオは、内部ネットワークの形式で厳密に定義されたセキュリティ境界を持ち、この境界を保護することで情報セキュリティを提供できる企業に最適です。ただし、企業の内部ネットワークは、常に単純で理解可能なものとはほど遠いものです。多くの場合、1つの大規模なネットワークには非常に多くの異なるサブネットがあり、その多くは、地理的に離れた支店やオフィスについて話している場合、VPNを介して接続されています。内部ネットワークの複雑な構造は、Zextrasチームでのビデオチャットおよびビデオ会議の正しい動作を妨げる可能性があります。ここで、すべてが正しく機能し、障害が発生しないようにするために何ができるかについて説明します。



Zextras Teamのインストールはできるだけ簡単です。Zextras Suite Proをインストールしたら、管理コンソールからcom_zextras_Team winterをアクティブ化するだけで十分です。その後、企業内のすべてのZimbra OSEユーザーが対応する機能を使用できるようになります。その後、システム管理者は、異なるユーザーグループと個々のアカウントの両方に対してZextrasチームの機能を制限できます。これは、次のコマンドを使用して行われます。

  • zxsuite config teamChatEnabled false
  • zxsuite config historyEnabled false
  • zxsuite config videoChatEnabled

最初のコマンドでは、さまざまなグループまたは個々のユーザーのテキストチャットに関連するいくつかの機能を無効にできます。 2番目のコマンドでは、チャット履歴の保存を無効にできます。このアクションは、すべてのユーザー、特定のサーバーのユーザー、およびさまざまなグループや個々のユーザーに対して実行できます。 3番目のコマンドでは、ビデオチャットに関連する機能を無効にできます。この機能は、グローバルに、別のサーバーで、ユーザーのグループや特定のアカウントに対して無効にすることができます。 

必要なすべての制限が導入された後、管理者は企業のビデオ通信が適切に機能していることを確認することしかできません。 Pext-to-Peer WebRTCテクノロジーはZextrasチームの中心にあるため、その操作には2つの要素が非常に重要です。1つは接続設定の容易さ、もう1つは十分なチャネル帯域幅です。また、管理者が内部ネットワークのチャネル幅や信号品質について心配する必要がない場合は、複雑なネットワークアーキテクチャが企業の従業員間の接続の確立を妨げることがあります。

クライアント間の接続を確立する際の問題を回避するために、Zextras Teamの開発者はTURN-serverをサポートしました。これは、最も分岐した内部ネットワークでさえ、ユーザー間の接続を確立するのに役立ちます。これを行うには、他のドメインから見えるエンタープライズの内部ネットワークに、オンボードのTURNを備えたノードを追加する必要があります。 

たとえば、企業ネットワークの対応するノードの名前がturn.company.ruであるとします。ビデオチャットを作成しようとする場合、Zextrasチームがユーザーの認証データを使用してTURNサーバーにアクセスし、問題がなければ、WebSocketのような接続を確立し、ユーザーが互いに正常に通信できるようにする必要があります。 

TURNサーバーをZextrasチームに接続するために、zxsuite Team iceServer add turn：turn.company.ru：3478？Transport = udp credential password username admin cos defaultの形式のコンソールコマンドを導入します。このチームの場合、新しいTURNサーバーをZextrasチームリストに追加して、ネットワークアドレスと管理者アカウント情報を示し、デフォルトのユーザーグループに割り当てて使用しました。同じ原理を使用して、複数のTURNサーバーを一度に追加して、異なるグループのユーザーが接続に異なるサーバーを使用するようにすることができます。 

新しいTURNサーバーを追加することに加えて、zxsuite Team iceServer remove turn.company.ruコマンドを使用して、追加されたリストからそれらを削除できます。また、zxsuite Team iceServer getコマンドを使用して、追加されたサーバーのリストを表示します  。Zimbra OSEと同じユーザーをTURNサーバーに作成する必要がないことに注意してください。TURNサーバーで快適に作業するには、管理者アカウントのみが必要です。

したがって、ローカルネットワークにTURNサーバーを追加し、小規模なセットアップを行うと、ネットワーク構造に関係なく、Zextrasチームユーザー間の接続が非常に迅速に確立されます。また、内部ネットワークのチャネル幅は、プライベートビデオチャット中とプライベートビデオチャット中の両方で一貫して良好な画像を提供する必要があります。ビデオ会議。
