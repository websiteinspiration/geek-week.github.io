数学とNumpyを使用した単純なニューラルネットワークの記述


ゼロからニューラルネットワークを作成する方法に関する次の記事を読む理由 悲しいかな、理論とコードがゼロから完全に機能するモデルまで記述された記事は見つかりませんでした。私はすぐにたくさんの数学があると警告します。私は、読者が線形代数、偏微分、および少なくとも部分的に確率理論の基本、ならびにPythonおよびNumpyに精通していることを前提としています。完全に接続されたニューラルネットワークとMNISTを扱います。

数学。パート1（シンプル）


完全結合層（FC層）とは何ですか？通常、彼らは「完全に接続された層は層であり、その各ニューロンは前の層のすべてのニューロンに接続されている」という精神の中で何かを言います。特にコードでは、ニューロンとは何か、それらがどのように接続されているかは明確ではありません。ここで、例を使用してこれを解析してみましょう。 100ニューロンの層があるとします。それが何であるかはまだ説明していませんが、100個のニューロンがあり、データが送信される入力と、データを送信する出力があるニューロンを想像してみてください。そして、28x28ピクセルの白黒の画像が入力に送られます-ベクトルに引き伸ばした場合、値は784だけです。画像は入力レイヤーと呼ばれます。次に、100個のニューロンのそれぞれが各「ニューロン」に接続されるようにします。必要に応じて、前のレイヤー（つまり、画像）の値が必要です。100個のニューロンのそれぞれが元の画像の784個の値をとるようにします。たとえば、100個のニューロンのそれぞれについて、画像の784の値に約784の数値を掛けてそれらを加算すると、結果として1つの数値が得られます。つまり、これはニューロンです。

=- 1⋅ 1 ++ ... + - 784⋅ 784


次に、各ニューロンには784個の数値があり、これらすべての数値は（この層のニューロンの数）x（前の層のニューロンの数）= 100×784= 78,400桁。これらの数値は一般にレイヤーの重みと呼ばれます。各ニューロンはその数を出力し、結果として100次元のベクトルを取得します。実際、この100次元のベクトルは、784次元のベクトル（元の画像）にサイズの重み行列を乗算することによって得られると書くことができます。100×784：

x100=W100×784⋅x784



さらに、結果として得られる100個の数値は、アクティベーション関数（一部の非線形関数）に渡され、各数値に個別に影響します。たとえば、シグモイド、双曲線正接、ReLUなどです。活性化関数は必然的に非線形です。それ以外の場合、ニューラルネットワークは単純な変換のみを学習します。



次に、結果のデータは、完全に接続されたレイヤーに、ただしニューロンの数が異なる状態で、再びアクティベーション機能に送られます。これは数回発生します。ネットワークの最後の層は、答えを生成する層です。この場合、答えは写真の数字に関する情報です。



ネットワークのトレーニング中に、画像に表示されている図を知る必要があります。つまり、データセットはマークアップされています。次に、別の要素-エラー関数を使用できます。彼女はニューラルネットワークの応答を見て、実際の答えと比較します。これのおかげで、ニューラルネットワークは学習しています。

問題の概要


データセット全体が大きなテンソルです（多次元データ配列をテンソルと呼びます）。 X=[x1,x2,…,xn]どこ xi-i番目のオブジェクト。たとえば、テンソルでもある画像。各オブジェクトにはyi-i番目のオブジェクトの正解。この場合、ニューラルネットワークは、オブジェクトを入力として受け取り、それにいくつかの答えを返す関数として表すことができます。

F(xi)=y^i


次に、関数を詳しく見てみましょう F(xi)。ニューラルネットワークはレイヤーで構成されているため、個々のレイヤーはそれぞれ機能です。そしてそれは

F(xi)=fk(fk−1(…(f1(xi))))=y^i


つまり、最初の関数-最初のレイヤー-では、テンソルの形式で画像が表示されます。関数f1いくつかの答えを与えます-これもテンソルですが、次元が異なります。このテンソルは内部表現と呼ばれます。この内部表現は関数の入力に送られますf2、その内部表現を提供します。など、関数までfk -最後の層-答えはありません y^i。

次に、ネットワークをトレーニングするタスクは、ネットワークの回答を正しい回答に一致させることです。まず、ニューラルネットワークがどれほど間違っているかを測定する必要があります。これを測定することはエラー関数です。L(y^i,yi)。そして、私たちは制限を課します：

1。y^i→yi⇒L(y^i,yi)→0
2。 ∃ dL(y^i,yi)
3。 L(y^i,yi)≥0

制限2は、レイヤーのすべての機能に課されます fj-それらをすべて区別可能にします。

さらに、実際には（これについては触れませんでした）、これらの関数の一部はパラメーター（ニューラルネットワークの重み）に依存していますfj(xi|ωj)。そして、全体のアイデアは、そのような重みをピックアップして、y^i と一致した yiデータセットのすべてのオブジェクト。すべての関数に重みがあるわけではないことに注意してください。

では、どこで停止したのでしょうか。ニューラルネットワークのすべての関数は微分可能であり、誤差関数も微分可能です。グラデーションのプロパティの1つを思い出してください-関数の成長方向を示します。これを利用して、制限1と3、という事実を

L(F(xi))=L(fk(fk−1(…(f1(xi)))))=L(y^i)


そして、私が複素関数の偏微分と微分を考慮できるという事実。今、あなたが計算するために必要なすべてがあります

∂L(F(xi))∂ωj


iとjの場合。この偏微分は変化する方向を示しますωj拡大する L。減らすには、横に一歩を踏む必要があります−∂L(F(xi))∂ωj何も複雑ではありません。

これは、ネットワーク学習プロセスが次のように構造化されていることを意味します。検討するデータセットオブジェクトごとに、データセット全体（これは時代と呼ばれます）を1サイクルで数回調べますL(y^i,yi) （これはフォワードパスと呼ばれます）そして偏微分を考慮してください ∂L すべての重み ωj、次に重みを更新します（これは逆方向パスと呼ばれます）。

まだ特定の関数とレイヤーを導入していないことに注意してください。この段階でこれをどうするかが明確でない場合は、読み続けることをお勧めします-数学はもっとありますが、今では例を示します。

数学。パート2（難しい）


エラー機能


最後から始めて、分類問題の誤差関数を導出します。回帰問題の場合、エラー関数の導出は、本「ディープラーニング」に詳しく説明されています。ニューラルネットワークの世界への没頭。」

簡単にするために、猫の写真を犬の写真から分離するニューラルネットワーク（NN）があり、正解がある猫と犬の写真のセットがあります。ytrue。

NN(picture|Ω)=ypred


次に行うことはすべて、最尤法とよく似ています。したがって、主なタスクは尤度関数を見つけることです。詳細を省略した場合、ニューラルネットワークの予測と正解を比較し、それらが一致する場合は大きな値が得られ、そうでない場合はその逆になります。正解の確率は、与えられたパラメーターで頭に浮かびます：

p(ypred=ytrue|Ω)


そして今、私たちはいくつかのフェイントを作成しますが、それはどこからも続いていないようです。ニューラルネットワークが2次元ベクトルの形式で答えを出すようにします。その値の合計は1です。このベクトルの最初の要素は、猫が写真に写っているという信頼度の指標であり、2番目の要素は犬が写真に写っているという信頼度の指標です。はい、それはほぼ確率です！

NN(picture|Ω)=[p0p1]


これで、尤度関数を次のように書き換えることができます。

p(ypred=ytrue|Ω)=pΩ(ypred)0t∗(1−pΩ(ypred))1t=p0t0∗p1t1


どこ t0,t1 たとえば、正しいクラスのラベル ytrue=catその後 t0==1,t1==0、 ytrue=dogその後 t0==0,t1==1。したがって、ニューラルネットワークによって予測されたはずである（ただし、必ずしもネットワークによって予測される必要はない）クラスの確率が常に考慮されます。これは、任意の数のクラス（たとえば、mクラス）に一般化できます。

p(ypred=ytrue|Ω)=∏0mpiti


ただし、どのデータセットにも多くのオブジェクト（たとえば、N個のオブジェクト）があります。ニューラルネットワークで、各オブジェクトまたはほとんどのオブジェクトに対して正しい答えを出す必要があります。そのためには、データセットの各オブジェクトについて、上記の式の結果を乗算する必要があります。

MaximumLikelyhood=∏j=0N∏i=0mpi,jti,j


良い結果を得るには、この関数を最大化する必要があります。しかし、最初に、最小化するのがより急になります。これは、確率的勾配降下法とそのためのすべてのバンがあるためです-マイナスを割り当てるだけです。次に、巨大な作業を処理することは困難です-対数です。

CrossEntropyLoss=−∑j=0N∑i=0mti,j⋅log⁡(pi,j)


すごい！その結果、クロスエントロピー、またはバイナリの場合はログロスが発生しました。この機能は数えやすく、区別がさらに簡単です。

∂CrossEntropyLoss∂pj=−tjpj


逆伝播アルゴリズムを区別する必要があります。エラー関数はベクトルの次元を変更しないことに注意してください。MNISTの場合のように、出力が回答の10次元ベクトルである場合、導関数を計算すると、導関数の10次元ベクトルが得られます。別の興味深いことは、導関数の1つの要素だけがゼロにならないことです。ti,j≠0、つまり、正解です。また、このオブジェクトでニューラルネットワークによって予測された正解の確率が低いほど、エラー関数が多くなります。

アクティベーション機能


ニューラルネットワークの完全に接続された各層の出力には、非線形活性化関数が存在する必要があります。それがなければ、意味のあるニューラルネットワークをトレーニングすることは不可能です。将来を見据えて、ニューラルネットワークの完全に接続されたレイヤーは、入力データに重み行列を乗算するだけです。線形代数では、これは線形マッピングと呼ばれます—線形関数。線形関数の組み合わせも線形関数です。しかし、これは、そのような関数が近似できるのは線形関数のみであることを意味します。悲しいかな、これがニューラルネットワークが必要な理由ではありません。

ソフトマックス


通常、この関数はネットワークの最後のレイヤーで使用されます。これは、最後のレイヤーのベクトルを「確率」のベクトルに変換するためです。ベクトルの各要素は0から1までで、それらの合計は1です。ベクトルの次元は変更されません。

Softmaxi=exi∑jexj


それでは、微分検索に移りましょう。なのでx ベクトルであり、そのすべての要素は常に分母に存在するため、導関数を取るとヤコビアンが得られます。

JSoftmax={xi−xi⋅xj,i=j−xi⋅xj,i≠j


次に、逆伝播について説明します。導関数のベクトルは前のレイヤーから来ます（通常、これはエラー関数です）dz。もしdz mnistのエラー関数から来た、 dz-10次元ベクトル。この場合、ヤコビ行列の次元は10x10です。入手するにはdznew、これは前のレイヤーにさらに進みます（エラーが戻ったときに、ネットワークの終わりから始まりに戻ることを忘れないでください）。 dz に JSoftmax （列ごとの行）：

dznew=dz×JSoftmax


出力では、導関数の10次元ベクトルを取得します。 dznew。

レル


ReLU(x)={x,x>00,x<0


ReLUは、2011年に記事「Deep Sparse Rectifier Neural Networks」が公開されてから、広く使用されるようになりました。ただし、そのような機能は以前から知られていました。「アクティベーションパワー」の概念はReLUに適用されます（詳細については、「ディープラーニング、ニューラルネットワークの世界への没入」を参照してください）。しかし、ReLUを他のアクティベーション関数より魅力的にする主な機能は、その単純な微分計算です。

d(ReLU(x))={1,x>00,x<0


したがって、ReLUは他のアクティブ化関数（シグモイド、双曲線正接など）よりも計算上効率的です。

完全に接続されたレイヤー


今こそ、完全に接続されたレイヤーについて議論する時です。他のすべての中で最も重要なのは、ニューラルネットワークを適切に機能させるために構成する必要があるすべての重みがあるのがこのレイヤーにあるためです。完全に接続されたレイヤーは、単にウェイトマトリックスです。

W=|wi,j|


重み行列が入力列で乗算されると、新しい内部表現が取得されます。

xnew=W⋅x


どこ x サイズあり input_shape、および xnew - output_shape。例えば、x -784次元ベクトル、および xnew100次元ベクトルの場合、行列Wのサイズは100x784です。このレイヤーのウェイトは100x784 = 78,400であることがわかります。

エラーの逆伝播では、この行列の各重みに関する微分を取る必要があります。問題を単純化し、微分に関してのみ取るw1,1。行列とベクトルを乗算するとき、新しいベクトルの最初の要素xnew 等しい xnew 1=w1,1⋅x1+...+w1,784⋅x784、および派生物 xnew 1 沿って w1,1 簡単になります x1、上記の金額のデリバティブを取得する必要があるだけです。他のすべての重みについても同様です。しかし、これが単なる導関数の行列である限り、これはエラー逆伝播アルゴリズムではありません。次のレイヤーからこれまで（エラーは最後から最初へ）に100次元の勾配ベクトルが来ることを覚えておく必要があります。dz。このベクトルの最初の要素dz1 作成に「参加」した導関数の行列のすべての要素が乗算されます xnew 1、つまり x1,x2,...,x784。同様に、残りの要素。これを線形代数の言語に変換すると、次のようになります。

∂L∂W=(dz, dW)=(dz1⋅x...dz100⋅x)100


出力は100x784マトリックスです。


次に、前のレイヤーに何を転送するかを理解する必要があります。これと今何が起こったのかをよりよく理解するために、この層の派生物を少し異なる言語で受け取ったときに何が起こったかを書き留めて、「乗算されるもの」の詳細から関数に（もう一度）触れないようにします。

重みを調整したいとき、これらの重みの誤差関数の導関数を取得したいと思いました。∂L∂W。エラー関数とアクティベーション関数の導関数を取る方法を上に示しました。したがって、このような場合を考えることができます（dz エラー関数とアクティブ化関数のすべての派生物はすでに存在しています）：

∂L∂W=dz⋅∂xnew(W)∂W


これは可能です。 xnewWの関数として： xnew=W⋅x。
これを上記の式に代入できます。

∂L∂W=dz⋅∂W⋅x∂W=dz⋅E⋅x


ここで、Eは単位で構成される行列です（単位行列ではありません）。

ここで、前のレイヤーからの導関数を取得する必要がある場合（計算を簡単にするために、それは完全に接続されたレイヤーになることもありますが、一般的には何も変更されません）、次に考慮する必要がありますx前のレイヤーの関数として x(Wold)：

∂L∂Wold=dz⋅∂xnew(W)∂Wold=dz⋅∂W⋅x(Wold)∂Wold==dz⋅∂W⋅Wold⋅xold∂Wold=dz⋅W⋅E⋅xold==dznew⋅E⋅xold


丁度 dznew=dz⋅W そしてあなたは前のレイヤーに送る必要があります。

この記事は、主にニューラルネットワークの数学を説明することを目的としています。コードにはほとんど時間をかけません。

これはエラー関数の実装例です：

class CrossEntropy:
    def forward(self, y_true, y_hat):
        self.y_hat = y_hat
        self.y_true = y_true
        self.loss = -np.sum(self.y_true * np.log(y_hat))
        return self.loss

    def backward(self):
        dz = -self.y_true / self.y_hat
        return dz

このクラスには、ダイレクトパスとリバースパスのメソッドがあります。直接パスのとき、クラスインスタンスはレイヤー内にデータを格納し、戻りパスのとき、それらを使用してグラデーションを計算します。残りのレイヤーも同じ方法で作成されます。これのおかげで、このスタイルで完全に接続されたニューラルを書くことが可能になります：
