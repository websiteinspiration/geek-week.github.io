1.Elasticスタック：セキュリティログ分析。前書き



Splunkロギングおよび分析システムのロシアでの販売終了に関連して、このソリューションをどのように置き換えることができるかという疑問が生じましたか？さまざまなソリューションに慣れるために時間を費やして、私は本当の人のためのソリューション「ELKスタック」に落ち着きました。このシステムは構成に時間がかかりますが、結果として、組織内の情報セキュリティインシデントに対するステータスと運用上の応答を分析するための非常に強力なシステムを得ることができます。このシリーズの記事では、ELKスタックの基本的な（またはそうでない可能性がある）機能、ログの解析方法、グラフとダッシュボードの作成方法、および例としてCheck PointファイアウォールまたはOpenVasセキュリティスキャナーからのログを使用して実行できる興味深い機能について検討します。まず、ELKスタックとは何か、およびELKスタックを構成するコンポーネントについて検討します。

ELKスタックは：3つのオープンソースプロジェクトの略称でElasticsearch、Logstash、およびKibana。すべての関連プロジェクトとともに、Elasticによって開発されました。Elasticsearchは、データベース、検索、分析システムの機能を組み合わせたシステム全体の中核です。Logstashはサーバー側のデータ処理パイプラインであり、同時に複数のソースからデータを受信し、ログを解析して、Elasticsearchデータベースに送信します。Kibanaを使用すると、Elasticsearchのチャートやグラフを使用してデータを視覚化できます。Kibanaを使用してデータベースを管理することもできます。次に、各システムを個別に詳細に検討します。



Logstash


Logstashは、さまざまなソースからのログイベントを処理するためのユーティリティであり、メッセージのフィールドとその値を選択できます。また、データのフィルタリングと編集を構成することもできます。すべての操作の後、Logstashはイベントを最終的なデータストアにリダイレクトします。ユーティリティは、構成ファイルを介してのみ構成されます。
典型的なlogstash構成は、いくつかの着信情報ストリーム（入力）、この情報用のいくつかのフィルター（フィルター）、およびいくつかの発信ストリーム（出力）で構成されるファイルです。これは、1つ以上の構成ファイルのように見えます。最も単純なバージョン（まったく何もしない）では、次のようになります。

input {
}

filter {
}

output {
}

INPUTでは、ログがどのポートに入るか、どのプロトコル、またはどのフォルダーから新しいファイルまたは永続的に記録されたファイルを読み取るかを構成します。FILTERでは、フィールドの解析、値の編集、新しいパラメーターの追加、削除などのログパーサーを構成します。FILTERは、大量の編集オプションを備えたLogstashに付属するメッセージ管理フィールドです。出力では、すでに解析済みのログを送信する場所を構成します。それがelasticsearchの場合、値を持つフィールドが送信されるJSONリクエストが送信されます。または、デバッグの一部として、標準出力に出力するか、ファイルに書き込むことができます。



Elasticsearch


最初はElasticsearchは全文検索ソリューションですが、簡単なスケーリング、レプリケーションなどの追加機能を備えているため、大量のデータを使用する高負荷プロジェクトに非常に便利で優れたソリューションとなっています。 Elasticsearchは、JSON形式のドキュメントの非リレーショナルリポジトリ（NoSQL）であり、Lucene全文検索に基づく検索エンジンです。ハードウェアプラットフォームはJava仮想マシンであるため、システムが動作するには大量のプロセッサとRAMリソースが必要です。
Logstashを使用するか、クエリAPIを使用して、受信メッセージはそれぞれ「ドキュメント」としてインデックス付けされます。これは、リレーショナルSQLのテーブルに類似しています。すべてのドキュメントはインデックスに保存されます-SQLのデータベースの類似物。

データベース内のドキュメントの例：

{
  "_index": "checkpoint-2019.10.10",
  "_type": "_doc",
  "_id": "yvNZcWwBygXz5W1aycBy",
  "_version": 1,
  "_score": null,
  "_source": {
	"layer_uuid": [
      "dae7f01c-4c98-4c3a-a643-bfbb8fcf40f0",
      "dbee3718-cf2f-4de0-8681-529cb75be9a6"
	],
	"outzone": "External",
	"layer_name": [
  	"TSS-Standard Security",
  	"TSS-Standard Application"
	],
	"time": "1565269565",
	"dst": "103.5.198.210",
	"parent_rule": "0",
	"host": "10.10.10.250",
	"ifname": "eth6",
    ]
}

データベースでのすべての作業は、REST APIを使用したJSONリクエストに基づいており、インデックスによってドキュメントを発行するか、質問-回答という形式の統計情報を発行します。リクエストに対するすべての回答を視覚化するために、WebサービスであるKibanaが作成されました。
