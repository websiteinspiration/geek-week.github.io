Node.jsとOracleデータベースを使用したREST APIの作成。パート2

パート2：REST APIの作成：データベースの基本


では最初の記事、あなたはここで、あなたが起動し、責任があるモジュールを作成し、Webサーバを作成したシャットダウン使用してデータベース接続プールノード-ORACLEDBを。また、プールから接続を自動的に取得および解放することにより、単純なステートメントの実行を簡素化する機能を追加します。

実行中の接続プール


node-oracledbはOCIクライアントライブラリの上に構築されているため、クライアント側で優れたパフォーマンス特性を持つOCIプールを作成するためのサポートが組み込まれています。接続プールを作成するには、と呼ばれる新しい設定ファイルの作成から始めdatabase.js内のconfigディレクトリを。次のコードをコピーしてファイルに貼り付け、変更を保存します。

module.exports = {
  hrPool: {
    user: process.env.HR_USER,
    password: process.env.HR_PASSWORD,
    connectString: process.env.HR_CONNECTIONSTRING,
    poolMin: 10,
    poolMax: 10,
    poolIncrement: 0
  }
};

同じように/ webserver.js設定ファイルを、このファイルには、環境変数を使用して、いくつかのプロパティを設定することができます。環境変数を使用すると、アプリケーションをさまざまな環境にデプロイするときに柔軟性が得られ、パスワードやその他の機密情報をソースコードから保護できます。端末から次のコマンドを実行して必要な環境変数を設定し、それらが将来の端末セッションで使用できることを確認します。

echo "export HR_USER=hr" >> ~/.bashrc
echo "export HR_PASSWORD=oracle" >> ~/.bashrc
echo "export HR_CONNECTIONSTRING=0.0.0.0/orcl" >> ~/.bashrc
source ~/.bashrc

poolMinとpoolMaxが同じで、poolIncrementが0に設定されていることに気付くかもしれません。これにより、必要な管理リソースが少ない固定サイズのプールが作成されます。

Node.jsはしばしば「シングルスレッド」と表現されますが、JavaScriptコードを実行するメインスレッドをブロックする特定の操作に使用できるスレッドプールがあります。このスレッドプールは、node-oracledbによって、接続の受信やSQLおよびPL / SQLコードの実行などのすべての非同期操作を実行するために使用されます。ただし、デフォルトのスレッドプールサイズは4です。プール内の10個の接続すべてを同時に機能させる場合は、それに応じてスレッド数を増やす必要があります。

UV_THREADPOOL_SIZE環境変数を使用して、スレッドプールサイズを調整できます。 UV_THREADPOOL_SIZEは、Node.jsアプリケーションを実行する前または内部で設定できますが、スレッドプールを使用して最初の呼び出しを行う前に設定する必要があります。これは、スレッドプールが最初の使用時に作成され、作成後にサイズが固定されるためです。アプリケーションのルートにあるindex.jsファイルを開き、最初の行（Webサーバーモジュールを含む）の後に次の行を追加します。

// *** line that requires services/web-server.js is here ***
const dbConfig = require('./config/database.js');
const defaultThreadPoolSize = 4;
 
// Increase thread pool size by poolMax
process.env.UV_THREADPOOL_SIZE = dbConfig.hrPool.poolMax + defaultThreadPoolSize;

スレッドプールが適切なサイズになったので、データベースモジュールに移動できます。servicesディレクトリにdatabase.jsという名前の新しいファイルを作成します。次のコードをコピーして貼り付け、変更を保存します。
const oracledb = require('oracledb');
const dbConfig = require('../config/database.js');
 
async function initialize() {
  const pool = await oracledb.createPool(dbConfig.hrPool);
}
 
module.exports.initialize = initialize;

このモジュールでは、最初にnode-oracledbと構成ファイルを紹介します。次に、initializeという名前で非同期関数が定義され、module.exportsオブジェクトを通じて提供されます。初期化関数は、デフォルトのプールとして内部接続プールキャッシュに格納される接続プールを作成します。

次に、Webサーバーが開く前に接続プールが開始されるように、すべてを接続する必要があります。index.jsに戻り、1行目の下に次の行を追加します。
//  *** line that requires services/web-server.js is here ***
const database = require('./services/database.js');

次に、Webサーバーを起動する既存のtryブロックの直前に、次のtryブロックをstratup関数に追加します。

try {
    console.log('Initializing database module');
 
    await database.initialize(); 
  } catch (err) {
    console.error(err);
 
    process.exit(1); // Non-zero failure code
  }
  
  // *** existing try block in startup here ***

この時点で、node-oracledbをインストールしてコードをテストできます。ターミナルでhr_appディレクトリから次のコマンドを実行します。

npm install oracledb -s
node .

データベースモジュールとWebサーバーが実行中であるというメッセージが表示された場合は、おめでとうございます。接続プールが機能しています。

接続プールのシャットダウン


今すぐアプリケーションを閉じると（以前と同様にctrl + cを使用）、接続プールが閉じる前にNode.jsプロセスが破棄されます。関連するすべてのデータベースプロセスを自動的にクリーンアップする必要がありますが、Node.jsプロセスを終了する前に接続プールを明示的に閉じることをお勧めします。services / database.js

ファイルに戻り、以下のコード行を最後に追加して、更新を保存します。

// *** previous code above this line ***
 
async function close() {
  await oracledb.getPool().close();
}
 
module.exports.close = close;

close関数は、oracledb.getPool（）メソッドを使用してデフォルトのプールを同期的に取得し、プールのcloseメソッドを呼び出して閉じます。

適切なタイミングでclose関数を呼び出すには、Webサーバーを停止する既存のtryブロックの直後に、シャットダウン関数内のindex.jsファイルに次のコード行を追加します。

// *** existing try-catch block in shutdown here ***
 
  try {
    console.log('Closing database module');
 
    await database.close(); 
  } catch (err) {
    console.log('Encountered error', e);
 
    err = err || e;
  }

アプリケーションを再度起動して閉じると、Webサーバーを閉じた後、プロセスが完了する前にデータベースモジュールが閉じていることがわかります。

単純なCRUD操作を簡素化する


node-oracledbを使用したSQLまたはPL / SQLコードの実行は、通常、3つのステップのプロセスです。接続を取得し、コードを実行してから、接続を解放します。実行する呼び出しが1つだけである場合（マルチステップトランザクションは必要ありません）、接続の受信と解放は標準コードのようになります。 3つの操作すべてを1回の呼び出しで実行する関数を作成したいと思います。services / database.jsファイルに戻り、以下のコードを追加して変更を保存します。

// *** previous code above this line ***
 
function simpleExecute(statement, binds = [], opts = {}) {
  return new Promise(async (resolve, reject) => {
    let conn;
 
    opts.outFormat = oracledb.OBJECT;
    opts.autoCommit = true;
 
    try {
      conn = await oracledb.getConnection();
 
      const result = await conn.execute(statement, binds, opts);
 
      resolve(result);
    } catch (err) {
      reject(err);
    } finally {
      if (conn) { // conn assignment worked, need to close
        try {
          await conn.close();
        } catch (err) {
          console.log(err);
        }
      }
    }
  });
}
 
module.exports.simpleExecute = simpleExecute;

通常、Webサーバーモジュールではデータベースモジュールを使用しませんが、ここで追加して、正しく機能していることを確認します。services / web-server.jsファイルを開き、上部の既存の定数宣言の下に次の行を追加します。

// line that requires ../config/web-server.js here
const database = require('./database.js');

次に、次のコードを使用して、「Hello World！」で応答するapp.getハンドラ全体を置き換えます。（すべて3行）。

// *** line that adds morgan to app here ***
 
    app.get('/', async (req, res) => {
      const result = await database.simpleExecute('select user, systimestamp from dual');
      const user = result.rows[0].USER;
      const date = result.rows[0].SYSTIMESTAMP;
 
      res.end(`DB user: ${user}\nDate: ${date}`);
    });

新しいハンドラーは、データベースモジュールのsimpleExecute関数を使用して、現在のユーザーとsystimestampの値をデータベースから取得します。次に、値がテンプレートリテラルで使用され、動的メッセージでクライアントに応答します。
