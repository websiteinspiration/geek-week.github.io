GraalVMでのユーティリティの開発

問題の定式化


定期的に、たとえばプロジェクトの同僚と、ローカルネットワーク上のファイルを共有するタスクがあります。


これには多くの解決策があります-Samba / FTP / scp。ファイルをGoogleドライブなどの公共の場所にアップロードしたり、Jiraのタスクに添付したり、メールで送信したりできます。


しかし、これはある程度、柔軟性がなく、事前調整が必要であり、独自の制限（最大投資額など）がある場合もあります。


そして、もっと軽量で柔軟なものを求めています。


Linuxで実用的なソリューションをすばやく構築できる機会を利用できることに驚かされました。


たとえば、次の単一行でシステムpythonを使用して前述の問題をしばしば解決しました


$ python3 -mhttp.server
Serving HTTP on 0.0.0.0 port 8000 ...

このコマンドは、現在のフォルダーでWebサーバーを起動し、ファイルのリストを取得して、Webインターフェースからそれらをダウンロードできるようにします。これらのものの多くはここにダンプすることができます。


いくつかの不便があります。ここで、ダウンロードリンクを同僚に転送するには、ネットワーク上のIPアドレスを知っている必要があります。


コマンドを使用すると便利です


$ ifconfig -a

次に、表示されるネットワークインターフェイスのリストから適切なインターフェイスを選択し、http：// IP：8000という形式のリンクを手動で作成します。これは送信できます。


2番目の不便：このサーバーはシングルスレッドです。つまり、同僚の1人がファイルをダウンロードしている間、2人目の同僚はファイルのリストをダウンロードすることさえできません。
