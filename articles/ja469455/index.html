NginxからEnvoyプロキシへの移行

こんにちは、ハブル！NginxからEnvoy Proxyへの移行という投稿の翻訳に注目します。


Envoyは、個々のサービスおよびアプリケーション用に設計された高性能分散プロキシサーバー（C ++で記述）であり、通信バスおよび大規模なマイクロサービス「サービスメッシュ」アーキテクチャ用に設計された「ユニバーサルデータプレーン」でもあります。作成時には、NGINX、HAProxy、ハードウェアロードバランサー、クラウドロードバランサーなどのサーバーの開発中に発生した問題の解決策が考慮されました。Envoyはすべてのアプリケーションと連携してネットワークを抽象化し、プラットフォームに関係なく共通の機能を提供します。インフラストラクチャ内のすべてのオフィストラフィックがEnvoyグリッドを通過すると、一貫した観察可能性、全体的なパフォーマンスの調整、および特定の場所での基本機能の追加により、問題領域を簡単に視覚化できます。


能力


  • プロセス外のアーキテクチャ：envoyは、少量のRAMを消費するスタンドアロンの高性能サーバーです。任意のアプリケーション言語またはフレームワークと連携して動作します。
  • http / 2とgrpcのサポート：envoyは、インバウンド接続とアウトバウンド接続のhttp / 2とgrpcをファーストクラスでサポートしています。これは、http / 1.1からhttp / 2への透過プロキシです。
  • 拡張されたロードバランシング：envoyは、自動再試行、開回路、グローバル速度制限、シャドウイングリクエスト、ローカルゾーンロードバランシングなどの高度なロードバランシング機能をサポートしています。
  • 構成管理API：envoyは、構成を動的に管理するための堅牢なAPIを提供します。
