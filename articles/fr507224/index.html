Comment éliminer les angles morts avec des tests visuels

Une traduction de l'article a été préparée avant le début du cours Python QA Engineer .


Les erreurs visuelles sont des erreurs dans la présentation de l'application. Ils apparaissent tout au long de la vie de l'application et apparaissent très souvent lorsque l'application est affichée sur les écrans d'appareils mobiles de différentes tailles (sur ordinateurs portables, téléphones, tablettes, montres).

Le pire aspect des erreurs visuelles est qu'elles ne peuvent pas être «détectées» à l'aide de tests automatisés conventionnels. En effet, la plupart des outils de test automatisés vérifient le DOM (modèle d'objet de document) pour signaler l'état de l'application.

Par exemple, l'application de ce tweet n'affiche pas le dernier chiffre du prix, cependant, le prix complet est dans le DOM. Dans le premier cas, le prix réel est de 1250 dollars néo-zélandais, mais il est affiché comme 125 dollars néo-zélandais. Très grande différence!



Malheureusement, les tests automatisés vous diront que tout est en ordre et que le prix est correct, même si les utilisateurs voient une image différente. Aujourd'hui, nous allons parler de la façon de détecter ces erreurs à l' aide de tests visuels .

Qu'est-ce qu'un test visuel?


Afin de détecter les erreurs visuelles, nos scripts d'automatisation nécessitent des «yeux», c'est-à-dire la possibilité de voir l'application telle que les utilisateurs la voient, afin de vérifier son apparence.

Les outils de test visuel nous permettent de faire exactement cela. Ils prennent une capture d'écran de votre application lorsqu'elle prend l'état souhaité, puis prennent une nouvelle capture d'écran chaque fois que vous exécutez le test.

Les captures d'écran des tests de régression sont comparées à la capture d'écran «correcte». Si des incompatibilités importantes sont détectées, le test échoue.

Certains de ces outils utilisent l'intelligence artificielle pour afficher des captures d'écran afin de ressembler davantage à des personnes. Ils sont beaucoup plus efficaces que les méthodes de comparaison d'images de base basées sur les pixels.

Dois-je tout recommencer?


Lorsque je parle aux ingénieurs des avantages des tests visuels, ils me demandent souvent s'ils devront maintenant «jeter» tous les tests existants et recommencer. La réponse est non.

Les tests visuels peuvent être intégrés à votre système de test existant. De nombreux outils de test visuel prennent en charge tous les principaux langages de programmation, ainsi que les cadres d'automatisation tels que Selenium WebDriver, Cypress, Appium et autres.

Malgré le fait que les assertions visuelles complètent vos assertions fonctionnelles existantes, l'avantage du test visuel est qu'il comporte également de nombreux tests fonctionnels .

Par exemple, dans l'exemple de la suite Wellington ci-dessus, les tests actuels incluent très probablement des assertions de prix. Bien sûr, vous pouvez les laisser et leur ajouter des tests visuels pour vous assurer que les prix sont affichés correctement. Une comparaison faite par des algorithmes de tests visuels garantit déjà que le prix est correct et affiché correctement. En conséquence, il s'avère que pour les tests, vous devez écrire moins de code et moins il y a de code, plus il est facile à maintenir.

Que se passe-t-il si je ne veux pas prendre une capture d'écran de tout l'écran?


Malgré le fait qu'une capture d'écran de tout l'écran peut vous aider à trouver les erreurs auxquelles vous ne pouviez même pas penser, cette approche des tests visuels n'est pas toujours vraie. Parfois, l'écran que vous testez est encore en cours de développement et change donc souvent, ou il y a des données sur l'écran qui ne sont pas pertinentes pour vos tests.

Ignorer les étendues


Certains outils de test visuel vous permettent d'ignorer des zones spécifiques de l'écran. Cette méthode convient aux domaines qui ne sont pas liés aux tests, par exemple, les barres d'état, les lignes de date, la publicité, etc.



Cadres de zone


Il arrive qu'il n'y ait pas de zone spécifique qui doive être ignorée, mais il y a une zone qui doit être testée, et ici vous viendrez également à l'aide de tests visuels. En annotant une capture d'écran ou en définissant le localisateur de zone dans le code, vous ne pouvez vérifier qu'une zone limitée.

Soyez prudent avec les limites


Vérifier l'intégralité de l'écran garantira qu'il ne manque rien, car, en fait, une telle vérification comprend les éléments que vous ne pourriez pas couvrir avec des tests. Lors de la réduction de la portée des tests visuels, une attention particulière doit être accordée aux zones entre les limites que vous pourriez avoir manquées.

Je conseille d'élargir la zone de test autant que possible (par exemple, en prenant une section entière, au lieu d'un élément spécifique dans une section), et, si nécessaire, combinez l'affirmation visuelle avec les assertions traditionnelles.

Qu'en est-il des données dynamiques qui varient d'un test à l'autre?


Vous pouvez penser que le test visuel n'est pas applicable lorsque les données changent d'un test à l'autre, mais ce n'est pas le cas. En utilisant l'intelligence artificielle, les tests visuels peuvent reconnaître les modèles qui composent la disposition de l'écran, puis utiliser un algorithme de comparaison différent qui vérifiera que la disposition reste inchangée pendant la régression, quelles que soient les données qu'il contient.

Cette méthode est idéale pour les applications comme les sites d'actualités, les réseaux sociaux et tout autre où les données ne sont pas statiques.

Les tests visuels s'appliquent-ils uniquement aux applications Web?


Les tests visuels ne se limitent pas aux applications Web. Les tests visuels peuvent être utilisés pour les applications mobiles , les systèmes de conception tels que le livre de contes, les fichiers PDF et les images individuelles.

Les tests multiplateformes peuvent également être effectués à l'aide de grilles visuelles. Ces grilles rendent les tests visuels plus rapides et plus intelligents, vous permettant d'exécuter des tests sur de nombreux appareils avec différentes tailles d'écran, où les erreurs sont principalement cachées!

Les grilles visuelles diffèrent des grilles normales en ce qu'elles n'ont pas besoin d'exécuter l'intégralité du script pour chaque configuration. Au lieu de cela, la grille visuelle exécute le script sur une configuration, capture l'état de votre application, puis duplique cet état particulier sur toutes les configurations prises en charge, économisant ainsi du temps et des efforts lors de l'écriture et de l'exécution de tests multiplateformes.

Comment démarrer avec les tests visuels


Il existe de nombreux outils gratuits que vous pouvez commencer à utiliser dès maintenant. Lorsque vous choisissez l'un d'entre eux, assurez-vous que l'outil offre la flexibilité qui répond aux spécificités de vos tests.
