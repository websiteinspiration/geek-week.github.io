コードを見てみましょう：


local function onGlobalCollision( event )
  if ( event.phase == "began" ) then

    local ball = event.object1
    local u, v = ball:getLinearVelocity()
    local speed = u*u + v*v
--    print( "speed: " .. speed  )
     if speed>10000 then
       if speed<20000 then
         audio.play( rockSound )
       else
         audio.play( woodenSound )
       end
    end
  end
end

ご覧のとおり、onGlobalCollision関数では、コロボックの衝突をキャッチし、衝突によって2種類のサウンドを失います。したがって、いくつかのレベルでは、正弦は2に等しく、500を超えるオブジェクトが同時に（または同時に？グリシュコベッツ）衝突しました。このまれな瞬間に、衝突の中でサウンドを呼び出すと、ランタイムでひどいクラッシュが発生します。<kznm！音を消さないでよね？ 直し方？onGlobalCollision関数から直接play（サウンド）を呼び出さないことにしました

、変数boom1とboom2に衝突の数を累積します。次に、runTimeLoopサイクルで（この関数はゲーム内で1秒あたり10回呼び出されます。理由はわかりません）、boom1とboom2を確認し、必要に応じてサウンドを再生します。

       if boom1>0 then
          audio.play( rockSound )
          boom1=0
       end

エラーは消えましたが、音が不快にリズミカルになりました（理解できる、1秒に10回）
次に、時間遅延ノイズを入力しました。100ミリ秒のランダムな遅延-そしてオッパ！すべてが非常に自然に聞こえ始めました：


local function play1()
  audio.play( rockSound )
end

local function play2()
  audio.play( woodenSound )
end

local function playSound()
  local t = math.random(100)
  if boom1>0 then
    boom1=0
    timer.performWithDelay(t, play1)
  end
  if boom2>0 then
    boom2=0
    timer.performWithDelay(t, play2)
  end
end
