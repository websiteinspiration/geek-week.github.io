gogrep: structural search and replacement of Go code

gogrep is one of my favorite utilities for working with Go. It allows you to find code by syntactic patterns, filter the results by type of expression, and perform replacements (also by pattern).


In this article I will tell you how to use it gogrep, as well as the VS Code extension for more convenient working gogrepdirectly from the editor.



Why gogrep is needed


If in the abstract, it gogrepcan be useful for:


  • Refactoring
  • Code base study
  • Search for suspicious code (example: ruleguard )

Consider an example that demonstrates the elegance and effectiveness of structural search.


Functions a()and b()perform the same operations:


func a(xs []int) []int {
  xs = append(xs, 1)
  xs = append(xs, 2)
  return xs
}

func b(xs []int) []int {
  xs = append(xs, 1, 2)
  return xs
}

Suppose we want to rewrite all the places where calls appendcan be collapsed.


Let's try gogrep:


  • Find all matching pairs using the -xtemplate.$x=append($x,$a); $x=append($x,$b)
  • Through the -stemplate $x=append($x,$a,$b)we get the desired replacement
  • By passing an argument -wall affected files will be updated.

gogrep -w -x '$x=append($x,$a);$x=append($x,$b)' -s '$x=append($x,$a,$b)' ./...
