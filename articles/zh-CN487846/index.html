RFC（征求意见），以改进Ember Registry API


Chris Cricho（@chriskrycho）建议为所有注册表接口添加一个新的面向对象的API。使用基于字符串的微语法声明当前的API 并引入一种属性capabilities来安全支持现有的识别引擎。


RFC：增强的Ember Registry API描述了当前存在的微语法问题以及缺乏TypeScript支持的问题。RFC建议添加一个新的API标识符，该标识符将包含字段name，type并且可以选择namespace替代当前的微语法。例如：


代替


getOwner(this).lookup('service:session');

将会


getOwner(this).lookup({ type: 'service', name: 'session' })

但反而


class Example {
  @service('global@session')
  session;
}

将会


class Example {
  @service({ namespace: 'global', name: 'session' })
  session;
}

如果您想了解更多信息或添加自己的评论，请查看GitHub上的RFC！
