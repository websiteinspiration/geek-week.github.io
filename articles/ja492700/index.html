Payment Request APIとAngularを使用したインターネット宣言型ショッピング

Google Pay、Apple Pay、またはブラウザの事前定義されたカードを使用して、ワンクリックでウェブサイトでどれくらいの期間支払いましたか？

私はめったにそれを得ません。

それとは正反対に、すべての新しいオンラインストアは私に別の型を提供しています。また、カードからサイトにデータを再印刷するために、常にカードを忠実に検索する必要があります。次の日、私は別の店で何かを支払い、このプロセスを繰り返したいと思います。

これはあまり便利ではありません。特に代替案について知っている場合：ここ数年で、Payment Request API標準により、最新のブラウザーでこの問題を簡単に解決できます。

それが使用されない理由を理解し、それを使用して作業を簡略化してみましょう。



あなたは何について話していますか？


最近のほとんどすべてのブラウザーは、支払い要求API標準を実装しています。これにより、ブラウザーでモーダルウィンドウを呼び出して、ユーザーが数秒で支払いを行うことができます。これは、ブラウザから通常のカードを使用してChromeで表示する方法です



。ApplePayを介して指紋で支払う場合は、Safariに



表示されます。高速であるだけでなく、機能的でもあります。ウィンドウでは、注文全体や内部の個々の商品やサービスに関する情報を表示できます彼は、顧客情報と配達の詳細を明確にすることができます。提供されたAPIの利便性については議論の余地がありますが、これらはすべてリクエストの作成時にカスタマイズされます。

Angularでどのように使用しますか？


AngularはPayment Request APIを使用するための抽象化を提供していません。 Angularのボックスからそれを使用する最も安全な方法は、依存性注入メカニズムからドキュメントを取得し、そこからウィンドウオブジェクトを取得して、window.PaymentRequestを操作することです。

import {DOCUMENT} from '@angular/common';
import {Inject, Injectable} from '@angular/core';
 
@Injectable()
export class PaymentService {
   constructor(
       @Inject(DOCUMENT)
       private readonly documentRef: Document,
   ) {}
 
   pay(
       methodData: PaymentMethodData[],
       details: PaymentDetailsInit,
       options: PaymentOptions = {},
   ): Promise<PaymentResponse> {
       if (
           this.documentRef.defaultView === null ||
           !('PaymentRequest' in this.documentRef.defaultView)
       ) {
           return Promise.reject(new Error('PaymentRequest is not supported'));
       }
 
       const gateway = new PaymentRequest(methodData, details, options);
 
       return gateway
           .canMakePayment()
           .then(canPay =>
               canPay
                   ? gateway.show()
                   : Promise.reject(
                         new Error('Payment Request cannot make the payment'),
                     ),
           );
   }
}

支払い要求を直接使用すると、暗黙的な依存関係のすべての問題が発生します。コードのテストが困難になると、支払い要求が存在しないため、アプリケーションがSSRで爆発します。抽象化のないグローバルオブジェクトを期待しています。@ ng-web-apis / commonからWINDOW

トークンを取得して、DIからグローバルオブジェクトを安全に取得できます。次に、新しいPAYMENT_REQUEST_SUPPORTを追加します。これにより、Payment Request APIのサポートを確認してから使用できるようになり、サポートしていない環境で誤ってAPIを呼び出すことはなくなります。

export const PAYMENT_REQUEST_SUPPORT = new InjectionToken<boolean>(
   'Is Payment Request Api supported?',
   {
       factory: () => !!inject(WINDOW).PaymentRequest,
   },
);


export class PaymentRequestService {
   constructor(
       @Inject(PAYMENT_REQUEST_SUPPORT) private readonly supported: boolean,
       ...
    ) {}
 
    request(...): Promise<PaymentResponse> {
       if (!this.supported) {
           return Promise.reject(
               new Error('Payment Request is not supported in your browser'),
           );
       } 
      ...
   }

Angularスタイルで書いてみましょう


上記のアプローチを使用すると、支払いを非常に安全に処理できますが、使いやすさはベアAPIブラウザーと同じレベルのままです。3つのパラメーターを使用してメソッドを呼び出し、大量のデータを収集して、目的の形式にして最終的に呼び出します支払方法。

しかし、Angularの世界では、依存関係の注入メカニズム、サービス、ディレクティブ、ストリームなどの便利な抽象化に慣れています。Payment Request APIをより速く、より簡単に使用できるようにする宣言型ソリューションを見てみましょう



。この例では、バスケットは次のようになります。

<div waPayment [paymentTotal]="total">
   <div
       *ngFor="let cartItem of shippingCart"
       waPaymentItem
       [paymentLabel]="cartItem.label"
       [paymentAmount]="cartItem.amount"
   >
       {{ cartItem.label }} ({{ cartItem.amount.value }} {{ cartItem.amount.currency }})
   </div>
 
   <b>Total:</b>  {{ totalSum }} ₽
 
   <button
       [disabled]="shippingCart.length === 0"
       (waPaymentSubmit)="onPayment($event)"
       (waPaymentError)="onPaymentError($event)"
   >
       Buy
   </button>
</div>

3つのディレクティブのおかげですべてが機能します。

  • waPayment , PaymentItem
  • — waPaymentItem. PaymentItem .
  • Payment Request API . PaymentResponse . waPaymentSubmit .

したがって、支払いを開いてその結果を処理するためのシンプルで便利なインターフェースが得られます。そして、それはAngular Wayのすべての標準に従って機能します。

ディレクティブ自体はかなり単純な方法で接続されています。

  • 支払いディレクティブは、ContentChildrenを使用して内部のすべての商品を収集し、PaymentDetailsInit（Payment Request APIを操作するときに必要な引数の1つ）を実装します。

@Directive({
   selector: '[waPayment][paymentTotal]',
})
export class PaymentDirective implements PaymentDetailsInit {
   ...
   @ContentChildren(PaymentItemDirective)
   set paymentItems(items: QueryList<PaymentItem>) {
       this.displayItems = items.toArray();
   }
 
   displayItems?: PaymentItem[];
}

  • ボタンのクリックを追跡し、最終的な支払い結果を出力する出力ディレクティブは、Dependency Injectionツリーから支払いディレクティブを引き出し、支払い方法とDIトークンによって設定される追加オプションを引き出します。

@Directive({
   selector: '[waPaymentSubmit]',
})
export class PaymentSubmitDirective {
   @Output()
   waPaymentSubmit: Observable<PaymentResponse>;
 
   @Output()
   waPaymentError: Observable<Error | DOMException>;
 
   constructor(
       @Inject(PaymentDirective) paymentHost: PaymentDetailsInit,
       @Inject(PaymentRequestService) paymentRequest: PaymentRequestService,
       @Inject(ElementRef) {nativeElement}: ElementRef,
       @Inject(PAYMENT_METHODS) methods: PaymentMethodData[],
       @Inject(PAYMENT_OPTIONS) options: PaymentOptions,
   ) {
       const requests$ = fromEvent(nativeElement, 'click').pipe(
           switchMap(() =>
               from(paymentRequest.request({...paymentHost}, methods, options)).pipe(
                   catchError(error => of(error)),
               ),
           ),
           share(),
       );
 
       this.waPaymentSubmit = requests$.pipe(filter(response => !isError(response)));
       this.waPaymentError = requests$.pipe(filter(isError));
   }
}
