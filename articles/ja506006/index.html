Pythonists、角かっこを使用して辞書から値を取得するのをやめる

こんにちは、ハブル！Jonathan Hsuによる記事「Pythonで角括弧表記を使用して辞書の値を取得するのをやめる」の翻訳を紹介します。

従来の方法を超えてコードを保護する


辞書は、キーと値のソートされていないコレクションです。だということだ：

  • 辞書の各要素は、用語（キー）とその定義（意味）で構成されています。
  • キーはこの辞書に対して一意である必要があります-繰り返しはありません。
  • 辞書のキーには、リストアイテムのような明示的な配置がありません。

辞書を作成するには、中括弧を使用します。中括弧には、コンマで区切られたkey：valueペアを記述します。このような：

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}

辞書から意味を取得する従来の（そして安全でない）方法。

辞書から要素の値に通常アクセスするには、角括弧を使用します。そのようなエントリを使用する場合、キーは辞書名の後に角括弧で示されます。

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author['username']) #  Friday1719
print(author['middle_initial']) #   KeyError: 'middle_initial'

辞書にないキーで値にアクセスしようとすると、KeyError例外がスローされることに注意してください。これは、特に予測できない作業データを処理する場合に、深刻な問題を引き起こす可能性があります。

もちろん、try / except構文を使用するか、ifステートメントを使用できます。しかし、そのような単純な操作への注意の高まりは、コードを散らかすだけです。

author = {}
try:
   print(author['username'])
except KeyError as e:
   print(e) # 'username'

if 'username' in author:
   print(author['username'])

JavaScript開発の経験がある場合は、ドット表記を使用して辞書から値を取得することをお勧めします。しかし、Pythonでは、これは機能しません。

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author.username)
# AttributeError: 'dict' object has no attribute 'username'

get（）メソッドを使用します


辞書から値を取得する必要がある場合、最も安全な方法はget（）メソッドを使用することです。このメソッドには2つのパラメーターがあります。

  • 最初（必須）：値を取得するキーの名前。この名前は文字列にすることも、プログラム中にキーを変更できる場合は変数名にすることもできます。
  • 2番目（オプション）：キーが突然辞書に表示されなくなった場合に使用される値。

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author.get('username')) #   Friday1719
print(author.get('middle_initial', “Monday1917”)) #  Monday1917

キーがディクショナリに存在する場合、get（）メソッドは角括弧内のキーにアクセスするのとまったく同じように機能します。ただし、ディクショナリにそのようなキーがない場合、get（）メソッドはデフォルト値を返すため、例外を処理する必要がなくなります。

デフォルト値は、このコンテキストで有効な任意のオブジェクトにすることができます。このパラメーターはオプションであることに注意してください。したがって、明示的に指定しない場合、辞書に存在しないキーにアクセスしようとすると、get（）メソッドはNoneオブジェクトを返します。

setdefault（）メソッドを使用します


辞書からデータを安全に受け取るだけでなく、新しいデータを辞書に安全に追加する必要がある場合があります。このため、辞書にはsetdefault（）メソッドがあります。これにはget（）メソッドと同じパラメーターがありますが、後者とは異なり、存在しないキーを使用してディクショナリにアクセスすると、デフォルト値が返されるだけでなく、このキーと渡された値を使用してディクショナリに新しい要素が作成されます。setdefault（）メソッドを使用してディクショナリにアクセスするときに、送信されたキーがすでにディクショナリにある場合、このメソッドはディクショナリを変更しません。

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}

print（author.setdefault（ 'username'））＃金曜日を出力します1719
print（author.setdefault（ 'middle_initial'、“ Monday”））＃月曜日を出力します
＃辞書にキー 'middle_initial'を持つ要素を作成し、このキーの値は“ Monday 」

上記の例では、setdefault（）メソッドの動作は、get（）メソッドの動作や、既存のキーを使用して辞書にアクセスするときの角括弧の使用と同じです。辞書にそのようなキーがない場合、setdefault（）メソッドは2番目の引数の値を（get（）メソッドのように）プログラムに返すだけでなく、渡されたキーと値を使用して辞書に要素を作成します。これはsetdefault（）メソッドの動作であり、get（）メソッドとは異なります。

ここで、上記の例に従って辞書の要素を表示すると、その後、次の結果が得られます。

print(author)
{
  "first_name": "Robinson",
  "last_name": "Crusoe",
  "username": "Friday1719"
  "middle_initial": "Monday"
}

get（）メソッドとsetdefault（）メソッドの使用は、辞書の値にアクセスするための第一級の手法です。古い習慣を捨ててこのテクニックを実際に使い始めるのに時間がかかりません。

辞書から値を取得するだけの場合、ヘルパーはget（）メソッドです。

辞書に新しい値を安全に追加する必要がある場合は、setdefault（）メソッドを呼び出します。
