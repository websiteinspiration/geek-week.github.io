Fast.aiを使用してX線で肺炎を探す

School of Data社のブログの記事に出くわし、 Fast.aiライブラリがその記事で言及されているものと同じデータセットで何ができるかを確認することにしました。ここでは、肺炎をタイムリーかつ正確に診断することがどれほど重要であるか、放射線技師が技術開発の状況で必要になるかどうか、ニューラルネットワーク予測を医療診断と見なすことができるかどうかなどについての議論はありません。主な目標は、最新のライブラリでの機械学習が非常に単純で（文字通り数行のコードが必要）、優れた結果が得られることを示すことです。記事の結果（精度= 0.84、再現率= 0.96）を思い出して、何が起こるか見てみましょう。

ここから トレーニング用のデータを取得します。データは、肺炎の兆候の有無にかかわらず、2つのクラスに分布する5856のX線です。ニューラルネットワークのタスクは、肺炎の兆候を特定するためのX線画像の高品質のバイナリ分類器を提供することです。

まず、ライブラリといくつかの標準設定をインポートします。

%reload_ext autoreload
%autoreload 2
%matplotlib inline
from fastai.vision import *
from fastai.metrics import error_rate
import os

次に、バッチサイズを決定します。GPUで学習する場合、メモリがいっぱいにならないようにGPUを選択することが重要です。必要に応じて、半分にすることができます。

bs = 64

重要な更新：
以下のコメントに正しく記載されているように、モデルがトレーニングされ、その有効性をテストするデータを明確に監視することが重要です。ここでは、trainフォルダーとvalフォルダーの画像でモデルをトレーニングし、testフォルダーの画像で検証します。

データへのパスを決定します

path = Path('storage/chest_xray')
path.ls()

すべてのフォルダーが配置されていることを確認します（valフォルダーはトレーニングのために移動されています）。

Out:
[PosixPath('storage/chest_xray/train'),
 PosixPath('storage/chest_xray/test')]

ニューラルネットワークに「ダウンロード」するためのデータを準備しています。Fast.aiには、イメージラベルを照合する方法がいくつかあることに注意してください。from_folderメソッドは、画像が配置されているフォルダーの名前からラベルを取得する必要があることを通知します。

サイズパラメータは、すべての画像のサイズを299x299に変更することを意味します（アルゴリズムは正方形の画像で機能します）。get_transforms関数は、トレーニング用のデータ量を増やすための画像拡張を提供します（ここではデフォルト設定のままにします）。

np.random.seed(5)
data = ImageDataBunch.from_folder(path, train = 'train', valid = 'test', size=299, bs=bs, ds_tfms=get_transforms()).normalize(imagenet_stats)

データを見てみましょう：

data.show_batch(rows=3, figsize=(6,6))



確認するために、取得したクラスと、トレーニングと検証の間の画像の定量的な分布を確認します。

data.classes, data.c, len(data.train_ds), len(data.valid_ds)

Out:
(['NORMAL', 'PNEUMONIA'], 2, 5232, 624)

Resnet50アーキテクチャに基づくトレーニングモデルを定義します。

learn = cnn_learner(data, models.resnet50, metrics=error_rate)

そして、1サイクルポリシーに基づいて8時代から学習を開始します。

learn.fit_one_cycle(8)



検証サンプルではすでに89％の精度が得られています。ここでは、モデルの重みを書き留めて、結果の改善を試みます。

learn.save('step-1-50')

モデル全体を「解凍」します。その前に、最後のグループのレイヤーでのみモデルをトレーニングし、残りのウェイトはImagenetで事前トレーニングされたモデルから取得し、「凍結」しました。

learn.unfreeze()

学習を継続するための最適な学習率を探しています。

learn.lr_find()
learn.recorder.plot()



レイヤーのグループごとに異なる学習率で10の時代のトレーニングを開始します。

learn.fit_one_cycle(10, max_lr=slice(1e-6, 1e-4))



検証サンプルでは、​​モデルの精度がわずかに89.4％に向上したことがわかります。

重みを書き留めます。

learn.save('step-2-50')

混乱マトリックスの作成：
interp = ClassificationInterpretation.from_learner(learn)
interp.plot_confusion_matrix()


この時点で、特に不均衡なクラスの場合、精度パラメーター自体は十分ではないことを思い出します。たとえば、実際にX線検査を受けた人の0.1％だけが肺炎にかかっている場合、システムはすべての場合に肺炎がないことを単純に示すことができ、その精度は実用性がまったくなく99.9％のレベルになります。

ここで、PrecisionとRecallの指標が役立ちます。

  • TP-真の陽性予測;
  • TN-真の否定的予測;
  • FP-誤検知予測;
  • FN-偽陰性予測。

Prec私s私oん=TP/（TP+FP）=385/446=0、863


Recall=TP/（TP+FN）=385/390=0、987



結果は、記事で述べたものよりもわずかに高いことがわかります。タスクのさらなる作業では、リコールが医学的問題の非常に重要なパラメータであることを覚えておく価値があります。偽陰性のエラーは、診断の観点から最も危険です（つまり、危険な診断を単に「見落とす」ことができます）。
