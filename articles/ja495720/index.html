Android上のLinuxシステムの多くが使用する主要なテクノロジーの1つがpRootです。これはchrootユーティリティのユーザー空間実装であり、Linuxデスクトップおよびサーバーで非常に人気があります。ただし、chrootツールにはrootユーザー権限が必要です。これは、Androidではデフォルトでは使用できません。pRootは、ディレクトリへのバインディングを確立することにより、この利点を提供します。

Android上のLinux


前述したように、インストールするソフトウェアパッケージはユーザー空間で実行されます。

つまり、現在のユーザー、つまりAndroid OSの場合は常に通常のユーザー（つまり、管理者権限がありません。ただし、Linuxデスクトップをインストールするには、XやWaylandなどのグラフィカルサーバーをインストールする必要があります。これをLinuxオペレーティング環境で実行した場合、Android OSのグラフィカルレベルにアクセスすることなく、通常のユーザーの権限で開始されます。したがって、サーバーをハードウェアにアクセスしてグラフィカル環境をサポートできるように、「標準」のAndroidの方法でサーバーをインストールすることを検討する必要があります。

開発者コミュニティの賢い人たちは、この問題に対する2つの解決策を提案しています。 1つ目は、独自のバージョンのLinux（通常はXサーバー）を使用することです。彼らがバックグラウンドで作業を開始すると、VNCを通じてこのバックグラウンドプロセスにアクセスできるようになります。 Androidデバイスに他のコンピューターとリモートで対話するためのVNCビューアプログラムがすでにある場合は、それを使用してローカルホストへのリモートアクセスを取得します。これは使いやすいソリューションですが、一部のユーザーはプログラムで問題が発生していると報告しています。

2番目のオプションは、Androidデバイス専用に設計されたサーバーをインストールすることです。一部のサーバーは、無料バージョンと有料バージョンの両方でPlayストアで入手できます。インストールする前に、選択したオプションがサポートされているか、少なくともインストールするAndroidのLinuxソフトウェアパッケージで動作するかどうかを確認する必要があります。X-Serverシステムを選択したため、XServer XSDLソフトウェアパッケージ（リンク）を使用しました。この記事では、このサーバーのインストールプロセスについて説明しますが、別のアプリケーションをインストールしている場合やVNCを使用している場合は、多少異なる場合があります。

システムの選択


X-Serverの場合と同様に、PlayストアにLinuxシステムディストリビューションをインストールするためのアプリケーションがいくつかあります。ここでは、Termuxと同様に、スーパーユーザー権限を必要としないオプションに集中します。これには、ある程度のリスクが伴います。これらのアプリケーションは、データを安全に保ちながら、ほとんどのユーザーに必要なすべての機能を提供します。以下は、Playストアでのそのようなアプリケーションの例です。

- UserLAnd：ユーザーの非常に人気のある選択肢。アプリケーションには、Debian、Ubuntu、Arch、Kaliなどの一般的なディストリビューションのセットが含まれています。不思議なことに、RPMベースのオプションがないにもかかわらず、UserLAndにはメモリの少ないデバイス用のAlpine Linuxが含まれています。

- AnLinux：このアプリケーションは、1つ以上の大量の配布リストをインストールするのに役立ち、Ubuntu / Debian、Fedora / CentOS、openSUSE、さらにはKaliを含めることができます。そこでは、Xfce4、MATE、LXQt、LXDEなどの低コストのデスクトップオプションも選択できます。動作するには、Termuxをインストールする必要があり、Androidオペレーティングシステムは5.0以降である必要があります。

- Andronixはある AnLinuxと非常によく似ています。以前のアプリケーションよりもデザインが優れている可能性がありますが、サポートされるディストリビューションは少なくなっています。

- GNURoot WheezyX：このプロジェクトはAndroidのLinuxバリアントとして始まり、オープンソースプログラム用に開発されました。名前が示すように、Debianディストリビューションに焦点を当てていますが、末尾の「X」は、アプリケーションがグラフィカルデスクトップに焦点を当てていることを意味しています。そして、作成者がUserLAndのためにプロジェクトの開発を中止したという事実にもかかわらず、GNURoot WheezyXは、だれかがそれを必要とする場合、引き続きPlayストアで入手できます。

この記事の作成者は、UserLAndアプリケーションを使用してLinuxデスクトップをAndroidにインストールしますが、これにはいくつかの理由があります。まず、アプリケーションはオープンソースです（AnLinuxにもあります）。第二に、ディストリビューションの適切なセットを提供します（ただし、FedoraまたはCentOSは含まれていません）。また、スマートフォンのメモリであまりスペースを取らない最小システム要件でディストリビューションをインストールすることもできます。しかし、UserLAndの主な利点は、ディストリビューション全体ではなく、個々のアプリケーションをインストールするためのサポートツールが含まれていることです。これが私たちにとって何を意味するのかは後でわかります。次に、デバイスにUserLAndをインストールします。

UserLAndアプリケーション


Google PlayまたはAndroid OSのF-Droid（リンク）からアプリケーションをダウンロードします。他のアプリケーションと同じようにインストールされます-ここで特別なことをする必要はありません。その後、アプリケーションパネルから起動します。

最初に目にするのはディストリビューションのリストです。最後に、LXDEとXfce4という2つのデスクトップオプションがあります。彼のアプリケーションであるFirefox、2つのゲーム、およびいくつかのオフィスユーティリティ（GIMP、Inkscape、LibreOfなど）を閉じます。このタブは「アプリケーション」と呼ばれます。アプリケーションをインストールするように設計されています。

何かをインストールすると、対応するエントリが[セッション]タブに表示されます。ここでは、現在のセッションを開始または停止したり、実行中のプロセスを確認したりできます。

「ファイルシステム」は、すでに完了したインストールを表示する最後のタブです。ファイルシステムからアイテムを削除すると、そのアイテムに関する情報が[セッション]タブから消去されることに注意してください。ただし、それ以外の場合は証明されません。つまり、現在のファイルシステムに基づいて新しいセッションを作成できます。この関係が実際にどのように機能するかを理解すれば、はるかに簡単になるので、まず、UserLAndシステム環境にアプリケーションをインストールすることから始めます。


スマートフォンにディストリビューションをインストールする前に、リポジトリへのUserLAndアクセスを提供する必要があります。

UserLAndでの分布


アプリ画面にあるディストリビューションの1つを選択して、デバイスにインストールします。例として、Ubuntuを使用します。アイコンをクリックすると、VNCのユーザー名、パスワード、パスワードを入力するダイアログボックスが表示されます。次に、ディストリビューションへのアクセス方法を選択します。ダウンロードが開始され、その間に、選択したディストリビューションのベースイメージが含まれます。ファイルは、UserLAndディレクトリに解凍されます。

ダウンロードが完了したら、xtermターミナルエミュレータに戻ります。ユーティリティコマンドを入力して、インストールしたLinuxのバージョンを確認できます。

uname –a

次のステップは、Ubuntuユーティリティコマンドを使用してデスクトップをインストールすることです。

sudo apt install lxde

最後のステップは、新しいデスクトップ環境を実行する準備ができていることを確認することです。これを行うには、現在/ usr / bin / twmが 1行しかない.xinitrcfileファイルを編集する必要があります。/ usr / bin / startlxdeに変更する必要があります。XSDLセッションを終了し（通知領域のSTOPボタンを必ず押してください）、[セッション]タブの[Ubuntuリスト]ボタンを押したまま、[セッションの停止]をクリックしてセッションを再開します。数秒後、LXDEシステム環境が表示されます。通常のデスクトップと同じように実行できます。デバイスのボタンを押すには、キーボードとマウスで操作する場合よりも長く待つ必要があります。スマートフォンのLinuxシステム環境をどのように改善できるかを見てみましょう。

デスクトップの内容を注意深く調べると、デスクトップバージョンの正確な再現を見つけることができます。キーボードとマウス（Bluetooth接続またはその他の方法で接続）を備えたデバイスでUserLAndを使用する場合、この形式のLinuxシステム環境の使用に簡単に適応できます。 X-WindowsカーソルがAndroid上のデバイスのカーソルと同期しているという事実によるわずかな遅延を除いて、すべてがスムーズに動作します。

デスクトップフォントのサイズが電話の画面には大きすぎるため、おそらく最初に行うことは、デフォルトのフォントシステムを調整することです。メインメニューに移動し、[設定]→[外観とウィジェットのカスタマイズ]→[ウィジェット]を選択します。ここで、デフォルトのフォントサイズを携帯電話に適したオプションに変更できます。

後でLinuxシステム環境にお気に入りのプログラムをインストールすることができます。上記のように、この場合ユーティリティコマンドは機能しません。UserLAndシステム環境にインストールされている、ASAPと呼ばれる本当にかけがえのないツールを自由に使用してください。

ディストリビューションをインストールしたら、標準コマンドを使用してデスクトップ環境を追加できます。

おそらく、ディストリビューションの接続を確立する別の方法も必要になります。最初にXSDLをインストールしたからといって、常に変更されている必要はありません。 [セッション]タブで別のアカウントを作成し、別のサーバーを選択できます。必ず同じファイルシステムで指定してください。 UserLAndは、XSDL、ConnectBot for SSH、またはbVNCのいずれかの新しいタイプの接続を確立するために、目的のアプリケーションにユーザーを誘導しようとします。

ただし、再接続を試みるときにアプリケーションが自動的にPlayストアに誘導する永続性は、煩わしい場合があります。これを停止するには、特別なアプリケーションをインストールしてサーバーを変更するだけです。 SSHをインストールするには、古い実績のあるVX ConnectBotを選択します。ユーザー名とパスワードを使用して、ワークステーションのポート2022を入力するだけです。 VNCサーバーに接続するには、コマーシャルをインストールするだけで十分ですが、多くの点で高度なJumpデスクトップアプリケーションを使用し、アドレス127.0.0.1►951をダイヤルします。

ファイルシステムを作成したときに設定したVNCパスワードを覚えておいてください。
