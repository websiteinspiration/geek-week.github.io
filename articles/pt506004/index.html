Primeiras reflexões sobre Deno

A tradução do artigo foi preparado antes do lançamento do desenvolvedor Node.js curso .


No Codegram, adoramos experimentar novas tecnologias, e o Deno saiu há apenas uma semana! Leia o artigo se você quiser saber sobre esse novo tempo de execução do JavaScript e o que o Codegram pensa sobre ele.

Você já deve ter ouvido falar sobre Deno, ou talvez não, mas agora a Internet está cheia de informações sobre isso e há uma razão para isso! Um novo tempo de execução JavaScript apareceu com um dinossauro fofo como um logotipo (algumas pessoas pensam que o logotipo tem uma meia triste, mas não ...). Nós até tweetamos sobre ele uma semana antes do primeiro lançamento, mas, para dizer a verdade, ouvimos falar de Deno dois anos atrás, depois de assistir a essa incrível performance de Ryan Dahl .

Neste artigo, abordaremos Deno superficialmente e compartilharemos nossos pensamentos sobre isso. Vamos começar!

Olá Deno!


Deno é um tempo de execução JavaScript como o Node. Para usá-lo, você deve primeiro instalar a CLI do deno. Para fazer isso, siga as instruções para o seu sistema operacional em deno.land (nosso nome de domínio é um elogio!). Deno não é um fork do Node, foi criado do zero no Rust. Eu escrevi um artigo sobre Rust for JS programadores , você pode lê-lo se quiser saber mais sobre a linguagem!

Depois de instalar a CLI, você pode executar o programa clássico "Olá, mundo!" Da seguinte maneira:

deno run https://deno.land/std/examples/welcome.ts

Estamos executando o TypeScript pronto para uso? Sim! O TS é um convidado bem-vindo no Deno. Portanto, você não precisa de mais nada para executar o código no TS. Você também pode executar o código JS, mas eu sou um grande fã do TS, então fui o primeiro a falar sobre isso.

Lembre-se de que o código do nó não é compatível com o Deno. No entanto, agora eles estão trabalhando para garantir a compatibilidade , para que, no futuro, você possa usar seus pacotes NPM favoritos.

Há mais uma coisa que vale a pena mencionar que Deno é uma sandbox por padrão. Ou seja, inicialmente você não tem acesso à rede ou sistema de arquivos. Se o seu programa precisar de acesso, você deverá permitir explicitamente (ou seja, usar --allow-net para conceder privilégios de rede).

Não mais package.jsonounode_modules
Quem não viu essa piada na Internet?



Apesar de um leve exagero, ainda há alguma verdade nessa piada. À medida que seu aplicativo cresce, a lista de dependências aumenta e você acaba com uma enorme árvore de dependências. O problema é que todo aplicativo possui uma cópia do pacote @ versão que você está usando.
Um dos principais objetivos de Deno é espelhar o navegador. Quando você deseja importar algo para um navegador, escreve uma tag <script>e usa o URL. Deno funciona da seguinte maneira:

import { assertEquals } from "https://deno.land/std/testing/asserts.ts";
assertEquals("hello", "hello");
assertEquals("world", "world");
console.log("Asserted! ");

Eu concordo, parece um pouco estranho. Como não temos nenhum package.json, não há como especificar dependências externas, por isso as importamos quando são necessárias. No primeiro início, o Deno fará o download do arquivo e o salvará no cache. Você pode especificar o sinalizador --reload se desejar baixar o pacote novamente.

Você provavelmente tem muitas perguntas sobre esse assunto (eu também as tinha), então convido você a visitar esta página de perguntas frequentes no manual do Deno para obter mais informações sobre o assunto.

Tempo de execução compatível com a API do navegador


Outro objetivo importante do Deno era a compatibilidade do navegador, portanto, ele tem um conjunto de recursos disponíveis no escopo global, como busca ou addEventListener . Você também pode usar o objeto de janela global (no entanto, por enquanto, eu recomendo usar globalThis). Você pode aprender mais sobre funções globais na documentação .

O tempo de execução também possui um Deno global para APIs que não são um padrão da Web. Você pode usá-lo para operações de baixo nível, como ler um arquivo, abrir um soquete TCP etc.

Biblioteca padrão rica


Os desenvolvedores da Deno criaram uma coleção de módulos padrão prontos para uso. A biblioteca também está hospedada no domínio deno.land , para que você possa importar qualquer módulo para seu aplicativo usando o seguinte URL:

import { v4 } from "https://deno.land/std/uuid/mod.ts";

console.log(v4.generate());

A biblioteca padrão inclui muitos módulos úteis, mas existem muitos de terceiros, e essa lista é atualizada todos os dias. Há mais uma boa notícia: você pode importar qualquer módulo localizado em uma URL pública, o que significa que, com o mesmo sucesso, você pode importar módulos do GitHub. O Deno possui um serviço de reescrita de URL que você pode usar para disponibilizar seus módulos. Você pode aprender mais sobre esse mecânico nesta página .

Ferramentas integradas


O Deno CLI também inclui ferramentas internas para executar algumas tarefas comuns, como executar testes, por exemplo, ou até criar um pacote! Isso significa que o Deno pode substituir o gracejo, o mais bonito e o webpack? Provavelmente não. O Deno é uma solução relativamente nova e ainda não possui um ecossistema tão extenso, mas a presença de todas essas ferramentas, cujo acesso pode ser obtido com a instalação de um binário, ainda é impressionante.

Conclusão


Ir nesta onda de hype! Estou realmente impressionado com o Deno. Após dois anos de desenvolvimento e um longo caminho para o sucesso, o novo tempo de execução do JavaScript parece ótimo e estou ansioso para começar a experimentar. O Node apareceu há cerca de 11 anos e se tornou um novo marco na história do JavaScript. Ele melhorou a linguagem e acho que alcançamos um novo marco, é hora de Deno!
