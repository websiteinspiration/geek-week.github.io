Notfallwiederherstellung und Migration mit VMware vCloud Availability. Teil 2

Hallo! In einem früheren Beitrag habe ich über die Funktionen von VMware vCloud Availability (vCAV) gesprochen und gezeigt, wie Disaster Recovery (DR) und Migration innerhalb mehrerer Cloud-Anbieterstandorte organisiert werden. Heute erfahren Sie, wie Sie mithilfe von vCAV eine Wiederherstellung durchführen oder einfach von einer lokalen Plattform in die Cloud eines Dienstanbieters migrieren können. In unserem Beispiel konfigurieren wir DR vom lokalen Standort des Kunden in die Cloud in St. Petersburg. 



In der Vorbereitungsphase müssen Sie entscheiden, wie der Zugriff auf die Server nach deren Wiederherstellung in der Cloud bereitgestellt werden soll. Dazu müssen Sie die Netzwerkkonnektivität zwischen dem lokalen Standort und der Cloud organisieren. Über die wichtigsten Möglichkeiten, eine Verbindung zur Cloud herzustellen, habe ich hier geschrieben .

Der Einfachheit halber habe ich eine schnelle Navigation gemäß den Anweisungen durchgeführt:

  • Installation und Konfiguration Availability Appliance für vCloud-Mandanten ;
  • Schaffung von DR-Arbeitsplätzen ;
  • Testwiederherstellung ;
  • Failover zwischen Standorten .
  • Rückwärtsschaltung .
