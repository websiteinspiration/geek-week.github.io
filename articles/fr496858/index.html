COVID-Net, développé par la startup canadienne IA DarwinAI, est un réseau neuronal convolutionnel profond conçu pour dépister les patients suspects d'infection à coronavirus en détectant les signes de contrôle de la maladie sur une radiographie pulmonaire, rapporte ZDNet. Alors que les tests d'infection à coronavirus sont traditionnellement effectués avec un tampon de l'intérieur de la joue ou du nez, les hôpitaux manquent souvent de kits de test et de testeurs, et la radiographie pulmonaire est rapide et généralement l'équipement nécessaire est disponible dans les hôpitaux. Le goulot d'étranglement entre l'obtention d'une image radiographique et son interprétation, en règle générale, est la recherche d'un radiologue pour compiler un rapport à partir des données de numérisation - au lieu de cela, si l'IA le lit, cela peut signifierque les résultats de l'analyse seront reçus beaucoup plus rapidement. Selon Sheldon Fernandez, PDG de DarwinAI, après l'ouverture des sources de COVID-Net, "la réponse était tout simplement écrasante . " « Nos boîtes aux lettres étaient jonchées de lettres de personnes qui recommandent des améliorations et parlent de la façon dont nous utilisons ce que nous faisons » , a- t-il ajouté.

Sortie de Chrome 81




Google a présenté la sortie du navigateur Web Chrome 81. Dans le même temps, selon OpenNET, une version stable du projet gratuit Chromium, qui sert de base à Chrome, est disponible. Ainsi, la publication rappelle que le navigateur Chrome se distingue par l'utilisation des logos Google, la présence d'un système de notification en cas de crash, la possibilité de charger le module Flash à la demande, des modules de lecture de contenu vidéo protégé (DRM), un système pour installer automatiquement les mises à jour et transmettre lors de la recherche des paramètres RLZ. La prochaine version de Chrome 82 sera ignorée, la sortie de Chrome 83 est prévue pour le 19 mai. Quelques innovations:

  1. Le support FTP est désactivé;
  2. pour tous les utilisateurs, la fonction de regroupement des onglets est activée, ce qui vous permet de combiner plusieurs onglets similaires dans des groupes visuellement séparés;
  3. Des modifications ont été apportées aux conditions d'utilisation de Google, dans lesquelles une section distincte pour Google Chrome et Chrome OS est apparue;
  4. il est stabilisé et l'interface du programme Badging est désormais distribuée en dehors de Origin Trials, ce qui permet aux applications Web de créer des indicateurs affichés sur un panneau ou un écran d'accueil;
  5. améliorations des outils pour les développeurs Web;
  6. La désactivation de la prise en charge de TLS 1.0 et TLS 1.1 a été retardée jusqu'à la sortie de Chrome 84.

La mise à jour de Chrome OS est également sortie, elle est équipée de gestes de navigation simples et d'un nouveau dock Quick Shelf, rapporte CNet.

FlowPrint, une boîte à outils pour identifier les applications basées sur le trafic chiffré, est disponible





Le code de la boîte à outils FlowPrint a été publié, ce qui permet de déterminer les applications mobiles du réseau en analysant le trafic chiffré généré pendant le fonctionnement de l'application, rapporte OpenNET. Il est possible d'identifier à la fois les programmes types pour lesquels des statistiques sont accumulées et l'identification de l'activité de nouvelles applications. Le code est écrit en Python et distribué sous licence MIT. Le programme met en œuvre une méthode statistique qui définit les caractéristiques de l'échange de données qui sont caractéristiques pour différentes applications (retards entre les paquets, caractéristiques des flux de données, changements de taille des paquets, caractéristiques d'une session TLS, etc.). Pour les applications mobiles Android et iOS, la précision de reconnaissance de l'application est de 89,2%. Au cours des cinq premières minutes de l'analyse de l'échange de données, 72,3% des demandes peuvent être déterminées. La précision de l'identification de nouvelles applications qui n'ont jamais été vues auparavant,est de 93,5%.

Microsoft a proposé un module de noyau Linux pour vérifier l'intégrité du système





Les développeurs de Microsoft ont présenté le mécanisme IPE (Integrity Policy Enforcement), implémenté en tant que module LSM (Linux Security Module) pour le noyau Linux. Le module vous permet de définir une politique d'intégrité générale pour l'ensemble du système, indiquant quelles opérations sont valides et de quelle manière l'authenticité des composants doit être vérifiée. À l'aide d'IPE, vous pouvez spécifier les fichiers exécutables autorisés à s'exécuter et vous assurer que ces fichiers sont identiques à la version fournie par une source fiable. Le code est ouvert sous la licence MIT.
