DRAGおよびDROPエフェクトを作成する方法

初めてDRAGとDROPに来たとき、私はそれについて非常に難しい説明に出くわしました（これは私の主観的な意見です。私は彼に同意しないで、可能な限りすべてをもう一度読んで、この質問をさまざまな角度から見てください）。そして、私はZenを学びたい初心者向けの記事をいくつか書くことにしました。

記事は2つの部分で構成されます。

  • DRAGおよびDROPエフェクトを作成する方法。
  • DRAGとDROPを使用してソートを作成するために得られた知識の実用的なアプリケーション

パラグラフ1 DRAGおよびDROPエフェクトの作成方法


深い分析を始める前に、表面的に見てみましょう。自分をローダーとして想像してみてください。ボックスをある場所から別の場所に移動する必要があります。ローダーについては、「まあ、私はそれを手に入れました。完了しました！」、そしてプログラマーのために、「私は箱に行って、曲がって、箱を手に取り、箱を持ち上げて、Nステップ歩いて、曲がって、箱から離しました。」つまり、仕事を始める前に、あなたの頭の中のすべてを段階的に失い、あなたは真実にはるかに近づくでしょう。

このエフェクトを作成するにはいくつかの方法があることを予約したいと思います。すべてについて必ずお読みください。私が現在最も便利だと思うものを利用します。

DRAGおよびDROPを作成する場合、最初のステップは、移動するオブジェクトをdraggable = 'true'に設定することです。

<html>
  <head>
    <style>
      .ddd {
	display:block;
	float:left;
	padding:10px;
	border:1px solid #000;
      }
    </style>
  </head>
  <body>
    <div id='d1' class='ddd' draggable='true'>Pervii 1</div>
  </body>
</html>

、
最初の段階でプロセス自体を示したいと思います。その後、すべてのオブジェクトにそれを展開します。私たちは現在JSで作業しており、ご存知のように、ブラウザーにはさまざまなイベントがあり、そこに独自の一連のアクションをアタッチできます。 DRAGとDROPの作成に必要なイベントを見てみましょう：

dragstart-ユーザーがアイテムのドラッグを開始すると発生します。
drag-アイテムがドラッグされているときに発生します。
dragend-ユーザーがアイテムのドラッグを完​​了したときに発生します。
dragenter-ドラッグされたアイテムがターゲットに到達したときに発生します。
dragleave-ドラッグされたアイテムがターゲットを離れると発生します。
ドラッグオーバー-ドラッグされたアイテムがターゲットの上にあるときに発生します。
ドロップ -ドラッグされたアイテムがターゲットオブジェクト上にあるときに発生します。

今非常に重要な情報！イベントは2つのグループに分けられます。移動したアイテム（ドラッグしているアイテム）の場合：dragstart、Drag、Dragend。受信要素（ドラッグする場所）の場合：Dragenter、Dragleave、Dragover、Drop。そして、これらのイベントは逆に機能することはできませんが、お互いがなくても機能します。

例：オブジェクトを移動し、マウスボタンを離した場所に置いたままにする必要があります。このタスクにはホストパーツは必要ありません。

<html>
  <head>
    <style>
      .ddd {
  	display:block;
	float:left;
	padding:10px;
	border:1px solid #000;
      }
    </style>
  </head>
  <body>
    <div id='d1' class='ddd' draggable='true'>Pervii 1</div>
    <div id='d2' class='ddd'>Vtoroy 2</div>
    <div id='d3' class='ddd'>Tretii 3</div>
    <div id='d4' class='ddd'>Chetverty 4</div>
    <div id='d5' class='ddd'>Pyatii 5</div>	
    <script>
	var d1 = document.getElementById('d1');
	var startCursorX;
	var startCursorY;
	var startX;
	var startY;
	d1.addEventListener('dragstart',function() {
	  startCursorX = event.pageX;//     X
	  startCursorY = event.pageY;//     Y
	  startX = d1.style.marginLeft.replace('px','')*1; //      PX
	  startY = d1.style.marginTop.replace('px','')*1;
	});
	d1.addEventListener('dragend',function() {
	  d1.style.position = 'absolute';//CSS   "" :) 
	  d1.style.marginLeft = startX + event.pageX-startCursorX; //  +   -     
	  d1.style.marginTop = startY + event.pageY-startCursorY; //       ,    
	});
    </script>
  </body>
</html>

間違いなく、この例は膝で行われていますが、受信オブジェクトのオプション性を完全に示しています。

段落番号2。DROPはDRAGおよびDROPでは機能しません


すべてのイベントを試すと、ドロップが機能しないことがわかります。この方法を開発したのは、「それだけです...ええ、ナンセンス」と決定した人にatatを作成します。

さて、すべてが単純です。ドロップイベントの前に、同じ要素でイベントをハングアップする必要があります

d2.addEventListener('dragover',function() {
  event.preventDefault();
});

それを事実として考えてください。それなしでは機能しません。
