Shadow DOMのないWebコンポーネント

こんにちは、ハブル！

最近、あらゆる種類のWebコンポーネントを批判している記事に気づきました。時には、この批判は非常に厳しいものであり、憎しみの小片ですらある。私の意見では、ここでの主な問題は、開発者コミュニティの間で、この標準のグループで作業する確立された慣習の欠如です。多くの使い慣れたモデルは、カスタム要素やShadow DOMを含むプロジェクトに必ずしも有機的に適合するとは限りません。多くのことを新しい角度から見る必要があり、誰もがそれを好むわけではありません。私は数年前からWebコンポーネントの操作に成功しており、独自のライブラリを開発していますそれらに基づいているので、この状況はあまり公平ではないと思います。少なくとも部分的には、私の適度な強さの範囲で修正するように努めます。私は、一連のコンパクトな出版物を作成することにしました。そのそれぞれで、批評の頻繁な側面の1つに触れ、バリケードのどちら側に配置すべきかをまだ決定していない人にとって興味深いかもしれない多くの技術的手法を実証する予定です。今日はシャドウDOMなしでコンポーネントを作成する方法についてお話します。

何のために？


今回伝えたい主なアイデアは、WebコンポーネントとShadow DOMは同じものではないということです。Shadow DOMを使用すると、2つの主な利点があります。

  • スタイルが外部の影響や「漏れ」から安全であると感じるドキュメントの分離されたセクション
  • 文書をコンポーネント自体の構造とその内容（ツリー内のDOM要素の子孫）に分割できる構成メカニズム

ただし、このメカニズムには、孤立したスペースを作成してスタイルを設定するためのコストも伴います。これは非常に自然なことです。場合によっては（大きなリスト、データを含むテーブルセルなど）、パフォーマンスの最適化のためにこれらのコストを回避したいと思います。これを修正します。

const MY_CSS = {
  title: 'color: #00f; font-size: 2em',
  item: 'color: #f00; font-size: 1.2em',
};

const DATA = [
  {text: 'Text 1'},
  {text: 'Text 2'},
  {text: 'Text 3'},
];

let template = document.createElement('template');
template.innerHTML = /*html*/ `
<div style="${MY_CSS_.title}">List items:</div>
<div class="my-list">
  ${DATA.map(item => /*html*/ `<div style="${MY_CSS.item}">${item.text}</div>`).join('')}
</div>
`;

class ShadowlessComponent extends HTMLElement {
  constructor() {
    super();
    this._contents = new DocumentFragment();
    this._contents.appendChild(template.content.cloneNode(true));
  }
  connectedCallback() {
    this.appendChild(this._contents);
  }
}

window.customElements.define('shadowless-component', ShadowlessComponent);

カスタム要素標準に既に慣れている場合は、何が起こっているかすぐattachShadowにわかります。コンポーネントコンストラクターでメソッドを呼び出す代わりに、事前に用意されたテンプレートを複製するDocumentFragmentを作成しました。この段階では、コンポーネントはブラウザーによってレンダリングされず、データをバインド/挿入するなど、比較的安全に変更できます。

次の重要なステップは、カスタム要素のライフサイクルに関連しています。コンポーネントは一般的なドキュメントに追加されますが、コンストラクタが完全に機能し、その瞬間まで、要素の親または子孫、および属性を処理するDOM APIの部分は使用できません。したがって、コンポーネントにコンテンツを直接追加するには、を使用しますconnectedCallback。

テンプレートを作成するときは、簡単にするためにこの方法を使用しましたinnerHTML。この操作は1回だけ実行されます。「テンプレート」要素を作成する場合、コンポーネントのインスタンスが作成されるたびに繰り返されるわけではありません。ただし、この点は、パターンを強制的に作成することによってさらに最適化することもできます。

全体として、マークアップshadowless-componentでカスタムタグを使用すると、ブラウザーで次の結果が得られます。

<shadowless-component>
  <div id="caption" style="color: #00f; font-size: 2em">List items:</div>
    <div class="my-list">
      <div style="color: #f00; font-size: 1.2em">Text 1</div>
      <div style="color: #f00; font-size: 1.2em">Text 2</div>
      <div style="color: #f00; font-size: 1.2em">Text 3</div>
    </div>
</shadowless-component>

ShadowRootを削除したため、スタイルの分離が失われたため、属性を使用してテンプレートにスタイルを追加しました。この場合、それらが優先され、これにより問題が部分的に解決され、重要な場所で使用できます。他のすべてのケースでは、一般的なスタイルシートを通じてクラシックなスタイルを利用でき、カスタムタグは便利なセレクターとして機能します。

しかし、追加のコンテナーやカスタムタグがまったく必要ない場合はどうでしょうか。純粋なHTMLを与える！

OK：

  constructor() {
    super();
    this._contents = new DocumentFragment();
    this._contents.appendChild(template.content.cloneNode(true));
    this._titleEl = this._contents.querySelector('#caption');
    window.setInterval(() => {
      this._titleEl.textContent = Date.now();
    }, 1000);
  }
  connectedCallback() {
    this.parentNode.prepend(this._contents, this);
    this.remove();
  }

CSSコンポーネント


規格によれば、カスタムタグには必ず「-」文字を追加して名前を付ける必要があります。マークアップでタグを使用するが、JSでコンポーネントを作成せず、そのコンストラクタをコンポーネントレジストリに追加しない場合、ブラウザはタグを「不明な要素」（HTMLUnknownElement）と見なします。デフォルトでは、これらの要素の動作はspanタグと似ています。これは、CSSルール:: before、:: after、およびattr（）式で十分な単純な構造を持つ単純なダムコンポーネントを作成する必要がある場合に使用できます。例：

  my-container {
    display: block;
    padding: 10px;
    border: 1px solid currentColor;
  }
  my-container::before {
    content: attr(caption);
    margin-bottom: .6em;
  }

マークアップで使用：

<my-container caption=""></my-container>
