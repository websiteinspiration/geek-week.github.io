java.netのHttpUrlConnectionの機能

こんにちは。

本日は、JREライブラリのURLConnectionを使用して、リクエストを送信し、HTTPサーバーからの応答を読み取る方法についてお話します。

現在、Javaをオンラインで学習しています。チーム全体が仕事とコミュニケーションにSlackを使用しています。ユーザーについては、Slack APIを使用します。長い間API自体について話さないようにするために（これは別の記事のトピックです）、簡単に言います。SlackAPIはHTTPプロトコルに基づいて構築されています。ユーザーに関する情報を取得するには、APIからホスト名APIへのメソッド名が必要なURIを使用してリクエストを送信する必要があります。 slack.comは、いくつかのメソッドのリストです。
  • users.list
  • chat.postMessage
  • conversations.create
  • files.upload
  • im.open

ユーザーリストを取得するには、users.listメソッドが必要です。URIを形成します- /api/users.listをリクエスト本文に含めます。application / x-www-form-urlencodedの形式の認証トークンが必要です。つまり、リクエストは次のようになります（ただし、ニュアンスが1つ低くなります）。

GET /users.list HTTP/1.1
Content-Type: application/x-www-form-urlencoded

token=xoxp-1234567890-098765-4321-a1b2c3d4e5


私はApache HttpComponentsライブラリについて知っていましたが、研究目的のために、標準のJava 8ライブラリで利用可能なツール、つまりjava.net.URLConnectionの実装を使用します。

URLConnectionエンティティを取得するには、java.net.URLクラスのオブジェクトを使用する必要があります。そのコンストラクタはString型を取り、すべてに加えて、プロトコル（この場合はhttps）を指定する必要があります。

URLエンティティを受け取った後、opens（）メソッドを呼び出し、HttpsUrlConnectionエンティティを返します。

String url = “https://slack.com/api/users.list”;
URLConnection connection = new URL(url).openConnection();

この場合、MalformedUrlExceptionおよびIOExceptionを処理またはスローする必要があります。

その後、接続変数はHttpsUrlConnectionImplオブジェクトへの参照を格納します。デフォルトでは、GETリクエストが生成されます。ヘッダーを追加するには、キーと値を受け入れるsetRequestProperty（）メソッドを使用します。ここで、値application / x-www-form-urlencodedを持つContent-Typeを設定する必要があります。さて、私たちはそれを行います！
connection.setRequestProperty(“Content-Type”, “application/x-www-form-urlencoded”);

トークンと制限を本文に書き込んでリクエストを送信するだけです。これを行うには、setDoOutput（）メソッドを使用して、接続オブジェクトのdoOutputフィールドをtrueに設定します。

connection.setDoOutput(true);

次に、最も興味深い部分-何とかしてリクエスト本文をOutputStreamに渡す必要があります。OutputStreamWriterを使用します。

OutputStreamWriter writer = new OutputStreamWriter(connection.getOutputStream());

注意点が1つあります。getOutputStream（）メソッドを呼び出した後、GETはリクエスト本文を提供しないため、リクエストメソッドはPOSTに変更されますが、スラックはメソッドに厳密な制限を課さないため、すべて正常でした。したがって、GETリクエストは次のようになります。
GET /users.list?token=xoxp-1234567890-098765-4321-a1b2c3d4e5&limit=100 HTTP/1.1
Content-Type: application/x-www-form-urlencoded

しかし、私はそれをやり直し始めませんでした。そして代わりに、私たちの要求は次のように判明しました：

POST /users.list HTTP/1.1
Content-Type: application/x-www-form-urlencoded

token=xoxp-1234567890-098765-4321-a1b2c3d4e5

（*一部のヘッダーはHttpsUrlConnection自体によって設定され、ここにはありません）

そして、リクエスト本文を書き込むには、write（）;を使用します。
String reqBody = “token=xoxp-1234567890-098765-4321-a1b2c3d4e5&limit=100”;
writer.write(reqBody);
writer.close();

その後、リクエストが送信され、受信したレスポンスを読み取ることができます。InputStreamを受信する前に、OutputStreamを閉じるか、flush（）を実行することが重要です。そうしないと、データがバッファーを離れません（または、PrintStreamを使用できます-println（）メソッドでは、flush（）がデフォルトで呼び出されます）。読み取りには、BufferedReaderを使用しました。
StringBuilder respBody = new StringBuilder();
BufferedReader reader = new BufferedReader(connection.getInputStream());
reader.lines().forEach(l -> respBody.append(l + “\r\n”);
reader.close();

（*行（）を使用して出力でストリームを取得します; \ r \ n-CRLF文字-新しい行への遷移を挿入します）

そして、認証に成功した場合、respBody変数はサーバーからの応答（この場合はJSONオブジェクト）を格納する必要があります。その後、次の処理段階に送ることができます。

最適化後、すべては次のようになります。

package main.java.com.bilichenko.learning;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.io.OutputStreamWriter;
import java.net.HttpURLConnection;
import java.net.MalformedURLException;
import java.net.URL;
import java.nio.charset.Charset;
import java.util.Optional;
import java.util.stream.Collectors;

public class SlackClient {

    private static final String HOST = "https://api.slack.com";
    private static final String GET_USERS_URI = "/api/users.list";

    private static final String TOKEN = "xx-ooo-YOUR-TOKEN-HERE";

    public static void main(String[] args) throws IOException {
        SlackClient slackClient = new SlackClient();
        System.out.println(slackClient.getRawResponse(HOST + GET_USERS_URI,
                "application/x-www-form-urlencoded",
                "token=" + TOKEN).orElse("no response"));
    }

    public Optional<String> getRawResponse(String url, String contentType, String requestBody)
            throws MalformedURLException, IOException {
        HttpURLConnection connection = (HttpURLConnection) new URL(url).openConnection();
        connection.setRequestProperty("Content-Type", contentType);
        connection.setConnectTimeout(10000);
        connection.setRequestMethod("POST");

        connection.setDoOutput(true);

        try(OutputStreamWriter writer = new OutputStreamWriter(connection.getOutputStream())) {
            writer.write(requestBody);
        }

        if (connection.getResponseCode() != 200) {
            System.err.println("connection failed");
            return Optional.empty();
        }

        try(BufferedReader reader = new BufferedReader(
                new InputStreamReader(connection.getInputStream(), Charset.forName("utf-8")))) {
            return Optional.of(reader.lines().collect(Collectors.joining(System.lineSeparator())));
        }
    }
}
