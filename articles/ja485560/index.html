アンチパターンDevOpsインタビュー

親愛なる読者の皆さん、こんにちは！

今日、私は長年にわたって差し迫ったトピックに関する私の考えを共有したいと思います、そしておそらくコメントでそれを議論したいと思います。
プログラマーにとっての面接の悪い習慣に関する記事に出くわすことがよくあります。私の意見では、これは非常に重要であり、大企業ではなく大企業の人事部門が読むことができると思います。


私たちの知る限りでは、DevOpsエンジニアなどの興味深いエンティティが求められています。私はそのようなフレーズ（yes、DevOps方法論など）を実際に認識していない人たちに属しているので、このスペシャリストグループの開発パスにはいくつかの違いがあります。
まず第一に、ワークスペースであっても、一人一人が自分の興味の輪を持っていると私は強く信じています。つまり、誰かがクラウドを好きで、誰かがアプリケーションサーバーを深く掘り下げ、深いJavaを設定し、誰かがPythonでコードを書いていますまたは神はyamlコードを禁止しました。つまり、いわゆるインフラストラクチャエンジニア、ビルドエンジニア、シニアYaml開発者です:)
これにより、一方ではタスクプールに最適な人物を見つけることができ、他方では面接で誤解を生む可能性があります。
個人的な経験に基づいて、私は数十の面接を実施し、回答者としてさまざまな面接に参加したので、私の意見を共有したいと思います起こるすべて。

最初の、そしておそらく私のお気に入りのアンチパターンは、誰かがすべてを実行したいという願望です。または、誰が必要かははっきりしていません。たくさんの候補者を見てみましょう。そこで理解します。おそらくこれはどの分野にも当てはまりますが、いくつかの特徴があります。
私が気付いたように、人々はシステム管理者よりもDevOpsという単語の欠員の影響を受けやすいですが、私のシニアレベルでは、これら2つの領域のタスク領域は可能な限り異なります。
本当にシステム管理者を必要とする雇用主は、欠員ヘッダーにdevopsを書き込み、リクエストの本文、K8S / Java / gradle / oracleDBなどのすべてをリストで完全にリストしますが、内部からは、K8SクラスターのサポートとOracleDBスタックのサポートを分離して扱う必要があります。チームから。
さて、つまり、Developers / Operations形式とはどのようなやり取りですか？
さらに、チームとのこのようなやり取りのプロセスはなく、一般に、部門としての運用はなく、開発者のコ​​ンピューターを構成する必要があることが明らかになりました。
実際、このオプションは一部の応募者に適していますが、正直に言って、これは上級システム管理者です。そのため、そのように書きたくないのですが、本当に恥ずかしいのはなぜですか。職業の異なる名前間のsnの違い？ただし、会社には1つの予算があり、船には名前が付いていないため、予算に合わせて変動します。
まあ、これについても聞いたことがあります。候補者はすぐにすべてを自動化し、Pythonでの製品の開発に参加します。違いは何ですか。Pythonはどこでも同じです。世界観とアプローチの違いは考慮されていません。

さらに、私は通常、来るスペシャリストのレベルを区別し、それぞれについて私の問題を見る
ジュニア-私にとって、ジュニアDevOpsはシステム管理/開発を平均的なレベルで習得した人です。新しい分野で成長したい強いLinux Linuxユーザーと、他の開発者のために成功したい開発者を区別するのは良いことです。強力、ある種のデバッグ能力、ログの検索、またはエンコードされたプロジェクトのストックが必要です。
私は何かを試してクラウドに触れたいシステム管理者と、フロント、バック、そして何らかの理由でDevOpsプロセスに興味を示したシステム管理者の両方に会いました。
このレベルでは、Puppet、Ansibleの膨大な技術スタックがダウンし始めたとき、私はいつも混乱します-なぜ私はすべてを試していないのですか？ K8S、K3S-違いは何ですか？何種類のデータベースを知っていますか？なぜそんなに少ないのですか？暗号化はJavaでどのように機能しますか？特に開発から来た方はとても便利な人材ですが、常にこの分野で活躍しています。
これが起こったとき、私は常に昏迷に追い込まれます、私が最初に聞きたいのはなぜですか??? 2番目に頭に浮かぶのは、インタビュアーがこのような多様なスタックに関する質問に答える準備ができているかどうかです。彼らは本当にジュナを取り、それにすべてを置きたいですか？
多くの場合、これはあらゆる種類のボディショップで見られます。ある種のプロジェクトで人を販売する必要があり、履歴書でもっとクールな言葉が必要な場合や、会社が誰も連れて行きたくないが、6月の種類を調べているだけなのです。 ここでの

中間レベル
はいくつかの極端なものです。私の意見では、最初にその人が何を真ん中に引いているのかを正確に判断するのは難しいです。個人的なものはありません）
私が見た最も驚くべきことは、コーディングに深く入り、小さなpythonをダウンさせ、Java GCを苦しめることです。つまり、より具体的なトピックで、またはその逆で、長い未使用の知識のギャップを開き、ネットワーク全体でドライブし、OSドライバーのタイプ、ニヤリと膨らみます。どうすれば男はそれを忘れることができるでしょう。そして、ここからが楽しい部分です。
私の意見では、専門家は中レベルに向けて、関心の輪と彼が作業したいものの個人的な見解を形成しています-サイコロをキューブに押し込むことによって最も新鮮なスタックに飛び込むか、またはコードパフォーマンスに深く入り込む恐ろしい企業に向かってスイングします。
私の意見では、人が働いたプロセスについてここで質問し、最も興味深いものとそうでないものを尋ね、この知識に基づいて、質問をスタックに適用するのではなく、クラスタを構築する価値があります。それ以外の場合は、OpenShiftクラスターの構成について1〜2時間魅力的な会話をした後、担当者を雇って監視担当に任せます。おそらく両方がそれを好きになるでしょう。

上級レベル
ああ、私の好きなレベル。
これは、さまざまなプロジェクトで育った強力なスペシャリストで、自分が何を望んでいて、何が嫌いかをすでに知っている人です。
そして、ショーが始まります：
-システム管理に関する深い質問（最初のアンチパターンを参照）

-Linux 全体としての実践的な知識とはかけ離れた理論分野からの深い質問（OSIレベルはトップレベルの質問です）-コーディングに関する学術的な質問（インタビュアー自身は実際にフィールドを知らないため、奇妙な開発者にインタビューするように求められただけです）
ここで簡単に述べます。ある時、面接で私はいくつかのコードを書くように頼まれました。一枚の紙の上。まあ、誰もが愛するように、彼らは毎日書いて、私たちのリーフレットがすべてです。
タスクに対処した後、私のリーフレットとソリューションを確認した後、アルゴリズムが最適ではないという判断が下されました。私はインタビュアーが彼自身のアルゴリズムを書くことを提案しました、そしてそれに対して私は「これはインタビューの一部ではありません」という答えを受けました。私は少し尋ねて、小さなコードをやり直して、それがより速くなるのか遅くなるのかと尋ねました。返事をいただいた次の質問に進みます。違いは、ループ内とループなしのコードの作業にありました。そうするよりも、なぜそうする方が良いのか、答えを用意しました。さて、その後私は質問に答えたり、この人と一緒に仕事をしたくありませんでした。
私たち全員が異なっており、あなたにとって取るに足らないものは候補者を怖がらせることができることを考慮に入れなければなりません。
-通常、上級レベルのスペシャリストは明確に定義されたワーキングスタックを持っていますが、すぐに運転を開始する必要はありません。たとえば、Ansibleが書かれていて、優れていて、Puppetがあります。Puppetと呼んでいます。Puppetについて教えてください。いいね！ OpenShiftを使用しましたか？ K8がありますが、違いはわかりませんが、あなたの経験は重要ではありません。すごい！

学生にとってはすべてが簡単です。大学が何をしているか、何をすべきか、目が光っている問題を調べるのは素晴らしいことです。もし彼らが光っている場合は、なぜそれが専門家にあるのか、そしてそれについて一般的に知られていることを尋ねてください。この人のことを教えたいのであれば、その人を感じ、彼をいじり続けることがいいかどうかを理解してください。
「ITを始めたい」という人にとっては、少し厳しくなっています。人がどれだけ学んだか、面接に行く前に彼が何をしたかを確認するには、githubを見る良い選択肢がすでにあります。もちろん、コミットの密度と演習は製。同じことを聞くと、フロントエンドの方が楽しくて複雑なので、なぜそれでもdevopsなのですか？
