Memecahkan masalah permintaan N + 1 tanpa meningkatkan konsumsi memori di Laravel

Salah satu masalah utama pengembang ketika mereka membuat aplikasi dengan ORM adalah permintaan N +1 di aplikasi mereka. Masalah permintaan N + 1 bukanlah cara yang efisien untuk mengakses database ketika aplikasi menghasilkan permintaan untuk setiap panggilan ke objek. Masalah ini biasanya terjadi ketika kita mendapatkan daftar data dari database tanpa menggunakan malas atau bersemangat. Untungnya, Laravel dengan ORM Eloquentnya menyediakan alat untuk pengoperasian yang mudah, tetapi mereka memiliki beberapa kelemahan.

Pada artikel ini, kami mempertimbangkan masalah N +1, metode untuk menyelesaikannya, dan mengoptimalkan konsumsi memori.

Class User extends Authenticatable
{
    public function articles()
    {
        return $this->hasMany(Aricle::class, 'writter_id');
    }

    public function first_article()
    {
        return $this->hasOne(Aricle::class, 'writter_id')->orderBy('created_at', 'asc');
    }

    public function scopeWithFirstArticle($query)
    {
        $query->addSelect(['first_article_id' => Article::select('id')
                ->whereColumn('writter_id', 'users.id')
                ->orderBy('created_at', 'asc')
                ->take(1)
        ])->with('first_article')
    }

}

Faktanya, kami tidak menerapkan lazy loading untuk komunikasi dinamis. Kami hanya menetapkan hasil permintaan untuk menerima artikel pertama dari pengguna. Ini harus berfungsi sama baiknya saat mengakses properti first_article untuk pemuatan serakah dan pemuatan malas.







Koneksi Dinamis dalam Laravel 5.X



Sayangnya, solusi kami hanya berlaku untuk Laravel 6 ke atas. Laravel 6 dan versi sebelumnya menggunakan implementasi addSelect yang berbeda. Untuk digunakan dalam versi kerangka kerja yang lebih lama, kita perlu mengubah kode kita. Kita harus menggunakan selectSub untuk menjalankan subquery:







public function scopeWithFirstArticle($query)
{
    if(is_null($query->toBase()->columns)) {
        $query->select([$query->toBase()->from . '.*']);
    }

    $query->selectSub(
                Article::select('id')
                ->whereColumn('writter_id', 'users.id')
                ->orderBy('created_at', 'asc')
                ->take(1)
                ->toSql(), 
            'first_article_id'
            )->with('first_article');
}
