Linux Install Fest-側面図

数日前、ニジニノヴゴロドで「インターネットの制限」の時代の古典的なイベントがありました-Linux Install Fest 05.19。



このフォーマットは、NNLUG（Linuxユーザーの地域グループ）によって長い間（2005年まで）サポートされています。

今日では、「ねじからねじへ」をコピーしてブランクを新しい配布で配布することは、もはや慣例ではありません。インターネットは誰にでもアクセス可能であり、文字通りすべてのティーポットから輝きます。
同時に、教育的要素は引き続き重要です。今回はその重要性を確認しました。

主催者は、Linuxとフリーソフトウェアの分野からの興味深いトピックについて話すように講演者を招待しました。その結果、最終的なリストには、深刻な「管理」タスク、グラフィック、ゲームセクター、オーディオおよび音楽アプリケーションが含まれていました。

スピーカーがNNLUGサイトでトピックを登録している間、主催者はHabréを含む発表を行いました。すぐにタスクリストがGDで編集され、準備に参加したい人なら誰でも参加できます。

主催者は何を計画しましたか？


8つのレポート、新しいLinuxディストリビューションの使用デモ、さまざまな口径のゲームスタンド、そして後で判明したように、ファイナルのミュージックセッション。

これはすべてNTRCの広々とした集会ホールにあり、まともな音響と隅にティーテーブルがあります。

少し遅れて、イベントは正式に始まりました。次は、レポートに関する主観的な見方です。6時間のストリームのレコーディングで、彼らと完全に知り合うことができます。



Denisが興味深いMeshroomパッケージについて語っています。要するに、プログラムはさまざまな角度からのオブジェクトの50〜100枚の写真に基づいて、形成されたテクスチャで3Dモデルを構築します。チューニングのニュアンスと結果が示されました。



ウラジミールは、Proxmox VEについて言及する仮想重力を追加しました：主にユーザーケースと一般的な印象。このdebianベースのツールのリリースは頻繁ではありませんが、それをサポートして更新するには、より特別な条件が必要です。



イノセントは、子供向けの優れたゲーム開発パッケージGComprisについて語った。このソフトウェアは、3歳以下の小さな人々にとって手頃な価格で、さまざまな種類の小さなゲームが含まれています。ゲーム形式は、子どもの発達をさらに刺激する可能性があります：視野を広げ、論理を発達させ、自閉症の子どもへのアプローチ。

Denis（彼の2番目のレポート）によると、Blender 2.8の新しいバージョンは、一部のアナログよりも安定しています。機能の調整。インターフェイスの変更。



Artyom Kashkanov（Radiolok）Nextcloudをたたえます。彼はそれを長い間使用し、非常に成功していると言います。ローカルの「DropBoxアナログ」を用意することをお勧めします。

Artyom Poptsov（avvvp）素晴らしいAudacityに言及し、正しい正弦波を示すことによって哲学的な種を導入しました。優れたパッケージ、優れたコンプレッサー、Linuxでの初期サウンド処理の事実上の標準（私の個人的な意見）。

オーディオ処理の詳細については、Ilyaによる次のレポートで説明およびデモされました。彼はプロのミュージシャンおよび作曲家として、Ubunty Studioを彼の作品で成功裏に使用しました。ストーリーでは、Linuxのオーディオシステムの基本と、より具体的なSupercolliderおよびPure Dataパッケージに触れました。



最終的に、FedorはフリーソフトウェアのパラダイムをN次元で展開し、緊張することなく完全に「緊張した数字」を出しました。歴史的背景、事実、比較-レポートは、FOSSの陰湿な批判であることが判明しました。一連のフリーソフトウェア支持者は傷つけられ、次第に物語は「火花を浴びたまま」で本当に状況に無関心ではない人々の円卓になりました。



インストールゾーン（最後の行で2、3席）で、ミニプロジェクト「Install FreeDOS on PentiumMMX」が沸騰し始めました。同時に、アイアンマシンには20GBのIDE HDDしかなく、USBはありませんでした。 DVD ROMは手元にありませんでした。

FreeDOSの公式イメージのフォーマットはIvanに助けられました。



それからジャンパーとのちょっとした大騒ぎがありました。DIN取り外し可能キーボードが機能しないことが判明しました-両方のEnterキーが押されませんでした...ローカルCADRハッカーが救助しました。シェルフのシェルフはまったく同じものであることが判明しました。確かに、時間は失われ、「テスター」はシステムインストーラーのHDDからの起動のみを達成しました。同じHDDにシステムをインストールする方法がすぐにわかりませんでした。プロジェクトは次のフェスティバルまで残っているようです。
