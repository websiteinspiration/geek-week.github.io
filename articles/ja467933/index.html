成人検査の可能性

多くの人が、その形成時の革命的なアイデアが専門家のコミュニティによって拒否された歴史から、少なくとも1つのケースをすぐに思い出すことができると思います。原則として、そのような振る舞いの欠点は、すでに蓄積された知識の膨大な手荷物であるため、古い問題を新しい観点から見ることはできません。したがって、新しいアイデアは、開発の前の段階で重要であると見なされたメトリックによってのみ評価されるため、確立されたアプローチの特性の点で失われます。

これは、Positフォーマットが今日直面している一種の嫌悪感です。批評家は、しばしば「間違った方向を見る」だけでなく、実験でPositを間違った方法で使用することさえあります。この記事では、その理由を説明します。

Positの利点はすでにかなり言われています：数学的な優雅さ、低指数値での高精度、広範囲の値、NaNとゼロの1つのバイナリ表現のみ、非正規値の不在、オーバーフロー/アンダーフローとの戦い。批評家はかなり声を上げてきました：非常に大きい値または非常に小さい値の役に立たない精度、複雑なバイナリ表現形式、そしてもちろん、ハードウェアサポートの欠如。

私はすでに言った議論を繰り返すのではなく、原則として見落とされている側面に焦点を当てようと思います。

ゲームのルールが変更されました


IEEE 754標準は、約40年前にIntel 8087に実装された浮動小数点数を記述しています。私たちの業界の基準によると、これは信じられないほどの時間です。その後、プロセッサのパフォーマンス、メモリコスト、データ量、計算規模など、すべてが変化しました。 Positフォーマットは、IEEE 754の最良のバージョンとしてだけでなく、当時の新しい要件を満たす数値を処理するためのアプローチとして開発されました。

高レベルのタスクは同じままです-精度の損失を最小限に抑えて、有理数のフィールド全体で効果的な計算が必要です。しかし、課題が解決される状況は根本的に変化しています。

まず、最適化の優先順位が変更されました。 40年前、コンピュータのパフォーマンスは、ほぼ完全にプロセッサのパフォーマンスに依存していました。今日、ほとんどの計算のパフォーマンスはメモリにかかっています。これを確認するには、ここ数十年のプロセッサ開発の主要な領域を見てください。3レベルのキャッシング、投機的実行、計算のパイプライン処理、分岐予測などです。これらのアプローチはすべて、高速コンピューティングとメモリへの低速アクセスの条件下で高性能を達成することを目的としています。



第二に、新しい要件、つまり効果的なエネルギー消費が前面に出てきました。過去数十年にわたって、コンピューティングの水平スケーリングのテクノロジーは非常に進歩してきており、これらの計算の速度ではなく、電気代に関心を持ち始めました。ここで、理解しておくべき重要な部分を強調しておきます。エネルギー効率の観点からは、プロセッサのレジスタがコンピュータに非常に近いため、計算は安価です。プロセッサとメモリの間（x100）と長距離（x1000 ...）の両方でのデータ転送の料金ははるかに高くなります。



これは、Positが使用することを計画している科学プロジェクトのほんの一例です。



この望遠鏡ネットワークは、200ペタバイトを生成します1秒あたりのデータ、その処理には小さな発電所の電力が必要です-10メガワット。明らかに、そのようなプロジェクトでは、データとエネルギー消費の削減が重要です。

最初に


では、Posit標準は何を提供するのでしょうか？これを理解するには、引数の最初に戻って、浮動小数点数の精度の意味を理解する必要があります。

実際には、精度に関連する2つの異なる側面があります。最初の側面は、計算の精度です。計算の結果がさまざまな操作中にどれだけ逸脱するかです。 2番目の側面は、プレゼンテーションの正確さです。つまり、有理数のフィールドから特定の形式の浮動小数点数のフィールドに変換するときに、元の値がどれだけ歪んでいるかです。

今、意識の重要な瞬間があります。Positは主に有理数を表すための形式であり、それらに対して演算を実行する方法ではありません。言い換えると、Positは非可逆の有理圧縮形式です。32ビットのPositが64ビットのFloatの代わりになるという主張を聞いたことがあるかもしれません。したがって、同じ数のセットを格納および送信するために必要なデータ量を半分にすることについて話している。メモリアクセスが2分の1になります。メモリアクセスの期待値が低くなるため、消費電力が約2分の1になり、プロセッサのパフォーマンスが向上します。

スティックの2番目の端


ここでは、論理的な質問が必要です。高精度で計算を実行できない場合に有理数を効率的に表すポイントは何ですか。

実際、正確な計算を行う方法があり、Quireと呼ばれています。これは、Positと密接に関連する有理数を表す別の形式です。Positとは異なり、Quireフォーマットは、計算および中間値をメインメモリではなくレジスターに格納するために特別に設計されています。



つまり、Quireはワイド整数バッテリー（固定小数点演算）にすぎません。単位は、Quireのバイナリ表現として、Positの正の最小値に対応します。最大Quire値は最大Posit値に対応します。各Posit値は、精度を失うことなくQuireで一意の表現を持っていますが、すべてのQuire値が精度を失うことなくPositで表現できるわけではありません。

Quireの利点は明白です。これにより、Floatよりも比類のない精度で演算を実行できます。また、加算および乗算演算では、精度がまったく失われません。これに対して支払う必要のある価格は、広いプロセッサーレジスター（es = 2の32ビットPositは512ビットQuireに対応）ですが、これは最新のプロセッサーでは深刻な問題ではありません。そして、40年前に512ビット整数を超える計算が許容できない贅沢に思えた場合、今日のメモリへの幅広いアクセスに代わる適切な代替手段となりました。

パズルを組み立てる


したがって、PositはFloat / Doubleに代わる形式の新しい標準を提供するだけでなく、数値を処理するための新しいアプローチを提供します。Float-精度とストレージ効率と計算効率の間の妥協点を見つけようとする単一の表現であるのとは異なり、Positは2つの異なる表示形式を提供します。1つは数値の格納と送信用-Posit自体と、もう1つは計算とその中間用値-必要なもの。

浮動小数点数を使用して実際的な問題を解決する場合、プロセッサーの観点から、それらの操作は次のアクションのセットとして表すことができます。

  1. メモリから数値の値を読み取ります。
  2. 一連の操作を実行します。操作の数が非常に多い場合があります。この場合、計算のすべての中間値がレジスターに格納されます。
  3. 操作の結果をメモリに書き込みます。


Float / Doubleでは、すべての操作で精度が失われます。Posit + Quireでは、計算中の精度の低下はごくわずかです。Quire値をPositに変換するときに、最終段階でのみ失われます。これが、Posit + Quireの「エラー累積」問題のほとんどが単に関連しない理由です。

Float / Doubleとは異なり、Posit + Quireを使用すると、通常、数値をよりコンパクトに表現できます。その結果、メモリからのデータへのアクセスが高速になり（パフォーマンスが向上）、情報の保存と送信がより効率的になります。

道徳


多くの実際的なケースで、正しく使用すると、Positフォーマットは、一方でメモリを節約し、Floatよりも数値を圧縮し、Quireのおかげで計算精度を高めることができます。

しかし、これは単なる理論です！精度やパフォーマンスについては、いずれかのアプローチを盲目的に信頼する前に、必ずテストを行ってください。実際、実際には、あなたの特定のケースは、理論よりもはるかに例外的であることが判明します。

まあ、クラークの最初の法則（自由解釈）を忘れないでください。尊敬され経験豊富な専門家が新しいアイデアがうまくいくと主張するとき、彼はほぼ間違いなく正しいです。新しいアイデアがうまくいかないと彼が主張するとき、彼は間違いを犯される可能性が非常に高いです。私はあなたが私の意見に頼ることができる経験豊富な専門家だとは思っていませんが、経験豊富で尊敬されている人々でさえも批判しないように注意してください。結局のところ、悪魔は詳細にあり、経験豊富な人々でさえそれらを見逃す可能性があります。
