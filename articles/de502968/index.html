Grundlagen des Datentresors

Derzeit ist es im Bereich Datenanalyse und BI nicht mehr möglich, ein Konzept wie DATA VAULT nicht zu erfüllen. Meiner Meinung nach fehlen jedoch einige Informationen zu diesem Thema, insbesondere im russischsprachigen Segment des Internets.


Sie finden interessante Artikel zur Anwendung von DATA VAULT in Unternehmen, aber die Grundlagen und Methoden werden nicht ausreichend behandelt.


Im englischen Segment sieht es viel besser aus. Sie können Bücher von Autoren und Erfindern der DATA VAULT-Methodik kaufen, aber es gibt auch öffentlich zugängliche Artikel, die sich auf die Grundlagen konzentrieren.


Ich werde mich von einem dieser Artikel inspirieren lassen und versuchen, die grundlegenden Dinge der DATA VAULT-Methodik auf Russisch zu vermitteln.


DATA VAULT - die Ursprünge


Die Hauptvoraussetzung für die Entstehung von DATA VAULT war die zunehmende Variabilität der Umgebung und die Notwendigkeit, schnell auf diese Änderungen zu reagieren. Beispielsweise gibt es in EDW (Enterprise Data Warehouse) eine neue Datenquelle mit zuvor detaillierter Datengranulation. Es wird davon ausgegangen, dass Sie mit der DATA VAULT-Methode schnell Daten aus einer neuen Quelle hinzufügen können. Darüber hinaus ist es mit DATA VAULT einfacher, ein System zu erstellen, mit dem Sie historische Daten speichern können.
