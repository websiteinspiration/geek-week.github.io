SIPコネクタテレグラム+ Voximplant =携帯電話およびソフトフォンへの通話


テレグラム用にSIPコネクタ（@siptg）が開発されました。Voximplantは電話プラットフォームです。そうです、彼らは完全に一緒に働くことができ、今日はその方法を示します。猫へようこそ！

Voximplantにソフトフォンを登録する


SIPコネクタは、ソフトフォンとSIPゲートウェイの2つのモードで動作します。Voximplantアプリケーションの内部ユーザーの「偽装」ボットを登録できるため、最初のオプションを検討します。そのため、まずVoximplantアプリケーションとその内部のユーザーを作成する必要があります。

  1. アカウントhttps://manage.voximplant.com/にログインします。
  2. 「アプリケーション」セクションに移動して、sipというアプリケーションを作成します。
  3. アプリ内から[ユーザー]タブに移動し、ユーザーテレグラムユーザーを作成します。パスワードを忘れないでください。すぐに必要になります。

  4. ここでuserbobユーザーを作成します（後で少し役立ちます）。

. @siptg : . Voximplant : usertelegram, userbob .. camelCase, , .
今、私たちはメッセンジャーの側に行きます。ソフトフォン自体は@siptgという名前で、その設定は@siptg_botという名前であり、それらを開きます。Startを押してから/ tariffコマンドを入力してFreeを選択します（これで十分ですが、制限があります-たとえば、5回の呼び出しが失敗した後、一時的なブロックが発生します。正確なロック解除時間は、/ tariffコマンドを使用して確認することもできます）。

次に、/ softphoneコマンドを使用して、[新規]ボタンをクリックします。ここでは、Voximplantユーザーデータを1つずつ入力して、[適用]をクリックします。サーバーフィールドはsipで構成されます。+ Voximplantアカウント名+ .voximplant.com：


すべてが正しく入力されると、登録されたクライアントが緑色で強調表示されます。


さて、私たちは彼らが電報に来るかどうかを確認することができます...

…着信


これを行うには、Voximplant側のSIPアプリケーションに移動して、スクリプトと呼び出しを処理するためのルールを作成する必要があります。シナリオは単純です。sipアプリケーションのユーザーからクラウドへの呼び出しが発生すると、クラウドは同じアプリケーションの2番目のユーザーを呼び出して接続します。次のようになります。

VoxEngine.addEventListener(AppEvents.CallAlerting, e => {
   const inc = e.call
 
   const out = VoxEngine.callUser(e.destination, e.callerid)
 
   VoxEngine.easyProcess(inc, out)
})

スクリプトをcallUserとして保存します。そのままの状態で、SIPアプリケーションで[ルール]タブに切り替えて、次の設定で通話ユーザールールを作成します（マスクに注意してください）。


最初に作成したuserbobユーザーを覚えていますか？ユーザー名とパスワードを使ってログインしてください。


usertelegramを入力して[通話]をクリックします。インスタントメッセンジャーを使用しているすべてのデバイスの@siptgボットから着信通話を受信します。


Voximplantユーザーへの発信


シナリオとルールは同じで、新しいものを作成する必要はありません。
上で述べた唯一のニュアンスは、Voximplantのユーザー名は小文字でなければならないということです。ユーザー名がuserBobの場合、指定すると、@ siptgボットは最終的にそれをuserbobに変換し、Voximplantアプリケーションにそのようなユーザーは存在しません。その結果、ダイヤルアップエラーが発生します。
@siptgボットを開き、userbobメッセージを送信すると、ボットからの着信の疑いがあります（逆に、そのままです）。


緑色の受話器をクリックすると、コールはブラウザのソフトフォン、つまりuserbobに送られます。


セル番号への発信


電報ボットが固定電話番号に電話をかけるには、次のものが必要です。

  • もう1つのシナリオとルール。
  • 有効なコールバックの電話番号（発信者ID）。これは、パネルで確認した個人の電話番号か、Voximplantから購入した番号のいずれかです。テスト番号はこの目的には適していません。

SIPアプリケーションで、次のコードを使用してcallPSTNスクリプトを作成します（4行目に注意してください-ここでは有効な番号に置き換える必要があります）。

VoxEngine.addEventListener(AppEvents.CallAlerting, e => {
   const inc = e.call
 
   const out = VoxEngine.callPSTN(e.destination, "real callback number")
 
   VoxEngine.easyProcess(inc, out)
})

次に、通話ルールPSTNルールを作成します。デフォルトではマスクをそのままにしておくことができます。


重要：この新しいルールをルールのリストの下に移動します。したがって、彼の優先順位は下がり、ユーザーのルールが守られない場合にのみ機能します。


さて、電話をかけることができます。ターゲットの電話番号を@siptgボットに送信します。これにより、通話インターフェイスが表示され、緑色の受話器を押すと、指定された携帯電話/都市番号に電話がかかります。

結論


テレグラムをクラウドテレフォニーと統合することは、これが顧客に連絡するもう1つの方法であるオムニチャネルの読み取りであるため、ビジネスに便利です。電報が企業のメッセンジャーである場合は、内部的に適用することもできます。おそらく、個人的なニーズのために、これも要求されるかもしれません。電報を発信する機能は、このメッセンジャーを使用する場合に関係があり、発信したいメッセンジャーは使用しません。その結果、コミュニケーションのための環境を変更する必要はありません。通話はメッセンジャーから直接送信され、対話者は何も変更せず、ソフトフォン/電話への通話を受信します。
