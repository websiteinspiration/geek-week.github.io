フォルマントP432の内面の世界



思いがけずシンセサイザーを購入して修理し、仕組みを理解してエミュレータを作成したという話です。

このストーリーは、2019年8月の先週末のカオスコンストラクションフェスティバルで始まりました。夕方、過ぎ去った時代の興味深いアーティファクトを探してAvitoを眺めていたところ、安価なFormant P432シンセサイザーに出くわしました。それはデジタルシンセサイザーに非常に馴染みのあるものに見えました。下部には41キー（3.5オクターブ）のキーボードがあり、上部にはトーンを選択するボタン、コーラスとユニゾンエフェクトをオンにするボタン、ビブラートエフェクトを調整するボタンがあります。ジョイスティックはキーボードの左側に取り付けられており、ピッチを上下に移動したり、ビブラートをスムーズに接続したりできます。

私にとって最も興味深いことは、このシンセサイザーはデジタルであり、ウェーブテーブル方式を使用してサウンドを生成したことです。つまり、キーを押したときに、ROMチップに格納された表形式のデータに基づいてサウンドを生成しました。インターネット上の回路をすぐに知った後、シンセサイザーの制御はよく知られたマイクロプロセッサーKP580VM80（i8080）に基づいており、サウンドは8つのROMチップに保存され、合計ボリュームは8 * 2 = 16 Kbです。

シンセサイザーが購入され、ゆっくりと売り手の街から運送会社の地方事務所へと出発しました。そして私はその計画を研究し続けました。販売者は広告に、このツールには複数のキーを押すと喘鳴という形で現れる欠陥があると書いています。いっそうおもしろいと思った。 「それを見つけて排除する必要があります。」

しかし、購入の主な動機は、内部デバイスを処理し、シンセサイザーのROMからサウンドのサンプルを抽出することでした。

免責事項：シンセサイザノードと信号のコンポーネントの名前（「引用符」と大きな文字で示されているものを除く）は私が開発したものであり、開発者の意図とは異なる場合があります。ノードの操作の説明は、私の（限られた）知識、インターネットからの資料、および経験豊富な人々のアドバイスに基づいています。

ROMを備えた別のボード上のMIDIノードが削除されたのは、P432ラインに別のシンセサイザー、MIDIインターフェイスとHORエフェクトを持たない「フォルマントMINI」が存在するためです。 MINIプロセッサノードはファームウェアが少し異なり、ジェネレータノードはP432と同じです。

そのため、初めて電源を入れたときに、2つの問題に気づきました。リニア出力の信号レベルが非常に弱いことと、一度に複数のキーを押したときの前述の音です。なお、P432はポリフォニックシンセサイザーで、1つの音色の4つのボイス（周波数）を同時に再生できます。また、UNISONエフェクトを含めると、同時に発声されるボイスの数が8に増え、キーごとに2つになります。

静かな音の理由はほとんどすぐにわかりました。焦げた出力OU K544UD1Aが原因でした。

交換後、パッシブヘッドフォンが出力に接続されていても、楽器の音が大きくなり始めました。

しかし、奇妙な音では、個々のキーが押されても音の音がyoutubeのP432デモのようではなかったため、状況は非常にわかりにくいものでした。

問題を見つけるのにかなり時間がかかりましたが、最終的にはメインDACのレベルにローカライズされました。デジタル入力の上位ビットが論理1を認識するのに不十分だったため、アナログ出力信号が歪んでいました。この話の犯人が発見され、排除された後、問題は解決されました。数回タップするだけで、すべての音が問題なくきれいに聞こえ始めました。

歴史には終わりがありますが、すでに修理に多くの時間が費やされていたので、そこで止めるべきではないと判断し、それを理解する必要があります。

マルチバイブレーター、カウンター、ロジックエレメントを使用した同期回路は、ジェネレーターユニットのさまざまなモジュールの制御パルスを生成し、現在の音声番号を出力「アドレス」に発行します。 UNISON「アドレス」モードがアクティブな場合、値は[0,1,2,3,4,5,6,7]で、非アクティブの場合は[0,2,4,6]です。マスターオシレーターの周波数は、フロントパネルの調整つまみとキーボードの左側にあるジョイスティックで少し変更できます。

クロックパルスのタイミング図。説明：Unison-ログ。 0アクティブレベル、FA0-2-「アドレス」出力、DD8-12-位相アキュムレータレジスタ、DD33-35-投票トータライザレジスタ、DD36-37-DACレジスタ。


「アドレス」出力の値の変化と同時に、4（8）票のそれぞれのジェネレーターボードのデジタル部分は、次の操作を実行します。プロセッサノードから「周波数」入力は信号周波数値を受け取ります。周波数の線形値は、ROMのデコーダーによって位相増分に変換されます。増分は、加算器と8ワードRAMで構成される位相アキュムレータ（20ビット）で合計されます。合計の最上位9ビットは、波形ROMのアドレス入力に送られます。特定のROM領域は、「制御」入力の最下位3ビットによって選択されます。 ROMの出力からの10ビットは、「アンプ」の入力での値と合計され、DACのバイポーラコードを生成するスケーリングROMの入力に送られます。出力は、すべての4（8）票のコード値を順番に合計する、加算器とレジスターに構築された累積バッファーに分類されます。各サイクルの終わりに、最終コードはストレージレジスタによってラッチされ、12ビットDACに到達します。 DACでコードを発行する周波数は信号のサンプリング周波数で、P432の場合は約33キロヘルツです。ナイキストによると、P432ジェネレーターで使用できる最大音響周波数は約16.5 kHzです。

選択したトーンがローパスフィルターパラメーターの動的な変更を必要とする場合、エンベロープ値はフィルター入力に出力され、10ビットDACを通過して、オペアンプフィルターの制御電圧を形成します。ローパスフィルターは、可変状態フィルターの非コンデンサーフィルターに従って作成され、温度安定化回路、QのQ係数を変更するためのブロック、および初期バイアスを含みます。


このタイミングチャートは、ジェネレータボードの2つの離散化期間を示しています。UNISONはオフです。アドレスの変更と同期して周波数入力（Freq_in）は、4つの異なるキーの周波数番号を受け取ります。ただし、最後の2つだけが押されます（Volume_in = 0xFF）。同時に、定数値（Filter_in = 0x60）が「フィルター」入力に発行されます。最後のコードは、ストレージレジスタ（DD36-37）にスナップした後、DAC（DAC）に入力されます。

CPU構成とノード


そこで、ジェネレーターを見つけました。ここで、すべての制御信号がどこから来て、それらを記述するパラメータがどこに保存されるかを理解する必要があります。

最初は、ロジックアナライザーですべての制御信号のダンプを削除し、それらを記述するアルゴリズムを考え出すことを計画していました。たとえば、アンプとフィルターのエンベロープは、限られた数のパラメーターを必要とする区分的線形関数で表すことができます。

「周波数」入力を使用すると、各音色の単一のキーの周波数値を記録し、それに基づいて他のすべての音を計算することでも十分でした。

取扱説明書によると、P432には41キーのキーボードがあり、音量は3および5/12オクターブで、範囲は科学表記のC1-E4またはヘルムホルツ表記のC1-e1です。ただし、実際には（トーン00の場合）対応A4 = 440 Hzに基づいて、左2オクターブのキーAがこの周波数の音を生成することがわかりました。後で判明したように、アクティブなトーンに応じて、再生される周波数の範囲は1または2オクターブシフトして変化する可能性があります。

また、可能なすべてのトーンについて、ロジックアナライザーによって「コントロール」入力からのバイト値を記録し、それらをプレートにもたらしました。

ボイスとコントロール入力値のリスト

これらの構成パラメータはすべてシンセサイザーROMの1つに保存する必要があることは明らかで、少し検索した後、それらはプロセッサノードのDD16ファームウェアの後半で見つかりました。ROM 573RF2の容量は2048バイトです。



構成領域の1024バイトは簡単に32トーンに分割され、各ティンバーの設定に32バイトが割り当てられました。トーン設定とエンベロープの実際のフェーズ、サウンドの周波数などの間の相関関係を探すのにかなりの時間を費やしました。ロジックアナライザーの多くのダンプを見なければなりませんでした。

私が最初に見つけたアンプのエンベロープの例を使用して、それらをより詳細に検討してみましょう。別の方法で正しいエンベロープを取得できなかったため、ハードコードを使用する必要があったいくつかのトーンを除いて、フィルターエンベロープは同様に形成されます。

接尾辞* _dtが付いたパラメーターは、対応するフェーズの信号レベルの変化のデルタを指定します。たとえば、トーン00の場合：A_min = 0、A_dt = 48およびA_max = 255。つまり、Attack 0の初期値は、255に達するまで48ずつインクリメントする必要があります。立ち下がりフェーズでは、S_ft> 0の場合、D_dtの値をA_maxからレベルS_lvlまでデクリメントする必要があります。サポートレベルは、S_ft間隔ごとに1ずつ減少する必要があります。スムーズにフェードする音。 R_dtパラメータは、減衰フェーズのデルタを示します。現在のエンベロープレベルは、レベル0に到達するまで、時間間隔ごとにR_dtずつ減らす必要があります。パラメータ* _dt <= 1の場合、実際のdtは1で、その他の場合は値* _dtです。

* _ftパラメーターは、値が変更されないままでいる必要がある時間を決定します。これは、エンベロープのサンプリング間隔の一種です。経験的に、このパラメータのゼロ値は、〜100 DACサンプリング周期または〜3 msに対応することがわかりました。ゼロより大きい値の場合、時間は式int（256 / * _ft）* 100を使用して計算する必要があります。 * _ftは、100 DAC周期の基本間隔の乗数です。

すべての封筒オプションを示す図。右側では、パラメータ* _ft / * _ dtの本質をよりよく理解するために、明示的な離散化で複製されています。



構成領域の「管理」バイトは、ロジックアナライザーを使用して経験的に受け取ったものと一致しました。

周波数バイトは、周波数入力の値を決定します。すでに上で述べたように、シンセサイザーは4つのボイスを同時に再生できます（ポリフォニー）。それぞれのUNISONをオンにすると、わずかに離調またはメインオクターブから1〜2オクターブずれた別のジェネレーターを同時に追加できます。

周波数は2バイトでエンコードされます。メインの場合はFREQ_1、FREQ_2、ユニゾンの場合はFREQ_u1、FREQ_u2です。 P432キーボードの2オクターブのノートAの周波数値を計算するための実験式は、実験的に導出されました。

freq_offset = {4:0xd0, 5:0x90, 6:0x50}
freq = (config['FREQ_1'] << 8) + freq_offset[config['FREQ_1']] + config['FREQ_2']

トーン00の例。FREQ_1= 6、FREQ_2 = 8、次のようになります。6* 0x100 + 0x50 + 8 = 0x658。これは、すべてのトーンのロジックアナライザーデータと一致しています。ユニゾンの場合、計算は似ています。

トーン00の場合、値0x658は440 Hzの周波数またはノートA4に対応することに注意してください。
隣接する（黒を含む）キーの「周波数」間隔は16（0x10）単位であることが実験的に確立されました。したがって、左側の22番目のキーの値（2番目のオクターブのA）-0x658がわかっている場合、左端（No. 1）の値：0x508と右端（No. 41）の値：0x788を簡単に取得できます。

これに基づいて、キーボードの範囲とさまざまなトーンの再生周波数を計算できます。「Frequency」列は、単純な波形に対してのみ正しいです。

PythonでのP432エミュレーションの作成


設定の保存とサウンドの生成を分離した方がいいと思い、2つのメインクラスを作成しました。

  • P432data、ROMイメージから値をロードし、トーンの構成を便利な形式で使用できるようにします。
  • P432。エンベロープ形状を作成し、実際にデジタルサンプルを生成する方法が含まれています。
P432dataについては詳しく説明しません。ソースコードはgithubで確認できます。
しかし、P432の方法はより詳細に検討されています。

もちろん、主なものはgen_samplesで、返すサンプルの数を示すパラメーターが含まれています。この方法では、入力「周波数」での値の受信からDACへの出力までの信号の処理が、プロトタイプに可能な限り近く再現されます。原則として、P432スキームに焦点を当てると、ジェネレーターユニットのモジュールに相当するPythonをすばやく見つけることができます。明示的に追加する必要があったのは、変数を所定の次元に削減することだけでした。位相アキュムレータに20ビット、エンベロープ加算器に11ビット、投票加算器に12ビットです。この制限は、これらの変数のオーバーフローのロジックが正常に機能するために必要です。

エンベロープアンプgetADSおよびフィルターgetFADSフォームを作成するメソッドは、キーを押したときにgen_samplesが値をとる振幅エンベロープの値をリストします。彼らは社内にgetRメソッドとgetFRメソッドをいくつか持っており、エンベロープ減衰フェーズを形成します。それらは、レベルパラメーターをエンベロープレベルの現在の値として受け入れ、0から開始します

。4つのメインボイスは、アンプとフィルターのエンベロープ形状が同じですが、アンプごとに独自のエンベロープがあり、キーを押すと開始します。フィルターの場合、すべての声に共通です。最初のプレスから始まり、次の圧力で再開し、最後のリリースまで続きます。上記を参照

取扱説明書によると、P432はキーボードのキー1から41に対して、それぞれ36から77（実際には76）のMIDIノート番号を発行します。ホイールを再発明しないために、私は押されたキーの番号としてエミュレータで36〜76の範囲を使用することにしました。そのため、選択したトーンを考慮して、キー番号を「周波数」値に変換するmidi2freqメソッドが必要でした。

メソッドの最後のペアは、key_press（midi_note）とkey_release（midi_note）でした。これらは、音声と基本音の周波数、エンベロープの現在の位置、位相との対応の辞書を形成しました。

最後のメインメソッドはsetprog（prog、clean = True）で、アクティブなトーンを選択するために使用されます。Setprog目的のトーンの構成をロードし、クリーンフラグがクリアされていない場合は、内部変数をクリアします。 cleanがFalseに設定されている場合、内部変数はゼロ化されないため、実質的にクリックやその他の不快なサウンドなしに、サウンド生成中にトーンを切り替えることができます。

生成されたサンプルをWAVファイルに書き込むために、waveモジュールに基づいてwrite_wavユーティリティメソッドが作成されました。各サンプルについて、スケーリングを実行して振幅を増やし、符号付き16ビット形式にします。

これらの方法は、シンセサイザーサウンドを数秒録音したwavファイルを取得するのに十分です。

#   
p432=P432()
#  
p432.setprog(0)
#     (  )
p432.unison=False
p432.filter_en = False
#   A  
p432.key_press(57)
#  3  
samples=p432.gen_samples(33000*3)
#  
p432.key_release(57)
#   
p432.write_wav('test_00.wav',samples)

ティンバー00の波形は正弦波なの​​で、出力ではエンベロープが重畳された正弦波が得られます。聞いて。



もう少し興味深いのは、ユニゾンとフィルターをオンにしたトーン14の録音です。聞いて。



GUI


コンソールのサウンドジェネレーターは良いですが、キーボードでも演奏したいです。そこで、TkinterでシンプルなGUIを作成することにしました。

このインターフェースは、キーボードの凡例、MIDI入力選択ボタン、およびフィルターをオフにする機能が追加され、実際のP​​432の影響下で作成されました。


python-rtmidiモジュールがインストールされている場合、Midi Inボタンが表示されます。それを押してMidi入力デバイスを選択すると、プログラムは、16のMIDIチャンネルすべてでNote-On、Note-Off、Program Changeのイベントをキャッチするコールバック関数をアクティブにします。 Program Changeにより、アクティブなエミュレータトーンの数を変更できます。

キーボードのキーストロークをインターセプトするには、Tkinter bind（ '<KeyPress'）、bind（ '<KeyRelease'）メソッドを使用し、コールバック関数で許可されたキーのリストを使用してキー番号が確認され、MIDIノートに変換され、p432.key_press（）メソッドが呼び出されます、p432.key_release（）。インターセプトはプログラムの開始時にアクティブになり、最後に非アクティブになります。

ストリーミングサウンドを出力するために、pyaudioモジュールが非ブロッキングモードで使用されます。インターフェイスをレンダリングした後、別のスレッドスレッドで開始し、tkinterは主に機能し続けます。コールバック関数では、pyaudioは生成されるサンプルの数を渡し、それに応答してgen_samplesとステータスフラグからサンプルを受け取ります。（キーを押すことによる）ノートオンイベントとサウンドの出現との間のレイテンシを短縮するために、200サンプルというかなり小さいバッファが選択されています。これは、open pyaudioメソッドのframes_per_bufferパラメーターによって設定されます。

P432_emulator for Windowsのコンパイルされたpyinstallバージョンは、リリースからダウンロードできます。
