Android- und 3D-Kamera. Gesichtserkennung mit Betrugsschutz

Hallo! Mein Name ist Vladimir Shalkov, ich bin ein Android-Entwickler bei Surf.



Vor nicht allzu langer Zeit mussten wir auf Android ein Gesichtserkennungssystem mit Betrugsschutz implementieren. In diesem Artikel werde ich die interessantesten Aspekte der Implementierung mit Codebeispielen und Links teilen. Ich bin sicher, Sie werden etwas Neues und Interessantes für sich finden, also lehnen Sie sich zurück und fangen Sie an.







Gesichtserkennungssysteme werden immer beliebter: Die Anzahl der Geräte mit Gesichtsentsperrfunktion sowie die Anzahl der Tools für Entwickler nimmt zu.



Apple verwendet FaceID in seinen Produkten . Außerdem haben sie sich um die Entwickler gekümmert und eine API hinzugefügt, um auf diese Funktionalität zuzugreifen. FaceID gilt als ziemlich sicher und kann zum Entsperren von Bankanwendungen verwendet werden. Android SDK hatte bis vor kurzem keine fertige Lösung. Obwohl Gerätehersteller die Möglichkeit hinzugefügt haben, das Gerät mit ihren Gesichtern in ihrer Firmware zu entsperren, konnten Entwickler die Funktionalität in Anwendungen nicht nutzen, und die Sicherheit dieser Methode zum Entsperren ließ zu wünschen übrig.



Vor kurzem KlasseDem FingerprintManager , mit dem Anwendungen per Fingerabdruck entsperrt wurden, wurde API 28 und höher zugewiesen. Entwicklern wird empfohlen, BiometricPrompt zu verwenden . Diese Klasse enthält Logik in Bezug auf Biometrie, einschließlich der Identifizierung von Personen. Es funktioniert jedoch nicht auf jedem Smartphone, da das Gerät nach Angaben von Google eine hohe Sicherheitsbewertung aufweisen muss.



Einige Geräte verfügen nicht über einen eingebauten Fingerabdruckscanner. Sie lehnten diesen aufgrund des hohen Schutzes vor Betrug bei der Gesichtserkennung und dank des nach vorne gerichteten ToF- Sensors (Time-of-Flight) ab . Mit ihm können Sie eine Tiefenkarte erstellen und so die Rissbeständigkeit des Systems erhöhen.



Bedarf



Die Anwendung, die wir aufgrund ihrer Funktionalität implementiert haben, ist ein Zugriffskontrollsystem, bei dem eine Person eine Möglichkeit darstellt, eine Person zu identifizieren. Mit speziellen Algorithmen wird die Zugehörigkeit einer Person zu einer realen Person überprüft. Sie können der Datenbank direkt vom Gerät aus einen neuen Benutzer hinzufügen, indem Sie ein Bild aufnehmen und einen Namen angeben. Wenn die Anwesenheit einer Person in der Datenbank festgestellt werden muss, wird die Suche anhand eines Fotos durchgeführt, das in Echtzeit vom Gerät aufgenommen wurde. Algorithmen bestimmen die Ähnlichkeit mit Personen aus der Datenbank. Wenn diese gefunden werden, werden Informationen über diese Person ausgegeben.



Unser Hauptziel war es, ein Höchstmaß an Sicherheit zu gewährleisten: Es war notwendig, die Möglichkeit der Umgehung des Gesichtserkennungssystems zu minimieren, beispielsweise durch Verwendung eines Fotos, das in den Sucher gebracht wurde. Zu diesem Zweck haben wir uns für die Intel RealSense 3D-Kamera (Modell D435i) entschieden, die über einen integrierten ToF-Sensor verfügt. Dank dieser Kamera können Sie alle erforderlichen Daten zum Erstellen einer Tiefenkarte abrufen.







Als funktionierendes Gerät mussten wir ein Tablet mit einer großen Bildschirmdiagonale verwenden, das keinen eingebauten Akku hatte und eine ständige Verbindung zum Stromnetz benötigte.



Eine weitere nicht weniger wichtige Einschränkung ist die Offline-Arbeit. Aus diesem Grund konnten wir Cloud-Dienste nicht zur Gesichtserkennung verwenden. Darüber hinaus ist es unangemessen, Gesichtserkennungsalgorithmen unter Berücksichtigung von Zeitbeschränkungen und Arbeitskosten von Grund auf neu zu schreiben. Es stellt sich die Frage: Warum das Rad neu erfinden, wenn es bereits vorgefertigte Lösungen gibt? Aufgrund des Vorstehenden haben wir uns für die 3DiVi Face SDK-Bibliothek entschieden .



Abrufen von Bildern von der Intel RealSense-Kamera



In der ersten Phase der Implementierung mussten zwei Bilder von einer 3D-Kamera abgerufen werden: eine Farbe, die zweite mit einer Tiefenkarte. Anschließend werden sie von der Face SDK-Bibliothek für weitere Berechnungen verwendet.



Um mit der Intel RealSense-Kamera in einem Android-Projekt arbeiten zu können, müssen Sie die Abhängigkeit des RealSense SDK für Android-Betriebssystems hinzufügen : Es ist ein Wrapper über die offizielle C ++ - Bibliothek. In den offiziellen Beispielen finden Sie Informationen zum Initialisieren und Anzeigen eines Bildes von den Kameras. Wir werden hier nicht aufhören, da ist dort alles ganz einfach. Gehen wir direkt zum Bildaufnahmecode: 



private val pipeline = Pipeline()
private val streamingHandler = Handler()

private var streamRunnable: Runnable = object : Runnable {
    override fun run() {
        try {
            FrameReleaser().use { fr ->
                val frames = pipeline.waitForFrames(1000).releaseWith(fr)

                val orgFrameSet = frames.releaseWith(fr)
                val processedFrameSet = frames.applyFilter(align).releaseWith(fr)

                val orgFrame: Frame = orgFrameSet.first(StreamType.COLOR, StreamFormat.RGB8).releaseWith(fr)
                //    
                val videoFrame: VideoFrame = orgFrame.`as`(Extension.VIDEO_FRAME)

                val processedDepth: Frame = processedFrameSet.first(StreamType.DEPTH, StreamFormat.Z16).releaseWith(fr)
                //    
                val depthFrame: DepthFrame = processedDepth.`as`(Extension.DEPTH_FRAME)

                upload(orgFrame) //     
            }
            streamingHandler.post(this)
        } catch (e: Exception) {
            Logger.d("Streaming, error: " + e.message)
        }
    }
}
streamingHandler.post(streamRunnable) //


Mit FrameReleaser () erhalten wir aus dem Videostream einzelne Frames vom Typ Frame. Sie können verschiedene Filter auf applyFilter () anwenden.



Um einen Frame des gewünschten Formats zu erhalten, muss der Frame in den entsprechenden Typ konvertiert werden. In unserem Fall der erste mit einem Typ VideoFrame, der zweite DepthFrame.



Wenn wir ein Bild auf dem Bildschirm des Geräts anzeigen möchten, gibt es hierfür die Methode upload (). In den Parametern wird der auf dem Bildschirm anzuzeigende Rahmentyp angegeben. Wir haben Bilder von einer Farbkamera.



Konvertieren Sie Frames in Bilder



Der nächste Schritt besteht darin, die Bilder in dem von uns benötigten Format vom VideoFrame und DepthFrame abzurufen. Wir werden diese Bilder verwenden, um festzustellen, ob die Person auf dem Bild zu einer realen Person gehört, und um Informationen zur Datenbank hinzuzufügen.



Bildformat:



  • Farbe mit der Erweiterung .bmp von VideoFrame
  • Mit einer Tiefenkarte mit der Erweiterung .tiff, die von DepthFrame bezogen wurde


Um die Konvertierung zu implementieren, benötigen wir eine OpenCV-Open-Source-Computer-Vision-Bibliothek. Alle Arbeiten bestehen darin, das Mat-Objekt zu formen und in das gewünschte Format zu konvertieren: 



fun videoFrameToMat(videoFrame: VideoFrame): Mat {
    val colorMat = Mat(videoFrame.height, videoFrame.width, CvType.CV_8UC3)
    val returnBuff = ByteArray(videoFrame.dataSize)
    videoFrame.getData(returnBuff)
    colorMat.put(0, 0, returnBuff)

    val colorMatNew = Mat()
    Imgproc.cvtColor(colorMat, colorMatNew, Imgproc.COLOR_RGB2BGR)
    return colorMatNew
}


Um das Farbbild zu speichern, muss eine Matrix vom Typ CvType.CV_8UC3 erstellt und anschließend in BRG konvertiert werden, damit die Farben einen normalen Farbton haben.

Speichern Sie mit der Methode Imgcodecs.imwrite auf dem Gerät: 



fun VideoFrame.saveToFile(path: String): Boolean {
    val colorMat = videoFrameToMat(this)
    return Imgcodecs.imwrite(path + COLOR_IMAGE_FORMAT, colorMat)
}


Dasselbe muss für DepthFrame getan werden, mit dem einzigen Unterschied, dass die Matrix vom Typ CvType.CV_16UC1 sein muss, da das Bild aus einem Frame erstellt wird, der Daten vom Tiefensensor enthält: 



fun depthFrameToMat(depthFrame: DepthFrame): Mat {
    val depthMat = Mat(depthFrame.height, depthFrame.width, CvType.CV_16UC1)
    val size = (depthMat.total() * depthMat.elemSize()).toInt()
    val returnBuff = ByteArray(size)
    depthFrame.getData(returnBuff)
    val shorts = ShortArray(size / 2)
    ByteBuffer.wrap(returnBuff).order(ByteOrder.LITTLE_ENDIAN).asShortBuffer().get(shorts)
    depthMat.put(0, 0, shorts)
    return depthMat
}


Bild mit Tiefenkarte speichern: 



fun DepthFrame.saveToFile(path: String): Boolean {
    val depthMat = depthFrameToMat(this)
    return Imgcodecs.imwrite(path + DEPTH_IMAGE_FORMAT, depthMat)
}


Arbeiten mit dem Face SDK



Das Face SDK verfügt über eine große Anzahl von Softwarekomponenten, die meisten benötigen wir jedoch nicht. Die Bibliothek ist wie das RealSense SDK in C ++ geschrieben und verfügt über einen Wrapper, mit dem Sie bequem unter Android arbeiten können. Face SDK ist nicht kostenlos, aber wenn Sie Entwickler sind, erhalten Sie eine Testlizenz.



Die meisten Bibliothekskomponenten werden mithilfe von XML-Konfigurationsdateien konfiguriert. Abhängig von der Konfiguration wird dieser oder jener Algorithmus angewendet.

Um mit der Arbeit zu beginnen, müssen Sie eine Instanz der FacerecService-Klasse erstellen. Sie wird zum Initialisieren anderer Komponenten verwendet. Der Pfad zu den DLLs, Konfigurationsdateien und Lizenzen wird in den Parametern übergeben.



Als Nächstes müssen Sie mit diesem Dienst Objekte der Klassen FacerecService.Config und Capturer erstellen: 



private val service: FacerecService = FacerecService.createService(
                dllPath,
                confDirPath,
                onlineLicenseDir
        )
private val confManual: FacerecService.Config = service.Config("manual_capturer.xml")
private val capturerManual: Capturer = service.createCapturer(confManual)


Die Capturer-Klasse wird zur Gesichtserkennung verwendet. Die Konfiguration manual_capturer.xml bedeutet, dass wir die Algorithmen aus der OpenCV-Bibliothek verwenden werden - dies ist der Viola-Jones-Frontal-Gesichtsdetektor; Haar-Funktionen werden zur Erkennung verwendet. Die Bibliothek bietet einen vorgefertigten Satz von XML-Dateien mit Konfigurationen, die sich in Bezug auf Erkennungsqualität und Laufzeit unterscheiden. Langsamere Methoden haben eine bessere Erkennungsleistung. Wenn wir Gesichter in einem Profil erkennen müssen, sollten wir eine andere XML-Konfigurationsdatei verwenden - common_lprofile_capturer.xml. Es gibt viele Konfigurationen, über die Sie in der Dokumentation mehr lesen können. In unserem Fall musste die Konfiguration common_capturer4_singleface.xml verwendet werden. Dies ist eine Konfiguration mit einem niedrigeren Qualitätsschwellenwert, wodurch immer nicht mehr als eine Person zurückgegeben wird.



Um das Gesicht im Bild zu finden, wird die Methode capturerSingleFace.capture () verwendet, in die das Bytearray des Bildes übertragen wird, das das Gesicht der Person enthält: 



fun createRawSample(imagePath: String): RawSample? {
    val imageColorFile = File(imagePath)
    val originalColorByteArray = ImageUtil.readImage(imageColorFile)
    return capturerSingleFace.capture(originalColorByteArray).getOrNull(0)
}


Das RawSample-Objekt speichert Informationen über das gefundene Gesicht und enthält eine Reihe verschiedener Methoden. Wenn Sie beispielsweise getLandmarks () aufrufen, können Sie anthropometrische Punkte des Gesichts abrufen .



Zugehörigkeit einer Person zu einer realen Person



Um festzustellen, ob sich eine reale Person im Rahmen befindet und kein Foto an der Gesichtserkennungskamera angebracht ist, stellt die Face SDK-Bibliothek das DepthLivenessEstimator-Modul bereit. Sie gibt eine Aufzählung mit einem von vier Werten zurück:



  • NOT_ENOUGH_DATA - Zu viele fehlende Werte auf der Tiefenkarte
  • REAL - die beobachtete Person gehört einer lebenden Person.
  • FAKE - Die beobachtete Person ist ein Foto.
  • NOT_COMPUTED - Berechnung fehlgeschlagen


Modulinitialisierung: 



val depthLivenessEstimator: DepthLivenessEstimator = service.createDepthLivenessEstimator(
           "depth_liveness_estimator_cnn.xml"
   )


Bestimmung der Zugehörigkeit einer Person zu einer realen Person: 



fun getLivenessState(
            rgbPath: String,
            depthPath: String
    ): DepthLivenessEstimator.Liveness {
        
val imageColorFile = File(rgbPath + COLOR_IMAGE_FORMAT)
        val originalColorByteArray = readImage(imageColorFile)

        val originalRawSimple = capturerSingleFace.capture(originalColorByteArray).getOrNull(0)
        val originalRawImage = RawImage(
                SCREEN_RESOLUTION_WIDTH,
                SCREEN_RESOLUTION_HEIGHT,
                RawImage.Format.FORMAT_BGR,
                originalColorByteArray
        )

        val originalDepthPtr = Natives().readDepthMap(depthPath + DEPTH_IMAGE_FORMAT)

//  
        val hFov = 69.4f 
        val vFov = 42.5f 

        val depthMapRaw = DepthMapRaw()
        with(depthMapRaw) {
            depth_map_rows = originalRawImage.height
            depth_map_cols = originalRawImage.width
            depth_map_2_image_offset_x = 0f
            depth_map_2_image_offset_y = 0f
            depth_map_2_image_scale_x = 1f
            depth_map_2_image_scale_y = 1f
            horizontal_fov = hFov
            vertical_fov = vFov
            depth_unit_in_millimeters = 1f
            depth_data_ptr = originalDepthPtr
            depth_data_stride_in_bytes = (2 * originalRawImage.width)
        }
        return depthLivenessEstimator.estimateLiveness(originalRawSimple, depthMapRaw)
}


Die Methode getLivenessState () als Parameter erhält Links zu Bildern: Farbe und mit einer Tiefenkarte. Wir bilden ein RawImage-Objekt aus Farbe. Diese Klasse bietet Rohbilddaten und optionale Zuschneideinformationen. DepthMapRaw wird aus der Tiefenkarte gebildet - einer Tiefenkarte, die gemäß dem ursprünglichen Farbbild registriert wurde. Dies muss erfolgen, um die EstimationLiveness-Methode (originalRawSimple, depthMapRaw) zu verwenden, die uns eine Aufzählung mit Informationen darüber zurückgibt, ob sich die reale Person im Frame befand.



Es lohnt sich, auf die Bildung des DepthMapRaw-Objekts zu achten. Eine der Variablen heißt depth_data_ptr - dies ist ein Zeiger auf Tiefendaten, aber wie Sie in Java wissen, gibt es keine Zeiger. Verwenden Sie die JNI-Funktion, um einen Zeiger zu erhalten, der als Argument einen Link zu einem Bild mit einer Tiefenkarte verwendet: 



extern "C" JNIEXPORT jlong JNICALL Java_ru_face_detect_Natives_readDepthMap
(JNIEnv *env, jobject obj, jstring jfilename)
{
    const char * buf = env->GetStringUTFChars(jfilename, NULL);
    std::string filename = buf;
    env->ReleaseStringUTFChars(jfilename, buf);

    cv::Mat depth_map = cv::imread(filename, -1);
    unsigned char * data = new unsigned char[depth_map.rows * depth_map.cols * depth_map.elemSize()];
    memcpy(data, depth_map.data, depth_map.rows * depth_map.cols * depth_map.elemSize());

    return (jlong) data;
}


Um in C in Kotlin geschriebenen Code aufzurufen, müssen Sie eine Klasse dieses Typs erstellen: 



class Natives {

    init {
        System.loadLibrary("native-lib")
    }

    external fun readDepthMap(fileName: String): Long
}


Der Name der CPP-Datei wird an System.loadLibrary () übergeben, wo die Methode readDepthMap () enthalten ist. In unserem Fall handelt es sich um native-lib.cpp. Es ist auch erforderlich, einen externen Modifikator zu installieren, was bedeutet, dass die Methode nicht in Kotlin implementiert ist.



Gesichtserkennung



Eine ebenso wichtige Funktion ist die Identifizierung der im Rahmen gefundenen Person. Mit Face SDK können Sie dies mithilfe des Erkennungsmoduls implementieren. Initialisierung: 



val recognizer: Recognizer = service.createRecognizer(
        "method8v7_recognizer.xml",
        true,
        true,
        true
)


Wir verwenden die Konfigurationsdatei method8v7_recognizer.xml, die die höchste Erkennungsgeschwindigkeit aufweist, deren Erkennungsqualität jedoch geringer ist als die der Methoden 6v7 und 7v7.



Bevor Sie eine Person identifizieren, müssen Sie eine Liste von Gesichtern erstellen, anhand derer wir die Korrespondenz gemäß dem Beispielfoto finden. Zum Implementieren müssen Sie einen Vektor aus den Vorlagenobjekten erstellen: 



var templates = Vector<Template>()
val rawSample = createRawSample(imageUrl)
val template = recognizer.processing(rawSample)
templates.add(template)


Zum Erstellen einer Vorlage wird die Methode Recognizer.processing () verwendet. RawSample wird als Parameter übergeben. Nachdem die Liste mit Gesichtsvorlagen erstellt wurde, müssen Sie sie zu Recognizer hinzufügen und den resultierenden TemplatesIndex speichern, der für die schnelle Suche in großen Datenbanken erforderlich ist: 



val templatesIndex = recognizer.createIndex(templates, SEARCH_THREAD_COUNT)


Zu diesem Zeitpunkt haben wir ein Erkennungsobjekt erstellt, das alle für die Identifizierung erforderlichen Informationen enthält: 



fun detectFaceSearchResult(rgbPath: String): Recognizer.SearchResult {
    val rawSample = createRawSample(rgbPath + COLOR_IMAGE_FORMAT)
    val template = recognizer.processing(rawSample)

    val searchResult = recognizer.search(
            template,
            templateIndex,
            searchResultCount,
            Recognizer.SearchAccelerationType.SEARCH_ACCELERATION_1
    ).firstElement()
    return searchResult
}


Die Funktion Recognizer.search () gibt das Ergebnis zurück, in dem wir den Index des gefundenen Elements abrufen, ihn mit einer Liste von Personen aus der Datenbank abgleichen und die Person identifizieren können. Darüber hinaus können wir den Ähnlichkeitswert ermitteln, eine reelle Zahl von 0 bis 1. Diese Informationen finden Sie in der Recognizer.MatchResult-Klasse, Bereichsvariable: 



val detectResult = detectFaceSearchResult(rgbPath)
//    -    0  1.
val scoreResult = detectResult.matchResult.score


Fazit



Es besteht kein Zweifel, dass solche Systeme in Zukunft überall eingesetzt werden: Wenn Sie sich dem Eingang nähern, öffnet sich die Tür automatisch und die Kaffeemaschine im Büro wählt Ihren bevorzugten Mahlgrad.



Im Android SDK wird nach und nach eine API hinzugefügt, mit der der Entwickler mit einem Personenidentifikationssystem arbeiten kann. Jetzt befindet sich alles in der Anfangsphase der Entwicklung. Wenn wir über ein Zugangskontrollsystem mit einem Android-Tablet, einer Face SDK-Bibliothek und einer Intel RealSense 3D-Kamera sprechen, möchte ich auf große Flexibilität und Erweiterbarkeit hinweisen. Es besteht keine Bindung zum Gerät, die Kamera kann an jedes moderne Smartphone angeschlossen werden. Sie können das Angebot an unterstützten 3D-Kameras erweitern und mehrere Teile an ein Gerät anschließen. Es ist möglich, die schriftliche Anwendung für Android Things anzupassenund verwenden Sie es in Ihrem Smart Home. Wenn Sie sich die Funktionen der Face SDK-Bibliothek ansehen, können wir mit ihrer Hilfe die Gesichtserkennung in einem kontinuierlichen Videostream hinzufügen und Geschlecht, Alter und Emotionen bestimmen. Diese Möglichkeiten bieten Raum für viele Experimente. Und wir können aus eigener Erfahrung sagen: Hab keine Angst vor Experimenten und fordere dich selbst heraus!
