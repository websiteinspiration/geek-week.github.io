パート1. QInst：1日を失ってから5分で飛行する方がよい（私たちは簡単に計装を記述します）

で前の部分、私は約あなたがELFファイルからeBPF機能をロードする方法を説明しました。次に、ファンタジーからソビエト漫画に移行するときが来ました。賢明なアドバイスに従って、一度一定の努力を払った後、普遍的な計測ツールを作成してください（または、要するにUII !!!）。その際、ゴールデンハンマーのアンチパターンデザインを使用し、比較的使い慣れたQEMUからツールを構築します。これに対するボーナスとして、仮想コンピューター全体のレベルでのインストルメンテーションだけでなく、クロスアーキテクチャーのインストルメンテーションも取得できます。インストゥルメンテーションは、「小さなネイティブsoファイル+ eBPFを含む小さな.oファイル」の形式になります。この場合、eBPF関数は、最適化とコード生成の前に、QEMUの内部表現の対応する命令の前に置き換えられます。


その結果、コード生成中に追加される（つまり、数キロバイトの通常のシステムランタイムをカウントしない）インストルメンテーション自体は次のようになり、これは疑似コードではありません。


#include <stdint.h>

extern uint8_t *__afl_area_ptr;
extern uint64_t prev;

void inst_qemu_brcond_i64(uint64_t tag, uint64_t x, uint64_t y, uint64_t z, uint64_t u)
{
    __afl_area_ptr[((prev >> 1) ^ tag) & 0xFFFF] += 1;
    prev = tag;
}

void inst_qemu_brcond_i32(uint64_t tag, uint64_t x, uint64_t y, uint64_t z, uint64_t u)
{
    __afl_area_ptr[((prev >> 1) ^ tag) & 0xFFFF] += 1;
    prev = tag;
}
