それでも、なぜmakeが必要なのですか？



すべてが始まったように思われますが、最初に気が遠くなるような簡単な質問がありました-「なぜ作成する必要があるのですか？なぜbashスクリプトでうまくいかないのですか？」そして私は思った-本当に、なぜ私はメイクが必要なのですか？（そして最も重要な）彼はどんな問題を解決しますか？


それから私は少し考えることにしました-もし私たちがメイクしていなかった場合、どのように私たちのプロジェクトを集めますか？ソースコードを含むプロジェクトがあるとしましょう。それらから、実行可能ファイル（またはライブラリ）を取得する必要があります。一見、タスクは単純に見えますが、さらに進んでいきました。プロジェクトを最初は1つのファイルで構成します。



コンパイルするには、1つのコマンドを実行します。


$ gcc main.c -o main

とてもシンプルでした。しかし、時間が経つとプロジェクトが発展し、いくつかのモジュールが表示され、ソースファイルが大きくなります。



コンパイルするには、次の数のコマンドを条件付きで実行する必要があります。


$ gcc -c src0.c
$ gcc -c src1.c
$ gcc -c main.c
$ gcc -o main main.o src0.o src1.o

同意する、これはかなり長くて骨の折れるプロセスです。これを手動で行うには、しません。このプロセスは、これらのコマンドを含むbuild.shスクリプトを作成するだけで自動化できると思います。わかりました、それはずっと簡単です：


$ ./build.sh

#!/bin/bash

function modification_time
{
    date -r "$1" '+%s'
}

function check_time
{
    local name=$1
    [ ! -e "$name.o" ] && return $?
    [ "$(modification_time "$name.c")" -gt "$(modification_time "$name.o")" ] && return $?
}

check_time src0 && gcc -c src0.c
check_time src1 && gcc -c src1.c
check_time main && gcc -c main.c
gcc -o main main.o src0.o src1.o
