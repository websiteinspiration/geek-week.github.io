discord.pyライブラリを使用して簡単なDiscordボットを作成する

非同期ライブラリdiscord.pyには、ボットに必要なすべてのものが含まれており、サーバーの音声チャネルを操作することもできます。この記事では、discordサーバー用のシンプルなボットを作成する方法を説明します。



ボットのトークンとクライアントIDを取得する


トークンとボットIDを取得するには、独自のアプリケーションを作成し、[一般情報]セクションにクライアントIDをコピーする必要があります。



そして、設定セクションでボットを作成し、そのトークンをコピーします。難しいことではありませんが、誰でも扱えると思います。



実際にボットを書く


discord.pyをpipでインストールします。

pip install discord

インストールが成功したら、bot.pyファイルを作成します。ここにボットを記述します。

必要なものはすべてインポートします。

import discord
from discord.ext import commands

上記で書いたように、トークンを使用して変数を作成します。

TOKEN = ' '

トークンについて
, - , .

ボットの本体を作成します。

bot = commands.Bot(command_prefix='!') #    '!'

まず、ボットが転送するだけの簡単なコマンドを作成しましょう。


@bot.command(pass_context=True) #  
async def test(ctx, arg): #   
    await ctx.send(arg) #

そして最後に、トークンを使用してボットを起動します。

bot.run(TOKEN)

最後に、これを取得する必要があります。

コード

import discord
from discord.ext import commands

TOKEN = ' '
bot = commands.Bot(command_prefix='!')


@bot.command(pass_context=True)  #   
async def test(ctx, arg):  #    
    await ctx.send(arg)  #   


bot.run(TOKEN)


次に、サーバーにボットを追加する必要があります。リンクを使用してこれを行うことができます：

https://discordapp.com/oauth2/authorize?&client_id={Client ID}&scope=bot&permissions={,  66395456}

必要な権限の数はボット設定セクションで取得できます。

これでボットを実行できます。

python bot.py

数秒後、ネットワーク上でそれを確認できます。



最後に、コマンドを送信してみます。



結論


これにより、サーバー上でボットを簡単に起動できます。ご覧のとおり、ライブラリはほとんどすべてのことを行い、Pythonを使用して独自の機能を追加することしかできません。次回は、イベントをフォローする方法、音声チャネルに接続する方法（LinuxとWindowsの問題を回避する方法）、参加者の役割と権限を使用する方法などを紹介します。
