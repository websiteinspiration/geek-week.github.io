Windowsサーバーをより安全にする

この記事では、著者がWindowsサーバーの管理に関するいくつかの良いヒントを提供したいと思います。これは、何らかの理由でインターネットでは見つけることができませんでした。

これらの原則に従って、ローカル環境とパブリック環境の両方でWindowsベースのサーバーを大幅に保護します。


1. ADにコンピューターを正しく入力します


管理者特権を持つユーザーを使用して、Active Directoryにコンピューターを入力しないでください。キーロガーが従業員のコンピューターにまだ現れていないことを誰も保証しません。別のアカウントを持っています。資格情報を盗んだ攻撃者は、ADにコンピューターを追加することしかできません。

1.1。新しいユーザーを作成する


これを行うには、ADUC（Active Directoryユーザーとコンピューター）に移動します。「ユーザーとコンピュータ」フォルダの空のスペースをクリックして、ユーザーを作成します。簡単にするために、JoinAdminと呼びましょう。

1.2。権限を委任する


ユーザーが作成された後、彼は許可を割り当てられる必要があります。これを行うには、「ユーザー」フォルダを右クリックし、「制御の委任」を選択します。「次へ」をクリックして、管理者を選択します。


ユーザーに1つの機能のみを実行させたいので、2番目のオプションを選択します。


次に、「コンピュータオブジェクト」とそのすべてのサブアイテムを選択します。



このユーザーを使用して、誰かが管理者からパスワードを盗むことを心配しないでください。

2.フォルダーアクセスの制御


Windows Server 2019では、非常に過小評価されている機能であるフォルダーアクセスの制御が導入されました。選択したフォルダーまたはセクションのホワイトリストにないプログラムの録画へのアクセスを禁止します。 

オンにすると、サーバーが感染している場合でも、ランサムウェアウイルスからデータを保護するのに役立ちます。ただし、これはデータを盗難から保護するものではなく、読み取ることができます。

この機能は、WindowsセキュリティセンターまたはPowershellで有効にできます。

Set-MpPreference -EnableNetworkProtection Enabled

許可されたプログラムの追加：

許可されたプログラムは、Windowsセキュリティセンターを通じて追加できます。Powershellを通じて、実行ファイルへのフルパスをカンマで区切って入力する必要があります。

Set-MpPreference -ControlledFolderAccessAllowedApplications "C:\Folder\File.exe"

            .

$ApplicationToAdd = "C:\Fodler\file.exe"
 
$ExistingApplicationList = (Get-MpPreference).ControlledFolderAccessAllowedApplications
 
$FullList = $ApplicationToAdd + $ExistingApplicationList 
 
Set-MpPreference -ControlledFolderAccessAllowedApplications $FullList

保護されたフォルダーの追加：

フォルダーはプログラムとまったく同じ方法で追加されます。たとえば、次のコマンドでは、C：ドライブ全体を追加します。

Set-MpPreference -ControlledFolderAccessProtectedFolders "C:\"

3. SYSWOW64およびSMB


すべての32ビットプログラムは、互換性レイヤー-SYSWOW64を介して64ビットで実行されます。このコンポーネントを無効にすると、ウイルスを含むすべての32ビットソフトウェアに互換性がなくなります。

32ビットアプリケーション自体はより脆弱であり、64ビットプログラムはバッファオーバーフローによる攻撃を受けず、これらのプログラムで提供されていないコードを実行することははるかに困難です。埋め込みコンポーネントのみまたは64ビットプログラムのみがサーバーで動作する場合は、必ずこのコンポーネントを削除してください。

SMB1を歩いたWannacryを覚えていますか？最初のバージョンのSMBはWindows Serverの標準コンポーネントであり、各エディションにデフォルトでインストールされます。
WannacryはEternalBlueの脆弱性を突破し、脆弱性にパッチを適用しましたが、堆積物は残りました。
このコマンドは、SMB1とSYSWOW64の両方を削除します。

Remove-WindowsFeature WoW64-Support, FS-SMB1


この推奨事項は、これらの特定のコンポーネントを削除するための言い訳としてではなく、一般的にすべての未使用のコンポーネントを無効にすることとして価値があります。

4. pingを無効にする


既定では、WindowsベースのコンピューターはローカルネットワークでのみICMPに応答します。ICMPを無効にすると、サーバーのセキュリティが少し向上します。部分的には、これは、kulhackerおよび他のスクリプトキディフォーラムに、ブルートフォースになる可能性のある目標を探す方法に関するガイドがあるという事実によるものです。

ICMPを無効にすると勝つことを示すために、2つのサーバーが作成されました。両方でRDPが開かれましたが、サーバーの1つがICMPに応答しませんでした。

ICMPは無効になっています。


ICMPが有効：


スクリーンショットからわかるように、これは全員を止めたわけではなく、多くを止めました。彼らがあなたをどの程度ハッキングしたいのかについてのデータを収集するには、次のスクリプトを使用できます。
 
function Get-Bruteforce {
 
    $Last = 4
    $Attempts = 10
 
    #Getting date -one hour (default)
    $DateTime = [DateTime]::Now.AddHours(-$Last)
 
    $BruteEvents = Get-EventLog -LogName 'Security' -InstanceId 4625 -After $DateTime -ErrorAction SilentlyContinue | Select-Object @{n='IpAddress';e={$_.ReplacementStrings[-2]} }
    $TopPunks = $BruteEvents | Group-Object -property IpAddress | Sort-Object Count
 
    #Get bruteforsers that tried to login greated or equal than 4 times (default)
    $GetPunks = $TopPunks | where {$_.Count -ge $attempts} | Select -property Name
 
    Write-host Unique attackers IP: $GetPunks.Length -ForegroundColor Green
    Write-Host Total bruteforce attempts: $BruteEvents.Length -ForegroundColor Green
 
    #Output-punks
    foreach ($i in $TopPunks | where {$_.Count -ge $attempts}) {
 
    $PunkRdns = (Resolve-DnsName $i.Name -ErrorVariable ProcessError -ErrorAction SilentlyContinue).NameHost 
 
    if ($ShowRDNS) {
        if ($PunkRdns) {
            Write-Host "attempts": $i.count IP: $PunkRdns
        }
        else {
            Write-Host "attempts": $i.count IP: $i.name
        }
    }
    else {
 
        Write-Host "attempts": $i.count IP: $i.name
 
    }
 
    }
   
}
