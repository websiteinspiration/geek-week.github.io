In diesem Artikel werde ich Ihnen GCP SQL vorstellen und zeigen, wie Sie in diesem Dienst einen PostgreSQL-Server erstellen.

So erstellen Sie PostgreSQL in GCP SQL


Beginnen wir mit der Erstellung eines PostgreSQL-Servers in GCP SQL.

Bevor Sie beginnen, melden Sie sich bei Ihrem GCP-Konto an. So erstellen Sie einen PostgreSQL-Datenbankserver:> Wählen Sie im Google Services-Bereich SQL aus.



So erstellen Sie eine SQL-Instanz:> Klicken Sie auf INSTANZ ERSTELLEN.



Wählen Sie die gewünschte Datenbank aus → Klicken Sie auf PostgreSQL.



Geben Sie die Konfigurationsdetails ein:

Instanz-ID: - Benennen Sie Ihre Instanz,

legen Sie ein Kennwort fest - Kennwort,

wählen Sie die Region aus, die Ihnen am nächsten liegt - Region,

wählen Sie die Version des Datenbankservers aus - Datenbankversion.

Klicken Sie auf Konfigurationsoptionen anzeigen, um zusätzliche Informationen zu konfigurieren.



Erweitern Sie Konfigurationsdetails



Klicken Sie auf Konnektivität → Standardmäßig kann von externen Netzwerken nicht auf die Instanz zugegriffen werden. Um Zugriff auf eine Instanz zu gewähren, lassen Sie eine bestimmte IP-Adresse oder Freigabe zu. Klicken Sie auf + Netzwerk hinzufügen.



Durch Eingabe von 0.0.0.0/0 in das Netzwerk lassen Sie alle IPv4-IPs zu und können von überall auf die Instanz zugreifen.



Als nächstes warten wir darauf, dass die Einstellungen gespeichert werden.



Um mit zusätzlichen Ressourcen und der API zu kommunizieren → aktivieren Sie Private IP und wählen Sie das zugeordnete Netzwerk (das von Ihnen erstellte VPC-Netzwerk oder Standard).



Wir warten wieder auf die Rettung.



Sie können private Dienstverbindungen auch überprüfen, indem Sie auf VPC → VPC-Peering klicken.



Wählen Sie den Maschinentyp und den Speicher aus: Anzahl der Kerne und Speicher.



Wenn Sie die Instanz zu Testzwecken verwenden möchten, wählen Sie eine einzelne Zone aus.



Wenn Sie eine Produktionsumgebung haben, wählen Sie Hochverfügbarkeit, dies erhöht jedoch die Kosten.



Klicken Sie auf Element hinzufügen, um die Datenbankflags hinzuzufügen.



Wählen Sie die Flagge aus, die Sie benötigen.


.
GCP aktualisiert die Datenbank alle paar Monate. Wir werden daher gebeten, im Voraus ein Datum und eine Uhrzeit festzulegen, um Ihre Datenbank neu zu starten. Dies dauert nicht lange. Wählen Sie also ein Datum und eine Uhrzeit, wenn Sie über eine Produktionsdatenbank verfügen.



Geben Sie den Schlüssel und den Wert zur Verdeutlichung auf der Registerkarte Beschriftungen an.



Klicken Sie auf Erstellen, um die Datenbank zu erstellen.



GCP erstellt PostgreSQL Server:



Erfolg! Der PostgreSQL-Server wurde erfolgreich erstellt.



Sie sehen Informationen und eine Übersicht über das neu erstellte PostgreSQL.



Wenn Sie auf Verbindungen klicken, können Sie die Details der Verbindung anzeigen und feststellen, dass die Datenbank mit der GCP App Engine kommunizieren durfte.



Um auf den erstellten PostgreSQL-Server zugreifen zu können, müssen Sie auf der Registerkarte USERS einen Benutzer erstellen.



Sie können auch eine Datenbank auf der Registerkarte DATENBANKEN erstellen.



Sie können Backups in BACKUPS verwalten.
