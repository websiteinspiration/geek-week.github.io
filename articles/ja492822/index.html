CSS


2001年から2010年の10年間、インターネットが世界を征服したとき、CSSは止まりました。Internet Explorer 6-7-8が主要なブラウザーでした。当時のマイクロソフトはCSS仕様にあまり注意を払わず、新しい機能を追加することに関心を示していませんでした。CSSはここ数年バージョン2.1のままです。

2007年以降、最新のブラウザであるFirefoxとChromeが登場し始めました。現時点でも、Internet Explorerをサポートする必要がありました。これは私たちの開発を妨げました。当時の主要な管理ブラウザとして、IEはほとんど存在しない、ゼロに近いCSS開発速度をインストールしました。それはインターネットの「暗黒時代」でした。



CSSの新時代の基本的なイベント


私は当時開発者だったので、CSSとWebの世界全体を変える2つの重要なイベントを見ました。

  1. スマートフォン時代の幕開け（2007年6月29日） -すべては最初のiPhoneのリリースから始まりました。このリリースでは、Webデバイスの新しいカテゴリに適合する新しいCSS機能のサポートが必要でした。
  2. Chromeブラウザリリース（2008年9月2日） -Chromeは、デスクトップアプリケーションとして開始された新しいブラウザです。それまでは、Internet Explorerがインターネットの主要な支配者であり、その開発を指示していました。



フロントエンド開発者がCSSとJavaScriptの両方を扱う理由


当時、フロントエンドの開発者がクライアントのすべての機能を管理するべきだという考えはかなり一般的でした。その後、CSSルールはわずかしかありませんでした。CSSは現在拡張されており、そのようなビジョンはもはや関係ありません。

CSS 3、HTML 5、JavaScript：新しい時代


CSS 3が登場した時期は、1999年以来開発されているため、正確にはわかりません。インターネット上での実際のサポートは2011年に始まりました。当時、ChromeとFirefoxは市場シェアを拡大​​していました。スマートフォンは非常に人気があります。独自のブラウザーを使用して、すべてのユーザーが最初の新しいCSS 3機能をサポートし始め、

これらの変更によりInternet Explorerの人気が低下しました。当時はまだ重要なブラウザでしたが、Microsoftは仕様とCSS開発に注意を向ける必要があることに気づきました。彼らはInternet Explorer 9をリリースしました。

新しいCSS


CSS 3は10歳になりましたが、なぜCSS 4でないのですか？CSS 3はCSS 2.1とは異なり、オープンスタンダードです。これのおかげで、それは常に更新されます。CSS 3は継続的な開発であり、常に新しいバージョンであるため、新しいバージョンに「CSS 4」という名前を付ける必要はありません。


2011年以降、すべてのブラウザー更新には新しいCSS機能が含まれています。過去数年間、ほとんどのブラウザー（Chrome、Firefox、Opera、Edge）はほぼ毎月更新されています（Safariは主に年に1回更新されます）。この頻度は、CSSの新機能が登場するとすぐに入手できることを意味します。

実際には、Web開発者がこれらすべての新機能について基本的なレベルでさえ理解することはほとんど不可能です。さらに、JavaScriptは並行して開発されました。そして彼は、おそらくCSS以上のものを開発しました。これにより、一人の人がすべての開発についていくことがさらに難しくなります。

Angular 2：最初のJavaScript準拠フレームワーク


新しいAngularは、カプセル化されたスタイルを初めて導入したものです。これは、経験の浅いWeb開発者がCSSカスケードを理解する必要をなくす方法でした。この方法を使用すると、グローバルスタイルを作成し、コンポーネントごとにローカルスタイルを維持できます。

すべてのさまざまな開発アイデアのうち、カプセル化されたスタイルのアイデアは、私の意見では、コンポーネントのグローバルスタイルとカプセル化されたスタイルの両方を作成できるため、良いと思います。さらに、CSS / SASS、HTML、およびJavaScriptについては、まだ懸念の分離があります。必要なスタイルだけを簡単にダウンロードできます。



反応-セマンティクスの死！


Reactはこのアイデアをさらに取り入れ、すべてのHTMLおよびCSSをJavaSscript内に書き込みます。再利用可能なスタイルを作成するという目標は、Reactでなくなりました。CSSクラス名を自動的に生成します。したがって、CSSクラス名と非セマンティックHTMLの両方が原因で、コードが読みにくくなり、理解できなくなります。私の観点から見ると、これは、CSSやHTMLが動作しているときの外観が問題ではなかったときに、インターネットの初めにCSSやHTMLがどのように書かれたかに似ています。


React-判読できないcssクラス名

なぜそんなに悪いのですか？実際のCSS問題があり、通常のフロントエンド開発者がそれを解決できず、経験豊富なCSS開発者が必要な場合、混乱するReactコードが邪魔になります。はい、両方のWeb開発者が一緒に問題を解決できますが、経験豊富な2人のWeb開発者には時間がかかります。

新しい役職は何ですか？


過去8年間、私は何度かポジションを変えてきました。私のLinkedInプロフィールを見る人に私の能力を説明するためにこれを行いました。

最初、私は自分自身をCSSマスターと呼びました（私が設立したコミュニティの名前で-CSS Masters Israel）。このような見出しの仕事が発表された直後に見るのは面白かった。CSS開発者を必要とするが、適切に名前を付ける方法がわからない会社があることに気付きました。



その後、自分の能力を作品名に合わせるために、投稿の名前を何度か変更しました（UI / CSS-開発者/インターフェース開発者/エンジニア/建築家/エキスパート）。

単なるCSS開発者ではない


最近では、CSSやSVGなどのビジュアルパーツを専門とする複数のタイプの開発者がいます。これらのタイプをここで定義しようとします：

CSS開発者 -この名前はCSS中級レベルの専門家向けです。これらは、デザインを取得してCSSに簡単に変換できる開発者です。

CSSアーキテクト-このカテゴリは、CSSを効率的に構築することに加えて、CSSの分離を計画し、複雑な建物構造に構築する専門家である私などの開発者を対象としています。 CSS開発者とは異なり、CSSアーキテクトは経験豊富で、大規模なプロジェクトをスマートにスケーリングする方法を知っており、チームで作業しているCSS開発者もいます。さらに、他のWeb開発者、プロジェクトマネージャー、およびWebデザイナーと通信します。


バラエティが素晴らしい！

CSSアニメーション開発者 -今日では、CSSアニメーションまたはSVGアニメーションを使用して、ユーザーエクスペリエンスをほとんど苦労せずに改善できるため、優れたプロジェクトを素晴らしいプロジェクトに変えることができます。この作業は、大企業に適している場合があります。
