asyncioループは、タスクを含むように起動する必要はありません。サイクルが停止しているときでも、タスクを作成および停止できます。停止すると、一部のタスクが未完了のままになる場合があります。

import asyncio


async def printer():
    try:
        try:
            while True:
                print('*')
                await asyncio.sleep(1)
        except asyncio.CancelledError:
            print('')
    finally:
        await asyncio.sleep(2)
        print('')  # never happens


loop = asyncio.get_event_loop()
run = loop.run_until_complete
task = loop.create_task(printer())

run(asyncio.sleep(1))  # printer works here
print('||')

run(asyncio.sleep(1))  # printer works here
task.cancel()  # nothing happens
run(asyncio.sleep(1))  #  printed

結果：

*
*
||
*

すべてのタスクが完了するのを待ってから、サイクルを停止してください。これが行われない場合、一部のブロックfinallyをスキップでき、一部のコンテキストマネージャは無効になりません。


Pythonでは、ビット単位のシフト演算子を含む多くの演算子をオーバーライドできます。この演算子を使用して関数の構成を作成する例を次に示します。矢印はデータ転送の方向を示します。

from collections import deque
from math import sqrt


class Compose:
    def __init__(self):
        self._functions = deque()

    def __call__(self, *args, **kwargs):
        result = None
        for f in self._functions:
            result = f(*args, **kwargs)
            args = [result]
            kwargs = dict()
        return result

    def __rshift__(self, f):
        self._functions.append(f)
        return self

    def __lshift__(self, f):
        self._functions.appendleft(f)
        return self


compose = Compose


sqrt_abs = (compose() << sqrt << abs)
sqrt_abs2 = (compose() >> abs >> sqrt)

print(sqrt_abs(-4))  # 2.0
print(sqrt_abs2(-4))  # 2.0


クラスを定義するときに、そのメタクラスに引数を渡すことができます。表記法classはキーワードを引数としてサポートします：class Klass(Parent, arg='arg')。キーワードはmetaclassメタクラスを選択するために予約されていますが、他のキーワードは自由に使用できます。

以下は、属性の1つを持たないクラスを作成するメタクラスの例です。属性名は引数で提供されますremove：

class FilterMeta(type):
    def __new__(mcs, name, bases, namespace, remove=None, **kwargs):
        if remove is not None and remove in namespace:
            del namespace[remove]

        return super().__new__(mcs, name, bases, namespace)


class A(metaclass=FilterMeta, remove='half'):
    def half(x):
        return x // 2

    half_of_4 = half(4)
    half_of_100 = half(100)


a = A()
print(a.half_of_4)  # 2
print(a.half_of_100)  # 50
a.half  # AttributeError


ジェネレーターを使い果たす必要がある場合もありますが、同時に、ジェネレーターが生成する値には関心がありませんが、いくつかの副作用があります。たとえば、例外、ファイルへの書き込み、グローバル変数の変更など。

これを行うには、便利で一般的な方法がありますlist(gen())。ただし、この方法ではすべての値がメモリに保存され、すぐに削除されます。これは冗長な場合があります。この動作を回避したい場合dequeは、サイズ制限付きで使用できます。

from collections import deque

def inversed(nums):
    for num in nums:
        yield 1 / num

try:
    deque(inversed([1, 2, 0]), maxlen=0)
except ZeroDivisionError:
    print('E')

親子、 -あなたはクラスのカップルを持っていると仮定UserしてAdmin。また、ユーザーのリストを引数として取る関数もあります。管理者のリストを提供できますか？いいえ：関数は別のユーザーを管理者のリストに追加できますが、これは誤りであり、リストが提供する保証に違反しています。

ただし、タイプSequenceは読み取り専用であるため指定できます。より正確には、この場合、それSequenceは参加者のタイプで共変です。

のcovariant=True引数としてを指定することにより、共変タイプを定義できますTypeVar。

from typing import TypeVar, Generic

T = TypeVar('T', covariant=True)


class Holder(Generic[T]):
    def __init__(self, var: T):
        self._var: T = var

    def get(self) -> T:
        return self._var


class User:
    pass


class Admin(User):
    pass


def print_user_from_holder(holder: Holder[User]) -> None:
    print(holder.get())


h: Holder[Admin] = Holder(Admin())
print_user_from_holder(h)

逆に、機能には管理者を配置するためにのみコンテナーが必要な場合があります。記録のみに使用できるこのようなコンテナは、参加者のタイプによって反変です。

from typing import TypeVar, Generic

T = TypeVar('T', contravariant=True)


class Holder(Generic[T]):
    def __init__(self, var: T):
        self._var: T = var

    def change(self, x: T):
        self._var = x


class User:
    pass


class Admin(User):
    pass


def place_admin_to_holder(holder: Holder[Admin]) -> None:
   holder.change(Admin())


h: Holder[User] = Holder(User())
place_admin_to_holder(h)
