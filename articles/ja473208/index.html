オープンソースプロジェクトの作業プロセスを簡略化した方法


この記事では、いくつかの主要なオープンソースプロジェクトの開発にどのように参加しようとしたかを説明します。自分を嫌い、ルーチンを自動化し、人生を楽しむことを学びました。詳細-カットの下。


なぜこれが必要なのですか？


通常、開発者はいくつかの理由でオープンソースコミュニティへの参加を望んでいます。それらのいくつか（おそらくすべてではない）を次に示します。


  • これと他のプログラムを無料で使用する機会に感謝します
  • 新しい経験を積む
  • 履歴書を送る

私は常に、新しいことを学び、ソフトウェア開発のベストプラクティスを吸収する機会に常に惹かれていましたが、それ以外のすべては、このプロセスに劣らず快適な付属物です。


はい、どこにいますか？


最初に行うことは、作業するタスクを見つけることです。仕事のニーズに合わせてライブラリを変更する必要がある場合、あなたは本当に幸運です。タスクを開始し、所有者と話し合って、実装を開始してください！それ以外の場合は、プロジェクトページの開いているタスクのリストを参照して、自分用に何かを選ぶことができます。適切なものを見つけることはその実装と同じくらい重要であり、ここではそれほど単純ではありません。あなたが経験豊富なエンジニアであっても、より単純なタスクから始めて、コードベースに精通し、受け入れられた開発プロセスを理解してから、より大きな機能を引き継ぐことは有用かもしれません。

発売


開始するには、次のものが必要です。


  • コンピューターにドッカーがインストールされている
  • APIトークンを書き込みます。対応するGithub設定ページで行うことができます。
  • パラメータを介してアクセストークンを渡すことでコンテナを開始します

    docker pull igorperikov/mighty-watcher:latest
docker run -e TOKEN={} --rm igorperikov/mighty-watcher:latest

その他の設定（言語によるフィルタリング、並列処理のレベル、リポジトリのブラックリスト）は、プロジェクトページにあります。プロジェクトへのリンク。


プロジェクトにいくつかのタグがない場合は、PRを作成するか、個人的に私に書いてください。追加します。
