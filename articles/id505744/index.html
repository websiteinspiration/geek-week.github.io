Pohon pencarian biner




Penampilan pohon sangat tergantung pada bagaimana elemen-elemen array pada awalnya berada. Berdasarkan data acak, pohon dengan tingkat probabilitas tinggi akan berubah menjadi tidak seimbang, dengan jumlah tingkat yang jauh lebih besar dibandingkan dengan yang optimal. Dalam contoh kami, 7 level terbentuk. Jika elemen array diatur, misalnya, dalam urutan ini:

25, 28, 29, 23, 30, 27, 24, 22, 21, 26

, pohon pencarian akan seimbang, dengan level sesedikit mungkin (= 4). Menyortir menggunakan pohon seimbang akan memiliki kompleksitas waktu: O ( n log n ) . Tetapi dalam kasus pohon yang tidak seimbang, ia dapat terdegradasi ke O ( n 2 ) .

Dalam habrastatia sebelumnya, pengurutan berdasarkan pohon pencarian biner dipertimbangkan , serta modifikasi yang menarik , yang memungkinkan pengurutan yang dijamin dengan kompleksitas waktu O ( n log n ) .

Semakin banyak pohon pencarian yang tidak seimbang, semakin mahal untuk melewatinya.

Di sisi lain, menyimpan data di pohon pencarian sangat mudah dari sudut pandang organisasi, memungkinkan Anda untuk melakukan banyak operasi dengan data ini, termasuk penyortiran.

Orang tua di pohon pencarian biner tidak harus lebih dari seorang anak. Tetapi pada saat yang sama, anak kiri selalu lebih kecil dari orang tua, anak kanan selalu lebih besar dari orang tua (atau sama dengan itu).

Elemen maksimum array ketika membangun pohon pencarian jatuh di suatu tempat di subtree kanan. Untuk mengekstrak maksimum ini, pohon pencarian harus sepenuhnya dielakkan.

Pohon kartesius


Dan akhirnya, pohon Cartesian. Hubungan diatur sedemikian rupa sehingga, di satu sisi, itu banyak jika Anda melihat nilai - nilai elemen. Semua orang tua lebih dari keturunan mereka:



Di sisi lain, struktur adalah pohon pencarian biner ketika Anda melihat indeks elemen. Untuk kejelasan, nilai-nilai elemen dalam node telah dihapus dari gambar di pohon, hanya indeks yang tersisa. Setiap keturunan kiri lebih kecil dari induknya, setiap keturunan kanan lebih besar dari induknya:



Di satu sisi, kami memiliki akses cepat ke elemen maksimum. Karena ini adalah banyak, maksimum ada di root.

Di sisi lain, Anda dapat dengan mudah menghapus maksimum dari struktur dan bekerja dengan data yang tersisa lebih lanjut. Karena ini adalah pohon pencarian biner, menghapus root tidak mahal. Perubahan relatif kecil terjadi dalam hubungan orangtua-anak, maksimum baru menyentuh akar pohon Cartesius, dan Anda dapat terus bekerja.

Setiap elemen ditandai oleh dua angka - nilai elemen dan indeks elemen dalam array. Indeks dapat dianggap sebagai koordinat X , nilai sebagai Y , dan elemen itu sendiri harus ditafsirkan sebagai titik pada bidang koordinat. Algoritma terakhir:

  • I. Berdasarkan pada array yang tidak disortir, kami membuat pohon Cartesius:
    • I.1. Iterasi elemen-elemen array dari kiri ke kanan.
    • I.1. Jika ini adalah elemen pertama dari array, maka itu hanyalah simpul pertama dari pohon Cartesian.
    • I.2. Jika ini bukan elemen pertama dari array, pertimbangkan ini sebagai pohon Cartesi yang terdiri dari satu elemen. Kami melakukan operasi Gabung pohon ini dengan pohon yang dibangun dari elemen sebelumnya.
  • II Kami menghapus elemen maksimum saat ini dari pohon, setelah itu kami mengembalikan pohon Cartesius.
    • II.1. Elemen maksimum saat ini adalah akar dari pohon.
    • II.2. Kami menghapus root dari pohon, kami mentransfer elemen maksimum ke array tambahan.
    • II.3. Setelah menghapus root, struktur terbagi menjadi dua pohon Cartesian.
    • II.4. Untuk dua pohon ini, kami melakukan operasi Gabung . Maksimum baru saat ini adalah salah satu akar dari pohon yang bergabung.
    • II.5. Untuk ketinggian baru, ulangi langkah II.1-II.4 lagi.




Implementasi dalam C ++:

/*
 * c++ 11 code to construct a Cartesian tree. The method cartesianTreeSort
 * will sort the contents of the array.
 */

#include <iostream>
#include <vector>
#include <queue>

struct Node{
    int value;
    Node *left, *right;
    Node *parent;

    Node(){
        value = 0;
        parent = NULL;
        left = NULL;
        right = NULL;
    }
};

// Used by priority queue
struct compare{
    bool operator()(Node *left, Node *right){
        return left->value > right->value;
    }
};

class CartesianTree{
private:
    // last pointer to keep track of last node added
    Node *root, *last;

    Node * findLowestNode(Node *node, int x){
        if(node->value < x)
            return node;
        else if(node->parent != NULL)
            return findLowestNode(node->parent, x);
        else
            return NULL;
    }

public:
    Node * getRoot(){
        return root;
    }

    void addNode(int x){
        Node *new_node = new Node;
        new_node->value = x;
        if(root == NULL){
            last = new_node;
            root = new_node;
            return;
        }
        Node *z = findLowestNode(last, x);
        if(z == NULL){
            new_node->left = root;
            root->parent = new_node;
            root = new_node;
        }
        else{
            new_node->left = z->right;
            z->right = new_node;
            new_node->parent = z;
        }
        last = new_node;
    }

    CartesianTree(std::vector<int> ar){
        root = NULL;
        last = NULL;
        // Call addNode function for each element of the array
        for(auto x : ar){
            addNode(x);
        }
    }

    void InorderTraversal(Node *node){
        // To print inorder traversal of the tree
        if(node == NULL)
            return;
        InorderTraversal(node->left);
        std::cout << node->value << ' ';
        InorderTraversal(node->right);
    }

    // Function to sort and store values in array
    void cartesianTreeSort(std::vector<int> &sorted_ar){
        // Initialize input array
        sorted_ar.assign(0, 0);

        // Initialize priority queue
        std::priority_queue<Node *, std::vector<Node *>, compare> p_queue;
        p_queue.push(root);
        Node *temp = NULL;
        while(!p_queue.empty()){
            temp = p_queue.top();
            p_queue.pop();
            sorted_ar.push_back(temp->value);
            if(temp->left){
                p_queue.push(temp->left);
            }
            if(temp->right){
                p_queue.push(temp->right);
            }
        }
    }
};

int main(){
    std::vector<int> ar = {1, 14, 5, 0, 8};

    CartesianTree tree(ar);
    std::cout << "Inorder Traversal\n";
    tree.InorderTraversal(tree.getRoot());
    std::cout << '\n';

    std::vector<int> sorted;
    tree.cartesianTreeSort(sorted);
    std::cout << "Sorted array is\n";
    for(auto x : sorted)
        std::cout << x << ' ';
    std::cout << '\n';
}


Kompleksitas


Sebuah pohon Cartesian dengan biaya minimal memproses bagian yang diurutkan dan hampir diurutkan (dan tidak masalah, dalam naik atau turun) dari array. Di sini, misalnya, adalah proses untuk array yang dipesan terbalik:



Bagian yang paling mahal dari algoritma ini adalah operasi Gabung, yang harus dilakukan dua kali untuk setiap n elemen - pertama ketika membangun deramide dan kemudian ketika dibongkar. Pada data acak, operasi Gabung satu kali biaya O (log n ) , tetapi untuk elemen yang terletak di daerah terurut array, biaya operasi O (1).

Dengan demikian, rata-rata dan kompleksitas waktu penyortiran terburuk adalah O ( n log n ) , dan yang terbaik adalah O ( n ).

Menurut memori tambahan, semuanya lebih buruk - O ( n) Jika pohon Cartesian hanya semacam tumpukan, maka biayanya adalah O (1). Tapi pohon Cartesian juga pohon pencarian biner, yang mengapa untuk semua n elemen perlu untuk secara terpisah menyimpan hubungan orangtua-anak.

Selain itu, implementasi dalam formulir ini berarti penyortiran eksternal - elemen yang diurutkan dikumpulkan dalam array yang terpisah. Dan ini O ( n ) untuk memori tambahan.
