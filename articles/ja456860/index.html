Extreme Networks Insightスイッチ または、なぜスイッチ上で仮想マシン

Extreme NetworksのSLX、VSP、およびEXOSスイッチの範囲には、総称して「Insight」と呼ばれる特別なハードウェアアーキテクチャを備えたモデルが含まれています。標準バージョンとは異なり、スイッチの制御とデータプレーンがPCIeバスによってのみ接続されている場合（結果として生じるすべての帯域幅制限がある）、「Insight」スイッチには、コントロールプレーンにデプロイされた仮想マシンに直接接続されているいくつかのデータプレーンインターフェイスがあります。それがどのように機能し、どのリソースが利用されるかを検討してください：



「インサイト」インターフェースは通常、ASICからマルチプレクサーを介してコントロールプレーンに接続されているいくつかの専用10GEリンクです。そこで、それらは分離されたUbuntu Linux 16.04仮想マシンのイーサネットインターフェースとしてマウントされ、簡単にTPVM（サードパーティ仮想マシン）と呼ばれます。この物理的な分離により、制御またはデータプレーンのパフォーマンスを低下させずに、最小限の遅延で大量のトラフィックを転送できます。仮想化自体はKVMに基づいて構築されています。これにより、追加のネットワークインフラストラクチャを展開することなく、tcpdump、p0f、snort、分散DPIまたは分析などのツールやサービス、およびその他の多くのサービスをデバイス上で直接実行できます。 TVPM自体の展開を高速化するために、イメージには多くの必要なパッケージがあります。事前定義されています：

  • ビルド必須
  • checkinstall
  • iperf
  • mtools
  • netperf
  • qemu-guest-agent
  • シャーク
  • Valgrind
  • vim-gnome
  • ワイヤーシャーク
  • xterm



「Insight」スイッチには、Intel XEON x86 CPU、増設されたRAMおよびSSDドライブが搭載されており、複数のVMを導入する場合にリソースを割り当てることができます。スイッチ自体のOSを中断しないように、リソース自体は専用で分離されています。


TPVMを本来の目的で使用するには、次の4つの手順を実行するだけで済みます。

  1. TVPMをダウンロードしてインストールする
  2. Insightインターフェイスを構成する
  3. これらのインターフェースにACLまたはミラートラフィックを追加する
  4. TPVMアプリケーションを起動する

以下は、実際にこれがどのように展開して機能するかの小さな例です。TPVMイメージがscp2経由でダウンロードされた後、VMをインストールするためにいくつかのコマンドを実行する必要があります。

slx# show tpvm status
TPVM is not installed
slx# tpvm install
Installation starts. To check the status use ‘show tpvm status’ command
slx# show tpvm status
TPVM is being installed now

TPVMがすでに起動している場合、TPVMにアクセスするにはいくつかの方法があります。デフォルトでは、VMはDCHP経由でアドレスを取得しようとするか、リンクローカルIPv6アドレスを割り当てることができます。その後、SSHアクセスを取得できます。TTYを介したTPVMへのコンソール接続もあります。

スタッキングテクノロジーと同様に、スイッチのフロントパネルの最後のポートには二重の目的があり、それらをInsightモードに転送する必要があります。その後、それらを介したデータ転送は一時的に不可能になります。

slx# conf t
slx(config)# hardware
slx(config-hardware)#connector 0/48
slx(config-connector-0/48)# no breakout
slx(config-connector-0/48)# insight mode

システムが使用するインターフェイス0/125を作成します

slx(config)# interface Port-channel 22
slx(config-Port-channel-22)# insight enable
slx(config-Port-channel-22)# no shutdown
slx(config)# interface Ethernet 0/125
slx(conf-if-eth-0/125)# channel-group 22 mode on type standard
slx(conf-if-eth-0/125)# no shutdown

スイッチのインターフェースを確認します。

slx(conf-if-eth-0/125)# do show interface ethernet 0/125
Ethernet 0/125 is up, line protocol is up (connected)
slx(conf-if-eth-0/125)# do show interface port-channel 22
Port-channel 22 is up, line protocol is up
