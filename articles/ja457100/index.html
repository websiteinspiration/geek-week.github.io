AWS Lambdaのクイックツアー


それは何ですか？


AWS Lambda：これは、ほとんどすべてのタイプのアプリケーションまたはサーバーサービスのコードを実行できるコンピューティングサービスです-すべて管理なし。AWS Lambdaは、サーバーとオペレーティングシステムのメンテナンス、リソースの割り当てと自動スケーリング、コードのモニタリングとロギングなど、すべての管理を行います。AWS Lambdaがサポートする言語の1つでコードを送信するだけです。

なぜそれを使うのですか？


  • お金。サービスが実行されている時間に対してのみ支払います。
  • 速度。ラムダ自体が上昇し、非常に速く動作します。
  • 便利。Lambdaには、AWSサービスと統合するための多くの機能があります。
  • パフォーマンス。並行して、地域に応じて、最大1000〜3000コピーを実行できます。必要に応じて、サポートに書面でこの制限を引き上げることができます。

このアプローチには欠点があり、コードが実行されるオペレーティングシステムを制御できず、CPU、メモリ、リソースを制御できません。これはすべてAWSによって行われます。

サポートされているAWS Lambdaから言語を選択するだけです。

なにができる？


以下は、AWS Lambdaの主な機能の概要です。次に、すべてを順番に検討します。



1.トリガー


トリガーはラムダ病原体です。ある意味で、ラムダはPHPと比較できます。つまり、ラムダは実行されて死ぬという意味です。さらに、作業のメカニズムを詳しく検討します。とりあえず、ラムダはトリガーからのリクエストで実行される1つの関数であることを理解する必要があります。

それらのそれぞれについて、これらのトリガーで使用可能な固有のパラメーターを構成する必要があります。1つのラムダに複数のトリガーを設定することもできます。ラムダが同期的に実行されるか非同期的に実行されるかは、トリガーのタイプによって異なります。
注意：ラムダは、AWS CLI、AWS SDKを手動モードで使用して、必要なすべてのパラメーターを渡すことで実行できます。同期して実行するかどうかを含む

例を見てみましょう：


1. Gateway API -httpリクエストでラムダをプルすることができ、結果をユーザーに返す必要があります。このような操作は非同期では実行できません。答えが必要です。同期操作の場合、一部の機能は使用できません。

2. SQS-たとえば、ラムダがSQSからのメッセージを処理する場合、どこにも結果を返す必要はなく、非同期で実行できます。非同期実行では、いくつかの新機能が表示されます。たとえば、エラーが発生した場合の繰り返し実行を構成したり、そのような要求を「デッド」SQSキューにさらに送信したりできます。

2. AWSサービスへのアクセス許可


これらは、lambdaがデフォルトでアクセスできるAWSサービスです。どういう意味ですか？AWS SDKは、作成する関数にいつでも接続でき、キーや認証パラメーターなしで、利用可能なサービスを使用できます。このラムダに使用するIAMロールで使用可能なすべてのサービスを定義します。

使用される各言語には、メインのAWSサービスと通信できる独自のSDKがあります。
注意：ラムダごとに、ラムダを実行する人からIAMロールを設定します

3. VPC


たとえば、RDSに安全に接続するために、ラムダの仮想ネットワークを構成できます。

4.オンラインエディター


AWS Lambdaには、ブラウザーのインターフェイスから直接関数コードを編集する機能もあります。



5.ロギング


ラムダのすべてのリクエストはCloudWatchにログインし、ランタイムとメモリのデータもそこに記録されます。このデータは制限の設定に非常に役立ちます。また、コードには独自のデータをログに記録する機能があります（たとえば、console.logを介してNode.jsに）。

さらに、[ 監視 ]タブでラムダの使用状況に関する統計をいつでも確認できます

6.環境変数


環境変数を安全にコードに転送する機会があり、コードを配信せずにシステムの重要な部分を構成できます。キーを使用して環境変数を暗号化することが可能です。

7. Code


今、最も興味深い部分は、ラムダ自体がいくつかの部分で構成されていることです。

1. レイヤー -下のレイヤー。必須ではありませんが、ラムダを使用するためにいくつかのライブラリを追加する必要がある場合は、それらをメインコードとは別に配置する必要があるため、メインコードの量と関数自体の速度を大幅に節約できます。

AWS Lambdaのレイヤーは、関数に関しては永続的であり、個別に変更する必要があるという点で、dockerのレイヤーと多少似ています。他のラムダでも再利用できます。

2.関数環境 -コードには、ラムダ（ハンドラー）が起動されるたびに直接実行される関数が必ず含まれている必要があります。。下の彼女について。そして彼女の前には彼女の環境があります。事実、リソース管理は、この環境が機能とは別に、その完了後しばらくの間保存されるように行われます。そして次の開始時に、初期化に時間とリソースを費やすことなく再開します。したがって、構成、ライブラリの接続など、可能なすべてのものを関数自体に初期化する必要があります。

3.ハンドラー -言語に応じて、直接実行可能なコード自体は異なる方法で定義されます。たとえば、Node.Jsを取ります。コードを実行するには、次のものが必要です。

  1. jsファイル-1個
  2. exports.yourFunction =（）=> {//あなたのコード}-1pc
