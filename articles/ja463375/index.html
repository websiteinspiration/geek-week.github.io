Windows 10のWine。

Wine  は、Unixを実行しているコンピュータでWindowsアプリケーションを実行するためのプログラムです。

WindowsでWineを起動することは、少なくとも2004年以降、誰かがCygwinでWineをコンパイルしてホストレジストリを壊そうとしたとき、「私たちは実行する義務がないので、実行する必要があることを実行します」というファンの声に従うファンの夢でした。システム。

正当化：「しかし、最近のWindowsで適切に動作しない古いアプリケーションはどうですか？」しかし、本当の理由は「私は自分のアートのために長い間苦しんできた、今はあなたの番だ」と理解しています。

2008年の終わりに、私は強迫観念に打ちひしがれ、そして私（たぶん私だったようです）がWineウィキにWineOnWindowsページを作成しました。つまり、すべてがCygwin、MinGW、またはWindows Services for Unixで機能していたため、これはまったく不可能でした。現在のページはそれほど成功していません。

Windows 10ではLinux用のWindowsサブシステムが導入され、UbuntuはMicrosoftストアから簡単にダウンロードできます。その結果、DISPLAY環境変数を手動で設定する必要がない限り、私たちの愚かなアイデアはほとんどそれ自体で実現されます。

今まで、それは息をのむほど役に立たない。 Wineは64ビットWindowsアプリケーションでのみ機能します。つまり、Windows 10自体に付属するすべてのアプリケーションでさえ機能しません。

（最初のプロジェクトは、誰かがWindows 10でEncarta 97を実行できなくなった後に開始されました。したがって、実際のマニアックなソリューションと同様に、元のユーザーの問題は実際には解決しません）。

しかし、私はもう一度強調したいと思います：今それはささいに動作します。私は素晴らしいプロジェクトを実現したような狂気の天才ではありません-この事実を公に発表したのは私が初めてのようです。

この愚かなことをする方法


1. 64ビットのWindows 10を使用していますか？これはWSLを備えた唯一のバージョンです。

2. WSLをインストールします。コントロールパネル->アプリケーション->プログラムと機能-> Windowsの機能をオンまたはオフにします。 [LinuxのWindowsサブシステム]チェックボックスをオンにします。 Windowsを再起動します。

3. Microsoftストアに移動し、Ubuntuをインストールします（基本的に、それを起動するためにWSLが作成されています）。 Ubuntu 18.04 LTSをインストールしました。 Ubuntuを開くと、bashターミナルが表示されます。

4. bashコマンドラインから以下をインストールします。

sudo dpkg --add-architecture i386
sudo apt update; sudo apt upgrade
sudo apt install wine-stable

いじくり回し たい場合は、Wineの最新バージョンをインストールできます（wine-devel> = 4.5に必要な、Ubuntuにまだ含まれていない2つの新しいライブラリを忘れないでください！）。これが機能するかどうか教えてください。

5..bashrc次の行を追加します。

export DISPLAY=:0.0

これを現在のbashウィンドウでも実行したいと思うでしょう。

6. Windows用にコンパイルされたxorgの美しくパッケージ化されたバージョンであるVcXsrvをインストールします。最新の実行可能ファイルを取得して実行するだけです。 [スタート]ボタンとXLaunchからXサーバーを起動します。デフォルトをガイドします-それらのほとんどをそのままにしておきます。念のため、「アクセス制御を無効にする」をチェックしました。構成を保存します。

6a。 Xサーバーが正しく構成されていることを確認したい場合は、インストールsudo apt install x11-appsして実行xeyesし、80-90年代に戻ってください。

7. 32ビットバイナリはWSLで起動しないwineため、それ自体は機能しません。 64ビットWindows 10でのエラー/usr/bin/wine: 40: exec: /usr/lib/wine/wine: Exec format error。どうやら、これはWSL 2で修正されました。

それまでの間、Wineでメモ帳を実行してください！

wine64 /usr/lib/x86_64-linux-gnu/wine/fakedlls/notepad.exe
