最初のAIを書いたとき

こんにちは、ハブル。私の物語は2019年1月に始まります。

私のTimaゲーム開発者と私は、歴史上最大のプロジェクトである2Dプラットフォーマーに取り組むことにしました。いいえ、これまでFlappyBirdsやヘビを使用したことはありませんでしたが、このプロジェクトの作業量は頭を痛めました。そもそも、通常の縦階段をあきらめて、階段を上った。「鍵で」ロックできるドアのロジックを書き、破壊可能なブロックをねじ込みました。人工知能を書く必要がある時が来ました。経験の浅いチームで最も経験豊富なスクリプト作成者として、AIを書くことの名誉は私にありました。私は枕を叫びましたが、何をすべきか理解できませんでした。私は、プロジェクトで最も複雑なメカニズムの1つを書くことを非常に誇りに思いました。

ステージ1：パスを見つけてパスに沿って移動する


ステップ1.1：方法を見つける

主要な場所は戸外ではなく建物であるため、数十のドア、階段、部屋の間のルートを検索する必要がありました。チームリーダーは考えた結果、A *アルゴリズムの特定のパロディーを実行する価値があると判断しました。このアルゴリズムでは、ボットが実行されるノードが存在します。テストシーンを作成し、ノードを配置しました。わかりやすくするために、SprateRenderersを吊り下げました。次はどうする？

私はこの質問を頭の中で3日間続けました。これまでのところ、脳神経細胞のようにノードが興奮するとき、私の仲間の1人は興味深いアルゴリズムを提案していません。

そう。ノードAがあり、その近くにAIがあり、ノードBがあり、そこにAIが来るはずです。すべてのノードにIDを割り当て、信号を送信する接続ノードをマークしました。各ノードには、独自のブール変数「isChecked」と変数「triggeredBy」があり、これには「興奮」したノードのIDが含まれていました。したがって、ノードBが影響を受けると、ノードAへのチェーンを経由して、信号が通過したノードのすべてのIDを認識します。したがって、ボットが通過するノードのIDからパスを取得しました。これがどのように機能するかが突然わからなくなったら、話をします。

ステップ1.2.1：パスに沿った移動

ノードIDのリストがあり、ボットがあります。次は何ですか？次に、このパスに沿ってボットを移動するために必要なもの。

さて、私はこのオプションを考えました：ボットはノードに到達し、チェックされ、次のものを探し、次のノードに行きました。できました。獲得した。嬉しかった…でも…

ステージ1.2.2：ラダーとAIとの相互作用

白黒のヒーローの一人が言ったように：「階段...私の主な敵..」

次のノードがAIの上にあるのか、レベルの下にあるのか、レベル上にあるのかを判断する必要がありました。この情報に応じて、彼は階段を通過する（衝突を無視する）か、階段に上ります（衝突と相互作用します）。ああ、そしてエンジンとのこの戦いで多くの神経細胞が死んだ...フォーラムでは、すべてをレイヤーに配置し、[編集]-> [Physics2D]タブで、レイヤーとレイヤーの衝突を無視して設定できると書いています。すべてうまくいきました！
彼にドアを開けるように教えるだけです。問題ありませんでした。

ステージ2：感情と手がかり


ステージ2.1：感情

はい、私たちは感情を付けることにしました...そして、キュー。

感情は表情とアクションアニメーションで強調されます。

レプリカはテキストオーバーヘッドで表示されます。

私は感情を一度にねじ込みました...このため、感情のIDを格納する変数「emotionID」をすでに作成しました。そして、これがレプリカです...

ステップ2.2：レプリカ

美しさのために別のクラスのフレーズを作りました

[System.Serializable] //     
class Phrases
{
    public string Name;     // ,   
    public int byEmotionID; // ,     
    public string[] Phrases;//  
}

このクラスの配列を作成しました。さらに、感情IDに応じて、リストから任意のフレーズを追加しました。N秒ごとに更新されます。

しかし、私はさらに行くことにしました！各文字について、.phrs拡張子を持つファイルを作成し、ファイル内の各文字のバイト数にXバイトを追加することでこれをエンコードしました。結果は、判読不能で変更不可能なテキストでした。マークアップのようなものを行い、このマークアップによってこの配列のすべてを取得してPhrasesクラスに変換するアルゴリズムを作成しました。
いいね！すべてが機能しています！
