クリーンアップ属性

GCCドキュメント[1]からの引用：

cleanup属性は、変数がスコープ外になったときに関数を実行するために使用されます。この属性は自動変数にのみ適用でき、パラメーターや静的変数と共に使用することはできません。関数は1つのパラメーター、つまり変数と互換性のある型へのポインターを受け取る必要があります。関数の戻り値は無視されます。

-fexceptionsオプションが有効になっている場合、例外処理中にスタックが展開されると、cleanup_function関数が起動します。 cleanup属性は例外をキャッチせず、アクションを実行するだけであることに注意してください。 cleanup_functionが正常に戻らない場合の動作は未定義です。



cleanup属性は、gccおよびclangコンパイラでサポートされています。

この記事では、クリーンアップ属性を実際に使用するためのさまざまなオプションについて説明し、クリーンアップを使用してCでstd :: unique_ptrおよびstd :: shared_ptrアナログを実装するライブラリの内部構造を検討します。

メモリの割り当て解除をクリーンアップしてみましょう。

#include<stdlib.h>
#include<stdio.h>

static void free_int(int **ptr) 
{
    free(*ptr); 
    printf("cleanup done\n");
}

int main()
{
    __attribute__((cleanup(free_int))) int *ptr_one = (int *)malloc(sizeof(int));
    // do something here
    return 0;
}

プログラムが「クリーンアップ完了」と出力します。すべてがうまくいきます。

しかし、すぐに1つの欠点が明らかになります。

__attribute__((cleanup(free_int)))

cleanup属性によって呼び出される関数は、解放された変数へのポインターを引数として取る必要があり、割り当てられたメモリ領域へのポインターがあるため、ダブルポインターを受け取る関数が必要です。これを行うには、追加のラッパー関数が必要です。

static void free_int(int **ptr) 
{
    free(*ptr); 
    ...
}

さらに、ユニバーサル関数を使用して変数を解放することはできません。変数が異なるタイプの引数を必要とするためです。したがって、次のように関数を書き換えます。

static void _free(void *p) {
    free(*(void**) p);
    printf("cleanup done\n");  
}

今、彼女はどんなポインターも受け入れることができます。

        struct __useless_struct_to_allow_trailing_semicolon__

これは後でこのように使用できます：

DEFINE_TRIVIAL_CLEANUP_FUNC(FILE*, pclose);
#define _cleanup_pclose_ __attribute__((cleanup(pclosep)))

しかし、それだけではありません。この属性を使用して、unique_ptrおよびshared_ptrプラスのアナログを実装するライブラリがあります。https : //github.com/Snaipe/libcs​​ptr

使用例（[2]から取得）：

#include <stdio.h>
#include <csptr/smart_ptr.h>
#include <csptr/array.h>

void print_int(void *ptr, void *meta) {
    (void) meta;
    // ptr points to the current element
    // meta points to the array metadata (global to the array), if any.
    printf("%d\n", *(int*) ptr);
}

int main(void) {
    // Destructors for array types are run on every element of the
    // array before destruction.
    smart int *ints = unique_ptr(int[5], {5, 4, 3, 2, 1}, print_int);
    // ints == {5, 4, 3, 2, 1}

    // Smart arrays are length-aware
    for (size_t i = 0; i < array_length(ints); ++i) {
        ints[i] = i + 1;
    }
    // ints == {1, 2, 3, 4, 5}

    return 0;
}

まあ、それが私たちがCを愛している理由です。ライブラリにもドキュメントがあります（神聖な人々、誰もがそれらから例をとることをお勧めします）：smalloc（）

関数はアロケータ（デフォルトではmalloc（3））を呼び出し、返されるポインタは「スマート」ポインタです。<...>サイズが0の場合、NULLが返されます。nmembが0の場合、smallocは少なくともsizeバイトのメモリブロックへのスマートポインターを返し、nmembが0と等しくない場合、少なくともsize * nmembのサイズのメモリブロックへのポインターが返され、ポインターの型は配列になります。

元の
«The smalloc() function calls an allocator (malloc (3) by default), such that the returned pointer is a smart pointer. <...> If size is 0, then smalloc() returns NULL. If nmemb is 0, then smalloc shall return a smart pointer to a memory block of at least size bytes, and the smart pointer is a scalar. Otherwise, it shall return a memory block to at least size * nmemb bytes, and the smart pointer is an array.»

smallocのソースは次のとおりです。

__attribute__ ((malloc)) void *smalloc(s_smalloc_args *args) {
    return (args->nmemb == 0 ? smalloc_impl : smalloc_array)(args);
}

スカラー型のオブジェクトを割り当てるコードsmalloc_implを見てみましょう。ボリュームを減らすために、共有ポインターに関連付けられているコードを削除し、インライン置換とマクロ置換を行いました。

static void *smalloc_impl(s_smalloc_args *args) {
    if (!args->size)
        return NULL;

    // align the sizes to the size of a word
    size_t aligned_metasize = align(args->meta.size);
    size_t size = align(args->size);

    size_t head_size = sizeof (s_meta);
    s_meta_shared *ptr = malloc(head_size + size + aligned_metasize + sizeof (size_t));

    if (ptr == NULL)
        return NULL;

    char *shifted = (char *) ptr + head_size;
    if (args->meta.size && args->meta.data)
        memcpy(shifted, args->meta.data, args->meta.size);

    size_t *sz = (size_t *) (shifted + aligned_metasize);
    *sz = head_size + aligned_metasize;

    *(s_meta*) ptr = (s_meta) {
        .kind = args->kind,
        .dtor = args->dtor,
        .ptr = sz + 1
    };

    return sz + 1;
}

ここでは、変数のメモリが割り当てられていることがわかります。加えて、タイプs_metaの特定のヘッダーと、単語のサイズに合わせたargs-> meta.sizeサイズのメタデータ領域に、さらに1つの単語（sizeof（size_t））が配置されています。関数は変数のメモリ領域へのポインターを返します：ptr + head_size + aligned_metasize +1。

ここでスマートはマクロです：

# define smart __attribute__ ((cleanup(sfree_stack)))

ポインターがスコープを離れると、sfree_stackが呼び出されます。

CSPTR_INLINE void sfree_stack(void *ptr) {
    union {
        void **real_ptr;
        void *ptr;
    } conv;
    conv.ptr = ptr;
    sfree(*conv.real_ptr);
    *conv.real_ptr = NULL;
}

Sfree関数（省略）：

void sfree(void *ptr) {
    s_meta *meta = get_meta(ptr);
    dealloc_entry(meta, ptr);
}

dealloc_entry関数は、unique_ptr引数で指定した場合、基本的にはカスタムデストラクタを呼び出し、それへのポインタはメタデータに格納されます。そうでない場合は、フリー（メタ）のみが実行されます。
