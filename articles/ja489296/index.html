ステロイドの機能オプション

こんにちは、ハブル！MárkSági-Kazárによる記事「ステロイドの機能オプション」の翻訳を紹介します。 機能オプションは、クリーンで拡張可能なAPIのためのGoのパラダイムです。彼女はデイブ・チェイニーとロブ・パイクによって大衆化されました。この投稿は、テンプレートが最初に導入されて以来、テンプレートを中心に行われてきた慣行に関するものです。 機能オプションは、オプションのパラメーターを含む構成でAPIを作成する方法として登場しました。これを行うには明らかな方法が数多くあります（コンストラクター、構成構造、セッターなど）。ただし、数十のオプションを渡す必要がある場合、それらは十分に読み取られず、機能的なオプションなどの優れたAPIを提供しません。







はじめに-機能オプションとは何ですか？


通常、「オブジェクト」を作成するときは、必要な引数を使用してコンストラクターを呼び出します。

obj := New(arg1, arg2)

（Goには従来のコンストラクターがないという事実を無視しましょう。）

機能オプションを使用すると、追加のパラメーターでAPIを拡張し、上記の行を次のように変更できます。

// I can still do this...
obj := New(arg1, arg2)

// ...but this works too
obj := New(arg1, arg2, myOption1, myOption2)

機能オプションは基本的に、複合または中間構成タイプをパラメーターとして取る変数関数の引数です。変数の性質上、デフォルト値を使用する場合でも、オプションなしでコンストラクターを呼び出し、それをクリーンに保つことは完全に許容されます。

パターンをよりよく示すために、現実的な例（機能オプションなし）を見てみましょう。

type Server struct {
    addr string
}

// NewServer initializes a new Server listening on addr.
func NewServer(addr string) *Server {
    return &Server {
        addr: addr,
    }
}

タイムアウトオプションを追加すると、コードは次のようになります。

type Server struct {
    addr string

    // default: no timeout
    timeout time.Duration
}

// Timeout configures a maximum length of idle connection in Server.
func Timeout(timeout time.Duration) func(*Server) {
    return func(s *Server) {
        s.timeout = timeout
    }
}

// NewServer initializes a new Server listening on addr with optional configuration.
func NewServer(addr string, opts ...func(*Server)) *Server {
    server := &Server {
        addr: addr,
    }

    // apply the list of options to Server
    for _, opt := range opts {
        opt(server)
    }

    return server
}

結果のAPIは使いやすく、読みやすいです。

// no optional paramters, use defaults
server := NewServer(":8080")

// configure a timeout in addition to the address
server := NewServer(":8080", Timeout(10 * time.Second))

// configure a timeout and TLS in addition to the address
server := NewServer(":8080", Timeout(10 * time.Second), TLS(&TLSConfig{}))


比較のために、コンストラクターと構成構成構造を使用した初期化は次のようになります。

// constructor variants
server := NewServer(":8080")
server := NewServerWithTimeout(":8080", 10 * time.Second)
server := NewServerWithTimeoutAndTLS(":8080", 10 * time.Second, &TLSConfig{})


// config struct
server := NewServer(":8080", Config{})
server := NewServer(":8080", Config{ Timeout: 10 * time.Second })
server := NewServer(":8080", Config{ Timeout: 10 * time.Second, TLS: &TLSConfig{} })

コンストラクタよりも機能オプションを使用することの利点はおそらく明白です。それらは保守と読み取り/書き込みが簡単です。パラメーターがコンストラクターに渡されない場合、機能パラメーターも構成構造を超えます。次のセクションでは、構成構造が期待どおりに機能しない可能性があるさらに多くの例を示します。

概要のリンクをクリックして、機能オプションの完全な履歴を読んでください。（翻訳者注-Dave Cheneyによるオリジナル記事のブログは、ロシアでは常に利用できるとは限らず、閲覧するにはVPN接続が必要な場合があります。また、記事の情報は、dotGo 2014に関する彼のレポートのビデオでご覧いただけます）

機能オプションの実践


関数オプション自体は、コンストラクタに渡される関数にすぎません。シンプルな機能の使いやすさは柔軟性をもたらし、大きな可能性を秘めています。したがって、何年にもわたってテンプレートの周囲に多くの慣行が登場していることは当然のことです。以下に、私が最も人気があり有用なプラクティスと考えるもののリストを提供します。

何か不足していると思われる場合は私にメールしてください。

オプションの種類


機能オプションテンプレートを適用するときに最初にできることは、オプション機能のタイプを決定することです。

// Option configures a Server.
type Option func(s *Server)

これは大きな改善とは思えないかもしれませんが、関数を定義する代わりに型名を使用することにより、コードが読みやすくなります。

func Timeout(timeout time.Duration) func(*Server) { /*...*/ }

// reads: a new server accepts an address
//      and a set of functions that accepts the server itself
func NewServer(addr string, opts ...func(s *Server)) *Server

// VS

func Timeout(timeout time.Duration) Option { /*...*/ }

// reads: a new server accepts an address and a set of options
func NewServer(addr string, opts ...Option) *Server

オプションタイプを持つことのもう1つの利点は、Godocがオプション関数をそのタイプの下に配置することです。



オプションタイプ一覧


通常、機能パラメータは何かの単一のインスタンスを作成するために使用されますが、これは常に当てはまるわけではありません。複数のインスタンスを作成するときにデフォルトのパラメーターリストを再利用することも珍しくありません。

defaultOptions := []Option{Timeout(5 * time.Second)}

server1 := NewServer(":8080", append(defaultOptions, MaxConnections(10))...)

server2 := NewServer(":8080", append(defaultOptions, RateLimit(10, time.Minute))...)

server3 := NewServer(":8080", append(defaultOptions, Timeout(10 * time.Second))...)

これは非常に読みやすいコードではありません。特に、機能オプションを使用する目的が使いやすいAPIを作成することであることを考えると、幸い、これを単純化する方法があります。[]オプションをオプションで直接スライスする必要があるだけです。

// Options turns a list of Option instances into an Option.
func Options(opts ...Option) Option {
    return func(s *Server) {
        for _, opt := range opts {
            opt(s)
        }
    }
}

オプション関数でスライスを置き換えた後、上記のコードは次のようになります。

defaultOptions := Options(Timeout(5 * time.Second))

server1 := NewServer(":8080", defaultOptions, MaxConnections(10))

server2 := NewServer(":8080", defaultOptions, RateLimit(10, time.Minute))

server3 := NewServer(":8080", defaultOptions, Timeout(10 * time.Second))

With / Setオプションのプレフィックス


オプションは、タイムアウトや接続の最大数とは異なり、多くの場合、複雑なタイプです。たとえば、サーバーパッケージでは、ロガーインターフェイスをオプションとして定義できます（デフォルトのロガーがない場合は、それを使用します）。

type Logger interface {
    Info(msg string)
    Error(msg string)
}

ロガー名はインターフェースですでに使用されているため、オプション名として使用することはできません。オプションLoggerOptionを呼び出すことができますが、あまりわかりやすい名前ではありません。コンストラクタを文として見ると、私たちの場合、WithLogger with Withという単語が思い浮かびます。

func WithLogger(logger Logger) Option {
    return func(s *Server) {
        s.logger = logger
    }
}

// reads: create a new server that listens on :8080 with a logger
NewServer(":8080", WithLogger(logger))

複合型オプションのもう1つの一般的な例は、値のスライスです。

type Server struct {
    // ...

    whitelistIPs []string
}

func WithWhitelistedIP(ip string) Option {
    return func(s *Server) {
        s.whitelistIPs = append(s.whitelistIPs, ip)
    }
}

NewServer(":8080", WithWhitelistedIP("10.0.0.0/8"), WithWhitelistedIP("172.16.0.0/12"))

この場合、オプションは通常、値を上書きするのではなく、既存の値に追加します。既存の値のセットを上書きする必要がある場合は、オプション名に単語セットを使用できます。

func SetWhitelistedIP(ip string) Option {
    return func(s *Server) {
        s.whitelistIPs = []string{ip}
    }
}

NewServer(
    ":8080",
    WithWhitelistedIP("10.0.0.0/8"),
    WithWhitelistedIP("172.16.0.0/12"),
    SetWhitelistedIP("192.168.0.0/16"), // overwrites any previous values
)

プリセットテンプレート


特別なユースケースは通常、ライブラリでそれらをサポートするのに十分普遍的です。構成の場合、これは、一緒にグループ化され、使用するプリセットとして使用されるパラメーターのセットを意味する場合があります。この例では、サーバーは、タイムアウト、速度制限、接続数などをさまざまな方法で設定するオープンな内部使用シナリオを持つことができます。

// PublicPreset configures a Server for public usage.
func PublicPreset() Option {
    return Options(
        WithTimeout(10 * time.Second),
        MaxConnections(10),
    )
}

// InternalPreset configures a Server for internal usage.
func InternalPreset() Option {
    return Options(
        WithTimeout(20 * time.Second),
        WithWhitelistedIP("10.0.0.0/8"),
    )
}

プリセットは場合によっては役立つかもしれませんが、おそらく内部ライブラリでは非常に価値があり、パブリックライブラリではその価値は低くなります。

デフォルトのタイプ値とデフォルトのプリセット値


Goには常にデフォルト値があります。数値の場合、これは通常ゼロで、ブール型の場合はfalseになります。オプションの構成では、デフォルト値に依存することをお勧めします。たとえば、null値は、それがない場合（通常は意味がない）ではなく無制限のタイムアウトを意味する必要があります。

場合によっては、デフォルトのタイプ値は適切なデフォルト値ではありません。たとえば、ロガーのデフォルト値はnilであり、パニックを引き起こす可能性があります（条件チェックでロガー呼び出しを保護しない場合）。

これらの場合、コンストラクターで（パラメーターを適用する前に）値を設定することは、フォールバックを決定する良い方法です。

func NewServer(addr string, opts ...func(*Server)) *Server {
    server := &Server {
        addr:   addr,
        logger: noopLogger{},
    }

    // apply the list of options to Server
    for _, opt := range opts {
        opt(server)
    }

    return server
}

デフォルトのプリセットの例を見ました（前のセクションで説明したテンプレートを使用）。しかし、私はこれを良い習慣とは考えていません。これは、コンストラクターでデフォルト値を設定するよりもはるかに表現力に欠けます：

func NewServer(addr string, opts ...func(*Server)) *Server {
    server := &Server {
        addr:   addr,
    }

    // what are the defaults?
    opts = append([]Option{DefaultPreset()}, opts...)

    // apply the list of options to Server
    for _, opt := range opts {
        opt(server)
    }

    return server
}

構成構造オプション


構成オプションを機能オプションとして持つことはおそらく一般的ではありませんが、これは時々見つかることがあります。アイデアは、作成される実際のオブジェクトを参照するのではなく、機能オプションが構成構造を参照するというものです。

type Config struct {
    Timeout time.Duration
}

type Option func(c *Config)

type Server struct {
    // ...

    config Config
}

このテンプレートは、多くのオプションがあり、構成構造を作成すると、関数呼び出しですべてのオプションをリストするよりもクリーンに見える場合に役立ちます。

config := Config{
    Timeout: 10 * time.Second
    // ...
    // lots of other options
}

NewServer(":8080", WithConfig(config))

このテンプレートのもう1つの使用例は、デフォルト値を設定することです。

config := Config{
    Timeout: 10 * time.Second
    // ...
    // lots of other options
}

NewServer(":8080", WithConfig(config), WithTimeout(20 * time.Second))

高度なテンプレート


数十の機能オプションを記述した後、これを行うより良い方法があるかどうか疑問に思うかもしれません。使用の観点からではなく、長期的なサポートの観点から。

たとえば、型を定義してパラメータとして使用できるとしたらどうでしょうか。

type Timeout time.Duration

NewServer(":8080", Timeout(time.Minute))

（APIの使用方法は同じであることに注意してください）

Optionタイプを変更することで、これを簡単に実行できることがわかります。

// Option configures a Server.
type Option interface {
    // apply is unexported,
    // so only the current package can implement this interface.
    apply(s *Server)
}

オプションの関数をインターフェイスとしてオーバーライドすると、機能オプションを実装するための多くの新しい方法への扉が開か

れます。ラッパー関数なしで、さまざまな組み込み型をパラメーターとして使用できます。

// Timeout configures a maximum length of idle connection in Server.
type Timeout time.Duration

func (t Timeout) apply(s *Server) {
    s.timeout = time.Duration(t)
}

オプションと構成構造のリスト（前のセクションを参照）は、次のように再定義することもできます。

// Options turns a list of Option instances into an Option.
type Options []Option

func (o Options) apply(s *Server) {
    for _, opt := range o {
        o.apply(s)
    }
}

type Config struct {
    Timeout time.Duration
}

func (c Config) apply(s *Server) {
    s.config = c
}

私の個人的なお気に入りは、複数のコンストラクターでオプションを再利用できることです。

// ServerOption configures a Server.
type ServerOption interface {
    applyServer(s *Server)
}

// ClientOption configures a Client.
type ClientOption interface {
    applyClient(c *Client)
}

// Option configures a Server or a Client.
type Option interface {
    ServerOption
    ClientOption
}

func WithLogger(logger Logger) Option {
    return withLogger{logger}
}

type withLogger struct {
    logger Logger
}

func (o withLogger) applyServer(s *Server) {
    s.logger = o.logger
}

func (o withLogger) applyClient(c *Client) {
    c.logger = o.logger
}

NewServer(":8080", WithLogger(logger))
NewClient("http://localhost:8080", WithLogger(logger))

概要


機能パラメーターは、数十のパラメーターを持つクリーンで拡張可能なAPIを作成するための強力なテンプレートです。これは、単純な構成構造をサポートするよりも少し多くの作業を行いますが、代替手段よりもはるかに柔軟性が高く、よりクリーンなAPIを提供します。
