V16 Update 2 Betaの新しい3CX SBCリモート接続マネージャー

先週、次の3CX v16 Update 2ベータアップデートをリリースしました。最初から書き直した3CX SBC（Session Border Controller）リモート接続マネージャーが含まれています。

リコールその3CX SBC2つ以上のIP電話が離れた場所にある場合は、3CXを使用してリモートオフィスを中央交換機に接続することをお勧めします。同時に、PBXへのローカル接続と同様に、SIPシグナリングは3CXサーバーを通過し、メディアストリーム（音声）はオフィスを離れることなくリモートデバイス間を直接通過します。これにより、通信の品質が向上するだけでなく、外部通信チャネルの負荷が軽減され、ローカルコールがローカルネットワーク内に「残されます」。さらに、3CX SBCはすべてのVoIPトラフィックを単一のTCPポート経由でトンネリングするため、境界ルーターの構成が大幅に簡素化されます。 SBCを使用すると、IP電話をリモートで管理したり、ファームウェアを再構成したり、更新したりできます。また、リモートオフィスと3CXサーバー間のVoIPトラフィックを暗号化して、プライバシーを強化できます。

V16 Update 2 Betaとともに、ロシア語の3CXクイックスタートガイドもリリースしました。取引所で番号を作成するときに、ユーザーに送信されます（内線のパラメーターで指定された電子メールによって）。

3CX SBC v16の機能


新しい3CX SBC v16には、次の機能があります。

  • 簡素化されたサービスのインストール。SBCをインストールするには、3QXサーバーのFQDNとSBCトンネル認証パスワード（3CXトンネル技術）のみを指定する必要があります（以下を参照）。
  • サービス管理。SBCがリモートで更新され、3CXインターフェイスで再構成されます（以下を参照）。SBC自動更新は、LinuxおよびRaspberry Piでのみ使用できます。
  • 接続されているすべてのSBCの集中監視。SBCトランクごとに、管理者は接続ステータスの変更を確認します。サービスの統計、イベントログ、電子メール通知も利用できます。

SBC v16は3CX V16 Update 2でのみ機能することに注意してください。信頼できるSSL証明書を3CXサーバーにインストールする必要があります（自己署名証明書はサポートされていません）。

3CX Updateをインストールする


3CXインターフェースの標準アップデート（Linuxのみ）に加えて、もちろん、完全なシステムディストリビューションをダウンロードできます。

  • Windows：3CX v16 Update 2 Beta
  • Linux：Debian 9 ISO上の3CX v16-3CX Update Channel画面から、 Betaを選択します。
  • DebianおよびRaspberry Piに 3CXを手動でインストールします。

完全な変更ログ。

3CX SBC接続の構成


3CX SBCは最新バージョンのRaspberry Pi（Raspbian Linux）にインストールでき、最大30台のIP電話を処理できます。各電話について、10個以下のBLFインジケータを設定することをお勧めします。ステータスメッセージングダウンロードサービス。ただし、SBCをWindowsにインストールするか、Debian Linuxを推奨ハードウェアにインストールすると、追加の電話が許可されます。

SBCをインストールする予定のコンピューターには、空きポート5060（TCPおよびUDP）があり、DNS、DHCP、SQLサーバー、または同様のネットワークリソースであってはなりません。一般に、このコンピューターを他のタスクに使用することはお勧めしません。

サーバー側3CXからのSBC接続


[SIP Trunks]セクションの[Add 3CX SBC]ボタンで新しいSBC接続を作成します。



HeadquartersToMoscowBranchなど、SBC接続の名前を指定します（特殊文字は使用しないでください）。次に、接続設定ページで、認証IDとWebリンクをコピーします。これらのパラメータは、リモートオフィスにSBCをインストールするときに指定されます。



SBC接続のパラメータを検討してください。

基本タブ：

  • 接続情報-接続名とトンネルパスワード。
  • ネットワーク情報-リモートSBCに接続するためのURLおよびID。

統計タブ：

  • トンネルステータス、アップタイム、ソフトウェアバージョン、この接続のコール数とユーザー。

オプションタブ：

  • トンネルセキュリティレベル（TLSまたはTCP）
  • 最初のRTPオーディオポート番号は、SBCを通過するオーディオストリームが使用する最初のポートです。
  • フォールトトレランスを有効にする-3CXバックアップサーバーの外部IPアドレスを入力します。プライマリサーバーに障害が発生した場合、SBCはバックアップサーバーに切り替えます。
  • ロギングとトラブルシューティング-ロギングレベルを選択します。「長い間詳細なログレベル」を設定しないでください。これは、特にRaspberry Piを使用している場合、過負荷につながります。    

ボタンは、3CX SIPトランクインターフェイスセクションでも使用できます。

  • 「構成のプッシュ」-選択したSBCサービスに更新された構成を送信します。
  • 「更新」-SBCサービスを最新の安定バージョンに更新します。

リモートオフィスにSBCサービスをインストールする


3CXでSBC接続を構成した後、リモートオフィスにSBCをインストールできます。

DebianおよびRaspbian LinuxへのSBCのインストール




コマンドラインインストール：

安定

wget http://downloads.3cx.com/downloads/sbc/3cxsbc.zip -O- | sudo bash

ベータ版

wget http://downloads.3cx.com/downloads/sbc/3cxsbcV16.zip  -O- | sudo bash



ISOイメージからのインストール：

Linuxのインストールが完了したら、[3CXアップデートチャネル]画面で[3CX SBC安定]または[3CX SBCベータ]を選択します。



次に、SBC接続設定から3CXサーバーのURLと認証IDを指定します。



これで、リモートIP電話を接続できます（接続モード-SBC経由の接続）。

WindowsにSBCをインストールする


3CX SBC for Windows ディストリビューションを 実行し、インストールプロセス中にサーバーのURLと認証IDを指定します。



LinuxでのSBCの手動更新（以前のバージョンからのアップグレード）


APTリポジトリを更新します。 

sudo apt update

安定版に更新：

sudo apt upgrade 3cxsbc

ベータ更新：
sudo apt -t testing upgrade 3cxsbc

以前のバージョンのSBC（Linuxのみ）を手動で更新すると、対応するSBC接続が3CX管理インターフェースに自動的に表示されます。接続を選択し、「構成の送信」ボタンをクリックします。設定を更新したら、IP電話を接続できます。
