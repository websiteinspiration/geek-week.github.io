クローンナンピー



サードパーティのライブラリを使用せずにPythonで配列を操作するための小さなモジュール（NumPyクローン、ただし純粋なPythonのみ）。

大学の宿題は、行列の基準と分解を計算するプログラムを書くように求められましたが、サードパーティのライブラリを使用することは禁止されていました。プログラミング言語の選択は制限されません。私はpythonを選択しました（JavaおよびC / C ++よりもはるかに遅いため、これは間違いでした）。したがって、NumPyを使用できません。その過程で、配列を使った操作を実行するための関数、未成年者を見つけるための関数、行列式などを書く必要がありました。結果は、配列を操作するためのミニライブラリです。

私の純粋なpythonコードは、CおよびFortranの計算を行うNumPy（さらに私のコードは最適化されていません）よりもはるかに低速です。

  • 逆行列

  • マトリックス条件番号
  • 1番目、2番目（ファイナライズされていない）、ユークリッドおよび無限行列ノルム
  • 方程式AX = Bの解
  • LU分解
  • コレスキー分解
  • ザイデル法

MathLOLパフォーマンス


サイズNxNの行列の積を計算する速度を見てみましょう。行列は、-100〜100のランダムな整数で埋められます。

コード
from mathlol import mathlol

import time
import random
import matplotlib.pyplot as plt

#   
data = {}
for i in range(10, 110, 10):
    array = []
    for i_ in range(i):
        temp = []
        for j_ in range(i):
            temp.append(random.randint(-100, 100))
        array.append(temp)
    data[i] = array

#     
mlol_dot = {}
for key in data.keys():
    matrix = mathlol()
    matrix.set(matrix = data[key])
    
    start = time.process_time()
    result = matrix * matrix
    end = time.process_time() - start
    
    mlol_dot[key] = end

#  
plt.plot(mlol_dot.keys(), mlol_dot.values())

plt.title("MathLOL \n ")

plt.xlabel("  (NxN)")
plt.ylabel(" ()")




100x100から1000x1000までの行列積サイズ



の計算速度numpyとmathlolの計算速度を比較します。残念ながら、mathlolは速度が非常に遅いため、numpyの場合は100x100から1000x1000、mathlolの場合は10x10から100x100のマトリックスを取ることにしました。



MathLOLは0.16秒でマトリックス100x100の積を単独で計算し、NumPyは0.002（!!!）秒でマトリックス1000x1000の積を単独で計算しました。違いは大きいです。

私たちの仕事は、行列を操作するためのさまざまな関数を実装することでしたが、大きな行列を使用するプログラムは、思ったほど速く動作しません。プログラムを改良し、さらにいくつかの関数（たとえば、トッド数を計算するための関数）を追加することは残ります。コードを見て、エラーを指摘し、おそらくコードの改良に役立てていただければ幸いです。
