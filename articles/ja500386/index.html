古典的なソリューションを見てみましょう：SDWebImageライブラリを使用して画像を非同期にアップロードします。



便宜上、ライブラリの操作はImageManagerにカプセル化されています。

画像のダウンロードとキャッシュ用。

伝統的に、受信UIImageView結果との通信は2つの方法で実装されます：

  • この同じUIImageViewへの弱いリンクを渡すことによって;
  • クロージャーブロックをImageManagerメソッドに渡す


ImageManagerの呼び出しは通常、UIImageView拡張機能のいずれかにカプセル化されます。

extension UIImageView {
 func setup(by key: String) {
        ImageManager.sharedInstance.setImage(toImageView: self, forKey: key)
    }
}

後継クラスのいずれか：

class CachedImageView : UIImageView {
    private var _imageUrl: String?
    var imageUrl: String?  {
        get {
            return _imageUrl
        }
        set {
            self._imageUrl = newValue
            if let url = newValue, !url.isEmpty {
                self.setup(by: url)
            }
        }
    }
    
    func setup(by key: String) {
        ImageManager.sharedInstance.setImage(toImageView: self, forKey: key)
    }
}

次に、このソリューションをSwiftUIにねじ込みましょう。ただし、適応するときは、フレームワークの次の機能を考慮する必要があります。-

ビュー-構造。継承はサポートされていません


-通常の意味での拡張は役に立たない。もちろん、機能を拡張するためにいくつかのメソッドを作成することもできますが、これを何らかの方法でDataFlowにバインドする必要があります。

フィードバックを得るという問題と、UIとの対話のロジック全体をDataDriven Flowに適合させる必要性が生じます。

ソリューションについては、ビュー側とデータフロー適応側の両方から行くことができます。

ビューから始めましょう。

まず、SwiftUIはそれ自体ではなく、UIKitのアドオンとして存在することを思い出してください。。SwiftUI開発者は、SwiftUI UIViewで使用するためのメカニズムを提供しています。その類似体は既製のコントロールには含まれていません。そのような場合のために、UIViewおよびUIViewControllerをそれぞれ適応させるためのUIViewRepresentableおよびUIViewControllerRepresentableプロトコルがあります。

メソッドを再定義するUIViewRepresentableを実装するビュー構造を作成します。

  • makeUiView;
  • updateUIView

ここでは、使用しているUIViewを示し、その基本設定を設定します。そして、変更可能なプロパティのPropertyWrappersを忘れないでください。

struct WrappedCachedImage : UIViewRepresentable {
    let height: CGFloat
    @State var imageUrl: String
    
    func makeUIView(context: Context) -> CachedImageView {
        let frame = CGRect(x: 20, y: 0, width: UIScreen.main.bounds.size.width - 40, 
                                     height: height)
        return CachedImageView(frame: frame)
    }
    
    func updateUIView(_ uiView: CachedImageView, context: Context) {
        uiView.imageUrl = imageUrl
        uiView.contentMode = .scaleToFill
    }
}

結果の新しいコントロールをView SwiftUIに埋め込むことができます。



このアプローチには利点があります。

  • 既存のライブラリの操作を変更する必要はありません
  • ロジックは、埋め込まれたUIViewにカプセル化されます。

しかし、新しい責任があります。まず、View-UIViewバンドルでメモリ管理を監視する必要があります。ビュー構造以降、それらを使用するすべての作業は、フレームワーク自体によってバックグラウンドで実行されます。しかし、新しいオブジェクトのクリーニングは開発者の肩にかかります。

次に、カスタマイズ（サイズ、スタイル）に追加の手順が必要です。 Viewでこれらのオプションがデフォルトで有効になっている場合は、UIViewと同期する必要があります。

たとえば、サイズを調整するには、GeometryReaderを使用して、画像が画面の幅全体と定義した高さを占めるようにします。

 var body: some View {
      GeometryReader { geometry in 
        VStack {
           WrappedCachedImage(height:300, imageUrl: imageUrl) 
           .frame(minWidth: 0, maxWidth: geometry.size.width,
                     minHeight: 0, maxHeight: 300)
        }
    }
}

原則として、このような場合、埋め込まれたUIViewの使用はオーバーエンジニアリングと見なすことができます。それでは、DataFlow SwiftUIを介して解決してみましょう。

表示されるビューは、状態変数または変数のグループに依存しています。それ自体がこの状態変数である特定のモデルから。本質的に、この相互作用はMVVMパターンに基づいています。

次のように実装します。

  • SwiftUIコントロールを使用するカスタムビューを作成します。
  • モデルを操作するロジック（ImageManager）を転送するViewModelを作成します。


ViewとViewModelの間に接続を確立するには、ViewModelがObservableObjectプロトコルを実装し、ObservedObjectとしてビューに接続する必要があります。

class CachedImageModel : ObservableObject {
    @Published var image: UIImage = UIImage()
    
    private var urlString: String = ""
    
    init(urlString:String) {
        self.urlString = urlString
    }
    
    func loadImage() {
        ImageManager.sharedInstance
        .receiveImage(forKey: urlString) {[weak self] (im) in
            guard let self = self else {return}
            DispatchQueue.main.async {
                self.image = im
            }
        }
    }
}

ライフサイクルのonAppearメソッドでのビューは、ViewModelメソッドを呼び出し、@ Publishedプロパティから最終的な画像を取得します。 

struct CachedLoaderImage : View {
    @ObservedObject var  model:CachedImageModel
    
    init(withURL url:String) {
        self.model = CachedImageModel(urlString: url)
    }
    
    var body: some View {
        Image(uiImage: model.image)
            .resizable()
            .onAppear{
                self.model.loadImage()
        }
    }
    
}

DataFlow SwiftUIを操作するための宣言型Combine APIもあります。これを操作することは、リアクティブフレームワーク（同じRxSwift）を操作することと非常に似ています。サブジェクトがあり、サブスクライバーがあり、同様の管理方法があり、（ディスポーザブルではなく）キャンセル可能です。

class ImageLoader: ObservableObject {
 @Published var image: UIImage?
 private var cancellable: AnyCancellable?

 func load(url: String) {
     cancellable = ImageManager.sharedInstance.publisher(for: url)
         .map { UIImage(data: $0.data) }
         .replaceError(with: nil)
         .receive(on: DispatchQueue.main)
         .assign(to: \.image, on: self)
 }
