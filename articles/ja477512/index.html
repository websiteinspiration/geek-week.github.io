L2TP、RRAS対SoftEtherの戦い


「注目を集めるために西部から盗まれた写真」

これまでの記事では、Windows Server Core 2019でVDSを使用する方法について、月額99ルーブルの新しいUltraLight料金で話しました。この関税を使用する別の方法を提供します。今回は、遅延または静的IPアドレス用のVPNが必要な場合に選択する方が良い方法について話します。ローカルネットワークで本当にヒーローやWarcraft 3をプレイしたい場合は、Hamachiや他のすべての代わりに使用すると便利です。それはチューニングではなく、パフォーマンスについて話しましょう。

テスト方法


RRASとSoftEtherは、インストールの容易さ、L2TPプロトコルのサポート、およびGUiを介した制御機能に基づいて選択されました。

SoftEtherおよびRRASの場合、共有キーを使用したL2TP接続が標準のWindowsツールを介して使用されました。それが確立されたので、それはテストされました。

SoftEtherのオペレーティングシステムは、RRAS Windows Server Core 2019のUbuntu 18.04 LTSです。テスト前のすべてのオペレーティングシステムは、2019年11月21日の状態に関する最新の更新を受け取りました。 

第2世代のHyper-V仮想マシンには、1 GBのRAMとプロセッサの制限がインストールされています。テストグループの順序は次のとおりです

すべてのVPNサーバーがそのまま設定を適用し、NATが有効になりました。すべての仮想マシンは同じホスト上にあり、同じ仮想スイッチ上にあります。

システムメモリ消費


SoftEtherによって消費されるRAMの量は、コアの数から122 MBから177 MBに増加しましたが、RRASよりは少ないです。

RRASサービス自体の重量は、メモリ全体で約200メガバイトからシステムの総消費量を差し引いたものです。

結論：


SoftEtherへの接続は、プロセッサが制限されたときに初めて成功しませんでした。最初に制限を増やし、接続してから制限を下げる必要がありました。これは、非常に薄い環境でのインストールに制限を課します。RRASは常に即座にログインしました。

コアの多いマシンを使用している場合は、RRASを選択してください。SoftEtherの場合、4つのコアを残すことができます。もし作者がそれを使っていたら、コアは1つだけ残していたでしょう。
