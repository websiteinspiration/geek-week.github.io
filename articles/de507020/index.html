Referenzarchitekturen: Wie erstelle ich ein softwaredefiniertes Rechenzentrum?

Zum Erstellen eines softwaredefinierten Rechenzentrums können von Anbietern vorgeschlagene Referenzarchitekturen verwendet werden. Im Folgenden wird beschrieben, wie Sie ein VMware Software Defined Data Center mit Lenovo ThinkAgile VX-Knoten erstellen.

Was ist SDDC?


Was ist ein softwaredefiniertes (oder softwarekonfigurierbares) Rechenzentrum (Software Defined Data Center, SDDC)? Erstens ist es ein Rechenzentrum mit einem hohen Automatisierungsgrad, das praktisch keinen physischen Eingriff erfordert. Alle Infrastrukturkomponenten eines solchen Rechenzentrums werden virtualisiert und die Verwaltung mithilfe von Software vollständig automatisiert. Bei VMware SDDC handelt es sich um vSphere-Software. VMware vSphere bietet Virtualisierung auf allen Ebenen, einschließlich der Ebenen der Computer-, Netzwerk- und Speicherressourcen.


Mit SDDC können Sie Benutzern flexibel Computer-, Netzwerk- und Speicherdienste bereitstellen.

In einem softwaredefinierten Rechenzentrum bietet die richtlinienbasierte Automatisierung die Ressourcenzuweisung und -verwaltung. Das Ergebnis ist die Flexibilität und Effizienz der IT, die alle Arten von Geräten und Anwendungen unterstützt. Heute ist SDDC ein bewährter Architekturansatz, der auf Virtualisierung und Automatisierung basiert. Er kann schrittweise implementiert werden, ohne die vorhandene IT-Infrastruktur sofort zu ersetzen.


In einem softwaredefinierten Rechenzentrum wird die gesamte Infrastruktur virtualisiert und die Verwaltung mithilfe von Software automatisiert. VMware fördert das SDDC-Konzept seit 2012.

Wofür ist das?


Mit der zunehmenden Komplexität der IT wird es schwieriger, die IT-Infrastruktur im Rechenzentrum zu verwalten und die Auswirkungen der Änderungen zu antizipieren. Die Anforderungen des Unternehmens überlagern sich mit Infrastrukturproblemen: Kostenreduzierung, flexiblere Verwendung von Anwendungen, die die Benutzeranforderungen hinsichtlich Stabilität, Leistung, Verfügbarkeit und einfacher Aktualisierung erfüllen können.


VMware SDDC enthält die gesamte Software, die zum Aufbau einer Unternehmensinfrastruktur erforderlich ist.

Zu den VMware SDDC-Produkten gehören die VMware vSphere Computing-Virtualisierungssoftware, die VMware vSphere- und VMware vSAN-Speichervirtualisierung, die VMware NSX-Netzwerkvirtualisierung sowie die Cloud-Verwaltungsplattform VMware vRealize und das integrierte VMware EVO-System.

Ziel ist eine flexible, einfach zu verwaltende IT-Infrastruktur, die an sich ändernde Anforderungen angepasst werden kann. Dank der Virtualisierung von Computer-, Speicher- und SDDC-Netzwerken ist es weniger abhängig von physischer Hardware, der zugrunde liegenden Hardwareplattform. Die richtlinienbasierte Konfiguration und Bereitstellung von IT-Ressourcen bei Bedarf vereinfacht die Verwaltung von Rechenzentren und erweitert und aktualisiert die Infrastruktur.

Das Ergebnis der Erstellung von SDDC ist eine hohe Effizienz und niedrige Kosten. Virtualisierte IT-Services und automatisiertes Betriebsmanagement eröffnen neue Möglichkeiten für den Einsatz von Ressourcen und die Steigerung der Mitarbeiterproduktivität. Durch die richtlinienbasierte Konfiguration können Sie Workloads schnell bereitstellen, und die Ressourcennutzung wird automatisch so konfiguriert, dass sie sich ändernden Anforderungen entspricht. Die Automatisierung von Sicherheitsfunktionen unter Berücksichtigung der Virtualisierung verlängert die Verfügbarkeit.

Anbieter bieten verschiedene Hardware- und Softwarekonfigurationen für die Bereitstellung von SDDC an, einschließlich schlüsselfertiger hyperkonvergenter Lösungen - integrierte Hardware- und Softwaresysteme sowie Referenz- und Referenzarchitekturen. Sie sind für eine Vielzahl von Anwendungen und Workloads konzipiert.

Im Folgenden wird eine Referenzarchitektur zum Erstellen eines softwaredefinierten VMware-Rechenzentrums mit zertifizierten Lenovo ThinkAgile VX-Knoten und -Geräten sowie Komponenten zur Integration einer lokalen VMware vRealize-Cloud in Amazon AWS vorgestellt.

VMware vSAN


VMware vSAN ist eine SDS-Lösung (Software Defined Storage), die in ESXi-Hypervisoren integriert ist. VMware vSAN kombiniert den Flash-Cache auf drei oder mehr mit 10 GbE verbundenen Servern in einem gemeinsam genutzten Datenspeicher. Laut einer Gartner-Studie wird der Anteil von softwaredefiniertem Speicher am Weltmarkt in den nächsten fünf Jahren auf 50% steigen.


VMware vSAN kann bis zu 64 Server skalieren, wobei jeder Server bis zu fünf Laufwerksgruppen unterstützt. Die Gruppe umfasst Solid-State-Laufwerke (SSD) und bis zu sieben Festplatten (HDD). Die Speicherleistung und -kapazität kann leicht durch Hinzufügen von Festplatten, Flash-Speicher oder Servern erhöht werden. Der Cache wird verwendet, um das Lesen und Schreiben zu beschleunigen.

Die wichtigsten auf vSAN gehosteten Workloads. 64% sind geschäftskritische Anwendungen (laut VMware-Umfrage).

VMware vSAN verwendet eine verteilte RAID-Architektur: Daten werden über einen Cluster verteilt. Die verwendete Methode ist RAID-0, RAID-1 oder RAID-5/6. Diese Lösung wendet SPBM (Storage Policy Based Management) an. Speicherressourcen werden dynamisch basierend auf der angeforderten Richtlinie und nicht im Voraus zugewiesen, wie bei vielen herkömmlichen Lösungen.

Dieser Test wird verwendet, um die relative Speicherleistung für verschiedene Cluster-Computing-Lasten anzuzeigen. YCSB ist eine NoSQL-Testplattform. Der Test enthält verschiedene Ladevorgänge zum Ausführen von Lese-, Einfüge-, Aktualisierungs-, Lösch- und Scanvorgängen, mit denen die Last beim Arbeiten mit NoSQL-Datenbanken und die Leistung von E / A-Systemen bewertet werden können. VMware Cloud Foundation


VMware Cloud Foundation (VCF) ist eine einzelne Hybridplattform für die Bereitstellung von VMware SDDC als private Cloud und die Integration in öffentliche Clouds. Diese Lösung bietet softwaredefinierte Dienste für die Verwaltung von Computern, Speichern, Netzwerken und der Cloud für eine Vielzahl von Workloads. Es vereinfacht die Installation, das Upgrade und die SDDC-Komponenten durch die Verwaltung ihres Lebenszyklus.

Automatisierungsfunktionen werden von der neuen VMware SDDC Manager-Komponente bereitgestellt. Die SDDC Manager-Software automatisiert die Installation und das Lebenszyklusmanagement von vSphere, vSAN und NSX - vom Start und der Konfiguration bis zum Upgrade sowie die Installation und Konfiguration von vRealize Log Insight, vRealize Operations und vRealize Automation.

Die Cloud Foundation kombiniert VMware vSphere (Computerressourcen), vSAN (Speicher) und NSX-Virtualisierung (Netzwerk) in einem integrierten Standardpaket. Cloud Foundation kann in einer lokalen privaten Cloud bereitgestellt oder als öffentlicher Cloud-Dienst verwendet werden.
VCF wird entweder als integrierter Hardware-Stack oder als Softwarepaket für die Installation auf Kundengeräten geliefert. VCF unterstützt die Bereitstellung von SDDC-Komponenten auf einer Vielzahl von physischen Servern (vSAN Ready Nodes) und Netzwerk-Switches und schafft so eine flexible heterogene Infrastruktur zur Unterstützung einer Vielzahl von Workloads. Insbesondere kann VCF auf Lenovo ThinkAgile VX-zertifizierten Standorten installiert werden.

Die Lösung wird vom Anbieter getestet, validiert und unterstützt. Kunden erhalten alle notwendigen Komponenten „aus einem Fenster“, was den Bestell- und Supportprozess erleichtert. Vorgefertigte Referenzarchitekturen vereinfachen die Implementierung und den Betrieb solcher Komplexe erheblich.
