スタイリッシュなアバロニア

デフォルトでは、wpf、Avaloniaのスタイルは非常に中立的であり、必要なタスクに常に適しているわけではありません。この記事では、スタイルを操作するための基本について説明し、いくつかの興味深い例を示します。



Avaloniaフレームワークにスタイルが必要なのはなぜですか？

  1. スタイルは、要素をカスタマイズするときの自由度を高めます。
  2. 要素の説明を重複させないでください。
  3. マークアップ自体とそのデザインを分離します。

いくつかの例


スタイルの理解を深めるために、まず例を見てみましょう：

丸いボタン（gnome ubuntuに似たもの）
これを行うには、パネルスタックからシンプルなレイアウトを作成し、標準テンプレートのボタンをいくつか作成します（テンプレートの作成方法）。

  <StackPanel>
        <Button Margin="20" Content="Btn" />
        <Button Margin="20" Content="Style Btn" />
    </StackPanel>

これで、ペンを使用してボタンをメインの背景とフォントの色に設定できますが、コードを複製する必要があります。スタイルは、この繰り返しを避けるのに役立ちます。

ウィンドウに適切なスタイルを追加します。

    <Window.Styles>
        <Style Selector="Button">
            <Setter Property="Foreground" Value="#FFFFFFFF" />
            <Setter Property="BorderThickness" Value="2" />
            <Setter Property="Background" Value="#FFDD4812" />
            <Setter Property="BorderBrush" Value="#FFFFFFFF" />
        </Style>
    </Window.Styles>



似たようなものが得られますが、縁を丸くするといいでしょう。残念ながら、どのように試しても、ボタンの端を丸めることは不可能ですが、ボタンを表示するテンプレートの端を丸めることができます。これを行うには、別のスタイルを追加します。

<Style Selector="Button /template/ ContentPresenter">
            <Setter Property="CornerRadius" Value="10" />
        </Style>



ほぼ同じように見えますが、問題があります。ボタンにカーソルを合わせると、灰色のフレームが表示されます（標準のスタイルからこんにちは）。



これを行うには、css：pointerover 疑似クラスに移動し、別のスタイルを追加します。

  <Style Selector="Button:pointerover /template/ ContentPresenter">
            <Setter Property="BorderBrush" Value="#FFDD4812" />
        </Style>



良くなったね。

しかし、特定のボタンだけにスタイルを適用したい場合はどうなりますか？

テンプレートに別のボタンを追加し、クラス属性を「装飾」するボタンに書き込みます。

 <StackPanel>
        <Button Margin="20" Content="Btn" />
        <Button Margin="20" Classes="btn" Content="Style Btn" />
        <Button Margin="20" Classes="btn" Content="Style Btn" />
    </StackPanel>

また、特定のクラスのみに影響を与える必要があることをスタイルに示します。

<Style Selector="Button.btn /template/ ContentPresenter">
            <Setter Property="CornerRadius" Value="10" />
        </Style>
        <Style Selector="Button.btn">
            <Setter Property="Foreground" Value="#FFFFFFFF" />
            <Setter Property="BorderThickness" Value="2" />
            <Setter Property="Background" Value="#FFDD4812" />
            <Setter Property="BorderBrush" Value="#FFFFFFFF" />
        </Style>
        <Style Selector="Button:pointerover.btn /template/ ContentPresenter">
            <Setter Property="BorderBrush" Value="#FFDD4812" />
        </Style>

スタイルを使用する場所


スタイルはウィンドウ内（上記の例で行ったように）
とアプリケーション全体（スタイルはすべてのウィンドウに適用されます）の両方で使用できます。

これを行うには、スタイルをApp.xamlファイルに追加します。

 <Application.Styles>
        <StyleInclude Source="avares://Avalonia.Themes.Default/DefaultTheme.xaml"/>
        <StyleInclude Source="avares://Avalonia.Themes.Default/Accents/BaseLight.xaml"/>
        <Style Selector="Button">
            <Setter Property="Background" Value="Blue"></Setter>
        </Style>
    </Application.Styles>

より大きな分離


コードが複雑になるため、スタイルをウィンドウレイアウト内に保持する必要はなくなりました。代わりに



、スタイルを別のファイルに配置します。これを行うには、Stylesディレクトリを作成し、そこにスタイルをxmlまたはxamlファイルの形式で配置し、名前空間の方向を追加します。

<Styles
        xmlns="https://github.com/avaloniaui"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:sys="clr-namespace:System;assembly=netstandard">

    <Style Selector="Button /template/ ContentPresenter">
        <Setter Property="CornerRadius" Value="10" />
    </Style>
    <Style Selector="Button">
        <Setter Property="Foreground" Value="#FFFFFFFF" />
        <Setter Property="BorderThickness" Value="2" />
        <Setter Property="Background" Value="#FFDD4812" />
        <Setter Property="BorderBrush" Value="#FFFFFFFF" />
    </Style>
    <Style Selector="Button:pointerover /template/ ContentPresenter">
        <Setter Property="BorderBrush" Value="#FFDD4812" />
    </Style>
</Styles>

新しいリソースをプロジェクトファイル（* .csproj）に追加します。

 <AvaloniaResource Include="Styles\**" />

そして、StyleIncludeタグを使用してそれらを接続します

<Window.Styles>
      <StyleInclude Source="/Styles/Style.xml"/>
    </Window.Styles>

しかし、これだけでは不十分な場合があり、スタイルを別のdllに入れて、すべてのプロジェクトで使用したいと考えています。

これを行うには、ClassLibraryを作成し、リソースavaloniyaとして将来のファイルをサポートします。

  <ItemGroup>
        <AvaloniaResource Include="**\*.xaml">
            <SubType>Designer</SubType>
        </AvaloniaResource>
    </ItemGroup>
    <ItemGroup>
        <AvaloniaResource Include="**\*.xml">
            <SubType>Designer</SubType>
        </AvaloniaResource>
    </ItemGroup>

ここで、すでに知っているStyles.xmlと2つの新しいファイルをプロジェクトに追加します。xaml



ファイルで、名前空間とそれが属するクラス、および必要なリソースの接続を指定する必要があります。

<Styles xmlns="https://github.com/avaloniaui"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        x:Class="ClassLibrary1.DarkCustom">    
        <StyleInclude Source="avares://ClassLibrary1/Style.xml"/>
</Styles>

そして、csファイルで、これがスタイルであり、これ以上ではないことを定義します。

namespace ClassLibrary1
{
    public class DarkCustom:Styles
    {
        
    }
}

私たちのライブラリに依存関係を追加し、スタイルを再接続し、使用してavaloniaリソースとしてスタイルを指定するStyleIncludeをし、

    <Window.Styles>
        <StyleInclude Source="avares://ClassLibrary1/DarkCustom.xaml"/>
    </Window.Styles>
