ビットコインブロックチェーンでのデータの書き込みと読み取り

金融取引だけでなく、事実上あらゆるデータのビットコインのブロックチェーンに書き込むことが可能です。ビットコインブロックチェーンに記録されたデータは公開され、それを削除または変更することはほとんど不可能です。この機能を使用して、たとえば、ブロックを適用できない分散サイトを作成できます。

blockchaindata-libライブラリを使用すると、ブロックチェーンでのデータの書き込みと読み取りを数行のコードで構成できます。

ツール


blockchaindata-libライブラリはnode.jsで記述されています。サンプル付きのソースコードはgithub.com/3s3s/blockchaindata-lib githubで入手できます。プロジェクトにライブラリを追加するには、コンソールで1つのコマンドを実行するだけです

npm install blockchaindata-lib

ブロックチェーンを使用するには、ビットコインのウォレットまたはビットコインと互換性のあるフォーク（ライトコイン、ドージェコインなど）へのRPCアクセスを構成する必要があります。デフォルトでは、blockchaindata-libは次の詳細でRPCサーバーへのアクセスを試みます。

: http://127.0.0.1:18332
: rpc_btc_test
: rpc_btc_password_test

ライブラリ関数によりデフォルトを変更可能

updateNetwork(url, user, password)

したがって、ライブラリを操作するには、最初にビットコインテストウォレットを実行する必要があります。

  1. ビットコインのウォレット（または互換性のあるフォーク）をダウンロードします。
  2. 〜/ .bitcoin / bitcoin.confファイルを作成します（またはwindows％APPDATA％/ Bitcoin / bitcoin.confに）
  3. bitcoin.confファイルに、次の設定を書き込みます

    
testnet=1
server=1
rpcbind=127.0.0.1
rpcallowip=127.0.0.1
rpcuser=rpc_btc_test
rpcpassword=rpc_btc_password_test
txindex=1

  4. 変更をbitcoin.confに保存し、bitcoindを実行します（またはWindowsのbitcoin-qt.exeで）。
  5. ブロックチェーンの同期を待つ

上記の設定で、ウォレットはビットコインテストネットワークで起動します。

データの記録


ビットコインは費用のかからない取引を禁止しています。したがって、ビットコインブロックチェーンに何かを書き込む前に、必ずウォレットを補充する必要があります。「ビットコインテストネットの蛇口」などをGoogleで検索すると、テストネットワーク用のビットコインを無料で入手できます。ウォレットが補充されると、ようやくデータを記録できます。

ライブラリには、データを記録するためのいくつかの関数があります。

SaveTextToBlockchain( dataString )
SaveJSONToBlockchain( objectJSON )
SaveFileToBlockchain( data )

原則として、これらの関数の機能は名前から明らかです。関数結果-オブジェクト
{結果：<true | false>、メッセージ：<string>、txid：<string>}


使用例を考えてみましょう：

'use strict';

const blockchaindata = require('blockchaindata-lib')

async function test1()
{
    try {
        //           
        const ret1 = await blockchaindata.SaveTextToBlockchain("     ");
        if (ret1.result == false) throw new Error("SaveTextToBlockchain failed, message: "+ret1.message);

        console.log("SaveTextToBlockchain success! txid="+ret1.txid+"\n--------------------------")
    }
    catch (e) {
        console.log(e.message)
    }
}

test1();

このコードがエラーなしで実行されると、トランザクションハッシュがコンソールに表示されます。このハッシュからデータを読み取ることができます。

データの読み取り


ブロックチェーンの初期に記録されたデータを読み取るには、トランザクションのハッシュを知るだけで済みます。ウォレットが空の場合があります。つまり、データを読み取るだけの場合は、ウォレットを補充する必要はありません。

コード例


'use strict';

const blockchaindata = require('blockchaindata-lib');

async function test2()
{
    //   
    try {
        const savedObject = await blockchaindata.GetObjectFromBlockchain("8af6633160b982a0b0b4d4962ad28e0d5b3dd97e05e27cc2dd64ec0c56820df5");
        if (savedObject.type == 'error') throw new Error(savedObject.message)
            
        if (savedObject.type == 'text')
            console.log(Buffer.from(savedObject.base64, 'base64').toString('utf8'));
        else
            console.log(savedObject.base64);
    }
    catch(e) {
        console.log(e.message)
    }
}

test2();

このコードがエラーなしで機能する場合、以前にブロックチェーンに保存したページのHTMLテキストがコンソールに表示されます。

どこに申請しますか？


たとえば、ブロックチェーンでは、静的サイトを保存できます。このサイトを削除したり、どういうわけかブロックすることは事実上不可能です。ただし、そのようなサイトを読むには、追加のソフトウェアが必要になります。新しいブラウザーまたはブラウザー拡張機能です。

ブラウザ拡張機能が最も簡単なソリューションです。そのような拡張の例はここで見ることができgithub.com/3s3s/blockchaindata

あなたはすることができ、ここで、Firefox用のこの拡張機能をダウンロードしてインストールします。yadi.sk/d/a3xM9BCepP4nBw
インストール後、Firefoxが傍受し、形式のサイトへのプロセスへのリンク
http：// tbtc / <txid>


たとえば、このようなリンクはhttpで機能します。

効率


blockchaindata-libライブラリは、deflateアルゴリズムを使用してデータをブロックチェーンに配置する前にデータを圧縮するため、この方法はテキストドキュメントの書き込みに特に効果的です。テスト中に、最大70 kbの圧縮データをブロックチェーンに書き込むことができました。着信データのサイズに制限はありませんが、ビットコインコードの内部実装が原因で、大きなデータセットでエラーが発生します。

データをブロックチェーンに書き込むとき、財務コンポーネントも考慮する必要があります。blockchaindata-libライブラリを使用したビットコインテストネットワークへのデータの読み取りと書き込みは、シンプルで無料です。ただし、ビットコインの有効なネットワークにデータを書き込む場合は、ビットコインのゼロ以外の価格を考慮する必要があります。
