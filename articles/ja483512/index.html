さまざまなバージョンのpythonをサポートする完璧なpypiパッケージの開発

これは、Pythonの「完璧な」pypiパッケージを作成する方法に関する小さなマニュアル/ストーリーであり、誰もが欲するコマンドをインストールできます。


pip install my-perfect-package

初心者向けですが、「パーフェクト」なパッケージを改善する方法について、専門家に意見を述べてほしい。だから猫をお願いします。


「理想的な」パッケージとはどういう意味ですか？


次の要件から始めます。


  • github上のオープンソース。
    誰もが開発に貢献でき、作者に感謝できるはずです。
  • 現在/人気のあるすべてのバージョンのPython（2.7、3.5、3.6、3.7、3.8）のサポート。
    Pythonは異なり、どこかでまだ2.7を積極的に書いています。
  • 単体テストで100％カバー。
    単体テストはアーキテクチャを改善し、回帰チェックを自動化します。
    大切な番号が付いたバッジは、FAQを上げ、他の人のための基準を設定します。
  • CIの使用：
    自動チェック-とても便利です！そして、たくさんのかっこいいバッジ
    • すべてのプラットフォームとPythonのすべてのバージョンで単体テストを実行します。いつでもバグに遭遇する可能性があるため
      、pythonとインストールされたパッケージはクロスプラットフォームであると主張する人を信用しないでください。
