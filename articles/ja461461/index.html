バックアップはクラウド時代に成功していますが、テープリールを忘れることはありません。Veeamとの会話



Alexander Baranovは、VeeamでR＆Dディレクターとして働いており、2つの国の間に住んでいます。彼は彼の時間の半分をプラハで過ごし、残りの半分はサンクトペテルブルクで過ごします。これらの都市には最大のVeeam開発オフィスがあります。

2006年には、仮想マシンをバックアップするためのソフトウェアに関連するロシアからの2人の起業家のスタートアップでした（仮想マシンのV [ee] [a] Mという名前は、同じ場所から来ました）。今日、それは世界中に4000人以上の従業員を持つ巨大な企業です。

アレクサンダーは、そのような会社で働くのはどんな感じで、それに入るのがどれほど難しいかを教えてくれました。以下は彼の独白です。
伝統的に、マイサークルでの会社の評価についてお知らせします。VeeamSoftwareは、従業員から平均4.4の評価を受けました。彼は、優れた社交パッケージ、チーム内の快適な作業環境、興味深いタスク、そして会社が世界をより良い場所にするということで高く評価されています。





Veeamが開発する製品


ITインフラストラクチャの復元力を提供する製品。幸い、時間の経過とともに鉄は非常に信頼性が高くなり、クラウドはフォールトトレランスを提供します。しかし、人々の過ちはまだ存在しています。

たとえば、組織のインフラストラクチャと更新の互換性がないという古典的な問題です。管理者が未確認の更新をロールアウトしたか、更新が自動的に行われたため、エンタープライズサーバーの操作が中断されました。別の例：誰かが、適切であると見なして、共通のプロジェクトまたは一連のドキュメントに変更を加えた。その後、問題が発見され、1週間前の状態に戻す必要がありました。時々、そのような変化は意識的な人間の行動とさえ関連付けられていません：cryptolockerウイルスは最近人気を得ています。ユーザーは、疑わしいコンテンツを含むフラッシュドライブを動作中のコンピューターに持ってくるか、シールを貼ったWebサイトにアクセスし、その結果、ネットワーク上のコンピューターが感染します。

すでに悪が起こっている状況では、変更をロールバックする機会を与えます。変更のみが計画されている場合は、データセンターのバックアップから再作成された、隔離されたインフラストラクチャでの影響を確認できます。

多くの場合、バックアップは組織の監査中に「サイレント証人」の役割を果たします。公開企業は外部規制機関の要件（たとえば、サーベンスオクスリー法）に準拠する必要がありますが、これは偶然ではありません。 2008年、金融市場の一部の参加者が大まかに言って、彼らの活動の結果を偽造したという事実により、世界経済の状態は揺さぶられました。これは雪だるま式に始まり、経済は落ち込んだ。その瞬間から、規制当局は上場企業のプロセスを綿密に監視してきました。 ITインフラストラクチャ、メールシステム、レポート期間のドキュメント管理システムの状態を復元する機能は、監査人の要件の1つです。

Microsoft、Amazon、Google、その他のクラウドプロバイダーには、クラウド内のリソースをバックアップするネイティブソリューションがあります。しかし、彼らの決定は「それ自体のもの」です。問題は、ほとんどの場合、大企業がハイブリッドITインフラストラクチャを持っていることです。その一部はクラウドにあり、一部は地上にあります。クライアント指向のWebプロジェクトおよびアプリケーションは、通常、クラウドにあります。機密情報や個人データを保存するアプリケーションやサーバーは、ほとんどの場合、地上に配置されています。

さらに、組織は複数の異なるクラウドを使用して1つのハイブリッドを構築し、リスクを最小限に抑えます。多国籍企業がハイブリッドクラウドを構築した場合、インフラストラクチャ全体に対して単一の共通のフォールトトレランスシステムが必要です。




そのような製品を開発するのはどれほど難しいか


研究、適応、経験を必要とする新しいテクノロジーが常に出現しています。私たちが最初に現れてスタートアップだったとき、仮想化を真剣に考えている人はほとんどいませんでした。物理的なデータセンターをバックアップするためのアプリケーションがありました。仮想化されたデータセンターはおもちゃとして見られました。

仮想化を念頭に置いてバックアップをサポートし始めたのは、そのテクノロジーがマニアのみに使用された当初からです。そして、その爆発的な成長と規格による認識が起こりました。これで、同じ質的な飛躍を待っている他の領域が見えてきて、波に乗ろうとします。鼻を風下に保つ能力は、会社のDNAのどこかに縫い付けられています。

現在、同社はすでにスタートアップの時代を生き抜いています。現在、安定性と信頼性は多くの大規模な顧客にとって重要であり、フォールトトレランスの決定は数年続く可能性があります。適応、製品検証、多数の要件への準拠。おかしな状況が判明しました。一方では製品の信頼性と信頼性を確保し、もう一方では最新の状態を維持する必要があります。

しかし、新しいものは常に、技術、市場、またはその両方に対する一定レベルの無知に関連しています。

たとえば、数年の作業の後、バックアップを高速化するにはストレージシステムの組み込み機能を使用する必要があることに気付きました。したがって、鉄の生産者との統合のための全体的な方向性が生まれました。今日、このプログラムにおけるVeeamのパートナーはすべて、この市場で最大のプレーヤーです-HP、NetApp、Dell EMC、Fujitsuなど。

仮想化が従来のサーバーに取って代わるようにも思えました。しかし、生命は、物理サーバーの最後の10％が残っていることを示しました。その仮想化は不可能であるか、または意味がありません。また、それらもバックアップする必要があります。これが、Veeam Agent for Windows / Linuxの登場です。

かつて、私たちはUnixが博物館に登場する時がきたと感じ、それをサポートすることを拒否しました。しかし、長い歴史を持つ顧客に到達するとすぐに、Unixはすべての生きているものよりも生きていることに気づきました。それにもかかわらず、彼らは彼のために解決策を書きました。

同じことがテープドライブにも当てはまります。私たちは「現代の世界で誰がそれらを必要としているのか」と考えました。次に、きめ細かなデータリカバリや合成フルコピーを使用した増分バックアップなどの機能に取り組みました。これはテープでは実行できず、ディスクが必要です。その後、テープドライブは長期保存に必要な未変更のバックアップを提供する手段の1つとして機能することがわかりました。5年後には、テープをシェルフから取り出して監査を行います。さて、顧客のサイズ-私たちは小さなものから始めました-そして誰もそこでテープを使用していません。そして、テープなしでは製品を購入しないと私たちに言った顧客に育ちました。




Veeamで使用されているテクノロジー


ビジネスロジックに関連するタスクには、.NETを使用します。私たちはそれから始め、最適化を続けています。現在、多くのソリューションで.NET Coreを使用しています。スタートアップが設立されたばかりのとき、チームにはこのスタックの支持者が何人かいました。彼はビジネスロジック、開発速度、およびツールの利便性を書くのが得意です。その後、それは最も人気のあるソリューションではありませんでしたが、今ではそれらの支持者が正しかったことは明らかです。

同時に、Unix、Linux、ハードウェアで作業するため、他のソリューションを使用する必要があります。バックアップに保存するデータに関する情報に関連するシステムパーツ、データ取得アルゴリズム、アイロンの操作に関連するアルゴリズム-これらはすべてC ++で記述されています。




従業員が世界中にどのように分布しているか


現在、同社は約4000人を雇用しています。彼らの約千人がロシアにいます。会社には2つの大きなグループがあります。 1つ目は、製品の開発と技術サポートです。 2つ目は、製品を外の世界に見えるようにすることです。その能力には、販売とマーケティングが含まれます。グループ間の比率は約30から70です。

世界中に約30のオフィスがあります。販売はより広範囲に分配されますが、開発もそれほど遅れていません。一部の製品は、一部はサンクトペテルブルク、一部はプラハの複数のオフィスで同時に処理されています。一部は1つだけで開発されています。たとえば、物理的なLinuxバックアップを提供する製品はプラハで開発されています。カナダだけが取り組んでいる製品があります。

お客様のご要望にお応えするため、分散開発を行っています。製品が機能するのと同じ地域で開発が行われると、大規模な顧客はより安全だと感じます。

チェコにはすでに非常に大きなオフィスがあり、来年はプラハに500の開発者とテスターのために別のオフィスを開設する予定です。ハブレの「最初の波」でチェコの首都に引っ越した人は、ヨーロッパで働く機会に興味があるすべての人と経験とライフハックを共有できてうれしいです。ロシアでは、オフィスはサンクトペテルブルクにあり、内部プロジェクトの一部はイジェフスクで進行中で、サポートは部分的にモスクワで行われています。一般に、世界中の数百人が技術サポートに従事しています。さまざまなレベルの技術トレーニングと専門分野のスペシャリストがいます。最高レベルは、ソースコードレベルで製品を理解できる人であり、開発と同じオフィスで作業します。




プロセスの配置方法


年に1回、新しい機能を備えた大規模なリリースをリリースし、2〜3か月ごとに、緊急の市場要件またはプラットフォームの変更に対応するバグ修正と改善を含むアップデートをリリースします。優先度は要件に割り当てられます-マイナーからクリティカルまで、リリースなしでは不可能です。後者は「エピック」と呼ばれます。

古典的な三角形があります-品質、リソースの量、用語（一般的な人々では、「迅速に、効率的に、安価に、2つを選択してください」）。私たちは不十分なことはできません。品質は常に高くなければなりません。リソースも限られていますが、常に拡大を図っています。時間管理にははるかに柔軟性がありますが、多くの場合、修正されます。したがって、変更できる唯一のことは、リリースの機能の量です。

エピコフは、原則として、予測されるリリースサイクルの日付の30〜40％を超えないようにします。残りを切り取り、転送し、修正し、修正することができます。これが私たちの操縦スペースです。

要件ごとに、リリースに一時的なチームが作成されます。複雑さにもよりますが、3人で50人です。私たちは柔軟な開発方法論を順守し、週に1回、各機能で実行され、近々行われる作業のレビューとディスカッションを組織します。

リリースサイクルタイムの半分は開発に費やされ、半分は製品の完成に費やされます。しかし、「破産プロジェクトの技術的負債はゼロである」という格言があります。したがって、コードを際限なくなめるよりも、機能していて需要のある製品を作ることが重要です。製品が人気のある場合、それはすでにそれをさらに開発し、将来の変更に適応させる価値があります。




Veeamが従業員を雇用する方法


選択アルゴリズムは多段階です。最初のレベルは、候補者と採用担当者との間の、本人の希望についての会話です。この段階では、候補者に適しているかどうかを理解しようとしています。プロジェクトに人を紹介することは非常に高い喜びであるため、私たちが会社として興味深いことは私たちにとって重要です。

興味がある場合は、第2レベルで、候補者の経験がどの程度関連しているか、および候補者が専門家として何を実証できるかを理解するためのテストタスクを提供します。例えば、ファイルコンプレッサーを作ってください。これは標準的なタスクであり、ソリューションを使用する文化とスタイルに準拠するコードと人との関係を示しています。

テストタスクでは、通常、すべてが完全に表示されます。手紙を知ったばかりで、最初に手紙を書いた人は、常に手紙を書く人とは著しく異なります。

次に面接を行います。通常は、すべてができるだけ客観的になるように、3人のチームリーダーがすぐに実行します。さらに、最終的に異なるチームで作業している場合でも、ほぼ同じ開発方法とアプローチを持つ技術的に互換性のある人材を採用するのに役立ちます。

平日は、空席のある面接を数回実施し、今後誰と協力するかを決定します。

多くの場合、彼らは私たちのところにやって来て、彼らが仕事を探していると言っています。なぜなら、彼らは現在の仕事に行く場所がないからです。ボスが引退するときだけ、増加を待つことができます。ダイナミクスは少し異なります。 12年前、Veeamは従業員10人のスタートアップでした。今では数千人が働く会社です。

人々は荒れ狂う川のようにここに行きます。新しい方向性が常に現れています;昨日のランクアンドファイルデベロッパーはチームリーダーになります。人々は技術的に成長し、管理的に成長します。小さな機能を開発しているが、それを開発したい場合は、戦いの半分はすでに完了しています。サポートは、チームリーダーから会社のオーナーまで、すべてのレベルで行われます。管理上何かを行う方法がわからない-コース、内部トレーナー、経験豊富な同僚がいます。開発における十分な経験はありません-Veeam Academyプロジェクトがあります。ですから、私たちはプロと初心者の両方に誰でも参加できます。
