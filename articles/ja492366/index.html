JavaScriptコンソールの使用方法：console.log（）を超えて

こんにちは、ハブル！Yash Agrawalによる「JavaScriptコンソールの使用方法：console.log（）を超えて」という記事の翻訳を紹介します。

JavaScriptで何かをデバッグする最も簡単な方法の1つは、console.logを使用して資料を出力することです。しかし、デバッグの改善に役立つコンソールによって提供される他の多くのメソッドがあります。

はじめましょう。

最も基本的な使用例は、JavaScriptオブジェクトの文字列またはグループを出力することです。ものすごく単純

console.log('Is this working?');

次に、コンソールに出力する必要のあるオブジェクトの束があるシナリオを想像してみてください。

const foo = { id: 1, verified: true, color: 'green};
const bar = { id: 2, verified: false, color: 'red' };

最も直感的な出力方法は、単にconsole.log（変数）を数回実行することです。コンソールでどのように表示されるかを確認すると、問題がより明らかになります。



ご覧のとおり、変数名は表示されないため、あまり便利ではありません。これは、すべての変数を1つのconsole.log（{foo、bar}）に結合することで解決できます。また、コード内のconsole.log行の数を減らします。

console.table（）


これをすべてテーブルに入れて読みやすくすることで、次のステップに進むことができます。共通のプロパティを持つオブジェクトまたはオブジェクトの配列がある場合は常に、console.table（）を使用します。ここでは、console.table（{foo、bar}）を使用でき、コンソールに次のように表示されます。



console.group（）


ログを簡単に読み取れるように、対応する要素をグループ化またはネストする場合に使用できます。

関数内に複数のログステートメントがあり、各ステートメントに対応するスコープを明確に表示したい場合にも使用できます。

たとえば、ユーザーデータを登録すると、次のようになります。


console.group('User Details');
    console.log('name: John Doe');
    console.log('job: Software Developer');

    console.group('Address');
        console.log('Street: 123 Townsend Street');
        console.log('City: San Francisco');
        console.log('State: CA');
    console.groupEnd();
console.groupEnd();



グループをデフォルトで折りたたむ場合は、console.group（）の代わりにconsole.groupCollapsed（）を使用することもできます。展開するには、左側のハンドルボタンをクリックする必要があります。

console.warn（）およびconsole.error（）


状況に応じて、コンソールを読みやすくするために、console.warn（）またはconsole.error（）を使用してログを追加できます。一部のブラウザでは「i」アイコンを表示するconsole.info（）もあります。



独自のスタイルを追加することで、さらに先へ進むことができます。％cディレクティブを使用して、任意のログステートメントにスタイルを追加できます。これは、API呼び出し、ユーザーイベントなどを区別するために使用できます。

console.log('%c Auth ‘, 'color: white; background-color: #2274A5’, 'Login page rendered');
console.log('%c GraphQL ','color: white; background-color: #95B46A', 'Get user details');
console.log('%c Error ',  'color: white; background-color: #D33F49',  'Error getting user details’);

フォントサイズ、フォントスタイル、その他のCSS要素を変更することもできます。



console.trace（）


console.trace（）は、スタックトレースをコンソールに出力し、特定の時点でコードがどのように終了したかを示します。データベースからの削除など、一度だけ呼び出したい特定のメソッドがあります。console.trace（）を使用して、コードが意図したとおりに動作することを確認できます。

console.time（）


フロントエンド開発に関してもう1つ重要なことは、コードが高速であることです。console.time（）を使用すると、テストのためにコード内の特定の操作を同期できます。

let i = 0;
console.time("While loop");
while (i < 1000000) { i++;}console.timeEnd("While loop");
console.time("For loop");
for (i = 0; i < 1000000; i++) { // For Loop}console.timeEnd("For loop");
