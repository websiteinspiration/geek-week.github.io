Generic Recycler Viewまたは定型コードを作成しない方法

私たちはみんなアプリケーションを書き、みんなリストを持っています。そして最も明白な解決策はRecyclerViewです。実装自体は複雑ではなく、RecyclerViewでガイドを書くことはもはや関係ありません。しかし、1つあります。リストが必要になるたびに、クラスを作成し、その中にテンプレートメソッドを規定し、テンプレートクラスを作成します。 2つから3つのリストがある場合、問題はありません。ただし、それらが10個以上ある場合は、これを実行する必要はありません。

そして、問題に直面して、私は捜索を始めました。 Kotlinで非常に興味深い実装が1つ見つかりました。気に入ったのですが、いくつかの要素が欠けていました。さらに数時間を費やした後、それを調整することができ、アダプターの実装には数行かかります。そしてここでそれをあなたと共有したいと思います。

まず、アダプタを作成する必要があります。

abstract class GenericAdapter<T> : RecyclerView.Adapter<RecyclerView.ViewHolder> {
    private var itemList = mutableListOf<T>()

    constructor()

    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): RecyclerView.ViewHolder {
        return getViewHolder(
            LayoutInflater.from(parent.context)
                .inflate(viewType, parent, false)
            , viewType
        )
    }

    override fun onBindViewHolder(holder: RecyclerView.ViewHolder, position: Int) {
        (holder as Binder<T>).bind(itemList[position], itemClickListener)
    }

    override fun getItemCount(): Int = itemList.size

    override fun getItemViewType(position: Int): Int = getLayoutId(position, itemList[position])

    fun update(items: List<T>) {
            itemList = items.toMutableList()
            notifyDataSetChanged()
    }

    protected abstract fun getLayoutId(position: Int, obj: T): Int

   protected open fun getViewHolder(view: View, viewType: Int): RecyclerView.ViewHolder {
        return ViewHolderFactory.create(view, viewType)
    }

    internal interface Binder<T> {
        fun bind(data: T)
    }
}

ここで何が起こっているのですか？パラメーター化されたアダプターを作成し、その中で基本的なテンプレートメソッドを再定義します。ViewHolderが実装する必要があるパラメーター化されたBinderインターフェースを作成します。抽象getLayoutId（）メソッドで、レイアウトを設定します。

ViewHolderのファクトリを作成した後。

object ViewHolderFactory {
    fun create(view: View, viewType: Int): RecyclerView.ViewHolder {
        return when (viewType) {
            R.layout.item_data -> DataViewHolder(view)
            R.layout.item_other_data -> OtherDataViewHolder(view)
            else -> throw Exception("Wrong view type")
        }
    }

class DataViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView),

すべてがクールで便利、高速です。ほぼこの形で、私はこの実装を見つけました。しかし、私は、クリック可能な要素についてはどうだと思いました。そして、これが私の決断です。

まず、インターフェースを作成します

interface OnItemClickListener<T> {
    fun onClickItem(data: T)
}

そしてそれを私たちのバインダーインターフェースに渡します
internal interface Binder<T> {

そしてアダプターで、追加のコンストラクターを作成します。

 private var itemClickListener: OnItemClickListener<T>? = null

constructor(listener: OnItemClickListener<T>) {
        itemClickListener = listener
}

class DataViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView),
        GenericAdapter.Binder<Data> {
        override fun bind(data: Data, listener: OnItemClickListener<Data>?) {
            itemView.apply {
                dateTextView.text = data.dateTitle
                setOnClickListener { listener?.onClickItem(data) }
            }
        }

結局のところ、3行で作成されたアダプターと、あらゆる種類の要素のためのユニバーサルインターフェイスです。クリックを処理する必要がない場合は、リスナーをコンストラクタに渡さないだけです。しかし、それだけではありません。

DiffUtils.Callbackをアダプターにバインドする場合はどうでしょうか。

class GenericDiffUtil<T>(
    private val oldItems: List<T>,
    private val newItems: List<T>,
    private val itemDiff: GenericItemDiff<T>
) :
    DiffUtil.Callback() {
    override fun getOldListSize() = oldItems.size
    override fun getNewListSize() = newItems.size
    override fun areItemsTheSame(oldItemPosition: Int, newItemPosition: Int) =
        itemDiff.isSame(oldItems, newItems, oldItemPosition, newItemPosition)

    override fun areContentsTheSame(oldItemPosition: Int, newItemPosition: Int) =
        itemDiff.isSameContent(oldItems, newItems, oldItemPosition, newItemPosition)
}

interface GenericItemDiff<T> {
    fun isSame(
        oldItems: List<T>,
        newItems: List<T>,
        oldItemPosition: Int,
        newItemPosition: Int
    ): Boolean

    fun isSameContent(
        oldItems: List<T>,
        newItems: List<T>,
        oldItemPosition: Int,
        newItemPosition: Int
    ): Boolean
}

これは、DiffUtilsの基本クラスです。アダプターにメソッドを追加する

アダプタメソッドの更新を少し変更します（）
 fun update(items: List<T>) {
        if (diffUtil != null) {
            val result = DiffUtil.calculateDiff(GenericDiffUtil(itemList, items, diffUtil!!))

            itemList.clear()
            itemList.addAll(items)

adapter.setDiffUtilCallback(dataDiffUtil)
private val dataDiffUtil = object : GenericItemDiff<Data> {
        override fun isSame(
            oldItems: List<Data>,
            newItems: List<Data>,
            oldItemPosition: Int,
            newItemPosition: Int
        ): Boolean {
            val oldData = oldItems[oldItemPosition]
            val newData = newItems[newItemPosition]
            return oldData.id == newData.id
        }

        override fun isSameContent(
            oldItems: List<Data>,
            newItems: List<Data>,
            oldItemPosition: Int,
            newItemPosition: Int
        ): Boolean {
            val oldData = oldItems[oldItemPosition]
            val newData = newItems[newItemPosition]
            return oldData.name == newData.name && oldData.content == newData.content
        }
