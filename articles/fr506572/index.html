Création d'un prototype de boutique en ligne SPA sur htmlix.js

Cet article décrira la création étape par étape de l'interface pour le prototype de boutique en ligne SPA. La partie serveur est écrite en node.js en utilisant express, le moteur de template est twig, la base de données est NeDB, le formidable module est utilisé pour traiter les données du formulaire, la partie client est écrite en htmlix.

Pour ceux qui ne connaissent pas Htmlix, lisez les bases de la création d'applications ici .

Les données source de cette leçon peuvent être téléchargées ici .

Ceci est un exemple prêt à l'emploi, nous supprimons simplement tout ce qui se trouve dans le fichier /static/js/front.js et l' insérons à la place:

En allant à l'adresse "localhost: 3000 /", on nous donne un menu supérieur avec trois éléments, une page principale avec six fiches produits et une liste de catégories à gauche.

Ensuite, créez un composant de carte de produit cart_single , mais avant de le créer, voyons comment fonctionne htmlix:

Ainsi, lors du chargement de la page à partir de l'adresse "/" ou "/ category /: idCategory", le serveur nous envoie le code html de deux composants, plus précisément trois, un autre "menu", mais nous ne l'utilisons pas encore, nous utilisons du code html pour les composants des chariots et catégories, sur la base de ce code, htmlix crée des modèles pour les composants, mais le modèle pour le composant cart_single ne nous est pas transmis à ces adresses, car il n'est pas là. Comment pouvons-nous le «charger»? Pour ce faire, nous créons le composant cart_single dans la description de l'application et le plaçons dans l'objet spécial "fetchComponents", et dans les paramètres `stateSettings.templatePath`, spécifions l'adresse où charger les modèles pour ce composant, et tous les autres qui apparaîtront. Ainsi, l'application créera d'abord les composants que le serveur a envoyés avec html,puis après avoir envoyé la deuxième demande à stateSettings.templatePath, il créera les composants restants situés dans l'objet fetchComponents.

Donc, nous le ferions si nous avions toujours accès aux applications à partir des adresses "/" ou "/ category /: idCategory", mais que se passe-t-il si nous chargeons d'abord l'application à partir de l'adresse "/ cart /: idCart"? L'application donnera une erreur indiquant qu'elle ne peut pas trouver de modèle pour le tableau de chariots, car maintenant elle ne sera pas à cette adresse et nous devons maintenant mettre des chariots dans l'objet "fetchComponents", et au contraire, obtenir single_cart à partir de là, car il doit être initialisé en premier avec le composant categories, que dois-je faire?

Pour résoudre ce problème, nous utiliserons HTMLixRouter (), avant de créer l'instance d'application, après avoir chargé la page, il vérifie quelle est l'adresse URL maintenant et la compare avec celles que nous lui indiquerons, puis il met celles pour lesquelles il n'y a pas de modèle lors du premier chargement, dans l'objet "fetchComponents", après quoi il initialise l'application, ainsi nous éviterons de créer plusieurs options pour décrire l'application pour différentes URL et, par conséquent, la duplication de code.
