Nodemailer

		text: 'hello crazy world',

Socket.io


Entwickelt, um eine ereignisbasierte Zwei-Wege-Verbindung in Echtzeit einzurichten.

	const server = require('http').createServer()
	const io = require('socket.io')(server)
	io.on('connection', client => {
		client.on('event', data => { ... })
		client.on('disconnect', () => { ... })
	})
	server.listen(3000)

Gut für

  • Implementierung von Live-Analysen, binärem Datenfluss, Messaging und Zusammenarbeit mit Dokumenten

Morgan


Es ist eine Middleware zum Registrieren von Anfragen.

	morgan
	(':method:url :status :res[content-length] - :response-time ms')

	---

	const express = require('express')

	const morgan = require('morgan')

	const app = express()

	app.use(morgan('combined'))

	app.get('/', (req, res) => res.send('hello crazy world'))

Gut für

  • Schreibanforderungen an die Konsole, Datei, Datenbank
  • Debuggen und "Protokollieren"

Körperparser


Es ist eine Middleware zum Parsen von Daten.

Analysiert eingehende Anforderungen (req.body), bevor sie verarbeitet werden.

	const express = require('express')
	const bodyParser = require('body-parser')

	const app = express()

	//  application/x-www-form-urlencoded
	app.use(bodyParser.urlencoded({ extended: false }))

	//  application/json
	app.use(bodyParser.json())

Gut für

  • Interpretationen des Abfragekörpers
  • enthält viele nützliche Funktionen (Aufblasen, Tippen, Überprüfen usw.)

Reisepass


Es ist eine Express-kompatible Middleware zur Authentifizierung.

Die Hauptaufgabe besteht darin, Anforderungen mithilfe einer Reihe von Plugins zu authentifizieren - Strategien.

	passport.use(new LocalStrategy(
		(username, password, done) => {
			User.findOne({ username: username }, (err, user) => {
				if(err) { return done(err) }

				if(!user) { return done(null, false) }

				if(!user.verifyPassword(password)) { return done(null, false) }

				return done(null, user)
			})
		}
	))

Gut für

  • Authentifizierung
  • Integration mit OAuth und OpenID (melden Sie sich mit Ihrem Facebook-Konto, Twitter usw. an)

810 Tausend

Dotenv


Es ist ein Modul ohne Abhängigkeit (Punkt) zum Laden von Variablen aus der .env-Datei in process.env.

Hält Variablen vom Code getrennt.

	require('dotenv').config()

	const db = require('db')

	db.connect({
		host: process.env.DB_HOST,
		username: process.env.DB_USER,
		password: process.env.DB_PASS
	})

	//  .env
	DB_HOST = localhost
	DB_USER = root
	DB_PASS=12345

Gut für

  • Laden von Umgebungsvariablen wie AWS, SQL-Benutzernamen, Kennwörtern, die zum Bereitstellen von Anwendungen benötigt werden, und Herstellen einer Verbindung zu anderen Tools
  • Speichern von Umgebungseinstellungen getrennt vom Code

Multer


Middleware zur Verarbeitung von Multipart- / Formulardaten - Laden von Bildern.

	const express = require('express')

	const multer = require('multer')

	const upload = multer({ desc: 'uploads/' })

	const app = express()

	app.post('/profile', upload.single('avatar'), (req, res, next) => {
		// req.file -   'avatar'
		// req.body     
	})

	app.post('/photos/upload', upload.array('photos', 12), (req, res, next) => {
		// req.files -    'photos'
		// req.body      
	})

Gut für

  • Schneller Download von mehrteiligen / Formulardatendateien

CORS


Connect / Express-Middleware zum Definieren von Richtlinienregeln für gemeinsamen Ursprung.

	const express = require('express')

	const cors = require('cors')

	const app = express()

	app.use(cors())

	app.get('/products/:id', (req, res, next) => {
		res.json({ msg: '    !' })
	})

	app.listen(80, () => {
		console.log('ok')
	})

Gut für

  • CORS-bezogene Aufgaben
