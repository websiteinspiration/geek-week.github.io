rpm-gpg-repository-mirroring-Nexusでyumプロキシを作成できないリポジトリからRPMをダウンロードするスクリプト

一部の組織では、サーバーはインターネットにアクセスできません。そのような場合、メインのリポジトリのミラーを作成します。







しかし、サーバーからのアクセスが制限されており、必要なrpmパッケージをインストールする必要がある場合はどうでしょうか。通常はreposyncを使用するか、手動でダウンロードしてローカルリポジトリを作成します。

プロキシサーバーを介してオンラインになるリポジトリを追加することもできます。プロキシサーバーは多くの作業を行うことができます。







しかし、rpm-gpg-repository-mirroringスクリプトを使用して、必要なrpmパッケージをダウンロードし、ローカルミラーを作成することができます。







この投稿の目的は、Grafana、Kubernetes、Gitlab、packagecloud.ioなどのリポジトリからrpmパッケージの最新バージョンをダウンロードするスクリプトについて説明することです。







reposyncよりもスクリプトを使用したrpm-gpg-repository-mirroringの利点は何ですか







  • 特定のバージョンから始まるすべてのrpmパッケージをダウンロードできます
  • 最新のNバージョンのrpmパッケージをダウンロードできます
  • reposyncと比較してトラフィックを大幅に節約
  • reposyncに比べてディスク容量を大幅に節約
  • 手で作成されたリポジトリとは異なり、最新バージョンをダウンロードし、rpmは手で追加されました


ソース



https://github.com/patsevanton/rpm-gpg-repository-mirroring







Selinuxをオフにする



sudo sed -i s/^SELINUX=.*$/SELINUX=disabled/ /etc/selinux/config
sudo reboot


rpm-gpg-repository-mirroringのインストールと実行（egin-releaseはnginxに必要です）



yum install -y epel-release
yum -y install yum-plugin-copr
yum copr enable antonpatsev/rpm-gpg-repository-mirroring
yum -y install rpm-gpg-repository-mirroring


rpm-gpg-repository-mirroringのセットアップ



ファイルの編集/etc/rpm-gpg-repository-mirroring.conf





すべてが詳細に説明されています。







特定のバージョン以降のすべての最新のrpmパッケージをダウンロードする必要があるリポジトリ。グラファナの例



以下の内容で/etc/yum.repos.d/grafana.repoを作成します：







[grafana]
name=grafana
baseurl=https://packages.grafana.com/oss/rpm
repo_gpgcheck=0
enabled=1
gpgcheck=0
gpgkey=https://packages.grafana.com/gpg.key
sslverify=1
sslcacert=/etc/pki/tls/certs/ca-bundle.crt


設定/etc/rpm-gpg-repository-mirroring.conf：







# ,     rpm 
DOWNLOAD_DIR=/var/www/repos

# , rpm       ,   
REPOS={"grafana":"6.5.3"}


スクリプトを実行する







rpm-gpg-repository-mirroring
