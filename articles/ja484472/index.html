2.3遅延

コルーチンで遅延を構成するには、2つのオプションがあります。より長い遅延の場合、および期間が正確である必要がない場合は、以下を使用できます。

      await  asyncio.sleep ( delay_secs )
      print ( 'Hello' )
      await  asyncio.sleep_ms ( delay_ms )

そのような遅延の間、スケジューラは他のコルーチンを実行します。現在実行中のコルーチンが実行されたときにのみ、呼び出しコルーチンが起動されるため、これにより時間の不確実性が生じる可能性があります。遅延の量はアプリケーション開発者によって異なりますが、おそらく数十または数百msのオーダーになります。これについては、ハードウェアデバイスとの相互作用（セクション6）でさらに説明します。

3.同期


多くの場合、コルーチン間の同期を確保する必要があります。一般的な例は、複数のコルーチンが同時に同じリソースにアクセスする必要がある場合に、いわゆる「競合状態」を回避することです。例はastests.pyで提供され、ドキュメントで説明されています。別の危険は、各コルーチンが他のコルーチンの完了を待つ「死の抱擁」です。

単純なアプリケーションでは、グローバルフラグまたは関連する変数を使用して同期を実現できます。より洗練されたアプローチは、同期クラスを使用することです。asyn.pyモジュール提供の「マイクロ」の実装、イベント、バリア、セマフォ、およびConditiosのクラスをのみでの使用を目的とし、asyncio。これらはスレッド指向ではないため、特に指定がない限り、 _threadモジュールまたは割り込みハンドラーと一緒に使用しないでください。 Lockクラスも実装されています。これは、公式の実装の代替です。

コルーチンのプロデューサーとコルーチンのコンシューマーで別の同期の問題が発生します。コルーチンプロデューサーは、コルーチンコンシューマーが使用するデータを生成します。これを行うために、 asyncioは Queueクラスを提供します。コルーチンプロデューサーはデータをキューに入れ、コルーチンコンシューマーはその完了を待ちます（他の操作は時間どおりにスケジュールされます）。クラスキュー受け取った順にアイテムを削除することを保証します。または、プロデューサーコルーチンがコンシューマーコルーチンがデータにアクセスする準備ができるまで待機する必要がある場合は、バリアクラスを使用できます。

Lockは、共有リソースへの一意のアクセスを保証します。次のコード例は、共有リソースにアクセスするすべてのクライアントに渡されるロッククラスLockのインスタンスを作成します。各コルーチンはロックを取得しようとし、成功するまで実行を一時停止します。

        print("Acquired lock in task", i)
        await asyncio.sleep(0.5)

    await asyncio.sleep(10)

lock = Lock()  # The global Lock instance

loop.create_task(task(1, lock))
loop.create_task(task(2, lock))
loop.create_task(task(3, lock))

loop.run_until_complete(killer())  #  10s

        return b'\n'
