純粋なPowerShellを使用してMinecraftの支払い受け入れシステムを作成した方法


この記事では、Powershellを使用して、神のないドーナツをMinecraftバニラサーバーにねじ込みます。この方法の利点は、Minecraftがコンソールコマンドを使用した自動支払いの実装の特殊なケースにすぎないことです。支払いシステムから送られるものに耳を傾け、それをチームにまとめます。そして最も重要なこと-プラグインなし。
そして、PayPalを通して支払いを受け入れます。最も重要なのは、コードを変更する必要のない支払いの受け入れを開始するために、PayPalが必要なものをすべて送信することです。サイトのボタンを使用して、サイトが純粋なHTMLで処理できるようにします。支払いシステム自体の複雑さから抽象化し、コードの主要な点のみに集中します。

ちなみに、作者は、すべてのモジュールを調べて、指摘したり修正したりしている子供のエラーを見つけたら、とても喜ばれます。ここにgithubプロジェクトへのリンクがあります。

IPNについて一言


IPN


お支払いはボタンから受け付けます。ボタンはユーザーからのバックエンドを必要とせず、純粋なHTMLで動作し、独自のフィールドも持っています。

ボタンは、IPN-即時支払い通知をトリガーします。この通知では、データがWebListenerに送信されます。以下のIPN構造を検討します。

さらに、PayPalアカウントを持っている人なら誰でも自分のボタンを作成できます。
IPNには完全なPayPal REST APIはありませんが、基本的な機能を実装できます。実際、私たちが検討しているIPNは、PayPal自体がコード200以外は何も期待していないからといって、完全な意味でのREST APIではありません。

Minecraft Nuances


そこで、支払いに関するアラートを受け取る方法を理解しました。これで、クレジットにクレジットを付けることができます。しかし、ここもそれほど単純ではありません。問題は、ゲームがアイテムを提供しないか、サーバー上にないプレイヤーのステータスを変更しないことです。つまり、誰かがサーバーにアクセスして、彼が支払った金額を提供するまで待つ必要があります。

したがって、あなたの注意は、支払いをクレジットするための喫煙者の一般的な概念に提示されます。



支払いは上記のリスナーを介して受信され、オブジェクトをファイルに書き込むために1行だけ追加されました。 Complete-Payment（Processor）はニックネームを確認し、ファイル名と照合します。ファイルが見つかった場合、rconのコマンドをコンパイルして実行します。

スタートマインクラフト著者が以前の記事で書いたものについては、わずかに変更されました。今、彼は結論を聞いて、プレーヤーのニックネームを見て、支払いプロセッサーにそれらを渡します。

実際のコールバックを行う


プラグインを使用せずに、真のコールバックを作成します。このため、Start-Minecraftが変更されました。現在、彼はStdOutをファイルに追加する方法を知っているだけでなく、定期的に各行に沿って歩きます。幸い、プレイヤーがサーバーに入るとminecraftは非常に具体的なメッセージを残します。

[04:20:00 INFO]: UUID of player XXPROHUNTERXX is 23e93d2e-r34d-7h15 -5h17-a9192cd70b48

この行からニックネームを選ぶのは非常に簡単です。Stdout文字列からデータを取得するために必要なすべてのコードを次に示します。

$Regex = [Regex]::new("of player ([^ ]+)")
 
powershell.exe -file ".\Start-MinecraftHandler.ps1" -type $type -MinecraftPath $MinecraftPath | Tee-Object $LogFile -Append | ForEach-Object {
 
     Write-host $_
        
    $Player = $Regex.Matches($_).value -replace "of player "

        #   
    }
}


新しい行が$ _パイプラインに入力され、コンソールウィンドウにそれを書き込んで定期的に実行します。レギュラー自体が動作すると通知してくれるのでとても便利です。

ここから任意のコードを呼び出すことができます。たとえば、同じRCONを使用して、PMのプレーヤーに挨拶し、不和のボットを使用して、誰かがサーバーにログオンしたこと、マットの参加を禁止したことなどを通知できます。

すべてを便利なモジュールで作成します


JavaプロセスとWebListenerプロセスには異なるスレッドが必要ですが、作成者はWebListenerとサーバーを別々に実行する必要性に満足していません。著者は、1つのチームですべてを一度に望んでいます。

したがって、Powershell 7を使用して、これとその両方を起動します。そしてそれは私たちを助けます：

ForEach-Object -Parallel {}

コマンドレットはinputObjectで動作するため、複雑でない配列をフィードし、スイッチを使用してストリームを共有します。

"A", "B" | ForEach-Object -Parallel {
 
    Import-Module ".\Start-Minecraft.ps1"
 
    Import-Module ".\Start-WebListener.ps1"
 
    switch ($_) {
        "A" {
            Start-WebListener -Path "C:\mc\"
        }
        "B" {
            Start-Minecraft -Type Vanilla -LogFile ".\stdout.txt" -MinecraftPath "C:\mc\"
        }
        
    }
}

したがって、私たちは松葉杖で、1つの端末から2つの異なるプロセスを開始し、入力を失うことさえありませんでした。しかし、別の問題がありました。WebListenerは、通常のサーバーのシャットダウン後にコンソールをロックし、どこにも行きたくありません。

毎回端末を再起動しないように、ランダムキーがStart-MinecraftHandler.ps1とStart-WebListener.ps1に追加されました。これにより、WebListenerのPOSTを介してサーバーが停止します。

Start-MinecraftHandler.ps1は、正常終了を記録すると、次のコマンドを実行します。

Invoke-WebRequest -Method Post -Uri localhost -Body $StopToken | Out-Null

$ StopTokenには、起動スクリプトによってリスナーとハンドラーの両方に事前に渡されるランダムな数値が含まれています。リスナーは、リクエストで受け取った内容を確認し、リクエストの本文が$ StopTokenと一致する場合はオフにします。

if ($DecodedContent -eq $StopToken) {
        Write-Host "Stopping WebListener"
        $context.Response.Headers.Add("Content-Type", "text/plain")
        $context.Response.StatusCode = 200
        $ResponseBuffer = [System.Text.Encoding]::UTF8.GetBytes("")
        $context.Response.ContentLength64 = $ResponseBuffer.Length
        $context.Response.OutputStream.Write($ResponseBuffer, 0, $ResponseBuffer.Length)
        $context.Response.Close()
        $http.Close()
        break
      }

安全です。トークンについてはRAMだけが知っており、他の人は知りません。すべてのモジュールはPowerShell 7の下から起動され、PowerShell 7のモジュールへのパスはWindows Powershellのパスとは異なります。すべてがここに積み上げられました。自分で書くときは覚えておいてください。
