PVS-Studio-進歩のエンジン


PVS-Studioチームと当社の製品は、ソフトウェア品質の開発に大きく貢献しています。さらに、閉じたプロジェクトと開いたプロジェクトでのエラーの明示的な検出に加えて、コンパイラーやその他のコード分析ツールの開発に間接的に貢献しています。私たちは時々トレンドセッターであることを嬉しく思っており、私たちはこれについてブログに小さなメモを書くことにしました。

PVS-Studioは、C、C ++、C＃、およびJavaで記述されたプログラムのソースコードのエラーおよび潜在的な脆弱性を検出するための静的アナライザーです。

静的コード分析の手法とツールを普及させるために、さまざまなオープンソースプロジェクトの検証に関するメモを書きます。特に、さまざまなコンパイラを随時チェックしています。たとえば、GCC、LLVM、PascalABC.NET、Roslynなどのプロジェクトでエラーを確認しました。

興味深い事実に繰り返し気づきました。たとえばLLVMまたはGCCを確認するとすぐに、1つまたは2つのリリース後、PVS-Studioがコードで検出できるエラーを特定することを目的として、これらのコンパイラーにいくつかの新しい診断が表示されます。残念ながら、日付と関連するイノベーションへのリンクを書き出すことは想定していなかったため、ここで私たちの言葉を伝える必要があります。さまざまなC ++コンパイラーが診断の一部を借用しており、これは完全に正常で、正しく、有用であると信じています。

C ++コンパイラーに加えて、C＃アナライザーが借用診断に接続されるようになりました。これは、PVS-Studioに実装されたC＃アナライザーが別のロールモデルになることを意味します。これを実現することは素晴らしいことです。

この場合、私は修正することができます、これがどのように起こるかをリアルタイムで言うことができます。 2019年8月13日に、.NET Core Libraries（CoreFX）のチェックに特化した大きな記事を公​​開しました。特に、この記事では、補間された文字列の使用に関連するエラーパターンについて説明しました（V3138診断を参照）。 CoreFX開発者は私たちのパブリケーションに興味を持って対応し、発見したバグの修正に取り掛かりました。そしてすでに8月14日に、これらの非常に内挿された線に関連して検出されたエラーに遭遇しました。

楽しい部分です。同日、Roslyn Analyzersプロジェクトは、新しい診断を実装するという課題に直面しました。新しい規則：特殊文字$ 2767が欠落している補間文字列。これはCoreFXで修正されたエラーに関連付けられています。

この作業がCoreFX開発者に役立ち、診断がRoslynアナライザー開発者が従うべき例になったことを嬉しく思います。 PVS-Studioツールがディスカッションのどこにも言及されていないのは残念です。彼ら自身がこれらのエラーを見つけて診断を思いついたようでした。もちろん、私たちがソースとして言及されていれば、私たちはお世辞になります。

なぜこのすべてについて書くことにしたのですか？私たちは非常に満足しており、少しでも誇りに思っています！私たちの経験を検討すると、他のコンパイラは新しい診断を実装し、開発されたソフトウェア全体の品質を向上させます。私たちはコンパイラがエラーを見つける能力の開発に影響を与えるだけではないことを理解しています。ただし、このプロセスに貢献していることを承知させていただきます。

他のツールがPVS-Studioと同じエラーを見つけることを徐々に学習していることは気になりますか？番号。私たちのツールは存在し、コンパイラよりも常に優れているという理由で販売されています。私たちの仕事は常に先を行くことです。私たちが常に追いついているという事実を認識することは、私たちがリラックスすることを許さず、それはすべての人に利益をもたらします。これに加えて、PVS-Studioは警告であるだけでなく、次のことも理解する必要があります。

  • 高速で高品質のサポート（プログラマーのみが手紙に答えます）;
  • Visual Studio、IntelliJ IDEA、SonarQube、Jenkins、IncrediBuildとの統合。
  • ローカルとクラウドの両方で使用できる機能（Docker、Travis CI）。
  • 大規模な古いプロジェクトに分析を統合するためのツール（大量抑制）。
  • 各エラーパターンの例を含む詳細なドキュメント。
  • 開発者に手紙を送るためのメカニズム（BlameNotifier）。
  • コンパイラの監視（コンパイラの監視）;
  • 等。

ご清聴ありがとうございました。PVS-Studioについてご満足いただけたと思います。そして、してみてください、あなたのプロジェクトのコードの継続的な品質管理のための私達のアナライザを。
