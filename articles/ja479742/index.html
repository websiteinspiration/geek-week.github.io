裏庭-マルチクラウドおよびハイブリッドインフラストラクチャ上の自動サービスメッシュ



自分のクラスタで裏庭を遊んでくださいcurl https://getbackyards.sh | sh && backyards install -a --run-demo。興味がある場合は、最新リリースの詳細をご覧ください。

6月に、Istioオペレーターに加えて自動サービスメッシュであるBackyardsを導入しました。マルチクラウドおよびハイブリッドインフラストラクチャでのサービスメッシュ管理を大幅に簡素化します。裏庭は、バンザイクラウドのパイプラインコンテナー管理プラットフォームに統合されていますが、スタンドアロン製品としても利用できます。もちろん、Pipelineとの統合には多くの特定の利点（複数のクラウドでのアプリケーションの管理など）がありますが、BackyardsはすべてのKubernetesインストールで機能します。

では裏庭発表、私たちは、主な機能のいくつかを議論：ユニークなグラフィック表現を通じてアプリケーションを監視し、便利なUIからIstio機能を管理し、数回のクリックでサービスメッシュを作成します。次に、プラットフォームがマルチクラウドおよびハイブリッドデプロイでIstioマルチクラスターサービスネットワークを管理できることを説明しました。

この記事では、この機能である自動マルチクラスターサービスメッシュについて詳しく見ていきます。

1つ以上のクラスター、マルチクラウド環境、ハイブリッド環境のいずれであっても、Backyardsは同じ機能セットと便利なインターフェースを使用します。1つのクラスターから開始するのは非常に簡単で、数回のクリックでサービスメッシュを他のクラスター（マルチクラウドまたはハイブリッドクラウド）に展開します。

動機


ハイブリッドおよびマルチクラウドシステムは、企業で急速に一般的になりつつあります。 Googleが今年4月にAnthosを発表して以来、関心と意思決定は大きく成長しています。

また、このトピックについて話し合ったほとんどの企業で、このような関心が高まっていることに気づきました。誰かが自分のデータセンターでデータを処理し、負荷のピーク時にのみクラウドサービスにアクセスすることを好みます。また、クラウドプロバイダーからの最低料金とベストディールを常に探しており、価格が許す場合にのみワークロードを増やしています。規制上の規制やビジネス要件により、特定のクライアントの要求に応じて異なるクラウド間を移動したり、いくつかの異なるトポロジを構築したりするための便利な方法がすべて必要ですこのマルチクラウドとハイブリッドインフラストラクチャの上に。

ハイブリッドおよびマルチクラスターソリューションの需要が高まるにつれ、私たちはお客様のニーズを満たす方法を考え始めました。

パイプラインユーザーは1つ以上のクラウドで作業します。それは彼らが必要とする特定の機能に依存します。3つの主要な機能を区別できます。

  • 複数のクラウドアプリケーション管理
  • マルチクラウドおよびハイブリッドインフラストラクチャ上のIstioベースの自動サービスメッシュ
  • Kubernetes Federation v2によるリソースとアプリケーションの統合デプロイメント

バックグラウンド


Pipelineプラットフォームは当初から、さまざまなレベル（クラスター、デプロイメント、サービス）でさまざまなクラウドプロバイダーの接続をサポートしていました。これは常に私たちの優先事項の1つです。

同時に、Istioにも積極的に取り組みました。オペレーターは、Istioのデプロイと管理を簡素化するために最初にリリースされ、次にマルチクラスター機能を調査しました。その結果、オープンソースのオペレーターであるIstioを使用して、パイプラインに統合されたマルチクラウドおよびハイブリッドインフラストラクチャ向けの自動化製品であるBackyardsを作成しました。
