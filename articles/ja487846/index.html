EmberレジストリAPIを改善するRFC（Request For Comments）


Chris Cricho（@chriskrycho）は、すべてのレジストリインターフェイスに新しいオブジェクト指向APIを追加することを提案しました。文字列ベースのマイクロ構文を使用して現在のAPIを宣言する capabilities既存の認識エンジンを安全にサポートするプロパティを紹介します。


RFC：Enhanced Ember Registry APIは、microsyntaxの現在の問題とTypeScriptサポートの欠如について説明しています。RFCは、フィールドを含む新しいAPI識別子を追加することをお勧めしますname、type必要に応じて、namespace代わりに現在のmicrosyntaxの。例えば：


代わりに


getOwner(this).lookup('service:session');

になります


getOwner(this).lookup({ type: 'service', name: 'session' })

代わりに


class Example {
  @service('global@session')
  session;
}

になります


class Example {
  @service({ namespace: 'global', name: 'session' })
  session;
}

詳細を知りたい、または独自のレビューを追加したい場合は、GitHubのRFCをチェックしてください！
