WebComponentsのクイックスタート

Webコンポーネントは、コンポーネントアーキテクチャを整理するためのソフトウェアインターフェイスを定義する一連の標準です。これらはすべて、最新バージョンのブラウザに実装されています。ライブラリやコードトランスパイラを接続する必要はありませんが、たとえばInternet Explorer 11との互換性が必要な場合は、おそらくライブラリとトランスパイラを使用する必要があります。

この記事は、初期レベルのトレーニングと、何らかのフロントエンドフレームワークの経験を持つ開発者を対象としていますが、おそらくいくつかのトリックのおかげで、経験豊富な専門家にとっては興味深いでしょう。

以下に引用されている実験はすべてChromeでテストされており、Firefoxは最新バージョンではない場合もあります。

それでは始めましょう。

まず、プロジェクト用のディレクトリを作成し、そこに移動します。

mkdir mywebcomp
cd mywebcomp

実行： 

npm init

このディレクトリでは、デフォルトですべての質問に回答します。

ディレクトリに最も単純なコンテンツを含むindex.htmlファイルを作成します。

<html lang="en">

<body>

</body>
</html>

要素のタグを追加します。名前にはハイフンが含まれている必要があります。これは、CusomElementsサブシステムがこの要素を標準の要素上に構築するものとして定義しようとするための信号です。

<html lang="en">

<body>

<my-webcomp></my-webcomp>

</body>
</html>

ハンドラークラスをスクリプトタグに追加します。

<script type="module">
   class MyWebComp extends HTMLElement {
       connectedCallback() {
           this.insertAdjacentHTML('beforeEnd', `<div>Hello</div>`)
       }
   }
   customElements.define('my-webcomp', MyWebComp);
</script>

モジュラースクリプトタグで、customElements.define（）メソッドを使用してmy-webcompタグの背後で定義した新しいクラスを定義しました。そして、connectedCallback（）メソッドにコードを追加することで、コンポーネントの実装をツリーに追加するときにその呼び出しを提供しました。結果はすでにブラウザーで表示できます。



ただし、HTMLレイアウトをコードに直接配置することは、小さな断片に便利な場合は一般に正しくありません。これは、断片がまともなサイズに成長するときに特に当てはまります。たとえば、20個の要素を持つフォームでは、サブコンポーネントに分割することも常に便利であるとは限りません。したがって、最初に、同じhtmlに配置されるテンプレートでレイアウトを作成しますが、必要に応じて別のファイルからそれをロードすることを妨げるものはありません。

<html lang="en">

<body>

<template id="myWebCompTemplate">
   <div>Hello</div>
</template>

<my-webcomp></my-webcomp>

<script type="module">
   class MyWebComp extends HTMLElement {
       connectedCallback() {
           let tplEl = document.querySelector('#myWebCompTemplate');
           let html = document.importNode(tplEl.content, true);
           this.appendChild(html);
       }
   }
   customElements.define('my-webcomp', MyWebComp);
</script>

</body>
</html>

コンポーネントコードでは、idでテンプレート要素を取得することにより、行挿入をhtmlに置き換えました。インポート、つまり この要素のコピーを作成し、現在の要素のコンテンツにリンクします。

idはcamelCase表記で名前が付けられています 要素のすべての識別子は、ハイフンまたはその他の特殊文字を使用すると、グローバル名前空間にスローされます。それらへの文字アクセスはあまりエレガントではないかもしれません。それら。代わりに：

           let tplEl = document.querySelector('#myWebCompTemplate');
           let html = document.importNode(tplEl.content, true);

一行で書く： 

let html = document.importNode(myWebCompTemplate.content, true);

このコードは同じように機能しますが、あまり安全ではないと考えられています。また、要素にIDを割り当てると、メソッドを呼び出してプロパティ値を取得することにより、グローバル名前空間のインスタンスとしてコンテキスト内のどこからでもそれにアクセスできます。
たとえば、次のようになります。

<my-webcomp id="myWebComp"></my-webcomp>

<script type="module">
   class MyWebComp extends HTMLElement {
       connectedCallback() {
           let html = document.importNode(myWebCompTemplate.content, true);
           this.appendChild(html);
       }
   }
   customElements.define('my-webcomp', MyWebComp);
</script>
<script type="module">
   myWebComp.showMessage();
</script>

このシナリオでは、ページが読み込まれるとすぐにアラートが表示されます。

次に、アラートメッセージを表示するコンポーネントのマウスクリックハンドラーを切断します。

<my-webcomp id="myWebComp" onclick="this.showMessage(event)"></my-webcomp>

<script type="module">
   class MyWebComp extends HTMLElement {
       connectedCallback() {
           let html = document.importNode(myWebCompTemplate.content, true);
           this.appendChild(html);
       }
       showMessage(event) {
           alert("This is the message");
           console.log(event);
       }
   }
   customElements.define('my-webcomp', MyWebComp);
</script>

これで、メッセージをクリックすると、ユーザーのアクションに反応します。showMessage（）



メソッドへの引数は、クリックの座標や要素自体へのリンクなどのイベントデータを格納するイベントオブジェクトも宣言します。 多くの場合、特定の各要素はそのために独自の方法で構成する必要があります。これは属性を使用して行うことができます。 要素の2番目のインスタンスを追加し、それぞれに異なるgreet-nameプロパティを定義します。そのプロパティの値は、要素をクリックしたときに表示されます。





<my-webcomp id="myWebComp" greet-name="John" onclick="this.showMessage(event)"></my-webcomp>
<my-webcomp id="myWebComp2" greet-name="Josh" onclick="this.showMessage(event)"></my-webcomp>
<script type="module">
   class MyWebComp extends HTMLElement {
       connectedCallback() {
           let html = document.importNode(myWebCompTemplate.content, true);
           this.appendChild(html);
       }
       showMessage(event) {
           alert("This is the message " + this.getAttribute('greet-name'));
           console.log(event);
       }
   }
   customElements.define('my-webcomp', MyWebComp);
</script>

これで、最初のボタンをクリックすると「これはジョンへのメッセージです」が表示され、2番目をクリックすると「これはジョシュへのメッセージです」が表示されます。

イベント処理ではなく属性を使用する必要がある場合がありますが、テンプレートに直接レンダリングされます。これには、ターゲット要素にidを追加し、テンプレートオブジェクトのコピーをレンダリングした直後にapiの値を代入します。

<template id="myWebCompTemplate">
   <div id="helloLabel">Hello</div>
</template>

<my-webcomp id="myWebComp" greet-name="John" onclick="this.showMessage(event)"></my-webcomp>
<my-webcomp id="myWebComp2" greet-name="Josh" onclick="this.showMessage(event)"></my-webcomp>

<script type="module">
   class MyWebComp extends HTMLElement {
       connectedCallback() {
           let html = document.importNode(myWebCompTemplate.content, true);
           this.appendChild(html);
           this.querySelector('#helloLabel').textContent += ' ' + this.getAttribute('greet-name');
       }
       showMessage(event) {
           alert("This is the message " + this.getAttribute('greet-name'));
           console.log(event);
       }
   }
   customElements.define('my-webcomp', MyWebComp);
</script>

代わりに.innerHTML が存在するか、最初のセレクターからオブジェクトに対して同じ.insertAdjacentHTML（）メソッドを呼び出すことができます。 長い間、IDの使用は不適切な形式と見なされていました。大量のコードではIDが重複する可能性があり、衝突が発生したためです。ただし、シャドウツリーテクノロジーの出現により、識別子、スタイル、およびその他のリソースを使用して、要素の内部コンテンツを恐れることなく分離できます。Webコンポーネントの場合、シャドウツリーは次のように有効になります。



           this.attachShadow({mode: 'open'});

真実は、これに対するすべてのDOM呼び出しはそれほど多くないため、this.shadowRootで置き換える必要があるということです。

<script type="module">
   class MyWebComp extends HTMLElement {
       connectedCallback() {
           let html = document.importNode(myWebCompTemplate.content, true);
           this.attachShadow({mode: 'open'});
           this.shadowRoot.appendChild(html);
           this.shadowRoot.querySelector('#helloLabel').textContent += ' ' + this.getAttribute('greet-name');
       }
       showMessage(event) {
           alert("This is the message " + this.getAttribute('greet-name'));
           console.log(event);
       }
   }
   customElements.define('my-webcomp', MyWebComp);
</script>

視覚的には、このコードの動作は再び変更されることはありませんが、グローバルネームスペースにhelloLabelがなくなり、ページにはすでにこの識別子を持つ2つの要素があります。そして、例えば、次のようにそれらにアクセスできます：

myWebComp.shadowRoot.getElementById('helloLabel');

そして、.attachShadow（）メソッドで対応する属性を渡してツリーを閉じない場合。

テストを書くことも開発にとって重要であり、コンポーネントの開発中にコードを実行可能に保つのに役立ちます。Webコンポーネントをテストするには、ブラウザーから起動モードでmochaを使用できます。

パッケージを追加します。

npm i mocha chai wct-mocha --save-dev

これを行うには、テストディレクトリを作成し、それに次のall.htmlファイルを追加します。

<!doctype html>
<html>
<head>
   <meta charset="utf-8">
   <script src="../node_modules/mocha/mocha.js"></script>
   <script src="../node_modules/chai/chai.js"></script>
   <script src="../node_modules/wct-mocha/wct-mocha.js"></script>
</head>
<body>

<script>

   WCT.loadSuites([
       'my-webcomp.tests.html',
   ]);

</script>
</body>
</html>

index.htmlを コピーしてテストし、my-webcomp.tests.htmlという名前を付けて、all.htmlと同じヘッドコンテンツを追加します。

ただし、通常の初期化と操作を、テスト実行の一部として実行されるものに置き換える必要があります。

<script type="module">
   import { MyWebComp } from "../my-webcomp.js";

   customElements.define('my-webcomp', MyWebComp);
   suite('MyWebComp', () => {
       test('is MyWebComp', () => {
           const element = document.getElementById('myWebComp');
           chai.assert.instanceOf(element, MyWebComp);
       });
   });
</script>

今入ると

http://127.0.0.1:8080/test/all.html

テストレポートが表示されます。



ただし、テストを自動的に実行する必要がある場合や、別のブラウザで特別なユーティリティをインストールする必要がある場合があります。

sudo npm install --global web-component-tester --unsafe-perm

wct --npm
