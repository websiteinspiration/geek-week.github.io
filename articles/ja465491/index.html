Zabbix + Voximplant：通話によるモニタリング、または心配をやめて迅速にセットアップする方法

Zabbix-スマート、柔軟、カスタマイズ可能-何でもできます。またはほとんどすべて。特にクラウドテレフォニーが便利なHTTP APIを備えている場合は、彼がクラウドテレフォニーと友達になれないのは奇妙です。今日は、Voximplantを介した通知呼び出し用にZabbixを構成する方法を示します。これは、緊急の場合に非常に役立ちます（「説明する時間はありません。呼び出す必要があります-戦闘サーバーが落ちました」）。

映画「マトリックス」のフレームに基づいています（マトリックス（1999）、ワーナーブラザーズ）。
注：このチュートリアルでは、動作中のZabbixがすでにあり、Zabbixフロントエンドが設定されていることを前提としています。Zabbixの最初のインストールはこの記事の目的ではありません。好奇心が強い/そのようなガイドが必要な場合は、公式ドキュメントやGoogleを参照できます。

私たちが取り組むもの


この例では、Zabbix 4.2.6がインストールされているCentOS 7（執筆時の最新の安定バージョン）でVDSを使用しています。Zabbixが動作する同じサーバーを監視します。Voximplantには、プラットフォームのアカウントとJSのコール処理スクリプトが必要です。そしてもう少し。

私たちは何をしますか


ロケットサイエンスではありません。/tmp/habrTriggerFileファイルの出現に応答する非常に簡単な監視を実行してみましょう。そのようなファイルが出現すると、ZabbixはHTTP APIを介してVoximplantに連絡し、指定された電話番号を呼び出すスクリプトが機能します。サーバーにファイルが作成されました。そして、電話を切ります。ファイルを削除または名前を変更してから元に戻すと、再び呼び出しが行われます。何もしなければ、呼び出しは1つだけになります。

ボキシムプラント


アカウントにログイン（または登録）して、 habr アプリを作成します。アプリケーション内で、[シナリオ]タブに移動し、次のコードで通知スクリプトを作成します。

VoxEngine.addEventListener(AppEvents.Started, () => {
  const call = VoxEngine.callPSTN(" ", "   ")
  call.addEventListener(CallEvents.PlaybackFinished, () => {
    VoxEngine.terminate()
  })
  call.addEventListener(CallEvents.Connected, () => {
    setTimeout(
      () => call.say(".     .", Language.RU_RUSSIAN_FEMALE),
      1000
    )
  })
})

このスクリプトはHTTPリクエストで実行できます（電話番号を購入することなく実行できます）。callPSTNの電話番号を置き換えることを忘れないでください。1つ目はプラットフォームが呼び出す相手、2つ目はcalleridで、呼び出されたサブスクライバーに表示されます。プラットフォームで確認済みの個人番号か、以前Voximplantから購入した番号のいずれかです（注：仮想番号は使用できません）。スクリプトは番号を呼び出します。接続後、ファイルが作成され、電話が切れたことが報告されます。

スクリプトを保存し、[ルーティング]タブに移動してルールを作成します。

ルールを保存して、そのIDをコピーします。


次に、APIキーが必要になります。メインメニューの適切なセクションで取得してください。
APIキーとルールIDがわかっているので、それらをHTTP APIのStartScenariosメソッドに置き換えることができます。これは次のようになります。

https://api.voximplant.com/platform_api/StartScenarios/?account_id=1&api_key=4ed5695e-4690-448r-95e8-3f8f0001287c&rule_id=28

少し後、監視を設定するときにこの要求を使用します。監視はすでに開始されています。

Zabbix


ユーザー（または管理者）としてZabbixフロントエンドを入力し、[構成-ホスト]セクションで[ホストの作成]をクリックします。名前にlocalhostと書き込み、グループにtest groupを指定して、下の[追加]をクリックします。


ホストのリストに、デフォルトのZabbixサーバーとlocalhostの両方が表示されます。ここから、設定を続けます。

Zabbix監視は、3つのエンティティのチェーンに基づいています：アイテム-トリガー-アクション。3つのリンクはすべて必須であるため、それぞれを順番に作成する必要があります。

項目


アイテムはデータアキュムレータであり、データモニタリングがないと何にも反応しません。アイテムを作成するには、ホストのリストで、ローカルホスト名の右側にあるアイテムをクリックします。ホスト名自体をクリックして、開いた画面で[アイテム]タブに移動することもできます-本質は同じです。[アイテム]タブで、右上隅にある[アイテムの作成]をクリックします。名前-habr itemを入力し、[Key]フィールドの横にある[Add]をクリックします。下にスクロールします。標準のキーvfs.file.existsが必要です。それを選択すると、[Key]フィールドで置き換えられます。fileを現在のパスに置き換えます。この場合、キーは次のようになります：vfs.file.exists [/ tmp / habrTriggerFile]。[更新間隔]フィールドで5sを設定して、ホストでファイルを作成するときに監視がより速く応答するようにします。ページの下部にある[追加]をクリックすると、すぐに新しいアイテムが表示されます。



引き金


トリガーは収集されたデータを評価し、指定されたしきい値に達すると反応し、アクションを開始します。アイテムが作成された画面で、[トリガー]タブに移動します。


[トリガーの作成]をクリックし、habr triggerという名前を付けます。[式]フィールドの値は、コンストラクターの[追加]ボタンを使用して簡単に収集できます。スクリーンショットのように設定し、[挿入]をクリックします。


ページの下部にある[追加]をクリックします。完了したら、トリガーが作成されます。

アクション


アクションは、トリガーされたトリガーを起動するアクションです。 「構成-アクション」タブを開き、「アクションの作成」をクリックします。その名前は突然の行為です。以下のNew条件ブロックでは、すべてをデフォルトのままにしてhabrを追加します。その結果、トリガー名にhabr条件が含まれるようになります。 [追加]をクリックして条件を追加します-条件ブロックにすぐに表示されます。スクリーンショットを参照してください。


ここで、[操作]タブに移動し、[操作]ブロックで[新規]をクリックしてアクションを追加します。[操作の種類]で[リモートコマンド]を選択すると、[ターゲットリストブロック]と[コマンド]入力フィールドが表示されます。[ターゲットリストブロック]で、[新規]をクリックすると、デフォルトのターゲットが表示されます-現在のホスト、必要です-[追加]をクリックすると、ターゲットがリストに表示されます。入力フィールドに、curl +以前に生成されたリクエストをVoximplantに挿入します。

curl "https://api.voximplant.com/platform_api/StartScenarios/?account_id=1&api_key=4ed5695e-4690-448r-95e8-3f8f0001287c&rule_id=28"

次に、このアクションを追加し（リンクとして追加）、追加ボタンをクリックしてアクションの作成を完了します。


重要なポイント：リモートコマンドアクションのタイプを選択しました。これは、Zabbixエージェント構成でデフォルトでオフになっています。これを修正するには、ホストで構成を開き、EnableRemoteCommandsに値1を入力します。

vi /etc/zabbix/zabbix_agentd.conf
// EnableRemoteCommands = 1

その後、エージェントサービスを再起動します。

service zabbix-agent restart


一連のアクションを開始します


Zabbixパネルで、監視-問題セクションに移動します。トリガーが機能したという警告がすぐに表示されます。

ホストで、ファイルを作成します。

touch /tmp/habrTriggerFile

問題パネルに5秒後、新しいメッセージが表示されます。問題が[ステータス]列で点滅し、[アクション]列のアイコンをクリックすると、アクションの詳細が表示されます（スクリーンショットはクリック可能です）。



途中で、Voximplantスクリプトで指定した電話番号に電話がかけられます。

映画「マトリックス」（The Matrix（1999）、Warner Bros.）のフレームに基づいてい

ます。ファイル名を変更すると、

mv /tmp/habrTriggerFile /tmp/habrTriggerFile_

その後、5秒後に問題のステータスが解決されます。
