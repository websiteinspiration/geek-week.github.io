在Zimbra Collaboration Suite开源版中管理多个通讯录

Zimbra OSE中的全局通讯簿是使用单独的galsync系统帐户实现的，该帐户可在每个Zimbra OSE邮件域上使用。有关该域中每个用户的信息已添加到此帐户的联系人列表中，而域用户可以访问此地址列表，并且可以在输入电子邮件地址时使用它来搜索用户并自动完成。 

全局地址簿的这种实现方式可以简化向其中导入新数据的操作，但是默认情况下，它仅在一个域中工作。大型公司通常将不同的邮件域用于不同的部门或子公司，并且经常需要有一个通用的全局通讯簿，其中将包含几个甚至所有公司域的邮件地址和用户数据。


为了合并来自不同邮件域的galsync帐户的联系人列表，您需要配置它们的同步。以msk.company.ru和spb.company.ru域为例，我们将为公司在莫斯科和圣彼得堡的分支机构配置通用地址簿。

默认情况下，Zimbra OSE域中的galsync帐户的后缀是数字和字母的随机集合。这可能会使我们将来难以使用它，因此，请首先删除galsync.kf9song4@msk.company.ru帐户，然后使用以下命令创建新的galsync @ msk company.ru：

zmgsautil deleteAccount -a galsync.kf9song4@msk.company.ru
zmgsautil createAccount -a galsync@msk.company.ru -n GAL --domain msk.company.ru -t zimbra -s mail.company.ru -f GAL

我们提请您注意以下事实：永远不要以与普通帐户相同的方式删除galsync帐户。这将导致Zimbra OSE在尝试访问没有galsync帐户的域时不断报告错误。

之后，将新的SPB数据源添加到Moscow域的galsync帐户，该联系人将从中自动添加到_SPB contacts文件夹。使用命令zmgsautil addDataSource -a galsync@msk.company.ru -n SPB --domain msk.company.ru -t ldap -f _SPB -p 1d完成此操作。 

作为信息源，我们指定域spb.company.ru并设置一个过滤器以搜索联系人：zmprov mds galsync @ msk。 = zimbraDistributionList））））“zimbraGalSyncLdapSearchBase DC = SPB，DC =公司，DC = RU zimbraGalSyncLdapURL LDAP（（zimbraHideInGAL = TRUE！）：//192.168.0.15：389。

现在仅需打开由我们添加的联系人源zmprov mds galsync@msk.company.ru SPB zimbraDataSourceEnabled TRUE，并使用命令zmgsautil forceSync -a galsync@msk.company.ru -n SPB强制进行第一次同步。

要确保全局通讯簿之间的联系人同步有效，您可以检查我们刚刚创建的_SPB文件夹的内容  zmmailbox -z -m galsync@msk.company.ru gaf。如果一切正常，仅需输入命令  zmprov md msk.company.ru zimbraGALMode  并检查从圣彼得堡域拨打联系人时自动完成的工作方式。

如果您需要spb.company.ru域的用户还必须能够看到msk.company.ru域的用户，则需要执行相同的操作，但是要交换域：

  1. zmgsautil addDataSource -a galsync@spb.company.ru -n MSK-域spb.company.ru -t ldap -f _MSK -p1d
  2. zmprov mds galsync@spb.company.ru SPB zimbraGalSyncLdapBindDn $zimbra_ldap_userdn zimbraGalSyncLdapBindPassword $zimbra_ldap_password zimbraGalSyncLdapFilter '(|(&(mail=*)(zimbraAccountStatus=active)(!(zimbraHideInGAL=TRUE)))(&(mail=*)(objectClass=zimbraDistributionList)(!(zimbraHideInGAL=TRUE))))' zimbraGalSyncLdapSearchBase dc=msk,dc=company,dc=ru zimbraGalSyncLdapURL ldap://192.168.0.15:389
  3. zmprov mds galsync@spb.company.ru MSK zimbraDataSourceEnabled TRUE
  4. zmgsautil forceSync -a galsync@spb.company.ru -n MSKzmmailbox -z -m galsync@spb.company.ru gaf
  5. zmprov md spb.company.ru zimbraGALMode both

galsync帐户中的数据源数量是无限的。

您可以在管理控制台中为您的galsync帐户添加新的联系人来源。但是，为此，无论如何，您仍然必须在命令行上工作。例如，要将联系人从卡卢加州的分支添加到莫斯科域的galsync，请使用命令  zmgsautil addDataSource -a galsync@msk.company.ru -n KLG --domain msk.company.ru -t ldap -f _KLG -p 1d添加一个KLG数据源和一个名为_KLG的文件夹。



然后，在管理控制台中，转到“设置”“域”“配置GAL”，在打开的全局通讯簿配置向导中，选择“ KLG源”并在其中输入所有必需的参数，类似于先前示例中使用的那些参数。



您还可以将Active Directory连接为外部联系人源。这样做的方式与使用常规LDAP的方式相同：创建了AD数据源-zmgsautil addDataSource -a galsync@msk.company.ru -n AD --domain msk.company.ru -t ldap -f _AD -p 1d，然后在管理控制台中指示搜索过滤器和自动完成数据。
