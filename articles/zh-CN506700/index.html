我们为Android配置GitHub Action，并在PlayMarket中进行后续部署

您好，habrozhiteli！有一天，我开始学习Android的GitHub Actions。早些时候，我曾成功尝试在Flutter上为项目配置此功能，但没有进行部署，因为它包含了很多信息，并且在英语资源和俄语资源上均提供了指南，但是对于本地android系统而言，一切并不是那么简单。我决定写下主要问题及其解决方案。

第一步：设置自动签名完成的APK


我遇到的第一个问题是进行通用设置，使您可以在本地和全局收集发行版本。

我使用该选项并使用一个文件keystore.properties，该文件允许我们将开发人员密钥添加到项目文件夹中，而不会发出密码，这是这样完成的：

apply plugin: ...

def keystorePropertiesFile = rootProject.file("keystore.properties")
def keystoreProperties = new Properties()
keystoreProperties.load(new FileInputStream(keystorePropertiesFile))

android {
  ...

  signingConfigs {
    release {
        storeFile file("../MyKey.jks")
        storePassword keystoreProperties['RELEASE_STORE_PASSWORD']
        keyAlias keystoreProperties['RELEASE_KEY_ALIAS']
        keyPassword keystoreProperties['RELEASE_KEY_PASSWORD']
    }
    debug {
        storeFile file('../debug.keystore')
    }
  }

  buildTypes {
    release {
        proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        minifyEnabled false
        signingConfig signingConfigs.release
        buildConfigField "String", "PIN_ALIAS", keystoreProperties['PIN_ALIAS']
        buildConfigField "String", "DB_PASS_ALIAS", keystoreProperties['DB_PASS_ALIAS']
    }
    debug {
        minifyEnabled false
        signingConfig signingConfigs.debug
        buildConfigField "String", "PIN_ALIAS", keystoreProperties['PIN_ALIAS']
        buildConfigField "String", "DB_PASS_ALIAS", keystoreProperties['DB_PASS_ALIAS']
    }
  }
}

dependencies {
  ...
}

然后出现了一个问题，如何解决这个问题，以便我们可以从中获取密钥，${{ secrets.MY_KEY }}同时了解是否拥有keystore.properties，然后从中获取密钥，如果没有，则从秘密中获取密钥？在其中一种使用flutter的指南中找到了解决方案，在该指南中他们使用了环境（顺便说一句，这是一种很酷的方法，以免与我们的开发人员键擦肩而过），但这并不能解决问题。在尝试了几种方法并引入了其他文件等之后，我决定采用最简单的方法：我们另外引入了几个变量（取决于所需的数量），并检查keystore.properties文件的存在：

def release_store_password
def release_key_password
def release_key_alias
def pin_alias
def db_pass_alias

def keystoreProperties = new Properties()
if (rootProject.file("keystore.properties").exists()) {
    keystoreProperties.load(new FileInputStream(rootProject.file("keystore.properties")))
    release_store_password = keystoreProperties['RELEASE_STORE_PASSWORD']
    release_key_password = keystoreProperties['RELEASE_KEY_PASSWORD']
    release_key_alias = keystoreProperties['RELEASE_KEY_ALIAS']
    pin_alias = keystoreProperties['PIN_ALIAS']
    db_pass_alias = keystoreProperties['DB_PASS_ALIAS']
} else {
    release_store_password = System.env.RELEASE_STORE_PASSWORD
    release_key_password = System.env.RELEASE_KEY_PASSWORD
    release_key_alias = System.env.RELEASE_KEY_ALIAS
    pin_alias = System.env.PIN_ALIAS
    db_pass_alias = System.env.DB_PASS_ALIAS
}

android {
   signingConfigs {
        release {
            storeFile file("../my_key.jks")
            storePassword = release_store_password
            keyAlias = release_key_alias
            keyPassword = release_key_password
        }
    buildType{
       release {
          buildConfigField "String", "PIN_ALIAS", "\"$pin_alias\"" //     
          buildConfigField "String", "DB_PASS_ALIAS", "\"$db_pass_alias\"" //  .
      }
    }
}

因此，现在我们的收藏家可以收集并立即签署我们的apk。

第二阶段：组装版本。


没有什么比自然界更重要的了，我想获得某种相当通用的，最小化复杂性的选择。谷歌搜索，仔细研究了有多少开发人员-有很多选择，每个人都尽可能地变态。我不需要任何超级复杂的方法BUILD_NUMBER，但是我已经想使用它，但是随后我遇到了GitHub操作的y参数：$ {{github.run_number}}。

$ {{github.run_number}}
. 1 . , . ( — ).

因此，在权衡了所有利弊之后，我们有以下解决方案：

def versionPropsFile = rootProject.file('version.properties')
Properties versionProps = new Properties()
versionProps.load(new FileInputStream(versionPropsFile))
def verCode = versionProps['VERSION_CODE'].toInteger()

android {
  defaultConfig {
    versionCode verCode
    versionName "1.1.$verCode"
  }
}

//version.properties 
VERSION_CODE=1

在工作流程中，执行以下操作：

- name: Output version code
        run: echo VERSION_CODE=${{ github.run_number }} > ./version.properties

最终工作流程-将与冰雹文件一起进行优化和改进


Android CI.yaml：


name: Android_CI

on:
  push:
    branches:
      - beta_release

jobs:
  build:
    runs-on: ubuntu-latest
    name: Build release-apk and deploy to PlayMarket
    steps:
      - uses: actions/checkout@v2
      - name: set up JDK 1.8
        uses: actions/setup-java@v1
        with:
          java-version: 1.8
      # Without NDK not compile and not normal error message. NDK is required
      - name: Install NDK
        run: echo "y" | sudo ${ANDROID_HOME}/tools/bin/sdkmanager --install "ndk;20.0.5594570" --sdk_root=${ANDROID_SDK_ROOT}
      # Some times is have problems with permissions for ./gradle file. Then uncommit it code
      #    - name: Make gradlew executable
      #      run: chmod +x ./gradlew
      - name: Output version code
        run: echo VERSION_CODE=${{ github.run_number }} > ./version.properties
      - name: Build with Gradle
        run: ./gradlew assemble
        env:
          RELEASE_STORE_PASSWORD: ${{ secrets.RELEASE_STORE_PASSWORD }}
          RELEASE_KEY_PASSWORD: ${{ secrets.RELEASE_KEY_PASSWORD }}
          RELEASE_KEY_ALIAS: ${{ secrets.RELEASE_KEY_ALIAS }}
          PIN_ALIAS: ${{ secrets.PIN_ALIAS }}
          DB_PASS_ALIAS: ${{ secrets.DB_PASS_ALIAS }}
      - name: Upload APK
        uses: actions/upload-artifact@v2
        with:
          name: notification_tracker
          path: app/build/outputs/apk/release/notification_tracker.release.apk
      - name: Upload to PlayMarket
        uses: r0adkll/upload-google-play@v1
        with:
          serviceAccountJsonPlainText: ${{ secrets.SERVICE_ACCOUNT_JSON }}
          packageName: com.guralnya.notification_tracker
          releaseFile: app/build/outputs/apk/release/notification_tracker.release.apk
          track: beta
          userFraction: 0.33
          whatsNewDirectory: distribution/whatsnew
