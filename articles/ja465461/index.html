Reactアプリケーションでの静的型付け

TypeScriptは2016年に新たな高みを踏み出しました。開発者は、多くの一般的なテクノロジーを完全に書き換え、既存のプラットフォームに静的分析のサポートを追加し始めました。このようなグローバルプロセスにより、数千、さらには数万のプロジェクトのコードベースに安定性が追加されました。

なぜ反応するのか 今日の時点で、このライブラリは間違いなく競合他社の背景を支配しています。Reactを中心に、世界最大の開発者コミュニティが形成されました。SPAは3つごとにこのプラットフォームで作成されます。React.jsをベースにしたiOS、UWP、AndroidアプリケーションのプラットフォームであるReact Nativeの使用に関連する多くの素晴らしいプロジェクトもあります。

したがって、今日は、TypeScriptとReactの2つの超人気ツールの統合がもたらす可能性を見ていきます。



例


まず、Reactに使用できる型を見てみましょう。
単純なものから始めて、機能コンポーネントにタイプを追加しましょう。

import * as React from 'react';
 
const HelloWorld: React.FunctionComponent<{
  name: string;
}> = ({ name = 'World' }) => {
  return <div>Hello, {props.name}</div>;
};
 
export default HelloWorld;

機能コンポーネントまたは統計レスコンポーネントの場合、タイプReact.FunctionComponentの定義を使用する必要があります。親がコンポーネントに渡すフィールドであるprops引数のタイプを定義することもできます。この場合、propsは文字列型の名前フィールドのみを含むことができます。

これはすべて複雑に見えません。クラスコンポーネントについてはどうですか？

import * as React from 'react';
 
interface State {
  name: string;
}
 
interface Props {}
 
class HelloWorld extends React.Component<Props, State> {
  state = {
    name: 'World'
  }
  
  setName(name: string) {
    this.setState({ name });
  }
  
  redner() {
    return (
      <React.Fragment>
        <hI>Hello, {this.state.name}</hI>
        <input value={this.state.name} onChange={(e) => this.setName(e.target.value)} />
      </React.Fragment>
    );
  }
}

クラスの例では、PropsとStateの2つのインターフェイスを作成しました。彼らの助けを得て、機能コンポーネントの例のように、着信小道具（空）の署名とコンポーネントの状態の署名を特定しました。

デフォルトのプロパティ値を追加することもできます。

import * as React from 'react';
 
interface Props {
  name?: string;
}
 
export default class HelloWorld extends React.Component<Props> {
  static defaultProps: Props = {
    name: 'World'
  };
 
  render () {
    return <hI>Hello, {this.props.name}</hI>;
  }
}

それで全部です！私たちの小さなReactアプリケーションは、コンポーネントのパラメーターと状態値のレベルですでに強く型付けされています。

これが私たちに与えた利点を見てみましょう：

  • コンパイル段階では、すべての型の不一致が見られます。
  • 適切に構成されたエディターは、シグニチャーまたはデータ・タイプの不一致を強調表示するだけで、開発段階でもミスを回避するのに役立ちます。
  • インターフェイスと型定義からのドキュメント。


パラメータの列挙



Enumは列挙データ型です。このタイプを変数またはインターフェースフィールドに追加する場合、このフィールドまたは変数の値はEnum内の特定の値のみにすることができます。
例えば。

 import * as React from 'react';
 
enum Colors {
  RED,
  BLUE,
  GREEN
}
 
const ColorResult: React.FunctionComponent<{
  color: Colors;
}> = ({ color = Colors.Red }) => {
  return <div>Your color is {props.color}</div>;
};
 
export default ColorResult;

すでに知っている機能コンポーネントでは、ユーザーが選択した色を表示します。enum Colorsタイプでは、コンポーネントに送信できるすべての可能なカラーオプションを指定しました。TypeScriptコンパイラが型の不一致をどこかで見つけた場合、エラーをスローすることでこれを示します。

厳格なRedux


2019年も、Reduxで実行されている多くのアプリケーションがあります。TypeScriptはこの状況で役立ちます。

import * as React from 'react';
 
const initialState = { name: 'World' };
type HelloWorldStateProps = Readonly<typeof initialState>;
 
interface Action {
	type: string;
  name?: string;
}
 
const worldNameReducer = (
	state: HelloWorldStateProps = initialState,
	action: Action
): HelloWorldStateProps => {
	switch (action.type) {
		case "SET":
			return { name: action.name };
		case "CLEAR":
			return { name: initialState.name };
		default:
			return state;
	}
};
 
const set = (name): Action => ({ type: "SET", name });
const clear = (): Action => ({ type: "CLEAR" });
 
const store = createStore(
	combineReducers({
		world: worldNameReducer
	})
);
 
type StateProps = ReturnType<typeof mapStateToProps>;
type DispatchProps = typeof mapDispatchToProps;
 
interface AppProps extends StateProps, DispatchProps {}
interface AppState extends StateProps {}
 
class App extends React.Component<AppProps, AppState> {
  state = {
    name: initialState.name
  }
  
  setName(name: string) {
    this.setState({ name });
  }
 
	render() {
		const { set, clear, name } = this.props;
		return (
			<div>
				<hI>Hello, {name}</hI>
        <input value={this.state.name} onChange={(e) => this.setName(e.target.value)} />
        
        <button onClick={() => set(this.state.name)}>Save Name</button>
        <button onClick={() => clear()}>Clear</button>
			</div>
		);
	}
}
 
const mapStateToProps = ({ world }: { world: HelloWorldStateProps }) => ({
	name: world.name,
});
 
const mapDispatchToProps = { set, clear };
 
const AppContainer = connect(
	mapStateToProps,
	mapDispatchToProps
)(App);
 
render(
	<Provider store={store}>
		<AppContainer />
	</Provider>,
	document.getElementById("root")
);

この例では、複数のレベルで一度にタイプをアプリケーションに追加します。まず第一に、それは減速機自体です。入力では、レデューサーはActionを実行し、HelloWorldStatePropsタイプに対応するオブジェクトを常に返す必要があります。最新のアプリケーションにリデューサーがいくつあるかを考えると、これは非常に便利な革新です。また、私たちが持っているすべてのアクションには、厳密なアクション署名があります。

タイピングの次のレベルはコンポーネントです。ここでは、AppPropsとAppStateに型の継承を適用​​しています。そのようなシグネチャを持つデータ型がすでにあるのに、なぜもっと書くのですか？システムのメンテナンスが簡単になります。一部の要素を変更すると、すべての相続人に対して変更が発生します。
