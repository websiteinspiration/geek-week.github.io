10:  00-サービス間対話プラットフォームが必要な理由と、今日の構築方法

11:  00-カフカ。1つの戦いの説明

12:00-  Apache Kafkaのアプリケーション設計パターン

13:  00-開発者の目を通した最新のk8sプロジェクトの監視

14:00-  ヒューストン、問題があります。障害システムの設計、Amazonクラウドの内部サービス開発パターン

良好な監視は、問題が発生する前であっても警告します。ただし、誤動作による損傷を最小限に抑えるような方法でサービスを最初に設計すると、さらに信頼性が高まります。障害に対する設計は、Amazon Web Servicesクラウドサービスの設計の基本です。Vasily Pantyukhinと一緒にこのアプローチを扱い、サービス障害の理由のいくつかを検討し、セルベースのアーキテクチャ、Constant Work、Shuffle Shardingについて説明します。
