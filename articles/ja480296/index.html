リアクティブ開発Telegramボット

こんにちは、Habr'ovchan。

インターネットでのテレグラムチャット用ボットの開発に関するさまざまな情報を検索したところ、すべてのチュートリアルには、telegram-bot-apiライブラリーとの対話のさまざまなメカニズムがかなり含まれていることがわかりました。同志の生活を常に改善すべきだという結論には誰もいませんでした。労働円滑化の側面（プログラマーは本質的に怠惰です）なので、私は自分のラッパーライブラリを使用して自分のチュートリアルを書くことにしました。現時点では、メッセージの読み取りと回答の送信のみが可能ですが、最初にそれを行います。

チャットボットが必要なのはなぜですか？



チャットボットの導入以来、さまざまなソーシャルネットワークのために私たちの時代にチャットボットを開発するというトピックは、デリケートな問題です。

  1. ユーザーが興味のある質問（およびユーザーの興味のために、ユーザーとの迅速な連絡）に対するユーザーへの応答時間を短縮します。
  2. 従業員が手動のアプリケーションを処理し続けるために、会社のウォレットNの負担を軽減します

チャットボットの導入は、特定の問題を解決する責任者の参加を必要とする特定の状況が常にあるため、その人がまったく必要とされないことをまったく意味しませんが、家庭の問題の解決、アプリケーションの処理、および責任者への情報の配信、維持を維持する問題を簡素化および自動化します魂が望んでいる統計と他のもの。

バーチャルアシスタントの導入はそれほど活発ではなく、21世紀の20世紀に入りつつあるため、このトピックのしきい値を下げる必要があると思います。

始めましょう。

注意

実装されたラッパーライブラリを使用するプロジェクトは、herokuなどの特定のサービスにデプロイする目的がないため、webhookを使用しませんが、長いプーリングを使用します（ただし、基本的な機能を備えているため、プロジェクトは長持ちし、ディストリビューターの最初の呼び出しでスリープ状態になりません他のherokuユーザーのさまざまなサービスの稼働時間）。プロジェクトの主なアイデアは、基本的な機能を備えたチャットボットを開発するときに、開発を簡素化し、「ジェスチャー」の数を減らすことでした-着信ユーザー要求を処理することでした。

このライブラリではどのように機能しますか？




主なデータソースは、チャットボットのベースとなる辞書（形式のJSONファイルに含まれています）ですが、独自のプログラムロジックを記述してユーザーメッセージを処理することも意味します。

まず、BotFatherでボットを作成する必要があります（ボットを作成してトークンを受け取ったと想定します）。次のコマンドを実行して、ライブラリを自分で「受け取る」必要があります
go get "github.com/liquiddeath13/reactivetgbot"。

次に、.goファイルを作成し、そこに次のコードを配置する必要があります。

package main

import (
	reactivetgbot "github.com/liquiddeath13/reactivetgbot"
)

func main() {
    BotInstance := reactivetgbot.Init("YOUR_BOT_TOKEN", "PATH_TO_JSON_DICTIONARY")
    if BotInstance != nil {
        go BotInstance.Logic()
        //if we need host our application on Heroku and shouldn't think about uptime
        reactivetgbot.HerokuServiceUP("Telegram ChatBot by liquiddeath13")
    }
}

BotInstance変数には、指定されたトークンを持つチャットボットインスタンスへのリンクと、質問と回答を保存する辞書へのパスが含まれています。

辞書の例：

[
    {
        "Question" : "/about",
        "Answer" : "ChatBot created by liquiddeath13 in 2019 year"
    },
    {
        "Question" : "Hello, how can i contact with your company?",
        "Answer" : "Hello, you can contact us by provided mail address. Email: coolnickname@hostname.domain"
    }
]

次に、チャットボットはTelegram APIの使用を開始し、チャットボットが配置されているチャットで発生したイベントに関する情報を取得します。 [質問]フィールドの辞書に記載されているメッセージがボットのアドレスに届くと、[応答]フィールドでペアになっている回答でメッセージに応答します（つまり、「こんにちは、どのように会社と連絡を取ることができますか？」この例では、ボットは「こんにちは。提供されたメールアドレスでご連絡いただけます。メール：coolnickname@hostname.domain」と返信します。

また、前述のように、高度な同志は、ハンドラーを追加することにより、メッセージを受信したときのボットの動作を決定できます。次に例を示します。

package main

import (
	"fmt"
	reactivetgbot "github.com/liquiddeath13/reactivetgbot"
)

func main() {
	BotInstance := reactivetgbot.Init("YOUR_BOT_TOKEN", "PATH_TO_JSON_DICTIONARY")
	AskCounter := 0
	if BotInstance != nil {
		BotInstance.AppendHandler("how much time people asked you?", func(Msg reactivetgbot.TGMessage) string {
			AskCounter++
			return fmt.Sprintf("Hello.\n%d - so many time people asked me", AskCounter)
		})
		go BotInstance.Logic()
		reactivetgbot.HerokuServiceUP("Telegram ChatBot by liquiddeath13")
	}
}

この例では、「人々があなたに尋ねた時間」という質問を処理するメカニズムについて説明します。答えとして、ボットは、カウンターを増やした後、ユーザーが指定された質問を何回尋ねたかを示す値をユーザーに提供します。

Herokuへの展開（広告なし）


チャットボットは、何らかの理由で、herokuなどのサービスに不快に感じます。これは、サービスが早期に「眠りに落ちる」ことを防ぐ、つまりネットワークアクティビティをエミュレートする（5分に1回Webリソースにアクセスする）ための小さな機能が記述されているためです。
プロジェクトをHerokuにデプロイするには、ユーティリティが必要です。 godep。依存関係をプロジェクトに「パック」します。チームgo get github.com/tools/godepで「ピックアップ」できgodep saveます。プロジェクトフォルダーでコマンドを実行して、依存関係をパックできます。

次に、Herokuでアカウントを作成し、公式の導入ガイドを理解する必要があります。読む時間があまりない場合、または「すべてを一度に1か所に」したい場合は、次の一連のコマンドを使用します。

  1. コマンドを実行してCLIを使用するHerokuリソースの承認 heroku login
  2. git initプロジェクトフォルダーでコマンドを実行する
  3. コマンド実行 git remote add heroku ** git' , Heroku**
  4. git add .すべてのファイルをローカルリポジトリに追加するコマンドを実行する
  5. コマンド実行 git commit -m " , commit'"
  6. コマンドを使用してherokuリポジトリにローカルリポジトリを配置する git push heroku master

アクションが完了したら、何もする必要はありません。Herokuは自動モードでプロジェクトをアセンブルし、ボットはあなたやあなたの部下のためにすべてのルーチン作業を行います。
Webサービスのライフサイクルを監視するために、Herokuには、heroku logs --tail最近のアクティビティのログを監視できるチームがあります。
