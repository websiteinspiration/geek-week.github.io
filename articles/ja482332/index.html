マニアがDOSでWi-Fiネットワークを立ち上げた方法

今月初め、OS / 2ミュージアムポータルに資料が掲載されました。その著者は、古いThinkPad 760XLラップトップを最新のワイヤレスネットワークに接続しました。彼がどのように成功したかを説明します。


/ PD / Marus

プロジェクトの著者はマイケルNecasek、オープンソースのメンバーであるオープンWatcomののコミュニティマルチプラットフォームコンパイラはWatcom C、C ++およびFortranを開発し、。MichaelはDOSを古いラップトップに配備し、息子がレトロゲーム（SierraとLucasArtsのクラシック）をプレイできるようにしました。

ラップトップには、1994年にサンディスクで開発された CFカード用のポートが装備されていました。Michaelはこれらのドライブを使用して（アダプター経由で）必要なデータをコピーするだけで済みましたが、SMBまたはNFSプロトコルを使用してネットワーク経由でファイルを送信する方が便利だと判断しました。しかし、このためには、古いオペレーティングシステムとThinkPadを最新のWi-Fiで「友達にする」必要がありました。

DOSでWi-Fiを利用することはできません


箱から出して、システムは90年代に登場したWaveLANワイヤレスネットワーク標準のみをサポートします。IEEE 802.11bの前身であり、IEEE仕様との互換性はありません。最初のWi-Fi規格は、企業および家庭用コンピューターからDOSがほとんど姿を消した1999年にのみ公開されました。したがって、その作業は最も簡単なものではありませんでした。

Habréに関するブログの資料：

  • 「Stayin 'alive、stayin' alive」：新しいプロトコルにより、Wi-Fiの使用可能範囲が60メートル広がります
  • 別のレジストラがIPv4アドレスの最後のブロックを提供しました

PCMCIA拡張モジュール をラップトップに接続することで最新のワイヤレスネットワークに接続できますが、使用可能なアダプターはWPAテクノロジー（WPA2は言うまでもありません）をサポートしていませんが、WEPのみをサポートしています。 WEPは、ゼロで使用されていたWi-Fiネットワークを保護するためのアルゴリズムです。発見された暗号化の脆弱性のために放棄されました。

その結果、ラップトップをWi-Fiに接続するために、マイケルは Cisco Aironet PCM-352拡張モジュールを使用しました。彼のチップはIEEE 802.11bをサポートし、最大11 Mbpsの速度で実行できます。 2000年代初頭にこのカードの下に適切なドライバーが登場しました。彼はサードパーティのリソースでそれらを見つけました。

WPAの問題を回避するために、マイケルはWEPをサポートする古いTP-LINKネットワーク機器を使用しました。ただし、標準のシスコユーティリティでは、Aironet PCM-352を操作するためのWEPキーを登録できませんでした。したがって、暗号化を無効にした（および非表示のSSID）別のアクセスポイントが必要でした。


/ PD / phreakindee

ラップトップでネットワークを開始するために、マイケルはダウンロードしたアーカイブからのNDISドライバーと、NetBIOS over TCP / IPおよびDHCPをサポートするネットワーククライアント3.0クライアントを使用しました。システムが正しく機能するには、PROTOCOL.INI構成ファイルで、特定の変更を行う必要がありました。適切なSSIDを作成し、ソケットサービスソケットクライアントを構成してください。その後、マイケルはホームネットワーク上のラップトップにファイルを転送することができました。

その他のDOSプロジェクト


このオペレーティングシステムがまもなく40年を迎えるという事実にもかかわらず、DOSコミュニティは依然として進化を続けています。たとえば、1年前、Microsoftの開発者がMS-DOSバージョン1.25および2.0 のソースコードをGitHubに投稿しました。彼らはできるも見つけることが 2014年以来、コンピューター歴史博物館のウェブサイトで。コードベース全体は、コメントを含む12,000行で構成されています。

さまざまなプロジェクトがまだDOSに基づいて構築されています。2016年、ゲーム開発者会議のエンジニアであるブライアンプロビンチャーノは、ゲームRetro Retro RampageをPS4から486番目のプロセッサを搭載したDOSのコンピューターに移植した方法を語りました。彼の報告では 彼は、ゲームが占めるスペースを1.44 MBに、RAMの消費量を4 MBに削減する方法を説明しました。

  • GUIを介してHotSpotを管理する方法
