Pemantauan melalui UptimeRobot

Setiap situs perlu pemantauan, tidak ada sumber daya yang pemiliknya lebih suka untuk tidak tahu tentang downtime. Sangat lucu bahwa banyak pengembang tidak memantau proyek kesayangan mereka, dan pekerja lepas hampir tidak pernah repot dengan pemeriksaan aksesibilitas, meskipun ini biasanya tidak lebih sulit daripada menghubungkan beberapa analisis dan dapat menghemat banyak keberanian.



Ada lebih dari selusin layanan untuk situs pemantauan, sebagian besar dari mereka dibayar penuh atau sangat berkurang dalam versi gratis. Terlepas dari mereka adalah UptimeRobot , yang menyediakan fungsionalitas besar secara gratis dan memungkinkan Anda untuk secara fleksibel mengkonfigurasi langganan berbayar untuk kebutuhan Anda.



Dalam kebanyakan kasus, UptimeRobot mengirimkan permintaan HTTP ke alamat yang ditentukan dan menentukan ketersediaan oleh header respons (2xx - semuanya baik-baik saja, 4xx / 5xx - ini mem-request ulang permintaan dari titik lain selama 30 detik, jika jawabannya negatif lagi, downtime dihitung).



Bebas



  • 50 monitor dengan kecepatan refresh 5 menit
  • Permintaan HTTP (S) ke alamat apa pun
  • Cari kata kunci di halaman
  • Ping (permintaan ICMP)
  • Pemeriksaan port
  • Lansiran melalui surat, kait web, pesan di Twitter, Telegram, Slack, dan layanan lainnya
  • Statistik Waktu Tanggapan
  • Log terperinci selama dua bulan
  • Kemampuan untuk mempublikasikan statistik pada halaman publik ( contoh )
  • API terbuka


Dibayar



  • Hingga 10.000 monitor dengan interval penyegaran menit
  • Lacak kedaluwarsa sertifikat SSL dan pantau masalah HTTPS
  • Berbagai peringatan logika (dengan pemasangan interval dan lainnya)
  • Kustomisasi permintaan HTTP (termasuk tajuk dan status)
  • Pemantauan detak jantung (pemantauan server untuk NAT)
  • Peringatan SMS / panggilan (hingga 900 per bulan)
  • Log tersedia dalam dua tahun


Seperti yang Anda lihat, bahkan ada lebih banyak nishtyaks gratis dalam layanan daripada yang dibeli, dan kemungkinan besar inilah yang membawa popularitas dan pengakuan UptimeRobot. Meskipun, setelah beberapa desain ulang, ia masih disalahkan untuk antarmuka yang mengerikan.







Kata kunci



Jika Anda menggunakan komponen apa pun dengan rendering bersyarat (misalnya, sepiring tentang pekerjaan teknis), Anda dapat menggunakan pencariannya dengan kata kunci untuk melacak statusnya. Secara umum, benda ini memiliki banyak kasus penggunaan keren: karena waktu respons disimpan untuk setiap permintaan, Anda dapat mengukur render berbagai bagian aplikasi "dari samping" - permintaan dikirim dari berbagai belahan dunia dan dapat memberikan gambaran yang sama sekali berbeda. Anda dapat mengonfigurasi peringatan untuk konten yang tidak diinginkan pada halaman dan sebaliknya, periksa keberadaan teks yang diinginkan. API



UptimeRobot menyediakan API XML / JSON / JSON-P sederhana yang sepenuhnya menduplikasi fungsi versi web. Deskripsi terperinci tentang metode dan parameter ada di halaman API , singkatnya - Anda dapat membaca statistik dan log, membuat / menghapus / mengedit / me-reset monitor dan mengelola pengguna.



Ada panduan menarik Mengotomatiskan pemantauan waktu aktif di Kubernetes dengan UptimeRobot , klien on Go , Python, dan banyak interpretasi untuk bahasa lain, panel, dan halaman status khusus .



Denyut jantung



Untuk memantau server yang tidak dapat diakses secara langsung, monitor detak jantung tersedia dalam versi berbayar. Saat membuat ini, tautan unik dihasilkan di mana layanan UptimeRobot mendengarkan permintaan masuk berkala dari server jauh. Jika permintaan berhenti tiba, downtime dihitung.



Apa yang bisa dilakukan sekarang



Satu akun gratis bergantung pada 50 monitor, yang biasanya cukup untuk mencakup situs dan proyek pribadi. Jika Anda belum menghubungkan pemantauan gratis di mana-mana, maka inilah saatnya untuk melakukannya. Dalam 5 menit dan tanpa ada gangguan dengan kode. Dan jika Anda membutuhkan integrasi pemantauan yang lebih dalam ke dalam proyek, pelajari API. Hanya saja, jangan lupa untuk mencari solusi siap pakai untuk tumpukan Anda, mungkin secara massal.
