Citrix Workspace Environment Managementは良い製品ですが、マーケティングは悪いですか？

この記事では、Citrixのポートフォリオからの製品、つまりWEMについてお話します。約3年半前に、CitrixはNorskaleと呼ばれる小さなソフトウェアメーカーとその製品Norskale VUEMを買収しました。この買収の発表は、Citrixコミュニティから非常に好評です。おそらく、RES Softwareが当時のNorskaleライバルの買収に対応して、RES SoftwareがRES ONE Workspace Coreと呼ばれるRES ONE Workspace製品の無料版をリリースしたのでしょう。

過去3年間、Citrixというブランド名で、製品に小さな変更が加えられました。過去のメーカーの名前がす​​べての製品のコンソールから消えたわけではないという事実に注意を払う価値があると思います。

もちろん、多くの読者は、メーカーの公式統計に基づいていない私のアプローチをアマチュア的だと思いますが、私の個人的な見積もりによると、多くの同僚とのコミュニケーションに基づいて、WEMはあまり人気がありません。 WEMの使用は、アドバンストまたはプレミアムCitrix Virtual Apps and Desktopsライセンスのすべての所有者が利用できます。

WEMアーキテクチャについて一言


まず、この製品のアーキテクチャと明記された目的の表面的な見直しから始めることを提案します。 Citrixは、3つの使用シナリオを提供します。

パフォーマンスの最適化 -このシナリオは、リアルタイムでのパフォーマンスの評価とその後の構成の即時変更に基づいて、システムリソース（CPU、メモリ、I / O）の効率的な使用を提供します。最大70％のリソース節約の可能性。

プロファイル管理設定 -このシナリオでは、ネットワークプリンター、フォルダー、ログインスクリプトなど、ユーザー環境のさまざまな要素の設定をGPOからWEMに転送します。 WEMは、Citrix User Profile Managementのグラフィカルなカスタマイズを提供します。このシナリオでは、ログイン時間が大幅に短縮されることを想定しています。
安全保障 -このシナリオは後のバージョンで登場し、2つの主要コンポーネントに基づいています。

  • プロセス管理 -この機能はパフォーマンス最適化セクションから移動し、ブラック/ホワイトリストに基づいて個々のプロセスを開始することの禁止または許可を表します。
  • Application Securityは、単一のユーザーおよびグループに対して、個々のプログラム、スクリプト、およびDLLへのより幅広いアクセス（有効/無効）アクセスを提供します。アプリケーションセキュリティはMicrosoft AppLockerに似ており、そこで作成されたルールをエクスポートすることもできます。

トランスフォーマー -Norskaleは、この製品を、Windows PCをThinClientに変える（PCをシンクライアントに変換する）独立したソリューションとして位置づけました。私のように、私が何らかの組み込みLinuxについて話していると思うなら、あなたは間違っています。 Transformerは、いわゆるWindowsシェルを非表示にする機会にすぎませんキオスクモード。 Transformerは、Citrix独自の「Desktop Lock」製品と直接競合しています。

私の意見では、シェアウェア製品のためのまともなオプションのセット。

アーキテクチャに関しては、ブローカー、エージェント、ライセンスサーバー、管理コンソールなど、すべてが非常にシンプルで論理的です。

  • Broker (Infrastructure Service) – WEM , : , , AD.
  • Database – .
  • Agent – WEM. ( VDA), .
  • Admin Console – .




-私の控えめな意見では、いくつかの同様の理由があります。これらの1つ目は、数千または数万のユーザーがいる大規模なインフラストラクチャに対するWEMの初期の不十分さです。たとえば、1つのブローカーが一度に処理できるエージェントは3,000以下です。おそらく誰かが外部のロードバランシングの助けを借りて1つのインフラストラクチャに複数のWEMブローカーを組み合わせることが可能であるという私の声明に反対するでしょう。

-2つ目の理由は、やはり控えめな意見ですが、使いやすさ、またはその欠如にあります。上記のように、ブローカーは重要なコンポーネントですが、インストール後、2つのコンソールは互いに独立しています。それらの1つ（WEMデータベース管理ユーティリティ）の目的は、最初からインストールする場合はデータベースを作成すること、または新しいバージョンに切り替える場合はデータベースをアップグレードすることです。 2番目のコンソール（WEMインフラストラクチャサービス構成）は、インフラストラクチャ構成用に設計されています。 Citrix開発者が2つのコンソールを1つに統合する（PVSの構成ウィザードなど）か、設定を管理コンソールに転送する（CVADコンソールのウィザードなど）のを妨げたのはなぜですか？

管理コンソール自体は、非論理的であり、絶対に直感的ではありません。 「科学的突く」の方法はそれを習得しません。何が起こっているのかを簡単に思い出せるように、使用シナリオ（最適化、一般設定、ユーザープロファイル（UEM））に応じて自分用にリボンを分割しました。

-スクリプトユーザー環境管理。インストールが不要になり、Webコンソールとしてユーザーに提供される製品が増えているため、WEMが多少古くなっている可能性があります。 Windows XPとWindows 7では、スタートメニューでプログラムをグループ化する必要がありました。複数のネットワークドライブを接続すると、単一のSharePointまたはShareFileページが置き換えられます。 2020年にWindows GPOを介して接続される膨大な数のネットワークプリンターの必要性はどの程度緊急ですか？また、ユーザーが実際に20秒未満でログインすることはどれほど重要ですか。確かにあると思いますが。

-シナリオリソースの最適化。私の意見では、この機能はWEMの最も興味深く魅力的な部分であり、今日に至るまで関連性を失っていません。主な問題は、この機能の配置にあります。最大70％のサーバーのスケーラビリティの約束は、多くのクライアントにとって真実であるにはあまりにも良さそうです。私自身の経験から、WEMはCPUリソースを本当に効果的に管理し、ユーザーセッションのプロセスの優先順位を変更することでこれを行うので、プロセッサーが完全にロードされている場合でもシステム全体が動作し続けると言えます。また、WEMは、アイドル状態（RAMからページファイル）のアプリケーションが占有していたメモリを非常に効果的に解放します。

私の経験では、すべてが正しく確実に機能します。しかし、クライアントに鉄を70％少なく取れることを納得させることは非常に困難です。ほとんどのクライアントは、盲目的に信頼して1つのプログラムのみに依存することを望んでいませんが、必要な鉄分を計算するときは、信頼できる実績のあるスキームに従って行動することを好みます。

起こり得るマーケティングの間違い


彼らのプレゼンテーションでは、Citrixの従業員またはパートナー企業は、WEMがすべてのライセンスを受けた顧客に関心を持つべきであることに賭けました。WEMは常に、すべてのシナリオの同時使用を含む完全なソリューションとして提供されます。私の信念では、これがまさにエラーの原因です。

WEMを使用するように顧客を魅了する方法は？


インフラストラクチャ全体ではなく、パフォーマンスの問題がすでに存在するサイロにWEMを使用することをお勧めしませんか？ローカルの問題を解決できる場合、すべてを同時に改善しようとしても意味がない場合があります。

パフォーマンスの最適化 -この機能の実装には特別な費用は必要ありませんが、すばやく目に見える結果を得ることができます。高速ログオフの1つのオプションだけで、製品の有用性を簡単に実証できます。

結論として、WEMはインフラストラクチャの負荷を軽減し、ユーザーエクスペリエンスに良い影響を与えることができると言えるだけです。製品をどこにでも導入する必要はなく、本当に必要な場所に導入するだけです。

私は常に建設的なコメントと製品でのあなたの経験の説明をうれしく思います。
