wxpython 可用性甚至更实惠。

大家好。

尽管许多应用程序正越来越成为WEB，但在本文中，我想提出应用程序软件的主题。

更具体地说，关于精彩的wxpython GUI库，它是WX小部件的包装。

它非常方便，易于学习，具有许多用于构建任何复杂性的图形界面的组件。

最重要的是，就可访问性而言，它是最实惠的。

尽管大多数组件都是开箱即用的，但仍有一些组件的可用性很差。

但是不久之前，出现了一个新工具，该工具应该可以帮助开发人员进一步提高新的和已经创建的早期组件的可用性。

wx.Accessible


实际上，他出现在较早的版本4.0.4中。但是，在此版本中，wxpython开发人员打破了基本可访问性，这是对可访问性的极大回归。

例如，可以在这里查看此问题：wx.Accessible：如果控件选择不实现IAccessible对象或未给出wx.Control构造函数的name关键字，则屏幕阅读器会看到该标签。

但是，在上一个较大的更新4.1.0中，他们解决了此问题，现在您可以全面了解新工具。

有了它，您既可以使早期无法访问的组件可用，又可以扩展现有的和完全可访问的组件的功能。因此，当我寻找在不更改图形表示的情况下为屏幕阅读器输出扩展文本信息的可能性时，我遇到了它。

由于可访问性主题不是最相关的，并且此组件相对较新，因此即使在互联网的英语部分，甚至在俄语的部分中，我都没有找到任何可理解的示例来使用它。

我将尽我所能为这一领域做出一点贡献。

为了清楚起见，请考虑使用它的示例。

我不会为您提供用于创建wx.Frame窗口的额外标准代码。这些示例可以满足各种口味。因此，我将仅简要地引用关于使用wx.Accessible的创新。
对于我们的示例，请使用adv.CalendarCtrl小部件。

在此小部件中捕获焦点时，您可以使用键盘上的箭头导航日期，但与此同时根本不会发声，并且用户无法理解他当前的日期。

此外，小部件还存在其他问题，例如，无法使用键盘从其更改焦点并移至下一个界面组件（ps：如果有人知道将焦点从此类小部件返回到窗口的选项，请在注释中写上），但现在的任务不是使日历窗口小部件完全可访问，而是显示使用wx.Accessible的工作。

因此，在我们的程序中，我们对此小部件进行了初始化：

self.date = wx.adv.CalendarCtrl(self.panel, wx.ID_ANY,
    wx.DateTime.Today(),
    style=wx.adv.CAL_MONDAY_FIRST |
          wx.adv.CAL_SEQUENTIAL_MONTH_SELECTION)

在下面，您需要添加以下内容：

self.date.SetAccessible(AdvCalendarAccessible(self.date))

AdvCalendarAccessible类是从wx.Accessible继承的我们全新的类，如下所示。

它覆盖了屏幕阅读器的必要方法。
因此，创建我们的新类：

class AdvCalendarAccessible(wx.Accessible):

    def __init__(self, window):
        self.window = window
        super().__init__(window)

    def GetName(self, index):
        date = self.window.GetDate()
        return (wx.ACC_OK, date.Format('%d.%m.%Y'))

    def GetDescription(self, index):
        text = ''
        date = self.window.GetDate()
        attr = self.window.GetAttr(date.day)
        if attr is not None:
            text = attr.GetBackgroundColour().GetAsString()
        return (wx.ACC_OK, text)

在此选项中，我们会根据屏幕阅读器的要求，将日期返回给他们，并返回所选日期的背景色（如果该日期当然由程序标记）。


这就是全部。现在，使用此小部件上的键盘箭头导航时，用户可以听到所有日期。

除了这些方法外，还有其他方法。例如，要表达组件的值或其字符串提示。

完整的文档位于：wx.Accessible。

我希望有人会发现该信息有用，并且您的程序将变得更容易访问。
