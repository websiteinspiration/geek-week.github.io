音声制御による叙事詩の継続


前の部分では、アリスを通じて、音声はMiLightコントローラーによって制御されました。しかし、そのようなコントローラーはすべての家にあるわけではなく、それは私たちの能力を大きく制限します。したがって、私は音声コントロールを通常のランプ、ランプ、シャンデリアなどに拡張することを試みましたが、標準の照明システムの変更は最小限に抑えました。このようなシステムに必要なのは、従来のスイッチをZigBeeプロトコルに従って動作するスマートスイッチに置き換え、USB ZigBeeコーディネーターCC2531をインストールすることだけです。当然のことながら、前回の記事から、スマートなYandexカラムとカスタマイズされた混乱がすでにあります。

CC2531のコストは250ルーブル、プログラマーはCC-Debugger Texas Instrument-500ルーブルです。 Livolo VL-C701Z-11タッチスマートスイッチが最も高価でした-約2000ルーブルで、少し後、AqaraウォールスイッチZigbeeボタンスイッチも2000年頃に追加されました。

当初、私は独自のZigBeeゲートウェイルーターLivolo C700ZW-12を介してスイッチを制御し、それをオープンハブに接続し、yandex2mqttを介してAliceを操縦することを計画していました。腺を電気に接続し、電話でLivoloからアプリケーションをセットアップしました。ゲートウェイスイッチが接続されているのを確認しましたが、すべて電話から問題なく動作しました。しかし、ルーターを備えたオープンハブは、必要な機能がオープンハブにないため、統合できませんでした。

ゲートウェイを放棄し、スイッチを直接CC2531ドングルとマージする必要がありました。そして、ここで、友人のKoenkkのzigbee2mqttプロジェクトがこれを手助けしました。その結果、ZigBeeコーディネーターをスマートスイッチに接続し、分析のために最初にmqttトピックのクエリを通じて動作を確認し、次にAliceを使用してコントロールを構成しました。

したがって、セットアップはどのように行われましたか：

1. コーディネーターのファームウェアで CC2531スニファーをフラッシュし、それをUSBポートに接続します。電源不足に関するメッセージを回避するには、少なくとも2Aを5V電源に接続することをお勧めします。システムのスティックが正しく定義されていることを確認します。ls-l / dev / serial / by-idは、usb-Texas_Instruments_TI_CC2531_USB_CDC ___ 0X00124B0458ED3DDF-if00-> ../../ttyACM0のように表示されます。その場合、出力の最後の部分 "-> ../../ttyACM0"の存在は非常に重要です。スティックはこの部分なしで決定でき、何も機能しません。スティックを点滅させることで、このような問題を解決できました。

2. Zigbee2mqttをインストールしますが、起動しないでください。

3. /opt/zigbee2mqtt/data/configuration.yamlを正しく設定することが重要です。実際のところ、Livoloスマートスイッチはチャネル26で動作するため、このチャネルはファイルで明示的に設定する必要があります。また、ファイルの行の先頭にあるスペースの数に注意することも重要です。設定ファイルの構文を確認するには、YAMLバリデーターを使用できます。

4.構成ファイルが正しく登録され、ドングルが動かなくなった後、zigbee2mqttを実行します。次に、スイッチに踏み込み、ペアリングを開始して、コンソール出力を確認します。データ交換の行、デバイス間の相互rassharkivaniya、そして最後に次のようなものがありますMQTT publish: topic 'zigbee2mqtt/0x001*********a8c9', payload '{"state_left":"OFF","state_right":"OFF","linkquality":60}'。これは、スイッチが接続され、そのステータスが送信されたことを意味します。さらに2つ以上のコンソールを実行する場合は、mqtt-topicを使用してライトを制御できます。- mosquitto_pub -d -h localhost -t zigbee2mqtt/0x001*********a8c9/left/set -m OFF -u a**** -P *****ターン- mosquitto_pub -d -h localhost -t zigbee2mqtt/0x001*********a8c9/left/set -m ON -u a**** -P *****そしてトピーカでチームをチェックすることもできますmosquitto_sub -d -h localhost -t zigbee2mqtt/0x001*********a8c9/left/set -u a**** -P *****。ちなみに、LivoloとAqaraの場合、トピックは識別子0x0 ...だけでなく、構造も異なります。サポートされているデバイスのトピックの構造はここにあります。

5. mqttでデバイスを管理できることを確認したら、すべてをYandexとペアリングします。若干のニュアンスがここに現れました。トピックのzigbee2mqttを介してデバイスを制御するコマンドは、ON / OFFの形式で提供され、結果としてyandex2mqttは1/0になります。この問題の解決策は、yandex2mqttでファイル/mnt/data/root/yandex2mqtt/device.jsを編集することでした。 「オン」の場合、スイッチのint = valを置き換える必要がありますか？ 'on'： 'off'の代わりに '1'： '0'。当然、対応するトピックのオープンハブで切り替えコマンドを修正する必要がありました。これがなければ、Milightのライトは機能しなくなります。そして、最も興味深いことがすぐに明らかになりました-yandex2mqttとzigbee2mqttはローカルのmqttサーバーですべてを公開するため、zenbeeデバイスはデバイスを管理するためにオープンハブを必要としないようです。バンドルは直接動作します-yandex（Alice）-yandex2mqtt-mqtt-zigbee2mqtt-device zigbee。なんとなくなんとなく。しかし、続けましょう。設定ファイル/mnt/data/root/yandex2mqtt/config.jsのデバイス制御トピックを新しい仮想デバイスにファイルし、Yandexダイアログシステムでデバイスのリストを更新すると、Aliceを起動してライトをオンにすることができます。

6. Xiaomiセンサーのペア、漏れセンサー、温度/湿度/圧力センサーをシステムに接続することも可能でした。彼らは2032バッテリーで動作しますが、当然のことながら制御されていませんが#033[32mzigbee2mqtt:info #033[39m 2020-05-04 21:12:21: MQTT publish: topic 'zigbee2mqtt/0x0015************', payload '{"battery":91,"voltage":2985,"temperature":26.2,"humidity":28.65,"pressure":1006.9,"linkquality":52}' - # #033[32mzigbee2mqtt:info #033[39m 2020-05-04 21:21:07: MQTT publish: topic 'zigbee2mqtt/0x001************7', payload '{"battery":100,"voltage":3025,"linkquality":0,"water_leak":true}' 、コップ1杯の水のリークセンサーなどの情報を定期的にトピックにドロップします。これまでのところ、特にメリットはありませんが、これらすべてがotparsitであり、SMSまたは数字をWebマズルに送信するなどの反応がある場合、それは非常に良いことです。
