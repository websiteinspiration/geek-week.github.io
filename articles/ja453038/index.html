こんにちは、ハブル。第3の部分はパイソンを使用して受信機SDRにアクセスする方法を説明しました。ここで、GNUラジオプログラム（コードを1行も書かずに無線デバイスのかなり複雑な構成を作成できるシステム）について理解します。 例として、1台の受信機で複数のFMステーションを並列受信するタスクを考えてみましょう。レシーバーと同じRTL SDR V3を使用します。 カットの下に続きました。









取り付け


はじめに、GNU Radioをインストールする必要があります。Windows用の配布キットはここからダウンロードできます。このシステムはクロスプラットフォームで、LinuxとOSXのバージョンもあります（GNU RadioはRaspberry Piで正常に起動されたようですが、100％保証することはできません）。

実際、GNU Radioはデジタル信号処理のフレームワーク全体であり、プログラムは個別のモジュールから「アセンブル」されます。既製のブロックが多数あり、必要に応じて独自のブロックを作成することもできます。モジュール自体はC ++で記述されており、Pythonを使用して相互に対話します。 APIをより詳細に見ることができますが、実際にはこれはおそらく役に立たない可能性があります。すべてのアクションは、GNU Radio Companionプログラムで視覚的に実行できます。

システムはデータストリームの処理に重点を置いているため、通常、各ブロックには入力と出力があります。次に、エディターでブロックを接続して、既製のシステムを作成します。GNU Radioインターフェース自体は非常にシンプルで、ブロックが何をしているかを理解するのが困難です。前述のように、SDRを使用した低レベルの作業には高い入力しきい値があり、DSPと数学に関するある程度の知識が必要です。ただし、特別な知識を必要としない単純なタスクについて考えます。それでは始めましょう。

仕事の始まり


スクリーンショットに示すように、GNU Radio Companionを起動して、新しいプロジェクトを作成し、プロジェクトタイプWX GUIを選択して画面に追加し、2つのブロックを接続します。



ブロックには、ソース（ソース）とシンク（出力、「ドレイン」）の2つのタイプがあります。 RTL-SDRは当社のレシーバーであり、FFT GUIは仮想スペクトルアナライザーです。

サンプルレート変数は2048000に設定されています。これは、レシーバーのサンプリングレートです。デフォルトのRTL-SDR周波数は100 MHzです。

私たちはプロジェクトを開始します-すべてが機能し、さまざまなFMステーションが表示されます。 GNU Radioの最初のプログラムの準備が整いました！

FMラジオを受信する


次に、ステーションの1つを取得してみます。スクリーンショットからわかるように、受信機の中心周波数は100 MHzで、帯域幅は約2 MHzです。スペクトルには、それぞれ100.1 MHzと100.7 MHzの2つのステーションがあります。

最初のステップは、ステーションのスペクトルを中央に転送することです。これで、100KHzが右側になります。これを行うために、余弦を乗算するための学校の公式を思い出します-結果として、合計と差の2つの周波数があります-目的のステーションが中心に移動します。

周波数freq_center = 100000000とfreq_1 = 100100000を格納するための2つの変数を作成します。また、freq_center周波数を持つ信号発生器を追加します-freq_1。



なぜならシステムはPythonに基づいているため、パラメーター入力フィールドで式を使用でき、非常に便利です。

結果として、回路は次のようになります。



ここで、一度に複数のブロックを追加する必要があります-入力信号のクロック周波数を下げ（2048KHz）、信号をフィルターにかけ、それをFMデコーダーに適用し、クロック周波数を再び48KHzに下げます。

結果は図に示されてい



ます。慎重に検討します。 Rational Resamplerブロック（512KHzを取得）でクロック周波数2048KHzを4で割り、次にローパスフィルターの後に、デシメーション10（51.2KHzを取得）のWBFMデコーダーがあります。原則として、この信号はすでにサウンドカードに適用できますが、ピッチは少し異なります。ここでも、クロック周波数を48/51に変更します。その結果、クロック周波数は48.2 KHzになり、その差はすでに無視できます。

2番目の重要なポイントは、入力のタイプです。レシーバーから統合IQ信号（入力/出力は青色）、マテリアルシグナルはFMデコーダーから出力されます-入力と出力は黄色です。混同した場合、何も機能しません。Habréについてはすでに多くのことがありましたが、一般的な原則を理解するだけで十分です。

一般的に、実行して、すべてが機能することを確認します。プログラムを実行してラジオを聞くことができます。さらに進みます-まだSoftware Definedラジオがあります-2番目のステーションの同時受信を追加します。

記録


オリジナルの方法で2つのステーションを聴くが、実際にはあまり役に立たない。たとえば、サウンドレコーディングを別のファイルに追加するなど、さらに必要な作業を行います。これは非常に便利です。1つの物理レシーバーから複数のチャンネルを同時に録音できます。

スクリーンショットに示すように、各出力にFile Sinkコンポーネントを追加します。



何らかの理由でWindowsバージョンでは絶対ファイルパスが必要です。それ以外の場合、記録は機能しません。まず、すべてが正常であると確信しています。保存されたファイルのサイズは非常に大きいです。デフォルトの形式は浮動小数点です。 int形式で入力すると、読者は宿題のままになります。

結果のファイルはCool Editで開くことができ、サウンドが正常に録音されていることを確認します。





もちろん、記録されたチャンネルの数は増やすことができます、それは受信機の帯域幅とコンピュータのパワーによってのみ制限されます。File Sinkに加えて、UDP Sinkも使用できるため、プログラムをネットワーク上のブロードキャストに使用できます。

コマンドラインから実行する


そして最後。プログラムを自律的に、たとえばマルチチャンネルの記録に使用する場合、原則として、UIは必要ありません。オプションの左上のブロックで、[実行オプション]パラメーターを[UIなし]に変更します。プログラムを再度実行し、すべてが機能することを確認します。次に、生成されたファイルtop_block.pyを保存します。コマンドラインから、たとえばbatファイルやコンソールから実行できます。

        ##################################################
        self.samp_rate = samp_rate = 2048000
        self.freq_center = freq_center = 100000000
        self.freq_2 = freq_2 = 100700000
        self.freq_1 = freq_1 = 100100000

        ##################################################
        # Blocks
        ##################################################
        self.rtlsdr_source_0 = osmosdr.source( args="numchan=" + str(1) + " " + '' )
        self.rtlsdr_source_0.set_sample_rate(samp_rate)
        self.rtlsdr_source_0.set_center_freq(freq_center, 0)
        self.rtlsdr_source_0.set_freq_corr(0, 0)
        self.rtlsdr_source_0.set_dc_offset_mode(0, 0)
        self.rtlsdr_source_0.set_iq_balance_mode(0, 0)
        self.rtlsdr_source_0.set_gain_mode(False, 0)
        self.rtlsdr_source_0.set_gain(10, 0)
        self.rtlsdr_source_0.set_if_gain(20, 0)
        self.rtlsdr_source_0.set_bb_gain(20, 0)
        self.rtlsdr_source_0.set_antenna('', 0)
        self.rtlsdr_source_0.set_bandwidth(0, 0)

        self.rational_resampler_xxx_1_0 = filter.rational_resampler_fff(
                interpolation=48,
                decimation=51,
                taps=None,
                fractional_bw=None,
        )
        self.rational_resampler_xxx_1 = filter.rational_resampler_fff(
                interpolation=48,
                decimation=51,
                taps=None,
                fractional_bw=None,
        )
        self.rational_resampler_xxx_0_0 = filter.rational_resampler_ccc(
                interpolation=1,
                decimation=4,
                taps=None,
                fractional_bw=None,
        )
        self.rational_resampler_xxx_0 = filter.rational_resampler_ccc(
                interpolation=1,
                decimation=4,
                taps=None,
                fractional_bw=None,
        )
        self.low_pass_filter_0_0 = filter.fir_filter_ccf(1, firdes.low_pass(
        	1, samp_rate/4, 100000, 500000, firdes.WIN_HAMMING, 6.76))
        self.low_pass_filter_0 = filter.fir_filter_ccf(1, firdes.low_pass(
        	1, samp_rate/4, 100000, 500000, firdes.WIN_HAMMING, 6.76))
        self.blocks_multiply_xx_0_0 = blocks.multiply_vcc(1)
        self.blocks_multiply_xx_0 = blocks.multiply_vcc(1)
        self.blocks_file_sink_0_0 = blocks.file_sink(gr.sizeof_float*1, 'D:\\Temp\\1\\audio2.snd', False)
        self.blocks_file_sink_0_0.set_unbuffered(False)
        self.blocks_file_sink_0 = blocks.file_sink(gr.sizeof_float*1, 'D:\\Temp\\1\\audio1.snd', False)
        self.blocks_file_sink_0.set_unbuffered(False)
        self.audio_sink_0 = audio.sink(48000, '', True)
        self.analog_wfm_rcv_0_0 = analog.wfm_rcv(
        	quad_rate=samp_rate/4,
        	audio_decimation=10,
        )
        self.analog_wfm_rcv_0 = analog.wfm_rcv(
        	quad_rate=samp_rate/4,
        	audio_decimation=10,
        )
        self.analog_sig_source_x_0_0 = analog.sig_source_c(samp_rate, analog.GR_COS_WAVE, freq_center - freq_2, 1, 0)
        self.analog_sig_source_x_0 = analog.sig_source_c(samp_rate, analog.GR_COS_WAVE, freq_center - freq_1, 1, 0)

        ##################################################
        # Connections
        ##################################################
        self.connect((self.analog_sig_source_x_0, 0), (self.blocks_multiply_xx_0, 1))
        self.connect((self.analog_sig_source_x_0_0, 0), (self.blocks_multiply_xx_0_0, 1))
        self.connect((self.analog_wfm_rcv_0, 0), (self.rational_resampler_xxx_1, 0))
        self.connect((self.analog_wfm_rcv_0_0, 0), (self.rational_resampler_xxx_1_0, 0))
        self.connect((self.blocks_multiply_xx_0, 0), (self.rational_resampler_xxx_0, 0))
        self.connect((self.blocks_multiply_xx_0_0, 0), (self.rational_resampler_xxx_0_0, 0))
        self.connect((self.low_pass_filter_0, 0), (self.analog_wfm_rcv_0, 0))
        self.connect((self.low_pass_filter_0_0, 0), (self.analog_wfm_rcv_0_0, 0))
        self.connect((self.rational_resampler_xxx_0, 0), (self.low_pass_filter_0, 0))
        self.connect((self.rational_resampler_xxx_0_0, 0), (self.low_pass_filter_0_0, 0))

        self.connect((self.rational_resampler_xxx_1, 0), (self.blocks_file_sink_0, 0))
        self.connect((self.rational_resampler_xxx_1_0, 0), (self.audio_sink_0, 1))
        self.connect((self.rational_resampler_xxx_1_0, 0), (self.blocks_file_sink_0_0, 0))
        self.connect((self.rtlsdr_source_0, 0), (self.blocks_multiply_xx_0, 0))
        self.connect((self.rtlsdr_source_0, 0), (self.blocks_multiply_xx_0_0, 0))

結論


もちろん、GNU Radioは、ブロックを描画するという点ではなく、すべてがどのように機能するかを理解するという点で、かなり複雑なシステムであると言えます。しかし、いくつかの簡単なことをすることは非常に実行可能で興味深いものです。GNU Radioは、トレーニング用の「仮想実験室」としても便利です。仮想オシロスコープまたはスペクトルアナライザーを回路の任意の部分に接続して、信号がどのように見えるかを確認できます。

個別の願いがない場合、SDR受信のトピックはおそらく終了する可能性があります。すべての主要なポイントはすでに検討されており、第1パートから第3パートまでのビューの数はほぼ指数関数的に減少します（ただし、転送について書くことはできますが、テストにはRTL SDRよりも高価なハードウェアが必要です）。それでも、これがどのように機能するかについての理解が読者に残っていることを願っています。まあ、すべての成功した実験。
