Amazon AWS Marketplaceから3CXをインストールする



Amazonのクラウドサービスでは、追加の送信は必要ないと考えています。しかし最近まで、AWSへの3CX PBXのインストールは、特に初心者エンジニアにとっては困難でした。そのため、BYOL（持参ライセンス）の原則に基づいてAWSに迅速に展開されるターンキーソリューションとして認定され、3CXを配置しました。これは、経験豊富な専門家は、AWSインフラストラクチャのスキルを使用して高速化3CXを起動できるようになります、と初心者の管理者は、ランダムエラーをすることはありません。

開始3CXは、3つの簡単なステップにダウンしています：

  1. 無料の3CXライセンスを取得してAmazon AWSに登録する 
  2. VM PBXインスタンスのインストール
  3. ドキュメント3CXセットアップ

これらのステップを詳しく見てみましょう。

3CXライセンスの取得


3CXに登録するか、最寄りのパートナーにお問い合わせください。電子メールで無料の3CXライセンスを受け取ります。最初の40日間は、Proエディションのすべての機能で動作します。

VM PBXインスタンスのインストール


すでにアマゾンのアカウントをお持ちでない場合は、行くを通じて登録プロセス。ある段階で、Amazonから自動呼び出しを受信します。それに応答して、確認PINを入力する必要があります。次に、AWSコンソールに移動します。Amazon AWS

マーケットプレイスに移動し、3CXキーワードを検索します。 クリックして購読を続け、その後、設定を続行新しいインスタンスを構成します。 次に、[ このソフトウェアの構成]ページで、システムのソフトウェアバージョンが3CX 16以降であることを確認します。地域を確認または選択してください







。ホスティングの毎月の費用に注意してください。

次に、[ 起動して続行 ]をクリックします。起動時に、このソフトウェアのページへの注意を払う設定の詳細とで設定に必要なVMオプションEC2インスタンスタイプ、VPC設定、およびサブネット設定のセクション。

仮想マシンにパブリックIPアドレスが割り当てられていない場合は、[VPCダッシュボード]> [サブネット]セクションに移動し、このVMに割り当てられているVPC（仮想プライベートクラウド）で [ パブリックIPv4アドレスの自動割り当て ]オプションが有効になっていることを確認します。



ではセキュリティグループ設定のセクション、クリック[セラーの設定に基づいて新規作成]で、名前と説明を指定し、[ 保存 ]をクリックして新しいセキュリティグループを作成します。[キーペアの設定]



セクションで、既存のキーペアを選択するか、[ EC2でキーペアを作成する ]をクリックして新しいキーペアを作成します。名前を指定し、[ 作成 ]をクリックして証明書ファイルをダウンロードします（ファイルは再取得できないため、安全な場所に保存してください）。ペアを作成したら、[ このソフトウェアの起動 ] ページに戻り、ペアフィールドの近くにある更新アイコンをクリックして、作成したペアを選択します。 次に、「起動」をクリックして新しいインスタンスをインストールします。 VMをインストールした後、





EC2 Console > Instancesをクリックし、作成されたVMのインスタンスIDをクリックします。必要に応じて、名前の横にある[編集 ]をクリックして、VMの名前を変更できます。VMパブリックDNS（IPv4）

のパブリックDNS名をコピーし、ブラウザーで開きます（例：
example.eu-west-1.compute.amazonaws.com）。3CXユーザー名でログインします。パスワード=作成された仮想マシンのインスタンスID。

その後、ドキュメントに従って新しいPBXの構成を続行します。セットアップ中に、3TX HTTPS Webサービスポートを 5001または 443に設定し、HTTPポートを常に 5000に設定します。
