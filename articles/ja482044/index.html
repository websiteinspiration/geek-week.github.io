5. Dockerイメージに機密データを残さない


Dockerイメージ内にアプリケーションを作成するときに、プライベートリポジトリからコードを抽出するためのプライベートSSHキーや、クローズドパッケージをインストールするためのトークンなどの機密データが必要になる場合があります。これらを中間のDockerコンテナーにコピーすると、後で削除した場合でも、追加されたレイヤーにキャッシュされます。これらのトークンとキーは、外部に保存する必要がありますDockerfile。

マルチステージビルドを使用する


マルチステージビルドのDockerサポートを使用して、イメージの中間層のシークレットを操作します。シークレットはその後削除され、機密データが最終ビルドに到達しないようにします。次のコードを使用して、中間層に秘密データを追加します。

FROM: ubuntu as intermediate

WORKDIR /app
COPY secret/key /tmp/
RUN scp -i /tmp/key build@acme/files .

FROM ubuntu
WORKDIR /app
COPY --from=intermediate /app .

Dockerシークレットコマンドを使用する


Dockerのalpha関数を使用して機密データを管理し、機密ファイルをキャッシュせずにマウントします。

# syntax = docker/dockerfile:1.0-experimental
FROM alpine

# shows secret from default secret location
RUN --mount=type=secret,id=mysecret cat /run/secrets/mysecre

# shows secret from custom secret location
RUN --mount=type=secret,id=mysecret,dst=/foobar cat /foobar

機密データの管理について詳しくは、Docker Webサイトをご覧ください。

9.マルチステージアセンブリを使用して、小さく安全な画像を作成する


を使用してアプリケーションを作成するとDockerfile、ビルド時にのみ必要な多くのアーティファクトが作成されます。これらは、コンパイルに必要な開発ツールとライブラリ、または単体テストの実行に必要な依存関係、一時ファイル、シークレットなどです。

これらのアーティファクトを本番環境で使用できる基本的なイメージに保存すると、Dockerイメージのサイズが大きくなります。これは、ダウンロードにかかる時間に大きな影響を与える可能性があります。その結果、より多くのパッケージがインストールされるため、攻撃対象が増える可能性があります。同じことが、使用しているDockerイメージにも当てはまります。アプリケーションコードを実行するのではなく、ビルドするために特定のDockerイメージが必要になる場合があります。

Golangはその良い例です。Golangアプリケーションを作成するには、Goコンパイラが必要です。コンパイラーは、クリーンなイメージを含め、依存関係なしに、任意のオペレーティングシステムで実行される実行可能ファイルを作成します。

これが、Dockerが段階的に構築する機能を備えている正当な理由です。この機能を使用すると、アセンブリプロセス中にいくつかの一時的なイメージを使用して、コピーした情報とともに最後のイメージのみを保存できます。したがって、2つの画像があります。

  • 最初のイメージは非常に大きいサイズであり、アプリケーションの作成とテストの実行に使用される多くの依存関係が組み合わされています。
  • 2番目のイメージは、ライブラリのサイズと数の点で非常に軽量であり、アプリケーションを本番環境で実行するために必要なアーティファクトのコピーのみが含まれています。

10.リンターを使う


リンターを使用して、一般的なミスを回避し、エンジニアが自動的に実行できるベストプラクティスを確立します。

そのようなリンターの1つはhadolintです。Dockerfileを分析し、推奨事項に準拠しないエラーについて警告を出します。



Hadolintは、統合開発環境（IDE）で使用するとさらに強力になります。たとえば、hadolintをVSCode拡張機能として使用すると、入力中にリンティングエラーが表示されます。これは、最高のdockerfilesをより速く書き込むのに役立ちます。
