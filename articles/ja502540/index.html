X-SHOWおよびX-CLOAK


<div x-data="{ show: false }">
  <button x-on:click="show= ! show" x-text="show ? '' : '' ">
  </button>
  <p x-show="show" x-cloak>!</p>
</div>

上記の例では、2つの新しいディレクティブを一度に分析します。X-showは、x-ifと似ていますが、混同しないでください。X-showはページツリーを変更しませんが、要素の表示プロパティを制御するだけです。つまり、アイテムはレンダリングされますが、表示されません。


X-cloakは、スクリプトの開始直後に削除されるディレクティブです。どうやら、それは何が良いのでしょうか？実用的なアプリケーションを検討してください。上記のコードを実行すると、x-showディレクティブのfalse値にもかかわらず、「Hello！」というテキストを含む要素が一瞬表示されます。ここでx-cloakが役立ちます。彼に次のスタイルを書くだけです。


[x-cloak] { 
  display: none;
}

これで、スクリプトが開始するまで要素は非表示になります。


要素の突然の出現と非表示はそれほど明白ではないため、.transition修飾子を使用してこれらのイベントをアニメーション化します。

X-REF $REFS


このディレクティブは、コンポーネントの外部を含む、ページの任意の部分の目的の要素へのアクセスを提供します。すべてのリンクは、グローバル$ refsフィールドを通じて利用可能になります。例を使用してこの機能をデモンストレーションしましょう：


<div x-data="{ }">
  <input type="text" x-ref="myInput"/>
  <button x-on:click="alert($refs.myInput.value)">Alert</button>
<hr />
</div>

x-ref = "myInput"属性で入力に入力されたテキストは、$ refs.myInput.valueで使用できます
