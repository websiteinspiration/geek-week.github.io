病気だ！


Pythonでのロギングに関しては、すぐに頭に浮かびloggingます。


logging-Pythonエコシステムに緊密に組み込まれた堅牢で安定したソリューション。必要に応じてインポートし、いくつかの操作を行いlogger.exception('--')ます。それだけで、大事にしています。そして、エントリー'--'はある種のジャーナルに分類されます。


その後、通常、開発者は'--'このレコードで正確にどこで何が発生するかに関心があります。プログラマーはロガーの設定に入り込み、さまざまなオプションの助けを借りて、レコードをファイル、データベース、エラーコレクター、およびログエントリを保存できるその他の場所に適切にプッシュするように教え始めます。


ロガーの構成が表示されます。これは扱いにくいですが、多かれ少なかれ理解できます。


LOGGER_CONFIG = {
  "version": 1,
  "disable_existing_loggers": False,
  "formatters": {
    "simple": {
      "format": "%(asctime)s - %(name)s - %(levelname)s - %(message)s"
    }
  },
  "handlers": {
    "console": {
       "class": "logging.StreamHandler",
       "level": "DEBUG",
       "formatter": "simple",
       "stream": "ext://sys.stdout"
    },
  },
 "root": {
    "level": "INFO",
    "handlers": ["console"]
  }
}
