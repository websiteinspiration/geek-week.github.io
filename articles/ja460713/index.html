Redux、BindableObject、およびEnvironmentObject




私は約2年間Reduxと仕事をしてきたので、これは比較的よく知っています。特に、React Webサイトのフロントエンドで使用するほか、ネイティブiOS（Swift）およびAndroid（Kotlin）アプリケーションの開発にも使用しています。

SwiftUIでアプリケーションを構築するためのデータフローアーキテクチャとしてReduxを選択したことを後悔したことはありません。 UIKitアプリケーションでReduxを使用する際に最も困難なのは、ストアの操作、データの取得と取得、ビューやコンポーネントとの比較です。これを行うには、一種のコネクタライブラリ（ReSwiftとReKotlin）を作成する必要がありました。うまく機能しますが、かなりのコードです。残念ながら、それは（まだ）オープンソースです。

朗報！ SwiftUIで心配するのは、ストア、ステート、レデューサーだけです。 SwiftUIは、@ EnvironmentObjectのおかげで、ストアとの対話を引き継ぎます。したがって、ストアはBindableObjectから始まります。Reduxの基本的な使用法を提供

するシンプルなSwiftパッケージSwiftUIFluxを作成しました。私の場合、これはMovieSwiftUIの一部です。また、このコンポーネントの使用に役立つステップバイステップのチュートリアルも書きました。

使い方？

final public class Store<State: FluxState>: BindableObject {
    public let willChange = PassthroughSubject<Void, Never>()
        
    private(set) public var state: State
    
    private func _dispatch(action: Action) {
        willChange.send()
        state = reducer(state, action)
    }
}

アクションを開始するたびに、ギアボックスをアクティブにします。アプリケーションの現在の状態に従ってアクションを評価します。次に、アクションとデータのタイプに応じて、変更された新しい状態を返します。

まあ、storeはBindableObjectなので、PassthroughSubjectによって提供されるwillChangeプロパティを使用して、値の変更をSwiftUIに通知します。これは、BindableObjectがPublisherTypeを提供する必要があるが、プロトコル実装がそれを管理する責任があるためです。全体として、これはAppleの非常に強力なツールです。したがって、次のレンダリングサイクルで、SwiftUIは状態の変化に応じて表現の本文を表示するのに役立ちます。

実際、これがすべてです。SwiftUIの心と魔法です。これで、状態にサブスクライブされているすべてのビューで、状態から受信したデータと変更内容に応じてビューが表示されます。

そして今、あなたはアクションをアクティブにして、新しい状態を公開することを試みることができます。次に、より複雑な例を示します。

struct CustomListDetail : View {
    @EnvironmentObject var store: Store<AppState>
 
    let listId: Int
    
    var list: CustomList {
        store.state.moviesState.customLists[listId]!
    }
    
    var movies: [Int] {
        list.movies.sortedMoviesIds(by: .byReleaseDate, state: store.state)
    }
    
    var body: some View {
        List {
            ForEach(movies) { movie in
                NavigationLink(destination: MovieDetail(movieId: movie).environmentObject(self.store)) {
                    MovieRow(movieId: movie, displayListImage: false)
                }
            }.onDelete { (index) in
               self.store.dispatch(action: MoviesActions.RemoveMovieFromCustomList(list: self.listId, movie: self.movies[index.first!]))
            }
        }
    }
}

上記のコードでは、各IPに対してSwiftUIからの.onDeleteアクションを使用しています。これにより、リスト内の行に通常のiOSスワイプを表示して削除できます。したがって、ユーザーが削除ボタンに触れると、対応するアクションが開始され、リストからムービーが削除されます。

まあ、リストプロパティはBindableObjectの状態から派生し、EnvironmentObjectとして実装されているため、ForEachは計算されたムービープロパティに関連付けられているため、SwiftUIはリストを更新します。

これがMoviesStateレデューサーの一部です。

func moviesStateReducer(state: MoviesState, action: Action) -> MoviesState {
    var state = state
    switch action {
    
    // other actions.
    
    case let action as MoviesActions.AddMovieToCustomList:
        state.customLists[action.list]?.movies.append(action.movie)
        
    case let action as MoviesActions.RemoveMovieFromCustomList:
        state.customLists[action.list]?.movies.removeAll{ $0 == action.movie }
        
    default:
        break
    }
    return state
}

上記のように、アクションを送信して新しい状態を返すと、レデューサーが実行されます。
