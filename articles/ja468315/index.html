OK.RU APIで正しい署名（SIG）を生成する方法。エラー104で動作します

少し前に私は困難に遭遇しました。OK.RU（クラスメート）でAPIを要求したところ、正しいSIG（エラー104）を作成できませんでした。結局のところ、私だけではありませんでした。ネットワークで提供されているレシピはうまく機能しませんでした（pokeメソッドを使用して多くをチェックしました）。実際に機能しましたが、正しいクエリを正しく作成する方法がわからなかったので、SIGも正しいものでした。

具体的には、この投稿は正確に104 OK.RU APIエラーに特化しています。

PARAM_SIGNATURE	104

以下は、アルバムのリストを取得するための簡単なコードです。コードは少し松葉杖です（私は自分のために適切なクラスを作成しましたが、それは特定のものであり、誰にとってもあまり有用ではありません）。


	protected $params = array(
	'application_id' => '5*******6',
	'application_key' => 'C***********A',
	'session_secret_key' => '*************0'
	
	);
	
	protected $apiUrl = 'https://api.ok.ru/fb.do?';
	
	public function getalbums( Request $request ) {
		
		//access_token        .
		$userok = Userok::where('useroks.user_id', '=', $request->user_id)->first();
		
		$paramsArray = array(
		'application_key' => $this->params['application_key'],
		'format' => 'json',
		'method' => 'photos.getAlbums',
		);
		ksort($paramsArray);
		
		foreach($paramsArray as $k=>$v) {
			$paramsStr .= $k . "=" . $v;
		}
		
		
		
		/***    .   ,     (    ).
			
			$sig = strtolower(
			md5(
			$paramsStr)
			. md5(
			$userok->access_token
			.$this->params['session_secret_key']
			)
			);	
			
		**/
		
		
		$sectret_key = md5($userok->access_token.$this->params['session_secret_key']);
		$sig = md5($paramsStr.$sectret_key);
		
		
		$paramsArray['sig'] = $sig;
		$paramsArray['access_token'] = $userok->access_token;
		
		
		return $this->sendRequest(
		$this->apiUrl,
		$paramsArray,
		'POST'
		);
		
		
	}
	
	
	protected function sendRequest($url = '', $params = array(), $method = 'POST') {
		if(is_array($params)) {
			$params = http_build_query($params);
		}
		$ch = curl_init();
		if($method == 'GET') {
			$url .= $params;
			} else if($method == 'POST') {
			curl_setopt($ch, CURLOPT_POST, 1);
			curl_setopt($ch, CURLOPT_POSTFIELDS, $params);
		}
		curl_setopt($ch, CURLOPT_URL, $url);
		curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
		curl_setopt($ch, CURLOPT_FOLLOWLOCATION, 1);
		$result = curl_exec($ch);
		curl_close($ch);
		return json_decode($result, true);
	}

ただし、これを機能させるには、正しい$ paramsArrayが必要です。そうしないと、104番目のエラーが解決されますが、別のエラーが発生します。多くの初心者プログラマーはおそらく、開発者を支援するためのOK.RUの努力を無視するでしょう。

したがって、リンクhttps://apiok.ru/dev/methods/をたどってください。
