相対ロケーター


Selenium 4は私たちに相対ロケーター-Relative Locators（元はFriendly Locatorsと呼ばれていました）をもたらしました。この機能は、他のアイテムの隣にあるアイテムを見つけやすくするために追加されました。

利用可能なオプション：

  • 上（）：検索項目は指定された項目の上にあります
  • below（）：検索されたアイテムは指定されたアイテムの下にあります
  • toLeftOf（）：検索アイテムは指定されたアイテムの左側に配置されます
  • toRightOf（）：検索アイテムは指定されたアイテムの右側に配置されます
  • near（）：探しているアイテムは、指定したアイテムから50ピクセル以内にあります。距離を指定するためのオーバーロードメソッドもあります。

これらのメソッドはすべて、ByまたはWebElementを受け入れるためにオーバーロードされます。

相対ロケーターの使用


この書店アプリを 例として使用して、JavaのAdvanced Seleniumの左側にある本がJava For Testersであることを確認します。相対ロケータを使用すると、これを行うことができます。


これは、

メソッドは、オブジェクト（Byの子孫）を返すメソッドWebDriver::findElementを取ることができwithTagName()ますRelativeLocator.RelativeBy。

driver.findElement(withTagName("li")


ここで、すでに相対ロケーターを指定できます。「テスター向けのJava」は「Javaの高度なセレン」（pid6）の左側にあり、「実世界のテストオートメーション」（pid1）の下にあることを知っています。だから、私は両方から指摘することができます：

driver.findElement(withTagName("li")
                .toLeftOf(By.id("pid6"))
                .below(By.id("pid1")));

そして、テスター向けJava（pid5）を入手しました。

@Test
public void test_book5_is_left_of_book6_and_below_book1(){
    String id = driver.findElement(withTagName("li")
            .toLeftOf(By.id("pid6"))
            .below(By.id("pid1")))
            .getAttribute("id");
 
    assertEquals(id, "pid5");
}

我々は使用することができますabove()し、方法はtoRightOf()テストの自動化（PID2）の経験を見つけること：

@Test
public void test_book2_is_above_book6_and_right_of_book1(){
    String id = driver.findElement(withTagName("li")
                    .above(By.id("pid6"))
                    .toRightOf(By.id("pid1")))
            .getAttribute("id");
 
    assertEquals(id, "pid2");
}

SeleniumはJavaScript関数getBoundingClientRect()を使用して相対要素を検索します。この関数は、right、left、bottom、topなどの要素のプロパティを返します。

これら3冊の本の特性を見ると、実際のテストオートメーション（pid1）とJava For Testers（pid5）の両方が同じx軸の位置にあることがわかります。



したがって、どちらもJava For Testersの左側にあり、実際のテストオートメーション（pid1）が最初に見つかりました。
