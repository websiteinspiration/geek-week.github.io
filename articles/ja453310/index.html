RxJSライブラリを使用してReactコンポーネント間でデータを交換する

これは、blog.bitsrc.ioに公開されたChidume Nnamdiによる記事の翻訳です。翻訳は著者の許可を得て公開されています。



RxJSライブラリの出現により、JSの世界に多くの新しい可能性が開かれました。RxJSの目標は、少量のコードで多くを達成することです。この記事を読んだ後、RxJSの機能を使用してReact上のアプリケーションコンポーネント間でデータを交換する方法を学びます。

ヒント：ビットを使用して、Reactコンポーネントを整理および共有します。これにより、チームはアプリケーションをより迅速に開発できます。試してみてください。

関係のないReactコンポーネント間のデータ交換は、状態ライブラリが作成された目的です。状態管理テンプレートは多数ありますが、FluxとReduxの2つが最もよく知られています。

Reduxは、その単純さと純粋な機能の使用のために人気があります。実際、彼らのおかげで、レジューサーを使用しても副作用が生じないことは間違いありません。

Reduxを使用する場合、最初に行うのは、一元化されたデータウェアハウスを作成することです。



次に、コンポーネントをこのリポジトリに関連付け、必要に応じて状態を更新または削除します。ボールトに加えられた変更は、ボールトに関連付けられたコンポーネントに反映されます。したがって、データフローは、ネストの程度に関係なく、すべてのコンポーネントに拡張されます。階層構造のn番目のレベルにあるコンポーネントは、最上位レベルのコンポーネントにデータを送信できます。後者は、次に、コンポーネント21レベルにデータを送信できます。

Rxjs


RxJSの登場により、状態管理ライブラリの使用がはるかに簡単になりました。多くの人がRxJSによって提供されるObserverパターンを気に入っていました。

単にストリームObservableを作成し、すべてのコンポーネントがそれをリッスンできるようにします。コンポーネントがストリームに追加されると、リスナー（または「署名された」）コンポーネントがDOMの更新に応答します。

取り付け


を使用して、Reactでアプリケーションを作成しますcreate-react-app。ない場合はcreate-react-app、まずグローバルにインストールします。

npm i create-react-app -g

次に、Reactでプロジェクトを生成します。

create-react-app react-prj

ディレクトリに移動します。

cd react-prj

rxjsライブラリをインストールします。

npm i rxjs

新しいインスタンスを作成するファイルが必要BehaviourSubjectです。

BehaviorSubjectを使用する理由


BehaviorSubjectRxJSライブラリのサブジェクトの1つです。Subjectの子コンポーネントであるBehaviorSubjectため、多くのオブザーバーがストリームを聞くことができ、これらのオブザーバーにイベントを大量にブロードキャストすることもできます。BehaviorSubject最後の値を保存し、それをすべての新しい署名済みコンポーネントに渡します。

したがってBehaviorSubject：

  • 大量メールを許可します。
  • それは、サブスクライバーによって公開された最新の値を格納し、これらの値を大量メール送信します。



srcフォルダにはmessageService.js、インスタンスBehaviorSubjectとオブジェクトをサブスクライバにエクスポートするファイルが含まれていますmessageService。サブスクライバーオブジェクトはファイルの先頭に作成されるため、どのインポートコンポーネントでも使用できます。オブジェクトにmessageServiceは、パラメーターを取る送信関数がありますmsg。これには、すべてのリスニングコンポーネントへの送信に必要なデータが含まれています。関数の本体では、メソッドを呼び出しますemit。サブスクライバーオブジェクトの署名されたコンポーネントにデータを一括送信します。

次のコンポーネントがあるとします。

  • ConsumerA;
  • 消費者B;
  • プロデューサーA;
  • プロデューサーB。

階層構造では、次のようになります



。アプリケーションコンポーネントがメッセージをProducerAとConsumerBに送信します。 ProducerAはデータをConsumerAに送信し、ConsumerBからのメッセージはProducerBに送信されます。



ConsumerAおよびConsumerBコンポーネントには、個別のステータスカウンターがあります。彼らの方法では、componentDidMountそれらは同じスレッドにサブスクライブされていますsubscriber。イベントが発行されるとすぐに、両方のコンポーネントのカウンターが更新されます。

ProducerAとProducerBは持っているIncrement Counterと、ボタンDecrement Counterを押す、ということ、ディスプレイ1または-1。署名されたコンポーネントConsumerAとConsumerBがイベントを受け取り、それらのコールバック関数を起動して、ステータスカウンターとDOMの値を更新します。



階層構造をもう一度見てみましょう。



ProducerBは、ConsumerAデータを送信しますが、完全に無関係です。ProducerAは、親コンポーネントではなく、ConsumerBにデータを転送します。これがRxJSの要点です。中央のイベントフローノードを作成し、コンポーネントにそれをリッスンさせました。いずれかのコンポーネントがイベントを生成すると、リスニングコンポーネントはすぐにそれらをピックアップします。

結論


したがって、RxJSを使用してReactコンポーネント間でデータを交換する方法を確認しました。以前BehaviourSubjectは一元化されたデータストリームを作成し、残りのコンポーネントがこのストリームをサブスクライブできるようにしました。これで、コンポーネントの1つがデータを生成すると、他のコンポーネントもそれらを受け取ります。階層構造のコンポーネントのレベルは重要ではありません。
