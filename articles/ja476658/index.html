ビデオカードを備えたVPS（パート2）：コンピューティング機能

で、以前の私たちは新しいの話記事、VPSのサービスビデオカードを搭載した、私たちは、ビデオアダプタで仮想サーバーを使用していくつかの興味深い側面には触れませんでした。テストを完了する時間です。



仮想環境で物理ビデオアダプターを使用するために、MicrosoftハイパーバイザーでサポートされているRemoteFX vGPUテクノロジを選択しました。同時に、SLATをサポートするプロセッサー（IntelのEPTまたはAMDのNPT / RVI）と、Hyper-Vの作成者の要件を満たすビデオカードをホストにインストールする必要があります。このソリューションを物理マシンのデスクトップアダプターと比較するべきではありません。これは通常、グラフィックスで作業するときに最高のパフォーマンスを発揮します。私たちのテストでは、vGPUは仮想サーバーの中央プロセッサと競合します。これはコンピューティングタスクにとって非常に論理的です。また、RemoteFXに加えて、NVIDIA Virtual GPUなどの他の同様のテクノロジーがあることにも注意してください。これにより、ハイパーバイザーでブロードキャストせずに、各仮想マシンのグラフィックコマンドをアダプターに直接転送できます。 

テスト


テストでは、3.4 GHzの4つのコンピューティングコア、16 GBのRAM、100 GBのソリッドステートドライブ（SSD）、512 MBのビデオメモリを搭載した仮想ビデオアダプターを搭載したマシンを使用しました。プロフェッショナルNVIDIA Quadro P4000グラフィックスカードが物理サーバーにインストールされており、ゲストシステムがWindows Server 2016 Standard（64ビット）と標準のMicrosoft Remote FXビデオドライバーを実行している。


▍GeekBench5


まず、OpenCLアプリケーションのシステムパフォーマンスを測定できるGeekBench 5ユーティリティの最新バージョンを起動します。


以前の記事でこのベンチマークを使用したところ、明らかなことが確認されました。vGPUは、典型的な「グラフィック」問題を解決するための生産的なデスクトップビデオカードよりも弱いです。

▍GPUCaps Viewer 1.43.0.0


Geeks3D によって作成されたユーティリティは、ベンチマークとは言えません。パフォーマンステストは含まれていませんが、使用されているハードウェアおよびソフトウェアソリューションに関する情報を取得できます。ここでは、物理サーバーにNVIDIA Quadro P4000ビデオアダプターがインストールされているにもかかわらず、vGPU仮想マシンがOpenCL 1.1のみをサポートし、CUDAをサポートしていないことがわかります。


▍FAHBench2.3.1


Folding @ Home分散コンピューティングプロジェクトの公式ベンチマークは、タンパク質分子の凝固のコンピューターシミュレーションという高度に専門化された問題を解決することを目的としています。これは、欠陥タンパク質に関連する病理の原因を研究するために必要です-アルツハイマー病とパーキンソン病、狂牛病、多発性硬化症など。FAHBenchユーティリティは、仮想ビデオアダプターの計算能力を包括的に評価することはできませんが、CPUとvGPUのパフォーマンスを複雑な計算で比較できます。 


OpenCLを使用したFAHBenchを使用したvGPU計算のパフォーマンスは、かなり強力な中央プロセッサの同様のインジケーターよりも約6倍（暗黙的なシミュレーション手法の場合は約10倍）高いことがわかりました。

次に、倍精度の計算結果を示します。


▍SiSoftwareサンドラ20/20


コンピュータを診断およびテストするためのもう1つのユニバーサルパッケージ。これにより、サーバーのハードウェアおよびソフトウェアの構成を詳細に調査でき、膨大な数のさまざまなベンチマークが含まれています。CPUを使用したコンピューティングに加えて、Sandra 20/20はOpenCL、DirectCompute、CUDAをサポートしています。私たちは主に、Sandra Liteの無料バージョンに含まれているハードウェアアクセラレータを使用した汎用ベンチマーク計算ベンチマークテスト（GPGPU）に関心があります。 


結果はかなり良いですが、NVIDIA Quadro P4000ビデオアダプターの予想よりも少し低くなっています。仮想化のオーバーヘッドが影響を受ける可能性があります。


Sandra 20/20には、CPUの同様のベンチマークセットがあります。それらを実行して、結果をvGPU計算と比較します。


ビデオアダプターの利点ははっきりと見えますが、テストパッケージ全体の設定は完全に同一ではなく、結果では必要な詳細度のインジケーターを見ることができません。いくつかの個別のテストを実施することにしました。最初に、OpenCLを使用した一連の簡単な数学的計算を使用して、vGPU のピークパフォーマンスを決定しました。このベンチマークは、基本的にはCPUのSandraマルチメディア（算術ではない！）ベンチマークと同じです。比較のため、同じ図に中央プロセッサVPSのマルチメディアテストの結果を示します。 4コアのCPUでも、vGPUが大幅に失われます。


模擬試験から実用的なものまで合格します。暗号化テストは、データのエンコードとデコードの速度を判断するのに役立ちました。ここでは、vGPUと  CPUの結果を比較すると、アクセラレータの明らかな利点がわかりました。


vGPUのもう1つのアプリケーション分野は財務分析です。このような計算を並列化することは難しくありませんが、それらを実行するには、倍精度の計算をサポートするビデオアダプターが必要です。繰り返しになりますが、結果はそれ自体を物語っています。かなり強力なプロセッサはGPUを完全に失います。


私たちが最後に行ったテストは、高精度の科学計算でした。グラフィックスアダプターは、行列の乗算、高速フーリエ変換、その他の同様のタスクを備えた中央プロセッサーよりも上手く対応しました。


調査結果


vGPUは、グラフィックエディターの起動や、3Dレンダリングやビデオ処理のアプリケーションにはあまり適していません。デスクトップシステム用のアダプターはグラフィックスに非常によく対応しますが、仮想アダプターはCPUよりも高速に並列計算を実行できます。このため、生産的なRAMとより多くの算術論理モジュールに感謝しなければなりません。さまざまなセンサーからのデータ収集と処理、ビジネスアプリケーションの分析計算、科学計算と工学計算、トラフィック分析と料金計算、トレーディングシステムとの連携-GPUが不可欠な計算タスクがたくさんあります。もちろん、このようなサーバーは自宅でもオフィスでも構築できますが、ハードウェアとライセンスソフトウェアを購入するには、きちんとした金額を支払う必要があります。資本コストに加えて、メンテナンスのための運用コストがあり、電気代を含む。減価償却があります-機器は時間の経過とともに摩耗し、さらに早く陳腐化します。仮想サーバーにはこれらの欠点はありません。仮想サーバーは必要に応じて作成でき、計算能力の必要性がなくなったときに削除できます。必要なときにのみリソースに支払うことは、常に有益です。
