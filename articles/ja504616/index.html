Idle Detection APIを使用して非アクティブなユーザーを検出する

今日翻訳している記事は、新しいIdle Detection APIに関するものです。このAPIは、ユーザーがアクティブでないときに開発者に通知し、ユーザーがキーボード、マウス、画面を操作していないことを示します。APIは、スクリーンセーバーのアクティブ化、画面ロック、ユーザーの別の画面への遷移を報告します。この場合、開発者はユーザーアクティビティインジケーターのしきい値レベルを設定できます。これを達成すると、APIがトリガーされます。



アイドル検出APIの推奨される使用法


新しいAPIを使用できるプロジェクトの例を次に示します。

  • チャットアプリケーションまたはソーシャルネットワーキングサイトは、このAPIを使用して、通信相手が現在対応可能かどうかをユーザーに通知できます。
  • 美術館などの情報スタンドで実行されているパブリックアプリケーションは、このAPIを使用して、誰も情報スタンドで作業していない状況で「ホーム」画面に切り替えることができます。
  • 特定の図の表示など、リソースを大量に消費する計算を実行するアプリケーションは、ユーザーがデバイスを操作していないときに計算を制限できます。

アイドル検出APIの使用


Idle Detection APIを使用する際の最初のステップは、ページに権限があることを確認することnotificationsです。そのような許可がない場合は、ユーザーに要求する必要があります。

//    notifications.
const hasPermission = await navigator.permissions.query({name: 'notifications'});
if (hasPermission.state !== 'granted') {
  //    .
  return console.log('Notifications permission not granted.');
}

次に、インスタンスを作成しますIdleDetector。thresholdユーザーの非アクティブ時間のしきい値を設定するパラメーターの最小値は60,000ミリ秒（1分）です。start()オブジェクトのメソッドを呼び出すことにより、ユーザーの非アクティブを検出するシステムを起動できますIdleDetector。このメソッドは、thresholdミリ秒単位の値とオプションの値signal（AbortSignalオブジェクト）を含むパラメーターを持つオブジェクトを受け入れます。これにより、ユーザーアクティビティの監視を無効にできます。

try {
  const controller = new AbortController();
  const signal = controller.signal;

  const idleDetector = new IdleDetector();
  idleDetector.addEventListener('change', () => {
    const userState = idleDetector.userState;
    const screenState = idleDetector.screenState;
    console.log(`Idle change: ${userState}, ${screenState}.`);
  });
  
  await idleDetector.start({
    threshold: 60000,
    signal,
  });
  console.log('IdleDetector is active.');
} catch (err) {
  //   ,   ,
  //        .
  console.error(err.name, err.message);
}

AbortControllerオブジェクトのabort（） メソッドを呼び出すことにより、ユーザーの非アクティビティ検出を無効にできます。

controller.abort();
console.log('IdleDetector is stopped.');

セキュリティと権限


Chrome開発チームは、このドキュメントで定義されている基本原則に焦点を当てて、Idle Detection APIを設計および実装しました。これには、ユーザーがブラウザの機能を制御できるようにする原則や、透明性と人間工学に基づいたテクノロジーを保証する原則が含まれます。このAPIを使用する機能は、通知の許可によって制御されます。アプリケーションでこのAPIを使用するには、アプリケーションをトップレベルのセキュリティコンテキストで実行する必要があります。私たちが知っているユーザーのニーズを満たすためにこの許可を使用しましたが、将来的にこれを拡大する準備ができています。これがこのAPIを使用するシナリオに対応していない場合は、待機しています あなたのフィードバック。
