Lima tahun Karat

Dalam kekacauan yang sedang terjadi di dunia saat ini, mudah untuk melupakan bahwa lima tahun telah berlalu sejak rilis 1.0 pada 2015! Rust telah banyak berubah selama lima tahun ini, jadi kami ingin mengingat kembali pekerjaan semua anggota komunitas, mulai dari saat bahasa mulai stabil.


Ingat jika seseorang lupa: Rust adalah bahasa pemrograman untuk tujuan umum yang memiliki sarana untuk membangun perangkat lunak yang andal dan efisien. Karat dapat digunakan di bidang apa saja: dari kernel sistem operasi Anda ke aplikasi web Anda berikutnya. Bahasa ini sepenuhnya dibangun oleh anggota komunitas multi-segi terbuka, terutama sukarelawan, yang dengan murah hati berbagi waktu dan pengetahuan mereka untuk membantu membuat Rust seperti sekarang.


Perubahan besar sejak versi 1.0


2015


1.2 - Pembuatan Kode Paralel: Mengurangi waktu kompilasi selalu menjadi tema utama dalam setiap rilis Rust, dan sulit untuk membayangkan bahwa pernah ada periode waktu singkat ketika Rust tidak memiliki pembuatan kode paralel sama sekali.


1.3 - The Rustonomicon: edisi pertama kami dari buku fantastis "The Rustonomicon", sebuah buku yang mengeksplorasi sisi gelap Rust, Unsafe Rust, dan topik terkait. Ini telah menjadi sumber informasi yang sangat baik bagi siapa saja yang ingin belajar dan memahami aspek-aspek bahasa yang paling sulit.


1.4 - Dukungan untuk Windows MSVC level 1: memindahkan platform ke mendukung level 1 membawa dukungan asli untuk Windows 64-bit dengan toolkit Microsoft Visual C ++ (MSVC). Sebelum 1.4, Anda harus menginstal MinGW (port environment GNU untuk Windows) untuk menggunakan dan mengkompilasi program Anda di Rust. Dukungan Windows telah menjadi salah satu peningkatan terbesar Rust dalam lima tahun. Baru-baru ini, Microsoft mengumumkan pra-rilis publik dukungan resmi untuk Rust WinRT API! Dan akhirnya, sekarang jauh lebih mudah untuk membangun aplikasi lintas platform yang berkualitas tinggi daripada sebelumnya.
