3CX WebMeetingサービスの更新、Elastixオンラインコンバーター、新しいビデオチュートリアル

先週、3CXシステムの開発者向けに、いくつかの新製品とトレーニングビデオコースをリリースしました。しかし、まず最初に-本当にたくさんのニュースがあります！
 

3CX WebMeetingアップデート


無料の3CX WebMeetingビデオ会議サービスの次のリリースには、AndroidおよびiOS向けの新しいモバイルアプリケーション、セキュリティの強化、多数の新しいサーバープラットフォーム機能およびその他の改善が含まれています。更新されたWebMeetingがより安全で便利になりました。新機能について簡単に説明します。

  • TLS / SSLの強化。WebMeetingマイクロサービス用の拡張TLS / SSL暗号化-コンバーター、MCU、ポータル、およびファイル転送。
  • 長い会議。1つのWeb会議の期間が90時間に延長されます。つまり、ビジネスで必要な場合は、さまざまなタイムゾーンまたはユーザーグループのマラソン会議を開催できます。また、このソリューションは遠隔医療で需要があるかもしれません。
  • 高度なリモートコントロール。このリリースでは、参加者のPCのリモートコントロールがオンデマンドの「高速」会議に登場しました（以前は、スケジュールされた会議でのみ利用可能でした）。
  • 更新されたユーザーインターフェイス。使い慣れたビデオ会議インターフェイスも更新され、最適化されました。

また、AndroidおよびiOS向けのモバイルビデオ会議アプリにも力を入れました。主な変更点：

  • Google WebRTCの最新バージョンのサポート。アプリケーションは、最新バージョンのChromeおよびFirefoxに実装されている最新のWebRTC SDPスキームをサポートしています。
  • バッテリーの電力を節約します。複数の参加者がいるテレビ会議に接続すると、スマートフォンのバッテリー電力を3倍節約できました。

注意-更新された3CX WebMeetingサービスを使用するには、AndroidおよびiOSのアプリケーションの必須更新が必要です-古いアプリケーションは機能しません。リリース

の完全な変更ログ。

MCU WebMeeting可用性監視


ビデオ会議サーバー（MCU-マルチポイントコントロールユニット）の分散システムである3CX WebMeetingは、毎月10万件を超えるビデオ会議セッションに対応し、参加者の総数は最大20万人に達します。サーバーは世界中のデータセンターに配置されていますが、残念ながら、一部のサーバーは利用できない場合があります。問題について事前に知っているか、または最高のpingでMCUを選択できるように、WebMeetingサービスの可用性監視ページを作成しました。



監視の使用は非常に簡単です。マップ上で目的のサーバーにカーソルを合わせると、その可用性が表示されます。以下は、すべてのMCU WebMeetingのリストで、ユーザーからの最小のpingでソートされ、右側に表示されます。特定の機能の可用性に関する重要なニュースです。

特定のサービスのクラウドインフラストラクチャが100％の稼働率を保証すると言っても不当です。したがって、管理者は特定のノードの動作を手動で確認し、最も近いものが利用できない場合は別のノードを選択する機会を与えます。

  • 3CX Webクライアントでビデオ会議を作成するときに、レンチアイコンをクリックして、適切なMCUを選択します（提案されたMCUが利用できないか、pingが多すぎる場合）。
  • 3CXサーバーインターフェイスで、[設定]> [会議]に移動し、優先MCUを指定します。この場合、すべてのユーザー会議はデフォルトでこのMCUで作成されます。



FreePBX、Elastix、Askozia用のオンライン構成コンバーター


3CXが他のソフトウェアPBXメーカー（FreePBX、Elastix、Askozia）を買収したことをご存知でしょうか。これらのメーカーの古い電話交換機のユーザーが3CX（受け取ったテクノロジーの一部を含む）に簡単に切り替えることができるように、シンプルなオンライン構成コンバーターをリリースしました。



3CXに切り替えるには、古いPBXの構成をバックアップし、コンバーターにロードし、3CXで必要な内線番号を指定して、新しい構成をダウンロードするだけです。次に、3CX初期インストールウィザードでこの構成ファイルを指定します。このフォームに記入して、無料の3CXライセンスを事前に取得してください。

次のPBXがサポートされています。

  • Askozia-v2.x / v3.x / v4.x / v5.x（.xmlファイル）
  • Elastix-v2.5 / 4.0（.tarファイル）
  • FreePBX-v12 / v13 / v14（.tgzファイル）

変換すると、次のPBXオブジェクトが転送されます。

  • 内線番号（ユーザー）
  • SIPトランク
  • 受信および送信ルーティングルール
  • 音声メニュー（IVR）
  • ブロックリスト
  • PBXシステムコード

古い電話交換機がアナログまたはデジタルのテレフォニーカードを使用してPSTNに接続している場合は、物理的な「銅線」の回線をSIPトランクに転送することをお勧めします。しかし、何らかの理由で「銅」を拒否できない場合は、2つの解決策があります。

  • ElastixをVoIPゲートウェイとして3CXに接続する
  • 事前に構築されたアナログ/デジタルVoIPゲートウェイの取得

3CX Instance ManagerおよびCall Flow Designerビデオチュートリアル


3CX Instance Managerは、3CX v16のリリースで導入されました。これはかなり機能的な3CXサーバーのフリート管理ポータルです。クライアントの多くの自動電話交換をサポートする場合、または大規模な組織で企業のテレフォニーを管理する場合は、これを使用することをお勧めします。インスタンスマネージャは、すべての監視対象システムのステータスをすぐに表示します。多くの場合、そこからすぐに保守作業を開始できます。

クライアントPBXをインスタンスマネージャーに接続するには：

  1. PBXインターフェイスのメインページで、アフィリエイトIDをこの3CXインスタンスに関連付けます。
  2. 3CX管理インターフェースで、[設定]> [3CXインスタンスマネージャー]に移動し、必要なオプションを有効にします。ATSが監視されます！


3CX Call Flow Designer（CFD）は、音声アプリケーションの強力な開発環境であり、着信および発信の処理を自動化できることをご存じでしょう。アプリケーションは、既製のコンポーネントをドラッグアンドドロップすることによって作成されます。プログラミングなしで、または最小限のコード使用で実行できます。3CX CFDは3CX PROに無料で含まれています。

これで、ドキュメントとともに、開発者はCFDの作成者自身による開発環境のビデオチュートリアルを見ることができます。

  1. 最初の音声アプリケーション（コールフロー）を作成し、3CXサーバーにアップロードします。
  2. PBXへの非稼働時間のインストールや電子メールメッセージの送信など、既存の音声アプリケーションの変更。
  3. 新しいコンポーネントによる既存のプロジェクト（アプリケーション）の拡張。
  4. コールフローへの複雑なロジックの追加-IDによるデータベースでのユーザー認証。
  5. 検証アルゴリズムを使用してユーザー入力をチェックし、一連の番号をサブスクライバーに再生します。

音声アプリケーションの開発者向けに、 CFD作成者も責任を負うフォーラムの特別セクションをサポートしていることを思い出してください。
