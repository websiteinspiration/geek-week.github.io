React Nativeでのアニメーションの実装

この記事の翻訳は、特にReactJS / React Native-Developerコースの学生向けに作成されました。



アニメーションがないモバイルアプリケーションを想像することは困難です。Webでは、アニメーションは存在する場合でも通常は単純です（最大-1つのページが別のページを置き換える）。モバイルアプリケーションのアニメーションには、まったく異なる注意が必要です。

CSS3には、シンプルなアニメーションを作成できる比較的シンプルなAPIがあります。しかし、React Nativeにはこのツールはありません。はい、そうであっても、それは十分ではありません。

では、React Nativeを使用していて、本格的なアニメーションを実装する必要がある場合、何を知る必要がありますか？

React NativeにはAnimatedのようなツールがありますが、少なくとも最初は威圧的に見えます。

したがって、必要に応じて3つの「グループ」または「ブロック」があり、アニメーションを操作するために知っておく必要があります。

ブロック0：変化の必要性


アニメーションは、ある状態から別の状態への変化です。

  • 「非表示」から「表示」への移行。
  • 円から正方形への移行。

アニメーションの前後に何を表示するかを考えます。

1: , Animated.Value


論理的な観点から見ると、どのコンポーネントも表示または非表示にすることができます。これらは「true」と「false」の意味であり、ここに中間的な根拠はありません。状態として、オブジェクトがtrueまたはfalseのどちらであるかが表示されます。

優れたユーザーインターフェイスでは、オブジェクトが突然非表示になったり、突然表示されたりすることもありません。それらは徐々に表示され、ユーザーがインターフェースを理解するのに役立ちます。
したがって、視覚状態は、真の状態と偽の状態の間の何かである可能性があることが明らかになります。

どうして？
別の変数を導入して、視覚状態の範囲を表すことができます。これは数値である必要があります。論理状態がどのようなものであっても、数値は中間値を表すのに役立ちます。

this._shown = new Animated.Value(0);


論理状態は2進数（つまり、、trueまたはfalse、1または0）にすることができますが、表示状態は浮動小数点数です。

ブロック2：として知られている遷移 Animated.timing


一部のコンポーネントが非表示になっているとしましょう。これは、その可視性パラメーターの論理状態がfalseになり、表示状態も0.0になることを意味します。しかし、コンポーネントを表示したい場合はどうなりますか？論理状態はすぐにtrueになるはずですが、視覚状態は最初に0.1、0.2、...で徐々に通過し、最後に完全に1.0で通過するはずです。

これを行うには、視覚状態を1.0に遷移させる方法が必要です。
そして、そのような方法があります。実際には、いくつかあります。
最も簡単な方法は、基本的には次のとおりです。

Animated.timing(this._shown, {
  toValue: 1,
  duration: 300,
}).start();


ここでは、300ミリ秒の間隔でのAnimated変更_shownと言います1.0。
他のトランジションと、いくつかのトランジションを整理する方法がありますが、今はそれを使用できますAnimated.timing。

ブロック3：Animated.Viewと呼ばれるピクセルinterpolate


私たちの_shown間の移行は0.0、1.0それらが見えなければ何も意味しません。それでは、どうすればよいでしょうか。

何らかの方法で_shown子コンポーネントの透明度を設定する必要があります。

アニメーションで作業を始める前に、次のコードがあるとします。

<View style={{ opacity: this.state.shown ? 1 : 0 }}>
  <SomeComponent />
</View>


0コンポーネントを非表示に するときと表示するときの両方に透明度を設定1します。

もう1つ：補間


この言葉は恐ろしく聞こえますが、アイデア自体は非常に単純です。補間により、0と1のフレームワークで視覚的な状態を維持できますが、「比較」の可能性を他のものに与えることができます。

たとえば、継承されたコンポーネントを作成するだけでなく、コンポーネントを「上からドロップアウト」したいとします。そしてこれを行うには、非表示コンポーネントを40px高く配置し、状態が可視に変化したときに、その動きを目的の位置までアニメーション化します。

通常の呼び出しを使用して、0から1までの値を-40から0までの値と「一致」させることができますinterpolate。

const top = this._shown.interpolate({
  inputRange: [0, 1],
  outputRange: [-40, 0],
});


これは新しいものを作成しAnimated.Value、-40から0にされるの値
つまりは、それがあろう-40、それは場合_shown等しい0、-20と_shown= 0.5と0で_shown等しいです1.0。



暗い秘密：interpolateあなたの助けを借りて、色や度の値を変更することもできます。

調査結果：


  • 視覚的な状態は数値であり、あるスタイルから別のスタイルへの移行を反映する必要があります。
  • Animated.Value 視覚状態の数値を反映できます。
  • Animated.timingAnimated.Value別の番号に移動するために使用できます。
  • Animated.Value交換したときに、スタイルのために使用することができるViewことでAnimated.View。

補間を使用すると、1つの範囲をマッピングすることができAnimated.Value、例えば、別の範囲0の1範囲へ5の25、あるいは黒と緑との間の範囲にします。
