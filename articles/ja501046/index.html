確かな利点について：会議のトップ10レポートC ++ Russia 2019 Piter

C ++には真剣なアプローチが必要です。したがって、ほとんどのプラスは真面目な専門家であり、休暇中に家にいる必要がある場合、秘密の道でケバブに出かけることはありませんが、現時点では専門家として成長することを好みます。そして、これらの休暇中のそのような専門家は、C ++ Russia 2019 Piterカンファレンスからのトップ10レポートから利益を得るでしょう：



そしてもう一つの素晴らしいニュース！間もなくオンラインになるC ++ Russia 2020モスクワ会議で、伝説のBjarne Stroustrupが講演します。

10位：


「標準C ++ライブラリの内部」-Pavel Novikov


Pavelは、標準C ++ライブラリの実装のニュアンスを明らかにし、標準ライブラリの型とアルゴリズムの使用がアプリケーションのパフォーマンスにどのように影響するかを示しました。




9位：


「C ++ rvalueの生涯災害」-Arno Schoedl


Arnoは、非常に重要で危険なトピック、つまり右辺値オブジェクトの寿命について話しました。それらは、メモリ管理をより効率的にするために言語に導入されましたが、予期しない問題が発生しました：const＆パラメーターの形式で右辺値を渡す場合、および右辺値の寿命を一時的に延長する場合。これらは、アプリケーションメモリの複雑なデバッグ破損につながります。Arnoは問題の本質を説明し、言語を変更せずにライブラリコードを使用したソリューションを提供します。




8位：


「コンパイル時の正規表現の状態」-HanaDusíková


ハナは、コンパイル時間正規表現（CTRE）ライブラリの変更、彼女の新しいエンジン（有限オートマトン（FA））、および以前のエンジン（バックトラッキング）との比較について話しました。




7位：


コンパイル時のタイプのタグ付け-IvanČukić


C ++ロシアに関するIvanのレポートは常に非常に興味深く、有用です。前回のC ++ Russia 2019 Piterで、彼はC ++のタグ付け手法について話しました。レポートが何であるかわからない場合は、必ずレポートをチェックしてください。




6位：


「C ++での非同期の統一抽象化」-Eric Niebler


範囲ライブラリに関するエリックの作業により、最新のC ++が変更されました。彼の報告では、将来が非常に遅い理由、エグゼキュータとは何か、コルーチンとコールバックの間で一般的なこと、および抽象化「タスク」があらゆる種類の非同期アルゴリズムにうまく適合する理由を説明しました。




5位：


「コードをドライに保つための最新のテクニック」-BjörnFahller


会議での彼の講演で、BjörnFullerは、最新のC ++のさまざまなテクニックを収集しました。




4位：


「コンパイラー最適化のプリズムによるC ++例外」-Roman Rusyaev


Roman Rusyaevは、-fno-exceptionsを使用してアプリケーションをビルドするのが理にかなっている場合、およびC ++例外を使用できる場合に、コードがパフォーマンスを低下させていないことを示しました。例外の実装とコンパイラの最適化の例は、LLVMに基づいて検討されています。




3位：


「最新のC ++での型押し」-Timur Doumler


ティムールは、C ++ 20で登場した型パンニング（type punning）のテクニックについて、また、未定義の動作（未定義の動作）を引き起こす危険を冒さずにタイピングのpunをいつどのように使用できるかについて話しました。

Timurは、C ++での重要な概念についても説明しました。オブジェクトの存続期間（オブジェクトの存続期間）、その表現（オブジェクトの表現）、およびエイリアスルール（エイリアスルール）です。




2位：


「指定子、修飾子、テンプレート」-ミハイル・マトロソフ


彼のスピーチで、ミハイルは次のような質問に答えました：変数または関数に静的でインラインを指定する価値があるのはいつですか？constおよびconstexpr？constevalとconstinitはどうですか？これらはすべてテンプレートとどのように相互作用しますか？




1位：


「C ++タクシートリック-アントンポルキン」


C ++ Russia 2019 PiterのAnton Polukhinによる最高の最高のレポートを閉じます。これには、一般的なタスクに対する非定型かつ効果的なソリューションが含まれています。
-Pimplパターン-動的割り当てなし。
-ユーザータイプの高速ロギング-ただし、可能であればstd :: ostreamとロケールなし;
-JSON / XML / YAML / ...からユーザータイプへの変換-ADLとコードの再利用（ほとんどの場合、戻り値による関数のオーバーロード）。
