  • All photos will be stored in cloud storage .

Going serverless



Add logic for processing photos, which will be implemented as Azure Functions . Azure Functions is a programming model. Allows you to write functions that fire on events in the cloud.



Azure Function responds to events, such as the appearance of an object in the queue, and automatically communicates with storage and cloud objects. If we want to perform some action when a new photo is placed in the repository, a ready-made binary file is fed to the input of the corresponding function, and we only need to describe the logic of work.



Azure Function has two pricing modes. In the first case, the function works on a dedicated virtual machine, and in the second, we don‚Äôt think what, where and how is executed - the cloud will take care of this. It is this function that we will use.

Anatomy of Azure Function



Azure Function has two important parts. The first is a Python function __init__.py



. This function receives arguments and returns a response. 



def main(req:func.HttpRequest) -> func.HttpResponse:
    logging.info('Execution begins‚Ä¶')

    name = req.params.get('name')
    
    if name:
        return func.HttpResponse(f"Hello {name}!")
    else:

The second is a file function.json

{

        {

A request is submitted to the input, in which the GET and POST methods are used, and the function is triggered by an HTTP trigger. Inputs and outputs may be different. For example, as an output, a file in the storage can be used so that the function works invisibly inside the cloud, performing certain actions. Using Azure Function it is possible to build the entire microservice architecture of our application.
