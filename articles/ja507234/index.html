SnortまたはSuricata。パート2：Suricataのインストールと初期設定

統計によると、ネットワークトラフィックの量は毎年約50％増加します。これにより、機器の負荷が増加し、特にIDS / IPSのパフォーマンス要件が増加します。高価な専用ハードウェアを購入することもできますが、より安価な選択肢があります。オープンソースシステムの1つを導入することです。多くの初心者管理者は、無料のIPSをインストールして構成するのが難しいと感じています。Suricataの場合、これは完全に真実ではありません。インストールして、数分で一連の無料のルールを使用して一般的な攻撃を撃退することができます。


SnortまたはSuricata。パート1：企業ネットワークを保護するための無料のIDS / IPSを選択する

なぜ別のオープンIPSが必要なのですか？


長い間、標準と見なされているSnortシステムは90年代後半から開発されたため、もともとシングルスレッドでした。長年にわたって、IPv6サポート、アプリケーションレベルのプロトコルまたはユニバーサルデータアクセスモジュールの分析の可能性など、すべての最新チップが登場しています。

Snort 2.Xの基本エンジンは、複数のコアで動作することを学習しましたが、シングルスレッドのままであり、したがって、最新のハードウェアプラットフォームの利点を最適に使用できません。

問題はシステムの3番目のバージョンで解決されましたが、ゼロから作成されたSuricataがなんとか市場に出回るように準備されていました。2009年に、IPS機能をそのまま使用できるSnortに代わるマルチスレッドの代替として正確に開発を開始しました。コードはGPLv2ライセンスの下で配布されますが、プロジェクトの金融パートナーはエンジンのクローズドバージョンにアクセスできます。システムの最初のバージョンではスケーラビリティに関するいくつかの問題が発生しましたが、すぐに解決されました。

なぜスリカタ？


Suricataには（Snortのような）いくつかのモジュールがあります：キャプチャ、キャプチャ、デコード、検出、出力。デフォルトでは、キャプチャされたトラフィックは1つのストリームでデコードされる前に送信されますが、これによりシステムに負荷がかかります。必要に応じて、スレッドを設定で分割し、プロセッサ間で分散することができます。Suricataは特定のハードウェア向けに非常に最適化されていますが、これは初心者向けのHOWTOレベルではありません。 SuricataがHTPライブラリに基づく高度なHTTP検査ツールを備えていることも注目に値します。また、トラフィックを検出せずに記録するためにも使用できます。このシステムは、IPv4-in-IPv6、IPv6-in-IPv6トンネルなどのIPv6デコードもサポートしています。

トラフィックを傍受するためにさまざまなインターフェイス（NFQueue、IPFRing、LibPcap、IPFW、AF_PACKET、PF_RING）を使用できます。また、Unixソケットモードでは、別のスニファによってキャプチャされたPCAPファイルを自動的に分析できます。さらに、Suricataのモジュラーアーキテクチャにより、新しい要素を簡単に接続して、ネットワークパケットをキャプチャ、デコード、分析、および処理できます。 Suricataでは、オペレーティングシステムの標準フィルターによってトラフィックがブロックされることに注意することも重要です。 GNU / Linuxには、IPSを操作するための2つのオプションがあります。NFQUEUEキュー（NFQモード）とゼロコピー（AF_PACKETモード）です。最初のケースでは、iptablesに入るパケットはNFQUEUEキューに送信され、そこでユーザーレベルで処理できます。 Suricataは独自のルールでそれを実行し、NF_ACCEPT、NF_DROP、NF_REPEATの3つの判定のいずれかを行います。最初の2つの説明は必要ありません。最後の説明では、パケットにマークを付けて、現在のiptablesテーブルの先頭に送信できます。 AF_PACKETモードは、より高速であるという特徴がありますが、システムにいくつかの制限を課します。2つのネットワークインターフェイスがあり、ゲートウェイとして機能する必要があります。ブロックされたパケットは、2番目のインターフェースに転送されません。

Suricataの重要な機能は、Snortの開発を使用できることです。管理者は、特にSourcefire VRTとOpenSource Emerging Threatsルールセット、および商用Emerging Threats Proにアクセスできます。統一された出力は、一般的なバックエンドを使用して分析でき、PCAPおよびSyslogへの出力もサポートされています。システム設定とルールはYAMLファイルに保存され、読みやすく、自動的に処理できます。 Suricataエンジンは多くのプロトコルを認識するため、ルールのポート番号にバインドする必要はありません。さらに、フロービットの概念は、スリカータのルールで積極的に実践されています。応答を追跡するために、セッション変数を使用して、さまざまなカウンターとフラグを作成および適用します。多くのIDSは異なるTCP接続を別個のエンティティと見なし、それらの間の接続を認識しない場合があり、攻撃の始まりを示します。 Suricataは全体像を見ようとし、多くの場合、さまざまな接続に分散された悪意のあるトラフィックを認識します。長い間、その利点について話すことができます。インストールと構成に移ります。

インストールする方法？


Ubuntu 18.04 LTSを実行している仮想サーバーにSuricataをインストールします。すべてのコマンドはroot（ルート）として実行する必要があります。最も安全なオプションは、通常のユーザーの権限でSSH経由でサーバーに接続し、sudoユーティリティを使用して権限を増やすことです。まず、必要なパッケージをインストールする必要があります。

sudo apt -y install libpcre3 libpcre3-dev build-essential autoconf automake libtool libpcap-dev libnet1-dev libyaml-0-2 libyaml-dev zlib1g zlib1g-dev libmagic-dev libcap-ng-dev libjansson-dev pkg-config libnetfilter-queue-dev geoip-bin geoip-database geoipupdate apt-transport-https

外部リポジトリを接続します：

sudo add-apt-repository ppa:oisf/suricata-stable
sudo apt-get update

Suricataの最新の安定バージョンをインストールします。

sudo apt-get install suricata

必要に応じて、構成ファイル名を編集し、デフォルトのeth0を外部サーバーインターフェイスの実際の名前に置き換えます。デフォルト設定は/ etc / default / suricataファイルに保存され、ユーザー設定は/etc/suricata/suricata.yamlに保存されます。IDSの設定は、ほとんどの場合、この構成ファイルの編集に限定されています。名前と目的がSnortの対応物と一致する多くのパラメーターがあります。ただし、構文は完全に異なりますが、ファイルはSnortの構成よりもはるかに読みやすく、よくコメントされています。

sudo nano /etc/default/suricata



そして

sudo nano /etc/suricata/suricata.yaml



注意！開始する前に、varsセクションの変数の値を確認することをお勧めします。

設定を完了するには、suricata-updateをインストールして、ルールを更新およびダウンロードする必要があります。これを行うのは非常に簡単です：

sudo apt install python-pip
sudo pip install pyyaml
sudo pip install <a href="https://github.com/OISF/suricata-update/archive/master.zip">https://github.com/OISF/suricata-update/archive/master.zip</a>
sudo pip install --pre --upgrade suricata-update

次に、suricata-updateコマンドを実行して、Emerging Threats Openルールセットを設定する必要があります。

sudo suricata-update



ルールソースのリストを表示するには、次のコマンドを実行します。

sudo suricata-update list-sources



ルールのソースの更新：

sudo suricata-update update-sources



更新されたソースを再確認します。

sudo suricata-update list-sources

必要に応じて、利用可能な無料のソースを有効にすることができます。

sudo suricata-update enable-source ptresearch/attackdetection
sudo suricata-update enable-source oisf/trafficid
sudo suricata-update enable-source sslbl/ssl-fp-blacklist

その後、ルールを再度更新する必要があります。

sudo suricata-update

これに関して、Ubuntu 18.04 LTSでのSuricataのインストールと初期構成は完了したと見なすことができます。最も興味深い部分が始まります。次の記事では、仮想サーバーをVPNを介してオフィスネットワークに接続し、すべての送受信トラフィックの分析を開始します。DDoS攻撃、マルウェアアクティビティのブロック、およびパブリックネットワークからアクセス可能なサービスの脆弱性を悪用する試みに特に注意を払います。明確にするために、最も一般的なタイプの攻撃がモデル化されます。
