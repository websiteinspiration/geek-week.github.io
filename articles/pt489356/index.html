CI / CD na AWS, Azure e Gitlab. Novo curso da OTUS

Atenção! Este artigo não é de engenharia e destina-se a leitores interessados ​​em educação na área de CI / CD. Provavelmente, se você não estiver interessado em aprender, este material não será do seu interesse.



Se você é um desenvolvedor ou administrador responsável pela configuração de processos de integração contínua / entrega contínua , a OTUS abriu um conjunto de cursos especificamente para você: um curso intensivo prático sobre o método popular de desenvolvimento contínuo e entrega do software de Integração Contínua e Entrega Contínua nas plataformas Amazon Web Service, Azure, GitLab e Jenkins .

Durante o treinamento, os alunos aprenderão como configurar o processo de montagem e teste do aplicativo e o processo de instalação com três fornecedores líderes, além de entender a arquitetura dos provedores de nuvem e aprender a automação da análise de código e pesquisa de vulnerabilidades.

Como resultado do treinamento, cada aluno criará um trabalho de graduação, que será a implementação dos processos de CI / CD para qualquer projeto de código-fonte aberto. Após o treinamento, é claro, cada aluno receberá materiais para todas as aulas, um certificado de conclusão e, mais importante, configurará a montagem e o teste do aplicativo e poderá encontrar vulnerabilidades.



Claro, este curso não é para todos. Mas se você tem experiência :

  • Trabalhar com git
  • Administração de sistema Linux ou Windows
  • Desenvolvimento ou operação
  • Trabalhar com um provedor de nuvem

então OTUS está esperando por você! Você pode fazer um teste de entrada para determinar se possui conhecimento suficiente para concluir o curso de CI / CD na AWS, Azure e Gitlab.

Na véspera do início do curso “CI / CD para AWS, Azure e Gitlab” , o OTUS sediou o Open Day. O professor falou sobre o programa do curso em mais detalhes, respondeu a perguntas dos alunos e também descreveu o processo de aprendizagem.


Também está aberto o acesso a uma visualização gratuita do webinar aberto sobre o tópico "Usando Jenkins com K8S", conduzido por Boris Nikolaev , professor do curso de CI / CD sobre AWS, Azure e Gitlab :


O processo de treinamento no curso "CI / CD na AWS, Azure e Gitlab" ocorre no formato de seminários on-line. Durante o treinamento (e dura 3 meses), os alunos podem perguntar a professores experientes que estão sempre em contato. Os exercícios práticos serão concluídos usando o Google Cloud Platform (GCP), Amazon Web Service e Microsoft Azure.

O programa do curso consiste em quatro módulos principais:

  1. Desenvolvimento na nuvem (Código)
  2. Automação de montagem e teste (Integração Contínua)
  3. Automação de Instalação (Entrega Contínua)
  4. Módulo Resumo

Cada um deles será totalmente compreendido nas aulas no formato de seminários on-line, e a lição de casa ajudará a consolidar o conhecimento adquirido, de acordo com o qual, se necessário, você poderá obter um feedback detalhado dos professores.

Muitos especialistas chamam o CI / CD de uma das melhores técnicas de desenvolvimento de software para tarefas modernas. Você está de acordo com esta afirmação?
