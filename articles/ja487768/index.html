Xiaomi Gateway 2でポート4321および9898を開く

前書き


年末年始で休むと、「スマートホーム」を作るというアイデアにワクワクしました。 「人気のある」エコシステムの中で、あらゆる種類の接続デバイスが安価で豊富であるために最も人気のあるエコシステムは、Xiaomiのソリューションのように思えました。中国のサーバー（遅延、セキュリティ、同じメーカーのデバイスへのバインドなど）による家の管理をすぐに放棄し、あらゆる種類のスマートホームコントロールシステムを研究し、互換性のある機器を選択し始めました。

何時間も検索した結果、ホームアシスタントが自分に適していると判断しました。最初の段階で、Raspberry Pi 4 Model B（2GB）とXiaomiのセンサーを使用してすべてを構築することにしました。ゲートウェイを購入した（製品番号DGNWG02LMのバージョンが必要））そしてLAN通信プロトコルをオンにすると、ホームアシスタントへの統合に必要なポート4321および9898がまだ閉じられたままであるという事実に直面しました。

コマンドnmap -sU -Pn <ゲートウェイIP> -p 9898,4321,54321を使用して、Nmap からこれを確認できます



。この記事では、これらのポートを開く方法と、それが何を行うかについて説明します。

楽しい部分に行く


あなたはあなた自身のリスクですべてのさらなる行動を実行します。ゲートウェイを開くと、保証が無効になります。

最初のステップは、ネジを覆う3つのゴムパッドを取り外すことです。パッドは接着されていません。薄いものでこじってください：私はナイフを使いました。



パッドの1つの下に保証シールがあります。これはノーリターンのポイントです。
ネジを緩め、ゲートウェイを静かに開きます。その上部にはスピーカーが下部のボードにワイヤーで接続されています。何も干渉しないようにワイヤを外します。



ゲートウェイボードに3本のケーブルをはんだ付けする必要があります。 2点ですべてが明確になり、3点目はボタンの側面にあります。写真に示されている場所：



次のようなものが表示されます。



次のステップは、コンバーターの接続を構成することです。コンバーターからジャンパーを取り外し（私が購入したとき、それはVCCおよび3V3にありました）、USB 2.0ポートに接続します。

Windows 10では、ドライバーが自動的に読み込まれ、COM3ポートの接続がデバイスマネージャーに表示されました（覚えておいてください。後で便利になります）。



デバイスのプロパティで、[ポート設定]タブに移動して値を変更します。

  • 115200での「ビット/秒」
  • 「フロー制御」から「いいえ」

念のため、スクリーンショットを参照してください：



完了したコンバーターの構成で、USBポートから引き出します。ゲートウェイの接続に進みます。TXをRXコンバーターに接続し、GNDをGNDに接続します。RXゲートウェイはまだ接続していません。

ゲートウェイを電源コンセントに接続します。すべてが正常に完了すると、青色に点滅し始めます。完全なダウンロードを待って（点滅は停止します）、コンバーターをセットアップ段階と同じUSBポートに接続します。

管理者としてPuTTYを実行します。 [セッション]セクションで、[ 接続タイプ]をSerialに変更します。フィールド「シリアルライン」と「速度」は、以前に記憶したポート（私の場合-COM3）、および115200を示します



。「シリアル」セクションで、COMポートが指定されていることを確認します。「速度（ボー）」 - 115200および「フロー制御」 - なし。



「セッション」セクションに戻り、ゲートウェイに接続します。すべてが正しく行われた場合、読みやすいテキストがコンソールに表示されます。私にはこのように見えました。



次に、RXゲートウェイをTXコンバーターに接続する必要があります。コンソールにエラーはないはずです。読みやすいテキストが引き続き表示されますが、今回はJSON形式です。psm-set network.open_pf 3

コマンドを入力しますEnterキーを押します。

私がフォローしたいくつかのビデオや記事では、いくつかの回答がコンソールに表示されるはずであると書かれています。コマンド入力後、1分ほど待ったが返事がなかった。コンソールで、新しいエントリがときどきフラッシュされるだけでした。したがって、念のため、コマンドに対して他に3つのオプションを試しました。

  • psm-get network open_pf 3（ピリオドの代わりにスペース）
  • ＃psm-get network.open_pf 3（コンソールのエントリの後、Enterキーを押して＃が表示されるようにし、コマンドを記述します）
  • ＃psm-get network open_pf 3（前のものと同様ですが、スペースがあります）

4回回答した後、私は待つことなく、ポートが開いているかどうかを確認することにしました。これを行うには、コンバーターの電源を切り、ゲートウェイをソケットから引き出して再起動します。コンバーターをコンピューターに接続せずに、ゲートウェイをオンにしてダウンロードを待ちます。その後、nmap -sU -Pn <GATEWAY IP> -p 9898,4321,54321コマンドを使用してポートを確認します。私の場合、彼らは開いていました！
