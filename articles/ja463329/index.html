SVGとVue.jsを使用した動的ツリー図の開発

本日翻訳する資料は、動的ツリー図の視覚化システムの開発プロセスに特化しています。3次ベジェ曲線の描画には、SVGテクノロジー（Scalable Vector Graphics、スケーラブルベクターグラフィックス）がここで使用されます。データを使用した事後処理は、Vue.jsによって編成されます。

これは、実験可能なシステムのデモ版です。


インタラクティブツリーチャート

SVGの強力な機能とVue.jsフレームワークの組み合わせにより、データベースでインタラクティブでカスタマイズ可能なダイアグラムを構築するためのシステムを作成できました。

ダイアグラムは、1点から始まる3次ベジェ曲線のコレクションです。曲線は、互いに等距離にある異なる点で終了します。最終的な位置は、ユーザーが入力したデータによって異なります。その結果、グラフはデータの変化に反応的に対応できます。

最初に、ベジェ3次曲線がどのように形成されるかについて説明し、次に要素の座標系でそれらを表す方法を理解し<svg>、画像のマスクを作成する方法について説明します。

資料の作者は、彼女が彼のために多くのイラストを用意し、彼を理解可能で興味深いものにしようとしていると言います。資料の目的は、誰もが独自のチャート作成システムを開発するために必要な知識とスキルを得るのを助けることです。

Svg


▍3次ベジェ曲線はどのように形成されますか？


このプロジェクトで使用される曲線は、3次ベジェ曲線と呼ばれます。次の図は、これらの曲線の主要な要素を示しています。


3次ベジエ

曲線の主要要素曲線は、4組の座標で記述されます。最初のペア(x0, y0)は、曲線の開始アンカーポイントです。最後の座標のペア(x3, y3)は、終了基準点です。

これらの点の間には、いわゆる制御点があります。ポイント(x1, y1)とポイント(x2, y2)です。

制御点に対する制御点の位置によって、曲線の形状が決まります。曲線は、開始点と終了点によってのみ設定された場合、座標(x0, y0)とは(x3, y3)、この曲線は対角線上に位置する直線セグメントのように見えるであろう。

次に、上記の4つの点の座標を使用して、SVG要素を使用して曲線をプロットします。<path>。次の構文は、要素で使用され、<path>3次ベジェ曲線を作成します。

<path D="M x0,y0  C x1,y1  x2,y2  x3,y3" />

コードに表示される 文字は、Cubic Bezier Curveの略です。小文字（c）は相対値の使用を意味し、大文字（C）は絶対値の使用を意味します。絶対値を使用して図を作成していますが、これは例で使用されている大文字で示されています。

a対称図の作成


対称性は、このプロジェクトの重要な側面です。対称的な図を作成するために、1つの変数のみを使用して、特定のオブジェクトの中心の高さ、幅、または座標などの値に基づいて受け取りました。

この変数を呼び出しましょうsize。ダイアグラムは水平方向を向いているので、変数sizeは、ダイアグラムにアクセスできる水平空間全体と見なすことができます。

この変数に現実的な値を割り当てます。この値を使用して、グラフ要素の座標を計算します。

size = 1000

curve曲線パラメータの計算


曲線の作成に必要な値のほとんどは単一の変数（size）に基づいて計算されるため、計算されたプロパティを使用してすべての固定座標を見つけました。ここで「定数座標」と呼ぶものはに基づいて計算され、sizeその後は変化せず、ダイアグラムに含まれる曲線の数に依存しません。

あなたがいる場合size、それを変更- 「一定の座標は」詳述されます。その後は、次の変更までsize変更されません。上記を前提として、ベジェ曲線を描くために必要な5つの値を次に示します。

  • topHeight — size * 0.2
  • bottomHeight — size * 0.8
  • width — size
  • halfSize — size * 0.5
  • distance — size/arrayLength

これで、不明な意味が2つだけ残っています- x2とx3。すでに導出したそれらを計算するための式：

x = index * distance + (distance * 0.5)

特定の値を見つけるには、この数式の配列要素のインデックスを代入する必要があります。

次に、計算されたプロパティがを見つけるのに適しているかどうかを考えてみましょうx。簡単にこの質問に答えてください-いいえ、それはしません。

計算されたプロパティにパラメータを渡すことはできません。実際には、これは関数ではなくプロパティです。さらに、何かを計算するためにパラメーターを使用する必要があるということは、キャッシュに関して計算されたプロパティを使用する明白な利点がないことを意味します。

上記の原則には例外がありますのでご注意ください。 Vuexについてです。関数を返すVuexゲッターを使用する場合は、パラメーターをパラメーターに渡すことができます。

この場合、Vuexは使用しません。しかし、この状況でも、この問題を解決する方法はいくつかあります。
