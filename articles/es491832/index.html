8 ventajas de Flutter en comparación con React Native

El desarrollo móvil no depende cada vez más del desarrollo nativo, sino del uso de marcos que ayudan a crear aplicaciones para múltiples plataformas a la vez. Hoy quiero decir algunas palabras sobre Flutter, que gradualmente está comenzando a ganar el mercado de React Native. Detalles - debajo del corte.



Cuando comienza el proyecto para desarrollar una aplicación móvil, es necesario tomar una decisión sobre qué plataforma programaremos. A menudo, esta pregunta está influenciada por el propio cliente, que quiere ver el software creado de forma nativa para cada plataforma o, por el contrario, quiere ahorrar dinero y pide desarrollarse en React Native.

En el sitio web del proyecto puede descubrir que Flutter está utilizando activamente no solo Google, sino también algunas otras compañías. Por ejemplo, Flutter ha escrito la nueva aplicación The New York Times, así como las versiones móviles de eBay y Aliexpress. Pero, en general, la lista no parece muy impresionante, ya que incluye no más de 20 empresas.

Sin embargo, la experiencia de los desarrolladores de nuestra empresa muestra que, en algunos casos, Flutter es mejor y más conveniente para crear aplicaciones móviles. La mente colectiva de Neti totalizó 8 más de este marco más joven.

Actuación


Las aplicaciones desarrolladas por Flutter se ejecutan mucho más rápido que con React Native. Las ventajas de velocidad se logran a través de nuestro propio motor y el uso de otro lenguaje de programación. A diferencia de React Native, que funciona con puentes y el lenguaje Java Script, Flutter le permite resolver rápidamente problemas de aplicaciones, especialmente aquellos relacionados con la interfaz de usuario. Los compiladores nativos ya están integrados en el marco, y el código escrito en Dart se convierte en instrucciones nativas para procesadores ARM.

Por cierto, los chicos de InVerita llevaron a cabo interesantes pruebas comparativas. Su artículo sobre Medium comparó el rendimiento de las aplicaciones terminadas en dispositivos iPhone 6 y Xiaomi Redmi Note 5 reales.



Como puede ver, la prueba de Gauss en iOS se ejecuta en Flutter incluso más rápido que en Swift, ¡y RN muestra un retraso de 20 veces!



En el caso de Android, Flutter requiere un 20% más de tiempo para completar la prueba que Java y Kotlin, pero aún funciona 15 veces más rápido que RN.

Tamaño de la aplicación


Como resultado del párrafo anterior, las aplicaciones creadas sobre la base de Flutter y escritas en Dart contienen menos componentes intermedios. La compilación de instrucciones ARM nativas le permite almacenar nada más. Como resultado, los binarios de la aplicación ocupan hasta dos veces menos espacio en la unidad del dispositivo que cuando se desarrollan en React Native.

Animación rápida


En cierto sentido, esta ventaja se deduce de la anterior. Decidimos usar Flutter para aplicaciones que tienen mucha animación incorporada. El hecho es que Flutter demuestra de manera estable 60 FPS, lo que significa que los usuarios no tienen que mirar la imagen temblorosa, como suele ser el caso con React Native.

Lenguaje de Programación Mecanografiado


La base de Flutter es Dart, un lenguaje de programación mecanografiado que ya es popular entre los entusiastas. En Dart, la codificación se convierte en programación orientada a objetos ... y aquí, por supuesto, podríamos recordar TypeScript y argumentar que también está escrito. Pero, por desgracia, en última instancia, se transcompila y todavía se convierte en JavaScript sin tipo. Es decir, todas las ventajas de TS permanecen en el nivel de conveniencia del codificador y no afectan el rendimiento. Dart, por el contrario, se escribe de forma nativa. El lenguaje de programación se desarrolla activamente, con el apoyo de la comunidad. Puede escribir mucho más código estructural en Dart, lo que significa que puede crear aplicaciones más complejas y estructuras jerárquicas.

Excelente documentación


Para aquellos que quieren lidiar con la nueva plataforma en Internet, hay tutoriales en video, las posibilidades para trabajar con código en línea están pensadas. La comunidad dirigida por Google ha desarrollado instrucciones y guías integrales. Además, la red incluso tiene una receta lista para cambiar de React Native a Flutter. Puedes verlo aquí . Por supuesto, también hay documentación para React Native, pero a menudo es inútil debido a descripciones como "removeClippedSubviews - Esto puede mejorar el rendimiento de desplazamiento para listas grandes". La comunidad de Flutter tiene un enfoque mucho más serio de la documentación.

IU de uniformidad


La conocida ventaja de React Native es la capacidad de llevar a cabo el desarrollo de varias plataformas a la vez. Sin embargo, nadie garantiza que sin finalizar con un archivo obtendrá de inmediato la misma interfaz. Una aplicación escrita en Flutter, por el contrario, se ve y funciona igual en iOS y Android. Nuestros programadores nunca han reescrito el código para que la aplicación funcione en iOS al igual que en Android. En el caso de React Native, todo es diferente. Dichas mejoras deben hacerse con envidiable regularidad.

Soporte para una gran cantidad de plataformas.


Flutter ya es compatible con Android, iOS y Web. A juzgar por las discusiones en la comunidad, el soporte para MacOS, ChromeOS y Windows estará disponible muy pronto. Y, lo más interesante, Flutter ya es compatible con la futura plataforma Google Fuchsia. Por lo tanto, cualquier desarrollo puede ser portado inmediatamente al nuevo sistema operativo.

Herramientas de desarrollo


Lo último que quiero señalar es el DevTools desarrollado. Flutter cuenta con un buen ecosistema de herramientas para desarrolladores. Puedes verlos aquí . Por cierto, Facebook está tratando de hacer algo similar, pero hasta donde podemos ver, RN solo tiene una versión beta de DevTools hasta ahora, y por funcionalidad no llegan a las contrapartes de Flutter.

Ejemplo de código


Por ejemplo, a continuación se muestra el código "¡Hola, mundo!" para los marcos React Native y Flutter

// React Native
import React from 'react';
import { StyleSheet, Text, View } from 'react-native';

export default class App extends React.Component {
  render() {
    return (
      <View style={styles.container}>
        <Text>Hello world!</Text>
      </View>
    );
  }
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center'
  }
})

Y así es como se verá este código en Flutter

// Flutter
import 'package:flutter/material.dart';

void main() {
  runApp(
    Center(
      child: Text(
        'Hello, world!',
        textDirection: TextDirection.ltr,
      ),
    ),
  );
}

¿Flutter tiene alguna desventaja?


Por supuesto que lo son. Flutter es una plataforma joven. No hay suficientes soluciones preparadas para ello. A modo de comparación, el mismo React Native ya tiene muchas más plantillas, recetas y opciones de implementación listas para usar.

Escribir una IU en Flutter lleva más tiempo. Y la escritura misma del código Dart es más larga. En React Native, una aplicación se crea como un constructor de componentes web. Flutter todavía requiere una programación completa.

Además, Flutter requiere habilidades de programación más profundas. En el caso de React Native, puede confiar el desarrollo a cualquier especialista con conocimiento de Java Script. Con Dart, las cosas son más complicadas, ya que este lenguaje tiene un umbral de entrada más alto que JS. El programador debe tener una formación más profunda.

Pero al mismo tiempo, las vacantes para desarrolladores en Flutter son aún bastante pequeñas. Todavía no hay confianza por parte de las empresas rusas en la nueva plataforma. Sin embargo, esto no nos impide llevar a cabo parte del desarrollo de este marco. Y los resultados hasta ahora son más que positivos. Si tiene experiencia desarrollando en Flutter, comparta sus impresiones en los comentarios.
