Snort或Suricata。第3部分：保护办公室网络

在上一篇文章中，我们讨论了如何在Ubuntu 18.04 LTS上运行稳定版本的Suricata。在一个节点上配置IDS并连接免费规则集非常简单。今天，我们将探讨如何使用虚拟服务器上安装的Suricata保护企业网络，这是最常见的攻击类型。为此，我们需要Linux上具有两个内核的VDS。RAM的数量取决于负载：2 GB足以容纳某人，而对于更严重的任务则可能需要4甚至6个内存。另外，虚拟机也可以进行试验：您可以从最小的配置开始，并根据需要增加资源。

进行测试可能主要需要将IDS删除到虚拟机。如果您从未处理过此类解决方案，则不要急于订购物理硬件并更改网络体系结构。最好安全地运行系统，而无需花费额外费用来确定计算需求。重要的是要了解所有企业流量都必须通过单个外部节点传递：要将局域网（或多个网络）连接到安装了IDS Suricata的VDS，您可以使用SoftEther（SoftEther），这是一种易于配置的跨平台VPN服务器，可提供强大的加密功能。办公室的Internet连接可能没有真正的IP，因此最好将其升级到VPS。 Ubuntu存储库中没有现成的软件包，您将必须通过以下方式下载该软件：项目站点，或者从Launchpad服务上的外部存储库中（如果您信任他）： 



sudo add-apt-repository ppa:paskal-07/softethervpn
sudo apt-get update


您可以使用以下命令查看可用软件包的列表： 



apt-cache search softether






我们将需要softether-vpnserver（测试配置中的服务器在VDS上运行），以及softether-vpncmd-命令行实用程序来对其进行配置。 



sudo apt-get install softether-vpnserver softether-vpncmd


要配置服务器，使用特殊的命令行实用程序： 



sudo vpncmd






我们不会详细讨论该设置：此过程非常简单，在许多出版物中都有很好的描述，并且不直接适用于本文的主题。简而言之，启动vpncmd之后，您需要选择第1项以转到服务器管理控制台。为此，请输入名称localhost并按Enter键，而不要输入集线器的名称。使用serverpasswordset命令在控制台中设置管理员密码，删除虚拟的DEFAULT集线器（hubdelete命令），并使用名称Suricata_VPN创建一个新的密码，并设置其密码（hubcreate命令）。接下来，使用hub Suricata_VPN命令转到新集线器的管理控制台，以使用groupcreate和usercreate命令创建组和用户。使用userpasswordset设置用户密码。



SoftEther支持两种流量传输模式：SecureNAT和本地网桥。第一种是专有技术，用于使用其自己的NAT和DHCP构建虚拟专用网络。 SecureNAT不需要TUN / TAP，也不需要设置Netfilter或其他防火墙。路由不会影响系统的核心，并且所有进程都已虚拟化并且可以在任何VPS / VDS上运行，而不管所使用的管理程序如何。与将SoftEther虚拟集线器与物理网络适配器或TAP设备连接的本地桥接模式相比，这导致处理器负载增加和速度降低。



这种情况下的设置很复杂，因为路由是使用Netfilter在内核级别完成的。我们的VDS建立在Hyper-V上，因此在最后一步中，我们创建一个本地网桥，并使用bridgecreate Suricate_VPN -device：suricate_vpn -tap：yes命令激活TAP设备。退出集线器管理控制台后，我们将在系统中看到尚未分配IP的新网络接口： 



ifconfig






接下来，如果非活动状态，则必须启用接口之间的数据包路由（ip转发）： 



sudo nano /etc/sysctl.conf


取消注释以下行： 



net.ipv4.ip_forward = 1


我们将更改保存到文件，退出编辑器并使用以下命令应用它们： 



sudo sysctl -p


接下来，我们需要为虚拟网络定义一个具有虚拟IP的子网（例如10.0.10.0/24），并为该接口分配一个地址： 



sudo ifconfig tap_suricata_vp 10.0.10.1/24


然后，您需要阐明Netfilter规则。



1.如有必要，允许传入的数据包到侦听端口（专有协议SoftEther使用HTTPS和端口443） 



sudo iptables -A INPUT -p tcp -m tcp --dport 443 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 992 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 1194 -j ACCEPT
sudo iptables -A INPUT -p udp -m udp --dport 1194 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 5555 -j ACCEPT


2.配置从10.0.10.0/24子网到主IP服务器的NAT 



sudo iptables -t nat -A POSTROUTING -s 10.0.10.0/24 -j SNAT --to-source 45.132.17.140


3.我们允许从子网10.0.10.0/24传递数据包 



sudo iptables -A FORWARD -s 10.0.10.0/24 -j ACCEPT


4.我们允许传递已建立连接的数据包 



sudo iptables -A FORWARD -p all -m state --state ESTABLISHED,RELATED -j ACCEPT


使用初始化脚本重新启动系统时，过程自动化将作为作业交给读者。



如果要自动颁发IP客户端，则还必须为本地网桥安装某种DHCP服务。这样就完成了服务器设置，您可以转到客户端。SoftEther支持许多协议，其使用取决于LAN设备的功能。 



netstat -ap |grep vpnserver






由于我们的测试路由器也可以在Ubuntu下运行，因此我们将从外部存储库上安装softether-vpnclient和softether-vpncmd软件包，以使用专有协议。您将需要启动客户端： 



sudo vpnclient start


要进行配置，我们使用vpncmd实用程序，选择localhost作为运行vpnclient的计算机。所有命令都在控制台中完成：您将需要创建一个虚拟接口（NicCreate）和一个帐户（AccountCreate）。



在某些情况下，您需要使用AccountAnonymousSet，AccountPasswordSet，AccountCertSet和AccountSecureCertSet命令指定身份验证方法。由于我们不使用DHCP，因此将手动设置虚拟适配器的地址。



另外，我们需要启用ip转发（/etc/sysctl.conf文件中的参数net.ipv4.ip_forward = 1）并配置静态路由。如有必要，在带有Suricata的VDS上，您可以配置端口转发以使用本地网络上安装的服务。在这一点上，网络的结合可以被认为是完整的。

战斗使用外部IDS服务可能有几种情况：



防范DDoS攻击（主要目的）



由于必须将分析包发送到查看Internet接口的系统，因此很难在公司网络中实现这种选择。即使IDS阻止了它们，虚假的通信也会占用数据通道。为避免这种情况，您需要订购具有足够高效的Internet连接的VPS，该连接可以跳过本地网络上的所有流量以及所有外部流量。这通常比扩展办公渠道更容易，更便宜。作为替代方案，值得一提的是针对DDoS防护的专门服务。他们的服务成本可与虚拟服务器的成本相媲美，但是不需要费力的设置，但是有缺点-因为这笔钱，客户只能从DDoS获得保护，而自己的IDS可以根据需要进行配置。



防范其他类型的外部攻击



Suricata能够应对利用可从Internet访问的公司网络服务（邮件服务器，Web服务器和Web应用程序等）中各种漏洞的尝试。通常，为此，将IDS安装在边界设备之后的LAN内，但是外带具有存在的权利。



防止内部入侵者



尽管有系统管理员的最大努力，公司网络中的计算机仍可能感染恶意软件。另外，流氓有时会出现在局域网中并试图执行某些非法操作。 Suricata可以帮助阻止此类尝试，尽管最好将其安装在外围以保护内部网络，并将其与能够将流量镜像到一个端口的受管交换机一起使用。在这种情况下，外部IDS也不是没有用-至少它能够捕获存在于LAN中的恶意软件与外部服务器联系的尝试。

git clone https://github.com/Soldie/xerxes-DDos-zanyarjamal-C.git
