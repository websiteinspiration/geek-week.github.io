APT30を求めて：数年の沈黙の後のグループの新しい活動に気づいたように







サイバーグループAPT30はかなり前から知られています-2015年に、FireEyeの同僚がその活動について説明しました。このグループのメンバーは通常、サイバースパイ活動を目的として東南アジアおよび東南アジア（インド、タイ、マレーシア、その他の国）の政府機関を攻撃し、そのツールは少なくとも2005年以降開発されています。



過去数年間、APT30はそれほど活発ではありませんでしたが、2020年の春に、ハッカーによって開発された新しいマルウェアの痕跡を検出することができました。完全な研究は参考文献として公開されており、この記事ではその主要な点に触れます。



前書き



2020年4月8日、PT Expert Security Centerの専門家が有名なサイバー犯罪グループの活動を発見しました。動的なマルウェア分析のための人気のあるリソースが、長い間何も聞いていなかったAPT30のアクティビティを説明するネットワークシグネチャをトリガーしました。これが私たちの研究の出発点でした。



結局のところ、ハッカーは10年前の古くて有名なツールを引き続きサポートするだけでなく、ネットワークリソースを整理するためのアプローチにも忠実です。



BACKSPACEおよびNETEAGLEバックドア

最初のファイル（MD5：634e79070ba21e1e8f08aba995c98112）は、Microsoft Officeテンプレート- ％APPDATA％\ Microsoft \ Windows \ Templates \ AGENDA.docxのあるディレクトリに書き込まれ、起動します。これは、マレーシア政府の1つの部署での会議の計画が含まれているオフィス文書です。このドキュメントの目的は、ユーザーの注意を引くことです。

バックドアRCtrl



これは、MFCライブラリを使用して開発され、UPXを使用してパッケージ化されたx86ベースの実行可能PEファイルです。



マルウェアは、番号0x23のシングルバイトXORを使用して、メインの攻撃者サーバーのアドレス（103.233.10 \ .152）を復号化します。ポート4433でTCPを介して彼に向かい、接続の成功を確認します。接続が失敗した場合は、追加のデータを使用してサーバーアドレスを取得します。



追加のデータはアドレスでhxxp://www.gordeneyes.com/infos/p



ありhxxp://www.techmicrost.com/infos/p



、0x25のシングルバイトXORでエンコードされます。マルウェアはアドレスを復号化した後、GETリクエストを介してそれらに交互に接続しようとします。サーバーの応答には8バイトが必要です：サーバーのIPアドレスとポート。







攻撃者のサーバーのアドレスを取得する： '0xAC 0xF7 0xC5 0xBD'→ '172 247 197 189'、 '0xBB 0x01 0x00 0x00'→0x1BB→443



攻撃者のサーバーの有効なIPアドレスを受け取った後、マルウェアはサーバーに再接続し、サーバーからJo * Po * Hello行を受け取ることを期待します。この文字列は、マルウェアの本体で、0x24のシングルバイトXORでエンコードされます。奇妙なトリック：原則として、トロイの木馬自体がデータ交換を開始します。



回線が受信されると、システムのフィンガープリントが作成されます。システムのバージョン、IPアドレス、プロセッサーのブランドと周波数に関する情報、およびディスク領域の量です。収集されたデータは、ビット単位の循環シフトとXOR（つまり、4 + 3 = 7ビットの左シフトと0x23のXOR）に基づく独自のアルゴリズムで暗号化され、攻撃者のサーバーに送信されます。



次に、別のストリームが作成され、30秒ごとに同じデータバッファーがサーバーに送信されます。次に、コントロールは、受信したデータを解読するコマンド処理関数を受信し（解読アルゴリズムは、考慮される暗号化アルゴリズムとは逆のアクションで構成されます）、コマンド番号を取得します。



結論



これは、攻撃者によって作成されたソフトウェアが、高級なプログラムコード、システムに隠れたり、検出を回避するためのいくつかのトリックによって区別されていると言っているのではありません。目標が変化しているようには見えません。おそらく、より洗練された悪意のある武器は必要ありません。新しいツールキットについて言えば、その不完全さに気付きました。おそらく、グループは戦闘で新しいソフトウェアをテストし、欠陥を特定しています。したがって、将来的には、分析を隠したり複雑にしたりする手法を使用して、改善されたバックドアRHttpCtrlおよびRCtrlの出現を期待する必要があります。
