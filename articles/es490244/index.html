Optimizaciones básicas


Antes de reescribir el código fuente de Python en C, considere los métodos básicos de optimización en Python.

Estructuras de datos incorporadas


Las estructuras de datos integradas de Python, como set y dict, están escritas en C. Funcionan mucho más rápido que sus propias estructuras de datos escritas como clases de Python. Otras estructuras de datos además del conjunto estándar, dict, list y tuple se describen en la documentación del módulo de colecciones .

Lista de expresiones


En lugar de agregar elementos a la lista usando el método estándar, use expresiones de lista. 

# Slow
      mapped = []
      for value in originallist:
          mapped.append(myfunc(value))
      
      # Faster
      mapped = [myfunc(value) in originallist]

tipos


El módulo ctypes le permite interactuar con el código C de Python sin usar un módulo subprocessu otro módulo similar para iniciar otros procesos desde la CLI.

Solo hay dos partes: compilar el código C para cargar en calidad shared objecty configurar estructuras de datos en el código Python para asignarlas a los tipos C.

En este artículo conectaré mi código Python con lcs.c , que encuentra la subsecuencia más larga en dos listas de cadenas. Quiero que lo siguiente funcione en Python:

list1 = ['My', 'name', 'is', 'Sam', 'Stevens', '!']
      list2 = ['My', 'name', 'is', 'Alex', 'Stevens', '.']
      
      common = lcs(list1, list2)
      
      print(common)
      # ['My', 'name', 'is', 'Stevens']

Un problema es que esta función C particular es la firma de una función que toma listas de cadenas como tipos de argumento y devuelve un tipo que no tiene una longitud fija. Resuelvo este problema con una estructura de secuencia que contiene punteros y longitud.

Compilar código C en Python


Primero, el código fuente C ( lcs.c ) se compila lcs.sopara cargar en Python.

gcc -c -Wall -Werror -fpic -O3 lcs.c -o lcs.o
      gcc -shared -o lcs.so lcs.o

  • - Muro mostrará todas las advertencias;
  • - Werror envolverá todas las advertencias en errores;
  • - fpic generará instrucciones independientes de la posición que necesitará si desea usar esta biblioteca en Python;
  • - O3 maximiza la optimización;

Y ahora comenzaremos a escribir código Python usando el archivo de objeto compartido resultante .

Estructuras en Python


A continuación hay dos estructuras de datos que se usan en mi código C.

struct Sequence
      {
          char **items;
          int length;
      };
      
      struct Cell
      {
          int index;
          int length;
          struct Cell *prev;
      };

Y aquí está la traducción de estas estructuras a Python.

import ctypes
      class SEQUENCE(ctypes.Structure):
          _fields_ = [('items', ctypes.POINTER(ctypes.c_char_p)),
                      ('length', ctypes.c_int)]
      
      class CELL(ctypes.Structure):
          pass
      
      CELL._fields_ = [('index', ctypes.c_int), ('length', ctypes.c_int),
                       ('prev', ctypes.POINTER(CELL))]

Algunas notas

  • Todas las estructuras son clases que heredan de ctypes.Structure.
  • El único campo _fields_es una lista de tuplas. Cada tupla es ( <variable-name>, <ctypes.TYPE>).
  • Hay ctypestipos similares en c_char (char) y c_char_p (* char) .
  • También ctypeshay uno POINTER()que crea un puntero de tipo a partir de cada tipo que se le pasa.
  • Si tiene una definición recursiva como en CELL, debe pasar la declaración inicial y luego agregar los campos _fields_para obtener un enlace más adelante.
  • Como no utilicé CELLPython en mi código, no necesité escribir esta estructura, pero tiene una propiedad interesante en el campo recursivo.

Llame su código en C


Además, necesitaba un código para convertir los tipos de Python en nuevas estructuras en C. Ahora puede usar su nueva función C para acelerar el código de Python. 

def list_to_SEQUENCE(strlist: List[str]) -> SEQUENCE:
          bytelist = [bytes(s, 'utf-8') for s in strlist]
          arr = (ctypes.c_char_p * len(bytelist))()
          arr[:] = bytelist
          return SEQUENCE(arr, len(bytelist))
      
      
      def lcs(s1: List[str], s2: List[str]) -> List[str]:
          seq1 = list_to_SEQUENCE(s1)
          seq2 = list_to_SEQUENCE(s2)
      
          # struct Sequence *lcs(struct Sequence *s1, struct Sequence *s2)
          common = lcsmodule.lcs(ctypes.byref(seq1), ctypes.byref(seq2))[0]
      
          ret = []
      
          for i in range(common.length):
              ret.append(common.items[i].decode('utf-8'))
          lcsmodule.freeSequence(common)
      
          return ret
      
      lcsmodule = ctypes.cdll.LoadLibrary('lcsmodule/lcs.so')
      lcsmodule.lcs.restype = ctypes.POINTER(SEQUENCE)
      
      list1 = ['My', 'name', 'is', 'Sam', 'Stevens', '!']
      list2 = ['My', 'name', 'is', 'Alex', 'Stevens', '.']
      
      common = lcs(list1, list2)
      
      print(common)
      # ['My', 'name', 'is', 'Stevens']

Algunas notas

  • **char (lista de cadenas) coincide directamente con una lista de bytes en Python.
  • Hay lcs.cuna función lcs()con la firma struct Sequence * lcs (struct Sequence * s1, struct Sequence * s2) . Para configurar el tipo de retorno, yo uso lcsmodule.lcs.restype = ctypes.POINTER(SEQUENCE).
  • Para hacer una llamada con una referencia a la estructura de secuencia , utilizo ctypes.byref()una que devuelve un "puntero de luz" a su objeto (más rápido que ctypes.POINTER()).
  • common.items- esta es una lista de bytes, se pueden decodificar para obtener retla forma de una lista str.
  • lcsmodule.freeSequence (común) solo libera la memoria asociada con común. Esto es importante porque el recolector de basura (AFAIK) no lo recogerá automáticamente.

El código optimizado de Python es el código que usted escribió en C y lo envolvió en Python.
