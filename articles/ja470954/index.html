Zimbra OSE 8.8.15とZextras Suite ProをUbuntu 18.04 LTSにインストールする

Zimbra Collaboration Suiteオープンソースエディション8.8.15 LTSの最新パッチにより、Ubuntu 18.04 LTSオペレーティングシステムの「長時間」リリースの完全サポートが追加されました。これにより、システム管理者は、Zimbra OSEを使用してサーバーインフラストラクチャを作成できます。これは、2022年末までサポートされ、セキュリティアップデートを受け取ります。 3年以上関連性があり、同時にメンテナンスに多大な人件費を必要としないコラボレーションシステムを企業に実装する機能は、企業にとってITインフラストラクチャの所有コストを削減する絶好の機会であり、SaaSプロバイダーにとってこのオプションはZimbraです。 OSEを使用すると、顧客に対してより収益性の高いサービスを提供できると同時に、プロバイダーに対してより多くの限界料金を提供できます。理解しようUbuntu 18.04にZimbra OSE 8.8.15をインストールする方法。



Zimbra OSEをインストールするためのサーバーシステム要件には、4コアプロセッサ、8ギガバイトのRAM、50ギガバイトのハードディスク容量、FQDN、リダイレクトDNSサーバー、MXレコードが含まれます。 Zimbra OSEのパフォーマンスを制限する「ボトルネック」は通常、プロセッサやRAMではなく、ハードドライブであることをすぐに指摘します。そのため、サーバー用の高速SSDを購入するのが賢明です。サーバーの総コストに大きな影響はありませんが、Zimbra OSEのパフォーマンスと応答性が大幅に向上します。 Ubuntu 18.04 LTSとZimbra Collaboration Suite 8.8.15 LTSを搭載し、ドメイン名mail.company.ruでサーバーを作成しましょう。

初心者向けのZimbraのインストールで最も困難なのは、FQDNとリダイレクトDNSサーバーの作成です。すべてが機能するために、dnsmasqユーティリティに基づいてDNSサーバーを作成します。これを行うには、最初にsystemd-resolvedサービスを無効にします。これは、sudo systemctl disable systemd-resolvedコマンドとsudo systemctl stop systemd-resolvedコマンドを使用して行われます。また、sudo rm /etc/resolv.confコマンドを使用してresolv.confファイルを削除してから、echoコマンド「nameserver 8.8.8.8 "> /etc/resolv.confを使用して新しいファイルを作成します  。 

このサービスを無効にした後、dnsmasqをインストールする必要があります。これは、sudo apt-get install dnsmasqコマンドを使用して行われ  ます。インストールが完了したら、構成ファイル/etc/dnsmasq.confを編集してdnsmasqを構成する必要があります。結果は次のようになります。

server=8.8.8.8
listen-address=127.0.0.1
domain=company.ru   # Define domain
mx-host=company.ru,mail.company.ru,0
address=/mail.company.ru/***.16.128.192

これにより、Zimbraでサーバーアドレスを設定し、リダイレクトDNSサーバーとMXレコードを設定して、他の設定に移ることができます。sudo hostnamectl set-hostname mail.company.ru

コマンドを使用して、  Zimbra OSEでサーバーのドメイン名を設定し、echo "***。16.128.192 mail.company.ru"コマンドを使用して対応する情報を/ etc / hostsに追加します  | sudoのティー-a / etc / hostsファイル。

その後、sudo systemctl restart dnsmasqコマンドを使用してdnsmasqサービスを再起動し、dig A mail.company.ruコマンドとdig MX company.ruコマンドを使用してAレコードとMXレコードを追加する必要があります。。これがすべて完了したら、Zimbra Collaboration Suite Open-Source Edition自体のインストールを開始できます。

Zimbra OSEのインストールは、ディストリビューションのダウンロードから始まります。これは、wget files.zimbra.com/downloads/8.8.15_GA/zcs-8.8.15_GA_3869.UBUNTU18_64.20190917004220.tgzコマンドで実行できます。配布をダウンロードしたら、tar xvf zcs-8.8.15_GA_3869.UBUNTU18_64.20190917004220.tgzコマンドを使用して配布を解凍する必要があります  。アンパックが完了したら、使用してアンパックフォルダに移動する必要があります  * /コマンドのCD ZCSをした後、使用してインストールスクリプトを実行し  ます。./install.shコマンドを。

インストーラーを起動した後、利用規約に同意し、公式のZimbraリポジトリを使用して更新をインストールすることに同意する必要があります。その後、インストールするパッケージを選択するよう求められます。パッケージを選択すると、インストール中にシステムが変更されるという警告が表示されます。ユーザーが変更に同意すると、欠落しているモジュールと更新のダウンロード、およびそれらのインストールが開始されます。インストールが完了すると、インストーラーはZimbra OSEの初期セットアップを実行するように提案します。この時点で、管理者パスワードを設定する必要があります。これを行うには、まずメニュー項目7に移動してから、項目4を選択する必要があります。その後、Zimbra Open-Source Editionのインストールが完了します。

Zimbra OSEのインストールが完了した後は、操作に必要なWebポートを開くだけです。これは、ufwと呼ばれる標準のUbuntuファイアウォールを使用して行うことができます。すべてが機能するためには、最初にufw allow from 192.168.0.1/24コマンドを使用して管理サブネットからの無制限のアクセスを許可し、次に/etc/ufw/applications.d/zimbra設定ファイルにZimbraプロファイルを作成する必要があります  。

[Zimbra]  

title=Zimbra Collaboration Server
description=Open source server for email, contacts, calendar, and more.
ports=25,80,110,143,443,465,587,993,995,3443,5222,5223,7071,9071/tcp

次に、sudo ufw allow Zimbraコマンドを使用して、作成したZimbraプロファイルをアクティブにし、sudo ufw enableコマンドで ufwを再起動する必要があります  。また、sudo ufw allow sshコマンドを使用して、SSH経由でサーバーにアクセス  できます。必要なポートが開いたら、Zimbra管理コンソールにアクセスできます。 、タイプこれを行うにはmail.company.ru:7071をブラウザのアドレスバーに、または、プロキシ、使用している場合  mail.company.ru:9071をした後、ユーザー名としてadminと入力し、パスワード、パスワードとして設定されていることZimbraをインストールするとき。



Zimbra OSEのインストールが完了すると、完全な電子メールおよびコラボレーションソリューションがエンタープライズインフラストラクチャに表示されます。ただし、メールサーバーの機能は、拡張機能Zextras Suite Proを使用して大幅に拡張できます。これらを使用すると、モバイルデバイスのサポート、ドキュメント、スプレッドシート、プレゼンテーションとのコラボレーションをZimbra Collaboration Suite Open-Source Editionに追加でき、必要に応じて、Zimbra OSEでテキストおよびビデオチャット、ビデオ会議のサポートを追加できます。

Zextras Suite Proのインストールは非常に簡単です。これを行うには、wget コマンドwww.zextras.com/download/zextras_suite-latest.tgzを使用して公式のZextras Webサイトから配布パッケージをダウンロードし、このアーカイブを解凍します。 tar xfz zextras_suite-latest.tgz、解凍したファイルcd zextras_suite /が含まれているフォルダーに移動  し、コマンド./install.sh allを使用してインストールスクリプトを実行します。その後、zmprov fc zimletコマンドを使用してZimbra OSEキャッシュをクリアするだけで、  Zextras Suiteの使用を開始できます。

Zextras Docs拡張機能が機能するため、従業員がテキストドキュメント、テーブル、プレゼンテーションを共同で作業できるようにするには、別のサーバーアプリケーションをインストールする必要があります。 ZextrasのWebサイトで、Ubuntu 18.04 LTSオペレーティングシステム用のディストリビューションをダウンロードできます。。さらに、Zextrasチームの従業員間のオンライン通信のためのソリューションの機能は、アプリケーションを使用してモバイルデバイスで利用できます。これは、Google PlayおよびApple AppStoreから完全に無料でダウンロードすることもできます。さらに、Zextras Driveクラウドストレージにアクセスするためのモバイルアプリケーションがあり、iPhone、iPad、Androidデバイスでも利用できます。

したがって、Zimbra OSE 8.8.15 LTSとZextras Suite ProをUbuntu 18.04 LTSにインストールすることで、長期的なサポートと低いライセンスコストにより、企業のITインフラストラクチャの所有コストを大幅に削減できる、完全に機能するコラボレーションソリューションを入手できます。
