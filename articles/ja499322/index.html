Windows ServerまたはLinuxディストリビューション？サーバーOSの選択



オペレーティングシステムは、現代の産業の基盤です。一方では、より有用なものに使用できる貴重なサーバーリソースを消費します。一方、オペレーティングシステムはサーバーアプリケーションのオーケストラとして機能し、シングルタスクコンピューティングコンプレックスからマルチタスクプラットフォームを作成できるようにし、すべての関係者と機器との対話を容易にします。現在の主流のサーバーOSは、Windows Server +さまざまな種類のLinuxディストリビューションです。これらの各オペレーティングシステムには、長所、短所、およびアプリケーションのニッチがあります。今日は、サーバーに付属するシステムについて簡単に説明します。

Windowsサーバー


このオペレーティングシステムは企業セグメントで非常に人気がありますが、ほとんどの一般ユーザーはWindowsをPCのデスクトップバージョンにのみ関連付けています。会社をサポートするために必要なタスクとインフラストラクチャに応じて、Windows Server 2003以降で最新バージョンのWindows Server 2019で終了するWindows Serverのいくつかのバージョンが同時に動作します。記載されているすべてのオペレーティングシステム、つまりWindows Server 2003を備えたサーバーを提供します。 2008 R2、2016、2019。

Windows Server 2003は、主にWindows XPに基づいて構築されたエンタープライズシステムとネットワークをサポートするために使用されます。驚いたことに、約5年前にサポートされて入手されたMicrosoftのデスクトップOSのバージョンは、運用用の多くのプロプライエタリソフトウェアが一度にその下で作成されたため、まだ動作しています。同じことがWindows Server 2008 R2とWindows Server 2016にも当てはまります。これらは、古くは機能しているソフトウェアと最も互換性があるため、今日まで使用されています。

Windowsベースのサーバーの主な利点は、管理が比較的簡単なこと、かなり大量の情報、マニュアル、ソフトウェアがあることです。さらに、会社のエコシステムに、Microsoftシステムのカーネルのライブラリと一部を使用するソフトウェアまたはソリューションがある場合、Windows上のサーバーなしでは実現できません。また、ここでは、サーバーアプリケーションへのユーザーアクセスとシステムの汎用性のためにRDPテクノロジを追加できます。さらに、Windows Serverには、Linuxディストリビューションのレベルでリソースを消費するGUIのない​​軽量バージョンがあります。これは、以前に説明した Windows Server Core です。 Windowsを実行しているすべてのサーバーにアクティブ化されたライセンスを提供します（新規ユーザーは無料です）。

Winserverのマイナス面では、ライセンスコストとリソース消費という2つのパラメーターを一度に書き留めることができます。すべてのサーバーオペレーティングシステムの中で、Windows Serverは最も大胆で、カ​​ーネルと標準サービスが機能するためだけに、少なくとも1つのプロセッサコアと1.5〜3ギガバイトのRAMを必要とします。このシステムは低電力構成には適しておらず、RDPおよびグループとユーザーのポリシーに関連する多くの脆弱性も持っています。

ほとんどの場合、Windows Serverは企業のイントラネットを管理し、特定のソフトウェア、MSSQLデータベース、ASP.NETツール、またはWindows専用に作成されたその他のソフトウェアのパフォーマンスを保証するように設計されています。同時に、ルーティングの導入、DNSの引き上げ、その他のサービスを実行できる本格的なOSです。

Ubuntu


UbuntuはLinuxファミリの最も人気があり安定した開発ディストリビューションの1つであり、2004年に初めてリリースされました。かつてはGnomeシェルの「主婦のテナント」でしたが、Ubuntuはその広範なコミュニティと継続的な開発により、やがてデフォルトのサーバーOSになりました。最新の人気バージョンは18.04ですが、サーバーは16.04未満で提供され、約1週間前にバージョン20.04がリリースされました。

特定のWindows指向のソフトウェアをサポートするOSとしてWindows Serverを使用した場合、LinuxディストリビューションとしてのUbuntuは、オープンソースとWeb開発の物語です。したがって、現在人気のあるPostgreSQLやMySQL、またはスクリプト開発言語と連携するために、nginxまたはApache（Microsoft IISではなく）でWebサーバーをホストするために使用されるのはLinuxサーバーです。 Ubuntuを搭載したサーバーでは、ルーティングおよびトラフィック管理サービスも完全に機能します。

プラスは、Windows Serverよりも少ないリソース消費を書き留めるだけでなく、コンソールとパッケージマネージャーを備えたすべてのUNIXシステムでネイティブに機能することです。さらに、Ubuntuは最初は「デスクトップホームUnix」であり、ユーザーフレンドリーで管理が容易です。

主なマイナス点はunixであり、そのすべてが意味します。Ubuntuは友好的かもしれませんが、他のLinuxシステムにのみ相対的です。そのため、特に本格的なサーバー構成で（つまり、端末のみで）使用するには、特定のスキルが必要になります。さらに、Ubuntuは個人的な使用に重点を置いており、企業のケースを解決するのに必ずしも適しているわけではありません。

Debian


皮肉なことに、Debianは、前述した非常に人気のあるUbuntuの原型です。最初のDebianビルドは25年以上前（1994年）に公開され、Ubuntuの基礎を形成したのはDebianコードでした。実際、DebianはLinuxファミリーのシステムの中で最も古く、同時にハードコアなディストリビューションの1つです。 Ubuntuのすべての類似点について、その「相続人」とは異なり、Debianはジュニアシステムとしてのユーザーフレンドリーのレベルを受け取りませんでした。ただし、これには利点もあります。 DebianはUbuntuよりも柔軟性があり、より詳細に構成でき、企業のタスクを含む多くの特定のタスクをより効率的に解決できます。

Debianの主な利点は、Ubuntu、特にWindowsと比較して高いセキュリティと安定性です。そしてもちろん、他のLinuxシステムと同様に、特に端末を実行するサーバーOSの形で、リソース消費量が少なくなります。さらに、Debianコミュニティはオープンソースであるため、このシステムは主に、無料のソリューションを使用した正確で効果的な作業に焦点を当てています。

ただし、柔軟性、ハードコア、およびセキュリティの費用を支払う必要があります。Debianは、ブランチウィザードシステムによる明確なカーネルなしで、オープンソースコミュニティによって開発されています。ある時点で、Debianには、安定版、不安定版、およびテスト版の3つのバージョンが同時に存在します。問題は、安定した開発ブランチがテストのブランチの真後ろにあるということです。つまり、古い部品やモジュールがカーネルに見つかることがよくあります。タスクが安定版のDebianの機能を上回っている場合、これらすべてにより、手動でカーネルが再構築されるか、テストブランチに移行することになります。Ubuntuにはバージョンの不連続性に関するそのような問題はありません。開発者は2年ごとにシステムの安定したLTSバージョンをリリースしています。

CentOS


それでは、CentOS上のサーバーOS RUVDSについての会話を終了しましょう。より大規模なUbuntu、特にDebianの中で、CentOSは10代の若者のように見えます。そして、システムはDebianやUbuntuほど昔から大衆の間で人気がありましたが、最初のバージョンのリリースはUbuntuで一度に、つまり2004年に行われました。

CentOSは主に仮想サーバーに使用されます。これは、UbuntuやDebianよりもリソース消費が少ないためです。このOSの2つのバージョン（CentOS 7.6.1810と古いCentOS 7.2.1510）を実行する構成を提供します。主な使用例は企業のタスクです。 CentOSは仕事についての物語です。たとえば、Ubuntuのような家庭用システムではないため、CentOSはすぐにオープンソースコードに基づくRedHatのようなディストリビューションとして開発されました。 CentOSに主な利点を与えるのはRedHatからの遺伝です-企業タスク、安定性、およびセキュリティに焦点を当てています。システムの最も一般的な使用例はWebホスティングで、CentOSは他のLinuxディストリビューションよりも優れた結果を示します。

ただし、システムにはいくつかのマイナスがあります。Ubuntuよりも開発と更新のサイクルが抑制されているため、ある時点で、他のディストリビューションですでに解決されている脆弱性や問題に耐えなければならないという事実につながります。コンポーネントを更新およびインストールするためのシステムも異なります。apt-getはなく、yumおよびRPMパッケージのみです。また、CentOSは、UbuntuとDebianが明らかにDocker / k8sコンテナソリューションをホスティングおよび操作するのに適していません。コンテナ化によるWebサーバーとアプリケーションの仮想化は、近年DevOps環境で勢いを増しているため、後者は重要です。そしてもちろん、CentOSは、より人気のあるDebianやUbuntuに比べてはるかに小さなコミュニティを持っています。

出力の代わりに


ご覧のとおり、どのOSにも長所と短所があり、独自のニッチを持っています。 Windowsベースのサーバーは際立っています。Microsoft環境では、いわば、独自の雰囲気と仕事のルールです。
すべてのLinuxディストリビューションは、リソース消費の点では互いに似ていますが、タスクによって独自の機能と違いがあります。 Ubuntuは使いやすく、Debianはより細かく設定されています。 CentOSは有料のRedHatの代わりとして機能することができます。これは、UNIXパフォーマンスで本格的な企業OSが必要な場合に重要です。しかし同時に、コンテナ化とアプリケーションの仮想化の点で弱いので、いずれの場合も専門家に連絡して、タスクに基づいて必要なソリューションと構成を見つけることができます。
