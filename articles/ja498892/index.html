最後のRenga APIレビュー

Rengaの建物と構造の設計システムは活発に開発されており、建築家、KZhとKM、VK、OV、EO、ESエンジニアの設計者向けのツールを備えています。各バージョンで、Renga Architecture、Renga Structure、Renga MEPに分割されたシステムは、すぐに1つになり、機能を構築します。これに伴い、それは成長し、変化し、ますます人気のあるAPIになります。実際、Rengaでは建物の情報モデルを作成できますが、パイプラインの建設や計算の見積もりである美しいレンダリングを得ることができません（計画もしていません）。もちろん、モデルをRengaからOBJ、IFC、STEPなどの一般的な形式にエクスポートできますが、仲介なしで実行し、エクスポート/インポート中にデータを誤って解釈しないようにするために、APIを介して動作することをお勧めします。



当初、Renga APIの作成に関する質問があったとき（Renga Architectureの最初のバージョンのリリース後）、その時点での要件に基づいて、C ++ APIのみに焦点を当てました。

Rengaが開発されるにつれて、Rengaの下でC＃およびその他の言語で記述できるかどうか、およびサポートされているVisual Studioのバージョンについて、開発者から質問を受け始めました。

次に、ユーザーのAPIの可用性を分析した後、テクノロジーの変更を検討しました。

実際のところ、C ++ APIはかなり厳しいものです。これはコンパイラに大きく依存しています。 SDKがVisual Studioコンパイラーによってコンパイルされた場合、おそらくgccなどを使用してコンパイルすることはできません。名前のマングリングは標準化されていないため、仮想テーブルやテンプレートなどの実装に違いがある可能性があります。 Visual Studio自体では、コンパイル設定と標準のサポートがバージョン間で異なる場合もあります。

一般的に、たとえば、テンプレートを使用せずに、使用される標準と開発環境を一定期間修正せずに、仮想関数を最小限に（結果として継承として）使用することでこれに対処できます。

さらに、さまざまな環境やシステムのバージョンに応じて、APIのいくつかのバージョンを作成できます。
しかし、それでも、私たちは別の見方をすることにしました。そして調査の後、彼らはCOMを選びました。
この標準は1993年に開発されました。つまり、それは古いだけでなく、スーパースターです）しかし、C ++、C＃、または他のCOM互換言語でプラグインを記述できる単一のAPIをユーザーに提供でき、さらにCOM APIを使用してアクセスできますたとえばPythonなどの動的型付けを使用する言語で記述されたアプリケーションを含む、サードパーティアプリケーションからRengaへ



私はと言わなければならないレンガAPIのヘルプを開始する方法、何とそれを使用する方法について詳しく説明します。ただし、Rengaの拡張機能を作成するために必要なものはまだ説明しています。

したがって、任意の言語でRengaの拡張機能を作成するには、次のものが必要です。

  1. Rengaと、システムを操作するときに自動化したいものを正確に理解します。
  2. Rengaソフトウェア開発キット。
  3. Microsoft Visual Studio。2012より新しいバージョン。
  4. プログラミングの基礎に関する知識。

Rengaシステムとその機能については、システムのWebサイト、ヘルプ、開発者のブログ、VKとFacebookのグループで確認できます。そこには、ウェビナーのアナウンスと録音が投稿されています。また、システムの実際のユーザーを見つけてチャットすることもできます。したがって、システムの知識は省きますが、APIが満たすことができる要件を指定します。

現時点では、APIから新しいオブジェクトを作成することはできません。これは計画中ですが、次のリリースではできません。

Renga APIはユーザーの要求に応じて拡張されていることに注意してください。このリストに必要な自動化機能が見つからなかった場合は、お問い合わせください。アイデアについて話し合います。

C ++ APIのみで作業を続けた場合、機能のセットは同じである可能性が高いですが、すべてのRenga拡張機能が生まれるとは限りません。ちなみに、あなたは私たちが知っている拡張機能のリストをサイトで見ることができます。

最後に、APIに関してユーザーに何を提供するかを見てみましょう。 Renga SDKを入手するには、ダウンロードするだけです。そして、それを使用するには、アーカイブにある例を研究して使用することができます。



SDKには、C ++およびC＃で拡張機能を作成するための例と、PythonでRengaに接続する例があります。方法セクションのRenga APIヘルプに、このセクションのテクニックを使用する例がリストされています。

C ++およびC＃プラグインの例は、ほとんどすべてのAPI機能を示しています。Pythonの例では、それを使用してサードパーティのアプリケーションからRengaにカスタムプロパティを作成する方法と、スクリプトを使用してRengaプロジェクトを自動的に開いて保存して閉じる方法を説明します。

現在、RengaはC ++バイナリDLLおよび.NETアセンブリからプラグインをロードでき、Python、VBScript、1Cなどの動的に型付けされた言語を使用して、サードパーティプログラムからRenga APIにアクセスし、必要なデータを取得できます。
一般的に、COM APIへの移行と、ユーザーに開かれた機会に満足しています。

Rengaから呼び出されるすべてのRenga拡張機能は、プラグインのバイナリファイルと、拡張子が.rndescの.xmlファイルで構成され、次の内容が含まれている必要があります。

<RengaPlugin>
   <!--   -->
   <Name>Best Plugin</Name>
   <!--   -->
   <Version>1.0</Version>
   <!--    -->
   <Copyright>Copyright text</Copyright>
   <!--   Renga API -->
   <RequiredAPIVersion>2.3</RequiredAPIVersion>
   <!--   cpp/net -->        
   <PluginType>Plugin type</PluginType>
   <!--    -->                     
   <PluginFilename>Best_Plugin.dll</PluginFilename> ->
   <Vendor>Renga</Vendor>                           
</RengaPlugin>

拡張機能をRengaに接続するには、拡張機能をRenga \ Pluginsインストールフォルダーにインストールまたはコピーする必要があります。.rndescファイルに含まれる情報は、[Renga設定]ウィンドウの[拡張子]タブに表示されます。

習得した知識とプログラミングの知識を武器に、開発者なら誰でもRengaの拡張機能を作成できると確信しています。ただし、Rengaを拡張するアイデアがあり、その実装に問題がある場合は、フォーラムに書き込むか、ServiceDeskにリクエストを登録してください。ご質問お待ちしております！
