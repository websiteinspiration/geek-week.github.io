Kirim perintah ke beberapa perangkat secara bersamaan menggunakan SecureCRT

Insinyur jaringan sering memiliki tugas melakukan serangkaian tindakan yang identik pada sejumlah besar peralatan. Cara utama untuk melakukan ini dengan cepat adalah menulis tindakan ke notepad dan menyalin / melewati ke setiap perangkat. Ada cara untuk mempercepat proses yang sudah cepat ini menggunakan perangkat lunak SecureCRT dan fungsi jendela Perintah. Yang memungkinkan Anda mengirim perintah ke banyak perangkat secara bersamaan.
Untuk apa artikel ini? Artikel ini berasal dari siklus Mulai Cepat dan bertujuan untuk menghemat waktu bagi teknisi jaringan saat mengatur peralatan (satu tugas) pada beberapa perangkat. Ini menggunakan perangkat lunak SecureCRT dan fungsionalitas jendela Perintah bawaan.

pengantar


Dalam proses menyiapkan peralatan jaringan, seringkali muncul tugas melakukan tindakan sederhana dan identik pada sejumlah besar perangkat. Untuk melakukan tugas-tugas ini, Anda dapat menggunakan metode standar: Notepad + Copy / Paste. Dan Anda dapat menggunakan fungsionalitas SecureCRT bawaan - jendela perintah - jendela khusus untuk mengedit perintah sebelum mengirimnya ke peralatan. Dan fungsi tambahan: mengirim perintah tidak hanya ke sesi saat ini, tetapi juga ke beberapa sesi terbuka.

Menggunakan jendela Command:
  • Jendela Command Command, mengatur mode operasi
  • Menggunakan jendela Command untuk mengkonfigurasi beberapa perangkat secara bersamaan
  • Menggunakan jendela Command untuk mengkonfigurasi beberapa perangkat secara real time pada waktu yang bersamaan.


Jendela Command Command, mengatur mode operasi


Jendela perintah dipanggil melalui menu: SecureCRT Menu => View => Jendela perintah. Atau Menu SecureCRT => Lihat => Jendela obrolan.


Mode operasi mungkin:
  • Mode edit
  • Real-time ("Kirim Karakter Segera")

Mode pengeditan - memungkinkan Anda untuk mengedit perintah sebelum mengirimnya ke perangkat.
Mode real-time - semua penekanan tombol secara otomatis dikirim ke perangkat - apakah itu spasi atau penekanan tombol (abjad atau Enter).

Pengaturan penerusan perintah memiliki mode operasi berikut:
  • Teruskan ke sesi aktif (hanya yang sekarang).
  • Di semua sesi.
  • Ke grup (tab) atau grup yang dipilih.

Secara resmi, pilihannya adalah sebagai berikut:
Active Session — Send commands only to the active session. This is the default.
All Sessions — Send commands to all your open sessions at once.
Selected Tab Group(s) — Send commands to one or more tab groups. This setting is automatically selected when you right-click on a tab and enable Send Commands to This Group. When commands are being sent to selected tab groups, the tabs of the sessions that commands will be sent to contain an asterisk in front of the session name.
Visible Sessions — Send commands to all visible tabbed sessions. When there are multiple tab groups, commands will be sent to the currently active session in each tab group.

Pengaturan real-time diaktifkan menggunakan menu konteks-sensitif di jendela Command. (Klik kanan pada Command Window - menu sensitif konteks muncul dan pilih item menu "Kirim Karakter Segera")
Pengaturan penerusan perintah juga dipanggil menggunakan menu sensitif konteks di jendela Command. (Klik kanan pada Command Window - menu sensitif konteks muncul, pilih "Kirim Perintah Ke", pilih sub-item menu {"Semua sesi" | "Tab yang dipilih" | "Sesi aktif")


Menggunakan jendela Command untuk mengkonfigurasi beberapa perangkat secara bersamaan


Sangat sering, untuk bekerja, Anda perlu menjalankan perintah sederhana secara bersamaan di beberapa perangkat. Untuk melakukan ini, sangat mudah (karena cepat) menggunakan jendela Command untuk mengirim data ke beberapa perangkat secara bersamaan.
Untuk mengkonfigurasi beberapa perangkat:
  1. Kami terhubung ke semua perangkat dengan tipe yang sama.
  2. Di menu SecureCRT, panggil jendela Perintah
  3. Di menu konteks jendela Perintah, pilih Semua sesi.
  4. Kami memasukkan perintah sebagai satu unit atau satu per satu. (Dalam sekali swap, perintah dimasukkan melalui Control + Enter. - Umpan baris tanpa mentransfer ke peralatan.)

Ini biasanya seperangkat perintah standar:
screen-len 0 tem
dis cu
save
y
q

Koneksi cepat ke beberapa sesi


Pilih sesi yang ingin kita sambungkan sekaligus dan tekan Enter.

Kami mengirim perintah ke semua perangkat secara bersamaan:
ter len 0
show ip int br

Menggunakan jendela Command untuk mengkonfigurasi beberapa perangkat secara real time pada waktu yang bersamaan.


Apa input real-time dapat digunakan dan benar-benar digunakan? Jawab: ketika parameter identik dikonfigurasi, tetapi dengan sedikit perbedaan. Contoh nama host, alamat ip pada antarmuka.
Bagaimana kelihatannya dalam kenyataan:
1. Sesi dibuka di mana perlu untuk memasukkan parameter yang hampir sama.

2. Mode operasi dari jendela perintah dikonfigurasikan: "Kirim Karakter Segera" + "Kirim perintah ke: Semua Sesi".

3. Data berikut dimasukkan ke dalam jendela perintah:
interface gi0/0/0.10
dot1q termination vid 10
ip address 172.31.255.

4. Kursor ditransfer menggunakan mouse dari jendela Command ke jendela sesi dan untuk setiap perangkat dalam sesi, alamat yang diperlukan telah dikonfigurasi:
R1 – 1
R2 – 2
R3 – 3

5. Kursor ditransfer menggunakan mouse ke jendela Command dan suffix ditambahkan:
 255.255.255.0
quit
dis ip int br


Trik kecil. Untuk memeriksa ejaan - pertama saya memeriksa ejaan dalam satu sesi. Setelah verifikasi, perintah dikirim ke semua sesi.


Kesimpulan


Anda harus selalu berhati-hati.
Dalam beberapa kasus, lebih cepat untuk memutuskan sambungan dari semua sesi. Hubungkan hanya ke sesi khusus. Masukkan perintah di Command windows. Setelah terhubung ke semua sesi lainnya. Menghubungkan ke SecureCRT sangat cepat. Dan menghubungkan kembali tidak menyebabkan masalah.
Sangat sering, setelah memasukkan perintah grup, pekerjaan di setiap sesi diperiksa (menggunakan kombinasi [ctrl + tab] kita beralih di semua jendela dan membandingkan hasil perintah di setiap jendela). Kami memeriksa kebenaran dari pelaksanaan semua perintah, kesalahan yang bisa terjadi dalam satu sesi. Contoh: ketika terhubung ke 3 router Huawei, salah satunya secara default terhubung ke mode pengguna. Untuk bekerja di utas umum, saya mengalihkan mode konfigurasi ke mode privilege.
