画像：

Positive Technologies Security Center（PT Expert Security Center）のUnsplash Expertsが、中国のインターネットセグメントで拡散しているマルウェアの興味深いサンプルを発見しました。このソフトウェアは、とりわけMITM攻撃を実行するために使用され、その主な機能は、検出を回避するためのさまざまな手法の組み合わせです。それらを解析して、悪意のあるソフトウェアの開発者がその活動をどのように隠すかを示すことにしました。

すべての始まり


ネットワークトラフィック分析システムは、悪意のあるアプリケーションが追加のコンテンツを含む画像を定期的に要求するという事実に注意を向けました。ダウンロードは、画像を保存するための正当なリソース-imgsa.baidu.comから行われました。さらに、結局のところ、圧倒的な可愛さの写真でした:)その後、攻撃者がそれを使用してさまざまな悪意のある負荷を隠す試みが冒とくされたように見えました！

データの復号化

マルウェアがその活動を隠す方法


与えられたペイロードのシーケンスに従って、以下の研究に進みます。そのサフィックスは「AR」です。クライアントは、既存のスキームに従って、Baidu Images画像ストレージサービスから暗号化されたロードで画像の次の連結をダウンロードし、モジュールを復号化して、ランダムな名前で新しいプロセスで開始します。私たちの意見では、この機能は悪意のあるアプリケーションを無害に見せるために役立ちます。多くの場合、これはオンラインゲームクライアントです（図13）。そして、これは別の変装テクニックでした。



図。 13.オンラインゲームインターフェイス

この気を散らす操作を実行した後、悪意のあるプロセスは感染したノードを修正する段階に進みます。これを行うには、ルートキットプログラムの機能と同様の機能を使用します。特に、システムへの独自の安全なドライバーの導入。

そして、これはそれが起こる方法です。復号化された構成から、サフィックス「AE」が付いた負荷が選択されます。これはTaskReportDLL.dllライブラリです。これは、前のステップのTkRep.dllライブラリと同じ機能を備えています。システムに関する情報を送信し、保護装置が利用可能かどうかを確認します。

次に、RealWorkDll.dllライブラリがダウンロードされます。 RealWorkDll.dllの機能の中で、VMPROTECTによって部分的に保護されたドライバーと、このドライバーがシステムにインストールするPEファイルをダウンロードすることが重要です。



図。 14.ドライバデータベースへのパス

その後、ドライバーのインストールに使用したPEファイルが削除され、この修正段階が完了します。

ドライバーデータベースの行を検索すると、ルートキット[。] Comリソースミラーリポジトリに移動しました。ここで、パス「objfre_wxp_x86」に対応する名前のFUToルートキットのインスタンスが見つかりました（図14）。弊社のブログでは、このルートキットは2006年に考慮されました。

RealWorkDll.dllモジュールによってインストールされたSDriverBlogx86ドライバーのシステムでの作業をより詳細に検討します。最初の段階では、クライアントの登録データがネットワークに送信されます。 POSTは要求として使用されますが、現在はポート番号8081にあります（図15）。どうやら、このポートは、感染したシステムでのアクティビティがルートキット「FUTo」のアクティブ化段階に達した場合にデータを受信するために使用されます。



図。 15.システムにインストールされているドライバーからのC2の要求

攻撃者のサーバーに対するアピールは暗号化された形式で行われ、暗号化前のデータはシステムに関する情報です。データフィールドのセパレータ、表示形式、フィールド数はすべてのモジュールで同じです（図16）。

このマルウェアは、ブートローダー、偽装ファイル、ルートキットドライバー、および中間者攻撃を実行するためのモジュールで構成されています。秘密のペイロード配信の場合、ソフトウェアはJPEG画像を使用してデータを接合する手法を使用します。コマンドサーバーの場合、攻撃者はドメインゾーンtop、bid、およびクラウドプラットフォームに基づいて名前を登録します。
