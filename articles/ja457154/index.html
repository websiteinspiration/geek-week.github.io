すべてのユーザー向けのレスポンシブなアプリ設計

私は、モバイルアプリケーションをよりパーソナライズする必要があると考えています。一部の機能は必要ありませんが、他の機能は頻繁に使用します。しかし、何らかの理由で1つの櫛の下で全員を平等にして、より多くの人々に少しでも対応できるようにします。パーソナライゼーションの面でもう少し進んだのは、ニュースフィードとソーシャルネットワークです。すべての人にユニークなコンテンツを提供するネットワーク。

しかし、さらに進んで、コンテンツの他にパーソナライズされたUIを人々に提供したらどうなるでしょうか。

理論


概念

  1. アプリケーション自体は、頻繁に使用するものを理解し、よく使用する機能を最初の画面に表示します。
  2. ページの重要度に応じて要素を配置し、親指で手を伸ばす必要がないようにします。
  3. この要素が使用される頻度に応じて、その内容も大きく異なります
  4. トリガーもあります：プッシュ通知が到着した、特定の日付またはユーザーアクション。このトリガーには独自の固有の重みがあり、個々の要素に短時間割り当てられます。

ロジック

キャリブレーションステップ

人がアプリケーションを使用します。

そして、プログラム自体が各要素のクリック数を分析し、要素に比重を与えます。

スムーズな実装の段階

最初の調整後、メインページへの最も一般的なリクエストを別のブロックに慎重に実装できます。

要素検証フェーズ

訪問の頻度を分析し、要素がメインページにとどまる価値があるかどうかを判断します。

そして、それが価値があり、頻繁に使用される場合は、ユーザーがターゲットアクションをさらに速く実行できるように、このセルを拡張するオプションを検討しています。

アプリケーションの実装

どうして？

  1. 彼らは多機能です。
  2. ユーザーとして私が必要とする機能のほとんどはまったく必要ありませんが、他の機能のほうが重要な場合があります。
  3. 一部の機能は、特定の瞬間にのみ必要になる場合があります
  4. 誰もがこれらのアプリケーションを使用するので、概念を理解するのは簡単です。
