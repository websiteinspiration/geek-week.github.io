トークン、リフレッシュトークン、およびRESTリクエストの非同期ラッパーの作成

このチュートリアルでは、ユーザーの承認を必要とするAPIへのRESTリクエストがどのように実装されるかを簡単に調べ、リクエストの非同期「ラッパー」を作成します。これにより、承認がチェックされ、タイムリーに更新されます。

ログイン情報


ユーザー名とパスワードを送信したAPIにRESTリクエストを送信すると、次の形式のjsonが返されます（値はランダムで、通常、行は長くなります）。

{
"access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJpc3MiOiJPbmxpbmUgSld",
"refresh_token": "1eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJpc3MiOiJPbmxpbmUgS",
"expires_in": 124234149563
}

「token_type」、「expires_on」などの ように、応答にはさらに多くのフィールドがある場合がありますが、この実装では、上記の3つのフィールドのみが必要です。
それらを詳しく見てみましょう：

  • access_token-応答でデータを受信するために各リクエストのヘッダーで送信する必要があるトークン
  • refresh_token-古いトークンの有効期限が切れたときに新しいトークンを受け取るために送信する必要があるトークン
  • expires_in-トークンの有効期間（秒）

トークンを受け取る


上記のjsonを受け取る関数を作成して保存します。必要に応じて、

承認用のデータをsessionStorageまたはlocalStorageに保存します。最初のケースでは、ユーザーがセッションを完了するかブラウザを閉じるまでデータが保存されます。2番目のケースでは、何らかの理由でlocalStorageがクリアされるまで、データは無制限の時間ブラウザに保存されます。

トークンをsessionStorageに保存する関数：


function saveToken(token) {
    sessionStorage.setItem('tokenData', JSON.stringify(token));
}

トークンを受け取る関数：



function getTokenData(login, password) {
    return fetch('api/auth', {
        method: 'POST',
        credentials: 'include',
        headers: {
            'Accept': 'application/json',
            'Content-Type': 'application/json',
        },
        body: JSON.stringify({
            login,
            password,
        }),
    })
        .then((res) => {
            if (res.status === 200) {
                const tokenData = res.json();
                saveToken(JSON.stringify(tokenData)); //     sessionStorage,   ,  
                return Promise.resolve()
            }
            return Promise.reject();
        });
}

したがって、我々はフィールドでトークンを受け取った「access_tokenは」、「refresh_token」と「expires_in」と、それを保存したのsessionStorageさらなる使用のために。

トークンの更新


先ほど受け取ったトークンには有効期限があり、「expires_in」フィールドで設定されます。有効期限が切れると、ユーザーはリクエストでこのトークンを送信しても新しいデータを受信できなくなるため、新しいトークンを取得する必要があります。

トークンは2つの方法で取得できます。1つ目は、ユーザー名とパスワードをサーバーに送信して再度ログインする方法です。しかし、これは私たちには適していません。一定の時間が経過した後、毎回ユーザーに認証データの再入力を強制するのは間違っているからです-これは自動的に行われるべきです。しかし、自動送信のためにメモリ内のログイン/パスワードのペアのどこかに保存することは、我々が必要とする理由です、安全ではないrefresh_tokenで先に受信された、access_tokenはをそしてsessionStorageに保存されます。このトークンをAPIが提供する別のアドレスに送信することで、応答として新しい「新しい」トークンを取得できます。

トークン更新の機能



function refreshToken(token) {
    return fetch('api/auth/refreshToken', {
        method: 'POST',
        credentials: 'include',
        headers: {
            'Accept': 'application/json',
            'Content-Type': 'application/json',
        },
        body: JSON.stringify({
            token,
        }),
    })
        .then((res) => {
            if (res.status === 200) {
                const tokenData = res.json();
                saveToken(JSON.stringify(tokenData)); //      sessionStorage,   ,  
                return Promise.resolve();
            }
            return Promise.reject();
        });
}

上記のコードを使用して、sessionStorageのトークンを書き換え、新しい方法でAPIにリクエストを送信できるようになりました。

ラッパー関数を作成する


次に、リクエストヘッダーに認証データを追加する関数を作成し、必要に応じて、リクエストを行う前にそれを自動的に更新します。

トークンの有効期限が切れている場合、新しいトークンを要求する必要があるため、関数は非同期になります。このため、async / await構文を使用します。

ラッパー機能



export async function fetchWithAuth(url, options) {
    
    const loginUrl = '/login'; // url   
    let tokenData = null; //    tokenData

    if (sessionStorage.authToken) { //   sessionStorage  tokenData,   
        tokenData = JSON.parse(localStorage.tokenData);
    } else {
       return window.location.replace(loginUrl); //   ,      
    }

    if (!options.headers) { //     headers,   
        options.headers = {};
    }
    
    if (tokenData) {
        if (Date.now() >= tokenData.expires_on * 1000) { //       
            try {
                const newToken = await refreshToken(tokenData.refresh_token); //  ,      refresh_token
                saveToken(newToken);
            } catch () { //   -   ,      
               return  window.location.replace(loginUrl);
            }
        }

        options.headers.Authorization = `Bearer ${tokenData.token}`; //    headers 
    }

    return fetch(url, options); //   ,       headers
}

上記のコードを使用して、APIのリクエストにトークンを追加する関数を作成しました。この関数を使用すると、認証が必要な場合に必要なリクエストのフェッチを置き換えることができます。このため、構文を変更したり、引数にデータを追加したりする必要はありません。
それをファイルに「インポート」して、標準のフェッチをそれに置き換えるだけで十分です。

import fetchWithAuth from './api';

function getData() {
    return fetchWithAuth('api/data', options)
}
