عميل Web Sip قائم على JsSIP + FreePBX

لدى FreePBX عميل Web Sip. كنت أتساءل كيف يعمل. بدأت في تعلم تقنية WebRTC ، صادفت 3 مكتبات - هذه هي JsSIP ، شوكة SIP.js ، وأيضًا sipml5.



تعذر تشغيل JsSIP و SIP.js في المرة الأولى ، لم تكن هناك خبرة كافية. لكن sipml5 كان أسهل استخدام. بعد تنزيل نسخته التجريبية ، اتضح إجراء المكالمة الأولى.



جميع المكتبات الثلاث تعمل. ولكل منها نقاط القوة والضعف الخاصة بها. بعد اختبار جميع المكتبات الثلاث ، اخترت JsSIP.



إيجابيات وسلبيات واجهت:



1.



إيجابيات JsSIP :



  • وزن خفيف؛
  • مدعوم بنشاط من قبل المطورين.


السلبيات:



  • في المرحلة الأولية ، من الصعب البدء ؛
  • اضطررت إلى تحرير المصادر للعمل مع علامة النجمة.


2.



إيجابيات Sip.js :



  • وزن خفيف؛
  • يتم التركيز على العمل مع علامة النجمة.


السلبيات:



  • تعذر بدء تشغيل الصوت للوسائط المبكرة (الصوت غير متصل لرسائل الرسائل المسبقة للمشغل).


3.



إيجابيات Sipml5 :



  • بسيطة نسبيا للمبتدئين.


السلبيات:



  • رمز معقد ومربك ، لم تكن هناك رغبة في فهمه.


تدريب



لا يعمل عميل الويب إلا على صفحة بها دعم تشفير. نحن بحاجة إلى إنشاء شهادة موقعة ذاتيا. كيف يتم ذلك هناك مقالات جاهزة على الإنترنت.



يتيح لك Firefox العمل مع الشهادات الموقعة ذاتيًا. للعمل في Chrome ، سيكون عليك القيام بشهادة الجذر أولاً ، وعلى أساسها قم بالفعل بالتوقيع على شهادة ليعمل الهاتف. يجب استيراد شهادة الجذر إلى مخزن الشهادات.



تمكين الدعم لعميل الويب sip على FreePBX



المسؤول -> إدارة المستخدم -> علامة التبويب "المجموعات" -> علامة التبويب "UCP" -> علامة التبويب "الهاتف"

بالنسبة لخاصية "تمكين الهاتف" ، حدد "نعم".



إذا كان لديك أرقام امتداد في FreePBX ، على سبيل المثال 200 ، فعند تنشيط عميل sip على الويب ، ستظهر أرقام إضافية مع البادئة 99 ، أي 99200.



في مستعرض العميل ، بعد منح الإذن لاستخدام شهادة موقعة ذاتيًا ، لا تزال بحاجة إلى منح الإذن لشهادة العلامة النجمية. للقيام بذلك ، اكتب في المتصفح 



https://192.168.1.70:8089/ws


يجب تحديد عنوان IP الذي يتم تشغيل العلامة النجمية عليه.



تحرير مصدر JsSIP



في مصادر مكتبة JsSIP ، من الضروري استبدال النقل = ws بالنقل = wss. بدون هذا التعديل ، لن يعمل العميل. لا تحتاج إلى تحرير أي شيء في Sip.js ، فقد قدموا لهذه النقطة هناك.



رمز عميل رشفة الويب القائم على JsSIP

<script>
JsSIP.debug.enable('JsSIP:*');
var socket = new JsSIP.WebSocketInterface('wss://192.168.1.70:8089/ws');
var configuration = {
	sockets  : [ socket ],
	//  	
	uri      : '99XXX@192.168.1.70',
	// 
	password : 'passw'
};

var remoteAudio = new window.Audio();
remoteAudio.autoplay = true;

var ua = new JsSIP.UA(configuration);

//   
ua.on('connected', function(e) { /*   */ });
ua.on('disconnected', function(e) { /*   */ });

ua.on('registered', function(e) { /*   */ });
ua.on('unregistered', function(e) { /*   */ });
ua.on('registrationFailed', function(e) { /*   */ });

//  
ua.start();

//   . 
var eventHandlers = {
	'progress': function(e) {
		console.log('call is in progress');

		session.connection.ontrack = function(e) {
			console.log(e);
			remoteAudio.srcObject = e.streams[0];
		};
	},
	'failed': function(e) {
		console.log('call failed with cause: ' + e.cause);
		$('#call').css({'display' : 'flex'});
		$('#hangup').css({'display' : 'none'});
	},
	'ended': function(e) {
		console.log('call ended with cause: ' + e.cause);
		$('#call').css({'display' : 'flex'});
		$('#hangup').css({'display' : 'none'});
	},
	'confirmed': function(e) {
		console.log('call confirmed');
		console.log(e);
	}
};

var options = {
	'eventHandlers'    : eventHandlers,
	'mediaConstraints' : { 'audio': true, 'video': false }
};

//   
$('#call').click(function(e) {
	session = ua.call($('#num').val(), options);
	$('#call').css({'display' : 'none'});
	$('#hangup').css({'display' : 'flex'});
});

//    
$('#hangup').click(function() {
	if (session) {
		session.terminate();
	}

	$('#call').css({'display' : 'flex'});
	$('#hangup').css({'display' : 'none'});
});
</script>
</html>

ملخص



على مدار العامين الماضيين من استخدام عميل Web sip ، حدثت مشاكل ظهرت عند تحديث المتصفح.



على سبيل المثال ، في الإصدار 63 من Firefox ، اضطررت إلى تغيير بروتوكول SDP حتى لا يتم إنهاء الجلسة. توقف المتصفح عن الاتصال بدون المعلمة a = mid: 0. في الإصدارات الأخيرة لم تعد ذات صلة ، تم إصلاح الخطأ ، في الإصدارات اللاحقة لا توجد مشكلة من هذا القبيل.



في الإصدار 65 من Firefox مع المرجع. بدأ صوت المحاور في الاختفاء أثناء المكالمة ، لم تكن هناك مثل هذه المشاكل في الشبكة المحلية. المشكلة هي فقط عند استدعاء من خلال البوابة. في الكروم ، لم يلاحظ مثل هذه المشكلة ، كل شيء يعمل بثبات.



يعمل معظم المستخدمين في Firefox ، ولهذا السبب كان من الضروري إزالة إعادة التفاوض على بروتوكول SDP في المكتبة. مشاكل في فقدان الصوت بعد توقف الجواب عن الإزعاج.
