カスタムボトムシート：仕組み

序文


少し前に、Bottom Sheetに関する記事を読み、同僚と相談した後、このことをアプリケーションに実装することにしました。ボトムシートにいくつかのステップ（Googleマップなど）で移動する方法を教えるのに十分な時間を費やした後、私はその鈍さと限界に気づきました。2つの解決策が見つかりました：リアルタイムで細長いカーテンの状態を監視するストリームにすべてをラップすること、またはBottomSheetBehaviorクラスのいくつかのメソッドを書き直すことです（幸い、Apache 2.0ライセンスではこれを行うことができます）。2番目のオプションを選択したのは、それがより実行可能であるように思えたためです。今、私の経験に基づいて、何がそれから生じたのかを説明しようと思います。これが誰かに役立つことを願っています。さて、始めましょう。


カスタマイズ


プロジェクトを作成する段階、Googleのボトムシートを追加する段階は省略します。これを読んだ場合、上記のすべてがすでに解決されているためです。BottomSheetBehaviorクラスに移動し、そのコンテンツを、作成した新しいCustomBottomSheetBehaviorクラスにコピーする必要があります。コピーしたら、赤でマークされているすべての名前を変更することを忘れないでください。また、BottomSheetの.xmlファイルで、この行の名前を変更し、新しいクラスを参照する必要があります。次のような結果になります。


app:layout_behavior="com.my.custombottomsheetexample.CustomBottomSheetBehavior"

それで、恐ろしく見える700行以上のJavaコードがあるので、それらを使って何をしますか？
まず、onViewReleased（）メソッドを見つける必要があります。


public void onViewReleased(@NonNull View releasedChild, float xvel, float yvel) {

        ...

}

数行変更するだけです。最初のif {...}を見つけ、その内容を次のコードに置き換えます。


if (yvel < 0.0F) {
        currentTop = releasedChild.getTop();
        if (currentTop > CustomBottomSheetBehavior.this.halfExpandedOffset) {
        top = CustomBottomSheetBehavior.this.halfExpandedOffset;
        targetState = 6;
} else {
        top = 0;
        targetState = 3;
    }
}

これで、BottomSheetは2つの段階で進み、途中で停止します。


（）


高さ変更


しかし、それだけではありません。平均の高さを調整できます。これを行うには、すべて同じクラスで、変数halfExpandedOffsetの初期化を見つけます。


this.halfExpandedOffset = this.parentHeight / 2;

この線を見ると、パーセンテージでも高さを好きなように調整できることがわかります。個人的に、私はこのように変更することを好みました：


this.halfExpandedOffset = this.parentHeight * 60 / 100;

（）


BottomSheetは、メインビューの高さの40％を拡張します。それで全部です！
