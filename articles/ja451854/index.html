Swiftとは何ですか？


Swiftは、iOS、macOS、watchOS、tvOS、さらにはLinux向けのソリューションを開発するためにAppleが開発した、ユニバーサルでマルチパラダイムのプログラミング言語です。

  • 読みやすさ -Swiftの構文は非常に簡潔で、書き込みと読み取りが簡単です。
  • 簡単なサポート -出力はコードと継承レベルがはるかに少なく、プロジェクト全体が単一の迅速なファイルになります。
  • 安全なプラットフォーム -コードの作成中にバグをコンパイルして修正します。
  • 高速 -信じられないほど高速で効率的なLLVMコンパイラがSwiftコードをネイティブコードに変換し、デバイスを最大限に活用します。構文自体と標準ライブラリも迅速な作業のために最適化されています。
  • 動的ライブラリのサポート。
  • オープンソース。

クラスと構造の違いは何ですか？


Swiftでは、クラスと構造の間に4つの主な違いがあります。クラスには、構造にはないいくつかの機能があります。

  • 型キャスト -実行時にクラスを確認および解釈できます。
  • リンクカウント -クラスのインスタンスごとに複数のリンクを使用できます。
  • 継承 -あるクラスが別のクラスの特性を継承できるようにします。
  • Deinitializers-クラスの各インスタンスが、それに割り当てられたすべてのリソースを解放できるようにします。

構造体がコードで渡されると、参照カウントなしで常にコピーされます。構造体のインスタンスは常に値によって渡され、クラスのインスタンスは常に参照によって渡されます。

どの場合にクラスを使用し、どのような場合に-構造？


簡単なチートシートとして：次の条件の1つ以上に該当する場合、構造体を使用する必要があります。

  • この構造の目的は、いくつかの比較的単純なデータ値をカプセル化することです。
  • カプセル化された値は、参照ではなくコピーされることが期待できます。
  • 構造体に格納されているプロパティは、それ自体もコピーされる値型であり、参照されません。
  • 構造体は、別の既存のタイプのプロパティと動作を継承してはなりません。

その他の場合は、クラスを使用します。クラスを定義し、インスタンスを作成して管理し、参照渡しします。

変数をリンクとして渡す方法は？


変数は、inoutパラメータを使用して参照として渡すことができます。Inoutは、ローカル変数を変更すると、渡されるパラメーターも変更されることを意味します。

var value: String = “Apple”

func changeString(newValue:inout String) {
 newValue = “Samsung”
 print(newValue) // Output:Samsung
 print(value) // Output:Samsung
}

changeString(newValue:&value)


モジュールとは？


  • モジュールは、コードの配布における個別のユニットです。
  • 別のユニットとして作成および配布され、別のモジュールでインポートできるプラットフォームまたはアプリケーション。
  • 各ビルド目標-Xcodeのアプリケーションパッケージまたはフレームワークは、個別のモジュールと見なされます。

Swiftのアクセスレベルの違いは何ですか？


Swiftには、コード内のエンティティに対して5つの異なるアクセスレベルがあります。

  • オープンアクセス - オープンアクセスのクラスは、サブクラスになるか、それらが定義されているモジュール内、またはそれらが定義されているモジュールをインポートする他のサブクラスによってオーバーライドされます。
  • パブリックアクセス - パブリックアクセスを持つクラスは、サブクラスにすることも、それらが定義されているモジュール内でのみサブクラスによって再定義することもできます。
  • 内部アクセス -エンティティは、定義モジュールの任意のソースファイルで使用できますが、このモジュール外のソースファイルでは使用できません。
  • ファイルプライベートアクセス -エンティティの使用は、独自の定義ソースファイルによって制限されます。
  • プライベートアクセス -エンティティの使用は、同じファイルにある添付の宣言とこの宣言の拡張子によって制限されます。

オープンアクセスは最も高く、最も制限の少ないレベルですが、プライベートアクセスは最も低く、したがって最も制限されています。デフォルトでは、コード内のすべてのエンティティに内部アクセスレベルがあります。

初期化の遅延とは何ですか？


遅延初期化は、オブジェクトの作成または別のプロセスの実行を、このプロセスが必要になるまで遅らせる手法です。遅延はクラスと構造体でのみ使用できます。ただし、遅延プロパティは自動的に初期化されないため、安全ではないことを理解する必要があります。

常にvarを使用して、遅延プロパティを変数として宣言する必要があります。定数のプロパティは、初期化が完了する前に常に値を持つ必要があるため、遅延することはありません。

タプルとは何ですか？


タプルは、単一の値として表される0個以上の値のグループです。それらは通常、呼び出し関数から複数の値を返すために使用されます。Swiftには2種類のタプルがあります。

Motorcadeに名前を付ける

let nameAndAge = (name:”Midhun”, age:7)
Access the values like:
nameAndAge.name //Output: Midhun
nameAndAge.age //Output: 7


名前のないタプル
このタイプのタプルでは、​​要素の名前を指定しません。

let nameAndAge = (“Midhun”, 7)
Access the values like:
nameAndAge.0 //Output: Midhun
nameAndAge.1 //Output: 7


転送とは何ですか？


列挙を使用して、関連する値のグループの一般的なタイプが決定され、コードでこれらの値を安全に処理する機能が提供されます。CやObjective-Cとは異なり、Swiftでは、列挙型は作成時に整数のデフォルトに割り当てられません。

関連する値とは何ですか？


関連する値は、列挙型のケースの1つに関連する変数と非常に似ています。 

enum Barcode {

case upc(Int, Int, Int, Int)

case qrCode(String)
}


バーコード 列挙のタイプを判別します。これは、タイプの関連する値（Int、Int、Int、Int）を持つupc値、またはStringタイプの関連する値を持つqrCode値のいずれかを取ることができます。

場合によっては、ケース値の隣に他のタイプの関連する値を格納する機能が役立つことがあります。これにより、追加のユーザー情報をケース値とともに保存でき、このケースをコードで使用するたびにこの情報を変更できます。
