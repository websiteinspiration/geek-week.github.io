コンパイルせずにJava 11で単一ファイルプログラムを実行する



ソースファイルHelloUniverse.javaに、クラス定義とmain1行のテキストを端末に出力する静的メソッドを含めます。

public class HelloUniverse{
      public static void main(String[] args) { 
            System.out.println("Hello InfoQ Universe");
      }
}

通常、このクラスを実行するには、まずJavaコンパイラー（javac）を使用してコンパイルする必要があります。これにより、HelloUniverse.classファイルが作成されます。 

mohamed_taman$ javac HelloUniverse.java

次に、Java仮想マシンコマンド（インタープリター）を使用して、結果のファイルを実行する必要があります。

mohamed_taman$ java HelloUniverse
Hello InfoQ Universe

次に、virtualkaが最初に起動し、クラスをロードしてコードを実行します。

そして、コードの一部をすばやく確認する必要がある場合はどうでしょうか。それとも、Java（この場合は重要なポイント）を初めて使用し、言語を実験したいですか？説明されている2つの手順は、事態を複雑にする可能性があります。

Java SE 11では、中間コンパイルなしで単一のソースファイルを直接実行できます。

この機能は、単純なプログラムで作業したい初心者に特に役立ちます。jshellと組み合わせると、初心者を教育するための優れたツールセットが得られます。

専門家はこれらのツールを使用して、言語の革新を学び、慣れていないAPIをテストできます。私たちの意見では、スクリプトの形式でJavaプログラムを作成し、その後OSシェルから実行するなど、多くのタスクを自動化することをお勧めします。その結果、シェルスクリプトを柔軟に処理し、Javaのすべての機能を使用できます。これについては、記事の後半で詳しく説明します。

Java 11のこの優れた機能により、コンパイルせずに単一のソースファイルを直接実行できます。話し合いましょう。

Javaで.javaを実行する


JEP 330 関数（ソースコードで単一ファイルプログラムを実行）は、JDK 11で登場しました。これにより、インタープリターを使用せずに、Javaソースコードでソースファイルを直接実行できます。ソースコードはメモリ内でコンパイルされ、ディスク上に.classファイルを作成せずにインタープリターによって実行されます。

ただし、この関数は、単一のファイルに格納されているコードに限定されます。複数のソースファイルを同時に実行することはできません。

この制限を回避するには、すべてのクラスを1つのファイルで定義する必要があります。それらの数に制限はありません。さらに、それらが同じファイルにある間、それらがパブリックであるかプライベートであるかは問題ではありません。

ファイルで定義されている最初のクラスはメインクラスと見なされ、メインメソッドを配置する必要があります。つまり、順序が重要です。

JavaインタープリターがHelloUniverseプログラムを実行する方法


JDK 10では、Javaランチャーは次の3つのモードで動作できます。

  1. クラスファイルを実行します。
  2. JARファイルからのメインクラスの実行。
  3. モジュールのメインクラスの実行。

そしてJava 11では、4番目のモードが登場しました：

  1. ソースファイルで宣言されたクラスの実行。

このモードでは、ソースファイルがメモリでコンパイルされ、このファイルの最初のクラスが実行されます。

システムは、次の2つの基準に従って、ソースファイルを入力する意図を決定します。

  1. コマンドラインの最初の項目は、オプションでもオプションの一部でもありません。
  2. 行にはオプションを含めることができます--source <vrsion>。

最初のケースでは、Javaは最初にコマンドの最初の要素がオプションかそれの一部かを調べます。これが.javaで終わるファイル名である場合、システムはソースコードをコンパイルして実行すると見なします。ソースファイルの名前の前にJavaコマンドにオプションを追加することもできます。たとえば、ソースファイルが外部依存関係を使用するときにクラスパスを設定する場合などです。

2番目のケースでは、ソースファイルを操作するモードが選択され、オプションではないコマンドラインの最初の要素は、コンパイルして実行する必要があるソースファイルと見なされます。

ファイルに.java拡張子がない場合は、オプション--sourceを使用して、ソースファイルを操作するモードに強制的に移行する必要があります。

これは、ソースファイルが実行する必要のある「スクリプト」であり、ファイル名がJavaコードでソースファイルに名前を付けるための通常の規則に準拠していない場合に重要です。

このオプションを使用する--sourceと、ソース言語のバージョンを判別できます。これについては以下で説明します。

    }

Javaでスクリプトが重要なのはなぜですか？


まず、スクリプトとは何かを思い出してみましょう。

スクリプトは、人が順番に実行できるタスクまたはコマンドの実行を自動化する特定のランタイム環境用に作成されたプログラムです。

この一般的な定義から、スクリプト言語の簡単な定義を導き出すことができます。これは、高水準の構造を使用して一度に1つまたは複数のコマンドを解釈および実行するプログラミング言語です。

スクリプト言語は、ファイルに記述された一連のコマンドを使用します。多くの場合、このような言語は（コンパイルではなく）解釈され、手続き型プログラミングスタイルに準拠しています（ただし、一部のスクリプト言語にはオブジェクト指向言語のプロパティもあります）。

一般に、スクリプト言語は、Java、C、C ++などのより構造化されたコンパイル済み言語と比較して、学習が簡単でタイピングが高速です。サーバー側のスクリプト言語には、Perl、PHP、Pythonが含まれ、クライアント側ではJavaScriptが含まれます。

長い間、Javaは適切に構造化され、高度に型付けされたコンパイル言語であり、仮想マシンによって解釈されて任意のコンピューティングアーキテクチャで実行されると考えられていました。ただし、Javaは、他のスクリプト言語に比べて、習得やプロトタイプ作成が簡単ではありません。

それにもかかわらず、Javaはすでに24歳になり、世界中の約1,000万人の開発者によって使用されています。最近のリリースでは、若いプログラマがこの言語を習得しやすくするだけでなく、コンパイルやIDEなしで言語やAPI関数を使用できるようにするために、いくつかの新機能が追加されています。たとえば、Java SE 9では、対話型プログラミングをサポートするJShellツール（REPL）が導入されました。

また、JDK 11のリリースにより、この言語はスクリプトをサポートできるようになりました。これにより、簡単なコマンド呼び出しでコードを実行できるようになりましたjava。

Java 11でスクリプトを使用するには、主に2つの方法があります。

  1. チームへの直接電話java。
  2. コマンドラインで* nixスクリプトを使用する（Bashスクリプトと同様）。

最初のオプションについてはすでに検討しましたが、ここでは2番目のオプションについて説明します。それは私たちに多くの可能性を開きます。

シバンファイル：Javaをシェルスクリプトとして実行する


そのため、Java SE 11は* nixの世界の従来のシバンファイルを含むスクリプトのサポートを導入しました。それらをサポートするために、言語仕様は必要ありませんでした。

シバンファイルでは、最初の2バイトは0x23と0x21でなければなりません。これは、ASCII文字エンコーディング＃！です。ファイル内の後続のバイトはすべて、このプラットフォームのデフォルトのエンコーディングシステムに基づいて読み取られます。

したがって、OSの組み込みシバンメカニズムを使用してファイルを実行するには、最初の行が＃！で始まるという1つの要件のみを遵守する必要があります。つまり、Javaランチャーが明示的に使用されている場合、特別な最初の行は不要です。 HelloUniverse.javaの場合と同様に、ソースファイルからコードを実行します。macOS Mojave 10.14.5を

実行しているターミナルで次の例を実行します。しかし、最初に、シバンファイルを作成するときに従うべき重要なルールを定義します。

  • JavaコードをOSシェルスクリプトのスクリプト言語コードと混在させないでください。
  • 仮想マシンのオプションを追加する必要がある場合は、shebangファイルの実行可能ファイルの名前の後に最初のオプションを設定する必要があります--source。仮想マシンのオプションについては、次のとおり--class-path、--module-path、--add-exports、--add-modules、--limit-modules、--patch-module、--upgrade-module-path、および任意の変動が挙げられます。また、JEP 12で--enable-preview説明されている新しいオプションがこのリストに含まれる場合があります。
  • ソースファイルで使用されるJavaのバージョンを指定する必要があります。
  • ファイルの最初の行は、シバン文字（＃！）で始まる必要があります。例えば：
    #!/path/to/java --source <vrsion>

    chmod +x <Filname>.<Extnsion>。
