Walkthrough Bandit dari overthewire.org

Selamat siang bandit. Setelah menyaksikan musim pertama Mr. Robot, saya memutuskan bahwa itu tidak lebih buruk daripada karakter utama dan menginstal Kali Linux untuk saya sendiri, siap untuk berselancar di Internet untuk mencari kerentanan dan gambar gadis-gadis telanjang di telepon saudara saya . Setelah meluncurkan nmap tiga kali dan menemukan, tanpa banyak kejutan, bahwa hidup bukanlah film, saya memutuskan untuk mengambil pendekatan yang lebih terstruktur untuk pencarian saya dan mulai dengan dasar-dasar keamanan informasi dan pencarian kerentanan.

Langkah saya berikutnya adalah mengunjungi r / meretas reddit.com dan membaca posting yang disematkan yang menyebutkan overthewire.orgdan game Bandit, sebagai salah satu "peretas" yang cocok untuk pemula. Saya yakin bahwa pembaca memiliki gagasan tentang game Capture The Flag (mengingat pusat di mana saya menerbitkan), tetapi, untuk berjaga-jaga, saya akan menjelaskan. Anda terhubung ke mesin jarak jauh dan di ruang terisolasi Anda mencari kunci untuk terhubung ke mesin berikutnya menggunakan rekomendasi di situs. Selanjutnya, untuk saya dan Anda semua, saya akan memberikan Anda bagian dari 10 level pertama, karena merekam semata-mata agar tidak lupa dan tidak berbagi dengan siapa pun adalah dosa. Pergilah!

PS Semua kata sandi disembunyikan di belakang spoiler, baca dan jangan khawatir.

Level 0


Anda perlu terhubung ke host di bandit.labs.overthewire.org, port 2220 dan masuk sebagai bandit0 dengan kata sandi bandit0. Tidak lebih cepat dikatakan daripada dilakukan. Kami menggunakan sintaks perintah standar ssh.

:~$ ssh bandit0@bandit.labs.overthewire.org -p 2220
This is a OverTheWire game server. More information on http://www.overthewire.org/wargames

bandit0@bandit.labs.overthewire.org's password: <b>bandit0</b>
...
bandit0@bandit:~$

Tingkat 1


Kata sandi untuk pergi ke tingkat berikutnya ada di file readme di direktori home. Untuk membukanya, kita membutuhkan tim cat.

bandit0@bandit:~$ cat readme

Kata sandi
boJ9jbbUNNfktd78OOpsqOltutMc3MY1

Di akhir setiap level, kami terhubung menggunakan sshpengguna berikut untuk localhost:

Saya tidak memperhatikan peringatan tentang localhost, karena saya tidak punya izin untuk membuat file, jelas. Tapi ini tidak berhenti bergerak ke level selanjutnya.

Level 2


Kata sandi untuk tingkat berikutnya disimpan dalam file dengan nama -dan ini adalah satu-satunya cara berbeda dari yang sebelumnya. Kami menggunakan perintah yang sama catdengan sedikit perubahan:

bandit1@bandit:~$ cat ./-

Kata sandi
CV1DtqXWVFXTvM2F0k09SHz0YwRINYA9

bandit1@bandit:~$ ssh bandit2@localhost

Kami melewati peringatan di telinga kami, terhubung ke bandit2 dan pergi ke tingkat berikutnya.

Tingkat 3


Pada level ini, file yang spaces in this filenameberisi kata sandi memiliki spasi dalam namanya. Ini juga mudah ditangani:

bandit2@bandit:~$ cat "spaces in this filename"

Kata sandi
UmHadQclWmgdLOKQ3YNgjWxGoRMb5luK

bandit2@bandit:~$ ssh bandit3@localhost

Level 4


Kata sandi untuk tingkat berikutnya terletak di file tersembunyi di direktori inhere, untuk menunjukkannya di konsol kami menggunakan perintah lsdengan opsi -a. Ayo pergi ke direktori.

bandit3@bandit:~$ cd inhere
bandit3@bandit:~/inhere$ ls
bandit3@bandit:~/inhere$ ls -a
.  ..  .hidden
bandit3@bandit:~/inhere$ cat .hidden

Kata sandi
pIwrPrtPN36QITSp3EQaw936yaFoFgAB

bandit3@bandit:~/inhere$ ssh bandit4@localhost

Level 5


Kata sandi ada dalam satu file yang dapat Anda baca (secara harfiah) dalam suatu direktori inhere. Untuk mendapatkan informasi tentang file kami akan menggunakan perintah file. Pertama, mari kita pergi ke direktori dan melihat apa yang ada di dalamnya:

bandit4@bandit:~$ cd inhere
bandit4@bandit:~/inhere$ ls
-file00  -file01  -file02  -file03  -file04  -file05  -file06  -file07  -file08  -file09

Semua file dimulai dengan -, jadi kami menggunakan sintaks dari level kedua dan melengkapi *untuk mendapatkan informasi tentang semua file sekaligus

bandit4@bandit:~/inhere$ file ./-*
./-file00: data
./-file01: data
./-file02: data
./-file03: data
./-file04: data
./-file05: data
./-file06: data
./-file07: ASCII text
./-file08: data
./-file09: data

Seperti yang bisa kita lihat, satu-satunya file yang bisa kita baca adalah ini -file07. Apa yang akan kita lakukan dengan menggunakan tim yang sudah kita kenal cat.

bandit4@bandit:~/inhere$ cat ./-file07

Kata sandi
koReBOKuIDDepwhWk7jZC0RTdopnAYKh

bandit4@bandit:~/inhere$ ssh bandit5@localhost

Sekarang, selain keterbacaan file, Anda perlu mempertimbangkan ukuran 1033 byte dan fakta bahwa file tersebut tidak dapat dieksekusi. Untuk melakukan ini, gunakan perintah finddengan parameter -size 1033c. Pertama, ubah direktori, tentu saja, dan lihat apa yang ada di dalamnya.

bandit5@bandit:~/inhere$ ssh bandit6@localhost

Kami dengan mudah menghindari berkeliaran di folder sebagai sebuah tim file, ini adalah kabar baik!

Kata sandi
millionth cvX2JJa4CFALtqS87jk27qwqGhBM9plV

bandit7@bandit:~$ ssh bandit8@localhost
