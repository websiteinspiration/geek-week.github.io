VPSの選択方法

必要なものとどのような種類のWebホスティングについては、「ホスティング：オプション、比較、ユーザー統計」の投稿を参照してください。ここでは、すぐに記事のトピックに進みます。そのタイトルは、紹介よりも重要です。  もちろん、この記事では、RUVDSサービスについて説明しながら、実践からの例を示します。しかし、投稿の主な目的は、ホスティングプロバイダーを選択し、レンタルする仮想専用VPSまたはVDSサーバーを構成するための包括的なチェックリストを提供することです。





ホスティングプロバイダーの選択


▍評判とレビュー


さまざまなサイトのレビューを読むとき（ここにはリストしません。通常の検索「[プロバイダー名]レビュー」には多くのオプションがあります）、不満のあるユーザーの赤いケースだけでなく、ホスティングプロバイダー自体の回答に注意してください。競合他社は眠っていないことを覚えておいてください。 

あなたの将来の隣人は、プロバイダーの評判について雄弁に話します：どの会社がこのホスティング業者のクライアントであるかを見て、結論を引き出してください。さらに、IT分野でよく知られているサイトを定期的に実施する独立したサーバーテストの結果を読んでください：オタク向けのWebマガジン（xakep.ruなど）、分析センター（acexpert.ruなど）

この情報は、民間の「専門家」の意見よりも客観的です「これらのブロガーの収入源であることが知られている彼らのブログのページ。

▍技術サポート


たとえば、オンラインストアサイトがブラックフライデーに「かかっている」場合や、頻繁にアクセスするサイトへのアクセスが中断された場合など、テクニカルサポートホスティング業者の回答の速度と品質が決定的な要素になる場合があります。通常モードでのチケットの平均応答時間は、15〜20分を超えてはなりません。もちろん、プロバイダーにはさまざまな状況がありますが、回答が遅れる場合は、単なる「申し訳ありません」ではなく、この遅れの理由を適切に説明したいと思います。これは、ホスティングサービスのレベルも決定します。

resourcesリソースの支払い能力


「リソースの」料金は、サーバーが日中に提供した実際の負荷を考慮に入れています。このオプションは、常にではなく、ピーク時に大容量のサーバーを必要とする可能性があるプロジェクトや企業に適しています。この場合、コンピューティングリソースは常に在庫にあり、自動的に接続されます。使用料は、予約された分ではなく、使用された分だけ差し引かれます。詳細はこちら。

ただし、毎月の請求-サーバーの毎月の固定料金プラン-には、月額費用を100％制御し、選択した構成の仮想サーバーをリソースで保証できるため、利点もあります。さらに、無料トライアル期間は通常このレートで提供されます。

ちなみに、RUVDSでは、最も安価な仮想VPSサーバーを1か月あたり30ルーブルでレンタルできます。高速SSDディスク上のストレージ用に10 GB、無制限のトラフィック、1つの専用IPv4アドレス、1秒あたりの料金と返金保証。詳細はこちら（料金の詳細はこちら）。

▍返金保証


まず、無料の試用期間があれば良いです。おそらくすべての構成で提供されるわけではありませんが、ホスティング業者が限られたオプションであなたに適しているかどうかを理解することは可能です。次に、ホスティングが気に入らなかったが、すでに「支払われた」場合の、クライアントへの実際の払い戻し。これを返金保証といいます。

▍使用されているオペレーティングシステム


VPS / VDSをレンタルするには、オペレーティングシステムとソフトウェアのインストールなど、最大の権限を持つ専用サーバーハードウェアリソースを使用してクライアントを使用する必要があります。ただし、すべてのOSがホストのサーバーでサポートできるわけではなく、サーバーにインストールできるアプリケーションに依存します。 

▍プリセットプログラム


基本的なホスティングサービスに加えて、プロバイダーは顧客に追加の「製品」を提供します。これは、非専門家や、時間のないWebホスティングのすべての複雑なことを理解したくない人でもサーバーで機能します。そのような追加は、単一のインターフェース（Plesk、ISPmanagerなど）による便利なサーバー管理のためのコンテンツ管理システム（CMS）またはWebコンソールです。

▍サーバーホストのリソース制限


現在の要件と計画された要件が異なる複数のプロジェクトがある場合は、選択するホスティングプロバイダーが、これらのプロジェクトのライフサイクルを通じてこれらの要件を満たすことができたことを確認することが重要です。パフォーマンスに影響を与えるサーバーリソースは、プロセッサ能力（CPU）、ディスク容量、およびRAMです。サーバー、FTPアカウント、データベース、メールボックスでホストできるサイトの最大数は、これに依存します。これらの指標は広い範囲にあるはずです。簡単に言えば、専用の仮想サーバーの構成を選択するための多くのオプションが必要です。

VPS / VDSサーバーの構成


▍OSの選択 


インストールする予定のプログラムをサポートするオペレーティングシステムによって異なります。データベース、スクリプト、コントロールパネル、アプリケーションなどです。それらの要件をお読みください。

CPU CPUの選択


ここでは、プロセッサのタイプを選択します。これは、コアの数と1秒あたりの処理されたプロセスの数に関与するクロック周波数によって特徴付けられます。数値が大きいほど、プロセッサは強力になります（たとえば、3.4 GHzの周波数は、プロセッサが1秒間に30億から4億の操作を処理できることを意味します）。

RAM RAMの選択


これは、プロバイダーが保証するRAMの量です。まあ、範囲が1GBから16GBの間であれば、プロジェクトのスケーラビリティは向上します。
 

▍ビデオRAM


ゲームのあるサイトでは、メモリ容量を選択できるビデオカード接続サービスを用意することが重要です。たとえば、RUVDSには最近そのようなサービスがあり、64 Mb（最大サイズ1024 Mb）で月額125ルーブルの費用がかかります。注意！ビデオカードを搭載したサーバーは、プロセッサ周波数3.4 GHzのRUCLOUDデータセンターで注文できます。

さらに、ビデオカードのリソースは、人気のあるソフトウェアの開発者によって積極的に使用されています。最新のブラウザは、グラフィックアクセラレータを使用できる場合、サイトのページをより高速にレンダリングします。これらの同じブラウザには、WebGLで実行される3Dアプリケーションとゲームがあることは言うまでもありません。

discディスクタイプを選択


仮想サーバーが配置されているハードディスクの種類とサイズは、仮想サーバーに格納できる情報の量、耐負荷、およびサーバーが対応するアプリケーションにどの程度適切にサービスできるかに影響します。通常、これらはHDDまたはSSDドライブで、20〜600 TBのボリュームを選択できます。

RUVDSは「Huge Disk」サービスを提供します。これは、HDD RAIDにある追加の大容量仮想ディスクを接続する機会です。1TBで500 r /月、最大容量18 TBです。 

allocated割り当てられたIPアドレスの数 


サイトのドメイン名は、このサイトが配置されているサーバーのIPアドレスに対応しています。ブラウザのアドレスバーにサイトのURLを入力すると、DNSサーバーが呼び出されます。DNSサーバーは、ドメイン名によって目的のWebサーバーのIPを判別し、要求をサーバーにリダイレクトします。 1つのIPアドレスは、サーバーでホストされている1つ、複数、またはすべてのWebサイトに対応できます。最近のバージョンでは、ネイバーのリソースへの多数のリクエストにより、サーバーでホストされている他のサイトの読み込み時間が長くなります。さらに、このIPアドレスにあるサイトの1つからスパムが送信された場合、メールサーバーはIPアドレスをブラックリストに追加できます。そして、すべてのサイトの受信メールと送信メールがブロックされます。したがって、複数の専用IPアドレスを取得できる必要があります。最大数は16個です。

webウェブホスティングとサーバーを管理するためのパネル


サーバー管理は、経験の浅いサイト所有者にとっては暗い森であり、創造的な仕事でもあり得ることは既に述べました。これは、この制御が行われるインターフェースによって異なります。ユーザーが快適なWeb​​ブラウザー環境でさまざまなサーバーサービスを管理できるWebコンソール（コントロールパネル）があります。それらが何であるかについては、レビュー記事「What's New in Web Consoles 2019」で読むことができます。RUVDSでは、ISPマネージャー（年末まで共有で無料で利用可能）、Cpanel、Plesk、およびPlesk Obsidianの新しいバージョン（VPSを注文する場合）のWebコンソールを提供しています2019年12月31日まで無料）。注意！RUVDSのコントロールパネルは、LinuxベースのOSでのみ使用できます。

ホスティングオプションの選択は、サイトの種類、負荷、および予想されるスケーラビリティによって異なります。サイトは、静的（会社のサービスの説明とサイトに関する一般情報を含む複数のページを含むHTMLサイト）、コンテンツの公開（テキスト、写真、ビデオ）、ブログ、または商品の販売（オンラインストア）にすることができます。それぞれにホスティングに関する独自の要件があります。サイトが記述されているプログラミング言語、そのためのアプリケーション言語、データベースのタイプ、保存されているファイルの数とボリューム（たとえば、ビデオが豊富であるか、トラフィックが多いと推定されるサイトでは、高いネットワーク帯域幅と大容量）など。適切なホスティングを選択するには、これらの要件を理解し、プロジェクト（サイト）の目標を理解する必要があります。
