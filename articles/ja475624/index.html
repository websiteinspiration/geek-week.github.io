PHP-Watcher：長寿命のアプリケーションの開発を簡素化するツール



私たちはPHPのシンプルさを気に入っています。コードを記述し、ブラウザーでページを更新すると、すぐに変更が表示されます。しかし、コンソールコマンドに関しては、これは長期間のプロセスになる可能性があります。たとえば、非同期のHTTPサーバーを作成してファイルをアップロードする場合、開発は非常に苦痛になる可能性があります。

PHPエコシステムには、ソースコードが変更されたときにアプリケーションを自動的に再起動する適切なソリューションがありませんでした。そこで私は自分のツールを作成することにしました-純粋なPHPで、Composerを介してアクセスできます。

Nodemonに似ていますが、PHPで


しばらくの間、私自身はNodemonを使用していました。これはNode.jsの世界のツールですが、少し調整すれば、PHPスクリプトで使用できます。しかし、実際には、Node.jsをインストールして、不明なNPMパッケージの束を非同期PHPアプリケーションにドラッグして再起動したくありません。

私はツイッターで積極的に話しているので、他にこの問題があり、解決策を得たいと思っている人に尋ねました。関心を見て、私は、Nodemonと同じ機能を提供するツールを書くために座っただけでPHPにし、PHPのために。



1か月が経過しました。PHP -Watcherを使用すると、長寿命のPHPアプリケーションを開発するためにNodemonまたはその他のNPMパッケージをインストールする必要がなくなります。

それはそれがどのように機能するかです


ライブラリはComposerからインストールできます。

composer global require seregazhuk/php-watcher

Symfonyをベースにした長寿命のアプリケーションに取り組んでいるところを想像してみてください。このアプリケーションのエントリポイントは、public / index.phpファイルです。srcフォルダーとconfigフォルダーの変更を追跡する必要があります。また、ソースコードや構成パラメータを変更するとすぐに、アプリケーションを自動的に再起動する必要もあります。この問題を解決する方法は次のとおりです。

php-watcher public/index.php --watch src --watch config

このコマンドは、スクリプトpublic / index.phpを実行します。これにより、srcおよびconfigディレクトリの変更の追跡が開始されます。それらのいずれかでファイルが変更されるとすぐに、PHP-Watcherはスクリプトを再起動します。

デフォルトでは、PHPファイルへの変更のみを追跡します。しかし、symfonyはその設定をyamlに保存します。したがって、PHPとyamlの両方のファイルを追跡するように、ウォッチに明示的に通知する必要があります。これは--extオプションを使用して行われます。

php-watcher public/index.php --watch src --watch config --ext php,yaml

srcディレクトリ内で変更を加えてもアプリケーションを再起動する必要がないことを理解しているとします。たとえば、src / Migrationsサブディレクトリの変更は無視します。この場合、-ignoreオプションを使用できます。

php-watcher public/index.php --watch src --watch config --ext php,yaml --ignore Migrations

これで、PHP-Watcherはsrcおよびconfigディレクトリの変更の追跡を開始しますが、Migrationsサブディレクトリ内の変更は無視します。さらに、デフォルトでは、すべてのドットファイルとVCSファイルの変更が無視されます。

Watcherは、コマンドラインオプションだけでなく、構成ファイルを使用して、その動作のカスタマイズをサポートしています。コマンドラインで毎回一連のオプションとパラメーターを渡したくない場合は、.php-watcher.yml構成ファイルを作成できます。たとえば、前のコマンドは次の構成ファイルで置き換えることができます。

watch:
  - src
  - config
extensions:
  - php
  - yaml
ignore:
  - Migrations

このようなファイルがあれば、再起動する必要のあるPHPスクリプトのみを指定するだけで、「watch」をアクティブ化できます。他のすべての設定はファイルから取得されます。

php-watcher public/index.php

構成ファイルとコマンドライン引数の両方がある場合はどうなりますか？この場合、コマンドラインから渡されたすべての引数は、構成ファイルの対応する値を上書きします。

デフォルトでは、PHP-WatcherはPHP実行可能ファイルを使用してスクリプトを実行します。ターミナルでコマンドを書きます：

php-watcher public/index.php

内部では、phpコマンドpublic / index.phpを使用して子プロセスが作成されます。ほとんどの場合、これが必要です。ただし、環境に別の実行可能ファイルがある場合は、実行するコマンドを明示的に指定できます。たとえば、同じシステム上に複数のバージョンのPHPがあり、php7.4実行可能ファイルを使用してアプリケーションを実行する場合、-execオプションを使用して実行可能ファイルを指定できます。

php-watcher public/index.php --exec php7.4

構成ファイルを介して同じこと：

watch:
  - src
  - config
executable: php7.4

PHP-Watcherは、クラッシュしてもアプリケーションを自動的に再起動しません。開発環境では、これは特に必要ありません。新しいアプリケーションを開発しているときに、クラッシュすることがあるのが普通だからです。アプリケーションがクラッシュした場合（0以外のコードで終了）、「ウォッチ」がそれを知らせます。コードを修正するとすぐに、ソースで変更が検出され、アプリケーションが再起動します。
