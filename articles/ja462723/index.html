TLA +で猫を捕まえる

形式的な方法は、ソフトウェアの信頼性を確保するための効果的ではあるが不合理に複雑な方法と見なされています。このために使用されるツールは、プログラマーが使い慣れているものとは大きく異なります。この記事は、このツールキットに参加するためのしきい値を下げるために書かれています。モデルチェックシステムは、ソフトウェア仕様の難解に定式化されたタスクを解くのではなく、小学生でも理解できるパズルを解くために使用します。

片側に7つの部屋がある直接の廊下にいます。そのうちの1匹は猫です。1つのステップで、部屋の1つを調べることができます。猫がいる場合は、捕まえられます。ドアが閉まるとすぐに、猫は隣の部屋の1つに移動します。仕事は猫を捕まえることです。

TLA +と時相論理


このような問題を解決できるツールはたくさんあります。多くの場合、このような場合、SMTソルバーが使用されます。原則として、このようなシステムは通常の述語ロジックを使用し、アクションのシーケンスの表現は非常に複雑です（配列のようなものを使用する必要がありますが、操作するにはあまり便利ではありません）。TLA +、時相論理は、現在では、次のステップの両方で1つのステートメントでシステムの状態を使用可能にする、使用されています。

CatDistr' = CatDistr \ {n}

この条件は、部屋nをチェックした後に猫が配置できる部屋のセットが、nが取得された部屋のセットと一致することを示しています（猫がそこにいた場合、猫は捕らえられ、それ以上行うことはありません）。

命令型プログラミング言語では、これは古い変数から計算された新しい値を変数に割り当てることとほぼ同じです。

セットについて少し


ご存じのように、猫の位置は、シミュレーションシステムの場合のように、特定の部屋ではなく、可能なすべての部屋のセットを含む変数によってモデル化されます。特定の値の代わりに可能な値のセットを使用することは、正式な方法で頻繁に使用される手法であり、初心者プログラマーにとって困難になります。そのため、セットの使用が適切なタスクを選択しました。

TLA +のプログラム構造


最も単純なケースでは、プログラムは、初期状態、現在の状態と次の状態の関係、および使用可能なすべての状態で実行する必要がある不変条件を記述する宣言とステートメント（述語）で構成されます。補助述語も存在する場合があります。述語はパラメータ化できます。

---- MODULE cat -------------------------------------------------------------

EXTENDS Integers

CONSTANTS Doors
VARIABLES CatDistr, LastDoor

Init ==
        /\ CatDistr = 0..Doors
        /\ LastDoor = -1

OpenDoor(n) == 
        /\ CatDistr' =
              0..Doors \intersect UNION { {x-1, x+1} : x \in CatDistr \ {n} }
        /\ LastDoor' = n

Next == \E door \in 0..Doors : OpenDoor(door)

CatWalk == CatDistr /= {}

=============================================================================

最初と最後の行は、モジュール宣言の構文機能です。

==は「定義上等しい」を意味しますが、単一の=は式の計算値の正確な等価です。

Doorsはモデル定数であり、構成ファイルで設定する必要があります。
CatDistr-部屋での猫の分布。

プログラムの出力を読みやすくするために、最後にチェックされた部屋であるLastDoor変数が入力されています。大規模なモデルの場合、このような変数は、分析された状態の数を増やすことによってプログラムを大幅に遅くする可能性があります（状態には発生の履歴が含まれ、異なる方法で作成された同じ状態は異なるため）。

述語Initは初期状態を説明します（0..Doorsはすべての部屋のセットです）。 OpenDoor（n）は、部屋nのドアを開いたときに何が起こるかを示します（最悪の場合、猫はそこにいません。

次の述語は少し奇妙に見えます-調べる余地があります。事実は、何かが存在するということは、TLA +がどの部屋を調べるかがわからないことを意味するため、すべての場合において不変条件の正確性がチェックされます。

書くほうがより明確になります

Next == OpenDoor(0) \/ OpenDoor(1) \/ ... \/ OpenDoor(Doors)

ただし、コードは一定数の部屋でのみ機能するため、パラメータ化することをお勧めします。

そして最後に、CatWalkの不変条件-猫が空でないかもしれない多くの部屋。不変条件の違反とは、猫がもともとどこにいても捕まったことを意味します。仕様を検証する場合、不変条件の違反はエラーを意味しますが、意図された目的のためにツールを使用しておらず、見つかった「エラー」は問題を解決することを意味します。

モデル構成


CONSTANTS
  Doors = 6

INIT Init
NEXT Next

INVARIANT CatWalk

ここではすべてが単純です-部屋の数、初期条件、状態変化の条件、およびテスト中の不変条件を示しました。

モデルはコマンドラインtlc2 -config cat.cfg cat.tlaから起動されます。

TLA +には、tla-toolboxチームによって起動された高度なGUIがあります。残念ながら、彼は.cfgファイルを理解しておらず、モデルのパラメーターはメニューから設定する必要があります。

結論


このタスクでは、すべてがかなり単純であることがわかりました。もちろん、正式な方法を適用する実際に重要なケースでは、はるかに大量のモデルとさまざまな言語構造の使用が必要になります。しかし、私はそのようなパズルを解くことにより、作業プロジェクトにフォーマルな方法を実装するためのシンプルで退屈ではない方法が提供されることを願っています。
