Pythonエントリポイント

多くの人は、エントリポイントは、パッケージをコマンドラインから実行できるようにするsetup.pyの指示であると考えています。これは一般的に当てはまりますが、エントリポイントの機能はこれに限定されません。

以下では、パッケージのプラグインシステムを実装して、他の人がそれと対話したり、たとえば、機能を動的に拡張したりできるようにする方法を示します。

サービスとしてのヘビ


残念ながら、平均的な消費者はまだPythonを習得しておらず、インタープリターやsnek.pyの場所を考慮せずに、コンソールからプログラムを実行したいと考えています。まあ、私たちの最高のスペシャリストは、インストール中にコンソールコマンドを自動的に作成するようにスクリプトをパックできた最高のスペシャリストです。

再配布可能なパッケージを作成するには、依存関係、ライセンスなどに関する情報を含むsetup.pyファイルが必要です。さらに、そこにエントリポイントを指定できます。

from setuptools import setup

setup(
    name='snek',
    entry_points={
        'console_scripts': [
            'snek = snek:main',
        ],
    }
)

専門家が説明したように、console_scriptsは特別なエントリポイントです。setuptoolsはその要素を"<console script> = <Pythonオブジェクトへのパス>"として読み取り、パッケージをインストールするときに各要素のコンソールユーティリティを作成します。

国境を越えた蛇


何百万人もの人々がヘビなしでは自分の一日を想像することはできません。 Googleの買収後も、Zmeikaのリソースは世界中のユーザーのニーズを満たすには不十分です。私たちのインフラストラクチャに基づいて自分のヘビを作成する機会を人々に与える時が来たようです。

""" ASCII .

:
    snek [--type=TYPE]
    
"""
import docopt
import pkg_resources

normal_snek = """\
    --..,_                     _,.--.
       `'.'.                .'`__ o  `;__.
          '.'.            .'.'`  '---'`  `
            '.`'--....--'`.'
              `'--....--'`
"""

fancy_snek = """\
                          _,..,,,_
                     '``````^~"-,_`"-,_
       .-~c~-.                    `~:. ^-.
   `~~~-.c    ;                      `:.  `-,     _.-~~^^~:.
         `.   ;      _,--~~~~-._       `:.   ~. .~          `.
          .` ;'   .:`           `:       `:.   `    _.:-,.    `.
        .' .:   :'    _.-~^~-.    `.       `..'   .:      `.    '
       :  .' _:'   .-'        `.    :.     .:   .'`.        :    ;
       :  `-'   .:'             `.    `^~~^`   .:.  `.      ;    ;
        `-.__,-~                  ~-.        ,' ':    '.__.`    :'
                                     ~--..--'     ':.         .:'
                                                     ':..___.:'
"""

def get_sneks():
    sneks = {
        'normal': normal_snek,
        'fancy': fancy_snek,
    }
    for entry_point in pkg_resources.iter_entry_points('snek_types'):
        sneks[entry_point.name] = entry_point.load()
    return sneks


def main():
    args = docopt.docopt(__doc__)
    snek_type = args['--type'] or 'normal'
    print(get_sneks()[snek_type])
    
if __name__ == '__main__':
    main()

これで、snekが起動するたびに、snek_typesエントリポイントを使用して、システムに登録されている他のヘビを検索します。そのような各ヘビは、そのタイプの名前で登録されます。これにより、コンソールのパラメーターに応じて目的のヘビを選択できます。

最も重要なことはすべてget_sneks内で発生します。pkg_resources.iter_entry_points（ 'snek_types'）を呼び出すと、「snek_types」という名前で登録されたすべてのエントリポイントをどこでも通過できます。したがって、サードパーティのパッケージは、setup.pyにエントリポイント「snek_types」を作成して、スクリプトで読み込むことができます。「snek_types」
