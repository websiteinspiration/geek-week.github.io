YandexがTestsuiteをオープン



本日、Yandex.Taxiによって開発および使用されたHTTPサービスをテストするためのフレームワークであるtestsuiteのソースコードを開きます。ソースはMITライセンスの下でGitHubで公開されています。

testsuiteを使用すると、HTTPサービスをテストするのに便利です。次のことを行うための既製のメカニズムを提供します。

Yandex.Taxiバックエンドは数百のマイクロサービスで構成され、新しいサービスが常に表示されています。私たちは、独自のuserverフレームワークを使用してすべての高負荷サービスをC ++で開発し、Habréですでにそれについて話しました。Pythonで要求の少ないサービスとプロトタイプを作成します。

サービスが問題を適切に解決し、APIを他のサービスと最終的なアプリケーションに提供することを確認するために、主にブラックボックスの原則に基づいて、サービス全体をテストします。

このための既製のツールはありません。次のようなテスト環境をセットアップするには、コードを記述する必要があります。

  • データベースを上げて埋める;
  • HTTPリクエストの傍受とスプーフィング。
  • この環境でテストサービスを実行します。

単体テストのフレームワークを使用してこの問題を解決するのは、タスクが異なるため、非常に難しく、間違っています。それは、コンポーネント、クラス、関数などの小さな構造ユニットの単体テストです。

Testsuiteは、標準のPythonテストフレームワークであるpytestに基づいています。テストするマイクロサービスがどの言語で書かれているかは問題ではありません。現在、testsuiteはGNU / Linux、macOSオペレーティングシステムで実行されます。

testsuiteは統合シナリオ、つまり複数のサービスの相互作用（およびサービスがPythonで記述されている場合は低レベルのサービス）には便利ですが、これらのケースは考慮しません。さらに、単一のサービスのテストのみに焦点を当てます。

動作原理


最終的な目標は、サービスがHTTP呼び出しに正しく応答することを確認することです。そのため、HTTP呼び出しでテストします。

サービスの開始/停止は日常的な操作です。したがって、以下をチェックします。

  • サービスの開始後、HTTP経由で応答します。
  • 外部サービスが一時的に利用できない場合のサービスの動作。




テストスイート：

  • データベースを起動します（PostgreSQL、MongoDB ...）。
  • 各テストの前に、データベースにテストデータを入力します。
  • テスト済みのマイクロサービスを別のプロセスで開始します。
  • 独自のWebサーバー（mockserver）を起動し、サービスの外部環境を模倣（乾燥）します。
  • テストを実行します。

テストで確認できるもの：

  • サービスがHTTPリクエストを正しく処理するかどうか。
  • サービスがデータベースで直接機能する方法。
  • 外部サービスへの呼び出しの有無/シーケンス。
  • Testpointに渡す情報を使用したサービスの内部状態。

モックサーバー


単一のマイクロサービスの動作をテストします。外部サービスのHTTP APIへの呼び出しはブリッジする必要があります。テストスイートでの作業のこの部分では、独自のプラグインmockserverとを満たしますmockserver_https。Mockserverは、リクエストごとにカスタマイズ可能なリクエストプロセッサと、どのリクエストが処理され、どのデータが転送されるかについてメモリを備えたHTTPサーバーです。

データベース


Testsuiteを使用すると、テストでデータベースに直接アクセスして読み書きを行うことができます。データを使用して、テストの前提条件を作成し、結果を確認できます。箱から出してすぐにサポートされているPostgreSQL、MongoDB、Redis。

ご利用方法


テストスイートテストを作成するには、開発者はPythonと標準のpytestフレームワークを知っている必要があります。

簡単なチャットの例を使用して、ステップバイステップでテストスイートを使用してデモンストレーションしましょう。アプリケーションとテストのソースコードは次のとおりです。



フロントエンドのchat.htmlは、チャットバックエンドサービスとやり取りします。

サービスの相互作用を示すために、チャットバックエンドはメッセージストレージをリポジトリサービスに委任します。ストレージは、chat-storage-mongoとchat-storage-postgresの 2つの方法で実装されます。

これで、インフラストラクチャは開始chat-backend方法と要求の送信方法を認識しました。これでテストの作成を開始できます。

テストであることに注意してくださいchat-backend、我々はストレージサービスを使用していない、どちらもchat-storage-mongo、またchat-storage-postgres。chat-backend通常の呼び出しを処理するために、ストレージAPIをで濡らしますmockserver。

メソッドのテストを書いてみましょうPOST messages/send。以下を確認します。

  • リクエストは正常に処理されます。
  • リクエストを処理するときにchat-backend、ストレージメソッドを呼び出しますPOST messages/send。

ソース

async def test_messages_send(server_client, mockserver):
    #    mockserver   POST messages/send
    @mockserver.handler('/storage/messages/send')    
    async def handle_send(request):
        # ,       ,
        #     chat-backend
        assert request.json == {
            'username': 'Bob',
            'text': 'Hello, my name is Bob!',
        }
        return mockserver.make_response(status=204)

    #    chat-backend
    response = await server_client.post(
        'messages/send',
        json={'username': 'Bob', 'text': 'Hello, my name is Bob!'},
    )
    
    # ,        HTTP-
    assert response.status == 204

    # ,  chat-backend       POST messages/send
    assert handle_send.times_called == 1

メソッドのテストを書いてみましょうPOST messages/retrieve。以下を確認します。

  • リクエストは正常に処理されます。
  • リクエストを処理するときにchat-backend、ストレージメソッドを呼び出しPOST /messages/retrieveます。
  • chat-backend 最新のメッセージがリストの最後になるように、リポジトリから受信したメッセージのリストを「反転」します。

ソース

async def test_messages_retrieve(server_client, mockserver):
    messages = [
        {
            'username': 'Bob',
            'created': '2020-01-01T12:01:00.000',
            'text': 'Hi, my name is Bob!',
        },
        {
            'username': 'Alice',
            'created': {'$date': '2020-01-01T12:02:00.000'},
            'text': 'Hi Bob!',
        },
    ]

    #    mockserver   POST messages/retrieve
    @mockserver.json_handler('/storage/messages/retrieve')
    async def handle_retrieve(request):
        return {'messages': messages}

    #    chat-backend
    response = await server_client.post('messages/retrieve')

    # ,        HTTP-
    assert response.status == 200

    body = response.json()
    
    # ,    chat-backend    ,
    #   ,       
    assert body == {'messages': list(reversed(messages))}

    # ,  chat-backend       POST messages/retrieve
    assert handle_retrieve.times_called == 1


チャットストレージpostgres



このサービスchat-storage-postgresは、PostgreSQLデータベースへのチャットメッセージの読み取りと書き込みを担当します。

サービス


これは、PostgreSQLからのメッセージのリストをメソッドで読み取る方法POST /messages/retrieveです。

ソースコード

@routes.post('/messages/retrieve')
    async def get(request):
        async with app['pool'].acquire() as connection:
            records = await connection.fetch(
                'SELECT created, username, "text" FROM messages '
                'ORDER BY created DESC LIMIT 20',
            )
        messages = [
            {
                'created': record[0].isoformat(),
                'username': record[1],
                'text': record[2],
            }
            for record in records
        ]
        return web.json_response({'messages': messages})

async def test_messages_send(server_client, pgsql):
