ユーザーの桁を囲む

私たちとの遠隔作業は、長い間続き、今日の大流行の境界を越えます。ガートナーが調査した317社の74％は、引き続き在宅勤務を利用しています。彼女の組織のITツールは、将来的に積極的に求められます。デジタルワークスペースの作成に不可欠な要素であるCitrix Workspace Environment Managerの紹介。この記事では、製品のアーキテクチャと主な機能について検討します。



ソリューションアーキテクチャ


Citrix WEMには、クラシックなクライアント/サーバーアーキテクチャソリューションがあります。


WEMエージェント\ WEMエージェント -Citrix WEMソフトウェアのクライアント部分。ワークステーション（仮想または物理、シングルユーザー（VDI）またはマルチユーザー（ターミナルサーバー））にインストールされ、ユーザーの環境を制御します。

WEMインフラストラクチャサービス\インフラストラクチャサービス -WEMエージェントにサービスを提供するサーバー部分。

MS SQL Server -Citrix WEM構成情報が保存されているWEMデータベースのサービスに必要なDBMSサーバー。

WEM管理コンソール -WEM環境管理コンソール。

Citrix WebサイトのWEMインフラストラクチャサービスコンポーネントの説明を少し調整してみましょう（スクリーンショットを参照）。


このサイトは、WEMインフラストラクチャサービスがターミナルサーバーにインストールされていると誤って述べています。本当じゃない。ターミナルサーバーでは、ユーザー環境を管理するためにWEMエージェントがインストールされます。また、WEM agnetとWEMサーバーを同じサーバーにインストールすることはできません。WEMサーバーが機能するためには、ターミナルサービスの役割は必要ありません。このコンポーネントはインフラストラクチャであり、他のサービスと同様に、別の専用サーバーに配置することが望ましいです。4つのvCPU、8 GBのRAMを備えた1台のWEMサーバーは、最大3000人のユーザーに対応できます。フォールトトレランスを確保するには、環境に少なくとも2つのWEMサーバーをインストールします。

主な特徴


IT管​​理者のタスクの1つは、ユーザーワークスペースの編成です。従業員が使用する作業ツールは、手元にあり、適切にセットアップされている必要があります。管理者は、アプリケーションへのアクセスの提供（デスクトップと[スタート]メニューにショートカットを配置する、ファイルの関連付けを構成する）、情報リソース（ネットワークドライブの接続）へのアクセスの提供、ネットワークプリンターの接続、ユーザードキュメントを一元的に保存する機能、ユーザーに構成機能を提供する必要があります。環境と最も重要なこと-快適なユーザーエクスペリエンスを保証するため。一方、管理者は、ユーザーが作業する特定の条件とソフトウェアライセンスポリシーを遵守するための条件に応じて、データのセキュリティを担当します。Citrix WEMは、これらの問題を解決するように設計されています。

したがって、Citrix WEMの主な機能は次のとおりです。

  • ユーザー環境管理
  • コンピューティングリソース管理
  • アプリケーションへのアクセスを制限する
  • 物理ワークステーション管理

ユーザーワークスペース管理


Citrix WEMはユーザーのデスクトップ作成オプションを管理するためにどのような機能を提供しますか？次の図は、Citrix Workspace Environment Manager管理コンソールを示しています。 「アクション」セクションには、管理者が作業環境をセットアップするために使用できるアクションがリストされています。つまり、デスクトップとスタートメニューにアプリケーションのショートカットを作成します（Citrix Storefrontとの統合による公開アプリケーションのほか、アプリケーションをすばやく起動するためのホットキーを割り当てたり、画面上の特定の場所にあるショートカットの場所の座標を調整したりできます）。接続します。ネットワークプリンターとネットワークドライブ、仮想ディスクの作成、レジストリキーの管理、環境変数の作成、セッションでのCOMポートとLPTポートのマッピングの構成、INIファイルの変更、プログラム\スクリプトの実行（LogOn、LogOff、再接続操作中）、ファイルとフォルダーの管理（作成\コピー\ファイルとフォルダーの削除）、SQLサーバーでのデータベース接続を構成するユーザーDSNの作成、ファイルの関連付けの構成。


管理を容易にするために、作成された「アクション」をアクショングループに組み合わせることができます。

作成されたアクションを適用するには、[割り当て]タブでドメインユーザーのセキュリティグループまたはKMにそれらを割り当てる必要があります。次の図は、「評価」セクションと、作成された「アクション」を割り当てるプロセスを示しています。すべての「アクション」を含むアクショングループを割り当てるか、左側の列「利用可能」から右側の「割り当て済み」にドラッグして、必要な「アクション」のセットを個別に追加できます。


「アクション」を割り当てる場合、システムが特定の「アクション」の使用の必要性を決定する分析に基づいて、フィルターを選択する必要があります。デフォルトでは、システムは1つのフィルターAlways Trueを作成します。これを使用すると、割り当てられたすべての「アクション」が常に適用されます。より柔軟な管理のために、管理者は[フィルター]セクションでフィルターを作成します。フィルターは、「条件」と「ルール」の2つの部分で構成されます。この図は、左側に条件を作成するウィンドウ、右側に目的の「アクション」を適用するために選択された条件を含むルールの2つのセクションを示しています。


かなり多くの「条件」がコンソールで利用可能です—図にはそれらの一部のみが表示されています。サイトまたはActive Directoryグループのメンバーシップの確認に加えて、個々のAD属性、フィルターを使用して、PCまたはIPアドレスの名前を確認し、OSバージョンを照合し、日付と時刻、公開されたリソースのタイプなどを確認できます。

アクションを使用してユーザー環境設定を管理することに加えて、Citrix WEMコンソールには別の大きなセクションがあります。このセクションはポリシーとプロファイルと呼ばれます。追加設定の可能性を提供します。このセクションは、環境設定、Microsoft USV設定、Citrixプロファイル管理設定の3つのセクションで構成されています。

最後のサブセクションは、Citrixプロファイル管理設定です。ユーザープロファイルを管理するためのCitrix UPMのセットアップを担当しています。このセクションには、前の2つの組み合わせよりも多くの設定があります。設定はセクションにグループ化され、タブとして編成され、Citrix StudioコンソールのCitrix UPM設定に対応しています。以下は、メインの[Citrixプロファイル管理設定]タブと、一般的なプレゼンテーション用に追加された使用可能なタブのリストを含む図です。


ユーザーの作業環境設定の集中管理は、WEMが提供する主なものではありません。上記の機能の多くは、標準のグループポリシーを使用して実現できます。 WEMの利点は、これらの設定がどのように適用されるかです。標準ポリシーは、ユーザーを順番に接続するときに使用されます。そして、すべてのポリシーを適用した後にのみ、ログオンプロセスが終了し、ユーザーはデスクトップを使用できるようになります。グループポリシーで有効にする設定が多いほど、適用に時間がかかります。これにより、ログイン時間が大幅に長くなります。グループポリシーとは異なり、WEMエージェントは処理順序を変更し、設定を複数のスレッドに並列かつ非同期に適用します。ユーザーのログイン時間が大幅に短縮されます。

コンピューティング管理


Citrix WEMを使用する別の側面、つまり、リソース管理の観点からシステムを最適化する可能性を検討してください。設定はシステム最適化セクションにあり、いくつかのブロックに分かれています。

  • CPU管理
  • メモリ管理
  • I \ O管理
  • 高速ログオフ
  • Citrixオプティマイザー

CPU管理には、プロセッサリソースを管理するためのパラメータが含まれています。一般的にリソースの消費を制限し、プロセッサリソースの消費の急増/急増を処理し、アプリケーションレベルでリソースに優先順位を付けます。主な設定は[CPU Manager Settings]タブにあり、次の図に示されています。


一般に、パラメーターの目的はその名前から明らかです。興味深い機能は、Citrixがスマート最適化-CPU \インテリジェントCPU最適化と呼ぶ、プロセッサリソースを管理する機能です。大きな名前は、シンプルですが非常に効果的な機能を隠します。アプリケーションが起動すると、プロセスには最大のCPU使用率の優先順位が割り当てられます。これにより、アプリケーションがすばやく起動し、システムを操作するときの快適さが全体的に向上します。ビデオのすべての「魔法」。


「メモリ管理」と「I \ O管理」セクションにはいくつかの設定がありますが、それらの本質は非常に単純です。ディスクを操作するときのメモリと入出力プロセスの管理です。メモリ管理はデフォルトで有効になっており、すべてのプロセスに適用されます。アプリケーションが起動すると、そのプロセスは作業用にRAMの一部を予約します。原則として、この予備は現時点では必要以上のものです。予備は、アプリケーションの高速動作を保証するために「拡張用」に作成されます。メモリの最適化は、設定された時間、非アクティブな状態（アイドル状態）であったプロセスのメモリを解放することから成ります。これは、未使用のメモリページをページファイルに転送することで実現されます。ディスクの最適化は、アプリケーションに優先順位を付けることで実現されます。下の図は、使用可能なオプションを示しています。


高速ログオフセクションを検討してください。作業セッションの通常の終了時に、ユーザーはアプリケーションがどのように閉じられるか、プロファイルがコピーされるかなどを確認します。高速ログオフオプションを使用する場合、WEMエージェントは通話を監視して作業セッションを終了し（ログオフ）、ユーザーセッションを切断します-それを切断状態にします。ユーザーの場合、セッションは即座に終了します。そして、システムは定期的にすべての作業プロセスをバックグラウンドで完了します。高速ログオフオプションは1つの「daw」によって有効になりますが、例外を割り当てることができます。


そして最後に、Citrix Optimizerのセクションです。Citrix管理者は、ゴールデンイメージを最適化するツールであるCitrix Optimizerをよく知っています。このツールは、Citrix WEM 2003に統合されています。次の図は、使用可能なテンプレートのリストを示しています。


管理者は、現在のテンプレートを編集したり、新しいテンプレートを作成したり、テンプレートに設定されているパラメーターを表示したりできます。次の図に設定ウィンドウを示します。


アプリケーションアクセス制限


Citrix WEMは、アプリケーション、スクリプト、DLLのロード-ライブラリの起動/インストールを制限するために使用できます。これらの設定は[セキュリティ]セクションで収集されます。以下の図は、システムが各サブセクションに対してデフォルトで作成することを提案しているルールをリストしています。デフォルトでは、すべてが許可されています。管理者はこれらの設定を上書きしたり、新しい設定を作成したりできます。ルールごとに、2つのアクション（許可\拒否）のいずれかを使用できます。サブセクションの名前が付いた括弧内に、その中に作成されたルールの数が示されています。 [アプリケーションセキュリティ]セクションには独自の設定はありません。サブセクションのすべてのルールが表示されます。管理者は、ルールの作成に加えて、組織で使用されている場合は既存のAppLockerルールをインポートし、単一のコンソールから環境設定を集中管理できます。


[プロセス管理]セクションでは、ブラックリストとホワイトリストを作成して、実行ファイル名でアプリケーションの起動を制限できます。


物理ワークステーション管理


VDIとターミナルサーバーの操作に関して、ユーザーの作業環境を作成するためのリソースとパラメーターを管理するための以前の設定に興味がありました。接続先の物理ワークステーションを管理するために、Citrixは何を提供していますか？上記で強調されたWEM機能は、物理ワークステーションに適用できます。さらに、このツールを使用すると、PCを「シンクライアント」に「変える」ことができます。この変換は、ユーザーがデスクトップへのアクセスをブロックし、一般にWindowsの組み込み機能を使用するときに発生します。デスクトップの代わりに、WEMエージェントのグラフィカルシェルが起動され（VDI \ RDSHと同じWEMエージェントが使用されます）、公開されているCitrixリソースが表示されるインターフェイスに表示されます。 CitrixにはCitrix DesktopLockソフトウェアがあり、PCを「TC」に変換することもできます。ただし、Citrix WEMの機能はより広いです。以下は、物理コンピューターの制御に使用できる主なパラメーターの画像です。




以下は、職場を「シンクライアント」に変換した後の職場のスクリーンショットです。[オプション]ドロップダウンメニューには、ユーザーが必要に応じて環境をカスタマイズできる要素が一覧表示されます。それらの一部またはすべての要素をインターフェイスから削除できます。


管理者は、会社のWebリソースへのリンクを「サイト」セクションに集中的に追加でき、ユーザーが「ツール」セクションに入るのに必要な物理PCにインストールされているアプリケーションを追加できます。たとえば、「サイト」でユーザーサポートポータルへのリンクを追加すると、VDIへの接続に問題がある場合に従業員がリクエストを作成できるため便利です。


このようなソリューションは、本格的な「シンクライアント」とは言えません。その機能は、類似のソリューションの商用バージョンと比較して制限されています。しかし、システムでの作業のインターフェースを簡素化して統一し、PCのシステム設定へのユーザーアクセスを制限し、特殊なソリューションの一時的な代替として古いPCフリートを使用するだけで十分です。

  • リソースの管理：プロセッサ、メモリ、ディスク

もちろん、WEMを使用してデモを懐疑的に評価することもできます。私たちの経験が示すように、WEMを使用しないほとんどの企業の場合、平均ログイン時間は50〜60秒で、ビデオの時間と大差ありません。WEMを使用すると、ログオン時間を大幅に短縮できます。また、会社のリソースを管理するための単純なルールを使用して、サーバー上のユーザーの密度を高めたり、現在のユーザーにシステムで最高品質の作業を提供したりできます。
