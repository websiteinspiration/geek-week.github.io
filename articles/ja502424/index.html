完全にインポートされ、その機能の一部のみを必要とするパッケージの鮮やかな例は、Lodashライブラリーです。このライブラリについて聞いたことがない場合は、そのドキュメントをご覧ください。特に、プロジェクトのWebサイトでは、Lodashが配列、数値、オブジェクトなどを操作するタスクを実行することでJavaScript開発を簡略化していることがわかります。



Lodashライブラリには200を超える関数が含まれています。これは、彼女が実際にプログラマーが多くの問題を解決するのを助けることができることを示唆しています。ただし、ライブラリ全体をインポートすると、4〜5個の関数のみが呼び出される場合があります。これにより、その機能の2〜3％しか使用されない状況でパッケージ全体をインポートすることの妥当性の問題につながります。これをどう扱うか考えてみましょう。

Lodash get関数


Lodashライブラリの私のお気に入りの機能の1つは機能ですget。ネストされたオブジェクトへの安全なアクセスを整理し、デフォルト値の使用をサポートします。

この関数の使用例を次に示します。

const _ = require('lodash');

let employee1 = {
  name : "Jon",
  address : {
    street : "North Avenue",
    area : "DAC",
    zip : "87344",
    contact : [ 12444554, 9384847 ]
  },
  designation : "Manager"
};

let employee2 = {
  name : "Jake",
  designation : "Senior Manager"
};

function getHomeContact(employee) {
  return employee.address.contact;
}

getHomeContact(employee1); // [12444554, 9384847]
getHomeContact(employee2); // Uncaught TypeError: Cannot read property 'contact' of undefined

function getHomeContactWithLodash(employee) {
  return  _.get(employee, "address.contact", []);
}

getHomeContactWithLodash(employee1); // [12444554, 9384847]
getHomeContactWithLodash(employee2); // []

この関数を使用すると、コードを以前よりもずっときれいにすることができます。これは、空の配列が予期されている場合、関数がを返さないために発生するエラーを回避するのに役立ちますnull。この関数のおかげmapで、空の配列のメソッドを誤って呼び出すことはできません。他のトラブルから守ります。

関数のみを使用する予定の場合、プロジェクトにLodashライブラリを含めるとバンドルのサイズにどのように影響するかを見てみましょうget。実験は、Reactプロジェクトを使用して行われます。バンドルのサイズは、ライブラリをインポートする前、およびライブラリをプロジェクトに接続するためのさまざまなオプションの後に分析されます。

破壊的な割り当てを使用してget関数をインポートするときのファイルサイズ

ご覧のとおり、上記の両方のアプローチを使用すると、ファイルサイズが約23 KB増加しました。そして、これは非常に重要な増加です。特に、200を超える関数を含むライブラリから単一の関数を使用することについて話していることを考えると、その結果、バンドルのサイズは、これらの200個の関数すべてがプロジェクトで使用された場合に予想されるほど大きくなりました。

たぶん、必要な機能だけを使用するのに23 Kbはそれほど大きな価格ではありませんか？いいえ、多すぎます。 

必要なものだけをプロジェクトにインポートできる方法はありますか？はい、そのような方法があります。

Lodashマテリアルが保存されているフォルダーを分析してみましょう。

これを行うには、パスに沿って進みますnode_modules/lodash。このフォルダには、個々の関数のコードが格納されている多くのファイルがあります。その中から、get.js目的の関数のコードが配置されているファイルを簡単に見つけることができますget。つまり、関数のみが必要な場合は、getこのファイルのみをプロジェクトにインポートするだけで十分です。これは、インポートの3番目の方法につながります。

▍3。Lodashからget.jsファイルをインポートする


ここでも、2つの方法を使用できます。

最初：

import get from 'lodash/get';

第二：

const get = require('lodash/get');

すべてのパッケージが選択的インポートアプローチをサポートしていますか？


いいえ、すべてではありません。パッケージファイルの編成に完全に依存します。しかし、幸いなことに、かなり大きな人気のあるパッケージのほとんどは構造化されているため、それらを使用すると、個々の機能の選択的なインポートを簡単に整理できます。
