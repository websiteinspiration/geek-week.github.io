リファレンスアーキテクチャ：ソフトウェアデファインドデータセンターの構築方法

ソフトウェアデファインドデータセンターを作成するには、ベンダーが提案したリファレンスアーキテクチャを使用できます。以下では、Lenovo ThinkAgile VXノードを使用してVMware Software Defined Data Centerを構築する方法について説明します。

SDDCとは


それでは、ソフトウェア定義（またはソフトウェア構成可能）データセンター（ソフトウェア定義データセンター、SDDC）とは何ですか？まず、それは高度な自動化を備えたデータセンターであり、実際には物理的な介入を必要としません。このようなデータセンターのすべてのインフラストラクチャコンポーネントは仮想化されており、その管理はソフトウェアを使用して完全に自動化されています。 VMware SDDCの場合、これはvSphereソフトウェアです。 VMware vSphereは、コンピューティング、ネットワーク、ストレージリソースのレベルを含むすべてのレベルで仮想化を提供します。


SDDCを使用すると、ユーザーにコンピューティング、ネットワーク、ストレージサービスを柔軟に提供できます。

ソフトウェア定義のデータセンターでは、ポリシーベースの自動化により、リソースの割り当てと管理が提供されます。その結果、ITの柔軟性と効率が実現し、あらゆる種類の機器とアプリケーションをサポートします。今日、SDDCは仮想化と自動化に基づく実証済みのアーキテクチャアプローチであり、既存のITインフラストラクチャをすぐに置き換えることなく段階的に実装できます。


ソフトウェア定義のデータセンターでは、インフラストラクチャ全体が仮想化され、ソフトウェアを使用して管理が自動化されます。VMwareは、2012年からSDDCコンセプトを推進しています。

それはなんのためですか？


ITの複雑さが増すにつれ、データセンターのITインフラストラクチャを管理し、変更の影響を予測することがより困難になっています。ビジネスの要件は、インフラストラクチャの問題に重なっています。コストの削減、安定性、パフォーマンス、可用性、更新の容易さなどのユーザー要件を満たすことができるアプリケーションのより柔軟な使用です。


VMware SDDCには、エンタープライズインフラストラクチャの構築に必要なすべてのソフトウェアが含まれています。

VMware SDDC製品には、VMware vSphereコンピューティング仮想化ソフトウェア、VMware vSphereおよびVMware vSANストレージ仮想化、VMware NSXネットワーク仮想化、およびVMware vRealizeクラウド管理プラットフォームと統合されたVMware EVOシステムが含まれます。

目標は、変化するニーズに適応できる、柔軟で管理しやすいITインフラストラクチャを提供することです。コンピューティング、ストレージ、SDDCネットワークの仮想化により、基盤となるハードウェアプラットフォームである物理ハードウェアへの依存度が低くなります。ポリシーベースの構成とオンデマンドでのITリソースのプロビジョニングにより、データセンターの管理が簡素化され、インフラストラクチャが拡張および更新されます。

SDDCを作成した結果、高効率と低コストが実現しました。仮想化されたITサービスと自動化された運用管理は、リソースの使用とスタッフの生産性の向上の新たな可能性を切り開きます。ポリシーベースの構成により、ワークロードをすばやく展開でき、変化するニーズに合わせてリソース使用率が自動的に構成されます。仮想化を念頭に置いたセキュリティ機能の自動化により、稼働時間が延長されます。

ベンダーは、ターンキーハイパーコンバージドソリューション-統合されたハードウェアおよびソフトウェアシステム、リファレンス、リファレンスアーキテクチャなど、SDDCを展開するためのいくつかのハードウェアおよびソフトウェア構成を提供しています。これらは、幅広いアプリケーションとワークロード向けに設計されています。

VMware vSAN


VMware vSANは、ESXiハイパーバイザーと統合されたソフトウェア定義ストレージ（SDS）ソリューションです。 VMware vSANは、3つ以上の10GbE接続サーバー上のフラッシュキャッシュを1つの共有データストアに結合します。ガートナーの調査によると、今後5年間で、グローバル市場におけるソフトウェアデファインドストレージのシェアは50％にまで増加すると予測されています。


VMware vSANは最大64台のサーバーを拡張でき、各サーバーは最大5つのドライブグループをサポートします。グループには、ソリッドステートドライブ（SSD）と最大7台のハードドライブ（HDD）が含まれます。ストレージのパフォーマンスと容量は、ディスク、フラッシュメモリ、またはサーバーを追加することで簡単に増やすことができます。キャッシュは、読み取りと書き込みを高速化するために使用されます。

vSANにかかる主な負荷。64％はビジネスクリティカルなアプリケーションです（VMwareの調査によると）。

VMware vSANは分散RAIDアーキテクチャを使用します。データはクラスター全体に分散されます。使用される方法は、RAID-0、RAID-1、またはRAID-5 / 6です。このソリューションは、ストレージポリシーベースの管理（SPBM）を適用します。ストレージリソースは、多くの従来のソリューションのように、事前ではなく、要求されたポリシーに基づいて動的に割り当てられます。

VMware NSXプラットフォームは、物理および仮想のデータセンターに既製のネットワーク環境を提供し、集中管理を提供し、IT運用を簡素化し、新しいアプリケーションの起動にかかる時間を短縮します。

オープンで柔軟性があり、シンプルで経済的なネットワークテクノロジーのために独自のソリューションを放棄し、ネットワークを高速化して、サービスの起動にかかる時間を短縮し、新しいアプリケーションのデプロイにかかる時間を短縮し、ネットワーク仮想化、分離、セグメンテーションテクノロジーを利用して、物理ワークロードを仮想ネットワークに接続できます。さらに、アプリケーションフローを自動的に検出して分離することで、サービスの生産性と品質を向上させることができます。セキュリティの目的で、データセンターはマイクロセグメンテーションを使用しています。

VMware NSXを使用すると、ネットワークリソースのプールから新しいアプリケーション用の安全な環境を迅速に展開できる仮想ネットワークを作成できます。仮想ネットワークは、物理インフラストラクチャによって課される制限を取り除くことができます。さらに、ネットワーク仮想化のおかげで、単一のアプリケーションの仮想ネットワークは、モビリティと特定の物理インフラストラクチャからの独立性を獲得します。

Yahooテスト（YCSB）：Lenovoノードの結果


このテストは、さまざまなクラスターコンピューティングの負荷に対する相対的なストレージパフォーマンスを示すために使用されます。 YCSBはNoSQLテストプラットフォームです。テストには、読み取り、挿入、更新、削除、およびスキャン操作を実行するためのさまざまな負荷が含まれています。これにより、NoSQLデータベースを操作するときの負荷と入出力システムのパフォーマンスを評価できます。さまざまな読み取り/書き込み比率でワークロードのカスタマイズをサポートします。

YCSBテストは、mongodb v3.6.3とWiredTigerストレージエンジンを搭載したUbuntu 16.04.4 LTS仮想マシンで実行されました。各仮想マシンには、2つの仮想CPU、4 GBのメモリ、および50 GBのメモリが搭載されていました。

テストは、48および96の仮想マシンで実行されました。さらに、YCSBクライアントを備えた他の4つのUbuntu 16.04.4 LTS仮想マシンを使用して、mongodb仮想マシンの負荷をシミュレートしました。I / Oシステムを評価するために、125の同時スレッドを使用して、1秒あたり5,000データベース操作という目標速度に到達しました。
3つの異なるワークロードがテストされ、それぞれ読み取り操作と書き込み操作の比率が異なります。次のワークロードが使用されます。

  • ワークロードA-ダウンロード（100％レコード）-1000万レコードを作成


100％レコードのワークロードのパフォーマンス比較。

読み取り操作が50％、書き込み操作が50％のワークロードのパフォーマンス比較。


読み取り操作の80％と書き込み操作の20％のワークロードのパフォーマンス比較。

YCSBテストは、4つのLenovo ThinkAgile 1U認定サイトで実行されました。 ESXi 6.7のテストに使用されたノードには2 x 6130プロセッサー（16コア、2.1 GHz）と384 GBのシステムメモリが搭載されていましたが、ESXi 6.7u1のノードには2つのIntel Xeon Scalable Gen2 2 x 6230プロセッサー（20コア、2 、10 GHz）。 384 GBのメモリ。仮想マシンは4台のマシンに均等に分散されました。 YCSBテストでは、48個のVMが9.8 TBを使用し、96個のVMが19.6 TBの容量を使用します。

vSANハイブリッドアレイの場合、4つのサーバーのそれぞれが2つのドライブグループを使用し、それぞれに800 GBのソリッドステートドライブと3つの1.2 TBハードドライブ、つまり合計8つのキャッシュドライブと24のハードドライブ、合計で27 TBのストレージ容量があります。 vSANフラッシュアレイの場合、4台のサーバーのそれぞれが2つのグループを使用し、それぞれに800 GBソリッドステートドライブと2つの3.84 TBソリッドステートドライブが含まれていました。つまり、合計8台のキャッシュSSDと16台のSSDで、合計容量は43 TBです。

48個の仮想マシンの結果は、フラッシュメモリを使用した場合のわずかな改善を示しています。96台の仮想マシンの結果は、ハイブリッドシステムと比較して、フラッシュメモリ（オールフラッシュ）を使用した場合の96台の仮想マシンのパフォーマンスが大幅に向上します。これは、予想通り、I / O負荷が高い場合にフラッシュがはるかに高速であることを証明しています。

VMware Cloud Foundation


VMware Cloud Foundation（VCF）は、VMware SDDCをプライベートクラウドとして展開し、パブリッククラウドと統合するための単一のハイブリッドプラットフォームです。このソリューションは、さまざまなワークロードのコンピューティング、ストレージ、ネットワーク、クラウドを管理するためのソフトウェア定義サービスを提供します。ライフサイクルを管理することにより、インストール、アップグレード、およびSDDCコンポーネントを簡素化します。

自動化機能は、新しいVMware SDDC Managerコンポーネントによって提供されます。 SDDC Managerソフトウェアは、vSphere、vSAN、NSXのインストールとライフサイクル管理を、起動と構成からアップグレードまで自動化し、vRealize Log Insight、vRealize Operations、vRealize Automationのインストールと構成も自動化します。

Cloud Foundationは、VMware vSphere（コンピューティングリソース）、vSAN（ストレージ）、およびNSX仮想化（ネットワーク）を1つの標準統合パッケージに統合します。 Cloud Foundationは、ローカルのプライベートクラウドにデプロイするか、パブリッククラウドサービスとして使用できます。
VCFは、統合ハードウェアスタックとして、または顧客の機器にインストールするためのソフトウェアパッケージとして提供されます。 VCFは、さまざまな物理サーバー（vSAN対応ノード）とネットワークスイッチ全体にわたるSDDCコンポーネントの導入をサポートし、さまざまなワークロードをサポートする柔軟な異種インフラストラクチャを作成します。特に、VCFはLenovo ThinkAgile VX認定サイトにインストールできます。

ソリューションはベンダーによってテスト、検証、サポートされています。顧客は必要なコンポーネントをすべて「1つのウィンドウから」受け取り、注文とサポートのプロセスを容易にします。既製のリファレンスアーキテクチャにより、このような複合システムの実装と操作が大幅に簡素化されます。
