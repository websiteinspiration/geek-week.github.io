子供の頃のような万華鏡


時々、鏡での反射はオブジェクト自体よりリアルです
...-ルイス・キャロル（鏡の中のアリス）

若い頃、私はおもしろいおもちゃ、万華鏡を持っていました。何時間も、壊れたガラスの複数の色の破片からなる正しいパターンを調べました。この瞑想的な瞑想には、妖艶な何かがありました。今、私は父親と​​して、カオスの正しい構造の美しさを子供たちに見せたかったのです。

子供たちは今や現代的で、普通のおもちゃには興味がなく、コンピューターやタブレットを与えています。したがって、私は万華鏡の変形のデジタルプロトタイプを再現し、同時に私のコンピューターグラフィックススキルを練習したいと思いました。

私と一緒に反射の世界に飛び込んでください。

当初のアイデアは、万華鏡の完全な物理モデルを構築することでした。このデバイスは、互いに角度を付けて配置されたいくつかの鏡で構成されています。子供の頃の万華鏡は3枚の鏡で構成されていたので、このデザインを再現することにしました。

私にとって最も明白な解決策は、レイトレーシングを使用することでした。 3つの鏡面が互いに120度の角度で作成されました。



ミラーの遠端を超えてオブジェクトを配置し、光線の複数の再反射（約20回の反射）を使用すると、非常に有効な万華鏡が得られます。



ルーティングを作成するには、計算シェーダーが使用されます。画像の出力はテクスチャーで行われ、後で画面に表示されます。球体は、より単純な形状の描画オブジェクトとして使用されます。リアルタイムレンダリングモードのビデオカードでは、20〜25 FPSをなんとか達成できました。これは、3つのオブジェクトと1つの光源でのみ発生します。これは悲しいことです。多くの異なる形状のカオス的な動きとリアルタイムの光源が欲しかったのですが、これはさらに大きな減速につながるでしょう。

最適化へのいくつかのアプローチの後、私はこのモデルを見込みがないと延期しました。

#version 430 core
layout( local_size_x = 32, local_size_y = 32 ) in;
layout(binding = 0, rgba8) uniform image2D IMG;
layout(binding = 1, std430) buffer InSphere {vec4 Shape_obj[];};
layout(binding = 2, std430) buffer InSphere_color {vec4 Sphere_color[];};

uniform vec2 u_InvScreenSize;
uniform float u_ScreenRatio;
uniform vec3 u_LightPosition;
uniform vec3 u_CameraPosition;

//
const vec3 ray00 = vec3(-1*u_ScreenRatio,-1, -1.2);
const vec3 ray01 = vec3(-1*u_ScreenRatio,+1, -1.2);
const vec3 ray10 = vec3(+1*u_ScreenRatio,-1, -1.2);
const vec3 ray11 = vec3(+1*u_ScreenRatio,+1, -1.2);
const ivec2 size = imageSize(IMG);

const mat3 mat_rotate = mat3(-0.5, -0.86602540378443864676372317075294, 0, 0.86602540378443864676372317075294, -0.5, 0, 0, 0, 1);
struct plane {
vec3 v_plane;
vec3 n_plane;
vec3 p_plane;
};

//
plane m[3];
int last_plane;

//----------------------------------------------------------
float ray_intersect_sphere(vec3 orig, vec3 dir, vec4 Shape_obj) {
vec3 l = Shape_obj.xyz - orig;
float tca = dot(l,dir);
float d2 = dot(l,l) - tca * tca;
if (d2 > Shape_obj.w * Shape_obj.w) {return 0;}
float thc = sqrt(Shape_obj.w * Shape_obj.w - d2);
float t0 = tca - thc;
float t1 = tca + thc;
if (t0 < 0) {t0 = t1;}
if (t0 < 0) {return 0;}
return t0;
}

'float ray_intersect_plane(in vec3 orig, in vec3 dir, inout plane p) {
vec3 tested_direction = p.v_plane - orig;
float k = dot(tested_direction, p.v_plane) / dot(dir, p.v_plane);
if (k>=0) {
vec3 p0 = orig + dir * k;
// z
if ((p0.z>-80)&&(p0.z<3)) {
p.p_plane = p0;
return length(p0-orig);
}
}
return 1000000;

//---------------------------------------------------------
bool all_obj(inout vec3 loc_eye, inout vec3 dir, inout vec3 c) {
float min_len = 1000000;
uint near_id = 0;
float len;
float min_len2 = 1000000;
int near_id2 = -1;
for (int i=0; i<3; i++) {
if (i!=last_plane) {
len = ray_intersect_plane(loc_eye, dir, m[i]);
if (len<min_len2) {
min_len2 = len;
near_id2 = i;
}
}
}

//
if (near_id2>=0) {
loc_eye = m[near_id2].p_plane;
dir = reflect(dir, m[near_id2].n_plane);
last_plane =near_id2;
return true;
}

for (uint i=0; i<Shape_obj.length(); i++) {
len = ray_intersect_sphere(loc_eye, dir, Shape_obj[i]);
if ((len>0)&&(len<min_len)) {
min_len = len;
near_id = i;
}
}
//
if (min_len>=1000000) {return false;}

vec3 hit = loc_eye + dir * min_len;
vec3 Normal = normalize(hit - Shape_obj[near_id].xyz);
vec3 to_light = u_LightPosition - hit;
float to_light_len = length(to_light);
vec3 light_dir = normalize(to_light);
float diffuse_light = max(dot(light_dir, Normal), 0.0);
c = min(c + Sphere_color[near_id].xyz * (diffuse_light*0.8+0.2),1);
return false;
}

void main(void) {
if (gl_GlobalInvocationID.x >= size.x || gl_GlobalInvocationID.y >= size.y) return;

vec3 dir = normalize(mix(mix(ray00, ray01, pos.y), mix(ray10, ray11, pos.y), pos.x));

m[1].n_plane = mat_rotate * m[0].n_plane;
m[2].v_plane = mat_rotate * m[1].v_plane;

別のアプローチでは、万華鏡パターンの周期特性を使用しました。各頂点は常に他の2つと接続されています。ここでは、3つの頂点が3つの色で示されています。
菱形を形成する正三角形の頂点の座標でバッファオブジェクトを塗りつぶします。



図では、色が数字に置き換えられています。注意：偶数行と奇数行は1シフトして繰り返されます。余分な要素を切り取り、必要な頂点インデックスのみを表示します。その結果、簡単にスケーリングできる六角形が得られます。



次に、色をミニテクスチャテンプレートのテクスチャ座標で置き換えます。



ランダムな色の長方形でテクスチャを塗りつぶす例。

表示を改善するには、六角形を画面のサイズまで増やし、軸回転も追加します。

一方向の回転から数分の熟考の後、それはかき混ぜ始めました。この不快な影響を排除するために、回転は各方向に順次実行されました。

当初、テクスチャはランダムな要素で埋められていましたが、その後、カラーイメージまたは写真を使用するというアイデアが思い付きました。表示要素は、スライドウィンドウの形でランダムな方向に画像を通過し、定期的に方向を変えます。したがって、パターンはより飽和していて興味深いものです。

結果はかなりいい画像です
