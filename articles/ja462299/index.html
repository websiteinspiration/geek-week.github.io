.NET全体への丸め

すべてのひげを生やした区、同志！

私たちは皆、丸めとは何かを知っています。誰かが忘れた場合、丸めは、有効桁数を減らして書かれた数値をその近似値で置き換えることです。6.5を整数に丸めるとどうなるかをその場で誰かに尋ねると、彼はためらうことなく「7」と答えます。私たちは学校から、数字は最も近い整数に丸められ、絶対値が大きいと教えられました。つまり、丸められた数値で小数部分が全体の放電の半分以上である場合、元の数値を最も近い大きい方に丸めます。

簡単に言えば： 
6,4 = 6 
6,5 = 7 
6,6 = 7
等

そのため、学校を出てプログラマになると、強力なプログラミング言語から同じ動作が期待されることがよくあります。私たちは学校で「数学的な丸め」を教えられたことを完全に忘れていますが、実際にははるかに多くの種類の丸めがあります。ウィキペディアだけでも、0.5から最も近い整数までの丸めオプションの数を掘ることができます。

  • 数学的な丸め
  • ランダム丸め
  • 交互丸め
  • 銀行丸め

最初のタイプである「数学の丸め」は、私たち全員が学校から学んだものです。 2番目と3番目のタイプについては自由に読むことができますが、今日の記事ではそれらは興味深いものではありません。

しかし、「銀行丸め」はすでに興味深いものです。 "なぜ？" - あなたが尋ねる。サブネットでは、あるデータ型を別のデータ型に変換するための多くのメソッドを提供するConvertクラスをよく使用します（キャストと混同しないように、以下で説明します）。そして今、Convert.ToInt32メソッドを介して浮動小数点数（double、float、decimal）を整数型intに変換すると、「バンキング」丸めが内部で機能することがわかります。ここではデフォルトで使用されます！

そして、このささいなことの無知はあなたの仕事に大きな影響を与えていないようですが、あらゆる種類の記録と数値の束に基づいて統計と計算指標を扱う必要があるとすぐに、これは横向きになります。 （無知から）計算におけるすべての変換/丸めは「数学的」丸めのルールに従って機能することを期待しているためです。そして、6.5を丸めた結果（6）の結果、新しいゲートのRAMのように見えます。

これを見たプログラマーの最初の考えは次のとおりです。「おそらく、丸めは反対方向に機能し、ルールに従って最小数に丸めますか？」、「学校の数学から何かを忘れたのでしょうか？」それから彼はグーグルに行き、彼らが何も忘れていないこと、そしてある種の暴徒が続いていることを理解します。この段階で、怠惰な開発者は、これがConvert.ToInt32メソッドの標準の動作であると判断し、最小の整数に丸め、さらに検索するためにスコアを付けます。そして、彼は、Convert.ToInt32（6,5） = 6であれば、アナロジーでConvert.ToInt32（7,5） = 7であると考えます。しかし、それはありませんでした。将来、そのような開発者は、QA部門からの多数のバグに頭を悩まされます。

実際のところ、「バンキング」丸めは少し注意が必要です。数値を最も近い偶数の整数に丸め、最も近い整数を法として丸めません。このタイプの丸めは、銀行業務での適用の場合にはおそらくより正直です-奇数の整数部分を持つ操作と同じ数の操作があると仮定すると、銀行はそれ自体またはクライアントを奪うことはありません。しかし、私に関しては-それはまだはっきりしていません:)それがConvert.ToInt32（6.5）が6の結果を与え、そしてConvert.ToInt32（7.5）の結果が7ではなく8になる理由です:)

何をすべきか誰もが馴染みのある「数学的な」四捨五入を得るために？クラスメソッドの変換追加の丸め設定はありません。このクラスは主に丸めではなく、型変換に使用されるため、事実です。Roundメソッドを備えたすばらしいMathクラスが役に立ちます。ただし、ここでも注意してください。デフォルトでは、このメソッドはConvert.ToInt32（）の丸めと同じように機能するため、「バンキング」ルールに従います。ただし、この動作は、Roundメソッドの一部である2番目の引数を使用して変更できます。だから、恐らくMath.round（someNumber、MidpointRounding.ToEvenは）私たちのデフォルトの丸め「銀行業」を提供します。そして、ここで恐らくMath.round（someNumber、MidpointRounding.AwayFromZeroは）「数学的」丸めの通常のルールに従って動作します。

ちなみに、Convert.ToInt32（）は内部ではSystem.Math.Round（）を使用しません。特にgithub でこのメソッドの実装を掘り下げます-残差に従って丸めが考慮されます：

public static int ToInt32(double value) {
	if (value >= 0) {
		if (value < 2147483647.5) {
			int result = (int)value;
			double dif = value - result;
			if (dif > 0.5 || dif == 0.5 && (result & 1) != 0) result++;
			return result;
		}
	}
	else {
		if (value >= -2147483648.5) {
			int result = (int)value;
			double dif = value - result;
			if (dif < -0.5 || dif == -0.5 && (result & 1) != 0) result--;
			return result;
		}
	}
	throw new OverflowException(Environment.GetResourceString("Overflow_Int32"));
}

そして最後に、型キャストについてのいくつかの言葉：

var number = 6.9;
var intNumber = (int)number;

この例では、浮動小数点型（この場合はdouble）を整数intにキャストします。そのため、整数型にキャストする場合、非整数部分全体が単に切り捨てられます。したがって、この例では、変数「intNumber」に数値6が含まれます。ここには丸め規則はありません。小数点以下のすべてを切り捨てるだけです。これを覚えて！
