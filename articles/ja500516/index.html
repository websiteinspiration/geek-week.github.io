インスタントメッセンジャーからの高速転送-QIWI Androidウォレット

こんにちは！

私の名前はアレックスです。私はQIWIの開発者です。

TL; DR

メッセンジャーから支払いフォームに直接転送する方法：

  1. マニフェストでは、空のアクティビティCを入れintent-filter タイプACTION_VIEWや ACTION_DIALスキーム「TEL」と。
  2. アクティビティでは、既存のディープリンクを介して支払いフォームに転送し、元のデータからのデータでそれを充実させます intent- “tel:XXXXX”



利益：メッセンジャーでハイライト表示された電話番号をクリックすると、転送先のフィールドに入力された転送フォームが表示されます。
おまけ：実行時にマニフェストのインテントフィルターリストを変更することなく、この機能を美しく有効にする方法を説明します。

何のために？


ある人が一括誕生日プレゼントにお金を転送するタスクを持っているとします。受け取り側は、便利な送金オプションを選択できます：カード、電子財布への送金、口座への直接送金など。翻訳サービスを提供する多くの企業は、電話番号を二次または一次ユーザー識別子として受け入れます。誤った転送を回避するために、受信者はテキストメッセージ（電子メール、インスタントメッセンジャー、SMS）で電話番号を伝えることができます。メッセージには、翻訳に必要なシステムも示されます。

一般的なユーザーパスは、メッセンジャーから電話番号をコピーして、翻訳のためにアプリケーションに貼り付けることです。この方法では、アプリケーション間の切り替え、場合によってはデスクトップへのアクセス、さらには翻訳サービスプロバイダーのインストール済みアプリケーション間の検索までもが行われます。ステップ数を減らすことでこのパスを短くすることは可能ですか？オプションがあります。

理論


スマートフォンの多くのインスタントメッセンジャーは電話番号を認識し、ハイパーリンクの形でそれらを強調表示します。デフォルトでは、GSMモジュールを備えたすべてのモバイルデバイスでこの種のリンクを処理するためのアプリケーションがあります。通常は「電話」と呼ばれます。 Androidでは、共有電話番号を処理する機能はAndroidManifestで宣言されています。アプリケーションの例：「電話」、Viber、Skype。これらのイベントを傍受した場合は、可能なアクションのリストと電話番号をすぐにユーザーに提供できます。私たちの場合、これはQIWI Walletを介した転送オファーになります。

私たちは種類のイベントに興味があるIntent.ACTION_VIEW、Intent.ACTION_DIAL, Intent.ACTION_CALLTEL方式に。イベントIntent.ACTION_CALLAndroidのセキュリティポリシーに制限があり、その初期化にはMarshmallowのバージョンを含むユーザーからの許可が必要です。より一般的な私たちの研究メッセンジャーによるとIntent.ACTION_VIEW、Intent.ACTION_DIAL。

特に緊急電話用に似たタイプがいくつかありますが、これは明らかに私たちのケースではありません。インテントボディの形式は「tel：12345678」です。android.content.UriMatcherホストがここにないため、この種のインテントは処理できません。この種のUriを処理する最も簡単な方法は、それを尋ねてschemeSpecificPart 、それが有効な電話番号であるかどうかを確認することです。

インテントフィルターのリストはAndroidManifestで指定する必要があります。アプリケーションの実行中にリストを変更することはできません。この機能をリモートで無効にする方法は？最初のオプションは、アクティビティが開かれたときにユーザーのリクエストを拒否することです。メッセージを表示するか、アプリケーションを閉じます。私たちの観点からは、ユーザーの流れを途中で中断するため、これは最適なUXではありません。機能が無効になっている場合は、ユーザーにアプリケーションをまったく選択させない方が良いでしょう。実行時にインテントフィルタリストを使用してコンポーネントを無効にする方が簡単です。これにより、システムインターフェースでユーザーに誤った約束が与えられることはありません。 PackageManagerを使用する場合、コンポーネントが含まれているかどうかのチェックはあまり信頼できないことに注意してください。enabled 値を使用して現在の状態に依存せずにコンポーネントのフラグを強制することが望ましいCOMPONENT_ENABLED_STATE_ENABLED とCOMPONENT_ENABLED_STATE_DISABLED。PackageManagerを介して行われた変更は、アプリケーションがデバイスからアンインストールされるまで保存されます。

練習


この機能を無効にするには、それを無効にするAndroidコンポーネントが必要です。
最も簡単な方法は、空のアクティビティを使用することです。同様のインテントフィルターのセットでサービスを使用する場合は、特に「サービスのインテントフィルターを宣言しないでください」というGoogleガイドを確認してください。デフォルトではコンポーネントは無効になっていることに注意してください：android:enabled="false"

  <activity
            android:name=".messengerP2P.view.MessengerP2PActivity"
            android:configChanges="orientation"
            android:label="@string/title_activity_messenger_p2_p"
            android:enabled="false"
            android:screenOrientation="portrait">
            <!-- Open shared telephone number as dial application -->
            <intent-filter

                android:label="@string/title_activity_messenger_p2_p">
                <action android:name="android.intent.action.VIEW" />
                <action android:name="android.intent.action.DIAL" />

                <category android:name="android.intent.category.DEFAULT" />
                <data android:scheme="tel" />
            </intent-filter>
        </activity>

アクティビティ自体は、既存のディップリンクを介して支払い方法にリダイレクトするだけです。

class MessengerP2PActivity : Activity() {

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        var phoneNumberFromDial: String? = intent?.data?.schemeSpecificPart
        phoneNumberFromDial?.let {
            if (ru.mw.utils.Utils
                    .isPhoneNumber(phoneNumberFromDial)) {
                val phoneNumberFromDialLink = PaymentActivity.getUriForProviderId(
                        resources.getInteger(R.integer.providerIdQiwiWallet).toLong(), null, null)
                    .buildUpon()
                    .appendQueryParameter(PaymentActivity.QUERY_PARAM_ACCOUNT, phoneNumberFromDial)
                startActivity(
                    Intent(Intent.ACTION_VIEW, phoneNumberFromDialLink.build()))
            }
        }
        finish()
    }
}

android：enabledフラグは、このヘルパークラスによって制御できます。

import android.content.ComponentName
import android.content.Context
import android.content.pm.PackageManager

class MessengerP2PUtils {

    companion object {
        private const val componentName = "ru.mw.messengerP2P.view.MessengerP2PActivity"

        private fun switchMessengerP2P(enabled: Boolean = true, packageName: String, packageManager: PackageManager) {
            val compName = ComponentName(packageName, componentName)
            packageManager.setComponentEnabledSetting(
                compName,
                when (enabled) {
                    true -> PackageManager.COMPONENT_ENABLED_STATE_ENABLED
                    else -> PackageManager.COMPONENT_ENABLED_STATE_DISABLED
                },
                PackageManager.DONT_KILL_APP)
        }

        fun enableMessengerP2P(applicationContext: Context) {
            val packageName = applicationContext.packageName
            val packageManager = applicationContext.packageManager
            switchMessengerP2P(enabled = true, packageName = packageName, packageManager = packageManager);
        }

        fun disableMessengerP2P(applicationContext: Context) {
            val packageName = applicationContext.packageName
            val packageManager = applicationContext.packageManager
            switchMessengerP2P(enabled = false, packageName = packageName, packageManager = packageManager);
        }

        fun isMessengerP2PEnabled(packageName: String, packageManager: PackageManager): Boolean {
            val state = packageManager.getComponentEnabledSetting(ComponentName(packageName, componentName))
            return state == PackageManager.COMPONENT_ENABLED_STATE_ENABLED || state == PackageManager.COMPONENT_ENABLED_STATE_DEFAULT
        }

    }
}

「isMessengerP2PEnabled」メソッドは、テスト中に信頼できない結果を生成したため、ここではコンポーネントの現在の状態を確認するために使用されていません。使用する場合は、非同期ビジネスロジックの条件で、およびアプリケーションをダウンロードするときに、その動作を慎重に確認してください。

フラグ値を変更する時間enabled はそれほど重要ではありません。機能フラグ構成が完全に読み込まれたときを選択しました。

メッセンジャーの電話番号をクリックすると、選択可能なシステムダイアログが表示されるので、インテントのデフォルト設定である「tel：XXXXX」を破棄する方法をユーザーに提供する必要があります。例：「番号をクリックしたときにのみ電話をかけることができる場合は、スマートフォンの設定に移動→アプリケーションリストで[電話]を選択→設定を[デフォルトで開く]にリセットします。


外出先でコンポーネントを無効にした場合、ユーザーにこのシステムダイアログが表示されても問題はありません。対応するアイテムはすぐにリストから消えます。私たちの場合は「送金」です。

ユーザーがデフォルトでアプリケーションを介してリンクを処理すると、興味深い動作が始まります。これを行うには、項目「選択を記憶する」または「常に」をポークするだけで十分です。コンポーネントを無効にすると、ユーザーは使用するアプリケーションを選択できるようになり、最初の起動時に「選択を記憶」項目はありません。機能を有効にすると、フロー全体が復元され、ユーザーはアプリケーションに直接転送されます。

結論


この機能のアイデアはずっと前に私たちに思い浮かびました、元のチケットは2017年に作成されました。アイデアは今でもその関連性を失っていません、私は同様の機能を持つ銀行アプリケーションを見つけることができませんでした。4月29日に「メッセンジャーからの転送」をリリースしました。初日には、約8,000人のユニークユーザーがそれを利用しました。近い将来のビジネス指標が私たちを満足させるならば、私たちはこの機能をさらに発展させます。
