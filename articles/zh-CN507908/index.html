在Mac OS，Linux和Windows WSL2上管理多个JDK

再次问好。预期Java Developer课程的开始，我们已经为您准备了一篇有趣的文章的翻译。





如果像我一样，您必须同时处理多个项目（旧项目和现代项目），并且也许您正在为使用不同Java版本的用户开发某种库或实用程序，那么您将了解在不同Java之间切换的问题JDK。







如今，在开发过程中安装，管理JDK和在JDK之间进行切换已成为一项艰巨的任务，因为尽管许多开发人员已经迁移到Java 11，但他们仍在使用Java 8。在本文中，我们将考虑其中的一些。



手动方式



您可以从JDK供应商的网站下载发行版，然后手动安装所有发行版（或仅将其解压缩到诸如此类的文件夹中$user/jdks



）。但是，您每次都必须检查和更新，JAVA_HOME



以便在那里找到所需的JDK。



这可以使用bash脚本，bash函数等来完成。那么问题是什么？您可以简单地忘记调用脚本/函数，而只有在看到脚本/函数时才意识到出了点问题UnsupportedClassVersionError



。



如果您喜欢手动执行所有操作，则可以浏览Internet来找到解决方案。对于高级用户而言，这可能是一个不错的选择。但是我为此使用了一些实用程序。因此，让我们看看它们。



最好的方式



在我看来，理想的解决方案是将已经存在一段时间的两个开源工具结合起来，也许您至少熟悉其中一个。但是您不知道它们可以一起使用。



  • SDKMAN！
  • 环保


让我们看看他们的分享。



安装多个JDK





安装SDKMAN之后！类型： 



$ sdk list java


查找所需的供应商，版本并输入： 



$ sdk install java <id>


让我们安装四个JDK：Oracle OpenJDK的最后两个版本和AdoptOpenJDK的两个LTS版本。



安装时，请勿选择默认的JDK。



  1. 采用OpenJDK构建OpenJDK 8u252 -LTS
  2. 采用OpenJDK构建OpenJDK 11.0.7 -LTS
  3. Oracle OpenJDK 14-最新GA
  4. 甲骨文的OpenJDK 15 -早期访问构建


这四个版本应能满足您对旧版Java 8项目，现代Java 11项目以及在Java 14和Java 15中进行创新试验的需求



。您使用SDKMAN在本地安装了四个JDK！我认为该实用程序对于安装JDK确实非常方便，并且希望您也喜欢它。



可以在以下目录中找到所有已安装的JDK： 



$ cd /Users/bruno/.sdkman/candidates/java
$ ls
11.0.7.hs-adpt  14.0.1-open     15.ea.20-open   8.0.252.hs-adpt


如果您不需要经常在不同的JDK之间切换，那么就足够了。您可以使用SDKMAN！选择JDK： 



$ sdk current java
Using java version 15.ea.19-open
$ sdk default java 15.ea.20-open
Default java version set to 15.ea.20-open


您可以在此停止并仅使用SDKMAN！，但是迟早会忘记以与手动方法相同的方式切换JDK。因此，我更喜欢JDK自动切换。



使用jEnv管理多个JDK







尽管事实是SDKMAN！安装JDK很方便，但是在从一个项目切换到另一个项目时在不同的JDK之间自动切换时却无济于事。在这里，jEnv将为我们提供帮助。



使用SDKMAN安装多个JDK之后！或以任何其他方式，您需要将它们添加到jEnv中： 



$ cd ~/.sdkman/candidates/java
$ ls -1
11.0.7.hs-adpt 
14.0.1-open    
15.ea.20-open
8.0.252.hs-adpt
$ jenv add 15.ea.20-open
openjdk64-15-ea added
15-ea added
15-ea added
...
$ jenv versions
 system
 1.8
 1.8.0.252
 11.0
 11.0.7
 14.0
 14.0.1
* 15-ea (set by /Users/bruno/.jenv/version)
 openjdk64-1.8.0.252
 openjdk64-11.0.7
 openjdk64-14.0.1
 openjdk64-15-ea


jenv add



对其余三个JDK版本 重复该命令。



jEnv对于经常使用命令行的用户特别方便。他会让你：



  1. 为您的系统全局安装Java版本。
  2. 为您所在的当前目录/项目安装Java版本。
  3. 为当前外壳安装Java版本。


jEnv使用填充程序并自动管理JAVA_HOME环境变量。因此，在配置系统后，jEnv将根据上述优先级和当前目录将JDK版本切换到最合适的版本。凉！它还具有其他有用的功能-有关更多信息，请访问网站。



在JDK之间自动切换



现在，您已经安装了Java 8、11、14和15-EA。那么如何在它们之间切换？让我们自定义版本并测试开关。



首先，您需要在系统中指定默认的全局JDK。为此，我通常使用抢先体验。每当我开始使用新项目时，我都会自动使用即将发布的版本，这有助于我确定可以报告给OpenJDK项目的潜在问题。您也这样做... 



$ jenv global 15


现在，对于您所在的任何文件夹，执行后java -version



都将收到OpenJDK 15 EA。



如果您有一个应使用Java 8的项目，请转到该文件夹​​并运行： 



$ jenv local 1.8




结果，将.java-version



使用以下内容在文件名下创建文件： 



$ cat .java-version
1.8


当您位于此文件夹中时，此文件告诉jEnv使用哪个JDK。



最后，如果您想暂时切换到其他版本的JDK，请使用jenv shell



。



这里有一个演示，可以解释以上所有内容以及自动切换的工作原理。



在Reddit上发表文章后，这里有一些有趣的评论值得一提：



  • SDKMAN！从5.8.1+版本开始，它支持sdk env，并且在那里也提供自动切换。
  • direnv非常好，不依赖Java。它与jEnv相同，只是它不能帮助您轻松切换JDK。您必须手动编辑文件。
  • autoenv是具有相同限制的另一个direnv替代方案。
  • jabba是SDKMAN + jEnv组合的最有前途的替代方案，因为它支持所有功能，并且可以在Windows（PowerShell）中运行，而上述其他工具均不支持。
