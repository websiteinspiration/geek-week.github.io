Menjelajahi cybergrouping Cobalt: alat dan serangan baru apa yang digunakan peretas



Spesialis di PT Expert Security Center telah memantau aktivitas kelompok Cobalt sejak 2016 . Hari ini, kelompok ini menyerang lembaga keuangan di seluruh dunia. Kerusakan dari serangan ini setahun yang lalu melebihi 1 miliar rubel. Selama empat tahun terakhir, kami telah menerbitkan beberapa laporan serangan yang terkait dengan grup ini.

Selama tahun lalu, grup Cobalt tidak hanya memodifikasi alat CobInt utamanya dan dropper COM-DLL bersamaan dengan backdoor more_eggs JavaScript, tetapi juga menggunakan metode pengiriman baru dan teknik baru untuk memintas perlindungan pada tahap awal serangan. Memperbarui taktik dan alat mungkin karena fakta bahwa kelompok tersebut telah lama menjadi sasaran para peneliti dari seluruh dunia, dan perlu satu langkah di depan untuk mem-bypass peralatan pelindung.

Rata-rata, pada 2019, kelompok itu melakukan tiga serangan per bulan. Dalam artikel baru kami - analisis taktik baru dan alat pengelompokan cyber. Versi lengkap dari penelitian ini tersedia di sini .

Situs phishing Bank Sentral Eropa


Pada akhir Agustus 2019, ketika menyelidiki ancaman keamanan informasi, kami mencatat serangan menggunakan CobInt. Serangan itu diduga ditujukan ke lembaga keuangan Eropa. Kami tidak memiliki data tentang seberapa sukses itu. Pemasang NSIS khusus berfungsi sebagai dropper untuk CobInt. Kami menemukan tiga versi penetes seperti itu: untuk Chrome, Firefox, dan Opera. Di dalam masing-masing adalah versi yang sama dari CobInt dan penginstal yang sesuai untuk browser. Setelah memulai, dropper menyimpan CobInt ke folder% TEMP%, dan installer juga meluncurkannya. Setelah menganalisis malware, kami menemukan bahwa distribusi dropper terjadi dari situs phb ecb-european [.] Eu.



Beranda Situs Phishing

Situs itu adalah salinan situs Bank Sentral Eropa - kecuali ketika dikunjungi, pengguna diperlihatkan jendela pop-up yang memintanya untuk memperbarui browser.



Jendela sembulan di situs web ECB palsu.

Akibatnya, penetes tersebut diunduh ke komputer pengguna. Di kode sumber halaman ada tautan ke skrip yang menampilkan jendela ini.

Kemungkinan besar, pengguna masuk ke situs menggunakan email phishing yang mirip dengan yang paling sering digunakan oleh kelompok Cobalt.

2. VHD berbahaya


Pada akhir Desember 2019, kami menemukan bootloader CobInt Cobalt berikutnya. Unusual adalah wadah yang berisi bootloader ini. Itu adalah hard disk virtual (VHD), yang seharusnya dikirimkan.

Format VHD telah digunakan sejak penciptaan teknologi virtualisasi Hyper-V. File VHD dapat berisi semua yang dapat ditemukan pada HDD biasa: partisi disk, sistem file dengan folder dan file. Mulai dengan Windows 7, pengguna dapat menghubungkan VHD secara manual, misalnya melalui konsol MMC. Dimulai dengan Windows 8, pengguna dapat menghubungkan VHD hanya dengan mengklik dua kali dengan mouse, dan itu akan muncul di sistem sebagai disk biasa.

Pada bulan September 2019, sebuah artikel dipublikasikan di blog CERT / CCbahaya file VHD dan penggunaannya sebagai vektor potensial dalam serangan. Will Dorman menunjukkan bahwa antivirus maupun tanda di web tidak akan memberi sinyal kepada pengguna bahwa isi file VHD yang diunduh dari Internet dapat membahayakan komputer. Ada kemungkinan bahwa kelompok Cobalt menggunakan hasil penelitian ini. File VHD mereka juga tidak terdeteksi oleh antivirus pada saat itu muncul di VirusTotal. Enam bulan kemudian, file ini terdeteksi oleh hanya satu antivirus: ini adalah indikator yang sangat rendah.



Di dalam VHD adalah dua file CobInt. Dua sertifikat Google yang tidak valid dilampirkan di akhir salah satu file untuk mengurangi kemungkinan deteksi oleh fitur keamanan.

Setelah VHD dipasang, pengguna harus memulai salah satu file secara manual. Secara fungsi, kedua file tersebut identik. Setelah memulai salah satu CobInt, pustaka utama dimuat. Pustaka utama dimuat oleh CobInt dari server kontrol sebagai file HTML.

3. BIFF makro


Pada bulan Maret 2020, kami merekam dokumen grup Cobalt dalam format XLS, yang mengunduh dan menjalankan dropper COM-DLL. Dokumen tersebut berisi format makro Excel 4.0 yang cukup lama dan praktis tidak terdeteksi oleh antivirus (1 vonis dari 60 menurut layanan VirusTotal).



Jumlah vonis antivirus pada pengunduhan pertama file makro Excel 4.0 ke VirusTotal
Standar makro ini telah digunakan selama 20 tahun. Keunikannya adalah bahwa makro tidak disimpan dalam proyek VBA, itu terletak di sel-sel halaman, sementara halaman dapat disembunyikan di Excel itu sendiri.

Ini berarti bahwa makro tidak akan berada di aliran VBA, tetapi dalam catatan BIFF (Binary Interchange File Format).
Jika Anda membuka dokumen di Excel, maka kami hanya akan melihat satu halaman dan tidak adanya makro dalam proyek VBA. Namun, Excel akan tetap mendeteksi keberadaan makro dan memblokir eksekusi.

Membuka dokumen sumber di pengelola rumus, kita akan melihat bahwa salah satu rumus dimulai secara otomatis saat Anda membuka dokumen.



Rumus makro yang dimulai saat dokumen dibuka.

Rumus awal meluncurkan rantai besar perintah COMBIN, EXECUTE, SYMBOL, CALL, yang pada akhirnya mengarah ke pemuatan dan peluncuran dropper COM-DLL. Keseluruhan rantai perintah tersebar di bidang sel besar di halaman Excel, yang membuat analisis menjadi sulit.

4. Analisis dropper COM-DLL


Pada awal April 2020, kami menemukan versi baru dropper COM-DLL. Dalam hal fungsi, ini berbeda dari semua yang kita lihat sebelumnya. Namun, payload dalam bentuk JavaScript backdoor more_eggs tetap sama.

Dropper COM-DLL muncul di gudang Cobalt pada musim panas 2017 dan masih digunakan oleh grup untuk mengirimkan JavaScript backdoor more_eggs, yang terkandung di dalamnya dalam bentuk terenkripsi dan diarsipkan.

Fitur penetes:

  • Sepenuhnya ditulis dalam PureBasic.
  • Menggunakan sejumlah besar teknik anti-analisis yang berbeda.
  • Ini berisi loader JavaScript terenkripsi dan diarsipkan, JavaScript backdoor, utilitas konversi baris perintah yang sah untuk meluncurkan JavaScript backdoor more_eggs.
  • Ini memiliki obfuscator bawaan untuk JavaScript backdoor dan pemuat JavaScript yang terhubung.

Kesimpulan


Cobalt Group terus menyerang lembaga keuangan di seluruh dunia, mengambil taktik baru, memperbarui alat, dan mencoba untuk secara lebih efektif memotong pertahanan. Karena karantina, banyak pekerja sektor kredit dan keuangan sekarang bekerja "dari jauh", di luar jangkauan pemulihan perusahaan . Selain itu, banyak penyerang menggunakan tema coronavirus dalam serangan mereka, seperti kelompok Higaisa . Ada kemungkinan bahwa kelompok Cobalt akan mencoba menggunakan tema ini dalam serangannya di masa depan.
