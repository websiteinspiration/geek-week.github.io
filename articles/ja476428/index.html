Nginx VTS Stats、Mockify-モックAPI、WiremockおよびMockifyの比較

この投稿では、Nginx VTS + Prometheus + Grafanaのデモを作成したいと思いました。デモでは、アップストリームが異なるhttpコードを発行できることが必要でした。これを行うことができるプロジェクトは、Golangで記述されたMockifyとJavaで記述されたWireMockです。


小さな目次


  • Nginx VTS + Prometheus + Grafanaのインストールと構成。
  • Mockifyは、Golangで記述された、軽量で構成可能なエミュレーションAPIです。
  • Golangで記述されたMockifyとJavaで記述されたWireMockのCPU使用率の比較。

Nginx VTSの場合、nginx-module-vtsモジュールを使用してnginxをコンパイルする必要があります。これはNginx-builderを使用して行います。彼の設定：


nginx_version: 1.16.1
output_package: rpm
modules:
  - module:
      name: nginx-module-vts
      git_url: https://github.com/vozlt/nginx-module-vts.git
      git_tag: v0.1.18

組み立てられたnginxをインストールします。ここにその主要な設定があります（vhost_traffic_status_zoneを指定することを忘れないでください）：


user  nginx;
worker_processes  auto;
worker_rlimit_nofile 40960;

error_log  /var/log/nginx/error.log warn;
pid        /var/run/nginx.pid;

events {
    use epoll;
    worker_connections 1024;
    multi_accept on;
}

http {
    vhost_traffic_status_zone;
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

    #access_log /var/log/nginx/access.log main;
    access_log off;
    sendfile on;
    tcp_nodelay on;
    tcp_nopush on;
    keepalive_timeout  65;
    include /etc/nginx/conf.d/*.conf;
    open_file_cache max=200000 inactive=20s;
    open_file_cache_valid 30s;
    open_file_cache_min_uses 2;
    open_file_cache_errors on;
}

nginx-vts-exporterをインストールします。


wget https://github.com/hnlq715/nginx-vts-exporter/releases/download/v0.10.3/nginx-vts-exporter-0.10.3.linux-amd64.tar.gz
tar zxvf nginx-vts-exporter-0.10.3.linux-amd64.tar.gz
cp nginx-vts-exporter-0.10.3.linux-amd64/nginx-vts-exporter /usr/local/bin/nginx-vts-exporter

/etc/systemd/system/nginx_vts_exporter.serviceファイルを作成します。


[Unit]
Description=Nginx vts exporter  
Wants=network-online.target
After=network-online.target
[Service]
ExecStart=/usr/local/bin/nginx-vts-exporter -nginx.scrape_uri=http://localhost:7070/status/format/json
Restart=always
RestartSec=3
[Install]
WantedBy=default.target

VTS構成ファイル/etc/nginx/conf.d/vts-exporter.confを作成します


server {
 listen 7070;
 listen [::]:7070;

 location / {
 }

 location /status {
   vhost_traffic_status_display;
   vhost_traffic_status_display_format html; 
 }

}
