アニメーション化されたファビコンの開発

これは、ユーザーがブラウザのタブを切り替えるときに最初に探すものです。ファビコンと呼ばれるものの可能な説明の1つを示しました。Webページのタブタブを占める画面スペースは、多くの人が考えるよりもはるかに価値のあるリソースです。ラベルを適切に処理すると、ページ識別子の役割を引き続き果たすだけでなく、ページで何が起こっているかを正確に伝える「メッセージボード」のようなものになる可能性があります。


ファビコン

実際、ファビコンは、非アクティブなブラウザータブに関しては特に便利です。例を考えてみましょう。

最近の夏休みに撮影した写真を、そのWebページを使用してクラウドサービスにアップロードするとします。写真がアップロードされている間に、新しいブラウザタブを開いて、以前に行った場所の詳細を見つけて写真のキャプションで使用します。このすべてを探している間、一方がもう一方に固執し、ブラウザの7番目のタブでCasey Neistatを監視しています。しかし、YouTubeマラソンを安全に続けることはできません。クラウドサービスページを定期的に見て、写真が既にアップロードされているかどうかを確認します。

これはあなたが創造的になることができるまさにその場合です！ファビコンを構成するピクセルを動的に変更し、ファイルアップロードプロセスに関する情報をページショートカットに表示できるとしたらどうでしょう。実際のところ、これはここで行うことです。

これをサポートするブラウザでは、JavaScript、HTML要素、<canvas>および古いジオメトリルールを使用して、一部のアニメーションをファビコンとして表示できます。

これは、ブラウザのタブでの読み込みアニメーションの外観です。

すぐにビジネスに取り掛かり、最も単純なものから始めます。すなわち-ページ要素のHTMLコードに追加する<link>、と<canvas>：

<head>
    <link rel="icon" type="image/png" href="" width=32px>
</head>

<body>
    <canvas width=32 height=32></canvas>
</body>

この手法を実際に使用する場合は、おそらく要素を非表示にする必要があり<canvas>ます。これを行う1つの方法は、HTML属性を使用することですhidden。

<canvas hidden width=32 height=32></canvas>

要素を<canvas>表示したままにして、ファビコンと要素の両方にアニメーションが同時に表示される様子を確認できるようにします<canvas>。どちらも標準のファビコンのサイズ（32x32ピクセル）です。

デモの目的で、ページにボタンを追加し、クリックするとアニメーションが開始されます。次のコードをページのHTMLコードに追加します。

<button>Load</button>

それでは、JavaScriptコードに入りましょう。最初にサポートを確認してくださいcanvas：

次に、要素のアニメーションを開始するために使用されるボタンクリックイベントハンドラーを接続します<canvas>。

button = document.querySelector('button');
button.addEventListener('click', function() {

drawLoader渡しsetIntervalた エンティティは、60ミリ秒の間隔で呼び出され、要素に何かを出力する関数です<canvas>。この関数のコードを記述する前に、ロードインジケーターである正方形を徐々に描画するときに使用する線のスタイルについて説明します。

/*  ,         */

関数でdrawLoaderは、アニメーションステップの何パーセントが既に完了したかに基づいて線を描画します。この場合、1つのアニメーションステップは1％に相当します。つまり、最初の25％のステップでは、正方形の上辺を表す線が徐々に表示されます。ステップの次の25％は、正方形の右側を表示します。つまり、各側のアニメーション時間の25％になります。

アニメーション効果は、要素のコンテンツを消去<canvas>し、前のステップと比較して少し長い線を出力することによって実現されます。アニメーションの各ステップで、要素への画像の出力が<canvas>終了した後、画像はすぐにPNG形式に変換され、ページファビコンとして設定されます。

次に、必要な線を引くのに役立つ計算を行ってみましょう。

以下は、アニメーションの最初の25ステップで正方形の上面を徐々に表示するために使用されるデータです。

  • 開始点は、画像の左上隅（0,0）です。
  • nは、現在のアニメーションステップの番号です。
  • x- x現在のアニメーションステップでのラインの終点の座標。
  • yはy終点の座標で、0です。

アニメーションの25ステップすべてを完了した後、値xは32（つまり、ファビコンと要素のサイズ<canvas>）になる必要があります。したがって、次の数式を使用xして、現在のアニメーションステップで値を検索しますn。

x = (32/25) * n

この推論が実装された線画コードは次のようになります。

moveTo(0, 0); lineTo((32/25)*n, 0);

次の25のアニメーションステップ（正方形の右側を描画）は、同様の方法で実行されます。

moveTo(0, 0); lineTo(32, 0);
moveTo(32, 0); lineTo(32, (32/25)*(n-25));

そしてここに完全な関数コードがありますdrawLoader：

function drawLoader() {
  with(context) {
    clearRect(0, 0, 32, 32);
    beginPath();
    /*  25% ,   */
    if (n<=25){ 
      /*
        (0,0)-----(32,0)
      */
      moveTo(0, 0); lineTo((32/25)*n, 0);
    }
    /*  25  50  */
    else if(n>25 && n<=50){ 
      /*
        (0,0)-----(32,0)
                  |
                  |
                  (32,32)
      */
      moveTo(0, 0); lineTo(32, 0);
      moveTo(32, 0); lineTo(32, (32/25)*(n-25));
    }
    /*  50  75  */
    else if(n>50 && n<= 75){ 
      /*
        (0,0)-----(32,0)
                  |
                  |
        (0,32)----(32,32)
      */
      moveTo(0, 0); lineTo(32, 0);
      moveTo(32, 0); lineTo(32, 32);
      moveTo(32, 32); lineTo(-((32/25)*(n-75)), 32);
    }
      /*  75  100  */
    else if(n>75 && n<=100){
      /*
        (0,0)-----(32,0)
            |      |
            |      |
        (0,32)----(32,32)
      */
      moveTo(0, 0); lineTo(32, 0);
      moveTo(32, 0); lineTo(32, 32);
      moveTo(32, 32); lineTo(0, 32);
      moveTo(0, 32); lineTo(0, -((32/25)*(n-100)));
    }
    stroke();
  }
  //  ,     <canvas>,   PNG,     
  favicon.href = canvas.toDataURL('image/png');
  /*  ,    */
  if (n === 100) {
    clearInterval(loadingInterval);
    return;
  }
  //  ,     
  n++;
}
