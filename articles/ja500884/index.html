DjangoでのVueJSの使用

記事の翻訳は、Python Web Developerコースの開始前に準備されました。



前書き


現在、非常に興味深いプロジェクトに取り組んでいます。そして、プロジェクト自体は非常に大規模であり、私はフルタイムで働いている間、私は自分の自由時間に一人でそれを扱います。したがって、私は効果的でなければなりません。幸い、私はDjangoを「バッテリー内蔵」アプローチで使用しています。

私はDjangoのすべての機能を使用しています。これにより開発がスピードアップし、テンプレートエンジンを見失いたくありません。したがって、DjangoのバックエンドとJavaScript SPAのフロントエンドは、私にとって選択肢ではありません。ただし、最も熱心なバックエンド開発者でさえ、クライアント側で実装する必要があるものがあることを認めなければなりません。マイナーなユーザーアクションでは、ページの再読み込みは必要ありません。さらに、私が作成するWebアプリケーションの一部には、かなり複雑なユーザー操作が必要です。

伝統的に、DjangoとjQueryを混合して、望ましい動作を得ることができました。しかし今、ReactやVueのような新しいJavaScriptテクノロジーがあります。私たちの目標は、すべてを最初から書き直さなくてもDjangoで使用できるフレームワークを見つけることなので、より簡単な代替手段としてVueを使用します。この記事では、DjangoでVueを最小限の労力で使い始める方法を紹介します。

インストールとセットアップ


Vueを使用する理由の1つは、その優れたドキュメントです。多くの例があり、まともな検索と明確な目次があります。この記事の目的は、Djangoプロジェクトで複雑な時間の長いセットアップをせずにVueの使用をすぐに開始できることを示すことです。したがって、最も簡単な接続方法を使用Vue.jsして、タグ経由で追加し<script>ます。

<!-- development version, includes helpful console warnings -->
<script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>

これで、最初のインスタンスを作成する準備ができました Vue.js。

<div id="app">
  {{ message }}
</div>

var app = new Vue({
  delimiters: ["[[", "]]"],
  el: '#app',
  data: {
    message: 'Hello Vue!'
  }
})

私は公式の入門ガイドからこの例を取り上げました。ただし、他にもあります。デフォルトでは、DjangoとVueは同じテンプレートタグを使用します。したがって、Djangoテンプレートエンジンとの競合を回避するには、VueをDjangoから明示的に分離する必要があります。

VueからDjangoデータにアクセスする


データを取得する最も簡単な方法はjscon_script、Djangoの組み込みフィルターを使用することです。このようにして、DjangoモデルをVueインスタンスのデータとしてすぐに使用できます。

HTML：

{{ django_template_variable|json_script:"djangoData" }}

次に、JavaScriptでデータを変数にロードします。

let jsVariable = JSON.parse(document.getElementById('djangoData').textContent);


そして、彼女はVueで使用する準備ができています。

new Vue({
  delimiters: ["[[", "]]"],
  el: '#app',
  data: jsVariable
})

バックエンドで非同期リクエストを実行する


Vueのフロントエンドで最も一般的なタスクの1つは、バックエンドのサーバーアプリケーションにクエリを実行することです。完全なDjangoアプリでは、すべてのユーザーアクションでこれを行う必要はありません。場合によっては、ページ全体の再読み込みが優れており、Djangoテンプレートシステムがすべての利点を提供します。しかし、ユーザーエクスペリエンスを改善し、Vueを最大限に活用するために、場合によってはバックエンドリクエストを行う必要があるかもしれません。

Vue自体はリクエストの処理方法を知りません。この記事ではaxiosを使用します公式のVueドキュメントでも推奨されているためです。他の選択肢を選択できます。DjangoのCSRF保護メカニズムを渡すには、axiosがリクエストに適切なCookieを含める必要があります。最も簡単な方法は、デフォルトでグローバルaxios値を設定することです：

axios.defaults.xsrfCookieName = 'csrftoken';
axios.defaults.xsrfHeaderName = "X-CSRFTOKEN";

さらに、必要な設定でaxiosインスタンスを作成できます。

var instance = axios.create({
    xsrfCookieName: 'csrftoken',
    xsrfHeaderName: "X-CSRFTOKEN",
});

Djangoテンプレートにはタグを含める必要があります。{% csrf_token %}または、対応するビューでデコレータを使用する必要がありますensure_csrf_cookie()。認証用のDjangoセッションのデフォルトのバックエンドの残りはそのまま使用できます。つまり、バックエンド上のサービスにloginRequiredこのような注釈を付けることができ、機能するだけです。リクエストを行うには、axiosを使用できます。

axios.post('/django/backend/endpoint', {
    data: jsVariable 
})
    .then(function (response) {
        // handle response
    })
    .catch(function (error) {
        // handle error
    });

この呼び出しmountedは、Vueインスタンスのフックから、またはJavaScriptコードを配置できる他の場所から行うことができます。設定でDjango

を有効にCSRF_USE_SESSIONS CSRF_COOKIE_HTTPONLYした場合、DOMからCSRFトークンを読み取る必要があります。詳しくは、Djangoの公式ドキュメントをご覧ください。

結論


Django + Vueをググると、ほとんどの検索結果は、バックエンドでのDjangoの使用方法と、フロントエンドでの別のプロジェクトのVueに関するものになります。2つの独立したプロジェクトがあると作業が複雑になり、Djangoテンプレートシステムにアクセスできなくなります。これは時間を節約するのに非常に効果的な方法です。一方、Vueのようなフレームワークへのアクセスは、CRUD機能を超えるWebアプリケーションの世界で不思議に思うことがあります。
