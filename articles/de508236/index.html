Die Idee der Superauflösung besteht darin, die Auflösung des Bildes zu verbessern, indem die fehlenden Pixel im neuronalen Netzwerk basierend auf den verfügbaren Daten gezeichnet werden. Die meisten Algorithmen sind jedoch ressourcenintensiv, und ich wollte ein Skript erstellen, um das Spiel in Echtzeit zu verbessern.



Ein bisschen Theorie



Alle folgenden Punkte gelten für Faltungs-Neuronale Netze - ein Subtyp von Neuronalen Netzen, die zum Arbeiten mit Bildern verwendet werden. Schauen wir uns zunächst an, wie ein neuronales Netzwerk funktioniert, um das Problem der Superauflösung zu lösen. Die Aufgabe ist der Lösung des Auto-Encoder- Problems sehr ähnlich. Es ist erforderlich, ein Bild an den Eingang des Netzwerks zu senden, um dasselbe Bild am Ausgang zu erhalten. Autoencoder werden jedoch normalerweise verwendet, um das Problem der effizienten Datenkomprimierung zu lösen. Daher ist ein Merkmal ihrer Architektur der Engpass - ein Engpass, dh eine Netzwerkschicht mit einer kleinen Anzahl von Neuronen. Das Vorhandensein einer solchen Schicht lässt die verbleibenden Teile lernen, Informationen effizient zu codieren und zu decodieren. Um Superauflösungsnetzwerke zu trainieren, wird die Auflösung von Bildern hoher Qualität zunächst absichtlich reduziert und dem Eingang des neuronalen Netzwerks zugeführt. Die Ausgabe wird als Quellbild in hoher Qualität erwartet.



Die Superauflösungsaufgabe definiert die Architektur der verwendeten Netzwerke:

In der Regel besteht eine Verbindung zwischen Eingabe- und Ausgabedaten (Sprungverbindung), was das Lernen erheblich beschleunigt. Die Pixelgröße der Eingabedaten nimmt zu und wird zur Ausgabe des Faltungsnetzwerks hinzugefügt. Sie müssen das Netzwerk also nicht trainieren, um das Bild fast gleich zu machen. Sie müssen sie nur trainieren, um den Unterschied zwischen der Vergrößerung durch Vergrößern des Pixels und dem realen Bild zu ermitteln. Die Idee, Verbindungen auf verschiedenen Ebenen und über eine unterschiedliche Anzahl von Schichten zu überspringen, ist äußerst effektiv und hat zur Entstehung der Netzwerkklasse der Residual Networks geführt. Heutzutage werden solche Verbindungen in fast allen gängigen Architekturen verwendet. Einen guten Überblick über modernste Architekturen zur Lösung von Superauflösungsproblemen finden Sie hier.. Meine Aufgabe war es, ein neuronales Netzwerk zu erstellen, um das Problem der Superauflösung in Echtzeit zu lösen.



Zuerst habe ich mich für die edsr-Architektur mit 4 Blöcken entschieden (normalerweise werden mehr als 16 Schichten von Blöcken verwendet). ResNet mit einer vierfachen Auflösungserhöhung. Hier habe ich teilweise die Entwicklungen von github.com/krasserm/super-resolution und den Datengenerator aus demselben Projekt verwendet.



Die allgemeine Netzwerkarchitektur ist im Diagramm dargestellt. Jeder Block ist ein X * Y * N-Bild, wobei die Breite der Anzahl der Kanäle entspricht. Übergänge - entsprechen 3x3 Faltungen (im Fall von Res-Blöcken, gefolgt von der Aktivierung von ReLU wegen Nichtlinearität). Upscaling-Schritt - Vergrößerung der Abmessung durch Abflachung der Kanäle.







Wenn Sie sich ein so kleines Netzwerk genau ansehen, werden Sie feststellen, dass die letzten Blöcke linear sind und das Vorhandensein mehrerer Upscaling-Schritte nicht gerechtfertigt ist. Sie können sie durch eine Schicht mit viel weniger Filtern ersetzen, ohne die Qualität zu verlieren, ohne die Qualität des Netzwerks zu verlieren:







Es stellte sich heraus, dass ein solches Netzwerk die Superauflösungsaufgabe fast auf die gleiche Weise bewältigt, jedoch viel schneller und gestartet werden kann, um eine kleine Videoauflösung in Echtzeit zu verarbeiten .



Hochwertige Fotos wurden von data.vision.ee.ethz.ch/cvl/DIV2K heruntergeladen . Grundsätzlich können Sie jedes Foto zum Training verwenden. 



import tensorflow as tf
from model import resolve_single
from model. rtsr import rtsr
import numpy as np
from utils import load_image, plot_sample
from data import DIV2K
from tensorflow.keras.optimizers.schedules import PiecewiseConstantDecay
# Model creation
rtsr_model= rtsr(scale=4, num_res_blocks=4)
#Data loading and generator creation
train_loader = DIV2K(scale=4,             
                     downgrade='bicubic',
                     subset='train')      
                     


train_ds = train_loader.dataset(batch_size=16,
                                random_transform=True,
                                repeat_count=None)  
valid_loader = DIV2K(scale=4,  
                     downgrade='bicubic',
                     subset='valid')  
                     

valid_ds = valid_loader.dataset(batch_size=16,           # use batch size of 1 as DIV2K images have different size
                                random_transform=True, # use DIV2K images in original size
                                repeat_count=40)         # 1 epoch
#Saving checkpoints during training
cbcks=tf.keras.callbacks.ModelCheckpoint('callback_x4{epoch:02d}-{loss:.2f}', monitor='loss', verbose=0, save_best_only=False)

learning_rate=PiecewiseConstantDecay(boundaries=[20000], values=[1e-4, 5e-5])
#model compilation before training
rtsr_model.compile(loss='mae', optimizer=tf.keras.optimizers.Adam(learning_rate=learning_rate))
#Training
rtsr_model.fit(train_ds,validation_data=valid_ds, steps_per_epoch=3000, epochs=100, validation_steps=10,callbacks=[cbcks])
#saving weights
rtsr_model.save_weights('weights/ myedsr-lastdiv2_4block-x4_weights.h5')


Die Ausgabe ist ein relativ kleines (2,6 MB) neuronales Netzwerk mit einer einfachen Architektur. Gleichzeitig ergibt die Prüfung einen subtilen Unterschied zum vorab trainierten 16-Block-Netzwerk:







Das Bild links ist das Original, rechts wird es mit dem 16-Layer-Edsr in der Mitte verbessert - mit rtsr. Der Unterschied zwischen 2 und 3 ist aus meiner Sicht nicht signifikant.

Wie man anfängt?



Um schnell arbeiten zu können, benötigen Sie eine Grafikkarte mit Unterstützung für cuda und cudnn (https://developer.nvidia.com/cuda-gpus) und die installierten cuda / cudnn-Bibliotheken. Sie müssen Python 3.7 und Tensorflow installiert haben (Version höher als 2.0 mit GPU-Unterstützung). Dies kann eine entmutigende Aufgabe sein und Kompatibilitätsprobleme können auftreten .



Am einfachsten wäre es, die Anaconda-Distribution (https://www.anaconda.com/products/individual) zu installieren und dann tensorflow-gpu mit conda zu installieren: 



conda install tensorflow-gpu


Wenn es aufgrund von Konflikten nicht funktioniert, können Sie es versuchen 



conda install cudnn
pip install tensorflow-gpu


sollte arbeiten.



Andere Bibliotheken können mit pip installiert werden: 



pip install opencv-python
pip install pywin32
pip install mss


Als nächstes müssen Sie das Skript mit dem folgenden Befehl ausführen: 



python superres.py


Das Aufnahmefenster wird mit den Tasten wsad und ijkl gesteuert (Größe ändern). q- Schließen des Fensters. 0 - Superauflösungsmodus ein- / ausschalten. Nummer 1 und 2 - Bildmodus (2 bedeutet standardmäßig, dass das Spiel eine Erhöhung aufgrund einer Vergrößerung des Pixels um das Zweifache verwendet).
