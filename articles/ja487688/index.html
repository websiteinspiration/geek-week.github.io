Pythonでのタイプ2019-nCoVの対話型コロナウイルス分布マップの開発

コロナウイルスタイプ2019-nCoVは、中国の武漢市で発生した後、急速に世界中に広がっています。元の記事を書いている時点（2020年1月30日）で、9,000人以上の感染者と213人の死者が報告されました。コロナウイルス感染の症例は、フランス、オーストラリア、ロシア、日本、シンガポール、マレーシア、ドイツ、イタリア、スリランカ、カンボジア、ネパール、その他多くの国で検出されています。ウイルスがいつ停止するかは誰にもわかりません。一方、コロナウイルスの確定患者数は増加しているにすぎません。

今日翻訳中の記事の著者は、Pythonを使用して、コロナウイルスの蔓延を追跡する簡単なアプリケーションを作成する方法についてお話したいと思います。このアプリケーションでの作業が完了すると、ウイルスの広がりのマップを表示するHTMLページと、データがマップに表示される日付を選択できるスライダーが読者に表示されます。


ここでは、

Python 3.7、Pandas、Plotly 4.1.0、Jupyter Notebookなどのコロナウイルスタイプ2019-nCoVテクノロジーの分布のインタラクティブマップを使用します。

ライブラリをインポートする


まず、Jupyter Notebook、PlotlyおよびPandasライブラリに基づくプロジェクトにインポートします。

import plotly.offline as go_offline
import plotly.graph_objects as go
import pandas as pd

次に進む前に、コードを実行してみてください。エラーメッセージが表示されない場合は、必要なすべてのライブラリがインストールされ、正しく機能しています。コードの実行に失敗した場合は、PlotlyとPandasの公式ページを確認し、これらのライブラリのリファレンスとインストール資料を読んでください。ハードウェアでJupyter Notebookシステムを実行していない場合は、Jupyter Notebook を操作できるクラウドプラットフォームであるGoogle Colabを使用することをお勧めします。

コロナウイルスデータの最初の5行

左下隅に、データテーブルに47列あるという情報が表示されます。ここでは最初の5つの列の名前です：country、location_id、location、latitudeとlongitude。 ：その他の列は、名前が以下のように構築されているペアですconfirmedcase_dd-mm-yyyyとdeaths_dd-mm-yyyy。この資料の執筆時点での表の列の総数は47でした。これは、21日間のデータを自由に使えることを意味します（（47-5）/ 2 = 21）。データ収集の開始日が10.01.2020の場合、終了日は30.01.2020です。

テーブルの最初の5列の名前は変更されませんが、時間の経過とともに新しい名前の列がテーブルに追加されます。私たちのインタラクティブマップが出力するのは、コロナウイルスの分布を視覚化したもので、マップが形成された日付を示すことができます。したがって、データセット全体から日ごとの情報を選択し、テーブルの最初の5列は変更されず、毎日が2列で記述されることを考慮して、データセット全体を分離する必要があります。次に、データ、たとえば10.01.2020のデータを詳しく見ると、多くの日付がこの日付に対応していることがわかります。実際、この日、コロナウイルスの検出が確認されたのは1か所だけで、対応する番号が付けられています。この日付の他のすべての行にはゼロのみが含まれます。その意味は、これらのラインをマップ作成プロセスから除外する必要があることを確認します。

視覚化のためのデータを準備するプロセスは、サイクルで実行されます。

#  
fig=go.Figure()
col_name=data.columns
n_col=len(data.columns)
date_list=[]
init=4
n_range=int((n_col-5)/2)

#,          
for i in range(n_range):
    col_case=init+1
    col_dead=col_case+1
    init=col_case+1
    df_split=data[['latitude','longitude','country','location',col_name[col_case],col_name[col_dead]]]
    df=df_split[(df_split[col_name[col_case]]!=0)]
    lat=df['latitude']
    lon=df['longitude']
    case=df[df.columns[-2]].astype(int)
    deaths=df[df.columns[-1]].astype(int)
    df['text']=df['country']+'<br>'+df['location']+'<br>'+'confirmed cases: '+ case.astype(str)+'<br>'+'deaths: '+deaths.astype(str)
    date_label=deaths.name[7:17]
    date_list.append(date_label)
    
    #  Scattergeo
    fig.add_trace(go.Scattergeo(
    name='',
    lon=lon,
    lat=lat,
    visible=False,
    hovertemplate=df['text'],
    text=df['text'],
    mode='markers',
    marker=dict(size=15,opacity=0.6,color='Red', symbol='circle'),
    ))

動作中、各データセットの出力は、を使用して、Scattergeoグラフに追加されfig.add_traceます。執筆時点では、画像の構築に使用されるデータは21個のオブジェクトで表されています。コマンドを使用してこれを確認できfig.dataます。

スライダー作成


ここでは、マップ上に視覚化されたデータの選択を整理するのに役立つスライダーを作成します。関連するコードは次のとおりです。

# 
steps = []
for i in range(len(fig.data)):
    step = dict(
        method="restyle",
        args=["visible", [False] * len(fig.data)],
        label=date_list[i],
    )
    step["args"][1][i] = True  #  i-     "visible"
    steps.append(step)
    
sliders = [dict(
    active=0,
    currentvalue={"prefix": "Date: "},
    pad={"t": 1},
    steps=steps
)]

スライダーコードは2つの主要なフラグメントで構成されています。1つ目はsteps、スライダースライダーを移動するために使用されるリストが入力されるループです。スライダーを動かすと、対応するデータセットが視覚化され、前に表示されていたものが非表示になります。コードの2番目の部分は、以前に作成されたリストstepsをSliderオブジェクトに含めることです。スライダーが移動すると、対応する要素がから選択されstepsます。

概要


コロナウイルスタイプ2019-nCoVの分布を視覚化するためのインタラクティブマップの作成に関するガイドのレビューを完了しました。この資料を学習した後、Googleの公開スプレッドシートからデータを読み取る方法、パンダを使用してデータを処理する方法、およびスライダーとPlotlyを使用してインタラクティブな地図上でこのデータを視覚化する方法を学びました。プロジェクトの結果はHTMLページとしてここからダウンロードできます。。マップに表示される情報は、データテーブルによって異なります。プロジェクトコードが実行されるたびにマップが更新され、テーブルからの新しいデータが使用可能になります。これは非常にシンプルなマップです。それを改善する方法はたくさんあります。たとえば、追加のグラフやいくつかの要約データなどを追加できます。興味があれば、このすべてを自分で行うことができます。
