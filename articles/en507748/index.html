However, this is not the only way to build a user interface based on Node-red. There is a fairly popular module containing a number of nodes for creating dashboards in real time from the creators of the node-red-dashboard platform itself .







The Template widget can contain any valid html and Angular / Angular-Material directives. This node can be used to create a dynamic user interface element that changes its appearance based on the input message and can send messages back to Node-RED. Using the Template node, you can create an object of the Scadavis screen page, set the necessary properties, emulate the behavior of the object, and associate the tags (variables) of the screen page with the external environment

substationsynoptic.on("click", function (event, tag) {
    var v = substationsynoptic.getValue(tag);
    if (event.currentTarget.id === "TAPUP")
        substationsynoptic.setValue(tag, v + 1, false, false);
    else
        if (event.currentTarget.id === "TAPDOWN")
            substationsynoptic.setValue(tag, v - 1, false, false);
    if (event.currentTarget.id === "XCBROPEN")
        substationsynoptic.setValue(tag, false, false, false);
    else
        if (event.currentTarget.id === "XCBRCLOSE")
            substationsynoptic.setValue(tag, true, false, false);
    if (v === true)
        substationsynoptic.setValue(tag, false, false, false);
    else
        if (v === false)
            substationsynoptic.setValue(tag, true, false, false);

  • 0 - not initialized state
  • 1 - false (off) state

PLC plugin for Siemens



The Node-red arsenal has a plug-in for working with Siemens PLC - node-red-contrib-s7 , based on the nodeS7 framework . The plugin is quite simple and intuitive to use. It contains three nodes: S7 in, S7 out, S7 control. The first two nodes are mainly used for reading and writing tags in the PLC. One nuance that must be taken into account when working with S7-1200 is described in the plug-in dock. I came across this earlier when I used the Snap7 library .







Some additional configuration steps are required in the PLC:



"Optimized block access" should be disabled for those DB blocks that we want to access




In the "Protection" section of the CPU properties, select the "Allow access using PUT / GET" checkbox
