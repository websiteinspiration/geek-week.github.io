NestJSに会う

Node.js開発者コースの開始に先立って、記事の翻訳が準備されました。



現代の開発者は、Webサービスやその他のサーバーアプリケーションの作成に関して、多くの選択肢を持っています。ノードは非常に人気のある選択肢になっていますが、多くのプログラマーはJavaScriptよりも堅牢な言語を好みます。特に、C＃、C ++、Javaなどの最新のオブジェクト指向言語を使用する人はそうです。 TypeScriptがNodeJSにうまく適合している場合、NestJSフレームワークはそれをまったく新しいレベルに引き上げ、バックエンド開発者がコンポーネント、プロバイダー、モジュール、およびその他の有用な高レベルの抽象化を使用して耐久性のある高性能アプリケーションを作成するための最新のツールを提供します。

この記事では、NestJSでシンプルなAPIサーバーを作成して、スーパーマーケット製品のリストを作成、保存、取得するという基本的なアプリケーションシナリオを処理するプロセスを見ていきます。
プロジェクトのソースコードに慣れる必要がある場合は、こちらで見つけることができます。

プロジェクトの作成


Nestを使用するには、Node環境が必要です。まだ持っていない場合は、彼らのWebサイトにアクセスしてダウンロードしてください。

フレームワークのインストールは非常に簡単です：

$ npm i -g @nestjs/cli

このプロジェクトは、次のコマンドを実行した後にNest CLIを使用して作成されました。

$ nest new nest-js-example

このようなチームは、必要な設定ファイル、フォルダ構造、サーバーテンプレートを備えた完全に新しいNestプロジェクトを作成します。

アプリケーションのエントリポイント


サーバーを設定して起動する主なファイルはsrc/main.ts次のとおりです。

import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
	
    const app = await NestFactory.create(AppModule);
    await app.listen(3000);
}

bootstrap();

このファイルは、アプリケーションの作成に使用されるNestFactoryクラスとメインのAppModuleファイル（後で説明します）をインポートし、アプリケーションをダウンロードしてインスタンス化し、ポート3000でリッスンします。

アプリモジュール


アプリケーションコンポーネントが宣言されているファイルは、次のように呼び出されsrc/app.module.tsます。

import { Module } from '@nestjs/common';
import { ItemsController } from './items/items.controller';
import { ItemsService } from './items/items.service';

@Module({
    imports: [],
    controllers: [ ItemsController ],
    providers: [ ItemsService ],
})

export class AppModule {}

これは、前のファイル（main.ts）にインポートされたモジュールで他のコンポーネントがインポートおよび宣言されているファイルです。ネストCLIの新しいコンポーネントを作成するように指示するとき、必要に応じてツールが自動的にこのファイルを更新します。ここでは、アイテムのコントローラーとサービスがインポートされ、モジュールに追加されます。

アイテムコントローラー


次に検討するファイルは次のsrc/items/items.controller.tsとおりです。

import { Controller, Req, Get, Post, Body } from '@nestjs/common'
import { CreateItemDto } from './dto/create-item.dto'
import { ItemsService } from './items.service'
import { Item } from './items.interface'

@Controller('items')
export class ItemsController {

    constructor(private readonly itemsService: ItemsService) {}

    @Post()
    create(@Body() data: CreateItemDto): Object {
        return this.itemsService.create(data)
    }

    @Get()
    findAll(): Array<Item> {
        return this.itemsService.findAll()
    }
}

このファイルは、アイテムを作成し、以前に作成されたアイテムのリストを取得するためのコントローラーを定義します。いくつかの主要なコンポーネントがここにインポートされます：

  • CreateItemDto：ネットワーク経由でアイテムデータを送信する方法を定義するData-Transferオブジェクト（つまり、これはJSONデータ構造です）。
  • ItemsService：アイテムデータの操作または保存を処理するプロバイダー。
  • 項目：のための内部データ構造を定義するインターフェイスアイテム。

デコレータ@Controller('items')は、このクラスがREST / アイテムエンドポイントを提供することをフレームワークに伝え、ItemsControllerコンストラクタは、2つのHTTPメソッドを提供するために内部的に使用されるItemsServiceのインスタンスを取得します。

  • POST / items（JSONリクエストから新しいアイテムを作成します）;
  • GET / items（以前に作成されたアイテムのリストを取得）。

これら2つのメソッドに対する要求は、@Post()およびデコレータを使用して対応するHTTPメソッドにバインドされているcreateおよびFindAllメソッドによって処理されます@Get()。追加のメソッドも同様に、@Put()または@Delete()などの方法でデコレータによってサポートできます。

アイテムオブジェクトインターフェイス


次に、アイテムを格納するためのインターフェイスを定義する2つのファイルを扱います。1つはコンパイル時の型チェックなどの内部使用（Item）で、もう1つは着信JSONの予想される構造を決定するための外部インターフェイス（CreateItemDto）です。

export interface Item {
	
    name: string,
    description: string,
    price: number
}

export class CreateItemDto {

    @IsNotEmpty()
    readonly name: string;

    @IsNotEmpty()
    readonly description: string;

    @IsNotEmpty()
    readonly price: number;
}

Item インターフェースは、通常のストア製品の3つのプロパティ、名前、説明、価格を定義します。これにより、アイテムとは何か、アイテムのプロパティについては、アプリケーションアーキテクチャに混乱が生じません。CreateItemDto

クラスは、Itemプロパティを反映し、各プロパティを装飾して、REST APIエンドポイントからこれらのプロパティがすべて要求されるようにします。@IsNotEmpty()

両方のクラスのすべてのプロパティは厳密に型指定されています。これは、TypeScript（したがって名前）の主な利点の1つです。一見すると、これによりコンテキストの理解度が向上し、コード分析ツール（VSCodeのIntelliSenseなど）と正しく使用すると、開発時間が大幅に短縮されます。これは特に、数百または数千もの異なるクラスとインターフェースを持つ大規模なプロジェクトに当てはまります。無限の容量を持つ完璧な写真の記憶を持たない人（たとえば、私にとって）は、何千もの特定の詳細を覚えようとするよりもはるかに簡単です。

アイテムサービス


最新のものは、作成および受信するためのサービスですitems: items.service.dart。

import { Injectable } from '@nestjs/common'
import { Item } from './items.interface'

@Injectable()
export class ItemsService {

    private items: Array<Item> = []

    create(item: Item): Object {

        this.items.push(item) 
        return { id: this.items.length.toString() }
    }

    findAll(): Array<Item> {

        return this.items;
    }
}

ItemsService クラスは、サンプルプロジェクトのメモリ内データストアとして機能するItemオブジェクトの単純な配列を定義します。このリポジトリに対して書き込みと読み取りを行う2つの方法は次のとおりです。

  • 作成（アイテムをリストに保存し、そのIDを返します）;
  • findAll（以前に作成されたItemオブジェクトのリストを返します）。

テスト中


サーバーを起動するには、標準のnpm run startコマンドを使用します。アプリケーションが起動したら、CURL経由でHTTPリクエストを送信してテストできます。

$ curl -X POST localhost:3000/items -d '{"name":"trinket", "description":"whatever", "price": 42.0}'

このコマンドを実行すると、itemによって生成されたIDを持つJSON応答が返されます。作成済みのアイテムのリストを表示するには、次を使用します。

$ curl localhost:3000/items

上記の / アイテムへのGETリクエストは、すでにメモリ内にあるアイテムに関する情報を含むJSON応答を返します。答えは次のようになります。

[{"{\"name\":\"trinket\", \"description\":\"whatever\", \"price\": 42.0}":""}]

結論
