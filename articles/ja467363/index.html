Kerasでカスタム損失関数と学習品質メトリックを使用する

トレーニングサンプルでニューラルネットワークをトレーニングする場合、ニューラルネットワークの出力で、学習効率の2つの重要なパラメーター（エラーと予測精度）が計算されます。このために、損失関数と精度メトリックが使用されます。これらのメトリックは、タスク（分類または画像セグメンテーション、オブジェクト検出、回帰）によって異なります。Kerasでは、特定のタスクについて、独自の損失関数と精度メトリックを定義できます。このようなカスタム関数については、記事で説明します。猫の下で誰が気にかけてください。

Kerasのカスタム損失関数


たとえば、平均平均誤差（MAE）エラー関数を実装する必要があるとします。カスタムMAE損失関数は、次のように実装できます。

from keras import backend as K

def mae(y_true, y_pred):
    true_value = K.sum(y_true * K.arange(0, 100, dtype="float32"), axis=-1)
    pred_value = K.sum(y_pred * K.arange(0, 100, dtype="float32"), axis=-1)
    mae = K.mean(K.abs(true_value - pred_value))
return mae

ここでは、Kerasで定義された合計、範囲、平均、およびabs関数を使用します。

同様に、精度メトリックを定義できます。たとえば、2つのヒストグラムを比較するためにearth_movers_distanceメトリックを定義します。

from keras import backend as K

def earth_movers_distance(y_true, y_pred):
    cdf_true = K.cumsum(y_true, axis=-1)
    cdf_pred = K.cumsum(y_pred, axis=-1)
    emd = K.sqrt(K.mean(K.square(cdf_true - cdf_pred), axis=-1))
    return K.mean(emd)

maeおよびearth_movers_distanceメトリックを使用するには、対応する関数を別のモジュールからインポートし、モデルのコンパイル時に損失およびメトリックパラメーターに追加します。

from utils.metrics import mae, earth_movers_distance

loss = earth_movers_distance
model.compile(optimizer=optimizer, loss=loss, metrics=[mae, "accuracy"])

カスタム損失関数を使用したKerasモデルの読み込み


Kerasでモデルをトレーニングする場合、モデルの重みをh5ファイルに保存して、トレーニングされたモデルを予測段階で後で読み込むことができます。カスタム損失関数と品質メトリックを使用すると、問題が発生する可能性があります。load_weightsメソッドを使用してモデルのh5ファイルからトレーニング済みの重みを読み込むと、次のエラーが発生する可能性があります。

ValueError: Unknown loss function:earth_movers_distance

これはKerasの有名なバグです（githubの公式リポジトリに書いてあります）。

この問題を解決するには、カスタムの損失関数と品質メトリックをKerasに追加する必要があります。

from keras.utils.generic_utils import get_custom_objects
get_custom_objects().update({"earth_movers_distance": earth_movers_distance, "age_mae": age_mae})

それは今のところすべてです。皆さん、頑張ってくださいね！
