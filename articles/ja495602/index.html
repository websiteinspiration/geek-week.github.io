この記事では、Core Dataの機能をさらに明らかにしたいと思います。この記事は続きです。Core Dataは多くのリレーショナルデータベースの1つではないが、自尊心のあるIOS開発者の武器の不可欠な武器になることができる非常に強力なツールであるという考えが得られることを本当に期待しています。

さあ、始めましょう！

今日は、2つのNSManageObjectContextとNSFetchedResultsControllerの操作について見ていきます。

2つのNSManageObjectContextとなぜこれが必要なのですか？


アプリケーションがデータの受信に使用するAPIをアクティブに使用していて、このデータをどこかに保存したい場合は、Core Dataが最適です。これでどんな問題がありますか？ 1つ目は最も明白です。データ量が非常に多いため、データを保存しようとすると、アプリケーションがしばらくフリーズし、ユーザーエクスペリエンスに影響します。通常のGCDは独立しているため省略できません。ここで説明したコアデータスタックは同期的に動作し、NSManageObjectContextへのアクセスにはループの終了まで待機する必要があります。

しかし、ここでは、バックグラウンドスレッドで機能するプライベートNSManageObjectContextが役に立ちます。これを呼び出すには、まずNSPersistentContainerに連絡する必要があります。

初期化は次のようになります。

lazy var persistentContainer: NSPersistentContainer = {
    let container = NSPersistentContainer(name: "Preset")
    container.loadPersistentStores { (persistent, error) in
        if let error = error {
            fatalError("Error: " + error.localizedDescription)
        }
    }
    container.viewContext.mergePolicy = NSMergeByPropertyStoreTrumpMergePolicy
    container.viewContext.shouldDeleteInaccessibleFaults = true
    container.viewContext.automaticallyMergesChangesFromParent = true
    return container
}()

マージポリシー


ここでは、NSManageObjectContextのマージポリシーを指定します。ここでは、データの競合が発生した場合のCore Dataの動作を明示的に示します。

MergePolicyオプション：

  1. NSRollbackMergePolicy-競合が発生した場合、発生するまですべての変更を破棄します
  2. NSOverwriteMergePolicy-データに関係なく新しい値を保存します
  3. NSMergeByPropertyStoreTrumpMergePolicy-変更されたオブジェクトのプロパティをプロパティごとに保存します。この場合、保存されたデータが優先されます
  4. NSMergeByPropertyObjectTrumpMergePolicy-プロパティごとに変更されたオブジェクトのプロパティを保存します。この場合、新しいデータが優先されます

AutomaticallyMergesChangesFromParent-コンテキストがデータを自動的にマージするかどうかを示し、
次に新しいコンテキストを作成します。

let context = persistentContainer.viewContext
let context = persistentContainer.newBackgroundContext()

これで2つのNSManageObjectContextができました。1つ目はUIを操作するためのもので、メインスレッドで実行され、2つ目はバックグラウンドで作業するためのprivateQueueConcurrencyTypeを持っています。

データのダウンロードに使用します。

let object = NSEntityDescription.insertNewObject(forEntityName: "Name", into: context)
saveChanges(with: context)

ここでエンティティを作成し、それに必要なプロパティを割り当てることができます。その後、saveメソッドを呼び出すと、次のようになります。 

func saveChanges(with context: NSManagedObjectContext) {
        context.performAndWait {
            if context.hasChanges {
                do {
                    try context.save()
                } catch {
                    context.rollback()
                }
            }
            context.reset()
        }
    }

2つの方法から選択できます。

  • performAndWait-コンテキストスレッドで同期的にアクションを実行します
  • 実行-コンテキストストリームに対して非同期にアクションを実行します


NSFetchedResultsController


NSFetchedResultsController-特定のリクエストを実行し、ユーザーに必要なデータを表示するコントローラー。

lazy var fetchedResultsController: NSFetchedResultsController<Pack> = {
        let fetchRequest = NSFetchRequest<Pack>(entityName:"Pack")
        fetchRequest.fetchBatchSize = 20
        fetchRequest.sortDescriptors = [NSSortDescriptor(key: "name", ascending:true)]
        let controller = NSFetchedResultsController(fetchRequest: fetchRequest, managedObjectContext: context, sectionNameKeyPath: nil, cacheName: nil)

        controller.delegate = self
        do {
            try controller.performFetch()
        } catch {
            print(error.localizedDescription)
        }
        return controller
    }()

現時点では、データをテーブルに表示するNSFetchedResultsControllerがあります。データを更新するには、デリゲートメソッドを呼び出す必要があります。

extension ViewController: NSFetchedResultsControllerDelegate {
    func controller(_ controller: NSFetchedResultsController<NSFetchRequestResult>, didChange anObject: Any, at indexPath: IndexPath?, for type: NSFetchedResultsChangeType, newIndexPath: IndexPath?) {
       collectionView.reloadData()
    }
}

このデリゲートメソッドは、コンテキストに変更がある場合に機能します。この実装では、データをprivateContextに保存した後にこれが発生します。この時点で、デリゲートメソッドがトリガーされ、データが更新されます。
