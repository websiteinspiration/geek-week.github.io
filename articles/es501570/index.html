Aserrar el cuestionario web Meduza: una guía paso a paso para principiantes

Mi nombre es Egor, soy un desarrollador Full-stack en Leader-ID. En este artículo, quiero compartir una receta simple para crear un cuestionario bello y conveniente basado en la web similar a los realizados por Meduza. Él sabe cómo mostrar estadísticas después de responder preguntas individuales, calcular la puntuación total, emitir comentarios, cargar datos para su análisis y buscar resultados en redes sociales. Para lograr esto, elegí Django, DRF, Python y la base de datos PostgreSQL.



Todos los detalles están debajo del corte.

Después de una hora de mirar el ladrillo (sin embargo, una lección difícil ) , el primer resultado apareció en forma de modelos confeccionados, que se describieron en Dzhang después de diez minutos.

Si es un principiante, le aconsejo que siga el tutorial de Djnago , que describe cómo crear una encuesta paso a paso. Y después del tutorial DRF , para finalmente sumergirnos en el tema.

Entonces, en el proyecto, usé:

  • Django 3.0.3. Para el backend;
  • django-rest-framework. Para crear rest-api;
  • Pitón
  • PostgreSQL como una base de datos;

class Question(models.Model):
    title = models.CharField(max_length=4096)
    visible = models.BooleanField(default=False)
    max_points = models.FloatField()

    def __str__(self):
           return self.title

class Choice(models.Model):
    question = models.ForeignKey(Question, on_delete=models.DO_NOTHING)
    title = models.CharField(max_length=4096)
    points = models.FloatField()
    lock_other = models.BooleanField(default=False)

    def __str__(self):
        return self.title

class Answer(models.Model):
    user = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.DO_NOTHING)
    question = models.ForeignKey(Question, on_delete=models.DO_NOTHING)
    choice = models.ForeignKey(Choice, on_delete=models.DO_NOTHING)
    created = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.choice.title

from rest_framework import serializers
from .models import Answer, Question, Choice


class ChoiceSerializer(serializers.ModelSerializer):
    percent = serializers.SerializerMethodField()

    class Meta:
        model = Choice
        fields = ['pk', 'title', 'points', 'percent', 'lock_other', ]

    def get_percent(self, obj):
        total = Answer.objects.filter(question=obj.question).count()
        current = Answer.objects.filter(question=obj.question, choice=obj).count()
        if total != 0:
            return float(current * 100 / total)
        else:
            return float(0)


class QuestionSerializer(serializers.ModelSerializer):
    choices = ChoiceSerializer(many=True, source='choice_set', )

    class Meta:
        model = Question
        fields = ['pk', 'title', 'choices', 'max_points', ]


class AnswerSerializer(serializers.Serializer):
    answers = serializers.JSONField()

    def validate_answers(self, answers):
        if not answers:
            raise serializers.Validationerror("Answers must be not null.")
        return answers

    def save(self):
        answers = self.data['answers']
        user = self.context.user
        for question_id, in answers:  #     ,    
            question = Question.objects.get(pk=question_id)
            choices = answers[question_id]
            for choice_id in choices:
                choice = Choice.objects.get(pk=choice_id)
                Answer(user=user, question=question, choice=choice).save()
                user.is_answer = True
                user.save()


Luego escribimos dos vistas DRF en views.py, que dan todas las preguntas con opciones y aceptan todas las respuestas del usuario.


Texto del código aquí 
from .serializers import QuestionSerializer, AnswerSerializer
from rest_framework.permissions import IsAuthenticated
from rest_framework.generics import GenericAPIView
from rest_framework.response import Response
from .models import Question


class GetQuestion(GenericAPIView):
    permission_classes = (IsAuthenticated,)
    serializer_class = QuestionSerializer

    def get(self, request, format=None):
        questions = Question.objects.filter(visible=True, )
        last_point = QuestionSerializer(questions, many=True)
        return Response(last_point.data)


class QuestionAnswer(GenericAPIView):
    permission_classes = (IsAuthenticated,)
    serializer_class = AnswerSerializer

    def post(self, request, format=None):
        answer = AnswerSerializer(data=request.data, context=request)
        if answer.is_valid(raise_exception=True):
            answer.save()
            return Response({'result': 'OK'})

    )

    )
    list_filter = ('question',)


class AnswerAdmin(admin.ModelAdmin):
    list_display = (
        'user',
        'question',
        'choice',
    )
    list_filter = ('user',)


admin.site.register(Question, QuestionAdmin)
admin.site.register(Choice, ChoiceAdmin)
admin.site.register(Answer, AnswerAdmin)

Para calcular el puntaje total de las preguntas, agregamos al método una función de cálculo simple, de acuerdo con los puntos que se calculan. No publicaré el código de función, porque puede usarse para descifrar nuestro cuestionario. Ahora cada respuesta tiene su propio "peso".

Accedemos al panel de administración a través del navegador usando el enlace provisto en la consola ( http://127.0.0.1:8000 / admin por defecto), y creamos preguntas y respuestas a ellos, colocamos puntos.



Era importante para mí darles a nuestros socios listas de personas que completaron la encuesta y sus respuestas. Pero para esto, no es suficiente simplemente vincular las respuestas a las preguntas. Por lo tanto, agregué otra tabla: "Opciones". Entonces se formó una conexión entre las respuestas del usuario a las preguntas con varias opciones de respuesta. Esto nos permite cargar datos en la forma en que los socios pueden interpretarlos fácilmente.

En él eliminamos la lista de preguntas y respuestas, pasamos por cada elemento hasta el último. Dependiendo del tipo de pregunta, cambiamos el componente con su propia lógica y estilo. En consecuencia, cuando no quedan preguntas en la lista, enviamos el resultado al final y obtenemos una respuesta con el número de puntos. En esta etapa, ya tenemos un cuestionario listo que puede hacer todo lo necesario: hacer preguntas, recibir y recopilar respuestas, dar resultados en forma de puntos y comentarios.

Agregar bollos


En primer lugar, necesitaba cargar periódicamente los datos. Para esto, acabo de agregar el comando de administración.

En segundo lugar, sería bueno implementar el intercambio de resultados de la encuesta en las redes sociales. OKAY. Funcionalidad de corte que le permitirá compartir una imagen con puntos VKontakte y Facebook.

Generamos cien variantes de imágenes que reflejan puntos para VK y Facebook por separado (diferentes resoluciones). Ahora conectamos la transferencia del enlace a la imagen en el componente social de la parte frontal. Con VKontakte, todo resultó ser simple: pasamos el parámetro de imagen con la URL directa a la deseada. Pero tuve que jugar con Facebook. Resultó que no aceptan medios por API, y si transfirí una imagen o una imagen desde la URL de la imagen, se mostró un gran campo vacío en la publicación. Como resultó más tarde, toma una imagen del metainf (og: imagen) del sitio en sí, que compartió (pasamos el parámetro u en el enlace). Y ella, entre otras cosas, tuvo que ser cambiada dinámicamente. No quería hacer redirecciones innecesarias y una mecánica en la parte posterior, y decidí convertir SPA (aplicación de una sola página) a SSR (renderizado del lado del servidor) en la parte delantera,de modo que, según la solicitud, la url de la imagen con una puntuación en head-meta cambia antes de que se inicie JavaScript en el navegador. Afortunadamente, el marco Nuxt.js tomado como base permite que esto se haga simplemente cambiando de modo. Ahora queda por dibujar etiquetas solo para clientes y agregar la lógica para cambiar la cabeza de la presencia de una puntuación de consulta.



Además, en el servidor, era necesario iniciar el servicio de daemon para proporcionar las páginas generadas, y dejar las estadísticas también en nginx. Todo beneficio!



Revivimos el cuestionario


Para mantener el nivel de interés de los participantes en el proceso de completar la encuesta, agregué una visualización dinámica de estadísticas a cada pregunta individual. Después de responder la pregunta, el usuario ve cómo respondieron los demás. A veces no está claro para una persona por qué se le hacen estas preguntas. Por lo tanto, completé cada pregunta con explicaciones divertidas. Bueno, el truco más importante para revitalizar mi cuestionario fue realizado por los diseñadores de nuestra empresa.
