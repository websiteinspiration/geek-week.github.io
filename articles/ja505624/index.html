数十億レコードのデータフレームを数秒で処理する方法

Pythonのビッグデータ分析はルネサンスの時代です。 NumPyライブラリから始まりました。ちなみに、このライブラリは、この資料で説明するツールのコンポーネントの1つです。 2006年、ビッグデータ処理の話題は次第に勢いを増し、このプロセスはHadoopの登場とともに加速しました。その後、通常は単に「データフレーム」と呼ばれるDataFrameデータ構造を持つpandasライブラリが登場しました。 2014年にビッグデータが主流になり、同じ年にApache Sparkプラットフォームが登場しました。 2018年には、DaskライブラリとPythonでデータを分析するための他のツールがリリースされました。



毎月、私は本当に学びたいPythonのデータ分析のための新しいツールに出くわします。それらを研究するために1、2時間を費やすことで、長期的には、多くの時間を節約できます。さらに、あなたが興味を持っているテクノロジーの領域での新しい発展を監視することは重要です。おそらく、この記事はDaskライブラリに関するものだと思うかもしれません。しかし、これはそうではありません。今日は、最近発見された1つのPythonライブラリーについてお話します。これは、データ分析に携わっている人には知っておく価値があります。

Vaex Pythonライブラリ



Vaexは、データフレームの大規模なセットの視覚化と研究のために設計された（pandasデータフレームに類似した）データフレームの「遅延」処理を整理するために設計された高性能Pythonライブラリです。このライブラリは、調査中のデータの主要な統計的指標を計算できます。同時に、10億件のレコードの処理には、1秒程度の時間がかかる場合があります。ライブラリは、インタラクティブにビッグデータを探索するのに役立つ多くの視覚化ツールをサポートしています。

VaexとDaskの比較



VaexライブラリはDaskとは異なりますが、そこで使用されるデータ構造はDaskで使用されるDataFrameエンティティに似ています。それらはパンダのデータフレームに基づいて構築されています。これは、Dask がパンダからデータフレームの操作に関する特定の制限を継承することを意味します。たとえば、処理するデータをメモリに完全にロードする必要があるという要件。 Vaexの場合、これはそうではありません。

Vaexはデータフレームのコピーを作成しないため、このライブラリを使用すると、少量のRAMを搭載したコンピューターで大規模なデータフレームを処理できます。

VaexとDaskはどちらも遅延データ処理エンジンを使用しています。2つの主な違いは、Vaexが必要なときにフィールド値を計算することと、Daskでは値計算関数を明示的に使用する必要があることです。

Vaexの可能性を最大限に引き出すためには、データをHDF5またはApache Arrow形式で提示する必要があります。

Vaexをインストールする


Vaexのインストールは、他のPythonパッケージと同じくらい簡単です。

pip install vaex

Vaexの実験


100万行と1000列を含むpandasデータフレームを作成します。これにより、大きなデータファイルを取得できます。

import vaex
import pandas as pd
import numpy as np
n_rows = 1000000
n_cols = 1000
df = pd.DataFrame(np.random.randint(0, 100, size=(n_rows, n_cols)), columns=['col%d' % i for i in range(n_cols)])
df.head()

メモリの使用に関する情報

後でVaexを使用して読み取るために、このデータフレームをディスクに保存します。

file_path = 'big_file.csv'
df.to_csv(file_path, index=False)

Vaexを使用してこのCSVファイルを直接読み取る場合、それによるメリットはあまりありません。読み取り速度は、パンダによって提供される速度と同様になります。私のラップトップでは、Vaexとパンダの両方がこのデータを約85秒で読み取ります。

Vaexの機能を確認するには、CSVデータをHDF5（Hierarchical Data Formatバージョン5）に変換する必要があります。 Vaexにはこのような変換を実行する機能があり、使用可能なRAMの容量を超えるボリュームのデータを操作できます。これは、ソースデータがフラグメントに分割されているために実現されます。

メモリ不足が原因でパンダを使用して十分に大きなファイルを開くことができない場合、このファイルをHDF5形式に変換し、Vaexを使用して処理できます。

dv = vaex.from_csv(file_path, convert=True, chunk_size=5_000_000)

上記の関数は、HDF5ファイルを自動的に作成し、ディスクに保存します。

変数のタイプを確認しますdv。

type(dv)
# 
vaex.hdf5.dataset.Hdf5MemoryMapped

次に、Vaexを使用して7.5 GBのデータセットを読み取ります。今すぐ読む必要はありません。実際、既に変数で表されていますdv。ただし、次のコマンドを実行して、この操作の速度を調べるためにこれを実行します。

dv = vaex.open('big_file.csv.hdf5')

Vaexはこのコマンドを完了するのに1秒もかかりません。しかし、実際にはVaexは「遅延」データ処理のメカニズムを使用しているため、このファイルは読み取れません。

次に、列の値の合計を計算して、Vaexにファイルを読み取らせますcol1。

suma = dv.col1.sum()
suma
# array(49486599)

しかし、このチームは私を本当に驚かせました。Vaexの金額を計算するのに1秒もかかりませんでした。これはどのようにして可能ですか？そしておそらくこれはメモリマッピングが原因です。

データの視覚化


Vaexは、データの視覚化にも優れた速度を示しています。ライブラリには特別な関数があります：plot1d、plot2dおよびplot2d_contour。

dv.plot1d(dv.col2, figsize=(14, 7))

仮想列


Vaexは、データセットに新しい列を追加すると、仮想列を作成します。彼は記憶をとらない。そこに格納されている値は、「オンザフライ」で言うように計算されます。

dv['col1_plus_col2'] = dv.col1 + dv.col2
dv['col1_plus_col2']



新しい列

効果的なデータフィルタリング


Vaexは、データのフィルタリング時にデータフレームのコピーを作成しません。これは、メモリのより効率的な使用に貢献します。

dvv = dv[dv.col1 > 90]

データ集約


Vaexは、pandasとは少し異なる方法でデータを集計します。しかし、ここで最も重要なことは、Vaexではこの操作が非常に高速であることです。

比較結果を含む新しい列を作成しますdv.col1 >= 50。

dv['col1_50'] = dv.col1 >= 50

Vaexは、データをグループ化および集約する操作を1つのチームに結合します。次のコマンドは、データを列ごとにグループ化し、列col1_50の合計を計算しますcol3。

dv_group = dv.groupby(dv['col1_50'], agg=vaex.agg.sum(dv['col3']))
dv_group



チームの結果

データ接続


Vaexを使用すると、メモリにデータコピーを作成せずにデータを結合できます。これは、ライブラリの他の機能と同様に、メモリの経済的な使用に貢献します。join以下に示す関数は、pandasユーザーにはおなじみのようです。

dv_join = dv.join(dv_group, on=’col1_50')
