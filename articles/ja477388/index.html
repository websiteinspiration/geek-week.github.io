NILFS2-/ホーム向けの防弾ファイルシステム



ご存知のように、トラブルが発生した場合、必ず発生します。おそらく、誰もが新しい重要なファイルを誤って削除したり、テキストエディターのテキストを誤って選択して削除したりしたことがあります。

あなたがサイトのホストまたは所有者である場合、ユーザーアカウントまたはサイトのハッキングに遭遇した可能性があります。このような場合、年代を復元し、侵入の方法と攻撃者が使用した脆弱性を見つけることが重要です。

このような問題を解決するには、NILFS2ファイルシステムが最適です。

バージョン2.6.30以降、Linuxカーネルに存在しています。

このファイルシステムの特徴は、バージョン管理システムに似ていることです。いつでもシステムの状態をロールバックして、以前の状態を確認できます。

この機能を提供するために、cronスクリプトを構成したり、スナップショットを作成したりする必要はありません。NILFS2ファイルシステムは、これをすべて単独で行います。彼女は古いデータを上書きすることはなく、十分な空きディスク領域がある場合は常にディスクの新しい領域に書き込みます。コピーオンライトの原則に完全に準拠しています。

実際、ファイルを変更すると、ファイルシステムの新しいスナップショットが自動的に作成されるため、このファイルシステムをタイムマシンとして使用して、ファイルの状態を巻き戻すことができます。

物語


NILFS2は、日本電信電話株式会社の腸内で開発されました。実際には、国（管理株）であり、日本最大の通信会社です。具体的には、小西隆介率いるサイバースペース研究所。

それが具体的に開発された理由-不明ですが、「タイムマシン」の機能を備えたそのようなFSは、SMSや電子メールなどの全体像を再生するために特別なサービスをさらに掘り下げたいデータを格納するのに理想的であると想定できます。

NILFS2は、メールデータベース内のすべての削除された手紙を復元し、後でファイルを削除または変更することで偽装しようとする従業員の側を開くことができるため、潜在的に、内部セキュリティサービスにとって非常に貴重なツールでもあります。

NILFS2はどのようなものですか？


使用の観点から：バージョン管理システムSVNで。各FSチェックポイントは、削除、ファイルの内容の変更、アクセス権の変更など、ユーザーが気付かないうちに自動的に行われるコミットです。各コミットには、直線的に増加する数があります。

プログラマーの観点から：循環バッファー上。ファイルシステムは変更を保存し、それらを約8 MBに等しい部分に書き込みます（2048 * 4096、2048はブロック内の要素の数、4096はメモリページのサイズ）。ディスク全体がそのようなチャンクに分割されます。録音は順番に行われます。空き容量がなくなると、最も古い画像が削除され、チャンクが上書きされます。

NILFS2 /home


Unixライクなオペレーティングシステムでは、原則として、ユーザーデータが格納される/ホームフォルダがあります。さまざまなプログラムは、特定のユーザーに関連する設定をこのフォルダに保存します。

そして、もしユーザーでなければ、ほとんどの場合、誰が刈っていますか？したがって、彼らが言うように、神自身は/家でNILFS2を使用するように命じました。

さらに、SSDが広く分布しているため、CoWファイルシステムを使用する際の強いドローダウンについて心配する必要がなくなりました。

はい、ZFSとBTRFSで好きなだけFSスナップショットを作成できますが、スナップショット間で失われたファイルの変更が発生するリスクは常にあります。そして、写真はまだ管理する必要があります：古いものを削除してください。 NILFS2では、これらすべてが文字通り数秒ごとに自動的に行われます。

lvcreateを使用して論理ボリュームを作成しました（nvmeボリュームグループのシンシンプール）。後で簡単に拡張できるため、lvmボリュームに作成することをお勧めします。適切なバージョンの深さのために、NILFS2で50％の空きディスク領域を確保することをお勧めします。

lvcreate -V10G -T nvme/thin -n home

NILFS2でフォーマットします。

mkfs.nilfs2 -L nvme_home /dev/nvme/home

mkfs.nilfs2 (nilfs-utils 2.1.5)
Start writing file system initial data to the device
      Blocksize:4096  Device:/dev/nvme/home1  Device Size:10737418240
File system initialization succeeded !!

その後、現在/ホームからすべてのデータをコピーする必要があります。

これは、コンピューターを起動した直後、アカウントを入力する前に、rootユーザーから実行しました。ユーザーとしてログインした場合、一部のプログラムがユーザーのフォルダー/ home /ユーザーのソケットとファイルを開き、コピーが困難になります。ご存知のように、rootユーザーのホームフォルダーは通常パス/ rootにあるため、/ homeセクションでファイルは開かれません。

mkdir /mnt/newhome
mount -t nilfs2 /dev/nvme/home /mnt/newhome
cp -a /home/. /mnt/newhome

最後の行については、記事を参照してください。

次に、/ homeのファイルシステムがマウントされている/ etc / fstabを

/dev/disk/by-label/nvme_home /home nilfs2    noatime 0 0

オプション  で編集して、noatimeパフォーマンスが向上し、ファイルアクセスのたびにatimeが変更されないようにします。次に、再起動します。

データを回復します。


NILFSを使用すると、メインのFSブランチを操作しながら、必要なだけ古いスナップショットをマウントできます。ただし、読み取りモードのみ。

すべてがこのように配置されています。NILFS2が行う通常のチェックポイントはいつでも（ディスク領域がなくなるか、nilfs_cleanerdのルールに従って）自動的に削除されるため、マウントする前に、チェックポイントをスナップショットに変換するか、ロシア語で画像を修正する必要があります。

chcp ss _

その後、たとえば次のようにスナップショットをマウントできます。

mount -t nilfs2 -r -o cp=_ /dev/nvme/home /mnt/nilfs/_

次に、回復したファイルをスナップショットから/ホームにコピーします。
次に、画像から区別不可能のフラグを削除して、将来、自動ガベージコレクターが古いデータを削除できるようにします。

chcp cp _

NILFS2のユーティリティ


しかし、これは問題です。もちろん、ファイルシステムを作成し、そのサイズをオンラインで変更し、チェーンポイントのリストを表示し、作成および削除することができます。nilfs2-utilsパッケージは、最小限の紳士のセットを提供します。

NTTは資金を削減したため、ファイルの変更履歴を表示したり、スナップショット間で差分を作成したりできる高速で低レベルのユーティリティはありません。

私のn2uユーティリティ


この空白を埋めるために、特定のファイル/ディレクトリの変更履歴を表示できるn2uユーティリティを作成しました。

n2u log filename

出力は次のようになります。

          CHECKPOINT        DATE     TIME     TYPE          SIZE  MODE
             1787552  2019-11-24 22:08:00    first          7079    cp
             1792659  2019-11-25 23:09:05  changed          7081    cp

選択した実装方法に対して非常に迅速に機能します。2分割法を使用してファイル間の差異を検索し、さまざまなイメージのファイル/ディレクトリをすばやくマウントして比較します。または

キーを使用して、チェックポイントの範囲を設定できます。 特定のファイルまたはディレクトリのチェックポイント間の違いも確認できます。-cp CP1:CP2-cp {YEAR-MM-DD}:{YEAR-MM-DD}



n2u diff -r cp1:cp2 filename

変更の年代順全体を表示できます。特定のファイル/ディレクトリのチェックポイント間のすべての違い：

n2u blame [-r cp1:cp2] filename

このコマンドの日付範囲もサポートされています。
