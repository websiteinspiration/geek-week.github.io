Discussion of the voting system developed by DIT Moscow

Open source



Recently, the source code of the voting system , which is likely to be applied by electronic voting under the Constitution in Moscow and Nizhny Novgorod, has been posted publicly .







With this message, I would like to open the discussion to the developers what happened to the developers and how to live with it now.







Introduction and request for help



This development of the Department of Information Technology (DIT) of Moscow, the previous version of which is probably already familiar to some of the readers: as I understand it, it was it that was used in the elections of municipal deputies last year.







Due to the lack of time, having very briefly and superficially got acquainted with the submitted source texts, and also having passed the test vote, I ask the habr community for help in a deeper analysis of this software from the point of view of its technical, administrative, software, network and other vulnerabilities, which can distort the will of citizens who have entrusted their vote to this system.

How does it work



This is how I understand the system.







The web shell performs all interface functions, up to the formation of a newsletter - a separate page with code that accesses directly from the browser the server side of the blockchain subsystem through a special light client (part of exonium).







The bulletin code forms a transaction with a voice and sends it to the server for inclusion in the transaction log. The transaction is encrypted and signed, which is designed to protect the secrecy of the vote and the immutability of the vote.
