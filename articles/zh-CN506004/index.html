关于迪诺的初步思考

在Node.js开发人员课程启动之前，准备了本文的翻译。


在Codegram，我们喜欢尝试新技术，而Deno就在一周前问世！如果您想了解这个新的JavaScript运行时以及Codegram对它的看法，请阅读本文。

您可能已经听说过Deno，也许不是，但是现在Internet上充满了有关它的信息，这是有原因的！出现了一个新的JavaScript运行时，其中带有可爱的恐龙作为徽标（有些人认为徽标带有可悲的袜子，但没有...）。甚至在发布第一周之前，我们就发布过有关他的推文，但是，说实话，两年前，我们在看了Ryan Dahl的惊人表演后首次听说了Deno 。

在本文中，我们将简要介绍Deno，并分享我们的想法。让我们开始吧！

你好，迪诺！


Deno是像Node这样的JavaScript运行时。要使用它，必须首先安装deno CLI。为此，请按照deno.land上适用于您的操作系统的说明进行操作（我们的域名称赞！）。 Deno不是Node的分支，它是在Rust上从头开始创建的。我为JS程序员写了一篇关于Rust的文章，如果您想了解更多有关该语言的信息，可以阅读。

安装CLI后，您可以运行经典程序“ Hello，World！”。通过以下方式：

deno run https://deno.land/std/examples/welcome.ts

我们是开箱即用地运行TypeScript吗？是! TS是Deno的受欢迎的来宾，因此您无需任何其他操作即可在TS上运行代码。您也可以运行JS代码，但是我是TS的忠实拥护者，所以我是第一个谈论它的人。

请记住，Node代码与Deno不兼容。但是，现在他们正在努力确保兼容性，以便将来您将能够使用自己喜欢的NPM软件包。

值得一提的是，默认情况下，Deno是沙箱。也就是说，最初您无权访问网络或文件系统。如果您的程序需要访问权限，则需要明确允许它（即使用--allow-net授予网络特权）。

没有更多package.json或node_modules
谁没有在互联网上看到这个笑话？



尽管有些夸张，但这个笑话仍然有些道理。随着应用程序的增长，依赖项列表也随之增长，最终您将得到一个巨大的依赖关系树。问题是每个应用程序都有您正在使用的程序包@版本的副本。
Deno的主要目标之一是镜像浏览器。要将某些内容导入浏览器时，可以编写标签<script>并使用URL。 Deno的工作原理如下：

import { assertEquals } from "https://deno.land/std/testing/asserts.ts";
assertEquals("hello", "hello");
assertEquals("world", "world");
console.log("Asserted! ");

我同意，这看起来有些奇怪。由于我们没有任何package.json，因此无法指定外部依赖项，因此我们在需要它们时将其导入。首次启动时，Deno将下载文件并将其保存在缓存中。如果要再次下载软件包，可以指定--reload标志。

您可能对此主题有很多疑问（我也有），因此，我邀请您访问Deno手册中的FAQ页面，以获取有关该主题的更多信息。

浏览器API兼容的运行时


Deno的另一个重要目标是浏览器兼容性，因此它具有一组可从全局范围使用的功能，例如fetch或addEventListener。您也可以使用全局窗口对象（但是，到目前为止，我建议使用globalThis）。您可以在文档中了解有关全局函数的更多信息。

运行时还具有非Web标准的API 全局Deno。您可以将其用于低级操作，例如读取文件，打开TCP套接字等。

丰富的标准库


Deno开发人员已创建了一组标准的即用型模块。该库还托管在deno.land域中，因此您可以使用以下URL将任何模块导入到您的应用程序中：

import { v4 } from "https://deno.land/std/uuid/mod.ts";

console.log(v4.generate());

标准库包括许多有用的模块，但是有许多第三方模块，并且此列表每天都会更新。还有一个好消息：您可以导入公共URL上托管的任何模块，这意味着您可以轻松地从GitHub导入模块。Deno提供了URL重写服务，您可以使用它来使模块可用。您可以在此页面上了解有关此机制的更多信息。

内建工具


Deno CLI还包括用于执行一些常见任务的内置工具，例如，运行测试，甚至创建包！这是否意味着Deno可以取代玩笑，漂亮和Webpack？可能不是。Deno是一个相当新的解决方案，还没有如此广泛的生态系统，但是所有这些工具的存在仍然令人印象深刻，可以通过安装一个二进制文件来访问这些工具。

结论


跳上这个炒作浪潮！我对Deno印象深刻。经过两年的发展和漫长的成功之路，新的JavaScript运行时看起来很棒，我渴望开始尝试它。Node大约在11年前出现，并成为JavaScript历史上的新里程碑。他使语言变得更好，我认为我们已经达到了一个新的里程碑，现在是Deno的时候了！
