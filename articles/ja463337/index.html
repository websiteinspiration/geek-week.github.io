Immerを使用してReactアプリケーションのステータスを管理する

状態は、Reactアプリケーションからのデータのモニターを編成するために使用されます。ユーザーがアプリケーションを操作すると、状態が変化します。ユーザーがなんらかのアクションを実行する場合、状態を更新する必要があります。これは、ユーザーが画面に表示するものが形成される基礎となるデータのセットです。setStateメソッドを使用して、Reactアプリケーションの状態を更新します。 状態は直接更新されるべきではないので（Reactでは、状態は不変である必要があります）、状態の構造が複雑になるため、それらの操作は非常に簡単な作業になります。つまり、プログラマーがその状態でナビゲートし、アプリケーションで自分のデータを使用することが困難になります。





このような状況では、Immerライブラリを使用できます。Reactアプリケーションでの使用は、今日私たちが公開している資料に専念しています。

ReactアプリケーションでのImmerの使用の基本


Immerを使用すると、Reactアプリケーションの状態構造を簡略化できます。つまり、操作がより簡単になります。イマーは、いわゆる「ドラフト」の概念を使用しています。 「ドラフト」は、州のコピーとして理解できますが、州自体としては理解できません。

イマーは、CMD + Cキーを押して状態をコピーし、次にCMD + Vキーを使用して、コピーしたデータを、元の資料に影響を与えずに表示できる場所に挿入します。状態に含まれるデータの変更は「ドラフト」で行われ、その後、「ドラフト」に加えられた変更に基づいて、アプリケーションの現在の状態が更新されます。

アプリケーションの状態が次のようになっているとします。

this.state = {
  name: 'Kunle',
  age: 30,
  city: 'Lagos',
  country: 'Nigeria'
}

こちらがユーザーデータです。結局のところ、このユーザーは31歳の誕生日を祝っています。つまり、その年齢（プロパティage）を更新する必要があります。この問題を解決するためにイマーを使用する場合、この状態のコピーが最初に作成されます。

条件のコピーが作成され、宅配便に渡され、彼がこのコピーをKunleに配達したとしましょう。これは、状態のコピーが2つあることを意味します。 1つはアプリケーションの現在の状態で、もう1つはユーザーに転送された「ラフ」コピーです。 「ドラフト」を編集しているユーザーは、年齢を31歳に変更します。その後、変更されたドキュメントを宅配便業者が返し、「ドラフト」をアプリケーションに渡します。そこでは、ドキュメントの2つのバージョンの比較が実行され、「ドラフト」では何も変更されていないため、ユーザーの年齢に関する変更のみがアプリケーションの現在の状態に加えられます。

そのような作業計画は、状態の免除の概念に違反しません-現在の状態は直接更新されません。一般的に、Immerの使用は、免疫状態の使いやすさの向上に役立つと言えます。

const {produce} = immer

class App extends React.Component {
  state = {
    red: 'red', 
    yellow: 'black', 
    green: 'black',
    next: "yellow"
  }

  componentDidMount() {
    this.interval = setInterval(() => this.changeHandle(), 3000);
  }
  
  componentWillUnmount()  {
    clearInterval(this.interval);
  }

  handleRedLight = () => {
    this.setState(
      produce(draft => {
        draft.red = 'red';
        draft.yellow = 'black';
        draft.green = 'black';
        draft.next = 'yellow'
      })
    )
  }
  
  handleYellowLight = () => {
    this.setState(
      produce(draft => {
        draft.red = 'black';
        draft.yellow = 'yellow';
        draft.green = 'black';
        draft.next = 'green'
      })
    )
  }
  
  handleGreenLight = () => {
    this.setState(
      produce(draft => {
        draft.red = 'black';
        draft.yellow = 'black';
        draft.green = 'green';
        draft.next = 'red'
      })
    )
  }

  changeHandle = () => {
    if (this.state.next === 'yellow') {
      this.handleYellowLight()
    } else if (this.state.next === 'green') {
      this.handleGreenLight()
    } else {
      this.handleRedLight()
    }
    
  }

  render() {
    return (
      <div className="box">
        <div className="circle" style={{backgroundColor: this.state.red}}></div>
        <div className="circle" style={{backgroundColor: this.state.yellow}}></div>
        <div className="circle" style={{backgroundColor: this.state.green}}></div>
      </div>
  );
}
};

Produceイマーからインポートされた標準機能です。これを値としてメソッドに渡しsetState()ます。関数produceは、引数としてを受け入れる関数を受け入れますdraft。「ドラフト」状態を編集して、それを実際の状態にする必要のある形式にすることができるのは、この関数内です。

これらすべてが複雑すぎると思われる場合は、上記のコードと同じタスクを解決するコードを作成する別の方法を次に示します。まず、関数を作成します。

const handleLight = (state) => {
  return produce(state, (draft) => {
    draft.red = 'black';
    draft.yellow = 'black';
    draft.green = 'green';
    draft.next = 'red'
  });
}

produce引数としての 関数は、アプリケーションの現在の状態と引数を取る別の関数を実行しますdraft。次に、コンポーネントでこれをすべて活用しましょう。

handleGreenLight = () => {

  this.setState(nextState)

class App extends React.Component {
  constructor(props) {
      super(props)
      
      this.state = {
        item: "",
        price: 0,
        list: [
          { id: 1, name: "Cereals", price: 12 },
          { id: 2, name: "Rice", price: 10 }
        ]
      }
    }

    handleInputChange = e => {
      this.setState(
      produce(draft => {
        draft[event.target.name] = event.target.value
      }))
    }

    handleSubmit = (e) => {
      e.preventDefault()
      const newItem = {
        id: uuid.v4(),
        name: this.state.name,
        price: this.state.price
      }
      this.setState(
        produce(draft => {
          draft.list = draft.list.concat(newItem)
        })
      )
    };

  render() {
    return (
      <React.Fragment>
        <section className="section">
          <div className="box">
            <form onSubmit={this.handleSubmit}>
              <h2>Create your shopping list</h2>
              <div>
                <input
                  type="text"
                  placeholder="Item's Name"
                  onChange={this.handleInputChange}
                  name="name"
                  className="input"
                  />
              </div>
              <div>
                <input
                  type="number"
                  placeholder="Item's Price"
                  onChange={this.handleInputChange}
                  name="price"
                  className="input"
                  />
              </div>
              <button className="button is-grey">Submit</button>
            </form>
          </div>
          
          <div className="box">
            {
              this.state.list.length ? (
                this.state.list.map(item => (
                  <ul>
                    <li key={item.id}>
                      <p>{item.name}</p>
                      <p>${item.price}</p>
                    </li>
                    <hr />
                  </ul>
                ))
              ) : <p>Your list is empty</p>
            }
          </div>
        </section>
      </React.Fragment>
    )
  }
}

ReactDOM.render(
  <App />,
  document.getElementById('root')
);

関数でhandleSubmitは、最初にユーザーが入力したデータに基づいてオブジェクトを作成します。オブジェクトへの参照は定数に書き込まれますnewItem。新しいアプリケーション状態を形成するには、メソッドを使用し.concat()ます。このメソッドは配列で呼び出され、元の配列の要素と新しい要素を含む新しい配列を返します。新しい配列がに書き込まれdraft.listます。その後、Immerはアプリケーションの状態を更新できます。

calculateAmount = (list) => {
  let total = 0;
    for (let i = 0; i < list.length; i++) {

    produce(draft => {

フックイマー


Use-immerは、開発者がReactアプリケーションの状態を制御できるようにするフックです。クラシックカウンターアプリケーションをベースに実装して、このフックがどのように機能するかを見てみましょう。

import React from "react";
import {useImmer} from "use-immer";

const Counter = () => {
  const [count, updateCounter] = useImmer({
    value: 0
  });

  function increment() {
    updateCounter(draft => {
      draft.value = draft.value +1;
    });
  }

  return (
    <div>
      <h1>
        Counter {count.value}
      </h1>
      <br />
      <button onClick={increment}>Increment</button>
    </div>
  );
}

export default Counter;

この関数はuseImmer、useStateメソッドによく似ています。この関数は、状態と、状態を更新する関数を返します。コンポーネントが最初にロードされたとき、状態のコンテンツ（この場合、このプロパティcount）は渡された値に対応しますuseImmer。返された関数を使用して状態を更新するincrementと、stateプロパティの値をインクリメントする関数を作成できますcount。

そして、これがuseReducerを連想させるImmerのフックを使用するコードです。

import React, { useRef } from "react";
import {useImmerReducer } from "use-immer";
import uuidv4 from "uuid/v4"
const initialState = [];
const reducer = (draft, action) => {
  switch (action.type) {
    case "ADD_ITEM":
      draft.push(action.item);
      return;
    case "CLEAR_LIST":
      return initialState;
    default:
      return draft;
  }
}
const Todo = () => {
  const inputEl = useRef(null);
  const [state, dispatch] = useImmerReducer(reducer, initialState);
  
  const handleSubmit = (e) => {
    e.preventDefault()
    const newItem = {
      id: uuidv4(),
      text: inputEl.current.value
    };
    dispatch({ type: "ADD_ITEM", item: newItem });
    inputEl.current.value = "";
    inputEl.current.focus();
  }
  
  const handleClear = () => {
    dispatch({ type: 'CLEAR_LIST' })
  }
  
  return (
    <div className='App'>
      <header className='App-header'>
        <ul>
          {state.map(todo => {
            return <li key={todo.id}>{todo.text}</li>;
          })}
        </ul>
        <form onSubmit={handleSubmit}>
          <input type='text' ref={inputEl} />
          <button
            type='submit'
          >
            Add Todo
          </button>
        </form>
        <button
          onClick={handleClear}
        >
          Clear Todos
        </button>
      </header>
    </div>
  );
}
export default Todo;

この関数useImmerReducerは、レデューサー関数と初期状態を受け入れます。状態と機能を返しますdispatch。その後、その中の要素を表示するための状態をバイパスできます。アクションはdispatch、新しい項目がTo Doリストに追加されたとき、およびリストがクリアされたときに、関数を使用して送信されます。送信されるアクションには、特定のアクションを処理するために何を実行する必要があるかについて、レデューサー関数で決定が行われることに基づいてタイプが割り当てられます。
