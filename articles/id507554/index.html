Klien Web Sip berdasarkan JsSIP + FreePBX

FreePBX memiliki klien Web Sip. Saya bertanya-tanya bagaimana cara kerjanya. Mulai belajar teknologi WebRTC, saya menemukan 3 perpustakaan - ini adalah JsSIP, garpu SIP.js, dan juga sipml5.



JsSIP dan SIP.js tidak dapat dimulai pertama kali, tidak ada cukup pengalaman. Tapi sipml5 adalah yang paling mudah digunakan. Setelah mengunduh versi pengujiannya, ternyata melakukan panggilan pertama.



Semua 3 perpustakaan operasional. Masing-masing memiliki kekuatan dan kelemahannya sendiri. Setelah menguji semua 3 perpustakaan, saya memilih JsSIP.



Pro dan kontra utama yang saya temui:



1. JsSIP



Pro:



  • ringan;
  • didukung secara aktif oleh pengembang.


Minus:



  • pada tahap awal, lebih sulit untuk memulai;
  • Saya harus mengedit sumber untuk bekerja dengan asterisk.


2. Sip.js



Pro:



  • ringan;
  • Penekanan ditempatkan pada bekerja dengan tanda bintang.


Minus:



  • Tidak dapat memulai suara untuk media awal (suara tidak terhubung untuk pesan pra-pesan operator).


3. Sipml5



Pro:



  • relatif sederhana untuk pemula.


Minus:



  • kode yang rumit dan membingungkan, tidak ada keinginan untuk memahaminya.


Latihan



Klien web hanya bekerja pada halaman dengan dukungan enkripsi. Kita perlu membuat sertifikat yang ditandatangani sendiri. Bagaimana ini dilakukan, ada artikel siap pakai di Internet.



Firefox memungkinkan Anda bekerja dengan sertifikat yang ditandatangani sendiri. Untuk bekerja di Chrome, Anda harus melakukan sertifikat root terlebih dahulu, dan berdasarkan itu, sudah menandatangani sertifikat agar ponsel berfungsi. Sertifikat root harus diimpor ke toko sertifikat.



Mengaktifkan dukungan untuk klien sip web pada FreePBX



Admin -> Manajemen Pengguna -> Tab "Grup" -> Tab "UCP" -> Tab "Telepon"

Untuk properti "Aktifkan Telepon", pilih "Ya".



Jika Anda memiliki nomor ekstensi di FreePBX, katakan 200, maka ketika Anda mengaktifkan klien web sip, nomor tambahan dengan awalan 99 akan muncul, mis. 99200.



Pada browser klien, setelah Anda memberikan izin untuk menggunakan sertifikat yang ditandatangani sendiri, Anda masih perlu memberikan izin untuk sertifikat tanda bintang. Untuk melakukan ini, ketik browser 



https://192.168.1.70:8089/ws


Alamat IP harus ditentukan di mana tanda bintang sedang berjalan.



Mengedit sumber JsSIP



Dalam kode sumber perpustakaan JsSIP, Anda perlu mengganti transport = ws dengan transport = wss. Tanpa hasil edit ini, klien tidak akan berfungsi. Anda tidak perlu mengedit apa pun di Sip.js, mereka sudah menyediakan untuk titik ini di sana.

ua.on('disconnected', function(e) { /*   */ });

Ringkasan



Selama 2 tahun terakhir menggunakan klien Web sip, ada masalah yang muncul saat memperbarui browser.



Sebagai contoh, di Firefox versi 63, saya harus mengubah protokol SDP agar sesi tidak dikalahkan. Browser berhenti berkomunikasi tanpa parameter a = mid: 0. Di versi terbaru itu tidak lagi relevan, kesalahan telah diperbaiki, di versi selanjutnya tidak ada masalah seperti itu.



Dalam versi 65 dari Firefox dengan ref. suara lawan bicara mulai menghilang saat panggilan berlangsung, tidak ada masalah seperti itu di LAN. Masalahnya hanya ketika memanggil melalui gateway. Di chrome, masalah seperti itu tidak diamati, semuanya bekerja secara stabil.



Sebagian besar pengguna bekerja di Firefox, dan untuk alasan ini perlu untuk menghapus negosiasi ulang protokol SDP di perpustakaan. Masalah dengan hilangnya suara setelah jawaban berhenti mengganggu.
