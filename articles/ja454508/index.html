デスクトップアプリケーションの.NET Coreへの移植

.NET Frameworkから.NET Coreへのデスクトップアプリケーションの移植についてコミュニティと協力し始めたので、「2つのキャンプ」があることに気づきました。1人の代表者は、アプリケーションを.NET Coreに移植するための非常にシンプルで短いリストの手順を望んでいます、他の代表は多くの初期情報を備えたより原則的なアプローチを好みます。「スイスアーミーナイフ」のようなドキュメントを作成する代わりに、「キャンプ」ごとに1つずつ、2つのブログ投稿を公開します。

  • この投稿は単純なケースです。簡単な指示と小さなアプリケーションに焦点を当て、アプリケーションを.NET Coreに移植する最も簡単な方法を検討します。
  • 後で、より複雑なケースについて別の投稿を公開します。WCFやサードパーティのUIパッケージに依存するWPFアプリケーションなど、重要なアプリケーションに焦点を当てます。

読むのではなくビデオを見たい方は、以下のビデオをご覧ください。

エラー訂正


次のようなエラーが発生した場合

The type or namespace <some name> could not be found

または

The name <some name> does not exist in the current context

移植性レポートが緑色で表示されている場合は、適切なライブラリを備えたNuGetパッケージを追加するだけで簡単に修正できます。ライブラリが不足しているNuGetパッケージが見つからない場合は、Microsoft.Windows.Compatibilityに連絡してみてください。このパッケージは、.NET Frameworkから約21Kの.NET APIを追加します。
