symfonyでスキンを整理する方法

これは、Symfonyの例を使用して、Twigでページをブランディングするためのスキンの使用を整理する方法についての短いメモです。このソリューションはSymfonyに関連付けられていません。類推すると、Twigを使用して任意のプロジェクトにスキンを実装できます。


オンラインストア、オンライン映画館、イベントのポスター、テレビ番組のカタログなどがあります。ある晴れた日には、ユーザーを引き付けて何らかのアクションの売り上げを増やすために、カタログページをブランド化するタスクを取得します。カタログ内のすべての製品がエンジンと同等である場合、これをどのように行うのですか？


最も簡単な解決策は、カタログから製品IDをハードコードすることです。テンプレートに条件を追加し、bodyタグに追加のCSSクラスを課すことができます。それに従って、ページを一般的なスタイルでスタイル設定できます。


{% block body_class -%}
    {{ parent () }} product-{{ product.id }}
{%- endblock %}

body.product-12345 {
   # custom style
}

特にflexを使用する場合、スタイルは多くのことができますが、スタイルは全能ではありません。場合によっては、スタイルの可能性ではページをブランディングするのに十分ではなく、ページのHTMLマークアップ（レイアウト）を変更する必要があります。これは、スタイルとの類推によって行われます。


{% if product.id == 12345 %}
    {# custom code #}
{% else %}
    {# original code #}
{% endif %}

public function show(Product $product, Twig $twig): Response
{
    $template = $twig->resolveTemplate([
        sprintf('product/skin/%s/show.html.twig', $product->skin),
        'product/skin/default/show.html.twig',
    ]);
    $content = $template->render([
        'product' => $product,
    ]);

    return new Response($content);
}
