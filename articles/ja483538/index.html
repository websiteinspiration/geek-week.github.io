FMCを介して10 "ER-TFT101-1ディスプレイをSTM32F429に接続する

全てにおいて良い日。この記事では、FMC（フレキシブルメモリコントローラー）モジュールを使用して、16ビット8080パラレルインターフェイスを介してSTM32F429L DiscoveryボードへのTFTディスプレイER-TFT101-1（10インチ、RA8876ドライバー）の接続を分析します。




ディスプレイ組立について


EastRisingのER-TFT101-1は、解像度が1024x600の10インチTFTマトリックスとRA8876ドライバーを搭載したボードのアセンブリです。ドライバーが搭載されたマザーボードには、必要なすべての電源が供給され、16メガバイトのSD-RAMメモリ（16ビットバス、最大周波数166 MHz、最大ボリューム64 MB）、標準のmicroSDカードスロットがあります。外部フォントを備えたEEPROMの下、およびそれらをプログラミングするための出力コネクタを備えた画像のフラッシュメモリの下には、空のフットプリントがあります。また、抵抗式または容量式のタッチパネルをオプションでアセンブリに取り付けることもできます。

ボードには、120 MHzの最大動作周波数を持つトップエンドのRAiO RA8876ドライバーがあり、必要に応じて、それ自体が制御マイクロコントローラーとして機能できます。小さなプログラム（12命令のみ）を作成して、外部フラッシュメモリに配置できます。ディスプレイが開始すると、このプログラムは最初から実行を開始し、基本的にすべての制御オプションを外部インターフェースを介して複製します。



RA8876には独自のRAMがないため、外部SD-RAMメモリを使用します。 DMAを使用してフラッシュメモリから画像を読み取り、フレームバッファにロードすることができ、非常に柔軟な構成になっています。ドライバは、標準の18ビット幅のRGBインターフェースを介してマトリックス自体に接続されます。赤チャネルあたり6ビット、緑あたり6ビット、青チャネルあたり6ビットのみが使用されます。各チャネルの下位2ビットは使用されず、理論上は262,144色になります。

RA8876のDMAモジュールはSTMのDMA2Dと非常によく似ています-メモリの長方形のセクションをある場所から別の場所にコピーでき、色変換、透明度、その他のチップを備えています。

RA8876には、組み込みの英語と中国語のフォント（8x16、12x24、16x32ピクセル）があり、ディスプレイ（回転、スケールなど）を柔軟に設定でき、ハードウェアボタン（スタンドアロンおよび使用するだけでなく）、長押しや短押しなどの一連の設定で、ボタンを押して同時に複数のボタンを押すことでディスプレイを起動します。

ポップアップウィンドウとメニューを表示するためのピクチャーインピクチャー機能（透過性サポートなし）があります。

ドライバー自体は、四角形、円、曲線、楕円形、三角形、丸みを帯びた四角形などのグラフィックプリミティブを描画できます。ちなみに、RA8875ではRA8876で、三角形を塗りつぶすという小さなバグがあり、各ドライバーには独自のバグがあります。しかし、RAiOはそれを高い鐘楼から気にしませんでした...私は彼らに手紙を書こうとしたので、彼らは答えさえしませんでした。このようなプリミティブを描画すると、低速のマイクロコントローラーでも美しいグラフィックを作成できます。

RA8876は、外部の世界では、8080/6800 8/16ビット、3/4ワイヤーSPIおよびI2Cインターフェースを介して通信します。さらに、ドライバチップ自体がSPIおよびI2Cマスターとして機能できます。RA8876には、バックライトの柔軟な制御に使用できる2つのPWM出力があります。最大SPI CLK周波数は、120 MHzのドライバー周波数で66 MHzで宣言されており、理論的には、フルスクリーン更新（1024 x 600 x 16ビット）で毎秒6フレームを提供します。この接続は私がテストしたものであり、画面にビデオを表示しなければ、生きる権利があることを示しました。

今回のケースでは、FMCモジュール（フレキシブルメモリコントローラー）を介して、幅80ビットの8080プロトコルを使用するディスプレイをSTM32F429ZIT6に接続します。これにより、マイクロコントローラーの画面充填速度と負荷を軽減できます。

データシートのXnWAITについては、以下をご覧ください。
連続データ書き込み速度は、表示更新速度を決定します。XnWaitを採用せずに待機状態を挿入する場合、サイクル間間隔はシステムクロック周期の5より大きくなければなりません。仕様を超えると、xnwaitメカニズムを使用しない場合、データが失われたり、機能が失敗したりする可能性があります。
文字通り、8080プロトコルのデューティサイクルの間に、ユーザーがXnWAITメカニズムを使用してRA8876のリリースを待機しない場合、5つのRA8876システムユニットの遅延を挿入する必要があります。このピンをもう一度使用します。実際には、5サイクルの遅延を挿入しようとしましたが、機能しませんでした。

FMCユニットの完全なアドレスバスの代わりに、ピンA16を1つだけ使用します。

  1. データピン（D0〜D15）を、プッシュプール、最大速度、サスペンダーなしなどの代替機能＃12として構成します。
  2. ピンXnWAIT、XnWR、XnRD、XA0、およびXnCSは、プラスへのリフトを伴うプッシュプル（PULL UP）などの代替機能＃12として構成されます。
  3. XnRSTをサスペンダーなしの通常のGPIOとして構成します（ボード自体にあります）。
  4. XnINTRは、プラスへの上昇を伴う入力用のGPIOとして構成されます。

また、PWMで制御せずにバックライトを100％に接続しました。これを行うには、ディスプレイアセンブリコネクタのピン＃14をVDDに接続します。

ピンの構成コードは提供していません。私は自分の構成ライブラリーを使用していますが、GPIO構成自体は、ハブや他のソースで既に100回噛まれています。

FMC設定


各バンクの3つのバンクは、FMCモジュールのバンクのセットアップを担当します（NORフラッシュ/ PSRAM / SRAM / ROM / LDC 1）。これらは、FMC_BCRx、FMC_BTRx、およびFMC_BWTRxです。 STM32F429 MK定義では、FMC_BCRxおよびFMC_BTRxレジスタは、8つの要素を持つFMC_BTCRと呼ばれる1つの共通配列に結合されています。 FMC_BWTRxは、7つの要素を持つFMC_BWTRアレイに結合されますが、4つ必要です。理由を聞かないでください...

FMC_BCRxには基本設定が含まれ、FMC_BTRxには一般的なタイミングが含まれ、FMC_BWTRxには、デバイスが必要とする場合、読み取り用の個別のタイミングが含まれます。

STM32F429とRA8876の相互作用のタイミング図とタイミング。

ディスプレイの初期化


ディスプレイの操作は、メモリの特定の領域の読み取りまたは書き込みになります。残りの作業はFMCが処理します。作業を簡略化するために、2つの定義を定義します。

#define LCD_DATA 0x60020000
#define LCD_REG  0x60000000

次に、低レベル関数について説明します。

void LCD_CmdWrite (unsigned char cmd)
{
	*(unsigned short *)(LCD_REG) = cmd;
};
void LCD_DataWrite (unsigned short data)
{
	*(unsigned short *)(LCD_DATA)= data;
};
unsigned char LCD_StatusRead(void) 
{	
	unsigned short data = *(unsigned short *)(LCD_REG);
	return data;
};
unsigned char LCD_DataRead(void) 
{
	unsigned short data = * (unsigned short *)(LCD_DATA);
	return (unsigned char)data;
};
void LCD_RegisterWrite(unsigned char cmd, unsigned char data)
{
	*(unsigned short *)(LCD_REG)  = cmd;
	*(unsigned short *)(LCD_DATA) = data;
};
unsigned char LCD_RegisterRead (unsigned char cmd)
{
   volatile unsigned char data = 0;
   LCD_CmdWrite (cmd);
   data = LCD_DataRead ();
   return data;
};

各ページ（PAGEx_START_ADDR）は、SDRAMの開始アドレスです。16メガバイトのメモリでは、13のフルレイヤーを1228800バイト（1024 * 600 * 2）のサイズで配置できます。
Frame_Buffer_Start_Address関数は、フレームバッファー（現在表示されているもの）の開始メモリー領域を設定します。

Canvas_Window_Start_Address関数は、キャンバスの初期メモリ領域を設定します。さらに、キャンバスがフレームバッファよりも大きい場合があるため、画面上で画像をスクロールできます。たとえば、プラットフォームゲームの場合、13312 x 600ピクセルの長いキャンバスを作成し、水平方向にスクロールして、フレームバッファーを水平方向に移動できます。

グラフィック出力をSTM32のLTDMと比較すると、すべてがそれほどバラバラではありません。一度に、ドライバー自体は最後の（バッファー）レイヤーを1つだけ表示でき、LTDCはすぐに2つを表示し、それらを混合して、このプロセスに参加する必要はありません。

操作コードの説明（操作コード）：

0000：MKを使用してROPでメモリに書き込みます。
0001：MKを使用してROPなしでメモリを読み取ります。
0010：ROPを使用してメモリブロックを順方向にコピーします。
0011：ROPを使用してメモリブロックを逆方向にコピーしています。
0100：MKを使用してROPなしで（透過性ありで）メモリに書き込みます。
0101：ROPなしでメモリブロック（透過性あり）を順方向にコピー（移動）します。
0110：ROPを使用してパターンを入力します。
0111：テンプレートにクロマキーを入力します。
1000：カラー拡張
1001：透明度のあるカラー拡張
1010：アルファブレンディングでメモリブロックを順方向に移動する
1011：MKを使用してアルファブレンディングでメモリに書き込む。
1100：メモリ領域を単色で塗りつぶします。
1101：予約済み
1110：予約済み
1111：予約済み
