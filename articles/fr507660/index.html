Surveillance via UptimeRobot

Tout site doit être surveillé; il n'y a aucune ressource dont le propriétaire préférerait ne pas connaître les temps d'arrêt. C'est drôle que de nombreux développeurs ne surveillent pas leurs projets préférés, et les pigistes ne se soucient presque jamais des vérifications d'accessibilité, bien que cela ne soit généralement pas plus difficile que de connecter certaines analyses et peut économiser beaucoup de nerfs.



Il existe plus d'une dizaine de services de surveillance de sites, la plupart d'entre eux sont entièrement payés ou fortement réduits dans la version gratuite. En dehors d'eux est UptimeRobot , qui fournit gratuitement d'énormes fonctionnalités et vous permet de configurer de manière flexible un abonnement payant à vos besoins.



Dans la plupart des cas, UptimeRobot envoie des requêtes HTTP à l'adresse spécifiée et détermine la disponibilité par les en-têtes de réponse (2xx - tout va bien, 4xx / 5xx - il déclenche des requêtes répétées d'autres points pendant 30 secondes, si la réponse est à nouveau négative, les temps d'arrêt sont comptés).



Est libre



  • 50 moniteurs avec un taux de rafraîchissement de 5 minutes
  • Demandes HTTP (S) vers toutes les adresses
  • Rechercher un mot-clé sur une page
  • Ping (requêtes ICMP)
  • Vérification du port
  • Alertes par courrier, webhooks, messages sur Twitter, télégramme, Slack et autres services
  • Statistiques sur les temps de réponse
  • Journaux détaillés pendant deux mois
  • La possibilité de publier des statistiques sur une page publique ( exemple )
  • API ouverte


Payé



  • Jusqu'à 10 000 moniteurs avec un intervalle de rafraîchissement d'une minute
  • Suivez l'expiration du certificat SSL et surveillez les problèmes HTTPS
  • Une variété d'alertes logiques (avec l'installation d'intervalles et plus)
  • Personnalisation des requêtes HTTP (y compris les en-têtes et les statuts)
  • Surveillance du rythme cardiaque (surveillance du serveur pour NAT)
  • Alertes SMS / appels (jusqu'à 900 par mois)
  • Journaux disponibles dans deux ans


Comme vous pouvez le voir, il y a encore plus de nishtyaks gratuits au service que ceux achetés, et c'est probablement ce qui a apporté la popularité et la reconnaissance d'UptimeRobot. Bien que, après plusieurs remaniements, il soit toujours blâmé pour la terrible interface.







Mots clés



Si vous utilisez n'importe quel composant avec un rendu conditionnel (par exemple, une plaque sur les travaux techniques), vous pouvez utiliser sa recherche par mots clés pour suivre son état. En général, cette chose a de nombreux cas d'utilisation intéressants: comme le temps de réponse est enregistré pour chaque demande, vous pouvez mesurer le rendu de différentes parties de l'application «de côté» - les demandes sont envoyées de différentes parties du monde et peuvent donner une image complètement différente. Vous pouvez configurer des alertes pour l'apparition de contenu inapproprié sur la page et vice versa, vérifier la présence du texte souhaité. API



UptimeRobot fournit une API XML / JSON / JSON-P simple qui duplique complètement les fonctionnalités de la version Web. Une description détaillée des méthodes et des paramètres peut être trouvée sur la page API , en bref - vous pouvez lire les statistiques et les journaux, créer / supprimer / modifier / réinitialiser les moniteurs et gérer les utilisateurs.

Battement de coeur



Les moniteurs Heartbeat sont disponibles pour surveiller les serveurs qui ne sont pas directement accessibles. Lors de sa création, un lien unique est généré sur lequel le service UptimeRobot écoute les demandes entrantes périodiques d'un serveur distant. Si les demandes ont cessé d'arriver, les temps d'arrêt sont comptés.



Que peut-on faire maintenant



Un compte gratuit repose sur 50 moniteurs, ce qui est généralement suffisant pour couvrir les sites et projets personnels. Si vous n'avez pas encore connecté de surveillance gratuite partout, c'est le moment de le faire. En 5 minutes et sans aucune interférence avec le code. Et si vous avez besoin d'une intégration plus approfondie de la surveillance dans le projet, étudiez l'API. N'oubliez pas de chercher d'abord des solutions prêtes à l'emploi pour votre pile, probablement en vrac.
