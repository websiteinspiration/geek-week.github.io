GradleアクションとGithubアクションを使用してJavaプロジェクトをSonatype Mavenセントラルリポジトリに公開する

この記事では、Gradleコレクターを使用して、Sonatype MavenセントラルリポジトリのGithubアクションを通じて、Javaアーティファクトを最初から公開するプロセスについて詳しく説明します。


通常のチュートリアルが1か所にないため、この記事を書くことにしました。すべての情報は、完全に新鮮ではありませんが、さまざまなソースから断片的に収集する必要がありました。誰も気にしない、猫へようこそ。


Sonatypeでリポジトリを作成する


最初のステップは、Sonatype Maven Centralでリポジトリーを作成することです。これを行うには、ここに移動し、登録して新しいタスクを作成し、リポジトリを作成するように依頼します。プロジェクトのGroupId、プロジェクトへのプロジェクトURLリンク、およびプロジェクトが存在するバージョン管理システムへのSCM urlリンクをドライブします。ここでのGroupIdは、com.example、com.example.domain、com.example.testsupportの形式である必要があります。また、github.com / yourusername-> io.github.yourusername へのリンクにすることもできます。いずれの場合も、このドメインまたはプロファイルの所有権を確認する必要があります。 githubプロファイルを指定した場合、希望する名前でパブリックリポジトリを作成するように求められます。


確認後しばらくすると、GroupIdが作成され、次のステップであるGradle構成に進むことができます。


Gradleの構成


この記事の執筆時点では、アーティファクトの公開に役立つGradleのプラグインは見つかりませんでした。これは私が見つけた唯一のプラグインですが、作者は彼のさらなるサポートを拒否しました。そのため、すべてを自分で行うことにしました。難しいことではないからです。


最初に調べることは、Sonatypeの公開要件です。それらは次のとおりです。


  • ソースコードとJavaDocの存在。ファイル-sources.jarと-javadoc.jarファイルが存在している必要があります。ドキュメントに記載されているように、ソースコードまたはドキュメントを提供することができない場合は、ダミーを作成する-sources.jarか-javadoc.jar、簡単なREADMEを使用してテストに合格できます。

def customizePom(pom) {
    pom.withXml {
        def root = asNode()

        root.dependencies.removeAll { dep ->
            dep.scope == "test"
        }

        root.children().last() + {
            resolveStrategy = DELEGATE_FIRST

            description 'Some description of artifact'
            name 'Artifct name'

                name 'com.github.login'

            }

name: publish

on:
  push:
    tags:
      - 'v*'

  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v1
      - name: Set up JDK 8
        uses: actions/setup-java@v1
        with:
          java-version: 8

      - name: Prepare to publish
        run: |
          echo '${{secrets.GPG_KEY_CONTENTS}}' | base64 -d > publish_key.gpg
          gpg --quiet --batch --yes --decrypt --passphrase="${{secrets.SECRET_PASSPHRASE}}" \
          --output secret.gpg publish_key.gpg
          echo "::set-env name=RELEASE_VERSION::${GITHUB_REF:11}"

      - name: Publish with Gradle
        uses: eskatos/gradle-command-action@v1
        with:
          gradle-version: current
          arguments: test publish -Psigning.secretKeyRingFile=secret.gpg -Psigning.keyId=${{secrets.SIGNING_KEYID}} -Psigning.password=${{secrets.SIGNING_PASSWORD}} -PsonatypeUsername=${{secrets.SONATYPE_USERNAME}} -PsonatypePassword=${{secrets.SONATYPE_PASSWORD}}
