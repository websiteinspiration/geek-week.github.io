航空運賃を予測するには？

みなさん、こんにちは！

これは私が小さくて快適なサービスをどのように作るかについての3番目の記事で、理論的には旅行の計画に役立つはずです。この記事では、Clickhouse、Catboost、および1TB *のデータを使用して航空運賃を予測する方法について説明します。



それはなんのためですか？


cheapster.travel の主な機能の 1つは、複雑なルートの柔軟な組み合わせです（以前の記事で詳しく説明しています）。「all-with-all」を組み合わせるために、アグリゲーターキャッシュが使用されます。この場合、めったに検索されないチケットが常にあるわけではなく、複雑なルートを構築するのに非常に欠けています。それら。そこに複雑なルートのベースとなるホットチケット（安い）ですが、「通常の」チケットの1〜2セグメントでは十分ではありません（通常の価格で、最も人気のある方向ではありません）。航空運賃を予測できるモデルを構築する必要に私を導いたのはこの問題でした。

タスクの形式化


  • 直行便のチケットを予測できる必要があります（往復のみ）
  • これを定期的に予測してデータベースに保存できる必要があります（単純なシナリオ）
  • 「オンザフライ」で予測できる必要がある（複雑なシナリオ）
  • これはすべて非常に限られたハードウェアで行われるため、大量のデータを使用した最小限の操作

どうやってするの？


まず、モデルをトレーニングします。データセットを準備し、列内のフィーチャの最大数を選択し、それをtsvにアップロードし、DataFrame /プールにロードし、分析し、パラメーターを選択します...停止、データが多すぎてメモリに収まりません、-次のエラーをキャッチします。

MemoryError: Unable to allocate array with shape (38, 288224989) and data type float64
OSError: [Errno 12] Cannot allocate memory

この制限を回避するには、小さな部分を繰り返し学習する必要があり、次のようになります。

結果はRMSE〜100のモデルでした-一般に、私はそのような結果に満足しているでしょうが、予測の少しの分析と「正規化」の後（負の値と履歴の最小値/最大値と大きく異なる値が、履歴価格の対応する境界にもたらされます） 。その後、目標の指標は約80です。これは、私の経験では、航空券の価格設定にはロジックや常識がほとんどないという事実を考慮に入れています。

すばらしい、モデルがあります-今がそれを使用する時です。まず最初に、KXモデルを追加します。これは簡単な設定で行われます：

構成
<models>
    <model>
        <!-- Model type. Now catboost only. -->
        <type>catboost</type>
        <!-- Model name. -->
        <name>price</name>
        <!-- Path to trained model. -->
        <path>/opt/models/price_iter_model_2.bin</path>
        <!-- Update interval. -->
        <lifetime>0</lifetime>
    </model>
</models>

通常のSQLを使用した「オンザフライ」予測の場合：

select origin, destination, date,
         modelEvaluate('price', *)  predicted_price -- ,   
from log.history

+--------+-------------+------------+-----------------+
| origin | destination | date       | predicted_price |
+--------+-------------+------------+-----------------+
| VKO    | DEB         | 2020-03-20 | 3234.43244      |
+--------+-------------+------------+-----------------+
--* ,

そのようなアプローチが選択されたという事実を置き換えたいのですが、それは実装がより簡単だからというだけではありません-まだプラスがあります：

  • KHの外部にデータをアップロードする必要はありません（これは、アイロンの負荷がより速く、コストがかからないことを意味します）
  • etlプロセスを実行する必要はありません（簡単=信頼性が高い）

APIとフロントエンドを少し修正し、待望の予測を取得。

これらの予測は、「航空券の価格履歴」セクションにもよく適合します。この



機能は、cheapster.travel / historyで利用できます（モバイルでは曲がって開き、大画面のみになります）。
