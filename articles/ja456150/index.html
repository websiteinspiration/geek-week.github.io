ささいな喜び＃4：ラドン-数で測定されたコード品質

エンジニアは測定と数値が非常に好きです。したがって、コードの品質などの重要なものを数値形式で測定しようとしていることは当然のことです。



プログラムのテキストを評価するための測定基準は、プロジェクトの膨大な数のコード行から、それほど明確ではない「保守性インデックス」まで、数多く考案されてきました。あらゆる種類のメトリックでコードをコーティングする既存のすべての方法の詳細は、この記事に記載されています。


もちろん、Pythonの世界には、コードの品質を評価する独自の機能があります。それはラドンと呼ばれています。それは同じPythonで書かれており、プチファイルでのみ動作します。


私たちは自分自身にそれを置きます


pip install radon

コードを含むフォルダーに移動し、測定を開始します。


生の統計


ソースの行数のバナー数。また、コードを直接含む行数とコメント行数も含まれます。あまり有益な指標ではありませんが、それ以上の計算には必要です。


radon raw ./

それに応じて、プロジェクト内のファイルのリストと各ファイルの統計情報が表示されなくなります。
