開発者のための楽しい練習

人は1000日間初心者のままです。彼は10、000日の練習の後に真実を見つけます。

これはかなり記事を書いている大山ますたつからの引用です。クールな開発者になりたい場合は、努力してください。これが全体の秘密です。キーボードで何時間も過ごし、練習することを恐れないでください。その後、開発者として成長します。

ここでは、開発に役立つ7つのプロジェクトを示します。テクノロジースタックを自由に選択してください-心の欲望を何でも使用してください。

（以前のトレーニングタスクリスト：1）8つのトレーニングプロジェクト 2）トレーニングするプロジェクトの別のリスト）

プロジェクト1：パックマン




パックマンのバージョンをビルドします。これは、ゲームの開発方法を理解し、基本を理解するのに最適な方法です。JavaScriptフレームワーク、React、またはVueを使用します。

あなたは学びます：

  • アイテムの動き
  • どのキーを押すかを決定する方法
  • 衝突の瞬間を判断する方法
  • あなたはそこで停止してゴーストコントロールを追加することはできません

このプロジェクトの例は、 GitHub リポジトリにあります。

「主人は初心者よりも多くの間違いを犯します-試み」

発行サポートは、Vivaldiドキュメントリポジトリを開発および診断する会社であるEdison です。

プロジェクト2：ユーザー管理




GitHub リポジトリプロジェクト

ユーザー管理用のCRUDタイプのアプリケーションを作成すると、開発の基本がわかります。これは、初心者の開発者に特に役立ちます。

あなたは学びます：

  • ルーティングとは何ですか？
  • データ入力フォームを処理し、ユーザーが入力した内容を確認する方法
  • データベースの操作方法-アクションの作成、読み取り、更新、削除

プロジェクト3：自分の場所の天気を確認する



GitHub リポジトリー内のプロジェクト

アプリケーションを作成する場合は、気象検出アプリケーションから始めます。このプロジェクトは、Swiftを使用して完了することができます。

アプリケーションの作成経験を積むだけでなく、次のことも学びます。

  • APIを使用する方法
  • ジオロケーションの使い方
  • テキスト入力を追加して、アプリケーションをより動的にします。ユーザーは自分の場所を入力して、特定の場所の天気を確認できます。

APIが必要になります。OpenWeather APIを使用して、気象データを取得します。OpenWeather APIの詳細については、こちらをご覧ください。

チャットウィンドウを作成するには、ソケットを使い始めるための理想的な方法です。技術スタックの選択は膨大です。たとえば、Node.jsは優れています。

プロジェクト5：GitLab CI




ソース

継続的インテグレーション（CI）を初めて使用する場合は、GitLab CIを試してみてください。いくつかの環境をセットアップし、いくつかのテストを実行してみてください。これはそれほど難しいプロジェクトではありませんが、そのおかげで多くのことを学ぶことができると思います。現在、多くの開発チームがCIを使用しています。使えるようになると便利です。

あなたは学びます：

  • GitLab CIとは何ですか？
  • .gitlab-ci.ymlGitLabユーザーに何をすべきかを指示する設定方法
  • 他の環境にデプロイする方法

プロジェクト6：Site Analyzer




Webサイトのセマンティクスを分析し、ランキングを作成するスクレーパーを作成します。たとえば、画像に欠落しているaltタグがないかどうかを確認できます。または、ページにSEOメタタグがあるかどうかを確認します。スクレーパーは、ユーザーインターフェイスなしで作成できます。

あなたは学びます：

  • スクレーパーはどのように機能しますか？
  • DOMセレクターの作成方法
  • アルゴリズムの書き方
  • そこで停止したくない場合は、ユーザーインターフェイスを作成します。確認した各Webサイトについて報告することもできます。

ソース
