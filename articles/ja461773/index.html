Airtest IDE-モバイルゲームの自動化をテストする新しい方法？

今日は、Airtestと呼ばれる自動テスト用のかなり新しいオープンソースツールについてお話ししたいと思います。将来的には、このツールキットの個々の要素とそれらの使用方法についての詳細なストーリーを備えた記事をさらにいくつか作成し、今、あなたにそれを紹介し、それについての一般的な考えを与えることを目標としています。


これは3つの記事の最初です。第二の部分は、説明の画像認識（画像認識）するためのフレームワーク - AirTestを、第3および最後の部分がに捧げられるポコ - コードを書き込むことによって、UIオートメーションフレームワーク。


AirTest IDEはNetEaseによって開発され、ゲームなどの「自動化が難しい」アプリケーション用に設計されています。実際、これらは開発者の主な焦点ですが、他のアプリケーションでのAirTestの使用を停止するわけではありません。


AirTest IDEは、Windows、MacOS X、Linuxで動作し、AirTestとPocoの2つのフレームワークが含まれています。この開発は、Android Game Developer Summit 2018でXin Liuによって発表されました。テストの記述には、Python 2.7およびPython 3.3以降が使用されます。多数のデバイス、オープンソースコード、無料のアプリケーションですぐにテストを実行するためのサポート、テストの簡単な記述などの重要なこと-これらは開発者によって強調されている利点です。アプリケーションはWindowsとMacOSの両方で使用でき、Windows、iOS、AndroidのOSからアプリケーションを自動化できます。


GitHub のAirTestプロジェクトには、Airtest、Poco、iOS-Tangent、multi-device-runnerの 4つのプロジェクトが含まれています。
また、NetEaseがAirLabと呼ばれる自動テスト用の物理デバイスファームを提供していることにも言及する価値があります。
