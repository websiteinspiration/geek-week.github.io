Zabbix、Ansible、およびNapalmでCiscoスイッチポートのロックを解除

こんにちは。これは、2つの記事からなるシリーズの2番目の部分です。では最初の部分、我々はZabbixのキャッチportSecurityのはしごをスイッチからが、ここで我々は反対の問題を解決すると言うことができます-私たちは、のフロントエンドでのマウスクリックでスイッチポートのロックを外すのZabbix。


たまたまこの問題は2つの異なるツールと数か月の差で2回解決されました。最初は、Ansibleが使用されましたが、これでうまく対処できました。しかし、ある時点でそれは（再び）壊れ、同じ問題が狭い円で広く知られているNapalmネットワークライブラリを使用した単純なPythonによって解決されました。


これが必要な理由から始めましょう。


多くの場合、スイッチポートのロックは誰かの悪意のあるアクティビティではなく、予期されるイベントです。ほとんどの場合、コンピューターやIP電話がある部屋から別の部屋に移動すると、ポートがブロックされます。ただし、これは通常、制御されたプロセスであり、テクニカルサポートによって提供されます。そして、ブロックしたポートのロックを解除するためのテクニカルサポートツールを提供するとよいでしょう。


Zabbixで外部スクリプトを実行するための組み込み機能を使用します。つまり、問題の確認時にスクリプトを起動します（謝辞）。したがって、テクニカルサポートオフィサーが自分の行動で港を封鎖した場合、問題を確認するだけで、自分の力で港の封鎖を解除できます。つまり、[ Ack ]をクリックして確認フォームに記入するだけです（フォームは空白のままでもかまいませんが、もちろん、そのことについては伝えません）。


さらに、Zabbixは確認時にスクリプトを呼び出します。このスクリプトは、特定のポートの特定のスイッチについて、スタックされているすべてのMACアドレスをクリアします。また、可能な場合は、スイッチ全体で指定されたMACアドレスを消去します。

        commands:

    - name: run a command that clears mac-address
      ios_command:
        host: "{{ ansible_host }}"
        username: "{{ user }}"
        password: "{{ pass }}" 
        commands: 
          - clear port-security sticky address {{ mac | hwaddr('cisco') }}
      when: mac is defined
