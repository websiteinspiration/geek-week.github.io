大人のおもちゃについて話すようにYandex.Aliceに教えた方法

Yandex.AlisaとセックスショップをSpring Bootを通じて統合します。

音声アシスタントの話題が勢いを増しているので、私は親密な商品の最大の卸売業者とYandex.Alisaを統合することによってこの傾向に加わることに決めました。

アイデア：サプライヤーはオンラインストアの所有者の個人アカウントを持っており、アリスの音声チャネルを通じてサプライヤーから分析データを提供する必要があります。

統合は次の2つの部分で構成されています。

  • 商品のサプライヤーのWebサービスとREST APIの統合。
  • Yandex.Aliceとユーザー要求を処理するためのWebサービスの統合。

商品プロバイダーのWebサービス統合とREST API


サービスは、ユーザーが正確に何を要求したかを理解し、サプライヤーのサーバーにこのデータを要求する必要があります。Spring Bootでサービスを作成することにしました。

そこで、サプライヤーからのリクエストとリクエストされたデータを整理しました。しかし、アリスは厳格な女の子なので、3秒後でなければ答えを受け入れません。また、ユーザーは過去2年間の自分の店の比較説明を取得したいと思うかもしれません。この問題を解決するために、ehCacheライブラリを介してキャッシュを有効にしましたユーザーデータをロードするように設定されています。新しいニュアンスに直面しました：アリスはほとんど個人化されたユーザーデータを提供しません。つまり、受信リクエストにメールは表示されません。デバイスの一意の識別子を表示するuserIdのみがあります。ユーザーを特定する鍵となりました。

鍵はそこにあり、キャッシュはそこにあり、情報をいつロードするかを理解する必要があります。ユーザーはすぐにはデータを要求しませんが、最初にスキルを起動します（これはアリスで使用される用語です。以下で詳しく説明します）。サーバーの場合、これは空のタグを含むリクエストのように見えますCommand。この時点で、クライアントデータのダウンロードを開始できます。その後、最初のユーザーリクエストが送信されるまでに、すべてのデータがすでにサービスにアップロードされています。

完全なテストと製品でのさらなる作業のために、アプリケーションをHerokuのクラウドにデプロイすることにしました。これを行うには、Heroku Webサイトにアクセスし、登録して、heroku cliをダウンロードしてインストールします。次に、コンソールを開いてと入力しheroku loginます。プロジェクトcd〜/ myappのあるディレクトリに移動して実行しheroku createます。

統合を構成し、サービスを作成したら、テストできます。

[テスト]タブに移動します。



ここでは、スキルのパフォーマンスを確認できます。便宜上、機能ボタンへのクイックアクセスをスキルに追加しました。

制限があります。開始フレーズの正確さについてスキルをチェックできず、音声入力/出力をチェックできません。フレーズに問題がない場合は、スキルの公開後にのみ、回答の正しい発音がチェックされます。
すべてのテストの後で、スキルはモデレートのために送信できます。これを行うには、「一般情報」タブに移動し、「管理」をクリックします。



モデレート後、このボタンの代わりに[公開]ボタンが表示されます。公開後、スキルはYandexの公式カタログに表示されます。

機能と制限


現時点では、スキルはアリス自身とは多少異なります（私は「スキルを起動する」と言わなければなりません）。そのため、アリスを本格的な音声アシスタントとして認識することは困難です。スキルコンテキストを形成し、アリスに直接質問するのは素晴らしいことです。デバイスIDであるuserIdを除いて、ユーザーに関する情報を取得する方法はありません。したがって、各デバイスで再認証が必要になるため、単一のユーザーセッションを保存することが難しくなります。また、スキルはプッシュメッセージを送信できないため、リマインダースキルを作成することはできません。

一方、スキルは、家電製品など、音声制御を実装するための優れたプラットフォームになる可能性があります。また、スキルは、トレーニングや実験の実施、注文、価格などの情報の明確化に役立ちます。一般に、ユーザーが正確で短い質問をしたり、短いコマンドを出したりできる場所。
