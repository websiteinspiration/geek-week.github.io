挑発的なブラウザが行動ファジングでクラッシュする



この記事では、ファジングを使用してFirefoxでいくつかのクラッシュを見つける方法を説明します。通常、ファジングの目的はメモリの破損を示すエラーを見つけることですが、私の目標は異なります。予期しないブラウザの反応を検出したいのです。これらは、通常とは異なるタグを開いたり閉じたりする文字、またはJavaScriptパーサーによって無視される可能性のある文字です。このような予期しない反応は、セキュリティフィルターをバイパスし、JavaScriptサンドボックスを回避して、XSS攻撃を行うためによく使用されます。

私が話したい最初のエラーは、HTMLコメントを別の方法で閉じる方法です。 HTML仕様を読むと、コメントを->または-！>で閉じることができることがわかります。しかし、どのように別の方法で行うのですか？ファジングを開始する素晴らしい質問です！答えを出すコードを生成するだけです。

2008年にShazzerを組み立てたときファジングブラウザーの動作では、ページあたりの方向が約10,000に制限されていました。今日、2019年にはすべてがより速く機能するため、ファズをより頻繁に行うことができます。また、DOMを使用すると、現在のドキュメントに各方向を追加する必要がなくなるため、ファジングが高速化されます。異なる結果が得られる可能性があるため、これは信頼性の低いアプローチであることは注目に値します。実際、DOMがhrefなどの属性引数でnull値を許可しているが、HTMLパーサーでは許可していない場合があります。これらは依然として重大ではないエラーですが、HTMLパーサーが何を行うかを完全に把握するための結果を常に信頼できるとは限りません。ただし、ほとんどの場合は機能し、サーバー側からのHTML出力よりもはるかに高速です。

最初のステップはすでに行われています-「HTMLコメントを閉じることができる文字は何ですか？」という質問があります。回答するには、HTMLコメントを閉じる既存の文字と、不明なファズ文字を使用する必要があります。次のステップは、適切なファジングソフトウェアを使用することです。私の場合、ハックバーツールを使用していますが、ローカルWebサーバーを使用しても同じ結果が得られます。このツールのアイデアは、入力フィールドに入力を入力し、タグを少し変換して、出力を処理することです。変換するものがないので、コードを出力フィールドに直接配置できます。したがって、テキスト出力領域をクリックして、あいまいな文字を格納するための配列と、HTMLをテストするためのdiv要素を作成します。

log = [];
div=document.createElement('div');

次に、1,000,000を超えるUnicode文字、より正確には0x10ffffをファジングする必要があります。必要なのは単純なforループだけです。

for(i=0;i<=0x10ffff;i++){

次に、各文字に対して作成したdiv要素を再利用します。この例では、後の位置をテストしています！、文字が後に入力されるようにします！..次に、img要素を使用して、ファジングが成功したかどうかを確認します。この要素が存在する場合、HTMLコメントは閉じられており、興味深い文字がいくつかあります！

div.innerHTML = '<!-- --!'+String.fromCodePoint(i)+'><img>-->';

最後に、querySelectorを使用して、imgが存在するかどうかを確認し、文字をログに追加します。次に、ifステートメントとforループを閉じます。最後に、左側の入力フィールドに結果を入力します。

if(div.querySelector('img')){
 log.push(i);
 }
}
input.value=log

ここでのフルバージョンのコードが。 FirefoxでURLを開き、出力フィールドに入力文字を入力し、「JSを実行」ボタンをクリックして文字をファズする必要があります。ファジングが完了すると、入力フィールドに数字が表示されます。数字は、成功した文字コードに対応しています。執筆時点では、Firefox（バージョン67）では、！の後の改行文字-\ nおよび\ r-を使用してコメントを閉じることができます。これは、Firefoxの将来のバージョンで修正されると通知されました。つまり、ファジングの最後の段階はペイロードを構築することです。これは非常に簡単です。文字コードを文字に置き換え、XSSペイロードを追加する必要があります。

<!-- --!
><img src=1 onerror=alert(1)> -->

Hackvertorを再度使用して、上記を出力フィールドに貼り付け、[HTMLのテスト]をクリックすることにより、その動作をテストできます。 Firefox（バージョン67）では、最終コメントの一部として改行が許可されているため、警告ウィンドウが表示されます。

そのため、Firefox HTMLパーサーで重大ではないエラーを見つけることができました。別のものを見つけましょう！新しい質問が必要です：「どの文字がHTMLコメントを開くことができますか？」。既存のHTMLコメントを超える代わりに、HTMLコメントを使用して既存のHTML属性を超えます。私はきっと、HTMLコメントを<！ -。よろしいですか？同じコードを再度使用しますが、今回はinnerHTMLの目的を変更して、コメントが開いているかどうかを確認します。



したがって、最初のハイフンの後に、あいまいな記号が表示されます。文字がHTMLの開始コメントを正常に作成すると、div要素にコメントし、title属性が終了します。今回は、「Execute JS」を実行すると、Firefox（バージョン67）で「0.45」という2つの結果が得られます。ハイフンなのでコード45が必要ですが、0はNULL文字です！これは、Firefoxがシーケンス<！-NULL-をオープンコメントとして解釈することを意味します。ある種のゲーム！ （私は、ブラウザープロバイダーがより多くの動作ファジングを必要としていると思います=））このテストケースを完了するには、方向を作成する必要があります。同じことをしましょう。ここでも、String.fromCodePoint関数をNULLに置き換え、XSSシンボルを方向に置き換えます。

document.body.innerHTML = '<!-\x00- ><div title="--><img src=1 onerror=alert(1)>"></div>';

HTMLではなくJavaScriptに切り替えましょう。私はすべてのブラウザをテストしました、そしてMozillaに申し訳ありませんが、Firefoxは再びある種のゲームを作っています。jinmo123 ツイートからのファジングが興味深い新しいES6機能を使用して、括弧なしで関数を呼び出すという事実に触発されました。ファジングのために思いついた質問は、「inステートメントまたはinstanceofステートメントの後にどの文字を使用できるか？」でした。次に、Hackvertorでコードを再度作成します。コードは同様のパターンに従いますが、今回はDOMを使用しません。まず、配列とforループを作成します。

次に、innerHTMLの代わりにevalを使用して、値をファジングします。まず、無効な文字が原因の例外を検出するために、try catchブロックを使用する必要があります。
