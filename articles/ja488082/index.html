ステートマシン（IAsyncStateMachine）

非同期/待機を使用する場合


一般に、非同期/待機が適切なソリューションである2つのシナリオがあります。

  • I / O関連の作業：コードは、データベースからのデータ、ファイルの読み取り、Webサービスの呼び出しなど、何かを期待します。この場合、Task Parallel Libraryではなく、Async / Awaitを使用する必要があります。
  • CPU関連の作業：コードは複雑な計算を実行します。この場合、Async / Awaitを使用する必要がありますが、Task.Runを使用して別のスレッドで作業を開始する必要があります。Task Parallel Libraryの使用も検討してください。

ライブラリー・コードを作成する場合、以前のコンテキストに戻る必要はほとんどありません。 Task.ConfigureAwait（false）を使用すると、コードは以前の場所から再開を試みなくなり、代わりに、可能であれば、タスクを完了したスレッドでコードが終了するため、コンテキストの切り替えが回避されます。これにより、パフォーマンスがわずかに向上し、デッドロックを回避できます。

非同期タスク進捗レポート


非同期メソッドのかなり一般的な使用例は、バックグラウンドで動作し、他のタスクのためにユーザーインターフェイススレッドを解放し、応答性を維持することです。このシナリオでは、ユーザーがプロセスの進行状況を監視し、操作を操作できるように、進行状況をユーザーインターフェイスに報告することができます。

この一般的な問題を解決するために、.NETはIProgress <T インターフェイス>を提供します。これは、非同期タスクによって呼び出されるReport <T メソッドを提供>し、呼び出し元に進行状況を報告します。このインターフェイスは、非同期メソッドのパラメーターとして受け入れられます。呼び出し元は、このインターフェイスを実装するオブジェクトを提供する必要があります。

.NETは、Progress <Tを提供>します。これは、同期コンテキストの保存と復元に関連するすべての低レベルのロジックを処理するため、実際に推奨されるIProgress <Tのデフォルトの実装です>。 Progress <T >は、Action <T イベントとコールバックも提供します>。どちらもタスクが進行状況を報告するときに呼び出されます。

IProgress <T >とProgress <Tを一緒に使用>すると、進行状況情報をバックグラウンドタスクからユーザーインターフェイススレッドに簡単に転送できます。

それに注意してください<Tを>これは、intなどの単純な値、または完了率、現在の操作の文字列の説明、ETAなどのコンテキスト進行情報を提供するオブジェクトにすることができます。
進捗状況を報告する頻度を検討してください。実行している操作によっては、コードレポートが1秒あたり数回進行状況を報告する場合があり、ユーザーインターフェイスの応答性が低下する可能性があります。このようなシナリオでは、進行状況をより大きな間隔で報告することをお勧めします。この記事の

非同期タスクをキャンセルする


バックグラウンドタスクのもう1つの一般的な使用例は、実行をキャンセルする機能です。.NETはクラスCancellationTokenを提供します。非同期メソッドは、CancellationTokenオブジェクトを受け取ります。このオブジェクトは、発呼者コードと非同期メソッドによって共有され、キャンセルを通知するメカニズムを提供します。

最も一般的なケースでは、キャンセルは次のように行われます。

  1. 呼び出し元は、CancellationTokenSourceオブジェクトを作成します。
  2. 呼び出し元は、キャンセルされた非同期APIを呼び出し、CancellationTokenSource（CancellationTokenSource.Token）からCancellationTokenを渡します。
  3. 呼び出し元は、CancellationTokenSource（CancellationTokenSource.Cancel（））オブジェクトを使用してキャンセルを要求します。
  4. タスクはキャンセルを確認し、通常はCancellationToken.ThrowIfCancellationRequestedメソッドを使用して自分自身をキャンセルします。

このメカニズムが機能するためには、定期的に（つまり、コードの反復ごとに、またはロジックの自然なブレークポイントで）要求されたキャンセルをチェックするコードを記述する必要があることに注意してください。理想的には、キャンセルリクエストの後、非同期タスクをできるだけ早くキャンセルする必要があります。

完了するまでに時間がかかる可能性があるすべてのメソッドに対して、取り消しを使用することを検討する必要があります。この記事の

詳細は、公式のMicrosoft .NETブログをご覧ください。
