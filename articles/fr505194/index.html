Windows Defender SmartScreen nuit aux développeurs indépendants

Supposons que vous soyez un développeur indépendant et que vous ayez publié votre application. Pour simplifier l'installation, vous créez un programme d'installation et commencez à le distribuer. Certains courageux expérimentateurs téléchargent, lancent le programme d'installation - mais voient un avertissement strict:


Windows SmartScreen bloque le lancement du fichier exécutable

En effet, dans l'environnement actuel, Windows bloque activement le lancement des fichiers binaires via la fonction SmartScreen.

Mais qu'est-ce qu'un écran intelligent?


SmartScreen collecte les données d'installation de tous les utilisateurs de Windows pour déterminer la «réputation» de l'application. Si le programme n'a pas une bonne réputation, vous recevez un tel avertissement. La plupart des utilisateurs dans cette situation supprimeront .exe, le considérant comme un programme malveillant, mais vous pouvez contourner SmartScreen en cliquant sur le bouton "Détails" puis sur "Exécuter quand même".

Raquette à signature numérique


Mais comment se bâtir une réputation? Tout d'abord, Microsoft devrait pouvoir collecter des informations sur qui a publié l'application, et cela se fait à l'aide d'un certificat de signature de code. La conclusion la plus évidente est que les applications non signées appellent toujours SmartScreen. Une conclusion plus insidieuse est que l'acquisition d'un certificat de signature de code est une grosse dépense pour un développeur individuel. Il n'y a actuellement aucun équivalent de Let's Encrypt pour signer un code, vous devrez donc acheter un certificat auprès d'autorités de certification réputées. La fourchette de prix est large, mais un certificat d'un an coûte généralement environ 100 $.


SSL.com propose l'un des certificats de signature de code «les moins chers» sur le marché, au prix de 129 $ par an.

Supposons que vous picoriez un crochet, que vous vous achetiez un nombre surestimé de nombres premiers multiplié par un ordinateur, que vous signiez votre code et que vous republiez votre application. Maintenant, les utilisateurs vont commencer à installer votre application, non? Faux.

Construire une réputation est un truc 22


Même avec une signature numérique, SmartScreen fonctionnera toujours. Après tout, vous êtes un nouvel éditeur inconnu et le processus de «création de réputation» de SmartScreen est une boîte noire absolue. Alors, voici l'astuce 22: pour créer une réputation, vous avez besoin de personnes pour installer votre logiciel - et Microsoft collectera des données avec des statistiques d'installation. Mais pour que les utilisateurs installent votre logiciel, vous devez vous débarrasser d'un message qui suggère clairement que votre code endommagera leurs ordinateurs.

Pire encore.

Si vous essayez de publier votre logiciel à l'aide du gestionnaire de packages WinGet (en bref: lorsqu'il quitte la version bêta, ce gestionnaire copié depuis AppGet remplacera probablement le Windows Store), vous recevrez le message suivant de Microsoft:


Microsoft WinGet considère le lancement de SmartScreen comme une «erreur» et votre logiciel est classé comme «malveillant».

Selon Microsoft, votre application est considérée comme malveillante si elle lance SmartScreen. Ils recommandent d'envoyer le fichier pour examen par un lien spécial. Si vous suivez ce processus, vous obtiendrez la réponse suivante:

, . Windows Defender .

, , SmartScreen. , .

, . , , , «». , .

, - , , , . , .

Essentiellement: votre application est propre, mais nous ne ferons rien avec l'erreur SmartScreen. Nos seigneurs du cloud bot décideront quand vous pouvez faire confiance.

Et puis pire.

Disons que vous êtes enfin approuvé par les algorithmes en tant qu'éditeur fiable. Votre certificat expirera et vous le renouvellerez. Une chose courante sur Internet. Mais dans le monde des logiciels Windows, votre réputation sera annulée. SmartScreen ne reconnaît pas les mises à jour de certificats, il vous considère comme un nouvel éditeur et vous devez recommencer ce processus douloureux.

Cela peut être atténué en achetant un certificat avec une période de validité plus longue, mais le coût devient alors une barrière insurmontable pour la plupart des développeurs indépendants.

Certificats de signature de code EV


Pour les développeurs indépendants, c'est la fin de la route. Pour les éditeurs, il existe une autre solution: un certificat de vérification avancé pour la signature de code (EV). En signant le code à l'aide de l'un d'eux, vous contournez automatiquement SmartScreen. Il est réservé uniquement aux entreprises et, comme son nom l'indique, prévoit une vérification supplémentaire des informations. Elle est menée par des centres de certification.

L'autorité de certification populaire Digicert.com vend de tels certificats pour la modique somme de 699 $ par an.

Un développeur indépendant devra passer par les difficultés de former une entité juridique et débourser à des prix scandaleux pour ces certificats. Bien sûr, pour la plupart, ce n'est pas réaliste.

"Développeurs, développeurs, développeurs!"




"Développeurs, développeurs, développeurs!" Cria Steve Ballmer de la scène. L'un des discours légendaires du PDG de Microsoft. Ces mots notoires ont été prononcés en 2006. Quatorze ans plus tard, Microsoft chez Sathya Nadella a été félicité pour avoir rendu l'entreprise plus ouverte que jamais. Mais ironiquement, il s'avère que Microsoft a rendu son environnement Windows extrêmement hostile à ses développeurs préférés. Pour résoudre cette terrible situation, vous devez modifier SmartScreen ou le fonctionnement des certificats de signature de code.
