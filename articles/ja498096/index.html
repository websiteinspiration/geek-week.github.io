メモリーズデモが256バイトに収まる方法


前書き


こんにちは！私の名前はHellMoodで、この記事はMemoriesと呼ばれる小さなMS DOSプログラムに関するものです。このプログラムのサイズは256バイトで、2020年に«PC 256バイト»競技デモシーン «改訂»のカテゴリで優勝し、オーディエンス賞も受賞しました。ここでプログラムの出力のビデオを見ることができ、ここでオンライン視聴者とモデレーターの反応を含むビデオを見ることができます。リリースをダウンロードしてコメントをここに残してください。この記事では、プログラムの詳細な分析を示し、歴史的な参考文献と開発段階について説明します。記事はsizecoding wikiに投稿されています。メモリの内部構造を理解するだけでなく、似たようなものを作成するのにも役立ちます。探検しよう！サイズコーディング（希望のサイズ内でプログラムを作成する）またはx86アセンブラーを初めて使用する場合は、このwikiの基本から始めることをお勧めします。原則は簡単に理解できますが、実装の詳細を理解するのはそれほど簡単ではありません。

短いレビュー


この記事では、DosBoxのコンテストに送信されるバージョン（256バイト）について説明します。アーカイブには、FreeDosとWindows XP DOSのバージョンも含まれています。この記事の執筆時点では、一部のコンピューターでは動作しませんでした。これらの代替バージョンは、プログラムがエミュレータで機能するだけではないことを示すための概念実証としてアーカイブに含まれています。 2020年の「改訂」コンテストの「PC 256バイト」カテゴリでは、プラットフォームとして「FreeDos」または「DosBox」を指定することができました（特定の構成では後者）。代替バージョンが証明するように、実際には、DosBoxのバージョンを変更して、FreeDos、MS DOS、WinXP、およびWin98で動作するようにすることができますが、記事ではそれについて触れません。

いずれにせよ、すべてのプラットフォームとコンピューターの信頼できるバージョンが開発中です。さらなる最適化については、コードに最適化できる場所がすでにいくつか見つかっていますが、コンテストに送信されるバージョンについてのみ説明します。これは小さな効果の履歴のデモンストレーションであるため、完璧なコードはほとんどなく、さらに減らすことができます。異なるバージョンで混乱しないように、競争についてのみ説明します。

小さな効果の歴史



http://www.pouet.netのディメンションカテゴリ

私たち、sizcoders、サイズカテゴリの理由。MS DOSの場合、これらのカテゴリは256b、128b、64b、および32bです。これらは、デモシーンの最大のアーカイブの1つであるwww.pouet.netの標準です。。 16bカテゴリはありませんが、多くの小さな効果を16バイトで実装できます。 「メモリーズ」からのほとんどすべてのエフェクトは、以前に私が作成して最適化したものであり、基本的に、それらの実装は、エフェクトの既存のサイズを小さくする、または似たような小さいものを作成する試みでした。ここでのサイズの縮小とは、次の小さいカテゴリのいずれかに縮小することを意味します2 ^N。たとえば、効果が33〜64バイトで実装された場合、32バイト以下に削減されます。ほとんどの場合、効果をより低いカテゴリーに「下げる」ことができるたびに、結果の小さなプログラムをデモシーン専用のイベントに送信しました。このイベントでは、リモートの参加者はカテゴリー256bへの参加を許可され、結果はwww.pouet.netに公開されました。。このセクションでは、効果を紹介し、その起源と作者について説明します。

フェージング効果


2つの効果間の移行は、それ自体、前任者のいない効果です。むしろ、それは数年にわたって進化してきたアイデアであり、おそらく他の多くの人々によって同様に実装されたのでしょう。つまり、フレームを計算するときに、各ピクセルの位置がランダム化され、使用する効果を決定する時間が、以前に削減されたこのランダム化値によってシフトされます。これにより、スムーズなミキシングのために独自の色を作成するのではなく、標準のVGAパレット（画像、ソース）を使用できるため、スペースを節約できます。

私の「小さなメガデモ」の枠組み

円をズームする



ズーム円

点（X、Y）から中心（0,0）までの距離Dはsqrt（X²+Y²）です。フレームワークは以前、DLに中心座標Xを含め、DH = Yはコードの中心に置く必要があります。x86で平方根計算操作を実行するには、かなり多くのコードが必要ですが、実際には省略できます。色を注意深く選択すると、平方根のない円が収束する感覚は非常に説得力があります。

	mov al,dh		; get Y in AL
	sub al,100		; align Y vertically
	imul al			; AL = Y²
	xchg dx,ax		; Y²/256 in DH, X in AL
	imul al			; AL = X²
	add dh,ah		; DH = (X² + Y²)/256
	mov al,dh		; AL = (X² + Y²)/256
	add ax,bp		; offset color by time
	and al,8+16		; select special rings

シェルピンスキーロトズーマー



シェルピンスキーロトズーマー


www.google.com

を使用して作成されたグラフ1 / cos（atan（x））スケーリングで回転するには、通常、三角関数またはその適切な近似値が必要です。のは、見てみましょう通常の2D回転式や計算で三角法を取り除くために特別な方法でそれを解釈：

x' = x*cos(a) - y*sin(a)
y' = x*sin(a) + y*cos(a)

スケーリングを追加すると、次のようになります。

x' = z * (x*cos(a) - y*sin(a))
y' = z * (x*sin(a) + y*cos(a))

ここで、zを自分で定義せず、1 / cos（a）を角かっこから外します。

x' = 1/cos(a) * (x - y*tan(a))
y' = 1/cos(a) * (x*tan(a) + y)

ここで、tan（a）を時間変数に置き換えます。これは、タンジェント関数が180°に近づくにつれて無限大になる傾向があるためです。

x' = 1/cos(atan(T)) * (x - y*T)
y' = 1/cos(atan(T)) * (x*T + y)

倍率を設定する機能がないことを気にせず、角度を直接制御しない場合は、三角関数を使用せずに-180°から+ 180°に回転できるようになりました。結果として、スケーリングファクターは時間Tに関連付けられることがわかります。係数の関数は画像に示されています。これにより、スケーリングは、小数から1（元のサイズ）に、そして小数に戻ります。この効果の装飾、時間オフセットの修正、アニメーションの高速化、シェルピンスキーの三角形のピクセル数の増加、美しい色の作成にかなりのバイトが費やされましたが、それだけの価値はあったと思います。

	lea cx,[bp-2048]; center time to pass zero
	sal cx,3		; speed up by factor 8!
	movzx ax,dh		; get X into AL
	movsx dx,dl		; get Y int DL
	mov bx,ax		; save X in BX

	add bh,dl		; BH = X*T/256+Y

レイキャストベントトンネル

これは「新時代へ」（上記参照）からの荒れ狂うトンネルの一種です。同様のエッセンスプログラムに対するこの効果の詳細な説明が redditに公開されています。古いコンピューターとDosBoxのパフォーマンスを向上させるために、ユニークな色を取り除き、傾斜の方向を変更し、ジオメトリをより閉じました。

	mov cl,-9		; start with depth 9 (moves backwards)
	fx5L: 
	push dx			; save DX, destroyed inside the loop
		mov al,dh	; Get Y into AL
		sub al,100	; Centering Y has to be done "manually".
		imul cl		; Multiply AL=Y by the current distance, to get a projection(1)
		xchg ax,dx	; Get X into AL, while saving the result in DX (DH)
		add al,cl	; add distance to projection, (bend to the right)

		mov al,dh	; Get projection(1) in AL

	pop dx			; restore DX
	loopz fx5L		; repeat until "hit" or "iter=max"

	xor al,cl		; XOR pattern for texture

	add al,20		; offset into grayscale palette

オーシャンナイトトゥデイ



オーシャンナイト


oceanday

オーシャンエフェクトは「fluke」の優れた例です。値を整数としてFPUにロードし、それを浮動小数点として格納してから、再び整数として解釈すると、優れたパターンが得られます。これを逆除算と組み合わせると、美しい波の効果が得られます。 DXレジスタと密接に連携しており、必要な位置に符号が反転しているため、空と海を簡単に分離できます。しかし、最後のタッチは色です。グローバルフレームワークの構造に従って、ALの値は関数の入力で決定され、エフェクトのアドレスが含まれます。コードを少しシャッフルすることにより、エフェクトのアドレスでもある「今日への移行」の色と同じように、指示を使用せずに空の色を「無料」にすることができます。そして、これは幸せな偶然ではありません。 DosBoxバージョン以外のバージョンでは、このため、空の色は異なる場合があります。

	sub dh,120			; check if pixel is in the sky
	js fx6q				; quit if that's the case
	mov [bx+si],dx		; move XY to a memory location

MIDI音楽


コードのこのセクションでは、時間の値を増分し、サウンドを作成します。チャネル3を選択すると、「ツールチャネルの変更」命令を「RET」として再利用できます。エフェクトコードをシフトすることにより、ボリュームに適した値を作成し、別のバイトを節約できます。コードのこの部分は、MIDIデバイスがすでにUARTモードである場合にのみ機能することを考慮する価値があります。そうでない場合は、さらに3バイトを費やす必要があります。観客や主催者の多くは、この曲はマイクオールドフィールドの呪文に少し似ていると言いましたが、非常にシンプルな手法で作成されました。Hypnoteye 2015以降、私は手続き型MIDIを実験し、最終的に小さなMIDIフレームワーク（64バイト）をリリースしました。彼の作品の主な原則は、トーン空間で固定ピッチでジャンプし、高い値を低い値に変換することです（除算モジュロによって）。ステップ幅とmodvalueの単純な組み合わせにより、興味深い効果を得ることができます。たとえば、stepwidth = 3を指定すると、継続的に継続するマイナーコードが減少し、stepwidth = 4またはstepwidth = 6-トリトナルエフェクトになります。適切に選択されたmodvalueにより、これらのパターンはシーケンスを作成できます。私は適切な理論的分析を実行していませんが、単に音の空間を調査し、興味深い音に気づきました。
