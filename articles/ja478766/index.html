Spring MVCコントローラーのマッピングをカスタマイズする

前書き


最近、リクエストパラメータがある場合とない場合のさまざまな方法で状況を処理するコントローラを実装するタスクに直面しました。この問題は、コントローラーで正確に2つの異なるメソッドが必要だったという事実によってさらに悪化しました。Spring MVCの標準機能ではこれができませんでした。私はもう少し深く掘り下げる必要がありました。誰も気にしない-猫へようこそ。


私達何が欲しいの


これは、タスクの本質を説明する短いテストです。 


TestControllerTest.java
@SpringJUnitWebConfig(WebConfig.class)
class TestControllerTest {

    @Autowired
    WebApplicationContext webApplicationContext;

    @Test
    void testHandleRequestWithoutParams() throws Exception {
        MockMvc mockMvc = MockMvcBuilders.webAppContextSetup(webApplicationContext).build();

        mockMvc.perform(MockMvcRequestBuilders.get("/test"))
                .andExpect(status().isOk())
                .andExpect(result ->
                        assertEquals(TestController.HANDLE_REQUEST_WITHOUT_PARAMS, result.getResponse().getContentAsString()));
    }

    @Test
    void testHandleRequestWithParams() throws Exception {
        MockMvc mockMvc = MockMvcBuilders.webAppContextSetup(webApplicationContext).build();

        mockMvc.perform(MockMvcRequestBuilders
                .get("/test")
                .param("someparam", "somevalue"))
                .andExpect(status().isOk())
                .andExpect(result ->
                        assertEquals(TestController.HANDLE_REQUEST_WITH_PARAMS, result.getResponse().getContentAsString()));
    }
}

すべてが自動的に解決されることを願っています


TestController.java

あきらめないで


試みは、アノテーションを経由して、これらのメソッドのマッピングを分離するために行われてきた@RequestMapping(params = "some condition")が、残念ながら春5.1.8の現在のバージョンに含まれている要求のための条件を設定する方法はありません任意のパラメータが。特定の名前のパラメーターの存在を指定するか、特定の名前のパラメーターの非存在を指定できます。どちらも適切ではありませんでした。パラメータはリクエストごとに異なる場合があります。@RequestMapping(params = "*")リクエストにいくつかのパラメーターを含めるか、または@RequestMapping（params = "！*"） `リクエストにパラメーターを含めないことを示すために、このようなものを書きたいと思います。

public class CustomRequestMappingHandlerMapping extends RequestMappingHandlerMapping {

    @Override
    protected RequestCondition<?> getCustomMethodCondition(Method method) {
        return method.isAnnotationPresent(NoRequestParams.class) ? NO_PARAMS_CONDITION : null;
