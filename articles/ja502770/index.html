ProLock：MITER ATT＆CKを使用した新しいランサムウェアオペレーターのアクションの分析を明らかにします



世界中の組織に対するランサムウェア攻撃の成功により、ますます多くの新しい攻撃者が「ゲームに参加」するようになっています。これらの新しいプレーヤーの1つは、ProLockランサムウェアグループです。 2020年3月に、2019年後半に運用を開始したPwndLockerプログラムの後継として登場しました。 ProLockランサムウェア攻撃は、主に金融および医療組織、政府機関、小売部門を標的としています。最近、ProLockオペレーターは最大のATMメーカーの1つであるDiebold Nixdorfへの攻撃に成功しました。

この投稿では、コンピューターフォレンジックグループ-IBのラボの主任スペシャリストであるOleg Skulkinが、ProLockオペレーターが使用する基本的な戦術、手法、および手順（TTP）について説明します。記事の最後は、さまざまなサイバー犯罪者グループが使用する標的型攻撃の戦術を含む公開データベースであるMITER ATT＆CKマトリックスとの比較です。

初期アクセスを取得する


ProLockオペレーターは、主要な侵害の2つの主要なベクトルを使用します。QakBotトロイの木馬（Qbot）と、脆弱なパスワードを持つ保護されていないRDPサーバーです。

外部からアクセス可能なRDPサーバーを介した侵害は、ランサムウェアオペレーターに非常に人気があります。通常、攻撃者は侵害されたサーバーへのアクセスをサードパーティから購入しますが、グループのメンバーが自分で入手することもできます。

より興味深い主要な侵害ベクトルはQakBotマルウェアです。以前は、このトロイの木馬は、別の暗号化ファミリであるMegaCortexに関連付けられていました。ただし、現在はProLockオペレーターが使用しています。

通常、QakBotはフィッシングキャンペーンを通じて配布されます。フィッシングメールには、添付されたMicrosoft Officeドキュメントや、クラウドストレージにあるそのようなファイル（Microsoft OneDriveなど）へのリンクが含まれる場合があります。

QakBotに別のトロイの木馬-Emotetをロードするケースもあります。Emotetは、Ryukランサムウェアを配布したキャンペーンに参加することで広く知られています。

パフォーマンス


感染したドキュメントをダウンロードして開いた後、ユーザーはマクロの実行を許可するように求められます。成功すると、コマンドサーバーからQakBotペイロードを読み込んで実行するためにPowerShellが起動します。

同じことがProLockにも当てはまることに注意してください。ペイロードはBMPまたはJPGファイルから抽出され、PowerShellを使用してメモリにロードされます。場合によっては、スケジュールされたタスクを使用してPowerShellを起動します。

タスクスケジューラを介してProLockを起動するバッチスクリプト：

schtasks.exe /CREATE /XML C:\Programdata\WinMgr.xml /tn WinMgr
schtasks.exe /RUN /tn WinMgr
del C:\Programdata\WinMgr.xml
del C:\Programdata\run.bat

システムピン


RDPサーバーに危害を加えてアクセスできる可能性がある場合は、既存のアカウントを使用してネットワークを保護します。QakBotは、さまざまな固定メカニズムが特徴です。ほとんどの場合、このトロイの木馬はRunレジストリキーを使用して、スケジューラにタスクを作成します。


Runレジストリキーを使用してシステム上のQakbotを保護


する場合によっては、スタートアップフォルダーも使用されます。ブートローダーを指すショートカットがそこに配置されます。

保護バイパス


コマンドサーバーとの通信を通じて、QakBotは定期的に自身を更新しようとします。したがって、検出を回避するために、マルウェアは現在のバージョンを新しいバージョンに置き換えることができます。実行可能ファイルは、侵害された署名または偽の署名で署名されています。PowerShellによって読み込まれた初期ペイロードは、PNG拡張子が付いたコマンドサーバーに格納されます。さらに、実行後、正当なcalc.exeファイルに置き換えられます。

また、悪意のある活動を隠すために、QakBotを使用しているプロセスにコードを注入するための技術使用EXPLORER.EXEをこのために。

すでに述べたように、ProLockペイロードはBMPまたはJPGファイル内に隠されています。保護を回避する方法としても考えられます。

資格情報を取得する


QakBotにはキーロガーの機能があります。さらに、追加のスクリプト、たとえば、有名なユーティリティMimikatzのPowerShellバージョンのInvoke-Mimikatzをロードして実行できます。このようなスクリプトは、サイバー犯罪者が資格情報をダンプするために使用できます。

ネットワークインテリジェンス


特権アカウントにアクセスした後、ProLockオペレーターはネットワークインテリジェンスを実行します。これには、特にポートスキャンやActive Directory環境の分析が含まれる場合があります。攻撃者はさまざまなスクリプトに加えて、ランサムウェアを使用するグループに人気のある別のツールであるAdFindを使用して、Active Directoryに関する情報を収集します。

ウェブプロモーション


従来、ネットサーフィンを行う最も一般的な方法の1つは、リモートデスクトッププロトコルです。プロロックも例外ではありませんでした。攻撃者は、RDPを介してターゲットホストへのリモートアクセスを取得するスクリプトを武器に持っています。

RDP経由でアクセスするためのBATスクリプト：
 リモートスクリプト実行の場合、ProLockオペレーターは別の一般的なツールであるSysinternals SuiteパッケージのPsExecユーティリティを使用します。 ホスト上のProLockは、Windows Management Instrumentationサブシステムを操作するためのコマンドラインインターフェイスであるWMICを使用して起動されます。このツールは、ランサムウェアオペレーターの間でも人気を得ています。
reg add "HKLM\System\CurrentControlSet\Control\Terminal Server" /v "fDenyTSConnections" /t REG_DWORD /d 0 /f
netsh advfirewall firewall set rule group="Remote Desktop" new enable=yes
reg add "HKLM\System\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" /v "UserAuthentication" /t REG_DWORD /d 0 /f





データ収集


他の多くのランサムウェアオペレーターと同様に、ProLockを使用するグループは、侵害されたネットワークからデータを収集して、身代金の可能性を高めます。抽出前に、収集されたデータは7Zipユーティリティを使用してアーカイブされます。

漏出


データをアップロードするために、ProLockオペレーターはRcloneを使用します。Rcloneは、OneDrive、Google Drive、Megaなどのさまざまなクラウドストレージサービスとファイルを同期するように設計されています。攻撃者は常に、実行可能ファイルの名前を正当なシステムファイルのように変更します。

「同僚」とは異なり、身代金の支払いを拒否した会社に属する盗まれたデータを公開するための独自のWebサイトはまだProLockオペレーターにはありません。

究極の目標を達成する


データを引き出した後、チームはエンタープライズネットワーク全体にProLockを展開します。バイナリファイルは、PowerShellを使用して、PNGまたはJPG拡張子のファイルから抽出され、メモリに埋め込まれます。


まず、ProLockは組み込みリストに示されているプロセスを終了します（たとえば、「winwor」などのプロセス名から6文字だけを使用するのは興味深いことです）。セキュリティに関連するサービス（CSFalconService（CrowdStrike Falcon）など）を含み、net stopコマンドを使用します。

次に、他の多くのランサムウェアファミリーと同様に、攻撃者はvssadminを使用してWindowsのシャドウコピーを削除し、サイズを制限するため、新しいコピーは作成されません。

vssadmin.exe delete shadows /all /quiet
vssadmin.exe resize shadowstorage /for=C: /on=C: /maxsize=401MB
vssadmin.exe resize shadowstorage /for=C: /on=C: /maxsize=unbounded

ProLockは、拡張子.proLock、.pr0Lock、または.proL0ckを各暗号化ファイルに追加し、[HOW TO RECOVER FILES] .TXTファイルを各フォルダーに配置します。このファイルには、被害者が一意の識別子を入力して支払い情報を受け取る必要があるサイトへのリンクを含め、ファイルを復号化する方法に関する指示が含まれています。


ProLockの各インスタンスには、買い戻し額に関する情報が含まれています-この場合、それは35ビットコインで、約312,000ドルです。

結論


多くのランサムウェアオペレーターは、同様の方法を使用して目標を達成しています。同時に、いくつかのテクニックは各グループに固有です。キャンペーンで暗号化機能を使用するサイバー犯罪グループが増えています。場合によっては、同じオペレーターがランサムウェアの異なるファミリーを使用した攻撃に参加できるため、使用される戦術、手法、および手順の交差をますます観察するようになります。
