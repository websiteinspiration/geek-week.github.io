Kuadran Gartner 2020 untuk Solusi Pemantauan Aplikasi (APM)

Beberapa hari yang lalu, Gartner menerbitkan laporan baru tentang sistem APM (Manajemen Kinerja Aplikasi). Di saluran telegram saya , saya bertanya kepada pelanggan tentang perkiraan mereka untuk para pemimpin kuadran ini sebelum dipublikasikan. Secara umum, intrik tidak berhasil. Detail di bawah potongan.



Meskipun intrik tidak berhasil dengan para pemimpin, pemain ambisius baru muncul di kuadran - Datadog, Instana dan SignalFx (sekarang bagian dari Splunk). Mereka berkembang sangat aktif dan, saya kira, dalam beberapa tahun ke depan memiliki potensi untuk menjadi pemimpin pasar bersama dengan sektor lama sektor ini - Appdynamics, New Relic dan Dynatrace.

Pada tahun 2025, 50% aplikasi pemantauan berbasis cloud baru akan menggunakan alat sumber terbuka dan bukan agen vendor untuk meningkatkan kompatibilitas, dibandingkan dengan 5% pada 2019.

Dan banyak perhatian diberikan pada tren ini dalam laporan. Kehadiran dukungan teknologi terbuka digambarkan sebagai keunggulan vendor, dan absennya sebagai kerugian. Di bawah ini saya akan memberikan daftar vendor yang berpartisipasi dalam kuadran beserta kekuatan dan kelemahannya menurut Gartner.

Tahun ini, kriteria untuk memasukkan solusi APM dalam laporan semua juga mencakup tiga persyaratan utama (tetapi, dibandingkan dengan tahun lalu , persyaratan telah sedikit berubah):

Pemantauan front-end. Pendekatan ini untuk aksesibilitas dan pemantauan kinerja adalah bagian dari metodologi Digital Experience Monitoring ( DEM ). Untuk keperluan laporan ini, pendekatan ini akan mencakup dukungan untuk memantau pengguna nyata ( RUM) dan memantau transaksi sintetis untuk pengguna aplikasi web dan seluler. Pemantauan front-end adalah pengembangan kuadran ajaib tahun lalu untuk solusi APM , di mana persyaratan ini disebut pemantauan pengalaman digital. DEM berisi serangkaian teknologi yang lebih besar daripada yang digunakan dalam APM.

Penemuan Aplikasi, Pelacakan, dan Diagnostik ( ADTD ). Ini berlaku untuk penemuan otomatis server web, server aplikasi dan layanan Microsoft, serta lingkungan dan platform aplikasi (seperti wadah, orkestra, dan service mesh). ADTD menentukan hubungan antara elemen-elemen ini dengan memonitor perilaku transaksi HTTP / S dari aplikasi menggunakan bytecode toolkit (BCI) dan / atau penelusuran aplikasi terdistribusi.

Analisis: Solusi APM harus menyediakan AIOps yang berorientasi domain menggunakan AI / ML , secara kolektif disebut sebagai analitik di sini (lihat Solusi AIOps Gartner 2019-2020 ). Untuk melakukan ini, solusi APM harus menggunakan algoritma korelasi, deteksi anomali, dan akar penyebab ( RCA ) berdasarkan data yang tersedia di APM dalam konteks topologi. Solusi ini juga dapat memberikan kemampuan untuk menerima acara dari sumber pihak ketiga. Kriteria ini adalah evolusi kuadran ajaib tahun lalu untuk solusi APM , di mana persyaratan ini disebut AIOps. AIOps sendiri dapat mencakup rentang pendekatan yang lebih luas untuk digunakan daripada solusi APM yang ada.

Gartner Magic Quadrants dibagi menjadi 4 sektor: pemimpin, penantang, ahli strategi, dan pemain khusus. Setiap vendor ditempatkan di kuadran berdasarkan kekuatan dan kelemahannya, pangsa pasar dan ulasan pengguna, serta indikator lainnya. 15 vendor yang termasuk dalam Gartner saat ini: Broadcom (CA Technologies), Cisco (AppDynamics), Datadog, Dynatrace, Instana, IBM, ManageEngine, Micro Focus, Microsoft, Peninggalan Baru, Oracle, Tempat Tidur Sungai, SolarWinds, Splunk (SignalFX) dan Tingyun.
