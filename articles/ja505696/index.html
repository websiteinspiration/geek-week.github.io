Java 15での密封されたクラスの探索

Javaに表示される新機能については、引き続き調査しています。過去には、詳細 instanceofと記録を調査してきましたが、今日は学習の対象となるsealedクラスで、第15版の Java でリリースされる予定です。


「密封された」型を言語に導入するアイデアは、2019年2月に「Javaのデータクラスと密封された型」で詳細に説明され、その年の7月にJEP 360を受け取りました。これは、 Oleg Chirukhin（Olegchir） Olegが翻訳したJEP'aのバージョンは少し古く、現在のバージョンとは異なることに注意してください。特に、新しいバージョンでは、sealed型は型ではなく、クラスとインターフェースです。この名前変更は、用語「タイプ」はあまりにも過負荷であるという事実と接続され、クラスおよびインタフェースのみならずを意味することができるだけでなく、アレイのようなそれらの誘導体（Object[]、String[]（）とジェネリックのパラメータList<Integer>、List<?>）。この名前の変更について詳しくは、こちらをご覧ください。


でも歌詞は綺麗なので、直接sealed授業に行きましょう。ScalaとKotlinに精通しているプログラマーにとって、この種のクラスは精通している必要があります。Javaでは、これはまったく同じことを意味しsealedます。修飾子は、このクラスから継承できるクラスの範囲を制限します。しかし、C＃では、まったく別のことをsealed意味します。したがって、C＃から来た場合は、注意sealedしてください。Java のキーワードの類似物は、finalではなくthis sealedです。
