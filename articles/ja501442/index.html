一部のアプリケーションはスクリプトを介して実行されます。たとえば、WildFlyアプリケーションサーバーを起動するには、binフォルダーに移動し、そこでstandalone.shを実行する必要があります。拡張子が.shのファイルはスクリプトです。

スクリプトを実行するには、スクリプトへのフルパスを指定する必要があります。

/opt/cdi/jboss/bin/standalone.sh —   standalone.sh

大事です！スクリプトが置かれているフォルダー内にいても、shスクリプトの名前を指定するだけでは見つかりません。次のように書く必要があります：

./standalone.sh —  ,

検索は、PATH変数に書き込まれたディレクトリでのみ行われます。したがって、スクリプトが頻繁に使用される場合は、そこにパスを追加し、名前で呼び出すだけです。

standalone.sh ---   standalone.sh,      PATH

アーカイブを作成する方法


ベアシステムでも機能する標準ユーティリティはtarです。確かに、それを使用するには、フラグを覚えておく必要があります。アーカイブを作成するには、cvzfの標準的な組み合わせ：

tar -cvzf archive.tar.gz /home/test



この例では、/ home / testディレクトリをパックしました。この中には、502.jpgと504.jpgの 2つの画像がありました。

解凍するには、フラグ「c」を「x」に変更し、「z」を削除します。

tar -xvf archive.tar.gz



システムは「/ home / test」を解凍したと書き込みますが、実際には「test」フォルダが現在の場所に表示されます。

これらすべてのフラグの意味を見てみましょう：

  • c -Linuxでアーカイブを作成する
  • x-アーカイブからファイルを抽出します
  • v-ワークプロセスに関する詳細情報を表示します（これがないと、どのファイルがパック/アンパックされたかはわかりません）。
  • f-アーカイブを記録するためのファイル
  • z-圧縮

パッケージングには、c - C reateオプションを使用し、x -e X tractのアンパックには、使用してください。



本当にrarを使用したい場合は、キャストアウトする必要があります。インストールはyumで機能しません：

yum install rar
yum install unrar

そのようなパッケージはないと言います：

No package rar available.
Error: Nothing to do

たくさんのチームを完了する必要があります！まず、ダウンロードして解凍し、コンパイルします。

wget http://rarlabs.com/rar/rarlinux-x64-5.4.0.tar.gz
tar xzf rarlinux-x64-5.4.0.tar.gz
cd rar
make install

インストール：

mkdir -p /usr/local/bin
mkdir -p /usr/local/lib
cp rar unrar /usr/local/bin
cp rarfiles.lst /etc
cp default.sfx /usr/local/lib

そして適用：

unrar x test.rar

以前に使用したコマンドを表示する方法


したがって、いくつかの複雑なアクションを実行したとしましょう。それともあなたではなく、開発者か管理者か！何かがあなたのために壊れました、同僚が入って、ペンでスワイプして出入りしました、魔法-それは機能します。彼が何をした？それは面白いです！

あるいは、長いコマンドを書いて、今度はそれを繰り返す必要があるかもしれません。またペン？気まずい！さらに、アシスタントがいます：

↑  ( «») —   
history —   1000

閉じるのに「巻き戻す」必要がある場合は、矢印をスクロールする方が簡単です。一度押すと、システムは最後のコマンドを表示しました。もう一度クリック-最後から2番目。そして、1,000回まで続きます（これは、履歴にどれだけ保存されているかです）。



大きなボーナスは、コンソールを再起動してもLinuxが履歴を保持することです。これはWindowsとは異なり、テキストをコピーして別のテキストをコピーしましたが、最初のテキストはすでに失われています。そして、システムを再起動すると、すべてが失われました。

  • mkdir-ディレクトリを作成する
  • touch-ファイルを作成します（または既存のファイルを変更しますが、同じ名前のファイルがない場合、コマンドは新しい空のファイルを作成します）

かっこ内の式は、ループで実行され、手動で作業を行う関数の役割を果たします。

LinuxでIOPSをテストする方法


これは、顧客が車を渡した場合に非常に役立ちます。SSDはありますか？そして、彼らは良いiopsを与えますか？サーバーアプリケーションを開発していて、負荷に耐える必要がある場合は、ToRに従ってディスクが提供されていることを確認する必要があります。

しかし、反対側の管理者をその言葉で信頼することはできません。アプリケーションの実行速度が遅い場合、当然、彼らはすべてが問題なく、「彼ら」が責任を負うと言います。したがって、ディスクを自分でテストする必要があります。

ドライブのテスト方法についてお話します。発行されたIOPSの数を確認する方法。



fioユーティリティ-https ://github.com/axboe/fio/releasesを使用します。

1）最新バージョンをダウンロードして解凍し、ディレクトリに移動します。以下のコマンドでは、「fio-3.19」をリストの現在のバージョンに置き換える必要があります

cd /tmp
wget https://github.com/axboe/fio/archive/fio-3.19.tar.gz
tar xvzf fio-3.19.tar.gz
rm fio-3.19.tar.gz
cd fio-fio-3.19

2）組み立て用のパッケージが必要です

apt-get install -y gcc make libaio-dev | yum install -y make gcc libaio-devel

3）集める

make

4）テスト

./fio -readonly -name iops -rw=randread -bs=512 -runtime=20 -iodepth 32 -filename /dev/sda -ioengine libaio -direct=1
