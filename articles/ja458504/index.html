1.基本的なコマンドを記録するための簡単なオプション


基本から始めましょう。最も一般的なnpmコマンドの要約バージョンを探索するために少しの時間を費やすことにより、将来、多くの時間を節約できます。

  • パッケージをインストールします。通常バージョン：npm install。ショートバージョン：npm i。
  • テストを実行しています。通常バージョン：npm test。ショートバージョン：npm t。
  • 助けて。通常バージョン：npm --help。ショートバージョン：npm -h。
  • グローバル設定フラグ。通常バージョン：--global。ショートバージョン：-g。
  • 開発依存としてのパッケージインストールフラグ。通常バージョン：--save-dev。ショートバージョン：-D。
  • npm initデフォルト設定を受け入れます。通常のオプション：npm init --yesまたはnpm init --force。ショートカット：npm init -yまたはnpm init -f。

現在、パッケージを保存するために--saveor フラグを使用する必要はありません-S。パッケージはデフォルトで保存されます。保存せずにパッケージをインストールするには、フラグを使用できます--no-save。

2. npm initのデフォルト値を設定する


コマンドnpm initを常に実行して新しいプロジェクトを作成すると、システムの質問に答えて同じデータを何度も入力していることに気付くでしょう。たとえば、あなたが作成したほとんどのプロジェクトの作成者である可能性があります。同じデータを入力する時間を節約するために、対応するフィールドに独自のデフォルト値を設定できます。

npm config set init.author.name  "Joe Bloggs"
npm config set init.author.email "joebloggs@gmail.com"
npm config set init.author.url   "joebloggs.com"
npm config set init.license      "MIT"

そのような情報をシステムに追加することの正しさを確認するには、コマンドを入力しnpm config editます。これにより、システムエディターで構成ファイルが開きます。グローバルnpmパラメータを編集する場合は、コマンドを使用しnpm config edit -gます。

デフォルト設定に戻すには、次のスクリプトを使用できます。1行目はファイルから構成データを削除し、2行目はデフォルト値を設定データに入力します。

echo "" > $(npm config get userconfig)
npm config edit

上記のスクリプトは、ユーザーのデフォルト値をリセットします。次のスクリプトは、グローバルデフォルトをリセットします。

echo "" > $(npm config get globalconfig)
npm config --global edit

3.クロスプラットフォームスクリプト


コマンドラインで実行されるコードには、互換性の問題が発生する可能性があります。これは、WindowsベースのシステムとUnixベースのシステム（MacとLinuxを含む）の間の非互換性に特に当てはまります。あなたとあなただけが特定のプロジェクトに取り組んでいる場合、これは問題ではありません。しかし、多くの場合、クロスプラットフォームのスクリプト互換性が重要です。たとえば、オペレーティングシステムから独立して作業することは、オープンソース、トレーニング、またはデモンストレーションプロジェクトにとって大きなプラスになります。

幸い、スクリプトの互換性の問題を解決することは特に難しくありません。いくつかのオプションがあります。ただし、ここでは、そのうちの1つについて、最も良い結果が得られた使用方法について説明します。それはクロス環境パッケージについてです。チームを使用して開発依存関係として設定する必要がありますnpm i -D cross-env。次にcross-env、各環境変数の前に置く必要があります。たとえば、次のようになります。

{
  "scripts": {
    "build": "cross-env NODE_ENV=production webpack --config build/webpack.config.js"
  
}

このパッケージcross-envは、クロスプラットフォームのスクリプト互換性を実現するための最も便利なツールであることがわかりました。ただし、この問題の解決に役立つ次の2つの一般的なツールを確認すると便利です。

  • リムラフパッケージ。クロスプラットフォームスクリプトを実行するために、グローバルにインストールできます。
  • ShellJSパッケージ。これは、Unixコマンドシェルの移植可能なNode.js実装です。

6.リソースが利用可能になるまでスクリプトの起動を遅らせる


フルスタックアプリケーションを開発する場合、アプリケーションのクライアント部分とサーバー部分の両方を実行する必要が生じることがよくあります。待機パッケージを使用すると、特定のプロセスの実行を同期できます。私たちの場合、アプリケーションのクライアント部分を開始する前に、特定のポートが利用可能である必要があります。

たとえば、インターフェースがReactを使用して記述されているElectronアプリケーションで使用される開発スクリプトがあります。スクリプトは、をconcurrently同時に使用して、アプリケーションのプレゼンテーションレイヤーとElectronウィンドウを起動します。ただしwait-on、Reactプレゼンテーションレイヤーが使用可能になり、アドレスでアクセスできる場合にのみ、Electronウィンドウを開くことができますhttp://localhost:3000。使用方法は次のwait-onとおりです。

"dev": "concurrently \"cross-env BROWSER=none npm run start\" \"wait-on http://localhost:3000 && electron .\"",

さらに、Reactプロジェクトの標準的な動作は、ブラウザーウィンドウを自動的に開くことです。React with Electronを使用する場合、これは必要ありません。これは、環境変数を使用して無効にできますBROWSER=none。その前に、ソリューションのプラットフォーム間の互換性のために、コマンドがありcross-envます。

9.アプリケーションのバージョン管理


プログラマーは、アプリケーションのバージョンを手動で変更する代わりに、特別なコマンドを使用できますnpm。アプリケーションのバージョン番号の対応する部分を増やすには、コマンドのnpm version後にmajor、minorまたはを続けますpatch。

// 1.0.0
npm version patch
// 1.0.1
npm version minor
// 1.1.0
npm version major
// 2.0.0

アプリケーションを更新する頻度に応じて、アプリケーションの新しいバージョンをデプロイするたびにバージョン番号を自動的に増やすことで時間を節約できます。たとえば、次のようになります。

{
  "predeploy": "npm version patch"
}

13. npm initを使用して最初のコミットをGitHubリポジトリに送信する


gitファイル内の コマンドを実行するに.npm-init.jsは、コマンドラインを操作する方法を見つける必要があります。このためのモジュールを使用できますchild_process。これをファイルの先頭に接続します。必要なのは関数だけなので、構造化execSyncを使用してそれだけをインポートします。

const { execSync } = require('child_process');

さらに、コンソールに関数の結果を表示する補助関数を作成します。

function run(func) {
  console.log(execSync(func).toString())
}

最後に、GitHubリポジトリのURLを処理する適切なプロンプトブロックを作成します。URLがスクリプトの自由に使える場合、ファイルを作成しREADME.md、最初のコミットをリポジトリに送信します。

したがって、module.exportsファイルオブジェクトの要素の1つは.npm-init.js次のコードである必要があります。

repository: prompt('github repository url', '', function (url) {
  if (url) {
    run('touch README.md');
    run('git init');
    run('git add README.md');
    run('git commit -m "first commit"');
    run(`git remote add origin ${url}`);
    run('git push -u origin master');
  
  return url;
})
