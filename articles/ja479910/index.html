tcpserverとnetcatを使用してKubernetesポッドまたはコンテナーでトンネルを開く方法

注意 ペレフ。：LayerCIの作成者からのこの実用的なメモは、Kubernetesのいわゆるヒントとテクニックの優れた例です（そしてそれだけではありません）。ここで提案されている解決策は数​​少ないものの1つであり、おそらく最も明白ではありません（コメントに記載されているK8の「ネイティブ」な解決策が適している場合がありますkubectl port-forward）。ただし、これにより、少なくとも古典的なユーティリティとそれらの組み合わせ（単純、柔軟、強力）の観点から問題を調べることができます（インスピレーションについては、最後にある「その他のアイデア」を参照してください）。



典型的な状況を想像してみてください。ローカルコンピューターのポートで、魔法のようにトラフィックをポッド/コンテナー（またはその逆）にリダイレクトしたいとします。

可能なユースケース


  1. 運用/healthzクラスターでHTTPエンドポイントポッドが返すものを確認します。
  2. ローカルマシンのポッドにTCPデバッガーを接続します。
  3. 認証に煩わされることなくデータベースを操作するためのローカルツールから本番データベースへのアクセスを取得します（通常、localhostにはroot権限があります）。
  4. ステージングクラスターのデータ用のコンテナーを作成せずに、そのデータ用の移行スクリプトを1回だけ実行します。
  5. VNCセッションを仮想デスクトップを実行しているポッドに接続します（XVFBを参照）。

必要なツールについて一言


Tcpserverは、ほとんどのLinuxパッケージリポジトリで利用可能なオープンソースユーティリティです。これにより、ローカルポートを開き、stdin / stdoutを介して受信したトラフィックを、指定したコマンドからそのポートにリダイレクトできます。

colin@colin-work:~$ tcpserver 127.0.0.1 8080 echo -e 'HTTP/1.0 200 OK\r\nContent-Length: 19\r\n\r\n<body>hello!</body>'&
[1] 17377
colin@colin-work:~$ curl localhost:8080
<body>hello!</body>colin@colin-work:~$

kubectlとリンクします：ホストでリッスンしてポッドに接続します


上記のツールをkubectlと組み合わせると、次のようなコマンドが得られます。

tcpserver 127.0.0.1 8000 kubectl exec -i web-pod nc 127.0.0.1 8080

ユーティリティ相互作用図

反対方向：ポッドで聞いてホストに接続する


nc 127.0.0.1 8000 | kubectl exec -i web-pod tcpserver 127.0.0.1 8080 cat

このコマンドにより、ポッドはローカルマシンのポート8000​​にアクセスできます。

バッシュのスクリプト


上記の方法を使用して Kubernetes LayerCI本番環境クラスターを管理できるようにするBash用の特別なスクリプトを書きました。

kubetunnel() {
    POD="$1"
    DESTPORT="$2"
    if [ -z "$POD" -o -z "$DESTPORT" ]; then
        echo "Usage: kubetunnel [pod name] [destination port]"
        return 1
    fi
    pkill -f 'tcpserver 127.0.0.1 6666'
    tcpserver 127.0.0.1 6666 kubectl exec -i "$POD" nc 127.0.0.1 "$DESTPORT"&
    echo "Connect to 127.0.0.1:6666 to access $POD:$DESTPORT"
}

この関数を~/.bashrcに追加すると、podコマンドで簡単にトンネルを開いて実行できkubetunnel web-pod 8080ますcurl localhost:6666。

  • Dockerのトンネルの場合、メイン行を次のように置き換えることができます。

    tcpserver 127.0.0.1 6666 docker exec -i "$CONTAINER" nc 127.0.0.1 "$DESTPORT"
  • K3sのトンネル-次のように変更します。

    tcpserver 127.0.0.1 6666 k3s kubectl exec …
  • 等
