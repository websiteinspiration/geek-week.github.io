MS SQL Serverのスキーマとデータベースデータを同期するためのコンパレータの比較

変更同期の全体的な必要性の説明


データベースを使用する場合、変更の同期の問題を解決する必要があることがよくあります。
会社が1つの産業環境のみを使用している場合、それに加えて、加えられた変更をテストするために、少なくとも1つの追加の追加環境が必要です。この場合、テスト環境から工業用環境に変更を転送する必要があります。

会社の成長に伴い、必要なサーバーと仮想環境の数が増加し、データベース管理システムのインスタンスの数も増加する可能性があるため、より複雑な同期が必要になります。

原則として、データベースは専用の環境で開発されます。次に、変更はさまざまなチェック（単体テスト、自動テスト、負荷テストなど）のためにテスト環境に転送されます。そして開発と検査のすべての段階を経て初めて、変化が産業環境に移転されます。

十分に大きなインフラストラクチャでは、変更を複数の産業環境に一度に転送する必要があります。また、データベース開発は機能ごとに異なる環境に分割できるため、これらの環境間で定期的に変更を転送する必要があります。

開発環境の外で変更が適用され、それらを転送する必要がある場合があります。ただし、システムを正しく動作させるための脅威が発生しないように、このような状況を最小限に抑える必要があります。


一般に、変更の転送シーケンスは次のように表すことができます。
  1. 開発環境間。
  2. 開発環境からテスト環境まで。
  3. テスト環境間。開発環境以外からのテスト環境への変更を行う場合、そのような変更は開発環境にも転送されます。原則として、すべての変更は開発環境からテスト環境へと行われますが、その逆は行われません。
  4. テスト環境から、特定の産業環境ごとに変更が収集される産業化以前の環境まで、最新の統合テストが実行され、事前に指定された段階的な障害が発生した場合に変更をロールバックするテストも実施されます。
  5. 産業革命以前の環境から産業環境へ。変更をすばやくロールバックできます。

環境間の変更を同期するためのさまざまなツールがあります。次に例を示します。
  1. Visual Studio Comparer
  2. SQL Server統合サービス
  3. DevartによるdbForge Comparer
  4. Quest Software ApexSQL
  5. RedGateソリューション

通常、.NETで記述されたソリューション（たとえば、C＃で記述）は、MS SQL Serverで動作します。

この記事では、Devart、Quest Software、およびRedGateのツールを使用して、あるサーバーから別のサーバーにMS SQL Serverデータベースへの変更を転送する例を検討します。最後に、これらのツールの比較が行われます。

この記事では、例として、MS SQL Server DBMSを提供するように設計されたSRVデータベースについて検討します。ここからソースをダウンロードできます。
このSRVデータベースは、あらゆる目的で自由に配布されます。

ApexSQL Diffツール の公式サイトでは、このプログラムとSQL Compareツールを比較できます。

同様に、ApexSQLデータ差分ツールのために、比較 SQLデータの比較ツールでは、提供されます。

上記の比較から、ApexSQLには何かがCompareにはなく、反対に何かがCompareにはあるがApexSQLにはないことが簡単にわかります。ただし、検討されているすべてのユーティリティには、スキーマとデータベースデータ自体の両方を比較するために必要な基本機能のセットがあります。

機能と価格の観点から、割引や上記で説明した資料を考慮に入れると、ApexSQL DiffおよびApexSQL Data Diff製品が最適です。

結論


データベーススキーマとデータを同期するための一般的なユーティリティが検討されました。
  1. dbForge Schema Compare for SQL Server Devart
  2. dbForge Data Compare for SQL Server Devart
  3. ApexSQL Diff Quest Software
  4. ApexSQL Data Diff Quest Software
  5. SQL Compare RedGate
  6. SQL Data Compare RedGate


これらの製品の価格と機能についても簡単に比較しました。

レビューの結果、価格と機能の最良の選択は、Quest SoftwareのApexSQL DiffとApexSQL Data Diffであるという結論に達しました。ただし、主要な必要な機能はDevart製品で十分に利用できます：dbForge Schema Compare for SQL ServerおよびdbForge Data Compare for SQL Server。これらの製品は、Quest Softwareの対応するApexSQL DiffおよびApexSQL Data Diffよりもはるかに安価に購入できます。

Oracle DBMSにも同様のソリューションがあります。
