AnandTechドライブテスト-駆逐艦


Destroyerは、大量のI / Oでアプリケーションアクセスパターンを複製する非常に長いテストです。ドライブは実際の条件で使用される場合と同様に、バックグラウンドガベージコレクションとキャッシュクリーニングを使用できる短い中断が発生する場合がありますが、これらのダウンタイムは25ミリ秒に制限されているため、1つのテストを実施するために1週間を費やす必要はありません。 AnandTech Storage Bench（ATSB）テストには、ワークロードを生成した実際のアプリケーションの起動は含まれていないため、新しいテストベンチのCPUとRAMのパフォーマンスの変化にはそれほど影響はありませんが、新しいバージョンのWindowsと新しいドライバーへの移行は顕著な影響を与える可能性があります。



このテストの結果を評価するには、平均ディスクスループット、平均I / Oレイテンシ、およびテスト中にドライブが消費した総エネルギーを報告します。



Intel Optane Memory H10は、実際にはOptaneが完全に非アクティブの状態で、キャッシュが無効になっているDestroyerでパフォーマンスが向上します。このテストでは、データ配置のバックグラウンド最適化に時間をかけられず、移動されるデータの総量は32 GB Optaneキャッシュよりもはるかに大きくなります。 512 GBのQLC NANDは、タイムリーなキャッシュクリーニングのパフォーマンスが不足しています。





Optane Memory H10自体のQLC側は、99パーセンタイルの平均レイテンシスコアが低く、キャッシュメモリの不足は状況を悪化させるだけです。 7200RPMのハードドライブでさえ、より優れたパフォーマンスを発揮しました。





Optane Memory H10の平均読み取りレイテンシは、すべてのTLCベースのドライブよりも劣りますが、Optaneキャッシュをその前に備えている、または備えていないHDDよりははるかに優れています。 QLC H10レコードの場合、SLCキャッ​​シュが終了すると、SSDが最後の場所に送信されます。





Optaneのキャッシュは、H10読み取りレイテンシの99パーセンタイルにプラスの効果をもたらし、ドライブをCrucial MX500 SSDに近づけ、より大きな1LC 660p QLCモデルを大幅に上回ります。99パーセンタイルでの書き込みレイテンシはひどいですが、キャッシュオーバーフローが原因で冗長な書き込みが発生しても、H10はDRAMなしのToshiba RC100ほど悪くありません。

AnandTechドライブテスト-ヘビー


重い負荷「重い」のテストでは、「破壊者」よりも比例して大きな記録負荷が得られますが、時間ははるかに短くなります。ヘビーテストで記録されたデータの総量は、ディスクをいっぱいにするのに十分ではないため、パフォーマンスは安定した動作状態に落ちることはありません。このテストは、毎日のエネルギー消費量の指標にはるかに適しています。そのピークパフォーマンスは、ドライブのピークパフォーマンスに大きく影響されます。ヘビーテストの詳細なデータは、AnandTechの対応する記事に記載されています。このテストは2回実行されます。1回は完全に消去されたディスクで、もう1回はディスクを連続記録で満たした後です。



ヘビーテストでは、キャッシュによりIntel Optane Memory H10が明確に加速され、テストが空のディスクで実行された場合、平均データレートがTLCベースの適切なNVMe SSDの範囲にもたらされます。四輪駆動のパフォーマンスは、キャッシュがない場合よりも優れていますが、最終的には、SLCにデータを入力した後のQLC NANDの動作をOptaneで隠すことはできません。 TLCベースのドライブは、QLCドライブと同じくらいいっぱいになると速度が低下することはありません。





空のディスクでテストが実行されている場合のみ、H10の平均および99％のレイテンシは他のTLCドライブとほぼ同じです。ヘビーテストがフルドライブで実行され、フルSLCキャッ​​シュとアイドルOptaneキャッシュがある場合、レイテンシはOptaneキャッシュハードドライブよりもさらに悪くなります。全輪駆動の場合の平均H10遅延は、QLCの一部のみを使用する場合よりもはるかに優れていますが、Optaneキャッシュは遅延の99パーセンタイルをまったく改善しません。





H10の平均読み取りレイテンシは、Heavyテストをフルドライブで実行すると著しく悪化しますが、SATA SSDよりもわずかに優れています。平均書き込み遅延は、QLCが特に悪く見える場所で、整数H10はHDDのそれよりも悪く、Optaneキャッシングが無効になっていると、書き込み遅延はTLC SSDの10倍になります。





99-OptaneキャッシュなしのH10の読み取りレイテンシパーセンタイルは、フルディスクテスト中の深刻な問題ですが、Optaneキャッシュを使用すると、読み取りQoSがSSDの適切な範囲に戻ります。99-記録遅延のパーセンタイルは、Optaneキャッシュがないと悪く見え、それがあるとさらに悪く見えます。

AnandTechドライブテスト-ライト


私たちのライトドライブテストは、デストロイヤーまたはヘビーテストよりも連続セッションが多く、キュー深度が少ないため、これは全体としては断然最短のテストです。これは主にドライブのパフォーマンスにあまり依存しないアプリケーションに基づいているため、テスト結果はアプリケーションの起動時間とファイルのダウンロードを示す可能性が高くなります。このテストは、毎日の使用におけるすべての小さな遅延の合計と見なすことができますが、ダウンタイムが25ミリ秒に短縮された場合、完了までに30分もかかりません。ライトテストの詳細については、AnandTechの対応する記事を参照してください。 ATSBヘビーテストの場合と同様に、このテストは2回実行されます。完全にクリーニングされたドライブ上と、ディスクに順次記録を書き込んだ後です。



Intel Optane Memory H10は、空のドライブでLightテストを実行すると、NVMeローエンドドライブと競合します。 QLC自体の単一部分のパフォーマンスが高いことは、H10全体の評価がおそらくいくらか過小評価されていることを示していますが。フルディスクのパフォーマンスは、すべてのTLCベースのSSDよりも劣りますが、OptaneキャッシュなしのHDDよりもはるかに優れています。





Optane Memory H10の平均および99％のレイテンシはTLC NANDに比べて競争力があり、空のディスクでテストを実行すると、レイテンシインジケーターは機械的なハードディスクよりも優れています。





フルドライブでの平均書き込み遅延は、H10をエントリーレベルより上のNVMeドライブとして分離および識別する唯一のものです。TAMベースのDRAMなしのToshiba RC100は、このシナリオではさらに悪いことがわかりました。





平均遅延とは異なり、Optane H10の読み取りと書き込みの両方の99パーセンタイル遅延は、充填時に重大な問題が発生していることを示しています。OptaneキャッシュはSLCキャッ​​シュの不足を補うのに十分ではありません。

ランダム読み取りパフォーマンス


最初のランダム読み取りパフォーマンステストは、キューなしで一度に1つずつ実行される非常に短い操作のバッチを使用します。ディスクはパッケージ間で十分なダウンタイムが発生するため、全体のデューティサイクルは20％になるため、熱調整は不可能です。各パッケージは、4 MBの32 MBのランダム読み取り、16 GBのディスク上のデータで構成されます。読み込まれるデータの総量は1 GBです。



短いパケットのランダム読み取りテストでは、データはOptane Memory H10のOptaneキャッシュに簡単に収まるため、すべてのフラッシュベースのソリッドステートドライブを超えますが、純粋なOptaneを備えたストレージデバイスよりもはるかに低速です。

長いランダム読み取りパフォーマンス


連続ランダム読み取りのパフォーマンステストは、2015年のテストスイートのテストに似ています。1〜32のキューの深さがチェックされ、QD1、QD2、QD4に基づく平均生産性とエネルギー効率が主な指標として示されます。各キューの深さは、1分間または32 GBの送信データについてチェックされ、より高速です。各キューの深さをチェックした後、ドライブは1分間冷却されます。そのため、熱の蓄積がキューのより深い部分に影響を与えることはほとんどありません。個別の読み取り操作は4 KBであり、64 GBのディスクを占有します。



Optaneキャッシュが処理できるドライブよりも広い範囲にわたる長いランダム読み取りテストにより、H10のパフォーマンスはTLCベースのSSDと同等です。

ランダム書き込みパフォーマンス


短いパケットのランダム書き込みの最初のパフォーマンステストは、読み取りテストと同様に構成されていますが、各パケットのサイズは4 MBのみであり、テストの合計の長さは128 MBです。 4 KBのランダム書き込み操作は16 GBのディスクに分散され、キューなしで一度に1つずつ実行されます。



キャッシングを有効にした短いH10パケットのランダム記録のパフォーマンスは、ディスクのどの半分でも個別に処理できるよりは高くなりますが、2つの部分の合計よりはるかに低くなります。 TLCドライブ上の適切なSLCキャッ​​シュは、QLC上のOptaneキャッシュよりも優れています。

継続的なランダム読み取りテストと同様に、安定した4 KBのランダム書き込みテストは、キューの深さあたり最大1分または最大32 GBで実行され、64 GBのディスクにまたがり、キューの深さの間に最大1分のダウンタイムを与えて、キャッシュのフラッシュを確実にします。そしてディスクを冷却します。



Optaneキャッシュが耐えられるよりもはるかに広い範囲にわたる長いランダム書き込みテストにより、Optane Memory H10は、フラッシュメモリに基づくすべての競合製品に遅れをとっています。キャッシングソフトウェアは、最終的に、SLCキャッ​​シュのみを使用して、QLCパーツ自体よりもはるかに低いパフォーマンスを提供する追加の負荷を作成します。

順次読み取りパフォーマンス


最初のシーケンシャル読み取りパフォーマンステストでは、128 KBの順不同の操作によって発行された128 MBの短いデータパケットを使用します。このテストでは、8パケットのパフォーマンスを平均化し、16 GBのデータを含むディスクから転送されたデータの合計が1 GBになります。各パッケージ間で、ドライブには20％の全体的なデューティサイクルを維持するのに十分なダウンタイムが与えられます。



Optane Memory H10のシーケンシャル読み取りパフォーマンスは、高性能TLCベースのドライブのパフォーマンスよりもはるかに低くなりますが、PCIe3 x2に制限されている低レベルのNVMeドライブに匹敵します。 Optaneのメモリキャッシングは、純粋なQLC速度よりも約10％だけ速度が向上するだけなので、キャッシングドライバーがOptaneとNANDの間でアクセスを効果的に共有できる場合は、明らかにそうではありません。

2番目のテスト-連続順次読み取り-キューの深さは1〜32を使用し、パフォーマンスと消費電力はQD1、QD2、QD4の平均として計算されます。各キューの深さは、64 GBのデータを含むディスクから受信した1分間または最大32 GBのデータについてテストされます。このテストは2回実行されます。1回はテストデータの順次記録によって準備されたドライブから、もう1回はランダム書き込みテストがすべてを混合した後、SSD内の断片化がOSから見えなくなります。これら2つの見積もりは、実際のディスク使用量の2つの極端を表しています。摩耗の分布と既存のデータの変更により、パフォーマンスに悪影響を与える内部の断片化が発生しますが、通常、ここに示す極端な範囲ではありません。



より長いシーケンシャルリードテストでは、Optaneキャッシングは依然としてOptaneとNAND H10パーツのパフォーマンスを効果的に組み合わせません。ただし、順番に書き込まれていないデータを読み取る場合は、Optaneキャッシュが非常に役立ちます。

順次書き込みパフォーマンス


最初の順次書き込みテストのデータパケットは、データ転送の方向を除いて、順次読み取りテストと同じように構成されています。各パケットは、QD1で実行される128 KB操作として128 MBを書き込みます。合計1 GBのデータが、16 GBのデータを含むディスクに書き込まれます。



Optaneパーツ自体の短いパケットのシーケンシャル書き込み速度は非常に遅いため、QLC NAND側がOptane H10を大きく支援する場合がこれに該当します。したがって、QLC H10はTLCベースのドライブと競合しますが、キャッシュソフトウェアがH10に干渉すると、SATAレベルのパフォーマンスが得られます。

連続順次書き込みテストは、データ転送の方向を除いて、同じ読み取りテストと同じように構成されています。キューの深さは1〜32であり、各キューの深さは1分間または最大32 GBの転送データについてチェックされ、次にディスクが冷却されてガベージを収集するときに最大1分のダウンタイムがチェックされます。テストは、64 GBのディスク容量によって制限されます。



状況は前のテストとほぼ同じですが、低レベルのNVMeが非常に低くなり、Optane Memory H10のスコアがそれほどひどく見えなくなりました。ただし、QLC部分自体は、キャッシュされた構成よりも連続シーケンシャルレコーディングの方が優れています。
