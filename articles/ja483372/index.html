GPUを使用してラップトップでDeepPavlovニューラルネットワークをトレーニングする方法

この記事では、専門家の履歴書を含むPDF / DOC / DOCXファイルを解析する1つのタスクの実装例で、このオープンソースライブラリを使用した経験を共有したいと思います。

ここでは、データセットを準備するためのツールの実装の段階についても説明します。次に、テキストからエンティティを認識するタスクの一部として、受信したデータセットでBERTモデルをトレーニングすることができます（名前付きエンティティ認識 -以下NER）。

だから、どこから始めるか。当然、最初に、ツールを実行するには、環境をインストールして構成する必要があります。Windows 10にインストールします。

Habréには、このライブラリの開発者からの記事がすでにいくつかあり、詳細なインストールガイドしかありません。そして、この記事では、ローンチからモデルトレーニングまで、すべてをまとめたいと思います。また、このライブラリで作業中に発生した問題のいくつかの解決策を示します。
重要：多くの場合、互換性のないバージョンに問題があるため、インストールするときは、すべての製品とコンポーネントのバージョンに準拠することが重要です。これは特にTensorFlowライブラリに当てはまります。一部のタスクでは、GitHubでの必要なコミットまで、それを使用する必要がある場合もあります。DeepPavlovの場合、サポートされているバージョンのみに準拠すれば十分です。

動作構成の製品バージョンと、ニューラルネットワークのトレーニングプロセスを開始したラップトップの仕様を示します。また、オープンソースのDeepPavlovライブラリのインストールと構成を説明するリンクもいくつか提供します。

REST API上のDeepPavlov


REST APIのサービスを開始してインストールするには、次のコマンドを実行する必要があります。

  1. アクティブな仮想環境にインストールする 

    python -m deeppavlov install ner_ontonotes_bert_mult
  2. DeepPavlovサーバーからner_ontonotes_bert_multモデルをダウンロードします 

    python -m deeppavlov download ner_ontonotes_bert_mult
  3. REST APIを実行する 

    python -m deeppavlov riseapi ner_ontonotes_bert_mult -p 5005

このモデルはhttp：// localhost：5005で入手できます。ポートを指定できます。

すべてのモデルは、デフォルトで途中でダウンロードされます。

C:\Users\<_>\.deeppavlov

トレーニングのためのDeepPavlovのセットアップ


学習プロセスを開始する前に、ビデオカードのメモリがいっぱいであるというエラーで学習プロセスが「クラッシュ」しないように、DeepPavlovを構成する必要があります。このために、モデルごとに構成ファイルがあります。

開発者の例のように、ner_ontonotes_bert_multモデルも使用します。DeepPavlovのすべてのデフォルト構成は、パスに沿って配置されています。

<_>\env\Lib\site-packages\deeppavlov\configs\ner

私の場合、ファイルの名前はner_ontonotes_bert_mult.json modelのようになります。

私のラップトップ構成では、trainブロックのbatch_size値を4 に変更する必要がありました。 そうしないと、ビデオカードが数分後に「チョーク」し、学習プロセスが失敗します。

重要：文の終わりの後には改行が必要です。オファーに75を超えるトークンが含まれている場合は、改行する必要があります。そうしないと、モデルを学習するときにプロセスが失敗します。

DeepPavlovトレーニング


だから私たちは最も興味深い部分に行きました。学習プロセスでは、最初にner_ontonotes_bert_multモデルをダウンロードする必要があります。まだダウンロードしていない場合は、DeepPavlovセクションから上記のREST APIまでの最初の2つのステップを実行する必要があります。

学習プロセスを開始する前に、2つのステップを完了する必要があります。

  1. トレーニング済みモデルのあるフォルダーを完全に削除します。

    C:\Users\<_>\.deeppavlov\models\ner_ontonotes_bert_mult

    このモデルは別のデータセットでトレーニングされたため。
  2. 準備したデータセットファイルtrain.txt、valid.txt、test.txtをフォルダーにコピーします 

    C:\Users\<_>\.deeppavlov\downloads\ontonotes

これで、学習プロセスを開始できます。

トレーニングを開始するには、次の形式の簡単なtrain.pyスクリプトを記述できます。

from deeppavlov import configs, train_model
ner_model = train_model(configs.ner.ner_ontonotes_bert_mult, download=False)

またはコマンドラインを使用します：

python -m deeppavlov train <_>\env\Lib\site-packages\deeppavlov\configs\ner\ner_ontonotes_bert_mult.json



結果


サイズが115,540トークンのデータセットでモデルをトレーニングしました。このデータセットは、100人の従業員の履歴書ファイルから生成されました。学習プロセスには5時間18分かかりました。

モデルの意味は次のとおりです。

  • 精度：76.32％;
  • リコール：72.32％;
  • FB1：74.27;
  • 損失：5.4907482981681826;

データセットの自動生成に関するいくつかの問題を編集した後、私は以下の損失を受けました。しかし、一般的に、私は結果に満足しました。もちろん、このライブラリの使用についてはまだ多くの質問があります。ここで説明したのは、ほんの数滴です。

そのシンプルさと使いやすさから、ライブラリは本当に気に入りました。少なくともNERタスクの場合。このライブラリの他の機能についてお話しできることをうれしく思います。また、この記事の資料が役立つと思います。
