Elementary、Watson：Voximplantと統合します


自然言語（NLU、NLP）での作業は、ITの巨人の間の激しい競争の領域です。だけでなく、AIの開発。まあ、それに応じて、これらのセットの交差点もホットなニッチであり、新しいものを観察して学ぶのは興味深いことです。Voximplantは長い間Google Dialogflowと親しくなり、この統合Dialogflowコネクタのラッパーを作成しました。IBMにはアナログであるWatsonベースの音声エージェントがあり、クライアントとの柔軟で適切な通信を自動化する同じタスクを実行して、クラシックIVRを置き換えます。簡単なデモでこのテクノロジーを試してみましたが、今日は段階的にそれを行う方法も説明します。その間、あなたはこれを読むでしょう、私たちの開発者はこの統合のためのラッパーを作り続けます...

私たちは何をしますか？


ロケットサイエンスはありません。私たちが呼び出すシンプルなボットを作成します。彼は挨拶し、2つのオプションを提供します。1つは会話をループし、もう1つは会話を終了し、ボットは別れを告げます。ニュアンス：執筆時点では、ワトソンは4つの言語（英語、アラビア語、ポルトガル語、中国語（簡体））しかサポートしていません。そのため、デモは英語で最も馴染み深いものになります。

リソースの作成


IBM Cloudアカウントをお持ちでない場合は、こちらから登録してください。アカウントに移動し、右上隅にある[リソースの作成]をクリックすると、リソースディレクトリが開きます。左側で「Webおよびアプリケーション」カテゴリーを選択し、次に画面の右側で「Watsonを使用した音声エージェント」をクリックします。


デフォルトではすべてのフィールド（サービス名、地域など）をそのままにしておくことができます。画面下部にある[作成]をクリックしてお茶を入れます（これは冗談です。IBMクラウドは常にすぐに向きを変えるわけではないためです）。リソースが作成されると、この新しく作成されたリソースのダッシュボードが表示されます。ここでは、リソースの名前、場所（ワシントンDC）、およびリソース管理メニューが表示されます。これは左側にあります。まもなく、このメニューからManageアイテムが必要になりますが、最初に購入する必要があります...

ボキシムプラント番号


エージェントは、彼に連絡できる番号を必要とします。これを行うには、Voximplantパネルの[番号]-> [私の電話番号]セクションに移動し、右上隅にある[新しい電話番号を購入]をクリックします。私たちの目的には、仮想番号が適しています-テスト番号スイッチを選択し、リストから1つの番号を確認して、下部にある[選択した購入]をクリックし、開いたウィンドウで[購入]をクリックします。


この番号は、番号のリストにすぐに表示されます。それをコピーして、IBM Cloudに戻り、Watsonリソースを備えた音声エージェントに戻します。

エージェントの作成


ヒント。リソースのあるタブを閉じた場合、メインページからリソースをすばやく開く方法は次のとおりです。cloud.ibm.com ダッシュボードで、「サービス」という単語をクリックすると、アカウントの現在のリソースのリストが開きます。


この場合、Servicesグループはすでに展開されており、その中にVoice Agentが表示されます。それをクリックしてください-乾杯、あなたは再び希望のリソースを開きました。



左側のメニューで[管理]を選択し、画面の右側にある[音声エージェントを作成]ボタンをクリックします。エージェントの名前を入力し、購入した仮想番号を挿入して、下にスクロール-音声エージェントを作成します。


IBM Cloudは少し考え、エージェントと3つの組み込みサービスの作成が成功したことについてメッセージを表示します。さて、今私たちはエージェントに方法を教える必要があります...

革のバッグと話す


左上隅のハンバーガーアイコンをクリックし、[リソースリスト]を選択します。リソースのリストで、VoiceAgent-WatsonAssistant（リストの一番下）を選択します。コントロールパネルで、起動ツールをクリックします。Watson Assistant Control Panelが開きます。上の[スキル]タブを選択します。VoiceGatewayConversationスキルが既にあることがわかります。これは、エージェントの作成時にウィザードが追加した一連のフレーズとダイアログのフローチャートです。このスキルは使用できますが、興味を引くために自分で作成することをお勧めします。

これを行うには、[スキル]タブで[スキルの作成]をクリックします。「simple-skill」という名前を付け、言語を英語（米国）のままにして、[Create dialog skill]をクリックします。


スキル内には3つのタブが必要です。
  • 意図-クライアントの意図。本質的に、これらはフレーズとそのバリエーションです。
  • エンティティ-認識のためのキーワードとその同義語。
  • ダイアログ-ダイアログのフローチャート。

[Intents]タブで、ステートメント#whatcanidoを追加し、いくつかのオプションを記述します（「何ができるか」、「オプションを表示」など）。

「エンティティ」タブで、次を追加します。
  • 継続する 同義語は「続行」、「続行」、
  • オプション 同義語は「オプション」、「オプション」、「選択肢」。
  • やめる 「停止」、「終了」の同義語。

[ダイアログ]タブで[作成]ダイアログをクリックすると、[ようこそ]ブロックとその他のブロックが表示されます。それ以外のものを開き、スクリーンショットのように設定を行います（テキスト：「この無意味な会話を続けるか、停止するかを選択できます。続行するか、停止するかを選択してください。」）。なお、認識条件にはインテントだけでなく、セキュリティのためのキーワードも追加しています。


次に、Anything elseブロックの3つのポイントをクリックし、[子ノードの追加]を選択して、ブロックを作成します 継続する。他のものの中に作成し、別の子ブロックを作成します-やめる。ブロック設定：



したがって、ボットはユーザーが「続行」と言っている間、選択に関するテキストを無限に繰り返すか、ユーザーが「停止」と言ったときにダイアログを終了します。確かに、無意味な会話。
これらすべてを行うのが面倒な場合は、jsonファイルをダウンロードしてスキルとしてインポートできます。これを行うには、[スキル]タブで[スキルの作成]をクリックし、[スキルのインポート]タブに切り替え、[JSONファイルの選択]をクリックし、ダウンロードしたファイルを指定して、最後に[インポート]をクリックします。

最後になりますが、この段階では重要ではありません。cloud.ibm.comに戻り、リソースリストを開き、Watsonを使用して音声エージェントをクリックし、[管理]タブに移動します。habr-watsonエージェントで、3つのポイントをクリックして、[エージェントの編集]を選択します。会話セクションで、作成したスキルを示すようにスキル名を変更します。

Voximplantアプリとスクリプト


でVoximplantコントロールパネル、ワトソンアプリケーションを作成します。次の手順はすべて、このアプリケーション内で実行する必要があります。「シナリオ」タブに移動し、コードを使用してwatson-scenarioを作成します。
require(Modules.ASR)

VoxEngine.addEventListener(AppEvents.CallAlerting, (e) => {
  let call2 = VoxEngine.callSIP("sip:699100484@us-east.voiceagent.cloud.ibm.com")

  const recognition = VoxEngine.createASR({
    lang: ASRLanguage.ENGLISH_US
  })
  let botSpeech = ""

  recognition.addEventListener(ASREvents.Result, e => {
    botSpeech += e.text
    botSpeech.includes("goodbye") ? VoxEngine.terminate() : Logger.write("There is no 'goodbye' yet.")
  })

  call2.addEventListener(CallEvents.Connected, () => call2.sendMediaTo(recognition))

  VoxEngine.easyProcess(e.call, call2)

})

callSIPメソッドの代わりに、以前に購入し、IBMエージェントの設定で指定した電話番号をそのまま呼び出すことを忘れないでください。

次に、「ルーティング」タブに移動して、ワトソン規則を作成します。watson-scenarioスクリプトを指定します。

最後に、[番号]タブに移動します。ここでは、添付されたセクション（現時点では空）と[利用可能]があります。利用可能に切り替え、購入した番号をマークして、[添付]をクリックします。


開いたウィンドウで、watson-ruleを指定し、次にAttachを指定します。


これで、番号が[添付]セクションに表示されます。ここにあなたが呼び出すことができる番号が表示され、購入した仮想番号をトーンモードで入力し、IBMボットと非常にインテリジェントな会話を開始します。
