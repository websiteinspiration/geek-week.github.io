Yiiフレームワーク2カスタム権利

Yiiフレームワーク2-データベースを介したユーザー権限。

すべてのコントローラーが継承されるコントローラーとbeforeAction-アクションの前に実行されるメソッドを作成します。

PS：モデルとCRUDは、Giiモジュールを使用して生成する必要があります。

<?php

/*
 * To change this license header, choose License Headers in Project Properties.
 * To change this template file, choose Tools | Templates
 * and open the template in the editor.
 */
namespace app\helper;

use yii\web\Controller;
use app\models\Users;
use app\models\Rights;

use Yii;

class AppController extends Controller{
    
    public function beforeAction($action){
        $controller = $action->controller->id;
        //$action     = $action->id;
        
        
        
        
        
        $rights     = Rights::find()->where('controller = :controller', [':controller' => $controller])->one();
        
        
        if(is_null($rights)){
            echo 'controller dont have a rights';
            exit;
        }
        
        
          
        $roles  = explode(",",$rights->rights);
        
        $access = false;
        
        $userRole = "guest";
        
        if(!\Yii::$app->user->isGuest){
            $id       = \Yii::$app->user->id;

            $user     = Users::find()->where('id = :id', [':id' => $id])->one();
            
            $userRole = $user->righgts;
        }
        
        foreach ($roles as $role){
            if($role == $userRole){
                $access = true;
            }
        }
        
        if(!$access){
            $this->redirect(array('/site/login'));
            exit;
        }

        return parent::beforeAction($action);
        
    }
}

モデルと権利の表：

CREATE TABLE `rights` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `controller` varchar(255) NOT NULL,
  `action` varchar(255) NOT NULL,
  `rights` varchar(255) NOT NULL,
  PRIMARY KEY (`id`)
) ENGINE=MyISAM AUTO_INCREMENT=32 DEFAULT CHARSET=utf8

モデルとユーザーの表の特徴は、riggsフィールドがあることです。

CREATE TABLE `users` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `email` varchar(255) NOT NULL,
  `phone` varchar(22) NOT NULL,
  `password` varchar(64) NOT NULL,
  `righgts` varchar(60) NOT NULL,
  `verify_key` varchar(65) NOT NULL,
  `activated` int(11) NOT NULL,
  `recovery_key` varchar(65) NOT NULL,
  `recovery_key_life` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `auth_key` varchar(255) NOT NULL,
  `accessToken` varchar(255) NOT NULL,
  PRIMARY KEY (`id`)
) ENGINE=MyISAM AUTO_INCREMENT=37 DEFAULT CHARSET=utf8

usersテーブルでは、コントローラのアクションへのアクセス権を担当する権限フィールドを変更することができます。



権限テーブルには、コントローラー、アクション、およびそれらにアクセスできるロール（権限、権限）を記述します。



アルゴリズム：

  1. 親コントローラーは、データベースからのユーザー権限をチェックします（権限列）。
  2. 親コントローラーは、呼び出されたコントローラーの名前と呼び出されたアクションの名前を取得します。
  3. 親コントローラーは権限テーブルにアクセスし、呼び出されたコントローラーとアクションを使用するために必要な権限を確認します。
  4. ユーザー権限が権限テーブルの権限と一致する場合、アクションが実行され、一致しない場合、リダイレクトが実行されます。

権利をカンマで区切って権利テーブルにリストします。

管理者モジュールでこの方法を使用することをお勧めします。ユーザー部分の標準権限が適しているためです。
