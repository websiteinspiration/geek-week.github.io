ユニバーサルRESTサービスの実装例（Avalanche-Javaのアプリケーションフレームワーク）


「なだれ-Javaのアプリケーションフレームワーク」は、
ローカルコードコールとリモートコードコールの違いを解消するテクノロジの実装です。フォールトトレランス、スケーラビリティ、
変更可能性、継続的な可用性が素晴らしいボーナスにバンドルされています。


この記事では、ユニバーサルRESTサービスの実装例について説明します。サービスを実装するという考えは、FrontEndが常に、操作するデータと、BackEndから要求するデータを「認識」しているというアサーションに基づいています。このステートメントにより、バックエンドでのORMの開発と、ユニバーサルサービスの実装でのJPAの使用を放棄することができました。サービスは、データベーステーブルの内容を操作できます。構造の変更または作成されたデータベースオブジェクトは、すぐにサービスメソッドで使用できるようになります。


サービスは、3つのグループのメソッドを実装します。


  • ... / data / table / ...-データベーステーブルの内容を操作します。
  • ... / data / query / ...-サービスの機能を拡張する任意のクエリの実行。
  • ... / data / info / ...-データベースオブジェクトの構造に関する情報を取得します。

サービスは、独自にエラーの原因と場所を理解できるエラーシステムを実装します。デフォルトでは、ruおよびenメッセージのローカリゼーションがサポートされています。ランタイムのローカライズに依存するSQLクエリ実行エラーメッセージなど、すべてのメッセージをローカライズできるわけではありません。


このサービスは、XMLおよびJSON形式のデータの操作をサポートしています。


このサービスは、Jersey 2.0ライブラリを使用して実装されています。


一般に、サービスリクエスト文字列のURL形式は次のようになります。


http://host:port/name_context/map_servlet/data/type_group/{object}[?request_parameters]
