nginx経由のプロキシでWebogramインスタンスを発生させます

こんにちは、ハブル！

最近、インターネットへのアクセスが不完全な企業ネットワーク内で作業する必要があり、見出しから推測できるように、テレグラムがその中でブロックされている状況になりました。この状況は多くの人にとって身近なものだと思います。

メッセンジャーがいなくても大丈夫ですが、仕事に必要なのはテレグラムでした。稼働中のマシンにクライアントをインストールすることも、個人のラップトップを使用することもできませんでした。もう1つの解決策は、公式のWebバージョンを使用することですが、ご想像のとおり、これも使用できませんでした。非公式なミラーを検索して、オプションをすぐに削除します（明らかな理由を期待しています）。

幸い、Webogramはオープンソースプロジェクトであり、そのソースコードはgithubから入手できます。その作者（
多くの方に感謝します！）インストールと起動自体は大した問題ではありませんが、テレグラムサーバーへのアクセスがブロックされたネットワーク内の動作条件下では、Webバージョンがユーザーのマシンからの電報サーバー。

幸いなことに、これは修正するのが非常に簡単です（しかしそれほど明白ではありません）。私はこのソリューションの作成者ではないことを警告します。私と同じような問題について話し合っているスレッドでなんとか見つけました。ユーザーgithub tecknojockによって提案された解決策は私に大いに役立ちましたが、私は他の誰かを助けることができると確信しているので、このチュートリアルを書くことにしました。

猫の下には、ngogramxを使用したテレグラムサーバーへのWebogramミラーとそのリクエストのプロキシ設定の段階的な設定があります。

例として、インストールして更新したばかりのUbuntu Server 18.04.3を選択しました。

注：このチュートリアルでは、nginxでドメインを設定する手順はありません。これは個別に行う必要があります。このチュートリアルでは、sslを使用してドメインをすでに構成しており、構成する予定のサーバーがテレグラムサーバーに（任意の方法で）アクセスできることを前提としています。

このサーバーのIPが10.23.0.3で、ドメイン名がmywebogramであるとします。 localhost

これらの規則に基づいて、構成の例を示します。値を自分のものに変更することを忘れないでください。

だから

ここに行きます：Webogramを実行するには、nodejsが必要です。デフォルトでは、Ubuntuリポジトリからインストールすると、nodejsバージョン8.xが取得されます。 12.xが必要です。

curl -sL https://deb.nodesource.com/setup_12.x | sudo -E bash - 
sudo apt update && sudo apt -y install nodejs

Webogramのベースとなる場所を選択します。

例として、ホームディレクトリのルートに配置します。これを行うには、公式リポジトリをサーバーに複製します。

cd ~ && git clone https://github.com/zhukov/webogram.git

次のステップでは、アプリケーションの実行に必要なすべての依存関係をインストールします。

cd webogram && npm install

テストランを実行してみましょう。次のコマンドを実行します。

npm start

その後、ブラウザで開こうとします 

 http://10.23.0.3:8000/app/index.html

この時点までにすべてを正しく行った場合、Webogram承認ページが開きます。

次に、アプリケーションの起動をサービスとして構成する必要があります。これを行うには、ファイルを作成します

sudo touch /lib/systemd/system/webogram.service

任意のエディターで開き、次のような外観にします（WorkDirectoryへのパスを入力します）。

[Unit]
Description=Webogram mirror
[Service]
WorkingDirectory=/home/tg/webogram
ExecStart=/usr/bin/npm start
SuccessExitStatus=143
TimeoutStopSec=10
Restart=on-failure
RestartSec=5
[Install]
WantedBy=multi-user.target

次に、次のコマンドを実行します。

変更を適用します

sudo systemctl daemon-reload

自動起動をオンにします。

sudo systemctl enable webogram.service

サービスを開始します。

sudo systemctl start webogram.service

手順が完了した後も、Webogramは引き続きポート8000​​で

使用できます。nginxを介してWebogramへのアクセスを構成するため、外部からの要求に対してポート8000​​を閉じます。

これには、udfユーティリティ（または便利な方法）を使用します。

sudo ufw deny 8000

まだudfを使用することに決めたが、サーバーでオフになっている場合-ルールを追加して（すべてがバラバラにならないように）、udfをオンにします。

sudo ufw allow ssh
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable

nginx設定に追加するもの：

  • ルートの場所を変更し、Webogramが応答するポート8000​​にリクエストをプロキシします。
  • 基本認証でルートの場所を閉じます。これは、アプリケーションを詮索好きな目やボットから閉じるための純粋に象徴的なステップです。（そしてロックにも問題がなかったことも）
  • Telegramサーバー上のproxy_pathを持つロケーションヒープは、リクエストをプロキシするエンドポイントにすぎません

また、ファイルを作成し/etc/nginx/passwd.htpasswd;て、nginxがユーザーパスワードと比較するものを含むようにします。

sudo apt install apache2-utils
sudo htpasswd -c /etc/nginx/passwd.htpasswd tg



nginxを再起動します。

sudo systemctl restart nginx

これで、htpasswdコマンドの作成時に定義したログインとパスワードを入力すると、Webogramはmywebogram.localhost / app / index.htmlでのみ利用できるようになります。
