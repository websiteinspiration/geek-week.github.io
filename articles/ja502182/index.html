再びMikroTikまたは待望のSOCKS5について



ユビキタスなMikrotikに飽きたことはよくわかりますが、とても興味深いです。実際、記事は4か月遅れて作成されました（2020年1月にSOCKS5のサポートが登場しました）。

それでは始めましょう！

バックグラウンド


Mikrotikは非常に強力なツールであり、（IMHO）はあらゆるネットワークエンジニアのファームに配置する必要があります。巨大な機能と一連のツール。リリースごとに常に補充されています。RouterOS 6.47（ベータ）の変更ログは、SOCKS5のサポートに気づきました。これは、私が数年間待っていた（そして、他の多くの場合、forum.mikrotik.comの多数の「機能リクエスト」から



判断すると）：前述のフォーラムのブランチの1つによる判断、組み込みのサポートSOCKS5サーバーはバージョン6.47beta19で起動しました。

カスタマイズ


MikroTikでいつものように、セットアップは非常にシンプルで、4行に収まります。
6.47beta53に更新されたCHR（Cloud Hosted Router）を備えた仮想サーバーがテスト対象として機能しました。

RouterOSバージョンのベータプレフィックスは、[at] mikrotik.comをサポートするバグレポートのテストと送信のみを目的としていることを意味します。負荷のかかっているハードウェアを急いでアップグレードせずに、長期または少なくとも安定したリリースを待つことを強くお勧めします。アップグレードする場合は、変更ログをよく読んで、すべての変更と考えられる問題を確認してください。

SOCKS5サーバーを構成し、パスワードによる認証を選択し（それを開くことは可能です）、ボットネットスキャンリストに入らないように3327（標準1080以外）をポートとして指定します。

ip socks set enabled=yes version=5 port=3327 auth-method=password

次に、ユーザーを追加します。

ip socks users add name=test_socks5 password=Str0nGP@SSw0RD

ファイアウォールで例外ルールを作成します。

ip firewall filter add chain=input protocol=tcp dst-port=3327 action=accept comment="SOCKS5 TCP"

そして、「ドロップ」タイプのルールの前にルールを移動します。

ip firewall filter move [/ip firewall filter find comment="SOCKS5 TCP"] 0

できた！私たちのMikrotikでは、SOCKS5プロキシサーバーが発生しています。
