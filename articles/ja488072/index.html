Koin Dependency InjectionまたはService Locatorはありますか？

前書き


従来のDI用のAndroid開発では、非常に強力なコード生成フレームワークであるDagger 2を使用しています。しかし、問題があります。初心者が使用するのは難しいです。DIの原則自体は単純で単純ですが、Daggerはそれらを複雑にします。プログラマーのリテラシーの絶対的な低下について文句を言うことができますが、問題はこれから消えることはありません。


Kotlinの出現により、Javaを使用してはほとんど不可能であった便利なものを書くことが可能になりました。これらのものの一つでした光陰として多くの人に扱われるDIないが、しかし、サービスロケータ、アンチパターン多くの人々は、原則的にそれを使用していない理由です。ただし、コードの記述と保守を簡素化する非常に簡潔なAPIがあるため、無駄です。


この記事では、初心者が依存性注入とService Locatorの違いを理解できるようにしますが、Koin自体はわかりません。


依存性注入


まず、Dependency Injectionとは何ですか？簡単に言うと、これはオブジェクトがそれ自体を作成または抽出するのではなく、外部から依存関係を受け入れるときです。例を挙げましょう。インターフェースEngine、その実装、およびにCar依存するクラスがあるとしEngineます。DIがないと、次のようになります。


interface Engine
class DefaultEngine: Engine

class Car {
  private val engine: Engine = DefaultEngine()
}

fun main() {
  val car = Car()
}

CarDIアプローチを使用してクラスを書き直すと、これが発生する可能性があります。


class Car(private val engine: Engine)

fun main() {
  val car = Car(DefaultEngine())
}

それは単純です-クラスCarは実装がどこから来るのかを知りませんが、Engineこの非常に実装を置き換えるのは簡単です。それをコンストラクタに渡すだけです。

interface ServiceLocator {
  fun <reified T> register(factory: () -> T)
  fun <reified T> resolve(): T
}

class DefaultServiceLocator: ServiceLocator { ... }

class Car(private val serviceLocator: ServiceLocator) {
  private val engine = serviceLocator.resolve<Engine>()
}

fun main() {
  val serviceLocator = DefaultServiceLocator()
  serviceLocator.register<Engine> { DefaultEngine() }
  val car = Car(serviceLocator)
}

, Car , . .. . :


class Car(private val engine: Engine)

fun main() {
  ServiceLocator.register<Engine> { DefaultEngine() }
  val car = Car(ServiceLocator.resolve<Engine>())
}

これは、純粋な依存性注入です。Koinを使用すると、次のようになります。


interface Engine
class DefaultEngine: Engine
class Car(private val engine: Engine)

fun carModule() = module {
  factory<Engine> { DefaultEngine() }
  factory { Car(get<Engine>()) }
}

fun main() {
  val koin = startKoin {
    modules(carModule())
  }.koin

  val car = koin.get<Car>()
}

残念ながら、依存関係についてはKoinに問い合わせる必要がありますが、これは依存関係注入の原則と矛盾するものではありません。


更新 リクエストにより悪党 Dagger 2の最も単純な例を示します。


interface Engine
class DefaultEngine: Engine
class Car @Inject constructor(private val engine: Engine)

@Module
class AppModule {
  @Provides
  fun provideEngine(): Engine = DefaultEngine()
}

@Component(modules = [AppModule.class])
interface AppComponent {
  fun car(): Car
}

fun main() {
  // DaggerAppComponent –  ,   Dagger
  val daggerComponent = DaggerAppComponent.create()
  val car = daggerComponent.car()
}
