あなたが見逃した10の便利なAngular機能


Angularで定期的にアプリケーションを作成し、それを100時間以上費やしている場合、これらの関数について知っている可能性が最も高くなります。しかし、それでも、確かに-読んでください:)

Angularの機能に進む前に、1つの非常に便利なツールについて言及する価値があります。ビット（Github）を使用すると、Angularコンポーネントを一緒に操作して簡単に共有できます。一貫したインターフェースを維持し、開発をスピードアップし、コード内のエラーの数を最小限にすることを心からお勧めします。


例：bit.devを使用したAngularの循環ローダー


1.タイトル


titleタグは、Webページのタイトルを定義するHTML要素です。見出しは、検索結果ページ（SERP）に結果のクリック可能なタイトルとして表示されます。これらは、ユーザビリティ、SEO、ソーシャルプロモーションにとって非常に重要です。ネットワーク。

Angularアプリケーションは、index.htmlのタグ...からタイトルを取得します。デフォルトでは、Angularのコンポーネント間の遷移はこのヘッダーを変更しません。

タイトルを各コンポーネントとは別に設定できることをご存知ですか？

AngularにはTitleサービスがあります角度のある/ platform-b​​rowser。他のコンポーネントに簡単に埋め込むことができ、setTitleメソッドを使用してタイトルをオーバーライドできます。

import { Title } from "@angular/platform-browser"@Component({
   ...
})
export class LoginComponent implements OnInit {
   constructor(private title: Title) {}    ngOnInit() {
       title.setTitle("Login")
   }
}

アプリケーションのLoginComponentに移動すると、ブラウザウィンドウのタイトルが「ログイン」に変わります。

同じことはどのコンポーネントでも行うことができるため、アプリケーション内を移動すると、ユーザーのアクションに応じてタイトルが変化します。

2.メタ


Angularアプリケーションのメタタグもindex.htmlファイルから取得されます。そして、それらをコンポーネントから設定するには、角度のある/ platform-b​​rowser。

繰り返しになりますが、メタタグの使用は、ソーシャルでのSEOやページプロモーションの点で役立ちます。ネットワークには、検索エンジンが検索結果で正しくランク付けして正しく表示するために使用するWebページに関する情報が含まれています。

メタタグコンポーネントを使用するには、からメタをインポートします角度のある/ platform-b​​rowserと目的のコンポーネントに埋め込みます。

import { Meta } from "@angular/platform-browser"@Component({
    ...
})
export class BlogComponent implements OnInit {
    constructor(private meta: Meta) {}    ngOnInit() {
        meta.updateTag({name: "title", content: ""})
        meta.updateTag({name: "description", content: "Lorem ipsum dolor"})
        meta.updateTag({name: "image", content: "./assets/blog-image.jpg"})
        meta.updateTag({name: "site", content: "My Site"})
    }
}

BlogComponentがTwitter、Facebookなどで正しく表示できるようになりました。同時に、説明、見出し、および画像を正しい、読みやすい形式で表示します。
これも聞いたことがありますか？次に、もっと興味深いものがあります。

3.パターン補間のオーバーライド


パターンの標準の補間子は{{}}です。{{and}}の間に変数を入力すると、その値が最終的なDOMに表示されます。

任意の文字の標準のカプセル化区切り文字を上書きできることを知っていますか？それは簡単です。コンポーネントデコレータの内挿プロパティで新しい値を指定するだけです。

@Component({
   interpolation: ["((","))"]
})
export class AppComponent {}

AppComponentテンプレートで使用される補間が（（））に変更され、{{}}が機能しなくなりました。

@Component({
   template: `
       <div>
           ((data))
       </div>
   `,
   interpolation: ["((","))"]
})
export class AppComponent {
   data: any = "dataVar"
}

ブラウザでは、（（data））の代わりに文字列「dataVar」が表示されます。

4.場所


位置情報サービスを使用して、ブラウザで開いているページの現在のURLを取得できます。使用されているLocationStrategyに応じて、場所はURL内のパス（/ example / page /）またはURLのポンド記号の後の部分（＃test / page /）のいずれかによって格納されます。

ロケーションを使用すると、URLに移動したり、ストーリーを進めたり、戻ったり、現在のURLを変更したり、ストーリーの一番上のアイテムを置き換えたりできます。

CommonModuleのLocationサービスが役立ちます。

import { Location } from "@angular/common"@Component({
   ...
})
export class AppComponent {
   constructor(private location: Location) {}    navigateTo(url) {
       this.location.go(url)
   }    goBack() {
       location.back()
   }    goForward() {
       location.forward()
   }
}


5.ドキュメント


Angularアプリケーションから直接DOM操作を実行するために、ドキュメントモデルを取得する必要がある場合があります。

DOCUMENTはこの目的を果たすだけです。DOCUMENTは、メインのレンダリングコンテキストを表すDIトークンです。ブラウザでは、これはDOMドキュメントです。ランタイムに関係なくDOM操作を提供します。

注：アプリケーションとレンダリングのコンテキストが一致しない場合（アプリケーションをWebワーカーで起動するときなど）、ドキュメントはアプリケーションコンテキストで使用できない場合があります。

ある種のHTML要素があるとしましょう：

<canvas id="canvas"></canvas>


キャンバスHTMLElementを取得するには、DOCUMENTを実装します。

@Component({})
export class CanvasElement {
   constructor(@Inject(DOCUMENT) _doc: Document) {}
}


getElementById（）を使用してHTMLElementキャンバスを取得します

@Component({})
export class CanvasElement {
   constructor(@Inject(DOCUMENT) _doc: Document) {}    renderCanvas() {
       this._doc.getElementById("canvas")
   }
}


警告：ドキュメントは慎重に使用してください！DOMとの相互作用は直接危険であり、XSSのリスクを高めます。

6.デコレーター@Attribute


基本的に、Angularアプリケーションでは、コンポーネント、モジュール、およびディレクティブデコレーターを使用します。

ただし、不要なパフォーマンスオーバーヘッドなしに静的文字列を渡して、変更チェックをバイパスできる属性デコレータもあります。

属性デコレータの値は一度だけチェックされます。デコレータのように使用入力：

@Component({
   ...
})
export class BlogComponent {
   constructor(@Attribute("type") private type: string ) {}
}


7. HttpInterceptor


HttpInterceptorはAngularの非常に強力な機能で、HttpRequestリクエストをインターセプトして処理できます。

ほとんどのインターセプターは、next.handle（transformedReq）を呼び出して、チェーン内の次のインターセプターに渡す前に、アウトバウンド要求を変換します。

まれに、インターセプターがリクエスト自体を完全に処理し、チェーンの残りの部分をそれ以上委任できない場合があります。この動作は許可されています。

HttpInterceptorは、以下で最も頻繁に使用されます。

  • 認証
  • キャッシング
  • 偽のバックエンド
  • URL変換
  • ヘッダーSubversion

このインターセプターを使用するには、サービスを作成し、HttpInterceptorインターフェースを実装する必要があります。

@Injectable()
export class MockBackendInterceptor implements HttpInterceptor {
   constructor() {}    intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
       ...
   }
}

メインモジュールに挿入します。
@NgModule({
   ...
   providers: [
       {
           provide: HTTP_INTERCEPTORS,
           useClass: MockBackendInterceptor,
           multi: true
       }
   ]
   ...
})
export class AppModule {}


8. AppInitializer


Angularアプリケーションを起動するときに、特定のコードを初期化する必要がある場合があります。たとえば、設定の読み込み、キャッシュの読み込み、構成の読み込み、チェックの実行などです。AppInitialzerトークンは、このために特別に作成されます。
APP_INITIALIZER：アプリケーションの起動時にすぐに実行される関数。
Angularアプリケーションを起動するときに、いくつかのrunSettings関数を実行する必要があるとします。

function runSettingsOnInit() {
   ...
}

これを行うには、メインのAppModuleモジュールに移動して、NgModuleデコレータープロバイダーセクションに追加します。

@NgModule({
   providers: [
       { provide: APP_INITIALIZER, useFactory: runSettingsOnInit }
   ]
})

9.ブートストラップリスナー


AppInitialzerと同様に、Angularにはコンポーネントのロードを追跡できる機能があります。これはAPP_BOOTSTRAP_LISTENERです。

このトークンによって返されるすべてのコールバックは、ダウンロード可能なコンポーネントごとに呼び出されます。

多くの場合、この機能を使用してコンポーネントの起動を追跡できます。たとえば、Routerモジュールはこれを使用して、ルートナビゲーションに基づいてコンポーネントを破棄および作成します。

APP_BOOTSTRAP_LISTENERを使用するには、コールバック関数を使用してAppModuleのプロバイダーセクションに追加する必要があります。

@NgModule({
   {
       provide: APP_BOOTSTRAP_LISTENER, multi: true,
       useExisting: runOnBootstrap
   }
   ...
})
export class AppModule {}


10. NgPlural


複数化はプログラマにとって頭痛の種です。単数形/複数形の使用に応じて単語形式を文法的に調整する作業は常に発生します。一部のサイトでは、このために末尾を使用しています。例えば：

1 component(s) removed
3 component(s) removed

つまり、読者自身が何を意味するかを理解する必要があります:)

Angularは、NgPluralディレクティブを使用して、この問題をより文明的な方法で解決することを提案しています。

NgPluralは、数値に基づいてDOMサブブランチを追加または削除します。複数形にするには、これが必要です。

このディレクティブを使用するには、[ngPlural]属性をスイッチ式として要素コンテナーに設定する必要があります。[ngPluralCase]属性を持つ内部要素は、指定された条件に応じて表示されます：

<p [ngPlural]="components">
   <ng-template ngPluralCase="=1">1 component removed</ng-template>   
   <ng-template ngPluralCase=">1">{{components}} components removed </ng-template>   
</p>


これで「（s）」を破棄できます。NgPluralディレクティブは、意味に応じて正しい単語形式を導き出すのに役立ちます。合計：

// if 1 component
1 component removed

// if 5 components
5 components removed
