SMSや登録なしでスマートテレビで急流を見る

こんにちは、Habr。

私は、カジノでプレイしたいダムサイトにアクセスせずに、お気に入りの映画/シリーズを高品質で視聴する方法の1つを共有することにしました。

ただちに予約しますので、海賊版コンテンツのご利用はお勧めできません。海賊行為は悪です。常にライセンスされたコンテンツを購入します。

さて、ダウンロードを待たずに、スマートTVで著作権で保護されていない急流を視聴したい人にとって、以下は興味深い方法です。

私はサムスンのスマートテレビを持っています。そのテレビでさまざまな方法で映画を見ようとしました。

Ubuntuにminidlnaをインストールしようとしました。しばらくの間、これは非常に便利に機能しましたが、ある日dlnaでテレビのアプリケーションが定期的に飛び出し始めました。私はまだこれが何に関連しているのか理解していません、いくつかの失敗した更新が到着した可能性があります。いずれにしても、完全にダウンロードされたコンテンツしか見ることができませんでした。

昔ながらの方法で映画を外部HDDにアップロードし、テレビに接続しました。数回後、それは私を悩ませました。

その結果、長い間、私はテレビに組み込まれたブラウザーを調べただけでした。しばらくすると、この方法はひどく退屈でした。

便利でシンプルなものが欲しかったので、すべてのコンテンツが読み込まれるまですぐに待機しませんでした。

Chromecastがあるという。しかし、私は私の場所に余分なデバイスなしですべてをやりたかったです。

結局のところ、組み込みのブラウザー（デスクトップクロムとは異なります）はHLSをサポートしています。さて、ラップトップでhlsストリームを取得し、ブラウザで監視しませんか。

そして、行きましょう。

1.私たちは急流のクライアントを置きます


トレントと呼ばれる非常にクールなnpmパッケージがあります。すべてうまくいくでしょうが、彼はダウンロードするファイルとリストを1つ選択し、ダウンロード速度を制限する方法を知りません。それは非常に重要です。しかし、それは最小限であり、ごちそうをすぐに拾い、シーケンシャルモードで完全に揺れます。これが私たちに必要なことです。

私は根性を調べ、必要な機能を追加する必要がありました。そして、私たちはフォークから急流を入れました

npm install 'https://github.com/zim32/torrent.git#master' -g

トレントファイルまたはマグネットリンクをダウンロードします。

ファイルのリストを取得するには、infoコマンドを実行する必要があります

torrent info some.torrent | less

必要なファイル名（パスではない）を見つけ、コマンドを実行します

torrent some.torrent --select 'FILE_NAME' --downloadLimit 1000000

すべてが順調であれば、ダウンロード統計、ピアの数などが表示されます。ファイルは現在のディレクトリに追加されます。速度制限（バイト/秒）。

私の場合、急流がチャネル全体を詰まらせるので、制限が必要です。ルーターが悪くなり、ラップトップとスマートTVの間でさらにブレーキが発生します。主なことは、速度がストリームのビットレートよりも高いことです。そうしないと、ロードとバッファリングが発生します。

HLSストリームの作成


ffmpegをインストールします。私の場合、すべてがリポジトリの通常のffmpegで動作しました。

apt install ffmpeg

ストリームが移動するパブリックディレクトリを作成します。

さらに、いくつかのオプションが可能です。ソースファイルのビデオがh264形式で、オーディオがAACの場合、ストリームを再エンコードしないようにすることができます。この場合、この方法でストリームを実行します

ffmpeg -re -i 'torrent_dir/movie.mkv' -codec copy -map 0:0 -map 0:1 -map_chapters -1 -movflags default_base_moof+frag_keyframe -f hls -hls_playlist_type event ~/www/player/out.m3u8

コーデックが異なる場合は、オンザフライでトランスコードする必要があります。私のラップトップでは、ffmpegはストリームに追いついていました。

ffmpeg -re -i 'torrent_dir/movie.avi' -c:v libx264 -preset slow -r 24 -x264opts fps=24:bitrate=2000:pass=1:vbv-maxrate=4000:vbv-bufsize=8000:keyint=24:min-keyint=24:scenecut=0:no-scenecut  -c:a aac -b:a 256k -map 0:0 -map 0:1 -map_chapters -1 -movflags default_base_moof+frag_keyframe -f hls -hls_playlist_type event ~/www/player/out.m3u8

オプション-map 0：0および-map 0：1を使用して、目的のチャネルを振動させます。ビデオとロシアのトラック。-map_chapters -1オプションは、念のため、章情報などの形ですべてのゴミを削除します。

ffprobe movie.mkvコマンドで利用可能なチャンネルを見ることができます

サーバーを作成する


ストリームと同じフォルダーに、次のコンテンツを含むindex.htmlファイルを配置する必要があります

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>

<video src="out.m3u8" controls></video>

</body>
</html>

単純なhttpサーバーをインストールします。

npm i http-server -g

サーバーを起動します

http-server -a 0.0.0.0 -c-1

映画を見て


テレビで、ラップトップのアドレス（例：192.168.1.200：8080）でブラウザを開いてお楽しみください。
