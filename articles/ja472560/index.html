ゲシュタルトテスト：ベイジアン理論と機械学習に基づくメーリングリスト最適化の新しいアプローチ



多変量（A / B / Nまたは分割）テストは、メーリングリストをテストする最も一般的な方法です。このツールは効果的であることが証明されていますが、主にテストと主要な出荷が時間的に分離されているという事実に関連する欠点があります。

私たちの側では、DashaMailは状況に影響を与えることを決定し、メーリングをテストするための異なるアプローチを見つけました。彼はベイズ理論、ニューラルネットワークの技術、機械学習を使用しています。その結果、これらすべてにより、文字の開封性を平均20％向上させることができます。

バックグラウンド


メールマガジンの効果を向上させるツールの1つはテストです。メッセージの件名、送信者の名前、メールの送信時間など、多くの要因が文字のオープン性と視聴者の関与に影響を与えます。

それほど前のことではありませんが、ブレーンストーミングセッションの1つで、現在人気のある機械学習アルゴリズムがメール配信のテストに違いをもたらすことができるという結論に達しましたつまり、オープン性とエンゲージメントにプラスの影響を与えます。よく知られているスプリットテストは、私たちが望むように完璧というわけではありませんが、実際には改善のオプションがあります。

A / B / Nテストは、電子メールマーケティングの主要な仮説テストオプションです。主な難点：このようなテストの結果は、常に事後的に分析することができます。これにより、プロセス全体が非常に長く時間のかかるものになります。最初にいくつかの郵送オプションを送信し、次に結果を調べ、テストパラメータを最適化して、もう一度送信する必要があります。そして、そのような多くの反復があり得る。

しかし、同時にテストと最適化を行う方法を作成するとどうなるでしょうか。このアイデアは、DashaMailのGestaltテストツールを生み出しました。

ベイジアンアプローチ：オンザフライでテストおよび最適化


さまざまなタイミングで受信したさまざまなメッセージオプションに対するサブスクライバーの応答は、大きく異なる可能性があります。メインニュースレターを送信するときに多変量テストの結果として決定された優勝オプションは、それほど効果的ではない場合があります。

この問題を回避し、メーリングのすべての重要なパラメーターをリアルタイムで考慮することができるように、意思決定と統計評価へのベイジアンアプローチが使用されました。はい、私たちダーシャメールは数学と確率論がとても好きです。

ベイズvs A / B / Nテスト


A / B / Nテストでは、一方ですべてが単純であり、他方でそれらの精度が非常に疑わしい場合があります。すべてが非常に簡単に思えます。たとえば、異なるデザインのメールの有効性をテストする必要がある場合、2つのオプションの場合、一方をサブスクライバーのベースに送信し、もう一方を2番目のオプションに送信できます。次に、結果を分析します。

ただし、統計的に有意な結果を得るには、両方のオプションが表示する必要があるユーザーの最小数を理解する必要があります。実際、サブスクライバーベースの20％のみをテストに割り当てるだけで十分な場合、残りの80％で最も効果的なバージョンのレターを起動して、最良の結果を得ることができます。ただし、10％の2つのグループを単純に割り当てると、正しい結果が得られる保証はありません。文字の1つのバージョンに赤い色がある場合、この色を好まない人が誤ってユーザーの10％のグループに分類される場合があります。さらに、より多くの人々がテストに参加した場合、このオプションが勝つ可能性があります。したがって、第1種と第2種のエラーの概念に到達します。Habréには、それらに関する十分な記事があります。これらのエラーには、独自の発生確率があります。

結果として、このテスト方法の分析は、不確実性をまったく排除しないという事実につながります。つまり、テストは「何が良いのか」という質問に対する正確な回答を与えません。作業は完了しましたが、明確になりませんでした。

この方法とは対照的に、いわゆるベイジアン多腕バンディットが使用されます。この方法の本質は、仮説検定を実施できるだけでなく、より効果的である可能性が高い質問に対する答えを得ることも可能にすることです。そして重要なことは、各仮説のサンプルサイズがリアルタイムで決定されるのと同じ方法で、見積もりが動的に変化することです（つまり、特定のオプションをテストするために送信する必要があるトラフィック/文字の量）。

「ワンアームバンディット」タイプのスロットマシンがあるカジノに来たときの状況を想像してみてください。私たちはお金が限られている、時間も無限ではありません。 「有望な」マシンをできるだけ早く決定すると同時に、最小限のコストでそれが必要です。これは多腕バンディットタスクです。それを解決するための多くのオプションがあり、それらの1つは、トンプソンサンプリングとベイズの定理に基づいています; Habréに関するこの記事で詳しく説明されています。

メーリングリストの場合、これは次のように機能します。 2つ以上の仮説（郵送オプション）をテストするプロセスでは、明らかにパラメーターを失った文字を送りすぎないようにします（A / Bテストでは、等しいシェアを送信する必要があります）。しかし、同時に、私はそのようなバリエーションにも従いたいと思います。なぜなら、時間が経つと彼らはうまくいき始め（最初は不運だった）、リーダーになることさえあり、さらに多くのトラフィックが彼らに行く可能性があるからです。

この理論は、ゲシュタルト試験と呼ばれる新しいツールの基礎を形成しました。



従来のA / Bテストとの主な違い：ほとんどの文字が勝者オプションで送信されるという事実にもかかわらず、サブスクライバーの行動パターンが変化した場合、時間内に応答し、最も適切なオプションを状況に送信する必要があるため、他のオプションは常に最後のチャンスがあります。

さらに、ゲシュタルトテストは、ニュースレターで感情的なマーケティングを使用して、感情的な色の異なる文字を作成する機能です。これは次のように機能します。ニュースレターを送信する電子メールマーケターが基本的なトピックを設定し、このトピックをさまざまな感情で言い換えることができます-最大10のオプション（恐怖、感謝など）があります。



ニューラルネットワークは、与えられた感情的な色を使用して、トピックのテキストを言い換え、検討のためにそれらを提供します。この場合、メールマーケターは彼の裁量で変更を行うことがあります。

感情と関連トピックの例、およびそれぞれの発見のインジケーター：



開始後、システムはグループで手紙を送信し始めます-各パッケージには提案されたすべてのオプションが含まれています。すべての郵送には約10時間かかり、30分ごとに1パックです。ご覧のとおり、このツールは、迅速に発送する必要がある短期在庫には適していません。むしろ、中期的なプロモーションまたはコンテンツ配信のオプションを検討できます。各オプションの統計情報を利用できるので、すぐに何が効果的かを確認できます。

以下の例では、オープニングとクリックに従って、「愛」という感情のニューラルネットワークによって書き換えられたテーマを持つバリアントがリードします。「あなたはオフィスで最も美しいです！弊社のセレクションのオフィスモデルは-30％。」ただし、他のすべてのオプションの中で最も高い、退会率も示しています。これは、レターの内容がトピックよりも弱かったか、または以前に眠っていたサブスクライバーのセグメントの注意を引くことができたことを示している可能性があります。



Gestaltテストを使用したメーリングの送信は時間が延長されるため、送信の特定の瞬間のテストも自動的に実行されます。さらに、サービスはニュースレターのどの感情を、どの特定のサブスクライバーがいつ反応するかを記憶し、その後の送信中にこの機能を使用してそれに適応します。したがって、時間の経過とともに、ゲシュタルトテストの効果が高まります。



なぜうまくいくのか


新しいテストツールの考え方は、受信者がドライテキストよりもパーソナライズされた感情的な色のメッセージによく反応するという事実を考慮に入れることができるということです。

同時に、ゲシュタルトテストでは、機械学習手法がトピックのすべてのバリアントに適用されます。テスト中に最も成功したオプションが最もアクティブに使用されますが、他の比較参加者は少しトラフィックを受け取ります。これにより、定期的にサブスクライバーの動作パターンを監視できます。一度に良好なパフォーマンスを提供したトピックが、残りのオプションを強打して失うことがよくあります。システムがこのようなパターンの変化を「検出」した場合、ニュースレターはその場で最適化され、最大の効率を維持します。

各サブスクライバーの行動パターンが分析されます。特定の受信者の発見履歴に基づいて、個々の送信時間を選択します。一時的なパターンも変化する可能性があります。たとえば、人が就業日の開始時刻と終了時刻を変更したり、個人のメールをチェックする機能が別のときに発生したりする場合があります。ゲシュタルト機能は、このような変化に自動的に適応します。

重要な点：ゲシュタルトテストは、特定の量のデータを必要とする方法です。それ以外の場合、高効率を維持することは困難です。そのため、1万以上のアドレスのデータベースでのみ使用できます。

結論：信頼できる結果


論理的に聞こえますが、提案されたテストツールを使用すると、実際にどのような結果を期待できますか？例を見てみましょう。これは、メーリング用のゲシュタルト関数の使用に関するレポートの見た目です。最終的な開封率（OR）、ベーストピックに対する結果、および件名ごとに同じ文字の分布を持つ通常の多変量テストで達成されたインジケーターとの比較が含まれます。



DashaMailクライアントの統計によると、ゲシュタルトテストを使用したメールの開封率の平均増加率は20％です。システムは特定のサブスクライバーがいつどの感情でよりよく反応するかを学習および記憶し、その結果、メールの開封率（OR）を1.5〜2倍に増やすことができるため、時間の経過とともにこの機能を使用する効果が高まります。基本的なテーマ。
