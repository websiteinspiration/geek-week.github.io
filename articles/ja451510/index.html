JetBrainsのC ++ツールに関する少し技術的な歌詞と、ユニコーンがそれとどう関係するか

典型的な「Hello、Habr！ここにも別のクールなリリースがあります。「こんにちは。Nastyaと申します。私はJetBrainsのPMMであり、C ++ツールの責任者です。」そうではない場合は、次のようにもう一度試してみます。アイデアはすべてこだわる。」

いいえ、これは空席の候補者を検索することに関する従来の投稿ではありません。私が話をしてみましょう、我々はいくつかのC ++のツール持っている理由と、私たちが彼らのために持っているアイデアや計画を、なぜあなたが開発者として、その上に書いて停止した場合、あなたはC ++を忘れないだろう、そしてPMMになります（ネタバレ、C ++言語標準化委員会のメンバーでない場合は、言語をよりよく学習できる可能性が高くなります）。その後、PMMとしてこれに参加したい場合は、anastasia.kazakova @ jetbrains.comで履歴書をいただければ幸いです。

なぜC ++用のIDEを取得して作成できないのですか？


C ++言語コンパイラからIDE用のパーサーを作成するのは非常に簡単なようです。数年前のACCU、C ++ NowおよびCppConカンファレンスで、物事がそれほど単純ではない理由について話し始めました。たとえば、2017年のレコードをACCU（言語ツールのメガネを通してC ++を見る）とCppCon（救出の新しい標準：IDEのメガネを通して見る）で見ることができます。重要なポイント：環境がスマートになるほど、C ++は難しくなります。

  • エディターのパフォーマンス（および応答性）を維持します。
  • 正しくないコードを処理できるようにする（コンパイラーはエラーを出し、動作を停止します）。
  • 翻訳単位（TU）ではなく、プロジェクト全体の規模で「考えて」ください（名前を変更すると、同じ名前だけでなく、プロジェクト全体で、コンテキストシンボルが正確に必要になるため）。

したがって、2014年には、1つではなく、2つ（またはさらに正確には3つ）のC ++開発用の開発環境が生まれました。そして、それはすべて突然起こりました。AppCodeでObjective-Cを実行したところ、C ++パーサーを作成していることがわかりました。そして去ります！ちなみに、私は最近のモスクワでのC ++ Russia 2019カンファレンスのインタビューでこの面白い話をしました：


その結果、チームの一部はIntelliJプラットフォーム-CLionに基づくIDEを作成することを決定しました。そしてもう1つの部分は、異なるアーキテクチャで異なるアプローチを実装し始めました-Visual Studioの拡張機能であるReSharper C ++です。そして、別のclangdベースのパーサーが登場しました。一般的に、いくつかの製品と多くのパーサーがあります。

三頭竜とその売り方


同時に、C ++製品の読者は少し異なります。

CLionは、C ++でのクロスプラットフォーム開発に焦点を当てています。つまり、いくつかのプラットフォーム（Linuxを含み、オプションが多くない）でIDEを実行したい人向けです。これは、他のツール（Valgrind Memcheck、Google Sanitizers、DTrace、Perf、Conan）および言語（Python、Rust、Swift、Kotlin / Native）との多くの統合（直接およびプラグインを通じて、サードパーティと当社の両方）が統合された独立型のフル機能環境です。 ） CLion では、組み込み開発市場のサポートに向けて取り組んでいます。 IDEは、金融分野、自動運転車の開発市場などで人気があります。BMWの広告にも最近掲載されました。

ReSharper C ++-Windows環境で開発している人向けに設計され、対応するツールチェーン（msbuild、M​​SVC）によってガイドされるVisual Studioの拡張機能。ここでは、既にVisual Studioにある機能を実装しようとはしていませんが、特に最新のC ++を使用すると、コードをより便利、迅速、かつ生産的に操作できるようにしています。したがって、この製品には、C ++開発の第一人者になることができるクールなオタク機能がたくさんあります。これで、Unreal Engineでゲーム開発者をサポートするためにReSharper C ++で行っているアクティブな作業を確認できます。そのようなゲームの主な対象者は、MS環境のWindowsで開発されているため、これは非常に論理的です。したがって、UE4のゲーム専用のパフォーマンス最適化と特別な機能を取り上げました。

また、CLionによるC ++サポートは、AppCode（実際に発生した場所）およびAndroid Studio（GoogleがIntelliJプラットフォームに基づいて作成）に存在します。

そして、どういうわけかこれをユーザーに説明するために、Because C ++というマーケティングキャンペーンを考え出しました。C ++カンファレンスでのスタンドを見たり、C ++ Nowカンファレンス（ビデオスポンサーとしてサポートしている）からのレコーディングを見たり、緑のボトルやC ++アイコンを配布資料として受け取ったことがある場合、これが何であるかを正確に理解できます。



しかし、ユニコーンはどうですか？


この多様性のユニコーンは1つになりました-私です。「JetBrainsのユニコーン」の概念に慣れていない場合は、次の投稿からアブレスラフこれは、JetBrainsでのPMMの位置をかなり正確に説明しています。かつて、PMMサマースクールに多くのエネルギー（精神的および肉体的）を投資し、自分自身について多くのことに気づき、自分が誰であるか、そして私たちが何をしているのかを他の人に伝えました。パウラリオノフここで、ハブレはその様子を語りました（同時に講義スライドへのリンクも含まれています）。ビューアに興味がある場合、JetBrainsからではありません。

私は最初はマーケティングの人ではありません。私はC / C ++開発からJetBrainsに来ました：組み込みアウトソーシングで5年、Yota / Roox / Scartelで3年（多くの名前があり、基本的に1つ）、PCRFを実行し、ひどく飛んだすべてを最適化しました（そしてHabréにそれについて書きました））、そして突然... ...実際には、C ++では交差が少なくなりませんでした。もちろん、私は既成の商用システムを記述していませんが、言語の複雑さを詳しく調べ、豊富なQAとともにIDEのサポートを中断し、製品のブログですべてを説明しています。私はテクニカルライターがこれまたは次の機能のシナリオをどれほどうまく説明しているかを評価し、エンドユーザー（つまり、C ++開発者）と常に連絡を取り、あらゆる種類の「興味深いケース」を示します。チームと製品計画や現在の問題について話し合い、開発者の弁護士やコミュニティと協力しています。さらに、標準化委員会とより緊密に連絡を取り合い、彼の会議に出席するようになりました。また、会議でC ++とそのエコシステムについて話し、サンクトペテルブルクでC ++コミュニティの会議を組織することも好きです。

しかし、PMM向けの製品は、広告キャンペーン、会議の準備、さまざまなマーケティング資料など、それほど技術的なタスクがありません。それは私の成長するTODOリストにもすべて含まれています。
