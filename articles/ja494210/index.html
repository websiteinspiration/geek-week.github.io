Microsoft Surface ProにUbuntuをインストールする

MicrosoftのSurface Proタブレットは、従来のデスクトップアプリケーションをインストールできるため、魅力的な作業ツールのように見えます。

Linux on Surfaceのインストールに関するテーマは新しいものではなく、非常に人気があります。

  • 6年前のHabréには、「Surface Pro 3とLinuxを友達にする方法」のような記事がすでにありました。
  • Redditには、8000人のサブスクライバーr / SurfaceLinuxに関するトピックがあり、現状の詳細な説明があります。
  • linux-surfaceリポジトリは、定期的にLinuxカーネルの新しいリリースをgithub に公開しています。


UbuntuがインストールされたMicrosoft Surface Pro 19.10（Eoan Ermine）

アメリカのオンラインストアAmazonが前世代の正式に復元された12インチタブレットを2、3倍の価格で正式に販売していることを知ったとき、Microsoft Surface Proに興味が湧きました最新世代のデバイスよりも安価です。しかし、メーカーによって復元されたそのようなタブレットを購入すると、実際には映画の新しいデバイスと、改修マークが付いた箱と、これらすべてが低価格で手に入ります。

一般的に、私は抵抗することができず、特にLinuxをインストールして仕事で使用するためにMicrosoft Surface Proを購入しました。

Surfaceの製品ラインは非常に幅広く、タブレット、ラップトップ、インタラクティブホワイトボードが含まれます。Amazonの販売者がどのモデルを意味するかは必ずしも明確ではありません-発表「Microsoft Surface Pro 4 (2736 x 1824) Tablet 6th Generation (Intel Core i5-6300U, 8GB Ram, 256GB SSD, Bluetooth, Dual Camera) Windows 10 Professional (Renewed)」のように第6世代が示されていることもありますが、実際には販売者は誤解を招きます。第6世代-これはSurface Pro 6だと思って混乱するかもしれませんが、実際はSurface Pro 4です。マインドフルネスが害を及ぼすことはありません。

なぜLinuxなのか？


私はWindowsを25年以上使用していますが、近年ますます少なくなっています。Windowsにのみ存在する特別なプログラムを使用する必要はありません。すべてがブラウザーのみに依存するクラウドサービスにスムーズに移行しています。

さらに、スマートホームへの私の情熱は、特にコマンドライン、そして一般にLinuxの使用に貢献しています。 Linuxコンピューターを使用している場合、一部のアクションはより簡単または高速です。

Surface Proタブレットについては、システムをWindowsからLinuxに置き換えるのがどれほど簡単かについていくつかの疑問がありましたが、将来判明したので、これらの疑問は完全に無効になりました。Ubuntuのインストールは、通常のラップトップのようにスムーズに進みました。これはタブレットではなくラップトップ上にありますが、Surface Proはメーカーによってタブレットとして位置付けられています。私の意見では、これはまだラップトップであり、タブレットではありません。WindowsおよびLinuxのタッチコントロールは、私の意見ではまだ優先順位ではありません。


購入後のSurface ProタブレットはWindowsで動作します。

Linuxをインストールする前に


購入後すぐに、タブレットは英語版のWindows 10で動作し、Linuxをインストールする前に、Windowsバックアップディスクを作成します。

Linuxのインストール自体では、デバイスに物理ポートが1つしかなく、Linuxのインストール中はタッチスクリーンが機能しないため、キーボードとマウスを接続するためのUSBハブは必ず必要です。

Surface Proでリカバリディスクを作成する

WindowsをSurface Proタブレットに残してLinuxのみを使用する予定がない場合でも、デバイスを販売する前などにシステムを復元できるように、リカバリディスクを作成することをお勧めします。これを行う方法に関するロシア語の詳細な説明は、Microsoftの Webサイトにあります。メニュー項目の英語名は、Microsoft Webサイトの同じ手順の別の言語のセクションにあります。

Linuxインストール


特定の配布キットを選択する前に、Surface Pro 4以降のタブレットモデルの両方のカメラがLinuxでは機能しないことを知っておく必要があります。これは、以前のモデルのように、カメラが通常のUSBバスではなくPCIバス上にあるためです。また、Surface Pro 7では、ペン（スタイラス）はLinuxではまだ機能しません。 Debian / Arch Linux / Fedora / Gentooに基づくディストリビューションを選択できます。

私の考えでは、Ubuntuを選びました。私の意見では、大きなボタンを備えたデスクトップシェルの方がタッチコントロールに適しているからです。 HiDPI高解像度ディスプレイの簡単なセットアップは、このディストリビューションのプラスに過ぎません。

起動可能なUSBフラッシュドライブからタブレットをロードすると、起動オプションのメニューが表示され、画面キーボードを使用して起動オプションを選択できます。

カスタムカーネルをインストールする


タッチパッドとペンを復元するには、対応するディストリビューションのカスタムカーネルを使用します。Ubuntuのコマンドを以下に示します。

リポジトリを追加する前に、開発者がパッケージの署名に使用するキーをインポートする必要があります。

wget -qO - https://raw.githubusercontent.com/linux-surface/linux-surface/master/pkg/keys/surface.asc \
    | sudo apt-key add -

その後、次のようにしてリポジトリ自体を追加できます。

echo "deb [arch=amd64] https://pkg.surfacelinux.com/debian release main" | sudo tee /etc/apt/sources.list.d/linux-surface.list

次に、パッケージのリストを更新して更新します。

sudo apt-get update


linux-surfaceリポジトリを追加するコマンドを含む端末のスクリーンショット。

カーネルをインストールしますが、最新のカーネルsurface（2020年3月にバージョン5.5.10）はインストールしません。このバージョンのカーネルでは、ペンもタッチスクリーンもマルチタッチも正しく機能しませんでした。開発者は4.19カーネルのインストールを推奨しています（2020年末までの長期サポート）：

sudo apt-get install linux-image-surface-lts linux-headers-surface-lts linux-libc-dev-surface-lts surface-ipts-firmware linux-surface-secureboot-mok libwacom-surface

しかし、それでも最新のカーネルを試したい場合は、他のコマンドを使用する必要があります。

sudo apt-get install linux-headers-surface linux-image-surface linux-libc-dev-surface surface-ipts-firmware linux-surface-secureboot-mok libwacom-surface

新しいカーネル4.19（LTS）を追加するためのコマンドを含むターミナルのスクリーンショット次に、

2つのオプションがあります（セキュアブートのセキュリティ設定を変更していない場合）。

  1. Linuxカーネルバージョン5.5.10をSurface Proにインストールした場合は、新しいカーネルをインストールした直後に再起動surfaceし、上記のスクリーンショットのように、インストール中に指定したパスワード「」を入力します。これを行うには、メニュー項目を選択します。MOKの登録/キーの登録->はい/パスワード->サーフェス。

    再起動後のセキュアブートの構成
  2. カーネル4.19.110をインストールした場合、タブレットは通常どおり起動し、[MOKの登録]メニューは表示されません。

この安全なダウンロードの背後にある考え方は、信頼できるソフトウェアのみをコンピューターにダウンロードできるようにすることで、それ以外の場合はオペレーティングシステムによって検出されない潜在的なウイルスやルートキットをブロックすることです。私たちが使用したものなどのカスタムカーネルは任意のユーザーが収集および配布できるため、ブートローダーでは信頼できないと見なされます。したがって、セキュアブートを有効にしている場合、ダウンロードは許可されません。

デフォルトでロードする別のカーネルを設定するためのGRUBの編集


新しいカーネルをインストールした後surface、このカーネルを指すように、GNU GRUBオペレーティングシステムのブートローダーを更新する必要があります。デフォルトでは、カーネルはディストリビューションによって提供されるメインディストリビューションとともにインストールされます。したがって、問題が発生した場合に使用できるバックアップコアがあることがわかります。

インストールされているコアに応じて、2つのオプションもあります。

  1. カーネルバージョン5.5-ブートローダーでは、最初から自動的にインストールされます。追加の操作は必要ありません。
  2. カーネルバージョン4.19（LTS）。この特定のカーネルでデフォルトのブートを実行するには、追加の手順が必要です。

GRUBにsurfaceデフォルトのカーネルロードを登録するために実行するアクション：

  1. まず/ etc / default / grubをバックアップします。問題が発生した場合は、よく知られているコピーに簡単に戻ることができます。

    sudo cp /etc/default/grub /etc/default/grub.bak
  2. 次に、選択したテキストエディター（Ubuntuのgeditなど）を使用してファイルを編集します。

    sudo -H gedit /etc/default/grub
  3. GRUB_DEFAULTを含む行を見つけます。これは、デフォルト値を設定するために編集する必要があるものです。私の場合、私は書きます：

    GRUB_DEFAULT="Advanced options for Ubuntu>Ubuntu, with Linux 4.19.110-surface-lts"
  4. ファイルaを保存し、次のコマンドで更新されたGRUBメニューを作成します。

    sudo update-grub
