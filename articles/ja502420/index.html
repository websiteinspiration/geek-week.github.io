ユーザーアバターのアスペクト比


プロフィールページのユーザーアバターの興味深い実装に気づきました。この実装では、CSS手法を使用して要素のアスペクト比を維持します。


プロファイルページ

のユーザーアバター次のHTMLおよびCSSコードは、アバターの実装を示しています。

<a href="#" class="avatar">
    <div class="avatar-aspect-ratio"></div>
    <img alt="" src="me.jpg">
</a>
.avatar {
  position: relative;
  width: 25%;
  display: block;
}

.avatar-aspect-ratio {
  width: 100%;
  padding-bottom: 100%;
}

.avatar img {
  position: absolute;
  top: 0; 
  right: 0; 
  bottom: 0;
  left: 0;
  width: 100%;
  height: 100%;
}

要素に縦のインデント（プロパティpadding-bottomまたはpadding-top）がある場合、インデントは要素の幅に依存するため、要素のアスペクト比を維持する手法は有効です。次の例を見てください。

.element {
    width: 250px;
    padding-bottom: 100%;
}

計算値padding-bottomは等しい250pxです。これは、完全な正方形があることを意味します。Twitterチームも同じ手法を使用しましたが<img>、親要素に対して絶対的に配置されている要素に適用されました。どうして？これが理由です。


アバターに問題があります。

画像が完全に配置されていない場合、アバターは前の図のようになります。画像は100%、幅と高さの値を使用して配置する必要があります。このアプローチでは、そのサイズはラッパー要素のサイズに従って設定されます。

このソリューションの背後にあるアイデアを理解したところで、このアイデアがTwitterでどのように実装されたかに戻りましょう。

このプロパティはwidth: 25%、ラッパー要素の幅に基づいています。私の画面では-600px。どうしてそんなテクニックが選ばれたのかしら。CSSコードについて詳しく調べたところ、同じコンポーネントが[プロファイルの編集]モーダルウィンドウで使用されていることに気付きました。確かに、次のスタイルを使用しているため、ここの要素は小さくなっています。

.avatar {
    max-width: 8rem; /* 112px  150px (25%) */
}

プロパティのみに影響を与えることで画像を管理するという考え方が本当に好きwidthです。これは、このアイデアを説明するビデオです。
