いくつかの新しいJavaScript機能の使用例

資料の作成者は、その翻訳を本日公開していますが、ES2019標準に該当する新しいJavaScript機能は、Chrome、Firefox、Safariブラウザー、およびNode.jsですでに公式に利用可能であると述べています。古いブラウザをサポートする必要がある場合は、Babelを使用してJSコードをトランスパイルすることにより、革新を利用できます。ここでは、JSの新機能の例をいくつか紹介します。



Object.fromEntriesメソッド


ES2017にメソッドが登場しましたObject.entries。オブジェクトを配列に変換します。たとえば、次のようになります。

let students = {
  amelia: 20,
  beatrice: 22,
  cece: 20,
  deirdre: 19,
  eloise: 21
}

Object.entries(students) 
// [
//  [ 'amelia', 20 ],
//  [ 'beatrice', 22 ],
//  [ 'cece', 20 ],
//  [ 'deirdre', 19 ],
//  [ 'eloise', 21 ]
// ]

このメソッドは、言語の機能に素晴らしい追加となっています。実際には、プロトタイプに組み込まれた多数のメソッドを使用してオブジェクトデータを簡単に処理できるようになりましたArray。これらの方法の中でも、例えば、それが注目されてもよいですmap、filter、reduce。しかし、配列をオブジェクトに戻すために、残念ながら、便利なツールはありませんでした。すべてをループを使用して手動で行う必要がありました。

let students = {
  amelia: 20,
  beatrice: 22,
  cece: 20,
  deirdre: 19,
  eloise: 21
}

//          .filter()
let overTwentyOne = Object.entries(students).filter(([name, age]) => {
  return age >= 21
}) // [ [ 'beatrice', 22 ], [ 'eloise', 21 ] ]

//      
let drinkingAgeStudents = {}
for (let [name, age] of overTwentyOne) {
    drinkingAgeStudents[name] = age;
}
// { beatrice: 22, eloise: 21 }

このメソッドはObject.fromEntries、そのようなサイクルを取り除くために作成されました。これにより、はるかに少ないコードで同じ問題を解決できます。これは、開発者が配列メソッドを使用して、配列に変換されたオブジェクトをより頻繁に処理するのに役立つ場合があります。

let students = {
  amelia: 20,
  beatrice: 22,
  cece: 20,
  deirdre: 19,
  eloise: 21
}

//          .filter()
let overTwentyOne = Object.entries(students).filter(([name, age]) => {
  return age >= 21
}) // [ [ 'beatrice', 22 ], [ 'eloise', 21 ] ]

//      
let drinkingAgeStudents = Object.fromEntries(overTwentyOne); 
// { beatrice: 22, eloise: 21 }

配列とオブジェクトには、データ構造が異なる理由がないわけではないことに注意してください。場合によっては、あるデータを別のデータに変換すると、データが失われることがあります。次の例では、配列をオブジェクトに変換するときに、オブジェクトの重複キーであることが判明した配列の要素がどのように失われるかを確認できます。

let students = [
  [ 'amelia', 22 ], 
  [ 'beatrice', 22 ], 
  [ 'eloise', 21], 
  [ 'beatrice', 20 ]
]

let studentObj = Object.fromEntries(students); 
// { amelia: 22, beatrice: 20, eloise: 21 }
//    beatrice!

Array.prototype.flatメソッド


多次元配列は、プログラマーが頻繁に遭遇するデータ構造です。特に-一部のデータをロードする場合。同時に、配列の次元を減らすことは常に重要な仕事でした。この問題を解決することは常に可能でしたが、そのソリューションのコードは魅力的とは言えませんでした。

次の例を考えてみましょう。ここでは、関数を使用してオブジェクトの配列を処理した結果map、多次元配列ができました。もっと「フラット」にしたい。

let courses = [
  {
    subject: "math",
    numberOfStudents: 3,
    waitlistStudents: 2,
    students: ['Janet', 'Martha', 'Bob', ['Phil', 'Candace']]
  },
  {
    subject: "english",
    numberOfStudents: 2,
    students: ['Wilson', 'Taylor']
  },
  {
    subject: "history",
    numberOfStudents: 4,
    students: ['Edith', 'Jacob', 'Peter', 'Betty']
  }
]

let courseStudents = courses.map(course => course.students)
// [
//   [ 'Janet', 'Martha', 'Bob', [ 'Phil', 'Candace' ] ],
//   [ 'Wilson', 'Taylor' ],
//   [ 'Edith', 'Jacob', 'Peter', 'Betty' ]
// ]

//       -  [].concat.apply([], courseStudents)

これArray.prototype.flatで、配列の要素を「上げる」必要があるレベルを示すオプションの引数を取るメソッドを自由に使用できます。

let courseStudents = [
  [ 'Janet', 'Martha', 'Bob', [ 'Phil', 'Candace' ] ],
  [ 'Wilson', 'Taylor' ],
  [ 'Edith', 'Jacob', 'Peter', 'Betty' ]
]

let flattenOneLevel = courseStudents.flat(1)
console.log(flattenOneLevel)
// [
//   'Janet',
//   'Martha',
//   'Bob',
//   [ 'Phil', 'Candace' ],
//   'Wilson',
//   'Taylor',
//   'Edith',
//   'Jacob',
//   'Peter',
//   'Betty'
// ]

let flattenTwoLevels = courseStudents.flat(2)
console.log(flattenTwoLevels)
// [
//   'Janet',   'Martha',
//   'Bob',     'Phil',
//   'Candace', 'Wilson',
//   'Taylor',  'Edith',
//   'Jacob',   'Peter',
//   'Betty'
// ]

このメソッドに引数を渡さないと、配列の要素が1レベル上がることに注意してください。この場合、配列を完全にフラットなデータ構造に変換する必要があるため、これは非常に重要です。パラメータなしでこのメソッドを使用した結果は次のとおりです。

let courseStudents = [
  [ 'Janet', 'Martha', 'Bob', [ 'Phil', 'Candace' ] ],
  [ 'Wilson', 'Taylor' ],
  [ 'Edith', 'Jacob', 'Peter', 'Betty' ]
]

let defaultFlattened = courseStudents.flat()
console.log(defaultFlattened)
// [
//   'Janet',
//   'Martha',
//   'Bob',
//   [ 'Phil', 'Candace' ],
//   'Wilson',
//   'Taylor',
//   'Edith',
//   'Jacob',
//   'Peter',
//   'Betty'
// ]

この方法のこのようなデバイスの正当化は、デフォルトでは、配列を1次元に変換しようとはせず、配列を変換するための特定の命令を必要とするという事実にあります。正確なパラメータが不明な配列を1次元配列に変換する必要flatがある場合は、メソッドに値を渡すことができますInfinity。

let courseStudents = [
  [ 'Janet', 'Martha', 'Bob', [ 'Phil', 'Candace' ] ],
  [ 'Wilson', 'Taylor' ],
  [ 'Edith', 'Jacob', 'Peter', 'Betty' ]
]

let alwaysFlattened = courseStudents.flat(Infinity)
console.log(alwaysFlattened)
// [
//   'Janet',   'Martha',
//   'Bob',     'Phil',
//   'Candace', 'Wilson',
//   'Taylor',  'Edith',
//   'Jacob',   'Peter',
//   'Betty'
// ]

通常、このような操作は注意して使用する必要があります。このアプローチは、おそらく、処理された配列の深さが本当に不明であるような場合には良い選択とは言えません。

Array.prototype.flatMapメソッド


メソッドと一緒に、flat私たちは自由に新しい組み合わせメソッドを持っていますArray.prototype.flatMap。上記では、この方法が役立つ状況の例をすでに実際に見ましたが、別の例を見てみましょう。

特定の要素を配列に挿入するタスクに直面していると仮定します。新しいJS機能が登場する前に、それをどのように解決しますか？たとえば、次のようになります。

let grades = [78, 62, 80, 64]

let curved = grades.map(grade => [grade, grade + 7])
// [ [ 78, 85 ], [ 62, 69 ], [ 80, 87 ], [ 64, 71 ] ]

let flatMapped = [].concat.apply([], curved) 
//    .       flat,      JS  
// [
//  78, 85, 62, 69,
//  80, 87, 64, 71
// ]

メソッドArray.prototype.flatができたので、このコードを改善できます：

let grades = [78, 62, 80, 64]

let flatMapped = grades.map(grade => [grade, grade + 7]).flat()
// [
//  78, 85, 62, 69,
//  80, 87, 64, 71
// ]

ここで解決している問題は、比較的一般的なパターンに基づいています（これは特に関数型プログラミングに当てはまります）。したがって、メソッドを使用してどれほど美しくそれを解決できるかはflatMap喜ばざるを得ません。

let grades = [78, 62, 80, 64]

let flatMapped = grades.flatMap(grade => [grade, grade + 7]);
// [
//  78, 85, 62, 69,
//  80, 87, 64, 71
// ]

デフォルトでは、メソッドArray.prototype.flatはユニットに渡されたかのように機能することを思い出してください。このメソッドflatMapはまったく同じように動作します。つまり、配列の要素を1レベルだけ「レイズ」します。mapとメソッドの組み合わせの結果ですflat。

let grades = [78, 62, 80, 64]

let flatMapped = grades.flatMap(grade => [grade, [grade + 7]]);
// [
//   78, [ 85 ],
//   62, [ 69 ],
//   80, [ 87 ],
//   64, [ 71 ]
// ]

String.prototype.trimStartおよびString.prototype.trimEndメソッド


ES2019のもう1つの優れた新機能は、いくつかの文字列メソッドにわかりやすい名前を付けるエイリアスです。以前、我々は我々の処分方法で持っていたString.prototype.trimRightとString.prototype.trimLeft：

let message = "   Welcome to CS 101    "
message.trimRight()
// '   Welcome to CS 101'
message.trimLeft()
// 'Welcome to CS 101   '
message.trimRight().trimLeft()
// 'Welcome to CS 101'

方法は素晴らしいですが、目的に関連する名前が付けられているのは良いことです。そして、それらの目標は、文字列から先頭と末尾の空白を削除することです。

let message = "   Welcome to CS 101    "
message.trimEnd()
// '   Welcome to CS 101'
message.trimStart()
// 'Welcome to CS 101   '
message.trimEnd().trimStart()
// 'Welcome to CS 101'

オプションのcatchブロック引数


ES2019のもう1つの優れた機能は、ブロックの引数try-catchがオプションになったことです。以前は、すべてのブロックcatchが例外オブジェクトをパラメーターとして渡す必要がありました。議論はcatch使われなくても伝達されなければなりませんでした。

try {
  let parsed = JSON.parse(obj)
} catch(e) {
  // e    
  console.log("error")
}

今はそうではありません。例外オブジェクトがブロックcatchで使用されていない場合、このブロックに転送する必要はありません。

try {
  let parsed = JSON.parse(obj)
} catch {
  console.log("error")
}

これは、プログラマーがどのような緊急事態が対応するブロックに陥るのかを事前に知っている場合に便利な素晴らしい機会catchです。

Function.prototype.toStringメソッドの変更


ES2019標準は、関数メソッドの動作方法に変更をもたらしました toString。以前は、出力コードのレイアウトを少し歪めていました。

function greeting() {
  const name = 'CSS Tricks'
  console.log(`hello from ${name}`)
}

greeting.toString()
//'function greeting() {\nconst name = \'CSS Tricks\'\nconsole.log(`hello from ${name} //`)\n}

現在、このメソッドは、関数のソースコードの実際の表現を反映しています。

function greeting() {
  const name = 'CSS Tricks'
  console.log(`hello from ${name}`)
}

greeting.toString()
// 'function greeting() {\n' +
//  "  const name = 'CSS Tricks'\n" +
//  '  console.log(`hello from ${name}`)\n' +
//  '}'

▍サポート


  • Chrome 75
  • Firefox 60
  • Safari 12（部分的に）
