Python Newsビットコイン価格予測

記事の翻訳は、OTUSの機械学習コースの開始前に準備されました。



仕事


このガイドでは、ビットコインと米ドルのデータセットを使用しています。



上記のデータセットには、毎日の価格の概要が含まれています。ここで、CHANGE列は、前日の価格（PRICE）と新しい価格（OPEN）の割合としての価格の変化です。

目標：タスクを簡略化するために、価格が翌日上昇する（CHANGE> 0）か下降する（CHANGE <0）かを予測することに焦点を当てます。（したがって、「実際の」予測を使用できる可能性があります）。

必要条件

  • Python 2.6+または3.1+がシステムにインストールされている必要があります
  • パンダ、sklearn、openblenderをインストールします（pipを使用）

$ pip install pandas OpenBlender scikit-learn


ステップ1.ビットコインデータを取得する


はじめに、必要なライブラリをインポートしましょう：

import OpenBlender
import pandas as pd
import json


次に、OpenBlender APIを介してデータをプルします。
まず、パラメーターを定義しましょう（この場合、これはビットコインデータセットのidです ）：

# It only contains the id, we'll add more parameters later.
parameters = { 
 'id_dataset':'5d4c3af79516290b01c83f51'
}

注：openblender.io（無料）でアカウントを作成し、トークンを追加する必要があります（[アカウント]タブにあります）。

parameters = {     
   'token':'your_token',
   'id_dataset':'5d4c3af79516290b01c83f51'
}

次に、データをデータフレーム「df」に配置します。

# This function pulls the data and orders by timestamp
def pullObservationsToDF(parameters):
    action = 'API_getObservationsFromDataset'
    df = pd.read_json(json.dumps(OpenBlender.call(action,parameters)['sample']), convert_dates=False,convert_axes=False) .sort_values('timestamp', ascending=False)
    df.reset_index(drop=True, inplace=True)
    return df
df = pullObservationsToDF(parameters)

そしてそれらを見てください：



注：データセットは毎日更新されるため、値は異なる場合があります！

ステップ2.データの準備


まず、私たちは「かどうかになります予測対象、作成する必要がありCHANGEが」増加または減少しますが。これを行うには、ターゲットのしきい値パラメーターに'success_thr_over'：0を追加します。

parameters = {
   'token':'your_token',   
   'id_dataset':'5d4c3af79516290b01c83f51',
   'target_threshold':{'feature':'change', 'success_thr_over': 0}
}

APIからデータを再度プルすると、次のようになります。

df = pullObservationsToDF(parameters)
df.head()



属性「CHANGE」は新しい属性「change_over_0」に置き換えられ、「CHANGE」が正の場合は1、そうでない場合は0になります。これは機械学習のターゲットになります。

「明日」の観測を予測したい場合、明日からの情報が利用できなくなるので、1周期の遅延を加えましょう。

parameters = { 
   'token':'your_token',
   'id_dataset':'5d4c3af79516290b01c83f51',
   'target_threshold':{'feature':'change','success_thr_over' : 0},
   'lag_target_feature':{'feature':'change_over_0', 'periods' : 1}
}
df = pullObservationsToDF(parameters)
df.head()



これは単に'change_over_0'を前日（期間）のデータに合わせ、その名前を'TARGET_change_over_0'に変更するだけです。

依存関係を見てみましょう：

target_variable = 'TARGET_change_over_0'
df = df.dropna()
df.corr()[target_variable].sort_values()



それらは線形独立であり、有用である可能性は低いです。

ステップ3.ビジネスニュースデータを取得する


OpenBlenderで 依存関係を検索した後、ターゲットの良い予測を生成するのに役立つFox Business Newsデータセットを見つけました。ニュースの要約で単語と単語のグループの繰り返しを数えることにより



、「title」列の値を数値特性に変換し、ビットコインデータセットと時間内で比較する方法を見つける必要があります。思ったより簡単です。

最初に、ニュースの'title'属性のTextVectorizerを作成する必要があります。

action = 'API_createTextVectorizer'
vectorizer_parameters = {
    'token' : 'your_token',
    'name' : 'Fox Business TextVectorizer',
    'sources':[{'id_dataset' : '5d571f9e9516293a12ad4f6d',
                'features' : ['title']}],
    'ngram_range' : {'min' : 1, 'max' : 2},
    'language' : 'en',
    'remove_stop_words' : 'on',
    'min_count_limit' : 2
}

すべての標識を数字の形でトークンワードとして取得するためのベクトライザーを作成します。上記では、次のことを示しました。

  • 名前：'Fox Business TextVectorizer'としましょう。
  • anchor：データセットのIDおよびソースとして使用する必要がある特性の名前（この場合、「title」列のみ）。
  • ngram_range：トークン化のための一連の単語の最小長と最大長。
  • 言語：英語
  • remove_stop_words：ソースからストップワードを削除します。
  • min_count_limit：トークンと見なす必要のある繰り返しの最小数（1回発生することはほとんどありません）。

これを実行します。

res = OpenBlender.call(action, vectorizer_parameters)
res


回答： 

{  
    'message' : 'TextVectorizer created successfully.'
    'id_textVectorizer' : '5dc1a404951629331f6359dd',
    'num_ngrams': 4270
}

TextVectorizer が作成され、構成に応じて4270 n- gramが生成されました。少し後で、生成されたIDが必要になります：

5dc1a404951629331f6359dd

ステップ4.ビットコインデータセットと互換性のあるニュースの概要


次に、ニュースの要約とビットコインの為替レートのデータを時間で比較する必要があります。一般に、これは、タイムスタンプをキーとして使用して2つのデータセットを組み合わせる必要があることを意味します。結合したデータを元のデータ抽出オプションに追加してみましょう。

parameters = { 
   'token':'your_token',  
   'id_dataset':'5d4c3af79516290b01c83f51',
   'target_threshold' : {'feature':'change','success_thr_over':0},
   'lag_target_feature' : {'feature':'change_over_0', 'periods':1},
   'blends':[{'id_blend':'5dc1a404951629331f6359dd',
               'blend_type' : 'text_ts',
               'restriction' : 'predictive', 
               'specifications':{'time_interval_size' : 3600*12 }}]
}

上記では、次のことを示しました。

  • id_blend：textVectorizerの ID。
  • blend_type：'text_ts'これにより、Pythonはテキストとタイムスタンプの混合であることを認識します。
  • 制限：'予測的'で、将来のニュースとすべての観察結果の「混合」はなく、指定された時間より前に発行されたものだけです。
  • blend_class：'closest_observation'。これにより、最も近い観測のみが「混合」されます。
  • 仕様：観測の転送にかかる最大可能な経過時間、この場合は12時間（3600 * 12）。つまり、ビットコインの価格のすべての観測は、過去12時間のニュースに基づいて予測されます。

最後に、8月20日から始まる「date_filter」の日付でフィルターを追加します。これは、Fox Newsがデータの収集を開始したためです。「drop_non_numeric」を使用すると、数値のみが取得されます。

parameters = { 
   'token':'your_token',
   'id_dataset':'5d4c3af79516290b01c83f51',
   'target_threshold' : {'feature':'change','success_thr_over':0},
   'lag_target_feature' : {'feature':'change_over_0', 'periods':1},
   'blends':[{'id_blend':'5dc1a404951629331f6359dd',
               'blend_type' : 'text_ts',
               'restriction' : 'predictive',
               'blend_class' : 'closest_observation', 
               'specifications':{'time_interval_size' : 3600*12 }}],
   'date_filter':{'start_date':'2019-08-20T16:59:35.825Z',
                   'end_date':'2019-11-04T17:59:35.825Z'},
   'drop_non_numeric' : 1
}

注：11月4日は「end_date」として示しました。このコードを記述した日なので、日付を変更できます。

もう一度データを取得しましょう：

df = pullObservationsToDF(parameters)
print(df.shape)
df.head()

（57、2115）



これで、トークンと57個の観測値を持つ2000個を超える標識があります。

ステップ5.予測ターゲットにMLを適用する


さて、最後に、クリーンなデータセットができました。ターゲットと関連する数値データの時間オフセットを使用して、必要なものとまったく同じに見えます。'Target_change_over_0'

との最も高い相関関係を見てみましょう。



これで、いくつかの相関属性があります。データセットを時系列でトレーニングとテストに分割して、初期の観測でモデルをトレーニングし、後の観測でテストできるようにします。

X = df.loc[:, df.columns != target_variable].values
y = df.loc[:,[target_variable]].values
div = int(round(len(X) * 0.29))
# We take the first observations as test and the last as train because the dataset is ordered by timestamp descending.
X_test = X[:div]
y_test = y[:div]
print(X_test.shape)
print(y_test.shape)
X_train = X[div:]
y_train = y[div:]
print(X_train.shape)
print(y_train.shape)



トレーニング用に40個、テスト用に17個の観測があります。
次に、必要なライブラリをインポートします。

from sklearn.ensemble import RandomForestRegressor
from sklearn.metrics import mean_squared_error
from sklearn.metrics import accuracy_score
from sklearn.metrics import roc_auc_score
from sklearn import metrics

ここで、ランダムフォレスト（RandomForest）を使用して予測を行います。

rf = RandomForestRegressor(n_estimators = 1000)
rf.fit(X_train, y_train)
y_pred = rf.predict(X_test)

物事を簡単にするために、予測とy_testをデータフレームに入れましょう：

df_res = pd.DataFrame({'y_test':y_test[:,0], 'y_pred':y_pred})
df_res.head()

私たちの本当の「y_test」はバイナリですが、予測はfloat型であるため、0.5より大きい場合は価格の上昇を意味し、0.5未満の場合は低下すると仮定して切り上げます。

threshold = 0.5
preds = [1 if val > threshold else 0 for val in df_res['y_pred']]

ここで、結果をよりよく理解するために、AUC、エラーマトリックス、および精度インジケーターを取得します。

print(roc_auc_score(preds, df_res['y_test']))
print(metrics.confusion_matrix(preds, df_res['y_test']))
print(accuracy_score(preds, df_res['y_test']))



正しい予測の64.7％が0.65 AUCで得られました。

  • 9回、下落を予測し、価格が下がった（右）。
  • 5回、私たちは減少を予測し、価格は（誤って）上昇しました。
  • 1回、増加を予測しましたが、価格が不正確に減少しました）;
  • 2倍の増加を予測し、価格が上昇しました（true）。
