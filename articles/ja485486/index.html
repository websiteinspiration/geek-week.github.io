Spring Boot 2.3.0でのビルドパックのサポート

数日前、Spring Boot 2.3.0.M1がリリースされました。その説明では、最初の行がCloud Native Buildpacksプロジェクトのサポートを示しています。これは、開発者の生活を簡素化し、ソースコードからのイメージの最も自動化されたアセンブリを可能にする試みです。私たちのマイクロサービスは現在のプロジェクトのコンテナーに存在する必要があるため、私はそれを試して、何が利点であるかを理解することにしました。カットの下の短い続き。

Cloud Native Buildpacks（CNB）を使用すると、開発者は独自のDockerfileを作成したりサポートしたりできません。 CNBプロジェクトを構築する唯一のチームを起動することにより、イメージに組み込む依存関係を決定します。これは次のようになります。



さらに、さまざまなテクノロジーにより、OCIサポート、モジュール構造、依存層キャッシュ、CNBシステムは広く使用でき、開発者によるとかなり高速です。

彼が実際に何であるかを確認してください。サイトstart.spring.io空のプロジェクトをダウンロードします。依存関係を少し追加して、コンテナーがまったく機能するかどうかを確認します。

これで準備は終わりです。アプリケーションをコンテナに配置する準備ができました。これを行うには、gradle "./gradlew bootBuildImage"で実行するか、maven "./mvnw spring-boot：build-image"とビルドが始まります。組み立て後、プロセス全体の説明を含む便利なログを取得します。
