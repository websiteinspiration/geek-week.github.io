創造の秘密からベールを取り除くために、私はいつも最初のことに興味がありました。おそらく、そのような人々が考古学者になる前に、そして土、砂、粘土を掘りました。これですべてが変わったので、データを掘る必要があります。

さらに、そのような「肥料」は何倍にもなり、ツールは自動化されています。長年にわたり、世界中の天文学者があらゆる種類の宇宙物体に関するビットごとのデータを収集し、それらをタイプとクラスに分け、太陽中心の位置を計算し、光度、赤方偏移などを測定してきました。

ハーフ法


空間を太陽を通る平面で2つの半球に分割すると、要素AとBの2つのセットが得られ、それぞれに特定の数の要素[QuantityA]と[QuantityB]があり、合計KKS [RSh_SumA]と[RSh_SumB]があります。その結果、要素あたりの平均KKS数[RSh_midA]と[RSh_midB]、およびそれらの差[RSh_dif]。

便宜上、座標系を修正すると便利です。

軸Xは、原点（太陽）と、0度0分、および0度の傾きの赤経点を含む直線です。つまり、春分点の方向と一致します。 Y軸も赤道面にあります-赤経9時間0分0秒、傾き0度。 Z軸-90度の傾斜、任意の右上がり。

3つの参照面も定義します。これを垂直線で行うと便利です。

平面αはzに垂直で、xとyを含みます。
平面βはyに垂直で、xとzを含みます。
平面γはxに垂直で、yとzを含みます。

そして、これらの平面の平均変位の違いを見てください：

私は各オブジェクトの宇宙座標系の平均偏差の値に読者を焦点を合わせます：北半球の各空間オブジェクト（以下、太陽に対する2番目の赤道座標系）、または上半分の空間、すべてのオブジェクトよりもスペクトルが赤に約0.075シフト南半球で。

約11,230 km / sの速度で、相対的に移動しているかのように（α= 22,460.1 km / sに対する差の値を2で割ります）。ここで2に分割するのは、空間の反対側を基準にして赤方偏移の値を取得したという事実によるものです。その対象には、紫の方向にシフトするオブジェクトがあり、赤方偏移の値に等しいため、方向の変位に2倍の差が生じます。

しかし、「世界中の」平均値のこのような違いは、太陽系と天の川の既知の特異な速度と比較して非常に大きく、その最大値は遺物に対して627±22 km / sにすぎません。

それはうまくレイアウト。

最初は、この違いの主な理由は、Friedmann-Robertson-Walkerメトリックに従って赤方偏移のスケール成分が分離されていないことにあると想定していました。

つまり、光源からの距離が長くなると、赤方偏移は時間的に非線形に増加し、宇宙の膨張のかなりの成分が含まれ、放射の「古い層」に対する受信点の固有の速度は「明るく」表現されます。
ただし、あらゆる年代の放射線に対する受信ポイントの固有の速度は、同じスケールなしの結果になります。

違いの主な理由は、時代のスケールで発生した天の川の中心を中心とした反復および同一方向の動き（回転）にあります。したがって、固定された違いに多大な貢献をすると同時に、穀物を籾殻から分離する可能性を提供します。また、おそらく、さらに長い方向性のある動きが存在する場合。

これについては、「COPのメモリの影響」の章の後半で説明しますが、ここでは、データを参照しながら、さらに深く掘り下げます。どうすれば物理現象の理解を深めることができますか？

まず、平面上の特定のデータは、特定のモーションベクトルの投影を作成できます。絶対値では、特定のモーションベクトルよりも大きくなるはずです。つまり、そのような動きがあるという仮定を確認したい場合（たとえば、現象のフレームワーク内）暗いストリームまたは大きなアトラクタ）、最大値を探すために分割平面を回転する価値があります。
次に、1点からアクションを実行できます-平面による回転、両側のオブジェクトの選択を徐々に減らし、分割平面を定義するベクトルの周りのトロイダル領域にある要素を削除することにより、オブジェクトを制限します。あたかも分割面の両側にスポットライトビームを狭めるかのように。

赤方偏移の差が宇宙空間の側面に対する観測点の動きによるものである場合、除外されたオブジェクトはこの性質の赤方偏移への寄与が小さいという事実により、このような狭めは平均赤方偏移を一貫して増加させるはずです。

3番目に、分割平面の回転中に、最大値とその方向がどのように変化したかを追跡するために、KKS範囲の一部のみを考慮することができます。これを2番目の段落のトリックと組み合わせます。

分割平面の回転


次のように1回のパスを15〜20分に短縮してラップトップの作業を容易にしたため、これはいくつかの仮定のテストにすぎません。オブジェクトの角度の秒は除外されます（値への寄与は無視できます）。平面は5°ずつ回転します。

回転のメカニズムは次のとおりです。0°から90°までの傾斜値ごとに、5°の増分で0°から360°まで、5°の増分で右の上昇値が渡されます。

したがって、すべての可能な角度で半球を通過させます。2番目の半球を通過する意味はありません-それは反対の符号で完全にミラーリングされます（たとえば、0°の傾きでの赤経の最初の通過）。

WHILE (@counter1 < 19)

	WHILE (@counter2 < 72)

		SET @Q1 = @counter1;

		SET @RA_surf_ort_angle = @counter2 * @RA_surf_step;
		SET @DEC_surf_ort_angle = @counter1 * @DEC_surf_step;

		SELECT @RSh_sumB = SUM([Z]) FROM dbo.RSh8 where ([Z]>@DIAPASON_L) AND ([Z]<@DIAPASON_H) AND ([Z]<>0) AND ((SQUARE([Xd]+@X_ort_neg)+SQUARE([Yd]+@Y_ort_neg)+SQUARE([Zd]+@Z_ort_neg))>@threshold);
		SET @RSh_sumCheck = @RSh_sumA+@RSh_sumB;
		SET @RSh_midA = @RSh_sumA / @QuantityA;
