F＃8：差別的な組合

したがって、F＃の旅は続きます。レコード/タプルなど、いくつかの基本的なタイプのビルディングブロックを確認しました。次に、マークアップされた関連付けを確認します。

ラベル付き共用体は、いくつかの可能な値の1つである可能性がある値をサポートします。可能な値は「複合ケース」と呼ばれ、以下に示す形式を取ります。

case-identifier1 [of [ fieldname1 : ] type1 [ * [ fieldname2 : ] type2 …]

この構文が威圧的に見えても心配する必要はありません。実際に重要なのは、各ケースを他のケースと区別（区別）できるようにするラベルの存在と、組み合わせるケースのタイプです。名前には、次のような特定の規則があります。

  • 大文字で始める必要があります
  • ユニオンタイプ自体の名前を含む識別子にすることもできます。これは少し混乱するかもしれませんが、マージの場合を説明するのに役立ちます。





これは 不正な識別子の例です。ユニオンレジスタに一致するラベル識別子を使用すると、前述のように完全に有効なラベル識別子を使用すると次のようになります。

type LabelUnionType = Int of int | String of string
