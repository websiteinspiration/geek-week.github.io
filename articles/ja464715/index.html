最小グラフの大きなグラフの視覚化



グラフを描画する必要があるが、腕の下にあるツールが髪の毛のようなものを描画したり、すべてのRAMを食い込んでシステムをハングアップしたりする場合はどうすればよいですか？過去2、3年間、大きなグラフ（数億の頂点とエッジ）を扱ってきましたが、多くのツールとアプローチを試しましたが、まともなレビューはほとんど見つかりませんでした。したがって、今、私はそのようなレビューを自分で書いています。

なぜグラフを視覚化するのですか？


何を探すかを見つける


入力には通常、頂点とエッジのセットがあり、これは本質的にグラフです。いくつかの統計的特性を計算し、そこからメトリックをグラフ化できますが、これではそれが何であるかを明確に把握できません。優れた視覚化は、クラスターグラフにブリッジがあるか、それとも同質であるか、または1つの塊に集まり、いくつかの主要な魅力の中心があるかを示します。

誇ります


データの視覚化は、定義により、表現の問題を解決します。いくつかの作業が既に行われている場合は、結論を壮観な写真で示すことができます。たとえば、グラフをクラスタリングした場合、グラフをクラスタに色分けして、さまざまなラベルが互いにどのように関連しているかを示すことができます。

サインを入手


グラフ視覚化アルゴリズムは主に画像を取得するためのツールとしてのみ開発されたという事実にもかかわらず、それらは次元を削減する方法として使用できます。グラフ自体は、隣接行列で表されている場合、高次元空間のデータであり、レンダリングすると、頂点ごとに2つの座標が取得されます（3つ以上の場合もありますが、これは例外です）。視覚化における頂点の近接性は、プロパティの類似性を表すことがよくあります。

大きなグラフの問題は何ですか？


大きなグラフとは、たとえば1万の頂点またはエッジから始まる次元を意味します。小さいサイズの場合、通常は問題はありません。インターネットでクイック検索して見つけられるほとんどすべてのツールは、基本的にそのようなボリュームの問題をうまく解決するためです。大きなグラフの何が問題になっていますか？2つの問題：これは読みやすさと速度です。オブジェクトが多いほど、ナビゲートが難しくなることが予想されます。大きなグラフの場合、理解できない写真がしばしば得られます。さらに、グラフアルゴリズムは一般に非常に遅く、多くは頂点やエッジの数の二乗（場合によっては立方体）に比例したアルゴリズムの複雑さを持っています。レンダリングを待つ場合でも、満足のいく結果を得るために多くのオプションを実行する余裕はまだありません。

理論


スタッキングは、座標をグラフの各頂点にマップする方法です。通常は平面上の座標について話しますが、一般的に言えば平面である必要はありません。単に2次元以上を使用する必要はほとんどありません。

良いスタイリングとは何ですか？



見た目が悪いとか言うのは簡単ですが、そのような評価をどのように行うかをマシンに説明することは困難です。 「良い」スタイルを作成するために、通常、いわゆる「美的」メトリックが最適化され、より客観的に定式化されます。それらの一部を以下に示します。

エッジの最小交差
すべてが単純です。交差が多い場合、小さい場合は「混乱」し、画像は「よりきれい」に見えます。

隣接するピークは互いに近く、隣接しないピークは遠くにあります。
グラフは、定義により、頂点と頂点のセット間の接続のセットです。接続された頂点を互いに近づけることは、グラフデータの基本的なプロパティを表現するための直接的かつ論理的な方法です。

コミュニティはクラスター化されている
これは前の段落から続きます。グラフの他の部分よりも相互に接続されているノードが多い場合、それらは「コミュニティ」を形成し、図では密集したクラスターのように見えるはずです。

頂点とエッジの最小の重なりは
かなり明白です。ここで1つまたは複数のオブジェクトを作成できない場合、視覚化の読み取りは不十分です。

頂点またはエッジ、あるいはその両方の分布は均一です。
この条件は、グラフのプロパティで少なくとも一部の構造を確認できない場合に役立ちます。たとえば、グラフ全体が1つの密集したクラスターである場合、1つの塗りつぶしスポットにマージできるようにするよりも、少なくとも接続の不均一性を確認するために、グラフをぼかします。



スタイリングとは


分類とタイプはたくさんありますが、これらの3つのタイプのスタイリングを検討することが重要だと思います。ただし、これらのタイプの知識は、主題をナビゲートするには十分です。

  • 力指向でエネルギーベース
  • 次元削減
  • ノードの機能

力指向でエネルギーベース




これらの方法は、体力のシミュレーションを使用します。頂点は、互いに反発する荷電粒子、およびエッジ-隣接する頂点を引き寄せる弾性ストリングとして表されます。次に、そのようなシステムの頂点の動きは、定常状態が確立されるまでシミュレートされます。他のアプローチでは、そのようなシステムのポテンシャルエネルギーを記述し、最小値に対応する頂点の位置を見つけようとします。

このアルゴリズムファミリの利点を図で示します。通常、グラフのトポロジーを反映する非常に優れたスタイルが得られます。構成する必要があるパラメーター間の短所。まあ、もちろん、計算の複雑さ。各頂点について、他のすべての頂点から作用する力を計算する必要があります。

重要なファミリーアルゴリズムは、Force Atlas、Fruchterman-Reingold、Kamada Kawaii、およびOpenOrdです。最後のアルゴリズムは、トリッキーな最適化を使用します。たとえば、長いエッジをカットオフして計算を高速化し、副作用として、密集した密集したピークのクラスターを取得します。

次元削減




グラフは、隣接行列、つまり正方行列NxNによって定義できます。Nは頂点の数です。これは、次元Nの空間内のN個のオブジェクトとして解釈できます。この表現により、tSNE、UMAP、PCAなどの次元を削減する普遍的な方法を使用できます。別のアプローチは、エッジの重みとローカルトポロジーの知識に基づいて頂点間の理論上の距離を計算し、より小さな次元のスペースに移動するときにこれらの距離間の関係を維持しようとすることです。

機能ベースのレイアウト




通常、データは頂点の隣接性に関する情報だけではなく、現実の世界から取得されます。ピークは、独自のプロパティを持つ実際のオブジェクトです。これを思い出して、頂点のプロパティを使用して平面に表示することができます。これを行うには、表形式のデータに一般的に使用される任意のアプローチを使用できます。これらは、前述のPCA、UMAP、tSNE、オートエンコーダーの次元を削減する方法です。または、フィーチャのペアに対して単純な散布図（散布図）を描画し、結果のビューの上に既にエッジを描画できます。これとは別に、Hive Plotについて言及することができます。これは、属性の値が、頂点が配置されている中心から向けられたさまざまな軸に対応し、エッジがそれらの間の円弧によって描画される場合の興味深い方法です。

大きなグラフ視覚化ツール



視覚化タスクは古くて比較的人気があるという事実にもかかわらず、大きなグラフ用のツールには大きな問題があります。それらのほとんどはサポートされていません。ほとんど誰もが我慢しなければならない彼らの深刻な欠点のいくつかを持っています。言及する価値があり、大きなグラフに使用できるものについてのみ説明します。小さなグラフの場合、問題はありません-ツールは見つけやすく、基本的にはうまく機能します。

これどうやってやるの？


最も明白な方法は、単純に隣人の平均を取ることです。何が発生し、平均が失う情報についてもう少し考える場合は、標準偏差、最小値、最大値などの他の統計を追加できます。

今それを整理する方法は？まず、グラフは本質的に多くの頂点と多くのエッジです。関心があるのは複数の頂点からの接続された断片のみなので、この場合のエッジのリストはグラフを完全に定義します。これは便宜的に表の形で書かれています。最初の列では、エッジが出てくる頂点、2番目の列では、これらのエッジが入る頂点です。次に、統計を読み取る必要があります。これは非常に一般的なタスクであるため、すべてが最適化されたフレームワークを使用できます。

グラフ分析におけるテーブルフレームワークの力


グラフを指定するプレートがあり、いくつかの量の統計を読み取る必要があるという結論に達しました。テーブルと統計-これはすべてパンダで行われるため、以下に実装例を示します。

まず、グラフを設定してみましょう。

ara = np.arange(101).reshape(-1, 1)
sample = np.vstack((np.hstack((ara[:-1], ara[1:])), # forward links
                    np.hstack((ara[1:], ara[:-1])))) # backward links

これは、図のように次々に接続された101個の頂点のチェーンです。



次に、頂点の符号をランダムに設定します。

feats = np.random.normal(size=(101, 10))

そして、このすべてからパンダデータフレームを作成します。

edges = pd.DataFrame(sample, columns=['source', 'target'])
cols = ['col{}'.format(i) for i in range(10)]
feats = pd.DataFrame(feats, columns=cols)

次に、畳み込み関数自体を設定します。

def make_conv(edges, feats, cols, by, on, size=1000000, agg_f='mean',
                      prefix='mean_'):
    """
    edges -- edgelist: pandas dataframe with two columns (arguments by and on)
    feats -- features dataframe with key column (argument on) 
             and features columns (argument cols)
    by -- column in edges to be used as source nodes
    on -- column in edges to be used as neighbor nodes
    size -- number of unique source nodes to be used in one chunk
    agg_f -- can be interpreted as pooling function. 
             Pandas has several optimised functions for basic statistics,
             that can be passed as string arg (see pandas docs),
             but you also can provide any function you like
    prefix -- prefix for new column names             
    """
    res_feats = [] # used to stack result chunks

    # get chunk of unique source nodes
    for chunk in tqdm(chunker(edges[by].unique(), size=size), 
                      total=(len(edges[by].unique()) // size) + 1):
        # for each chunk we get feature matrix for neighbours
        temp = edges[edges[by].isin(chunk)]\
                .merge(feats, on=on, how='left')
        # convolve and pool
        tempgb = temp[cols + [by, on]]\
                .groupby(by).agg({col: agg_f for col in cols}).reset_index()
        res_feats.append(tempgb.rename(columns={c: prefix + c for c in cols}))
    # concat results
    return pd.concat(res_feats, axis=0).reset_index(drop=True)

何が起きてる


最初に、コンボリューションを作成する頂点の一部を選択し、それらのすべてのエッジを取得し、それを近傍の記号と連結して、一時データフレームに書き込みます。次に、ソースの頂点でグループ化し、関数agg_fを使用して集計します。これは、デフォルトでは単純に平均です。現在のピースの結果をリストに追加し、最後に結果を連結します。

もう少し数学が欲しいなら


以前にグラフのたたみ込みについて聞いたことがある場合は、おそらくグラフのたたみ込みネットワーク（Graph Convolutional Networks-GCN）のコンテキストであったと考えられます。一部の人にとって、ここで説明されているタブレットのトリックは「難しくない」ように見えるかもしれません。実際、非常に興味深い記事の1つは、ディープラーニングを行わずにグラフのたたみ込みを使用することです。「グラフのたたみ込みネットワークの単純化」です。それは畳み込みのそのような定義を与えますH(k)←SH(k−1)ここでHは符号の行列であり、Sはグラフの正規化ラプラシアンで、次のように定義されます。S=D~−12A~D~12こちらA~単位行列と合計グラフの隣接行列であり、D~グラフの頂点の度合いが書き込まれるの対角に行列です。そして、これらすべてはscipyとnumpyを使用してpythonで数行に書かれています：

S = sparse.csgraph.laplacian(adj, normed=True)
feats_conv = S @ feats

ここで、sparseはscipy内のモジュールであり、sparse行列を操作します。adjは隣接行列であり、featsとfeats_convは畳み込みの前後の符号です。このアプローチはより厳密ですが、スケーリングが非常に不十分です。さらに、発生していることの意味について少し考えると、これは頂点でのフィーチャの値と頂点の近傍の平均値の差に相当します。つまり、1つの操作を追加すると、前のスキームを使用してテーブルで完全に解決されます。
