組み込みのGo機能


Goでは、アセンブラーで記述できます。しかし、言語の作者はそのような標準ライブラリを書いたので、実行する必要はありませんでした。ポータブルで高速なコードを同時に書く方法があります。どうやって？カットへようこそ。

goのアセンブラーで関数を書き始めるのは非常に簡単です。たとえば、Add2つのint64を追加する（前方宣言）関数を宣言します。

func Add(a int64, b int64) int64

これは通常の関数ですが、関数の本体がありません。コンパイラーは、パッケージをコンパイルしようとするときに合理的に誓います。

% go build
examples/asm
./decl.go:4:6: missing function body

拡張子が.sのファイルを追加し、アセンブラーで関数を実装します。

TEXT ·Add(SB),$0-24
    MOVQ a+0(FP), AX
    ADDQ b+8(FP), AX
    MOVQ AX, ret+16(FP)
    RET

これでAdd、通常の機能としてアセンブル、テスト、使用できます。これは、ランタイム、数学、bytealg、syscall、reflect、cryptパッケージで言語開発者自身が広く使用しています。これにより、言語で表されていないハードウェアプロセッサの最適化とコマンドを使用できます。

しかし、問題があります-asmの関数を最適化して組み込むことができません（インライン）。これがないと、オーバーヘッドが非常に大きくなる可能性があります。

Go組み込み関数は通常のgoで記述されています。しかし、コンパイラはそれらをより最適なものに置き換えることができることを知っています。Go 1.13では、埋め込み関数はmath/bitsおよびに含まれていsync/atomicます。

これらのパッケージの関数には、派手な署名があります。実際、それらはプロセッサコマンドの署名を繰り返します。これにより、ターゲットアーキテクチャがサポートしている場合、コンパイラーは関数呼び出しをアセンブラー命令に透過的に置き換えることができます。

以下では、goコンパイラーが組み込み関数を使用してより効率的なコードを作成する2つの方法について説明します。

人口数


数値のバイナリ表現におけるこの数の単位は、重要な暗号プリミティブです。これは重要な操作であるため、最新のCPUのほとんどはハードウェアでの実装を提供します。
パッケージmath/bitsは、この操作を関数で提供しますOnesCount*。これらは認識され、プロセッサ命令に置き換えられますPOPCNT。

これがいかに効率的であるかを確認するために、3つの実装を比較してみましょう。1つ目はカーニガンアルゴリズムです。

func kernighan(x uint64) (count int) {
	for x > 0 {
		count++
		x &= x - 1
	}
	return count
}

アルゴリズムのサイクル数は、設定されたビット数と一致します。ビット数が多い-実行時間が長くなるため、サードパーティのチャネルで情報が漏洩する可能性があります。

2番目のアルゴリズムは、Hacker's Delightの本から引用したものです。

func hackersdelight(x uint64) uint8 {
	const m1 = 0b0101010101010101010101010101010101010101010101010101010101010101
	const m2 = 0b0011001100110011001100110011001100110011001100110011001100110011
	const m4 = 0b0000111100001111000011110000111100001111000011110000111100001111
	const h1 = 0b0000000100000001000000010000000100000001000000010000000100000001

	x -= (x >> 1) & m1
	x = (x & m2) + ((x >> 2) & m2)
	x = (x + (x >> 4)) & m4
	return uint8((x * h1) >> 56)
}

分割統治戦略により、このバージョンは、数値の長さのO（log₂）、および暗号化にとって重要なビット数の一定時間で機能します。とパフォーマンスを比較してみましょうmath/bits.OnesCount64。

func BenchmarkKernighan(b *testing.B) {
	var r int
	for i := 0; i < b.N; i++ {
		r = kernighan(uint64(i))
	}
	runtime.KeepAlive(r)
}

func BenchmarkPopcnt(b *testing.B) {
	var r int
	for i := 0; i < b.N; i++ {

原子カウンター


組み込み関数は通常のGoコードであり、命令ストリーム内でインラインにすることができます。たとえば、アトミックパッケージの関数の奇妙なシグネチャからのメソッドを使用して、カウンターを抽象化します。

package main

import (
	"sync/atomic"
)

type counter uint64

func (c *counter) get() uint64 {
	return atomic.LoadUint64((*uint64)(c))
}

func (c *counter) inc() uint64 {
	return atomic.AddUint64((*uint64)(c), 1)
}

func (c *counter) reset() uint64 {
	return atomic.SwapUint64((*uint64)(c), 0)
}

func F() uint64 {
	var c counter
	c.inc()
	c.get()
	return c.reset()
}

func main() {
	F()
}

誰かがそのようなOOPはオーバーヘッドを追加すると考えるでしょう。ただし、GoはJavaではありません。明示的にインターフェースを使用しない限り、ランタイムではバインディングは使用されません。上記のコードは、プロセッサ命令の効率的なストリームにまとめられます。メインはどのようになりますか？


順番に。c.incに変わりますlock xadd [rax], 1-x86のアトミック追加。x86ではすでにアトミックなc.get通常の命令になりmovます。無効化されたレジスタとメモリ間c.resetのアトミック交換にxchgなります。
