私の目標は、ソニーのカメラ用のコンパクトで最も安価なリモコンを開発することです

まず、カメラのメニューでWiFiがオンになっていることを確認する必要があります。

[セットアップ]-> [CONFG]-> [Wi-Fi]-> [オン]

識別子とパスワードは、カメラの取扱説明書の別のステッカーに記載されている必要があります。このステッカーが紛失した場合は、MicoUSB経由でカメラをMicrosoft Windowsを実行しているコンピューターに接続し、電源を入れることができます。 [USB]が画面に表示され、コンピュータに2つのネットワークドライブが表示されます。カメラにメモリカードが挿入され、PMHOMEカメラの内部メモリが挿入されています。 2つ目のディスクに関心があり

ます。ファイル\\ INFO \ WIFI_INF.TXTには、WiFi経由でカメラにアクセスするための識別子とパスワードが含まれ、ファイル\\ INFO \ WPS_PIN.TXTには、WPS経由で接続するためのキーピンが含まれています。情報へのアクセスは読み取り専用であるため、識別子、パスワード、またはPINを変更することはできません。

これで、このデータを使用してカメラに接続できます。接続後のカメラのIPアドレスは192.168.122.1です。カメラを制御するには、HTTP POSTリクエストをアドレス192.168.122.1：10000 / sony / camera に送信する必要があります。

3番目に、リクエスト構造と応答。各リクエストには、JSON形式のコマンドが含まれており、JSON形式でも応答を返します。コマンドの完全なリストと使用例は、同じPDFファイルに含まれています。

また、HTTPプロトコルを介して、写真付きの完成したファイルとカメラから表示するためのビデオストリームが返されます。

最も複雑なプロトコルは、情報の要求です。リクエストのバージョン（1.0-1.3）に応じて、JSON形式で34から62のパラメーターの配列を提供します。これは、配列にすることもでき、2つの操作モードがあります-カメラの状態に関する完全な情報への即時応答と、イベントの応答カメラ上（例：モードの切り替えまたは記録の開始）

ファームウェア


ESP8266コアがインストールされたArduino IDEでファームウェアを開発したため、この部分はarduino-hatersにスキップすることをお勧めします。便宜上、追加のライブラリを使用しました。

  • Adafruit GFXグラフィックライブラリ
  • OLED Adafruit SSD1306
  • WiFiManager
  • ArduinoJson — JSON

グラフィックライブラリを使用すると、すべてが明確になります。 OLEDディスプレイを操作するために必要です。 WiFiManagerは、ESPでWiFi接続をセットアップするための非常に便利なライブラリです。設定モードに入る場合、アクセスポイントをESPとその最小限のWEBサーバーに上げ、接続時にリダイレクトします。自動モードで開始することもできますが、カメラが接続されていないときは常に設定モードが開始されます。 「BOOT / MODE」ボタンを長押しして設定モードに入り、120秒のタイムアウトで終了することにしました。

WiFiManagerライブラリーは、カスタマイズ用の独自のフィールド（このプロジェクトでは私には役立ちませんでした）を追加したり、イベントを処理したり（セットアップモードに入って設定を保存したり）できるという点でも便利です。

カメラに接続するための設定が構成された後、プログラムは画面に書き込まれる接続を行います。接続が確立されると、プログラムは現在のステータスを毎秒カメラに要求し、変更を画面に表示します。これは、カメラコントロールをボタンから直接追跡できるようにするためです。

「BOOT / MODE」ボタンを使用して、「VIDEO」、「PHOTO」、「LAPSE」の3つのメインカメラモードを周期的に切り替えます。 PLAY / STOPボタンを使用して、VIDEOまたはLAPSEモードでの記録を有効/無効にし、PHOTOモードで1枚の写真を記録します。録音がオンになると、赤いLEDが点灯します。記録中は、写真が1回点滅します。

GPIO14には追加の入力もあり、本質的に「PLAY / STOP」ボタンを複製します。この入力は外部トリガーを接続するために必要です。外部トリガーは、タイムラプスショットの写真キャプチャを外部イベントと同期させることができます。この入力は、3Dプリンターに接続して、タイムラプスビデオプリントパーツを撮影する予定です。

追加機能のうち、プログラムは定期的にADC入力の値を測定します。ADC入力の値はバッテリーから直接接続されており、アイコンでバッテリーの状態を示します。プログラムは、分周器R12 = 1.6KおよびR11 = 10Kとリチウム電池で調整されます。

残念ながら、AS100は、タイムラプスモードでの記録時間とショット数のJSONでの出力をサポートしていません。スケッチでは、これらのパラメータは変数に書き込まれ、誰かが繰り返すと、他のカメラでそれらを表示することもできます。
