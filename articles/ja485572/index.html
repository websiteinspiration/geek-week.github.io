Visual Studio for MacでBlazorサーバーアプリケーションを使い始める

Visual Studio 2019 for Mac v8.4では、Blazorサーバーアプリケーション開発サポートが追加されました。この記事では、Visual Studio for Macを使用して新しいBlazorサーバーアプリケーションの作成を開始する方法を説明します。Blazorでは、JavaScriptの代わりにC＃を使用してインタラクティブなWebインターフェイスを作成できます。Blazorアプリケーションは、C＃、HTML、およびCSSを使用して実装された再利用可能なWebインターフェイスコンポーネントで構成されています。クライアントとサーバーのコードはC＃で記述されているため、コードとライブラリを共有できます。



新しいBlazorプロジェクトを作成する


Visual Studio for Macを初めて起動すると、次のダイアログボックスが表示されます。



新しいBlazorサーバーアプリケーションを作成するには、[新規]をクリックするか、[ファイル]メニューから[新しいソリューション]を選択します。



これを行うと、[新しいプロジェクト]ダイアログボックスが開きます。 Blazorサーバーアプリケーションを作成するには、.NET Core- > Appセクションに移動し、Blazorサーバーアプリを選択します。



[次へ]をクリックすると、.NET Coreのバージョンを選択するように求められます。公開時にデフォルトバージョンの.NET Core 3.1を選択するか、別のバージョンに変更できます。 Blazorアプリケーションには、.NET Coreバージョン3.0以降が必要です。 「次へ」をクリックすると、セットアップウィザードの次のページに移動し、新しいプロジェクトに名前を付けることができます。私はこの新しいプロジェクトをHelloBlazorと呼びました。



これで新しいプロジェクトが設定されたので、[作成]をクリックして（または戻って）作成を完了できます。プロジェクトを作成すると、IDEで開きます。 Visual Studio for MacエディターでIndex.razorファイルを開きました。下のスクリーンショットで確認できます。



プロジェクトが作成されたので、最初に行う必要があるのは、アプリケーションを起動して、すべてが期待どおりに機能していることを確認することです。[実行]> [デバッグの開始]または[実行]> [デバッグなしで開始]を使用して、新しいBlazorアプリケーションを開始できます。



この場合は「デバッグなしで開始」に進みましょう。デバッグセッションよりも速く開始されるため、ここではデバッグを実行しません。デバッグせずに開始するには、同じ名前のメニュー項目（上の図に示すように）または⌥⌘⏎キーの組み合わせを使用できます。アプリケーションを起動すると、デフォルトのブラウザで開きます。次の図に示すツールバーのブラウザセレクタを使用して、実行中のブラウザを変更できます。



「デバッグなしで開始」 のホットキーを使用して、このアプリケーションを起動してみましょう。プロジェクトがアセンブルされると、アプリケーションがブラウザーで開かれます。

これでプロジェクトが稼働し始めたので、その設定を少し楽しんでみることをお勧めします。

作成されたプロジェクトには、ボタンをクリックしてカウンターの数を増やすことができるカウンターページがあります。ユーザーに増分値を指定させて、このページを変更してみましょう。これを行うには、Counterページに入力フィールドを追加し、アカウントをインクリメントするために使用される新しいインクリメントプロパティにリンクします。次のスクリーンショットで更新されたCounter.razorコードを見てください。



このコードをコピーしてプロジェクトに貼り付ける場合は、コードスニペットを以下に示します。

@page "/counter"

<h1>Counter</h1>

<input type="number" min="1" step="1" @bind-value="increment" />
<p>Current count: @currentCount</p>

<button class="btn btn-primary" @onclick="IncrementCount">Click me</button>

@code {
    public int increment = 1;
    private int currentCount = 0;

    private void IncrementCount()
    {
        currentCount += increment;
    }
}


上記のコードのスクリーンショットで、矢印で示されている行は、コードの新しい行または編集された行です。ここでは、増分を設定するには、ユーザーのための新しい入力フィールド（ライン5）を加え、私たちは、インクリメント値を保存するために、インクリメントプロパティ（ライン11）を加え、そして最後に、我々は1のハード一連の増分の代わりに増分値を使用するためにライン16を変更

するには行った変更が適切に機能していることを確認するために、デバッグセッションを開始します。currentCountを増やすときにブレークポイントを設定しましょう16行目。このブレークポイントを設定した後、ショートカットを使用してデバッグを開始します。ブレークポイントに到達すると、増分の値が[カウンター]ページの入力フィールドから取得されていることを確認できます。以下のGIFで、ブレークポイントの作成、アプリケーションのデバッグ、およびブレークポイントに到達したときの増分の値の確認を確認できます。



すべてが順調に進んだ場合、増分値は[カウンタ]ページの入力フィールドから取得され、アプリケーションは正しく動作しています。Blazorサーバーアプリケーションを作成、編集、デバッグする方法を説明したので、今度はこの記事を完成させましょう。

最終的に


この記事では、新しいBlazorサーバーアプリケーションを作成し、それをVisual Studio for Macで操作する方法を理解しました。Visual Studio for Macをまだお持ちでない場合は、ダウンロードして開始してください。Visual Studio for Macユーザーの場合は、Visual Studio for Macをバージョン8.4以降にアップグレードして、Blazorサーバーアプリケーションのサポートを取得してください。Blazorサーバーアプリケーションの開発に加えて、それらをAzure App Servicesに公開することもできます。

Visual Studio for Macでの作業中に問題が発生した場合は、問題の報告機能を使用して問題をお知らせください。それらを修正して製品を改善できるようにします。終了する前に、ここにいくつかの追加リソースがあります。
