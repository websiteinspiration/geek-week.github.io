フィールドスタッフのトレーニングと認定のデジタル変換

今日は、継続的な従業員トレーニングのプロセスを自動化し、このプロセスから事務処理を除外するのに役立つソリューションについて説明します。

連邦製造会社です。彼女のマーチャンダイザーは、店舗での商品の在庫、棚の表示、POSM資料（広告）、製品に関する販売者の知識などを担当します。監督者は、現場の従業員を訓練、評価、および管理します。これには企業基準と明確に規制されたプロセスがあります。

以前は、トレーニングまたは認定の結果に関する貴重な情報は紙にのみ記録されていました。日中、スーパーバイザーはマーチャンダイザーと一緒に20の販売店を通過し、それぞれに要約用紙フォームに記入しました。それから、それは人にフィードバックを与えるために処理されることでした。



私たちTrue Engineeringは、監督者がチームの開発を計画し、結果を記録し、棚卸するのに役立つ技術ツールを作成しました。この記事では詳細を説明します。

開発プロセスはどのように組織されましたか


開発プロセスは多段階です。小売店でのマーチャンダイザーの操作中にトレーニングが提供されます。伴奏はトレーニングと認定と交互になります。1.現場での頻繁なエスコートによる初心者の専門家の急速な成長

： スーパーバイザーはマーチャンダイザーを選択し、プレゼンテーションスキルの開発と異論への取り組みなど、伴奏の2つの目標を設定します。それから彼はアンケートを印刷し、20のアウトレットへのマーチャンダイザーとのルートに沿って行きます。彼は一日中マーチャンダイザーの作業を監視し、従業員が前回行ったアドバイスやコメントを考慮に入れているかどうかを確認し、推奨事項を提供し、アンケートのすべての結果を修正します。 1日の終わりに、彼は次の伴奏で考慮される開発計画をまとめて作成します。





2.経験豊富な従業員の仕事の質を定期的な認証とアウトレットの監査を通じて維持する証明

はトレーニングと同様に行われます。監督者はアウトレットの従業員に付き添い、彼の仕事の質を監視します。監査はマーチャンダイザーなしで行われます。スーパーバイザーは、各店舗の主要なインジケーターを印刷した後、自分で買い物に行きます。訪問中、彼は要約フォームに記入し、電話で写真レポートを作成します。結果に基づいて、監督者は開発計画を調整し、提案を行います。

したがって、1日の終わりまでに、監督者は紙のフォームに記入しました。次に、従業員へのフィードバックのために最終結果を準備し、チーム全体の結果をExcelスプレッドシートに転送するために、追加の時間を必要としました。これらすべてが、チームの発展のダイナミクスを追跡し、次のイベントを計画するのに役立ちました。1か月以内に、監督者は各部下と面会する必要があり、平均して7人います。

このプロセスには明らかな困難がありました：


  1. たくさんのペーパーワーク。人々は紙のアンケートに記入した-それは不便で疲れる。
  2. データは断片化されていました。マーチャンダイザーの見積もりは紙の形式で、製品のレイアウトの写真は電話にありました。そしてフィードバックを与えるために、それらはまとめられて処理される必要がありました。
  3. , Excel.
  4. — . Excel .


私たちのソリューションは、WebポータルとiPad用のモバイルアプリケーションで構成されています。



ポータルでは、スーパーバイザーが1週間の販売店のトレーニング、認定、監査を計画しています。エスコート中に、監督者はモバイルアプリケーションでアンケートに記入します。会社の基準に従って仕事の質を評価し、目標を計画し、従業員の長所と短所を示し、自己啓発のためのタスクを策定します。次に、マーチャンダイザーにフィードバックを提供し、結果をストレージにポータルに送信します。

ポータルに戻り、スーパーバイザーはその期間のチームまたは特定の従業員の結果と開発ダイナミクスを分析します。そして、この情報に基づいて、彼は以下のイベントを計画しています。



ポータルAngular 5（当時の最新バージョン）、. Net Core 2.2のバックエンド、およびSwiftのモバイルアプリケーションを開発しました。

バックエンドはマイクロサービスアーキテクチャに基づいて実装され、単一の責任に基づいて9つの分離されたサービスに分割されます。 1つのサービスは計画のみを扱い、もう1つは結果の保存と表示、3つ目はモバイルアプリケーションとの対話、4つ目は写真の操作などです。このため、あるサービスの変更が別のサービスの作業に影響を与えることはありません。



マイクロサービスは、RabbitMQ +大量輸送バスを介して相互に通信します。各サービスには独自のデータベースがあり、「外部」データベースへのアクセスは禁止されています。

また、ユーザーとその役割のリスト、従属、アウトレットのリスト、製品情報、およびその他のデータが格納されている個別の顧客データベースもあります。このデータは、Service.Integration統合サービスによって同期されます。 （アーキテクチャの詳細以前にHabrについて書いた。

トレーニングと認定のデジタル変換


• イベントの計画

計画プロセスでは、監督者が非常に多くの要因に依存していることがわかりました。

  • 従業員の能力レベル、
  • 一連の活動
  • 最近のトレーニング、認定、監査の結果、
  • 強みと弱み
  • 開発の目標と目的を設定し、
  • 1か月間、各従業員に費やした時間。

したがって、私たちはUI / UXだけでなく、よく考えられた直感的な計画方法を提案しました。これにより、監督者は、意思決定のためのすべての要約情報を確認し、学習プロセスで何も見落とすことがなくなります。

ポータルでは、監督者は自分のチーム全体を見て、かかった時間と以前に実行されたアクティビティに関するすべての情報を考慮して、来月の一連のイベントを計画します。このアプローチにより、チーム全体により均一なトレーニングが提供されます。焦点は、適切な人と指標すべてにあります。初心者が今週トレーニングを行った場合、来週はそれ以降の進行状況を理解するために認定を計画する必要があり、その後、アウトレットを監査できます。これらすべては、毎月のプレゼンテーションの一部として簡単に整理でき、プランはドラッグアンドドロップで簡単に変更できます。



毎週のパフォーマンスは、プロセスを見失うことがないように、計画されたアクティビティのより詳細なチンキにすでに使用されています。



• 活動の設定

スーパーバイザーは、計画された活動カードを開きます。これは、過去の随伴からの従業員の長所と短所に関する情報を自動的に置き換えます。これにより、次のトレーニングの目標を選択するときに、レベルを思い出し、情報に基づいた決定を行うことが容易になります。同時に、ユーザーはコメントを追加して、以前のエスコートの結果を見ることができます。



たとえば、店舗で予定されている従業員トレーニングは次のようになります。



計画された活動のカードは、このイベントのアンケートテンプレートに直接関連しています。トレーニング、認定、監査の目的でのサポートなど、あらゆるタイプのイベントのテンプレートを作成しました。テンプレートは、質問と回答のさまざまなブロックのセットであり、それに基づいてアンケートが作成されます。

テンプレートでは、質問のリスト、順序、タイプ、重み、質問の形式を作成できます。

  • 答えはい/いいえ
  • リストからオプションを選択し、
  • テキストボックスなど。

さまざまな質問をグループ化できます。たとえば、製品のリストを含むブロックを生成でき、各フィールドの反対側に数量を入力するフィールドがあります。または、質問のリストをアクティビティの領域ごとにグループにまとめます。たとえば、すべての質問で、異論への取り組みのスキルをテストします。

Webポータルはスケジューリングサービスと連携して動作します。スケジューリングサービスは、データバスを介して、計画されたアクティビティのすべての変更について他のサービスに通知します。

• モバイルアプリケーションとポータルの同期ポータルでの

計画後、スーパーバイザーはService.Mobileに自動的に接続し、スケジュールされたメンテナンスと監査のリストと対応するアンケートテンプレートを要求するモバイルアプリケーションを開きます。

以下の図は、モバイルアプリケーションが機能する構造と、それに基づいてUIがメンテナンスまたは監査のために構築される構造を示しています。

セクション（セクション）-タイプ（プロパティobjectType）に応じてモバイルアプリケーションのUIに描画される要素。たとえば、名前付きグループや、ある種のコントロールにすることができます。セクションには、ネストされたセクションのリストへのリンクがあります。セクションのネストは制限されていません。

• サポートとアンケート

の記入モバイルアプリケーションのメイン画面には、従業員とその能力レベルに関する情報、およびメンテナンスまたは監査の一部として完了する必要がある一連のアンケートが表示されます。

  • フィールドトレーニングの準備-1日の初めに完了します。
  • アンケートテンプレート-各アウトレットに記入
  • 就業基準-就業日の終わりに完了します。



スーパーバイザーは店内の部下に付き添い、自分の仕事を観察し、アンケートに答えます。各プロファイルには、従業員の長所と短所の概要が表示されるため、上司は特定のスキルの開発に集中できます。また、現在のサポートの結果をスケールで表示して、監督者が部下に今ここで作業できるようにします。

伴奏の過程で、監督者は質問のグループにコメントを付け、写真レポートを作成します。これにより、勤務日の終わりに複雑なフィードバックを提供できます。各アンケートには、長所と短所のブロックが1日中記入され、従業員の能力開発をさらに計画するためのガイドラインになります。



• サーバーへのプロファイルの送信

1日の終わりに、スーパーバイザーはマーチャンダイザーに拡張フィードバックを提供し、メンテナンス結果をポータルに送信します。最初に、完成した申請書の基本データがなくなり、次に写真がなくなります。



• 結果の処理

アンケートはサーバーに送信され、Webポータルに表示されます。その構造はテンプレートに似ています。データは、汎用的な形式で保存され、他のコンポーネントから分離されています。これにより、報告書においてもアンケート構造の変化からの独立を獲得しました。

結果により、リーダーはチーム全体と各従業員など、パフォーマンスのさまざまなセクションを含むレポートを作成できます。

  • 1か月に作成された会議の数と種類、開催された会議の数に関する情報があります。
  • 部下の能力に関するレポートを取得できます（すべてがポイントで評価されるため、ダイナミクスを視覚化できます）。
  • チーム全体のダイナミクスを分析できます。また、必要に応じて、特定の伴奏または監査の詳細な結果、質問への回答、コメント、写真レポート、その他のメモを見て、変更の理由を理解してください。
  • ロシア全体の統計が、各地域および都市ごとにあります。

SQL Server Reporting Servicesを使用してレポートを作成します。実際、レポートでは、テーブルでもデザインスタイルの継続性を維持したかったため、別の記事ではニュアンスで十分だったため、すべてが簡単ではありませんでした。近い将来、使用したソリューションに関する詳細な記事を書くことを約束します。

結果


このソリューションにより、定性的に異なるレベルで従業員のトレーニングと認定を実施し、従業員の能力とスキルを分析することが可能になりました。 10か月余り稼働しており、その間にすでに3,700を超える保守と監査が実施されています。

iPadアプリケーションは、概要用紙プロファイルに代わるものです。スーパーバイザの充填速度と効率を大幅に向上させます。これで、観察、トレーニング、詳細なフィードバックを行うための時間が残ります。

ポータルパーツを使用すると、レポートをワンクリックで表示し、ダイナミクスを従業員別、チーム別、および各タイプのアクティビティ別にアップロードできます。レポートから、記入済みのアンケート、各従業員との各ミーティングの詳細を確認できます。変更の理由を理解します。写真レポートとコメントを表示します。そして、多くの要因と統合分析に基づいて、チームのさらなる発展を計画します。
