データの完全なガイド-* HTML属性

属性の使用に関する記事の翻訳をご紹介しますdata-*。これらは、さまざまな有用な情報の標準HTML要素に格納するのに便利な属性です。特に、この情報はJavaScriptおよびCSSで使用できます。



一般情報


HTML要素には、データの提供から支援技術、要素のスタイリングまで、さまざまな問題を解決するために使用される属性を含めることができます。

<!--  `class`      CSS,  `role`    -->
<div class="names" role="region" aria-label="Names"></div>

独自の属性を作成したり、既存の属性を設計されていない方法で適用したりすることはお勧めしません。

<!-- `highlight`   HTML--->
<div highlight="true"></div>

<!-- `large`      `width` -->
<div width="large">

これは多くの理由で悪いです。HTMLが正しく生成されません。そして、これは実際に悪い結果をもたらすことはないかもしれませんが、これは開発者が有効なHTMLコードを作成するという事実によって引き起こされる温かい感覚を奪います。ただし、HTMLは常に進化している言語であるため、これを行わないでください。その結果、特定の属性が現在言語で使用されていなくても、将来何も変更されないという意味ではありません。

確かに、誰かがそのような属性を使用する必要がある場合、彼はこれを行うための完全に通常の方法を持っています。これは、名前がprefixで始まる独自の属性を作成することですdata-。これらの属性は簡単に操作でき、プログラマーが必要とする方法で適用できます。

構文


カスタムHTML属性を作成してデータを書き込む機能は、非常に便利です。ご存知のように、これは属性が原因で可能ですdata-*。このような属性が意図されているのはこのためです。次のようになります。

<!--      -->
<div data-foo></div>

<!-- ...     -->
<div data-size="large"></div>

<!--      HTML,    HTML-, , ,      -->
<li data-prefix="Careful with HTML in here."><li>

<!--   -      -->
<aside data-some-long-attribute-name><aside>

これらの属性は、常にプレフィックスdata-で始まるため、属性data-*またはデータ属性と呼ばれることがよくあります。これらの属性の名前を形成する場合、開発者の必要に応じて、単語が最初dataに来て、次にダッシュ（-）、次に残りの名前が続きます。

dataという名前の属性を使用できますか？


以下は、という名前の属性を使用するサンプルコードですdata。

<div data=""></div>

この名前の属性はおそらく誰にも害を与えることはありませんが、それを使用すると、以下で説明するJavaScriptツールを使用できなくなります。この例では、実際、開発者は自分の特定の属性を作成しますが、これはすでに述べたように、お勧めできません。

data- *属性で行うべきでないこと


このような属性には、支援技術がアクセスできるコンテンツを含めないでください。一部のデータがページに表示される必要がある場合、または画面から読み取るための手段にアクセスできる必要がある場合、属性に入れるだけでは不十分ですdata-*。このようなデータは、通常のHTMLマークアップにも表示されます。

<!--      ,     -->
<div data-name="Chris Coyier"></div>

<!--      ,       ,    ... -->
<div>
  <span class="visually-hidden">Chris Coyier</span>
</div>

これは、 Webページ要素を非表示にする方法に関する資料です。

data- *属性を使用した要素のスタイル


CSSでは、属性とその値に基づいてHTML要素を選択できます。

/*      ,    */
[data-size="large"] {
  padding: 2rem;
  font-size: 125%;
}

/*    , ,  -  */
button[data-type="download"] { }
.card[data-pad="extra"] { }

これは面白そうに見えるかもしれません。HTML / CSSでのスタイル設定には、主にクラスが使用されます。また、クラスは素晴らしいツールです（具体的な平均レベルは異なりますがclassList、elementのプロパティを介して便利なJavaScriptメソッドを使用して操作できます）、要素には特定のクラスがある場合とない場合があります（つまり、要素内のクラスは「含まれている」 、または「オフ」）。属性を使用する場合data-*、クラスの開発者のオプション（「オン/オフ」）と、同じレベルの特異性を持つ属性の値に基づいて要素を選択する機能は、開発者が自由に使用できます。

/*  ,      */
[data-size] { }

/*  ,      */
[data-state="open"],
[aria-expanded="true"] { }

/*  " ",      ,    "3",    -   ,   3 -  "3.14" */
[data-version^="3"] { }
/*  ""   ,      -    */
[data-company*="google"] { }

属性セレクターの特異性


属性セレクターの特異性は、クラスの場合と同じです。多くの場合、特異性は4つの部分からなる値と見なされます。

  • インラインスタイル
  • ID
  • クラスと属性
  • タグ

その結果、属性セレクターのみを使用してスタイルを設定したアイテムの特異性の値を想像すると、のようになります0, 0, 1, 0。 

そしてここに別のセレクタがあります：

div.card[data-foo="bar"] { }

既にの意味で説明されています0, 0, 2, 1。2クラス（.card）と属性の両方があるため、ここに数値が表示されます([data-foo="bar"]。そして、1ここでは、タグ（div）が1つしかないためです。

ここでは、わかりやすくするために、これらの引数の図解版を示します。


タグ1つ、クラス1つ、属性1つ属性

セレクターの場合、特異性は識別子（ID）よりも低くなりますが、タグ（要素）よりは高くなります。それらの特異性は、クラスの特異性と同じです。

大文字と小文字を区別しない属性値


セレクターで小文字と大文字の異なる組み合わせを使用して記述された文字列が値に含まれる可能性がある属性を選択する場合は、大文字と小文字を区別しないセレクターオプションを使用できます。

/*     
<div data-state="open"></div>
<div data-state="Open"></div>
<div data-state="OPEN"></div>
<div data-state="oPeN"></div>
*/
[data-state="open" i] { }

この動作により、セレクタで文字が確実に使用されiます。

CSSを使用すると、属性値を取得data-*してページに表示できます。

JavaScriptでのdata- *属性の操作


属性値は、data-*メソッドを使用して、他の属性の値のように、アクセス可能なgetAtributeデータおよび方法読み出すためのsetAttribute書き込みのために。

//   
let value = el.getAttribute("data-state");

//   .

//  data-state   "collapsed"
el.setAttribute("data-state", "collapsed");

ただし、属性にdata-*は独自の特別なAPIがあります。複数の属性を持つ要素があるとしますdata-*（これは完全に正常です）。

<span 
  data-info="123" 
  data-index="2" 
  data-prefix="Dr. "
  data-emoji-icon=":-)"
></span>

この要素へのリンクがある場合、次のようにその属性の値を読み書きできます。

// 
span.dataset.info; // 123
span.dataset.index; // 2

// 
span.dataset.prefix = "Mr. ";
span.dataset.emojiIcon = ";-)";

JSコードの最後の行では、CamelCaseスタイルの属性名エントリを使用しています。システムは、ケバブスタイルで記述されたHTML属性の名前をラクダスタイルで記述された名前に自動的に変換します。つまり、それはにdata-this-little-piggy変わりdataThisLittlePiggyます。

このAPIは、当然のことながら、同じくらい簡単APIのようではありませんCLASSLISTのような明確な方法をサポートしている、add、remove、toggleそしてreplace、それは何もないよりはましです。

data-*JSONデータ を属性に書き込みませんか？結局のところ、これらは有効なJSONデータ（引用符など）としてフォーマットできる単なる文字列です。必要に応じて、このデータを属性から抽出して解析できます。

const el = document.querySelector("li");

let json = el.dataset.person;
let data = JSON.parse(json);

console.log(data.name); // Chris Coyier
console.log(data.job); // Web Person

ここでの考え方は、属性を使用して、data-*JavaScriptからアクセスして特定のアクションを実行できるHTMLコードにデータを配置することです。

このシナリオの一般的な実装は、データベースでの作業を整理することを目的としています。ボタンがあるとしますLike：

このボタンには、サーバーへのAJAX要求を実行するクリックハンドラーがある場合があります。ハンドラは、ユーザーがボタンを使用して何かを気に入った場合、サーバーデータベースのいいねの数を増やします。ハンドラーは、属性からこの情報を取得するため、更新する必要がある特定のレコードを認識していますdata-*。
