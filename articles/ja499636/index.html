クロスプラットフォーム


アプリケーションをAndroidとiOSの両方で動作させたいと考えました。同時に、プラットフォームに応じてさまざまな実装ができるようにしたいと考えました。 React Nativeでこれを行うには2つの方法があります。

最初の方法は、プラットフォームモジュールを使用することです。たとえば、アプリケーションを作成し、プラットフォームに応じて高さを設定します。高さは、iOSの場合は200、Androidの場合は100です。この方法を使用してPlatform.OS、ユーザーが使用しているプラ​​ットフォームを特定します。

import {Platform, StyleSheet} from 'react-native';

const styles = StyleSheet.create({
    height: Platform.OS === 'ios' & 200 : 100,
});

そのような属性が40ある場合、コードを複製する必要はありません。これにはPlatform.select同じモジュールからのメソッドがあります。これは、プラットフォームキーによって一連のプロパティを返します。iOS backgroundColor = 'red'では、Androidでは- が返されますbackgroundColor = 'blue'。結果として、iOSの属性セット（少なくとも20、少なくとも40ある可能性があります）、Androidを返すことができます-完全に異なるセットです。

import {Platform, StyleSheet} from 'react-native';

const styles = StyleSheet.create({
    container: {
        flex: 1,
        ...Platform.select({
            ios: {
                backgroundColor: 'red',
            },
            android: {
                backgroundColor: 'blue',
            },
        }),
    },
});

このメソッドをPlatform.select使用すると、クロスプラットフォームコンポーネントを作成できます。

const Component = Platform.select({
    ios: () => require('ComponentIOS'),
    android: () => require('ComponentAndroid'),
})();

<Component />;

React NativeはHTMLベースのWebViewテクノロジーを使用しないため、アニメーションはありません。ここでは、アニメーションコンポーネントが役立ちます。それはいくつかのラッパーの基本的な要素をサポートしていますView、Image、Text、ScrollView。それは別のスレッドに入り、アプリケーションを遅くしませんが、マイナスがあります-「非人道的な」種類のコードです。

以下は、ユーザープロジェクトのリストを含むアプリケーションの画面の例です。


上記Headerの名前は「プロジェクト」で、プロジェクトSearchBarをフィルタリングします。SearchBar下にあるように見えますHeaderが、実際には、プロジェクトのリストの下にある一般的なブロックの流れの中にあります。Animatedコンポーネントのおかげで、そのようなアニメーションはSearchBar「消える」ように実装されています。

実装の短縮バージョン。

export default class ProjectsScreen extends Component {

    state = {
        scrollY: new Animated.Value(
            Platform.OS === 'ios' ? -HEADER_MAX_HEIGHT : 0,
        ),
    };

    render() {
        const scrollY = Animated.add(
            this.state.scrollY,
            Platform.OS === 'ios' & HEADER_MAX_HEIGHT : 0,
        );
        const headerTranslate = scrollY.interpolate({
            inputRange: [0, HEADER_SCROLL_DISTANCE],
            outputRange: [0, -HEADER_SCROLL_DISTANCE],
            extrapolate: 'clamp',
        });

        return (
            <View>
                <Animated.View style={{ translform: [{ translatY: headerTranslate }] }}>
                    <SearchBar/>
                </Animated.View>
                <Animated.ScrollView
                    onScroll={Animated.event(
                        [{ nativeEvent: { contentOffset: { y: this.state.scrollY } } }],
                        { useNativeDriver: true },
                    )} contentInset={{
                        top: HEADER_MAX_HEIGHT,
                    }}
                    contentOffset={{
                        y: -HEADER_MAX_HEIGHT,
                    }}
                >
                    {this._renderScrollViewContent()}
                </Animated.ScrollView>
            </View>
        );
    }
}

見た目は簡単ですが、iOSですべてのコンテンツを上下に移動する必要があります。Androidでは、これは必要ありません。アニメーションモジュールには、補間やさまざまなメソッドもあります。

初回起動時の画面の「ちらつき」の問題


アプリケーションを作成し、アプリケーションアイコンとスプラッシュ画面（または起動画面）を追加することにしました。これは、メモリからアンロードされた場合に最初の起動時に表示されるアプリケーション画面です。画面はオレンジ色で、中央にSEMrushのロゴがあります。



React Nativeアーキテクチャーでは、ネイティブ部分はいくつかのJavaScriptブリッジ・バンドルと相互作用します。起動時にネイティブパーツが機能していて、スクリプトバンドルがまだ読み込まれていないという問題が発生しました。オレンジ色のスプラッシュスクリーンとプロジェクトリストの外観の間に、白い空白（ミリ秒の端数）が画面に輝いていました。

ちらつきの問題は、react-native-splash-screenライブラリを使用して解決されました。アプリケーションの起動画面をプログラムで表示または非表示にするJavaScript APIを提供します。このAPIを使用して、メソッド内のプロジェクトのリストを含むコンポーネントのネイティブスプラッシュスクリーンを削除しましたcomponentDidMount()。ボーナスとして、2つの画面の間にスピナーを配置して、見栄えを良くしています。

接続の喪失


フレンドリーで美しいインターフェースを開発し、UXとUIを管理したいと考えました。しかし、ユーザーが地下鉄に乗り、連絡を失い、プロジェクトのリストを更新すると、アプリケーションには空白の画面しか表示されませんでした。既に読み込まれている情報と、インターネット接続の問題に関するメッセージを表示することをお勧めします。Netinfo

ライブラリはオフライン作業を支援しました。FacebookがReact Nativeをアンロードして軽量化するため、現在は別のパッケージになっています。



Netinfoを使用すると、次のことができます。

  • イベントをサブスクライブし、インターネット接続を変更するときにユーザーにメッセージを表示します。
  • 接続の品質を確認してください。
  • 複雑なアプリケーションの接続タイプ（LTE、WiFi、3G）を確認します。
  • リクエストを使用してインターネット接続を確認してください。

アプリケーションのデバッグ


新しい依存関係、特にネイティブコードを使用する依存関係を追加した後にエラーが発生しないようにするには、デバッガーが必要です。 React Nativeではそれが-iPhoneシミュレーターの画像に示されています。


iOSでは、Xcodeのエミュレーターを使用しました。このエミュレーターでは、OSとデバイスのバージョンを選択できます。 Android用シミュレーター-Android Studio。

エミュレータのダイアログメニューには、リモートでのJSデバッグモードとライブリロードモードがあります。 CSSのようなスタイルやビューのネストなど、コンポーネントのプロパティを表示できます。

ネイティブ開発では、コードを記述してアセンブリに送信し、実行する必要があります。これは長い時間です。デバッガーでは、すべてがWeb上のReactJSと同じです。LiveReloadingをオンにすると、再起動せずにシミュレーターでリアルタイムの再構築を確認できます。

さらに、React Developer Toolsを使用して、コンポーネントを検査し、階層を表示し、それらのプロパティとスタイルを変更します。



しかし、これでも十分ではありません。より優れたソリューションであるReact Native Debuggerがあります。



React Inspector、Redux DevTools、インターネットインタラクション、コンソールロギング、メモリ、アプリケーションプロファイリング、すべてのアクションを表示します。最初の2つのソリューションとは異なり、React Native Debugger自体でスタイルを変更できます。

ネイティブモジュールの接続


これまでは、すべてが簡単かつ透過的でした。私たちは、ある種のソリューションを採用し、接続し、機能します。しかし、ニュアンスがあります-React Nativeのアーキテクチャの結果です。JavaScript部分に加えて、すべてのライブラリーはネイティブのものを使用します。これは、React Nativeに関連付ける必要があります。これを行うには、必要なすべての変更を自動的に行い、プロジェクトのネイティブコードにリンクするreact-native linkコマンドがあります。

ただし、自動接続が常に機能するとは限らないため、ネイティブファイルを手動で変更する必要がある場合があります。手動インストールがライブラリの説明に記載されていない場合は、それを使用する価値があるかどうかを検討し、可能であれば手動リンクを避けてください。
