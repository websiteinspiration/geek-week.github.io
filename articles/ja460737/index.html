背景を聞く：プロジェクト管理ポッドキャスト

世界のポッドキャスティングはますます多様化しています。放送のテーマはガジェットから科学まで多岐にわたりますが、ITギルドで毎日直面しているトピックに関するポッドキャストを取り上げることにしました。これらは、ITSM、ITIL、DevOps、およびITプロジェクト管理です。

CIOトークネットワークポッドキャスト


[ iTunes ] [ SoundCloud ]

ポッドキャストのストーリーは2003年に始まりました。その後、「テクノロジーの管理：正しい方法！」ラジオの生放送だった。彼のマスターは、現在も、そして今も残っているSandzhong Aul（Sanjog Aul ）、ITコンサルタント、起業家で、Diversity MBAマガジンバージョンのトップ100アンダー50リーダーのリストを作成しています。今日、ポッドキャストは、フォーチュン500やフォーチュン1000の大企業のリーダーへの一連のインタビュー

です。彼らが言うこと：特別な専門分野はありません。ビッグデータ、クラウドテクノロジー、モノのインターネットについても話します。ただし、IT管理のトピックはしばしば扱われます。たとえば、最新の問題の1つでは、ITSM方法論を使用して顧客サービスの品質を改善することでした。別の- 参加者技術サポートのための人工知能システムの展望について議論しました。ポッドキャストは、これらの分野でなじみのないトピックを「最初に見る」のに適していると考えられています。

それが出てくる頻度：新しいエピソードは約10日に1回表示されます。

所要時間： 40〜60分。


DevOpsラジオ


[ iTunes ]

ショーの主催者は、DevOpsエバンジェリストであるブライアンドーソンであり、マーケティング担当副社長であるCloudBeesのアンドレピノです。同社は継続的デリバリーツールに取り組んでいます。

彼らが話していること： DevOpsからDockerまで、ソフトウェアの開発と展開に関連するすべてに特化した一連のポッドキャスト。ゲストの中には、ボッシュ、フォレスター、エレクトロニックアーツ、その他多くの企業のスペシャリストがいます。ショーの存在全体を通して、スピーカーはIT業界の多くの象徴的な人物でした。

ジェンキンスの歴史について（こことここ）、作者の川口浩介氏に話を聞いた。Jenkins Xの開発者と、このKubernetesベースのシステムが登場した理由と、DevOpsエンジニアにとってどの機能が役立つかについて話し合いました。また、Capital Oneを保有する銀行の代表者に、「大規模」のDevOps変換についてインタビューを行いました。

それが出てくる頻度： 2週間ごとの新しいリリース。

所要時間： 30〜50分。


ピンクの象


[ iTunes ] [ Player.fm ]

ピンクエレファントのポッドキャスト。2人の副大統領が主催している。組織自体は1980年以来、IT管理の分野でトレーニングと高度なトレーニングに従事しています。

彼らが話していること：問題はDevOps、アジャイル、およびITSMのベストプラクティス全般に焦点を当てています。たとえば、そのうちの1人はかんばんの方法論と、ITサービス管理の分野でそれをどのように使用するかを検討しました。では、別の方法で、人工知能システムは、資産管理を支援することができるもの。トランスミッションのリリースは興味深い実用的なケースとユーモアで薄められています。

それが出てくる頻度： 2016年までは月に1回だったが、今では出てくる頻度ははるかに少ない。

所要時間： 30分。


ServiceNow TechBytes


[ iTunes ] [ SoundCloud ]

多機能ITSM自動化プラットフォームであるServiceNowのポッドキャスト。ガートナーは、彼女を彼の分野のリーダーと繰り返し呼びました。プログラムは2015年からリリースされています。

彼らが話していること： ServiceNowのエキスパートとエンジニアは、プラットフォームの機能、新しいリリースについて話し、実際的な事例を提供します。たとえば、問題の1 つでは、IBM Watsonスーパーコンピューターを使用したプラットフォームの動作について話しました。テクニカルサポートを組織するためのウォークアップアプローチに関するもう1つの問題は、ユーザーまたはクライアントが専門家のところに行き、自分の問題について個人的に話すことができる場合です。このアプローチの例は、Apple Genius Barsです。

それが出てくる頻度：月に2〜3回。

所要時間： 20〜30分。

ソフトウェア工学研究所ポッドキャスト


[ iTunes ]

発表者はカーネギーメロン大学のCERTです。

彼らが言うこと：主な方向はサイバーセキュリティです。したがって、DevOpsとITSMに関する議論のほとんどは、何らかの方法で、ITインフラストラクチャの保護に関連しています。で1つの問題、専門家が医療や金融などの、高度に規制産業でのIT管理プロセスを構築する方法について話しました。

インフラストラクチャを保護するための実用的な推奨事項があります。スペシャリストは、米国およびその他の国の警察が大企業の企業メールの侵害を含む事件で74人の逮捕を行った昨年の話を分析しています。過去の情報に加えて、問題でそのような攻撃から組織を保護する方法に関するヒントが提供されます。

それが出てくる頻度：月に2〜3回発行。

期間： 30〜40分。
