DockerコンテナーからのWebstorm NodeJsインタープリター

この記事では、DockerコンテナーにあるNodeJsインタープリターをプロジェクトとして（デフォルトで）使用してNodeJsアプリケーションをデプロイするプロセスについて説明します。最終的にインタープリターはDockerHubの公式ノードイメージから取得されるため、このソリューションはおそらくLinux OSのユーザーにのみ関連する可能性があります。


何のために？


アプリケーションはドッカーのみでホストマシンにデプロイできます。インタープリターを自分でインストールする必要はありません。これにより、開発者は特定のプロジェクトに特定のバージョンのインタープリターを使用できます。また、プロジェクトのイメージのバージョンを変更する場合、開発者は手動で設定する必要がなく、インタープリターの新しいインスタンスが使用されます。自動的に投影します。


決定


たとえば、アプリケーションはReactJs + Typesciptで作成されました。インタープリターが正しく構成されていない場合、少なくともコードの強調表示が壊れるためです。


1.完成したテンプレートからアプリケーションを作成します。


$ npx create-react-app reactjs-typescript-custom-nodejs --template typescript

2. Dockerfileを作成します。


FROM node:14
WORKDIR /app

CMD cp /usr/local/bin/node /app/docker/node \
    && npm install \
    && npm run start

CMD, Docker /app/docker/node, /app . .

docker build -t client . && docker run -it --rm --volume=${PWD}:/app -p 3000:3000 --name client client
