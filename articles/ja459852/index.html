銀行のモバイルアプリケーションで宝くじライブラリを使用する方法

こんにちは、ハブル！

かつて、プロダクトオーナーから、Android / iOSアプリケーションにアニメーションを実装するための効果的なプロセスを作成することを検討するように依頼されました。その時点で、ローン商品の個人データをアプリケーションに事前に入力するタスクを実行し、サーバーが応答するまでに少し時間がかかりました。その間、美しい読み込みアニメーションを表示したいと考えました。


タスクは明確でした：デザイナーは美しく描き、ソースコードを両方の
プラットフォームに直接提供し、仕上げ作業を一切行わず、古いデバイスで遅れが生じないようにしました（そうです、私たちはまだAndroid 4.1をサポートしています）。

アニメーションを導入するための私の選択肢は何でしたか：

  1. アニメーションのベクタードローアブルを使用したペン。利点は、レンダリング
    が別のストリーム（api 25以降）で機能することです。欠点は、そのようなアニメーションを作成する複雑さと、オブジェクトを使用した操作の数が少ないことです。単純なアニメーションの場合、これはすべてうまくいきますが、もう少し複雑で、地獄が始まります。はい、異なるプラットフォームでは開始しません。
  2. GIF-重量が大きく、サイズが固定されているため、通常のスケーリングができません。また、特別な操作は行いません。
  3. pngシーケンス（コメントなし）。

しばらくの間、androidとgifのネイティブのベクターアニメーションに向けて掘り下げたところ（まあ、私はまだこのオプションを考えていました）、素晴らしいLottieライブラリーを思い出し、それを同僚に見せました。

さまざまなデバイスでのいくつかのテストの後、特にその機能が印象的だったため、ライブラリを実装する必要があると判断しました。デザイナーは特に喜んでいましたが、今ではAdobe After Effectsでほとんどすべてのアニメーションを実行でき、数回のクリックでjsonファイルにエクスポートできました。嬉しかったですが、まず最初に。

Lottieは、クロスプラットフォームアニメーションへの需要の高まりに応えてAirbnbによって開発および実装されたため、すべてのプラットフォームで（ほぼ）同等に機能します。開発者自身も、目標はAfter Effectsの機能の最大数を実現することであると主張し、これに成功しました。Lottieアニメーションの埋め込みは、ImageViewに画像を挿入するのと同じくらい簡単です。

主要な3つのクラス：

  1. LottieAnimationViewはImageViewの後継であり、アニメーションを使用する最も簡単な方法です。アニメーションをxmlで記述したり、コードで記述したりできます。ほとんどのメソッドがサポートされています。
  2. LottieDrawable-Drawableの子孫は、前のクラスと同じ機能を持ち、アニメーションを任意のビューに適用できます。
  3. LottieCompositionとそのコンパニオンLottieCompositionFactoryを使用すると、さまざまなソースからアニメーションをプリロードして、LottieDrawableとLottieAnimationViewに適用できます。

リソースの読み込み


以下からのリソースのロードをサポート：

  1. res / raw
  2. src /アセット
  3. JSONストリング
  4. jsonまたはzipファイルにつながるネットワークからの任意のURL（外部依存関係を追加しないようにHttpURLConnectionを介して実装されます。画像付きのアニメーションがある場合は、zipを使用する必要があります）
  5. InputStream jsonまたはzipファイル

アニメーションのキャッシュ


クールなのは、res / rawまたはアセットを介してダウンロードされたすべてのアニメーションがLRUキャッシュによって保存されることです。これにより、複雑なアニメーションの場合は時間がかかる場合があるため、ユーザーがアニメーションを再読み込みして解析する時間を無駄にすることがなくなります。さらにすごいのは、アニメーションをプリロードして、次のフラグメントで即座にアニメーションを表示する必要がある場合、コードを使用できることです。

LottieCompositionFactory.fromRawRes(context, rawFile)

アニメーションはrawFileキーを使用してキャッシュされ、実際に使用する必要がある場合は、ほぼ瞬時に開始されます。

進捗管理


Lottieでは、setProgress（...）を使用して現在のアニメーションの状態を設定できます。これは、ファイルのアップロードステータス、スクロール位置、さまざまなジェスチャーなどをアニメーション化する場合に便利です。BottomSheets、PullToRefresh、CollapsingToolbarLayoutのさまざまな実装を見ました。

AppBarLayoutで進行状況を使用する方法は次のとおりです。


appBarLayout.addOnOffsetChangedListener(AppBarLayout.OnOffsetChangedListener
{ appBarLayout, verticalOffset ->
  val percent = Math.abs(verticalOffset).toFloat()/appBarLayout.totalScrollRange
  animationView.progress = percent
})

ループ


Lottieは、ループ全体のsetRepeatMode（）またはsetRepeatCount（）をサポートします。アニメーション全体だけでなく、（0.0 ... 1.0）内のフラグメントもサポートします。これは、プロパティsetMinFrame、setMaxFrame、setMinAndMaxFrameによって実装されます。これを使用して、ファイルアップロードのさまざまな状態（アイドル、進行中、完了）の3つのアニメーションを実装しません。これを解決する小さなコードがあります：


when (loadingStatus) {
  LoadingStatus.IDLE -> {
    animationView.setMaxProgress(0.1f)
  }
  LoadingStatus.PROGRESS -> {
    animationView.setMinAndMaxProgress(0.2f, 0.9f)
    animationView.repeatCount = LottieDrawable.INFINITE
    animationView.playAnimation()
  }
  LoadingStatus.COMPLETE -> {
    animationView.setMinAndMaxProgress(0.9f, 1f)
    animationView.repeatCount = 1
    animationView.playAnimation()
}}


画像


私たちにとってロッティの主な利点の1つは、ライブラリがアニメーションへの画像の直接挿入をサポートしていることです。さらに、インターネットからダウンロードした静止画像と動的画像の両方を挿入できます。次に、それがどのように機能するかを説明します。

静的な画像の場合、すべてが簡単です。デザイナーはjsonと画像自体を含むアーカイブをアンロードします。

このimg_0.png、これはsrc /アセットに配置する必要がある画像で、アニメーション内にあります。

動的ロードの場合は、ビットマップを渡す必要があるsetImageAssetDelegateメソッドが使用されます。画像をGlideでプリロードするので、アニメーションと画像でフラグメントを開く段階で、すべてがキャッシュから取り出されるため、すべてがかなり高速です。これがコードです：


glideLoader.loadAsBitmap(imageUrl).into(object: CustomTarget<Bitmap>() {
  override fun onResourceReady(resource: Bitmap, transition: Transition<in Bitmap>?) {
     viewAnimation.setImageAssetDelegate(object: ImageAssetDelegate {
       override fun fetchBitmap(asset: LottieImageAsset?): Bitmap {
         asset?.let {
           val resizeBitmap =Bitmap.createScaledBitmap(resource, it.width, it.height, true);
             return resizeBitmap
           } ?: run {
             return resource
           }
         }
       })
    setAnimation(viewAnimation, animationImage)
  }
  override fun onLoadCleared(placeholder: Drawable?) {}
})


パフォーマンス


もちろん、ユーザーが多くの時間を費やしている画面では、プロセッサに負荷がかかるため、アニメーションを大量に使用しない方がよいでしょう。テストによると、一部のアニメーションのプロセッサ負荷は20％に達しています。したがって、このようなアニメーションの理想的なケースは、一度起動するインタラクティブな要素です。

一部のデバイスのアニメーションが遅くなる場合、時にはそれが役立ちます

viewAnimation.useHardwareAcceleration(true)

ただし、開発者はこの方法を慎重に使用することをお勧めします。電話ごとにハードウェアアクセラレーションが異なるため、アクセラレーションの代わりに、逆の効果が得られる可能性があります。

結論


一般に、Lottieライブラリを使用すると、アプリケーションでのアニメーションの実装が大幅に簡略化されます。

私たちが強調した宝くじの主な利点：

  1. 小さなライブラリサイズ（300 kb）
  2. クロスプラットフォームソリューションiOS / Android / Web
  3. Webからアニメーションをダウンロードする
  4. 進捗管理とどこでもループ
  5. アフターエフェクトからの多数の機能により、設計者は意図したエフェクトを実現できます。

マイナス：

  1. レンダリングはメインスレッドで行われ、fpsはアプリケーションで大幅に低下する可能性があります。
  2. くじ引きアニメーションの解析は、複雑なアニメーションではかなりの時間がかかる場合があります。


ちなみに、リソースを大量に消費するアニメーションをチェックするには、Google Playの公式Lottieアプリを使用できます。フレームをレンダリングする時間を確認できるレンダーグラフがあり、フレームにカットした場合のアニメーションの外観や、hardwareAccelerationなどがどのように影響するかを確認できます。
