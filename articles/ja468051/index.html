100ルーブルのWindowsサーバーがライセンスされたVDS：神話か現実か？

低コストのVPSは、GNU / Linuxで実行される仮想マシンとして最もよく理解されています。今日は、火星の窓に生命があるかどうかを確認します。国内外のプロバイダーの予算案がテストリストに含まれています。



商用のWindowsオペレーティングシステム上の仮想サーバーは、ライセンス料が必要であり、コンピューターの処理能力要件がわずかに高いため、通常はLinuxマシンよりも高価です。負荷の少ないプロジェクトの場合、安価なWindowsソリューションが必要でした。開発者はアプリケーションをテストするためのインフラストラクチャを作成する必要があり、これらの目的のために強力な仮想サーバーまたは専用サーバーを用意するのはかなりコストがかかります。平均して、最小構成のVPSのコストは1か月あたり約500ルーブル以上ですが、市場では200ルーブル未満のオプションが見つかりました。そのような安価なサーバーからパフォーマンスの奇跡を期待することは困難ですが、それらの機能をテストすることは興味深いことでした。結局のところ、テストの候補を見つけるのはそれほど簡単ではありません。

オプションを検索


一見すると、Windowsを搭載した超低予算の仮想サーバーで十分ですが、問題が発生したらすぐに、実際に注文してみる価値があります。私たちは約24の提案を検討し、そのうちの5つだけを選択することができました。残りはそれほど予算的ではないことが判明しました。最も一般的なオプションは、プロバイダーがWindowsとの互換性を主張する場合ですが、料金プランにOSのライセンスを借りるコストは含まれず、サーバーに試用版をインストールするだけです。この事実がサイトに記載されているのは良いことです。多くの場合、ホスティング業者はそれに集中していません。ライセンスは、自分で購入するか、月に数百から数千ルーブルというかなり印象的な価格で借りることができます。ホスティングサポートの典型的なダイアログは次のようになります。



同様のアプローチは理解できますが、ライセンスを個別に取得してWindows Serverの試用版をアクティブ化する必要があるため、意味がありません。VPS自体の価格を超えるソフトウェアをレンタルする価格も魅力的ではありません。特に21世紀には、アカウントで数回クリックするだけで、高価な追加サービスなしでオペレーティングシステムの合法的なコピーを備えた既製のサーバーを入手することに慣れているためです。その結果、ほとんどすべてのホスティング事業者が辞任し、Windowsに正直な超予算VPSを使用する企業がZomro、Ultravds、Bigd.host、Ruvds、およびInoventicaサービスという「競争」に参加しました。その中には、ロシア語を話す技術サポートを持つ国内と外国の両方があります。この制限は私たちにとって非常に自然なことです。ロシアのクライアントでのサポートが重要でない場合、彼は業界の巨人を含む多くのオプションを持っています。

構成と価格


テストのために、いくつかのプロバイダーからWindowsの最も安価なVPSオプションを選び、価格を考慮してそれらの構成を比較しようとしました。注目に値するのは、最高のCPUではなく、1 GBまたは512 MBのRAM、10、20、または30 GBのハードディスク（HDD / SSD）を備えたユニプロセッサ仮想マシンが、超予算の仮想マシンのカテゴリに分類されたことです。毎月の支払いには、プレインストールされたWindows Server、通常はバージョン2003、2008、または2012も含まれます。これは、おそらくシステム要件とMicrosoftライセンスポリシーによるものです。ただし、一部のホスティング業者は古いシステムを提供しています。

リーダーは価格ですぐに特定されました：Ultravdsは最も安価なWindows用VPSを提供します。毎月の支払いでは、ユーザーにVAT付きで120ルーブルの費用がかかり、1年間の支払いは1152ルーブル（1か月あたり96ルーブル）になります。それは何よりも安くなりますが、ホスティング業者はメモリをあまり割り当てません-512 MBだけで、ゲストマシンはWindows Server 2003またはWindows Server Core 2019を実行します。最後のオプションは最も興味深いです：象徴的なお金のために、最新バージョンの仮想サーバーを取得できますOS、ただしグラフィカル環境はありませんが、以下で詳しく説明します。RuvdsおよびInoventicaサービスの提供も同様に興味深いものでした。これらのサービスは約3倍高価ですが、最新バージョンのWindows Serverを搭載した仮想マシンを入手できます。

プロバイダーのサイトで仮想サーバーを注文することに特別な問題はありませんでした-それらのすべては非常に便利で人間工学的に作られています。承認のために、ZomroはGoogleからキャプチャを入力する必要があり、少し激怒します。また、Zomroには電話によるテクニカルサポートはありません（24 * 7モードのチケットシステムを通じてのみ提供されます）。また、Ultravdsの非常にシンプルで直感的な個人アカウント、Bigd.hostのアニメーションを備えた美しいモダンなインターフェース（モバイルデバイスで使用するのは非常に便利です）、およびクライアントVDSファイアウォールの外部にRuvdsを構成する機能にも注目したいと思います。さらに、各プロバイダーには、特に理解していなかった独自の追加サービスセット（バックアップ、ストレージ、DDoS保護など）があります。全体として、印象はポジティブです：以前は、より多くのサービスを提供している業界大手とのみ連携していましたが、その管理システムははるかに複雑です。

テスト


参加者の数が十分に多く、構成が弱いため、高価な負荷テストを実行しても意味がありません。一般的な模擬テストとネットワーク機能の表面的なテストに限定することをお勧めします。VPSの大まかな比較については、これで十分です。

インターフェースの応答性


最小構成の仮想マシンから、プログラムの即時ダウンロードとグラフィカルインターフェイスの迅速な応答を待つのは困難です。ただし、サーバーの場合、インターフェイスの応答性は最も重要なパラメーターとはほど遠いため、サービスの低コストを考慮に入れると、遅延を我慢する必要があります。これらは、512 MBのRAMを備えた構成で特に顕著です。また、ギガバイトのRAMを搭載したシングルプロセッサマシンでWindows Server 2012より古いバージョンのオペレーティングシステムを使用することは意味がないことも判明しました。これは非常にゆっくりと悲しく動作しますが、これは私たちの主観的な意見です。

一般的な背景に対して、UltravdsのWindows Server Core 2019のオプションは（主に価格で）際立って優れています。本格的なグラフィカルデスクトップがないため、コンピューティングリソースの要件が大幅に削減されます。RDPまたはWinRMを介してサーバーにアクセスでき、コマンドラインモードを使用すると、グラフィカルインターフェイスでプログラムを起動するなど、必要なアクションを実行できます。すべての管理者がコンソールの操作に慣れているわけではありませんが、これは良い妥協策です。顧客は古いバージョンのOSを弱いハードウェアで使用する必要がないため、ソフトウェアの互換性の問題を解決できます。 



デスクトップは禁欲的に見えますが、Server Coreアプリ互換性機能オンデマンド（FOD）コンポーネントをインストールすることで少し仕上げることができます。すでに使用されているシステムに加えてかなりの量のRAM（使用可能な512のうち約200 MB）がすぐに失われるため、これを行わない方がよいでしょう。その後は、一部の軽量プログラムのみをサーバーで実行することができますが、本格的なデスクトップに変える必要はありません。結局のところ、Windows Server Core構成は、アドミンセンターを介したリモート管理用に設計されているため、作業マシンへのRDPアクセスを無効にする価値があります。

キーボードショートカット「CTRL + SHIFT + ESC」を使用してタスクマネージャーを呼び出し、そこからPowershellを起動することをお勧めします（インストールキットには古き良きcmdも含まれていますが、オプションは少なくなっています）。次に、コマンドのペアを使用して、必要なディストリビューションがアップロードされる共通のネットワークリソースが作成されます。

New-Item -Path 'C:\ShareFiles\' -ItemType Directory
New-SmbShare -Path 'C:\ShareFiles\' -FullAccess Administrator -Name ShareFiles

サーバーソフトウェアをインストールして起動するときに、オペレーティングシステムの切り捨てられた構成に関連する問題が発生することがあります。原則として、これらは克服可能であり、Windows Server 2019が512 MBのRAMを備えた仮想マシンで適切に動作する場合、おそらくこれが唯一のオプションです。

模擬テストGeekBench 4


今日、Windowsを実行しているコンピューターのコンピューティング機能をチェックするための最高のユーティリティの1つです。合計で、彼女は4つ以上のテストを実施し、暗号化、整数、浮動小数点、メモリの4つのカテゴリに分類されます。このプログラムは、さまざまな圧縮アルゴリズムを使用し、JPEGとSQLiteでの作業、およびHTML解析をチェックします。先日、GeekBenchの5番目のバージョンが利用可能になりましたが、多くの人がアルゴリズムの重大な変更を気に入らなかったため、実績のある4つを使用することにしました。 GeekBenchは、Microsoftオペレーティングシステムの最も包括的な模擬テストと呼ぶことができますが、ディスクサブシステムには影響しません。個別に確認する必要がありました。明確にするために、すべての結果は一般的な図に要約されています。



Windows Server 2012R2がすべてのマシンにインストールされ（Ultravds UltraLiteを除く-Windows Server Core 2019にはServer Coreアプリ互換性機能オンデマンドコンポーネントがあります）、結果は予想に近く、プロバイダーによって宣言された構成に対応しています。もちろん、模擬テストは指標ではありません。実際のワークロードでは、サーバーはまったく異なる動作をすることができ、さらに、クライアントゲストシステムが置かれる物理ホストのワークロードに大きく依存します。ここでは、Geekbenchが提供する基本周波数と最大周波数の値を確認する価値があります：
