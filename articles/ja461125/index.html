Visual Studioのソースコードでメッセージに番号を付けるための連続した数値コードを作成するタスク（例：C＃）

みなさん、こんにちは


Visual Studioでソースコードを直接編集するプロセスで、ソースコード内のテキストメッセージ用の連続した数値コードを作成する、私が約2年間使用してきた小さな人生のハックを共有しましょう。



ユーザー/エラー/例外ハンドラーへの数十のメッセージがコード内で「形成」され始めたときにこのタスクについて考え始め、実行時に次のメッセージを受信すると、それがどこから来たかをナビゲートすることが困難になったという事実に我慢できなくなった。各メッセージの冒頭に表示される順次インデックスを使用してすべてのメッセージに番号を付けたいという強い要望がありましたが、このシーケンスは、オンザフライで（オンデマンドで）キーボードを使用してのみソースコードを入力するときに作成でき、エンコードプロセスの邪魔にならないようにしました。 これは次のように行われます。





  1. エラーコードの列挙を作成します。
  2. 列挙型のコードの特別な構文： "_number"。（enumはまだシンボル名を必要とするため、番号の先頭にアンダースコアがあります）。
  3. 「_number」形式の「デジタイザ」を実際の数値に。
  4. その場で数値コードをインクリメントする魔法。

1.2：


/// " "   
public enum MCodes{
        _000,
        _001,
        _002,
}

3：実際の数値の「_number」形式の「デジタイザ」


static class _MCodeExtensions{
        /// mini formar error message -    .
        public static string mfem(this MCodes mcode) {
            //string str = $"{nameof(rcode)} = {rcode}, {nameof(mcode)} = {mcode}";
            int val = Int32.Parse(mcode.ToString().Substring(1));
            string str = $"{nameof(mcode)} = {val}";
            return str;
        }
}

4.マジック


マジックは、IntelliSense for Visual Studioの機能を使用する ことです。実際、これらのアクションは非常に迅速に実行されます（スローモーション）。









使用する


「通常」下線付きの番号がソースコードで使用されることはほとんどないため、Ctrl-F（現在のファイルで検索）またはCtrl-Shift-F（プロジェクト全体で検索）を使用してこの番号を見つけると、エラーの場所が正確に示されます。

（もちろん、列挙型を開いてコードを見つけ、Shift-F12を押すこともできますが、これは放電からの正しい道です...）

欠点


1.エラーコードが挿入されたコードをコピーすると、当然、エラーコードは一意でなくなります。それらに対抗するには、いくつかのコードが1度しか使用されていないことを確認して、列挙型MCodeを定期的に確認する必要があります 。 2.ダイヤルされた形式を間違え、 "_ number"ではなく、数値に変換できない何かを書き込むことができます。はい、例外があります。







結論


特にエラーに関するメッセージに番号を付けると、さまざまなメッセージが表示される理由をデバッグしたり、その理由を見つけたりするのに非常に役立つ場合があります。この方法がこのタスクを簡略化するのに役立つことを願っています。

これは最低限の体動だと思われますか？

PS


これはエラーコードに対する私の主観的な態度ですが、突然何かを助けるでしょう。エラーコードのインクリメントだけでなく、他のシーケンスにも使用できます。当然、好みに合わせてソリューションをカスタマイズできます。

enumの数値形式を "_number"として使用する理由は、intは実際にはenumの後ろに隠されており、enumメンバーにはシーケンスの先頭から番号が付けられているためです（構文的に割り当てることができますが、これはIntelliSenseによって自動的に行われるのではなく、時間を費やす必要はありません）したい）。また、このメンバーの意味は場所によって異なります。そして、場所が変わった場合、その数はすでに異なります。したがって、値自体はコードのすべての場所で無視されます。Enum Enums
