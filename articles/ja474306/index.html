ポインティング、フォーカス、アクティブ状態のスタイルを変える。


長年にわたり、私は要素の状態を設計し:hover、:focusそして中に:active同じように。いつ始めたのか正確には覚えていません。しかし、これは最善のアプローチとはほど遠いものです。なぜ、この記事で説明しようと思います。


これは私がいつも使っているコードの例です。

.selector {
  &:hover, 
  &:focus,
  &:active {
    ...
  }
}

キーボードを操作するときにインターフェイスの可用性（特にフォーカスの状態）にもっと注意を向け始めたとき、要素の異なる状態を同じようにスタイル設定するべきではないという結論に達しました。

ガイダンス、フォーカス、アクティブ状態は、異なるスタイルにする必要があります。

理由は簡単です。これらは異なる条件です！

今日私はあなたに多くの努力なしに3つの州すべてを配置する魔法の方法を示したいと思います。

から始めましょう:hover。

ホバースタイリング（：ホバー）


:hoverユーザーがマウスカーソルで要素の上に移動するとトリガーされます。

通常、この状態は背景色background-colorやテキストの変化ですcolor。ユーザーはすでに何らかの要素の上にカーソルを置いていることを知っているため、違いは明白である必要はありません。

button {
  background-color: #dedede;
}
 
button:hover {
  background-color: #aaa;
}



スタイリングフォーカス（：focus）


:focus要素がフォーカスを受け取ったときにトリガーされます。これは、次の2つの方法で実現されます。

  1. タブボタンでアイテムを選択するとき
  2. マウスでアイテムをクリックすると

焦点の要素は次のとおりです。

  1. リンク（<a>）
  2. ボタン（<button>）
  3. フォーム要素（<input>、<textarea>など）
  4. の属性 tabindex

留意すべきいくつかの重要なポイント：

  1. ユーザーは、タブボタンを使用して属性を持つ要素を選択することはできませんtabindex="-1"が、マウスで要素をクリックすることはできます。クリックすると、フォーカスの状態が発生します。
  2. SafariおよびFirefox Mac OSブラウザでは、クリックで要素がフォーカスされない <button>
  3. <a>リンクをクリックすると、マウスボタンが押されている間、フォーカスがリンクに留まります。ボタンを離すと、属性hrefが同じページ上の既存の場所を指定している場合、フォーカスが別の場所にリダイレクトされます。id

フォーカス状態をスタイル設定することにより、マウスを使用するユーザーよりも、キーボードインターフェイスを使用するユーザーの方に注意を向けます。

ユーザーがTabキーを押すと、フォーカスが移動する要素はわかりませんが、推測することしかできません。そのため、フォーカスされた要素にユーザーの注意を引くために、状態の顕著な変化が必要です。

ほとんどの場合、デフォルトのフォーカス設計で問題ありません。あなたがしたい場合は、独自の方法でそれをスタイル、これら四つの点に注意してください。

  1. ストロークを追加する（概要）
  2. アニメーションを作成する
  3. 変化する background-color
  4. 変化する color

特性の変化以来background-colorとcolorするとき頻繁に行われ:hover、それは状態に理にかなって:focus脳卒中やアニメーションを経由して登録します。

あなたは、プロパティの組み合わせを使用することができoutline、borderかつbox-shadow興味深いフォーカススタイルを作成します。これを行う方法については、「カスタムフォーカススタイルの作成」の記事で説明しました。

button {
  background-color: #dedede;
}
 
button:hover {
  background-color: #aaa;
}
 
button:focus {
  outline: none;
  box-shadow: 0 0 0 3px lightskyblue;
}



アクティブ状態のスタイル設定（：アクティブ）


実生活で何かと相互作用するとき、あなたは一種の反応を期待します。たとえば、ボタンを押すと、ボタンが押されることが期待されます。

Webサイトでは、この応答も役立ちます。で「ボタンを押す」瞬間をスタイリングできます:active。この状態は、要素を操作するときに呼び出されます。この場合、相互作用とは次のことを意味します。

  1. 要素上でマウスの左ボタンを押したまま（フォーカスが外れている場合でも）
  2. スペースバーを押したままにします（ボタン上）

button:active {
  background-color: #333;
  border-color: #333;
  color: #eee;
}



注意すべき2つのポイント：

  1. スペースを:active押し続けるとボタンの状態（<button>）が発生しますが、Enterを押し続けると
  2. Enterはリンクを起動しますが、アクティブな状態にはなりません。スペースバーがリンクをまったく起動しない

デフォルトのリンクスタイル


リンクには、デフォルトでアクティブな状態スタイルがあります。押すと赤くなります



関係：アクティブとフォーカス


フォーカスされた要素でマウスの左ボタンを押すと、そのアクティブな状態が呼び出されます。しかし同時に、フォーカス状態も呼び出されます。

マウスの左ボタンを離しても、フォーカスは要素に残ります。

これは、リンクとボタンを除くほとんどのフォーカス可能なアイテムに適用されます。

リンクについて：

  1. FirefoxとChromeでマウスの左ボタンを押し続けると、状態:activeとが発生します:focus。Safariのみのステータス:active（Mac OSでのみ確認済み）
  2. マウスボタンを離しても、ボタン:focusはリンクに残ります（同じページで属性hrefがリンクしidていない場合）。Safariでは、フォーカスが<body>

ボタンの場合：

  1. マウスの左ボタンを押したとき：両方の状態:activeと:focusChromeでのみ呼び出されます。:focusSafariおよびFirefox（Mac）ではState はまったく呼び出されません。私はこの奇妙な行動についてここに書きました。

クリックでボタンにフォーカスを当てたい場合は、このJavaScriptをできるだけ早く追加する必要があります（必要なものについては、上記のリンクを参照してください）。

document.addEventListener('click', event => {
  if (event.target.matches('button')) {
    event.target.focus()
  }
})

このコードを追加すると、ボタンの動作が次のように変わります。

  1. マウスボタンを押したままにする:activeと、すべてのブラウザで呼び出され:focus、Chrome でのみ呼び出されます
  2. マウスボタンを離すと、:focusSafariおよびFirefox（Mac OS）で呼び出されます。:focusすべてのブラウザのボタンに残ります

魔法の組み合わせ


魔法の組み合わせにより、ユーザーは、要素を狙ったり、焦点を合わせたり、操作したりしたときに応答を得ることができます。必要なコードは次のとおりです。

.element:hover,
.element:active {
  /*   /  */
}
 
.element:focus {
  /*   */
}

マウスユーザーの場合：

  1. ユーザーが要素にカーソルを合わせるbackground-colorと、変化します（および/またはcolor）。応答があります。
  2. ユーザーが要素をクリックすると、フォーカスのストロークの輪郭が表示されます。応答があります。



キーボードユーザーの場合：

  1. ユーザーがタブボタンで項目を選択すると、フォーカスストロークが表示されます。応答があります。
  2. 要素と相互作用すると、要素が変化しますbackground-color（および/またはcolor）。応答があります。



両方の長所を兼ね備えています。

  1. 私は魔法の組み合わせを注意深くテストしていませんが、これはこの概念の概念を支持する議論にすぎません。チェックして問題の可能性についてお知らせいただければ幸いです。
  2. チェックする場合は、Codepenを使用しないでください。Codepenのリンクのフォーカス状態は非常に奇妙です。リンクにカーソルを合わせると、フォーカスストロークが削除されます。どうして？知りません。追加のツールを使用せずにそのようなものをチェックするのが最善であるように思えることがあります。古き良きHTML、CSS、JS。

不思議な（しかしもっと良い）組み合わせではない


上で述べたように、Mac OSのSafariとFirefoxではボタンのクリックが奇妙な動作をします。上記で提案したJavaScriptコードのスニペットを追加した場合でも、魔法の組み合わせは機能します。しかし、完璧ではありません。

Mac OSのSafariとFirefoxで何が起こるかを次に示します。

  1. ユーザーがマウスボタンを押したままにしても、何も変化しません
  2. ユーザーがボタンを離すと、アイテムにフォーカスが移ります。



これで十分だと思えば、魔法の組み合わせが機能します。あなたはそこで止めることができます。
