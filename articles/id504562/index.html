Proxmox VE incremental backup dengan VBR


Dalam salah satu artikel sebelumnya dalam seri hypervisor Proxmox VE, kita sudah bicara tentang cara membuat cadangan menggunakan alat biasa. Hari ini kami menunjukkan cara menggunakan alat Veeam® Backup & Replication ™ 10 yang sangat baik untuk tujuan yang sama. “ Cadangan

memiliki esensi kuantum eksplisit. Sampai Anda mencoba memulihkan dari cadangan, ia berada di posisi superposisi. Dia sukses dan tidak. ” (ditemukan di Internet)

Penafian:

Tidak, saya tidak mengatakan bahwa ini adalah cara sempurna untuk membuat cadangan. Tidak, ini tidak dapat direkomendasikan untuk produksi. Tidak, saya tidak menjamin integritas sempurna dari cadangan yang dibuat.

Namun, semua ini berfungsi dan sangat cocok untuk banyak pengguna dan administrator sistem pemula yang mengambil langkah pertama dalam studi tentang virtualisasi dan sistem cadangan.

Cadangan mungkin merupakan salah satu proses terpenting di mana pekerjaan perusahaan mana pun bergantung. Tidak ada yang lebih mahal daripada data yang disimpan dalam sistem informasi perusahaan, dan tidak ada yang lebih buruk daripada kurangnya kemampuan untuk memulihkannya jika terjadi kegagalan.

Sering terjadi bahwa Anda perlu memikirkan kebutuhan untuk cadangan dan memilih alat hanya setelah keadaan darurat yang telah terjadi, terkait dengan hilangnya data penting. Seiring perkembangan teknologi virtualisasi, aplikasi cadangan mulai fokus pada interaksi yang erat dengan hypervisors. Tidak terkecuali Veeam® Backup & Replication ™, dengan opsi cadangan yang luas di lingkungan tervirtualisasi. Hari ini kami akan menunjukkan kepada Anda cara mengkonfigurasinya agar berfungsi dengan Proxmox VE.

Pengaturan Hypervisor


Kami akan menggunakan versi Proxmox saat ini pada saat penulisan - 6.2-1. Versi ini dirilis pada 12 Mei 2020 dan berisi banyak perubahan bermanfaat, yang akan kita bahas di salah satu artikel berikut. Untuk saat ini, mari mulai menyiapkan hypervisor. Tugas utama adalah menginstal Veeam® Agent untuk Linux pada host yang berlebihan dengan Proxmox. Namun sebelum itu kami mengambil beberapa langkah.

Persiapan sistem


Kami memasang utilitas sudo , yang tidak ada dalam sistem jika Proxmox tidak diinstal pada sistem Linux yang ada, tetapi sebagai OS mandiri dari gambar resmi . Kita juga membutuhkan header pve dari kernel. Kami pergi ke server melalui SSH dan menambahkan repositori yang berfungsi tanpa berlangganan dukungan (secara resmi tidak disarankan untuk produksi, tetapi berisi paket yang kami butuhkan):

echo "deb http://download.proxmox.com/debian/pve buster pve-no-subscription" >> /etc/apt/sources.list

apt update

apt install sudo pve-headers

Setelah prosedur ini, pastikan untuk me-restart server.

Instal Agen Veeam®


Unduh paket deb Veeam® Agent untuk Linux dari situs web resmi (akun diperlukan), bekali diri Anda dengan klien SFTP dan unggah paket deb yang diterima ke server. Instal paket dan perbarui daftar program di repositori yang ditambahkan paket ini:

dpkg -i veeam-release-deb_1.x.x_amd64.deb

Memperbarui repositori lagi:

apt update

Instal agen itu sendiri:

apt install veeam

Kami memeriksa bahwa semuanya telah diinstal dengan benar:

dkms status

Jawabannya akan seperti ini:

veeamsnap, 4.0.0.1961, 5.4.41-1-pve, x86_64: installed

Menambahkan repositori

Sebagai contoh, mari kita ambil tes penyimpanan SMB, saya punya ini adalah QNAP biasa:

Isi nama pengguna dan kata sandi untuk mengakses penyimpanan SMB, lalu klik OK dan kembali ke jendela sebelumnya, Berikutnya :


Jika semuanya dilakukan tanpa kesalahan, program akan terhubung ke repositori, meminta informasi tentang ruang disk yang tersedia, dan menampilkan kotak dialog berikut. Di dalamnya, atur parameter tambahan (jika perlu) dan klik Berikutnya :


Di jendela berikutnya, Anda dapat meninggalkan semua pengaturan default dan juga mengklik Berikutnya :


Kami memeriksa bahwa komponen yang diperlukan sudah diinstal dan dalam status sudah ada , dan klik tombol Terapkan :

Program akan secara otomatis menawarkan untuk menyimpan file konfigurasi Anda di repositori baru. Kami tidak membutuhkan ini, jadi kami menjawab TIDAK :

Menciptakan pekerjaan cadangan


Di jendela utama Veeam® Backup & Replication ™, klik Backup Job - Linux computer . Pilih jenis Server dan Dikelola oleh mode server cadangan :


Kami memberi nama pada tugas dan secara opsional menambahkan deskripsi. Kemudian klik Berikutnya :


Selanjutnya, kita perlu membuat semua server dengan Proxmox yang akan kita buat cadangannya. Untuk melakukan ini, klik Tambah - Komputer individual . Masukkan nama host atau alamat IP server dan detail akses. Dengan demikian, kami membentuk daftar komputer yang dilindungi dan klik Berikutnya :


Sekarang poin yang sangat penting, yaitu pilihan data yang akan ditambahkan ke cadangan. Semuanya akan tergantung di mana tepatnya mesin virtual Anda berada. Jika Anda hanya ingin menambahkan volume logis apa pun, Anda memerlukan mode cadangan level Volume dan memilih jalur ke volume atau perangkat logis, misalnya / dev / pve . Semua tindakan lain identik.

Untuk artikel ini, kami akan menunjukkan cara mode cadangan tingkat File berfungsi :


Di jendela berikutnya, kami membuat daftar direktori untuk cadangan. Klik Tambah dan tentukan direktori tempat file konfigurasi mesin virtual disimpan. Secara default, ini adalah direktori / etc / pve / nodes / pve / qemu-server / . Jika Anda tidak hanya menggunakan mesin virtual, tetapi juga wadah LXC, tambahkan direktori / etc / pve / node / pve / lxc / . Dalam kasus saya, ini juga merupakan direktori / data .

Setelah membentuk daftar direktori dengan cara ini, klik Berikutnya :


Dari daftar drop-down repositori, pilih Storage yang dibuat sebelumnya. Kami menentukan panjang rantai untuk cadangan inkremental. Semakin banyak poin dalam kebijakan Retensi , semakin banyak ruang yang akan Anda hemat. Namun seiring dengan ini, keandalan cadangan akan menurun. Keandalan lebih penting bagi saya daripada jumlah ruang penyimpanan, jadi saya menempatkan 4 poin. Anda dapat mengambil nilai standar 7 . Kami terus mengonfigurasi tugas dengan mengklik Berikutnya :

Kami mengkonfigurasi penjadwal. Ini adalah salah satu fitur paling keren untuk membuat hidup lebih mudah bagi administrator sistem. Dalam contoh, saya memilih untuk memulai pencadangan secara otomatis setiap hari pukul 2 pagi. Fitur hebat lainnya adalah kemampuan untuk mengganggu tugas cadangan jika kita melampaui batas waktu "jendela cadangan" yang diberikan. Jadwal pastinya dikonfigurasi melalui tombol Window :


Sekali lagi, misalnya, misalkan kita melakukan pencadangan hanya selama jam tidak bekerja pada hari kerja, dan pada akhir pekan kita umumnya tidak dibatasi waktu. Kami membentuk tabel yang sedemikian indah, kembali ke jendela sebelumnya dan klik Terapkan :


Tetap hanya untuk memeriksa informasi ringkasan tentang tugas dan klik tombol Selesai :


Ini melengkapi pembuatan pekerjaan cadangan.

Cadangkan


Semuanya dasar di sini. Di jendela utama program, pilih tugas yang dibuat dan klik Mulai . Sistem akan secara otomatis terhubung ke server kami (atau beberapa server), memeriksa ketersediaan penyimpanan dan mencadangkan jumlah ruang disk yang diperlukan. Kemudian, pada kenyataannya, proses pencadangan akan dimulai, dan setelah selesai kami akan menerima informasi yang komprehensif tentang proses tersebut.
Jika masalah bentuk gagal memuat modul [veeamsnap] dengan parameter [zerosnapdata = 1 debuglogging = 0] terjadi selama proses startup cadangan , maka Anda perlu membangun kembali modul veeamsnap sesuai dengan instruksi .

Yang sangat menarik adalah bahwa pada server itu sendiri kita tidak hanya dapat melihat daftar semua tugas cadangan yang sudah selesai, tetapi juga menonton perintah veeam secara real time :


Memprediksi pertanyaan mengapa konsol terlihat sangat aneh, saya akan katakan segera: Saya sangat suka bagaimana konsol terlihat di layar monitor tabung CRT yang hangat. Ini dilakukan dengan menggunakan emulator terminal keren-retro-term .

Pemulihan data


Sekarang pertanyaan yang paling penting. Tetapi bagaimana cara memulihkan data jika sesuatu yang tidak dapat diperbaiki terjadi? Sebagai contoh, mereka secara tidak sengaja menghapus mesin virtual yang salah. Dalam GUI Proxmox, semuanya lenyap sama sekali, tidak ada yang tersisa di penyimpanan di lokasi mesin.

Proses pemulihannya sederhana. Kami pergi ke konsol Proxmox dan masukkan perintah:

veeam

Kami akan melihat daftar cadangan yang selesai. Pilih panah dengan anak panah dan tekan R key . Selanjutnya, pilih titik pemulihan dan tekan Enter :


Setelah beberapa detik, titik pemulihan akan dipasang di direktori / mnt / cadangan .

Yang tersisa adalah menyalin drive virtual dan file konfigurasi mesin virtual di tempatnya masing-masing, setelah itu mesin "terbunuh" akan muncul secara otomatis di Proxmox VE GUI. Anda bisa menjalankannya dengan cara biasa.

Untuk meng-unmount titik pemulihan, Anda tidak harus melakukan ini secara manual, tetapi Anda harus menekan tombol U di utilitas veeam .
