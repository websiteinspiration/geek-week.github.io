GitLabリポジトリにプッシュするときにシェルランナーでJavaプロジェクトを実行する方法

私の名前はIvan Sidorenkoです。私はDigital Designのテストエンジニアです。私の仕事は、Java Seleniumで書かれたセルフテストの立ち上げを実装することでした。

CI / CDは初めてなので、多くの問題に遭遇しました。このタスクの詳細な手順はインターネット上で見つかりませんでした。そのため、開発者の助けを借りてそれを理解した後、私と同じように新規参入者向けのマニュアルを書くことにしました。

Javaを使用して記述された単純なHelloWorldプログラムを実行するなど、GitLabを使用してCI / CDを構成するために必要なすべての手順を詳しく説明しました。



自動テストは、開発者がGitLabリポジトリに変更をプッシュするたびに実行する必要があります。このタスクは、CI / CDの原則の1つである製品開発の各段階での自動テストの開始に対応しています。このマニュアルでは完全なCI / CDの概念については説明しません。この情報はインターネットで簡単に見つけることができます。

ただし、必要なすべての手順の説明とともに、初心者向けにGitLabを使用してCI / CDを構成する方法はわかりません。これについては以下で説明します。

1. GitをインストールしてGitLabリポジトリにアクセスする


プロジェクトを開始するために必要なすべてのものをリモートマシンにデプロイしましたが、このマニュアルの目的はCI / CDのデプロイの基本を理解することなので、ローカルマシンで作業します。

[ プロジェクト ]ページのGitLabに移動し、表示されるウィンドウで[ 新しいプロジェクト ]ボタンをクリックします。プロジェクト


名を設定し、[可視性プロジェクト]フィールドで値を[プライベート]に設定して、緑色の[ プロジェクトの作成 ]ボタンをクリックします。



おめでとうございます。GitLabのプロジェクトが作成されました。



次に、変更をGitLabにプッシュするローカルリポジトリをデプロイする必要があります。これを行うには、最初にGitをインストールする必要があります。インストール中に、すべてのデフォルト値を選択できます。 gitとそのインストールの経験がない場合は、このコースに慣れることをお勧めしますwww.youtube.com。

次に、Git環境変数user.nameおよびuser.emailを構成する必要があります。 GitLab-> Project / Detailsページから値を取得します（プロジェクトを作成するとすぐにこのページが表示されます）。

git config --global user.name "user name"
git config --global user.email "email@examle.com"

Git Bashを開きます。これを行うには、デスクトップ上の任意の場所をクリックして、[Git Bash Here]を選択します。次に、これらの2つのコマンドを順番に入力します。

その後、GitLabのすべての変更をプッシュするローカルリポジトリを作成する必要があります。これを行うには、ローカルマシンにフォルダーを作成してそこに移動し、Git Bashを開いてこれらのコマンドを順番に実行します（GitLab-> Project / Detailsページでも利用できます）。ここでは、プロジェクトへのパスを示します。 これで、リモートリポジトリが追加されました。ただし、GitLabリポジトリへの変更のプッシュを開始する前に、SSHキーを構成する必要があります。

git init
git remote add origin



2. SSHキーの構成


SSHプロトコルは、リモートリポジトリからメインgitリポジトリへの安全なデータ転送にSSHキーが必要です。鍵ペアを作成します。

Git Bashを開き、次のように入力します。

ssh-keygen -o -t rsa -b 4096 -C email@example.com

email@example.comの代わりに、私たちはあなたのメールを置き換えます。

次に、キーが保存されるメッセージが表示されます。ここでは、Enterキーを押すだけです。キーは次の形式のフォルダに保存されます：

C:\Users\User_name\.ssh

フォルダを選択すると、キーのパスワードを入力するように求められます。Enterキーを2回押すと、この手順をスキップできます。

次に、公開鍵をGitLabアカウントに追加する必要があります。 .sshフォルダーに移動し、メモ帳でid _rsa.pub idファイルを開きます。ファイルの内容をコピーして、GitLabに移動します。ここで、ユーザー設定に移動する必要があります（右上のボタンをクリックして[設定]を選択します）。



[SSHキー]セクションに移動します。

[ キー ]フィールドに公開キーを挿入し、[ キーの追加 ]ボタンをクリックします。



次に、秘密鍵をリモートマシンに保存する必要があります。これを行うには、次のコマンドを実行します。

ssh-agent bash
ssh-add ~/.ssh/key

ここで、keyは生成された秘密鍵（「id_rsa」という名前のファイル）です。ssh
-agentは、公開鍵による認証に使用される秘密鍵（この場合はRSA鍵）を格納するためのプログラムです。

これで、GitLabでファイルをプッシュする準備ができました。すべてが機能するかどうかを確認します。以前に初期化されたリポジトリに移動します。空のtxtファイルを追加します。その後、これらのコマンドをGit Bashで順次実行します。

git add .
git commit -m "Initial commit"
git push -u origin master

プロジェクト/詳細ページのGitLabに移動し、すべてが正常に機能していることを確認します。



3. GitLabランナー（シェル）の作成


gitlabにプッシュするときにプロジェクトを開始するには、ランナーをデプロイする必要があります。 GitLab Runnerは、特別な.gitlab-ci.ymlファイルから命令を実行するためのエージェントです。ランナーにはいくつかのタイプがあります。最も簡単なものの1つであるシェルランナーのインストールと起動を検討してください。

Shell-runnerを使用すると、このランナーがインストールされているマシンでプロジェクトを実行できます。このランナーをローカルマシンにインストールすることを検討してください

。1.ランナー用のフォルダーを作成します。たとえば、C：\ GitLab-Runner

2.リンクからランナーをダウンロードし
ます。https：//gitlab-runner-downloads.s3.amazonaws.com/latest/binaries/ gitlab-runner-windows-386.exe gitlab-runner.exeに
名前を変更します。

3.管理者としてPowerShellを実行します。

4.フォルダーに移動します。

cd C:\GitLab-Runner

5. GitLabへのリンクを入力します。GitLabで、[設定]-> [CI / CD]-> [ランナー]に移動します。



6.トークンを入力します



。7.ランナーの説明を入力します。

8.ランナーのタグを入力します。

9.ランナーのタイプ-シェルを選択します。

10.ランナーをインストールします。

gitlab-runner install

11.そして実行：

gitlab-runner start

ランナーが開始したことを確認するには、GitLab->設定-> CI / CD->ランナーに移動します。緑の円は、ランナーが走っていることを意味します。



4.環境変数を設定する


Javaプログラムを実行するには、次のものが必要です

。1. javaをインストールします：
https : //java.com/en/download/

2. jdkをインストールします：https : //www.oracle.com/technetwork/java/javase/downloads/index .html

3.環境変数を設定します。

環境変数を設定するには、次の場所に移動します。
マイコンピュータ->プロパティ->詳細環境設定->環境変数
ここで、「システム変数」セクションでJAVA_HOME変数を作成し、「変数値」の行にjdkへのパスを指定します。



この値をPath変数に追加します。また、Javaのパスがあることを確認する必要があります。



5.プロジェクトの作成とその立ち上げ


「Hello World！」を出力する単純なJavaプロジェクトを作成してみましょう コンソールで。ローカルリポジトリに配置します。これは次のようになります。



ここでは、拡張子がymlの.gitlab-ciファイルを作成しました。

コンテンツ.gitlab-ci.yml
image: java:latest

stages:
  - build
  - execute
     
build:
 stage: build
 script: 
  - cd C:\GitAuto\src
  - javac Main.java
  
 artifacts:
 tags:
    - first-runner

execute:
  stage: execute
  script:
    - cd C:\GitAuto\src
    - java Main
  artifacts:
  tags:
    - first-runner


ビルドと実行の2つの段階があります。javacコマンドを使用してソースコードをコンパイルし、javaコマンドを使用して実行します。各ステップで、使用するランナーを明示的に指定する必要があります。ランナーはそのタグの値によって決定されます（GitLabのランナー設定で確認できます）。

ファイルの内容を設定したら、ローカルリポジトリに移動してGit Bashを開きます。コマンドを順次実行し始めます：

git add .
git commit -m "Initial commit"
git push -u origin master

変更が開始され、プログラムが正常に完了したことを確認します。GitLab-> CI / CD-> Jobsに移動し、最後のジョブを開きます



。プログラムが完了しました。
