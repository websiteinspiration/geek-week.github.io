Tidak ingin Visual Studio Code: 7 alternatif open source

Dalam terjemahan ini, kami akan memberi tahu Anda tentang editor kode alternatif yang tidak kalah dengan VS Code, dan bahkan mengungguli jika Anda memilih ekstensi yang tepat.







Visual Studio Code Editor, juga dikenal sebagai VS Code berjalan di Linux, Windows dan macOS. Ini adalah perantara antara editor kode sumber teks sederhana dan lingkungan pengembangan terintegrasi (IDE) yang mengelola seluruh basis kode Anda. Fungsionalitas Kode VS dapat diperluas dengan plugin. Ini adalah editor kode yang andal yang dapat bersaing dengan kompetitor.



VS Code pada awalnya dibuat sebagai produk open source. Namun, build yang telah selesai didistribusikan di bawah lisensi eksklusif dari Microsoft. Tapi itu tidak terlalu buruk.





Menggunakan VS Code sebagai sumber terbuka



Sumber VS Code tersedia di GitHub . Namun, ketika Anda mengunduh rakitan yang sudah selesai , Lisensi Microsoft mulai berlaku . Dan kemudian Kode VS Anda tiba-tiba berhenti menjadi produk sumber terbuka. Ternyata dalam hal ini menyelesaikan proses perakitan.



Chris Diaz, pengembang proyek VS Code, menjalankan contoh serupa browser Chrome dan proyek sumber terbuka Chromium. VS Code benar-benar dibangun berdasarkan solusi open source. Namun, build resmi Microsoft disesuaikan untuk fitur-fitur tertentu, termasuk merek dagang, galeri ekstensi, debugger berpemilik C #, dan telemetri. Tetapi ketika Anda mengkloning dan membuat perakitan sendiri, Anda menghasilkan versi yang lebih bersih yang disebut Code-OSS (OSS singkatan dari perangkat lunak open source).



Dalam praktiknya, perbedaan antara VS Code dan Code-OSS minimal. Secara khusus, telemetri berfungsi dalam VS Code: yaitu, Microsoft mengumpulkan data tentang penggunaan produk. Dalam keadilan, perlu dicatat bahwa saat ini ada banyak produk yang mengumpulkan data tersebut.



Jika Anda suka VS Code, tetapi ingin mempertimbangkan alternatif open source yang "benar", maka Anda harus membaca.



Vscodium











Alternatif paling jelas untuk VS Code adalah VS Code tanpa add-on milik Microsoft. Proyek VSCodium hanya berisi komponen gratis yang dibangun berdasarkan basis kode VS Code. Pengembang VSCodium telah melakukan segala yang mungkin sehingga kami dapat menonaktifkan telemetri. Mereka telah menyiapkan versi bersih dari kode sumber VS Code, yang dapat Anda unduh tanpa membuatnya sendiri.



Pengembang ingat bahwa VS Code mencakup beberapa alat berpemilik yang mungkin tidak dikirimkan dengan sumber terbuka. Jika Anda membutuhkannya, ada cara yang terdokumentasi untuk menyelesaikan masalah ini. Tetapi jika Anda memerlukan fitur yang sangat spesifik dari Microsoft VS Code build, Anda perlu memastikan bahwa itu bekerja di VSCodium.



Selain itu, Anda perlu memeriksa bahwa semua telemetri dinonaktifkan .



Kode-OSS



Jika Anda tidak ingin menggunakan rakitan VSCodium, Anda dapat mengkompilasi Kode VS sendiri dari kode sumber dan berakhir dengan hal yang sama. Executable disebut Code-OSS, bukan VSCode. Persyaratan lisensi dan solusi untuk masalah menggunakan fitur eksklusif sama dengan VSCodium.



Pada awal pertama, Anda perlu memeriksa bahwa telemetri dinonaktifkan.



Atom











Atom adalah editor teks mirip IDE sumber terbuka yang diperoleh Microsoft dengan GitHub. Seperti VS Code, Anda dapat memperluas editor Atom dengan plugin, menambahkan tema, dan membuat kombinasi alat tambahan sendiri. Ini adalah produk lintas platform yang memiliki integrasi terintegrasi dengan GitHub. Singkatnya, Atom ideal jika ekstensi yang Anda butuhkan sudah ada atau Anda siap untuk menulisnya.



Seperti VS Code, Atom menggunakan telemetri secara default.. Itu dapat dinonaktifkan, dan tidak seperti VS Code, tidak ada batasan pada ekstensi, jadi Anda tidak perlu mengubah alur kerja karena persyaratan privasi. Atom tentu saja merupakan alat yang berguna bagi pengembang. Selain itu: pada prinsipnya cocok untuk semua orang yang menggunakan komputer. Jika Anda mencari editor teks untuk keperluan umum yang baik, Anda dapat mencoba Atom.



Pembuat GNOME











GNOME Builder adalah editor kode Linux yang dirancang khusus untuk membangun aplikasi berbasis GNOME. Jika Anda membuat aplikasi untuk Linux dan ingin dengan mudah mencapai kompatibilitas, maka Builder adalah pilihan terbaik. Pembuat GNOME tersedia di Flathub.org . Ketika Anda pertama kali memulainya, itu akan meminta Anda untuk menginstal GNOME SDK jika Anda belum.



Namun, Anda dapat menggunakan Builder tidak hanya untuk mengembangkan aplikasi GNOME. Ini mendukung puluhan bahasa pemrograman, termasuk Python, Rust, C dan C ++, Java, Go, JavaScript, TypeScript, VB.NET, beberapa bahasa markup, penurunan harga, dan banyak lainnya. Beberapa dari mereka memiliki dukungan penuh dengan pelengkapan otomatis dan cuplikan, sementara yang lain memiliki fasilitas minimal, seperti penyorotan sintaksis dan pencocokan tanda kurung otomatis. Sangat menyenangkan bekerja dengan IDE, apakah Anda menganggap diri Anda seorang programmer yang serius atau hanya ingin menikmati HTML dan CSS.



Geany











Geany adalah editor yang kuat, andal, dan ringan dengan fitur yang berguna untuk membantu Anda menulis kode yang baik dalam Bash, Python, Lua, XML, HTML, LaTeX dan sebagainya. Ini mendukung 50 bahasa pemrograman dan scripting yang berbeda, serta bahasa markup dan file (seperti .diff dan .po). Geany setidaknya menyediakan tanda kurung dan penyorotan sintaks, dan biasanya menawarkan lebih banyak.



Menggunakan plugin, Anda dapat menambahkan fitur seperti browser proyek, explorer, debugging, terminal, dan sebagainya. Anda dapat terus memperluas fungsinya dan pada titik tertentu akan berubah menjadi IDE nyata. Jika Anda tidak dapat menjalankan VS Code di komputer karena keterbatasan CPU atau RAM, Geany mungkin merupakan alternatif yang baik. Ini dimulai dengan cepat dan mengkonsumsi sejumlah kecil memori. Meskipun Geany lebih lambat dari Vim di terminal, itu tidak melambat bahkan pada Raspberry Pi.



Kurung











Kurung adalah editor teks dan IDE untuk pengembang web. Ini memiliki dukungan kuat untuk HTML, CSS, JavaScript, PHP, dan bahkan Python. Kurung memiliki ekosistem ekstensi yang kaya, jadi dengan plugin yang sesuai itu akan memungkinkan Anda untuk menulis dalam hampir semua bahasa modern.



Ada ekstensi untuk membantu menganalisis, menjalankan skrip, dan bahkan mengkompilasi dan mengeksekusi kode secara langsung di jendela editor. Kurung memiliki antarmuka tradisional yang dapat dipahami oleh hampir semua orang yang melampaui notebook. Jika Anda menghabiskan sedikit waktu menambahkan ekstensi yang sesuai dan mengenalnya, tanda kurung menggunakan pelengkapan otomatis dan linter akan membantu Anda menghindari kesalahan pengetikan yang bodoh. Dan jika Anda menulis kode, ini akan mempercepat proses pengujian dan debugging.



Che











Jika Anda ingin menjadi yang terdepan, maka Anda harus mencoba Che . Ini adalah IDE berbasis cloud, ini bekerja pada model SaaS. Namun demikian, ini adalah proyek open source yang lengkap. Jika Anda memiliki salinan Kubernetes, Anda dapat sepenuhnya mentransfernya sendiri.



Che bukan hanya IDE yang hidup di awan. Ini adalah IDE yang dirancang untuk pengembangan cloud. Itu tidak mencoba untuk menyimpan apa pun di sistem file lokal Anda. Jika Anda memiliki server Git, Anda dapat menganggapnya sebagai sistem file dan mengerjakan proyek langsung dari repositori. Tentu saja, Anda juga dapat mengunduh proyek apa pun yang sedang Anda kerjakan jika Anda ingin memiliki cadangan lokal.



Namun fitur utamanya yang dikagumi oleh pengembang cloud adalah Che adalah lingkungan pengembangan sumber terbuka lengkap yang berfokus pada Kubernetes. Jika Anda membuat aplikasi, situs atau wadah (atau kombinasi dari semuanya) untuk cloud, maka Anda harus mencoba Che.



Apa yang kamu gunakan?



Apakah Anda menggunakan salah satu dari alternatif ini untuk VS Code? Atau mungkin, sambil masih berpikir untuk mencoba salah satu dari mereka? Bagikan pengalaman Anda di komentar?
