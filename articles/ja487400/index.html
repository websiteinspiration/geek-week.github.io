マスターSCADA 4D。ARMに生命はありますか？



産業オートメーションの分野でかなり幅広い経験を持っているため、私たちは常に問題を解決するための最良のオプションを模索しています。お客様の技術仕様に応じて、ソフトウェアとハ​​ードウェアのベースに立ち寄る必要がありました。また、TIAポータルと組み合わせてSiemens機器をインストールするための厳密な要件がない場合、原則として、選択はMasterSCADA 3.XXに当てはまります。しかし、永遠に続くものはありません...

MasterSCADA 4Dへの切り替えの経験、前提条件、この記事のカットの下での組み込みARMアーキテクチャーコンピューターでのその作業の機能について。

バックグラウンド


私たちは、Insatの比較的新しい開発であるMasterSCADA 4Dのテストを始めました。これにはいくつかの前提条件がありました。最初に、どのSCADAシステムが最も人気があるかを見つけるために、産業オートメーションの専門家の間でいくつかの独立した調査を行いました（図1）。調査によると、国内のシステムの中で最初の場所は、MasterSCADAシステムによって占められています。


図1-最も人気のあるSCADAシステムの調査結果（画像はクリック可能です）

2番目の前提条件を検討できます...

次に、MasterSCADA 4D自体に直接移動します。つまり、開発環境とランタイム環境という2つのソフトウェア製品で構成されています。これらの各部分の作業については、以下で説明します。

開発環境


システムプロジェクトはMasterSCADA 4D開発環境で作成されます。これを行うには、Insat Webサイトで無料バージョンを取得し、プロンプトに従ってインストールする必要があります。


図2-開発環境のインターフェース（写真はクリック可能です）

最初に目につくのは、開発環境の快適なインターフェースとプロジェクトを構築するための便利な階層構造です。これで、1つのプロジェクトで、ワークステーションだけでなく、コントローラ全体からサーバーまたはオペレータのワークステーションまで、オブジェクト全体のプログラムを作成できます。

開発環境はWindowsでのみ機能し、比較的馴染みがあり寛容ですが、ランタイム（RunTime）は、さまざまなオペレーティングシステムやプロセッサアーキテクチャに統合できることで驚きましたが、後で詳しく説明します。

また、視覚化要素の大規模なライブラリにも満足しています。さまざまな分野のスペシャリストが、インターネットでアイコンを描画したり検索したりすることなく、視覚化要素を見つけることができます。


図3-視覚化の要素（画像はクリック可能）

通信プロトコル


システムは、MasterSCADA 4Dにデフォルトで統合されているさまざまなドライバー（交換プロトコル）をサポートしています。

  • Modbus TCP / RTU、RTU over TCP
  • DCON
  • OPC UA / DA / HDA
  • IEC61850
  • SNMP
  • PostgreSQL
  • MQTT
  • IEC104
  • MSSQL
  • MySQL
  • 水星（別館）など


ランタイム


ランタイムは、さまざまなオペレーティングシステムやパソコンアーキテクチャで実行できます。また、RunTimeをローカルマシンで実行することもできます。ランタイムは開発環境と共にインストールされ、制限なしで1時間（または32タグ）実行されます。

AntexGateデバイス


MasterSCADAランタイムは、ARMプロセッサアーキテクチャとDebianオペレーティングシステムを備えたAntexGate組み込みPCに個別のオプションとしてプリインストールされています。このデバイスでテストを実施します。

  • 1.2Mhz RAM：LPDDR2 1024MB
  • 不揮発性メモリ：8/16 / 32GB eMMC

デバイスの詳細については、こちらをご覧ください。

アクチュエータでプログラムを実行します。たとえば、Modbus RTUプロトコルを使用して調査とデバイス制御を作成しました。調査の設定プロセスは直感的で、使い慣れたOPCサーバーの設定と多少似ています。データを交換するためのプロトコルドライバーがRunTimeに組み込まれました。

例として、3つのポンプと2つのバルブを制御する簡単なプロジェクトを作成します。これは、抽象的な生産プロセスに必要です。開発環境では、図5のように次のようになります。

HMI情報表示オプション


WEB経由でランタイム環境に接続することが可能です。このオプションは、模倣図でデータを表示するためのクライアントの選択を制限しません。
私たちの場合、デバイスはHDMI、イーサネット、3Gを介して情報を出力します。
HDMI経由で接続する場合、AntexGateの組み込みブラウザーを介してLocalHost 127.0 0.1：8043にアクセスするか、インターネット上の固定IP：8043アドレスまたは別のシンクライアントを使用して企業のローカルネットワークに接続します。


図7-WEB監視構造（クリック可能な画像）

興味深いニュースは、待望のMQTTプロトコルでした。これは通常、SCADAシステムでリモートオブジェクトを監視するには不十分でした。
今日、誰もがインターネット上で固定IPアドレス（たとえば、会社のWebサイトサーバー）を備えた安価なVDSサーバーを入手し、MQTTブローカー（たとえば、Mosquito）を展開する機会があります。
MQTTブローカーを備えた1台のサーバーを受け取ったので、オペレーターの高価なサービス（固定IP）を簡単に取り除くことができ、3G接続では4000ルーブルではなく、年間900ルーブルを支払うことができます。


図8-MQTT監視構造（クリック可能な画像）

インターネット上のModbus TCPプロトコルを介したデータ送信は通信のセキュリティと品質を保証しないため、このようなネットワーク構造はトラフィックを節約するだけでなく、データも保護します。
したがって、クライアント自身がインターネットプロバイダーを選択する複製プロジェクトを販売できます。また、IPアドレスの設定と割り当てに頭を悩ませることはありません。クライアント自身がSIMカードを挿入するか、DHCPサーバーを使用してルーターに接続します。

パフォーマンス


プロジェクトの主なものはスピードです。いわゆる「タスク」がこれを支援します。デフォルトでは、作成時の各ノードには1つしかありません-メインタスク。プロジェクト開発者は、特定のプロジェクトに必要な数だけそれらを作成できます。計算サイクルなどの計算の機能は、特定のタスクの設定によって異なります。それらのそれぞれは、デバイス内の他のものから独立して動作します。プロジェクトのさまざまなプログラムを計算するさまざまなサイクルを提供する必要がある場合は、いくつかのタスクを作成することをお勧めします。

この機能は、いくつかのコアを持つプロセッサを搭載したボード上のデバイスにとって特に興味深いものです。各「タスク」はシステムの個別のプロセスとして起動され、負荷はプロセッサに均等に分散されます。 AntexGateデバイスには、1.2 GHzの4つのコアと1 GBのRAMを備えたARMプロセッサがあり、これにより、少なくとも4つの大きなタスクを作成し、コア全体に負荷を分散できます。 PLCと比較して、AntexGateは同じ価格で少なくとも4倍の計算能力を提供できます。


図9-ランタイムでのAntexGateのコンピューティング機能のダウンロード（画像はクリック可能）

図9からわかるように、CPU負荷は2.5％以下であり、メモリは61MBしか割り当てられていません。したがって、実行時の小さなプロジェクトは、埋め込みリソースをほとんど消費しません。
このデバイスは、コントローラーとしてだけでなく、すでに2000を超えるI / Oポイントの調査と100を超えるWEBクライアントをサポートする機能を備えた本格的なサーバーとして使用できます。

たとえば、9台のWEBクライアントをデバイスに接続して、リソース消費の進行状況を確認します（図10）。


図10-9つのWEBクライアントを接続するときのAntexGateコンピューティング機能のダウンロード（画像はクリック可能）

上の図からわかるように、CPU負荷は平均で2.5％から6％に増加し、割り当てられたメモリは3MBだけでした。
デバイスのコンピューティングリソースが大量に供給されるため、開発者はMasterSCADA 4Dで作成されたプログラムの品質を節約する必要がありません。

クロスプラットフォーム


また、検討中のSCADAシステムのクロスプラットフォームの性質にも注目したいと思います。これにより、インテグレーターは、プロジェクトを実装するためのプラットフォームを非常に豊富に選択できます。このアプローチのおかげで、オペレーティングシステムまたはPCアーキテクチャ間の移行は非常に簡単です。

結論


MasterSCADA 4Dは、Insatの比較的新しい製品です。これまでのところ、このソフトウェア製品の操作に関する情報は、私たちが望むほど多くはありません。ただし、会社の公式Webサイトから無料の開発環境をダウンロードできます。プログラムの操作については非常に詳細なヘルプがあります。
