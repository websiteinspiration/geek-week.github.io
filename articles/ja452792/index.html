企業ユーザー向けのソリッドステートSSDレビューKingston DC500R

キングストンは最近、高一定負荷向けに設計されたエンタープライズクラスのソリッドステートドライブKingston DC500Rをリリースしました。現在、多くのジャーナリストが新製品を積極的にテストし、興味深い資料をリリースしています。私たちはキングストンDC500Rの詳細なレビューの1つをHabrと共有したいと思います。そのテストは読者を喜ばせます。オリジナルはStoragereview Webサイトにあり、英語で公開されています。お客様の便宜のために、資料をロシア語に翻訳し、カットの下に配置しました。読書を楽しむ！



ドライブキングストンDC500Rは、フラッシュメモリテクノロジー3D TLC NANDに基づいています。オプションは480 GB、960 GB、1.92 TB、および3.84 TBの容量で利用可能であり、コストを節約したい企業や、単に大容量ドライブを必要としない企業に追加の選択肢を提供します。このレビューでは、宣言された順次読み取りおよび書き込み速度がそれぞれ555 MB /秒および520 MB /秒である3.48 TBバージョンと、98,000および28,000入力操作の一定負荷での4 KBブロック読み取りおよび書き込み速度について説明します。 1秒あたりの出力（IOPS）、それぞれ。この製品ファミリーの一部として、キングストンは、混合使用アプリケーション用に最適化されたDC500Mも提供しています。

テスト情報

エンタープライズレベルのテストラボStorageReviewは、実際の条件に近い環境でストレージデバイスをテストするための十分な機会を提供します。ラボには、さまざまなサーバー、ネットワークデバイス、電源システム、およびその他のネットワークインフラストラクチャが含まれています。これにより、従業員は実際の状態に近い状態で機器の性能を正確に評価できます。
環境とプロトコルに関する情報はレビューに含まれているため、ITプロフェッショナルとストレージシステムの購入を担当する担当者は、対応する結果が得られた条件を評価できます。テスト対象の機器の製造元はレビューの費用を負担せず、管理もしません。

アプリケーションのワークロード分析

エンタープライズクラスのストレージデバイスのパフォーマンスパラメーターを適切に評価するには、実際の環境に適したインフラストラクチャとアプリケーションのワークロードをモデル化することが重要です。したがって、Samsung 883 DCT SSDを評価するために、SysBenchユーティリティを使用してMySQL OLTPデータベースのパフォーマンスと、TCP-CワークロードをエミュレートすることでMicrosoft SQL Server OLTPデータベースのパフォーマンスを測定しました。この場合、アプリケーションでは、各ドライブが2〜4台の均等に構成された仮想マシンを処理します。

SQL Serverのパフォーマンス

SQL Server仮想マシンごとに2つの仮想ディスクが構成されています。100GBの容量を持つブートディスクと、データベースとログファイルをホストするための500 GBの容量を持つディスクです。システムリソースに関しては、各仮想マシンに16個の仮想プロセッサ、64 GBのDRAM、およびLSI Logic SAS SCSIコントローラが搭載されていました。以前は、Sysbenchワークロードを使用して、I / O速度とドライブ容量の使用率をテストしました。次に、SQLテストはレイテンシの評価に役立ちます。

テストの一部として、SQL Server 2014はWindows Server 2012 R2を実行するゲスト仮想マシンに展開されます。負荷は、Quest Benchmark Factory for Databasesソフトウェアを使用して作成されます。StorageReviewのMicrosoft SQL Server OLTPデータベーステストプロトコルは、現在のバージョンのTransaction Processing Performance CouncilベンチマークCパフォーマンス評価プログラム（TPC-C）を使用しています。このリアルタイムトランザクション処理パフォーマンス評価プログラムは、複雑なアプリケーション環境のプロセスをシミュレートします。 TPC-Cテストでは、人為的なパフォーマンステストではなく、データベース環境におけるストレージインフラストラクチャの長所と短所をより正確に特定できます。テストでは、SQL Server仮想マシンの各インスタンスが333 GB（1500スケール）のSQL Serverデータベースで動作しました。トランザクション処理のパフォーマンスとレイテンシは、15,000の仮想ユーザーの負荷で測定されました。

VDBenchワークロード分析

ストレージデバイスをテストする場合は、模擬テストよりもアプリケーションテストの方が適しています。ただし、それらの結果は実際の条件に対応していませんが、タスクの再現性による合成テストは、基本的な指標の決定や競合ソリューションの比較に便利です。これらのテストは、四隅のテストやサンプルデータベース移行テストから、さまざまなVDI環境からのキャプチャの追跡まで、幅広いプロファイルを提供します。これらすべてのケースで、スクリプトプロセッサを備えた単一のvdBenchワークロードジェネレーターを使用して、計算テストの大規模なクラスターで結果を自動化および収集します。これにより、フラッシュアレイや個々のドライブを含む幅広いドライブに同じワークロードを使用することが可能になります。テストの一環として、アプリケーションの負荷をシミュレーションしてドライブの動作を評価するために、ドライブにデータを完全に入力し、元の容量の25％のパーティションに分割しました。このアプローチは、負荷が一定の場合、ディスク全体がすぐに使用される完全エントロピーテストとは異なります。このため、以下の結果は、より安定した記録速度を反映しています。

最初のVDBenchワークロードテスト（4Kランダム読み取り）で、キングストンDC500Rは印象的な結果を示しました：80,000 IOPSの速度レベルと1.59の遅延で80,209 IOPSの最大速度に達するまでの1 msの遅延MS



テストされたすべてのドライブは、2番目のテスト（ランダム書き込み4 KB）でほぼ同じ結果を示しました。速度は2 msの遅延で63,000 IOPSよりわずかに高くなっています。



シーケンシャルロードに移って、最初に64Kの読み取りを調べました。この場合、キングストンドライブは、5200 IOPS（325 MB /秒）に達するまでミリ秒以内の遅延を維持しました。 2.22ミリ秒の遅延がある7183 IOPS（449 MB /秒）の最大値により、このディスクは総合順位で2位になりました。



シーケンシャルレコーディング操作をテストしたところ、Kingstonデバイスは、遅延を1 ms未満から5700 IOPS（356 MB /秒）の速度まで維持することで、すべての競合他社を上回りました。最大速度は6291 IOPS（395 MB /秒）で、遅延は2.51ミリ秒でした。



その後、SQLタスクに移りました。キングストンDC500Rは、3つのテストすべてで遅延レベルがミリ秒を超えた唯一のデバイスでした。最初のケースでは、ディスクの最大速度は26411 IOPS、遅延は1.2 msでした。



SQL 90-10テストでは、キングストンが27.339 IOPSの最大速度で1.17ミリ秒の遅延で最後の位置を占めました。



SQL 80-20テストでも同じことが起こりました。この場合のKingstonデバイスは、1.08ミリ秒の遅延で29576 IOPSの最大速度を示しました。



Oracleワークロードのテスト結果により、DC500Rは最後の場所に戻りましたが、2つのテストでは、デバイスは依然としてミリ秒以内の遅延を示しました。最初のケースでは、キングストンの最大ディスク速度は29098 IOPS、遅延は1.18ミリ秒でした。



2番目のテスト（Oracle 90-10）では、DC500Rは894.3μsの遅延で24555 IOPSを達成しました。



3番目のテスト（Oracle 80-20）では、キングストンのデバイスの最大速度は26401 IOPSで、遅延レベルは831.9μsでした。



次に、VDIのコピーに進み、完全なリンクされたコピーを作成しました。 VDIのフルコピーのダウンロードをテストしたところ、キングストンドライブは競合他社を再び上回りませんでした。デバイスは、1 ms未満から約12000 IOPSまでの遅延をサポートし、最大速度は2.203 msの遅延で16203 IOPSでした。



VDIの初期ログインコピーをテストすると、キングストンデバイスのパフォーマンスが向上し、最終的に（わずかなマージンを確保して）2位になりました。ドライブは、11000 IOPSの速度に達するまでのミリ秒以内の遅延をサポートし、最大速度は2.182ミリ秒の遅延で13652 IOPSでした。



また、わずかなマージンで、キングストンドライブは月曜日のログインで2位になり、VDIの完全なコピーのテスト結果が出ました。 Seagate Nytro 1351ドライブはわずかに高い最大速度を示しましたが、キングストンデバイスは一般に、テスト全体を通して低いレイテンシレベルを示しました。 DC500Rの最大速度は1.3897ミリ秒の遅延で11897 IOPSでした。



VDIのリンクされたコピーのダウンロードをテストする際、キングストンは最後の場所にありました。遅延はすでに6000 IOPS未満の速度で1 msを超えました。 DC500Rの最大速度は2.03ミリ秒の遅延で7861 IOPSでした。



ただし、初期ログインテストの結果によると、ドライブは再び2位になりました。遅延は、ピークパフォーマンスに達した後にのみミリ秒の制限を超え、最終的には1.001ミリ秒の遅延で7950 IOPSに達しました。



リンクされたVDIのコピーの最後のテスト-月曜日のログイン-ディスクにも2番目の結果が表示されました。最大速度は9205 IOPSで、遅延は1.72 msです。ミリ秒外では、速度が6400 IOPSに達したときに遅延が超過しました。



結論


DC500Rは、エンタープライズユーザー向けのキングストンの最新のソリッドステートドライブです。 DC500Rは、2.5インチフォームファクターで利用できます。利用可能な容量の範囲は480 GB〜3.84 TBです。ドライブは3D TLC NANDフラッシュメモリテクノロジーに基づいており、大容量のリソースと高レベルのパフォーマンスを兼ね備えています。 3.48 TBのディスクの場合、それぞれ555および520 MB /秒の順次読み取りおよび書き込み速度、それぞれ98,000および28,000 IOPSの一定負荷での読み取りおよび書き込み速度、3504 TBWのリソース容量が要求されます。

Kingston DC500Rのパフォーマンスを評価するために、Samsung 860 DCTおよび883 DCTドライブを含む他の一般的なSATA SSDと比較しましたSeagate Nytro 3530ドライブも同様です。 Kington DC500Rは、競合他社のレベルを維持することができ、場合によってはそれを超えることもありました。アプリケーションのワークロードをテストする際、Kingston DC500RはSQLタスクの処理時に非常に優れたパフォーマンスを発揮し、1秒あたりのトランザクション数（6291.8 TPS）と待機時間（26.5 ms）で総合2位を獲得しました。より強力な書き込みパフォーマンスを備えたSysbenchテストでは、DC500Rはリストの最後に達し、1,680.5 TPSのパフォーマンスで76.2ミリ秒の平均待機時間と134.9ミリ秒で最も好ましくないシナリオの待機時間となりました。

4 KBのランダムな読み取りおよび書き込みブロックのテストの一環として、Kingston DC500Rは80209 IOPSの速度と読み取り時の1.59ミリ秒の遅延、および63000 IOPSの速度と書き込み時の2ミリ秒の遅延を示しました。 64 KBの読み取りおよび書き込みテストの一部として、DC500Rは、それぞれ2.22ミリ秒の遅延で7183 IOPS（449 MB /秒）の速度に達し、2.51ミリ秒の遅延で6291 IOPS（395 MB /秒）の速度に達しました。 SQLおよびOracleデータベースを使用した模擬テストと書き込み速度の要件の増加のコンテキストでは、DC500Rのパフォーマンスは低くなりました。 SQLワークロードに関して、Kingston DC500Rは3つのテストすべてで最後の位置を占め、遅延レベルがミリ秒を超えた唯一のドライブであることが判明しました。ただし、Oracleのテストでは、状況ははるかに良好でした。3つのテストのうち2つで、ドライブは1 ms未満の遅延をサポートし、次点となりました。 Kingston DC500Rは、VDIのコピーを使用してテストしたときに、フルレベルと関連レベルの両方でまともなレベルのパフォーマンスを示しました。

キングストンDC500R SSD 全体-注目に値するクラスの高品質デバイス。高性能テクノロジ（NVMeとアナログ）がどれほど気に入っても、信頼性が重要な役割を果たす処理タスク（サーバーやストレージコントローラーの読み込みなど）に最適なソリューションであるのは、SATAドライブです。このようなドライブは、費用対効果が非常に重要な状況でサーバーデータを保存するための経済的なソリューションでもあります。さらに、それらは、ハードドライブのバックグラウンドからソリッドステートドライブが際立つという総所有コストの点ですべての利点を提供します。 DC500Rパフォーマンスにより、他の価値のあるドライブと比較して行った多くのテストの結果によると、デバイスはトップラインを占めることができます。DC500Rは、大容量と幅広い容量を備えた信頼性と生産性の高いドライブを必要とするシナリオに最適なSATAドライブです。
