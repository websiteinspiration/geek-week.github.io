.NET 5プレビュー1の紹介

昨年末、.NET Core 3.0および3.1をリリースしました。これらのバージョンでは、Windowsフォーム（WinForms）とWPF、単一ページアプリケーションを作成するためのASP.NET Blazorデスクトップアプリケーションモデル、および契約ベースのクロスプラットフォームメッセージング用のgRPCが追加されました。また、サービスを作成するためのテンプレート、gRPCと通信するためのクライアントコードのクールな生成、REST APIサービスなども追加しました。 .NET Core 3が.NETの最も速く受け入れられたバージョンになり、昨年1年間でさらに100万人のユーザーがいることを嬉しく思います。

また、これらのリリースにも取り組み、.NET Frameworkからのアプリケーションモデルの移行を完了しました。 .NET Core 3では、最も使用頻度の高いアプリケーションモデルをすべて移行し、移植されなかったものではなく、新しいクロスプラットフォームインフラストラクチャを導入しました。

.NET 5の次のメジャーリリースを見越して、.NET 5に.NETモバイルアプリケーションモデル（Xamarin）を含めることにより、引き続き.NETを単一のプラットフォームに統合します。.NET5には、ASP.NET Core、Entity Framework Core、WinForms、 WPF、Xamarin、ML.NET。初めて、プラットフォーム全体ですべてのアプリケーションモデルに統一されたBCL（基本クラスライブラリ）が使用されます。 .NET Coreおよび.NET Frameworkよりも高いバージョン5の存在は、.NET 5があらゆるタイプのアプリケーションを作成するための統合された統合プラットフォームである.NETの未来であることも明らかにしています。

これは何度も言っていますが、繰り返します。.NET Core、次に.NET 5は.NETであり、これを使用してすべての新しいアプリケーションを作成する必要があります。.NET Frameworkは、Windows自体がサポートされている限りサポートされます。引き続きセキュリティを提供し、バグを修正するとともに、ネットワークと暗号APIを更新します。.NET Frameworkで古いアプリケーションを実行するために、安全でサポートされたままになります。

イベントプロファイリングAPI（イベントパイプ）


イベントパイプは、.NET Core 2.2に追加した新しいサブシステムおよびAPIであり、任意のオペレーティングシステムでパフォーマンス診断などを実行できるようにします。.NET 5.0では、イベントパイプが拡張され、プロファイラーがイベントパイプイベントを記録できるようになりました。このシナリオは、以前にETWを使用してアプリケーションの動作とパフォーマンスを監視していたプロファイラーツールにとって重要です。
