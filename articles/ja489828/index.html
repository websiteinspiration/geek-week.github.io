Node.jsでのGeoJSONの操作：実用的な入門

GeoJSONは、JSONに基づく地理データ構造を表すための標準化された形式です。GeoJSONデータを視覚化するための多くの優れたツールがあります。さらに、このフォーマットは特定のポイントの座標の保存だけでなく、優れています。ポイントに加えて、ライン、ポリゴン、オブジェクトのコレクションなど、他のオブジェクトを記述することができます。



ポイント-ポイントオブジェクト


GeoJSONポイントは次のようになります。

{
  "type": "Point",
  "coordinates": [-80.1347334, 25.7663562]
}

このポイントは、米国フロリダ州マイアミビーチの公園を表しています。geojson.ioプロジェクトを使用すると、この点を地図上で簡単に視覚化できます。


地図上のポイント

プロパティの座標coordinatesはの形式で記述されていることに注意してください[lng, lat]。GeoJSONの経度は緯度より前になります。これは、経度が東西方向（x典型的なマップの軸）を表し、緯度が南北方向（y典型的なマップの軸）を表すためです。GeoJSONの作者は、座標の順序を維持しようとしましたx, y。

GeoJSONポイントの典型的な使用例は、ジオコーディング —経度と緯度で表された座標への「429 Lenox Ave、Miami Beach、FL」のようなアドレス変換です。たとえば、 APIを使用しますジオコーディングマップボックス。このAPIにアクセスするには、次のエンドポイントに対してHTTPリクエストを行う必要があります。

https://api.mapbox.com/geocoding/v5/mapbox.places/429%20lenox%20ave%20miami.json?access_token=pk.eyJ1IjoibWF0dGZpY2tlIiwiYSI6ImNqNnM2YmFoNzAwcTMzM214NTB1NHdwbnoifQ.Or19S7KmYPHW8YjRz82v6g&cachebuster=1581993735895&autocomplete=true

それに応じて、次のコードが表示されます。

{"type":"FeatureCollection","query":["429","lenox","ave","miami"],"features":[{"id":"address.8052276751051244","type":"Feature","place_type":["address"],"relevance":1,"properties":{"accuracy":"rooftop"},"text":"Lenox Avenue","place_name":"429 Lenox Avenue, Miami Beach, Florida 33139, United States","center":[-80.139145,25.77409],"geometry":{"type":"Point","coordinates":[-80.139145,25.77409]}, ...}

答えをよく見るとfeatures[0].geometry、JSONコードではGeoJSONポイントであることがわかります。

{"type":"Point","coordinates":[-80.139145,25.77409]}




API静的マップの座標の視覚化 Mapboxは、マップ上にポイントを表示するための優れたツールです。以下は、渡された文字列をデコードし、最初の検索結果を示す画像へのURLを返すスクリプトです。

const axios = require('axios');

async function search(str) {
  const geocoderUrl = 'https://api.mapbox.com/geocoding/v5/mapbox.places/' +
    encodeURIComponent(str) +
    '.json?access_token=' +
    'pk.eyJ1IjoibWF0dGZpY2tlIiwiYSI6ImNqNnM2YmFoNzAwcTMzM214NTB1NHdwbnoifQ.Or19S7KmYPHW8YjRz82v6g';

  const res = await axios.get(geocoderUrl).then(res => res.data);
  const point = res.features[0].geometry;

  return 'https://api.mapbox.com/styles/v1/mapbox/streets-v11/static/' +
    'pin-l-1+333(' + point.coordinates[0] + ',' + point.coordinates[1] + ')/' +
    point.coordinates[0] + ',' + point.coordinates[1] +
    ',14.25,0,0/600x600/' +
    '?access_token=pk.eyJ1IjoibWF0dGZpY2tlIiwiYSI6ImNqNnM2YmFoNzAwcTMzM214NTB1NHdwbnoifQ.Or19S7KmYPHW8YjRz82v6g';
}

search('429 Lenox Ave, Miami Beach').then(res => console.log(res));

const axios = require('axios');

async function directions(fromPt, toPt) {
  const fromCoords = fromPt.coordinates.join(',');
  const toCoords = toPt.coordinates.join(',');
  const directionsUrl = 'https://api.mapbox.com/directions/v5/mapbox/driving/' +
    fromCoords + ';' + toCoords + '?' +
    'geometries=geojson&' +
    'access_token=pk.eyJ1IjoibWF0dGZpY2tlIiwiYSI6ImNqNnM2YmFoNzAwcTMzM214NTB1NHdwbnoifQ.Or19S7KmYPHW8YjRz82v6g';

  const res = await axios.get(directionsUrl).then(res => res.data);
  return res.routes[0].geometry;
}

const wework = { type: 'Point', coordinates: [-80.139145,25.77409] };
const airport = { type: 'Point', coordinates: [-80.2752743,25.7938434] };

directions(wework, airport).then(res => {
  console.log(res);
});

ポリゴン-ポリゴンオブジェクト


GeoJSONポリゴン、オブジェクトはPolygon、マップ上の閉じた領域を説明するために使用されます。これらは、三角形、正方形、十二角形、または固定数の辺を持つ他の任意の図の形の領域にすることができます。たとえば、次のGeoJSONオブジェクトは、米国のコロラド州の境界を大まかに示しています。

{
  "type": "Polygon",
  "coordinates": [[
    [-109, 41],
    [-102, 41],
    [-102, 37],
    [-109, 37],
    [-109, 41]
  ]]
}


地図上のポリゴンの視覚化

GeoJSONポリゴンは、非常に複雑な形状を表すために使用できます。たとえば、しばらくの間、Uberはサンフランシスコ湾岸地域の3つの主要空港すべてを含む唯一のGeoJSONトレーニング場を使用していました。


複雑なGeoJSONポリゴン

True、GeoJSONポリゴンは円と楕円を表すことができないことに注意してください。

ポリゴンは何に使用されますか？通常-ジオフェンスを説明します。たとえば、UberまたはLyftで働いているとします。空港からの旅行を予約するユーザーに特別な画面を表示する必要があります。これを行うには、旅行が注文されたポイントが空港（または前の図のようにいくつかの空港）を表すポリゴン内にあるかどうかを確認する必要があります。

GeoJSONポイントがポリゴン内にあることを確認する1つの方法は、 Turpm npmモジュールを使用することです。このモジュールで @turf/boolean-point-in-polygonは、ポイントがポリゴン内にあるかどうかを確認できます。

const pointInPolygon = require('@turf/boolean-point-in-polygon').default;

const colorado = {
  "type": "Polygon",
  "coordinates": [[
    [-109, 41],
    [-102, 41],
    [-102, 37],
    [-109, 37],
    [-109, 41]
  ]]
};

const denver = {
  "type": "Point",
  "coordinates": [-104.9951943, 39.7645187]
};

const sanFrancisco = {
  "type": "Point",
  "coordinates": [-122.4726194, 37.7577627]
};

// true
console.log(pointInPolygon(denver, colorado));

// false
console.log(pointInPolygon(sanFrancisco, colorado));

Turfパッケージでは、Node.jsを使用して、ポイントがポリゴン内にあるかどうかを確認できます。しかし、データベースにクエリを実行して同じ情報を取得したい場合はどうでしょうか。この場合、組み込みのMongoDB ステートメント$geoIntersectsがGeoJSON をサポートしていることに注意してください。したがって、たとえば、マップ上の特定のポイントに対応する米国の州を見つけることができるクエリを作成できます。

const mongoose = require('mongoose');

run().catch(err => console.log(err));

async function run() {
  await mongoose.connect('mongodb://localhost:27017/geotest', {
    useNewUrlParser: true,
    useUnifiedTopology: true
  });
  await mongoose.connection.dropDatabase();

  const State = mongoose.model('State', mongoose.Schema({
    name: String,
    location: mongoose.Schema({
      type: String,
      coordinates: [[[Number]]]
    })
  }));

  const colorado = await State.create({
    name: 'Colorado',
    location: {
      "type": "Polygon",
      "coordinates": [[
        [-109, 41],
        [-102, 41],
        [-102, 37],
        [-109, 37],
        [-109, 41]
      ]]
    }
  });

    const denver = {
    "type": "Point",
    "coordinates": [-104.9951943, 39.7645187]
  };

  const sanFrancisco = {
    "type": "Point",
    "coordinates": [-122.4726194, 37.7577627]
  };

  //     ?
  let res = await State.findOne({
    location: {
      $geoIntersects: { $geometry: denver }
    }
  });
  res.name; // 

  //     -?
  res = await State.findOne({
    location: {
      $geoIntersects: { $geometry: sanFrancisco }
    }
  });
  res; // null
}

概要


GeoJSONは、点の座標の格納だけではありません。この形式でパスを保存できます。 GeoJSONデータを使用して、ユーザーがいつジオフェンスに入ったかを確認できます。そして、必要であれば、その後にGeoJSONも、あなたが作成することができます等時を。 GeoJSONフォーマットを中心に、一連の優れたツールが形成されました。したがって、リソースgeojson.ioを使用すると、地図上の座標を簡単に視覚化できます。Mapboxプロジェクトは、高度な地理APIへのアクセスを提供します。Turfパッケージを使用すると、ブラウザーおよびNode.jsで地理空間コンピューティングを実行できます。

MongoDBは、地理データに関連するクエリをサポートしています。また、GeoJSON形式を使用せずに、ポイントの地理座標を値のペアとして保存すると、優れた開発ツールを使用する機会を逃していることになります。
