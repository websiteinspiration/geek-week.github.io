これらの5つの原則でCSSを改善してください。



CSSの記述は、かなり単純で簡単なプロセスですが、なぜいくつかの原則とベストプラクティスが必要なのでしょうか。

プロジェクトの規模とそれに取り組む人々の数が増加するにつれて、将来深刻な問題を引き起こす可能性のある新しい困難がますます明らかに現れ始めます。コードの複製、複雑なプロパティの再定義チェーン、using！リモートHTML要素からの重要な残りの未使用のCSSプロパティなど。このようなコードは、読み取りや修正がより困難です。

CSSを専門的な方法で作成すると、コードの保守性、拡張性、理解性、およびクリーンさが向上します。CSSを次のレベルに引き上げる5つのシンプルで非常に効果的な原則を見てみましょう。


命名原則


「コンピューターサイエンスでは、キャッシュの無効化と命名の2つの複雑なことしかありません」-Phil Carleton

CSSセレクターに適切な名前を付けて構造化することは、CSSをより読みやすく、構造化し、クリーンにするための最初のステップです。命名規則にルールと制限を定義すると、コードが標準化され、信頼性が高く、理解しやすくなります。

これが、BEM（Block-Element-Modifier）、SMACSS（CSSのスケーラブルおよびモジュラーアーキテクチャ）、OOCSS（オブジェクト指向CSS）などの概念が多くのフロントエンド開発者に人気がある理由です。

低特異性の原則


CSSプロパティのオーバーライドは非常に便利ですが、より複雑なプロジェクトでは、かなり手早く手に負えなくなる可能性があります。チェーンのオーバーライドは非常に長く複雑になる可能!important性があり、コードのデバッグまたは新しい関数の追加時に簡単に失われる可能性がある特定性の問題を使用する必要があります

/*     */
.card {}
 
/*     */
.card .title {}
 
.blog-list .card img {}
 
.blog-list .card.featured .title {}
 
#js-blog-list .blog-list .card img {}

ブラウザーと特異性


特異度が低いという原則に従う利点の1つはパフォーマンスです。ブラウザはCSS を右から左に解析します。

次の例を見てみましょう。

.blog-list .card img {}

ブラウザは次のようにセレクタを「読み取り」ます。

  1. ページ上のすべての<img>タグを検索します
  2. それらから、クラス.cardの子孫である要素を選択します
  3. それらから、.blog-listクラスの子孫である要素を選択します

特にdiv、img、liなどの共通要素をグローバルに選択する必要がある場合に、高い特異性を持つセレクターがパフォーマンスにどのように影響するかを確認できます。

同じレベルの特異性を使用する


固有性の低いCSSセレクターを、BEM方法論または前のセクションで説明した他の命名原則のいずれかと組み合わせて使用​​することで、強力で柔軟性があり、理解しやすいコードを作成できます。

CSSクラスを使用する理由 柔軟性を維持し、複数の要素を選択できるようにしながら、同じレベルの特異性を維持したいと考えています。要素セレクターとIDセレクターは、必要な柔軟性を提供しません。

前の例をBEMを使用して書き直し、特異度を低く保つことにしましょう。

/*     */
.card {}
 
/*       */
.card__title {}
 
.blogList__image {}
 
.blogList__title--featured {}
 
.blogList__img--special {}

これらのセレクターがよりシンプルで明確であり、必要に応じて簡単に再定義および拡張できることがわかります。また、それらの特異度を低く（単一クラス）維持して、最適なパフォーマンスと柔軟性を保証します。

乾燥原理


DRYの原則（自分を繰り返さない、自分を繰り返さない）もCSSに適用できます。CSSの重複は、コードの膨張、不必要なオーバーライド、不十分なサポートなどにつながる可能性があります。この問題は、適切なコード構造と品質のドキュメントで修正できます。

Storybookは、利用可能なWebインターフェイスコンポーネントの概要を作成し、高品質のドキュメントを作成できる優れた無料ツールです。

/*    DRY */
.warningStatus {
  padding: 0.5rem;
  font-weight: bold;
  color: #eba834;
}
 
.errorStatus {
  padding: 0.5rem;
  font-weight: bold;
  color: #eb3d34;
}
 
.form-errorStatus {
  padding: 0.5rem 0 0 0;
  font-weight: bold;
  color: #eb3d34;
}

DRY原則に従ってコードをリファクタリングしましょう

/*    DRY */
.status {
  padding: 0.5rem;
  font-weight: bold;
}
 
.status--warning {
  color: #eba834;
}
 
.status--error {
  color: #eb3d34;
}
 
.form__status {
  padding: 0.5rem 0 0 0;
}

単独責任の原則


CSSの唯一の責任原則を使用すると、CSSクラスを簡単に拡張して再定義することができます。次の例を見てみましょう

.button {
  padding: 1rem 2rem;
  font-size: 2rem;
  border-radius: 0.2rem;
  background-color: #eb4934;
  color: #fff;
  font-weight: bold;
}
 
.button--secondary {
  border-radius: 0;
  font-size: 1rem;
  background-color: #888;
}

.button-セカンダリ修飾子 を使用して.buttonクラスを展開する必要がある場合は、スタイルをデフォルトのままにして、背景色のみを変更したいものの、多くのオーバーライドを実行する必要があることがわかります。 問題は、.buttonクラスにいくつかの役割があることです。



  • ブロックモデルを定義します（パディング）
  • タイポグラフィを定義します（font-size、font-weight）
  • 外観を決定します（color、background-color、border-radius）

これにより、CSSクラスの拡張と他のクラスとの統合が複雑になります。それを念頭に置いて、BEMとOOCSSでCSSを改善しましょう。

/*   */
.button {
  padding: 1rem 2rem;
  font-weight: bold;
  color: #fff;
}
 
/*   */
.button--radialBorder {
  border-radius: 0.2rem;
}
 
.button--large {
  font-size: 2rem;
}
 
.button--primary {
  background-color: #eb4934;
}
 
.button--secondary {
  background-color: #888;
}

ボタン のスタイルを、基本クラスを拡張するために使用できるいくつかのクラスに分割しました。必要に応じて、モディファイアを適用し、デザインの変更や新しい要素の追加に応じて新しいモディファイヤを追加できます

オープン/クローズの原則


ソフトウェアエンティティ（クラス、モジュール、関数など）は、拡張のために開いている必要がありますが、変更のために閉じている必要があります

。すべての新機能とオプションは、拡張機能によって追加する必要があります。この例を見てみましょう。

.card {
  padding: 1rem;
}
 
.blog-list .card {
  padding: 0.5em 1rem;
}

.blogリスト.cardセレクタは、いくつかの潜在的な問題があります。

  • 一部のスタイルは、.card要素が.blog-list要素の子である場合にのみ適用できます
  • .blog-list要素に配置されている場合、スタイルは.card要素に強制的に適用されます。これにより、予期しない結果や不要なオーバーライドが発生する可能性があります

前の例を書き直してみましょう：


.card {
  padding: 1rem;
}
 
.blogList__card {
  padding: 0.5em 1rem;
}

単一のクラスセレクターを使用する場合、問題を修正します。このセレクターを使用すると、予期しない影響を回避でき、条件付きでネストされたスタイルを使用しません。

最終的に


これらのいくつかの単純な原則を適用することで、CSSの記述方法を大幅に改善できる例を検討しました。

  • 命名と構造の標準化、BEM、OCSSなどを使用した読みやすさの向上
  • 特異度の低いセレクターを使用したパフォーマンスと構造の改善
  • DRYの原理を使用して膨らみを減らし、コード品質を改善
  • オープン/クローズの原則を使用した柔軟性と保守性
