自分のインターネットラジオ

私たちの多くは、午前中にラジオを聞くのが好きです。そしてある晴れた朝、私は地元のFMラジオ局を聴きたくないことに気づきました。興味がない。しかし、その習慣は有害であることがわかりました。そして、FM受信機をインターネット受信機に置き換えることにしました。すぐにAliexpressの部品を購入し、インターネット受信機を組み立てました。

インターネット受信機について。レシーバーの心臓部はESP32マイクロコントローラーです。 KA-radioのファームウェア。部品のコストは私に12ドルかかりました。組み立てが簡単なため、数日で組み立てることができました。それはうまくそして安定して働きます。 10か月を超える作業で、数回ハングしただけでしたが、それは私の実験のためだけでした。便利で思いやりのあるインターフェースにより、スマートフォンやコンピューターから操作できます。つまり、これはすばらしいインターネット受信機です。

すべて大丈夫です。しかし、ある早朝、私は何万ものラジオ局にアクセスできるので、興味深い局はないという結論に達しました。私は広告、愚かなジョークが主導することに悩まされました。常にあるステーションから別のステーションにジャンプします。 SpotifyとYandex.Musicが好きです。しかし、悲しいことに、彼らは私の国では機能しません。そして、インターネットの受信機で聞いてみたいです。

子供の頃を思い出しました。私はテープレコーダーと20個のカセットを持っていました。カセットは友達と交換しました。そしてそれは素晴らしかった。オーディオアーカイブをインターネットレシーバーにのみストリーミングする必要があると判断しました。もちろん、オーディオプレーヤーまたはiPodをスピーカーに接続して、スチーム接続しないオプションもあります。しかし、これは私たちのやり方ではありません！コネクタの接続は大嫌いです）

既製のソリューションを探し始めました。 Radio-Tochka.comから独自のインターネットラジオを作成するという提案が市場に出ています。 5日間テストしました。すべてが私のインターネット受信機で完璧に動作しました。しかし、価格は私にとって魅力的ではありませんでした。このオプションを拒否しました。

私は10 GBの有料ホスティングを利用しています。私は、mp3ファイルのオーディオストリームをストリーミングするスクリプトを何かに書くことにしました。 PHPで書くことにしました。すぐに書いて立ち上げました。すべてがうまくいった。かっこいい！しかし、数日後、ホスティング行政から手紙が来ました。プロセッサの制限時間の超過と、より高い料金に切り替える必要性について話しました。スクリプトを削除して、このオプションを放棄する必要がありました。

どうやってそうなった？ラジオなしでは生きられない。他の誰かのホスティングでスクリプトを実行することが許可されていない場合は、独自のサーバーが必要です。私の魂が望むことをどこでやります。

バッテリーなしの古いネットブックを持っています（CPU-900 MHz、RAM-512 Mb）。その老人は11歳です。サーバーの場合は問題ありません。 Ubuntu 12.04を入れました。次に、Apache2とphp 5.3、sambaをインストールします。私のサーバーは準備ができています。

Icecastを試すことにしました。私はそれについてたくさんのマナを読みました。しかし、それは私には複雑に思えました。そして、PHPのスクリプトでオプションに戻ることにしました。このスクリプトのデバッグには数日かかりました。そして、それは完璧に機能しました。次に、ポッドキャストを再生するスクリプトも作成しました。そして、とても気に入ったので、小さなプロジェクトを作ることにしました。それをIWScastと呼んだ。githubに投稿しました。



すべてが非常に簡単です。 mp3ファイルとindex.phpファイルをApache / var / www /のルートフォルダーにコピーすると、ランダムに再生されます。一日で約300曲で十分です。
index.phpファイルはスクリプト自体です。スクリプトは、mp3ディレクトリファイルのすべての名前を配列に読み込みます。オーディオストリームを作成し、mp3ファイルの名前を置き換えます。曲を聴いて、気に入ったときがあります。誰がこれを歌うと思いますか？この場合、ログlog.txtに聴いているトラックの名前の記録があります。
完全なスクリプトコード
<?php
set_time_limit(0);
header('Content-type: audio/mpeg');
header("Content-Transfer-Encoding: binary");
header("Pragma: no-cache");
header("icy-br: 128 ");
header("icy-name: your name");
header("icy-description: your description"); 
$files = glob("*.mp3");
shuffle($files); //Random on

for ($x=0; $x < count($files);) {
  $filePath =  $files[$x++];
  $bitrate = 128;
  $strContext=stream_context_create(
   array(
     'http'=>array(
       'method' =>'GET',
       'header' => 'Icy-MetaData: 1',
       'header' =>"Accept-language: en\r\n"
       )
     )
   );
//Save to log 
  $fl = $filePath; 
  $log = date('Y-m-d H:i:s') . ' Song - ' . $fl;
  file_put_contents('log.txt', $log . PHP_EOL, FILE_APPEND);
  $fpOrigin=fopen($filePath, 'rb', false, $strContext);
  while(!feof($fpOrigin)){
   $buffer=fread($fpOrigin, 4096);
   echo $buffer;
   flush();
 }
 fclose($fpOrigin);
}
?>


トラックを順番に再生したい場合は、index.phpの行をコメント化する必要があります
shuffle($files); //Random on


ポッドキャストには/ var / www / podcast /を使用します。別のindex.phpスクリプトがあります。ポッドキャストトラックを記憶しています。次にインターネットレシーバーをオンにすると、次のポッドキャストトラックが再生されます。再生されたトラックのログもあります。
counter.datファイルでトラック番号を指定すると、そこからポッドキャスト再生が開始されます。

彼はポッドキャストを自動的にダウンロードするためのパーサーを書いた。彼はRSSから最後の4トラックを取得してダウンロードします。これらはすべて、ブラウザーのスマートフォン、IPTVセットトップボックスで適切に機能します。

最近の朝、トラックの再生位置を覚えておくといいなと思いました。しかし、PHPでこれを行う方法はまだわかりません。

スクリプトはgithub.com/iwsys/IWScastからダウンロードできます。
