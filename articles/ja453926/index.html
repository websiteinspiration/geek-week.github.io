クライアントとサーバーの相互作用


カーシェアリングサービスを利用する一般のユーザーは、特定のオペレーターが発行した特別なモバイルアプリケーションを使用して、フリートにアクセスします。ただし、ニュースの多くでは、さまざまなTelegramチャネルで偽のアカウントについて耳にします。以下のスクリーンショットは、誰でもアカウントを購入できるテレグラムチャネルの例です。



偽のアカウントを登録する原理は簡単です！登録に必要なデータ（パスポート、運転免許証）、攻撃者はさまざまなソースから取得します。「被害者」自身がデータを送信することもあります。データを受け取った後、攻撃者はアカウントを登録し、カーシェアリングオペレーターのセキュリティサービスがこのアカウントを承認して、売りに出します。そのようなアカウントの下で乗ることは刑事犯罪です。セキュリティサービスは、現在のほぼすべての旅行を積極的に監視しています。

攻撃者がアカウントを登録するために使用する2番目の方法は、ソフトウェアの脆弱性を使用することです。そのため、比較的最近、カーシェアリング事業者のTimCarのユーザーデータベースが盗まれました。そしてつい先日、首都の事業者の1つのアカウントにアクセスするために、SMS経由で送信される確認コードを入力する必要さえありませんでした。オペレーターのサーバー自体がリクエストへの応答としてそれをクライアントに送信しました：
{
    "success": 1,
    "return": {
        "veryfy_code": 3634,
        "sms_id": "201907-1000009"
    }
}

「veryfy_code」フィールドからコードを入力することにより、電話番号のみを知っていて、被害者のアカウントに簡単にログインできます。
