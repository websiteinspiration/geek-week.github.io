Visual C ++コンパイラの#lineディレクティブのエラー


#lineディレクティブはプリプロセッサによって追加され、前処理されたファイル内の特定のコードが属するファイルと行を理解するために使用できます。#lineディレクティブは、内部に保存されている行番号とコンパイラファイルの名前を指定された行番号に変更するようにコードツールに指示します。次の行は、指定された位置を基準としてすでにカウントされています。明示的な前処理は、主にデバッグまたはさまざまなジェネレーターによって使用されます。いずれの場合も、このような機能のエラーは、さまざまな悪影響をもたらす可能性があります。これらの問題の1つは、Visual Studio 2019のPVS-Studioのユーザーに発生しました。

エラー履歴と説明


#lineディレクティブは、PVS-Studioアナライザーがユーザーコードの必要な行に警告を出すために使用する前処理済みファイルの重要な部分です。ここからエラー検索のストーリーが始まります。

大規模なコードベースを持つユーザーの1人が、Visual Studio 2019でプラットフォームツールセットバージョンv141からv142へのプロジェクト全体の変換を開始しました。その後、分析結果で、V002警告が急増し、他の数十の警告が誤ったコード行（1行エラー）に対して発行されました）

Diagnostics V002は、このような状況（行番号の誤り）を検出するだけです。これは通常、複数行のマクロまたは他のプリプロセッサディレクティブ（#pragmaなど）が原因で発生します。この問題はそれほど頻繁には発生しませんが、主にMicrosoftのコンパイラーで発生します。1つのプロジェクトでのこのような問題の急増は予期せぬものでした。私たちのテストシステムはそのような深刻な問題の存在を示さなかったため、前処理されたユーザーファイルでアナライザーのデバッグを開始しました。

数時間のデバッグの後、アナライザーはコード行に関する誤ったデータをすでに処理しており、テスト例を作成できることがわかりました。

#define MACRO(a, b)              // line1
                                 // line2
int main(int argc, char* argv[]) // line3
{                                // line4
    MACRO("Some string",         // line5
        42);                     // line6
    return 0;                    // line7 <=
}

前処理されたファイルは次のとおりです。

#line 1 "D:\\_Tests\\TestApp\\Project2\\rnd_bs.cpp"


int main(int argc, char* argv[])
{
  ;
#line 8 "D:\\_Tests\\TestApp\\Project2\\rnd_bs.cpp"
  return 0;
}

return文は、上にある第七ソースファイルの行、およびへのプリプロセッサポイント8行。このエラーは、おそらく複数行のマクロが原因で、プリプロセッサで発生しました。

リリースバージョンリリース


執筆時点では、問題はVisual Studioの最新バージョンであるバージョン16.4.3（2020年1月14日）までに関連していました。バグレポート「前処理後の無効な#lineディレクティブ（VS2019、v142）」を開き、 4日後に興味深い回答を受け取りました。

この問題は、以前に報告されたフィードバックのリンクとしてここにリンクされています：developercommunity.visualstudio.com/content/problem /738407/preprocessor-generates-incorrect-line-information.html。ここで報告または投票した場合、投票は元の問題に適用されています。他の誰もが、上記のリンクされたフィードバックに直接投票を追加できます。投票は、顧客への影響を1つのフィードバックにまとめることで、問題の優先度を高めるのに役立ちます。ありがとうございました！

この問題はすでに報告されており、2019年9月18日 -ほぼ半年前に、「プリプロセッサが誤った行情報を生成する」ことが判明しました。ユーザーは、エラーを再現するために同様のコード例を提供しました。著者によると、エラーは少なくともVisual Studio 2019 16.2に存在します。

これが偶然であるかどうかはわかりませんが、数日間のアクティビティの後、エラーには「修正済み-保留中のリリース」というステータスが割り当てられました。

2020年5月19日、Visual Studio 2019 16.6.0の改訂版がリリースされました。 PVS-Studioのすべてのユーザーに切り替えることをお勧めします。
