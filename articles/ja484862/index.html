ハイブリッドアプリによるAngularJSからAngular7への移行



タスクは最も簡単ではありませんが、実行可能です。

新しい会社に引っ越すときに彼女に出会いました。

それが実行可能であるという事実は、マニュアルで見つけることができます。

これは、新しいプロジェクトを作成して各コンポーネントを最初から書き直す必要なしに、移行を単純化するために特別に作成されました。

開発者が考えたように、私たちのプロジェクトは、AngularJSとAngularの両方で新しいバージョンのコードを徐々に置き換えて作業を開始する必要があります。

だから私はそれをすることにしました。

このガイドはかなり詳細ですが、基本的には理論的な部分が含まれています。

Vueを主に扱っていた人にとって、フレームワークの編成の複雑さをすぐに理解することはそれほど簡単ではありません。

私にとっての主な困難は、AngularJSとAngular7のプロジェクトの構造が異なることでした。

したがって、すべての依存関係、サービス、およびコンポーネントを手動で書き直す必要があります。

実際、アプリケーションを完全に書き換える必要があるため、このアプローチとのハイブリッド性は意味を失います。

したがって、私は最初に、より安価な方法を見つけることはしませんでした。

ただし、より良いオプションがあります。

ここで私はたくさんの便利なものを見つけました。

私がやったことを詳しく説明します。

新しいプロジェクトを作成する必要があります。

Node.jsの場合：

npm install -g @angular/cli

Angularをグローバルにインストールします。

次に、アプリケーション用のフォルダーを作成し、Node.jsを介してそのフォルダーに移動する必要があります。

目的のフォルダーを見つけるため、

cd _ およびcd .. 上記のディレクトリに戻るためにここで必要となる可能性がある主なコマンド：

したがって、プロジェクトフォルダーが見つかったら、その中にプロジェクトを作成する必要があります。

Node.jsで次のように記述します。

ng new _

プロジェクトを作成するプロセスでは、目的のパラメーターを指定する必要があるアプリケーションのオプションがいくつか提供されます。

アプリケーションが作成されたら、次のコマンドを使用してブラウザーで開くことができます。

ng serve --open

ただし、まだ開くものはありません。

次に、AngularJSとAngularの両方が機能するために必要な依存関係をインストールする必要があります。

npm install --save @angular/upgrade

これから、アプリケーションは両方のフレームワークをサポートします。

ここで古いプロジェクトを開き、package.jsonファイルを探します。

アプリケーションの操作に必要なすべての依存関係をリストします。

手動でインストールする必要があります。

例：

npm install --save angular @uirouter/angularjs

この手順の後、すべてのグローバルスタイルを新しいアプリケーションに転送できます。

次に、アプリからすべてのファイルを転送します。

上記のガイダンスについては、さらに進んでください。

ルートコンポーネントの名前を変更します。

app.module.tsに次の行を追加します。

import * as angular from 'angular';

これはマニュアルには記載されていませんが、それなしではハイブリダイゼーションは達成できません。

これで、理論的にはハイブリッドになるはずのアプリケーションができました。

実際には、主に依存関係に関連するさまざまなエラーがこの段階で発生する可能性があります。これについては、別の記事で説明します。

表示されない場合は、AngularJSモジュールの後のモジュールを目的のバージョンのAngularに変更してください。
