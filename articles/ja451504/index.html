CSS


ほとんどの場合、SCSS、LESS、PostCSS、StylusなどのCSSプリプロセッサのいずれかを既に使用しており、これらにより作業が大幅に簡略化されます。SCSSを使用してApple Webサイトのコードを最適化してみましょう。メディア表現については、メディアミックスインを取り、写真のミックスインを記述します。

@mixin image($width, $height, $url) {
  width: $width;
  height: $height;
  background-size: $width $height;
  background-repeat:no-repeat;
  background-image:url($url);
}

変数を設定しましょう：

$breakpoints: (
  'phone': 735px,
  'tablet': 1068px
);

$media-expressions: (
  'screen': 'only screen',
  'retina': '(-webkit-min-device-pixel-ratio: 1.5), (min-resolution: 1.5dppx), (min-resolution: 144dpi)'
);

AppleのCSSで1対1でコンパイルするSCSSを取得します。

ティルダ：ぼかし効果のある背景画像


Tildaは、画像を表示するためのさまざまなコンポーネントを提供するサイトビルダーです。画像を「遅延」アップロードする最も簡単な方法を検討してください。画像をロードする前のhtmlは次のようになります。

<div data-original="https://static.tildacdn.com/image.jpg" style="background: rgba(0, 0, 0, 0) url(https://static.tildacdn.com/image-small.jpg) no-repeat scroll center center / cover;">
</div>



プレビューのサイズが小さい（20x20ピクセル）ためにぼかし効果が得られますが、カバーのcssプロパティを使用すると、画像は幅と高さ全体に引き伸ばされます。マイナス：このぼやけた見栄えはあまりよくありませんが、非常にシンプルで、プロセッサに過負荷をかけません。

ロード後、フルサイズの画像はインラインスタイルに置き換えられます。

最初に、Appleの方法について説明しましたが、cssには画面サイズとピクセル密度の両方のメディア表現があることがわかりました。また、同時に使用して、表示する特定の画像を選択することもできます。重要な欠点は、背景の画像がそのアスペクト比を「認識」しておらず、「ゴム」にはできないことだけです。さらに、Webのセマンティクスの観点から、すべての画像を連続して表示するために背景を使用することは正しくありません。

imgタグにはsrcset属性を含めることができますが、画面サイズまたはピクセル密度を選択する必要があります。これらのパラメーターを組み合わせることはできません。

画像タグにはこの欠点がなく、imgと同様に、アスペクト比を「認識」しています。すでに92％でサポートされていますブラウザ。このタグは、IEに適応しない余裕がある人に適しています。

交差オブザーバーAPI


最後に、遅延読み込みイメージの実装について話しましょう。古典的なアプローチは、スクロール、サイズ変更、およびオリエンテーションチェンジのイベントを組み合わせて使用​​し、画面の表示領域に対する特定の画像の位置を計算することです。

2019年には、松葉杖のセットを特別なブラウザーAPI- 交差オブザーバーに置き換えることができます。最新のすべてのブラウザでサポートされています。古いiOSとIEでは、公式のw3c polyfillを使用する必要があります。

未来への展望：完全な幸福のために他に欠けているもの


そのため、ブラウザーに十分対応している画像タグがすでにありますが、遅延読み込みの問題が残っています。それでも、交差オブザーバーまたはJSイベントの組み合わせに基づく独自の実装に依存する必要があります。ブラウザはいつネイティブソリューションを提供しますか？たとえば、Chromeバージョン75は、imgおよびiframeタグの読み込みプロパティのサポートを提供することを約束します。画像のプロトタイプでプロパティの可用性を確認することにより、コンポーネントを更新して遅延読み込みをネイティブにサポートできます。成功した場合は、ブラウザーによってすべてが行われるため、独自の遅延読み込みコードを放棄できます。

CMSとユーザーコンテンツ


ユーザーがアップロードした画像を表示する最良の方法は何ですか？原則として、携帯電話、タブレット、デスクトップ用に別々に写真をアップロードすることを強制することなく、すべてをできるだけ簡単に行う価値があります。したがって、すべてのデバイスに対して画像が1つしかありません。このような状況では、Tildaのアプローチ（シンプルさを最大にするため）またはMediumが最適です。実際のぼかし効果を作りたい場合は、おそらく、stackblur-canvasを標準のcss：blur（）に置き換えて、パフォーマンスを向上させます。

概要


画像を表示するための単一の「最も適切な」方法はまだありません。古いブラウザとIEをサポートする必要がある場合は、メディア式と単純なケースのimgを含む背景画像を選択します。新しいプロジェクトでは、背景画像を画像に置き換える方が便利な場合があります。美しい遅延読み込みを行いたい場合は、stackblur-canvasを試してください。パフォーマンスが重要な場合は、チルダスタイルでCSSフィルターまたは背景の拡大画像を使用してください。
