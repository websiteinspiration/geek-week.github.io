コードから、5つのレベルを通過する必要があることがわかります。順番に見ていきます。



最初のレベルでは、プログラム引数の数が100であることを確認します。この場合、65番目の要素は文字列「\ x00」、66番目の要素は「\ x20 \ x0a \ x0d」である必要があります。/ tmp /ディレクトリにディレクトリを作成し、そこにPythonスクリプトを作成します。



最初のレベルを解くために、100行の「A」の配列を作成します。そして、必要な引数を目的の値に割り当てます。次のように、これらの引数を使用してプロセスを開始できます。

from pwn import *

a = ['A']*100
a[0] = '/home/input2/input'
a[ord('A')] = '\x00'
a[ord('B')] = '\x20\x0a\x0d'

ex = process(argv=a)

ex.interactive()



最初のレベルを通過しました。2番目を見てください。



このレベルでは、2つの行が読み取られます。1つは標準入力stdin、もう1つはstderrから読み取られます。これらの行を含む2つのファイルを作成する必要があります。



これらのファイルへのストリームを開き、1つのファイルのストリームの記述子を記述子stdinとして指定し、別のファイルを記述子stderrとして指定します。

from pwn import *

a = ['A']*100
a[0] = '/home/input2/input'
a[ord('A')] = '\x00'
a[ord('B')] = '\x20\x0a\x0d'

fin = open('/tmp/ex/in.txt', 'r')
ferr = open('/tmp/ex/err.txt', 'r')

ex = process(argv=a, stdin=fin, stderr=ferr)

fin.close()
ferr.close()

ex.interactive()



3段階目の決定に進みます。



getenv（）関数は、環境変数の値を返します。これは、参照値と等しい必要があります。したがって、特定の値を持つ環境変数を作成する必要があります。

from pwn import *

a = ['A']*100

a[0] = '/home/input2/input'
a[ord('A')] = '\x00'
a[ord('B')] = '\x20\x0a\x0d'

fin = open('/tmp/ex/in.txt', 'r')
ferr = open('/tmp/ex/err.txt', 'r')

e={'\xde\xad\xbe\xef':'\xca\xfe\xba\xbe'}

ex = process(argv=a, stdin=fin, stderr=ferr, env=e)

fin.close()
ferr.close()

ex.interactive()



彼らは環境変数を管理していたので、今度は第4レベルから始めましょう。



コードによると、プログラムは「\ x0a」という名前のファイルを開いて4文字を読み取り、その後バイト「\ x00」と比較します。文字自体とファイル名の両方が印刷できない文字で構成されているため、Pythonを使用します。





最後のレベルは残ります。始めましょう。



プログラムは、66番目の引数で指定されたポートでソケットを開きます。次に、ネットワーク経由で4バイトを受信し、それを参照文字列と比較します。プログラムに別の引数を追加する必要があります-ポート番号、接続を確立し、必要な4バイトを送信します。

from pwn import *

a = ['A']*100
a[0] = '/home/input2/input'
a[ord('A')] = '\x00'
a[ord('B')] = '\x20\x0a\x0d'
a[ord('C')] = '1234'

fin = open('/tmp/ex/in.txt', 'r')
ferr = open('/tmp/ex/err.txt', 'r')

e={'\xde\xad\xbe\xef':'\xca\xfe\xba\xbe'}

ex = process(argv=a, stdin=fin, stderr=ferr, env=e)

fin.close()
ferr.close()

ex.interactive()
