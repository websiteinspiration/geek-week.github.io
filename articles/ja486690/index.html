高品質の矢印関数を作成するための5つのヒント

JavaScriptアロー関数はかなり人気があります。簡潔な構文、字句バインディングthis、およびコールバックとして使用するのに非常に便利であるという点で異なります。 本日翻訳する資料には、矢印関数の使用に関する5つの推奨事項が含まれています。これらの推奨事項を利用する人は、そのような機能の可能性をより完全に明らかにする機会があります。





1.矢印関数名の表示を使用する


JavaScriptの矢印関数は匿名です。それらのプロパティnameには空の文字列-が含まれています''。

( number => number + 1 ).name; // => ''

匿名関数はanonymous、デバッグ中またはコールスタックの分析中にマークされます。残念ながら、この単語にanonymousは、関連する特定のコードに関するヒントは含まれていません。

これは、匿名関数が呼び出されるコードのデバッグのポイントの1つです。


無名関数を含むコードのデバッグ

図の右側の呼び出しスタックには、とマークされた2つの関数への参照が含まれていanonymousます。コールスタックに含まれているそのような情報からは、コードについて有用なことを学ぶことはできません。

幸い、JavaScriptには関数名を導出するメカニズム（ES2015機能）があり、特定の条件下で関数名を検出できます。名前を表示するアイデアは、JavaScriptが、その構文上の位置から、つまり、関数オブジェクトへのリンクを格納する変数の名前に基づいて、矢印関数の名前を理解できるということです。

実際に関数名を導出するメカニズムを見てみましょう。

const increaseNumber = number => number + 1;

increaseNumber.name; // => 'increaseNumber'

アロー関数への参照は変数increaseNumberに格納されるため、JavaScriptは名前が'increaseNumber'関数に適していると判断します。その結果、アロー関数もそのような名前を取得します。

矢印関数の命名には、関数名の派生メカニズムを使用することをお勧めします。

次に、関数名を取得するメカニズムが使用されているデバッグプロセスを見てみましょう。

ため、コールスタックは実行中のコードに関するより有用な情報を提供します。

  • 関数の名前は、handleButtonClickイベントハンドラの呼び出しを示しますclick。
  • 名前increaseCounterは、カウンターをインクリメントする関数の呼び出しを示します。

2.可能な限り、インライン関数を作成するように努めます


「インライン」関数は、単一の式で構成される関数です。アロー関数では、短いインライン関数を作成するのに非常に便利であるという点が気に入っています。

たとえば、以下はアロー関数の「完全な」形式です。

const array = [1, 2, 3];

array.map((number) => { 
  return number * 2;
});

この機能は簡単にコンパクトにすることができます。つまり、関数には式が1つしか含まれていないため、中括弧とreturn：

const array = [1, 2, 3];

array.map(number => number * 2);

関数に式が1つだけ含まれている場合は、インラインにすることをお勧めします。

3.矢印関数と比較演算子を慎重に使用します。


比較演算子は>、<、<=および>=矢印に非常に類似している=>（また、「太い矢印」と呼ばれるような矢印）機能を矢印宣言れます。

インライン関数で比較演算子を使用すると、結果のコードがあまり明確にならない場合があります。

演算子を使用する矢印関数を宣言します<=。

const negativeToZero = number => number <= 0 ? 0 : number;

1行の文字の存在は、コードリーダー=>を<=混乱させます。

矢印と比較演算子を明確に区別するために、いくつかの方法を使用できます。

最初に、式を括弧で囲むことができます。

const negativeToZero = number => (number <= 0 ? 0 : number);

次に、より長い構造を使用して、矢印関数を意図的に宣言できます。

const negativeToZero = number => {
  return number <= 0 ? 0 : number;
};

これらの変換により、1行での矢印記号と比較演算子の両方の使用によって引き起こされる不確実性が取り除かれます。

1ライン機能アン矢印がステートメントが含まれている場合にすることが推奨され>、<、<=そして>=、括弧内の対応する表現を締結または広告が機能を矢印マルチフォームを使用します。

4.矢印関数で単純なオブジェクトを作成する場合は、括弧または複数行構造を使用します


埋め込まれた矢印関数内でオブジェクトリテラルを使用すると、構文エラーが発生します。

const array = [1, 2, 3];

//  SyntaxError!
array.map(number => { 'number': number });

JavaScriptは中括弧をオブジェクトのリテラルではなくコードのブロックと見なします。

オブジェクトリテラルを括弧で囲むと、この問題は解決されます。

const array = [1, 2, 3];

// !
array.map(number => ({ 'number': number }));

オブジェクトリテラルに多くのプロパティがある場合は、ここで改行を使用することもできます。アロー関数はまだ埋め込まれたままです：

const array = [1, 2, 3];

// !
array.map(number => ({
  'number': number
  'propA': 'value A',
  'propB': 'value B'
}));

オブジェクトリテラルをインライン関数で使用する場合は、括弧で囲むことをお勧めします。

5.ネストが深くなりすぎないように注意してください。


アロー関数は簡潔な構文が異なります。それは良いです。しかし、これが原因で、相互に埋め込まれた多くのアロー関数は、読みにくい構造を形成する可能性があります。

次のシナリオを考えます。ボタンをクリックすると、サーバーへのリクエストが実行されます。特定の要素のセットを含むサーバー応答を受信すると、これらの要素の名前がコンソールに表示されます。

myButton.addEventListener('click', () => {
  fetch('/items.json')
    .then(response => response.json())
    .then(json => {
      json.forEach(item => {
        console.log(item.name);
      });
    });
});

ここでは、3つのレベルのネスト矢印関数が表示されています。このコードで何が起こっているのかを詳しく調べるには、少し時間をかける必要があります。

ネストされた関数の可読性を向上させるために、いくつかのアプローチを使用できます。

最初のアプローチは、変数への関数参照を記述することです。変数名は、関数の本質を簡潔に説明する必要があります（関数名の派生に関する推奨事項1をご覧ください）。

const readItemsJson = json => {
  json.forEach(item => console.log(item.name));
};

const handleButtonClick = () => {
  fetch('/items.json')
    .then(response => response.json());
    .then(readItemsJson);
};

myButton.addEventListener('click', handleButtonClick);

リファクタリング中に、ネストされた矢印関数を抽出し、それらをreadItemsJsonand 変数に書き込みましたhandleButtonClick。コードのネストレベルが3から2に減少しました。コードがより理解しやすくなりました。

このコードをリファクタリングする別のオプションは、関数全体をフォーマットに変換することですasync/await。これは、入れ子関数の問題を解決する優れた方法です。

const handleButtonClick = async () => {
  const response = await fetch('/items.json');
  const json = await response.json();
  json.forEach(item => console.log(item.name));
};

myButton.addEventListener('click', handleButtonClick);

矢印関数の入れ子のレベルを深くしすぎないようにし、それらを個別の関数として変数に抽出するか、可能であれば構文を使用することをお勧めしますasync/await。

概要


JavaScriptアロー関数は匿名です。コードのデバッグをより生産的にするために、関数参照を格納する変数を使用することをお勧めします。これにより、JavaScriptで関数名を表示できます。

埋め込まれた矢印関数は、関数本体に式が1つだけ含まれている場合に役立ちます。

演算子>、<、<=および>=矢印などの=>ファンクションスイッチを宣言するときに使用されます。これらの演算子を埋め込み矢印関数の本体で使用する場合は、コード変換を検討する必要があります。

のようなオブジェクトリテラルの構文は{ prop: 'value' }、コードのブロックのようなものです{}。その結果、オブジェクトリテラルが埋め込み矢印関数内に配置される場合は、括弧で囲む必要があります。() => ({ prop: 'value' })。

ネストのレベルが高すぎると、コードを読む人を混乱させます。関数を抽出して変数に書き込むことにより、関数のネストレベルを下げることをお勧めします。コードのネストのレベルを下げるもう1つの方法は、構文を使用することasync/awaitです。
