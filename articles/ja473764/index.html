小包の特徴


Parcelで最も気に入っているのは、構成がまったく必要ないことです。そのとおりです。設定する必要はありません。この点で、Parcelはwebpackよりも優れており、設定は複数のファイルに分散され、コードの「シート」全体を表すことができます。 Webpackを設定する開発者は、他のプログラマーの構成ファイルから何かを取得した可能性があります。設定が他のプロジェクトから単純に完全にコピーされる可能性があります。構成の複雑さは開発者に依存することは明らかですが、小規模なプロジェクト用にWebpackをセットアップする場合でも、特定のプラグインセットを使用して特定のオプションをインストールする必要があります。

そのため、小包の使用は私にとってクーデターのようです。たとえば、開発者がSCSSまたはLESSを使用してプロジェクトのスタイルを設定することを計画している場合、不必要な動きなしに、適切なコードを記述するだけで済みます。最新のJavaScript機能を使用したいですか？その場合-何も考えずに、これらの機能を使用してコードを記述できます。開発用のサーバーが必要ですか？ Parcelを使用する人には、このようなサーバーがあります。実際、ここでは、パーセルの膨大な機能の氷山の一角をほんの少し触れただけです。

Parcelを使用すると、プログラマーは二次的なものに時間を費やすことなく、プロジェクトに取り掛かり、作業を開始できます。これは、バンドラーとして使用する主な利点です。さらに、Parcelはマルチコアプロセッサの機能を使用して、ファイルを処理し、プロジェクトを非常に迅速にビルドします。一方、webpackを含む他のバンドルは、低速で複雑なコード変換の実行が遅くなります。

パーセルは、他のツールと同様に、普遍的なツールではなく、いつでもどこでも等しく適用できます。しかし、パーセルが特に成功している状況があります。

Parcelを使用すると、機能するプロジェクトにすばやく入ることができることについてはすでに説明しました。これにより、厳しい時間枠での作業やプロトタイプの作成に最適です。時間のかかる状況、および開発者の目標が実用的なアプリケーションをできるだけ早く作成することである場合について話します。

これは、Parcelが複雑なアプリケーション、またはプログラマーの大規模なチームが参加するプロジェクトに適していないことを意味しません。パーセルは、このような条件でうまく機能します。ただし、大規模なプロジェクトではワークフローを手動で調整することでメリットが得られることに気付きました。

パイロットプロジェクトページ

Sassと少しのJavaScriptを使用する簡単なサイトを作成します。現在の曜日の情報と Unsplash Sourceからダウンロードしたランダムな画像をウェブサイトのページに表示します。

▍プロジェクトの基本構造


Parcelを使用する予定のこのプロジェクトでは、ファイルとフォルダーの特別な構造は必要ありません。特定のフレームワークを使用する必要はありません。プロジェクトの基本的な構造は3つのファイルで構成され、それらの名前はそれ自体が物語っています。これはindex.html、style.scssとindex.js。好きなように作成できます。たとえば、次のコマンドラインを使用します。

mkdir simple-site
cd simple-site
touch index.html && touch style.scss && touch index.js

index.htmlプロジェクトの機能のベースとなる ファイルに、小さなテンプレートコードとマークアップを追加します。

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta http-equiv="X-UA-Compatible" content="ie=edge">
  <link href="https://fonts.googleapis.com/css?family=Lato&display=swap" rel="stylesheet">
  <link rel="stylesheet" href="style.scss">
  <title>Parcel Tutorial</title>
</head>
<body>
  <div class="container">
    <h1>Today is:</h1>
    <span class="today"></span>
    <h2><font color="#3AC1EF">and the image of the day:</font></h2>
    <img align="center" src="https://source.unsplash.com/random/600x400" alt="unsplash random image">
</div>
<script src="index.js"></script>
</body>
</html>

お気づきかもしれませんが、ここではGoogle FontsからWebフォント（Lato）をダウンロードしています。ダウンロード可能なフォントの使用は任意です。ここでは、CSSファイルとJavaScriptファイルの接続、および曜日に関する情報を出力する場所とUnsplashからランダムな画像を出力する場所を提供するドキュメント本文のHTMLコードに注目する価値があります。実はこれでプロジェクトの基本構造の作業は完了です。
