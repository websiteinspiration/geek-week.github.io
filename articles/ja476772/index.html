NLPモデルとの相互作用のモードDeepPavlov


DeepPavlovは現在、NLPモデル（事前トレーニング済みとユーザー作成の両方）と対話する4つの方法をサポートしています。

  • RESTサーバー（riseapiモード）は、モデルを統合するための主要なツールであり、以前の記事 （ドキュメント）で詳細に説明されていました。
  • TCPまたはUNIXソケットサーバー（risesocketモード）-低レベルの統合（ドキュメント）が必要な場合。
  • テレグラムのボット（テレグラムモード）-デモモードでは、テレグラム（ドキュメント）を介してモデルを操作できます。
  • コマンドライン（対話モード）-デモおよびデバッグモードでは、コマンドラインを介してモデルと対話できます。コマンドラインを介した対話モードのモデルは、次のコマンドで初期化されます。
    python -m deeppavlov interact


<config_path> パラメーター（4つすべてのモードで必須）は、モデル構成ファイルへの絶対パスまたは拡張子なしの構成ファイルの名前のいずれかを取ることができます。後者の場合、モデル設定はライブラリに登録する必要があります。

DeepPavlovで提供されるすべてのモデルの構成は、ライブラリに登録されています。提供されているモデルのリストは、DeepPavlov ドキュメントの MODELSセクションにあります。それらの構成はここにあります。

GPUの使用


上記のいずれのモードでも、NLPモデルは初期化され、ニューラルネットワークに基づいています。これにより、コンピューティングリソースに対する要求が非常に高くなります。GPUを使用すると、モデルのパフォーマンスを向上させることができます。これを行うには、十分なビデオメモリ（実行しているモデルによって異なります）とサポートされているバージョンのCUDAフレームワークを備えたnVidiaグラフィックカードが必要です。ここでは、GPU での DeepPavlovモデルの起動に必要なすべての情報を見つけることができます。

ライブラリ設定ファイル


すべてのライブラリ設定は、3つのファイルに含まれています。

  • server_config.json -RESTとソケットサーバーの設定、およびテレグラムコネクタ
  • dialog_logger_config.json-モデルへのリクエストをログに記録するための設定
  • log_config.json-ライブラリのログ設定

デフォルトでは、構成ファイルは<deep_pavlov_root> / utils / settingsにあります。<deep_pavlov_root>は、DeepPavlovのインストールディレクトリです（通常、これはlib / python <XX> / site-packages / deeppavlov in virtual environment）です。コマンドを使用する

python -m deeppavlov.settings

設定ファイルがあるディレクトリへの正確なパスを見つけることができます。また、環境変数DP_SETTINGS_PATHで指定することにより、便利なディレクトリへのパスを設定することもできます。上記のコマンド（トレーニング済みのDeepPavlovモデルを備えたサーバー）を最初に実行した後、デフォルトディレクトリのファイルがDP_SETTINGS_PATHのディレクトリにコピーされます。チーム

python -m deeppavlov.settings -d

ファイルの上のデフォルトディレクトリからDP_SETTINGS_PATHに設定ファイルをコピーして、設定をリセットします。

DeepPavlovの設定から、次のことに注意する必要があります。

  • server_config.json、パラメータmodel_args_names：私たちが覚えて
    いる最後の記事から：
    -名前が付けられたDeepPavlov REST API引数。
    -DeepPavlovのすべてのモデルは、その構成の名前で識別されます。
    したがって、各モデルのデフォルトの引数名はその構成から取得されます。

    以下の記事でモデル構成の構造を詳しく分析しますが、REST APIの引数名は次のように再定義できることに注意してください。

    model_args_names: [“arg_1_name”, ..., “arg_n_name”]

    引数名のシーケンスは、モデル構成で引数が定義されているシーケンスに対応し、model_args_namesパラメーターの値としての空の文字列はデフォルト名に対応します。
  • log_config.json：
    ロガーは、個別に構成されているuvicornのログに使用されることに注意してください。ロギングPythonモジュールの構成構造については、こちらをご覧ください。

Dockerで事前トレーニング済みモデルを実行する


事前トレーニング済みのすべてのDeepPavlovモデルは、RESTサービスモードのDockerコンテナーで起動できます。詳細な手順は、DockerHubのリポジトリにあります：CPU- こちら、GPU- こちら。コンテナ内のAPIモデルは、前回の記事の説明と完全に一致しています。

Deeppavlovクラウド


DeepPavlovの事前トレーニング済みのNLPモデルを使いやすくするために、SaaSモードでの提供を開始しました。モデルを使用するには、サービスに登録し、個人アカウントのトークンセクションでトークンを取得する必要があります。APIドキュメントはInfoセクションにあります。1つのトークンの下で、モデルに最大1000のリクエストを送信できます。

現在、このサービスはAlphaバージョンで開始されており、その使用は無料で提供されています。さらに、モデルの提供のためのセットとフォーマットは、ユーザーの要求に応じて拡張されます。リクエストフォームは、デモページの下部にあります。

以下のモデルがDeepPavlov Cloudで利用可能になりました。

  • 名前付きエンティティの認識（多言語）-名前付きエンティティの認識;
  • 感情（RU）-テキストの調性の分類;
  • SQuAD（多言語）-このテキストのフラグメントとしてのテキストへの質問に対する回答。
