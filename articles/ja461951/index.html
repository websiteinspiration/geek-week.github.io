SAP HANAデータベースのバックアップとリカバリのためのVeeamプラグイン

今シーズン、Veeam開発者はSAP HANAサーバーとデータベースのバックアップとリカバリのためのソリューションを導入しました。私たちのブログの読者は新製品への関心を示しました-そしてここに私の同僚のClemens Zerbeからの有用な記事が到着しました。今日はそれをあなたと共有し、少し追加します。猫へようこそ。



図からわかるように、SAP HANAの完全なソリューションには次のものが含まれます。

  • Veeam Backup＆Replication-イメージレベルまたはファイルレベルでSAP HANAサーバーをバックアップし、リカバリをサポートします。

    サーバーが仮想化されている場合、最高のバックアップにはVeeam Backup＆Replicationが推奨されます。物理サーバーにはVeeam Agent for Linuxがあります。どちらにもかなり詳細なドキュメントと、こちらのブログの記事があります。
  • Veeam — SAP HANA .

    SAP Backint ( API) Veeam ( — SOBR). .


上記のように、プラグインはSAP Backint APIを使用して、SAP HANAデータベースを直接操作します。 HANAは、独自のバックアップディレクトリを維持し、カスタム保持ポリシーと作成間隔を設定します。したがって、Veeam Backup＆Replicationは、データパイプからデータを取得してリポジトリに配置するだけです。
リカバリ中、SAP HANAはVeeam Backup＆Replicationに復元するデータを通知し、Veeamはそれをリポジトリから引き出します。

データ転送操作は、トランスポートエージェントであるVeeamトランスポートエージェントコンポーネントを使用して実行されます。これは、展開を計画するときに留意する必要があります。

注： SAP HANA Backintはデータベースでのみ機能し、さまざまなタイプのバックアップ（完全、差分、増分）とログを実行することに注意してください。彼（Backint）は回復にも使用されます。さらに、実際のデータに加えて、SAP HANAのインストールファイルと構成ファイルもあり、これらはすべてRed HatまたはSUSEオペレーティングシステムで実行されます。これも忘れてはならない。したがって、サーバー自体またはファイルのバックアップが必要な場合は、別のツールキットが必要になります。

はい、プラグイン自体はEnterprise PlusエディションのVeeam Backup＆Replication 9.5 Update 4（または4a）に含まれています。

インストールを開始


もちろん、最初に、システム要件への準拠を確認します。サポートされるOS：

  • SAPアプリケーション用のSUSE Linux Enterprise Server 12 SP1 / SP2 / SP3 / SP4（x86_64）
  • SUSE Linux Enterprise Server for SAP Applications 15（x86_64）
  • SAPアプリケーション用Red Hat Enterprise Linux 7.2 / 7.3 / 7.4 / 7.5（x86_64）

SAP HANA 2.0 SPS 02 / SPS 03 / SPS 04 はサポートされていますが、Express Editionはサポートされていません。

プラグインのテストインストールを実行し、データベースをバックアップおよび復元するには、次のものが必要です。

  • 構成済みのVeeamバックアップリポジトリ（暗号化なし！）とそれに必要なアクセス権。
  • VeeamリポジトリとHANAシステムの場合、DNS解決がサポートされている必要があります（フォワードとリバースの両方）。
  • x86で動作するHANAシステムバージョン2.0 SPS02以降。
  • 友人のサポート-Veeam管理者の場合は、SAP Basisでの作業経験を持つ同僚がいることをお勧めします。逆も同様です。

インストールファイルは、Veeam Backup＆Replication ISOの/ Plugins / SAP HANA / x64ディレクトリに格納されています。

  1. ISOをマウントし、ディレクトリ/ Plugins / SAP HANA / x64に移動します。目的のファイル（.rpmまたはtar.gz）をSAP HANAサーバーにコピーします。
  2. たとえば、このサーバーで適切なコマンドを実行します。

    sudo rpm -i VeeamPluginforSAPHANA-X.X.X.XXXX.x86_64.rpm

    注： Puttyまたはその他のsshクライアントを使用して、システムに接続できます。インストールするには、sudo権限を持つアカウントが必要です。
  3. インストールが完了し、SapBackintConfigTool --wizard構成ウィザードを起動するための招待を待機しています。

便利：実際、インストールはとても陽気ですが、Veeam Backup＆Replication 9.5 Update 4aを使用しているさらに高速のファンのために、HANAにそのような特別なパフォーマンスパッチがあります。
