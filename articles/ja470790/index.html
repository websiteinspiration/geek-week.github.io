Yandex.Interviewタスクを関数形式で解決します

数ヶ月前、記事は Yandexののブログに登場しインタビューのアルゴリズムの部分の通過を議論しました。とりわけ、この記事では、Yandexが候補者に提供したものと同様のタスクを含む特別なコンテストへのリンクが作成されました。


システムに登録すると、Haskellの問題を解決する機能にすぐに注意を向けました。事実、私はこの言語でのプログラミングが好きですが、教育用オンラインプラットフォームのさまざまなコースからのタスクの実装ほど進んでいません。彼らの解決策は興味深い挑戦になる可能性があり、開発者としてのレベルが上がると判断したので、私はそれらの解決に進みました。


誰が最終的にそれから来たのか気にしています、猫へようこそ。


A.石とジュエリー


小文字のラテン文字の2行が与えられます：文字列Jと文字列S。文字列Jに含まれる文字は「宝石」であり、文字列Sに含まれる文字は「石」です。Sから同時にいくつの文字が「宝石」であるかを判別する必要があります。簡単に言えば、Sの文字がJにいくつあるかを確認する必要があります。

最初のタスクはウォームアップです。「額」で解決します。関数jewelleryCount :: String-> String-> Intを定義します。これは、2番目の引数によって渡されたリストの畳み込みを使用して、最初のリストの処理済みアイテムのすべてのケースを合計します。これらの目的のために、elemInt関数をelem関数に基づいて定義します。これは、後者とは異なり、TrueまたはFalseではなく、0または1を返します。メイン関数では、2行を読み取るだけで、対応する関数に渡して結果を出力します。テストシステムの判定はOKです。次のタスクに進みます。


jeweleryCount :: String -> String -> Int
jeweleryCount j = foldr ((+).(elemInt j)) 0
                where elemInt s x = fromEnum $ elem x s

main :: IO ()
main = do
    j <- getLine
    s <- getLine
    print $ jeweleryCount j s
