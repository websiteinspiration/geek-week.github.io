サイト上でアダプティブコンテンツの読み込みを行います

こんにちは、ハブル！

プロジェクトのウェブサイトを作成しています。このサイトでは、同時に多数のgifを表示する必要があります。それぞれの重量はかなり大きいです。すべてを一度に表示すると、ページが非常に長時間ロードされます。同時に、GIFなしでページをダウンロードすることもできます（ダウンロードできるようにするため）。

私がこの問題にどう対処したかに興味がある人なら誰でも猫をお願いします。

実際には、問題


先ほど述べたように、サイトには非常に多くのGIFがあり、見栄えがよく、異なっています。あなたがそこにあるどのようにそれらの多くは自分で見ることができます：reface.techを



できるだけ早く私は着陸のベータ版を見たように、ロードに問題があった：一部のユーザーのためのページが長い時間のためにロードしました。深刻なビジネスの場合、もちろんこれは当てはまりません。

これをどうにかして解決する必要がありました。すでにその時点ですべてのメディアコンテンツが既にcompresser.ioを介して駆動されていたため、単純な可逆圧縮では問題を解決できませんでした。したがって、損失を伴う圧縮を行い、品質の悪いGIFを作成する必要がありました。

しかし、インターネットが良好なユーザーに悪いメディアコンテンツを配布することは、一種の冒涜です。したがって、これらのインターネットの速度を、どういうわけか、適切な品質を実現する速度に応じて決定することが決定されました。

ロードするメディアコンテンツの説明を含む配列があります。

例：

ここで、適切な品質の基本要素には、URLとファイルサイズがあります。

私たちは単に：

  1. 配列を調べます（ページをロードするとき）
  2. 品質の程度ごとに読み込まれた合計サイズを読み取る
  3. すべてが4秒でロードされるかどうかを判断します（これは私が個人的に気になる期間です）

したがって、次のことが必要です。

  1. すべてがペンで数えられないようにそのような配列を作るある種のものを見ました（Pythonで）
  2. インターネット速度の決定要因を見た

スピードゲージを切る


比較的簡単です。ブラウザのアドレスバーにeu.httpbin.org/stream-bytes/51200と入力します。51200バイトの長さのファイルがダウンロードされます。私たちは（ホスティングへの速度を測定するために）それを公共の場で自分に押し込みました。

次に、ファイルがどの程度変動しているかを確認する必要があります。このための単純な関数を提出します。これにより、速度がメガバイト/秒で返されます。

async checkDownloadSpeed(baseUrl, fileSizeInBytes) {
        return new Promise((resolve, _) => {
            let startTime = new Date().getTime();

            return axios.get(baseUrl).then( response => {
                const endTime = new Date().getTime();
                const duration = (endTime - startTime) / 1000;

                const bytesPerSecond = (fileSizeInBytes / duration);
                const megabytesPerSecond = (bytesPerSecond / 1000 / 1000);

                resolve(megabytesPerSecond);
            });
        }).catch(error => {
            throw new Error(error);
        });
}

つまり、ダウンロードの開始時刻と終了時刻を記録し、差を測定し、ファイルサイズがわかっているため、単純に分割しました。

次に、この特定の速度で何かを実行する関数を記述します。

async getNetworkDownloadSpeed() {
        const baseUrl = process.env.PUBLIC_URL + '/51200';
        const fileSize = 51200;

        const speed = await this.checkDownloadSpeed(baseUrl, fileSize);
        console.log("Network speed: " + speed);

        if (speed.mbps === "Infinity") {
            SpeedMeasure.speed = 1;
        }

        else {
            SpeedMeasure.speed = speed * 5;
        }
}

一般に、このコードには問題があります。ファイルサイズが小さいため、正確な接続速度が決定されないということです。しかし、それ以上のダウンロードはコストがかかりすぎるため、結果として得られる速度に5を掛けるだけです。実際には速度がさらに高くなるため、余裕を持ってそれを採用します。

次に、速度に応じて、対応する品質を達成する関数を提出します。

 static getResolution(gifsArray) {
        let totalSizeLevel1 = 0;
        let totalSizeLevel2 = 0;
        let totalSizeLevel3 = 0;

        for (let i = 0; i < gifsArray.length; i++) {
            for (let a = 0; a < gifsArray[i].length; a++) {
                let element = gifsArray[i][a];

                totalSizeLevel1 += element.small.size;
                totalSizeLevel2 += element.middle.size;
                totalSizeLevel3 += element.large.size;
            }
        }

        if (isNaN(SpeedMeasure.speed)) {
            SpeedMeasure.speed = 1;
        }

        let timeLevel1 = totalSizeLevel1 / SpeedMeasure.speed;
        let timeLevel2 = totalSizeLevel2 / SpeedMeasure.speed;
        let timeLevel3 = totalSizeLevel3 / SpeedMeasure.speed;

        if (timeLevel3 < APPROPRIATE_TIME_LIMIT) {
            return "large";
        }

        else if (timeLevel2 < APPROPRIATE_TIME_LIMIT) {
            return "middle";
        }

        else {
            return "small";
        }
}

速度をカウントする関数は非同期であるため、SpeedMeasure.speedはゼロになる場合があります。デフォルトでは、接続速度は1メガバイト/秒であると考えています。関数が速度を計算するとき、コンテナーを再レンダリングするだけです。

配列の配列をgetResolution関数に渡します。どうして？ページに複数のコンテナがある場合、対応するコンテンツを配列内のそれぞれに転送する方が便利ですが、一度にすべてのダウンロード速度を考慮する必要があります。

使用例


以下は使用例です（React）：

    async runFunction() {
        let speedMeasure = new SpeedMeasure();
        await speedMeasure.getNetworkDownloadSpeed();
        this.forceUpdate()
    }

    componentDidMount() {
        this.runFunction();
    }

    render() {
        let quality = SpeedMeasure.getResolution([
            Control.getControlArray(),
            Health.getHealthArray()

            <div className="app">
                <Presentation />
                <Control quality={quality} />
                <Health quality={quality} />
            </div>
        );
}

したがって、速度決定が読み込まれると、コンテナは再びレンダリングされます。
コンテナー内（たとえば、Control内）では、対応するGIFを（インデックスによって）配列から取得し、品質によってオブジェクトを取得し、URLによって参照を取得します。すべてが簡単です。

Pythonスクリプトの作成


次に、gifを何らかの方法で自動的に圧縮し、コンテンツの説明を含む配列を出力する必要があります。

はじめに、圧縮用のスクリプトを作成しましょう。gifsicleを使用します。中間圧縮コンテンツの場合、圧縮レベルは80（200のうち）になり、非常に強力な機能の場合は160になります。

import os

GIFS_DIR = "/home/mixeden//Landingv2/"
COMPRESSOR_DIR = "/home/mixeden//gifsicle-static"

NOT_OPTIMIZED_DIR = "not_optimized"
OPTIMIZED_DIR = "optimized"
GIF_RESIZED_DIR = "gif_not_optimized_resized"
GIF_COMPRESSED_DIR = "gif_compressed"
COMPRESSION_TYPE = ["middle", "small"]

for (root, dirs, files) in os.walk(GIFS_DIR, topdown=True):
    if len(files) > 0 and GIF_RESIZED_DIR in root:
        for file in files:
            path = root + "/" + file

            for compression in COMPRESSION_TYPE:
                final_path = path.replace(GIF_RESIZED_DIR,
                                          GIF_COMPRESSED_DIR + "/" + compression + "/" + OPTIMIZED_DIR)
                print(path, final_path)

                if compression == COMPRESSION_TYPE[0]:
                    rate = 80

                else:
                    rate = 160

                os.system("echo 0 > " + final_path)
                os.system(COMPRESSOR_DIR + " -O3 --lossy={} -o {} {}".format(rate, final_path, path))

ファイルへのパスをわかりやすくするために、gifを含むファイルディレクトリの説明を次に示します。

  1. NOT_OPTIMIZED_DIR-最適化されていないGIF
  2. GIF_RESIZED_DIR-最適化されていないGIFですが、それらのコンテナのサイズに従ってサイズ変更されます
  3. GIF_COMPRESSED_DIR-圧縮されたGIF

フォルダ内には、GIFカテゴリ名の付いたフォルダがあります。特定のフォルダーを含むフォルダー内には、「大」、「中」、「小」が含まれます（圧縮ファイルのタイプの下）。

スクリプトでは、gifを含むディレクトリを調べ、各ファイルに対して2種類の圧縮を行い、対応するコマンドラインコマンドを呼び出します。

それでは、情報を含む配列を作成するスクリプトに移りましょう。

import json
import os

GIFS_DIR = "/home/mixeden//Landingv2/"
COMPRESSOR_DIR = "/home/mixeden//gifsicle-static"

NOT_OPTIMIZED_DIR = "not_optimized"
OPTIMIZED_DIR = "optimized"
GIF_RESIZED_DIR = "gif_not_optimized_resized"
GIF_COMPRESSED_DIR = "gif_compressed"
COMPRESSION_TYPE = ["large", "middle", "small"]

OUTPUT = {}

for (root, dirs, files) in os.walk(GIFS_DIR, topdown=True):
    if len(files) > 0 and GIF_COMPRESSED_DIR in root and NOT_OPTIMIZED_DIR not in root:
        files.sort()

        type = root.split(GIFS_DIR)[1].split(GIF_COMPRESSED_DIR)[0].replace("/", "")
        print(type)

        if type not in OUTPUT:
            OUTPUT[type] = []

        if len(OUTPUT[type]) == 0:
            for file in files:
                OUTPUT[type].append(
                    {
                        "large": {
                            "url": "",
                            "size": 0
                        },
                        "middle": {
                            "url": "",
                            "size": 0
                        },
                        "small": {
                            "url": "",
                            "size": 0
                        }
                    })

        for file in files:
            full_path = root + "/" + file
            bytes_size = os.path.getsize(full_path)
            kilobytes_size = bytes_size / 1000
            megabytes_size = kilobytes_size / 1000

            index = int(file.split("_")[1].replace(".gif", "")) - 1

            for typer in COMPRESSION_TYPE:
                if typer in root:
                    local_type = typer
                    
            new_url = "gifs/" + full_path.replace(GIFS_DIR, "").replace("/" + GIF_COMPRESSED_DIR, "").replace("/" + OPTIMIZED_DIR, "")

            OUTPUT[type][index][local_type]['url'] = new_url
            OUTPUT[type][index][local_type]['size'] = megabytes_size

print(OUTPUT)
print(json.dumps(OUTPUT, indent=4, sort_keys=True))

ここでは、ファイルが含まれているフォルダーを調べ、ファイルサイズを決定し、圧縮の種類を確認して、一般的な情報をOUTPUT配列に入れます。次に、この配列をコンソールに美しく出力してコピーします。
