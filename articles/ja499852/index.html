Angularの英語ドキュメント

Angular24.ru Webサイトのすべての人がAn​​gularドキュメントのロシア語版を利用できるようになりました。



これは、6か月の作業、すべての必要な注釈プラグイン、カスタムパーサー、および2つのMagic Stringのチェーンを書き込もうとして数回失敗した結果です。これにより、フォーマットを失うことなくMarkdownファイルを翻訳できます。

そんなに苦しむことはそれだけの価値がありましたか？はい。

第1に、フォーマットを失うことなくマークダウンファイルを翻訳するための拡張可能なユーティリティを作成することが可能でしたが、これまではあまりうまく解決されていませんでした。これを確認するには、サンプルコード、テーブル、およびHTMLの部分を含むマークダウンファイルを使用します。

2番目に、Angularのドキュメントを、Google翻訳がサポートするすべての103言語に翻訳できるようになりました。

3番目に、Angularドキュメントコードを個別の回路図に取り込むことが可能でした。これにより、Angularドキュメントをプロジェクトに追加できるようになります（検索、サービスワーカー、オフラインモードのサポート付き）。

このすべてについて1つの記事で説明することはできません。そのため、あなたはどのように支援できるか、そして今後どのような記事があなたを待っているかについて知ることができます。

どうやって助けることができる？


ドキュメントはGoogle翻訳を使用して翻訳されたため、まだ完全ではありません。ここでサポートしてください。どうやって？ 2つのオプションがあります

。1）あまりよく定式化されていない、または誤った文が表示された場合、それを選択します。いくつかのオプションからポップアップメニューが表示されます。



[翻訳を提供]をクリックして、表示されるウィンドウに翻訳オプションを入力します。 Github Issueが自動的に生成されます-必要なのは、Githubにログインして、[Submit New Issue]をクリックすることだけです。

このページのオリジナルを表示する場合は、[オリジナルを開く]を選択すると、ページが自動的にオリジナルの英語バージョンで開きます。ブラウザ行で「24.ru」を「.io」に変更しても同じ効果が得られます。また、アドレスバーの「.io」アドレスバーを「24.ru」に置き換えるだけで、ドキュメントの英語版からロシア語版に切り替えることができます。

2）MarkdownとGithubに十分な自信がある場合は、次のことができます。

  • angular-ru-docsリポジトリに直接移動する
  • ブランチangular-v9-finalに切り替えます
  • aio / contentディレクトリから.mdファイルを取得して修正し、angular-v9-finalブランチでプルリクエストを実行します
