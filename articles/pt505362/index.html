Vite - desenvolvimento de leads sem pacotes no Vue

Olá, Habrovsk. Em junho, a OTUS lança o curso Vue.js Developer . Antecipando o início do curso, tradicionalmente preparávamos uma tradução de material interessante.


Um dia, Ewan Yu (criador do Vue.js) decidiu não dormir à noite e acabou criando o Vite .



O Vite permite criar aplicativos no Vue com componentes de arquivo único sem a etapa de montagem de pacotes configuráveis. As importações são solicitadas pelo navegador como módulos ES nativos. O servidor dev intercepta pedidos de arquivos .vue e os compila em tempo real. E isso acontece instantaneamente.

Observe que o Vite ainda é uma solução experimental e não sei o que acontecerá no futuro e por quanto tempo este artigo permanecerá útil. Pense nisso como um aperitivo ou apenas como algo divertido.

vamos tentar


Crie os seguintes arquivos:

Comp.vue

<template>
  <button @click="count++"> {{ count }}</button> 
</template>

<script>
export default {
  data: () => ({ count: 0 })
}
</script>

<style scoped>
button{
  font-size: 2rem;
}
</style>

index.html

<div id="app"></div>
<script type="module">
  import { createApp } from 'vue'
  import Comp from './Comp.vue'

  createApp(Comp).mount('#app')
</script>

Então faça:

npx vite

Vá para http://localhost:3000, edite o arquivo .vuepara ver alterações instantâneas.

Montagem de Produção


Agora vamos criar um aplicativo pronto para produção .

Criopackage.json

{
  "scripts": {
    "build": "vite build"
  }
}

Corre:

npm i -D vite
npm run build


Verifique o diretório com o código.
Problemas que encontrei:

  • Erro: Não foi possível encontrar o módulo 'tslib' Resolvido
    executando o npm i -D tslib
  • Erro: ENOENT: esse arquivo ou diretor não foi
    resolvido criando manualmente um diretório vazio

Pensamentos


Foi muito fácil começar e o desenvolvimento foi mais rápido, porque eu vi imediatamente o resultado das alterações. Algo como compilação em tempo real. Portanto, parece realmente promissor, talvez depois de um tempo, passemos a esse método de desenvolvimento para o Vue.

Evan Yu, faça sua mágica.

Curiosidade: vue é traduzido do francês como uma visualização e vite o mais rápido.

Parece que, neste momento, o trabalho no Vite foi concluído e possivelmente continuará após o Vue 3. A maneira mais fácil de experimentar é o Vue 3 beta.

Eu disse que foi concluído com base em um tweet de Evan Yu. No entanto, observando os commits no repositório, podemos dizer que o trabalho ainda está em andamento.
