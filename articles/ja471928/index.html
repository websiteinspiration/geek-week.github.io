Mikrotikルーターで最後に保存された構成を自動的に復元する

HPEスイッチなど、多くの優れた機能が見つかりました。何らかの理由で設定が手動で保存されなかった場合、以前に保存された設定は再起動後にロールされます。このテクノロジーは冷酷です（保存するのを忘れていました-新しい方法で保存してください）が、公平で信頼できます。

しかし、Mikrotikでは、そのような機能はデータベースにはありませんが、「ルーターのリモート構成は長い道のりです」という前兆は長い間知られていました。そして、あなたの側のルーターでさえ、「レンガからリゼットへ」に非常に簡単に変えることができます。

奇妙なことに、私はこの主題についての単一のマニュアルを見つけられなかった、私はペンでそれをしなければならなかった。

まず、構成バックアップスクリプトを作成します。将来的には、この特定のスクリプトで状態を「保存」します。[システム]-> [スクリプト]に

移動し、「fullbackup」などのスクリプトを作成します（引用符なし）。 パスワードは使用しません。それ以外の場合は隣接するスクリプトで明示的に指定する必要があるため、そのような「保護」の理由はわかりません。 起動するたびに構成を復元する2番目のスクリプトを作成します。それをfull_restoreと呼びましょう。 このスクリプトはもう少し複雑です。実際には、設定が復元されると、再起動も発生します。制御メカニズムを使用せずに、周期的な再起動が行われます。

system backup save dont-encrypt=yes name=Backup_full







制御メカニズムは、やや "オーク"であることがわかりましたが、信頼できます。スクリプトが実行されるたびに、まず「restore_on_reboot.txt」ファイルの存在が確認されます。
そのようなファイルが存在する場合、バックアップからのリカバリが必要です。ファイルを削除し、回復してから再起動します。

そのようなファイルがない場合は、このファイルを作成し、何もしないでください（つまり、これは2回目のダウンロードであり、バックアップからの復元後です）。 タスクをスケジューラに追加する前に、この時点でスクリプトをテストすることをお勧めします。 すべて問題なければ-3番目の最後のアクションに進みます-各ロードでスクリプトを実行するタスクをスケジューラに追加します。[システム]-> [スケジューラ]に 移動して、新しいタスクを追加します。 フィールドで

:if ([/file find name=restore_on_reboot.txt] != "") do={ /file rem restore_on_reboot.txt; system backup load name=Backup_full password=""} else={ /file print file=restore_on_reboot.txt }






開始時間は起動を示します（はい、文字で入力します）[ イベント時]
フィールドに「 次へ」と入力し、構成を保存するスクリプトを実行します！もう二度とやりたくないですか？ 確認のために設定に何らかの「ゴミ」を追加するか、重要なものを削除して、最後にルーターを再起動してみます。 はい、多くの人がおそらく言うでしょう：「セーフモードがあります！」ただし、作業の結果としてルーターに再接続する必要がある場合（たとえば、接続しているwifiネットワークのアドレスまたはパラメーターを変更した場合）は機能しません。また、このモードを有効にすることを「忘れる」可能性を忘れないでください。 PS今の主なことは「保存」することを忘れないことです。
/system script run full_restore
