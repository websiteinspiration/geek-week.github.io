クラスにまだテストがない場合、コマンドを使用してテストを作成できます。

階層を上に移動


別の一般的なケースは、現在のメソッド（またはクラス全体）のクラス階層を上に移動する必要がある場合です。ほとんどのIDEでは、メソッド定義の横に特別なアイコンが表示されます。そのようなアイコンをクリックすると、スーパークラス（またはインターフェース）の対応するメソッド定義に移動できます。これに加えて、IntelliJ IDEAにはCtrl + U（OS Xの場合はCmd + U）を介して使用できるスーパーメソッドと呼ばれる特別なコマンドがあります。このコマンドを使用すると、1回の動きで移動できるため、マウスに触れる必要さえありません。

階層を下に移動


クラスの階層を上に移動できる場合は、下に移動してみませんか？少数のIDEだけがこの機能を提供しており、IntelliJ IDEAはその1つです。スーパークラスにつながるアイコンに加えて、IntelliJ IDEAは、派生クラスの適切なメソッド定義に移動するためのアイコンを提供するため、上下の両方向に移動できます。



階層をさらに速く移動したい場合は、C trl + Alt + B（OS Xの場合はCmd + Alt + B）に割り当てられた実装コマンドを使用できます。このコマンドを使用すると、派生クラスのリスト全体から、目的の実装の1つを選択できます。



メソッド階層


しかし、現在のメソッドを実装またはオーバーライドするクラスの階層全体を確認する必要がある場合はどうでしょうか。IntelliJ IDEAでは、これを行うには、Ctrl + Shift + H（OS Xの場合はCmd + Shift + H）から使用できるメソッド階層コマンドを呼び出します。このコマンドは、どのクラスに行きたいかを理解するのに役立ちます。

メソッド呼び出し階層


階層、クラス、およびメソッド定義のナビゲートについて言えば、メソッド呼び出しの階層のナビゲートという1つを除くほとんどすべてを調べました。各メソッドは、他のメソッドを呼び出す他のメソッドなどを呼び出します。したがって、現在のメソッドから呼び出されたメソッドの1つに移動するか、現在のメソッドを呼び出す必要がある場合があります。IntelliJ IDEAでは、Ctrl + Alt + Hに割り当てられたCall Hierarchyコマンドを使用して、メソッド呼び出しの階層を含むポップアップウィンドウをすばやく取得できます。



シンボルの使用法を表示


もう1つの一般的なテンプレートは、シンボルが使用されている場所（定数、フィールド、変数、メソッド、クラス、またはインターフェース）の1つへの遷移であり、その上にカーソルがあります。これを行うには、任意のIDEで、[使用法を検索]コマンドを使用できます。このコマンドの短所の1つは、原則として重いことです。これは、長い検索プロセスを開始し、結果を含むツールウィンドウを開きます。そのため、IntelliJ IDEAは、Alt + Ctrl + F7（OS Xの場合はCmd + Alt + F7）を介して使用可能なShow usagesと呼ばれるこのコマンドの軽量バージョンも提供します。



このコマンドは結果をポップアップウィンドウに表示し、デフォルトでスコープを使用しますプロジェクトレベル。このコマンドをもう一度呼び出すと、スコープがプロジェクトレベルとライブラリレベルに変更されます。

最終編集ファイル


この記事の最初のテンプレートは、最新のファイルのナビゲートに関するものでした。このリストを最近編集されたファイルに絞り込む場合はどうなりますか？これは、IntelliJ IDEAでCtrl + Shift + E（OS Xの場合はCmd + Shift + E）に割り当てられた[ 最近編集されたファイル]コマンドで簡単に実行できます。



ビューとコントローラー間のナビゲーション


IDEはプロジェクトファイル内の一般的な関係（クラス階層、テストカバレッジ、使用など）のみを認識すべきだと思いますか？より具体的なものはどうですか？特別な関係に基づくナビゲーションのために、IntelliJ IDEAはCtrl + Alt + Home（Cmd + Alt + Home for OS X）に割り当てられた関連シンボルコマンドを提供します。このコマンドを使用すると、たとえば、Spring MVC、Grails、またはJavaFXアプリケーションのビューとコントローラーの間を簡単に移動できます。



ビューコントローラーへのナビゲーション
