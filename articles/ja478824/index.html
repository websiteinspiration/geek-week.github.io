Unity、ECS、アクター：最適化するものが何もないときに、ゲームでFPSを10回上げる方法[編集あり]

→ ECSは何ですか
→ 俳優は何

私は、多くの場合、どのように良い聞いたことがECSのテンプレートがあり、そのジョブはとバーストからユニティライブラリのすべてのパフォーマンスの問題の解決策があります。「たぶん」とか「たぶん」という言葉を毎回追加しないように、コードの速度について話し合って、すべてを個人的にチェックすることにしました。

私の目標は、この開発ツールの速度と、計算に並列化を使用するかどうかについて心を開くことでした。そうであれば、Unity.JobsまたはSystem.Threadingを使用する方が良いでしょうか？同時に、実際のタスクでのECSの使用方法を知りました。


テスト条件（実際のゲームタスクに近い）：

  • I5 2500プロセッサ（ハイパートレーディングなしの4コア）およびUnity2019.3.0f1
  • フレームごとの各ゲームオブジェクト...

    A）開始点から終了点まで10分間、2次ベジエ曲線に沿って移動します。

    B）math.sincos、math.asin、math.sqrt（すべてのテストで同じ非常に複雑な計算）を使用するスクエアコライダー（ボックス10f10f）を計算します。
  • FPS測定前のオブジェクトは、720fx1280fゾーン内のランダムな位置に設定され、このゾーンのランダムなポイントに移動します。
  • PC上のIL2CPPのリリースですべてがテストされています
  • テストは起動後数秒で記録されるため、すべての初期の予備的な計算とUnityシステムの組み込みがFPSに影響を与えません。同じ理由で、各フレームの更新コードのみが表示されます。
  • レンダリングがFPSに影響しないように、リリースではオブジェクトに視覚的な表示はありません。

[BurstCompile]
struct JobPositionUpdate : IJobParallelForTransform
{
	public NativeArray<SetMove> nSetMove;
	public NativeArray<Obj> nObj;
	[Unity.Collections.ReadOnly] public float deltaTime;
	
	public void Execute(int index, TransformAccess transform)
	{
		var setMove = nSetMove[index];
		var velocityToOneFrame = nSetMove[index].velocityToOneSecond * deltaTime;
		
		//   
		setMove.observedDistance += velocityToOneFrame;
		var t = setMove.observedDistance / setMove.distanceFull;
		if (t > 1f) t = 1f;
		var newPos = t.CalculateBesierPos(setMove.posToMove.c0, setMove.posToMove.c2,setMove.posToMove.c1);
		nSetMove[index] = setMove;
		
		//  
		var obj = nObj[index];
		obj.properties.c0 = newPos;
		var posAndSize = new float2x2
		{
			c0 = newPos,
			c1 = obj.collBox.posAndSize.c1
		};
		obj.collBox = obj.entity.NewCollBox(posAndSize, new float2(10f, 10f), obj.rotation.ToEulerAnglesZ());
		nObj[index] = obj;		
		
		//    
		transform.position = (Vector2) newPos;
	}
}


public struct SetMove
{
	public float2x3 posToMove;
	public float distanceFull;
	public float velocityToOneSecond;	
	public float observedDistance;

Actors + Threaded ( Actors System.Threading)


アクターには、ゲームのすべてのコンポーネントをクラスではなく構造体で保持する機能があります。これはコードを書く観点からは痔ですが、そのような条件下では、プログラムはスタックでより多く機能し、マネージヒープでは機能しません。これは、その動作速度に大きく影響します。
