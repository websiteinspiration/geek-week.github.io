フレンドリーなオープンスペースJS：クライアント側のレンダリングとラッパーの作成

Friendly Open Spaceは非常に若いフレームワークですが、すでに実行方法を知っています:-)


「Friendly Open Space」の開発に関するこの記事では、ブラウザーでのテンプレートのレンダリングをマスターし、ローカルファイルデータベースでアプリケーションを起動します。

フレームワークのコアは、クライアントで2種類のテンプレートアセンブリをサポートします。

  1. 完全なクライアント側レンダリング
  2. テンプレートをサーバーにレンダリングするリクエストが行われ、続いてブラウザウィンドウにその出力が表示されます。

最初のモードには1つの機能があります。テンプレートのレンダリングに必要なデータは、サーバーから要求されます。クライアントはFSQLクエリを実行します。アクセス制限とfosAccessモジュールを使えば特に危険はありませんが、コピーパスツールで生データをアンロードする可能性があります。ただし、この方法ではサーバーの負荷が大幅に軽減されます。

2番目のタイプのレンダリングには、すでにこの機能がありません。クライアントはテンプレートパラメータを送信し、HTMLサーバーから既に受信します。はい、サーバーの負荷は確実に増加しますが、そのような方法はオープンインターネットソリューションに適しています。

次に、application.jsアプリケーションファイル自体を作成します。

var fos = require("./fos/fos");

fos.module({
  name: "application.js",
  dependencies: [
    "fos:NServer/Application.js",
  ],
  module: function(application) {
    application.setSettings({
      port:                       3001,
      packages:                   [],
      dynamicLoading:             true,
      clientRenderingMode:        "client",
      dataClient: {
        defaultConnection: "default",
        connections: {
          default: {
            type: "file",
            file: "data.fosdb"
          }
        },
      },
      onInitialize: function(a_event) {
        if (!a_event.error) {
          application.run();
        } else {
          console.error(a_event.error);
        }
      }
    });

    application.getRouter().add([
      {
        route:      "",
        controller: "fos:NServer/NControllers/Tmpl.js",
        source:     "templates/page.tmpl",
      },
      {
        route:      "/css/*",
        controller: "fos:NServer/NControllers/File.js",
        source:     "css",
      },
      {
        route:      "/templates/*",
        controller: "fos:NServer/NControllers/File.js",
        source:     "templates",
      },
      {
        route:      "favicon.ico",
        controller: "fos:NServer/NControllers/File.js",
        source:     "favicon.ico",
      },
    ]);

    application.initialize();

  }
});

「クライアント側レンダリング」リンクをクリックすると、onClientSideRendering（）関数が呼び出され、「templates / window.tmpl」テンプレートがレンダリングされます。完全にクライアント側でアプリケーション設定で、clientRenderingModeパラメータが「client」に設定されていました。これがデフォルト値であるという事実にもかかわらず:-)。

ポップアップテンプレートの作成とラッパーテンプレートの作成


ポップアップウィンドウテンプレート自体は非常に単純です。templates / window.tmplファイルを作成します。以下がその内容です。

//~OPTIONS
{
  args:{
    fosWrapper: true,
    fosClass:   "window",
    fosInclude: ["css/window.css"],
    title:       "",
    context:     "",
  }
}

//~BLOCK main default
<div class="window-container">
  <div class="window-close" name="close">x</div>
  <div class="window-title">${{args.title}}$</div>
  <div class="window-context">${{args.context}}$</div>
</div>

ここでは、2つの新しいパラメーターfosWrapperとfosClassがあります。

fosWrapperから始めましょう。このフラグがtrueに設定されている場合、HTMLテンプレートはスパンコンテナーに配置され、ラッパーオブジェクトfos :: NRender ::ラッパーが作成されます。

アプリケーションを起動し、localhost：3001に移動して、[クライアント側レンダリング]リンクをクリックします。ポップアップウィンドウはテンプレートです。ウィンドウポップアップは完全にcss（css / window.cssファイル）によって実装されています-隠しJSを探しているわけではないと述べました:-)。

DevToolsブラウザー（Ctrl + Alt + i）を開き、[要素]タブに移動して、ウィンドウの構造を確認します。



図の青い線は、fos :: NRender :: Wrapperオブジェクトが接続されているスパンラッピングコンテナーを示しています。

テンプレートの次のシステム引数はfosClassです。 CSSクラスをラッパースパンコンテナーに追加するだけです。

これですべてがうまくいきましたが、ポップアップを閉じようとしても何も起こりません。はい、まだクローズオペレーションを書いていません！

前に述べたように、fosWrapperシステム引数がtrueの場合、テンプレートに対してラッパーオブジェクトfos :: NRender :: Wrapperが作成されます。クライアント上のテンプレートと対話するための標準インターフェースを提供します。テンプレートの標準ラッパーを再定義するには、次の形式[PATTERN NAME] .wrapper.jsに対応する名前のモジュールを作成するだけで十分です。モジュールはクラスfos :: NRender :: Wrapperから継承する必要があります。

次に、templates / window.tmplテンプレートのラッパーファイルを作成します。新しいクラスは、記号「x」をクリックすると、ポップアップウィンドウを閉じる必要があります。

ファイルテンプレート/ window.wrapper.js：

fos.module({
  name: "templates/window.wrapper.js",
  dependencies: ["fos:NRender/Wrapper.js"],
  module: function(Wrapper){
    return function(a_initializeOptions) {
      var self = this;
      Wrapper.call(this, a_initializeOptions);

      var parentAttach = this._attach;
      this._attach = function(a_cb) {
        parentAttach.call(this, function(){
          fos.addDomListener(fos.select(self.getDomElement(), "[name=close]")[0], "click", function(){
            self.getDomElement().classList.add("window-hide");
            setTimeout(function() { self.destroy(); }, 2000);
          });
          a_cb();
        });
      }

    }
  }
});

シンボル「x」をクリックするイベントに接続するには、fos :: addDomListenerを呼び出します。これにより、DOM要素の親を変更したときに、接続されたイベントがリセットされません。

イベントハンドラーで、ウィンドウを閉じます。まず、CSSクラス「window-hide」を追加して、画面からウィンドウを削除します。アニメーションが完了したら、fos :: NRender :: Wrapper :: destroyメソッドを呼び出して、テンプレートとラッパーオブジェクトを削除します。

すべてのアプリケーションが書面になり、発表され、取り消されます!!!
