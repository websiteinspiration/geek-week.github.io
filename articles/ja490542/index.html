PythonとBashを一緒にします。python-shellおよびsmart-env vライブラリのリリースです。1.0.1

全てにおいて良い日！

2020年2月29日、smart-envおよびpython-shellライブラリの公式マイクロリリースが開催されました。知らない人のために、まず最初の投稿を読むことをお勧めします。

つまり、変更には、コマンドのオートコンプリート、コマンドを実行する機能の拡張、少しのリファクタリング、バグ修正が含まれます。

猫の下で詳細をお願いします。

python-shellの新機能


すぐにデザートから始めます。

チームの完成


同意します-エディタ/ IDE /ターミナルがコマンドの名前を要求し、時にはパラメータを呼び出すと便利です？そのため、Pythonシェルはそのような機能を提供するために少し進んでいます。事実上のフードの下にあるShellクラスのフィールドはそのフィールドではない（ユビキタスな__getattr__）ため、オートコンプリートも最初から作成されます（それぞれ__dir__メソッドをオーバーロードすることにより）。オートコンプリートは現在、BPythonおよびIPython環境で実行されています。もちろん、PyCharmのようなより由緒ある製品との統合を見たいので、この方向で実装の可能性を模索しています。

プロパティを追加する


リリースの一部として、Shellクラスは新しいlast_commandプロパティを受け取りました。ShellExceptionがゼロ以外の戻りコードを持つコマンドによってスローされたときに、コマンドオブジェクトがコマンドオブジェクトの__call __（）呼び出しから返されなかったため、この必要性が生じました。今、そうする機会があります：

try:
    command = Shell.touch('/foo.txt')
except ShellException:
    command = Shell.last_command

Commandオブジェクトのプロパティのリストも拡張されました。コマンドの出力をエラーストリームに返すエラーフィールドが追加されました。

Pythonの無効な名前でコマンドを実行する


ほとんどすべてのシステムには、名前がPythonの識別子として適合しないプログラムが少なくとも1つあります（たとえば、よく知られているユーティリティ2to3）。彼女に電話する

Shell.2to3()

失敗-インタープリターはスキップしません。
解決策は、回避策でコマンドを呼び出すことです。

Shell("2to3")  #

同じ方法で、インタプリタの観点から有効なコマンドを実行できるため、次のような柔軟なスクリプトを作成できることに注意してください。

cmd = "python{}".format(sys.version_info[0])
Shell(cmd)(*args, **kwargs)

マイナーな変更


  • Commandオブジェクトの__repr __（）メソッドと__str __（）メソッドが実装され、直感的な値が生成されるようになりました（それぞれパラメーターを持つコマンドとその標準出力の排気）。
  • コードによるマイナー修正。
  • テストカバレッジを追加するだけでなく、既存のカバレッジを再編成します。
  • サブプロセスおよびプロセスクラスを追加します。サブクラスの目的は、サブプロセスモジュールで作業するときに、追加の抽象化レベルを作成することです。ほとんどの場合、Python 2/3を使用する場合はコードの繰り返しを排除する必要がありますが、他のボーナスが発生する可能性があります。

smart-envの新機能


python-shellとは異なり、smart-envライブラリの変更は少なくなっています。これの理由は単純です-空き時間の不足、その間にいくつかの潜在的な改善（たとえば、環境変数の完了）が次のリリースに引き継がれたことです。

実際、ライブラリには次の変更が加えられています。

  • コードによるマイナー修正。
  • リファクタリング
  • 既存のテストの再編成と改良。
