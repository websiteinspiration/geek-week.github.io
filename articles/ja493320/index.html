独自のコンポーネントでLaravelを拡張する

仕事


プロバイダーを選択するオプション付きのSMSメッセージを介してユーザーにアラートシステムを追加します。

決定


私の意見では、最良の解決策は独自のコンポーネントを追加することです。
コンポーネントは、さまざまなドライバーを通じて割り当てられたタスクを解決できるアクション（契約）のセットが明確に定義されたプログラムブロックです。例えば、コンポーネント内蔵Cache ドライバを使用することができる：file、memcachedまたはredis。

コンポーネントを構築するときは、Laravelコンポーネントを構築するのと同じ原則であるBridge設計原則を適用します。

それでは始めましょう。

運転手


仮定の1つが言うように：実装ではなく、インターフェイスに従ってプログラムします。

まず始めに、コンポーネントのコンテキストでは、契約という用語を使用する方がより正確です。コントラクトは基本的に、コンポーネントの機能を説明するための一連のインターフェースです。

interface SmsContract
{
    public function send();
}

次に、コントラクトを実行するドライバーを追加します。抽象クラスで一般的なロジックを取り出します。

abstract class Driver implements SmsContract
{
    protected $message;

    protected $recipient;

    public function to($phoneNumber)
    {
        $this->recipient = $phoneNumber;
    }

    public function content($message)
    {
        $this->message = $message;
    }

    abstract public function send();
}

そして、各ドライバークラスの送信ロジックを直接説明します。

class SmsRuDriver extends Driver
{
    protected $client;

    protected $from;

    public function __construct(SmsRuClient $smsRuClient, $from)
    {
        $this->client = $smsRuClient;
        $this->from = $from;
    }

    public function send()
    {
        return $this->client->sendSms([
            'type' => 'text',
            'from' => $this->from,
            'to' => $this->recipient,
            'text' => trim($this->message)
        ]);
    }
}


class SmartSenderDriver extends Driver
{
    protected $client;

    protected $from;

    public function __construct(SmartSenderClient $smartSenderClient, $from)
    {
        $this->client = $smartSenderClient;
        $this->from = $from;
    }

    public function send()
    {
        return $this->client->message()->send([
            'type' => 'text',
            'from' => $this->from,
            'to' => $this->recipient,
            'text' => trim($this->message)
        ]);
    }
}

コンポーネント管理


ドライバーを選択して構成するには、クラスSmsManagerから継承したクラスを追加しますManager。

use Illuminate\Support\Manager;

class SmsManager extends Manager
{
    public function setDriver($name = null)
    {
        return $this->driver($name);
    }

    public function createSmsRuDriver(): SmsContract
    {
        return new SmsRuDriver(
            $this->createSmsRuClient(),
            $this->app['config']['sms.sms_ru.from']
        );
    }

    public function createSmartSenderDriver(): SmsContract
    {
        return new SmartSenderDriver(
            $this->createSmartSenderClient(),
            $this->app['config']['sms.smart_sender.from']
        );
    }

    public function getDefaultDriver()
    {
        return $this->app['config']['sms.default'];
    }

    protected function createSmsRuClient()
    {
        return new SmsRuClient(
            $this->app['config']['sms.sms_ru.key'],
            $this->app['config']['sms.sms_ru.secret']
        );
    }

    protected function createSmartSenderClient()
    {
        return new SmartSenderClient(
            $this->app['config']['sms.smart_sender.key'],
            $this->app['config']['sms.smart_sender.secret']
        );
    }

}

これで、構成ファイルで必要なパラメーターを指定するだけで十分であり、クラスSmsManager はSMSメッセージを送信するドライバーを介してコンポーネントに通知します。

ファサード


アプリケーションのどこからでもコンポーネントの機能にアクセスするには、ファサードを追加します。
まず、ファサードクラス自体を作成します。

use Illuminate\Support\Facades\Facade;

class Sms extends Facade
{
    protected static function getFacadeAccessor()
    {
        return 'sms';
    }
}

次に、サービスプロバイダーSmsManager を使用してクラスをファサードに関連付け、エイリアスを登録します。

class SmsServiceProvider extends ServiceProvider
{
    protected $defer = true;
    
    public function register()
    {
        $this->app->singleton('sms', function ($app) {
            return new SmsManager($app);
        });
    }
    
    public function provides()
    {
        return ['sms'];
    }
}

そして最後に、サービスプロバイダーを他のプロバイダーと一緒に構成ファイルに登録します（app/config/app.php）。

'providers' => [
    ...
    App\Providers\SmsServiceProvider::class,
],

応用


コンポーネントのファサードが作成され、パラメーターが構成ファイルで指定されている場合、SMSメッセージを送信するには、次のコードを追加するだけです。

Sms::to($phone)->content('Beware! He`s not who he is')->send();

または、ドライバーを明示的に指定する必要がある場合：

Sms::setDriver('sms_ru')->to($phone)->content('why don`t you answer me?')->send();

結論


私たちのコンポーネントは、ブリッジを設計するという原則に基づいて構築されています。つまり、実装と抽象化は、独立して変更できるように分離されています。SmsManagerアプリケーションコード自体は変更されないまま、クラスを介してドライバーを管理できます。
ファサードから、コンポーネントの機能に簡単にアクセスできます。

コンポーネント設計へのこのアプローチは、それらの使用においてシンプルさと柔軟性を提供します。
