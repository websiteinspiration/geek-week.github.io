Linux向けの新しいPunto Switcher：xswitcher

xneurサポートの廃止により、過去6か月間にいくつかの苦痛が生じました（私のデスクトップでのOpenSUSE 15.1の登場により、xneurをオンにすると、ウィンドウがフォーカスを失い、キーボード入力に合わせて面白くちらつきます）。

「ああ、いまいましい、間違ったレイアウトでもう一度入力し始めた」-それは私の仕事ではよくわいせつです。プラスを追加しません。


同時に、私は（設計エンジニアとして）自分が何を望んでいるかを明確に説明できます。そして、私は（最初にPunto Switcherから、次にWindows Vistaのおかげで、ついにLinuxにxneurから再移植して）正確に1つを望みました。画面上のゴミがそのレイアウトに含まれていないことに気づき（これは通常、新しい単語セットの最後に発生します）、「一時停止/中断」を実行します。そして、私が印刷したものを入手してください。

現時点では、この製品は（私から見て）最適な機能/複雑さの比率を持っています。共有する時が来ました。

TL.DR


あらゆる種類の技術的な詳細が続くので、最初に- せっかちな人のための「タッチ」リンク。

現在、次の動作がハードコードされています。

  • 「一時停止/中断」：最後の単語をバックスペースし、アクティブウィンドウのレイアウトを切り替え（0と1の間）、再度入力します。
  • 何もない左Ctrl：アクティブウィンドウのレイアウトを切り替えます（0と1の間）。
  • 「左シフトなし」：アクティブウィンドウのレイアウト0をオンにします。
  • 「何もせずに右シフト」：アクティブなウィンドウのレイアウト1をオンにします。

今後は動作をカスタマイズする予定です。フィードバックがないと面白くありません（とにかくそれで満足です）。Habréでは、同様の問題を抱える聴衆の十分な割合がいると思います。

NB以来 現在のバージョンでは、キーロガーは "/ dev / input /"にねじ込まれています。xswitcherはroot権限で実行する必要があります。

chown root:root xswitcher
chmod +xs xswitcher

注意： suidを持つファイルの所有者はrootでなければなりません。所有者が誰であるかはsuidであり、起動をオンにします。

パラノイド（私も例外ではありません）はGITから複製してその場で組み立てることができます。そのように：

go get "github.com/micmonay/keybd_event"
go get "github.com/gvalkov/golang-evdev"

### X11 headers for OpenSUSE/deb-based
zypper install libX11-devel libXmu-devel
apt-get install libx11-dev libxmu-dev

cd "x switcher/src/"
go build -o xswitcher -ldflags "-s -w" --tags static_all src/*.go

Autostart add to taste（DEによって異なります）。

「お粥を要求しないでください」（1日あたりCPU約30秒、RSSで約12 MB）。

細部


詳細については、今。

リポジトリ全体はもともと私のペットプロジェクト専用でしたが、別のプロジェクトを開始するのは面倒です。したがって、すべてが（フォルダーに）積み上げられ、AGPLでカバーされます（「特許逆も同様」）。

xswitcherコードはCの最小限のスプラッシュを使用してgolangで記述されています。このアプローチは（これまでのところ）最小限の労力で済むと想定されています。CGOを介して不足しているものを接続する機能を維持しながら。

解説のテキストは、それがどこから借りられたのか、そしてその理由からレイアウトされています。なぜなら xneurコードは「私を刺激しませんでした」、loloswitcherが出発点として採用されました。

"/ dev / input /"を使用することには、利点（オートリピート付きの押されたキーを含むすべてが表示される）とマイナスがあります。短所は次のとおりです。

  • 自動繰り返し（コード「2」のイベント）は、Xでの繰り返しとは相関しません。
  • X11インターフェースを介して入力を見ることができない（たとえば、VNCが機能する）。
  • ルートが必要です。

または、「XSelectExtensionEvent（）」を通じてXイベントをサブスクライブすることもできます。xinputコードを覗くことができます。とりあえず、そのようなものは何も見つかりませんでした。ドラフトの実装では、100行のCコードがすぐに提供されました。これまでのところ脇に置いておきます。

これまでの「戻る」という結論は、仮想キーボードをねじ込むことによって行われました。作者のkeybd_eventに感謝しますが、抽象化が高すぎるため、さらにやり直す必要があります。たとえば、私は正しいWinキーを持っています。3行目が選択します。そして、左の勝利だけが放送されます。

既知のバグ


  • 「複合」入力については何も知りません（例：½）。今は必要ありません。
  • 正しいWinを誤って再現します。私の場合、アクセントの配置が崩れています。
  • 入力の明確な解析はありません。代わりに、いくつかの関数があります：Compare（）、CtrlSequence（）、RepeatSequence（）、SpaceSequence（）。感謝nsmcan注意力：ここのコードで修正されました。一定の確率で、交換時にバグを見つけることができます。
    この場所では「方法」がわからないので、どんな申し出も喜んで受けます。
  • （ああ、ホラー）チャネル（keyboardEvents、miceEvents）の競争的な使用。

結論


コードは最も単純な手続きです。そして、私のように愚か。だから、私はほとんどすべての技術者が彼が望んでいることを終えることができることを望んで自分を慰めています。そして、このため、この製品は、ほとんどの楽しみのようなサポートなしでは消えません。
