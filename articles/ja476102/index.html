PHP配列を使用したタイプセーフ

みなさん、こんにちは。私は、PHPで配列を使って作業するのに便利な自分の自転車についてお話します。

タイプヒント


型ヒントはPHP7で登場しました。これにより、IDEはコードの静的分析をより適切に実行できるようになり、コードの品質が向上しました（または正確に「良くなった」と言っています）。

もちろん、以前はIDEがコードへのコメントにヒントを書き込むことが可能でしたが、タイプはコードの一部になり、リファクタリングが可能になり、どこかで何かを忘れることを恐れなくなりました（もちろん、クラスやインターフェースの名前を変更するという意味でのリファクタリング）。

出力タイプを指定できるようになったほか、入力引数のタイプを指定できるようになりました。

しかし、優れた機能に加えて、型ヒントは責任を課します。つまり、変数の型は、実際にはメソッドシグネチャに示されているとおりでなければなりません。

タイプをチェックしないと、メソッドとコンストラクターでエラーが発生する可能性があります（デザイナーのエラーは特に助長されます）。

手動で小切手を書くのは面倒です。私はこのビジネスを自動化することにしましたが、検証ではなく、目的のタイプにキャストすることで決めました。

私の作業では、多くの場合、ゼロから作成する必要があります。通常、これはプロトタイプ、パーサー、または新しいデータソースのETL、実際にはパーサーのいずれかです。

もちろん、配列を操作する場合（たとえば、* .csvから読み取る場合）は、ORMを介してデータベースを操作できますが、私のタスクでは扱いにくいので、PDOを使用してデータベースを操作すると便利です。「お気に入り」Bitrixは、配列以外でデータを返す方法を知りません。

配列からデータを取得する方法に関係なく。そこで、配列を扱うためのラッパーを書きました。

プロジェクト間でコードをコピーして貼り付けないようにするため、Composerのパッケージを設計しました。

composer require sbwerewolf/language-specific

Arrayhandler


次の要件は、配列を1つの値から正確にこの値に単純化できることでした。

ドキュメントから例を示します：

$connection = new PDO ($dsn,$login,$password);

$command = $connection->prepare('select name from employee where salary > 10000');
$command->execute();
$data = $command->fetchAll(PDO::FETCH_ASSOC);
/*
$data =
    array (
        0 =>
            array (
                'name' => 'Mike',
            ),
        1 =>
            array (
                'name' => 'Tom',
            ),
        2 =>
            array (
                'name' => 'Jerry',
            ),
        3 =>
            array (
                'name' => 'Mary',
            )
    );
*/
$names = new ArrayHandler($data);
$result = $names->simplify();

echo var_export($result,true);
/*
LanguageSpecific\ArrayHandler::__set_state(array(
   '_data' => 
  array (
    0 => 'Mike',
    1 => 'Tom',
    2 => 'Jerry',
    3 => 'Mary',
  ),
))
*/

もちろん、リクエストから返される配列を実行して、この割り当てを行うことができます。

$response[] = $element[0];

ですが、あまり好きではないので、自動的に発生させるため、simplify（）メソッドが登場しました。

まあ、我々は、配列のラッパーを持っているので、私たちはインデックスの有無をチェックするためにメソッドを追加します- あり）（あなたは、配列の要素の上に行きたい場合は、次の（）メソッドが役立ちます。

これは、自動化レベルが快適なレベルに達したために停止した可能性がありますが、ネストされた配列のネストされた配列を操作する必要がある場合があり、ターゲット配列のArrayHandlerをすぐに取得する方が便利なので、ネストされた配列のArrayHandlerを返すpull（）メソッドを追加しました。 次のようになります。



$address = new ArrayHandler($item)->pull('metaDataProperty')->pull('GeocoderMetaData')->pull('Address')->asIs();

もちろん次のように書くことができます：

$address = $item['GeoObject']['metaDataProperty']['GeocoderMetaData']['Address'];

、しかし私の目には角括弧の数に波紋があり、pull（）を使うとより便利です。

一般的な推論


コードがComposerから接続されている場合は、コピーと貼り付けの必要がなくなり、1つのコマンドでライブラリを取得できるので、非常に便利です。

パッケージを作成する前に、類似物を調べたところ、そのようなものは見つかりませんでした。配列を単純にラップするプロジェクトがいくつかあります。これらのプロジェクトでは、配列を操作するための多くのメソッドを単純にラップし、タイプセーフはどこにもありません。

どうやら、変数名の前に（int）または（bool）を書き込むことは誰にとってもシンプルで便利であり、このことのために別のリポジトリを使用する理由は誰にもわかりません。

ライブラリの機能は、記事で説明されている機能よりも少し幅が広く、詳細情報はドキュメント（README.md）で入手できます。

PHP5はまだ一般的ではありません。そのため、ライブラリにはPHP5の別のバージョンがあり、PHP7のバージョンとはいくつかのメソッドの名前が異なります。もちろん、すべての型のヒントはコメントのみです。
PHP7.2にはライブラリバージョンがあり、それは、戻り値の型がオブジェクト（）メソッドのシグニチャー（オブジェクト）に表示されることのみが異なります。

コードはテストで完全にカバーされていますが、原則として何も壊すことはありません:)

別の使用例


foreach ($featureMember as $item) {
    $pointInfo = extract($item);
    $info = new ArrayHandler($pointInfo);

    $address = $info->get('formatted')->default('')->str();
    $longitude = $info->get('longitude')->default(61.402554)->double();
    $latitude = $info->get('latitude')->default(55.159897)->double();

    $undefined = !$info->get('formatted')->has();

    $properties = ['longitude' => $longitude, 'latitude' => $latitude, 'address ' => $address ,'undefined'=>$undefined,];
    $result = json_encode($properties);
    output($result);
}
