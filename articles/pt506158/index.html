useSWR é minha nova biblioteca favorita do React

Uma tradução do artigo foi preparada antes do início do curso "React.js developer" .


Nos últimos meses, tenho trabalhado em um aplicativo no NextJS. Toda semana, torna-se cada vez mais. O aplicativo usa axios para chamadas de API e unstated-next para gerenciamento de estado. Existem muitas chamadas de API, mas não queremos que os usuários vejam várias telas de carregamento. Portanto, armazenamos os resultados de chamadas axios em repositórios não declarados .

No entanto, encontramos um problema. As próprias abóbadas estão se tornando cada vez mais complexas. Às vezes, as páginas exigem várias chamadas de API e, por sua vez, dependem dos resultados de outras chamadas de API. Dias se transformaram em semanas e semanas em meses, enquanto nossos cofres não declarados se tornavam cada vez mais pesados. Ocorremos erros estranhos enquanto nossa lógica de cache caseira lutava para lidar com casos extremos inesperados.

Então pensamos que o caminho deveria ser mais simples.
E aqui está ele. E é ainda desenvolvido pela Vercel , criadora do NextJS .

Conheça o SWR


O nome SWR vem de stale-while-revalidate , um método de cache que agora está ganhando popularidade no desenvolvimento de front-end. Ele permite que você baixe conteúdo em cache imediatamente e atualize-o imediatamente para lidar com novo conteúdo no futuro. No nosso caso, conseguimos o compromisso perfeito entre desempenho e experiência do usuário.

Como usar useSWR


useSWR é uma biblioteca de ganchos React desenvolvida pela Vercel . Ele permite recuperar dados de uma API ou outra fonte externa, salvá-los no cache e renderizá-los.

Vamos começar examinando um exemplo de um componente React que recebe uma lista TODO de um servidor JSON e a renderiza.

import React from "react";
import "./App.css";
const todosEndpoint = "http://localhost:3001/todos";
const TodoApp = () => {
  const [todos, setTodos] = React.useState([]);
  React.useEffect(() => {
    const getData = async () => {
      const response = await fetch(todosEndpoint);
      const data = await response.json();
      setTodos(data);
    };
     getData();
    }, []);
  return (
    <div className="App">
      {todos.map((todo) => (
        <div key={todo.id}>{todo.title}</div>
      ))}
    </div>
  );
};
export default TodoApp;

Agora vamos ver o mesmo componente, mas reescrito usando useSWR .

import React from "react";
import useSWR from "swr";
const todosEndpoint = "http://localhost:3001/todos";
const getData = async () => {
  const response = await fetch(todosEndpoint);
  return await response.json();
};
const TodoApp = () => {
  const { data: todos } = useSWR(todosEndpoint, getData);
  
return (
    <div>
      {todos && todos.map(todo => (
        <div key={todo.id}>{todo.title}</div>
      )}
    </div>
  );
};
export default TodoApp;

Como você pode ver, é muito semelhante à implementação anterior.

Neste exemplo, usamos useSWR (key, buscador, opções) para obter nossa lista TODO. A chave useSWR é necessária para o armazenamento em cache. No nosso caso, usamos todosEndpoint . Como buscador, passamos por uma função assíncrona que recupera uma lista TODO.

Note-se que useSWR não tem idéia de como você recupera dados. Você pode usar qualquer estratégia de mineração de dados assíncrona que desejar. Você pode usar busca , axios ou mesmo GraphQL . Enquanto sua função está retornando dados de forma assíncrona,useSWR ficará feliz.

A chamada useSWR retorna os seguintes parâmetros:

  • data : dados principais obtidos usando o buscador (não definido, se ainda não estiver carregado);
  • erro : erro causado pelo buscador ;
  • isValidating : um booleano que informa se uma solicitação ou revalidação está em andamento;
  • mutate (data?, shouldRevalidate) : função para modificar dados em cache.

Exemplo mais complexo


Vejamos um exemplo mais complexo que usa mais parâmetros retornados por useSWR . Neste exemplo, recuperaremos um elemento da lista TODO e renderizaremos a caixa de seleção para o status completo de nossa tarefa da lista. Quando a caixa de seleção é clicada, enviamos uma solicitação PUT para atualizar o TODO e, em seguida, chamamos mutate para atualizar o cache useSWR .

import React from "react";
import useSWR from "swr";
import "./App.css";
const todosEndpoint = "http://localhost:3001/todos";
const getTodo = async (id) => {
  const response = await fetch(`${todosEndpoint}/${id}`);
  return await response.json();
};
const updateTodo = async (id, todo) => {
  const response = await fetch(`${todosEndpoint}/${id}`, {
    method: "PUT",
    headers: {
      "Content-type": "application/json; charset=UTF-8",
    },
    body: JSON.stringify(todo),
  });
  return await response.json();
};
const TodoApp = () => {
  const todoId = 1;
  const key = `${todosEndpoint}/${todoId}`;
  const { data: todo, mutate } = useSWR(key, () =>
    getTodo(todoId)
  );
  const toggleCompleted = async () => {
    const newTodo = {
      ...todo,
      completed: !todo.completed,
    };
    await updateTodo(todoId, newTodo);
    mutate(newTodo);
  };
  if (!todo) {
    return <div>Loading...</div>;
  }
  return (
    <div>
      <p>{todo.title}</p>
      <div>
        <input
          type="checkbox"
          id="completed"
          name="completed"
          value="completed"
          checked={todo.completed}
          onChange={toggleCompleted}
        />
        <label htmlFor="completed">Completed</label>
      </div>
    </div>
  );
};

Usar o mutate é uma ótima maneira de aumentar o desempenho visível do seu aplicativo da web. Podemos alterar dados localmente e alterar a visualização, sem esperar até que a fonte de dados remota seja atualizada. O useSWR irá revalidá- los e substituí-los pelos dados mais recentes em segundo plano.

Por que preciso de uma biblioteca para isso?


Você pode fazer esta pergunta. Talvez você já tenha um mecanismo de gerenciamento de estado em seu aplicativo e não considere usar uma biblioteca de terceiros para algo tão simples como extração e armazenamento em cache de dados.

E eu responderei que o armazenamento em cache de dados não é fácil. E à medida que seu aplicativo cresce, fica ainda mais difícil. Você encontrará casos extremos e, processando-os, obterá repositórios e provedores complexos, difíceis de entender e de manter. Erros surgirão do nada.
“Na ciência da computação, existem apenas dois problemas complexos: invalidação de cache e invenção de nomes” - Phil Carleton

Em vez de implementar sua própria solução de mineração de dados, por que não confiar no comprovado, desenvolvido por uma das empresas mais respeitadas no ecossistema React?

Como o useSWR nos ajudou


Portar a questão da extração de dados em nosso aplicativo para usar o SWR deu muitas vantagens.

1. Ele nos permitiu remover as

solicitações pull de código que excluem código desnecessário, mas não adicionam um novo - meus favoritos. Quanto menos código no aplicativo, menor a probabilidade de ocorrerem erros. E isso é bom. À medida que envelheço, começo a apreciar cada vez mais a simplicidade. O useSWR nos permitiu remover completamente o armazenamento não declarado, o que simplificou nosso aplicativo e facilitou o entendimento.

2. Ele simplificou a conexão de novos desenvolvedores ao projeto

useSWRcomo a maioria dos projetos de código aberto, possui excelente documentação. A implementação de nossa própria solução significaria que precisamos escrever nossa própria documentação e ensinar aos novos desenvolvedores do projeto como lidar com a extração de dados. Agora que usamos o useSWR , não precisamos fazer isso.

3. Ele simplificou as coisas difíceis:

nosso aplicativo, como a maioria dos outros aplicativos, contém muitas chamadas de API. Algumas dessas chamadas dependem de outras chamadas da API. Com useSWR, é fácil escrever ganchos para consultas dependentes.

4. Melhorou o desempenho visível.O

aplicativo parece mais rápido. Os usuários classificaram e deram um bom feedback.

5. Atualiza dados obsoletos

O useSWR atualiza dados desatualizados em foco. Isso significa que os usuários sempre têm a versão mais atualizada dos dados, excluindo o tempo de download.

Conclusão


O useSWR teve um grande impacto em nosso aplicativo. Essa biblioteca simplificou nosso código e melhorou a experiência do usuário. Não consigo nem imaginar outra biblioteca que seria tão fácil de implementar e tenha tantas vantagens quanto o useSWR . Para saber mais sobre o useSWR , você pode visitar o site ou o repositório no GitHub .

Bibliotecas alternativas


useSWR não é a única biblioteca boa para recuperar dados. Embora tenha tido um bom desempenho em nosso caso, talvez outra biblioteca seja mais adequada para você para extração de dados.

React-query


O react-query é uma biblioteca de ganchos para recuperar, armazenar em cache e atualizar dados assíncronos no React. Essa solução popular, escrita por Tanner Linsi , tem escopo muito semelhante ao useSWR .

Apollo


O cliente Apollo é um cliente GraphQL que cuida de recuperar e armazenar em cache seus dados, além de atualizar a interface do usuário. Se você usa o GraphQL , o Apollo será uma ótima solução.
