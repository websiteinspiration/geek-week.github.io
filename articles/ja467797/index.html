「ために笑う」：「戦闘」アプリケーションのないソフトウェアツールが必要になる理由

GitHubは1億を超えるリポジトリをホストしています。それらのいくつかは、大企業が使用するアプリケーションです。他にも、開発者がプロ​​ジェクトで積極的に使用する小さなライブラリがあります。しかし、実際には役に立たないユーティリティを備えたリポジトリがあります。

それらが注目される理由を説明します。ヒント：水曜日は少し金曜日です。


— Pineapple Supply Co. — Unsplash


ある状況を想像してみてください。新しいモジュールの作成が完了し、現在のタスクをすべて閉じたところです。しかし、就業日の終わりまではまだ1時間あり、会社が上司よりも早く帰宅することは慣例ではありません。同時に、経営陣は従業員がアイドル状態のときは気に入らず、常に追加のタスクをスローする準備ができています。特別なアプリケーションが助けになり、嵐の活動を描写するのに役立ちます。

そのようなアプリケーションの1つがHackerTyperです。入力した文字セットを一種のプログラムコードに変換します。ユーティリティの例は、プロジェクトのWebサイトにあります。 HackerTyperも代替を持っている- GeekTyper。 Redditの居住者がこのツールを使用したのは、自分の街のローカルTVチャンネルがゲーム開発会社での彼の仕事のストーリーを撮影しました：


GitHubには、労働時間を延長することはできませんが、逆にそれを削減するのに役立つツール（たとえば、フォルクスワーゲンアプリケーション）があります。コードが継続的インテグレーション（CI）サーバーによってチェックされると、フォルクスワーゲンはエラーの数を減らし、テストが成功するようにします。したがって、テストについて心配する必要はなく、他の「より重要な」タスクにもっと注意を払うことができます。ユーティリティは、自動車メーカーの周りに広まったスキャンダルの直後に開発されたので、ドイツの自動車メーカーの名前を冠しています。 4年前、ディーゼル車で、企業はテスト中に排気ガス中の有害ガスの量を数十回過小評価するソフトウェアを発見しました。

また、注目に値するのは、No Codeユーティリティです。。キーボードにまったく触れたくない場合は、彼女に連絡することができます。ユーティリティと呼ぶのは非常に困難ですが、リポジトリでは1行のコードも見つかりません。コードなしで作業を開始するには、何もしないでください。著者によると、これは信頼性が高く安全なアプリケーションを作成するための最良の方法です。

データを保存するタイミング


GitHubリポジトリーでは、πfsファイルシステムを見つけることができます。これは、Pi番号でデータを保存することによりディスクスペースを節約することを提供します。ファイルシステムの操作は、πでは任意の数字のシーケンスに対応できるという記述に基づいています。次のコマンドを使用してデータを保存できます。

πfs -o mdd=<metadata directory> <mountpoint>

大括弧内の最初のパラメーターは、πfsが保存されたファイルのメタデータを保管するディレクトリーであり、2番目のパラメーターは、標準ファイル・システムのマウント・ポイントです。ファイルのオフセットとその長さがπでわかっている場合は、ベイリー-ボーウェイン-プラフの式を使用してファイルのオフセットを抽出し、必要な数字のシーケンスPiを見つけることができます。


写真-ozz314 -CC BY /写真の変更

Pi番号は、既知のリポジトリとしてだけでなく、不明なデータのリポジトリとしても使用できるという一般的な見方があります。理論的には、写真から、書かれていない本、未完成の写真、撮影されていない映画を抽出できます。最も重要なのは、それらのメタデータを取得することです。

変わったサービスが必要なとき


IT業界では、XaaS（Everything as a Service）などの「Everything as a Service」という用語があります。クラウドコンピューティングを使用するすべてのサービスについて説明しています。ネットワーク上で、この定義を新しいレベルに引き上げるユーティリティを見つけることができます。たとえば、「サービスとしてのブール式」を提供するbooleans.ioサービス。これは、ブールオブジェクトを作成、削除、および更新できるAPIセットです。たとえば、リクエストに対して：

curl -X POST https://api.booleans.io --header "Content-Type: application/json" --data '{"value": true, "label": "Hello world!"}' --header "Authorization: Token [token]

サービスは次の応答を生成します。
 
{
  "id": "4ee4b8f7-8d31-4ae2-93b0-554e19af740d",
  "created_at": "2018-11-24 14:56:26",
  "updated_at": "2018-11-24 14:57:43",
  "value": true,
  "label": "Hello world!"
}

別の例はshoutcloud.ioです。これはすべて大文字のサービスで、小文字を自動的に大文字に変換します。特別なAPIを使用するだけで十分です。

curl -X POST \
    -d '{"INPUT": "hello world"}' \
    -H 'Content-Type: application/json' \
    HTTP://API.SHOUTCLOUD.IO/V1/SHOUT

    { "INPUT":"hello world",
      "OUTPUT":"HELLO WORLD" }

もちろん、説明されている各ツールが本番環境でアプリケーションを見つけることはほとんどありません。しかし、彼らは面白い経験をし、仕事の合間に楽しませることができます。ジョークとして作成された他のユーティリティを知っている場合は、コメントで共有してください。
