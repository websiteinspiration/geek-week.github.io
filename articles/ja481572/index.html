2020年の私の新しいウェブ技術スタック

Webテクノロジースタックがシンプルだった時代を覚えていますか？これらのスタックのレベルを、LAMP、LEMP、LEPPのような4文字の削減の形で指定できるのはいつですか。サイトの作成と保守に必要なものがすべて、完全に通常のハードウェア、いくつかのオープンソースソフトウェアに、そして目標の達成に固執するようになったときはどうでしょうか。

私の最初の成功したサイトは、1999年に古いプロジェクトになりましたが、片手で数えることができるテクノロジー（HTML4、CSS2、JavaScript3、Apache 1.1）を使用して作成されました。これはすべて、Linux 2.0を搭載したサーバーで実行されていました。サイトには38,000ページが含まれていました。そして今日、20年経った今でも、彼はそれらを発行しています。 それ以来、すべてが変わった。これは、Webテクノロジースタックにも適用されます。今、彼らは以前とまったく同じではありません。





本日の翻訳を公開している記事の著者は、フルスタックから2020スタックに移行した経緯についてお話ししたいと思います。この旅の間、いくつかのテクノロジーは予想外にお気に入りになり、いくつかのテクノロジーは以前の魅力を失いました。

2020 Webスタック


2020年は新しい10年の始まりです。今こそ、新しいWebテクノロジースタックについて話す時です。

2020スタックはどのように見えますか？これは、サイト開発者が達成しようとしていることに大きく影響されていると言わざるを得ません。適切なレベルの選択は、プロジェクトに必要なスケーラビリティの度合いに大きく依存します。

私は特に小さなウェブサイトに興味があります。仮想サーバー上で気分が良いもの。これらのサイトには、ロードバランサーや永続的なデータストアは必要ありません。これは、WordPressが長い間使用してきたCMSのニッチです。しかし、すべての中心にあるのは、ある種のミニマルなサーバーではありません。代わりに、ピーク時に電力を自動的に増加させる必要なしに、一定のトラフィックフローに耐えることができるシステムについて話しています。

今、私の興味のある分野でのプロジェクトの開発とサポートのために、12レベルからなるテクノロジースタックを使用しています。

▍1。クラウドプロバイダー


私のスタックのベースは、Webプロジェクトが実行される環境の微調整に慣れている人々のニーズを考慮に入れたクラウドプロバイダーです。サポートのコストが高くなるまで、私は自分のサーバーを使用しました。サーバーラックのスペース、専用IPアドレスを借りて、必要な帯域幅を確保します...これらすべてが、物理サーバーの月額コストの原因になります。しかし、本当のランサムウェアは電気代です。クラウドプロバイダーは、私が電力会社に送った1日あたり1.25ドルよりはるかに安価です。そのような経費を拒否することで、私は年間数百ドルを節約することができました。

▍2。SELinuxを使用したFedora Linuxディストリビューション


セキュリティは本当に私たち全員を悩ませるものです。SELinuxは、Linuxで実行される強力なセキュリティシステムと比較できます。これに適切に構成されたiptables-firewallを追加すると、サイトの所有者が夜に安らかに眠ることができるようになります。これらすべてが必要かどうかわからない場合は、次の実験を行ってください。お気に入りのクラウドプロバイダーに新しいサーバーを展開し、攻撃されるまでの時間を監視します。新しいサーバーへのSSHログイン試行によるブルートフォース攻撃が、作成されてから10分以内に開始されることを確認しました。

▍3。Webサーバー読み取り/書き込みサーバー


LetsEncryptからのTLS証明書を備えたRead Write Serve Webサーバーを使用しています。以前はApacheのファンでしたが、新しいWebサイトをセットアップして起動するのに数分しかかかりませんでした。しかし、PHPからJavaScriptに切り替えたので、Apacheを忘れる必要がありました。Expressサーバーは私には非常にシンプルなツールのように見えましたが、Apacheが提供していたすべての機能をそのサーバーで再現しようとするまでしかありませんでした。コンテンツマッチングメカニズム、条件付きキャッシング、データ圧縮、SEO、CORSのURL書き換え、コンテンツ保護ポリシーについて話しています。その結果、Read Write Serveサーバーに切り替えました。サーバーには、これらの機能がすべてデフォルトで存在しています。

▍4。Node.jsアプリケーションランタイム


Node.jsは、サーバーで実行されるアプリケーションロジックを担当します。NPMエコシステムには、あらゆる機会に対応できるパッケージがあるように感じます。したがって、利用可能なパッケージから私が必要とするものをアセンブルし、これらすべてを読み取り/書き込みサーバーで起動するタスクは、単純で理解しやすいことがわかりました。最新のWebプロジェクトに必要なすべての作業を整理するために、過度の努力をする必要はありません。これは、電子メールの送信、支払いサービスの操作、データベースへのアクセスなど、サーバーAPIの操作を意味します。

▍5。MariaDBデータベース


MariaDBデータベースサーバーを使用しています。これは、オープンソースコミュニティによってリブランドおよびマスターされたMySQLのフォークです。非構造化JSONデータを格納する必要がある場合は、PostgreSQLを使用します。これにより、特定のJSONプロパティで直接リクエストを実行できるようになります。これはMongoDBに少し似ていますが、使い慣れたSQL構文に基づいています。

▍6。HTTP / 2


アプリケーションのパーツ間で通信するために、永続的な接続とストリームの多重化を備えたHTTP / 2機能に依存しています。これらは、立派なHTTP / 1.1プロトコルへの2つの追加です。文書形成への私のアプローチを変更しました。まず、キューの先頭をブロックする問題がなくなりました。その結果、小さな画像を何十枚も持っていても、スプライトリストの必要性がなくなりました。次に、JavaScriptとCSSファイルをバンドルに結合して最適化する必要がなくなりました。クライアント/サーバー接続が確立された後、これらすべての小さなファイルは、この接続を介して中断されることなく転送されます。

▍7。ブルーフレーズを使用したHTMLテンプレート


Blue Phraseは、HTML構造をコンパクトな形式で正確に記述できるテンプレートシステムです。私にとっては、HTMLコードのハッシュが判読不能になり、開始タグと終了タグの間に矛盾が生じた時代は終わりました。テンプレートでは、通常、少数の変数（タイトル、説明、キーワード、SEOデータ、ロード画面、日付など）のみを使用し、それらを宣言型のスタイルでテンプレートに配置します。

▍8。Read Write Docを使用したページコードの記述


新しいページを作成するときは、レイアウトではなく、表現しようとしていることに焦点を当てています。この問題を解決するには、Read Write Docを使用します。このツールは、何にも邪魔されることなくビジネスを行うのに役立ちます。私が取り組んでいるものがMediumで公開される予定である（そして優れたオンラインWYSIWYGエディターがある）場合でも、それを使用します。私は自分自身をWeb開発のベテランだと思っているので、等幅フォントに慣れていて、キーボードとマウスの間で急いではなく、キーボードの上に手を置くようにします。いずれにせよ、CSSを適用して作業している内容を確認する必要がある場合は、単純なキーの組み合わせを使用して、表示モードと編集モードを切り替えることができます。

▍9。標準のWebコンポーネント


Webコンポーネントの操作の分野では、W3C標準に準拠しています。これらは、シャドウDOM、ユーザー要素、HTMLタグ<template>、ECMAScriptモジュールです。これにより、NPMを通じて配布するパッケージに必要なすべてのものを完全に組み込むことができます。私にとって、これらすべての最大の利点は、シャドウDOMが提供する分離レベルでした。これにより、CSSの呪い、名前空間の汚染を取り除くことができました。

▍10。クライアントスクリプトのJavaScript


クライアントスクリプトを作成するには、モジュール式のオブジェクト指向JavaScriptコードを使用します。ECMAScript標準の新機能は、最新のブラウザーリリースでサポートされている場合にのみ適用します。つまり、すべての主要なブラウザがcaniuse.comで「グリーン」になるのを確認した時点で、それらを武器に含めています。ポリフィルは避けます。

▍11。CSSスタイリング


CSSはタイポグラフィとページレイアウトです。タイポグラフィは正しいフォントの選択から始まります。私にとって最も重要なことは、テキストの読みやすさです。最近、使用するフォントのファイルを自分のサーバーでホストすることにしました。これにより、一部のサードパーティサービスの制限された帯域幅が原因でブロックされる可能性からリソースが保護されます。たとえば、大まかに次の構文を使用します。

<link href='/fonts/source-serif-pro-400-latin.woff2' rel=preload as=font crossorigin />

このアプローチの追加の利点は、FOUTと呼ばれる問題（スタイルのないテキストのフラッシュ、通常のフォントのフラッシュ）を完全に排除できることです。

▍12。GIMPおよびInkScapeを使用したグラフィックリソースの準備


そして最後に、グラフィックリソースの準備のために、私はいくつかのエディターを使用します。GIMPを使用してビットマップPNG画像を準備し、InkScapeを使用してベクターSVGマテリアルを準備します。
