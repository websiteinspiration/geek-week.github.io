地震データに基づくニューラルネットワークの実験

地震データを解釈することの困難さは、そのようなデータの各セットが一意であるため、各タスクに対して個別のアプローチを模索する必要があるという事実によるものです。手作業による処理には多大な人件費が必要であり、その結果には人的要因に関連するエラーが含まれることがよくあります。解釈にニューラルネットワークを使用すると、手作業を大幅に削減できますが、データの一意性により、この作業の自動化が制限されます。

この記事では、例として北海の完全にマークされたデータを使用して、2D画像の地層の割り当てを自動化するニューラルネットワークの適用性を分析する実験について説明します。


図1.アクアトリアル地震調査（出典）

対象領域について少し


地震探査は、弾性波-地震波を使用して地質学的対象を研究するための地球物理学的手法です。この方法は、地震波の伝播速度が、伝播する地質環境の特性（岩の組成、多孔性、破壊、水分飽和度など）に依存するという事実に基づいています。異なる特性を持つ地層を通過すると、地震波が反射されます。さまざまなオブジェクトとレシーバーに返されます（図1を参照）。それらの性質が記録され、処理後、2次元画像-地震断面、または3次元データ配列-地震立方体を形成できます。


図2.耐震キューブの例（ソース）

立方体の横軸は地表に沿って配置され、縦軸は深度または時間を表します（図2を参照）。場合によっては、キューブは、受振器の軸（いわゆるインライン、インライン）または横（クロスライン、クロスライン、構築線）に沿って垂直セクションに分割されます。各垂直立方体（およびスライス）は、別個の地震トレースです。

したがって、インラインとクロスラインは同じ地震トレースから構成され、順序は異なります。隣接する地震の軌跡は非常によく似ています。フォールトポイントでさらに劇的な変化が発生しますが、類似点はあります。これは、隣接するセクションが互いに非常に類似していることを意味します。

このすべての知識は、実験を計画するときに役立ちます。

解釈タスクとそのソリューションにおけるニューラルネットワークの役割


得られたデータは、立方体または各スライスで岩石の個々の地層とその境界（地平線、地平線）、塩の堆積物、断層、および調査地域の地質構造の他の特徴を直接識別する通訳者によって手動で処理されます。通訳者は、立方体またはスライスを使用して、地層と地平線の手作業による綿密な選択から作業を開始します。各地平線は、カーソルをポイントしてマウスをクリックすることにより、手動で（英語の「picking」-コレクションから）選択する必要があります。


図3. 2Dスライスの例（左）および対応する地層のマーキングの結果（右）（ソース）

主な問題は、ますます困難な地質条件（たとえば、海の深さが深い水中セクション）で得られる地震データの量の増加と、これらのデータの解釈のあいまいさに関連しています。さらに、締め切りが厳しく、ボリュームが大きい状況では、通訳は間違いを犯します。たとえば、地質断面のさまざまな特徴を見落とすことがあります。

この問題は、ニューラルネットワークを使用して部分的に解決でき、手作業を大幅に削減して、解釈プロセスを高速化し、エラーの数を減らします。ニューラルネットワークの操作には、特定の数の既製のラベル付けされたセクション（キューブのセクション）が必要です。その結果、すべてのセクション（またはキューブ全体）の完全なマークアップが得られます。ネットワークが正しく認識できませんでした。

ニューラルネットワークを用いた解釈の問題に多くのソリューションがここにありますが、ほんの一例です：1、2、3。難しさは、各データセットが一意であるという事実にあります-研究領域の地質岩の特殊性のために、生データを既製のものに変えるために使用されるさまざまな方法のために、さまざまな技術的手段と地震探査の方法のために。外部ノイズ（犬の吠え声やその他の大きな音など）がある場合でも、完全に除去できるとは限りません。したがって、各問題は個別に解決する必要があります。

しかし、これにもかかわらず、解釈のさまざまな問題を解決するための個別の一般的なアプローチを模索することを可能にする多くの作品があります。

私たちは、MaritimeAI（ソーシャルグッズの機械学習ODSコミュニティから開発されたプロジェクト、私たちに関する記事）にいます）興味のある分野（海の研究）の各ゾーンについて、公開された作品を研究し、独自の実験を行います。これにより、特定のソリューションの適用の境界と特徴を明確にし、独自のアプローチを見つけることができます。

この記事で説明する1つの実験の結果。

ビジネス研究の目標


データサイエンスの専門家は、図3を見て呼吸が容易になるだけで十分です。これは、多くのニューラルネットワークアーキテクチャと教育方法が発明された、セマンティックイメージセグメンテーションの一般的なタスクです。あなただけの適切なものを選択し、ネットワークを訓練する必要があります。

しかし、それほど単純ではありません。

ニューラルネットワークの助けを借りて良い結果を得るには、学習するためのマークアップ済みのデータをできるだけ多く必要とします。しかし、私たちの仕事は、手作業の量を減らすことです。また、地質構造が大きく異なるため、他の地域のタグ付きデータを使用することはほとんど不可能です。

上記をビジネスの言葉に翻訳します。

ニューラルネットワークの使用を経済的に正当化するためには、主要な手作業による解釈と得られる結果の改良の量を最小限に抑える必要があります。ただし、ネットワークをトレーニングするためのデータを削減すると、結果の品質に悪影響を及ぼします。ニューラルネットワークは、通訳者の作業をスピードアップして促進し、ラベル付き画像の品質を向上させることができますか？それとも通常のプロセスを複雑にするだけですか？

この研究の目的は、ニューラルネットワークのマークアップされた地震の立方体データの最小の十分な量を決定し、得られた結果を評価する試みです。私たちは、手動または部分的に自動化された解釈を決定する際に地震調査の結果の「所有者」を助けるはずである次の質問への回答を見つけようとしました。

  1. ? ?
  2. ? ? – ?


実験では、解釈問題の1つ、つまり、地震立方体の2D断面上の地層を分離するタスクを選択しました（図3を参照）。私たちはすでにこの問題を解決しようとしましたが（ここを参照）、著者によると、ランダムに選択されたスライスの1％で良い結果が得られました。立方体の体積を考えると、これらは16個の画像です。ただし、この記事では比較のための指標を提供しておらず、トレーニング方法論（損失関数、オプティマイザー、学習速度を変更するためのスキームなど）の説明がないため、実験を再現できません。

さらに、私たちの意見では、提示された結果は、提起された質問に対する完全な回答を得るには不十分です。この値は1％で最適ですか？または、別のスライスのサンプルの場合、それは異なるでしょうか？より少ないデータを選択できますか？もっと取ってみる価値はありますか？結果はどのように変化しますか？等。

実験では、北海のオランダのセクターから完全にラベル付けされた同じデータセットを使用しました。ソース地震データは、Open Seismic Repository：Project Netherlands Offshore F3 Blockの Webサイトで入手できます。簡単な説明はSilva et al。にあります。 「オランダのデータセット：地震の解釈における機械学習のための新しい公開データセット。」

私たちのケースでは2Dスライスについて話しているので、元の3Dキューブを使用しませんでしたが、ここで入手可能なオランダのF3 Interpretation Datasetで既に作成されている「スライス」を使用しました。

実験中に、次の問題を解決しました。

  1. ソースデータを確認し、手動マーキングに品質が最も近いスライスを選択しました。
  2. ニューラルネットワークのアーキテクチャ、トレーニングの方法論とパラメータ、トレーニングと検証のためにスライスを選択する原理を記録しました。
  3. 結果を比較するために、20の同じニューラルネットワークを同じタイプのスライスの異なるデータボリュームでトレーニングしました。
  4. 結果を比較するために、異なるタイプのスライスの異なる量のデータで別の20のニューラルネットワークをトレーニングしました。
  5. 予測結果の必要な手動調整の量を推定しました。

このセクションは、まずデータサイエンスのスペシャリストにとって重要であるため、適切な用語を使用します。

インラインとクロスラインは同じ地震トレースで構成されているため、相互に排他的な2つの仮説を立てることができます。

  1. 遅延選択として別のタイプのイメージを使用して、トレーニングは1つのタイプのスライス（たとえば、インライン）でのみ実行できます。これにより、結果をより適切に評価できます。トレーニングで使用された同じタイプの残りのスライスは、トレーニングスライスと同様です。
  2. これは既成の拡張機能であるため、トレーニングには、異なるタイプのスライスを混合して使用することをお勧めします。

見てみな。

さらに、同じタイプの隣接するスライスの類似性と再現可能な結果を​​得たいという要望から、任意の原理ではなく、キューブ全体で均一に、トレーニングと検証のためにスライスを選択するための戦略が導き出されました。スライスが可能な限り離れているため、さまざまなデータを最大限にカバーします。

検証のために、2つのスライスが使用され、トレーニングサンプルの隣接する画像間で均等に分散されました。たとえば、3つのインラインのトレーニングサンプルの場合、検証サンプルは、それぞれ8つのスライスの3つのインラインと3つのクロスラインの4つのインラインで構成されていました。

その結果、2つのシリーズのトレーニングを実施しました。

  2. 3-10 . 40+40 80+80 .

このアプローチでは、トレーニングサンプルと検証サンプルのサイズが大幅に異なるため、比較が困難になることを考慮する必要がありますが、残りの画像のボリュームはそれほど減少していないため、結果の変化の適切な評価に使用できます。

トレーニングサンプルの再トレーニングを減らすために、任意のクロップサイズ448x64を使用してオーグメンテーションを行い、確率0.5で縦軸に沿ってイメージをミラーリングしました。

結果の品質がトレーニングサンプルのスライス数にのみ依存することに関心があるため、画像の前処理は無視できます。変更を加えずに単一レイヤーのPNG画像を使用しました。

同じ理由で、この実験のフレームワーク内では、最適なネットワークアーキテクチャを探す必要はありません。主なことは、すべてのステップで同じであることです。このようなタスクには、シンプルだが実績のあるUNetを選択しました。


図6.ネットワークアーキテクチャ

損失関数は、Jacquard係数とバイナリクロスエントロピーの組み合わせで構成されていました。

def jaccard_loss(y_true, y_pred):
    smoothing = 1.
    intersection = tf.reduce_sum(y_true * y_pred, axis = (1, 2))
    union = tf.reduce_sum(y_true + y_pred, axis = (1, 2))
    jaccard = (intersection + smoothing) / (union - intersection + smoothing)
    return 1. - tf.reduce_mean(jaccard)
def loss(y_true, y_pred):
    return 0.75 * jaccard_loss(y_true, y_pred) + 0.25 * keras.losses.binary_crossentropy(y_true, y_pred)

keras.optimizers.SGD(lr = 0.01, momentum = 0.9, nesterov = True)
keras.callbacks.EarlyStopping(monitor = 'val_loss', patience = 10),
keras.callbacks.ReduceLROnPlateau(monitor = 'val_loss', patience = 5)

各ネットワークは、30〜60エポックのGeForce GTX 1060 6Gbでトレーニングされました。サンプルサイズに応じて、各時代のトレーニングには10〜30秒かかりました。

第1に、Jacquard係数によって測定された、すでに9インラインの予測品質は非常に高い値に達し、その後は増加し続けますが、それほど集中的ではありません。それら。ニューラルネットワークをトレーニングするための少数のラベル付き画像の十分性の仮説が確認されます。

第二に、トレーニングと検証にインラインのみが使用されたという事実にもかかわらず、クロスラインについて非常に高い結果が得られました。1種類のスライスのみの十分性の仮説も確認されています。ただし、最終的な結論として、インラインとクロスラインの混合に関するトレーニングと結果を比較する必要があります。

3番目に、さまざまなレイヤーのメトリック、つまり彼らの認識の質は非常に異なります。これは、たとえば、弱いクラスに重みや追加のネットワークを使用したり、本格的な「1対すべて」のスキームを使用したりして、異なる学習戦略を選択するというアイデアにつながります。

タスク5.手動による調整の量の評価


結果の最終的な結論を得るには、得られたネットワーク予測を手動で調整する量を見積もる必要があります。これを行うために、得られた各予測について、接続されたコンポーネント（つまり、同じ色の塗りつぶされたスポット）の数を決定しました。この値が10の場合、レイヤーは正しく選択されており、最大のマイナーホライズン補正について話していることになります。これ以上ない場合は、画像の小さな領域を「クリーン」にするだけで済みます。それらが大幅に多い場合は、すべてが不良であり、完全な再レイアウトが必要になる場合もあります。

テストでは、110のインラインと360のクロスラインを選択しました。これらは、考慮されたどのネットワークのトレーニングでも使用されていません。

表1.両方のタイプのスライスで平均化された統計


表1は、以前の結果の一部を確認しています。特に、トレーニングに1％のスライスを使用する場合、違いはなく、1種類のスライスまたは両方を使用すると、結果は次のように特徴付けられます。

  • 予測の約10％が理想に近い、つまり 地平線の個々のセクションを調整するだけで済みます。
  • 予測の50％に含まれるスポットは15以下です。追加は5つまで。
  • 予測の75％に含まれるスポットは20以下です。追加は10以下。
  • 残りの25％の予測では、個々のスライスの完全な再設計を含め、さらに大幅な調整が必要です。

最大5％のサンプルサイズの増加により、状況が変わります。特に、セクションの混合でトレーニングされたネットワークは、非常に高い指標を示しますが、コンポーネントの最大値も増加します。これは、非常に低い品質の個別の解釈の出現を示しています。ただし、サンプルを5倍に増やし、スライスの混合を使用する場合：

  • 予測の約30％が理想に近い、つまり 地平線の個々のセクションを調整するだけで済みます。
  • 予測の50％に含まれるスポットは12個以下です。追加は2つまで。
  • 予測の75％に含まれるスポットは14以下です。追加は4つまで。
  • 残りの25％の予測では、個々のスライスの完全な再設計を含め、さらに大幅な調整が必要です。

サンプルサイズをさらに増やしても、結果は改善されません。

一般に、調査したデータキューブでは、ニューラルネットワークから良好な結果を得るには、総データ量の1〜5％で十分であるという結論を導き出すことができます。

そのようなデータによれば、上記のメトリックスおよび図と併せて、ニューラルネットワークを使用して通訳者を支援することの妥当性および専門家が対処する結果について結論を出すことがすでに可能です。

ネットワークを適切に予測するには、スライスの総数の1〜5％を事前にマークしておけば十分です。ボリュームのさらなる増加は、以前にマークされたデータの数の増加に匹敵する結果の改善につながりません。ニューラルネットワークを使用してこのような小さなボリュームでより良いマークアップを取得するには、他のアプローチを試す必要があります。たとえば、アーキテクチャと学習戦略の微調整、画像の前処理などです。

予備マーキングの場合、インラインとクロスラインの両方のタイプのスライスを選択する価値があります。

そのような出力では何が起こりますか？ニューラルネットワークの予測を手動で調整する必要がありますか？もしそうなら、どれほど複雑で膨大ですか？

その結果、そのようなニューラルネットワークでラベル付けされた画像の重要な部分は、個々の不十分に認識されたゾーンの修正からなる、最も重要な改良を必要としません。その中には、修正を必要としないような解釈があります。また、単一の画像の場合のみ、新しい手動レイアウトが必要になる場合があります。

もちろん、学習アルゴリズムとネットワークパラメータを最適化すると、その予測機能を向上させることができます。私たちの実験では、そのような問題の解決策は含まれていません。
