非同期機能


ES2017（ES8）に非同期関数を追加することで、promiseを処理するタスクが簡略化されました。非同期関数はpromiseの上で機能することに注意してください。これらの関数は、質的に異なる概念を表していません。非同期関数は、promiseを使用するコードの代替として考えられました。

Async / Awaitを使用すると、非同期コードを使用した作業を同期スタイルで整理できます。

したがって、約束の知識は、非同期/待機の原則を理解しやすくします。

構文

典型的な状況では、それはasyncとawaitの2つのキーワードで構成されます。最初の単語は関数を非同期にします。これらの関数は待機を許可します。それ以外の場合、この関数を使用するとエラーが発生します。

// With function declaration
 
async function myFn() {
  // await ...
}
 
// With arrow function
 
const myFn = async () => {
  // await ...
}
 
function myFn() {
  // await fn(); (Syntax Error since no async)
}

非同期は、関数宣言の最初に挿入され、矢印関数の場合は、「=」記号と角括弧の間に挿入されます。

これらの関数は、メソッドとしてオブジェクトに配置するか、クラス宣言で使用できます。

// As an object's method
 
const obj = {
  async getName() {
    return fetch('https://www.example.com');
  }
}
 
// In a class
 
class Obj {
  async getResource() {
    return fetch('https://www.example.com');
  }
}

NB！クラスコンストラクターとゲッター/セッターは非同期にできないことを覚えておく価値があります。 Async関数

を実行するためのセマンティクスとルール

は、原則として標準のJS関数に似ていますが、例外があります。

したがって、非同期関数は常にpromiseを返します。

async function fn() {
  return 'hello';
}
fn().then(console.log)
// hello

特に、fnは文字列helloを返します。これは非同期関数であるため、文字列値はコンストラクターを使用してpromiseにラップされます。

以下は、非同期のない代替設計です。

function fn() {
  return Promise.resolve('hello');
}
 
fn().then(console.log);
// hello

この場合、約束の返済は「手動」で行われます。非同期関数は常に新しいプロミスにラップします。

戻り値がプリミティブである場合、async関数は値を返し、promiseでラップします。戻り値がpromiseのオブジェクトである場合、そのソリューションは新しいpromiseに返されます。

const p = Promise.resolve('hello')
p instanceof Promise;
// true
 
Promise.resolve(p) === p;
// true

しかし、非同期関数内でエラーが発生するとどうなりますか？

async function foo() {
  throw Error('bar');
}
 
foo().catch(console.log);

処理されない場合、foo（）はredject付きのpromiseを返します。この状況では、Promise.resolveの代わりに、Promise.rejectはエラーを含んで戻ります。

出力の非同期関数は、何が返されるかに関係なく、常に約束を与えます。

非同期関数は待機するたびに一時停止します。

Awaitは式に影響します。したがって、式がpromiseの場合、async関数はpromiseが実行されるまで中断されます。式がプロミスでない場合、Promise.resolveによってプロミスに変換されて終了します。

// utility function to cause delay
// and get random value
 
const delayAndGetRandom = (ms) => {
  return new Promise(resolve => setTimeout(
    () => {
      const val = Math.trunc(Math.random() * 100);
      resolve(val);
    }, ms
  ));
};
 
async function fn() {
  const a = await 9;
  const b = await delayAndGetRandom(1000);
  const c = await 5;
  await delayAndGetRandom(1000);
 
  return a + b * c;
}
 
// Execute fn
fn().then(console.log);

そして、これがfn関数がどのように機能するかの説明です。

  • 呼び出した後、最初の行はconst a = await 9;から変換されます。in const a = await Promise.resolve（9）;。
  • Awaitを使用した後、その値を取得するまで関数の実行が中断されます（現在の状況では9です）。
  • delayAndGetRandom（1000）は、それ自体が完了するまで（1秒後）、fn関数の実行を一時停止します。これは、実際にはfn関数を1秒間停止しています。
  • delayAndGetRandom（1000）からresolveはランダムな値を返し、それが変数bに割り当てられます。
  • まあ、変数cの場合は変数aの場合と似ています。その後、すべてが1秒間停止しますが、必要がないため、delayAndGetRandom（1000）は何も返しません。
  • その結果、値は式a + b * cによって計算されます。結果は、Promise.resolveを使用してpromiseにラップされ、関数によって返されます。

これらの一時停止はES6のジェネレーターに似ていますが、これには理由があります。

これは、数値を出力するソリューションです。2つのオプションがあります。

const wait = (i, ms) => new Promise(resolve => setTimeout(() => resolve(i), ms));
 
// Implementation One (Using for-loop)
const printNumbers = () => new Promise((resolve) => {
  let pr = Promise.resolve(0);
  for (let i = 1; i <= 10; i += 1) {
    pr = pr.then((val) => {
      console.log(val);
      return wait(i, Math.random() * 1000);
    });
  }
  resolve(pr);
});
 
// Implementation Two (Using Recursion)
 
const printNumbersRecursive = () => {
  return Promise.resolve(0).then(function processNextPromise(i) {
 
    if (i === 10) {
      return undefined;
    }
 
    return wait(i, Math.random() * 1000).then((val) => {
      console.log(val);
      return processNextPromise(i + 1);
    });
  });
};

そして、これは非同期関数を使用したソリューションです。

async function printNumbersUsingAsync() {
  for (let i = 0; i < 10; i++) {
    await wait(i, Math.random() * 1000);
    console.log(i);
  }
}

エラー処理

未処理のエラーは、拒否されたpromiseにラップされます。ただし、非同期関数では、try / catch構文を使用して同期エラー処理を実行できます。

async function canRejectOrReturn() {
  // wait one second
  await new Promise(res => setTimeout(res, 1000));
 
// Reject with ~50% probability
  if (Math.random() > 0.5) {
    throw new Error('Sorry, number too big.')
  }
 
return 'perfect number';
}

canRejectOrReturn（）は、成功する（「完全な数」）か、エラーで失敗する（「申し訳ありません、数が多すぎます」）非同期関数です。

async function foo() {
  try {
    await canRejectOrReturn();
  } catch (e) {
    return 'error caught';
  }
}

上記の例ではcanRejectOrReturnが実行されることが予想されているため、それ自体が異常終了すると、catchブロックが実行されます。その結果、foo関数は、未定義（tryブロックで何も返されない場合）またはエラーをキャッチして終了します。その結果、try / catchはfoo関数自体を処理するため、この関数は失敗しません。

次に別の例を示します。

async function foo() {
  try {
    return canRejectOrReturn();
  } catch (e) {
    return 'error caught';
  }
}

fooからの例ではcanRejectOrReturnが返されるという事実に注意する価値があります。この場合、Fooは完全な数値で完了するか、エラー（「申し訳ありません、数値が大きすぎます」）エラーを返します。catchブロックは実行されません。

問題は、fooがcanRejectOrReturnから渡されたpromiseを返すことです。したがって、foo関数の解はcanRejectOrReturnの解になります。この場合、コードは2行だけで構成されます。

try {
    const promise = canRejectOrReturn();
    return promise;
}

しかし、awaitとreturnを一緒に使用するとどうなりますか。

async function foo() {
  try {
    return await canRejectOrReturn();
  } catch (e) {
    return 'error caught';
  }
}

上のコードでは、fooは完全な数とエラーの両方をキャッチして成功しています。故障はありません。しかし、fooはcanRejectOrReturnで終わり、undefinedではありません。return await canRejectOrReturn（）という行を削除して、これを確認しましょう。

    const value = await canRejectOrReturn();

これはかなり頻繁に起こります-約束の前に、awaitキーワードは忘れられます：

ご覧のとおり、コードには待機も戻りもありません。したがって、fooは常に1秒の遅延なしで未定義で終了します。しかし、約束は果たされるでしょう。エラーまたはredjectが発生した場合は、UnhandledPromiseRejectionWarningが呼び出されます。

const url = 'https://api.github.com/users';

const fetchPublicReposCount = async (username) => {

  const json = await response.json();

}

const users = [

.mapコールバックでAwaitに注意する必要があります。ここでcountsはpromiseの配列で、.mapは指定された各ユーザーの匿名コールバックです。

awaitの過度に一貫した使用

例として、次のコードを見てください。

async function fetchAllCounts(users) {
  const counts = [];
  for (let i = 0; i < users.length; i++) {
    const username = users[i];
    const count = await fetchPublicReposCount(username);
    counts.push(count);
  }
  return counts;
}

ここでは、レポ番号がcount変数に入れられ、次にこの番号がcounts配列に追加されます。コードの問題は、最初のユーザーデータがサーバーから到着するまで、後続のすべてのユーザーがスタンバイモードになることです。したがって、1度に処理されるユーザーは1人だけです。

たとえば、1人のユーザーを処理するのに約300ミリ秒かかる場合、すべてのユーザーにとって、これはすでに1秒です。費やされる時間は、ユーザー数に直線的に依存します。ただし、リポジトリの数を取得することは互いに依存しないため、プロセスを並列化できます。これには.mapとPromise.allでの作業が必要です。

async function fetchAllCounts(users) {
  const promises = users.map(async username => {
    const count = await fetchPublicReposCount(username);
    return count;
  });
  return Promise.all(promises);
}

入力のPromise.allは、Promiseの配列とともにPromiseの配列を受け取ります。配列内のすべてのpromiseが完了した後、または最初のredjectで最後のものが完了します。それらすべてが同時に開始しない場合があります-同時起動を確実にするために、p-mapを使用できます。
