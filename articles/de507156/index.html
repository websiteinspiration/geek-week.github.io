Tetris auf Bitboards: alte Songs auf neue Weise



Bitbordy ( Bitboard ) - spezielle Bitmuster, um die Bewegungen in Brettspielen effektiv zu zählen. Auf einem habr schrieb über die Anwendung von Bitboards auf Schach und sogar auf Dame . Heute werden wir die Technik der Bitboards auf ein etwas unerwartetes, aber bekanntes Spiel anwenden - auf Tetris. Das Ergebnis unserer Forschung wird ein Konsolenspiel sowie eine automatische Suche nach den besten Zügen (für eine bestimmte Folge von Teilen) sein, deren Geschwindigkeit nur durch die Effizienz von Bitmanipulationen sichergestellt wird. Gleichzeitig werden wir die Wiedergabe der gefundenen Züge im automatischen Modus unterstützen, um die Computerintelligenz voll zu genießen. Am Ende des Artikels finden Sie einen Link zum Github mit dem Spielcode in C # sowie ein kurzes Videospiel mit 114 Zügen, die der Computer gefunden hat, indem er fünfzehn Minuten lang eingehend gesucht hat.

Normalerweise ist ein Bitboard ein Maschinenwort, das aus mehreren Bytes besteht, von denen jedes Bit einer Zelle des Feldes im Spiel entspricht. Im Schach gibt es also nur 64 Zellen, was dem 8-Byte-Wort entspricht (ulong in C #), und in checkers - 32 (uint in C #). Fans von Tetris werden sich wahrscheinlich daran erinnern, dass die Größe des Feldes in Standard-Tetris 10 mal 20 Zellen beträgt, dh 200 Bit, was in keinen Zahlentyp passt. Natürlich können Sie das Feld in vier Teile unterteilen und vier Acht-Byte-Wörter verwenden, oder Sie können ein Array von zweiundzwanzig Byte-Wörtern, ein Wort für jede Zeile des Feldes, nicht klein machen und verwenden. Alle Tetris-Implementierungen auf den Bitboards, die ich gefunden habe (in Höhe von einem Stück), tun genau das. Aber wir werden den anderen Weg gehen ...

Vektoren


Wir werden eine spezielle Anleitung Advanced Vector Extensions 2 verwenden , die wir bereits beim letzten Mal verwendet haben , um die Berechnungen zu beschleunigen. Sie arbeiten mit 256-Bit-Werten (im Gegensatz zu den üblichen Anweisungen, die mit 64-Bit-Werten arbeiten):

//  , 8 
ulong 
    x = 0,
    y = 0,
    z = x & y; // z[i] = x[i] & y[i], i = 0..7
// SIMD-, 32 
Vector256<byte> 
    X = Vector256<byte>.Zero,
    Y = Vector256<byte>.Zero,
    Z = Avx2.And(X, Y); // Z[i] = X[i] & Y[i], i = 0..31

Ich muss sagen, dass das Schreiben statischer Methodenaufrufe anstelle von arithmetischen Operationen jedes Mal nicht sehr bequem ist; Darüber hinaus verfügen nicht alle Operationen über direkte Analoga in AVX2. Daher verwenden wir das c # -Power- Tape und schreiben eine spezielle Struktur, die ein 256-Bit-Wort zum Schreiben von Code in diesem Stil enthält:

V256 X = 0, Y = 0, Z = X & Y;

Um den vorherigen Code aussagekräftig zu machen, definieren wir V256 als eine Struktur, in der ein 256-Bit-Datenvektor gespeichert ist und die implizite Konstruktoren und Operatoren überladen:

readonly unsafe struct V256
{
    private readonly Vector256<byte> _data;

    public V256(Vector256<byte> data) { _data = data; }

    //    Vector<256>:
    //  V256 x = new V256(data)    V256 x = data
    public static implicit operator V256(Vector256<byte> data) => new V256(data);

    //     x & y
    public static V256 operator &(V256 x, V256 y) => Avx2.And(x._data, y._data);

    //    :    V256 x = 42;
    public static implicit operator V256(byte b) => Avx2.BroadcastScalarToVector256(&b);

    ...
}

//  readonly,   JIT-  
readonly unsafe struct V256
{
    //  ,    AVX2
    static V256() { if (!Avx2.IsSupported) throw new NotSupportedException("AVX2"); }

    //         , . 
    public static readonly V256 ZERO = 0;

    //  . JIT-       ,
    //         256- 
    //   YMM0 - YMM15
    private readonly Vector256<byte> _data;

    //   32    
    public V256(byte* b) { _data = Avx2.LoadVector256(b); }

    //      32   
    public V256(params byte[] buffer)
    {
        if (buffer.Length < 32) Array.Resize(ref buffer, 32);
        fixed (byte* p = buffer) _data = Avx2.LoadVector256(p);
    }

    //      .  
    //    unmanaged,      T
    //    .    (int, uint, long, ulong,
    //  ..)   
    public static V256 Load<T>(params T[] buffer) where T : unmanaged
    {
        fixed (T* p = buffer) return new V256((byte*)p);
    }

    public void Store<T>(T[] buffer) where T : unmanaged
    {
        fixed (T* p = buffer) Avx2.Store((byte*)p, _data);
    }

    public T[] ToArray<T>() where T : unmanaged
    {
        T[] array = new T[32 / sizeof(T)];
        fixed (T* ptr = array) Avx2.Store((byte*)ptr, _data);
        return array;
    }

    //   broadcast     
    //  ,   32   b
    public static V256 FromByte(byte b) =>
        Avx2.BroadcastScalarToVector256(&b);
    //  16  short- s
    public static V256 FromShort(short s) =>
        Avx2.BroadcastScalarToVector256(&s).AsByte();
    //      
    public static V256 FromULong(ulong ul) =>
        Avx2.BroadcastScalarToVector256(&ul).AsByte();

    //    :    V256 x = 42;
    public static implicit operator V256(byte b) => FromByte(b);

    //   AVX2
    //     ,  result[i] = 
    // Math.Min(this[i], other[i]), i=0..31,   100  
    public V256 Min(V256 other) => Avx2.Min(_data, other._data);
    //  table lookup: result[i] = this[data[i]], i = 0..31
    public V256 Shuffle(V256 mask) => Avx2.Shuffle(_data, mask._data);
    //  result[i] = this[i] == other[i] ? 0xFF : 0x00;
    public V256 CompareEqual(V256 other) => Avx2.CompareEqual(_data, other._data);
    
    //     x & y, x | y  .. 
    public static V256 operator &(V256 x, V256 y) => Avx2.And(x._data, y._data);
    public static V256 operator |(V256 x, V256 y) => Avx2.Or(x._data, y._data);
    public static V256 operator +(V256 x, V256 y) => Avx2.Add(x._data, y._data);
    public static V256 operator -(V256 x, V256 y) => Avx2.Subtract(x._data, y._data);
    public static V256 operator -(V256 x) => Avx2.Subtract(ZERO._data, x._data);

    //      256-   .
    //      AVX2  AVX512,   
    //   ,    :
    public static bool operator ==(V256 x, V256 y) =>
        -1 == Avx2.MoveMask(Avx2.CompareEqual(x._data, y._data));

    //    -  .  ,    
    //  256-  (   AVX512,  ,  !), 
    // :
    public static V256 operator >>(V256 x, int count)
    {
        //  ,  4  8- 
        Vector256<ulong> data = Avx2.ShiftRightLogical( 
            x._data.AsUInt64(), (byte)count);
        //        
        Vector256<ulong> carry = Avx2.ShiftLeftLogical(
            x._data.AsUInt64(), (byte)(64 - count));
        // ...
        carry = Avx2.Permute4x64(carry, 0x39);
        //   !
        carry = Avx2.Blend(ZERO._data.AsUInt32(), carry.AsUInt32(), 0x3F).AsUInt64();
        data = Avx2.Or(data, carry);
        return data.AsByte();
    }

    //   ,   ,  .
    //       :   
    //   ,     ,
    //      .
    public override string ToString()
    {
        byte* buffer = stackalloc byte[32];
        Avx2.Store(buffer, _data);
        return string.Join(" ", Enumerable.Range(0, 32).Select(i => buffer[i].ToString("X2")));
    }
}

Zuerst müssen wir den Schnittpunkt der gefüllten Zellen des Feldes und der Form bestimmen, was in einer logischen Operation erfolgt:

bool Intersects(V256 board, V256 piece) => (piece & board) != 0;

Das Verschieben einer Figur nach unten, links oder rechts ist einfach eine logische Verschiebung um die entsprechende Anzahl von Bits (beachten Sie, dass das Verschieben einer Figur nach rechts eine Verschiebung ihrer Bits in Richtung der Zunahme ist, was einer logischen Verschiebung der Bits des Vektors nach links entspricht):

const int width = 12; //    

V256 MoveDown(V256 piece) => piece <<= width;

V256 MoveRight(V256 piece) => piece <<= 1;

V256 MoveLeft(V256 piece) => piece >>= 1;

//      ,

V256 Drop(V256 piece)

    while(!Intersects(board, MoveDown(piece)) piece = MoveDown(piece);

V256 fullLine = new V256(0, 0, 0, ..., 0, 0, 255, 15);
V256 emptyLine = new V256(1, 8);

void RemoveFullLines(ref V256 board)
{
    //     
    V256 lineMask = fullLine >> width;
    //      
    V256 underLineMask = fullLine;
    while (lineMask != 0)
    {
        //  ,   
        while ((board & lineMask) == lineMask)
        {
            //        
            board = (board & underLineMask) | 
                        ((board & ~underLineMask & ~lineMask) << width) | 
                        emptyLine;
        }
        //      
        underLineMask |= lineMask;
        lineMask >>= width;
    }
}

Suche nach der besten Strategie


Für das Spiel wäre es eigentlich nicht sinnvoll, diese kleinen Tricks zu fechten. Aber um die optimale Strategie zu finden - das war's! Machen wir die einfachste Suchoption - eine rekursive Suche in der Tiefe durch umfassende Suche: Da die Reihenfolge der herausfallenden Figuren festgelegt ist, sortieren wir für jede Figur alle möglichen Varianten ihrer Rotationen und horizontalen Verschiebungen. Parallel dazu speichern wir die Abfolge der Aktionen, damit wir die gefundenen Optionen reproduzieren können. Der vereinfachte Code ist ziemlich einfach:

void Solve() => Solve(Tetris.board, 0, 0);

void Solve(V256 board, int step, int score)
{
    //

        for(int i = 1; i <= 5; i++)

        }
    }
}
