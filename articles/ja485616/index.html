プロジェクトAngularの外部に静的ファイルを追加する

みなさん、こんにちは。連絡先Omelnitsky Sergey。本日は、元の角度プロジェクトのルートの外にある静的コンテンツへのアクセスを提供する場合の対処方法をお伝えします。



angular, angular.json. assets . :


"assets": [
  "src/favicon.ico",
  "src/assets"
],

, , . — ? assets, — An unhandled exception occurred: The configs asset path must start with the project source root.


assets, : . :


  • glob: . input.
  • input：ワークスペースのルートからの相対パス。
  • output：結果のディレクトリへの相対パス（デフォルトはdist /プロジェクト名）。
  • ignore：globsテンプレートの除外リスト。

私たちassetsを新しい外観に変えましょう：


"assets": [
  { "glob": "**/*", "input": "src/assets/", "output": "/assets/" },
  { "glob": "favicon.ico", "input": "src/", "output": "/" }
]

次に/common、プロジェクトの外部にあるフォルダーを追加します。彼女にアクセスを許可する/tmp/**/*：


"assets": [
  "src/favicon.ico",
  "src/assets",
  {"glob": "**/*", "input": "./common", "output": "/tmp/" }
],

まあ、それだけです。これの詳細については、角の公式ドキュメントをご覧ください。
