現在、機能的パラダイム（FP）の周りには多くの誇大宣伝があります。ただし、すべての問題の万能薬ではなく、長所と短所もあります。

機能的パラダイムとは何かについて簡単に


関数型パラダイムは、数学からプログラミングに移りました。プログラムの次の要件を形成します。

  • 既存のデータに変更はありません。
  • 隠し状態はありません。

これは私たちに何を与えますか？

私たちの機能は、サードパーティの影響なしで機能します。つまり、関数は値を返すだけで、外部データには影響を与えません。

純粋な機能を使用する。入力データに関係なく、関数の再テストの信頼性が向上します。つまり、プログラムのテストの信頼性が高まり、結果がより予測可能になります。

したがって、Golangには、機能的パラダイムを実装する機会があります。

クロージャーとカレー機能


現代の多くのプログラミング言語には短絡があります。クロージャーは、親関数のフリースコープ変数を参照する関数です。関数カリー化とは、関数をビューからビューfunc(a,b,c)に変更することfunc(a)(b)(c)です。

Goでのクロージャーとカレーの例を次に示します。

// 
func multiply(x int) func(y int) int { //   
	return func(y int) int { //   ,       JS
		return x * y
	}
}

func main() {
	//    
	var mult10 = multiply(10)
	var mult15 = multiply(15)

	fmt.Println(mult10(5)) //50
	fmt.Println(mult15(15))//225
}


純粋な機能


前に述べたように、純粋な関数とは、入力された引数にのみ関連付けられ、グローバルな状態には影響を与えない値を返す関数です。

失敗したダーティ関数の例を次に示します。

var arrToSave = map[string]int{} //map -    -   Golang

func dirtySum(a, b int) int {
	c := a + b
	arrToSave[fmt.Sprintf("%d", a, b)] = c
	//   ,  "%d" -      
	return c
}

ここで、関数は可能な限り予測どおりに機能することを受け入れる必要があります。

func simpleSum(x, y int) int {
	return x + y
}

func main() {
	fmt.Printf("%v", dirtySum(13, 12))

	//     
	//   ""     
	fmt.Printf("%v", simpleSum(13, 12))
}

再帰


機能パラダイムでは、による単純な反復を使用する代わりに、再帰を優先する-純粋さと透明性のために慣習がありますfor。

命令型と宣言型のパラダイムを使用して階乗を計算する例を次に示します。

func funcFactorial(num int) int {
	if num == 0 {
		return 1
	}
	return num * funcFactorial(num-1)
}

func imperativeFactorial(num int) int {
	var result int = 1
	for ; num > 0; num-- { //    for
		result *= num
	}
	return result
}

func main() {
	fmt.Println(funcFactorial(20)) //       
	fmt.Println(imperativeFactorial(20)) //     
}


現在、再帰関数は非常に非効率的です。計算速度を最適化するために、少し書き直してみましょう。

func factTailRec(num int) int {
	return factorial(1, num) //    "" 
}

func factorial(accumulator, val int) int {
	if val == 1 {
		return accumulator
	}
	return factorial(accumulator*val, val-1)
}

func main() {
	fmt.Println(factTailRec(20)) // 2432902008176640000
}


階乗計算速度がわずかに向上しました。ベンチマークは行いません）。

残念ながら、Goはそのままでは再帰の最適化を実装していないため、再帰の末尾を自分で最適化する必要があります。間違いなく、このトピックに関する有用なライブラリは確かに見つけることができます。たとえば、このトピックには「GoashのLoadash」というすばらしいものがあります。
