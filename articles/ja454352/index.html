Pythonの数学的不確実性

みなさん、こんにちは！私たちはしばしば、お気に入りのプログラミング言語で使い慣れた数字を使用します。習慣性は1、-1.5、pi、または複雑です。しかし、NaN、+ Inf、-Infは、あまり使用されません。

注：

この記事では、Pythonバージョン3.6.8の基本関数を使用していますが、JSなどの他の言語にも多くの関連があります。

数学にはかなり多くの不確実な状況があります。たとえば、これは0で除算され
ます。コンソールに入力すると

1/0

ZeroDivisionErrorエラーが発生します。予想通りですか？はい。別の例を試してみましょう：

0**0

ご存じのように、0の0乗は、古典的な数学的不確実性です。
しかし、Pythonはこれは1であると述べています。合理的な質問が発生します。



「足で撃つ」を0で除算することが許可されないのはなぜですか。

たぶん、誰かが今憤慨して、答えが単純であると叫ぶ：IEEE754。そして、私が間違っていなければ、0の数が1

であることに同意したのは彼の助けでした。ここで、最も興味深いものが始まります。JavaScriptはIEEE（0.1 + 0.2！= 0.3）もサポートしていますが、その1/0は無限大であり、0による除算エラーではありません。

その結果、次のようになります（Pythonのすべての例）：

1.不正なアクション中にエラーがスローされました

例：

1/0 # ZeroDivisionError
log(0) #ValueError

2.不正な値を返す

例：

0**0  #1
1**inf #1.0

3.特別な値を返す：

+ -Inf、nan

例：

inf=float('inf')
inf-inf #nan

私の意見では、これは混乱を招きます。

ちょうど今日、PHPの興味深い動作に気づきました。

var_dump(mt_rand(INF,INF)); //  INF  INF - NULL??
//        false.

私は2つのオプションに断固として反対し、おそらく1をサポートします

。PHPの例が示すように、infとnanは慎重に使用する必要があるためです。どう思いますか？
