Fitur Baru di Python 3.9

Ulasan fitur terbaik yang disertakan dalam iterasi Python terbaru.



Waktunya telah tiba, rilis versi baru Python tidak bisa dihindari. Sekarang dalam versi beta (3.9.0b3), tetapi segera kita akan melihat versi lengkap Python 3.9.

Beberapa fitur terbaru sangat menarik, dan akan luar biasa melihat penggunaannya setelah rilis. Kami akan mempertimbangkan hal berikut:

  • Operator Kombinasi Kamus
  • Ketik mengisyaratkan
  • Dua metode string baru
  • Parser Python Baru sangat keren

Pertama mari kita lihat fitur-fitur baru dan bagaimana kita akan menggunakannya.

Kombinasi Kamus


Salah satu fitur favorit saya yang baru dan sudah dengan sintaks. Jika kami memiliki dua kamus adan bkami perlu menggabungkan, kami sekarang menggunakan operator serikat.

Kami memiliki operator gabungan " |":

a = {1: 'a', 2: 'b', 3: 'c'}
b = {4: 'd', 5: 'e'}
c = a | b
print(c)

[Out]: {1: 'a', 2: 'b', 3: 'c', 4: 'd', 5: 'e'}

Dan pernyataan pembaruan " |=", yang memperbarui kamus sumber:

a = {1: 'a', 2: 'b', 3: 'c'}
b = {4: 'd', 5: 'e'}
a |= b
print(a)

[Out]: {1: 'a', 2: 'b', 3: 'c', 4: 'd', 5: 'e'}

Jika kamus kami memiliki kunci umum , pasangan nilai kunci dari kamus kedua akan digunakan:

a = {1: 'a', 2: 'b', 3: 'c', 6: 'in both'}
b = {4: 'd', 5: 'e', 6: 'but different'}
print(a | b)

[Out]: {1: 'a', 2: 'b', 3: 'c', 6: 'but different', 4: 'd', 5: 'e'}

Pembaruan kamus menggunakan iterasi


Perilaku menarik lainnya dari operator " |=" adalah kemampuan untuk memperbarui kamus dengan pasangan nilai kunci baru menggunakan objek iteratif - misalnya, daftar atau generator:

a = {'a': 'one', 'b': 'two'}
b = ((i, i**2) for i in range(3))
a |= b
print(a)

[Out]: {'a': 'one', 'b': 'two', 0: 0, 1: 1, 2: 4}

Jika kami mencoba mengulangi hal yang sama dengan operator serikat standar " |" yang kami dapatkan TypeError, karena itu hanya akan memungkinkan penggabungan antar tipe dict.

Metode String


Tidak sehebat fitur-fitur baru lainnya, tetapi layak disebutkan karena mungkin bermanfaat. Dua metode string baru telah ditambahkan untuk menghapus awalan dan sufiks:

"Hello world".removeprefix("He")

[Out]: "llo world"

Hello world".removesuffix("ld")

[Out]: "Hello wor"

Parser baru


Ini lebih dari perubahan tersembunyi, tetapi mungkin salah satu perubahan paling signifikan untuk pengembangan Python di masa depan.

Python saat ini menggunakan tata bahasa terutama berdasarkan LL (1), yang pada gilirannya dapat diuraikan oleh parser LL (1), yang mem-parsing kode dari atas ke bawah, dari kiri ke kanan, dengan kemampuan untuk melihat hanya satu token.

Saya hampir tidak tahu bagaimana ini bekerja, tetapi saya dapat memberi tahu Anda tentang beberapa masalah aktual dalam Python karena penggunaan metode ini:

  • Python mengandung tata bahasa non-LL (1); karena itu, beberapa bagian dari tata bahasa saat ini menggunakan solusi, menciptakan kompleksitas yang tidak perlu.
