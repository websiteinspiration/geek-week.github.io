ご想像のとおり、このような構造をセットアップした後、分散環境では、集中サービスがクライアントのグループの計算ロジックの管理を開始できます。



コード化可能な構造


それでは、どうすればよいでしょうか。最初に、シリアライズおよびデシリアライズ可能な構造を開発します。SwiftプロトコルでCodable は、JSONEncoderおよびを使用してこれを行うことができますJSONDecoder。さらに、構造体は、ストリーム内の値の最小単位（複雑な演算子のチェーンまで）のデータと動作を正しく表す必要があります。

これから作成する構造に必要なコンポーネントを理解する前に、前の一連の投稿で作成したメインストリームを思い出してみましょう。

数字の流れ




これが最も簡単なストリームです。ただし、さらに深く見ると、これが配列のシーケンスではないことに気付くでしょう。ラウンドブロックのそれぞれに独自の遅延演算子（delay）があり、送信する実際の時間を決定します。Combineの各値は次のようになります。

Just(value).delay(for: .seconds(1), scheduler: DispatchQueue.main)

そして一般的に、それはすべてこのように見えます：

let val1 = Just(1).delay(for: .seconds(1), scheduler:   DispatchQueue.main)
let val2 = Just(2).delay(for: .seconds(1), scheduler: DispatchQueue.main)
let val3 = ....
let val4 = ....
let publisher = val1.append(val2).append(val3).append(val4)

各値は1秒間遅延され、同じステートメントが次の値に追加されますdelay。

したがって、観察から2つのことを学びます。

  1. ストリームは構造体の最小単位ではありません。最小値はストリームの値です。
  2. 各ストリーム値には、いつどの値を送信するかを制御する無制限の演算子を含めることができます。

StreamItemを作成する


ストリームとその演算子の値は最小単位なので、その構造を作成することから始めます。彼女に電話しましょうStreamItem。

struct StreamItem<T: Codable>: Codable {
 let value: T
 var operators: [Operator]
}

StreamItemストリームの値と演算子の配列が含まれます。我々の要求によると、我々は両方のような構造で、すべてを保存できるようにしたいvalue、とStreamItemプロトコルに準拠しますCodable。

ストリームの値は、あらゆるタイプの値に対応するためにユニバーサルでなければなりません。

StreamModelを作成する


オペレーターの構造については後で説明します。配列StreamItemをに接続しましょうStreamModel。

struct StreamModel<T: Codable>: Codable, Identifiable {
 var id: UUID
 var name: String?
 var description: String?
 var stream: [StreamItem<T>]
}

StreamModelの配列が含まれていますStreamItem。StreamModel識別子、名前、説明のプロパティもあります。繰り返しになりますが、すべてのものがStreamModelストレージと配布のためにコード化可能である必要があります。

オペレーター構造を作成する


前述したように、オペレーターdelayは送信時間を変更できますStreamItem。

enum Operator {
 case delay(seconds: Double)
}

遅延時間を格納するために 、演算子を1つの関連付けられた値を持つdelay列挙（enum）と見なします。

もちろん、列挙Operatorもに一致する必要がありCodableます。これには、関連する値のエンコードとデコードが含まれます。以下の完全な実装を参照してください。

enum Operator {
    case delay(seconds: Double)
}

extension Operator: Codable {

    enum CodingKeys: CodingKey {
        case delay
    }

    struct DelayParameters: Codable {
        let seconds: Double
    }

    enum CodingError: Error { case decoding(String) }

    init(from decoder: Decoder) throws {
        let container = try decoder.container(keyedBy: CodingKeys.self)
        if let delayParameters = try? container.decodeIfPresent(DelayParameters.self, forKey: .delay) {
            self = .delay(seconds: delayParameters.seconds)
            return
        }
        throw CodingError.decoding("Decoding Failed. \(dump(container))")
    }

    func encode(to encoder: Encoder) throws {
        var container = encoder.container(keyedBy: CodingKeys.self)
        switch self {
        case .delay(let seconds):
            try container.encode(DelayParameters(seconds: seconds), forKey: .delay)
        }
    }

}

これで、この順次ストリームを表す適切な構造が得られました。これは、2番目の遅延間隔で1から4までの値を生成します。

operators: [.delay(seconds: 1)]) }

description: nil, stream: streamA)

StreamModelをパブリッシャーに変換


これでストリームのインスタンスが作成されました。しかし、それをパブリッシャーに変換しないと、すべてが無意味になります。やってみよう。

まず、各オペレーターモデルは実際のCombineオペレーターを参照します。このオペレーターは、このパブリッシャーに追加して、操作されたパブリッシャーに返す必要があります。

extension Operator {
func applyPublisher<T>(_ publisher: AnyPublisher<T, Never>) -> AnyPublisher<T, Never> {
  switch self {
    case .delay(let seconds):
    return publisher.delay(for: .seconds(seconds), scheduler: DispatchQueue.main).eraseToAnyPublisher()
  }
 }
}

現在、演算子のタイプは1つだけdelayです- 。今後はさらに追加していきます。

これで、すべての人にパブリッシャーを使用できるようになりましたStreamItem。

extension StreamItem {
 func toPublisher() -> AnyPublisher<T, Never> {
   var publisher: AnyPublisher<T, Never> =
                  Just(value).eraseToAnyPublisher()
   self.operators.forEach {
      publisher = $0.applyPublisher(publisher)
   }
  return publisher
}
}

値から始めJust、メソッドを使用して一般化しeraseToAnyPublisher、次に、関連するすべての演算子のパブリッシャーを使用します。

レベルStreamModelでは、ストリーム全体のパブリッシャーを取得します。

extension StreamModel {
 func toPublisher() -> AnyPublisher<T, Never> {
   let intervalPublishers =
        self.stream.map { $0.toPublisher() }
   var publisher: AnyPublisher<T, Never>?
   for intervalPublisher in intervalPublishers {
     if publisher == nil {
       publisher = intervalPublisher
       continue
     }
     publisher =
        publisher?.append(intervalPublisher).eraseToAnyPublisher()
   }
   return publisher ?? Empty().eraseToAnyPublisher()
 }
}

ご想像どおり、この方法を使用appendしてパブリッシャーを結合します。
