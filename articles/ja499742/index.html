サービス拒否の種類：DDoSの進行状況

古き良きDDoS ...サイバー犯罪のこのセグメントがどのように変化しているかを毎年分析し、今年の最後と最初の結果によると、そのような攻撃の数は1.5倍以上に増加していることがわかります。この間、攻撃者は攻撃力を大幅に高め、戦術を変更しました。さらに、DDoSはさらにアクセスしやすくなりました。教育リソースやさまざまな「電子日記」への攻撃の数が最近急激に増加していることから判断して、学校のハッカーはDDoSの世界をますます発見しています。この投稿では、2019年と2020年の初めに記録したDDoS攻撃と、最近のDDoSの変化について説明します。



高速かつ強力


ハッカーはスピードアップしました。数日間続く低電力の攻撃は、今や彼らにとって興味のないものです。たとえば、レポート期間で最長の攻撃は約1日続きました（1年前には、攻撃者はサーバーを11日間と16時間「拷問」しました）。

もちろん、攻撃者はグリップを緩めませんでした-期間とは対照的に、DDoS攻撃の威力は増加しました。過去の期間で最も強力な攻撃は、405ギガビット/秒の強度で実行されました。もちろん、これはRostelecomネットワークに対する2018年モデルの記録的な攻撃（450 Gbit / s）よりわずかに低いですが、平均して、DDoS電力インジケーターは依然として増加しています。ハッカーの戦術の変化は、攻撃中に、貴重な資産であるボットネットが防御側によって明らかにされる可能性があるという事実によって説明できます。攻撃を無力化すると、情報交換の一環として他のオペレーターがボットネットアドレスデータを利用できるようになり、ボットネットは適切ではなくなります。強力なリクエストの波で被害者をすばやく攻撃する場合、DDoS対策サービスにブラックリストを埋める時間はありません。

DDoSテクノロジー-アーセナルNew


スプリアストラフィックの爆発を伴うスプリントを実行するには、優れたテクニカルベースが必要です。そして最近、攻撃者はそれをうまくポンプしました。私たちは、DDoS攻撃で積極的に使用され始めた新しい強力なアンプの出現に注目しました。

まず、これはUDPベースのApple Remote Desktopプロトコル（UDPポート3283）であり、その増幅率は35.5：1（Netscoutデータ）です。Apple Remote Desktopは、macOSワークステーションのリモート管理アプリケーションです。ARMS（Appleのリモート管理サービス）は、ファイアウォールのセキュリティ設定と競合していても、リモート管理モードが有効になっている場合、macOSを実行しているワークステーションで常に開いているUDPポートにアクセスします。現在、UDPポートが開いているmacOSを実行している16千台を超えるコンピューターがグローバルネットワークに登録されています。

また、サイバー犯罪者は、IoTソリューションで使用されているWS-Discovery（Web Services Dynamic Discovery）デバイス検出プロトコルを積極的に使用しています。その増幅係数は500です：1。 Apple Remote Desktopと同様に、WS-DiscoveryはUDPベースのプロトコルを使用してパケットを送信します。これにより、ハッカーはパケットスプーフィング（IPアドレススプーフィング）を使用できます。 BinaryEdgeは、インターネット上のこの脆弱性のある約63万のデバイスをカウントしました（2019年9月のデータ）。利用して-私は欲しくない！

DDoSの主要なツールの1つであるIoTデバイスからのボットネットを忘れないでください。私たちが記録した最も強力なスマート攻撃は2019年でした：178 Mppsが賭博会社の1つにヒットしました。攻撃者は、8000台の実際のデバイス（ホームルーター、カメラ、その他のIoT、携帯電話など）のボットネットを使用しました。プレッシャーを逃れたくありませんが、5GとIPv6が新しい「記録」をもたらすだろうという明確な気持ちがあります。

最も人気のある攻撃タイプ


一般に、DDoS攻撃の最も一般的な方法は依然としてUDPフラッドであり、攻撃の総量の約32％です。 2番目と3番目の場所-SYNフラッド（27.4％）と断片化されたパケットによる攻撃（14.2％）。リーダーからそれほど遠くないところにDNS増幅攻撃があります（13.2％）。

攻撃者は、現在ほとんど使用されていないプロトコルである16（CHAOS）および111（IPX over IP）も実験しています。ただし、このような攻撃は例外である可能性が高くなります。これらのプロトコルは実際のクライアントでは使用されないため、攻撃を抑制すれば簡単に検出して削除できます。

昨年の異常な攻撃の中で、いわゆる「カーペット爆撃」に注目することができます。それらは、被害者のアドレス空間で標的を破壊し、それらのうち最も脆弱なものを探すことを特徴とします。これにより、インターネットインフラストラクチャ全体ではなく個々のIPアドレスのみが保護される場合、IPごとの保護モデルでのこのような攻撃の検出と対応が複雑になります。私たちが記録した最も大量の「カーペット爆撃」には、ほぼ3,000のターゲットが含まれていました。1つのクライアントのすべてのネットワークの各アドレスが試行されました。

最近、攻撃者はさまざまな種類の攻撃を組み合わせ始めました。このようなDDoSカクテルの例は、SYN + ACKリフレクションです。このタイプの攻撃では、サードパーティのパブリックリソースが使用され、SYNパケットは偽の犠牲者アドレスをソースとして送信されます。サードパーティのサーバーは、SYN + ACKパケットで、TCPスタックがセッションの外部に到着したパケットの処理に苦しんでいる被害者に応答します。被害者と反映に使用されたサードパーティのリソースの両方であるお客様が私たちの顧客である状況を観察しました。前者の場合はSYN + ACKリフレクションのように見え、後者の場合はSYNフラッドのように見えました。

新旧の犠牲者


私たちの方法論についての最初の2つの言葉。この調査は、Rostelecom Anti-DDoSサービスの顧客に対する攻撃に関するデータに基づいているため、最も攻撃されている業界を特定する場合、2つの分析スライスを形成します。

  1. 業界ごとの記録された攻撃の総数の単純な分布、
  2. 業界クライアントあたりの攻撃数の平均増加。このインジケーターは、特定の業界からの顧客数の増加（したがって、記録された攻撃の数の増加）に関連する潜在的なエラーを排除します。

だから私たちは何を見てきました。

業界別


上記のように、DDoS攻撃の数は増加しています。多くの重要なストレッサーの閉鎖、実際にはDDoSを整理するためのサービス（Quantum Stresser、ExoStress.in、QuezStresser.comなど）でさえ、このダイナミクスには影響しません。 DDoSはまだ安価で簡単です。攻撃のピークは、オンラインストアがブラックフライデーの準備をしている2019年10月に発生しました。当時のDDoSは、「最大90％割引」と同じ伝統になりました。同時に、大規模市場のプレーヤーは、マルチベクターターゲット攻撃を使用して攻撃されます-確かに。

ゲーム業界はDDoSのリーダーであり続けており、攻撃の総数の34％を占めています（2018年は64％）。これは、ハッカーがゲームへの関心を失ったという意味ではありません。セグメントへの攻撃の数は減っていませんが、新たな被害者の出現により、この業界のシェアは「低下」しています。

それにもかかわらず、eSportsは全体としては一口にとどまります。ゲームプロトコルは、トランスポートとしてUDPを使用します。これにより、送信者アドレスを置き換えるだけでなく、セッションの追跡が困難になります。レポート期間中に、ゲームサーバーに対する2つの主要なタイプのアプリケーションレベルのDDoS攻撃が確認されました。 1つ目は、サードパーティの保護サービスの場合は正当なパッケージのように見える多数のパッケージを送信することで、アクセス不能を実現しています。このような攻撃に対する保護は、正規のトラフィックのプロファイリングと、正規表現やチャレンジ/レスポンスなどの手段に基づいています。

2番目のタイプは、ゲームプロトコルおよびプラットフォームで特定された脆弱性の悪用に関連しています。この場合、プロアクティブな保護には、異なるクラスのデバイス（ゲームアプリケーションファイアウォール）が必要です。大規模なゲームホストやメーカーは、クライアントプログラムにさまざまなチェックを組み込んでサーバーを保護し、独自に開発した保護システムと関連付けます。

通信会社の人気は2番目で、攻撃の総量に占める割合は10％から31％に大幅に増加しました。原則として、これらは小規模な地域ISP、さまざまなホスティングおよびデータセンターであり、強力な攻撃を撃退するためのリソースを持たないため、ハッカーにとって簡単に被害者になります。

通信に加えて、攻撃者は教育機関や公共部門にも注目を集めました。以前は、DDoS攻撃の総量に占める割合はそれぞれ1％と2％でしたが、年間を通じて、各セグメントで5％に増加しました。これは、デジタル化と独自のインターネットリソースの立ち上げに起因すると考えられます。このような組織の活動は、特に自己分離の期間中にますます依存しています。

クライアントあたりの攻撃


クライアントあたりの攻撃数の平均的な増加は大幅に加速しています。2018年に21％だった場合、2019年と2020年の初めの結果によれば、58％に達しました。クライアントあたりの攻撃数の最大の増加は、教育機関で153％でした。これらは、電子日記、課題やクイズのあるサイト-学生が人生を楽しむのを妨げるすべてのものです。このような攻撃の開始者は、多くの場合、母親のハッカーである生徒自身である可能性があります。これは、たとえば、学校のサイトなどの安全でないリソースでDDoSを整理する単純さを証明しています。

何をすべきか


DDoS攻撃の数は増加し、新しいアンプ、攻撃の種類、攻撃者の当然の新しいターゲットが出現します。しかし、ニュートンの第三法が言うように、すべての 
行動には反対があります。

DDoS防止方法は既知です。この問題に最終的に対処したいという要望があります。 IoTボットネットを生成しないようにするには、すべてのデバイス、ポートを制御し、IPv6を使用しないようにして、脆弱性をタイムリーに排除します。

急速に変化する攻撃ベクトルの状況で保護を適切に構成するには：事前に準備されたオプションを使用して、特定の種類の既知の攻撃に対抗し、インフラストラクチャを定期的にチェックします。

一般に、DDoSの分野における現在の問題は何らかの形で相互に関連していることがわかります。したがって、フィルタリングのさまざまな段階で、アプリケーションとインフラストラクチャ、特にビジネスクリティカルなセグメントを包括的に保護する必要があります。
