Clean Swift Architectureのワーカー

読者こんにちは！

以前、VIPサイクルがどのように配置されているか、価格とデータ転送の間でどのように移行するかを検討しました。次に、過剰なロジックからInteractorをアンロードし、その一部を取り出して他のシーンで再利用する方法を理解する必要があります。そして労働者はこれで私たちを助けます。トピックはボリュームはかなり控えめですが、言及することが重要です。



理論


ワーカーは、Interactorのビジネスロジックの一部を引き継ぐことを目的とする補助クラス/構造（サービスやヘルパーと混同しないでください）です。Interactorのメソッドが拡張されている場合は、Workerの膨大なロジックを削除する番です。でクリーンスウィフト、それらはデータストレージの論理に、ネットワークと、アプリケーションの個々の層であり、上のような作業に使用されています。つまり、すべてが膨大で低レベルであり、アプリケーションのビジネスロジックとは関係ありません。

労働者は主に2つのタイプに分けられます。

  1. シーン内でのみ使用されるローカル。
  2. どのシーンでも使用できるグローバル。

ローカルワーカーはシーン内に直接配置され、彼女にちなんで名前が付けられます-SceneNameWorker。グローバル労働者は、のルートディレクトリに配置されている労働者と命名されている応じてその対象に。ローカルワーカーは、シーンのニーズに合わせた拡張ロジックを使用して、「グローバル全体の装飾」としても機能します。

プロジェクトがレイヤー（プレゼンテーション、ビジネスロジック、永続性、ネットワークロジック）に分割されている場合、プレゼンテーションレイヤーとビジネスロジック間の接続ブリッジの役割をワーカーに割り当てることができます。。したがって、Interactorをアンロードし、より予測可能な動作と再利用の容易さを実現します。

練習


作業労働者は、私たちは、ネットワークでの作業の例について検討します。 2つの画面があります-最初にユーザーのリストが表示され、2番目にこのユーザーの投稿のリストが表示されます。すべてのデータはAPIによって取得されます。このタスクを3つのワーカーに分割します。2つはローカル、もう1つはグローバルで、他の2つのエントリポイントとして機能します。メソッドの実装自体は記事に隠します。実際に試してみたい人は誰でも、記事の最後に完成したプロジェクトへのリンクがあります。



このプロジェクト構造は、ネットワークを操作するためのリファレンスではなく、Clean Swiftでネットワークを操作する方法をまったく反映していません。これはすべて、ローカルおよびグローバルワーカーの役割の良い例としてのみ行われます。

開始するには、グローバル作成ワーカー - NetworkWorkerを。シーンディレクトリと同じレベルのワーカーディレクトリに配置します。以下の例では、ローカルのWorkersに共通のsendRequest（to：params：completion）メソッドがあります。彼はルーチンタスクを実行します。パラメータからリンクを形成し、リクエストを送信して、結果を完了まで送信します。

struct NetworkWorker {

  // MARK: - Private Properties
  private let session = URLSession.shared

  // MARK: - Public Methods
  ///  Worker.    API
  ///
  /// - Parameters:
  ///   - to: ,    
  ///   - params:   
  ///   - completion:    
  func sendRequest(to: URL, params: [String: String], completion: @escaping (Data?, Error?) -> Void) {
    // ...
  }
}

最初のシーンでは、すべてのユーザーのAPIリストを取得する必要があります。これを行うには、ユーザーをロードするためのパラメーターを構成するローカルHomeWorkerを作成し、これらのパラメーターを使用してNetworkWorkerでsendRequest（to：params：completion）を呼び出します。次に、シーンInteractorでfetchUsers（完了:)を呼び出し、受信したデータを処理のためにPresenterに送信する必要があります。

ユーザー名のあるテーブルセルをクリックすると、遷移が行われ、選択したユーザーが別のシーンに転送されます。

struct HomeWorker {

  // MARK: - Private Properties
  private let networkWorker = NetworkWorker()
  private let usersURL = URL(string: "https://jsonplaceholder.typicode.com/users")

  // MARK: - Public Methods
  ///  Worker.   API    
  ///
  /// - Parameter complete:    
  func fetchUsers(_ complete: @escaping ([User]?) -> Void) {
    // ...
  }
}

ユーザー投稿のあるページで、PostsWorkerを作成しますが、fetchPosts（userId：completed :)メソッドのみを使用します。その中に、ダウンロードしたい投稿のあるユーザーのIDを渡します。このメソッドでは、パラメーターを形成し、NetworkWorkerでsendRequest（to：params：completion）を呼び出します。そして、以前と同じように、シーンのInteractorでfetchPosts（userId：completed :)メソッドを呼び出し、受け取ったデータをPresenterに渡します。

struct PostsWorker {

  // MARK: - Private Properties
  private let networkWorker = NetworkWorker()
  private let postsURL = URL(string: "https://jsonplaceholder.typicode.com/posts")

  // MARK: - Public Methods
  ///   API     
  ///
  /// - Parameters:
  ///   - userId: ID ,    
  ///   - completed:    
  func fetchPosts(userId: Int, _ completed: @escaping ([Post]?) -> Void) {
    // ...
  }
}

これで、実装全体が個別のファイルに移動され、Interactorにビジネスロジックをロードすることなく再利用できます。
