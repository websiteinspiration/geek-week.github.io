PHP検閲


PHPプロジェクトの構築を容易にするオープンソースのCIサーバー。これは、PHPCIプロジェクトのフォークです。PHPCI自体はまだ開発中ですが、以前ほど活発ではありません。

PHP Censorは、GitHub、GitLab、Mercurialなどのリポジトリで動作します。コードをテストするために、ツールはAtoum、PHP Spec、Behat、Codeceptionライブラリを使用します。ここで例のコンフィギュレーション・ファイルの最初のケースについては：

test:
    atoum:
        args: "command line arguments go here"
        config: "path to config file"
        directory: "directory to run tests"
        executable: "path to atoum executable"

PHP Censorは小規模なプロジェクトをデプロイするのに悪くないと考えられていますが、自分でホストして構成する必要があります（セルフホスト）。このタスクはかなり詳細なドキュメントによって簡略化されています-GitHubにあります。


レックス


レックスはリモート実行の略です。このシステムは、エンジニアのFerenc Erkiがデータセンターのプロセスを自動化するために開発しました。Rexの作業はPerlスクリプトに基づいていますが、ツールと対話するためにこの言語を知る必要はありません。ほとんどの操作（ファイルのコピーなど）は関数ライブラリに記述されており、スクリプトは多くの場合10行に収まります。次に、複数のサーバーにログインして稼働時間を実行する例を示します。

use Rex -feature => ['1.3'];

user "my-user";
password "my-password";

group myservers => "mywebserver", "mymailserver", "myfileserver";

desc "Get the uptime of all servers";
task "uptime", group => "myservers", sub {
   my $output = run "uptime";
   say $output;
};

現在追加中の公式ガイドや電子書籍 のツールから知り合いを始めることをお勧めします。


オープンビルドサービス（OBS）


これは、ディストリビューション開発を最適化するためのプラットフォームです。そのコードはオープンで、GitHubのリポジトリにあります。このツールの作成者はNovellです。彼女はSuSEディストリビューションの開発に参加し、このプロジェクトはもともとopenSUSEビルドサービスと呼ばれていました。当然のことながら、Open Build Serviceは、openSUSE、Tizen、およびVideoLANでプロジェクトをビルドするために使用されます。このツールは、Dell、SGI、Intelでも動作します。しかし、通常のユーザーや小規模な新興企業の中にもあります。特に彼らのために、著者は事前設定されたソフトウェアパッケージをコンパイル（p。10）しました。システム自体は完全に無料です-それを展開するためにホスティングまたは鉄のサーバーにお金を費やす必要があります。

しかし、その存在の全期間にわたって、ツールは幅広いコミュニティを獲得していません。けれどもそれはだった、オープンOSの標準化を担当するのLinux開発者向けネットワークの一部、。トピックフォーラムで質問に対する回答を見つけるのは難しい場合があります。しかし、Quoraの居住者の1人は、Freenode でのIRCチャットで、コミュニティのメンバーはかなり喜んで応答したと指摘しました。多くの問題の解決策が公式ドキュメント（PDFおよびEPUB）に記載されているため、小さなコミュニティの問題はグローバルなものではありません。OBSを使用するためのベストプラクティスを見つけることができます（例とケースがあります）。


ランデッキ


スクリプトを使用してデータセンターとクラウドのタスクを自動化 するオープンツール（GitHub）。特別なスクリプトサーバーがその実行を担当します。Rundeckは、ControlTierアプリケーション管理プラットフォームの「娘」であると言えます。Rundeckは2010年にそれと分離し、新しい機能を獲得しました。たとえば、Puppet、Chef、Git、Jenkinsとの統合です。

このシステムは、ウォルトディズニーカンパニー、Salesforce、Ticketmasterで使用されています。しかし、このプロジェクトは新興企業にも適しています。これは、RundeckがApache v2.0ライセンスでライセンスされているためです。さらに、ツールは非常に使いやすいです。

Rundeckで働いたRedditの居住者は、自分でほとんどの困難を解決したと言います。これにより、開発者によって発行されたドキュメントと電子書籍が彼を助けました。

Gocd


オープンツール（GitHub）は、コードのバージョン管理を自動化します。それは2007年にThoughtWorksによって導入されました-その後、プロジェクトはクルーズと呼ばれました。

祖先家系図サービスであるAutoCD AutoderとクレジットカードプロバイダーであるBarclaycardがGoCDと連携しています。ただし、ツールのユーザーの4分の1 は中小企業です。

スタートアップの間でのサービスの人気は、そのオープン性によって説明できます-これは、Apache v2.0ライセンスの下で配布されています。同時に、GoCDには、承認システムやクラウドソリューションなどのサードパーティソフトウェアと統合するためのプラグインがあります。真のシステムはかなり複雑です開発中-彼女には多数のオペレーターとチームがあります。また、一部のユーザーは、貧弱なインターフェースと、スケーリングのためにエージェントを構成する必要性について不満を述べています。


写真-Matt Wildbore -Unsplash

実際にGoCDを試したい場合は、プロジェクトのWebサイトで公式ドキュメントを見つけることができます。追加情報のソースとして、セットアップガイド付きのGoCD開発者ブログを推奨することもできます。


ジェンキンス


Jenkinsは広く知られており、CI \ CDの分野では一種の標準と見なされています。もちろん、これがないと、この選択は完了しません。このツールは2011年に登場し、Oracle Hudsonプロジェクトの支流となりました。

今日、彼らは仕事ジェンキンスとNASAで、任天堂、およびその他の大規模な組織。ただし、ユーザーの8％以上が最大10人の小さなグループに属しています。製品は完全に無料で、MITライセンスの下で配布されます。ただし、自分でJenkinsをホストして設定する必要があります-専用サーバーが必要です。

楽器の寿命全体にわたって、その周りに広範なコミュニティが形成されました。ユーザーはRedditのスレッドで積極的に通信しますおよびGoogleグループ。ジェンキンスの資料も定期的にハブレに掲載されています。コミュニティの一員になり、Jenkinsを使い始めたい場合は、公式ドキュメントと開発者によるガイドがあります。以下のガイドと書籍もお勧めします。

  • Jenkins入門：公式ガイド ;
  • ジェンキンスへの完全なガイド。
  • Jenkinsスターターガイド
  • ブレント・ラスターの本「ジェンキンス2.はじめに」;
  • Rafale Leshkoの著書、DockerとJenkinsによる継続的デリバリー：スケーラブルなアプリケーションの提供。

Jenkinsには、いくつかの有用なサードパーティプロジェクトがあります。1つ目は、Configuration as Codeプラグインです。ツールの深い知識がなくても管理者でも理解できる、読み取り可能なAPIでJenkinsのセットアップを簡素化します。2つ目はJenkins Xクラウドシステムです。いくつかのルーチンタスクを自動化することにより、大規模なITインフラストラクチャに展開されたアプリケーションの配信を高速化します。


ビルドボット


これは、アプリケーションの組み立てとテストのサイクルを自動化するための継続的な統合システムです。コードに変更が加えられるたびに、コードの機能を自動的にチェックします。

ツールの作成者はエンジニアのブライアンワーナー（ブライアンワーナー）でした。今日、彼は6人の開発者を含むBuildbot Oversight Committeeイニシアチブグループに置き換えられました。

Buildbotは、LLVM、MariaDB、Blender、Dr.Webなどのプロジェクトで使用されます。ただし、wxWidgetsやFlathubなどの小規模なプロジェクトでも使用されます。システムはすべての最新のVCSをサポートし、Pythonを使用してそれらを記述することで柔軟なビルド設定を行います。公式ドキュメントはそれらすべてに対処するのに役立ちます。たとえば、サードパーティのチュートリアルは、IBMの簡単なチュートリアルです。
