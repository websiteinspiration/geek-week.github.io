ギャップ-（グリッドのように）Flexboxのパディングの明るい未来



CSSグリッド仕様の私のお気に入りの部分の1つはこれgrid-gapです。グリッド内の要素間のインデントを簡単にします。

マージンと、さまざまな状況でマージンを実装するために使用するトリックは、CSSの主な問題の1つです。

W3Cはgrid-gap、simple gap とFlexboxおよびMulti-Columnでの使用を優先してプロパティを放棄することを推奨しました。

このガイドでは、以前にマージンがFlexboxに追加された方法と、プロパティがこれをどのように行うかを見て、gap追加のトリックなしでインデントを取得できるようにします。


通常のFlexboxグリッドのマージン


この例では、ブロックのグループを取り、Flexboxを使用してメッシュスタイルを作成し、マージンを使用してブロックを互いに分離します。

基本的なHTMLから始めましょう。flex-containerセットがありますflex-item.

<div class="flex-container">
    <div class="flex-item"></div>
    <div class="flex-item"></div>
    <div class="flex-item"></div>
    <div class="flex-item"></div>
    <div class="flex-item"></div>
    <div class="flex-item"></div>
    <div class="flex-item"></div>
    <div class="flex-item"></div>
</div>

Flexboxの動作のおかげで、コンテンツは隣同士に配置されます。コンテナーの幅の値に基づいて、子要素のサイズを決定し、プロパティを使用して子要素を転送しますflex-wrap。

.flex-container {
    display: flex;
    flex-wrap: wrap;
}
.flex-item {
    width: calc(100% / 3);
}


これにより、コンテナの幅の1/3に等しい完全なサイズのブロックが得られます。マージンを設定して、各要素間に垂直および水平のスペースを追加しましょう。

.flex-item {
    width: calc(100% / 3);
    margin-right: 1rem;
    margin-bottom: 1rem;
}


ああ！コンテナの1/3である要素は、親要素の幅に適合しなくなりました。行間のマージンは非常に正確であることが判明し、問題は発生しませんでした。

マージンによって作成される追加のスペースを考慮して、子要素の幅を設定する必要があります。またmargin-right、3番目の要素ごとにプロパティをリセットする必要があります。

これで、2つのマージンが等しく1remなりました2rem。3つすべての要素の幅からこれらを均等に差し引く必要があります。

flex-item {
    // width: calc(100% / 3);
    width: calc((100% / 3) - (2rem / 3)); // one-third - two margins divided equally among 3 items
    margin-right: 1rem;
    margin-bottom: 1rem;
}
.flex-item:nth-child(3n) {
    margin-right: 0;
}



難しすぎますか？はい、そうです。これを行う簡単な方法がありますが、列間の正確な1remの間隔も得られません。この複雑なコードは、レスポンシブデザインを大幅に複雑にします。

すべてのブラウザのFlexboxでギャッププロパティを使用できるようになると、コードがより明確になります。また、プロパティを使用して子供のための幅を設定することから移動することができflex-grow、flex-shrinkそしてflex-basis。

ギャップ付きインデント


ギャッププロパティを使用すると、要素の幅を使用してほとんどのトリックを実行する必要がなくなります。また、flex-grow / flex-shrink値の使用に戻すこともできます。

次の例では、プロパティdisplay: flexとflex-wrap: wrapコンテナを引き続き使用していますが、プロパティも追加していますgap。これは、row-gapとを組み合わせた省略形のプロパティですcolumn-gap。すべてのメソッドについては、MDNのドキュメントを確認してください。

ここで、各flex要素の幅を設定する代わりに、値flex-grow、flex-shrink、およびflex-basisを設定します。プロパティflex-basisは、ブラウザがコンテナにインストールする列の数を決定します。関数は引き続き使用しますcalc() これのために、しかしコードは最終的にきれいになります

.flex-container {
    display: flex;
    flex-wrap: wrap;
    gap: 1rem;
}
.flex-item {
    flex: 1 1 calc((100% / 3) - 2rem);
}



注意深い読者は、これにより、最後の要素のサイズが大きくなり、要素が欠落している行のスペースを埋めることができるようになったことに気付くでしょう。これは、要素の幅に基づくCSSグリッドとFlexboxが実行できないことです。

おまけ：ギャップも応答性を促進します。


元の例では、特定の制御点でコンテナー列の数を変更したい場合、幅を再カウントし、nth-childセレクターを変更してマージンを取り除く必要がありました。

を使用した例でgapは、flex-basisプロパティを調整するだけで済みます。

.flex-item {
    flex: 1 1 100%; // 1 across at mobile
}
@media (min-width: 640px) {
    .flex-item {
        flex-basis: calc((100% / 2) - 1rem); // 2 across at tabletish
    }
}
@media (min-width: 1024px) {
    .flex-item {
        flex-basis: calc((100% / 3) - 2rem); // 3 across at desktop
    }
}

嘘はつきませんが、そのようなデザインパターンにはCSSグリッドを好みますが、この信じられないほどの新機能の使用例をいくつかご覧いただければ幸いです。

翻訳者からの追加


たとえば、nth-child(3n)コンテナーの右端に隣接するFlex要素の右マージンを削除し、Flexコンテナー内の列数が変更されるたびにそれを修正するなど、今でも必要性を取り除くことができるトリックがあるようです。

もちろん、これも完璧ではなく、次のようになります。

  1. HTMLマークアップでは、オプションのラッパータグでFlexコンテナーをラップします
  2. 子供の場合、たとえば、代わりmargin-right: 10pxにmargin-left: 10px
  3. 増加する左マージンを補うには、Flexコンテナのプロパティに負の値を設定する必要があります margin-left: -10px;
