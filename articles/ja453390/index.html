軸とキャベツについて

AWSソリューションアーキテクトアソシエイトの認定を取得したいという願望はどこから来たのでしょうか。


最初の動機：「軸」

どの専門家にとっても最も有用な原則の1つである「ツールを理解する」（または「のこぎりを研ぐ」のバリエーションの1つ）。

私たちは長い間クラウドの中にいましたが、当分の間、それらはEC2インスタンスにデプロイされたベースを備えた単なるモノリシックアプリケーションでした-安くて陽気です。

しかし徐々にモノリスの枠組みの中で窮屈になりました。モジュール化、そして今流行のマイクロサービスについて、いい意味でのカットのコースを受講しました。そして、すぐに百本の花がこの土壌に「咲き」ました。

しかし、これから先には何がありますか-私が現在実施しているアクティビティログプロジェクトには、次のものが含まれます。

  • 高密度のレガシーの裏材から.Net Coreのトレンディなマイクロサービスまで、当社の製品のさまざまなアプリケーションの形でお客様。
  • 顧客に何が起こっているかのログを追加するAmazon SQSキュー。
  • キューからメッセージを取得してAmazon Kinesis Data Streams（KDS）に送信する.Net Coreマイクロサービス。また、バックアップチャネルとして、および手動テスト用のWeb APIとSwagger UIも備えています。docker linuxコンテナーにラップされ、Amazon ECSの下でホストされます。ログの大量のストリームの場合、自動スケーリングが提供されます。
  • KDSから、消防ホースのデータがAmazon Redshiftに送信され、中間倉庫はAmazon S3にあります。
  • 開発者向けの操作ログ（デバッグ情報、エラーメッセージなど）は見栄えの良いJSONでフォーマットされ、Amazon CloudWatch Logsに送信されます



このようなAWSサービスの動物園を使用して、武器の中に何があるのか​​、そして何かを使用する方が良いのか知りたいです。

想像してみてください-あなたは木をうまく切り、釘をうまく詰まらせる古い証明された斧を持っています。長年にわたって、あなたは彼を上手に扱い、犬小屋、いくつかの小屋、そしておそらく小屋さえまとめることを学びました。時々問題が発生します。たとえば、斧でねじをねじ込むと、すぐにうまくいかない場合がありますが、通常は忍耐力などの母親の助けを借りて解決されます。

そして、近くに裕福な隣人が現れ、電気のこぎり、釘打ち機、ねじ回し、そして神は他に何を知っているかなど、さまざまな道具のくもりのある雲を抱えています。彼はこのすべての富を24時間リースする準備ができています。何をすべきか？斧を取り、私たちを政治的非識字として却下するという選択肢。どのようなツールがあり、どのように異なる仕事でお互いを補完できるか、どのような条件で貸し出されているかを調べるのが最も合理的です。

この動機が私にとって主なものだったので、準備はそれに応じて構成されました-基本的なガイダンスを見つけ、それを注意深く研究するためです。そして、そのようなガイドが見つかりました。本は少々乾燥していますが、フィヒテンホルツでマタンを教えた人々を怖がらせることはまずありません。

私はそれを最初から最後まで読んで、それが設定された目標を完全に満たしていると思います-それはサービス自体と試験で発生することができるより一般的な概念の両方の良い概要を提供します。さらに、すばらしいボーナスは、Sybexのやや奇妙な登録手続きを経て、本からオンラインですべてのテストの質問とテスト試験に答える機会です。

重要なポイント：2016年版の本に従って勉強しましたが、AWSではすべてが非常に動的に変化するため、準備時に最新版を探します。たとえば、トライアルテストでは、さまざまなクラスのS3およびGlacierの可用性と耐久性について疑問が生じることがよくありますが、2016年と比べて一部の数値は変更されています。さらに、新しいものが追加されました（たとえば、INTELLIGENT_TIERINGまたはONEZONE_IA）。

第2の動機：「オレンジの65の色合い」

。しかし、多くのプログラマーが不可解なタスク、質問、そして場合によっては試験さえすることから、自虐的な喜びを経験することは秘密ではありません。

この楽しさは何を演奏するのとよく似ていると思いますか？どこ？いつ？"または、たとえば、チェスの良いゲームです。

この意味で、現在のAWS Solutions Architect Associate試験は非常に優れています。テストの質問の準備中に、「VPCで使用できる最大エラスティックIPアドレスの数」や「S3 IAの可用性とは何ですか」などの「窮屈な」問題がときどきありましたが、」、彼らはそのような試験に出くわしませんでした。実際、65の質問のほとんどすべてが設計のためのミニタスクでした。これは公式ドキュメントのかなり典型的な例です：
A web application allows customers to upload orders to an S3 bucket. The resulting Amazon S3 events trigger a Lambda function that inserts a message to an SQS queue. A single EC2 instance reads messages from the queue, processes them, and stores them in an DynamoDB table partitioned by unique order ID. Next month traffic is expected to increase by a factor of 10 and a Solutions Architect is reviewing the architecture for possible scaling problems. Which component is MOST likely to need re-architecting to be able to scale to accommodate the new traffic?
A. Lambda function B. SQS queue C. EC2 instance D. DynamoDB table

私の知る限り、以前のバージョンの試験には55問が含まれており、80分が割り当てられていました。どうやら、彼らは彼にうまく取り組んだ：今それらに65の質問と130分があります。質問の時間は増加しましたが、実際には問題はありません。それぞれが、時には2分以上も考えなければなりませんでした。

ちなみに、ここから実用的な結論があります。通常、勝利の戦術は、すべての質問をすばやく調べ、すぐに答えられたものに答えることです。 SAA-C01の場合、これは通常は機能しません。ほとんどすべての質問にフラグを立てる必要があります。そうしないと、詳細に気付かず、誤って回答するリスクがあります。結局、私は答えて、各質問に1〜2分を費やし、それからシャッターに戻り、残りの20分をそれらに費やしました。

第3の動機：「若者が知っていれば、老後も可能か」

ご存知のように、プログラマーが40を超える障害の最も頻繁な原因の1つは、若者と比べて学習能力が低下していることです。

一方で、忍耐力と経験が向上しているため、学生時代に比べて学習能力が向上している地域もあるようです。これにより、馴染みのない類推に馴染みのある類推を引き付けることができます。

しかし、感情はだまされている可能性があり、客観的な基準が必要です。試験の準備をして合格する-オプションではないものは何ですか？

検証は成功したと思います。私は自分で準備しましたが、準備は十分スムーズに進みました。ええ、はい、マニュアルを読んでいるハンモックで数回寝ましたが、これは誰にでも起こり得ることです。

2番目の動機ではありません：「会社の要件」

企業は、認定を奨励したり、認定を要求したりすることもできます（特に、Amazonの場合のAWS APNへの参加など、パートナーシップに必要な場合）。

しかし、私たちの場合、独立した製品が製造され、さらに、ベンダーロックインを回避しようとしています。したがって、誰も証明書を必要としません。一定の努力が認められた場合、彼らは試験を称賛し、支払います。それが公式の全体です。

第三の動機ではない：「雇用」

おそらく、証明書を入手できることは、仕事を得るための明確なプラスになるでしょう。しかし、転職の予定はありません。多くの新しいアプローチとAWSサービスを積極的に使用する複雑な製品に取り組むのは興味深いことです。現在の場所ではこれで十分です。
