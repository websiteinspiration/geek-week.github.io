Analysieren von YouTube, einschließlich hochgeladener Daten, ohne die YouTube-API

Einführung


Zum Laden von Inhaltsdaten auf YouTube verwenden sie normalerweise entweder Selenium oder die YouTube-API. Es gibt jedoch überall Nachteile.

  1. Selen ist zu langsam zum Parsen. Stellen Sie sich vor, Sie analysieren eine Wiedergabeliste mit ~ 1000 Clips nach Selen.
  2. Die YouTube-API ist natürlich die beste Option, wenn Sie eine Anwendung oder ein Projekt haben, aber dort müssen Sie die Anwendung usw. registrieren. In der "Test" -Version müssen Sie sich ständig anmelden, um die API zu verwenden. Außerdem gibt es ein schnelles Kontingent.
  3. In unserer Methode würde ich sehr komplexe Datenstrukturen sagen, die von YouTube generiert wurden. Das Parsen von YouTube-Suchanfragen ist besonders instabil.

Wie lade ich Daten auf YouTube?


Hierfür gibt es ein Token, das sich im HTML-Code der Seite befindet. Dann werden wir es in Zukunft als Parameter für eine Anfrage an YouTube verwenden, die uns neuen Inhalt gibt. YouTube selbst lädt Inhalte mithilfe einer Anfrage herunter, bei der dieses Token verwendet wird.

Es gibt zusätzliche ausgehende Parameter, die wir im nächsten Schritt benötigen.



Ein Token durch ein Skript erhalten


Wir erstellen den Header-Parameter für die Anfrage an YouTube. Zusätzlich zum Benutzeragenten fügen wir zwei weitere ein, die Sie unten sehen.

headers = {
    'User-Agent': '      ,      ',
    'x-youtube-client-name': '1',
    'x-youtube-client-version': '2.20200529.02.01'
}

Wir stellen eine Anfrage über die Anforderungsbibliothek. Fügen Sie einen Link zu der Seite ein, die Sie laden möchten, und fügen Sie außerdem Überschriften hinzu.

token_page = requests.get(url, headers=headers)

Das Token kann von Parsern nicht gefunden werden, da es im Tag-Tag versteckt ist. Um es in einer Variablen zu speichern, schreibe ich einen so hässlichen Code:

nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]

Normalerweise ist dies ein Token mit einer Länge von 80 Zeichen.

Eine Anfrage nach Inhalten stellen


service = 'https://www.youtube.com/browse_ajax'
params = {
"ctoken": nextDataToken,
"continuation": nextDataToken
}
r = requests.post(service, params=params, headers=headers)

Verschiedene Arten geladener Daten haben unterschiedliche Dienstverknüpfungen. Unsere eignet sich für Wiedergabelisten und Videos von Kanälen.

YouTube sendet Daten im JSON-Format. Deshalb schreiben wir r.json (), aber die Liste wird zu Ihnen fliegen, wir brauchen das zweite Element der Liste, daher r.json () [1]. Außerdem haben wir bereits Daten. Es bleibt zu analysieren.

Analyse der JSON-Antwort


Sie können lange Wörterbuchketten sehen, aber wir werden sie so kürzen, dass sie mehr oder weniger klar sind.

r_jsonResponse = r_json['response']
dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]

Hier erhalten wir einen neuen Token für weitere Anfragen. Wenn die geladenen Daten beendet sind, wird das Token nicht angezeigt.
for content in dataContainer["contents"]:
	videoId = content['playlistVideoRenderer']['videoId']

Auf diese Weise können Sie die ID des Videos extrahieren, indem Sie den Vorlagenteil hinzufügen. Sie erhalten einen Link zum Video.

Um die folgenden Daten zu erhalten, müssen Sie dasselbe tun - ein Token anfordern, analysieren und dann erneut.

Der voll funktionsfähige Code sieht folgendermaßen aus:

import requests, json

url = input('url: ')
headers = {
		"User-Agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/81.0.4044.122 Safari/537.36",
		'x-youtube-client-name': '1',
		'x-youtube-client-version': '2.20200429.03.00',
		}
token_page = requests.get(url, headers=headers)
nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]
sleep = False #  ,    
ids = []
while not sleep:
	service = 'https://www.youtube.com/browse_ajax'
	params = {
	"ctoken": nextDataToken,
	"continuation": nextDataToken
	}
	r = requests.post(service, params=params, headers=headers)
	r_json = r.json()[1]
	r_jsonResponse = r_json['response']
	dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
	try: #  
		nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]
	except:
                #  . ,    .    
		sleep = True
	for content in dataContainer["contents"]:
		videoId = content['playlistVideoRenderer']['videoId']
		ids.append(videoId)
print(len(ids))
