Styled-Systemを使用せずにコンポーネントを作成し、独自の自転車を発明した方法

みなさん、こんにちは！私の名前はサーシャです。私はクォークリーの共同創設者兼非常勤主任開発者です。この記事では、私たちが遵守しているアトミックCSSの概念と、Styled-System機能の欠点（およびこのライブラリを使用する特別なケースとしてのRebass）がどのようにしてAtomizeと呼ばれる独自のツールの作成を促したかについてお話します。

小さなプリアンブル。Quarklyプロジェクトは、グラフィカルエディター（Figma、Sketchなど）とサイトデザイナー（Webflowなど）を組み合わせたもので、クラシックIDEに固有の機能が追加されています。 Quarklyについては、別の投稿を書く予定ですが、説明する内容と表示する内容がありますが、本日は上記のAtomizeについて説明します。

Atomizeはプロジェクト全体の基礎であり、Styled-SystemとRebassを使用して解決することが不可能または困難である問題を解決することを可能にします。少なくとも、このソリューションはそれほどエレガントではありません。

今、投稿全体を圧倒する時間がほとんどない場合は、GitHubでAtomizeに慣れることができます。

また、知り合いをより快適にするために、Atomizeを使用して反応コンポーネントを組み立てるための競争を開始します。これについては、投稿の最後で詳しく説明します。

すべての始まり


Quarklyの開発を開始し、コンポーネントをタイプセットする機能をユーザーに提供したいが、別のCSSファイルを使用する必要がないことに同意しました。インラインスタイルとは対照的に、コードを可能な限り最小限に抑えながら、すべてのCSS機能を保持するため。

このタスクは革新的ではなく、一見すると、スタイル付きシステムとリベースの助けを借りて完全に解決されます。しかし、この機能では十分ではなく、さらに次の問題が発生しました。

  • ブレークポイントを使用した不快な作業。
  • 状態ホバー、フォーカスなどのスタイルを書く能力の欠如;
  • トピックを扱うメカニズムは、十分に柔軟ではないように見えました。

Atomizeとは（簡単に）




Atomizeの主要な機能のうち、以下を強調できます。

  • 複合css-propertiesでトピックの変数を使用する機能。
  • ホバーおよびその他の疑似クラスのサポート。
  • 各プロパティの短いエイリアス（emmetなど）。
  • マークアップの読みやすさを維持しながら、特定のブレークポイントのスタイルを指定する機能。
  • ミニマルなインターフェース。

Atomizeには2つの主な目的があります。

  • アトミックCSSおよびテーマをサポートするコンポーネントの作成。
  • Quarklyプロジェクトでインタラクティブな編集用のウィジェットを作成します。

使用説明書を噴霧


作業を開始する前に、依存関係を確立する必要があります。 

npm i react react-dom styled-components @quarkly/atomize @quarkly/theme

Atomizeは、styled-componentのラッパーであり、同様のAPIを持っています。必要な要素の名前でメソッドを呼び出すだけで十分です。

import atomize from '@quarkly/atomize';
 
const MyBox = atomize.div();

出力では、CSSを小道具として受け入れることができる反応コンポーネントを取得します。
使いやすさのために、プロパティエイリアスのシステムが開発されました。たとえば、bgc === backgroundColor

ReactDOM.render(<MyBox bgc="red" />, root);

プロパティとエイリアスの完全なリストはここにあります。

React継承メカニズムも提供されています。

const MySuperComponent = ({ className }) => {
   // some logic here
   return <div className={className} />;
};
 
const MyWrappedComponent = atomize(MySuperComponent);

テーマを操作する


これについては、もう少し詳しく教えていただく必要があるようです。QuarklyのテーマはCSS変数に基づいています。重要な機能は、テンプレート関数の形式での追加の抽象化やその後のユーザーによる追加の処理を必要とせずに、小道具とトピック自体の両方から変数を再利用できることです。

トピックの変数を使用するには、トピックのプロパティを記述し、接頭辞「-」を使用してこのプロパティにアクセスするだけで十分です。

変数はJSXと同様に使用できます。

import Theme from "@quarkly/theme";
 
const theme = {
   colors: {
       dark: "#04080C",
   },
};
export const MyComp = () => (
   <Theme>
       <Box bgc="--colors-dark" height="100px" width="100px" />
   </Theme>
);

（カラー＃04080Cは--colors-darkプロパティから利用できます）

したがって、サブジェクト自体では次のようになります。

import Theme from "@quarkly/theme";
 
const theme = {
   colors: {
       dark: "#04080C",
   },
   borders: {
       dark: "5px solid --colors-dark",
   },
};
export const MyComp = () => (
   <Theme>
       <Box border="--borders-dark" height="100px" width="100px" />
   </Theme>
);

（私たちは色から変数を再利用し、それを境界線テーマに接続しました）

JSXマークアップの色については、簡略化された構文が提供されています。

import Theme from "@quarkly/theme";
 
const theme = {
   colors: {
       dark: "#04080C",
   },
};
export const MyComp = () => (
   <Theme>
       <Box bgc="--dark" height="100px" width="100px" />
   </Theme>
);

メディア表現を扱う場合、テーマにはブレークポイントがあります。
ブレークポイントキー名の形式で任意のプロパティにプレフィックスを追加できます。

import Theme from "@quarkly/theme";
 
const theme = {
   breakpoints: {
       sm: [{ type: "max-width", value: 576 }],
       md: [{ type: "max-width", value: 768 }],
       lg: [{ type: "max-width", value: 992 }],
   },
   colors: {
       dark: "#04080C",
   },
   borders: {
       dark: "5px solid --colors-dark",
   },
};
export const MyComp = () => (
   <Theme>
       <Box
           md-bgc="--dark"
           border="--borders-dark"
           height="100px"
           width="100px"
       />
   </Theme>
);

効果


AtomizeとStyled-Systemの主な違いは、「効果」です。それは何ですか、なぜそれが必要なのですか？
Buttonコンポーネントを作成し、その色と境界線を変更するとしますが、ホバー、フォーカスなどにスタイルを割り当てる方法を想像してみてください。ここで効果が助けになります。

コンポーネントを作成するときは、構成を指定してオブジェクトを転送するだけです。

const MySuperButton = atomize.button({
 effects: {
   hover: ":hover",
   focus: ":focus",
   active: ":active",
   disabled: ":disabled",
 },
});

キーはプロパティ名のプレフィックスであり、値はCSSセレクターです。したがって、すべての疑似クラスの必要性を閉じました。

ここで、CSSプロパティにホバープレフィックスを指定すると、特定の効果が適用されます。たとえば、以下にカーソルを合わせると、

ReactDOM.render(<MySuperButton hover-bgc="blue" />, root);

エフェクトをメディア表現と組み合わせることもできます。

ReactDOM.render(<MySuperButton md-hover-bgc="blue" />, root);

いくつかの例


上記の情報を視覚化するために、いくつかの興味深いコンポーネントを組み立てましょう。2つの例を用意しました。

  • ライブラリのすべての機能を示すシンプルなコンポーネント。
  • ポケモンカードを使ったより複雑な例。

2番目の例では、ほとんどの機能と外部APIを使用しました。

しかし、それだけではありません


上記のように、Atomizeの2番目の目的は、カスタム反応コンポーネントに基づいてQuarklyでウィジェットを作成することです。

これを行うには、コンポーネントをAtomizeでラップし、その構成を記述して、Quarklyがインタラクティブに編集できるプロパティを理解できるようにします。

export default atomize(PokemonCard)(
 {
   name: "PokeCard",
   effects: {
     hover: ":hover",
   },
   description: {
     // past here description for your component
     en: "PokeCard — my awesome component",
   },
   propInfo: {
     // past here props description for your component
     name: {
       control: "input",
     },
   },
 },
 { name: "pikachu" }
);

コンポーネントの構成フィールドは次のようになります。

  • エフェクト -ブラウザの疑似クラス（ホバー、フォーカスなど）を定義します。
  • description-カーソルを名前の上に置いたときに表示されるコンポーネントの説明。
  • propInfo-右側のパネル（[ プロパティ ]タブ）に表示されるコントロールの構成。

右側のパネル（プロップタブ）に表示されるプロップを決定する方法：

propInfo: {
   yourCustomProps: { //  
       description: { en: "test" }, //    
       control: "input" //  
   }
}
