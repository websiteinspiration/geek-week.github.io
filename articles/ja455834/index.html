Linuxサーバーのベンチマーク：5つのオープンツール

今日は、プロセッサ、メモリ、ファイルシステム、およびストレージシステムのパフォーマンスを評価するためのオープンツールについて説明します。

リストには、GitHubの居住者が提供するユーティリティとRedditのテーマ別スレッドのメンバー（Sysbench、UnixBench、Phoronix Test Suite、Vdbench、IOzone）が含まれています。


/アンスプラッシュ/ ベリ・イワノワ

Sysbench


これは、Lua言語用の仮想マシンを開発するLuaJITプロジェクトに基づくMySQLサーバーの負荷テスト用のユーティリティです。このツールの作成者は、MySQL Alexey Kopytovのプログラマーでありエキスパートです。プロジェクトは趣味として始まりましたが、やがてコミュニティの認識を得ました。今日、sysbenchはIEEEのような大規模な大学やIT組織を業務に使用しています。

SECR-2017カンファレンス（YouTubeでのスピーチの録音があります）の間、アレクセイは、sysbenchを使用すると、新しい機器に移動したり、DBMSバージョンを更新したり、クエリ数を大幅に変更したりするときに、データベースのパフォーマンスを評価できると述べました。一般に、テストのコマンド構文は次のとおりです。

sysbench [options]... [testname] [command]

このコマンドは、タイプ（cpu、メモリ、fileio）と負荷テストパラメータ（スレッド数、リクエスト数、トランザクション処理速度）を決定します。一般に、このツールは1秒あたり数百万のイベントを処理できます。Alexey Kopytovは、ソフトウェア開発ポッドキャストポッドキャストの問題の 1つで、sysbenchのアーキテクチャと内部構造について詳しく説明しました。


Unixbench


Unixシステムのパフォーマンスを評価するためのツールのセット。彼は1983年にモナッシュ大学のエンジニアによって紹介されました。それ以来、多くの人々がツールをサポートしてきました。たとえば、マイクロコンピュータ技術マガジンの作者であるByte Magazineや、LKMLのメンバーであるDavid Niemiです。MicrosoftのAnthony Voellmが、ツールの次のバージョンのリリースを担当しています。

UnixBenchは、個々のテストのコレクションです。彼らは、Unixを実行しているマシンでのコード実行の速度を、SPARCstation 20-61 である参照システムのパフォーマンスと比較します。この比較に基づいて、パフォーマンススコアが生成されます。

Phoronix Test Suite


このテストスイートは、GNU / Linuxディストリビューションに関するニュースを公開しているPhoronix Webリソースの作成者によって開発されました。最初のテストスイートは2008年に導入され、その後23の異なるテストが含まれました。その後、開発者はクラウドサービスOpenBenchmarking.orgを開始しました。そこでユーザーは独自のテストスクリプトを公開できます。今日では、機械学習やレイトレーシングテクノロジーに関連するものを含め、約60のベンチマークキットを提供しています。

特殊なスクリプトのセットにより、個々のシステムコンポーネントをテストできます。彼らの助けを借りて、カーネルのコンパイル時間とビデオファイルのエンコード、アーカイバーの圧縮速度などを評価できます。テストを実行するには、コンソールで適切なコマンドを記述します。たとえば、次のコマンドはCPUパフォーマンスの評価を開始します。

phoronix-test-suite benchmark smallpt

テスト中、Test Suiteは個別に機器の状態（CPU温度とクーラー速度）を監視し、システムを過熱から保護します。

Vdbench


Oracleが開発したディスクシステムでI / O負荷を生成するためのツール。ストレージシステムのパフォーマンスと整合性を評価するのに役立ちます（ディスクシステムの理論上のパフォーマンスを計算する方法について簡単なヘルプを用意しました）。

ソリューションは次のように機能します。実際のシステムでは、SWATプログラム（Sun StorageTekワークロード分析ツール）が起動され、一定期間のすべてのディスクアクセスを含むダンプが作成されます。タイムスタンプ、操作の種類、アドレス、データブロックサイズが記録されます。次に、ダンプファイルを使用して、vdbenchは他のシステムの負荷をエミュレートします。

ユーティリティを管理するためのパラメータのリストは、オラクルの公式ドキュメントに記載されています。ユーティリティのソースコードは、会社のウェブサイトにあります。


イオゾン


ファイルシステムのパフォーマンスを評価するためのコンソールユーティリティ。ファイルの読み取り、書き込み、上書きの速度を決定します。何十人ものプログラマーがツールの開発に参加しましたが、その最初のバージョンのエンジニアはエンジニアのウィリアム・ノーコットであると考えられています。この開発は、Apple、NetApp、iXsystemsなどの企業によってサポートされていました。

テスト中にスレッドを制御し、それらを同期するために、ツールはPOSIXスレッド標準を使用します。作業が完了すると、IOzoneは結果をテキスト形式またはスプレッドシート（​​Excel）としてレポートを発行します。このツールには、テーブル内のデータから3次元グラフを作成するgengnuplot.shスクリプトも含まれています。このようなグラフの例は、機器のドキュメント（p。11-17）にあります。

IOzoneは、前述のPhoronixテストスイートのテストプロファイルとして利用できます。
