Merge texture sets, or how to make one set of textures in Substance Painter

Merge Texture Set quickly and without too much effort ...



Good day, my name is Julia , and today I will tell you how to combine Texture Sets into Substance Painter and export one set of textures.I will explain by the example of my project, which I did in Blender 2.8.

Introduction


For starters, a few words about software. Substance Painter is a fairly powerful and stable program that is familiar to many, and maybe all 3D artists and developers of computer games. What could be easier to load a model with UV- scan and immediately ‚Äútexture‚Äù and ‚Äúbake‚Äù? But once in front of me there was a problem ...

Problem


As many (who work with Substance Painter ) know that when importing a 3D model with a set of colors or textures into the environment, this model is divided into separate areas or ‚ÄúTexture Sets‚Äù . For example, I took my old ax model.



Texture Set is a convenient tool, since an artist, when working with him, can simply remove from view the parts of the model that he does not need, which will be painted in a different color. And all would be fine, but what did I get when I decided to export textures from Substance Painter to work with the model in another editor (for example, in the same Blender )?

Namely, what I should have, I got a set of textures for eachTexture Set , namely 6 * 3 = 18 different textures ( BaseColor, Roughness, Metallic, Normal, Height, Emissive ).

Maybe this will seem very convenient to someone, but definitely not to me. Customizing the render by adding 18 textures to one small project is too much ... I tried to solve this problem. I searched for solutions on the net, but there were few answers that were not clear to me. Now, after a while, I want to talk about how I solve it now.

Stage 1 - Model Preparation


When exporting from any 3D editor (after creating low-poly and UV ), we must assign one color to all parts of the model - this is the most important thing to do at the first stage. By importing this model later in Substance Painter , we will have one Texture Set and, therefore, only one texture set when exporting.



Further, we work with our high-poly (on the basis of which they usually ‚Äúbake‚Äù different cards). Assign each object its own color ID (I also worked in Blender ), that is, we simply color them in different colors that are as different as possible from each other.

Stage 2 - The Bake


After everything has been prepared, export the low-poly to Substance Painter. Further, the most important ... I will not talk about what ‚Äúbaking‚Äù and Normal Map are , there are a lot of such articles, and if you are interested in this topic, you probably know what I'm talking about.
When baking in Substance Painter, I select only 2 Normal Map and ID cards . And as an object, I select my high-poly , click Bake and you 're done.

(To make the card ‚Äúbake‚Äù without artifacts, you need to do some additional operations in the settings: on the Common tab in the Match fieldset the value to ‚ÄúBy Mesh Name‚Äù , on the ID tab in the Color Source field set the value to ‚ÄúMaterial Color‚Äù .)



Stage 3 - Project Setup


Now you need to properly configure the project, so that working with it is convenient and understandable.

  1. Create a folder ( Folder ) in the layers and name it as the texture would be called (In my case, Wood, Rings or Steel ). You need as many folders as there will be textures on your model.
  2. We select one of the folders and add mask with color selection to it in the masks menu, select Picker in its settings and see a ‚Äúbaked‚Äù ID on our low-poly model. This operation must be done for all parts of the object.

Now these folders are our variants of the Texture Set , when working with which we will not touch other parts of the model. Now hierarchically filling folders (for example, Fill Layer ) you can paint parts of the model in your texture.
