PythonのAlbion Onlineゲームで釣りボットを作成しています



皆さんこんにちは、私はGNU / Linuxオペレーティングシステムの幸せなユーザーです。多くの人が知っているように、タンバリンを使った追加のダンスなしでLinuxに行くおもちゃは、Windowsよりもはるかに少ないです。

そして、MMORPGジャンルのゲームはさらに少なくなります。

しかし、約半年または一年前に、彼らがゲームAlbion OnlineをLinuxに移植したことがわかりました。
ゲームはとても面白いですが、かなり長い時間がかかります。そして、貴重な人生の時間を無駄にしないために、ボットを書くことにしました。私のビジネスに取り掛かっている間、これは私に資源を養います

ゲームにはさまざまな種類の工芸品があり、木を切ったり、石を掘ったり、庭を育てることができますが、私の選択は釣りを優先して行われました。

したがって、オンラインのアルビオンでの釣りの本質は簡単です。釣り竿を取り、池に行き、ボタンを押して餌を投げ、一定のバイト時間を待ちます。



噛みついたら、もう一度ボタンを押してミニゲームをします。

ここでは、フロートの動きに応じて、釣り竿を交互に引っ張ってから、引っ張らないようにする必要があります。フロートは常にランダムに移動し、シーケンスと速度は異なります。そして、すべてを正しく行い、フロートが許容範囲を超えない場合は、魚を捕まえます。

したがって、このビジネスをすべて自動化します。コンピューター言語のpythonによる。私は最も難しいものから、つまりゲームがフロートで始まる瞬間から始めることにしました。ここでも、私のお気に入りのOpenCVコンピュータービジョンライブラリが助けになりました。それを起動することにより、例えば写真の中の物体を検出することができます。ただし、ライブラリ自体は、何を発見する必要があるのか​​正確にはわかりません。もちろん、それらを定義するためにさまざまなオブジェクトが提示される多くのパターンがあります。ただし、Albion Onlineにはフロートはありません。

しかし、このライブラリには、指定されたテンプレートに対する素晴らしい検索機能があります。テンプレートとして、フロートのスクリーンショットを撮りました。



そして、私たちが見るように、すべてが写真で完全に見つかりました。

また、画像とストリーミングビデオの違いはそれほど大きくありません。実際、これは非常に高速な画像のストリームと数行のコードであるためです。そして今、私たちはすでにストリーミングビデオのフロートを見つけることができます。

コード
import numpy as np
import cv2
from mss.linux import MSS as mss
from PIL import Image
import time
import pyautogui as pg
import cv2
import mss
import numpy

template = cv2.imread("2019-07-02_06-55_1.png", cv2.IMREAD_GRAYSCALE)
w, h = template.shape[::-1]

with mss.mss() as sct:
    monitor = {"top": 40, "left": 0, "width": 800, "height": 640}

    while "Screen capturing":
        last_time = time.time()
        img = numpy.array(sct.grab(monitor))
        cv2.imshow("OpenCV/Numpy normal", img)
        print("fps: {}".format(1 / (time.time() - last_time)))
        gray_frame = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
        res = cv2.matchTemplate(gray_frame, template, cv2.TM_CCOEFF_NORMED)
        loc = np.where(res >= 0.7)
        for pt in zip(*loc[::-1]):
            cv2.rectangle(img, pt, (pt[0] + w, pt[1] + h), (0, 255, 0), 3)
        cv2.imshow("Frame", img)
        key = cv2.waitKey(1)
        if cv2.waitKey(25) & 0xFF == ord("q"):
            cv2.destroyAllWindows()
            break

次に、フロートを監視する必要がある元の部分に戻ります。

それはすべて少し異なります。たとえば、フロートがスローされる場所を決定し、フロートの存在について画面のその部分を分析することができます。しかし、水中に投入されたフロートはさまざまな角度で出現し、常に揺れています。

この場合、その少し異なる方法を使用します。その本質は、追跡するフラグメント内のピクセルの違いを分析することです。フロートがない場合はゼロに近づきます。

その結果、フロートが消えてアクションができる最適な値が見つかりました。

import numpy as np
import cv2
from mss.linux import MSS as mss
from PIL import Image
import time
import pyautogui as pg
import imutils
import mss
import numpy
import pyautogui

template = cv2.imread("2019-07-02_06-55_1.png", cv2.IMREAD_GRAYSCALE)
w, h = template.shape[::-1]

color_yellow = (0,255,255)

mon = {'top': 80, 'left': 350, 'width': 100, 'height': 100}

def process_image(original_image):

    processed_image = cv2.cvtColor(original_image, cv2.COLOR_BGR2GRAY)

    processed_image = cv2.Canny(processed_image, threshold1=200, threshold2=300)
    return processed_image

def ss():
    op = 1
    with mss.mss() as sct:

        monitor = {"top": 40, "left": 0, "width": 800, "height": 640}

        while "Screen capturing":
            last_time = time.time()

            img = numpy.array(sct.grab(monitor))


            gray_frame = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
            res = cv2.matchTemplate(gray_frame, template, cv2.TM_CCOEFF_NORMED)
            loc = np.where(res >= 0.7)
            op += 1
            print (op)
            for pt in zip(*loc[::-1]):
                cv2.rectangle(img, pt, (pt[0] + w, pt[1] + h), (0, 255, 0), 3)
                for p in img:
                    pts = (pt[0],pt[1])
                    x = (pt[0])
                    y = (pt[1])
                    print (x)
                    if 100 < x < 490:
                            pyautogui.mouseDown(button='left')
                            time.sleep(2)
                            pyautogui.mouseUp(button='left')
                            x = 0
                            break
                    else:
                            continue
                    break
                else:
                        continue
                break
            key = cv2.waitKey(1)
            if cv2.waitKey(25) & 0xFF == ord("q"):
                cv2.destroyAllWindows()

                return
