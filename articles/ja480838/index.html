SQLクエリは高速です。パート1

前書き


SQL言語は、そのシンプルさ、利便性、普及により、ビジネスアナリストの生活と候補者の要件に非常に強く統合されています。私自身の経験から、ほとんどの場合、SQLは、アップロード、ストアフロント（およびこれらのストアフロントに基づくレポートのその後の構築）、およびデータベース管理の形成に使用されていると言えます。また、アナリストの毎日の作業はデータのアップロードとストアフロントに必然的に関連付けられるため、SQLクエリを書くスキルは、候補者が有利になるか、または排除される要因になる可能性があります。悲しいニュースは、誰もがそれを学生のベンチに置くことを期待できるわけではないということです。良い知らせは、SQLの学習は複雑ではなく、高速であり、クエリ構文は単純で簡単です。これは特に、すでにより複雑な言語に出会ったことがある人に当てはまります。

SQLトレーニングを3つの部分に分けました。この部分は、基本的な構文に特化しており、80〜90％のケースで使用されています。次の2つの部分は、サブクエリ、Join'amおよび特別な演算子に当てられます。このガイドの目的は、SQL構文をすばやく実用的に実践してスキルの武器に追加することです。

SQLクエリ構造


リクエストの一般的な構造は次のとおりです。

SELECT ('  *    ; ')
FROM ('; ')
WHERE ('/, , city = 'Moscow'; ')
GROUP BY (',     ; ')
HAVING ('/    ; ')
ORDER BY (',     ; ')

構造を分析しましょう。便宜上、リクエストの現在の学習項目はCAPSによって割り当てられます。

SELECT、FROM


SELECT、FROM-選択した列、その順序、およびデータソースを定義するクエリの必須要素。

Customersテーブルから（*で示されている）すべてを選択します。

SELECT * FROM Customers

Customersテーブルから列CustomerID、CustomerNameを選択します。

SELECT CustomerID, CustomerName FROM Customers


どこ


WHERE-必要な条件でデータをフィルタリングする必要がある場合に使用されるオプションのクエリ要素。多くの場合、where要素内では、IN / NOT INを使用して列を複数の値でフィルタリングし、AND / ORを使用してテーブルを複数の列でフィルタリングします。

1つの条件と1つの値によるフィルタリング：

select * from Customers
WHERE City = 'London'

IN（包含）またはNOT IN（例外）を使用した1つの条件と複数の値によるフィルタリング：

select * from Customers
where City IN ('London', 'Berlin')

select * from Customers
where City NOT IN ('Madrid', 'Berlin','Bern')

AND（すべての条件が満たされている）またはOR（少なくとも1つの条件が満たされている）といくつかの値を使用したいくつかの条件によるフィルタリング：

select * from Customers
where Country = 'Germany' AND City not in ('Berlin', 'Aachen') AND CustomerID > 15

select * from Customers
where City in ('London', 'Berlin') OR CustomerID > 4

GROUP BY


GROUP BY-必要な列で集計を指定できるオプションのクエリ要素（たとえば、各都市に住んでいる顧客の数を調べる必要がある場合）。

GROUP BYを使用する場合は、次のことを行う必要があります。

  1. 切り取りに使用される列のリストは、SELECT内とGROUP BY内で同じでした。
  2. 集計関数（SUM、AVG、COUNT、MAX、MIN）も、そのような関数が適用される列を使用してSELECT内で指定する必要があります。

都市ごとに顧客数をグループ化：

select City, count(CustomerID) from Customers
GROUP BY City

国および都市ごとに顧客数をグループ化： 

select Country, City, count(CustomerID) from Customers
GROUP BY Country, City

さまざまな集計関数を使用して製品IDで売上をグループ化：この製品の注文数と販売された個数：


select ProductID, COUNT(OrderID), SUM(Quantity) from OrderDetails
GROUP BY ProductID

ソーステーブルのフィルタリングによる売上のグループ化。この場合、出力はドイツの都市の顧客数の表になります。


select City, count(CustomerID) from Customers
WHERE Country = 'Germany'
GROUP BY City

AS演算子を使用して集計列の名前を変更します。デフォルトでは、集計のある列の名前は、適用された集計関数と同じです。さらに、これは、認識にはあまり便利ではない場合があります。

select City, count(CustomerID) AS Number_of_clients from Customers
group by City

持っている


HAVINGは、グループ化されたデータのレベル（実際にはWHEREですが、1レベルだけ高いレベル）でのフィルタリングを担当するオプションのクエリ要素です。

都市ごとに顧客の数で集計テーブルをフィルタリングします。この場合、少なくとも5人の顧客がアンロードされている都市のみを残します。


select City, count(CustomerID) from Customers
group by City
HAVING count(CustomerID) >= 5


HAVING内で名前が変更された列の場合、集計構成カウント（CustomerID）と新しい列名number_of_clientsの両方を指定できます。 


select City, count(CustomerID) as number_of_clients from Customers
group by City
HAVING number_of_clients >= 5

WHEREおよびHAVINGを含むクエリの例。このクエリでは、ユーザーごとの最初のテーブルが最初にフィルター処理され、都市ごとの顧客数が計算され、顧客数が5以上の都市のみが残ります。


select City, count(CustomerID) as number_of_clients from Customers
WHERE CustomerName not in ('Around the Horn','Drachenblut Delikatessend')
group by City
HAVING number_of_clients >= 5

注文する


ORDER BYは、テーブルのソートを担当するオプションのクエリ要素です。

1列で並べ替える簡単な例。このリクエストでは、クライアントが指定した都市によるソートが実行されます。


select * from Customers
ORDER BY City

複数の列で並べ替えることができます。この場合、指定した列の順に並べ替えが行われます。


select * from Customers
ORDER BY Country, City

デフォルトでは、並べ替えは数値では昇順、テキスト値ではアルファベット順になります。逆ソートが必要な場合は、ORDER BY句の列名の後にDESCを追加します。


select * from Customers
order by CustomerID DESC

1列による逆ソートと2列目のデフォルトのソート：

select * from Customers
order by Country DESC, City

参加する


JOIN-両方のテーブルに存在するキーによってテーブルを結合するために使用されるオプションの要素。キーの前にはONステートメントがあります。

OrderテーブルとCustomerテーブルをCustomerIDキーで結合し、テーブル名をドットを介してキーの列名の前に追加するクエリ：

select * from Orders
JOIN Customers ON Orders.CustomerID = Customers.CustomerID

あるテーブルを別のテーブルの値でマップする必要がある場合、状況が発生することがあります。タスクに応じて、さまざまなタイプの接続を使用できます。INNER JOIN-交差、RIGHT / LEFTJOIN。1つのテーブルを別のテーブルからの知識でマッピングします。


select * from Orders
join Customers on Orders.CustomerID = Customers.CustomerID
where Customers.CustomerID >10
