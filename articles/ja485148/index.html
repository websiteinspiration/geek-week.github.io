Unicode標準は、主要な現代言語の文字のコードを定義しています。これらは、ヨーロッパのアルファベット文字、中東の文字、右から左、および多くのアジアの文字です。

標準には、句読点、分音記号、数学記号、技術記号、矢印、絵文字、絵文字なども含まれています。ティルダ（〜）などの文字記号を変更する分音記号のコードを提供します。これらは、アクセント付き文字（例えば、ñ）を表すために基本的なものと組み合わせて使用​​されます。一般に、Unicodeバージョン9.0は、世界のアルファベット、表意文字セット、および文字コレクションからの128,172文字のコードを提供します。

最も一般的な文字は、最初の64Kコードポイント、メインの多言語面と呼ばれるコードスペースの領域、または略してBMPに配置されます。他の文字をエンコードするために使用できる他の16の追加プレーンがあり、850,000を超える未使用のコードポイントがあります。これらは、標準の将来のバージョンに新しい文字を追加するのに便利です。

Unicode規格は、個人使用のためにコードポイントも予約しています。ベンダーまたはエンドユーザーは、独自のシステムで文字を指定したり、特殊なフォントで使用したりできます。 BMPには、6400では特定のアプリケーションに十分ではない場合、私用の6400コードポイントと私用の別の131 068コードポイントがあります。

Unicode文字エンコーディング


文字コーディング標準は、各文字のIDとその数値またはコードポイントだけでなく、この値がビットでどのように表されるかを決定します。

Unicode規格では、同じデータの伝送を可能にする3つの形式のエンコーディングが定義されています。バイト、ワード、ダブルワード（つまり、コードの単位あたり8、16、または32ビット）。3つの形式はすべて同じ共通の文字セットをエンコードし、データを失うことなく互いに効果的に変換できます。Unicodeコンソーシアムは、これらのエンコード形式の使用を、Unicode標準を実装するための合意された方法として完全に支持しています。

UTF-8はHTMLおよび類似のプロトコルで一般的です。 UTF-8は、すべてのUnicode文字を可変バイト長エンコーディングに変換する方法です。その利点は、おなじみのASCIIセットに対応するUnicode文字がASCIIと同じバイト値を持ち、UTF-8に変換されたUnicode文字が、ソフトウェアを大幅に変更しなくても、既存の多くのソフトウェアで使用できることです。

UTF-16は、文字への効率的なアクセスと経済的なストレージのバランスをとる必要がある多くの環境で人気があります。これは非常にコンパクトであり、頻繁に使用されるすべての文字は1つの16ビットコードブロックに配置されますが、他のすべての文字は16ビットコードブロックのペアを通じて使用できます。

UTF-32は、メモリの量が問題にならない場合に便利ですが、単一の固定幅コードの文字にアクセスする必要があります。ここでは、各Unicode文字が単一の32ビットコードブロックにエンコードされています。

3つの形式のエンコーディングはすべて、各文字に4バイト（または32ビット）以下を必要とします。

サロゲートペアの計算


UTF-16のUnicode文字「Pile of shit」（U + 1F4A9）は、サロゲートペア、つまり2つのサロゲートとしてエンコードする必要があります。コードポイントをサロゲートペアに変換するには、（JavaScriptで）このアルゴリズムを使用します。16進数表記を使用していることに注意してください。

 var High_Surrogate = function(Code_Point){ return Math.floor((Code_Point - 0x10000) / 0x400) + 0xD800 };
 var Low_Surrogate  = function(Code_Point){ return (Code_Point - 0x10000) % 0x400 + 0xDC00 };

 // Reverses The Conversion
 var Code_Point = function(High_Surrogate, Low_Surrogate){
	return (High_Surrogate - 0xD800) * 0x400 + Low_Surrogate - 0xDC00 + 0x10000;
 };



構成と分解


Unicodeには、文字の形状を変更するためのメカニズムが含まれており、サポートされるグリフのセットを大幅に拡張します。これは、結合可能な発音区別符号に適用されます。彼らは主人公の後に挿入されます。複数の結合可能な発音区別符号を同じマークに適用できます。 Unicodeには、通常の使用のためにこれらのほとんどの組み合わせのコンパイル済みバージョンも含まれています。

一部の文字シーケンスは、合成文字（複合文字）と呼ばれる単一の文字として表すこともできます。たとえば、文字[ü]は、単一のコードポイントU + 00FCまたはベース文字U + 0075（u）としてエンコードでき、その後に非独立文字U + 0308（¨）を続けます。 Unicode標準は、[ü]や[ñ]などの多くの複合文字を含むLatin 1などの確立された標準との互換性のために複合文字をエンコードします。

複合文字は、一貫性または分析のために拡張できます。たとえば、アルファベット順に並べ替える場合、記号[ü]は[u]に分解され、その後に独立記号[¨]が続きます。このような分解の後、一連の文字を処理するアルゴリズムは簡単になります。これにより、文字修飾子がアルファベット順に影響しない言語での並べ替えが簡単になります。Unicode標準は、すべての複合文字の分解順序を設定します。また、文字の一意の表現を提供する正規化の形式も定義します。

function transformAllTags (newName){
   // querySelectorAll doesn't actually return an array.
   Array.from(document.querySelectorAll('*'))
     .forEach(function(x){
         transformTag(x, newName);
   });
}

function wonky(str){
  return str.split('').join('\u{20de}') + '\u{20de}';
}

function transformTag(tagIdOrElem, tagType){
    var elem = (tagIdOrElem instanceof HTMLElement) ? tagIdOrElem : document.getElementById(tagIdOrElem);
    if(!elem || !(elem instanceof HTMLElement))return;
    var children = elem.childNodes;
    var parent = elem.parentNode;
    var newNode = document.createElement(tagType||wonky(elem.tagName));
    for(var a=0;a<elem.attributes.length;a++){
        newNode.setAttribute(elem.attributes[a].nodeName, elem.attributes[a].value);
    }
    for(var i= 0,clen=children.length;i<clen;i++){
        newNode.appendChild(children[0]); //0...always point to the first non-moved element
    }
    newNode.style.cssText = elem.style.cssText;
    parent.replaceChild(newNode,elem);
}
