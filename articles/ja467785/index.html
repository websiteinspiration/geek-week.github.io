大規模なデータセットを扱うときにパンダを最適化する方法（エッセイ）

車のメモリや小さなデータセットを最適化せずにパンダに安全に投入できる場合。ただし、データが大きい場合は、それをどのように処理するか、少なくとも数える方法が問題になります。

ネットワークから巨大なデータセットを引き出さないように、最適化をミニチュアで見ることが提案されています。

データセットとして、私たちは1人の勤勉なユーザーのおかげで公開されている、2019年のためのユーザーのコメントをhabrastatisticsを使用します。
データセットを

情報基盤として、我々は以前に翻訳使用します記事面白いことをたくさんミックスHabr、からを。

参加する代わりに


habrastatisticsデータセットは小さいと見なされますが、288 MBを占め、448533行で構成されています。
もちろん、もっと多くのデータを見つけることができますが、車を吊るさないように、詳しく考えてみましょう。

操作の便宜上、列名を追加します（ファイルの最初の行を書き込むだけです）。

a,b,c,d

ここで、データセットをパンダに直接ロードして、使用するメモリの量を確認すると

import os
import time
import pandas as pd
import numpy as np
gl = pd.read_csv('habr_2019_comments.csv',encoding='UTF')
def mem_usage(pandas_obj):
    if isinstance(pandas_obj,pd.DataFrame):
        usage_b = pandas_obj.memory_usage(deep=True).sum()
    else: #     ,     DataFrame,   Series
        usage_b = pandas_obj.memory_usage(deep=True)
    usage_mb = usage_b / 1024 ** 2 #    
    return "{:03.2f} MB".format(usage_mb)
print (gl.info(memory_usage='deep'))

また、オブジェクトタイプのデータを含む列を扱っていることも示しています。
したがって、記事によれば、列については、オブジェクトに対して計算された最適化に焦点を合わせる必要があります。1つを除くすべての列。

列bには日付が含まれています。さらに計算しやすくするため、データセットのインデックスに送信することをお勧めします。これを行うには、データセットを読み取るときに使用されるコードを変更します。

gl = pd.read_csv('habr_2019_comments.csv', parse_dates=['b'], encoding='UTF')

これで、日付がデータセットのインデックスとして読み取られ、メモリ使用量がわずかに減少しました。

memory usage: 407.0 MB

次に、列とインデックスの外側のデータセット自体のデータを最適化します


最適化は、「カテゴリ変数を使用したオブジェクトタイプのデータストレージの最適化」と呼ばれます。

ロシア語に翻訳する場合は、列内のデータをカテゴリごとに組み合わせる必要があります。

有効性を判断するには、列内の一意の値の数を知る必要があります。列内の値の総数の50％未満の場合は、カテゴリ内の値を組み合わせると効果的です。

データセットを見てみましょう：


gl_obj=gl.select_dtypes(include=['object']).copy()
gl_obj.describe()

列ご覧のとおり、行aとcの一意の行から、カテゴリの結合が有効です。列aの場合、これは25100ユーザー（明らかに448533未満）です。cの場合、「+」と「-」を含むスケールの185の値（これも448533を大幅に下回ります）。

列を最適化します。

for col in gl_obj.columns:
    num_unique_values = len(gl_obj[col].unique())
    num_total_values = len(gl_obj[col])
    if num_unique_values / num_total_values < 0.5:
        converted_obj.loc[:,col] = gl_obj[col].astype('category')        
    else:
        converted_obj.loc[:,col] = gl_obj[col]

便宜上使用されるメモリ量を理解するために、関数を紹介します。

def mem_usage(pandas_obj):
    if isinstance(pandas_obj,pd.DataFrame):
        usage_b = pandas_obj.memory_usage(deep=True).sum()
    else: #     ,     DataFrame,   Series
        usage_b = pandas_obj.memory_usage(deep=True)
    usage_mb = usage_b / 1024 ** 2 #    
    return "{:03.2f} MB".format(usage_mb)

そして、最適化が有効であったかどうかを確認します。

>>> print('  : '+mem_usage(gl_obj))
  : 407.14 MB
>>> print('  : '+mem_usage(converted_obj))
  : 356.40 MB
>>>

ご覧のとおり、さらに50 MBのゲインを受け取りました。

これで、最適化のメリットがあることがわかったので（発生する可能性があり、その逆も可能です）、処理するデータをすぐに考慮するために、読み取り時にデータセットパラメーターを設定します。

gl = pd.read_csv('habr_2019_comments.csv', parse_dates=['b'],index_col='b',dtype ={'c':'category','a':'category','d':'object'}, encoding='UTF')
