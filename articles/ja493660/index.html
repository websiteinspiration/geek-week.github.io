CSS：完全なcalc（）関数ガイド

CSSには、calc()簡単な計算を実行するために使用される特別な関数があります。以下はその使用例です。

.main-content {
  /*  80px  100vh */
  height: calc(100vh - 80px);
}

ここでは、それが使用されているCSSコードをcalc()試すことができます。 この記事の著者は、その翻訳を今日公開していますが、この非常に便利な機能について知っておく価値のあるすべてのことについて話したいと思っています。





Calc（）関数とCSSプロパティ値


関数を使用できる唯一の場所calc()は、CSSプロパティ値です。この関数を使用して、さまざまなプロパティの値を設定する次の例を見てください。

.el {
  font-size: calc(3vw + 2px);
  width:     calc(100% - 20px);
  height:    calc(100vh - 20px);
  padding:   calc(1vw + 5px);
}

この関数calc()は、プロパティの別の部分を設定するためにも使用できます。

.el {
  margin: 10px calc(2vw + 5px);
  border-radius: 15px calc(15px / 3) 4px 2px;
  transition: transform calc(1s - 120ms);
}

この関数は、特定のプロパティの一部を形成する役割を持つ別の関数の一部である場合もあります。たとえば、ここではcalc()、グラデーションの色の変化の位置を調整するために使用されます。

.el {
  background: #1E88E5 linear-gradient(
    to bottom,
    #1E88E5,
    #1E88E5 calc(50% - 10px),
    #3949AB calc(50% + 10px),
    #3949AB
  );
}

calc（）関数は、数値プロパティを操作するためのツールです。


上記のすべての例では、calc()数値プロパティを操作するときに関数を使用していることに注意してください。また、数値プロパティを操作するいくつかの機能についても説明します（これらの機能は、測定単位の使用が不要な場合があるという事実に関連しています）。この関数は、文字列やその他の関数ではなく、数値を使用して演算を実行するように設計されていることに注意してください。

.el {
  /* ! */
  counter-reset: calc("My " + "counter");
}
.el::before {
  /* ! */
  content: calc("Candyman " * 3);
}

要素とその部品のサイズを指定するCSSで使用することができ、測定の多くの単位がありますpx、%、em、rem、in、mm、cm、pt、pc、ex、ch、vh、vw、vmin、vmax。これらの単位はすべて、機能で使用できますcalc()。

この関数は、単位を指定せずに使用される数値でも機能します。

line-height: calc(1.2 * 1.2);

角度の計算にも使用できます。

transform: rotate(calc(10deg * 5));

この関数は、渡された式で計算が実行されない場合にも使用できます。

.el {
  /*   ,    */
  width: calc(20px);
}

Calc（）はメディアクエリでは使用できません


この関数はCSSプロパティ値を設定するように設計されていますが、メディアクエリでは機能しません。

@media (max-width: 40rem) {
  /* 40rem   */
}

/*  ! */
@media (min-width: calc(40rem + 1px)) {
  /* ,  40rem */
}

このような構成が機能することが証明されれば、非常に論理的に見える相互に排他的なメディアクエリ（たとえば、上記のもの）を作成できるため、これは非常に適切です。

1つの式で異なる単位を使用する


1つの式で異なる測定単位を使用することの許容性は、おそらく最も貴重な機会calc()です。上記の例のほとんどすべてで同様のものが使用されています。ここでは、注意を引くために、さまざまな測定単位の使用を示す別の例を示します。

/*      */
width: calc(100% - 20px);

この式は次のように解釈されます。「幅は、20ピクセルが差し引かれる要素の幅と同じです。」

要素の幅が変化する可能性がある状況では、ピクセルで表されたインジケーターだけを使用して目的の値を事前に計算することは完全に不可能です。つまり、calc()Sassなどを使用して前処理を行ったり、ポリフィルなどを取得したりすることはできません。はい、ブラウザはcalc()非常によくサポートしているので、これは必要ありません。ここでのポイントは、異なる単位で表された値が混合されるそのような計算は、ブラウザーで実行される必要があることです（ページの「実行中」）。つまり、そのような計算を実行する能力が主な値calc()です。

さまざまな単位で表された値の使用例をいくつか次に示します。

transform: rotate(calc(1turn + 45deg));

animation-delay: calc(1s + 15ms);

これらの式は、ページがブラウザーで実行されている間に決定されたものに関連しない単位が混在しているため、おそらく前処理することができます。

calc（）とプリプロセッサー計算の比較


先ほど述べたように、最も便利な機能はcalc()前処理に適していません。ただし、前処理により、開発者は機能に一致するいくつかの機能を利用できますcalc()。たとえば、Sassを使用する場合は、プロパティ値も計算できます。

$padding: 1rem;

.el[data-padding="extra"] {
  padding: $padding + 2rem; //  3rem;
  margin-bottom: $padding * 2; //  2rem; 
}

ここでは、測定単位を示す計算を行うことができます。ここでは、同じ測定単位で表された数量を追加できます。特定の値に、単位が示されていない値を掛けることができます。ただし、異なる測定単位で表された値を使用して計算を実行することはできません。そのような計算には、制限に似た制限が課されますcalc()（たとえば、何かを乗算または除算する数値は、単位のない値でなければなりません）。

CSSで使用されている数値の意味を明らかにする


ヘルプcalc()でのみ達成できる機会を利用しない場合でも、この関数を使用して、 CSSで使用される値の意味を明らかにできます。プロパティの値を要素の幅のちょうど1/7にしたいとします。

.el {
  /*   , */
  width: calc(100% / 7);

  /*   */
  width: 14.2857142857%;
}

このアプローチは、一種の自己スタイルのCSS-APIのようなもので役立つ場合があります。

[data-columns="7"] .col { width: calc(100% / 7); }
[data-columns="6"] .col { width: calc(100% / 6); }
[data-columns="5"] .col { width: calc(100% / 5); }
[data-columns="4"] .col { width: calc(100% / 4); }
[data-columns="3"] .col { width: calc(100% / 3); }
[data-columns="2"] .col { width: calc(100% / 2); }

calc（）関数の数学演算子


助けを借りて評価される表現calc()に使用できる演算子+、-、*と/。しかし、それらのアプリケーションにはいくつかの機能があります。

（+）を加算および（）を減算するとき-は、指定された単位で値を使用する必要があります

.el {
  /*  */
  margin: calc(10px + 10px);

  /*  */
  margin: calc(10px + 5);
}

誤った値を使用すると、特定のアナウンスも正しくなくなることに注意してください。

（/）を除算する場合、2番目の数字に測定単位を示さないようにする必要があります

.el {
  /*  */
  margin: calc(30px / 3);

  /*  */
  margin: calc(30px / 10px);

  /*  (   ) */
  margin: calc(30px / 0);
}

（*）を乗算する場合、数値の1つに測定単位があってはなりません。

.el {
  /*  */
  margin: calc(10px * 3);

  /*  */
  margin: calc(3 * 10px);

  /*  */
  margin: calc(30px * 3px);
}

スペースの重要性


式で使用されるスペースは、加算および減算演算で重要です。

.el {
  /*  */
  font-size: calc(3vw + 2px);

  /*  */
  font-size: calc(3vw+2px);

  /*  */
  font-size: calc(3vw - 2px);

  /*  */
  font-size: calc(3vw-2px);
}

ここでは、負の数を使用することはかなり可能です（たとえば、のような構成でcalc(5vw — -5px)）。これは、スペースが必要であるだけでなく、式の明確さの点でも役立つ状況の例です。

タブアトキンス氏は、加算演算子と減算演算子をスペースで区切る必要がある理由は、実際には式の解析の特殊性だと私に言った。これを完全に理解したとは言えませんが、たとえば、パーサーは式を測定単位を持つ2px-3px数値として処理します2px-3px。そして、誰もそのような奇妙な測定単位を必要としません。加算演算子を使用した式の解析には、独自の問題があります。たとえば、パーサーはこの演算子を数値の記述に使用される構文の一部として使用できます。カスタムプロパティ（--）の構文を正しく処理するにはスペースが必要だと思いましたが、そうではありません。

演算子の周りのスペースを乗算およ​​び分割する場合は必要ありません。しかし、コードを読みやすくするため、およびスペースを忘れないようにするため、および加算と減算を使用する式を入力するときに、これらの演算子でスペースを使用することをお勧めできると思います。

大括弧とcalc()式を区切るスペースは何の役割も果たしません。式は、必要に応じて、新しい行に移動して選択することもできます。

.el {
  /*  */
  width: calc(
    100%     /   3
  );
}

確かに、ここでは注意が必要です。関数の名前（calc）と最初の開き角括弧の間にスペースを入れないでください：

.el {
  /*  */
  width: calc (100% / 3);
}

入れ子構造：calc（calc（））


関数calc()を使用する場合は、ネストされた構造を使用できますが、これは必ずしも必要ではありません。これは、なしでブラケットを使用することに似ていますcalc：

.el {
  width: calc(
    calc(100% / 3)
    -
    calc(1rem * 2)
  );
}

ネストされた構造を関数から構築しても意味がありません。これは、calc()ブラケットのみを使用して書き換えることができるためです。

.el {
  width: calc(
   (100% / 3)
    -
   (1rem * 2)
  );
}

さらに、演算子の優先順位を決定するためのルールは、ここに示されている式を計算するときに適用されるため、この例では括弧は必要ありません。除算と乗算は、加算と減算の前に実行されます。その結果、コードは次のように書き直すことができます。

.el {
  width: calc(100% / 3 - 1rem * 2);
}

しかし、追加のブラケットがコードをより明確にするように思われる場合は、それらを使用できます。さらに、括弧がない場合、演算子の優先順位のみに基づいて式が正しく計算されないため、そのような式には括弧が必要です。

.el {
  /* ,   , */
  width: calc(100% + 2rem / 2);

  /*     ,    */
  width: calc((100% + 2rem) / 2);
}

カスタムCSSプロパティとcalc（）


優れた機能の1つ、calc()異なる単位で値を使用する計算についてはすでに学習しました。この関数のもう1つの興味深い機能は、カスタムCSSプロパティに適用する方法です。カスタムプロパティには、計算で使用できる値を割り当てることができます。

html {
  --spacing: 10px;
}

.module {
  padding: calc(var(--spacing) * 2);
}

カスタムCSSプロパティの値を設定することにより、多くの設定が1か所で行われるCSSスタイルのセットを想像するのは簡単だと思います。これらの値は、CSSコード全体で使用されます。

さらに、カスタムプロパティは相互に参照できます（calc()ここでは使用されません）。取得した値は、他のCSSプロパティの値を設定するために使用できます（ただし、ここではcalc()これを行う必要があります）。

html {
  --spacing: 10px;
  --spacing-L: var(--spacing) * 2;
  --spacing-XL: var(--spacing) * 3;
}

.module[data-spacing="XL"] {
  padding: calc(var(--spacing-XL));
}

カスタムプロパティにアクセスするときは、について覚えておく必要があるため、これはあまり便利ではないように見えるかもしれませんcalc()。しかし、これはコードの読みやすさの点で興味深いと思います。

カスタムプロパティのソースはHTMLコードにすることができます。これは非常に役立つ場合があります。たとえば、Splitting.jsでは、単語と記号にインデックスが追加されます。

<div style="--index: 1;"> ... </div>
<div style="--index: 2;"> ... </div>
<div style="--index: 3;"> ... </div>
div {
  /*     HTML (   ) */
  animation-delay: calc(var(--index, 1) * 0.2s);
}

これらのプロパティを宣言した後のカスタムプロパティへのユニットの割り当て


カスタムプロパティに測定単位なしで数値を書き込むのが理にかなっている状況、または実際に使用する前に、そのような数値が便利な状況で、なんらかの方法で単位を使用せずに変換するとします。このような場合、単位を指定せずにカスタムプロパティに値を割り当てることができます。この数値を特定の測定単位で表された新しい数値に変換する必要がある場合1は、目的の測定単位を示すことで乗算できます。

html {
  --importantNumber: 2;
}

.el {
  /*    ,  ,        */
  padding: calc(var(--importantNumber) * 1rem);
}

花を扱う


RGBやHSLなどの形式を使用して色を記述する場合、数値が使用されます。これらの数値は、上で処理することができますcalc()。たとえば、いくつかの基本的なHSL値を設定し、必要に応じて変更できます（例は次のとおりです）。

html {

calc（）とattr（）を組み合わせることはできません


CSS機能attr()は非常に魅力的に思えるかもしれません。そして真実：HTMLから属性値を取得して、それを使用します。だが…

<div data-color="red">...</div>
div {
  /*    */
  color: attr(data-color);
}

残念ながら、この関数は値の「タイプ」を認識しません。そのため、attr()文字列の操作と、それを使用したCSSプロパティの設定にのみ適していcontentます。つまり、そのような設計は非常にうまく機能していることがわかります。

div::before {
  content: attr(data-color);
}

誰かがHTMLコードからattr()特定の数値を抽出して計算に使用しようとする可能性があるため、ここでこれを述べました。

<div class="grid" data-columns="7" data-gap="2">...</div>
.grid {
  display: grid;

  /*         */
  grid-template-columns: repeat(attr(data-columns), 1fr);
  grid-gap: calc(1rem * attr(data-gap));
}

ただし、HTMLのカスタムプロパティはそのような問題を解決するのに優れているため、問題はないというのは良いことです。

<div class="grid" style="--columns: 7; --gap: 2rem;">...</div>
.grid {
  display: grid;

  /* ! */
  grid-template-columns: repeat(var(--columns), 1fr);
  grid-gap: calc(var(--gap));
}

ブラウザのサポート


ここでは、この機能のcalc()ブラウザサポートについて確認できます。最新のブラウザについて言えば、サポートのレベルはcalc()97％以上です。かなり古いブラウザ（IE 8やFirefox 3.6など）をサポートする必要がある場合は、通常、これを行います。calc()プロパティの前に同じプロパティを追加して、古いブラウザが理解できる形式で設定されている値を計算します。

.el {
  width: 92%; /*   */
  width: calc(100% - 2rem);
}

この機能にcalc()は多くの既知の問題がありますが、問題が発生するのは古いブラウザのみです。上Caniuse 13のこのような問題の説明を見つけることができます。それらの一部を次に示します。

  • バージョン59より前のFirefoxブラウザcalc()は、色の設定に使用される関数をサポートしていません。次に例を示しますcolor: hsl(calc(60 * 2), 100%, 50%)。
  • IE 9-11はbox-shadow、値の決定に使用される場合、プロパティで指定されたシャドウをレンダリングしませんcalc()。
  • width: calc()テーブルセルに適用した場合、IE 9-11もEdgeもビューデザインをサポートしません。
