一般レベルでのC ++ / Visual BasicとJavaの違い（初心者と学生向け）

C ++とJavaは、非常に人気のあるプログラミング言語の一部です。それらは、速度、タイピング、コンパイラーなどを比較して非常に詳細に比較できますが、平均的な素人にとっては多すぎるため、一般的で基本的なレベルでそれらの違いを検討するのははるかに簡単です。このエリアを見ているだけです。すでにこの段階で、どのパラダイムのどの言語があなたに適しているかを結論付けることができます。そして、はい、そのような選択の状況はかなり現実的です-たとえば、教育目的の学生にとって。

これを行うために、プログラミング言語の主な構成、つまり次のことを簡単に検討します。

式：数学、論理、文字列;
演算子：複合ステートメント、代入ステートメント、条件付きIFステートメント、FORおよびWHILEループ、GOTOステートメントなど。
サブプログラム：プロシージャと関数。
データ型：基本型、構造、クラス、インターフェース、およびテンプレート。

（この記事は翻訳ではありませんが、収集された情報を個人的に分析しています）

式


ジャワ


数式


数式は、結果として数値になる式です。変数、関数、定数が含まれます。

Javaの算術演算子を使用すると、加算、減算、乗算などの標準的な数学演算を実行できます。言語には、デクリメント演算子とインクリメント演算子も含まれています（値を1ずつ増減します）。

論理式


論理式は、真（true）または偽（false）になる式の記述です。論理演算子はブール型のオペランドでのみ機能します。
2つのオペランドを持つすべての論理演算子は、2つの論理値を組み合わせて、結果の論理値を形成します。

Javaは、AND（論理および省略）、OR（論理および省略）、XOR、NOTなど、10を超えるステートメントをサポートしています。

文字列式


文字列式-文字列操作。数式のように見えますが、結果は新しい行です。 

Java言語では、文字列を追加、削除、分割、および結合できます。Javaの特徴は、文字列を操作するために特別に提供されているStringクラスの存在です。

C ++ / Visual Basic


数式


C ++およびVisual Basicには、算術演算子、減分および増分演算子も含まれています。また、数式構文では、角かっこと単項マイナスもサポートされています。

Java言語と同様に、組み込み関数として使用できるより複雑な演算、たとえば平方根sqrt（x）と指数exp（x）があります。

論理式


Visual Basicは6つの演算子を使用します。AND、OR、NOT、XORに加えて、等価演算子と含意演算子も使用されます。

C ++では、AND、NOT、ORの3つの論理演算子を使用します。検討中のすべての言語で、いくつかの演算子は一度に複数のオペランドに使用できます。

文字列式


Javaとは異なり、C ++では、文字列などのオブジェクトの組み込みサポートはなく、必要に応じて、ヘッダーファイル文字列を含む文字列（加算、減算など）を処理します。

Visual Basicでは、文字列の操作が重要であるため、文字のシーケンスを表すStringデータ型がサポートされています。このタイプの変数は、文字列になる任意の式を取ることができます。

オペレーター


ジャワ


問題のすべての言語は、IF条件演算子であるループ演算子（FOR、WHILE）をサポートするという点で類似していますが、それらの重要な違いは、Javaにはgoto遷移演算子がないため、constキーワードとgotoキーワードが保存されることです。ただし、使用されません。

C ++ / Visual Basic


Javaよりはるかに早く開発されたC ++およびVisual Basicには、goto遷移演算子がありますが、この演算子の使用は一般に望ましくありません。

手順と機能


ジャワ


サブプログラムは、名前の付いた論理的に完全な言語演算子のグループであり、プログラムのさまざまな場所から何度でも呼び出すことができます。関数はメインプログラムに値を返しますが、プロシージャは戻りません。
Javaでは、プロシージャや関数はありません。これらのエンティティはどちらもメソッドと呼ばれますが、値を返すこともしないこともできます。また、パラメータの有無にかかわらず呼び出すことができます。

C ++ / Visual Basic


C ++では、ルーチンは関数として実装されます。C ++プログラムは、使用する前に説明する必要がある1つ以上の関数で構成されています。
Visual Basicプログラムは、手順と機能に分割されたいくつかのルーチンで構成されています。

データ型


ジャワ


Javaでは、すべての基本的な数値型に対して、特定の規則が独立して使用されます。これらの規則は、環境の特定の実装に依存せず、幅広いプラットフォームで再現可能な結果を​​保証します。また、Javaには構造体と共用体の構成体がなく、これらすべてがクラスの概念に組み込まれています。

さらに、Javaはポインター（いわゆる「リンク」がJavaで役割を果たす）、テンプレート、オペレーターのオーバーロード、構造などをサポートしていません。

C ++ / Visual Basic


C ++は、Cからすべての通常のデータ型を継承します。これらの型は、さまざまなサイズと精度の整数と実数を表すために使用されます。残念ながら、これらのタイプの実際の範囲と精度は、トランスレータの特定の実装によって異なります。

異なるプラットフォームのトランスレータへの依存関係のため、C ++およびVisual Basicコードの動作は変更される可能性があります。C ++は、構造体（構造体）および共用体（共用体）を使用した多態性を宣言することにより、プログラマーにデータをカプセル化する機能を提供しようとしています。これら2つの設計は、サイズと配置の重要で壊滅的なマシン固有の制限をカバーしています。

C ++およびVisual Basicは、ポインターとポインター演算、演算子のオーバーロードなどをサポートしています。

概要


Java言語は、Visual Basic、特にC ++を含む、それ以前に存在していた言語の影響下で開発されました。つまり、これらの言語の特定の数の構造が一致していますが、多くのC ++の欠陥がJavaで修正され、プログラムのパフォーマンスと結果に影響を与えましたまた、C ++と比較して言語を「単純化」するための対策が取られました。これは、JavaとC ++の間に大きな違いがあることを意味します。類似の構成要素があるにもかかわらず、JavaはC ++やVisual Basicとは大きく異なります。
