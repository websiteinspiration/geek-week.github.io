セキュリティウィーク33：Black Hatで興味深い/ DEF CON 2019

次回のBlack Hat / DEF CONダブルカンファレンスは先週ラスベガスで開催されました。最初のイベントがビジネスのライバルに向かって順調に進んだ場合、2番目のイベントはハッカーにとって最良の会議であり（主に言葉の意味で）、ハードウェアとソフトウェアの弱点の検索は主に芸術であり、その後はお金を稼ぐ方法です。生活のために。昨年、これらの2つの会議に基づいてダイジェストでは、我々は話を時代遅れVIA C3プロセッサの脆弱性については、アップルコンピュータ上で、不要のWi-Fi SDアダプタをハッキングについてのサプライチェーンの攻撃。

今年の議題には、プリンター、オフィスの電話、子供用タブレットへの攻撃、Apple FaceIDの顔認識システムの半円形のバイパス、そしてキャノンのカメラのランサムウェア暗号化機能が含まれていました。プラス2、たとえば、アートプロジェクト：バックドア付きのiPhone用ケーブルとスマートエアロバイクでの不正行為。最後に、GDPRを使用して他の人の個人情報を盗むことについての興味深い研究。待ってください、GDPRは個人データの保護を目的とした法律ですか？まあ、私達もそう考えました。

プリンター、オフィス電話、子供用タブレットのハッキング


比較的退屈な研究から始めましょう。 NCCグループの専門家は、HP、リコー、ゼロックス、レックスマーク、京セラ、およびブラザーによって製造されたオフィスプリンターに多くの脆弱性を発見しました。問題の大きさは、HPプリンターのレポートの例で評価できます。IPPネットワーク印刷プロトコルに脆弱性があり、組み込みWebサーバーにバッファーオーバーフローがあり、クロスサイトスクリプティング攻撃につながるエラーがあります。最も危険な脆弱性を利用すると、DoS攻撃を組織化したり、理解できない結果をもたらす任意のコードを実行したりすることができます。

昨年発生した事件a）保護されていない、b）インターネットからアクセス可能なプリンターに何が起こるかを示しています。 5万台のデバイスで、匿名の「活動家」が1つのいやらしいYouTubeをサブスクライブするためのコールを印刷しました。 NCCグループの調査では、プリンターが企業ネットワークへのさらなる攻撃のエントリポイントになる可能性がある場合、愚かさが少なく危険な攻撃の兆候が見られます。



しかし、プリンターではなくオフィスの電話を使用するとどうなるでしょうか。このオプションは、McAfeeの担当者によって調査されました。彼はAvaya VoIP電話に10年前のファームウェアを見つけました。特に、電話で使用されているdhclientクライアントには、2009年以降に知られている脆弱性がありました。この問題は、サブネットマスクパラメータが長すぎてDHCPクライアントに渡せない場合にバッファオーバーフローを引き起こす可能性があり、繰り返しになりますが、理論的には任意のコードが実行される危険があります。したがって、「仕事-触らないでください」というアプローチは、オフィスの電話などの比較的単純なデバイスの場合でも適用できません。ローカルネットワークにアクセスできる場合、dhclientの脆弱性が悪用されますが、パッチを電話に適用する価値はあります。触れる必要があります！



調査LeapFrogのLeadPad Ultimateの子供用タブレットは、子供時代の脆弱性を明らかにしました。 Chekmarxの専門家は、すぐに利用できるツールを使用してデバイスを特定し、トラフィックを傍受する方法を示しました。最も深刻な問題は、追加のアプリケーションペットチャットによって引き起こされます。これは、タブレットに「ワイヤード」された一連のフレーズを使用して、親が子供と通信する（または子供が互いに通信する）ことを可能にします。接続を確立するために、タブレットはペットチャットと呼ばれるオープンアクセスポイントを作成します。特定の数のタブレットの地理位置情報は、Wi-Fiアクセスポイントのオープンデータベースで見つかりました。 「親のスマートフォン-子のタブレット」のペアの認証は提供されないため、ワイヤレスネットワークの範囲内の誰でもデバイスに接続できます。私たちはメーカーに敬意を払う必要があります：この問題は、アプリケーションをパブリックアクセスから削除することですぐに解決されました。そうそう、いずれにしても、タブレットはHTTP経由で製造元のサーバーと通信し、中間者攻撃などの攻撃を可能にしていました。


電気テープ付きメガネ、カメラ内のトロイの木馬、驚きのiPhone用ケーブル


より興味深いものに移りましょう。ただし、Black Hat / DEF CONでのプレゼンテーションの実用的でないトピックかもしれません。Tencentの研究者が明らかにした（ニュース、別のニュースHabré）AppleスマートフォンのFaceID顔認識システムを部分的にバイパスする方法。 FaceIDは、所有者の写真を使用してロックを解除しようとしたり、たとえば、眠っている人に携帯電話を持ち込んだりした場合に十分に保護されます。このため、顔の3次元モデルが作成され、目の動きが監視されます。しかし、ユーザーが眼鏡をかけている場合、おそらく認識の問題を回避するために、顔のこの部分の詳細な3Dモデルは作成されません。この場合でも、iPhoneは人の目が開いているかどうかをチェックし、カメラで生徒を識別します。しかし、認識品質が意図的に損なわれているため、「オープンアイ検出器」は電気テープを使用して回避されました。レンズに貼り付けられた暗い背景の薄い正方形は、電話によって生徒として識別されます。

わかりました。これはすばらしい調査ですが、実際には他の人の電話のロックを解除する方が少し簡単です。眠っている人に忍び寄り、電気テープで眼鏡をかける必要があります。または、人が無意識である最も楽しいシナリオではないと想定します。いずれにせよ、認識アルゴリズムは強化する価値があります。たとえ何らかの理由でメガネを使って3Dモデルを構築することが不可能な場合でも、そのような状況では、従来のカメラからの画像分析の品質を向上させることをお勧めします。


Check Point Software は Canon EOS 80Dをハッキングしました。なぜカメラ？彼らができたので！合計で6つの脆弱性がカメラで発見されましたが、最も深刻なのはコンピューターと通信するためのPTPプロトコルに存在します。結局のところ、PCに接続すると、ユーザーの確認なしでデバイスのファームウェアの更新を開始できます。強制的なフラッシュにより、研究者はカメラに本物のランサムウェア型トロイの木馬をインストールすることができました。それはメモリカード上の写真を暗号化し、画面に身代金要求を表示します。キヤノンはニュースレターをリリースしましたカメラの所有者向け（何らかの理由で、新しいファームウェアバージョンへの直接リンクがない）。報告によると、実際の被害者はいないため、疑わしいコンピューターにはカメラを接続しないことをお勧めします。実際、誰がカメラを攻撃するのでしょうか。


Vice は、Black Hatにバックドアが組み込まれた自家製のiPhoneケーブルを配布（および$ 200で販売）していた活動家について書きました。ただし、このケーブルの被害者は電話ではなく、同期のためにデバイスが接続されているコンピューターです（同期も機能します）。バックドアはWi-Fiアクセスポイントであり、明らかにキーボードエミュレーターです。記事の冒頭では、肯定的な意味で「ハッカー」という用語を使用しましたが、この実験は、いわば肯定的なものの下限にあります。ケーブル作成者は匿名のままで、会議エリアでブックマークを使用してケーブルを販売しました。そして彼はすべての200個を成功裏に販売しました。

最後に、Registerニュースサイトが注目を集めましたオックスフォード大学の学生であるJames Pavurによるプレゼンテーション。彼は社会実験を行い、ヨーロッパの法律GDPRの規範に従って情報を提供するための要件を150社に送信しました。法律は、ユーザーに個人データを収集および処理する会社から自分自身に関する情報を受け取る権利を与えます。歴史の重要なニュアンスは、ジェームズが彼自身についてではなく彼の花嫁についての情報を要求しなかったことです。調査した150社のうち、72％が要求に回答しました。

このうち84％は、研究者の花嫁についての情報があることを認めています。その結果、社会保障番号、生年月日、旧姓、クレジットカード番号、一部住所が取得されました。漏洩したパスワードに関する情報を収集しているある企業が、メールアドレスのパスワードのリストを送信しました。そして、これはすべての企業がデータを提供することに同意したわけではないという事実にもかかわらず：24％のケースでは、（自発的）被害者の電子メールアドレスと電話番号は識別に十分であり、16％のケースではIDスキャン（簡単に偽造）が必要であり、さらに3％の企業が必要でした提供せずにすべてのデータを削除しただけです。この実験は、GDPRの利点を反証するものではありません。むしろ、企業が常に新しい義務を履行する準備ができているわけではなく、ユーザーを識別するための一般に受け入れられている方法がまだないことを示しています。しかし、それは必要です：そうでなければ、法律を守ろうとする試みは、まったく逆の効果をもたらします。


ボーナスマテリアル：ブラッドディクソン研究員は、ゲーミフィケーションが組み込まれたエアロバイクをだます方法を発見しました（Viceの記事、ウェブサイト）事業）。 Zwiftシミュレーターを使用すると、後輪を負荷を提供するモジュールに交換することで、実際の自転車を自宅のトレーニング用のルームバージョンに変えることができます。シミュレーターには、仮想バイクに乗るためのソフトウェアが付属しており、記録と競争があります。 Zwiftは簡単な詐欺師（ドライバーをシミュレーターにねじ込む）を識別するため、Dixonは、センサーからの信号の傍受、妥当なトラフィックのエミュレートなど、重要な方法を使用して保護を回避する必要がありました。その結果、アクセルペダル（非常に速く進む）とクルーズコントロール（一定の速度で進み、センサーからの偽のデータをわずかに変更する）がシミュレーターに追加されました。 「これで、トレーニングを中断することなく、ビールまたは何か他のものに行く機会があります」と研究の著者はコメントしました。
