BeerTenderビールシステムのGrafanaダッシュボード

説明。 BeerTender-KrupsとHeinekenのビールを冷却して瓶詰めするための装置。メーカーによると、樽の開封後30日間は生ビールの品質を維持しているそうです。もちろん、システム管理者や開発者が通常のオンラインダッシュボードを使用して、樽内のビールの温度とレベルを監視すると便利です。前の記事では、BeerTenderをWarp 10に接続する方法について説明しました。



次に、ヨーロッパ最大のホスティングおよびクラウドプロバイダーであるダッシュボードGrafana OVHcloud を構成し、Warp 10プラットフォームをアクティブに使用します。監視データはすべて1つのWarp 10クラスターに流れます。これらは、40万台のサーバー、27のデータセンター、1秒あたり合計数百万のメトリックです。

それらには多くのダッシュボードがあり、OVHcloudはオープンソースプラグインWarp 10 Grafanaのメンテナーになりました。私たちは彼について話します。自分で試してみたい場合、データはパブリックドメインにあります。以下のWarpScriptをコピーできます。


BeerTenderをWarp 10に接続する方法についてはすでに説明しました。

取り付け


まず、Grafanaをインストールします。彼らのウェブサイトの指示に従ってください。

デフォルトのURLに接続し、デフォルトのhttp://localhost:3000/パスワードadmin / adminに移動して、管理者パスワードを変更します。

Grafanaの最新バージョンでは、プラグインを手動でインストールする必要があります。方法c grafana-cliは機能しません。プラグインリポジトリをGrafanaプラグインディレクトリに複製し、すべての場所で権限を修正する必要があります。

sudo systemctl stop grafana-server.service
sudo chown -R grafana:mygroup /var/lib/grafana/
sudo chmod g+rw /var/lib/grafana/plugins
git clone git@github.com:ovh/ovh-warp10-datasource.git /var/lib/grafana/plugins/ovh-warp10-datasource
sudo chown -R grafana:mygroup /var/lib/grafana/plugins
sudo systemctl start grafana-server.service

構成


Grafana構成で、Datasourcesセクションに移動し、タイプWarp 10のデータソースを追加します。それがWarp 10の必要なインスタンスを指していることを確認してください。同じ設定を指定できます。



BeerTenderからデータを取得する時が来ました...

温度表示


新しいダッシュボードを作成し、グラフ視覚化タイプのパネルを追加して、クエリモードに入ります：


初期温度（センサーからの整数値）

  • データソースで、ワープ10（またはデフォルト値）を選択します。
  • WarpScriptエディターをオンにする
  • 以下のWarpScriptをコピー
  • 右上隅で、「過去30日間」を選択します
  • 結果を保存する

"gCA1SVjbDkaxtmTx9ydI4TI2iGc5hFgcbCWnzMRZSt45XW8dZ53Z7VK_if28i0kXwNDLazXHgLrXUKgxLK0RbS79eJmBCpyBlIxw9US7bPfdWH4Fta51.kXN.D4Hsk5OZOwl.vLRBzMpP7F2pAMfclMXSGtCOT6F"
 'rt' STORE
 [ $rt 'beertender.rawtemperature' {} NOW 30 d ] FETCH 0 GET

このスクリプトは、ADCセンサーからの初期値を表示します。GrafanaのWarpScriptエディターは以前ほど強力ではないことに注意してください。ここではオートコンプリートはありません。WarpStudioまたはVSCodeでスクリプトを記述できるため、構文の強調表示とオンラインドキュメントが得られ、その結果をコピーしてGrafanaに貼り付けます。

自動間隔


この例では、30日間の間隔を選択し、過去30日間の表示モードをgraphanで設定します。最終日または最終年の表示モードを選択すると、WarpScriptは常に30日ごとにデータを要求します。プラグインはこの問題を解決します。WarpScriptで使用できる2つの変数- $endと$interval。

[ $rt 'beertender.rawtemperature' {} $end $interval ] FETCH 0

予想される補間

センサーからのソースデータは整数の形式で提供されます。補間する最も簡単な方法は、最初に曲線を作成し、次に ATTICK関数を使用してそこから度単位で値を読み取ることです。

WarpScriptでの補間は簡単です。バケットを作成して目的の期間を設定し、補間を開始して欠落している値を入力します。

[ $linearInterpolation bucketizer.last 0 1 0 ] BUCKETIZE 0 GET
INTERPOLATE 'truthtableGTS' STORE




いいよ 次に、センサーからの各値を実際の物理的な温度値に置き換えて、グラフェンで使用される形式に結果をもたらすために、独自のマッパーを作成する必要があります。

警告：値は10秒ごとに到着します。3か月で27万の値が蓄積されます。Grafanaには多すぎます... 1時間の最大値を維持してください。WarpScriptでは、バケット化バケットバケット化.maxを使用したBUCKETIZE です。

このWarpScriptをGrafanaにコピーできます。

// raw value interpolation
"gCA1SVjbDkaxtmTx9ydI4TI2iGc5hFgcbCWnzMRZSt45XW8dZ53Z7VK_if28i0kXwNDLazXHgLrXUKgxLK0RbS79eJmBCpyBlIxw9US7bPfdWH4Fta51.kXN.D4Hsk5OZOwl.vLRBzMpP7F2pAMfclMXSGtCOT6F"
'rt' STORE

NEWGTS 'linearInterpolation' RENAME
680 NaN NaN NaN 18.3 ADDVALUE
870 NaN NaN NaN 41.5 ADDVALUE
841 NaN NaN NaN 37.6 ADDVALUE
586 NaN NaN NaN 10 ADDVALUE
596 NaN NaN NaN 11 ADDVALUE
559 NaN NaN NaN 8 ADDVALUE
467 NaN NaN NaN 1 ADDVALUE
500 NaN NaN NaN 4.5 ADDVALUE
535 NaN NaN NaN 6.5 ADDVALUE
SORT 'linearInterpolation' STORE

[ $linearInterpolation bucketizer.last 0 1 0 ] BUCKETIZE 0 GET
INTERPOLATE 'truthtableGTS' STORE

[ $rt 'beertender.rawtemperature' {} NOW $end $interval ] FETCH 
// subsampling, keep max of every hour
[ SWAP bucketizer.max 0 1 h 0 ] BUCKETIZE
UNBUCKETIZE // do not try to interpolate missing buckets.
// interpolation
[ SWAP 
  <% 
    'l' STORE
    [
      $l 0 GET //same tick
      NaN NaN NaN  //no latitude/longitude/elevation
      $truthtableGTS $l 7 GET 0 GET ATTICK 4 GET //take the interpolated value
    ]
%> MACROMAPPER 0 0 0 ] MAP
'temperature (°C)' RENAME

バレルレベル


ひずみゲージはバレル内のレベルを測定します...しかし、これはさびた金属製のスタンドに取り付けられた安価な3線式センサーです。そのような機器から正確な情報を取得することは不可能です。したがって、出力が本当にノイズの多い結果であり、絶対値が信頼できないことは当然のことです。私たちのBeerTenderでは、現在のレベルをパーセンテージで表示するための実験的な数式をGrafanaで見つけました。

ヒント：Warp 10プラグインは、GTSからのみデータを取得します。単一の値を表示したい場合でも、GTSデータのある時点で表示する必要があります。

以下は、最後のデータポイントを取得してパーセンテージ値に変換し、1つのポイントで時系列を作成するWarpScriptです。

"gCA1SVjbDkaxtmTx9ydI4TI2iGc5hFgcbCWnzMRZSt45XW8dZ53Z7VK_if28i0kXwNDLazXHgLrXUKgxLK0RbS79eJmBCpyBlIxw9US7bPfdWH4Fta51.kXN.D4Hsk5OZOwl.vLRBzMpP7F2pAMfclMXSGtCOT6F"
'rt' STORE

//read for beer level
[ $rt 'beertender.rawvalue' {} NOW -1 ] FETCH 
0 GET VALUES 0 GET 'rawSensorValue' STORE

//raw correction: 
// empty: 30.8e6
// full (and cold): 30.5e6 

30.8e6 $rawSensorValue - 0 MAX 
2800 / 100 MIN   
'beerlevelPercent' STORE

$beerlevelPercent
NEWGTS 'Barre Level (%25)' RENAME
1 NaN NaN NaN $beerlevelPercent ADDVALUE

このWarpScriptを、ゲージの視覚化タイプを備えたパネルのリクエストとして使用し ます。
