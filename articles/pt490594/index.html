«FreeBSD: GNU/Linux» –




Um estudo interessante, embora controverso, do autor foi publicado no Habré, que há 20 anos se dedica exclusivamente aos sistemas UNIX, aproximadamente igualmente com o FreeBSD e GNU / Linux. O autor compara esses dois sistemas com vários parâmetros, de uma olhada no dispositivo do sistema operacional como um todo à análise de pontos específicos, como suporte a sistemas de arquivos individuais e tecnologias de rede, e resume que o FreeBSD é "alta qualidade, confiabilidade, usabilidade e facilidade de uso" e O GNU / Linux é "um zoológico, um depósito de lixo de código vagamente acoplado, pouco que pode ser concluído até o fim, falta de documentação, caos, um bazar".

olhar alternativo ao tópico e explicação da prevalência do GNU / Linux




A Free Software Foundation anunciou planos para criar um novo código de hospedagem com suporte para ferramentas de desenvolvimento colaborativo e atendendo aos critérios éticos para hospedagem de software livre desenvolvidos anteriormente. A nova plataforma será criada além da hospedagem Savannah existente, cujo suporte continuará. O objetivo de criar uma nova plataforma é resolver o problema com a infraestrutura para o desenvolvimento de software de código aberto. Agora, muitos projetos gratuitos dependem de plataformas que não publicam seu código e forçam o uso de software proprietário. A plataforma está planejada para ser comissionada em 2020, construindo-a com base em soluções gratuitas já criadas para trabalhar no código desenvolvido por comunidades independentes que não estão vinculadas aos interesses de empresas individuais. O licitante mais provável é considerado a plataforma Pagure,Desenvolvido por Fedora Linux Developers.

Versão de distribuição do Manjaro Linux 19.0




De acordo com o OpenNET, foi lançada a versão GNU / Linux da distribuição Manjaro Linux 19.0, construída com base no Arch Linux, mas voltada para iniciantes. O Manjaro possui um instalador gráfico mais simples, suporte para detecção automática de hardware e instalação de driver. A distribuição vem na forma de montagens ao vivo com ambientes gráficos KDE, GNOME e Xfce. O Manjaro usa seu próprio kit de ferramentas BoxIt, projetado à imagem do Git, para gerenciar repositórios. Além de seu próprio repositório, há suporte para o uso do repositório AUR (Arch User Repository). Na versão 19.0, o kernel Linux 5.4 é introduzido, versões atualizadas do Xfce 4.14 (com um novo tema para Matcha), GNOME 3.34, KDE Plasma 5.17, KDE Apps 19.12.2. O GNOME oferece uma opção de modo de área de trabalho com diferentes temas. Até a versão 9.3 o gerenciador de pacotes Pamac foi atualizado e, por padrão, é incluído o suporte para pacotes independentes nos formatos snap e flatpak, que podem ser instalados através da nova interface de gerenciamento de aplicativos Bauh.

O primeiro lançamento do Monado, uma plataforma para dispositivos de realidade virtual




O OpenNET anuncia o primeiro lançamento do projeto Monado, destinado a criar uma implementação aberta do padrão OpenXR. O OpenXR é um padrão aberto e sem licença para acesso a plataformas e dispositivos para realidade virtual e realidade aumentada. O código do projeto é escrito em C e distribuído sob a licença gratuita Boost Software 1.0, compatível com a GPL. O Monado fornece um ambiente de tempo de execução que cumpre totalmente os requisitos do OpenXR, que pode ser usado para trabalhar com realidade virtual e aumentada em smartphones, tablets, PCs e outros dispositivos. No âmbito do Monado, vários subsistemas básicos são desenvolvidos:

  1. motor de visão espacial;
  2. mecanismo de rastreamento de caracteres;
  3. servidor composto;
  4. mecanismo de interação;
  5. Ferramentas.

Mirantis Open Source




Linux.com escreve sobre Mirantis. A empresa, que ganhou popularidade com suas soluções baseadas no OpenStack, agora está se movendo muito agressivamente em direção ao Kubernetes. No ano passado, a empresa adquiriu o negócio Docker Enterprise. “ Somos especialistas na área de código aberto e continuamos a oferecer a maior flexibilidade e escolha em nosso setor, mas fazemos isso de maneira a instalar cercas protetoras para que as empresas não tenham algo muito complicado e não gerenciado ou mal configurado ” - concluiu Van Everen.

Open Cybersecurity Alliance




O ZDNet anuncia o advento da OpenDXL Ontology, uma estrutura projetada para a troca de dados e comandos entre programas relacionados à segurança cibernética. A nova estrutura, projetada para colmatar a fragmentação entre ferramentas de segurança cibernética, foi introduzida na comunidade Open Source. O OpenDXL Ontology é desenvolvido pela Open Cybersecurity Alliance (OCA), um consórcio de provedores de segurança cibernética, incluindo IBM, Crowdstrike e McAfee. A OCA disse que o OpenDXL Ontology é "a primeira linguagem de código aberto para conectar ferramentas de segurança cibernética por meio de um sistema de mensagens comum". A ontologia OpenDXL tem como objetivo criar uma linguagem comum entre ferramentas e sistemas de segurança cibernética, eliminando a necessidade de integrações personalizadas entre produtos que possam ser mais eficazes ao interagir entre si,entre sistemas finais, firewalls e mais, mas sofrem com a fragmentação e a arquitetura específica do fornecedor.



Brave archive.org




Segundo o OpenNET, o projeto Archive.org (Internet Archive Wayback Machine), que mantém um arquivo de muitos sites desde 1996, falou sobre uma iniciativa conjunta com os desenvolvedores do navegador Brave, que pode aumentar o acesso à Internet se houver algum problema com a disponibilidade dos sites. Se você tentar abrir uma página inexistente ou inacessível no Brave, o navegador verificará a existência da página em archive.org e, se for encontrada, exibirá uma proposta para abrir a cópia do arquivo morto. Esse recurso foi implementado no lançamento do Brave Browser 1.4.95. Existem complementos com funcionalidade semelhante para o Safari, Chrome e Firefox. O desenvolvimento do navegador Brave é liderado por Brendan Ike, criador da linguagem JavaScript e ex-chefe da Mozilla. O navegador é baseado no mecanismo Chromium,Ele se concentra em garantir a privacidade e a segurança dos usuários e é distribuído sob a licença MPLv2 gratuita.

O programador e o músico geraram algoritmicamente todas as melodias possíveis e as transformaram em domínio público




Vamos terminar com Habr. A verdade também não se aplica diretamente ao software livre e de código aberto, mas os direitos autorais e copyleft são os mesmos, apenas no art. Dois entusiastas, o advogado de software Damien Riel e o músico Noah Rubin, tentaram resolver radicalmente o problema de processos por violação de direitos autorais devido a alegações de música plagiada. Usando o algoritmo de software que eles desenvolveram (disponível no GitHub sob a licença Creative Commons Attribution 4.0) chamado make all the music, eles “geraram todas as melodias possíveis contidas em uma oitava, as salvaram, protegeram esse arquivo com direitos autorais e o tornaram em domínio público, para que no futuro essas melodias não sejam abrangidas por direitos de propriedade intelectual. ” Todas as músicas geradas são publicadas no Internet Archive, 1,2 TB no formato MIDI.Damien Riel também falou sobre essa iniciativa no TED.
