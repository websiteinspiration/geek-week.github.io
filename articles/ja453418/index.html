CQRSとイベントソーシングの主題について、Habréと他のリソースに関する多くの記事が書かれました。この記事は、CQRSとイベントソーシングについてすでに理解している読者を対象としています。記事では、アッカに集中したいと思います。


ドメイン駆動設計


ドメイン駆動設計（DDD）について多くの資料が書かれています。このアプローチには反対者と支持者の両方がいます。イベントソーシングとCQRSに切り替えることを決定した場合、DDDを研究することは不要ではないことを自分で付け加えたいと思います。さらに、DDDの哲学はすべてのAkkaツールで感じられます。

static class DemoActor extends AbstractActor {
  static Props props(Integer magicNumber) {
    return Props.create(DemoActor.class, () -> new DemoActor(magicNumber));
  }

  private final Integer magicNumber;

  public DemoActor(Integer magicNumber) {
    this.magicNumber = magicNumber;
  }

  @Override
  public Receive createReceive() {
    return receiveBuilder()
        .match(
            Integer.class,
            i -> {
              getSender().tell(i + magicNumber, getSelf());
            })
        .build();
  }
}
