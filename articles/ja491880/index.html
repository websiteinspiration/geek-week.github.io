ISO / IEC 27001認証の採用の必然性の5段階。怒り

変化に対する感情的な反応の第2段階は怒りです。これは、認証の最初の準備の困難との闘いの私たちの段階に対応しています-これが私たちの今日の話です。



次の初期データで証明書へのパスを開始しました。

  • 認証条件：できるだけ早く;
  • 予算：小さいほど良いです（ただし、すべてがまともです）。
  • チーム：1.5〜2人（プロジェクトマネージャー+定期的に接続されているIT部門のスタッフと管理者）。
  • 情報セキュリティの分野におけるチームの知識：まあまあ。



あまり見た目は良くないですよね？私たちは、作業の過程でどれほど多くの困難が発生し、どれほどの重大な決定を下さなければならないかさえ想像もしていませんでした。

何も知りません！


主な問題の1つは、当社の誰も情報セキュリティの分野で十分な専門知識を持っていなかったことです。従業員は、情報セキュリティ管理システムの実装に関する専門的な証明書や専門的な経験はありませんでした。これは深刻な懸念を引き起こしました：これに対処しますか？最初にいくつかのトレーニングが必要なのでしょうか？それとも、そのような経験をすでに持っている人を雇う必要がありますか？

実際、あなたはコンサルタントを雇うことができますが、私たち自身がこれについて何も理解していない場合、彼のプロ意識をどのように評価できますか？

先を見れば、このような初期データがあっても、問題はかなり解決可能であることがわかりました。主なことは、チームには論理、常識、および会社が認定を必要とする理由を明確に理解していることです。

多分それをググる？


私たちには専門知識はまったくありませんでしたが、現代のテクノロジーの時代では、ほとんどすべての情報を無料で、またはほとんどお金をかけずに入手できます。そのため、プロジェクトの開始時に、インターネットで認証の準備を成功させるために必要なすべての情報を簡単に見つけ、必要なすべてのドキュメントのサンプルを簡単にダウンロードできると思いました。

実際には、すべてが完全に間違っていることが判明しました。

まず第一に、原則として、「グーグル」するために具体的に何が必要かを私たちは完全に理解していませんでした。

第二に、私たちがパブリックドメインで見つけたものはすべて非常にぼやけていました-詳細、実際の事例はありません。

第三に、インターネットで見つかったドキュメントのサンプルはすべて、当社とはまったく無関係でした。また、英語でも、ステップバイステップのわかりやすい説明や、認証に合格した企業の事例はほとんどありませんでした。したがって、証明書へのパスを自分で模索する必要がありました。

もつれを解明し始めるのはいつですか？


インターネットで情報を集中的に検索した結果、最初に次のことを決定する必要があることに気付きました。

  • 認証局;
  • 認定コンサルタント（私たちは本当に専門知識を持っていないため-すでにそれを持っている人を見つける必要があるため）;
  • システムの開発と保守のための技術ツール（以降の記事で、この重要なポイントをさらに詳しく説明します）。

最初の2つは、認証中の主要な取引相手です。その選択は非常に慎重に行う必要があります（これを行いました）。したがって、最初に焦点を当てることに決めたのは、これらの主要な取引相手を選択するための2つの入札を行うことです。

証明機関の選択方法は？


もちろん、認証機関の選択は、認証の準備を促した理由によって異なります。記事のこの場所に来た場合は、おそらくショーだけでなく証明書も必要です。それ以外の場合は、1時間1万ルーブルで証明書を作成することを提供している企業のサービスを利用したでしょう。したがって、国際的な慣行が広く、関心のある国で認定されている認証機関に焦点を当てる必要があります。

ISO 27001規格に従ってロシアで認定する準備が整っている企業はそれほど多くありません。入札には約10名の参加者が選ばれました。選択の主な基準は次のとおりです。

  • 国際認定の利用可能性、
  • 顧客ポートフォリオとその推奨事項
  • 価格。

最後のポイントで約10倍のスプレッドを得たことは驚くべきことです。ただし、一部の入札者は、外国人監査人しか提供できないと述べた。これは自動的に英語で認定監査に合格することを意味します。これは、すべての主要な従業員が高いレベルでそれを知っているため、原則として私たちにとって大きな問題ではありませんでしたが、誰かにとってこれは間違いなく問題になる可能性があります。

後に、私たちの国では、この基準に従って認証監査を実施できる専門家はほとんどいないことがわかりました。それらのほとんどすべてが複数の認証機関で機能しており、相互に精通しています。

認定準備コンサルタントの選び方は？


認定に向けてサービスを提供している企業はかなりたくさんあります。ただし、それらすべてが実際に役立つわけではありません。実際には、ビジネスプロセスを詳しく調べることなく、会社の名前を挿入する必要がある場所にポリシーテンプレートを送信するだけのものもあります。当然のことながら、このアプローチは認定に少し役立ちます。

概念的には、この問題には2つの解決策があります。

  • すべてのターンキードキュメントのコンサルタントによる準備。このアプローチにより、認定の準備で従業員が過負荷にならないことは間違いありません。ただし、プロセスと手順が適切に文書化されないリスクがあります。
  • コンサルタントがあなたの従業員が作成した文書をチェックします。ここでは、おそらくドキュメントの品質が向上します。これは、プロセスに精通している従業員がドキュメントを作成するためです。

認証の準備として、2番目のシナリオで行動しました。私たちの経験に基づいて、認定のためのコンサルタントを選択する上でいくつかのヒントを与えることができます：



  • 認証機関のコンサルタント会社からの提案を求めてください。その間、入札を行っています-それが私たちが見つけた方法です。
  • 作業の範囲と範囲、および各当事者の責任を事前に交渉して修正します。
  • 認定の準備の全期間を通じて、コンサルタントと定期的に連絡を取り合ってください。これにより、時間を節約でき、大量のドキュメントをやり直す必要がなくなります。

大丈夫ですが、今はすべて大丈夫ですか？


認証の準備に必要な資料を集める過程で、意外なことがわかりました。たとえば、ISO 27001が多くの関連する標準に関連付けられているという事実（少なくとも表面的に読む必要があります）。

たとえば、次のような標準があります。

  • ISO 19011-管理システム監査のガイドライン
  • ISO 22301-事業継続管理システム
  • ISO 31000-リスク管理。原則とガイドライン
  • ISO 27003-安全の方法と手段。情報セキュリティ管理システム

上記のリストは基本的なものですが、包括的ではありません。各企業は独自のニーズに基づいてそれを形成します。私たちは「車輪を再発明」しないことを選択しました。たとえば、リスク管理と管理システムの監査の問題では、それぞれISO 31000とISO 19011に依存していました。サポート標準であるISO 27003は、27001の実装に関する付随情報を提供してくれました。しかし、ほとんどの場合、ISO 22301で作業しました。これは、ビジネス継続性計画（BCP）を担当するポリシーの一部を説明するために必要です。
