2019年以降に開発者のためにOracleを開発するもの

少し前に、私はもう一度仕事を変えました。新しい雇用主はとても親切だったので、好きなIDEを選択してライセンスの支払いをすることができました（ただし、合理的な範囲内）。また、（一度だけ）動作しているコンピューターの管理者権限を持っているので、何でもインストールして好きなだけ試すことができます。私が試しました：Oracle SQL Developer、TOAD、Alt SQL Developer、および（SUDDENLY！）IntelliJ IDEA（PL / SQL Developerは試していませんでしたが、以前はかなり機能していました）。定期的にPL / SQLコードを書く場合、または少なくとも時々Oracleデータベースにリクエストを出す場合、私が結論を出したことに興味があると思います。


すぐに警告します。私は各IDEを徹底的に研究したとは言えません。特にPL / SQL Developerは、5年前に最後に見ました。したがって、「これはIDEにありません」と言ったときに、実際にはそこにあるのに、見つからなかったことが常にわかります。PL / SQL Developerについて話すとき、バージョンは8（5年前に書いた）のようです。そこでは多くのことが変わった可能性があり、必ずしも良くなるとは限りません。


競合他社


それで私たちは何を持っていますか？まず、これは、Oracle開発の大きな3つであるTOAD、PL / SQL Developer、Oracle SQL Developerと呼ばれるものです。これらの製品は長い間市場に出回っていますが、誰もが知っているように、各製品には独自のファンがいます。TOADはDBAのようなものであり、PL / SQL Developerは開発者のようなものだという意見に何度か出会いました。Oracle SQL Developerについては、私はこのようなことを聞​​いたことがありません、そして彼のファンの軍隊はどうやら、主にマネージャーで構成されています、そしてこれはそれが無料だからです。


さらに、Alt SQL Developer（製品、長い間、そしておそらく1人で開発されたもの）とIntelliJ IDEA（別名 ゴッシュ、彼はゴガだ DataGrip）。
