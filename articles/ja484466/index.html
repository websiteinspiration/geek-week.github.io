誰もが知っておくべき共通のJavaScript約束事項

こんにちは！

JavaScriptとプロミスを勉強したときに、これらのエラーについて知りたいのですが。

一部の開発者が私に電話をかけ、彼のコードが機能していないか、実行速度が遅いと文句を言うときはいつでも、まずこれらのエラーに注意を払います。 4年前にプログラミングを始めたとき、私はそれらについて知らず、それらを無視することに慣れていました。しかし、数分で約100万のリクエストを処理するプロジェクトに割り当てられた後、コードを最適化するしかありませんでした（それ以上の垂直スケーリングが不可能になったため）。

したがって、この記事では、JSでプロミスを処理する際に最もよくある間違いについてお話ししたいと思いますが、多くの人はこの点に注意を払っていません。

間違い＃1. Promise内でtry / catchブロックを使用する


promise内でtry / catchブロックを使用することは実際的ではありません。コードが（promise内で）エラーをスローすると、promise自体のエラーハンドラーによってインターセプトされるためです。

これはこれについてです：
new Promise((resolve, reject) => {
  try {
    const data = someFunction()
    //  
    resolve()
  } catch(e) {
    reject(e)
  }
})
  .then(data => console.log(data))
  .catch(error => console.log(error))

代わりに、コードにpromiseの外部でエラーを処理させます。
new Promise((resolve, reject) => {
  const data = someFunction()
  //  
  resolve(data)
})
  .then(data => console.log(data))
  .catch(error => console.log(error))

これは、以下で説明する場合を除いて、常に機能します。

間違いその2。Promise内で非同期関数を使用する


非同期関数を使用すると、約束の範囲内でいくつかの不快な副作用が発生します。

非同期タスクを実行することに決め、キーワード「async」をpromiseに追加すると、コードがエラーをスローします。ただし、.catch（）またはawaitのいずれでもこのエラーを処理できなくなりました。
//      
new Promise(async() => {
  throw new Error('message')
}).catch(e => console.log(e.message))

//       
(async() => {
  try {
    await new Promise(async() => {
      throw new Error('message')
    })
  } catch(e) {
    console.log(e.message)
  }
})();

promise内で非同期関数に出会うたびに、それらを分離しようとします。そして、私は10ケースのうち9ケースでそれを得ます。ただし、これは常に可能であるとは限りません。この場合、promise内でtry / catchブロックを使用するしかありません（そうです、これは最初のエラーと矛盾しますが、これが唯一の方法です）。
new Promise(async(resolve, reject) => {
  try {
    throw new Error('message')
  } catch(error) {
    reject(error)
  }
}).catch(e => console.log(e.message))

//   async/await
(async() => {
  try {
    await new Promise(async(resolve, reject) => {
      try {
        throw new Error('message')
      } catch(error) {
        reject(error)
      }
    })
  } catch(e) {
    console.log(e.message)
  }
})();

間違い番号3. .catch（）を忘れる


これは、テストを開始するまで、疑わしいことのないエラーの1つです。または、テストを信じない無神論者であれば、コードは確かに本番環境でクラッシュします。プロダクションはマーフィーの法則に厳密に従っているため、「問題が生じる可能性がある場合はすべて問題が発生します」（これは、「問題が発生する可能性がある場合に発生します」と翻訳できます。ロシア語での類推は「卑劣の法則」です。 -約

コードをよりエレガントにするために、.then（）を使用する代わりに、promiseをtry / catchでラップすることができます。Catch（）。

エラー番号4。Promise.all（）を使用しないでください


Promise.all（）はあなたの友達です。

あなたがプロの開発者であれば、おそらく私が言いたいことを理解しているでしょう。互いに依存しない複数のpromiseがある場合、それらを同時に実行できます。デフォルトでは、promiseは並行して実行されますが、（awaitを使用して）順次実行する必要がある場合は、かなりの時間がかかります。Promise.all（）はレイテンシを大幅に削減できます：
const {promisify} = require('util')
const sleep = promisify(setTimeout)

async function f1() {
  await sleep(1000)
}
async function f2() {
  await sleep(2000)
}
async function f3() {
  await sleep(3000)
}

//  
(async() => {
  console.time('sequential')
  await f1()
  await f2()
  await f3()
  console.timeEnd('sequential') //  6 
})();

今Promise.all（）で：
(async() => {
  console.time('concurrent')
  await Promise.all([f1(), f2(), f3()])
  console.timeEnd('concurrent') //  3 
})();

間違い＃5. Promise.race（）の誤った使用


Promise.race（）は常にコードを高速化するわけではありません。

奇妙に見えるかもしれませんが、実際には奇妙です。Promise.race（）が役に立たないと言っているわけではありませんが、なぜそれを使用しているのかを明確に理解する必要があります。

たとえば、Promise.race（）を使用して、Promiseを解決した後でコードを実行できます。ただし、これは、promiseに続くコードの実行が、promiseの1つが解決された直後に開始されることを意味するものではありません。Promise.race（）はすべてのpromiseの解決を待機し、その後にのみストリームを解放します。
const {promisify} = require('util')
const sleep = promisify(setTimeout)

async function f1() {
  await sleep(1000)
}
async function f2() {
  await sleep(2000)
}
async function f3() {
  await sleep(3000)
}

(async() => {
  console.time('race')
  await Promise.race([f1(), f2(), f3()])
})();

process.on('exit', () => {
 console.timeEnd('race') //  3 ,    !
})

間違い番号6.約束の乱用


プロミスはコードを遅くしますので、悪用しないでください。

多くの場合、コードの見栄えをよくするために、長い.then（）チェーンを使用している開発者を見る必要があります。このチェーンが長くなりすぎて、瞬きする時間がありません。このような状況の悪影響を視覚的に確認するには、（記事よりも詳細にプロセスを説明するために、元のテキストから少し離れて説明します-約Transl。）：

1）次の内容のscript.jsファイルを作成します（追加の約束）：
new Promise((resolve) => {
  //  ,   
  const user = {
    name: 'John Doe',
    age: 50,
  }
  resolve(user)
}).then(userObj => {
    const {age} = userObj
    return age
}).then(age => {
  if(age > 25) {
    return true
  }
throw new Error('Age is less than 25')
}).then(() => {
  console.log('Age is greater than 25')
}).catch(e => {
  console.log(e.message)
})

2）コマンドラインを開きます（Windowsユーザーの場合：目的のファイルを含むフォルダーでコマンドラインを開くには、Shiftキーを押しながら右クリックし、[コマンドウィンドウを開く]を選択します）。次のコマンドを使用してscript.jsを実行します（ノードがインストールされている必要があります。 js）：
node --trace-events-enabled script.js

3）Node.jsはスクリプトフォルダーにログファイル（私の場合はnode_trace.1.txt）を作成します。

4）Chromeを開きます（動作するのはChromeだけなので）、アドレスバーに「chrome：//トレース」と入力します。

6）[約束]タブを開きます。
