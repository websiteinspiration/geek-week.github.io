Greenplum 6：新機能のレビュー

過去16年間、オープンで大規模な並列並列データベース管理システムであるGreenplumは、さまざまな企業がデータ分析に基づいた意思決定を支援してきました。

この間、Greenplumは、小売、フィンテック、テレコム、業界、eコマースなど、さまざまなビジネス分野に進出しました。水平方向に数百のノードへのスケーリング、フォールトトレランス、オープンソースコード、PostgreSQLとの完全な互換性、トランザクション性、ANSI SQL-分析用DBMSのプロパティのより良い組み合わせを想像することは困難です。 Morgan Stanley（200ノード、25 Pbのデータ）やTinkoff（> 70ノード）などの世界の巨大企業の巨大なクラスターから始まり、居心地の良い新興企業で小規模な2ノードのインストールに至るまで、ますます多くの企業がGreenplumを選択しています。ロシアでこの傾向を観察することは特にうれしいです-過去2年間で、Greenplumを使用する国内の大企業の数は3倍になりました。

2019年の秋に、別の主要なDBMSリリースがリリースされました。この記事では、GP 6の主な新機能について簡単に説明します。

Greenplumバージョン5の以前のメジャーリリースは2017年9月に公開されました。詳細はこの記事で確認できます。それでもGreenplumが何であるかわからない場合は、この記事から簡単な紹介を取得できます。古いですが、DBMSアーキテクチャは正しく反映されています。

現在のリリースは、まさしく集合的な発案者と呼ぶことができます。世界中のいくつかの企業が開発に参加しました-その中には、Pivo​​tal、Arenadata（この記事の著者が作業する場所）、Alibabaがあります。

それでは、Greenplum 6の新機能は何ですか？

複製されたテーブル


Greenplumでは、クラスター全体で2種類のテーブルの分布があったことを思い出してください。

  • ランダム均一分布
  • 1つまたは複数のフィールドでの分布

ほとんどの場合、2つのテーブル（JOIN）の結合は、クエリの実行中にクラスターセグメント間でデータが再分散されて実行され、両方のテーブルが最初にJOIN結合キーによって分散された場合にのみ、セグメント間でデータを転送せずにセグメントでローカルに発生しました。

GP 6は、アーキテクトに新しいストレージスキーム最適化ツール-複製されたテーブルを提供します。このようなテーブルは、クラスターのすべてのセグメントで完全に複製されます。右側のそのようなテーブルへの接続は、データの再配布なしでローカルに実行されます。基本的に、この機能は大量のディレクトリを格納することを目的としています。

Zstandard圧縮アルゴリズム（ZSTD）


Zlib（デフォルトでGreenplumで使用されています）と比較して圧縮率が高く、圧縮と解凍に必要な時間が少ないため、Facebook開発者が2016年に発表したロスレス圧縮アルゴリズムは、すぐにArenadataチームの魂に落ち込みました。


出典：cnx-software.com

圧縮効率は、最新の分析DBMSの最も重要なパラメーターの1つです。実際、比較的安価なCPUをロードすることで、クラスターの比較的高価なディスクサブシステムの負荷を軽減できます。大量のデータを順次読み書きすることにより、システムのTCOを大幅に削減できます。

2017年に、私たちのチームはGreenplumの列テーブルのZSTDサポートを追加しましたが、リリースポリシーによると、この改訂はGreenplumの公式のマイナーリリースには含まれていません。今日まで、これはArenadataの商用ユーザーのみが利用でき、6.0のリリースで、誰でもそれを使用できます。

クラスター拡張の最適化（拡張）


以前のバージョンのGPでは、水平クラスター拡張（新しいノードの追加）にいくつかの制限がありました。

  • データの再配布がダウンタイムなしでバックグラウンドで発生した場合でも、新しいノードを追加するときにシステムの再起動が必要でした
  • ハッシュおよびデータ分散アルゴリズムでは、拡張中にすべてのテーブルの完全な再分散が必要でした-バックグラウンドのデータ分散プロセスには、特に大規模なクラスターの場合、数時間または数日かかる場合があります
  • テーブルのバックグラウンド配布中に、すべての結合が配布されました

Greenplum 6は、次の理由により、まったく新しいクラスター拡張アルゴリズムを導入しました。

  • システムを再起動せずに拡張が行われるようになりました-ダウンタイムは必要ありません
  • 一貫性のあるハッシュアルゴリズムにより、ノードの追加時にブロックの一部のみを再配布できます。つまり、テーブルのバックグラウンド再配布は何倍も速く機能します
  • システムディレクトリを変更するためのロジックが変更されました。拡張プロセス中でも、すべての接続（JOIN）がローカルと分散の両方で通常どおり機能します。

Greenplumの拡張は数時間ではなく数分になりました。これにより、クラスターはビジネスユニットの増え続ける食欲に従うことができます。

Arenadata DBディストリビューションの新機能

お知らせします。Arenadata、私たちはオープンソーステクノロジー（Greenplum、Kafka、Hadoop、Clickhouse）に基づくデータストレージプラットフォームを開発、実装、およびサポートしています。私たちのクライアントは、小売、通信、フィンテックなどの分野で最大のロシア企業です。一方で、私たちはオープンソースプロジェクト自体の貢献者です（カーネルへの変更をコミットします）。一方、私たちは商業顧客のみが利用できる追加機能を開発しています。さらに、主な機能について説明します。

Tkhemali Connector別名Connector Greenplum->クリックハウス


プロジェクトでは、しばしばGreenplum + Clickhouseの束を使用します-一方では、これにより、多くの接続、開発されたANSI SQL構文、トランザクション性、およびGreenplumが持つその他のチップを必要とするデータウェアハウス（ソースからデータマートまで）を構築する最高の古典的なモデルを使用できます。もう1つは、組み込みのワイドショップウィンドウへのアクセスを、かなりの数のユーザーに最大速度で提供することです。これには、Clickhouseには競合相手がいません。

このようなバンドルを効果的に使用するために、トランザクション的に（つまり、ロールバックトランザクションの場合でも一貫性のある）GPからKHにデータを転送できる特別なパラレルコネクタを開発しました。一般に、このコネクタのアーキテクチャは、別の純粋な技術記事に値します。実際、内部では、挿入ごとのスレッド数とデータフローを動的に選択するシステムで並列非同期キューを実装する必要がありました。

その結果、素晴らしい相互作用速度が得られます。通常のSATAディスクでのテストでは、1組のGreenplumサーバー-Clickhouseの挿入ごとに最大1 Gb / sを得ています。お客様の平均的なGPクラスターが20台以上のサーバーで構成されている場合、対話速度は十分以上です。

カフカコネクター


Kafkaメッセージブローカーとの統合でも同じことが行われました-ほぼリアルタイムモード（数秒または数十秒）でデータをKafkaからGreenplumにオーバーロードするタスクに遭遇することがよくあります。ただし、Kafkaのコネクタアーキテクチャは異なります。コネクタは、自動検出機能を備えた個別の同期プロセス（Dockerで起動）のクラスターであり、一方ではカフカ消費者であり、他方では、データをGreenplumセグメントに直接挿入します。コネクタはKafkaレジストリと連携して動作し、ハードウェア障害が発生した場合でも、転送されたデータの完全な一貫性を保証します。

管理および監視システム


本番環境でのシステムの運用では、クラスターのデプロイ、更新、モニタリングに高い要求が課されます。DBMSで発生するすべてのことは、運用およびDBAの専門家に透過的であることが重要です。

Arenadata Cluster Manager（ADCM）管理および監視システムは、運用専門家に必要なすべてのツールを提供します。実際、Greenplumクラスターのデプロイと更新は、グラフィカルインターフェースのボタンをクリックするだけで行われ（すべてのOS、サービス、ディスクのマウント、ネットワーク設定が自動的に行われます）、さらに、完全に構成されたモニタリングスタックが得られ、企業システムと統合する準備が整います。ちなみに、Arenadata Cluster ManagerはGreenplumだけでなく、Hadoop、Kafka、Clickhouseも管理できます（これらのサービスのアセンブリが必要です。ADCM自体のような無料バージョンは、ポップアップを入力するだけで、当社のWebサイトから完全に無料でダウンロードできます）。

結論


Greenplum 5.Xを使用している場合は、今後2〜3か月以内にクラスターを現在のバージョン6.Xにアップグレードすることを検討することをお勧めします。

Greenplumをまだ使用していない場合は、ぜひご参加ください。私たちArenadataは、いつでもこれをお手伝いする準備ができています。
