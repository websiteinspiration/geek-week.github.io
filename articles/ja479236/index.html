昨日のモバイル開発のPython記事で、KivyMDライブラリ（Kivyクロスプラットフォームフレームワークで使用するマテリアルデザインスタイルのウィジェットのコレクション）について説明しましたが、コメントでAndroidプラットフォーム向けのパッケージのビルドプロセスについて話してもらいました。多くの人にとって、このプロセスは、残念ながら、一連の魔法のシ​​ャーマニズムの一部であり、初心者向けではありません。さて、すべてが本当に複雑かどうか、そして私が本当に魔法使いで魔法使いであるかどうかを理解しましょう...



もちろん彼はできました！つまり、PythonとKivyでコードを記述しました。Androidデバイスで実行するには何が必要ですか？KivyMDリポジトリに移動し、あなたがいることがわかりますこれらの命令は限り、あなたはAPKパッケージを収集できるようになります手順綴られています。

  1. XUbuntu 18.04をダウンロード

コンピュータにVirtual Boxをインストールします。
ロードされたXUbuntuイメージに基づいて新しい仮想マシンを作成し
、XUbuntu 仮想マシンを起動してターミナルを開き、次のコマンドを実行します。

wget https://github.com/HeaTTheatR/KivyMD-data/raw/master/install-kivy-buildozer-dependencies.sh

chmod +x install-kivy-buildozer-dependencies.sh

./install-kivy-buildozer-dependencies.sh

すべて！これで、KivyアプリのAPKパッケージを構築するための仮想マシンができました。次は何ですか？実際にテストアプリケーションを作成してみましょう。空のmain.pyファイルを使用して、仮想マシンのホームディレクトリにTestKivyMDディレクトリを作成します。


次に、main.pyファイルを開き、KivyMDライブラリを使用するテストアプリケーションのコードを記述します。

from kivy.lang import Builder

from kivymd.app import MDApp


KV = """
Screen:

    MDToolbar:
        title: "My firt app"
        elevation: 10
        md_bg_color: app.theme_cls.primary_color
        left_action_items: [["menu", lambda x: x]]
        pos_hint: {"top": 1}

    MDRaisedButton:
        text: "Hello World"
        pos_hint: {"center_x": .5, "center_y": .5}
"""


class HelloWorld(MDApp):
    def build(self):
        return Builder.load_string(KV)


HelloWorld().run()

保存し、main.pyファイルのあるディレクトリでターミナルを開いて、KivyMDライブラリをインストールします。


sudo pip3 install kivymd

インストール後、コードをテストできます。


python3 main.py

スクリプトの結果は、ツールバーと「Hello World」ボタンが1つある画面になります。


次に、main.pyファイルと同じディレクトリにあるbuildozer.spec仕様ファイルを作成する必要があります。


ターミナルを閉じなかった場合（ターミナルが閉じられていた場合） 、それをディレクトリTestKivyMDで開き、次のコマンドを入力します。


buildozer init

このコマンドは、デフォルトの仕様ファイルを作成します。それを開いて編集します：


[app]

# (str) Title of your application
title = KivyMDTest

# (str) Package name
package.name = kivymd_test

# (str) Package domain (needed for android/ios packaging)
package.domain = com.heattheatr

# (str) Source code where the main.py live
source.dir = .

# (list) Source files to include (let empty to include all the files)
source.include_exts = py,png,jpg,jpeg,ttf

# (list) Application version
version = 0.0.1

# (list) Application requirements
# comma separated e.g. requirements = sqlite3,kivy
requirements = python3,kivy==1.11.1,kivymd

# (str) Supported orientation (one of landscape, sensorLandscape, portrait or all)
orientation = portrait

# (bool) Indicate if the application should be fullscreen or not
fullscreen = 1

# (list) Permissions
android.permissions = INTERNET,WRITE_EXTERNAL_STORAGE

# (int) Target Android API, should be as high as possible.
android.api = 28

# (int) Minimum API your APK will support.
android.minapi = 21

# (str) Android NDK version to use
android.ndk = 17c

# (bool) If True, then skip trying to update the Android sdk
# This can be useful to avoid excess Internet downloads or save time
# when an update is due and you just want to test/build your package
android.skip_update = False

# (bool) If True, then automatically accept SDK license
# agreements. This is intended for automation only. If set to False,
# the default, you will be shown the license when first running
# buildozer.
android.accept_sdk_license = True

# (str) The Android arch to build for, choices: armeabi-v7a, arm64-v8a, x86, x86_64
android.arch = armeabi-v7a

[buildozer]

# (int) Log level (0 = error only, 1 = info, 2 = debug (with command output))
log_level = 2

# (int) Display warning if buildozer is run as root (0 = False, 1 = True)
warn_on_root = 0

# (str) Path to build artifact storage, absolute or relative to spec file
build_dir = ./.buildozer

# (str) Path to build output (i.e. .apk, .ipa) storage
bin_dir = ./bin

ここではすべてが明確であるため、追加のコメントは不要です。デフォルトの仕様を注意深く読み、アイコンへのパスを指定したり、アプリケーションのロード時にフラッシュしたりすることができます。テストパッケージをビルドするために必要なものだけを残しました。そして、実際には、ターミナルのコマンドでビルドプロセスを開始します。


buildozer android debug

ライブラリをダウンロードしてコンパイルするプロセスには初めて長い時間がかかるため、安全にキッチンに行ってコーヒーを作ることができます。以降のすべての組み立てには10〜20秒かかります。
