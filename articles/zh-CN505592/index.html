使用Android为BK0010编程



故事发生在BK00010电报频道，我是参与者。

问题出现了：如何编写程序？该仿真器不支持刻录光盘，因此在仿真器中使用汇编程序不太可能成功。最重要的是，智能手机上的图片很小。

提出了一个解决方案（斯玛斯洛夫斯基），我将对其进行描述。您需要下载Termux，此处提供了适用于Android 5.0的版本，即termux-app-git-debug.apk文件。该程序的版本是0.83，它支持自定义键栏。

在安装并启动Termux之后，我重新配置了密钥，过程如下：

mkdir $HOME/.termux

echo "extra-keys = [['ESC','/','-','HOME','UP','END','PGUP'],['TAB','CTRL','ALT','LEFT','DOWN','RIGHT','PGDN']]" > $HOME/.termux/termux.properties

termux-reload-settings

变得更加方便



现在您需要安装必要的 Finish，MACRO-11将用作创建.BIN的汇编程序：
pkg install build-essential
pkg install git




mkdir src
cd src
git clone https://gitlab.com/Rhialto/macro11.git

然后我安装了外壳：

pkg install mc

和文本编辑器

pkg install nano

在src \ macro11文件夹中，您需要修复Makefile，更改参数

OBJFORMAT = -DDEFAULT_OBJECTFORMAT_RT11=1

输入make并等待结果，

复制接收到的文件

cp macro11 $PREFIX/bin

cp dumpobj $PREFIX/bin

要进行编译，您需要一个脚本来构建

cd ~/src 
git clone https://github.com/smaslovski/pdp11dasm.git
cd pdp11dasm/utils
chmod 755 obj2bkb
cp obj2bkb $PREFIX/bin

编译脚本，我叫compile：

#!/data/data/com.termux/files/usr/bin/sh
n=$1
macro11 $n.mac -o $n.obj -l $n.lst
obj2bkb --binary --base=01000 -o $n.bin $n.obj
du -b $n.bin
cp $n.bin /sdcard

（您可以创建nano compile）

需要复制脚本

chmod 755 compile
cp compile $PREFIX/bin
