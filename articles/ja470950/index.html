bear_hug：Python3.6のASCIIアートのゲーム+



ASCIIアートのゲームでは、イベントキュー、ウィジェットのコレクション、ECSサポート、およびその他の便利な小さなものを使用して、bear_hugライブラリを作成しました。この記事では、それを使用して最小限の動作するゲームを作成する方法について説明します。

もちろん、ライブラリは比較的大規模なプロジェクト用に作成されました。ただし、ゲームの設計とアーキテクチャに気を取られないようにするために、この記事では簡単なものを作成します。ライブラリの基本的な機能を示すものがある1泊のプロジェクト。つまり、Dandyと同じタンクの簡略化されたクローンです（これらはBattle Cityでもあります）。プレイヤーの戦車、敵の戦車、破壊可能な壁、音、得点があります。ただし、メインメニュー、レベル、選択したボーナスは対象外です。それらを追加することが不可能になるのではなく、このプロジェクトがHalloworldに過ぎないからです。

端末はゲームの実際のウィンドウです。その上にウィジェットを配置でき、必要に応じて入力イベントをスローします。端末を作成するときのキーとして、すべての端末オプションbearlibterminalを使用できます。この場合、対象となるフォント、ウィンドウサイズ（文字数）、ウィンドウタイトル、および入力方法を設定します。

イベントキューについては、非常にシンプルなインターフェースを備えています。dispatcher.add_event（イベント）はイベントをキューに追加し、dispatcher.register_listener（リスナー、event_types）はそれをサブスクライブできます。署名者（ウィジェットやコンポーネントなど）には、イベントを単一の引数として受け取り、何も返さないか、別のイベントまたはイベントのセットを返すon_eventコールバックが必要です。イベント自体はタイプと値で構成されます。ここでのタイプは、strやintの意味ではなく、「変化」の意味、たとえば、「key_down」または「tick」です。キューは、既知のタイプ（組み込みまたはユーザーが作成したタイプ）のイベントのみを受け入れ、このタイプにサブスクライブしているすべてのイベントをon_eventに送信します。彼女は決して値をチェックしませんが、ライブラリ内にはこれは、イベントのタイプごとに有効な値です。

まず、いくつかのリスナーをキューに入れます。これは、イベントにサブスクライブできるオブジェクトの基本クラスですが、ウィジェットやコンポーネントではありません。原則として、署名者がon_eventメソッドを持っている限り、これを使用する必要はありません。

クラスHealthComponent
class HealthComponent(Component):
    def __init__(self, *args, hitpoints=3, **kwargs):
        super().__init__(*args, name='health', **kwargs)
        self.dispatcher.register_listener(self, 'ac_damage')
        self._hitpoints = hitpoints

    def on_event(self, event):
        if event.event_type == 'ac_damage' and event.event_value[0] == self.owner.id:
            self.hitpoints -= event.event_value[1]

    @property
    def hitpoints(self):
        return self._hitpoints

    @hitpoints.setter
    def hitpoints(self, value):
        if not isinstance(value, int):
            raise BearECSException(
                f'Attempting to set hitpoints of {self.owner.id} to non-integer {value}')
        self._hitpoints = value
        if self._hitpoints < 0:
            self._hitpoints = 0
        self.process_hitpoint_update()

    def process_hitpoint_update(self):
        """
        Should be overridden by child classes.
        """
        raise NotImplementedError('HP update processing should be overridden')

    def __repr__(self):
        #          
        return dumps({'class': self.__class__.__name__,
                      'hitpoints': self.hitpoints})


コンポーネントを作成するとき、 'name'キーはsuper（）.__ init__に渡されます。コンポーネントがエンティティに追加されると、このキーの名前で、エンティティの__dict__に追加され、entity_object.healthを介してアクセスできます。インターフェースの利便性に加えて、このアプローチは、いくつかの同種コンポーネントのエンティティの発行を禁止するため、優れています。また、コンポーネント内にハードコードされているため、たとえばWidgetComponentをヘルスコンポーネントのスロットに誤って挿入することはできません。作成直後に、コンポーネントは対象のイベントクラス（この場合はac_damage）をサブスクライブします。そのようなイベントを受け取ったon_eventメソッドは、それがその所有者についてかどうかを1時間チェックします。もしそうなら、彼はヒットポイントから望ましい値を差し引き、コールバックをプルしてヘルスを変更します。基本クラスは抽象です。 __repr__メソッドもあり、これを使用してJSONでのシリアル化（保存など）。追加する必要はありませんが、すべての組み込みコンポーネントとほとんどの組み込みウィジェットに追加されています。

VisualDamageHealthComponentの基になるヘルスコンポーネントから継承すると、ヘルスの変更時にコールバックがオーバーライドされます。

クラスVisualDamageHealthComponent
class VisualDamageHealthComponent(HealthComponent):
    """
          .
       HP=0
    """
    def __init__(self, *args, widgets_dict={}, **kwargs):
        super().__init__(*args, **kwargs)
        self.widgets_dict = OrderedDict()
        for x in sorted(widgets_dict.keys()):
            self.widgets_dict[int(x)] = widgets_dict[x]

    def process_hitpoint_update(self):
        if self.hitpoints == 0 and hasattr(self.owner, 'destructor'):
            self.owner.destructor.destroy()
        for x in self.widgets_dict:
            if self.hitpoints >= x:
                self.owner.widget.switch_to_image(self.widgets_dict[x])


ヘルスが0を超えている間、彼はウィジェットを担当するコンポーネントに希望の状態で壁を描くように依頼します。ここで、上記の呼び出しは、エンティティオブジェクトの属性を通じて使用されます。ヒットポイントが終了すると、エンティティとすべてのコンポーネントの正しい破棄を担当するコンポーネントが同じ方法で呼び出されます。

他のエンティティについては、すべてが類似しており、コンポーネントのセットのみが異なります。タンクには、コントローラー（プレイヤーの入力、対戦相手のAI）と回転ウィジェット（シェル）が追加されます-衝突コンポーネントは、それらがヒットしたものにダメージを与えます。それはかさばり、かなり取るに足らないので、私はそれらのそれぞれを分析しません。発射体コライダーだけを見てください。これにはcollided_intoメソッドがあり、ホストエンティティが何かにクラッシュしたときに呼び出されます。

def collided_into(self, entity):
        if not entity:
            self.owner.destructor.destroy()
        elif hasattr(EntityTracker().entities[entity], 'collision'):
            self.dispatcher.add_event(BearEvent(event_type='ac_damage',
                                                event_value=(
                                                entity, self.damage)))
            self.owner.destructor.destroy()


犠牲者（たとえば、背景要素では間違っている可能性があります）を取得できることを確認するために、発射物はEntityTracker（）を使用します。これは、作成および破棄されたすべてのエンティティを追跡するシングルトンです。これにより、名前でエンティティオブジェクトを取得し、そのコンポーネントで何かを行うことができます。この場合、entity.collision（被害者衝突ハンドラー）が存在することが確認されます。

class ScoreLabel(Label):
    """
     
    """

        super().__init__(text='Score:\n0')
        self.score = 0

    def on_event(self, event):
        if event.event_type == 'ecs_destroy' and 'enemy' in event.event_value and 'bullet' not in event.event_value:
            #

            self.text = f'Score:\n{self.score}'
