Cara menonaktifkan peringatan tentang bahaya mendengarkan audio yang lama (Android)

Mungkin, banyak yang mendengarkan musik (dan tidak hanya) dari perangkat Android telah menemukan peringatan ini:


Dalam artikel ini, kita akan melihat mengapa dan kapan peringatan ini terjadi, dan bagaimana memastikan bahwa peringatan itu tidak lagi terjadi.

Ini muncul hanya ketika mendengarkan audio melalui perangkat eksternal (headphone / speaker). Bagi yang belum pernah bertemu ini, sedikit penjelasan: bayangkan Anda sedang mendengarkan musik dengan headphone, cukup keras. Tiba-tiba, suara menjadi lebih tenang. Anda mencoba menaikkan volume menggunakan tombol-tombol pada case, tetapi tidak berhasil. Mengambil perangkat dari saku Anda dan membukanya, Anda akan melihat peringatan seperti itu. Hanya setelah persetujuan dengannya akan mungkin untuk menaikkan volume kembali.

Ya, peringatan itu masuk akal, tetapi muncul pada saat yang tidak terduga, kadang-kadang paling tidak tepat: ketika Anda berada di angkutan umum / mengemudi / di musim dingin di jalan / ketika Anda memiliki tangan yang kotor, dll. Sangat tidak nyaman untuk melepas perangkat, melepaskan kunci, menerima peringatan, meletakkan kembali perangkat dalam kasus ini. Dan jika speaker terhubung, dan bukan headphone, pesannya tidak sepenuhnya tepat.

Mengapa itu muncul?


Peringatan ini bukan inisiatif dari penulis platform. Masalahnya adalah bahwa ada standar WHO-ETU untuk "mendengarkan dengan aman" (mendengarkan dengan aman). Di Eropa dan beberapa negara lain, implementasinya wajib. Standar ini menjelaskan berapa lama Anda bisa mendengarkan audio tergantung pada volume dengan risiko gangguan pendengaran minimal. Misalnya, untuk orang dewasa, "dosis" suara mingguan yang aman adalah 1,6 Pa 2 jam, yang setara dengan 20 jam mendengarkan pada volume 83 dB.


Penerapan


Bergantung pada mcc (kode negara seluler), mode mendengarkan aman dapat dihidupkan atau dimatikan. Ini ditentukan oleh nilai sumber daya R.bool.config_safe_media_volume_enabled.

Jika mode aktif, sistem menghitung waktu mendengarkan pada volume yang tidak aman (di atas 85 dB), dan secara berkala menyimpan nilai dalam suatu variabel Settings.Secure.UNSAFE_VOLUME_MUSIC_ACTIVE_MS. Ketika nilainya mencapai 20 jam, sebuah peringatan ditampilkan. Setelah menerima peringatan, nilainya diatur ulang dan penghitungan dimulai lagi.

Implementasi semacam itu cukup sederhana dan tidak memperhitungkan, misalnya, berapa lama pengguna mendengarkan 20 jam ini: mungkin dalam beberapa hari, atau mungkin mendengarkan selama 6-7 menit selama enam bulan (sesuai dengan standar, ini bukan ancaman untuk mendengar )

Logika mendengarkan aman terkonsentrasi di kelas AudioService.java kelas, Anda dapat melihat bidang yang sesuai di dalamnya:

// mMusicActiveMs is the cumulative time of music activity since safe volume was disabled.
// When this time reaches UNSAFE_VOLUME_MUSIC_ACTIVE_MS_MAX, the safe media volume is re-enabled
// automatically. mMusicActiveMs is rounded to a multiple of MUSIC_ACTIVE_POLL_PERIOD_MS.
private int mMusicActiveMs;
private static final int UNSAFE_VOLUME_MUSIC_ACTIVE_MS_MAX = (20 * 3600 * 1000); // 20 hours
private static final int MUSIC_ACTIVE_POLL_PERIOD_MS = 60000;  // 1 minute polling interval

Bidang ini mMusicActiveMs berisi jumlah milidetik yang didengarkan oleh pengguna pada volume yang tidak aman sejak konfirmasi terakhir dialog. Nilai awal diambil dari variabel Settings.Secure.UNSAFE_VOLUME_MUSIC_ACTIVE_MS. Dalam variabel yang sama, nilai mMusicActiveMs baru ditulis setiap menit.

Ada juga bidang mSafeMediaVolumeState, yang berisi kondisi saat ini dari sistem pendengaran yang aman:

  • DISABLED: mati
  • ACTIVE: diaktifkan, dan pada saat yang sama batas pendengaran telah tercapai, yang berarti Anda tidak dapat mengizinkan pengguna untuk meningkatkan volume sampai dia setuju dengan peringatan
  • INACTIVE: pada, batas belum tercapai


// mSafeMediaVolumeState indicates whether the media volume is limited over headphones.
// It is SAFE_MEDIA_VOLUME_NOT_CONFIGURED at boot time until a network service is connected
// or the configure time is elapsed. It is then set to SAFE_MEDIA_VOLUME_ACTIVE or
// SAFE_MEDIA_VOLUME_DISABLED according to country option. If not SAFE_MEDIA_VOLUME_DISABLED, it
// can be set to SAFE_MEDIA_VOLUME_INACTIVE by calling AudioService.disableSafeMediaVolume()
// (when user opts out).
private static final int SAFE_MEDIA_VOLUME_NOT_CONFIGURED = 0;
private static final int SAFE_MEDIA_VOLUME_DISABLED = 1;
private static final int SAFE_MEDIA_VOLUME_INACTIVE = 2;  // confirmed
private static final int SAFE_MEDIA_VOLUME_ACTIVE = 3;  // unconfirmed
private Integer mSafeMediaVolumeState;

Metode untuk memeriksa kelebihan batas terlihat seperti ini:

private void onCheckMusicActive(String caller) {
   synchronized (mSafeMediaVolumeState) {
       if (mSafeMediaVolumeState == SAFE_MEDIA_VOLUME_INACTIVE) {
           int device = getDeviceForStream(AudioSystem.STREAM_MUSIC);

           if ((device & mSafeMediaVolumeDevices) != 0) {
               sendMsg(mAudioHandler,
                   MSG_CHECK_MUSIC_ACTIVE,
                   SENDMSG_REPLACE,
                   0,
                   0,
                   caller,
                   MUSIC_ACTIVE_POLL_PERIOD_MS);
               int index = mStreamStates[AudioSystem.STREAM_MUSIC].getIndex(device);
               if (AudioSystem.isStreamActive(AudioSystem.STREAM_MUSIC, 0) &&
                   (index > safeMediaVolumeIndex(device))) {
                   // Approximate cumulative active music time
                   mMusicActiveMs += MUSIC_ACTIVE_POLL_PERIOD_MS;
                   if (mMusicActiveMs > UNSAFE_VOLUME_MUSIC_ACTIVE_MS_MAX) {
                       setSafeMediaVolumeEnabled(true, caller);
                       mMusicActiveMs = 0;
                   }
                   saveMusicActiveMs();
               }
           }
       }
   }
}

Cara mematikan peringatan


Untuk mematikan mendengarkan dengan aman, Anda perlu memastikan bahwa variabel tersebut mSafeMediaVolumeStatediberi nilai selama fase konfigurasi DISABLED.

Mari kita lihat di mana nilai awalnya ditetapkan:

private void onConfigureSafeVolume(boolean force, String caller) {
   ...
   boolean safeMediaVolumeEnabled =
           SystemProperties.getBoolean("audio.safemedia.force", false)
                   || mContext.getResources().getBoolean(
                 com.android.internal.R.bool.config_safe_media_volume_enabled);

   boolean safeMediaVolumeBypass =
           SystemProperties.getBoolean("audio.safemedia.bypass", false);

   int persistedState;
   if (safeMediaVolumeEnabled && !safeMediaVolumeBypass) {
       persistedState = SAFE_MEDIA_VOLUME_ACTIVE;
       /*  ,  mSafeMediaVolumeState   ACTIVE,  INACTIVE */
...
   } else {
       persistedState = SAFE_MEDIA_VOLUME_DISABLED;
       mSafeMediaVolumeState = SAFE_MEDIA_VOLUME_DISABLED;
   }

Kami melihat bahwa selain nilai sumber daya R.bool.config_safe_media_volume_enabled, ada dua properti yang mengaktifkan / menonaktifkan sistem mendengarkan aman: audio.safemedia.force dan audio.safemedia.bypass .

Untuk menonaktifkan peringatan, Anda perlu mengatur nilai audio.safemedia.bypass = true dalam file system / build.properties . Tetapi ini membutuhkan hak root. Jika tidak, maka Anda perlu memahami lebih lanjut dan mencari cara lain.

Cara menonaktifkan peringatan tanpa root


Mari kita lihat apa yang terjadi ketika Anda menutup dialog peringatan dengan mengklik OK, dan mencoba mereproduksi ini:

@Override
    public void onClick(DialogInterface dialog, int which) {
        mAudioManager.disableSafeMediaVolume();
    }

Metode disableSafeMediaVolumeinstance disebut AudioManager.

/**
* Only useful for volume controllers.
* @hide
*/
public void disableSafeMediaVolume() { … }

Itu ditandai dengan anotasi @hide. Ini berarti bahwa metode ini tidak akan dimasukkan dalam API publik meskipun ada pengubah publik. Sebelum Android 9, ini dapat dengan mudah dielakkan menggunakan refleksi. Namun sekarang, metode seperti itu masih bisa dipanggil, tetapi dengan bantuan trik yang disebut double-reflection :

val audioManager = getSystemService(Context.AUDIO_SERVICE) as AudioManager
val getDeclaredMethod = Class::class.java.getDeclaredMethod("getDeclaredMethod", String::class.java, arrayOf<Class<*>>()::class.java)
val disableSafeMediaVolumeMethod = getDeclaredMethod.invoke(AudioManager::class.java, "disableSafeMediaVolume", arrayOf<Class<*>>()) as Method
disableSafeMediaVolumeMethod.invoke(audioManager)

Panggilan berakhir dengan pengecualian.
java.lang.SecurityException: Only SystemUI can disable the safe media volume: Neither user 10307 nor current process has android.permission.STATUS_BAR_SERVICE.
Izin STATUS_BAR_SERVICE memiliki perlindunganLevel = "signature | privilege", itu tidak akan berfungsi.

Nah, coba ini. Kami akan memantau variabel Settings.Secure.UNSAFE_VOLUME_MUSIC_ACTIVE_MSdi mana nilai saat ini disimpan secara berkala mMusicActiveMs. Ketika nilai mulai mendekati 20 jam, kami akan meresetnya. Maka Anda perlu membuat AudioService membaca nilai baru dari pengaturan.

Anda Settings.Secure.UNSAFE_VOLUME_MUSIC_ACTIVE_MSdapat membaca nilai seperti ini:

val unsafeMs = Settings.Secure.getInt(contentResolver, "unsafe_volume_music_active_ms")

Hal yang sama menggunakan adb:

 adb shell settings get secure unsafe_volume_music_active_ms

Dan untuk menulis nilainya, aplikasi akan membutuhkan izin android.permission.WRITE_SECURE_SETTINGS.

Ia memiliki protectionLevel = "signature | privilege | pengembangan ", yang artinya dapat dikembalikan ke aplikasi menggunakan adb:

adb shell pm grant com.example.app android.permission.WRITE_SECURE_SETTINGS

Nilai itu sendiri dapat ditulis seperti ini:

Settings.Secure.putInt(contentResolver, "unsafe_volume_music_active_ms"
, 1)

Anda dapat melakukan hal yang sama dengan adb:

adb shell settings put secure unsafe_volume_music_active_ms 1

Lebih baik mengatur ulang ke 1, seperti yang dilakukan di AudioManager, tetapi tidak ke 0. Karena 0 sesuai dengan status AKTIF.

Sekarang Anda memerlukan AudioService untuk membaca nilai baru dan memperbarui nilai variabel lokal mMusicActiveMs.

Ada metode yang cocok di AudioManager.java

Metode ini ditandai dengan anotasi @hide. Panggilannya menggunakan refleksi ganda menimbulkan pengecualian:
java.lang.SecurityException: Izin Penolakan: pengaturan get / set untuk pengguna meminta dijalankan sebagai pengguna -2 tetapi memanggil dari pengguna 0; ini memerlukan android.permission.INTERACT_ACROSS_USERS_FULL
Ya, anotasi di @hidesini juga bukan kebetulan. Tentu saja, kami tidak dapat memperoleh izin ini. Ia memiliki protectionLevel = "signature | installer".

Masih ada satu cara untuk membuat AudioService membaca nilai baru - memulai kembali. Anda tidak dapat memulai kembali layanan sistem. Anda harus me-reboot perangkat, atau beralih ke pengguna lain, lalu kembali.

Sekarang adalah waktunya untuk menguji teorinya.

Installunsafe_volume_music_active_ms = 71 990 000 (10 detik tersisa, di mana Anda dapat mendengarkan musik pada volume tinggi)

adb shell settings put secure unsafe_volume_music_active_ms 71990000


Kami me-restart perangkat (Anda dapat beralih ke pengguna lain, dan kemudian kembali):

adb reboot

Kami menghubungkan headphone, nyalakan musik lebih keras. Dialog muncul dalam satu menit.

Sekarang kami mengulangi tindakan yang sama, tetapi menetapkan unsafe_volume_music_active_ms = 1. Nyalakan musik, tunggu sebentar. Dialog tidak muncul.

Ringkasan


Untuk menonaktifkan peringatan, Anda dapat melakukan hal berikut:

Jika Anda memiliki hak root

Tetapkan nilai audio.safemedia.bypass = true dalam sistem file / build.properties

Tanpa hak root

Anda perlu memantau nilai Settings.Secure.UNSAFE_VOLUME_MUSIC_ACTIVE_MSdan mencegahnya naik di atas 72.000.000 (20 jam) ) Setelah mengatur ulang nilai, Anda harus memulai ulang perangkat (atau beralih ke pengguna lain, lalu kembali).

Saya menulis kode untuk aplikasi sederhana yang melakukan pekerjaan ini, dan mengingatkan saya pada kebutuhan untuk reboot perangkat / masuk.
