Java / Spring：Speedmentを使用してCRUD REST APIを完全に生成する方法

この記事では、Speedmentを使用してデータベース用の完全なCRUD REST APIを作成する方法を学びます。



毎年、Spring FrameworkがJavaでのWeb開発に最も広く使用されているフレームワークの1つであることはますます明らかになっています。

次の10年が近づくにつれ、Springの最も人気のあるモジュールであるSpring Bootがメジャーアップデートを受けました。Spring Boot "2.2.0"の新しいバージョンと2020は、ほぼ完全に一致しています。したがって、Speedment開発チームは、Spring Bootプラグインを更新してギャップを埋めました。

これがSpeedmentを初めて聞く場合は、Spring Boot Speedmentプラグインを使用して、開発者がSpeedment、ORM、強力なStream APIベースのクエリインターフェースを使用してSpringデータベースアプリケーションを簡単に作成できる方法を示すので、お楽しみいただけます。

スピードメントについて


Speedmentは、JavaコミュニティにふさわしいORMとして説明できます。これは、迅速な開発とさらに高速なアプリケーションを愛する人のためのツールキットです。

選択したデータベースと通信する主要な手段としてJava Stream APIを使用することで、SpeedmentはJavaの経験を持つ開発者がくつろげるようにします。使いやすいAPIに加えて、Speedmentは開発者に、数秒でドメインモデルを生成するグラフィカルツールを提供します。

Speedment全般に興味がある場合は、次のリンクをクリックして、例を含む詳細なドキュメントをご覧ください。この記事の残りの部分では、Spring Bootプラグインに焦点を当てます。

春のブーツ：最速のスタート


Speedment Spring Bootプラグインは以前から存在していましたが、Speedment 3.2.2はメインの機能、つまりデータベースモデルから直接CRUD REST API全体を生成する機能を導入しました。

最初から始めるか、すでにプロジェクトを持っているかに応じて、手順は異なりますが、プロジェクトの状態に関係なく、統合プロセスが単純であることを確認してください。

Spring Bootを既存のSpeedmentプロジェクトに統合する場合は、pom.xmlに次の2つの要素を追加する必要があります。

<plugin>
    <groupId>com.speedment.enterprise</groupId>
    <artifactId>speedment-enterprise-maven-plugin</artifactId>
    <version>${speedment.version}</version>

    <configuration>
        <components>
            component>com.speedment.enterprise.plugins.spring.SpringGeneratorBundle</component>
        </components>
        <appName>${project.artifactId}</appName>
        <packageName>${project.groupId}</packageName>
    </configuration>
    ... // Database connector dependency           
</plugin>
<dependencies>
 ...
<dependency>
    <groupId>javax.annotation</groupId>
    <artifactId>javax.annotation-api</artifactId>
    <version>1.3.1</version>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <version>2.2.1.RELEASE</version>
</dependency>
<dependency>
    <groupId>com.speedment.enterprise.plugins</groupId>
    <artifactId>spring-runtime</artifactId>
    <version>${speedment.version}</version>
</dependency>
...
</dependencies>

ドメインモデルの次の再生成時にこれらの要素を追加した後、Spring用のSpring固有の構成を生成する必要があります。生成された構成は、Speedmentによって生成されたさまざまなマネージャーのデータベース接続とビン登録のプロパティを制御します。

プロジェクトを最初から開始する場合は、プロジェクトイニシャライザーに移動します。ここで、Spring Boot用の既成のプロジェクトが作成されます。プロジェクトの構成方法は自由ですが、生成されたプロジェクトにSpring Bootプラグインを含めるには、プラグインセクションでSpringの横のチェックボックスがオンになっていることを確認してください。プロジェクト構成が適切な場合は、[ ダウンロード ]ボタンをクリックすると、プロジェクトアーカイブを使用できるようになります。



CRUDを簡略化


プラグインをインストールすると、Speedmentツールで特定のSpring Bootオプションが利用可能になり、REST APIの構成に使用できます。次のコマンドを実行すると、Speedmentツールを実行できます。

mvn speedment:tool

初めてSpeedmentを使用する場合は、Hello Speedmentの簡単なチュートリアルに従ってワークフローに慣れることをお勧めします。

デフォルトでは、SpeedmentツールはREST APIを生成しません。特定のテーブルのCRUD REST APIを生成するには、ツリービューでテーブルを選択し、[REST コントローラーの作成/更新/削除を有効にする]オプションとともに[ @RestControllerを生成]チェックボックスを選択します。[ 生成 ] をクリックすると、一連のRESTマッピングが作成されます-特定のテーブルのCRUD操作を実装するURL。



それで全部です！手動でコーディングする必要はありません。次のコマンドを実行して、生成されたSpringプロジェクトを実行するだけです。

mvn spring-boot:run

RESTを使用したエンティティの作成


作成したREST APIを使用して新しいエンティティを作成するには、ツールで指定されたパスでPOSTリクエストを実行する必要があります。デフォルトでは、このパスの名前は、スキーマ名の接頭辞が付いたテーブルの名前と一致します。[ RESTエンドポイント]チェックボックスをオフにすると、任意のパス名を指定できます。リクエストの本文には、JSONオブジェクトにキーと値のペアを含める必要があります。ここで、Speedmentツールからの「RESTフィールド名」の値はキーです。

たとえば、列id、name、およびageを持つpersonという名前のテーブルがある場合、次のクエリを送信して新しいエンティティpersonを作成します。

POST localhost:8080/db/person
{
    "id": 1,
    "name": "Jane",
    "age": 25
}

または

curl -d '{"id": 1,"name": "Jane","age": 25}' -H "Content-Type:application/json" -X POST localhost:8080/db/person

id 列の値が自動的に生成される（またはシーケンスを使用する）場合は、その値をPOST本文から除外できます。特定の列をPOST本文から強制的に除外する場合は、ツールツリービューで対応する列をクリックし、[ Create Body in Create Body ]ボックスをオフにして、ドメインモデルを再作成します。一方、リクエストの実行時にPOSTボディに特定の列が存在することを確認する場合は、[ボディの作成に必要]をオンにして、ドメインモデルを再作成します。



RESTを使用したエンティティの取得


新しく作成したエンティティを取得するには、エンティティの作成に使用したのと同じ方法でGETリクエストを実行する必要があります。

GET localhost:8080/db/person

または

curl localhost:8080/db/person/1

完了したリクエストは、既存のすべてのエンティティのJSON配列を返します。私たちの場合、それは私たちの唯一のエンティティでのみ配列を返します：

[
    {
        "id": 1,
        "name": "Jane",
        "age": 25
    }
]

フィルターやソーターなどの高度な概念については、Speedmentの公式ドキュメントをご覧ください。

RESTを使用して既存のエンティティを更新する


REST APIを使用して既存のエンティティを更新することは、エンティティを作成することに似ています。POSTリクエストの代わりに、PATCHリクエストを実行し、パスはエンティティ識別子で展開されます。エンティティのどの識別子がこのテーブルの主キー列に依存します。PK（主キー）列は数値型であるため、エンティティ識別子は整数になります。

前の例で作成したエンティティを更新するには、次のクエリを実行します。

PATCH localhost:8080/db/person/1
{
    "name": "Mike",
    "age": 43
}

または
curl -d '{"name": "Jane","age": 25}' -H "Content-Type:application/json" -X PATCH localhost:8080/db/person/1

デフォルトでは、すべての列がPATCH本文に含まれます（主キー列を除く）が、必須ではありません。特定の列を強制的にリクエストの本文から除外する場合は、ツールツリービューで対応する列をクリックし、[ 更新本文に含める]チェックボックスをオフにして、ドメインモデルを再作成します。一方、リクエストの本文に列が存在することを確認する場合は、[更新本文で必須]をオンにして、ドメインモデルを更新します。

RESTを使用したエンティティの削除


REST APIを使用したエンティティの削除は非常に簡単です。エンティティの更新に使用されるのと同じ方法でDELETEリクエストを実行します。

前の例で作成および更新したエンティティを削除するには、次のクエリを実行します。

DELETE localhost:8080/db/person/1

または

curl -X DELETE localhost:8080/db/person/1

概要


新しいプロジェクトを立ち上げるのは面倒なことがあります。開発者として、これらの不便をできるだけ回避し、直接コーディングに突入したいと考えています。新しいSpeed Boot Boot Springプラグインを使用すると、開発者は必要なすべての構成とRESTコントローラーをデータベースから直接自動的に生成することにより、新しいプロジェクトの立ち上げを加速できます。

Speedment Spring Bootプラグインが提供する新しいCRUD機能のこの短いガイドをお楽しみいただけたと思います。新しい関数の詳細な説明に興味がある場合は、公式ドキュメントにアクセスしてください。新しいCRUD関数の詳細な説明と例が掲載されています。
