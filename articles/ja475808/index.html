GitHubは千年前のリポジトリを作成し、オープンソースリポジトリを後世に保存します


北極世界アーカイブをホストしていた元炭鉱。写真：Guy Martin / Bloomberg Businessweek

フリーソフトウェアは、現代文明の基盤であり、すべての人類の共通の遺産です。 GitHubアーカイブプログラムの使命は、アレクサンドリア図書館の歴史が決して繰り返されないように、このコードを将来の世代のために保存することです。

これを行うために、GitHubはSvalbard上の長期の Arctic Code Vaultストレージを含む、さまざまなメディア上の多くのバックアップを維持します。それは、永久凍土で250メートルの深さの元炭鉱にあり、少なくとも1000年の貯蔵寿命のために設計されています。

人間のコードのスナップショットが2020年2月2日に撮影されます。

長期データストレージプロジェクトは、Long Now Foundation、Internet Archive、Software Heritage Foundation、Arctic World Archiveおよびその他のパートナーと共同で立ち上げられました。

LOCKSSプロジェクト


今日の重要なコードは、時間の経過とともに忘れられたり失われたりする可能性があります。最悪の事態は、地球規模の大惨事が発生した場合に、数十年に渡って設計されたHDD、SSD、CD、DVDの「短命」メディアに保存されていたすべての情報が失われ、30年の条件付き寿命で温度と湿度を厳密に制御する必要があるテープである場合です。 。

この問題の解決策は、バックアップの複製、つまり、複数の組織によるソフトウェアのアーカイブと異なる形式でのアーカイブです。LOCKSSと呼ばれるこのプロジェクトは、ほぼ20年前から始まりました。 2019年5月、LOCKSS 2.0-alphaプログラムが導入され ました。これは、多くの参加者と外部ストレージのサポートを受けた、長期間にわたる分散データストレージ用の最初のプロトタイプソフトウェアです。

システムの開発者は、ハードウェアが短命のメディアよりもはるかに耐久性が高いという事実から進んでいます。したがって、「現代の動作するコンピュータが存在する可能性のある将来のオプションは数多くありますが、ソフトウェアはほとんど失われています」。

GitHubは、役に立つかもしれない多くの失われた技術を思い起こします：ローマのコンクリート（そのレシピは2014年にのみ再発見されました）、抗マラリア薬DFDT、Saturn-5ロケットの失われた図面。GitHubプログラムのウェブサイトは、「バックアップと同様に、GitHubアーカイブプログラムも予期しない将来に向けて設計されている」と思われるまで、今日のソフトウェアが奇妙で長い間忘れられていると考えられる未来を想像するのは簡単です。アーカイブ

Githubアーカイブ


GitHub Archiveは、3つのレベルのバックアップを提供します。

  • ホット：ほぼリアルタイム
  • 暖かい：月から年に更新
  • コールド：5年以上ごとに更新

GitHubユーザーによるアクションの後、すべてのGitデータが世界中のいくつかのデータセンターに複製されます。 GitHub上のGitバックアップ、問題、プールリクエスト、およびすべてのユーザーデータは、いくつかの場所に保存されます。この情報は、GitHub APIを通じてリアルタイムで利用できます。

さらに、再帰的なインデックス作成はGHTorrentクローラーによって編成され、毎日または毎月アーカイブをアップロードします。 GH Archiveを使用すると、アーカイブのスナップショットをBigQueryで取得できます。コードのその他のコピーは、インターネットアーカイブ用のよく知られた「Time Machine」にあり、コピーは複数の場所に保存されます。最後に、Software Heritage Foundationは定期的にGitHubをスキャンし、そのパブリックリポジトリをアーカイブに追加します。アーカイブにはパブリックAPIがあります。

Arctic GitHubストレージ


2020年2月2日、GitHubはすべてのアクティブなパブリックリポジトリのコピーを作成し、それらをArctic GitHubリポジトリに配置します。

データは、長期データストレージを専門とするノルウェーの企業Piqlが提供する3,500フィートのフィルムリールに保存されます。ISO測定によると、ポリエステルにハロゲン化銀を使用したこのフィルムの寿命は500年です。シミュレーションエージングテストでは、Piqlフィルムが情報を少なくとも2倍保持することが示されています。

さらに、GitHub ArchiveはMicrosoft Silicaプロジェクトの研究者と協力して、フェムト秒レーザーを使用してすべてのパブリックリポジトリを石英ガラスプレートに記録しています。このメディアは、10,000年以上にわたってデータの安全性を保証します。

GitHub Arcticコードリポジトリは、永久凍土の深さ250メートルにあるArctic World Archive（AWA）に基づいています。アーカイブは、北極からさほど遠くないスピッツベルゲン諸島の元炭鉱にあります。地球温暖化は数メートルの永久凍土層にのみ影響し、近い将来（数千年）に鉱山を脅かすことはありません。

スバールバル諸島は国際条約により非武装地帯として規制されています。これは、地球上で最も遠隔で地政学的に安定した人間居住地の1つであるとGitHubは言います。有名な世界の種子貯蔵庫が近くにあります。


スバールバル諸島ワールドシードストア

AWAは、ノルウェーの国営鉱業会社Norske Spitsbergen Kulkompani（SNSK）とデジタル保護プロバイダーのPiql ASの共同イニシアチブです。イタリア、ブラジル、ノルウェー、バチカンおよびその他の国の歴史的および文化的データはすでにそこに保存されています。


写真：Guy Martin / Bloomberg Businessweek

GitHubコイルは、気密チャンバー内の鋼壁コンテナーに保管されます。すべてのアクティブなGitHubリポジトリと非アクティブなリポジトリの大部分（星や依存関係などで判断）、100 KBまでのすべてのバイナリファイルは、スナップショット02.02.2020に分類されます。個別のtarファイル内の各リポジトリ。すべては120 GBの200コイルに適合します。

アーカイブとともに、QR、ファイル形式、文字エンコーディング、その他の重要なメタデータをデコードするための人間が読めるカタログとテクニカルマニュアルを作成し、子孫がデータをソースコードに変換できるようにします。

アーカイブには、将来の読者が稼働中のコンピューターを持たず、テクノロジーをゼロから復元する必要がある場合に備えて、Tech Techの一般ガイドも含まれます。
