This short article will tell you about integrating JDI Light with Selenium Grid.

JDI is a family of frameworks for testing automation. At the moment, it consists of JDI Light for UI automation, JDI Dark for backend automation and JDI Mobile. The name stands for Just do it : the main idea of ‚Äã‚ÄãJDI is to use it to easily test and maintain tests, and all successful working finds to save and accumulate.



Selenium Grid is a cluster of several Selenium servers. It is designed to organize a distributed network that allows you to simultaneously run many browsers on a large number of machines. There is a dedicated server, which is called a ‚Äúhub‚Äù or ‚Äúswitch‚Äù, and the rest of the server is called ‚Äúnodes‚Äù or ‚Äúnodes‚Äù.



Integration:



The first thing we need is the selenium grid itself, you can download it on the official selenium website ( selenium.dev/downloads/ ), you can also see the documentation there.



Next, you need to run selenium grid, for this, in the folder with the downloaded file, open the command line and execute the commands.



  1. Running selenium grid in hub mode: 



    java -jar selenium-server-standalone-3.141.59.jar -role hub -port 4444


    • selenium-server-standalone-3.141.59.jar - selenium grid file
    • port 4444 - the port on which the hub will work, you need to know to connect the nodes and create RemoteWebDriver.


  2. Connecting a node to a hub: 



    java -Dwebdriver.chrome.driver=chromedriver.exe -jar selenium-server-standalone-3.141.59.jar -role node -hub http://localhost:4444/grid/register -browser "browserName=chrome,maxInstances=3,platform=WINDOWS" -port 5557


You can also use json files to configure the nodes. You can read more about the capabilities and settings of Selenium Grid on the official website of the selenium project ( selenium.dev/documentation/en/grid/ ).



You can check whether the hub has started, and if the nodes are connected to it, from the console or by clicking on the link where the hub is registered. In the configuration presented above, we created it locally on port 4444, so after opening the link ( localhost: 4444 / grid / console ) in the browser, we will see the following site:







The selenium grid configuration is complete, and we proceed to use it.



If you use pure Selenium WebDriver, to get the WebDriver object, we will need to create an object 



RemoteWebDriver(URL remoteAddress, Capabilities capabilities)
- remoteAddress ‚Äì url 
- capabilities ‚Äì


When creating an object, we will turn to the hub and try to take a node from it that has the browser we need. If such a node is not found, an exception will be thrown.

Out of the box, Selenium WebDriver does not provide the ability to change the browser on the fly or store the browser we need in the settings file. To realize these opportunities, developers spend time and energy.



But the developers of JDI Light took care of the implementation of these features. Using JDI, you can save time and effort, because to get a WebDriver object, we only need to add the hub address to the JDI settings file: 



driver.remote.url=http://localhost:4444/wd/hub


and specify in the settings file the browser on which to run the tests: 



driver=chrome


Additional information about the settings file and other features of JDI Light ( jdi-docs.github.io/jdi-light/#jdi-settings ).



The hub will try to find a suitable node. If the node is not found, an exception will be thrown: 



java.lang.RuntimeException: java.lang.RuntimeException: Failed to do screenshot. No Drivers run
