IoTのユニバーサルサーバーとしての4Gルーター


ICR-3200 シリーズのルーターは、クラシックバンドル（シングルボードコンピューター+モデム+ルーター）を置き換えるように設計されています。これで、必要なすべてのロジックをルーターで直接実行できます。強力なARMプロセッサ、512 MBのRAM、および最大2 GBの内部フラッシュメモリのおかげで、ルーターでnodejsサーバーを起動することもできます。

シリアルRS-232 / 485インターフェースもルーターに組み込まれており、オペレーティングシステムですぐに使用でき、完全なルートアクセスが可能です。低レベルのハードウェアAPIを操作するためのネイティブプログラムとC / C ++ライブラリを構築するためのオープン環境も利用できます。
ディープスリープ機能は、バッテリー電源での作業時にエネルギーを節約します。ルーターは、通信が本当に必要な場合にのみアクティブ化できます。

ディープスリープモード

シリアルインターフェース

デフォルトでは、両方のインターフェイスがLinuxの標準シリアルデバイスとして機能するように使用できます。

ls -la /dev/ttyS*

シェルスクリプトによるハードウェアリソース


単純な自動化タスクの場合は、ライトの表示を制御し、デジタルピンのステータスを受信/設定できるシェルスクリプトを使用すると便利です。

ライト表示は、コマンド実行の現在の状態を示すのに便利です。たとえば、データ転送表示-頻繁な点滅、接続試行-まれな点滅、単純-が連続的に点灯します。ledコマンドを考えてみましょう。入力では、引数は1つだけです。LEDの点滅の種類です。

# led
led on|off|fast|slow

これが実際の生活の様子です。


シェルコマンドによるユーザーインジケーターの動作モードのデモ io

コマンドは、デジタルピンを操作するために使用されます。

$ io
Usage: io [get <pin>] | [set <pin> <value>]

io set out0 1 #    OUT0   1 (LOW)
io get bin0   #      BIN0

この場合のロジックが逆になることを覚えておくことが重要です。したがって、1 =低、0 =高。ioコマンドを使用すると、bashスクリプトからデジタルピンを簡単に操作して、自動化と制御のための単純なロジックを構築できます。

デジタル入力とインジケーターの操作が簡単であることを示すために、LEDを使用してデジタル入力のステータスを表示する役に立たないスクリプトを作成します。

#!/bin/bash
while true
do
	[ $(io get bin0) -eq 0 ] && led fast || led on
done

スクリプトの原理は単純です。BIN0が状態0の場合、ダイオードは頻繁に燃焼し、そうでなければ継続的に燃焼します。ライブの様子は次のとおりです。
