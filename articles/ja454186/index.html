リアルタイムパイプライン用のApache BeamとDataFlow


今日の投稿は、私が最近仕事で取り組んだタスクに基づいています。これを実装し、ブログポストの形式で行われた作業について説明できることを本当に嬉しく思います。これにより、データエンジニアリングに取り組む機会と、チームにとって非常に役立つ何かを行う機会が与えられました。少し前に、私たちのシステムには、データを操作するための製品の1つに関連付けられたかなり大量のユーザーログが保存されていることがわかりました。誰もこのデータを使用していなかったことが判明したので、定期的に分析し始めたときに何がわかるかすぐに気になりました。しかし、途中でいくつかの問題がありました。最初の問題は、データがインスタント分析に利用できない多くの異なるテキストファイルに保存されていたことです。第二の問題はクローズドシステムに保存されていたため、お気に入りのデータ分析ツールを使用できませんでした。

私は、ユーザーが簡単にアクセスできるようにする方法を決め、このデータソースをユーザーインタラクションソリューションの一部に埋め込むことにより、少なくとも何らかの価値を追加する必要がありました。しばらく考えた後、このデータをクラウドデータベースに転送するパイプラインを構築して、私とチームがデータにアクセスして結論を​​出し始めることにしました。しばらく前にCourseraでデータエンジニアリングの専門を終えた後、私はプロジェクトでコースツールのいくつかを使用することを熱望していました。

したがって、クラウドデータベースにデータを配置することは、最初の問題を解決するためのスマートな方法のように見えましたが、問題番号2で何ができるでしょうか？幸い、PythonやGoogle Cloud Platform（GCP）などのツールにアクセスできる環境にこのデータを転送する方法がありました。しかし、それは長いプロセスだったので、データ転送の終了を待つ間、開発を継続できるようにする必要がありました。私が見つけた解決策は、PythonのFakerライブラリを使用して偽のデータを作成することでした。私はこのライブラリを以前に使用したことがありませんでしたが、すぐにそれがいかに便利かを理解しました。このアプローチを使用することで、コードを記述し、実際のデータなしでパイプラインをテストすることができました。

前述のことに基づいて、この投稿では、GCPで利用可能ないくつかのテクノロジーを使用して、上記のパイプラインをどのように構築したかを説明します。特に、Apache Beam（Pythonバージョン）、Dataflow、Pub / Sub、Big Queryを使用してユーザーログを収集し、データを変換して、さらに分析するためにデータベースに転送します。私の場合、データがリアルタイムで届かなかったため、Beamのバッチ機能のみが必要で、Pub / Subは必要ありませんでした。ただし、実際にはストリーミングバージョンに焦点を当てます。

  • そして最後に、Apache Beamを使用します。つまり、Pythonバージョンに焦点を当てて、パイプラインを作成します。このツールを使用すると、GCPと統合するストリーミングまたはバッチ処理用のパイプラインを作成できます。これは特に並列処理に役立ち、抽出、変換、読み込み（ETL）などのタスクに適しているため、変換や計算を使用してデータをある場所から別の場所に移動する必要がある場合は、Beamが適しています。

  1. ユーザーのログデータは、Pub / Subセクションで公開されます。

Fakerを使用して疑似データを作成する


前述したように、データへのアクセスが制限されているため、実際と同じ形式で疑似データを作成することにしました。データを期待している間にコードを記述してパイプラインをテストできたので、これは本当に便利な練習でした。このライブラリのその他の機能を知りたい場合は、Fakerのドキュメントをご覧になることをお勧めします。ユーザーデータは、一般的に次の例のようになります。このフォーマットに基づいて、リアルタイムデータをシミュレートするための行ごとのデータを生成できます。これらのログは、日付、リクエストのタイプ、サーバーからの応答、IPアドレスなどの情報を提供します。

from faker import Faker
import time
import random
import os
import numpy as np
from datetime import datetime, timedelta



LINE = """\
{remote_addr} - - [{time_local}] "{request_type} {request_path} HTTP/1.1" [{status}] {body_bytes_sent} "{http_referer}" "{http_user_agent}"\
"""


def generate_log_line():
    fake = Faker()
    now = datetime.now()
    remote_addr = fake.ipv4()
    time_local = now.strftime('%d/%b/%Y:%H:%M:%S')
    request_type = random.choice(["GET", "POST", "PUT"])
    request_path = "/" + fake.uri_path()

    status = np.random.choice([200, 401, 404], p = [0.9, 0.05, 0.05])
    body_bytes_sent = random.choice(range(5, 1000, 1))
    http_referer = fake.uri()
    http_user_agent = fake.user_agent()

    log_line = LINE.format(
        remote_addr=remote_addr,
        time_local=time_local,
        request_type=request_type,
        request_path=request_path,
        status=status,
        body_bytes_sent=body_bytes_sent,
        http_referer=http_referer,
        http_user_agent=http_user_agent
    )

    return log_line
