ZPM-InterSystems IRISのパッケージマネージャー

さまざまなプラットフォームのパッケージマネージャーが非常に広く使用されており、既成のコンポーネントやライブラリをすばやくインストールして構成できます。また、独自のソリューションを展開するためにも使用されます。パッケージマネージャーが依存関係を処理することが重要です。アプリケーションが特定のバージョンのある種のライブラリを使用している場合、パッケージマネージャは、アプリケーションをインストールするときに、このライブラリの必要なバージョンもインストールします。

インターシステムズのIRISでは、パッケージマネージャーであるZPMも利用できるようになりました。

ZPMを使用すると、モジュールを検索、インストール、更新でき、モジュールの公開にも使用できます。各モジュールは、個別のアプリケーション、ライブラリ、フレームワーク、ユーティリティ、またはインターシステムズのテクノロジーの使用例です。

ZPMパッケージマネージャー


ZPMは2つのコンポーネントで構成されています。

  • クライアント（CLI）
    IRISにインストールされ、モジュールの管理（インストールなど）に使用されるユーティリティ
  • レジストリ
    モジュールの管理に使用されるモジュールとメタ情報のデータベース。

デフォルトでは、ZPMクライアントはレジストリを使用するように構成されています。pm.community.intersystems.comは開発者コミュニティのレジストリです。このレジストリは、多くの例、ユーティリティ、ライブラリを公開しています。

独自のレジストリを取得して使用することもできます（この記事の後半で詳しく説明します）。

どこから始めるか


1. ZPMクライアントをインストールする


このリンクから最新のインストーラーをダウンロードしてください。

ダウンロードしたクラスを、ポータル、スタジオ、またはターミナルを介して、便利な方法で任意の領域にインポートします。

USER>Do $System.OBJ.Load("/path/zpm.xml", "ck")

Dockerを使用する場合、InterSystems IRIS Community EditionおよびInterSystems IRIS for Health Community Editionイメージを使用できます。これらのイメージには、最新バージョンのZPMが含まれています（詳細については、ドキュメントを参照してください）。

2. ZPMを起動する


インストール後、ZPMはどの地域でも利用できます。
ZPMを使用するには、ターミナルでzpmコマンドを入力するだけで、ZPMパッケージマネージャーのコマンドラインインターフェイス（CLI）を実行できます。

MYNS> zpm
zpm: MYNS>

helpコマンドを使用すると、使用可能なすべてのコマンドのリストを取得できます。

zpm: MYNS>help


3.モジュールのインストール


まず、検索コマンドを使用して、レジストリで使用可能なモジュールのリストを確認します。

zpm: MYNS>search
 
registry https://pm.community.intersystems.com:
analyzethis 1.1.4
blocksexplorer 2.2.1
dsw 2.1.41
…

モジュールまたはアップデートの最新バージョンをインストールするには、installコマンドを使用します。インストールは現在のエリアで実行されます。

zpm: MYNS>install dsw

ご覧のように、mdx2jsonモジュールが最初にインストールされ、dswモジュールが依存するモジュールがインストールされ、次にdswがインストールされます。

これですべてです-モジュールがインストールされ、設定されました！

多くのコマンドには特別なフラグがあります。たとえば、-vまたは-verboseフラグを使用すると、次のように詳細を表示できます。

zpm: MYNS>install dsw -v

ドキュメント ページで、またはヘルプコマンドを使用して、引数とフラグのコマンドの完全なリストを確認してください。

独自のパッケージを準備して公開する


独自のパッケージを公開するには、以下を行う必要があります。

  • InterSystems IRISでコードを機能させる。
  • module.xmlファイルを準備します。

module.xmlファイルには、パッケージ、パッケージに含まれるリソース、パッケージのインストール時に必要な依存関係とコンポーネント（Webアプリケーションの作成など）が記述されています。

以前にインターシステムズのIRISインストールマニフェスト（％インストーラー）を使用したことがある場合は、module.xmlとXDATAブロックに多くの共通点があります。

ドキュメントでのxml形式の説明

公開されたモジュールのmodule.xmlの例を見つけることができます：
github.com/isc-zpm/DeepSeeWeb/blob/master/module.xml
github.com/isc-zpm/Samples-BI/blob/master/ module.xml

パッケージを公開レジストリpm.community.intersystems.comに公開するには、アプリケーションをOpen Exchangeで公開し、その公開リポジトリを指定するだけで十分です。

独自のレジストリを設定する


独自のレジストリの作成はオプションですが、それを使用すると、モジュールがどのようにアセンブルおよびインストールされるかを確認できます。

アプリケーションをパブリックレジストリpm.community.intersystems.comに公開する場合は、まずローカルレジストリのすべての手順を確認することをお勧めします。

独自のレジストリをインストールするには、installコマンドを使用できます。
レジストリ領域に移動し（以下、これをREGISTRYと呼びます）、実行します

REGISTRY>zpm
zpm: REGISTRY>install zpm-registry

レジストリが機能していることを確認します。ブラウザでlocalhost：52773 / registry / _ping ページを開きます（IRISの設定によっては、別のポートを指定する必要がある場合があります）。ページにアクセスするとき、IRISユーザーのログインとパスワードを指定する必要があります（例：_system / SYS）。

{"message"： "ping"}が表示された場合-レジストリは正常にインストールされています。

次に、新しいレジストリで動作するようにZPMクライアントを構成します。

次のコマンドを実行します。

zpm: MYNS>repo -r -n registry -url http://localhost:52773/registry/ -user _system -pass SYS

このコマンドは、ZPMクライアントが、指定されたURLでアクセス可能なレジストリをリモートレジストリとして使用し、アクセス時に引数-userおよび-passでそれぞれ指定されたユーザー名とパスワードを渡す必要があることを示しています。

公開モジュール


module.xmlとモジュールのクラスが配置されている別のフォルダーが作成されていることを確認します（クラスは通常/ srcフォルダーに配置されます）。以下の例では、このモジュールはdemo-moduleと呼ばれます（名前はmodule.xmlで指定されています）。

テストレジストリに切り替えたことを確認します。

loadコマンドを使用して、コードを現在の領域にロードします。

zpm: MYNS>load /path/to/module/folder
[demo-module]  Reload START
[demo-module]  Reload SUCCESS
[demo-module]  Module object refreshed.
[demo-module]  Validate START
[demo-module]  Validate SUCCESS
[demo-module]  Compile START
[demo-module]  Compile SUCCESS
[demo-module]  Activate START
[demo-module]  Configure START
[demo-module]  Configure SUCCESS
[demo-module]  Activate SUCCESS

このコマンドは、いくつかのアクションを実行します-コードをダウンロードしてコンパイルし、モジュールを構成します。

これで、ZPMクライアントはモジュールに関するすべてを認識し、それを公開できます。

パブリッシュするには、publish引数を指定してmodule-actionコマンドを使用します。

zpm: MYNS>module-action demo-module publish

module-actionコマンドは省略でき、短く書くことができます。

zpm: MYNS>demo-module publish
[demo-module]  Reload START
[demo-module]  Reload SUCCESS
[demo-module]  Module object refreshed.
[demo-module]  Validate START
[demo-module]  Validate SUCCESS
[demo-module]  Compile START
[demo-module]  Compile SUCCESS
[demo-module]  Activate START
[demo-module]  Configure START
[demo-module]  Configure SUCCESS
[demo-module]  Activate SUCCESS
[demo-module]  Package START

 	/var/folders/9f/r62h3fxj42b5fzb0hgnb28m40000gn/T/direyLtX5/demo-module-1.0.0/
 
Module package generated:
 	/var/folders/9f/r62h3fxj42b5fzb0hgnb28m40000gn/T/direyLtX5/demo-module-1.0.0.tgz

[demo-module]  Register SUCCESS

このコマンドを実行すると、モジュールは最初に「アセンブル」（つまり、公開の準備）され、一時ディレクトリにエクスポートされ、アーカイブされてから、現在アクティブなレジストリに公開されます。

検索コマンドを実行すると、リストにモジュールが表示されます。

zpm: MYNS>search                          
 
registry http://localhost:52773/registry/:
demo-module 1.0.0

インストールが成功したことを確認するには、ZPMを使用してテストレジストリからモジュールを新しいIRISインストールまたは新しい領域にインストールします。

デフォルトのレジストリに切り替える


デフォルトのレジストリ（pm.community.intersystems.com）での作業に切り替えるには、-reset-defaultsフラグを使用します。

zpm: MYNS>repo -r -n registry -reset-defaults


モジュールをコミュニティレジストリに投稿する


モジュールをコミュニティレジストリ（pm.community.intersystems.com）に公開して、インターシステムズのすべての開発者がソリューションにインストールして使用できるようにすることができます。
これを行うには、InterSystems Open Exchange（InterSystems上のアプリケーションのマーケットプレイス）でソリューションを公開し、インストール中にソリューションがパッケージマネージャーモジュールであることを示します。

さらに、Open Exchangeでのソリューションのインストールに関するこのビデオを見ることができます。
