SSD。内部構造とその構築原理

お客様は、ソリッドステートドライブの内部設計に関心を寄せることがよくあります。ドライブの信頼性を決定するものは何ですか？重要なデータをフラッシュドライブに保存することが不可能で、信頼性の高いフラッシュドライブがあるのはなぜですか？ SSDコントローラーにマルチコアプロセッサが配置されるのはなぜですか？フラッシュドライブが大きなボリュームをすばやく書き込み、小さなボリュームがゆっくりと書き込むのはなぜですか？多くの質問は、トピックに長い間浸ることを必要とし、議論されるであろうソリッドステートドライブの内部構造に関する小さな教育プログラムの後にいくつかはそれ自体で消えます。


USBフラッシュドライブ、microSD / SDメモリカード、およびSSDドライブは、実際には「近親者」ですが、一見すると異なるデバイスです。 3種類のデバイスはすべて、NANDフラッシュメモリに基づくソリッドステートドライブです。家庭では一般に「SSDディスク」と呼ばれているおかしな事実は、文字通りの意味ではディスクではありません。 「ディスク」という名前は、歴史的にハードディスクドライブ（HDD）から継承されています。

アプリケーションの違いにもかかわらず、一般的にすべてのSSDのアーキテクチャは同じに見えます。

ソリッドステートドライブは次のとおりです。

  • コントローラは、フラッシュメモリのNAND構造で発生するビットエラーの読み取り、書き込み、データ整合性制御、および訂正の機能を実行するソリッドステートドライブの主要な要素です。一方では、コントローラーは外部SATA / USB / SD / PCIeインターフェースを介してホストに接続し、他方では、ONFI /トグル接続インターフェースを介してNANDフラッシュチップに接続します
  • NANDフラッシュメモリ -ドライブのメモリサイズを形成するマイクロ回路の配列


コントローラによって実行されるタスクを理解するには、NANDフラッシュメモリの構成の基本を理解している必要があります。NANDフラッシュメモリチップは、接続インターフェイスから始まり、情報ストレージの信頼性に至るまで、使用方法が非常に特殊です。

コントローラ


コントローラはホストへの接続を提供し、実際にはドライブのように見えます。一般に、任意のドライブの任意のコントローラーのアーキテクチャーは典型的に見えます。ホスト（EXT_IF）およびNANDメモリー（FLASH_IF）とデータを交換するためのハードウェアインターフェイスブロックがあります。インターフェイスのブロック間には、必須のバッファーメモリ（MEM_BUF）があり、データのオンラインキャッシュと書き込み/読み取りデータストリームの平滑化のために設計されています。 USBストレージコントローラでは、バッファメモリは数十キロバイトで、コントローラ自体に直接配置されています。 SSDなどの高性能システムは、外部メモリチップを使用します。インターフェイスユニットとバッファメモリ間のデータは、直接メモリアクセスチャネル（DMA、ダイレクトメモリアクセス）。データ伝送へのプロセッサの参加は、DMAチャネルを構成し、ブロックの動作を同期させることです。


プラグインドライブはいずれも「ブロックデバイス」です。ブロックデバイスは、データが論理ブロックの順次チェーンとして格納されているデバイスであり、論理ブロックアドレス（LBA）でアクセスできます。ほとんどのデバイスは、「セクター」と呼ばれる512バイトのブロックサイズをサポートしています。セクターは、ブロックデバイスで上書きできる最小の個別情報です。つまり、1バイトを置き換えるには、ホストがセクター全体をストレージデバイスに転送する必要があります。

物理レベル（物理アドレスへの書き込み/読み取り）でドライブを操作するプログラマーにとって、あるセクターからの書き込みと読み取りが他のセクターのデータに影響を与えないことは明らかです。ブロックデバイスのユーザーにとっても明らかですが、前述のように、NANDフラッシュメモリはそのような機会を提供しません。 NANDメモリアレイの1つのセクタを置き換えるには、フラッシュメモリのNANDブロック全体を上書きする必要があります。そのサイズはメガバイトです。この問題を解決するこの方法は、NANDフラッシュメモリへの書き込み速度が許容範囲を超えて許容できないほど低下するため、非常に非効率的です。さらに、オペレーティングシステムは同じデバイスアドレス（FATレコードなど）に書き込むことが多いため、その場合、消去リソースが限られているため、NANDフラッシュブロックはすぐに使用できなくなります。データの書き込み/読み取りの速度を上げ、NANDフラッシュメモリの寿命を延ばすために、ドライブの論理アドレス（LBA）をNANDフラッシュメモリの物理アドレスに変換する、より高度なアドレス指定方法が使用されます。外国文学のNANDフラッシュアドレス変換アルゴリズムは呼ばれていますFlash Translation Layer （FTL）。SSDコントローラーの説明（例えばMarvellから）を見ると、コントローラーに最大4つのプロセッサーコアが含まれていることがわかります。SSDのこのような高性能システムは、主にブロードキャストアドレスの計算に必要です。

FTL、論理および物理アドレス指定


以下を含むすべてのユーザー要求を満たす単一のユニバーサルFTLアルゴリズムはありません。

  • シリアルアドレスでのデータの書き込み/読み取り速度
  • ランダムアドレスでのデータの書き込み/読み取り速度
  • ドライブライフ
  • 信頼性の高いデータストレージ
  • 使用されるメモリの量とタイプ

FTLアルゴリズムは、ドライブのタイプ（SD / USB / SSD）によって、および1つのタイプによって異なります。たとえば、SSDは目的によって分離されています。サーバー用、パーソナルコンピューター（PC）内のキャッシュ用、ラップトップ用、デスクトップPC用です。さらに、アプリケーションごとに、要件と独自のFTL実装オプションに優先順位があります。各FTLオプションは、特定のドライブアプリケーションの優先順位を考慮した、パラメーターの妥協です。

FTLの本質は、デバイスの論理アドレスを物理的なNANDフラッシュアドレスに変換することです。


各論理LBAには、NAND内のメモリ領域が割り当てられています。これは、論理ユニット番号テーブル（LUNテーブル）と呼ばれます。論理ブロックを書き換えるには、空き物理ブロックのデータを消去してから、LUNテーブルで置き換えます。これにより、ユーザーは書き換えのように見えます。LUNテーブルのサイズは、論理ブロックのサイズによって異なります（すべてのデバイスに大量のメモリを搭載することはお勧めしません）。FTL実装には、ブロックアドレッシングとページアドレッシングという2つの根本的に異なるアプローチがあります。

ブロックのアドレス指定


ブロックアドレス指定では、論理ブロックのサイズは物理ブロックのサイズに対応します。次に、論理ブロックのアドレスに等しいインデックスを持つLUNテーブルアレイでは、物理ブロックのアドレスに対応する値が示されます。ブロックの一部を変更するには、ブロック全体を書き換える必要があります。

ブロックアドレス指定の利点は、LUNテーブルのサイズが小さいことです。これは、USBフラッシュドライブやmicroSDメモリカードなど、RAMが少ないデバイスでは重要です。欠点は、ブロックサイズが非常に大きく（約メガバイト）、少量のデータ（たとえば、512バイト）を上書きするには、ブロック全体を書き換える必要があることです。

ページのアドレス指定


ページアドレス指定では、物理ページのアドレスがLUNテーブルに格納されます。論理ユニットのサイズは約数十キロバイトです。

ページングの利点は、シーケンシャルにもランダムにも高速にデータを上書きできることです。欠点は、LUNテーブルのサイズが大きいことです。したがって、ほとんどのSSDには100 MBを超える容量のメモリチップが含まれています。

実際には、主に両方のアドレス指定オプションを組み合わせたハイブリッドアルゴリズムが使用されました。たとえば、コンパクトドライブ（USBフラッシュドライブ、SDメモリカード）では、ボリュームの大部分はブロック方式でアドレス指定されますが、ボリュームの頻繁にアクセスされる部分はページごとにアドレス指定されます。


これは、NANDフラッシュドライブの構造と動作原理です。FTLアドレッシング方法について多数の科学記事が書かれており、多くのソリューションが特許を取得しています。コントローラーメーカーは、製品が市場に投入された後でも、ソフトウェアの改善に常に取り組んでいます。一般に、ソフトウェアはコントローラーの不可欠な部分であり、その説明は別の記事に値します。
