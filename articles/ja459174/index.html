Subject-ソースとブラウザの両方になることができるクラス。これにより、たとえば、外側にオブザーバブル、内側にオブザーバーとして通知するさまざまな種類のコントローラーで使用できます。次に、このクラスのさまざまな実装について説明します。

AsyncSubject / AsyncProcessorストリームが正しく完了するまで最後のイベントを保持し、それをサブスクライバーに渡します。エラーが発生した場合、イベントは転送されません。



PublishSubject / PublishProcessorは、端末シグナルが到着するまで、そこに着信するイベントをさらに転送します。ストリームまたはエラーの終了後、適切なイベントを返します。



BehaviorSubject / BehaviorProcessorは PublishSubject / PublishProcessorと同様に機能しますが、サブスクリプション時に最後のイベントがあればそれを返し、Subjectが最終状態に渡されていない場合にそれを返します。



ReplaySubject / ReplayProcessor-BehaviourSubject /ステロイドのBehaviorProcessor。最後の1つのイベントではなく、魂が望むだけのイベントを返します。完了したReplaySubjectまたはReplayProcessorをサブスクライブすると、蓄積されたすべてのデータが受信されます。


したがって、ReplaySubject.createWithSize（1）とBehaviourSubject.create（）は、最終状態に移行した後、動作が異なります。サブスクリプション中、最初のイベントは最後のイベントを返し、2番目のイベントは返しません。ReplayProcessorについても同様です。
CompletableSubject、MaybeSubjectとSingleSubject作品同様PublishSubjectに、唯一たぶんシングル、それぞれ、Completableで使用するために設計されています。

UnicastSubject / UnicastProcessorは、実際には1つのサブスクライバーだけが存在することを保証するReplaySubjectです。再サブスクライブしようとすると、IllegalStateExceptionがスローされます。
