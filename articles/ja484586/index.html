PHPでIPv6を使用する

最近、LIRステータスと29 IPv6ブロックを受け取りました。そして、指定されたサブネットを追跡する必要がありました。私たちの請求はPHPで書かれていたので、この問題に少し触発されて、この言語がIPv6での作業に関して最も友好的ではないことを理解しなければなりませんでした。アンダーカット-アドレスと範囲の操作に関する問題の解決策。おそらく最もエレガントではありませんが、タスクを実行します。



理論のビット


免責事項 IPv6とは何か、何を食べるのかをよく知っているとしたら、この部分は退屈かもしれません。そうではないかもしれません。

IPv6アノテーションを最初に目にする人は落胆するかもしれません。エレガントな64.233.177.101の後、突然2607：f8b0：4002：c08 :: 8bに遭遇し、混乱する可能性があります。それとそれ以外の両方-それぞれ人間が読める32ビットと128ビットのみの表現。すべてのIPパケットには、厳密に標準化された各ビットの割り当てを持つヘッダーが含まれています。ヘッダーの構造をこれ以上深く掘り下げることなく、ここから1つのことを得る必要があります。IPアドレスと範囲を使用した演算では、一般的にバイナリ演算とビットごとの演算を使用すると便利です。また、IPv4の場合はBINARY（4）、IPv6の場合はBINARY（16）としてデータベースに格納することも最も便利です。

考慮すべきもう1つの重要な側面は、ネットワークマスクとCIDR表記です。 CIDR-Classless Inter-Domain Routing（Classless Inter-Domain Routing）の頭字語。この概念は、IPアドレスのどの部分がネットワークプレフィックスであり、どの部分がこのネットワーク内のネットワークインターフェイスアドレスであるかを決定する問題のクラス1を置き換えました。実際には、プレフィックスに対応する最初のnビットは1に設定され、残りは0に設定されます

。人間が読める形式では、これはip.add.re.ss. / cidrとして記述されます。たとえば、64.233.177.0 / 24は、最初の24ビットがプレフィックスを参照することを意味します。最後の8ビットは、人間が読み取れるエントリの最後の数字であり、サブネット内のアドレスを参照します。さらにいくつかの演習。64.233.177.101/32および2607：f8b0：4002：c08 :: 8b / 128-1つの特定のアドレス。2607：f8b0：4002：c08 :: / 64-最初の64ビット（最初の4つのグループ）-プレフィックス、残りの64ビット-ローカル部分。ちなみに、エントリの「::」に困惑している人がいる場合、二重コロンは、0を含む任意の数のセクションを置き換えます。これは、注釈内で1回だけ発生します。つまり、2607：f8b0：4002：c08 :: 8b = 2607：f8b0：4002：c08：0：0：0：8bです。

これらすべてから何を学ぶ必要がありますか？まず、最初と最後のサブネットアドレスは、バイナリ形式のマスクを知っているバイナリANDおよびORを使用して取得できます。次に、次のサイズのサブネット（CIDRを使用）nは、nに 1を追加することで計算できます。バイナリ表現の-番目の位置。バイナリビューとは、関数pack（）とinet_pton（）を使用した結果と、ビットごとの演算子をさらに使用した結果を、バイナリで示したものです。これは、たとえばbase_convert（）を使用して取得できるバイナリシステムの表現です。

練習に移りましょう


実際には、私には思われるように、最も可能性の高い3つのタスクを実装します。

  1. 範囲の最初と最後のアドレスを取得します。
  2. 与えられたサイズ（CIDR）の次の範囲を取得します。
  3. アドレスが範囲に属していることを確認します。

実装はIPv6用ですが、必要に応じて、ロジックを簡単に適合させることができます。私はここからいくつかのアイデアを得ましたが、実装方法が少し異なります。また、例では入力エラーのチェックはありません。じゃ、行こう。

すでに述べたように、範囲の最初と最後のアドレスは、範囲の始まりとバイナリサブネットマスクを知っているビット単位の演算を使用して決定できます。したがって、最初に行う必要があるのは、CIDRをバイナリマスクに変換することです。これを行うには、16進数表現を収集してバイナリにパックします。

function cidrToMask ($cidr) {
    $mask = str_repeat('f', ceil($cidr / 4));
    $mask .= dechex(4 * ($cidr % 4));
    $mask = str_pad($mask, 32, '0');
    return pack('H*', $mask);
}

コールパック（「H *」、$マスク）と同じように16進数表現パックはinet_pton（）を。唯一の違いは、人間が読めるエントリとは対照的に、pack（）が呼び出されるとき、すべて0が配置されている必要があり、エントリにコロンが含まれていてはならないことです。

次のステップは、範囲の開始と終了を計算することです。そしてここにニュアンスがあります。ビット演算は、プロセッサの容量によって制限されます。したがって、32ビットのCubieTruckは、時々テストの甘やかしに使用しますが、アドレスの128ビットすべてを1回の操作で処理することはできません。ただし、32ビットのグループに分割することを妨げるものは何もありません（万が一、実行するプロセッサを知っている人がいるかもしれません）。

function getRangeBoundary ($ip, $cidr, $which, $ipIsBin = false, $returnBin = false) {
    $mask = cidrToMask($cidr);
    if (!$ipIsBin) {
        $ip = inet_pton($ip);
    }
    $ipParts   = str_split($ip, 4);
    $maskParts = str_split($mask, 4);
    $rangeParts  = [];
    for ($i = 0; $i < count($ipParts); $i++) {
        if ($which == 'start') {
            /*  &       . */
            $rangeParts[$i] = $ipParts[$i] & $maskParts[$i];
        } else {
            /*  |    (~)           1. */
            $rangeParts[$i] = $ipParts[$i] | ~$maskParts[$i];
        }
    }
    $rangeBoundary = implode($rangeParts);
    if ($returnBin) {
        return $rangeBoundary;
    } else {
        return inet_ntop($rangeBoundary);
    }
}

今後の使用のために、IPを送信し、バイナリと人間が読み取り可能な形式の両方で結果を取得する機能を提供します。ここの$ whichパラメータは、範囲の最初と最後のどちらを取得するかを設定します（値はそれぞれ「開始」または「終了」です）。

次のタスク（当社にとって最も実用的なもの以外）は、次の範囲を計算することです。このタスクでは、アドレスをバイナリ文字列に分解し、目的の位置に1を追加し、すべてを折りたたむ方法を除いて、何も良いことは思い浮かびませんでした。アーティファクトがどこにも表示されないようにするため、分解とアセンブリの際にアドレスをバイト単位で分割することにしました。

function getNextBlock ($ipStart, $cidr, $ipIsBin = false, $returnBin = false) {
    if (!$ipIsBin) {
        $ipStart = inet_pton($ipStart);
    }
    $ipParts = str_split($ipStart, 1);
    $ipBin   = '';
    foreach ($ipParts as $ipPart) {
        $ipBin .= str_pad(base_convert(unpack('H*', $ipPart)[1], 16, 2), 8, '0', STR_PAD_LEFT);
    }
    /*  1       "" :) */
    $i = $cidr - 1;
    while ($i >= 0) {
        if ($ipBin[$i] == '0') {
            $ipBin[$i] = '1';
            break;
        } else {
            $ipBin[$i] = '0';
        }
        $i--;
    }
    $ipBinParts = str_split($ipBin, 8);
    foreach ($ipBinParts as $key => $ipBinPart) {
        $ipParts[$key] = pack('H*', str_pad(base_convert($ipBinPart, 2, 16), 2, '0', STR_PAD_LEFT));
    }
    $nextIp = implode($ipParts);
    if ($returnBin) {
        return $nextIp;
    } else {
        return inet_ntop($nextIp);
    }
}

出力では、$ cidrで指定された次のサイズ範囲のプレフィックスを取得します。この機能により、お客様に住所ブロックを割り当てます。

最後に、アドレスが範囲に属しているかどうかを確認します。たとえば、1ブロック/ 48をクライアントにブロック/ 64を配布するために割り当て、割り当て時に、割り当てられたブロックの境界を超えないようにする必要があります（実際にはこれはすぐに発生しますが、まだ可能性があります）。ここではすべてが簡単です。範囲の開始と終了をバイナリ形式で取得し、アドレスが範囲内かどうかを確認します。

function ipInRange ($ip, $rangeStart, $cidr) {
    $start = getRangeBoundary($rangeStart, $cidr, 'start',false, true);
    $end = getRangeBoundary($rangeStart, $cidr, 'end',false, true);
    $ipBin = inet_pton($ip);
    return ($ipBin >= $start && $ipBin <= $end);
}
