AWS Landing ZoneのTerraformモジュールの紹介


数か月のハードワークの後、AWS Landing ZoneのTerraformモジュールを紹介できることを嬉しく思います。ソースコードはGitHubに保存され、安定リリースバージョンはTerraform Module Registryで公開されています。

開始するには、main.tfコードに次のコードを含めます。

module "landing_zone" {
  source     = "TerraHubCorp/landing-zone/aws"
  version    = "0.0.6"
  root_path  = "${path.module}"
  account_id = "${var.account_id}"
  region     = "${var.region}"
  landing_zone_components = "${var.landing_zone_components}"
}

注：variables.tfから必要になる可能性があるものをすべて含めてくださいoutputs.tf。

理解を容易にするために、デフォルト値をに追加しましたterraform.tfvars：

account_id = "123456789012"
region = "us-east-1"
landing_zone_components = {
  landing_zone_pipeline_s3_bucket = "s3://terraform-aws-landing-zone/mycompany/landing_zone_pipeline_s3_bucket/default.tfvars"
  [...]
}

つまり、このモジュールterraformを使用する場合、次のものが必要になります。

  1. AWS組織のデータに対応する値account_idをregion自分で変更します。
  2. landing_zone_componentsAWS Landing Zoneの使用に合わせて値を変更します。
  3. 変更s3://terraform-aws-landing-zone/mycompanyあなたのS3のブロックとS3キープレフィックス、ファイル保存します.tfvars（またはファイルへの絶対パス.tfvars、ローカルストレージ内に）。

このモジュールには、数十、数百、数千の展開可能なコンポーネントを含めることができますが、それらのすべてが展開される必要はなく、今後も展開されます。実行時に、変数マップの一部ではないコンポーネントはlanding_zone_components無視されます。
