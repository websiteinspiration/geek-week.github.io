この記事をマスターするには、次のことを知る必要があります。ビットとは何か、バイナリシステム、負の次数の知識レベルでの算術。この記事は、プロセッサレベルでの実装のエンジニアリングの詳細、および正規化された数値と非正規化された数値には影響しません。数値を2進数に、またはその逆に変換することにさらに重点が置かれ、浮動小数点数が一般にビットの形式で格納される方法が説明されています。

浮動小数点数は、正しく使用するために必要な非常に強力なツールです。それらは整数レジスターほど簡単ではありませんが、それらが十分にゆっくりとそれらに侵入される場合、それらは複雑ではありません。

今日の記事では、例として32ビットレジスタを使用します。倍精度数（64ビット）はまったく同じロジックで機能します。

まず、浮動小数点数がどのように格納されるかについて話しましょう。最も古い31ビットが重要です。単一とは、数値が負であり、ゼロがそれぞれ反対であることを意味します。次は指数の8ビットです。これらの8ビットは通常の符号なし数値です。そして最後には仮数の23ビットがあります。便宜上、符号をS、指数をE、仮数を奇妙なことにM

と表記します。一般式 仮数の場合、1つ暗黙単位ビットを考慮することが通例です。つまり、仮数は24ビットになりますが、最上位の23番目のビットは常に1であるため、それを書き留めることはできません。したがって、この「制限」により、任意の数を表す独自性が得られます。 仮数は通常の2進数ですが、整数とは異なり、最上位ビットは2 ^ 0度で、次第に次数が下がります。出品者が重宝するところです。その値に応じて、上位ビット2の累乗が増減します。それがこのアイデアの天才です。 これを良い例で示してみましょう：（-1）s×M×2E-127

#include <stdio.h>

union IntFloat {
    unsigned int integerValue;
    float floatValue;
};

void printBits(unsigned int x) {
    int i;
    for (i = 31; i >= 0; i--) {
        if ((x & ((unsigned int)1 << i)) != 0) {
            printf("1");
        }
        else {
            printf("0");
        }
        if (i == 31) {
            printf(" ");
        }
        if (i == 23) {
            printf(" ");
        }
    }
    printf("\n");
}

int main() {
    union IntFloat b0;
    b0.floatValue = 59.578125;

    printBits(b0.integerValue);

    b0.integerValue = 0b01000010011011100101000000000000;

    printf("%f\n", b0.floatValue);

    return 0;

}

グリッドステップは、隣接する2つの浮動小数点数の最小差です。そのような数のビットのシーケンスを通常の整数として表す場合、隣接する浮動小数点数のビットは単位あたりの整数として異なります。

違う表現ができます。 2つの隣接する浮動小数点数は2 ^（E-127-23）異なります。つまり、最下位ビットの値に等しい差。

証明として、コードのmainを変更して、再度コンパイルできます。
