QEMU-KVMの操作の一般原則



私の現在の理解：

1）KVM


KVM（カーネルベースの仮想マシン）-ハイパーバイザー（VMM-仮想マシンマネージャー）。LinuxOSでモジュールとして動作します。ハイパーバイザーは、一部のソフトウェアを存在しない（仮想）環境で実行すると同時に、このソフトウェアから、このソフトウェアが動作する実際の物理ハードウェアを隠すために必要です。ハイパーバイザーは、物理ハードウェア（ホスト）と仮想OS（ゲスト）の間の「ストリップ」として機能します。

KVMはLinuxカーネルの標準モジュールであるため、カーネルから必要なすべてのnishtyaki（メモリー、スケジューラーなどで作業）を受け取ります。そしてそれに応じて、最終的には、これらすべての利点がゲストにもたらされます（ゲストはLinuxカーネル上で実行されるハイパーバイザーで動作するため）。

KVMは非常に高速ですが、それだけでは仮想OSを実行するには不十分です。これにはI / Oエミュレーションが必要です。I / O（プロセッサ、ドライブ、ネットワーク、ビデオ、PCI、USB、シリアルポートなど）の場合、KVMはQEMUを使用します。

2）QEMU


QEMU（Quick Emulator）-あるアーキテクチャ用に設計されたオペレーティングシステムを別のアーキテクチャで実行できるようにするさまざまなデバイスのエミュレーター（たとえば、ARM-> x86）。 QEMUは、プロセッサに加えて、ネットワークカード、HDD、ビデオカード、PCI、USBなどのさまざまな周辺デバイスをエミュレートします。

これは次のように機能します。

命令/バイナリコード（たとえば、ARM）は、TCG（Tiny Code Generator）コンバーターを使用して、プラットフォームに依存しない中間コードに変換されます。次に、このプラットフォームに依存しないバイナリコードは、ターゲットの命令/コード（x86など）にすでに変換されています。

ARM->ミドルウェア-> x86

実際、Intel VT-x（Intel Virtualization Technology）/ AMD SVM（AMD Secure Virtual Machine）をサポートしていない古いプロセッサモデルでも、任意のホストのQEMUで仮想マシンを実行できます。ただし、この場合、バイナリはTCGを使用して2回オンザフライで再コンパイルする必要があるため（TCGはジャストインタイムコンパイラーです）、動作が非常に遅くなります。

それら。QEMU自体は非常に優れていますが、動作が非常に遅いです。

3）保護リング




プロセッサのバイナリプログラムコードはそのように機能しませんが、最も特権の高い（Ring 0）から最も制限された、過度に規制された「きついナット付き」（Ring 3）まで、データへのさまざまなアクセスレベルを持つさまざまなレベル（保護リング）に配置されます。 ）

オペレーティングシステム（OSカーネル）はリング0（カーネルモード）で実行され、任意のデータとデバイスで何でも実行できます。ユーザーアプリケーションはリング3（ユーザーモード）レベルで動作し、必要な操作を実行する権限はありませんが、代わりに操作を実行するために毎回アクセスを要求する必要があります（したがって、ユーザーアプリケーションは自分のデータにしかアクセスできず、 「他の誰かのサンドボックスに「入る」」）。リング1と2は、ドライバーが使用するためのものです。

Intel VT-x / AMD SVMが発明される前は、ハイパーバイザーはリング0で動作し、ゲストはリング1で動作していました。リング1にはOSが正常に機能するための十分な権限がないため、ゲストシステムからのすべての特権呼び出しで、ハイパーバイザーはこの呼び出しをオンザフライで変更する必要がありましたリング0で実行します（QEMUのようなものです）。それら。ゲストバイナリコードはしないプロセッサ上で直接実行され、その都度その場でいくつかの中間の修正がありました。

オーバーヘッドが大きく、これは大きな問題でした。その後、プロセッサの製造元は、互いに独立して、ゲストOSコードを直接実行できる拡張命令セット（Intel VT-x / AMD SVM）をリリースしましたホストプロセッサ上（以前のように、あらゆる種類のコストのかかる中間ステップをバイパスします）。

Intel VT-x / AMD SVMの登場により、特別な新しいリング-1レベルが作成されました（マイナス1）。そして今ハイパーバイザーはそれに取り組んでおり、ゲストはリング0に取り組んでおり、CPUへの特権アクセスを取得しています。

それら。最終的に：

  • ホストはリング0で実行されます
  • ゲストはリング0で働く
  • ハイパーバイザーはリング-1で実行されます

4）QEMU-KVM


KVMはゲストにリング0へのアクセスを提供し、QEMUを使用してI / O（プロセッサー、ディスク、ネットワーク、ビデオ、PCI、USB、シリアルポートなど、ゲストが「見て」操作する）をエミュレートします。

したがって、QEMU-KVM（またはKVM-QEMU）:) 注目を集める
