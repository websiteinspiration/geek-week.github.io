Sorgen Sie für Ihre Hausaufgaben: Sichern Sie Ihren Laptop oder Desktop mit den kostenlosen Veeam-Agenten

Im normalen Leben arbeite ich, wie viele von Ihnen, liebe Leser, die meiste Zeit im Büro, und es passt perfekt zu mir. Aber heute ist die Situation so, dass sowohl ich als auch meine Kollegen in verschiedenen Teilen der Welt und ich denke, die Mehrheit der Leute, die diesen Artikel lesen, alle von zu Hause aus arbeiten.

Und hier ist es ziemlich unangebracht, sich an die Notwendigkeit zu erinnern, einen Heimcomputer oder Laptop zu sichern. Wenn sich die Arbeitsumgebung ändert, ändert sich auch die übliche Art, mit Daten zu arbeiten. Bei der Arbeit im Büro ist alles offensichtlich: direkter Zugriff auf die erforderlichen Ressourcen, zentralisierte Backups. Wenn überhaupt, können Sie sich jederzeit an den Helpdesk wenden. Bei der Arbeit von zu Hause aus müssen Sie viele Dinge für sich selbst bereitstellen, einschließlich Backup.

Ich glaube, dass viele von Ihnen bereits Veeam Agent für Microsoft Windows oder Veeam Agent für Linux verwenden. Wenn noch nicht, dann interessieren Sie sich vielleicht für den heutigen kurzen Rückblick. Also willkommen bei cat.



Als 2014 die erste Version unseres Produkts, das jetzt als Veeam Agent für Microsoft Windows bekannt ist , veröffentlicht wurde, hieß es Veeam Endpoint Backup Free und hatte die einzige nichtkommerzielle Edition.

Natürlich hat sich die Lösung seitdem stark verändert, und es wurde eine kommerzielle Version herausgebracht, in der es viele erweiterte Funktionen gibt - zentrales Management, CBT-Treiber, Cluster-Unterstützung, parallele Festplattenverarbeitung und vieles mehr. Veeam Agent für Linux wird ebenfalls entwickelt, mit dem Sie physische Maschinen sichern und Daten aus Sicherungen wiederherstellen können. Aber jetzt, im Jahr 2020, die neuen Versionen von V eeam Agent für Microsoft Windows und Veeam Agent für LinuxEs gibt auch eine kostenlose Edition für einzelne Computer.

Wenn Sie also Ihren Heimdesktop, Laptop oder sogar ein Tablet unter Windows sichern müssen, verwenden Sie Ihre Gesundheit.

Für Veeam Agent für Linux ist die Geschichte ähnlich:

  1. Ein Snapshot der erforderlichen Volumes wird erstellt - dies erfolgt mit dem speziellen Veeamsnap-Modul. (Ein Snapshot wird auf einem Volume mit ausreichend freiem Speicherplatz erstellt.)

    Wenn ein Volume gesichert wird, kopiert Veeam Agent die Partitionstabelle und den Bootloader an den Zielspeicherort.
  2. Veeam Agent für Linux liest Daten vom Volume, führt die Komprimierung durch und kopiert sie an den Zielspeicherort.
  3. Nachdem alle erforderlichen Daten kopiert wurden, wird der Schnappschuss gelöscht. Die Daten werden in einer Sicherungsdatei gespeichert.

Veeam Agent für Microsoft Windows: Unterstütztes Betriebssystem


64-Bit- und 32-Bit-Betriebssystemversionen (sofern zutreffend) werden unterstützt:

  • Microsoft Windows Server 2019
  • Microsoft Windows Server 2016
  • Halbjährlicher Microsoft Windows Server-Kanal (einschließlich 1909)
  • Microsoft Windows Server 2012 R2
  • Microsoft Windows Server 2012
  • Microsoft Windows Server 2008 R2 SP1
  • Microsoft Windows 10 (von Version 1607 bis 1909)
  • Microsoft Windows 8.1
  • Microsoft Windows 7 SP1

Anmerkungen:

  • Für Server-Betriebssysteme werden die Editionen Small Business Server und Server Essentials unterstützt.
  • Server Core-Versionen für Serversysteme in einem eigenständigen Bereitstellungsszenario werden nicht unterstützt.
  • Unterstützte Windows Embedded / Windows IoT-Betriebssysteme (außer benutzerdefinierte Builds).
  • Microsoft Failover-Cluster werden unterstützt, wenn Veeam Agent für Microsoft Windows Veeam Backup & Replication ausführt (werden in eigenständigen Bereitstellungsszenarien nicht unterstützt).
  • Microsoft Windows 10 Education wird ab Build 10586 unterstützt.

Die übrigen Systemanforderungen sind in der Dokumentation (in englischer Sprache) ausführlich beschrieben .

Sicherungsjob in Veeam Agent für Microsoft Windows


Konfigurieren Sie für die Sicherung die Sicherungsaufgabe, indem Sie den Assistenten für neue Sicherungsjobs starten . Wählen Sie darin die Art der Sicherung aus:

  • Gesamter Computer - die ganze Maschine.
  • Sicherung auf Volume-Ebene - Sicherung ausgewählter Volumes.
  • Datei - Level - Backup - Sicherung bestimmter Dateien.

Veeam Agent für Microsoft Windows kann verschiedene Arten von Speichersystemen als Sicherungsspeicherort verwenden:

  • Lokaler Speicher - externes Laufwerk (USB, Firewire oder eSata). Wie Sie wissen, besteht die Hauptsache hier nicht darin, sich auf der integrierten Festplatte des Computers zu sichern.
  • Freigegebener Ordner - SMB-Freigabe auf einem Dateiserver oder NAS.
  • Veeam-Backup-Repository - Veeam-Repository (lokal).
  • Das Veeam Cloud Connect-Repository ist ein Cloud-basiertes Repository, das vom Veeam-Dienstanbieter bereitgestellt wird.
  • Microsoft OneDrive

In meinem Beispiel wird die Sicherung auf das USB-Laufwerk übertragen. Wählen Sie dazu im Schritt Ziel die Option Lokaler Speicher :



Anschließend legen Sie die Einstellungen für den ausgewählten Speichertyp fest.

Wichtig! Wenn Sie mit austauschbaren USB-Laufwerken arbeiten, sollten Sie berücksichtigen, dass ein solches Laufwerk, das für FAT32 formatiert ist, das Speichern von Dateien mit mehr als 4 GB nicht unterstützt. Es wird daher nicht empfohlen, solche Festplatten als Zielsicherungsspeicher zu verwenden.

Konfigurieren Sie als Nächstes den Zeitplan im Schritt Zeitplan . Hier können Sie angeben, ob zu bestimmten Zeiten regelmäßig gesichert werden soll oder ob die ausgewählten Ereignisse auftreten sollen:

  • Sperren - wenn Sie das Auto gesperrt haben
  • Abmelden - wenn für das aktuelle Konto abgemeldet
  • Wenn das Sicherungsziel verbunden ist - wenn der Zielsicherungsspeicher verbunden ist

Wenn die dritte Option angegeben ist, können Sie Veeam Agent auch anweisen, das Trennen (Auswerfen) des Zielspeichers am Ende der Sicherung abzuschließen, damit die Sicherung offline ist. Dies verringert das Risiko, einer Sicherungskopie des Verschlüsselungsvirus ausgesetzt zu sein, wenn einer erkannt wird.

Veeam Agent für Linux


Veeam Agent für Linux wurde entwickelt, um die Daten von Computern unter Linux zu schützen. Diese Lösung verfügt außerdem über eine kostenlose Edition. Der Linux-Kernel 2.6.32 und höher wird unterstützt.

Unterstützt werden 64-Bit- und 32-Bit-Versionen (sofern zutreffend) von Distributionen:

  • Debian 8.0 - 10.3
  • Ubuntu 14.04, 16.04, 18.04, 19.10
  • RHEL 6.0 - 8.1
  • CentOS 6.0 - 8.1
  • Oracle Linux 6 - 8.1 (RHCK)
  • Oracle Linux 6 (beginnend mit UEK R1) - Oracle Linux 7 (bis UEK R5 U2)
  • SLES 11 SP4, 12 SP1 - 15 SP1
  • SLES für SAP 11 SP4, 12 SP1 - 15 SP1
  • Fedora 30 - 31
  • openSUSE-Sprung 15.1
  • openSUSE Tumbleweed

Anmerkungen:

  • Fedora wird bis einschließlich Kernel Version 5.4.14 unterstützt.
  • openSUSE Tumbleweed wird bis einschließlich Kernel Version 5.1.7 unterstützt.
  • Der Linux-Kernel 2.6.32-754.6.3 wird unter CentOS / RHEL und Oracle Linux (RHCK) nicht unterstützt.

Weitere Informationen finden Sie in der Dokumentation (in englischer Sprache).

Es werden dieselben Arten von Sicherungen unterstützt wie in Veeam Agent für Microsoft Windows, einschließlich Sicherungen auf Dateiebene. Die Sicherungseinstellungen können mithilfe der Schritte im Assistenten oder über die Befehlszeile festgelegt werden.

Weitere Informationen zu den einzelnen Lösungen finden Sie im Benutzerhandbuch für Windows (auf Englisch) oder Linux (auch auf Englisch).

Über die Einschränkungen der kostenlosen Edition


Für Veeam Agent für Linux (im Vergleich zur kommerziellen Version von Server):

  1. Es kann nur 1 Sicherungsjob erstellt werden.
  2. Das Veeam Cloud Connect-Repository wird nicht als Zielsicherungsrepository unterstützt.
  3. Die Skripte vor dem Einfrieren und nach dem Auftauen werden für den Sicherungsjob nicht unterstützt.
  4. Backup unter Berücksichtigung der Arbeit des DBMS Oracle, MySQL, PostgreSQL wird nicht unterstützt.

Weitere Details finden Sie in der Tabelle hier . Lesen Sie den Artikel von Sergey Shtepa, einem Entwickler in einem Blog über Habré, darüber, was „hinter den Kulissen“ passiert, wie unsere Ingenieure diesen „Zoo“ unterstützen und wie kompatibel die Veeam-Lösung ist .

Die Einschränkungen für Agent für Windows sind in derselben Tabelle ausführlich aufgeführt . Von den wichtigsten stellen wir Folgendes fest:

  1. Anwendungsbasierte Sicherungen werden nicht unterstützt.
  2. Das Veeam Cloud Connect-Repository wird nicht als Zielsicherungsrepository unterstützt.
  3. Microsoft OneDrive wird nicht als Sicherungsspeicher für Computer mit Windows Server-Betriebssystem unterstützt.
  4. Microsoft OneDrive kann nur für Sicherungen des Systemstatus verwendet werden. (Weitere Informationen zu Sicherungstypen finden Sie hier .)
