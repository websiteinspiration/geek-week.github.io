同様に、ルーティング、コントローラー、データベースロジックを追加して、「従業員」APIエンドポイントへのHTTP GETリクエストを処理します。

ルーティングロジックの追加


Express には、適切なコントローラーロジックへのHTTPリクエストのルーティングを簡素化するルータークラスが付属しています。ルートパスは APIのURLのエンドポイントを定義し、含んでいてもよいルートパラメータを URLに捕捉値こと。

アプリケーションのルートを決定するには多くの方法があります。たとえば、アプリケーションの起動時に、コントローラーディレクトリ内のすべてのファイルを読み取り、それらが提供するファイル名やプロパティなどの事前定義されたルールに基づいてルーティングロジックを自動的に生成できます。または、ファイルを構成ディレクトリに追加して、起動時に読み取ることもできます。

このアプリケーションでは、わずかに低いレベルのアプローチを使用して、新しいルーターモジュールを通るルートをプログラムで定義します。servicesディレクトリにrouter.jsという新しいファイルを作成します。次のコードをファイルに追加し、変更を保存します。

const express = require('express');
const router = new express.Router();
const employees = require('../controllers/employees.js');
 
router.route('/employees/:id?')
  .get(employees.get);
 
module.exports = router;

ルーターモジュールはExpressモジュールを要求することから始まり、次にRouter Expressクラスの新しいインスタンスを作成します。ルーターモジュールのルートメソッドは、送信されたデータに基づいてルートを決定するために使用されます。パスには、idという名前のオプションの（疑問符による）パラメーターが含まれています。ルーターから返されるルートには、HTTPメソッドと一致するメソッドがあり、ハンドラーを定義できます。この場合、getメソッドを使用して、受信GET要求を従業員コントローラー（以下で作成されます）で定義されたget関数にマップします。

現時点ではルーターがありますが、アプリケーションでは使用されていません。それを使用するには、services / web-server.jsファイルを開きますそして、データベースモジュールが必要な最上部の行を削除します（前の投稿でのテストにのみ使用されていました）。その場所に次のコード行を追加します。

// *** line that requires ../config/web-server.js is here ***
const router = require('./router.js');

次に、次のコードを使用してapp.getハンドラ全体を置き換えます。app.getハンドラは、データベースモジュールを使用してGETリクエストに応答します（7行すべて）。

// *** line that adds morgan to app here ***
 
    // Mount the router at /api so all its routes start with /api
    app.use('/api', router);

これで、ルーターはWebサービスモジュールで要求され、/ apiに「マウント」されます。これは、従業員エンドポイントの完全なURLがhttp：// server：port / api / employee /：idになることを意味します。

コントローラロジックの追加


コントローラーロジックは、エンドポイントURLとHTTPメソッドが判明した瞬間から動作します。 WebサーバーはExpressを使用して構築されているため、コントローラーロジックは、要求オブジェクトと応答オブジェクト、および次の関数にアクセスできる特別なミドルウェアまたは関数を使用して決定されます。

ミドルウェア機能は、要求オブジェクトからの入力を使用して、応答オブジェクトに送信される応答を生成します。次の関数は、通常、パイプライン内の次のミドルウェア関数を呼び出すために使用されます。ただし、このAPIでは、コントローラーロジックがパイプラインの最後のステップになり、HTTP応答を完了します。次の関数は、標準のExpressエラーハンドラーに制御を渡すエラーが発生した場合にのみ呼び出されます。controllers

ディレクトリに移動し、employee.jsという名前の新しいファイルを作成します。次のコードをコピーしてファイルに貼り付け、変更を保存します。

1行目：従業員データベースAPI（以下で作成）。
3〜23行目： getという非同期関数が宣言されています。関数本体でtry-catchブロックを使用して、メインスレッドでスローされた例外をキャッチし、次の関数に渡します。
5〜7行目：名前付きコンテキストを持つ定数が宣言されています-これは、データベースAPI検索メソッドに関連するプロパティを含むユニバーサルオブジェクトです。 idプロパティは、req.params.idから取得した値に基づいてコンテキストに追加されます。
9行目： findメソッドを使用して、データベース内の適切な従業員レコードを取得します。
11-19行目：条件付きロジックを使用して、正しいHTTPステータスコードと応答本文を決定します。1人の従業員が要求されたが見つからなかった場合、エラーコード「404 Not Found」が応答として送信されます。それ以外の場合は、200 OKコードがJSONベースの応答本文とともに送信されます。
25行目：他のモジュールに追加できるようにするためのモジュール

のエクスポートreq.paramsオブジェクトは、着信要求オブジェクトからデータを取得するために使用されるいくつかのプロパティの1つにすぎません。その他の一般的なプロパティには、URL内のクエリ文字列値のreq.query、リクエスト本文のreq.body、req.cookiesなどがあります。HTTPヘッダーは、req.getメソッドを使用して取得できます。

データベースロジックの追加


、従業員データベースモジュールを起動するに移動するにはdb_apisのディレクトリと呼ばれる新しいファイルを作成employee.jsを。次のコードをファイルに追加します。

const database = require('../services/database.js');
 
const baseQuery = 
 `select employee_id "id",
    first_name "first_name",
    last_name "last_name",
    email "email",
    phone_number "phone_number",
    hire_date "hire_date",
    job_id "job_id",
    salary "salary",
    commission_pct "commission_pct",
    manager_id "manager_id",
    department_id "department_id"
  from employees`;
 
async function find(context) {
  let query = baseQuery;
  const binds = {};
 
  if (context.id) {
    binds.employee_id = context.id;
 
    query += `\nwhere employee_id = :employee_id`;
  }
 
  const result = await database.simpleExecute(query, binds);
 
  return result.rows;
}
 
module.exports.find = find;

従業員データベースモジュールは、一般的なデータベースモジュールを導入し、従業員テーブルのSQLクエリ用にbaseQueryという名前の定数を初期化します。二重引用符で囲まれた列のエイリアスは、返されるキーの大文字と小文字を制御するために使用されます。

次に、findという関数が宣言され、クエリを実行して抽出された行を返すために使用されます。渡されたコンテキストパラメータがtrueに設定されている場合、where句がリクエストに追加されるため、1人の従業員のみが返されます。

context.id値はリクエストに直接追加されなかったことに注意してください。代わりに、employee_idという名前のプレースホルダーが使用されました-これはバインド変数と呼ばれます。 Oracleデータベースバインディング変数の使用は、セキュリティとパフォーマンスの観点から非常に重要です。バインド変数の値はバインドオブジェクトに割り当てられ、クエリと一緒にdatabase.simpleExecuteに渡されます。最後に、データベースから取得した行が呼び出し元に返されます。

アプリケーションを起動し、ブラウザーでhttp：// localhost：3000 / api / employeeに移動します。次のように従業員のリストが表示されます（いくつか折りたたみました）。

たとえば、URLの末尾に識別子を追加することで1人の従業員を選択できます。例：http：// localhost：3000 / api / employee /100。
