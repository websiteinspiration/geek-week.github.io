自動機械学習


データ処理と機械学習の分野でのタスクは、システムの複雑さに起因して発生する多くの要因に関連し、それらのソリューションを複雑にします。これらには（Charles Suttonによると）：

  • 不確実性と不確実性の存在。これは、データと必要な依存関係のアプリオリな知識の欠如につながります。したがって、研究要素は常に存在します。
  • 「千カットからの死。」実際には、データ処理と分析、およびその後のモデリングのためのパイプラインを構築する場合、大小さまざまな決定を行う必要があります。たとえば、データを正規化する必要がある場合、その場合、どのメソッドを使用し、このメソッドにはどのパラメーターを指定する必要がありますか？等。
  • 不確実性に起因するフィードバックループの存在。タスクとデータへの没入が長くなるほど、それらについて学ぶことができます。これにより、一歩下がって既存の処理および分析メカニズムに変更を加える必要が生じます。
  • さらに、機械学習アルゴリズムによって取得されたモデルの結果は、現実の近似にすぎません。明らかに正確ではありません。



したがって、データ処理および分析の完全なパイプラインを取得するプロセスは、複雑なシステム（つまり、複雑なシステム）と見なすことができます。

一方で、これらの要因の存在は、マシンとディープラーニングの問題の解決とその自動化の両方を複雑にします。一方、増え続けるアクセスしやすいコンピューティング機能により、タスクにより多くのリソースを割り当てることができます。

共通のCRISP-DM標準によれば、データ分析に関連するプロジェクトのライフサイクルは、ビジネスタスクの理解（ビジネス理解）、データの理解と調査（データ理解）、データの処理（データ準備）、モデリング（モデリング）、品質評価（評価）、実用化（展開、応用）。実際には、これらのステップのすべてが今日効果的に自動化できるわけではありません。

ほとんどの作品または既存のライブラリ（h2o、auto-sklearn、autokeras）は、モデリングの自動化と部分的に品質評価に焦点を当てています。ただし、データ処理の自動化へのアプローチの拡大により、より多くのステージをカバーできます（たとえば、Google AutoMLサービスで適用されました）。

問題の定式化


教師による機械学習のタスクはさまざまな方法で解決できますが、そのほとんどは損失関数を最小化するために削減されます J または尤度関数を最大化する L、パラメータの推定値を取得するために θ^m 利用可能なサンプルに基づく-トレーニングデータセット yt：
θ^m=argmaxθm(L(yt;θm)) または θ^m=argminθm(J(yt;θm))どこ θm-トレーニングされたモデルパラメータ（たとえば、回帰の場合の係数）。

自動化をモデリングのみに限定しないために、メソッドのスコープをパイプラインの他のステージに拡張することが可能です。たとえば、適用するデータ処理方法、モデルまたはその組み合わせの選択、最適なハイパーパラメータに近いものの選択に関する意思決定を自動化します。いくつかのハイパーパラメーターの選択を含む、

2つのデータ処理方法（標準のスケーラーと分位数のスケーラー）と2つのモデル（ランダムフォレストとニューラルネットワーク）のどちらかを選択できるフレームワークで、簡単な例で説明する内容を示します。選択構造はツリーとして表すことができます：



行われた各選択はシステムのパラメーターであり、ツリー自体が可能なパラメーターのスペースになります。問題をこのように見ることで、抽象化のレベルをさらに高め、関数を最小化または最大化するプロセスとして、データ処理メソッド、モデル、およびそれらのパラメーターを含む最終的なパイプラインを取得するタスクを定式化できます。

ω^=argmaxω(L(yt,ycv;ω)) または ω^=argminω(J(yt,ycv;ω))どこ ω -非学習パラメータ、 ycv-コントロール選択の遅延（交差検証用のデータセット）。

このような学習自動化の主な利点は次のとおりです。

  • 単一の最適化プロセスのフレームワーク内で1つの入力ポイントが存在する場合の、より多くのシステムパラメーターの選択。
  • 「千カット」から研究者や開発者を救う自動化ルーチン。
  • 自動化による機械学習の「民主化」。これにより、多くの非専門家が多くの方法を適用できます。

ただし、自動化には欠点がないわけではありません。

  • パラメータの数が増えると、それらのスペースも大きくなり、遅かれ早かれ組み合わせの爆発につながります。これには、アルゴリズムの開発とコンピューティングリソースの数の増加が必要です。
  • 完全に自動化された方法では、「ブラックボックス」の原則に基づく柔軟なソリューションが常に提供されるわけではなく、結果の制御が低下します。
  • パラメータ空間ωは非線形であり、最適化プロセスを複雑にする複雑な構造を持っています。

内部プロジェクトでの作業の過程で、私たちはハイブリッド機能宣言型構成システムに基づいて半自動機械学習の問題を解決できるツールを作成しました。この構成アプローチでは、標準のデータ型だけでなく、機械学習やディープラーニング用の一般的な最新ライブラリの関数も使用します。このツールを使用すると、簡単なデータ処理メソッドの作成、機能の基本的な構築（機能エンジニアリング）、モデルとそのハイパーパラメーターの選択を自動化し、SparkまたはGPUクラスターで計算を実行することもできます。リストは、記事の前半で示した例を形式化しています。この例では、パラメーターの分散と最適化のために、sk-learnとhyperopt（オープンソースコードにわずかな貢献をすることさえできた）の単純なモデルを使用しています。

'preprocessing': {
        'scaler': hp.choice('scaler', [
            {
            'func': RobustScaler,
            'params': {
                'quantile_range': (10, 90)
            }},
            {
            'func': StandardScaler,
            'params': {
                'with_mean': True
            }}
        ]),
},

'model': hp.choice('model', [
    {
        'func': RandomForestClassifier,
        'params': {
            'max_depth': hp.choice('r_max_depth', [2, 5, 10]),
            'n_estimators': hp.choice('r_n_estimators', [5, 10, 50])

    },
    {
        'func': MLPClassifier,

            'learning_rate_init': hp.choice('learning_rate_init', [0.1, 0.01]) 
        }
    },
])


構成メカニズムを含むこのような半自動システムにより、たとえば、特定のモデルファミリーが問題の解決により適している場合に、事前に準備された標準シナリオを作成できます。これらには、特に、クレジットスコアリングが含まれる場合がありますが、そのようなステップには、同様の幅広いタスクに関する追加の調査が必要です。また、検索メカニズムに取り組んでいるとき、トレーニングサンプルと交差検証サンプルの両方で最適化された関数の値を同時に考慮することにより、バイアスと分散のトレードオフジレンマのバランスを自動的に維持することが可能です。

結論


サイクル内の1つのハイパーパラメーターの値を列挙することさえ、すでに自動化への第一歩であるため、実際に自動化が完全に欠如していることは非常にまれです。同時に、パイプライン建設プロセス全体を完全に自動化することも、今日ではほとんど不可能です。したがって、最新のプロジェクトの開発では、自動化アプローチが意識的にまたは無意識に適用されます。

半自動機械学習を使用すると、ルーチンを自動化することで、研究の柔軟性を大幅に損なうことなく、研究者や開発者のリソースをより効率的に使用できます。ご覧のように、提案されたソリューションには人の参加が必要であり、可能なシステムパラメータのスペースが制限されています。さらに、構成システムに基づいて取得された標準シナリオの導入により、部分的な自動化アプローチだけでなく、人間の関与を必要としない完全なアプローチも使用できます。
