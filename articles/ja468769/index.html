Xavier Noria on Rails 6、コンサルティングなど

今週の土曜日には、このテクノロジーに特化したメインカンファレンス-RubyRussia 2019ですべてのルービストと会います。 9月28日は非常に間近です。このインタビューでは、Xavier Noriaへの質問が Evrone開発者Pavel Argentovによって作成され、MSTUでコース「インターネットプログラミング言語」に取り組んでいる生徒と教師が彼を助けました。


前回のシャビエルのロシア訪問の珍しい写真-RailsClub Moscow 2012で

RoRの世界で最近起こったことの中で最も興味深く、重要なものは何だと思いますか？

Rails 6は印象的なリリースだったと思います。新しいコアコンポーネント、複数のデータベースのサポート、並行テスト。有名なオートレーキに「歴史を送る」新しいブートローダー。

あなたはRailsコンサルティングで生計を立てています。これはプログラマーとして何をもたらしましたか？

ほとんどの場合、それは私に通常の仕事では提供できない経験と意識を与えました。コンサルティングが充実します。会社やチームはそれぞれ異なります。ビジネス、文化、ワークフロー、アーキテクチャ、機能、好みのさまざまな分野に出会います。モバイル環境への適応、戦わなければならないタスクは、問題を解決するために鋭く磨かれたスキルを保持します。

あなたの経験に基づいて、前の質問を続けて、Rails開発者は何を覚えておくべきですか？

開発者は最初にRubyとRailsをできるだけ深く理解する必要があります。これは、バラバラなブログ投稿ではなく、本やドキュメントを読むことを意味します。マスターは彼の楽器を所有している必要があります。たとえば、アクティブレコードがプロセスの触媒であり、ブラックボックスではないように、SQLを十分に理解する必要があります。 Active Recordを使用すると、手書きのSQLを簡単かつ安全に生成し、結果のデータセットを簡単に処理できます。 HTTPは、アプリケーションが外部の世界（WebSocketの記憶を含む）と話すため、適切な理解が重要です。他にも覚えておくべきことがたくさんありますが、特にパフォーマンスに注目します。ベンチマークを推進する方法を知る必要があります。MRIのGVLなどについてすべて知っておく必要があります。このトピックに関するリソースを探し、それらを研究し、十分な情報に基づいた直感を養います。

Rubyは本当にプログラマーの心を良くする言語なのか？これを最大限に活用するには？

私はそのような一般化に個人的に懐疑的です。 Rubyは私を興奮させ、ScalaはScalaプログラマーを興奮させます。私は数年間Perlをやっていて、彼をとても愛していますが、多くは彼を好きではありません。さまざまな脳はさまざまな形式で喜びを見つけます。自分の好みも時間とともに変化する可能性があります。プログラマーとして、できるだけ多くの言語と可能な限り多くのパラダイムを使用するほうがよいでしょう。私の心を壊した言語はプロローグでした。私に最も影響を与えたパラダイムは、Knuthスタイルの「文学プログラミング」でした。これらの完全に異なるプログラミング方法を実行することは、あなたをより良くします。 Rubyは特に柔軟で高レベルです。
高レベルは、「タスクに近い」と主張するのに役立ちます。これは私を超高級言語（VHLL）に引き付けます。私は教育によって数学者であり、彼らが植え付けるあらゆる形式主義を扱うことができます。プログラミング言語は子供のおもちゃです。たとえば、チューリングの完全性は、実際の等価性とは一致しません。実際の問題を解決するために、VHLLより優れたものを思いつくことは困難です。人工的な複雑さが減り、問題の実際の解決策に集中できるため、本当に強力で生産的な感じがします。本当の問題を解決することが、プログラミングのキャリアを追求するきっかけになりました。一般的に言って、「Cが必要な場所-Cを選択」。

カンファレンスは「on Rails」よりも「Ruby」になっています。 Rails以外に、またはwebdev以外にも、どのRubyアプリケーションがありますか？

メモリが適切に機能する場合、最初に使用したRubyプログラムはFreeBSD Port / Package Managerでした。それはRailsより前の2000年代初頭でした。 Web開発に直接関係しない人気のRubyソフトウェアは、Vagrant、Metasploit、Jekyll、Chef、Puppet、またはHomebrewです。さらに、Rails企業のシステムがWebインターフェースのみで構成されることはほとんどありません。通常、ソフトウェアのエコシステム全体が存在します。 Rubyをメイン言語とする会社には、通常、純粋なRubyバックエンドソフトウェアも（おそらく他のテクノロジーと組み合わせて）あります。

たとえば、現在スチュアートデリバリーと相談している会社で、Sidekiqワーカーによる処理のためにKafkaからRedisへのメッセージを受け入れるサービスを実装しました。また、重要なデータ移行プロジェクトと非常に興味深いビジネスエミュレータにも取り組みました。すべてが純粋なRubyで記述されています。会社のサーバー側のフロントエンドはRailsですが、舞台裏とプラットフォーム全体で、はるかに多くの異なるプログラムと言語があります。これは私の実践ではよくあることです。

ネタバレの時間-RubyRussia 2019でのあなたのレポートはどうですか？

Rubyプロジェクトのオートロード、リブート、アクティブブートを実装する新しいRuby gem、Zeitwerkについて説明します。Rails 6はデフォルトでこれらの機能をZeitwerkに委任します。

Rubyの世界で一番印象に残っていることは何ですか？たぶん、コード、テクノロジー、知識、態度？

これは主観的な印象ですが、Rubyの世界は驚くほど革新的だったと思います。これについては先ほどテレンスリーと話しました。手始めに、RailsはWeb開発における真の革命を表すという発言に同意するのは難しいと思います。 Heroku、New Relic、Shopify、GitHubなどの企業は革命的でした。しかし、私の経験では、主に特定のコミュニティに引き寄せられる人々は、近所のコミュニティで起こっていることに部分的に盲目です。イノベーションは、Scala、Haskell、Brainfuckなど、どのコミュニティでも見つかるはずです。したがって、各コミュニティをグローバルに観察できれば、Rubyコミュニティは、いずれにしても革新的であるため、それほど革新的ではないことがわかります。

あなたの意見では、Railsには標準のCMSコンポーネントライブラリのようなものが必要ですか？標準のCMSパーツを構築するための足場が好きですか？

私の意見では、これはRailsにとって少し外れたトピックです。この質問を使用して、足場は会議で簡単にデモンストレーションを実行したり、アプリケーションを生成して何かを確認したり、エラーを再現したり、特定の機能がどのように機能するかを確認したりするための手頃な方法です。 「戦闘中」の足場は見たことがなく、実際の使用には適していません。

最新のソリューションは一般に異種混合です。MLにはJuliaを、重いビジネスワークロードにはJavaを使用できます。 Rubyは言語インタラクションのためのツールを十分に提供していますか？これを簡単にするツールはありますか？

JRubyはJavaとネイティブに通信します。 MRIのC拡張機能を記述したり、FFIを使用してCライブラリをポータブルな方法でラップしたり、Helixを使用してRustで拡張機能を記述したりできます。一般的に言って、ファイルシステム、ソケット、HTTP、データベース、gRPC、メッセージブローカーを使用して、標準的な方法でプロセスと通信する方が簡単です。

多くの人が気づいているように、Rubyはもはや「新しくて光沢のある」ものではありません。今ではより「古き良き言語」となっています。一方、学生は若いです。 Rubyの学習への興味を動機づける方法は？

2000年代、私は自由時間の7年間、バルセロナ大学の学生にPerlを教えました。私のコースの前に、彼らは主にCとJavaを扱っていました。私はスクリプト言語について話し、Perlを例として使用しました。とても面白かった。私たちはプログラミングを教え、これらの言語で何が利用できるかを強調し、特定の選択肢を掘り​​下げることができる魅力的なプログラムを必要としています。 Rubyの場合、これは単純なOOP実装から素晴らしいメタプログラミング機能までです。たとえば、クラス本体に1行のコードがなくても、Active Recordモデルがどのように機能するかを示すことができます。

数十のプログラミング言語から選択できます。 Rubyを選択する理由、PythonやJavaScriptを選択する理由、Clojure、Rust、C ++、Goはどうですか？すべてのオプションは物議を醸している、またはその逆です-これらはすべて素晴らしいオプションです。いずれかを選択し、コースを思い出深いものにし、学生にプログラミング、ベストプラクティス、イディオムの概念を学習させ、ドキュメントを確認する方法を学び、形式主義を超える重要な関連問題を調査させます。これは彼らを成長させ、職業に根本的に備えるためのものです。私の意見では、これは大学の役割です。

同じもののための多くの異なるツールがあります：Active Recordを備えたRailsコア、Dry-rbなど。代わりのものをつかむべきですか？彼らはRailsを「パス」ではなく「パス」にしませんか？

それは主に個人の好みに依存します。あなたは間違いなくあなたに合ったツール、レイヤー、パターンの組み合わせを研究して理解するべきです。 「Railsパス」は「アプリケーション全体をデフォルトのフォルダにドロップする」ことを意味しないことに注意してください。これはよくある誤解です。デコレーターが欲しいですか？サービスオブジェクト、PORO？いいね！これは、いわゆる「The Rails Way」に反するものではありません。 Railsは一連のデフォルトの抽象化とコードの編成を提供します。私たちの意見では、これは少なくとも出発点として、多くのアプリケーションにとって十分なものです。そして、コントローラのアクションを単一のファイルとして記述して、すべての作業をサービスオブジェクトにアンロードする場合は、そうすることができます。一部はこれを好まないかもしれません。異なる人々-異なる脳。
