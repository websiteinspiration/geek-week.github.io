抜粋。第6章タプル


プログラミングでタプルなどの概念に遭遇したことがないかもしれませんが、これはSwiftで利用できる非常に便利な機能の1つです。たとえば、タプルを使用して座標を操作できます。そうですね、座標軸ごとに個別の変数を作成するよりも、1つの変数で書かれた構造（x、y、z）を使う方がはるかに便利です。

6.1。タプルの基本


タプルは、単一の複合値内のさまざまなタイプの値をグループ化するオブジェクトです。

タプルは、Swiftが任意の型の値を組み合わせる最も簡単な方法です。タプル内の個々の値には独自のデータ型があり、他のデータ型にはまったく依存していません。

Motorcadeリテラル


タプルは、タプルリテラルを使用して作成できます。

構文

(_1, _2, ..., _N)

•value：Any-タプルに含まれる任意のデータ型の次の値。

タプルリテラルは、タプルとして定義された値を返します。リテラルは括弧で囲まれ、コンマで区切られた一連の独立した値で構成されます。要素の数は任意ですが、7つを超える数を使用することはお勧めしません。

タプルは、基本的なデータ型の値と同じ方法で変数と定数に格納されます。
構文

let  = (_1, _2, ..., _N)
var  = (_1, _2, ..., _N)

変数と定数を宣言し、N個の要素で構成されるリテラルタプルを値として初期化します。タプルを変数に書き込むには、varステートメントを使用し、定数に書き込むには、letステートメントを使用する必要があります。
次の例を考えてみましょう。定数を宣言し、Int、String、およびBoolの3つの型要素で構成されるタプルで初期化します（リスト6.1）。

リスト6.1

let myProgramStatus = (200, "In Work", true)
myProgramStatus // (.0 200, .1 "In Work", .2 true)

この例では、myProgramStatusは、特定のプログラムのステータスを表す値としてタプルを含む定数であり、3つの要素で構成されています。

  • 200-Int型の整数、プログラムステータスコード。
  • 「作業中」-タイプStringの文字列リテラル、プログラムの状態のテキストによる説明。
  • true-タイプBoolの論理値。プログラムの操作を続行する機能。

このタプルは、単一の初期化された定数内の3つの異なるデータ型の値をグループ化します。

タプルのデータ型


しかし、タプルがさまざまなデータタイプの値を1つにグループ化する場合、タプル自体とその値を格納するパラメーターはどのタイプのデータですか？

タプルデータ型は、タプル要素のデータ型名の固定され、順序付けられたシーケンスです。
構文

(____1, ____2, ...,
____N)

データ型は括弧で囲まれ、要素型名はコンマで区切られます。タイプ名の順序は、タプル内の要素の順序に対応している必要があります。

例


(Int, Int, Double)

リスト6.1のmyProgramStatusタプルのデータ型は（Int、String、Bool）です。データ型は、タプルの要素に基づいて自動的に決定されるため、暗黙的に設定されます。データ型を指定する順序はタプル内の要素の順序と一致する必要があるため、型（Bool、String、Int）は（Int、String、Bool）とは異なります。

リスト6.2は、さまざまなタプルのデータ型を比較しています。

リスト6.2

var tuple1 = (200, "In Work", true)
var tuple2 = (true, "On Work", 200)
print( type(of:tuple1) == type(of:tuple2) )

コンソール

false

タプルのデータタイプを比較するために、渡されるオブジェクトのタイプを決定するグローバル関数タイプ（of :)によって返される値が使用されます。
myProgramStatusタプルで、最初の要素が整数ではなくFloat値であると仮定します。この場合、タプルのデータ型（パラメーター名の後のコロン）を明示的に判別できます（リスト6.3）。

リスト6.3

//      
let floatStatus: (Float, String, Bool) = (200.2, "In Work", true)
floatStatus // (.0 200.2, .1 “In Work”, .2 true)

タプル要素の特定の数に制限されません。タプルには必要な数の要素を含めることができます（ただし、7つを超える要素を使用することはお勧めしません）。リスト6.4は、同じタイプの4つの要素のタプルを作成する例を示しています。禁止されていないIntデータ型のエイリアスを使用します。

リスト6.4

//     Int
typealias numberType = Int
//      
let numbersTuple: (Int, Int, numberType, numberType) = (0, 1, 2, 3)
numbersTuple // (.0 0, .1 1, .2 2, .3 3)

6.2。タプル要素との相互作用


タプルは、特定の値のセットを設定および保存するだけでなく、これらの値と対話することも目的としています。このセクションでは、タプル要素の値を操作する方法について説明します。

パラメータへの値の初期化


1つの式で新しいパラメーターを宣言し、それらのタプルのすべての要素の値を初期化できます。これを行うには、括弧とコンマでvarキーワード（または定数を宣言する場合はlet）の後に、新しいパラメーターの名前を指定し、初期化演算子の後にタプルを渡す必要があります。宣言されたパラメーターの数は、タプル要素の数と一致する必要があることに注意してください（リスト6.5）。

リスト6.5

//     
var (statusCode, statusText, statusConnect) = myProgramStatus
//  
print("  — \(statusCode)")
print("  — \(statusText)")
print("   — \(statusConnect)")

コンソール：

  — 200
  — In Work
   — true

この構文を使用すると、任意の値を一度に複数のパラメーターに簡単に初期化できます。これを行うには、式の右側で、初期化演算子の後に、タプルを含むパラメーターではなく、タプルリテラルを渡す必要があります（リスト6.6）。

リスト6.6

/*     
   */
var (myName, myAge) = ("", 140)
//   
print("  \(myName),   \(myAge) ")

コンソール：

  ,   140

変数myNameおよびmyAgeは、タプル要素（Troll、140）の対応する値で初期化されます。この構文を使用すると、任意のタプル要素を無視できます。これを行うには、無視する要素に対応する変数の名前としてアンダースコア文字を指定する必要があります（リスト6.7）。

リスト6.7

/*   
  */
var (statusCode, _, _) = myProgramStatus

その結果、タプルの最初の要素であるmyProgramStatusの値がstatusCode変数に書き込まれます。他の値は無視されます。

注意

Swiftの下線は、パラメーターを無視することを示します。これが使用できる唯一の例ではありません。将来、本の素材を勉強するとき、彼と何度も会うことになるでしょう。

インデックスを介してタプル要素にアクセスする


タプルの各要素には、値に加えて、この要素へのアクセスに使用できる整数インデックスが含まれています。インデックスは常にゼロから始まり、順番になります。したがって、N要素のタプルでは、​​最初の要素のインデックスは0になり、最後の要素にはインデックスN-1を使用してアクセスできます。

単一の要素にアクセスする場合、インデックスは、タプルが格納されているパラメーターの名前の後にドットで示されます。リスト6.8は、タプルの個々の要素にアクセスする例を示しています。

リスト6.8

//     
print("   — \(myProgramStatus.0)")
print("   — \(myProgramStatus.1)")
print("    — \(myProgramStatus.2)")

コンソール：

  — 200
  — In Work
   — true

名前を通じてタプル要素にアクセスする


インデックスに加えて、タプルの各要素に一意の名前を割り当てることができます。要素名は必須ではなく、タプルを使用するためにのみ使用されます。名前は、全員と要素の一部の両方に付けることができます。要素の名前は、タプルのリテラルと、その型を明示的に定義することの両方で定義できます。

リスト6.9は、リテラルを介してタプル要素の名前を判別する例を示しています。

リスト6.9

let statusTuple = (statusCode: 200, statusText: "In Work",
statusConnect: true)

指定された要素名を使用して、これらの要素の値を取得できます。この場合、インデックスがパラメータ名の後のポイントを介して指定された場合、インデックスを介してアクセスする場合と同じ構文が使用されます。名前を定義しても、インデックスを使用できます。タプル内のインデックスは常に使用できます。

リスト6.10は、要素名がインデックスと組み合わせてどのように使用されるかを示しています。

リスト6.10

//     
print("   — \(statusTuple.statusCode)")
print("   — \(statusTuple.statusText)")
print("    — \(statusTuple.2)")

コンソール：

  — 200
  — In Work
   — true

名前を使用した要素へのアクセスは、インデックスを介したアクセスよりも便利で視覚的です。
前述のように、要素名はタプルリテラルだけでなく、データ型を明示的に決定するときにも指定できます。リスト6.11は対応する例を示しています。

リスト6.11

/*   
  
   */
let anotherStatusTuple: (statusCode: Int, statusText: String,
statusConnect: Bool) = (200, "In Work", true)
//   
anotherStatusTuple.statusCode // 200

タプル編集


同じタイプのタプルの場合、あるタプルの値を別のタプルに初期化する操作を実行できます。リスト6.12の例を検討してください。

リスト6.12

var myFirstTuple: (Int, String) = (0, "0")
var mySecondTuple = (100, "")
//      
myFirstTuple = mySecondTuple
myFirstTuple // (.0 100, .1 "")

タプルmyFirstTupleとmySecondTupleは同じデータ型であるため、1つの値を別の値に初期化できます。最初のタイプは明示的に設定され、2番目のタイプは初期化された値を介して設定されます。
インデックスと名前を使用して、タプル内の個々の要素の値を変更できます（リスト6.13）。

リスト6.13

//  
var someTuple = (200, true)
//    
someTuple.0 = 404
someTuple.1 = false
someTuple // (.0 404, .1 false)

タプルはSwiftで非常に普及しています。たとえば、彼らの助けを借りて、関数から1つではなくいくつかの値を簡単に返すことができます（少し後で関数に慣れます）。

注意


タプルは複雑なデータ構造の作成を許可しません。それらの唯一の目的は、異種または類似のパラメーターの特定のセットをグループ化し、それらを目的の場所に転送することです。複雑な構造を作成するには、オブジェクト指向プログラミング（OOP）のツール、より具体的にはクラスまたは構造を使用する必要があります。私たちは後でそれらを知るようになります。
