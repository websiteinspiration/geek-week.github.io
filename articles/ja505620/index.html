HackTheBox。ウォークスルーネスト。NTFSストリーム、逆C＃およびSMBウォーカー


私は、HackTheBoxサイトからさらに処理するために送信されたソリューションを引き続き公開しています。

この記事では、SMBリソースを長時間さまよって、代替のNTFSストリームを見つけて、C＃でアプリケーションを逆にします。

実験室への接続はVPN経由です。情報セキュリティの分野で何かを知っている人々とプライベートネットワークに行き着くため、仕事用のコンピュータや、重要なデータが利用可能なホストから接続しないことをお勧めします。

偵察


このマシンのIPアドレスは10.10.10.178で、/ etc / hostsに追加しています。

10.10.10.178    nest.htb

まず、開いているポートをスキャンします。nmapですべてのポートをスキャンするには時間がかかるため、最初にmasscanでこれを実行します。tun0インターフェースからすべてのTCPおよびUDPポートを毎秒500パケットの速度でスキャンします。

masscan -e tun0 -p1-65535,U:1-65535 10.10.10.178  --rate=500



ここで、ポートで動作するサービスの詳細については、-Aオプションを指定してスキャンを実行します。

nmap -A nest.htb -p445,4386



まず、SMBリソースに注目しましょう。ゲストとしてログインしてみましょう。


サウナ

そして利用可能なリソースを発見してください。私たちが利用できるすべてのコンテンツを再帰的に見てみましょう。



上記のファイルのうち、最も興味深いのは「Welcome Email.txt」です。リソースに接続し、このファイルをローカルマシンに保存します。



そしてメッセージ自体には、それに接続するためのリソースと資格情報が報告されます。



リソースのすべてのコンテンツを表示してみましょう。ただし、すでに見つかった資格情報を使用しています。



この場合、リソース、サービス、または資格情報の名前を見つけることができるため、構成ファイルに常に関心があるはずです。したがって、再びリソースに接続し、すべての構成を取得します。



1つのファイルでプライベートリソースの名前「Secure $」を見つけ、もう1つのファイルでユーザーの資格情報を見つけます。





しかし、パスワードは適合しません。明らかに暗号化されています。しかし、判明したのは既存の資格情報を持つリソースです。



ただし、内部には1つのディレクトリがあるため、ITディレクトリの内容を表示することはできません（構成から追跡されます）。 \ Secure $ \ IT \ Carl \の内容全体を確認します。



VisualBasicプロジェクトはそこにあります。エクスプローラを介して接続し、プロジェクトフォルダをローカルマシンにコピーしました。



プロジェクトを開いて、パスワードが構成から読み取られ、復号化される場所を見つけます。



暗号化の原理自体をざっと見た後、あまり登りたくなかったので、プロジェクトをVisual Studioに読み込み、コードを変更して、デバッグ時にパスワードを見つけます。





したがって、ユーザーの実際のパスワードがわかります。接続してトークンを取得します。



ルート


ユーザーの作業ディレクトリをさらに調べます。



構成ファイルから、2番目のポートで何が機能するかがわかります。



しかし、接続してヘルプを見ると、パスワードが必要であることがわかります。



ディレクトリに移動してアプリケーションをダウンロードします。おそらくそれを分析することで、パスワードを見つけることができます。



このアプリケーションがC＃で記述されているのは幸いです。これにより、dnSpyを使用してプロジェクト全体を簡単に逆コンパイルできます。





したがって、プログラムは、プログラムが入力として構成ファイルを受け取り、そこから特定のパラメーターを取得します。



しかし、プロジェクトには何も見つかりません。さらに、認識が空のファイル「Debug Mode Password.txt」を覗き込むまでに多くの時間が費やされました。ネットワークリソースを接続します。



次に、このファイルのすべてのストリームを見てみましょう。また、パスワードストリームには15文字が格納されていると判断します。



私たちはそれを読んで、デバッグモードのパスワードを見つけます（鈍い...それが起こります）。



再度接続し、このパスワードを入力してヘルプを表示します。





その後、逆コンパイルされたプロジェクトのなんらかの設定を見つける必要があるため、アプリケーションを少し散策する価値がありました。



アプリケーションの名前から判断して、LDAPに移動しましょう。



そして再び、すでに持っているアプリケーションと設定ファイルを見つけます。



これらは、アプリケーションで変換されるパラメーターです。復号化関数を見つけ、それを返す前にコンソールに値を描画します。



さて、ファイルのチェック時にアプリケーションが停止しないように、条件の1つを変更します。



プロジェクトを再コンパイルしてアプリケーションを実行し、構成を引数として指定します。



そして、管理者パスワードを取得します。次に、psexecを使用して正常に接続します。



マシンを完全に制御できます。Telegramで

参加できます。興味深い資料、統合されたコース、ソフトウェアを見つけることができます。ITの多くの分野に精通している人々が集まるコミュニティをまとめましょう。そうすれば、ITと情報セキュリティの問題について常に互いに助け合うことができます。
