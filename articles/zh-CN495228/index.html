WebStorm 2020.1：界面改进，Vuex支持以及保存文件时启动Prettier

大家好！我们很高兴介绍今年的第一个主要WebStorm更新。在新版本中，您会发现许多新功能和期待已久的改进，包括Vuex支持以及保存文件时启动Prettier的新选项。在线或使用工具箱应用程序



下载WebStorm 2020.1的30天试用版。完整版可以由WebStorm或所有产品包的有效订阅的持有者以及开源项目的免费学生和开发人员使用。 现在，让我们更详细地看一下主要改进。



用于处理代码的新字体


在过去的一年中，我们开发了一种特殊的字体以用于处理代码，这将帮助我们的用户编程时非常舒适，并最大程度地减少了眼睛疲劳。我们努力的结果是开发了一种新的开源字体JetBrains Mono。从版本2020.1开始，在WebStorm和我们的其他IDE中默认选择JetBrains Mono。当然，保留其他字体的能力也保持不变。



更有用的快速文档。


WebStorm 2020.1可帮助您快速找到有关符号的相关信息，因为现在将鼠标悬停在该符号上（而不是仅按F1时）会显示快速文档。如果您的代码中有问题，WebStorm可以提供解决方案的帮助，那么它还将通过在文档弹出窗口中显示可用的快速修复来使您知道它。



使用JavaScript和TypeScript代码时，快速文档将显示有关字符的类型和可见性以及定义该字符的位置的详细信息。



禅意模式专注工作


新的Zen模式可帮助您完全专注于代码。结合两种模式：无干扰和全屏，您可以快速打开这两种模式，并暂时隔离可以分散注意力的内容。



要尝试新模式，请转到查看| 外观| 从WebStorm主菜单进入Zen模式，或使用“ 切换”弹出窗口（Ctrl +`|视图模式|进入Zen模式）。

支持Vuex和Vue Composition API


Vue.js越来越受欢迎，因此我们正在积极努力使WebStorm成为使用此框架的最有用的IDE。这次，我们添加了两个主要改进。

首先，在使用Vuex库时，您将在编辑Vue组件时看到Vue商店符号和模块的代码完成选项。WebStorm还可以帮助您快速定义吸气剂，突变和动作。



其次，由于您已经可以使用即将推出的Vue 3版本中提供的Composition API，因此我们决定在WebStorm 2020.1中添加对此功能的支持。



保存文件时运行更漂亮


借助 新的“在文件上保存时运行”选项，您可以将Prettier格式应用于WebStorm设置中指定并在当前项目中编辑的所有文件，同时保存这些文件-您不再需要配置文件监视程序或使用第三方插件。



帮助语法和文体错误


从版本2020.1开始，WebStorm与Grazie捆绑在一起，后者是我们用于检查拼写，语法和文本样式的工具。这将帮助您避免在添加注释，提交和各种语言构造时出现语法错误。



默认情况下，仅对英语启用Grazie，并且不会检查所有文件类型是否存在语法错误。您可以添加更多语言并在首选项/设置|更改预定义设置。编辑器 校对（如果要配置扫描范围，请转到“ 语法”部分，请参阅现有规则并添加例外）。

新的意图行动和检查


与往常一样，我们添加了一些新的意图操作和快速修复，以帮助您节省使用JavaScript和TypeScript代码的时间。例如，通过按Alt + Enter可以使用的一种新的意图动作将迅速将现有代码转换为可选的链接和/或无效合并。



有关新的意图操作和快速修复的更多信息，请参见发行博客文章（英文）。

支持流行技术的最新版本


在WebStorm 2020.1中，您将找到对TypeScript 3.8功能的内置支持，包括纯类型的导入/导出，私有字段和顶级await，这将帮助您更有效地使用它们。此外，通过完全支持新的元数据格式，我们使Angular 9项目的工作变得更加容易。

重新提交更方便


Git窗口的“ 日志”选项卡上的“ 从此处交互式地变基” 操作使您可以编辑，合并和删除以前的提交。这有助于使提交历史更加线性和易于理解。 在WebStorm 2020.1中，您将找到一个改进的，更具交互性的“ 重新提交基准”对话框，该对话框在调用“ 从此处进行交互式基准”操作时打开。更新的对话框显示可以应用于每次提交的操作。它还显示有关每个提交的详细信息，并允许您查看差异并快速放弃已应用的更改。





使用WebStorm快速编辑文件


借助 新的LightEdit模式，您可以在单独的文本编辑器窗口中打开文件，而无需创建或加载整个项目。如果WebStorm尚未运行，则该模式将起作用。如果WebStorm正在运行，则文件将照常在其中打开。

要尝试使用新模式，请右键单击要编辑的文件，然后从建议的程序列表中选择“ WebStorm”。或者，您也可以按此处所述配置命令行启动器，并使用命令行打开文件。
