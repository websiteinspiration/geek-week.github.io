React on Mobx @ Quantmart / Mobx-form-validation-kitの新しい検証パッケージ

こんにちは。
私はに基づいているプロジェクトで、非同期の検証のために新しいパッケージについてお話したいと思います今日はリアクト、Mobxとで書かれた活字体。
最新のフロントエンド開発では、ページにドキュメント、アンケート、およびローンのドキュメント、支払い注文、サイトの登録ページを入力するときに、大量のロジックが含まれます。主な論理的負担は検証チェックにあります。Angular開発者はこの点を熟考し、これらの目的のために組み込みのFormControlメカニズムを使用することを開発者に提案します。これには多くの欠点がありますが、Reactにそのようなソリューションがないことよりも優れています。React開発の現在の傾向には、mobxを使用してビジネスロジックを編成することが含まれているため、状況は複雑です。
これらの問題に直面して、私たちはパッケージを書くことでそれらすべてを解決しました：@quantumart / mobx-form-validation-kit

パッケージのプラス：
  • TypeScript全体
  • Mobxと互換性があります（バージョン4、サポートしている、みんなのお気に入り、IE10）
  • Reactで動作するように設計されています（Reactなしでプロジェクトで使用できます）
  • 非同期検証用に設計されています
  • 既存のプロジェクトに簡単に組み込むことができます。

カットの下でパッケージを操作するための手順。


最初に、@ quantumart / mobx-form-validation-kitパッケージの機能について説明します。記事の最後では、サイトの登録フォームの例を含む完全に機能するページを作成します。

フォームコントロール


@quantumart / mobx-form-validation-kitを使用すると、ソースデータと表示するフォームの間にレイヤーを作成できます。これにより、それらを検証し、必要に応じて、元のオブジェクトに到達する前にデータを変更できます。@quantumart / mobx-form-validation-kit

ライブラリには、フォームを管理するための3つの主要なクラス（検証コンポーネント）が含まれています。
  • FormGroup-検証コンポーネントを組み合わせることができます。クラスは型指定されており、フィールドのリストをジェネリックパラメーターとして使用してインターフェイスを再作成できます。デフォルトではどれも登録されていますが、タイプせずに使用しないことを強くお勧めしますが、可能性があります。
  • FormControl-特定のフィールドを検証するために使用されます。最も一般的に使用されるクラスです。クラスは型指定されており、ジェネリックパラメーターとして、格納する変数の型を受け取ります。デフォルトでは、文字列は登録されています。フォームの最もプライベートなオプションとして、デフォルトは文字列です。
  • FormArray-検証コンポーネントの配列を作成および管理できます。

さらに、基本的な抽象クラスがあります
  • AbstractControl-リストされたすべての検証クラスの型ではなく、基本クラス。
  • FormAbstractControl-入力されていない、FormGroupおよびFormArrayの基本クラス。
  • FormAbstractGroup - FormControlの型付き基本クラスではなく、レンダリングされるHTML要素へのリンクが含まれます。

    RegistrationStore.tsファイルに追加の変更が加えられました。
    彼は以下の構造を獲得した。
    ユーザーに関する情報を持つメインオブジェクト（ソースオブジェクト）はuserInfoのままでしたが、これに加えてレイヤーがの形式で表示されました。検証とuserInfoオブジェクトへのデータの割り当てを担当するのはこのレイヤーです。
    import { observable } from "mobx";
import {
  FormControl,
  FormGroup,
  AbstractControls
} from "@quantumart/mobx-form-validation-kit";

interface IUserInfo extends AbstractControls {
  name: FormControl;
}

export class RegistrationStore {
  @observable
  public userInfo = {
    name: ""
  };

  @observable
  public form: FormGroup<IUserInfo>;

 public initForm(): void {
    this.form = new FormGroup<IUserInfo>({
      name: new FormControl(
        this.userInfo.name,
        [],
        v => (this.userInfo.name = v)
      )
    });
  }
}
export const registrationStore = new RegistrationStore();
