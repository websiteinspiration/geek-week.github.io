初心者マーケッターとソシニコフのための39の英語の略語

マーケティングおよび関連分野の専門家が使用するほとんどすべての用語は英語に強く結びついています。したがって、すでに職業訓練の段階で、あなたはそれらを研究する必要があります。

私たちの経験では、略語を覚えるのは最も困難です。略語に費やすエネルギーのほぼ2倍です。結局のところ、最初に復号化を学び、次に翻訳自体を学ぶ必要があります。したがって、彼らは1つの記事で最も一般的な頭字語を収集しました。

仕事の初心者のために、タスクを少し簡略化することにしました。私たちは、この辞書が初心者とジュニアのソシニコフとマーケティング担当者に役立つことを願っています。

重要！多くの略語が英語の元の形式で使用されていることはわかっています。しかし、初心者にとっての難点は、それらがすべてかなり似ていることです。そのため、誤って記憶するリスクがあります。それらを学ぶ最善の方法は、筆記録と翻訳を最初に理解し、次に意味とニュアンスを掘り下げることであると信じています。

CPC-クリック単価-クリック単価

広告をクリックするたびに支払いが考慮され、広告主のWebサイトまたはページに移行する標準的な広告モデル。

CPC =（広告費用）：（クリック数）

例。Webサイトに広告バナーを配置し、120クリックを受け取りました。あなたは広告に支払った：800ルーブル。式を適用すると、CPC = 6.6ルーブルになります。移行のため。

CPL-リードあたりのコスト-潜在的なクライアント（リード）を受け取るためのコスト

引き付けられた各リードに対して支払いが考慮される標準的な広告モデル。Leadは、広告に反応して連絡先情報を残した潜在的な顧客です。CPLは、広告キャンペーンの効果を評価する必要があるb2b企業で最もよく使用されます。

CPL =（広告費用）：（完了したリードの数）

例。サイトにバナーを配置し、3つのアプリケーションを受け取りました。あなたは広告に1200ルーブルを費やしました。式を適用すると、CPL = 400ルーブルになります。鉛のため。

CPM-マイルあたりのコスト-CPM

1000回の広告インプレッションごとに支払いがカウントされる標準的な広告モデル。このモデルを使用すると、比較的少ない予算で、できるだけ多くの人々に広告を表示できます。

CPM =（広告費用）x 1000：（広告の表示回数）

例。あなたはサイトに広告を出しており、2,000回表示されています。あなたは広告に1,500ルーブルを費やしました。式を適用すると、CPM = 750ルーブルになります。1000インプレッションあたり。

CR-コンバージョン率-コンバージョン率

1000人の訪問者あたりのコンバージョン数を示す係数。パーセンテージとして計算されます。

CR =（コンバージョン数）：（訪問者数）x 100％

例。1000人の訪問者がサイトを訪れました。このうち、45件の商品購入申込みが残り、計算式を適用するとCR = 4.5％となります。

CRM-顧客関係管理-顧客関係管理

これは、プロセス自動化の助けを借りて、バイヤーとの対話をより効率的に構築し、作業のミスを回避し、結果として彼をより多く販売するのに役立つシステムです。CRMは、各クライアントとのすべての通信の完全な記録を保持します。

ここでは、通話の聞き取り、購入履歴の確認、テンプレートによるドキュメントの作成、電子メールまたはSMSの作成、タスクの設定を行うことができます。一般に、ひどくではなく、体系的に販売したい企業にとって不可欠なツールです。

サイトのコンバージョンを増やすことを目的とした一連のアクション。つまり、マーケティング担当者とSEOの主な機能の1つです。

通常、CROには、購入インターフェースの改善、製品またはサービスのプレゼンテーションの作成、販売目標到達プロセスの分析と改善、ランディングページの最適化、A / Bテストが含まれます。

CTA-行動を促すフレーズ-行動を促すフレーズ

CTA要素は、ボタン、画像、図形、または何らかのアクションを必要とするテキストです。

無料登録または今すぐ購入-クラシックCTA。

CTR-クリック率-クリック率

コンテキスト広告の効果を評価するための最も重要な指標の1つ。結局のところ、Google AdwordsとYandex Directのクリックの価格に影響を与えるのは彼です。パーセンテージとして計算されます。

CTRが高いほど、CPCは低くなります。

CTR =（広告のクリック数）：（広告のインプレッション数）x 100％の

例。20,000人のユーザーが目にした広告を立ち上げました。800人がバナーをクリックしました。式を適用すると、CTR = 4％になります。

CX-カスタマーエクスペリエンス-カスタマーエクスペリエンス（カスタマーエクスペリエンス）

DA-ドメイン権限

Googleのリンク指標の1つ。この値は、ドメイン（サブドメイン）が検索エンジンでどのようにランク付けされるかを示します。

この予測は、対数スケールで1から100までのポイントで測定されます。つまり、10から20の範囲でパフォーマンスを向上させることは、70から80に比べてはるかに簡単です。

FOMO-行方不明の恐怖-失われた利益の恐怖（Lost Profit Syndrome）

実際、IOを受け入れると、請負業者は本格的な契約に署名します。

NPS-ネットプロモータースコア-消費者ロイヤルティインデックス

1つのことを決定しようとする複雑なスキーム：どれだけの消費者がブランドを愛するか。

その根拠は、サービス後に企業でよく尋ねられる質問です。「あなたが友人や同僚に私たちの会社に助言する可能性はどれくらいですか？」回答は通常1から10のスケールで配置されます。

次に、回答の全体が3つのグループに分類されます。批評家（1から6まで回答）、中立（7-8回答）、ブランド支持者（9-10回答）。NPSを決定する式は次のとおりです

。NPS=（（ブランド支持者の数）-（批評家の数））：（回答者の数）x 100％

たとえば、100人の調査では、そのうち20人が批評家、45人が中立、35人が-ブランド支持者。式を使用すると、NPS = 15％になります。

ただし、PBNが検索エンジンによって検出された場合、制裁が課される可能性があるため、この戦略にはリスクがあります。

ページへのリンクが多いほど、ページの重要性が増します。ページ「A」の重みは、ページ「B」によって送信されるリンクの重みに依存します。これがどのように行われるかの良い例を次に示します。



QDD-クエリは多様性に値する-内線。「リクエストにはさまざまな価値があります」

システムが不明なまたはあいまいな質問に使用するGoogleランキングアルゴリズム。つまり、ユーザーが「ペンを購入する」と書いた場合、ユーザーがどのペンを念頭に置いているのか明確ではありません：執筆、ドア、窓、ホルダー、その他。

このアルゴリズムの存在と機能は100％明確ではありません。しかし、実際には、無期限のリクエストの場合、標準のアルゴリズムとあまり整合性のない項目が問題に含まれることが示されています。

QDF-クエリは新鮮に値する-追加。「リクエストは新鮮に値する」

Googleのランキングアルゴリズム。ページの重みだけでなく、ページの公開時間も考慮されます。新鮮なコンテンツは古いものより上位にランク付けされます。

このアルゴリズムの特徴も研究されていません。SEOスペシャリストは、検索結果を分析することによってのみ、それを主張できます。検索結果では、新鮮な材料が古いものよりも上位にランク付けされることがよくあります。

ROI-投資収益率-投資収益率

特定の投資の収益性や損失を示します。ROIは、個々の広告キャンペーンを含め、ほとんどすべての投資について考慮する必要があります。
ROI =（広告に反応した顧客からの収益）：（広告費用）

例。6月の広告キャンペーンの予算は10,000ルーブルに達した。広告は、40,000ルーブルを購入した顧客からのものです。式、ROI = 4を使用し

ます。比率はパーセントで示される場合があります。つまり、上記の例では400％になります。

簡略化すると、SEM = SEO +コンテキスト広告。

SC-補足コンテンツ

これは、UX（ユーザーエクスペリエンスまたはユーザーエクスペリエンス）を向上させるコンテンツの名前です。

それはページやサイトの目的を果たさないかもしれませんが、ページの意味に直接関連しているべきです。

全体としての補助コンテンツは、訪問者のリードへのコンバージョンとして機能しない任意の要素と呼ぶことができます。しかし、多くの場合、これらはネイティブの写真やビデオ、ソーシャルネットワークから挿入された投稿、ヘッダー、ナビゲーションメニュー、コメントブロック、推奨される資料のブロックです。

はい、この略称は検索エンジンの結果ページに表示される広告を非表示にします。GoogleとYandexでは、これは通常、ページの最上部と最下部のいくつかの位置にあり、「広告」または「広告」アイコンでマークされています。

ここでは、たとえば、「タブレットを購入する」という要求に応じて4（！）ポジションのブロックが表示されます。



SEO-検索エンジン最適化-検索エンジン最適化

これは、ターゲットを絞った検索クエリに対する検索エンジンでのサイトの可視性を高めるための一連の対策です。

つまり、SEOには、検索エンジンアルゴリズムの分析、需要分析、セマンティックコアのコンパイル、外部および内部のWebサイトの最適化、行動要因の改善、結果の分析、そして簡単に説明できない車とタスクの小さなカートが含まれます。

SERM-検索エンジンの評判管理-検索エンジンの評判管理

これは、潜在的な顧客の目に組織の前向きなイメージを作成することを目的とした一連のツールです。多くの場合、これらは2つの主要なタスクです。ブランドまたは製品（サービス）のポジティブなイメージに対する直接作業と、ネットワーク上にすでに存在するネガティブの処理です。

SERMは、主にフォーラム、ソーシャルネットワーク、レビュー、画像やニュースコンテンツの投稿で機能します。

SERP-検索エンジン結果ページ-検索エンジンページ

SMM-ソーシャルメディアマーケティング

これは、ソーシャルメディアとしてマーケティングによって認識されているソーシャルネットワークで商品やサービスを宣伝するための一般的な名前です。

定義が大きすぎて簡単に説明できないので、SMMが解決するタスクに焦点を当てましょう。

  • PR
  • ブランディング
  • 貿易マーケティング
  • マーケティングイベントのソーシャルおよびインターネットサポート
  • バルク広告
  • 製品、製品、またはブランドのソーシャルプロモーション
  • 会社のリソースの出席の増加
  • 競争力のない企業やブランドとの相互プロモーション。


そして、はるかに。実際、これはソーシャルネットワークに焦点を当てた本格的なマーケティング業界です。

SMO-ソーシャルメディアの最適化-ソーシャルネットワークのリソースの最適化

これらは、ネットワークコミュニティで最も簡単に使用できるようにサイトのコンテンツを変換することを目的としたイベントです。

ほとんどの場合、次の手順はSMOの一部として実行されます。
  • コンテンツは、定期的なソーシャルメディアの訪問者の期待と好みに沿ったものです。
  • サイトは、ソーシャルネットワークの技術的なメカニズム（出版物などのコメントを残す機能）に準拠するように変換されます。
  • リソースの使いやすさが向上します。
  • コンテンツを簡単にエクスポートするためのチャネルを作成します。そのため、あらゆる出版物をソーシャルネットワークで拡張できます。


ToS-利用規約-利用規約

これは、リソースの使用に関する詳細なルールが記載されたオファー形式の公式ドキュメントです。実際、これはユーザーのアクションを管理する主要なドキュメントです。

これを使用すると、商品またはサービスの購入者の関係を法的機関に転送できます。これは、紛争が発生した場合に役立ちます。サイトに単にアクセスしただけでも、ユーザーは利用規約に同意したことになります。

UI / UX-ユーザーインターフェース/ユーザーエクスペリエンス-ユーザーインターフェース/ユーザーエクスペリエンス

UIとUXは、一緒に混在させることはできませんが、互いに存在しなければ存在し得ない設計の領域です。優れた製品はUXで始まり、UIが密接に続きます。

ユーザーエクスペリエンスは、ユーザーの問題を特定し、それらの問題を解決してユーザーの満足度を高めるプロセスです。ユーザーがサイトを使いやすくするため-考えやウィッシュリストがすぐに機能に反映されるようにします。

ユーザーインターフェイスは、デバイスと人との間の仲介者です。これは見た目だけでなく、製品の双方向性でもあります。つまり、この便利さも美しく、目を楽しませてくれます。

WP-WordPress
