Dockerコンテナー内のワークステーション

何のために？あらゆる種類のアイデアが常に思い浮かび、そのうちのいくつかはすぐに試してみたいと思いますが、ワークステーションが常に手元にあるわけではないので、手に入るすべてのものに対してIDEをセットアップしました。その結果、デバイスが乱雑になり始め、それらを維持および更新することが困難になりました。

この問題を解決するために、私はそのような「ノートブック」をクラウドに置くことにしました。それは毎日更新され、蓄積されたゴミをすべて削除することになります。作業するには、リモートで接続します。



その結果、彼はそれを疑うことなく、多数のタスクを解決するための非常に便利なツールを作成しました。など記事では、そのようなコンテナを作成する方法を共有したいと思います。

このようなコンテナーを作成するには、Dockerfileを作成する必要があります。そして記事では、その作成のプロセス全体について説明します。自分で作成したくないが、試してみたい人は、記事の下部に既製の画像へのリンクがあります。

オペレーティング・システム


すべてのLinuxで実行できます。個人的には、私は美しいOpenSuseを好みますが、そのメモリ消費量をCentOSと比較して、後者を選択することにしました。実際、コンテナが消費するメモリが少ないほど、そのホスティングを節約できます。

dockerfileの作成を始めましょう：

FROM centos:7

リモートアクセス


コンテナーに接続するには、VNCリモートアクセスサーバーを構成する必要があります。このためには、2つの要素を理解する必要があります。

  1. システムにはモニターがありません。したがって、エミュレートする必要があります。このための特別なx0vncserverサーバーがあります。
  2. VNCサーバーは機知に富んでいてはなりません。結局のところ、あなたはそれらのために支払う必要があります。そして、信号伝送の遅れは最小限でなければならず、そうでなければ不快感を引き起こします。

両方の問題を解決するために、サーバーにはTigerVNCサーバーを、クライアントにはTightVNCを選択しました。TigerVNC Serverは、軽量で高速なLinuxの提供に含まれており、x0vncserverを介したモニターなしの作業もサポートしています。TightVNCクライアントは、リモート接続ではなく、コンピューター上で実行されているプログラムのように感じるような画像の高速転送を提供します。

Dockerfileを拡張します。

FROM centos:7

RUN yum install -y epel-release dnf \
        && \
        dnf install -y \
            tigervnc-server \
        && \
        yum clean all && dnf clean all \
        && \
        rm -rf /var/cache/yum/* && rm -rf /var/cache/dnf/*

プログラムをインストールしたら、キャッシュをクリアして画像の重みを軽くします。

デスクトップ


私はBreezeテーマのKDEが大好きですが、KDEは非常に大食いのデスクトップです。ノームとその派生物はさらに大食いでした。Xfce、氷はあまり美しくない。幸い、Kde-PlasmaテーマのLXQTデスクトップという解決策があります。

デスクトップをインストールします。

FROM centos:7

RUN yum install -y epel-release dnf \
        && \
        dnf install -y \
            tigervnc-server \
            openbox obconf-qt \
            lxqt-about lxqt-common lxqt-config lxqt-globalkeys lxqt-notificationd \
            lxqt-openssh-askpass lxqt-panel lxqt-policykit lxqt-qtplugin lxqt-runner \
            lxqt-session pcmanfm-qt \
            dejavu-sans-mono-fonts \
            xterm nano htop expect sudo \
        && \
        yum clean all && dnf clean all \
        && \
        rm -rf /var/cache/yum/* && rm -rf /var/cache/dnf/*

もう少し、あなたは走ることができます。

ユーザー作成


コンテナでは、一部のユーザーから作業する必要があります。これを行うには、それを作成してパスワードを設定するとともに、rootユーザーのパスワードを設定します。

... Dockerfile

ENV HOME=/home/headless

RUN /bin/dbus-uuidgen --ensure && \
        useradd headless && \
        echo "centos" | passwd --stdin root && \
        echo "centos" | passwd --stdin headless

ここでヘッドレスとは、私たちが作成し、そこから作業するユーザーです。「centos」は、ユーザーとルートによって指定されたパスワードです。コンテナーの開始時に外部パラメーターから渡すことをお勧めしますが、この形式でもコンテナーは脆弱ではありません。接続は、引数のパスワードを介してVNCでパスワード保護されます。

vncサーバーの起動を構成する


開始するには、VNCサーバーを構成する補助スクリプトが必要です。

#!/usr/bin/expec

spawn /usr/bin/vncserver :1 -fg -geometry 1820x960
expect "Password:"
send "$env(password)\r"
expect "Verify:"
send "$env(password)\r"
expect "Would you like to enter a view-only password (y/n)?"
send "n\r"

set timeout -1
expect eof

Dockefileの隣に配置する必要があります。後でコンテナに含まれ、プログラムが開始されます。このファイルでは、作業を計画している解像度を指定する必要があります。たとえば、1820x960を設定しています。クライアントウィンドウのサイズより小さい値を指定すると、メモリ不足が原因でサーバーがクラッシュする可能性があります。さらに指定する場合は、画像を拡大縮小する必要があり、デスクトップの要素が小さすぎます。また、このスクリプトでは、コンテナー起動パラメーターで渡される変数からパスワードが設定されます。

このファイルをコンテナにコピーして、起動用のパラメータを追加することは残ります。

... Dockerfile

COPY ./startup.sh ${HOME}
RUN mkdir -p ${HOME}/.vnc \
        && \
        echo '#!/bin/sh' > ${HOME}/.vnc/xstartup && \
        echo 'exec startlxqt' >> ${HOME}/.vnc/xstartup && \
        chmod 775 ${HOME}/.vnc/xstartup \
        && \
        chown headless:headless -R ${HOME}


WORKDIR ${HOME}
USER headless
ENTRYPOINT ["expect", "./startup.sh"]

実行できます。Dockerfileのコンパイルで混乱している場合は、完全版が私のリポジトリにあり、完成したコンテナはdocker hubにあります。

完成したコンテナを起動するには、次のコマンドを実行する必要があります。

docker run -it --rm -e password='YOUR_VNC_PASSWORD' -p5901:5901 labeg/centos-lxqt-vnc

そして、TightVNCクライアントを使用して接続します。



接続後、前に指定したユーザーパスワードを入力する必要があります。安全性を高めるため、すぐに交換することもお勧めします。

上のスクリーンショットのように美しいデザインを有効にするには、[スタート]> [設定]> [外観]> [LXQtテーマ]に移動し、Kde-plasmaテーマを選択します。

仕事のためのプログラム


これで、作業プログラムで2番目のイメージを作成できます。これを行うには、上記で作成したイメージを取得し、パッケージをインストールするためのスクリプトを追加します。

FROM labeg/centos-lxqt-vnc:latest

USER root

# dotnet vscode monodevelop nodejs git2
RUN rpm -Uvh https://packages.microsoft.com/config/rhel/7/packages-microsoft-prod.rpm \
        && \
        rpm --import https://packages.microsoft.com/keys/microsoft.asc && \
        sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/yum.repos.d/vscode.repo' \
        && \
        rpm --import "http://keyserver.ubuntu.com/pks/lookup?op=get&search=0x3FA7E0328081BFF6A14DA29AA6A19B38D3D831EF" && \
        su -c 'curl https://download.mono-project.com/repo/centos7-vs.repo | tee /etc/yum.repos.d/mono-centos7-vs.repo' \
        && \
        yum install -y https://centos7.iuscommunity.org/ius-release.rpm \
        && \
        curl -sL https://rpm.nodesource.com/setup_13.x | bash - \
        && \
        dnf install -y geany git2u git2u-gui code monodevelop firefox dotnet-sdk-3.1 nodejs gnome-terminal gnome-system-monitor \
        && \
        npm install -g gulp typescript npm-check-updates \
        && \
        chown headless:headless -R ${HOME}

USER headless

RUN code --install-extension ms-vscode.vscode-typescript-tslint-plugin && \
    code --install-extension dbaeumer.vscode-eslint && \
    code --install-extension mrmlnc.vscode-scss && \
    code --install-extension visualstudioexptteam.vscodeintellicode && \
    code --install-extension ms-dotnettools.csharp && \
    code --install-extension joelday.docthis && \
    code --install-extension mrmlnc.vscode-remark && \
    code --install-extension eamodio.gitlens

このスクリプトは、LinuxでTypescriptおよびC＃開発用のツールをインストールします。これは、NodeJS、必要な拡張機能を備えたVSコード、およびMonodevelop（別名Visual Studio for Mac）です。

前の画像と同じくらい簡単に始まります。

docker run -it --rm -e password='YOUR_VNC_PASSWORD' -p5901:5901 labeg/devpc

数秒でクリーンな作業環境を展開できます。
