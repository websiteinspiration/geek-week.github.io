Redisとは何ですか？


Redisは、データベース、キャッシュ、およびメッセージブローカーとして使用できるKey-Valueデータ構造のオープンソースデータウェアハウスです。実装に関して、キーバリューストアは、NoSQLの世界で最大かつ最も古い代表の一部です。Redisは、文字列、ハッシュ、リスト、セット、ソートされたセットなどのデータ構造を範囲クエリでサポートします。

Spring Data Redis フレームワークを使用すると、Redisストレージを使用するSpringアプリケーションを簡単に作成でき、データストレージを簡単に抽象化できます。

Redisの設定


アプリケーションをRedisサーバーに接続する必要があります。接続を確立するには、Redisのクライアント実装であるJedisを使用します。

構成


構成Beanの定義から始めましょう。

@Bean
JedisConnectionFactory jedisConnectionFactory() {
    return new JedisConnectionFactory();
}
@Bean
public RedisTemplate<String, Object> redisTemplate() {
    final RedisTemplate<String, Object> template = new RedisTemplate<String, Object>();
    template.setConnectionFactory(jedisConnectionFactory());
    template.setValueSerializer(new GenericToStringSerializer<Object>(Object.class));
    return template;
}

JedisConnectionFactoryBeanとして表示されるため、RedisTemplateクエリするデータを作成できます。

投稿者


SOLID の原則に従って、インターフェースを作成しますMessagePublisher。

public interface MessagePublisher {
    void publish(final String message);
}

MessagePublisherメッセージRedisTemplateの形式で任意のオブジェクトを転送できるため、高レベルのRedisTemplateを使用してメッセージを公開 するインターフェースを実装します。

@Service
public class MessagePublisherImpl implements MessagePublisher {
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    @Autowired
    private ChannelTopic topic;
    public MessagePublisherImpl() {
    }
    public MessagePublisherImpl(final RedisTemplate<String, Object> redisTemplate, final ChannelTopic topic) {
        this.redisTemplate = redisTemplate;
        this.topic = topic;
    }
    public void publish(final String message) {
        redisTemplate.convertAndSend(topic.getTopic(), message);
    }
}

また、これをBeanとして定義しますRedisConfig。

@Bean
MessagePublisher redisPublisher() {
    return new MessagePublisherImpl(redisTemplate(), topic());
}

メッセージの受信者


メッセージをサブスクライブするには、インターフェースを実装する必要がありますMessageListener。新しいメッセージが到着するたびに、メソッドのユーザーコードがonMessage呼び出されます。このインターフェイスは、メッセージ、メッセージの受信に使用されたチャネルへのアクセスを提供し、チャネルのサブスクライブに使用される任意のテンプレートを使用できるようにします。

@Service
public class MessageSubscriber implements MessageListener {
    public static List<String> messageList = new ArrayList<String>();
    public void onMessage(final Message message, final byte[] pattern) {
        messageList.add(message.toString());
        System.out.println("Message received: " + new String(message.getBody()));
    }
}

また、このクラスはとしてBeanとして登録する必要がありますRedisConfig。

@Bean
MessageListenerAdapter messageListener() {
    return new MessageListenerAdapter(new MessageSubscriber());
}

リポジトリインターフェース


他のSpring Dataプロジェクトとは異なり、Spring Data Redisは他のSpring Dataインターフェース上で作業するために必要なすべてを提供します。これは、他のSpring Dataプロジェクトの経験がある人には奇妙に見えるかもしれません。

多くの場合、Spring Dataプロジェクトでリポジトリインターフェースの実装を記述する必要はありません。インターフェースを操作するだけです。Spring Data JPAは、CRUD操作、派生クエリ、ページネーションなどの機能を提供するために拡張できる多数のリポジトリインターフェースを提供します。

したがって、残念ながら、独自のインターフェースを作成してから、メソッドを定義する必要があります。

public interface RedisRepository {
    Map<Object, Object> findAllMovies();
    void add(Movie movie);
    void delete(String id);
    Movie findMovie(String id);
}

リポジトリの実装


クラスは、redisTemplate構成クラスで定義されたものを使用しますRedisConfig。Spring Data Redisを提供

するHashOperationsものを使用します。

@Repository
public class RedisRepositoryImpl implements RedisRepository {
    private static final String KEY = "Movie";
    private RedisTemplate<String, Object> redisTemplate;
    private HashOperations hashOperations;
    @Autowired
    public RedisRepositoryImpl(RedisTemplate<String, Object> redisTemplate){
        this.redisTemplate = redisTemplate;
    }
    @PostConstruct
    private void init(){
        hashOperations = redisTemplate.opsForHash();
    }
    public void add(final Movie movie) {
        hashOperations.put(KEY, movie.getId(), movie.getName());
    }
    public void delete(final String id) {
        hashOperations.delete(KEY, id);
    }
    public Movie findMovie(final String id){
        return (Movie) hashOperations.get(KEY, id);
    }
    public Map<Object, Object> findAllMovies(){
        return hashOperations.entries(KEY);
    }
}

メソッドに注目しましょうinit()。このメソッドでは、名前を持つ関数を使用しopsForHash()ます。このキーに関連付けられたハッシュ値で実行された操作を返します。次に、すべてのCRUD操作hashOpsにinit()、で定義されたを使用します。

Webページを介して映画を追加できるようにしたいと考えています。キーは映画の識別子であり、値は実際のオブジェクトです。ただし、後でこれに戻るため、値として映画の名前のみが表示されます。

次に、フォームを送信するときにJavaScriptを使用して値を保存します。

$(document).ready(function() {
    var keyInput = $('#keyInput'),
        valueInput = $('#valueInput');
    refreshTable();
    $('#addForm').on('submit', function(event) {
        var data = {
            key: keyInput.val(),
            value: valueInput.val()
        };
        $.post('/add', data, function() {
            refreshTable();
            keyInput.val('');
            valueInput.val('');
            keyInput.focus();
        });
        event.preventDefault();
    });
    keyInput.focus();
});

@RequestMappingPOSTリクエストの パラメータを設定し、キーと値をリクエストし、オブジェクトMovieを作成してリポジトリに保存します。

@RequestMapping(value = "/add", method = RequestMethod.POST)
public ResponseEntity<String> add(
    @RequestParam String key,
    @RequestParam String value) {
    Movie movie = new Movie(key, value);
    redisRepository.add(movie);
    return new ResponseEntity<>(HttpStatus.OK);
}

コンテンツを見る


Movieオブジェクトを追加したら、テーブルを更新して新しい値を表示します。JavaScriptコードのブロックでは、関数を呼び出しましたrefreshTable()。GETリクエストを実行して、リポジトリ内の現在のデータを取得します。

function refreshTable() {
    $.get('/values', function(data) {
        var attr,
            mainTable = $('#mainTable tbody');
        mainTable.empty();
        for (attr in data) {
            if (data.hasOwnProperty(attr)) {
                mainTable.append(row(attr, data[attr]));
            }
        }
    });
}

GETリクエストはfindAll()、リポジトリに格納されているすべてのMovieオブジェクトを取得し、データ型をからMap <Object, Object>に変換するメソッドによって処理されますMap <String, String>。

@RequestMapping("/values")
public @ResponseBody Map<String, String> findAll() {
    Map<Object, Object> aa = redisRepository.findAllMovies();
    Map<String, String> map = new HashMap<String, String>();
    for(Map.Entry<Object, Object> entry : aa.entrySet()){
        String key = (String) entry.getKey();
        map.put(key, aa.get(key).toString());
    }
    return map;
}

途中/deleteでPOSTリクエストを実行するスクリプトを記述し、テーブルを更新して、便利な入力のためにキーボードフォーカスを切り替えます。

function deleteKey(key) {
    $.post('/delete', {key: key}, function() {
        refreshTable();
        $('#keyInput').focus();
    });
}

キーを要求し、redisRepositoryこのキーに基づいてオブジェクトを削除します。

@RequestMapping(value = "/delete", method = RequestMethod.POST)
public ResponseEntity<String> delete(@RequestParam String key) {

}
