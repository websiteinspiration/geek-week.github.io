イジングモデル


Isingモデルは、強磁性の性質を理解するために導入され、相転移と臨界現象の研究に影響を与えました。強磁性とは、特定の温度-キュリー点以下の強磁性体における自発磁化の出現を表します。キュリー点（狭い温度範囲）では、秩序化が発生します。この場合、磁気モーメントの整列が発生します。これは、相転移、つまり、物質の特性が急激に変化することを伴います。

当初、このモデルは1920年にV. Lenzによって提案され、Isingによって1次元のケースについて調査されました。このモデルでは、結晶格子の各頂点は数値をとります：+1または-1-フィールド「上」または「下」。頂点と一致する数はスピンと呼ばれます。したがって、Ｎ個のスピンの格子は２Ｎ個の状態になり得る。各状態は、隣接する原子のスピンのペアワイズ相互作用から得られるエネルギーに対応し



ます。Jは隣接するスピンの相互作用のエネルギーです（交換相互作用定数はすべてのペアで同じです）、s iとs jはスピンです。

さらに、J> 0は強磁性体の振る舞いを表し、J <0は反強磁性体です。

モンテカルロ法


前述のように、合計2 Nの状態が可能であり、Nスピンの数が十分に多い場合、数値結果を取得することは困難です。たとえば、N = 10の場合、直接モデル化するのはそれほど簡単ではない2 10の状態が得られるため、統計的手法を使用してモデル化します。

このアプローチでは、システムは特定の温度Tで熱力学的平衡状態にあると見なされます。環境とのエネルギー交換中、エネルギーは平衡状態の近くで変化し、1つの粒子の平均エネルギーはTに比例します。平衡状態の周りの一定のランダム変化の実装では、モンテカルロ法を使用し、モデリングは段階に分けることができます：

  1. 再生状態は、α のIを固定Tで（例えば、ランダムに）システム。
  2. これらの状態について、平衡に近い熱力学的特性（エネルギーE、磁化M）を検討します。
  3. 得られた値の平均


メトロポリスアルゴリズム。


統計力学の問題では、「モンテカルロ法」と「メトロポリスサンプリング法」はほぼ同義語です。スピンまたは粒子のシステムの例として、メトロポリスアルゴリズムの最も一般的な形式を示します。

  1. 初期設定を行います。
  2. 初期設定でランダムなテスト変更を行います。たとえば、ランダムにスピンを選択し、それを倒そうとします。または、ランダムなパーティクルを選択して、ランダムな距離に移動してみてください。
  3. Δconfiguration、つまり、テスト構成の変更によるシステムのエネルギーの変化を計算します。
  4. ∆がゼロ以下の場合、新しい構成を受け入れて手順8に進みます。
  5. Δが正の場合、「遷移確率」を計算します。

  6. 間隔（0、1）で乱数rndを生成します
  7. rnd≤Wの場合、新しい構成を受け入れます。それ以外の場合は、以前の構成を保存します。
  8. 必要な物理量の値を決定します。
  9. 手順2〜8を繰り返して、十分な構成または「テスト」を取得します。
  10. 互いに統計的に独立している構成の平均を計算します。

C ++の2次元イジングモデル

  • ユーザーは、SIZE火格子の側面のサイズ、最大許容温度T_MAXを設定します。
  • ISING SIZE×SIZE, , (0), , , (1), .

  • – . T=2,269℃ .
  • 構成の変更は、画面上で10ミリ秒（ミリ秒）ごとに直接行われます。構成を変更するたびに、モンテカルロステップの数が増加します。
  • ユーザーは、スピンラティスの構成を変更する前に、温度を変更できます。
  • 10秒（秒）ごとに、データがコンソールに出力されます（受け入れ係数、スピンあたりの平均エネルギー、スピンあたりのエネルギーの平均二乗、平均磁化、磁化の平均二乗）。

#include <iostream>
#include <random>
#include <locale>
#include "glut.h"

using namespace std;

#define SIZE 256
#define SIZE_PX 800
#define T_MAX 4
#define CURIE_SCALE 1000. 

double quadSize = SIZE_PX / (double)SIZE;

short lattice[SIZE][SIZE];
double w[5];
double T = T_MAX, M, E;

int ratio = 0;
size_t nmcs = 0;
double ecum = 0, e2cum = 0, mcum = 0, m2cum = 0;

void display()
{
	glClear(GL_COLOR_BUFFER_BIT);
	
	glBegin(GL_QUADS);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			glColor3f(lattice[i][j], lattice[i][j], lattice[i][j]);
			glVertex2d(i, j);
			glVertex2d(i, j + 1);
			glVertex2d(i + 1, j + 1);
			glVertex2d(i + 1, j);
		}
	glEnd();

	glBegin(GL_QUADS);
	{
		glColor3f(1, 0, 0);
		glVertex2d(SIZE, 0);
		glVertex2d(SIZE, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, 0);

		glColor3f(1, 1, 1);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
	}
	glEnd();

	glutSwapBuffers();
	glFlush();
}

void calcW()
{
	double e4 = exp(-4 / T), e8 = e4 * e4;
	w[0] = w[4] = e8;
	w[1] = w[3] = e4;
	w[2] = 0;
}

void init()
{
	M = E = 0;
	static std::default_random_engine generator;
	std::uniform_int_distribution<int> distribution(0, RAND_MAX);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			lattice[i][j] = ((distribution(generator) / (double)RAND_MAX >= 0.5) - 1) * 2 + 1;
			M += lattice[i][j];
		}

	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			E += (i + 1 != SIZE) ? lattice[i][j] * lattice[i + 1][j] : 0;
			E += (j + 1 != SIZE) ? lattice[i][j] * lattice[i][j + 1] : 0;
		}

	calcW();
}

void metropolis()
{
	int x, y, sum;
	for (int i = 0; i < SIZE * SIZE; i++)
	{
		x = rand() % SIZE;
		y = rand() % SIZE;
		sum = lattice[(x - 1 + SIZE) % SIZE][y] +
			lattice[(x + 1 + SIZE) % SIZE][y] +
			lattice[x][(y - 1 + SIZE) % SIZE] +
			lattice[x][(y + 1 + SIZE) % SIZE];

		if (sum * lattice[x][y] <= 0 || (rand() / (double)RAND_MAX) < w[sum / 2 + 2])
		{
			lattice[x][y] = -lattice[x][y];
			::ratio++;
			M += 2 * lattice[x][y];
			E -= 2 * lattice[x][y] * sum;
		}
	}
}

void timer1(int)
{
	display();

	metropolis();

	nmcs++;

	ecum += E;
	e2cum += E * E;
	mcum += M;
	m2cum += M * M;


	glutTimerFunc(10, timer1, 0);
}

void outputData(int)
{
	double norm = 1 / (double)(nmcs * SIZE * SIZE);

	cout << "  = " << ::ratio * norm << endl;
	cout << "    = " << ecum * norm << endl;
	cout << "     = " << e2cum * norm << endl;
	cout << "  = " << mcum * norm << endl;
	cout << "   = " << m2cum * norm << endl << endl;

	glutTimerFunc(10000, outputData, 0);
}

void keyboard(unsigned char key, int x, int y)
{
	switch (key)
	{
	case 'w': case 'W':
	{
		T += 0.01;
		T = (T >= T_MAX) ? T_MAX : T;
		calcW();
		break;
	}
	case 's': case 'S':
	{
		T -= 0.01;
		T = (T <= 0.01) ? 0.01 : T;
		calcW();
		break;
	}
	}
}

void motion(int x, int y)
{
	double k = 1 - y / (double)SIZE_PX;
	T = T_MAX * k;
	T = (T >= T_MAX) ? T_MAX : T;
	T = (T <= 0.01) ? 0.01 : T;
	calcW();
}

int main()
{
	setlocale(LC_ALL, "ru");
	init();
	glutInitDisplayMode(GLUT_SINGLE | GLUT_RGB);
	glutInitWindowSize(SIZE_PX + 100, SIZE_PX);
	glutCreateWindow("ISING");
	glMatrixMode(GL_PROJECTION);
	glLoadIdentity();
	gluOrtho2D(0, SIZE + SIZE / 8, 0, SIZE);
	glutDisplayFunc(display);
	glutKeyboardFunc(keyboard);
	glutMotionFunc(motion);
	glutTimerFunc(1, timer1, 0);
	glutTimerFunc(10000, outputData, 0);

	glutMainLoop();
}

このソフトウェア製品でいくつかの関数を作成する目的に注意してください。

  • init（）-初期構成を作成します。
  • display（）-スピンの格子と温度スケールのレンダリング。
  • calcW（）-遷移確率の計算;
  • metropolis（）-Metropolisアルゴリズムの実装。
  • timer1（）-構成を変更するたびにデータを更新し、モンテカルロステップの数を増やします。
  • outputData（）-スピンあたりの平均の計算、必要なデータの出力。
  • キーボード（）とモーション（）-2つの方法でスケールを使用して温度を変更する機能-キーボードのボタンとマウスのクリック。

いくつかの結論を出す価値があります。

  • 臨界温度T = 2.269℃では、無秩序状態から秩序だった強磁性状態への相転移が起こります。
  • 温度が下向きに変化すると、次の物理的および統計的特性が次のように変化します。
    • 受け入れ率が低下します。
    • スピンあたりの平均エネルギーは減少します。
    • スピンあたりのエネルギーの平均二乗が増加します。
    • 平均磁化は減少します。
    • 磁化の平均二乗が増加します。
