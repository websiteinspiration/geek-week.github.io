テーブルを使用した高速整数乗算

私は、バイトが発明されただけでなく、スタックが発言するのが怖い、そして優れたダイクストラがまだオペレーターを呪っていない遠い時代にそのようなトリックの選択を含むある種の翻訳本で出会ったプログラミングトリックについて読者に伝えたいGOTO（シック、大文字）。

私はそのシンプルさと優美さからこのトリックをとても気に入ったので、この千年紀にすでに次の課題の形で生徒にそれについて喜んで教えました。

2030年に月から戻る過程で、搭載コンピューターが整数乗算演算を誤って実行していることが突然判明し、着陸時に事故につながることを想像してください。

この物語には特に素晴らしいものはありません。たとえば、Pentiumプロセッサで一度発生した問題を思い出してください。、そしてあなたが月に送られたときまでに、あなたはまだ完全な輸入代替品に達していませんでした。一般に、プロセッサが特別にドリルされたかどうかを確認する必要があります。

しかし、要点は。乗算をプログラムで実装して、リアルタイムですばやく動作し、使用可能なリソースに収まるようにすることが急務です。

算数の学校のコースから、多値の数値は列で乗算でき、個々の数値の乗算の結果は乗算テーブルから取得できることを思い出します。

ただし、数値が短く選択された場合（たとえば、0と1）は、九九が短く、列が長くなるため、計算に時間がかかります。

反対に、長い数値（たとえば、0から65535）を取る場合、16ビット演算の場合、
結果はテーブルから直接取得され、列はありません。ただし、古典的なピタゴラスのテーブルのサイズは約17 GB（4 * 65536 * 65536）になります。対角線に関する対称性を考慮すると、半分は約8.5 GBになります。

少し多すぎるかもしれません。

代数を締めて覚えてください。

(a+b)2=a2+b2+2ab （1）

(a−b)2=a2+b2−2ab （2）

減算（2）から（1）

(a+b)2−(a−b)2=4ab

そしてさらに

ab=((a+b)2−(a−b)2)/4

したがって、sqr配列に正方形のテーブルがあると、

a * b =（sqr [a + b]-sqr [a-b]）/ 4 （*）が得られ

ます。8*テーブル（65535 + 65535）のサイズは約8.4MBで、これはすでに許容できる場合があります。

8バイトのテーブル要素のサイズは、最大のaとbの場合、4バイトの合計の2乗が収まらないためです。2ビットが足りません。

次に、本にはなかったいくつかの改善について説明します。すでにこのメモを書いているときに自分で発明しました。

偶数の正方形の最下位2ビットは常に00であり、奇数の正方形は常に01であることに注意してください。一方、数値のペアの場合、それらの合計と差は同じパリティになります。
したがって、式（*）では、括弧内の減算プロセスにハイフンを含めることはできません。
2つの最下位ビットに関連付けられています。したがって、正方形のテーブルの要素の内容を
2ビット右に事前にシフトして、メモリの半分を節約できます。

最後に、

a * b = sqr4 [a + b]-sqr4 [a-b] （**）

があります。ここで、sqr4は正方形の修正されたテーブルです。

この例では、サイズは約4.2 MBです。

以下に、このアプローチを説明するために、Luaプログラムのテキストが含まれています。

function create_multiplier(N)  -- N    
 local sqr4 = {}		--  
 for i = 1, 2 * N - 1 do
  local temp = 0
  for j = 1, i - 1 do		--   
   temp = temp + i - 1	--   
  end					-- ..  ""
  sqr4[i] = math.floor(temp / 4)  --  2 
 end
 return 			--   ()
  function (i, j)
   if i < j then i, j = j, i end
   return sqr4[1 + i + j] - sqr4[1 + i - j]
  end
end
N = 256
mpy = create_multiplier(N)
for i = 0, N - 1 do
 for j = 0, N - 1 do
  if i * j ~= mpy(i,j) then
   print("", i, j, i * j, mpy(i,j))
  end
 end
end
print(" .")
