通常のアバロニア

作成した正規表現がどのように機能するかを理解できず、確認したい場合があります。さまざまな環境に組み込まれた、またはオンラインの多くのエキサイティングなアプリケーションがあります。このリストにもう1つ追加したかったのです。

この記事では、クロスプラットフォームグラフィックスでRegexをラップし、正規表現をテストするための簡単なアプリケーションを取得する方法について説明します。



開始


avalonia uiを使用してプロジェクトを作成するには、GitHubからテンプレートをインストールする必要があります。
そして、mvvmテンプレートからプロジェクトを作成します。

dotnet new avalonia.mvvm -o MyApp

デザインとレイアウト


メインウィンドウを作成するには、すべてのコンポーネントをグリッド内に配置します。

 <Grid>
        <Grid.RowDefinitions>
            <RowDefinition Height="5" />
            <RowDefinition Height="Auto" />
            <RowDefinition Height="5" />
            <RowDefinition Height="*" />
            <RowDefinition Height="5" />
        </Grid.RowDefinitions>
    </Grid>

私たちのアプリケーションは2つの部分から構成されています。ツールバーの上部の類似物とワークスペースです。



トップを考える


ここには、正規表現を入力するためのフィールド、テキストを分析する方法を示すチェックボックス（行ごとまたはフラグメント全体）、およびヘルプボタンがあります。

コンテナはDockパネルです。このコントロールを使用すると、コンテナーを要素で簡単に埋め、要素を別の側に押し、内部に空きスペースがないことを確認できます。

 <DockPanel Grid.Row="1">

そしてそれをテキスト入力フィールドで埋めます：

 <TextBox Margin="5, 0, 5, 0" Watermark="Your regexp" AcceptsReturn="False"  Text="{Binding RegText}"/>

ここで、テキストフィールドに透かしが入っている-Avalonia uiの楽しい些細なことに注意する価値があります。IsThreeState = "False"の

ように、2つの状態のチェックボックスをパネルに追加します。

 <CheckBox DockPanel.Dock="Right" Content="By row" IsThreeState="False" IsChecked="{Binding IsChecked}" ToolTip.Tip="Check for each row" />

そして、ヘルプを呼び出すためのボタン：

 <Button DockPanel.Dock="Right" Content=" ? " Margin="5, 0, 5, 0" ToolTip.Tip="Show hints" Command="{Binding ShowHelp}" />

                <TextBlock  ScrollViewer.VerticalScrollBarVisibility="Auto" Text="{Binding OutputText}" />
            </Border>





新しいウィンドウを作成するためのヘルプウィンドウ。ターミナルに移りましょう

dotnet new avalonia.window -na MyApp -n MyNewWindow

このウィンドウには参照情報のみが含まれ、他には何も含まれないため、コンテナを無視して、ボックスシートをこのウィンドウの子にします。

 <ListBox ScrollViewer.VerticalScrollBarVisibility="Visible" VirtualizationMode="None">
 </ListBox>

必要に応じて同じ要素を入力します。

     <ListBoxItem>
            <StackPanel Orientation="Horizontal">
                <TextBlock Margin="5"  Text="A single character of: a, b or c" />
                <TextBlock Margin="5"  Classes="green" Text="[abc]" />
            </StackPanel>
        </ListBoxItem>

レイアウトは非常にシンプルです。2つのフィールドを持つスタックパネルです。ここで言及する価値があるのは、CSS（Avaloniaのもう1つの優れた機能です）のようにウィンドウのスタイルを設定できるようにするクラス属性です。

実際にルール自体を緑色で描画します。

 <Window.Styles>
        <Style Selector="TextBlock.green">
            <Setter Property="Foreground" Value="Green" />
        </Style>
    </Window.Styles>

MVVMの機能


まず、すべてのマークアップ要素のフィールドに注意しましょう。

private bool _isChecked;
private string _inputText;
private string _regText;
private string _outputText;

元のテキストとの一致の結果を返すメソッドを作成してみましょう。

private void SetNoRowResult()
        {
            OutputText = string.Join(Environment.NewLine,new Regex(_regText).Matches(_inputText));
        }

そして、Splitを使用するソーステキストを行に沿って
 private void SetRowResult()
        {
            var r = new Regex(_regText);
            var s = string.Empty;
            foreach (var line in _inputText.Split( Environment.NewLine,StringSplitOptions.RemoveEmptyEntries))
            {
               
                s += $"--{Environment.NewLine}";
                s+=string.Join(Environment.NewLine,r.Matches(line));
                s += $"{Environment.NewLine}--{Environment.NewLine}";
            }

            OutputText = s;
        }

そして、ヘルプウィンドウを表示するメソッドを追加します

public void ShowHelp()
        {
            new HelpWindow().Show();
        }

プロパティとバインディングAvalonia

の興味深い機能は、メソッドをボタンに直接アタッチできることです。

<Button  Command="{Binding ShowHelp}" />

どちらを使用するか。

残りのフィールドについては、プロパティにバインドするだけです。ここで興味深い機能は、アバロニアでは、コントロールのコンテンツが変更されたときにバインディングの更新が発生することです。これは、vpfで行われました。アバロニアではUpdateSourceTrigger = PropertyChangedがデフォルトで機能します。

また、Avalonia mvvmテンプレートはReactive UIに関連付けられているため、このライブラリを使用してプロパティが作成されます。

public string RegText
        {
            get => _regText;
            set
            {
                this.RaiseAndSetIfChanged(ref _regText, value);
                if(_isChecked) SetRowResult();
                else SetNoRowResult();
            }
        }
