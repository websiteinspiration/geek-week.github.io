C ++、メタプログラミング、およびマイクロコントローラーのレジスター

こんにちは、ハブル！


ここ数年、私はstm32マイクロコントローラーラインのすべてのプロジェクトをC ++で行っています。


この間、他の開発者の興味を引くかもしれない一定量の資料を蓄積してきました。


不要な質問を避けるために、私はQtCreator + gcc + gdb + openocdの束を使用します。それを使用する方法は何度も説明されているので、ここでは止めませんが、マイクロコントローラーを使用する私のアプローチについて詳しく説明します。


プロジェクトの下位レベルには、通常、周辺機器ドライバーがあります。それらから始めましょう。


可能であれば、SPL、HALなどのモンスターを台無しにしないようにし、他の人のプロジェクトをお金で仕上げる場合を除いて、CubeMXを許してください。フレームワークのイデオロギー「大規模マシンについて」は、私の意見では悪質です。柔軟で使いやすい（誰かがこれに異議を唱えるかもしれませんが）関数は、かなり大規模で最適ではないマシンコードに変換され、ランタイムにマイクロコントローラーで多くの不要な操作が実行されます。はい、最新のコントローラーは強力ですが、プロジェクトを組み立てるコンピューターははるかに強力なので、すべてを実行します。


周辺管理を実装するための基本的な要件を自分で決定しました。


  • すべての定数式はコンパイル段階で計算され、マシンコードに数値として分類されます。
  • コンパイラーは、作業の大部分を行うだけでなく、関数に渡される型を制御する必要があります。
  • マクロ 必要ありません これは型コントロールの弱体化につながらない、最小限に使用する必要があります。
  • 実装プログラミングインターフェイスは、コードの読みやすさをサポートする必要があります。

たぶん怠惰な人だけがレジスターフィールドを操作するために彼の自転車を思い付かなかったでしょう。これはシフトとCMSISによる論理演算であり、複雑な腸内で同じ演算を隠し、STのライブラリスタイルにあるあらゆる種類のマクロです。かつては、ビットフィールドについても作業をしていましたが、ようやく解決策が見つかるまで紹介しました。

template<typename T, typename V, typename... Args>
INLINE constexpr static void setRegister(volatile uint32_t& reg,
                                         const T field,
                                         const V value,
                                         const Args... args)
{
    uint32_t mask = setMaskR(field, value, args...);
    uint32_t val = setBitsR(field, value, args...);

    reg = (reg & (~mask)) | val;
}

template<typename V, typename T>
INLINE constexpr static uint32_t setBitsR(T field, V val)
{
    return (val << (field[0]));

template<typename V, typename T, typename... Args>
INLINE constexpr static uint32_t  setBitsR(T field, V val, Args... args)
{
    return (val << (field[0])) | setBitsR(args...);
}  

template<typename V, typename T>
INLINE constexpr static uint32_t setMaskR(T field, V val)
{
    return ((((1 << field[1]) - 1) << field[0]));
}
