二叉搜索树




树的外观在很大程度上取决于数组元素的初始放置方式。根据随机数据，一棵具有高概率的树会变得不平衡，与最佳树相比，树的数量要大得多。在我们的示例中，形成了7个级别。如果数组的元素按以下顺序排列：例如

25、28、29、23、30、27、24、22、21、26

，则搜索树将是平衡的，并且级别数最少（= 4）。使用平衡树排序将具有时间复杂度：O（n log n）。但是在不平衡树的情况下，它可以降级为O（n 2）。

在以前的habrastatia中，考虑了通过二叉搜索树进行排序及其有趣的修改，该方法允许以时间复杂度O（n log n）保证排序。

搜索树越不平衡，遍历它的成本就越高。

另一方面，从组织的角度来看，将数据存储在搜索树中非常方便，它允许您使用此数据执行很多操作，包括排序。

二叉搜索树中的父级不一定要比子级多。但是同时，左子对象始终小于父对象，右子对象始终大于父对象（或等于父对象）。

构建搜索树时，数组的最大元素位于右侧子树中的某个位置。要提取此最大值，必须完全规避搜索树。

笛卡尔树排序::笛卡尔树排序


最终算法：

  • I.基于未排序的数组，我们构造笛卡尔树：
    • I.1。从左到右遍历数组的元素。
    • I.1。如果这是数组的第一个元素，那么它仅仅是笛卡尔树的第一个节点。
    • I.2。如果这不是数组的第一个元素，则将其视为由一个元素组成的笛卡尔树。我们使用从先前元素构建的树执行此树的合并操作。
  • 二。我们从树中删除当前的最大元素，然后还原笛卡尔树。
    • II.1。当前的最大元素是树的根。
    • II.2。我们从树中删除根，将最大元素转移到另一个数组。
    • II.3。除去根后，该结构分为两棵笛卡尔树。
    • II.4。对于这两棵树，我们执行Merge操作。当前的新最大值是已连接树的根之一。
    • II.5。对于新的高点，请再次重复步骤II.1-II.4。




用C ++实现：

/*
 * c++ 11 code to construct a Cartesian tree. The method cartesianTreeSort
 * will sort the contents of the array.
 */

#include <iostream>
#include <vector>
#include <queue>

struct Node{
    int value;
    Node *left, *right;
    Node *parent;

    Node(){
        value = 0;
        parent = NULL;
        left = NULL;
        right = NULL;
    }
};

// Used by priority queue
struct compare{
    bool operator()(Node *left, Node *right){
        return left->value > right->value;
    }
};

class CartesianTree{
private:
    // last pointer to keep track of last node added
    Node *root, *last;

    Node * findLowestNode(Node *node, int x){
        if(node->value < x)
            return node;
        else if(node->parent != NULL)
            return findLowestNode(node->parent, x);
        else
            return NULL;
    }

public:
    Node * getRoot(){
        return root;
    }

    void addNode(int x){
        Node *new_node = new Node;
        new_node->value = x;
        if(root == NULL){
            last = new_node;
            root = new_node;
            return;
        }
        Node *z = findLowestNode(last, x);
        if(z == NULL){
            new_node->left = root;
            root->parent = new_node;
            root = new_node;
        }
        else{
            new_node->left = z->right;
            z->right = new_node;
            new_node->parent = z;
        }
        last = new_node;
    }

    CartesianTree(std::vector<int> ar){
        root = NULL;
        last = NULL;
        // Call addNode function for each element of the array
        for(auto x : ar){
            addNode(x);
        }
    }

    void InorderTraversal(Node *node){
        // To print inorder traversal of the tree
        if(node == NULL)
            return;
        InorderTraversal(node->left);
        std::cout << node->value << ' ';
        InorderTraversal(node->right);
    }

    // Function to sort and store values in array
    void cartesianTreeSort(std::vector<int> &sorted_ar){
        // Initialize input array
        sorted_ar.assign(0, 0);

        // Initialize priority queue
        std::priority_queue<Node *, std::vector<Node *>, compare> p_queue;
        p_queue.push(root);
        Node *temp = NULL;
        while(!p_queue.empty()){
            temp = p_queue.top();
            p_queue.pop();
            sorted_ar.push_back(temp->value);
            if(temp->left){
                p_queue.push(temp->left);
            }
            if(temp->right){
                p_queue.push(temp->right);
            }
        }
    }
};

int main(){
    std::vector<int> ar = {1, 14, 5, 0, 8};

    CartesianTree tree(ar);
    std::cout << "Inorder Traversal\n";
    tree.InorderTraversal(tree.getRoot());
    std::cout << '\n';

    std::vector<int> sorted;
    tree.cartesianTreeSort(sorted);
    std::cout << "Sorted array is\n";
    for(auto x : sorted)
        std::cout << x << ' ';
    std::cout << '\n';
}

成本最小的笛卡尔树将处理数组中已排序和几乎已排序的部分（升序或降序无关紧要）。例如，以下是逆序数组的过程：



该算法最昂贵的部分是合并操作，该合并操作必须对n个元素中的每个元素进行两次-首先是在构建神经酰胺时，然后在分解时。对于随机数据，一次性合并操作的成本为O（log n），但对于位于数组的有序区域中的元素，操作的成本为O（1）。

因此，排序的平均和最差时间复杂度为O（n log n），最佳排序为O（n）。

根据其他记忆，情况更糟-O（n）如果笛卡尔树只是一种堆，那么代价将是O（1）。但是笛卡尔树也是二叉搜索树，这就是为什么对于所有n个元素都必须单独存储父子关系的原因。

此外，这种形式的实现意味着它是一种外部排序-排序后的元素收集在单独的数组中。这是O（n）用于增加内存。
