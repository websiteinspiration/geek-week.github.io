課題1-「ハッキングに対する許可はありません」

参加者にはリンクが与えられ、その後、「Hello world！」という唯一の碑文があるページに移動しました。そのような場合、最初から何をしますか？もちろん、ページのソースコードをクロールしています。



このケースを解決するには、GITのフルパワーを使用してください-GITはよく知られている名前です。しかし、なぜ別の手紙が強調されているのですか？それらを接続すると？.. CGIT？..まさに、そのようなことを聞​​きました！

CGIT-リポジトリWebインターフェイス。

それで、私たちはそれを理解しました。リンクをたどる213.170.100.213/cgitでは、すべてのフォルダーをざっと見て、キーを見つけようとしますが、一定時間経過すると、ポイントが異なることがわかります。

そして今、人々は2つのカテゴリーに分けられます：cgitの注目に値する脆弱性について聞いた人と聞いていなかった人。

いずれにせよ、グーグル検索してCVE-2018-14912を見つける必要があります。次に、

手抜き のみで詐欺はありません：213.170.100.213/cgit/cgit.cgi/my_repo.git/objects/?path=.. / .. / .. / .. / .. / .. / .. / etc/ passwd
パスワード付きのファイルを取得し、それからキーの最初の部分を取得します：
NQ2020Gka2rFseNPexB4JsnP9k9RKulFVQDCcXwYy1aPKI +詳細...

2番目の部分を見つけることは残ります。ここでは簡単です。リポジトリにはそれほど多くのファイルが含まれていません。列挙すると、ファイルには何もないことがわかります。しかし、私たちはGitリポジトリにいます。つまり、そこにあったすべての変更を確認できます。そこで、情報は明らかにより完全になります...

「もっと見る」ポインタでdiffに移動するか、手動で「see more = e9a3c19a544e6589825fd643f4e6d5c1c4e9」を見つけ、最初の部分と連結してキーを取得してください！

タスク番号9-「プログラミングに専念する」

「目を信用しないでください」-これが、このタスクに与えるモットーです。 Cのソースコードを含むファイルをダウンロードする場合、AESアルゴリズムに従って暗号化/復号化するコードのみが内部にあるように見えますが、「そこにはありませんでした」-お知らせします。コードにエラーがあるかどうか、疑わしいスペースやインデントに注意する必要があります。私たちは秘教を約束しましたか？受け取って署名してください！

実際のところ、このファイルには、Cコードに加えて、難解なプログラミング言語のコードもあります-ホワイトスペースとスプーン！ Whitespace言語のコードはキーを提供し、Spoon言語のコードは暗号文を提供します。コードのエラーを修正し、受信したデータを使用して実行し、キーを取得します！
