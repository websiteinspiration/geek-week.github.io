蛇糖或用JavaScript编写范围

许多人喜欢Python。初学者对分号的缺乏感到钦佩，而高级分号则享有真正的简单性。今天，我们将讨论如何在JavaScript中实现非常简单的Python，特别是range函数。

在Python中，可以对range函数进行迭代，或例如将其转换为数组列表（range（开始，结束））。

但是问题是，JavaScript是否有可能创建类似的东西，同时使解决方案看起来既原生又简单？

首先想到的是编写一个类似的类：

function range(from, to, step = 1){
    this.current = from
    this.to = to
    this.step = step

    this.next = () => this.current++ % to
}

是的，我写道下一次返回调整后的值，但这也是一个需要解决的问题-达到最大值时抛出错误或执行其他操作？

在一般情况下，这样的解决方案将没有添加方便，它会更容易写一个标准的，例如，您可以添加端功能和每次迭代检查，如果我们出国后，但在开发这一解决方案是没有意义的。

ES6引入了新的原始数据类型Symbol。他向我们敞开了元编程的大门。

为了解决范围问题，我们将使用Symbol.iterator。这样就可以使用特定协议创建迭代器对象。

为了使对象成为迭代器，您需要在其中定义[Symbol.iterator]（）函数，但是由于我们要创建一个通用函数而不是类，因此我们将拥有一个返回迭代器对象的函数。

看起来是这样的：

function range(from, to, step = 1){
    return {
        [Symbol.iterator](){
            return {
                current: from,
                to: to,
                from: from,
                step,
                next(){
                    const it = { done: this.current >= this.to, value: this.current }
                    this.current += this.step
                    return it
                }
            }
        }
    }
}

发生了什么？我们有一个range函数，它返回相同的迭代器对象，其中[Symbol.iterator]（）函数返回一个对象。最重要的是，该对象包含下一个函数，否则该对象将不可迭代，并且在尝试使用它时将出现错误。下一个函数应该（！）返回具有done和value属性的对象，其中done（bool）表示迭代结束，并且value包含当前值。

原则上，一切正常，您已经可以编写类似于python的内容：

    for(let i of range(0, 10, 2))
        console.log(i)

结论：

0
2
4
6
8

我们可以在这里停止...但是您可以使用生成器函数编写更优雅的解决方案：

function range(from, to, step = 1){
    return {
        *[Symbol.iterator](){
            for(let val = from; val < to; val += step){
                yield val;
            }
        }
    }
}

一个额外的好处是我们为迭代器提供了其他机会。例如，您可以轻松地将我们的范围转换为数组：

[...range(0, 10, 2)]
// [ 0, 2, 4, 6, 8 ]
