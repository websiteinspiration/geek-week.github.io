データサイエンスの最初のステップ。タイタニック

小さな紹介


何をどのように行うかを説明する段階的な指示が提供されれば、もっと多くのことができると思います。自分がどこから始めればいいのかわからなかったために事業が始まらなかったときのことを思い出します。たぶん、インターネットで昔、「データサイエンス」という言葉を見て、それとはかけ離れており、別の世界のどこかでこれをやっている人々はそうではないと判断しました。いいえ、彼らはここにいます。そして、おそらく、この領域の人々のおかげで、記事がフィードに表示されました。この技術に慣れるのに役立つ多くのコースがあります。ここでは、あなたが最初の一歩を踏み出すのを助けます。

さて、準備はいいですか？ここではPython 3を使用するので、Python 3を知っている必要があることをすぐに言っておく必要があります。また、Jupyter Notebookにプレインストールするか、google colabの使用方法を確認することをお勧めします。

第一歩




Kaggleはこの問題であなたの重要なアシスタントです。原則として、これはなくても問題ありませんが、これについては別の記事で説明します。これは、データサイエンスコンテストを主催するプラットフォームです。初期段階のこのような各コンテストでは、さまざまな問題を解決する非現実的な量の経験、開発経験、チームワークの経験を受け取りますが、これは私たちの時代には重要です。

そこから仕事を引き受けます。それは「タイタニック」と呼ばれています。条件はこれです：一人一人を予測することは生き残るでしょう。一般的に言って、DSに関わる人のタスクは、データの収集、その処理、モデルのトレーニング、予測などです。 kaggleでは、データ収集段階をスキップできます-それらはプラットフォームで表示されます。私たちはそれらをダウンロードする必要があり、あなたは始めることができます！

第二段階


このデータをどのようにダウンロードしますか？

まず、必要なライブラリをインポートします。

import pandas as pd
import numpy as np

Pandasでは、.csvファイルをダウンロードしてさらに処理することができます。

Numpyは、データテーブルを数値の行列として表示するために必要です。
進め。train.csvファイルを取得してアップロードします。

dataset = pd.read_csv('train.csv')

データセット変数を通じてサンプルのtrain.csvデータを参照します。そこにあるものを見てみましょう：

dataset.head()



head（）関数を使用すると、データフレームの最初の数行を表示できます。

存続列は、このデータフレームで知られている単なる結果です。問題の問題では、test.csvデータのSurvived列を予測する必要があります。このデータには、タイタニック号の他の乗客に関する情報が格納されています。これらの乗客は、意思決定者であり、結果を認識していません。

したがって、テーブルを従属データと独立データに分割します。ここではすべてが簡単です。依存データとは、結果に含まれるものとは無関係のデータです。独立したデータは、結果に影響を与えるデータです。

たとえば、次のようなデータセットがあります。

「Vovaはコンピュータサイエンスを教えました-いいえ。
Vovaはコンピュータサイエンス2を受賞しています。 "

コンピュータサイエンスの評価は、Vovaがコンピュータサイエンスを教えましたか？はっきりしていますか？次に進み、目標に近づきます！

独立データの従来の変数はXです。従属データの場合はyです。

次のことを行います。

X = dataset.iloc[ : , 2 : ]
y = dataset.iloc[ : , 1 : 2 ]

それは何ですか？iloc関数[：、2：]を使用して、Pythonに次のように伝えます。変数Xで、2番目の列から始まるデータを確認します（包括的に、カウントがゼロから始まる場合）。2行目では、1列目のデータを確認したいとします。

[a：b、c：d]は、括弧内で使用するものの構造です。変数を指定しない場合、デフォルトのままになります。つまり、[：、：d]を指定すると、データフレームに、番号d以降のすべての列を除きます。変数aとbは文字列を定義しますが、すべて必要なので、このデフォルトのままにします。

何が起きたのか見てみましょう：

X.head()



y.head()



この小さなレッスンを簡略化するために、特別な「ケア」が必要なカラム、または生存にまったく影響を与えないカラムを削除します。タイプstrのデータが含まれています。

count = ['Name', 'Ticket', 'Cabin', 'Embarked']
X.drop(count, inplace=True, axis=1)

素晴らしい！次のステップに進みます。

ステップ3


ここでは、マシンがこのデータが結果にどのように影響するかをよりよく理解できるように、データをエンコードする必要があります。ただし、すべてをエンコードするのではなく、残したstrタイプのデータのみをエンコードします。列「セックス」。どのようにエンコードしたいですか？ベクトルによる人間の性別のデータを想像してください：10-男性、01-女性。

はじめに、テーブルをNumPy行列に変換します。

X = np.array(X)
y = np.array(y)

そして今私たちは見る： 

from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder

ct = ColumnTransformer(transformers=[('encoder', OneHotEncoder(), [1])],
                       remainder='passthrough')
X = np.array(ct.fit_transform(X))

sklearnライブラリは、データサイエンスですべての作業を実行できるようにする優れたライブラリです。興味深い機械学習モデルが多数含まれており、データの準備も可能です。

OneHotEncoderを使用すると、説明したように、その表現で人物の性別をエンコードできます。男性、女性の2つのクラスが作成されます。人が男性の場合、「男性」列には1が、0にはそれぞれ0が書き込まれます

。OneHotEncoder（）の後には[1]かかります。これは、列番号1をエンコードすることを意味します（ゼロから数えます）。

素晴らしい。さらに移動します！

原則として、これは一部のデータが空のままである（つまり、NaN-数値ではない）場合に発生します。たとえば、名前、性別など、人に関する情報があります。しかし、彼の年齢に関するデータはありません。この場合、この方法を使用します。すべての列の算術平均を求め、列にデータが欠落している場合は、算術平均をボイドに入力します。

from sklearn.impute import SimpleImputer
imputer = SimpleImputer(missing_values=np.nan, strategy='mean')
imputer.fit(X)
X = imputer.transform(X)

次に、データが非常に散在しているときにこのような状況が発生することを考慮します。一部のデータは[0：1]の間隔にあり、一部は数百および数千になる可能性があります。このような広がりを排除し、コンピューターがより正確に計算できるように、データをスケーリングします。すべての数が3を超えないようにします。これを行うには、関数StandartScalerを使用します。

from sklearn.preprocessing import StandardScaler
sc = StandardScaler()
X[:, 2:] = sc.fit_transform(X[:, 2:])

これで、データは次のようになります：



クラス。私たちはすでに目標に近づいています！

第四段階


最初のモデルをトレーニングしてください！sklearnライブラリから、興味深いものを大量に見つけることができます。このタスクにグラデーションブースティング分類子モデルを適用しました。私たちのタスクは分類タスクなので、分類子を使用します。予測を1（存続）または0（存続しなかった）に関連付ける必要があります。

from sklearn.ensemble import GradientBoostingClassifier
gbc = GradientBoostingClassifier(learning_rate=0.5, max_depth=5, n_estimators=150)
gbc.fit(X, y)

関数fitはpythonに指示します。モデルにXとyの間の依存関係を調べさせます。

1秒未満で、モデルの準備が整いました。



それをどのように適用しますか？これから見ていきます！

ステップ5 結論


次に、予測を行うために必要なテストデータをテーブルに読み込む必要があります。このテーブルを使用して、Xに対して行ったのと同じアクションをすべて実行します。

X_test = pd.read_csv('test.csv', index_col=0)

count = ['Name', 'Ticket', 'Cabin', 'Embarked']
X_test.drop(count, inplace=True, axis=1)

X_test = np.array(X_test)

from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder
ct = ColumnTransformer(transformers=[('encoder', OneHotEncoder(), [1])],
                       remainder='passthrough')
X_test = np.array(ct.fit_transform(X_test))

from sklearn.impute import SimpleImputer
imputer = SimpleImputer(missing_values=np.nan, strategy='mean')
imputer.fit(X_test)
X_test = imputer.transform(X_test)

from sklearn.preprocessing import StandardScaler
sc = StandardScaler()
X_test[:, 2:] = sc.fit_transform(X_test[:, 2:])

すでにモデルを適用します！

gbc_predict = gbc.predict(X_test)

すべて。予測しました。次に、csvに記録してサイトに送信する必要があります。

np.savetxt('my_gbc_predict.csv', gbc_predict, delimiter=",", header = 'Survived')

できました。各乗客の予測を含むファイルを取得しました。これらの決定をサイトにアップロードし、予測の見積もりを取得することは残っています。そのような原始的な解決策は、一般の人々の正解の74％を与えるだけでなく、データサイエンスへの推進力も与えます。最も好奇心が強い人はいつでも私にプライベートメッセージを書いて質問することができます。ありがとうございます！
