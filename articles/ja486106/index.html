Raspberry Pi TVのアダプティブバックライト-Ambilight Analog



これは、画面の上部と側面にアダプティブバックライトの37 RGBドットを備えた50インチテレビです。どのような画面やテレビにも同様のアダプティブバックライトを作成する方法は厄介なトピックですが、いくつかの理由からかなり関連があります。

まず第一に、すべてが変化しています。数年前、ソフトウェアやハードウェアの松葉杖なしではプロジェクトは機能しませんでした。いずれにせよ、私はそのような出版物を見たことがありません。インターネットで説明されているものは、ソフトウェアまたはハードウェアプラットフォームの陳腐化により関連性が失われているか、PCに接続されたモニターに関してのみこのテクノロジーを実装しています。

次に、このテクノロジーをサポートしているサムスン、ソニー、エルジなどのテレビは、フィリップスがこれから何年にもわたって特許を取得していたため、市場には存在しません。そして最後に、アリや彼のような他の人が利用できるすべてのソリューションは、設定の柔軟性とビデオコンテンツのフレームサイズへの適応、色域の調整など、多くの制限があり、ハードウェアの松葉杖だけで外部信号を分析します。そして、バックライトの形成にのみ取り組んでいます。

「ハードウェア」フレームキャプチャオプション


テレビ受信機自体の洗練と近代化のための理論的なオプションについて話さない場合は、どの場合でもHDMI信号を処理する必要があるという事実に同意する必要があります。フレーム分析とバックライトに使用されます。ここでは、2つの実装オプションのみが推奨されています。任意のソースからの信号処理（Ali付きのライトパックなど）と、適応型バックライト付きのTVセットトップボックスとしてのソース自体の実装です。

最初のケースでは、ある種の汎用性が得られます。ゲームコンソールやある種のプレーヤーをデバイスに接続できます。 2番目-バックライト機能が組み込まれた、これと同じ「ある種のプレーヤー」を収集します。実験中に両方のオプションを実装しましたが、ここでは2番目のみに焦点を当てます。その理由は次のとおりです。



最初のバリアントの所定のブロック図では、任意のソースからのHDMI信号がスプリッターに到達し、そこで2つの出力に分割されます。最初から、コンテンツは任意のテレビ受信機またはモニターで再生されます。 2番目から、信号は同時にハードウェアスケーラーに入り、低解像度フレーム-320x480のアナログ信号に変換されます。アナログ信号はハードウェアUSBグラバーに送信され、各フレームの「スクリーンショット」を取り、USB経由でシングルボードミニPC-Raspberry Pi 3 B +の入力に転送します。ラズベリーのソフトウェアは、Hyperionプログラムを使用してフレームを分析し、GPIO出力の1つを通じて、WS2812BやAPA102などのアドレステープの入力に制御シーケンスを提供します。これらはすべて正常に機能しましたが、接続のある鉄の山の形と、バックライト信号の発行にわずかな遅延といういくつかの欠点があります。ほとんどのフレームで色が急激に変化するテストビデオでのみ遅延が顕著でした。それ以外の場合、この実装のすべての鉄のコストを除いて、不満はありません。

このソフトウェアの基礎は、Kodiプレーヤー向けに特別に強化されたオープンLibreelec OSです。プロジェクトサイトには、必要な情報がすべて含まれています。

ビデオオーディオコンテンツ、IPTVなどを再生するための最も強力で柔軟なプラットフォームの機能について任意の検索エンジン「機会のコディ説明」で採点することによって読むことができます。このトピックに関するロシア語の情報はたくさんあります。私はおそらく、すべてのプラットフォーム機能の1/10を使用していますが、これで十分です。プラットフォームは積極的にサポートされており、レスポンシブコミュニティがあります。したがって、私たちは読んで研究します-これらのプロジェクトのウィキとフォーラムでは、必要な情報と質問への回答のほとんどすべてがあります。 LibreelecプロジェクトのWebサイトで、配布キットとインストールおよび構成の説明を取得します。 OSはクロスプラットフォームなので、私たちのプロジェクトはRaspberry上に構築されていることを忘れないでください。

奇妙ですが、定評のある伝統を守りながら、アドレステープの種類の説明と、家庭用テレビの後ろの壁に不格好に貼り付ける方法を説明するために、シングルボードソフトウェアのインストールと構成から始めます。残念ながら、設定のための段階的な指示を作成することはほとんど不可能です-スクリーンショットなしで500ページのテキストが必要になります。さらに、プレーヤーのプラットフォームは活発に開発されており、OSとKodiのバージョンは変更されています。現在の情報-間もなく変更されます。したがって、ここでは、主要なポイントと、役立つ情報のソースへのリンクのみを示します。

バックライトのソフトウェア部分を構成し、IRリモートコントロールとプレーヤー自体を制御するには、Libreelecディレクトリの配置方法、ローカルネットワークの動作、コンソール、コマンドラインでの処理方法などを最初に理解しておく必要があります。手動で。 microSDカードのインストーラーを使用して、Libreelec OSとハイライト用のアドイン-Hyperionのみがインストールされます。

Raspberry Piには、リアルタイムクロック、BIOS、バックアップ電源が組み込まれていないため、動作に必要なものはすべてmicroSDカードにあります。カードにOSをインストールするには、RaspberryプロジェクトのWebサイトからNOOBSパッケージ（インストール時にビルドバージョン3.2.1を使用しました）または特別なツール「LibreELEC USB-SD Creator」をダウンロードする必要があります。

メモリカードは、良好な読み取り速度と書き込み速度を備えている必要があります。つまり、少なくともグレード10である必要があります。ボリュームの点では、Libreelec OSはほとんどスペースを必要としません。ボリュームを追加購入する必要がない場合は、16 GBカードで十分です。任意のコンテンツを格納するために使用できます。

NOOBSの場合、zipアーカイブの内容がFAT32でフォーマットされたmicroSDカードのルートに解凍され、カードが非通電ラズベリーのスロットに挿入され、電源がオンになり、インストーラーで必要なOSを選択し、指示に従って、Libreelecを展開します。ただし、プラットフォーム開発者は、配布パッケージをダウンロードしてカードに書き込むLibreELEC USB-SD Creatorを使用してインストールすることをお勧めします。その後、カードはラズベリースロットにも挿入され、電源をオンにした後、インストーラーがOSを展開します。この場合、カードのセクションが再フォーマットされます。その後、Windowsからは、メディアの実際のボリューム全体が表示されなくなります。何らかの理由でパーティションを元の状態に戻す必要がある場合は、まず標準のWindowsユーティリティであるDiskPartを使用してカードをクリーニングする必要があります。次に、「ディスクの管理」セクションで、メディアの空き領域に新しいパーティションを作成し、フォーマットします。

OSのインストールと構成を簡略化するために、ラズベリーに一時的に別のモニターとキーボードをマウスで接続しました。
