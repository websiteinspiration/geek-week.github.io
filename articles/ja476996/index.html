Electron上のアプリケーションでLinuxのalt + shiftを切り替える問題を解決する

こんにちは！

ヘッダーに示されている問題の解決策を共有したいと思います。同僚に触発された記事を書くbrnovk誰もが怠惰ではなく、問題の部分的な（私にとっての）解決策を提案しました。助けてくれる「松葉杖」を作りました。私はあなたと共有します。

問題の説明


私は仕事にUbuntu 18.04を使用しましたが、最近、Electronを使用して作成されたVisual Studio Code、Skype、SlackなどのアプリケーションでAlt + Shiftでレイアウトを切り替えると、次の問題が発生することに気付きました：入力フィールドからのフォーカスがウィンドウの上部パネル（メニュー）。他の理由で、私はFedora + KDEに移動し、問題が解消されないことに気付きました。解決策を探していたら、素晴らしい記事Do it own Skypeを見つけました。どうもありがとうbrnovkそれは問題について詳しく話し、それを解決する彼の方法を共有しました。しかし、記事で指定された方法は、1つのアプリケーション、つまりSkypeのみで問題をカバーしていました。ジャンプメニューでメッセージを書くのは面倒ですが、開発している場合はそれほど多くないので、私がVisual Studio Codeを扱うことは依然として重要でした。さらに、同僚がアプリケーションメニューが完全に消えるソリューションを提案しましたが、VS Codeでメニューを失いたくありません。

どうしたのか理解しようとした


そこで、時間をかけて何が起こっているのかを理解することにしました。ここで私が行った方法について簡単に説明します。この問題についてより知識のある人が、私が遭遇した困難を明確にするのに役立つかもしれません。

Visual Studio Codeを開き、さまざまな組み合わせAlt + <％something％>をクリックして、アプリケーションの反応を確認しました。ほとんどすべての場合、Alt + Shiftを除くすべての組み合わせが、フォーカスを失うことなく機能しました。誰かが押されたShiftキーを押したように見え、Altを押したままにした後、アプリケーションはAltを押してから何も押さなかったと考え、Altを放しました。

キーボードレイアウトスイッチの設定を開いて（チェックボックスとキーのすべての種類の設定が含まれるこの長いリストをご存知でしょう）、追加のキーを押すことなく、Altボタンでキーボードレイアウトスイッチを設定しました。



その後、Alt + Tabはウィンドウを切り替えるために機能しなくなりました。タブのみが機能しました。つまり、誰かが再び私のAltを「食べました」。この「誰か」に誰も質問は残っていませんが、私は彼と一緒に何ができるかわかりません。

しかし、問題は少なくとも何らかの形で解決する必要があったため、解決策が思い浮かびました。

  1. 設定で、ホットキーをオフにしてキーボードレイアウトを切り替えます（[別のレイアウトに切り替え]セクションのすべてのチェックボックスを削除します）。
  2. レイアウトを切り替える独自のホットキーを作成する

ソリューションの説明


最初に、コマンドをXbindkeysキーに割り当てることができるプログラムをインストールします。残念ながら、通常のツールでは、Alt + Shiftタイプの組み合わせのホットキーを美しいインターフェイスで作成できませんでした。Alt + S、Alt + 1、Alt + Shift + Yなどに対して実行できます。など、これは私たちの仕事には適していません。

sudo dnf install xbindkeysrc

詳細はArchWikiにあります。
次に、プログラムのサンプル設定ファイルを作成します。サンプルは非常に短く、いくつかのコマンドだけで、それを操作する方法を理解するために必要なものがあります。

xbindkeys -d > ~/.xbindkeysrc

ファイル内の例からわかるように、使用するホットキーと実行するコマンドを指定する必要があります。シンプルに見えます。


# Examples of commands:
"xbindkeys_show"
  control+shift + q
# set directly keycode (here control + f with my keyboard)
"xterm"
  c:41 + m:0x4

ホットキーとして、人間が読める形式のスペルを使用したり、キーコードを使用したりできます。これはコードでのみ機能しましたが、少し実験することを禁止する人はいません。

コードを取得するには、次のコマンドを使用する必要があります。

xbindkeys -k

小さな「X」ウィンドウが開きます。このウィンドウにフォーカスしたキーを押すだけです！この場合のみ、ターミナルに次のようなものが表示されます。


[podkmax@localhost ~]$ xbindkeys -k
Press combination of keys or/and click under the window.
You can use one of the two lines after "NoCommand"
in $HOME/.xbindkeysrc to bind a key.
"(Scheme function)"
    m:0x4 + c:39
    Control + s

私の場合、キーボードショートカットのAlt + Shiftは次のようになります。

m:0x8 + c:50

この組み合わせをクリックすると、レイアウトが正確に切り替わることを確認する必要があります。レイアウトを指定するコマンドが1つだけ見つかりました。


setxkbmap ru
setxkbmap us

例からわかるように、彼女は1つまたは別のレイアウトをオンにする方法しかわかっていないため、スクリプトを書く以外は何も思いつきませんでした。


vim ~/layout.sh
#!/bin/bash
LAYOUT=$(setxkbmap -print | awk -F + '/xkb_symbols/ {print $2}')
if [ "$LAYOUT" == "ru" ]
        then `/usr/bin/setxkbmap us`
        else `/usr/bin/setxkbmap ru`
fi

これで、.xbindkeysrcファイルとlayout.shファイルが同じディレクトリにある場合、.xbindkeysrcファイルの最終的な形式は次のようになります。


# Examples of commands:

"xbindkeys_show"
  control+shift + q

# set directly keycode (here control + f with my keyboard)
"xterm"
  c:41 + m:0x4

# specify a mouse button
"xterm"
  control + b:2
#  ,   
"./layout.sh"
  m:0x8 + c:50

その後、変更を適用します。


xbindkeys -p

そして、あなたはチェックすることができます。標準設定でレイアウトを切り替えるためのオプションを無効にすることを忘れないでください。

合計
