LibreOffice拡張機能をビルドするためのSDKをインストールして構成する

前書き


DSS製品の開発プロセス中に、C ++でLibreOfficeの拡張機能を作成する必要がありました。言語の選択は、C ++の既存のプロジェクトの存在によって決まりましたが、プロジェクトに「動物園」を作成したくありませんでした。この問題

に関する資料、このリソースを含む記事の研究を始めました。多くのデータがあるように見えますが、実際には情報が非常に少ないか、LOの古いバージョンが原因でもはや関連性がありませんでした。

私は自分で問題を研究しなければなりませんでした。



SDKアセンブリ


Windowsでビルドするには、次のものが必要です。


  • LibreOffice 6.2.4 / 6.2.5 + LibreOffice SDK 6.2.4 / 6.2.5（LibreOffice＆SDK）以降。
  • GNU Zip Make CoreUtils Sed（GNU）
  • jdk（JDK）
  • C ++ + MSToolsおよびMicrosoft.NETのコンパイラがインストールされたMicrosoft VS

主な手順


LibreOfficeとLibreOffice SDKを、できれば "スペースなし"ディレクトリにインストールします。

例C：\ App \ Libreoffice6.2.4.2x64およびC：\ App \ Libreoffice6.2.4.2x64 \ sdk。

私たちは、ダウンロードしたSDKを含むディレクトリに移動し、コンソールを開きます。
CD Cは：\アプリケーション\ Libreoffice6.2.4.2x64 \ SDK
setsdkenv_windows.bat実行

このスクリプトは、最初に、システム上のユーザーのためのバッチファイルを生成し、パスをするために登録することを尋ねます：

  1. LibreOffice
  2. LibreOffice SDK;
  3. パッケージをビルドするためのGNUユーティリティ、C ++、C＃およびVB.NETコンパイラ、およびJava SDKディレクトリ。

適切なフォルダを指定します。

重要！対応するビット深度のバージョンへのすべてのパスを指定する必要があります。 「UNOコンポーネントの自動展開（YES / NO）」は、アセンブリの直後にパッケージをインストールします。NOを選択しました。バッチファイルへのパス、たとえばC：\ Users \ yurev.admin \ AppData \ Roaming \ libreoffice6.2_sdkをたどり、ハンドルを使用してVCVARS32へのパスを調整することも必要です。これを新しい変数で行いました。

set OO_SDK_CPP_HOME=C:\Program Files (x86)\Microsoft Visual Studio\2019\Enterprise\VC\Tools\MSVC\14.20.27508\bin\HostX64\x64
set OO_SDK_VCVARS_HOME=C:\Program Files (x86)\Microsoft Visual Studio\2019\Enterprise\VC\Auxiliary\Build
if defined OO_SDK_VCVARS_HOME call "%OO_SDK_VCVARS_HOME%\VCVARS64.bat"

ビット深度のパスを選択する必要があります。

環境変数を正常に設定すると、次のメッセージが表示されます。



コメントはありません-これは後でアセンブリに影響します。コマンドラインは、SDKモード用に準備されたシェルに入ります。

また、GNUパッケージのあるディレクトリ（C：\ Program Files（x86）\ GnuWin32 \ bin）に移動し、link.exeリンカーの名前を変更して、将来Microsoftリンカーと混同しないようにします。必要に応じて、常に新しい名前で呼び出すことができます。

SDKコンパイル


次のステップは、ヘッダーまたは（Javaの）クラスをコンパイルすることです。それ自体、SDKは.idl拡張子を持つコンポーネントのコレクションであり、.udlブロックにアセンブルされ、次に.rdb拡張子を持つレジストリにアセンブルされます。

もちろん、この形式では使用できませんが、このアプローチの目標は、独自のコンポーネントを記述して、一般的なツリーに追加する機能です。

LibreOfficeの例からプロジェクトを取り上げます。

C：\ App \ Libreoffice6.2.4.2x64 \ sdk \ examples \ cpp \ complextoolbarcontrolsディレクトリに移動し、makeを実行します。

最初のステップのデータが正しく入力され、環境変数が正しく設定されている場合は、

  • Cのダウンロードファイル：\ libreoffice6.2_sdk \ WINexample.out \ incディレクトリ、
  • manifest.xml C:\libreoffice6.2_sdk\WINexample.out\misc\complextoolbarcontrols\complextoolbarcontrols\META-INF,

本質的に、Makefileは環境変数を正しく入力することで、すべてのことを実行してくれます。

Cppmakerは、offapi.rdbおよびtypes.rdbレジストリファイルに基づいて、cxxファイルのビルドに必要な依存関係をダウンロードします。

コンパイラーがこれらのファイルをオブジェクトに収集し、リンカーがそれらをLibreOfficeディレクトリーからぶら下げて出力dllを収集し、manifest.xml description.xml .componentsファイルが生成され、zipを使用して.oxtアーカイブにすべてパッケージ化されます。吊り下げられたコンソールが原因でビルドできないため

、cigwin（wiki）のバージョンは放棄する必要がありました。

Linuxでビルドするには、次のものが必要です。


ibcurl4-openssl-dev zip git build-essential wget curl gstreamer1.0-libav libkrb5-dev nasm graphviz ccache libpython3-dev libreoffice-dev*

これらのパッケージはスペースをあまり取らないため、Dockerイメージを使用して拡張機能をビルドします。

準備プロセスはWindowとほとんど同じです。setsdkenv_windows.batの代わりに、/ usr / lib / libreoffice / sdkからsetsdkenv_unixを実行します。

その後、Windowsと同様に、/ usr / lib / libreoffice / sdk / examples / cpp / complextoolbarcontrolsに移動し、makeを実行します。

その結果、アセンブリの完了時に、setsdkenv_unixを介して構成中に指定されたディレクトリにある完成した拡張機能を取得します。
