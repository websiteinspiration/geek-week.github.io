Citrix NetScaler CVE-2019-19781の重大な脆弱性が非表示のIT問題を公開する方法

親愛なる読者、まず第一に、私はドイツの居住者として、私は主にこの国の状況について説明していることを指摘したいと思います。おそらくあなたの国では状況は根本的に異なります。

2019年12月17日、Citrix Knowledge Centerは、Citrix Application Delivery Controller（NetScaler ADC）および一般にNetScaler Gatewayとして知られるCitrix Gateway製品ラインに重要な脆弱性情報を公開しました。将来的には、SD-WANラインにも脆弱性が発見されました。この脆弱性は、10.5から現在の13.0までの製品のすべてのバージョンに影響を及ぼし、不正な攻撃者がシステムで悪意のあるコードを実行することを許可し、実質的にNetScalerを内部ネットワークへのさらなる攻撃のためのプラットフォームに変えました。

  • CTX267027：CVE-2019-19781-Citrix Application Delivery Controllerの脆弱性
  • CTX267679：CVE-2019-19781の緩和策
  • CTX269180：CVE-2019-19781-検証ツール（1/15/2020に投稿！）

脆弱性情報の公開とともに、Citrixは回避策ガイドラインを公開しました。脆弱性の完全な閉鎖は、2020年1月末までに約束されました。

この脆弱性の重要度（CVE-2019-19781）は、10ポイント中9.8と評価されました。Positive Technologiesによると、この脆弱性は世界中の8万社以上に影響を及ぼしています。

ニュースに対する考えられる反応


私は責任者として、インフラストラクチャにNetScaler製品を使用しているすべてのITプロフェッショナルが次のことを実行すると信じていました。

  2. NetScalerから内部ネットワークへの許可されたトラフィックに関してファイアウォール設定を再確認しました。

残念ながら、後で判明するように、標準的なアプローチである上記の手順はほとんどの人によって無視されています。

Citrixインフラストラクチャの責任者である多くの専門家は、中央ニュースから 2020年1月13日のみ脆弱性について知りました。彼らは、彼らが担当していた膨大な数のシステムが侵害されたときを知りました。状況の不条理は、これに必要なエクスプロイトがインターネット上で非常に合法的にダウンロードできるようになりました。
どういうわけか、ITスペシャリストはメーカーからのニュースレターを読み、委託されたシステムはTwitterを使用でき、その分野の主要な専門家に登録されており、時事問題を認識する必要があります。

実際、3週間以上にわたり、多くのCitrix顧客が製造元の推奨を完全に無視してきました。また、Citrixのお客様は、ほぼすべてのドイツの大規模および中規模の企業と、ほぼすべての政府機関です。まず、脆弱性は州の構造に影響を及ぼしました。

答えよりも多くの質問。


Citrixの多数のパートナーであるプラチナとゴールドは何をしたのでしょうか。2020年の第3週にのみ必要な情報が一部のCitrixパートナーのページに表示されたのはなぜですか？明らかに、高給の外部コンサルタントもこの危険な状況を寝過ごしました。私は誰も気分を害したくはありませんが、パートナーの仕事はまず第一に、発生する問題を防ぐことであり、それらを排除するための支援を提供することではありません。

実際、この状況は、ITセキュリティの分野での実際の状況を示していました。企業のIT部門の従業員とCitrixのパートナー企業のコンサルタントの両方が1つの真実を明確にする必要があります。脆弱性がある場合、それを排除する必要があります。まあ、重大な脆弱性はすぐに修正する必要があります！
