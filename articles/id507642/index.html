Pure Function - Fungsi yang deterministik dan tidak memiliki efek samping.



Artinya, agar suatu fungsi menjadi murni, ia harus

bersifat deterministik - yaitu, setiap kali dengan serangkaian argumen yang sama menghasilkan hasil yang sama.

Contoh Fungsi yang Ditentukan 

def sum(a,b):
  return a+b

print(sum(3,6))
print(sum(3,6))
print(sum(3,6))

>>>9
>>>9
>>>9


Dan contohnya tidak deterministik 



from datetime import date

todays_weekday = date.today().weekday()
def sum(a,b):
  if todays_weekday == 1:
    result = a+b
    print(f'sum is {result} and result class is {type(result)}')
  else:
    result = str(a+b)
    print(f'sum is {result} and result class is {type(result)}')

sum(4,5)
todays_weekday = 4
sum(4,5)

>>>sum is 9 and result class is <class 'int'>
>>>sum is 9 and result class is <class 'str'>


Setiap kali Anda mengubah hari dalam seminggu (yang bukan merupakan argumen untuk fungsi), fungsi menghasilkan hasil yang berbeda.



Contoh yang paling jelas dari fungsi non-deterministik adalah acak 



import random
print(random.random())
print(random.random())
print(random.random())

>>>0.5002395668892657
>>>0.8837128676460416
>>>0.5308851462814731




Kualitas penting kedua dari fungsi murni adalah kurangnya efek samping yang

akan terlihat jelas 

my_list = [32,3,50,2,29,43]

def sort_by_sort(li):
  li.sort()
  print(li)

sort_by_sort(my_list)
print(my_list)

>>>[2, 3, 29, 32, 43, 50]
>>>[2, 3, 29, 32, 43, 50]


Fungsi sort_by_sort memiliki efek samping karena ia mengubah daftar elemen asli dan menampilkan sesuatu di konsol. 



my_list = [32,3,50,2,29,43]

def sort_by_sorted(li):
  return sorted(li)

print(sort_by_sorted(my_list))
print(my_list)

>>>[2, 3, 29, 32, 43, 50]
>>>[32, 3, 50, 2, 29, 43]


Tidak seperti contoh sebelumnya, fungsi sort_by_sorted tidak mengubah larik asli dan mengembalikan hasilnya tanpa mengeluarkannya sendiri ke konsol.



Fungsi murni bagus karena

- Mereka lebih mudah dibaca

- Mereka lebih mudah untuk dipelihara

- Mereka lebih mudah untuk diuji

- Mereka tidak tergantung pada urutan di mana mereka dipanggil



Pemrograman fungsional dapat dicirikan oleh fakta bahwa ia hanya menggunakan fungsi murni



Fungsi tingkat tinggi - dalam pemrograman, fungsi yang mengambil fungsi lain sebagai argumen atau mengembalikan fungsi lain sebagai hasilnya.





Contoh FVP 



def func(num):
  return num**2

def higher_order_func(fun, num):
  return fun(num)+fun(num)

print(func(4))
print(higher_order_func(func,4))

>>>16
>>>32




Dengan dasar-dasar sedikit disortir dan sekarang beralih ke langkah berikutnya



Jadi, mari kita mulai



Pertama, Anda perlu memahami yang berikut - apa itu Pemrograman Fungsional secara umum. Secara pribadi, saya tahu dua paradigma yang paling sering disebutkan dalam pemrograman sehari-hari - OOP dan FP.



Jika Anda menyederhanakannya sepenuhnya dan menjelaskannya dengan jari-jari Anda, maka Anda dapat menggambarkan dua paradigma ini sebagai berikut:



  • OOP adalah Pemrograman Berorientasi Objek - pendekatan untuk pemrograman, saat menggunakan objek mana yang dapat dilewatkan sebagai parameter dan digunakan sebagai nilai.
  • Menggunakan logika ini, adalah mungkin untuk menetapkan bahwa FP adalah pendekatan untuk pemrograman, ketika menggunakan fungsi mana yang dapat ditransfer ke fungsi lain sebagai parameter dan digunakan sebagai nilai yang dikembalikan oleh fungsi lain ... Jawabannya tersembunyi dalam nama itu sendiri.




Seperti kata guru favorit saya zverokVictor Shepelev: ‚ÄúSemua pekerjaan dalam pemrograman adalah bekerja dengan data. "Dia mengambil beberapa data, bermain dengan mereka dan mengembalikannya."



Ini juga berlaku untuk FP - mengambil beberapa data, mengambil beberapa fungsi, bermain dengan mereka dan memberikan sesuatu.



Saya tidak akan mengecat semuanya, kalau tidak, akan lama sekali. Tujuan artikel ini adalah untuk membantu memahami, dan tidak menjelaskan bagaimana dan apa yang berhasil, jadi di sini kita melihat fungsi utama dari FP.



Untuk sebagian besar, FP (seperti yang saya rasakan) hanyalah penulisan kode yang disederhanakan. Setiap kode yang ditulis dalam gaya fungsional dapat dengan mudah ditulis ulang dengan gaya biasa tanpa kehilangan kualitas, tetapi lebih primitif. Tujuan FP adalah untuk menulis kode yang lebih sederhana, lebih mudah dimengerti, dan mana yang lebih mudah untuk dipelihara, dan juga yang memakan lebih sedikit memori, dan di mana tanpa itu - tentu saja, moral abadi utama pemrograman adalah KERING (Jangan Ulangi Diri Sendiri - Jangan Ulangi) .

Fungsi lambda



Lambda adalah alat dalam python dan bahasa pemrograman lain untuk memanggil fungsi anonim. Kemungkinan besar ini tidak akan mengatakan apa-apa dan tidak akan menjelaskan bagaimana cara kerjanya, jadi saya akan memberitahu Anda hanya mekanisme ekspresi lambda.



Semuanya sangat sederhana.



Pertimbangkan sebuah contoh. Sebagai contoh, kita perlu menulis fungsi yang akan mempertimbangkan luas lingkaran dengan jari-jari yang diketahui.



Rumus area lingkaran adalah di



S = pi*(r**2)







mana

S adalah area lingkaran

pi - konstanta matematika sama dengan 3.14 yang akan kita dapatkan dari perpustakaan standar Matematika

r - jari-jari lingkaran adalah satu-satunya variabel yang akan kita lewati untuk fungsi kita







Sekarang mari kita format semuanya dengan python 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

#              
def area_of_circle_simple(radius):
  return pi_const*(radius**2)

print(area_of_circle_simple(5))
print(area_of_circle_simple(12))
print(area_of_circle_simple(26))
>>>78.5
>>>452.16
>>>2122.64




Tampaknya bagus, tetapi semuanya bisa terlihat jauh lebih keren jika Anda merekamnya melalui lambda 



import math #  math

pi_const = round(math.pi, 2) # pi    
#       
#  3.141592653589793     

print((lambda radius: pi_const*(radius**2))(5))
print((lambda radius: pi_const*(radius**2))(12))
print((lambda radius: pi_const*(radius**2))(26))

>>>78.5
>>>452.16
>>>2122.64


Untuk membuatnya lebih jelas, panggilan fungsi anonim menyiratkan bahwa Anda menggunakannya tanpa menyatakannya di mana saja, seperti dalam contoh di atas.



Fungsi Lambda bekerja sesuai dengan prinsip berikut 




print((lambda     :     )( ))

>>>




Pertimbangkan contoh dengan dua argumen input. Misalnya, kita perlu menghitung volume kerucut menggunakan rumus berikut: 



V = (height*pi_const*(radius**2))/3






Mari kita tulis semuanya dengan python: 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

#      
def cone_volume(height, radius):
  volume = (height*pi_const*(radius**2))/3
  return volume

print(cone_volume(3, 10))

>>>314.0


Dan sekarang bagaimana tampilannya dalam bentuk lambda: 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

print((lambda height, radius : (height*pi_const*(radius**2))/3)(3, 10))

>>>314.0




Jumlah variabel di sini tidak terbatas. Misalnya, kami menghitung volume kerucut terpotong, di mana kami memperhitungkan 3 variabel berbeda.



Volume kerucut terpotong dihitung dengan rumus: 



V = (pi_const*height*(r1**2 + r1*r2 + r2**2))/3






Dan di sini adalah bagaimana itu akan terlihat dalam python secara klasik 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

#      
def cone_volume(h,r1,r2):
  return (pi_const * h * (r1 ** 2 + r1 * r2 + r2 ** 2))/3

print(cone_volume(12, 8, 5))
print(cone_volume(15, 10, 6))
print(cone_volume(20, 12, 9))

>>>1620.24
>>>3077.20
>>>6970.8




Sekarang mari kita tunjukkan bagaimana tampilannya dengan lambda: 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

print((lambda height, radius1, radius2 : (height*pi_const*(radius1**2 + radius1*radius2 + radius2**2))/3)(12, 8, 5))
print((lambda height, radius1, radius2 : (height*pi_const*(radius1**2 + radius1*radius2 + radius2**2))/3)(15, 10, 6))
print((lambda height, radius1, radius2 : (height*pi_const*(radius1**2 + radius1*radius2 + radius2**2))/3)(20, 12, 9))

>>>1620.24
>>>3077.20
>>>6970.8
