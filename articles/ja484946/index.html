GPRモデリング

Georadar現在非常に広く使用されている（地下響きの無線技術機器、GPR、地中レーダ）、 -からウサギの穴をマッピングし、トカゲを研究する鉱山の検索は、かなり高価な喜びのまま。


GPR表示（イギリスのテレビ番組「コマンドオブザタイム」のフレーム）

しかし、その能力を評価し、「鉄」デバイスを取得またはレンタルすることなく、GPR電磁界と環境との相互作用のさまざまな側面を研究できます。gprMaxのパッケージ（GPR - GPRの略語から、マックス -電気力学の基礎を築いたジェームズ・クラーク・マクスウェルの名前の頭文字、）、GNU GPL v3のライセンスの下で配布さは、私たちを助けます。1996年に始まったこのプロジェクト
の作成者は、ノーサンブリア大学のクレイグウォーレンとエジンバラ大学のアントニスジャンノプロスです。パッケージは元々Cで開発された後、Python 3 / Cythonの組み合わせで完全に書き直されました。


パッケージをインストールするには、OpenMP（Microsoft Visual C ++ 2015ビルドツール（このバージョンをお勧めします！）for Windows / gcc for Linux）、NumPyライブラリ、およびCythonコンパイラをサポートするインストール済みコンパイラが必要です。GitHub のリポジトリからダウンロードし、プロジェクトのソースコードを解凍した後、ルートフォルダーに移動してコマンドを実行します。

python setup.py build
python setup.py install

「クイックスタート」として、単純な2次元の例を使用してパッケージを操作することを検討します。パルスレーダーの送信アンテナT（インパルスGPR）は電磁パルスを放出し、そのエネルギーの一部は直接波（DW-直接波）の形で受信アンテナRに直接到達し、一部は透過します。砂を通して、それは導電性シリンダーの表面から反射され、反射波（RW-反射波）の形で受信アンテナに到達します。

アンテナ

シミュレートされたGPRには、送信と受信の2つのアンテナが装備されています。
私たちのケーススタディでは、長さがサンプリングステップと等しいヘルツダイポールを備えた送信アンテナを想像してください（3次元の場合は、広範囲のライブラリからアンテナを選択できます）。領域の中央の左5 cmの距離（x = 35）に砂の中（プローブされた媒体と接触して動作）にあります。 cm = 0.35 m、y = 38 cm = 0.38 m）：

#hertzian_dipole: z 0.35 0.380 0.0 pulse

（zは双極子分極軸です（2次元の場合（2D TMzモード）はzのみが許容されます）、pulseはアンテナによって放射されるパルスの形状マークです）

受信アンテナは通常、アンテナユニットのベースと呼ばれる受信アンテナから小さな一定の距離にあります（これは相互の変形です）アンテナの位置は「共通オフセット」と呼ばれます。基準として、10 cmの距離を選択します。水平座標は35 + 10 = 45 cm = 0.45 m（領域の中央の右5 cm）です。

#rx: 0.45 0.38 0.0

モデリング

したがって、入力ファイルの内容は次のとおりです。

hello.in
#domain: 0.80 0.60 0.002
#dx_dy_dz: 0.002 0.002 0.002

#waveform: gaussian 1 1e9 pulse
#hertzian_dipole: z 0.35 0.380 0.0 pulse
#rx: 0.45 0.38 0.0
#box: 0 0 0 0.80 0.38 0.002 sand
#cylinder: 0.40 0.1 0 0.40 0.1 0.002 0.06 pec


モデリングプロセスを開始します。

python -m gprMax models\hello.in

シミュレーションを実行するために、有限時間領域法（FDTD、有限時間領域）が使用されます（この方法の基本アルゴリズムはKane Yeeによって提案されました）。その後、モデルが分割されたセルはYeeセルと呼ばれます。 。各セルのマクスウェル方程式を解くことにより、時間領域で数値解が得られます。

2次元の場合（2D TMzモード）、コンポーネントのみが計算されますEz 電界とコンポーネント Hx そして Hy磁場。

使用可能なメモリの量を超えると、メモリ不足についてのメッセージが表示され、シミュレーションが実行されます。


オブジェクトのいずれかがシミュレーションの範囲外にある場合、エラーメッセージが表示されます。


記述されたモデルでシミュレーションを実行するには、約56 MBのRAMしか必要としませんでした（ステップを半分（最大1 mm）削減すると、メモリ要件は99 MBに増加します）。

シミュレーションが完了すると、hello.outファイルがmodelsフォルダーに表示され、数値データを格納するために特別に設計されたHDF5形式のシミュレーション結果が含まれます。ルートの 構築結果を視覚的に表現するために、ルートを構築します。





python -m tools.plot_Ascan models\hello.out

開いたウィンドウに表示された各トラック（Aスキャン）には、受信アンテナの位置での電磁界の成分の1つの時間グラフが表示されます。


経路上では、送信アンテナ（DW）から直接到来する直接波と、ターゲットから反射された波（RW）が見えます。

時間の横軸は、物質内の電磁波の速度を通じて信号を反射するターゲットの深さに関連しています。

しかし、入力ファイルのboxコマンドの前にシリンダーコマンドを配置するとどうなりますか？


円柱から反射された信号が消えました-砂が円柱を吸収しました（これは、オブジェクトが構築される順序の重要性の例です）。

プロファイルの作成

しかし、レーダーレーダー（レーダーグラム）-プロファイル（B-スキャン）は、特定の方向に沿ってジオレーダーを移動したときに作成された多数のトラックの組み合わせです-これは、調査エリアに沿ってレーダーでトロリーを移動するまさにその手順です。

アンテナを水平軸の始点に移動して、アンテナの説明を変更します。

#hertzian_dipole: z 0.10 0.38 0.0 pulse
#rx: 0.20 0.38 0.0

アンテナを移動するステップを1 cm = 0.01 mに設定します。

#src_steps: 0.01 0 0
#rx_steps: 0.01 0 0

したがって、入力ファイルの内容は次のとおりです。

hello.in
#domain: 0.80 0.60 0.002
#dx_dy_dz: 0.002 0.002 0.002
#time_window: 5e-9
#material: 3 0.01 1 0 sand
#waveform: gaussian 1 1e9 pulse
#hertzian_dipole: z 0.10 0.38 0.0 pulse
#rx: 0.20 0.38 0.0
#src_steps: 0.01 0 0
#rx_steps: 0.01 0 0
#box: 0 0 0 0.80 0.38 0.002 sand
#cylinder: 0.40 0.1 0 0.40 0.1 0.002 0.06 pec


シミュレーションをバッチモードで実行します。

python -m gprMax models\hello.in -n 50

（50はレーダーが動くステップ数です）。

開始後、シミュレーションは50個のGPR位置に対して順次実行されます。


シミュレーションが完了すると、モデルフォルダーに50個のファイルhello1.out ... hello50.outがあります。
次のコマンドを使用して、これらのファイルをhello_merged.outファイルに結合します。

python -m tools.outputfiles_merge models/hello

プロファイルを作成します。

python -m tools.plot_Bscan models\hello_merged.out Ez

（Ezは、プロファイルを作成する電磁界のコンポーネントです。コンポーネントは直接電圧に変換されます）


ご覧のとおり、上のプロファイルには直接波による水平バーが表示され、下にはターゲットからGPRまでの距離の変化を反映した反射波による特徴的な双曲線が表示されます。

右側の凡例は、一致する色とフィールドの強さを示しています。Ez（トラックに表示）：
赤-Ez>0
白い - Ez=0
青い - Ez<0
このようなプロファイルを分析することで、ターゲットの深さ、サイズ、さらには形状について結論を出すことが可能であり、ニューラルネットワークもこれに使用されます。

見てわかるように、直接波（DW）の「トレース」の下に、2つの砂層の境界面から反射された波（RW）の線形セグメントが現れました。

この例の範囲外では、電磁波の分散を考慮した3次元モデリング、複雑な表面形状、詳細なアンテナモデルなどのgprMax 機能 ...さらに、このパッケージは、GPRをシミュレートするだけでなく、さまざまな環境での電磁波の伝播を研究するためにも使用できます。NVIDIA CUDAテクノロジーを使用して計算を高速化します。これにより、OpenMPを使用するCPUでの並列計算と比較して、速度が10倍向上します。モデリングの柔軟性を高めるために、Pythonのコードブロックを入力ファイルに配置できます。

gprMaxパッケージの使用例：

  • 対人地雷の探索におけるGPRの有効性に対する土壌特性の影響の評価
  • トウモロコシの種を植える深さの非破壊的な決定のために地中レーダーを使用する可能性をチェックする
  • 高速道路を通る地下通路の構造の強さの研究におけるGPRフィールドデータの解釈の質の向上
  • 反射信号の振幅とコンクリート中の鉄筋の直径との関係の研究
  • 木の幹の空隙を検出するためのGPRの適合性の評価
