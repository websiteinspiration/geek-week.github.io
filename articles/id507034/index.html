Webstorm NodeJs interpreter dari wadah Docker

Artikel ini menjelaskan proses penggunaan aplikasi NodeJs menggunakan juru bahasa NodeJs yang terletak di wadah Docker sebagai proyek (secara default). Solusinya kemungkinan besar hanya akan relevan bagi pengguna OS Linux, karena pada akhirnya interpreter akan diambil dari gambar node resmi pada DockerHub .


Untuk apa?


Aplikasi ini dapat digunakan dengan hanya buruh pelabuhan pada mesin host, tidak perlu menginstal sendiri juru bahasa, yang akan memungkinkan pengembang untuk memiliki versi tertentu dari juru bahasa untuk proyek tertentu, dan ketika mengubah versi gambar dalam proyek, pengembang tidak perlu mengkonfigurasi secara manual, contoh baru juru bahasa akan digunakan dalam memproyeksikan secara otomatis.


Keputusan


Misalnya, aplikasi dibuat pada ReactJs + Typescipt, karena jika penerjemah tidak terkonfigurasi dengan benar, setidaknya penyorotan kode akan rusak.


1. Buat aplikasi dari template yang sudah selesai:


$ npx create-react-app reactjs-typescript-custom-nodejs --template typescript

2. Tulis Dockerfile:


FROM node:14
WORKDIR /app

CMD cp /usr/local/bin/node /app/docker/node \
    && npm install \
    && npm run start

CMD, Docker /app/docker/node, /app . .


3. node docker/node .gitignore, :


*

4. :


docker build -t client . && docker run -it --rm --volume=${PWD}:/app -p 3000:3000 --name client client
