問題の発生：なぜ誰もが正規化する必要があるのか​​？


無視するだけで十分な場合、実験の測定は次のように説明できます。プロセスのスペクトルまたは信号をキャプチャし、測定結果に応じていくつかの数値を表示する特定のデバイスがあります。これらの数値を見てデバイスの構造を理解する研究者としての私たちの仕事は、測定されたスペクトルまたは信号が何であったかを理解することです。つまり、いわゆる逆問題に直面します。これを数学的に想像すると、次の方程式が得られます（これは、偶然にも、第1種のフレドホルム方程式と呼ばれます）。

f(y)=∫abdxK(x,y)φ(x)

実際、この方程式は次のことを表しています。ここでは、測定デバイスはハードウェア関数で表されています。 K(x,y)、これは調査されたスペクトルまたは他の入力信号に作用しますφ、これにより研究者は出力信号を観察しますf(y)。研究者の目標は信号を復元することですφ既知の φf(y)およびK(x,y)。この式を行列形式で定式化して、関数をベクトルと行列に置き換えることもできます。

fm=Kmnφn

フレドホルム方程式と線形方程式系（過決定であっても）の両方が正確な解を持っているため、信号の再構成は難しい作業ではないように思われます。それでは、試してみましょう。測定された信号を2つのガウスの合計として説明します。

φ(x)=2∗N(2,0.16)+N(4,0.04)

計測器として、最も単純な積分器を使用します。これは、Heaviside関数を使用して信号を累積和に変換する行列です。

Kmn=θ(xm−yn)

測定された信号の種類とデバイス、および測定結果がグラフに表示されます。

実際の測定には誤差があることが重要です。そのため、通常のノイズを追加して5％の測定誤差を与えると、結果が少し損なわれます。

しかし、最初に、この失敗の原因を突き止めましょう。明らかに、ポイントは測定誤差ですが、それらは何に影響しますか？事実は、ジャックアダマール（コーシー-アダマールの式にダッシュを追加したのと同じもの）がすべてのタスクを正しいポーズと正しくないタスクに分けたことです。

古典を思い出す：「ソリューションがあるとしても、それを探すことは意味がありません。解決策のない課題への対処方法です。これは非常に根本的な質問です...」-正しいタスクについては説明せず、すぐに間違ったタスクを取り上げます。幸いなことに、これはすでに満たされています。上記のフレドホルム方程式は正しくない逆問題です。入力データに微小な変動があっても（そして測定誤差も無限小ではありません）、正確な分析方法で得られた方程式の解は、真の方程式とは恣意的に異なる場合があります。

この声明の証明は、学者A.N.の古典作品の最初の章で読むことができます。 Tikhonova「悪質な問題を解決する方法」この本には、間違ったタスクをどうするかに関するヒントがありますが、そこに概説されている手法には、Turchinメソッドで修正された多くの欠点があります。しかし、最初に、誤ったタスクを処理する一般的な原則について説明します。そのようなタスクに遭遇した場合はどうすればよいですか？

タスク自体は何も提供できないため、小さな犯罪を犯す必要があります。タスクをデータで補完して、正しくなるようにします。つまり、タスクに関するいくつかの*事前情報を入力します*（このプロセスは、タスクの正則化と呼ばれます）。古典的なティホノフ法とは対照的に、パラメーター化された正則化汎関数の導入に基づいて、Turchin法はベイズ法を使用します。

例

例として、モジュールstaregを使用して行列と積分方程式のデータを復元する方法を考えます。

必要な科学パッケージをインポートします。

import numpy as np
import matplotlib.pyplot as plt
from scipy.stats import norm
from scipy.integrate import quad
%matplotlib inline

真の信号を決定し、それを復元します。

a = 0
b = 5
#  
phi = lambda x: 4*norm.pdf(x-2, scale=0.4) + 2*norm.pdf(x-4, scale = 0.5)
x = np.linspace(a, b,100)
plt.plot(x, phi(x));


カーネルと関数のたたみ込み演算を定義します（注：np.convolution特に配列の場合）。

kernel = lambda x,y : np.heaviside(x-y, 1) #  
convolution =  np.vectorize(lambda y: quad(lambda x: kernel(x,y)*phi(x), a,b)[0])

測定データを生成し、正規分布を使用してノイズを加えます。

y = np.linspace(a, b, 50)
ftrue = convolution(y)
sig = 0.05*ftrue +0.01 #  
f = norm.rvs(loc = ftrue, scale=sig)
plt.errorbar(y, f, yerr=sig);



積分方程式を解く

離散化のためにソルバーと補助クラスをインポートします。

from statreg.model import GaussErrorUnfolder
from statreg.basis import CubicSplines

離散化の機能的基礎として、3次スプラインを使用し、追加条件として、関数がエッジでゼロ値を取ることを示します。

basis = CubicSplines(y, boundary='dirichlet')
model = GaussErrorUnfolder(basis, basis.omega(2))

次の方程式を解きます。

phi_reconstruct = model.solve(kernel, f, sig, y)

ソリューションスケジュールを作成しています。

plt.plot(x,phi(x))
phir = phi_reconstruct(x)
phiEr = phi_reconstruct.error(x)
plt.plot(x, phir, 'g')
plt.fill_between(x, phir-phiEr, phir + phiEr, color='g', alpha=0.3);



行列方程式を解く

離散化のためにソルバーと補助クラスをインポートします。

from statreg.model import GaussErrorMatrixUnfolder
from statreg.basis import CubicSplines

行列を取得するには、関数ベースを使用しますが、行列はどのような方法でも取得できることは明らかです。

cubicSplines = CubicSplines(y, boundary='dirichlet')
omega = cubicSplines.omega(2)
Kmn = cubicSplines.discretizeKernel(kernel,y)

行列方程式を解きます。

model = GaussErrorMatrixUnfolder(omega)
result = model.solve(Kmn, f, sig)

チャートを作成します。

phir = lambda x: sum([p*bf(x) for p, bf in zip(result.phi,cubicSplines.basisFun)])
plt.plot(x,phir(x))

result = solve(model, Kmn, f, sig)
phivec = PhiVec(result, basis)

x = collect(range(a, stop=b, length=5000))
plot(x, phi.(x))

phi_reconstructed = phivec.phi_function.(x)
phi_reconstructed_errors = phivec.error_function.(x)

plot(x, phi_reconstructed)
fill_between(x, phi_reconstructed - phi_reconstructed_errors, phi_reconstructed + phi_reconstructed_errors, alpha=0.3)
