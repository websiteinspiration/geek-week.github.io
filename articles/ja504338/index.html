自宅でのUSB over IP

ラップトップの隣のテーブルに保持せずに、USB経由で接続されたデバイスを操作したい場合があります。私が持っているそのようなデバイスは、500 mWのレーザーを備えた中国人の彫刻家であり、密接に接触するとかなり不快なものです。目への即時の危険に加えて、レーザーの動作中に燃焼の有毒な生成物が放出されるため、デバイスは換気の良い場所に配置し、できれば人から隔離する必要があります。しかし、そのようなデバイスを管理する方法は？古いD-Link DIR-320 A2ルーターにふさわしいアプリケーションを見つけるためにOpenWRTリポジトリを閲覧しているときに、誤ってこの質問の答えを見つけました。接続については、以前にHabrで説明したUSB over IPトンネルを使用することを決定しましたが、そのインストールに関するすべての手順は関連性を失うことがありました。そのため、自分で作成しました。

OpenWRTは導入する必要のないオペレーティングシステムであるため、インストールのペイントは行いません。彼のルーターについては、07/19/3にOpenWrtの最新の安定版リリースを取得し、メインのWi-Fiアクセスポイントにクライアントとして接続し、ファイアウォールを苦しめないようにlanモードを選択しました。

サーバ側


私たちは公式の指示に従って行動します。ssh経由で接続した後、必要なパッケージをインストールします。

root@OpenWrt:~# opkg update
root@OpenWrt:~# opkg install kmod-usb-ohci usbip-server usbip-client

次に、デバイスをルーターのUSBポート（私の場合、デバイス：USBハブ、ルーターのファイルシステムがマウントされているUSBフラッシュドライブ（内部ドライブのスペースが不足しているため）、および直接彫刻機）に接続します。

接続されているデバイスを一覧表示しようとします。

root@OpenWrt:~# usbip list -l

空っぽです。

犯人はグーグルで発見され、libudev-fbsdライブラリであることが判明しました。OpenWRTリリース17.01.7から
最新の作業バージョンのlibudev_3.2-1を、アーキテクチャーの下のリポジトリーから手動でプルします。私の場合はlibudev_3.2-1_mipsel_mips32.ipkです。wget / scpを使用して、ルーターのメモリにロードし、再インストールします

root@OpenWrt:~# opkg remove --force-depends libudev-fbsd
root@OpenWrt:~# opkg install libudev_3.2-1_mipsel_mips32.ipk

私たちはチェックします：

root@OpenWrt:~# usbip list -l
 - busid 1-1.1 (090c:1000)
   Silicon Motion, Inc. - Taiwan (formerly Feiya Technology Corp.) : Flash Drive (090c:1000)

 - busid 1-1.4 (1a86:7523)
   QinHeng Electronics : HL-340 USB-Serial adapter (1a86:7523)

USBハブに接続されている中国人の男はbsuid受け取っ1から1.4を。覚えておいてください。

デーモンを実行します。

root@OpenWrt:~# usbipd -D

そして中国人の束縛

root@OpenWrt:~# usbip bind -b 1-1.4
usbip: info: bind device on busid 1-1.4: complete

すべてが機能することを確認します。

root@OpenWrt:/home# netstat -alpt
Active Internet connections (servers and established)
Proto Recv-Q Send-Q Local Address           Foreign Address         State       PID/Program name
tcp        0      0 0.0.0.0:3240            0.0.0.0:*               LISTEN      1884/usbipd

デバイスをさらに自動的にバインドするには、/ etc/rc.localを編集し、exit 0の前に以下を追加します。

usbipd -D &
sleep 1
usbip bind -b 1-1.4

クライアント部分


上記のopenwrt.orgの手順を使用して、デバイスをWindows 10に接続してみましょう。私はすぐに言わなければなりません：その考えは失敗する運命にあります。まず、Windows 7 x64のみが考慮されます。次に、sourceforge.netのスレッドへのリンクがあり、ドロップボックスから2014年にパッチされたドライバーをダウンロードすることを提案しています。 Windows 10で実行してデバイスに接続しようとすると、エラーが発生します。

c:\Utils\usbip>usbip -a 192.168.31.203 1-1.4
usbip for windows ($Id$)

*** ERROR: cannot find device

これは、バージョン3.14より前のカーネル用に構築されたサーバーではクライアントが機能しないためです。
OpenWRT 07.19.3の下のusbipサーバーは、4.14.180カーネル上に構築されています。

検索を続けると、githubでのWindowsクライアントの現在の開発に遭遇しました。 OK、Windows 10 x64のサポートが要求されていますが、クライアントは専らテスト用であるため、いくつかの制限があります。

そのため、最初に証明書をインストールするように依頼され、さらに2回インストールされます。 OK、それを信頼されたルート証明機関と信頼された発行元に配置します。

次に、オペレーティングシステムをテストモードにする必要があります。これはチームによって行われます

bcdedit.exe /set TESTSIGNING ON

私が初めて成功しなかったとき、セキュアブートはそれを妨げました。これを無効にするには、UEFIで再起動し、セキュアブート-無効を設定する必要があります。一部のラップトップモデルでは、スーパーバイザーパスワードのインストールが必要な場合があります。

その後、Windowsを起動し、bcdedit.exe / set TESTSIGNING ONを実行
します。もう一度再起動すると、右下隅にテストモード、OSのバージョンとビルド番号が表示されます。

これらすべての操作は何のためですか？署名されていないUSB / IP VHCIドライバーをインストールするには。これを行うには、ファイルusbip.exe、usbip_vhci.sys、usbip_vhci.inf、usbip_vhci.cer、usbip_vhci.catをダウンロードし、管理者として実行することをお勧めします。

usbip.exe install

または2番目の方法では、レガシーハードウェアを手動モードでインストールします。私は2番目のオプションを選択し、署名されていないドライバーのインストールに関する警告を受け、それに同意しました。

次に、次のコマンドを実行して、リモートUSBデバイスに接続できることを確認します。

usbip.exe list -r <ip  >

デバイスのリストを取得します。

c:\Utils\usbip>usbip.exe list -r 192.168.31.203
usbip: error: failed to open usb id database
Exportable USB devices
======================
 - 192.168.31.203
      1-1.4: unknown vendor : unknown product (1a86:7523)
           : /sys/devices/ssb0:1/ehci-platform.0/usb1/1-1/1-1.4
           : unknown class / unknown subclass / unknown protocol (ff/00/00)

usbipエラーに注意 を払いません：エラー：usb idデータベースを開くことができませんでした。操作には影響しません。

今すぐbindimデバイス：

c:\Utils\usbip>usbip.exe attach -r 192.168.31.203 -b 1-1.4

Windowsが新しいデバイスを発見したので、ラップトップに物理的に接続されているかのように操作できます。

中国の刻印機で少し苦労しました。刻印機に取り付けられたインストーラー（そう、Arduinoの刻印機）からドライバーCH341SERをインストールしようとすると、USB / IP VHCIがWindowsをBSODに落としたからです。ただし、usbip.exe 経由でデバイスを接続する前に CH341SERドライバーをインストールすると、問題は解決しました。

結論：彫刻家は、窓を開け、ドアを閉めた状態で、キッチンで音を出し、煙を出します。私は、自分のソフトウェアを使用して別の部屋から燃え尽きるプロセスを観察します。汚れたトリックを感じません。
