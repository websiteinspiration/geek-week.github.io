HuaweiでWi-Fiを構築した方法



前の3回の記事（では1、2、3）、我々はジェットインフォシステムズのオフィスで有線ネットワークをアップグレードする方法について話しました。次に、ワイヤレスネットワークのアップグレードについて説明します。

古いネットワークの欠点


私たちのオフィスのワイヤレスネットワークは、別のベンダーの機器の上に構築されました。そして、このネットワークの運用は、ユーザーとシステム管理者の両方に深刻な不満を引き起こしました。

ユーザーは何が気に入らなかったのですか？


ネットワークへの接続は簡単ではありませんでした。ユーザーがデバイスを個別にネットワークに接続できるBYOD（独自のデバイスを使用する）の概念を実装しました。たとえば、ネットワークに接続されているデバイスが従業員の会社のラップトップである場合、彼は会社のリソースとインターネットの両方へのアクセスを提供する必要があります。そして、ユーザーがデバイス（スマートフォン、タブレット）を持ってきた場合は、インターネットへのアクセスのみを許可する必要があります。この可能性にもかかわらず、個別に対処しなければならない状況が多かったため、ワイヤレスネットワークのサポートには多くの管理作業が必要でした。さらに、標準化されたジョブがない場合の断片化されたプラットフォームの多くのニュアンス。その結果、すべての側面を解決するためのリソースコストは完全に非実用的であることが判明しました。すべてを再構築する方が論理的でした。

管理者はWi-Fiネットワークについてどのような不満を持っていますか？


接続に関するさまざまな問題や機器の設定ができないなどのユーザーから膨大な数のアプリケーションが生まれました。さらに、2012〜2013年頃に古いネットワークが作成され、それ以降、規格が変更され、速度が向上しています。
Huaweiのアーキテクチャと機器を使用して既存のネットワークを完全に新しいものに置き換えるよりも、既存のネットワークをアップグレードする方がコストが高くなることを計算して認識しました。彼らはそれを決めました。

ソリューションの技術的基礎として、Huawei AP6050DNおよびAP6052DNアクセスポイントが選択されました。これらは、IEEE 802.11b / g / n / ac規格をサポートし、2.4 GHzおよび5 GHz帯域で動作する最新のデバイスです。また、Huawei AC6605ワイヤレスネットワークコントローラー（より正確には2つのコントローラーをクラスターにまとめたもの）とHuawei Agile Controller Campus（HACC）ハードウェアおよびソフトウェアシステムを使用して、ユーザー認証とアクセス制御を行うことも決定しました。

ラジオ調査




既存のWi-Fiネットワークについて、接続方法だけでなく、無線範囲の質にも不満があったため、無線調査から始めました。従来、これは、測定アダプタと構成済みのAirMagnetソフトウェアがちりばめられたラップトップですべての部屋をバイパスすることによって行われていました。各部屋で、いくつかの測定を行い、ラップトップの位置を固定する必要があります。

Alexander Gulyaevによるナレーションasha77：私たちのオフィスには長い廊下がたくさんあり、自分で測定機器を運ぶのは長くて疲れます。 1日か2日このように見えたので、私たちは疲れていて、もっと技術的なものを作ることにしました。 Googled。 Avitoでは、半死んだスクーターが2500ルーブルで購入されました。ビデオウォールのプロファイルの残骸から、ベースの外観をねじり、内部にラズベリーPiを加えた容赦なく切断されたジャイロスクーターのピースを統合しました。同僚がプログラマーと独創的なドイツ人を調整しました（またはオランダ人ですか？）githubから-ジャイロスコープコントローラーに注がれるソフトウェアで、Raspberry PIからRS232を介してコマンドを制御できます。スマートフォンからコントローラーを制御するためのNode.js上のアプリケーションがPI自体にアップロードされました。それは、オペレーターのためにラップトップを運ぶロボット（Robikと呼ばれる）であることがわかりました。Robikのおかげで、オフィスの詳細な無線調査は通常のバージョンの半分の時間で完了しました。これは、時間がないために事実上何も完了しなかったという事実にもかかわらずです。


信号レベルが非常に良好であることが判明したのは、ロビックの助けがなかったわけではありません。これは、アクセスポイントが通常の密度で適切な場所に配置されたことを意味します。しかし、2.4 GHz帯域の一部は相互に干渉しました。そのため、一部のポイントに高密度のポイントがある場所では、アップグレード後に2.4 GHzを切断して5 GHzのみを使用することにしました。
また、高密度の場所にはAP6050DNではなくAP6052DNシリーズのアクセスポイントを設置しました。それらは、特に多数の加入者がいる場所のためにベンダーによって位置付けられます。AP6052DNには2つのトランスミッタがあり、1つは2つの帯域で動作し、自動的に最適に切り替わります。2つ目のトランスミッタは5 GHzでのみ動作します。

安全性


無線はオープンな環境であり、誰でも聴くことができるため、ワイヤレスアクセスは常に認証メカニズムを意味します。したがって、セキュリティの問題-暗号化、承認-には最大限の注意を払う必要があります。これにより、ワイヤレスネットワークの設計を開始する必要があります。

アクセスには、プロダクションアクセス、ゲストアクセス、および従業員自身のデバイスの接続の3つのタイプがあります。

生産アクセス


本番環境へのアクセスは、企業のラップトップで使用されます。従業員は、有線で接続する場合と同じ企業リソースにアクセスできます。このオプションの業界では、長い間、WPA2標準に準拠した標準の認証スキームが開発されてきました。

承認は、個人ユーザー証明書に従って行われます。以前のネットワークでは、ベンダー承認ソリューションは個別の証明機関であり、独立して生成されたユーザー証明書でした。 「セットアップが完了してすべてが機能する」という原則に基づいて機能しましたが、何か問題が発生した場合、テクニカルサポートは生成された証明書を抽出してユーザーのデバイスに手動で置くことができませんでした。
したがって、私たちは独自のソリューションを持たなくなり、既存のマイクロソフト企業の証明機関と統合することを決定しました。ユーザーのコンピューターにユーザー証明書をインストールするには、3つの方法があります。ADグループポリシーを使用して、ラップトップをIT部門に持ち込み、IT部門の従業員が証明書を発行するか、自動プロビジョニングを使用します。

3番目の方法について話しましょう。使い方？ユーザーがワイヤレスプロダクションネットワークに接続しようとしています。彼がこれを初めて行う場合、彼は自動的に内部ポータルHuaweiにリダイレクトされます。これは、Agile Controller Campus製品を使用して実装され、小さなユーティリティを実行するように提供されています。

このユーティリティは、企業認証局へのHuawei Agile Controller Campusの呼び出しを開始し、個々のユーザー証明書を生成して、ローカルコンピューターの証明書ストアにインストールします。

一般に、すべての作業環境が多かれ少なかれ標準である企業では、Active Directoryを使用するMicrosoftグループポリシーを使用してワークステーションに証明書を配布する方が便利です。しかし、多くのコンピューターがLinuxで実行されているため、これは組織全体では不可能です。したがって、さまざまな国内バージョンのLinuxをインストールした開発者が最も影響を受けます。残念ながら、そのようなOS向けのさまざまな最新のITソリューションが最後に登場します。

上記のすべてに関連して、ワークステーションを手動で構成する機能が常に必要です。

ゲストアクセス




当社の従業員なら誰でもゲストを招待できます。彼がインターネットアクセスを必要とする場合、企業Wi-Fiネットワークに接続すると、システムはそれを自動的に承認ポータル（キャプティブポータル）にリダイレクトします。この場合、ゲストは招待した従業員からユーザー名とパスワードを受け取ります。

しかし、従業員はどこでこのログインをパスワードで取得しますか？以前は、2〜2か月に1度、ITスペシャリストが一連のログインとパスワードを生成し、受付で入手することができました。従業員は誰でも秘書に行き、ネットワークに入るためのデータを要求できます。次に、秘書は情報と技術サポートサービスに、従業員と訪問者に情報が提供されたという情報を含む手紙を送りました。

ご想像のとおり、この回路は不便で時代遅れでした。したがって、新しいネットワークでは、まったく異なるアプローチを採用しました。これで、内部ポータルにセクションがあり、会社名とゲストの携帯電話を入力するだけで十分です。ログインデータが記載されたSMSが電話に送信され、従業員は操作が成功したことの確認を電子メールで受け取ります。移行期間のバックアップオプションとして、通常のスキームには秘書を残しました。

Huawei Agile Controller Campus（HACC）製品には、ゲスト認証ポータル、管理インターフェイス、セルフサービスポータル、SOAPモジュールなど、さまざまな目的で使用するためのいくつかのWebインターフェイスがあります。 SOAPモジュールには、サードパーティソフトウェアからHACCを制御するために使用できるさまざまなAPIがあります。そのため、企業ポータルとHuaweiソリューションとの相互作用のためのインターフェースを作成しました。



ゲストWi-Fiの場合、インターネットへのアクセスと、外部ネットワークに開かれている会社のリソースへのアクセスを提供します。ボットネットと見なされる急流やネットワークトラフィックは許可されません。アダルトリソースへのアクセス、および制限を回避するために設計されたホストへのアクセスは禁止されています。このポリシーのおかげで、ゲストWi-Fiからの異常なトラフィックのバーストは発生しておらず、帯域幅を制限していません。

自分のモバイルデバイス


新しいネットワークでは、EAP-TLSではなく、EAP-TTLSプロトコルを使用して独自のデバイスを認証します。つまり、任意のデバイスに証明書を自動的にインストールすることは非常に難しいため、ユーザー側では証明書は使用されません。私はベンチャーがそれを解決することができないことを提案することに挑戦します。

同時に、EAP-TTLS接続には独自の欠点があります。攻撃者はアクセスポイントを設定し、ユーザーのログインとパスワードハッシュを無線でキャッチしようとする可能性があります。したがって、企業リソースへのアクセスに使用されるログインとパスワードでは、ゲストネットワークに接続できません。

個人のモバイルデバイスを接続するための個別のユーザー名とパスワードを導入しました。これらはゲストのものと同じ原則に従って生成されます。どのユーザーもモバイルデバイスを特別なセルフサービスポータルに登録できます。現在、Huawei Agile Controller CampusのWebフォームの1つを介して実装されています。登録は一定期間有効であり、ISの企業ポリシーに従って必須のパスワード変更の日付とともに終了します。

従業員の1人が自分のラップトップ、タブレット、またはスマートフォンを持参して、インターネットだけでなく企業リソースにもアクセスしたい場合は、これをセキュリティサービスの管理と調整してから、IT部門にアクセスします。企業のラップトップ接続スキームと同様に、別の証明書を生成してインストールします。

フリーランサー




フルタイムの従業員に加えて、Wi-Fiを含むオンラインで仕事をする必要のあるフリーランスの従業員もいます。彼らのために、私たちは正社員のモバイルデバイスを接続するのと同じプロファイルを使用します。これにより、フリーランサー用の個人アカウントを作成する必要がなくなり、企業のActive Directoryシステムにアカウントを入力する必要がなくなり、セキュリティサービスとの複雑な調整に対処する必要がなくなります。通常、頭痛の種ははるかに少なくなります。しかし、同時に、私たちが管理するインターネットアクセスを提供することもできます。

または、ある従業員が個人用のラップトップを購入したとします。彼はすぐにそれを接続したい、最新のアップデートなどを入れたいと思っています。この場合、彼は情報部門とテクニカルサポートに行き、ラップトップのネットワークへの接続を調整する必要はありません。彼は独自にワイヤレスネットワークに接続し、すぐに作業を開始します。また、社内の企業リソースへのアクセスはブロックされていますが、インターネットで更新を確認してください。

ネットワーキング


アクセスポイントは、ユーザーワークステーションと同じアクセススイッチに接続します。それらは別々のVLANに接続され、別々のVRFに設定されます。アクセスポイントは、CAPWAPプロトコルを介してHuawei AC6605 WLANコントローラに接続します。コントローラはフェイルセーフクラスタに統合され、1つのコントローラがプライマリとして機能し、2つ目がバックアップとして機能します。 Huawei Agile Controller Campusは、それらの認証サーバーとして機能します。

その結果、Wi-Fiトラフィックは、コントローラを介してのみアクセスポイント間を行き来します。パフォーマンスの低下は私たちにとって重要ではありませんが、制御性とさまざまな情報保護機能の実装の観点からは、このようなスキームの方がはるかに便利です。
コントローラは、ワイヤレストラフィックをモバイルデバイスからメインネットワークに転送します。異なるSSIDからのトラフィックは、異なるVRFを使用して送信されます。ゲストと従業員のデバイス用に2つ、作業用ネットワーク用に2つ用意しました。

ユーザーは、企業のDHCPサーバーからIPアドレスを受け取ります。以前は、SSIDごとに512台のデバイス用に1つの大きな範囲がありましたが、ゲストアクセス用に4つのIP範囲（それぞれ256台のデバイス）があり、作業用アクセスと私たちのデバイスには8台の範囲（それぞれ256台のデバイス）があります。ユーザーが最初にIPアドレスを受け取る範囲から、オフィス内を移動する際に、このIPアドレスはユーザーによって保存されます。

同時に、私たちが選択したネットワークアーキテクチャには欠点があります。監視する制御点が多くあります。特に、ユーザーが接続すると、デフォルトゲートウェイだけでなく、他のIPアドレスもネットワークインフラストラクチャからユーザーに表示されます。ゲストアクセスの場合、これは情報セキュリティの観点からはあまり良くありません。ゲストが利用できるインフラストラクチャに関する情報が少ないほど良いです。



ゲストネットワークでのこれらの問題を解決するために、次の基本的なセキュリティ対策を採用しました。

  • 接続されたワイヤレスデバイス間のトラフィックの転送を除外するために、いわゆるプライベートモードをオンにしました。
  • 通常のアクセスリストを使用して、ゲストIP範囲内のルーターであるすべてのネットワークデバイスへのアクセスをブロックしました。

Huawei Agile Controller Campus


Huawei Agile Controller Campusは、ハードウェアとソフトウェアの複合体です。ソフトウェアは、Tomcatフレームワークを使用してJavaで記述され、Windows Server 2012を実行しているサーバーで実行されます（SUSE Linuxでも動作します）。 Microsoft SQL ServerまたはOracleをデータベースとして使用できます。

オフィスネットワークの再構築プロジェクトを開始したとき、Huaweiは物理バージョンでのみHACCを提供していたため、3台ものサーバーを購入する必要がありました。それらの1つはSystem Managerと呼ばれ、データベースとすべてのWebインターフェースが含まれています。他の2つのサーバーは互いに同等に予約します。これらはシステムコントローラーです。ユーザーの接続および承認時に、WLANコントローラーが（RADIUSプロトコルを介して）対話するのは、これらのコントローラーです。つまり、WLANコントローラーは、利用可能なシステムコントローラーに毎回、そのような資格情報を持つそのようなユーザーがWi-Fiにアクセスできるかどうかを尋ねます。

私たちがプロジェクトを実装している間、ファーウェイはサーバーを仮想化する機能を提供しました。もちろん、すぐにこれを利用しました。これで、3つの物理サーバーではなく、2つの仮想マシンができました。これにより、特にフォールトトレランス、System Managerバックアップ、データベースバックアップなどの多くの問題が即座に解消されました（仮想マシンはスナップショットを使用してコピーされ、仮想化ファームによってバックアップされるため）。

SMSニュースレター



最後に、ゲストアクセス用のログインとパスワードを使用したSMS配布メカニズムについて少し説明します。物理HACCサーバーで完成したのは、シリアルポートを介して接続するGSMモデムでした。残念ながら、COMポートを仮想マシンに接続することはできませんが、解決策が見つかりました。オフィスのアクセス制御に携わるエンジニアが、シリアルポートからのトラフィックをイーサネットでラップできる特別なゲートウェイを購入していることがわかりました。

EverFocus EA-LAN1モデルを使用しました。通常、これらは電子パスのリーダーを接続するために使用されますが、中国のGSMモデムの場合、このゲートウェイは完全に適合します。ゲートウェイには仮想シリアルポートドライバーが接続されており、HACCは物理的なドライバーと同じように機能します。そのため、仮想マシンからのSMSの送信をシームレスに実装しました。
