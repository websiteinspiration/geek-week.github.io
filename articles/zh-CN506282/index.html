.NET Core与Node.js



本文的目的不是选择这两个平台中的最佳平台，而是寻找它们各自的优点和缺点。两种技术在Web开发领域均已确立。他们每个人都有自己的粉丝，这些粉丝获得了不错的成绩，并在互联网上分享了他们的成就。网络上有很多关于开发人员在切换到.Net Core或Node.js时所获得的好处的评论。让我们尝试找出答案。

一点背景


自2015年以来， 我一直在一家从事自定义开发的公司工作，首先我们在.NET上编写，然后切换到.NET Core，然后逐渐将堆栈更改为Node.JS。主要原因是发展速度。

一般信息




.Net Core

.NET Core是用于创建跨平台应用程序的开源通用框架。允许您为各种操作系统Windows，macOS和Linux创建应用程序。支持 x64，x86，ARM32和ARM64处理器。首次正式发布于2016年6月27日。

Node.js

根据官方网站，Node.js是基于V8构建的软件平台。反过来，V8是Google的开源JavaScript引擎（用C ++编写），可将JavaScript转换为机器代码。 Node.js允许您在macOS，Linux，SmartOS，FreeBSD，Microsoft Windows，AIX和Android上创建应用程序。第一个版本是2009年。

程式语言




使用Net Core

Mostly C＃-Microsoft在1998-2001年开发的一种面向对象的编程语言。严格的类型允许您在编译阶段捕获错误，从而获得更快的反馈循环，并以更少的错误更快地开发应用程序。使用关键字“ dynamic”可以模拟动态类型。

Node.js

JavaScript是一种具有动态类型的解释型，面向对象的语言。它还具有功能语言固有的许多功能。

标准的JavaScript语言是ECMAScript。 JavaScript易于使用，但与C＃不同，许多错误必须在运行时进行调试。为消除此缺点，您可以使用Microsoft在2012年推出的TypeScript。 TypeScript是JavaScript附加组件，与JavaScript向后兼容，并已编译为后者。但是，TypeScript也不能提供C＃提供的强类型的面向对象语言的所有功能。 TypeScript与JavaScript有着千丝万缕的联系，JavaScript具有结构化的代表，因此，它只能像C＃一样模仿类型的工作。

在组装阶段未出现的错误示例：

interface A {
    x: number;
}
 
let a: A = {x: 5}
 
let b: {x: number | string} = a; 
b.x = "unsound";
 
a.x.toFixed(0); // Not a function error occurs.

该代码的问题在于，作为赋值的结果，“ ax”的类型将为字符串，并且错误仅在运行时出现。

模块和工具




Net Core

社区通过NuGet软件包分发工具。当前有205,000个程序包。对于开发而言，最常见的是Visual Studio和Rider，它们可以简化并加快应用程序的开发。

Node.js该

工具包是NPM，具有1,298,879个软件包。尚未宣布正式的开发环境，但已将Visual Studio Code与用于Node.js开发的插件一起使用。
通常，Node.js看起来拥有更多的工具和替代软件包。

可扩展性

Net Core

平台提供了编写多线程应用程序的功能，因此下载单独服务器的任务比Node.js更加容易。对于部署，扩展和弹性服务，Microsoft已发布Azure Service Fabric SDK。您可以在Azure或AWS的服务器上使用此SDK。该文档发布在这里。

Node.js一个

实例仅使用一个线程，因此要加载所有服务器内核，您将必须使用“ Cluster ”模块或外部过程控制。要跨多个服务器扩展，您需要一个负载平衡器（AWS ELB / NGINX）。还有一些工具可以管理服务和流量，例如Istio。

通常，缩放的问题是在两种技术的基础上解决的，这两种技术都为此提供了所有必要的工具。

结论

再次值得注意的是，没有适用于所有场合的通用工具。如果您是初学者，并且想要快速创建具有用户界面和服务器代码的应用程序，那么Node.js将是一个不错的选择。它足以处理JavaScript，并且可以成为服务器端和客户端的一种语言。
