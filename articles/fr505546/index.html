Flutter et développement de bureau

L'idée d'écrire des applications multi-plateformes est loin d'être nouvelle. Flutter offre également la possibilité de le faire. Dans cet article, je vais essayer de décrire deux approches pour lancer une application mobile sur le bureau, que j'utilise moi-même pour développer des applications mobiles. J'ai arrêté d'exécuter l'émulateur et le simulateur, principalement parce qu'il y avait une possibilité de s'en passer. Pour ceux qui sont intéressés par l'idée, bienvenue au chat.

Alors pourquoi prendre la peine d'écrire une version de bureau en contournant les émulateurs et les simulateurs?

Tout d'abord, les ressources du poste de travail . Mon projet commercial actuel utilise beaucoup d'animations et divers widgets «lourds». Sur mon très bon ordinateur portable avec i7, 16 Go de RAM et une carte graphique GTX1050Ti, un processeur avec des transitions et des animations très complexes passe à 30-35% en mode débogage, l'émulateur Android RAM «mange» jusqu'à 2 Go. Les images, bien sûr, sont perdues et les animations se déplacent «saccadées», ce qui ne nous permet pas d'évaluer correctement l'intégralité de l'image. Lorsque la version de bureau est lancée, mon processeur "se repose" et est chargé avec les mêmes animations jusqu'à 3-4% et cela, pendant une minute, enregistre le processeur 8-10 fois et la RAM 300-400MB.

Deuxièmement, la possibilité de visualiser rapidement le placement correct des widgets sur divers appareils. Il suffit de prendre une fenêtre et de l'étirer, ou de la serrer à la taille minimale, ce qui représente une énorme économie de votre temps personnel et du temps des autres testeurs.

La première option pour lancer la version de bureau


Utilisation, désormais intégrée à Flutter - flutter-desktop-embedding.

Tout est simple ici. Nous copions les dossiers windows, linux, macos dans le projet de travail. Nous écrivons dans la console la commande permission pour utiliser les bibliothèques OS:

flutter config --enable-windows-desktop
flutter config --enable-linux-desktop
flutter config --enable-macos-desktop

Tout. Maintenant, lors du choix d'un appareil, un émulateur de bureau avec le nom Linux, Macos, Windows apparaît, selon la plate-forme. Nous sélectionnons, lançons, travaillons.
Nous obtenons la plate-forme identique à l'émulateur avec Hot Reload et Hot Restart.

Exemple de plugin pour émuler les tailles de fenêtre d'émulateur


Le moyen le plus simple consiste à copier le dossier Go à partir du référentiel de projet exemple, par exemple, à partir de github.com/go-flutter-desktop/examples/tree/master/text_demo

Ensuite, dans le répertoire Go, créez les dossiers plugins / desktop_utils et dans celui-ci un fichier avec notre futur plugin, nommons-le main.go et y écrire

code suivant: 
package desktop_utils

import (
	"github.com/go-flutter-desktop/go-flutter"
	"github.com/go-flutter-desktop/go-flutter/plugin"
	"github.com/go-gl/glfw/v3.3/glfw"
)

const channelName = "go-test/desktop_utils"

type DesktopUtilsFlutterPlugin struct {
	window *glfw.Window
	channel *plugin.MethodChannel
}

var _ flutter.Plugin = &DesktopUtilsFlutterPlugin{}
var _ flutter.PluginGLFW = &DesktopUtilsFlutterPlugin{}

func (p *DesktopUtilsFlutterPlugin) InitPlugin(messenger plugin.BinaryMessenger) error {
	channel := plugin.NewMethodChannel(messenger, channelName, plugin.StandardMethodCodec{})
	
	channel.HandleFunc("set_window_size", p.setWindowSizeHandler)

	return nil
}

func (p *DesktopUtilsFlutterPlugin) InitPluginGLFW(window *glfw.Window) error {
	p.window = window

	return nil
}

func (p *DesktopUtilsFlutterPlugin) setWindowSizeHandler(arguments interface{}) (reply interface{}, err error) {
	argumentsMap := arguments.(map[interface{}]interface{})

	var minW = int(argumentsMap["minW"].(int32))
	var minH = int(argumentsMap["minH"].(int32))
	var maxW = int(argumentsMap["maxW"].(int32))
	var maxH = int(argumentsMap["maxH"].(int32))

	p.window.SetSizeLimits(minW, minH, maxW, maxH);

	return nil, nil
}

channelName — Flutter
minW, minH, maxW, maxH — /

Nous initialisons notre fichier en tant que Go-module: go mod init desktop_utils (le fichier go.mod apparaîtra).

Nous allons dans le répertoire Go, trouvons le fichier go.mod, ajoutons le répertoire local de notre plugin: replace desktop_utils => ./plugins/desktop_utils
et ajoutons notre plugin à options.go dans la ligne d'options:

var options = []flutter.Option{
	flutter.WindowInitialDimensions(InitW, InitH),

	flutter.AddPlugin(&desktop_utils.DesktopUtilsFlutterPlugin{}),
}

Sur cette partie Go est terminée.

La partie Flutter est aussi simple que possible: il suffit de créer un canal (MethodChannel) avec notre nom: go-test / desktop_utils et de lui transmettre les paramètres de longueur et de largeur dont nous avons besoin:

const MethodChannel _utilsChannel = MethodChannel('go-test/desktop_utils');

_utilsChannel.invokeMethod(
  'set_window_size',
   <String, dynamic>{
     'minW': 400,
     'minH': 600,
     'maxW': 800,
     'maxH': 1200,
   },
 );

C'est tout. Notre plugin est prêt.
