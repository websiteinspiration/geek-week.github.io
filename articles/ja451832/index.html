プロキシが嘘をついているときを理解する方法：アクティブなジオロケーションアルゴリズムを使用したネットワークプロキシの物理的な場所の検証



世界中の人々が本当の場所やアイデンティティを隠すために商用プロキシを使用しています。これは、ブロックされた情報へのアクセスやプライバシーの確保など、さまざまな問題を解決するために行うことができます。

しかし、サーバーが特定の国にあると宣言した場合、そのようなプロキシのプロバイダーはどの程度正しいのでしょうか。これは根本的に重要な質問であり、個人情報の保護を懸念している顧客に対して特定のサービスをまったく使用することが可能かどうかを決定する答えです。

マサチューセッツ大学のカーネギーメロンとストーニーブルックのアメリカ人科学者のグループは、7つの人気のあるプロキシプロバイダーのサーバーの実際の場所を確認する研究を発表しました。主な結果の簡単な説明を用意しました。

前書き


多くの場合、プロキシオペレーターは、サーバーロケーションステートメントの正確性を確認する情報を提供しません。データベースIP-to-locationは、通常、そのような企業の宣伝論文を確認しますが、これらのデータベースにはエラーの証拠が大量にあります。

調査中、アメリカの科学者は、7つのプロキシ会社によって管理され、合計222の国と地域にある2269のプロキシサーバーの場所を推定しました。分析の結果、すべてのサーバーの少なくとも3分の1が、企業がマーケティング資料で宣言している国に配置されていないことがわかりました。代わりに、安価で信頼できるホスティングを利用できる国、つまりチェコ共和国、ドイツ、オランダ、英国、米国にあります。

サーバーの場所の分析


商用VPNおよびプロキシプロバイダーは、IP-to-locationデータベースの精度に影響を与える可能性があります。たとえば、企業は、ルーターの名前のロケーションコードを操作することができます。その結果、マーケティング資料は、ユーザーがアクセスできる場所を多数宣言することができますが、実際には、費用を節約して信頼性を高めるために、サーバーは少数の国に物理的に配置されていますが、IP-to-locationデータベースはその反対を示しています。

サーバーの実際の位置を確認するために、研究者たちはアクティブな位置情報アルゴリズムを使用しました。これは、サーバー側およびインターネット上の他の既知のホストに送信されるパケットの往復を評価するために使用されました。

同時に、テストされたプロキシの10％未満しかpingに応答せず、科学者はサーバー自体で測定用のソフトウェアを実行できませんでした。プロキシを介してパケットを送信する機能しかなかったため、空間内の任意のポイントへの往復は、パケットがテストホストからプロキシに、およびプロキシから宛先に移動する時間の合計です。



調査中、CBG、オクタント、スポッター、およびハイブリッドオクタント/スポッターという4つのアクティブなジオロケーションアルゴリズムに基づいて、専用のソフトウェアが開発されました。ソリューションコードは GitHub で入手できます。

データベースのIP-to-locationに依存することが不可能だったため、研究者は実験にアンカーホストのRIPE Atlasリストを使用しました-このデータベースの情報はオンラインで入手でき、常に更新され、文書化された場所は正確です。さらに、リストのホストは常にping信号を相互に送信しますパブリックデータベースの往復データを更新します。

科学者によって開発されたこれは、安全でない（HTTPS）TCP接続を安全でないHTTP 80ポートで確立するWebアプリケーションです。サーバーがこのポートをリッスンしない場合、1つの要求後に失敗しますが、サーバーがこのポートをリッスンしている場合、ブラウザはSYN- TLS ClientHelloパケットによるACK応答。これによりプロトコルエラーがトリガーされ、ブラウザにエラーが表示されますが、これは2回目のラウンドトリップの後でのみです。



したがって、Webアプリケーションは1回または2回の往復の時間を測定できます。同様のサービスが、コマンドラインから起動されるプログラムとして実装されました。

テストされたプロバイダーはどれも、プロキシサーバーの正確な場所を提供していません。最良のケースでは、都市が言及されますが、ほとんどの場合、国に関する情報しかありません。都市が言及されている場合でも、インシデントが発生する可能性があります。たとえば、研究者はusa.new-york-city.cfgというサーバーの1つの構成ファイルを調査しました。このファイルには、chicago.vpn-provider.exampleというサーバーへの接続手順が含まれています。したがって、多かれ少なかれ正確に、サーバーのみが特定の国に属していることを確認できます。

結果


アクティブジオロケーションアルゴリズムを使用したテストの結果によると、研究者は2269個のIPアドレスのうち989個の場所を確認できました。642の場合、これは不可能であり、638は、プロキシサービスの保証に従っているべき国には絶対にありません。これらの偽住所の400以上が実際には宣言された国と同じ大陸にあります。正しいアドレスは、サーバーをホストするために最も頻繁に使用される国にあります（画像をクリックするとフルサイズが開きます） テストされた7つのプロバイダーのそれぞれで疑わしいホストが見つかりました。研究者は企業にコメントを求めたが、全員が連絡を拒否した。
