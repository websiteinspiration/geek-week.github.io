Premières réflexions sur Deno

Une traduction de l'article a été préparée avant le lancement du cours Node.js Developer .


Chez Codegram, nous aimons expérimenter les nouvelles technologies et Deno est sorti il ​​y a à peine une semaine! Lisez l'article si vous voulez en savoir plus sur ce nouveau runtime JavaScript et ce que Codegram en pense.

Vous avez peut-être déjà entendu parler de Deno, ou peut-être pas, mais maintenant Internet regorge d'informations à ce sujet et il y a une raison à cela! Un nouveau runtime JavaScript est apparu avec un mignon dinosaure comme logo (certaines personnes pensent que le logo a une chaussette triste, mais non ...). Nous avons même tweeté à son sujet une semaine avant la première sortie, mais pour vous dire la vérité, nous avons entendu parler de Deno pour la première fois il y a deux ans après avoir regardé cette incroyable performance de Ryan Dahl .

Dans cet article, nous allons passer en revue Deno superficiellement et partager nos réflexions à ce sujet. Commençons!

Bonjour Deno!


Deno est un runtime JavaScript comme Node. Pour l'utiliser, vous devez d'abord installer la CLI deno. Pour ce faire, suivez les instructions de votre système d'exploitation sur deno.land (nos compliments de nom de domaine!). Deno n'est pas un fork de Node, il a été créé à partir de zéro sur Rust. J'ai écrit un article sur Rust pour les programmeurs JS , vous pouvez le lire si vous voulez en savoir plus sur la langue!

Après avoir installé la CLI, vous pouvez exécuter le programme classique "Hello, World!" de la manière suivante:

deno run https://deno.land/std/examples/welcome.ts

Sommes-nous en train d'exécuter TypeScript hors de la boîte? Oui! TS est un invité bienvenu chez Deno, vous n'avez donc besoin de rien d'autre pour exécuter le code sur TS. Vous pouvez également exécuter du code JS, mais je suis un grand fan de TS, donc j'ai été le premier à en parler.

Gardez à l'esprit que le code Node n'est pas compatible avec Deno. Cependant, en ce moment, ils travaillent à assurer la compatibilité , afin qu'à l'avenir vous puissiez utiliser vos packages NPM préférés.

Il y a encore une chose à noter que Deno est un bac à sable par défaut. Autrement dit, au départ, vous n'avez pas accès au réseau ou au système de fichiers. Si votre programme a besoin d'un accès, vous devez l'autoriser explicitement (c'est - à - dire utiliser --allow-net pour accorder des privilèges réseau).

Pas plus package.jsonounode_modules
Qui n'a pas vu cette blague sur Internet?



Malgré une légère exagération, il y a encore du vrai dans cette blague. À mesure que votre application grandit, la liste des dépendances s'allonge et vous vous retrouvez avec une énorme arborescence de dépendances. Le problème est que chaque application possède une copie du package @ version que vous utilisez.
L'un des principaux objectifs de Deno est de refléter le navigateur. Lorsque vous souhaitez importer quelque chose dans un navigateur, vous écrivez une balise <script>et utilisez l'URL. Deno fonctionne comme suit:

import { assertEquals } from "https://deno.land/std/testing/asserts.ts";
assertEquals("hello", "hello");
assertEquals("world", "world");
console.log("Asserted! ");

Je suis d'accord, ça a l'air un peu étrange. Comme nous n'avons pas de package.json, il n'y a aucun moyen de spécifier des dépendances externes, nous les importons donc quand elles sont nécessaires. Au premier démarrage, Deno téléchargera le fichier et l'enregistrera dans le cache. Vous pouvez spécifier l'indicateur --reload si vous souhaitez télécharger à nouveau le package.

Vous avez probablement beaucoup de questions sur ce sujet (je les avais aussi), je vous invite donc à visiter cette page FAQ dans le manuel Deno pour obtenir plus d'informations sur le sujet.

Runtime compatible avec l'API du navigateur


Un autre objectif important de Deno était la compatibilité du navigateur, de sorte qu'il dispose d'un ensemble de fonctionnalités disponibles à partir de la portée globale, telles que fetch ou addEventListener . Vous pouvez également utiliser l'objet de fenêtre globale (cependant, pour l'instant, je recommande d'utiliser globalThis). Vous pouvez en savoir plus sur les fonctions globales dans la documentation .

Le runtime dispose également d'un Deno global pour les API qui ne sont pas une norme Web. Vous pouvez l'utiliser pour des opérations de bas niveau telles que la lecture d'un fichier, l'ouverture d'un socket TCP, etc.

Bibliothèque standard riche


Les développeurs de Deno ont créé une collection de modules standard prêts à l'emploi. La bibliothèque est également hébergée sur le domaine deno.land , vous pouvez donc importer n'importe quel module dans votre application à l'aide de l'URL suivante:

import { v4 } from "https://deno.land/std/uuid/mod.ts";

console.log(v4.generate());

La bibliothèque standard comprend de nombreux modules utiles, mais il existe de nombreux modules tiers, et cette liste est mise à jour quotidiennement. Il y a une autre bonne nouvelle: vous pouvez importer n'importe quel module placé sur une URL publique, ce qui signifie que vous pouvez tout aussi facilement importer des modules depuis GitHub. Deno dispose d'un service de réécriture d'URL que vous pouvez utiliser pour rendre vos modules disponibles. Vous pouvez en savoir plus sur ce mécanicien sur cette page .

Outils intégrés


Deno CLI inclut également des outils intégrés pour effectuer certaines tâches courantes, telles que l'exécution de tests, par exemple, ou même la création d'un package! Est-ce à dire que Deno peut remplacer la plaisanterie, la plus jolie et le webpack? Probablement pas. Deno est une solution assez nouvelle et ne dispose pas encore d'un écosystème aussi étendu, mais la présence de tous ces outils, dont l'accès peut être obtenu en installant un binaire, est toujours impressionnante.

Conclusion


Sautez sur cette vague de battage médiatique! Je suis vraiment impressionné par Deno. Après deux ans de développement et un long chemin vers le succès, le nouveau runtime JavaScript a fière allure et je suis impatient de commencer à l'expérimenter. Node est apparu il y a environ 11 ans et est devenu un nouveau jalon dans l'histoire de JavaScript. Il a amélioré le langage et je pense que nous avons franchi une nouvelle étape, il est temps Deno!
