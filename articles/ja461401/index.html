イベントループは何ですか？または、Chromeブラウザーでのイベントループの動作

このコードをブラウザコンソールで実行するとどうなると思いますか？

function foo() {
  setTimeout(foo, 0);
}

foo();

そしてこれ？

function foo() {
  Promise.resolve().then(foo);
}

foo();

私と同じように、イベントループ、メインスレッド、タスク、マイクロタスクなどに関する多数の記事を読んだことがあるが、上記の質問に答えるのが難しい場合、この記事はあなたのためのものです。

それでは始めましょう。ブラウザの各HTMLページのコードは、メインスレッドで実行されます。メインスレッドは、ブラウザがJSを実行し、再描画を実行し、ユーザーアクションを処理するメインスレッドです。基本的に、これはJSエンジンがブラウザーに統合される場所です。

これを理解する最も簡単な方法は、図 を見ることです。


図1

タスクがコールスタックに入り、完了することができる唯一の場所がイベントループであることがわかります。あなたが彼の代わりにいたと想像してください。そしてあなたの仕事は「レーキ」タスクに追いつくことです。タスクには次の2つのタイプがあります。

  1. 個人-サイトでのメインのJavaScriptコードの実行（以下では、すでに実行されていると仮定します）
  2. カスタマータスク-レンダリング、マイクロタスク、およびタスク

ほとんどの場合、個人的なタスクが優先されます。イベントループはこれに同意します:)顧客からのタスクを合理化することは残っています。

もちろん、最初に頭に浮かぶのは、各顧客に優先順位を付けて並べることです。 2つ目は、各顧客からのタスクをどのように処理するかを正確に決定することです。一度に1つずつ、一度に、またはバッチで実行できます。

この図を見てください。


図2

この図に基づいて、イベントループ操作全体が構築されます。
スクリプトの実行を開始すると、このスクリプトを実行したタスクがタスクキューに入れられます。このコードが実行されると、さまざまな顧客からのタスクが発生し、適切なキューに入れられます。スクリプト実行タスクが完了すると（タスクからのタスク）、イベントループはマイクロタスクに移動します（タスクからのタスクの後、イベントループはマイクロタスクからタスクを取得します）。イベントループは、タスクが完了するまでタスクを受け取ります。これは、それらの追加の時間が実行の時間と等しい場合、イベントループはそれらを無限にレーキすることを意味します。
次に、彼はレンダリングに行き、彼からのタスクを実行します。レンダリングのタスクはブラウザによって最適化され、このサイクルで何かを再描画する必要がないと見なされた場合、イベントループはさらに先に進みます。次に、イベントループは再びタスクからタスクを取得し、キューの最初のタスクの1つだけを要求し、それをCallStackに転送して、ループをさらに進みます。

顧客の1人がタスクを持っていなかった場合、イベントループは次のタスクに進みます。逆に、顧客のタスクに時間がかかる場合、残りの顧客は順番を待ちます。そして、ある顧客からのタスクが無限である場合、コールスタックがオーバーフローし、ブラウザーが誓い始めます。


図3

イベントループの仕組みを理解したところで、この記事の冒頭でコードスニペットが実行された後に何が起こるかを理解する時が来ました。

function foo() {
  setTimeout(foo, 0);
}

foo();

関数fooは、内部でsetTimeoutを通じて再帰的に呼び出しますが、呼び出しごとにTasksカスタマータスクを作成しています。記憶しているように、イベントループループでは、タスクからタスクキューを実行すると、ループ内で1つのタスクしかかかりません。そして、マイクロタスクとレンダーからのタスクがあります。したがって、このコードによってイベントループが影響を受けたり、タスクが永久に実行されたりすることはありません。しかし、それは各ラップで顧客タスクの新しいタスクをスローします。

このスクリプトをGoogle Chromeブラウザーで実行してみましょう。これを行うために、簡単なHTMLドキュメントを作成し、このコードを含むscript.jsを接続しました。ドキュメントを開いたら、開発者のツールに移動し、[パフォーマンス]タブを開いて[プロファイリングの開始とページの再読み込み]ボタンをクリックします。


図4

Tasksからのタスクは一度に1つ、約4msに1回実行されることがわかります。

2番目のパズルについて考えてみましょう。

function foo() {
  Promise.resolve().then(foo);
}

foo();

ここでは、上記の例と同じものが表示されますが、fooを呼び出すと、マイクロタスクからタスクが追加され、すべて終了するまで完了します。イベントループがそれらを終了するまで、彼は次の顧客に移動することができません:(そして、我々は悲しい見ること、これは手段絵をもう一度。
