Angularで大きなリストをレンダリングする3つの方法

2020年に、フロントエンドフレームワークはより良く、より効率的でより高速になりました。しかし、これを念頭に置いても、ブラウザを「フリーズ」せずに大きなリストをレンダリングすることは、最速の既存のフレームワークであっても、依然として困難な作業になる可能性があります。

これは、「フレームワークが高速でコードが遅い」ケースの1つです。 Webページでのユーザーの操作をブロックせずに多数の要素を表示できるようにする多くのアプローチがあります。 この記事の著者は、今日翻訳を公開していますが、Webページに大きなリストを表示するための既存の方法を探り、それらのアプリケーションの領域について話したいと考えています。







この資料はAngularを対象としていますが、ここで説明する内容は、純粋なJavaScriptで記述された他のフレームワークおよびプロジェクトに適用されます。特に、大きなリストをレンダリングする次のアプローチがここで考慮されます：

  • 仮想スクロール（Angular CDKを使用）。
  • 手動レンダリング。
  • プログレッシブレンダリング。

仮想スクロール


仮想スクロールは、大きなリストを操作するためのおそらく最も効率的な方法です。確かに、彼にはいくつかの欠点があります。Angular CDKと他のプラグインのおかげで、仮想スクロールはどのコンポーネントにも非常に簡単に実装できます。

仮想スクロールの背後にある考え方は単純ですが、実装が必ずしも容易ではありません。ここで重要なのは、コンテナと要素のリストがあるということです。要素は、それがコンテナーの可視領域内にある場合にのみレンダリングされます。仮想スクロールを整理するために設計されたAngular CDKの

モジュールを使用しますscrolling。これを行うには、最初にCDKをインストールする必要があります。

npm i @angular/cdk

次に、モジュールをインポートする必要があります。

import { ScrollingModule } from '@angular/cdk/scrolling';
@NgModule({
 ...
 imports: [ ScrollingModule, ...]
})
export class AppModule {}

その後、コンポーネントで使用できますcdk-virtual-scroll-viewport：

<cdk-virtual-scroll-viewport itemSize="50">
 <div *cdkVirtualFor="let item of items">
   {{ item }}
 </div>
</cdk-virtual-scroll-viewport>

プログレッシブレンダリング


プログレッシブレンダリングの考え方は簡単です。これは、要素のサブセットを段階的にレンダリングし、イベントループ内の他の要素のレンダリングを延期することで構成されます。これにより、ブラウザは「スローダウン」することなくすべての要素を表示できます。

プログレッシブリストレンダリングを実装する以下のコードは非常に簡単です。

  • まず、これを使用してsetInterval、定期的に10ミリ秒ごとに確立します。呼び出されると500要素をレンダリングする関数を呼び出します。
  • 現在の要素のインデックスの分析に基づいて決定したすべての要素が推定された後、通常の関数呼び出しを停止し、サイクルを中断します。

private buildData(length: number) {
  const ITEMS_RENDERED_AT_ONCE = 500;
  const INTERVAL_IN_MS = 10;

  let currentIndex = 0;

  const interval = setInterval(() => {
    const nextIndex = currentIndex + ITEMS_RENDERED_AT_ONCE;

    for (let n = currentIndex; n <= nextIndex ; n++) {
      if (n >= length) {
        clearInterval(interval);
        break;
      }
      const context = {
        item: {
          id: n,
          label: Math.random()
        },
        isEven: n % 2 === 0
      };
      this.container.createEmbeddedView(this.template, context);
    }

    currentIndex += ITEMS_RENDERED_AT_ONCE;
  }, INTERVAL_IN_MS);

転送される関数の呼び出しごとに表示される要素の数、setIntervalおよびこの関数の呼び出しの頻度は、状況に完全に依存することに注意してください。たとえば、表示されたリストの要素が非常に複雑な場合、一度に500個のそのような要素の出力は許容できないほど遅くなります。

このソリューションのパフォーマンスを測定したところ、以前に受け取った結果よりも悪く見える結果を得ました。コードの実行-907 ms、レンダリング-2555 ms、描画-16 ms。
