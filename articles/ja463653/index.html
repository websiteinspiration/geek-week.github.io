Unityでの16ビットゲームとそのレクリエーションの制限


当社では最初のレトロゲームのチュートリアル、我々は2Dピクセルパーフェクトツールとどのようにグラフィックスを8ビットゲームの時代に作成されたをカスタマイズする方法を示しました。この投稿では、16ビット時代に移行します。Mega Cat Studiosでは、Unityオプション、グラフィック構造、およびカラーパレットを使用して、セガジェネシス（メガドライブ）およびスーパーファミコンのスタイルでゲーム用の本格的なグラフィックを作成する方法を学びます。

本物の創世記スタイルのグラフィックを作成する


このセクションでは、さまざまなコンソールの画像を模倣するグラフィックを作成するワークフローについて検討します。8ビットプロジェクトよりも制限が少なく、色を扱う際の自由度は高くなりますが、それでもジェネシスには制限があります。これらの制限を独自のレトロプロジェクトに適用できるように、コンソールのハードウェアがどのように機能したかを説明することも役立つと考えています。

パレットとサブパレット


機器の複雑さによる8ビットコンソールから16ビットコンソールへの移行は、開発者により多くの機会を提供しました。ただし、高品質のNESグラフィックを作成するための原則は引き続き有効です。すべてのグラフィックは引き続きタイル（たとえば、サイズが8×8）に格納され、それらから収集されてスプライトと背景要素の大きな画像になります。アーティストは依然として共通の透明色のサブパレットに制限されていますが、16ビットはパレットを操作するための柔軟性を提供します。 16ビットのコンソールにハードコードされたカラーパレットがなかったことに、おそらく満足するでしょう。つまり、使用可能な色の数は、NESに比べて大幅に拡張されています。

Genesisは、15色に加えて、スプライトとレイヤーの透明度に使用される全体的な色を含むサブパレットを誇っています。ただし、ジェネシスのグラフィックを作成する場合の欠点の1つは、サブパレットを使用することです。サブパレットはスプライトまたは背景タイルに自由に割り当てることができますが、ジェネシスでは一度に最大4つのサブパレットを使用できます。したがって、アーティストは、スプライトと背景の数が最大になるように、サブパレットで使用する色について考える必要があります。 Genesisサブパレットには通常、背景とスプライトの両方に使用される色が含まれています。



ジェネシスのゲームのシーンを上に示し、使用したサブパレットを下に示します。

16ビットプラットフォーム用のグラフィックスを作成するには、インデックス付きパレットを使用する必要があります。これを行うには、Gimpエディターを使用できます。これは、Photoshopの無料の代替ツールであり、インデックス付きのパレットを操作できます。

Gimpでインデックス付きパレットを作成するには、Image> Mode> Indexed ...に移動します。


[インデックス付きカラー変換]ウィンドウが開きます。


[最大色数]の値を15に設定します。自動ディザリングパターンを使用することもできますが、通常は手動で作成した方が見栄えがよくなります。


画像の色にインデックスが付けられました。この情報は画像とともに自動的に保存されるため、カラーインデックスを使用できます。インデックス内の色の順序を変更する必要がある場合は、カラーマップを右クリックして[カラーマップの再配置 ]を選択します...


ウィンドウが開き、色をドラッグアンドドロップして順序を変更できます。


色深度を増やすための一般的なトリックは、特定のスキャンラインでパレットを制御することです。これは、視差スクロールがNESに実装される方法と同様です。ジェネシスコンソールは、指定したスキャンラインのグラフィックサブパレットの選択を変更できます。このトリックは、レベルの一部が水面下にあるような錯覚を作成するためによく使用されます。水中の色は別のサブパレットに追加され、このサブパレットは、指定されたスキャンラインが画面にレンダリングされるときに読み込まれます。

タイルでの保存とこのコンソールでの読み込み


一般に、16ビットのコンソールは、8ビットのタイルとは異なる方法でグラフィックタイルをロードしました。 8ビットコンソールは、スプライトと背景タイルに大きなデータのチャンクをロードして処理能力を節約し、16ビットコンソールのリソースは柔軟性を高めました。彼らはその場で別々のタイルをロードして置き換えることができました。そのおかげで、必要なグラフィックのみをロードすることができました。したがって、1つの画面またはレベルで、さまざまなグラフィックを使用できます。

Genesis / Mega Driveのもう1つのユニークな側面は、ゲーム中にVRAMコンソールに読み込まれるデータがグラフィックスとパレットタイルだけではないことです。このため、特定の時点でメモリに読み込むことができるグラフィックタイルの数は、ゲームの他の状況に依存していたため、コンソールのグラフィックデザインを作成することが困難でした。一般的なケースでは、ほとんどのゲームには約1000のタイルを読み込むのに十分なスペースがあり、動的要素を使用すると、タイルをいつでも自由にダウンロードしてメモリに読み込むことができます。


上記のシーンでメモリに読み込まれたタイル。中央の大きな空きスペースとその下のアーティファクトは、ゲーム内の敵やその他の要素のためにメモリに割り当てられたスペースです。

VRAMではより多くの種類のタイルを同時にロードできるという事実にもかかわらず、ほとんどの場合、この追加のスペースはスプライト用に予約されていました。これにより、より複雑なアニメーションが提供され、画面上により多くのスプライトを同時に表示できるようになりました。つまり、タイルセグメントが繰り返されるという基本的な設計哲学は、16ビットグラフィックスで依然として積極的に使用されていたため、背景は利用可能なスペースの多くを占めていませんでした。 NES、Genesis、およびSNESの権限はほぼ同じであるため、通常、16×16セグメントがこれらのタイプのデザインの開始点でした。


ここでは、アーティストは32×32のブロックパターンを使用して、床の主要部分（背景）を作成しました。

背景レイヤーを操作する


Genesis / Mega Driveを使用すると、画面上に2層の背景を同時に表示できます。つまり、オーバーレイの要素は、デザインの背景を作成するのにより便利です。ただし、レイヤーは2つしかないので、シーンの奥行きをより深くするために、アーティストと開発者はスキャンラインでトリックを使用できます。幸いなことに、これらすべてを2番目のレイヤーに転送できるため、デザイナーは錯覚を壊すことなく、前景オブジェクトを背景の前に自由に配置できました。

さらに、第2レイヤーの存在により、開発者はスプライトの優先順位を使用してトリックを適用する必要がなくなりました。オンザフライでスプライトの優先順位を常に変更する代わりに、プレーヤーの前に表示される背景の2番目のレイヤーを設定できるようになりました。ただし、より複雑なオーバーレイの場合は、スプライトの優先順位をすばやく操作する必要があります。2番目の背景レイヤーにも、インターフェイスに使用できる領域があります。この領域は固定されており、スクロールすることはありません。


ツリーのトップダウンビューのため、スプライトレイヤーの順序を操作するために特別なタイルを作成する必要がありました。

スプライトの制限


16ビットに切り替えると、スプライトを操作する自由度が大幅に向上します。Genesis / Mega Driveでは、1つの水平線上に最大80のスプライトと最大20のスプライトを同時に画面に表示できます。後続のすべてのスプライトはレンダリングされません。さらに、スプライトは個別の8×8タイルとは見なされなくなりました。Genesisは、複数のタイルで構成される単一のスプライトを生成できます。1タイルから4x4タイルまで可能です。大きな画像は、いくつかのスプライトで構成する必要があります。


最終的なボスのグラフィックは、多数のアニメーション化された背景要素、レイヤー、および多くのスプライトを使用します。これはすべて、8ビットプラットフォームでは不可能です。

ディザパターンとコントラスト


16ビット時代のグラフィックの特徴の1つは、ディザリングの使用です。当時、ゲームはCRTテレビでプレイされていました。通常、画面のピクセルは互いに統合されています。アーティストは、グラフィックにディザパターンを適用することでこの機能を利用しました。1つのピクセルが繰り返しパターンで別のピクセルと結合すると、コンソールが許容するよりも多くの色の錯覚を引き起こしました。オリジナルの美学を伝えるために、ディザリングは、より優れたディスプレイにもかかわらず、ピクセルアートで依然として積極的に使用されています。


ディザリングは16ビットコンソールでよく使用されていました。CRT画面では、ピクセルパターンが混合され、これが不可能な場所で新しい色または透明効果が作成されました。

SNESと比較して、Genesis / Mega Driveはより高いコントラストで色を表示します。これは、サブパレットを選択するときにも考慮する必要があります。ミュートされた鈍い色でゲームを作成する場合、元のハードウェアでのレンダリング中に、パレットが異なる場合があります。通常、グラフィックスは高コントラストのカラーパレットで作成して、完成した結果がアーティストの元のビジョンと一致するようにします。

本物のSNESスタイルのグラフィックを作成する


Super NESゲームは8×8ピクセルのグリッド/タイルでも機能するため、繰り返しタイルを使用すると非常に便利です。一般に、サイズは8の倍数にする必要があります。

カラーパレット


Genesis / Mega DriveとSNESの最初の基本的な違いは、カラーパレットに関係しています。Mega Driveと同様に、SNESコンソールにはハードウェアにステッチされたカラーパレットがないため、独自の色を選択できます。

SNESの複雑さは、ピクセルあたり5ビット（ピクセルあたりのビット数、BPP）の深さの色を使用していることです。Gimpを使用してそれらを実装し、RGBの32シェードで画像を単純にポスタライズできます。これは、5 BPPの色に対応します。これにより、コンソールに画像の色が正確に表示されます。


Gimpのこのオプションは、メニュー項目の[ 色 ] > [ポスタリゼーション ]にあります。ポップアップウィンドウが表示され、ポスタリゼーションレベルを32に設定して、5 BPPと互換性のある色を作成できます。


画面の解像度


2つのシステムのもう1つの大きな違いは、画面の解像度です。SNESはNESの子孫であるため、両方のコンソールの画面解像度は同じでした。SNESの内部解像度は256×224です。ほとんどのCRTテレビの安全な領域で、画像の一部をトリミングせずに画像をレンダリングできます。この解像度は変更されないため、画像のサイズとなり、アーティストが使用する必要があります。


この画像は、ほとんどの画面モードで使用される全画面SNES解像度を占めます。

コンソール画面モード


このセクションでは、さまざまな画面モードについて簡単に説明します。


コンソール間の最大の違いは、SNESが7つの異なる画面モードで背景グラフィックスをレンダリングできることです。一部の画面モードでは、SNESは画面上の1つのサブパレットから256色を同時にレンダリングできます。最も一般的な画面モードは次のとおりです。

  • モード1： SNESの最も一般的な画面モードの1つ。これは、コンソールの機能を示す最も平均的なモードです。モード1では、3つの背景レイヤーで作業できます。そのうち2つには独自の16色のサブパレットがあり、最後のレイヤーには4色のサブパレットがあります。
  • モード3：このモードは、スプラッシュスクリーンやシーンなどの大きな静止画像に一般的に使用されます。2つの背景レイヤーがあります。1つ目はフル256色のサブパレットを使用し、2つ目は16色のサブパレットを使用します。
  • Mode 7: SNES. . Mode 7 : , , . -3D, SNES.

これらの機能を確実に機能させるために、モード7での唯一の背景面の処理は、他の画面モードとは大きく異なります。まず、256色のカラープレーンは1つしかありません。つまり、すべてのスプライトは、この背景プレーンのサブパレットからのカラーを持っている必要があります。次に、通常のSNES画面サイズの代わりに、モード7の背景面は1024×1024ピクセルでした。設計者が必要に応じてスケーリングおよび調整します。


このオフィスは、1つのサブパレットでモード1を使用します（他の2つはUIに使用されます）。「再生してくれてありがとう」画面では、モード3を使用しています。これにより、アーティストは256色のパレットを適用できます。

スプライトのサイズ


バックグラウンドスクリーンモードの複雑さと比較すると、スプライトの操作に関するルールは比較的単純です。SNESにはいくつかの異なるスプライトモードがあり、ほとんどMega Driveに似ていますが、厳密な制限があります：コンソールはゲーム内で2つの異なるスプライトモードしか使用

できません。しかし、それだけではありません。デザイナーは、既製のスプライトサイズのリストから組み合わせを選択する必要がありました。SNESのゲームでは、次の組み合わせを使用できます。

  • 8×8、16×16
  • 8×8、32×32
  • 8×8、64×64
  • 16×16、32×32
  • 16×16、64×64
  • 32×32、64×64

これらのサイズは「1回限り」で選択され、ゲーム内のすべてのスプライトはそれらに一致する必要があります。SNESは、1つの水平走査線上に32個のスプライトをレンダリングでき、一度に画面上に128個以下のスプライトをレンダリングできます。他のすべてはレンダリングされません。


Fork ParkerのCrunch-Outは、ゲーム内のすべてのスプライトに32×32と16×16の組み合わせを使用します。

スプライトは8つの16色サブパレットで動作します。すべてのレトロコンソールと同様に、サブパレットの最初の色は、透明度に使用される全体的な色です。他のコンソールに比べて多数のサブパレットがあるため、スプライトの色を自由に選択できます。256色の厳密な制限があることを覚えておく必要があります。

2D Pixel Perfectを使用してレトロゲームを作成する


では、前のレトロなチュートリアル、我々は2Dピクセルパーフェクトオプションをカバーし、8ビットスタイルのグラフィックスを再作成します。

Unity 2019.2以降、2D Pixel Perfectパッケージは2Dレンダラーの一部であり、2DレンダラーはLightweight Render Pipeline（LWRP）パッケージの一部です。LWRPを使用しない場合は、2D Pixel Perfectを別のパッケージとして使用することもできます。このチュートリアルでは、LWRPでプロジェクトを設定する方法を示します。

Unity 2019.2およびLWRPで新しいプロジェクトを準備する


  1. Unity Hubを起動し、[新規]をクリックして2Dを選択し、プロジェクトの名前を指定します。

  2. 2D Pixel Perfect, Window Package Manager. Lightweight RP , 6.9.0 .

  3. 2D Renderer Editor Pipeline Asset. Project Assets Create > Rendering > Lightweight Render Pipeline > Pipeline Asset.
  4. Assets Project 2D Renderer, Create > Rendering > Lightweight Render Pipeline > 2D Renderer.

  5. 作成したパイプラインアセットを選択します。[ 一般]を選択し、レンダラータイプを[ フォワードレンダラー]から[ カスタム]に切り替えます。
  6. 作成した2Dレンダラーをデータフィールドの値として設定します。

  7. グラフィックスオプションで、スクリプト可能なレンダーパイプライン設定を変更して、作成した新しいパイプラインアセットを使用します。

    これで、2Dレンダラーと2Dピクセルパーフェクトカメラのセットアップが完了しました。

Unity 2019.2では、2次元のスプライトに2Dライティング条件に応答できるSprite-Litマテリアルを含めることができます。プロジェクトに2Dライティングがない場合は、2Dライティングを必要としないマテリアルを使用して、スプライトが見えるようにします。つまり、「Sprites-Default」です。
