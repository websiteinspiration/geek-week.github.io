GOST R 57100-2016。それが何だった？

2017年9月には、ロシア連邦の国家標準を指定GOST R 57100から2016を（ステータスが受信され、導入された、ここで示されているテキストができ、ここで見つける）（私は、このようなギャグをオーバースローされるトマトの危険性を認識し、簡単にするために「百」と呼んでます）。今後は、この基準の順守に関するお客様の本当の要求に対応する必要があるため、情報システムの概念を説明する場合でも、この基準をより詳しく検討する時がきたようです。

GOST RがISO / IEC / IEEE 42010：2011システムおよびソフトウェアエンジニアリング-アーキテクチャの説明に記載されている完全に非ゲストアプローチを隠しているという事実を隠すことさえできません。さらに、GOST R 57100-2016はどのGOSTともまったく関係がないだけでなく、従来のGOSTとは異なる独自の哲学も導入しているため、このアプローチは一言で言うとGostのものではありません（標準のテキストの品質については触れません。他の規格は、ロシア語の「ISO / IEC 10746」または英語の「ISO 15704」のいずれかで示されています。

率直に言って、国家規格が他の関連する国家規格と組み合わせて導入されなかったが、評判のよい国際機関によって採用されたにもかかわらず、直接外国規格を参照していることはやや残念です。

連邦法「ロシア連邦における標準化について」の第26条のパラグラフ3によると、「国家規格の適用を含む製品の国家規格への適合に関する公的声明の場合、製造業者および（または）請負業者には国家規格の適用が必須です。ラベル付け、運用上またはその他の文書化、および（または）製品に国の標準化システムの記号が付いたラベル付けの標準。」

言い換えれば、顧客は自分の参照条件でこの標準に従う必要があると規定しており、ISO / IEC / IEEE 42010標準が一度に採用されたパッケージで、請負業者と彼自身の両方が国内GOSTだけでなく、国際標準の束にも従うことを義務付けています。 ：2011。また、GOSTの要件が国際基準と矛盾する場合は、国際基準の要件に従う必要があります。彼らのテキストが変更されるとどうなるか、「百人」の著者は沈黙している。

用語の恐ろしい翻訳。たとえば、翻訳の「フレームワーク」は「構造」に変わり、元の意味が失われます。「構造」という用語は特定のドキュメントのテンプレートを指し、「フレームワーク」という用語はベース、フレームとして理解されることがよくあります。それは実際には何であり、標準ISO / IEC / IEEE 42010：2011に準拠しています。確かに、この標準はアーキテクチャの構造を説明していませんが、アーキテクチャを説明するための基礎として使用できるアプローチ、原則、および実践を説明しています。

「懸念」という用語は「関心」に変わり、「織り」でのデコードがなければすべて問題ありません。関心は「1つ以上の利害関係者に関連するシステムの利益または問題」であることがわかります。当初は、「懸念（システム）-1つ以上の利害関係者に関連するシステムへの関心」のように見えました。 GOSTの作成者は、定義の問題をどこで入手しましたか？

一般に、私の意見では、GOST R 57100-2016のテキストを著者から提出された形式で使用することは不可能です。 ISO / IEC / IEEE 42010：2011の元のテキストを使用します。

そして、あなたの文書でGOSTを示すことを避けてください、これは法律の下であなたに追加の義務を課すだけです。ただし、配当はありません。すぐにISOを参照することをお勧めします。
