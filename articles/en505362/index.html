Vite - lead development without bundles on Vue

Hi, Habrovsk. In June, OTUS launches the Vue.js Developer course . In anticipation of the start of the course, we traditionally prepared a translation of interesting material.


Once Evan Yu (creator of Vue.js) decided not to sleep at night and eventually created Vite .



Vite allows you to create applications on Vue with single-file components without the assembly step of bundles. Imports are requested by the browser as native ES modules. The dev server intercepts requests for .vue files and compiles them on the fly. And it happens instantly.

Please note that Vite is still an experimental solution, and I don‚Äôt know what will happen in the future and how long this article will remain useful. Think of it as an aperitif or just as something fun.

let's try


Create the following files:

Comp.vue

<template>
  <button @click="count++"> {{ count }}</button> 
</template>

<script>
export default {
  data: () => ({ count: 0 })
}
</script>

<style scoped>
button{
  font-size: 2rem;
}
</style>

index.html

<div id="app"></div>
<script type="module">
  import { createApp } from 'vue'
  import Comp from './Comp.vue'

  createApp(Comp).mount('#app')
</script>

Then do:

npx vite

Go to http://localhost:3000, edit the file .vueto see instant changes.

Production Assembly


Now let's create a production-ready application.

Createpackage.json

{
  "scripts": {
    "build": "vite build"
  }
}

Run:

npm i -D vite
npm run build


Check the directory with the code.
Problems I encountered:

  • Error: Cannot find module 'tslib' Solved by
    running npm i -D tslib
  • Error: ENOENT: no such file or director
    Solved by manually creating an empty directory

Thoughts


It was really easy to start, and development went faster, because I immediately saw the result of the changes. Something like compilation on the fly. So it looks really promising, maybe after a while, we will move on to just such a development method for Vue.
