Today we turn to the practical part. Let's start by setting up our certification authority based on a complete openSSL cryptographic open source library. This algorithm was tested using windows 7.

Having installed openSSL, we can perform various cryptographic operations (for example, creating keys and certificates) through the command line.

The algorithm of actions is as follows:

  1. Download the installation distribution package openssl-1.1.1g.
    OpenSSL has various versions. The documentation for Rutoken said that openSSL version 1.1.0 or later is required. I used openssl-1.1.1g version. You can download openSSL from the official site, but for an easier installation, you need to find the installation file for windows on the network. I did it for you: slproweb.com/products/Win32OpenSSL.html You
    need to scroll down the page and download Win64 OpenSSL v1.1.1g EXE 63MB Installer.
  2. Install openssl-1.1.1g on the computer.
    Installation must be carried out according to the standard path, which is automatically specified in the C: \ Program Files folder. The program will be installed in the OpenSSL-Win64 folder.
  3. In order to configure openSSL as you need, there is an openssl.cfg file. This file is located in the path C: \ Program Files \ OpenSSL-Win64 \ bin if you installed openSSL as described in the previous paragraph. Go to the folder where openssl.cfg is stored and open this file using, for example, Notepad ++.
  4. You probably guessed that the certification center would be set up somehow by changing the contents of the openssl.cfg file, and you are absolutely right. To do this, configure the [ca] command. In the openssl.cfg file, the beginning of the text where we will be making changes can be found as: [ca].
  5. Now I will give an example of configuration with its description:

    [ ca ]
default_ca	= CA_default		

 [ CA_default ]

serial		= $dir/private/serial 		
crlnumber	= $dir/crlnumber	
					
crl		= $dir/crl.pem 		
private_key	= $dir/private/ca.key
x509_extensions	= usr_cert
