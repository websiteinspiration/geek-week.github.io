Analyser YouTube, y compris les données téléchargées, sans l'API YouTube

introduction


Pour charger des données de contenu sur YouTube, ils utilisent généralement Selenium ou l'API YouTube. Cependant, il y a des inconvénients partout.

  1. Le sélénium est trop lent à analyser. Imaginez analyser une liste de lecture de ~ 1000 clips par sélénium.
  2. L'API YouTube, bien sûr, est la meilleure option si vous avez une sorte d'application ou de projet, mais là, vous devez enregistrer l'application, etc. Dans la version "d'essai", vous devez vous connecter en permanence pour utiliser l'API, il existe également un quota à fin rapide.
  3. Dans notre méthode, je dirais des structures de données très complexes générées par YouTube. L'analyse des recherches YouTube est particulièrement instable.

Comment charger des données sur YouTube?


Il existe un jeton pour cela, qui peut être trouvé dans le code html de la page. Ensuite, à l'avenir, nous l'utiliserons comme paramètre pour une demande à YouTube, ce qui nous donne un nouveau contenu. YouTube lui-même télécharge du contenu en utilisant une demande où ce jeton est utilisé.

Il y aura des paramètres sortants supplémentaires dont nous aurons besoin à l'étape suivante.



Obtenir un jeton via un script


Nous composons le paramètre des en-têtes pour la demande à YouTube. En plus de user-agent, nous en insérons deux supplémentaires, que vous voyez ci-dessous.

headers = {
    'User-Agent': '      ,      ',
    'x-youtube-client-name': '1',
    'x-youtube-client-version': '2.20200529.02.01'
}

Nous faisons une demande en utilisant la bibliothèque de demandes. Mettez un lien vers la page que vous souhaitez charger et ajoutez également des en-têtes.

token_page = requests.get(url, headers=headers)

Le jeton ne peut pas être trouvé par les analyseurs, car il est masqué dans la balise tag. Pour l'enregistrer dans une variable, j'écris un code aussi laid:

nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]

Il s'agit généralement d'un jeton de 80 caractères.

Faire une demande de contenu


service = 'https://www.youtube.com/browse_ajax'
params = {
"ctoken": nextDataToken,
"continuation": nextDataToken
}
r = requests.post(service, params=params, headers=headers)

Différents types de données chargées ont différents liens de service. La nôtre convient aux listes de lecture et aux vidéos des chaînes.

YouTube envoie des données au format json. Par conséquent, nous écrivons r.json (), mais la liste volera vers vous, nous avons besoin du deuxième élément de la liste, donc r.json () [1]. De plus, nous avons déjà des données. Il reste à analyser.

Analyse de la réponse JSON


Vous pouvez voir de longues chaînes de dictionnaires, mais nous les raccourcirons pour que ce soit plus ou moins clair.

r_jsonResponse = r_json['response']
dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]

Ici, nous obtenons un nouveau jeton pour une demande supplémentaire. Si les données chargées sont terminées, vous ne verrez pas le jeton.
for content in dataContainer["contents"]:
	videoId = content['playlistVideoRenderer']['videoId']

C'est ainsi que vous pouvez extraire l'id de la vidéo en ajoutant la partie modèle, vous obtiendrez un lien vers la vidéo.

Pour obtenir les données suivantes, vous devez faire la même chose - demander un jeton, analyser, puis à nouveau.

Le code pleinement fonctionnel ressemble à ceci:

import requests, json

url = input('url: ')
headers = {
		"User-Agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/81.0.4044.122 Safari/537.36",
		'x-youtube-client-name': '1',
		'x-youtube-client-version': '2.20200429.03.00',
		}
token_page = requests.get(url, headers=headers)
nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]
sleep = False #  ,    
ids = []
while not sleep:
	service = 'https://www.youtube.com/browse_ajax'
	params = {
	"ctoken": nextDataToken,
	"continuation": nextDataToken
	}
	r = requests.post(service, params=params, headers=headers)
	r_json = r.json()[1]
	r_jsonResponse = r_json['response']
	dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
	try: #  
		nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]
	except:
                #  . ,    .    
		sleep = True
	for content in dataContainer["contents"]:
		videoId = content['playlistVideoRenderer']['videoId']
		ids.append(videoId)
print(len(ids))
