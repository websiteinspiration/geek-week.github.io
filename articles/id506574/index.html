Get best of the week

Visualisasi daftar perempuan pemenang Hadiah Nobel dalam bentuk kristal di 3d menggunakan Vue, WebGL, three.js



Tahun 1 | inspirasi


Bulan ini saya mencoba untuk waktu yang sangat lama untuk memutuskan dataset dan ide untuk memprosesnya. Meskipun saya mulai memikirkannya kembali pada bulan Mei, saya berhasil menyelesaikan fakta hanya setelah 8 bulan (sial, saya buruk), dan saya berhasil menggambarkan proyek sebulan kemudian (oh, saya sangat buruk).

Ide proyek muncul setelah saya menonton film Madly Rich Asians . Saya benar-benar menyukai aktris Michelle Yeo , tetapi gagasan itu terbentuk hanya setelah saya membaca lebih banyak tentang dia dan mengetahui betapa luar biasa dan kerennya dia. Itu membuat saya berpikir tentang wanita luar biasa yang saya tidak tahu. Dan sekarang - ide muncul entah bagaimana memvisualisasikannya.


1 minggu | data


Setelah pembentukan konsep, pertanyaan berikut muncul - bagaimana cara mengumpulkan data yang diperlukan? Saya dengan cepat memunculkan ide untuk menggunakan Wikipedia, mencoba mengurai perempuan "terbaik" dari sana. Saya tidak tahu cara menentukan "terbaik" (tampilan halaman? Panjang halaman? Tautan halaman?). Meskipun saya melihat sebuah artikel di The Pudding tentang visualisasi berdasarkan data Wikipedia, saya masih tidak yakin bahwa saya ingin bergerak ke arah ini. Dan saya benar-benar tidak punya cukup waktu luang.

Kemudian terlintas dalam benak saya bahwa mungkin saya harus menggunakan daftar wanita yang tetap, dan ini entah bagaimana membawa saya ke halaman 51 wanita pemenang Hadiah Nobel . Saya belum pernah mendengar tentang sebagian besar dari mereka, dan saya ingin tahu lebih banyak. Untuk proyek, ini sempurna.



Sejak saat itu, saya mulai mempelajari berbagai metode menggunakan API Wikipedia. Tidak mudah untuk mengetahuinya (saya tidak yakin apakah saya perlu menggunakan Wikidata atau MediaWiki - ini adalah opsi yang muncul ketika mencari "Wikipedia API" di Google), tetapi, untungnya, artikel Puding yang telah disebutkan di atas membantu saya menutup masalah ini . Melihat sekilas repositori wiki-billboard-data membuat saya memilih salah satu skrip dengan wikijs , dan saya bisa menggunakan paket ini.

Yang harus saya lakukan adalah menyalin dan menempelkan daftar wanita pemenang hadiah ke spreadsheet dan melakukan sedikit format / pembersihan.

Lalu saya mengekspor spreadsheet sebagai CSV, lalu saya menggunakan konverter online untuk mendapatkan data dalam format JSON .

Setelah itu, saya menulis skrip sederhana menggunakan halaman wiki untuk mendapatkan informasi lebih lanjut tentang setiap wanita, termasuk jumlah tautan ke halaman mereka ("tautan balik") dan jumlah sumber di bawah ini untuk mendapatkan kumpulan data akhir .

Terlepas dari kenyataan bahwa pembuatan ide itu berlangsung selamanya, pengumpulan data hanya memakan waktu beberapa jam.

minggu 2 | sketsa


Saya tidak akan berbohong, saya memikirkan proyek ini begitu lama sehingga saya bahkan tidak dapat mengingat semua ide yang saya miliki dalam proses. Yang saya ingat adalah bahwa saya melihat nominasi untuk penghargaan Information is Beautiful ketika saya memutuskan bahwa saya perlu menyorot catatan yang sangat saya sukai di simpan saya. Ini membuat saya sadar bahwa saya harus mengatur papan tulis saya dengan lebih baik di Pinterest , seperti sebelum saya membuang semua ide dan visi saya di satu papan. Dalam proses pembersihan, saya menemukan gambar yang luar biasa ini dengan kristal dari artis Rebecca Chaperonyang saya simpan untuk masa depan beberapa tahun yang lalu. Hampir seketika, saya menyadari bahwa saya ingin membuat ulang secara terprogram. Lagi pula, alangkah baiknya memvisualisasikan wanita-wanita luar biasa ini sebagai kristal multi-warna yang cerah?



Mungkin saya punya ide lain sampai saat ini (ya, itu sangat mungkin), tetapi saya tidak dapat mengingatnya - ide dengan kristal itu sempurna.

Setelah itu, saya memikirkan detail lainnya: ukuran kristal harus mencerminkan pengaruh wanita - jumlah tautan ke halaman Wikipedia-nya. Jumlah wajah pada kristal akan dibandingkan dengan jumlah sumber di bagian bawah halamannya (karena "beragam"), dan warnanya akan ditentukan oleh kategori premium. Satu-satunya hal yang saya masih tidak mengerti adalah bagaimana mengatur kristal. Saya berpikir untuk waktu yang lama bahwa saya hanya perlu meletakkannya dalam 2 dimensi x / y dan membuat pembaca menggulirnya.

Dan kemudian saya ikut serta dalam Seminar Creative Coding Matt DesLoriersdi mana ia mengajar kanvas, three.js dan WebGL. Seminar membuka konsep dimensi ketiga kepada saya, dan saya segera menyadari bahwa saya akan menggunakan tahun menerima penghargaan sebagai sumbu z. Lebih lanjut, seorang teman saya menyarankan untuk membuat utas-utas benang yang menghubungkan mereka yang berkolaborasi satu sama lain sehingga posisi mereka juga berubah tergantung pada ini (tetapi pada akhirnya saya tidak punya waktu untuk mengimplementasikan ide ini).

Seluruh konsep datang kepada saya begitu cepat dan alami sehingga saya tidak membuat sketsa tunggal dalam proses. Saya melihat notebook saya, tetapi tidak ada sama sekali.

minggu 3 & 4 | coding


Itu bulan yang bagus untuk coding.

Selama beberapa tahun terakhir, saya ingin menciptakan sesuatu dengan perwujudan fisik, dan tahun lalu saya akhirnya memutuskan untuk menjadikannya tujuan saya. Tetapi sebagian besar tahun saya tidak tahu apa yang akan terjadi. Setiap kali saya memikirkannya, saya hanya terjebak pada proyeksi dua dimensi di dinding dan tidak tahu bagaimana menggunakan seluruh volume.

Dan begitu saya sadar (saya tidak yakin apa alasannya) bahwa saya tidak tahu bagaimana berpikir dalam ruang fisik tiga dimensi, justru karena saya terus bekerja dalam format digital yang datar. Jika saya bisa belajar bekerja dengan 3D dalam digital, saya akan dapat berpikir dalam volume dan di dunia fisik. Saya menambahkan three.js dan WebGL ke atas daftar saya untuk belajar.

Pada akhir Oktober, saya menghadiri lokakarya pengkodean kreatif Matt dan mempelajari dasar-dasar three.js, sebuah pengantar untuk shader terfragmentasi dan verteks. Saya belajar aturan gimlet : gunakan ibu jari untuk sumbu X (naik ke kanan), jari telunjuk untuk sumbu Y (meningkat, yang berlawanan dengan SVG dan kanvas) dan jari tengah untuk sumbu Z (untuk memperbesar layar dan memperbesar). Saya menemukan bahwa WebGL tidak berfungsi dalam piksel, tetapi dalam satuan (asosiasi yang serupa adalah kaki atau meter).

Kemudian pada bulan November saya masuk ke dalam satu kontes WebGL. Meskipun saya tidak punya bisnis dengannya sebelumnya, saya berharap tenggat waktu akan memberi saya motivasi yang dibutuhkan untuk menyelesaikan proyek. Saya mulai pada 1 Desember dan menetapkan tujuan saya untuk melakukan sedikit setiap hari kerja sampai saya dapat mencapai sesuatu yang layak pada 23 Desember (final kompetisi).

Pertama, saya membaca dua bab pertama dari Panduan Pemrograman WebGL , yang menjelaskan cara mengatur WebGL. Kemudian saya mengulangi pengetahuan saya tentang three.js. Setelah itu saya meluncurkan notebook dari Observable untuk memahami pengaturan minimum yang diperlukan untuk menggambar sesuatu di three.js (renderer, kamera dan adegan, dan kemudian panggil renderer.render (adegan, kamera) untuk ditampilkan). Saya selalu suka memahami dasar-dasar kode, jadi itu sangat membantu.

Setelah penyetelan, saya memutuskan untuk membuat bentuk kristal. Saya menggunakan PolyhedronGeometry karena dengan cara itu saya hanya bisa mendefinisikan satu set simpul dan kemudian menentukan simpul yang akan membentuk wajah segitiga. Pada hari pertama, saya hanya berhasil membuat satu segitiga.



Dan pada hari kedua - kristal (yang mengambil dua upaya, karena pertama kali saya menyesuaikan dengan matematika).



Dan kemudian, pada akhirnya, bentuk kristal yang ada di kepalaku.



Kemudian, saya menyadari bahwa ada cara yang lebih baik untuk melakukan apa yang saya inginkan (dan PolyhedronGeometry sebenarnya adalah cara yang agak membosankan untuk melakukan ini), tetapi saya sangat senang bisa mempraktikkan koordinat x / y / z di WebGL.

Segera setelah saya puas dengan bentuk yang dibuat, saya melanjutkan untuk mempelajari pewarnaan mereka dalam warna. Untuk melakukan ini, saya mulai bermain dengan vertex dan fragmen shader .



Tugas selanjutnya adalah mempelajari cara menggunakan shader fragmen untuk mewarnai kristal. Di sini saya harus menyimpang sedikit dari rencana, karena saya tidak bisa mengerti cara menggunakan glslify (sistem modular berbasis node untuk GLSL, di mana shader yang saya ambil sebagai sampel) ditulis. Sebagai gantinya, saya mulai belajar berbagai alat build / build untuk akhirnya menyebarkan kode saya di Internet. Pada akhirnya, saya memutuskan untuk menggunakan Parcel (bukan Vue CLI , yang telah saya gunakan secara teratur selama enam bulan terakhir), karena ia memiliki dukungan bawaan untuk Vue dan GLSL.

Inilah kristal dengan pola suara yang sama yang menempel pada saya.



Saya tidak suka hasilnya, jadi saya memutuskan bahwa saya perlu belajar lebih banyak tentang shader dan penggunaan warna dalam shader. Pada saat itulah saya membuka Buku Shader oleh Patricio GonzÃ¡lez Vivo dan, khususnya, ke bab "Membentuk Fungsi" . Saya belajar tentang sinus, cosinus, polinomial, dan eksponen - fungsi yang dapat mengambil angka (atau sekumpulan angka) dan mendapatkan yang lain berdasarkan padanya. Saya juga belajar tentang pencampuran warna dan bagaimana kita dapat mengambil dua warna, dan tidak hanya mendapatkan warna baru di tengah di antara keduanya, tetapi juga mencampur warna RGB dan membuat yang benar-benar baru berdasarkan mereka. Ini diperoleh dengan mencampur biru dan kuning, serta mengubah nilai RGB di setiap posisi menggunakan derajat, sinus, nilai absolut, langkah dan perataan.

Segera setelah saya puas dengan warna yang dihasilkan, saya beralih ke bekerja dengan data: jumlah tautan di bagian bawah untuk jumlah pihak, jumlah backlink untuk ukuran, kategori Hadiah Nobel untuk warna.



Tapi saya tidak suka hasilnya karena dua alasan:

  • Saya secara kaku mengatur bentuk masing-masing gambar, jadi saya hanya memiliki tiga bentuk berbeda: persegi panjang, persegi panjang dengan segitiga di bagian atas dan persegi panjang dengan segitiga di bagian atas dan bawah. Ini berarti bahwa sebagian besar "kristal" hanyalah persegi panjang.

Sekitar waktu yang sama, saya menemukan demo Bloom (efek post-processing untuk membuat cahaya), dan berisi benda bulat seperti permata yang terlihat sangat mirip dengan kristal yang ingin saya dapatkan:



Saya melihat kode sumber dan menyadari bahwa mereka menggunakan SphereGeometry, tetapi dengan flatShading dihidupkan - jadi alih-alih menampilkan permukaan yang halus, itu menunjukkan setiap wajah masing-masing. Saya menyadari bahwa bagaimana saya dapat memanipulasi dan mencampur warna untuk mendapatkan warna baru yang sama sekali berbeda dari aslinya, jadi saya benar-benar dapat mengontrol parameter geometris (misalnya, jumlah wajah) dan mendapatkan geometri baru yang terlihat serupa tetapi berbeda dari aslinya .

Jadi saya mengganti PolyhedronGeometry dengan SphereGeometry, mengatur tinggi menjadi 4 dan lebar untuk data saya, meregangkan bentuk, mengatur skala vertikal dua kali lebih lebar secara horizontal, menambahkan jitter (jitter) untuk setiap titik, dan saya mendapatkan bentuk yang jauh lebih menarik:



Sekarang, ketika saya memutuskan formulir, sudah waktunya untuk kembali ke warna. Kali ini saya menggunakan dua warna dan mencampurnya dengan fungsi bentuk:



Saya suka opsi pertama - sepertinya kentang

Karena shader default hanya membungkus bentuk (setidaknya saya suka berpikir tentang hal itu), saya kehilangan tepi jernih. Untungnya, saya diajarkan untuk mengembalikan formulir kembali dengan memanggil computeFlatVertexNormals () pada geometri, mendapatkan normals dalam vertex shader dan meneruskannya ke fragmen shader sambil menambahkannya ke warna. Ini tidak hanya membuat tepi jelas, tetapi juga memberikan ilusi cahaya dan bayangan:


Kode untuk menambahkan normal ke sisi.

Setelah itu, saya bermain-main dengan dua set gradien: satu untuk "humaniora" (dunia, sastra, ekonomi), dan yang lainnya untuk "ilmu alam" (fisika, kimia, kedokteran).



Lalu dia mengambil latar belakang. Saya membuat "lantai" menggunakan PlaneGeometry dan secara acak mengguncang posisi-y dari setiap titik (terinspirasi oleh artikel ini dari codrops), serta "langit", menciptakan bola besar di sekitar tempat kejadian. Saya bereksperimen dengan tiga jenis sumber cahaya: belahan bumi, tersebar (untuk menciptakan perasaan matahari terbenam / matahari terbit untuk "langit"), dan arah (untuk melemparkan bayangan dari kristal ke "lantai").



Pada akhirnya, saya menambahkan "bintang-bintang" yang memvisualisasikan semua pria yang memenangkan penghargaan untuk periode waktu yang sama, serta anotasi untuk setiap kristal dan selama beberapa dekade. Itu sulit, karena dengan teks yang saya miliki, menggunakan TextGeometry praktis tidak berguna. Solusi yang saya temukan setelah mencari di Internet adalah membuat teks dalam Kanvas HTML5, membuat PlaneGeometry dan menggunakan Kanvas ini sebagai tekstur untuk mengisi PlaneGeometry. Pendekatan yang sangat menarik.

Saya sangat suka bahwa saya dapat menemukan alasan sebenarnya untuk menggunakan dimensi ketiga - dekade menerima penghargaan. Semakin dekat mereka ke garis depan, semakin banyak hadiah pada waktunya, dan semakin jauh mereka dari kita, semakin tua penghargaan ini. Tapi saya tidak menunjukkan dekade ini sampai pengguna "lepas landas" untuk melihat kristal dari atas. Jika mereka "berjalan" di antara kristal di permukaan tanah, saya hanya menunjukkan informasi tentang setiap wanita, karena saya ingin pengunjung hanya berkonsentrasi pada wanita selama berjalan.
