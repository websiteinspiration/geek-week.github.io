TensorFlow 2への自動コードアップグレード



この資料は、更新スクリプトを使用してコードをTensorFlow 1.xからTensorflow 2に自動的に更新するためのマニュアルの翻訳を提供しますtf_upgrade_v2。

TensorFlow 2.0には、引数の順序の変更、文字の名前の変更、パラメーターのデフォルト値の変更など、多くのAPIの変更が含まれています。これらすべての変更を手動で修正するのは面倒で、エラーが発生しやすくなります。変更を簡素化し、TF 2.0への移行をできるだけスムーズにするために、TensorFlowチームはtf_upgrade_v2、レガシーコードから新しいAPIへの移行を支援するユーティリティを作成しました。
注： tf_upgrade_v2 TensorFlow 1.13以降（すべてのTF 2.0ビルドを含む）では自動的にインストールされます。
スクリプトの一般的な使用方法は次のとおりです。

tf_upgrade_v2 \
  --intree my_project/ \
  --outtree my_project_v2/ \
  --reportfile report.txt

このコードは、既存のTensorFlow 1.x PythonスクリプトをTensorFlow 2.0に変換することで、アップグレードプロセスを高速化します。

変換スクリプトはプロセスを可能な限り自動化しますが、スクリプトでは修正できない構文およびスタイルの変更がまだあります。

互換性モジュール


一部のAPI文字は、単に文字列置換を使用して更新することはできません。コードがTensorFlow 2.0で確実に機能するように、更新スクリプトにはモジュールが含まれていますcompat.v1。このモジュールは、などのTF 1.x文字をtf.foo同等のリンクに置き換えますtf.compat.v1.foo。互換性モジュールは良好ですが、置き換えを手動で差し引いて、できるだけ早くtf. *名前空間ではなく名前空間の新しいAPIに転送することをお勧めしますtf.compat.v1。

TensorFlow 2.xモジュール（tf.flagsおよびなどtf.contrib）の廃止により、一部の変更はに切り替えることで回避できませんcompat.v1。このようなコードを更新するには、追加のライブラリを使用する必要がある場合があります（たとえば、でabsl.flagsパッケージに切り替えるなど）tenorflow/addons。

"セーフモード


変換スクリプトには、モジュールを使用するようにインポートを変更するだけの、侵襲性の低い "SECURE"モードもありますtensorflow.compat.v1。

!tf_upgrade_v2 --mode SAFETY --infile dropout.py --outfile dropout_v2_safe.py > /dev/null

コード例：

import tensorflow as tf

d = tf.nn.dropout(tf.range(10), 0.2)
z = tf.zeros_like(d, optimize=False)


このモードでは、次のように変換されます。

import tensorflow.compat.v1 as tf
tf.disable_v2_behavior()

d = tf.nn.dropout(tf.range(10), 0.2)
z = tf.zeros_like(d, optimize=False)

ご覧のとおり、コードは更新されていませんが、TensorFlow 1のコードはTensorFlow 2で実行されています。
