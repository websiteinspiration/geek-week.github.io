成長：チームでのスキルの評価方法

こんにちは！DocDoc（Sberbankグループの一員）のデザインディレクター、Vanya Solovyovです。今日は、ドリームティムの見出しです。これは、ドリームチームの作成方法を示しています。 私たちの設計チームを開発するために、2つのツールを使いました。そして、それらは互いに補完し合う。



最初のツール：デザインのレベル


長期的な問題を解決する透明な成長システムを実装しました。

  • デザイナー - 「給与を増やす方法と、どこでさらに発展させるか、どのコースをとるかは明確ではありません。」;
  • 企業 - 「デザイナーの開発を正しい方向に体系化する方法」。

2番目のツール：スキル評価


各デザイナーのハードスキルとソフトスキルを正確に開発するのに役立ちます。

これら2つのツールの動作をよりよく理解するために、次のことを想像してください。

  • 設計のレベル -このツールは、デザイナーの方向性と成長のポイント全体を示すガイドブックに似ています。
  • スキルの評価は、指標を見て、デザイナーが正しい方向に成長できるかどうかを理解できるダッシュボードに似ています。



以前の記事の1つで 、デザインのレベルについて話しました。今日は、スキルの評価について話します。なぜこのツールが必要なのか、どのようにそれを実装するのか、そして取得したデータを使って何をするのかです。

前書き


デザイナーの仕事は、多様なスキルの組み合わせです。十分に発達しているものもあれば、不十分なものもあります。アプリケーションの範囲に応じてスキルを「ポンプオーバー」する場合、これは正常です。

しかし内省は難しい。したがって、私たちは専門的なスキルを収集し、それらをセクションにグループ化しました：

  • ビジュアルデザイン-UI ;
  • インターフェースデザイン-UX ;
  • 研究と分析;
  • 自己およびプロジェクト管理;
  • 設計成果物。


テーブルへのリンク

アイデアの基本的なコンセプトを提供してくれたYura Vetrovに感謝します。

評価


私たちは6か月に1度評価を行い、それが合理的に客観的であるために、各デザイナーのこの一連の参加者について詳しく説明します。

  • 彼が過去6か月間密接に協力していた2〜3人のマネージャー。
  • 客観的な評価を行うことができる1〜2人の設計者。
  • 監督者;
  • デザイナー自身。

誰もがスキルの用語を等しく理解できるように、それらの上にカーソルを合わせると、説明付きのヒントが表示されます。


各スキルには短い説明があり、

スキルは5つのポイントシステムを使用して評価されます。

  • -・「わからない」 ;
  • 1・悪い、「何も知らない」 ;
  • 2・表面的な、「信用しない」 ;
  • 3・平均、「まあ、まあまあ」。
  • 4・優秀、「あなたは信頼できる」 ;
  • 5・権威ある、「彼は教えることができる。」

いくつかの評価の後、連中は小数値を入れることが多いことがわかりました-つまり、スケールが小さいことを意味します。バージョン3.0では、10ポ​​イントシステムに切り替えます。

投票数


客観的な結果を得るために、最終成績を2つに分けました。

  • 結果は、マネージャー、デザイナー、およびヘッドの評価の平均値です。
  • 私の評価は、デザイナーが自分をどう評価したかです。



これは、デザイナー自身が世界の2つの写真を見るのに役立ちます-彼の同僚が彼をどのように見るか、そして彼が自分自身をどのように配置するか これは、3人以上が評価した場合に機能します。

別のタブで、各設計者は同僚の結果を確認し、自分と比較することができます。


そんな時、10-16人のデザイナーのチームがそれらを比較することを夢見ています

調査結果


どのスキルに開発が必要か-2つの要因によって決定されます。

私たちは良いものを改善し、悪いものに触れないでください


問題だけに焦点を当てることは疑わしい考えです。注意と努力の70％は、デザイナーがすでに得意とする分野の開発に向けられ、残りの30％は、スキルのテールとパッチホールのクリーンアップに向けられるべきです。
70％は4ポイント以上のスキルの開発に費やされ、30％は3ポイント以上のスキルの開発に費やされています。

製品への関心


たとえば、1か月でプロダクトマネージャーはユーザーに関する調査を積極的に行います。このスキルに対する設計者の評価が4ポイントに近い場合、彼の開発に集中することは理にかなっています。

最後に、各デザイナー向けにドキュメントが作成され、そのドキュメントが書き込まれる場所、注意を払う価値のあるスキルとその開発に関する推奨事項が示されます。短期的には、本、記事、そして人々。長期的には、コースとミタップ。


記事、本、コース、ミタップ

すべてが純粋に推奨であり、強制はありません。スキル評価の目的は、成長のポイントを示すことです。そして、デザイナーは自分で決める-開発するか採点するか。

2つの評価後の結果


評価が肯定的だった後のフィードバック-彼らは推奨を取り、良い結果を示しました。何もしなかった人もいましたが、自分のスキルを横から見ると便利だとおっしゃっていました。

過去のミスを勘案し、8月下旬に行われる3回目のスキル評価の準備に着手しました。

そしてあなたの会社はデザイナーの開発にどのように取り組みますか？コメントに興味を持って聞きます。
