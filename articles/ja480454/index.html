ポートスキャン


このマシンのIPアドレスは10.10.10.135で、/ etc / hostsに追加します。
10.10.10.135 smasher2.htb
まず、開いているポートをスキャンします。nmapですべてのポートをスキャンするには時間がかかるため、最初にmasscanでこれを実行します。tun0インターフェースからすべてのTCPおよびUDPポートを毎秒1000パケットの速度でスキャンします。

masscan -e tun0 -p1-65535,U:1-65535 10.10.10.135 --rate=1000



ホストには3つのポートが開いています。nmapでスキャンして詳細を取得してください。

nmap -A 10.10.10.135 -p22,53,80



したがって、SSH、DNS、およびWEBがあり、403コード（禁止、アクセス拒否）を返します。

DNS


DNSをチェックしましょう。これを行うには、ホストクライアントを-lオプションとともに使用し、AXFR要求を使用してドメイン内のすべてのホストのリストを表示します。

host -l smasher2.htb 10.10.10.135



したがって、/ etc / hostsに新しいエントリを追加する必要があります。
10.10.10.135 wonderfulsessionmanager.smasher2.htb

次に、smasher2.htbにアクセスしたときにWEBが何を提供するかを見てみましょう。

関数の乏しいリスト。PWNについては、メモリを操作する関数に関心があります。名前から判断すると、これらはdev_readとdev_mmapです。



さらにグーグルで調べたところ、mmapとは言えない、読み取りに関連するドライバーの脆弱性に関する情報は特に見つかりませんでした。それで私は彼女のところへ行きました。



通常、最初はデバイスが物理メモリをまったく使用しないため、デバイスをメモリにマップし、オンデマンドでページを選択するには、ドライバのmmapが必要です。

このコードで興味深いのは、デバイスのメモリをユーザーのアドレス空間に線形マッピングできるremap_pfn_range関数を呼び出すことだけです。
int remap_pfn_range（struct vm_area_struct * vma、unsigned long virt_add、unsigned long pfn、unsigned long size、pgprot_t prot）;

仮想アドレスvirt_addのpfnで指定されたページ番号で始まる、物理アドレスのsizeバイトを表示します。仮想空間に関連するセキュリティビットはprotで指定されます。

いつものように、以前に処理されていないパラメーターを調べます。これらはpfnパラメータとsizeパラメータであり、読み取りおよび書き込み用に任意の量のメモリを表示できます。
