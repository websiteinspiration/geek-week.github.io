Funciones de llamada de óxido desde Go

Hubo un tiempo en que había un artículo sobre Habr sobre cómo llamar al código Rust desde Go . El artículo no es malo, pero es bastante difícil de entender y de hecho repele a los principiantes del deseo de mirar hacia ambos idiomas. El propósito de esta publicación no es tanto entrar en las tripas de las llamadas en varios idiomas, sino mostrar cuán fácil se puede hacer.



No llegaremos lejos y tomaremos un ejemplo de un libro sobre el aprendizaje del idioma Rust.

Todo lo que hace este ejemplo es iniciar 10 subprocesos, dentro de los cuales incrementa la variable 'x' a 5 millones y muestra un mensaje sobre el final de la secuencia.

use std::thread;

#[no_mangle]
pub extern "C" fn process() {
    let handles:Vec<_> = (0..10).map(|_|{
        thread::spawn(||{
            let mut x = 0;
            for _ in 0..5_000_000 {
                x += 1
            }
            x
        })
    }).collect();

    for h in handles {
        println!("Thread finished with count={}",
                 h.join().map_err(|_| "Could not join thread!").unwrap());
    }
    println!("Done!");
}


También debe editar el archivo de carga agregando una línea
crate-type = ["cdylib"]
Como resultado, se creará una biblioteca con la capacidad de invocar funciones a través de la Interfaz de funciones externas (FFI) .

Vale la pena señalar que libembed.dylib es una biblioteca en Mac OS, en Linux será libembed.so y en Windows será libembed.dll

En general, esto es todo lo que tiene que hacer en la biblioteca Rust. Un ejemplo del libro describe esto con más detalle y no nos detendremos en esto.

Compilamos la biblioteca con el comando:
construcción de carga - liberación
Ahora nuestro objetivo es llamar a este código desde la aplicación Go. Creamos una aplicación simple y, dentro de nuestro proyecto, agregamos la carpeta lib en la que copiamos el archivo /target/release/libembed.dylib . En el interior, creamos un archivo con el nombre de la función y describimos su firma de llamada.

lib / process.h
void process();


Dentro del archivo Go agregamos tales directivas y nuestro main.go se verá así

package main

/*
#cgo LDFLAGS: -L./lib -lembed
#include "./lib/process.h"
*/
import "C"

func main() {
	C.process()
}

También vale la pena mencionar por separado que puede transferir las enseñanzas del programa Go a la biblioteca Rust. Para hacer esto, transformamos la función en la biblioteca Rust para que tome un valor de cadena.

extern crate libc;
use std::thread;
use std::ffi::CStr;

#[no_mangle]
pub extern "C" fn process(name: *const libc::c_char) {

    let buf_name = unsafe { CStr::from_ptr(name).to_bytes() };
    let str_name = String::from_utf8(buf_name.to_vec()).unwrap();

    let handles:Vec<_> = (0..10).map(|_|{
        thread::spawn(||{
            let mut x = 0;
            for _ in 0..5_000_000 {
                x += 1
            }
            x
        })
    }).collect();

    for h in handles {
        println!("{}:Thread finished with count={}\n",
                 str_name,
                 h.join().map_err(|_| "Could not join thread!\n").unwrap());
    }
    println!("Done!");
}


Ensamblamos nuestro proyecto, copiamos la biblioteca a la carpeta lib nuevamente, modificamos el archivo process.h de esta manera
proceso nulo (char * name);

Pasamos la línea desde la aplicación Go (en nuestro caso: "Hola desde Golang") .

package main

/*
#cgo LDFLAGS: -L./lib -lembed
#include "./lib/process.h"
*/
import "C"

func main() {
	C.process(C.CString("Hello from Golang !!"))
}


Todas. Ejecute el programa y obtenga la salida.
