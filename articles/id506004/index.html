Pikiran pertama pada Deno

Sebuah terjemahan dari artikel itu dipersiapkan menjelang peluncuran Pengembang Node.js saja .


Di Codegram, kami senang bereksperimen dengan teknologi baru, dan Deno keluar seminggu yang lalu! Baca artikel jika Anda ingin tahu tentang runtime JavaScript baru ini dan apa pendapat Codegram tentang itu.

Anda mungkin sudah pernah mendengar tentang Deno, atau mungkin tidak, tetapi sekarang Internet penuh dengan informasi tentang hal itu dan ada alasan untuk itu! Sebuah runtime JavaScript baru telah muncul dengan dinosaurus lucu sebagai logo (beberapa orang berpikir bahwa logo tersebut memiliki kaus kaki yang sedih, tetapi tidak ada ...). Kami bahkan men - tweet tentang dia seminggu sebelum rilis pertama, tetapi sejujurnya, kami pertama kali mendengar tentang Deno dua tahun lalu setelah menonton penampilan luar biasa ini oleh Ryan Dahl .

Dalam artikel ini, kita akan membahas Deno secara dangkal dan membagikan pemikiran kita tentang hal itu. Ayo mulai!

Halo Deno!


Deno adalah runtime JavaScript seperti Node. Untuk menggunakannya, Anda harus terlebih dahulu menginstal deno CLI. Untuk melakukan ini, ikuti instruksi untuk sistem operasi Anda di deno.land (pujian nama domain kami!). Deno bukan garpu Node, itu dibuat dari awal pada Rust. Saya menulis artikel tentang programer Rust for JS , Anda dapat membacanya jika Anda ingin tahu lebih banyak tentang bahasa ini!

Setelah menginstal CLI, Anda dapat menjalankan program klasik "Halo, Dunia!" dengan cara berikut:

deno run https://deno.land/std/examples/welcome.ts

Apakah kita menjalankan TypeScript di luar kotak? Iya! TS adalah tamu sambutan di Deno, jadi Anda tidak perlu hal lain untuk menjalankan kode pada TS. Anda dapat menjalankan kode JS juga, tapi saya penggemar TS, jadi saya yang pertama membicarakannya.

Ingatlah bahwa kode Node tidak kompatibel dengan Deno. Namun, saat ini mereka sedang bekerja untuk memastikan kompatibilitas , sehingga di masa depan Anda akan dapat menggunakan paket NPM favorit Anda.

Ada satu hal lagi yang layak disebutkan bahwa Deno adalah kotak pasir secara default. Artinya, pada awalnya Anda tidak memiliki akses ke jaringan atau sistem file. Jika program Anda membutuhkan akses, Anda harus mengizinkannya secara eksplisit (mis. Gunakan --allow-net untuk memberikan hak istimewa jaringan).

Siapa yang belum melihat lelucon ini di Internet?



Meskipun sedikit berlebihan, masih ada beberapa kebenaran dalam lelucon ini. Ketika aplikasi Anda tumbuh, daftar dependensi bertambah, dan Anda berakhir dengan pohon ketergantungan yang besar. Masalahnya adalah bahwa setiap aplikasi memiliki salinan paket @ versi yang Anda gunakan.
Salah satu tujuan utama Deno adalah untuk mencerminkan browser. Saat Anda ingin mengimpor sesuatu ke browser, Anda menulis tag <script>dan menggunakan URL. Deno bekerja seperti berikut:

import { assertEquals } from "https://deno.land/std/testing/asserts.ts";
assertEquals("hello", "hello");
assertEquals("world", "world");
console.log("Asserted! ");

Saya setuju, ini terlihat sedikit aneh. Karena kami tidak memiliki package.json, tidak ada cara untuk menentukan dependensi eksternal, jadi kami mengimpornya ketika dibutuhkan. Pada awal pertama, Deno akan mengunduh file dan menyimpannya dalam cache. Anda dapat menentukan flag --reload jika Anda ingin mengunduh paket lagi.

Anda mungkin memiliki banyak pertanyaan tentang hal ini (saya juga memilikinya), jadi saya mengundang Anda untuk mengunjungi halaman FAQ ini di manual Deno untuk mendapatkan informasi lebih lanjut tentang topik ini.

Runtime kompatibel dengan Browser API


Tujuan penting lain dari Deno adalah kompatibilitas browser, sehingga ia memiliki serangkaian fitur yang tersedia dari lingkup global, seperti mengambil atau menambahEventListener . Anda juga dapat menggunakan objek jendela global (namun, untuk saat ini, saya sarankan menggunakan globalThis). Anda dapat mempelajari lebih lanjut tentang fungsi global dalam dokumentasi .

Runtime juga memiliki Deno global untuk API yang bukan standar web. Anda dapat menggunakannya untuk operasi tingkat rendah seperti membaca file, membuka soket TCP, dll.

Perpustakaan standar yang kaya


Pengembang Deno telah membuat koleksi modul siap pakai standar. Perpustakaan juga dihosting di domain deno.land , sehingga Anda dapat mengimpor modul apa pun ke aplikasi Anda menggunakan URL berikut:

import { v4 } from "https://deno.land/std/uuid/mod.ts";

console.log(v4.generate());

Perpustakaan standar mencakup banyak modul yang berguna, tetapi ada banyak pihak ketiga, dan daftar ini diperbarui setiap hari. Ada satu lagi berita baik: Anda dapat mengimpor modul apa pun yang ditempatkan di URL publik, yang berarti Anda dapat dengan mudah mengimpor modul dari GitHub. Deno memiliki layanan penulisan ulang URL yang dapat Anda gunakan untuk membuat modul Anda tersedia. Anda dapat mempelajari lebih lanjut tentang mekanik ini di halaman ini .

Alat bawaan


Deno CLI juga menyertakan alat bawaan untuk melakukan beberapa tugas umum, seperti menjalankan tes, misalnya, atau bahkan membuat paket! Apakah ini berarti bahwa Deno dapat menggantikan gurauan, lebih cantik dan webpack? Mungkin tidak. Deno adalah solusi yang cukup baru dan belum memiliki ekosistem yang begitu luas, tetapi keberadaan semua alat ini, akses yang dapat diperoleh dengan menginstal satu biner, masih mengesankan.

Kesimpulan


Langsung saja hype wave ini! Saya sangat terkesan dengan Deno. Setelah dua tahun pengembangan dan jalan panjang menuju sukses, runtime JavaScript baru tampak hebat dan saya ingin segera mulai bereksperimen dengannya. Node muncul sekitar 11 tahun yang lalu dan menjadi tonggak baru dalam sejarah JavaScript. Dia membuat bahasa lebih baik, dan saya pikir kita telah mencapai tonggak sejarah baru, saatnya Deno!
