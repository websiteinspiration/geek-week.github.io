我将采访Java程序员。任何级别的一组问题

在我的职业生涯中，我必须是一名开发人员和一名经理，甚至曾经是一家小型初创公司的“ CTO”。而且，当然，我不得不对候选人进行大量采访。因此，我自己制定了进行技术面试的规则：

  • 向所有候选人提出相同的问题
  • 问题应该使得大三学生和大三学生都回答了，但是方式不同，
  • 经过三个问题，应了解候选人的水平。

因此，我为Java开发人员提出了这三个问题。

告诉我们有关HashMap的信息


是的，是一个非常标准的问题，其中有50％的访谈从此开始。我为什么要问他？因为答案可以非常非常不同。

  • 理想的答案始于对问题的陈述。我们为什么需要这样的结构？它解决什么具体任务？解决这个问题的简单解决方案看起来像“直率”，为什么不好呢？
  • 进行历史考察的“奖励积分”，对解决方案作者的了解，至少是其出现的大约年份（当然不是Java）
  • 至少初级的答案必须包括使用该结构的方法。她支持什么方法。
  • 要求中级开发人员了解内部设备，以及对hashCode（）和equals（）函数的协定的描述，其功能要求
  • 奖励点，用于描述hashCode（）和equals（）的非功能要求
  • 考虑在多线程环境中使用HashMap的奖励积分。
  • 考虑HashMap替代方案的“奖励点”-列出它们，比较实施，使用，速度的复杂性（一般，理想和最坏情况）。
  • HashMap替代方案的“奖励积分”以单独的系统组件（例如，数据库和“外部”缓存）形式存在。
  • “奖励积分”，用于描述在数据库中如何使用HashMap，以及为什么通常使用基于二叉树的索引而不是HASH索引。

如您所见，您可以与Senior'om讨论大量的“奖励选项”。您至少可以谈10分钟多少点？

如果您觉得无聊，请用TreeMap替换HashMap。

编写SQL ...


是的，再一次是10年前的每次采访中的“标准”问题。现在，它变得不那么普遍了，显然是由于nosql的流行。但是，处理数据的能力始终是必需的，并且理解笛卡尔积的程序员总是比那些只限于复述教科书的程序员更有用（并且更昂贵）。

问题是：
数据库中有两个表：students和groups。每个学生都有他所属的group_id组的标识符。编写不使用嵌套的SQL查询，列出没有一个学生的组（或组数）

当然，正确答案不仅包含JOIN，而且包含LEFT JOIN（将值与NULL比较）。最有趣的事情是与应聘者讨论编写请求时会犯什么错误，以及如果提出请求将导致请求失败的原因。为什么不能通过相等性与NULL比较？为什么简单的JOIN工作不起作用？

索引的优点是可以加快此查询的操作速度，并可以考虑使用不同方言（不同DBMS）中的SQL选项。

查找代码中的错误


向候选人表达了这样的任务：
假设您从另一个开发人员那里收到了一个Pull请求（合并请求）以进行代码审查。该请求具有大量不同的文件，但是其中一个看起来如下（给出了整个类代码）：

package ru.mycompany.utils;

/** Wrap around Lock to simplify interface */
class LockWrapper {
    private Lock lock;
    public wait() {
        this.lock.wait()
    }
}

您将对此代码发表评论吗？您会进一步跳过带有此类文件的请求（到master分支），还是需要重做，为什么？

当然，这里至少有三个级别。

  • 初级人员应指出此代码中的语法错误。开发人员必须指出为什么代码无法编译。当然，问题不仅在于语法。
  • 开发人员必须知道为什么即使编译代码也无法正常工作。以及如何更正此代码以使其起作用。开发人员必须能够使用java.util.concurrent包中的syncize构造和类，并且知道为什么一秒钟不是三分之一。
  • 理想的开发人员应首先询问为什么需要此代码，他要解决什么问题，即使他解决了此代码中的所有现有问题，他是否也可以解决它。

在回答过程中，我们与候选人一起将代码转换为“理想”形式。当然，最终形式取决于候选人的资格。我最好的候选人在这里发现了13个错误。除了“最重要的错误”。

另外，您可以讨论应该使用哪种自动代码控制方式来发现至少部分错误。

而不是结论


我想给所有不得不面试未来同事的人一个建议。不要试图回答这个人是否合适。以等级的形式（如果您的公司采用了该等级）或您愿意付给候选人以在公司工作的金额来表达您的意见。首先，这将简化您的决定，因为它不会是黑白的；其次-对于将要做出最终决定或比较几次面试结果的人。
