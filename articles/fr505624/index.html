Comment traiter une trame de données avec des milliards d'enregistrements en quelques secondes?

L'analyse Big Data de Python est dans une ère de la renaissance. Cela a commencé avec la bibliothèque NumPy. Cette bibliothèque, soit dit en passant, est l'un des composants des outils qui seront abordés dans ce document. En 2006, le thème du big data prend progressivement de l'ampleur; ce processus s'accélère avec l'avènement de Hadoop. Puis vint la bibliothèque pandas avec ses structures de données DataFrame, qui sont généralement appelées simplement «trames de données». En 2014, le big data est devenu courant; la même année, la plate-forme Apache Spark est apparue. En 2018, la bibliothèque Dask et d'autres outils d'analyse des données en Python ont été publiés.



Chaque mois, je rencontre de nouveaux outils d'analyse de données en Python que je veux vraiment apprendre. En passant une heure ou deux à les étudier, vous pouvez, à long terme, gagner beaucoup de temps. De plus, il est important de suivre les nouveaux développements dans le domaine de la technologie qui vous intéressent. Vous pensez peut-être que cet article portera sur la bibliothèque Dask . Mais ce n'est pas le cas. Aujourd'hui, je vais vous parler d'une bibliothèque Python récemment découverte, qui vaut la peine d'être connue de ceux qui sont engagés dans l'analyse de données.

Bibliothèque Vaex Python



Vaex est une bibliothèque Python hautes performances conçue pour organiser le traitement «paresseux» des trames de données (similaire aux trames de données pandas), conçue pour la visualisation et la recherche de grands ensembles de données tabulaires. Cette bibliothèque est capable de calculer les principaux indicateurs statistiques des données étudiées. Dans le même temps, le traitement d'un milliard d'enregistrements peut prendre environ une seconde. La bibliothèque prend en charge de nombreux outils de visualisation, ce qui permet d'explorer les mégadonnées de manière interactive.

Comparaison de Vaex et Dask



La bibliothèque Vaex n'est pas comme Dask, mais les structures de données qui y sont utilisées sont similaires aux entités DataFrame utilisées dans Dask. Ils sont construits sur la base de trames de données pandas. Cela signifie que Dask hérite des pandas de certaines restrictions sur l'utilisation des trames de données. Par exemple, une exigence selon laquelle les données à traiter doivent être entièrement chargées en mémoire. Dans le cas de Vaex, ce n'est pas le cas.

Vaex ne fait pas de copies de trames de données; par conséquent, en utilisant cette bibliothèque, vous pouvez traiter des trames de données à grande échelle sur des ordinateurs avec une petite quantité de RAM.

Vaex et Dask utilisent tous deux des moteurs de traitement de données paresseux. La principale différence entre eux est que Vaex calcule les valeurs de champ lorsqu'elles sont nécessaires, et dans Dask, vous devez utiliser explicitement la fonction de calcul de valeur.

Afin de libérer tout le potentiel de Vaex, il est nécessaire que les données soient présentées au format HDF5 ou Apache Arrow.

Installer Vaex


L'installation de Vaex est aussi simple que n'importe quel autre package Python:

pip install vaex

Expériences avec Vaex


Créez une trame de données pandas contenant 1 million de lignes et 1000 colonnes. Cela nous permettra d'obtenir un fichier de données volumineux.

import vaex
import pandas as pd
import numpy as np
n_rows = 1000000
n_cols = 1000
df = pd.DataFrame(np.random.randint(0, 100, size=(n_rows, n_cols)), columns=['col%d' % i for i in range(n_cols)])
df.head()

Informations sur l'utilisation de la mémoire

Enregistrez ce bloc de données sur le disque afin de le lire ultérieurement à l'aide de Vaex.

file_path = 'big_file.csv'
df.to_csv(file_path, index=False)

Si vous lisez directement ce fichier CSV à l'aide de Vaex, nous n'en tirerons pas grand avantage. La vitesse de lecture sera similaire à la vitesse fournie par les pandas. Sur mon ordinateur portable, Vaex et les pandas lisent ces données en 85 secondes environ.

Afin de voir les capacités de Vaex, les données CSV doivent être converties au format HDF5 (Hierarchical Data Format version 5). Vaex a une fonction pour effectuer une telle conversion, qui vous permet de travailler avec des données dont le volume dépasse la quantité de RAM disponible. Cela est dû au fait que les données source sont divisées en fragments.

S'il ne parvient pas, par manque de mémoire, à ouvrir un fichier suffisamment volumineux à l'aide de pandas, ce fichier peut être converti au format HDF5 et traité à l'aide de Vaex.

dv = vaex.from_csv(file_path, convert=True, chunk_size=5_000_000)

La fonction ci-dessus crée automatiquement un fichier HDF5 et l'enregistre sur le disque.

Vérifiez le type de variable dv:

type(dv)
# 
vaex.hdf5.dataset.Hdf5MemoryMapped

Lisons maintenant l'ensemble de données de 7,5 Go à l'aide de Vaex. Nous n'avons pas besoin de le lire maintenant - le fait est qu'il est déjà représenté par une variable dv. Mais nous exécutons toujours la commande suivante, en faisant cela afin de savoir quelle est la vitesse de cette opération:

dv = vaex.open('big_file.csv.hdf5')

Vaex prend moins d'une seconde pour terminer cette commande. Mais Vaex, en effet, du fait de l'utilisation de mécanismes de traitement de données "paresseux", ce fichier ne se lit pas.

Maintenant, faisons lire à Vaex le fichier en calculant la somme des valeurs de la colonne col1:

suma = dv.col1.sum()
suma
# array(49486599)

Mais cette équipe m'a vraiment surpris. Il a également fallu moins d'une seconde pour calculer le montant du Vaex. Comment est-ce possible? Et cela est peut-être dû au mappage de la mémoire.

Visualisation de données


Vaex montre également une bonne vitesse de visualisation des données. La bibliothèque dispose d' une fonction spéciale: plot1d, plot2det plot2d_contour.

dv.plot1d(dv.col2, figsize=(14, 7))



Résultat de visualisation des données

Colonnes virtuelles


Vaex, lors de l'ajout d'une nouvelle colonne à l'ensemble de données, crée une colonne virtuelle. Il ne prend pas de mémoire. Les valeurs qui y sont stockées sont calculées, comme on dit, «à la volée».

dv['col1_plus_col2'] = dv.col1 + dv.col2
dv['col1_plus_col2']



Nouvelle colonne

Filtrage efficace des données


Vaex ne crée pas de copies de trames de données lors du filtrage des données. Cela contribue à une utilisation plus efficace de la mémoire.

dvv = dv[dv.col1 > 90]

Agrégation de données


Vaex agrège les données un peu différemment des pandas. Mais le plus important ici, c'est qu'à Vaex, cette opération est très rapide.

Créez une nouvelle colonne contenant les résultats de la comparaison dv.col1 >= 50:

dv['col1_50'] = dv.col1 >= 50

Vaex combine les opérations de regroupement et d'agrégation de données en une seule équipe. La commande suivante regroupe les données par colonne col1_50et calcule la somme de la colonne col3:

dv_group = dv.groupby(dv['col1_50'], agg=vaex.agg.sum(dv['col3']))
dv_group



Résultats d'équipe

Connexions de données


Vaex vous permet de combiner des données sans créer de copies de données en mémoire. Cela, comme d'autres fonctionnalités de la bibliothèque, contribue à l'utilisation économique de la mémoire. La fonction joinci-dessous semblera familière aux utilisateurs de pandas:

dv_join = dv.join(dv_group, on=’col1_50')
