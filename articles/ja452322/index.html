風がどこから来るのか知りたい場合は、ハリケーンのまさに中心に向かってください。

おおよそこの気持ちは、Google Cloud Next '19カンファレンスに参加したことから生まれました。カンファレンスでは、3日間、開発者、プロダクトマネージャー、データスペシャリスト、その他の有識者が、アプリケーション開発とアーキテクチャ、セキュリティ、コスト管理、データ分析、ハイブリッドクラウドの分野で最もホットなニュースを共有しました。テクノロジー、MLとAI、オフサーバーデータ処理など... 30,000人、業界と技術分野の流れ、ほとんどノンストップの講義。かき混ぜる！

2019-2020年のGoogle Cloudに関する重要なニュースをいくつかお話ししましょう。カンファレンスに参加したOWOXの CEOであるVlad Flaxのメモに基づくと、その122のアップデートは見逃せません。

製品：Anthos（以前のCloud Services Platform）


製品はクローズドベータ版ですが、開発者はすでにAnthosの導入を待っています。Anthosを使用すると、開発環境に関係なくハイブリッドアプリケーションを作成して実行できます。ローカルでもクラウドでも、同じレベルのセキュリティと簡単なアクセスが保証されます。

コンテナーとマイクロサービスシステムのおかげで、仮想マシン、サーバー、サービスを世界中（そして場合によっては月）から起動できます



。Anthosの場合：

  • 重要な要素であるGKE On-Premは、コンテナインフラストラクチャのローカルオーケストレーションを担当します。
  • マイクロサービス管理-Istio
  • クラウドの機会-GCP

すべての技術スタッフィングはオープンソースに基づいていますが、そのような贅沢を組み合わせても、月額10,000ドルからの費用がかかります。Anthosビデオプレゼンテーションはこちらからお問い合わせください。

製品：サーバー外ですが、まだ少し曇っていますCloud Run（ベータ）


開発者にとってもう1つの喜びです。ローカルデータ処理とサーバーデータ処理のどちらを選択するかを選択せず​​に、ハーフキックからアプリケーションを実行する方法の問題に答えることができます。

スケーリング、サポート、サーバー設定の問題、およびローカルソリューションの標準的な手順、ライブラリ、言語の制限（カウント、ブルドッグとサイ）の問題を解決する製品をWebアプリケーション開発者向けに統合し、これまでのところベータ版のCloud Runが登場しました。

意味は次のとおりです。これらの2つのアプローチから役立つものをすべて取り入れてください。状態を保持せずに、必要なすべてのライブラリと依存関係が自動的にプルアップされる任意の言語で動作するHTTPコンテナーを、トラフィック負荷が変化した場合にサーバーの容量が自動的にプルアップされる本格的な「クラウド」に配置します。複雑に聞こえますか？

しかし、それは開発プロセス中に価値実現までの時間を実際にもたらします。サーバー側の価格を考えたり、選択したり、尋ねたりする必要がなくなり、コンテナ開発はCloud Codeを使用して簡単に管理および適応されます。作成、起動、結果を得た。

API管理：ハイブリダイゼーション


Apigeeハイブリッド（ベータ版）は、API管理システムの新機能であり、クロスクラウドハイブリッド環境（データセンターまたはパブリッククラウド環境）でAPIを管理する機能を提供します。

また、Google Cloudの基本機能へのアクセスを拡張するための多くの更新が含まれています。必要なセキュリティ警告APIも同様です。

これにより、あらゆる場所で最短かつ安全な方法で行きたいビジネスのタスクの実行が簡単になります。

データ管理：どこでも、どこでも


Google Cloudのデータ管理は、データの量に関係なく、管理、移動、データへのアクセスをシンプルにするという原則に基づいています。したがって、今年は以下のサービスの改善が発表されました。

  • Microsoft SQL ServerでのCloud SQL（特別な表示モード）
  • PostgreSQL、バージョン11でのCloudSQLがサポートされるようになりました
  • Cloud Bigtableのマルチリージョンコピーが利用可能

そして、多くのセキュリティとアクセスの問題が更新されました。たとえば、Google Filestoreクラウドファイルシステムの管理が改善され、統合データベースアクセス調整システムが便利になりました。ベータテスト後にAWS V4キーを使用したフルアクセスが完全に利用可能になります。

これはビジネスにとっても朗報です。世界中のどこからでもデータを管理するためのアクセスが高速化および簡素化され、Google Cloudの運用およびソフトウェアとハ​​ードウェアの基盤が拡大しています。

このサービスをすぐに使い始めるには、超人的なことをする必要はありません。結局のところ、Googleのもう1つの命令は、適切なレベルのセキュリティとプライバシーを維持しながら、技術者ではない専門家のためにすべてを簡素化することです。
