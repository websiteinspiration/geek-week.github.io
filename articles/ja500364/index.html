MoodleとBigBlueButtonの学校学習ポータルを拡張する

2020年3月の終わりに、小さなカムチャッカの学校は、わが国のすべての学校と同じ問題に直面しました：学校に行けない場合に子供を教える方法？

独立した学習のために資料を単純に捨てることは不可能です。フィードバックが必要です。生徒が資料を正しく理解していることを確認する必要があります。

連邦ポータルの調査により、利用可能なレッスンはトピックにはあまりにも原始的であり、素材をアップロードできるレッスンは、増加した負荷に耐えられない可能性が高いことが明らかになりました。したがって、トレーニングポータルの要件が策定されました。

  1. サーバー上のソリューション（自己ホスト型）。負荷に対応できない可能性が高い公共サービスの拒否（後で確認されました）。
  2. 手動登録。生徒はクラスの教師からログイン/パスワードを受け取り、リソースにアクセスして個別に登録しないでください。
  3. サードパーティのソフトウェアをインストールする必要なく、すべてのデバイスのブラウザで作業できます。両親と子供はどちらも家に座っており、すべての人にとって十分なコンピュータがないため、トレーニングポータルは、現在学生が利用できるすべてのデバイス（コンピュータ、タブレット、スマートフォン）で実行する必要があります。追加のソフトウェアをインストールすると問題が発生する可能性があるため、追加のプラグイン（Flashなど）がなくてもすべてがブラウザで機能するはずです。
  4. レッスン資料の構成、完了した各課題に対するフィードバックの存在。生徒が行う作業は、タスクと同じ場所にある必要があります。先生は宿題を整理すべきではありません。教師は、生徒の間違いを自分の仕事ですぐに確認し、そこにコメントを残すことができます。
  5. 生徒の活動を監視する。教師は、特定の生徒がいつ入ってきたのか、彼が何をしていたのかを知る必要があります。
  6. ビデオチュートリアルのエンドツーエンドの承認。生徒にはビデオレッスンで自己紹介する機会を与えないでください。教師は、ビデオレッスンをモデレートするためのすべてのツール（各参加者のマイクとカメラのオン/オフ、プライベートチャットの禁止、一般的なチャットのオン/オフ）を用意する必要があります。

簡単な検索とテストの結果、これらすべての要件を同時に満たすために、Moodle + BigBlueButtonを選択しました。

インストールと構成には1日の休暇があり、教師のトレーニングと組織の問題があり、1週間の休暇がありました。

まず、ドメイン（私はどこでも架空のmyportal.ruを使用します）を購入し、それをDNSサーバーに委任して、インストールを開始しました。

次の形式でcsvに保存します。

username:firstname:lastname:email:cohort1:city:country:lang:password
g1petrov:::g1petrov@myportal.ru:1  (2020)::RU:ru:drkcwf,f,

ファイルエンコーディングはUTF-8で、メモ帳++に簡単に変換できます。

[管理]-[ユーザー]-[ユーザーのダウンロード]を開きます。セパレーター "："プレビュー行 "1000"。エラーの有無を確認し、問題がないかどうかをダウンロードします。

これですべての生徒がいます。教師を作成することは残り、レッスンを作成することができます。

たとえば、BigBlueButton会議。会議を開始するには、[セッションに接続]をクリックします。ビデオチュートリアルが始まりました。



1か月以上にわたり、ビデオチュートリアルの経験が蓄積されています。

  1. オンライン学習には、Fifine K669やDEXP U700などの安価なデスクトップUSBマイクを使用するのが最適です。マイクのボタンが少ないほど良いです。
  2. プレゼンテーションと音声が主なものであったため、高校の教師は月の半ばまでにウェブカメラの使用をほぼ中止しました。ビデオは生徒の注意をそらし、教師が8:10に起きてすぐにレッスンを開始するのを防ぎます。
  3. オンラインレッスンの30分で、教師はほとんどの生徒が自分で同時に学ぶ時間よりもはるかに多くのことを教えてくれます。

他の学校から、「リンクにズームと同じようにしたい」という要望があったため、BigBlueButtonとGreenlightをコントロールパネルとして使用する別のサーバーが発生しました。

これはBBBと同じ方法で設定されますが、最後に-gスイッチが付いています。

wget -qO- https://ubuntu.bigbluebutton.org/bbb-install.sh | bash -s -- -v xenial-22 -s vks.myportal.ru -e help@myportal.ru -g
cd greenlight/
docker exec greenlight-v2 bundle exec rake admin:create

レッスンの中断（ズーム爆撃と呼ばれる現象）に対する保護があります。会議を作成するときに、「接続前のモデレーターの承認」モードを有効にすることができます。生徒が入るたびに、教師の確認が必要になります。教師は、生徒を慣れ親しんだ姓でのみ許可します。



個別の相談のために、部屋をすばやく作成してリンクを共有できるJitsiサーバーが導入されましたが、メインポータルで各生徒用のBigBlueButton会議を作成することをあまり好んでいません。

私たちの経験が誰かにとって有用であり、世界を少し良くしたことを願っています。
