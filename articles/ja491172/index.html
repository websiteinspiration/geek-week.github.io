あなたはReact開発者であり、Vueを試すことにしました。はじめましょう！

ReactとVueはどちらもコカコーラとペプシです。Reactで実行できることの多くは、Vueでも実行できます。同時に、いくつかの違いがあり、そのうちのいくつかは、Vueに対するAngularフレームワークの影響を反映しています。

この記事では違いに焦点を当てているため、すぐにVueの世界に飛び込んで、その利点を確認します。

ReactとVueはどのくらい違いますか？


ReactとVueには、違いよりも類似点があります。

  • これらは、UIを作成するための2つのライブラリです。
  • 速くて軽い;
  • コンポーネント指向のアーキテクチャを持っている。
  • 仮想DOMアプローチを使用します。
  • どちらもHTMLファイルで直接接続することも、Webpackのより複雑なビルドシステムのモジュールとして接続することもできます。
  • どちらにも、ルーティングと状態管理のために、独立しているが広く使用されているサードパーティのライブラリがあります。

顕著な違いは、Vueは通常HTMLテンプレートファイルを使用し、ReactはJavaScriptファイルのみを使用することです。Vueライブラリには、変更可能な状態と、反応性と呼ばれる自動テンプレート再描画システムもあります。

部品


Vueライブラリーでは、コンポーネントは.componentid引数と定義変数を持つオブジェクトを取るAPIメソッドを使用して接続されます。React開発者は、このコードでVueコンポーネントを定義することのなじみのある、またはなじみのない側面に気付くでしょう。

Vue.component('my-component', {
  //<strong> </strong> Props
  props: [ 'myprop' ],
  // <strong> </strong> Local state
  data() {
    return {
      firstName: 'John',
      lastName: 'Smith'
    }
  },
  // <strong>  </strong>Computed property
  computed: {
    fullName() {
      return this.firstName + ' ' + this.lastName;
    }
  },
  // <strong> </strong>Template
  template: `
    <div>
      <p>Vue components typically have string templates.
      <p>Here's some local state: {{ firstName }}
      <p>Here's a computed value: {{ fullName }}
      <p>Here's a prop passed down from the parent: {{ myprop }}
    </div>
  `,

  // <strong>    </strong>Lifecycle hook
  created() {
    setTimeout(() => {
      this.message = 'Goodbye World' 
    }, 2000);
  }
});

テンプレート


ご覧のとおり、コンポーネントにはプロパティがありtemplate、これはHTMLマークアップを含む文字列です。Vueライブラリには、実行時にこの文字列をレンダリング関数に変換するコンパイラが含まれています。これらのレンダリング関数は、仮想DOMエンジンを使用します。独自のレンダリング関数を作成する場合

は、このプロパティを使用できませんtemplate。Reactの世界のJSXマークアップを使用することもできます。しかし、Vueに切り替えてこのようにするのは奇妙です-イタリアに来てピザを拒否する方法...

ライフサイクルフック


Vueコンポーネントには、Reactコンポーネントと同様のライフサイクルメソッドがあります。たとえば、created上記の例のフックstateは、コンポーネントの状態（）がready であるときに、コンポーネントがページにアタッチされる前に開始されます。世界からの反応- getDerivedStateFromProps。

ただし、大きな違いが1つあります。Vueライブラリには、Reactライブラリに類似するものはありませんshouldComponentUpdate。Vue反応性システムのため、これは必要ありません。

コンポーネントの再描画


コンポーネントを初期化する手順の1つは、すべてのプロパティを表示し、それらをゲッターとセッターに変換することです。あなたが見ることができるように、プロパティはmessage、元のデータ（から取られたprops）二つの追加方法を受け- getとset：


Vueはこれらの関数をアタッチして、プログラムが読み取りまたは変更したときに、依存関係の追跡とプロパティの変更通知を有効にします。

可変状態


Vueで状態を変更するためにthis.setState、Reactのように別のメソッドを呼び出す必要はありません。変更を直接書き込むだけです。

// React
this.setState({ message: 'Hello World' });
// Vue
this.message = 'Hello World';

このように状態値が変更されると、setこの値のメソッドが開始されます（上記を参照）。Set新しい値を書き込みますが、彼には別のタスクもあります。彼は、この値が変更されたことと、状態のこの変数の値に依存するレイアウトのその部分を再描画する必要があることをライブラリに通知します。

変数がmessageプロパティ（prop）として子コンポーネントにさらにスローされると、ライブラリはこれを記憶し、子コンポーネントも自動的に再描画されます。そのため、VueライブラリはReactのようなライフサイクルメソッドを必要としませんshouldComponentUpdate。

メインテンプレート


メインプロジェクトテンプレートファイルに関しては、VueはよりAngularに似ています。Reactと同様に、Vueテンプレートはページのどこかにマウントする必要があります。

<body>
  <div id="root"></div>
</body>

// React
ReactDOM.render('...', document.getElementById('root'));

// Vue
new Vue({
  el: '#root'
});

ただし、Reactとは異なり、このルート要素に通常のレイアウトを追加できます。コンポーネントをマウントするときにReactが要素内のすべてを消去する#rootと、Vueは保存します。

<div id="root">
  <div>You can add more markup to index.html</div>
  <my-component v-bind:myprop="myval"></my-component>
</div>

x-templateやinline-templateなどのHTML5拡張機能を使用して、ルートhtmlファイルで子コンポーネントのテンプレートを定義する方法もあります。テンプレート全体をコンポーネント全体の定義から分離するため、これはベストプラクティスとは見なされません。

ディレクティブ


ここでも、Angularと同様に、Vueでは「ディレクティブ」のロジックを使用してコンポーネントの機能を拡張できます。v-たとえば、v-if条件付きのレンダリングやv-bind、正規のHTML属性に式を添付するための、プレフィックス付きの特別なHTML属性があります。

new Vue({
  el: '#app',
  data: {
    mybool: true,
    myval: 'Hello World'
  }
});
<div id="app">
  <div v-if="mybool">This renders if mybool is truthy.</div>
  <my-component v-bind:myprop="myval"></my-component>
</div>

ディレクティブに割り当てられる値はJS式であるため、3項演算子の使用など、データプロパティを参照できます。

プロジェクト組織


Vueライブラリには、react-create-appに相当する公式のものはありませんが、Vue開発者コミュニティによって作成されたビルド、create-vue-appがあります。

Vueでプロジェクトを開始するための公式の推奨事項は、vue-cliです。1つのHTMLファイルを含む単純なプロジェクトから、完全にパックされたWebpack +サーバー側レンダリングプロジェクトまで、すべてを生成できます。

$ vue init template-name project-name

単一のHTMLファイルを含むプロジェクト


Vue.jsの作成者であるEwan Yu（You Yu Xi）は、複雑なアプリケーション用にスケーリングしたり、小さなアプリケーション用に簡略化したりできるため、彼のプロジェクトを「プログレッシブフレームワーク」と呼んでいました。

もちろん、Reactでもそれが可能です。違いは、Vueプロジェクトは通常、使用するES6機能が少なく、JSXをほとんど使用しないため、Babelを使用してコードを転置する必要がないことです。さらに、Vueライブラリ全体が1つのファイルに収まるため、ReactDOMのようなReactの同等のファイルが別に存在することはありません。

これは、単一のHTMLファイルを使用してプロジェクトにVueを追加する方法です。

<script src="https://unpkg.com/vue/dist/vue.js"></script>

重要！テンプレート文字列を使用しないので、テンプレートコンパイラが不要な場合は、これらの関数を持たないライブラリの簡易バージョン-vue.runtime.jsを使用してください。フルよりも約20 KB少なくなります。

1つのファイル内のコンポーネント


Webpackなどのツールを使用してプロジェクトのビルドフェーズを追加できる幸運な場合は、Vueの単一ファイルコンポーネント（SFC、単一ファイル内のVueコンポーネント）を使用できます。これらは拡張子がのファイルで、.vueコンポーネントテンプレート、JS構成、およびスタイルを1つのファイルにカプセル化します。

<template>
  <div class="my-class">{{ message }}</div>
</template>
<script>
  export default {
    data() {
      message: 'Hello World'
    }
  }
</script>
<style>
  .my-class { font-weight: bold; }
</style>

これは間違いなく、Vueライブラリの最も注目すべき機能の1つです。正しいHTMLテンプレートを取得できますが、JSロジックはそのすぐ隣にあり、プレゼンテーションとロジックの分離曲線はありません。

SFCの構築を担当するvue-loaderと呼ばれるWebpack-loaderがあります。アセンブリプロセス中に、テンプレートはレンダリング関数に変換されます。これは、ブラウザーでストリップされたアセンブリvue.runtime.jsを使用するのに理想的です。

Reduxと残り

ReactからVueに切り替えると、開発者は何を得ますか？


この問題を完全に理解するために、私はこの記事を補足して、Reactを選択した方がよい場合と、いつVueを使用すべきかを説明しました。

Vueに切り替えると、新しい人々がプロジェクトに参加しやすくなります。 Vueテンプレートは直接HTMLで記述できます。また、小規模なプロジェクトの場合、最新のES ++機能を理解し、WebpackやBabelなどのツールを使用してコードアセンブリを構成する必要はありません。 VueのコードはjQueryスタイル、つまりテンプレートに直接書き込むことができます。

Vueには、すぐに使用できるステータスメッセージ配信システムもあります。開発者は、非同期メソッドの呼び出しsetState（Reactなど）や、状態の配信を制御するための外部ライブラリ（Mobxなど）に接続する必要はありません。

最後に、Vueを使用して記述されたアプリケーションは著しく簡単になります。多くの場合、必要なのは、ライブラリー自体、またはvue.runtime.jsの簡略化されたバージョンさえ必要になります。Reactの最新の開発は、最大で数メガバイトのBabelとかさばるバンドルがなければほとんど不可能です。

一方、Vueプロジェクトは、クライアントに大規模なアプリケーションまたは複雑なロジックがある場合、複雑すぎる可能性があります。HTMLテンプレートでレイアウトを行うと、アプリケーションの一部をテストすることがより困難になります。JSコードのみを使用するReactは、コードを再利用および拡張するためのより多くの機能を提供します。また、Reactで作成すると、React-Nativeを使用してデバイス用のアプリケーションを作成するときにコードを再利用できます。これはVueには当てはまりません。
