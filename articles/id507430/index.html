Kalender produksi DIY di Postgresql







Halo, nama saya Victor dan saya seorang pengembang di Permata Pengembangan. Saya ingin memberi tahu bagaimana kami menerapkan pembuatan dan penyelesaian kalender produksi di Postgresql.



Alur kerjanya tergantung pada hari libur nasional dan internasional resmi. Dalam aplikasi perusahaan, tugas menghitung tanggal sering memperhitungkan hari kerja dan akhir pekan. Misalnya, layanan seperti "Penerbitan izin bangunan", "Penerbitan izin commissioning" harus diberikan tepat waktu.



Setelah menganalisis tugas, kami sampai pada kesimpulan bahwa cukup untuk menyimpan akhir pekan dan hari libur dalam kalender, karena merekalah yang sulit untuk perhitungan dan dapat berubah sesuai dengan kalender produksi di setiap tahun.



Aturan untuk bekerja dengan kalender:



  • Definisi hari kerja tergantung pada apakah tanggal tersebut ada dalam daftar akhir pekan dan hari libur atau tidak.
  • Jika hari kerja sehubungan dengan transfer menjadi tidak berfungsi, maka itu juga dicatat dalam tabel.
  • Jika hari libur adalah hari kerja, maka, karenanya, catatan dari tabel ini dihapus.


Untuk demonstrasi, kami akan membuat skema terpisah dan menetapkan variabel di sesi sehingga setiap kali kami tidak menentukan skema dalam permintaan: 



create schema calendar;
set search_path=calendar;


Untuk mendukung perhitungan seperti itu, sebuah tabel dibuat dalam database: 



CREATE TABLE d_holidays
(
  holiday_date date,
  comment character varying(1000),
  CONSTRAINT d_holidays_pk PRIMARY KEY (holiday_date)
);


Serta indeks yang memungkinkan Anda untuk mempercepat pencarian tanggal yang diinginkan dalam tabel: 

CREATE UNIQUE INDEX d_holidays_pk_idx
  ON d_holidays
  USING btree
  (holiday_date);


Untuk membuat data, kami menggunakan fungsi standar dari Postgresql yang mengembalikan satu set. Kami akan memeriksa operasi indeks dengan mengisi awal tabel dengan set data yang dihasilkan cukup besar: sepanjang hari dari 01/01/2017 hingga 12/31/2025. 



insert into d_holidays(holiday_date)
select generate_series(to_date('01.01.2017','dd.mm.yyyy'),
            to_date('31.12.2025','dd.mm.yyyy'),
            '1 day');


Kami mendapat 3287 catatan, yang sesuai dengan jumlah hari dari tanggal yang ditentukan. Penting untuk memastikan bahwa parameter "autovacuum = on" diaktifkan dalam konfigurasi Postgresql.



Mari kita lihat rencana kueri: 



explain(analyze)
select comment from d_holidays where holiday_date='09.09.2020';

  Index Cond: (holiday_date = '2020-09-09'::date)

Perhitungan liburan akan dilakukan dalam tahun ini. Setelah memeriksa semua tanggal dari 1 Januari tahun yang ditentukan hingga 31 Desember, pilih hari libur dan tuliskan dalam tabel. Untuk mengimplementasikan algoritma, kami menggunakan bahasa prosedural plpgsql dan konstruksi blok anonim.



Kami akan menyiapkan desain yang membentuk tanggal untuk 1 Januari dan 31 Desember. 



do
$$
declare
    calc_year integer:=2020; -- 
    begin_date date; --    
    end_date date; --    
    dow integer; --  . -1,...,-6,-0
begin
    --   
    if calc_year between 1988 and 2099 then
        begin_date :=to_date('01.01.'||calc_year, 'dd.mm.yyyy');
        end_date :=to_date('31.12.'||calc_year, 'dd.mm.yyyy');
    end if;
end$$;


Untuk menghitung hari dalam seminggu, kami menggunakan fungsi ekstrak, yang menentukan jumlah hari dalam seminggu (dengan mempertimbangkan bahwa hari Minggu adalah 0). Untuk menentukan nama hari dalam seminggu, kami menggunakan konstruksi kasus sederhana.



Kami mendapatkan skrip berikut: 



do
$$
declare
    calc_year integer:=2020; -- 
    begin_date date; --    
    end_date date; --    
    dow_value integer; --  . -1,...,-6,-0
begin
    --   
    if calc_year between 1988 and 2099 then
        begin_date :=to_date('01.01.'||calc_year, 'dd.mm.yyyy');
        end_date :=to_date('31.12.'||calc_year, 'dd.mm.yyyy');
        
        while begin_date<=end_date
        loop
            raise info '%',to_char(begin_date,'dd.mm.yyyy');
            begin_date:=begin_date+interval '1 day';
            dow_value:=extract(dow from begin_date);
            if dow_value in(0,6) then
                insert into d_holidays(holiday_date,comment)
                        values(begin_date,
                            case when dow_value=6 then '' else '' end);
            end if;
        end loop;
    end if;
end$$;


Mari kita menganalisis algoritme untuk menambahkan liburan: 



insert into d_holidays(holiday_date,comment) 
    values(to_date('01.01.2020','dd.mm.yyyy'),' ') 
        on conflict(holiday_date) do update set comment=EXCLUDED.comment;
…        
insert into d_holidays(holiday_date,comment) 
    values(to_date('04.05.2020','dd.mm.yyyy'),'  04.01.2020') 
        on conflict(holiday_date) do update set comment=EXCLUDED.comment;


Mungkin ada situasi ketika hari libur menjadi hari kerja. Dalam kasus ini, algoritme adalah sebagai berikut (misalnya, 2018 dipilih, karena tidak ada hari libur pada tahun 2020): 



delete from d_holidays where holiday_date=to_date('29.12.2018','dd.mm.yyyy'); 
delete from d_holidays where holiday_date=to_date('09.06.2018','dd.mm.yyyy');


Perhatikan desainnya: 



on conflict(holiday_date) do update set comment=EXCLUDED.comment;


Inovasi ini muncul di Postgresql versi 9.5 dan memungkinkan Anda untuk secara efektif menangani konflik saat memasukkan catatan. Dalam contoh kami, jika hari libur sudah merupakan hari libur, maka untuk menghindari konflik atas kunci utama saat menambahkan catatan dengan tanggal yang ada, kami memperbarui deskripsi tanggal tanpa membuat duplikat.
