VPN / Mikrotik两因素身份验证-简单且可扩展

你好！

在撰写本文时，系统提示我阅读用户文章的类似内容恩库斯涅佐夫。视图数量表明社区对该主题感兴趣。

因此，我决定与您分享我自己的解决方案，该解决方案以前由我实施，其基本形式包括：

  • 低级别的输入和简单的代码（供另一位员工理解/调试）
  • 简单的ROS脚本不会创建任何负载，甚至无法在hAP Lite上运行
  • 可扩展性-连接大量VPN网关以减少负载或地理分布的能力
  • 能够将Mikrotik CHR用作VPN服务器
  • “ 1xN”-1个SMS网关，用于无限数量的路由器，并可能随着负载的增加而扩展
  • 能够将单独的路由器绑定到“特定的”调制解调器（用于什么？-稍后再介绍）
  • 在远程服务器上仅使用一个PHP脚本
  • 不论哪个设备启动了VPN连接，通过SMS链接授权
  • 能够在服务器上记录员工授权
  • 通过从多个调制解调器随机发送SMS来提高容错能力并降低系统负载


UPD 1.0-更新了PHP代码。现在，它包括记录对脚本的所有请求，并在指定多个网关的情况下通过多个网关随机发送消息。


问题陈述和解决方案


任务1


在开发此解决方案的过程中，我们的任务是最大程度地减少USB调制解调器的数量-降低拥有成本，简化管理，将调制解调器本地放置在一个地方，从而提高系统的可维护性。

决定使用安装在“可靠通道”上的一台路由器作为基本SMS网关。我唯一不知道的是系统的负载，调制解调器可以应付的费用以及操作员是否会阻止它。因此，我可以连接多个调制解调器，并将系统进一步分为几组。此外，可以通过简单的USB集线器将其他调制解调器连接到基本网关。但是，要想发挥全部功能，用最少的功能，则无论路由器数量如何，都需要一个调制解调器。

任务2


提供在路由器安装区域中使用本地SIM卡的可能性。
示例：在哈萨克斯坦拥有多家商店的广泛分支机构网络。从俄罗斯联邦发送短信会非常昂贵。该解决方案使哈萨克斯坦的员工可以从本地号码接收短信。

但是在反思的过程中，事实证明，该解决方案已在任务1中找到并实现。

任务3


来自移动设备的隧道授权，而不必实际位于授权设备上。

目标是不仅可以授权用户隧道，还可以授权任何vpn连接：Mikrotik-> Mikrotik，Server-> Mikrotik等。在这种情况下，负责这些隧道的用户只需要遵循SMS消息中的链接即可，显示要登录的隧道。

为了能够以这种方式进行授权，有必要将脚本传输到外部资源-可以从任何地方访问的服务器，并使用授权代码在防火墙中标记地址列表并随后进行解锁。

该问题的解决方案已经明确暗示要使用RouterOS API（或SSH）。API是最简单的选择。

逻辑学


  1. 用户使用预先添加的用户名和密码登录
  2. VPN网关将其ip放入地址等待列表中，然后向服务器发送POST请求
  3. 服务器接收数据，检查并生成授权码，然后将SMS发送给以下形式的联系电话：“为用户79001112233连接开放-http_：//synome.ru/？Ruid = vrG7yYMbZ6＆auth = YU6zc”
  4. 用户可以从任何设备访问链接
  5. 服务器向路由器发出请求，并在注释的授权码中检查条目的地址列表中是否存在。如果找到记录，则将其删除，然后用户显示成功的登录页面

在所有其他情况下，如果某些内容未通过测试，则用户将收到请求错误。

/ip firewall address-list

添加一条规则，阻止VPN用户从等待列表到其他任何主机的流量 
/ip firewall raw
add action=drop chain=prerouting dst-address-list=!Allow-list src-address-list=VPN-blocked disabled=no
