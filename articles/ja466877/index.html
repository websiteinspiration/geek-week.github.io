マルウェアSustesが更新され、Eximの脆弱性（CVE-2019-10149）を介して拡散しています。

暗号マイナーSustesの新しい波が、Eximメールサーバーの6月の脆弱性を感染に使用するようになりました。8月11日以降、PT Network Attack Discoveryネットワークセンサーは、受信ネットワークトラフィックでメールサーバーを操作する試みを検出します。

スキャンはアドレス154.16.67 [。] 133から行われ、RCPT TOフィールドのコマンドは、アドレスhxxpに悪意のあるbashスクリプトをロードします：//154.16.67 [。] 136 / main1。スクリプトのチェーンにより、ホストにXMRマイナーがインストールされ、crontabに追加されます。スクリプトは、公開SSHキーを現在のユーザーのauthorized_keysリストに追加します。これにより、攻撃者は将来、マシンにパスワードなしのSSHアクセスを取得できるようになります。

さらに、SustesはSSHを介してknown_hostsリストから他のホストに伝播しようとしています。それらは公開鍵を使用して自動的に接続されることが期待されています。感染プロセスは、利用可能なSSHホストで再び繰り返されます。



配布には別の方法があります。 Sustesは一連のPythonスクリプトを実行します。最後のスクリプトは-hxxp：//154.16.67 [。] 135 / src / sc-ランダムなRedisサーバースキャナーを含み、自動実行のためにcrontabに自身を追加し、そのキーを信頼できるSSHのリストに追加します-脆弱なRedisサーバー上のキー：
