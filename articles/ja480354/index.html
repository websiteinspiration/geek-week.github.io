JavaScript配列メソッド.map（）、. filter（）、. reduce（）の概要

私は、JS-配列のメソッドを使用する方法を理解すると.map()、.filter()と.reduce()私は見て、読んで、に耳を傾けていることはすべて、私には非常に困難に思えました。これらの概念は、他とは何の関係もないいくつかの独立したメカニズムと見なされていました。彼らの本質を理解して理解するのは私にとって大変でした。 これらは基本的なものであり、その理解は「開始者」と「未開始者」の境界のようなものだと聞きました。それから私はそれらについて真実を伝えてもらいたいのです。これは、これらの3つのメソッドが、反復可能なオブジェクトを分類する理由が3つの機能カテゴリのいずれかに当てはまる場合が多いという事実を象徴していることにあります。





以前に作成したコードを調べたところ、95％の場合、文字列または配列の要素を繰り返し処理したときに、次のいずれかを実行したことがわかりました。

  • 特定の命令シーケンスの各値への適用（メソッドのアナログ.map()）。
  • 特定の基準に一致する値のフィルタリング（と同じ.filter()）。
  • データセットを単一の集計値（アナログ.reduce()）に削減。

それは真実の瞬間でした。そのとき、私はこれらの方法の本質を理解し、私が長い間知っていたものとのそれらの関係を見ました。

練習するために、私は古いコードを取り、これらのメソッドを使用してリファクタリングしました。これは非常に便利であることがわかりました。

さて、さっそくこれらの方法についてお話ししましょう。特に、ループを使用するための広範なスキームの代わりに、これらの方法を使用する方法を見てみましょう。

.Map（）メソッド


このメソッドは.map()、以下を実行する必要がある場合に使用されます。

  1. 反復可能オブジェクトの各要素に対して特定のシーケンスのアクションを実行する必要があります。
  2. おそらく変更された値を返す必要があります。

価格を含む配列の各要素について、元の価格と消費税を含む新しい金額を見つける必要がある単純な例を考えてみます。

const prices = [19.99, 4.95, 25, 3.50];
let new_prices = [];
for(let i=0; i < prices.length; i++) {
   new_prices.push(prices[i] * 1.06);
}

これを使って同じことをする方法は.map()次のとおりです：

const prices = [19.99, 4.95, 25, 3.50];
let new_prices = prices.map(price => price * 1.06);

かなり簡潔な構文構造を使用しています。それでは、この例を見てみましょう。このメソッド.map()はコールバックを受け入れます。これは、配列の要素に適用される関数です。この場合、メソッドの宣言に続く括弧内で直接宣言されるのはアロー関数です。

パラメータ名priceは、配列要素を操作するときに使用される名前です。アロー関数にはパラメーターが1つしかないため、宣言するときに括弧なしで実行できます。

矢印（=>）の後の式は、コールバックの本体です。関数本体には式が1つしかないため、中括弧やキーワードなしで実行できますreturn。

そのようなエントリがわかりにくい場合は、この例の少し拡張したバージョンを次に示します。

const prices = [19.99, 4.95, 25, 3.50];
let new_prices = prices.map((price) => {
   return price * 1.06
});

.Filter（）メソッド


このメソッド.filter()は、反復可能オブジェクトから特定の要素を選択する必要がある場合に適用されます。この方法を使用する場合、フィルターに対応する値は最終結果に含まれ、フィルターから除外されないことに注意する必要があります。つまり、渡された関数.filter()が返すすべてのものはtrue破棄されます。

整数の配列から奇数の要素のみを選択する必要がある例を考えてみます。ここでは、演算子を使用して除算の余りを取得し、配列の各要素を2で除算した余りがあるかどうかを調べます。余りが1の場合、これは対応する数が奇数であることを示しています。最初に、通常のループを使用してこの問題を解決する方法を見てみましょう。

const numbers = [1,2,3,4,5,6,7,8];
let odds = [];
for(let i=0; i < numbers.length; i++) {
   if(numbers[i] % 2 == 1) {
      odds.push(numbers[i]);
   }
}

の.filter()ようなメソッドは、.map()1つのコールバックを受け入れます。コールバックには、反復可能なオブジェクトの要素が順番に渡されます。

const numbers = [1,2,3,4,5,6,7,8];
let odds = numbers.filter(num => num % 2);

ここでの作業は、の例と同じように構成されてい.map()ます。渡さ.filter()れた矢印関数は1つのパラメーターのみを使用するため、括弧なしで使用します。彼女の体には1つの式しか含まれていないため、中括弧で囲むことはできませんreturn。

.reduce（）メソッド


そして、ついにメソッドに到達しました.reduce()。彼は、今日考えられている3つの方法の中で最も理解しにくい人だと思います。このメソッドの名前は、いくつかの値を1つに減らすために使用されていることを示しています。しかし、それをパーツか​​ら特定の値を収集できる方法と考える方が簡単で、何かを「縮小」または「縮小」できる方法とは思えないように思えます。

このメソッドが呼び出されるコードを作成するとき、特定の初期値が最初に設定されます。メソッドが配列の値を反復処理するときに、この初期値が変更され、変更された形式で次の反復に渡されます。

これは、配列の要素の合計を計算するために必要な古典的な問題です。私たちの場合、それは特定の慈善プロジェクトへの寄付の額を見つけることにあります：

const donations = [5, 20, 100, 80, 75];
let total = 0;
for(let i=0; i < donations.length; i++) {
   total += donations[i];
}

メソッド.map()と.filter()は異なり、メソッド.reduce()は2つのパラメーターを受け取るコールバックを必要とします。これはバッテリーと現在の値です。バッテリーは最初のパラメーターです。各反復で変更され、以下に渡されるのは彼です。

const donations = [5, 20, 100, 80, 75];
let total = donations.reduce((total,donation) => {
   return total + donation;
});

メソッドに.reduce()2番目の引数を渡すこともできます。これは、バッテリーの初期値の役割を果たすものです。昨日の金額が450ドルで、今日の寄付に関する情報が配列に格納されているとすると、2日間の寄付の合計額を知りたいとします。

const donations = [5, 20, 100, 80, 75];
let total = donations.reduce((total,donation) => {
   return total + donation;
}, 450);

概要


私は今あなたがJSの配列の方法を考え出したことを願って.map()、.filter()と.reduce()。それらを、コードを読みやすくするメカニズムと考えてください。これらを使用すると、従来のループを使用して取得したプログラムよりもコンパクトなプログラムを作成できます。しかし、その主な長所は、コードの根底にある意図を明確に表現できることです。

これらのメソッドのおかげで、かなり長い間書かれたコードは読みやすくなります。ループ内に配置された構造を掘り下げるのではなくfor、最終的な目標を理解するためだけにこれを行うのではなく、これらのメソッドのいずれかの名前を確認することによって、特定のコードの存在理由の一般的な考えを形成することができます。
