左再帰を障害として数回言及しましたが、それを処理する時が来ました。主な問題は、左再帰下降を伴うパーサーがスタックオーバーフローが原因で即座にクラッシュすることです。

この架空の文法規則を考えてみましょう。


expr: expr '+' term | term

この文法のフラグメントを左の再帰パーサーメソッドに実装すると、次のような結果が得られます。


def expr():
    if expr() and expect('+') and term():
        return True
    if term():
        return True
    return False

したがって、それexpr()は、call expr()で始まる呼び出しで始まる呼び出しexpr()で始まります...これは、例外として表されるスタックオーバーフローでのみ終了できますRecursionError。


従来の解決策は、文法を書き直すことです。前のパートでは、それだけを行いました。実際、上記の文法規則は次のように書き換えることができます。


expr: term '+' expr | term

ただし、構文解析ツリーを構築する段階では、その形状は異なります。演算子'-'を文法に追加した場合、これは状況を台無しにする可能性があります（それa - (b - c)はとは異なるため(a - b) - c）。これは通常、グループ化や反復などのより強力なPEG関数を使用して解決され、上記のルールを次のように書き換えることができます。


expr: term ('+' term)*
