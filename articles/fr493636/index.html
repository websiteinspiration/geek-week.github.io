I. Calcul des rendements annuels: développement d'outils


Immédiatement, la formule suivante vient à l'esprit
[profit] / [fonds investis] / [nombre de jours] * 365 = [rentabilité en% / par an]
Elle travaille vraiment quand elle a investi de l'argent et après un certain temps a décollé plus.

Que dois-je faire si je dépose de l'argent en plusieurs fois à des moments différents ou lorsque des paiements sont effectués périodiquement? Dans cette situation, il est difficile de comprendre combien et combien de temps était sur le compte. Naturellement, je ne suis pas le premier à rencontrer un tel problème et pour le moment il existe deux solutions:

Si vous additionnez tous les flux de trésorerie (soustrayez les investissements et ajoutez les retraits), la différence qui en résulte est le revenu. Par exemple, ils ont investi 10 000 roubles dans une banque, un an plus tard, ils ont retiré 10 600 roubles, alors que les revenus étaient de 600 roubles. Vous pouvez également obtenir un revenu en multipliant le montant de l'investissement par le taux d'intérêt. Par exemple, nous multiplions 10 000 roubles par un taux bancaire de 6%, nous obtenons 600 roubles.

Plus intéressant, nous modifions les flux de trésorerie d'un nombre qui compenserait le profit. Voici une description plus compétente , mais en substance toute la logique réside dans la formule suivante:

IRR est le rendement annuel inconnu. Nous substituons les valeurs de l'exemple par un dépôt bancaire: - 10 000 roubles / (1 + 6%) ^ 0 + 10 600 roubles / (1 + 6%) ^ 1 = 0. Tout converge. Il reste maintenant à exprimer l'IRR et à le calculer déjà par programme à l'aide de boucles ou de la fonction PURE dans Excel.

function XIRR(values, dates, guess) {
  // Credits: algorithm inspired by Apache OpenOffice
  
  // Calculates the resulting amount
  var irrResult = function(values, dates, rate) {
    var r = rate + 1;
    var result = values[0];
    for (var i = 1; i < values.length; i++) {
      // result += values[i] / Math.pow(r, moment(dates[i]).diff(moment(dates[0]), 'days') / 365);
      result += values[i] / Math.pow(r, daysLag(dates[i],dates[0]) / 365);
    }
    return result;
  }

  // Calculates the first derivation
  var irrResultDeriv = function(values, dates, rate) {
    var r = rate + 1;
    var result = 0;
    for (var i = 1; i < values.length; i++) {
      // var frac = moment(dates[i]).diff(moment(dates[0]), 'days') / 365;
      var frac = daysLag(dates[i], dates[0]) / 365;
      result -= frac * values[i] / Math.pow(r, frac + 1);
    }
    return result;
  }

  // Check that values contains at least one positive value and one negative value
  var positive = false;
  var negative = false;
  for (var i = 0; i < values.length; i++) {
    if (values[i] > 0) positive = true;
    if (values[i] < 0) negative = true;
  }
  
  // Return error if values does not contain at least one positive value and one negative value
  if (!positive || !negative) return '#NUM!';

  // Initialize guess and resultRate
  var guess = (typeof guess === 'undefined') ? 0.1 : guess;
  var resultRate = guess;
  
  // Set maximum epsilon for end of iteration
  var epsMax = 1e-10;
  
  // Set maximum number of iterations
  var iterMax = 50;

  // Implement Newton's method
  var newRate, epsRate, resultValue;
  var iteration = 0;
  var contLoop = true;
  do {
    resultValue = irrResult(values, dates, resultRate);
    newRate = resultRate - resultValue / irrResultDeriv(values, dates, resultRate);
    epsRate = Math.abs(newRate - resultRate);
    resultRate = newRate;
    contLoop = (epsRate > epsMax) && (Math.abs(resultValue) > epsMax);
  } while(contLoop && (++iteration < iterMax));

  if(contLoop) return '#NUM!';

  // Return internal rate of return
  return resultRate;
}

II. Calcul des rendements annuels: analyse de la demande


Avant de comprendre les formules et d'écrire du code pour calculer une seule valeur, j'ai cherché sur Internet tout entier et je n'ai pas trouvé de solution toute faite. Mais maintenant que la table est prête, «et il a vu que ça s'est bien passé», il est devenu intéressant de vérifier - existe-t-il une demande potentielle pour une telle solution?

Je n'ai pas beaucoup d'investisseurs familiers pour mener une interview problématique, nous nous tournons donc vers des méthodes de vérification rapides et abordables.



Les statistiques de Yandex montrent un certain intérêt dans ce domaine, mais la taille du marché n'est pas suffisante pour qu'il soit rentable de concevoir et de développer un service distinct.

D'autre part, mon enquête sur vc.ru a montré que 46% utilisent des tableaux pour la comptabilité.



Peut-être, en raison du petit échantillon, les résultats sont dans la marge d'erreur.

Néanmoins, il y a clairement des gens qui n'ont pas trouvé le service fini et ont compilé une tablette. Que se passe-t-il si c'est l'option la plus pratique pour eux ou si les difficultés sont si insignifiantes que vous ne voulez même pas ouvrir Google Play ou l'App Store?

Pour vérifier cette hypothèse, j'ai créé une enquête dans les formulaires Google avec la possibilité de laisser du courrier pour une version brute du programme.



Pour remplir au moins un peu de trafic, j'ai écrit un article sur vc. Résultat: 1095 vues, 1 personne a laissé une demande. La conversion de 0,1% ne prouve pas une faible demande, car elle dépend également de nombreux facteurs: la clarté de l'article, l'intérêt du public, la localisation de l'action cible, etc. La demande reçue suggère que la personne n'était pas trop paresseuse pour donner du courrier pour résoudre des problèmes de comptabilité.

Pour en savoir plus sur le problème et les lacunes des solutions existantes, vous avez besoin de personnes. Le trafic provenant d'une publication gratuite est temporaire et n'est pas suffisamment ciblé. J'ai décidé de faire de la publicité contextuelle pour les requêtes clés.

Vous ne pouvez pas envoyer des gens à la table immédiatement - ils ne comprendront pas ce qui se passe. En conséquence, vous avez besoin d'une page de destination, et le plus simple sera le mieux.
