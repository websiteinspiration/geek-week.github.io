マイクロコントローラーが1.6 Gbpsでデータを読み取る方法

全てにおいて良い日！それは決して起こらなかったし、ここで再び。前回の記事から十分な時間が経過し、新たな課題が生じています。また、100 Mbpsの速度でデータを送信していた場合は、1600 Mbpsでスイングする必要がありました

。KDPV（小説のヒーロー）では、その速度でデータを読み取ることができました。



したがって、私の次のプロジェクトでは、事前にわかっている量で32ビットのデータストリームを50 MHzの速度で（ただし、これは1.6 Gbpsと同じになります）読み取る必要がありました。10000にしてください。1つのポートからDMAを使用してすぐに読み取るのは問題ありません。 -しかし、残念ながら、適切なプロセッサがありませんでした（誰かがコメントでこの問題を修正したことを願っています）。何らかの理由で、速度に適したすべてのポートは何らかの理由で16ビットです。

しかし、そのような些細なことは私たちを止めません-私たちは一度に2つのポートから読みます！確かに、これは一般的なケースでは必要な程度の制御と同期で常に実行できるとは限りませんが、私たちの場合、すべてがそれほど悲しいことではありません-信号があり、その後20 nsのデータがポートに保持されます。

そして、プロセッサ-400 MHzのstm32h750と200 MHzのバスとタイマーがあるので、すべてがうまくいくはずです。

信号で単一のDMA転送をトリガーするのは簡単なケースのようです。しかし、DMAだけがそのような機会を持ちません。ポートは割り込みを発行できますが、DMAを制御できません。しかし、私たちのプロセッサには良い点があります-DMAMUXにはDMAチャネル用のイベントジェネレータがありますが、このジェネレータには2つの適切なオプションがあります-EXTIT0割り込みまたはTIM12タイマーからの信号を使用します（チップ開発者には奇妙なファンタジーがありました）。

中断する時間はありません-空になるには約47クロックサイクルも必要です。クロックサイクルは2.5 nsです...

しかし、タイマーに間に合うように。 100 MHzの外部信号からタイマーを接続し、タイマーの長さを1に設定するだけで、TRGO出力がDMAMUXジェネレーターをトリガーし、コマンドを発行してDMAを送信し、ポートを読み取ってデータをメモリに送信します。

しかし、やめて！ポートは16ビットであり、32があります...さて、別の2番目のポートを読み取ろうとすることができます...これだけのために2番目のDMAチャネルが必要で、同じバスを使用します。つまり、読み取る時間がありますが、メモリにデータを書き込む時間がありません。理論的には、このプロセッサにはさまざまな種類のメモリがあり、プロセッサ構造の全体像を見ると、DMAとRAM_D1の両方のメモリが200 MHzの周波数で同じバス上にあることがわかります。実際に検証する必要があります。

DMA1->LIFCR |= ~0;

そしてもちろん、dataおよびdata2配列を目的のメモリセグメントに配置する必要があります。これは次のように行われます。

__attribute__((section(".dma_buffer"))) uint16_t data[10240],data2[10240];

中央プロセッサ（まだ400 MHz）を使用して愚かなコピーを実装しました。

       uint16_t * ptr = cpudata;
        volatile uint16_t * src = &(GPIOE->IDR);
        volatile uint16_t * src2 = &(GPIOD->IDR);
        for (register int i = 0; i < 10000; i++) {
                *ptr++ = *src;
                *ptr++ = *src2;
        }

検証のために、cpudataデータは別のメモリに配置され、最速（まあ、確かに64Kのみ）が最速のメモリ（これも400 MHz）DTCMRAMでした。

結果


テスト中に、CPUの助けを借りて2つのポートから12.5 MHzの速度で読み取ることが可能であることがわかりました。そして、25 MHz。そのため、オプションは機能しません...

DMAおよびそのような母親の助けを借りて、 TIM12は50 MHzの速度で正常に読み取ることができ、数時間のテストでエラーは発生しませんでした。両方のポートが読み取られましたが、2番目のDMAでの読み取りがどれだけ遅れているかを測定することはまだできていません...
