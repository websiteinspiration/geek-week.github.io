なぜfinalで継承を制限するのですか？

GoFから「クラス継承よりも構成を優先する」というこの有名な声明を聞いたことがあるでしょう。そして、原則として、静的に決定された継承は動的な構成と比較してそれほど柔軟ではないという長い考えがありました。


柔軟性はもちろん便利な設計機能です。ただし、アーキテクチャを選択するときは、主に保守性、テスト容易性、コードの可読性、およびモジュールの再利用に関心があります。したがって、これらの優れた設計基準では、継承も問題になります。「そして今、継承をまったく使用しないのですか？」- あなたが尋ねる。


継承によるクラス間の強い依存関係が、システムアーキテクチャを非常に厳しく脆弱なものにする方法を見てみましょう。そして、コードで最も神秘的でとらえどころのないキーワードの1つを使用する理由- final。策定されたアイデアは、簡単な横断的な例で示されています。記事の最後には、finalクラスでの作業に便利なトリックとツールがあります。

    {

    }

    public function getCommentKeys(): array
    {
        return $this->commentBlock->getCommentKeys();
    }

    public function viewComment(string $key): string
    {
        $this->cache->increment($key);
        return $this->commentBlock->viewComment($key);
    }

    public function viewComments() : string
    {
        $commentKeys = $this->getCommentKeys();

        }
        return $this->commentBlock->viewComments();
    }
}
