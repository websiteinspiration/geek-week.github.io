本「コトリン。専門家のためのプログラミング」

こんにちは、habrozhiteli！Josh SkinとDavid Greenhallによる本は、Big Nerd Ranchの人気のあるKotlin Essentialsコースに基づいています。鮮明で有用な例、主要な概念と基本的なAPIの明確な説明は、Kotlin言語を紹介するだけでなく、その機能を効果的に使用する方法を教え、JetBrains IntelliJ IDEA開発環境を習得できるようにします。

あなたがJavaを超えて行きたいと思っている経験豊富な開発者であるか、または最初のプログラミング言語を学んでいるかどうかは問題ではありません。JoshとDavidが、基本原理からKotlinの高度な使用法までをガイドし、信頼性の高い効率的なアプリケーションを作成できるようにします。


この本は誰のためのものですか。


私たち（著者）は、さまざまな能力の開発者向けにこの本を書きました。Java機能が不足している経験豊富なAndroid開発者、Kotlin機能に関心のあるサーバーコード開発者、そして効果的なコンパイル言語を学ぶことにした初心者向けです。

この本はAndroidのサポートに興味を持っているかもしれませんが、Android向けのKotlinでのプログラミングに限定されていません。さらに、この本では、Android向けのKotlinプログラミング手法について1つの章（21章）でのみ説明しています。それでも、Kotlinを使用してAndroidアプリケーションを開発するトピックに関心がある場合、この本では、KotlinでのAndroidアプリケーションの作成プロセスを簡略化する基本的なテクニックを紹介します。

Kotlinは他のいくつかの言語の影響を受けていますが、Kotlinで正常に動作するように設計されている方法を知る必要はありません。時々、JavaコードとKotlinコードを比較します。Javaでの開発経験がある場合は、2つの言語間の関係を理解するのに役立ちます。また、そのような経験がない場合は、別の言語で同じ問題を解決する例が、Kotlinの形成に影響を与えたアイデアを理解するのに役立ちます。

この本の使い方


この本は参考資料ではありません。私たちの目標は、Kotlin言語の一貫した学習です。プロジェクトで作業し、その過程で言語を学びます。より効果的にするには、本を読むときにすべてのコード例を試すことをお勧めします。例を使用すると、「筋肉」の記憶を発達させ、ある章から別の章に移動するための洞察を提供するのに役立ちます。

次の各章は、前の章に基づいています。章をスキップしないことをお勧めします。他の言語で作業中にトピックを学習した場合でも、少なくともここでそれを読むことをお勧めします。多くのことは、Kotlinでは異なる方法で実装されています。まず、変数やリストなどの紹介トピックから始め、次にオブジェクト指向の関数型プログラミング手法に移り、Kotlinがこのような強力なツールになる理由を理解してもらいます。本の終わりまでに、Kotlinの初心者から上級者まで行きます。

急ぐべきではないことを付け加えておきます。開発中に、リンクでKotlinのドキュメントを使用します：kotlinlang.org/docs/reference。実験中に発生する多くの質問に対する回答があります。

抜粋。拡張


拡張機能を使用すると、型宣言を明示的に変更せずに、型に機能を追加できます。カスタムタイプや、リスト、文字列、Kotlin標準ライブラリのその他のタイプなど、制御できないタイプの拡張機能を使用します。
拡張機能は継承の代替手段です。これらは、クラス定義を使用できない場合や、クラスにサブクラス化を許可するオープン修飾子がない場合に、タイプに機能を追加するのに適しています。

多くの場合、Kotlin標準ライブラリは拡張機能を使用します。たとえば、第9章で学習した標準関数は宣言された拡張であり、この章ではそれらの宣言の例をいくつか示します。

この章では、まずサンドボックスプロジェクトに取り組み、次にこの知識を適用してNyetHackコードを最適化します。まず、Sandboxプロジェクトを開き、Extensions.ktという新しいファイルを作成します。

拡張関数宣言


最初の拡張機能を使用すると、任意の程度の熱意をStringに追加できます。Extensions.ktで宣言します。

リスト18.1 タイプString（Extensions.kt）の拡張機能の追加

fun String.addEnthusiasm(amount: Int = 1) = this + "!".repeat(amount)

拡張関数は他の関数と同じ方法で宣言されますが、1つ異なる点があります。拡張関数を定義するときに、拡張機能が機能を追加する受信型と呼ばれる型も指定します。 （拡張可能な型を「レシーバー」と呼んだ第9章を思い出してください。）addEnthusiasm関数では、受け入れ可能なString型が指定されています。

addEnthusiasm関数の本体は、文字列を返す1つの式にすぎません。この引数の内容と、1つ以上の感嘆符が、amount引数の値に応じて異なります（1がデフォルト値です）。 thisキーワードは、拡張機能が呼び出される宛先オブジェクトのインスタンス（この場合はStringインスタンス）を参照します。
これで、Stringの任意のインスタンスに対してaddEnthusiasm関数を呼び出すことができます。main関数で行を宣言し、addEnthusiasm拡張関数を呼び出して結果を表示することにより、新しい拡張関数を試します。

リスト18.2 Stringレシーバーオブジェクト（Extensions.kt）のインスタンスの新しい拡張機能を呼び出す

fun String.addEnthusiasm(amount: Int = 1) = this + "!".repeat(amount)

fun main(args: Array<String>) {
      println("Madrigal has left the building".addEnthusiasm())
}

Extensions.ktを実行し、拡張関数が意図したとおりに感嘆符を文字列に追加するかどうかを確認します。

この機能を文字列インスタンスに追加するために文字列をサブクラス化することは可能ですか？IntelliJで、Shiftキーを2回押してSearch Everywhereダイアログボックスを開き、検索ボックスに「String.kt」と入力して、String宣言のソースコードを確認します。次のクラス宣言が表示されます。

public class String : Comparable<String>, CharSequence {
      ...
}

openキーワードはStringクラスの宣言にないため、継承によってStringをサブクラス化して新しい機能を追加することはできません。前述のように、制御できない、またはサブクラスの作成に使用できないクラスに機能を追加する場合は、拡張機能が適しています。

スーパークラス拡張宣言


拡張機能は継承に依存しませんが、継承と組み合わせてスコープを拡大できます。Extensions.ktでこれを試してください：easyPrintという名前でAny型の拡張を宣言します。拡張はAnyに対して宣言されているため、すべてのタイプで使用できます。主に、println関数呼び出しを、StringへのeasyPrint拡張呼び出しに置き換えます。

リスト18.3 任意の拡張子（Extensions.kt）

fun String.addEnthusiasm(amount: Int = 1) = this + "!".repeat(amount)

fun Any.easyPrint() = println(this)

fun main(args: Array<String>) {
      println("Madrigal has left the building".addEnthusiasm()).easyPrint()
}

Extensions.ktを実行し、出力が変更されていないことを確認します。

Anyタイプの拡張機能を追加したので、サブタイプにも使用できます。Intの内線呼び出しを追加します。

リスト18.4 easyPrintはすべてのサブタイプ（Extensions.kt）で利用できます。

fun String.addEnthusiasm(amount: Int = 1) = this + "!".repeat(amount)

fun Any.easyPrint() = println(this)

fun main(args: Array<String>) {
      "Madrigal has left the building".addEnthusiasm().easyPrint()
      42.easyPrint()
}

ジェネリック拡張関数


しかし、addEnthusiasmの前後に「Madrigalが建物を出ました」という行を印刷したい場合はどうでしょうか。

これを行うには、チェーンで呼び出す機能をeasyPrint関数に追加します。関数呼び出しのチェーンはすでに見ました。関数がレシーバーオブジェクトまたは後続の関数を呼び出すことができる別のオブジェクトを返す場合、関数はチェーンに参加できます。
easyPrintを更新してチェーンを呼び出します。

リスト18.5 easyPrintを呼び出しチェーンを変更する（Extensions.kt）

fun String.addEnthusiasm(amount: Int = 1) = this + "!".repeat(amount)

fun Any.easyPrint()= println(this): Any {
     println(this)
     return this
}
...

次に、easyPrint関数を2回呼び出してみます。addEnthusiasmの前と後です。

リスト18.6 easyPrintを2回呼び出す（Extensions.kt）

fun String.addEnthusiasm(amount: Int = 1) = this + "!".repeat(amount)

fun Any.easyPrint(): Any {
     println(this)
     return this
}

fun main(args: Array<String>) {
      "Madrigal has left the building".easyPrint().addEnthusiasm().easyPrint()
      42.easyPrint()
}

コードはコンパイルされませんでした。最初のeasyPrint呼び出しは許可されましたが、addEnthusiasmは許可されませんでした。タイプ情報を見て、これが発生する理由を確認します。easyPrintをクリックし、Ctrl-Shift-P（Ctrl-P）を押して、表示される拡張機能のリストから、最初の拡張機能を選択します（「Madrigalが建物.easyPrint（）を残しました」）。 （図18.1）

easyPrint関数は、呼び出された文字列を返しますが、Anyを使用してそれを表します。addEnthusiasmは、Stringでのみ使用できるため、easyPrintから返された値に対して呼び出すことはできません。


この問題を解決するには、一般化された拡張を作成します。easyPrint拡張関数を更新し、Anyではなくジェネリック型をアクセプターとして使用します。

リスト18.7 easyPrintの一般化（Extensions.kt）

fun String.addEnthusiasm(amount: Int = 1) = this + "!".repeat(amount)

fun <T> AnyT.easyPrint(): AnyT {
     println(this)
     return this
}
...

ここで、拡張機能が一般化されたタイプTのパラメーターをレシーバーとして使用し、AnyではなくTを返すと、レシーバーオブジェクトの特定のタイプに関する情報がチェーンに渡されます（図18.2）。


Extensions.ktをもう一度実行してみてください。今回は行が2回出力されます。

Madrigal has left the building
Madrigal has left the building!
42

新しい一般化された拡張関数は、どのタイプでも機能し、それに関する情報も処理します。ジェネリック型を使用する拡張機能を使用すると、プログラムでさまざまな型を処理できる関数を作成できます。

ジェネリック型の拡張機能は、Kotlin標準ライブラリでも利用できます。たとえば、let関数の宣言を見てください。

public inline fun <T, R> T.let(block: (T) -> R): R {
      return block(this)
}

letは一般的な拡張関数として宣言されており、すべての型で機能します。これは、引数（T）としてレシーバーオブジェクトを取り、新しい型Rの値を返すラムダを取ります。

第5章で学習したインラインキーワードに注意してください。以前に与えたのと同じアドバイスがここに適用されます。ラムダを受け入れる場合、拡張関数を組み込みとして宣言すると、メモリのオーバーヘッドが減少します。
