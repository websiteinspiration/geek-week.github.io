Node-RedによるHomebridgeオートメーション

私の意見では、homebridgeの主な欠点の1つは、高度なスクリプトを作成する機能がないことです。すべてのオートメーションはホームセンターに割り当てられます。ホームセンターは、iPad（充電に接続されている）、Apple TV、またはHomePodです。誰もがこれらのデバイスを持っているわけではありません。さらに、HomeKitのオートメーションは非常に貧弱です。この状況はNode-Redを使用して修正できます。この記事は、homebridge がすでにインストールおよび構成されているユーザーを対象としています。



簡単に言うと、Node-REDは、さまざまなノード（ブロック）間にリンクを構築することにより、ハードウェア、API、オンラインサービスを接続するためのツールです。

公式ウェブサイトでの定義：
Node-RED is a programming tool for wiring together hardware devices, APIs and online services in new and interesting ways.
It provides a browser-based editor that makes it easy to wire together flows using the wide range of nodes in the palette that can be deployed to its runtime in a single-click.


Raspbianは次のようにインストールされます：

sudo apt-get install nodered
sudo systemctl enable nodered.service
sudo service nodered start

デフォルトでは、Node-Redはポート1880で実行されます。さまざまな機能のブロックのセットを備えたモジュールの

巨大なライブラリがあります。惜しみなく、homebridge。node-red-contrib-homebridge-automationをインストールします。

cd ~/.node-red
npm install node-red-contrib-homebridge-automation
sudo service nodered restart

電球をつけてみましょう。これを行うには、hb-control、ファンクションブロック（このブロックはオンにするコマンドを送信します）をドラッグして挿入します（この記事のフレームワーク内では、スクリプトを手動でアクティブにするだけで済みます）。初めて、hbブロックの場合、PINを指定し、デバイスのリストを更新する必要があります。次に、必要なブロックを追加でき



ます。注入ブロックのボタンをクリックすると、ライトが点灯します。hb-contollに加えて、hb-event、hb-status、hb -resumeブロックもあります。ブロックの詳細な説明は公式リポジトリにあります。

機能ブロックコード
var ret_msg={};
ret_msg.payload = {};
ret_msg.payload.On = true;
return ret_msg;


関数 ブロックは、JavaScript機能を使用してデータを操作するように設計されています。ブロックは、ペイロード変数とともにmsg変数を受け入れます。デフォルトでは、Node-Redのほとんどのブロックはペイロードパラメーターで機能します。Debug ブロックについて説明する価値があります。これを使用すると、ブロックの出力を確認して、アイデアが機能しない場合のエラーを理解できます。 では、面白いことをしましょう。たとえば、MagicCubeを 90度回転させた後、ライトの状態を変更します。最後の記事で



、homebridgeでzigbeeデバイスを友達にする方法を示しました。 MagicCubeは、他のzigbeeデバイスと同様に、ペアリング後にMQTTブローカーに表示されます。

Node-RedはそのままMQTTプロトコルをサポートします。 MQTTブロックを追加し、サーバーアドレスとトピックキューブを指定します。



キューブは次の状態を転送できます：
シェイク、ウェイクアップ、フォール、タップ、スライド、flip180、flip90、rotate_left、rotate_right スイッチ

ブロックを追加して、ある状態を別の状態から分離します。スイッチでFlip90を定義し、シェイクとタップの条件を設定します上の出力をhb-statusに接続して、電球の現在のステータスを取得します。HB-ステータスブロックは、ブロックに接続されていますInvert On Valueは反転状態であり、その出力はhb-controllに接続されており、キューブが回転すると、電球のオン/オフ状態が変化します。次のようになります。



値コードで反転
var ret_msg=msg;
ret_msg.payload.On = !msg.payload.On;
return ret_msg;
