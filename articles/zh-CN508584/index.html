今年的ARKit开发和ARKit 4中的新功能：定位锚



近年来，我一直在为Apple平台开发本地游戏方面非常活跃，并且对AR中游戏流程的可能集成感兴趣。因此，我尝试监视WWDC每年宣布的所有更新。尽管近年来SpriteKit，SceneKit和GameplayKit没有更新，但Apple继续积极更新和推广可与这些游戏工具集成的ARKit和RealityKit。



尽管在WWDC上的一次演讲中，苹果公司很少关注增强现实，但本周发布的新版软件开发工具包（SDK）对于参与创建AR应用程序的每个人来说确实非常有用和有趣。在ARKit 4中，为开发人员带来了新的机遇，所有具有以上处理器A12 Bionic的 iOS / iPadOS设备都可以使用。



如您所见，近年来，苹果在AR技术开发方面的投资已被证明是合理的，因为使用计算机可视化，机器学习和运动检测功能，您现在可以使用固定相机创建更复杂的交互式计算机图形世界。事实证明，与竞争对手创建基于智能手机的虚拟现实系统的尝试相比，该技术的开发更加成功和富有成效。



该工具还增加了重要的新深度检测功能，这些功能可在配备Apple LiDAR Scanner的设备上使用（当前仅在最新的iPad Pro上可用）。但是，ARKit 4中最重要的事情也许是位置绑定，它允许开发人员将虚拟对象放置在现实世界中的特定位置。



LiDAR：先进的远程跟踪系统



在带LiDAR扫描仪的iPad Pro系列发布之后， Apple 在ARKit 3.5中引入了Scene Geometry API 。我预计苹果将在其下一代iPhone中增加LiDAR扫描仪，该产品计划在今年年底发布，因此该功能可能是下一次发布中讨论最多的功能。



新的Scene Geometry API使您可以使用LiDAR扫描仪收集信息，并根据获取的数据创建世界的拓扑图。此信息可用于识别某些物理对象，将它们放置在舞台上，并创建真实世界和虚拟世界中对象之间交互的游戏模拟。



ARKit 4帮助您将虚拟对象放置在真实人物的前面或后面，并使用“ 光线投射”功能来真实地显示事件，以模糊真实和数字内容之间的界限。





在iOS 14中， Apple通过LiDAR扫描仪的支持扩展了iOS设备的功能，以更好地确定它们与虚拟环境中的对象之间的距离。机器学习可让您将从设备的广角相机获得的彩色RGB图像与来自LiDAR扫描仪的深度指示器进行组合，以创建密集，深沉的图像。深度数据的更新频率为60hz，这使iOS不仅可以在舞台上显示对象，还可以确保在虚拟环境中实时放置对象。



LiDAR还允许您完善一个称为“射线投射”的功能，该功能是一种渲染方法，它使用计算几何在二维平面上创建三维空间。此外，Apple在ARKit的早期版本中提供了跟踪功能，但仅在配备True-Depth前置摄像头的设备上可用。 ARKit 4通过使面部跟踪可用于配备A12 Bionic处理器或更高版本的所有设备（包括最近发布的下一代iPhone SE），大大扩展了这些功能。面部跟踪功能使开发人员可以创建将人的图像放置在虚拟内容之上并实时观看其表情的应用程序。





Location Anchors



当然，在Apple宣布的所有新功能中，LiDAR扫描仪提供的功能令人印象深刻。 ARKit 4的新Location Anchors技术现在为街道带来了更高质量的AR内容，从而使开发人员可以为虚拟对象设置经度和纬度。然后，ARKit 4使用来自Apple Maps的这些坐标和数据将AR对象放置在现实世界中特定高度的特定位置。



下一代AR接口管理过程称为位置锚。它使您可以根据环境准确锁定设备，并且据Apple称，这比使用GPS更为精确。这整个过程是基于设备本地的高级机器学习技术来控制的。



结果，当开发人员将虚拟对象（例如，虚拟雕塑）放置在现实世界中时，该对象将被保存并显示在该位置，以便使用Apple AR设备查看该对象的每个人都可以在此看到该对象。位置。定位锚将首先出现在洛杉矶，旧金山，芝加哥，迈阿密和纽约等主要城市，然后，在今年夏天结束之前，将有更多的城市出现。



WWDC会话中的位置锚示例




许多开发人员都可以理解位置锚的重要性，这证明了苹果公司在开发与AR相关的设备和技术方面的深远计划。如今，许多专注于增强现实技术的初创公司正试图占领创新市场，以开发其功能，而苹果本周悄悄地在ARKit 4中推出了许多新功能，而且没有任何失败。



由于该公司拥有自己的地图数据，并且拥有数亿个不断收集位置信息的设备，因此它在确定许多开发人员的位置的基础上提供AR功能最为有利。这些新功能将有助于创建下一代应用程序，这些应用程序最终将使AR成为领先技术。



定位锚的工作原理



借助ARKit 4，您现在只需设置坐标即可轻松将AR内容绑定到现实世界中的特定位置。以前基于图像识别或代码扫描以在特定位置触发动作的基于AR的应用程序现在具有实时更新的功能，并可以利用基于位置的功能。



作为Apple Maps数据库转换的一部分，该公司长期以来从世界各地的城市街道上的摄像机和3D LiDAR收集数据。要绑定到某个位置，ARKit将从云中下载设备周围的虚拟地图并将其映射到摄像机数据。使用GPS，ARKit可以快速准确地确定您在现实世界中的位置。所有信息处理均直接在您的设备上使用机器学习技术进行。



可用性



正如我已经指出的那样，所有具有GPS和A12芯片及更新型号的设备都支持位置锚点功能，即地理跟踪。由于此功能要求该区域先前已映射到Apple卡，因此仅在美国某些城市可用。截至2020年6月，其支持区域只有五个城市，但我真的希望苹果会继续扩大其支持区域。



地理跟踪基于名为ARGEoTrackingConfiguration的新ARConfiguration，可轻松检查兼容设备及其可用性。



首先，检查您的设备是否支持此功能： 



guard ARGeoTrackingConfiguration.isSupported else { return }


现在，您可以检查设备是否在受支持的城市中。如果是这样，请在ARView中启动地理跟踪配置。如果使用RealityKit，则无法应用自动配置，因此必须手动启动它。 




ARGeoTrackingConfiguration.checkAvailability { available, error in
    guard available else { return }
    arView.session.run(ARGeoTrackingConfiguration())
}


重要的提示。checkAvailability功能需要网络连接才能加载AR资源。如果设备未连接到Internet，则可用将为false。



建筑物位置锚



定位锚如何工作？毕竟，众所周知，ARKit使用自己的坐标系来确定设备的相对位置，而在现实世界中，位置是通过纬度和经度来描述的。在地理跟踪中，通过创建单个坐标系解决了该问题。 ARKit轴自动与罗盘重合，其中X轴指示向东的方向，Z轴指示向南的方向。



创建ARGeoAnchor所需要知道的就是单独的GPS坐标。这是创建快照的方法，例如，定位金门大桥。为了提高可靠性，我们使用精确到小数位的坐标系。 




let coordinate = CLLocationCoordinate2D(latitude: 37.8185, longitude: -122.4738)
let geoAnchor = ARGeoAnchor(name: "Golden Gate Bridge", coordinate: coordinate)


如果需要，可以指定以米为单位的高度，默认情况下，通过将参数设置为高于海平面可以接受高度。 

let geoAnchor = ARGeoAnchor(name: "Golden Gate Bridge", coordinate: coordinate, altitude: 67)


现在，您可以将锚添加到主场景。在RealityKit，这个工程基础上，AnchorEntity工具从ARGeoAnchor。 


arView.session.add(anchor: geoAnchor)
let geoAnchorEntity = AnchorEntity(anchor: geoAnchor)
arView.scene.addAnchor(geoAnchorEntity)


现在，创建应用程序，您可以添加与所选位置相关的任何其他对象，并根据需要旋转并放置在现实空间中。



空间坐标之间的转换



如果要获取应用程序场景中特定点的GPS坐标，则ARKit可以使用ARGeoAnchor和XYZ ARKit坐标系轻松实现此目的。只需在屏幕上一键就可以了！

let geoAnchor = ARGeoAnchor(coordinate: coordinate, altitude: altitude)}
