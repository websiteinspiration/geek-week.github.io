DINS Java EVENINGへようこそ6月26日

19歳のStaro-Petergofskyの私たちのオフィスで開催される次のオープンなDINS Java EVENINGに、Java開発に関心のあるすべての人を招待して喜んでいます。

会議では、TextBackの経験を知り、Vert.xからSpringにどのように移行するかを学びます。Genestackの同僚が、大規模なプロジェクトでGradleのビルドシステムをゼロから構築した経験を共有します。

カットの下-レポートとスピーカーの詳細、会議に参加するために登録するためのリンク。




報告書


「非同期マイクロサービス-Vert.xまたはSpring？」（TextBackのCTO、Alexander Fedorov）


AlexanderがTextBackサービスについて話します。彼らがどのようにVert.xからSpringに移行するか、彼らが直面する困難とそれらがどのように生き残るかについて。このレポートは、非同期サービスの使用を開始し、このためのフレームワークを選択したいユーザーにとって興味深いものになります。

Alexanderは、Javaで8年以上の産業開発の経験があり、現在はスタートアップでの生活を楽しんでいます。彼はサンクトペテルブルグ工科大学のメレンゲでポリテクニックの大学院を卒業し、学生にコース「ニューロインフォマティクスとネオテクノロジー」を教え、Dell EMCはDevOpsでコースを教えました。

高度なGradleビルド（Genestackのバックエンドチームリーダー、Nikita Tukkel）


Nikitaは、大規模および超大規模なビルドに特徴的な特定の問題の解決策について説明します。このレポートには、Gradleの基礎に関する情報はほとんど含まれていません。Gradleを完全に初めて使用する人にとっては、その一部が十分に理解されていない可能性があります。このレポートは、プロジェクトで効果的なビルドシステムを構築する際の問題（100を超えると確信できるモジュールの数）を心配している人にとって興味深いでしょう。

Nikitaは30年以上プログラミングを行っており、そのうち過去15年間はJavaでした。電子ワークフロー、フィンテック、そして今-バイオインフォマティクス。彼はGradle、Kotlin、およびJ2EEに見えないすべてのものを愛しています。
