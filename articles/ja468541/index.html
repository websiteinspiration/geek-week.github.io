ドラッグ＆＆-視覚障害者向けのコンポーネントをドロップしますか？冗談ですか？

Trello、Gmail、Aviasales、Google Keepを一体にするものは何ですか？それぞれに、マウスでドラッグできるDrag-＆-Drop-コンポーネントがあります。これらのコンポーネントを使用すると、Webサイトが非常に便利になります。現実には、このパターンを使用することが多いためです。物事を移動するのは自然なことなので、ウェブ上では、黒板のステッカーや冷蔵庫の磁石など、画面上の要素をある場所から別の場所に移動したいと考えています。 ToDoリストの並べ替え、ダッシュボードの整理、ファイルのダウンロード-ページに要素をドラッグアンドドロップしないと、これらすべてのイベントを想像することはできません。



しかし、ニュアンスがあります。そのようなコンポーネントを開発するとき、ユーザーは画面や要素を物理的に見て、マウスでそれらをつかんでドラッグできると思います。多くの場合、そうですが、視覚に問題のあるユーザーのセグメントを忘れます。私たちは、単にそれらを見ない多くの人々のグループが使用できないコンポーネントを作成します。ドラッグ-＆-ドロップコンポーネントにもアクセスできるようにする方法は？

Sergey Krieger  はSinnerSchraderのフロントエンド開発者です。フロントエンドの開発に加えて、彼はWebアクセシビリティのトピックに積極的に興味があり、それを増やすいくつかの方法について話します。カットの下-画面を表示せず、他のデバイスを使用してブラウザーと対話する人のためのドラッグ＆ドロップコンポーネントの作成に関するストーリー。

ファイルをアップロードする


ファイルのアップロード機能は、メールにファイルを添付するなど、サイトにもあります。したがって、私の例では、メールはGmailです。



画面上に点線の要素があります。ここにファイルをドラッグします。



これを行うには、ファイルが配置されているエクスプローラを開き、マウスでファイルを破線のゾーンにドラッグします。ドラッグアンドドロップ後、アイテムの状態が変化し、ファイルを解放すると、ダウンロードが開始されます。ダウンロードの最後に、すべてが正常にダウンロードされたことをユーザーに通知します。

キーボード


キーボードがあれば、すべてが簡単に解決されます。inputタイプの意味要素がありfileます。使用する場合、この要素をクリックすると、エクスプローラーダイアログボックスが自動的に開き、OSからファイルを選択できます。ウィンドウスタイルは、デザイナーの要望に応じて、完全に任意に設定できます。



アイテムをクリックすると、ダイアログボックスが開きます。ネイティブエレメントを使用したのでinput、Tabキーでこのエレメントに移動し、スペースバーを押すと、ダイアログボックスが開きます。要素はEnterキーで選択され、すべてが同じように行われます。

すべてがマウスとキーボードで動作します。スクリーンリーダーの場合、デフォルトではネイティブ要素によって提供されない処理を実行できます。

スクリーンリーダー


「select」要素が機能し、OSでダイアログボックスを開くことができます。ただし、ロードが開始されると、ダウンロードのステータスについてユーザーのスクリーンリーダーを更新するために非常に興味深いことができます。


スクリーンリーダーをオンにします。キーボードを使用して、OSからファイルを選択する項目に移動します。

-ボタンアプリケーションを選択します。

スクリーンリーダーでボタンを押します。

-選択ボタンを押します。対話の始まり... Image image.png。3月31日... 0％... 16％ダウンロード... 73％ダウンロード... 100％ロード Image.pngファイルが正常にアップロードされました。

どうした ダイアログボックスを開いた後、ファイルを選択しました。スクリーンリーダーはOSに関連しており、ブラウザーとは関係がないため、ここでは何もする必要はありません。ファイルシステムは、他のすべてのアプリケーションと同様に、適切に処理できます。

ファイルを選択してダウンロードを開始すると、ダウンロードステータスに関するユーザーのスクリーンリーダーの更新を開始しました。進行状況バーに、ダウンロードされたファイルの数が視覚的に表示されます-10％、20％... 100％。ダウンロードが正常に完了すると、ファイルがダウンロードされたという通知が表示されます。

ここでは、先ほどお話しした通常のライブリージョンテクノロジーを使用しました。スクリーンリーダーを更新し、ARIA Live属性で作成した要素にテキストを配置しました。スクリーンリーダーのユーザーは、何パーセントがダウンロードされ、ダウンロードが成功したかを知っていました。このような単純な方法で、アクセス可能な要素を作成できます。

概要


ネイティブ要素。ネイティブ要素の使用は素晴らしいです。時間と神経を節約し、スクリーンリーダーを使用してユーザーに優れたサービスを提供します。それらはこれらの要素に慣れ、それらがどのように機能するかを知っており、慣れ親しんだ使用経験を壊す必要はありません。

HTML + JavaScript + ARIA。ほとんどすべてのドラッグアンドドロップ要素は、適切なマークアップ、JavaScript、およびARIA属性によってアクセス可能にすることができます。私はファンタジーを追加しますが、それがなくても多くの機会があります。

「選択モード」。サイト内で要素を移動するには、「選択モード」を使用します。要素を選択し、移動してチェックを外すと、要素は元の場所に残ります。

コンテキストメニュー。前の方法の代替方法：リストから要素を移動する場所を選択し、選択すると、要素がそこに移動します。
