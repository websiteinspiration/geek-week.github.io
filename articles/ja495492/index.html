1. App Store Connectの最初のステップ

メンバーシップ（登録/ Apple Developer Programへの参加）を仮想カードで支払った後（これは、インターネット経由で支払う場合、私の名前ではなく「デジタルカード」を示す必要があります。これは私の銀行にあります。別のカード所有者がいる場合があります）。本人確認が必要でした。私のアカウント情報はカード所有者のデータと一致しませんでした。
これは私の登録をチェックする人からの手紙のテキストの一部です：

1.2支払い確認


アプリケーションが収益性のあるものである場合（有料、または組み込みの購入がある場合）、プロファイルで支払いデータを指定する必要があります。

私たちは行く：

  • App Store Connect。
  • 契約、税金、銀行。
  • 「有料アプリ」に行きます。
  • 必要事項を記入します。これは、連絡先情報、納税フォーム、通貨銀行口座の表示になります。「受益者の種類」の納税フォームには「個人/個人事業主」と記載したので、個人情報（氏名と電話番号）を常に記載するとともに、銀行の個人口座に開設した外貨口座も一分で記載しました。
  • 確認のために送信し、確認と「アクティブ」の大事なステータスを待ちます。

アプリ内購入を設定するためのワークフローで これについて読むこともできます。

1.3アプリケーションをプレビューするためのビデオ


アプリケーションページの設定では、スクリーンショットに加えて、より多くのビデオをダウンロードできます。YouTube動画へのリンクは、Google Playに挿入するだけです。App Storeでは、異なる見積もり：
アプリケーションをプレビューするためのビデオをダウンロードするには、SafariおよびOS Xバージョン10.10以降を使用する必要があります。
結局のところ、画像は任意のOSの任意のブラウザーからダウンロードでき、ビデオはMacOSでのみ利用可能なSafariからのみダウンロードできます。

私はすでにビデオを完成させましたが、側面のサイズに合いませんでした。

動画の簡単な仕様（動画の読み込みに失敗した場合のApp Store Connectのエラーテキスト）：

仕様には、さまざまなデバイスに対するより多くの権限がリストされています。私のページには、上記のリストの権限を持つ4本のビデオがありました。

2.4プロジェクトの組み立てについて


私の組み立てには、平均20〜50分かかります。コンテンツが変更されなかったが、構成の何かを修正できる20分間。

ただし、エンジンが更新された場合や、コンテンツに重大な変更があった場合は、アセンブリが増える可能性があります。平均して50分まで増加しました。

VPSでリモートコンパイルすると、エンジンのソースコード、プロジェクト、一時ファイルなどがコピーされます。これはすべて[[ホーム] / UE4 /]にあります。

プロジェクトの設定によっては、生成されるIPAファイルのサイズが異なる場合があります。
受信したファイルのサイズは、オプション「ビルド構成」（プロジェクト設定-プロジェクト-パッケージ）-開発または出荷の影響のみを受けます。

「配布用」フラグ、および選択した証明書とプロファイルは、サイズに影響しません。

「配布用」フラグが設定されているのは、配布証明書とプロファイルを選択することだけです。それ以外の場合、プロジェクトはアセンブルされません。

また、「配布用」フラグが設定されている場合、「Distro_」接頭辞がファイルに追加され、配送アセンブリ中に、接尾辞「-IOS-Shipping」が追加されます。

たとえば、私のアセンブリの寸法：

  • 開発ビルド-186 Mb。
  • 出荷時の組み立て-491 Mb。

どちらのファイルも、テスト用にUSB経由で電話にアップロードできます。

Shippingアセンブリ中に取得された大きなファイルについては、私が理解しているように、さまざまなデバイスのリソースが含まれています。

3.2 IPhonePackager


アプリケーションは、IPhonePackagerプログラム（パス " [EnginePath] \ Engine \ Binaries \ DotNET \ IOS \ IPhonePackager.exe "）を介してインストールされます。
起動時に、プロジェクトを選択するためのダイアログボックスが開きます。閉じる必要があります。開いたプログラムで、[高度なツール]タブに移動し、[ハードウェアにIPAをインストール...]ボタンをクリックします。ipaファイルを選択します。インストールが完了するまで待ちます。
電話でのインストールには、20〜80秒かかります（ファイルサイズと月相によって異なります）。

私は定期的にIPhonePackagerを介してアプリケーションをインストールできませんでした（UE 4.24への移行前の実際の状況です）。

エラー（UE 4.24より前）：「FileName.ipaの「インストール\更新」は、「...

エラーの特異性は、ファイルの送信がハングする（ファイルが部分的に送信される）か、非常に迅速に電話にコピーされる（たとえば、1秒あたり0.5ギガバイト）ことです。

解決策（常に機能するとは限らない）：ケーブルを電話に再接続するか、別のケーブルを接続するか、電話を再起動します。しかし、これが役に立たないこともあるので（iTunesが電話を見たにもかかわらず）、テストを延期する必要がありました。

3.3 TestFlight


TestFlightを使用してアプリケーションをテストすることもできます。テストは招待により終了しました。

App Store ConnectのTestFlightで：

  • 「App Store Connectユーザー」サブセクションで、自分を追加します。
  • 「テスターとグループ」セクションで、テスターのグループを追加してから、アプリケーションをテストするユーザーの電子メールを追加します。招待状と、テストプログラムに接続してテスト対象のアプリケーションをインストールするためのコードが送られます。

：テスターの作成したグループは、2つのセクションがあるテスターやアセンブリを：
  • 「テスター」には、テストに接続するユーザーのメールアドレスと、これらのユーザーがアプリケーションをインストールできるアプリケーションへのリンクがあります。
  • 「アセンブリ」には、テストに使用できるアセンブリがあり、手動で指定されます。

また、携帯電話にTestFlightアプリケーションをインストールする必要があります。

開発者、管理者、またはアカウント所有者はコードを入力する必要はありません。アプリケーションの最新のダウンロードバージョンは、App Store Connectにダウンロードした直後に利用可能になります。

ただし、他のすべてのユーザーを招待する必要があり（アクティベーションコードが送信されます）、テスターグループで示されるバージョンを利用できます。

5ゲーム内購入


5.1 App Store Connectでの設定


確認のためにアプリケーションを送信する前に、すぐにゲーム内購入を作成しました。結局のところ、それらはアプリケーションと共にチェック/承認されます。
承認された購入の場合は、購入ウィンドウが表示された
ショッピングウィンドウのスクリーンショット（購入ごと）を添付することをお勧めします。すべての購入のスクリーンショットが1つありました。スクリーンショット自体は、ゲームからではなくエディターから取得され、修正されました。

なぜならスクリーンショットは特定のサイズ（640x920ピクセル）にする必要があります。画像の一部を互いに押し付けなければなりませんでした。したがって、スクリーンショットとゲームでのショッピングウィンドウの外観は異なります。



購入ボタンは、クリックしても何も起こらない場合でも、表示されている必要があります。

最初は購入ボタンを隠し、それらに関する情報（価格、名前、通貨など）を受け取った後にのみ表示しました。これを行う必要はありません。このため、私のアプリケーションと購入は検証に合格しませんでした。

5.2設定ファイルとメソッド「RestoreInAppPurchase」のオプション「bUseStoreV2」


デフォルトでは、プロジェクトのオプション「bUseStoreV2 = true」が有効になっています。「[EngineFolder] /Config/IOS/IOSEngine.ini」を参照してください。ゲーム内での購入は彼女にはうまくいきませんでした。そして、この問題は、フォーラムとAnswerHubへの投稿数に基づくと、私だけの問題ではありませんでした。

解決策：「[UnrealProject] /Config/IOS/IOSEngine.ini」ファイルに「bUseStoreV2 = false」という行を登録しました。ファイルがない場合は作成します-プロジェクトの例を見るか、インターネットで検索します。 もちろん、購入するたびにApple IDのユーザー名/パスワードを入力する必要がありますが、私にとっては何もないよりはましであり、理解する時間はありませんでした。 購入データの復元はまだ機能していません（方法「RestoreInAppPurchase



"）、そして最も重要なことに、購入の価格は表示されませんでしたが（" ReadInAppPurchaseInformationメソッド " ）、Androidではすべて正常に機能しました。しばらくして、「RestoreInAppPurchase」の使用を削除しました（オフにできる広告がありましたが、広告を削除したため、この購入も削除しました。購入した購入のデータを復元する必要もありませんでした）。その後、奇跡的に、ゲームでの価格の読み込み。
