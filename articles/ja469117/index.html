2019年のBC 0010でのプログラミング

何のために？


あなたがレトロなコンピューターの愛好家であれば、動機付けのスピーチをスキップして次のセクションに進むことができます。

2018年8月中、私と私の13歳の息子Ivanqは、グッドアップルデモの執筆に費やしました。Chaos Constructionsフェスティバルでは、私たちの仕事が2位になり、35,000ルーブルの賞金を受け取りました。子供にとっては、これは良い収入ですが、サイトを作成することにより、彼はより少ない時間で同じ額を稼ぎました。明らかに、私はより大きな経済的利益で私の時間を過ごすことができました...しかし、8月ではありません！一度仕事を休まなければならない。あなた自身の喜びのためのプログラミングは、あなたが休暇中にしか余裕がない贅沢です。



もちろん、物質的な報酬は動機としては役立ちませんが、前向きな感情を強化します。驚いたことに、忘れられていたソビエトのコンピューターのデモは、YouTubeで何千ものビューを収集し、最終的には約10万回の再生リストになりました。しかしさらに驚くべきことに、2018年には彼女に賞金が授与されます！おそらく、起こっていることのあり得ないことは、そのような感情的な高揚を引き起こします。

ただし、もっと重要な点があります。

「良いApple」を作成するとき、エミュレーターが正しく再現しないコンピューターの機能を使用したため、実際のハードウェアで作業する必要がありました。まず、IDEフォーマットのハードディスクとコンパクトフラッシュの形の最新の代替品を使用して、同様に安定した作業を実現したいと考えました。

私の13歳の息子が産業開発において非常に貴重な経験を得たのはこのプロジェクトでした。欠陥のある鉄。ドキュメントに記載されているように動作しない良好なハードウェア。エラーを伴って作成されたドキュメント。多くの問題に関するドキュメントの欠如。独自のテストを作成して、鉄の問題を特定します。発信期限。さまざまなマシンと構成でのテスト。独自のクロスアセンブラを作成する（既存のソリューションが開発プロセスを妨げることが明らかになったとき）。そして最後に、成功の短いお祝いの後-最終バージョンの必須リリース、いくつかのバグの修正。

一見すると、いくつかの現代のArduinoでも同様の経験が得られるようです。実際、ジャングルの回路に突入しなければなりませんでした。命令が実行されるメジャーの数を計算しました。メモリの種類によって、今回は異なります。メモリコントローラとプロセッサは異なる周波数で動作するため、同じタイプのメモリでも、同じコマンドを異なるタイミングで実行できます。サブプログラムの実行期間は、このサブプログラムの命令の期間の合計と等しくありません。実際のハードウェアでコードをテストおよび最適化するための独自のツールを作成する必要がありました。



その結果、息子がBK 0011回路（RPLYがトリガーされたときに信号エッジが到達するなど）を研究していたことがわかりました。これについては、もう何も理解できません。それで彼は、ビートに正確な実際のハードウェアと互換性のある独自のエミュレーターBKを作成するというアイデアを思い付き、カーネルさえ作成しました...しかし、これは別の話です。

合計で、1か月-アセンブラの研究から独自の開発ツールの作成まで、ハードテストから完成したマルチメディア作業まで。これはすべて、主要な計画なしには不可能です。おそらく、学んだ最も価値のある教訓。あなたは不可能な仕事をします。あなたは乗り越えられない困難に遭遇することを理解しています。計画しています あなたがやる。その結果は、すべての人を驚かせ、専門家でさえ浮気したとして非難されます（「あなたがプロセッサをオーバークロックした！」）。

なぜZXスペクトラムではないのですか？




繰り返しになりますが、奇妙なことから始めましょう。金融です。 Avitoの広告から判断すると、平均してBC 0010はZX Spectrumの数倍の費用がかかります。完璧な状態にあるネイティブのスペクトラムのまれなインスタンスでは、彼らが合計を要求することは明らかです。ただし、完全に搭載されたBC 0011mは、より高価になります。見つけることができれば。価格はそれ自体が物語っています：BC 0010のコレクション価値はより高いです。そしてBK 0011m-そしてさらにもっと。そのようなコンピュータを自宅に置くのはとても良いことです。

2番目の引数は16ビットです。 BC 0010には8ビットはありません。プロセッサは8ビットの数値を追加する方法すら知りません; ADDBコマンドはありません。すべてで16ビット。この理由は、DEC PDP-11アーキテクチャです。多くの人がPDP-11プロセッサコマンドシステムを、これまでに作成された中で最も成功し、最も便利であると呼びます。もちろん、この発言に異議を唱えたい人もいます。しかし、ここに一つの事実があります：Yandex Demodulation Festival「3つのワークショップはプロセッサアーキテクチャに特化しており、そのうち2つはPDP-11に関するものでした。これは確かに伝説であり、コンピュータ技術の歴史における画期的な出来事であり、愛好家の心を刺激し続けています。 BK 0010は、この伝説に触れることを可能にします。触れるだけでなく、細部まで理解し、DECアセンブラのすべての美しさと優雅さを感じる：完全に等しいレジスタ、8つのアドレス指定方法、線形メモリ-美しさ！

3番目の引数：多くのデモがZXスペクトラム用に書かれています。 BC 0010の場合-50未満で、小さな256バイトと4キロバイトのものをカウントします。デモシーンの場所はほぼ自由で、初心者でも見せる場所があります！

そして最後の考察は、純粋に主観的なものです。私は1994年以来、ロシアのデモシーンを見てきました。パーティーspektrumistovは、残念ながら、より有毒で敵対的です。もちろん、素敵な人達もいます！私は彼らと彼らの仕事をとても尊敬しています。しかし、一般的には-そのマスキャラクターのせいか-スペクトラムシーンは、対立、関係の明確化、さらには名前投票（コンテストで「自分自身」のみに投票する場合）などの陰謀でさえ満たされていました。小さなBK-snoyステージにいる間、彼らは「誰がより涼しい」のような質問をせず、すべての新しい参加者に喜んでいます。繰り返しますが、これは純粋に個人的な印象であり、無視してかまいません。

本物の鉄


私がBCのためのプログラムを書くことに戻る前に20年が経過しました。 20年間で、多くの変化がありました。私個人的には、「異なると思う」という大きな変化が頭の中で起こりました。このスローガンは、デモ「Good Apple」の最終クレジットに入れられます。

以前は、テープレコーダーからゲームを5分間ロードしていました。それからドライブコントローラーが来ました。それらの背後にはハードドライブがあります。次に、ボード上にコンパクトフラッシュを搭載したコントローラーの新しいレプリカが見つかりました。今では、レトロなシーンでフラッシュドライブをエミュレートするのが通例です...しかし、待ってください、2019年には高品質のポータブルサウンドソース-iPhoneがあります。それでは、標準のDIN-5オーディオコード-ミニジャック（はんだ付けする必要すらない）を使用して、iPhoneからBKを高速でロードします。

私はROMトレースを通過し、明らかになりましたどのようにすれば、テープレコーダーのロードアルゴリズムをわずかに上回り、BCがデータを4倍速く読み取ることができますか。

次のステップは、特別に設計されたターボ形式でデータを読み取るマイクロブートローダーを作成することでした。ローダーとデータの両方が1つのWAVファイルに次々に配置されました（コンバーターはLenar Zakirovによって作成されました）。 BK 0010はマイクロローダーを読み取って自動的に起動し、マイクロローダーはファイルから残りのデータを読み取ります。ターボ形式の周波数は22 KHzに達するため、音源の品質に対する要件は高くなります。 iPhone対応中です。良い音楽プレーヤーはますます。コンピュータのサウンドカードも。

その後、クロスアセンブラが来ました。プログラムをWAVで保存するためのオプションが追加されました（標準形式とターボ形式の両方で）。さらに、クロスアセンブラはサウンドカードを介してWAVをすぐに再生できます。開発がいかに加速されたか想像してみてください。コンパイルのたびに、CFカードにディスクイメージを書き込む必要はありません。コードに変更を加えて[ビルド]をクリックすると、サウンドがBCに流れ込み、数秒後、プログラムは実際のハードウェアで既に実行されています。標準形式でサウンドを受信するには、BC 0011でL（ロード）キーとEnterキー（最初に検出されたプログラムをロード）を押します。ターボ形式でサウンドを送信するには、まずマイクロブートローダーを実行する必要があります（私にとっては、BCがオンになっていると、ハードドライブから自動的に起動します。いつでも、STOPキーを押してシステムにログインできます）。



通常のコンポジットAV入力では、BKをモノクロモードでテレビに接続するのが最も簡単です。ケーブルの一端を「チューリップ」で、もう一端をDIN-5ではんだ付けする必要があります。モノクロ信号は、BKコネクタのDIN-5の4番目のピン（「TV」と指定）から送信されます。アースは、伝統的にDIN-5の2番目のピンに関連付けられています。

個人的には、私はモノクロCRTディスプレイのファンです。アパーチャーグリルを備えたカラーモニターでは達成できない画像の鮮明さを提供します。しかし、ほとんどのゲームとすべてのデモは、カラーで見るのが望ましいです。これを行うには、「TVC」BK出力を使用し、RGB SCARTを介してTVに接続します。 DIN-5のコンタクト3、4、5-それぞれ赤、青、緑。ピン1-同期。ピン2はグラウンドです。 LCD TVに接続する場合、SCARTの16番目のピンに（200オームの抵抗などを介して）+5ボルトを印加すると便利です。通常、隣接する「TV」コネクタ（ピン1）から5ボルトが取得されます。

SCART-HDMIコンバーターを使用できます。BCでは、毎秒48フレームではなく、48.83の頻度で画像を表示するため、LCDモニターをスムーズにスクロールする代わりに、周期的な動きが顕著になることをすぐに警告します。この問題は、 12メガヘルツの水晶振動子を12.288 MHzで置き換えることで解決しました。ただし、けいれんはいくつかのデモでのみ顕著でした。ほとんどのBCプログラムはフレームレート同期を使用しません。

実際のコンピュータがまったく必要になる可能性のあるエミュレータに満足してみませんか？エミュレートが不十分な4つのことがわかりました。

  1. 高周波でのスピーカーの動作。
  2. 画像とパレットとビームの同期。
  3. コマンドの正確な実行時間（Covoxによる音楽にとって重要）。
  4. IDEハードドライブを高速で操作します。

上記のいずれも行わない場合は、エミュレータで十分です。

こんにちは世界！


BC 0010の画面メモリの領域は、アドレス40000（左上隅）で始まり、アドレス77777（画面の右下隅）で終わります。ご想像のとおり、PDP-11アーキテクチャは8進数のシステムを使用しています。ただし、PDPy11クロスアセンブラーでは、もちろん、2進数、10進数、16進数のシステムでも数値を書き込むことができます。好きなようにしてください。画面にドット

を付けるには、画面メモリの領域にいくつかの数字を書き込む必要があります。DECアセンブラーの引数は、左から右に記述されます：ソース、レシーバーの例：

MOV #100000,@#60040 ;

＃記号は、引数が（アドレスではなく）単なる数値であることを意味します。@＃記号は、引数が絶対アドレスであることを意味します（つまり、プログラムが別のメモリ位置に移動されても、引数は変更されません）。
画面のクリーニングは次のようになります。

   MOV #40000,R1 ;  
   MOV #20000,R0 ;  
1: CLR (R1)+ ;    ,   R1
   SOB R0,1 ;

間接アドレッシング（R1）は、R1レジスタをアドレスポインタとして使用します。 BCのメモリはバイト単位でアドレス指定されますが、CLR命令は隣接する2つのバイトを最初にクリアします。最初の40000と40001、サイクル40002と40003の次のステップなど。表記（R1）+は、引数を使用した後、それを増やす必要があることを意味します。この場合、コマンドは2バイトを処理するため、2ずつです。ループカウンターは任意のレジスタにすることができます。 SOBはレジスタからユニットを減算してラベル1に移動します。ローカルラベルは番号とコロンで示され、2つのグローバルラベル間のスコープです。グローバルラベルは文字で始まり、コロンで終わる必要があります。

CLR命令は、文字「B」を追加することにより、バイトを処理するようにできます。次に、CLRB（R1）+は、レジスタR1を2ではなく1増やします。

   MOV #40000,R1
   MOV R1,R0 ;       
1: CLRB (R1)+
   SOB R0,1

同じことをもっと短くできます。インデックスアドレス指定方法を使用して、画面を下から上にクリアします。

   MOV #40000,R0
1: CLRB 37777(R0) ;    (37777+R0)
   SOB R0,1

高速が必要な場合は、メモリをバイト単位ではなく、ワード単位ですぐにクリアすることをお勧めします。2倍の速度になります。しかし、それ以上に高速化できます。何らかの理由で、CLRコマンドはMOVよりも低速です。したがって：

   CLR R2 ;     R2
   MOV #40000,R1 ;  
   MOV #20000,R0 ;  
1: MOV R2,(R1)+ ;    ,   R1
   SOB R0,1 ;

画面をクリアする4つの方法の1つを選択すると、すでにポイントを設定できます。カラーモードでは、各ポイントは2ビットに対応します。モノクロ、1ビット。BCには、画面モードのプログラム切り替えはありません。モニターがどの出力に接続されているか、そのような画像を受け取りました。
わかりやすくするために、ドットの色を2進数で表記します。

   MOVB #0b00001100,@#50010
   MOVB #0b00110000,@#50112
   MOVB #0b11000000,@#50313

ビットの各ペアの色は次のようにエンコードされます。偶数ビットのみが設定されている場合-緑のドット、奇数ビットのみ-青、両方のビットが設定されている-赤、両方のビットがリセットされている-黒。まあ、モノクロのモニターは各ビットを別々のポイントとして表示します。
MOVBコマンドは一度に4ポイントを画面上のメモリに書き込み、MOVコマンドは8ポイントを書き込みます。隣接するポイントに影響を与えたくない場合は、BIC（ビットクリア）およびBIS（ビットセット）コマンドを使用します。次に例を示します。

   BICB #0b00001100,@#50112 ;  
   BISB #0b00000100,@#50112 ;

興味深い点：本文では、最下位ビットを最上位ビットの右側に記述しています。逆に、画面では、最下位ビットに対応するポイントが左側に表示されます。
それは、実際には、オンスクリーンメモリBK 0010のデバイスに関するすべてです。
