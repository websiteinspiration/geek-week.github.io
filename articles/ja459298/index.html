Angular：2019年のステータス

今、夏は世界中でさまざまな会議やその他のイベントが開催される時期です。現時点では、プログラマーは文字通り新しい情報で圧倒されますが、これは通常、同化するのが非常に困難です。

フロントエンドの開発ユニバースは、ノンストップで動く列車に似ています。私たちは皆、この列車に遅れずについていくのがいかに難しいか知っています。会議のビデオを見るには時間がかかるので、結果として、私たちの多くは、事務処理後、Netflixで映画を見る方を好むのは明らかです。 私たちが翻訳を公開している資料の作者は、Angularの革新を理解したいすべての人を支援することにしたのはそのためです。ここでは、Angularの現在の状態、このフレームワークの新しい機能、現在の傾向について説明します。





Angular Scaling Upについて


Angular開発チームメンバーのBrad Greeneがng-conf 2019で提供したデータによると、Angularの使用規模は年間で約50％増加しています！

このデータは、多くのプロジェクトがAngular 1.Xからフレームワークのより新しいバージョンへの移行の過程にあるという事実によって影響を受ける可能性があることは明らかです。しかし、50％は依然として印象的な指標です。

AngularはSvelteやVueほど速く成長しません。このフレームワークは、Reactほど広く使用されていません。ただし、RedditやTwitterなどのリソースで他のフレームワークについて読むことができるものとは異なり、Angularは正常であり、何百万もの開発者やチームによって使用されています。

Angularには大規模で動的なインフラストラクチャがあります


Angularには、コア開発チームによって作成された多くの便利な組み込みツールがありますが、そのエコシステムは、開発に貢献する愛好家のコミュニティであり、非常に大規模です。さらに、彼女は非常に質の高いプロジェクトに代表されています。

基本的なフレームワークと一緒に、多くの興味深いツールを使用できます。つまり、次のことを話しているのです。

  • 優れた動的開発ツールとの緊密な統合。その中で、TypeScriptとRxJSに注目できます。これらのテクノロジーの開発は、Angularに直接プラスの影響を与えます。
  • アプリケーションの状態を管理するためのさまざまな種類のライブラリ。
  • コード品質を分析および改善するためのツール。などCodelyzer。
  • ユーザーインターフェイス要素、コンポーネント、ディレクティブ、パイプラインなどのライブラリ。
  • さまざまなIDE用のプラグインの豊富な選択肢。
  • アプリケーションをテストするためのライブラリとフレームワーク。
  • コミュニティメンバーとAngularチームの両方が取り組んでいる高品質の詳細なドキュメント。これには、実践的なガイドライン、ガイド、トレーニング資料、書籍、ビデオコースが含まれます。これはこのリストの最後の項目ですが、それでも他の項目と同じくらい重要です。

次のセクションでは、Angularプロジェクトの開発時に使用できる特に興味深いツールに焦点を当てます。

Angular CLI


▍AngularCLIと非常にシンプルなAngular更新プロセス


これらは空の言葉ではありません。ここでは、毎年5億人のユーザーにサービスを提供しているBlueWebプロジェクトが、1営業日でAngular 7からAngular 8にアップグレードされた方法についての良い話を見つけることができます。Angularの2番目から4番目のバージョンへの移行が同じプロジェクトから30日かかったことを考えると、これは印象的な成果です。

これは、Angular CLIの大きな価値を証明しています。このツールを使用すると、私が個人的にこれまで会ったことがないレベルに生産性を上げることができます。

今日、Angular CLIなしで作業することを想像することは非常に難しくなっています。

▍回路図ツール


Angular Schematicsツールは、Angular開発者が日常的に使用するものです。たとえば、次のコマンドを頻繁に使用すると思います。

ng generate component my-component

これは、Angular CLIに組み込まれたSchematicsツールに基づいています。

おそらく、開発者が独自のSchematicsルールを作成する機会があることは、あなたにとってはニュースになるでしょう。それらを使用すると、頻繁に使用されるコードフラグメントでの作業が容易になります。

▍APIビルダー


APIビルダーを使用すると、既存のチームを拡張または拡張できます。たとえば、JasmineとProtractorを使用する代わりに、JestとCypressを使用してテストを実行するチームを作成できます。たとえば、別のリンターを使用できます。

フレームワークの新しい興味深い機能


この資料の執筆時点で、Angularのバージョン8はすでにリリースされており、さまざまな改善と新機能が含まれています。

このフレームワークの使用方法を大幅に変更するものはここにはありません。ツールの新しいバージョンに関する同様のステートメントは、通常、良いニュースとして認識されます。しかし、Angular 8には、いわば肉眼では見えない多くの改善点があります。それらのおかげで、Angularプロジェクトはより小さく、より速くなります。これらの改善により、Angularアプリケーション開発プロセスは以前よりもさらに楽しくなります。さらに、フレームワークの新しいバージョンには、ほぼ完全に使用できる状態の新しいレンダリングシステムがあります。

▍差動荷重


この技術の名前から判断すると、「差分読み込み」は非常に複雑に思えるかもしれませんが、実際はそうではありません。一言で言えば、この機能のおかげで、コンパイラーは2つのバンドルを生成します。1つは、最新のブラウザー用です。その中にポリフィルコードはありません。2つ目は、古いバージョンのブラウザ用です。

最近のブラウザは最新のバンドルをダウンロードします。また、専用に設計されたポリフィルダウンロードアセンブリを必要とする古いブラウザ。

これは非常に便利なイノベーションです。ほとんどのユーザーは最新バージョンのブラウザーで作業する可能性が非常に高いため、これは、最新のバンドルよりも大きいポリフィル付きのバンドルを使用するユーザーが少数であるという事実につながります。

▍ウェブワーカー


あなたはおそらくWebワーカーについてすでに聞いたことがあるので、このテクノロジーの本質の説明は扱いません。しかし、新しいAngularリリースでは、Webワーカーをアプリケーションに簡単に統合するのがどれほど簡単かわからないかもしれません。ここでこれについてもっと読むことができます。

▍CDKツールボックス


CDKは、Angularコンポーネントを開発するチームによってリリースされたツールボックスです。その中に、スタイルに依存しない、Angular Materialによって使用される抽象化を見つけることができます。

たとえば、CDKのおかげで、開発者には次の機能を実装できるディレクティブがあります。

  • 「ドラッグアンドドロップ」のスタイルでオブジェクトを操作します。
  • 自動的にサイズ変更されるテキスト領域を使用します。
  • ポップアップウィンドウ。
  • 仮想スクロール。

CDKは非常に便利なツールです。これは、ほとんどのアプリケーションが、限られた機能のみを実装する高品質で広範な抽象化を使用できるためです。これは、広範囲に渡るコンポーネントよりもはるかに優れており、常に正当化できるわけではありません。Angular Materialは素晴らしいコンポーネントのセットですが、CDKの力は、開発者が基本的なビルディングブロックからカスタムコンポーネントを作成できることです。

▍Ivyレンダリングエンジン


Ivyは非常に興味深い新しい開発であり、レンダリングエンジンです。Ivyの作業はまだ完了していません。デフォルトでは、このエンジンは無効になっています。ただし、Angular 7以降では、すでに有効にして使用できます。Ivyは、Angular 9のメインレンダリングエンジンであることが期待されています。

フル機能のIvyエンジンから期待されるものは次のとおりです。

  • バンドルのサイズを小さくする。
  • テンプレートのデバッグ。
  • プロジェクトの迅速な組み立て、テストの加速。
  • 既存のエラーの修正。

Angularおよびモバイルプラットフォーム


▍イオン


Ionicは、モバイルアプリケーション開発フレームワークです。基本コンポーネントはステンシルを使用して作成されますが、IonicをAngularやその他のフレームワークで使用できるようにする開発者向けの抽象化を提供します。

▍NativeScript


NativeScriptは、React Nativeに似たネイティブモバイルアプリ開発フレームワークです。Angularとの緊密な統合が特徴です-Vueのサポートと純粋なJavaScriptでの開発にそれを使用する機能。残念ながら、彼との私の経験は特に成功していませんでした。React Nativeに匹敵することはできません。ただし、モバイルアプリケーションを真剣に開発している場合は、NativeScriptを試してください。

テスト中


▍ジャスミンと分度器


ジャスミンと分度器は紹介する必要はありません。これらは、Angularアプリケーションをテストするための標準ツールです。それらは時間ごとに十分にテストされており、定期的に更新されており、Angularとの相性も良好です。

▍ジェストとサイプレス


Angular CLIの一部であるAPIビルダーに基づく新しいツールのおかげで、JestとサイプレスでAngularアプリケーションをテストすることが可能になりました。

Jestは、Jasmineベースの単体テストフレームワークです。Facebook上で開発されています。これはReactプロジェクトをテストするための標準的なフレームワークであり、その速度により広く配布されています。多くの人々はジャスミンよりもそれを使用することを好みます。あなたもJestに傾倒しているなら-Angularアプリケーションのテストにそれを使うことができます。

ヒノキ-これは、エンドツーエンドのアプリケーションテストを実施するための非常に尊敬されるフレームワークです。開発コミュニティがそれを非常によく扱っているのは偶然ではありません。SeleniumやWebDriverからは独立しています。サイプレスは、チームログの操作を可能にし、ネットワークトラフィックの管理方法を知っています。それは非常に安定しているので、その助けを借りて得られた結果は非常に理解可能で予測可能であることを期待できます。

▍Angular Testing Library Toolbox


Angular Testing Libraryは、ユーザーアクションの再現に重点を置いてユーザーインターフェイスコンポーネントをテストするためのツールセットです。

これは、このライブラリーがプログラマーにコンポーネントをプログラムで操作しないようにすることを要求することを意味します。コンポーネントの動作をテストすることを目的としています。ライブラリは、ユーザーが通常実行するアクションを実行します。

たとえば、このライブラリのツールを使用する場合、そのような構造は使用されません。

myComponent.onClick();

代わりに、ユーザーの自然な動作がシミュレートされます。

const { getByText, click } = await render(CounterComponent, {
    componentProperties: { counter: 5 }
});
click(getByText('+'));

状態管理


▍NgRx


NgRxは、アプリケーションの状態を管理するためのライブラリであり、Reduxに組み込まれたアイデアの影響下で作成されます。NgRxは、おそらくAngular開発で最も広く使用されている状態管理ライブラリです。ライブラリの名前は、RxJsストリームを真剣に使用していることを示しています。

▍NGXS


NGXSは、Reduxと同様に、Angularアプリケーションで状態を管理するための代替ライブラリです。NGXSとRxJSを比較すると、NGXSの特徴は、多くのクラスとデコレーターを使用していることです。これは、ボイラープレートコードの量を減らすために行われます。おそらく、そのようなライブラリを探している誰かがクラスの使用に慣れている場合、NGXSのこの機能は、状態を管理するためのライブラリを選択するときに決定的なものになる可能性があります。

▍秋田


秋田はDatoramaによって開発された状態管理ライブラリです。このライブラリもRxJSに基づいています。前の2つのライブラリと比べると、Angularが使われていないプロジェクトでも使えるのが特徴です。つまり、長期的に秋田ライブラリを選択すると、プロジェクトコードを再利用する機会が増える可能性があります。

RたぶんRxJをとるだけ？


このセクションのタイトルで提起された質問に肯定的な答えを出すことができます。それはすべて、Reduxを連想させるライブラリが好きかどうか、プロジェクトが状態を管理するためにライブラリを必要とするかどうかによって異なります。必要に応じて、標準のRxJSツールを使用して状態を保存するサービスを作成できます。

ユーザーインターフェイスコンポーネントライブラリ


▍ストーリーブック


StoryBookは、実際にはUIコンポーネントのライブラリではありません。これは、孤立したコンポーネントを作成し、コンポーネントとそのバリアントを調査できるようにするツールです。

これは素晴らしいツールです。以前は、React開発者のみが使用できました。私はそれらをうらやましく思った。しかし、今ではAngular開発者がそれを使用できるので、言及せざるを得ません。

▍角材


ここでは、悪名高いAngular Materialライブラリについて簡単に説明します。これにより、現代の開発者は、Angularの最新バージョン用に設計されたコンポーネントの素晴らしいセットを手に入れることができます。

Angularアプリケーションのバックエンド


▍Firebase


Firebaseは、Googleが所有するアプリケーション開発プラットフォームです。この結果、FirebaseとAngular- AngularFireの公式ライブラリができます。このライブラリは、RxJSオブザーバブルを使用してデータをストリーミングします。Angularと深く効率的に統合されています。

▍GraphQL


GraphQLを使用する楽しみは、Reactプログラマーだけが利用できると思うかもしれません。もしそうなら、あなたは間違っています。Angular用のApolloライブラリのバージョンもあります。このライブラリで作成できるものは、Firebaseの優れた代替手段です。ApolloはGraphQLでの作業を好む人向けです。

▍NestJS


NestJSは、サーバーアプリケーションの開発用に設計されたNode.js用のWebフレームワークです。例からそれに至るまで、Angularの基本概念がそれに大きな影響を与えたと結論付けることができます。これらは、モジュール、コントローラー、コンベヤーなどです。Angularが好きなら、おそらくNestJSも好きになるでしょう。Nxワークスペース

を使用する場合、AngularとNestJSに基づくフルスタックアプリケーションのフレームワークを作成するには、コンソールで1つのコマンドを実行するだけです。
