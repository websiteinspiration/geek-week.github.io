独自のJSONコンバーターまたはExpressionTreesについてもう少し



シリアライゼーションとデシリアライゼーションは、現代の開発者が取るに足らないものとして扱う典型的な操作です。私たちはデータベースと通信し、HTTPリクエストを生成し、REST APIを介してデータを受信します。また、REST APIがどのように機能するかさえ考えません。今日は、JSONのシリアライザとデシリアライザを記述して、内部の内容を確認することをお勧めします。

責任の否定


前回 と同様に、私は気づくでしょう：自転車と言ってもよいような、プリミティブシリアライザーを作成します。ターンキーソリューションが必要な場合は、Json.NETを使用してください。これらの人たちは、高度にカスタマイズ可能で、多くのことができ、JSONでの作業時に発生する問題をすでに解決している素晴らしい製品をリリースしました。独自のソリューションを使用することは本当に素晴らしいですが、最大のパフォーマンス、特別なカスタマイズが必要な場合、または私が好きな方法で自転車が好きな場合に限ります。

主題領域


JSONからオブジェクト表現への変換サービスは、少なくとも2つのサブシステムで構成されています。デシリアライザは、有効なJSON（テキスト）をプログラム内のオブジェクト表現に変換するサブシステムです。逆シリアル化にはトークン化が含まれます。つまり、JSONを論理要素に解析します。シリアライザは、逆のタスクを実行するサブシステムです。データのオブジェクト表現をJSONに変換します。

消費者は、ほとんどの場合、次のインターフェースを見ます。私は特にそれを簡略化して、最も頻繁に使用される主な方法を強調しています。

public interface IJsonConverter
{
    T Deserialize<T>(string json);
    string Serialize(object source);
}

「内部では」、逆シリアル化にはトークン化（JSONテキストの解析）と、後でオブジェクト表現を簡単に作成できるようにするいくつかのプリミティブの構築が含まれます。トレーニングのために、中間プリミティブ（たとえば、Json.NETのJObject、JProperty）の構築をスキップし、データをすぐにオブジェクトに書き込みます。カスタマイズのオプションが減るため、これはマイナスですが、1つの記事のフレームワーク内でライブラリ全体を作成することは不可能です。

トークン化


トークン化または字句解析のプロセスは、テキストに含まれるデータの異なる、より厳密な表現を取得することを目的としたテキストの解析であることを思い出してください。通常、この表現はトークンまたはトークンと呼ばれます。 JSONを解析するために、プロパティ、それらの値、構造の最初と最後のシンボル、つまりコードでJsonTokenとして表すことができるトークンを強調表示する必要があります。

JsonTokenは、値（テキスト）とトークンのタイプを含む構造です。 JSONは厳密な表記なので、すべてのタイプのトークンを次の列挙型に減らすことができます。もちろん、受信データ（行と列）の座標をトークンに追加するのは素晴らしいことですが、デバッグは実装の範囲を超えています。つまり、JsonTokenにはこのデータが含まれていません。

したがって、テキストをトークンに解析する最も簡単な方法は、各文字を順番に読み取ってパターンと比較することです。これまたはその記号の意味を理解する必要があります。キーワード（true、false、null）がこの文字で始まるか、これが行の先頭（引用符）であるか、またはこの文字がトークン（[、]、{、}）である可能性があります。一般的な考え方は次のようになります。

var tokens = new List<JsonToken>();
for (int i = 0; i < json.Length; i++) {
    char ch = json[i];
    switch (ch) {
        case '[': 
            tokens.Add(new JsonToken(JsonTokenType.ArrayStart));
            break;
        case ']': 
            tokens.Add(new JsonToken(JsonTokenType.ArrayEnd));
            break;
        case '"':
            string stringValue = ReadString();
            tokens.Add(new JsonToken(JsonTokenType.String, stringValue);
            break;
        ...
    }
}

配列の逆シリアル化


JSONから配列を変換するための汎用クラスコードも比較的単純です。Type.GetElementType（）を抽出できる要素のタイプによってパラメーター化されます。型が配列であることを判別するのも簡単です：Type.IsArray。配列の逆シリアル化は、ArrayEndタイプのトークンに到達するまでtokenizer.MoveNext（）と言うことになります。配列要素のデシリアライゼーションは、配列要素タイプのデシリアライゼーションであるため、ArrayConverterを作成すると、要素デシリアライザがそれに渡されます。

一般的な実装のインスタンス化に問題がある場合があるので、その方法をすぐに説明します。リフレクションを使用すると、ジェネリック型をリアルタイムで作成できます。つまり、作成した型をActivator.CreateInstanceの引数として使用できます。これを活用してください：

Type elementType = arrayType.GetElementType();
Type converterType = typeof(ArrayConverter<>).MakeGenericType(elementType);

var converterInstance = Activator.CreateInstance(converterType, object[] args);

オブジェクトのデシリアライザーを作成する準備が完了すると、デシリアライザーの作成と格納に関連するすべてのインフラストラクチャコードをJConverterのファサードに配置できます。彼はすべてのJSONシリアライゼーションおよびデシリアライゼーション操作を担当し、サービスとしてコンシューマが利用できます。
