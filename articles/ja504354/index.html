登録とSMSなしでMLモデルのサイズを縮小する



機械学習に遭遇した人は誰でも、これには深刻な計算能力が必要であることを理解しています。この記事では、MITで開発されたアルゴリズムを適用してニューラルネットワークを圧縮しようとします。これにより、トレーニング済みモデルの重みの次元が削減され、学習とモデル起動の両方が速くなります。

ニューラルネットワークは、さまざまなタスクを解決するための優れたツールであることが証明されていますが、残念ながら、それらを使用するにはかなりの計算能力が必要ですが、それでも中小企業にはありません。ハードウェア、低レベル、数学に分類できるニューラルネットワークの圧縮には多くの種類がありますが、この記事では、2019年にMITで開発され、ニューラルネットワーク自体を直接操作する方法について説明します。

この方法は「当選チケット仮説」と呼ばれています。一般的に言えば、次のように聞こえます。ランダムに初期化された重みを持つ完全に接続されたニューラルネットワークには、同じ重みのサブネットが含まれており、個別にトレーニングされたそのようなサブネットは、元のネットワークと同じ精度である場合があります。

正式な証明と完全な記事はここにあります。実用化の可能性に興味があります。つまり、アルゴリズムは次のとおりです。

  1. モデルを作成し、そのパラメーターをランダムに初期化します
  2. イテレーションjのネットワークの学習
  3. 最小値を持つネットワークパラメーターを切り取ります（最も簡単なタスクは、いくつかのしきい値を設定することです）。
  4. 残りのパラメーターを初期値にリセットし、必要なサブネットを取得します。


理論的には、このアルゴリズムはn番目のステップを繰り返す必要がありますが、例では1回の反復のみを実行します。tensorflowとKerasを使用して、完全に接続されたシンプルなネットワークを作成します。

import tensorflow as tf
from tensorflow import keras

model = keras.Sequential([
    keras.layers.Flatten(input_shape=(28, 28)),
    keras.layers.Dense(300, activation='relu'),
    keras.layers.Dense(150, activation='relu'),
    keras.layers.Dense(10, activation='softmax')
])

model.compile(optimizer='SGD',
              loss='sparse_categorical_crossentropy',
              metrics=['accuracy'])

次のネットワークアーキテクチャを取得します



。60,000画像のMNIST形式のデータセットでトレーニングします。検証データの精度は0.8594になります。このアルゴリズムのネットワーク1反復のパラメーターに適用します。コードでは、次のようになります。

#  
threshold = 0.001

#       np.array
weights = model.weights
weights = np.asarray(weights)

#    
first_h_layer_weights = weights[1]
second_h_layer_weights = weights[3]

def delete_from_layers(one_d_array, threshold):
    index = []
    for i in range(one_d_array.shape[0]):
        #   ,       
        if abs(one_d_array[i]) <= threshold:
            index.append(i)
    #    ,   
    new_layer = np.delete(one_d_array, index)
    return new_layer

new_layer_weights = delete_from_layers(second_h_layer_weights, threshold)

したがって、このコードを実行すると、実際には使用されていない重みが削除されます。注意が必要な点が2つあります。この例では、しきい値は経験的に選択されており、このアルゴリズムは入力層と出力層の重みに適用できません。

新しいウェイトを受け取ったら、元のモデルを再定義して、余分な部分を削除する必要があります。その結果、以下が得られます。



パラメータの総数がほぼ2倍に減少したことがわかります。これは、最初のネットワークをトレーニングするときに、パラメータの半分以上が単に不要だったことを意味します。同時に、サブネットの精度は0.8554で、メインネットワークよりもかなり低いです。もちろん、この例は指標であり、通常、ネットワークはパラメーターの初期数の10〜20％削減できます。ここでは、このアルゴリズムを適用しなくても、元のアーキテクチャを選択するのが面倒すぎたことは明らかです。

結論として、この手法は現時点では十分に開発されていないと言えます。実際の問題では、この方法でモデルの重みを最適化しようとすると、学習プロセスが長くなるだけですが、アルゴリズム自体にはかなり大きな可能性があります。
