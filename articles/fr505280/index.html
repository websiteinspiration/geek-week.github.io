Magento 2: CSS critique



Dans cet article, nous verrons comment configurer et activer les CSS critiques dans Magento 2.

Que faites-vous, css critique?


HTML et CSS ralentissent la sortie des pages. Un document HTML doit être traité pour créer un modèle DOM, sans lequel il n'y aura pas de contenu sur la page. Le document CSS doit d'abord être chargé puis traité (modèle d'objet CSS).

Pour accélérer le processus, vous devez simplifier CSS et utiliser des requêtes multimédias. Les CSS critiques contribuent également à l'amélioration des performances.

Comme nous l'avons dit ci-dessus, tous les styles CSS chargés à partir de fichiers externes sont traités comme un blocage de rendu. Cela signifie que la page Web ne sera pas affichée tant que ces fichiers n'auront pas été téléchargés. En utilisant css critique, nous fournissons le «CSS critique» intégré et rendons le chargement de tous les autres styles asynchrone.

Le chargement asynchrone est obtenu en ajoutant une nouvelle directive de préchargeà l'attribut rel

Pour prendre en charge le cross-browser rel = "preload", Magento utilise un polyfill.

Une caractéristique importante, le «css critique» comprend uniquement la partie visible de la page.



Ainsi, nous pouvons considérablement améliorer le temps du "premier" affichage de nos pages.

Comment l'allumer?


Magasins - Paramètres - Configuration - Avancé - Développeur

L'onglet Avancé est disponible uniquement en mode développeur

Définir Utiliser le chemin critique CSS à oui

Ou utiliser la ligne de commande: bin / magento config: set dev / css / use_css_critical_path 1

Le CSS critique doit être situé dans app / design / frontend / vendor_name / theme_name / theme_name /web/css/critical.css



Comment générer?


Le processus de création entièrement manuel prend beaucoup de temps et de temps.

Des générateurs comme Penthouse ou Critical peuvent être utilisés pour créer des CSS critiques pour votre thème.

Ou optez pour une assez bonne ressource en ligne.



Il est nécessaire d'insérer l'adresse URL de la ressource dans le champ (1), et d'insérer css, qui est utilisé sur le site, et de cliquer sur le bouton Créer dans le champ (2).



Par conséquent, nous obtenons le css critique généré et minifié



Une certaine partie du travail manuel, bien sûr, nous devrons:

  • Vérification et ajustement du CSS critique généré
  • N'oubliez pas, également, que critical.css dans Magento est une pour toutes les pages, il est donc important de ne pas oublier la liste, la page du produit, la page d'inscription et la connexion
