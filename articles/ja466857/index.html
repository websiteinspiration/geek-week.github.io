iOS 13でのバックグラウンドアプリケーションの実行

iOSおよびMacOS開発者の世界での今年のメインイベントであるWWDCは、業界に関わるすべての人々に、オペレーティングシステム、フレームワーク、ハードウェアの新しいバージョンのリリースを常に喜ばせています。一般的に、取り組むべきことはすべてです。

今年も例外ではなく、活発な議論のテーマは、iOSの新しいバージョンであるSwiftUI、そしてあなた自身が理解しているデバイス向けの独自のiPadOS OSでした。

ただし、あまり取り上げられていないトピックもありますが、これからはそれほど興味深いものではありません。たとえば、アプリケーションのバックグラウンド実行に触れたような便利なものがあります。

トピックについて簡単に


バックグラウンド実行とは何かをよく理解していない人のために説明しましょう。バックグラウンドで実行すると、アプリケーションはユーザーのスマートフォン画面で物理的に実行されていない状態（つまり、フォアグラウンドではない状態）で何らかの処理を実行しています。 。

また、このセッションのApple開発者はリソースの消費に焦点を合わせました（バックグラウンドで作業しているときに、アプリケーションがエネルギーを消費し続けることは明らかです）。しかし、レポートに直接移動する時が来ました。

WWDCで何が起こったか


実際、スピーカーは私と同じことから始まりました。バックグラウンドでの作業について簡単に説明し、そこで行われるタスクについて説明しました。



開発に近づいています。バックグラウンドパフォーマンスに関しては、Appleの主な考慮事項が3つあります。

  • パワー
  • パフォーマンス
  • 守秘義務

すべてが非常に単純です。アプリケーションタスクがバックグラウンドで実行されている場合、この実行は別のアプリケーションがフォアグラウンドで実行されている間に発生する可能性があります。したがって、リソースが過度に消費されないようにするために、リソース制限が課されます。これらの制限を超えると、アプリケーションがAppleによって強制終了される可能性があり、それにより、さらに起動時間が遅くなります。プライバシーに関しては、開発者がAPIのすべての機能を使用して、アプリケーションが作業中に（バックグラウンドであっても）どのデータを使用するかをユーザーに通知することを強くお勧めします。

ちなみに、これらのAPIのリストはかなり印象的でした。個別のレビューは行っていませんが、注意を払うことをお勧めします。

はい、そしてセッション自体で、Apple開発者はメッセンジャーアプリケーションの例を使用して可能性の概要を示しました（WWDCのビデオを見る価値があります）。

BackgroundTasks


今年のレポートのほぼ半分は、このフレームワークに特化しています。

この目新しさが提供する2種類のタスク：

  • アプリ更新タスク
  • バックグラウンド処理タスク

アプリ更新タスク


BGアプリの更新タスクは、アプリケーションデータの更新に使用できる特別なタイプのバックグラウンドタスクです。このタイプのタスクを非常に特別なものにする1つのことは、ユーザーの行動です。iOSは、ユーザーがアプリケーションを起動する頻度と時間を学習し、ユーザーがアプリケーションを使用する可能性が低いときにBGAppRefreshTaskを起動しようとします。



はい、30秒より長く機能しません-この機能は非常に不快で、以前のバージョンのOSから残っています。

インターネットからのコードの例を考えてみましょう。

import UIKit
import BackgroundTasks

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate {

    func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        BGTaskScheduler.shared.register(
            forTaskWithIdentifier: "pl.snowdog.example.refresh",
            using: DispatchQueue.global()
        ) { task in
            self.handleAppRefresh(task)
        }
        return true
    }

    private func handleAppRefresh(_ task: BGTask) {
        let queue = OperationQueue()
        queue.maxConcurrentOperationCount = 1
        let appRefreshOperation = AppRefreshOperation()
        queue.addOperation(appRefreshOperation)

        task.expirationHandler = {
            queue.cancelAllOperations()
        }

        let lastOperation = queue.operations.last
        lastOperation?.completionBlock = {
            task.setTaskCompleted(success: !(lastOperation?.isCancelled ?? false))
        }

        scheduleAppRefresh()
    }
}

停止してよく検討する必要があるいくつかのポイントがあります。

  • タスクオブジェクト（expirationHandler）に有効期限ハンドラをインストールします。これは、システムが作業を完了するための限られた時間を提供するため、それを超えると、リソースをクリアする必要があるためです。
  • 操作が完了したらすぐに、setTaskCompletedメソッドが呼び出されていることを確認してください。
  • BGTaskSchedulerは、バックグラウンド作業をスケジュールするために使用されるメインクラスです。

バックグラウンド処理タスク


バックグラウンドハックのもう1つのタイプは、BG処理タスクです。これを使用して、デバイスでMLモデルをトレーニングしたり、データベースをクリーンアップしたりできます。Appleは、システムがこのタイプのタスクに最大数分の時間を割り当てることができると約束しています。これは、悲惨な30秒に収まらない重い作業のための非常に重要な革新です。

再び例：

import UIKit
import BackgroundTasks

@UIApplicationMain
class AppDelegate: UIResponder, UIApplicationDelegate {

    func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        
        BGTaskScheduler.shared.register(
            forTaskWithIdentifier: "pl.snowdog.example.train",
            using: DispatchQueue.global()
        ) { task in
            self.handleMLTrain(task)
        }
        return true
    }

    private func scheduleMLTrain() {
        do {
            let request = BGProcessingTaskRequest(identifier: "pl.snowdog.example.train")
            request.requiresExternalPower = true
            request.requiresNetworkConnectivity = true
            try BGTaskScheduler.shared.submit(request)
        } catch {
            print(error)
        }
    }
}

重要なポイント：

  • requiesExternalPowerは、電源に接続されたデバイスが処理タスクに必要かどうかを示すブール値です。
  • requireNetworkConnectivity-処理タスクにネットワーク接続が必要かどうかを示すブール値。

デバッグ中


ここにもアメニティがないわけではありません。バックグラウンドタスクのスケジュールされた実行時間と、システムがアプリケーションを起動して完了するまでの遅延は、数時間になる場合があります。アプリケーションを開発する場合、2つのプライベート関数を使用してタスクを起動し、選択したタイムラインに従ってタスクを早期に完了できます。デバッグ機能はデバイスでのみ機能します。

  • タスクの起動：

    e -l objc -- (void)[[BGTaskScheduler sharedScheduler] _simulateLaunchForTaskWithIdentifier:@"TASK_IDENTIFIER"]
  • 割り当ての強制終了：

    e -l objc -- (void)[[BGTaskScheduler sharedScheduler] _simulateExpirationForTaskWithIdentifier:@"TASK_IDENTIFIER"]
