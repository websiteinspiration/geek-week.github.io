フォト用紙なしのフォトグラム



フォトグラムは非常に興味深い芸術で、かつてはテレビ番組でも宣伝されていました。フォトペーパーにオブジェクトを配置し、最も興味深いシルエットが得られる方向から露光し、マニフェストし、修正します。これで作業の準備が整います。しかし、今ではすべての段階で写真用紙を購入することはできません。しかし、写真の後にデジタル画像がデジタル化されたらどうなるでしょうか。

これには、長い間知られているランニングビームスキャナーが適しています。現在では、CRTの代わりにLEDマトリックスを使用する方が便利です。それはさらに良いです：管にはわずかな残光がありますが、LEDにはありません。センサーはTIL78フォトトランジスターで、外部からはLEDと区別できません（ベース出力は取り出されませんが、必要ありません）。このフォトトランジスタは廃止されました。多くの類似のフォトトランジスタに置き換えることができます。

フォトトランジスターをエミッターで共通のワイヤーに接続し、コレクターをA5 Arduino UNOの入力に接続してから、小さなデバッグスケッチを入力して実行する必要があります。

#define SENSOR A5
void setup() {
  Serial.begin(9600);
  pinMode(SENSOR, INPUT_PULLUP);
}
void loop() {
  // Read analog value continuously and print it
  Serial.println(analogRead(SENSOR));
}

特にプログラムは、Arduinoに組み込まれているプルアップ抵抗をアクティブにするため、外部抵抗は必要ありません。 Arduino IDEメニューで、[ツール]-[シリアルプロッター]を選択し、ライトスケジュールを確認します。フォトトランジスタのベースは表示されていないため、接続の極性は実験的に決定されています。

次のように、32x32 LEDマトリックスを組み込み制御デバイスに接続します。



マトリックスはArduinoから電力を供給できません。5V、4 Aの外部電源が必要です。このように接続します。



まずマトリックスに電力を供給し、次にArduinoに電力を供給し、逆の順序で電源を取り外します。ここで説明するように、必要なライブラリをArduino IDEに「ポンプで送り」、マトリックスを確認します。機能したら、スケッチを入力して写真を取得します。

その中で、パラメータを変化させることができます：

static constexpr uint8_t  READINGS_PER_PIXELS = 1;
static constexpr uint8_t  THRESHOLD           = 15;
static constexpr bool     INVERT              = false;
static constexpr bool     CLEAR               = true;

最初の2つのパラメーターで感度を調整できます。3番目のパラメーター-ポジティブとネガティブを切り替えます。4番目のパラメーターは、前のイメージの後続の各イメージのオーバーレイを有効にします。すばらしいアニメーションGIFを見る。

フォトトランジスタからの光がマトリックスからのものかどうかを判断する最良の方法は、LEDをオフにしてからオンにして結果を比較することでした。現在読み取られているオブジェクトのポイントの上のLEDがブロックされている場合、それらは同じになります。ブロックされていない場合、それらは異なります。 LEDは、OEライン（出力イネーブル）を切り替えることにより数回オン/オフし、読み取り結果は2つのバッテリーに合計されます。これらの量を差し引いた結果は、しきい値と比較されます。

マトリックスには、上半分と下半分の赤、緑、青のLEDの行全体に対応する6つのシフトレジスタが含まれています。 LATライン（ラッチ）は、シフトレジスタからLEDドライバーに値を転送するバッファーを制御します。 4つのアドレスラインA、B、C、Dを使用すると、上半分と下半分のアクティブラインを選択できます。OEラインを使用すると、ドライバーを無効または有効にできます。

通常、以前のデータはバッファに格納されてLEDドライバに送信されますが、このとき新しいデータはシフトレジスタに書き込まれます。行が完全に書き込まれた後、それらはバッファに転送され、プロセスは次の行に対して繰り返されます。 1ピクセルのみの状態を変更する必要がある場合でも、行全体を書き換える必要があります。

ここでは、マトリックスは他の目的に使用されます-スキャンのため、著者はこのモードでシフトレジスタを使用しましたが、バッファーは使用しませんでした。1つをLATラインに供給し、CLKラインの各パルスでシフトレジスタの内容が直ちにバッファに転送されます。また、ダビングのたびに、OEを介してLEDを数回点滅させ、フォトトランジスターで結果を読み取り、上記のように2つのバッテリーを追加します。最後に、LEDがブロックされているかどうかを確認した後、ライブラリで提供されているフレームバッファーに結果を記録します。

フォトグラム全体がスキャンされたら、割り込みを再度オンにするだけで十分です。フレームバッファーからの画像は、標準的な方法でマトリックスに書き込まれ、表示されます。その後、次のスキャンが始まるまでそこに残ります。
