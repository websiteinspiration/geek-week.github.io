ページの読み込みを高速化するための7つのCSS最適化のヒント

最新のWebでは、サイトページの読み込み時間は最も重要な指標の1つです。ミリ秒でも収益に大きな影響を与える可能性があり、ページの読み込みが遅いと、コンバージョン率が低下する可能性があります。あなたのウェブサイトをスピードアップするために使用できる多くのツールとテクニックがあります。この記事では、インターフェースのパフォーマンスを向上させるために使用できるCSS最適化のヒントを紹介します。

1.パフォーマンスのボトルネックを見つける


すべてのタイプの最適化で最も重要なことは、徹底的な監査から始めることです。幸い、パフォーマンスのボトルネックを見つけるのに役立つCSS診断ツールがたくさんあります。何よりもまず、ブラウザの「開発者ツール」を使用して、データの読み込み速度を確認できます。ほとんどのブラウザでは、F12キーを押して開発者ツールを開くことができます。

たとえば、Firefoxブラウザの「開発者ツール」では、「ネットワーク」タブを使用して、ページに読み込まれたすべてのCSSファイルのサイズと読み込み時間を確認できます。また、CSSファイルがキャッシングありまたはなしでロードされる速度を確認することもできます。外部CSSは、このようなサードパーティのCDNサーバからダウンロードしたGoogleのフォント、フォントファイルやCSSファイルなど、ここで示されているので、あなたがいることを多くのソースを見つけることができます



PingdomのツールとGoogleの灯台がさえ知りませんでした前に- 2つの無料ツールの開発サイトの速度とインターフェースのパフォーマンスを分析するためによく使用されます。たとえば、Pingdom Toolsを使用すると、簡単なWebサイトの速度テストを実行するときに、CSSの最適化に役立つヒントが得られます。



2. CSS-


ほとんどのWebサイトは複数のCSSファイルを使用しています。ほとんどの場合、モジュラーCSSファイルの使用が最良のソリューションと見なされていますが、個々のファイルのロードには時間がかかる場合があります。しかし、まさにこの理由から、CSSを縮小および圧縮するためのツールがあります。それらを賢く使用すると、ページの読み込み時間を大幅に短縮できます。CSSファイルを単純なフォームにコピーするだけでCSSファイルを縮小できる

、CSS Minifyなどのオンラインサービスがあります。このタイプのサービスは、小規模なプロジェクトでうまく機能します。ただし、多くのCSSファイルを含む大規模なプロジェクトでは、これらを使用するのは面倒で時間がかかる場合があります。このような状況では、自動化されたソリューションを優先することをお勧めします。

現在、ほとんどのビルドツールでは、自動的に圧縮することができます。たとえば、デフォルトでは、Webpackはすべてのプロジェクトファイルを縮小パッケージとして返します。PostCSSには、CSS Nanoなどのスマートプラグインもあり、ファイルを縮小するだけでなく、ファイルに対して多くの特別な最適化を実行します。



3. FlexboxとCSSグリッドを使用する


CSSを記述し、マージン、パディング、フロートを使用してページ上の要素を揃える際に従来のブロックモデルのみに依拠している場合は、FlexboxおよびCSSグリッドと呼ばれるより高度なメソッドへの切り替えを検討する必要があります。これらを使用すると、はるかに少ないコードで複雑なレイアウトを実装できます。

古いアプローチを使用すると、要素を垂直方向に中央揃えにするなどの単純なことでも、多くのトリックを使用する必要があります。ただし、これはFlexboxおよびCSSグリッドには適用されません。新しいアプローチを学ぶのには時間がかかるかもしれませんが、CSSファイルのサイズがはるかに小さくなるので、それは価値があります。これは特に、現在非常に優れたブラウザーサポート（世界全体で98.3％）を備えているFlexboxに当てはまります。



CSSグリッドはブラウザーで十分にサポートされていませんが（グローバルでは92.03％）、古いブラウザーをサポートする必要がない場合、またはフォールバックを実装する準備ができている場合は、既にこのアプローチを使用できます。



4.ルールの代わりに<link>タグを使用します @import


CSSファイルをWebページにロードするには、主に2つの方法があります。

  1. <link>タグを使用して<head>セクションにHTMLページを追加する
  2. CSS宣言を使用して他のスタイルシートからインポートする @import

@importメインCSSファイルの上部に 広告を追加する必要があります。ほとんどの場合、このアプローチは、フォントやその他のデザイン要素などの小さなリソースを読み込むために使用されます。一見すると、これは良い解決策のように見えるかもしれませんが、ブラウザが追加のスタイルシートをロードするには、HTMLページが<link>タグを使用してスタイルシートを直接ロードする状況よりもはるかに多くの時間が必要です。

HTMLページに複数のCSSファイルを追加するときは、常にCSSの特異性を考慮してください。一般的なスタイルシートから始めて、より具体的なスタイルシートを指定します。後で追加されたスタイルシートは以前のCSSファイルのルールを上書きするため、この原則に従う必要があります。 CSSファイルが正しい順序で追加される例：

<link rel="stylesheet" href="main.css">
<link rel="stylesheet" href="page.css">
<link rel="stylesheet" href="component.css">

5. SVG


Webページ上のすべての画像のアップロードには時間がかかる場合があります。この時間を短縮するために、開発者は外部CDNからの画像の読み込みやTinyJPGなどの画像圧縮ツールの使用など、多くの画像最適化手法を使用しています。これらのソリューションは非常に役立ちますが、状況によっては、リソースを大量に消費するJPGおよびPNG画像の使用をCSS効果に置き換えることができます。

たとえば、巨大な背景画像の代わりにグラデーションを使用できます。これにより、ページ訪問者のブラウザの速度がわずかに低下する可能性があります。 CSSグラデーション関数を使用して、線形グラデーション、放射状グラデーション、繰り返しグラデーションを作成できます。これらの組み込みCSS関数を使用して、色だけでなく、グラデーションの角度も指定できます。

たとえば、次のルールは、どの画像よりもはるかに高速に読み込まれる美しいグラデーションの背景を作成します。

div {
    background: linear-gradient(45deg, lightgreen, royalblue);
}

より複雑なグラデーションやテクスチャの場合は、CSSmatic（下の画像）やColorZillaなどのジェネレーターを使用することもできます。



グラデーションに加えて、従来のJPGおよびPNG画像をスケーラブルベクターグラフィックス（SVG）に置き換えることもできます。ロードが高速になるだけでなく、イメージの1つのバージョンのみをダウンロードする必要もあります。これは、SVG画像がベクターの性質により品質を損なうことなく任意のサイズに拡大縮小できるためです。さらに、通常のHTMLファイルのように、CSSでSVGのスタイルを設定することもできます。

6.ルールを避けてください！重要


！重要な ルールは特定の状況では実際に見つけることができますが、最後の手段としてのみ使用する必要があります。このルールは、カスケードから例外をスローします。つまり、追加すると、CSS宣言にとって重要であり、より具体的な宣言であっても、他の宣言は上書きされます。次に構文を示します。

h1 {
   margin-bottom: 20px !important;
}

CSSの重要なルールが多数ある場合、ユーザーのブラウザはコードで追加のチェックを実行する必要があるため、ページがさらに遅くなる可能性があります。絶対に使用しないことをお勧めします！サイト全体、またはテーマやプラグインを作成するときに重要です。可能であれば、サードパーティのライブラリからCSSをオーバーライドする場合にのみ、このルールを使用してください。

7. CSSリファクタリング


CSSのリファクタリングは簡単な作業ではありませんが、多くの場合、Webサイトのパフォーマンスを大幅に向上させることができます。たとえば、CSSファイルが大きすぎる場合、古いコードベースを取得した場合、またはページの読み込み時間が非常に短く、変換に重大な悪影響を与える場合などです。CSSリファクタリングの目標は、コードをよりエレガントで、保守しやすく、ロードを高速化することです。

CSSリファクタリングは、CSSコードのあらゆる側面を分析する必要がある複数のステップからなるプロセスです。次の点を確認する必要があります。

  • 未使用または重複するCSSルールまたはリソースはありますか
  • フレックスボックスやCSSグリッドなどのより近代的な技術を使用することは可能ですか
  • あまりにも多くの特異度が使用されています（これは視覚的特異度計算機を使用して計算できます）
  • CSSファイルの構造は適切に整理されていますか（たとえば、大きいファイルよりも小さいファイルを維持する方が簡単です）
  • 自動ビルドツールの使用を開始する必要がありますか
  • そして、はるかに。

リファクタリングを開始する前に、測定可能な目標を設定し、ガイドされる基準（ページの読み込み速度や最初にレンダリングされたコンテンツの時間など）を選択して、前後の値を比較できるようにします。

また、Gitなどのバージョン管理システムを使用することも忘れないでください。この場合、何か問題が発生した場合は、以前のバージョンのコードに戻すことができます。

要約する


Webサイトのパフォーマンスを向上させるために使用できるCSS最適化のヒントがたくさんあります。それらのほとんどは実装が簡単ですが、ページの読み込み時間に大きな影響を与える可能性があります。ページの読み込みが速くなると、使いやすさが向上するだけでなく、Googleや他の検索エンジンでの位置も改善されます。

CSS最適化のベストプラクティスに加えて、キャッシング、Google AMP、HTTPSプロトコルなど、他の読み込み高速化手法を使用できます。それらについて詳しく知りたい場合は、記事10ステップガイドを読んで、ウェブサイトの読み込み速度を改善することもできます。
