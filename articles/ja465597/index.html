1.マイクロコントローラーの選択


LQFP32パッケージにSTM8S105K6がいくつかありますが、これはこのファミリーのものなので、選択しました。

シリーズ内の任意のMKを選択できます。

STのディスカバリーボードは準備が整っており、中国語から購入するか、他のリソースから注文できます。中国製のSTM8S103F3P6などの既製のボードがあります。

ただし、既製のボードはトレーニング用に設計されていることに注意してください。

メーカーのウェブサイトst.comで、MKを見つけてドキュメントをダウンロードします。

リファレンスマニュアル：RM0016-すべてのMKファミリSTM8SおよびSTM8AFの共通ドキュメント。次はRMです。
データシート：STM8S105C4 / 6、STM8S105K4 / 6およびSTM8S105S4 / 6のDS5855ドキュメントがあります。さらにDS。

彼が持っているおいしいものから（表面的に）：

  • 16 MHzサポート
  • 32 KBフラッシュ、1 KB EEPROMおよび2 KB RAM
  • 電源2.95-5.5V
  • 8ビットおよび16ビットタイマー
  • UART、SPI、I2C
  • 10ビットADC
  • 最大38のディスクリート入力/出力

3. IDEの選択


私にとって、選択の問題はまだ関係ありません。私はSTM8のIARで作業します。これは、以前のバージョンと同じ理由で、現在作業中のプロジェクトを作成したためです。

サイトiar.comにアクセスし、検索でMKのモデルを入力します。
STM8用のIAR Embedded Workbenchをダウンロードし、インストール時にサイズ制限のある評価を選択して、8 KBのコードの制限に耐えます。または購入。ロシア連邦の領土で禁止されているリソースにハッキングされたバージョンがありますが、商業プロジェクトには適していません。

インストール手順は、インストーラーまたはGoogleにあります。
バージョン6.5.3.2581を使用しています。
プログラマー用のドライバーと一緒に環境をインストールします。

4.プロジェクトを作成する


IDEに移動し、[プロジェクト]-> [新しいプロジェクトの作成]-> [C ++]を

クリックします。[OK]をクリックし、ディレクトリ（キリル文字とスペースを除く）を選択し、名前（スペースとキリル文字を含まない）を選択します。

IDEのルートフォルダーに移動して、incフォルダーを見つけます。

C:\Program Files (x86)\IAR Systems\Embedded Workbench 6.5\stm8\inc


iostm8ヘッダーファイルの中には、私のためにMKモデルがあります。 

iostm8s105k6.h

最初に接続し、無限ループを追加します。

結果：
#include<iostm8s105k6.h>  // 

int main()  
{
  while(1){  // 
  }
  return 0;
}


[プロジェクト]-> [オプション]-> [一般オプション]列の[デバイス]で、MKを選択します。

Project-> Options-> DebuggerのDriver列でST-LINKを選択します。

ツールバーの[ダウンロードしてデバッグ]ボタンを見つけます。
最初のコンパイル時に、環境は.ewwファイルの名前を選択するように要求します。（私はそれをプロジェクトと同じと呼びます。）

デバッグログウィンドウの下部には、次のようなメッセージが表示されます。

  • STM8のプリプロセッサー-プリプロセッサーが開始されました
  • STM8用デバッガ-デバッガが起動しました
  • STM8 SWIMデバッグシステムに接続-MKに接続
  • ロードされたデバッグ対象-ファームウェアのダウンロードが発生しました
  • ターゲットのリセット-リセットが発生しました

デバッグモードですが、プログラムは一時停止しています。
ツールバーの同じ場所で、[検索]ボタンを見つけて押します。
MKをデバッグモードで起動しました。

[デバッグの停止]をクリックすると、デバッグが停止しますが、MKはプログラマーによって提供され、プログラムを実行します。つまり、無限ループをスピンし、何もしません。

概要


その結果、簡単な概要と、STM8SシリーズのMKをプログラミングせずにプログラミングするための概要が紹介されました。レイアウトの組み立てから、これは通常すべてから始まり、作業の基礎のみが判明しました。
