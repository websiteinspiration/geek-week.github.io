複数のdocker-compose.ymlファイルの使用


複雑な構成があり、コンテナーの特定の基本レイヤーがあり、これは、たとえば、常に必要です。そして、通常、私たちは隣のチーム\別のプロジェクト\インターネットから取得して、あなたのニーズに合わせて仕上げます。ただし、いくつかのコマンドがある場合は、基本的な部分を共通の内部リポジトリに移動できます。また、ほとんどのプロジェクトで同じバージョンのベースパーツが使用されますが、これもバージョン管理されています。


基本的なdocker-compose-base.ymlの例を説明しましょう。


これが、証明書、チューニング、および発言指標を含むカスタマイズされたnginxイメージであるとします。そしてプロメテウスの輸出業者：


version: '2'
services:
  nginx:
    image: nginx
  nginx-exporter:
    image: nginx/nginx-prometheus-exporter

次に、docker-compose-app.ymlアプリケーションの例について説明します。


version: '2'
services:
  backend:
    image: internal.local/super-app:0.1.2

まず、通常のチームが必要ですが、1つだけ違いがあります。2つのdocker-composeファイルを示します。


docker-compose up -d -f docker-compose-base.yml -f docker-compose-app.yml

ほら、1つのdocker-composeファイルで記述されているかのように、一連のサービスを取得します。

#!/bin/bash

dc=${1}

if [ ! -z ${dc} ] && [ -f ${dc} ]; then
  echo "Saving docker images from file ${dc}..."
  images=`grep image: ${dc} | awk '{print $2}'`
  docker save ${images} | gzip > docker-images.gz
  echo "Success!"
else
  echo "ERROR! You must set path to docker-compose.yml as argument!"
fi
