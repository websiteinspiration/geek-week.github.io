LinuxのC＃でのセレンテスト

Seleniumを使用したWebアプリケーションテストの自動化は自動テスト開発者の間で一般的なソリューションであり、C＃は最も人気のあるプログラミング言語の1つであるため、これらのツールを組み合わせても問題は発生しません。これらのテクノロジーを使用した開発では、Microsoftの人気のあるWindows専用ソフトウェアがよく使用されますが、このタスクのためにSelenium + C＃スタックを離れずに使用できる無料のアナログを知りたいと思いました。

私はこのテーマに関するロシア語の記事を見つけられなかったので、LinuxのC＃で自動テストを開発およびデバッグするための環境をセットアップした経験を共有します。

OSは、公式サイトからダウンロードしたISOイメージからインストールしたLinuxカーネル4.15.0-99-genericを搭載したKubuntu 18.04 64-bitを使用しました。最近のかなり人気のあるLinuxディストリビューションなら何でもできると思います。

C＃のCLRの役割は、Mono JITコンパイラバージョン6.6.0.166によって実行されました。インストールは、このページからターミナル（KubuntuではKonsole）へのコマンドの順次コピーと実行で構成されていました。

そしてIDEとして、MonoDevelop 7.8.4（ビルド2）が使用され、Monoと同様にインストールされました。

Seleniumは多くのブラウザをサポートしていますが、私は面倒で、Chromeに限定して64ビットの.debパッケージをインストールしました。

次に、MonoDevelopでソリューションを作成します。

  • MonoDevelopを実行する
  • 「ファイル」メニューに行く
  • 「ソリューションを作成」を選択します
  • 「.NET」をクリック
  • 「NUnit Library Project」を選択し、「Next」をクリックします
  • 名前とソリューションを指定して、[作成]をクリックします



ブラウザーを制御するには、いくつかのNuGetパッケージが必要です。

  • [プロジェクト]メニューに移動し、[NuGetパッケージの追加]を選択します
  • Selenium.WebDriverパッケージを検索してインストールします
  • Selenium.WebDriver.ChromeDriverパッケージを検索してインストールします



実際には、これですべてです。すべてが正しく設定されていることを確認するためのコードを記述する必要があります。ソリューションを作成すると、テストメソッドTest.csのファイルが自動的に作成されます。それに次の数行のコードを挿入します。

using NUnit.Framework;
using System;
using OpenQA.Selenium.Chrome;
using OpenQA.Selenium;

namespace SeleniumTests
{
    [TestFixture()]
    public class Test
    {
        [Test()]
        public void TestCase()
        {
            IWebDriver driver = new ChromeDriver();
            driver.Navigate().GoToUrl("http://habr.com/");
            Assert.IsTrue(driver.Url.Contains("habr.com"), "-   =(");
            driver.Quit();
        }
    }
}

テストは「単体テスト」タブから起動されます。表示されない場合は、「表示」メニューから「テスト」を選択します。
