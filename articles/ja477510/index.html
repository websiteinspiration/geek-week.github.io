Mikrotikの動的アドレスリストの自動バックアップと復元

MikrotikのRouterOSはアドレスリストの動的要素を記憶しておらず、再起動または電源喪失の場合、静的要素のみがリストに残ります。たとえば、スパマーやポートスキャナーをキャッチし、IPアドレスを1か月間禁止するルールがあるとします。通常、このようなアドレスは動的エントリとしてリストに保存されますが、再起動中には保存されません。つまり、たとえば数か月の継続的なゲートウェイ操作で収集された「大根」データベースが失われやすくなります。

このスキームが機能するためには、読み取り/書き込み権限を持つアカウントを持つFTPサーバー（私はWindowsを持っていますが、バッチファイルはnixesに簡単に再利用できます）が必要です。このメカニズムは、ゲートウェイ上のbackupDynList_to_FTPスクリプトとBackupDynListFromFTPスクリプト、およびFTPサーバー上のconvertmtik.batスクリプトの要素の相互作用に基づいています。 Mikrotikの内部メモリをオフロードするために（リストは非常に大きくなる可能性があり、不揮発性メモリは誰にとっても大きくない）サードパーティのサーバーが必要であり、組み込みのスクリプト言語mikrotikが持つより強力なテキスト処理手段でファイルを処理するために必要です。 DynListExport関数呼び出しは、リストのサイズに関するシステム制限を回避するために使用されます。リストを変換する必要があるのは、データベースにすでに同じ要素（たとえば、静的レコード）がある場合にそうなります。その後、インポート中にエラーが発生し、インポートが停止します。これを回避するために、静的レコードが追加されるたびにエラーハンドラーがリセットされ、一致した場合、レコードは単に追加されず、インポートプロセスはさらに進みます。

BackupDynList_to_FTPスクリプトは1時間ごと（00:00）に実行され、すべてのリスト内の動的レコード（少なくとも30分は存続可能）に関するすべてのデータをiplist_dyn.srcファイルに収集し、このファイルをftpに送信します。サーバーでは、最初のスクリプト（00:30）から30分シフトして1時間ごとに、convertmtik.batバッチファイルが実行されます。これにより、ゲートウェイから受信した元のスクリプトが、偶然エラーから保護されたスクリプトに変換され、iplist_dyn_done.srcという名前で保存され

ます。ゲートウェイがクラッシュまたは再起動した場合、動的レコードに関するデータは起動時に自動的に消えますが、起動から60秒後に、BackupDynListFromFTPスクリプトがiplist_dyn_done.srcファイルをFTPからダウンロードして実行を開始し、リストを復元します。

Hourly_Dynlist_Backup_on_FTP（/ sys sheduler）アクティベーターは、毎時0分に実行されます。

/system script run BackupDynList_to_FTP

スクリプトBackupDynList_to_FTP（FTPアドレス、ユーザー名、パスワードを変更することを忘れないでください）：

/system script environment remove [ find where name="DynListExport" ];
:global DynListExport do={
:foreach i in=[/ip firewall address–list find where dynamic=yes and timeout>0d00h30m] do={
:local list [/ip firewall address–list get $i list];
:local address [/ip firewall address–list get $i address];
:local timeout [/ip firewall address–list get $i timeout];
:local comment [/ip firewall address–list get $i comment];
:put "/ip firewall address–list add list=$list address=$address timeout=$timeout comment=\"$comment\";"};
}
:log info "Starting Backup to FTP Script..."
:global iplistfile ("iplist_dyn.rsc")
:if ([/file find name=$iplistfile]!= "") do={/file rem $iplistfile};
/execute script="\$DynListExport" file=$iplistfile
:delay 60s
/tool fetch address="_FTP_" port=21 mode=ftp src–path="iplist_dyn.rsc.txt" user= password= dst–path="iplist_dyn.src" upload=yes
:delay 20s
/file rem $iplistfile
:log info "Finished Backup to FTP!"

ゲートウェイの起動時（起動時）に、Activator BackupDynList_from_FTP（/ sys sheduler）が実行されます。

{:delay 60s};
/system script run BackupDynListFromFTP

スクリプトBackupDynListFromFTP（FTPアドレス、ユーザー名、パスワードを変更することを忘れないでください）：

:local BackupFile "iplist_dyn_done.src"
/file remove [find name=$BackupFile]
/tool fetch address="_FTP_" port=21 mode=ftp src–path="$BackupFile" user= password=
/import file–name=$BackupFile
{:delay 30s};
/file remove [find name=$BackupFile]
/log info "$BackupFile imported"

Convertmtik.batサーバースクリプト-サーバーのパスに登録する必要があるgnuwin32などからの作業にLinux SEDユーティリティのポートを使用します。

echo # ––––––––––––––––––––––––––––––––––––––––––––––––––––– — > iplist_dyn_done.src
echo # %date% %time% >> iplist_dyn_done.src
echo # ––––––––––––––––––––––––––––––––––––––––––––––––––––– — >> iplist_dyn_done.src
echo /ip firewall address–list > iplist_dyn_done.src
sed –e "s/;//" –e "s/\/ip firewall address–list //" –e "s/.*/:do { & } on–error={}/" iplist_dyn.src >> iplist_dyn_done.src
