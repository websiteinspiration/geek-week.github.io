Quarkus：JBoss EAPクイックスタートのhelloworldサンプルを使用したアプリケーションのアップグレード

このブログの皆さん、こんにちは。Quarkusシリーズの4番目の投稿をお届けします。 （ちなみに、ウェビナー「これはQuarkus-KubernetesネイティブJavaフレームワークです。」を参照してください。既製のソリューションをゼロから開始または転送する方法を見てみましょう）



以前の投稿は、QuarkusがMicroProfileとSpringを組み合わせる方法についてでした。Quarkusは「超高速サブアトミックJava」として位置付けられていることを思い出してください。それは「Kubernetes指向のJavaスタックであり、GralalVMとOpenJDK HotSpotによって強化され、最高のライブラリーと標準からコンパイルされます」。今日は、Red Hat JBoss Enterprise Application Platform（JBoss EAP）クイックスタートリポジトリから helloworld アプリケーションを使用して、Quarkus機能を使用して既存のJavaアプリケーションをアップグレードする方法を示しますQuarkusがサポートするCDIおよびサーブレット3テクノロジーを使用します。

ここで、QuarkusとJBoss EAPはどちらも、最大限の標準に基づいて構築されたツールの使用に重点を置いていることに注意することが重要です。JBoss EAPでアプリを実行していませんか？問題ありません。RedHat Application Migration Toolkitを使用して、現在のアプリケーションサーバーからJBoss EAPに簡単に移行できます。その後、アップグレードされたコードの最終的な作業バージョンがgithub.com/mrizzi/jboss-eap-quickstarts/tree/quarkusリポジトリのhelloworldモジュールで利用可能になります。

この投稿は、Quarkusのチュートリアルを使用して作成されました。主に、最初のアプリケーションの作成とネイティブ実行可能ファイルの作成です。

helloworldのアップグレード


次に、元のhelloworldアプリケーションをアップグレードします。

新しいブランチを

作成するクイックスタートプロジェクトが完了したら、新しい作業ブランチを作成します。

$ git checkout -b quarkus 7.2.0.GA

pom.xmlファイルの変更pom.xmlファイル

からアプリケーションの変更を開始します。QuarkusがXMLブロックを挿入できるように、helloworldフォルダーで次のコマンドを実行します。

$ mvn io.quarkus:quarkus-maven-plugin:0.23.2:create

この記事の執筆時には、バージョン0.23.2が使用されていました。多くの場合、Quarkusには新しいバージョンがあります;最新バージョンはgithub.com/quarkusio/quarkus/releases/latestで確認できます。

上記のコマンドは、pom.xmlに次の要素を挿入します。

  • 使用するQuarkusのバージョンを指定する<quarkus.version>プロパティ。
  • 各Quarkus依存関係のバージョンを追加しないように、Quarkus BOM（部品表）をインポートするための<dependencyManagement>ブロック。
  • quarkus-maven-pluginプラグイン。アプリケーションのパッケージ化と開発モードの提供を担当します。
  • アプリケーション実行可能ファイルを作成するためのネイティブプロファイル。

さらに、pom.xmlに次の変更を手動で加えます。

  1. <groupId> <parent> <artifactId>. <parent>, <groupId>.
  2. <parent>, Quarkus pom JBoss.
  3. <version> <artifactId>. .
  4. <packaging>, WAR, JAR.
  5. :
    1. javax.enterprise:cdi-api io.quarkus:quarkus-arc, <scope>provided</scope>, ( ) Quarkus- injection CDI.
    2. org.jboss.spec.javax.servlet：jboss-servlet-api_4.0_spec依存関係をio.quarkus：quarkus-undertowに変更し、（ドックに従って）このQuarkus拡張機能がサーブレットサポートを提供するため、<scope>が提供する</ scope>を削除しますの
    3. org.jboss.spec.javax.annotation：jboss-annotations-api_1.3_spec依存関係を削除します。これには、変更した依存関係が含まれているためです。


すべての変更を含むpom.xmlファイルのバージョンは、github.com/mrizzi/jboss-eap-quickstarts/blob/quarkus/helloworld/ pom.xmlにあります。

上記のmvn io.quarkus：quarkus-maven-plugin：0.23.2：createコマンドはpom.xmlファイルを変更するだけでなく、プロジェクトにいくつかのコンポーネント、つまり次のファイルとフォルダーを追加することに注意してください。

  • mvnw and mvnw.cmd .mvn: Maven Wrapper Maven Maven .

import javax.enterprise.context.ApplicationScoped;

helloworldをJARパッケージにビルドします
。コードが正しく機能するようになったので、次のコマンドでコードをパックします。

$ ./mvnw clean package

このコマンドは、/ターゲットフォルダーに2つのJARファイルを作成します。helloworld-.jarファイルは、プロジェクトクラスとリソースとともにMavenチームによって組み立てられた標準のアーティファクトです。また、helloworldファイルは、実行可能JARであるrunner.jarです。

これはuber-jarではないことに注意してください。すべての依存関係が単に/ target / libフォルダーにコピーされます（JARファイルにパッケージ化されていないため）。したがって、このJARを別のフォルダーまたは別のホストから実行するには、JARパッケージのMANIFEST.MFファイルのClass-Path要素に、JARの明示的なリストが含まれている場合、JARファイルと/ libフォルダーの両方をコピーする必要があります。 libフォルダー。
uber-jarアプリケーションの作成方法については、Uber-Jar作成チュートリアルを参照してください。

JARにパッケージ化されたhelloworldを起動します

これで、標準のjavaコマンドを使用してJARを実行できます。
