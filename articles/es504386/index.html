Variables virtuales de Vassbotn H. Class

¡Saludos a todos los que comenzaron a leer este artículo! Quiero invitarlos (y en gran medida a los fanáticos de Object Pascal, amantes de los trucos de programación, técnicas y refinamientos sintácticos) para que se familiaricen con la traducción de la publicación bastante antigua (2007) "Hack # 17: Variables de clase virtual, Parte I" y "Hack # 17: Variables de clase virtual, Parte II , de Hallvard Vassbotn, un reconocido desarrollador y autor de numerosas técnicas orientadas a Delphi.


El mensaje de Hallward está dedicado a uno de los temas más interesantes: la posibilidad de colocar y utilizar datos asociados con una clase particular de objetos de aplicación. Si pensabas en constantes o variables de clase, entonces tendrás razón, pero solo parcialmente.
Este artículo trata sobre la clase virtual var., en ruso, esta frase suena como una variable virtual de una clase (es precisamente esta frase la que se coloca en el nombre de esta traducción). Dirás que no existe tal construcción sintáctica en Object Pascal, y tendrás toda la razón. Además, estoy seguro de que los expertos y desarrolladores en otros lenguajes de programación OO declararán: en mi práctica, no he escuchado, encontrado o usado algo como esto. Como todo, "no veo un gopher ...", pero argumentaré que puede existir. Y esto se discutirá en el texto de la traducción propuesta.


Prefacio del traductor


Durante los muchos años que han pasado desde que leí por primera vez la publicación de Hallward, no pude decidir usar su maravillosa idea en mis proyectos. Pero su belleza y la solución ofrecida por Hallward no me dejaron y todavía no me dejan solo.


Era hora de recolectar piedras: tenía una tarea para la cual la mejor solución para su implementación sería simplemente un "hack" (y todavía llamaría a tales métodos tecnológicos hacks) de Hallward. Una vez que pueda usarlo, debe estar preparado para lo inesperado: una vez más, estudie el material, vuelva a leer la carta, las instrucciones ...


Me gustaría aclarar mis motivos para trabajar en la traducción de mensajes de Hallward.

// Self-modifying code - change one DWORD in the code segment
var
  RestoreProtection, Ignore: DWORD;

  if VirtualProtect(Code, SizeOf(Code^), PAGE_EXECUTE_READWRITE,
    RestoreProtection) then

    Code^ := Value;

  end;

class procedure TBasicObject.InitVMTPlaceholders;
begin
  // First, check if the VMT-mapping came thru the compiler alright :
  if Pointer(ClassGetClassInfo) = Addr(TBasicObject.VMT_Placeholder1) then
  begin
    // Now, empty the variable default, 
    // very important for later code !
    PatchCodeDWORD(@PBasicObjectOverlay(Self).OurClassInfo, DWORD(nil));

    // Now check that we see a cleaned up variable :
    Assert(ClassGetClassInfo = nil, 'Failed cleaning VMT of ' + ClassName);
  end

    // When there's no original content anymore, this initialization 
    // has already been done - there _has_ to be a nil here :
    Assert(ClassGetClassInfo = nil, 
      'Illegal value when checking initialized VMT of ' + ClassName);
end;
