p5.j​​sを使用して3Dオブジェクトに2Dテクスチャを重ね合わせる（パート2-キューブにパターンを適用する）



これは、3Dオブジェクトに動的な2Dパターンを課すプロジェクトの2番目の部分です。最初のパート（パート1）から開始することも、結果のコードを数段落下にコピーすることもできます。

少し前に、p5.jsライブラリーの数行のコードを使用して生成されるパターンを作成しました。
p5.j​​sは2014年に作成されたJavaScriptライブラリです。その主な目標は、デザイナー、アーティスト、教師、その他のクリエイティブな職業の代表者のためのプログラミングの世界への窓になることです。アニメーションと図の両方をサポートしており、簡単にWeb形式に変換できます。

最後に、3Dオブジェクト（この場合は立方体）にパターンを適用する方法を説明します。

ゼロから作業を開始するには、オンラインエディターeditor.p5js.orgでプロジェクトを作成する必要があります。次のコードは引き続き変更します。

let r=10
let a=0
let =20

function setup() {
  background(220);
  createCanvas(400, 400);
  
}

function draw() {
translate(200,200)
  let x = r+*cos(a);
  let y = r+*sin(a);
  fill(255,0,0);
  ellipse (x,y,10,10);

  a = a + 0.3

キャンバスを大きくするには、キャンバスを作成する関数createCanvas（400,400、WEBGL）にWEBGLパラメータを追加する必要があります。そのようなオブジェクトの基準点はすでに中心にあるため、フラットな画像で補正を実行した移動（200,200）を削除できます（必要があります！）。

色域を多様化し、より壮観な画像を作成するために、fill（）パラメーターを変数r、a、cに置き換えましょう。今度は、無限描画（）ループの新しい反復ごとに、楕円の色がわずかに変化します。また、変数cとaの変化率を変更して、円と傾斜角の間の距離を追加します。

   =  + 0.06
  a = a + 0.3

最後の改善として、パターンコードの前にpush（）を追加し、その後にpop（）を追加しましょう。この方法でフレーミングすると、コードが分離されます。これは、作成する他のオブジェクトの影響を受けません。

let r=10
let a=0
let c=20

function setup() {
  background(220);
  createCanvas(400, 400, WEBGL);
  
}

function draw() {
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  fill(r,a,c);
  ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
}



結局、このようなものが得られます。キューブを作成します。

同じdraw（）関数で、pop（）の後にボックスコードを追加します。

  push()
  box(200)
  pop()

これで、サイズが200の箱が片側だけに向けられ、正方形のように見えます。ボリュームを見積もるには、デプロイする必要があります。これを行うには、4番目の変数topを、既にそこにあるr、a、cの1つにデフォルト値0の角度で追加します。このパラメーターがキューブに影響を与えるには、2回目のプッシュ（）の後、ボックスを作成する前に行います。 3つの関数、rotateX（）、rotateY（）、rotateZ（）をそれぞれの角度パラメーターとともに追加します。

回転角度が0度のオブジェクトが回転しないことは論理的であるため、可変角度の値の変更を0.003に追加します（最初はそれで十分です）。その結果、次のコードが取得されます。

let r=10
let a=0
let c=20
let angle = 0

function setup() {
  background(220);
  createCanvas(400, 400, WEBGL);
  
}

function draw() {
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  fill(r,a,c);
  ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
  
  push()
  rotateX(angle), 
  rotateY(angle), 
  rotateZ(angle)
  box(200)
  angle = angle + 0.003
  pop()
}

私たちの背景は1回だけ作成されるため（設定中）、キューブを回転すると背景が消去される最初のバグをキャッチします。



これを修正するには、単にdraw（）関数内でバックグラウンド（200）を転送します。ただし、別の問題が発生しています。円は最初と同じように消えます。これは、新しいサイクルで上書きされ、一瞬だけ表示されるためです。無。

この問題を解決するには、デフォルト値のない新しい変数（たとえば、art）を、最初に他のトップレベルの変数の隣に作成する必要があります。

setup（）関数内では、それをcreateGraphics（400,400）関数と同等にして、他の場所で使用します。
次元を維持するために、キャンバスの作成時と同じパラメーターを設定してください。

次に、パターンコード内で、ドットを介して楕円とそのプロパティ（fill（）など、作成した場合は他のプロパティ）にart変数を追加します。 

art.fill(r,a,c);
  art.ellipse (x,y,10,10);

その後、それはテクスチャとしてキューブにパターンを課すために残ります。これを行うには、キューブボックスコード内にテクスチャ（アート）プロパティを追加します。ロジックは次のとおりです。指定したプロパティを使用してパターンを作成するための関数を呼び出す、グラフィック分離オブジェクトアートのエッセンスを作成し、その後、アートのプロパティパラメーターとしてキューブのテクスチャを指定します。

let r=10
let a=0
let c=20
let angle = 0
let art

function setup() {
  createCanvas(400, 400, WEBGL);
  art = createGraphics(400,400);
}

function draw() {
  background(220);
  push()
  let x = r+c*cos(a);
  let y = r+c*sin(a);
  art.fill(r,a,c);
  art.ellipse (x,y,10,10);
  c = c + 0.09
  a = a + 0.8
  pop()
  
  push()
  texture(art)
  rotateX(angle), 
  rotateY(angle), 
  rotateZ(angle)
  box(200)
  angle = angle + 0.003
  pop()
}



それは最後の問題を解決するために残っています：私たちのパターンは立方体の隅から現れ始めます。中心から開始するには、単純に立方体の幅と長さの半分をxおよびyパラメータに追加して、パターンの描画の開始の中心をシフトします。

art.ellipse (x+200,y+200,10,10);



パターンを多様化したい場合は、パラメーターを変更するか、たとえば別のプロジェクトからいくつかの交差するパターンを追加することもできます- 初心者のためにp5.jsライブラリーでモアレパターンの効果を再現する方法（クイックガイド）
