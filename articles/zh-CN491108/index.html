使用helmfile在许多k8s环境中进行组织部署

Helmfile - helm的包装器，它使您可以在一处描述许多helm版本，为多个环境参数化其图表的图表，还可以设置其部署顺序。


您可以在自述文件和最佳实践指南中了解helmfile本身及其使用示例。


我们将熟悉在Helmfile中描述发布的非显而易见的方法


假设我们有一堆舵图（例如，有postgres和某种后端应用程序）和几个环境（几个kubernetes群集，几个namespace'ov或两者兼有）。我们使用helmfile，阅读文档并开始描述我们的环境和发行版：

helmfile.yaml


environments:
  devel:
  production:

releases:
  - name: postgres
    labels:
      app: postgres
    wait: true
    chart: stable/postgresql
    version: 8.4.0
    values:
      - envs/{{ .Environment.Name }}/values/postgres.yaml
  - name: backend
    labels:
      app: backend
    wait: true
    chart: private-helm-repo/backend
    version: 1.0.5
    needs:
      - postgres
    values:
      - envs/{{ .Environment.Name }}/values/backend.yaml

最后，我想补充一下，默认环境的变量可以依次由运行程序的运行程序的OS环境变量进行参数化，从该运行程序将启动部署，从而获得动态环境


helmfile.yaml


environments:
  default:
    values:
    - global:
        clusterDomain: {{ env "CLUSTER_DOMAIN" | default "cluster.local" }}
        ingressDomain: {{ env "INGRESS_DOMAIN" }}
