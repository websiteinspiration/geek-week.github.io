また会いましたね！記事のタイトルはそれ自体について語っています。「データエンジニア」コースの開始を見越して、データエンジニアが誰であるかを理解することをお勧めします。記事には役立つリンクがたくさんあります。読書を楽しむ。



データエンジニアリングの波を捉え、それがあなたを深淵に引き込まないようにする方法に関する簡単なガイド。

今日では誰もがデータサイエンティストになりたいと思っているという印象を受けます。しかし、データエンジニアリングについてはどうでしょうか？本質的には、データアナリストとデータサイエンティストのハイブリッドのようなものです。データエンジニアは通常、ワークフロー、処理パイプライン、およびETLプロセスの管理を担当します。これらの機能の重要性のため、これは現在勢いを増しているもう1つの人気のある専門用語です。

高い給与と大きな需要は、この仕事を非常に魅力的なものにするほんの一部にすぎません！ヒーローの仲間入りをしたい場合は、学習を始めるのに遅すぎることはありません。この投稿では、最初のステップを実行するために必要なすべての情報を収集しました。

データエンジニアリングとは

正直なところ、これ以上の説明はありません。
「科学者は新しい星を発見できますが、作成することはできません。彼はエンジニアに彼のためにそれをするように頼まなければならないでしょう。」

したがって、データエンジニアの役割は非常に重要です。

その名前から、データエンジニアリングはデータ、つまりその配信、保存、処理に関連していることがわかります。したがって、エンジニアの主なタスクは、データに信頼できるインフラストラクチャを提供することです。ニーズのAI階層を見ると、データエンジニアリングは最初の2〜3段階を実行します。データの収集、移動、保存、準備



データエンジニアは何をしますか？

ビッグデータの出現により、責任の領域は劇的に変化しました。以前は、これらの専門家は大きなSQLクエリを記述し、Informatica ETL、Pentaho ETL、Talendなどのツールを使用してデータを上書きしていましたが、現在はデータエンジニアの要件が増えています。

データエンジニアの地位に空席があるほとんどの企業には、次の要件があります。

  • SQL Python.
  • , Amazon Web Services.
  • Java/Scala.
  • SQL NoSQL ( , ).

覚えておいてください、これは最も必要なことです。このリストから、データエンジニアはソフトウェア開発とバックエンドの分野の専門家であると想定できます。
たとえば、企業がさまざまなソースから大量のデータを生成し始めた場合、データエンジニアとしてのあなたの仕事は、情報の収集、その処理、および保管を整理することです。

この場合に使用されるツールのリストは異なる場合があります。すべて、このデータの量、受信速度、および不均一性によって異なります。ほとんどの企業はビッグデータにまったく遭遇しません。そのため、集中型リポジトリ、いわゆるデータリポジトリとして、SQLデータベース（PostgreSQL、MySQLなど）を使用して、データをリポジトリに送信する小さなスクリプトセットを使用できます。

Google、Amazon、Facebook、DropboxのようなITの巨人には、Python、Java、Scalaの知識など、より高い要求があります。

  • ビッグデータの経験：Hadoop、Spark、Kafka。
  • アルゴリズムとデータ構造の知識。
  • 分散システムの基本を理解する。
  • TableauやElasticSearchなどのデータ視覚化ツールでの経験は、大きなプラスになります。

つまり、ビッグデータへの明確なシフトがあります。つまり、高負荷での処理です。これらの企業は、システムのフォールトトレランス要件を強化しています。
