Tentang kematian dual boot dan kesatuan Windows dan Linux

Mesin kerja saya dulunya adalah laptop yang dibuat oleh Apple. Saya dapat melakukan hampir semua hal di dalamnya: mengembangkan program, menulis teks, menggubah musik, dan banyak lagi. Tetapi saya dihantui oleh pemikiran bahwa saya terikat pada ekosistem Apple, bahwa saya bergantung pada keinginan perusahaan ini. Jadi saya mulai mencari sesuatu yang baru.



Saya mulai mengumpulkan workstation untuk tugas pembelajaran mesin. Saya memasukkannya, antara lain, prosesor yang luar biasa, banyak memori, kartu video yang bagus. Hampir semua tugas saya diselesaikan di Ubuntu. Benar, saya membutuhkan Microsoft Office untuk bekerja dengan teks. Office Online belum muncul, dan mari kita sebut sekop sekop, LibreOffice hanyalah semacam horor. Bagi saya, solusinya adalah boot ganda dalam konfigurasi Ubuntu - Windows 10. Saya sangat menyukai perasaan kebebasan yang Anda alami ketika beralih dari OS dari Apple ke Ubuntu. Dan kemungkinan yang terbuka bagi mereka yang merakit komputer mereka sendiri hampir tidak terbatas.







Dual boot untuk waktu yang lama sangat cocok untuk saya. Dan ketika saya menggunakannya jutaan kali, teknologi WSL (Windows Subsystem untuk Linux, Windows Subsystem untuk Linux) muncul. Ketika ini terjadi, saya mulai menyelesaikan beberapa masalah Linux saya di Windows. Benar, meski begitu, saya masih belum memiliki banyak untuk pekerjaan penuh. Tapi sekarang, dengan rilis WSL 2, saya merasa bahwa versi baru WSL mampu mengubah situasi secara radikal. Hari ini saya mengusulkan untuk berbicara tentang bagaimana, menggunakan WSL 2, untuk mentransfer tugas mengembangkan program dari Linux ke Windows 10. Saya akan berbicara tentang fitur-fitur baru WSL 2, dan apa yang dapat diharapkan dari subsistem ini di masa depan.



Ulasan WSL 2



WSL 2 adalah versi baru dari subsistem Windows untuk Linux. Ada beberapa perubahan dalam versi ini yang menentukan bagaimana distribusi Linux berinteraksi dengan Windows.









Microsoft mencintai Linux



Dalam rilis WSL ini, pengguna telah meningkatkan kinerja sistem file dan kompatibilitas penuh dengan panggilan sistem. Tentu saja, kami memiliki kemampuan untuk menjalankan Linux baik di WSL 1 atau di WSL 2, dan, lebih lagi, Anda dapat beralih di antara berbagai versi WSL kapan saja. WSL 2 adalah hasil dari perbaikan besar dalam arsitektur dasar sistem, fitur-fiturnya yang baru didasarkan pada teknologi virtualisasi dan pada fungsi kernel Linux. Pada saat yang sama, Microsoft menangani perincian kecil, sehingga pengguna WSL dapat dengan mudah menjalankan bisnis mereka tanpa gangguan.



Instalasi



Microsoft berjanji bahwa menginstal WSL 2 akan sangat mudah dalam waktu dekat, dan bahwa Linux dapat diperbarui menggunakan mekanisme pembaruan Windows. Sementara itu, proses menginstal WSL 2 membutuhkan beberapa upaya, tetapi, pada kenyataannya, tidak ada yang perlu dikhawatirkan.



Dalam contoh ini, kita akan menginstal Ubuntu 20.04 di Windows 10. Perlu dicatat bahwa proses instalasi akan sama untuk semua distribusi Linux yang tersedia di Microsoft Store. Pertama, Anda perlu mengaktifkan komponen Subsistem Windows untuk Linux. Untuk melakukan ini, buka PowerShell sebagai administrator dan jalankan perintah berikut: 



dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart


Selanjutnya, Anda perlu memutakhirkan WSL ke WSL 2. Untuk ini, Windows 10 harus diperbarui ke versi 2004. Teknologi Virtualisasi Intel harus diaktifkan di BIOS. Sekali lagi, gunakan PowerShell dengan hak administratif dan jalankan perintah berikut: 



dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart


Untuk menyelesaikan instalasi WSL dan meningkatkan ke WSL 2, restart komputer. Selanjutnya, Anda perlu menginstal WSL 2 sebagai versi standar WSL ketika menginstal distribusi Linux baru. Untuk melakukan ini, sama saja, menggunakan PowerShell dengan hak administrator, kami menjalankan perintah ini: 



wsl --set-default-version 2


Setelah Anda menjalankan perintah ini, pesan berikut mungkin muncul: 



WSL 2 requires an update to its kernel component. For information please visit https://aka.ms/wsl2kernel








Pesan ditampilkan ketika mencoba menginstal WSL 2 sebagai subsistem Windows default untuk Linux



Ikuti tautan dan instal file .msi yang sesuai, terima kasih untuk mana kernel Linux untuk WSL 2 akan diinstal pada mesin Anda. Setelah kernel diinstal, jalankan perintah di atas lagi. Sekarang seharusnya berhasil diselesaikan tanpa memberikan pesan serupa.



Sekarang yang tersisa adalah menginstal distribusi Linux yang diperlukan. Untuk melakukan ini, buka Microsoft Store dan cari Ubuntu 20,04 LTS di sana . Setelah menginstal distribusi, jalan pintas akan muncul pada menu Mulai untuk memulai Ubuntu. Mulai sistem dan ikuti instruksi untuk menyelesaikan instalasi (secara umum, penyelesaian instalasi adalah membuat pengguna baru).



Untuk memeriksa apakah distribusi benar-benar diinstal pada WSL 2, jalankan perintah berikut: 



wsl --list --verbose


Jika ternyata Anda menggunakan WSL 1, maka Anda dapat beralih ke WSL 2 menggunakan perintah ini: 



wsl --set-version <distribution name> <versionNumber>


Itu saja. Anda sekarang memiliki distribusi Ubuntu lengkap yang berjalan pada Windows 10.



Menyiapkan lingkungan kerja untuk seorang programmer



Sekarang Anda telah menjalankan Ubuntu yang Anda inginkan, Anda dapat menginstal semua yang Anda butuhkan. Misalnya, jika Anda seorang ilmuwan data, Anda dapat menginstal distribusi Anaconda terbaru. Jika Anda adalah pengembang front-end, maka Anda, misalnya, mungkin tertarik pada Angular, npm, dan banyak lagi. Di sini saya ingin fokus pada dua alat. Ini adalah Visual Studio Code dan sekelompok Docker + Kubernetes.

VS Code adalah editor kode yang disukai banyak pengembang. Salah satu kekuatan editor ini adalah dukungannya untuk jumlah ekstensi yang tak terbatas. Dan sekarang kita telah memasukkan WSL 2, Remote Development dapat disebut ekstensi yang mutlak diperlukan untuk VS Code .



Ekstensi ini memungkinkan Anda untuk bekerja dari jarak jauh pada kode yang tersedia di lingkungan yang dibuat oleh WSL 2, dalam sebuah wadah, atau bahkan pada mesin virtual jarak jauh yang diakses melalui SSH. Ekstensi ini memungkinkan, misalnya, membuat proyek di Linux yang menjalankan WSL 2 dan menggunakan editor VS Code yang diinstal pada Windows 10 untuk mengerjakan proyek ini.



Faktanya, VS Code berbeda dalam peluang besar. Ini adalah sistem intelijen IntelliSense, dan integrasi dengan git, dan debugger, dan terminal, dan banyak lagi. Saya percaya bahwa Anda akan berhasil mengonfigurasi VS Code dan segera Anda akan dapat membuat apa yang Anda butuhkan dari editor ini.

Rencana Pengembangan WSL



WSL 2 masih kekurangan beberapa fitur, tetapi subsistem Windows untuk Linux berkembang ke arah yang benar. Dalam waktu dekat, Anda dapat menginstal WSL dengan satu perintah. Untuk melakukan ini, cukup membuka terminal Windows dan melakukan hal berikut di dalamnya: 



wsl.exe --install


Selain itu, WSL 2 akan menjadi versi subsistem yang, ketika Anda pertama kali menginstal WSL, diinstal secara default.



Perlu dicatat bahwa pengembang menantikan fitur-fitur berikut dari WSL: dukungan untuk kartu grafis dan dukungan untuk aplikasi dengan antarmuka grafis.



Sejak rilis WSL 1, tim pengembangan subsistem ini paling sering diminta untuk mengimplementasikan dukungan CUDA atau GPU Compute di WSL. Dalam kode terbaru, tim pengembangan WSL, sistem virtualisasi, DirectX, Windows Driver sedang mengerjakan fitur ini. Oleh karena itu - tetap disini.









Melatih model yang menggunakan teknologi pembelajaran mendalam di WSL 2 (menggunakan CUDA)



Selain itu, tentang cara dan dukungan untuk aplikasi grafis untuk Linux. Misalnya, pada sistem Linux yang diinstal pada WSL, Anda dapat menggunakan editor teks atau IDE. Pada mesin Windows, Anda bahkan dapat mengembangkan aplikasi GUI untuk Linux!

Pada pembuangan programmer seperti itu adalah kinerja tinggi dan banyak fitur yang bermanfaat, satu set yang, dari waktu ke waktu, akan diisi ulang.
