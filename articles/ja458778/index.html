Satellite 6.5 Reporting Engine：何を、そしてなぜ

Red Hat Satelliteは、物理環境、仮想環境、およびクラウド環境でのRed Hatインフラストラクチャの導入、スケーリング、および管理を容易にするシステム管理ソリューションです。サテライトを使用すると、ユーザーはシステムを調整および更新して、さまざまな標準に従って効率的で安全な運用を確保できます。システムのパフォーマンスの維持に関連するほとんどのタスクを自動化することにより、Satelliteは組織の効率の向上、運用コストの削減、戦略的ビジネスリクエストへの対応を容易にします。



Red Hat Enterprise Linuxサブスクリプションに含まれているRed Hatサービスを使用して基本的な管理タスクを実行できますが、Red Hat Satelliteは広範なライフサイクル管理機能を追加します。

これらの機能には次のものがあります。

  • パッチのインストール。
  • サブスクリプション管理;
  • 初期化;
  • 構成管理。

単一のコンソールから、数千のシステムを単一のシステムと同じくらい簡単に管理できるため、可用性、信頼性が向上し、システム監査機能が提供されます。

そして今、私たちは新しいRed Hat Satellite 6.5を手に入れました！

Red Hat Satellite 6.5の優れた点の1つは、新しいレポートエンジンです。

多くの場合、Satellite ServerはRed Hatの企業システムに関するすべての情報が流れる中心として機能します。この最新のエンジンにより、Satelliteクライアントホスト、ソフトウェアサブスクリプション、および該当するソフトウェアの欠陥（該当するエラッタ）に関する情報を含むレポートを作成およびエクスポートできます。等 レポートは、組み込みRuby（ERB）言語でプログラムされています。

Satellite 6.5には既製のレポートが付属しており、エンジンにより、ユーザーはこれらのレポートをカスタマイズしたり、独自のレポートを作成したりできます。Satellite 6.5の埋め込みレポートはCSV形式で生成されますが、この投稿では、HTML形式でレポートを作成する方法を示します。

Satellite 6.5の組み込みレポート


Satellite 6.5には、4つの組み込みレポートが含まれています。

  • 該当する正誤表 -コンテンツホストで修正されるソフトウェアの欠陥（正誤表）のリスト（オプションでホストまたは欠陥によってフィルター処理されます）;
  • ホストのステータス -サテライトホストのステータスに関するレポート（オプションでホストによってフィルタリングされます）;
  • 登録済みホスト -サテライトホストに関する情報：IPアドレス、OSバージョン、ソフトウェアサブスクリプション（オプションでホストでフィルター）。
  • サブスクリプション -ソフトウェアのサブスクリプションに関する情報：サブスクリプションの総数、無料サブスクリプションの数、SKUコード（オプションでサブスクリプションパラメーターでフィルター処理されます）。

レポートを生成するには、[ 監視 ]メニューを開き、[ レポートテンプレート ] を選択して、目的のレポートの右側にある[生成]ボタンをクリックします。フィルターフィールドを空白のままにしてレポートにすべてのデータを含めるか、そこに何かを入力して結果を制限します。たとえば、登録済みホストレポートにRHEL 8ホストのみを表示する必要がある場合は、次のスクリーンショットに示すように、フィルターos = RedHatおよびos_major = 8を指定します。



レポートを生成した後、それをダウンロードしてLibreOffice Calcタイプのスプレッドシートで開くことができます。これにより、CSVからデータがインポートされ、列に配置されます。たとえば、下のスクリーンショットの該当するエラッタレポートとして表示されます。



埋め込みレポートのプロパティでデフォルトオプションが有効になっているため、サテライトで作成したすべての新しい組織と場所に自動的に追加されることに注意してください。

埋め込みレポートのカスタマイズ


組み込みのサブスクリプションレポートを例として使用して、カスタマイズを分析しましょう。デフォルトでは、このレポートにはサブスクリプションの総数（1）と、利用可能なサブスクリプションの数（無料サブスクリプション）（2）が反映されます。 （1）-（2）として定義されている、使用されたサブスクリプションの数を含む別の列を追加します。たとえば、合計で50のRHELサブスクリプションがあり、そのうちの10が無料の場合、40のサブスクリプションが使用されます。

埋め込みレポートの編集はブロックされており、変更することは推奨されていないため、埋め込みレポートのクローンを作成して新しい名前を付け、このクローンコピーを変更する必要があります。

そのため、サブスクリプションレポートを変更する場合は、まずそれを複製する必要があります。したがって、[ 監視 ]メニューを開き、レポートテンプレートをクリックし、サブスクリプションテンプレートの右側にあるドロップダウンメニューで[ クローン ]を選択します。次に、クローンレポートの名前を入力し（カスタムサブスクリプションと呼びましょう）、Available行とQuantity行の間に「Used」という行を追加します。pool.quantity-pool.available、 -行の終わりにあるコンマに注意してください。スクリーンショットは次のようになります。



次に、[ 送信 ]ボタンをクリックすると、[ レポートテンプレート]ページに戻ります。そこで、先ほど作成したカスタムサブスクリプションレポートの右側にある[ 生成 ]ボタンをクリックします。[サブスクリプションフィルター]フィールドを空白のままにして、[ 送信 ]をクリックします。その後、レポートが作成されてロードされます。このレポートには、Used列が追加されています。



組み込みRuby言語のヘルプは、レポート編集ウィンドウの[ ヘルプ ]タブにあります。構文の概要、および使用可能な変数とメソッドを提供します。

独自のレポートを作成する


次に、Satelliteのホストに割り当てられたAnsible-Rolesレポートの例を使用して、独自のレポートを作成する方法を見てみましょう。 [ 監視 ]メニューを開き、[ レポートテンプレート ] をクリックして、[ テンプレートの作成 ]ボタンをクリックします。レポートAnsible Roles Reportを呼び出して、次のERBコードを挿入します。

<%#
name: Ansible Roles Report
snippet: false
template_inputs:
- name: hosts
 required: false
 input_type: user
 description: Limit the report only on hosts found by this search query. Keep empty
   for report on all available hosts.
 advanced: false
model: ReportTemplate
-%>
<% load_hosts(search: input('hosts'), includes: :ansible_roles).each_record do |host| -%>
<%   report_row({
       'Name': host.name,
       'All Ansible Roles': host.all_ansible_roles
     }) -%>
<% end -%>
<%= report_render -%>

このコードは、ホストごとに「all_ansible_roles」属性を表示して、ホストごとにレポートを生成します。

次に、[ 入力 ]タブに移動し、[ +入力を追加 ]ボタンをクリックします。名前はhostsで、説明のタイプはFilter by hosts（optional）です。次に[ 送信 ] をクリックし、作成したレポートの右にある[ 生成 ]ボタンをクリックします。次に、ホストフィルターを設定するか、[すぐに送信 ]をクリックしてすべてのホストのレポートを生成できます。生成されたレポートは、LibreOffice Calcでは次のようになります。



HTMLレポート


Satellite Reporting Engineを使用すると、CSV形式だけでなくレポートを生成できます。例として、統合されたホストステータスレポートに基づいて独自のレポートを作成しますが、ステータスベースのセルが色分けされたHTMLテーブルの形式でのみ作成します。これを行うには、ホストステータスを複製し、そのERBコードを次のコードに置き換えます。

<!DOCTYPE html>
<html>
<head>
   <title>Host Statuses</title>
   <style>
       th {
           background-color: black;
           color: white;
       }
       td.green {
           background-color:#92d400;
           color:black;
       }
       td.yellow {
           background-color:#f0ab00;
           color:black;
       }
       td.red {
           background-color:#CC0000;
           color:black;
       }
       table,th,td {
               border-collapse:collapse;
               border: 1px solid black;
       }
   </style> 
</head>
<body>
<table>
<tr> 
       <th> Hostname </th>
       <th> Status </th> 
<% load_hosts(search: input('hosts'), includes: :host_statuses).each_record do |host| -%>
   <% all_host_statuses_hash(host).each do |key, value|  -%>
       <th> <%= key %> </th>
   <% end -%>
   <% break -%>
<% end -%>
</tr>

<%- load_hosts(search: input('hosts'), includes: :host_statuses).each_record do |host| -%>
   <tr> 
   <td> <%= host.name   %> </td> 
   <% if host.global_status == 0 -%>
       <td class="green"> OK </td>
   <% elsif host.global_status == 1 -%>
       <td class="yellow"> Warning </td>
   <% else -%>
       <td class="red"> Error (<%= host.global_status %>) </td>
   <% end -%>

   <% all_host_statuses_hash(host).each do |key, value|  -%>
       <% if value == 0 -%>
           <td class="green"> OK </td>
       <% elsif value == 1  -%>
           <td class="yellow"> Warning </td>
       <% else -%>
           <td class="red"> Error (<%= value %>) </td>
       <% end -%>
   <% end -%>
   </tr>
<% end -%>

</table>
</body>
</html>

コマンドラインからレポートを実行する


hammer コマンドは、コマンドラインからレポートを実行するために使用され、cronユーティリティはこのプロセスを自動化します。

ハンマーのreport-template generate —name ""コマンドを使用します。次に例を示します。

# hammer report-template generate —name "Host statuses HTML"

レポートの内容がコンソールに表示されます。情報をファイルにリダイレクトしてから、シェルスクリプトを実行してレポートを生成し、電子メールで送信するようにcronを設定できます。 HTML形式は電子メールクライアントで完全に表示されます。これにより、関係者へのレポートの定期的な配信を、認識に便利な形式で整理できます。

したがって、Satellite 6.5のレポートエンジンは、企業がSatelliteに保持している重要なデータをエクスポートするための強力なツールです。非常に柔軟性があり、組み込みレポートとその改訂バージョンの両方を使用できます。さらに、ユーザーは独自のレポートを最初から作成できます。サテライトレポートエンジンの詳細については、YouTubeビデオをご覧ください。
