Webstorm NodeJs Interpreter aus Docker-Container

Dieser Artikel beschreibt den Prozess der Bereitstellung von NodeJs- Anwendungen mithilfe des NodeJs-Interpreters im Docker- Container als Projekt (standardmäßig). Die Lösung wird höchstwahrscheinlich nur für Benutzer des Linux-Betriebssystems relevant sein, da der Interpreter am Ende dem offiziellen Node-Image auf DockerHub entnommen wird .


Wozu?


Die Anwendung kann nur mit Docker auf dem Hostcomputer bereitgestellt werden. Der Interpreter muss nicht selbst installiert werden, sodass der Entwickler eine bestimmte Version des Interpreters für ein bestimmtes Projekt haben kann. Wenn die Version des Abbilds im Projekt geändert wird, muss der Entwickler nicht manuell konfigurieren, sondern es wird eine neue Instanz des Interpreters verwendet automatisch projizieren.


Entscheidung


Beispielsweise wurde eine Anwendung auf ReactJs + Typescipt erstellt, da bei falscher Konfiguration des Interpreters zumindest die Code-Hervorhebung unterbrochen wird.


1. Erstellen Sie eine Anwendung aus der fertigen Vorlage:


$ npx create-react-app reactjs-typescript-custom-nodejs --template typescript

2. Schreiben Sie eine Docker-Datei:


FROM node:14
WORKDIR /app

CMD cp /usr/local/bin/node /app/docker/node \
    && npm install \
    && npm run start
