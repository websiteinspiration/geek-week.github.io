Swift 5.3：新機能

こんにちはHabr！Swiftプログラミング言語は、iOS用のアプリケーションの記述に使用されるため非常に人気があり、その開発はこの領域に関係するすべての人に関心があります。

Swiftバージョン5.3開発者向けの新機能を見てみましょう。

複数の末尾クロージャー


新しいバージョン（SE-0279）は、複数のクロージャーをより読みやすい方法で関数パラメーターとして呼び出すことができる有限クロージャーの新しい構文を提供します。これにより、関数シグネチャでの括弧の使用が最小限に抑えられます。

import Foundation

func load(url: String, success: (Data) -> Void, failure: (Error) -> Void) {
    
}

// old  version

load(url: "someURL", success: { data in }, failure: { error in })

// new multiple trailing closures

load(url: "someURL") { (data) in
    // do smth
} failure: { (error) in
    // do smth
}

2つの呼び出しオプションの例では、視覚的に2番目のオプションが少し見栄えが良いことがわかります。

マルチパターンキャッチ句


現在、do-catchステートメントの各catch句に含めることができるパターンは1つだけです。この問題を回避するために、開発者はスイッチオプションの機能を使用して、catchステートメントの本文にパターンマッチングを含めることにより、ネストされたコードと複製されたコード（SE-0276）を増やします。

キャッチアイテムを使用すると、スイッチのように変数をキャッチボディにバインドする機能を使用して、コンマ区切りのテンプレートのリストを指定できます。

enum NetworkError: Error {
    case failure, timeout
}

// old  version
func networkCall(){
  do{
    try someNetworkCall()
  } catch NetworkError.timeout{
    print("timeout")
  } catch NetworkError.failure{
    print("failure")
  }
}

// new multi-Pattern catch clauses
func networkCall(){
  do{
    try someNetworkCall()
  } catch NetworkError.failure, NetworkError.timeout {
    print("handle for both")
  }
}

Enumの合成された同等の適合性


これまで、列挙型の2つの要素を比較することは「楽しい」ことではありませんでした。元の値が別の値より小さいかどうかを判断するには、Comparableに準拠し、静的関数を記述する必要がありました。

現在、Swift（SE-0266）では、これらすべての実装について「汗をかく」ことはできません。列挙型が同じ型である限り、プロトコルを明示的に実装する必要はありません。対応する値が指定されていない場合、リストは宣言の意味順に比較されます。

enum Brightness: Comparable {
    case low(Int)
    case medium
    case high
}

([.high, .low(1), .medium, .low(0)] as [Brightness]).sorted()
// [Brightness.low(0), Brightness.low(1), Brightness.medium, Brightness.high]

自分の数を減らすことができます


今後は、不要になった場所で自己を省略できます（SE-0269）。以前は、外部スコープから値をキャプチャするときに、クロージャでselfを使用する必要がありました。

struct OldView: View {

    var body: some View {
        Button(action: {
            self.sayHello()
        }) {
            Text("Press")
        }
    }
    
    func sayHello() {}
}

struct NewView: View {

    var body: some View {
        Button {
            sayHello()
        } label: {
            Text("Press")
        }
    }
    
    func sayHello(){}
}

ジェネリックのどこに


言語の新しいバージョン（SE-0267）では、ジェネリック型と拡張機能内の関数のどこに接続する機能が導入されました。

struct Stack<Element> {
    private var array = [Element]()

    mutating func push(_ obj: Element) {
        array.append(obj)
    }

    mutating func pop(_ obj: Element) -> Element? {
        array.popLast()
    }
}

extension Stack {
    func sorted() -> [Element] where Element: Comparable {
        array.sorted()
    }
}

スタックに新しいsorted（）メソッドを追加できますが、スタック内の要素がComparableに準拠している場合のみです。

新しいタイプのFloat16


Float16半精度浮動小数点（SE-0277）が導入されました。近年のモバイルデバイスでの機械学習の出現により、これはこのトピックを宣伝するというAppleの野心を示しているにすぎません。Float 16は、モバイルGPUでコンピューティングに、またMLアプリケーションのスケールの圧縮形式として一般的に使用されます。

let f16: Float16 = 7.29

Swift 5.3では、Swift Package Manager（SPM）に多くの拡張機能が導入されています。それらを考慮してください。

  1. SE-0271（パッケージマネージャーリソース）を使用すると、SPMに画像、オーディオ、JSONなどのリソースを含めることができます。
  2. SE-0272（Package Manager Binary Dependencies）により、SPMは既存のソースパッケージサポートとともにバイナリパッケージを使用できます。つまり、Firebaseなどの共有のクローズドソースSDKをSPMを使用して統合できるようになりました。
  3. SE-0273（Package Managerの条件付きターゲット依存関係）を使用すると、特定のプラットフォームと構成に対してのみ依存関係を持つようにターゲットを構成できます。たとえば、Linux用にコンパイルするときに、特別な追加フレームワークを指定できます。
  4. SE-0278では、ローカライズされたリソースを追加できます。

したがって、バージョン5.3は、開発者によって長期的に採用される優れたイノベーションをいくつか導入したと言えます。
