WindowsサブシステムLinux（WSL）へのDockerのインストールと構成

前提条件


始める前に、システムがおおよそ次の構成になっていることを確認してください。

  • Windows 10バージョン1803ビルド1734以降
  • Ubuntu for WSL 16.0.4 LTSまたはそれに似たもの（バージョンは若干異なる場合があります）

Windowsのバージョンとビルドがここに示されているものよりも低くないことが重要です。実際、これらの数値から始めて、cgroup（コントロールグループ-2008年にLinuxカーネルに追加）を使用できるようにするWindows WSLカーネルに根本的な変更が加えられました。また、Dockerがコンテナー内のシステムのリソースを管理するために必要です。



Docker-CE 09.17.0をインストール


Docker Community Edition 17.09.0をインストールします（これを書いている時点では、WSLにインストールすると、より新しいバージョンが落ちています）。

最初に行うのは、以前のDockerインストール（存在する場合）をすべて削除することです。これを行うには、bashターミナルを実行し、次のように記述します。

$ sudo apt-get remove docker docker-engine docker.io containerd runc


次に、Docker CEのリポジトリをインストールします。apt-getを使用すると、自分でソースからビルドすることなく、必要なバージョンをインストールできます。Dockerの公式Webサイトの推奨事項に従って、次のコマンドを記述します。

#   
$ sudo apt-get update
#   ,   apt    HTTPS
$ sudo apt-get install apt-transport-https ca-certificates curl gnupg2 software-properties-common

#      GPG   Docker
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
#  Docker   :
$ sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"

#        Docker    
$ sudo apt-get update

aptを使用して新しいリポジトリを追加したことがある場合、これらすべては全体としておなじみです。そうでない場合は、私と一緒にいることを願っています:)

次に、Dockerをインストールします。

$ sudo apt-get install docker-ce=17.09.0~ce-0~ubuntu

UbuntuスタイルのWSLを使用していない場合は、次のように使用可能なバージョンのリストを表示できます。

$ apt list -a docker-ce


...またはAPTを放棄した場合は、他のライブラリから同様のコマンドを使用します。

最後に、Docker Engineを正しく使用できるようにするために、現在のユーザーをdockerグループに追加する必要があります。ルートとして。

sudo usermod -aG docker $USER

WindowsでのDockerの実行


Dockerがインストールされたので、Windowsを起動するたびにWSLでサービスとして開始するために、かなりトリッキーな方法を使用する必要があります。これは見かけほど簡単ではありません。スーパーユーザーに代わって行動し、昇格した特権でコマンドシェルを実行する必要があるため、タスクが少し複雑になります。

プロセスを少し単純化するには、/ usr / local / sbin /に新しいファイルを作成します-Dockerサービスを開始するための適切なコマンドを含むスクリプト：

$ sudo nano /usr/local/sbin/start_docker.sh

次の内容で：

#!/usr/bin/env bash
sudo cgroups-mount
sudo service docker start

最初のsudoコマンドはcgroupをマウントすることで（リブート時に実行されます）、2番目はDockerサービスをsystemdに提供します（これはサービスを初期化および管理するためのUbuntuサブシステムです）。

スクリプトを実行して実行します。

$ sudo chmod +x /usr/local/sbin/start_docker.sh
#  
$ sudo chmod 755 /usr/local/sbin/start_docker.sh
$ /bin/sh /usr/local/sbin/start_docker.sh

ただし、Dockerサービスは次の2つの理由で開始されません。

  1. スクリプトはスーパーユーザーとして呼び出しを行うため、意図したとおりに機能させるには、スクリプト自体をスーパーユーザー権限で実行する必要があります。理論的には、これはsudoコマンドを書くのと同じくらい簡単ですが、これを行うには、ユーザーはダウンロードのたびにデータを入力する必要があり、非常に面倒です。
  2. cgroupをマウントする最初のコマンドは、昇格した特権でbashで実行する必要があります。ユーザー入力なしで実行するには、Windowsタスクスケジューラを使用する必要があります。

すべてクリア？それから要点に...

ユーザー入力なしでroot権限でスクリプトを実行する


LinuxファミリーのOSでは、/ etc / sudoersファイルによって、スーパーユーザー権限で実行できるユーザーと実行可能なユーザーが決まります。これを変更して、ユーザーがrootパスワードを常に入力しなくてもスクリプトを呼び出せるようにします。（しかし、このファイルには細心の注意を払ってください。自分が何をしているかわからない場合は、システムから簡単に切り離されてしまいます。）

$ sudo nano /etc/sudoers

ファイルの最後に新しい行を追加します。この場合、ユーザー名を正確に知る必要があります（不明な場合は、echo $ USERを実行してください）。

<your username here> ALL=(ALL:ALL) NOPASSWD: /bin/sh /usr/local/sbin/start_docker.sh

これで、ユーザー入力なしでrootとしてDockerサービスを開始できます。

$ sudo /bin/sh /usr/local/sbin/start_docker.sh

おっと、同志！

Windowsブートでの適切なDocker起動


これで、システムの起動時に、ユーザーの操作を必要としない別のコマンドを使用して、管理者としてDockerを起動できるようになります。



Windows タスクスケジューラを起動し、左側のペインでタスクスケジューラライブラリを選択します。次に、右側のペインで「タスクの作成」を選択します。



次に、タスクの基本設定を編集するように求められます。タスクに明確な名前を付け、残りの設定が私のものと一致することを確認してください。 [ 最高の特権で実行]を選択することが重要です。



[ トリガー ]タブで、ローカルユーザーがログインするとすぐにタスクを開始する新しいトリガーを追加します。



[ アクション ]タブに切り替え、bashを実行する新しいアクションを作成し、-cスイッチを使用してスクリプトをすぐに実行します。

-c "sudo /bin/sh /usr/local/sbin/start_docker.sh"


これは管理者特権で実行する必要があるため、[ 全般 ]タブの[ 最高の特権で実行 ] チェックボックスをオンにしてください。



残りのパラメーターは主に個々の設定に依存しますが、設定を表示します。必要に応じて、それらをコピーできます。停車中の車では、一部の電源設定が表示されない場合があります。個人的にはノートパソコンを使っています。



まあ、すべては私たちが意図したとおりに行きます。昇格された特権を使用して前の段階でDockerを起動しなかった場合は、タスクスケジューラで作成したタスクを右クリックし、[実行]を選択してください！それで全部です！

すべてが機能することを確認する


安全にプレイしたい場合は、マシンを再起動してください（そうするためには、しばらくの間ブラウザのタブを閉じる必要があります）、bashを起動して次のように入力します。

$ docker run --rm hello-world

問題がなければ、Dockerの標準の挨拶が表示されます。サービスはWSLで自動的かつ直接開始されます！おめでとう！

四捨五入


WSLにDockerをインストールして開始しました。つまり、コンテナーは、システムで既に実行されているUbuntuを共有できるようになります。したがって、通常のDocker for Windowsのように、新しい（仮想化された）OSをデプロイする必要はありません。

これまでのところ（元の記事の執筆時点で）WSLでのDockerのパフォーマンスは低いですが、プラットフォームが開発され、Windowsの更新プログラムがリリースされると、改善されるだけです。
