システムでのユーザーの登録と承認にRutokenテクノロジーを使用した経験（パート2）

こんにちは！私たちはこのトピックを扱い続けます（前の部分はここにあります）。

今日は実用的な部分に移ります。完全なopenSSL暗号化オープンソースライブラリに基づいて認証局を設定することから始めましょう。このアルゴリズムはWindows 7

を使用してテストされました。openSSLをインストールすると、コマンドラインからさまざまな暗号化操作（鍵や証明書の作成など）を実行できます。

アクションのアルゴリズムは次のとおりです。

  1. インストール配布パッケージopenssl-1.1.1gをダウンロードします。
    OpenSSLにはさまざまなバージョンがあります。Rutokenのドキュメントによると、openSSLバージョン1.1.0以降が必要です。openssl-1.1.1gバージョンを使用しました。openSSLは公式サイトからダウンロードできますが、簡単にインストールするには、ネットワーク上のWindows用のインストールファイルを見つける必要があります。私はあなたの ためにそれをしました：slproweb.com/products/Win32OpenSSL.html
    ページを下にスクロールして、Win64 OpenSSL v1.1.1g EXE 63MBインストーラーをダウンロードする必要があります。
  2. openssl-1.1.1gをコンピューターにインストールします。
    インストールは、C：\ Program Filesフォルダーに自動的に指定される標準パスに従って実行する必要があります。プログラムはOpenSSL-Win64フォルダーにインストールされます。
  3. 必要に応じてopenSSLを設定するために、openssl.cfgファイルがあります。前の段落で説明したようにopenSSLをインストールした場合、このファイルはパスC：\ Program Files \ OpenSSL-Win64 \ binにあります。openssl.cfgが格納されているフォルダーに移動し、メモ帳++などを使用してこのファイルを開きます。
  4. おそらく、openssl.cfgファイルの内容を変更することにより、なんとかして認証センターがセットアップされると推測していましたが、間違いはありません。これを行うには、[ca]コマンドを構成します。openssl.cfgファイルでは、変更を加えるテキストの先頭は次のように見つかります：[ca]。
  5. 次に、設定の例とその説明を示します。

    [ ca ]
default_ca	= CA_default		

 [ CA_default ]
dir		= /Users/username/bin/openSSLca/demoCA

certificate	= $dir/ca.crt 	
serial		= $dir/private/serial 		
crlnumber	= $dir/crlnumber	
					
crl		= $dir/crl.pem 		
private_key	= $dir/private/ca.key
x509_extensions	= usr_cert
