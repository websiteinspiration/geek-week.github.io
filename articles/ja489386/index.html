かつて、別のコードクエストのユニット用にRubyチームを作成するように頼まれました、つまり、私はTimlid（地獄、はい!!!）になり、さらに重要なことに、すべての人事部から新しい開発者を募集しようとする人（hmm ...）になります。明らかに、これらの開発者のほとんどは平均的な知識を持つ初心者でした。それぞれの面接に近づくと、私自身もテーブルの向こう側に座っている初心者だったので、辛抱強く、親切に、正直で役立つフィードバックを提供するように心がけています。適切ではありません。とにかく、私はいくつかのパターンを明らかにしました-多くの開発者、特に初心者は、間違いを犯すことで開始を難しくするだけでなく、誰かがこれについて以前に知らせていれば簡単に回避できます。私は自分の時間のほとんどの間違いを犯し、誰かがそれについて警告していた場合、それはあったかもしれませんが、私は耳を傾けませんでした。ソフトウェア開発の権威ある方向で開発者として効果的に開発したい場合は、初心者（または中級者）の方が効果的であることを指摘しておきます。

GitHub


コミュニティの誰もが、GitHubリポジトリが将来の雇用者にとって本当に重要なものであることを知っています。そうではありませんか？人々が使用したり、貢献したりしたオープンソースライブラリを作成した場合、そうです、それは非常に貴重な切り札になります。アプリケーションを作成した場合、それがあなたが持っている日常の問題のいくつかを解決するトレーニングアプリケーションであっても、それはまたあなたのスキルを示すことができます。ただし、表示できるすべてのリポジトリが、過去9か月間に触れられていない他社の半分完了したタスク、または最初の「新しいレール」である超基本的なRailsである場合、これを実際に見てほしいですか？おそらく、あなたは今9か月前よりもずっと良くなっているでしょう、いまいましい、おそらくそれは、確かではありませんが、コードの墓地へのリンクを送信すると、よくわかりますが、彼らが言うように、良い第一印象を与えるチャンスは1回だけですよね？

そしてこれはジュニアのものだけに適用されません-私のGitHubリポジトリを見れば、どこでもまったく同じものを見つけたでしょう-過去数年または私がプロセスで研究した核物質に関するいくつかの放棄された実験からの募集割り当てなど。私はGitHubで毎日コミットしていますが、これらのコミットはプライベートリポジトリであり、確認するものはありません。GitHubで何もスポーンしない場合、私が言いたいことはすべて正常ですが、それを表示しないでください。

ルビー


私たちはみんなRailsが大好きです！そして、私たちは彼らを憎んでいます。そして、時には私たちは同時に愛し、憎しみます。あるいは、ある日彼を愛し、翌日私たちは彼を憎んで呪いました。ただし、RubyでのWebアプリケーションの作成に関しては、Railsが事実上の標準であることに誰もが同意します。このビジネスで働くには多くの献身と努力を要し、少しも触れないでください（そうしたと主張する）。したがって、RubyのWeb開発者になることを決定したときに、Railsを学ぶのは当然のことです。そして、Rubyを学ばずにRailsを学ぶのは良くない！

これにより進捗が速くなると思うかもしれません-結局のところ、基本的なRailsアプリケーションの設定は非常に簡単です-しかしすぐに、なぜそれが機能するのか本当に理解できず、機能することがわかるだけですが、これが最初の単純な間違いです。あなたはすでにスタックオーバーフローを利用しており、同様の問題を抱えている人を探しています。

このように考えてください。Google翻訳とお気に入りの映画の引用だけを使用して、外国語で本を書こうとはしないでしょう。アプリケーションを作成する場合も同様です。文字どおりに意味のあるものを作成し、言語をよく理解すればするほど、意図した意味をよりよく伝えることができ、文章を作成する際の効果が高まります。 Railsだけを知っている場合、語彙は限られます。そして、「限定」を必要とする企業で働きたくないと断言できます。すぐに、基本的なCRUD機能のみを実装するロボットに変わります。たとえば、2か月間は刺激的ですが、すぐに退屈、歯を磨くように、あなたはそれをしなければなりません、あなたはこれをしなければなりませんが、それは本当ではありません、何にしますか？

私は各インタビューを非常にシンプルで基本的な質問から始めます：

-Rubyのクラスとは何か、モジュールとは何か、それらの違いは何で、それらは何のためのものですか？

この質問の答えを知らない人がどれほどいるのか、あなたは驚くでしょう！そして、これらはソフトウェア開発者になりたい人々です。すべてのユーザーが数分で投稿とコメントのあるブログを作成できます-それだけです。中間開発者のポストには、チームリーダーとして1年間クライアントアプリケーションに取り組んだと主張する候補者が1人いました。そのため、彼はまだRubyモジュールが何であるかを知りません。

そのようにしないでください。 Rubyを学ぶ。これには多くのリソースがあります。私が知っている最高の1つはRubyリファレンスです-Ruby online-公式ドキュメントに言及しているが、読むのがより便利で理解しやすい本。それを読んで、学び、あなたのプロ意識がどのように成長しているかを感じてください。
