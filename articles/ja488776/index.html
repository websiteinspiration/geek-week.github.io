Visual Studio 2019 for Macのアクセシビリティの強化



Visual Studio 2019 for Macバージョン8.4.4のリリースには、アイコン、警告、エラーメッセージのカラーパレットに対する多数の変更が含まれています。新しい外観は印象的で、新しい配色はさらに目立ちます。この記事では、これらの変更が本当に重要である理由と、何が変更されたのかを説明します。

現在、10億人を超える人々が何らかの形で障害を抱えています。人々はほぼ毎日さまざまな問題に対処しなければなりません。これらには、運動制限、認知障害および神経障害、言語および聴覚の問題が含まれます。これに関して、最新バージョンのVisual Studio for Macにおける視覚的なアクセシビリティの改善についてお話ししたいと思います。

世界保健機関によると、現在約2億人が何らかの形の視覚障害を抱えて生活しています。私たちの目標は、Visual Studio for Macを誰でもアクセスできるようにすることです。これを行うには、視覚に問題があるかどうかに関係なく、各ユーザーに最適なユーザーインターフェイスを提供する必要があります。低視力、色または完全な失明、白内障を持つ多くのユーザーがいます。更新されたインターフェースは、プログラムでより完全に機能するのに役立ちます。結局のところ、太陽フレアのような一般的なものでさえ、プログラムのユーザーインターフェイスを使用するときに問題になる可能性があります。視覚障害を持つユーザーがアプリケーションとより効果的に対話できるようにする方法の1つは、カラーパレットの可用性です。

アクセシブルなユーザーインターフェイスを作成する主な方法の1つは、すべてのインタラクティブコンテンツ（主にテキストとアイコン）のコントラストしきい値を上げることです。 Macでは、背景とテキストのコントラスト比は少なくとも3：1、高コントラストモードでは少なくとも4.5：1である必要があります。後でこのトピックに戻ります。

もちろん、色の違いだけに頼って情報の深刻さを示すことはできません。たとえば、アクティブなアイコンと非アクティブなアイコンの間で状態が変化した場合などです。つまり、エラーメッセージや警告などの要素は、ステータスを伝えるために背景色だけを使用すべきではありません。さらに何かが必要です。たとえば、エラーまたは警告のはっきりとした兆候です。以前のバージョンのVisual Studio for Macでは、状態の違いがカラーでしか表示されないことがよくありました。現在は、色だけに依存するのではなく、アクティブ化、無効化、および停止されたアイコンのより明確なレンダリングを使用しています。視覚的明瞭性を高めるために、このような状況は除外しました。

ハイコントラストモード


Macでは、[ システム環境設定]の[ユーザー補助の設定]セクションにアクセスし、[ コントラストを上げる]のチェックボックスをオンにすることで、ハイコントラストモードをアクティブにできます。ハイコントラストモードでは、インターフェイス全体のカラーコントラストが上がります。コントロールは、ストロークと見やすい形状とラベルを使用し始めます。色が明るく見えるように調整され、前景と背景の明るさの違いがはるかに目立ちます。 残念ながら、デスクトップPCのすべてのアプリケーションがハイコントラストモードをサポートしているわけではありません。







もちろん、視覚的なアクセシビリティを向上させる唯一の方法は、新しい色とアイコンを使用することではありません。また、スクリーンリーダーを使用する際の利便性を確保し、ホットキーを使用できるようにしたいと考えました。まだ多くの計画があり、そのうちのいくつかは今日話し合い、残りは少し後で発表します。この記事では、新しいカラーパレットと改善されたアイコンセット、およびVisual Studio for Macのすべてのユーザーが現在利用できる新機能に焦点を当てます。

新しいカラーパレット


何年も前に作成されたVisual Studio for Macの古いカラーパレットは、特にIDEの明るいテーマで低すぎるコントラスト比を使用していました。したがって、それを修正する時が来ました。古いパレットと新しいパレット、アイコンと背景のコントラスト比を比較します。



古いパレットには2つのオプションがありました。1つはライト用、もう1つはダークIDEテーマ用です。上記のように、古いパレットには多くの問題があり、特に明るいトピックの警告アイコンの色のコントラストが問題でしたが、これは理想的ではありませんでした。白またはライトグレーの黄色は非常に見にくいです。

新しいパレットは、これらすべての問題を修正し、カラーグループの意味論的意味を改善して、よりシンプルになりました。さらに、ハイコントラストモードで作業する準備ができています。

バッジの改善


Visual Studio for Macには常に多くのアイコンがありました。この記事で説明した変更をリリースするまでに、1,142個のバッジがありました。ほとんどのアイコンは4つのバージョンで表示されました。1つは明るいテーマと暗いテーマ、もう2つは選択した状態です。

今、私たちはより多くのバッジを持っており、これのために大きな仕事が行われました。上記のように、それぞれの可用性の問題をチェックし、新しい高コントラストパレットを使用して変換、複製、再描画する必要がありました。つまり、コントラストの高い新しいアイコンを導入するだけでなく、既存のアイコンを改善しています。 Visual Studio for Macは現在13,704のアイコンファイルを使用しています。

以前は通常の状態とアクティブな状態の違いを示すために色だけが強調されていたため、一部のアイコンは再描画または修正する必要がありました。



警告とエラーの新しい色


この機会に、Visual Studio for Macが表示する警告とエラーメッセージの色を変更しました。これは、以前は明るすぎて読みにくい可能性があった軽いトピックの警告テキストで最もよく見られます。



エラー警告ポップアップの新しい色が追加され、標準モードとハイコントラストモードの外観が改善されました。



最終的に


上記の変更は、Visual Studio for Macのインターフェイスをすべての開発者にとってより簡単にすることを目的としています。これで、視覚に問題のあるユーザー向けの読みやすいアイコンだけでなく、すでにインストールされている標準アイコンセットも利用できます。これにより、ユーザーがアクセシビリティ機能にまったく頼らなくても済むように、色のコントラストが目立つようになりました。いずれにせよ、まだ多くの作業がありますが、Visual Studioは日々改善されています。

これらの変更についての意見を共有したい場合は、@ VisualStudioMacをアドレス指定して、元の記事のコメントまたはTwitterに書き込んでください。。Visual Studio for Macの使用中に問題が発生した場合は、問題の報告機能を使用して通知してください。Visual Studio開発者コミュニティサイトでの機能の提案も歓迎します。
