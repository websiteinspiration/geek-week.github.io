UptimeRobotによるモニタリング

どのサイトも監視が必要です。所有者がダウンタイムについて知りたくないリソースはありません。多くの開発者がペットプロジェクトを監視しておらず、フリーランサーがアクセシビリティチェックに煩わされることはほとんどありませんが、これは通常、いくつかの分析を接続するよりも難しくなく、多くの神経を節約できます。



サイトを監視するためのダース以上のサービスがあり、それらのほとんどは完全に支払われているか、無料版では大幅に削減されています。それらとは別に、無料で巨大な機能を提供し、ニーズに合わせて有料サブスクリプションを柔軟に構成できるUptimeRobotがあります。



ほとんどの場合、UptimeRobotはHTTPリクエストを指定されたアドレスに送信し、応答ヘッダーによって可用性を判断します（2xx-すべて正常、4xx / 5xx-他のポイントから繰り返しリクエストを30秒間起動します。回答が再び否定の場合、ダウンタイムがカウントされます）。



無料



  • リフレッシュレートが5分の50台のモニター
  • 任意のアドレスへのHTTP（S）要求
  • ページでキーワードを検索する
  • ping（ICMP要求）
  • ポートチェック
  • メール、ウェブフック、Twitter上のメッセージ、電報、Slack、その他のサービスによるアラート
  • 応答時間統計
  • 2か月間の詳細なログ
  • 統計を公開ページに公開する機能（例）
  • オープンAPI


有償



  • 1分の更新間隔で最大10,000モニター
  • SSL証明書の有効期限を追跡し、HTTPSの問題を監視する
  • さまざまなロジックアラート（間隔のインストールなど）
  • HTTPリクエストのカスタマイズ（ヘッダーとステータスを含む）
  • ハートビート監視（NATのサーバー監視）
  • SMS /通話アラート（毎月最大900）
  • 2年間で利用可能なログ


ご覧のように、サービスには購入したものよりも無料のニシュチャクが多くあり、これがおそらくUptimeRobotの人気と認知度を高めたものです。いくつかの再設計の後、彼はまだひどいインターフェースのために非難されています。







キーワード



条件付きレンダリングでコンポーネントを使用する場合（たとえば、技術作業に関するプレート）、キーワードによる検索を使用して、そのステータスを追跡できます。一般に、これには多くの優れたユースケースがあります。各リクエストの応答時間が節約されるため、アプリケーションのさまざまな部分のレンダリングを「側面から」測定できます。リクエストは世界のさまざまな部分から送信され、完全に異なる画像を提供できます。ページ上の不要なコンテンツやその逆のアラートを設定したり、目的のテキストが存在するかどうかを確認したりできます。私のお気に入り-2回のクリックでリマインダーが作成されます。これは、Habréの特定の記事が1日/週などでトップに到達したときにトリガーされ











ます。通知は問題なくインターフェースで設定されます。









API



UptimeRobotは、Webバージョンの機能を完全に複製するシンプルなXML / JSON / JSON-P APIを提供します。メソッドとパラメーターの詳細な説明はAPIページにあります。つまり、統計とログの読み取り、モニターの作成/削除/編集/リセット、ユーザーの管理を行うことができます。



面白いガイドがあり、稼働時間がUptimeRobotでKubernetesに監視の自動化、ゴー上のクライアントやPython、および他の言語、パネルやカスタムステータスページの解釈の束が。



ハートビート



直接アクセスできないサーバーを監視するために、有料版ではハートビートモニターを利用できます。これを作成すると、UptimeRobotサービスがリモートサーバーからの定期的な着信要求をリッスンする一意のリンクが生成されます。リクエストが届かなくなった場合、ダウンタイムがカウントされます。



今何ができるか



1つの無料アカウントは50のモニターに依存しており、通常は個人のサイトやプロジェクトをカバーするのに十分です。どこにでも無料の監視をまだ接続していない場合は、今がその時です。5分で、コードに干渉することなく。また、モニタリングをプロジェクトにさらに深く統合する必要がある場合は、APIを検討してください。おそらく、一括で、スタックの既製のソリューションを最初に探すことを忘れないでください。
