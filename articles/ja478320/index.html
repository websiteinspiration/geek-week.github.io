プログラマーの作業をスピードアップするのに役立つVS Codeの10の機能

プログラマーがフローの状態に陥ると、彼の思考はしばしば宇宙の速度で動きます。彼はできるだけ早くコードを記述し、自分の考えをつかみ、それらが空中で溶けたり、新しいイメージの裏通りで迷子になったりしないようにあらゆる努力をしています。

Visual Studio Code Editor（VS Code）は、登場してすぐに、プログラミングツールの世界で新星になりました。長い間、私はあえて試さなかった。この理由は、それが単なる通常のコードエディターであったということではありませんでした。問題は、私のすべての仕事がSublimeでうまく整理されていたことです。



私にとって、「仕事」と「コードを書く」という概念はまったく同じものです。そして、概して、どのエディターを常に使用するかは問題ではありません。しかし、変化は常に容易ではありません。 VS Codeで1週間働いた後、私の生産性が大幅に低下していることがわかりました。この間、私は自分がやったことよりも少ないことをしました。

これは、私が再調整する必要があり、新しいツールに慣れる必要があったために、そうでした。私に役立つコマンドを見つける必要があり、便利なキーボードショートカットを覚えて、コードをナビゲートする方法を学ぶ必要がありました。そして、それに加えて、自分の生活を楽にするために、どのエディター拡張機能をインストールする必要があるかを理解する必要がありました。

その結果、再び高いパフォーマンスを達成しました。新しいエディターへの移行を再開することはまずありません。VS Codeは、信頼性が高くリソースの少ないツールです。拡張機能開発者の強力なコミュニティが彼の周りに集まりました。おそらく、これはマイクロソフトの最高の開発の1つです。

その結果、プログラマーは、これらの構成要素のコードを記述するよりも、特定の構成要素がどのように適合し、どのように連携するかについて考えることに、より多くの時間を費やすことができます。奇妙に聞こえるかもしれませんが、特定のプロジェクトに取り組む時間が限られている場合、コードの構造とアーキテクチャについて考えることに専念できる時間が長いほど、最終的に信頼性の高いスケーラブルなものを作成できる可能性が高くなります。 。

さらに、スニペットの使用は、プログラマーが特定のフレームワークまたは特定のライブラリーをすばやく習得するのに役立ちます。ここでのポイントは、人が特定のエラーの原因を理解するためにより少ない時間を費やす必要があるということです。彼が必要とするものは、初めて実行可能になる可能性が高くなります。

2. IntelliSense


IntelliSenseはコード補完システムです。それは他の同様のシステムからそれを区別する非常にインテリジェントです。

IntelliSenseは、高い生産性を提供するVSコードのコアツールの1つと言われています。ただし、VS Codeを選択する多くの人がこの機能を使用することは、フルキャパシティーにはほど遠いものです。

コードのセクションの上にマウスを置くと、IntelliSenseはタイプに関する追加情報を表示し、それらの説明のソースに関する情報にアクセスする機会を提供します。この機能は、他の人が作成したなじみのないエンティティを操作するときに役立ちますが、IntelliSenseの主な価値は、このシステムがプログラマーのアクションを予測できることです。

コードフォーマッタは、多くの場合、特定の規則に基づいています。プログラムのテキストのデザインに一貫性を持たせることができます。たとえば、コードの位置合わせの規則、角括弧の配置、および一般に、コードの外観について話し合っています。

自動フォーマットは、プログラムのテキストのスタイルに関する論争の数を減らし、コードの外観を改善するのに役立ちますが、技術的な負債を取り除くのには役立ちません。そのため、Tech Debt Tracker拡張機能に注意を払うことをお勧めします。

6.キーボードショートカット


プログラマーがキーボードから手を離さなければならないたびに、彼は数秒を失います。これにより、プロジェクトの作業期間が長くなります。このような時間の損失は取るに足らないように見えるかもしれませんが、手の位置を変更し、キーボードからマウスに切り替えるだけで、ワークフローが中断され、人がフロー状態から外れる可能性があることに注意してください。

多くの開発者は、キーボードを使用してできる限りのことをするよう努めています。彼らがより速く働くことができるからといって。

キーボードショートカットを使用すると、ほとんどの問題をキーボードのみで解決できます。これは、メニューで必要なコマンドを検索するのに費やす時間が短縮されることを意味し、プログラマーが目標をより早く達成するのに役立ちます。

キーボードショートカットバーは、キーの組み合わせを使用して表示できますCTRL + K + S。


VS Codeのキーボードショートカットバー

まったく新しい「ホットキー」のセットを学習したくない場合は、自分でそのようなリストを作成するか、他の一般的なエディターからキーボードショートカットをインポートできます。キーボードショートカットをカスタマイズする柔軟性により、既に慣れているユーザーは、VS Codeで高い生産性をすばやく達成できます。

ここでは、VS Codeのキーボードショートカットのカスタマイズに役立つ拡張機能を見つけることができます。

7. Zenモード


外に気を取られずに仕事をしたくない人は誰ですか？ Zenモードでは、VS Codeはフルスクリーンモードになり、すべてがインターフェイスから消えて、コードの操作を妨げることがあります。エディターをこのモードにするには、特別な一連のアクションを実行する必要があります。つまり、キーボードショートカットではなく、一連のキーボードショートカットを使用します。自分の場所でZenモードがどのようにオンになっているかを正確に確認するには、キーボードショートカットパネルで単語を探しますzen。

私の場合、Zenモードをオンにするには、次のキーボードショートカットのシーケンスを使用します：最初に- CTRL + Z、その直後に- K。つまり、このモードに入るには、まずキーCTRL + Zを押してから放し、次に押して放しますK。見た目が異なる場合があります。たとえば-のようCTRL + KにZ。

Zenモードはフローの状態を維持するのに役立ち、プログラマーが潜在的な注意散漫から解放されます。

8. Git


VS Codeから直接、Gitリポジトリでコミットできます。結局のところ、VS CodeはGitを非常によくサポートしています。つまり、変更したファイルをコミット用に準備し、コミットし、変更をロールバックし、コメントを付けることができます。一般に、通常コマンドラインを使用して行われるすべてのことは、エディターを終了せずに行うことができます。ここでSource Controlは、画面の左側にある対応するボタンで開くパネルが必要になります。たとえば、コミット用に変更されたファイルを準備+するには、このパネルの上部にある記号付きのボタンを使用できます。その後、このファイルのコンテキストメニューを呼び出して、その中の追加のコマンドを見つけることができます。

これはすべてコンソールで、GitHubデスクトップアプリケーションを使用して実行できますが、エディターを終了せずにGitを操作できるため、プログラマーの生産性を向上させることができます。

ここでも、全体のポイントは流れの状態を維持することです。 Gitを使用するプログラマーは、通常の作業環境を離れません。彼は端末とエディタを切り替える必要はありません、彼は常に自分がどこにいるのかを考える必要はありません。さまざまな問題を解決するために必要なすべてが常にあなたの指先にあります。

バージョン管理システムでの作業のさまざまな側面を自動化するVSコードの拡張機能があります。たとえば、GitLens拡張機能リポジトリの詳細を取得できます。たとえば、誰がコードにどのような変更を加えたのか、どのような変更がcommitに向けて準備されており、どの変更が行われていないのかについて説明します。この拡張機能はコードに注釈を付けることができ、バージョン管理システムからの情報にアクセスできます。たとえば、コミットの時間やプロジェクトへのコミットによって行われた変更などです。GitLensは、プロジェクトのチームワークに特に役立つツールです。

9.デバッガ


VS Codeには、コードを実行してブレークポイントを設定できるデバッガーがあり、プログラムを中断する場所を示します。これにより、エディターを終了せずにプログラムをデバッグし、実行中にプログラムの内部状態を分析できます。

デバッグは、単にChrome Developer Toolsコンソールにメッセージを出力するだけではありません。デバッグ中に、プログラム内で何が起こっているかを知ることができ、その結果、さまざまな問題の原因を検出して修正できます。たとえば、プログラマーが特定の関数の正しい動作について疑いを持っている場合、彼は、デバッグ中に、ステップバイステップモードでこの関数を実行し、このプロセス中にプログラムの状態を調べることができます。

プログラムのデバッグを開始するには、デバッグパネルを開く必要があります。これを行うDebugには、プログラムウィンドウの左側にあるボタンを使用できます。


デバッグボタンとデバッグパネルデバッグ

を開始するにStart Debuggingは、デバッグパネルの上部にあるボタンをクリックします。デバッグ中は、プログラムウィンドウの下部に表示されるコンソールウィンドウを操作できます。

デフォルトでは、VS CodeはNode.jsを使用してデバッグ済みコードを実行しますが、デバッグにGoogle Chromeブラウザーデバッガーを使用できるようにすることができます。一般に、コードのデバッグは、通常のコード実行中に見つけるのが非常に難しい問題の原因を検出できる操作モードであることに注意してください。
