WSL（LinuxのWindowsサブシステム）を操作するためのクールなライフハック

私は深くWSL（Linux用のWindowsサブシステム）に浸漬していて、今という  WSL2が  で提供され  、Windowsのインサイダー、これは本当に可能な選択肢を模索するための素晴らしい時間です。WSLで見つけた非常に興味深い機能は、ワールド間でデータを「クリーンに」移動できることです。これは、本格的な仮想マシンで簡単に取得できるエクスペリエンスではなく、LinuxとWindowsの緊密な統合について述べています。

カットの下では、ピーナッツバターとチョコレートを混ぜることでさらに面白いことができます！



LinuxからWindows Explorerを起動し、配布ファイルにアクセスします


WSL / bashコマンドラインでファイルに視覚的にアクセスしたい場合は、「explorer.exe。」を実行できます。現在のディレクトリが配置されている場所にWindowsエクスプローラーウィンドウが表示され、サーバーからLinuxファイルが配信されます。 LANプラン9。



Windowsから真のLinuxコマンド（CGYWINではない）を使用する


これについては以前に書いていますが  、Windows内から実際のLinuxコマンドを使用できるようにする PowerShell関数のエイリアスがあります。

このように、WSL.exeの後に単に配置するだけで、DOS / Windows /何からでもLinuxコマンドを直接呼び出すことができます。

C:\temp> wsl ls -la | findstr "foo"
-rwxrwxrwx 1 root root     14 Sep 27 14:26 foo.bat

C:\temp> dir | wsl grep foo
09/27/2016  02:26 PM                14 foo.bat

C:\temp> wsl ls -la > out.txt

C:\temp> wsl ls -la /proc/cpuinfo
-r--r--r-- 1 root root 0 Sep 28 11:28 /proc/cpuinfo

C:\temp> wsl ls -la "/mnt/c/Program Files"
...contents of C:\Program Files...

WindowsへのパスはWindowsへの$ PATHにあるため、Windows実行可能ファイルはWSL / Linuxから呼び出され、実行できます。あなたがしなければならないすべては、最後に.exeでそれを明示的に呼び出すことです。これがExplorer.exeの仕組みです。notepad.exeまたはその他のファイルを作成することもできます。

WSL 2の利点は次のとおりです

  • 仮想マシンはリソースを集中的に使用し、非常に独立したエクスペリエンスを作成します。

Pengwin  は、非常に優れた特別なWSL Linuxディストリビューションです。Windowsストアで入手できます  。PengwinをX410 などのXサーバーと組み合わせると、非常に優れた統合システムが得られます。

Windowsシステム間でWSLディストリビューションを簡単に移動できます。


Ana Betts氏は、理想的なWSL2ディストリビューションを1台のマシンからn台のマシンに簡単に転送できるこの素晴らしい手法を指摘しています。

wsl --export MyDistro ./distro.tar

#   -, Dropbox, Onedrive, - 

mkdir ~/AppData/Local/MyDistro
wsl --import MyDistro ~/AppData/Local/MyDistro ./distro.tar --version 2

それで全部です。すべてのシステム間で同期された完璧なLinuxセットアップを取得します。
