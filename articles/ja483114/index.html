私たちがふさわしいモーダルウィンドウ

はじめましょう


さまざまな基準と品質マニフェストには、モーダルウィンドウが開いているときにページスクロールをブロックするなどの項目があります。モーダルウィンドウはそれとの相互作用のみを意味します。

多くの人が「既製のソリューションを使用しないのはなぜですか」という質問をします。問題は、この機能がまったく実装されていないか、非常に貧弱に実行されていることです。最も単純で最悪のソリューションを使用できます。

.body { overflow: hidden;}

ここで、ページのわずかなひねりを取得します。コード。これは注目に値します。この問題はページ上でスクロールが行われる場合にのみ発生します。たとえば、Safariではそのような問題は発生しません。

アニメーションの問題（GIF）


npmのオープンスペースにあるいくつかの解決策は、スクロールをスクロールとしてスタイル設定されたインデントまたはブロック（単純な灰色のバー）で置き換えることを提供しますが、多くの魔法がありますが、問題は修正されません。

最初のコードを少し調整すると、すべてが機能します。

.body { 
    position:fixed;
    overflow-y: scroll;
}

ここで別の問題が発生しました。ページがジャンプします。コード。jsによるソリューションは簡単です。ページを開くと、ページがスクロールされる値が取得され、この値が<body> コードタグのtopプロパティに設定されます。

function getBodyScrollTop() {
    return self.pageYOffset || (document.documentElement && document.documentElement.ScrollTop) || (document.body && document.body.scrollTop);
  }

openModalButton.addEventListener('click', e => {
  e.preventDefault()
  
  body.dataset.scrollY = getBodyScrollTop() //   
  body.style.top = `-${body.dataset.scrollY}px`
  
  modal.classList.add('modal--open')
  body.classList.add('body-lock')
})

closeModalButton.addEventListener('click', e => {
  e.preventDefault()
  
  modal.classList.remove('modal--open')
  body.classList.remove('body-lock')
  window.scrollTo(0, body.dataset.scrollY) //  ,  
})

サイトには、コンテンツがほとんどなく、スクロールもないページがある場合がありますが、モーダルウィンドウを開くと、コードによって追加されます。スクロールを少し変更しましょう。すぐに使えるソリューション。

function existVerticalScroll() {
   return document.body.offsetHeight > window.innerHeight
}

openModalButton.addEventListener('click', e => {
  e.preventDefault()
  
  body.dataset.scrollY = getBodyScrollTop()
  
  modal.classList.add('modal--open')
  
  if(existVerticalScroll()) { //  
     body.classList.add('body-lock')
     body.style.top = `-${body.dataset.scrollY}px`
   }
})

closeModalButton.addEventListener('click', e => {
  e.preventDefault()
  
  modal.classList.remove('modal--open')
  
  if(existVerticalScroll()) { //  
   body.classList.remove('body-lock')
   window.scrollTo(0,body.dataset.scrollY)    
  }
})

インターネットでは、この問題に対する簡単な解決策はありません。また、focus-trap.jsライブラリがある場合、松葉杖を書くのはあまり合理的ではありません。ここで、著者は図書館とその問題のトピックをより詳細に明らかにしました。

const modalFocusTrap = createFocusTrap(".modal"); //     

openModalButton.addEventListener('click', e => {
  e.preventDefault()
  
  body.dataset.scrollY = getBodyScrollTop()

   }

closeModalButton.addEventListener('click', e => {

  modalFocusTrap.deactivate(); //  .

   body.classList.remove('body-lock')
   window.scrollTo(0,body.dataset.scrollY)    
  }
})
