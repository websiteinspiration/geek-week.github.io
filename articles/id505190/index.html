Cara membuat ulang efek pola moire di p5.js library untuk pemula (panduan cepat)



Pola moire adalah ilusi optik yang terjadi ketika dua pola mesh periodik diterapkan. Elemen berulang dari kedua pola itu tumpang tindih atau membentuk celah.

Dalam kehidupan sehari-hari, kita sering menjumpainya, misalnya ketika kita melihat tirai tulle. Tidak mengherankan, nama Prancis Moire sendiri awalnya berarti sejenis kain sutera.



Efek ini sering dapat diamati di berbagai bidang - mulai dari fotografi hingga radiologi dan navigasi.

Untuk mencapai efek ini, dan pada saat yang sama berlatih di p5.js, saya memutuskan untuk menulis kode sederhana yang membuatnya kembali.

p5.js adalah perpustakaan Javascript yang dibuat pada 2014. Tujuan utamanya adalah untuk menjadi jendela ke dunia pemrograman bagi perancang, seniman, guru, dan perwakilan profesi kreatif lainnya. Ini mendukung animasi dan diagram, dengan mudah diterjemahkan ke format web.

Untuk memulai, Anda perlu membuat editor online proyek baru. Secara default, proyek baru adalah sebagai berikut:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
}

function setup () - mengatur fungsi yang dieksekusi hanya sekali
function draw () - mengulangi dalam loop yang tak terbatas.

Kita mulai dengan mengatur "kanvas" kita. Untuk fungsi createCanvas (), kami mengatur parameter 720 dan 500 (masing-masing lebar dan tinggi).

Untuk membuat elips muncul di latar belakang, kami menyisipkan parameter berikut ke dalam fungsi draw () (yang, seperti kita ingat, bertanggung jawab untuk pengulangan):

{
  noFill()
  stroke ('purple') 
  strokeWeight(3)
  ellipse(360,250,100,100) }

Apa yang terjadi disini?

noFill () - tidak mengisi bagian dalam elips dengan warna apa pun.
stroke ('ungu') - menciptakan garis dengan warna ungu.
strokeWeight (3) - mengatur ketebalan garis ini.
ellipse (360.250.100.100) - mengatakan bahwa garis harus diputar menjadi elips dengan parameter yang diberikan (posisi sepanjang X, posisi sepanjang Y, diameter sepanjang X, diameter sepanjang Y). Parameter pertama dan kedua sama dengan setengah parameter dalam fungsi createCanvas (400, 400) - tepatnya di tengah. Parameter 3 dan 4 yang sama menciptakan bentuk lingkaran, bukan oval.

Hasilnya, Anda mendapatkan sesuatu seperti ini.



Belum mengesankan, kan?

Sekarang kita perlu membuat elips ini lebih banyak, masing-masing berikutnya sedikit lebih banyak dari yang sebelumnya. Untuk melakukan ini, kita perlu membuat loop, menghapus elips kita ke dalamnya, dan membuat variabel parameter ketiga dan keempat.

 background(220);
  for(let i=0;i<1000; i+=10) {
  noFill()
  stroke ('purple') 
  strokeWeight(3)
  ellipse(360,250,i,i)

Jika Anda pernah berurusan dengan siklus setidaknya sekali dalam hidup Anda, Anda akan mengerti apa yang terjadi. Kami menetapkan nilai awal i = 0, jadi pada awalnya kami menggambar sebuah elips dalam radius nol. Kemudian kita menetapkan batas untuk nilai apa saya akan mengulangi siklus (hingga 1000), setelah itu kita menentukan langkah perubahan i + = 10 - setiap kali 10 akan ditambahkan ke hasil sebelumnya saya.

Kami mengganti parameter i ke dalam fungsi gambar elips sebagai parameter ketiga dan keempat bertanggung jawab atas diameter lebar dan tinggi.

Sekarang, ketika kita menjalankan kode, kita mendapatkan yang berikut.

Jangan lupa untuk memasukkan noFill () jika tidak, Anda akan menyimpan dan hanya menampilkan hasil dari iterasi terakhir.

Sekarang, untuk mendapatkan elips kedua, kita masukkan ke dalam loop yang sama, tetapi dengan parameter yang sedikit berbeda:

function draw() {
  background(220);
  for(let i=0;i<1000; i+=10) {
  noFill()
  stroke ('purple') 
  strokeWeight(3)
  ellipse(360,250,i,i)
    
  stroke("red")
  strokeWeight(3)
  ellipse(360, 250,i-500,i-500)
  }

Akibatnya, Anda akan mendapatkan urutan lingkaran, yang terbesar dengan diameter sama dengan 1000 minus 500. Ketinggian kanvas kami hanya 500, sehingga perfeksionis internal akan puas. Ya, itu akan berwarna merah.

Sudah waktunya untuk tindakan terakhir - untuk membuat lingkaran ini bergerak melewati satu sama lain.

Ada suatu kondisi. Jika x kurang dari lebar kanvas, maka kami mengatur ulang x ke nilai nol default. Sampai ini terjadi, kami menambahkan 3 ke nilai x kami. Seperti dapat dipahami, melalui X kita ingin mengatur koordinat sepanjang sumbu X untuk lingkaran kedua. Karena itu akan berubah sedikit setiap kali, lingkaran kita akan bergerak secara horizontal ke kanan hingga pusatnya mencapai tepi layar.
