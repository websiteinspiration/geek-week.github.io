DIY音響浮上


今日は、自宅で自分の手で超音波浮揚の効果を繰り返す方法を教えて示します。

以前の記事では、1、2。擬似浮上を実演しました。今回はそれが現実です。理論から始めましょう。

音の振動


浮上効果の本質全体を理解するには、音波と定在波が何であるかを理解する必要があります。

したがって、それらから始めます。音波は全方向に伝播し、正と負の2つの半波で構成されます。正の半波は圧縮ゾーンまたは高圧です。将来的には圧縮と呼び、負の半波は放電ゾーンと呼びます。これを減圧と呼びます。



実際には、次のようになります。スピーカーのディフューザーは、外側に移動すると圧縮を作成し、内側に移動すると圧縮解除します。これは画像にはっきりと示されています。



最大波力はダイナミックヘッドディフューザーの近くで作成され、エミッターから離れるにつれて徐々にパワーを失い、スピーカーから離れるほど弱くなります。

定常波-これは、位相が一致し、同じ波の周波数を持つ2つの対向する重なり合ったときに形成される波です。通常の波が空間での伝播の過程でそのパワーを失うと、短い距離の定在波はほぼ等しいパワーのノードを形成します。これは、多方向波を折りたたむことによって実現されます。弱まる波は、到来する波によって増幅されます。これがどのように発生するかを理解するには、以下の画像を見てください。ノードは灰色でマークされています。この例では、定在波の棚です。これらの棚（ノード）とアイテムを保持します。



波長音の速度を振動の周波数で割った値です。 20°Cの温度と50％の空気湿度で、音は340 m / sの速度でそのような環境で伝播します。ピエゾエミッターの共振振動周波数は約40,000 Hzです。その結果、340,000 mm / 40,000 Hz = 8.5 mmの波長が得られます。定常波長は同じ8.5mmになります。

エミッタは互いに異なる距離に配置できますが、常に波長の倍数である必要があります。エミッタ間の距離が短いほど、定在波のノードはより強力になります。音響送信機間のスペースが広いほど、それらの間のノードが多くなりますが、ノードのパワーは弱まり、その逆も同様です。また、波の節に多数のオブジェクトを保持し、超音波エミッタ間の距離を大きくするには、より強力なピエゾヘッドが必要になることを理解する必要があります。たとえば、駐車場レーダーや家庭用加湿器からです。この記事では、最も手頃な価格で手頃な価格の超音波エミッタを検討します。HC-SR04距離計から取り外すことができます。

接続図 -私は使用のArduino nanoをし、MX1508モータードライバー、NE555ジェネレーターで実行でき、HC-SR04距離計にインストールされたMAX232チップをエミッターのアンプとして使用しますが、私はより簡単なパスを選択したため、時間を節約できました。音響レビトロンを組み立てるのに5分もかからなかった。デュポンワイヤーキットで接続しました。



スケッチArduinoの説明


コードは特別なものではありません。タイマーを設定し、D Arduinoポートをぎくしゃくさせます。これを行うには、Timer1をリセット（CTC）時にリセットモードにし、アカウントレジスタTCNT1の値が比較レジスタOCR1Aの指定された数値と一致すると、割り込みがトリガーされ、ポートD全体が反転するプロセッサで割り込みがトリガーされます。その後、カウントレジスタはリセットされ、終了した後にリセットされます。請求プロセスのプロセッサは、新しいプロセスで開始されます。

必要なOCR1Aレジスタの値は次のように計算されます。TCCR1Bレジスタの分周器がオフになっているため、16,000,000 Hzのクロック周波数を取得し、必要な割り込み周波数80,000 Hzで除算すると、結果として数値200が得られ、これが値になります。レジスタOCR1A用。

なぜ40 kHzではなく80 kHzなのですか？ハンドラの期間は2つの割り込み操作で形成されるためです。最初の操作の間、出力はD3 = 0およびD4 = 1を形成し、2番目の操作では、D3 = 1およびD4 = 0を形成します。

Arduinoのコード


volatile uint8_t portD3_D4 = 8; //   D3    D4

void setup()
{
  pinMode(3, OUTPUT); 
  pinMode(4, OUTPUT);

//  Timer1
  TCNT1 = 0;
  TCCR1A = 0;
  TCCR1B = 0;
  OCR1A = 199; //    16  / 80  = 200
  TCCR1B = (1 << WGM12)|(1 << CS10); //  CTC,   
  TIMSK1 |= (1 << OCIE1A); //    
}

void loop () {}

ISR (TIMER1_COMPA_vect) //    
{
  PORTD = portD3_D4; //    
  portD3_D4 = 255-portD3_D4;//       
}
