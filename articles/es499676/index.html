Más de 90 herramientas útiles para Kubernetes: implementación, administración, monitoreo, seguridad y más


En el otoño de 2018, publicamos una lista de 25 herramientas útiles de Kubernetes . Desde entonces, la plataforma ha crecido en popularidad . El ecosistema de la orquestación de contenedores está en auge, puede encontrar herramientas auxiliares para casi cualquier tarea.

Por lo tanto, el equipo de Kubernetes aaS de Mail.ru actualizó y complementó la selección. Le presentamos una lista con casi un centenar de herramientas útiles que simplifican la vida de quienes trabajan con Kubernetes.

Herramientas de implementación de clúster


1. quilla


Declaración de Kubernetes que automatiza las actualizaciones de DaemonSet, StatefulSet, Helm e Deployment. Un comando, sin dependencias, archivos de configuración y bloqueos.

2. Kube-prod-runtime


Un conjunto de servicios de Kubernetes que simplifica el trabajo de producción bajo una carga pesada. Proporciona monitoreo del rendimiento del clúster, registro, gestión de certificación y descubrimiento automático de recursos en K8 a través de servidores DNS públicos. Este es un conjunto útil de servicios para otras necesidades de infraestructura.

3. K3sup


Después de instalar k3sup (pronunciado ketchup), puede generar kubeconfig en cualquier máquina virtual local o remota en cuestión de segundos.



4. Mail.Ru Cloud Solutions: Cloud Containers


Puede implementar clústeres de Kubernetes en la plataforma en forma de servicio en la nube: en pocos minutos obtendrá un clúster listo para trabajar sin la necesidad de configuración y actualizarlo a la versión correcta. Los clústeres también son fáciles de escalar: funcionan en la infraestructura Mail.Ru, diseñada para servicios altamente cargados.


5. Kubeadm


Kubernetes Cluster Initialization Tool en la configuración óptima para su infraestructura. La principal ventaja es la capacidad de ejecutar grupos de Kubernetes mínimamente viables en cualquier entorno. Los complementos y la configuración de red no están incluidos en la configuración lista para usar, todo tendrá que configurarse manualmente. 

6. Kubespray


Conjunto de roles de Ansible para la implementación y configuración de Kubernetes. Funciona en diferentes plataformas en la nube: AWS, GCE, Azure, Mail.Ru Cloud Solutions, OpenStack y bare metal IaaS. Este es un proyecto de código abierto construido en kubeadm. Adecuado para aquellos que están familiarizados con Ansible: con esta herramienta no necesita saber nada más para implementar todos los recursos necesarios. 

7. Conjurar 


Le permite implementar Kubernetes con solo unos pocos comandos, admite localhost, implementaciones completas en entornos de nube, incluido OpenStack.

8. Minikube


Un buen comienzo para aquellos que recién están conociendo a Kubernetes. La herramienta permite a los usuarios ejecutar fácilmente un clúster de nodo único localmente dentro de una máquina virtual en la computadora portátil de un usuario. Compatible con Mac OS X, Windows y Linux.

9. MicroK8s 


Una herramienta para que los usuarios de Kubernetes implementen un clúster independiente en un servidor Linux es muy adecuada para Edge e IoT.

10. Bootkube 


Lanza clústeres independientes y ayuda a configurar el plano de gestión temporal de Kubernetes. Bootkube también se puede utilizar para crear los recursos necesarios que se utilizarán cuando se inicie el nuevo clúster.

11. RKE por Rancher


Kubernetes certificó la distribución CNCF dentro de contenedores. Le permite simplificar y automatizar la instalación de Kubernetes, sin depender del sistema operativo y la plataforma en la que trabaja.

Herramientas de monitoreo


12. Kube-state-metrics


Una sencilla utilidad para escuchar el servidor API de Kubernetes, ayuda a generar métricas sobre el estado de los objetos. Se enfoca en la salud de varios objetos dentro del clúster, incluidos nodos, hogares e implementaciones.

13. Kubebox


Una consola de terminal que le permite administrar el clúster de Kubernetes y monitorear su estado en tiempo real. El monitor de clúster muestra lo que sucede con los recursos de hogar, los registros de contenedores y otros parámetros. Le permite navegar fácilmente al espacio de nombres deseado y ejecutar el comando en el contenedor deseado. Esto ayuda a hacer frente rápidamente a los problemas y restaurar el trabajo.  


14. Rakess


El complemento Rakess (Revisar acceso) muestra todos los derechos de acceso al clúster de Kubernetes. Por supuesto, para recursos individuales, puede verificar con el comando kubectl auth can-i list deployments, pero no proporciona información completa sobre todos los recursos en el servidor.

15. Kubetail


Script de Bash que le permite agregar los registros de muchos hogares en una sola secuencia. En la versión original, no sabe cómo filtrar o seleccionar, pero en Github hay un tenedor separado que le permite colorear los registros usando MultiTail.

16. popa


Otra herramienta de cola de hogar Kubernetes. Características: el uso de expresiones regulares para el filtrado conveniente de hogares (no necesita conocer identificadores específicos), de manera similar puede filtrar contenedores individuales para hogares solicitados, hay plantillas Go estándar y personalizadas para registros mostrados, restricción de salida de registro por período de tiempo o número de líneas y mucho más .

17. Prometeo


No podemos dejar de mencionar nuevamente esta herramienta de monitoreo y notificación de código abierto, que se ha convertido en el estándar para monitorear Kubernetes. Está integrado con todos los lenguajes de programación populares, ayuda a crear sus propias métricas y contiene muchas integraciones listas para usar con tecnologías populares, por ejemplo: PostgreSQL, MySQL, ETCD.

Con el Operador Prometheus, puede crear instancias Prometheus en clústeres de Kubernetes, incluida una estrecha integración con Grafana y Alertmanager.

18. Jaeger


Una herramienta de rastreo de código abierto. Capaz de controlar transacciones y dependencias de servicios en sistemas distribuidos, identificar y solucionar problemas. Una forma de comenzar con Kubernetes es utilizar el operador especial Jaeger . 

19. Searchlight


Operador de Kubernetes para Icinga . Capaz de realizar verificaciones periódicas en los grupos de Kubernetes y luego enviar notificaciones por correo electrónico, SMS o chat, si algo sale mal. La herramienta predeterminada incluye un conjunto de pruebas específicamente para Kubernetes. Con su ayuda, es posible ampliar las capacidades de monitoreo de Prometheus, también se convertirá en un sistema de respaldo si los sistemas de monitoreo interno fallan por completo.

20. Vista operativa de Kubernetes (vista de operaciones de Kube)


Panel de control del sistema de solo lectura capaz de trabajar con muchos clústeres de Kubernetes. Le permite moverse cómodamente entre clústeres, nodos de seguimiento y estado del hogar. Visualiza una serie de procesos, como la creación y destrucción de hogares. 


21. Kubewatch 


Se ejecuta en pods en el clúster de Kubernetes, monitorea los cambios del sistema, después del lanzamiento recibirá notificaciones a través de enlaces web. Puede personalizar sus notificaciones simplemente editando el archivo de configuración.

22. Alcance del tejido 


Rastree y solucione los problemas de los clústeres de Kubernetes y Docker para que pueda identificar y solucionar problemas fácilmente con aplicaciones en contenedores. Puede usarlo para identificar cuellos de botella en el rendimiento de la aplicación.


23. Turbonomic / Kubeturbo 


Proporciona visibilidad de toda su pila, le permite controlar la eficacia de la infraestructura subyacente y el rendimiento de la ejecución de microservicios en Kubernetes.

Pruebas


24. Kubeval


Herramienta para verificar el archivo de configuración Kubernetes YAML o JSON. La validación se realiza utilizando esquemas generados a partir de Kubernetes OpenAPI. Esto permite la validación de esquemas para diferentes versiones de Kubernetes.

25. Helm-kubeval


Helm plugin utilizado para validar diagramas contra esquemas de Kubernetes. Puede seleccionar versiones específicas de Kubernetes para consultar los diagramas.

26. BotKube


BotKube puede monitorear, depurar y ejecutar comprobaciones en clústeres de Kubernetes. La herramienta también se integra con varias plataformas de mensajería como Slack y Mattermost. Los beneficios son de código abierto y fácil configuración.


27. Sonobuoy


Sonobuoy es una herramienta de diagnóstico para verificar el cumplimiento, depurar cargas de trabajo y realizar pruebas personalizadas para ayudar a determinar el estado del clúster. Las pruebas se realizan de manera no destructiva, mientras que se generan informes claros e informativos.

28. Snyk Container


Snyk lo ayuda a encontrar y corregir rápidamente vulnerabilidades en los contenedores y aplicaciones de Kubernetes durante todo el ciclo de vida de desarrollo.


29. Kube-monkey


Siguiendo los principios de la ingeniería del caos, Kube-monkey eliminará aleatoriamente los módulos de Kubernetes en el clúster y comprobará el desarrollo de servicios tolerantes a fallas.

30. K8s-testuite 


Consta de dos diagramas de Helm para probar el ancho de banda de red y las pruebas de carga de clústeres de Kubernetes. Esto ayudará a garantizar que su configuración sea correcta, así como el estado de los servicios y la distribución correcta de la carga.

31. PowerfulSeal 


La herramienta es específica de Kubernetes y también sigue los principios de la ingeniería del caos, lo que le permite verificar los objetos que funcionan en contenedores. También se puede usar para verificar manualmente los componentes seleccionados del clúster a través del modo interactivo. Después de la implementación, la herramienta funciona de forma autónoma.


La seguridad


32. puerto


Los registros de puerto protegen las imágenes con contenedores mediante la introducción de control de acceso basado en roles. La herramienta también verifica las imágenes en busca de vulnerabilidades y las firma como confiables.

33. Kubesec


Una herramienta de análisis de riesgos de seguridad de recursos de Kubernetes de código abierto. Con él, puede controlar el sistema y obtener una lista completa de recomendaciones para mejorar su seguridad general. 

34. Administrador de permisos


Esta aplicación de desarrollo SIGHUP facilita la administración de roles de acceso para Kubernetes a través del control de acceso basado en roles. Cree usuarios, asigne un espacio de nombres / permiso y distribuya archivos Kubeconfig YAML.


35. Kube-scan


La herramienta de Octarine se centra en la evaluación de riesgos en las cargas de trabajo de Kubernetes. Kube-scan se ejecuta como en clústeres y evalúa 30 parámetros de seguridad para obtener el nivel de riesgo más aceptable. Luego, la herramienta analiza qué parámetros funcionan en conjunto para comprender qué combinaciones reducirán el nivel de amenazas.


36. K-rail


K-rail está diseñado para situaciones en las que necesita un poco más de control en la implementación de sus políticas. Hay muchas formas fáciles de elevar los privilegios, pero en un clúster de múltiples inquilinos, pueden ser peligrosos o causar inestabilidad.

37. KeyCloak


KeyCloak es una herramienta de gestión de identidad y usuario de código abierto. Agrega una función de autenticación de aplicaciones y ayuda con un esfuerzo mínimo para garantizar la seguridad de los servicios. Elimina la necesidad de comprender en detalle el mantenimiento de una lista de usuarios y su autenticación. Todo esto ahora funciona desde el primer momento.

38. Aquasec


La herramienta está diseñada para proteger las instalaciones de Kubernetes durante todo su ciclo de vida. Implementa un agente dedicado en cada contenedor, que actúa como un firewall y elimina posibles vulnerabilidades. Puede gestionar restricciones de seguridad a través de la consola central. Además, la herramienta le permite utilizar configuraciones de seguridad flexibles en entornos locales y en la nube. 

Otra herramienta de código abierto está asociada con él: Kube-Bench , que verifica el entorno de Kubernetes utilizando pruebas del documento de referencia de Kubernetes de la CEI.

39. Tigera


Una herramienta de los creadores del proyecto Calico , un conjunto de soluciones de seguridad de red de Kubernetes con soporte para entornos multi-cloud y heredados a través de una política de seguridad universal automatizada.

40. Klum


Klum, o Kubernetes Lazy User Manager, realiza tareas simples como crear / eliminar / cambiar usuarios. Emite archivos kubeconfig y gestiona roles de usuario.

41. StrongDM


StrongDM es un plano de gestión para verificar la seguridad y el acceso a sus servidores y / o bases de datos. Consiste en una API de autenticación, un servidor proxy que admite el protocolo y un repositorio de registros. 

42. Falco


Una herramienta de seguridad en la nube de código abierto que detecta riesgos para Kubernetes. Notifica el comportamiento inesperado de la aplicación y notifica las amenazas durante su ejecución.

43. Sysdig Secure 


Una plataforma para monitorear la seguridad de microservicios y contenedores. Kubernetes y Docker son compatibles. Se puede usar en la nube y localmente.

Utilidades útiles


44. Krew


Krew ayuda a los desarrolladores a encontrar complementos útiles de kubectl para programas y a instalarlos y administrarlos más tarde. Esta herramienta es similar a APT , DNF o Homebrew .

45. Ksniff


Un complemento para kubectl que utiliza efectivamente Wireshark y tcpdump para capturar de forma remota el tráfico desde cualquier pod en un clúster de Kubernetes.


46. Kube-ps1


El script Kube-ps1 agrega el contexto actual de Kubernetes y el espacio de nombres configurado desde kubectl a la consola Bash / Zsh, no se requieren comandos.

47. Kubefwd


Si está ejecutando los servicios de Kubernetes en un clúster remoto, Kubefwd lo ayudará a redirigirlos a su estación de trabajo local. No se requieren modificaciones: si usa kubectl, ya cumple con todos los requisitos.


48. Kubeterminal


Es más una herramienta auxiliar que complementa kubectl y su consola en Kubernetes.


49. Skaffold


Skaffold es una utilidad de consola que ayuda a garantizar el desarrollo continuo de las aplicaciones de Kubernetes. La herramienta es muy ligera y no requiere componentes en el lado del clúster.

50. Kubectl-alias


Generador de alias simple y muy potente para kubectl. Con él, puede escribir comandos muy rápidamente para la administración diaria de Kubernetes, ya que proporciona más de 800 alias cortos para todas las ocasiones.

51. Kubectx / Kubens


La utilidad de código abierto, que complementa Kubectl, le permite cambiar el contexto y conectarse a múltiples clústeres de Kubernetes al mismo tiempo, así como moverse entre espacios de nombres. Hay soporte para autocompletar en bash / zsh / fish shells.

52. Kube-shell


Una herramienta que acelera el trabajo con kubectl. Autocompletar comandos, ofrece diferentes opciones, busca y corrige los comandos que se ingresan incorrectamente, muestra la ayuda en línea sobre los comandos que se ejecutan.


53. inclinación


Si rara vez sale de la consola, Tilt sincroniza todos los cambios con el clúster y actualiza los servidores para que pueda ver de inmediato cómo afectan los cambios realizados al sistema. La herramienta muestra el estado de cada recurso, emite registros para cada uno de ellos o todos juntos. Todas las actualizaciones se realizan dentro del contenedor, lo que las hace muy rápidas.



54. Kail (cola de Kubernetes)


La herramienta le permite rastrear los registros de Docker para los hogares deseados. Filtra los pods por servicio, implementación, etiqueta y otros parámetros. De acuerdo con los criterios de filtrado, los hogares después del lanzamiento se agregarán automáticamente al diario o se eliminarán de él.

Herramientas de desarrollo


55. Helm 


El administrador de paquetes ayuda a administrar las aplicaciones de Kubernetes con Helm Charts. Esto permite a los usuarios crear ensamblajes reproducibles que se pueden compartir.

56. Helm-2to3


Este complemento ayuda a los desarrolladores a transferir la configuración de Helm v2 a Helm v3 con la limpieza de configuración adecuada .


57. Grajo


Rook ayuda a automatizar varias tareas de almacenamiento de datos, como implementación, carga, escalado, actualización, etc. Esto asegura que la solución de cualquier proveedor (Ceph, EdgeFS, CockroachDB, Cassandra, NFS, Yugabyte DB) funcionará de manera estable en Kubernetes.

58. contorno


Contour, un controlador de ingreso de Kubernetes, proporciona un plano de control para Ingress y el proxy de servicio.

59. Shell-operator


Shell Operator facilita la creación de operadores de Kubernetes. Proporciona integración entre los eventos de clúster de Kubernetes y los scripts de shell. Simplifica la gestión del clúster. 

60. Helm-operator-get-begin


Ayuda a administrar sus lanzamientos de Helm.


61. Helmfile


Helm chart lanzamiento herramienta de gestión. Le permite describir muchas versiones de timones en un solo lugar, establecer el orden de su implementación y hacer otras cosas útiles.

62. Kudo


Kudo simplifica la creación de declaraciones de Kubernetes, principalmente utilizando YAML. Proporciona operadores listos para usar que se pueden configurar de fábrica.

63. Helm-docs


Esta herramienta genera automáticamente documentación a partir de diagramas de Helm en el archivo de descuento. Este archivo contiene metadatos, incluida una tabla con todos los valores del gráfico y los valores predeterminados.

64. Telepresencia 


Permite la depuración local del servicio Kubernetes, simplificando el proceso de desarrollo.

65. Kubectl-debug


Le permite ejecutar un contenedor adicional en el pod que le interesa. El nuevo contenedor utilizará el espacio de nombres junto con el contenedor / contenedores de destino. 

66. Ksync 


Casi instantáneamente sincroniza los archivos de su sistema local con el clúster de Kubernetes. Adecuado si está utilizando scripts donde el problema principal es entregar código a un contenedor en ejecución. 

67. calabaza 


Se utiliza para depurar procesos mientras se ejecutan en un clúster. Fácil de usar, puede seleccionar interactivamente el depurador deseado y el espacio de nombres / bajo el proceso de interés. 

Transportador CI / CD


68. Rafay


Rafay es una herramienta de software que facilita a una empresa o desarrollador individual crear su propia plataforma, sistema de automatización y gestión del ciclo de vida de la aplicación. Rafay también puede ejecutar grupos de Kubernetes.

69. ganadero


Rancher es una plataforma de software completa que implementa fácilmente entornos de contenedor que van más allá de los instaladores de Kubernetes como Kops y Kubespray. Proporciona muchas características, incluida la gestión de la infraestructura, la planificación y organización de contenedores, la supervisión, las comprobaciones de estado, el registro y un potente sistema de control de acceso basado en roles.

70. Borrador


Utilidad de los desarrolladores de Helm. Su objetivo es simplificar las aplicaciones que están desarrolladas para funcionar en Kubernetes. Con dos comandos simples, puede trabajar con aplicaciones de contenedor sin tener que instalar Docker o Kubernetes.

71. Jenkins


Quizás el servidor CI / CD de código abierto más popular del mundo. Hay un complemento gratuito para él , que ayuda a implementar aplicaciones en Kubernetes, actualizarlas con un tiempo de inactividad mínimo y proporcionar una implementación de actualizaciones Verde / Azul. 


72. TeamCity


Servicio conocido de CI / CD de JetBrains. Hay un complemento con el que puede usar la infraestructura de clúster de Kubernetes para ejecutar agentes de compilación de TeamCity (en la versión 2017.1.xy posterior). 

73. Apolo 


Una solución de implementación continua (CD) que proporciona una interfaz de autoservicio para los equipos. Se puede integrar con los procesos de construcción existentes. Esto le permite administrar clústeres de Kubernetes, otorgando a cada usuario permisos específicos para garantizar la seguridad de la implementación.


74. Werf


La herramienta CLI de código abierto escrita en Go está diseñada para simplificar y acelerar la entrega de aplicaciones. Werf crea imágenes de Docker usando Dockerfiles o un enlazador rápido integrado alternativo basado en la sintaxis nativa. También elimina imágenes no utilizadas del registro de Docker. Werf luego despliega su aplicación en Kubernetes usando un gráfico en formato compatible con Helm, con configuraciones convenientes y un mecanismo mejorado para rastrear la implementación, la detección de errores y la salida de registros.

La herramienta le permite crear tuberías que se pueden integrar en cualquier sistema de CI / CD existente. 

75. jardín


Garden es una herramienta para desarrolladores que automatiza sus flujos de trabajo y hace que desarrollar y probar aplicaciones Kubernetes sea más rápido y fácil. Adecuado para el desarrollo colaborativo en un clúster remoto.


Rejillas de servicio


76. Kiali


Kiali ayuda a crear definiciones, verificar y monitorear el funcionamiento de microservicios y conexiones en la red del servicio Istio. La herramienta crea una representación gráfica visual de la topología de la cuadrícula de servicio y da una idea de funciones tales como disyuntores, enrutamiento de solicitudes, demoras y otros. 

77. Kuma


Panel de control universal para redes de servicio y microservicios. Puede funcionar de forma nativa tanto en un entorno virtual como en Kubernetes. Ingrese fácilmente en el arsenal de herramientas de cualquier equipo de la organización.


78. Tenkai


Tenkai es un administrador de microservicios basado en diagramas Helm. Una herramienta con una interfaz web gráfica le permite llamar a repositorios desde diagramas de Helm, son fáciles de configurar e implementar.


Descubrimiento de servicio


79. Descubriendo los servicios Vert.X


Un repositorio con muchas herramientas para descubrir los servicios que son visibles desde sus aplicaciones de microservicio. Los servicios también se pueden importar desde Kubernetes (así como desde Docker y Consul).

Visualización y gestión


80. Octant


Una herramienta web de código abierto que visualiza sus cargas de trabajo de Kubernetes y proporciona actualizaciones en tiempo real sobre ellas.

81. Kubernetic


Kubernetic lo ayuda a implementar fácil y rápidamente diagramas públicos o privados, ver todos los objetos de clúster asociados y sus dependencias en una pantalla. Presenta características tales como visualización en tiempo real, así como soporte para múltiples clústeres.



82. Panel de control de Kubernetes


Interfaz web genérica para clústeres de Kubernetes. El uso de este panel de control nativo facilita la resolución de problemas y el monitoreo de clústeres. 


83. Kubeapps


Interfaz web para el directorio de aplicaciones en clústeres de Kubernetes. Le permite instalar, actualizar y eliminar gráficos de Helm con solo hacer clic en un botón, sin usar la línea de comando.

 

84. lente


Aplicación de escritorio, funciona en Windows, Mac y Linux. Se puede conectar a un clúster local K8, adecuado para una pequeña cantidad de clústeres.


85. Kubevious


Software de código abierto, interfaz gráfica fácil de usar. Muestra todas las configuraciones relacionadas con la aplicación en un solo lugar. Esto ahorra tiempo al eliminar la necesidad de buscar configuraciones y profundizar en selectores y etiquetas. Uno de los inconvenientes de la herramienta es que funciona directamente en el clúster K8. Por lo tanto, debe implementar Kubevious en cada clúster y no solo apuntar al existente.



86. Kubelive


Interfaz de usuario basada en terminal usando Node.js. Bastante fácil de usar, pero actualmente limitado a unos pocos comandos kubectl. Le permite navegar fácilmente por los distintos espacios de nombres del clúster K8 y mostrar rápidamente el estado de un conjunto de hogares dado.


87. K9s


Otra interfaz de usuario conveniente facilita la navegación, el monitoreo y la administración de sus aplicaciones.

Herramientas para la computación / funciones sin servidor


88. Kubeless


Kubernetes es una infraestructura sin servidor de código abierto que le permite implementar pequeñas piezas de código. Admite los idiomas más populares, le permite editar e implementar funciones en tiempo real.

89. Fisión


Otro marco de servidor Kubernetes de código abierto. Soporta todos los lenguajes de programación. Escriba funciones de corta duración en cualquier idioma y compárelas con solicitudes HTTP (u otros desencadenantes de eventos): la herramienta le permite implementar funciones instantáneamente con un solo comando. No hay contenedores para construir ni registros de Docker para administrar.


90. Funktion


Este es un modelo de programación de estilo lambda de código abierto para Kubernetes. Permite a los desarrolladores centrarse en las funciones de escritura, mientras que Kubernetes se encarga del resto.

91. IronFunction


Una plataforma informática de servidor de código abierto para cualquier nube: privada, pública o híbrida. Con esta herramienta, los desarrolladores pueden simplemente descargar su código mientras la plataforma trabaja con la infraestructura. 

92. OpenFaaS


Simplifica la implementación de ambas funciones y el código existente en Kubernetes. Trabaja en nubes públicas y privadas. Le permite crear microservicios y funciones en cualquier idioma. 

93. Nuclio


Un proyecto de servidor que le permite usarlo como un contenedor Docker independiente o incluso encima de otro clúster de Kubernetes. Diseñado para trabajar con eventos de alto rendimiento y grandes cantidades de datos. También proporciona procesamiento de datos en tiempo real con una sobrecarga mínima.

94. Virtual-Kubelet


Es una implementación de código abierto de Kubernetes Kubelet. Comienza dentro del contenedor en su clúster actual y se disfraza como un nodo. A partir de ahí, controla los paquetes planificados como lo hace un Kubelet real.
