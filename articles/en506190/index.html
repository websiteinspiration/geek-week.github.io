How to subtract a series of time intervals and try the Bentley-Ottmann algorithm

Hello everyone!

Recently, I had to solve such a problem: there is a work schedule. For example, a doctor‚Äôs schedule. It is formed using rules and exceptions. It is necessary to subtract the exceptions from the rules, but they are periodic and it is not immediately clear at what point the intersection will occur. And for all this to work quickly, I had to call for help algorithms.

For example: doctor Ivanova O.I. works from 8:00 to 14:00 every day except weekends. But once on 5.06.20 she had to leave for family reasons from 8:00 to 10:00. Then this time she will work the next week from 15:00 to 17:00. Then the rules will look like this:

Rules:
1.06.20 8:00 1.06.20 14:00 Periodic 1 day
06/09/20 15:00 06/09/20 17:00 Not periodic

Exceptions:
6.06.20 00:00 06/08/20 00:00 Periodic Week 1
5.06.20 8:00 5.06.20 10:00 Not periodic


Graphically, this turns into such a scheme:


It turns out that in order to get the resulting schedule, you need to subtract the lower ones from the upper time periods.

And here it is possible in 4 options. See the picture below:



We are looking for a solution


The first solution that comes to mind: compare each rule with each exception and make a decision: whether to send it to the resulting schedule or not. But comparing each with each looks like an algorithm with complexity O (n 2 ). Not very fast.

Then I came across the Bentley-Ottmann algorithm . In the original, it is designed to search for the intersection of lines on a plane. But I have a simpler case: line segments. Those. we have only one dimension left.

The idea of ‚Äã‚Äãthe original Bentley-Ottmann algorithm in a moving scanning line that records the events of the beginning of a segment and the end of a segment. And then compare the second coordinate. But in my case, the scanning line goes from left to right and simply logs the events of the beginning and end of rules and exceptions.

This is how the result should graphically look (below):



Algorithm description


Next, I will tell you how to adapt the Bentley-Ottman algorithm for my task.

1. We start a collection with events. The collection item will look like

<DateTime, isRule, isOpen>

Then our rule

1.06.20 8:00 1.06.20 14:00 Periodic 1 day

falls into the collection in this form:

<1.06.20 8:00, True, True>

<1.06.20 14:00,True, False>

<2.06.20 8:00, True, True>

<2.06.20 14:00, True, False>

<3.06.20 8:00, True, True>

<3.06.20 14:00, True, False>

...

And the exception
6.06.20 00:00 06/08/20 00:00 Periodic Week 1

In such:

<6.06.20 00:00, False, True>

<8.06.20 00:00, False, False>

<13.06.20 00:00, False, True>

<15.06.20 00:00, False, False>

...

2. Sort our collection by the first argument of DateTime.

This is the most expensive part of the algorithm. Because the best sorting complexity is O (n log (n)).

As a result, we get a collection of events sorted by time: both rules and exceptions. It contains information: when the event occurred, the rule is either an exception and the period opens or closes.

3. Next, with a foreach loop, go through the sorted collection. For each element of the collection we will make a number of decisions to form the result.

In the picture below, a decision flow chart.



Comments on the flowchart:

isRuleInAction - a flag that is set to True when the rule starts. When the rule ends, it is set to False.

isExclusionInAction- a flag that is set to True when an exception starts. When the exception ends, then False

resultCandidate is set - a variable for storing the candidate for the resulting collection

4. As a result of the algorithm, we get a collection of sorted time periods.

Conclusion


In a more abstract version, such an algorithm can be used to intersect not only time segments, but also ordinary segments on a numerical line.
