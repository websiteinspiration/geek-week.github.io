仮想バックグラウンドとオープンソースツールを使用したビデオ通話

COVID-19が原因で私たちの多くが隔離されたため、ビデオ通話は以前よりもはるかに頻繁に発生しています。特に、ZOOMサービスは突然大人気に。おそらく最も興味深いズーム機能は、仮想背景のサポートです。これにより、ユーザーは背後の背景をインタラクティブに任意の画像またはビデオに置き換えることができます。



私は、企業のラップトップから通常、Kubernetesのオープンソース会議で、Zoomを長い間使用しています。現在、自宅での操作モードでは、より強力で便利なパーソナルデスクトップコンピューターを使用して、私のオープンソースタスクのいくつかを解決する傾向があります。

残念ながら、ズームは「クロマキー」または「グリーンスクリーン」と呼ばれる背景除去方法のみをサポートしています。この方法を使用するには、背景が無地、理想的には緑色で表現され、均一に照らされている必要があります。

私は緑色の画面を持っていないので、私は自分の背景除去システムを単に実装することにしました。そしてもちろん、これはアパートに物を並べたり、仕事用のラップトップを常に使用したりするよりもはるかに優れています。

結局のところ、既製のオープンソースコンポーネントを使用し、数行の独自のコードを記述するだけで、非常に適切な結果を得ることができます。

カメラデータの読み取り


最初から始めて、次の質問に答えましょう：「処理するWebカメラからビデオを取得する方法？」

私は自宅のコンピューターでLinuxを使用しているので（ゲームをしていないとき）、私は既に慣れているOpen CV Pythonバインディング を使用することにしました。Webカメラからデータを読み取るためのV4L2バインディングに加えて、便利な基本的なビデオ処理機能が含まれています。 python-opencvでウェブカメラからフレームを読み取るのはとても簡単です：



import cv2
cap = cv2.VideoCapture('/dev/video0')
success, frame = cap.read()

カメラを操作するときの結果を改善するために、カメラからビデオをキャプチャする前に次の設定を適用しました。

#    720p @ 60 FPS
height, width = 720, 1280
cap.set(cv2.CAP_PROP_FRAME_WIDTH ,width)
cap.set(cv2.CAP_PROP_FRAME_HEIGHT,height)
cap.set(cv2.CAP_PROP_FPS, 60)

ほとんどのビデオ会議プログラムは、ビデオを720p @ 30 FPS以下に制限しているようです。しかし、いずれにしても、すべてのフレームを読み取るとは限りません。このような設定は上限を設定します。

フレームキャプチャメカニズムをループに配置します。これで、カメラからのビデオストリームにアクセスできます。

while True:
    success, frame = cap.read()

次のように、テスト用にフレームを保存できます。

cv2.imwrite("test.jpg", frame)

その後、カメラが機能していることを確認できます。すごい！

バックグラウンド検出


ビデオストリームにアクセスできるようになったので、背景を見つけて置き換えることができるようにして、背景を検出する方法について考えます。しかし、これはすでにかなり難しい作業です。

Zoomの作成者は、プログラムが背景をどのように削除するかについて正確に話すことはないようですが、システムの動作により、ニューラルネットワークがなければ何ができたのかを考えさせられます。説明するのは難しいですが、結果はそのように見えます。さらに、Microsoft Teamsが畳み込みニューラルネットワークを使用して背景ぼかしを実装する方法に関する記事を見つけました。 原則として、独自のニューラルネットワークを作成することはそれほど難しくありません。画像のセグメンテーションに関する記事や科学論文はたくさんあります。

。オープンソースのライブラリとツールはたくさんあります。しかし、良い結果を得るには、非常に特殊なデータセットが必要です。

特に、前景にいる人物の完璧な写真を含む、ウェブカメラから取得したものを連想させる多くの画像が必要です。このような画像の各ピクセルは、背景とは異なるものとしてマークする必要があります。

ニューラルネットワークのトレーニングの準備としてそのようなデータセットを構築することは、それほどの労力を必要としない場合があります。これは、Googleの研究チームがすでにすべての努力を尽くしており、オープンソースに、人々をセグメント化するための事前トレーニング済みのニューラルネットワークを組み込んでいるためです。このネットワークはBodyPixと呼ばれます。とてもうまくいきます！

BodyPixは現在、TensorFlow.jsに適した形式でのみ利用できます。その結果、body-pix-nodeライブラリを使用して適用するのが最も簡単です。ブラウザーで

次に、結果を取得する方法について説明します。ただし、すぐに警告します。私はNode.jsのエキスパートではありません。これはちょうど私の夜の実験の結果なので、私に寛大になってください:-)。

次の簡単なスクリプトは、HTTP POST要求を使用してサーバーに送信されたバイナリマスクイメージの処理でビジーです。マスクは、ピクセルの2次元配列です。ゼロで表されるピクセルは背景です。

これがファイルコードですapp.js：

const tf = require('@tensorflow/tfjs-node-gpu');
const bodyPix = require('@tensorflow-models/body-pix');
const http = require('http');
(async () => {
    const net = await bodyPix.load({
        architecture: 'MobileNetV1',
        outputStride: 16,
        multiplier: 0.75,
        quantBytes: 2,
    });
    const server = http.createServer();
    server.on('request', async (req, res) => {
        var chunks = [];
        req.on('data', (chunk) => {
            chunks.push(chunk);
        });
        req.on('end', async () => {
            const image = tf.node.decodeImage(Buffer.concat(chunks));
            segmentation = await net.segmentPerson(image, {
                flipHorizontal: false,
                internalResolution: 'medium',
                segmentationThreshold: 0.7,
            });
            res.writeHead(200, { 'Content-Type': 'application/octet-stream' });
            res.write(Buffer.from(segmentation.data));
            res.end();
            tf.dispose(image);
        });
    });
    server.listen(9000);
})();

フレームをマスクに変換するには、Pythonスクリプトでnumpyおよびrequestsパッケージを使用できます。

def get_mask(frame, bodypix_url='http://localhost:9000'):
    _, data = cv2.imencode(".jpg", frame)
    r = requests.post(
        url=bodypix_url,
        data=data.tobytes(),
        headers={'Content-Type': 'application/octet-stream'})
    #     numpy-
    #     uint8[width * height]   0  1
    mask = np.frombuffer(r.content, dtype=np.uint8)
    mask = mask.reshape((frame.shape[0], frame.shape[1]))
    return mask

興味深い実験


マスクの方法を理解したので、結果を改善する方法を尋ねます。

最初の明白なステップは、マスクのエッジを柔らかくすることです。たとえば、これは次のように実行できます。

def post_process_mask(mask):
    mask = cv2.dilate(mask, np.ones((10,10), np.uint8) , iterations=1)
    mask = cv2.erode(mask, np.ones((10,10), np.uint8) , iterations=1)
    return mask

これにより状況は少し改善されますが、あまり進展がありません。そして、単純な交換はかなり退屈です。しかし、これはすべて自分で行ったので、背景を削除するだけでなく、画像に対して何でも実行できることを意味します。

スターウォーズの仮想背景を使用しているので、写真をより面白くするために、ホログラム効果を作成することにしました。さらに、マスクのぼかしを滑らかにすることができます。

最初に、後処理コードを更新します。

def post_process_mask(mask):
    mask = cv2.dilate(mask, np.ones((10,10), np.uint8) , iterations=1)
    mask = cv2.blur(mask.astype(float), (30,30))
    return mask

エッジがぼやけています。これは良いことですが、ホログラム効果を作成する必要があります。

ハリウッドホログラムには通常、次の特性があります。

  • 明るいレーザーで描いたような淡い色またはモノクロの画像。
  • 画像が複数の光線で表示されているかのように、スキャンラインまたはグリッドのようなものを連想させる効果。
  • 「ゴースト効果」-投影がレイヤーで実行される場合、または投影を作成するときに表示される正しい距離が維持されない場合と同様。

これらの効果はすべて、段階的に実装できます。

まず、画像を青色に着色するには、次のメソッドを使用できますapplyColorMap。

#     -  
holo = cv2.applyColorMap(frame, cv2.COLORMAP_WINTER)

次に、ハーフトーンを残すような効果のあるスイープラインを追加します。

#    bandLength    10-30%,
#    bandGap.
bandLength, bandGap = 2, 3
for y in range(holo.shape[0]):
    if y % (bandLength+bandGap) < bandLength:
        holo[y,:,:] = holo[y,:,:] * np.random.uniform(0.1, 0.3)

次に、現在の効果のシフトされた重み付きコピーを画像に追加することにより、「ゴースト効果」を実装します。

# shift_img : https://stackoverflow.com/a/53140617
def shift_img(img, dx, dy):
    img = np.roll(img, dy, axis=0)
    img = np.roll(img, dx, axis=1)
    if dy>0:
        img[:dy, :] = 0
    elif dy<0:
        img[dy:, :] = 0
    if dx>0:
        img[:, :dx] = 0
    elif dx<0:
        img[:, dx:] = 0
    return img

#    : holo * 0.2 + shifted_holo * 0.8 + 0
holo2 = cv2.addWeighted(holo, 0.2, shift_img(holo1.copy(), 5, 5), 0.8, 0)
holo2 = cv2.addWeighted(holo2, 0.4, shift_img(holo1.copy(), -5, -5), 0.6, 0)

最後に、元の色の一部を保持したいので、ホログラフィックエフェクトを元のフレームと組み合わせて、「ゴーストエフェクト」を追加するのと同じようなことを行います。

holo_done = cv2.addWeighted(img, 0.5, holo2, 0.6, 0)

次に、コマンドラインからv4l2loopbackをインストールします。

ダミーカメラをセットアップします。

sudo modprobe -r v4l2loopback
sudo modprobe v4l2loopback devices=1 video_nr=20 card_label="v4l2loopback" exclusive_caps=1

一部のアプリケーション（Chrome、ズーム）の機能を確保するには、設定が必要ですexclusive_caps。マークcard_labelは、アプリケーションでカメラを選択する際の利便性を確保するためにのみ設定されています。番号を表示すると、対応する番号がビジー状態ではなく、ビジー状態になる可能性が低い場合、video_nr=20デバイスが作成さ/dev/video20れます。

次に、スクリプトを変更してダミーカメラを作成します。

# ,  ,   ,   ,  width  height
fake = pyfakewebcam.FakeWebcam('/dev/video20', width, height)

pyfakewebcamはRGBチャネル（赤、緑、青-赤、緑、青）の画像を想定しており、Open CVはBGRチャネル（青、緑、赤）の順序で動作することに注意してください。

フレームを出力する前にこれを修正してから、次のようにフレームを送信できます。

frame = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
fake.schedule_frame(frame)

ここに完全なスクリプトコードがありますfakecam/fake.py：

import os
import cv2
import numpy as np
import requests
import pyfakewebcam

def get_mask(frame, bodypix_url='http://localhost:9000'):
    _, data = cv2.imencode(".jpg", frame)
    r = requests.post(
        url=bodypix_url,
        data=data.tobytes(),
        headers={'Content-Type': 'application/octet-stream'})
    mask = np.frombuffer(r.content, dtype=np.uint8)
    mask = mask.reshape((frame.shape[0], frame.shape[1]))
    return mask

def post_process_mask(mask):
    mask = cv2.dilate(mask, np.ones((10,10), np.uint8) , iterations=1)
    mask = cv2.blur(mask.astype(float), (30,30))
    return mask

def shift_image(img, dx, dy):
    img = np.roll(img, dy, axis=0)
    img = np.roll(img, dx, axis=1)
    if dy>0:
        img[:dy, :] = 0
    elif dy<0:
        img[dy:, :] = 0
    if dx>0:
        img[:, :dx] = 0
    elif dx<0:
        img[:, dx:] = 0
    return img

def hologram_effect(img):
    #    
    holo = cv2.applyColorMap(img, cv2.COLORMAP_WINTER)
    #   
    bandLength, bandGap = 2, 3
    for y in range(holo.shape[0]):
        if y % (bandLength+bandGap) < bandLength:
            holo[y,:,:] = holo[y,:,:] * np.random.uniform(0.1, 0.3)
    #  
    holo_blur = cv2.addWeighted(holo, 0.2, shift_image(holo.copy(), 5, 5), 0.8, 0)
    holo_blur = cv2.addWeighted(holo_blur, 0.4, shift_image(holo.copy(), -5, -5), 0.6, 0)
    #     
    out = cv2.addWeighted(img, 0.5, holo_blur, 0.6, 0)
    return out

def get_frame(cap, background_scaled):
    _, frame = cap.read()
    #      (  ,   )
    #       
    mask = None
    while mask is None:
        try:
            mask = get_mask(frame)
        except requests.RequestException:
            print("mask request failed, retrying")
    # -   
    mask = post_process_mask(mask)
    frame = hologram_effect(frame)
    #     
    inv_mask = 1-mask
    for c in range(frame.shape[2]):
        frame[:,:,c] = frame[:,:,c]*mask + background_scaled[:,:,c]*inv_mask
    return frame

#     
cap = cv2.VideoCapture('/dev/video0')
height, width = 720, 1280
cap.set(cv2.CAP_PROP_FRAME_WIDTH, width)
cap.set(cv2.CAP_PROP_FRAME_HEIGHT, height)
cap.set(cv2.CAP_PROP_FPS, 60)

#   
fake = pyfakewebcam.FakeWebcam('/dev/video20', width, height)

#    
background = cv2.imread("/data/background.jpg")
background_scaled = cv2.resize(background, (width, height))

#    
while True:
    frame = get_frame(cap, background_scaled)
    #    RGB-
    frame = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    fake.schedule_frame(frame)
