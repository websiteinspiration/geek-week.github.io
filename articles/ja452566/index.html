初心者のためのNode.js：ファイル操作の基本

今日は、Node.jsを使用してファイルシステムを操作する方法について説明します。ファイルで実行される基本的な操作を検討してください。これらの操作には、次のものがあります。

  • ファイル作成
  • ファイルを読む
  • ファイルへのデータの書き込み
  • ファイルの削除
  • ファイルの名前を変更

このような操作を実行する必要性は、さまざまな状況で発生します。



Fsモジュール


Node.jsには、fsファイルシステムを操作するための開発者ツールを提供する標準モジュール（ファイルシステムの略）があります。次のようにプロジェクトにインポートできます。

var fs = require(‘fs’);

このモジュールのメソッドは、同期形式と非同期形式で提供されます。非同期メソッドに渡されるコールバック関数は、最初のパラメーターとしてエラーオブジェクトを受け入れ、2番目のパラメーターとして操作が成功したときに返されるデータを受け入れます。例を考えてみましょう：

function fileHandler(){

    fs.readFile('textFile.txt', 'utf8', (err, data) => {
        if(err) throw err;
        console.log(data);
    });

}

.readFile()これから説明する 方法は、ファイルの読み取りです。この例では、コールバック関数に2つのパラメーター- errとがありdataます。ファイルを読み取ろうとしたときに発生する可能性のあるエラーは最初のパラメーターに該当し、操作が成功した後に取得されたデータは2番目のパラメーターに含まれます。.readFile()これは非同期モジュールメソッドであることに注意してくださいfs。同期バージョンはと呼ばれ.readFileSync()ます。他のモジュールメソッドに名前を付けるために、同様のアプローチが使用されます。

新しいファイルを作成する


例から始めましょう：

function fileHandler(){

    fs.open('testFile.txt', 'w', (err) => {
        if(err) throw err;
        console.log('File created');
    });
    
}

ここでは、このメソッドをfs.open()使用して新しいファイルを作成します。最初の引数としてファイル名を取ります。彼の2番目の引数は、ファイルに対して何をしたいかを正確にシステムに伝えるフラグです。この場合、これはフラグw（書き込みの略）で、書き込み用にファイルを開くことを示します。メソッド.open()はさまざまなフラグを受け入れることができます。それらの一部を次に示します。

  1. r：読み取り用にファイルを開く
  2. r+：読み取りおよび書き込み用にファイルを開く
  3. rs：同期モードで読み取るためにファイルを開く
  4. w：書き込み用にファイルを開く
  5. a：ファイルを開いてデータをファイルの終わりに書き込みます
  6. a+：ファイルの最後までデータを読み書きするためにファイルを開く

次の例に進む前に、testFile.txt何らかのテキストエディタを使用して、作成したファイルを編集します。次のテキストを追加します。

This is a test file.
We're learning about Node.js File System.
The End.

ファイルへのデータの書き込み


ファイルに何かを追加する方法について話しましょう：

function fileHandler(){

    fs.appendFile('testFile.txt', ' This line is beyond the end.', (err) => {
        if(err) throw err;
        console.log('Data has been added!');
    });

}

ここでは.appendFile()、既存のファイルの最後にデータを追加する方法を使用します。このメソッドは、最初の引数としてファイル名を受け入れ、2番目の引数としてファイルの末尾に追加されるデータを受け入れます。3番目の引数は、通常どおり、コールバック関数です。

上記のコードが正常に実行されると、ファイルの内容は次のようになります。

This is a test file.
We're learning about Node.js File System.
The End. This line is beyond the end.

データをファイルに書き込む別の方法があります。メソッドの使用を意味します.writeFile()。この方法はによく似て.appendFile()いますが、1つの重要な違いがあります。事実は、メソッドの助けを借り.appendFile()て、ファイルにすでにあるデータの後に新しいデータを追加することです。また、このメソッドを使用する.writeFile()と、ファイルの内容が新しいものに置き換えられます。この方法を試してみましょう：

function fileHandler(){

    fs.writeFile('testFile.txt', "I'm the replacement you've been looking for.", (err) => {
        if(err) throw err;
        console.log('Data has been replaced!');
    });

}

操作が成功すると、ファイルには次のテキストが含まれます。

I'm the replacement you've been looking for.

ご覧のとおり、ファイルの内容は完全に新しいものに置き換えられています。

ファイルを読む


このモジュールfsは、ファイルの読み取り方法を提供します.readFile()。これは、すでに使用した例です。最初のパラメーターとしてファイル名を受け入れ、2番目のパラメーターとしてエンコードを受け入れます。3番目のパラメーターはコールバック関数です。testFile.txtこの方法を使用して、ファイルの内容をコンソールに出力してみましょう。

function fileHandler(){

    fs.readFile('testFile.txt', 'utf8', (err, data) => {
        if(err) throw err;

        console.log('--------- [File Data] ---------');
        console.log(data);
        console.log('--------- [File Data] ---------');

    });

}

それが私たちが得るものです。

ファイルの名前を変更


ファイルの名前を変更するには、次のメソッドを使用します.rename()。

function fileHandler(){

    fs.rename('testFile.txt', 'newTestFile.txt', (err) => {
        if(err) throw err;
        console.log('File renamed successfully!');
    });

}

メソッドの最初の引数は既存のファイルの名前、2番目はこのファイルの新しい名前です。このメソッドの呼び出しが成功すると、ファイルtestFile.txtはに変わりますnewTestFile.txt。

ファイルの削除


ファイルを削除するには、次の方法を使用します.unlink()。

function fileHandler(){

    fs.unlink('newTestFile.txt', (err) => {
        if(err) throw err;
        console.log('File deleted successfully!');
    });

}

このメソッドの呼び出しが成功すると、ファイルが削除されますnewTestFile.txt。
