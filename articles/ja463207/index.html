チャットボットの開発（電報+ YouTube）

なぜここに現れたのですか？

私は最近、電報用にlaravel + botmanを使用したphp bot開発に関する記事を書きました。彼らが書いた一番最初のことは、「スズメの大砲からの射撃」です。私もこれに同意し、開発中にも気づきましたが、laravelに興味がありました。


現在、純粋なphp（ストラップなし）と電報APIでボットを開発しています。同様のトピックに関する記事がたくさんあるため、Google API（youtube api）について少し触れます。youtube APIについて学ぶのが面白くて便利だと思います。

少し余談です。現時点では、私はフロントエンドのプログラマーとして働いており、自分の喜びのためにのみphpを使用しています（それがどんなに楽しいことでもあります）。サーバー側で愚かなエラーがあるかもしれませんが、それは機能し、トレーニング時にはそれで十分です。

問題の定式化


ある晩、ボットとyoutubeを接続するアイデアを得ました。そのとき、api googleの存在を疑うことすらありませんでした。新しい動画がyoutubeチャンネルでリリースされると、ボットに通知をプッシュするボットが必要です。

原則として、機能は非常にシンプルですが、YouTubeでの作業方法は？google自体がこの問題の解決策であり、その名前はgoogle APIであることがわかりました。会社のすべてのアプリケーションを操作できますが、私はyoutubeだけに興味がありました。

カスタマイズと開発


偶然、YouTube APIの調査中に、私の問題に対するエレガントな解決策が見つかりました。それは、APIドキュメント自体によって提供されました。しかし、最初にAPIについて。それを使い始めるには、Googleコンソールに移動して、必要なAPIオプションを選択する必要があります。その後、パスはドキュメントまたはstackoverflowにのみ存在します。ドキュメントのロシア語版はなく、教訓もないことを私はすぐに言わなければなりません。英語の最小限の知識を持っているか、タイプして行動することをお勧めします。

こんな感じです。 Googleコンソールに移動し、プロジェクトを作成して、目的のAPIを選択します。



そして今、問題を解決します。ドキュメントを読んでいるときに、pubsubhubbub.appspot.com / subscribeサービスを見つけました（Googleがそれを自分で示しました）。YouTubeとアプリケーションをリンクできます。どのように機能しますか？ Webhookの原理に基づいて動作します。聴きたいチャンネルのYouTubeを挿入し、毎回チャンネルのアクション（動画の追加、削除、変更）を行って、スクリプトにデータを受信します。

$video = "null";

if (isset($_GET['hub_challenge'])) { echo $_REQUEST['hub_challenge']; } 
else { $video = parseYoutubeUpdate(file_get_contents('php://input')); }

function parseYoutubeUpdate($data) {
    $xml 		= simplexml_load_string($data, 'SimpleXMLElement', LIBXML_NOCDATA);
    $video_id 	= substr((string)$xml->entry->id, 9);
    $channel_id = substr((string)$xml->entry->author->uri, 32);
    $published  = (string)$xml->entry->published;

    return array(
        'video_id'=>$video_id,
        'channel_id'=>$channel_id,
        'published'=>$published 
    );
}

ここでは、ビデオに関する情報を必要とする変数を作成します。次の2行では、pubsubhubbubからのリクエストを確認できるかどうかを判断できます。確認できる場合は確認できます。確認できない場合は、データがYouTubeからのものであり、処理する必要があることを意味します。 parseYoutubeUpdate関数では、応答を処理し、必要なすべての情報を取得します。

ここで、私が理解しているように、YouTubeが（回答のために）呼び出している特別な（私の意見では、誤解の可能性がある）形式のデータについて脱線して話すことは価値があります（atom（IDEと混同しないでください、またはこれは通常のxmlです... 。それは次のようになります（youtubeからの回答の一部）：
