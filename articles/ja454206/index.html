tiny.cc/6fj06y

猫と犬の3 391枚の写真のDanデータセット。



クエストは「ステガノ」とマークされました。キルトの仕事はいくつかの情報を隠す必要があります。猫と犬は何かバイナリーであると推測するのは簡単に思えました。少し考えてみると、この一連の猫と犬は何らかのバイナリメッセージであると想定できます。シールが1で、犬が0であるとします。突然うまくいかない場合は、単純に交換できます。次に、猫と犬を分類するトレーニング済みモデルを見つけます。猫と犬の分類に関するレッスンの多くの例と、それらの後に訓練されたモデルがあります-githubで訓練されたモデルを見つけることができます。訓練されたモデルを使用しますが、極端な場合は自分で訓練します。各画像を0または1として予測します。この「バイト」のシーケンスは文字列に変換されます。

ここで著者の解決策を見ることができます
import time
import sys
import os
import numpy as np
from keras.models import load_model
from keras.preprocessing import image
data_path = "../data/"
picture_path = data_path + "{}.jpg"
CLASSES = [1, 0]
def run(model_path):
pictures_names = os.listdir(data_path)
pic_num = len(pictures_names)
model = load_model(model_path)
res = []
for pic in range(pic_num):
img = image.load_img(picture_path.format(pic), target_size=(224,224))
x = image.img_to_array(img)
x = np.expand_dims(x, axis=0)
preds = model.predict(x)
y_classes = preds.argmax(axis=-1)
res.append(CLASSES[y_classes[0]])
res = ''.join(map(str, res))
n = int("0b" + res, 2)
text = n.to_bytes((n.bit_length() + 7) // 8, 'big').decode()
print(text)
if __name__ == '__main__':
if len(sys.argv) < 2:
print("path to the model expected")
exit(1)
run(sys.argv[1])

ノート




このサービスは一種の「ブログ」で、各ユーザーがパブリックエントリーとプライベートエントリーを残すことができました。機能が小さいので、何らかの理由でプライベートレコードを取得する必要があると推測することは難しくありませんでした。

実際には、入力フィールドは1つしかありませんでした-レコードID。
何をすべきか？

セキュリティガードに最初に頭に浮かぶのは、SQLインジェクションを試すことです。ただし、サービスはAIによって保護されているとされています。そして、単純なSQLインジェクションを送信できませんでした。サービスはそのような攻撃に「ハッキングの試み！」で対応しました。多くの人がフラグのようにそれを渡そうとしましたが、本当にすべてがとても簡単だと思いましたか？

テストの内部には、SQLインジェクションのIDを分析するLSTMネットワークがありました。ただし、LSTMへの入力は固定長でなければなりません。簡単にするために、20文字に制限しました。つまり、ロジックは次のとおりです。20文字を超える場合はクエリを受け取り、それを切り捨てて残りを確認し、それ以下の場合は0を追加し

ます。したがって、実際には、単純なsql-injectionはすぐには機能しませんでした。
ただし、ネットワークが認識せず、適切な要求を処理できないベクトルを見つける可能性がありました。
