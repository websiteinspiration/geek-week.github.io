Iteraptor：ディープトランスペアレントマップのレデュス用ライブラリ

Elixirデータ構造は不変です。これは、データが他の無関係なコードの一部で認識できないほど破損しないという確信の観点からはすばらしいですが、深くネストされた構造を変更する必要がある場合は少し面倒です。


AccessデフォルトのKernel関数によってエクスポートされた関数を使用して、深くネストされたオブジェクトに対する4つの基本操作を大幅に簡素化する、すばらしい抽象化があります。


  • Kernel.get_in/2
  • Kernel.put_in/{2,3}
  • Kernel.update_in/{2,3}
  • Kernel.pop_in/{1,2}

これら4人の銃士（およびd'Artagnan Kernel.get_and_update_in/{2,3}は通常、次のように使用されます：


iex> users = %{"john" => %{age: 27, mood: ""}, "meg" => %{age: 23}}

#  
iex> get_in(users, ["john", :age])
#⇒ 27

#  
iex> put_in(users, ["john", :age], 28)
#⇒ %{"john" => %{age: 28, mood: ""}, "meg" => %{age: 23}}

#  
iex> update_in(users, ["john", :mood], & &1 + 1)
#⇒ %{"john" => %{age: 28, mood: ""}, "meg" => %{age: 23}}

#  
iex> pop_in(users, ["john", :mood])
#⇒ {"", %{"john" => %{age: 27}, "meg" => %{age: 23}}}

便利で多くの場合動作します...動作しない場合を除きます。を使用Accessするには、ターゲット要素へのパスを知る必要があります。ネストされた複数の値を一度に更新するには、大量のボイラープレートコードが必要です（たとえば、値を持つすべてのリーフを削除するnilか、ログに表示する必要のないすべてのフィールドのコンテンツにスターを付けます）。


ネストされた構造を扱う際の割引を提供するために、ライブラリが作成されましたIteraptor。

反復、マッピング、削減


# each
iex> %{a: %{b: %{c: 42}}} |> Iteraptor.each(&IO.inspect(&1, label: "each"), yield: :all)
# each: {[:a], %{b: %{c: 42}}}
# each: {[:a, :b], %{c: 42}}
# each: {[:a, :b, :c], 42}
%{a: %{b: %{c: 42}}}

# map
iex> %{a: %{b: %{c: 42}}} |> Iteraptor.map(fn {k, _} -> Enum.join(k) end)
%{a: %{b: %{c: "abc"}}}

iex> %{a: %{b: %{c: 42}, d: "some"}}
...> |> Iteraptor.map(fn
...>      {[_], _} = self -> self
...>      {[_, _], _} -> "********"
...>    end, yield: :all)
%{a: %{b: "********", d: "some"}}

# reduce
iex> %{a: %{b: %{c: 42}}}
...> |> Iteraptor.reduce([], fn {k, _}, acc ->
...>      [Enum.join(k, "_") | acc]
...>    end, yield: :all)
...> |> :lists.reverse()
["a", "a_b", "a_b_c"]

# map-reduce
iex> %{a: %{b: %{c: 42}}}
...> |> Iteraptor.map_reduce([], fn
...>      {k, %{} = v}, acc -> {​{k, v}, [Enum.join(k, ".") | acc]}
...>      {k, v}, acc -> {​{k, v * 2}, [Enum.join(k, ".") <> "=" | acc]}
...>    end, yield: :all)
{​%{a: %{b: %{c: 42}}}, ["a.b.c=", "a.b", "a"]}

# filter
iex> %{a: %{b: 42, e: %{f: 3.14, c: 42}, d: %{c: 42}}, c: 42, d: 3.14}
...> |> Iteraptor.filter(fn {key, _} -> :c in key end, yield: :none)
%{a: %{e: %{c: 42}, d: %{c: 42}}, c: 42}

深い入れ子→フラット構造およびその逆


iex> %{a: %{b: %{c: 42, d: [nil, 42]}, e: [:f, 42]}}
...> |> Iteraptor.to_flatmap(delimiter: "_")
#⇒ %{"a_b_c" => 42, "a_b_d_0" => nil, "a_b_d_1" => 42, "a_e_0" => :f, "a_e_1" => 42}

iex> %{"a.b.c": 42, "a.b.d.0": nil, "a.b.d.1": 42, "a.e.0": :f, "a.e.1": 42}
...> |> Iteraptor.from_flatmap
#⇒ %{a: %{b: %{c: 42, d: [nil, 42]}, e: [:f, 42]}}

パン


iex> Iteraptor.jsonify([foo: [bar: [baz: :zoo], boo: 42]], values: true)
%{"foo" => %{"bar" => %{"baz" => "zoo"}, "boo" => 42}}

iex> Iteraptor.Extras.bury([foo: :bar], ~w|a b c d|a, 42)
[a: [b: [c: [d: 42]]], foo: :bar]
