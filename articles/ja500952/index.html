Pythonプログラマの主要なスキル

私たちのダイナミックな時代において、プログラマーは求められているスペシャリストであり続けるために、遅れをとらず常に新しいスキルを学ぶ必要があります。

私は約2年間Pythonでプログラミングしてきましたが、今では意識的に新しいスキルの開発に取り組むときがきました。そのために、私は空席を分析し、必要なスキルをグラフの形で提示することにしました。スキルがさまざまな専門分野（バックエンド開発、データサイエンスなど）に対応するクラスターを形成することを期待していましたが、現実はどうですか？まず最初に。

データ収集


まず、データソースを決定する必要がありました。私はいくつかのオプションを検討しました：Habr Career、Yandex Work、HeadHunterなど。 HeadHunterが最も便利に見えました。空室には主要なスキルのリストがあり、便利なオープンAPIがあるためです。

HeadHunterのAPIを調べた結果、特定のキーワード（この場合は「python」）のジョブIDのリストを解析し、次に各ジョブに対応するタグのリストを解析することにしました。

空室を検索すると、空室はページごとに返されます。1ページあたりの空席の最大数は100です。最初に、すべての結果をページの回答のリストの形式で保存しました。

このため、requestsモジュールが使用されました。APIによると、「user-agent」フィールドに仮想ブラウザの名前が入力され、HHはスクリプトが仮想ブラウザにアクセスしていることを理解しました。サーバーに過負荷がかからないように、要求の間にわずかな遅延を設けました。

ses = requests.Session()
ses.headers = {'HH-User-Agent': "Mozilla/5.0 (X11; Linux x86_64; rv:10.0) Gecko/20100101 Firefox/10.0"}

phrase_to_search = 'python'
url = f'https://api.hh.ru/vacancies?text={phrase_to_search}&per_page=100'
res = ses.get(url)

# getting a list of all pesponses
res_all = []
for p in range(res.json()['pages']):
    print(f'scraping page {p}')
    url_p = url + f'&page={p}'
    res = ses.get(url_p)
    res_all.append(res.json())
    time.sleep(0.2)

その結果、回答辞書のリストを取得しました。各辞書は、検索結果の1ページに対応しています。

結局のところ、hh.ru APIは、空席の最大数を2,000に制限します。つまり、ページごとに100の空席がある場合、ページの最大数は20になります。Pythonキーワードの場合、20の空席ページが返されました。それでもなお。

タグのリストを取得するには、次のようにしました。
  • 検索結果の各ページで繰り返し、
  • ページ上の各ジョブを繰り返し、ジョブIDを取得しました。
  • APIを介して空席の詳細をリクエストし、
  • 空室で少なくとも1つのタグが指定されている場合、タグのリストがリストに追加されました。

# parcing vacancies ids, getting vacancy page and scraping tags from each vacancy
tags_list = []
for page_res_json in res_all:
    for item in page_res_json['items']:
        vac_id = item['id']
        vac_res = ses.get(f'https://api.hh.ru/vacancies/{vac_id}')
        if len(vac_res.json()["key_skills"]) > 0:  # at least one skill present
            print(vac_id)
            tags = [v for v_dict in vac_res.json()["key_skills"] for _, v in v_dict.items()]
            print(' '.join(tags))
            tags_list.append(tags)
            print()
        time.sleep(0.1)

興味深いことに、表示された2000件の求人のうち、タグが付いた求人は1579件のみでした。

データのフォーマット


次に、タグを処理して、グラフの形式で表示するのに便利な形式に変換する必要があります。
  • すべてのタグを1つのレジスタに持ってくるので、「機械学習」、「機械学習」、「機械学習」は同じ意味です
  • 各タグの出現頻度としてノードの値を計算し、
  • タグの相互の合同の頻度として接続の値を計算します。

単一のレジスタに削減し、各タグの発生頻度を計算し、ノードのサイズでフィルタリングします。

tags_list['items'] = [[i.lower() for i in line] for line in tags_list['items']]

# counting words occurrences
flattened_list = [i for line in tags_list for i in line]
nodes_dict_all = {i: flattened_list.count(i) for i in set(flattened_list)}
nodes_dict = {k:v for k, v in nodes_dict_all.items() if v > del_nodes_count}

次のように計算されたペアワイズオカレンス。最初に、キーがすべてタプルの形式のタグのすべての可能なペアであり、値がゼロである辞書を作成しました。次に、タグのリストを調べ、遭遇した各ペアのカウンターを増やしました。次に、値がゼロであるすべての要素を削除しました。

# tags connection dict initialization
formatted_tags = {(tag1, tag2): 0 for tag1, tag2 in itertools.permutations(set(nodes_dict.keys()), 2)}

# count tags connection
for line in tags_list:
    for tag1, tag2 in itertools.permutations(line, 2):
        if (tag1, tag2) in formatted_tags.keys():
            formatted_tags[(tag1, tag2)] += 1

# filtering pairs with zero count
for k, v in formatted_tags.copy().items():
    if v == 0:
        del formatted_tags[k]

Pythonの視覚化


グラフを視覚化するために、networkxモジュールを使用しました。これは、ノードをフィルタリングせずに初めて発生したことです。



この視覚化は、スキルグラフというよりは、絡み合った糸の玉のようなものです。接続は混乱しており、グラフに非常に密に浸透しているため、ノードを作成することは不可能です。さらに、グラフにノードが多すぎます。ノードが小さすぎるため、統計的有意性がありません。

したがって、サイズが5未満の最小のノードを除外し、灰色のリンクも作成しました。この図では、まだ単語を1つのレジスターに持っていませんが、接続を解除するために最大のPythonノードを削除しようとしました。



ずっと良くなりました。これでノードが分離され、リンクが視覚化を妨げません。基本的なスキルを見ることができるようになり、グラフの中央にある大きなボールと小さなノードに配置されています。しかし、このグラフにはまだ改善の余地があります。
