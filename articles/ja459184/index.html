スキル＃1-プロジェクト構造での作業


プロジェクトの秩序を維持するには、その構造を理解する必要があります。次のように表すことができます。

  1. モデル-プロジェクトで使用されるすべてのモデルを保存します
  2. プレハブ-すべてのモデルテンプレートを保存
  3. テクスチャ-すべてのテクスチャを保存します
  4. 材料-すべての材料を保存します
  5. スクリプト-すべてのスクリプトを保存します
  6. サウンド-すべてのサウンドを保存します
  7. アニメーション-すべてのアニメーションを保存します

キャラクターなどの複雑なオブジェクトがある場合は、別のフォルダーが作成され、上記の構造が繰り返されます。

スキル2-シーン内のオブジェクトの階層を操作する


同一のオブジェクトのセットでの作業を簡略化するために、空のオブジェクトが親として機能するグループに組み合わせることができます。

これは、場所全体または屋内で簡単に照明を消すのに役立ちます。すべてのオブジェクトの場所を見つけます。

各要素はそれが何であるかに関して呼び出され、最後に親の接尾辞が追加されます。たとえば、洞窟内の金の延べ棒は「goldBar_cv1」と呼ばれ、cv1はcave1を表し、森の木は「tree_frt1」で、frt1はforest1を表します。接尾辞の場合、単語の最初の文字、2番目の音節の最初の文字、および単語の最後の文字が使用されます。

スキル5-オブジェクトを操作する


シーンにキャラクターを追加したいですか？RigidBodyおよびCapsuleCollaiderコンポーネントを追加します。RigidBodyは、ソリッドステートフィジックスをこのオブジェクトに適用する必要があることをエンジンに通知します。

キャラクターとオブジェクトの衝突をチェックしたいですか？MeshCollaiderコンポーネントをオブジェクトに追加し、OnCollisionEnter関数のキャラクタースクリプトで、キャラクターが遭遇したオブジェクトのタグを確認します。

スキル7-ロジックの実装


コストのかかる操作は避けてください。例として、オブジェクトまでの距離を決定します。

コストのかかる操作の例
using UnityEngine;
using System.Collections;

public class ExampleClass : MonoBehaviour
{
    public Transform other;

    void Example()
    {
        if (other)
        {
//      
//        
            float dist = Vector3.Distance(other.position, transform.position); 
            print("Distance to other: " + dist);
        }
    }
}


より速い実装例
using UnityEngine;
using System.Collections;

public class ExampleClass : MonoBehaviour
{
    public Transform other;
    public float closeDistance = 5.0f;

    void Update()
    {
        if (other)
        {
            Vector3 offset = other.position - transform.position;
 //    
            float sqrLen = offset.sqrMagnitude;

            //    ,    
            if (sqrLen < closeDistance * closeDistance)
            {
                print("The other transform is close to me!");
            }
        }
    }
}
