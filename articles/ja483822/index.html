コードなしで作成できない5つのJavaScript機能

こんにちは！

始める前に、私が書いているコードについて少しお話ししましょう。私のコードのほとんどすべては、JavaScriptと、いくつかのHTMLおよびCSSで構成されています。クライアントとサーバーの両方のJSを作成します。コードをテストしています。私は世界中の何千人もの開発者が使用するオープンソースライブラリを作成して配布しています。フロントエンドには、React、バックエンドにはExpressまたはサーバーレスコンピューティングを使用します。

JSの5つの機能がなければ、コードを記述できませんでした。順不同。もちろん、「それがなければコードを記述できませんでした」は誇張表現です。これらは、私が本当に気に入っていて、常に使用している「機能」です。

2.モジュール


これは、ほぼすべてのファイルで使用するもう1つの機能です。モジュールが言語の一部になる前に、奇妙なライブラリーとツールを使用して大規模なプロジェクトを処理する必要がありました。モジュール（およびRollupやWebpackなどのビルダー）を使用して、コードを他のユーザーと共有する絶好の機会があります。

次に例をいくつか示します。
// a.js
// 
function add(a, b) {
  return a + b
}

const foo = 'bar'

const theAnswer = 42
const theQuestion = 'who knows'
   
//        , 
//     ""   
export default add
export {foo, theAnswer, theQuestion}

// b.js
// 
// 1.  
import './a'

// 2.   
import add from './a'

// 3.  `theAnswer`  `theQuestion`  './a'
import {theAnswer, theQuestion} from './a'

// 4.  `theAnswer`     `fortyTwo`
import {theAnswer as fourtyTwo} from './a'

// 5.  `add` ( )  `theQuestion`
import {default as add, theQuestion} from './a'

// 6.  `add`  `theQuestion`     
import add, {theQuestion} from './a'
   
// 7.     " "   `allTheThings`
import * as allTheThings from './a'

3.デフォルトのオプション


私はいつもこの機能を気に入って使用しています。これは、関数の引数と分解の両方に適用されます。オブジェクトを再構築するために使用する方法は次のとおりです。
const bench = {type: 'Piano', adjustable: false}
const {legs = 4} = bench
// `The bench has ${legs} legs`
// -> The bench has 4 legs
// bench - , leg -

ベンチオブジェクトには、legsプロパティがないことに注意してください。デフォルトのパラメーター構文がない場合、legs値は未定義になります。

この機能で破壊的な割り当てを使用することもできます。
const bench = {type: 'Piano', adjustable: false}
const {legs: legCount = 4} = bench
// `The bench has ${legCount} legs`
// -> The bench has 4 legs

パラメータリストでは次のようになります。
function getDisplayName(firstName = 'Unknown', lastName = 'Unknown') {
  return `${firstName} ${lastName}`
}

// getDisplayName()
// -> Unknown Unknown

// getDisplayName('Andrew')
// -> Andrew Unknown

// getDisplayName(undefined, 'Yang')
// -> Unknown Yang

// getDisplayName('Andrew', 'Yang')
// -> Andrew Yang

この機能を使用すると、「=」記号の右側の値が必要な場合にのみ計算されるため、かなり興味深いことができます。つまり、これを使用して必要なパラメーターを確認できます。
function getCandy(
  kind = requiredParam('kind'),
  size = requiredParam('size'),
  upperKind = kind.toUpperCase(),
  callback = function noop() {},
) {
  const result = {kind, size, upperKind}
  callback(result)
  return result
}

function requiredParam(argName) {
  throw new Error(`${requiredParam} is required`)
}

// getCandy('twix', 'king')
// -> {kind: 'twix', size: 'king', upperKind: 'TWIX'}
// getCandy('twix')
// -> : 'size is required'

このコードが複雑すぎると感じる人もいます。おそらく彼らは正しい。しかし、それがどのように機能するかを理解することは本当にクールです！

5.約束と非同期/待つ


JSはシングルスレッドであり、イベントシステム（コールスタック）上に構築されています。私のような話の大ファンだイベントループとは何ですか？（ロシア語字幕付き）。Promiseとasync / awaitは、これを管理するための優れたツールです。私のコードのほとんどは非同期であり、名前付きツールは私の作業を大幅に簡略化します。正直なところ、約束は深刻なトピックであり、慣れる必要がありますが、それは素晴らしいことです。

私はテストとバックエンドでasync / awaitをよく使用します。以下は非同期テストの例です：
test('Can fill out a form across multiple pages', async () => {
  mockSubmitForm.mockResolvedValueOnce({success: true})
  const testData = {food: 'test food', drink: 'test drink'}
  const {findByLabelText, findByText} = render(<App />)

  user.click(await findByText(/fill.*form/i))

  user.type(await findByLabelText(/food/i), testData.food)
  user.click(await findByText(/next/i))

  user.type(await findByLabelText(/drink/i), testData.drink)
  user.click(await findByText(/review/i))

  expect(await findByLabelText(/food/i)).toHaveTextContent(testData.food)
  expect(await findByLabelText(/drink/i)).toHaveTextContent(testData.drink)

  user.click(await findByText(/confirm/i, {selector: 'button'}))

  expect(mockSubmitForm).toHaveBeenCalledWith(testData)
  expect(mockSubmitForm).toHaveBeenCalledTimes(1)

  user.click(await findByText(/home/i))

  expect(await findByText(/welcome home/i)).toBeInTheDocument()
})

Expressでasync \ awaitを使用する例を次に示します。
async function getListItems(req, res) {
  const listItems = await listItemsDB.query({ownerId: req.user.id})
  res.json({listItems: await expandBookDataMultiple(listItems)})
}

不思議なことに、私はReactコードでasync / awaitを頻繁に使用しません（少なくとも直接）。これは、コンポーネントの外でほとんどの「非同期ロジック」を実行しようとするためです。したがって、たとえば、ReactでuseEffectを呼び出すときに非同期で何かを行う場合は、非同期関数の呼び出しを1つに制限します。これは、promiseを使用する方が簡単だからです。
React.useEffect(() => {
  getUser().then(
    user => setState({status: 'success', error: null, user}),
    error => setState({status: 'error', error, user: null}),
  )
}, [])
