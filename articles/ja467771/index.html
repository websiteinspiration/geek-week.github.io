興味深いことに、node-watchパッケージは、変更されたファイルの名前をパス\から\ file.cssの形式で送信しますが、hrefでは、パスは/ / / file.cssに書き込まれます。なぜならフルネームでファイルをチェックしていますが、チェックのためにスラッシュを逆に変更する必要がありました。

ただし、3つの問題が残っています。
1.オプションはクロムでは正常に機能し、エッジでは確実に機能しません。ここであなたは掘る必要があります、なぜならそれにもかかわらず、レイアウト内のマルチブラウザ（そしてこれはレイアウトの改善にすぎません）は非常に必要ですが、これはおそらく2つの問題が原因です。

2.ブラウザはスマートです。すでにアップロードされたファイルをキャッシュします。パラメータが変更されていない場合、何も変更されません。つまり、理論的には、同じ名前でファイルを保存すると、ブラウザは何も変更されていないと見なし、コンテンツを再ロードしません。これと戦うために、男たちは毎回名前を変えます。これがなくてもChromeで動作しますが、ニュアンスが重要です。

3.名前の明確な一致が必要です。それら。リンクする絶対パス（./で始まる）を設定すると、プログラムは一致を検出しません。
./path/to/file！=パス/ to /ファイル（コードのロジックを理解するため）。そして、これも修正する必要があります。

したがって、キャッシュが行われないようにファイル名を毎回更新する必要があります。
より正確には、cssファイルが変更されたlinkのhref属性を変更するたびに。

hrefへのパスは、「。」、「/」、またはすぐに名前で始めることができます。
暇なときにこの問題について考えました。
エントリポイント-index.html（および私の場合はentry.html）-は常に（通常は）トップレベルにあります。そして、スクリプトで接続されたcssファイルは常に（通常）深部のどこかにあります。したがって、繰り返しますが、パスは常に同じです（フォルダー名とファイル名）。最初の文字のみが異なります。
したがって、部分を疑問符で区切った後、同じ方法で改行しますが、すでに '/'になっているため、予想される最初の点を削除し、配列要素を1つの行に接続します。これにより、正確な検索を比較します。

const socket = io.connect();

const findFullPathString = (path) => path
  .split('/')
  .filter((item) => item !== '.')
  .filter((item) => item)
  .join('');

const makeCorrectName = (name) => name
  .replace('\\', '/')
  .split('?')[0];

const findCss = (hrefToReplace) => {
  const head = document.getElementsByTagName('head')[0];
  const replacedString = findFullPathString(hrefToReplace);
  const cssLink = Array.from(head.getElementsByTagName('link'))// special for IE
    .filter((link) => {
      const href = link.getAttribute('href').split('?')[0];
      const hrefString = findFullPathString(href);
      if (hrefString === replacedString) return link;
    });
  return cssLink[0];
};

const replaceHref = (cssLink, hrefToReplace) => {
  cssLink.setAttribute('href', `${hrefToReplace}?${new Date().getTime()}`);
  return true;
};

const tryReloadCss = (name) => {
  const hrefToReplace = makeCorrectName(name);
  const cssLink = findCss(hrefToReplace);
  return cssLink ? replaceHref(cssLink, hrefToReplace) : false;
};

socket.on('change', ({ name }) => {
  const isCss = tryReloadCss(name);
  if (!isCss) location.reload();
});
