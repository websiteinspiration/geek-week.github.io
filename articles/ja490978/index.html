SecureCRTを使用して複数のデバイスに同時にコマンドを送信する

ネットワークエンジニアは、多くの場合、多数の機器に対して一連の同一のアクションを実行するタスクを持っています。これをすばやく行う主な方法は、アクションをメモ帳に書き込み、各デバイスにコピー/貼り付けることです。SecureCRTソフトウェアとコマンドウィンドウ機能を使用して、この高速プロセスを高速化する方法があります。これにより、複数のデバイスに同時にコマンドを送信できます。
この記事の目的は何ですか？この記事はファストスタートサイクルからのものであり、複数のデバイスに機器（1つのタスク）をセットアップするときにネットワークエンジニアが時間を節約することを目的としています。SecureCRTソフトウェアと組み込みのコマンドウィンドウ機能を使用します。

前書き


ネットワーク機器のセットアッププロセスでは、多くの場合、多数のデバイスでシンプルで同一のアクションを実行するタスクが発生します。これらのタスクを実行するには、メモ帳+コピー/貼り付けという標準的な方法を使用できます。また、組み込みのSecureCRT機能（コマンドウィンドウ）を使用して、コマンドを機器に送信する前にコマンドを編集できます。また、追加機能：現在のセッションだけでなく、いくつかの開いているセッションにもコマンドを送信します。

コマンドウィンドウの呼び出し、操作モードの設定


SecureCRTメニュー=>表示=>コマンドウィンドウのメニューからコマンドウィンドウが呼び出されます。またはSecureCRTメニュー=>表示=>チャットウィンドウ。


動作モードは次のとおりです。
  • 編集モード
  • リアルタイム（「すぐに文字を送信する」）

編集モード-コマンドをデバイスに送信する前に編集できます。
リアルタイムモード-スペースまたはキープレス（アルファベット順またはEnterキー）を問わず、すべてのキープレスが自動的にデバイスに送信されます。

コマンド転送設定には、以下の動作モードがあります。
  • アクティブなセッションに転送します（現在のセッションのみ）。
  • すべてのセッションで。
  • 選択したグループ（タブ）またはグループへ。

正式には、オプションは次のとおりです。
Active Session — Send commands only to the active session. This is the default.
All Sessions — Send commands to all your open sessions at once.
Selected Tab Group(s) — Send commands to one or more tab groups. This setting is automatically selected when you right-click on a tab and enable Send Commands to This Group. When commands are being sent to selected tab groups, the tabs of the sessions that commands will be sent to contain an asterisk in front of the session name.
Visible Sessions — Send commands to all visible tabbed sessions. When there are multiple tab groups, commands will be sent to the currently active session in each tab group.

リアルタイムモード設定は、コマンドウィンドウの状況依存メニューを使用して有効にします。（コマンドウィンドウを右クリック-状況依存メニューがポップアップ表示され、[文字をすぐに送信]メニュー項目を選択）
コマンド転送設定は、コマンドウィンドウの状況依存メニューを使用して呼び出されます。（コマンドウィンドウを右クリック-状況依存のメニューがポップアップ表示され、[コマンドの送信先]を選択し、メニューのサブアイテム{[すべてのセッション] | [選択されたタブ] | [アクティブセッション]を選択します）


コマンドウィンドウを使用して複数のデバイスを同時に構成する


多くの場合、機能するには、いくつかのデバイスで簡単なコマンドを同時に実行する必要があります。これを行うには、コマンドウィンドウを使用して複数のデバイスに同時にデータを送信するのが（高速であるため）非常に便利です。
複数のデバイスを構成するには：
  1. 同じタイプのすべてのデバイスに接続されています。
  2. SecureCRTメニューで、コマンドウィンドウを呼び出します。
  3. コマンドウィンドウのコンテキストメニューで、[すべてのセッション]を選択します。
  4. コマンドは、単一のユニットとして、または一度に1つずつ入力します。（1回のスワップで、コマンドはControl + Enterを介して入力されます。装置に転送せずに改行します。）

これは通常、コマンドの標準セットです。
screen-len 0 tem
dis cu
save
y
q

結論


常に注意する必要があります。
場合によっては、すべてのセッションから切断する方が高速です。カスタムセッションにのみ接続します。コマンドウィンドウにコマンドを入力します。他のすべてのセッションに接続した後。SecureCRTへの接続は非常に高速です。また、再接続しても問題は発生しません。
多くの場合、グループコマンドを入力した後、各セッションでの作業がチェックされます（[ctrl + tab]の組み合わせを使用して、すべてのウィンドウを切り替え、各ウィンドウでのコマンドの結果を比較します）。すべてのコマンドの実行の正しさ、1つのセッションで発生する可能性のあるエラーをチェックします。例：3つのHuaweiルーターに接続されている場合、そのうちの1つはデフォルトでユーザーモードに接続されています。一般的なスレッドで作業するために、構成モードを特権モードに切り替えました。
