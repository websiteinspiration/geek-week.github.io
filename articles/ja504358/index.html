リソース制限を設定します。簡単な計算を使用して、実行できるアプリケーションのコピー数を計算できます。1つのコピーが1 GBのRAMを必要とし、10 GBのメモリがある場合、10のコピーを実行できます。システムのコアが規定された契約の履行を開始するだけであることを知っているので、これを監視する必要はありません。この契約、またはユーザーとシステムの間の合意は、それが利用可能であれば、すべてのツールがより適切に機能するため、非常に重要です。したがって、システムに実行の規律を導入します。



そのため、各レプリカが1 GBの空きメモリを取得した場合にのみ、スケジューラはこれを起動します。十分なメモリがない場合、プロセスは開始されません。そこで、kubectl createコマンドを入力すると、実行後にmysqlコンテナーが作成されます。



ステートフルシステムに関連する警告が1つあります-いくつかの選択肢があります。クラウドプロバイダーのPersistentDiskを使用することを示すコードを強調表示しました。



NFS、ISCSI、またはストレージデバイスへのネットワークブロックレベルのアクセスを提供するその他のプロトコルを使用できます。ストレージをマシンから切断するためにこれを行います。マシンの1つに障害が発生した場合、同じデータストアを使用して他のマシンでプロセスを再作成できます。障害が発生したホストからストレージをマウントすると、データが失われ、バックアップからすべてを再度復元する必要があります。

したがって、私たちの目標は、ネットワークの成長が速くなるにつれてストレージをより高速に実行することです。これは移行に関するものではなく、マシンの外部でストレージをすばやくマウントおよびアンマウントする機能に関するものです。これはかなり可能です。私たちの囲炉裏の状況を見てみましょう。まだ作成中です。今、他のアプリケーションが私たちのサービスを見つけられるように、サービスを作成したいと思います。



このサービスを作成するとすぐに、KubernetesがDNSレコードを生成するので、mysqlを呼び出すだけで、このコンテナーが実行されていることを自動的に検出できます。続けて、コマンド$ kubectl create –f services / mysql.yamlを入力します。ご覧のとおり、コンテナはまだ作成中です。ちなみに、このデモビデオは私のウェブサイトで見ることができます。 mysqlアプリケーションのサービスがどのように見えるかがわかります。これには、クラスターIPアドレス、外部IPアドレス、ポート番号、およびネットワークプロトコルが含まれています。



この特定のコンテナで何が起こるか見てみましょう。ご覧のとおり、動作します。



したがって、現時点では、mysqlアプリケーションが実際に機能していると思います。次に必要なのはWebアプリケーションです。そこで、「ロブスター」と呼ばれるこのようなアプリケーションをデプロイしてみましょう。GitHubで取得しました。これはHacker Newsのクローンです。これはRuby-on-Railsプロジェクトです。ここで指定したデータと基本構成に基づいてコンテナーを作成しました。



最新ではない場合：ハッカーニュースは、ハッカー会議であなたを超人気にします。ここに書かれていることを読むだけで、コンピューター技術の世界からのすべての人気のあるトピックについて議論できます。他の人を感動させたいなら、このポータルのニュースを読んでください。

だから、私はこのもののクローンを作成し、インターネットでお金を稼ぎたいと思っています。もちろん、これは実際のビジネスプロジェクトではなく、単なる機会のデモンストレーションです。
私は現在、Lobstersというアプリケーションをデプロイしています。私の秘密からデータベースのURLを取得します。そのために$ kubectl get secretsコマンドを使用します。シークレットにはユーザー名とパスワードもあります。



次に、アプリケーションと通信するコンテナーを作成します。そのために、$ kubectl create –f deployments / lobsters.yamlコマンドを使用します。ご覧のとおり、アプリケーションは実行中です。



さらに、IPアドレスがあります。 $ kubectl get svcコマンドを入力し、外部IPアドレスが101.198.12.60のページを指すグローバルロードバランサーを使用します。





ブラウザに移動し、HTTP経由でこのアドレスを入力しようとします。ええ、移行待機中にエラーが発生しました！これはRuby-on-Railsなので、そういうものを期待していました。



したがって、データベースの移行が必要です。このプロセスを1回実行する必要がありますが、それだけです。ただし、同じ方法でこれを実行したいのですが、サーバーでの承認、ジャンプボックスなどの特別なサーバーはありません。スケジューラーに連絡して、「このタスクを1回実行し、実行後にプロセスを強制終了するだけです！」つまり、コマンドを1つだけ実行して終了します。したがって、$ cat jobs / lobsters-db-schema-load.yamlコマンドを使用してバッチ処理を実行するために、このようなスキームを実装するJobsオブジェクトを作成します。



rakeコマンドフラグ「db：schema：load」はGitHub Webサイトに直接送信され、「コードイメージを取得する：kelseyhightower / lobsters：2.0.0でこのコマンドを1回実行する」と言います。コードの最後にrestartPolicy：行がないことは、Kubernetisにこれを一度だけ実行し、繰り返してはいけないことを伝えます。また、プロセッサとメモリのリソースを制限します。つまり、これを起動して実行できる適切なマシンのパラメータを示します。その後、データベース転送が完了します。そのため、$ kubectl create –f jobs / lobsters-db-schema-load.yamlコマンドを使用して、システムで実行する必要があるすべてのJobsオブジェクトを「レールに配置」します。
対応するジョブが作成されたことがわかります。その後、コマンド$ watch kubectl get jobsを入力します。



そのため、コンテナがマシンにプルされ、スケジューラが機能し、rakeタスクが作成されました。戻ってデータベースエラーページを更新してみましょう。ご覧のとおり、これでスキームが正常に実装されました。



次に、ログインする必要があります。 $ kubectl create –f jobs / lobsters-db-seed.yamlコマンドを使用します。スケジューラがまだコンテナを読み込んでいることがわかり、数秒後に作業が完了します。移行を開始するには、以前と同じレベルのコードを使用します。このページにログインすると、コンテンツを取得するだけで済みます。資金を「調達」したい場合は、コンテンツが必要です。これは、成長ハッキングがどのように見えるか、または「成長のハッキング」です。他の誰かのサイトにアクセスし、そこからコンテンツを取得して、オリジナルと同様に自分のサイトに投稿します。



しかし、コンテンツだけでなく、良いコンテンツも必要です。物事を漂わせるのは残念なことなので、手作業でニュースを借ります。コンテンツを自動的にコピーすることもできますが、これは合法ではありません。ニュースを選択し、リンクアドレスをコピーし、「テスト」タグを設定し、「このURLにあるストーリーの作成者です」チェックボックスをオンにして、[送信]ボタンを押します。盗んだニュースがどれほど素晴らしいか見てみましょう！



次に、アプリケーションをスケーリングします。これを行うには、実行していることの定義を変更するだけです。たとえば、1つのレプリカではなく、たとえば10に変更します。次に、コマンドブロックを再度実行します。



Kubernetesはこの情報を受け入れてアクションを実行します。これで、炉内でロブスターアプリケーションのコピーが10個実行されました。さらに、このプロセスは、サービスの働きにより、ロードバランサーに自動的に追加されます。



バックエンドで何が起こるか見てみましょう。これを行うには、$ kubectl get svcコマンドを使用し、短いステータスを取得して、$ kubectl describe svcコマンドで説明するように依頼します。 Kubernetesはすべてのエンドポイントを自動的に検出し、ロードバランサーの背後に配置します。



同時に、不要なものはすべて削除され、必要なものはすべて自動的に追加されます。このものを何度も作成する必要はありません。すべてがプラットフォームに完全に統合されています。
次の重要な質問は、更新方法とログの取得方法です。覚えていれば、私はあなたのSSHアクセスを取り除いたので、Log Stashや内部のGoogle Cloudロギングのようなものを使用してログを集中化する必要があります。ただし、アドホックログを表示するだけの場合は、マシンにアクセスできないことに注意してください。ただし、APIを使用して、コンテナー名を使用してログをキャプチャできます。これを行うには、コマンド$ kubectl logs lobsters-240734871-03rmn –fを入力します。ここで、03rmnはコンテナー内のlobsters-240734871アプリケーションの特定のコピーの名前です。したがって、必要に応じてトラブルシューティングできるように、レプリカを使用して各コンテナのログを表示できます。
