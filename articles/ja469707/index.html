言語サーバープロトコル -クライアントツール（IDE）とサーバー間で使用され、プログラミング言語に固有の機能（オートコンプリート、定義への移行など）を提供します。

Metals = Meta（Scalametaから）+ LS（Language Serverから-標準の言語サーバープロトコル）。

LSPは、任意のエディター、IDE、またはクライアントエンドポイントで任意のプログラミング言語を高レベルでサポートし、より単純なm-plus-nタスクを提供することで、m-times-nの複雑さを軽減します。LSPは、プログラミング言語アナライザーが話すことができる共通言語を定義するためにMicrosoftによって作成されました。 Microsoft Visual Studio Codeはそのままでこのプロトコルをサポートしています。

LSPにより、言語コミュニティは、コード補完、ツールチップ、定義への移動、リンクの検索などを提供できる1つの高性能言語サーバーに集中できます。一方、エディターとクライアントコミュニティは、単一の高性能で直感的なものの作成に集中できます。あらゆる言語サーバーと通信して、すぐに深い言語サポートを提供できる慣用的な拡張機能。

ブループ


BLOOPは、ビルドサーバーとSBTと連携してのような他のビルドツールのための実験的なサポートがあるScalaのためのCLIツールである Mavenの、 Gradleでは、およびミルを。ワークスペースに Bloop JSONファイルを含む .bloopディレクトリが含まれている場合、Metalsは自動的にそれに接続します。

Bloopの利点：

  • Scalaコードを可能な限り迅速にコンパイル、テスト、実行します。ソースコードが変更されるとコンパイルが自動的に行われ、プログラムの起動とテストで再コンパイルは必要ありません。
  • ビルドツール、コマンドラインアプリケーション、エディター、カスタムツールと簡単に統合できます。
  • IDEとは別に起動するため、その完了や再起動には依存しません。
  • Metalsは組み込みサーバーを実行する必要がないので、プロジェクトの開始時間を短縮できます。

BloopはIDEおよびテキストエディタと統合され、短いフィードバックループと信頼性の高いコンパイラ診断を提供します。

Bloopは Metalsで次の2つの方法で使用されます。
  1. Bloopがコンピューターにインストールされている場合、Metalsは既存のビルドサーバーに接続するだけです（推奨）。
  2. Bloopがインストールされていない場合、Metalsサーバーはそれをダウンロードし、ネイティブモードで実行します。

SemanticDBは、Scalaやその他の言語のプログラムに関する文字や型などのセマンティック情報のデータモデルです。プロジェクトをコンパイルした後、sbt-metalsはローカルのsemanticdb-scalaデータベースに情報を送信します。

bloopをインストールする


はじめに、最新バージョンをダウンロードしてください：

$ curl -L https://github.com/scalacenter/bloop/releases/download/v1.3.2/install.py | python

次に、現在のユーザーのbloopサービスのコピーを作成する必要があります。

$ cp ~/.bloop/systemd/bloop.service ~/.config/systemd/user/

また、スクリプトへのパスを常に書き込まないようにするには、bloopを/ usr / binにコピーします。

$ sudo cp /.bloop/bloop /usr/bin/

systemctlデーモンを再起動します。

$ systemctl --user daemon-reload

スタートアップにbloopを追加する：

$ systemctl --user enable bloop

ただし、Scalaを常に使用する予定がない場合は、このサービスをスタートアップから削除することはお勧めしません。次のコマンドで自動実行を無効にできます。

$ systemctl --user disable bloop

bloopの実行：

$ systemctl --user start bloop

アプリケーションの起動


bloopサービスが実行されている場合、プロジェクトディレクトリで次のコマンドを使用して、コンパイルされたプログラムを実行できます。

$ bloop run <proj-name>

〜/ .config / Code / User / keybindings.jsonに ホットキーを追加することもできます：

 {
    "key": "meta+r",
    "command": "workbench.action.terminal.sendSequence",
    "args": {
      "text": "bloop run ${workspaceFolderBasename}\u000D"
    }
 }

bloopサービスを実行していない場合は、sbtを介して直接アプリケーションを起動できますが、この場合、アプリケーションは毎回再コンパイルされ、プログラムの起動時間に影響します。 

$ sbt run
