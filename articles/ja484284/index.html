質問：


エントリーポイントはどこですか？

回答：


機能main()。

違い：


JavaScriptには事前定義されたエントリポイントがなく、開発者が定義した任意の関数を使用できます。ダートではそれだけmain()です。

質問：


コンソールに出力する方法は？

回答：


関数の使用print()。

違い：


JavaScriptでは、コンソール出力はを使用して行われconsole.log()ます。

例：


print('Hello world!');

質問：


変数/フィールドを作成して割り当てる方法は？

回答：


Dartは動的タイピングと強力なタイピングの両方をサポートしています。したがって、変数/フィールドは、都合のよい任意のタイプのタイピングで作成できます。代入は単一の=を通じて行われます。

違い：


JavaScriptでは厳密な型指定はサポートされていません。

追加情報：


詳細はこちら。

例：


String name = 'dart'; // Explicitly typed as a string.
var otherName = 'Dart'; // Inferred string.
// Both are acceptable in Dart.

変数/フィールドのデフォルト値は何ですか？

回答：


null

違い：


デフォルトではJavaScriptでundefined。

追加情報：

質問：


null0または0の 値を確認する方法

回答：


明示的なチェックを使用し==ます。

違い：


JavaScriptでは、チェックの一部として、if数値1または任意のnon-nullオブジェクトが同等になりtrueます。Dartでは、ブール値のみがtrue同等trueです。

例：


var myNull = null;
if (myNull == null) {
  print('use "== null" to check null');
}
var zero = 0;
if (zero == 0) {
  print('use "== 0" to check zero');
}

質問：


関数を宣言する方法は？

回答：


Dartでは、フィールドなどの関数は、戻り値の動的または厳密な型指定を使用できます。動的に型付けされたものは、関数名とオプションでパラメーターによって簡単に宣言されますが、最初に強く型付けされたものは、戻り型を持っています。

違い：


JavaScriptでは、関数はキーワードfunction、次に名前、オプションでパラメーターを使用して宣言されます。

追加情報：


詳細はこちら。

例：


fn() {
  return true;
}
// can also be written as
bool fn() {
  return true;
}

質問：


Promiseに 相当するものは何ですか？

回答：


未来

追加情報：


JavaScriptと同様に、Dartはシングルスレッド実行をサポートしています。DartのFutureは、React NativeのPromiseと同じ意味です。
詳細はこちら。。

例：


import 'dart:convert';
import 'package:http/http.dart' as http;

class Example {
  Future<String> _getIPAddress() {
    final url = 'https://httpbin.org/ip';
    return http.get(url).then((response) {
      String ip = jsonDecode(response.body)['origin'];
      return ip;
    });
  }
}

main() {
  final example = new Example();
  example
      ._getIPAddress()
      .then((ip) => print(ip))
      .catchError((error) => print(error));
}

質問：


アナログasyncとは何awaitですか？

回答：


async そして await

違い：


JavaScriptでは、非同期機能が返す約束は、ダーツで、彼らは返すの未来を。awaitは、非同期関数を呼び出した結果を同期的に期待します。

追加情報：


詳細はこちら。

例：


import 'dart:convert';
import 'package:http/http.dart' as http;

class Example {
  Future<String> _getIPAddress() async {
    final url = 'https://httpbin.org/ip';
    final response = await http.get(url);
    String ip = jsonDecode(response.body)['origin'];
    return ip;
  }
}

main() async {
  final example = new Example();
  try {
    final ip = await example._getIPAddress();
    print(ip);
  } catch (error) {
    print(error);
  }
}

質問：

回答：


Flutterでは、ウィジェットクラスを作成して再利用できます。使用されるパラメーターはクラスフィールドです。

違い：


React Nativeでは、再利用された要素のパラメーターを変更するには、それらを小道具としてマークする必要があります。

追加情報：


コンストラクターのパラメーターは、通常の列挙型とオプションの2つの方法で中括弧を使用して宣言できます{}。パラメータを使用してコンストラクタを呼び出すと、違いが生じます。
通常の列挙では、すべてのパラメーターが必要であり、コンストラクターを呼び出すときに、渡される特定のパラメーターを指定することはできません。彼らは注文によるものです。
オプションの場合、パラメーターが送信される任意の順序を使用できますが、送信する特定のパラメーターを指定するか、パラメーターをまったく送信しないようにする必要があります。オプションパラメータを必須としてマークするには、アノテーションを使用する必要があります@required。

例：


class CustomCard extends StatelessWidget {

  final Function onPress;

  Widget build(BuildContext context) {

      child: Column(
        children: <Widget>[
          Text('Card $index'),
          FlatButton(
            child: const Text('Press'),
            onPressed: this.onPress,
          ),
        ],
      )
    );
  }
}

class MyCanvasPainter extends CustomPainter {

  @override
  void paint(Canvas canvas, Size size) {

    paint.color = Colors.amber;

}

状態管理


質問：


ウィジェットの表示を更新するには？

回答：


StatefulWidgetとその状態の 使用。Flutterには、StatelessWidgetとStatefulWidgetの 2種類のウィジェットがあります。これらは同じように機能し、唯一の違いはレンダリング状態です。

違い：


StatelessWidgetには不変の状態があります。テキスト、ロゴなどの表示に適しています。それら。画面上の要素が表示時間全体で変化しない場合は、それが適しています。ステートフルウィジェットのコンテナとしても使用できます。

StatefulWidgetには、現在の状態に関する情報を格納するState state があります。なんらかのアクション（サーバーからの応答、ユーザーによるボタンのクリックなど）を実行しているときに画面上の要素を変更したい場合-これはオプションです。

例：


1）StatelessWidget-テキスト

Text(
  'I like Flutter!',
  style: TextStyle(fontWeight: FontWeight.bold),
);

2）StatefulWidget-（FloatingActionButton）ボタンをクリックすると、テキストウィジェットのテキストがI Like FlutterからFlutter is Awesomeに変わります。

import 'package:flutter/material.dart';

void main() {
  runApp(SampleApp());
}

class SampleApp extends StatelessWidget {
  //     .
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Sample App',
      theme: ThemeData(
        primarySwatch: Colors.blue,
      ),
      home: SampleAppPage(),
    );
  }
}

class SampleAppPage extends StatefulWidget {
  SampleAppPage({Key key}) : super(key: key);

  @override
  _SampleAppPageState createState() => _SampleAppPageState();
}

class _SampleAppPageState extends State<SampleAppPage> {
  //  
  String textToShow = "  Flutter";

  void _updateText() {
    setState(() {
      //  
      textToShow = "Flutter !";
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text("Sample App"),

ジェスチャーとタッチイベントの処理


質問：


クリックを処理するには？

回答：


ウィジェットがクリックをサポートしている場合、c onPressed()。そうでない場合は、GestureDetectorウィジェットを使用します。

違い：


React Nativeは、このためにPanResponderまたはTouchableを使用します。

例：


GestureDetector(
  child: Scaffold(
    appBar: AppBar(
      title: Text('Gestures'),
    ),
    body: Center(
      child: Column(
        mainAxisAlignment: MainAxisAlignment.center,
        children: <Widget>[
          Text('Tap, Long Press, Swipe Horizontally or Vertically '),
        ],
      )
    ),
  ),
  onTap: () {
    print('Tapped');
  },
  onLongPress: () {
    print('Long Pressed');
  },
  onVerticalDragEnd: (DragEndDetails value) {
    print('Swiped Vertically');
  },
  onHorizontalDragEnd: (DragEndDetails value) {
    print('Swiped Horizontally');

質問：


簡単なfade-inアニメーションを追加するには？

回答：


FadeTransitionの 使用。

例：


import 'package:flutter/material.dart';

void main() {
  runApp(Center(child: LogoFade()));
}

class LogoFade extends StatefulWidget {
  _LogoFadeState createState() => _LogoFadeState();
}

class _LogoFadeState extends State<LogoFade> with TickerProviderStateMixin {
  Animation animation;
  AnimationController controller;

  initState() {
    super.initState();
    controller = AnimationController(
        duration: const Duration(milliseconds: 3000), vsync: this);
    final CurvedAnimation curve =
    CurvedAnimation(parent: controller, curve: Curves.easeIn);
    animation = Tween(begin: 0.0, end: 1.0).animate(curve);
    controller.forward();
  }

  Widget build(BuildContext context) {
    return FadeTransition(
      opacity: animation,
      child: Container(
        height: 300.0,
        width: 300.0,
        child: FlutterLogo(),
      ),
    );
  }

  dispose() {
    controller.dispose();
    super.dispose();
  }
}
