Tetris sur les tableaux de bord: les vieilles chansons d'une manière nouvelle



Bitbordy ( Bitboard ) - modèles de bits spéciaux pour compter efficacement les mouvements dans les jeux de société. Sur un habr a écrit sur l'application des bitboards aux échecs et même aux dames . Aujourd'hui, nous appliquerons la technique des bitboards à un jeu quelque peu inattendu mais familier - à Tetris. Le résultat de nos recherches sera un jeu sur console, ainsi qu'une recherche automatique des meilleurs mouvements (pour une séquence donnée de pièces), dont la vitesse est juste assurée par l'efficacité des manipulations de bits. Dans le même temps, nous prendrons en charge la lecture des mouvements trouvés en mode automatique afin de profiter pleinement de l'intelligence informatique. À la fin de l'article, un lien vers le github avec le code du jeu en C # est donné, ainsi qu'un court jeu vidéo de 114 coups trouvés par l'ordinateur en cherchant en profondeur pendant quinze minutes.

Habituellement, un bitboard est un mot machine composé de plusieurs octets, dont chaque bit correspond à une cellule du champ dans le jeu. Ainsi, aux échecs, il n'y a que 64 cellules, ce qui correspond au mot de 8 octets (ulong en C #), et dans les dames - 32 (uint en C #). Les fans de Tetris se souviendront probablement que la taille du champ dans Tetris standard est de 10 par 20 cellules, soit 200 bits, ce qui ne correspond à aucun type de nombre. Bien sûr, vous pouvez diviser le champ en quatre parties et utiliser quatre mots de huit octets, ou vous ne pouvez pas jouer et utiliser un tableau de vingt-deux mots de deux octets, un mot pour chaque ligne du champ; toutes les implémentations de tetris sur les bitboards que j'ai trouvées (au montant d'une seule pièce) font exactement cela. Mais nous irons dans l'autre sens ...

Vecteurs


Nous utiliserons un ensemble spécial d'instructions Advanced Vector Extensions 2 , que nous avons déjà utilisé la dernière fois pour accélérer les calculs. Ils fonctionnent avec des valeurs 256 bits (contrairement aux instructions habituelles de travail avec des valeurs 64 bits):

//  , 8 
ulong 
    x = 0,
    y = 0,
    z = x & y; // z[i] = x[i] & y[i], i = 0..7
// SIMD-, 32 
Vector256<byte> 
    X = Vector256<byte>.Zero,
    Y = Vector256<byte>.Zero,
    Z = Avx2.And(X, Y); // Z[i] = X[i] & Y[i], i = 0..31

Je dois dire que l'écriture d'appels de méthode statique au lieu d'opérations arithmétiques à chaque fois n'est pas très pratique; en outre, toutes les opérations n'ont pas d'analogues directs dans AVX2, nous allons donc utiliser la bande d' alimentation c # et écrire une structure spéciale qui encapsule un mot de 256 bits pour écrire du code dans ce style:

V256 X = 0, Y = 0, Z = X & Y;

Pour rendre le code précédent significatif, nous définissons V256 comme une structure qui stocke un vecteur de données de 256 bits à l'intérieur et a des constructeurs implicites et une surcharge d'opérateur:

readonly unsafe struct V256
{
    private readonly Vector256<byte> _data;

    public V256(Vector256<byte> data) { _data = data; }

    //    Vector<256>:

//  readonly,   JIT-  
readonly unsafe struct V256
{
    //  ,    AVX2
    static V256() { if (!Avx2.IsSupported) throw new NotSupportedException("AVX2"); }

    //         , . 
    public static readonly V256 ZERO = 0;

    //  . JIT-       ,
    //         256- 
    //   YMM0 - YMM15
    private readonly Vector256<byte> _data;

    //   32    
    public V256(byte* b) { _data = Avx2.LoadVector256(b); }

    //      32   
    public V256(params byte[] buffer)
    {
        if (buffer.Length < 32) Array.Resize(ref buffer, 32);
        fixed (byte* p = buffer) _data = Avx2.LoadVector256(p);
    }

    //      .  
    //    unmanaged,      T
    //    .    (int, uint, long, ulong,
    //  ..)   
    public static V256 Load<T>(params T[] buffer) where T : unmanaged
    {
        fixed (T* p = buffer) return new V256((byte*)p);
    }

    public void Store<T>(T[] buffer) where T : unmanaged
    {
        fixed (T* p = buffer) Avx2.Store((byte*)p, _data);
    }

    public T[] ToArray<T>() where T : unmanaged
    {
        T[] array = new T[32 / sizeof(T)];
        fixed (T* ptr = array) Avx2.Store((byte*)ptr, _data);
        return array;
    }

    //   broadcast     
    //  ,   32   b
    public static V256 FromByte(byte b) =>
        Avx2.BroadcastScalarToVector256(&b);
    //  16  short- s
    public static V256 FromShort(short s) =>
        Avx2.BroadcastScalarToVector256(&s).AsByte();
    //      
    public static V256 FromULong(ulong ul) =>
        Avx2.BroadcastScalarToVector256(&ul).AsByte();

    //    :    V256 x = 42;
    public static implicit operator V256(byte b) => FromByte(b);

    //   AVX2
    //     ,  result[i] = 
    // Math.Min(this[i], other[i]), i=0..31,   100  
    public V256 Min(V256 other) => Avx2.Min(_data, other._data);
    //  table lookup: result[i] = this[data[i]], i = 0..31
    public V256 Shuffle(V256 mask) => Avx2.Shuffle(_data, mask._data);
    //  result[i] = this[i] == other[i] ? 0xFF : 0x00;
    public V256 CompareEqual(V256 other) => Avx2.CompareEqual(_data, other._data);
    
    //     x & y, x | y  .. 
    public static V256 operator &(V256 x, V256 y) => Avx2.And(x._data, y._data);
    public static V256 operator |(V256 x, V256 y) => Avx2.Or(x._data, y._data);
    public static V256 operator +(V256 x, V256 y) => Avx2.Add(x._data, y._data);
    public static V256 operator -(V256 x, V256 y) => Avx2.Subtract(x._data, y._data);
    public static V256 operator -(V256 x) => Avx2.Subtract(ZERO._data, x._data);

    //      256-   .
    //      AVX2  AVX512,   
    //   ,    :
    public static bool operator ==(V256 x, V256 y) =>
        -1 == Avx2.MoveMask(Avx2.CompareEqual(x._data, y._data));

    //    -  .  ,    
    //  256-  (   AVX512,  ,  !), 
    // :
    public static V256 operator >>(V256 x, int count)
    {
        //  ,  4  8- 
        Vector256<ulong> data = Avx2.ShiftRightLogical( 
            x._data.AsUInt64(), (byte)count);
        //        
        Vector256<ulong> carry = Avx2.ShiftLeftLogical(
            x._data.AsUInt64(), (byte)(64 - count));
        // ...
        carry = Avx2.Permute4x64(carry, 0x39);
        //   !
        carry = Avx2.Blend(ZERO._data.AsUInt32(), carry.AsUInt32(), 0x3F).AsUInt64();
        data = Avx2.Or(data, carry);
        return data.AsByte();
    }

    //   ,   ,  .
    //       :   
    //   ,     ,
    //      .
    public override string ToString()
    {
        byte* buffer = stackalloc byte[32];
        Avx2.Store(buffer, _data);
        return string.Join(" ", Enumerable.Range(0, 32).Select(i => buffer[i].ToString("X2")));
    }
}

Eh bien, maintenant balancez votre main, collez vos bras autour de manipuler doucement les bits!
Tout d'abord, nous devons déterminer l'intersection des cellules remplies du champ et de la forme, ce qui se fait en une seule opération logique:

bool Intersects(V256 board, V256 piece) => (piece & board) != 0;

Décaler un chiffre vers le bas, la gauche ou la droite sera simplement un décalage logique du nombre de bits correspondant (notez que le décalage d'un chiffre vers la droite est un décalage de ses bits vers le haut, ce qui correspond à un décalage logique des bits du vecteur vers la gauche):

const int width = 12; //    

V256 MoveDown(V256 piece) => piece <<= width;

V256 MoveRight(V256 piece) => piece <<= 1;

V256 MoveLeft(V256 piece) => piece >>= 1;

//      ,
//  ,     
V256 Drop(V256 piece)
{
    while(!Intersects(board, MoveDown(piece)) piece = MoveDown(piece);
    return piece;
}

Enfin, la partie la plus difficile est d'identifier les lignes remplies et de les supprimer. Ici, vous ne pouvez pas vous passer d'une boucle (au moins, je n'ai pas réussi), nous allons donc balayer le champ le long des lignes de bas en haut et vérifier avec des opérations binaires que la ligne est pleine. Pour supprimer la ligne remplie du champ, réinitialisez au niveau du bit la ligne correspondante dans le champ et déplacez la partie supérieure vers le bas:

Méthode entière 
V256 fullLine = new V256(0, 0, 0, ..., 0, 0, 255, 15);
V256 emptyLine = new V256(1, 8);

void RemoveFullLines(ref V256 board)
{
    //     
    V256 lineMask = fullLine >> width;
    //      
    V256 underLineMask = fullLine;
    while (lineMask != 0)
    {
        //  ,   
        while ((board & lineMask) == lineMask)
        {
            //        
            board = (board & underLineMask) | 
                        ((board & ~underLineMask & ~lineMask) << width) | 
                        emptyLine;
        }
        //      
        underLineMask |= lineMask;
        lineMask >>= width;
    }
}

Rechercher la meilleure stratégie


En fait, pour le jeu, il ne serait pas très logique de clôturer ces trucs. Mais pour trouver la stratégie optimale - c'est tout! Faisons l'option de recherche la plus simple - une recherche récursive en profondeur par une recherche exhaustive: étant donné que la séquence des figures qui tombent est fixe, alors pour chaque figure, nous trions toutes les variantes possibles de ses rotations et décalages horizontaux. En parallèle, nous stockons la séquence d'actions afin de pouvoir reproduire les options trouvées. Le code simplifié est assez simple:
