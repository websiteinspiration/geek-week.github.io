SwiftUIのプロパティラッパーについて

記事の翻訳は、コース「iOS Developer。上級コースv 2.0。」



先週、SwiftUIフレームワークに関する一連の新しい投稿を開始しました。今日は、SwiftUIのプロパティラッパーについて話し、このトピックを続けたいと思います。SwiftUIは私たちに、ラッパー特性を与える@State、@Binding、@ObservedObject、@EnvironmentObjectと@Environment。それでは、それらの違いと、いつ、なぜ、どれを使用するべきかを理解してみましょう。

プロパティラッパー


プロパティラッパー（以下、「プロパティラッパー」と呼びます）は、SE-0258で説明されています。主なアイデアは、プロパティをロジックでラップすることです。ロジックを別の構造に抽出して、コードベースで再利用できます。

状態


@State状態を示すために使用できるラッパーですView。SwiftUIは、構造体の外の特別な内部メモリにそれを保存しViewます。接続しているView人だけが彼女にアクセスできます。プロパティ値@Stateが変更されるとすぐに、SwiftUIはView状態の変化を考慮して再構築します。これは簡単な例です。

struct ProductsView: View {
    let products: [Product]

    @State private var showFavorited: Bool = false

    var body: some View {
        List {
            Button(
                action: { self.showFavorited.toggle() },
                label: { Text("Change filter") }
            )

            ForEach(products) { product in
                if !self.showFavorited || product.isFavorited {
                    Text(product.title)
                }
            }
        }
    }
}

上記の例では、ボタンと製品のリストが表示されたシンプルな画面があります。ボタンをクリックするとすぐに、stateプロパティの値が変更され、SwiftUIが再構築しViewます。

@Binding


@Binding値タイプの参照によるアクセスを提供します。時には、私たちViewは彼の子供たちに自分の州をアクセス可能にする必要があります。しかし、これは値の型なので、この値を取得して渡すことはできません。Swiftはこの値のコピーを渡します。ここで、プロパティラッパーが役に立ちます@Binding。

struct FilterView: View {
    @Binding var showFavorited: Bool

    var body: some View {
        Toggle(isOn: $showFavorited) {
            Text("Change filter")
        }
    }
}

struct ProductsView: View {
    let products: [Product]

    @State private var showFavorited: Bool = false

    var body: some View {
        List {
            FilterView(showFavorited: $showFavorited)

            ForEach(products) { product in
                if !self.showFavorited || product.isFavorited {
                    Text(product.title)
                }
            }
        }
    }
}

@ObservedObject


@ObservedObjectこれはと同様に機能します@Stateが、主な違いは、Viewサブスクライブしてこのオブジェクトの変更を監視できる複数の独立者間で分割でき、変更が表示されるとすぐに、SwiftUIこのオブジェクトに関連付けられているすべてのビューが再構築されることです。例を見てみましょう。

import Combine

final class PodcastPlayer: ObservableObject {
    @Published private(set) var isPlaying: Bool = false

    func play() {
        isPlaying = true
    }

    func pause() {
        isPlaying = false
    }
}

ここにPodcastPlayer、アプリケーション画面が共有するクラスがあります。アプリケーションがポッドキャストエピソードを再生しているとき、各画面にはフローティング一時停止ボタンが表示されます。SwiftUIトラックはに変わりObservableObjectラッパーを使用して@Published、すぐとしてマークされたプロパティとしてとして@Published変化し、それがSwiftUIすべて再構築しView、このオブジェクトに関連しますPodcastPlayer。ここでは、ラッパーを使用してクラスをクラス@ObservedObjectにバインドしますEpisodesViewPodcastPlayer

class SceneDelegate: UIResponder, UIWindowSceneDelegate {

@環境


前の章で述べたように、カスタムオブジェクトをSwiftUIEnvironment View内の階層で転送できます。しかし、SwiftUIにはすでに1つあり、システム全体の設定で満たされています。ラッパーで簡単にアクセスできます。Environment@Environment

struct CalendarView: View {
    @Environment(\.calendar) var calendar: Calendar
    @Environment(\.locale) var locale: Locale
    @Environment(\.colorScheme) var colorScheme: ColorScheme

    var body: some View {
        return Text(locale.identifier)
    }
}

プロパティをラッパー@Environmentでマークすることにより、システム全体の設定へのアクセスにアクセスし、変更をサブスクライブします。すぐにロケール、カレンダーやのColorSchemeのシステムの変更、SwiftUIは私たちを再現しますCalendarView。
