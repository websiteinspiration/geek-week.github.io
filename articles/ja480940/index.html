キュウリとセレノイドを使用したクロスブラウザーUIテストをGitlab CIで実行します。

Cucumberは、ソフトウェア開発者とビジネスマネージャーの間のギャップを埋めるのに役立つテストインフラストラクチャです。テストは、Given、When、Then（条件、操作、結果）のスタイルで単純な動作駆動型開発（BDD）言語で記述されており、すべてのユーザーが理解できます。次に、制御テストは、1つ以上のテストスクリプトにまたがる関数ファイルに書き込まれます。Cucumberは指定されたプログラミング言語でテストを解釈し、Seleniumを使用してブラウザーでテストを管理します。


SelenoidはSelenium Serverの代替ソリューションですが、本質は同じですが、ドライバーの編成です。


この投稿では、Gitlab CIでCucumberとSelenoidを使用した簡単なブラウザテストを開始し、最後にAllureレポートを表示します。


トレーニング


Dockerがインストールされ、オペレーティングシステムで実行されている必要があります。


Selenoidをインストールする


Redhatベースのオペレーティングシステムを使用している場合は、私のリポジトリを使用して構成マネージャーをインストールできます。


yum -y install yum-plugin-copr
yum copr enable antonpatsev/aerokube-cm-rpm
yum -y install aerokube-cm

Redhatベースのオペレーティングシステムがない場合は、Configuration Managerバイナリをダウンロードして使用できます。


構成マネージャーを使用してSelenoidを実行し、browsers.jsonを構築する


インターネットとDockerイメージに直接アクセスできない場合は、レジストリからダウンロードします。


aerokube-cm selenoid start --force --browsers "firefox:70.0;firefox:71.0;chrome:78.0;chrome:79.0" --registry -docker-registry

インターネットに直接アクセスできる場合。


aerokube-cm selenoid start --force --browsers "firefox:70.0;firefox:71.0;chrome:78.0;chrome:79.0"

キー--forceはbrowsers.jsonファイルを上書きします。

Gitlab CI


Selenoid Selenoid-UI , Selenoid Selenoid-UI Gitlab Runner Ansible, Puppet, Chef .


.gitlab-ci.yml


---
variables:
  MAVEN_OPTS: "-Dhttps.protocols=TLSv1.2 -Dmaven.repo.local=/home/gitlab-runner/.m2/repository -Dorg.slf4j.simpleLogger.log.org.apache.maven.cli.transfer.Slf4jMavenTransferListener=WARN -Dorg.slf4j.simpleLogger.showDateTime=true -Djava.awt.headless=true"
  MAVEN_CLI_OPTS: "--batch-mode --errors --fail-at-end --show-version -DinstallAtEnd=true -DdeployAtEnd=true"

before_script:
  - export PATH="/opt/rh/rh-maven33/root/usr/bin:$PATH"

build:
  stage: build
  script:
    - aerokube-cm selenoid start --force --browsers "firefox:70;firefox:71;chrome:78;chrome:79" --registry docker-registry
    - aerokube-cm selenoid-ui start --registry docker-registry
    - cat ~/.aerokube/selenoid/browsers.json
    - set +e
    - mvn -s maven_settings.xml clean install -Dmaven.test.skip=true
    - mvn -s maven_settings.xml clean test || EXIT_CODE=$?
    - mvn -s maven_settings.xml allure:aggregate;
    - export PATH_WITHOUT_HOME=$(pwd | sed -e "s/\/home\/gitlab-runner\/builds//g")
    - echo '***********************************************************************'
    - echo "http://$HOSTNAME:9090$PATH_WITHOUT_HOME/target/site/allure-maven-plugin/"
    - echo '***********************************************************************'
    - set -e
    - exit ${EXIT_CODE}

echo "http://$HOSTNAME:9090$PATH_WITHOUT_HOME/target/site/allure-maven-plugin/"

    }
