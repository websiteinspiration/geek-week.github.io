従来の方法


前回の記事では、シンプルなHello Worldアプリケーションを作成しました。

ただリマインダー-コードは次のようになりました：

@Override
public void start(Stage primaryStage) throws Exception {
    primaryStage.setTitle("Hello world Application");
    primaryStage.setWidth(300);
    primaryStage.setHeight(200);

    InputStream iconStream = getClass().getResourceAsStream("/icon.png");
    Image image = new Image(iconStream);
    primaryStage.getIcons().add(image);

    Label helloWorldLabel = new Label("Hello world!");
    helloWorldLabel.setAlignment(Pos.CENTER);
    Scene primaryScene = new Scene(helloWorldLabel);
    primaryStage.setScene(primaryScene);

    primaryStage.show();
}

ご覧のとおり、ユーザーインターフェイス全体がJavaコードで作成されています。

これは非常に単純な例ですが、アプリケーションがより複雑になると、ネストされたレイアウトと多くのコンポーネントのいくつかのレベルを入力する必要がある場合、結果のコードを理解するのが非常に難しくなることがあります。ただし、これだけではありません。同じクラスに、構造、視覚効果、および動作を同時に担当するコードがあります。

クラスには明らかに単一の責任はありません。これを、たとえば、各ページに明確に分離されたタスクがあるWebインターフェースと比較してください。

  • HTMLは構造です
  • CSSは視覚効果です
  • JavaScriptは動作です

FXMLの紹介


明らかに、すべてのコードを1か所に置くのは良い考えではありません。理解しやすく、管理しやすくするために、どうにかして構造化する必要があります。

実際には、これには多くの設計パターンがあります。通常、「モデルビューコントローラー」、「モデルビュープレゼンター」、「モデルビュービューモデル」などの「モデルビュービュー」オプションが表示されます。

さまざまなオプションの長所と短所について何時間も話し合うことができます。ここでは行わないでください。さらに重要なのは、JavaFxを使用すると、それらのいずれも使用できることです。

これが可能なのは、ユーザーインターフェイスの手順設計に加えて、宣言型XMLマークアップを使用できるためです。

XMLの階層構造は、ユーザーインターフェイスでコンポーネントの階層を記述する優れた方法であることがわかります。HTMLはかなりうまく機能しますよね？

JavaFX固有のXML形式はFXMLと呼ばれます。その中で、すべてのアプリケーションコンポーネントとそのプロパティを定義し、それらをコントローラーと関連付けることができます。コントローラーは、相互作用の管理を担当します。

FXMLファイルをダウンロードする


では、FXMLで動作するように起動方法を変更するにはどうすればよいでしょうか。

FXMLLoader loader = new FXMLLoader();
URL xmlUrl = getClass().getResource("/mainScene.fxml");
loader.setLocation(xmlUrl);
Parent root = loader.load();

primaryStage.setScene(new Scene(root));
primaryStage.show();

ここでrootはユーザーインターフェイスのルートコンポーネントを表し、他のコンポーネントはその中にネストされています。load

メソッドには一般的な戻り値があるため、Parentではなく特定のタイプを指定できます。次に、コンポーネント指向のメソッドにアクセスします。ただし、これによりコードがより脆弱になります。FXMLでルートコンポーネントのタイプを変更すると、アプリケーションは実行時に動作を停止する場合がありますが、コンパイル中にエラーは発生しません。これは、FXMLとJava FXMLローダーで宣言された型の不一致があるためです。

FXMLファイルの作成


これでFXMLファイルをロードする方法はわかりましたが、それでも作成する必要があります。ファイルには拡張子.fxmlが必要です。Mavenプロジェクトでは、このファイルをリソースフォルダーに配置するか、FXMLLoaderで外部URLからダウンロードできます。

ファイルを作成したら、最初の行にXML宣言を入力します。

<?xml version="1.0" encoding="UTF-8"?>

インポート


個々のコンポーネントをファイルに追加する前に、それらが正しく認識されていることを確認する必要があります。これを行うには、インポート文を追加します。これは、Javaクラスでのインポートとよく似ています。通常どおり、個々のクラスをインポートするか、ワイルドカードを使用できます。インポートセクションの例を見てみましょう。
<?import java.lang.*?>
<?import java.util.*?>
<?import javafx.scene.*?>
<?import javafx.scene.control.*?>
<?import javafx.scene.layout.*?>

すべてのインポート文を手動で追加する代わりに、IDEはJavaクラスにインポートするようにインポートを追加するのに役立つはずです。

コンポーネントの追加


次に、いくつかのコンポーネントを追加します。では前の記事、私たちは、それぞれのシーンが一つだけの子コンポーネントを持つことができることを学びました。まず、簡単なラベルを追加しましょう：

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.control.Label?>

<!--      -->
<Label>Hello World!</Label>

もちろん、ルートコンポーネントとしてのラベル付けは、あまり現実的な例ではありません。通常、いくつかの種類のレイアウト（レイアウト）を使用することをお勧めします。これは、いくつかのコンポーネントのコンテナーであり、それらの場所を編成します。このシリーズの後半でレイアウトについて説明しますが、ここでは、子を縦に重ねて配置する単純なVBoxを使用します。

<?xml version="1.0" encoding="UTF-8"?>

<?import javafx.scene.control.Label?>
<?import javafx.scene.layout.VBox?>
<?import javafx.scene.control.Button?>

<VBox>
    <Label text="Hello world!"/>
    <Label text="This is a simple demo application."/>
    <Button text="Click me!"/>
</VBox>

FX名前空間


デフォルトでは使用できないFXML要素と属性がいくつかあります。名前空間FXMLを使用可能にするには、それらを追加する必要があります。ルートコンポーネントに追加する必要があります。

<?xml version="1.0" encoding="UTF-8"?>
    ...

<VBox xmlns="http://javafx.com/javafx"
      xmlns:fx="http://javafx.com/fxml">
    ...
</VBox>

これで、fx名前空間の新しい要素を使用できます。コンポーネントに一意の識別子を追加してみましょう。

<Label fx:id="mainTitle" text="Hello world!"/>

fx：id 属性は、FXMLの他の部分やコントローラーからのコンポーネントを参照するために使用できるコンポーネントの一意の識別子です。

スクリプト


私たちのアプリケーションはまだ静的です。いくつかのラベルとボタンがありますが、アプリケーションは動的に何もしません。

ボタンのクリックに応答して、「Click me！」でタイトルを変更してみましょう。「もう一度クリックしてください！」

最初に、ボタンにonActionイベントハンドラーを追加します。

<Button fx:id="mainButton" text="Click me!" onAction="buttonClicked()"/>

fx：idに注意してください。これは、後でボタンを参照するために使用される識別子です。

次に、イベントを処理するために呼び出される関数を提供する必要があります。fx：スクリプトタグ内で定義できます。重要なのは、さまざまな言語を使用してスクリプト、JavaScript、Groovy、Clojureを記述できることです。JavaScriptの例を見てみましょう。次のよう



に宣言されたmainButton識別子を使用して、Buttonコンポーネントを参照していることに注意してください。

fx:id = "mainButton"

また、使用するスクリプト言語をFXMLファイルで示す必要があります。

<?language javascript?>

例の全文を見てみましょう：



これを使用する必要がありますか？


上記の例は、fx：idを使用してコンポーネントを参照する方法と、JavaScriptスクリプトを使用して単純な動作を追加する方法を示しています。それは本当にあなたがしなければならないことですか？

答えはほとんどの場合ノーです。このアプローチにはいくつかの問題があります。FXMLが導入された理由は、ユーザーインターフェイスの構造と動作を分離するために、関心の分離でした。このスクリプトでは、ユーザーインターフェイス構造とマージされた動作が再び返されました。さらに、JavaコードではなくXMLで作業するため、コンパイル時のすべてのコードチェックと型の安全性が失われました。これで、アプリケーションのすべての問題が、コンパイル時ではなく実行時に検出されます。アプリケーションは非常に壊れやすくなり、エラーが発生しやすくなります。

コントローラーの追加


では、利益を明確に分離するために何ができるでしょうか？コントローラをFXMLファイルにリンクできます。コントローラーは、アプリケーションでの動作とユーザー操作の処理を担当するJavaクラスです。このようにして、型の安全性とコンパイル時のチェックを返すことができます。

コントローラはPOJOであり、何かを拡張または実装したり、特別な注釈を付けたりしてはなりません。

コントローラークラスをFXMLに関連付けるにはどうすればよいですか？基本的に2つのオプションがあります。

Javaで


コントローラーのインスタンスを自分で作成するか、依存関係の注入など、インスタンスを作成する他の方法を使用できます。次に、FXMLLoaderをダウンロードします。

FXMLLoader loader = new FXMLLoader();
loader.setController(new MainSceneController());

FXML


コントローラーのクラスを属性fx：controllerとして指定できます。これはルートコンポーネントにある必要があります。

<VBox xmlns="http://javafx.com/javafx"
      xmlns:fx="http://javafx.com/fxml"
      fx:controller="com.vojtechruzicka.MainSceneController">
    ...
</VBox>

FXMLでControllerクラスを宣言すると、自動的に作成されます。このアプローチには1つの制限があります。コントローラーでは、コントローラークラスの新しいインスタンスを簡単に作成できるように、引数なしでコンストラクターを作成する必要があります。

自動的に作成されたコントローラのインスタンスにアクセスするには、FXMLローダーを使用できます。

FXMLLoader loader = new FXMLLoader();
loader.setLocation(getClass().getResource("/mainScene.fxml"));
MainSceneController controller = loader.getController();

コントローラメソッドの呼び出し


これでコントローラーができたので、スクリプトを削除して、コントローラーで直接ボタンを押すロジックを実装できます。

public class MainSceneController {
    
    public void buttonClicked() {
        System.out.println("Button clicked!");
    }
}

次のステップは、このメソッドの呼び出しをボタンのonActionイベントハンドラーとして登録することです。コントローラーからメソッドを参照するには、メソッド名の前に＃記号を使用する必要があります。

<VBox xmlns="http://javafx.com/javafx"
      xmlns:fx="http://javafx.com/fxml" fx:controller="com.vojtechruzicka.MainSceneController">
    <Label fx:id="mainTitle" text="Hello world!"/>
    <Label fx:id="subTitle" text="This is a simple demo application."/>
    <Button fx:id="mainButton" text="Click me!" onAction="#buttonClicked"/>
</VBox>

ボタンがクリックされると、MainSceneController.buttonClicked（）メソッドを呼び出します。これは、メソッドがパブリックとして宣言されている場合にのみ機能することに注意してください。アクセス修飾子がより厳密な場合は、メソッドに@FXMLアノテーションを付ける必要があります。

@FXML
private void buttonClicked() {
    System.out.println("Button clicked!");
}

コントローラにコンポーネントを埋め込む


これまでのところ、コンソールに出力しているだけです。ボタンのテキストをもう一度「もう一度クリックしてください」に変更したい場合はどうなりますか？コントローラ内のコンポーネントへのリンクを取得するにはどうすればよいですか？

幸い、これは簡単です。これらのfx：id属性を覚えていますか？

<Button fx:id="mainButton" text="Click me!" onAction="#buttonClicked"/>

JavaFXは、コンポーネントをfx：idと同じ名前でコントローラーに定義されたフィールドと自動的に照合しようとします。

上記のボタンがあるとします

fx:id="mainButton"

JavaFXは、ボタンオブジェクトをコントローラーのmainButtonというフィールドに挿入しようとします。

public class MainSceneController {

    //    fx:id = "mainButton"
    @FXML
    private Button mainButton;
}

前のメソッドと同様に、フィールドはパブリックであるか、注釈付きの@FXMLである必要があります。

ボタンへのリンクが作成されたので、テキストを簡単に変更できます。

public class MainSceneController {

    @FXML
    private Button mainButton;

    @FXML
    private void buttonClicked() {
        mainButton.setText("Click me again!");
    }
}

シーンビルダー


GUI構造をXMLで作成することは、Javaよりも自然な場合があります（特にHTMLに精通している場合）。しかし、それはまだ非常に便利ではありません。良い知らせは、ユーザーインターフェースの作成に役立つScene Builderと呼ばれる公式のツールがあることです。簡単に言うと、これはGUIのグラフィカルエディターです。



エディターには3つの主要な領域があります。

  1. 左側の部分には、中央の部分にドラッグできる使用可能なコンポーネントが表示されます。また、ユーザーインターフェイスのすべてのコンポーネントの階層も含まれているため、簡単にナビゲートできます。
  2. 中央の部分は、FXMLファイルに基づいて表示されるアプリケーションです。
  3. 右側は現在のコンポーネントインスペクターです。ここでは、選択した現在のコンポーネントのさまざまなプロパティを編集できます。階層の中央で選択されたコンポーネントは、インスペクターに表示されます。

スタンドアロン


Scene Builderは、FXMLファイルの編集に使用できるスタンドアロンアプリケーションとしてダウンロードできます。

IntelliJ IDEAとの統合


または、シーンビルダーはIDEとの統合を提供します。

IntelliJ IDEAでは、FXMLファイルを右クリックして、SceneBuilderの[開く]メニューオプションを選択できます。

または、IntelliJ IDEAはSceneBuilderを直接IDEに統合します。IDEAでFXMLファイルを開くと、画面の下部に2つのタブが表示されます。

  • テキスト
  • シーンビルダー

FXMLファイルごとに、直接またはSceneBuilderを介してFXMLファイルの編集を簡単に切り替えることができます。



IntelliJ IDEAでは、SceneBuilder実行可能ファイルの場所を構成できます。

Settings → Languages & Frameworks → JavaFX → Path to SceneBuilder
