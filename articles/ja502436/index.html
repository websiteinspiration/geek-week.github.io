CI TeamCity-AndroidおよびUIテストのビルドプロセスの自動化

この記事では、TeamCityをインストールして構成し、WindowsでのAndroidプロジェクトのアセンブリとテストを自動化する手順を説明します。

また、Androidプロジェクトの環境をセットアップする機能（調査プロセスで見つけたもの）や、初心者のテスターや開発者の生活を楽にするのに役立つさまざまな種類の機能にも注目します。

目的：


プロジェクトコードを更新すると、次のようになります。

  • 自動ビルドプロジェクト
  • UI自動テストを実行する
  • 後続の手動テストのためのデバッグおよびリリースAPKファイルのエクスポート
  • すべての開発チームメンバーに結果を通知する

予定：


  1. Java JDKをインストールして構成する
  2. Android SDKをインストールして構成する
  3. デバッグ用に別のGradleを設定する
  4. Androidプロジェクトの準備
  5. TeamCityサーバーのインストールとエージェントの構築
  6. TeamCityプロジェクトのセットアップ→プロジェクトビルド用のビルドとインストールAPKの取得
  7. UI自動テストでビルドステップを構成する

6.2。 TeamCityビルド構成

プロジェクトをコンパイルしてAPKを表示するビルドステップを追加します。このプロジェクトでは、Gradleを使用します。

7. UI

自動テストを使用したビルドステップの設定最も興味深い部分、つまりUIテストの自動起動に到達しました。

ステップは.BATファイルを使用して起動されます。コードを記述したり、TeamCity UIに移動しなくてもその場で変更したりする方がはるかに便利です。

7.1。まず、エミュレーターを実行した状態でビルドステップを作成します。

BATnikへのURLを完全に作成できます。便宜上、BAT_FILESBATが含まれるフォルダーへのパスを使用して環境をTeamCity に



作成しました。エミュレーターの開始コードをファイルに記述します。

例：

start /min "emulator" /D "C:\Users\1\AppData\Local\Android\Sdk\emulator" emulator.exe -avd Pixel_2_API_25 -no-snapshot-save -no-boot-anim
オフサイトのコマンドとパラメーターのリスト：リンク。

7.2。エミュレーターの読み込み中に一時停止するビルドステップを作成します。 .BATで最初のステップをすぐに登録できますが、より明確になります。

インターネットをスパイしているそのようなチームの例：

adb wait-for-device shell 'while [[ -z $(getprop sys.boot_completed) ]]; do sleep 1; done; input keyevent 82'

UIからは次のようになります：



オフサイトのすべてのチームのリスト：リンク。

7.3。テストを実行します。

コマンドラインからも。ここではすべてが個人的です。

コマンドの例：

7.4。エミュレータを停止します。

コマンド例：

adb -s emulator-5554 emu kill

プロセスが完了しました。

別のステップでレポートをアップロードすることもできます。デフォルトでは、プロジェクトフォルダーに保存されます。
