ZeroTierを搭載。仮想ネットワークを構築するための実用的なガイド。パート2

Powered by ZeroTierの記事に記載されている最初の5つのステップ。仮想ネットワークを構築するための実用的なガイド。パート1では、地理的に離れた3つのノードの仮想ネットワークを接続しました。そのうちの1つは物理ネットワークにあり、他の2つは2つの間隔のあるDCにあります。  


これらの各ノードが一度に1つずつネットワークに追加されましたが、これにはそれほど時間はかかりませんでした。しかし、ZeroTier仮想ネットワークを1つだけでなく、物理ネットワークにあるすべてのノードにも接続する必要がある場合はどうでしょうか。このような問題は、仮想ネットワークからネットワークプリンターおよびルーターへのアクセスを整理する問題に困惑したときに、かつて私に直面しました。 

私は上記の方法を使おうとしました-それは速くなく、いつも簡単ではないことが判明しました。たとえば、ネットワークプリンタ-接続しないでください。 Mikrotik-ZeroTierはサポートしていません。何をすべきか？たくさんググってハードウェアを分析した結果、ネットワークブリッジを編成する必要があるという結論に達しました。

ネットワークブリッジ（も橋英語から ブリッジ）-コンピュータネットワークのセグメント（サブネット）を単一のネットワークに結合するように設計された、OSIモデルの第2レベルのネットワークデバイス。

この記事で私がこれをどのように行ったかについての話を共有したいと思います。 

橋を架けるのにいくらかかりますか...


まず、管理者として、ネットワーク上のどのノードがブリッジとして機能するかを決定する必要がありました。オプションを検討した結果、ネットワークインターフェイス間のブリッジを構成する機能を備えた任意のコンピューターデバイスであることがわかりました。ルーター（OpenWRTまたはTeltonika RUTシリーズの機器を実行しているデバイス）か、通常のサーバーまたはコンピューターのいずれかです。 

最初はもちろん、OpenWRTを搭載したルーターの使用を検討しました。しかし、既存のMikrotikは完全に私に適していますが、ZeroTierとの統合はサポートしておらず、私は変態したくないし、「タンバリンでダンス」したくないので、コンピューターをネットワークブリッジとして使用することにしました。つまり、Raspberry Pi 3モデルBは、最新バージョンのRaspbian（Debianバスターベースのオペレーティングシステム）の下で常に物理ネットワークに接続されています。

ブリッジを構成できるようにするには、他のサービスで使用されていない1つのネットワークインターフェイスをデバイスで使用できる必要があります。私の場合、メインのイーサネットはすでに使用されていたので、2番目のイーサネットを編成しました。このタスクでは、Realtek RTL8152チップセットに基づくUSB-Ethernetアダプターを使用します。

アダプターを空きUSBポートに接続した後、システムを更新して再起動します。

sudo apt update && sudo apt upgrade -y
sudo reboot

システムがUSBイーサネットアダプターを認識しているかどうかを確認しました。

sudo lsusb

データを分析した

Bus 001 Device 004: ID 0bda:8152 Realtek Semiconductor Corp. RTL8152 Fast Ethernet Adapter
Bus 001 Device 003: ID 0424:ec00 Standard Microsystems Corp. SMSC9512/9514 Fast Ethernet Adapter
Bus 001 Device 002: ID 0424:9514 Standard Microsystems Corp. SMC9514 Hub
Bus 001 Device 001: ID 1d6b:0002 Linux Foundation 2.0 root hub

デバイス004は単なるアダプターです。

さらに、このアダプターに割り当てるネットワークインターフェイスを指定しました。

dmesg | grep 8152

[    2.400424] usb 1-1.3: New USB device found, idVendor=0bda, idProduct=8152, bcdDevice=20.00
[    6.363837] usbcore: registered new interface driver r8152
[    6.669986] r8152 1-1.3:1.0 eth1: v1.09.9
[    8.808282] r8152 1-1.3:1.0 eth1: carrier on

それはeth1 :)であることがわかりました、そして私はすでにそれとネットワークブリッジを設定することができます。 

以下のアルゴリズムに従って実際に私がしたこと：

  • インストールされているネットワークブリッジ管理パッケージ：

    sudo apt-get install bridge-utils
  • インストールされている ZeroTier ONE：
     
    curl -s https://install.zerotier.com | sudo bash
  • 既存のZeroTierネットワークに接続しました：

    sudo zerotier-cli join <Network ID>
  • IPアドレスとルートZeroTierの管理を無効にするコマンドを実行しました。

    sudo zerotier-cli set <networkID> allowManaged=0

また、ネットワークコントローラ上：

ではネットワーク、私はをクリックし、詳細、発見し、をクリックv4AssignModeのリンクや無効なIPのオフに、自動割り当てIPの割り当てプールからの自動割り当て ]チェックボックスを


その後、名前を設定し、承認済みブリッジとアクティブブリッジのチェックボックスを確認して、プラグインノードを承認しました。IPアドレスを割り当てていません。


次に、ノードでのネットワークブリッジの構成に戻り、ターミナルで編集するためのネットワークインターフェイス構成ファイルを開きました。

sudo nano /etc/network/interfaces

次の行を追加したところ

auto eth1
allow-hotplug eth1
iface eth1 inet manual

auto br0
allow-hotplug br0
iface br0 inet static
        address 192.168.0.10
        netmask 255.255.255.0
        network 192.168.0.0
        broadcast 192.168.0.255
        dns-nameservers 127.0.0.1
        bridge_ports eth1 ztXXXXXXXX
        bridge_fd 0
        bridge_maxage 0

ここで、eth1は、IPアドレスが割り当てられていない接続されたUSBイーサネットアダプターです。
br0-物理ネットワークのアドレス範囲から割り当てられた永続的なIPアドレスで作成されたネットワークブリッジ。
ztXXXXXXXXは、コマンドによって認識されたZeroTier仮想インターフェースの名前です。

sudo ifconfig

情報を入力した後、構成ファイルを保存し、次のコマンドでネットワークサービスをリロードしました。

sudo /etc/init.d/networking restart

ブリッジの操作性を確認するために、次のコマンドを実行しました。

sudo brctl show   

データによると-橋が上がりました。

bridge name	bridge id		STP enabled	interfaces
br0		8000.00e04c360769	no		eth1
							ztXXXXXXXX

次に、ネットワークコントローラーに切り替えてルートを設定しました。

ネットワークノードのリストで、ネットワークブリッジのIP割り当てリンクをクリックしたのはなぜですか。次に、開いたウィンドウで、[ 管理されたルート]をクリックします。私は新しいページに行ってきまし示さ0.0.0.0/0を通り、ターゲット、およびなどのゲートウェイ -以前に設定された組織のネットワークのアドレス範囲から、ネットワークブリッジのIPアドレス。私の場合、192.168.0.10


彼は入力されたデータを確認し、ノードのネットワーク接続の確認を開始し、物理ネットワークノードから仮想ネットワークのノードにpingを実行し、その逆も行いました。

それで全部です！


ただし、スクリーンショットを撮ったプロトタイプとは異なり、仮想ネットワークのノードのIPアドレスは、物理的なノードのIPアドレスと同じ範囲のものです。ネットワークをブリッジする場合、このモデルは可能です。主なことは、DHCPサーバーによって配布されるアドレスと重複しないことです。

この記事では、MS Windowsおよびその他のLinuxディストリビューションを実行しているノード側でネットワークブリッジをセットアップすることについて、この記事を個別に説明することはしません。インターネットには、このトピックに関する資料が満載です。ネットワークコントローラー側の設定は上記と同じです。

Raspberry PIは、ネットワークをZeroTierと組み合わせるための予算的で便利なツールであり、固定的なソリューションとしてだけではないことに注意してください。たとえば、アウトソーサーは、Raspberry PIに基づいて事前に構成されたネットワークブリッジを使用して、サービス対象のクライアントの物理ネットワークをZeroTierに基づいた仮想ネットワークとすばやく組み合わせることができます。
