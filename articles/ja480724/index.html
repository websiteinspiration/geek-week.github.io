トランスコンパイルされた言語：コードからコードへの変換プロジェクト


出典：Ward Cunningham。

翻訳とは、ある言語から別の言語へのコードの変換です。特別なトランスパイラー（トランスコンパイラー）を使用して、1つの汎用プログラミング言語を他の汎用プログラミング言語に転送できます。

別の言語に切り替える必要がある場合、トランスパイラーを使用すると、開発者はコード全体を手動で書き直すことなく、既存のコードのほとんどを保存できます。たとえば、プログラムをPython 2からPython 3に変換するとき、またはソフトウェアを古いAPIから新しいAPIに移動するときです。

トランスパイラーとトランスコンパイラーという用語はしばしば同じ意味で使用されますが、まだ違いがあると信じられています。たとえば、C ++コードをCに変換するにはトランスパイラーが必要で、Python-Rubyを変換するにはトランスコンパイラが必要です。バベルJavaScriptの場合、これはトランスパイラーであり、TypeScriptはトランスコンパイルされた言語です。

トランスポーターによる変換後、コードを正しく操作するために手動での構成が必要になる場合がありますが、トランスコンパイルされた言語では、コードは変更なしで機能する必要があります。本日は、これらについて説明しますが、最も重要なのは、開発者が日常業務で使用できる人気のあるトランスプロジェクトについてです。

フロントエンド転置


変換はフロントエンド開発の重要な部分です。ブラウザーでの新機能の表示が遅いため、実験的な機能を備えた言語が作成され、ブラウザー互換の言語に変換されています。

各プロジェクトで独自のJavaScriptエンジンを使用するブラウザー開発者は、JavaScriptの急成長するトランス言語にも責任があります。その結果、ブラウザーのパフォーマンスは異なり、独自のJavaScript関数のセットを実装し、さまざまな速度で仕様に完全に準拠します。



Sass、Less、およびStylusを使用すると、CSSコードを簡単に作成できます。そのため、Sassには柔軟な構文、論理（@if、それぞれ）、数学（数値、線、色を追加できます）があり、それによりコードの豊富さがすべて通常のCSS Rubyプログラムに変換されます。

Less（Leaner Style Sheets）はCSSと同じように見えますが、便利な追加機能があります。変数、ミックスイン、算術演算（Sassなど）などです。

スタイラスはNode.jsで記述され、SassやLessとは明確な構文が異なります。しかし、実際には、これらの3つの言語は互いに非常によく似ているので、それらのいずれかを選択した場合でも誤解しないでください。

Cに変換




2006年に、Valaプロジェクトが登場しました。これは、ValaとGenieの2つの言語で構成され、最も一般的なCコードを変換し、実行可能ファイルまたはライブラリにコンパイルします。

Valaの構文はC＃によく似ていますが、GObjectシステムとの互換性を高めるように調整されていますが、GenieはPythonと多くの共通点を持っています。第一言語はジニーよりも一般的になっているので、それをより詳しく見てみましょう。

なぜあなたはそのようなプロジェクトを必要としたのですか？多くの開発者は、高級言語でGNOMEのアプリケーションとライブラリを作成したいと考えていましたが、さまざまな理由により、C＃やJavaを使用できない、または使用したくありませんでした。 Valaを使用すると、複雑なオブジェクト指向コードをすばやく作成でき、必要なメモリが少ない標準のC APIとABIをサポートします。

C＃およびJavaライブラリーは、Cおよび他の言語のネイティブGObjectライブラリーと同じように使用することはできず、GNOMEプラットフォームの一部と見なすこともできません。ValaトランスコンパイラーであるValacは、ライブラリまたはアプリケーションをCで直接書き込んだ場合と同じように、ValaソースファイルからCファイルを作成します。CアプリケーションからValaライブラリを使用することは、他のGObjectベースのライブラリを使用することと同じです。

フラッシュからマルチプラットフォームへ




HaxeはValaと同時に作成されましたが、Flash、JavaScript、およびNekoでの変換を目的としています。そのような一連のスキルにより、言語は絶滅する運命にありましたが、それは生き残っただけでなく、非常に人気を得ました。

成功の秘訣はマルチプラットフォームです。やがて、Haxeは、JavaScript、C ++、C＃、Java、JVM、Python、Lua、PHP、Flashなどのさまざまな言語やプラットフォームへのトランスコンパイルをサポートする強力なツールセットに成長しました。

Haxeは、近代的で高水準の強く型付けされたプログラミング言語です。同時に、それは非常にシンプルで実用的であるため、コンパイルをさまざまなターゲットプラットフォームに適合させることができます。その構文は主にECMAScript標準ですが、必要に応じて変更されます。

Haxeの「Hello World」：

class Main {
static public function main():Void {
trace("Hello World");
  }
}

JavaScriptでは、これらのコード行は次のようになります。

haxe -main HelloWorld -js HelloWorld.js

そして、このようなJavaでは： 

haxe -main HelloWorld -java path/to/java/out

ほとんどのHaxeコードはクラスと関数に編成されており、Java、ActionScript 3、C＃に似たOOP言語になっています。ただし、Haxeはプラグマティズムと構文の容易さを重視して設計されました。

結論


他にも非常に多くの言語があるため、プロジェクトに最適な言語をいつでも選択できます。ただし、C ++からC、PHPからC ++、および「JavaScriptのすべて」の変換が頻繁に発生するため、コードからコードへの変換が必要な状況は珍しくありません。

ある言語から別の言語への「翻訳者」は良いアイデアですが、結果が明確に予測され、リスクなしで使用できる場合に限ります。したがって、JDK 1.4より前のJavaではメモリマップファイルがなかった ため、COBOLからJavaへの移行は困難でした。そして、記事で言及されているC2RustとCorrodeは、安全ではないとマークされ、生のポインターで満たされた非慣用的なコードを提供します。
