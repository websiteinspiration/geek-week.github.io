序文


みなさん、こんにちは！私の名前はサーシャです。私はバックエンド（LinuxサービスとAPI）を6年以上テストしています。この記事についての考えは、聞き慣れたテスターから、インタビューの前にLinuxコマンドで何を読むことができるかを次のように要求されたときに思い付きました。通常、QAエンジニアのポジションの候補者は、基本的なコマンド（もちろん、Linuxでの作業を意味する場合）を知る必要がありますが、Linuxの経験がほとんどないかまったくない場合に、面接の準備時に読む必要があるコマンドをどのように理解しますか？

そのため、これについては何度も書いていますが、「初心者のためのLinux」という別の記事を書き、Linuxを使用する部門（または会社）でのインタビューの前に知っておく必要がある基本的なコマンドをここにリストすることにしました。どのコマンドとユーティリティを使用し、どのパラメーターを最も頻繁に使用するかを考え、同僚からフィードバックを収集し、すべてを1つの記事にまとめました。記事は条件付きで3つの部分に分かれています。最初に、Linux端末での入出力の基本に関する簡単な情報、次に最も基本的なコマンドの概要、3番目の部分で、Linuxの一般的な問題の解決策について説明します。

Linuxでプロセスが開始すると、このプロセスの3つの標準データストリームが作成されます：stdin、stdout、およびstderr。これらの番号はそれぞれ0、1、2です。しかし、現在はstdoutに関心があり、程度は低いですがstderrに関心があります。名前から、stdoutがデータの出力に使用され、stderrがエラーメッセージの表示に使用されていることは容易に推測できます。デフォルトでは、Linuxでコマンドを実行すると、stdoutとstderrはすべての情報をコンソールに出力しますが、コマンド出力が大きい場合は、ファイルにリダイレクトすると便利です。これは、たとえば次のように行うことができます。

[user@testhost ~]$ man signal > man_signal

man_signal ファイルの内容を出力すると、 ` man signal`コマンドを実行しただけの場合と同じになります。

` > ` リダイレクト操作はデフォルトでstdoutを使用します。stdoutリダイレクトを明示的に指定できます： ` 1> `。同様に、stderrリダイレクトを指定できます： ` 2> `。これらの操作を組み合わせて、コマンドからの通常の出力とエラーメッセージの出力を分離できます。

[user@testhost ~]$ man signal 1> man_signal 2> man_signal_error_log

次のように、stdoutとstderrを1つのファイルに リダイレクトできます。

[user@testhost ~]$ man signal > man_signal 2>&1

リダイレクト操作「2>＆ 1」は、stderrをstdoutが向けられている場所にリダイレクトすることを意味します。

I / O（またはプロセス間通信の便利な手段）を操作するためのもう1つの便利なツールは、パイプ（またはパイプライン）です。パイプラインは、いくつかのコマンドをリンクするためによく使用されます。コマンドstdoutは、次のようにstdinにリダイレクトされます。

基本的なLinuxコマンド


pwd


現在の（作業中の）ディレクトリを出力します。

[user@testhost ~]$ pwd
/home/user

日付


システムの現在の日付と時刻を印刷します。

[user@testhost ~]$ date
Mon Dec 16 13:37:07 UTC 2019
[user@testhost ~]$ date +%s
1576503430

w


このコマンドは、システムにログインしているユーザーを表示します。さらに、稼働時間とLA（負荷平均）も表示されます。

[user@testhost ~]$ w
 05:47:17 up 377 days, 17:57,  1 user,  load average: 0,00, 0,01, 0,05
USER     TTY      FROM             LOGIN@   IDLE   JCPU   PCPU WHAT
user     pts/0    32.175.94.241    05:47    2.00s  0.01s  0.00s w

CD


現在のディレクトリを変更します。

[user@testhost ~]$ pwd
/home/user
[user@testhost ~]$ cd /home/
[user@testhost home]$ pwd
/home

引数としてディレクトリ名を渡さない場合、$ HOME環境変数、つまりホームディレクトリが使用されます。また、 `使用すると便利かもしれ〜 ` -意味特殊文字$ HOMEを：

[user@testhost etc]$ pwd
/etc
[user@testhost etc]$ cd ~/test/
[user@testhost test]$ pwd
/home/user/test

mkdir


ディレクトリを作成します。

[user@testhost ~]$ mkdir test
[user@testhost ~]$ ls -ltr
total 38184
-rw-rw-r-- 1 user user 39091284 Nov 22 14:14 qqq
drwxrwxr-x 2 user user     4096 Nov 26 10:29 test

特定のディレクトリ構造を作成する必要がある場合があります。たとえば、存在しないディレクトリ内のディレクトリなどです。mkdirを連続して何度も入力しないようにするために、-pオプションを使用できます。これにより、階層内に存在しないすべてのディレクトリを作成できます。また、このオプションを使用すると、ディレクトリが存在する場合、mkdirはエラーを返しません。

[user@testhost ~]$ ls
qqq  test
[user@testhost ~]$ mkdir test2/subtest
mkdir: cannot create directory ‘test2/subtest’: No such file or directory
[user@testhost ~]$ mkdir -p test2/subtest
[user@testhost ~]$ ls
qqq  test  test2
[user@testhost ~]$ ls test2/
subtest
[user@testhost ~]$ mkdir test2/subtest
mkdir: cannot create directory ‘test2/subtest’: File exists
[user@testhost ~]$ mkdir -p test2/subtest
[user@testhost ~]$ ls test2/
subtest

rm


ファイルを削除します。

[user@testhost ~]$ ls
qqq  test  test2
[user@testhost ~]$ rm qqq
[user@testhost ~]$ ls
test  test2

-r オプションを使用すると、ディレクトリとそのすべての内容を再帰的に削除できます。-fオプションを使用すると、削除時のエラー（たとえば、存在しないファイル）を無視できます。これらのオプションは、おおまかに言って、ファイルとディレクトリの階層全体を削除することを保証します（ユーザーがこれに対する権限を持っている場合）。したがって、これらのオプションは注意して使用する必要があります（古典的なジョークの例は " rm -rf / "であり、特定の状況下では削除されます。システム全体ではなく、そのパフォーマンスにとって重要なファイルがたくさんあります）。

[user@testhost ~]$ ls
test  test2
[user@testhost ~]$ ls -ltr test2/
total 4
-rw-rw-r-- 1 user user    0 Nov 26 10:40 temp
drwxrwxr-x 2 user user 4096 Nov 26 10:40 temp_dir
[user@testhost ~]$ rm -rf test2
[user@testhost ~]$ ls
test

cp


ファイルまたはディレクトリをコピーします。

[user@testhost ~]$ ls
temp  test
[user@testhost ~]$ cp temp temp_clone
[user@testhost ~]$ ls
temp  temp_clone  test

このコマンドには-rおよび-fオプションもありますが、これらを使用して、ディレクトリとフォルダの階層の別の場所へのコピーを保証できます。

mv


ファイルまたはディレクトリを移動または名前変更します。

[user@testhost ~]$ ls -ltr
total 4
drwxrwxr-x 2 user user 4096 Nov 26 10:29 test
-rw-rw-r-- 1 user user    0 Nov 26 10:45 temp
-rw-rw-r-- 1 user user    0 Nov 26 10:46 temp_clone
[user@testhost ~]$ ls test
[user@testhost ~]$ mv test test_renamed
[user@testhost ~]$ mv temp_clone test_renamed/
[user@testhost ~]$ ls
temp  test_renamed
[user@testhost ~]$ ls test_renamed/
temp_clone

ネコ


ファイルの内容を印刷します。

[user@testhost ~]$ cat temp
Content of a file.
Lalalala...

コマンドの先頭（ファイルの最初のn行またはバイトを出力）と末尾（後で詳しく説明）にも注意を払う価値があります。

尾


ファイルの最後のn行またはバイトを 印刷します。

[user@testhost ~]$ tail -1 temp
Lalalala...

-f オプションは非常に便利です。ファイルに新しいデータをリアルタイムで出力できます。

もっと少なく


テキストファイルが大きすぎて、catコマンドで表示するのが不便な場合があります。次に、lessコマンドでファイルを開くことができます。ファイルは部分的に表示され、ナビゲーションはこれらの部分で使用でき、検索やその他の単純な機能を使用できます。

[user@testhost ~]$ less temp

また、使用すると便利かもしれ少ないとパイプ：

[user@testhost ~]$ grep "ERROR" /tmp/some.log | less

プロセスはシグナルハンドラーを持っている可能性があるため、killは常に期待される結果（プロセスの即時終了）につながるとは限りません。プロセスを確実に「強制終了」するには、プロセスにSIGKILLシグナルを送信する必要があります。ただし、これによりデータが失われる可能性があるため（たとえば、プロセスが終了前に情報をディスクに保存する必要がある場合）、このコマンドを慎重に使用する必要があります。SIGKILLのシグナル番号は9なので、コマンドの短いバージョンは次のようになります。

[user@testhost ~]$ ps ux | grep sleep
531      1034930  0.0  0.0 107960   636 pts/1    S+   15:21   0:00 sleep 300
531      1034953  0.0  0.0 110516  2104 pts/2    S+   15:21   0:00 grep --color=auto sleep
[user@testhost ~]$ kill -9 1034930
[user@testhost ~]$ ps ux | grep sleep
531      1035004  0.0  0.0 110516  2092 pts/2    S+   15:22   0:00 grep --color=auto sleep

前述のSIGTERMとSIGKILLに加えて、さらに多くの異なるシグナルがあり、それらのリストはインターネットで簡単に見つけることができます。また、SIGKILL信号とSIGSTOP信号を傍受または無視することはできないことを忘れないでください。

Linuxの一般的なソリューション


ファイルの所有者を変更する


chownコマンド を使用して、ファイルまたはディレクトリの所有権を変更できます。

[user@testhost ~]$ chown user:user temp
[user@testhost ~]$ ls -l temp
-rw-rw-r-- 1 user user 31 Nov 26 11:09 temp

パラメータでは、このコマンドにコロンで区切って新しい所有者とグループ（オプション）を指定する必要があります。また、ディレクトリの所有者を変更する場合、-Rオプションが役立つ場合があります。その場合、ディレクトリのコンテンツ全体の所有者も変更されます。

ファイルのアクセス許可を変更する


この問題は、chmodコマンドを使用して解決されます。例として、「所有者は読み取り、書き込み、実行を許可され、グループは読み取りと書き込みを許可され、残りはすべて-なし」という権限のインストールを許可します。

[user@testhost ~]$ ls -l temp
-rw-rw-r-- 1 user user 31 Nov 26 11:09 temp
[user@testhost ~]$ chmod 760 temp
[user@testhost ~]$ ls -l temp
-rwxrw---- 1 user user 31 Nov 26 11:09 temp

パラメータの最初の7（これはビット表現の0b111です）は「所有者のすべての権利」を意味し、2番目の6（これはビット表現の0b110です）-「読み取りと書き込み」、まあ、0は残りの部分には何もありません。ビットマスクは3ビットで構成されます。最下位（「右」）ビットは実行を担当し、次の（「中央」）ビットは書き込みを担当し、最も古い（「左」）ビットは読み込みを担当します。
特殊文字を使用して権限を設定することもできます（ニーモニック構文）。たとえば、次の例では、現在のユーザーの実行権限が最初に削除されてから返されます。

[user@testhost ~]$ ls -l temp
-rwxrw---- 1 user user 31 Nov 26 11:09 temp
[user@testhost ~]$ chmod -x temp
[user@testhost ~]$ ls -l temp
-rw-rw---- 1 user user 31 Nov 26 11:09 temp
[user@testhost ~]$ chmod +x temp
[user@testhost ~]$ ls -l temp
-rwxrwx--x 1 user user 31 Nov 26 11:09 temp

このコマンドには多くの用途があるため、詳細について（特に、ここでは、ニーモニック構文などについて）読むことをお勧めします。

ファイル検索


findコマンド を使用して、ディレクトリツリーでファイルを名前で検索できます。

[user@testhost ~]$ find test_dir/ -name "*le*"
test_dir/file_1
test_dir/file_2
test_dir/subdir/file_3

他のオプションと検索フィルターも利用できます。たとえば、5日以上前に作成されたtestフォルダー内のファイルを見つけることができます。

[user@testhost ~]$ ls -ltr test
total 0
-rw-rw-r-- 1 user user 0 Nov 26 10:46 temp_clone
-rw-rw-r-- 1 user user 0 Dec  4 10:39 created_today
[user@testhost ~]$ find test/ -type f -ctime +5
test/temp_clone

ファイル内のテキストを検索する


grep コマンドは、このタスクに対処するのに役立ちます。彼女には多くの使用例がありますが、ここでは例として最も単純なものが示されています。

[user@testhost ~]$ grep -nr "content" test_dir/
test_dir/file_1:1:test content for file_1
test_dir/file_2:1:test content for file_2
test_dir/subdir/file_3:1:test content for file_3

grepコマンド を使用する一般的な方法の1つは、パイプで使用することです。

[user@testhost ~]$ sudo tail -f /var/log/test.log | grep "ERROR"

-v オプションを使用すると、grep ' の効果を得ることができ、grepに渡されたパターンを含まない反対の行だけが表示されます。

インストールされたパッケージを表示する


Linuxディストリビューションと使用するパッケージマネージャーにすべて依存するため、ユニバーサルコマンドはありません。ほとんどの場合、次のコマンドのいずれかが役立ちます。

yum list installed
apt list --installed
zypper se —installed-only
pacman -Qqe
dpkg -l
rpm -qa

ディレクトリツリーが占めるスペースを確認する


du コマンドの1つの使用例：

[user@testhost ~]$ du -h -d 1 test_dir/
8,0K test_dir/subdir
20K test_dir/

-dオプションの 値を変更して、ディレクトリツリーに関する詳細情報を取得できます。コマンドをsortと組み合わせて使用​​することもできます。

[user@testhost ~]$ du -h -d 1 test_dir/ | sort -h
8,0K test_dir/subdir
16K test_dir/subdir_2
36K test_dir/
[user@testhost ~]$ du -h -d 1 test_dir/ | sort -h -r
36K test_dir/
16K test_dir/subdir_2
8,0K test_dir/subdir

sortコマンド の-hオプションを使用すると、人間が読める形式（1K、2Gなど）で書き込まれたサイズをソートできます。- rオプションを使用すると、データを逆の順序でソートできます。

ファイル内、ディレクトリ内のファイル内を検索して置換する


この操作は、sedユーティリティを使用して実行されます（最後にgフラグがない場合、文字列内で最初に出現する「old-text」のみが置き換えられます）。

sed -i 's/old-text/new-text/g' input.txt

一度に複数のファイルに使用できます。

ネットワークトラフィックダンプ


Linuxでネットワークトラフィックを傍受するには、tcpdumpユーティリティを使用します。ポート12345でトラフィックをダンプするには、次のコマンドを使用できます。

[user@testhost ~]$ sudo tcpdump -i any -A port 12345

-A オプションは、出力をASCIIで表示することを示します（したがって、これはテキストプロトコルに適しています）。-i anyは、ダンプするネットワークインターフェース、ポート -どのポートトラフィックに関心がないことを示します。ポートの代わりに、ホスト、またはホストとポートの組み合わせ（ホストAとポートX）を使用できます。また、-nオプションも役立つ場合があります。出力でアドレスをホスト名に変換しないでください。
バイナリトラフィックの場合はどうなりますか？次に、-Xオプションが役立ちます-データを16進数とASCIIで出力します。

[user@testhost ~]$ sudo tcpdump -i any -X port 12345

どちらのユースケースでもIPパケットが出力されるため、それぞれの先頭にバイナリIPヘッダーとTCPヘッダーがあることに注意してください。以下は、ポート12345でリッスンしているサーバーに送信されたリクエスト " 123 "の出力例です。
