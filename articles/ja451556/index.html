Flutter：Android Studioを使用したアプリケーションのローカリゼーション


Flutterアプリケーションのローカリゼーション（国際化）については、かなり多くのことが書かれています。
公式ドキュメントはこの問題について詳しく説明しています。さらに、多くの愛好家が さまざまな アプローチについて説明しています。


この記事は、すべてを網羅しているように見せかけていませんが、私の意見では、手作業による入力を非常に効果的に削減する方法を説明しています。


私たちはflutter_i18nを使用したローカリゼーションについて話しています。これは、日常的な作業や定型コードを取り除くために使用できるAndroid Studioプラグインです。


トレーニング


最初にflutter-i18nプラグインをインストールします



再起動後、生成された/ i18n.dartおよびres / values / strings_en.arbファイルがプロジェクト構造に自動的に追加されます



（私はロシア語用のstrings_ru.arbファイルを手動で追加しました）


Android Studio 3.3.1-3.4には何らかのローカライズファイルが含まれていないため、ローカリゼーションファイルが生成されないことに注意してください。私はこの問題をstackoverflowで説明し、解決策を得ました。別の解決策はAS 3.2に一時的に留まることですが、これは素人です。
