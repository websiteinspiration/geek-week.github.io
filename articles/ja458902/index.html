5つの一般的な初心者のPythonの間違い

Pythonプログラマーとして働いた初期の頃は、コードでさまざまな種類のバグに遭遇しました。StackOverflowでの数時間の苦痛の後は、それはバグではなくPythonの機能です。以下は、ほとんどの新しいPythonプログラマーが犯す5つの最も一般的な間違いです。Facebookのページやグループで質問して数時間節約するために、それらについて少し調べてみましょう。

1.辞書またはリストをコピーする


辞書またはリストのコピーを作成する必要がある場合、代入演算子を使用するだけでは不十分です。

違う：

>>> dict_a = {"name": "John", "address":"221B Baker street"}
>>> dict_b = dict_a

これで、dict_bを変更または更新すると、dict_aも変更されます。これはすべて代入演算子のおかげです。この演算子を使用すると、dict_bはdict_aと同じオブジェクトを指すことになります。

>>> dict_b["age"] = 26
>>> dict_b
{'address': '221B Baker street', 'name': 'John', 'age': 26}
>>> dict_a
{'address': '221B Baker street', 'name': 'John', 'age': 26}
>>>

正しい例： copy（）またはdeepcopy（）メソッドを使用します。


>>> dict_c = dict_b.copy()
>>> dict_c["location"] = "somewhere"
>>> dict_c
{'address': '221B Baker street', 'name': 'John', 'age': 26, 'location': 'somewhere'}
>>> dict_b
{'address': '221B Baker street', 'name': 'John', 'age': 26}
>>> dict_a
{'address': '221B Baker street', 'name': 'John', 'age': 26}
>>>

コピーとディープコピーの違いをご覧ください。

2.辞書キー


辞書に値を追加してみましょう：

>>> dict_a = dict()
>>> dict_a
{}
>>> dict_a[1] = "apple"
>>> dict_a[True] = "mango"
>>> dict_a[2] = "melon"

辞書を表示しようとすると、何が表示されますか？

>>> dict_a
{1: 'mango', 2: 'melon'}

何が起こりましたか、Trueキーはどこにありますか？

ブールクラスは整数（整数）を継承することに注意してください。また、Trueに相当する整数は1です。False相当は0です。したがって、キー1の値は単に上書きされます。

>>> isinstance(True, int)
True
>>> isinstance(False, int)
True
>>> True == 1
True
>>> False == 0
True

3.リストまたは辞書の更新


リストにアイテムを追加するとします。

>>> list_a = [1,2,3,4,5]
>>> list_a = list_a.append(6)
>>> list_a
>>> # prints nothing

または、辞書を更新しようとしています。

>>> dict_a = {"a" : "b"}
>>> dict_a = dict_a.update({"c" : "d"})
>>> dict_a
>>> # prints nothing

それでは、リストを整理してみましょう。

>>> list_b = [2,5,3,1,7]
>>> list_b = list_b.sort()
>>> list_b
>>> # prints nothing

なぜ何も出てこない、何が悪いのか？

ほとんどのコンテナーメソッド（並べ替え、更新、追加、追加など）は、パフォーマンスの目的で最適化されており、不必要に別のコピーを作成することを避けています。

そのようなメソッドの戻り値を変数に割り当てようとしないでください。

正しく：

>>> list_a = [1,2,3,4,5]
>>> list_a.append(6)
>>> dict_a = {"a" : "b"}
>>> dict_a.update({"c" : "d"})
>>> dict_a
{'c': 'd', 'a': 'b'}
>>> list_a.sort()
>>> list_a
[1, 2, 3, 4, 5, 6]

4.インターンされた文字列


場合によっては、Pythonは既存の不変オブジェクトを再利用しようとします。文字列のインターンはそのようなケースの1つです。

>>> a = "gmail"
>>> b = "gmail"
>>> a is b
True

ここでは、2つの異なるオブジェクト（文字列）を作成しようとしました。しかし、同等性をテストしたところ、完全に一致していることがわかりました。これは、Pythonが別のオブジェクトbを作成せず、bが「gmail」の最初の値を指すようにしたためです。
長さが1のすべての文字列が抑止されます。ASCII文字、数字、アンダースコア以外の行はインターンされません。
確認しよう。

>>> a = "@gmail"
>>> b = "@gmail"
>>> a is b
False

また、==はis演算子とは異なります。==演算子は値が等しいかどうかを確認し、is演算子は両方の変数が同じオブジェクトを参照しているかどうかを確認します。

>>> a = "@gmail"
>>> b = "@gmail"
>>> a is b
False
>>> a == b
True

したがって、不変文字列または==およびis演算子を使用する場合は、そのことを覚えておいてください。

5.デフォルトの引数は1回評価されます。


例を考えてみましょう：

def func(a, lst=[]):
    lst.append(a)
    return lst
print(func(1))
print(func(2))

2枚印刷すると何が表示されると思いますか？

コードを実行してみましょう。

>>> def func(a, lst=[]):
... lst.append(a)
... return lst
... 
>>> print(func(1))
[1]
>>> print(func(2))
[1, 2]

2番目のケースが出力されるのはなぜですか[1、2]？[2]だけではいけませんか？

したがって、問題は、デフォルトの引数が1回だけ評価されることです。関数が初めて呼び出されたとき-func（1）-リストが評価され、リストが空であることがわかりました。これは、1を追加できることを意味します。ただし、2番目の呼び出し-func（2）では、リストにすでに1つの要素があるため、[1、2]が表示されます。

ボーナス：スペースとタブを混在させる必要はありません。しないでください。
