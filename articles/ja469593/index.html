Citrix Cloud


Citrix Cloudは、デジタルワークステーションを整理するために必要なすべてのサービスを収容するクラウドプラットフォームです。 Citrixはこのクラウドを直接所有しており、メンテナンスに従事しており、指定されたSLA（サービスの可用性-毎月少なくとも99.5％）を提供しています。

Citrixの顧客（顧客）は、選択したサブスクリプション（サービスパッケージ）に応じて、SaaSモデルを使用して特定のサービスリストにアクセスできます。彼らにとって、Citrix Cloudは同社のクラウドベースのデジタルワークステーションダッシュボードとして機能します。 Citrix Cloudにはマルチテナントアーキテクチャがあり、顧客とそのインフラストラクチャは互いに分離されています。

Citrix Cloudはコントロールプレーンとして機能し、次のような多数のCitrixクラウドサービスをホストしていますデジタルワークスペースインフラストラクチャのサービスおよび管理サービス。ユーザーアプリケーション、デスクトップ、データを含むデータプレーンは、Citrix Cloudの外部にあります。唯一の例外は、完全にクラウドベースのSecure Browser Serviceです。データプレーンは、お客様のデータセンター（オンプレミス）、サービスプロバイダーのデータセンター、ハイパークラウド（AWS、Azure、Google Cloud）に配置できます。顧客データが複数のサイトとクラウドにあり、それらの管理がCitrix Cloudから集中管理されている場合、混合および分散ソリューションが可能です。

Citrix Workspace


Citrix Workspaceは、超越的で根本的かつ包括的な概念です。私たちはそれをより詳細に扱い、それが明らかになるでしょう-理由。

全体として、Citrix WorkspaceはCitrixのデジタルワークプレイスのコンセプトを表しています。これは、ソリューション、サービス、およびサービスのセットであり、統一された安全で便利な管理可能なワークプレイスを作成します。

ユーザーはシームレスなSSOを取得して、アプリケーション/サービス、デスクトップ、およびデータに単一のコンソールから任意のデバイスからすばやくアクセスし、生産的な作業を行うことができます。多くのアカウント、パスワード、アプリケーションの検索の難しさ（ショートカット、スタートパネル、ブラウザなど、さまざまな場所にある）を忘れることができます。



ITサービスは、サービスとクライアントデバイスの集中管理、セキュリティ、アクセス制御、監視、更新、ネットワーク相互作用の最適化、分析のためのツールを受け取ります。

Citrix Workspaceは、次のリソースへの統一されたアクセスを提供します：

  • Citrix Virtual Apps and Desktops-アプリケーションとデスクトップの仮想化。
  • Webアプリケーション
  • クラウドSaaSアプリケーション;
  • モバイルアプリケーション;
  • を含むさまざまなストレージ内のファイル 曇り。



Citrix Workspaceリソースへのアクセスは次のとおりです。

  • 標準ブラウザ-Chrome、Safari、MS IEおよびEdge、Firefoxをサポート
  • または「ネイティブ」クライアントアプリケーション-Citrix Workspaceアプリ。

一般的なすべてのクライアントデバイスからアクセスできます。

  • Windows、Linux、MacOS、さらにはChrome OSを実行している本格的なコンピューター。
  • iOSまたはAndroidを搭載したモバイルデバイス。

Citrix Workspace Platformは、デジタルワークスペースを整理するための多くのCitrix Cloudクラウドサービスの一部です。WorkspaceにはCitrix Cloudに存在するほとんどのサービスが含まれていることは注目に値します。後で詳しく説明します。

したがって、エンドユーザーは、Workspaceアプリまたはそのブラウザーの代替（Workspaceアプリfor HTML5）を介して、お気に入りのクライアントデバイスでデジタルワークステーションの機能を利用できます。この機能を実現するために、Citrixは、会社の管理者がCitrix Cloudを通じて管理する一連のクラウドサービスの形式でワークスペースプラットフォームを提供しています。3つのパックで

利用可能なCitrixワークスペース：標準、プレミアム、プレミアムプラス。パッケージに含まれるサービスの数が異なります。また、一部のサービスをパッケージ外で個別に購入することもできます。たとえば、基本サービスである仮想アプリおよびデスクトップはプレミアムプラスパッケージにのみ含まれており、個別のコストは標準パッケージよりも高く、プレミアムとほぼ同じです。

ワークスペースは、クライアントアプリケーション-ワークスペースアプリとクラウドプラットフォーム（その一部）-ワークスペースプラットフォームの両方であり、さまざまなサービスパッケージの名前と、Citrix全体のデジタルワークスペースの概念であることがわかります。ここにそのような多様な実体があります。

アーキテクチャとシステム要件


従来、Citrix Digital Worksの構造には3つの領域があります。

  • ワークスペースアプリまたはデジタルワークステーションへのブラウザーベースのアクセスを備えた多くのクライアントデバイス。
  • cloud.comドメインのインターネット上のどこかにあるCitrix Cloudクラウドの直接ワークスペースプラットフォーム。
  • リソースの場所-Citrix Workspaceで公開されたアプリケーション、仮想デスクトップ、および顧客データを持つリソースをホストする、所有またはリースのサイト、プライベートまたはパブリッククラウド。これは、前述のデータプレーンと同じです。1人の顧客が複数のリソースロケーションを持っている可能性があることを覚えています。

リソースの例には、ハイパーバイザー、サーバー、ネットワークデバイス、ADドメイン、およびユーザーに適切なデジタルワークステーションサービスを提供するために必要なその他の要素が含まれます。

分散インフラストラクチャのシナリオには、次のものが含まれます。

  • お客様自身のデータセンター内のリソースのいくつかの場所、
  • パブリッククラウド内の場所
  • 離れた場所にある小さな場所。

場所を計画するときは、次のことを考慮してください。

  • ユーザー、データ、アプリケーションの近接性。
  • スケーリングする能力 迅速なキャパシティビルディングとワインドアップを保証
  • 安全要件とレギュレータ。

Citrix Cloudと顧客リソースの場所の間の通信は、Citrix Cloud Connectorと呼ばれるコンポーネントを介して行われます。これらのコンポーネントを使用すると、ユーザーはユーザーに提供されるリソースの維持に集中でき、すでにクラウドにデプロイされており、Citrixに付随しているサービスと管理サービスの踊りを忘れることができます。

負荷分散とフォールトトレランスのために、各リソースの場所に少なくとも2つのCloud Connectorをデプロイすることをお勧めします。 Cloud Connectorは、Windows Server（2012 R2または2016）を実行している専用の物理マシンまたは仮想マシンにインストールできます。 DMZではなく、内部リソース割り当てネットワークに配置することをお勧めします。

Cloud Connectorは、Citrix Cloudとリソースの場所の間のトラフィックをhttpsを介して認証および暗号化します。デフォルトでは、TCPポート443です。CloudConnectorからクラウドへの送信セッションのみが許可され、受信接続は禁止されています。

Citrix Cloudでは、お客様のインフラストラクチャにActive Directory（AD）が必要です。ADはメインのIdAMプロバイダーとして機能し、ワークスペースリソースへのユーザーアクセスを承認するために必要です。クラウドコネクタはADにアクセスできる必要があります。フォールトトレランスを実現するには、リソースの各場所にドメインコントローラーのペアを配置し、この場所のクラウドコネクタとやり取りすることをお勧めします。

Citrix Cloud Services


Citrix Workspaceプラットフォームの基礎となる主要なCitrix Cloudサービスに精通し、顧客が本格的なデジタルワークステーションを展開できるようにする価値があります。



これらのサービスの目的と機能を検討してください。

仮想アプリとデスクトップ


これは、アプリケーションおよび完全なVDIへのターミナルアクセスを提供するメインのCitrix Digital Workspaceサービスです。WindowsおよびLinuxアプリケーションとデスクトップの仮想化をサポートします。

Citrix CloudクラウドサービスであるVirtual Apps and Desktopsサービスには、下の図に示すように、従来の（非クラウド）Virtual Apps and Desktopsと同じコンポーネントがあります。違いは、サービスの場合のすべての制御コンポーネントがCitrix Cloudにあることです。お客様は、これらのコンポーネントを展開して維持し、コンピューティング能力を割り当てる必要がなくなりました。これはCitrixによって行われます。



その側では、お客様は次のコンポーネントをリソースの場所に展開する必要があります。

  • クラウドコネクタ
  • ADドメインコントローラー
  • 仮想配信エージェント（VDA）。
  • ハイパーバイザー-原則としてそうですが、物理を省略できる状況もあります。
  • オプションのコンポーネントは、Citrix GatewayとStoreFrontです。

Cloud Connectorを除くこれらのコンポーネントはすべて、お客様が独自に保守します。データプレーンはここにあり、特にユーザーアプリケーションとデスクトップが直接配置されているVDAを備えた物理ノードとハイパーバイザーの場合、これは論理的です。

Cloud Connectorはお客様がインストールする必要があるだけです。これは、Citrix Cloudコンソールから実行される非常に簡単な手順です。これらのサポートは自動的に実行されます。

アクセス制御


このサービスは、次の機能を提供します。

  • 人気のあるSaaSアプリケーションの多数のリストへのSSO（シングルサインオン）。
  • インターネットリソースへのアクセスのフィルタリング。
  • インターネット上のユーザーアクティビティを監視する。

Citrix Workspaceを介したSaaSサービスへのSSOクライアントは、通常のブラウザーアクセスよりも便利で安全な代替手段です。サポートされるSaaSアプリケーションのリストは非常に多く、常に拡大しています。

インターネットアクセスフィルタリングは、手動で作成したサイトのホワイトリストまたはブラックリストに基づいて構成できます。さらに、サイトカテゴリによるアクセス分類は、広範で更新された商用URLリストに基づいてサポートされています。ユーザーは、ソーシャルネットワーク、購入、成人向け、マルウェア、急流、プロキシなどのカテゴリのサイトへのアクセスを制限される場合があります。

サイト/ SaaSへの直接アクセスを許可するか、それらへのアクセスをブロックすることに加えて、クライアントをセキュアブラウザにリダイレクトすることが可能です。それら。リスクを軽減するために、選択したカテゴリ/インターネットリソースのリストへのアクセスは、セキュリティで保護されたブラウザを介してのみ可能になります。



このサービスは、インターネット上のユーザーアクティビティを監視するための詳細な分析も提供します。訪問したサイトとアプリケーション、危険なリソースと攻撃、ブロックされたアクセス、ダウンロード/ダウンロードされたデータの量。

安全なブラウザ


Citrix Workspaceユーザー用のインターネットブラウザ（Google Chrome）を仮想アプリケーションとして公開できます。 Secure Browserは、Citrixが管理および保守するSaaSサービスです。 Citrix Cloud（データプレーンを含む）で完全にホストされているため、お客様が独自のリソースの場所に展開して維持する必要はありません。

Citrixは、顧客向けに公開されたブラウザーをホストするVDAのクラウド内のリソースを割り当てる責任があり、OSとブラウザー自体のセキュリティと更新を保証します。

セキュアブラウザーへのクライアントアクセスは、Workspaceアプリまたはクライアントブラウザーを介して行われます。セッションはTLSを使用して暗号化されます。サービスを使用するために、クライアントは何もダウンロードしてインストールする必要はありません。

クラウドでセキュアブラウザスピンを介して起動されたサイトとWebアプリケーションは、クライアントがターミナルセッションの画像を受信するだけで、ターミナルデバイスでは何も行われません。これにより、セキュリティが大幅に向上し、ブラウザ攻撃から保護できます。

サービスの接続と管理は、Citrix Cloudカスタマーパネルを介して行われます。接続は数回のクリックで実行されます。


管理も非常に簡単で、ポリシーとホワイトシートを設定


することになります。ポリシーでは、次のパラメーターを調整できます。

  • クリップボード-ブラウザセッションでコピーと貼り付けのオプションを有効にできます。
  • 印刷-WebページをクライアントデバイスにPDF形式で保存する機能。
  • 非キオスク-デフォルトで有効になっており、ブラウザー（複数のタブ、アドレスバー）を完全に使用できます。
  • リージョンフェイルオーバー-メインリージョンが落ちたときに、別のCitrix Cloudリージョンでブラウザーを再起動する機能。
  • クライアントドライブマッピング-ブラウザセッションファイルをダウンロードまたはダウンロードするためのクライアントデバイスディスクをマウントする機能。

ホワイトシート（ホワイトリスト）では、クライアントがアクセスできるサイトのリストを指定できます。このリスト外のリソースへのアクセスは拒否されます。

コンテンツのコラボレーション


このサービスは、ワークスペースユーザーがお客様の内部リソース（オンプレミス）およびサポートされているパブリッククラウドサービスに投稿されたファイルおよびドキュメントに共同アクセスする可能性を提供します。これらは、ユーザーの個人用フォルダー、企業ネットワークボール、SharePointドキュメント、またはOneDrive、DropBox、Googleドライブなどのクラウドリポジトリです。

このサービスは、すべてのタイプのストレージリソースのデータにアクセスするためのSSOを提供します。Citrix Workspaceユーザーは、オフィス内だけでなくリモートからも、デバイスにある作業ファイルに安全にアクセスできます。追加の問題はありません。

コンテンツコラボレーションは、データを操作するための次の機能を提供します。

  • ワークスペースリソースとクライアントデバイス間のファイル交換（ダウンロードとダウンロード）、
  • すべてのデバイス上のユーザーファイルの同期
  • 複数のワークスペースユーザーのファイル共有と同期
  • 他のワークスペースユーザーのファイルとフォルダへのアクセス権の設定
  • ファイルへのアクセスの要求、ファイルを安全にアップロード（アップロード）するためのリンクの形成。

さらに、追加の保護メカニズムが提供されます。

  • ワンタイムパスワードを持つファイルへのアクセス
  • ファイルの暗号化
  • 透かし入りのファイル共有。

エンドポイント管理


このサービスは、デジタルワークステーションがモバイルデバイス（モバイルデバイス管理-MDM）およびアプリケーション（モバイルアプリケーション管理-MAM）を管理するために必要な機能を提供します。CitrixはそれをSaaS-EMMソリューションとして位置付けています-サービスとしてのエンタープライズモビリティ管理。

MDM機能により、次のことが可能になります。

  • お客様のリソースに接続するためのアプリケーション、デバイスポリシー、証明書を配布します。
  • デバイスを追跡する
  • デバイスをブロックまたは実行して、デバイスの完全または部分的なワイプを実行します。

MAM機能により、次のことが可能になります。

  • モバイルデバイス上のアプリとデータを保護する
  • 企業のモバイルアプリを提供します。

アーキテクチャの観点と顧客にサービスを提供する原則の観点から見ると、Endpoint Managementは、上記の仮想アプリとデスクトップのクラウドバージョンに非常に似ています。コントロールプレーンとそれを形成するサービスはCitrix Cloudにあり、メンテナンスはCitrixが担当しているため、このサービスをSaaSと見なすことができます。

お客様のリソースの場所にあるデータプレーンには、次のものが含まれます。

  • Citrix Cloudと対話するために必要なクラウドコネクタ、
  • Citrix Gatewayは、ユーザーに顧客の内部リソース（アプリケーション、データ）とマイクロVPN機能への安全なリモートアクセスを提供します。
  • Active Directory PKI
  • Exchange、ファイル、仮想アプリケーション、デスクトップ。



ゲートウェイ


Citrix Gatewayは次の機能を提供します：

  • リモートアクセスゲートウェイ-安全な境界の外にあるモバイルユーザーとリモートユーザーのための企業リソースへの安全な接続
  • 企業リソースにSSOを提供するためのIdAMプロバイダー（IDおよびアクセス管理）。

この文脈では、企業リソースは仮想アプリケーションおよびデスクトップとしてだけでなく、多くのSaaSアプリケーションとしても理解されるべきです。

ネットワークトラフィックを最適化してマイクロVPN機能を取得するには、通常はDMZ内の各リソースロケーションにCitrix Gatewayを展開する必要があります。この場合、必要な容量とサポートの割り当ては、お客様の肩にかかっています。

別の方法は、Citrix GatewayサービスをCitrix Cloudサービスとして使用することです。この場合、お客様は、Citrixがクラウドでこれを行うために、自分で何かを展開して保守する必要はありません。

分析


これは、上記のすべてのクラウドサービスと統合されたCitrix Cloud分析サービスです。ビルトインの機械学習メカニズムを通じて、Citrixサービスによって生成されたデータを収集および分析するように設計されています。これは、ユーザー、アプリケーション、ファイル、デバイス、およびネットワークに関連するメトリックを考慮に入れます。

その結果、セキュリティ、パフォーマンス、ユーザー操作に関するレポートが生成されます。



Citrix Analyticsは統計レポートを生成するだけでなく、積極的に行動することができます。これは、通常のユーザーの行動のプロファイルを形成し、異常を識別することです。ユーザーが非標準的または積極的にデータを操作するアプリケーションを使い始めた場合、ユーザーと彼のデバイスは自動的にブロックされます。危険なインターネットリソースにアクセスした場合も同じことが起こります。

安全性だけでなく、性能にも気を配っています。Analyticsを使用すると、長いユーザーログオンとネットワーク遅延に関連する問題を監視し、すばやく解決できます。

結論


Citrixクラウドのアーキテクチャ、Workspaceプラットフォーム、およびデジタルワークスペースのインフラストラクチャを整理するために必要なその基本的なサービスについて知りました。 Citrix Cloudのすべてのサービスを検討したのではなく、デジタルワークスペースを整理するための基本セットに限定したことは注目に値します。Citrixクラウドサービスの完全なリストには、ネットワークツール、アプリケーションやワークステーションを操作するための追加機能も含まれています。

また、デジタルワークプレースの基本的な機能は、Citrix Cloudがなくても、オンプレミスでのみ展開できることも言う必要があります。基本的な製品である仮想アプリとデスクトップは、VDAだけでなくすべての管理サービスでもクラシックバージョンで引き続き利用できます。この場合、お客様は自分のサイトに個別にデプロイして保守します。この場合、Cloud Connectorは必要ありません。同じことがエンドポイント管理にも当てはまります。オンプレミスの祖先はXenMobileサーバーと呼ばれていますが、クラウドバージョンでは少し機能的です。また、顧客はAccess Controlの一部の機能をサイトに実装することもできます。セキュアブラウザー機能はオンプレミスで実装でき、ブラウザーの選択はクライアント次第です。

サイトにすべてを展開したいという欲求は、セキュリティ、管理、ブルジョワの雲に対する制裁不信の点で優れています。ただし、Citrix Cloudがないと、コンテンツコラボレーションと分析の機能に完全にアクセスできなくなります。上記のように、他のCitrixオンプレミスソリューションの機能は、クラウドの実装よりも劣る場合があります。そして最も重要なことは、コントロールプレーンを維持し、自分で管理する必要があることです。
