ソフトウェア無線-それはどのように機能しますか？パート6

こんにちは、ハブル。前の部分、我々はGNUラジオを使用して単純な信号を送信する可能性を見ました。次に、さらに進んで、より複雑なものを伝える方法を確認します。まず、アマチュア無線信号WSPRから始めて、実際に動作するソフトウェアQAMモデムを作成します。 そして、前のケースと同様に、コードを1行も記述せずにこれを行います。プログラムもクロスプラットフォームであり、OSX / LinuxとWindowsの両方で動作します。また、ハードウェアなしで、GNU Radioを使用してモデムをデバッグする方法も示します。 カットの下に続きました。









GNU Radioを使用したことがない人は、プログラムの操作の原則を説明したパート4と5を読むことをお勧めします。

WSPR


WSPR を使用した簡単なものから始めましょう。このタイプの通信は、弱い信号の伝播テストのために特別に作成されました。必要なのは、LimeSDRなどのデバイスの電力レベルが100mW以下であることです。 WSPR信号は非常に低速（メッセージあたり約2バイト）で非常に狭い帯域で送信されるため、ノイズレベル未満でも受信できます。 GNU Radioを使用してそのような信号を送信します。

まず、信号を記録する必要があります。これを行うには、WSJTプログラムを利用します、必要なすべてのパラメーター（電源、コールサイン、場所など）を設定します。設定で出力オーディオデバイスとして仮想オーディオケーブルを指定し、WAVで信号を記録します。エッジでの一時停止は、任意のエディター（たとえば、Cool Edit）でトリミングする必要があります。その結果、約2分のファイルが作成されます。

次に、GNU Radio Companionでグラフを作成します。



この方法は最大効率であるとは主張していませんが、非常にシンプルで理解可能です。 WSPR信号の初期周波数は1500 Hzで、記録されたwavファイルのサンプリング周波数は22050 s / sです。最初に、信号を57/5回リサンプリングして、サンプリングレートを必要な250.000 s / sにします。次に、周波数を10KHzシフトし（有効な信号は11.5KHzの周波数になります）、信号をレシーバーが必要とする複雑な形式に変換し、余分なフィルターをカットして、周波数を11〜12KHzにします。

WSPR信号は時間制限があり、偶数分ごとに送信されます（0：00、0：02など）。私は手動でGNU Radioへの転送を開始しました。目で見て、間隔を1時間で決定します。Pythonでスクリプトを追加して転送を自動的に開始することができます。

適切なタイミングを待って、送信機と受信機の電源を入れ、結果を確認します。



必要に応じて、入力データ（コールサイン、場所、送信電力）に基づいてWSPRファイルの自動生成をプログラムに追加することもできます。Python用のWSPR生成の例は、githubで取得できます。

信号はまだデコードされていますが、432 MHzでの周波数ドリフトはすでにかなり顕著です。しかし、1.3 GHzの周波数では、ドリフトが非常に大きくなるため、WSPRは受け入れられなくなります。SDRには、より安定した信号を備えた外部基準発生器が必要です（または、送信中に少なくともソフトウェア周波数補正が必要ですが、これは不便です）。

SDRで低周波数での送信が可能であれば、HF帯域での送信を試すことができます。このように、HackRFを使用すると、室内アンテナから14 MHzで1000 kmの信号を送信することができ、これは良い結果と見なすことができます。高周波（433 MHzおよび1.3 GHz）は実験にとってさらに興味深いかもしれませんが、信号は直接見通し内でのみ送信されるため、このような実験では受信側に2人目の参加者が必要です。そのようなテストの2番目のプラスは、レイジーのみがwsprでHFに送信しなかったが、高周波数の習得ははるかに少ないことです。そのため、LimeSDRやUSRPなどのボードを使用すると、超高周波での受信と送信で非常に興味深い実験を実行できます。

QAMモデム


さらに進みましょう。 WSPRはかなり単純な形式です。もっと面白いことをしましょう-本格的な（まあ、ほぼ）モデムです。直交振幅変調、データが高速で伝送されることを可能にする信号を同時に変化の振幅及び位相の両方（ただし、占有帯域も大きくなっています）。

最初の部分-送信機を検討してください。



ご覧のように、data.txtファイルからデータを読み取り、25 KHzのサンプリング周波数で、データをパケットエンコーダーに送信します。パケットエンコーダーは、ストリームを4ビットコードに変換します。このストリームは直交変調器に送られ、サンプリング周波数が250KHzの送信機周波数に増加し、信号が80KHz上方にシフトします（多くの受信機はゼロ周波数にピークがあり、これが干渉します）。 Constellation Rectコンポーネントは、変調パラメーター（文字数、位相と振幅のシフト）を設定します。

最初の部分は準備ができています。 「転送」を開始して、信号を確認します。



機器がなくても送信機をテストできます。これには、特別なChannel Modelブロック（通信チャネルモデル）があります。そこでは、ノイズ、周波数シフトなどを設定できます。

これが送信前後の信号の様子です。ちなみに、ご覧のように、120dB未満の送信機のすべての「高調波」はノイズレベルよりもはるかに低くなっています。



今レセプション。実際、同じことですが、逆順です。



別に、最後のUDP Sinkブロックで停止できます。理由は明らかではありませんが、GNU Radioにはテキストデータを表示するためのコンポーネントがありません。したがって、UDPを介して任意のローカルポートにデータを送信するだけです（私は999を選択しました）。

受け取るには、Pythonで簡単なプログラムを記述します。

import socket

UDP_IP = "127.0.0.1"
UDP_PORT = 999

sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM) # UDP
sock.bind((UDP_IP, UDP_PORT))
sock.settimeout(1.0)

while True:
  try:
    data, addr = sock.recvfrom(64) # buffer size is 64 bytes
    print("Msg:", data)

  except socket.timeout:
   pass

結果：スクリプトを実行し、GNU Radioを実行して、受信したメッセージをコンソールで確認します。
