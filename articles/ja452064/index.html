PowerShellの望ましい状態の構成とファイル：パート1. SQLデータベースと連携するようにDSC Pullサーバーを構成する



PowerShellの望ましい状態の構成（DSC）を使用すると、数百台のサーバーがある場合に、オペレーティングシステム、サーバーの役割、およびアプリケーションを簡単に展開および構成できます。

ただし、オンプレミスでDSCを使用する場合、MS Azureではなく、いくつかのニュアンスがあります。組織が大きく（300のワークステーションとサーバーから）、まだコンテナーの世界を開いていない場合、これらは特に具体的です。

  • システムのステータスに関する完全なレポートはありません。一部のサーバーに必要な構成が適用されなかった場合、これらのレポートがなければ、それについてはわかりません。組み込みのレポートサーバーから情報を取得することは非常に困難であり、多数のホストの場合も長い時間がかかります。
  • スケーラビリティとフォールトトレランスはありません。単一のフォールトトレラントデータベースとmof構成ファイル、モジュール、および登録キーの共通リポジトリを持つポーリングDSC Webサーバーのファームを構築することは不可能です。

今日は、最初の問題を解決し、レポート用のデータを取得する方法について説明します。SQLをデータベースとして使用できれば、すべてが簡単になります。MS は、Windows Server 2019またはビルドWindowsサーバー1803でのみ組み込みサポートを約束します。DSCサーバーはOleDbCommandで完全にはサポートされていない名前付きパラメーターを使用するため、OleDBプロバイダーを使用してデータを取得することもできません。

私はこの方法を見つけました：Windows Server 2012および2016を使用するユーザーは構成できますSQLデータベースをポーリングDSCサーバーのバックエンドとして使用する。これを行うには、リンクテーブルを含む.mdbファイルの形式で「プロキシ」を作成します。これにより、クライアントレポートから受信したデータがSQLサーバーデータベースにリダイレクトされます。

注：Windows Server 2016の場合、Microsoft.Jet.OLEDB.4.0はサポートされなくなったため、AccessDatabaseEngine2016x86を使用する必要があります。

私はドエルポーリングDSCサーバーの展開プロセスに、それは非常によく説明されていないだろうここに。ほんの2、3点指摘しておきます。WSUSまたはKaspersky Security Centerと同じWebサーバーにポーリングDSCを導入する場合、構成作成スクリプトで次のパラメーターを変更する必要があります。

  1. UseSecurityBestPractices     = $false

    そうしないと、TLS 1.0が無効になり、SQLデータベースに接続できなくなります。Kaspersky Security Centerも機能しません（問題はKaspersky Security Center v11で解決されるはずです）。
  2. Enable32BitAppOnWin64   = $true

    この変更を行わないと、WSUSを使用するIISでAppPool DSCサーバーを起動できません。
  3. WSUSでDSCサーバーをインストールする場合は、DSCサイトの静的および動的キャッシュを無効にします。

SQLデータベースを使用するためのDSCサーバーの設定に移りましょう。

.mdb「プロキシ」ファイルの構成


SQLサーバーへのODBC接続の作成。DSCがインストールされているサーバーにMS Accessがインストールされていないことを前提としているため、databases.mdb構成は、MS Accessがインストールされている中間ホストで実行されます。

SQLサーバーへのシステムODBC接続を作成しましょう（接続のビット深度はMS Accessのビット深度と一致する必要があります-64または32）。以下を使用して作成できます：
-Powershellコマンドレット：

Add-OdbcDsn –Name DSC –DriverName 'SQL Server' –Platform '<64-bit or 32-bit>' –DsnType System –SetPropertyValue @('Description=DSC Pull Server',"Server=<Name of your SQL Server>",'Trusted_Connection=yes','Database=DSC') –PassThru

-または手動で、接続ウィザードを使用：

  1. 管理ツールを開きます。インストールされているMS Accessのバージョンに応じて、ODBCデータソースを選択します。[システムDSN]タブに移動して、システム接続を作成します（追加）。

  2. SQLサーバーに接続することを示します。完了をクリックします。

  3. 接続する名前とサーバーを指定します。次に、同じパラメーターを使用した接続をDSCサーバーで作成する必要があります。

  4. SQLサーバーに接続するには、以前に作成したDSCという名前のログインを使用することを示します。

  5. DSC接続設定でデータベースを指定します。

  6. 完了をクリックします。

  7. 設定を完了する前に、接続が機能していることを確認してください（テストデータソース）。

  8. できました。
