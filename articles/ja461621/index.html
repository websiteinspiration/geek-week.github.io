最初のVHDLプログラムを書くのは難しいですか？

最初のVHDLプログラムを書くのは難しいですか？言うのは難しいですが、ここでの主なものはモチベーションです...


たぶん私はこの瞬間を遅らせることができたかもしれませんが、近所の人が長方形のパルス発生器を作ってそれがはっきりと表示され、パルスの周波数と持続時間を制御できるようにするように頼みました。

そして、0.1マイクロ秒の精度で...

そして、インジケーターとボタンが付いたCPLD（200ルーブルなど）のスカーフに目が落ちました。あなたがそのようなことで働き始めるべきだと思ったら、そして...

VHDLとVerilogのどちらを書くかという選択はありませんでした-私はすべてCで記述しますが、それでもAdaは大好きです-したがって、VHDLはユニークです。さらに、FPGAの概要を読んだとき、複雑なものは何もないことに気付きました（少なくとも、このような単純なタスクでは）。

ですから、最初に私たち自身をジェネレーターにするという言葉がありました。ネイティブシュレッドの周波数は50 MHzです。つまり、クロックラインの切り替えが途中と最後になるように、周波数を10に減らします。これが起こったことです。

-- 100 ns signal generator
process(clk)
	variable t:integer range 0 to 5 := 0;
begin
 if rising_edge(clk) then
	t := t + 1;
  	if t = 5 then
	  t := 0;
	  tact <= not tact;
	end if;
  	if t = 2 then
	  tact <= not tact;
	end if;
 end if;
end process;

次に、何とか表示して管理する必要があります。私たちは2つの値を持っています-期間の長さとインパルスの長さ、つまり期間の長さに対して3つの親密度（10分の1を考慮に入れて）と期間の長さ-3を割り当てます。

shared  variable period : integer range 0 to 1000 := 500;
shared  variable duty : integer range 0 to 1000 := 250;

shared variable dig1:std_logic_vector(3 downto 0):="0000";
shared variable dig2:std_logic_vector(3 downto 0):="0101";
shared variable dig3:std_logic_vector(3 downto 0):="0010";

私は正直に言って、スカーフに付属のソースからコードの一部をドラッグしました-とてもクールで明確に書かれています！en_xhdl-この信号は、スイッチングサイクルでどのインジケーターがオンになるかを制御します。dataout_xhdl1-この信号は、LEDをオンにします。data4は一時レジスターであり、数字を格納します。

ジェネレーター自体-すべてを考慮する心を書くことは残っています。ここで、tactXは表示ジェネレーターで、cntはパルス位置カウンターです。まあ、linはジェネレータ自体の信号です。

process(tact)
 variable cntX : integer range 0 to 1000 := 0;
 variable cnt : integer range 0 to 1000 := 0;

begin
  if rising_edge(tact) then
   if cntX = 0 then
	 tactX <= tactX + 1;
	end if;
	
	cntX := cntX + 1;
	
	if cnt > period then
	 cnt := 0;
	else
	 cnt := cnt + 1;
	end if;

	if cnt = 0 then
	  lin <= '0';
	elsif cnt = duty then
	  lin <= '1';
	end if; 

	end if;
end process;
