Data Vaultの基本

現在、データ分析とBIの分野では、DATA VAULTのような概念を満たさないことはもはや不可能です。しかし、私の意見では、特にインターネットのロシア語セグメントでは、このトピックに関する情報が不足しています。


DATA VAULTの企業への適用に関する興味深い記事を見つけることができますが、基本と方法論は十分にカバーされていません。


英語のセグメントでは、物事ははるかに優れています。DATA VAULT方法論の著者および発明者による本を購入できますが、基本に焦点を当てたパブリックドメインの記事もあります。


これらの記事の1つに触発されて、私はDATA VAULT方法論の基本的なことをロシア語で伝えようとします。


DATA VAULT-起源


DATA VAULTの出現の主な前提条件は、環境の変動性の増大と、これらの変化に迅速に対応する必要性でした。たとえば、以前はEDW（エンタープライズデータウェアハウス）にきめ細かいデータグラニュレーションを持つ新しいデータソースがあります。DATA VAULT手法では、新しいソースからデータをすばやく追加できると想定されています。さらに、DATA VAULTを使用すると、履歴データを保存できるシステムを構築するのが簡単になります。


解剖学データボルト


DATA VAULTとデータウェアハウスを構築する他のアプローチの重要な違いは、同一のソース状態でデータをロードする必要があることです。ソースからDATA VAULTにデータを転送するプロセスには、変換や追加は含まれません。DATA VAULTアプローチは、ソースとの調整の可能性を意味します。データ変換プロセスは、後でDATA VAULTに基づいてデータマートを構築するときに実行されます。


ハブ（HUBS）


HUBはDATA VAULTの中核です。適切に構成されたHUBにより、企業ストレージ内のさまざまなデータソースを組み合わせることができます。ソースが独立していることが重要です。これに基づいて、各HUBは、他のビジネスオブジェクトに関連付けられていない独自の一意のビジネスキー（ビジネスキー）を持つ必要があります。
