サーバーWindowsの1つのオプションが原因で、サイトの速度が低下したという話



Cloud4Yが企業のクラウドプロバイダーであることをすでに多くの人が聞いています。したがって、私たちは自分自身について話すことはしませんが、いくつかのサイトへのアクセスに問題があった方法とこれを引き起こした原因についての短いストーリーを共有します。

ある晴れた日、マーケティング部門は、ブラウザでターミナルを介して作業しているとき、一部のサイトが長時間ロードされるとエンジニアに不平を言いました。特に、vk.comは彼らにとって不可欠です。私たちは信号を受け取り、問題が何であるかを理解し始めました。

したがって、状況は、インターネットプロバイダーMegafon、サーバーOS Windows、Firefoxブラウザーです。通常のWindows 10でVKontakteを開くと、サイトは10〜100ミリ秒で読み込まれます。 Windows Server 2012/16/19で開こうとすると、遅延は最大15秒、またはそれ以上になります。

彼らはVKピクセルを採用し、それを介して彼らは何が起こっているのか考えられるバージョンを解明し始めました。

仮説テスト＃1-ターミナルサーバーの問題。
確認されていません。同じネットワーク上の別のサーバーを介して開いているページをテストすると、問題が解決しません。

仮説テスト＃2-ゲートウェイの問題。
確認されていません。ローカルのラップトップでは、すべてが簡単かつ迅速に開くことに注意してください。しかし同時に、端末（および内部サーバー）にはまだ問題があります。外部および内部インターフェイスのICMP設定を試してみた-それは助けにはならなかった。

奇妙なことに、それが判明します。

ローカルのラップトップからサイトが遅くなることはありません。
内部スキャンマシン（スキャン用端末）から-速度が低下しません。
そしてマーケティングは減速します。めちゃくちゃ！

さらに進みましょう。

仮説テスト＃3-DNSの問題。
確認されていません。パブリックDNS（8.8.8.8）を介してピクセルを起動しました-同じ話。たとえば、初めてこのピクセルをシークレットモードでプルしたときに、問題がはっきりとわかります。

問題はブラウザに大きく依存しているという疑いがあります。 FFでは、ピクセルは常に最初のエントリのクロムで消滅します。マーケティングはすべてのブラウザで常に愚かです。

仮説テスト＃4-OSテンプレートのあるもの。
確認されていません。クリーンなWindows Server 2016を展開し、.0ネットワークからテストを開始しました。問題を抱えて。ネットワーク.200に転送されました。問題が解決しません。それがネットワーク.0のゲートです。それとは関係ありません。ただし、このネットワークのラップトップにはこの問題はありません。これが.200ネットワークゲートです。それとは関係ありません。

つまり、OSテンプレートにはありません。ピクセルの読み込みで仮想マシンの速度が低下します。しかし、VPN（別のネットワークカード）を作成し、そこにトラフィックを送ると、すべてが非常に速く機能します（あるはずです）。問題を引き起こす可能性のあるオプションが2つあることがわかります。オフィスのゲートウェイまたはオフィスのインターネットオペレーターです。

しかし、メガホンは特にVKontakteピクセルへのアクセスを遮断できますか？いいえ、ナンセンスです。私たちはまだかき回すようにしています。

仮説テスト＃5-VMware Toolsのせいです。
確認されていません。有害な影響は観察されません。カードの設定も変えてみました。 TTL変更-影響なし。まあ、それは一般的にWindows 10とWindows Serverの違いが何であるか不明です。しかし、違いがあります。 gopherの話のように。



私たちはかなり前からこの問題に対処しました。もちろん、Googleと同様の状況ですが、何も見つかりませんでした。したがって、彼らは手掛かりなしで行動し、すべての可能なバージョンを試しました。 Windows 2016ラップトップからテストして、ピクセルをロードするときの仮想化などがスローダウンのせいではないことを確認しました。ネットワークカードとIPスタックのすべての可能な設定を変更しました。すべての束を試しました。しかし、問題は残っており、マーケティングはひどい音を立てて、すべてを修正するように要求しました。

しばらくして、私たちはまだ犬が埋葬された場所を見つけました。それはすべてnetshインターフェイスtcp setglobal ecncapability = disabledオプション
に関するものでした

このオプションは、デスクトップWindowsではデフォルトで無効になっており、サーバーではデフォルトで有効になっています。サーバー上でオフにするとすぐに、デスクトップ上と同様に、すべてが瞬時に開きます。この問題は、オフィス（Megafon）でインターネットを提供しているプロバイダーから、Megafonのモバイルインターネット（携帯電話から共有し、Windows Server経由で接続している場合）、Yotaを通じて確認できました。モスクワの一部の地域で試行したところ、この問題はどこにでもありました。他のオペレーターに取り組んでいるとき、サイトへのアクセスは瞬時でした。

ある有名な政治家が言ったように、これはそのようなスカムです。原則として問題は解決しましたが、私たちは非常に興味があります。それは国内でのみ発生したのですか、それとも他の都市の企業に影響を与えている大規模な災害ですか？このケースが孤立していない場合、メガホンはこの問題の解決策を検討する必要があります。結局のところ、サーバーではECN（ecncapability）オプションがデフォルトで有効になっています。その意味を理解するには、多くの時間を費やす必要があります。

確認方法は？はい、私たちと同じように。 Firefoxブラウザーを使用して、任意のvk.comページを開き、ctrl + f5でもう一度開きます。問題がある場合は一定の遅延が発生し、問題がない場合はサイトがすぐに開きます。Cloud4Y
