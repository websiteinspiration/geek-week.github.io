什么是实施？


当一家小型企业决定实施CRM系统时，通常将其想象为“注册服务，入门”。众多CRM经销商大力鼓舞了这一策略，他们除了说服他们购买订阅外，还没有特别的能力（例如，著名CRM的合作伙伴，其主要业务是健身拉伸工作室？）。很大程度上由于这种用户期望，CRM市场收到了失败的项目，此后公司还没有准备好返回到另一个CRM。这是一次悲伤而昂贵的经历，触动了公司的精神和员工对管理的态度。

实施是选择，购买和调试CRM系统的过程，恰恰是符合公司要求并能够解决运营和分析活动问题的公司。

RegionSoft CRM中的远程工作方法-技术方面


System RegionSoft CRM支持通过Internet远程工作用户的功能。这仅适用于在不同地点设有多个办事处的公司，以及每个被迫不断或临时切换到远程工作的公司。同时，所有部门和员工都在同一信息系统中集中工作-所有数据都存储在一个地方，并由RegionSoft CRM中提供的所有信息安全工具可靠地保护。

有两种远程工作方式：

  • 终端模式
  • 通过DBMS端口的连接。

两种方法都支持在线工作，即 允许您实时访问位于远程计算机上的中央信息数据库。

方法1：终端模式（推荐方法）



终端模式是使用RDP（远程桌面管理）技术与中央计算机的连接。迄今为止，这种方法是最安全，最方便的。彼此远离的服务器和工作站都应使用建议的至少512 kbit / s的稳定通道连接到Internet。用户PC上的CRM使用RDP协议（使用远程桌面实用程序或其类似物）与服务器建立连接，系统在服务器端启动，或使用RemoteApp技术。此外，在线开展全面的工作。授予远程用户对系统所有功能的完全访问权限。

RegionSoft CRM支持的RDP协议基于瘦客户端技术。该技术的优点在于，在工作期间，将启动应用程序，并在服务器端执行所有数据操作，从而大大减少了数据传输通道上的负载。在这种模式下，可以以最小的通道和远程工作站负载处理大量数据。连接后，将由操作系统授权用户，以确保适当的安全性。

作为建立终端操作模式的一种手段，可以使用不同的技术。但是，最优质的解决方案以及允许您确保适当级别的安全性的解决方案被认为是Microsoft Windows Server附带的Microsoft终端服务。

当前，超过90％的具有远程单位或员工的RegionSoft CRM客户端成功使用了终端连接方法。在“自我隔离时代”引入CRM的正是他现在对我们的新客户绝对优先。 

方法2：通过DBMS端口连接


通过DBMS端口的连接是远程连接到中央数据库的另一种选择。这样可以从远程站直接访问Firebird端口。远程用户正在使用的数据将传输到客户端并在客户端进行处理。因此，这种远程连接方法对Internet通道的速度和稳定性提出了更高的要求。

彼此远离的服务器和工作站都应以建议的至少1 Mb / s的稳定通道连接到Internet。在从工作站连接到数据库的设置中，必须指定数据库服务器的静态IP地址和DBMS端口，例如“ 192.168.0.122/3059”。不需要其他实用程序。 CRM系统在远程工作站上运行。

工作是在线进行的。授予远程用户对系统所有功能的完全访问权限。

这种方法的缺点是连接安全性降低，并且在低连接速度和不稳定的Internet通道下数据下载时间较长。

远程连接设备


可以通过以下设备执行远程连接的终端方法：

  • 运行Windows，MacOS，Linux的个人计算机；
  • 运行Android，iOS，Windows的平板电脑和智能手机；支持RDP协议的任何其他设备，包括采用SmartTV技术的现代电视。

通过通过DBMS端口进行远程连接的方法，可以使用以下设备：

  • 运行Windows，Mac OS，Linux的个人计算机。
