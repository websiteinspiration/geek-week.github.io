脅威スポットライト：Neshta File Virus

敬礼、ハブロバイト！コース「Reverse Engineering 2.0」の開始を見越して、別の興味深い翻訳を皆さんと共有したいと思います。



短いレビュー


Neshtaはかなり古いファイルウイルスであり、依然として蔓延しています。 2003年に発見され、以前はBlackPOSマルウェアに関連付けられていました。感染したファイルに悪意のあるコードを追加します。基本的に、この脅威は意図しないダウンロードやその他の悪意のあるプログラムを通じて環境に侵入します。 Windowsの実行可能ファイルに感染し、ネットワークリソースやリムーバブルメディアを攻撃する可能性があります。

2018年、Neshtaは主に製造に焦点を当てましたが、金融、消費者、エネルギー部門も攻撃しました。安定性の理由から、Neshtaは自身の名前をsvchost.comに変更し、.exeファイルが起動されるたびに起動するようにレジストリを変更します。この脅威はシステム情報を収集し、POSTリクエストを使用して、攻撃者が制御するサーバー上のデータを引き出します。私たちの分析で使用されたNeshtaバイナリは、データ漏えいの動作や機能を示していません。

ファイル感染


Neshtaの主な機能は、ローカルドライブ上の.exeファイルを検索するファイル侵入者です。Neshtaは「.exe」ファイルをターゲットにしていますが、ショートカットに次の行のいずれかを含むファイルのみを除外しています。

  • ％Temp％
  • ％SystemRoot％（通常はC：\ Windows）
  • \プログラ〜1 \


感染プロセスの概要を以下と図3に

示します。

  1. ターゲットソースファイルの先頭から41,472（0xA200）バイトを読み取ります。
  2. 2つのパーティションを作成し、ソースファイルの最初と最後にPAGE_READWRITE属性を指定してメモリを割り当てます。
  3. 悪意のあるヘッダーとコードをソースファイルの先頭に配置します。記録されるデータは41,472バイトです。
  4. エンコードされたソースヘッダーとコードを、サイズが41.472バイトのファイルに書き込みます。


これらのアクションにより、感染したファイルを起動した直後に悪意のあるコードを実行できます。


図3：ファイルの感染

感染したファイルが起動されると、ソースプログラムが配置され%Temp%\3582-490\<filename>、WinExec APIを使用して起動されます。

BlackBerry CylanceがNeshtaを停止


BlackBerry Cylanceは、何百万もの安全なファイルと安全でないファイルの両方で脅威を検出するようにトレーニングされたAIベースのエージェントを使用します。当社の自動セキュリティエージェントは、特定のファイル署名に依存するのではなく、さまざまなファイル属性と悪意のある動作に基づいてNeshtaをブロックします。ゼロデイ脅威に対して予測優位性を提供するBlackBerry Cylanceは、新しい既知のサイバー攻撃に対して訓練され、効果的です。詳細については、https：//www.cylance.comにアクセスしてください。
