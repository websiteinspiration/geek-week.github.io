Apartemen insinyur otomatis. Bagian 1: pernyataan masalah, pilihan besi

Selama beberapa bulan sekarang saya telah menjadi pemilik kotak dua kamar yang konkret. Setelah mendapatkan apartemen, permintaan perbaikan minimal - untuk membuat kesenangan dan kenyamanan untuk uang minimal, jika mungkin, dengan tangan Anda sendiri. Akibatnya, saya harus beralih ke perancang untuk kenyamanan, dan konsep kenyamanan termasuk elemen rumah pintar.

Versi pertama dari garis besar rumah pintar diturunkan untuk mengendalikan pemanasan di bawah lantai dan kontaktor listrik pusat. Tapi, seperti yang Anda tahu, nafsu makan tumbuh bersama makanan. Saat ini, daftar kami telah berkembang ke poin-poin berikut:

  • kontrol lampu;
  • kontrol iklim (pemanas di bawah lantai, radiator, AC);
  • manajemen tirai;
  • manajemen sistem perlindungan kebocoran;
  • pengukuran otomatis konsumsi air dan listrik;
  • integrasi multi-kamar;
  • Kontrol TV;
  • kemampuan untuk mengendalikan segala sesuatu dari smartphone;
  • tamu tidak harus mengalami ketidaknyamanan, kontrol elemen pencahayaan harus seperti di apartemen biasa

Seperti yang Anda lihat, daftar Wishlist telah meluas secara signifikan, sehubungan dengan masalah pemilihan besi untuk implementasi menjadi akut. Sebagai seorang insinyur otomasi industri, saya segera menolak opsi seperti arduino dan semua jenis perangkat "Wi-Fi" pintar yang ditawarkan Xiaomi. Hanya PLC prefabrikasi dengan modul I / O dan sakelar kabel.

Sebagai hasil dari mempelajari semua jenis informasi selama beberapa bulan, saya sampai pada kesimpulan bahwa akan lebih optimal untuk menggunakan PLC bersamaan dengan OpenHAB atau platform rumah pintar Home Assistant. Tapi sekarang landasannya adalah pilihan PLC. Pilihannya sekarang datang dari beberapa opsi.

Opsi 1: Segnetics Matrix + Modul I / O FMR


Opsi mid-budget.



Pro:

  • Lingkungan pengembangan SMLogix sederhana
  • Ada komunitas forum dan pengguna.

Minus:

  • kurangnya server web bawaan;
  • tidak jelas apakah akan mungkin untuk memperketat MQTT;
  • controller baru, mungkin ada tiang temboknya.

Opsi 2. ARIES PLC210 + modul input / output MV210, MU210


Opsi mid-budget.



Pro:

  • Lingkungan pengembangan Codesys 3.5 dengan server WEB terintegrasi;
  • dukungan perspektif untuk MQTT;
  • ada forum dan komunitas

Minus:

  • pengontrol baru yang tidak dapat digulung, tiang tembok dimungkinkan;
  • tidak ada kepercayaan pada pabrikan (ada pengalaman negatif dalam implementasi proyek).

Opsi 3. PLC dan Modul Berghof I / O


Opsi anggaran tinggi.



Pro:

  • Lingkungan pengembangan Codesys 3.5
  • pengontrol yang teruji andal;
  • Modul I / O EtherCAT.

Minus:

  • harga (opsi termahal yang tercantum);
  • tidak ada dukungan untuk MQTT.

Opsi 4: Modul WirenBoard 6 dan I / O


Opsi anggaran rendah.



Pro:

  • harga (opsi termurah yang tercantum);
  • ada dukungan untuk berbagai protokol rumah pintar, termasuk MQTT;
  • semua modul yang diperlukan untuk rumah pintar tersedia dari pabrikan;
  • Eksekusi modular, diperlukan kabinet standar dengan plastron.

Minus:

  • desain server WEB tertanam tidak dapat dikonfigurasi;
  • Bahasa pemrograman berbasis JavaScript (akan sulit untuk mengimplementasikan semua ide, karena lingkungan baru bagi saya);
  • sedikit pemrograman dan dokumentasi konfigurasi.
  • tidak ada komunitas pengguna.

Secara umum, dari empat opsi yang sekarang saya pilih antara Aries dan WirenBoard. Di sisi WirenBoard sekarang ada banyak protokol dan penajaman diklaim untuk rumah pintar, di sisi Aries pengalaman saya dengan Codesys dan jaminan implementasi kontrol dari antarmuka WEB tanpa OpenHAB dijamin.

Sementara itu, saya tersiksa oleh pertanyaan memilih produsen PLC, saya tidak membuang-buang waktu dengan sia-sia dan membuat rangkaian listrik dari listrik apartemen, yang akan saya ceritakan di artikel berikutnya.
