Xcode 11に登場したXCTestPlanツールを使用すると、テストの構成をいくつか作成できます。現在、構成は、アプリケーションの言語の選択など、テストの実行方法を決定するために最もよく使用されています。この記事では、XCTestPlanを使用してスクリーンショットを自動化する方法を学びます。

スクリーンショットの自動化


スクリーンショットを作成するUIテストは、添付ファイルを作成することを除いて、通常のテストとほとんど変わりません。 XCTAssertチェックを追加して、問題が発生した場合にテストが失敗するようにすることもできます。

別の重要なポイント：スクリーンショットが撮られるまでにUIの準備ができていない場合（キーボードの表示 /非表示、アニメーションの終了）には、waitForExpectationsが必要になる場合があります。

以下は、すべてのヘルパーメソッドの例です。

func testScreenshots() {
    let app = XCUIApplication()
    let searchButton = app.buttons["search"]
    searchButton.tap()
    
    let keyboard = app.keyboards.firstMatch
    waitForExistence(of: keyboard)
    app.typeText("Cupertino")
    add(takePromoShot(name: "Search"))
    
    let firstResult = app.cells.firstMatch
    firstResult.tap()
    waitForDisappearance(of: keyboard)
    add(takePromoShot(name: "Result"))
}


func waitForExistence(of element: XCUIElement) {
    let predicate = NSPredicate(format: "exists == TRUE")
    expectation(for: predicate, evaluatedWith: element, handler: nil)
    waitForExpectations(timeout: 5.0, handler: nil)
}


func waitForDisappearance(of element: XCUIElement) {
    let predicate = NSPredicate(format: "exists == FALSE")
    expectation(for: predicate, evaluatedWith: element, handler: nil)
    waitForExpectations(timeout: 5.0, handler: nil)
}


func takePromoShot(name: String) -> XCTAttachment {
    let lang = Locale.preferredLanguages[0]
    let screenshot = XCUIScreen.main.screenshot()
    let attachment = XCTAttachment(screenshot: screenshot)
    attachment.lifetime = .keepAlways
    attachment.name = "\(lang)-\(name)"
    return attachment
}

最新の機能に注意してください。スナップショットの名前を取得し、言語識別子を追加し、スクリーンショットを取得して、.keepAlwaysパラメーターの値を含むXCTAttachmentを返します。デフォルトでは、テストが成功した場合、テスト中に取得したスクリーンショットは削除されるため、これは重要です。Lifitemeは、テストの完了後にスクリーンショットが確実に保存されるため、重要なパラメーターです。UIテスト中、スクリーンショットは常に撮られます。なぜ、いつテストが落ちたのかを判断する必要があるときに、これらは非常に役立ちます。ただし、テストが成功した場合、デフォルトでスクリーンショットは削除されます。したがって、testScreenshots（）が成功すると、必要なスクリーンショットのみが表示されます。

通常のステータスバー


ほとんどの開発者は、適切な時間、完全なネットワーク信号、およびその他の詳細を示すステータスバーをスクリーンショットで確認することを好みます。Xcode 11では、これはさらに簡単になります。これらの設定を指定できるユーティリティxcrun simctl status_barがあります。これはビルドフェーズとして追加できます：

xcrun simctl status_bar booted override --time 9:41 --operatorName ' ' --cellularMode active --cellularBar 4 --wifiBars 3 --batteryState charged

このコマンドの後、ステータスバーはAppleのマーケティング資料と同じように見えます。残念ながら、チームは実際のデバイスでは機能せず、シミュレータでのみ機能します。スクリーンショットを収集するためのテストを開始する前に、使用するシミュレーターが実行されている必要があることにも注意してください。

結果のスクリーンショットを収集する


これで、UIテストを実行できます。合格すると、Xcodeのレポートナビゲーターに結果が表示されます。言語構成ごとにグループ化された手順を含むテストもあります。添付ファイルがある手順には、ペーパークリップが付いています。画像自体はQuickLookで開くことができます。



特にアプリケーションが多くの言語をサポートしている場合は、結果の画像をすべて個別に収集するのは非常に時間がかかります。残念ながら、これらの添付ファイルはテスト結果でpngまたはjpgとして表示されません（Xcode 11の別の変更）。ただし、テスト結果を解析して、実際の添付ファイルをポイントして収集するJSONを見つけることができます。これは、xcparseユーティリティを使用して実行できます。。これはHomebrewを使用してインストールされ、次のコマンドで起動されます。

xcparse screenshots /path/to/Test.xcresult /path/to/outputDirectory

追加のフラグは、デバイス、OS、または構成によってファイルをグループ化します。ファイルの名前には言語識別子が含まれているため、名前で並べ替えることができます。
