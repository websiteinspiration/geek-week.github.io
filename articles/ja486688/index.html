Node.js、Tor、Puppeteer、およびCheerio：匿名のWebスクレイピング

Webスクレイピングは、Webサイトからデータを収集する方法です。この用語は通常、自動データ収集に関連して使用されます。今日は、サイトから匿名でデータを収集する方法について説明します。誰かがWebスクレイピングで匿名性を求める理由は、多くのWebサーバーがIPアドレスからの接続に特定のルールを適用し、そこから特定の期間に特定の数の要求が行われるためです。ここでは、次のツールを使用します。

  • Puppeteer -Webページにアクセスするため。
  • Cheerio -HTMLコードを解析するため。
  • Tor-各リクエストを異なるIPアドレスから実行します。

ウェブスクレイピングの法的側面は複雑で、多くの場合不明確な問題であることに注意してください。したがって、データを収集するページの「利用規約」を尊重してください。このトピックに関するいくつかの優れた資料を次に示します。



Torのインストール


最初から始めましょう-したがって、まず最初に、次のコマンドを使用してTorクライアントをインストールします。

sudo apt-get install tor

Torのセットアップ


次に、Torクライアントを構成します。Torのデフォルト設定はSOCKSポートを使用します。これにより、単一の出力ノード（つまり、1つのIPアドレス）への1つのパスが提供されます。Torを日常的に使用する場合、たとえばWebを閲覧するだけの場合は、これが適しています。ただし、いくつかのIPアドレスが必要です。これにより、ウェブスクレイピングプロセス中にそれらを切り替えることができます。

Torを必要に応じて構成するには、SOCKS接続をリッスンするために追加のポートを開くだけです。これはSocksPort、にあるプログラムのメイン構成ファイルにいくつかのエントリを追加することによって行われます/etc/tor。テキストエディタを使用し

てファイル/etc/tor/torrcを開き、ファイルの最後に次のエントリを追加します。

#  4 SOCKS-,       Tor-.

SocksPort 9050
SocksPort 9052
SocksPort 9053
SocksPort 9054

ここでは、次のことに注意する価値があります。

  • SocksPort . — , , SOCKS, .
  • SocksPort , , , - .
  • 9050 — , Tor- .
  • 9051. Tor , , , Tor.
  • 9051, , 1.

構成ファイルに加えられた変更を適用するには、Torクライアントを再起動します。

sudo /etc/init.d/tor restart

Puppeteerを使用してWebサイトを操作する


Puppeteerは、DevToolsプロトコルを使用してChromeまたはChromiumと対話するユーザーインターフェースのないブラウザです。ここでライブラリを使用してtor-requestなどのリクエストを処理しないのは、そのようなライブラリは、コンテンツが動的に読み込まれる単一ページのWebアプリケーションとして作成されたサイトを処理できないためです。 ファイルを作成し、次のコードをその中に入れます。このコードの主な機能はコメントに記載されています。

ドキュメントの構造を調べる

ページのHTMLコードから必要なものを抽出するには、次の一連のアクションを実行する必要があります。

  • Torプロキシに接続されたユーザーインターフェイスなしで新しいブラウザインスタンスを起動します。
  • 新しいページを作成します。
  • 住所に移動しますhttps://news.ycombinator.com/。
  • ページのHTMLコンテンツを取得します。
  • ページのHTMLコンテンツをにロードしますcheerio。
  • 記事のタイトルを保存する配列を作成します。
  • クラスを使用して要素にアクセスするstorylink。
  • cheerio slice（）メソッドを使用して最初の5つの要素を取得します。
  • cheerio each（）メソッドを使用して、結果の要素をトラバースします。
  • 見つかった各ヘッダーを配列に書き込みます。

これらのアクションを実装するコードは次のとおりです。

const puppeteer = require('puppeteer');

/**
 *   cheerio.
 */
const cheerio = require('cheerio');

async function main() {
  const browser = await puppeteer.launch({
    /**
     *       (   ).
     */
    headless: true,
    args: ['--proxy-server=socks5://127.0.0.1:9050']
  });

  const page = await browser.newPage();

  await page.goto('https://news.ycombinator.com/');

  /**
   *      HTML-.
   */
  const content = await page.content();

  /**
   *    cheerio.
   */
  const $ = cheerio.load(content);


  /**
   *      .
   */
  const titles = [];

  /**
   *   ,   `storylink`.
   *  slice()      5   .
   *      each().
   */
  $('.storylink').slice(0, 5).each((idx, elem) => {
    /**
     *   HTML-,   .
     */
    const title = $(elem).text();
  
    /**
     *    .
     */
    titles.push(title);
  })

  browser.close();
  
  /**
   *     .
   */
  console.log(titles);
}

main();

異なるIPアドレスを使用した継続的なスクレイピング


次に、ファイルで指定したさまざまなSOCKSポートの使用方法について説明しますtorrc。とてもシンプルです。それぞれがポート番号を含む配列を宣言します。次に、関数の名前を関数main()に変更し、その関数を呼び出すscrape()新しい関数main()を宣言して、呼び出しscrape()ごとに新しいポート番号を渡します。

これが完成したコードです：

const puppeteer = require('puppeteer');
const cheerio = require('cheerio');

async function scrape(port) {
  const browser = await puppeteer.launch({
    args: ['--proxy-server=socks5://127.0.0.1:' + port]
  });

  const page = await browser.newPage();
  await page.goto('https://news.ycombinator.com/');
  const content = await page.content();

  const $ = cheerio.load(content);

  const titles = [];

  $('.storylink').slice(0, 5).each((idx, elem) => {
    const title = $(elem).text();
    titles.push(title);
  });

  browser.close();
  return titles;
}

async function main() {
  /**
   *  SOCKS- Tor,    torrc. 
   */
  const ports = [
    '9050',
    '9052',
    '9053',
    '9054'
  ];
  
  /**
   *  -...
   */
  while (true) {
    for (const port of ports) {
      /**
       * ...  -    .
       */
      console.log(await scrape(port));
    }
  }
}

main();
