研究：ゲーム理論を使用したブロッキングプロキシサービスの作成



数年前、マサチューセッツ、ペンシルベニア、ドイツのミュンヘンの大学からの科学者の国際チームが、検閲と戦うためのツールとしての伝統的なプロキシの有効性に関する研究を行いました。その結果、科学者はゲーム理論に基づいてロックをバイパスする新しい方法を提案しました。私たちはこの作品のハイライトの翻訳を準備しました。

前書き


Torなどの一般的なブロッキングバイパスツールのアプローチは、ブロッキングにさらされているリージョンからのクライアント間のプロキシIPアドレスのプライベートで選択的な配布に基づいています。その結果、顧客はブロックする組織や当局に気付かれないようにする必要があります。 Torの場合、そのようなプロキシディストリビュータはブリッジと呼ばれます。

このようなサービスの主な問題は、インサイダー攻撃です。ブロッキングエージェントは、プロキシ自体を使用してアドレスを見つけ、ブロックすることができます。プロキシを計算する可能性を最小限に抑えるために、ロックバイパスツールはさまざまなアドレス割り当てメカニズムを使用します。

この場合、いわゆるアドホックヒューリスティックのアプローチが使用されますが、これは回避できます。この問題を解決するために、科学者たちは、ブロッキングに関連するサービスとそれらを回避するサービスの闘争をゲームとして提示することを決定しました。彼らはゲーム理論を使用して、各当事者に最適な行動戦略を開発しました-特に、これによりプロキシ配布メカニズムを開発することが可能になりました。

従来のロックバイパスシステムのしくみ


Tor、Lantern、Psiphonなどのブロッキングバイパスツールは、制限が導入されたリージョン外の多くのプロキシを使用します。これらのプロキシを使用して、ユーザーのトラフィックをこれらのリージョンからブロックし、リソースをブロックされたリソースに切り替えます。

検閲者がそのようなプロキシのIPアドレスを知った場合（たとえば、自分で使用した後）、ブラックリストに登録してブロックするのは簡単です。したがって、実際には、そのようなプロキシのIPアドレスは決して公開されず、ユーザーへのプロキシの割り当てはさまざまなメカニズムを使用して行われます。たとえば、Torにはブリッジシステムがあります。

つまり、主なタスクは、ブロックされたリソースへのアクセスをユーザーに提供し、プロキシアドレスを開示する可能性を最小限に抑えることです。

この問題を実際に解決するのはそれほど簡単ではありません。一般ユーザーを偽装した検閲者と正確に区別することは非常に困難です。ヒューリスティックなメカニズムは、情報を隠すために使用されます。たとえば、Torはクライアントが使用できるブリッジIPアドレスの数を1つのリクエストで3つに制限します。

これは、中国当局が短時間ですべてのTor橋を計算することを止めませんでした。追加の制限の導入は、バイパスシステムのユーザビリティに深刻な影響を与えます。つまり、一部のユーザーはプロキシにアクセスできなくなります。

ゲーム理論がこの問題を解決する方法


作品に記載されている方法は、いわゆる「大学入試ゲーム」に基づいています。さらに、打ち切りインターネットエージェントはリアルタイムで相互に通信し、複雑な戦術を使用できると想定されています。たとえば、さまざまな条件に応じて、プロキシをすぐにブロックしたり、即座にブロックしたりしないでください。

大学入学はどのように機能しますか？


n人の学生とm人の大学があるとします。各学生は、いくつかの基準に基づいて、教育機関間の好みのリストを編集します（つまり、ドキュメントが提出される大学のみがランク付けされます）。一方、大学はまた、自分の好みに基づいて書類を提出する学生をランク付けします。

まず第一に、大学は選択基準を満たしていない人を切り捨てます-欠乏した場合でも、彼らは採用されません。次に、必要なパラメータを考慮したアルゴリズムに従って応募者が選択されます。

「不安定な収入」が存在する可能性があります。たとえば、2人の学生1と2が大学aとbにそれぞれ入学したが、2人目の学生が大学aで勉強したい場合などです。説明した実験の場合、オブジェクト間の安定した関係のみが考慮されました。


すでに述べたように、大学はどんな状況でも受け入れない学生の数があります。したがって、据え置き受け入れアルゴリズムでは、これらの学生がこの大学に書類を提出することは許可されていないと想定されます。この場合、すべての学生が最も好きな大学に入ろうとします。

各学生が大学の待機リストにあるか、彼が行くことができるすべての教育機関を拒否された場合、手順は終了します。その結果、大学は最終的にすべての待機リストを登録します。

プロキシはそれと何をしているのですか


学生や大学との類推によって、科学者は各クライアントに特定のプロキシを割り当てました。結果は、代理割り当てゲームと呼ばれるゲームです。潜在的な検閲エージェントを含むクライアントは、大学の役割を果たすプロキシのアドレスを知りたい学生の役割を果たします-彼らは事前に決められた最終的なスループットを持っています。

説明されているモデルでは、n個のユーザー（クライアント）A =
{a 1、a 2、...、a n }がいて、プロキシへのアクセスを要求してロックをバイパスします。したがって、a iは「合計」クライアントの識別子です。これらのn人のユーザーの中で、mは検閲エージェントであり、J = {j 1、j 2、...、j m}、残りは普通のユーザーです。すべてのmエージェントは中央当局によって制御され、そこから指示を受け取ります。

また、プロキシセットP = {p 1、p 2、...、p l } があると考えられています。各リクエストの後、クライアントはディストリビュータからk個のプロキシに関するオブジェクト情報（IPアドレス）を受け取ります。時間はインターバルステージに分割され、tで示されます（ゲームはt = 0から始まります）。

各クライアントは、スコアリング関数を使用してプロキシを評価します。科学者は、関数を使用して、ユーザーa iがステージtでプロキシp xを割り当てたスコアをマークしました。同様に、各プロキシは関数を使用してクライアントを評価します。それは-プロキシp xステージtでクライアントにiを割り当てた。

ゲーム全体が仮想であること、つまり「ディストリビューター」自身がプロキシとクライアントに代わってプレイすることを覚えておくことが重要です。これを行うために、彼はクライアントのタイプ、プロキシに関する彼らの好みを知る必要はありません。ゲームは各ステージで行われ、遅延採用アルゴリズムも使用されます。
