2020年に注目すべきテスト傾向

敬礼、ハブロフスク。モバイルQAエンジニア2.0コースの開始を見越して、2020年のテスト傾向に関する別の資料の翻訳を用意しました。



ソフトウェアテストの分野は日々進化しています。2019年に出現したトレンドの発展と新しいトレンドの形成を目の当たりにしています。今年、テスト自動化の専門家チームは、ソフトウェアテストの主な傾向に関していくつかの予測を行いました。それらに慣れることをお勧めします！

2019年のソフトウェアテストの傾向については、こちらの記事をご覧ください。

1.テストにおける人工知能と機械学習


受け取った多くのレポートに基づいて、インテリジェントな自動化は2020年のテスターのレーダーに残ると言っても安全です。

人工知能と機械学習の使用は、ソフトウェアテストの分野ではもはや新しいアイデアではありません。 AIはテストをより読みやすくします。チームはAI / MLを使用して自動化戦略を最適化し、より迅速に適応して作業効率を向上させることができます。

2019年、品質保証チーム（QA）はAI / MLを使用して、テストの品質の予測、テストケースの優先順位付け、エラーの分類、テストオブジェクトの検出、テスト済みアプリケーションとの対話（AUT）、およびその他の多くの目標を達成しました。

人工知能は、情報技術のあらゆる分野のあらゆる場所に適用されることが期待されています。この分野への投資は、北米だけでも約60〜70億ドルのレベルです。予測によると、2025年までに、投資はほぼ2,000億ドルに達すると予測されています。もちろん、多数のテスト領域でAIが使用されることを期待していますが、それらの多くはレポートと分析に関連しています。

  • ログ分析：手動および自動テストを必要とする固有のテストケースを特定します。
  • テストスイートの最適化：冗長で無駄なテストケースを検出して排除します。
  • テスト要件カバレッジの提供：要件トレースマトリックス（RTM）からのキーワードの取得。
  • 予測分析：重要なパラメーターとユーザーの行動の詳細を予測し、注目すべきアプリケーションの領域を特定します。
  • エラー分析：ソフトウェア製品の領域とビジネスリスクに関連するエラーの特定。

インテリジェントな自動化の基礎となるもう1つの柱は、機械学習です。2020年に、MLは新しいレベルのアプリケーションに入ります。Capgemini World Qualityレポートによると、組織の38％が2019年に機械学習プロジェクトの実装を計画していました。業界の専門家は、この数字は今年増加すると予測しています。

これは組織にとって何を意味しますか？


テストにおけるAIとMLの需要の増加にもかかわらず、専門家はこれらのテクノロジーはテストでのみ出現していると信じています。しかし、すぐに彼らの成長が見られます。

AIが品質保証チームに新しい要求を課すにつれて、アジャイルチームは、データサイエンス、統計、数学の学習を含むAI関連のスキルの導入を開始する必要があります。新しいスキルは置き換えられませんが、自動テストおよび開発（S-DET）の分野での基本的なスキルを補完します。

さらに、ビジネスの洞察力は別の重要なスキルになります。優れたテスターは、優れたAIスキルと卓越したスキルを組み合わせる必要があります。昨年、AI QAアナリストやデータ分析およびテストスペシャリストなどの新しい投稿が登場しました。

自動化ツールの開発者としての彼らの仕事は、最も実用的なツールの作成に集中することです。

企業は、概念をテストし、予算を考慮しながらAIを最適に実装するための従来の方法を修正します。人工知能をサポートする優れたツールは、生産ログの表示、テストスクリプトの生成、生産活動への対応など、経済的ビジネス効率と技術的側面の両方を提供する必要があります。

2.アジャイルチームでのテストの自動化


テストの自動化は、品質保証の分野では目新しいものではありません。実際、IT企業の44％が2019年に 50％以上を自動化することを計画していました。予測によると、2020年には自動テストの実装の割合が増加するだけです。

ますます多くの企業が最新のアジャイルプロセスとDevOpsプロセスを実装して高品質の製品をできるだけ早く提供するようになるにつれて、テストの自動化がこのシステムの不可欠な部分になりつつあります。テストの自動化は、チームが反復的なタスクを完了し、エラーをすばやく検出し、継続的なフィードバックと完全なテストカバレッジを提供するのを支援する方法をリードしています。したがって、品質保証プロセスに自動テストを実装する組織は、財務、時間、および人的リソースを大幅に節約できます。

2020年のテスト自動化は、オープンツールと有料ツールを組み合わせて使用​​するミレニアル世代の起業家に特に歓迎されることが期待されています。

これはQAプロフェッショナルにとって何を意味しますか？

いずれにしても、テストの自動化は完全に手動のテストを妨げるものではありません。実際、優れたQAチームは、効率を最大化し、適切なソフトウェア品質を確保するために、手動テストと自動テストを適切に組み合わせる必要があります。自動テストの重要性に異議を唱えることは困難ですが、探索テストやユーザビリティテストなど、一部のタイプのテストは依然として手動で実行する必要があります。

さらに、QA専門家は、スマートな全体的かつ分野横断的な環境を開発する必要があります。組み立てから展開までのプロセスを自動化する必要性がますます高まっています。現在、テストの自動化は機能要件ではなく、製品ライフサイクルの不可欠な部分と見なされています。

言うより簡単だ。これが、多くの組織が自動化を最大限に活用して、望ましい投資収益を得ることができなかった理由です。 Capgemini World Qualityレポートには、自動化を機会と見なす代わりに、テスターはそれを大規模でインテリジェントな接続されたプラットフォームと考える必要があるという推奨事項があります。

4. QAOps：品質保証とDevOpsの変革


QAOpsという用語について聞いたことがない場合、これは時間の問題です。

情報技術サービス（Ops）の分野での開発（Dev）と運用を組み合わせたソフトウェア開発の一連の方法であるDevOpsについては、すでにご存じかもしれません。DevOpsの主な目標は、システム開発ライフサイクル（SDLC）を簡素化することです。一方、チームは、ビジネス目標を満たす新しい機能の作成、バグの修正、頻繁な更新に集中できます。DevOpsは、開発者とビジネス担当者間のコラボレーションを合理化します。

同様に、QAOpsは、テストをCI / CDパイプラインに統合することで、テスターと開発者間の直接コミュニケーションのフローを増やし、チーム内のテスターの作業を他のチームと確実に連携させるのに役立ちます。簡単に言えば、QAOpsは2つの主要な原則に基づいています。

  1. CI/CD.
  2. CI/CD.



FacebookはQAOpsの実装の最良の例の1つです。 2014年、FacebookチームはFacebook Graph API 2.0に切り替えて、すべてのアプリケーションでログインレビューを強制することを決定しました。スムーズな移行を確実にするために、チームは5000の最大のアプリケーションで新しいバージョンをテストしたいと考えました。社内テストは不可能であることが判明したため、外部委託のQAOpsを使用することにしました。最終的に、チームは1か月で5,000を超えるアプリケーションをテストでき、重大な問題を解決することができました。これは、Facebookのチームだけがこのプロセスに関与している場合は不可能です。

QAOpsは、グローバルテクノロジー企業だけでなく、中小規模のチームでも使用できます。このプラクティスは、あらゆるビジネスの規模に合わせて拡張できます。

DevOpsに引き寄せられるチームが増えるにつれ、2020年にQAOpsの上昇が見込まれます。

モノのインターネット（IoT）デバイスのテストの増加は、2019年に注目されました。Gartnerによると、2020年に

モノのインターネットデバイスの数は205億に達するはずです。IoTテストは、セキュリティ、使いやすさ、信頼性、デバイスバージョンとプロトコルの互換性、ソフトウェア要素の多様性、接続遅延の監視のためのモノのインターネットデバイスのテストですスケーラビリティ、データ整合性評価、デバイスの信頼性など

IoTテスターは、この領域で膨大な量の作業に直面することが多く、特に通信プロトコルとオペレーティングシステムの監視、およびモノのインターネットシステムのさまざまな要素の多数の組み合わせに直面しています。このため、テスターのチームは継続的に知識を拡大し、IoTテストのユーザビリティ、セキュリティ、パフォーマンスの分野でスキルレベルを向上させる必要があります。

IoTテスターが今後数年間で直面するもう1つの問題は戦略です。モノのインターネットのデバイスとアプリケーションの数が指数関数的に増加しているという事実にもかかわらず、回答者の34％は、製品にモノのインターネットモジュールが存在しているにもかかわらず、品質保証チームはまだ確立されたテスト戦略を持っていないと世界品質レポートで述べています。

6.サイバーセキュリティとリスク管理の要件


デジタル革命により、セキュリティの脅威が高まっています。多くの業界のほとんどすべての企業のCIOとCTOは、自社のソフトウェア、アプリケーション、ネットワーク、システムのセキュリティをテストすることの重要性を認識しています。ソフトウェア開発チームはパートナー企業と協力して、自社製品に適切なレベルのセキュリティを提供できるよう支援します。

セキュリティテストは、トランザクションのセキュリティ（お金かデータかに関係なく）だけでなく、ユーザーの個人データの保護も提供します。サイバー脅威はいつでも、どのような形式でも発生する可能性があるため、セキュリティテストは今後も人気のトピックとなるでしょう。

2020年のソフトウェアテストの主な傾向に関する予測のリストを以下に示します。デジタル変革がどのように行われても、テスターやソフトウェア会社が新しい変更や調整を続けていくのは間違いありません。その結果、この絶え間なく変化する業界で柔軟性を維持するために、品質の高いチーム、リーダー、専門家は常に進化しなければなりません。
