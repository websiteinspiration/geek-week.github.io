企業開発チームがGitLabとMattermost ChatOpsを使用して開発をスピードアップする方法

また会いましたね！2月に、OTUSはAWS、Azure、Gitlabで新しいCI / CDコースを開始します。コース開始前夜、役立つ資料の翻訳を用意しました。




DevOpsツール、オープンソースメッセンジャー、ChatOpsの完全なセット-どうすれば恋に落ちることができますか？

製品をより速く、より効率的に作成したいという願望から、現在のように開発チームにこれほどのプレッシャーはありませんでした。DevOpsの人気の高まりは、主に開発サイクルの高速化、柔軟性の向上、チームの迅速な対応を支援するという期待に基づいています。DevOpsツールの可用性と包括性は過去数年間で大幅に向上していますが、単に最も高度で最良のツールを選択するだけでは、スムーズで手間のかからない開発ライフサイクルは保証されません。

なぜgitlab


指数関数的に増加する選択肢と複雑さを備えたエコシステムにおいて、GitLabは、開発サイクルを加速し、コストを削減し、開発者の生産性を向上できる本格的なオープンソースDevOpsプラットフォームを提供します。計画とコードから展開と監視（およびその逆）まで、GitLabは多くの多様なツールを1つのオープンソーススイートに組み合わせます。

Mattermost ChatOpsが選ばれる理由


Mattermostでは、GitLabの大ファンです。そのため、MattermostにはGitLab Omnibusが付属しており、MattermostがGitLabで簡単に実行できるように取り組んでいます。

オープンプラットフォームのMattermost ChatOpsを使用すると、チームに関連情報を提供し、会話の場所を直接決定できます。問題が発生した場合、ChatOpsワークフローは、Mattermostで直接問題を解決するために協力している関連チームメンバーに警告することができます。

ChatOpsは、メッセージングを介してCI / CDタスクと対話する方法を提供します。今日、組織内では、問題に対する多くのディスカッション、コラボレーション、およびソリューションがメッセンジャーに提出されており、CI / CDタスクを実行して出力がチャネルに戻ってくると、チームの作業プロセスを大幅にスピードアップできます。

Mattermost + GitLab


DevOpsツール、オープンソースメッセンジャー、ChatOpsの完全なセット-どうすれば恋に落ちることができますか？開発者はGitLabとMattermostを使用して、DevOpsプロセスを簡素化するだけでなく、チームメンバーが問題について話し合い、協力し、意思決定を行う同じチャットインターフェイスに転送することもできます。

ChatOpsで生産性を向上させるために開発チームがMattermostとGitLabを共有する方法の例をいくつか示します。

Itkは、GitLabとMattermostを使用して、コードのタイムリーな配信と、年間のuvlichivaetプロダクションの数を6倍にしています
。Itkフランスのモンペリエに拠点を置き、農家が収穫プロセスを最適化し、品質を改善し、リスクをより効果的に管理するのに役立つツールとアプリケーションを開発しています。

彼らは2014年頃にGitLabの使用を開始し、ほとんどの場合、日常の仕事、メッセージング、ビデオ通話に古いチャットツールを使用していました。ただし、会社の成長に伴い、このツールは拡張できませんでした。永続的に保存され、簡単に見つけられるメッセージはなく、チームワークはますます困難になりました。そこで彼らは代替案の探求を始めました。

その後すぐに、GitLab OmnibusパッケージがオープンメッセージングプラットフォームMattermostに付属していることを発見しました。彼はすぐに、構文の自動強調表示やMarkdownの完全サポートなどのシンプルなコード共有機能に加え、GitLabと統合された新しいソリューションの開発に関するアイデアの共有、メッセージ検索、チームコラボレーションの容易さを気に入っていました。

Mattermostに移行する前は、チームメンバーは開発プロセスの通知を簡単に受け取ることができませんでした。しかし、彼らはプロジェクトを視覚的に追跡し、リクエストをマージし、GitLabで他のアクションを実行できることを望んでいました。

itk開発者のRomain ManeskiがMattermost用のGitLabプラグインの作成を開始したのは、MattermostでGitLab通知をサブスクライブし、新しい問題の通知とレビューのリクエストを1か所で受信できるようになったときです。

現在、彼の会社全体では、GitLabとMattermostの両方を使用して、ChatOpsでワークフローを高速化しています。その結果、彼らは更新をより速く提供することができ、チームが取り組んでいるプロジェクトとマイクロサービスの数が3倍に増加し、本番環境でのデプロイメントの数が年間で6倍に増加しました。これはすべて、開発チームと農学者の5倍の成長に伴います。



ソフトウェア会社は、コードと構成をより透過的かつ透過的にすることにより、生産性を向上させます。

メリーランド州に拠点を置くソフトウェア会社とデータサービス会社も、生産性を向上させ、シームレスにコラボレーションするためにGitLabと統合されたMattermostを実装しています。彼らは分析を行い、データを管理し、世界中の生物医学組織向けのソフトウェアを開発しています。

GitLabはチームで積極的に使用されており、DevOpsワークフローの大きな利点として使用を検討しています。

また、GitLabとMattermostを組み合わせて、Webhookを介してMattermostの1つのチャネルでGitLabとのコミットを収集しました。これにより、管理者は特定の日の「空中ビュー」で会社で何が起こっているかを知ることができました。また、構成とバージョン管理を管理するための更新が追加されました。これにより、日中に内部インフラストラクチャとシステムに加えられたさまざまな変更のスナップショットを取得できます。

また、チームは、アプリケーションイベントに関する通知を送信するために、個別の「ハートビート」チャネルを設定しました。このようなメッセージを特定のハートビートチャネルに送信することで、チームメンバーは通常のチャネルでの会話から注意をそらすことができなくなり、チームメンバーはハートビートチャネルで公開された質問に個別に切り替えることができます。

この統合の主な利点の1つは、バージョン間の変更の透過性とリアルタイムの構成管理です。変更がコミットされてプッシュされるとすぐに、通知がリアルタイムでハートビートチャネルに送信されます。誰でもそのようなチャンネルに登録できます。アプリケーション間の切り替え、チームメンバーへの追加の質問、またはコミットの追跡は不要になりました。現在、これらすべてがMattermostに集中しており、構成管理とアプリケーション開発はGitLabで行われています。

GitLabとMattermost ChatOpsは可視性と生産性を高め、開発をスピードアップします

MattermostはGitLab Omnibusパッケージの一部として提供されますビルド済みのGitLab SSOサポート、事前にパッケージ化されたGitLab統合、PostgreSQLサポート、およびシステムの監視とインシデントレスポンス管理を可能にするPrometheus統合を提供します。最後に、MattermostはGitLab Cloud Nativeを使用してデプロイできるようになりました。
