ユーザーエージェントとは


ブラウザのユーザーエージェントは、接続時にブラウザからウェブサイトに送信されるデータ文字列で、使用するプラットフォーム（オペレーティングシステム）、アーキテクチャ、使用するブラウザのバージョンに関するデータが含まれています。

正しく使用すれば、このアイデアは非常にポジティブです。ユーザーエージェントは、ブラウザの機能に応じて、Webサイトがユーザーに最大の機会を提供できるようにします。たとえば、ソフトウェアを使用するWebサイトの場合、ダウンロード用の適切なバージョンのインストールファイルが提供される場合や、ブラウザを設定するためのヘルプページの場合、ブラウザに適した手順が表示されます。

何が問題ですか？


熟練したWeb開発者がユーザーエージェントの定義と混同するか、さらに悪いことに、一部の大企業がこのテクノロジーを競争力のある目的で使用することを望まないまで、上記のすべてが素晴らしいように聞こえます。

ブラウザーコードの操作に関連しない多くのエラーメッセージが表示されます。ときどきVivaldiがブロックされ、そのバージョンが正しく検出されず、すべての種類の警告がユーザーに表示されます-これはすべてユーザーエージェントに接続されています。

たとえば、Webサイトの開発者がWebサイトの作成者自身が個人的にテストしたブラウザーのみがそのページを操作する権利を持ち、他のすべてのユーザーがこの権利を不当に奪われたと突然決定した場合など、より愚かな状況が発生することがあります。そのような決定は、誰にとってもインターネットの開放性というまさにその考えに矛盾し、ブラウザー市場における主要なプレーヤーの地位をさらに強化することにつながり、それによって私たちをインターネットの石器時代に戻します。そして、正直なところ、これはWeb開発者によくある間違いです。そのため、ほとんどすべてのブラウザーは、ユーザーエージェントだけでなく、多かれ少なかれ人気のあるすべてのブラウザーに関する情報もユーザーエージェントに含めています。 Linux x64用のVivaldi Browser User Agentを見てください-次のようなものです：

Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/78.0.3904.99 Safari/537.36 Vivaldi/2.9.1705.41

この行のほとんどはChromiumプロジェクトから継承され、ChromiumプロジェクトはSafariからコードを継承します。このような欺瞞的なトリックにもかかわらず、それはほとんどの場合に機能します。まるで自分を見知らぬ人として紹介するかのように、プライベートクラブに入ることができます。自分の名前を正しく付ければ、すべてのドアが開かれます。

お気に入りのクラブに入るのに役立つ名前をリストすることに加えて、逆の間違いを回避する必要もあります。問題が発生するような名前を使用してください。 Operaの最新バージョンまたはChromiumベースのEdgeブラウザのユーザーエージェントをご覧ください。彼らは過去の化身と混同したくないので、ユーザーエージェントに自分の名前を正しく書く危険はなく、それぞれエイリアス「OPR」と「EDG」と呼ばれることを好みます。ユーザーがこのエラーに関するメッセージを頻繁に送信することは明らかですが、実際にはこれはバグではなく機能です。

さて、上記のケースが一気に「間違い」に起因することができると仮定してください。しかし、もっと厄介な状況があります。ユーザーエージェントにVivaldiという単語が表示されたときに問題を通知せずに、ブラウザーをブロックするWebサイトがよくあります。時々、そのような技術的なトリックは、私たちが競合するブラウザで、ヘビー級によって使用されます。このようなロックが発生し、ユーザーがサイトへのアクセスに関する標準エラーメッセージを表示した場合、またはWebサイトの不完全なコピーがブラウザーに送信された場合、ユーザーは通常、Vivaldiブラウザーが原因であると見なします。誰かが意図的にブラウザに干渉する可能性があると考えるのは奇妙です！一方、ブラウザの名前を含むテキストを「Vivaldo」や「Vxvaldi」などのユーザーエージェント内の誤ったテキストに変更すると、すべてが魔法のように機能し始めます。この場合、コードにパッチを追加する必要があります（パッチを「修正」と呼ばないでください）。この問題を解決します。

この動作の例をいくつか示します。

  • Google.comでは、ユーザーエージェントでVivaldiを使用し、リダイレクト後にアクセスすると、ページの検索フィールドが下にシフトします
  • Googleドキュメントでは、Vivaldiとしてログインすると警告が表示されます
  • ユーザーエージェントのVivaldiという名前のWhatsApp Facebook Webインターフェイスでは、サイトへのアクセスは許可されず、別のブラウザーを使用するように提案されます
  • Vivaldiという名前のMicrosoft Teams（チャットとコラボレーション）では、彼らの素晴らしいWebサイトを使用することはできません。

また、ユーザーエージェントでVivaldiという名前に置き換えて、同様のWebサイトでの作業を試みることにより、使用可能な任意のブラウザーを試すことができます。

そのような場合はすべて、当然これらの企業に連絡して、ユーザーへのさらなる尊敬を求めます。たとえば、これは4年前に発見された問題に関するWhatsAppプロジェクトへのツイートです。さらに、私たちだけでなくユーザーも過去数年にわたって繰り返し信号を送ってきました。無駄に。

頭痛を刻む


一般的に、私たちはこれにうんざりしていて、Vivaldiの次のリリースでは、根本的な一歩を踏み出すことにしました。インターネットは無限になりがちであることを考えると、Vivaldiをブロックする問題について、すべてのサイトを物理的に確認することはできません。したがって、私たちが「非ヴィヴァルディ」であるWebサイトのリストを作成することは、まったく選択肢ではありません。代わりに、リバースすることにしました。私たちが尊敬している（そして私たちを尊敬している）一部のWebサイトでは、完全な正名のVivaldiで自己紹介し、他のすべての人にとっては、私たちが「Chrome」であると偽装します。ちなみに、私たちの友人には、duckduckgo.com、ecosia.org、qwant.com、startpage.comなどのリソースも含まれています。

この決定には重大な欠点があります。特に、インターネットメーターと個々のWebサイトの両方で収集されたブラウザー使用統計から消えるリスクがあります。しかし、私たちにとって、ユーザーの利便性は、見かけの人気よりも重要です。
