Beautiful and detailed geological map of Mars made in Python, GDAL



This week is a creative rethinking of the geological (or areoreological?) Map of Mars based on a map made by USCS . The same geological data was used as for the original, but more toponyms and signatures were added, the visual style was redesigned, and the terms from the map legend were simplified for general understanding.

One of the most difficult parts of the project was to translate the symbols into the language of non-specialists. The original map was made for geologists, so most of the vocabulary had to be reviewed. For example, a new interpretation of the value of the edge of the caldera is ‚ÄúThe boundary of the empty magma chamber remaining after the volcanic eruption‚Äù. In the original, there was the ‚ÄúOvoid tubercle, outlining one or several partially or completely closed hollows merged; the collapse of a volcano associated with eruptions and possibly explosive eruptions. ‚Äù

In many cases, the translated designations were more inaccurate or less informative than the original ones, therefore they also included original abbreviations for each type of geological element. These tags may contain links to the source data, if you suddenly want to learn more about each type of geological formation in scientific terms.



Some of the notable geological features on Mars. 1: Mount Olympus, the largest volcano in the solar system. 2: Mariner Valley, a system of deep canyons over 4000 km long. 3: Hellas Plain, the largest visible impact crater in the solar system. 4: Mars is geologically divided into the Northern Lowlands (pale green) and the Southern Highlands (brown). Impact craters formed by falling asteroids and comets (neon yellow) are scattered throughout the planet.



The following separate layers were used for the map. 1-3: Geological elements, geological contacts, and geological features from the USGS dataset. 4-5: tilt level and from two USGS sources. 6: Nomenclature from the IAU (International Astronomical Union). 7-8: grid lines and 3D effect developed in Photoshop.

It took a lot of time to choose a projection for the project. To accurately show which part of the planet consisted of each geological formation, the projection of Eckert IV was ultimately used.. This type of map projection distorts the contours of the object, but preserves the relative area of ‚Äã‚Äãthe figures over the entire surface of the planet. Such a projection is not suitable for visualization of the polar regions, therefore, we also added four map-insets into the corners to show each hemisphere of Mars (northern, southern, eastern and western).

To compare these different projections of the map, you can use the Tissot indicatrix - a set of circles of the same size, drawn in different places on the globe. All map projections distort the space, but you can see that the effects vary greatly depending on the projection.



1: Equidistant. 2: Eckert IV. 3-5: Orthographic projections centered at different longitudes and latitudes

(DEM)


The terrain model is data containing information about the height of various points on the planet. The project used data obtained from the US Geological Survey. Each pixel in the GeoTIFF file is a 16-bit number that describes the height of the point.

Note: many programs cannot read this file normally, so that it looks strange in the preview format or in other non-specialized programs is normal. Within the framework of the project, work with them went through GDAL (English Geospatial Data Abstraction Library - a library of abstraction of geospatial data).

Creating Map Projections


As mentioned above, the Eckert IV projection was used for the main map, which keeps the area unchanged, and the orthographic projection was used for embedded pole maps.

Change the projection in the DEM file: the source file used equal projection, so you need to enter the following code in the command line to make the translation. The code uses the original intif file and creates a new outtif, in eck4 (Eckert IV) projection format.

gdalwarp -t_srs "+proj=eck4" ./path_to_intif.tif ./path_to_outtif.tif

Then we decrease the DEM resolution by simply reducing the size of each pixel to 1500x1500 meters using the average method. This will reduce processing time, and a decrease in resolution at this point is easier to do than later.

gdalwarp -tr 1500 1500 -r average ./path_to_intif.tif ./path_to_outtif.tif

Washed and slope map


Next, we use the resulting file to create a map with washing and slope maps for each hemisphere of the planet.

After washing, the map shows shadows from an imaginary light source; light falls on the map from above. It is imaginary, because in the real world this does not happen - a single light source will create shadows at different angles in different parts of the ball. GDAL's built-in hillshade function sets the angle of incidence of the light to be the same for the entire map. For this project, z, the vertical magnification, was set to 20. This increases each height value by 20 times to make the relief more contrast and to provide shadows.

gdaldem hillshade -z 20 ./path_to_intif.tif ./path_to_hillshade.tif

Slope maps highlight the steepest parts of the map and add more information to the topographic wash (which emphasizes absolute height rather than steepness).

gdaldem slope ./path_to_intif.tif ./path_to_slope.tif



Orthographic projections


In addition to the map, four maps of the hemispheres of Mars are also made in the projection of Eckert IV. The Eckert IV projection is not well suited to display the North and South Poles, so these two insets are very useful for understanding them. To get a wash and a slope map, repeat the code with minor changes - ortho instead of eck4 and designate the center of longitude and latitude for each map (North: + lat_0 = 90 + lon_0 = 90, South: + lat_0 = -90 + lon_0 = 90, East: + lat_0 = 0 + lon_0 = 90, West: + lat_0 = 0 + lon_0 = 270).

gdalwarp -t_srs "+proj=ortho +lat_0=90 +lon_0=0" ./path_to_intif.tif ./path_to_outtif.tif
gdalwarp -tr 1500 1500 -r average ./path_to_intif.tif ./path_to_outtif.tif
gdaldem hillshade -z 20 ./path_to_intif.tif ./path_to_hillshade.tif
gdaldem slope ./path_to_intif.tif ./path_to_slope.tif



Legend card


The International Astronomical Union is responsible for naming extraterrestrial objects. You can simply download the file in CSV format, containing all the objects for each planet directly from their site. To do this, use the Advanced Search function to download All Feature Types for your planet - Target (Mars), but only with Approval Status of Adopted by IAU - approved by the MAC. In the Columns to Include section, select Feature ID, Feature Name, Clean Feature Name, Target, Diameter, Center Lat / Lon, Feature Type, and Feature Type Code. You can also enable Origin (origin) if you want to add additional information about objects to the project, such as, for example, in honor of whom it was named.

Geological structures and elements


The geological map shows various types of rocks and other features, such as fault lines and river beds. USGS has a beautiful geological map of Mars, which we use as a source. To work with this data, download the database archive (790 MB) from the USGS Geologic Map. The next section on map design in Python explains how to access and display each data type in this database.

Python Map Design


We create six drawings (plots) with geological elements, structures, features, two types of text labels and a grid. It is often worthwhile to separate the data for processing in order to easily apply effects from Photoshop or Illustrator if necessary. Matplotlib used gridspec to customize all elements so that each subplot occupies a specific place within the decorative frame.



Geological elements- These are different types of relief that make up the surface of the planet. Different rock elements are usually represented in different colors. The USGS dataset marks each breed with a 2-3 letter code for the type of item. Assign a color to each letter code by compiling a table of code-color pairs in Mars_geologic_units.csv. We refer to this file when constructing each element in cartopy and matplotlib. Saving graphic parameters in a separate file facilitates the use of various color schemes and separates the design from the code.



Geological contacts- these are the boundaries between geological formations. Some geological boundaries are approximate or hidden under dust. As for geological formations, we use the Mars_geologic_boundaries.csv configuration file to display each type of geological contact in a different color. On the final map, we configure the display of some geological contacts in the form of dashed lines, simply opening the PDF in Illustrator and selecting all objects of the same color (Select -> Same -> Stroke color).



Geological features- These are other visible lines on the surface of the planet, such as canals, edges of craters or ridges of mountains. There are 13 different types in the dataset. To display each of them in a special style, each was displayed in a different color, and then the lines were styled in Illustrator. This allowed the use of special effects, such as gradient tails, to show the direction of movement of the outflow channels and lava flows.



Legend


Two datasets for the tags were used for the map. The first is official from the IAU. Label sizes are created according to the size of the object, although you still need to finalize the dimensions in Illustarator.



The second set of labels adds an abbreviation for each major geological element. This makes it easier to use the map legend, as some colors change due to shadows. It was necessary to approach the code creatively, namely, to draw each abbreviation in the geometric center of the geological form, which is not ideal for all cases. But for this project, this was normal, because then, according to the plan, all the same, these signatures will move manually to avoid overlapping future signatures.



Landing places: To display the landing sites of Mars rovers and other spacecraft, an enhanced digital image from NASA was used as a background. To show all 4 hemispheres of the planet, I had to change the projection - landing_sites.ipynb.

Saving Results from Matplotlib


Typically, the results are saved in pdf format to make final edits to text and forms in Illustrator. Here are a couple of standard commands that facilitate subsequent editing:

import matplotlib
import matplotlib.pyplot as plt
import matplotlib.backends.backend_pdf as pdf

#     ,     :
matplotlib.rcParams['pdf.fonttype'] = 42

#     
matplotlib.rcParams['image.composite_image'] = False

#      subplot
ax.axis('off')

#  padding  margins 
plt.margins(0,0)
plt.subplots_adjust(top=1, bottom=0, right=1, left=0, hspace=0, wspace=0)
plt.gca().xaxis.set_major_locator(plt.NullLocator())
plt.gca().yaxis.set_major_locator(plt.NullLocator())

#  pdf
pp = pdf.PdfPages('./savename.pdf', keep_empty=False)
pp.savefig(fig)
pp.close()

#      
#     PNG     Photoshop:
plt.savefig('./savename.png', format='png', dpi=600, pad_inches=0, transparent=True)


After saving the PDF, you need to edit it so that each object can be edited independently. In Illustrator, select everything in the file and go to Object -> Clipping Mask -> Release. At this point, it is also worth deleting the background and borders, if you did them before.
