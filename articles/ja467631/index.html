JavaScriptコンテキストビュー

また会いましたね！9月末にOTUSでコース「Fullstack JavaScript Developer」の新しいストリームが始まるとすでに書いています。クラスの開始を見越して、特にコースの学生のために用意された著作権記事を引き続き共有します。今日は、JavaScriptのコンテキストタイプについて見ていきます。行け。

記事の作者：パベルヤクポフ



この記事の目的は、JavaScriptで使用されるスコープの基本概念を読者が理解することです。

スコープは、JavaScriptで最も重要なものの1つです（そして、ほとんどの最新のプログラミング言語で使用されています）。スコープは、変数または関数の長寿命、アクセス、変数の可視性などに関連しています。

なぜこのスコープが必要なのですか？


プログラミング言語のスコープは、次の機能を実行します：
セキュリティ（カプセル化）-変数と関数は、必要な場合にのみ使用できます。
変数名の競合を解消します。スコープの存在により、すべての変数を1つのヒープに「パイル」せずに、名前空間（名前空間）をシミュレートできます。
コードの再利用-作成されたコードを使用して、「無関係な」影響を回避できます。

スコープのタイプ


最も単純なレベルでは、JavaScriptにはローカルとグローバルの2つの可視性の領域があります。

この記事では、語彙やブロックなどのスコープについても触れます。

グローバルスコープ


JavaScriptでドキュメントを開いてコードの作成を開始すると、グローバルスコープに入ります。

グローバルスコープで作成されたすべてのもの（変数、関数）は、プログラムのどこからでもアクセスできます。また、グローバル変数は、アプリケーションの実行中にいつでも使用でき、プログラムが作業を終了したときにのみ削除されます。

多くの初心者は、最初はグローバル変数を頻繁に使用します。これはプログラミングがやや簡単ですが、これは悪い習慣と見なされ、最終的に実際に必要なメモリよりもはるかに多くのメモリを使用する不安定なプログラムにつながることがよくあります。結局のところ、変数が使用される関数内で確実にカプセル化された場合、関数の最後に、ガベージコレクターを使用して変数がメモリから削除され、クライアントメモリの占有が停止します。これは無限ではありません。

ローカルスコープ


ローカルで宣言された変数は、それらが宣言されたスコープでのみ使用できます。

新しいスコープを作成する最も簡単な方法は、新しい関数を作成することです。関数内で作成された変数は、内部からのみアクセスできます。さらに、ローカルスコープはブロックスコープを使用して取得できます（以下で説明します）。

function foo(){
  let x = 15; }
//console.log(x);  error
{
  let y = 14; }
// console.log(y);   error
{
  var z = 13; }
console.log(z); // var   ,
//        
//   -  13

字句スコープ


では、字句スコープとは何ですか？簡単に言うと、これは外部関数にアクセスするための内部関数の機能です。ここでは、クロージャーの実践に移る価値があります。あなたはおそらくそれらについてさらにいくつかの記事を書くことができますが、私はすぐにあなたに1つの古典的な例をあげます：

sum(5)(5) //    ?
function sum(a){
 var add = function(b){
    return a+b;
  }
 return add;
}
console.log(sum(5)(5)); // 10  
 //

ブロックスコープ


これまでは、関数と中括弧{}の操作に関連するスコープについてのみ説明し、varの操作の違いについては間接的にのみ説明しました。

varディレクティブはどのように機能しますか？変数がグローバルスコープで変数を使用して宣言されると、変数名はプロパティとしてグローバルウィンドウオブジェクト（ブラウザーを意味する場合）に割り当てられ、プログラムの実行中は常にそこに残ります。同時に、{}などのブロックスコープとして（そしてif、for、while、およびその他すべてが非常に論理的に含まれている場合）。

さらに、1つのスコープで宣言されたletとconstの別の機能があります。これらは同じスコープで再び宣言することはできません（インタプリタの不満はここではかなり論理的に見えます）。

let x = 15;
console.log(x); //  
{
let  x = 16; //      
console.log(x) //         
let x = 17; //        ,        
}

新しい機能


新しい関数を「新しい関数」として宣言するときのスコープの機能。関数を作成するこのオプションはほとんど使用されませんが、必要になる場合があります。
構文例：

//let newFunc = new Function([arg1, arg2…argN], functionBody);
let mult = new Function(‘a’, ‘b’, ‘return a * b’);
console.log(mult(3,4));

通常、関数はそれが生まれた場所を記憶しますが（レキシカル環境）、関数が新しいFunction構文を使用して作成される場合、通常の状況のように、環境変数に書き込まれるのは関数を囲む変数ではなく、グローバルにのみ宣言されます。 

//   ,      ()()    
let a = 3;
function outerFunc() {
  var a = 2;
  var func = new Function('console.log(a*a)');
  return func;
}
outerFunc()(); // 9,    window

巻き上げ（変数を上げる）


スコープについて話し合うと、変数のスコープを上げるというトピックに対処することはできませんでした。コードを読み取るインタープリターは、実際に2回読み取ります。つまり、関数宣言として宣言された関数を読み取り、var変数を使用してグローバルに宣言されたグローバル変数を読み取ります。ただし、変数は宣言された値ではなく、未定義の値によって記録されます。

console.log(x); // undefined
var x = 15;
console.log(y);// error
let y = 13;
