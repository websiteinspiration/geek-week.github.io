Tic Tac Toe (PixiJS)



Un tipógrafo vulnerable, estúpido y soñador decidió convertirse en programador y no salió nada de eso ... Pero no abandonó la programación y decidió comenzar con pequeños programas ...

Esta es la mejor descripción que se me ocurrió. Fue para este propósito que comencé a escribir programas simples para perfeccionar mis habilidades, conocer nuevos diseños en mi idioma habitual y, para ser honesto, incluso comenzó a darme placer.

Si tiene poca experiencia en desarrollo, entonces el artículo será útil, y si ya tiene experiencia en desarrollo, dedique tiempo a algo más valioso.

Esto no es entrenamiento. Más como un blog.

El objetivo era hacer 3 versiones del juego tic tac toe.

1 - El más simple ( sin una hermosa imagen, usando el DOM )
2 - Da la oportunidad de jugar juntos (una computadora )
3 - Transfiera todo esto al lienzo

No describiré el tic-tac-toe, espero que todos conozcan el principio del juego. Todos los enlaces útiles (repositorio, documentación) estarán al final del artículo.

¿Qué salió de esto? Hm ...

El bot camina al azar.

function botMove() {
	var items = document.querySelectorAll(".app_block:not(.active)");

	var step = getRandomInt(items.length);

	items[ step ].innerHTML = "0";
	items[ step ].classList.add("active");
	items[ step ].classList.add("active_o");

	var result = checkMap();
	if( result.val) {
		setTimeout(function() {
			exit(result);
		}, 1);
	}

	movePlayer = !movePlayer;
}

function getRandomInt(max) {
	return Math.floor(Math.random() * Math.floor(max));
}

Segunda versión


Dos jugadores en la misma computadora.

Tomé parte de la lógica del controlador de clics en una función separada y pasé el contexto de la llamada a la función, porque necesitamos determinar en qué botón se hizo clic.

var items = document.getElementsByClassName("app_block");
var movePlayer = true;
var game = true;

for (var i = 0; i < items.length; i++) {
	items[i].addEventListener("click", function() {
		var collecion = document.querySelectorAll(".app_block:not(.active)");
		if(collecion.length == 1) {
			exit({win: "other"});
		}

		if( !this.classList.contains("active") ){
			if( movePlayer) {
				firstPlayer(this);
			} else {
				secondPlayer(this);
			}
		}
	});
}

Me dividí en dos funciones, pero tienen duplicación de código. Idealmente, divida por 3. Uno principal y dos trabajando con contexto.

function firstPlayer(that) {
	if(that.innerHTML == "") {
		that.classList.add("active");
		that.classList.add("active_x");
		that.innerHTML = "x"
	}

	var result = checkMap();
	if( result.val) {
		game = false;
		setTimeout(function() {
			exit(result);
		}, 10);
	}

	movePlayer = !movePlayer;
}

function secondPlayer(that) {
	if(that.innerHTML == "") {
		that.classList.add("active");
		that.classList.add("active_o");
		that.innerHTML = "0"
	}

	var result = checkMap();
	if( result.val) {
		game = false;
		setTimeout(function() {
			exit(result);
		}, 10);
	}

	movePlayer = !movePlayer;
}

Tercera versión


Quizás este es el punto más interesante porque Ahora el juego realmente parece un juego, no la interacción de elementos DOM.

Elegí trabajar PixiJS. No puedo decir nada sobre el + y - de esta biblioteca, pero miré un ejemplo en el que había 60,000 elementos y todos estaban animados. La animación es simple, pero el FPS se mantuvo en 50-60. Me gustó y comencé a leer la documentación. Debo decir de inmediato que mi conocimiento del idioma inglés es mínimo, fue difícil, pero hay muy pocos artículos en ruso (o me he visto mal). Tuve que empujar y con la ayuda de un traductor de google para atravesar las espinas.

Solo vi un informe sobre este tema de Yulia Pucnina "Animación gorda con Pixi js" .

El informe es de 2014 y debe comprender que la API podría cambiar. Un ojo en la documentación y el segundo en el video. Entonces estudié. Me llevó 4 horas escribir un prototipo tan simple. Más cerca del código.

Hacemos la inicialización predeterminada pixi

const app = new PIXI.Application({
	width: 720,
	height: 390,
	resolution: window.devicePixelRation || 1,
});
document.body.appendChild(app.view);

y también cree un contenedor (el contenedor principal con celdas) y póngalo en nuestro lienzo

let wrapper = new PIXI.Container();
app.stage.addChild(wrapper);

En el bucle, creamos nuestras celdas, les establecemos los tamaños, las coordenadas necesarias y también agregamos el valor predeterminado a la celda como una cadena vacía desde Esto será útil en el futuro y colgará los controladores en las celdas, después de habilitar el indicador de interactividad del contenedor.

for (let i = 0; i < 9; i++) {
    let container = new PIXI.Container();
	let block = new PIXI.TilingSprite( PIXI.Texture.from("images/bg.png") , 240, 130);
    
	container.x = (i % 3) * 240;
    container.y = Math.floor(i / 3) * 130;
    container.addChild(block);
    
    let text = new PIXI.Text("");
    text.anchor.set(0.5);
    text.x = container.width / 2;
    text.y = container.height / 2;
    container.addChild(text);
    
    container.interactive = true;    
    container.on("mousedown", function () {
        addValueInBlock(this);
    });
    
    wrapper.addChild(container);
}
