Goプロジェクトを構築するためのMavenの方法

まず、少し背景。 2010年代の初めに、BK-0010エミュレータのBINファイルをWAVファイルに変換する小さなコンバータユーティリティを作成しました。このユーティリティは、移植性を最大化することを目的としてPythonで記述されており、問題なく動作し、しばらくの間忘れていました。しかし、2016年に、Pythonとそのインストール方法を知らなかったユーザーが現れました。彼は「うまくいく」単純な実行可能なモノリスを望んでいました。彼の要求は私には理にかなっているようで、私はユーティリティをメインプラットフォーム用のバイナリ実行可能ファイルのセットとして作り直すことにしました。



PythonとJavaはそのような機会を与えませんでした（もちろん、ユーティリティを何十メガバイトも増やしたいという欲求があった場合を除きます）。可能性としては、ソリューションはC / C ++で作成できますが、プラットフォームのそのようなターゲットカバレッジでは、クロスコンパイルの問題は、タスクに割り当てられた時間を超えてしまいます（64、32ビットのWindows、Linux、MacOSのクロスアセンブリをサポートする必要がありました）オプション）。それで、人気が高まっているGoという言語に注目しました。その頃にはすでにかなり成熟していて、「タンバリンで踊る」ことなく、必要なすべてのクロスコンパイルをボックスからすぐに提供できる唯一の人です（！）。

プロジェクトのルートからすべてのサンプルのビルドを開始することもできます。


mvn clean install -Pexamples

mvn clean install -Pexamples -T6
