パンダプロファイリングライブラリを使用したデータ探索の高速化

まず、新しいデータセットを使い始めるには、それを理解する必要があります。これを行うには、たとえば、変数によって受け入れられる値の範囲とそのタイプ、および欠損値の数を確認する必要があります。

パンダライブラリは、探索的データ分析（EDA）を実行するための多くの便利なツールを提供します。しかし、それらを使用する前に、通常、df.describe（）などのより一般的な関数から始める必要があります。確かに、そのような機能によって提供される可能性は限られていることに注意してください。EDA中にデータセットを処理する初期段階は、非常によく似ています。



私たちが今日公開している資料の著者は、彼は繰り返しの行動のファンではないと言っています。その結果、探索的データ分析を迅速かつ効率的に実行するツールを探して、パンダプロファイリングライブラリを見つけました。彼女の作業の結果は、特定の個別のインジケーターの形式ではなく、より詳細なHTMLレポートの形式で表現されます。これらのレポートは、より詳細な作業に進む前に知っておく必要がある分析データに関するほとんどの情報を含んでいます。

ここでは、タイタニックデータセットの例を使用して、パンダプロファイリングライブラリを使用する機能について検討します。

パンダ探査データ分析


タイタニックのデータセットにはさまざまなタイプのデータがあり、欠落している値があるため、タイタニックのデータセットでパンダのプロファイリングを試すことにしました。パンダのプロファイリングライブラリは、データがまだクリアされておらず、機能に応じてさらに処理が必要な場合に特に興味深いと思います。このような処理を正常に実行するには、どこから始めて何を探すべきかを知る必要があります。ここで、パンダのプロファイリング機能が役立ちます。

まず、データをインポートし、パンダを使用して説明的な統計指標を取得します。

#   
import pandas as pd
import pandas_profiling
import numpy as np

#  
df = pd.read_csv('/Users/lukas/Downloads/titanic/train.csv')

#    
df.describe()

標準パンダツールを使用して取得された記述統計のインジケーター

ここには多くの有用な情報がありますが、調査中のデータについて学ぶのに興味深いすべてのものはありません。たとえば、データフレームの構造DataFrameに891行あると想定できます。これを確認する必要がある場合は、フレームのサイズを決定する別のコード行が必要になります。これらの計算は特にリソースを集中的に使用するわけではありませんが、それらの一定の繰り返しは必然的に時間の損失につながり、おそらくデータのクリーニングに費やされます。

パンダプロファイリングを使用した探索的データ分析


次に、pandas-profilingを使用して同じことを行います。

pandas_profiling.ProfileReport(df)

上記のコード行を実行すると、探索的データ分析の指標を含むレポートを生成できます。上記のコードは、データについて見つかった情報の結論につながりますが、結果をHTMLファイルにして、たとえば誰かに見せることができるようにすることもできます。

レポートの最初の部分には、データに関する基本的な情報（観測数、変数数など）を提供する概要セクションが含まれます。さらに、特に注意する必要があることについてアナリストに通知する警告のリストが含まれます。これらの警告は、データを消去するときにどこに集中できるかについての手掛かりになる可能性があります。

探索的変数分析


レポートの概要セクションの後ろに、各変数に関する有用な情報があります。特に、各変数の分布を説明する小さなグラフが含まれます。


数値変数Ageに関する情報

前の例からわかるように、pandas-profilingは、欠損値のパーセンテージや数などのいくつかの有用な指標と、すでに見た記述統計の指標を提供します。Ageこれは数値変数なので、ヒストグラムの形でその分布を視覚化すると、直角分布であると結論付けることができます。

カテゴリー変数を検討する場合、表示される指標は数値変数で見られる指標とは少し異なります。


カテゴリー変数Sex

つまり、パンダプロファイリングライブラリは、平均、最小、最大の代わりにクラスの数を見つけました。それ以来、Sexバイナリ変数であり、その値は、二つのクラスによって表されます。

私のように、コードを調査したい場合は、パンダプロファイリングライブラリがこれらのインジケーターを正確に計算する方法に興味があるかもしれません。ライブラリコードがオープンでGitHubで利用可能であることを考えると、これを知ることはそれほど難しくありません。私はプロジェクトで「ブラックボックス」を使用するのがあまり好きではないので、ライブラリのソースコードを調べました。たとえば、 describe_numeric_1d関数で表される数値変数を処理するメカニズムは次のとおりです。

    """Compute summary statistics of a numerical (`TYPE_NUM`) variable (a Series).
    Also create histograms (mini an full) of its distribution.
    Parameters
    ----------
    series : Series
        The variable to describe.
    Returns
    -------
    Series
        The description of the variable as a Series with index being stats keys.
    """

    _percentile_format = "{:.0%}"
    stats = dict()
    stats['type'] = base.TYPE_NUM
    stats['mean'] = series.mean()
    stats['std'] = series.std()
    stats['variance'] = series.var()
    stats['min'] = series.min()
    stats['max'] = series.max()
    stats['range'] = stats['max'] - stats['min']
    # To avoid to compute it several times
    _series_no_na = series.dropna()
    for percentile in np.array([0.05, 0.25, 0.5, 0.75, 0.95]):
        # The dropna() is a workaround for https://github.com/pydata/pandas/issues/13098
        stats[_percentile_format.format(percentile)] = _series_no_na.quantile(percentile)

    stats['kurtosis'] = series.kurt()
    stats['skewness'] = series.skew()
    stats['sum'] = series.sum()
    stats['mad'] = series.mad()
    stats['cv'] = stats['std'] / stats['mean'] if stats['mean'] else np.NaN
    stats['n_zeros'] = (len(series) - np.count_nonzero(series))
    stats['p_zeros'] = stats['n_zeros'] * 1.0 / len(series)
    # Histograms

    stats['mini_histogram'] = mini_histogram(series, **kwargs)
    return pd.Series(stats, name=series.name)

このコードはかなり大きく複雑に見えるかもしれませんが、実際には非常に簡単に理解できます。重要なのは、ライブラリのソースコードに変数の型を決定する関数があるということです。ライブラリが数値変数を満たしていることが判明した場合、上記の関数は、検討したインジケーターを見つけます。この関数は、標準のパンダ操作を使用してSeries、などのタイプオブジェクトを操作しますseries.mean()。計算結果は辞書に保存されstatsます。ヒストグラムは、関数の適応バージョンを使用して形成されますmatplotlib.pyplot.hist。適応は、関数がさまざまなタイプのデータセットを処理できるようにすることを目的としています。

相関指標と調査したデータのサンプル


変数の分析結果の後で、パンダプロファイリングの[相関]セクションに、ピアソンとスピアマンの相関行列が表示されます。


ピアソン相関行列

必要に応じて、レポート生成を開始するコード行で、相関の計算に使用されるしきい値を設定できます。これを行うことにより、分析でどの相関強度が重要と見なされるかを示すことができます。

最後に、サンプルセクションのパンダプロファイリングレポートには、例として、データセットの先頭から取得したデータが表示されます。最初のいくつかの観察は、データセット全体の特性を反映しないサンプルである可能性があるため、このアプローチは不愉快な驚きにつながる可能性があります。


調査したデータのサンプルを含むセクション

概要


上記を要約すると、パンダプロファイリングライブラリは、データの一般的な大まかなアイデアをすばやく取得したり、誰かにインテリジェンスデータ分析に関するレポートを送信したりする必要がある場合に便利ないくつかの便利な機能をアナリストに提供することに注意できます。同時に、パンダのプロファイリングを使用せずに、データの機能を考慮した実際の作業が手動で実行されます。
