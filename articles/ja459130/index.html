この記事は、カザンでの最近の囲碁会議からの報告の詳細版です。Goに興味があり、カザン、イノポリス、美しいヨシュカルオラ、または近くの別の都市に住んでいる場合は、コミュニティページgolangkazan.github.ioにアクセスしてください。

マイクロサービス構造合意


エラー処理のルールに触れる前に、設計とコーディングの際に遵守すべき制限を決定することは価値があります。これを行うには、マイクロサービスがどのように見えるかを説明する価値があります。


まず、クリーンなアーキテクチャを尊重します。コードを3つのレベルに分割し、依存関係の規則を観察します。より深いレベルのパッケージは外部パッケージから独立しており、循環的な依存関係はありません。幸い、Goでは直接的なラウンドロビンパッケージの依存関係は禁止されています。用語の借用による間接的な依存関係、振る舞いや型へのキャストに関する仮定は依然として存在する可能性があるため、回避する必要があります。

func mergeErrors(err error, nextErr error) error {
    if err == nil {
        err = nextErr
    } else if nextErr != nil {
        err = errors.Wrap(err, nextErr.Error())
    }
    return err
}

func (m *errorHandlingMiddleware) logCall(start time.Time, err error, method string, fields log.Fields) {
    fields["duration"] = fmt.Sprintf("%v", time.Since(start))
    fields["method"] = method
    logger := m.logger.WithFields(fields)
    if err != nil {
        logger.Error(err, "call failed")
    } else {
        logger.Info("call finished")
    }
}
