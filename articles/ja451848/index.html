DNS-01チャレンジとAWSを使用してSSL証明書管理を暗号化しましょう

この投稿では、DNS-01チャレンジとAWSを使用してLet's Encrypt CAからのSSL証明書の管理を自動化する手順について説明します。


acme-dns-route53は、この機能を実装できるツールです。彼は、Let's EncryptからのSSL証明書を操作し、Amazon Certificate Managerに保存し、Route53 APIを使用してDNS-01チャレンジを実装し、最終的に通知をSNSにプッシュする方法を知っています。アクメ-DNS-route53がAWSラムダ内で使用するための機能を内蔵しており、これは我々が必要なものですされます。


この記事は4つのセクションに分かれています。


  • zipファイルを作成します。
  • IAMロールを作成します。
  • acme-dns-route53を実行するラムダ関数を作成します。
  • 1日に2回関数をトリガーするCloudWatchタイマーを作成します。

注：始める前に、GoLang 1.9+とAWS CLIをインストールする必要があります


zipファイルを作成する


acme-dns-route53はGoLangで書かれており、1.9以上のバージョンをサポートしています。


acme-dns-route53内部にバイナリを含むzipファイルを作成する必要があります。これを行うにはacme-dns-route53、次のコマンドを使用してGitHubからリポジトリをインストールしますgo install。


$ env GOOS=linux GOARCH=amd64 go install github.com/begmaroman/acme-dns-route53

バイナリは$GOPATH/binディレクトリにインストールされます。インストール中に2つの環境変数を指定したことに注意してください：GOOS=linuxとGOARCH=amd64。Linux OSとamd64アーキテクチャに適したバイナリを作成する必要があることをGoコンパイラに明確に示します。これはAWSで実行されるものです。
AWSでは、プログラムがzipファイルでデプロイされていると想定しているためacme-dns-route53.zip、新しくインストールされたバイナリを含むアーカイブを作成してみましょう。


$ zip -j ~/acme-dns-route53.zip $GOPATH/bin/acme-dns-route53

注：バイナリはzipアーカイブのルートにある必要があります。これには-jフラグを使用します。

# Cloudwatch event rule that runs acme-dns-route53 lambda every 12 hours
resource "aws_cloudwatch_event_rule" "acme_dns_route53_sheduler" {
  name                = "acme-dns-route53-issuer-scheduler"
  schedule_expression = "cron(0 */12 * * ? *)"
}

# Specify the lambda function to run
resource "aws_cloudwatch_event_target" "acme_dns_route53_sheduler_target" {
  rule = "${aws_cloudwatch_event_rule.acme_dns_route53_sheduler.name}"
  arn  = "${aws_lambda_function.acme_dns_route53.arn}"
}

# Give CloudWatch permission to invoke the function
resource "aws_lambda_permission" "permission" {
  action        = "lambda:InvokeFunction"
  function_name = "${aws_lambda_function.acme_dns_route53.function_name}"
  principal     = "events.amazonaws.com"
  source_arn    = "${aws_cloudwatch_event_rule.acme_dns_route53_sheduler.arn}"
}
