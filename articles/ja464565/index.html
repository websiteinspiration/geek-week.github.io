IP-KVM over QEMU



KVMのないサーバーにオペレーティングシステムをロードするときのトラブルシューティングは簡単な作業ではありません。リカバリイメージと仮想マシンを使用してKVM-over-IPを作成します。リモートサーバーの

オペレーティングシステムに問題がある場合、管理者はリカバリイメージをダウンロードして必要な作業を実行します。この方法は、障害の原因がわかっており、サーバーにインストールされているリカバリイメージとオペレーティングシステムが同じファミリのものである場合に正常に機能します。失敗の原因がまだわからない場合は、オペレーティングシステムの読み込みの進行状況を監視する必要があります。

リモートKVM


IPMIやIntel®vPro™などの組み込みツールを使用するか、IP-KVMと呼ばれる外部デバイスを使用して、サーバーコンソールにアクセスできます。リストされているテクノロジーのすべてが利用できない状況があります。しかし、これで終わりではありません。 Linuxファミリのオペレーティングシステムに基づいて、サーバーをリモートで再起動してリカバリイメージを作成できる場合は、KVM-over-IPをすばやく構成できます。

リカバリイメージは、RAMにある本格的なオペレーティングシステムです。したがって、仮想マシン（VM）を含むあらゆるソフトウェアを実行できます。つまり、サーバーオペレーティングシステムが動作するVMを起動できます。 VMコンソールへのアクセスは、たとえばVNCを介して調整できます。

VM内でサーバーオペレーティングシステムを起動するには、サーバーディスクをVMディスクとして指定する必要があります。Linuxファミリのオペレーティングシステムでは、物理ディスクは/ dev / sdX形式のブロックデバイスとして表され、通常のファイルとして使用できます。

QEMUやVirtualBoxなどの一部のハイパーバイザーでは、VMデータを未加工の形式、つまりハイパーバイザーメタデータのないストレージデータのみに保存できます。したがって、サーバーの物理ディスクを使用してVMを起動できます。

この方法では、リカバリイメージとその内部のVMを実行するためのリソースが必要です。ただし、4ギガバイト以上のRAMがあれば、これは問題になりません。

環境の準備


仮想マシンとして、軽量でシンプルなQEMUプログラムを使用できます。これは、ほとんどの場合、リカバリイメージの一部ではないため、個別にインストールする必要があります。私たちがお客様に提供するリカバリイメージは、pacmanパッケージマネージャを使用するArch Linuxに基づいています。 最初に、リカバリイメージが最新のソフトウェアを使用していることを確認する必要があります。次のコマンドを使用して、すべてのOSコンポーネントを確認および更新でき ます。更新後、QEMUをインストールする必要があります。pacmanを介したインストールコマンドは次のようになります。qemuが 正しくインストールされていることを確認してください 。



pacman -Suy



pacman -S qemu



root@sel-rescue ~ # qemu-system-x86_64 --version
QEMU emulator version 4.0.0
Copyright (c) 2003-2019 Fabrice Bellard and the QEMU Project developers



仮想マシンの起動


最初に、VMに割り当てられているリソースの量を決定し、物理ディスクへのパスを見つける必要があります。この例では、2つのコアと2ギガバイトのRAMを仮想マシンに割り当て、ディスクはパス/ dev / sdaおよび/ dev / sdbにあります。VMの起動： 各パラメーターの意味についてもう少し：

qemu-system-x86_64 \
-m 2048M \
-net nic -net user \
-enable-kvm \
-cpu host,nx \
-M pc \
-smp 2 \
-vga std \
-drive file=/dev/sda,format=raw,index=0,media=disk \
-drive file=/dev/sdb,format=raw,index=1,media=disk \
-vnc :0,password \
-monitor stdio



  • -m 2048M -VMで2 GBのRAMを選択します。
  • -net nic -net user -NAT（Network Address Translation）を使用して、ハイパーバイザーを介した単純なネットワーク接続を追加します。
  • -enable-kvm-完全なKVM仮想化（カーネル仮想マシン）を有効にします。
  • -cpu host-サーバープロセッサのすべての機能を取得するよう仮想プロセッサに指示します。
  • -M pc -PC機器のタイプ。
  • -smp 2 — ;
  • -vga std — , ;
  • -drive file=/dev/sda,format=raw,index=0,media=disk
    • file=/dev/sdX — , ;
    • format=raw — , «» , ;
    • index=0 — , ;
    • media=disk — ;
  • -vnc :0, password — VNC- 0.0.0.0:5900, ;
  • -monitor stdio-管理者は標準の入力/出力ストリームを通じてqemuと通信します。

すべてが正常であれば、QEMUモニターが起動します。 認証はパスワードで行われることを示しましたが、パスワード自体は指定していません。これを行うには、QEMUモニターにchange vnc passwordコマンドを送信します。重要な注意：パスワードは8文字を超えることはできません。 その後、サーバーのIPアドレスと指定したパスワードを使用して、RemminaなどのVNCクライアントに接続できます。 これで、読み込み段階で起こり得るエラーを確認できるだけでなく、それらと戦うことができます。 作業の最後に、仮想マシンを完了する必要があります。これは、シャットダウン信号を送信するか、system_powerdownコマンドを発行することにより、OS内部で実行できます。

QEMU 4.0.0 monitor - type 'help' for more information
(qemu)



(qemu) change vnc password
Password: ******









QEMUモニターで。これは、シャットダウンボタンを1回押すのと同じです。仮想マシン内のオペレーティングシステムがスムーズにシャットダウンします。

オペレーティングシステムのインストール


仮想マシンはサーバーのディスクにフルアクセスできるため、オペレーティングシステムを手動でインストールするために使用できます。唯一の制限はRAMの容量です。ISOイメージをRAMに配置できるとは限りません。私たちは、画像を保存するためのRAMの4ギガバイトを選択を/ mnt：

mount -t tmpfs -o size=4G tmpfs /mnt

我々はまた、FreeBSDの12.0オペレーティングシステムのインストールイメージをロードします。

wget -P /mnt ftp://ftp.freebsd.org/pub/FreeBSD/releases/amd64/amd64/ISO-IMAGES/12.0/FreeBSD-12.0-RELEASE-amd64-bootonly.iso

今、私たちは、VMを起動することができます：-boot dは フラグはCDドライブからの起動を設定します。VNCクライアントによって接続され、FreeBSDローダーが表示されます。

qemu-system-x86_64 \
-m 2048M \
-net nic -net user \
-enable-kvm \
-cpu host,nx \
-M pc \
-smp 2 \
-vga std \
-drive file=/dev/sda,format=raw,index=0,media=disk \
-drive file=/dev/sdb,format=raw,index=1,media=disk \
-vnc :0,password \
-monitor stdio \
-cdrom /mnt/FreeBSD-12.0-RELEASE-amd64-bootonly.iso \
-boot d






DHCPはインターネットへのアクセスに使用されたため、構成後に、新しくインストールされたシステムを起動してネットワーク設定を修正することが必要になる可能性があります。サーバーにインストールされ、VMでエミュレートされるネットワークカードが異なるため、ネットワークアダプタードライバーのインストールが必要になる場合があります。

結論


サーバーコンソールへのリモートアクセスを整理するこの方法は、サーバーのリソースの一部を消費しますが、サーバーのハードウェアコンポーネントに特別な要件を課すことはないため、ほとんどすべての条件で実行できます。このソリューションを使用すると、ソフトウェア障害の診断が大幅に容易になり、リモートサーバーの機能を復元できます。
