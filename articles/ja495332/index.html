データ検証用のライブラリquartetが作成されたときに、次のランドマークが設定されました。


  • TypeScript
  • 簡潔さと単純さ
  • パフォーマンス

この記事では、quartetTypeScriptに焦点を当てることを検討します。


動機


TypeScriptを使用してプロジェクトに取り組んでいます。したがって、このライブラリを作成したとき、TypeScriptを知っている人にはカルテットをまったく新しいものとして学習させず、このライブラリですでに知っていることを認識させたいと思いました。


ユーザー定義型ガード


例を考えてみましょう。APIを使用してユーザーに関するデータをリクエストします。次のタイプであると想定します。


interface User {
  id: string;
  name: string;
  gender: "male" | "female";
  age: number;
  phoneBook: {
    [name: string]: string;
  };
}

検証関数から取得したいもの：


const probablyUser: unkown = { ... }

if (checkUser(probablyUser)) {
    // probablyUser has type User
    console.log(probablyUser.name)
} else {
    // probablyUser has type unkown
    throw new Error('Probably User has not type User')
}

この目標を達成するために、ユーザー定義のタイプガードが使用されます。


つまり、関数宣言は次のようになります。


function checkUser(probablyUser: any): probablyUser is User {
  // ...
}

quartetこのような関数を作成するために使用してみましょう：


import { v } from "quartet";

const checkUser = v({
  id: v.string,
  name: v.string,
  gender: ["male", "female"],
  age: v.number
  phoneBook: {
    [v.rest]: v.string,
  }
});

そのようなコードを書いたら、TypeGuardではない関数を取得します。


chechUser: (value: any) => boolean;

TypeGuardにするには、この関数によって検証される型を宣言的に示す必要があります。これは次のように行われます：


const checkUser = v<User>({
  // ...
});

最終的に：


chechUser: (value: any) => value is User

この項目に関して2つのポイントがあります。


保証

const checkNumber = v<number>({ name: v.string });
// checkNumber: (value: any) => value is number
