UI / UX对Figma原型示例的分析和基本原理

本文针对的对象



在此出版物中，我想帮助初学者和不太出色的业务分析师，项目经理，想要建立第一个网站的产品所有者，以及需要建立网站或应用程序布局的每个人，他不知道如何开展业务。 。



Internet上有很多有关此主题的资料，但是在大多数情况下，它们要么太肤浅，要么需要术语知识和使用特定系统的经验来创建原型。在不假装“创造黄金”的前提下，我仍然希望我的文章对那些想进入使用现代工具而不是通过Paint，Word或Excel创建图纸的光荣世界的人有用和有趣。也就是说，本文适用于有关创建工作原型的假人。



首先，我将简要描述什么是原型和UI / UX设计，然后将描述用于创建它们的工具，然后在本文结尾，我将引导您通过一个小示例在其中一种工具中创建原型-Figma。



而不是介绍



在现代世界中开发新产品非常昂贵，有时甚至非常非常昂贵。即使是最小的更改也会花费金钱。错误的代价很大，不仅以完成成本来衡量，而且以利润损失，客户损失和市场份额来衡量。因此，甚至在开发开始之前就考虑并协调已实现功能的所有外部属性非常重要：用户类型，他们将要做什么以及为什么要做，页面和按钮的设计（包括文本和图片）以及使用场景。







怎么做？写在纸上-是可能的，也是必要的，但是用户根据他阅读的结果将了解什么，对开发人员来说，实现该系统会很方便吗？该解决方案已经发明了很久-首先创建一个布局，然后创建一个可用的原型。假设模型是“木制的”，而原型只能“打开门”，但是在这一步，客户将能够“触摸”他的产品并及时发表评论。



以上内容适用于任何类型的产品，包括物理产品和电子产品。接下来，我们将讨论IT领域（站点，应用程序，公司信息系统等）中的原型，定义UI和UX是什么，原型在产品开发中必须满足哪些标准，并考虑用于创建Figma原型的环境的某些功能。



什么是原型IT产品？应该是什么样？



原型是一种动态的产品模型，它可以通过界面模拟用户交互，您可以通过它快速而廉价地接收来自客户或目标受众代表的反馈。







重要的是要注意，原型不是站点（mocap）或布线框架（未来产品的一组块）的设计布局。外观上可能与最终产品有所不同。原型中最重要的是，它是一种用于测试假设和逻辑，制定用户脚本的工具。也就是说，原型必须是可点击的。

原型设计是一种非常昂贵且耗时的通信设计形式。还值得注意的是，原型通常不是可以提供的最佳文档，因为它们需要“阅读”一些了解界面的工作。但是，在实现大型项目时，不能没有它们，因为借助原型，您可以在实践中验证未来产品，分析其所有优点和缺点，从潜在用户那里获得最准确的反馈，并在最终完成不相关的功能时节省更多的钱。



必须记住，产品原型不是最终版本。但是，正是他将帮助使产品更接近理想，及时发现缺陷并正确计划进一步的措施清单。



什么是UI / UX？



UX / UI设计自远古时代就已经存在，并且不仅适用于应用程序和站点。只是之前的名称有所不同。例如，歌手缝纫机。毕竟，在19世纪，有人提出了它的外观和使用方法。同时，用户必须实现的目标是快速，精美，高效地缝制一些东西。







那么，UI和UX设计是什么？



UI-代表用户界面并翻译“用户界面”。该术语通常被理解为屏幕形式和界面的可视化。 UI是关于图形设计的。 UI设计器可以处理颜色，构图，动画，字体。 UI设计器基于原理图草图（线框）草稿，收集最终的设计布局。



Ux或“用户体验”翻译为“用户体验”。就是说，关于如何使用界面，是用户容易实现还是困难实现。UX设计人员将客户体验转移到界面上，建立和测试用户与系统交互的假设。



在UX / UI设计下，实现任何与可用性一样重要的设计用户界面。

UX和UI设计并排时常常会感到困惑，但是它们仍然是不同的概念，应该在路径的最后融合为一个整体。







链接原型和UX / UI设计



UX / UI信息系统的原型是设计吗？我认为是。

UI / UX设计是原型吗？是的，但是仅当业务逻辑以一种或另一种方式叠加在用户界面上时。如果在一张纸上某处描述了用户体验，并且无法在界面上向客户端演示，那么我们有独立的UI和UX设计，但是没有原型。







没有软件开发的原型就是一个很好的例子。该网站的页面绘制在纸上。实验者坐在用户面前，并在他面前放了一张网站首页的表格。用户在按钮上戳按钮或指向文本字段，说出他在此处输入的内容。对于每个用户操作，实验者将前一张纸替换为一张新纸，以显示系统对用户动作的响应。



用户界面是-这些是带有绘制按钮的纸。 UX是-这是用户操作。发展是-这些是实验者的行为。有一个原型，因为您可以通过这种方式检查用户体验并从客户端获得反馈。



但是，如果您删除实验者并将用户独自留在纸上会怎样？UX / UI设计将保留，并且原型将不起作用。因此，将不可能测试系统的逻辑。

接下来，为了说明原型的创建，请考虑在Figma中创建个人页面的示例。



什么是Figma？



Figma是面向Web设计人员的程序，使用它不仅可以创建原型，还可以创建站点和应用程序的最终接口。该程序的工作过程非常直观，并且在项目中同时进行联合工作的可能性吸引了越来越多的团队在Figma中实施原型。



Figma的优缺点



在Figma工作的优势包括：



  1. 如果在该项目上工作的用户不超过2个，则该版本是免费的。
  2. 在一个程序中创建和绘制布局
  3. 有大量现成的网站模板
  4. 要使用Figma，需要一个全新的浏览器版本，而桌面版本则不需要功能强大的现代计算机。
  5. 免费版本最多可保存30天的项目版本


缺点Figma：



  1. 缺乏俄罗斯化。Figma必须使用英语。
  2. 尽管Figma脱机工作，但是需要与云同步才能让同事看到所做的更改。


在Figma中创建最简单的原型的示例



Figma中最简单的原型甚至可以由学生创建。下面，作为使用该工具的示例，我将描述创建个人页面的过程。这将是一个两页的网站。在第一页上，我将放置一个小文本块和一张照片，在第二页上-联系人，还将创建菜单标题。



步骤1.注册



不必在Figma中注册一个单独的帐户。如果您有Google邮件，则可以在其帮助下转到该服务。在最初的开幕式上，Figma立即提供关于如何使用该程序的简短培训。



步骤2.创建一个项目



要创建一个新项目，只需单击屏幕右上角的“ +”。您可以在那里打开以前下载的模板之一。



步骤3.创建页面工作区



在为将来的站点或应用程序创建工作区之前，需要确定将在哪些设备上显示该工作区。这很重要，因为在不同型号的显示器，智能手机和平板电脑上，布局将以不同的方式显示，并且照片可能会移到其他地方。



但是现在我们不是在讨论自适应设计，而是在创建工作空间，实际上是在创建未来原型的接口基础，并且所选设备类型的用户将看到我们在Figma中所看到的布局。另外，我注意到在Figma中该站点的工作区域称为框架。



要创建将来页面的基础，必须按F键，然后在右侧显示的菜单中双击上面选择的设备类型。

我们会得到一张白板，您可以在其中更改大小，背景和其他参数，并在其上创建页面界面。







步骤4.添加图片



在Figma中，将图片添加和裁剪为任意形状非常容易。为此，只需将所需的形状和图像添加到工作区，使用ctrl组合键选择两个对象，然后按ctrl + alt +M。图片高于数字很重要。您可以通过单击形状名称在工作区和左侧面板中选择形状。







步骤5.添加文本字段



要添加文本，必须按下带有字母“ T”的键，然后在工作区中为文本字段选择一个位置。可以在右侧功能区中更改文本块选项。例如，您可以将块在工作区域的中心居中。







步骤6.创建菜单



在本文中，我并不打算教读者使用Figma的所有方面，因此我们拥有的菜单将是最简单的-两键式菜单。按钮是结合文本框的矩形。您可以通过按CTRL + G将对象组合成一个组，然后将文本字段居中放置在将来按钮的中心。



步骤7.复制页面的基础



该页面的基础包括：工作区，菜单和文本框。要创建其副本的副本，可以在首页上的对象组之外删除图像遮罩。为此，请在蒙版名称上单击鼠标左键并将图标向上拖动。然后，在对象列表中，选择第一页底部包含的对象组，然后按CTRL +D。







步骤8.添加外部链接



通常，创建单页站点仅出于一种目的-增加到主站点的访问量。也就是说，一页实际上是指向第三方资源的链接。不幸的是，在Figma中，您只能放置指向文本的链接。为此，请选择文本字段，然后单击屏幕顶部的链接图标，然后输入第三方站点的地址。







步骤9.在页面之间添加链接



在Figma内部向对象添加链接类似于向外部源添加链接，不同之处在于您需要在某个地方获取内部对象的链接。为此，请右键单击该对象，然后从下拉菜单中选择“复制/粘贴”和“复制链接”。







步骤10.演示原型



最后一步是向客户展示原型。最简单的方法是单击屏幕右上角的“共享”按钮，然后在弹出窗口中选择“复制链接”。







结果



您可以通过链接查看并单击完整的原型。
