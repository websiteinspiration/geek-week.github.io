ヘキサポッドデバイス


現在、Wi-Fi ESP8266を搭載したUNO R3ボードがメインコントローラーとして使用されています。実際、このボードには2つのコントローラーが搭載されており、UARTインターフェースを介して相互に対話します。



事実にもかかわらず、宇野は、コンピューティングリソースのかなり限られた量を持って、基本的なコマンドを実行する方法ロボットを教えるために十分です：

  • 与えられた速度と持続時間の直線運動
  • 左または右への円運動（所定の位置で回転）
  • 事前設定された四肢の位置を取る

ESP8266は、ワイヤレス通信チャネルの編成を担当し、Unoが制御コマンドを受信するためのゲートウェイとして機能します。

ロボットは、ロボットで確立されたTelnetセッション内のローカルネットワークを介して、またはコントローラーへの有線接続（ファームウェアまたはデバッグ用）を介して制御できます。便宜上、ロボットを制御するための最小限のインタラクティブ機能を実装するシンプルなAndroidアプリケーションも作成しました。



下の図は、ヘキサポッドの構造を概略的に示しています。



すべてのサーボはMultiservo Shield拡張カードに接続されており、18個のサーボを制御できます。 Arduinoとの通信はI²Cバスを介して行われます。したがって、18のサーボが同時にステアリングしても、ほとんどすべてのArduinoピンはフリーのままです。
拡張ボードには、接続されたサーボに電力を供給するためのコネクタがあることに注意してください。しかし、ボードが設計されている最大許容電流は約10Aであり、MG996Rサーボドライブに電力を供給するのに十分ではなく、その合計最大消費電流は指定値を超える可能性があります。したがって、私たちのバージョンでは、各サーボは、拡張ボードをバイパスして別々の電源ラインに接続されていました。

方法coxaAngle、femoraAngle、tibiaAngleはあなたが設定したり、個々の脚の関節の回転角度を見つけることができます。補助メソッドgetAnglesとgetPointは、あなたが彼女の手足の空間内の所与の点のために脚の角度の値を計算することができたとの直接および逆運動学を、コンピューティングのロジックを実装します。またはその逆、現在の角度値のスペースポイント。

各関節の平均位置は角度のゼロ値に対応し、関節の回転範囲は-90〜90度の範囲です。

マルチタスクエミュレーション


Geksapodの クラスはの実装を継承AJobManagerのクラスをとのインスタンスが含まれMotionJobのクラスを、そこからターン継承でAJobのクラス。これらのクラスを使用すると、いわゆる非プリエンプティブマルチタスクを実装でき、プログラムの線形性から抽象化して、同時に複数のタスクを実行できます。

クラスの仕事
class AJob {
  friend class AJobManager;
private:
  AJobManager* m_pAJobManager;   
  AJob* mJobNext;                      
  unsigned long m_counter;         //    onRun
  unsigned long m_previousMillis; //     onRun
  unsigned long m_currentMillis;  //   
  unsigned long m_delayMillis;    //    onRun
  void run();
public:
  AJob(AJobManager*, unsigned long delay = 0L);
  ~AJob();
                  
  void finish();  //     
  long counter(); //    onRun    
  long setDelay(unsigned long); //    onRun
  unsigned long previousMillis();//      onRun
  unsigned long currentMillis(); //    
                              
  virtual void onInit();  //      
  virtual void onRun();   //     
  virtual void onDone();  //        finish
};


AJob クラスは、同時実行が必要なすべてのタスクの基本クラスです。その相続人は、実行されているタスクのロジックを実装するonRunメソッドをオーバーライドする必要があります。プリエンプティブマルチタスクの詳細を考えると、このメソッドの呼び出しに時間がかかりすぎてはなりません。タスクロジックをいくつかの軽いサブタスクに分割することをお勧めします。各サブタスクは、個別のonRun呼び出しに対して実行されます。

クラスAJobManager
class AJobManager {
  friend class AJob;
  AJob* mJobFirst;  //      
  void attach(AJob*);   //    
  void dettach(AJob*); //    
  void dettachAll();   //   
public:  
  AJobManager();
  ~AJobManager();
  void setup();
  void loop();
};


AJobManager クラスにはより控えめな宣言があり、2つのパブリックメソッドのみが含まれています：setupとloop。メインプログラムループを開始する前に、セットアップメソッドを1回呼び出す必要があります。その中で、リストから各タスクに対応するonInitメソッドを呼び出すことにより、すべてのタスクの1つずつ初期化が行われます。
タスクは、コンストラクターが呼び出されると自動的にリストに追加され、タスク自体のpublic finishメソッドを呼び出すことによって削除できます。ループ

方式メインプログラムサイクルで繰り返し呼び出され、リストから各タスクのロジックを指定した間隔で順次実行します（インストールされている場合）。

したがって、AJobManagerクラスから継承されたGeksapodクラスのインスタンスを作成するときに、便利なマルチタスクツールを自由に利用できます。

ムーブメントの実装


体の動きは、特定の時点での体の位置を決定する関数によって説明できます。このような関数は複合することができます。つまり、関数のセットにすることができ、それぞれが特定の期間にのみ適用されます。

ヘキサポッドの四肢のさまざまなタイプの動きが定義されており、Motionクラスから継承したクラスを使用して拡張できます。

クラスモーション
class Motion {
  friend class MotionJob;
protected:
  long m_MaxTime;     //      
  long m_TotalTime;   //      
  bool m_IsLooped;    //   
  Motion* m_pNext;    //     
public:  
  Motion(long maxTime, bool isLooped, long totalTime = -1, Motion* pNext = NULL);
  ~Motion();
  
  inline long maxTime() { return m_MaxTime; }
  inline long totalTime() { return m_TotalTime; }
  inline bool isLooped() { return m_IsLooped; }

  //          

  //       0 <= time <= m_MaxTime 
  virtual int getPose(long time, Vector3D points[FOOTS_COUNT]) { return E_NOT_IMPL; };  
  //        0 <= time <= m_MaxTime
  virtual int getPose(long time, int pose[FOOTS_COUNT][3]) { return E_NOT_IMPL; };       
};


移動を実現するために、オーバーライドgetPoseの方法は、時間の一定期間ロボットの手足の位置を返さなければならない時間 0からの範囲のmMaxTime（ミリ秒）。ムーブメントがループしている場合（m_IsLooped == true）は、移動時間をm_TotalTimeに時間を設定することで制限できます。そして最後に、それらをリストに組み合わせることにより、一連の動きを整理することができます。

したがって、ロボットの動きを説明する機会があります。しかし、説明自体（この場合、Motionから継承されたクラスのインスタンス）はロボットを動かしません。ヘキサポッドの脚を再配置するメカニズムが必要です。そのメカニズムは、与えられた説明に従って導かれます。

このメカニズムは、Geksapodクラスで宣言されたMotionJobクラスのインスタンスです。
