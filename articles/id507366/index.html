Pengujian kinerja adalah proses optimalisasi yang berkesinambungan, pemantauan kinerja sistem dan memastikan toleransi kesalahan mereka baik pada tahap pemodelan dan desain, dan pada setiap iterasi perubahan.







Sekarang di Tinkoff kami secara aktif terlibat dalam pengembangan pengujian kinerja layanan kami, termasuk pengembangan alat pengujian. Kami menggunakan Gatling sebagai alat utama untuk menghasilkan beban . Berikut ini keunggulan utamanya:







  • Plugin Gatling SBT membuatnya sangat mudah untuk menjalankan tes dari sistem CI.
  • Gatling sangat cocok untuk kerja tim dalam proyek. Skripnya adalah kode DSL Gatling yang dapat dibaca manusia, lebih mudah menyimpannya di git dan versi.

import ru.tinkoff.gatling.config.SimulationConfig._
class SampleSimulation extends Simulation {
  val stageWeight    = getDoubleParam("stageWeight") // Double  
  val startIntensity = getDoubleParam("startIntensity")
  val warmUpDuration = getDurationParam("warmUp") // FiniteDuration 
  setUp(
    CommonScenario().inject(
      rampUsersPerSec(0) to startIntensity during warmUpDuration,
      incrementUsersPerSec(intensity * stageWeight)
        .times(stagesNumber)
        .eachLevelLasting(stageDuration)
        .separatedByRampsLasting(rampDuration)
        .startingFrom(startIntensity)
    )
  ).protocols(httpProtocol).maxDuration(testDuration)
}
