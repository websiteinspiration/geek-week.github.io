CCAの能力におけるネットワークモジュールのWorldSkillsタスクのソリューション。パート1-基本設定

WorldSkills運動は、現代の労働市場で求められている主に実践的なスキルを参加者に提供することを目的としています。コンピテンシー「ネットワークおよびシステム管理」は、ネットワーク、Windows、Linuxの3つのモジュールで構成されています。タスクはチャンピオンシップごとに異なり、会議の条件は変わりますが、タスクの構造はほとんど変わっていません。

LinuxとWindowsのアイランドに関してはネットワークアイランドが単純であるため、最初のネットワークアイランドになります。

この記事では、次のタスクが考慮されます。

  1. トポロジーに従ってすべてのデバイスに名前を付けます
  2. すべてのデバイスにwsrvuz19.ruドメイン名を割り当てます
  3. Ciscoパスワードを使用して、すべてのデバイス上にユーザーwsrvuz19を作成します。
    • ユーザーパスワードは、ハッシュ関数の結果として構成に保存する必要があります。
    • ユーザーには最大の特権レベルが必要です。
  4. すべてのデバイスに対して、AAAモデルを実装します。

    • リモートコンソールでの認証は、ローカルデータベースを使用して行う必要があります（RTR1およびRTR2を除く）
    • 認証が成功した後、リモートコンソールからログインする場合、ユーザーは最大の特権レベルですぐにモードに入る必要があります。
    • ローカルコンソールで認証の必要性を設定します。
    • ローカルコンソールで認証が成功したら、ユーザーは最小の特権レベルでモードに入る必要があります。
    • BR1では、ローカルコンソールで認証が成功すると、ユーザーは最大特権レベルでモードに入る必要があります
  5. すべてのデバイスで、wsrパスワードを設定して特権モードに入ります。
    • パスワードは、ハッシュ関数の結果ではなく、構成に保管する必要があります。
    • 構成内のすべてのパスワードが暗号化された形式で格納されるモードを設定します。

物理レベルのネットワークトポロジを次の図に示します。



ビデオ形式で情報を認識しやすい場合は、これらのタスクのソリューション全体を次のビデオで見つけることができます。



1.トポロジーに従ってすべてのデバイスの名前を設定します


デバイス名（ホスト名）を設定するには、グローバル構成モードからコマンドを入力する必要があります。この場合、SW1の代わりに、タスクで指定された機器の名前を書き込む必要があります。 設定を視覚的に確認することもできます-プリセットスイッチの代わりにSW1になりました：hostname SW1



Switch(config)# hostname SW1
SW1(config)#

設定後の主なタスクは、構成を保存することです。

これは、次のコマンドを使用して、グローバルコンフィギュレーションモードから実行できますdo write。

SW1(config)# do write
Building configuration...
Compressed configuration from 2142 bytes to 1161 bytes[OK]

または、次のコマンドを使用して特権モードからwrite：

SW1# write
Building configuration...
Compressed configuration from 2142 bytes to 1161 bytes[OK]

2.すべてのデバイスにwsrvuz19.ruドメイン名を割り当てます


コマンドを使用して、グローバルコンフィギュレーションモードからデフォルトのwsrvuz19.ruドメイン名を設定できますip domain-name wsrvuz19.ru。

確認は、グローバルコンフィギュレーションモードからdo show hosts summaryコマンドを使用して行われます。

SW1(config)# ip domain-name wsrvuz19.ru
SW1(config)# do show hosts summary
Name lookup view: Global
Default domain is wsrvuz19.ru
...

3. ciscoパスワードを使用して、すべてのデバイスでwsrvuz19ユーザーを作成します。


このようなユーザーを作成して、最大レベルの特権を付与し、パスワードをハッシュ関数として保存する必要があります。これらの条件はすべて、チームによって考慮されますusername wsrvuz19 privilege 15 secret cisco。

ここに：

username wsrvuz19-ユーザー名;
privilege 15-特権のレベル（0-最小レベル、15-最大レベル）。
secret cisco-MD5ハッシュ関数の形式でのパスワードの保存。

showコマンドをrunning-config使用すると、現在の構成の設定を確認できます。ここで、ユーザーが追加された行を見つけ、パスワードが暗号化された形式で保存されていることを確認できます。

SW1(config)# username wsrvuz19 privilege 15 secret cisco
SW1(config)# do show running-config
...
username wsrvuz19 privilege 15 secret 5 $1$EFRK$RNvRqTPt5wbB9sCjlBaf4.
...

4.すべてのデバイスに対して、AAAモデルを実装します


AAAモデルは、認証、承認、およびイベントアカウンティングシステムです。このタスクを完了するための最初のステップは、AAAモデルを有効にし、ローカルデータベースを使用して認証が実行されることを示すことです。

SW1(config)# aaa new-model
SW1(config)# aaa authentication login default local

a。リモートコンソールでの認証は、ローカルデータベースを使用して行う必要があります（RTR1およびRTR2を除く）

タスクは、ローカルとリモートの2種類のコンソールを定義します。リモートコンソールを使用すると、たとえばSSHまたはTelnetプロトコルを介してリモート接続を実装できます。

このタスクを完了するには、次のコマンドを入力する必要があります。

SW1(config)# line vty 0 4
SW1(config-line)# login authentication default
SW1(config-line)# exit
SW1(config)#

チームはline vty 0 4、0から4

の仮想端末回線の構成に進みます。チームlogin authentication defaultは、仮想コンソールでデフォルトの認証モードをオンにし、デフォルトのモードは、コマンドによって最後のタスクで設定されましたaaa authentication login default local。

リモートコンソールコンフィギュレーションモードを終了するには、コマンドを使用しexitます。

信頼できるテストは、あるデバイスから別のデバイスへのTelnetテスト接続です。このため、選択した機器の基本的なスイッチングとIPアドレッシングを設定する必要があることに注意してください。

SW3#telnet 2001:100::10
User Access Verification
Username: wsrvuz19
Password:
SW1>

b。認証が成功した後、リモートコンソールからログインすると、ユーザーは最大の特権レベルですぐにモードに入る必要があります

この問題を解決するには、仮想端末回線の構成に戻り、コマンドprivilege level 15で特権レベルを設定する必要があります。ここで、15は最大レベル、0は最小特権レベルです。

SW1(config)# line vty 0 4
SW1(config-line)# privilege level 15
SW1(config-line)# exit
SW1(config)#

最後の節からの解決策-リモートTelnet接続はチェックとして機能します。

SW3#telnet 2001:100::10
User Access Verification
Username: wsrvuz19
Password:
SW1#

認証後、ユーザーはすぐに特権モードに入り、非特権モードをバイパスします。つまり、タスクは正しく完了します。

CD。ローカルコンソールで必要性を設定し、認証が成功した場合、ユーザーは最小限の特権レベルでモードに入る必要があります

これらのタスクにおけるチームの構造は、以前に解決されたタスク4.aおよび4.bと一致します。コマンドline vty 0 4は次のように置き換えられconsole 0ます：

SW1(config)# line console 0
SW1(config-line)# login authentication default
SW1(config-line)# privilege level 0
SW1(config-line)# exit
SW1(config)#

すでに述べたように、最小特権レベルは数値0によって決定されます。検証は次のように実行できます。

SW1# exit
User Access Verification
Username: wsrvuz19
Password:
SW1>

認証後、タスクに記載されているように、ユーザーは非特権モードに入ります。

e。BR1では、ローカルコンソールで認証が成功すると、ユーザーは最大特権レベルでモードに入る必要があります

BR1でローカルコンソールを設定すると、次のようになります。

BR1(config)# line console 0
BR1(config-line)# login authentication default
BR1(config-line)# privilege level 15
BR1(config-line)# exit
BR1(config)#

検証は、前の段落と同じ方法で実行されます。

BR1# exit
User Access Verification
Username: wsrvuz19
Password:
BR1#

認証後、特権モードに移行します。

5.すべてのデバイスで、wsrパスワードを設定して特権モードに入ります


タスクでは、特権モードのパスワードは標準でクリアな形式で保存する必要があると述べていますが、すべてのパスワードの暗号化モードでは、パスワードをクリアな形式で表示できません。特権モードに入るパスワードを設定するには、コマンドを使用する必要がありますenable password wsr。キーワードを使用passwordして、パスワードを保存するタイプを決定します。ユーザーを作成するときにパスワードを暗号化する必要がある場合、キーワードはword secretであり、オープンストレージに使用されますpassword。

現在の構成を表示して、設定を確認できます。

SW1(config)# enable password wsr
SW1(config)# do show running-config
...
enable password wsr
!
username wsrvuz19 privilege 15 secret 5 $1$5I66$TB48YmLoCk9be4jSAH85O0
...

タスクで述べたように、ユーザーパスワードは暗号化された形式で格納され、特権モードに入るパスワードは平文で格納されていることがわかります。
すべてのパスワードを暗号化された形式で保存するには、コマンドを使用する必要がありますservice password-encryption。現在の構成を表示すると、次のようになります。

SW1(config)# do show running-config
...
enable password 7 03134819
!
username wsrvuz19 privilege 15 secret 5 $1$5I66$TB48YmLoCk9be4jSAH85O0
...

パスワードはクリアテキストで表示できなくなりました。
