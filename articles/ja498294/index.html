オブジェクト検出 認識し、支配する。パート1



コンピュータービジョンテクノロジーは、今日の現実において、生活とビジネスをより簡単に、より安く、より安全にすることができます。さまざまな専門家によると、この市場は今後数年間で成長の方向にのみ移動し、生産性と品質の方向に適切な技術の開発を可能にします。最も人気のあるセクションの1つは、オブジェクト検出（オブジェクト検出）です。これは、画像またはビデオストリーム内のオブジェクトの定義です。

オブジェクトの検出が古典的な機械学習（カスケード、SVMなど）によってのみ解決された時代はすでに過ぎており、現在、この領域ではディープラーニングに基づくアプローチが支配しています。 2014年に、この分野でのその後の研究開発に大きな影響を与えるアプローチが提案されました-R-CNNモデル。その後の改良（Fast R-CNNおよびFaster R-CNNの形で）により、これは最も正確なものの1つになり、それが今日まで使用される理由になりました。

R-CNNに加えて、オブジェクトを検索する多くのアプローチがあります：Yoloファミリー、SSD、RetinaNet、CenterNet ...それらのいくつかは代替アプローチを提供し、他のものはパフォーマンスインジケーターを増やす方向で現在のものを開発しています。チップとトリックが豊富なため、それらのほとんどすべてについての議論は別の記事で行うことができます:)

調査するために、2段階のオブジェクト検出モデルを分析した一連の記事を提案します。彼らのデバイスを理解する能力は、他の実装で使用される基本的なアイデアの理解をもたらします。この投稿では、最も基本的な、したがって最初のR-CNNについて検討します。

用語


境界ボックス -画像の特定の領域を区切る座標-ほとんどの場合、長方形の形状です。2つの形式の4つの座標で表すことができます。cx,cy,w,h）およびレギュラー（xmin,ymin,xmax,ymax）

仮説（提案）、 P-オブジェクトが配置されていると思われる画像の特定の領域（境界ボックスを使用して指定）。

エンドツーエンドのトレーニング -生画像がネットワーク入力に到着し、既製の回答が出てくるトレーニング。

IoU（Intersection-over-Union）-2つの境界ボックス間の交差の度合いのメトリック。

R-CNN


画像内のオブジェクトの位置を決定するために適用できる最初のアプローチの1つは、R-CNN（領域畳み込みニューラルネットワーク）です。そのアーキテクチャーはいくつかの連続したステップで構成され、図1に示されています。

  1. 一連の仮説を定義する。
  2. 畳み込みニューラルネットワークを使用して将来の領域から特徴を抽出し、それらをベクトルにエンコードします。
  3. ステップ2のベクトルに基づく仮説内のオブジェクトの分類。
  4. 仮説の座標の改善（調整）。
  5. ステップ1のすべての仮説が処理されるまで、すべてがステップ2から繰り返されます。

各ステップをより詳細に検討してください。



仮説検索


入力に特定の画像があると、最初に、異なるサイズの小さな仮説に分解されます。この記事の作成者は選択的検索を使用します-トップレベル。セグメンテーションに基づいて一連の仮説（オブジェクトのクラスはまだ問題ではありません）をコンパイルして、ピクセル強度、色差、コントラスト、テクスチャーによってオブジェクトの境界を決定できます。同時に、著者は同様のアルゴリズムを使用できることに注意しています。したがって、約2,000の異なる領域が際立っており、部分的に互いに重なり合っています。より正確な後続処理のために、各仮説は、コンテキストを追加するかのように、4方向すべてでさらに16ピクセル拡張されます。

合計：

  • 入力：元の画像。
  • 出力：さまざまなサイズとアスペクト比の一連の仮説。

画像エンコーディング


前のステップからの各仮説は、独立して互いに独立して、畳み込みニューラルネットワークの入力に入ります。最後のsoftmax-layerなしでAlexNetのアーキテクチャを使用しているため。ネットワークの主なタスクは、着信画像を、最後に完全に接続されたFC7レイヤーから抽出されたベクトル表現にエンコードすることです。したがって、出力は4096次元のベクトル表現です。

AlexNetの入力の次元は3×227×227であり、仮説のサイズはほぼすべてのアスペクト比とサイズであることがわかります。この問題は、入力を目的のサイズに圧縮または拡大するだけで回避されます。

合計：

  • 入力：前のステップで提案された各仮説。
  • 出力：各仮説のベクトル表現。

分類


仮説を特徴付けるベクトルを取得した後、そのさらなる処理が可能になります。目的の領域にあるオブジェクトを特定するために、著者は従来のSVMベースの分離平面分類法を使用します（サポートベクターマシン-サポートベクターマシンは、ヒンジ損失を使用してモデル化できます）。そしてそれはNc+1 個人（ここでは、 Ncオブジェクトの定義されたクラスの数を示し、OvRの原則に従ってトレーニングされたモデルのバックグラウンドを個別に決定するためにユニットが追加されます（One vs. Rest-すべてに対して1つ、マルチクラス分類のメソッドの1つ）。実際、バイナリ分類の問題は解決されています-提案された領域内にオブジェクトの具体的なクラスがあるかどうか。だから方法はNc+1仮説に含まれるオブジェクトの特定のクラスの信頼度を表す2次元ベクトル（背景は歴史的にゼロクラスで示され、Ci=0）

合計：

  • 入力：ネットワークの最後から2番目のレイヤーから提案された各仮説のベクトル（AlexNetの場合、これはFC7です）。
  • 出力：各仮説を順次起動した後、次元マトリックスを取得します 2000×Nc各仮説のオブジェクトのクラスを表します。

仮説座標の指定


手順1で得られた仮説には、常に正しい座標が含まれているとは限らないため（たとえば、オブジェクトが「トリミング」されて失敗する場合があります）、それらをさらに修正することは理にかなっています。著者によると、これによりメトリックがさらに3〜4％増加します。したがって、オブジェクトを含む仮説（オブジェクトの存在は分類ステップで決定されます）は、線形回帰によってさらに処理されます。つまり、「バックグラウンド」クラスの仮説には、領域の追加処理は必要ありません。実際にはそこにオブジェクトはありません...

クラスに固有の各オブジェクトには特定のサイズとアスペクト比があるため、論理的であり、各クラスに独自のリグレッサを使用することをお勧めします。

前のステップとは異なり、作成者は入力が最適に機能するようにFC7レイヤーの非ベクターを使用します、および最後のMaxPoolingレイヤーから抽出された機能マップ（AlexNetでは、 MaxPool5、寸法256×6×6）。説明は次のとおりです。ベクトルには、オブジェクトの存在に関する情報がいくつかの特徴的な詳細とともに格納され、機能マップには、オブジェクトの場所に関する情報が最もよく格納されます。

合計：

  • 入力：背景以外のオブジェクトを含む各仮説の最後のMaxPoolingレイヤーからの属性のマップ。
  • 出力：仮説の境界ボックスの座標への修正。

正と負の仮説の指定


先生と一緒に教えるときは、常にクラスの間に一定のバランスが必要です。逆の場合、分類精度が低下する可能性があります。たとえば、2つのクラスを持つサンプルで最初のケースが数パーセントのケースでのみ発生する場合、異常値として解釈される可能性があるため、ネットワークがそれを決定する方法を学習することは困難です。オブジェクト検出タスクの場合、まさにそのような問題があります-単一のオブジェクトを含む画像では、ほんの少しの仮説（〜2000）だけがこのオブジェクト（Ci>0）、そして他のすべての人が背景です（Ci=0）

必要な表記法を採用します。オブジェクトを含む仮説はポジティブ（ポジティブ）と呼ばれ、オブジェクトなし（背景のみ、またはオブジェクトの重要でない部分のみを含む）- ネガティブ（ネガティブ）と呼ばれます。

その後、画像の2つの領域間の交差を決定するために、Union上の交差のメトリックが使用されます。それは非常に単純であると考えられています：2つの領域の交差領域は、領域の合計領域で除算されます。下の画像では、メトリックカウントの例の図を見ることができます。



肯定的な仮説があれば、すべてが明確です。クラスが正しく定義されていない場合は、罰金を科す必要があります。しかし、ネガはどうですか？正の仮説だけではありません...まず、負の仮説のすべてが同じように認識するのが難しいというわけではありません。たとえば、背景のみを含むケース（簡単なネガティブ）は、別のオブジェクトまたは目的の小さな部分（ハードネガティブ）を含むよりも分類がはるかに簡単です。

実際には、イージーネガティブとハードネガティブは、バウンディングボックス（Union上のIntersectionで使用したもの）と画像内のオブジェクトの正しい位置との交差によって決定されます。たとえば、交差がない場合、または非常に小さい場合、これは簡単にネガティブです（Ci=0）ラージがハードネガティブまたはポジティブの場合。ハードネガティブマイニング

アプローチでは、ハードネガティブのみをトレーニングに使用することをお勧めします。それは、それらを認識することを学んだので、簡単なネガティブ仮説で自動的に最良の作業を達成するためです。ただし、そのようなイデオロギーは、後続の実装（Fast R-CNNから始まる）にのみ適用されます。

非最大抑制


多くの場合、モデルは同じオブジェクトを指す非常に信頼性の高いいくつかの仮説を識別します。非最大抑制（NMS）を使用すると、そのような場合を処理して、最良の境界ボックスを1つだけ残すことができます。ただし、同時に、イメージに同じクラスの2つの異なるオブジェクトが含まれる場合があることを忘れないでください。図3は、アルゴリズムの操作前（左）と操作後（右）の操作の効果を示しています。



1つのクラスで作業するためのアルゴリズムを検討してください（実際には、各クラスに個別に適用されます）。

  1. 入力では、関数は1つのクラスの一連の仮説と、仮説間の最大の交差を設定するしきい値を受け取ります。
  2. 仮説は「信頼度」によって分類されます。
  3. サイクルでは、最初の仮説が選択され（信頼値が最も高い）、結果セットに追加されます。
  4. サイクルでは、次の2番目の仮説が選択されます（ステップ3の後に残っている仮説の中から）。
  5. 選択した仮説間の共通部分が選択したしきい値より大きい場合（共通部分は結合の共通部分に基づいて計算されます）、2番目の仮説は破棄され、結果セットにはそれ以上存在しません。
  6. 仮説が完全に列挙されるまで、すべてがステップ3から繰り返されます。

擬似コードは次のようになります。

function nms(hypotheses, threshold):
    sorted = sort(hypotheses.values, key=hypotheses.scores)
    result = []
    for first in sorted:
        result.join(first)
        without_first = sorted / first
        for second in without_first:
            if IoU(first, second) > threshold:
                 sorted.remove(second)
    return result


トレーニング


仮説分離ブロックは学習できません。

ネットワークは互いに分離されたいくつかのブロックに分割されているため、エンドツーエンドでトレーニングすることはできません。つまり、学習は逐次的なプロセスです。

ベクタービュートレーニング


ImageNetで事前トレーニングされたネットワークがベースとして使用されます。このようなネットワークは、着信画像から重要な機能をすでに十分に抽出できます。必要なクラスで動作するようにトレーニングすることは残ります。これを行うには、出力レイヤーのディメンションを次のように変更しますNcすでに修正されたバージョンをトレーニングします。最初のレイヤーは、主要な（すべての画像でほぼ同じ）特徴を抽出し、トレーニング中の後続のレイヤーは目的のクラスの特徴に適応するため、ブロックできます。したがって、収束ははるかに速く達成されます。ただし、それでもトレーニングがうまくいかない場合は、プライマリレイヤーのロックを解除できます。既存のウェイトを正確に調整する必要があるため。高い学習率（学習率）を使用することはお勧めしません-既存の重みをすばやく消去できます。

ネットワークがオブジェクトを適切に分類することを学習した場合、SoftMaxがアクティブ化された最後のレイヤーは破棄され、FC7レイヤーが出力になり、その出力は仮説のベクトル表現として解釈できます。

このステップで肯定的なのは、オブジェクトの正しい位置（IoU）と0.5以上交差する仮説です。他のすべては否定的であると考えられます。スケールを更新するには、32の正の仮説と96の負の仮説で構成される128のミニバッチが使用されます。

分類子トレーニング


各仮説の分類には、 Nc+1仮説のベクトル表現の入力を受け取り、一方に対する他方の原則（One-vs-Rest）に基づくSVMモデルは、オブジェクトのクラスを決定します。これらは、1つの例外を除いて、通常のSVMモデルとしてトレーニングされます。このステップでは、ポジティブとネガティブの定義が少し異なります。ここでは、仮説はネガティブと見なされ、正しい位置との交差は0.3未満です。

結論として、このアプローチの主な欠点を強調します。


  1. ステップ1で提案された仮説は、部分的に相互に重複する可能性があります。異なる仮説は同一の部分で構成でき、そのような仮説はそれぞれニューラルネットワークによって個別に処理されました。ほとんどのネットワーク起動は、多かれ少なかれ互いに不必要に重複していることがわかります。
  2. 1つの画像（フレーム）（NVIDIA Titan Black GPU）の通過に約53秒かかるため、リアルタイム操作には使用できません。
  3. 仮説抽出アルゴリズムはまったく教えられていないため、品質をさらに向上させることはほとんど不可能です（悪い仮説をキャンセルした人はいません）。
