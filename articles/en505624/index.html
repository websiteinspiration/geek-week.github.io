Vaex Python Library



Vaex is a high-performance Python library designed to organize ‚Äúlazy‚Äù processing of data frames (similar to pandas data frames), designed for visualization and research of large sets of tabular data. This library is able to calculate the main statistical indicators for the data being studied. At the same time, processing a billion records can be something about a second. The library supports many visualization tools, which helps to explore big data online.

Comparing Vaex and Dask



The Vaex library is not like Dask, but the data structures used in it are similar to the DataFrame entities used in Dask. They are built on the basis of pandas dataframes. This means that Dask inherits from pandas certain restrictions on working with data frames. For example, a requirement that data to be processed must be fully loaded into memory. In the case of Vaex, this is not so.

Vaex does not make copies of data frames; as a result, using this library you can process large-scale data frames on computers with a small amount of RAM.

Both Vaex and Dask use lazy data processing engines. The main difference between them is that Vaex calculates the field values ‚Äã‚Äãwhen they are needed, and in Dask you need to explicitly use the function to calculate the values.

In order to unleash the full potential of Vaex, it is necessary that the data be presented in HDF5 or Apache Arrow format.

Experiments with Vaex

df = pd.DataFrame(np.random.randint(0, 100, size=(n_rows, n_cols)), columns=['col%d' % i for i in range(n_cols)])

If you directly read this CSV file using Vaex, we won‚Äôt get much benefit from it. The read speed will be similar to the speed provided by pandas. On my laptop, both Vaex and pandas read this data in about 85 seconds.

In order to see the capabilities of Vaex, CSV data must be converted to HDF5 (Hierarchical Data Format version 5). Vaex has a function to perform such a conversion, which allows you to work with data whose volume exceeds the amount of available RAM. This is achieved due to the fact that the source data is divided into fragments.

If it is not possible, due to lack of memory, to open a large enough file using pandas, this file can be converted to HDF5 format and processed using Vaex.

dv = vaex.from_csv(file_path, convert=True, chunk_size=5_000_000)

The above function automatically creates an HDF5 file and saves it to disk.

Effective data filtering

Data aggregation


Vaex aggregates data a bit differently than pandas does. But the most important thing here is that in Vaex this operation is very fast.

Create a new column containing the comparison results dv.col1 >= 50:

Vaex combines the operations of grouping and aggregating data in one team. The following command groups the data by column col1_50and calculates the sum of the column col3:
