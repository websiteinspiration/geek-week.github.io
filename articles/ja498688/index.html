素晴らしいA / Bテストガイド

インターネットにはA / Bテストについての情報がたくさんありますが、多くの人はまだそれを間違って行っています。間違いを犯しやすいので、そのような研究には真剣な事前準備が必要です。この記事では、効果的なWebページ分析のために考慮しなければならないA / Bテストの主な側面について説明します。

A / Bテストとは何ですか？


A / Bテスト（スプリットテスト）では、ページの異なるバージョン間でトラフィックを50/50の比率で分割します。本質的に、この方法は「制御された実験」として知られている古い技術の新しい名前です。

新薬の有効性をテストするために、専門家は分割テストを実施します。実際、ほとんどの研究実験はA / Bテストと呼ばれます。それらには、統計データの形式で提示された仮説、研究の主な目的、その変動および結果が含まれます。

それで全部です。例は単純なA / Bテストで、メインページとそのバリエーションの間で50/50のトラフィックが分割されます。



変換の最適化の場合、主な違いはインターネットトラフィックの変動性です。ラボでは、外部変数を制御する方が簡単です。インターネットでは、それらの影響を減らすことができますが、完全に制御されたテストを作成することははるかに困難です。
さらに、新薬のテストにはある程度の正確さが必要です。人の命が危機に瀕しています。技術的な観点から見ると、これはテストがより長く続く可能性があることを意味します。これは、研究者が最初の種類のエラー（誤検知）を回避するために可能な限りすべてを行う必要があるためです。

ただし、ビジネス目標を達成するために、WebリソースのA / Bテストが実行されます。リスクと報酬の分析、調査と開発、科学とビジネスに必要です。したがって、結果は異なる観点から考慮され、決定は研究室の研究者とは異なる方法で行われます。

もちろん、3つ以上のページバリエーションを作成できます。複数の要素を含む調査は、A / B / nテストと呼ばれます。十分なトラフィックがある場合は、必要な数のオプションをテストできます。以下は、バリエーションごとにトラフィックを割り当てたA / B / C / Dテストの例です

その人気にもかかわらず、A / Bテストはオンライン調査の1種類にすぎません。多変量テストを実施したり、多腕バンディット法を使用したりすることもできます。

A / B / nテストは、元のページとその1つ以上のパターンのコンバージョン率を比較する、制御された実験です。

各要素には特定の使用例があり、ページのパフォーマンスに影響します。次の方法でサイトを最大限に活用できます。

  • A / Bテストを実行して、最適なページレイアウトオプションを決定します。
  • 多変量テストを実施してレイアウトを改善し、すべてのページ要素が相互に適切に相互作用することを確認します。


多変量テストの可能性を考慮する前に、テストしたページに多数のユーザーを引き付ける必要があります。ただし、十分なトラフィックがあるため、両方のタイプの調査を使用してサイトを最適化する必要があります。
通常、クライアントは重要な変更をテストするため（ページにさらに影響を与える可能性があるため）、ほとんどの機関はA / Bテストを好みます。また、実行が簡単です。

多腕バンディット方式はA / B / nで、各バリエーションの有効性に基づいてリアルタイムで更新されるテストです。

実際、多腕バンディットアルゴリズムは、トラフィックを2つ（またはそれ以上）のページ（オリジナルとそのバージョン）に送信することから始まります。次に、どのバリエーションが最も効果的であるかに応じて更新されます。最終的に、アルゴリズムは可能な最良のオプションを決定します。



多腕バンディット方式の利点の1つは、最悪の場合のシナリオをテストするときに発生する変換損失を軽減できることです。Googleのこのグラフはすべてをうまく説明しています。



多腕バンディット法とA / B / nテストの両方に長所があります。1つ目は次の場合に最適です。

  • 見出しと短期キャンペーン;
  • 自動スケーリング;
  • ターゲティング
  • 同時最適化と帰属。


使用するテストの種類に関係なく、成功の可能性を高めることは重要です。つまり、テストの頻度が高いほど、変換は速くなります。

A / Bテストにはどのくらい時間がかかりますか？


最初のルール：統計的に有意になるという理由だけでテストを停止しないでください。これはおそらく、初心者のオプティマイザが犯す最も一般的な間違いです。

テストをすぐに停止すると、ほとんどの変更が収入の増加につながらないことがわかります（これが主な目標です）。
1000 A / Aテスト後に得られたこれらの統計に注意してください（2つの同一ページに対して実行されました）。

  • 1000のうち771の実験は90％の有意性に達しました。
  • 1000回のうち531回の実験は95％の有意性に達しました。


テストを途中で停止すると、誤検知のリスクが高まります。
サンプルサイズを決定し、少なくとも2つのワークサイクルを続けて数週間テストします。

この例では、コンバージョン率が3％であり、この率を少なくとも10％増加させることを示しています。このツールは、統計的有意性のレベルを確認する前に、51,486人が各バリエーションを訪問する必要があると述べています。

有意水準に加えて、統計的強度があります。統計的検出力は、タイプIIエラー（偽陰性）を回避しようとします。つまり、最も効果的なページ要素が見つかる可能性が高くなります。

電力の80％がA / Bテストツールの標準であることを覚えておいてください。このレベルを達成するには、大きなサンプルサイズ、壮大な効果、またはより長いテストが必要です。

マジックナンバーはありません


多くの記事では、マジックナンバー（「100コンバージョン」や「1000訪問者」など）をテストを停止するのに最適な時期として挙げています。しかし、数学は魔法とは何の関係もありません。実際、これらの数値のような単純化されたヒューリスティックスよりもすべてが複雑です。MalwarebytesのAndrew Andersonのコメントは次のとおりです。

「あなたの目標は特定の数のコンバージョンではありません。代表的なサンプルと代表的な行動に基づいて仮説を検証するのに十分なデータを収集するように努める必要があります。

100回の変換が可能なのは、非常にまれなケースで、動作に非常に大きな違いがある場合だけですが、時間の動作、一貫性、正規分布などの他の要件が満たされている場合に限ります。同時に、第1種のエラーのリスクは非常に高いままです。


したがって、代表的なサンプルが必要です。どうやって入手するの？次のような外部要因の影響を減らすのに役立つ2つの経済サイクルの間にテストを実施します。

  • \ 曜日。毎日のトラフィックは、曜日によって大きく異なります。
  • \ トラフィックのソース。特定のソースのエクスペリエンスをパーソナライズする必要がない限り。
  • \ ニュースレターやブログ投稿の送信をスケジュールします。
  • \ リピーター。人々はあなたのサイトにアクセスし、購入について考え、10日後に戻ってきて購入することができます。
  • 外部イベント。たとえば、月の途中の給与計算が購入に影響する場合があります。


少量のサンプルには注意してください。数学的エラーに満ちたインターネット上の多くのケーススタディがあります。

すべてを設定したら、終了するまでテスト結果を見ないでください（そして上司に覗かせないでください）。それ以外の場合は、「傾向を検出する」ことで時期尚早の結論を導き出すことができます。

意味する回帰


テストの最初の数日で結果が大きく異なることに気付くでしょう。その後、テストは数週間続くため、平均値に収束します。eコマースサイトの統計の例を次に示します。



  • 最初の2、3日：青（オプション番号3）がわずかに勝利します。バリエーションは、元のページがもたらす12.50ドルに対して、訪問者あたり16ドルをもたらします。多くの（誤って）この時点でテストを終了します。
  • 7日後：ページの青色バージョンが引き続き優先され、相対的な差は非常に大きくなります。
  • 14日後：オレンジバージョン（No.4）が登場！
  • 21日後：まだオレンジ色のバージョンが勝ちます！
  • テストの終了：オプション間に違いはありません。


4週間前にテストを完了した場合、誤った結論を下したことになります。

同様の問題があります：新規性の影響です。変更の目新しさ（大きな青いボタンなど）は、ページオプションにより注目を集めます。時間が経つにつれ、この影響はなくなります。変更が徐々に関連性を失うからです。

複数のA / Bテストを同時に実行できますか？


テストプログラムを高速化し、より多くのテストを実行したい。ただし、一度に複数のA / Bテストを実行することは可能ですか？成長の可能性を高めたり、データを歪めたりしますか？

一部の専門家は、一度に複数のテストを実施することは間違っていると主張しています。すべてが整然としていると言う人もいます。ほとんどの場合、複数のテストを同時に実行しても問題は発生しません。

本当に重要なもの（たとえば、ビジネスモデルや会社の将来に影響を与えるもの）をテストしていない場合、テスト量のメリットは、データの欠陥やランダムな誤検知を上回ります。
複数のテスト間の相互作用のリスクが高い場合は、同時テストの数を減らすか、テストをより長く実行して精度を向上させます。

A / Bテストの設定方法


優先度の高いテストアイデアのリストを作成したら、仮説を立てて実験を行う必要があります。仮説によって、問題が発生する理由を決定します。さらに、良い仮説：

  • 検証可能。測定可能ですので確認できます。
  • 変換の問題を解決します。分割テストは変換の問題を解決します。

A / Bテストで得られた統計


統計の知識は、A / Bテストの結果の分析に役立ちます。上記のセクションでそれらのいくつかを調べましたが、それだけではありません。

A / Bテストで得られた統計の詳細を学ぶ前に知っておくべき3つの概念があります。

  1. 平均。すべてのコンバージョン率を測定するのではなく、サンプルのみを測定します。平均は全体の代表にすぎません。
  2. 分散。確率変数の値のばらつきを、その数学的期待値と比較して測定します。テスト結果とその使用方法に影響します。
  3. 選択。真のコンバージョン率は測定できないため、代表的なサンプルを選択します。


P値とは何ですか？


多くの人が「統計的有意性」という用語を誤って使用しています。それ自体は、テストを停止する合図ではありません。それは何ですか、なぜそれがそれほど重要なのですか？
まず、P値を見てみましょう。これもほとんどの人が理解していません。科学者でさえ、時々混乱します！

P値は、帰無仮説が棄却されたときのエラー（第1種のエラー）の確率を特徴付ける値です。確率BがAよりも高いことは証明されません。これは一般的な誤解です。



要約すると、P値が統計的有意性のレベル（通常は0.05に設定されている）よりも小さい場合に、統計的有意性（または統計的に有意な結果）が達成されたと言えます。

片側および両側のA / Bテスト


一方向のテストでは一方向の変化を検出でき、双方向のテストでは二方向（正と負の両方）の変化を検出できます。

テストソフトウェアがA / Bテストのタイプの1つのみをサポートしているかどうか心配しないでください。必要に応じて、一方向のテストを双方向のテストに簡単に変換したり、その逆を行ったりできます（ただし、テストの前に行う必要があります）。唯一の違いは、しきい値の有意水準です。

ソフトウェアで一方向検定を使用している場合は、使用されているP値を2に分割するだけです。双方向テストの信頼性を少なくとも95％にするには、信頼レベルを97.5％に設定します。 99％の信頼性を実現する場合は、99.5％の値を選択する必要があります。


コンバージョン率はX％だけではありません。およそこの形で示されます：X％（±Y）。この式の2番目の数値は信頼区間であり、分割テストの結果を理解する上で非常に重要です。



A / Bテストでは信頼区間を使用して、サンプリングエラーのリスクを最小限に抑えます。この意味で、私たちは新しいバージョンのページの導入に関連するリスクを管理します。

したがって、ツールが「変換率がX％±Y％であることを95％確信している」と表示された場合、±Y％をエラーのマージンとして考慮する必要があります。

結果の信頼性は、エラーの大きさに大きく依存します。 2つの変換範囲が重複する場合は、テストを続行して、より真実に近い結果を得る必要があります。

ベイジアンおよび頻度統計


多くの一般的なツールでは、A / Bテストにベイズ法と周波数法の両方を使用できます。違いはなんですか？

簡単に言えば、仮説にはベイジアン統計で確率が割り当てられ、頻度統計では確率を割り当てずに検証されます。
