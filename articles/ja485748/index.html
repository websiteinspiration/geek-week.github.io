オプションのチェーニング、nullとの結合、およびコードを記述するためのアプローチをどのように変更するか

こんにちは！

TypeScriptのリリースに従うと、TypeScript 3.7でオプションの連鎖とヌル結合が導入されたことがわかります。また、これらの機能はBabel 7.8.0にデフォルトで含まれています。これは、JavaScript機能の役割の主要な候補の1つです。現在、それらはそれぞれステージ4と3にあります（1月23日、ECMAScript 2020標準が採用されました。ここで、オプションのチェーン、nullとの統合、およびいくつかの興味深いものがあります。最近、Habréに記事が掲載されたことを知っています。JS ES2020で導入された機能についての翻訳ですが、いくつかの追加の例で問題が発生しないと思います。

オプションの連鎖


多くの場合、オブジェクトの深く埋め込まれたプロパティにアクセスする必要があります。高品質ではないコードを100行書き込んだ場合、Uncaught TypeErrorが発生する可能性があります。

const data = {}
data.user.id // Uncaught TypeError: Cannot read property 'id' of undefined
//           -

さらなる例として、この疑似コード（サーバーの誤った応答）を使用します。

{
    'url': 'https://api.github.com/repos/sedighian/Hello-World/pulls/12',
    'number': 12,
    'state': 'open',
    'title': 'Amazing new feature',
    'user': {
        'login': 'sedighian',
        'id': 123234
    },
    'labels': [{
        'id': 208045946,
        'name': 'bug',
    }]
}

Uncaught TypeErrorを回避し、id値を取得しようとすると、「ダンスを行う」必要があります。以前に使用したアプローチは、ネストの各レベルでオブジェクトの真実を検証することでした。このテンプレートは、プロパティにアクセスする方法ではなく、ブール値を返す条件ステートメントに似ていますが、これは、これまでに見つけた中で最もクリーンで安全な方法です。

const userId = data && data.user && data.user.id

//    
const label = data && data.labels && data.labels[0]
const someFuncRes = navigator && navigator.serviceWorker && window.serviceWorker()

または、破壊を好む場合：

const { user: { id } = {} } = data || {}

//       
const { user } = data || {}
const { id } = user || {}

より人間工学的な方法は、LodashまたはEmberを使用することです。

// lodash
import _ from 'lodash'
_.get(data, 'user.id')

// ember
import { get } from '@ember/object'
get(data, 'user.id')

オプションのチェーンで同じことをするにはどうすればよいですか？

const userId = data?.user?.id

//    
const label = data?.labels?.[0]
const someFuncRes = someFunc?.()

nullとの組み合わせ（null coalescing）


アクセスしているオブジェクトのプロパティ値がnullまたは未定義の場合は、デフォルト値を使用します。以前は、演算子|| （論理または）。

デフォルトでログインプロパティにセディアンを書き込む場合は、次のようにします。

//     
data && data.user && data.user.login || 'sedighian'

//    
data?.user?.login || 'sedighian'

//        null
data?.user?.login ?? 'sedighian'

2番目と3番目の例は似ています。nullと組み合わせる利点は何ですか？左側が未定義またはnullの場合のみ、nullと組み合わせると右側に評価されます。これにより、実際の（有効な）偽の値を処理するときのランダムな結果に対する保護が得られます。

''（空の文字列）、falseまたは0を返したいとします。||演算子を使用して 彼は右側を返すため、これは機能しません。この場合、nullと組み合わせると便利です。

//  data.user.alias   ,    
data?.user?.alias ?? 'code ninja' // ''
data?.user?.alias || 'code ninja' // code ninja

//  data.user.volumePreference = 0
data?.user?.volumePreference ?? 7 // 0
data?.user?.volumePreference || 7 // 7

//  data.user.likesCats = false
data?.user?.likesCats ?? true // false
data?.user?.likesCats || true // true

別の方法として、サードパーティのライブラリを使用できます。Emberの場合は、組み込みユーティリティを使用できます。

// lodash
import _ from 'lodash'
_.get(data, 'user.likesCats', true)

// ember
import { getWithDefault } from '@ember/object'
getWithDefault(data, 'user.likesCats', true)

nullと組み合わせると、変数のデフォルト値よりも多くなることに注意してください。これは、対応する値がない場合にコードブロックを実行する別の方法です。

const temp = { celsius: 0 }
temp?.fahrenheit ?? setFahrenheit(temp)

何を覚えておくべきですか？


オプションの文字列の文字の順序を覚えておいてください（最初に疑問符、次にピリオド）。

data.?user // Uncaught SyntaxError: Unexpected token '?'
data?.user // ok

オプションのチェーンは、存在しない関数の呼び出しを防ぎません。

data?.user() // Ungaught TypeError: user is not a function

nullとの組み合わせは、lodash.getまたはEmberObject.getWithDefaultと同じではありません。主な違いは、c nullを組み合わせると値「null」がどのように処理されるかです。

const data = { user: { interests: null } }

// lodash
import _ from 'lodash'
_.get(data, 'user.interests', 'knitting') // null

// ember
import { get } from '@ember/object'
getWithDefault(data, 'user.interests', 'knitting') // null

//   null
data?.user?.interests ?? 'knitting' // knitting
