実際の知識伝達とニューラル機械翻訳

ニューラル機械翻訳（NMT）は急速に発展しています。現在、翻訳者を集めるために、2つの高等教育を受ける必要はありません。ただし、モデルをトレーニングするには、大きな並列コーパス（ソース言語の翻訳が文に関連付けられているコーパス）が必要です。実際には、少なくとも100万組の文について話している。電子フォーム（英語の低リソースNMT）で少量のデータを使用して言語ペアを教える方法を探る別の大きな領域のIMFもあります。

私たちはチュヴァシ-ロシア軍団を集めています。同時に、利用可能なデータ量で何ができるのかを見ています。この例では、90,000ペアのオファーのケースが使用されました。現時点での最良の結果は、知識移転の方法（Eng。Transfer Learning）によって与えられ、それについては記事で説明します。この記事の目的は、簡単に再現できる実装の実用的な例を示すことです。

研修計画は以下のとおりです。大きな（親の）建物を取り、その上でニューラルモデルをトレーニングしてから、娘モデルをトレーニングする必要があります。さらに、翻訳のターゲット言語は同じです：ロシア語。直観的には、これは第二言語を学ぶことと比較できます。 1つの外国語を知っていると、学習が容易になります。また、外国語の狭い領域、たとえば英語の医学用語を勉強しているようにも見えます。最初に一般的に英語を学ぶ必要があります。

親隊として、我々は100万文章のペア撮ってみました英語-ロシアパラレル隊から 100万をカザフ、ロシア隊から。カザフ語のデータには500万文あります。これらのうち、コンプライアンス係数（3列目）が2を超えるもののみが取得され、カザフ語バージョンの方がわずかに良い結果が得られました。チュヴァシュ語とカザフ語は互いに類似しているため、これは直感的に理解できるようです。しかし、実際には、これは証明されておらず、ケースの質にも大きく依存します。親の体の選択の詳細については、この記事を参照してください。 90,000ペアのオファーのドータービルディングについては、ここでサンプルデータを見つけてリクエストできます。

コードに移ります。独自の高速グラフィックスカードがない場合は、Colabサイトでモデルをトレーニングできます。トレーニングには、Sockeyeライブラリを使用しました。 Python3がすでにインストールされていることを前提としています。

pip install sockeye

ビデオカードの操作を担当 するMXNetを個別に調整する必要がある場合もあります。Colabには追加のライブラリをインストールする必要があります

pip install mxnet-cu100mkl

ニューラルネットワークについては、一般に、データをそのまま提供するだけで十分であり、彼らはそれを理解します。しかし、実際にはこれが常に当てはまるわけではありません。したがって、このケースでは、ケースを前処理する必要があります。まず、モデルがその「猫」を理解しやすいように、トークン化します。「猫」もほぼ同じです。たとえば、Python Tokenizerだけで十分です。

from nltk.tokenize import WordPunctTokenizer
def tokenize(src_filename, new_filename):
  with open(src_filename, encoding="utf-8") as src_file:
    with open(new_filename, "w", encoding="utf-8") as new_file:
      for line in src_file:
        new_file.write("%s" % ' '.join(WordPunctTokenizer().tokenize(line)))
        new_file.write("\n")

今回のケースでは、親ケースと子ケースの辞書を組み合わせたものが必要になるため、共通のファイルを作成します。

cp kk.parent.train.tok kkchv.all.train.tok
cat chv.child.train.tok >> kk.parent.train.tok
cp ru.parent.train.tok ru.all.train.tok
cat ru.child.train.tok >> ru.all.train.tok

子モデルのさらなるトレーニングが同じ辞書で行われるためです。

さて、小さいながらも重要な余談です。MPでは、文は単語の形でアトムに分割され、単語のシーケンスとして文を操作します。しかし、巨大な尾はコーパスで一度発生する単語で形成されるため、これは通常は十分ではありません。それらの確率モデルを構築することは困難です。これは、形態論が発達している言語（ケース、性別、数）に特に当てはまります。ロシア語もチュヴァシ語もそのような言語です。しかし、解決策があります。文を下位レベル、サブワードに分割できます。バイトペアエンコーディングを使用しました。

git clone https://github.com/rsennrich/subword-nmt.git

おおよそこのようなサブワードのシーケンスを取得します

接辞は単語とよく区別されていることがわかります。長い間@@でなく、@@は良いことです。
これを行うには、BPE辞書を準備します

python subword-nmt/subword_nmt/learn_joint_bpe_and_vocab.py --input kkchv.all.train.tok ru.all.train.tok -s 10000 -o bpe.codes --write-vocabulary bpe.vocab.kkchv bpe.vocab.ru

そして、それらをトークンに適用します。例えば：

python subword-nmt/subword_nmt/apply_bpe.py -c bpe.codes --vocabulary bpe.vocab.kkchv --vocabulary-threshold 50 < kkchv.all.train.tok > kkchv.all.train.bpe
!python subword-nmt/subword_nmt/apply_bpe.py -c bpe.codes --vocabulary bpe.vocab.ru --vocabulary-threshold 50 < ru.all.train.tok > ru.all.train.bpe

類推すると、すべてのファイルに対して行う必要があります：親モデルと子モデルのトレーニング、検証、テスト。

次に、神経モデルのトレーニングに直接移ります。最初に、一般的なモデル辞書を準備する必要があります。

python -m sockeye.prepare_data -s kk.all.train.bpe -t ru.all.train.bpe -o kkru_all_data

次に、親モデルをトレーニングします。簡単な例については、Sockeyeのページで詳しく説明しています。技術的には、プロセスは2つのステップで構成されています。以前に作成されたモデル辞書を使用してデータを準備する

python -m sockeye.prepare_data -s kk.parent.train.bpe -t ru.parent.train.bpe -o kkru_parent_data --source-vocab kkru_all_data/vocab.src.0.json --target-vocab kkru_all_data/vocab.trg.0.json

そして学習自体

python -m sockeye.train -d kkru_parent_data -vs kk.parent.dev.bpe -vt ru.parent.dev.bpe --encoder transformer --decoder transformer --transformer-model-size 512 --transformer-feed-forward-num-hidden 256 --transformer-dropout-prepost 0.1 --num-embed 512 --max-seq-len 100 --decode-and-evaluate 500 -o kkru_parent_model --num-layers 6 --disable-device-locking --batch-size 1024 --optimized-metric bleu --max-num-checkpoint-not-improved 10

コラボ施設でのトレーニングには約1日かかります。モデルのトレーニングが完了すると、次の助けを借りて翻訳できます

python -m sockeye.translate --input kk.parent.test.bpe -m kkru_parent_model --output ru.parent.test_kkru_parent.bpe

子モデルをトレーニングするには
python -m sockeye.prepare_data -s chv.child.train.bpe -t ru.child.train.bpe -o chvru_child_data  --source-vocab kkru_all_data/vocab.src.0.json --target-vocab kkru_all_data/vocab.trg.0.json

トレーニング開始コードは次のようになります

python -m sockeye.train -d chvru_child_data -vs chv.child.dev.bpe -vt ru.child.dev.bpe --encoder transformer --decoder transformer --transformer-model-size 512 --transformer-feed-forward-num-hidden 256 --transformer-dropout-prepost 0.1 --num-embed 512 --max-seq-len 100 --decode-and-evaluate 500 -o ruchv_150K_skv_dev19_model --num-layers 6 --disable-device-locking --batch-size 1024 --optimized-metric bleu --max-num-checkpoint-not-improved 10 --config kkru_parent_model/args.yaml --params kkru_parent_model/params.best

親モデルの構成と重みを開始点として使用する必要があることを示すために、パラメーターが追加されています。Sockeyeからの再トレーニングの例の詳細。子モデルの学習は約12時間で収束します。

要約すると、結果を比較します。通常の機械翻訳モデルは24.96 BLEU品質を生み出しましたが、知識伝達モデルは32.38 BLEUでした。この違いは、翻訳の例でも視覚的に確認できます。したがって、引き続きケースを組み立てながら、このモデルを使用します。
