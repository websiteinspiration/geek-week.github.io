Rhasspyは、オープンソースで完全にオフラインの音声ツールキットです。ロシア語の認識。クラウドへの漏れなし


手作りのスマートスピーカーなどのDIYデバイスのマイクアレイを比較した写真

Amazon Echoなどのシステムは、機密の会話をクラウドに送信して保存します（偶然に録音されたものでも）。場合によっては、録音はライブオペレーターによって聞こえます。これはプライバシーの喪失だけではありません。それは、24時間近くに立ち、耳を傾けて丁寧に録音し、役立つアシスタントのふりをしている「同志少佐」を自発的に入れさせるようなものです。

Google、Amazon、Yandexなどの企業から商用システムを購入する代わりに、Raspberry Pi 2-3 B / B +、パーソナルコンピューター、またはラップトップに基づいて、同様のオープンソースシステムを構築できます。

ラッシー-自律的に動作する安全な音声アシスタント。音声認識や音声コマンドにうまく対処しながら、リモートサービスには何も送信しません。

Rhasspyは、音声制御を追加するソフトウェアまたはハードウェアシステムに非常に簡単に統合できます。作者は、このツールはもともとHome Assistantプロジェクト用に作成されたが、現在は他のほとんどのホームオートメーションシステム（Has​​s.io、Node-RED、OpenHAB、Jeedom）と互換性があると説明しています。

Rhasspyは、MQTT、HTTP、またはWebsocketを介して外部サービスと連携するように最適化されています。文法構造が明確に定義された音声コマンド（ライトのオン/オフ、音楽の音量を大きく/小さくするなど）に最適化されています

。ロシア語を含む14の言語がサポートされています。

作業モデルは、ドキュメントに記載されています。特定のテンプレート言語による音声コマンドの認識に基づいていますこの領域に特別に適合。これらのコマンドは意図によって分類され、照明の色やコマンドが与えられる特定の器具の名前などのスロットやタグを含めることができます。

開始するには、意図（角括弧内）とそれらを呼び出す方法をリストします。テンプレートは次のようになります。

[LightState]
states = (on | off)
turn (<states>){state} [the] light

このパターンによれば、Rhasspyは、ホームオートメーションシステム、外部アプリケーション、またはハードウェアデバイス（Node-RED、Webソケット経由）で使用できるJSONコードを生成します。

{
    "text": "turn on the light",
    "intent": {
        "name": "LightState"
    },
    "slots": {
        "state": "on"
    }
}

音声認識は、pocketsphinx（ロシア語をサポートする軽量のオープンソースエンジン）によって直接実行されます。モバイルデバイスやRaspberry Piのようなシングルボードコンピューターに最適です。

サウンド処理がデバイスでオフラインになっています。サウンド自体は、Raspberry Piマイクロホンアレイ（ReSpeaker 4 MicアレイやReSpeaker 2 Mics pHATなど）から、またはネットワーク上のオーディオストリームから発生します。

Rhasspyは、音声認識エンジンをホームオートメーションシステムまたは音声制御を必要とするその他のシステムにリンクするための非常に便利なツールです。原則として、モバイルアプリケーションなど、どこでも使用できます。または、掃除機やバーテンダーなどのある種の家庭用ロボットで。

ロボットが以前と同じアクションをすべて実行すると便利ですが、今は音声コマンドで実行します。

Rhasspyの作成者は、voice2jsonプロジェクトの作成者でもあります。これは、ほぼ同じタスクのコンソールプログラムであり、人間の音声をコンピュータコマンドのリストに簡単に変換する（またはその逆）ことができます。

将来は音声インターフェースにあるようです。この場合、サウンドストリームの処理がローカルで行われ、インターネットへのアクセスを必要としないことが非常に重要です。
