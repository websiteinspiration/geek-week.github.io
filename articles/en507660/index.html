Monitoring through UptimeRobot

Any site needs monitoring, there is no resource about which the owner would prefer not to know about downtime. It's funny that many developers do not monitor their pet projects, and freelancers almost never bother with accessibility checks, although this is usually no more difficult than connecting some analytics and can save a lot of nerves.



There are more than a dozen services for monitoring sites, most of them are fully paid or greatly reduced in the free version. Apart from them is UptimeRobot , which provides huge functionality for free and allows you to flexibly configure a paid subscription to your needs.



In most cases, UptimeRobot sends HTTP requests to the specified address and determines the availability by the response headers (2xx - everything is ok, 4xx / 5xx - it fires repeated requests from other points for 30 seconds, if the answer is negative again, downtime is counted).



Is free



  • 50 monitors with a refresh rate of 5 minutes
  • HTTP (S) requests to any addresses
  • Search for a keyword on a page
  • Ping (ICMP requests)
  • Port check
  • Alerts by mail, webhooks, messages on Twitter, Telegram, Slack and other services
  • Response Time Statistics
  • Detailed logs for two months
  • The ability to publish statistics on a public page ( example )
  • Open API


Paid



  • Up to 10,000 monitors with minute refresh interval
  • Track SSL certificate expiration and monitor HTTPS issues
  • A variety of logic alerts (with the installation of intervals and more)
  • Customization of HTTP requests (including headers and statuses)
  • Heartbeat monitoring (server monitoring for NAT)
  • SMS / call alerts (up to 900 per month)
  • Logs available in two years


As you can see, there are even more free nishtyaks at the service than purchased ones, and most likely this is what brought UptimeRobot popularity and recognition. Although, after several redesigns, he is still blamed for the terrible interface.

API



UptimeRobot provides a simple API in XML / JSON / JSON-P that completely duplicates the functionality of the web version. A detailed description of the methods and parameters is on the API page , in short - you can read statistics and logs, create / delete / edit / reset monitors and manage users.



There is an interesting guide Automating uptime monitoring on Kubernetes with UptimeRobot , a client on Go , Python, and a bunch of interpretations for other languages, panels and custom status pages .



Heartbeat



To monitor servers that are not directly accessible, heartbeat monitors are available in the paid version. When creating this, a unique link is generated on which the UptimeRobot service listens for periodic incoming requests from a remote server. If requests ceased to arrive, downtime is counted.



What can be done right now



One free account relies on 50 monitors, which is usually enough to cover personal sites and projects. If you have not yet connected free monitoring everywhere, then this is the time to do it. In 5 minutes and without any interference with the code. And if you need a deeper integration of monitoring into the project, study the API. Just do not forget to first look for ready-made solutions for your stack, probably in bulk.
