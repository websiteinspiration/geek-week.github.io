ページネーション中に負荷を表示する別の方法



モバイルアプリケーションでのページごとの読み込みの操作は、かなり単純なトピックであり、落とし穴はありません。

開発者はよく似たタスクに遭遇するため、同じことを何度も繰り返すのは退屈で面倒です。

しかし、最も一般的なタスクでさえ、別のより興味深い方法で解決して、新しい知識を得て、ただ脳を伸ばすことができます。

ページネーション


つまり、ページネーションを実装するためのすべての作業は、以下にリストされた項目で構成されています。

  1. リサイクル業者ビューにリスナーを追加します。
  2. 一次データをダウンロードします。
  3. ユーザーがリストをスクロールするときにコールバックをキャッチします。
  4. すべてのアイテムの後にダウンロードをリストに表示します。
  5. 新しいアイテムのリクエストを送信します。
  6. データを再度表示します。

今回のケースでは、ページングメカニズムが実装されましたが、ユーザーがリストをスクロールしている間、負荷の表示はありませんでした。

わかりました、1つのアプリケーションでこれを1回行うことは問題ではありません。この機能が必要な画面がいくつかある場合は、追加のタイプのビューで機能する基本的なアダプターを作成できます。

しかし、タスクがやや複雑な場合はどうでしょうか。

基本的な機能を備えたコアモジュールが1つある新しいプロジェクトがいくつかあるとします。

プロジェクトは初年度からほど遠いため、結果として、このタスクに関連する多くのレガシーコード、つまり、異なる基本クラスを持つ多くのアダプターが蓄積されています。

どうすれば問題を解決できますか？


ダウンロードの進行状況を表示できる新しいベースアダプターを作成し、すべての古いアダプター+アダプターを使用する画面を書き換えます。各プレゼンターまたはビューで、各画面の負荷の表示状態を制御する必要があるためです。

これにはかなりの時間がかかります。また、大規模なリファクタリングは、特にプロジェクトのテストカバレッジがゼロになる傾向がある場合に、多くの新しいエラーを引き起こす可能性があります。さらに、QA部門の負荷が高く、すべての画面の回帰テストが行​​われ、複数のアプリケーションでページ分割が行われます。

その機能に基本的なソリューションを統合するために開発クライアントから最小限の時間を必要とするようなコードを書くことは素晴らしいことです。そのため、ロードの進行状況を表示する制御ロジックを実装する必要はありません-表示と非表示。

この時点で、ItemDecorationクラスの使用を検討しました。

このクラスを使用して、

  • 負荷を表示するタイミングと非表示にするタイミングの決定
  • 図面ダウンロードの進行状況

残念ながら、インターネット上で単一の既製のソリューションは見つかりませんでした-誰もそれを実装しようとしなかったか、単に実装後に経験を共有しませんでした。

ItemDecorationを使用してページ分割中に読み込み表示を実装するには何が必要ですか？


  • 負荷をレンダリングするためにインデントします。
  • 進捗状況を示す必要がある場合のItemDecorationの実装を理解します。
  • ダウンロードを描画します。

インデント


SpacingItemDecorationを作成した開発者は、少なくとも一度はインデントする方法を知っています。ItemDecoration getItemOffsetsメソッドは、これに役立ちます。

    override fun getItemOffsets(outRect: Rect, view: View,
                                recyclerView: RecyclerView, 
                                state: RecyclerView.State) {

        super.getItemOffsets(outRect, view, parent, state)
    }

リストのどの要素についても、どちら側にもインデントを追加できます。具体的には、このケースでは、ユーザーがリストを最後までロールバックし、最後の要素のインデントで、表示されるダウンロードの進行状況+進行状況自体のインデントの値に等しいことを理解する必要があります。

リストが一番下までスクロールされていることを理解するにはどうすればよいですか？

以下のコードはこれで私たちを助けます：

 private fun isLastItem(recyclerView: RecyclerView, view: View): Boolean {
        val lastItemPos = recyclerView.getChildAdapterPosition(view)
        return lastItemPos == recyclerView.adapter!!.itemCount - 1
    }

全体として、インデントを定義および実装するためのコードがあります。

    override fun getItemOffsets(outRect: Rect, view: View,
                                recyclerView: RecyclerView, 
                                state: RecyclerView.State) {

        super.getItemOffsets(outRect, view, recyclerView, state)

        when (isLastItem(recyclerView, view)) {
            true -> outRect.set(Rect(0, 0, 0, 120))
            else -> outRect.set(Rect(0, 0, 0, 0))
        }
    }

    private fun isLastItem(recyclerView: RecyclerView, view: View): Boolean {
        val lastItemPos = recyclerView.getChildAdapterPosition(view)
        return lastItemPos == recyclerView.adapter!!.itemCount - 1
    }

ダウンロードの進行状況を表示する時間を決定し、進行図を呼び出します


ItemDecoration onDrawOverメソッドは、これに役立ちます。

    override fun onDrawOver(canvas: Canvas, 
                            recyclerView: RecyclerView,
                            state: RecyclerView.State) {

        super.onDrawOver(canvas, recyclerView, state)
    }

onDrawOverメソッドは、onDrawメソッドとレンダリングの順序のみが異なります。OnDrawOverデコレーションは、リストアイテム自体が描画された後にのみレンダリングされます。

onDrawOverメソッドでは、進行状況をどの時点で描画する必要があるかを理解し、どの時点でそれを削除して、リストアイテムを更新して、不要なインデントを非表示にする必要があります。

上記のアクションを実装するためのコード：

    override fun onDrawOver(canvas: Canvas, 
                            recyclerView: RecyclerView,
                            state: RecyclerView.State) {

        super.onDrawOver(canvas, recyclerView, state)

        when (showLoading(recyclerView)) {
            true -> {
                PaginationProgressDrawer.drawSpinner(recyclerView, canvas)
                isProgressVisible = true
            }
            else -> {
                if (isProgressVisible) {
                    isProgressVisible = false
                    recyclerView.invalidateItemDecorations()
                }
            }
        }
    }

    private fun showLoading(recyclerView: RecyclerView): Boolean {

        val manager = recyclerView.layoutManager as LinearLayoutManager
        val lastVisibleItemPos = manager.findLastCompletelyVisibleItemPosition()

        return lastVisibleItemPos != -1 && 
                        lastVisibleItemPos >= recyclerView.adapter!!.itemCount - 1
    }
