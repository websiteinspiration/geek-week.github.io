最初の日。Node.jsの紹介


バックエンド開発に来ました。このIT企業はNode.jsプラットフォームを使用していますが、これは私がまったく知りませんでした。私は少し前に進んで、JavaScriptで何も開発したことがないことを読者に伝えるのを忘れていました（コピーコードを含むいくつかのスクリプトを除く）。 CRUDをJava、Python、Clojureで開発して以来、作業のアルゴリズムとWebアプリケーションのアーキテクチャ全体を理解しましたが、これでは不十分でした。したがって、私がNode.jsの研究に完全に専念した初日、このスクリーンキャストは本当に役に立ちました。Express

Webフレームワークについて学びながら、npmパッケージマネージャー、package.jsonやtsconfig.jsonなどのファイルだけでなく、私の頭の中は情報量を回っていました。別の教訓は、すべての素材を同時に習得することは不可能に近い作業であるということです。一日の終わりまでに、私はまだ環境を構成することができ、エクスプレスWebサーバーを実行することができました！しかし、彼は完全な誤解の感覚で家に帰ったので、喜ぶには早すぎました。私はJSの広大な世界で溺れていたという気持ちが1分間出なかったため、再起動が必要でした。

彼は以前、Java、C ++、C＃などのオブジェクト指向プログラミング言語の大部分を書いていました。TypeScriptの可能性を実感し、安心しました。このプログラミング言語は、この複雑な環境の生活に文字通り息を吹き込みました。その日の終わりに向けて、環境を完全にセットアップし、サーバーを起動し（既にTypeScriptで）、必要なライブラリーを接続しました。これについては、以下で説明します。結論：APIを開発する準備ができています。私たちは開発に直接渡します...

API開発


作業原理の説明と、REST APIとは何かに関するその他の説明については、フォーラムにさまざまなプログラミング言語での例や開発に関する記事がたくさんあるので、ここでは残しておきます。


タスクは次のとおりです

。RESTAPIを使用してサービスを作成します。ベアラートークンによる承認（/情報、/待ち時間、/ログアウト）。任意のドメインからのアクセス用にCORSを構成しました。DB-MongoDB。呼び出しごとにトークンを作成します。

APIの説明：

  1. /サインイン[POST]-IDとパスワードによるベアラートークンのリクエスト// JSONでデータを受信
  2. /サインアップ[POST]-新しいユーザーの登録：// jsonでデータを受信します
  3. / info [GET]-ユーザーIDとIDタイプを返します。認証時にベアラによって発行されたトークンが必要です
  4. /レイテンシ[GET]-遅延を返し（ping）、認証でベアラによって発行されたトークンが必要です
  5. / logout [GET]-パラメータall：true-すべてのユーザーベアラートークンを削除するか、false-現在のベアラートークンのみを削除します

私はすぐに、このタスクはWebアプリケーション開発者にとって信じられないほど単純に見えます。しかし、タスクはプログラミング言語で実装する必要があります。この言語については、3日前にはまったく何も知りませんでした。私にとっても、紙の上では完全に透明に見え、Pythonでは実装に少し時間がかかりましたが、そのようなオプションはありませんでした。開発スタックは問題を予告した。

ユーザー認証


承認には、authorizationChecker（サーバーを作成するときのパラメーターの1つ、上記を参照）を使用します。便宜上、別のファイルに配置します。

import { Action, UnauthorizedError } from 'routing-controllers';
import { getMongoRepository } from 'typeorm';

import { User } from '../models/User';

export async function authorizationChecker(action: Action): Promise<boolean> {

    let token: string;
    if (action.request.headers.authorization) {
        //     
        token = action.request.headers.authorization.split(" ", 2);
        const repository = getMongoRepository(User);
        const allUsers = await repository.find();

        for (let i = 0; i < allUsers.length; i++) {
            if (allUsers[i].token.accessToken.toString() === token[1]) {
                return true;
            }
        }
    }
    else {
        throw new UnauthorizedError('This user has not token.');
    }
    return false;
}

認証後、各ユーザーは独自のトークンを持つため、応答のヘッダーから必要なトークンを取得できます。これは、Bearer 046a5f60-c55e-11e9-af71-c75526de439eのようになります。これで、このトークンが存在するかどうかを確認できます。その後、関数は認証情報を返します。true-ユーザーは認証され、false-ユーザーは認証されません。アプリケーションでは、コントローラーで非常に便利なデコレーター@Authorized（）を使用できます。この時点で、authorizationChecker関数が呼び出され、応答が返されます。

多くの人はMVCパターンで何が必要でどのようにコントローラーが使用されるかを説明する必要はありませんが、それでも2つの言葉を言います。要するに、コントローラーはユーザーとユーザー間のデータをリダイレクトするアプリケーション間のリンクです。ロジックは上記で完全に説明されており、そのメソッドは、URIとIPサーバーで構成されるルートに従って呼び出されます（例：localhost：3000 / signin）。前にコントローラーのデコレーターについて述べました：取得する、 役職、@ Authorized、これらの中で最も重要なのは@JsonControllerです。このフレームワークのもう1つの非常に重要な機能は、JSONを送受信する場合は、代わりにこのデコレーターを使用することです。コントローラ。 

import * as express from 'express';
import {
    Authorized, Body, Get, Header, JsonController, NotFoundError, Post, QueryParam, Req,
    UnauthorizedError
} from 'routing-controllers';

import { IPingResult } from '@network-utils/tcp-ping';

import { User } from '../models/User';
import { UserService } from '../services/UserService';

//    JSON
@JsonController()
export class UserController {
   userService: UserService
   // 
   constructor() {
      this.userService = new UserService();
   }

   // 
   @Post('/signin')
   async login(@Body() user: User): Promise<string> {
      const responseSignin = await this.userService.userSignin(user);
      if (responseSignin !== process.env.USER_SERVICE_RESPONSE) {
         return responseSignin;
      }
      else {
         throw new NotFoundError(process.env.POST_SIGNIN_MASSAGE);
      }
   }

   // 
   @Post('/signup')
   async registrateUser(@Body() newUser: User): Promise<string> {
      const responseSignup = await this.userService.userSignup(newUser);
      if (responseSignup !== process.env.USER_SERVICE_RESPONSE) {
         return responseSignup;
      }
      else {
         throw new UnauthorizedError(process.env.POST_SIGNUP_MASSAGE);
      }
   }

   //  
   @Get('/info')
   @Authorized()
   async getId(@Req() req: express.Request): Promise<User> {
      return this.userService.getUserInfo(req);
   }

   //  
   @Authorized()
   @Get('/latency')
   getPing(): Promise<IPingResult> {
      return this.userService.getLatency();
   }

   @Get('/logout')
   async deleteToken(@QueryParam("all") all: boolean, @Req() req: express.Request): Promise<void> {
      this.userService.userLogout(all, req);
   }
}
