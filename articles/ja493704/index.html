TypeScriptを記述せずにJavaScriptでTypeScriptを使用する

私はTypeScriptの大ファンです。可能な限り、プロジェクトでこの言語を使用するようにしています。 TypeScriptは、JavaScriptおよびTypeScriptコードでのエラーレポートと型チェックを開発者に提供します。さらに、TypeScriptを作成する人は誰でも、コード補完に便利な機能を使用でき、迅速かつ確実にリファクタリングを実行できます。 TypeScriptは、コードのエラーに対する防御の最前線です。この言語のもう1つの長所は、プログラムを作成するときに最新のJavaScript機能を使用できることです。同時に、TypeScriptコードをJavaScriptに変換することで得られるものは、すべてのブラウザーでサポートされます。とてもいいです。



確かに、すべてのフロントエンドプロジェクトにTypeScriptが含まれているわけではありません。TypeScriptの機能を使用できても、プロジェクト全体（およびチーム全体）をそれに変換せずに、プロジェクトアセンブリパイプラインに新しいツールを追加するとどうなりますか？これは、VS CodeとJSDocのおかげで可能です。

作業環境のセットアップ


▍オプション番号1。VSコード設定（グローバル検証）


プレーンJavaScript（JS）で記述されたプロジェクトでTypeScript（TS）を使用する最初の方法は、TSを使用してすべてのJSファイルを検証することです。これは、1つのグローバルパラメータVSコードを含めることによって行われます。このパラメーターは、ユーザーまたはワークスペースの設定で有効にできます。

"javascript.implicitProjectConfig.checkJs": true

グラフィカルインターフェイスを使用したい場合は、以下に示すようにこれを有効にできます。


TypeScript JS検証の有効化

▍オプション番号2。jsconfig.jsonファイルの使用（グローバル検証）


TSを使用してグローバルJSコード検証を有効にする別のオプションは、ファイルを使用することですjsconfig.json。そのようなファイルが存在する場合、そのファイルで指定された設定は、VS Codeで指定された設定を上書きします。

{
  "compilerOptions": {
    "checkJs": true
  }
}

▍オプション番号3。個々のファイルの検証を有効にする


TypeScriptを使用してJSコードを制御する3番目の方法は、個々のファイルレベルで検証を有効にすることです。これは、対応するコメントをファイルの先頭に追加することで構成されています。

// @ts-check
let itsAsEasyAs = 'abc';
itsAsEasyAs = 123; // Error: Type '123' is not assignable to type 'string'

同じアイデアを使用して、単一のJSファイルのTSチェックをオフにすることができます。これは、上記の方法を使用してTSチェックがグローバルに有効化されている場合に行われます。特別なコメントもここで使用されます：

// @ts-nocheck
let easy = 'abc';
easy = 123; //

TypeScriptでファイルの一部のみを無視する場合は、次のようにします。

let easy = 'abc';
// @ts-ignore
easy = 123; //

JSDocを使用したコード入力


ファイルレベルのTSチェックを有効にする方法について説明しました。これにより、基本的な型チェック機能が提供されます。JSDoc形式のコメントを使用して型を記述することにより、拡張できます。

functions機能の類型化


JSDocを使用して、関数が入力として受け入れるものの説明を使用してコードの入力を開始できます。

/**
 * @param {number} shippingVal
 */
updateShipping(shippingVal) {
    ...
}

その後、エディターはタイプ別にヒントを与えることができます。


関数が受け入れる値のタイプに関するヒント

このメソッドは単純なタイプに適していますが、開発者が自分のタイプを記述する必要がある場合はどうでしょうか。これはタグを使用して行うことができます@typedef。タイプの説明をファイルの先頭に置くことをお勧めします。これにより、作業中にそのような記述の検出が容易になります。

/**
* @typedef {Object} CreditNoteTaxResponseViewModel
* @property {number} feeAmount
* @property {number} inclGst
* @property {number} subTotal
* @property {number} total
*
* @typedef {Object} ApiResponse
* @property {string} status
* @property {string} message
* @property {CreditNoteTaxResponseViewModel} response
*/

このような説明は、必要に応じて使用できます。

/**
                * @param {CreditNoteTaxRequestViewModel} req
                * @returns {Promise<ApiResponse>}
                */
                createCreditNoteTaxApiCall(req) {
        ...
                }

この手法のもう1つの使用例は、型宣言を特殊ファイルに移動することです。そのようなファイルを呼び出すことができるとしましょうmain.d.ts。

export interface ICreditNoteTaxRequestViewModel{
    orderID: number;
    shippingCredit: number;
    lines: IICreditNoteTaxLineViewModel[]
}

export interface ICreditNoteTaxLineViewModel{
    originalOrderLineID:number;
    creditQuantity: number;
}

export interface ICreditNoteTaxResponseViewModel{
    feeAmount: number;
    inclGst: number;
    subTotal: number;
    total: number;
}

export interface IApiResponse{
    status: string;
    status: message;
    response: ICreditNoteTaxResponseViewModel;
}

これらのタイプはJavaScriptで使用できます。

/**
   * @param {import("./main").ICreditNoteTaxRequestViewModel} req
   * @returns {Promise<import("./main").IApiResponse>}
   */
  function createCreditNoteTaxApiCall(req) {
    ///  
    return;
  }

libraries図書館の類型化


VS Codeには、サードパーティライブラリのタイプを自動的に取得するシステムがあります。対応する手順は、ファイルに記述されているすべてのパッケージに適用されますpackage.json。ただし、これを明示的に設定したい場合は、次の場所で適切な設定を行うことができますjsconfig.json。

}

/**
 * @param {JQuery<HTMLElement>} $itemRow
 */
initRow($itemRow) {
    ...
}

TypeScriptへの切り替え


TypeScript JavaScriptプロジェクトに変換する場合は、TypeScriptが使用されている一部の部分で、プロジェクトの名前を変更しjsconfig.jsonてtsconfig.json、パラメーターを含めることができますallowJs。

{
  "compilerOptions": {
    "allowJs": true,
    "checkJs": true
  }
}

その後、*.js-filesの名前を*.ts-filesに変更し、これらのファイルのコードを入力できます。プロジェクトをTypeScriptに変換するプロセスは、徐々に発生する可能性があります。

概要


この資料を読んだ後、JavaScriptプロジェクトでTypeScript機能を簡単に利用できることに気付くでしょう。これを行うには、それに応じてVSコードを構成します。ここで説明するアプローチを使用すると、プロジェクトのアセンブリプロセスに変更を加えたり、このプロセスに違反する危険を冒したり、開発チームに強制的に新しい言語に切り替えさせたりすることがなくなります。

TypeScript機能の一部のみを使用するJSプロジェクトが完全にTSに転送されると決定された場合も、これは簡単に実行できます。さらに、そのような移行は段階的に実行することができます。この記事で使用したコード例を見つけることができるGitHubリポジトリ
