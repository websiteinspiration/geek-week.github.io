Azure Analytics：瞬く間に明快さ


2020年が私たちに教えてくれたのは、世界が瞬時に変化するということです。数か月にわたって、私たちは世界中の各業界の仕事に失敗がどのように現れるかを観察しました。工場は凍結され、ホテルは空で、私たちをつなぐ交通動脈は静かです。これらの前例のない時代に、企業や組織は突然の変化の状況で働く必要があり、これは彼らが新しいレベルの柔軟性に到達することを必要とします。

この柔軟性を実現するには、絶えず変化するデータに基づいて関連する分析を継続的に受け取る必要があります。私たちは、ユーザーがAzureを介して安全かつ効率的に分析を行えるようにすることを約束します。昨年11月にAzure Synapse Analyticsを導入したとき、実際にエンタープライズデータストレージとビッグデータ分析の間の障壁を取り除きました。これにより、データ専門家は互いに協力し、独自の分析ソリューションを作成して簡単に管理できるようになりました。 Azure Synapse Analyticsは業界で画期的な出来事であり、昨年11月のデビュー以来、サービスに対する顧客の高い関心を観察できたことを非常に嬉しく思います。

しかし、それだけではありませんでした。カットの下の詳細。

別の長年の障壁が分析データから運用データを分離します。ハイブリッドトランザクション分析処理（HTAP）の負担を維持することは、常に困難でコストがかかります。また、トランザクションデータと分析データの処理を妥協するように顧客に強いてきました。分析とトランザクションの両方を1つのシステムの一部としてサポートするため、または個別のシステムをサポートするために、高価なリソース、特にメモリを過剰に増やす必要がありました。ほとんどのクライアントは後者を選択しました。その理由は、分析システムとオペレーティングシステム間のリンクを確立するために複雑なETLプロセスを処理する必要があり、データから分析結果を得るのに遅延が生じたためです。

現時点ですぐに、現在の状況に関する貴重なデータを受け取ることができる、シンプルで手頃な価格のクラウドベースのHTAPソリューションはまだありませんでした。まだ。

Azure Synapse Linkの紹介


本日、クラウドベースのHTAPソリューションであるAzure Synapse Linkのパブリックプレビューを発表します。このツールは、AzureオペレーティングデータベースとAzure Synapse Analyticsの間の障壁を取り除きます。Azure Synapse Linkを使用すると、情報を移動したり、運用データ処理システムを読み込んだりすることなく、ワンクリックで運用ベースからリアルタイムでトランザクションデータを受信できます。Azure Synapse LinkはAzure Cosmos DBで既に利用可能であり、将来的には、Azure SQL、Azure Database for PostgreSQL、Azure Database for MySQLなどの運用データベースを操作するための他のサービスにも登場する予定です。

使い方？Azure Synapse Linkの基礎は、クラウドアーキテクチャです。これを有効にするには、クライアントはお気に入りのAzureデータベースサービスのボタンをクリックするだけです。これにより、Azure Synapse Analyticsのデータとの直接接続が作成されます。その後、Azure Synapse Analyticsでは、カバーリングインデックスと同様に最適化された列構造としてトランザクションデータが自動的に利用可能になります。複雑なETLプロセスや追加のコンピューティング能力は必要ありません。お客様は、Azure Synapse Analyticsを使用してリアルタイムでデータをすぐにコスト効率よく分析し始めることができます。
