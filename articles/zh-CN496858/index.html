Linux exec系统调用在将来的内核中应该不太容易出现死锁





在Linux上对exec代码的不间断工作将使它在将来的内核版本中不易出现死锁。内核中当前的exec功能“极易出现死锁”，但是Eric Biederman和其他人已经努力清除此代码并将其置于更好的状态，以避免潜在的死锁。Linux 5.7内核的编辑是exec改进的第一部分，它使识别更复杂的情况变得更加容易，并且希望对于Linux 5.8，解决exec死锁的代码可能已经准备好。莱纳斯·托瓦尔兹（Linus Torvalds）接受了5.7的修改，但并没有对此表示满意。

Sandboxie已被释放并释放给社区





Sophos已宣布开放Sandboxie程序，该程序旨在启用基于Windows的应用程序的沙盒执行。Sandboxie允许您在与系统其余部分隔离的沙箱环境中运行不可靠的应用程序，仅限于不允许访问其他应用程序数据的虚拟磁盘。该项目的开发已移交给社区，该社区将协调Sandboxie的进一步开发和基础设施的维护（不是最小化项目，Sophos决定将开发移交给社区，他们计划今年秋天关闭论坛和旧项目站点）。该代码已在GPLv3许可下打开。

Windows 10计划在File Explorer中启用Linux文件集成





很快，您将能够直接在Windows资源管理器中访问Linux文件。早些时候，微软宣布计划在Windows 10中发布完整的Linux内核，现在该公司计划在内置的资源管理器中完全集成对Linux文件的访问。The Verge报告说，新的Linux图标将在资源管理器的左侧导航栏中提供，从而为Windows 10中安装的所有发行版提供对根文件系统的访问。我不知道怎么回事，但这让我感到不止让我快乐。GNU / Linux以前是隔离的，由于Windows暴露于病毒，因此您可以安全地在同一台计算机上运行Windows，而不必担心其他操作系统中的文件，但是现在您不必担心。

微软提出了Linux内核模块来验证系统完整性





Microsoft的开发人员介绍了IPE（完整性策略执行）机制，该机制被实现为Linux内核的LSM模块（Linux安全模块）。该模块允许您为整个系统定义通用的完整性策略，指示哪些操作有效以及应以哪种方式验证组件的真实性。使用IPE，您可以指定允许运行哪些可执行文件，并确保这些文件与受信任源提供的版本相同。该代码在MIT许可下是开放的。

Docker Compose正在准备开发适当的标准





Docker Compose是Docker开发的用于定义多容器应用程序的系统，它计划发展成为开放标准。名为Compose规范，应使Compose应用程序能够与Kubernetes和Amazon Elastic CS等其他多容器系统一起使用。开放标准草案现已发布，该公司正在寻找想要参与其支持和创建相关工具的人员。
