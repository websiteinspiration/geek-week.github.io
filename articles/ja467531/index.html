反応状態機械

5年前、RxJs、ビーコン、... etc.jsで、適切な解決策を探し、そしてflyd.jsで停止してからのみの3つの機能を使用して4-6のプロジェクトを書いた後、ストリーム、上で、ライブラリーを組み合わせました。残りは仕事がなかったので、バンドルの軽さを求めました。そのためalakが登場し、0.3より前の最初のバージョンはわずか20〜30ラインで、完全にフライングに置き換えられました。

1年後、バージョン0.4はすでにパターンマッチングとミックス（フライドからの結合またはvueからの計算のアナログ）を使用して72行で登場しました。3年と数十のプロジェクトを経て、良さを感じ、経験を共有し、1.0をリリースしました。



ファンタジーランド


教科書から古典的なOOPを書いている間、人のFRPに鋭敏に没頭すると、深部潜水艦の外の深部に鋭い没頭感のように頭を吹き飛ばすことができます。変数ではなく関数の考え方に慣れるまでに数週間かかりました。3か月後、私はすでにアプリケーションの反応グラフを完全に賞賛していました。プログラムは、木の静脈または根/枝として見ることができます。サンスクリットのような古代言語への単純な興味とエルフのケニアへの愛情が私を助けてくれたと思います。ダイバーの基本的なスキルは、鼻をつまみながら耳を吹き消すことです。成功した古典的なOOPプログラマーは、没入型の知識スキルを吹き飛ばす必要があります。経験豊富な機能主義者を助けるものは何もありませんが、既存のフォームやフレアの説明を克服するのに役立つかもしれません。

不明な名前


哲学では、モナドは宇宙の基本的な粒子と呼ばれます-神の粒子、かなり低レベルの概念。ファンタジーランドの仕様、およびFRPの一般的な理解では、モナドは内部にファンクタを含むより高いレベルの何かです。ではウィキペディア一般的な文脈とOCamlの中ファンクタの二つの定義があります。 alakのコンテキストでは、基本的な粒子ファンクターはデータを含む関数であり、変更されると、関連する関数/ファンクターが更新されます。すべてがExcelテーブルと同じです。



bacon.jsでは、これは「アトミックアップデート」と呼ばれていますが、 RxJはそうではありません。

alakでは、コードは次のようになります。

import A from 'alak'
 //    -    alak 
const userId = A.flow();
 //         
const followers = A.flow();
const profile = A.flow();
const tweets = A.flow();
//  userId -> profile  getProfile
const getProfile = id => api.getProfile(id).then(profile);
userId.up(getProfile);
//  
userId.up(id => {
  api.getFollowers(id).then(followers)
  api.getTweets(id).then(tweets);
});

codesandboxで実行userId
が変更されると、フォロワー、プロファイル、ツイートに反応的な変更が発生します。

angular.jsでさえ、これは複雑さと複雑さの状態を美しく解決するのに十分でした。私にとってangular2の交差点は、疑わしい反応性を持つRxJSの強制でした。 ReduXを見ると、PureMVCの実装に2歩戻った感じがしました。
