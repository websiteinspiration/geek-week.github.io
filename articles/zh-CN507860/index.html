没有互联网的时间同步



除了tcp / ip，还有许多同步时间的方法。其中一些只需要一部普通电话，而另一些则需要昂贵，稀有和敏感的电子设备。时间同步系统的广泛基础结构包括天文台，政府机构，广播电台，卫星星座等等。



今天，我将告诉您在没有Internet的情况下如何进行时间同步以及如何用自己的双手制作“卫星” NTP服务器。



短波广播



在美利坚合众国，NIST将准确时间和频率分别以2.5、5、10、15和20 MHz的频率从科罗拉多州柯林斯堡的WWVH站发射到无线电波，并以2.5、5、10和15 MHz的频率从考艾岛的WWVH站发射，夏威夷州。时间码以1 bps的速度每60秒间隔发送一次。在100 Hz的子载波上使用脉冲宽度调制。



加拿大国家研究委员会（NRC）从安大略省渥太华的CHU站以3.33、7.85和14.67 MHz的频率分发时间和频率信息。

短波站的信号传播通常是通过电离层上层的反射而发生的。可以在长距离上获得信号传输，但是时间精度约为1毫秒。



当前的NTPv4标准包括WWV，WWVH和CHU的音频驱动程序。

标准时间低频电台



与短波电台一样，时间码以60秒的间隔以1 bps的速度传输。两种标准的数据传输格式也相似。信号通过电离层的较低层传播，该较低层相对稳定并且在海拔高度上具有可预测的昼夜变化。由于物理环境具有这种可预测性，因此精度提高到50μs。

对地静止业务环境观测卫星



在美国，NIST还从对地静止操作环境卫星（GOES）发送大约468 MHz的准确时间和频率数据。时间码与用于轮询远程传感器的消息交替显示。它由60个BCD半字节组成，每30秒间隔发送一次。时间码信息类似于地面服务。



全球定位系统



美国国防部使用GPS在陆地，海上和空中进行精确导航。该系统通过将卫星按12小时的轨道（倾斜55°）进行分组来提供24小时的全球覆盖范围。



最初的24颗卫星星座已扩展为异构配置的31颗卫星，因此在世界上大多数地方，至少有6颗卫星一直在可见，并且有8颗或更多的卫星在可见。



GPS等服务正在由其他国家/地区运营或计划中。从2010年9月2日算起，俄罗斯GLONASS已经运行了十二年，当时卫星总数达到26颗-该星座已完全部署以完全覆盖地球。

欧盟的卫星导航系统称为伽利略（Galileo）。预计伽利略卫星将在2014-2016年开始运行，届时所有30颗计划中的卫星都将进入轨道，但是到了2018年，伽利略卫星星座没有达到所需的卫星数目。



还有中文“北斗”，意为“鲸鱼”。作为区域定位系统，这16颗卫星群已于2012年12月27日投入商业运营。计划到2020年该系统达到满负荷运行。就在今天，在哈布雷杂志上发表了一篇有关该系统卫星成功发射的文章。



SRNS确定坐标的数学



智能手机上的GPS / Glonass导航器如何使用无线电导航通信系统（SRNS）如此精确地确定位置？为了理解计算原理，需要回顾初中或高中物理系高年级的立体学和代数。



每颗卫星都会告诉接收机准确的时间。卫星具有原子钟，因此可以信任。知道光速后，确定卫星所在表面的球体半径就不难了。与地球接触的同一球体形成一个圆圈，GPS / glonass接收器位于该圆圈上。



当信号来自两个卫星时，我们已经具有地球和两个球体的交点，这仅在圆上给出了两个点。理想情况下，第三颗卫星的球面应落入这两个点之一，最终确定接收机的坐标。



原则上，即使是从两颗卫星上，也可以通过间接符号来理解两点中的哪一点更接近真相，现代导航软件算法可以应对这一任务。那为什么我们需要第四颗卫星？





使用卫星星座定位。



容易注意到，在这张理想化的图片中，有许多细微差别取决于计算的准确性。接收器上的时间可能是最明显的错误来源。为了使所有功能正常工作，接收机的GPS /格洛纳斯时间必须与卫星时间同步。没有这个，误差将是10万公里。



根据速度，时间和距离的公式S = v * t，我们得到了传输SRNS信号的基本公式。到卫星的距离等于光速和卫星与接收器之间的时间差的乘积。







S=∗(t−t)









这主要是由于这样的事实，即使在所有同步之后，我们也以足够的准确度知道了接收机tpr的时间。在真实时间和tpr之间始终会有Δt，因此，计算误差变得不可接受。这就是为什么需要第四颗卫星的原因。



为了更清晰地证明需要四颗卫星的数学依据，我们构建了一个方程组。







Ri=(x−Xi)2+(y−Yi)2+(z−Zi)2+Δt









为了确定四个未知数x，y，z和Δt，观察数必须等于或大于未知数。这是必要条件，但不是充分条件。如果正规方程的矩阵退化，则方程组将没有解。



不要忘记相对论和相对论的特殊理论，即相对于地面原子钟的时间膨胀对卫星原子钟的影响。







t=t1−(vc)2









如果我们假设卫星以14000 km / h的速度在轨道上运动，那么我们得到的时间膨胀约为7μs（微秒）。另一方面，广义相对论的相对论效应起作用。



事实是，轨道上的卫星与地球的距离很大，由于地球的质量，时空连续体的曲率小于地球表面的曲率。根据广义相对论，靠近大质量物体的时钟似乎比远离大质量物体的时钟慢。







t=t∗1−2GMrc2







  • G是引力常数；
  • M是物体的质量，在这种情况下为地球；
  • r是地球中心到卫星的距离；
  • c是光速。


通过此公式进行的计算可得出卫星上的时间扩展为45μs。总-7μs+45μs=38μs平衡-SRT和GR的影响。



在使用SRNS定位的应用问题中，还应考虑电离层和对流层延迟。此外，46 ns的校正与0.02 GPS卫星轨道的偏心率有关。



同时接收来自四个以上GPS / GLONASS卫星的信号的能力使您可以进一步提高确定接收器坐标的准确性。之所以能够实现这一目标，是因为导航器用四个未知数求解了四个方程组Cmk 次数并取平均值，从而根据数学统计定律提高最终评估的准确性。

2.将启用PPS的GPS接收器连接到串行RS232或USB端口。



普通的廉价GPS接收器将无法工作；您可能需要四处寻找合适的产品。



3.确保设备确实发出了PPS，为此，请使用gpsmon实用程序检查端口。



4.打开文件/etc/conf.d/gpsd并编辑以下行。



更换 



GPSD_OPTIONS=""


这样就变成了 



GPSD_OPTIONS="-n"


为了使gpsd在启动时立即开始搜索SRNS源，需要进行此更改。



5.启动或重新启动gpsd。 



(1:110)$ sudo /etc/init.d/gpsd start
(1:111)$ sudo /etc/init.d/gpsd restart


对于使用systemd的发行版，请使用适当的systemctl命令。



6.检查cgps命令的控制台输出。



您需要确保数据来自卫星。控制台应具有与插图相似的内容。





cgps控制台命令的输出。



7.现在该编辑/etc/ntp.conf文件了。 



# GPS Serial data reference (NTP0)
server 127.127.28.0
fudge 127.127.28.0 time1 0.9999 refid GPS

# GPS PPS reference (NTP1)
server 127.127.28.1 prefer
fudge 127.127.28.1 refid PPS


顶部NTP0记录指示几乎所有GPS设备上都可用的通用时间源。较低的NTP1条目定义了更准确的PPS源。



8.重新启动ntpd。 



(1:112)$ sudo /etc/init.d/ntpd restart

$ sudo systemctl重新启动ntp
