JavaScript + VSコードで最新のTDDの環境を収集します

TDDは長い間風変わりなものではありませんでした。このアプローチについては何百もの記事が見つかり、すべての初心者は極端なプログラミングについてどの本を読む必要があるか知っています。


私の同僚の多くもTDDを使用しています。彼らはテストを追加し、コードを書き、リファクタリングし、繰り返します。プロセスは同じように見えますが、1分かかるものもあれば、5分かかるものもあります。そして、後者が遅いと考えるということではありません。前者には、テストでの作業を最適化するための一連のトリックがあります。

テストにジャンプ


最も単純なものから始めましょう。テストに進む必要があります。
このために、Open Spec Fileを使用します。
この拡張機能は、次のタブのコードからテストファイルを開くのに役立ちます。



あらゆる好みに応じて、たくさんあります。
- UnitTest Switcherを使用すると、マウスの右ボタンで最初のメニュー項目を切り替えることができます
- テストに切り替えて、複雑な構造のプロジェクトのテストファイルへのパスをカスタマイズできます

私の経験では、最も速いオプションはホットキーです。リストからテストファイルを選択するには2秒かかります。ホットキー-0.5秒。

テストを実行する


それで、テストファイルに入りました。ここでAdd Only拡張は私たちに大いに役立ちます。これにより、1回のクリックでのみ修飾子を使用して、説明、テスト、テスト、コンテキストをカバーし、実行するだけです。シンプルで便利。



別のJavaScriptテストランナー拡張機能により、追加のみの横に、起動とデバッグのボタンがいくつか追加されます。テストパックを1回実行する必要がある場合に便利です。



コーティングをフォロー


カバレッジ雨どいの拡張子は、テストであなたのコードのカバレッジを強調しています。これにより、コードのカバーできない部分の手続きに時間を費やす必要がなくなります。カバーされていないすべてのブランチがすぐに表示されます。コードカバレッジ



拡張は、前のコードカバレッジの代替または追加のいずれかになります。これは、カバーされていない行を強調し、開いているファイルのカバーされている行の数をステータスバーに表示できます。



情報は、カバレッジフォルダーのlcov.infoファイルから取得されます。現在の問題を示す拡張機能については、監視フラグとカバレッジフラグを使用してテストを実行します。
