:


Flutter のUIViewアナログとは何ですか？

回答：


ウィジェット

違い：


UIViewは実際には画面に表示されるものです。SetNeedsDisplay（）が呼び出され、変更が表示されます。

ウィジェット -画面に表示される内容の説明。変更のために新しく作成されます。

追加情報：


Flutterには、クパチーノウィジェットライブラリが含まれています。Apple Designガイドラインを実装するウィジェットが含まれています。

質問：


ウィジェットの表示を更新するには？

回答：


StatefulWidgetとその状態の 使用。Flutterには、StatelessWidgetとStatefulWidgetの 2種類のウィジェットがあります。これらは同じように機能し、唯一の違いはレンダリング状態です。

違い：


StatelessWidgetには不変の状態があります。テキスト、ロゴなどの表示に適しています。それら。画面上の要素が表示時間全体で変化しない場合は、それが適しています。ステートフルウィジェットのコンテナとしても使用できます。

StatefulWidgetには、現在の状態に関する情報を格納するState状態があります。なんらかのアクション（サーバーからの応答、ユーザーによるボタンのクリックなど）を実行しているときに画面上の要素を変更したい場合-これはオプションです。

例：


1）StatelessWidget-テキスト

Text(
  'I like Flutter!',
  style: TextStyle(fontWeight: FontWeight.bold),
);

2）StatefulWidget-（FloatingActionButton）ボタンをクリックすると、テキストウィジェットのテキストがI Like FlutterからFlutter is Awesomeに変わります。

質問：


ウィジェットをアニメーション化する方法は？

回答：


使用AnimationControllerのクラスの子孫であり、アニメーション<T>抽象クラス。アニメーションを開始するだけでなく、一時停止、巻き戻し、停止、反対方向への再生もできます。画面の再描画を報告するTickerと連携します。

違い：


iOSでは、animate（withDuration：animations :)を使用してビューをアニメーション化できます。Flutterでは、アニメーションはAnimationControllerを使用してコードで記述する必要があります。

追加情報：


あなたにはもっと学ぶことができるアニメーション＆モーションウィジェット、アニメーションチュートリアル、およびアニメーションの概要。

例：

class SampleApp extends StatelessWidget {
  // This widget is the root of your application.
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Fade Demo',
      theme: ThemeData(
        primarySwatch: Colors.blue,
      ),
      home: MyFadeTest(title: 'Fade Demo'),
    );
  }
}

class MyFadeTest extends StatefulWidget {
  MyFadeTest({Key key, this.title}) : super(key: key);

  final String title;

  @override
  _MyFadeTest createState() => _MyFadeTest();
}

class _MyFadeTest extends State<MyFadeTest> with TickerProviderStateMixin {
  AnimationController controller;
  CurvedAnimation curve;

  @override
  void initState() {
    controller = AnimationController(duration: const Duration(milliseconds: 2000), vsync: this);
    curve = CurvedAnimation(parent: controller, curve: Curves.easeIn);
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text(widget.title),
      ),
      body: Center(
        child: Container(
          child: FadeTransition(
            opacity: curve,
            child: FlutterLogo(
              size: 100.0,
            )
          )
        )
      ),
      floatingActionButton: FloatingActionButton(
        tooltip: 'Fade',
        child: Icon(Icons.brush),
        onPressed: () {
          controller.forward();
        },
      ),
    );
  }

  @override
  dispose() {
    controller.dispose();
    super.dispose();
  }
}

質問：


使い方CoreGraphicsを？

回答：


Flutterは、CoreGraphicsではなく、Skia低レベルエンジンでCanvas APIを使用します。Androidも同様のCanvas APIを使用しています。

追加情報：


Flutterには、Canvasに描画するための2つのクラス、CustomPaintとCustomPainterがあります。2番目は、レンダリングアルゴリズムを実装します。

詳細はこちら：StackOverflow

例：


class SignaturePainter extends CustomPainter {
  SignaturePainter(this.points);

  final List<Offset> points;

  void paint(Canvas canvas, Size size) {
    var paint = Paint()
      ..color = Colors.black
      ..strokeCap = StrokeCap.round
      ..strokeWidth = 5.0;
    for (int i = 0; i < points.length - 1; i++) {
      if (points[i] != null && points[i + 1] != null)
        canvas.drawLine(points[i], points[i + 1], paint);
    }
  }

  bool shouldRepaint(SignaturePainter other) => other.points != points;
}

class Signature extends StatefulWidget {
  SignatureState createState() => SignatureState();
}

class SignatureState extends State<Signature> {

  List<Offset> _points = <Offset>[];

  Widget build(BuildContext context) {
    return GestureDetector(
      onPanUpdate: (DragUpdateDetails details) {
        setState(() {
          RenderBox referenceBox = context.findRenderObject();
          Offset localPosition =
          referenceBox.globalToLocal(details.globalPosition);
          _points = List.from(_points)..add(localPosition);
        });
      },
      onPanEnd: (DragEndDetails details) => _points.add(null),
      child: CustomPaint(painter: SignaturePainter(_points), size: Size.infinite),
    );
  }
}

違い：

Flutterには2つのナビゲーションメソッドがあります。

  • ルート名でマップを説明する
  • ルートに直接移動します。

スレッディングと非同期性


質問：


Flutterで非同期コードを記述する方法は？

回答：


Dartは、Isolatesで実行されるシングルスレッド実行モデルを実装しています。非同期実行の場合は、C＃、JavaScript、またはKotlinコルーチンでおなじみのasync / awaitを使用します。

例：


リクエストを実行し、UIを更新するための結果を返す：

loadData() async {
  String dataURL = "https://jsonplaceholder.typicode.com/posts";
  http.Response response = await http.get(dataURL);
  setState(() {
    widgets = json.decode(response.body);
  });
}

リクエストへの応答を受け取ったら、setState（）メソッドを呼び出して、新しいデータでウィジェットツリーを再描画する必要があります。

例：

質問：


バックグラウンドスレッドでコードを実行する方法

回答：


上記のように-async / awaitと分離（Isolate）を使用します。

違い：


iOSのそのままの状態で、メソッドを再定義する可能性のあるオペレーションを使用できます。Flutterでは、箱から出してすぐにasync / awaitを使用するだけで、あとはDartが処理します。

例：


ここでは、dataLoader（）メソッドが分離されています。分離すると、大きなJSONの解析、暗号化、画像処理などの重い操作を実行できます。

static dataLoader(SendPort sendPort) async {
  // Open the ReceivePort for incoming messages.
  ReceivePort port = ReceivePort();

  // Notify any other isolates what port this isolate listens to.
  sendPort.send(port.sendPort);

  await for (var msg in port) {

    SendPort replyTo = msg[1];

    http.Response response = await http.get(dataURL);

    });

// the entry point for the isolate
  static dataLoader(SendPort sendPort) async {
    // Open the ReceivePort for incoming messages.
    ReceivePort port = ReceivePort();

    // Notify any other isolates what port this isolate listens to.
    sendPort.send(port.sendPort);

プロジェクトの構造とリソース


質問：


異なる解像度のリソースをどこに保存しますか？

回答：


資産で。

違い：


iOSでは、グラフィックリソースにImages.xcassetがあり、assetsフォルダーにあります。Flutterにはアセットのみがあります。リソースフォルダーはプロジェクトの任意の場所に配置できます。最も重要なのは、そのパスをpubspec.yamlファイルに書き込むことです。

追加情報：

質問：


ライフサイクルイベントの処理方法

回答：


使用WidgetsBindingとdidChangeAppLifecycleState（）メソッドを。

追加情報：


FlutterはネイティブコードでFlutterAppDelegateを使用し、Flutterエンジンは処理状態の変更をできるだけ目立たなくします。ただし、状態に応じてさらに作業が必要な場合は、ライフサイクルが少し異なります。

  • 非アクティブ-アプリケーションは非アクティブであり、ユーザー入力を受け取りません。この状態はiOSにのみ存在し、Androidには類似物はありません。
  • 一時停止-アプリケーションは現在ユーザーからは見えず、ユーザー入力に応答しませんが、バックグラウンドで動作します。
  • 再開-アプリケーションが表示され、ユーザー入力に応答します。
  • 一時停止-停止処理中のアプリケーション。この状態はAndroidにのみ存在し、iOSには類似物はありません。

これについては、AppLifecycleStatusのドキュメントで詳しく説明しています。

例：


import 'package:flutter/widgets.dart';

class LifecycleWatcher extends StatefulWidget {
  @override
  _LifecycleWatcherState createState() => _LifecycleWatcherState();
}

class _LifecycleWatcherState extends State<LifecycleWatcher> with WidgetsBindingObserver {
  AppLifecycleState _lastLifecycleState;

  @override
  void initState() {
    super.initState();
    WidgetsBinding.instance.addObserver(this);
  }

  @override
  void dispose() {
    WidgetsBinding.instance.removeObserver(this);
    super.dispose();
  }

  @override
  void didChangeAppLifecycleState(AppLifecycleState state) {
    setState(() {
      _lastLifecycleState = state;
    });
  }

  @override
  Widget build(BuildContext context) {
    if (_lastLifecycleState == null)
      return Text('This widget has not observed any lifecycle changes.', textDirection: TextDirection.ltr);

    return Text('The most recent lifecycle state this widget observed was: $_lastLifecycleState.',
        textDirection: TextDirection.ltr);
  }
}

void main() {
  runApp(Center(child: LifecycleWatcher()));
}

質問：


検証エラーを表示する方法は？

回答：


InputDecorationとその状態について も同様です。

例：


class SampleApp extends StatelessWidget {
  // This widget is the root of your application.
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Sample App',
      theme: ThemeData(
        primarySwatch: Colors.blue,
      ),
      home: SampleAppPage(),
    );
  }
}

class SampleAppPage extends StatefulWidget {
  SampleAppPage({Key key}) : super(key: key);

  @override
  _SampleAppPageState createState() => _SampleAppPageState();
}

class _SampleAppPageState extends State<SampleAppPage> {
  String _errorText;

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text("Sample App"),
      ),
      body: Center(
        child: TextField(
          onSubmitted: (String text) {
            setState(() {
              if (!isEmail(text)) {
                _errorText = 'Error: This is not an email';
              } else {
                _errorText = null;
              }
            });
          },
          decoration: InputDecoration(hintText: "This is a hint", errorText: _getErrorText()),
        ),
      ),
    );
  }

  _getErrorText() {
    return _errorText;
  }

  bool isEmail(String emailString) {
    String emailRegexp =
        r'^(([^<>()[\]\\.,;:\s@\"]+(\.[^<>()[\]\\.,;:\s@\"]+)*)|(\".+\"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))$';

    RegExp regExp = RegExp(emailRegexp);

    return regExp.hasMatch(emailString);
  }
}

_incrementCounter() async {
