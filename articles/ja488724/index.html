Goのベストセキュリティプラクティス6

記事の翻訳は、特にGolang Developerコースの学生のために作成されました。




Golangの人気は近年大幅に高まっています。Docker、Kubernetes、Terraform などの成功したプロジェクトは、このプログラミング言語に大きな賭けをしています。 Goは最近、コマンドラインツールを作成するための事実上の標準になりました。セキュリティの観点から、Goは脆弱性レポートに従ってうまく管理しており、2002年以来、CVEレジストリは1つしかありません。

ただし、脆弱性がなくても、プログラミング言語の安全性が高いとは限りません。特定のルールに従わない場合、私たち人間は安全でないアプリケーションを作成できます。たとえば、OWASPから安全なコードを書くためのルールを知る、Goを使用するときにこれらのメソッドを適用する方法について考えることができます。そして、これがまさに私が今回行うことです。この投稿では、Goを使用して開発する際に考慮すべき6つのプラクティスを紹介します。

1.入力を確認する


ユーザー入力の確認は、機能上の目的だけでなく、システムに損傷を与える可能性のある侵入データを送信するサイバー犯罪者による攻撃を回避するのにも役立ちます。さらに、ユーザーがこのツールをより自信を持って使用できるように支援し、愚かな一般的な間違いを防ぐことができます。たとえば、ユーザーが複数のエントリを一度に削除しようとするのを防ぐことができます。

ユーザー入力をテストするには、strconvなどのネイティブGoパッケージを使用して、文字列を他のデータ型に変換します。 Goはregexp、複雑なチェックのための正規表現もサポートしています。 Goはネイティブライブラリの使用を好みますが、バリデーターなどのサードパーティパッケージがあります。。バリデーターを使用すると、構造または個々のフィールドの検証をより簡単に有効にすることができます。たとえば、次のコードは、構造Userに有効な電子メールアドレスが含まれていることを確認します。

package main

import (
	"fmt"

	"gopkg.in/go-playground/validator.v9"
)

type User struct {
	Email string `json:"email" validate:"required,email"`
	Name  string `json:"name" validate:"required"`
}

func main() {
	v := validator.New()
	a := User{
		Email: "a",
	}

	err := v.Struct(a)

	for _, e := range err.(validator.ValidationErrors) {
		fmt.Println(e)
	}
}


2. HTMLテンプレートを使用する


一般的な重大な脆弱性の1つは、クロスサイトスクリプティングまたはXSSです。主な悪用は、攻撃者が悪意のあるコードをアプリケーションに挿入して出力を変更できることです。たとえば、誰かがJavaScriptコードをURLのクエリ文字列の一部として送信する場合があります。アプリケーションがユーザーの値を返すと、JavaScriptコードを実行できます。したがって、開発者はこの可能性を認識し、ユーザーが入力したデータをクリアする必要があります。

Goには、アプリケーションがユーザーに返すものをエンコードするhtml /テンプレートパッケージがあります。したがって、ブラウザが次のような入力を行う代わりに<script>alert(‘ !’);</script>彼は警告を出します。入力をエンコードすると、アプリケーションはブラウザで出力される一般的なHTMLコードとして入力を処理します。HTMLテンプレートを返すHTTPサーバーは次のようになります。

package main

import (
	"html/template"
	"net/http"
)

func handler(w http.ResponseWriter, r *http.Request) {
	param1 := r.URL.Query().Get("param1")
	tmpl := template.New("hello")
	tmpl, _ = tmpl.Parse(`{{define "T"}}{{.}}{{end}}`)
	tmpl.ExecuteTemplate(w, "T", param1)
}
func main() {
	http.HandleFunc("/", handler)
	http.ListenAndServe(":8080", nil)
}


ただし、GoでWebアプリケーションを開発するときに使用できるサードパーティのライブラリがあります。たとえば、開発者がCookie認証値のエンコードなどを行うのに役立つライブラリを含むGorilla Webツールキット。nosurfもあります。これは、クロスサイトリクエストフォージェリ（CSRF）の防止に役立つHTTPパッケージです。

3. SQLインジェクションから身を守る


しばらく開発を続けている場合は、SQLインジェクションについて知っているかもしれません。SQLインジェクションは、OWASPトップの最初の位置を占めています。ただし、Goを使用する際に考慮すべき特定の点がいくつかあります。最初に行う必要があることは、データベースに接続するユーザーが制限された権限を持っていることを確認することです。前のセクションで説明したように、ユーザー入力をサニタイズするか、特殊文字をエスケープしてHTMLテンプレートパッケージのHTMLEscapeString関数を使用することもお勧めします。

ただし、含める必要がある最も重要なコードは、パラメーター化されたクエリの使用です。 Goでは、式を準備しません（これは準備済みステートメントです）） 接続;データベースで準備します。パラメータ化されたクエリを使用する方法の例を次に示します。

customerName := r.URL.Query().Get("name")
db.Exec("UPDATE creditcards SET name=? WHERE customerId=?", customerName, 233, 90)


ただし、データベースエンジンが準備された式の使用をサポートしていない場合はどうなりますか？または、クエリのパフォーマンスに影響する場合はどうなりますか？まあ、db.Query（）関数を使用できますが、前のセクションで示したように、最初にユーザー入力をサニタイズしていることを確認してください。SQLインジェクションを防ぐために、sqlmapなどのサードパーティライブラリもあります。

私たちのあらゆる努力にもかかわらず、脆弱性がサードパーティを介してアプリケーションに侵入または侵入することがあります。SQLインジェクションなどの重大な攻撃からWebアプリケーションを保護するには、Sqreenなどのアプリケーションセキュリティ管理プラットフォームの使用を検討してください。

4.機密情報を暗号化する


文字列がbase-64形式などで読みにくいという事実は、非表示の値が秘密であることを意味するものではありません。攻撃者が簡単に解読できない情報を暗号化する方法が必要です。暗号化する一般的な情報は、データベースのパスワード、ユーザーのパスワード、さらには社会保障番号です。

OWASPはいくつか持っている推奨例えば、暗号化アルゴリズムを使用したものにしbcrypt、PDKDF2、Argon2またはscrypt。幸い、情報を暗号化するための信頼できる実装を含むGoパッケージがあります- 暗号化。次のコードは使用例bcryptです。

package main

import (
	"database/sql"
	"context"
	"fmt"

	"golang.org/x/crypto/bcrypt"
)

func main() {
	ctx := context.Background()
	email := []byte("john.doe@somedomain.com")
	password := []byte("47;u5:B(95m72;Xq")

	hashedPassword, err := bcrypt.GenerateFromPassword(password, bcrypt.DefaultCost)
	if err != nil {
		panic(err)
	}

	stmt, err := db.PrepareContext(ctx, "INSERT INTO accounts SET hash=?, email=?")
	if err != nil {
		panic(err)
	}
	result, err := stmt.ExecContext(ctx, hashedPassword, email)
	if err != nil {
		panic(err)
	}
}


サービス間で情報を転送する場合は、引き続き注意が必要です。ユーザーデータをプレーンテキストで送信したくないでしょう。保存する前にアプリケーションがユーザーデータを暗号化するかどうかは関係ありません。インターネット上の誰かがトラフィックをリッスンし、システムのリクエストのログを保持できるとします。攻撃者はこの情報を使用して、他のシステムの他のデータと照合することができます。

5. HTTPS接続を提供する


ほとんどのブラウザは現在、すべてのサイトで動作するためにHTTPSを必要とします。たとえば、Chromeでは、サイトがHTTPSを使用していない場合に警告が表示されます。情報セキュリティ部門は、サービス間の通信に転送暗号化をポリシーとして使用できます。つまり、システムでの中継接続のセキュリティを確保するには、ポート443でアプリケーションをリッスンするだけではありません。適切な証明書を使用し、HTTPSを使用して、攻撃者がプロトコルをHTTPにダウングレードしないようにする必要もあります。

以下は、HTTPSプロトコルを提供および使用するWebアプリケーションのコードスニペットです。

package main

import (
    "crypto/tls"
    "log"
    "net/http"
)

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("/", func(w http.ResponseWriter, req *http.Request) {
        w.Header().Add("Strict-Transport-Security", "max-age=63072000; includeSubDomains")
        w.Write([]byte("This is an example server.\n"))
    })
    cfg := &tls.Config{
        MinVersion:               tls.VersionTLS12,
        CurvePreferences:         []tls.CurveID{tls.CurveP521, tls.CurveP384, tls.CurveP256},
        PreferServerCipherSuites: true,
        CipherSuites: []uint16{
            tls.TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384,
            tls.TLS_ECDHE_RSA_WITH_AES_256_CBC_SHA,
            tls.TLS_RSA_WITH_AES_256_GCM_SHA384,
            tls.TLS_RSA_WITH_AES_256_CBC_SHA,
        },
    }
    srv := &http.Server{
        Addr:         ":443",
        Handler:      mux,
        TLSConfig:    cfg,
        TLSNextProto: make(map[string]func(*http.Server, *tls.Conn, http.Handler), 0),
    }
    log.Fatal(srv.ListenAndServeTLS("tls.crt", "tls.key"))
}


アプリケーションはポート443でリッスンすることに注意してください。次の行はHTTPS構成を提供する行です。

w.Header().Add("Strict-Transport-Security", "max-age=63072000; includeSubDomains")


次のようにして、TLS構成でサーバー名を指定することもできます。
config := &tls.Config{ServerName: "yourSiteOrServiceName"}


Webアプリケーションが内部通信専用であっても、常にバックエンド暗号化を使用することをお勧めします。なんらかの理由で攻撃者が内部トラフィックを傍受できると想像してみてください。可能な限り、将来の可能性のある攻撃者の基準を引き上げることが常に最善です。

6.エラーとログに注意してください。


最後に重要なことですが、Goアプリケーションでのエラー処理とロギング。

本番環境の問題のトラブルシューティングを成功させるには、アプリケーションを適切に構成する必要があります。ユーザーに表示するエラーを考慮する必要があります。何が間違っているのかをユーザーに知られたくないでしょう。攻撃者はこの情報を使用して、使用しているサービスやテクノロジーを特定できます。さらに、ログは素晴らしいですが、どこかに保存されていることを覚えておいてください。また、ログが悪用された場合、それらを使用して、今後の攻撃をシステムに埋め込むことができます。

したがって、最初に覚えておく必要があるのは、Goに例外はないということです。つまり、他の言語とは異なる方法でエラーを処理する必要があります。標準は次のようになります。

if err != nil {
    //  
}


Goには、ログを操作するための組み込みライブラリも用意されています。最も単純なコードは次のようになります。

package main

import (
	"log"
)

func main() {
	log.Print("Logging in Go!")
}


ただし、ログを維持するためのサードパーティライブラリがあります。これらのいくつかはlogrus、glogおよびloggoです。ここに使用するコードの小さなスニペットがありますlogrus：

package main

import (
	"os"

	log "github.com/sirupsen/logrus"
)

func main() {
	file, err := os.OpenFile("info.log", os.O_CREATE|os.O_APPEND, 0644)
	if err != nil {
		log.Fatal(err)
	}

	defer file.Close()

	log.SetOutput(file)
	log.SetFormatter(&log.JSONFormatter{})
	log.SetLevel(log.WarnLevel)

	log.WithFields(log.Fields{
		"animal": "walrus",
		"size":   10,
	}).Info("A group of walrus emerges from the ocean")
}


最後に、ログに記録するデータの暗号化やサナレーションなど、以前のすべての推奨事項を必ず適用してください。

最終的に何をしても、常にアプリケーションのセキュリティを強化できることに注意してください。攻撃者は常に脆弱性を悪用する新しい方法を見つけるため、常にアプリケーションのセキュリティに取り組んでください。
