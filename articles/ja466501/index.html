Githubアクションとクロスプラットフォームの構築

こんにちは、ハブル。この記事では、githubアクションを使用してすべてのプラットフォームでビルドをセットアップする方法について説明します。


バックグラウンド


私はelectronで簡単なアプリケーションを書きました、私自身はlinuxを使用しましたが、私の友人はmacosを好みました。コンピューターでmacos用にコンパイルして、pkgを友達に渡したところ、起動しませんでした。その結果、macos用にアプリケーションをコンパイルする唯一のオプションは、macos上でコンパイルすることでした。タスクをできるだけ簡略化するために、ビルド：linux、ビルド：mac、ビルド：winの3つのスクリプトを作成しました。その結果、コンパイル後に、linux.deb、linux.AppImage、mac.pkg、win.exeのファイルが取得されました。しかし、別のシステムでコンパイルする必要がある1つの問題が残っていました。そして、gihubアクションの救済。


すべてが機能する方法


githubの新しいリリースボタンを押すと、マジックがgithubアクションのワークフローを開始し、すべてのオペレーティングシステムでコンパイルされ、バイナリをリリースに追加します


https://github.com/JasonEtco/upload-to-releasを使用してrelease にファイルを追加しましたが、このアクションには1つの問題がありました。これはコンテナーアクションであり、githubアクションでは、コンテナーアクションはLinuxでのみ使用できます。そのため、コンパイル用に3つ、ロード用に1つの4つのジョブを使用することにしました。環境はジョブごとに保存されないため、アーティファクトを使用してジョブ間で交換されます


練習


まず、コンテンツを含む.github / workflows / workflow.ymlフォルダー


name: CI

on: release

まあ、これがCIワークフローであることは明らかだと思います。リリース時にリリースされ、今では最も重要な仕事です


jobs:

 build-linux: 

   runs-on: ubuntu-latest 

   steps: 

   - uses: actions/checkout@v1 

   - name: Install bluetooth 

     run: sudo apt-get install -y build-essential libbluetooth-dev 

   - name: Install dependencies 

     run: npm install 

   - name: Build linux 

     run: npm run build:linux 

   - name: Creating out 

     run: | 

       mkdir out 

       cp dist/linux.AppImage out/

       cp dist/linux.deb out/ 

     - name: Upload build 

       uses: actions/upload-artifact@master 

       with: 

         name: linux 

         path: out

ジョブのステップ：これがすべての作業、build-linux：これはbuild-linux、runs-onという名前の作業です：ubuntu-latestは、最新のubuntuの下ですべてを実行する必要があることを示しています

upload: 

  runs-on: ubuntu-latest 

  needs: [build-linux, build-mac, build-win] 

  steps: 

  - uses: actions/checkout@v1 

  - name: Download linux artifact 

    uses: actions/download-artifact@master 

    with: 

      name: linux 

  - name: Download mac artifact 

    uses: actions/download-artifact@master 

    with: 

      name: mac 

  - name: Download win artifact 

    uses: actions/download-artifact@master 

    with: 

      name: win 

  - name: Upload to Release deb 

    uses: JasonEtco/upload-to-release@v0.1.1 

    env: 

      GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }} 

    with: 

      args: linux/linux.deb application/vnd.debian.binary-package 

   - name: Upload to Release AppImage 

     uses: JasonEtco/upload-to-release@v0.1.1 

     env: 

       GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }} 

     with: 

       args: linux/linux.AppImage application/x-executable 

   - name: Upload to Release pkg 

     uses: JasonEtco/upload-to-release@v0.31.1 

     env: 

       GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }} 

     with: 

       args: mac/mac.pkg application/x-xar 

   - name: Upload to Release exe 

     uses: JasonEtco/upload-to-release@v0.1.1 

     env: 

       GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }} 

     with: 

       args: win/win.exe application/octet-stream

非常に重要な部分はそのニーズです。この行は、すべてのビルドの後にのみ作業を開始する必要があることを示しています（ビルドが並行して実行される場合）。最初にアーティファクトをダウンロードし、次にそれらのバイナリをリリースに追加します。
