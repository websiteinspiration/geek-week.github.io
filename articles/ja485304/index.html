ここで何が面白いですか？

各ブロックにはタイトルがあるため、セクションでラップできます（仕様に従って、セクションと記事にはヘッダーが必要です）。

.viewport-フレームを含むブロック（以下、単にブロックと呼ぶ）。

src属性はフレームの内容を参照し、JSを使用して「カバー」に置き換えられます（これについては後で詳しく説明します）。

シームレスは、フレームのコンテンツがドキュメントの一部であるかのように表示されることを指定します（現在はサポートされていません）。

Scrolling =“ no”は、フレームにスクロールバーが表示されないようにします。

Window.openは、フレームのコンテンツ（新しいタブでは "_blank"）を開く1つの方法です。

Document.locationは、ファイルをダウンロードする1つの方法です。

問題番号1.ブロックにフレームを入力してください


Chromeのデフォルトのiframeサイズは304x154pxです。

セクションに次のスタイルを適用します。
section {
    margin: 1em;
    width: 300px;
    background: rgba(0, 0, 0, 0.15);
    border-radius: 5px;
    box-shadow: inset 0 0 50px rgba(0, 0, 0, 0.3);
}

セクションの幅を300px（+メタ名=“ viewport”およびdisplay：親要素のflex）に設定すると、解像度が異なる画面でも同じフレームが表示されます。

あなたが見ることができるように、問題は、コンテンツがフルサイズでフレーム内に表示され、フレーム自体がわずかにブロックの境界を越えて行くことです。

この問題を解決する1つの方法は、iframeをスケーリングすることです。
/*
      ""  , , 1024768px,
    25%,
  ,      - 0 0 (  )
*/
iframe {
    width: 1024px;
    height: 768px;
    transform: scale(.25);
    transform-origin: 0 0;
}

次に、ブロックのサイズを決定します。幅：1024 * 0.25 = 256px、高さ：768 * 0.25 = 192px。

問題＃2：ホバリングコンテンツ


ページにかなりの数のフレームがあり、特に動的コンテンツの場合、非常に長いロード/再ロードとスクロール時の一定の遅延が発生します。

この問題を解決するには、フレームのコンテンツをオフにして、画像に置き換えます。画像のサイズは1024x768ピクセルにする必要があります。より正確には、ファイルアドレスを変更します。値はsrcです（Codepenでは、この問題はフレームの内容の短いアニメーションで解決されます）。特定のフレームにカーソルを合わせると、その内容が表示されます。

解決策はこれかもしれません：
//     
let iframes = document.querySelectorAll("iframe")

//   (         Nodelist,   )
for (let i = 0; i < iframes.length; i++){
    let iframe = iframes[i],
    
    //    sr
    originalSrc = iframe.src
    
    //    
    iframe.src = "cover.jpg"
    
    //      
    iframe.addEventListener("mouseover", () => iframe.src = originalSrc)
    
    //  ""
    iframe.addEventListener("mouseout", () => iframe.src = "cover.jpg")
}
