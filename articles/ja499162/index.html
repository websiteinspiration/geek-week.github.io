バンドルサイズを縮小する方法-css-modulesの1文字クラスの戦略

標準のハッシュを1文字のプレフィックス+ファイルパスのハッシュに置き換えることで、バンドルの圧縮をファイルサイズの40％改善します。

Css-modulesを使用すると、BirdおよびCatコンポーネントを記述できます。それぞれのスタイルに同じ名前のstyles.cssおよび.blockクラスがあり、これらのクラスはこれらのコンポーネントごとに異なります。

/* Bird / styles.css */
.block { }
.name { }
/* Cat / styles.css */
.block { }
.name { }

ここではトリッキーなことは何もありません。Webpackは、[[hash：base64：8] "設定を使用して、すべてのファイルから各クラスをハッシュします。すべてのクラスの名前が変更され、どのクラスから来たかを理解するためにリンクが貼られます。アセンブリの基本バージョンでは、jsを操作するときに、スタイル用のstyles.cssファイルとリンク用のstyles.jsファイルがあります。

テストケースを続けると、k3bvEft8のような奇妙な名前を持つ4つの独立したクラスが取得されます。

/* Bird */
.k3bvEft8 { }
.f2tp3lA9 { }
/* Cat */
.epIUQ_6W { }
.oRzvA1Gb { }

本番アセンブリを実行し、ファイルを圧縮します。スタンドでは、300Kbのcssファイルがgzipを使用して70Kbにパッケージ化されました[またはbrotliを使用して50Kb]。ハッシュはランダムに生成される文字列であるため、圧縮率は非常に低く、圧縮率は低くなります。圧縮アルゴリズムはシーケンスを認識せず、各シンボルの位置を記憶するように強制されます。これらのセクションの内容を圧縮せずにそのまま転送します。

これで何かする必要があります。しかし、何ですか？動作中、Webpackはファイルツリーを非同期で読み取り、クラス名も調べます。毎回異なる方法で。あなたがしがみつくことができる唯一のものはCSS内の名前の順序です-それは一定です（そうでなければすべてが壊れます、CSSでは順序が重要です）。ファイル内のクラス位置番号は、1文字のプレフィックスでエンコードされます。エンコードは52（[a-zA-Z] +）または64（[a-zA-Z0-9 _-] +）文字で取得できます。ここでの主なことは、数字やハイフンがある場合に保護接頭辞を付けることを忘れないことです。

/* Bird */
.a { }
.b { }
/* Cat */
.c { }
.d { }

見た目は良いようです-名前は可能な限り圧縮されています。ただし、問題は、Webpackが非同期であり、起動のたびに、特にサーバーとクライアントの同時アセンブリが並行して起動されたときに、クラス名と同様に無秩序な方法でファイルを受信することです。スピードありがとうございますが、ここでは干渉します。

/* Bird */
.c { }
.d { }
/* Cat */
.a { }
.b { }

ご覧のとおり、ファイルの順序の不一致をキャッチしました。

この動作を修正するには、ファイルの場所、クラスの場所、およびそれらの位置番号を覚えておきます。

/* Bird */
.a { }
.b { }
/* Cat */
.a { }
.b { }

ファイル内に注文を保存しました。ただし、ファイルを互いに区別する必要があります。ファイルパスからのハッシュは、混乱を避けるのに役立ちます。

/* Bird */
.a_k3bvEft8 { }
.b_k3bvEft8 { }
/* Cat */
.a_oRzvA1Gb { }
.b_oRzvA1Gb { }

（ '_'はここでは必要ありません。これは例示のみを目的としています。ハッシュはプレフィックスとは異なり安定した長さであり、衝突はありません）

プロジェクトで完全に一意であるが、シーケンスが繰り返されているクラス名を取得しました。

このプロジェクトでは、css 50 Kbおよびjs 47 Kbのファイルからcss 30 Kbおよびjs 28 Kb [合計58 Kb、brotli]を取得しました。
約40Kbを節約。重要なcssのサイズとhtmlのサイズはわずかに減少します。

Webpackからのデータを処理するためのクラスを作成し、css-loader config（getLocalIdent）で呼び出しをスローするために残ります

PSさらに進んで、ファイルパスを保存し、パスを並べ替え、それらを1文字の方法で置き換えることもできますが、長期的なキャッシュの観点からはさらに悪いことに加えて、アセンブリで複数のパスを作成し、クライアント/サーバーを順番に組み立てる必要があります。
