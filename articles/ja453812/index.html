テーマ、スタイルなど


ほとんどすべての開発者はAndroidにテーマがあることを知っていますが、それらの使用は通常、スタックオーバーフローまたは他のリソースからのxmlの一部のコピーに制限されています。トピックに関するインターネット上の情報がありますが、これは通常、特定の結果を達成する方法に関する単なるレシピです。この記事では、Androidのスタイル設定メカニズムの概要を紹介しようとしました。

前書き


Androidのスタイルとテーマは、デザインの詳細（色、フォントサイズなど）をUI構造から分離できるメカニズムです。これがどのように機能するかを理解するには、カスタムビューを使用した簡単な例が役立ちます。

ビューの境界内に希望の色で長方形を描く単純なカスタムビューが必要です。

class RectView @JvmOverloads constructor(
    context: Context,
    attrSet: AttributeSet? = null,
    defStyleAttr: Int = 0
) : View(context, attrSet, defStyleAttr) {

    private val paint = Paint().apply {
        color = Color.BLUE
    }

    override fun onDraw(canvas: Canvas) {
        super.onDraw(canvas)
        canvas.drawRect(0f, 0f, width.toFloat(), height.toFloat(), paint)
    }
}



RectViewの属性


次に、レイアウト長方形の色を変更できるようにしたいと思います。
これを行うには、新しい属性を追加する必要があります。attrs.xmlをファイルに追加します

<resources>
    <attr name="rectColor" format="color"/>
</resources>

これで、コードでR.attr.rectColorを介してこの属性のIDにアクセスでき
、画面のレイアウトでapp：rectColor属性を使用できます。

<org.berendeev.themes.RectView
            android:id="@+id/text2"
            android:layout_width="100dp"
            app:rectColor="@color/colorPrimary"
            android:layout_height="100dp"/>

ただし、RectViewは、長方形の色を取得できる属性があることをまだ認識していません。

RectViewにrectColor属性を理解させ、属性のグループを追加してみましょう

<resources>
    <attr name="rectColor" format="color"/>

    <declare-styleable name="RectView">
        <attr name="rectColor"/>
    </declare-styleable>
</resources>

Rクラスでは、2つの新しいフィールドが生成されました
。R.styleable.RectViewはid属性の配列で、現在は1つの要素の配列です。R.attr.rectColor
R.styleable.RectView_rectColorはR.styleable.RectView配列の属性IDインデックスそれら。属性のIDを取得できるため、R.styleable.RectView [R.styleable.RectView_rectColor]

そして、rectColor属性のサポートをコードに追加します。

init {
        val typedArray = context.theme.obtainStyledAttributes(
            attrSet,
            R.styleable.RectView,
            0,
            0
        )
        try {
            paint.color = typedArray.getColor(
                R.styleable.RectView_rectColor,
                Color.BLUE
            )
        } finally {
            typedArray.recycle()
        }
}

完全なコード
class RectView @JvmOverloads constructor(
    context: Context,
    attrSet: AttributeSet? = null,
    defStyleAttr: Int = 0
) : View(context, attrSet, defStyleAttr) {

    private val paint = Paint().apply {
        color = Color.BLUE
    }

    init {
        val typedArray = context.theme.obtainStyledAttributes(
            attrSet,
            R.styleable.RectView,
            0,
            0
        )
        try {
            paint.color = typedArray.getColor(
                R.styleable.RectView_rectColor,
                Color.BLUE
            )
        } finally {
            typedArray.recycle()
        }
    }

    override fun onDraw(canvas: Canvas) {
        super.onDraw(canvas)
        canvas.drawRect(0f, 0f, width.toFloat(), height.toFloat(), paint)
    }
}


これで、レイアウトの長方形の色を変更できます。



RectViewスタイル


これで属性は1つだけになりましたが、10個あることを想像してください。同様の要素のレイアウトでこれらのすべての属性を毎回設定するのは非常に不便です。

すべてをスタイリッシュにする決断。

<style name="DefaultRectViewStyle">
        <item name="rectColor">@color/colorAccent</item>
</style>

これで、スタイルを指定して、同じ外観のあらゆる種類のビューを作成できます。

<org.berendeev.themes.RectView
            android:id="@+id/text2"
            android:layout_width="100dp"
            style="@style/DefaultRectViewStyle"
            android:layout_height="100dp"/>

デフォルトのスタイル


ここで、すべてのビューにデフォルトで何らかのスタイルを設定します。これを行うには、obtainStyledAttributesメソッドのデフォルトスタイルを指定します。

context.theme.obtainStyledAttributes(
            attrSet,
            R.styleable.RectView,
            0,
            R.style.DefaultRectViewStyle
)

それは十分だ。これで、レイアウトに追加するすべてのRectViewのデフォルトスタイルはDefaultRectViewStyleになります。

RectViewスタイルのテーマ属性


デフォルト値は良いですが、レイアウトをもっと柔軟にコントロールしたいです。アプリケーション全体または個別のアクティビティに対して特定のビューのスタイルを設定すると便利です。

これには、新しい属性が必要です。値をテーマに設定します。値は、RectViewの外観を決定するスタイルになります。

<attr name="rectViewStyle" format="reference"/>

そして、rectViewにこのテーマ属性を理解するように教えてください。3番目のパラメーターR.attr.rectViewStyleをobtainStyledAttributesメソッドに渡す。


context.theme.obtainStyledAttributes(
                attrSet,
                R.styleable.RectView,
                R.attr.rectViewStyle,
                R.style.DefaultRectViewStyle
            )

これで、rectViewStyleという名前のアイテムとtypeスタイルの値がトピックで指定されている場合、このスタイルはこのテーマを持つすべてのRectViewに適用されます。

活動レベルのテーマ


このトピックでは、rectViewStyle属性の値を設定します。

<resources>
    <style name="AppTheme" parent="Theme.AppCompat.Light.DarkActionBar">
        <item name="colorPrimary">@color/colorPrimary</item>
        <item name="colorPrimaryDark">@color/colorPrimaryDark</item>
        <item name="colorAccent">@color/colorAccent</item>
        <item name="rectViewStyle">@style/LocalRectViewStyle</item>
    </style>
</resources>

アプリケーションマニフェストでトピックを指定します。

<manifest xmlns:android="http://schemas.android.com/apk/res/android"
          package="org.berendeev.themes">
    <application
            android:allowBackup="true"
            android:icon="@mipmap/ic_launcher"
            android:label="@string/app_name"
            android:roundIcon="@mipmap/ic_launcher_round"
            android:supportsRtl="true"
            android:theme="@style/AppTheme">
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
    </application>
</manifest>

すべてのアクティビティは、デフォルトのAppThemeテーマに設定されます。アプリケーション全体のRectViewデザインを設定しました。

トピックは、マニフェストの特定のアクティビティに設定することもできます。

<activity android:name=".MainActivity" android:theme="@style/MyTheme">

アプリケーションレベルまたはアクティビティレベルで設定するテーマは、標準テーマから継承する必要があります。たとえば、Theme.AppCompatです。そうしないと、実行時にクラッシュします。
java.lang.IllegalStateException：このアクティビティでは、Theme.AppCompatテーマ（または子孫）を使用する必要があります。

トピックを表示


ビューレベルでは、テーマを再インストールしませんが、必要な属性値を上書きするため、標準テーマから継承する必要はありません。親は空にすることも、たとえば、ThemeOverlay.AppCompatオーバーレイの1つから継承することもできます。

ここでは、階層内のLinearLayoutグループとそのすべての子孫にMyOverlayテーマを設定します。

<LinearLayout
        xmlns:android="http://schemas.android.com/apk/res/android"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:orientation="vertical"
        android:theme="@style/MyOverlay">
    <org.berendeev.themes.RectView
            android:layout_width="100dp"
            android:layout_height="100dp"/>
 </LinearLayout>

サブジェクトでは、親なしで行うことができます。アクティビティのテーマを変更するだけです。

<style name="MyOverlay">
        <item name="rectViewStyle">@style/LocalRectViewStyle</item>
</style>

属性値を適用する方法



ビューについて、レイアウト、スタイル、テーマで属性値を定義する場合、値を選択する順序は次のようになります。最も高い優先度は、レイアウトで設定された値です。値がレイアウトに設定されている場合、スタイルとテーマは無視され、スタイルが続行され、テーマが続き、最後にデフォルトスタイルが配置されます。

リソーストピック


トピック値はリソースで使用できます。たとえば、ドローアブルでは、テーマに設定されている色に依存する色を設定できます。


<shape xmlns:android="http://schemas.android.com/apk/res/android"
       android:shape="rectangle" >
    <solid android:color="?attr/colorPrimary"/>
</shape>

長方形の色は、テーマのcolorPrimary属性の値に依存します。属性は任意です。たとえば、リソースに属性を設定し、サブジェクトにその値を設定できます。

このようなトリックは、セレクターやベクター描画など、すべてのリソースで使用できます。これにより、設計がより構造化され、柔軟になります。すべてのアクティビティのテーマをすばやく変更できます。

概要


  • リソースレベルのテーマとスタイルはまったく同じですが、使用方法が異なります。
  • テーマには、ビューの意味やスタイルだけでなく、他のテーマを含めることができます。
  • スタイルはレイアウトのビューレベルで示されます。LayoutInflaterはスタイル値を計算し、それらをAttributeSetとしてViewコンストラクターに渡します。
  • テーマは、アクティビティ全体の外観をグローバルに定義できるメカニズムです。
  • ビューの階層の要素（および子孫）のテーマ値を変更できます。
  • テーマの値は、ビューだけでなくリソースでも使用できます。
