今日は3つのトピックを探ります：物理デバイスとしてのルーター、Ciscoルーターの簡単な紹介、ルーターの初期設定。このスライドは、典型的なCiscoモデル1921ルーターの外観を示しています。



多くのポートを持つスイッチとは異なり、一般的なルーターには接続用のポートが2つしかありません。この場合、これらはギガビットイーサネットポートGE0 / 0およびGE / 1とUSBコネクタです。ルータには、拡張モジュール用のスロットと、1つのUSBポートを含む2つのコンソールポートもあります。シスコルータの特徴は、スイッチの存在です。シスコスイッチにはスイッチがありません。通常、ルータの前面は、スライドの左下に示されているものと同じように見えます。ルータの背面には、ケーブルを接続するためのジャックがあります。この場合、GE0 / 0またはGE / 1スロットからのケーブルがスイッチに接続されています。

右下にNME-X 23-ES-1GP拡張モジュールを示します。これは、ダミーパネルを取り外すことでルータに挿入できます。これらのモジュールを使用すると、従来のCiscoルーターの機能を拡張して、ニーズを満たすことができます。ご存知のように、シスコ製品は複雑で幅広い機能を備えているため、非常に高価であり、ユーザーは必要以上の機能を備えたデバイスに過剰な支払いをする機会がありません。 2つのポート用のシンプルなルーターを購入したら、ネットワークの発展に応じて必要な拡張モジュールを購入できます。一般に、シスコのデバイスは多くの機能を備えています。ルーターはシスコが発明したものではありませんが、シスコが今日知っている会社にしたのはルーターでした。シスコは最高品質のルーターの量産を開始しました。これにより、これらの製品はネットワークデバイス市場での主導的地位を確立しました。
シスコはそれ自体をソフトウェア会社、つまりソフトウェア会社と呼んでいます。シスコのハードウェアと同様のハードウェアは、適切なハードウェアを購入することにより、中国などのどのメーカーでも製造できます。しかし、この会社のデバイスを本当の姿にするのはCisco IOSソフトウェアです。同社は、スイッチとルーターの両方のすべてのシスコデバイスで動作するこのオペレーティングシステムを本当に誇りに思っています。

シスコの最も重要な発明は、CEF拡張、つまりシスコエクスプレスフォワーディングです。非常に高速なパケット伝送を提供します。これは、ネットワークの技術的能力で可能な最大速度です。これは、スイッチがほぼネットワーク速度でパケットを送信するように強制するCisco ASICの専用集積回路-特定用途向け統合回路のおかげで可能になりました。
すでに述べたように、ルーターはソフトウェアデバイスに近いため、ルーティングの決定はCisco IOSオペレーティングシステムによって行われます。

あなたはコンピュータゲーム用の高価なグラフィックカードがあることを知っています。そのため、このようなカードがない場合、面倒な計算、3Dアニメーション、複雑なグラフィックス処理はすべて、オペレーティングシステムによって実行され、コンピュータプロセッサをロードします。独自のGPUプロセッサと独自のメモリを備えた強力なグラフィックカードを使用している場合、個別のハードウェアがグラフィック部分を処理するため、ゲームのパフォーマンスは何倍も向上します。

パケットスイッチングに関するすべての決定は、ソフトウェアがこれらの決定を行うルーターをロードすることなく、別個のハードウェアによって行われるため、スイッチは同様に機能します。シスコはCEFセミソフトウェア、セミハードウェアテクノロジーを使用しており、これによりルータはより迅速なルーティング決定を行う必要があります。この機能はCiscoルーターに固有のものです。

スイッチパラメーターの初期構成を実行する方法については既に検討しましたが、ルーターも同じように構成されているので、すぐに説明します。 Cisco Packet Tracerプログラムを開き、モデル1921ルーターを選択してから、このルーターのオペレーティングシステムがどのように起動するかを確認できるIOSコンソールウィンドウを開きます。
バージョン15.1をダウンロードしたことがわかります。これは、IOSの最新バージョンであり、メモリ容量は512 MB、CISCO 2911プラットフォームです。その後、オペレーティングシステムの他のパラメーター、IOSイメージテストが配置されています。もちろん、ライセンス契約やその他の類似したものがあります。



Cisco IOS専用の別のビデオを作成するか、このオペレーティングシステムのさまざまなサービスについて説明します。バージョン番号によって、このOSの機能を判別できると言えます。 15.1から、iOSのすべてのバージョンがユニバーサルになりました。つまり、ユーザーが取得したライセンスに応じて、ユーザーはシステムのさまざまな機能を利用できます。たとえば、ネットワークセキュリティを強化する必要がある場合は、セキュリティサービスライセンスを購入します。音声サービス機能が必要な場合は、音声サービスライセンスなどを購入します。

バージョン15.1より前のルータには、Basic、Security、Enterprise、Voice Enableなどの異なるバージョンのOSがありました。友人のルーターにエンタープライズIOSバージョンがあり、私にはベーシックIOSバージョンがあったとしても、シスコがOSライセンスの概念を使用していなかったため、友人のバージョンを取得して私のルーターにインストールすることを妨げるものはなかったとします。

バージョン15.1から、同社はライセンスオプションの概念を適用し始め、対応するキーを購入するまで、オペレーティングシステムの追加サービスを使用できません。少し後で、シスコのライセンスポリシーを見るときに、IOSのさまざまなバージョンについて説明します。その間、これに注意を払うことはできず、ダウンロードログに直接移動できません。

ログの最後に、システムが起動されたハードウェアの説明が表示されます：プロセッサーブランド、3ギガビットインターフェイス、64ビットDRAM、256 Kb不揮発性メモリ。このメモリ量は少なすぎるように見えますが、ルーティングを決定するルーターにとっては、これで十分です。これらは完全に異なるものであるため、このメモリをコンピュータのメモリと比較しないでください。

Cisco IOSブートログは、「設定ダイアログを続行しますか？うーん、ダメ"。 「はい」と答えると、システムが一連の質問を案内し、デバイスの初期構成を実行するための質問に答えます。

CCNAコースを学ぶときは、これを行うべきではないので、この質問には常に「いいえ」と答えてください。もちろん、「はい」を選択して構成設定をスクロールできますが、実行方法がわからないため、「いいえ」を選択することをお勧めします。

[いいえ]を選択してRETURNキーを押すと、コマンドラインのプロンプトに移動して、さまざまなコマンドを入力できます。スイッチの場合と同様に、最初にコマンドRouter> enableを入力して、特権設定モードに切り替えます。次に、config t（configure terminal）と入力して、グローバル構成モードに入ります。

すぐにチームについて見ていきましょう。ホスト名を変更したいので、hostname R1コマンドを使用し、次に否定コマンドを使用します。最初に、do show ip interface briefコマンドを使用してルーターインターフェイスを表示するように依頼します。ギガビットイーサネットポート0/0が管理上のダウンモードになっていることがわかります。そのため、int gigabitEthernet 0/0を使用し、シャットダウンコマンドを使用しません。その後、ポートの状態がアップに変わります。ルーターのインターフェースのステータスをもう一度見ると、このポートのステータスが「オン」になっていることがわかります。ルーターに何も接続されていないため、プロトコルの状態はダウン状態のままであり、トラフィックがない場合はオフ状態です。しかし、トラフィックがルータポートに到着するとすぐに、プロトコルはステータスをアップに変更します。

次に、コンソールでパスワードを設定する必要があります。これを行うには、コマンドラインcon 0、password consoleを出力し、show runを実行して、コンソールのパスワードが設定されていることを確認します。パスワードの検証は、loginコマンドを入力した後にのみ実行されます。これで、ルータのコンソールポートがパスワードで保護されました。

パスワードの暗号化についてはすでにお話しました。誰かがこのデバイスの現在の構成にアクセスしたとします。その中にパスワードがはっきりと見えるので、この人はそれを簡単に盗むことができ、いつでもあなたはルータの設定に入り、システムをハッキングすることができます。

パスワードの暗号化を有効にする1つの方法は、service password-encryptionコマンドを使用することです。デフォルトでは、このコマンドは否定コマンドnoとともに使用され、no service password-encryptionという形式であるため、パスワードの暗号化は実行されません。グローバルコンフィギュレーションモードに入り、service password-encryptionコマンドを入力して、Enterキーを押します。このコマンドは、システムが私が設定したテキストパスワードを取得して暗号化することを意味します。

ここで、do show runコマンドを使用して現在の構成を確認し、パスワード行に移動すると、7番目のタイプのパスワードがランダムな数字のシーケンスの形式をとっていたことがわかります。さて、同僚の1人があなたの肩越しにこのパスワードを見ることができた場合、彼がこのシーケンスを覚えることは非常に困難になります。したがって、アクセスセキュリティシステムの防御の最前線を作成しました。



ただし、このパスワードをコピーして設定に移動し、パスワード文字列に貼り付けようとしても、システムは設定へのアクセスを許可しません。これは、この数字のセットがパスワード自体ではなく、暗号化された値であるためです。正しいパスワードはconsoleという単語で、入力するとコンソールポートにアクセスできます。したがって、誰かがこれらの番号をコピーしたとしても、デバイスにアクセスすることはできません。

ただし、攻撃者が必要とするのは、7番目のタイプのシスコパスワードを簡単に復号化できるサイトにアクセスすることだけであるため、実際には間違っています。サイトのページに移動し、コピーした番号を入力するだけで、復号化されたパスワード（この場合はコンソール）を取得できます。ハッカーはこの単語をコピーし、iOS設定に戻り、パスワードプロンプトに貼り付けるだけです。



この場合、単純なパスワードの有効化機能では必要なセキュリティが提供されません。セキュリティを提供する最良の方法は、enable secret ciscoコマンドを使用することです。その後、現在の構成を確認すると、パスワードの値がさまざまな文字のセットであることがわかります。この場合、5番目のタイプのシスコパスワードが使用されます。

このタイプのパスワードをオンラインで解読することは不可能であるため、デバイスのコンソールは完全に安全です。

次に、Telnetでパスワードを設定する必要があります。これを行うには、line vty 0 4コマンドを入力します。これにより、5人がこのルーターを使用できるようになり、password telnetコマンドを入力します。さて、誰かがTelnetプロトコルを使用してルーターに接続したい場合、彼はこのパスワードを入力する必要があります-単語telnet。

次に、スイッチがOSIレベル2に属しているため、スイッチに対して管理IP IPアドレスを構成しました。ただし、ルーターはレベル3のデバイスであるため、ルーターの各ポートには独自のIPアドレスがあります。



スイッチでは、VLAN1の設定、またはIPアドレスの登録が必要なその他のネットワークの設定に進みました。仮想インターフェースを作成し、それらにIPアドレスを割り当てました。しかし、ルーターの場合、これらのアドレスは物理ポートに割り当てる必要があるため、config tおよびint g0 / 0コマンドを入力します。次に、コマンドを使用して、VLANの場合とまったく同じようにIPアドレスを割り当てます。つまり、ip address 10.1.1.1 255.255.255.0コマンドを入力してから、no shutdownと入力します。

do show int briefコマンドを使用してポートの状態を確認すると、アドレス10.1.1.1がギガビットイーサネット0/0インターフェイスに割り当てられていることがわかります。そこで、IPアドレスを設定しました。
次に、ログオンバナーの設定に進みます。スイッチの場合と同じように、バナーのmotd＆コマンドを使用してから、「Welcome to NetworKing Router」などの任意のテキストを入力し、「アスタリスク」で下線を引いて、アンパサンド＆で閉じます。
さらに、ポートを無効にする場合は、Shutdownコマンドを使用します。設定を保存するには、copy running-config startup-configコマンドを使用します。現在の構成は、show running confコマンドを使用して表示できます。showstartup confコマンドを使用して、ブート構成を表示します。 「すぐに使える」新しいデバイスとデフォルト設定のブートを使用したため、ブート構成の表示要求への応答として、システムはまだ存在しないと応答します。

copy running-config startup-configコマンドを入力すると、システムは、書き換え可能なファイルがstartup-configブートパラメータファイルであることを確認するように求めます。ブート構成ファイルを上書きした後、show startup confコマンドで確認すると、デバイスの現在の状態のファイルが完全に繰り返されていることがわかります。ルーターをオフにしてから再びオンにすると、保存した設定を使用して起動します。

ルータステータスの確認は、show int briefコマンドを使用して行うのが最適ですが、すべてのポートのステータスを表示するshow intコマンドを使用することもできます。特定のポートのステータスを確認する場合は、show interface g0 / 0コマンドを使用できます。その後、システムはこのインターフェイスの完全な統計情報を表示します。

すでに述べたように、ルーターの最も重要な部分はルーティングテーブルです。 show ip routeコマンドを使用して表示できます。
