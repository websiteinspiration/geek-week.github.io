スタックカナリア


カナリアは、バッファオーバーフローを監視するためにスタック上のバ​​ッファと制御データの間に配置される既知の値です。バッファオーバーフローの後、破損する最初のデータは通常カナリアです。したがって、カナリアの値がチェックされ、チェックが失敗した場合は、バッファオーバーフローが通知されます。カナリアには3つのタイプがあります。

プログラムには、キャプチャチェックが組み込まれています。興味深い関数である2つの関数、my_hash（）とprocess_hash（）が表示されます。最初のものから始めましょう。



変数のタイプを再定義して、コードを分析しやすくしましょう：



したがって、関数は乱数を返します。同時に、v3はアドレスEBP-0xCのデータです。別の関数を見てみましょう。



ここで、変数v4はアドレスEBP-0xCで値を取得し、この値を使用して関数の出口でけんかをします。次に、v3変数に512バイトが割り当てられ、キーボード入力がg_buf変数に読み込まれます。その後、g_bufからの文字列がBase64でデコードされ、v3に書き込まれます。 v3からmd5ハッシュが計算されます。したがって、g_bufへの入力とv3へのコピーは制限されないため、バッファオーバーフローが発生します。スタックを見てみましょう。



変数v3は、バッファの後にあるスタックカナリアです。プログラムはシステムの機能も呼び出します。それでは、エクスプロイトのテンプレートを作成します。

from pwn import *

p = remote('127.0.0.1', 9002)
p.recvuntil('captcha : ')
captcha = int(p.recv())
p.sendline(str(captcha))

p.interactive()

まず、ペイロードを理解します。パラメータ「/ bin / sh」を使用してシステム関数を呼び出す必要があります。しかし、スタックは実行可能ではないので、システム関数を呼び出して、プログラム内のそのアドレスに制御を渡し、パラメーターとして、アドレスを「/ bin / sh」の行に渡します。これをg_bufに書き込みます。

したがって（スタックを見て）：512バイトのガベージを書き込み、次にカナリアの値を4バイト、さらに12バイトのガベージを書き込む必要があります。ここで、retの場合、システム関数のアドレス（4バイト）、文字列“ / bin / sh”（4バイト）のアドレス、および文字列“ / bin / sh”自体を指定する必要があります。

次に、未知数を見つけます。システムコールアドレスです。



これは0x8049187です。また、文字列のアドレスは「bin / sh」です。これを行うには、base64エンコーディングを考慮して、g_bufアドレスの行「/ bin / sh」にバイト数を追加する必要があります。これは元の値の4/3です。



つまり、行のアドレス：0x804b0e0 +（512 + 4 + 12 + 4 + 4 + 1）* 4/3 = 0x804b3ac。ペイロードを構成します。
