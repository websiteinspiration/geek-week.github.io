リモートアクセスを整理してハッカーの被害を受けない方法

会社の経営陣が緊急にすべての従業員にリモートでアクセスすることを要求すると、セキュリティの問題がバックグラウンドに後退することがよくあります。その結果、攻撃者は活動のための優れたフィールドを得ます。


では、企業リソースへの安全なリモートアクセスを整理する場合、何が必要で何ができないのでしょうか。これについてはカットの下で詳しく説明します。

安全なリソース公開


Webアプリケーションファイアウォールを介してWebリソースを公開します（一般的な人々-WAF）。迅速な導入と基本的な保護には、標準のOWASP Top 10セキュリティプロファイルを使用するだけで十分です。最初は、誤検知イベントをキャッチするという点で、多くのナットを締める必要があります。現在、WAFがない場合-絶望しないでください！WAFのある試用版をテストしている場合は、それを使用してこの問題を解決するか、オープンソースソリューションのNginx + ModSecurityをインストールしてください。

WAFを使用できない場合は、急いで（可能な限り）アプリケーションをHTTPSに転送し、公開されたアプリケーションのすべてのパスワード（ユーザー、管理者）が会社が確立したパスワードポリシーに準拠しているかどうかを確認します。つまり、オペレーティングシステムとCMSの鮮度と、必要なすべてのパッチの存在を確認することを忘れないでください。将来の公共サービスのすべての領域をサニタイズしてください。 Kali Linuxを展開し、脆弱性をスキャンするための組み込みのユーティリティセットを使用します。これに時間がない場合は、公開されている脆弱性スキャナー（Detectify、ImmuniWebなど）のいずれかを使用します。

何をしないのですか？何千もの脆弱性が存在する可能性があるHTTPのすばらしい自作の尻をインターネットに表示しないでください。ブルートフォースを利用したくない場合は、サーバーやネットワーク機器へのSSHアクセスを設定する必要はありません。また、RDPをターゲットステーション（hello、esteemaudit）に直接公開する必要もありません。アクセスを提供する必要がある特定のアプリケーションについて疑問がある場合は、VPNの背後に配置してください。

VPN


リソースの公開がわかったので、アクセスを公開できなかったサービスに移りましょう。これを行うには、VPNを編成する必要があります。

VPNを構成する際に考慮すべきことは何ですか？

まず、職場にVPNクライアントソフトウェアをすばやく展開できるか、クライアントレスアプローチをより適切に使用できるかを評価します。リモートアクセスを整理できるVPNゲートウェイまたはファイアウォールはありますか？

たとえば、ネットワークにバンドル（NGFW / NGTP / NGTX）を備えたFortinetまたはCheck Pointファイアウォールがある場合、おめでとうございます。IPsecVPN機能のサポートは標準であり、追加のものを購入またはインストールする必要はありません。あとは、クライアントを職場に配置してファイアウォールを構成するだけです。

現在VPNゲートウェイまたはファイアウォールがない場合は、任意のサーバーにすばやく展開できるオープンソースソリューション（OpenVPN、SoftEther VPNなど）を検討してください。幸いにも、インターネットはたくさんあります。

さらに、アカウントを集中管理するために、VPNゲートウェイをAD / RADIUSと統合することが望ましいです。また、パスワードポリシーを確認し、ブルートフォースに対する保護を設定することを忘れないでください。

リモートアクセスクライアントをユーザーのワークステーションにインストールする方法に従う場合は、使用するVPNモード（フルトンネルまたはスプリットトンネル）を決定する必要があります。特定のユーザーグループのアクセスに機密情報や非常に重要な情報の操作が含まれる場合は、フルトンネルモードを使用することをお勧めします。したがって、すべてのトラフィックはトンネルに送られ、ユーザーのインターネットアクセスはプロキシ経由で調整できます。必要に応じて、DLP経由でトラフィックをリッスンすることもできます。それ以外の場合は、通常のスプリットトンネルモードに制限することができます。このモードでは、トラフィックは会社の内部ネットワークにのみトンネルにルーティングされます。

職場の安全に移りましょう。

リモートユーザーのワークステーションのセキュリティを評価します。必要なすべてのセキュリティ機能（ウイルス対策、ホストベースのIPS /サンドボックスなど）がインストールされたゴールドイメージがインストールされたワークステーションをワークステーションに提供しますか？この質問に対する答えがホームデバイスである場合、リモートアクセスを許可した後、IDS / IPSを使用してNGFWに、そして理想的にはネットワークサンドボックスにもトラフィックを送る方が良いでしょう。

優れたオプションの1つは、仕事用の特定のアプリケーション（ブラウザー、電子メールクライアントなど）をVDIで公開することです。これにより、使用される特定のアプリケーションへのアクセスのみが許可されます。

会社でリムーバブルメディアの接続が許可されていない場合、リモートアクセスの場合は、このことを忘れないでください。これにより、新しく発行された企業のラップトップの可能性が制限されます。

いつものように、安全でないプロトコルとサービスが無効になっていることを確認します。ディスクの暗号化をオンにすると便利です（ユーザーがコワーキングで仕事に出かけ、会社のラップトップが盗まれた場合はどうなりますか）。特権アクセス権を選択することを忘れないでください（ラップトップが会社の場合）。

認証


リモートアクセス（AD / RADIUS）による集中アカウント管理を使用します。また、アイデンティティストアが使用できなくなるシナリオを考慮することを忘れないでください（たとえば、追加のローカルアカウントを作成します）。

クライアント証明書を使用することをお勧めします。自己署名証明書はMicrosoft CAでも発行できます。

予期しない状況が原因で、リモートユーザーが資格情報を盗まれたと想定します。2要素認証は、この惨劇（モバイルデバイスでのOTPプッシュ、SMS）に対処するのに役立ちます。ただし、企業の電子メールによる2要素認証はお勧めしません（多くの場合、リモートアクセスによる認証では、電子メールと同じアカウントが使用されるため、2番目の要素は簡単に引き出されます）。2要素認証をすばやく整理する必要がある場合は、公共サービス（Google認証システムなど）の方向性を調べることができます。

搾取


IT部門がリモートワークステーションをどのように操作し、ユーザーが日常の問題を解決できるようにするかを検討します。明らかに、リモートサポートスタッフは、ユーザーワークステーションへのリモートアクセスを必要とします。

ワークステーションがゴールデンイメージから「こぼれる」ことをお勧めします。従業員が何か間違ってインストールしたり、ランサムウェアを捕まえたりしたため、従業員のホームコンピューターを復元する必要はありません。従業員の自宅のPCで頭痛の種にならないように、既知の容量とインストールされたソフトウェアの構成を備えた企業のラップトップを配布することをお勧めします。これらは子供が使用できるため、システムによって処理速度が大幅に低下するか、必要な保護機器がない場合があるためです。

会社がセキュリティポリシーを持っていることをリモート作業に切り替える前にユーザーに思い出させることは役に立ちます：通常のユーザーが自宅で昼食時にどのようにリラックスしたいかは決してわかりません。
