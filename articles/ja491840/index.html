サービスワーカーとは何ですか？


Cersis Workerは、ブラウザーによってバックグラウンドプロセスで実行されるスクリプトです。Service Workerは、相互作用するページまたはサービスを提供する（提供する）ページから完全に独立していることに注意してください。

本質的に、Service Workerは、Webアプリケーション、ブラウザ、ネットワーク間のプロキシサーバーです。

Service Workerを使用すると、Webアプリケーションをネイティブアプリケーションのように機能させることができます。

サービスワーカーに関するいくつかの事実


  • Service WorkerはDOMに直接アクセスできません。これを行うには、postMessagesインターフェースを介して要求に応答するメカニズムを使用します。
  • サービスワーカーは、使用されていないときに強制的にキャンセル（停止）されます。つまり、イベント駆動型です。
  • サービスワーカーは、約束（約束）の使用を想定しています。
  • 優れた機能により、Service WorkerはHTTPS経由でのみ使用できます。ローカルサーバーでは、HTTPSがなくても実行できます。

一般的な使用例


サービスワーカーのしくみがわかったので、次に、それらが使用されている場所について話します。

キャッシング


上記のように、Service Workerはキャッシュに使用できます。ここではいくつかの例を示します。

  • キャッシングのみ-変化しない静的コンテンツがあります。
  • ネットワークまたはキャッシュ-クイックダウンロード条件でユーザーに関連するコンテンツを表示したい。
  • キャッシュと更新-コンテンツを即座に表示し、サーバーとの定期的な同期を気にしません。
  • キャッシュ、更新、再起動-可能な限り迅速にコンテンツを表示し、暗黙的に個々の部分を更新して、「シームレス」な方法で表示したい。

Webプッシュ


Webプッシュを使用すると、アプリケーションはプッシュ通知を送信し、これらの通知に応答して受信したコンテンツを表示できます。

  • コンテンツのプッシュと更新-利用可能なコンテンツを共有（配信および受信）したい。
  • プッシュとコンテンツ-テキストだけでなく、メッセージを充実させる他のタイプの情報も操作したい。
  • 飽和プッシュ-イメージ、ブートプロセス、および配信するメッセージを改善するその他の要素を表示します。
  • プッシュとクライアント-アプリケーションの状態に応じて通知を表示します。

サービスワーカーの登録


再びサービスワーカーのライフサイクルの図を見ると、まずインストールする必要があることがわかります。これを行うには、登録する必要があります。

//   
if('serviceWorker' in navigator){
    console.log('- ')
    //     
    //      "load"
    window.addEventListener('load', () => {
        //  -
        navigator.serviceWorker
        .register('/service-worker.js')
        .then(registration => {
            //   
            console.log(`-  , scope: ${registration.scope}`) // scope -  ,     -
        })
    })
    .catch(error => {
        //

次は何ですか？


次に、すべてのファイルをキャッシュする必要があります。キャッシュするファイルを選択できます。これは次のようになります。

//  
const CACHE_NAME = 'example.com-v1'
//    
const cacheAssets = ['index.html', 'about.html', 'js/main.js']
//    "install"
self.addEventListener('install', e => {
    console.log('- ')
    e.waitUntil(
        caches
            .open(CACHE_NAME)
            .then(cache => {
                console.log('  -:  ')
                cache.addAll(cacheAssets)
            })
            .then(() => {
                self.skipWaiting()
            })
    )
})

ここで何が起こるかです：

  1. キャッシュの名前を決定します（example.com-v1）。
  2. キャッシュするファイルを選択します。これを行うには、配列を作成します。
  3. 「インストール」イベントハンドラー内で、ブラウザーにPromiseの完了を待つように指示し、キャッシュを開きます。キャッシュは「example.com-v1」という名前で保存されます。
  4. 最後に、選択したファイルをキャッシュに追加します。

未使用のキャッシュを削除する


次に、古いキャッシュバージョンを削除する必要があります。

//    "activate"
self.addEventListener('activate', e => {
    console.log('- ')
    e.waitUntil(
        caches.keys().then(cacheNames => {
            return Promise.all(
                cacheNames.map(cache => {
                    if(cache !== CACHE_NAME){
                        console.log('   ')
                        return caches.delete(cache)
                    }
                })
            )
        })
    )
})

応答を受け取る


キャッシュされたコンテンツを取得する方法がない場合、上記のいずれも意味がありません。

これは、フェッチイベントハンドラーを使用して取得できます。

//    "fetch"
self.addEventListener('fetch', e => {
    e.respondWith(
        fetch(e.request)
            .then(res => {
                const copyCache = res.clone()
                caches.open(CACHE_NAME).then(cache => {
                    cache.put(e.request, copyCache)
                })
                return res
            })
            .catch(error => caches.match(e.request).then(res => res))
    )
})
