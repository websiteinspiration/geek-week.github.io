Gula ular atau tulis kisaran Anda dalam JavaScript

Banyak orang menyukai Python ... Para pemula dikagumi oleh tidak adanya titik koma, dan yang mahir menikmati kesederhanaan yang nyata. Hari ini kita akan berbicara tentang bagaimana menerapkan JavaScript mirip dengan kesederhanaan Python, dan khususnya fungsi jangkauan .

Dalam Python, fungsi rentang dapat diulang atau, misalnya, dikonversi ke daftar - array (kisaran (mulai, akhir)) .

Tetapi pertanyaannya adalah, mungkinkah JavaScript membuat sesuatu yang serupa dan pada saat yang sama membuat solusinya tampak asli dan sederhana?

Hal pertama yang terlintas dalam pikiran adalah menulis kelas yang serupa:

function range(from, to, step = 1){
    this.current = from
    this.to = to
    this.step = step

    this.next = () => this.current++ % to
}

Ya, saya menulis bahwa selanjutnya mengembalikan nilai yang dipangkas, tetapi ini juga merupakan masalah yang perlu dipecahkan bagaimana memecahkan - melempar kesalahan ketika pergi ke maksimum atau melakukan sesuatu yang lain?

Secara umum, solusi tersebut tidak akan menambah kenyamanan sama sekali dan akan lebih mudah untuk menulis standar untuk , misalnya, Anda dapat menambahkan akhir fungsi dan setelah setiap iterasi cek jika kita pergi ke luar negeri, tetapi tidak ada gunanya dalam mengembangkan solusi ini.

ES6 memperkenalkan tipe data primitif baru, Simbol. Dia membuka di depan kita pintu menuju metaprogramming.

Untuk mengatasi masalah kami dengan jangkauan, kami akan menggunakan Symbol.iterator . Itu memungkinkan untuk membuat objek iterator menggunakan protokol tertentu.

Agar objek menjadi iterator, Anda perlu mendefinisikan fungsi [Symbol.iterator] () di dalamnya , tetapi karena kami ingin membuat fungsi umum, bukan kelas, kami akan memiliki fungsi yang mengembalikan objek iterator.

Begini tampilannya:

function range(from, to, step = 1){
    return {
        [Symbol.iterator](){
            return {
                current: from,
                to: to,
                from: from,
                step,
                next(){
                    const it = { done: this.current >= this.to, value: this.current }
                    this.current += this.step
                    return it
                }
            }
        }
    }
}

Apa yang terjadi? Kami memiliki fungsi rentang , ia mengembalikan, sama saja, objek iterator kami, di mana fungsi [Symbol.iterator] () mengembalikan sebuah objek. Yang paling penting adalah bahwa objek ini berisi fungsi berikutnya , jika tidak maka objek tidak akan dapat diubah dan kesalahan akan muncul ketika mencoba menggunakannya. Fungsi selanjutnya harus (!) Mengembalikan objek dengan properti yang dilakukan dan nilai , di mana dilakukan (bool) memberi sinyal akhir dari iterasi, dan nilai berisi nilai saat ini.

Pada prinsipnya, semuanya berfungsi, dan Anda sudah dapat menulis sesuatu seperti python:

    for(let i of range(0, 10, 2))
        console.log(i)

Kesimpulan:

0
2
4
6
8

Kita bisa berhenti di sini ... Tetapi Anda dapat menulis solusi yang lebih elegan menggunakan fungsi generator:

function range(from, to, step = 1){
    return {
        *[Symbol.iterator](){
            for(let val = from; val < to; val += step){
                yield val;
            }
        }
    }
}

Nilai tambah lainnya adalah kenyataan bahwa kami memiliki peluang lain untuk iterator. Misalnya, Anda dapat dengan mudah mengkonversi rentang kami ke array:

[...range(0, 10, 2)]
// [ 0, 2, 4, 6, 8 ]
