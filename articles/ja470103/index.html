最初から単純なクリッカーゲームを作成する

ここ数年、「クリッカー」などのジャンルのゲームが人気を集めています。私自身はそれらをプレイすることに非常に興味がありますが、ゲームを作成することも同様に興味深いです-私自身クリッカー。Unityのようなエンジンのおかげで、初心者でも傑作を作るのは難しくありません。仕事に取り掛かりましょう。

何をすべきですか？


  1. なぜ、そして何をクリックすべきかを考えてみてください。
  2. クリックするとどうなりますか？
  3. クリックしないとどうなりますか？

私がこれらの質問に答えたとき：

  1. 画面上の任意の場所をクリックする必要があります。
  2. クリックすると、オブジェクトが変化するはずです。
  3. 無視した場合-損失。

ゲームは何についてですか？


プレイヤーは、上から何が落ちるかに応じて、剣か盾のどちらかを時間内に選択する必要があります。つまり、剣が落ちる-私たちは剣を選び、盾は落ちる-私たちは盾を選ぶ。間違った選択をすると、プレイヤーは単に負けます。つまり、すべてが非常に単純ですが、実装ではさらに単純です。

初心者はどこで美しいスプライトと背景を取得できますか？答え：自分を描きなさい。

Adobe Illustratorで描画するため、複雑なことは何もありません。あなたが絵を描く経験がなくても、すべてを詳細に説明するので、私たちは再び動揺しません。

私たちは何が必要なのか：

  1. 剣
  2. シールド
  3. 背景
  4. 葉と枝

もういい。

まず、スプライトを投げて、剣と盾を階層ウィンドウにドラッグします。それだけでなく、それらからプレハブを作成する必要があるためです。



  1. Rigidbody 2Dを追加し、Gravity Scaleフィールドの値を「0」に設定して、シールドが1か所に立つようにします。
  2. Circle Collider 2Dを追加し、Is Triggerチェックボックスをチェックしてアクションを追跡します。

プレハブの場合と同じアクションで、シールドが落ちるように「重力のスケール」フィールドに値「1」を設定する必要があるだけです。そして、それをPrefabsフォルダーにドラッグします。剣で同じアクション。できた！

次に、先ほど描いたスプライトでゲームを飾ります。最初に投げるのは背景です。

次に、画面全体を占めるボタンを作成します。階層内にボタンを作成するには、UI->ボタンをクリックします。さらに：

  1. 色をクリックします。
  2. ボタンの透明度を担当するフィールドAに値「0」を設定します。
  3. ボタンが希望どおりになっていることを確認します。

そして最後に、プログラミングに取り掛かりました。

using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.SceneManagement;
using UnityEngine.UI;

public class MainScript : MonoBehaviour
{
	public GameObject[] prefabs;    		// 
	public GameObject[] players;    		//  
	public static int score,record,n,k = 0;	//, , "n,k" - 
	public Text scoreT;				//  

    void Start()
    {
		score = 0;
		Time.timeScale = 1;								
		StartCoroutine (PrefabsCreate ());
		scoreT = scoreT.GetComponent<Text> ();
		record = PlayerPrefs.GetInt ("Record");
    }

 
    void Update()
    {
		n = Random.Range (0, prefabs.Length); //    
                                        // 0  - 
		scoreT.text = score + "/" + record;	  //     
		if (score >= record) {							
			record = score;
			PlayerPrefs.SetInt ("Record", record); //  
		}
			
    }

	IEnumerator PrefabsCreate(){	//   
		while (true) {
			Instantiate (prefabs [n], new Vector2 (0, 6.5f), Quaternion.identity);
			yield return new WaitForSeconds (0.5f);
		}
	}

	public void BtnClick(string btn){			//   
		if(btn == "Click"){
			Time.timeScale = 1;
			players [k].SetActive (false);      
                     //   (, ...)
			k++;										
			if (k >= prefabs.Length) {					
				k = 0;                 
                               //  ,  k  - 
			} 
			players [k].SetActive (true);
                        //   (, ...)
		}
		
	}

}

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class PlayerScript : MonoBehaviour
{

	void OnTriggerEnter2D(Collider2D other){//   (, )
//    
		if (other.gameObject.tag != gameObject.tag) {	
			Time.timeScale = 0;
			MainScript.score = 0;
		} else {
			MainScript.score++;
			Destroy (other.gameObject);//  
		}
	}
}

これらの方法で何が起こっているのですか？

（）を開始します。ゲームを開始するか、シーンを再開するたびに開始されます。このメソッド内に記述されるすべてのことは、一度だけ実行されます。したがって、ここに書き込みます。ゼロに等しいアカウント、コルーチンの起動（プレハブの作成に責任があるもの）、時間の値。

（）を更新します。フレームごとに1回実行します。常に更新されているものをここに書き留めておくと便利です。私たちの場合：スコアとレコードのインジケーター、ランダムな値nは作成されたものに依存します-剣または盾。

BtnClick（）。この方法が「パブリック」である理由を尋ねるかもしれません。 Unity自体に反映する必要があり、クリックするボタンを追跡するために「クリック」という単語を綴ることができます。ただし、ボタンは1つしかありませんが、もう少し知っていても害はありません。



ここでは、ボタンを押した後に何が起こるかを書きます。押すと、剣から盾に、またはその逆に変わります。失うとゼロにリセットされるため、時間の値を「1」に設定します。

OnTriggerEnter2D（）。オブジェクトの衝突をチェックします。両方のオブジェクトのタグが一致する場合、スコアが1つ増加し、プレハブが破棄されます。タグが異なる場合、損失-時間は停止し、スコアはリセットされます。

これらと同じタグを追加する必要があります。そしてここではそれは非常に簡単です：

  1. インスペクタウィンドウで、タグをクリックします。
  2. 一番下のドロップダウンリストで、[タグの追加]をクリックします。
  3. プラス記号をクリックし、必要に応じてタグに名前を付けます。



作成後、[タグ]をクリックして、目的のタグを選択します。

スクリプトを必要なオブジェクトに転送し、いくつかのフィールドに入力する必要があります。空のオブジェクトを作成し、「メインスクリプト」スクリプトを追加し、剣と盾に「プレーヤースクリプト」を追加します。



自信を持って、ゲームの準備はできていると言えます。私たちは最初の作品を発表し、楽しんでいます。
