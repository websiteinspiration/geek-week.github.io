Visual Studio for Mac：5つのクールな新機能

過去1年間、Visual Studio for MacチームはIDEのエディターを更新して、エディターをより速く、よりスムーズに、より生産的にしました。これは、WindowsのVisual Studioと同じエディターバックエンドに基づくネイティブmacOSエディターインターフェイスを作成することで実現しました。バージョン8.1では、新しいC＃エディターが導入されました。その後、8.2で新しいXAMLエディターが続きました。さらに最近では、バージョン8.3の新しいエディターを使用するようにWeb言語を更新し、1年前に予定していたプロセスを完了しました。

この功績を祝うために、新しいエディターの設計と実装、およびVisual Studio Code Editors for Macの5つのお気に入りの新機能について詳しく説明します。

Visual Studio for Macの更新されたエディターは、Visual Studio for Windowsの共通言語サービスに基づいています。これは、WindowsのVisual Studioバージョンをサポートする同じバックエンドがmacOSバージョンもサポートすることを意味します。これにはすべて、IntelliSense、Roslyn、テキストベースのロジック、および舞台裏のすべての言語サービスが含まれます。WindowsとmacOSで一般的ではない唯一の部分はユーザーインターフェイスレイヤーで、これは各プラットフォームにネイティブのままです。

macOSの場合、これは、CocoaやCoreTextなどのmacOSフレームワークを使用してユーザーインターフェイスを拡張することを意味します。ネイティブユーザーインターフェイスの使用、およびネイティブ入力メソッドを使用する機能のおかげで、右から左へのスペル、フォントの合字、およびその他の高度なグラフィック機能を備えた言語がサポートされています。



IDEに新しいエディターの機能が追加されたので、私の最も人気のある5つの新しいエディター機能を見てみましょう。本日お伝えしたいすべての機能は、開発体験をより生産的で面白く、刺激的なものにすることを目的としています。私たちが好きな方法でそれらを使用して楽しんでください！



複数の挿入ポイントを使用した編集（マルチキャレット）


マルチキャレットを使用すると、編集したファイルに任意の数のキャレット（テキスト挿入ポイント）を挿入できます。これは、マウスクリック、control-option-click、またはキーボードを使用して手動で実行できます。キーボードを選択するときは、パターンマッチングを使用して次のマッピングを挿入するか（Option + Shift +。）、マッピング全体を貼り付けることができます（Option + Shift +;）。Option + Shift +で

挿入された最後のキャレットを削除したり、Option + Shift + /で最後のキャレットを下に移動したりすることもできます。下のGIFでは、Option + Shift +を使用しています。次の適切なキャレットを2回挿入すると、このページで「double」の3つのインスタンスすべてを編集できます。

マルチキャレット編集は、一度に複数の行を編集するのにかかる時間を大幅に短縮できる非常に強力な機能です。たとえば、複数の変数のプレフィックスを変更する必要がある場合、または特定のvar宣言を厳密に型指定された宣言に切り替える必要がある場合、複数のキャレットを使用して編集すると、これを簡単に行うことができます。


 

IntelliSenseタイプフィルタリング


次に強調したい機能は、IntelliSense Type Filteringです。 IntelliSenseタイプフィルタリングを使用すると、補完リストを補完タイプでフィルタリングできます。たとえば、完了リストにクラスのみを表示したい場合は、クラスアイコンをクリックするか、「option-c」ホットキーを使用できます。

フィルタリングできるタイプの完全なリストと、Visual Studio for Macのドキュメントにある対応するアイコンとホットキーがあります  。以下のGIFでは、IntelliSenseタイプのフィルタリングを使用して、リストをインターフェイス、構造体、そして最後にデリゲートに集中させています。



この機能は、必要な要素の正確な名前を思い出せない場合、または特定のタイプにのみ焦点を当てたい場合に非常に役立ちます。また、次のお気に入りの機能である[インポートアイテムの表示]と組み合わせると非常にうまく機能します。

インポート項目を表示


多くの場合、プロジェクトで作業しているとき、特定のタイプのコードファイルにインポートする必要がある正確な名前空間を常に思い出せるとは限りません。これはしばしばパニックにつながり、必死にどこでも検索して必要なインポートを見つけさせます。次の関数は、既にインポートした完了だけでなく、インポート可能な完了も表示することで、この問題を修正します。さらに、まだインポートされていない補完の1つを最終的に選択すると、usingステートメントがコードファイルのヘッダーに追加されます。

下のgifに「System.ComponentModel.DataAnnotationsを追加します「インポート項目の表示」機能を使用してプロジェクトに。また、まだインポートされていないアイテムについては、完全な名前空間がタイプの横に示されているため、システムがタイトルに追加しようとしているものを簡単に確認できます。 現在、インポートアイテムの表示はデフォルトで無効になっていますが、Visual Studio>設定>テキストエディター> IntelliSenseを開き、「インポートアイテムの表示」をオンにすることで簡単に有効にできます。





右から左へのコードの書き込みとネイティブ入力のサポート


私たちのコミュニティからの人気のリクエストは、言語のサポートを右から左に追加すること、および双方向言語のサポートを追加することです。そして、そのようなサポートをVisual Studio 2019 for Macに追加できることを信じられないほど嬉しく思います。古いエディタでは、ペルシア語、ヘブライ語、アラビア語などの右から左への文字列を入力または貼り付けると、単語が反転しました。たとえば、単語helloはyuvtsvartsになり、テキストを上下逆に見えるように変更します。新しいエディターは、右から左への入力とすべてのタイプの双方向テキストをサポートします。



ネイティブ入力のサポートも追加しました。エディターはmacOSの組み込みツールを使用して作成されるため、エディターへのテキストの貼り付けは、他のmacOS組み込みアプリケーションへの貼り付けと同様です。つまり、アクセント記号や代替文字を長押ししたり、絵文字セレクターなど、macOSのすべての高度なテキスト入力機能にアクセスできます。

合字サポート


最近リリースされたCascadia Codeなどの合字をサポートするフォントを使用すると、Visual Studio for Mac 2019は通常の2文字のグリフの代わりに合字を自動的に挿入します。たとえば、二重等号（==）は、スペースのない拡張等号に変換されます。同様に、感嘆符と等号（！=）はスラッシュ付きの等号に変換され、「等しくない」記号をより正確に表します。
