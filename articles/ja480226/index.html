Xamarin.Forms 4.4で飛行を続ける

毎年多くのモバイルアプリケーションを作成している開発者と話し合い、「カルーセルを含まないモバイルアプリケーションを最後に作成したのはいつだったか思い出せません。」そこで、CarouselViewXamarin.Forms 4.4.0に新しいコントロールを導入できることをうれしく思います。またIndicatorView 、カルーセルを使用してページや要素を表示することもでき  ます。また  SwipeView 、のアイテムにコンテキストアクションを提供し  CollectionViewます。カルーセルのテーマは、画像のGIFアニメーションの新しいサポートのデモンストレーションなしでは完成しません。始めましょう。



Xamarin.Forms 4.4の機能


GIFアニメーションは、いくつかのアクションや詳細に注意を払いながら、ユーザーとのコミュニケーションを改善するためにアプリケーションにアニメーションを追加するための優れた簡単な方法です。もちろん、これは猫のgifにも最適です。 自動再生を有効または無効にするImageプロパティとともに、これで  IsAnimationPlaying十分です。

アニメーションのループはデフォルトで行われます。アニメーションを停止するには、IsAnimationPlaying Falseに設定  します。 それだけです、アニメーションがあります！これは小さな画像、特にアプリケーションに組み込まれている画像に最適です。GIF画像のループ設定が考慮されます。ユーザーエクスペリエンスを向上させるには、Androidで高速レンダリングを使用してください。





CarouselViewおよびIndicatorView


CollectionView新しい コントロールと同じ基本的なコントロールに基づいて構築されているCarouselViewため、水平方向にスクロールするか垂直方向にスクロールするかに関係なく、多機能カルーセルをアプリケーションに簡単に追加できます。CollectionView あなたと一緒  に：

  • ItemsLayout
  • ItemsSource
  • ItemTemplate
  • 空のテンプレート




<CarouselView HeightRequest="160">
    <CarouselView.ItemsSource>
        <x:Array Type="{x:Type x:String}">
            <x:String>h01.jpg</x:String>
            <x:String>h02.jpg</x:String>
            <x:String>h03.jpg</x:String>
            <x:String>h04.jpg</x:String>
        </x:Array>
    </CarouselView.ItemsSource>
    <CarouselView.ItemTemplate>
        <DataTemplate>
            <Image Source="{Binding .}"/>
        </DataTemplate>
    </CarouselView.ItemTemplate>
</CarouselView>

また、前の要素と次の要素を覗く距離などを管理するためのカルーセル固有のプロパティも取得します。



<CarouselView PeekAreaInsets="50"
              ...>

高度なヒント：上記の例では、前の要素と次の要素がスケーリングおよびシフトされていることにも注意してください。スクロールしながら動作を使用してレイアウトを変更することで、今すぐこれを実現できます。 4.5をリリースするときに、視覚的な状態を使用してこれを簡略化できます。

追加するには  IndicatorView、任意の場所に配置しCarouselView 、名前で関連付けます。

<CarouselView x:Name="walletCarousel">
// implementation here
</CarouselView>
<IndicatorView 
  IndicatorColor="LightGray"
  SelectedIndicatorColor="Black"
  IndicatorSize="10" HorizontalOptions="Center"
  IndicatorView.ItemsSourceBy="walletCarousel"/>




CarouselView and  IndicatorViewで 実行できるすべての強力な機能の  詳細については、ドキュメントを参照してください  。

注：以前にリリースされたNuGet  Xamarin.Forms.CarouselView（プレビュー）  はアーカイブされ、リリースされなくなります。このコントロールを使用した場合、NuGetを削除してコードの名前空間を更新できます。APIの一貫性を維持するように努めましたが、いくつかの設定が必要になる場合があります。 Github

スワイプビュー


このコントロールは通常、要素のリストにあり、ユーザーは任意の方向にスワイプして、編集や削除などの一般的なアクションのコンテキストボタンを開くことができます。このコントロールを使用するには、動作を追加する任意の要素をコントロールで囲みます。デフォルトで  SwipeItem は、タイトルとアイコン、コマンドまたはクリックハンドラを取得します。より複雑なものを追加したい場合は、テンプレートを使用してこれを行うことができます。



<SwipeView>
        <SwipeView.RightItems>
                <SwipeItems Mode="Execute">
                        <SwipeItem Text="Favourite" Command="{Binding Favourite}">
                                <SwipeItem.Icon>
                                        <FontIconSource Glyph=""/>
                                </SwipeItem.Icon>
                        </SwipeItem>
                </SwipeItems>
        </SwipeView.LeftItems>
 
        <!-- Swipeable content -->
        <Frame>
          // content here
        </Frame>
</SwipeView>

上記の  例は  、才能あるチームメンバーのハビエルスアレスルイスのブログで公開されました。

プラットフォームの仕様を使用して、Android  および  iOSのスワイプモードを制御することもできます  。詳細については、ドキュメントSwipeViewを  参照してください  。

Xamarin.Forms 4.4を今すぐ始めましょう


Xamarin.Forms 4.4を使い始めるには、NuGetパッケージマネージャーからアップグレードまたはインストールします。以前のバージョンからアップグレードする場合は、Xamarin.FormsとXamarin.EssentialsがAndroid 28.0.0.3のサポートに依存していることに注意してください。したがって、Androidサポートライブラリに依存する可能性のある、使用する他のパッケージと一緒に更新する必要があります。

新しいプレビュー機能を使用するSetFlags場合は、それらを有効にすることを忘れないでください。ソリューションのすべてのプラットフォームでこれをグローバルに行うには、以下をプラットフォームに追加しますApp.xaml.cs。

public App()
{
    InitializeComponent();
 
    Device.SetFlags(new[] { 
        "CarouselView_Experimental", 
        "IndicatorView_Experimental",
        "SwipeView_Experimental" 
    } );
 
    MainPage = new AppShell();
}
