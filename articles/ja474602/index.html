最小二乗法


最小二乗法に従って、偏差は二乗して計算する必要があります。このような手法では、偏差が反対の場合、偏差の相互返済が回避されます。たとえば、あるケースで偏差が+5（プラス5）で、他のケースでは-5（マイナス5）の場合、偏差の合計は相互に返済され、0（ゼロ）になります。偏差を浪費することはできませんが、モジュールのプロパティを使用すると、すべての偏差が正になり、私たちに蓄積されます。この点については詳しく説明しませんが、計算の便宜上、通常は偏差を二乗することを示します。

これは、二乗偏差（エラー）の最小合計を決定する助けを借りて、数式がどのように見えるかです。

ERR（バツ）=Σ私=1ん（a+bバツ私-y私）2=Σ私=1ん（f（バツ私）-y私）2→メートル私ん


どこ f（バツ私）=a+bバツ私 真の回答（つまり、当社が計算した収益）を概算する関数です。

y私 -これらは本当の答えです（サンプルで提供される収益）、

私-これは、サンプルのインデックス（偏差が決定された月の数）です。

差別化


微分は、関数の導関数を見つける操作です。

デリバティブとは何ですか？関数の導関数は、関数の変化率を特徴付け、その方向を示します。特定の点での導関数が正の場合、関数は増加し、そうでない場合、関数は減少します。そして、微分モジュロの値が大きいほど、関数の値の変化率が高くなり、関数のグラフの角度が急になります。

たとえば、デカルト座標系の条件下では、点M（0,0）での+25に等しい導関数の値は、特定の点で、値がシフトされたときにバツ 単位あたりの権利、値 y従来の25単位増加します。グラフでは、値の立ち上がりのかなり急な角度のように見えますy与えられたポイントから。

もう一つの例。-0.1の微分値は、オフセットしたときにバツ 従来の単位あたりの値 y従来の0.1単位だけ減少します。同時に、関数グラフでは、ほとんど目立たないチルトダウンを確認できます。山との類似を描くと、非常に急なピークをとらなければならない前の例とは対照的に、山から緩やかな傾斜を非常にゆっくりと下降しているように見えます:)

したがって、関数を微分することによってERR（バツ）=Σ私=1ん（a+bバツ私-y私）2 係数による a そして b、一次の偏微分の方程式を定義します。方程式を決定した後、2つの方程式のシステムを取得します。これを解決すると、係数のそのような値を選択できますa そして b所与の点での対応する導関数の値が非常に非常に小さい値で変化し、分析解の場合、それらはまったく変化しません。言い換えると、これらの点での偏微分の値はゼロになるため、見つかった係数の誤差関数は最小に達します。

したがって、微分の規則によれば、係数に関する1次の偏微分の方程式a 次の形式になります：

def Kramer_method (x,y):
        #   ( )
    sx = sum(x)
        #    (   )
    sy = sum(y)
        #      
    list_xy = []
    [list_xy.append(x[i]*y[i]) for i in range(len(x))]
    sxy = sum(list_xy)
        #   
    list_x_sq = []
    [list_x_sq.append(x[i]**2) for i in range(len(x))]
    sx_sq = sum(list_x_sq)
        #  
    n = len(x)
        #  
    det = sx_sq*n - sx*sx
        #   a
    det_a = sx_sq*sy - sx*sxy
        #   a
    a = (det_a / det)
        #   b
    det_b = sxy*n - sy*sx
        #   b
    b = (det_b / det)
        #   ()
    check1 = (n*b + a*sx - sy)
    check2 = (b*sx + a*sx_sq - sxy)
    return [round(a,4), round(b,4)]

勾配降下


まず、グラデーションとは何かを定義しましょう。簡単に言うと、勾配は、関数の最大成長の方向を示すセグメントです。勾配が見える上り坂を登るのと同じように、山の頂上に最も急な登りがあります。山の例を発展させて、私たちは実際にはできるだけ早く低地に到達するために最も急な下り坂、つまり最小値、つまり関数が増減しない場所が必要であることを思い出します。この時点で、導関数はゼロになります。したがって、勾配は必要ありませんが、反勾配が必要です。反勾配を見つけるには、勾配に-1（マイナス1）を掛けるだけです。

関数がいくつかの最小値を持つ可能性があることに注目し、以下に提案するアルゴリズムに従ってそれらの1つに下げると、見つかったものよりも低い可能性のある別の最小値を見つけることができなくなります。リラックスしてください、私たちは危険にさらされていません！私たちの場合、関数は1つなので、1つの最小値を扱っています。Σ私=1ん（a+bバツ私-y私）2グラフ上は普通の放物線です。そして、私たち全員が数学の学校コースから非常によく知っているはずなので、放物線には最低1つしかありません。

なぜ勾配が必要なのか、また勾配はセグメント、つまり特定の座標を持つベクトルであり、まったく同じ係数であることがわかった後a そして b勾配降下法を実装できます。

始める前に、降下アルゴリズムについてのほんの数文を読むことをお勧めします。

  • a b. , . , .
  • a 1- a. , , . , , , . , .
  • b: b.
  • 最小にジャンプしたり、遠方のスペースに飛んだりしないようにするには、ステップサイズを下降方向に設定する必要があります。一般に、ステップを正しく設定する方法と、降下中にステップを変更して計算コストを削減する方法についての記事全体を書くことができます。しかし、今は少し異なるタスクがあり、「ポーキング」の科学的方法を使用して、または一般の人々が経験的に言っているように、ステップサイズを確立します。
  • 与えられた座標から来た後 a そして b 導関数の値を差し引いて、新しい座標を取得します a そして b。計算された座標から、次のステップ（減算）を実行します。そして、必要な収束が達成されるまで、サイクルが何度も繰り返されます。

def gradient_descent_usual(x_us,y_us,l=0.1,tolerance=0.000000000001):
    #   ( )
    sx = sum(x_us)
    #    (   )
    sy = sum(y_us)
    #      
    list_xy = []
    [list_xy.append(x_us[i]*y_us[i]) for i in range(len(x_us))]
    sxy = sum(list_xy)
    #   
    list_x_sq = []
    [list_x_sq.append(x_us[i]**2) for i in range(len(x_us))]
    sx_sq = sum(list_x_sq)
    #  
    num = len(x_us)
    #   ,   
    a = float(random.uniform(-0.5, 0.5))
    b = float(random.uniform(-0.5, 0.5))
    #    ,     1  0
    #      
    errors = [1,0]
    #   
    #     ,        ,    tolerance
    while abs(errors[-1]-errors[-2]) > tolerance:
        a_step = a - l*(num*a + b*sx - sy)/num
        b_step = b - l*(a*sx + b*sx_sq - sxy)/num
        a = a_step
        b = b_step
        ab = [a,b]
        errors.append(errors_sq_Kramer_method(ab,x_us,y_us))
    return (ab),(errors[2:])

def gradient_descent_numpy(x_np,y_np,l=0.1,tolerance=0.000000000001):

    #    (   )

    #

    #

        a = a_step
        b = b_step
        ab = np.array([[a],[b]])

    return (ab),(errors[2:])

確率的勾配降下


確率的勾配降下法の動作原理をすばやく理解するには、通常の勾配降下法との違いを判断することをお勧めします。勾配降下法の場合、次の導関数の方程式でa そして a すべての属性の値の合計とサンプルで利用可能な真の回答を使用しました（つまり、すべての合計 バツ私 そして y私）確率的勾配降下法では、サンプルで利用可能なすべての値を使用するのではなく、疑似ランダムな方法で、いわゆるサンプルインデックスを選択してその値を使用します。

たとえば、インデックスが数値3（3）で決定される場合、値を取得しますバツ３=３ そして y３=4、8次に、導関数の方程式の値を代入して、新しい座標を決定します。次に、座標を決定したら、サンプルのインデックスを疑似ランダムに決定し、偏微分方程式のインデックスに対応する値を代入して、新しい方法で座標を決定しますa そして a等 収束を緑化する前。一見、これはうまくいくように見えるかもしれませんが、うまくいきます。確かに、すべてのステップでエラーが減少するわけではありませんが、傾向が確かに存在することは注目に値します。

通常よりも確率的勾配降下法の利点は何ですか？サンプルサイズが非常に大きく、数万の値で測定される場合、サンプル全体よりもランダムに数千個と言う方が処理がはるかに簡単です。この場合、確率的勾配降下法が起動されます。もちろん、私たちの場合、大きな違いはわかりません。

def stoch_grad_step_numpy(vector_init, X, ind, y, l):
    x = X[ind]
    y_pred = np.dot(x,vector_init)
    err = y_pred - y[ind]
    grad_a = err
    grad_b = x[1]*err
    return vector_init - l*np.array([grad_a, grad_b])
