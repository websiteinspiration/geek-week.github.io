レスポンシブフォント。レイアウトと最小値の間のテキストの適合

レスポンシブまたはレスポンシブフォントは、あらゆるWeb開発者の夢です。私は実装の多くのオプションを検討し、夕方に座って、信頼できる作業オプションを思いつきました。共有したいと思います。ボーナスSCSS Mixin！


だから挑戦。レイアウトに応じて値を指定するとともに、モバイルデバイス（iPhone 5（320px）など）でテキストまたはタイトルが適切に表示される最小値を指定します。この場合、フォントは画面の幅に応じて増減します。

レイアウトから初期値を取得します。たとえば、サイズが40pxのH1ヘッダーです。また、キャンバスレイアウト全体の幅（たとえば、1280px）も必要です。



最小フォント値を取得するには、レイアウトに従ってヘッダーを作成し、Chrome開発者ツール（F12）を使用して、目的の最小画面サイズとフォントサイズを選択し、テキストが適切に見えるまで値を変更します。



その後、CSSでfont-size：40pxではなく24pxを取得しました。次のように書いてください。

font-size: calc(24px + 16 * (100vw / 1280));

24pxが最小フォントである場合、16はレイアウト40のフォントと24の最小フォントの差です（40-24 = 16）。 100vwはビューポート（画面）の現在の全幅です; 1280はレイアウトの幅です。

レイアウトのサイズに完全に一致するレスポンシブフォントをテストして取得します。

ただし、320pxのサイズでは、望ましい結果、つまり24pxフォントは表示されません。これは、最小画面サイズを正確に320pxで指定しなかったためです。



この状況を解決するために、767pxのメディアリクエストを記述し、内部で式を変更します。

@media (min-width: 767px){
 h1 {
  font-size: calc(24px + (16 + 16 * 0.7) * ((100vw - 320px) / 1280));
 }
}

ここでは、ビューポートの全幅から320pxを減算し、さらに16を追加して70％（16 + 16 * 0.7）を追加

します。タスクの完全なコンプライアンスを確認します！同時に、ブレークポイントへの移行はシームレスに行われ、目立たなくなります。

もちろん、このすべてをSCSSミックスインで閉じることができます。

$maxWidth: 1280;
@mixin adaptiv-font($pcSize, $mobSize) {
	$addSize: $pcSize - $mobSize;
	$addMobSize: $addSize + $addSize * 0.7;
	@media (max-width: 767px) {
		font-size: calc(#{$mobSize + px} + #{$addMobSize} * ((100vw - 320px) / #{$maxWidth}));
	}
	@media (min-width: 767px) {
		font-size: calc(#{$mobSize + px} + #{$addSize} * (100vw / #{$maxWidth}));
	}
}

次のように接続します。

h1{
 @include adaptiv-font(40, 24);
}


あなたは完全にメディアクエリを取り除くことができます：

$maxWidth: 1280;
@mixin adaptiv-font($pcSize, $mobSize) {
	$addSize: $pcSize - $mobSize;
	$maxWidth: $maxWidth - 320;
	font-size: calc(#{$mobSize + px} + #{$addSize} * ((100vw - 320px) / #{$maxWidth}));
}

私はコメントでこの解決策をどのように改善できるかを嬉しく思います。ご清聴ありがとうございました！
