私は初心者のフロントエンド開発者です。現在、ミンスクのIT企業で勉強とトレーニングをしています。Web-uiの基本を学ぶには、例としてWebix JSライブラリを使用します。ささやかな経験を共有して、この興味深いUIライブラリの小さなチュートリアルとして保存したいと思います。

第六の問題


WebixはJavaScriptライブラリであり、操作は主にクライアント側で行われ、データを保存する必要がある場合にのみ、結果がサーバーに送信されます。ここでは、バックエンドに厳密な要件はありません。サーバーでどのデータが処理されるかは関係ありません。主なことは、特定の形式のリクエストを受け取り、正しい形式で回答を提供することです。プロジェクトに複雑なユーザー設定がないため、開発者が必要とするのは、サーバーに接続して単純なクエリを操作することだけです。

次のタスクを検討してください。

  • アプリケーションのサーバー側を接続します。
  • サーバー要求メソッド。
  • ダウンロード結果を追跡する方法;
  • 保存結果を追跡する方法。

この記事では、ドキュメントで詳細に説明されているテーブルウィジェットを使用しています。

保存結果の追跡


保存、追加、編集、削除などの操作でサーバーの応答を取得できます。テーブルウィジェットの例を考えてみます。

このため、DataProcessorモジュールを使用してサーバーとやり取りします。このモジュールのAPIを使用して、データストレージを微調整できます。保存する瞬間を追跡するだけでよいので、そのイベントの1つであるonAfterSyncを使用します。

DataProcessorモジュールは設定によってアクセスさwebix.dp(id)

れ、サーバーからの応答を受信して​​処理すると、onAfterSyncイベントが発生します。このイベントの場合、を介してメッセージを表示しますwebix.message()。保存後、メッセージが右上隅に表示され“Saved!”ます。

テーブルの最初の行を削除し、結果を確認します。



webix初期化後のscript.jsファイル内のイベントコード： 

let dp = webix.dp($$("film_list"));
dp.attachEvent('onAfterSync', () => {
    webix.message({
        text:"Saved!",
        type:"success"
    })
});

この場合のエラーは、onAfterSaveErrorイベントでキャッチできます。

dp.attachEvent("onAfterSaveError", function(id, status, response, details){
    webix.message("Error!");
});
