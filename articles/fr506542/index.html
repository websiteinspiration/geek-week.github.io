Enregistrement vidéo bricolage dans le cloud

Vous, cher lecteur, vous vous promenez dans la rue un soir d'été, ne touchez personne, et puis ... ils viennent sur vous (pah-pah-pah, comme on dit). Hooligans, juste des passants, des camarades spéciaux (comme ils chantaient dans une vieille chanson) - pas si important. Vous sortez votre téléphone et commencez à filmer ce qui se passe en vidéo. Ce n'est pas très agréable pour ceux qui le conduisent, et le téléphone vous est enlevé (ou ils sont légalement saisis - soulignons le nécessaire). Il n'y a pas de témoins, il n'y a plus d'enregistrement vidéo sur le téléphone, et il n'y a aucune preuve pour la police et le tribunal, respectivement.


Le moyen de sortir de cette situation est évident: la vidéo ne doit pas être enregistrée dans un fichier local sur votre téléphone, mais directement sur un serveur distant. Certes, il n'y a pas tellement de solutions logicielles prêtes à l'emploi pour mettre en œuvre cette idée (par exemple ici ): dans la plupart des cas, les applications proposées pour un téléphone mobile sont soit payantes soit fonctionnent très mal. Je ne considère pas les recommandations exotiques telles que "en cas d'attaque par des hooligans, commencez à diffuser sur YouTube", car dans une situation réelle, vous n'aurez tout simplement pas le temps de commencer à diffuser. De plus, la vidéo sera écrite dans le cloud de quelqu'un d'autre, et très souvent ce n'est pas bon.


Vous pouvez, bien sûr, apprendre Java ou Kotlin (et Swift en même temps) ou, au pire, maîtriser PhoneGap et écrire votre propre application. Cependant, tout est beaucoup plus simple: sous la coupe, une solution simple à ce problème consiste à utiliser l'API HTML5 vidéo / audio.


Contactez WebRTC


Bien sûr, WebRTC  est une chose très cool, vous permettant de diffuser directement sur le cloud. Cependant, la mise en œuvre d'une telle traduction est encore hémorroïdes, j'ai donc choisi la solution beaucoup plus facilement. La vidéo est écrite dans la mémoire RAM du téléphone (notez, pas sur la carte SD, mais uniquement dans la mémoire RAM) et toutes les minutes (par exemple), et aussi après l'enregistrement est envoyée au serveur. Autrement dit, même si les hooligans ont commencé à vous enlever votre téléphone, vous parvenez à appuyer sur le bouton d'arrêt et le dernier fichier vidéo va au serveur.

"use strict";

//      
const recTime = 60;

//    queryString
let pwd = location.search || 'a'; pwd = pwd.trim().replace('?', '');

const video = document.querySelector("video"),
      butt  = document.querySelector("button");

let media, playFlag = false;

//   
const play = async () => {
   try {
      //     ,   
      let c = /Android|iPhone/i.test(navigator.userAgent) ?
         {video:{facingMode:{exact:"environment"}}, audio:true} :
         {video:true, audio:true};

      //        
      let stream = await navigator.mediaDevices.getUserMedia(c);
      video.srcObject = stream;
      video.play();

      //      recTime 
      media = new MediaRecorder(stream);
      media.ondataavailable = d => {
         fetch("api.php", {
            method: "POST",
            headers: {"Content-Type": "video/webm", "X-PWD": pwd},
            body: d.data
         })
      };
      media.start(recTime * 1000);
   }
   catch(err) {alert(err);}
};

//    /
const go = () => {
   if (!playFlag) {
      butt.innerHTML = "&#9209;";
      play();
   }
   else {
      butt.innerHTML = "&#9210;";
      video.pause();
      video.srcObject = null;
      media.stop();      
   }
   playFlag = !playFlag;
}

Créez un raccourci pour ce fichier sur le bureau du téléphone (directement sur l'écran principal). En cas d'urgence, vous devez effectuer seulement trois actions:


  • allumez l'Internet mobile (s'il n'est pas allumé en permanence sur votre téléphone);
  • cliquez sur le raccourci alarm.html;
  • cliquez sur le bouton «Enregistrer» sur la page chargée.

La dernière action peut être exclue si vous corrigez légèrement le code frontal afin que l'enregistrement démarre immédiatement au chargement de la page.
