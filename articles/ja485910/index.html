Aws


AWSについて言えば、最初に、内部で使用する必要のあるいくつかのコンポーネントについて説明します。

AWS IAM

IAM（アイデンティティおよびアクセス管理）は、サインアップ時に最初に遭遇するものです。IAMを使用すると、アカウントのアクセス権を設定し、役割、グループ、ユーザーを管理できます。

Amazonでは、ベストプラクティスに従うことをお勧めしていますが、レッスン中はベストプラクティスに違反する必要があります。これらは次のプラクティスです。

  • 物理的な人ごとに-別のユーザー;
  • アプリケーションごと-個別の役割。
  • 発行されるアクセスはコミットされず、私たちは手探りでもなく、コードを使用していません。
  • 初期設定以外のrootアカウントは使用しないでください。誤ってどこかにパスワードを設定した場合、誰かがルートアカウントから仮想マシンを購入する可能性があります。また、必要なアラートをすべて設定したとしても、1,000ドルをすぐに受け取ることができます。

AWS EC2

EC2-Elastic Compute Cloud（IaaS）-仮想マシンをデプロイする機能を提供するWebサービス。EC2が提供するもの：

  • 私たちが使用する計算能力の管理（無料アカウントを登録すると、アクセスは1種類のインスタンスにのみ許可されます）;
  • Amazon Machine Image（AMI）スイート-アプリケーション、ライブラリなどを含む仮想マシンイメージ。

また、通常EC2で作業するには、Amazon S3（Simple Storage Service）-ファイルストレージを使用する必要があります。

環境から始まり、ネットワークアクセス設定で終わるすべてを自分で構成する必要があるため、EC2に直接触れることはありません。それにもかかわらず、内部的にはEC2が常に存在することを理解する必要があります。

AWS Elastic Beanstalk

Elastic Beanstalk-オーケストレーションサービス（オーケストレーションする内容に応じてPaaSまたはCaaS）。コンテナー化がコンテナー自体とその充填での作業である場合、オーケストレーションはコンテナーでの作業、たとえばメタレベルでの作業です。実際、オーケストレーションは、APIまたはコンソールを使用してコンテナー/仮想マシンを起動できるメカニズムです。

BeanstalkはOSに特定のプログラミング言語、Webサーバー、コンテナ化、一連のライブラリ、拡張機能などの環境レイヤーを追加し

ます。ApacheWebサーバーでPHP 7.3を使用します（nginxは提供されていませんが、これは良くも悪くもありませんが、心に留めておくべき事実です）。全部管理するわけではないので、基本的に気にしません。

インストールとセットアップ


さて、練習に移りましょう。最初の段階は、アクセス権の登録と設定です。

  1. amazon.comで登録します。最小限の車のセットで無料のアカウントを取得します。
  2. ログインする。Elastic Beanstalkはデフォルトでオレゴンリージョンを提供するため、AWSマネジメントコンソールで[オレゴン]を選択します。
  3. コンソールからIAMサービスに移動します（検索バーにiamと書き込みます）。
  4. そこには、特定のアクションを実行するダッシュボードが表示されます

  5. Elastic Beanstalkを使用するには、新しいユーザーを作成して構成します（プログラムによるアクセスのみ）。


  6. AWSElasticBeanstalkFullAccess権限を持つグループを追加します。
  7. アクセスの詳細をダウンロードして、安全な場所に保存してください。ファイルを紛失した場合、ユーザーはファイルを再作成する必要があります。




したがって、ユーザーがいるので、このユーザーで引き続き作業します。この段階では、すべて。

EB CLI

次に、ElasticBeanstalkコンソール自体をインストールする必要があります。これはかなり長いプロセスです。ここでは、実行する必要があることの簡単な概要を示します。

  1. リポジトリhttps://github.com/aws/aws-elastic-beanstalk-cli-setupのクローンを作成します。
  2. readmeを注意深く読み、OSに必要なアクションを実行します（潜在的な問題はreadmeにも記載されています）。
  3. インストール後、パス付きの変数をエクスポートすることを忘れないでください。
  4. すべてがチームとして機能することを確認しますeb –version。



Elastic Beanstalkの初期化次に、プロジェクトでElastic Beanstalk を初期化する

必要があります。このため：

  • アーカイブをソースコードで解凍します。
  • composer installを実行します。
  • eb initを実行します。
  • 地域（デフォルトはオレゴン）を選択し、ダウンロードしたファイルからアクセスの詳細を指定します。
  • アプリケーション名、PHPプログラミング言語、バージョン7.3を指定します。

このメモではCodeCommit、とssh-accessは使用しません。

初期化後、アプリケーション.elasticbeanstalk内にフォルダーが表示され、その中に設定ファイルがあります。



EC2インスタンスの作成と開始

次に、Beanstalkを介してEC2インスタンスを作成して開始する必要があります。このため：

  • 私たちは実行しeb createます。
  • 環境名DNS CNAMEを指定し、ロードバランサー（アプリケーション）を選択します。
  • スポット艦隊を拒否します（荷重下の弾性）;
  • eb status / eb healthのステータスを確認します。
  • eb open siteを開こうとしています。

奇妙なことに、エラー403が発生しています。何が問題だったのでしょうか。私たちのアプリケーションはLaravelにあるため、エントリポイントはディレクトリ/publicにあり、EBはデフォルトでルートディレクトリのエントリポイントを想定しています。



構成

の修正次のステップは、構成の修正です。

  • Elastic Beanstalkのコンソールに移動します。
  • アプリケーションを選択して、構成に移動します。
  • 「ソフトウェア」セクションで、「変更」ボタンをクリックします。
  • /にドキュメントルートをインストールします。
  • [適用]をクリックします。
  • 操作性を確認してください（/api/v1/goods）。

ヘルスチェックの設定

実際には、ヘルスチェックの設定が残っています。このため：

  • Elastic Beanstalkのコンソールに移動します。
  • アプリケーションを選択して、構成に移動します。
  • [ロードバランサー]セクションで、[変更]ボタンをクリックします。
  • 「プロセス」セクションで、デフォルトを選択し、「編集」アクションを選択します。
  • Health CheckセクションでPath / api / v1 / goodsおよびHTTPコード200を指定します。
  • [保存して適用]をクリックします。
