Use-soundパッケージ：Reactアプリケーションの効果音

たぶん私はプロとしてサウンドに携わっていたのかもしれませんが、ウェブをもっと大きくしたいのです。

多くの人がこの欲望で私をサポートしないことを知っています。そして理由なしではありません！歴史的に、インターネット上のサウンドは、非常に不成功かつ醜い方法で使用されていました。

  • Webの初期の頃は、一部のページでバックグラウンドMIDI音楽が再生されていました。
  • ポップアップ詐欺師は音を使って悪い目標を達成します。彼らは、音の助けを借りて、ユーザーの注意を引き、詐欺をより信頼できるものにしようとします。
  • 音声は自動的に開始するビデオに存在します（まあ、これは絶対に悪夢です）。

確かに、音を完全に放棄することは、子供が水をはねかけるようなものだと確信しています。サウンドはユーザーのアクションを強調でき、フィードバックを強化できます。音は少し退屈なアクションを明るくすることができます。 Webページのサウンドを好みに合わせて使用​​すると、プロジェクトのユーザーエクスペリエンスが向上し、より「具体的」で現実的なものになります。 このアイデアは新しいものではありません。ゲームやモバイルアプリケーションでは、サウンドが常に使用されています。実際、ウェブは一般的なトレンドに対する奇妙な例外です。私の頭に浮かぶほとんどのデジタル製品は音を使用しています。私は、モバイルアプリケーションでそれらを現実に近づけるために使用される触覚応答について話しているのではありません。





私のブログでの作業の過程で、私は音で実験したかったです。ユーザーが操作するブログのインターフェース要素の多くは、静かで短い音を出します。たとえば、ブログのホームページの右上には、テーマを切り替えたり、サウンドのオン/オフを切り替えるボタンがいくつかあります。

Web上のサウンドはほとんど使用されないため、その使用はユーザーによるサイトの認識に大きな影響を与える可能性があります。これは開発者の一種の秘密兵器であり、その使用は特定のプロジェクトに驚くほど大きなプラスの影響をもたらす可能性があります。

サウンドの使用を簡単にするために、私はサウンドを独立したnpmパッケージの 形式で処理するために使用するフックを設計し、use-soundそれを公開しました。ここでは、このフックの機能について説明し、Webでのサウンドの使用に関するヒントを共有します。

コードを確認してこのフックを使い始めたい場合は、GitHubリポジトリをご覧ください。

概観


パッケージuse-soundは、開発者が効果音を使用するのに役立つReactフックです。次に例を示します。

import useSound from 'use-sound';
import boopSfx from '../../sounds/boop.mp3';
const BoopButton = () => {
  const [play] = useSound(boopSfx);
  return <button onClick={play}>Boop!</button>;
};

このフックを使用すると、プロジェクトのビルドに約1 KBが追加されます。確かに、彼は非同期モードで、サイズが10 KBのサードパーティのハウラー依存関係をロードします。

フックは開発者に多くの素晴らしい機能を提供します。それらの中で、私は次のことに注意したいと思います：

  • 再生を途中で停止し、一時停止して再生を再開します。
  • サウンドの「スプライトシート」のようなものをダウンロードして、個別のサウンドに分割する。
  • サウンドの再生を加速および減速する機能を備えた再生速度の設定。
  • たくさんのイベントリスナー。
  • ハウラーのおかげで利用可能な多くの高度な機能。

ここにフックのドキュメントがあり、その使用法とAPIリファレンスの詳細なガイドを見つけることができます。

使用音の基本


▍インストール


まず、yarnまたはnpmを使用してパッケージをインストールする必要があります。

#  yarn
yarn add use-sound
#  NPM
npm install use-sound

▍インポート


パッケージuse-soundは唯一のデフォルト値をエクスポートします-これはフックuseSoundです。次のようにインポートできます。

import useSound from 'use-sound';

さらに、使用する予定のオーディオファイルをインポートする必要があります。

Reactプロジェクトで作業するときに、create-react-appGatsbyのようなものを使用する場合、画像などの他のリソースをインポートするのと同じ方法でMP3ファイルをインポートできるはずです。

import boopSfx from '../../sounds/boop.mp3';

自己記述型のWebpack構成を使用する場合は、ファイルローダーローダーを使用して.mp3ファイルを処理する必要があります。

さらに、フォルダーpublicまたはに配置されたサウンドファイルを操作できますstatic。たとえば、この記事で示すデモでは、パブリックフォルダーにある静的ファイルを使用しています。

音を検索して準備する


依存関係のインストールとコードの記述は、戦いの半分に過ぎません。ページをスコアリングするには、適切なサウンドを選択する必要があります。

サウンドを探すための私のお気に入りのリソースはfreesound.orgです。私のブログで使用されているほとんどすべての音は、私がそこで見つけました。Creative Commons Zeroの下でライセンスされた多くのサウンドがあります。ファイルをダウンロードするには、リソースに登録する必要があります。音は無料で使用できます。

適切なサウンドを選択するには、freesound.orgのコンテンツを真剣に検討する必要があるという事実に備えてください。ここでのポイントは、たとえば、多くの音は質が悪いということです。正しい音を見つけるプロセスは、干し草の山から針を探すプロセスと比較できます。そして実際に使用するために用意されたサウンドの準備は、ノーカットのダイヤモンドのファセットに似ています。

sounds音の準備


freesound.orgから取得した多くのサウンドには、いくつかの改善が必要です。

  • 文字列のようなサウンドは、空のスペースで囲まれます。通常、無音部分は切り取られ、その結果、対応する効果音は、有声イベントの発生直後に再現されます。
  • サウンドを準備するときに、1つのプロジェクトで使用されるすべてのサウンドが同じボリュームになるように、ボリュームを調整する必要がある場合があります。
  • Freesound.orgには、さまざまなオーディオ形式でサウンドが保存されています。好みのサウンドをMP3に変換する必要があるかもしれません。

Audacityを使用して、これらの編集をすべて完了することができます。これは無料のオープンソースクロスプラットフォームオーディオエディターです。ここでは、このエディターの使用方法については触れませんが、多くのすばらしいチュートリアルがあります。

なぜサウンドをMP3に変換することについて話しているのですか？実際には、以前はすべてのブラウザーでサポートされるオーディオ形式はありませんでした。したがって、MP3、AIFF、およびWAVファイルをプロジェクトに含め、それらをサポートする環境で使用することは一般的でした。

幸い私たちにとって現代の状況では、MP3形式はIE 9を含むすべての主要なブラウザーでサポートされています。MP3形式で保存されたサウンドも非常によく圧縮されています。また、これにより、品質を損なうことなくサウンドをエンコードする代替フォーマットを使用するよりもはるかに小さいファイルに移動できます。

サウンドと可用性


私はWebでのサウンドの使用を推奨していますが、すべてのユーザーがそれを好むわけではないことを認めます。そしてここでは、沈黙に対する主観的な欲求を超えた考慮事項について話しています。

視覚障害のある人は、スクリーンリーダーを使用してWebページを使用します。また、ページを見ない人だけの話ではありません。たとえば、スクリーンリーダーは、テキストを認識しにくい失読症の人や、他の同様の問題のある人が使用します。スクリーンリーダーは、ドキュメントを解析してテキストを読み取るプログラムです。サイトに多くの効果音が含まれている場合、画面からテキストを読み取るプログラムの通常の動作を妨害する可能性があります。そして、これにより、そのようなプログラムに依存している人々がサイトで作業することを防ぐことができます。

このため、サウンドのオン/オフを切り替えるボタンをサイトのページに含めることが重要です。さらに、そのようなボタンは、キーボードナビゲーション（Tabキーを使用）に使用できる必要があります。理想的には、ユーザーがTabキーを使用してそのようなボタンに到達するまで、サウンドは再生されません。さらに、Webプロジェクトはこのボタンの状態を記憶している必要があります。そのため、ユーザーは常にボタンを使用してサウンドをミュートする必要がありません。

一方、聴覚に障害のあるユーザーは、ページとの特定のやり取りに音声が伴うことさえ知りません。サウンドをオフにしてインターネットで作業する人にも同じことが言えます。このため、特定の重要なアクションを実行するときは、音だけに頼らないことが重要です。サウンドがユーザーアクションの確認として使用される場合は、このアクションの視覚的な確認もページに表示されることを確認してください。サイトは完全に機能し、無音のままである必要があります。

レシピ


いくつかの例を見てみましょう（この記事のオリジナルの対応するセクションでは、それらをインタラクティブに試して、さまざまなコントロールによって生成されたサウンドを聞くことができます）。

▍チェックボックス


私はこの旗が本当に好きです。マウスを使用する場合は、最初にすばやくクリックしてから、クリックしてからマウスボタンを離すまでの間に少し時間を置きます。

function Demo() {
  const [isChecked, setIsChecked] = React.useState(
    false
  );

  const [playActive] = useSound(
    '/sounds/pop-down.mp3',
    { volume: 0.25 }
  );
  const [playOn] = useSound(
    '/sounds/pop-up-on.mp3',
    { volume: 0.25 }
  );
  const [playOff] = useSound(
    '/sounds/pop-up-off.mp3',
    { volume: 0.25 }
  );

  return (
    <Checkbox
      name="demo-checkbox"
      checked={isChecked}
      size={24}
      label="I agree to self-isolate"
      onChange={() => setIsChecked(!isChecked)}
      onMouseDown={playActive}
      onMouseUp={() => {
        isChecked ? playOff() : playOn();
      }}
    />
  );
}

sound音声再生の早期中断


ユーザーが特定のコントロールを操作したときにのみサウンドを再生したい場合があります。次の例では、ポインターがコントロールの上にあるときにのみサウンドが再生される方法に注意してください。

function Demo() {
  // ,  ,  'rising-pops' :
  // - fanfare
  // - dun-dun-dun
  // - guitar-loop
  const soundUrl = '/sounds/rising-pops.mp3';

  const [play, { stop }] = useSound(
    soundUrl,
    { volume: 0.5 }
  );

  const [isHovering, setIsHovering] = React.useState(
    false
  );

  return (
    <Button
      onMouseEnter={() => {
        setIsHovering(true);
        play();
      }}
      onMouseLeave={() => {
        setIsHovering(false);
        stop();
      }}
    >
      <ButtonContents isHovering={isHovering}>
        Hover over me!
      </ButtonContents>
    </Button>
  );
}

pitchピッチを上げる


そして、これは興味深い効果の例です。それは、Likeボタンの次のクリックによって再生されるサウンドが、前のクリックによって再生されるサウンドよりも高いという事実にあります。

function Demo() {
  const soundUrl = '/sounds/glug-a.mp3';

  const [playbackRate, setPlaybackRate] = React.useState(0.75);

  const [play] = useSound(soundUrl, {
    playbackRate,
    volume: 0.5,
  });

  const handleClick = () => {
    setPlaybackRate(playbackRate + 0.1);
    play();
  };

  return (
    <Button onClick={handleClick}>
      <span role="img" aria-label="Heart">
      </span>
    </Button>
  );
}

playback再生を開始および一時停止するボタン


サウンドを開始および一時停止するボタンの実装コードは次のとおりです。これは、新しいSpotifyの開発に役立ちます。

function Demo() {
  const soundUrl = '/sounds/guitar-loop.mp3';

  const [play, { stop, isPlaying }] = useSound(soundUrl);

  return (
    <PlayButton
      active={isPlaying}
      size={60}
      iconColor="var(--color-background)"
      idleBackgroundColor="var(--color-text)"
      activeBackgroundColor="var(--color-primary)"
      play={play}
      stop={stop}
    />
  );
}

▍サウンド「スプライトリスト」


React-コンポーネントで多くのサウンドを使用することを計画している場合、サウンドの「スプライトリスト」を使用することは理にかなっています。サウンドスプライトリストは、さまざまなサウンドを含むファイルです。それらを単一のファイルにパックすることにより、それらを使用する際の利便性が高まり、個々のファイルをダウンロードするために多くの並列HTTPリクエストを行うことを回避します（いつかHTTP / 2多重化を使用できるようになることを願っています）。

ここでは、サウンド「スプライトリスト」を使用してドラムマシンを作成します。仮想リールは、マウスまたはキーボードの1〜4のキーを使用して再生できます。

function Demo() {
  const soundUrl = '/sounds/909-drums.mp3';

  const [play] = useSound(soundUrl, {
    sprite: {
      kick: [0, 350],
      hihat: [374, 160],
      snare: [666, 290],
      cowbell: [968, 200],
    }
  });

  //  ,   'keydown',
  //    .
  useKeyboardBindings({
    1: () => play({ id: 'kick' }),
    2: () => play({ id: 'hihat' }),
    3: () => play({ id: 'snare' }),
    4: () => play({ id: 'cowbell' }),
  })

  return (
    <>
      <Button
        aria-label="kick"
        onMouseDown={() => play({ id: 'kick' })}
      >
        1
      </Button>
      <Button
        aria-label="hihat"
        onMouseDown={() => play({ id: 'hihat' })}
      >
        2
      </Button>
      <Button
        aria-label="snare"
        onMouseDown={() => play({ id: 'snare' })}
      >
        3
      </Button>
      <Button
        aria-label="cowbell"
        onMouseDown={() => play({ id: 'cowbell' })}
      >
        4
      </Button>
    </>
  );
}
