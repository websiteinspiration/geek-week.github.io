1.仕様の読み方


1.1一般


仕様を一読すると、次のSDカードオプションがわかります。

  • SDカードへのデータ転送は1行で実行されます。
  • SDカードからのデータの読み取りは1行で実行されます。
  • SPIモードでは、電力は+ 3.3Vのみです。
  • 初期化モードでのクロック周波数は100〜400 kHzの範囲。
  • 初期化後のクロック周波数-25 MHz。

これはすぐに、理論上のピーク帯域幅のポイントを意味します。25MHz * 1ビット= 25 Mb / sで、これはやや小さいです。2番目のマイナスSDカードの使用、電源+ 3.3V。SDカードを取り付ける予定のプリント基板にはそのような電圧はありません。

MicroSDフォームファクタカードは、容量によって3つのカテゴリに分類できます。

  • SDSC 最大2 GBまでのカード容量。カード内のアドレスはバイトです。
  • SDHC。カードの容量は2 GB以上、最大32 GBです。カード内のアドレスは、512バイトのブロックを示しています。
  • SDXC。カードの容量は32 GB以上、最大128 TBです。カード内のアドレスは、512バイトのブロックを示しています。

抵抗器の定格は50 kOhmである必要があります

電源をオンにすると、SDカードはSDIOモードになります。SPIモードに切り替えるには、初期化を実行する必要があります。カードを操作するためのプロトコルでは、CRCアルゴリズムの形式でデータとコマンドを正しく転送するための制御方式を使用します。SPIモードで動作している場合、CRC検証はデフォルトで無効になっています。したがって、カードをSPIモードに切り替えるためにカードに送信される最初のコマンドには、正しいCRC値が含まれている必要があります。

SPIインターフェースを介して送信されるコマンドのサイズは48ビットです。コマンド形式：

  • ビット47（左端）には常に値0が含まれています。
  • ビット46には常に値1が含まれます。
  • ビット45..40には、コマンドインデックスが含まれています。
  • ビット39..8には、コマンドの引数が含まれています。
  • ビット7..1には、前のビットのCRCが含まれています。
  • ビット0には常に値1が含まれます。

静止状態のMOSIバスは1に等しいため、ビット47および46はカードにトランザクションの開始を明確に追跡する機能を提供します。



SDカードでの作業時に使用されるコマンドは、R1、R3、R7などの回答を受け取ります。


タイプR1、サイズ8ビットの回答。回答タイプR3、サイズ40ビット。回答タイプR7、サイズ40ビット。 すべてのコマンドとその回答は、Physical Layer Simplified Specificationで詳しく説明されています。

1.4情報を読む


SPI経由でSDカードから情報を読み取るには、2つの方法があります。

1.4.1単一のデータブロックの読み取り


1つのデータブロックを読み取るとき、マスターデバイスはSDカードが1つのデータブロックを読み取るためのコマンドを生成し、コマンドが処理され、カードからのデータを含むパケットを待機しているという応答を待ちます。カードからのデータを含むパケットを受信した後、読み取りトランザクションは終了します。


データの1つのブロックを読み取るトランザクションの一般的なビュー。

複数のデータブロックを読み取る場合、マスターデバイスはSDカードが複数のデータブロックを読み取るためのコマンドを生成し、コマンドが処理されたという応答を待ち、カードからのデータのパケットを待ちます。データパケットを送信した後、カードは次のデータパケットを送信します。これは、マスターデバイスからコマンドを受信して​​読み取りを完了するまで続きます。

1.5情報の記録


SPI経由でSDカードに情報を書き込むには、2つの方法があります。

1.5.1単一のデータブロックの書き込み


1つのデータブロックを記録する場合、マスターデバイスは、SDカードが1つのデータブロックを書き込むコマンドを生成し、コマンドが処理されたカードからの応答を待ち、記録用のデータを含むパケットをカードに送信し、データが書き込まれたカードからの応答を期待して、トランザクションを完了します。


1つのデータブロックの書き込みトランザクションの一般的なビュー。

読み取りと同様に、1つのデータブロックのレコードが元々実装されていました。速度の結果は不十分でした。

1.5.2複数のデータブロックの書き込み


複数のデータブロックを記録する場合、マスターデバイスは複数のデータブロックを記録するコマンドを生成し、コマンドが処理されたカードからの応答を待ち、記録用のデータを含むパケットをカードに送信し、データが記録されたカードからの応答を期待します。マスター応答を受信した後、デバイスはカードに記録するために次のデータを含むパケットを送信します。これは、マスターデバイスがストップデータトークンを送信するまで続きます。


複数のデータブロックの書き込みトランザクションの一般的なビュー。

データパケットの構造は、データを読み取るときのデータパケットの構造に似ています。

フォーマット：

  • データトークン 書き込みコマンドの場合、値0xFCが使用されます。
  • データブロック。カードに書き込まれたデータが含まれています。
  • CRC 前のフィールドのチェックサムが含まれています。

書き込みコマンドを完了するために使用されるストップトラントークンのサイズは1バイトで、0xFDと同じです。

データパケットの最後のビットがカードにプッシュされると、カードは次のクロックにデータレコードのステータスであるデータレスポンスで応答します。データレスポンスのサイズは1バイトで、ビット7..5は任意、ビット4は常にゼロ、ビット0は常に1、ビット3..1はデータレコードのステータスをエンコードします。カードがデータパケットを返した後、カードはMISOラインをゼロに引き、カードがビジーであることを示します。論理ユニットレベルがMISOライン上にあると、次のデータパケットをカードに転送できます。

2.ハードウェアでのアルゴリズムの実装


仕様を読んだ結果、ハードウェアに実装する必要があるアルゴリズムのいくつかの特性を取得しました。

可能な操作モード：

  • 初期周波数形成。CSおよびMOSIラインは論理ユニット状態でなければなりません。
  • SDカードにデータを転送します。
  • SDカードからデータを受信します。
  • チームの完成を待っています。これは、応答の生成後、SDカードがMISOラインをゼロに描画してユニットの出現を待機するときに使用されます。
  • データを書き込むときに応答を読み取ります。
  • データの読み取り中にトークンを待機しています。

可能なクロックモード：

  • 初期化するクロック信号。
  • 仕事用のクロック信号。

私の見解では、アルゴリズムは3つのコンポーネントを使用して最適に実装されています。

  • SDカードに直接接続されている物理層のコンポーネントは、SCLK、CS、DI信号を生成し、DOで読み取ります。
  • コマンドレベルのコンポーネント。物理層のコンポーネントのすべてのデータの準備を提供します。
  • 内部デバイス全体を隠し、コマンド（読み取り、書き込み、消去）とデータのインターフェースを提供する、外界との通信のコンポーネント。

FPGAには、クロック信号（PLL、MMCM）を処理するための特別なユニットがありますが、出力から5 MHz未満のクロック信号を受信すると問題が発生します。その結果、物理層は50 MHzの周波数で動作します。iPhyMode信号の各データとともに、これらのデータをSDカードに転送する（またはSDカードから受信する）頻度を示すビットが受信されます。速度ビットに応じて、クロックイネーブル信号が生成されます。

物理層のコンポーネントには、SDカードにデータを転送したり、SDカードからデータを受信したりするための2つのオートマトンが実装されています。

データ転送用のマシンコード：github。

  • SDummy状態は、初期周波数シェーピング、128スイッチングを提供します。
  • STxBitsステータスは、SDカードへのデータ転送を提供します。

データを受信するためのマシンのコード：github。

  • sRxBits状態は、SDカードからのデータ受信を提供します。
  • sBusy状態は、SDカードの準備ができていることを保証します（カードはMISOラインをユニットレベルに解放します）。
  • sResp状態は、データの書き込み時の応答の読み取りを実装します。
  • sToken状態は、データの読み取り中にトークン待機を実装します。

  • iSdStartWriteは、書き込みコマンドの実行を開始します。

残りの信号は物理層の信号と一致します。

コンポーネントには5つの自動機械があります。

  • smSdInit（github）-SDカードの初期化。
  • smSdErase（github）-SDカードからデータを消去します。
  • smSdRead（github）-SDカードからデータを読み取ります。
  • smSdWrite（github）-SDカードにデータを書き込みます。
  • smSdCommand（github）-生成された機能に基づいて、以前のすべてのマシンから物理層のデータを準備します。

アルゴリズムが作成され、シミュレータで検証されます。今は鉄でチェックする必要があります。DigilentのZyboボードが利用可能になったことから登場し 、+ 3.3Vの電圧を持つバンクに無料のFPGA端子があり、外部デバイスに簡単に接続できます。はい、使用されているFPGAのタイプはZynq-7000です。つまり、プロセッサコアがあります。Cでテストを記述できます。これにより、テストタスクが簡略化されます。 そのため、実装されたアルゴリズムをGPポート経由でプロセッサコアに接続します（PIOと同様に、4バイトの操作が可能です）。中断を気にする必要はありません。タイマーポーリングを実装しています。 プロセッサモジュールで作業する場合、データ記録アルゴリズムは次のようになります。









  • SDカードにアドレスを設定します。
  • コマンドコード2を設定します。
  • プログラマブルロジックにあるバッファにデータを書き込みます。
  • コマンドを実行します。
  • コマンドが完了するまで待ちます。
  • チームの完了ステータスをリセットします。

実装されたテスト：

for (SectorAddress = 0; SectorAddress < 1048576; SectorAddress ++)
{
	if ((SectorAddress % 1024) == 0)
	{
		xil_printf("Data write to %d sector \n\r", SectorAddress);
	}

	/** Set address */
	Xil_Out32(0x43c00008, SectorAddress);

	/** Set command */
	Xil_Out32(0x43c00004, 2);

	/** Write data to PL */
	for (int32_t i = 0; i < 1024; i++)
	{
		Xil_Out32(0x43c00014, cntrData);
		cntrData++;
	}

	/** Start */
	Xil_Out32(0x43c00000, 1);

	/** Wait end of operation */
	for (;;)
	{
		status = Xil_In32(0x43c0000c);
		if (status == 0x01 || status == 0x03)
		{
			if (status == 0x03)
			{
				xil_printf("Error in write \n\r");
			}
			break;
		}
		else
		{
			cntrDuration++;
			usleep(100);
		}
	}

	/** Duration operation */
	durationWrite += cntrDuration;

	if (cntrDuration > MaxWrite )
	{
		MaxWrite = cntrDuration;
	}

	cntrDuration = 0x00;

	/** Clear start */
	Xil_Out32(0x43c00000, 0);

	SectorAddress += 7;
}

ループで1024の外側の境界が使用される理由についての質問に対して、アルゴリズムはブロック数を8に設定します。1つのブロックのサイズは512バイトです。8つのデータブロックの合計サイズは、8 * 512バイト= 4096バイトです。プロセッサモジュールとプログラマブルロジック間のバスのサイズは4バイトです。プロセッサモジュールからプログラマブルロジックに4バイトの4096バイトを送信するには、4096/4 = 1024の書き込み操作を実行する必要があることがわかります。
