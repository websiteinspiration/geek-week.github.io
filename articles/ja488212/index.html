DORA State of DevOps 2019レポート、今はロシア語でも

こんにちは、ハブル！

私はDORA State of DevOps 2019レポートをロシア語に翻訳しました。これは、現時点で最も関連性の高いDevOps研究であるだけでなく、DevOps変換に関する既製の実用的なガイドでもあるため、レポートを読むことをお勧めします。DevOpsチームの報告書の国家は IT企業から政府や医療機関へ- DORA（DevOpsチームの研究と評価、Googleクラウドの一部）が毎年リリースされた研究報告書で、研究は全く異業種からの30,000人以上の回答者が出席しました。





最近、DevOpsコミュニティを密接にフォローしています。ほとんどすべての主要なDevOpsカンファレンスのレコーディングに参加したり見たりしています。そして、私はロシアと外国の報告の主題にわずかな食い違いがあることに気づきました。たとえば、Ansible、コンテナ、クラウド、そしてもちろん、ユビキタスなKubernetesなど、多くの技術的および実用的なレポートを提示しています。また、DevOpsサミットの記録を見ると、チーム、人、文化の構築に関するレポートがあり、大規模な組織におけるDevOpsの変革の成功事例とファカップに関するレポート（さらに多くのレポートがあります）があります。

今日のDevOpsは、一連の技術的なプラクティスとアプローチとしてだけでなく、リリースの迅速化と開発の効率化に役立つだけでなく、組織全体のレベルで方法論としてDevOpsを実装することを決定すると、はるかに大きな効果を得ることができます。これは、ITを節約し、コストをすばやく回収する方法に関するストーリーではありません。これは、DevOpsアプローチを使用して、組織内のすべてのチームに複製されたITが、企業が尊厳と競争することができない強力なビジネスプラットフォームを提供する方法に関するストーリーです。これは主観的な評価ではありませんが、レポートの結論の1つです。技術的な手法とアプローチを使用すると、組織全体の効果に影響を与えることができます。

このレポートは業界全体の標準であると考えています。同僚が話している会議の1つで、聴衆の反応から、それがどのような種類のレポートであるかほとんど誰も気付いていないことに気づいたとき、不愉快なことに驚きました。しかし実際には、すべてがそれほど悪いわけではありません。カンファレンスの1つでExpress 42のメンバーがすでにレポートについて話していました。大規模な組織でさえレポートに関心を示しています（hi mail.ru）。コミュニティが発展していることをとても嬉しく思います。

リンク（Yandex、Google）を使用すると、ロシア語のDORA State of DevOps 2019レポートの完全版を見つけることができます。
