ローコードプラットフォームを使用した機械設計者向けの企業検索システムの実装

小さな会社のデザイナーを選ぶ問題


偶然にも、私たちの会社のどのプロジェクトも機械設計者の参加なしでは成り立たず、それらは常に作業で過負荷になっています。現在、スタッフは4人です。過去2年間で、すでに3つを変更しています。コンストラクタの不足という問題が非常に深刻になり、私たちは新しい人を探すためにあらゆる努力をしました。

自分でデザイナーを選んでみました。彼は空席を投稿し、人々と話しました。彼は一般的なレビューのために最も興味深いものを引き出し、チームは彼と通信するかどうかを決定しました。その後、面接があり、参加できる従業員が出席しました。残りは後でどのように行われたかを尋ねられました。その結果、決定は必ずしも成功しませんでした。

候補者の手紙にすぐに答えることはできませんでした;私は本当に履歴書を見直すための重要なプロジェクトから従業員の注意をそらしたくありませんでした。デザイナーの検索は常に遅れてきました。クールなデザイナーを評価できる（エンジニアではなく）個々の従業員に仕事のほとんどを投入することは不可能だと私には思えました。

しかし、そうではありませんでした。私の経験を共有したいと思います。

技術設計会社


当社は生産の自動化の分野で働いています。高い精度と再現性が要求される組立作業から人的要因を排除する問題を解決します。

私たちのプロジェクトはいくつかの段階に分かれています：

  1. アイデアの検索、未来のワークステーション（AWP）のコンセプトの開発
  2. 3Dモデルの開発、図面（CD）、製造部品の注文
  3. 必要な機器の購入と供給
  4. マシンビジョンソフトウェア、PLCソフトウェア、オペレーターソフトウェアの開発
  5. ワークステーションアセンブリ
  6. 現場での実装、デバッグ
  7. AWPが適切な製品の生産を開始する状態にする。

私たちが成長し、新しい注文を受け取ることができるのは、最後のポイントの達成です。同社には、エンジニア、デザイナー、プログラマー、エレクトロニクスエンジニアを含む15人の従業員がいます。並行して、約5つのプロジェクトを実施しています。専門家に対する多大な労力と高い要件により、会社のデザイナーを見つけるプロセスを非常に真剣に受け止めました。採用ミスは、プロジェクトの3〜4か月の作業の損失につながります。

ローコードプラットフォームを使用したビジネスプロセス開発


どんな指示も欠席よりはましだと思います。しかし、命令が情報システム内で形成される場合、実装の効果は桁違いに強くなります。従業員は単に何か悪いことをする機会はありません。情報システムにデザイナーの企業検索プロセスを導入することにしました。ここでは、得たものです：

システム実装の主な目標は：

  1. 候補者選定の日常業務からスペシャリストやディレクターを解放するために可能な限り。
  2. 市場で最高のスペシャリストを見つけるための客観的なプロセスを構築する。
  3. 候補者の継続的な検索と処理のプロセスを作成します。空き時間がある週に一度ではなく、常に。
  4. 受け取った候補者の質を損なうことなく、HRスペシャリストを迅速かつ容易に置き換えることができるようにプロセスを調整します。
  5. すべてが設定されたら、1年後にすべてを再考して整理する必要がなくなります。

システムの選択


プログラマーを介さずにプロセスをすばやく変更および編集できるローコードプラットフォームを選択しました。私自身、テーブルを変更し、新しいロールを導入してプロセスを変更でき、変更にはプラットフォームでの作業に数時間を要しました。

次に、新しいデザイナーを探す際のルールを社内社員にルールをつけて説明します。

会社で新しいデザイナーを見つけるための指示


システムには2つの役割があります。

  • HRは、候補者との運用作業を担当するスペシャリストです。
  • お客様は私たちが人を探しているチームの責任者です。

システムを操作するための一般的なシナリオ：

  1. 適切な履歴書の選択（HR +お客様）。
  2. 適切な候補者への電話、標準的な質問（HR）への回答。
  3. 候補者の評価（顧客）。
  4. 候補者にテストタスク（HR）を送信します。
  5. テストタスク（HR）を保存しています。
  6. 顧客（顧客）によるテストタスクの評価。
  7. 候補者の面接（HR +顧客）の手順。
  8. 候補者（HR）との作業を完了するための手順。

設計者は、図面（付録）に示すリワインダー用のローラーアセンブリの独自のバージョンを提案しました。

  1. すべてが組み立てられ、何らかの形で機能しますが、設計の計算ミスまたは製造エラーのため、ローラーは軸と半径方向の両方で遊びを持っています。
  2. 概念的には、ローラーは軸の円錐部分を中心とすることが想定されていました。しかし、これは起こりません。製造エラーまたは角度/表面積が不十分です。したがって、「付録」に「axis_changed」が表示されました。

  4. 特別な要件の1つは、ローラーをハウジングと軸から簡単かつ迅速に取り外して洗浄することでした。
  5. ベアリング：4x7x2。

実施結果


  1. このシステムは1週間で起動できました。
  2. 紙ではなくプロセスを作成することに成功しましたが、情報システムでは、これにより人々の相互作用が大幅に簡素化され、ルールに従うことができました
  3. 私たちは、プロジェクトに関わる専門家から日常業務のすべてを、活動分野のデザイナーとはかけ離れた1人に転送することができました。
  4. 検索時間を短縮しながら、市場でより多くの候補者をカバーすることが可能でした
  5. すべての情報は永久にシステムに残り、次回は候補者との対話を続けることができます
  6. チームは候補者を決定するために集まる必要はありませんでした。誰もが彼にとって都合のよい時間に議論に参加することができました。
  7. 注目に値しない、または私たちに適さない候補者は、会議や会話をせずにカードを適切なステータスに転送することにより、最初から排除されました。
