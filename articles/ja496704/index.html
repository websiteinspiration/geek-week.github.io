Booster Digital Launch Computer（LVDC）およびBooster Data Adapter（LVDA）

LVDCは、コンピュータに入力/出力を提供するLaunch Vehicle Data Adapter（LVDA）と連携して機能しました。コンピューターとロケットの間のすべての通信はLVDAを介して行われ、LVDAはアナログロケット信号と28 V制御信号をコンピューターが必要とするシリアルバイナリデータに変換しました。 LVDAには、さまざまな機能のためのバッファ（ガラス遅延ライン上）と制御レジスタがありました。 LVDAには、ジャイロスコープを備えた慣性モジュールからデータを読み取るためのアナログ/デジタルコンバーターと、ミサイルに制御信号を供給するためのデジタル/アナログコンバーターがありました。彼はまた、地球に送信されたテレメトリ信号を処理し、コンピュータ宛ての地球からのコマンドを受信しました。そして最後に、LVDCはLVDAからの冗長性を備えたスイッチング電源によって電力供給されました。


Saturn 5のLVDAは、LVDA入力/出力を提供する80ポンドのボックスでした。彼はロケットの他の部分へのケーブルのための21の丸いコネクターを持っていました。

LVDAには非常に多くの異なる機能があったため、LVDCのほぼ2倍の大きさでした。以下は、80 kg LVDAに圧縮されたすべてのスキームの図です。これは、プリント回路基板または「ページ」で満たされた2つのセクションに分かれています。フロントロジックセクションとリアロジックセクションです（フロントセクションのボードが私の手に落ちました）。フィルターと電源は中央のセクションにありました。メタノールベースのクーラントは、LVDAチャンネルを通してポンプで送られました。LVDAは、21個の丸いコネクタを介してLVDCおよびロケットの他の部分に接続されていました。

ダイオードトランジスタロジック


論理ゲートはさまざまな方法で作成できます。 LVDCとLVDAには、「ダイオード-トランジスタロジック」（DTL）などの技術を使用しました。これにより、ダイオードのゲートとトランジスタを作成できます。これは、Apolloオンボードコントロールコンピューターで使用されている抵抗-トランジスタロジック（RTL）に比べてより高度な技術でしたが、1970年代に非常に普及したトランジスタ-トランジスタロジック（TTL）に劣っていました。

LVDCの標準論理ゲートはAND-OR-INVERT（AOI）で、（A•B + C•D） 'などの論理関数を実装します。これは、論理関数ANDを入力データセットに適用し、次にORを適用して、結果を反対に変更するために呼び出されます。 AOIバルブは機能的でした。たとえば、（A•B + C•D•E + F•G•H） 'のように、異なる数の入力でバルブから要素を形成することができたためです。また、AOIバルブは複雑に思えるかもしれませんが、それを実装するのに必要なトランジスタは1つだけでした。これは、数量を節約する必要があった時代には重要でした。

バルブの仕組みを理解するには、次の図を参照してください。 4つの入力と2つのANDメンバーを持つAOIバルブを示しています。 1つ目は入力AとBの役割を果たし、その値は現時点では1（高電圧）です。プルアップ抵抗がAND値をプルアップします（赤、1）。下部のANDゲートでは、入力Cが0であるため、電流は入力Cを流れ、AND値を引き下げます（青、0）。このようにして、ダイオードとプルアップ抵抗がANDゲートを実装します。次にORステップを見てみましょう。上からの電流AND（赤）は、ORステップを引き上げます（1）。最後に、この電流によりトランジスタがオンになり、出力が引き下げられ（青、0）、反転が行われます。 ANDステップが両方とも0の場合、ORステップはプルアップされません。代わりに、プルアップ抵抗がOR値を引き下げ（0）、トランジスタをオフにします。その結果、出力はプルアップされます（1）。



AOIゲートはより多くの抵抗またはダイオードで構成でき、必要な数の入力を提供します。このバルブは単一のチップに実装されると予想できますが、LVDCは各バルブに複数のチップを使用しました。チップごとに、ダイオード、抵抗器、トランジスタの組み合わせが異なり、柔軟に接続して必要な論理ゲートを形成します。

モジュラーロジックデバイス


LVDCとLVDAは、Unit Logic Devices（ULD）と呼ばれる興味深いハイブリッド技術を使用して作成されています。集積回路のように見えましたが、ULDモジュールにはいくつかのコンポーネントが含まれていました。彼らは単純なシリコン結晶を使用し、それぞれが1つのトランジスタまたは2つのダイオードのみを販売していました。これらの水晶は、厚膜抵抗器とともに、面積が2 cm2のセラミック基板に取り付けられました。これらのモジュールは、人気のあるIBM S / 360コンピュータで使用されているSolid Logic Technology（SLT）のバリエーションです。 IBMは、集積回路が商業的に実現可能になる前の1961年にSLTモジュールの開発を開始し、1966年までに年間1億個のSLTモジュールを生産しました。

写真に示すように、ULDモジュールはSLTモジュールよりも大幅に小さく、その結果、コンパクトスペースのコンピューターにより適していました。 ULDモジュールは、SLT金属缶の代わりに平らなセラミックバッグを使用し、ピンの代わりに上部に金属接点がありました。回路基板のクリップはULDモジュールを保持し、これらのピンに接続しました。 LVDCとLVDAは、50種類を超えるULDを使用しました。


右側はULDモジュールで、SLTモジュールまたは最新のDIP ICよりも大幅に小さい（左）。 SLTモジュールの長さは13 mm、ULDモジュールは8 mmで、はるかに薄いものでした。

ULDモジュールには、最大4つの小さな正方形のシリコン結晶が含まれていました。それらのそれぞれは、2つのダイオードまたは1つのトランジスタを販売しました。下の写真は、手つかずのモジュールの横にあるモジュールの内部コンポーネントを示しています。左側には、セラミック基板上の回路の経路が表示されており、4つの小さな正方形のシリコン結晶に接続されています。プリント基板のように見えますが、実際にはデバイスは釘よりもはるかに小さいことに注意してください。厚膜抵抗器はモジュールの底面に印刷されているため、見えません。


INVタイプULDは開いているので、4つのシリコン結晶が見えます。右上はトランジスタ、他の3つはダブルダイオードです。モジュールはピンクのシリコーンで保護されていました。

下の顕微鏡写真は、2つのダイオードを実装するULDモジュールのシリコン結晶を示しています。結晶は非常に小さいです-砂糖の粒がスケールのために写真に示されています。水晶には3つの外部接点があります。3つの円にはんだボールがはんだ付けされています。不純物（暗い領域）が2つの下側の円に追加されて2つのダイオードのアノードを形成し、上側の円は基板に接続されたカソードでした。この水晶は最も単純な集積回路よりもはるかに単純であることに注意してください。

インバータモジュール回路

ボードはANDゲートモジュール（タイプAAおよびAB）も使用します。これらは独立したゲートではなく、INVチップに接続してより多くの入力ANDおよびORを提供できるコンポーネントのみであることに注意してください。これらのモジュールはさまざまな方法で柔軟に接続され、特別な入力や出力はありません。一般的なオプションの1つは、AAチップの半分を3つの入力を持つANDゲートとして使用することです。 ABチップの一部は、必要に応じて、さらに2つの入力を提供できます。

PCB回路図


ボードの機能を理解するために、チップ間のすべての接続をマルチメーターで鳴らして配線図を描くという面倒な作業を費やしました。ただし、その直後に、すべてのスキームを含むLVDA命令の手に渡ったため、リバースエンジニアリングを行う私の試みは冗長でした。ボードは7つの入力を持つマルチプレクサを形成し、7つの入力の1つを選択して、受信した値をトリガーに保存します。1960年代のテクノロジーでは、このような単純なアクションで、複数のチップを備えたボード全体を作成する必要がありました。

以下の図は、ボードの簡略図を示しています。左側には、ボードに7つの入力があります。それらの6つは論理信号を受信するためにバッファリングする必要がある28 V信号で、7つ目は6 V論理信号です。 7つのラインの1つに電流を流して対応する入力を選択すると、データがトリガーに保存されます。電流が「リセットマルチプレクサ」と「マルチプレクサアドレス」に適用されると、トリガーがリセットされます。


簡略化された回路基板操作完全な回路基板。四角形は論理要素を示します。 NUは未使用の入力を示します-ボード上にトラックがありますが、チップは接続されていません。




図には多くの論理ゲートが描かれていますが、すべてが2つのAOIゲートで実装されています。黄色のバルブは1つの大きなAOIバルブを形成し、青色のバルブは2番目のバルブを形成します。2つの黄色のORが1つにマージされます。2つのゲートが8つのチップに実装されています-2つのINVチップ、4つのAAと2つのAB。これは、AOIロジックモデルの柔軟性と拡張性、および回路による多数のチップの使用を示しています。回路全体では、2つのトランジスタのみが使用されます。ほとんどすべてのロジックがダイオードに実装されています。

バッファスキーム


ボード上の26個のチップのうち、18個はアナログであり、入力信号のバッファリングと処理に従事していました。 28 Vの信号が入力され、ロジックには6 Vが必要でした。各入力（No. 7を除く）は、「ディスクリートインターフェース回路」（DIA）を通過し、入力が論理信号に変わりました。次の図は、チップ321、322、および323から組み立てられた回路を示しています（ボード上のほとんどのチップの場合、指定はINV、DLD、EDなどのアルファベットコードですが、アナログチップの場合、指定はデジタルであり、明らかに最後の3つだけですスペアパーツ番号の桁）。写真は各チップの内容です。 321チップは抵抗器（下部）のみで構成されているため、上から空に見えます。チップ322は1つのダイオードで構成され、チップ323は2つのトランジスタで構成されます（写真323には結晶はありません。これらは322と同じ小さな正方形です）。

次の図は、ボードの一般的な構造を示しています。中央の8つのロジックチップは緑色で囲まれています。6つの入力バッファはそれぞれ3つのチップ（321、322、および323）で構成されています。それらを通過する信号の経路は、青い矢印で示されています。ボード上のチップは35箇所あり、26箇所が使用されていますが、空いている場所にチップを追加すると、同じボードを他の目的に使用できます。



LVDAにおける取締役会の役割


このボードは、「システムデータサンプラー」と呼ばれるLVDAサブシステムのマルチプレクサーの一部であり、信号を選択し、テレメトリー用にコンピューターまたはEarthに送信します。 SDSは、8つの信号の1つを選択するマルチプレクサと、14ビットのデータをシリアル形式に変換するシリアライザセレクタで構成されます。マルチプレクサにはいくつかのデータソースがあります。発射前はロケットに接続されていたRCA-110地上コンピュータです。ロケット発射後、地上からコンピューターコマンドを受信する「コマンド受信装置」。コンピュータがロケットを制御するために使用した一連のリレーである「セレクタ」からのフィードバック。 Digital Data Acquisition System（DDAS）およびリアルタイムデータからのテレメトリ。

物理的には、これらのデータソースの多くは、ツールモジュールに配置された大きなボックスでした。たとえば、「コントロールディストリビューター」はLVDAの隣に取り付けられ、太いケーブルで接続された17 kgの箱でした。無線信号の送受信に関与する他のボックスに接続された4 kgのボックスである「コマンドデコーダー」から受信した信号は、LVDAの「コマンド受信デバイス」に入力されました。LVDAはケーブルで多くの異なる計測器モジュールデバイスに接続されていたため、21個のコネクタが必要でした。

ボードの物理的構造


LVDAとLVDCのボードは、ロケットの大きな加速と振動に耐え、要素を冷却するために、興味深い製造技術を使用しています。私の手に落ちたボードは破損していて、留め具はありませんでしたが、下の写真は「ページ」と呼ばれるモジュール全体を示しています。ページフレームはリチウムとマグネシウムの合金で作られています-熱をよく伝導する耐久性のある軽量素材です。ボードからの熱は、フレームを通過してLVDAおよびLVDCシャーシに達し、シャーシにドリルされたチャネルを通じて液体メタノールによって冷却されました。

各ページには、前面と背面の2つのプリント回路基板を収容できます。プリント回路基板には12層があります-1960年代にはかなり多くなっています（1970年代でさえ、市販のプリント回路基板には通常2層がありました）。このページには、98のコンタクト用コネクタがあり、各ボードに49あります。ボードは、ボードの上部で通過する30本の脚で接続されています。また、各ボードの上部には18個のテスト接点があり、すでに取り付けられているボードをチェックすることができました。その後、IBMはSystem / 4 Pi航空宇宙コンピューターの「ページ」でこの設計を再利用しました。

私にやってきたボードは、ページ上の他のボードから力で引き裂かれました。次の写真はその逆を示しています。スルーコンタクトは上部に表示されます-それらは別のボードに接続する必要があります。以下は、不足しているボードの49個の接点です。絶縁材の一部がボードから取り除かれ、所定のULDモジュールごとに12個のビアが見えます。それらのおかげで、チップ接点はプリント回路基板の12層のいずれにも接続できます。



結論


この小さな回路基板は、1960年代のコンピュータに関連するいくつかのものを示しています。

このボードは、当時のみ登場した集積回路ではなく、ハイブリッドモジュールの技術を使用しています。後退しているように見えるかもしれませんが、IBM System / 360シリーズの成功の鍵となっています。 56年前（1964年4月7日）に導入され、AOIロジックを備えたハイブリッドSLTモジュールを使用していました。そのようなコンピュータは長年市場を支配しており、System / 360アーキテクチャは引き続きIBMメインフレームでサポートされています。

LVDCとLVDAは、1967年に導入されたIBM System / 4ラインの航空宇宙コンピューターの作成にも役立ちました。これらのコンピューターも、このボードと同じ「ページ」とコネクターを使用しましたが、TTLフラットICを優先してULDモジュールを放棄しました。その後、システム/ 4 PiラインはスペースシャトルAP-101Sコンピュータに進化しました。

最後に、理事会は、1960年代以降どの程度の技術が改善されたかを示しています。各ULDモジュールには最大4つのトランジスタが含まれていたため、マルチプレクサのような単純な回路であっても、モジュールのボード全体を作成する必要がありました。今日のiPhoneプロセッサには80億を超えるトランジスタが含まれています。驚いたことに、そのような原始的な技術はロケットを月にもたらすことができました。
