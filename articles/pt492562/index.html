18 de março - Como entender o que é o Ignite SQL e começar a usá-lo


Se você se deparar com restrições no tamanho, escalabilidade ou desempenho do seu banco de dados relacional, talvez seja hora de atualizar para um sistema distribuído. O Apache Ignite é uma plataforma distribuída que pode funcionar como um banco de dados, fornecendo APIs SQL e JCache para trabalhar com dados. Neste webinar, examinaremos exemplos da vida real e discutiremos os prós e contras de cada abordagem:

  • Recursos SQL no Ignite
  • Como o SQL interage com a API Java (carregamento via API JCache via SQL e vice-versa)
  • Garantias de consistência (Ignite SQL e JCache API)
  • As complexidades do SQL distribuído e como a colocação de dados pode minimizá-lo.

25 de março - Computação distribuída com Apache Ignite


Para aproveitar todo o potencial da plataforma In-Memory, muitas vezes não é suficiente carregar dados no cluster e começar a trabalhar com eles usando o valor-chave ou a API do SQL. Você também deve colocar os dados com eficiência em um cluster e usar a computação distribuída que minimize a movimentação de dados pela rede. Neste webinar, você aprenderá como projetar e executar cálculos levando em consideração todos os prós e contras dos sistemas distribuídos.

Vamos considerar:

  • Como distribuir informações de forma eficiente em um cluster usando modelagem e disposição de dados adequadas;
  • Como a API do MapReduce é implementada e funciona no Apache Ignite;
  • Como minimizar os atrasos de operação usando a colocação de dados nos cálculos;
  • Como as consultas SQL podem ser executadas com eficiência em um ambiente distribuído.

O webinar será útil para quem começa a se familiarizar com o Apache Ignite e deseja aprender como projetar e usar de maneira ideal os recursos de computação distribuída dessa plataforma.

14 de abril - Guia de configurações de segurança do Apache Ignite e GridGain com exemplos


O Apache Ignite e o GridGain permitem computação rápida e consultas de dados distribuídos altamente eficientes. O Ignite e o GridGain fornecem uma configuração flexível que ajuda a tornar as operações do cluster mais seguras. Neste webinar, abordaremos os seguintes tópicos de segurança:

  • Conexão segura entre nós (SSL / TLS);
  • Autenticação de usuário
  • Autorização do usuário.

Usando exemplos, consideraremos configurações para:

  • SSL / TLS para criar uma conexão segura entre todos os nós, thin clients e conexões usando REST;
  • Autenticação através do Apache Ignite Advanced Security;
  • Plug-in de segurança GridGain.

O webinar ajudará desenvolvedores e especialistas em devops a entender os conceitos básicos de segurança do Apache Ignite e GridGain, fornecendo exemplos de configuração para os casos mais comuns.
