通知アプリiOSの内容を変更します

こんにちは！

この記事では、アプリケーションの通知テキストを変更し、UNUserNotificationCenterを介して追加のデータを送信するという明白ではない（少なくとも私にとっては）オプションを使用した私の経験を共有したいと思います。

この記事が、SwiftでのiOSプログラミングの初心者にとって役立つことを願っています。Swift for iOSでのプログラミング経験があることを前提としています。Swift 5とXcode 10.2.1を使用しました。それでは、始めましょう。


仕事


ユーザー通知を表示するアプリケーションがあります。通知は、本文テキストと背景テキストで構成されています。

したがって、アイデアは最初にユーザーにメインテキストのみを表示することです。ユーザーがセカンダリテキストを知りたい場合は、通知の[表示]ボタンをクリックする必要があります。

私は次のメソッドを使用して通知を表示します（すべての説明は以下にあります）

。1.まず、UNUserNotificationCenterのインスタンスを定義する必要があります。

class NotificationService: NSObject, UNUserNotificationCenterDelegate {
    let notificationCenter = UNUserNotificationCenter.current()
}


2.次に、通知の送信に使用するメソッドを宣言します。
私の通知リクエスト方法
// 1
//   ,      Tuple    .
// showBody -     Bool,           .
// withAction -     Bool,       ,   ""  .
// atDate date -    Date,      ,      .
func showNotification(with item: (String, String), showBody: Bool, withAction: Bool, atDate date: Date) {
    // 2
    //   
    let content = UNMutableNotificationContent()

    // 3
    //    
    let userActionsIdentifier = “showMe”

    // 4
    //      .
    content.title = item.0

    // 5
    //    ,      false,  ,       .
    if showBody { content.body = item.1 }

    // 6
    //     -    notificationCenter userInfo      .
    content.userInfo = [item.0: item.1]
    content.sound = UNNotificationSound.default

    // 7
    //        withAction  true,      "Show me".
    if withAction { content.categoryIdentifier = userActionsIdentifier }

    // 8
    //    ,        .
    let notificationID = item.0

    // 9
    //      .
    //       ,      .
    var dc = DateComponents()
    dc.hour = Calendar.current.component(.hour, from: date)
    dc.minute = Calendar.current.component(.minute, from: date)
    dc.second = Calendar.current.component(.second, from: date)

    // 10
    //   .     ,       UNCalendarNotificationTrigger.
    let trigger = UNCalendarNotificationTrigger(dateMatching: dc, repeats: false)

    // 11
    //        ,   .
    let request = UNNotificationRequest(identifier: notificationID, content: content, trigger: trigger)

    // 12
    //     notificationCenter,     .
    notificationCenter.add(request) { (error) in
        error == nil ? print(“notifacation request was added at “, trigger.nextTriggerDate()!) :
        print(error.debugDescription)
    }

    // 13
    //    .            .      .
    let action = UNNotificationAction(identifier: “showMe”, title: “Show me”, options: [])

    // 14
    //   .
    let category = UNNotificationCategory(identifier:userActionsIdentifier, actions: [action], intentIdentifiers: [], options: [])

    // 15
    //      notificationCenter.
    notificationCenter.setNotificationCategories([category])
}



このメソッドを呼び出すと、設定した時刻に通知が表示されます。
ユーザーが通知をプルダウンすると、[表示]ボタンが表示されます。ユーザーはこのボタンをクリックして背景テキストを表示する必要があります。これを行うには、UNUserNotificationCenterDelegateデリゲートメソッドを使用する必要があります。

func userNotificationCenter(_ center: UNUserNotificationCenter, didReceive response: UNNotificationResponse, withCompletionHandler completionHandler: @escaping () -> Void) { … }


したがって、このデリゲートのメソッドで使用するコードは次のとおりです。

私のデリゲートメソッド
func userNotificationCenter(_ center: UNUserNotificationCenter, didReceive response: UNNotificationResponse, withCompletionHandler completionHandler: @escaping () -> Void) {
    print(“didReceive response”)

    // 1
    //   switch    
    switch response.actionIdentifier {
        case “showMe”:
        print(“showMe action”)
 
        // 2
        //      userInfo,      showNotification   6,      String.
        let mainText = response.notification.request.content.userInfo.keys.first! as! String

        // 3
        //       userInfo,      showNotification   6,       String.
        let subText = response.notification.request.content.userInfo.values.first! as! String

        // 4
        //       showNotificationlet,    :
        // showBody    true,  withAction  false (       ).
        self.showNotification(with: (mainText, subText), showBody: true, withAction: false, atDate: Date(timeInterval: 3, since: Date()))
 
       default:
       print(“defaul action”)
    }
   
   completionHandler()
}
