Penyelesaian kartu bank dalam perdagangan - membuat dataset terbuka dan infografis di Google Data Studio

Ini adalah publikasi pertama saya tentang Habré. Saya tertarik dan sebagian mempraktikkan jurnalisme data dan ingin membagikan kepada Anda infografis yang menggambarkan pembayaran kartu bank dalam perdagangan. Dan juga untuk membagikan dataset terbuka di Data.World, dan berbicara tentang pembuatan proyek ini.

Jadi, infografis terakhir:



→ Tautkan ke laporan yang sama dalam bentuk interaktif yang jauh lebih menarik
→ Tautkan ke dataset terbuka (diperlukan pendaftaran Data.World).

By the way, sayangnya, itu tidak mungkin untuk menanamkan laporan dalam publikasi tentang Habré baik melalui iframe atau tag oembed.

Latar belakang proyek

Karena minat profesional saya di industri pembayaran, saya secara berkala memantau statistik Bank Rusia terkait dengan penyelesaian dengan kartu bank. Jadi, baru-baru ini saya beralih ke topik penyelesaian kartu di artikel “Lebih banyak mendapatkan!” di mana sudah dioperasikan dengan beberapa data dari proyek ini.

Kualitas statistik Bank Sentral sangat biasa-biasa saja. Jika diinginkan, Anda dapat menemukan beberapa wawasan, tetapi sifat data dan terutama format presentasi meninggalkan banyak hal yang diinginkan.

Kekurangannya adalah sebagai berikut:

  • Dataset selektif dan terfragmentasi.
  • Data disajikan hanya dalam xlsx. Kumpulan data normal juga tidak muncul di situs baru.
  • Struktur tabel Excel buruk, berbagai indikator dan periode dicampur.

Akibatnya, setiap kali saya perlu menyiapkan bahan analitis, saya harus memformat ulang file xlsx bank sentral secara manual dan membawanya ke bentuk yang dapat dicerna - misalnya, untuk grafik yang sama.

Permintaan untuk analitik berbeda: kadang-kadang Anda melihat bagian yang mengakuisisi, di lain waktu komponen emisi, statistik dalam konteks regional, dll.

Hampir setiap kali statistik Bank Sentral harus disekop secara manual, termasuk untuk menyatukannya dengan data dari sumber lain (seringkali dari Rosstat).

Dari waktu ke waktu, muncul pemikiran di benak saya bahwa saya perlu membuat beberapa dataset yang mudah digunakan, terus memperbarui, dan menggunakannya. Komunikasi dengan Ivan Begtinibegtin, pemimpin Infokultur dan Penginjil Data Terbuka, berpendapat bahwa data terbuka itu baik. Oleh karena itu, untuk menempatkan dataset semacam itu, tampaknya, masuk akal dalam beberapa repositori publik khusus.

Sejarah Proyek

Jadi, titik awal: penciptaan infografis yang menggambarkan keadaan pembayaran non tunai di Rusia.

Untuk apa?

  • Gunakan dalam pekerjaan analitis.
  • Gunakan sebagai bahan interaktif pada pertemuan, negosiasi, di acara yang relevan.
  • Keterampilan memompa dalam bekerja dengan data dan infografis.

Topik jurnalisme data dan infografis, layanan untuk pembuatannya, dan bahkan program studi yang saya minati sebelumnya. Ada banyak ulasan di web tentang topik ini, misalnya yang ini ).

Setelah menghabiskan beberapa waktu mempelajari beberapa dari mereka, saya mendapat kesan bahwa, tanpa ide dan set data, alat-alat ini, meskipun menarik, tidak mungkin berguna.

Kali ini, sebagai hasil dari penjelajahan yang agak kacau, saya menemukan layanan Google Data Studio (GDS).

Saya menyukai layanan ini, meskipun kesan pertama adalah layanan ini dirancang untuk mengumpulkan dan menganalisis data dari berbagai layanan Google untuk webmaster.

Apa yang kamu suka:

  • Pekerjaan fleksibel dan jelas dengan sumber data.
  • Seperangkat diagram besar yang biasanya digambar secara otomatis dengan benar dan estetis.
  • ( , ).
  • , , ..

Pada prinsipnya, layanan ini diatur cukup dimengerti, tetapi artikel yang sangat masuk akal dari Texterra.ru membantu saya dengan cepat menguasai GDS .

Gagasan untuk menggunakan produk Google lainnya secara paralel, termasuk untuk menyimpan data, juga membuat saya terkesan.

Untuk memulai, saya mulai bereksperimen dengan opsi penyimpanan paling sederhana - Google Sheets. Anehnya, banyak ini bekerja dengan bengkok. Format data terbang dalam tabel; Data Studio terkadang tidak mengenalinya sama sekali.

Plus, di kepala saya ada ide untuk membuat data terbuka, melalui Google Sheets ini entah bagaimana tidak sepenuhnya benar secara ideologis.

Produk Google lainnya yang terhubung ke Data Studio juga bagi saya sepertinya tidak cukup cocok atau kompleks.

Dalam mencari repositori lengkap data terbuka, saya mengalihkan perhatian saya ke Data.World - pada kenyataannya, layanan ini adalah salah satu yang pertama terdaftar dalam daftar koneksi GDS pihak ketiga.

Pendaftaran, pembuatan proyek pertama, dan pengunduhan data dari file xls tidak menyebabkan kesulitan apa pun. Teks Rusia ditransliterasi secara otomatis, format data tidak terpengaruh.
Menghubungkan dengan GDS dan penggunaan data lebih lanjut (tidak seperti Google Tables) juga berjalan dengan sempurna.



Saya tidak akan memikirkan membuat diagram dalam GDS secara detail. Pertama, saya menggunakan template yang cukup standar dan karenanya tidak menarik. Kedua, meskipun proses pembuatan bagan berbeda dari Excel, itu intuitif.

Apa berikutnya?

Saya berencana untuk melengkapi set data dengan data lain yang terkait dengan pembayaran kartu bank: jumlah terminal POS, omset ritel, pembayaran regional, dll.

Saya kira outputnya adalah 3-4 halaman laporan GDS, yang memungkinkan untuk mendapatkan kesan yang cukup lengkap tentang perkembangan pembayaran tanpa uang tunai di Rusia.
