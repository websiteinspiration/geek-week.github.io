単独でのDIY運動

誰もが自分自身を隔離しています。私の場合、主要な問題はすぐに身体を良好な健康状態に維持することとなりました。オフィスで働いて買い物をしていると、体が健康に保たれていることがわかりました。危機の前、どこでも売り切れる前に、一番安いエアロバイクを手に入れられたのは幸運だったので、問題は解決したと思いましたが、ありませんでした。ペダリングが単調な日常生活の必需品になるとき、次の問題は退屈です。そして、退屈なものすべてから、離れることは人間の本性です。もっと面白くするには？多分ゲームと組み合わせる？何故なの？飛行機のおもちゃで飛行機の速度がペダリングの速度に依存しているとしたら、それはどれほどクールだと私は長い間考えてきました。そのため、彼らはすでに「エクササイズ」という言葉を思いついています。

最初に考えたのは、速度制御用のホイールが付いた既成のジョイスティックを見つけ、どういうわけかそれをハックすることでした。幸い、壊れるのは残念ではないジョイスティックが手元になかったので、自分で作ることにしました。テーブルの引き出しにはArduino Microがあり、数年前に別のプロジェクトのために購入しましたが、使用されることはありませんでした。マイクロにはUSBポートがあり、コンピューターはそれをキーボードとして認識します。そして、キーボードがある場所では、他のHID（ヒューマンインターフェイスデバイス）デバイスからそれほど遠くありません。ジョイスティックのライブラリがすぐに見つかり、非常に便利でした（github.com/MHeironimus/ArduinoJoystickLibrary）。 SparkFun画像からいくつかのボタンとジョイスティックを追加します—完了しました！回路は完全に取るに足らないものです— Arduino、ボタン、ジョイスティックの2つの抵抗変数。

次の質問は、シミュレータからペダリングの速度を取得する方法です。ほとんどすべての最新のシミュレーターにはディスプレイとセンサーがありますが、私はシミュレーターを選びたくありませんでした（同時に保証が失われました）。さらに、すべてのプロセスを問題なく繰り返すことができるようにしたいと思いました。まず、ブレッドボードのシンプルなボタンを使用して、Arduinoが機能していることを確認しました。私はそれをチェックしましたが、テストの終わりまでにボタンが文字通りばらばらになるように夢中になりました。深刻なセンサーについて考える時間です。磁気センサーと光学センサーが最終的に到達し、問題が少ないと（正しく）想定して、磁気センサーに落ち着きました。

ホール効果に基づく磁気センサーの多くの異なるモデルがありましたが、極性を気にしないため、私はダイオードAH3572に落ち着きました。Arduinoで3.3ボルトしか必要とせず、比較的安価で、2種類のケースがあり、ブレッドボードに便利です。 PCBマウント。最も興味深い質問は、センサーがどこまで作動するかでした。ここで私は幸運でした。祖国の大箱には、辺が1センチメートルほどの立方体の形をしたかなり強力な中国の磁石がありました。この磁石を使用すると、センサーは最大3センチの距離で確実に動作しました。ペダルの磁石、ケースにテープが貼られたセンサー—機能します。







唯一の難点は、1回転あたり1回トリガーされるセンサーが、速度値としての使用に適した既製のデータを提供しないことです。革命の1回のクリックから、どういうわけか比較的スムーズな値を取得する必要がありますが、同時に、速度の変化、特に停止に対して十分に迅速に応答できるようにする必要があります。この問題は、約20行のコードで解決されました。

//    
void interruptHandler() {
  unsigned long now = millis();
  if (lastInterrupted + DEBOUNCE_MS < now) {
    lastInterrupted = now;
    throttleBuffer += THROTTLE_BOOST;
    if (throttleBuffer > MAX_AXIS) {
      throttleBuffer = MAX_AXIS;
    }
  }
}

//       
void calculateThrottle() {
  if (throttleBuffer < RELEASE_PER_TICK) {
    throttle += throttleBuffer;
    throttleBuffer = 0;
  } else {
    throttle += RELEASE_PER_TICK;
    throttleBuffer -= RELEASE_PER_TICK;
  }

  if (throttle > MAX_AXIS) {
    throttle = MAX_AXIS;
  }
  
  if (throttle > DECAY_PER_TICK) {
    throttle -= DECAY_PER_TICK;
  } else {
    throttle = 0;
  }
}

この時点まで、すべてが簡単で、さらなる困難が始まりました。適切なおもちゃを見つけるのは驚くほど困難でした。数日で私は多くのオプションを試しました。まず、比較的弱いオフィスのラップトップで正常に動作するためにそれが必要でした。この時点で、現代の美しい空飛ぶ戦闘機はすべて脱落しました。古いチラシが始動しないか、非常に鈍かった。最初の適切なゲームはIL-2 Sturmovikでしたが、本格的なシミュレーターはここでは適切ではないことに気付きました。まず、コントロールが少なすぎるため、ジョイスティックの精度が不十分です。次に、注意が必要で、頭の中で考えます。忘れないでください。ここではおもちゃを演奏していませんが、まずは激しくペダルを踏んでいます。さて、そして最も重要なことは、彼の配色を見た人が理解するだろうと同じように非常に憂鬱なことです。アーケードタイプのシンプルなものが必要でした明るく軽い。

最終的に、私はSky Rogue（store.steampowered.com/app/381020/Sky_Rogue）に落ち着きました。興味深いことに、有名な宇宙プロモーターのスコットマンリーは彼女のレビューを持っています（youtu.be/bRCsB5QoxEM）。シンプルだが明るく、8ビットグラフィックに様式化された、最小限の要件、アルゴリズムによって生成された（繰り返さない）地形、爽快なアクション、無限の武器—ノンストップの30分の動きに他に何が必要ですか？ Sky Rogueはゲームパッドに適していますが、幸いなことに、フルスクリーンモードのSteamでは、ジョイスティックコントロールを再割り当てしてゲームパッドをエミュレートできます。



これで最後のタッチが残っています—軍団。結局のところ、ブレッドボードを手で操作することは可能ですが、かなり不便です。 Autodesk Fusion 360を片手に、3Dプリンタをもう片方に。しかし、最初に少し考える必要があります。コントローラは片手で制御する必要があり、もう一方はシミュレータを握る必要があります。おもちゃで夢中になってシミュレーターを台無しにするのは、自己分離の恥です。形状は左右対称でなければならないので、どの手でも使用できます。まあ、それは印刷と組み立てが簡単であることが望ましい—自己分離では、誰もが希望する長さの3つの同一のM2ネジを見つけるとは限らない。



2つの半分を接着した画像のデザイン。あちこちに間違いなく瞬間接着剤があります。接着剤が乾くまで、半分の位置をすばやく調整するために、ガイドピンが使用されます。これは、プリンター用の5mmの素材です。ボタンは個別に印刷され、組み立て時にハウジングに挿入されます。軸がなく、プリント基板の表面を転がり、ケースの下半分で押し付けられています。もちろん、最も便利な形の選択に少し夢中になりました。最初のバージョンは、コンポーネントのサイズに正確に合わせて作られましたが、手に持っておくには小さすぎました。



シンセンでのプリント回路基板の製造（www.pcbsino.com）を含め、いくつかのコントローラーの製造には（シンガポールでは）約3,000ルーブルの費用がかかります。）—幸いなことに、中国のエレクトロニクス産業は、流行にもかかわらず継続して運営されました。コストの約半分— Arduino Micro、2番目に高価なコンポーネント、ジョイスティック。大量生産で3倍安くできるかもしれませんが、私にはそのようなグローバルな計画はありませんでした（興味があれば書いてください）。

しかし、最も重要なことは、私がコントローラーを作成して以来、シミュレーター上で30分間、完全に気付かれずに飛行することです。体調を整え、健康を維持しましょう！
