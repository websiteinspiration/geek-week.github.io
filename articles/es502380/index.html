Apio + asincio

Hola Habr! Quiero contar cómo resolví el problema de la ejecución competitiva efectiva de las tareas de asincio en Celery.



Introducción


El apio es un gran proyecto con una historia compleja y una gran carga de compatibilidad con versiones anteriores. Las decisiones arquitectónicas clave se tomaron mucho antes de que apareciera Asyncio en Python. Por lo tanto, es aún más interesante que de alguna manera pueda ejecutar la tarea de asincio en apio fuera de la caja.

Modo de orificio activado
#839. . mode off.

Ejecute tareas asincio en apio de vainilla


Puede iniciar la tarea asincio desde el cuadro:

import asyncio

from .celeryapp import celeryapp


async def async_task():
    await asyncio.sleep(42)


@celeryapp.task
def regular_task():
    coro = async_task()
    asyncio.run(coro)

Ventajas obvias:

  • ¡Funciona!
  • Simplemente
  • Sin dependencias externas adicionales

¿Qué está realmente mal?

  • Event Loop se crea dentro de cada trabajador.
  • No hay cambio de contexto entre la ejecución de corutinas
  • En un momento dado, el trabajador ejecuta no más de una rutina
  • Los recursos compartidos no se comparten entre tareas
  • Repetitivo

Es decir, asyncio en este caso se usa para asyncio, pero no hay ventajas

Vamos a intentar más complicado?


Luché por el rendimiento:

import asyncio
import threading

from .celeryapp import celeryapp

celeryapp.loop = asyncio.get_event_loop()
celeryapp.loop_runner = threading.Thread(
    target=celeryapp.loop.run_forever,
    daemon=True,
)
celeryapp.loop_runner.start()

async def async_task():
    await asyncio.sleep(42)


@celeryapp.task
def regular_task():
    coro = async_task()
    asyncio.run_coroutine_threadsafe(
        coro=coro,
        loop=celeryapp.loop,
    )

¡Bingo! Pros:

  • Aún trabajando
  • Aún más o menos efectivo
  • Incluso los recursos se mezclan entre las rutinas dentro del trabajador.
  • Todavía no hay dependencias externas adicionales

¿Bingo? Los problemas no tardaron en llegar:

  • El apio no sabe nada sobre correr corutina
  • Pierdes el control sobre la ejecución de tareas
  • Pierdes el control de las excepciones.
  • Repetitivo

Para ir con una solución de ingeniería tan maravillosa en prod, mi mano de alguna manera no se levantó

Formulación del problema


  • Deberia trabajar
  • Las rutinas deben ser competitivas
  • Los recursos deben compartirse entre muchos ejecutables
  • Sin repetitivo
  • API predecible simple

Es decir, mis expectativas:

import asyncio

from .celeryapp import celeryapp


@celeryapp.task
async def async_task():
    await asyncio.sleep(42)

Resumen


Implementé mis ideas en la biblioteca celery-pool-asyncio . Nuestro equipo utiliza esta biblioteca en el proyecto actual y ya la hemos implementado.

Características brevemente


Además de ejecutar directamente tareas de asyncio, celery-pool-asyncio también resuelve problemas:

  • Eliminación de tareas asincrónicas
  • Soporte de corutina en señales de apio

Para que el apio-pool-asyncio funcione, usé parches de mono . Para cada parche utilizado en tiempo de ejecución, es posible deshabilitarlo.

Puede leer más sobre todo esto en la documentación.
