Nginx


nginxサーバーをインストールして実行する方法については説明しません。公式文書は言うまでもなく、この件に関するインターネット上の記事は十分にあります。HTTPSとトークンによる2要素認証の設定に取り掛かりましょう。

nginx.confのサーバーセクションに次の行を追加します。

server {
	listen 443 ssl;
	ssl_verify_depth 1;
	ssl_certificate /etc/nginx/Server.crt;
	ssl_certificate_key /etc/nginx/ServerKey.pem;
	ssl_client_certificate /etc/nginx/CA.crt;
	ssl_verify_client on;
}

nginxのssl設定に関連するすべてのパラメーターの詳細な説明は、ここにあります-https ://nginx.org/en/docs/http/ngx_http_ssl_module.html#ssl_client_certificate

私が設定したものについて簡単に説明します：

  • ssl_verify_client-証明書の信頼チェーンを検証する必要があることを示します。
  • ssl_verify_depth-チェーン内の信頼されたルート証明書検索の深さを決定します。クライアント証明書はすぐにルート証明書に署名されるため、深さは1に設定されます。ユーザー証明書が中間CAに署名されている場合は、このパラメーターに2を指定する必要があります。
  • ssl_client_certificate-信頼されたルート証明書へのパスを指定します。これは、ユーザー証明書の信頼を確認するために使用されます。
  • ssl_certificate / ssl_certificate_key-サーバー証明書/秘密鍵へのパスを示します。

nginx -tを実行して、設定にタイプミスがないこと、およびすべてのファイルが必要な場所にあることなどを確認することを忘れないでください。

そして実際にはすべて！ご覧のとおり、設定は非常に簡単です。

  2. 最初はトークンなしでログインしてみましょう。次の画像を取得します。

  4. [ ロード]をクリックして新しいPKCS＃11デバイスドライバーを追加し、librtpkcs11ecp.soへのパスを指定します。

Apache


nginxと同様に、Apacheのインストールに問題はありません。このWebサーバーのインストール方法がわからない場合は、公式ドキュメントを使用してください。

そして、HTTPSと2要素認証の構成を開始します。

  1. 最初にmod_sslをアクティブにする必要があります：

    $ a2enmod ssl
  2. 次に、デフォルトのHTTPSサイト設定をオンにします。

    $ a2ensite default-ssl
  3. 次に設定ファイルを編集します：/etc/apache2/sites-enabled/default-ssl.conf：

        SSLEngine on
    SSLProtocol all -SSLv2

    SSLCertificateFile	/etc/apache2/sites-enabled/Server.crt
    SSLCertificateKeyFile /etc/apache2/sites-enabled/ServerKey.pem

    SSLCACertificateFile /etc/apache2/sites-enabled/CA.crt

    SSLVerifyClient require
    SSLVerifyDepth  10

    ご覧のとおり、パラメーターの名前はnginxのパラメーターの名前とほぼ一致しているため、説明しません。繰り返しますが、詳細に興味のある方はどなたでも、ドキュメントへようこそ。
    次に、サーバーを再起動します。

    $ service apache2 reload
$ service apache2 restart
