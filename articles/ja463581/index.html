コンピューターPC0はARP経由でスイッチに要求を送信し、スイッチはルーターにアドレスを指定し、スイッチはMACアドレスをコンピューターに送信します。その後、コンピューターは同じルートに沿ってpingパケットを送信します。ルーターは、VLAN20ネットワークがポートg0 / 1に接続されていることを認識しているため、スイッチにパケットを送信します。スイッチはパケットを宛先（PC1）に転送します。

このスキームは機能しますが、2つのルーターインターフェイスを占有するため、非効率的です。つまり、ルーターの技術的な機能を不合理に使用しています。したがって、同じことが単一のインターフェースでどのように実行できるかを示します。

2本のケーブルで回路を削除し、1本のケーブルでスイッチとルーターの以前の接続を復元します。スイッチのf0 / 1インターフェイスはトランクポートになるはずなので、スイッチ設定に戻り、このポートに対してswitchport mode trunkコマンドを使用します。ポートf0 / 4は使用されなくなりました。次に、show int trunkコマンドを使用して、ポートが正しく構成されているかどうかを確認します。



Fa0 / 1ポートは、802.1qカプセル化プロトコルを使用してトランクモードで動作することがわかります。 VLANテーブルを見てみましょう。F0/ 2インターフェースはVLAN10販売部門ネットワークの下にあり、f0 / 3インターフェースはVLAN20マーケティングネットワークの下にあることがわかります。



この場合、スイッチはルーターのg0 / 0ポートに接続されています。ルーターの設定で、int g0 / 0およびno ip addressコマンドを使用して、このインターフェイスのIPアドレスを削除します。ただし、このインターフェイスは引き続き機能し、シャットダウン状態ではありません。覚えていれば、ルーターは1.0と2.0の両方のネットワークからのトラフィックを受け入れる必要があります。スイッチはトランクによってルーターに接続されているため、最初のネットワークと2番目のネットワークの両方からのトラフィックはルーターを介してスイッチに流れます。ただし、この場合、ルーターインターフェイスにはどのIPアドレスを割り当てる必要がありますか？

G0 / 0は、デフォルトではIPアドレスを持たない物理インターフェースです。したがって、論理サブインターフェースの概念を使用します。この行にint g0 / 0と入力すると、システムはコマンドに2つのオプション、スラッシュ/またはピリオドを与えます。スラッシュは0/0/0などのインターフェースのモジュラー構成に使用され、ドットはサブインターフェースがある場合に使用されます。

行にint g0 / 0と入力すると、 ？の場合、システムは論理GigabitEthernetサブインターフェイスの可能な範囲の番号を提供します。これは、ポイントの後に示されます：<0-4294967295>。この範囲には40億を超える数が含まれています。つまり、非常に多くの論理サブインターフェイスを作成できます。



この後、VLAN10を表す10を示します。これで、CLI設定文字列のヘッダーがルーター（config-subif）＃に変更されたことからわかるように、サブインターフェイスの設定に移動しました。この場合、サブインターフェイスg0 / 0.10に関係しています。次に、IPアドレスを割り当てる必要があります。このIPアドレスには、ip add 192.168.1.1 255.255.255.0コマンドを使用します。このアドレスを設定する前に、カプセル化を実行して、使用するカプセル化プロトコル（802.1qまたはISL）を当社が作成したサブインターフェイスが認識できるようにする必要があります。行にカプセル化という単語を入力すると、このコマンドのパラメーターに使用できるオプションがシステムによって表示されます。



encapsulation dot1Qコマンドを使用しています。技術的には、このコマンドを入力する必要はありませんが、現時点ではスイッチがVLANトランキングを提供するスイッチとして機能しているため、VLANを使用するために使用するプロトコルをルーターに示すために入力します。このコマンドで、すべてのトラフィックをdot1Qプロトコルを使用してカプセル化する必要があることをルーターに示します。次に、コマンドラインで、このカプセル化がVLAN10に適用されることを示す必要があります。システムに使用されたIPアドレスが表示され、VLAN10のインターフェイスが機能し始めます。



同様に、g0 / 0.20インターフェイスをセットアップしています。新しいサブインターフェイスを作成し、カプセル化プロトコルを設定し、ip add 192.168.2.1 255.255.255.0コマンドでIPアドレスを設定します。



同時に、物理インターフェイスと論理サブインターフェイスがVLAN20ネットワークに対して同じアドレスを持っているため、物理インターフェイスのIPアドレスを削除する必要があります。これを行うには、コマンドint g0 / 1とIPアドレスなしを順に入力します。次に、必要がなくなったため、このインターフェイスをオフにします。



次に、再びg0 / 0.20インターフェイスに戻り、ip add 192.168.2.1 255.255.255.0コマンドを使用してIPアドレスを割り当てます。これですべてが確実に機能します。

次に、show ip routeコマンドを使用してルーティングテーブルを確認します。



ネットワーク192.168.1.0/24がGigabitEthernet0 / 0.10サブインターフェースに直接接続され、ネットワーク192.168.2.0/24がGigabitEthernet0 / 0.20サブインターフェースに直接接続されていることがわかります。次に、PC0コマンドラインターミナルに戻り、PC1にpingします。同時に、トラフィックはルータポートに到着し、対応するサブインターフェイスに転送して、スイッチ経由でPC1に送り返します。ご覧のとおり、pingは成功しています。最初の2つのパケットは破棄されました。ルーターインターフェイス間の切り替えには時間がかかるため、デバイスはMACアドレスを検出する必要がありますが、残りの2つのパケットは宛先に正常に到達しています。これが「スティックルーター」のコンセプトの仕組みです。
