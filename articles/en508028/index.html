What is a prototype IT product? What should it be like?



A prototype is a dynamic product model that simulates user interaction from interfaces, with which you can quickly and cheaply receive feedback from a customer or representatives of the target audience.







It is important to note that the prototype is not a design layout of the site (mocap) or wiring frame (a set of blocks for the future product). It may visually differ from the finished product. The most important thing in the prototype is that it is a working tool for testing hypotheses and logic, working out user scripts. That is, the prototype must be clickable.

What is UI / UX?



UX / UI design has existed since time immemorial and applies not only to applications and sites. It‚Äôs just that it was called differently before. For example, a Singer sewing machine. After all, someone else in the 19th century came up with how it should look and how to use it. At the same time, the goal that the user must achieve was to sew something fast, beautifully and efficiently.







So what are UI and UX design?



UI - stands for user interface and translates "user interface". This term is usually understood as the visualization of screen forms and interfaces. UI is about graphic design. UI-designer works with colors, composition, animation, fonts. Based on draft schematic sketches (wireframes), the UI designer collects the final design layout.



Uxor User Experience translates as "user experience." That is, it is about how to work with the interface, whether the user is able to achieve the goal, is it easy or difficult. UX designer transfers client experience to the interface, builds and tests hypotheses of user interaction with the system.



Under UX / UI design, realize any design user interfaces in which usability is just as important as appearance.

UX and UI designs are often confused, since they go side by side, but still these are different concepts that should merge into a single whole at the end of the path.







Link prototype and UX / UI design



Is the prototype of the UX / UI information system a design? In my opinion, is.

Is UI / UX design a prototype? Yes, but only if business logic is superimposed on the user interface in one way or another. If the user experience is described somewhere on a piece of paper and it is impossible to demonstrate it to the client on the interface, then we have separate UI and UX designs, but there is no prototype.







There is a good example of a prototype without software development. The pages of the site are drawn on sheets of paper. The experimenter sits down in front of the user and puts in front of him a sheet with the main page of the site. The user pokes a button on a button or points to a text field, saying what he enters there. For each user action, the experimenter replaces the previous sheet with a new one, which displays the response of the system to user actions.



UI is - these are sheets of paper with drawn buttons. UX is - this is user action. Development is - these are the actions of the experimenter. There is a prototype, since in this way you can check the user experience and get feedback from the client.



But what happens if you remove the experimenter and leave the user alone with sheets of paper? The UX / UI design will remain, and the prototype will not work. So, it will be impossible to test the logic of the system.

What is Figma?



Figma is a program for web designers, with which you can create not only prototypes, but also the final interfaces of sites and applications. The process of working in the program is intuitive, and the possibility of joint simultaneous work on a project attracts more and more teams to implement prototypes in Figma.



Pros and Cons of Figma



The advantages of working in Figma include:



  1. It is free if no more than 2 users work on the project.
  2. Creating and drawing a layout in one program
  3. There are a huge number of ready-made website templates
  4. To work with Figma, a fresh browser version is required, and the desktop version does not require a powerful and modern computer.
  5. Saving project versions up to 30 days on the free version


Cons Figma:



  1. The lack of Russification. Figma has to work in English.
  2. Although Figma works offline, synchronization with the cloud is required for colleagues to see the changes made.


An example of creating the simplest prototype in Figma



The simplest prototype in Figma can be created even by a student. Below, as an example of working with the tool, I will describe the process of creating a personal page. This will be a two-page site. On the first page I will place a small text block and a photo, on the second - contacts, a menu header will also be created.



Step 1. Registration



It is not necessary to register a separate account with Figma. If you have Google-mail, then you can go to the service with its help. At the initial opening Figma immediately offers to undergo a brief training on how to use the program.



Step 2. Creating a project



To create a new project, just click "+" in the upper right corner of the screen. There you can open one of the previously downloaded templates.



Step 3. Create a page workspace



Before creating a workspace for a future site or application, you need to determine on which devices it will be displayed. This is important, because on different models of monitors, smartphones and tablets, the layouts will be displayed differently and it may happen that your photo goes somewhere else.



But now we are not talking about adaptive design, but that creating a workspace, we are in fact creating the basis of the interface of the future prototype, and the user on the selected device type will see our layout as it looks in Figma. Separately, I note that in Figma the work area of ‚Äã‚Äãthe site is called a frame.



To create the basis of a future page, you must press the F key and in the menu that appears on the right, double-click on the type of device selected above.

We get a white sheet where you can change the size, background and other parameters and on which the page interface is created.







Step 4. Adding a picture



In Figma, it is very easy to add and crop pictures to any shape. To do this, just add the desired shape and image to the workspace, select both objects using the ctrl key combination and press ctrl + alt + M. It is important that the picture is higher than the figure. You can select shapes both on the workspace and in the panel on the left by clicking on their names.







Step 5. Adding a text field



To add text, you must press the key with the letter "T" and select a place for the text field on the workspace. Text block options can be changed in the ribbon on the right. For example, you can center the block in the center of the work area.







Step 6. Creating the menu



In this article, I do not pursue teaching the reader all aspects of working with Figma, so the menu we have will be the simplest - two-button. Buttons are rectangles combined with a text box. You can combine objects into a group by pressing CTRL + G, and then center the text field in the center of the future buttons.



Step 7. Duplicate the base of the page



The base of the page includes: workspace, menu and text box. To create a copy of its copy, you can remove the image mask outside the group of objects on the first page. To do this, left-click on the mask name and drag the icon up. Then, in the list of objects, select the group of objects included in the base of the first page, and press CTRL + D.







Step 8. Adding external links



Very often, single-page sites are created for only one purpose - to increase traffic to the main site. That is, the one-page is actually a link to a third-party resource. Unfortunately, while in Figma you can put a link only to the text. To do this, select the text field and click on the link icon at the top of the screen and enter the address of a third-party site.







Step 9. Adding links between pages



Adding links to objects inside Figma is similar to adding links to external sources, except that you need to take a link to an internal object somewhere. To do this, right-click on the object and select Copy / Paste and Copy Link from the drop-down menu.







Step 10. Demonstration of the prototype



The final step is to demonstrate the prototype to the customer. The easiest way is to click the ‚ÄúShare‚Äù button in the upper right corner of the screen and select ‚ÄúCopy Link‚Äù in the pop-up window.
