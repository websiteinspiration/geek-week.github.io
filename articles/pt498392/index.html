18 recursos do GitLab são de código aberto


Abrimos o código fonte dos recursos nos estágios Planejar, Criar, Verificar, Empacotar, Liberar, Configurar e Defender.


Sid Sijbrandij passou algum tempo estudando os recursos do GitLab e chegou à conclusão de que, de acordo com a estratégia para determinar os planos do GitLab direcionados ao comprador , o código fonte de dezoito recursos de sete estágios diferentes do ciclo de vida do DevOps deve estar aberto.


Quando, em 2018, introduzimos um novo modelo de planos GitLab, decidimos que os recursos serão distribuídos entre os quatro planos com base em quem é o consumidor desse recurso. Os recursos destinados a um usuário individual se enquadram no plano Core / Free, os recursos para gerentes se enquadram no Starter / Bronze, para diretores no Premium / Silver e para a gerência sênior no Ultimate / Gold.
