200人のためのハッカソン-整理する必要があるもの



大企業のプロジェクトが半年間行われる理由を知っていますか？なぜなら、最も遅いプロセスの1つは、顧客とのコミュニケーションがあり、顧客のニーズの詳細を特定するためです。仕様の簡単な説明（釘または接着剤を固定する必要があるため）には、最大3か月かかることがあります。もちろん、私は少し誇張していますが、実際には、手紙や電話で直接答えることはほとんど不可能です。私たちはすべての休日を待って集会をしなければなりません。

この領域の2つ目の機能は、自動化を開始するために、これが原則的に自動化できることを顧客が知っている必要があることです。そして彼は、安くても理解していなかった。技術開発のスピード（特にニューラルネットワーク検出器）を考えると、経験豊富な実業家のアイデアを超えるものもあれば、はるかに安価で手頃な価格のものもあります。しかし、それらを最初から作成するタスクは提起されていません。

私たちはこれを理解し、ハッカソンの組織と組織を支援した強力な産業パートナーであるSIBURのリーダーシップを理解しました。すでに行われていることと、自動化でできることとすべきことの間のギャップをなくす必要がありました。これを行うために、4つの面を一度に1つのサイトにまとめることにしました。

  1. 国で最大の工業企業。
  2. 変化する市場のテクノロジーベンダー。
  3. 若い開発者。
  4. 必要なフィールドまたは特定のテクノロジーの経験を持つITエンジニア。

重要なのは、大企業がタスクを伴うため、そのようなハッカソンの開発者はソリューションの概念を示しようとすることです。すべてが順調であれば、彼らはあなたが会社を設立することができるということに基づいて契約を取得します。しかし、顧客は2〜3日かけて質問に答えますが、テクノロジとソリューションの多くのプロトタイプを一度に非常によく把握できます。

ここでは、タスクとその解決策についてのレポートです。しかし、投稿自体は、イベントをどのように組織したかに関するものになります-おそらくこれはあなたのハッカソンに役立つでしょう。

タスクの収集に多くの時間が費やされました。簡単なテンプレートが記入され（以下は詳細になります）、専門家が追加の質問をしてから、タスクが公開されました。登録時に、参加者は1つ以上のタスクを選択し、ソリューションへのアプローチを送信しました。入り口では、解決する予定のチームの数とタスクを理解しました。スキームに従って承認された参加：1つのチームが1つの問題を解決します。私たちは、チームから送信されたソリューションの説明に基づいて事前に選択を行いました。これにより、イベントは顧客をパイロットする機会を持つことになります。

メンターのチームは非常に重要です。最初に、メンターと顧客は問題について質問され、次に技術スタックから特定の事柄の適用について質問し、次にソリューションに関するフィードバックを収集しました。実践が示しているように、ハッカソンの途中でワイヤーフレームを表示し、フィードバックを取得すること（または数回行うこと）は、非の打ちどころのない技術的ソリューションを実現することよりもはるかに重要です。顧客は間違いなく自分の分野に精通しており、ソリューション開発プロセスを劇的に変える何かを言うことができるからです。コンベヤー上のチョコレートの結婚を決定する場合、バーの理想的なモデルを特定する代わりに、欠陥のあるオプションについてモデルの追加トレーニングを使用し、偽陽性の割合を徐々に減らすことがより適切であることがわかりました。

提出物


一部のタスクでは、材料とツールが必要でした。はんだ付けゾーン、ビデオストリームを操作するためのゾーン、クラウド内の電源がありました。はんだ付けゾーンには、煙が発生する場合に備えてダイニングルームの排気システムを使用しました。



各タスクには、マテリアルの世界とソフトウェアのオブジェクトの独自のセットがありました。チョコレートの結婚を制御するために、これらの同じチョコレートのトレーニングサンプルの2つのボックスがありました。

彼らは大型モニターを提供しました-一部の参加者は事前に尋ね、一部はその場で尋ねました。私たちは設備を探すための完全な探求を開始し、週末には数十人の開発者が職場から譲り受け（すべてが月曜日の朝に再び設置されることを条件に）、一部はフィールドイベントファンドから使用されました。モニターは、ラップトップを接続するためだけでなく、Raspberry Piでの便利な操作のためにも必要でした。

当然、人々に食物を提供する必要がありました。温かいもの（それはダイニングルームで準備され、すべてがそこに行きました）、スナックとドリンク、そしてフィナーレのためのピザとビールとのディナーが必要です。私たちの間違いは、お菓子やペストリーがたくさんあり、野菜や果物はほとんどない他のハッカソンの間違ったデータをスナックに使用したことです。

スナックメニュー：カッテージチーズ、オートミールとナッツのクッキー、メレンゲ、ソフトドリンク、フルーツドリンク、新鮮なリンゴ。朝食：カッテージチーズのパンケーキ、ソースのクレープ（サワークリーム、フルーツ、コンデンスミルク）、チキンサンドイッチ、ハムとチーズのサンドイッチ、ヨーグルト。ランチ：素朴な方法でジャガイモを入れたミートボール、チキンヌードル、フライドポテトを添えた魚のコロッケ、ジャガイモを添えたグリルチキン、野菜を添えたヌードル、ジャガイモとピンクのサーモンスティック。夕食には、ビーフバーガーとフライドポテト、チキンケサディーヤがありました。また、ケータリングの便宜のため、多くの使い捨て食器が購入に含まれています。

スナックには11種類のチップ、ジュース、チョコレート、チョコレートバー、コーンスティック、チョコパイ、コーラゼロ、コーラ、スプライト、ジュース、5種類のクラッカーがあります。

部屋はダイニングルームオフィスの一部であり、そこから、屋外イベント用の家具店から、会議室からの家具がそれぞれあります。参加者のリラクゼーションのために、オンサイトエンジニア（サーバールームまたは近くで夜を過ごすことがある人々）は30枚のエアマットレスを推奨しました-購入したので便利でした。

また、グッズ（Tシャツや帽子）や、バッジ、パス、ステッカーパック（電報用を含む）など、さまざまな小さなものもたくさん作っていました。

技術ベンダースタック


ベンダーは、ハードウェアを持ち込むか、対応する問題を解決した人にライセンスを与えました。つまり、ベンダーのソリューションを使用することも使用しないことも可能でしたが、炭鉱作業のために家から二酸化炭素センサーを突然取り出さなかった場合は、設置されます。

PERGAMはセットを提供しました：ガス漏れセンサー、Raspberry Pi（2個）、5ボルトクーラー（Fsonsoning FSY50S05H 5V 0.25A 50 X 50 mm）、国産バイポーラトランジスタKT863A、一定抵抗1kΩ0.25 W-3個、取り付けワイヤMGTF 0.25平方mm-約3 m、ワイヤの切断とストリッピング用のツールセット-3セット、フラックス、はんだ、熱収縮チューブ、はんだ付けステーション：Ersa Digital 2000A-2個、CTブランドCT-936-1個、パワーバンク-3個。 、シリンダーCO。

参加者がクラウドパワーを必要とする場合、参加者は必要な特性を持つ仮想マシンを注文できます。通常の商用ユーザーと同じ方法で、仮想リソース管理コンソールへのアクセスが許可されました。ソフトウェアベンダーのテンプレートがありました。マシン-Intel Xeon Gold 6244プロセッサ、32コアと256 GBのRAM、および1万IOPSのフラッシュドライブを備えた構成が、当然のことながら最も一般的でした。合計で6つのアカウントが開かれ、20の仮想マシンが関与しました。ビデオを操作する参加者のために、AmazonでGPUマシンを購入しました。 Tesla V100 c 16 GBのグラフィックプロセッサメモリ、8つの仮想CPU、61 GBのRAM。このような機械は6台使用されました。
アクセスポイント、カメラ、Raspberry Piにビデオ認識用のソフトウェアも提供しました。

BitfuryExonumブロックチェーンフレームワークへのアクセスを許可しました-これは、ブロックチェーンで必要なすべてのものをすばやく整理する方法です。実際、Bitfuryライブラリを通じてブロックチェーンシステムを通常のデータベースとして扱うことができます。

BellSoftは、NVIDIA Jetson NanoおよびRaspberry Pi 4に基づいてデータを収集、分析、および管理するためのソリューションを提供しました。このソリューションには、Javaアプリケーションを起動するための開発環境がプリインストールされています-Liberica JDK。

シスコCisco VSMを与えた。このソリューションにより、誰もが問題なくビデオストリームを受信できます。VSMは、サーバーキャパシティでビデオストリームを収集し、必要に応じて保存、処理し、わずかな遅延で全員に配信します。同時に、ビデオストリームの需要が増加すると、ビデオ監視のパフォーマンスとは対照的に、生産性を向上させることができます。ストリームはサーバーに保存して後で表示できます。外部センサーまたはACSコントローラーを接続して、カメラの近くで起こっていることをより詳細に分析し、あらゆる種類のアラートを作成できます。一般に、ビデオを操作するための非常に優れたプラットフォームです。豚舎へのインストールが計画されただけです。

結果


350アプリケーション。 200人の参加者。 38チームの150人がハッカソンの決勝に臨みました。これらは、モスクワ、ノボシビルスク、サンクトペテルブルク、カザン、イジェフスクおよびその他の都市からの参加者です。セントピーターズバーグ州立大学のEmptySetチームは、ガスプロムネフトの課題を解決して遮断弁と制御弁の類似物を特定するプログラムを作成し、「最高の学生チーム」にノミネートされました。ほとんどのタスク作成者にとって、これは、たとえばチェリャビンスクのパイプ圧延プラント（ChTPZ）のハッカソンに参加した最初の経験でした。

このようなプロジェクトの管理は、常に大規模なチームの仕事です。ここでは、いくつかのコンポーネントが同時に重要になります。タスク計画、それらの実装への柔軟なアプローチ（アジャイルの原則はすべてです）、責任者の任命、プロジェクトのタイミングとマイルストーンです。プロジェクト管理には、Jiraを使用しました。



私たちはこのようなハッカソンの組織に参加し続けると思います-問題の著者にとっても参加者にとっても非常に役立つことがわかりました。これは、一般的に、ロシアのIT市場にとって重要です。
