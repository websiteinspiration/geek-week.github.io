Power Automate VS Logic Apps。一般情報

みなさん、こんにちは！今日はPower AutomateとLogic Appsについて話しましょう。多くの場合、人々はこれらのサービスの違いが何であるか、そしてサービスのどれが彼らの問題を解決するために選択されるべきかを理解していません。それを正しくしましょう。

Microsoft Power Automate


Microsoft Power Automateは、時間のかかるビジネスタスクとプロセスを自動化するスレッドを作成する機能をユーザーに提供するクラウドサービスです。このサービスは、100％開発者ではないが、アプリケーション開発とプロセスの自動化に関与している市民開発者向けです。

Microsoft Power AutomateはMicrosoft Power Platformの一部であり、Power Apps、Power BI、Power Virtual Agentsなどのサービスも含まれています。このプラットフォームにより、関連するOffice 365サービスから必要なすべての情報を簡単に取得し、それをアプリケーション、データストリーム、レポート、および補助ヘルパーサービスに組み合わせることができます。



Power Automateスレッドの作成は、「トリガー」=>「アクションセット」の概念に基づいています。ストリームは特定のトリガーによって開始されます。これには、SharePointリストでのアイテムの作成、メール通知の受信、HTTPリクエストなどがあります。開始後、このフローで構成されているアクションの処理が開始されます。アクションとして、さまざまなサービスへのコネクタを使用できます。現在、Microsoft Power Automateは、Google、Dropbox、Slack、WordPressなどの巨人の200を超えるさまざまなサードパーティサービスとサービス、およびさまざまなソーシャルサービス（Blogger、Instagram、Twitter、Youtube、Facebookなど）をサポートしています。もちろん、これに加えて、Office 365アプリケーションとの統合も可能です。Microsoft Power Automateの使用を簡素化するために、Microsoftは、必要なパラメーターのセットに入力するだけで使用できるさまざまなアプリケーションおよびイベント用の標準テンプレートを多数提供しています。また、ユーザーはデザイナーでテンプレートを自分で作成し、他のユーザーが使用できるように公開できます。

Microsoft Power Automateの特徴的な機能は次のとおりです。

  1. さまざまなサードパーティサービスへの多数のコネクタの存在。
  2. Office 365サービスとの統合のサポート。
  3. 統合シナリオなど、特定のトリガーに従ってストリームを開始する機能。Gmailのメールボックスへの手紙を受け取ったら、別のサービスで一連のアクションを開始する必要があります。たとえば、チームにメッセージを送信して、SharePointリストにエントリを作成します。
  4. 各ステージでのストリームの状態に関する詳細情報を含む、ストリームをデバッグする機能。

ただし、Microsoft Power AutomateはLogic Appsの簡易バージョンです。つまり、Power Automateストリームを作成すると、構成されたロジックを処理するLogic Appsストリームが内部で作成されます。簡単に言うと、Power AutomateはLogic Appsエンジンを使用してスレッドを実装します。

Microsoft Power Automateは現在、Office 365サブスクリプションの一部として、または別のプランで、ユーザーまたはスレッドによって購入できます。



プレミアムコネクタは、別のプランを購入しないと利用できないことに注意してください。Office 365サブスクリプションはプレミアムコネクタを提供していません。

ロジックアプリ


Logic Appsは、Azureアプリケーションサービスの一部であるサービスです。 Azure Logic Appsは、Azure APIにアクセスする機能を含むAzure Integration Servicesプラットフォームの一部です。 Power Automateと同様に、Logic Appsはビジネスタスクとプロセスを自動化するように設計されたクラウドベースのサービスです。ただし、Microsoft Power Automateはビジネスプロセスフローに重点を置いていますが、Logic Appsは統合された統合ソリューションの一部であるビジネスロジックブロックに重点を置いています。このような決定には、より慎重な管理と制御が必要です。 Logic Appsの主な違いの1つは、トリガーをチェックする頻度を指定できることです。 Power Automateにはこの設定はありません。

Logic Appsでは、Microsoft Power Automateを使用して、コードを記述せずにさまざまなレベルの複雑さのスレッドを作成できますが、ここでの価格は少し異なります。Logic Appsは、従量課金制のアプローチを使用しています。これは、個々のサブスクリプションを購入する必要がなく、すべてのコネクタがすぐに利用できることを意味します。ただし、ストリーム内でアクションを実行するたびに、一定の価値があります。



Logic Appsフローを開発するときは、標準コネクタと企業コネクタを実行するコストが異なることに注意してください。
