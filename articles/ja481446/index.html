5Gセキュリティアーキテクチャの概要：NFV、キー、2つの認証



明らかに、セキュリティメカニズムを考慮せずに新しい通信標準の開発に着手することは、非常に疑わしく、役に立たない問題です。

5Gセキュリティアーキテクチャ - 第5世代ネットワークに実装され、コアから無線インターフェースまでのすべてのネットワークコンポーネントをカバーする一連のセキュリティメカニズムと手順。

第5世代ネットワークは、本質的に第4世代LTEネットワークの進化形です。無線アクセス技術は最も重要な変化を遂げました。第5世代のネットワーク用に、新しいRAT（Radio Access Technology）が開発されました-5G New Radio。ネットワークコアについては、それほど大きな変化はありません。この点で、5Gネットワ​​ークのセキュリティアーキテクチャは、4G LTE規格で採用されている関連技術の再利用に重点を置いて開発されました。

ただし、無線インターフェースや信号プレーン（信号プレーン）への攻撃、DDOS攻撃、中間者攻撃などのよく知られた脅威の再考により、通信事業者は新しい標準を開発し、完全に新しいメカニズムを統合するようになりました。第5世代のネットワークセキュリティ。


バックグラウンド


2015年、国際電気通信連合は第5世代ネットワークの開発に関するこの種の最初のグローバル計画をまとめました。そのため、5Gネットワ​​ークでのセキュリティメカニズムと手順の開発の問題は特に深刻でした。

新しいテクノロジーは、非常に優れたデータ転送速度（1ギガビット/秒以上）、1ミリ秒未満の遅延、および半径1 km 2で約100万台のデバイスを同時に接続する機能を提供しました。第5世代ネットワークに対するこれらの最も高い要件は、組織の原則に反映されています。

主なものは分散化であり、ネットワークの周辺に多くのローカルデータベースとその処理センターを配置することを意味していました。これにより、M2Mでの遅延を最小限に抑えることができました。膨大な数のIoTデバイスのメンテナンスにより、通信とネットワークのコアの負荷を軽減します。このように、新世代ネットワークの境界は基地局にまで拡大され、重大な遅延やサービス拒否のリスクなしにローカルコミュニケーションセンターを作成してクラウドサービスを提供できるようになりました。当然のことながら、ネットワーキングへの変更されたアプローチと侵入者に関心のあるカスタマーサービス。これにより、サービス拒否を引き起こしたり、オペレーターのコンピューティングリソースを奪ったりするために、機密のユーザー情報とネットワークコンポーネント自体を攻撃する新しい機会が開かれました。

AMF（英語のアクセスとモビリティ管理機能から-アクセス制御とモビリティ）-提供：

  • コントロールプレーンインターフェイスの構成。
  • シグナリングトラフィックの交換の組織RRC、暗号化、およびそのデータの整合性の保護。
  • NAS信号トラフィックの交換の組織化、暗号化、およびそのデータの整合性の保護。
  • ネットワーク内のユーザー機器の登録を管理し、可能な登録条件を監視します。
  • ユーザー機器のネットワークへの接続を管理し、考えられる状態を監視します。
  • CM-IDLE状態のネットワーク上のユーザー機器の可用性の管理。
  • CM-CONNECTED状態のネットワーク上のユーザ機器のモビリティ管理。
  • ユーザー機器とSMF間のショートメッセージの送信。
  • 位置情報サービスを管理します。
  • EPSとの相互作用のためにEPSフロー識別子を強調表示します。

SMF（セッション管理機能）-以下を提供します：

  • アクセスネットワークとUPF間のトンネルのサポートを含む、通信セッションの管理、つまりセッションの作成、変更、解放。
  • ユーザー機器のIPアドレスの配布と管理。
  • 使用するUPFゲートウェイの選択。
  • PCFとの相互作用の組織。
  • QoSポリシーの管理。
  • DHCPv4およびDHCPv6プロトコルを使用したユーザー機器の動的構成。
  • 課金データの収集と課金システムとのやり取りの組織を監視します。
  • シームレスなサービス提供（SSCから-セッションとサービス継続性）。
  • ローミングの一部としてのゲストネットワークとの相互作用。

UPF（英語ユーザープレーン機能-ユーザープレーンの機能）-以下を提供します。

  • グローバルインターネットを含む外部データネットワークとの相互作用。
  • ユーザーパケットルーティング。
  • QoSポリシーに従ってパッケージにラベルを付ける。
  • ユーザーパッケージの診断（たとえば、署名ベースのアプリケーション検出）。
  • トラフィックの使用状況に関するレポート。
  • UPFは、同じ無線アクセステクノロジー内および異なる無線アクセステクノロジー間のモビリティをサポートするためのアンカーポイントでもあります。

UDM（統合データ管理-統合データベース）-以下を提供します：

  • ユーザーが利用できるサービスのリストと対応するパラメーターの保存と変更を含む、ユーザープロファイルデータの管理。
  • SUPI管理
  • 3GPP AKA認証資格情報の生成。
  • プロファイルデータに基づくアクセス許可（ローミング制限など）。
  • ユーザー登録管理、つまりサービスを提供するAMFのストレージ。
  • シームレスなサービスと通信セッションの維持、つまり現在の通信セッションに割り当てられたSMFの保存。

  • 複数の異なるUDMが、異なるトランザクションの一部として1人のユーザーにサービスを提供できます。

UDR（English Unified Data Repository-統合データのストレージ）-さまざまなユーザーデータのストレージを提供し、実際にはすべてのネットワークサブスクライバーのデータベースです。

UDSF（英語の非構造化データストレージ機能-非構造化データを保存する機能）-登録ユーザーの現在のコンテキストを保存するAMFモジュールを提供します。この情報は、一般に不確実な構造のデータとして表すことができます。ユーザーコンテキストを使用して、AMFの1つをサービスから計画的に撤回しているときと、緊急の場合の両方で、シームレスで中断のないサブスクライバーセッションを確保できます。どちらの場合も、スタンバイAMFは、USDFに保存されているコンテキストを使用してサービスを取得します。

同じ物理プラットフォーム上でUDRとUDSFを組み合わせることは、これらのネットワーク機能の典型的な実装です。

信頼ドメイン


第5世代ネットワークでは、要素がネットワークのコアから離れるにつれて、ネットワーク要素の信頼性が低下します。この概念は、5Gセキュリティアーキテクチャに実装された決定に影響します。したがって、ネットワークセキュリティメカニズムの動作を定義する5Gネットワ​​ーク信頼モデルについて説明できます。

ユーザー側では、信頼ドメインはUICCとUSIMによって形成されます。

ネットワーク側では、信頼ドメインの構造はより複雑です。

無線アクセスネットワークは、DU（英国の分散ユニット-分散ネットワークユニット）とCU（英国の中央ユニット-中央ネットワークユニット）の2つのコンポーネントに分かれています。一緒にgNBを形成-5Gネットワ​​ーク基地局無線インターフェース。安全でないインフラストラクチャのセグメントに展開できるため、DUはユーザーデータに直接アクセスできません。 CUは、ASセキュリティメカニズムのトラフィックの終端を担当するため、安全なネットワークセグメントに配置する必要があります。ネットワークの中核は、NASセキュリティメカニズムのトラフィックを終端するAMFです。現在の3GPP 5Gフェーズ1仕様では、AMFと、訪問した（サービスを提供している）ネットワークのルートキー（「アンカーキー」とも呼ばれる）を含むSEAFセキュリティ機能の組み合わせについて説明しています。Ausf認証が成功した後に取得したキーを保存します。ユーザーが複数の無線アクセスネットワークに同時に接続している場合は、再利用する必要があります。ARPFはユーザー資格情報を格納し、サブスクライバーのUSIMに相当します。UDRおよびUDMはユーザー情報を格納します。これは、資格情報、ユーザー識別子の生成、セッションの継続性の確保などのロジックを決定するために使用されます。

鍵の階層と配布スキーム


第5世代のネットワークでは、4G-LTEネットワークとは異なり、認証手順には、プライマリ認証とセカンダリ認証の2つのコンポーネントがあります。ネットワークに接続するすべてのユーザーデバイスにプライマリ認証が必要です。サブスクライバーが接続されている場合、外部ネットワークからの要求に応じて2次認証を実行できます。

1次認証が正常に完了し、ユーザーとネットワーク間の共有鍵K が作成された後、K SEAFが鍵Kから抽出されます。これは、サービス提供ネットワークの特別なアンカー（ルート）鍵です。その後、このキーからキーが生成され、RRCおよびNASシグナリングトラフィックの機密性とデータの整合性が保証されます。

一次認証


5Gネットワ​​ークでは、EAP-AKAおよび5G AKAが標準の主要な認証メカニズムです。プライマリ認証メカニズムを2つのフェーズに分けます。1つ目は認証の開始と認証方法の選択を担当し、2つ目はユーザーとネットワーク間の相互認証を担当します。


開始


ユーザーは、非表示のSUCIユーザーサブスクリプション識別子を含む登録要求をSEAFに送信します。

SEAFは、認証要求メッセージ（Nausf_UEAuthentication_Authenticate Request）を、SNN（Serving Network Name）とSU​​PIまたはSUCIを含むAUSFに送信します。

AUSFは、要求しているSEAFがこのSNNの使用を許可されているかどうかを確認します。サービングネットワークがこのSNNの使用を承認されていない場合、AUSFは「承認されていないサービングネットワーク」承認エラーメッセージ（Nausf_UEAuthentication_Authenticate Response）で応答します。

AUSF認証資格情報は、SUPIまたはSUCIおよびSNNを介してUDM、ARCFまたはSIDFによって要求されます。

SUPIまたはSUCIとユーザー情報に基づいて、UDM / ARPFは後で使用される認証方法を選択し、ユーザーの資格情報を提供します。

相互認証


任意の認証方法を使用する場合、UDM / ARPFネットワーク機能は認証ベクトル（AV）を生成する必要があります。

EAP-AKA：UDM / ARPFは、最初に分離ビットAMF = 1の認証ベクトルを生成し、次にCK、IK、およびSNN からCK 'およびIK'を生成し、新しい認証ベクトルAV（RAND、AUTN、XRES *、CK '、IKを作成します'）。EAP-AKAにのみ使用するように指示とともにAUSFに送信されます。 5G AKA：UDM / ARPFキー取得KのAUSFからCK、IK及びSNNを、次いで生成

5G HE AV英語5G家庭環境認証ベクター）。 5G HE AV認証ベクトル（RAND、AUTN、XRES、K AUSF）がAUSFに送信され、5G AKAにのみ使用する必要があることを示します。

この後、AUSFはアンカーキー取得K SEAFキーからK AUSFをしても、RAND、AUTNとRES *を含むメッセージ「Nausf_UEAuthentication_Authenticateレスポンス」、でSEAF「チャレンジ」にリクエストを送信します。次に、RANDとAUTNは、安全なNASシグナリングを使用してユーザー機器に送信されます。ユーザーUSIMは、受信したRANDおよびAUTNからRES *を計算し、それをSEAFに送信します。 SEAFはこの値を検証のためにAUSFに中継します。

AUSFは、格納されているXRES *とユーザーから受信したRES *を比較します。一致した場合、オペレーターのホームネットワークのAUSFとUDMに認証が成功したことが通知され、ユーザーとSEAF はK SEAFとSUPI から独立してK AMF鍵を生成し、さらに通信します。

二次認証


5G標準は、ユーザー機器と外部データネットワーク間のオプションのEAP-AKAベースの二次認証をサポートします。この場合、SMFはEAP認証システムの役割を果たし、ユーザーを認証および承認する外部ネットワークのAAAサーバーの操作に依存します。

5Gセキュリティアーキテクチャは既存のテクノロジーの再利用に基づいているという事実にもかかわらず、まったく新しい課題に直面しています。膨大な数のIoTデバイス、拡張されたネットワーク境界、および分散型アーキテクチャの要素は、サイバー犯罪者の想像力を自由に利用できる5G標準の主要な原則の一部にすぎません。

主な5Gセキュリティアーキテクチャ標準はTS 23.501バージョン15.6.0です。-セキュリティのメカニズムと手順の作業の重要なポイントが含まれています。特に、ユーザーデータとネットワークノードの保護の確保、暗号化キーの生成、および認証手順の実装における各VNFの役割について説明します。しかし、この標準でさえ、通信事業者がより頻繁に直面する緊急のセキュリティの質問に対する回答を提供していません。新世代のネットワークがより集中的に開発され、運用されています。

この点で、第5世代ネットワークの運用と保護の困難は、母の息子の息子のように伝送速度と応答が約束されている一般ユーザーに影響を与えず、新世代ネットワークの発表されたすべての機能を試してみることができないと信じたいと思います。
