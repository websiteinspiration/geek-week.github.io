アプリケーションフレームワークをインストールする


PHP開発者がLAMP（Linux Apache + Mysql + PHP）を使用してワークフローを加速し、Rails開発者が事前に準備したスタックを使用するように、Kubernetes開発者はアプリケーションフレームワークを使用できます。
PLONKスタックは、Prometheus、Linux、OpenFaaS、NATS、およびKubernetesの略です。
  • Prometheusは、システムの正常性をテストするためのメトリック、自動スケーリング、および可観測性を提供し、ゼロへのスケーリングに関する決定を行うためのメトリックを提供することにより、需要の急増に対応し、コストを削減できるようにします。
  • Linuxは、Kubernetesでワークロードを実行するための唯一のオプションではありませんが、標準で最も使いやすいオプションです。
  • OpenFaaSは当初、開発者にポータブルな機能を提供していましたが、APIとマイクロサービスをデプロイするときにうまく機能します。その汎用性は、HTTPサーバーを備えた任意のDockerコンテナーをデプロイおよび管理できることを意味します。
  • NATSは、メッセージングとパブ/サブに使用される人気のあるCNCFプロジェクトです。PLONKスタックでは、非同期でリクエストを実行し、キューイングする機能を提供します。
  • Kubernetesが私たちがここにいる理由です。これは、スケーラブルで自己修復可能な宣言型インフラストラクチャを提供します。そして、このすべての壮大さの一部が必要なくなった場合、そのAPIはOpenFaaSで簡素化されます。

func insert(description string) error {
       res, err := db.Query(`insert into todo (id, description, created_date) values (DEFAULT, $1, now());`,
               description)
       if err != nil {
               return err
       }
       defer res.Close()
       return nil
}

データスキーマに日付の追加フィールドがあるので、Todo構造を更新します。

type Todo struct {
   ID int `json:"id"`
   Description   string `json:"description"`
   CreatedDate   *time.Time `json:"created_date"`
   CompletedDate *time.Time `json:"completed_date"`
}


次に、selectTodos()リクエストコードをメソッドに追加します。

func selectTodos() ([]Todo, error) {
       rows, getErr := db.Query(`select id, description, created_date, completed_date from todo;`)
   if getErr != nil {
       return []Todo{}, errors.Wrap(getErr, "unable to get from todo table")
   }
   todos := []Todo{}
   defer rows.Close()
   for rows.Next() {
       result := Todo{}
       scanErr := rows.Scan(&result.ID, &result.Description, &result.CreatedDate, &result.CompletedDate)
       if scanErr != nil {
           log.Println("scan err:", scanErr)
       }
       todos = append(todos, result)
   }
   return todos, nil
}


以前と同様に、リクエストの行を閉じるのを遅らせる必要があります。各値は、rows.Scanメソッドを使用して新しい構造に挿入されます。メソッドの最後に、Todoコンテンツがあります。
