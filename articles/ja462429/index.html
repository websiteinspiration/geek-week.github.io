VueJs + VueRouter +モーダル。別の自転車

こんにちは。この記事では、住所を変更するときに必要に応じてモーダルウィンドウを表示する方法を分析します。公式ウェブサイトにモーダルコンポーネントの専用ソリューションがあることを知っています。これに加えて、VueJのモーダルウィンドウのトピックに特化したHabréに関する記事は他にもたくさんあります（たとえば、この記事）。

ただし、私の意見では、それぞれに欠点があります。たとえば、次のとおりです。

  • オフサイトで説明されているコンポーネントの場合、別の変数を作成してバインドする必要があります。
  • 2番目の記事では、サードパーティのソリューションについて説明します。この場合、時間を費やして、これらのコンポーネントの使用方法を理解する必要があります。私の意見では、プロジェクト内の依存関係を追加できます。そして私はその立場を堅持します。シンプルなほど良いです。

繰り返しになりますが、私はすべてを知ることはできません。ですから、何か言いたいことや指摘することがあれば、建設的な批判を受け入れるようにしています。さらに、使用するツール（Vue、VueRouter、Bootstrap Modal）の標準機能のみを使用します。なので、ポイントに近づけると......

「すべてのブートストラップ+ jqueryをプルするよりも、モーダルのnpm依存関係を接続する方がよい」と言う人もいるでしょう。同志、誰もあなたのニーズとツールにこの全体を適応させることを気にしません。上記のツールは、実装の例にすぎません。

すぐに機能するオプション。

HTML
<div id="app">
  <nav>
    <router-link :to="{ name: 'modal' }" exact>Open Modal</router-link>
  </nav>
  
  <router-view></router-view>
</div>


JavaScript
Vue.use(VueRouter)

const Modal = {
	template: `<div class="modal fade" tabindex="-1" role="dialog" ref="modal">
  <div class="modal-dialog" role="document">
    <div class="modal-content">
      <div class="modal-header">
        <h5 class="modal-title">Modal title</h5>
        <button type="button" class="close" data-dismiss="modal" aria-label="Close">
          <span aria-hidden="true">×</span>
        </button>
      </div>
      <div class="modal-body">
        <p>Modal body text goes here.</p>
      </div>
      <div class="modal-footer">
        <button type="button" class="btn btn-secondary" data-dismiss="modal">Close</button>
      </div>
    </div>
  </div>
</div>
mounted: function(){
	console.log('mounted')

	var context = this;

	$(this.$refs.modal).modal('show')
  
  $(this.$refs.modal).on('hidden.bs.modal', function(){
  	context.$router.go(-1);
  })
}
}


const routes = [
  { path: '/modal', name: 'modal', component: Modal },
]

const router = new VueRouter({
	routes,
})

// New VueJS instance
var app = new Vue({
	// CSS selector of the root DOM element
  el: '#app',
  // Inject the router into the app
  router,
})


実際、複雑なことは何もありません。

  1. リンクをクリックすると、モーダルコンポーネントがマウントされます。
  2. マウントするときに、モーダルウィンドウを開き、これを介して$ Refsがそのクローズを追跡します。
  3. それが閉じられると、プログラムによって人が1つ前のステップに戻り、コンポーネントがマウント解除されます。

したがって、独自のロジックを持つ多数のモーダルウィンドウを使用する場合は、このようなアプローチと構造が優れたソリューションです。

  • 簡単なカスタマイズ。
  • サードパーティの依存関係をロードする必要はありません。私個人的には、シンプルなほど良いです。
