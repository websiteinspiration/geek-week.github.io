Mac Pro 1.1にVMware ESXiをインストールする

この記事では、古いApple Mac Pro 1.1にVMware ESXiをインストールした経験について説明します。 タスクは顧客によって設定されました-ファイルサーバーを拡張することです。会社のファイルサーバーが2016年にPowerMac G5でどのように作成され、作成されたレガシーを提供する方法は、別の記事に値します。拡張機能とアップグレードを組み合わせて、既存のMacProからファイルサーバーを作成することが決定されました。また、Intelプロセッサ上にあるため、仮想化も実行できます。 タスクはかなり実行可能ですが、私は多くの困難に直面し、それらのソリューションでビットごとのデータを収集する必要がありました。また、解決策の検索は、「VMwareにMac OSをインストールする」という逆の問題の結果によってしばしば曖昧になりました。 得られた経験を統合し、すべての穀物を1か所に集め、ロシア語に翻訳するために、この記事が作成されました。









読者要件：HPサーバーなど、VMware ESXiと互換性のあるハードウェアへのVMware ESXiのインストールに精通している。アップルのテクニックに精通してください。特に、MacProの組み立て・分解の詳細については触れていませんが、微妙なニュアンスがあります。

1.ハードウェア


MacPro 1.1、別名MA356LL / A、別名A1186-インテルプロセッサーからの最初のアップルコンピューターは2006-2008年に製造されました。 10年以上の年齢にもかかわらず、コンピューターは優れた健康状態にあります。 4人の最も強力なファンはどれもうるさくありません。標準のクリーニングと分解が必要です。

プロセッサー-2つのデュアルコアXeon5150。完全に64ビットのアーキテクチャ、ただしEFIブートローダーは32ビット。これは非常に重要です、人生は非常に有毒です！

RAM-標準の4GB PC5300 DDR2 ECC667MHz。16GBまで簡単に拡張できます。サーバーのメモリは古いHP gen.5-6からのものであり、実際のところ、コンピューターはこのサーバーと非常に似ていますが、状況が異なります。

HDD-3.5インチ（LFF）用の4つのバスケット。いくつかの物理的な変更により、2.5 "（SFF）詳細については、[8] Apple Mac Pro 1.1のSSDを参照してください。

IDE DVDもあり、最大5枚の5.25インチフォーマットです。ただし、SATAコネクタもあります。マザーボードでは、ODD SATA（ODD = Optical Disk Drive）と呼ばれています。私の実験では、この場所にハードドライブとSSDをインストールできること、およびインストールする必要があることがわかっています。

2. ESXiの選択


VMware の互換性テーブルを使用すると、Xeon 5150が最大ESXi 5.5 U3をサポートしていることがわかります。インストールするのはこのバージョンです。

ESXi 6.0では、「廃止された」すべてのサポートが廃止されました。公式には、6.7のような新しいバージョンをここに配置することは実際には不可能です。インターネット上でこれが成功しているという言及がありました。しかし、今回はそうではありませんが、プロセッサーの非互換性は強力な魔術であると私は考えています。本番環境では、これは実験のためにのみ可能です。

ESXiの新しいバージョンの場合、ファイルの補完方法は同じです。

3.配布ファイルの完成


ディストリビューションは標準を採用しました。それはサイトから可能です、それは急流から可能です。ESXi 5.5 U3。

ただし、完全に64ビットのアーキテクチャに注意を払いましたが、EFIブートローダーは32ビットですか？そして、ここで彼は会います。インストーラーをダウンロードしようとしても、何も起こりません。
インストーラーブートローダーを古い32ビットブートローダーに置き換える必要があります。5.0よりも前のバージョンのようです。

これについては、記事[2] Mac ProとESXi 5.0のインストールとの互換性について詳しく説明しています。BOOTIA32.EFIファイルはそこから取得されます。

iso編集プログラム（たとえば、ultraiso）を使用します。iso内の\ EFI \ BOOTフォルダーを見つけ、BOOTIA32.EFIファイルを古いファイルに置き換えて保存します。これで、すべてが既に読み込まれています。



4. ESXiをインストールする


詳細はありません、いつものように。インストールは正常に完了しましたが、何もロードされていません。これは正常です。

5.ブートローダーファイルのファイナライズ


アクションのアルゴリズムは、記事[3] ESXi 6.0で古いMac Proを復活させるに示されています。32ビットブートファイルアーカイブへのリンクもあります。

5.1。ハードドライブを取り外し、別のコンピューターに接続します。

sata-usbアダプタを備えたMacBookのハードウェアバージョンを使用しました。Linuxを使用できます。別のコンピューターがない場合は、別のトレインを使用して、MacProにプラグインし、MacOSをインストールして、そこからESXiハードドライブをマウントできます。

Windowsは使えません！このディスクがWindowsシステムに含まれると、わずかな変更が要求なしで行われます。小さく、誰も気にしないでください。しかし私たちの場合、ESXiのダウンロードは「Bank6はvmwareブートバンクではなく、ハイパーバイザーが見つかりません」というエラーで終了します。



それでも、[4]バンク6がVMwareブートバンクではなく、ハイパーバイザーが見つからなかった場合に何が起こるかについての詳細が記載された記事です。しかし、解決方法はシンプルで迅速です。もう一度ESXiをインストールしてください！

5.2 EFIパーティションをマウントする


ターミナルを開き、必ずスーパーユーザーモードに切り替えてください

Sudo –s

今後のセクション用のディレクトリを作成する

mkdir  /Volumes/EFI

利用可能なセクションを見てください

diskutil list

これが私たちが必要とするもの、ESXi



Mount it と呼ばれるEFIセクションです

mount_msdos /dev/disk2s1 /Volumes/EFI

マウントされたドライブで、ファイルを古いバージョンに置き換える必要があります。古いバージョンは[3]にあります。32ビットブートファイルアーカイブ

置換用ファイル：

/EFI/BOOT/BOOTIA32.EFI
/EFI/BOOT/BOOTx64.EFI
/EFI/VMware/mboot32.efi
/ EFI / VMware / mboot64。 efi完了し



たら、マウントされたEFIパーティションを切断します

umount -f /Volumes/EFI
