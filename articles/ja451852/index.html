RDPプロトコルでの任意のコードのリモート実行



RDPプロトコルの危険な脆弱性について知られるようになりました。Microsoftは、CVE-2019-0708という識別子を持つ脆弱性に対する緊急パッチを用意しました。これにより、ターゲットシステムで任意のコードを実行できます。

認証されていない攻撃者がRDPを使用してターゲットシステムに接続し、特別に細工された要求を送信する場合、リモートデスクトップサービス（以前のターミナルサービス）にリモートでコードが実行される脆弱性が存在します。この脆弱性は事前認証を使用し、ユーザーの操作を必要としません。攻撃者がこの脆弱性を悪用した場合、標的のシステムで任意のコードを実行する可能性があります。

脆弱性（CVE-2019-0708）は、Windows 7、Windows Server 2008 R2、Windows Server 2008など、サポートされているバージョンのWindowsに組み込まれている「リモートデスクトップサービス」コンポーネントに存在します。また、Windows XPおよびMicrosoftがずっと前にセキュリティアップデートの配布を停止したWindows 2003、オペレーティングシステム。

この脆弱性を悪用するには、攻撃者は特別に細工したリクエストをRDP経由でターゲットシステムのリモートデスクトップサービスに送信する必要があります。

興味深い事実は、これまたは同様の脆弱性が少なくとも昨年の9月以降ダークネットで販売されていることです。
