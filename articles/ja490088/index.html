UE4でマップを作成し続ける方法

前回の記事では、Yandex.Mapセクションを参照として使用してレベルを作成する方法について説明しました。大きな地図を最適化する方法として、道路の「舗装」と、ワールドコンポジションを使用して遭遇したそれらのニュアンスについて話すことはまだ残っています。



道路


もちろん、引き伸ばされたテクスチャレベルによって「描かれた」道路は、使用するのに非常に適しています（たとえば、エアシミュレータの場合）。ただし、自動車シミュレータの場合は、より詳細なアプローチが必要です。UEで道路を作成するのに最も適したコンポーネントは、少なくとも「長い連続セクション」であるスプラインです。交差点やジャンクションにはまだ遭遇していません。おそらく手で別々のオブジェクトを作成する必要があります。


スプラインには、ブループリントスプラインとランドスケープスプラインの 2つのタイプがあります。前者は途方もないカスタマイズ機能を備えており、よく分解されています。チャンネルのエピックゲーム。彼らの助けを借りて、道路沿いのオブジェクト（標識、フェンス、排水溝、送電線）の作成を一度に制御できますが、ブループリントスプラインで風景と組み合わせる機能が見つかりませんでした。また、道路の各メーターを風景の下の手に合わせる（またはその逆）ことはできません。あなたがプロジェクトを始めたいもの。したがって、レベルの最初のバージョンで道路を作成するには、「スプラインの編集ツール」を使用してテクスチャをクリックするだけで、ランドスケープスプラインを選択しました。 2つのスプラインが判明しました。1つはアスファルト道路用、もう1つは未舗装道路用です。交差点は単に重ね合わせるだけで作られます。スプラインを「具体化」するには、セグメントがメッシュを設定する必要があります。私は完成したものをビークルゲームから取り出しましたが、トラックもランドスケープスプライン（エピックゲームランチャー>アンリアルエンジン>ライブラリ>ボールト）のように作成され、トラックから複雑な部分を素材から切り出しました。それは素晴らしい未舗装の道路であることが判明しました！アスファルトに対してこのようなトリックを繰り返すことはできなかったので、同じメッシュを取り、マテリアルのテクスチャを未舗装の道路からアスファルトの道路に変更しました（このように）、Zスケール= 0.2のスプラインに適用すると、かなりうまくいきました。最後に、スプラインの下の風景を変形することを忘れてはなりません。これはボタン1つだけで実行され、道路の準備が整います。

#include "MyGameInstance.h"
#include "Engine/World.h"
#include "Engine/WorldComposition.h"
#include "Engine/LevelStreaming.h"

void UMyGameInstance::LoadComplete(const float LoadTime, const FString & MapName)
{
	Super::LoadComplete(LoadTime, MapName);
	
	UWorld* World = GetWorld();
	
	// WorldComposition won't work in Standalone mode without next statement
	if ( World->WorldComposition ) {
		const TArray<ULevelStreaming*> TilesStreaming = World->WorldComposition->TilesStreaming;
		World->ClearStreamingLevels();
		World->SetStreamingLevels(TilesStreaming);
	}
	
	// "Pause" game till streaming complete
	GEngine->BlockTillLevelStreamingCompleted(World);
}
