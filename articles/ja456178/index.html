前書き


標準UI要素のデフォルトデザインがデザイナーに適しているとは限りません。SwitchCompatの例を使用して、要素の外観を変更する方法を見てみましょう。

私たちが必要とする問題を解決するには：

  • SwitchCompatの独自のスタイルを作成します。
  • どういうわけか、このSwitchCompatスタイルを設定します。

たとえば、SwitchCompatスタイルを設定するにはいくつかの方法があります。

  • 画面レイアウトの各ビューについて、style属性を使用して指定します。
  • オーバーライドされたswitchStyle属性でテーマを作成し、このテーマをアプリケーション全体または特定のアクティビティのマニフェストに割り当てます。これにより、アプリケーション/アクティビティ全体のビューの外観が変更されます。
  • トピックは、アクティベーションコードでプログラムで設定することもできます。必要に応じて、オンザフライで変更できます。

SwitchCompatの新しいスタイル




リソースで、新しいMySwitchStyleスタイルを作成し、親を設定してWidget.AppCompat.CompoundButton.Switchからデザインを継承します。継承することはできませんが、変更する予定がない場合でも、すべての値を指定する必要があります。

<style name="MySwitchStyle" parent = "Widget.AppCompat.CompoundButton.Switch">
</style>

何かを変更するには、必要な属性を再定義する必要があります。属性はドキュメントにあります。

ドキュメントには、いくつかの属性があります。それらは、コードでアクセスしているように示されます（たとえば、R.styleable.SwitchCompat_android_thumbなど）。間違いないように一部のみ解読します。残りの目的は、ドキュメントから理解しやすいです。
コードで
 XMLで
SwitchCompat_android_thumb
 android：親指
SwitchCompat_thumbTint
 thumbTint

 追跡
SwitchCompat_trackTint
 trackTint

  • android：thumb-SwitchCompatの可動部分のリソース
  • track-SwitchCompatの固定部分のリソース
  • thumbTint-SwitchCompatの状態に応じて、希望する色で可動部分をペイントできます
  • trackTint-SwitchCompatの状態に応じて、静止部分を目的の色に着色できます

例として、つまみ（円）の色を変更します。オンのときはオレンジ、オフのときは緑にします。醜いが、明らかに。

リソースのcolorフォルダーにセレクターが必要です。ファイルselector_switch_thumb.xml

<selector xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:state_checked="true" 
              android:color = "@android:color/holo_orange_dark"/>
    <item android:color="@android:color/holo_green_light"/>
</selector>

次に、thumbTint属性をスタイルに設定します。

<style name="MySwitchStyle" parent = "Widget.AppCompat.CompoundButton.Switch">
        <item name="thumbTint">@color/selector_switch_thumb</item>
</style>

これで、何らかの形でMySwitchStyleスタイルを取得したすべてのSwitchCompatが新しい方法で表示されます。



レイアウトスタイル


最も平凡で柔軟性のない方法。

  • スタイルは、レイアウトリソースを拡張するときに適用されます。
  • ソフトウェアに影響を与えることはできません。
  • レイアウトで毎回示すのは不便です。そして、私たちは忘れることができます。

<androidx.appcompat.widget.SwitchCompat
            android:text="Themed switch"
            style="@style/MySwitchStyle"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"/>

主題のスタイル。マニフェストを介して割り当てられたトピック


AppThemeテーマを作成し、switchStyle属性の値を設定します。値はMySwitchStyleスタイルです。

<resources>
    <style name="CustomTheme" parent="Theme.AppCompat.Light.DarkActionBar">
        <item name="colorPrimary">@color/colorPrimary</item>
        <item name="colorPrimaryDark">@color/colorPrimaryDark</item>
        <item name="colorAccent">@color/colorAccent</item>
        <!--    switchStyle -->
        <item name="switchStyle">@style/MySwitchStyle</item>
    </style>
</resources>

テーマは、アプリケーション全体のマニフェストで指定できます。

<application
            android:allowBackup="true"
            android:icon="@mipmap/ic_launcher"
            android:label="@string/app_name"
            android:roundIcon="@mipmap/ic_launcher_round"
            android:supportsRtl="true"
            android:theme="@style/CustomTheme"><!--     -->
</application>

または特定の活動のために

<activity 
           android:name=".MainActivity"
           android:theme="@style/CustomTheme"><!--    -->
</activity>

これで、すべてのSwitchCompatの外観が新しくなります。レイアウトに変更はありません。

  • 長所-アプリケーション全体の外観をすぐに変更できます。
  • 短所-その場でそれを変更することはできません。

主題のスタイル。テーマはプログラムで割り当てられます


プログラムでアクティブ化のテーマを設定するには、アクティブ化メソッドsetTheme（themeResId）を呼び出す必要があります。

スイッチの状態に応じてアクティビティのテーマを変えましょう。

private const val KEY_CUSTOM_THEME_CHECKED = "KEY_CUSTOM_THEME_CHECKED"

class MainActivity : AppCompatActivity() {

    private val preference by lazy {
        PreferenceManager.getDefaultSharedPreferences(this)
    }

    override fun onCreate(savedInstanceState: Bundle?) {
        val isCustomThemeChecked = preference.getBoolean(
            KEY_CUSTOM_THEME_CHECKED,
            true
        )

        if (isCustomThemeChecked) {
            setTheme(R.style.CustomTheme)
        } else {
            setTheme(R.style.StandardTheme)
        }
        super.onCreate(savedInstanceState)

        setContentView(R.layout.activity_main)
        customThemeCheckbox.isChecked = isCustomThemeChecked
        customThemeCheckbox.setOnCheckedChangeListener { _, isChecked ->
            preference.edit()
                .putBoolean(KEY_CUSTOM_THEME_CHECKED, isChecked)
                .apply()
            recreate()
        }
    }
}

  1. setThemeを呼び出して、テーマをプログラムでインストールします。このメソッドは、super.onCreate（savedInstanceState）の前に呼び出す必要があります。onCreateでは、フラグメントを初期化します（フラグメントがある場合）。
  2. トピックに応じて、スイッチの初期状態を設定します。
  3. Switchを変更すると、設定のテーマを変更し、recreate（）アクティベーションメソッドによってアクティビティを再開するリスナーをインストールします。
