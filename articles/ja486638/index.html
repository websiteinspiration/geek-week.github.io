Blazor WebAssembly 3.2.0 Preview 1が利用可能になりました

本日、Blazor WebAssemblyの新しいプレビューアップデートをリリースしました。多くの優れた新機能と拡張機能が含まれています。

このリリースの新機能は次のとおりです。

  • バージョンが3.2に更新されました。
  • 簡略化された起動
  • ダウンロードサイズの改善
  • .NET SignalRクライアントのサポート



開始


Blazor WebAssembly 3.2.0 Preview 1を開始  するには、.NET Core 3.1 SDK  をインストールしてから、次のコマンドを実行します。

dotnet new -i Microsoft.AspNetCore.Blazor.Templates::3.2.0-preview1.20073.1

それで全部です！https://blazor.netで追加のドキュメントと例を見つけることができ  ます。

既存のプロジェクトを更新する


既存のBlazor WebAssemblyアプリケーションを3.1.0プレビュー4から3.2.0プレビュー1にアップグレードするには：

  • Microsoft.AspNetCore.Blazor *パッケージへのすべてのリンクを3.2.0-preview1.20073.1に更新します。
  • で  Program.csの  Blazor WebAssemblyクライアントプロジェクトでは、交換してください  BlazorWebAssemblyHost.CreateDefaultBuilder() と  WebAssemblyHostBuilder.CreateDefault()。
  • Blazor WebAssemblyクライアントプロジェクトのルートコンポーネントの登録を移動  Startup.Configure する  ことにより、Program.csを  呼び出します  builder.RootComponents.Add<TComponent>( )。
  • Blazor WebAssembly  Startup.ConfigureServices  Program.cs   builder.Services.
  •  Startup.cs  Blazor WebAssembly client project.
  • Blazor WebAssembly ASP.NET Core,  Server   app.UseClientSideBlazorFiles<Client.Startup>(...)  app.UseClientSideBlazorFiles<Client.Program>(...).

3.2


このリリースでは、Blazor WebAssemblyパッケージのバージョンを3.2にアップグレードして、.NET Core 3.1長期サポート（LTS）の最近のリリースと区別しました。 .NET Core 3.2の対応するリリースは存在しません-新しいバージョン3.2はBlazor WebAssemblyにのみ適用されます。 Blazor WebAssemblyは現在.NET Core 3.1に基づいていますが、.NET Core 3.1 LTSのステータスを継承しません。代わりに、今年5月に予定されているBlazor WebAssemblyの最初のリリースはCurrentであり、.NET Core Support Policyに記載されているように、「CurrentまたはLTSの次のリリースから3か月間維持されます」。 5月のリリース3.2に続くBlazor WebAssemblyの次のリリースは、.NET 5での予定です。。つまり、.NET 5のリリース後は、Blazor WebAssemblyアプリケーションをサポートするために.NET 5にアップグレードする必要があります。

簡略化された起動


このリリースでは、Blazor WebAssemblyの起動とホスティングAPIを簡素化しました。当初、Blazor WebAssemblyの起動および展開APIは、ASP.NET Coreで使用されるパターンを反映するように設計されていましたが、すべての概念が関連しているわけではありませんでした。更新されたAPIでは、いくつかの新しいシナリオを使用することもできます。Program.csの

新しい起動コードは次のようになります。

public class Program
{
    public static async Task Main(string[] args)
    {
        var builder = WebAssemblyHostBuilder.CreateDefault(args);
        builder.RootComponents.Add<App>("app");

        await builder.Build().RunAsync();
    }
}

Blazor WebAssemblyアプリケーションはMain、アプリケーションへのエントリポイントの非同期メソッドをサポートするようになりました。

デフォルトのホストを作成するには、を呼び出しますWebAssemblyHostBuilder.CreateDefault()。ルートコンポーネントとサービスは、リンカーを使用して構成されます。別のクラスはStartup必要ありません。次の例が追加されたWeatherServiceため、依存関係注入（DI）を介して使用できます。

public class Program
{
    public static async Task Main(string[] args)
    {
        var builder = WebAssemblyHostBuilder.CreateDefault(args);
        builder.Services.AddSingleton<WeatherService>();
        builder.RootComponents.Add<App>("app");

        await builder.Build().RunAsync();
    }
}

ホストが作成されたら、コンポーネントが表示される前に、ルートDI領域からサービスにアクセスできます。これは、何かが表示される前に初期化ロジックを実行する必要がある場合に役立ちます。

public class Program
{
    public static async Task Main(string[] args)
    {
        var builder = WebAssemblyHostBuilder.CreateDefault(args);
        builder.Services.AddSingleton<WeatherService>();
        builder.RootComponents.Add<App>("app");

        var host = builder.Build();

        var weatherService = host.Services.GetRequiredService<WeatherService>();
        await weatherService.InitializeWeatherAsync();

        await host.RunAsync();
    }
}

ホストは、アプリケーションの中央構成のインスタンスも提供します。構成にはデフォルトのデータは入力されていませんが、アプリケーションで必要に応じて設定できます。

public class Program
{
    public static async Task Main(string[] args)
    {
        var builder = WebAssemblyHostBuilder.CreateDefault(args);
        builder.Services.AddSingleton<WeatherService>();
        builder.RootComponents.Add<App>("app");

        var host = builder.Build();

        var weatherService = host.Services.GetRequiredService<WeatherService>();
        await weatherService.InitializeWeatherAsync(host.Configuration["WeatherServiceUrl"]);

        await host.RunAsync();
    }
}

ダウンロードサイズの改善


Blazor WebAssemblyアプリケーションは、各アセンブリで.NET ILリンカーを実行して、アプリケーションから未使用のコードを削除します。以前のリリースでは、主要なフレームワークライブラリのみが切断されていました。このリリース以降、Blazorアセンブリもトリミングされ、サイズがわずかに削減されます（約100 KB）。以前と同様に、リンクを無効にする必要がある場合は<BlazorLinkOnBuild>false</BlazorLinkOnBuild> 、プロジェクトファイルに独自のものを追加します。

.NET SignalRクライアントのサポート


これで、.NET SignalRクライアントを使用してBlazor WebAssemblyアプリケーションからSignalRを使用できます。

Blazor WebAssemblyアプリケーションでSignalRを試すには：

  1. ASP.NET CoreでホストされているBlazor WebAssemblyアプリケーションを作成します。

    dotnet new blazorwasm -ho -o BlazorSignalRApp
  2. ASP.NET Core SignalRクライアントパッケージをクライアントプロジェクトに追加します。

    cd BlazorSignalRApp
dotnet add Client package Microsoft.AspNetCore.SignalR.Client
  3. サーバープロジェクトで、次のHub / ChatHub.csクラスを追加します。

    using System.Threading.Tasks;
using Microsoft.AspNetCore.SignalR;

namespace BlazorSignalRApp.Server.Hubs
{
    public class ChatHub : Hub
    {
        public async Task SendMessage(string user, string message)
        {
            await Clients.All.SendAsync("ReceiveMessage", user, message);
        }
    }
}
  4. サーバープロジェクトで、メソッドにSignalRサービスを追加しますStartup.ConfigureServices。

    services.AddSignalR();
  5. ChatHubat のエンドポイントも追加しますStartup.Configure。

    .UseEndpoints(endpoints =>
{
    endpoints.MapDefaultControllerRoute();
    endpoints.MapHub<ChatHub>("/chatHub");
    endpoints.MapFallbackToClientSideBlazor<Client.Program>("index.html");
});
  6. クライアントプロジェクトのPages / Index.razorを次のマークアップで更新します。

    @using Microsoft.AspNetCore.SignalR.Client
@page "/"
@inject NavigationManager NavigationManager

<div>
    <label for="userInput">User:</label>
    <input id="userInput" @bind="userInput" />
</div>
<div class="form-group">
    <label for="messageInput">Message:</label>
    <input id="messageInput" @bind="messageInput" />
</div>
<button @onclick="Send" disabled="@(!IsConnected)">Send Message</button>

<hr />

<ul id="messagesList">
    @foreach (var message in messages)
    {
        <li>@message</li>
    }
</ul>

@code {
    HubConnection hubConnection;
    List<string> messages = new List<string>();
    string userInput;
    string messageInput;

    protected override async Task OnInitializedAsync()
    {
        hubConnection = new HubConnectionBuilder()
            .WithUrl(NavigationManager.ToAbsoluteUri("/chatHub"))
            .Build();

        hubConnection.On<string, string>("ReceiveMessage", (user, message) =>
        {
            var encodedMsg = user + " says " + message;
            messages.Add(encodedMsg);
            StateHasChanged();
        });

        await hubConnection.StartAsync();
    }

    Task Send() => hubConnection.SendAsync("SendMessage", userInput, messageInput);

    public bool IsConnected => hubConnection.State == HubConnectionState.Connected;
}
  7. サーバープロジェクトを作成して実行する

    cd Server
dotnet run
  8. 2つの別個のブラウザータブでアプリケーションを開き、SignalRを介してリアルタイムで通信します。

既知の問題点


以下は、このリリースの既知の問題のリストであり、将来のアップデートで修正される予定です。

  • コマンドラインからASP.NET Coreでホストされている新しいBlazor WebAssemblyアプリケーションを起動すると、警告が生成されます。

    CSC : warning CS8034: Unable to load Analyzer assembly C:\Users\user\.nuget\packages\microsoft.aspnetcore.components.analyzers\3.1.0\analyzers\dotnet\cs\Microsoft.AspNetCore.Components.Analyzers.dll : Assembly with same name is already loaded.
    • 解決策：この警告は、<DisableImplicitComponentsAnalyzers>true</DisableImplicitComponentsAnalyzers> MSBuild プロパティを使用して無視または削除できます  。
