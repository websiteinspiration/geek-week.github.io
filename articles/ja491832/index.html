React Nativeと比較したFlutterの8つのプラス

モバイル開発は、ネイティブ開発ではなく、複数のプラットフォーム用のアプリケーションを一度に作成するのに役立つフレームワークの使用にますます依存しています。今日は、React Nativeから徐々に市場に勝ち始めているFlutterについて少しお話ししたいと思います。詳細-カットの下。



モバイルアプリケーションを開発するプロジェクトが開始されたら、どのプラットフォームをプログラムするかを決定する必要があります。多くの場合、この質問は、各プラットフォーム用にネイティブに作成されたソフトウェアを見たい、または逆にお金を節約してReact Nativeでの開発を求めている顧客自身の影響を受けます。

プロジェクトのウェブサイトで、FlutterがGoogleだけでなく他の会社も積極的に使用していることがわかります。たとえば、Flutterは新しいニューヨークタイムズアプリだけでなく、モバイルバージョンのeBayやAliexpressも作成しています。しかし、一般的に、このリストは20社以下しか含まれていないため、それほど印象的に見えません。

それにもかかわらず、当社の開発者の経験から、モバイルアプリケーションの作成にはFlutterの方が優れていて便利な場合があります。Netiの集合的な心は、この若いフレームワークの合計8つのプラスです。

パフォーマンス


Flutterで開発されたアプリケーションは、React Nativeを使用するよりもはるかに高速に実行されます。スピードの利点は、独自のエンジンと別のプログラミング言語を使用することで実現されます。ブリッジとJava Script言語で動作するReact Nativeとは異なり、Flutterを使用すると、アプリケーションの問題、特にユーザーインターフェイスに関連する問題をすばやく解決できます。ネイティブコンパイラはすでにフレームワークに組み込まれており、Dartで記述されたコードはARMプロセッサのネイティブ命令になります。

ちなみに、興味深い比較テストがInVeritaの人たちによって行われました。Mediumに関する彼らの記事は、実際のiPhone 6とXiaomi Redmi Note 5デバイスで完成したアプリケーションのパフォーマンスを比較しました。



ご覧のとおり、iOSでのガウステストは、SwiftよりもFlutterでさらに高速に実行され、RNは20倍の遅れを示しています。



Androidの場合、Flutterはテストを完了するのにJavaやKotlinよりも20％長い時間を必要としますが、RNの15倍の速度で動作します。

アプリケーションサイズ


前の段落の結果として、Flutterに基づいて作成され、Dartで記述されたアプリケーションには、中間コンポーネントがほとんど含まれていません。ネイティブARM命令までコンパイルすると、これ以上何も格納できなくなります。その結果、アプリケーションバイナリ自体は、React Nativeで開発する場合と比べて、デバイスのドライブ上で最大2分の1のスペースしか消費しません。

クイックアニメーション


ある意味で、このプラスは前のものに続きます。多くのアニメーションが組み込まれているアプリケーションにはFlutterを使用することにしました。実際のところ、Flutterは60 FPSを安定して示しています。つまり、React Nativeでよくあるように、ユーザーがぴくぴく動く画像を見る必要がないということです。

型付きプログラミング言語


Flutterの基礎は、すでに愛好家の間で人気のある型付きプログラミング言語であるDartです。Dartでは、コーディングはオブジェクト指向プログラミングに変わります...そしてもちろん、ここでTypeScriptを思い出して、それも型指定されていると主張することができます。しかし、悲しいことに、最終的にはトランスコンパイルされて、まだ型なしのJavaScriptに変わります。つまり、TSのすべての利点はエンコーダーの利便性レベルにとどまり、パフォーマンスに影響を与えません。対照的に、ダートはネイティブに型付けされます。プログラミング言語は積極的に開発され、コミュニティによってサポートされています。Dartでははるかに多くの構造コードを記述できます。つまり、より複雑なアプリケーションや階層構造を作成できます。

優れたドキュメント


インターネット上の新しいプラットフォームを扱いたい人のために、ビデオチュートリアルがあり、コードをオンラインで操作する可能性が考えられています。Googleが率いるコミュニティは、包括的な説明とガイドを作成しています。さらに、ネットワークには、React NativeからFlutterに切り替えるための既製のレシピさえあります。こちらでご覧いただけます。もちろん、React Nativeのドキュメントもありますが、「removeClippedSubviews-これにより、大きなリストのスクロールパフォーマンスが向上する可能性がある」などの説明があるため、役に立たないことがよくあります。Flutterコミュニティは、ドキュメントに対してより真剣なアプローチをとっています。

均一UI


React Nativeのよく知られた利点は、一度に複数のプラットフォームの開発を実行できることです。ただし、ファイルでファイナライズしないとすぐに同じインターフェイスが得られることを誰も保証しません。対照的に、Flutterで作成されたアプリケーションは、iOSとAndroidの両方で外観と動作が同じです。私たちのプログラマーは、AndroidのようにiOSでアプリケーションを機能させるためにコードを書き直したことはありません。React Nativeの場合、すべてが異なります。そのような改善はうらやましいほどの規則性で行われなければなりません。

多数のプラットフォームのサポート


FlutterはすでにAndroid、iOS、Webをサポートしています。コミュニティでの議論から判断すると、MacOS、ChromeOS、Windowsのサポートは間もなく利用可能になる予定です。そして最も興味深いことに、Flutterはすでに将来のGoogle Fuchsiaプラットフォームをサポートしています。そのため、あらゆる開発をすぐに新しいOSに移植できます。

開発者ツール


最後に注目したいのは、開発されたDevToolsです。Flutterは、開発者ツールの優れたエコシステムを誇っています。ここでそれらを見ることができます。ちなみに、Facebookは同様のことを試みていますが、今のところ、RNはDevToolsのベータ版しか持っておらず、機能的にはFlutterには対応していません。

コード例


たとえば、以下はコード「Hello、World！」です。React NativeおよびFlutterフレームワーク

// React Native
import React from 'react';
import { StyleSheet, Text, View } from 'react-native';

export default class App extends React.Component {
  render() {
    return (
      <View style={styles.container}>
        <Text>Hello world!</Text>
      </View>
    );
  }
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center'
  }
})

Flutterには短所がありますか？


もちろんそうです。 Flutterは若いプラットフォームです。十分な既製のソリューションはありません。比較のために、同じReact Nativeにはすでに多くの既製のテンプレート、レシピ、実装オプションがあります。

FlutterでのUIの記述には時間がかかります。そして、Dartコードの記述自体が長くなります。 React Nativeでは、アプリケーションはWebコンポーネントからコンストラクターとして構築されます。 Flutterはまだ完全なプログラミングを必要とします。

さらに、Flutterにはより深いプログラミングスキルが必要です。 React Nativeの場合、Java Scriptの知識を持つ専門家に開発を委託できます。 Dartを使用すると、この言語の入力しきい値がJSよりも高いため、状況はさらに複雑になります。プログラマーには、より詳細なトレーニングが必要です。

しかし同時に、Flutterの開発者の求人はまだかなり少ないです。新しいプラットフォームでのロシア企業の信頼はまだありません。ただし、これにより、このフレームワークの開発の一部を実施できなくなるわけではありません。そして、これまでの結果は肯定的な以上のものです。Flutterでの開発経験がある場合は、コメントで感想をお聞かせください。
