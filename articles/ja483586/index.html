Pythonでzmqを操作する基本、単純なキー/値ストアを作成する

前書き


memcacheなどの単純なキー/値のストレージの例を見てみましょう。それは単純に配置されます-データはハッシュマップ構造でメモリに保存されます。それらへのアクセスはtcpソケット経由です。Pythonでは、hashmapは通常の辞書です。アクセスには、zeromqを使用します。

カスタマイズ


このパッケージをdebian / ubuntuにインストールするには、コンソールに入力するだけです
sudo apt-get install libzmq-dev
sudo pip install zmq

コード


サーバーを操作するためのクラスを作成します。
使用されるzmqソケットのタイプはREQ（REQuest、要求）で、要求を送信します-応答を待機しています。
ネットワーク上であらゆるタイプのデータを保存および転送するために、標準のピクルモジュールを使用します。仕事の「プロトコル」-3つの値のタプル：（コマンド、キー、データ）

import zmq
import pickle

class SuperCacher:
    def __init__(self):
        context = zmq.Context()
        self.socket = context.socket(zmq.REQ)
        self.socket.connect('tcp://127.0.0.1:43000')

    def get(self, key):
        self.socket.send(pickle.dumps(('get', key, None)))
        return pickle.loads(self.socket.recv())

    def set(self, key, data):
        self.socket.send(pickle.dumps(('set', key, data)))
        return self.socket.recv() == b'ok'


使用する

cache = SuperCacher（）
cache.set（ 'key'、 'value'）
cache.get（ 'key'）

宿題として、クラスのインスタンスを作成するときにアドレス/ポートを指定する機能を追加して、実装を改善します。

次に、サーバー自体を記述します。
今回はREPソケットが使用されます（REPly、応答）-要求を待っています。ヘルメットが答えです。リクエストを解析し、書き込みの場合は「ok」、読み取りの場合はdata / Noneのいずれかで応答します。

import pickle
import json
import zmq

def run_daemon():
    memory = {}

    context = zmq.Context()
    socket = context.socket(zmq.REP)
    socket.bind('tcp://127.0.0.1:43000')

    while True:
        try:
            command, key, data = pickle.loads(socket.recv())
            if command == 'set':
                memory[key] = data
                socket.send(b'ok')
            elif command == 'get':
                result = memory.get(key, None)
                socket.send(pickle.dumps(result))
        except Exception as e:
            print(e)

if __name__ == '__main__':
    run_daemon()


すべてを一緒にテストするに
python daemon.py

は、次のコマンドでサーバーを起動します。次のタブで、対話モードでpythonを実行します。

>>> from lib import SuperCacher
>>> cache=SuperCacher()
>>> cache.set('key', 'value')
True
>>> cache.get('key')
'value'
