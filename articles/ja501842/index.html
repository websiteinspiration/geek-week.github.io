L2モードでのMetalLBのルーティングの微調整


少し前に、MetalLBのルーティングを構成するという非常に非標準的なタスクに直面しました。すべては何もないでしょう、なぜなら MetalLBは通常、追加のアクションを必要としませんが、この場合、非常に単純なネットワーク構成を持つかなり大きなクラスターがあります。


この記事では、クラスターの外部ネットワークにソースベースおよびポリシーベースのルーティングを構成する方法について説明します。


MetalLBのインストールと設定については詳しく説明しません。すでにある程度の経験があると思います。私は要点に直行すること、つまりルーティングを構成することを提案します。したがって、4つのケースがあります。


ケース1：構成が不要な場合


簡単なケースを分析してみましょう。



MetalLBによって発行されたアドレスがノードのアドレスと同じサブネットにある場合、追加のルーティング構成は必要ありません。


たとえば、サブネット192.168.1.0/24があり、ルーター192.168.1.1があり、ノードがアドレスを取得します192.168.1.10-30。MetalLBの場合、範囲192.168.1.100-120を構成して、追加の構成なしでそれらが機能することを確認できます。


何故ですか？ノードにすでにルートが設定されているため：

192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.10
