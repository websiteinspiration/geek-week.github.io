STM32のLua

こんにちは！

マイクロコントローラで何かをすぐに試したい、アイデアの小さな実用的なプロトタイプをプログラムしたい場合があります。これらの目的には、ご存知のように、スクリプト言語が適しています。この記事では、Embox を使用してSTM32でLuaインタープリター（eLuaではなく標準）を実行する方法を説明します。実例として、luasocketライブラリを使用してネットワーク上でLEDを点滅させ、httpで少し作業します。



Luaはスクリプトプログラミング言語であり、そのインタープリターは他のプロジェクトに簡単に統合できるほど軽量で、無料のMITライセンスがあります。私たちは長い間この言語に興味を持っていたため、Emboxの下ではqemu i386のLuaがサポートされています。インタプリタは軽量でPOSIX互換であるため、Pjsip、OpenCV、Qtなどの他のライブラリで行ったのと同じ方法でLuaを起動するというアイデアが生まれました。

私はすぐに私たちはluasocketサポートを開始しますので、それに注意しましょう、私たちは「北韓ないことを選択したstm32f4発見 - （1メガバイトのフラッシュ、192キロバイトのRAM）が、もう少しstm32f7発見を（1 MBフラッシュ、320 KB RAM、および追加のSDRAMがあります）。ネットワークサポートのないLuaの基本バージョンがstm32f4で簡単に開始できることは間違いありません（以下を参照）。

はじめに、簡単な例を準備しましょう-n番目のフィボナッチ数を計算します。

function fib(n)
    if n < 3 then
        return 1
    else
        return fib(n - 1) + fib(n - 2)
    end
end

print("fib(7) = " .. fib(7))

最初にLinuxで起動して、どれだけのメモリが必要かを見てみましょう。その後、変更せずにEmboxに転送します。まず、新鮮なluaをダウンロードします。5.3.5をダウンロードし、「make linux」としてコンパイルしました。しかし、実際には、私たちの目的のために、リポジトリから置くことができます。次に、fib.luaを実行します。

$ valgrind --tool=massif --massif-out-file=fib.massif lua fib.lua
$ ms_print fib.massif > fib.out

これで、fib.outを調べて、ヒープに割り当てられたメモリの最大量が約30 KBであることがわかります。ここではもちろん、これが最小サイズであるとは言いません。たとえば、「RAMの要件」セクションの記事では、Luaマシンのスイッチを入れたときのみ、要件が大幅に少なくなっています。しかし、いずれにしても、30 Kbは有望に見えます。320Kbです。

Emboxのarm / stm32f746g-discovery-luaテンプレートをビルドします。

$ make confload-arm/stm32f746g-discovery-lua
$ make -j $(nproc)

ファームウェアのビルド/ベース/ビン/ emboxは、私たちのwikiに示されているようにボード上で実行します。

embox> lua fib.lua 
fib(7) = 13

すばらしい、すべてがLinuxとまったく同じです（驚くべきことではありません:)）。

タスクを複雑にしてみましょう。しかし、ネットワークはほとんどどこでも必要です。インターネット上のどこかにHTTP / GETリクエストを送信します。これを行うには、luasocketライブラリが必要です。その後、ソースコードからの収集を開始しませんでしたが、既成のパッケージ（ "sudo apt-get install lua-socket"）を使用しました。

HTTPリクエストを送信するには、次の簡単な例を使用します。

local http = require("socket.http")

if not arg or not arg[1] then
    print("lua http_request.lua <url>")
    os.exit(0)
end

local body, code = http.request(arg[1])
print(code)
print(body)

os.exit(code == 200)

繰り返しになりますが、すべてLinuxで確認できます。

lua http_request.lua http://example.com

コード200とページコンテンツを返します。それを記憶から見てみましょう。valgrindを再度実行すると、ヒープ上のピークメモリ消費量が242 Kbに増加したことがわかります。そこで、ここで私はようやくstm32f4が追加の最適化なしには適合しないことを発見しました。stm32f7に乗ることができるようです-320 Kbありますが、ファイルシステムとネットワークを備えたOSがまだあることを考慮して、外部SDRAMの束を使用して少し工夫することにしました。

通常のヒープをRAMに追加し（構成に既に含まれています）、SDRAMに追加します。

    include embox.mem.static_heap(heap_size=0x10000)
    include embox.mem.fixed_heap(heap_size=0x40000, heap_start=0x60000000)

stm32f7-discoveryを確認します。Linuxと同じように機能します。

最後にLEDを「リモート」に点滅させます。

local socket = require("socket")

port = arg[1] or 1027
udp = assert(socket.udp())
assert(udp:setsockname('*', port))
print("Lua UDP server started on port " .. port .. "...")
while 1 do
    cmd, ip, port = assert(udp:receivefrom())
    print("Execute '" .. cmd .. "' from " .. ip .. ":" .. port)
    os.execute(cmd)
end

ここでは、指定したポートからUDPデータを取得し、それをos.execute（）に渡します。実行するコマンドとして、クライアントからピンコマンドを送信します。これはEmboxのシンプルなコマンドで、GPIOを制御します。たとえば、stm32f7-discoの場合、「pin gpioi 1 toggle」を実行する必要があります。これは、1行目をGPIOIに変更することを意味します。

luasocketの標準的な例をクライアントとして取り上げます。

echoclnt.lua
-----------------------------------------------------------------------------
-- UDP sample: echo protocol client
-- LuaSocket sample files
-- Author: Diego Nehab
-----------------------------------------------------------------------------
local socket = require("socket")

host = "localhost"
port = 1027
if arg then
    host = arg[1] or host
    port = arg[2] or port
end

udp = assert(socket.udp())
assert(udp:setpeername(host, port))
print("Using remote host '" ..host.. "' and port " .. port .. "...")
while 1 do
    line = io.read()
    if not line or line == "" then os.exit() end
    assert(udp:send(line))
end


いつものように、まずLinuxを確認します。Valgrindは、ヒープのピークメモリ消費量が約70 Kbであることを示しています。これは、フィボナッチの場合（30 Kb）の2倍以上ですが、HTTPの例（242 Kb）の場合よりも大幅に少なくなっています。

Emboxでの起動例をすぐに示します。

Linuxの場合：

$ lua echoclnt.lua 192.168.0.128 1027
Using remote host '192.168.0.128' and port 1027...
pin gpioi 1 toggle
pin gpioi 1 toggle

Emboxで：

embox>                                                   
embox>lua udp_server.lua                                                        
Lua UDP server started on port 1027...
Execute 'pin gpioi 1 toggle' from 192.168.0.102:34300
Execute 'pin gpioi 1 toggle' from 192.168.0.102:34300

結論


stm32f7-discoveryでLua + luasocketを実行できました。そして、彼らの助けを借りて、ネットワークを介してリモートでLEDを点滅させます。そして、最初に述べたように、スクリプト言語はプロトタイピングの時間を節約できるため、それは非常に単純であることがわかりました。

また、OSなしでベアメタルで実行されるeLuaプロジェクトがあることにも注意してください。しかし、彼にはいくつかのニュアンスがあります。まず、限定的なネットワークサポート：UDPは存在せず、ドキュメントでは、ネットワークは現在「進行中」であると述べています。次に、stm'kuタイプF3、F4（F1のみ）のサポートが見つかりませんでした。そして一般的に、あなたは同意しなければなりません、広くサポートされているプロジェクトを使用する方がはるかに便利です、そしてこの点で、eluaはもちろんメインプロジェクトよりも劣っています。

記事に記載されているすべての例は、リポジトリにあります。

Emboxには他のスクリプト言語もあります-python（tinipy）、Ruby（mruby）、Tcl。しかし、これはこの記事の範囲を超えています:)
