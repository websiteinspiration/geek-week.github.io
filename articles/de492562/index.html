18. März - Wie Sie verstehen, was Ignite SQL für Sie richtig ist, und es verwenden


Wenn Sie mit Einschränkungen hinsichtlich Größe, Skalierbarkeit oder Leistung Ihrer relationalen Datenbank konfrontiert sind, ist es möglicherweise an der Zeit, ein Upgrade auf ein verteiltes System durchzuführen. Apache Ignite ist eine verteilte Plattform, die als Datenbank fungieren kann und SQL- und JCache-APIs für die Arbeit mit Daten bereitstellt. In diesem Webinar werden Beispiele aus der Praxis betrachtet und die Vor- und Nachteile der einzelnen Ansätze erörtert:

  • SQL-Funktionen in Ignite
  • Wie SQL mit der Java-API interagiert (Laden über die JCache-API über SQL und umgekehrt)
  • Konsistenzgarantien (Ignite SQL und JCache API)
  • Die Komplexität von verteiltem SQL und wie Daten durch Colocation minimiert werden können.

Registrierung für das Webinar "Wie Sie verstehen, was Ignite SQL für Sie richtig ist, und es verwenden"

25. März - Distributed Computing mit Apache Ignite


Um das volle Potenzial der In-Memory-Plattform auszuschöpfen, reicht es häufig nicht aus, Daten in den Cluster zu laden und mit ihnen über die Schlüsselwert- oder SQL-API zu arbeiten. Sie müssen Daten auch effizient in einem Cluster platzieren und verteiltes Computing verwenden, das die Datenbewegung im Netzwerk minimiert. In diesem Webinar lernen Sie, wie Sie Berechnungen unter Berücksichtigung aller Vor- und Nachteile verteilter Systeme entwerfen und durchführen.

Wir werden überlegen:

  • Effiziente Verteilung von Informationen in einem Cluster mithilfe der richtigen Datenmodellierung und Kollokation;
  • Wie die MapReduce-API in Apache Ignite implementiert ist und funktioniert;
  • So minimieren Sie Betriebsverzögerungen mithilfe der Datenkolokation mit Berechnungen;
  • Wie SQL-Abfragen in einer verteilten Umgebung effizient ausgeführt werden können.

Das Webinar ist nützlich für diejenigen, die sich mit Apache Ignite vertraut machen und lernen möchten, wie sie die verteilten Computerfunktionen dieser Plattform optimal gestalten und nutzen können.

Registrieren Sie sich für das Distributed Computing mit Apache Ignite- Webinar

14. April - Handbuch zu Sicherheitseinstellungen für Apache Ignite und GridGain mit Beispielen


Apache Ignite und GridGain ermöglichen schnelles Rechnen und hocheffiziente verteilte Datenabfragen. Sowohl Ignite als auch GridGain bieten eine flexible Konfiguration, die dazu beiträgt, den Clusterbetrieb sicherer zu machen. In diesem Webinar werden die folgenden Sicherheitsthemen behandelt:

  • Sichere Verbindung zwischen Knoten (SSL / TLS);
  • Benutzerauthentifizierung
  • Benutzerberechtigung.

Anhand von Beispielen werden Konfigurationen betrachtet für:

  • SSL / TLS zum Herstellen einer sicheren Verbindung zwischen allen Knoten, Thin Clients und Verbindungen mithilfe von REST;
  • Authentifizierung durch Apache Ignite Advanced Security;
  • GridGain-Sicherheits-Plugin.

Das Webinar hilft Entwicklern und Entwicklern, die Grundlagen der Sicherheit für Apache Ignite und GridGain zu verstehen, und gibt Konfigurationsbeispiele für die häufigsten Fälle.
