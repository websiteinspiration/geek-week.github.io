グループ関数

テーブルEmployees を使用した集約データのレポート。最小の給与と最大の給与、職場への到着の早い日と遅い日、および従業員数を含むdepartment_idに関するレポートを取得します。従業員数でソート（降順）
決定
  SELECT department_id,
         MIN (salary) min_salary,
         MAX (salary) max_salary,
         MIN (hire_date) min_hire_date,
         MAX (hire_date) max_hire_Date,
         COUNT (*) count
    FROM employees
GROUP BY department_id
order by count(*) desc;


従業員テーブル。名前が同じ文字で始まる従業員は何人ですか？数量で並べ替えます。数量が1より大きい場所のみを表示
決定
SELECT SUBSTR (first_name, 1, 1) first_char, COUNT (*)
    FROM employees
GROUP BY SUBSTR (first_name, 1, 1)
  HAVING COUNT (*) > 1
ORDER BY 2 DESC;


従業員テーブル。同じ部門で同じ給与を受け取っている従業員は何人ですか。
決定
SELECT department_id, salary, COUNT (*)
    FROM employees
GROUP BY department_id, salary
  HAVING COUNT (*) > 1;


従業員テーブル。曜日ごとに雇用された従業員数のレポートを取得します。数量で並べ替え

    FROM employees
GROUP BY TO_CHAR (hire_Date, 'Day')
ORDER BY 2 DESC;

結合を使用して複数のテーブルのデータを表示する

SELECT *
  FROM employees

テーブルの従業員、部門、場所。従業員の総収入が少ない都市を手に入れましょう。
決定
SELECT city
    FROM employees e
         JOIN departments d ON (e.department_id = d.department_id)
         JOIN locations l ON (d.location_id = l.location_id)
GROUP BY city
  HAVING SUM (salary) =
         (  SELECT MIN (SUM (salary))
              FROM employees e
                   JOIN departments d ON (e.department_id = d.department_id)
                   JOIN locations l ON (d.location_id = l.location_id)
          GROUP BY city);

決定
SELECT *
  FROM employees
 WHERE     manager_id IN (SELECT employee_id
                            FROM employees
                           WHERE TO_CHAR (hire_date, 'YYYY') = '2005')
       AND hire_date < TO_DATE ('01012005', 'DDMMYYYY');


従業員テーブル。マネージャーが任意の年の1月に就職した従業員のリストを取得します。これらの従業員のjob_titleの長さは15文字を超えています
決定
SELECT *
  FROM employees e
 WHERE     manager_id IN (SELECT employee_id
                            FROM employees
                           WHERE TO_CHAR (hire_date, 'MM') = '01')
       AND (SELECT LENGTH (job_title)
              FROM jobs
             WHERE job_id = e.job_id) > 15;
