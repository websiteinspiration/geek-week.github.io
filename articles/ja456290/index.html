電報 無制限のネットワークドライブ。自由

こんにちは、ハブル。

きっと多くの人がそれについて考えました、多分誰かがTODOでこの考えを持っています。彼女は私と一緒に約1年間横たわっていましたが、それでも機能するプロトタイプの形で実装することができました。

TgCloud：


  • オープンソースの仮想ファイルシステムg ****コード。
  • ローカルディスク-メタデータのみ：名前、サイズ、フォルダー構造など
  • データはテレグラムに保存され、ファイルを使用する場合にのみダウンロードされます
  • ファイルのサイズとタイプは制限されず、どのOSでも使用できます

実装の詳細と猫の下のリポジトリへのリンク。



考え


Telegramに基づいてVFSを作成するというアイデアは、AudioTubeBotとVideoTubeBotで作業しているときに思いつきました。ボットのファイルをダウンロードする際の50MBの制限を回避する方法を見つけた後、Telegramのソースコードを調べたところ、多くの興味深いことがわかりました。たとえば、ダウンロード速度は主にクライアント自体に制限されており、サーバーには制限されていません。

その結果、Telegramクラウドへのダウンロード速度をセッションあたり最大240Mbit / s（1年以上安定して動作しています）に増やしました。これは、公式クライアントの速度の15倍の速さです。

論理的な手順は、ダウンロード速度の速い無制限のファイルストレージをネットワークドライブとして使用することでした。多くの場合、そうしますが、問題は、電報クライアントを使用してファイルを検索および整理するのが不便なことです。また、ファイルサイズごとに1.5 GBの制限があります。

ヒューズ


これらの問題を解決するために、最初にLinuxカーネルパッチを作成し、メタデータのみを使用してTelegramにアップロードされたファイルを便利にグループ化する独自の仮想ファイルシステムを作成することにしました。

この問題を調査する過程で、カーネルにパッチを当てる必要はまったくないことがわかりました。Linuxには特別なFUSEカーネルモジュールがあり、開発者は、ユーザーが特権なしでマウントできるアクセス可能な新しいタイプのファイルシステムを作成できます。モジュールのほとんどすべての一般的な言語のラッパーがあります。



テレソン


Telegramにアクセスするには、変更されたマルチスレッドファイルアップロード（TelegramClientX）を備えたTelethonライブラリバージョン0.19.1 を使用します。

Vfs


ファイルシステムを最初から書き込まないようにするために、Python dedupfsをベースにして、ファイルをブロックに分割し、sha1ハッシュを考慮してメタデータを保存し、同じブロックを再利用します。データをストレージにロードする方法を独自のものに置き換えるだけで済みます。

Tgcloud


これらすべてを組み合わせて、tgcloudのプロトタイプを作成しました。これまでのところ、スペースの速度はありませんが、まだ機能しているので、コミュニティを開発に引き付けたいと考えています。ダウンロードされたファイルは10 MBのブロックに分割され、sha1が計算されてSavedMessagesにロードされます。ハッシュはローカルデータベースに保存され、ブロックを読み取るときにファイルがそこに配置されます。ブロックは希望する順序で接着され、ディスクから読み取るかのようにバイトストリームが発行されます。

コードを実行するには


  • my.telegram.orgでapi_hashとapi_idを取得し、このデータをアカウントの電話番号と一緒にtg_access.pyに挿入する必要があります
  • プロジェクトフォルダーから実行してセッションを作成する 

    python3.6 download_service.py

    確認コードの入力
  • インストール 

    sudo yum install python-fuse
  • マウントフォルダーの作成 

    mkdir storage
  • VFSを起動：
    デバッグあり

    python dedupfs/dedupfs.py -df --block-size 10240000 storage/

    バックグラウンドで 

    python dedupfs/dedupfs.py --block-size 10240000 storage/
  • たとえば、このフォルダをファイルブラウザのソースとして指定できます
