2. TypeScriptの学習を開始する


Vue 3.0で予想される変更について最も話題になったものの1つ。 -これはTypeScriptのサポートです。 VueのコードベースはTypeScriptで書き直されています。これにより、開発者はIDEで型ヒントとコード補完ツールを使用できるようになります。

これは、誰もが絶対にTypeScriptに切り替える必要があるという意味ではないことに注意することが重要です。 Vueプロジェクトは引き続きJavaScriptで作成できます。ただし、TypeScriptを使用すると、コードの速度と品質が向上します。

TypeScript（TS）はJavaScriptのスーパーセットです。これは、コードがJavaScript（JS）でコンパイルされた言語です。 TSとJSの違いは、TypeScriptでは変数の型を指定できることです。型に関連付けられたコードのエラーは、コンパイル段階で検出できます。ここにいくつかのコードスニペットがありますTypeScript ドキュメント。1つはTypeScriptで記述され、もう1つはJavaScriptで記述されています。

ここにJSコードがあります：

function greeter(person) {
    return "Hello, " + person;
}

let user = "Jane User";

document.body.textContent = greeter(user);

そしてここにTSコードがあります：

function greeter(person: string) {
    return "Hello, " + person;
}

let user = "Jane User";

document.body.textContent = greeter(user);

TSコードにはエンティティタイプの表示があることに注意してください。greeter文字列ではなく、たとえば配列などのTS関数を渡すと、TS コードをコンパイルしようとするとエラーメッセージが表示されます。
