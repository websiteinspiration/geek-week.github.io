方法：マネまたはモネ？ニューラルネットワークが応答する

アーティストEduard ManetとClaude Monetは生涯混乱していました（Arzamasに関する非常に興味深い記事です）。彼らは両方とも印象派の創設者であり、同様の方法で書いたので、それは驚くべきことではありません。コースラでの畳み込みニューラルネットワークのコースを聞いて、どのアーティストが絵を描いたかを決定するモデルを作成することにしました。

仕事を始める前に、マネが「スキャンダラスな」「オリンピア」と「草の上での朝食」を書いたことだけを知りました。データを収集する過程で、次のことが明らかになりました。

  • 書体は本当によく似ています
  • マネはより多くの肖像画を描き、モネは風景を描きました
  • モネには一連の類似した絵画がすべてあります（たとえば、同じ風景、1日の異なる時間に描かれたもの）
  • モネはマネよりも多くの絵を描いた。

完全なデータセットは、両方の画像処理タスクで小さいことがわかりました。あるクラス（Manet）で100を少し超える画像と、別のクラス（Monet、彼のすべての絵ではありません）で400を超える画像です。したがって、このような問題で通常見られるような高い精度の分類は期待できませんでした。しかし、どのレベルに到達できるかは興味深いものでした。

追加のデータ処理なしのシンプルなモデル


まず、単純なCNNモデルを使用し、データを処理しません（再販を除く）。

model_simple_cnn_wo_augm = tf.keras.models.Sequential([
    tf.keras.layers.Conv2D(16, (3,3), activation='relu', input_shape=(input_size, input_size, 3)),
    tf.keras.layers.MaxPooling2D(2, 2),
    tf.keras.layers.Conv2D(32, (3,3), activation='relu'),
    tf.keras.layers.MaxPooling2D(2,2),
    tf.keras.layers.Conv2D(64, (3,3), activation='relu'),
    tf.keras.layers.MaxPooling2D(2,2),
    tf.keras.layers.Flatten(),
    tf.keras.layers.Dense(512, activation='relu'),
    tf.keras.layers.Dense(1, activation='sigmoid')
])

train_datagen_wo_augm = ImageDataGenerator(rescale=1.0/255.)

トレーニングの結果によると、大幅な過剰適合が顕著です



。検証AUCの最良の結果は0.883ですが、トレーニングサンプルのAUCは0.997です。

過剰適合に対処するために、データの前処理を追加してみましょう。

単純なデータ前処理モデル


前処理には、ケラスのImageDataGenerator関数を使用します。回転、垂直方向と水平方向のシフト、画像の「ストレッチ」とスケーリング、輝度と色の変更、水平方向の表示を追加します。

train_datagen_with_augm = ImageDataGenerator(rescale=1.0/255.,
                                  rotation_range=40,
      width_shift_range=0.2,
      height_shift_range=0.2,
      shear_range=0.2,
      zoom_range=0.2,
      brightness_range = [0.5, 1.5],
      channel_shift_range = 100,
      horizontal_flip=True,
      fill_mode='nearest')

モデルとすべてのトレーニングパラメーターは同じままにします。トレーニングの結果によると、トレーニングと検証のサンプルのメトリック値（AUC）が非常に近いこと、つまり、前処理により過剰適合が排除されたことは明らかです。



その結果、テストサンプルのAUCは0.919であり、検証では0.909です。

データの前処理と重み付けを備えた単純なモデル


前処理を追加することで、過剰適合がなくなりました。そして、AUCは悪くありませんでしたが、データの不均衡が、モデルにさらに重みを付けようとする動機になりました。私はケラスのマニュアルからウェイトを計算するアプローチを使用しました：

weight_for_manet = (1 / manet_test_size)*(manet_test_size + monet_test_size)/2.0 
weight_for_monet = (1 / monet_test_size)*(manet_test_size + monet_test_size)/2.0

class_weight = {0: weight_for_manet, 1: weight_for_monet}

このような計算の結果、クラス0（Manet）2.50、クラス1（Monet）0.62

の重み。次に、class_weight変数がmodel.fit（）に追加されます。

ただし、トレーニング結果は、重みなしのバージョンと大幅に異なりません。トレーニングセットのAUCは、0.892、検証-0.915。

転移学習：Inception V3モデルの使用


私たちのデータセットは機械学習タスクのために小さく、さらに深い学習です。このような場合、転移学習アプローチが役立つことがあります。

このアイデアは、大規模なデータセットでトレーニングされた、すぐに使用できるモデルを採用することであり、したがって、データのより多くのパターンを学習する機会がありました。その後、ネットワークの最後のレイヤーがタスクに適したレイヤーに置き換えられ、モデルがデータセットで学習します。

私はInception V3モデル（wikiと"Habr"に関する記事）を使用しました。結果は良好でした。AUCはトレーニングサンプル0.971、検証は0.970でした。

小さなデータセットと異なるクラスのオブジェクトの類似性を考えると、最初はAUCが約0.8になると予想していました。ただし、データの前処理を備えた単純なCNNモデルを使用しても、0.9に近い結果が得られ、InceptionV3を使用した転移学習を使用して0.97に改善することができました。
