そのため、インストール後、Zimbra Open Source Editionが搭載されたサーバーが当社に登場しました。 Zimbraのすべての利点を仕事で直接使用する人、つまり企業の従業員用のアカウントを作成する人にアクセスを与えるときです。 Zimbra Collaboration Suite OSEはActive Directoryと統合できます。この統合を実現する方法については、Habr.ruのZextrasブログをご覧ください。ただし、ZimbraおよびZextras Suiteをテストするために、新しいユーザーの作成を試みます。

コンソールでユーザーを作成するには、コマンドzmprov ca marketing@company.ru qwertyを入力します。実行後、marketing @ company.ruメールボックスとqwertyパスワードを持つユーザーがシステムに表示されます。また、会社にADを備えたサーバーがある場合は、そこから直接Zimbraにアカウントを追加できます。

Zimbraの信頼性の確保


コラボレーションシステムは常に企業の鍵の1つであるため、データの安全性を確保し、ダウンタイムを最小限に抑えることが非常に重要です。これを行うには、Zextras Backup拡張機能を使用してバックアップを構成します。管理コンソールで拡張機能を有効にする必要があります。左側のペインで、Zextras Suiteアイテムを見つけて展開し、ZxBackupに切り替えます。次に、Zextrasバックアップ設定の「基本バックアップ構成」セクションで、「今すぐ初期化」ボタンをクリックし、SmartScanをオンにする必要があります。これにより、Zimbraメールストレージで発生するすべての変更が段階的に保存されます。



すべてのバックアップ情報は/ opt / zimbra / backupフォルダーに蓄積されますが、実行中のシステムと同じサーバーにバックアップコピーを保存することは合理的ではないため、/ opt / zimbra / backupフォルダーに外部ドライブまたはネットワークドライブをマウントして、バックアップメディアがメインサーバーから地理的に離れています。これらのバックアップ方法の組み合わせも可能です。ネットワークドライブを/ opt / zimbra / backupに接続したが、Zimbraデータの完全なコピーを備えた外部ドライブが必要だとします。これを実行するには、外部バックアップオプションが役立ちます。そのおかげで、サーバーに接続された外付けドライブにデータの完全なコピーを作成できます。



ただし、Zimbraユーザーが生成するデータの量が増えると、ユーザーが使用するデータの量も増え、その結果、完全バックアップの作成にかかる時間が増加します。これを削減するために、Zextras PowerStore拡張機能を使用できます。これにより、圧縮と重複排除を使用できます。これにより、メールストレージスペースが占めるスペースを最大50％削減でき、LVMを使用せずに追加のストレージをサーバーに接続できます。

他のZextras拡張機能も非常に役立ちます。たとえば、モバイルでは、モバイルデバイスでZimbraおよびZextras Suiteのすべての機能を使用できます。管理者を使用すると、多くの部門を持つ大企業でソリューションを使用でき、それらを管理者として指定し、ユーザーグループを管理するグローバル管理者の権限を部分的に委任できます。企業内のコミュニケーションがインスタントメッセンジャーに転送されることが増えていますが、機密情報の保存と送信という観点からは安全ではありません。チーム拡張機能を使用すると、インスタントメッセンジャーのWhatsUpp、Telegramなどを使用するときに多くの人が持っているように、機密データがサイドに転送されるのを回避しながら、企業でビデオ通話サポートを備えた本格的な企業チャットを実装できます。
