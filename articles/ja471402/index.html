ユーザーのインターネットは高速である必要があり、ブラウザはこのページが利用できないと結論づけて30分待たないでください。さまざまなロックオプションを長時間検索した後、次のモデルに到達しました：
File 1-> / script / denied_host、禁止された名前のリスト：

test.test
blablabla.bubu
torrent
porno

ファイル2-> / script / denied_range、禁止されたアドレススペースとアドレスのリスト：

192.168.111.0/24
241.242.0.0/16

スクリプトファイル3-> ipt.sh。ipablesで動作します。

#      
HOSTS=`cat /script/denied_host | grep -v '^#'`
RANGE=`cat /script/denied_range | grep -v '^#'`
echo "Stopping firewall and allowing everyone..."
#    iptables,     
sudo iptables -F
sudo iptables -X
sudo iptables -t nat -F
sudo iptables -t nat -X
sudo iptables -t mangle -F
sudo iptables -t mangle -X
sudo iptables -P INPUT ACCEPT
sudo iptables -P FORWARD ACCEPT
sudo iptables -P OUTPUT ACCEPT
#     (  )
sudo sh rout.sh
#         
for i in $HOSTS; do
sudo iptables -I FORWARD -m string --string $i --algo bm --from 1 --to 600 -p tcp -j REJECT --reject-with tcp-reset;
sudo iptables -I FORWARD -m string --string $i --algo bm --from 1 --to 600 -p udp -j DROP;
done
#         
for i in $RANGE; do
sudo iptables -I FORWARD -p UDP -d $i -j DROP;
sudo iptables -I FORWARD -p TCP  -d $i -j REJECT --reject-with tcp-reset;
done

sudoの使用は、WEBインターフェイスを介して管理するための小さなハックがあるという事実によるものですが、そのようなモデルを1年以上使用した経験が示しているように、WEBはそれほど必要ではありません。実装後、データベースなどにサイトの一覧を作りたいという要望がありました。ブロックされたホストの数は、250 + 12を超えるアドレス空間です。実際、システム管理者のように、https接続を介してサイトに切り替えると問題が発生します。ブラウザについて不満があります:)しかし、これらは特別なケースであり、リソースへのアクセスの欠如に対するほとんどの応答はまだ私たちの側にあり、Opera VPN、プラグインも正常にブロックしていますMicrosoftのfriGateやテレメトリのようなものです。
