ビデオ@Databases Meetup：DBMSセキュリティ、IoTのTarantool、ビッグデータ分析のためのGreenplum



2月28日は、Mail.ru Cloud Solutionsによって組織された@Databases mitap でした。Mail.ru Groupには300人以上の参加者が集まり、最新の本番データベースの現在の問題について話し合いました。 カットされたビデオパフォーマンス：「Gazinformservice」がパフォーマンスを失うことなく安全なDBMSを準備する方法。Arenadataは、分析タスクのための強力な超並列DBMSであるGreenplumの中心にあるものについて説明しています。およびMail.ruクラウドソリューション-モノのインターネットのプラットフォームを構築した方法とその内容（ネタバレ：Tarantoolなしでは実現できませんでした）。




セキュリティとDBMS。Gazinformservice、ソフトウェア開発責任者、Denis Rozhkov氏



ユーザーデータをデータベースに保存するすべての人にとって、安全性とパフォーマンスは2つの痛点です。Denis Rozhkovは、パフォーマンスを維持しながら、ダークネットで自分のベースが見えないように保護手段を選択する方法を共有し、Gazinformservice Jatobaの開発例を使用してDBMSのセキュリティニュアンスについて話しました。

最新のIIoTプラットフォームのデータベース。Andrey Sergeev、Mail.ru Cloud Solutions、IoTソリューション開発グループ責任者



ご存知のように、ユニバーサルデータベースは存在しません。特に、センサーから毎秒数百万のイベントをほぼリアルタイムモードで処理できるモノのインターネットプラットフォームに必要な場合。Andrey Sergeyevは、Mail.ru Cloud SolutionsでIIoTプラットフォームをどのように構築したか、彼らがたどった道のり、Tarantoolがそれなしでは実現できない理由を語りました。

Greenplum：2台から数百台のサーバー。私たちは、ACID、ANSI SQL、そして完全にオープンソースで最新の分析を構築しています。Arenadata、最高製品責任者、Dmitry Pavlov




2009年以来、Dmitryは大規模なクラスターシステムと緊密に連携しており、他の誰も同様に、データ量が指数関数的に増加する状況では、従来のDBMSの分析問題を解決することが不可能になることを知っています。大規模分析システム向けの一般的なソリューションである大規模並列オープンソースDBMS Greenplumについて詳しく説明します。
