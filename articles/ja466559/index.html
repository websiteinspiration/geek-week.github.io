Letは新しいVarです



プログラミング言語を習得すると、変数の宣言など、いくつかの基本的な概念が現れます。それらは非常に単純なので、機能の知識がなくても使用できます。

私はあなたの考えを知っています。「なぜこれについて私たちに手紙を書くのに時間を無駄にしているのですか？」

これらの要素を理解するために数分を費やす価値があると思うので、それらの意味は何であり、どのようにそれらを使用する必要があります。これは、いくつかの一般的な間違いを回避し、コードをよりよく書くのに役立ちます。

この記事では、JavaScriptにデータを保存する方法に焦点を当てます。

Var


「むかしむかし、遠い銀河系で……」JavaScriptで変数を定義する唯一のオプションは、varキーワードを使用することでしたが、それでコード内のデータを操作できました。しかし、ES6標準の導入により、古いJSの奇妙な点と弱点のいくつかはかなり明白になりました。

  1. スコープ：varキーワードを使用すると、グローバルスコープとローカルスコープ（または関数のスコープ）でのみ変数を定義できます。ネストされたコードブロックがいくつあっても、これらの可能性は2つしかありません。
  2. 定数：コードの実行中に変更されないものを定義する場合は、開発者の常識にのみ依存できます。意味を明確にするためにいくつかの便利な規則（たとえば、大文字）を使用できますが、意味が変化しないことを保証するものはありません。
  3. : , , ( ), , .


技術的なことから始める前に、いくつかの一般的な定義を紹介しましょう：

変数は、再利用可能なデータが保存される「非常に簡単な」「コンテナー」です。

識別子は変数名です（これも非常に簡単です）。

ブロックは、中括弧（ccskrf）のペアで区切られたコードの一部です。たとえば、if、for、function、andなどです。 etc.

Scopeは、コード内の変数の可視性を決定します。疑問がある場合は、「コード内の変数はどこに表示されますか？」と自問してください。
注意。スコープと実行コンテキストを混同しないでください。

**コンテキスト（または実行コンテキスト）**は、JavaScriptコードが実行される環境です。簡単にするために、コンテキストはコードが属するオブジェクトであり、これはそれを参照するキーワードであると言えます。ですから、「これはどのオブジェクトを指しているのですか？」と自問してください。

ここで、開発者がスターウォーズに非常に情熱的である（これは良い）が、ES6（悪い）のような新しい標準に疑いがあると仮定します。したがって、彼は古いES5スタイルでコードを書くことを好み、それは次のようになります。

console.log("I am a %s", jedi);

var jedi = "Ani";

//   jedi   
// =>  undefined

function useTheForce(comeToTheDarkSide) {
  var jedi = "Obi-Wan Kenobi";
  var jedi = "Anakin Skywalker";

  if (comeToTheDarkSide) {
    var jedi = "Darth Vader";
    console.log("I am %s", jedi);
    // => I am Darth Vader
  }

  console.log("I am %s", jedi);
  //  Darth Vader
}

useTheForce(true);

console.log("I am %s", jedi);
// => Ani

ご覧のとおり、コードのブロックは3つあり（グローバルを含む）、スコープは2つだけです。これは、かっこ内のコードがスコープを作成しないためです。コンソールは「I am Darth Vader」を2回発行し、次に「I am Ani」をグローバルに発行します。

また、同じ変数が関数内の行で2回宣言され、その後ifステートメントで再度宣言されていることにも注意してください。つまり、同じスコープで同じ変数を3回宣言すると、例外がスローされます。

最後に重要なことですが、最初のログの出力は、変数を定義する前にその値を出力します。これはvarで完全に合法であり、「巻き上げ」と呼ばれます。

「上昇」は、宣言された変数と関数が物理的にコードの先頭に移動されていることを前提としています。技術的には、コンパイル段階でメモリに格納される変数と関数の宣言がありますが、それらは記述したとおりのコードに残ります。リフトの基本的な重要性は、コードで関数を宣言する前に関数を使用できることです。

あなたはそれについてここで読むことができます。

この例では、変数「jedi」の宣言がメモリに配置され、デフォルト値（未定義）に初期化されます。

しかし、彼はすぐにvarキーワードを変更して変更できないことに気づきました。これを行うには、いくつかの修正を行う必要があります。

「上昇」は同じようには機能しません。変数は一時デッドゾーンと呼ばれる状態になり、*まで初期化されません。定義が評価されるまで。この例では、参照エラーというエラーをスローします。

再割り当ては禁止されています。スコープ内の変数の定義は一意である必要があります。エラーを取得しようとすると：SyntaxError。

ifステートメントは有効なブロックスコープであるため、その中のjedi宣言は一意です。

定数


letに関するすべてを理解したので、constを簡単に紹介できます。実際、constはletに似ていると言えますが、別の値を再割り当てすることはできません。また、割り当てはconst宣言中にのみ許可されることも知っておく必要があります。

function useTheForce(comeToTheDarkSide) {

  const jedi = "Yoda";

  if (true) {
    jedi = "Darth Yoda";
    console.log("I am %s", jedi);

    // => TypeError: Assignement to constant variable.
  }

}

useTheForce(true);

この例では、「jedi」が値「Yoda」の定数であると仮定します。ifステートメント内の値を変更しようとすると、TypeErrorエラーが発生します。Yodaがダークサイドに参加することはないため、これは理解できます。

結論


言語でコンセプトを言葉で表現することが不可能な場合、そのギャップを埋めるために新しいコンセプトが導入されます。これは、英語（リンク）、イタリア語、エスペラント語、エボコック（おそらく）などに当てはまります。これは、JavaScriptなどのプログラミング言語にも当てはまります。

これで、次のことができるようになります。

  • ブロックのスコープ内で変数を宣言し、ブロックを評価するときにのみそれらにメモリを割り当てます
  • 変数宣言をスコープ内で一意に保ち、開発者がコードの別の部分で使用されている変数を（異なる値で）誤ってオーバーライドしないようにします。
  • constに格納されている変数がいつでも変更されないようにすることができます。

最後のヒントは、デフォルトの選択としてconstを使用することです。変数を再割り当てする必要がある場合は、（ループのように）** let **を使用します。そして、varを使用するのは...いいえ、varはもう必要ありません
