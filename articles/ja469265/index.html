RESTを介したマルチプロセッシングIntel Neural Computer Stickアクセス

シングルタスク問題


前のシリーズでは、Intel Neural Computer Stick 2を戦車に搭載し、すべてのニューラルネットワーク計算をそれに投入して、TensorflowとOpenCV-DNNを放棄しました。

そのとき私がすでに遭遇した問題がありました-複数のプロセスから同時にNCSを操作できないことです。それからそれは重要ではなかったが、今それを理解する時が来た。

2番目のプロセスからモデルを読み込もうとすると、OpenVinoは誓い始めました：

E: [ncAPI] [    926029] resetAll:348     Failed to connect to stalled device, rc: X_LINK_ERROR
E: [ncAPI] [    933282] ncDeviceOpen:672        Failed to find suitable device, rc: X_LINK_DEVICE_NOT_FOUND

インテルサポートフォーラムを検索したところ、同様の問題が見つかりました。

そこから、明確に述べられているドキュメントに転送されました。

単一のデバイスを複数のプロセスで共有することはできません。

この実験では、マルチプロセスアクセスを最小化して開始することができます。

NCSサービス


NCSとの直接の作業を別のサービスに配置し、それらが機能するすべてのクライアントにAPIを配布することは完全に論理的です。

一般的に、これはロボットとニューラルネットワークの観点から見た新しい成果についてのトピックであると考えられていました。しかし、NCS APIの資料は別の記事を作成するために非常に役立っていることがわかりました。

NCS API


低レベルでは、NCS APIは非常に単純です。-
モデルをロードする
-計算を開始する
-モデルのリストを取得する-
モデルプロパティを取得するモデルを

ロードするときにすべてが明確である場合、計算の排気は、クライアントが必要とする状況依存のテンソルであり、すべてではありません。

モデルのリストを取得することも非常に透過的であり、入力テンソルの次元がすぐに頭に浮かびます。つまり、人間の言葉で言えば、ネットワーク設定に先立って画像を調整しておくとよいでしょう。

また、低レベルは適切ですが、特殊な操作をサポートしている場合は、ロジックとデータが簡略化されます。

したがって、ベースに加えて、分類、検出、セグメンテーションのためのAPIをサポートするタスクがあります。

残念ながら、最も興味深いセグメンテーションモデルはNCSでサポートされていないため、道路とマークアップを使用して、最も単純なものに制限する必要があります。

これらの演算はいずれもモデルの基本的な計算を使用しますが、出力テンソルの解釈が異なります。

メインインターフェース


したがって、メインインターフェイスにはメソッドが含まれます。

  • POST：/ load-モデルをロードします
  • POST：/ unload / $ model-モデルを削除します（サービスから、デバイスから削除することは不可能です）
  • GET：/ list-モデルのリストを取得します
  • GET：/ input / shape / $ model-入力テンソルの次元を見つけます
  • POST：/推論/ファイル/ $モデル-メモリからのデータで計算を行います
  • POST：/推論/パス/ $モデル-ファイルシステムのデータを使用して計算を行います

メモリとファイルシステムからのデータに関する2つの単語を次に示します

。NCSサービスとそのユーザーが同じRaspberryで実行されている場合、画像の転送を節約し、代わりにパスを転送して、サービスがファイル自体を読み取ることは理にかなっています。
画像が既にメモリにある（またはファイルシステムに存在しない）場合は、そこから直接転送します。

テストは、メモリからのバイト転送がはるかに遅いことを示しています（測定は1000回の試行に対して行われました）：

メモリから：87.5秒
ファイルパス：63.3150秒

それにもかかわらず、これらの2つのオプションは、一般的な計算と特別な計算の両方で、すべての方法でサポートされています以下のケース。

一般に、推論メソッドは、numpy配列の形式の入力画像を受け取り、同じ形式のテンソルを生成します。
排気をどのように解釈するかはすでに顧客の問題です。
このタスクを容易にするために、サービスは出力テンソルから人間の形で重要な情報を抽出する特殊なメソッドをサポートしています。

分類


分類のために、出力テンソルをペアのセット（クラス、スコア）に変換する別のRESTメソッドを開始します。

def get_class_tensor(data):
    ret = []
    thr = 0.01
    while(True):
        cls = np.argmax(data)
        if data[cls] < thr:
            break;
        logging.debug(("Class", cls, "score", data[cls]))
        c = {"class" : int(cls), "score" : int(100 * data[cls])}
        data[cls] = 0
        ret.append(c)
    return ret

def classify(model_id, img):
    rc, out = run_inference(model_id, img)
    if not rc:
        return rc, out
    return True, get_class_tensor(out)

通常の出力の場合と同様に、メモリ内のファイルとディスク上のパスを介した2つの方法がサポートされています。

  • POST：/分類/ファイル/ $モデル
  • POST：/分類/パス/ $モデル

検出


検出器の出力テンソルにはセット（クラス、確率、正規化された座標）が含まれており、かなり扱いにくいように見えます。

ありそうもないオプションを切り捨てながら、それを理解可能な形式に変えます：

def get_detect_from_tensor(t, rows, cols):
    score = int(100 * t[2])
    cls = int(t[1])
    left = int(t[3] * cols)
    top = int(t[4] * rows)
    right = int(t[5] * cols)
    bottom = int(t[6] * rows)
    return {"class" : cls, "score" : score, "x" : left, "y" : top, "w" : (right - left), "h" : (bottom - top)}


def build_detection(data, thr, rows, cols):
    T = {}
    for t in data:
        score = t[2]
        if score > thr:
            cls = int(t[1])
            if cls not in T:
                T[cls] = get_detect_from_tensor(t, rows, cols)
            else:
                a = T[cls]
                if a["score"] < score:
                    T[cls] = get_detect_from_tensor(t, rows, cols)
    return list(T.values())


def detect(model_id, img):
    rc, out = run_inference(model_id, img)
    if not rc:
        return rc, out

    rows, cols = img.shape[:2]
    return True, build_detection(out[0], 0.01, rows, cols)

通常どおり、両方の方法がサポートされています。

  • POST：/検出/ファイル/ $モデル
  • POST：/検出/パス/ $モデル

セグメンテーション


セグメンテーションテンソルには、クラスごと、さらにはニューラルネットワークの次元での確率が含まれます。
これを単にクラスマスクに変換します。

def segment(model_id, img):
    rc, out = run_inference(model_id, img)
    if not rc:
        return rc, out

    out = np.argmax(out, axis=0)
    out = cv.resize(out, (img.shape[1], img.shape[0]),interpolation=cv.INTER_NEAREST)
    return True, out

  • POST：/セグメント/ファイル/ $モデル
  • POST：/セグメント/パス/ $モデル
