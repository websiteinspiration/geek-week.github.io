あなたは世界のチェスチャンピオン、バックギャモンを倒すつもりですか？

これを可能にする方法があります-強化トレーニング。

強化学習とは？


強化学習とは、環境内で一貫した意思決定を行うことを学習し、アクションごとに与えられる最大の報酬を受け取ります。

彼には教師はいません。環境からの報酬のしるしだけです。時間は重要であり、アクションは後続のデータに影響します。このような条件は、教師の有無にかかわらず学習を困難にします。


以下の例では、マウスはできるだけ多くの食べ物を見つけようとし、可能な場合は感電を回避します。


ネズミは勇敢で、チーズがたくさんある場所にたどり着くことができます。ただ静止して何も受け取らないよりはましです。

マウスは、特定の状況ごとに最良の決定を下したくありません。これは彼女からの多大な精神的支出を必要とし、それは普遍的ではありません。

強化学習は、電気ショックを回避し、できるだけ多くの食物を得る方法をマウスが学習できるようにするいくつかの魔法の方法を提供します。

マウスはエージェントです。壁、チーズ、スタンガンのある迷路が環境です。マウスは左、右、上、下に移動できます。これらはアクションです。


ネズミは感電ではなくチーズを欲しがっています。チーズは報酬です。マウスは環境を検査できます。これらは観察結果です。

氷強化訓練


マウスを迷路の中に置いて、氷の上に移動しましょう。「冬が来た。湖の真ん中に突然フリスビーを投げたとき、あなたとあなたの友達は公園でフリスビーを投げていました。基本的には湖の水は凍っていましたが、氷が溶けた穴がいくつかありました。（ソース）


「穴の1つを踏むと、氷水に落ちます。さらに、世界にはフリスビーが非常に不足しているため、湖を一周してドライブを見つけることは絶対に不可欠です。」（ソース）

同じような状況でどう思いますか？


これは強化学習の課題です。エージェントは、グリッドワールドでのキャラクターの動きを制御します。一部のグリッドタイルは通過可能ですが、他のグリッドタイルはキャラクターを水中に落とします。エージェントは、目標への適切なパスを見つけると報酬を受け取ります。学習アルゴリズムを開発して強化と比較するためのツールキットであるOpenAI Gym

を使用して、このような環境をシミュレートできます。この例のように、Frozen Lakeと呼ばれる標準化された環境セットへのアクセスを提供します。これは、数行のコードで作成できるテキスト環境です。

import gym
from gym.envs.registration import register

# load 4x4 environment
if 'FrozenLakeNotSlippery-v0' in gym.envs.registry.env_specs:
    del gym.envs.registry.env_specs['FrozenLakeNotSlippery-v0']
register(id='FrozenLakeNotSlippery-v0',
    entry_point='gym.envs.toy_text:FrozenLakeEnv',
    kwargs={'map_name' : '4x4', 'is_slippery': False},
    max_episode_steps=100,
    reward_threshold=0.8196
)

# load 16x16 environment
if 'FrozenLake8x8NotSlippery-v0' in gym.envs.registry.env_specs:
    del gym.envs.registry.env_specs['FrozenLake8x8NotSlippery-v0']
register(
    id='FrozenLake8x8NotSlippery-v0',
    entry_point='gym.envs.toy_text:FrozenLakeEnv',
    kwargs={'map_name' : '8x8', 'is_slippery': False},
    max_episode_steps=100,
    reward_threshold=0.8196
)

マルコフの意思決定プロセス


この例では、エージェントがグリッドワールドでのキャラクターの動きを制御します。この環境は、完全に観察可能な環境と呼ばれます。

将来のタイルは、現在のタイルを考慮して過去のタイルに依存しないため
（ランダムな状態のシーケンス、つまりMarkovプロパティを処理しているため）、いわゆるMarkovプロセスを処理します。

現在の状態は、次の動きを決定するために必要なすべてのものをカプセル化しています。覚えておく必要はありません。

次の各セル（つまり、シチュエーション）で、エージェントは次のセルにつながるアクション、つまりシチュエーションをある程度の確率で選択し、環境はエージェントに観察と報酬で応答します。

報酬関数と割引係数をマルコフプロセスに追加し、いわゆるマルコフ報酬プロセスを取得します。アクションのセットを追加すると、マルコフ意思決定プロセス（MDP）が取得されます。MDPのコンポーネントについては、以下で詳しく説明します。

状態


状態は、環境の一部であり、環境内の特定の時点でエージェントが観察するものの数値表現、湖のグリッドの状態です。Sは開始点、Gはターゲット、Fはエージェントが立つことができる固い氷、Hはエージェントが踏んだときに落下する穴です。4 x 4グリッド環境では16の状態、8 x 8環境では64の状態があり、以下ではOpenAI Gymを使用した4 x 4環境の例を示します。

def view_states_frozen_lake(env = gym.make('FrozenLakeNotSlippery-v0')):
    print(env.observation_space)
    print()
    env.env.s=random.randint(0,env.observation_space.n-1)
    env.render()
    
view_states_frozen_lake()

    print(env.action_space)
    print("Possible actions: [0..%a]" % (env.action_space.n-1))

状態遷移モデル


状態遷移モデルは、エージェントが現在の状態に基づいてアクションを実行したときに、環境の状態がどのように変化するかを記述します。

モデルは通常、遷移確率によって記述されます。これは、サイズN x Nの正方遷移行列として表されます。ここで、Nはモデルの状態数です。以下の図は、気象条件のそのようなマトリックスの例です。


凍った湖の環境では、湖は滑りにくいと仮定します。私たちが右に行けば、私たちは間違いなく右に行きます。したがって、すべての確率は等しくなります。

「左」は、エージェント1のセルを左に移動するか、エージェントが左の境界にある場合は同じ位置に残します。

「右」は、1セル右に移動するか、エージェントが右の境界にある場合は同じ位置に残します。

「上」は、エージェントを1セル上に移動します。エージェントが上限にある場合、エージェントは同じ場所に留まります。

「Down」は、エージェント1セルを下に移動します。または、下の境界にある場合は、同じ場所にとどまります。

報酬


各状態Fでは、エージェントは報酬を0受け取ります。状態Hでは、エージェントはこの状態に移行すると死亡するため、-1を受け取ります。エージェントがゴールに到達すると、+ 1の報酬を受け取ります。

遷移モデルと報酬モデルの両方のモデルが決定論的な関数であるため、これにより環境が決定論的になります。\

割引


割引は、将来の報酬の重要性を制御するオプションのパラメーターです。0から1の範囲で測定されます。このパラメーターの目的は、総報酬が無限大になるのを防ぐことです。

割引は、エージェントが将来受け取る可能性のある報酬よりも即時の報酬を好む場合のエージェントの動作もモデル化します。

値


フォーチュンの価値は、フォーチュンの割引を伴う予想長期収入です。

ポリシー（π）


エージェントが次のアクションを選択するために使用する戦略は、ポリシーと呼ばれます。利用可能なすべてのポリシーの中で、最適なものは、エピソード中に受け取るまたは予想される報酬の量を最大にするものです。

エピソード


エピソードは、エージェントが開始セルに表示されたときに始まり、エージェントが穴に落ちるか、ターゲットセルに到達したときに終了します。

それをすべて視覚化しましょう


マルコフの意思決定プロセスに関連するすべての概念を確認した後、OpenAI Gymを使用して、16x16環境でいくつかのランダムアクションをモデル化できます。毎回、エージェントはランダムなアクションを選択して実行します。システムは報酬を計算し、環境の新しい状態を表示します。

def simulate_frozen_lake(env = gym.make('FrozenLakeNotSlippery-v0'), nb_trials=10):
    rew_tot=0
    obs= env.reset()
    env.render()
    for _ in range(nb_trials+1):
        action = env.action_space.sample() # select a random action
        obs, rew, done, info = env.step(action) # perform the action
        rew_tot = rew_tot + rew # calculate the total reward
        env.render() # display the environment
    print("Reward: %r" % rew_tot)  # print the total reward
    
simulate_frozen_lake(env = gym.make('FrozenLake8x8NotSlippery-v0'))
