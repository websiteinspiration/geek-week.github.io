Transferindo arquivos usando pipes e outras pequenas coisas no Delphi

De alguma forma, eles me definiram a tarefa de escrever várias unidades no Delphi que fornecem acesso aos seguintes recursos do Windows:

  1. Memória compartilhada .
  2. Tubos .

Era impossível usar componentes de terceiros. A memória compartilhada deveria ser usada para classificar arquivos de texto grandes - várias dezenas ou até centenas de megabytes, nos quais o TStringList caiu do EOutOfMemory . Tubos - para transferir arquivos de um aplicativo para outro.

Reescrever o código de exemplos da documentação da Microsoft de C para Delphi não foi nem metade da batalha, mas para 10% .O principal era amarrar tudo isso, adicionar funções adicionais sempre que necessário e fazer com que tudo funcionasse, e não apenas existir como código-fonte. Como resultado, um monstro tão feio nasceu que não era algo para mostrar a ninguém, mas apenas olhando para ele era assustador. No entanto, talvez alguém encontre algo útil para si em suas entranhas (e se?), Então decidi publicá-la aqui. Para censura geral. O código fonte completo dessa abominação está disponível no GitHub . O texto do artigo fornecerá os nomes dos arquivos nos quais você pode ver uma ou outra parte da implementação.

A primeira tarefa é classificar o arquivo de texto - foi decidido assim:

  1. Abra o arquivo ( CreateFile ).
  2. Usamos CreateFileMapping e MapViewOfFile (common \ MemoryMappedFileUnit.pas).
  3. Determinamos a codificação do arquivo. Aqui, na verdade, me limitei a apenas duas codificações: UTF-16 e ACP - uma codificação de byte único com uma página de código padrão. No Windows de língua russa, o Windows-1251 é geralmente usado .
  4. Dividimos o texto em linhas. Ou seja, estamos procurando marcadores de fim de linha e, portanto, consideramos o texto de marcador em marcador como uma linha.
  5. Classifique a matriz resultante de seqüências de caracteres (SortingTest \ MergeSorterUnit.pas).
  6. Escrevemos a matriz classificada de seqüências de volta ao arquivo.
  7. LUCRO!

O cliente definitivamente queria fazer a classificação multithread, então o Merge Sort foi tomado como um algoritmo de classificação . (Graças ao guru que criou a implementação de trabalho e a postou no Stackoverflow .)

A segunda tarefa - transferir o arquivo de um aplicativo para outro - acabou sendo muito mais difícil. Obter a implementação de pipes no Delphi com base nos exemplos da Microsoft é uma coisa, mas fazer uma transferência de arquivos de trabalho é outra. Como um programa de servidor, a Microsoft oferece várias opções:

  1. Servidor de tubulação multithread .
  2. Servidor de tubulação usando E / S sobreposta nomeada .
  3. Servidor de tubos nomeado usando rotinas de conclusão .

Como resultou de duas semanas de tormento, a implementação do mecanismo de transferência de arquivos com base na primeira versão do servidor ( Multithreaded Pipe Server ) é muito complicada. É mais razoável usar a segunda opção ( servidor de pipes nomeados usando E / S sobreposta ), com isso a implementação é muito mais simples (embora o cliente realmente desejasse a primeira opção de servidor por algum motivo ). Mas ainda assim eu tinha que me atormentar para lembrar tudo isso. Como resultado, nasceu um transmissor de arquivos baseado em pipe : TCommunicationEndpointBase (CommunicationTest \ SimpleCommunicationUnit.pas).
