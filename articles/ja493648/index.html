NSG200 Nebula Security Gatewayルーターを分解します。内側と外側のビュー

自己分離モードの導入により、私は自宅に落ち着き、仕事の制限がZyxelから非常に興味深い鉄片が解除されるのを待っています。

所有者の許可を得て、この記事では本日、機能だけでなく、NSG200 Nebula Security Gatewayルーターについても分析します。



これは、ソフトウェア定義のSDNネットワークで動作するように完全に調整された機器の代表であり、アーキテクチャーレベルの制御はポータルnebula.zyxel.comを介して実行さ

れます。NSG200がどのように制御され、何がカバーされているかを詳しく見てみましょう。

コントロール


このルーターをコンピューターに接続し、ブラウザーに192.168.1.1と入力すると、Webインターフェイスが表示されなくなります。ユーザー名とパスワードadmin / 123456を入力すると、より信頼性の高いものに変更する必要があります。



パスワードを変更すると、メインインターフェースが表示されます。ほとんど何も表示されません。



実際には、NSG200が星雲コントロールノードに接続できるように、インターネット接続のみを構成できます。



これは、VPN、侵入検知、コンテキストフィルターなどを備えた本格的な強力なルーターのすべての機能が利用できる場所です。

星雲がなければNSGは役に立たないのですか？


Zyxel Nebulaは、長年にわたって実績のあるシステムです。多数の企業が、nebula.zyxel.comを通じて管理する機能を備えた機器を購入しています。

これにより、このデバイスのすべての機能にアクセスできます。Ansibleを使用してルーターの自動制御を構成できるため、CLIを使用すると、Nebulaを使用するよりもさらに多くを取得できるように思えました。

ただし、このデバイスは中央ノードからの制御のためにシャープにされています。したがって、この記事ではCLI機能を検討する意味がわかりません。

VPN


ローカルネットワークでオフィスを統合するには、少なくとも1つのルーターに白い静的IPアドレスがあれば十分です。サーバーであることを示します。また、サーバーのVPNルーターを別のルーターに変更するのも簡単です。たとえば、あるオフィスのチャネルに問題が発生した場合、サーバー機能を、インターネットが安定している別のオフィスのルーターに転送できます。



リモートオフィスにアクセスする権限を持つローカルネットワークを選択し、[保存]をクリックします。

別の都市/事務所に新しい支社を拡張することはそれほど簡単ではありません。これを行うには、地元の店舗のオフィスの責任者が星雲を使用したモデル（NSG50など）を購入し、プロバイダーのレースをWANポートに差し込み、ポピーの写真とルーターのシリアル番号を管理者に送信します。プロバイダーがDHCP経由で接続を発行しない場合（これは今日ではまれですが）、Webインターフェースにアクセスして、特定のプロバイダーを介して接続を構成する必要があります。それ以外はすべて中央局のシステム管理者が行います。
