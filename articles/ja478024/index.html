Pythonを使用したVMware vSphere API管理タスクの自動化


当社は仮想化プラットフォームVMware vSphereを積極的に利用しています。そのライブ製品テスト環境では、デモンストレーションスタンド、さまざまな顧客インフラストラクチャのエミュレーター、およびその他の同様に重要な「仮想マシン」が使用されます。インフラストラクチャには十分な容量がありますが、仮想マシンを管理するために多くの人がアクセスできるため、常に競合が発生し、ファームの生産性が低下します。部門（エンジニア、テスター、セールス、開発者）間でプールを分離しても問題は完全には解決されないため、誰が誰に迷惑をかけ、誰がすべてのリソースを食べたかを定期的に把握する必要があります。仮想マシンの数が100をはるかに超えるため、これを手動で行うのは問題があるため、APIの使用方法を学びました。 VMware vSphereには、非常に豊富なAPIがあります。アプリケーションはかなり広いですが。

この記事では、Pythonを使用した管理タスクの相互作用の例を示します。

次の記事 では、Ansibleを使用した管理について検討します。

Pythonの場合、VMware自体からのpyVmomiと呼ばれるライブラリがあります。

トレーニング


プロジェクトフォルダーで、次の内容のconfig.txt構成ファイルを作成します。

[VSphere]
host=10.10.1.2
user=admin@domain
pwd=password

それを読むには、configparserモジュールを使用してください。

config = configparser.ConfigParser()
config.read("config.txt")

球に接続するには、pyVim.connect.SmartConnectとpyVim.connect.SmartConnectNoSSL方法が使用されています。2つ目は自己署名証明書用です。

from pyVim.connect import SmartConnectNoSSL
connection = SmartConnectNoSSL(**config["VSphere"])

データストア上のローカルマシンのリスト


データストアのスペースが不足するとします。この問題を今解決します。目標は、データストア上の占有スペースの降順でソートされた仮想マシンのリストを取得することです。また、仮想マシンのディスクが拡張できる最大のファイルサイズと、ゲストシステムで使用されるスペースの量も出力します。

次のようにして、仮想マシンのリストを取得できます。

from pyVmomi import vim

def get_all_vms():
    content = connection.content
    container = content.viewManager.CreateContainerView(content.rootFolder, [vim.VirtualMachine], True)
    return [VirtualMachine(managed_object_ref) for managed_object_ref in container.view]

ご覧のとおり、VirtualMachineクラスのオブジェクトのリストが返されます。今それを実装しています：

class VirtualMachine:
    def __init__(self, machine_object):
        self.vm = machine_object

    @property
    def name(self):
        return self.vm.name

    @property
    def provisioned_space(self):
        return int(self.vm.summary.storage.committed + self.vm.summary.storage.uncommitted) / 1024**3

    @property
    def guest_disk_usage(self):
        return sum([int(i.capacity - i.freeSpace) for i in self.  vm.summary.vm.guest.disk]) / 1024**3

    @property
    def usage_storage(self):
        return int(self.vm.summary.storage.committed) / 1024**3

    def __repr__(self):
        return "{}  {:.2f}  {:.2f}  {:.2f}".format(self.name, self.provisioned_space,
                                                   self.guest_disk_usage, self.usage_storage)

データ出力はすでに必要性に依存しています。最も簡単なのは、コンソールに出力することです。

for virtual_machine in sorted(get_all_vms(), key=lambda vm: vm.usage_storage, reverse=True):
    print(virtual_machine)

スナップショットのサイズと数


仮想マシンのサイズを調整しました。各仮想マシンにあるスナップショットの数と、スナップショットが占めるスペースを調べたいとします。

前の例のロジックは保持されますが、VirtualMachineクラスは次のようになります。

class VirtualMachine:
    def __init__(self, machine_object):
        self.vm = machine_object
        self.snapshot_count, self.snapshot_size = self._snapshot_info()

    @property
    def name(self):
        return self.vm.name

    def _snapshot_info(self):
        disk_list = self.vm.layoutEx.file
        size = 0
        count = 0
        for disk in disk_list:
            if disk.type == 'snapshotData':
                size += disk.size
                count += 1
            ss_disk = re.search('0000\d\d', disk.name)
            if ss_disk:
                size += disk.size
        return count, size / 1024**3

    def __repr__(self):
        return "{}  {:.2f}  {}".format(self.name, self.snapshot_size, self.snapshot_count)

そして、出力はサイズまたは量によってすでにソートされます：

for virtual_machine in sorted(get_all_vms(), key=lambda vm: vm.snapshot_size, reverse=True):
    print(virtual_machine)

スナップショットリカバリ


ここではスナップショットについて説明しているので、スナップショットについて説明します。

架空のタスクは、最後に作成されたスナップショットを復元することです。その名前には、インフラストラクチャがあります。

スナップショットを復元するには、RevertToSnapshot_Task（）タスクを作成する必要があります。

まず、必要な車のリストを取得します。

def get_vms(part_of_name):
    content = connection.content
    container = content.viewManager.CreateContainerView(content.rootFolder, [vim.VirtualMachine], True)
    return [managed_object_ref for managed_object_ref in container.view if part_of_name in managed_object_ref.name]

最後の要素を返す、作成された最後のスナップショットを取得するにはvm.snapshot.rootSnapshotListのリストを、順にソートCREATETIMEのフィールド：

def get_last_snapshot(vm):
    try:
        return sorted(vm.snapshot.rootSnapshotList, key=lambda snap: snap.createTime)[-1]
    except AttributeError:
        return None

これで、このスナップショットを復元できます。

def restore_snapshot(snaphot_obj):
    task = snaphot_obj.snapshot.RevertToSnapshot_Task()
    WaitForTask(task, connection)

今すべてを組み合わせる：

part_of_name = "infra"
for vm in get_vms(part_of_name):
    last_snapshot = get_last_snapshot(vm)
    if last_snapshot:
        restore_snapshot(last_snapshot)

リストのオン/オフ


仮想マシンをシャットダウンするには、PowerOffVM_Task（）タスクを作成する必要があります  。

マシンがオンになっている場合は、オフにする関数を記述します。

def poweroff_if_enabled(vm):
    if vm.runtime.powerState == vim.VirtualMachinePowerState.poweredOn:
        vm.PowerOffVM_Task()

結果のスクリプト：

part_of_name = "infra"
for vm in get_vms(part_of_name):
    poweroff_if_enabled(vm)

このマシンのシャットダウンは強制されます。ShutdownGuest（）タスクを呼び出す方がはるかに正しいでしょう：

def shutdown_guest(vm):
    if vm.runtime.powerState == vim.VirtualMachinePowerState.poweredOn:
        vm.ShutdownGuest()
