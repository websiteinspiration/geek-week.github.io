id Softwareがコマンダーキーンのテクノロジーに基づいてWolfenstein 3Dを作成した方法


この記事は、ゲームエンジンのブラックブックの一部です： Wolfenstein 3D は、ナチスの破壊に関する一人称シューティングゲームのゲーム業界に大きな影響を与えた、歴史、コード、および開発の詳細な研究です。

この章では、ゲームの起源と、ゲームが成功とゲーム技術の基盤の上にどのように構築されたかを明らかにします。、任天堂のマリオ3 ポートをPC 用に販売するという失敗した試みの残りから構築されたキーン司令官の。

著者およびプログラマーであるFabien Sanglarによって書かれた完全な本は、紙またはデジタル形式で購入できます。

1990年、シュリーブポート（ルイジアナ州）の小規模企業Softdiskは、シェアウェア市場でビジネスを成功させました。

サブスクリプションベースのビデオゲーム配信サービスを提供することにより、Softdiskは毎月新しいゲームを作成し、メールで購読者に配信しました。事業は順調でしたが、会社の一部の従業員はより野心的でした。

彼らは、スケールアップするのに十分なスキルがあると考え、それを証明したかったのです。彼らは横スクロールをプログラムする新しい方法を作成し、このテクノロジーをアダプティブタイルリフレッシュと呼びました。これにより、PCでハードウェアスクロール（画面のスクロール）を実行できるようになり、コンピューターがNESと競合できるようになりました。1990年代初頭、彼らは週末にノンストップで働き、スーパーマリオ3を再現しました。 PCで任天堂のスキルを実演します。

ディープチームからのアイデアは、彼らが彼ら自身を呼んでいたように、クローンを作ることができました マリオの、しかし、残念なことに、彼女は任天堂に彼らに契約を結ぶように説得することができませんでした。日本の会社がどんなに感心したとしても、マリオシリーズは任天堂のコンソールだけに残したかったのです。

「私たちはデモをアメリカのニンテンドーに送り、そして彼女はそれを京都の本社に送りました。経営陣はデモを見て非常に感銘を受けました。しかし、彼らは知的所有権が機器以外のデバイスにあることを望まなかったので、彼らは単に私たちを賞賛し、それは不可能だと述べました。-John Romero、プログラマー


PC上のマリオ。頭字語IFDはIdeas from the Deepの略です。

このエピソードは、彼らの野心に合った才能があるだけでなく、彼らに大きな機会を潜在的に開く可能性のある労働倫理を持つうまく調整されたチームがあることを彼らに納得させるのに十分であることがわかりました。1991年2月、Softdiskの4人の従業員が飛躍し、独自のid Software会社を設立しました。


id Softwareの創設者。

（これについては、David KushnerのMasters of Doomで詳しく読むことができます。）

彼らはすぐにマリオ3の PC版のために開発された技術を利用しました、独自のゲームのリリースと知的財産の構築を開始しました。時間を無駄にすることなく、チームは1年に少なくとも3つのゲームをリリースしました。

  • 指揮官キーン エピソード1、2、3：渦の侵入（1990年12月14日）
  • 指揮官キーン エピソード4、5、6 ：さようなら銀河（1991年12月15日）
  • 別のゲーム司令官キーン：エイリアンは私の赤ちゃんシッターを食べた（1991年12月）

FormGenによって公開されたこれらのゲームはすぐに成功し、非常によく売れました。チームはまた、ソフトディスク用のゲームを書き続け、そのほとんどはアダプティブタイルアップデートを使用していました。

  • キーンドリームスのキーン司令官（1991）
  • お化け屋敷の危険なデイブ（1991）
  • レスキューローバー（1991）
  • レスキューローバー2（1991）
  • シャドウナイツ（1991）
  • ホバータンク3D ( 1991 ).
  • カタコンベ3D：新しい次元 ( 1991 )

1991年の春、新世代のid Softwareテクノロジーが登場し始めました。ゲームHovertank 3Dでは、プレイヤーは戦車の中にいました。これまでのところ、テクスチャマッピングはまだ行われておらず、ゲームのペースはかなり遅いままです。Catacomb 3Dは、テクスチャの出現とゲームへの没入感の向上を特徴づけました。プレイヤーは、一人称視点でウィザードを制御できるようになりました。

1991年11月、チームはSoftDiskに対するすべての義務を終了しました。彼女の次のゲームは、彼らが作成した3Dテクノロジーを使用し、Wolfenstein 3Dという名前でリリースされます。ゲームの規模と意欲のため、さらに4人がチームに参加し、現在は8人の従業員がいます。

1991年9月、トムとジェイソンの学校の思い出に導かれて、チームはシュリーブポートからマディソン（ウィスコンシン）に移動しました。男たちは、オフィスをハイリッジトレイルのパインズ住宅団地の2階建てのレンガ造りの建物に定着させました。彼らは、不便さを気にしないカーマックを除いて、オフィスから徒歩圏内に住んでいました。Wolfenstein 3D

の開発は1992年1月に始まりました。気温が下がり、空から雪が降ったので、チームは完全に仕事に没頭し、めったにオフィスを離れませんでした。開発は4か月続きました。Wolfenstein 3Dは1992年5月にリリースされました。


この4か月間、チームワークの編成はその時代のゲームスタジオではかなり標準的でした。同じ部屋に座っている4人の男性、速いペースの開発と強い友情に対するサポート（そしてジョンロメロとトムホールのコミュニケーションスタイルによる多くのノイズ）。

下の地図は、無数のF-ZeroゲームがプレイされたSNESとの角度、およびマスターズオブドゥームでしばしば参照されるダンジョンズ＆ドラゴンズゾーンを示しています。チームメンバーの1人（ジョンカーマック）がスタジオの真上に住んでいたというのは、ごく普通の状態でした。

「最初はフロッピーディスクでデータを転送することから始めましたが、最終的には同軸イーサネット*に基づくNovellネットワークを作成しました。バージョン管理システムはありませんでした。驚いたことに、開発にはVisual Source Safeを使い始めたQuake 3までありませんでした。」-John Carmack、プログラマー。

*ジョンロメロの解説：「1991年11月にマディソンに住んでいて、Novell Netware 3.11システムを7,000ドルで購入しました。ファイルサーバー、ケーブル、ネットワークカードが含まれています。」


誰もが、お金で買える最高のPC、つまり4MBのRAMを備えた最も強力な386-DX 33MHzに取り組みました。

プログラミング


開発は、デフォルトではVGAモード3で動作するBorland C ++ 3.1（ただしC言語）の環境で行われました。テキスト画面は80 x 25文字です。

John Carmackがランタイムコードを処理しました。 John Romeroは多くのツールをプログラミングしました（TED5マップエディター、IGRABリソースパッカー、MUSE音楽パッカー）。 Jason Blohovyakは、重要なゲームサブシステム（入力マネージャー、ページマネージャー、サウンドマネージャー、ユーザーマネージャー）を作成しました。


ボーランドのソリューションはオールインワンのシステムでした。BC.EXEと呼ばれるIDEは、いくつかの不安定性にもかかわらず、構文を強調表示する厄介なマルチウィンドウコード編集を提供しました。コンパイラとリンカもシステムの一部であり、BCC.EXEおよびTLINK.EXEという名前でした。

ただし、コマンドラインモードに切り替える必要はありませんでした。IDEを使用すると、プロジェクトを作成、アセンブル、実行、およびデバッグできます。


CRT画面の小さいサイズを補うために、一部の開発者は2台のモニターを使用しました。（John Romero氏の注：「ジョンカーマックと私は、ターボデバッガーを使って、12インチの小さな琥珀色の画面のモノクロモニターで作業しました。」）




ゲームグラフィックリソース


すべてのグラフィックリソースはAdrian Carmackによって作成されました（Kevin Cloudはいくつかのテクスチャを作成し、Wolfenstein 3Dマニュアルブックレットのデザインと構造にも取り組みました）。すべての作業はDeluxe Paint（Electronic ArtsのBrent Iversonによって作成）で行われ、ILBM（InterLeaved BitMap）ファイル（独自のDeluxe Paint形式）で保存されました。


VGAモードはパレットの使用に基づいているため（色は24ビットのRGB値ではなく、256色のテーブルを指すインデックスによって設定されています）、作成プロセスは複雑でした。最初に、エイドリアンは重要な決定をしなければなりませんでした-どの色がパレットに当てはまるか（たとえば、一部のゲームでは、モンキーアイランドはゲームの部分に応じて変化するいくつかのパレットを使用しました。IdSoftwareはよりシンプルなソリューションを選択し、ゲーム全体で1つのパレットを使用しました）、そしてすべてをこれらの色だけでペイントします。


Wolfenstein 3Dパレット。ゲームのすべてがこれらの256色を使用して描かれました。

パレットの座標は、水平方向に0x00から0x0Fまで、垂直方向に0x00から0xF0まで変化します。下の青の水平方向のグラデーションは0xF0で始まり、0xFEで終わります。0xFF（ピンクで示される）は、エンジンによって透明と見なされる特別な色であり、レンダリング時には常にスキップされます。

すべてのリソースはマウスで手動で描画されました。画面にフレームバッファーを表示するときに、VGAモードでフレームバッファーを拡大したため、Adrianは注意して、ゲームが動作するはずの解像度（320x200）ですべてを描画する必要がありました。

「スキャンツールがなかったのに、エイドリアンとケビンはデラックスペイントで直接働きました。」-John Carmack、プログラマー。

グラフィックリソースは2つのカテゴリに分類されます。

  • VGAGRAPH、VGAHEAD、VGADICTファイルでゲームに付属する2Dメニュー項目
  • VSWAPアーカイブで提供される3Dモード要素（壁とスプライト）

資源生産プロセス


グラフィックリソースを生成した後、IGRABツールはすべてのILBMを1つのアーカイブにパックし、リソース識別子を含むCヘッダーファイルを生成しました。エンジンは、これらのIDを使用してリソースを直接参照します。


2Dメニューのリソース作成パイプライン。


エンジンコードでは、リソースの使用は列挙で厳密に記述されています。この列挙型は、HEADテーブルのオフセットであり、DATAアーカイブのオフセットが取得されます。この間接リンクのレイヤーのおかげで、ソースコードを変更せずにリソースを再生成し、その順序を変更できます。

興味深い事実：このようなシステムは、ソースコードの公開後に問題を引き起こしました。バンドルの.hヘッダーは、シェアウェアバージョンまたは以前のWolfenstein 3Dバージョンのリソースファイルと一致しませんでした。発表された見出しは、運命の槍から取られました。サイトfabiensanglard.net [ Habréに関する記事の翻訳 ]の記事「1992年の庭にいるかのように編集する」の記事で、これがどのような混乱を引き起こしたかを確認できます。

で1992年に発行された公式のWolfenstein 3Dガイドでは、クリエイティブプロセスについて説明しています。トムホールの多くの絵が含まれ、トムによって作成され、アーティストのチームによってピクセルに変換された一連のスケッチを示しています。

「Idクリエイティブディレクターのトムホールが画面のアイデアを思いついたとき、彼はエイドリアンカーマックにスケッチを与えました。以下は、トムによるスプラッシュスクリーンデザインのスケッチの例です。ゲームの3番目のドラフトを選びました。」-「公式のWolfenstein 3Dガイド」
