フルスタックサーバーとしてのNuxt：フロントエンド+バックエンドAPIサーバー（パート1）



  • パート1：フルスタックサーバーとしてのNuxt：フロントエンド+バックエンドAPIサーバー
  • パート2：Nuxtフルスタックサーバーによる追加のSSRパフォーマンス

Nuxt 開発者は、APIにアクセスする3つの方法を提供します。

  1. 内蔵のNuxt Server ConnectとserverMiddlewareを使用
  2. 統合フレームワーク（Express、Koa、Hapiなど）
  3. 外部サーバーAPI

私が使ってAPIサーバーを整理する方法を簡単な例を示しますserverMiddlewareを同じにNuxtインスタンス私たちはフロントエンドを担当していること。

このアプローチの利点は何ですか。

  • テクノロジースタックのクイックスタートと最小化。フルスタックサーバーになるNuxtのみが必要です。フロントエンドページをレンダリングし、バックエンドAPIサーバーとして機能します。

  • , . , , .
  • Nuxt Connect, Node () 55 . , Nuxt' (Express, Koa, Hapi ..) .
  • ミニマリズムにも関わらず、アプリケーションのスケーリングは完全に自由であり、1つのデータベースと対話する複数のフルスタックNuxtサーバーを簡単にデプロイでき、さらにnginxをデプロイして複数のフルスタックNuxtサーバー間の負荷を分散できます。また、マイナーな変更により、各Nuxtインスタンスは、唯一のAPIサーバー（バックエンドサーバーのみ）、およびフロントエンドサーバー（フロントエンドサーバーのみ）として機能できます。
  • そして、大きなボーナス!!! フロントエンドとバックエンドに 1つのインスタンスを使用すると、サーバーページのレンダリング（SSR）を大幅に高速化できます。これについては、パート2で説明します。

    export default async (req, res, next) => {
  let url = req._parsedUrl.pathname.replace(/^\/+|\/+$|\.+/g, "");
  url = url.split("/");
  let method = url.pop();
  let controller = url.slice(1).join("/");
  let api = require("../api/" + controller);
  let result = await api[method](req.params);
  res.end(JSON.stringify(result));
};

async function getDataFromDB() {
  return (await require("axios").get(`https://reqres.in/api/users?page=1`))

async function index() {
  let res = await getDataFromDB();

  // some business logic
  let ret = res.data.map(el => {
    return {
      id: el.id,
      full_name: el.first_name + " " + el.last_name,
      highlight: el.first_name.charAt(0) === "J"
    };
  });
  ret = ret.sort(() => 0.5 - Math.random());

  return ret;
}

export { index };
