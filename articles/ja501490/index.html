mathモジュール
のいくつかの変更最大公約数を検索するmath.gcd（）関数は整数のリストを受け入れるようになり、1つの関数が2つ以上の数値の公約数を検索できるようになりました。無制限の整数を受け入れるmath.lcm（）の
最小公倍数を決定する関数があります。 次の2つの機能は相互に関連しています。math.nextafter（x、y） -y方向に移動する場合、xに最も近い浮動小数点数を計算します。math.ulp（x）


-「最後のユニット」の略で、コンピュータの計算の精度に依存します。正の数の場合、数値の最小値が返され、x + ulp（x）を加算すると、最も近い浮動小数点数が取得されます。
import math

math.gcd(24, 36)
>> 12
math.lcm(12, 18)
>> 36
math.nextafter(3, -1)
>> 2.9999999999999996
3 - math.ulp(3)
>> 2.9999999999999996
math.nextafter(3, -1) + math.ulp(3)
>> 3.0

有効な式をデコレータにすることができるようになりました（PEP-0614）
デコレータからの制限がなくなりました。これにより、名前のみがデコレータとして機能し、その構文ではドットによる分離のみが許可されます。

多くの人がそのような制限の存在について考えていたとは思いませんが、pepの説明は、革新によってコードがスリムになった場合の例を示しています。類推により、引用することができます
単純化された人工的な例： 
def a(func):
	def wrapper():
		print('a')
		func()
	return wrapper
	
def b(func):
	def wrapper():
		print('b')
		func()
	return wrapper
	
decorators = [a, b]

@decorators[0] #   3.8        
def some_func():
	print('original')
	
some_func()
>> a
>> original
