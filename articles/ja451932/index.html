PHDays 9：安全開発セクションへようこそ



Positive Hack Days 9フォーラムでは、Positive Development User Groupコミュニティの安全な開発に関するセクションが2日間開催されます。12のプレゼンテーションが参加者を待っています。毎日の最初の部分では、ビジネスプロセスに関する技術レポートが2番目に開催されます。

5月21日


Vladimir KochetkovとValery Pushkar（Positive Technologies）は、効果的な静的JavaScriptコードアナライザーの開発における経験を共有し、複雑な例を使用してアナライザーの操作をデモンストレーションします。

Sergey Khrenov（PVS-Studio）がSAST、CWE、CVE、SEI CERT、DevSecOpsについて話し、信頼できるアプリケーションの作成に役立つプログラミング標準を開発者に紹介します。

ミハイル・シェアバコブ（王立工科大学、スウェーデン）の報告がされたに専念 .NETで直列化復元のプロセスの脆弱性。受講者は、どの.NETシリアライザーが脆弱であるか、どのツールを使用して脆弱性を検索できるか、およびどのペイロードが.NETアプリケーションで既知であるかについても学びます。

アレクサンダーチェルノフ（モスクワ州立大学）とエカテリーナトロシーナ（HSE）が 、トレーニングの最初から一貫して安全な開発を推進する方法を説明します。彼らは、低レベルのプログラミングとオペレーティングシステムの基本的なコースの例について、安全な開発を教えるという目標と目的を策定します。Sergey Gorokhov（EPAM Systems）のスピーチ

から、学生はソフトウェア製品をヨーロッパのGDPR法に準拠させる方法と、顧客が「GDPR準拠の製品を作成する」と尋ねた場合の対処法を学びます。

 5月22日


 Androidアプリケーションの実際のセキュリティ問題は、Dmitry TereshinとNikolay Islamov（Tinkoff Bank）が対応します。彼らは、OWASPガイドで十分にカバーされていない、Androidアプリケーションの脆弱性の理由を強調します。Alexey Dremin（独立した専門家）

によるプレゼンテーション -アプリケーションの継続的なセキュリティテストのためのパイプラインの構築について。彼は、パイプラインを開始するポイント、CI / CDでどのようにどの統合を行う必要があるか、どこに保存し、どこで結果を処理するかを理解します。

安全なプログラミングプロセスの構築については、ウラジミールサドフスキー（ "M. Video"）のスピーチで聞くことができます。彼は、アーキテクチャ設計、自動テスト、ビジネスロジックエラーの特定、およびバグ報奨金について話します。

アレクセイRyzhkov（EPAMシステムズ）は、EPAMのセキュアな開発プロセスを実装の経験に基づいて、話をするプロジェクト（セキュリティ影響分析）のセキュリティへの影響の観点から各機能を解析するためのプロセスの構築について。

Sergey Prilutsky（MixBytes）は、スマートコントラクトセキュリティの自動監査のトピックを提起します。彼は、Ethereum Virtual Machineの例を使用してスマートコントラクトとアナライザーを実行するための実行可能コードの機能、およびスマートコントラクトの攻撃ベクトルとその自動検出の可能性について話します。セキュリティリスクの評価

に関するVitaliy Katunin（EPAM Systems）の報告：学生は、リスク評価をすべての関係者に透過的にして、脅威とセキュリティ要件の下位互換性を実現する方法を学びます。

Anton Basharin（Swordfish Security）は、AppSecプロセスの自動化、メトリックの収集、それらの視覚化と分析における彼の経験を共有します。
