Vite - Lead-Entwicklung ohne Bundles auf Vue

Hallo Habrowsk. Im Juni startet OTUS den Vue.js Developer- Kurs . Im Vorfeld des Kursbeginns haben wir traditionell eine Übersetzung von interessantem Material vorbereitet.


Eines Tages beschloss Ewan Yu (Schöpfer von Vue.js), nachts nicht zu schlafen und schuf schließlich Vite .



Mit Vite können Sie Anwendungen in Vue mit Einzeldateikomponenten erstellen, ohne den Zusammenstellungsschritt von Bundles ausführen zu müssen. Importe werden vom Browser als native ES-Module angefordert. Der Dev-Server fängt Anforderungen für .vue-Dateien ab und kompiliert sie im laufenden Betrieb. Und es passiert sofort.

Bitte beachten Sie, dass Vite immer noch eine experimentelle Lösung ist und ich nicht weiß, was in Zukunft passieren wird und wie lange dieser Artikel nützlich bleibt. Betrachten Sie es als Aperitif oder einfach als etwas, das Spaß macht.

Lass es uns versuchen


Erstellen Sie die folgenden Dateien:

Comp.vue

<template>
  <button @click="count++"> {{ count }}</button> 
</template>

<script>
export default {
  data: () => ({ count: 0 })
}
</script>

<style scoped>
button{
  font-size: 2rem;
}
</style>

index.html

<div id="app"></div>
<script type="module">
  import { createApp } from 'vue'
  import Comp from './Comp.vue'

  createApp(Comp).mount('#app')
</script>



npx vite

Gehen Sie dann wie http://localhost:3000folgt vor : Gehen Sie zu und bearbeiten Sie die Datei .vue, um sofortige Änderungen anzuzeigen.

Produktionsmontage


Jetzt erstellen wir eine produktionsbereite Anwendung.

Erstellenpackage.json

{
  "scripts": {
    "build": "vite build"
  }
}

Lauf:

npm i -D vite
npm run build


Überprüfen Sie das Verzeichnis mit dem Code.
Probleme, auf die ich gestoßen bin:

  • Fehler: Modul 'tslib' kann nicht gefunden werden. Wird durch
    Ausführen von npm i -D tslib behoben
  • Fehler: ENOENT: Keine solche Datei oder kein solcher Director.
    Durch manuelles Erstellen eines leeren Verzeichnisses behoben

Gedanken


Der Start war sehr einfach und die Entwicklung ging schneller, da ich sofort das Ergebnis der Änderungen sah. So etwas wie eine schnelle Zusammenstellung. Es sieht also sehr vielversprechend aus, vielleicht werden wir nach einer Weile zu einer solchen Entwicklungsmethode für Vue übergehen.

Evan Yu, mach deine Magie.

Unterhaltsame Tatsache: vue wird aus dem Französischen als Ansicht übersetzt und vite so schnell.

Es scheint, dass zu diesem Zeitpunkt die Arbeit an Vite abgeschlossen ist und möglicherweise nach Vue 3 fortgesetzt wird. Der einfachste Weg, dies zu versuchen, ist Vue 3 Beta.

Ich sagte, dass es basierend auf einem Tweet von Evan Yu fertiggestellt wurde. Wenn wir uns jedoch die Commits im Repository ansehen, können wir sagen, dass die Arbeit noch im Gange ist.
