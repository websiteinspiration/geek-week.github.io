モバイルアプリケーションテストの自動化：ツールの比較

テストの自動化は、モバイルアプリケーションの場合も含め、いくつかの問題を一度に解決するのに役立ちます。専門家は、手間のかかる日常的な手順を手動で実行する代わりに、それらの大部分をフレームワークに委任できます。自動化はテストを簡素化し、回帰テストの高速化を支援し、以前はアクセスできなかったタイプのテストを使用することも可能にします。

市場で定着したいくつかのツールを比較し、開発を続けます。この知識は、特定のモバイルアプリケーションのテストに使用するソリューションの選択に役立ちます。



この記事は、専門家に新しい展望を開く可能性は低いですが、モバイルテストの基本を学ぶことにした初心者や、ある程度、中級レベルの専門家にとっては役立つかもしれません。

Androidアプリケーションテストオートメーション


UI Automator


高度な外部統合を備えた強力なテストツール。つまり、このフレームワークを使用すると、アプリケーション自体をテストできるだけでなく、オペレーティングシステムや他のアプリケーションと「通信」できます。たとえば、Wi-Fi、GPSをアクティブまたは非アクティブにしたり、テスト中に設定メニューを開いたり、その他の外部のやり取りを行うことができます。

UI Automatorの目的はブラックボックステストです。つまり、分析は、コードにアクセスせずに外部ユーザーの立場から実行されます。

主な機能は次のとおりです。

  • テスト中に画面に表示されるコンポーネントを追跡および分析するためのUI Automator Viewer。要素とそのプロパティに関する情報を提供するため、より関連性の高いテストを簡単に作成できます。
  • デバイスのステータスとデバイスで実行中のプロセスに関する情報を取得するためのAPI。
  • クロスプラットフォームテスト用のUI Automator API。

エスプレッソ


UI Automatorよりも軽量なツールで、外部アプリケーションとの対話には適していませんが、特定のアプリケーションのソースコードにアクセスしてホワイトボックスをテストしたり、いくつかの内部プロセスと構造にアクセスできます。

ただし、Espressoは強力なAPI https://github.com/hamcrestで際立っています。インターフェースは

UI AutomatorとEspressoは相互補完的であり、同じプロジェクト内で組み合わせて使用​​できます。

iOSアプリケーションのテスト自動化


XCUITest


アプリケーションコードにアクセスせずにブラックボックステストを行うためのツール。これはネイティブ製品でのみ機能します-残念ながら、クロスアプリテストは機能しません。

一方、フレームワークのネイティブな性質は、XCUITestを使用すると、開発者とテスターの相互理解の度合いが、一方と他方が異なる言語を使用する場合よりもはるかに高いという観点からの利点です。

便利な追加機能はテストレコーダーです。テストレコーダーを使用すると、コードを使用しないユーザーでも、アプリケーションでアクションを記録してテストを作成できます。

このツールを使用すると、一般的なミスや、ユーザーコード操作にアクセスできない不要なミスを回避できます。ただし、XCUITestにはいくつかの欠点もあります。

XCUITestは、Espressoとは異なり、別のスレッドで機能します。テスト中は、特定の要素とパラメーターが表示されるまで待つ必要があります。アプリケーションの現在の状態は読み取られず、データの更新が遅れると、要求された要素を検出できなくなる可能性があります。

アールグレイ


EarlGreyは、ユーザーエクスペリエンスの再現に重点を置いています。画面上の要素が視覚的に表示されない限り、アプリケーションでの作業のシミュレーションは開始されません。

同時に、多くのアメニティと利点が記されています。まず第一に、エキスパートはフレームワークがリクエスト、UI、スレッドを同期する方法を好みます。waitforviewとwaitは必要ありません。

次に、すでに述べたように、要素の可視性の追跡に特別な注意が払われます。このツールには、インターフェイスの読み込みをチェックするための追加のレイヤーがあり、ユーザーのジェスチャー（スワイプ、クリック）をアプリケーションイベントレベルで直接再現します。

リポジトリリンク：github.com/google/EarlGreyおよびgoogle.github.io/EarlGrey。

ユニバーサルツール


ユニバーサルツール（または「組み合わせ」）を使用すると、選択をAndroidまたはiOSだけに制限するのではなく、両方のプラットフォームで使用できます。

このようなツールは、次のタイプのアプリケーションのテストに適用できます。

  • ネイティブアプリ（ネイティブアプリ）-Android、iOS、Windows SDKの直下に記述されています。
  • モバイルWebアプリ— SafariやChromeなどのモバイルブラウザーから利用できます。
  • ハイブリッドアプリ-ユーザーはWebアプリケーションのシェルを操作します。つまり、ネイティブアプリケーションのインターフェイスを介してWebコンテンツとやり取りします。

デトックス


私たちの意見では、DetoxはReact Nativeで記述されたアプリケーションに便利です。テストはJavaScriptで記述されていますが、iOSおよびAndroidアプリケーションは同じJavaScriptコードから生成され、可能な限り類似しています。これにより、両方のプラットフォームで同じテストを使用できます。

Detoxの主要な機能は、グレーボックステストです。この場合、フレームワークは内部メカニズムにある程度アクセスできます。これにより、アプリケーションの外部動作をより深いレベルで発生する動作と関連付けることができます。

デトックスはメモリにアクセスし、実行中のプロセスを追跡できます。グレーボックスの原則は、不安定性との戦いに役立ちます。つまり、エンドツーエンドのテストを通じて、

  • コードに変更を加えなくても、テストはランダムにクラッシュする可能性があります。

  • , .

奇妙なことに、「グレーボックス」は「ブラックボックス」に比べて安定性が高いだけでなく、速度も高速です。あらゆる種類の一時停止、waitUntilを回避すると、灰色のボックスが5〜10倍速くなります。

DetoxはWebDriverを必要とせず、JSONを介してネイティブドライバーを操作します。デバイスで直接ネイティブメソッドを使用します。このフレームワーク内では、iOS用のEarlGreyとAndroid用のEspressoが使用されます。

フレームワークは、エミュレーターと物理デバイスの両方で機能します。

Appium


Appiumの利点は、フレームワークと互換性のある特別な形式にアプリケーションを変換することなく、単一のAPIを使用して各プラットフォームのテストを記述できることです。

テスト時には、ベンダーのフレームワークが使用されます。つまり、元のアプリケーションで作業します。Android 4.2以降ではそれぞれUiAutomator / UiAutomator2が使用され、iOS 9.3以降ではXCUITestが使用されます。WebDriver（別名Selenium WebDriver）は、フレームワークフレームワークとして使用されます。

Appiumの原則：

  • アプリケーションを再コンパイルしたり、テストを自動化するために変更したりする必要はありません。
  • 1つの言語またはフレームワークにアタッチする必要はありません。
  • 自動化APIに関しては、ホイールを再発明する必要はありません。

Appiumの使用は、複数のプラットフォームでのテストを一度に自動化するツールが必要な場合に適しています。Webアプリケーションのテストの経験はあるが、モバイルアプリケーションのテストの自動化の経験がないスペシャリストがいる場合に役立ちます。

一般に、これは、限られた開発言語のセットに適応する必要なしに、プロジェクトのニーズに合わせて変更できる柔軟なツールです。

ラノレックス


デスクトップ、モバイル、およびWebアプリケーションをテストするための有料の包括的なツール。プログラミングを使用して、スクリプトをまったく使用せずにテストできます。エミュレータだけでなく、ライブデバイスでもテストする機能を提供します。

このツールを使用すると、テストを作成して構成し、一元的に管理できます。コントロールセンターでテストを作成し、さまざまな外部環境や任意のデバイスで実行できます。

JiraやTFSなどのアプリケーション管理システムや、GitやSVNなどのバージョン管理システムを使用して、既存のCI環境と簡単に統合できます。

Ranorexには、SQL、CSV、Excelからデータをロードするデータ駆動型テストがあります。

このツールは絶対にあらゆるデバイスに適しており、各デバイスでの並行テストをサポートしています。

3つのテスト方法すべてを組み合わせる：ブラックボックス、ホワイトボックス、グレーボックス。

テストコンプリート


モバイル、ウェブ、デスクトップアプリケーションの自動化をテストするための有料環境。AndroidとiOSをサポートし、アプリケーションタイプのコンテキストでは、ネイティブ、ウェブアプリケーション、ハイブリッドをサポートしています。

このツールは主に機能テストと単体テストに重点を置いており、他の多くのタイプのテストを実行する機能も提供します。

  • 回帰;
  • データ駆動型テスト;
  • 分散テストなど。

TestCompleteにはレコーダーがあります。テストはエディターでアクションを記録し、コマンドを設定することで作成されます。次に、ツール自体で直接起動するか、サードパーティのアプリケーションにエクスポートできます。

このツールは、オブジェクトとコントロールを認識するための特別なコマンドを提供することにより、オブジェクトとコントロールを認識します。Jenkins、Git、およびJiraと統合して、継続的なシームレスなテストを実行できるようにします。
