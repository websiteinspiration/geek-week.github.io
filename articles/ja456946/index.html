Kubernetes管理用のGrafanaパネル

みなさん、こんにちは！今日も、Kubernetes Based Infrastructure Platformコースの学生向けに翻訳された資料を引き続き共有しています。読書を楽しむ。



前書き




しばらくの間、私はKubernetesを管理するためのGrafanaダッシュボードの作成を実験しています。いくつかのKubernetesクラスターの所有者および管理者として、いくつかのメトリックを視覚的に表現したいと思いました。まず、Kubernetesコントロールプレーンとノードコンポーネントの通常のGrafanaダッシュボードが必要です。kube -api、kube-scheduler、kube-controller-manager、kubeletとkube-proxyです。次に、結果のパネルをGitHubに投稿して、残りが提案やエラーレポート（問題）を残し、変更の受け入れ要求（プルリクエスト）を送信し、パネルを更新（gitプル）して、作業を支援できるようにします。

Grafanaダッシュボードの公開


Grafanaパネルは現在、主にgrafana.com/dashboardsで公開されてい  ます。正しいパネルを取得するには、ページでそれを見つけ、識別子をコピーしてローカルのGrafanaインスタンスにインポートする必要があります。ダッシュボードのインポートは比較的簡単ですが、経験上、ほとんどのユーザーはダッシュボードでの作業が不便であると感じています。これにはいくつかの理由があります。

まず、人々はさまざまなタグセレクターを使用して、Prometheusでの監視目標を示します。第2に、新しいバージョンのPrometheusエクスポーターおよびメトリックにアクセスするためのアプリケーションでは、メトリックの名前が絶えず変更され、削除されます。第三に、本当に高品質のダッシュボードを見つけることは非常に困難です-インポート後に最終的に何が得られるかは決してわかりません。パネルを選択したら、すべてのグラフをくまなく調べて、ラベルのセレクターとともにメトリックを編集する必要があります。これは大量の手作業です。

Mixinsプロジェクトの監視


Monitoring Mixins という非常にクールなプロジェクトを見つけました。実際、これは一種のエコシステムであり、パネルのすべての問題を克服するように設計されています。アイデアはFrederick Branchiku（Frederic Branczyk）に思い浮かびました。オリジナルの設計ドキュメントを書いたのは彼でした。これを読むことを強くお勧めします。さらに、彼はダッシュボードを更新し、さまざまなGitHubリポジトリからそれらを抽出するための実際のメカニズムを考えました。このメカニズムはと非常によく似て  go get おり、私は本当に気に入っています。ここでは、このメカニズムを使用したjsonnet-bundlerパッケージマネージャーの簡単な説明を読むことができます。有用な開発をしてくれたFrederickに感謝します。

Kubernetes Grafana Mixinの使用


まず、とをインストールする必要が  jsonnet あり  jsonnet-bundlerます。

Jsonnet


macOSを実行している場合は、コマンドを使用できます  brew install jsonnet。それ以外の場合は、このコンポーネントを自分でコンパイルすることをお勧めします。

git clone https://github.com/google/jsonnet.git jsonnet_git
cd jsonnet_git
make
sudo mv jsonnet /usr/local/bin/

Jsonnetバンドラー


go get -u github.com/jsonnet-bundler/jsonnet-bundler/cmd/jb

パネルのインポート


プロジェクトの新しいディレクトリを作成し、チームに入力し  cd ます。mixin

をインストールします。

jb init
jb install https://github.com/povilasv/kubernetes-grafana-mixin

新しいファイルを作成します  config.libsonnet。

local kubedashboards = import 'kubernetes-grafana-mixin/mixin.libsonnet';

kubedashboards {
  _config+:: {
    kubeletSelector: 'job="kubernetes-nodes2"',
    kubeSchedulerSelector: 'job="kube-scheduler2"',
    kubeControllerManagerSelector: 'job="kube-controller-manager2"',
    kubeApiserverSelector: 'job="kube-apiserver2"',
    kubeProxySelector: 'job="kube-proxy2"',
  },
}

その後、情報パネルがインポートされ、ジョブセレクタが上書きされます。
Prometheusタグセレクターを環境に合わせて修正します。
次にディレクトリを作成します  dashboards 。最後に、jsonnetコンパイルして実行  します  config.libsonnet。

jsonnet -J vendor -m dashboards -e '(import "config.libsonnet").grafanaDashboards'

次のリストが表示されます。

dashboards/kube-apiserver.json
dashboards/kube-controller-manager.json
dashboards/kube-proxy.json
dashboards/kube-scheduler.json
dashboards/kubelet.json

Grafanaへのダッシュボードの追加


構成ファイルを介してパネルを配置することを強くお勧めします。これを行う方法の詳細については、Grafanaのドキュメントをご覧ください。

一方、Grafanaインターフェースを開いてパネルを追加することを妨げるものは何もありません。

ダッシュボードを更新する


時間の経過とともに、パネルは変化します。それらを更新するには、次のコマンドで十分です。

jb update
