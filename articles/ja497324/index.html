パートII：Canny 3 tinyの回路とプログラムの接続


インターネット上にUARTを介してArduinoを接続する方法の例はたくさんありますが、Cannyの例ははるかに少ないので、それから始めましょう。
記事の長さを膨らませたり、繰り返したりしないように、この記事ではCannyとCannyLab開発環境での作業方法の最も基本的な問題と、センサーをA / Dコンバーター（ADC）に接続して仮想COM経由でメッセージを送信する例を検討したことを思い出しますポート、私はこの記事で作りました。

これを読まないが、すぐにセンサーをADCに接続しようとする人にとって、重要なポイントがあります。私の間違いを繰り返さないでください。コントローラーの5番と6番の出力がADCモードで機能するためには、下の図のようにジャンパーを設定する必要があります。私がやったとしても、難しくはありません。


また、コントローラー出力のもう1つの重要なポイントは電源電圧です。つまり、USBではなく外部電源から接続する場合は、出力電圧がセンサーとUARTの許容電圧を超えないようにしてください。

この記事では、ジャンパーは1つで十分ですが、はんだごてを入手した場合、ささいなことに時間を費やす必要があるのはなぜですか。

端子台に注意してください。ネイティブケーブルハーネスはこのバージョンのコントローラーに標準で付属しています。車内で使用するのはタフでおそらく便利ですが、テーブルでの実験には適していません。ノウハウを思いついて、ネイティブのハーネスをコンピュータ機器によくバンドルされているハーネスに交換したり、DIYの電子機器を販売している店舗で販売したりすると、非常に便利でした。それらはブロックにひどく座っておらず、柔らかく、同時に追加の延長を与えます。

接続図に移りましょう：


すべてが非常にシンプルで、特別なスキルは必要ありません。ほとんど誰でも処理できます。

まず、取締役会の力について考えましょう。コントローラーの「-」出力から「グラウンド」を取得しますが、プラスを使用すると、すべてがそれほど明確ではありません。一見すると、「+」端子に接続するのは論理的に思えますが、実際には明らかに+ 5V INPUTの端子です。したがって、その代わりに、出力の1つを+ 5V出力として構成し、それに論理ユニットを供給して「プラス」に引き上げます。 3番出口をとることにしました。

圧力センサーは非常に簡単に接続されています。実際、これは抵抗器です。その「脚」の1つに電力を供給し、2番目の「脚」から、ADCに接続されたコントローラー入力（No. 5またはNo. 6）にワイヤを巻きます。 6番を選びました。

ちなみに、圧力センサーを２回紛失しました。 2つ目の間違いはセンサーの接続でした。ブレッドボードには収まりません。ネジ留め式端子台などに接続する必要があることがわかります。私は「ワニ」と一緒に出なければなりませんでした。

モーションセンサーはそれほど複雑には接続されていません。ボードから+ 5VをVinジャックに入れ、ブレッドボードのGNDジャックに接続し、OUTジャックをコントローラーの任意のデジタル入力に接続します。出力5に接続しました（ ADCモードに切り替えません）。

UARTを介してコントローラを「ラズベリー」に接続するために残ります。これを行うには、UARTモードでTX出力になるコントローラー出力No. 2をRaspberry GPIO RX入力に接続する必要があり、ボードのグランドとRaspberry PI GPIO GNDも接続する必要があります。

多くの人が電圧レベルを3.3 Vに調整することを勧めており、分圧器を介してCannyを接続しようとしましたが、それがない場合よりも動作が悪い（またはまったく動作しない）ため、この場合はRaspberry UARTに直接接続しました。燃え尽きたものは何もないようです。すべてが機能しています。


Bluetooth経由で可能な場合、UARTへの配線で「なぜ接続されているのですか？」実際、UARTを介してArduinoをネットワーク上のHC-06モジュールに接続する多くの例があります。あなたがフォーラムを信じているなら、キャニーも問題なくそれに接続します。しかし、私はそれを自分で試したことはありません。ワイヤーで接続したかったので、Bluetooth接続を次回まで延期することにしました。

パートIV：Raspberry PIとPythonの監視プログラム


まず、UARTとSSHおよびVNCを有効にしてから、スマートフォンを使用してRaspberryに接続する必要があり

ます。RaspberryPI設定に移動して、図のようにSSH、VNC、UARTを有効にします。



念のため、Bluetoothをオフにしました。

簡単なPythonプログラムを作成します。組み込みIDEを使用し、追加のPythonモジュールはインストールしませんでした。中サイズのデスクトップイメージを備えたRaspbianで、約1.1 GBを使用しました。

監視プログラムでは、コンソールとグラフィカルインターフェイスの両方を使用して情報を表示します。GUIを実装するために、Tkinterを使用することにしました。私は実際にそれを2回目に使いますが、必要なのは、いくつかのテキストフィールドとダイアログボックスだけです。

これがプログラムコードです：

import serial
import time
from tkinter import *
from tkinter import ttk
from tkinter import messagebox

serialport = serial.Serial("/dev/ttyS0", baudrate=9600, timeout=1.0) 

window = Tk()  
window.title("Security system for my pen")  
window.geometry('400x170')
#Presure sensor label
lbl_ps = ttk.Label(window, text="Pressure sensor", font=("Arial Bold", 20))
lbl_ps.grid(column=0, row=0)
lbl_ps_status = ttk.Label(window, text=" ",  font=("Arial Bold", 20))
lbl_ps_status.grid(column=1, row=0)
#Motion sensor label
lbl_ms = ttk.Label(window, text="Motion sensor", font=("Arial Bold", 20))
lbl_ms.grid(column=0, row=1)
lbl_ms_status = ttk.Label(window, text=" ", font=("Arial Bold", 20))
lbl_ms_status.grid(column=1, row=1)

while True:
   counter = 0
   rcv = serialport.readline().decode('utf-8').replace("\r\n","").split('  ')
   if (len(rcv)==2):
       ps=rcv[0]
       ms=rcv[1]
       print (ps+ " " +ms)
       if (int(ps)<380):
           lbl_ps_status.config(text = " Warning!")
           counter += 1
       else:
           lbl_ps_status.config(text = " Ok")

       if (int(ms)>0):
           lbl_ms_status['text']=" Warning!"
           counter += 1
       else:
           lbl_ms_status['text']=" Ok"  
       window.update_idletasks()  
       window.update()
       if (counter == 2):
            messagebox.showinfo("Alarm!", "heck your pen!")     
       time.sleep(1)

結論


結論として、スマートフォンからRaspberry PIに接続することを検討してください。原則として、何も複雑ではありません。コンピュータから接続できれば、電話から接続できます。あなたが知る必要があるすべてはラズベリーIP、ユーザー名とパスワードです。 AndroidスマートフォンにVNC Viewerをインストールしましたが、確かに別のクライアントを使用できます。

あまり便利ではありませんが、機能します。
