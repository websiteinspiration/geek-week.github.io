チュートリアル：リアクティブスプリングブート

先月、Trisha Gee（JetBrains）がSpringOneで、Spring Boot、Kotlin、JavaFXを使用してリアルタイム株価アプリを作成するデモを発表しました。ライブデモが録音され、70分のビデオとして視聴できます。

12月18日、Trishaはdzone.comで、このビデオに基づいたスプリングブートリアクティブチュートリアルの公開を完了しました。一連の短いビデオとして、各ステップをよりゆっくりと詳細に説明するブログの投稿を添付しています。

このチュートリアルをロシア語に翻訳するというアイデアがありました。しかし、トリシャはこれについての私の質問に丁寧に答えました：
「私の組織に確認させてください。私はそれを翻訳してもらいたいのですが、物事を翻訳するための好ましい方法があるかもしれません。」

そこで、次の一連のレッスンを含むこのチュートリアルの概要に限定することにしました。

  1. Kotlin RESTサービスの構築
  2. リアクティブストリーム用のRESTクライアント
  3. JavaFX Spring Bootアプリケーション
  4. JavaFX折れ線グラフ
  5. 共有Beanの自動構成
  6. 反応データの表示
  7. 複数のサブスクライバーのサブスクライブ
  8. Kotlin RSocketサーバー
  9. Java RSocketクライアント
  10. クライアントを切り替えるためのSpringプロファイル

このチュートリアルは、Kotlinの内部インターフェース、Javaクライアント、JavaFXユーザーインターフェースを使用して完全なSpring Bootアプリケーションを作成する一連のレッスンです。

パート1. KotlinでのRESTサービスの作成


チュートリアルの最初の部分では、アプリケーションのサーバー側として機能するKotlinでSpring Bootアプリケーションを作成する方法について説明します。マニュアルの以下の部分で接続できるリアクティブRESTサービスが作成されます。

このブログエントリには、ステップバイステップのプロセスを示すビデオと、テキスト形式のガイド（ビデオのトランスクリプトから適応）が書かれた形式を好む人のために含まれています。

チュートリアルのこの部分には、次の手順が含まれます。

  • Spring Boot Serviceプロジェクトの作成
  • Spring Bootプロジェクトの構造
  • RESTコントローラーの作成
  • 株価のデータクラスの作成
  • 価格の生成と返品
  • アプリケーションの起動

チュートリアルのパート1でこれらの手順を完了すると、Reactive Streamsを使用してランダムに生成された株価を毎秒1回発行する単純なKotlin Spring Bootアプリケーションが作成されます。

チュートリアルのパート1の完全なコードはGitHubで入手できます。

パート2.リアクティブフロー用のRESTクライアント


ではチュートリアルの2 RESTサービス、送信株価1秒に1回に接続春にジェットのJavaクライアントを作成する方法を示しています。このクライアントは、マニュアルの後続のセクションで使用されます。開発中、TDDベースのプロセスを使用してクライアントを作成し、テストします。

このブログエントリには、ステップバイステップのプロセスを示すビデオと、書面のフォーマットを好む人のためのテキストガイド（ビデオのトランスクリプトから適応）が含まれています。

チュートリアルのこの部分には、次の手順が含まれます。

  • クライアントのプロジェクトを作成する
  • 顧客クラスの作成
  • 顧客テストを作成する
  • 顧客基本価格メソッドの作成
  • 株価を保持するクラスを作成する
  • テストへのアサーションの追加
  • クライアントをRESTサービスに接続する
  • 統合テストを実行する
  • 統合テストのより徹底的なアサーション

リアクティブアプリケーションのテストは簡単なスキルではありません。これを行うには、この単純な例よりもはるかに優れた方法があります。ただし、統合テストは、株価のAPIおよびクライアント機能を決定するために正常に使用されました。

このクライアントはエンドポイントに接続し、エンドポイントはサーバーから送信されたイベントを生成し、別のサービスで使用できるStockPriceオブジェクトのストリームを返します。これを行う方法は、このシリーズの次のビデオで示されます。

完全な2パートコードはGitHubで入手できます。
