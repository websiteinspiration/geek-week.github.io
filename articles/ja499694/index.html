2020 Gartner Quadrant for Application Monitoring Solutions（APM）

数日前、GartnerはAPMシステムに関する新しいレポート（Application Performance Management）を公開しました。私の電報チャンネルで、この象限の発表前に、この象限のリーダーに対する彼らの予測について加入者に尋ねた。一般的に、陰謀は機能しませんでした。カットの下の詳細。



陰謀はリーダーとうまくいかなかったが、新しい野心的なプレーヤーが象限に現れました-Datadog、InstanaとSignalFx（現在はSplunkの一部）。それらは非常に活発に開発されており、今後数年のうちに、この分野の長い製品であるAppdynamics、New Relic、Dynatraceとともに市場のリーダーになる可能性があると思います。

2025年までに、新しいクラウドベースの監視アプリケーションの50％は、ベンダーエージェントの代わりにオープンソースツールを使用して互換性を高めます。

そして、レポートではこの傾向に多くの注意が払われました。オープンテクノロジーサポートの存在はベンダーの利点であり、存在しないことは欠点です。以下に、Gartnerによると、象限に参加しているベンダーとその長所と短所のリストを示します。

今年、レポートにAPMソリューションを含めるための基準には、3つの主要な要件も含まれています（ただし、昨年と比べて、要件はわずかに変更されています）。

フロントエンド監視。このアクセシビリティとパフォーマンス監視へのアプローチは、Digital Experience Monitoring（DEM）手法の一部です。このレポートの目的のために、アプローチには実際のユーザーの監視（RUM）、Webおよびモバイルアプリケーションのユーザーの代理トランザクションの監視。フロントエンド監視は、APMソリューションの昨年のマジッククアドラントの開発であり、この要件はデジタルエクスペリエンス監視と呼ばれていました。 DEMには、APMで使用されるテクノロジーよりも多くのテクノロジーが含まれています。

アプリケーションの検出、追跡、および診断（ADTD）。これは、Webサーバー、アプリケーションサーバー、マイクロサービス、およびアプリケーション環境とプラットフォーム（コンテナー、オーケストレーター、サービスメッシュなど）の自動検出に適用されます。 ADTDは、バイトコードツールキット（BCI）および/または分散アプリケーショントレースを使用して、アプリケーションのHTTP / Sトランザクションの動作を監視することにより、これらの要素間の関係を決定します。

分析： APMソリューションは、AI / MLを使用してドメイン指向のAIOpsを提供する必要があり、ここではまとめて分析と呼びます（ガートナーのAIOpsソリューション2019–2020を参照）。これを行うには、APMソリューションは、トポロジーのコンテキストでAPMで利用可能なデータに基づいて、イベント相関、異常検出、根本原因分析（RCA）アルゴリズムを使用する必要があります。このソリューションは、サードパーティのソースからイベントを受信する機能も提供します。この基準は、この要件がAIOpsと呼ばれていたAPMソリューションの昨年のマジッククアドラントの進化です。 AIOps自体は、APMソリューションよりも幅広いアプローチをカバーできます。

ガートナーのマジッククアドラントは、リーダー、チャレンジャー、ストラテジスト、ニッチプレーヤーの4つのセクターに分かれています。各ベンダーは、その長所と短所、市場シェア、ユーザーレビュー、およびその他の指標に基づいて、象限に配置されます。今回ガートナーが含めた15のベンダー：Broadcom（CA Technologies）、Cisco（AppDynamics）、Datadog、Dynatrace、Instana、IBM、ManageEngine、Micro Focus、Microsoft、New Relic、Oracle、Riverbed、SolarWinds、Splunk（SignalFX）、 Tingyun。
