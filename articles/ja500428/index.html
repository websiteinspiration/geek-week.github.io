パンダが大量のデータを処理するのを助ける方法は？

パンダライブラリは、探索的データ分析に最適なツールの1つです。しかし、これはパンダがあらゆる問題を解決するのに適した普遍的なツールであることを意味しません。特に、大量のデータの処理について説明します。私はたまたま、非常に長い時間を費やし、パンダが多くのファイルを読み取るか、それらを処理するのを待って、それらに含まれる情報に基づいて、興味のあるいくつかの指標に基づいて計算しました。実際、パンダは並列データ処理メカニズムをサポートしていません。その結果、このパッケージは最新のマルチコアプロセッサの機能を十分に活用できません。パンダの大きなデータセットはゆっくりと処理されます。



最近、私はビッグデータの処理に役立つものを見つけることに着手しました。探していたものを見つけることに成功し、見つかったツールをデータ処理パイプラインに埋め込みました。大量のデータを処理するために使用します。たとえば、10ギガバイトのデータを含むファイルを読み取り、それらをフィルタリングして集計する場合などです。このような問題をなんとか解決したら、パンダに適した小さなCSVファイルに取得したものを保存し、パンダを使用して取得したデータの操作を開始します。

これは、実験できるこの資料の例を含むJupyterノートブックです。

ダスク


大量のデータを処理するために使用するツールはDaskライブラリです。並列データ処理をサポートしているため、既存のツールの作業を高速化できます。これには、numpy、pandas、sklearnが含まれます。Daskは無料のオープンソースプロジェクトです。Python APIとデータ構造を使用しているため、Daskを既存のプロジェクトに簡単に統合できます。Daskについて簡単に説明すると、このライブラリーは通常の問題の解決を単純化し、非常に複雑な問題の解決を可能にするものと言えます。

パンダと薄暮の比較


Daskの機能についてここで説明できます。このライブラリーには多くの興味深いものがあるためですが、代わりに、1つの実用的な例を検討します。仕事の過程で、私は通常、大きなファイルのセットに遭遇します。これらのデータは、分析する必要があります。私の典型的なタスクの1つを実行して、10のファイルを作成してみましょう。各ファイルには100,000レコードが含まれています。このような各ファイルのサイズは196 MBです。

from sklearn.datasets import make_classification
import pandas as pd
for i in range(1, 11):
    print('Generating trainset %d' % i)
    x, y = make_classification(n_samples=100_000, n_features=100)
    df = pd.DataFrame(data=x)
    df['y'] = y
    df.to_csv('trainset_%d.csv' % i, index=False)

次に、パンダを使用してこれらのファイルを読み取り、それらの読み取りに必要な時間を測定します。pandasには組み込みのサポートがないglobため、ファイルをループで読み取る必要があります。

%%time
import glob
df_list = []
for filename in glob.glob('trainset_*.csv'):
    df_ = pd.read_csv(filename)
    df_list.append(df_)
df = pd.concat(df_list)
df.shape

パンダがこれらのファイルを読み取るのに16秒かかりました：

CPU times: user 14.6 s, sys: 1.29 s, total: 15.9 s
Wall time: 16 s

Daskについて話す場合、このライブラリーを使用すると、メモリーに収まらないファイルを処理できるようになります。これは、それらをブロックに分割し、タスクチェーンをコンパイルすることによって行われます。Daskがこれらのファイルを読み取るのに必要な時間を測定してみましょう。

import dask.dataframe as dd
%%time
df = dd.read_csv('trainset_*.csv')
CPU times: user 154 ms, sys: 58.6 ms, total: 212 ms
Wall time: 212 ms

ダスクは154ミリ秒かかりました！これはどのようにして可能ですか？実際、これは不可能です。Daskは遅延タスクパラダイムを実装しています。計算は、結果が必要な場合にのみ実行されます。Daskにタスクの実行を最適化する機能を提供する実行グラフについて説明します。実験を繰り返します。read_csvDask の関数には、を使用するためのサポートが組み込まれていることに注意してくださいglob。

%%time
df = dd.read_csv('trainset_*.csv').compute()
CPU times: user 39.5 s, sys: 5.3 s, total: 44.8 s
Wall time: 8.21 s

関数を使用すると、computeDaskは結果を返すように強制されます。その結果に対して、ファイルを実際に読み取る必要があります。その結果、Daskはパンダの2倍の速度でファイルを読み取ります。

Daskを使用すると、Pythonプロジェクトにスケーリングツールを装備できると言えます。

軽薄短所


おそらく、次のような考えがあるかもしれません。「Daskライブラリが非常に優れている場合-パンダの代わりにそれを使用しないのはなぜですか？」しかし、それほど単純ではありません。Daskに移植されているのは一部のパンダ関数のみです。実際、特定のタスクは並列化が困難です。たとえば、データを並べ替えたり、並べ替えられていない列にインデックスを割り当てたりします。Daskは、データの分析と処理のすべてのタスクを完全に解決するツールではありません。このライブラリは、メモリに完全に収まらないデータセットを操作する場合にのみ使用することをお勧めします。Daskライブラリはパンダに基づいているので、パンダでゆっくり動作するものはすべてDaskでは遅くなります。前に述べたように、Daskはデータ処理パイプラインに埋め込むことができる便利なツールですが、このツールは他のライブラリを置き換えるものではありません。

Daskをインストールする


Daskをインストールするには、次のコマンドを使用できます。

python -m pip install "dask[complete]"
