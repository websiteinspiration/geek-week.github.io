Zimbra Collaboration Suite开源版中的电子邮件归档

对于大型企业来说，归档具有将来查看功能的邮件是一项重要功能。它可用于处理各种投诉，进行调查以及在许多其他情况下使用。如果不道德的用户使用其服务进行非法操作，此功能对于SaaS提供商保护自己也很有用。

特别是出于这些目的，创建了Zimbra存档和发现插件，该插件可让您在每个邮箱中存档传出和传入的信件，甚至保存在草稿邮件中的信件。但是，这一决定并非没有缺点。首先，它仅与付费的Zimbra Collaboration Suite网络版一起使用，其次，它仅在Web客户端的框架内工作，并且在使用台式机或移动邮件客户端时，它不会存档任何内容。在这方面，我们将告诉您如何在免费的ZImbra Collaboration Suite开放源代码版中实现传入和传出邮件的归档。另外，它将存档从任何电子邮件客户端发送的信件。


存档邮件是通过内置的Postfix BCC功能实现的。它的工作方式如下：系统管理员为邮箱设置存档邮件地址，输入某些设置，然后将每个传入和传出的信件都复制到存档邮件中，您以后可以在其中找到所需的信件。我们建议为邮件存档创建一个单独的域。将来，这将大大简化存档邮箱的管理。

存档外发电子邮件




让我们配置外发信件的存档。例如，使用manager@company.ru帐户并为其创建一个存档邮箱manager@archive.company.ru。为了将外发邮件存档，您需要对Postfix设置进行一些更改。为此，请打开文件/opt/zimbra/postfix/conf/main.cf，并在其末尾添加sender_bcc_maps = lmdb行：/ opt / zimbra / postfix / conf / sender_bcc。之后，您需要创建文件/ opt / zimbra / postfix / conf / sender_bcc，并将您计划存档的邮箱添加到其中，以及将存档的信件发送到的邮箱。可以将多个邮箱归档为一个。这样做如下：
manager@company.ru manager@archive.company.ru
sales@company.ru manager@archive.company.ru
support@company.ru support@archive.company.ru



添加所有邮箱后，您要做的就是运行postmap / opt / zimbra / postfix / conf / sender_bcc命令，并使用postfix reload命令重新加载Postfix 。如我们的示例所示，在重新启动后，来自帐户manager@company.ru和sales@company.ru的所有传出信件将发送到同一邮箱manager@archive.company.ru，并且将从帐户support@company.ru的传出信件进行存档在邮箱support@archive.company.ru中

存档传入的电子邮件


现在设置传入电子邮件的自动存档。为此，您可以使用相同的Postfix BCC。与存档外发邮件的情况一样，您需要打开文件/opt/zimbra/postfix/conf/main.cf并在该行中添加以下行：receive_bcc_maps = lmdb：/ opt / zimbra / postfix / conf /收件人_bcc。之后，您需要创建文件/ opt / zimbra / postfix / conf / receiver_bcc，并以以前的格式向其中添加必要的电子邮件地址。



添加框后，您需要运行postmap / opt / zimbra / postfix / conf / receiver_bcc命令，并使用postfix reload命令重新启动Postfix。。现在，来自帐户manager@company.ru和sales@company.ru的所有来信都将存档在邮箱manager@archive.company.ru中，并将来自帐户support@company.ru的来信复制到邮箱support@archive.company中.ru。


过滤传入消息的示例

应该注意的是，每次在列表/ opt / zimbra / postfix / conf / sender_bcc和/ opt / zimbra / postfix / conf / receiver_bcc中添加或删除电子邮件地址时，都必须再次运行postmap命令显示修改后的列表，然后重新启动Postfix。我们还建议您根据发件人和收件人的名称使用Zimbra OSE邮件过滤器，以便将传入和传出的邮件分类到文件夹中，以后您将更容易找到所需的邮件。


筛选外发邮件的示例

要在已创建的邮件归档中搜索字母，以后可以使用内置的Zimbra OSE搜索。还应注意，电子邮件在存档中的存储时间比在帐户中的存储时间长得多，这意味着它们需要设置增加的配额以及更长的保留策略。如果您的存档邮箱位于单独的域中，则这样做会容易得多。
