Kelilingi digit pengguna

Pekerjaan jarak jauh dengan kami akan tetap untuk waktu yang lama dan di luar batas pandemi yang berkecamuk saat ini. 74% dari 317 perusahaan yang disurvei oleh Gartner akan terus menggunakan telecommuting. Alat TI untuk organisasinya akan dicari secara aktif di masa depan. Memperkenalkan Citrix Workspace Environment Manager, Elemen Penting untuk Menciptakan Workspace Digital. Pada artikel ini, kami akan mempertimbangkan arsitektur dan fitur utama produk.



Arsitektur Solusi


Citrix WEM memiliki solusi arsitektur klien / server klasik.


WEM agent \ WEM agent - bagian klien dari perangkat lunak Citrix WEM. Itu diinstal pada workstation (virtual atau fisik, pengguna tunggal (VDI) atau multi-pengguna (server terminal)) untuk mengontrol lingkungan pengguna.

Layanan Infrastruktur WEM \ layanan infrastruktur - bagian server yang menyediakan layanan untuk agen WEM.

MS SQL Server - server DBMS yang diperlukan untuk melayani basis data WEM tempat informasi konfigurasi Citrix WEM disimpan.

Konsol administrasi WEM - konsol manajemen lingkungan WEM.

Mari kita lakukan sedikit penyesuaian dalam deskripsi komponen layanan Infrastruktur WEM di situs web Citrix (lihat tangkapan layar):


Situs ini secara keliru menyatakan bahwa layanan Infrastruktur WEM diinstal pada server terminal. Ini tidak benar. Pada server terminal, agen WEM diinstal untuk mengelola lingkungan pengguna. Selain itu, tidak mungkin untuk menginstal agnet WEM dan server WEM pada server yang sama. Agar server WEM berfungsi, peran layanan terminal tidak diperlukan. Komponen ini adalah infrastruktur dan, seperti layanan apa pun, diinginkan untuk menempatkannya di server khusus yang terpisah. Satu server WEM dengan karakteristik 4 vCPU, 8 GB RAM mampu melayani hingga 3000 pengguna. Untuk memastikan toleransi kesalahan, instal setidaknya dua server WEM di lingkungan Anda.

Fitur utama


Salah satu tugas administrator TI adalah organisasi ruang kerja pengguna. Alat kerja yang digunakan oleh karyawan harus ada dan disiapkan dengan tepat. Administrator perlu memberikan akses ke aplikasi (letakkan pintasan di desktop dan menu Mulai, konfigurasikan asosiasi file), berikan akses ke sumber daya informasi (hubungkan drive jaringan), hubungkan printer jaringan, kemampuan menyimpan dokumen pengguna secara terpusat, memberikan kemampuan kepada pengguna untuk mengkonfigurasi lingkungan dan yang paling penting - untuk memastikan pengalaman pengguna yang nyaman. Di sisi lain, administrator bertanggung jawab atas keamanan data tergantung pada kondisi tertentu di mana pengguna bekerja dan kondisi untuk mengamati kebijakan lisensi perangkat lunak.Citrix WEM dirancang untuk mengatasi masalah ini.

Jadi, fitur utama Citrix WEM:

  • manajemen lingkungan pengguna
  • manajemen sumber daya komputasi
  • batasi akses ke aplikasi
  • manajemen workstation fisik

Manajemen ruang kerja pengguna


Kemampuan apa yang disediakan Citrix WEM untuk mengelola opsi pembuatan desktop pengguna? Gambar di bawah ini menunjukkan konsol manajemen Citrix Workspace Environment Manager. Bagian Tindakan mencantumkan tindakan yang dapat digunakan administrator untuk mengatur lingkungan kerja. Yaitu, buat pintasan aplikasi di desktop dan di menu Start (termasuk untuk aplikasi yang diterbitkan melalui integrasi dengan Citrix Storefront, serta kemampuan untuk menetapkan tombol pintas untuk peluncuran cepat aplikasi dan koordinat untuk lokasi pintasan di lokasi tertentu di layar), hubungkan printer jaringan dan drive jaringan, membuat disk virtual, mengelola kunci registri, membuat variabel lingkungan, mengkonfigurasi pemetaan port COM dan LPT dalam satu sesi, memodifikasi file INI,jalankan program \ skrip (selama Logon, LogOff, Sambungkan kembali operasi), kelola file dan folder (buat \ salin \ hapus file dan folder), buat Pengguna DSN untuk mengonfigurasi koneksi basis data pada server SQL, konfigurasikan asosiasi file.


Untuk memudahkan administrasi, "tindakan" yang dibuat dapat digabungkan ke dalam Grup Tindakan.

Untuk menerapkan tindakan yang dibuat, tindakan tersebut harus ditetapkan ke grup keamanan atau KM pengguna domain pada tab Tugas. Gambar di bawah ini menunjukkan bagian Penilaian dan proses menetapkan "tindakan" yang dibuat. Anda dapat menetapkan Grup Tindakan dengan semua "tindakan" yang termasuk di dalamnya atau menambahkan set "tindakan" yang diperlukan secara terpisah dengan menyeretnya dari kolom kiri Tersedia di sebelah kanan yang Ditugaskan.


Saat menetapkan "tindakan", Anda perlu memilih filter, berdasarkan analisis yang sistem akan menentukan kebutuhan untuk penggunaan "tindakan" tertentu. Secara default, sistem membuat satu filter Selalu Benar. Saat menggunakannya, semua "tindakan" yang ditugaskan selalu diterapkan. Untuk manajemen yang lebih fleksibel, administrator membuat filter mereka di bagian Filter. Filter terdiri dari dua bagian: "Ketentuan" dan "Aturan". Gambar menunjukkan dua bagian, di sebelah kiri jendela dengan menciptakan kondisi, dan di sebelah kanan aturan berisi kondisi yang dipilih untuk menerapkan "tindakan" yang diinginkan.


Sejumlah besar "kondisi" tersedia di konsol - hanya sebagian yang ditampilkan pada gambar. Selain memeriksa keanggotaan situs atau grup Direktori Aktif, atribut AD individu, filter tersedia untuk memeriksa nama PC atau alamat IP, mencocokkan versi OS, memeriksa tanggal dan waktu, jenis sumber daya yang diterbitkan, dll.

Selain mengelola pengaturan lingkungan pengguna melalui penggunaan Action, konsol Citrix WEM memiliki bagian besar lainnya. Bagian ini disebut Kebijakan dan Profil. Ini memberikan kemungkinan pengaturan tambahan. Bagian ini terdiri dari tiga bagian: Pengaturan Lingkungan, Pengaturan Microsoft USV, dan Pengaturan Manajemen Profil Citrix.

Pengaturan Lingkungan mencakup sejumlah besar pengaturan, secara tematis dikelompokkan ke dalam beberapa tab. Nama mereka berbicara sendiri. Mari kita lihat peluang apa yang tersedia bagi administrator untuk membentuk lingkungan pengguna.

Lewati pengaturan dari bagian Pengaturan USV Microsoft. Dalam blok ini, komponen Microsoft standar - Pengalihan Folder dan Profil Roaming - diatur serupa dengan pengaturan dalam kebijakan grup.


Dan subbagian terakhir adalah Pengaturan Manajemen Profil Citrix. Dia bertanggung jawab untuk mengatur Citrix UPM untuk mengelola profil pengguna. Ada lebih banyak pengaturan di bagian ini daripada di dua gabungan sebelumnya. Pengaturan dikelompokkan menjadi beberapa bagian dan disusun sebagai tab dan sesuai dengan pengaturan Citrix UPM di konsol Citrix Studio. Di bawah ini adalah gambar dengan tab Pengaturan Manajemen Profil Citrix Utama dan daftar tab yang tersedia ditambahkan untuk presentasi umum.


Manajemen terpusat pengaturan lingkungan kerja pengguna bukanlah hal utama yang ditawarkan WEM. Banyak fungsi di atas dapat dicapai dengan menggunakan kebijakan grup standar. Keuntungan WEM adalah bagaimana pengaturan ini diterapkan. Kebijakan standar digunakan saat menghubungkan pengguna secara berurutan. Dan hanya setelah menerapkan semua kebijakan, proses masuk berakhir dan desktop menjadi tersedia untuk pengguna. Semakin banyak pengaturan diaktifkan melalui kebijakan grup, semakin banyak waktu yang diperlukan untuk menerapkannya. Ini sangat memperpanjang waktu login. Tidak seperti kebijakan grup, agen WEM mengubah urutan pemrosesan dan menerapkan pengaturan pada beberapa utas secara paralel dan asinkron. Waktu login pengguna berkurang secara signifikan.

Keuntungan menerapkan pengaturan melalui Citrix WEM dibandingkan kebijakan grup ditunjukkan dalam video.


Manajemen Hitung


Pertimbangkan aspek lain dalam menggunakan Citrix WEM, yaitu kemungkinan mengoptimalkan sistem dalam hal Manajemen Sumber Daya. Pengaturan terletak di bagian Optimasi Sistem dan dibagi menjadi beberapa blok:

  • Manajemen CPU
  • Manajemen memori
  • I \ O Manajemen
  • Logoff cepat
  • Pengoptimal Citrix

Manajemen CPU berisi parameter untuk mengelola sumber daya prosesor: membatasi konsumsi sumber daya secara umum, memproses lonjakan / lonjakan konsumsi sumber daya prosesor, dan memprioritaskan sumber daya di tingkat aplikasi. Pengaturan utama terletak pada tab Pengaturan Manajer CPU dan disajikan pada gambar di bawah ini.


Secara umum, tujuan parameter jelas dari namanya. Fitur yang menarik adalah kemampuan untuk mengelola sumber daya prosesor, yang disebut Citrix optimasi pintar - CPU \ Intelligent optimasi CPU. Nama yang keras menyembunyikan fungsi yang sederhana, tetapi cukup efektif. Ketika aplikasi dimulai, proses tersebut diberikan prioritas penggunaan CPU maksimum. Ini menyediakan peluncuran cepat aplikasi dan secara keseluruhan meningkatkan tingkat kenyamanan saat bekerja dengan sistem. Semua "keajaiban" dalam video.


Ada beberapa pengaturan di bagian Memory Management dan I \ O Management, tetapi esensinya sangat sederhana: mengelola memori dan proses input-output saat bekerja dengan disk. Manajemen memori diaktifkan secara default dan berlaku untuk semua proses. Ketika aplikasi dimulai, prosesnya mencadangkan sebagian RAM untuk pekerjaan mereka. Sebagai aturan, cadangan ini lebih dari yang diperlukan saat ini - cadangan dibuat "untuk pertumbuhan" untuk memastikan operasi aplikasi yang cepat. Optimalisasi memori terdiri dari membebaskan memori dari proses-proses yang berada dalam keadaan tidak aktif (Status Idles) untuk waktu yang ditentukan. Ini dicapai dengan mentransfer halaman memori yang tidak digunakan ke file halaman. Optimalisasi disk dicapai dengan memprioritaskan aplikasi. Gambar di bawah ini menunjukkan opsi yang tersedia untuk digunakan.


Pertimbangkan bagian Logoff Cepat. Pada akhir sesi kerja yang biasa, pengguna melihat bagaimana aplikasi ditutup, profil disalin, dll. Saat menggunakan opsi Logoff Cepat, agen WEM memonitor panggilan untuk mengakhiri sesi kerja (Log Off) dan memutus sesi pengguna - menempatkannya dalam kondisi Putus. Untuk pengguna, sesi berakhir secara instan. Dan sistem secara teratur menyelesaikan semua proses kerja di latar belakang. Opsi Fast Logoff diaktifkan oleh satu "daw", tetapi pengecualian dapat diberikan.


Dan akhirnya bagian, Pengoptimal Citrix. Administrator Citrix sangat menyadari alat untuk mengoptimalkan gambar emas mereka - Citrix Optimizer. Alat ini diintegrasikan ke dalam Citrix WEM 2003. Gambar berikut menunjukkan daftar templat yang tersedia.


Administrator dapat mengedit template saat ini, membuat yang baru, melihat parameter yang ditetapkan dalam template. Jendela pengaturan ditunjukkan pada gambar di bawah ini.


Pembatasan Akses Aplikasi


WEM Citrix dapat digunakan untuk membatasi peluncuran / instalasi aplikasi, skrip, memuat DLL - perpustakaan. Pengaturan ini dikumpulkan di bagian Keamanan. Gambar di bawah mencantumkan aturan yang diusulkan sistem untuk dibuat secara default untuk setiap subbagian dan secara default, semuanya diizinkan. Administrator dapat mengganti pengaturan ini atau membuat yang baru. Untuk setiap aturan, satu dari dua tindakan tersedia - Izinkan \ Tolak. Dalam tanda kurung dengan nama ayat, jumlah aturan yang dibuat di dalamnya ditunjukkan. Bagian Keamanan Aplikasi tidak memiliki pengaturan sendiri, itu menampilkan semua aturan dari sub-bagiannya. Selain membuat aturan, administrator dapat mengimpor aturan AppLocker yang ada, jika digunakan dalam organisasi, dan mengelola pengaturan lingkungan secara terpusat dari satu konsol.


Di bagian Manajemen proses, Anda dapat membuat daftar hitam dan putih untuk membatasi peluncuran aplikasi dengan nama file yang dapat dieksekusi.


Manajemen Workstation Fisik


Kami tertarik pada pengaturan sebelumnya untuk mengelola sumber daya dan parameter untuk menciptakan lingkungan kerja bagi pengguna dalam hal bekerja dengan VDI dan server terminal. Apa yang ditawarkan Citrix untuk mengelola workstation fisik dari mana Anda terhubung? Kemampuan WEM yang disorot di atas dapat diterapkan ke workstation fisik. Selain itu, alat ini memungkinkan Anda untuk "mengubah" PC menjadi "thin client". Konversi ini terjadi ketika pengguna memblokir akses ke desktop dan menggunakan fitur bawaan Windows secara umum. Alih-alih desktop, shell grafis dari agen WEM diluncurkan (menggunakan agen WEM yang sama seperti pada VDI \ RDSH), di antarmuka di mana sumber daya Citrix yang dipublikasikan ditampilkan. Citrix memiliki perangkat lunak Citrix DesktopLock, yang juga memungkinkan Anda untuk mengubah PC Anda menjadi "TC",tetapi fitur Citrix WEM lebih luas. Di bawah ini adalah gambar dari parameter utama yang dapat digunakan untuk mengontrol komputer fisik.




Di bawah ini adalah tangkapan layar bagaimana tempat kerja terlihat setelah mengubahnya menjadi "thin client". Menu drop-down Options mencantumkan elemen-elemen yang dengannya pengguna dapat menyesuaikan lingkungan sesuai keinginannya. Beberapa di antaranya atau semua elemen dapat dihapus dari antarmuka.


Administrator dapat secara terpusat menambahkan tautan ke sumber daya web perusahaan ke bagian "Situs", dan aplikasi yang diinstal pada PC fisik, yang diperlukan bagi pengguna, ke bagian "Alat". Misalnya, penting untuk menambahkan tautan ke portal dukungan pengguna di "Situs", tempat karyawan dapat membuat permintaan jika ada masalah saat menyambung ke VDI.


Solusi semacam itu tidak dapat disebut "thin client" lengkap: kemampuannya terbatas dibandingkan dengan versi komersial dari solusi serupa. Tetapi itu cukup untuk menyederhanakan dan menyatukan antarmuka bekerja dengan sistem, untuk membatasi akses pengguna ke pengaturan sistem PC dan menggunakan armada PC yang menua sebagai alternatif sementara untuk solusi khusus.

***

Jadi, kami merangkum ulasan Citrix WEM. Produk "bisa":

  • kelola pengaturan lingkungan pengguna
  • mengelola sumber daya: prosesor, memori, disk
  • memberikan entri / keluar cepat dari Sistem (LogOn \ LogOff) dan peluncuran aplikasi
  • batasi penggunaan aplikasi
  • mengubah PC menjadi thin client

Tentu saja, Anda secara skeptis dapat mengevaluasi demo menggunakan WEM. Seperti yang ditunjukkan oleh pengalaman kami, untuk sebagian besar perusahaan yang tidak menggunakan WEM, waktu masuk rata-rata adalah 50-60 detik, yang tidak jauh berbeda dari waktu di video. Dengan WEM, waktu masuk dapat dikurangi secara signifikan. Selain itu, dengan menggunakan aturan sederhana untuk mengelola sumber daya perusahaan, Anda dapat meningkatkan kepadatan pengguna di server atau memberikan kualitas kerja terbaik dengan sistem untuk pengguna saat ini.

Citrix WEM sangat sesuai dengan konsep "workstation digital", tersedia untuk semua pengguna Aplikasi dan Desktop Citrix Virtual mulai dengan edisi Advanced dan dengan dukungan yang ada untuk Layanan Keberhasilan Pelanggan.
