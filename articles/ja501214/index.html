パンダのGroupby集約



集計は、データ分析で最も一般的な操作の1つです。さまざまなテクノロジーにより、関心のあるフィールド（列、属性）を効果的にグループ化および集約する方法が数多く提供されます。この記事では、パンダでの集計の実装について説明します。
私の専門分野では、Pythonをほとんど使用していませんが、特にデータの操作に関しては、この言語の長所と能力についてよく耳にします。したがって、ここではT-SQLを使用した並列操作を描画し、コード例をいくつか示します。データとしては、おそらく最も人気のあるデータセットであるIrises Fisherを使用します。

最初に頭に浮かぶのは、アイリスパラメータの最大値、最小値、または平均値を取得し、この植物の種ごとにグループ化することです。パンダを使用するpythonでは、次のようになります。 

import pandas as pd

df = pd.read_csv('iris.csv', delimiter = ',')
print(df.groupby('variety').max()[['sepalLength']].to_markdown())

結果：

| バラエティ| sepal.length |
|：----------- | ---------------：|
| せとさ| 5.8 |
| バーシカラー| 7 |
| バージニカ| 7.9 |

とか、ぐらい：

import pandas as pd
df = pd.read_csv('iris.csv', delimiter = ',')

print(df.groupby('variety').sepalLength.agg(
    maxSepalLength  = 'max',
    minSepalLength  = 'min',
    ).to_markdown())

結果：

| バラエティ| maxSepalLength | minSepalLength |
|：----------- | -----------------：| ----------------- ：|
| せとさ| 5.8 | 4.3 |
| バーシカラー| 7 | 4.9 |
| バージニカ| 7.9 | 4.9 |

またはラムダ式を使用する：

import pandas as pd
df = pd.read_csv('iris.csv', delimiter = ',')

print(df.groupby('variety').sepalLength.agg([
    lambda x: x.max(), 
    lambda x: x.min()
    ]).to_markdown())

結果：

| バラエティ| <lambda_0> | <lambda_1> |
|：----------- | -------------：| -------------：|
| せとさ| 5.8 | 4.3 |
| バーシカラー| 7 | 4.9 |
| バージニカ| 7.9 | 4.9 |

DataFrameインスタンス関数
to_markdown()
通常の（コンソール）形式でテーブル（DataFrame）を表示できます。

T-SQLでは、この操作は次のようになります。

select i.Variety, max(i.SepalLength) as maxSepalLength
    from Iris i
        group by i.Variety

結果：

Setosa 5.8 Versicolor
7.0
Virginica 7.9

しかし、今度は、アイリスのすべてのパラメーターの最大値と最小値（平均が好きな場合）の両方を取得したいとします。当然、植物のタイプごとに、T-SQLコードがここに生成されています。

select
	i.Variety 
	,max(i.SepalLength) as maxSepalLength 
	,min(i.SepalLength) as minSepalLength
	,max(i.SepalWidth) as maxSepalWidth
	,min(i.SepalWidth) as minSepalWidth
	,max(i.PetalLength) as maxPetalLength
	,min(i.PetalLength) as mibPetalLength
	,max(i.PetalWidth) as maxPetalWidth
	,min(i.PetalWidth) as minPetalWidth
from Iris i
	group by i.Variety

結果：

Setosa 5.8 4.3 4.4 2.3 1.9 1.0 0.6 0.1
Versicolor 7.0 4.9 3.4 2.0 5.1 3.0 1.8 1.0
Virginica 7.9 4.9 3.8 2.2 6.9 4.5 2.5 1.4

パンダでは、グループ集約の可能性は2019年7月18日のバージョン0.25.0でのみ出現しました（以前行われていたこと） ？）そしていくつかのバリエーションがあります、それらを考慮してください：

import pandas as pd
df = pd.read_csv('iris.csv', delimiter = ',')

df.groupby('variety').agg(
    maxSepalLength = pd.NamedAgg(column = 'sepalLength', aggfunc = 'max'),
    minSepalLength = pd.NamedAgg(column = 'sepalLength', aggfunc = 'min'),
    maxSepalWidth = pd.NamedAgg(column = 'sepalWidth', aggfunc = 'max'),
    minSepalWidth = pd.NamedAgg(column = 'sepalWidth', aggfunc = 'min'),
    maxPetalLength = pd.NamedAgg(column = 'petalLength', aggfunc = 'max'),
    minPetalLength = pd.NamedAgg(column = 'petalLength', aggfunc = 'min'),
    maxPetalWidth = pd.NamedAgg(column = 'petalWidth', aggfunc = 'max'),
    minPetalWidth = pd.NamedAgg(column = 'petalWidth', aggfunc = 'min'),
    )

結果：

Setosa 5.8 4.3 4.4 2.3 1.9 1.0 0.6 0.1
Versicolor 7.0 4.9 3.4 2.0 5.1 3.0 1.8 1.0
Virginica 7.9 4.9 3.8 2.2 6.9 4.5 2.5 1.4

関数
DataFrame.agg(self, func, axis=0, *args, **kwargs)

特定の軸で複数の操作を集約できます。パラメータとして、関数は** kwargs（名前付き引数、詳細についてはhabrの記事を参照）を受け取ります。これは、操作が実行される列と単一引用符で囲まれた集計関数の名前です。録音はかなりボリュームがあります。進め。

ラムダ式を使用した同じソリューションは、はるかに簡潔でシンプルに見えます。

import pandas as pd
df = pd.read_csv('iris.csv', delimiter = ',')

df.groupby('variety').agg([
    lambda x: x.max(),
    lambda x: x.min()
    ])

結果：

Setosa 5.8 4.3 4.4 2.3 1.9 1.0 0.6 0.1
Versicolor 7.0 4.9 3.4 2.0 5.1 3.0 1.8 1.0
Virginica 7.9 4.9 3.8 2.2 6.9 4.5 2.5 1.4

Pythonでは、他の言語と比較して同じタイプの問題を解決するときに、あまり書かれていないという話をよく聞きます。ここでは、T-SQLと比較してこれに同意できますが、SQLやT-SQLなどの言語ツールの表現の明快さと順序は完全に失われています（個人的な意見）。
