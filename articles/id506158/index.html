useSWR adalah perpustakaan React favorit saya yang baru

Terjemahan artikel disiapkan sebelum dimulainya kursus "Pengembang React.js . "


Beberapa bulan terakhir saya telah mengerjakan sebuah aplikasi di NextJS. Setiap minggu menjadi semakin banyak. Aplikasi ini menggunakan aksioma untuk panggilan API dan tidak dinyatakan berikutnya untuk manajemen negara. Ada banyak panggilan API, tetapi kami tidak ingin pengguna melihat banyak layar pemuatan. Oleh karena itu, kami menyimpan hasil panggilan aksioma di repositori yang tidak dinyatakan .

Namun, kami mengalami masalah. Kubah sendiri menjadi semakin kompleks. Halaman terkadang membutuhkan beberapa panggilan API, dan pada gilirannya bergantung pada hasil panggilan API lainnya. Hari-hari berubah menjadi minggu dan minggu menjadi bulan, sementara lemari besi kami yang tidak disebutkan namanya menjadi semakin rumit. Kami menemukan kesalahan aneh, karena logika cache buatan kami berjuang untuk menangani kasus ekstrim yang tidak terduga.

Kemudian kami berpikir bahwa jalan itu seharusnya lebih sederhana.
Dan ini dia. Dan itu bahkan dikembangkan oleh Vercel , pencipta NextJS .

Kenali SWR


Nama SWR berasal dari basi-sementara-memvalidasi , metode caching yang sekarang mendapatkan popularitas dalam pengembangan frontend. Ini memungkinkan Anda untuk mengunduh konten yang di-cache dengan segera, dan segera memperbaruinya untuk menangani konten baru di masa mendatang. Dalam kasus kami, kami mendapat kompromi sempurna antara kinerja dan pengalaman pengguna.

Cara menggunakan useSWR


useSWR adalah perpustakaan hook React yang dikembangkan oleh Vercel . Ini memungkinkan Anda untuk mengambil data dari API atau sumber eksternal lainnya, menyimpannya di cache, dan kemudian merendernya.

Mari kita mulai dengan melihat contoh komponen Bereaksi yang menerima daftar TODO dari server JSON dan merendernya.

import React from "react";
import "./App.css";
const todosEndpoint = "http://localhost:3001/todos";
const TodoApp = () => {
  const [todos, setTodos] = React.useState([]);
  React.useEffect(() => {
    const getData = async () => {
      const response = await fetch(todosEndpoint);
      const data = await response.json();
      setTodos(data);
    };
     getData();
    }, []);
  return (
    <div className="App">
      {todos.map((todo) => (
        <div key={todo.id}>{todo.title}</div>
      ))}
    </div>
  );
};
export default TodoApp;

Sekarang mari kita lihat komponen yang sama, tetapi ditulis ulang menggunakan useSWR .

import React from "react";
import useSWR from "swr";
const todosEndpoint = "http://localhost:3001/todos";
const getData = async () => {
  const response = await fetch(todosEndpoint);
  return await response.json();
};
const TodoApp = () => {
  const { data: todos } = useSWR(todosEndpoint, getData);
  
return (
    <div>
      {todos && todos.map(todo => (
        <div key={todo.id}>{todo.title}</div>
      )}
    </div>
  );
};
export default TodoApp;

Seperti yang Anda lihat, ini sangat mirip dengan implementasi sebelumnya.

Dalam contoh ini, kami menggunakan useSWR (key, fetcher, options) untuk mendapatkan daftar TODO kami. Kunci dalam useSWR diperlukan untuk caching. Dalam kasus kami, kami menggunakan todosEndpoint . Sebagai fetcher, kami melewati fungsi asinkron yang mengambil daftar TODO.

Perlu dicatat bahwa useSWR tidak tahu bagaimana Anda mengambil data. Anda dapat menggunakan strategi penambangan data asinkron apa pun yang Anda suka. Anda dapat menggunakan fetch , aksioma atau bahkan GraphQL . Saat fungsi Anda mengembalikan data secara tidak sinkron,useSWR akan senang.

Panggilan useSWR mengembalikan parameter berikut:

  • data : data kunci diperoleh dengan menggunakan fetcher (tidak ditentukan, jika belum dimuat);
  • error : error yang disebabkan oleh fetcher ;
  • isValidating : boolean yang mengatakan apakah permintaan atau validasi ulang sedang berlangsung;
  • mutate (data?, shouldRevalidate) : berfungsi untuk memodifikasi data yang di-cache.

Contoh yang lebih kompleks


Mari kita lihat contoh yang lebih kompleks yang menggunakan lebih banyak parameter yang dikembalikan oleh useSWR . Dalam contoh ini, kami akan mengambil satu elemen dari daftar TODO dan memberikan kotak centang untuk status lengkap tugas kami dari daftar. Ketika kotak centang diklik, kami mengirim permintaan PUT untuk memperbarui TODO, dan kemudian kami memanggil mutasi untuk memperbarui cache useSWR .

import React from "react";
import useSWR from "swr";
import "./App.css";
const todosEndpoint = "http://localhost:3001/todos";
const getTodo = async (id) => {
  const response = await fetch(`${todosEndpoint}/${id}`);
  return await response.json();
};
const updateTodo = async (id, todo) => {
  const response = await fetch(`${todosEndpoint}/${id}`, {
    method: "PUT",
    headers: {
      "Content-type": "application/json; charset=UTF-8",
    },
    body: JSON.stringify(todo),
  });
  return await response.json();
};
const TodoApp = () => {
  const todoId = 1;
  const key = `${todosEndpoint}/${todoId}`;
  const { data: todo, mutate } = useSWR(key, () =>
    getTodo(todoId)
  );
  const toggleCompleted = async () => {
    const newTodo = {
      ...todo,
      completed: !todo.completed,
    };
    await updateTodo(todoId, newTodo);
    mutate(newTodo);
  };
  if (!todo) {
    return <div>Loading...</div>;
  }
  return (
    <div>
      <p>{todo.title}</p>
      <div>
        <input
          type="checkbox"
          id="completed"
          name="completed"
          value="completed"
          checked={todo.completed}
          onChange={toggleCompleted}
        />
        <label htmlFor="completed">Completed</label>
      </div>
    </div>
  );
};

Menggunakan mutate adalah cara yang bagus untuk meningkatkan kinerja aplikasi web Anda yang terlihat. Kami dapat mengubah data secara lokal dan mengubah tampilan, tanpa menunggu hingga sumber data jarak jauh diperbarui. useSWR bahkan akan memvalidasi ulang dan menggantinya dengan data terbaru di latar belakang.

Mengapa saya perlu perpustakaan untuk ini?


Anda dapat mengajukan pertanyaan ini. Mungkin Anda sudah memiliki mekanisme manajemen negara di aplikasi Anda, dan Anda tidak melihat titik menggunakan perpustakaan pihak ketiga untuk sesuatu yang sederhana seperti ekstraksi data dan caching.

Dan saya akan menjawab Anda bahwa data caching tidak mudah. Dan ketika aplikasi Anda tumbuh, itu menjadi semakin sulit. Anda akan menemukan kasus-kasus ekstrem, dan memprosesnya, Anda akan mendapatkan repositori yang kompleks dan penyedia yang sulit dipahami dan sulit untuk dipelihara. Bug akan keluar entah dari mana.
"Dalam ilmu komputer, hanya ada dua masalah kompleks: pembatalan cache dan penemuan nama" - Phil Carleton

Alih-alih menerapkan solusi penambangan data Anda sendiri, mengapa tidak mengandalkan solusi terbukti yang dikembangkan oleh salah satu perusahaan paling dihormati di ekosistem Bereaksi?

Bagaimana useSWR membantu kami


Porting masalah ekstraksi data dalam aplikasi kami untuk menggunakan SWR telah memberikan banyak keuntungan.

1. Dia mengizinkan kami untuk menghapus kode

Tarik-permintaan yang menghapus kode tambahan, tetapi jangan menambahkan yang baru - favorit saya Semakin sedikit kode dalam aplikasi, semakin kecil kemungkinan kesalahan akan terjadi. Dan ini bagus. Seiring bertambahnya usia, saya mulai semakin menghargai kesederhanaan. useSWR memungkinkan kami untuk menghapus penyimpanan yang tidak disebutkan sepenuhnya, yang menyederhanakan aplikasi kami dan membuatnya lebih mudah untuk dipahami.

2. Dia menyederhanakan koneksi pengembang baru ke proyek

useSWRseperti kebanyakan proyek open source, ia menawarkan dokumentasi yang sangat baik. Menerapkan solusi kami sendiri akan berarti bahwa kami perlu menulis dokumentasi kami sendiri dan mengajari pengembang baru dalam proyek bagaimana menangani ekstraksi data. Sekarang kami menggunakan useSWR , kami tidak perlu melakukan ini.

3. Dia menyederhanakan hal-hal sulit.

Aplikasi kita, seperti kebanyakan aplikasi lain, mengandung banyak panggilan API. Beberapa panggilan ini bergantung pada panggilan API lainnya. Dengan useSWR, Anda mudah menulis kait untuk permintaan yang bergantung.

4. Ini telah meningkatkan kinerja yang terlihat.

Aplikasi ini terlihat lebih cepat. Pengguna memberi nilai dan memberikan umpan balik yang baik.

5. Ini memperbarui data yang usang

useSWR memperbarui data usang dalam fokus. Ini berarti bahwa pengguna selalu memiliki versi terbaru dari data, tidak termasuk waktu pengunduhan.

Kesimpulan


useSWR berdampak besar pada aplikasi kita. Perpustakaan ini telah menyederhanakan kode kami dan meningkatkan pengalaman pengguna. Saya bahkan tidak bisa membayangkan perpustakaan lain yang akan mudah diimplementasikan dan membawa banyak manfaat seperti yang dimiliki useSWR . Untuk mempelajari lebih lanjut tentang useSWR , Anda dapat mengunjungi situs atau repositori di GitHub .

Perpustakaan alternatif


useSWR bukan satu-satunya perpustakaan yang baik untuk mengambil data. Meskipun kinerjanya baik dalam kasus kami, mungkin perpustakaan lain lebih cocok untuk Anda untuk ekstraksi data.

Reaksi-permintaan


react-query adalah pustaka kait untuk mengambil, menyimpan, dan memperbarui data asinkron di Bereaksi. Solusi populer ini, yang ditulis oleh Tanner Linsi , sangat mirip dalam cakupannya dengan menggunakan SWR .

Apollo


The Apollo klien adalah GraphQL klien yang membutuhkan perawatan dari mengambil dan caching data Anda, serta memperbarui antarmuka pengguna. Jika Anda menggunakan GraphQL , maka Apollo akan menjadi solusi yang bagus.
