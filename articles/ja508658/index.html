ITで完璧な仕事を見つける方法は？心理的アプローチ。P1：地獄からのマネージャーの認識ジョブ分析の分析





さまざまな国のさまざまな企業の完全に異なる職種の300を超える求人応募を提出した後、仕事の説明が実際に職場であなたを待っているものをどのように反映するかについての明確なパターンに気付き始めました。たとえば、候補者の要件のリストの「ストレス耐性」の項目の下など、背後にあるものとHRが実際に何を意味するのかを簡単かつ即座に理解できるとは限りません。この記事では、完璧な仕事を見つけるための私の経験と、上記の300のアプリケーションからの何百ものインタビューの後に得たスキルを共有したいと思います。 「理想的な」仕事の基準に基づいて本当に自分に合った仕事を見つけるために、雇用前に空席の説明を分析する方法を示したいと思います。ここでは、就職活動を別の科学として考えます。例を示します。

この一連の記事は、卒業証書を受け取ったばかりのグリーン卒業生と、複数のチームを置き換えた経験豊富なシニア男性の両方を対象としています。また、人事部長にとって、外部からの活動の概要として、エラーを分析して作業するためにも役立ちます。

それでは始めましょう。仕事の説明を読むときに最初に覚えておくべきことは、人々がそれらを書いたということです。オイルオイル？しかし、仕事の説明を書くタスクが誰に与えられたのかを正確に理解することがはるかに重要です。HRJune / Timlid / HR Juneではありませんが、開発が何であるかについてはまだほとんど理解していません。このような分析は、その後の雇用段階で非常に役立ちます。私はN番目のポジションに信じられないほどのスキルを必要とする非常に多くの求人に出くわしました。そのため、このポジションの説明を書いた人は、候補者に期待することについて少しでも理解していないという結論につながりました（小さな補足、しばしばインタビューの後の段階では、HRジョーンズよりも比較的有能な従業員、たとえばチームリーダーは、この役割の候補者に何を期待しているのか正確に理解していません。しかし、このシリーズの記事の以下の部分でさらに詳しく説明します）。



: ,





必要なスキルの不明確で広範な説明は、あなたが「主導」され、専門分野の範囲外のタスクに対処する必要があることを示す非常に優れた予備的な指標です。たとえば、最初に私が話した300の空席の中には、開発者のバックエンドの位置に多くの空席があり、要件はフルスタックによってかなり引っ張られました。マネージャーとチームは全体として、タスクの明確な説明と、誰が何をしているのかを理解している必要があります。バックエンドデベロッパーを雇ってフロントに開発を強制することはできません。また、バックエンドの給与についてMLモデルをトレーニングすることもできません。たとえば、会社が小さく、チームにそのような人員の使用を正当化するほど多くの人がいない場合、これは仕事の説明にあるはずです。会社と同じように、この場合、探しているのはバックエンド開発者ではなく、すべての取引のジャック、または単にフルスタック開発者であることを明確にする必要があります。しかし、完璧なものは何もないので、あなたはそれが何であるかに我慢しなければならないので、仕事を探すときはそのような細部に注意を払います。



候補者の不明確な要件：彼ら自身が何を望んでいるかを知らない



候補者の要件に類似のテクノロジーのセットが混在している場合、あいまいに記述された空席の別のバリエーションがあります。以下は、私が2年前の就職活動中に見つけた300名の求人の中から見つけて保存した例です。



要件：

Python 3.6

、Django、Django-REST、Django-rq、Flask、Tornado、CherryPi。

MySQL / Postgres

docker / docker-composeの知識;

http、httpsを理解する。REST、RESTful、SOAP;

React Reduxの知識、AngularJS;


これらのフレームワークは比較的単純ですが、これらすべてのテクノロジーを同時に理解している開発者のバックエンドポジションの候補を想像するのは困難です。このポジションには1つのフレームワークの知識でおそらく十分であると考えられますが、やはり採用担当者は必要以上に多く、あなたが正確に何をするかという主な本質は完全には明らかではありません。実際、要件では、直接作業するテクノロジースタックを正確に指定する必要があります。要件は明確かつ正確である必要があります。これは、あなたを雇用する人々が、このポジションに誰を求めているか、そしてどのようにそれを開発するかについて明確な考えを持っていることを示しています。



基本要件：ストレス耐性;





最後に、候補リストで私のお気に入りの項目は「ストレス耐性」です。これは、就職活動サイトで見つけることができる最も論争の的であるアイテムです。彼はどういう意味ですか？私はまだそのような条項の下で候補者にどのような個人的品質の雇用者が求めているか完全には理解していませんが、確かに知っていることの1つは、説明にそのような条項がある職を得ないことにはいくつかの非常に良い理由があります。第一に、自発的傾向に苦しんでいない正常で健康な人は、仕事でストレスを感じたくないでしょう。すべての神話に反して、仕事はストレスにならないようにしてください！仕事はいいはず！第二に、これはチームで何が起こっているか、そしてあなたがどのような作業をしなければならないかもしれないかの別の指標です。無限の理由があります採用担当者がストレスに強い候補者を探している理由。ほとんどの場合、プロジェクトマネージャーは能力がなく、スキルやタスク数によってチケットを正しく割り当てることができません。それら。マネージャーは、人的資源の管理方法やチームの管理方法を知らない場合があります。もう1つの理由は、おそらく会社が従業員の数を節約していることです。チームには十分な手がなく、それらを処理できる人数よりも多くのタスクがあり、すべてが絶えず「やけど」しています。また、面接の給与を減らしただけの従業員を雇うことで、従業員の質を下げることもできます。いずれにせよ、マネージャーの存在の要点は、通常の開発者は、彼らが投げかけられたタスクのためにストレスを感じる必要がないということです。チームリーダーと部門マネージャーの主な仕事はそれを確実にすることですチームには、開発するプロジェクトに対して十分な数と質の従業員がいるようにします。
