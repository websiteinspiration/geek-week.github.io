iviオンライン映画ブログには、Hydra推奨システムのアーキテクチャに関する豊富な記事があります。ただし、推奨事項は外部APIだけでなく、「内部」に存在し、非常に複雑なビジネスロジックを実装するアルゴリズムでもあります。

この記事では、コンテンツの「コールドスタート」の問題について説明します。最近カタログに追加され、ユーザーからフィードバックを得ることができなかったコンテンツの推奨方法を知りたい場合は、catへようこそ。


この記事には、Kerasを使用した再現可能なPythonコードのサンプルが含まれます。

import implicit
import numpy as np
from scipy.sparse import load_npz

#   user-item
user_item_views_coo = load_npz('/srv/data/user_item_interactions.npz')

als_params = { 'factors': 40, 'regularization': 0.1, 'num_threads': 3, 'iterations': 5}

print('  ALS-')
als_model = implicit.als.AlternatingLeastSquares(**als_params)
als_model.fit(user_item_views_coo.T)
als_factors = als_model.item_factors
print('ALS- ,   ', als_factors.shape)

als_factors_filename = '/srv/data/fair_als_factors.npy'

最近サービスで出たコンテンツを推薦する方法は？そのようなコンテンツにはビューがありません（またはビューがほとんどありません）。つまり、機械学習の魔法の箱の枯渇では、そのようなコンテンツの機能がなく、ユーザーの推奨に表示されません。ちなみに、ユーザーとコンテンツの相互作用に基づいて取得する機能は、コラボレーティブと呼ばれます。

したがって、コールドスタートの問題が発生します。ユーザーからのフィードバックがないコンテンツをどのように推奨できますか？たとえば、新しいコンテンツをランダムな配信に混ぜて、「オーガニック」ビューが収集されるまで待つことができます。

別のオプションは、「冷たい」コンテンツの特徴を予測できるモデルを構築することです。

私たちは2番目の方法をとることに決めました、そしてそれは私たちが来た場所です

コールドスタートの問題を解決するために、たとえば、新しいコンテンツについて事前にわかっているコンテンツ機能が役立ちます。

  • 人：監督、脚本家、キャスト
  • コンテンツのジャンル：アクション、コメディーなど
  • カテゴリ：長編映画、漫画、ドキュメンタリー
  • 編集タグ

編集タグは、有限（通常は数百）の一連の特性の形式でコンテンツを簡潔に説明したものです。以下は、ビーバーゾンビコンテンツタグのセットです。


最初の近似では、コールドスタートの問題を次のように解決しました。

  • 「冷たい」コンテンツごとに、最も似ているコンテンツをタグでワイプします
  • 同様のコンテンツの共同機能を利用する
  • 冷たいコンテンツの共同機能は、その「熱い」隣人の機能の平均です


Pythonでは、次のようになります。
for row in new_items_neighbors:
    neighbors_als_indices = row.neighbors_ids[:self.cold_start_neighbors_count]
    neighbors_average_factors = item_factors[neighbors_als_indices].mean(axis=0)

この方法は何とか機能しましたが、2つの欠点がありました。

  • 拡張性が弱い：モデルにポスターの類似性などを追加するのが難しい
  • タグで類似したコンテンツのALS機能の類似性を保証する人はいません。これがないと、平均化の使用は奇妙に見えます


私たちはあなたがもはやそのように生きることができないことに気づき、より透明で拡張可能なモデルを思いつきました。

コールドスタートモデルのリファクタリング



ヒューリスティック（平均化など）を使用してALSコンテンツの特徴を計算する代わりに、たとえば、エディタータグによって、協調的なコンテンツの特徴を予測するニューラルネットワークをトレーニングできます。類似のモデルがすでにHabrでここで公開されており、Yandexの前に音楽サービスのSpotifyが類似のモデルについて話しました


モデルのプロトタイプコードはiviリポジトリで利用できます。コールドスタートのニューラルネットワークは次のとおりです。

cold_start_model.py
def _get_model(self, dims: List[int]) -> Sequential:
    model = Sequential()
    model.add(
        Dense(
            units=dims[1], activation='linear', input_dim=dims[0],
            kernel_initializer=WeightInitializer.custom_normal,
            bias_initializer='zeros'
        )
    )
    model.compile(
        loss=lambda y_true, y_pred: K.sum(K.square(y_pred - y_true), axis=-1),
        optimizer=optimizers.Adam(lr=self.learning_rate, decay=self.decay)
    )
    return model


この実験を実装する際にどのような困難に遭遇しましたか？

  • ネットワークのトレーニングは非常に困難であることがわかりました。機能はワンホットでエンコードされ、入力層のディメンションが大きいため、ネットワークのトレーニングが不十分です。私は慎重に機能を選択する必要があり、最終的にはカテゴリ、ジャンルのみを使用し、エディタータグからtf-idfを使用して最も「重要な」ものを選択しました
  • pipenvパッケージマネージャーを使用してKerasをインストールする際の問題：python、環境は使用できませんでした。Kerasが友達にならなかったサードパーティのmaxvolpyパッケージを終了する必要がありました。

実験結果



その結果、新しい機能は少し少なくなりました。これにより、開発者は2、3回のスプリントで約100時間を費やしました。これは、プロジェクトで本番環境でニューラルネットワークを使用した最初の経験です。今回は以下のように配信されました。

  • 記事を読んでプロトタイプ開発に60時間
  • プロトタイプをプロジェクトコードベースに統合するための30時間
  • 新しいモデルのデプロイ10時間-Kerasをpythonにドラッグするのは、特定の依存関係（maxvolpyなど）があるため、それほど簡単ではありませんでした

さらなる実験の余地がありました-ニューラルネットワークを使用すると、エディタータグだけでなく、写真、スクリプト、ユーザーコメント、ビデオシーケンスなどの他の機能についても学ぶことができます。
