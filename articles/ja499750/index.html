インターネット全体をスキャンする方法



IPv4アドレスの全範囲は40億のIPアドレスです。これは膨大な数のように見えますが、たとえば、世界中のすべてのWebサーバーまたは開いているすべてのSSHポートを見つけるために、IPv4インターネット全体を1つのTCPポートについて40分で完全にスキャンできます。この場合、1つのサーバーとギガビットチャネルで十分です。これは、たとえば、世界で使用されているテクノロジーに関する統計を収集したり、外部に公開されている脆弱なサービスの割合を推定したりする場合に、研究に役立ちます。Zmap

プログラム（nmapと混同しないでください）特別なアーキテクチャーにより、どのスキャナーよりもはるかに高速に広範囲のネットワークをスキャンできます。この記事では、zmapを使用して世界中のすべてのWebサーバーのリストをコンパイルする方法の例を見ていきます。HTTPポートが開いているホストのリストがあれば、よりインテリジェントなスキャナーを使用して、ターゲットの正確なリストを渡すことができます。

悪いnmapとは


小さなサブネットをスキャンするために、Nmapが伝統的に使用されています-多くのPentesterパンが中にある人気のあるオープンソースのマルチツール。狭い範囲や選択的なホストのスキャンに適しています。 Nmapには、lua（Nmap Script Engine）でカスタムスクリプトを作成するための独自のエンジンと、多くのクールな既製スクリプトがあります。しかし、nmapは、数百万または数十億ものアドレスなどの大規模ネットワークのスキャンにはあまり適していません。このような作業には数日から数週間かかります。実際のところ、nmapはオペレーティングシステムのネットワークサブシステムを使用し、リクエストごとに完全なソケットが開かれます。たとえば、ウェブサーバーにGETリクエストを行うなど、サービスとチャットする必要がある場合は、完全なTCP接続が必要です。ただし、これにより、高速SYNスキャンが必要な場合はパフォーマンスが大幅に低下します。ここでのタスクは、ポートが開いているかどうかを確認することだけです。

ZmapとMasscan


ZmapとMasscan非同期スキャナーの方が適しています。どちらもより高速に動作し、PF_RINGドライバー（パケットキャプチャを大幅に高速化するソケット）を使用し、アドレスをランダム化して致命的なDoSを回避します。Nmapとは異なり、それらはシステムTCP / IPを使用しません。最初はベアイーサネットフレームを生成し、2番目は自己作成のTCPスタックを使用します。

Masscanは、IPv4範囲全体を6分で通過できる最速のインターネットスキャナーと見なされていますが、並列に接続された少数の物理アダプターを使用する場合のみ、SYNパケットを含む送信トラフィックが一方から生成され、応答が他方に到達します。単一接続の条件では、変更されたZmapはほぼ1.5倍高速でした。もちろん、これらのレコードは10 Gbit / sチャネルのトップアイアンに設定されており、自分で繰り返すことは困難で費用がかかります。はるかに手頃なギガビット接続では、チャネルとCPUリソースをより効率的に使用できるため、ZmapはMasscanよりも高速に動作し、約45〜50分で完了できます。


ZmapとMasscanの簡素化されたワークフロー。発信トラフィックジェネレータと着信応答プロセッサは別々に動作します

技術的な詳細については説明しませんが、ZmapとMasscanは2つのエンティティを使用します。SYNリクエストを含む発信トラフィックジェネレータと、個別の着信応答プロセッサです。ここでのパフォーマンスは、チャネル幅とネットワークインターフェイスのパフォーマンス、つまりPPS制限（パケット/秒）によってのみ制限されます。したがって、ソースIPを着信パケットハンドラーのアドレスに置き換えることが可能な場合、スキャンプロセスは複数のインターフェイスまたは複数の物理ホストに分割できます。

スキャン準備


スキャナーはシステム全体、特にネットワークインターフェースをロードし、利用可能なチャネル全体を利用することに注意してください。ホストに警告せずにスキャンを開始すると、DDoSのようになり、ネットワークからすぐに切断されます。また、インターネット全体をスキャンすると応答が発生するため、準備を整えておく必要があります。つまり、ホストは「あなたは私たちをスキャンしている」と文句を言う一連の自動手紙を受け取ります。

NAT,


zmapが何百万ものTCPリクエストを同時に生成することを理解することが重要です。NATを備えたルーター、接続追跡を備えたファイアウォール、DDoS保護、または接続を追跡しようとするステートフルファイアウォールを備えたその他のシステムがインターネットとスキャンサーバーの間に設置されている場合、それほど多くの接続をダイジェストできないため、ルーターは故障します。したがって、たとえば家庭のWiFiルーターの背後など、NAT中はzmapを実行できません。

Zmapを試す


Zmapをインストール（指示）

簡単なものでテストします。habr.comから隣接するWebサーバーを確認します。

$ zmap -p 80 178.248.237.0/24 -B 100M -o habr.txt

オプション：

  • -p 80 -TCPポート80、つまりHTTPサーバーのみを探します
  • 178.248.237.0/24は、アドレスのターゲット範囲です。指定しない場合、インターネット全体がスキャンされます。
  • -B 100M -zmapが使用する最大チャネル幅。このオプションが指定されていない場合、使用可能なチャネル全体が破棄されます。
  • -o habr.txt-結果をテキストファイルに書き込みます。リクエストに応答したアドレス、つまりHTTPサーバーが存在するアドレスを取得します


zmapを使用して254のアドレスをスキャンすると数秒かかりました

Zmap出力フォーマット


デフォルトでは、zmapは見つかったアドレスをテキストファイルに追加し、改行で区切ります。ただし、結果をjsonおよびxml形式で書き込むこともできます。オプション--output-fieldsを使用すると、出力に追加される追加のフィールドを指定できます。

たとえば、jsonでより高度な出力形式を試し、応答要求とTTLパケットが到着したポートを示します。

$ zmap -p80  --output-module=json --output-fields=ttl,sport,dport,saddr 178.248.237.68 -o habr.com.json

#  
$ cat habr.com.json
{ "ttl": 58, "sport": 80, "dport": 51309, "saddr": "178.248.237.68" }
