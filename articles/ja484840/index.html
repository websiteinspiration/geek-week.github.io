GOTO Return

今では、GOTO演算子の使用は単に悪いだけでなく、ひどい習慣であることを誰もが理解しています。その使用に関する議論は20世紀の80年代に終わり、ほとんどの現代のプログラミング言語から除外されました。しかし、本当の悪にふさわしいので、彼はなんとかして自分自身を偽装し、例外を装って21世紀に復活しました。


一方、例外は、現代のプログラミング言語ではかなり単純な概念です。一方、それらはしばしば誤って使用されます。シンプルでよく知られているルールがあります-例外はダメージの取り扱いのみです。また、「内訳」の概念を自由に解釈することは、GOTOの使用に関するすべての問題を引き起こします。


理論的な例


内訳と否定的なビジネスシナリオの違いは、非常に単純なユースケースでログインウィンドウに明確に表示されます。


  1. ユーザーはログイン/パスワードを入力します。
  2. ユーザーは「ログイン」ボタンをクリックします。
  3. クライアントアプリケーションはサーバーにリクエストを送信します。
  4. サーバーはユーザー名/パスワードを正常にチェックします（対応するペアの存在を成功と見なします）。
  5. サーバーは、認証が成功したという情報と遷移ページへのリンクをクライアントに送信します。
  6. クライアントは指定されたページに移動します。

そして一つの否定的な拡張：


4.1。サーバーは対応するログイン/パスワードのペアを見つけられず、これについてクライアントに通知を送信します。


シナリオ4.1は「問題」であるため、例外を使用して実装する必要があることは、かなりよくある間違いです。これは実際にはそうではありません。ログインとパスワードの不一致は、スクリプトのビジネスロジックによって提供される標準のユーザーエクスペリエンスの一部です。私たちのビジネス顧客はこの発展を期待しています。したがって、これは内訳ではなく、ここで例外を使用することはできません。


内訳は、クライアントとノース間の接続の切断、DBMSへのアクセス不能、データベース内の不正なスキームです。さらに、アプリケーションを壊し、ユーザーのビジネスロジックとは何の関係もない100万以上の理由。

private Properties readPropertiesFromFile(String filepath, boolean throwIfNotExists){
    Properties loadedProperties = new Properties();
    System.out.println("Try loading workspace properties" + filepath);

    InputStream is = null;
    InputStreamReader isr = null;
    try{
        int loadingTryLeft = 3;
        String relativePath = "";
        while (loadingTryLeft > 0){
            try{
                File file = new File(relativePath + filepath);
                is = new FileInputStream(file);
                isr = new InputStreamReader( is, "UTF-8");
                loadedProperties.load(isr);
                loadingTryLeft = 0;
            } catch( FileNotFoundException e) {             
                loadingTryLeft -= 1;
                if (loadingTryLeft > 0)
                    relativePath += "../";
                else
                    throw e;
            } finally {
                if (is != null)
                    is.close();
                if (isr != null)
                    isr.close();
            }
        }
        System.out.println("Found file " + filepath);
    } catch( FileNotFoundException e) {
        System.out.println("File not found " + filepath);
        if (throwIfNotExists)
            throw new RuntimeException("Can`t load workspace properties." + filepath + " not found", e );
    }catch (IOException e){
        throw new RuntimeException("Can`t read " + filepath, e);
    }
    return loadedProperties;
}

private Properties readPropertiesFromFile(String filepath) {
    System.out.println("Try loading workspace properties" + filepath);

    try {
        int loadingTryLeft = 3;
        String relativePath = "";

        while (loadingTryLeft > 0) {
            File file = new File(relativePath + filepath);
            if (file.canRead()) {
                return read(file);
            } else {
                relativePath += "../";
                loadingTryLeft -= 1;
            }
        }
        System.out.println("file not found");
    } catch (IOException e) {
        throw new RuntimeException("Can`t read " + filepath, e);
    }

    return new Properties();
}

private Properties read(File file) throws IOException {
    try (InputStream is = new FileInputStream(file);
            InputStreamReader isr = new InputStreamReader(is, "UTF-8")) {
        Properties loadedProperties = new Properties();
        loadedProperties.load(isr);
        System.out.println("Found file " + file.getAbsolutePath());
        return loadedProperties;
    }
}
