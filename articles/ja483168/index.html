武器に！


さて、すでに主な落とし穴とそれらに対処する方法について話しましたが、今はビジネスに取り掛かるときです。

ディストリビューションをダウンロードしたフォルダーに移動します。pytorch-CycleGAN-and-pix2pixと呼ばれていたことを思い出させ

てください。ダミーのための指示があるので、必要なすべてのパラメーターはコマンドラインから設定できるため、配布コードには触れません。

この分布では、デフォルトのトレーニングは200年代を超えており、100番目のエポック後の学習率の線形減衰があります。
私の8ギガバイトのカードがマスターできる最大解像度は400x400でした。完全なトレーニングには約33時間かかったので、小さなライフハックを使用します。最初に、モデルを128x128の画像でトレーニングし、次に256x256でトレーニングします。最後の段階でのみ、ゴージャスな400x400を彼女に見せます。
第1ステージは100時代、第2および第3-それぞれ50時代続き、トレーニング時間をほぼ半分に短縮できます。

実際に示されているように、このアプローチの結果は、最大解像度ですぐにトレーニングを行う場合よりも悪くはなく、場合によってはそれよりも優れています。
おそらく十分な理論です。練習に移りましょう。

コマンドプロンプトで、次のように入力します。

第一段階 


python train.py  --dataroot {dataset root folder} --name {model name} --model cycle_gan --crop_size 128 --load_size 180 --init_type kaiming --netG resnet_9blocks --no_dropout --batch_size 4

中括弧を独自の値に置き換えることを忘れないでください。

いくつかのパラメーターをさらに詳しく分析します。

--batch_size {number}

このパラメータは、サイクルごとに処理される画像の数に関与し、速度にプラスの影響を与え、モデルの大食いにマイナスの影響を与えます。

各ステージで、可能な最大の--batch_sizeを選択します。これにより、GPUメモリの不足によるエラーは発生しません。

-- dataroot {dataset root folder}

-データセットのあるフォルダ。その中には、前の記事で説明したように、trainA、trainB、testA、testBのフォルダーがあります。

--name {model name}

-プロジェクトの名前。名前は任意ですが、モデルアーキテクチャ、最大解像度、データセットの名前を名前に含めることをお勧めします。
例：「resnet9_128to400_comics8」
したがって、異なるパラメーターとデータを使用した実験を区別できます。

トレーニングの最初の段階は、100世紀で停止できます。

ブラウザから進行状況を確認できます：localhost：8097 /
（またはコンソールに表示される別のリンク）

ステージ2


python train.py  --dataroot {dataset root folder} --name {model name} --model cycle_gan --crop_size 256 --load_size 290 --init_type kaiming --netG resnet_9blocks --no_dropout --batch_size 2 --epoch 100 --epoch_count 0 --continue_train

最初の段階で研究を終えた時代を示すことが重要です。
「--Epoch 100」は、100番目の時代のチェックポイントからモデルを読み込むことを意味します。
「-epoch_count 0」は、学習速度を最大にして、ゼロからトレーニングを開始することを意味します。

トレーニングの第2ステージは、50年代に停止できます。

ステージ3


python train.py  --dataroot {dataset root folder} --name {model name} --model cycle_gan --crop_size 400 --load_size 430 --init_type kaiming --netG resnet_9blocks --no_dropout --batch_size 1 --epoch 50 --epoch_count 0 --continue_train


トレーニングの第3ステージは50代で停止できますが、これは好みの問題です。最後まで仕上げ、好きな中間結果を選べます。200年代の結果は150年代よりも悪い場合があることを覚えておくことが重要です。

さあ外に出て、あなたが何をしたか見てください


トレーニング中、モデルと中間結果は
/ pytorch-CycleGAN-and-pix2pix / checkpoints / {model name} フォルダに保存されます。

モデルを確認するには、コマンドラインで次のように入力します。

python test.py --dataroot {dataset root folder} --name {model name} --model cycle_gan --netG resnet_9blocks --crop_size 512 --load_size 580 --epoch {epoch name}

チェックポイントのテストデータセットで結果を確認できます。{エポック名}と指定するだけです。{エポック名}が指定されていない場合、最後のチェックポイントが取られます。

結果はフォルダに保存されます：
/pytorch-CycleGAN-and-pix2pix/results/{model name}/test_{epoch name}
どのデータセットのどのモデルがどの結果をもたらすかを混乱させないために
、小さな日記をつけ始めます。ベースラインデータと学習結果を記録します。

たとえば、コマンドラインと、あなたが到達した時代です。このデータは、トレーニングパラメーターとトレーニングしたデータセットを保存するのに十分です。
