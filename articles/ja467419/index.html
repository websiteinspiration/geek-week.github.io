Jakarta EE 8最終リリースがリリースされました

Eclipse Foundationは、Jakarta EE 8プラットフォームとWebプロファイルの新しい仕様をリリースしました。
また、他の互換性のある実装をテストするためのJakarta EEアプリケーションサーバーとテストスイート（TCK）も導入しました。


Jakarta EE 8は2019年9月10日にリリースされました。

ジャカルタEE仕様はWebサイトで入手できます。

ジャカルタアノテーションからジャカルタサーバーページまでさまざまです。

Eclipse仕様とともに、Jakarta EE 8プラットフォームを実装するオープンソースアプリケーションサーバーであるEclipse Glassfish Server 5.1がリリースされました。

さまざまなJavaサーバーベンダーのJakarta EE 8サーバーの開発が期待されています。したがって、IBM Open LibertyサーバーはすでにJakarta EE 8互換として認定されています。

Eclipse は、2017年に開発を引き継ぐことに同意しました。新しいJakarta EE 9を導入する代わりに、Eclipseはスムーズな移行を目指しており、Java EEをJakarta EEに徐々に置き換えています。

ジャカルタEE開発


Jakarta EEは（Eclipseによれば）クラウドサービスやマイクロサービスなどの最新のソリューションを構築するために使用されます。

Jakarta EEをアップデートする将来の機能には、Jakarta EEの基礎となるJava SEアップデートが含まれます。

現在のバージョンはJava SE 8で、次の候補はJava SE 11です（これは、長期サポートサイクルを備えたSE 8の後の次のビルドです）。

また、Kubernetes、Dockerなどのクラウドテクノロジーとの連携も改善されます。

Jakarta RESTfull Webサービスなど、個別の仕様も改善されることが期待されています。

さらに、Eclipse Foundationは、javax名前空間をジャカルタに変更するプロセスを定義する必要があります。これにはプログラムの再コンパイルが必要になる場合がありますが、Eclipseは移行の重大度を下げるために下位互換性に取り組んでいます。

javax名前空間は20年間下位互換性がありますが、Oracleが所有しています。

Glassfish 5.1をダウンロードする場所


Eclipse GlassFish 5.1 はプロジェクトのWebサイトからダウンロードできます。
TCKもこのサイトでアクセスできます。
