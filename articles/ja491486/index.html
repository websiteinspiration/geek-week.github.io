ラップトップの蓋を閉じて、スリープなしで画面をロックするときのカスタムスクリプト

みなさん、こんにちは。私は自宅のラップトップでLubuntu 18.04を使用しています。ある晴れた日、私はラップトップのふたを閉じるときにPower Managerが提供するアクションに満足できないと判断しました。ラップトップのふたを閉じたとき、画面をブロックし、しばらくしてラップトップを休止状態にしました。これを行うために、私はスクリプトを作成し、それをあなたと共有することを急いでいます。

私は2つの問題に遭遇しました。

最初は、休止状態がボックスからのloubuntで機能しないことです;これを有効にするには、以下を実行する必要があります。

UUIDスワップを見つけます。これを行うには、次のことを行う必要があります。

grep swap /etc/fstab

私の場合、出力は次のようになります。

# swap was on /dev/mmcblk0p2 during installation
UUID=aebf757e-14c0-410a-b042-3d9a6044a987 none            swap    sw              0       0

次に、UUIDをカーネル初期化パラメーターに追加する必要があります。これを行うには、ファイル/ etc / default / grubに「GRUB_CMDLINE_LINUX_DEFAULT」という行を追加します。resume = UUID =％your UUID％

...
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash resume=UUID=aebf757e-14c0-410a-b042-3d9a6044a987"
...

そして、コマンドを実行します：

sudo update-grub

これで休止状態が機能するはずです。確認のために次のことができます。

sudo systemctl hibernate

2番目の問題は、ラップトップをスリープ状態にせずに、ユーザーの画面をrootからロックする方法でした。dbus-sendを使用して解決しました。コマンド自体は以下のスクリプトにあります。他のオプションを知っている人がいたら、コメントに書き込んでください。

さあ、スクリプトを書き始めましょう。

Power Managerで最初に行う必要があるのは、ふたを閉じるときのアクションとして[ディスプレイをオフにする]を選択して、スクリプトと競合しないようにすることです。



次に、次の内容で/ etc / acpi / events / laptop-lidファイルを作成します。

event=button/lid.*
action=/etc/acpi/laptop-lid.sh

そして、以下の内容でスクリプト/etc/acpi/laptop-lid.shを作成します：

#!/bin/bash

#set variables
# BUS   environ   lxsession
BUS=$(grep -z DBUS_SESSION_BUS_ADDRESS \
	/proc/$(pidof -s lxsession)/environ | \
	sed 's/DBUS_SESSION_BUS_ADDRESS=//g')
#     ,    
USER=$(grep -z USER /proc/$(pidof -s lxsession)/environ | sed 's/USER=//g')
#     
LID="/proc/acpi/button/lid/LID0/state"

#Check lid state (return 0 if closed)
check_lid () {
	grep -q closed $LID
}

#Lock screen without sleep
check_lid
if [ $? = 0 ]
then
	#TODO run command as root
	sudo -u $USER -E dbus-send --bus=$BUS \
				    --type=method_call \
				    --dest="org.freedesktop.ScreenSaver" \
				    "/org/freedesktop/ScreenSaver" \
				    org.freedesktop.ScreenSaver.Lock
fi

#Wait 10 minutes and hibernate if lid is closed
sleep 600
check_lid
if [ $? = 0 ]
then
	systemctl hibernate
fi

スクリプトを実行可能にします。

sudo chmod a+x /etc/acpi/laptop-lid.sh

そして、acpidデーモンを再起動して、変更を適用します。

sudo systemctl restart acpid.service

すべて準備ができています。

スクリプトのGnomeの場合、変更する必要があります。

  • lxsessin => gnome-session
  • org.freedesktop.ScreenSaver => org.gnome.ScreenSaver
