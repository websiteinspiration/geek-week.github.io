Apa Yang Harus Dikembangkan Pengembang JavaScript Tentang Curl





Selamat siang teman!



Saya mempersembahkan kepada Anda terjemahan artikel "Apa yang Harus Ketahui Pengembang JavaScript Tentang Curl" oleh Valery Karpov.



Curl adalah alat baris perintah populer yang sering digunakan untuk mengirim permintaan HTTP. Curl mendukung sejumlah besar protokol, namun, sebagai pengembang Node.js, Anda kemungkinan besar akan menggunakannya untuk mengirim permintaan http ke API ISTIRAHAT.



Dokumentasi curl adalah daftar 383 flag baris perintah, jadi sangat sulit untuk menemukan apa yang Anda cari. Dalam artikel ini, saya ingin berbagi dengan Anda beberapa templat yang sering saya gunakan. Sebagai contoh, layanan httpbin.org akan digunakan .

Unggah berkas



Wget adalah alat paling umum untuk mengunduh file melalui baris perintah. Ini disertakan dengan sebagian besar distribusi Linux. Namun, di OSX tidak.



Perintahnya wget url



mirip dengan perintah curl -OL url



. Opsi -



adalah opsi --remote-name



yang memberi tahu curl untuk menyimpan badan respons dalam file lokal. Opsi -L



memberi tahu curl untuk mengikuti arahan ulang.

Mengirimkan Permintaan Resmi



Header otorisasi digunakan untuk memasukkan data otorisasi dalam permintaan saat mengakses RESTful API. Untuk menambahkan data yang ditentukan, Anda harus menggunakan bendera -H



. Misalnya, jika kunci antarmuka Anda (kunci API) my-secret-token



, Anda dapat memasukkannya dalam permintaan http sebagai berikut: 



$ curl -H "Authorization: my-secret-token" https://httpbin.org/get
{
  "args": {}, 
  "headers": {
    "Accept": "*/*", 
    "Authorization": "my-secret-token", 
    "Host": "httpbin.org", 
    "User-Agent": "curl/7.58.0", 
    "X-Amzn-Trace-Id": "Root=1-5ee8e1a5-a3aa30e0765a7980b04ca4a0"
  }, 
  "origin": "69.84.111.39", 
  "url": "https://httpbin.org/get"
}


Perhatikan bahwa ia httpbin.org



mengembalikan header permintaan http di badan respons di properti headers



.



Curl juga mendukung otorisasi default melalui bendera -u



. Dalam contoh berikut, kami mengirim permintaan dengan nama pengguna user



dan kata sandi pass

$ curl -i -u "user:pass" https://httpbin.org/basic-auth/user/pass

Kesimpulan



Singkatnya, berikut adalah opsi ikal yang menurut saya paling berguna:



  • -X



    mendefinisikan metode permintaan, mis. curl -X POST url



  • -d



    mendefinisikan badan permintaan sebagai string dalam permintaan PUT dan POST. Gunakan @



    untuk mengekstrak data dari file
  • -H



    mendefinisikan header permintaan, misalnya curl -H "Authorization: my-secret-token" url



  • -u



    data otentikasi untuk otorisasi standar
  • -O



    menyimpan tubuh permintaan ke file
  • -i



    menunjukkan jawaban lengkap termasuk tajuk


Curl adalah alat yang berguna untuk berinteraksi dengan API melalui baris perintah, terlepas dari apakah itu API pihak ketiga atau API yang Anda kembangkan. Untuk pengujian cepat, curl lebih baik daripada Axios di Node.js atau penyetelan kueri di Postman jika Anda terbiasa dengan sintaks mereka.
