Unity + DOTSでの数十万のパーティクルの物理シミュレーション

ある時点で、あるサイトでWorld Wide Webの広大な範囲をさまよっているときに、インタラクティブなJS要素を発見しました。これは、マウスカーソルが近づくと粒子が飛び散る画像です。Unityでそのような動作のシミュレーションをプログラミングし、エンジンから絞り出せるパフォーマンスを確認したいという要望がありました。実装プロセスの間に受け取った技術的ソリューションと私の観察を共有します。



結果は次のとおりです。

アニメーション


それを実現するには、次の問題を解決する必要があります。

  • 素粒子物理学について説明する;
  • 各フレームでの位置計算の最適な速度を確保します。
  • 画面上に多数のパーティクルをすばやく描画する方法を選択します。

物理


望ましい粒子の振る舞いを説明するには、3つの原則のみを定式化する必要があります。開始点までの距離を短くしたいという願望、マウスカーソルから「飛び去る」という願望、そして動きの減衰です。正確な物理的相互作用と式は必要ありません。粒子が意図的に振る舞うに従って動く、一般的な原理のみが必要です。簡単にするために、質量はすべての式から除外されています-単一であると見なすことに同意します。

開始位置までの距離を短くする


粒子が元の位置に戻ろうとするためには、フックの法則が適しています。初期位置に向かう力は、その距離に線形に比例します。パーティクルは、最初の位置から2倍離れて飛んでいきました-2倍強く後方に「引っ張られ」、すべてが簡単です。

カーソルからの距離


興味深いことに、パーティクルは何らかの方法でカーソルと相互作用し、開始位置から「プッシュ」できる必要があります。私はこの相互作用の基礎として反対の符号で重力を取りました：粒子は、マウスカーソルの位置と粒子の現在の位置の間の距離の2乗に反比例する力によって反発され、カーソルから粒子へのベクトルに沿って方向付けられます。F=-C/r2どこ C -相互作用を調整する特定の定数。

減衰


上記の2つの式に制限すると、そのようなモデルのフレームワークではエネルギーを失う場所がないため、初期振幅を設定した後、粒子は永久に振動します。減衰は、媒体の粘性抵抗の力としてシミュレートします。これは、ストークスの法則に従って、粒子速度に線形に比例し、運動とは反対方向に向かいます。この場合、物理的な相互作用の絶対的な精度ではなく、原理にのみ関心があるため、この式の適用に課せられた制限に満足しています。

プログラミング動作


私たちのモデルでは、粒子は互いに独立しており、それらの動作はそれらの特性と共通の定数によってのみ決定されるため、更新された状態の計算は並列化に最適です。Unity DOTSに対する長年の関心を考慮して、定式化された問題を解決するためにこのテクノロジーを適用することにしました。

DOTSは、Unityエンジンの比較的最近の開発であり、高性能のマルチスレッドコードの作成に焦点を当てたテクノロジースタックです。Habréには、DOTSの紹介記事の翻訳があります。ECS、ジョブズシステムアーキテクチャパターン、バーストコンパイラを使用することで、DOTSは、高速マルチスレッドコードを記述し、足元でのショットの可能性を減らすことができます。

ECS のフレームワーク内でプログラムを作成する本質は、コードを、状態を記述するコンポーネント（Components）、コンポーネントの動作と相互作用を記述するシステム（Systems）、およびエンティティ -コンポーネントのセットを含むオブジェクトに分割することです。

コンポーネントを使用すると、すべてが非常に簡単です。システムに移動します。

//   
[UpdateBefore(typeof(MoveSystem))]
public class VelocityUpdateSystem : SystemBase
{
    protected override void OnUpdate()
    {
        var time = Time.DeltaTime;
        Entities.ForEach((ref VelocityData velocity, 
                in AccelerationData acceleration) =>
            {
                velocity.Value += time * acceleration.Value;
            })
            .ScheduleParallel();
    }
}

//   
public class MoveSystem : SystemBase
{
    protected override void OnUpdate()
    {
        var time = Time.DeltaTime;
        Entities.ForEach((ref Translation t, in VelocityData velocity) =>
            {
                t.Value.xy += time * velocity.Value;
            })
            .ScheduleParallel();
    }
}

動作を説明する最大のシステムは、次の派生式に従って動作する加速度更新システムです。

[UpdateBefore(typeof(VelocityUpdateSystem))]
public class AccelerationSystem : SystemBase
{
    protected override void OnUpdate()
    {
        float2 repulsorPos = Globals.Repulsor.Position;

        //   
        var attractionPower = Globals.SettingsHolder.SettingsModel.Attraction;
        //  
        var repulsionPower = Globals.SettingsHolder.SettingsModel.Repulsion;
        //  
        var dampingPower = Globals.SettingsHolder.SettingsModel.Damping;
        Entities.ForEach((
                ref AccelerationData acceleration, 
                in AttractorPosData attractorPos, 
                in Translation t,
                in VelocityData velocity) =>
            {
                //  
                var attraction = (attractorPos.Value - t.Value.xy) * attractionPower;

                var distSqr = math.distancesq(repulsorPos, t.Value.xy);
                //  
                var repulsion = -math.normalizesafe(repulsorPos - t.Value.xy) 
                    / distSqr * repulsionPower;

                //  
                var damping = -velocity.Value * dampingPower;

                acceleration.Value = attraction + repulsion + damping;
            })
            .ScheduleParallel();
    }
}

レンダリング


表示する必要のある潜在的に膨大な数のパーティクルにもかかわらず、すべてに対して、2つのポリゴンの同じ正方形メッシュと同じマテリアルを使用できるため、1つの描画呼び出しではなくても、1つの描画呼び出しでそれらすべてをレンダリングできます。しかし、」：すべての粒子は、一般的に、異なる色を持っています。そうでなければ、結果の画像は非常に退屈になります。

標準のUnityシェーダー「スプライト/デフォルト」は、GPUインスタンシングを使用して、スプライトとカラーが等しくないオブジェクトのレンダリングを最適化し、それらを1つの描画呼び出しに結合できますが、この場合、特定の各オブジェクトのテクスチャーとカラーへのリンクをスクリプトから設定する必要があります。 ECSからはアクセスできません。

ソリューションはGraphics.DrawMeshInstancedメソッドである可能性がありますを使用すると、同じGPUインスタンスを使用して、異なるマテリアルパラメータで1回の描画呼び出しで1回のメッシュを複数回描画できます。次のようになります。

[UpdateAfter(typeof(TrsMatrixCalculationSystem))]
public class SpriteSheetRenderer : SystemBase
{
    // DrawMeshInstanced     1023    
    private const int BatchCount = 1023;
    //   
    private readonly List<Matrix4x4> _matrixList = new List<Matrix4x4>(BatchCount);
    //  
    private readonly List<Vector4> _colorList = new List<Vector4>(BatchCount);

    protected override void OnUpdate()
    {
        var materialPropertyBlock = new MaterialPropertyBlock();
        var quadMesh = Globals.Quad;
        var material = Globals.ParticleMaterial;

        //        
        //   
        var shaderPropertyId = Shader.PropertyToID("_Color");

        var entityQuery = GetEntityQuery(typeof(ParticleData));
        //   ,   ParticleData
        var animationData = 
                   entityQuery.ToComponentDataArray<ParticleData>(Allocator.TempJob);
        var layer = LayerMask.NameToLayer("Particles");

        for (int meshCount = 0; 
              meshCount < animationData.Length; 
              meshCount += BatchCount)
        {
            var batchSize = math.min(BatchCount, animationData.Length - meshCount);
            _matrixList.Clear();
            _colorList.Clear();

            for (var i = meshCount; i < meshCount + batchSize; i++)
            {
                _matrixList.Add(animationData[i].matrix);
                _colorList.Add(animationData[i].color);
            }

            materialPropertyBlock.SetVectorArray(shaderPropertyId, _colorList);
            Graphics.DrawMeshInstanced(
                quadMesh,
                0,
                material,
                _matrixList,
                materialPropertyBlock,
                ShadowCastingMode.Off, false, layer);
        }

        animationData.Dispose();
    }
}

この方法を使用してオブジェクトのグループをレンダリングするには、遷移行列の配列と、変化するはずのマテリアルプロパティを収集する必要があります。ご覧のとおり、レンダリングシステムはTrsMatrixCalculationSystem-各パーティクルの遷移行列を計算する補助システムの後に実行されます。これは非常にシンプルに見えます。

[UpdateAfter(typeof(MoveSystem))]
public class TrsMatrixCalculationSystem : SystemBase
{
    protected override void OnUpdate()
    {
        //   scale   
        var scale = Globals.SettingsHolder.SettingsModel.ParticlesScale;
        Entities.ForEach((ref ParticleData data, in Translation translation) =>
            {
                data.matrix = Matrix4x4.TRS(translation.Value, Quaternion.identity,
                    new Vector3(scale, scale, scale));
            })
            .ScheduleParallel();
    }
}
