Androidアプリケーション内のスクリプトの調和



Android開発ハブの多くの読者は、JavaではClassLoaderを介してランタイムにインストール済みのアプリケーションのdex を変更できると聞いたと思います。これを使用すると、コンパイルされたコードを実行時にロードして使用できます。しかし、グーグルはそのような詐欺を軽度に、忠実にではなく扱い、そのようなアプリケーションに捕まった者を禁止します。

ただし、モバイルデバイスにスクリプトをダウンロードして実行する別の方法があります。猫の下の詳細について！

したがって、実行時にdexアプリケーションを更新することはできませんが、完全にJavaで記述されたスクリプト言語インタープリターを使用できます。したがって、Oracleはバージョン6以降、JhascriptエンジンRhinoをJVMに組み込んでいます。これは、Javaでのプログラミング言語のスクリプトのサポートを宣言するJSR-223仕様の実装のおかげで起こりました。

現在、Lua（Luaj）、Python（Jython）、Ruby（Jruby）、java-script（Rhinoなど）などの一般的なプログラミング言語用の組み込みエンジンがいくつかあります。それぞれ、Javaで記述されたスクリプトの実行と関数へのアクセスの両方を可能にします。
可能性を示すために、開発の「環境」を実装することを提案します。記事の最後にソースへのリンクを残しておきます。例を混乱させないために、Luaに焦点を当てますが、すべてのエンジンを同時に接続してそれらを切り替えることを妨げるものはありません。執筆時点でのJLuaの現在のバージョンは、mvnrepository：org.luaj：luaj-jse：3.0.1で入手できます。
それぞれの自尊心のある開発環境には、スクリプトを入力するためのフィールド、結果を表示するためのフィールド、そしてあなたが考えを完成させるためのボタンが必要です。

Luaスクリプトを実行するには、スクリプトが実行されるグローバル環境（Globals）を取得する必要があります。 Luajを使用すると、たとえば変数を設定したり、Javaクラスにバインダーを追加したりして、それをカスタマイズできます。ここで私たちにとって重要な機会は、メッセージ出力ストリームを設定することです。これは、デフォルトでjava.lang.System.out、java.lang.System.errが使用されるためです。これは、実行結果をTextViewで表示する必要がある場合、あまり便利ではありません。これを変更するには、Globals＃STDOUTおよびGlobals＃STDERRの値を再定義する必要があります。

したがって、ここで、クリークを環境にロードして実行するだけで済みます。

これは私の例では次のようになります。

private fun runLua(script: String) {
        val charset = StandardCharsets.UTF_8

        val globals = JsePlatform.standardGlobals()

        val outStream = ByteArrayOutputStream()
        val outPrintStream = PrintStream(outStream, true, charset.name())

        globals.STDOUT = outPrintStream
        globals.STDERR = outPrintStream

        try {
            globals.load(script).call()

            scriptOutput.setTextColor(Color.BLACK)
            scriptOutput.text = String(outStream.toByteArray(), charset)
        } catch (e: LuaError) {
            scriptOutput.setTextColor(Color.RED)
            scriptOutput.text = e.message
        } finally {
            outPrintStream.close()
        }
    }
