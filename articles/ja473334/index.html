猫ゴニム：猫が家の芝生でリラックスしないようにする方法は？

カリフォルニアを拠点とする65歳のプログラマであるRobert Bondがいました。そして彼は彼女のきれいな芝生をとても愛した園芸の妻がいました。しかし、これはカリフォルニアです。猫保護システムを備えた2メートルのフェンスはありません。隣の猫が芝生とがらくたの上を歩く！



問題は解決しなければなりませんでした。ロバートはどうやってそれを決めたのですか？彼は自分のコンピューターにアイロンを購入し、芝生を見渡す屋外監視カメラを接続して、それからやや珍しいことを行い、利用可能な無料のオープンソースソフトウェア（ニューラルネットワーク）をダウンロードし、カメラ画像で猫を認識するように彼女を訓練し始めました。そして、最初のタスクは簡単なようです。何かを学び、それが簡単であれば、それは猫です。猫はインターネットで散らかされているため、何千万もの猫がいるからです。すべてが非常に単純だったが、状況がさらに悪い場合、実際の生活では猫はほとんど夜にがらくたになります。インターネット上の芝生でおしっこをしている夜の猫の写真はほとんどありません。そして、猫の中には、仕事中に灌漑システムから何とか飲んでも、それでも捨てます。



以下に著者からのプロジェクトの説明を提供します。英語版はここにあります。

このプロジェクトの動機は2つありました。ニューラルネットワークソフトウェアについてもっと学びたいという欲求と、隣の猫が私の芝生以外の場所でたむろすることを奨励したいという欲求です。

：プロジェクトは、3つだけのハードウェアコンポーネントが含まのNvidia JETSON TX1のボード、FOSCAM FI9800P IPカメラと粒子光子に接続されたリレー。カメラは家の芝生の側面に取り付けられています。彼女はWIFIアクセスポイントに連絡し、次にJetsonに連絡します。パーティクルフォトンとリレーは、灌漑システムの制御ユニットに設置され、キッチンのWI-FIアクセスポイントに接続されています。

その過程で、カメラは庭の変化を監視するように設定されます。何かが変わると、カメラは1秒間に1つずつ、7つの画像のセットをJetsonに送信します。Jetsonを利用したサービスは、着信画像を追跡し、Caffeのディープトレーニングニューラルネットワークに転送します。ネットワークが猫を検出すると、Jetsonはクラウド内のパーティクルフォトンサーバーに信号を送り、クラウドフォトンサーバーはメッセージをフォトンに送信します。フォトンは、スプリンクラーを2分間オンにすることで応答します。

ネットワーク選択


画像認識用のニューラルネットワークは、通常、オブジェクトのセットを認識するようにトレーニングされています。各オブジェクトに1からnまでのインデックスを与えると仮定します。分類ネットワークは、「この画像のどのオブジェクトですか？」という質問に答えます。 0からn-1までの配列を返します。各配列エントリには0から1までの値があります。ゼロは、オブジェクトが画像内にないことを意味します。ゼロ以外の値は、値が1に近づくと、確率が高くなる可能性があることを意味します。これは、5つの要素の配列にある猫と男です。



セグメント化されたネットワークは、リストのオブジェクトが占める領域の画像ピクセルをセグメント化します。彼女は、画像の各ピクセルに対応するレコードを持つ配列を返すことで、質問に答えます。各レコードの値は、背景ピクセルの場合はゼロ、n個の異なるオブジェクトが認識できる場合は1からnの値になります。この架空の例は、人の足かもしれません。



このプロジェクトは、コンピューターを使用してラジコンカーを制御することを目的とした、より大きなプロジェクトの一部です。アイデアは、ニューラルネットワークを使用して、自動車にナビゲーションコマンドを送信するために自動車の位置（グローバルな3次元の位置と方向）を決定することです。カメラは固定されており、芝生はほとんど平らです。トリガーを少し使用して3D位置を変更し、ニューラルネットワークが画面のピクセルと方向を検出できるようにします。これらすべてにおける猫の役割は「意図された目的」です。

事前に訓練されたネットワークで猫を認識することは簡単だと仮定して、どうなるかわからなかったので、私は主に車について考えることから始めました。この記事では詳しく説明しませんが、多くの作業を行った後、かなり高い信頼性で車の向きを判断できると判断しました。以下は、292.5度の角度でのトレーニングショット



です。この作業のほとんどは、分類ネットワークであるモデルCaffe bvlc_reference_caffenetを使用して行われました。そのため、セグメンテーションネットワークタスクに画面上のマシンの位置を決定させることにしました。

私が最初に使用したネットワークはFaster R-CNN [1]です。ピクセルではなく、画像内のオブジェクトの境界ボックスを返します。しかし、Jetsonのネットワークはこのアプリケーションには遅すぎました。バウンディングボックスのアイデアは非常に魅力的だったので、運転指向ネットワーク[2]も検討しました。彼女も遅すぎた。 FCN [3]は、私が試した最速のセグメンテーションネットワークでした。 「FCN」は「完全たたみ込みネットワーク」、完全なたたみ込みネットワークを意味します。これは、特定の画像サイズを入力する必要がなく、たたみ込み/プーリングのみで構成されるためです。畳み込みレイヤーのみに切り替えると、大幅な高速化につながり、Jetsonで画像を約1/3秒で分類します。FCNには、トレーニングと簡単な導入のためのPythonスクリプトの優れたセットが含まれています。 Pythonスクリプトは、ネットワークのサイズを変更して着信イメージの任意のサイズに合わせ、メインイメージの処理を簡単にします。勝者がいた！

FCN GitHubリリースにはいくつかのオプションがあります。まずvoc-fcn32sを試してみました。それは完全に機能しました。 Voc-fcn32sは、20の標準Vocクラスで事前トレーニングされています。これは単純すぎるため、pascalcontext-fcn32sを試しました。彼は草や木を含む59のクラスで訓練を受けていたので、もっと上手いはずだと思いました。しかし、常にそうであるとは限らないことがわかりました。出力画像にははるかに多くのピクセルセットがあり、草や茂みに重ねられた猫と人のセグメンテーションはそれほど正確ではありませんでした。 siftflowからのセグメンテーションはさらに複雑だったので、すぐにvocオプションに戻りました。

vocネットワークを選択することは、考慮すべき3つのことを意味します：voc-fcn32s、voc-fcn16s、およびvoc-fcn8s。これらは、出力セグメンテーションの「ステップ」が異なります。ステップ32はネットワークの主なステップです。たたみ込み層が完了するまでに、640x360の画像が20x11のネットワークに縮小されます。この粗雑なセグメンテーションは、[3]で説明されているように、640x360に「デコンボリューション」します。ステップ16とステップ8は、より良いセグメンテーションのためにネットワークにより多くのロジックを追加することによって達成されます。私も試していません。32セグメントのセグメンテーションは最初に試したもので、思いついたのですが、このプロジェクトではセグメンテーションが十分に見えるため、他の2つのネットワークではトレーニングがより複雑に見えるため、こだわりました。

トレーニング


電源を入れてシステムを起動したときに最初に気付いたのは、ネットワークで認識されている猫は約30％だけでした。これには2つの理由があります。第一に、猫は夜に来ることが多いので、カメラはそれらを赤外線で見ます。これは簡単に修正できます-訓練のために猫のいくつかの分割された赤外線画像を追加するだけです。トレーニングキットの猫の写真を数百枚レビューした後に発見した2番目の問題は、多くの写真が「かわいい猫を見て」の品種に属していることです。これらは、猫の目の高さでの猫の正面画像です。猫は仰向けになるか、飼い主の膝の上に横たわります。猫が私の庭を歩き回っているようには見えません。繰り返しになりますが、いくつかのセグメント化された昼間の画像で簡単に修正できます。



トレーニング画像のオブジェクトをセグメント化する方法は？私のアプローチは、背景画像を差し引き、次に前景ピクセルを処理して、オブジェクトを追跡するように指示することです。実際には、これはかなりうまく機能します。カメラからのアーカイブには、通常、セグメント化された画像の数秒前に撮影された画像があるためです。ただし、クリーンアップが必要なアーティファクトがあり、セグメンテーションを明確にする必要があることが多いため、画像セグメントを編集するための大まかな準備ユーティリティsrc / extract_fg.cppを作成しました。ソースファイルの上部にある注意を参照してください。それは少し不格好で、小さな検証エラーがあり、いくつかの改良が必要ですが、タスクには十分に機能します。

Solver.py


solve.pyを実行してワークアウトを開始します。

     $ python rgb_voc_fcn32s / solve.py

それはカフェの通常のメカニズムのいくつかを変更します。特に、反復回数はファイルの下部に設定されています。この特定のセットアップでは、各イメージのネットワークサイズが変更され、イメージが一度に1つスキップされるため、反復は単一のイメージです。

Nvidiaでの作業の優れた点の1つは、非常に優れた機器を利用できることです。私はTitanをワークステーションに組み込んでいて、私の経営陣は、このプロジェクトのように疑わしい何かにTitanを使用させてもかまいません。私の最後のトレーニングは4,000回の反復で、タイタンでは2時間強かかりました。

いくつかのことを学びました


  • 夜間の侵入者を認識するようにネットワークを訓練するには、少数の画像（50未満）で十分でした。
  • 夜のショットは、歩道上の影が猫であると考えるようにネットワークに教えました。
  • ネガティブショット、つまりセグメント化されたピクセルのない画像は、影の問題の解決に役立ちます。
  • 固定カメラを使用してネットワークを再トレーニングするのは簡単です。これにより、異なるものはすべてランダムに分類されます。
  • ランダムな背景に重ねられた猫と人間は、過度のトレーニング中に発生する問題を助けます。
