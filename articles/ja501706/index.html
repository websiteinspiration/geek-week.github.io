良い短いレポートを作成する方法

5月の終わりと6月の初めは、大学の卒業証書の保護のための伝統的な時期です。最終的な作品に加えて、学生は試験委員会の前に短い（7〜10分の）プレゼンテーションを準備し、プレゼンテーションを行う必要があります。そして多くの場合、学生が通常のレポート/プレゼンテーションを準備することは、卒業証書を書くことよりもはるかに難しいことです。さらに、多くの場合、有益で興味深い作品ほど、高品質のパフォーマンスを実現することが難しくなります。この記事では、この問題を解決する方法についての見解を示します。



サンクトペテルブルクHSEの私たちは伝統的に、4月から5月にかけて、大学院生と弁護演説を行うために（そして過去6か月間は完全な報告を聞くために）費やしています。そして私たちは、生徒のレベルが非常に高いにもかかわらず、初めてリハーサルを行った生徒はほとんどいないという事実に何らかの形で同意しました。原則として、レポートを2〜3回聞いて、プレゼンテーションを編集し、資料のプレゼンテーションを追加/変更/修正してから、プレゼンテーションを適切に宣言し、生徒を保護のために解放する必要があります。そしてもちろん、毎年、レポートの作成、プレゼンテーションの整理などの同じ基本原則を生徒に説明する必要があります。この記事は、防衛について出てきた生徒に従来説明してきたすべての主要なポイントを書面で提示する試みの一部です。そして、ここで提示する考慮事項が私たちの学生だけでなく他のすべての学生にも役立つことが判明したら、私たちはうれしいです彼らの現在の仕事の結果によると。

注：修士課程「応用数学とコンピュータサイエンス」の卒業生であるエゴールボゴモロフの修士課程「データプログラミングと分析」の1年生の論文のプレゼンテーションから、さらに図解を引用しています。1。

まず、7〜10分のプレゼンテーションの形式では、プレゼンテーションのスライドの数が大幅に制限されます。経験上、スライドは12〜15枚以下にする必要があります。どうして？ 7分のレポートを取り、7をスライドの数で割ると、平均してスライドあたり約30秒になります。 30秒はかなり短い時間です。この間、意味のあることを言うのは困難です。スライドの数が多くなると、レポートが機関銃のキューになり、誰も何も理解する時間がないか、スピーカーが割り当てられた時間に収まるとは限りません。これに基づいて、スライドの数（12〜15）を修正し、スライドに何があるべきかを理解しましょう。



2。最初のスライドでは、すべてがだいたい明確です。レポートのタイトル、著者、監督者、組織を示しています。市と年を祝うこともお勧めです。唯一の微妙な点：最初のスライドに配置された情報を適切にフォーマットしたい。このスライドでは、プレゼンテーション全体のスタイルを設定します。ピンのスタイルとサイズ、後で使用する色です。たとえば、タワーのコーポレートカラーである青と白の使用をお勧めします。



残りのスライドのスタイルについて説明します。それらのそれぞれで、ヘッダーとフッターの両方を実行することが望ましいです。上部のプレゼンテーションブロックの名前（下のブロックについて考えてみましょう）と下部のプレゼンテーションブロックの名前を示すことが望ましい-スピーカーの姓、レポートの短い名前、そして重要なことに、x / y形式のスライドの番号（ここでx現在のスライドの数です。yはスライドの総数です。なぜこれが行われるのですか？リスナーは、原則として、レポートのどの時点で、どのくらい多くのスライドが先に進んでいるかを理解する必要があります。この情報が利用できない場合、レポートが永久に持続するか、いつでも終了するかのいずれかの印象が作成されます。これは迷惑です。

3.次に、スライドのコンテンツに移りましょう。レポート全体は、条件付きで2つの部分に分けることができます。導入部分（作業の目標と目的を策定する前）と実質的な部分（目標と目的の後）です。重要：良いレポートでは、紹介部分は約半分（！）プレゼンテーション全体、そしてここに理由があります。スピーカーが通常の紹介をしなかったり、全員に作業のトピックや問題の説明などを理解させたりしない場合、誰も実質的な部分を聞かないでしょう-聴衆は単に話が何であるかを理解しません。したがって、50％の成功率は、適切に構成された導入部分です。



重要：私たちの観点からは、導入部分は3つのブロックで構成する必要があります。最初のブロック-問題の説明、その中でスピーカーはタスクが何であるか、それがどの主題分野に属しているかを説明し、基本的な概念と定義を紹介し、なぜこの問題の解決が重要であるかを示します。そして、このブロックは、まれな例外を除いて、すべてをレポートに含めることを忘れないでください。しかし、悲しいことに、誰もが彼女に正しく伝える方法を知っているわけではありません。話者の典型的な誤解は、細かい部分を除いて、ほとんどすべての人がこのタスクについて知っているということです。そして、話者は彼の仕事が何であるかを指で説明する代わりに、原則として知られている用語、定義、事実を自由に使い始めます。

私はどちらかと言えば穏やかなことを言うことができますが、問題の説明は、「大学で何をしているのですか？」という質問に対するバーの親/友人/知人への回答とある程度似ているはずです。そして、あなたが両親と一緒にバーやキッチンに座っていると、あなたはなじみのない言葉で注ぎ、自分自身をクールなスペシャリストに育てることはまずありません。ほとんどの場合、少なくとも研究を行う対象分野を指で説明しようとするでしょう。そして、原則として、バー/キッチンに座っている間、あなたはこれをなんとかすることができますが、レポートの間-何らかの理由で、いいえ。あなたは理解しなければなりません：あなたの主題分野についてあなたがより単純で、より明確で、より理解しやすいほど、より多くの人々があなたの報告を最後まで聴くでしょう。次のブロックに



移りましょう-この領域で利用可能な結果のレビュー。ほとんどの話者は彼と問題を抱えています。何らかの理由で、特に短いレポートの場合は、安全に破棄できると考えられています。その間、これは巨大な誤りです;このブロックは断然必要です。レポートにない場合、これは通常2つのオプションを意味します。

最初のオプション -スピーカーは、彼に加えて、この分野で働いて結果を受け取っている研究者が数千ではないとしても数百人いることを理解していません。その結果、話者が語ろうとしていることのすべてが、はるか昔に誰かによって既に行われている可能性が高く、彼はただ車輪を再発明しているだけです。

2番目のオプション-タスクが面白くなく、誰にとっても不必要であるため、誰も実行していない。したがって、このタスクの関連性はゼロに近く、この作業を聞いても意味がありません。もちろん、これらは極端なケースであり、中間的なケースがあります。たとえば、1994年への2つの参照が機能します。そして、それがすべてである場合-再び問題は、上記の2つのオプションを参照してください。

親愛なる同僚たち、他の研究者の結果をレビューしない限り、あなたのすべての仕事は全く意味がありません！その中で、この分野の最新の研究をすべて知っていること、資料を完全に所有していること、過去4〜5年間で誰が、何を、いつ行ったかを知っていることをリスナーに納得させる必要があります。そうでない場合は、レポートを聞くことができなくなります。



文献のレビューでは、導入部を終えて、研究の目標と目的の策定に進むことができるようです。実際には、ほとんどの未経験のスピーカーがそうしようとします。そして、これは非常に大きな間違いです。どのプレゼンテーションでも、もう1つの非常に重要なブロック、つまりコード名「Big BUT」の下のブロックが必要です。このブロックでは、次のように言う必要があります：はい、私の領域ではすでに多くのことが行われています（必須の第1部を参照）（必須の第2部を参照）。ただし、このような作業には、そのような欠点があります。 this-and-suchはこの作業に対して行われていませんが、this-and-and-suchはこの作業からは理解できません。そして、私の仕事では、これらのポイントを排除し、これらの未完成のタスクを完了し、独自のより成功したソリューションを提供します。その後、レポートの中心部分、つまり作業の目標と目的の策定に進むことができます。



4.仕事の目標は1つでなければなりません。複数の目標は許可されていません。それらのいくつかが存在する場合、それはむしろ、研究の定式化された目的が間違った場所と間違った形式である。目的-これは、あなたの仕事で一体的に達成したいことです。ソフトウェア製品の作成、データセットの構築、モデルの構築、分析の実行など。あなたの仕事で達成したいことを考えてください。原則として、これが目標になります。

重要：非常に多くの作品で、かなり重大な間違いがこの場所で行われています。たとえば、「スパムフィルタリングシステムの作成」や「ソーシャルネットワーク上のボットの検出」など、目標があまりにも一般的に定式化されています。同僚、これらは数年前からGoogleやFacebookなどの巨大なIT企業によって解決されてきたタスクです。そして、このグローバルなタスクが最終的に解決されるのはあなたの仕事の中にあることが突然判明した場合、それは非常に面白いでしょう。非常に低い確率で、リスナーはそのような楽観的なシナリオを信じます。最も可能性が高いのは、同様に広く定式化された目標が依然として、原則として、経験不足および/または能力不足の兆候であることです。このような間違いをしないでください。具体的に目標を述べてください。

タスクに移りましょう。タスク-これは、本質的に、あなたの仕事の短い計画であり、あなたが上記の目標を達成する方法を簡単に説明しています。たとえば、「モデルを作成する必要があります」：1）そのようなツールを作成します。 2）そのようなデータセットを収集する。 3）プロトタイプモデルを作成します。 4）このプロトタイプをそのようなデータでテストします。多くの場合、タスクを正しく定式化するために、作業をもう一度見て、それを3〜4個の大きなブロックに分割し、何らかの形でこれらのブロックを呼び出す必要があります。実際には、これらはあなたの仕事になります。

重要：目標やタスクに新しい用語はありません。それらが突然現れた場合、これは、問題の説明を完了していない、そのような問題を解決するために通常使用されるすべての用語、ツール、メカニズムなどを完全に説明していないことを意味します。さらに、多くのタスクはありません。原則として、これらは3〜4つのグローバルタスクです。それらを粉砕したり、7〜8のタスクを実行したりしないでください。最後に、タスクは十分に短く、簡潔でなければなりません。理想的には、これらの各タスクは、後続のスライドのフッターに配置する必要があります。



5。次に、レポートの2番目の実質的な部分に移ります。その構造は非常に簡単です。約7枚のスライドの在庫があります。結論のために最後のスライドを残してください。残りのスライドは6枚です。これらのスライドは、タスクの数に応じてブロックに分割されています。また、短いレポートの各タスクについて、スライドを1〜2枚残します。悲しいかな、これは時間制限のために余裕のある最大値です。何を持ってくるかはすでにかなり具体的な質問です、誰もが自分でこの問題を解決する必要があります。ここでは、各タスクで本当に重要なポイントを選択し、それらについて簡単に説明することが重要です。この部分の一般的なレシピは十分ではありません。ここでの経験は非常に重要です。あなた、あなたの上司、そして同僚です。





多くの場合、この場所で学生は質問をします。まあ、まあ、どういうわけか私はなんとなく概略的に、指で1〜2枚のスライドに何かを示す時間があります。しかし、多くの重要な詳細、数十の重要な詳細があり、レポートを聞いている1人の専門家がこの追加情報なしで作業を評価することはできません。この場合はどうしますか？

ここでは、以下について理解する必要があります。原則として、あなたの報告が何であるかを本当に理解している聴衆の中には2-3人しかいない。ほとんどの場合、彼らは研究の詳細について学ぶことに本当に興味があるでしょう。残りの部分では、詳細は重要ではありません。一般的な考えが必要です。指で説明してください。専門家が細かい場合は、各ブロックから、プレゼンテーションの外側に追加のスライドをいくつか準備してください。たとえば、レポートには15枚のスライドがあります。番号はx / 15のままにします。xは現在のスライドの番号です。 15番目のスライドの後ろに、x / 15、x = 16.17という形式で追加の16〜17番目のスライドを作成します。メインブロックに収まらない情報を取り出すことができます。



レポートの後で突然（これは事実ではありません）、リスナーの1人が特定の質問をすることにした場合、これらのスライドを開いてこの質問に詳細に回答します。同時に、質問に答えると、レポートのために与えられた7〜10分に制限されることはありません。したがって、物理的にメインレポートに収まらなかったが、それについて何か伝えたかった情報を追加できます。



6。最後のスライドに移動します。このスライドでは、研究の結論または結果を説明します。実際、ここでは、レポートの途中で作成されたタスクを複製する必要があります。ただし、問題が発生します。なぜこのスライドが必要なのでしょうか。タスクをもう一度表示して、すべて完了したと言う方が簡単ではないですか？いいえ、重要な違いがあります。実際のところ、調査の目的を定式化する一方で、レポートの内容に含まれる情報にアピールすることはできません。あなたはまだ、目標を達成したアイデア、方法、アプローチ、どの程度成功したか、テストがどれほどうまく行われたかなどの助けを借りて、まだ開示していません。しかし、レポートの終わりには、聴衆はすでにトピックに没頭しており、彼らは用語を知っています、あなたが提案した解決策は既知であり、上記でまとめられたタスクを結論で安全に明らかにし、これらの問題をどのように正確に解決したかを簡単に説明できます。

さて、作品のこの部分のケーキの桜は、たとえば、調査の結果としてあなたが書いた記事へのリンク、特定の会議で作成したレポート、あなたが書いたコードを使い始めたソフトウェア製品などになります。 。

小さなメモ：最後のスライドに別の「ありがとう」は必要ありません。このフレーズは、言葉の終わりで完全に穏やかに言うことができます。 「あなたの質問」のようなスライドも不適切です。このフレーズは、この会議の議長を務める人が、自分のパンを取り上げないでください。

7.さて、結論として、7〜10分の短いレポートではなく、30〜40分の大きなレポートがない場合の対処方法について少し説明します。重要：上記の構造全体は、明確に変更されていません。 「問題の記述」、「利用可能な結果のレビュー」、「しかし」、スライド「目標と目的」、スライド「結論」のブロックが残っています。何が変わりますか？スライドの数は、主にコンテンツ部分で変化しています。1〜2枚のスライドの代わりに、最終的に各タスクに5〜7枚の詳細なスライドを挿入できます。

しかし、これだけではありません。コンテンツ部分を拡張する場合、導入部分を忘れないでください。また、拡張する必要があります。問題のステートメントについてより詳細に伝え、利用可能な結果のより詳細な概要を示す必要があります。導入部分は、少なくともレポートの重要な部分であることに注意してください。それを十分に行ってください-そして、誰も残りのすべての情報を理解できず、誰も聞きません。
