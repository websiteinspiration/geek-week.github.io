Transferring files using pipes and other little things on Delphi

Somehow they set me the task of writing several units in Delphi that provide access to the following Windows features:

  1. Shared memory .
  2. Pipes .

It was impossible to use third-party components. Shared memory was supposed to be used to sort large text files - several tens or even hundreds of megabytes, on which the TStringList fell from EOutOfMemory . Pipes - to transfer files from one application to another.

To rewrite the code of examples from Microsoft documentation from C to Delphi was not even half the battle, but to 10 percent. The main thing was to tie it all together, add additional functions where necessary, and make it all work, and not just exist as source code. As a result, such an ugly monster was born that it wasn’t something to show to anyone, but just looking at him was scary. However, maybe someone will find something useful for himself in his bowels (what if?), So I decided to post it here. For general reproach. The full source code for this abomination is available on GitHub . The text of the article will give the names of the files where you can see one or another piece of the implementation.

The first task is to sort the text file - was decided like this:

  1. Open the file ( CreateFile ).
  2. We use CreateFileMapping and MapViewOfFile (common \ MemoryMappedFileUnit.pas).
  3. We determine the encoding of the file. Here, I limited myself, in fact, to only two encodings: UTF-16 and ACP - a single-byte encoding with a default code page. In Russian-speaking Windows, Windows-1251 is usually used .
  4. We break the text into lines. That is, we are looking for end-of-line markers and, accordingly, we consider the text from marker to marker as one line.
  5. Sort the resulting array of strings (SortingTest \ MergeSorterUnit.pas).
  6. We write the sorted array of strings back to the file.
  7. PROFIT!

The customer definitely wanted to make sorting multithreaded, so Merge Sort was taken as a sorting algorithm . (Thanks to the guru who created the working implementation and posted it on Stackoverflow .)

The second task - transferring the file from one application to another - turned out to be much more difficult. Getting pipes implementation on Delphi based on Microsoft examples is one thing, but making a working file transfer out of it is another. As a server program, Microsoft offers several options:

  1. Multithreaded Pipe Server .
  2. Pipe Server Using Named Overlapped I / O .
  3. Named Pipe Server Using Completion Routines .

As it turned out as a result of two weeks of torment, the implementation of the file transfer mechanism based on the first version of the server ( Multithreaded Pipe Server ) is too complicated. It is more reasonable to use the second option ( Named Pipe Server Using Overlapped I / O ), with it the implementation is much simpler (though, the customer really wanted the first version of the server for some reason ). But still I had to torment myself to bring this whole thing to mind. As a result, a pipe- based file transmitter was born: TCommunicationEndpointBase (CommunicationTest \ SimpleCommunicationUnit.pas).
