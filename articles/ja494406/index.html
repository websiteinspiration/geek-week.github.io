教師なしの機械学習。本からの抜粋

こんにちは、habrozhiteli！私たちのニュースは印刷会社で印刷され、オフィスは離れた場所にありますが、ポールとハーベイのデイテルの本「Python：Artificial Intelligence、Big Data and Cloud Computing」からの抜粋を共有することにしました。

ケーススタディ：教師なしの機械学習、パート2-K平均クラスタリング


このセクションでは、おそらく教師なしの最も単純な機械学習アルゴリズムを紹介します。k平均法を使用したクラスタリングです。アルゴリズムは、ラベルのないサンプルを分析し、それらをクラスターに結合しようとします。 「k平均法」のkは、データが分割されるはずのクラスターの数を表すと説明します。

アルゴリズムは、k個の最近傍のクラスタリングアルゴリズムと同様の距離メトリックを使用して、サンプルを所定の数のクラスターに分散します。各クラスターは、クラスターの中心点である重心を中心にグループ化されます。最初に、アルゴリズムはデータセットサンプルからk個のランダムな重心を選択します。その後、残りのサンプルが最も近い重心を持つクラスターに分散されます。次に、重心の反復的な再計算が実行され、すべてのクラスターについて、特定の重心からそのクラスターに含まれるサンプルまでの距離が最小になるまで、サンプルがクラスター間で再分散されます。アルゴリズムの結果として、各サンプルが属するクラスターを指定するラベルの1次元配列と、各クラスターの中心を表す重心の2次元配列が作成されます。

アイリスデータセット


scikit-learnに含まれている一般的なIrisデータセットを使用します。このセットは、分類およびクラスタリング中に分析されることがよくあります。データセットにはラベルが付けられていますが、これらのラベルを使用してクラスタリングを示すことはしません。次に、ラベルを使用して、k平均アルゴリズムがどの程度サンプルをクラスター化するかを決定します。

Irisデータセットは、150個のサンプルと4つの属性のみで構成されるため、おもちゃのデータセットです。このデータセットは、アイリスの花の3つのタイプ（アイリスセトサ、アイリスバージカラー、アイリスバージニカ）の50個のサンプルを記述しています（下の写真を参照）。サンプルの特性：外側花被葉の長さ（がく片の長さ）、外側花被葉の長さ（がく片の幅）、内側花被葉の長さ（花弁の長さ）、および内側花輪の葉の幅（花弁の幅）（センチメートルで測定）。

14.7.2。アイリスデータセットの研究：パンダの記述統計

In [9]: pd.set_option('max_columns', 5)
In [10]: pd.set_option('display.width', None)

列名としてfeature_names配列の内容を使用して、データ配列の内容でDataFrameコレクションを作成します。

In [11]: iris_df = pd.DataFrame(iris.data, columns=iris.feature_names)

14.7.3。ペアプロットデータセットの視覚化


このデータセットの特性を視覚化します。データに関する情報を抽出する1つの方法は、属性が互いにどのように関連しているかを確認することです。データセットには4つの属性があります。1つの図で1つの属性と他の3つの属性の対応図を作成することはできません。それにもかかわらず、2つの機能間の対応を示す図を作成することは可能です。フラグメント[20]は、Seabornライブラリの関数pairplotを使用して、各機能が他の機能の1つにマッピングされている図表を作成します。

In [17]: import seaborn as sns
In [18]: sns.set(font_scale=1.1)
In [19]: sns.set_style('whitegrid')
In [20]: grid = sns.pairplot(data=iris_df, vars=iris_df.columns[0:4],
  ...: hue='species')
  ...:

主要な議論：

  • チャートにプロットされたデータセットを含むDataFrameコレクション。
  • vars-チャートにプロットされた変数の名前のシーケンス。DataFrameコレクションの場合は、列名が含まれます。この場合、DataFrameの最初の4つの列が使用され、それぞれ、外側の花束の長さ（幅）と内側の花束の長さ（幅）を表します。
  • 色相は、グラフにプロットされるデータの色を決定するために使用されるDataFrameコレクションの列です。この場合、虹彩の種類に応じてデータに色が付けられます。

前のpairplot呼び出しは、次の4×4ダイアグラムテーブルを構築します：


左上から右下隅につながる対角線上のダイアグラムは、値の範囲（左から右）とこれらの値を持つサンプルの数（上から下）とともにこの列に表示される属性の分布を示します。外花被りの長さの分布を見てみましょう：


最も高い陰影領域は、Iris setosa種の（x軸に沿った）外花被葉の長さの範囲が約4〜6 cmであり、ほとんどのIris setosaサンプルでは、​​値がこの範囲の中央（約5 cm）にあることを示します。右端の影付きの領域は、Iris virginica種の外側の花被葉の長さ（x軸に沿った）の範囲が約4〜8.5 cmであり、ほとんどのIris virginicaサンプルの値が6〜7 cmであることを示しています。

他の図では、列は、x軸に沿った特性に対する他の特性のデータ散布図を示しています。最初の列の最初の3つの図では、y軸は外花被の幅、内部花被の長さ、および内部花被の幅をそれぞれ示し、x軸は外部花被の長さを示します。

このコードを実行すると、画面にカラー画像が表示され、個々のキャラクターのレベルでの異なる種類のアイリス間の関係が示されます。興味深いことに、すべての図で、アイリスセトサの青い点は、他の種のオレンジ色と緑色の点から明確に分離されています。これは、アイリスセトサが実際に別のクラスであることを示唆しています。また、オレンジと緑のドットが重なっていることからわかるように、他の2種が混同されることもあります。たとえば、外側の花被葉の幅と長さの図は、アイリスバーシカラーとアイリスバージニカのポイントが混ざっていることを示しています。これは、外花被葉のみの測定が利用可能である場合、これら2つの種を区別することが困難になることを示唆しています。

ペアプロットの結果を1色で出力


色相キー引数を削除すると、関数pairplotは1つの色のみを使用してすべてのデータを出力します。これは、出力内のビューを区別する方法がわからないためです。

In [21]: grid = sns.pairplot(data=iris_df, vars=iris_df.columns[0:4])

次の図からわかるように、この場合、対角線上の図は、タイプに関係なく、この属性のすべての値の分布を持つヒストグラムです。図を調べると、データセットには3つのタイプが含まれていることがわかっていますが、クラスターが2つしかないように見えることがあります。クラスタの数が事前にわからない場合は、データに精通しているサブジェクトエリアの専門家に連絡できます。専門家は、データセットに3種類のデータがあることを知っているかもしれません。この情報は、データを使用して機械学習を行う場合に役立ちます。



ペアプロット図は、少数のフィーチャまたはフィーチャのサブセットでうまく機能するため、行と列の数が制限され、比較的少数のパターンでデータポイントが表示されます。フィーチャとパターンの数が増えると、データ散布図が小さくなりすぎてデータを読み取ることができなくなります。大規模なデータセットでは、フィーチャのサブセットをグラフにプロットし、オプションでランダムに選択されたパターンのサブセットをプロットして、データのアイデアを得ることができます。
