LINQによる全文検索を備えたEntity Framework 6

Entity Frameworkを使って友だちのフルテキストMSSQL検索を友だちにする方法という、かなりありふれた問題を解決するために私の松葉杖を共有したいと思います。トピックは非常に専門的ですが、今日関連しているように私には思えます。興味のある方は猫をお願いします。


すべては痛みから始まりました


私はC＃（ASP.NET）でプロジェクトを開発し、マイクロサービスを作成することもあります。ほとんどの場合、私はMSSQLデータベースを使用してデータを操作します。 Entity Frameworkは、データベースとプロジェクト間のリンクとして使用されます。 EFを使用すると、データを操作し、適切なクエリを生成し、サーバーの負荷を調整するための十分な機会が得られます。魔法のLINQメカニズムは、単にその機能で魅了されます。長年にわたって、私はもはやデータベースを操作するためのより速くてより便利な方法を想像していません。しかし、ほとんどすべてのORMと同様に、EFにはいくつかの欠点があります。まず、これはパフォーマンスですが、これは別の記事のトピックです。次に、データベース自体の機能について説明します。


MSSQLには組み込みの全文検索機能があり、そのまま使用できます。フルテキストクエリを実行するには、組み込みの述語（CONTAINSおよびFREETEXT）または関数（CONTAINSTABLEおよびFREETEXTTABLE）を使用できます。問題は1つだけです。EFは、単語からの全文クエリをまったくサポートしていません。


実際の経験から例を挙げます。Articleテーブル-Articleがあるとします。そして、このテーブルを記述するクラスを作成します。


/// c#
public partial class Article
{
    public int Id { get; set; }
    public System.DateTime Date { get; set; }
    public string Text { get; set; }

次に、これらの記事を選択する必要があります。たとえば、最新の10件の記事を出力します。


/// c#
dbEntities db = new dbEntities();
var articles = db.Article
    .Where(n => n.Active)
    .OrderByDescending(n => n.Date)
    .Take(10)
    .ToArray();

全文検索を追加するタスクが表示されるまで、すべてが非常に美しいです。EFではフルテキスト選択関数がサポートされていないため（.NETコア2.1には既に部分的に含まれています）、サードパーティのライブラリを使用するか、純粋なSQLでクエリを作成するかは変わりません。

SELECT TOP (10) 
    [Project1].[Id] AS [Id], 
    [Project1].[Date] AS [Date], 
    [Project1].[Text] AS [Text], 
    [Project1].[Active] AS [Active]
    FROM ( SELECT 
        [Extent1].[Id] AS [Id], 
        [Extent1].[Date] AS [Date], 
        [Extent1].[Text] AS [Text], 
        [Extent1].[Active] AS [Active], 
        [Extent2].[Rank] AS [Rank]
        FROM  [dbo].[Article] AS [Extent1]
        INNER JOIN [dbo].[FTS_Int] AS [Extent2] ON [Extent1].[Id] = [Extent2].[Key]
        WHERE ([Extent1].[Active] = 1) AND ([Extent2].[Query] LIKE @p__linq__0 ESCAPE N'~')
    )  AS [Project1]
    ORDER BY [Project1].[Rank] DESC
