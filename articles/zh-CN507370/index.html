rpm-gpg-repository-mirroring-从无法在Nexus中创建yum代理的存储库中下载RPM的脚本

在某些组织中，服务器无法访问Internet。在这种情况下，它们将镜像主要存储库。







但是，如果来自服务器的访问受到限制并且需要安装必需的rpm软件包，该怎么办？通常使用reposync或手动下载并创建本地存储库。

您还可以添加将通过代理服务器联机的存储库。代理服务器可以完成很多工作。







但是您可以使用rpm-gpg-repository-mirroring脚本，该脚本将下载必要的rpm软件包并创建本地镜像。







这篇文章的目的是讨论一个脚本，该脚本从存储库Grafana，Kubernetes，Gitlab，packagecloud.io等下载最新版本的rpm软件包。







在reposync上使用脚本进行rpm-gpg-repository-mirroring有什么好处







  • 可以从特定版本开始下载所有rpm软件包
  • 可以下载最新的N版本的rpm软件包
  • 与重新同步相比，可大大节省流量
  • 与重新同步相比，大大节省了磁盘空间
  • 下载最新版本，与手动创建的存储库不同，手动添加了rpm

关闭Selinux



sudo sed -i s/^SELINUX=.*$/SELINUX=disabled/ /etc/selinux/config
sudo reboot


安装并运行rpm-gpg-repository-mirroring（nginx需要使用epel-release）



yum install -y epel-release
yum -y install yum-plugin-copr
yum copr enable antonpatsev/rpm-gpg-repository-mirroring
yum -y install rpm-gpg-repository-mirroring


设置rpm-gpg-repository-mirroring



编辑文件/etc/rpm-gpg-repository-mirroring.conf





，其中详细介绍了所有内容。







从特定版本开始，您需要从该资源库下载所有最新的rpm软件包。Grafana示例



使用以下内容创建/etc/yum.repos.d/grafana.repo：







[grafana]
name=grafana
baseurl=https://packages.grafana.com/oss/rpm
repo_gpgcheck=0
enabled=1
gpgcheck=0
gpgkey=https://packages.grafana.com/gpg.key
sslverify=1
sslcacert=/etc/pki/tls/certs/ca-bundle.crt


配置/etc/rpm-gpg-repository-mirroring.conf：







# ,     rpm 
DOWNLOAD_DIR=/var/www/repos

# , rpm       ,   
REPOS={"grafana":"6.5.3"}


运行脚本







rpm-gpg-repository-mirroring
