機密データへの新しい脅威：アクロニスグローバル調査結果

こんにちは、ハブル！ 5回目の全世界調査で収集した統計を皆さんと共有したいと思います。データ損失がより頻繁に発生した理由、ユーザーが最も恐れている脅威、今日どのくらいの頻度でバックアップが行われ、どのメディアにバックアップが行われたか、最も重要なのは、データ損失が多くなる理由-猫の下で読む。



以前は、毎年3月31日にWorldwide Backup Dayを祝っていました。しかし、近年、データ保護の問題は非常に深刻になり、新しい検疫の現実では、データ保護を保証するための従来のアプローチとソリューションは、プライベートユーザーと組織の両方のニーズを満たすことができなくなりました。したがって、World Backup Dayは、調査結果を公開する枠組みの中で、World Cyber​​ Defense Week全体に変わりました。

5年間、私たちは情報技術に精通している個々のユーザーに、データのバックアップと復元、データ損失などの経験について質問しました。今年の調査には、11か国から約3,000人が参加しました。個人ユーザーに加え、IT専門家の回答者の拡大を図りました。また、調査結果がより明確になるように、2020年のデータと2019年の結果を比較しました。

個人ユーザー


個人ユーザーの世界では、データ保護状況は長い間バラバラでなくなりました。個人の91％がデータとデバイスをバックアップしていますが、68％は、偶発的な削除、ハードウェアまたはソフトウェアの誤動作、または非常にまれなバックアップの結果としてデータを失っています。データまたはデバイスの損失を報告した人の数は2019年に急増し、2020年にはさらに3％増加しました。



過去1年間、個々のユーザーはクラウドへのバックアップをより頻繁に行うようになりました。バックアップをクラウドに保存する人の数は5％増加し、ハイブリッドストレージ（ローカルとクラウドの両方）を好む人は7％増加しました。リモートバックアップのファンには、以前に内蔵および外付けハードドライブにコピーを作成していたユーザーが参加しました。

オンラインおよびハイブリッドバックアップシステムがより直感的で便利になったため、より重要なデータがクラウドに保存されるようになりました。同時に、まったくバックアップを作成しない人の割合が2％増加しました。これは興味深い傾向です。ユーザーは新しい脅威に直面しても降伏し、まだ対処できないと信じていると彼女は言っている可能性が高いです。



しかし、私たちはバックアップを取ることを望まない理由を自分自身に尋ねることに決めました。「単に必要ない」という意見になりました。したがって、多くの人々は依然としてデータ損失のリスクとバックアップの利点を過小評価しています。



一方、1年の間に、バックアップに時間がかかりすぎると考える人の数はわずかに増加しています（アクティブリストアなどの開発が進んでいることはわかっています）。また、保護の設定が複雑すぎると確信している人もいます。同時に、バックアップソフトウェアとサービスが高すぎると考える人は5％未満です。



おそらく近い将来、現代のサイバー脅威に対する個々のユーザーの意識が高まったため、バックアップをオプションであると考える人の数が少なくなる可能性があります。昨年のランサムウェア攻撃に関する懸念は29％増加しました。ユーザーに対して暗号化ジャッキングを使用できるという懸念は31％増加しましたが、ソーシャルエンジニアリング（フィッシングなど）を使用した攻撃は34％増加することを恐れています。

ITプロフェッショナルとビジネス


昨年以来、World Backup DayとWorld Cyber​​ Defense Weekに特化した私たちの調査と調査には、世界中の情報技術専門家が参加しています。したがって、2020年に初めて、私たちは答えを比較し、専門的な環境の傾向を追跡する機会を得ました。



ほとんどの場合、バックアップの頻度が増加しています。 1日に2回以上バックアップを作成するスペシャリストがおり、1か月に1〜2回バックアップを開始するスペシャリストははるかに少なかった。このような珍しいコピーはあまり役に立たないことがわかりましたが、まったくコピーを作成しない人も増えてきました。実際、もっと頻繁に行うことができず、ビジネスの月次コピーに実質的に意味がないのはなぜですか？ただし、最新の製品では会社全体で柔軟なバックアップを設定できるため、この見解は間違いです。これについては、すでにブログで何度か話しました。



バックアップを行う人々は、ほとんどの場合、レプリカを保存するための既存のアプローチを維持しています。それにもかかわらず、2020年に、リモートデータセンターをクラウドにコピーすることを希望する専門家が現れました。

回答者の3分の1以上（36％）が「クラウドストレージ（Google Cloud Platform、Microsoft Azure、AWS、Acronis Cloudなど）」にバックアップを保存しています。面接を受けたすべての専門家の4分の1が「ローカルストレージデバイス（テープドライブ、ストレージアレイ、専用バックアップデバイスなど）に」バックアップを保存しており、20％がローカルストレージとクラウドストレージのハイブリッドを使用しています。

これは興味深いデータです。ハイブリッドバックアップ方式は、他の多くのアプローチよりも効果的であり、レプリケーションよりも安価なので、5人中4人のITスペシャリストが使用していません。



バックアップの頻度と場所に関するこのような決定を考えると、ダウンタイムにつながるデータ損失を経験したITプロフェッショナルの割合が大幅に増加したことは当然のことです。今年、組織の43％が少なくとも1回はデータを失いました。

これは2019 年よりも12％多いです。2020年には、専門家のほぼ半数がデータの損失とダウンタイムに直面しました。しかし、1時間のダウンタイムでは、組織に30万ドルのコストがかかる可能性があります。

さらに詳しく：専門家の9％は、会社がデータ損失に苦しんでいるかどうか、ビジネスのダウンタイムがそれを必要としていたかどうかさえ知らないと述べています。つまり、約10人に1人の専門家が、組み込みの保護機能と、情報環境の少なくともある程度の保証されたアクセス可能性について自信を持って話すことができません。



しかし、これは調査の最も興味深い部分です。2019年と比較して、情報技術の専門家は、関連するすべてのサイバー脅威について心配する必要がなくなりました。技術者は、サイバー脅威を回避したり、脅威に対処したりする能力に自信を持つようになりました。しかし、ダウンタイム統計とこれらのデータを組み合わせると、サイバー脅威がより複雑で洗練されたものになり、専門家の過度の緩和が攻撃者の手に渡るようになるため、業界に問題があることを示しています。特定のアクセス権を持つ人々への攻撃におけるソーシャルエンジニアリングの唯一の問題は、特別な注意に値します。

結論


2019年の終わりには、さらに多くの個人ユーザーとビジネス担当者がデータ損失に直面しました。同時に、リアルタイムデータ保護と定期的なバックアップの実装の複雑さは、サイバー犯罪者が使用するセキュリティホールを作成する上で重要な役割を果たします。

セキュリティシステムの実装を簡略化するために、現在、Acronis Cyber​​ Protect Cloudに取り組んでいます。これは、ハイブリッドデータ保護の実装を簡略化するのに役立ちます。ところで、今ベータテストに参加することができます。次の投稿では、アクロニスの新しいテクノロジーとソリューションについて詳しく説明します。
