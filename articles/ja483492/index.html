json_encodeによる一般的な問題の解決（PHP）

これは、json_encodeで最も可能性の高い問題とその解決策に関する短い記事です。 phpでjson_encodeを使用してjsonでデータをエンコードすると、  期待した結果が得られないことがあります。プログラマーが直面する最も一般的な3つの問題を強調しました。


  • フィールドへのアクセス
  • テキスト値エンコーディング
  • 数値

フィールドへのアクセス


問題は、json_encodeがオブジェクトのパブリックフィールドにのみアクセスできることです。たとえば、クラスがある場合


class Example {
    public $publicProperty;
    protected $protectedProperty;
    private $privateProperty;

    public function __construct($public, $protected, $private)
    {
        $this->publicProperty = $public;
        $this->protectedProperty = $protected;
        $this->privateProperty = $private;
    }
}

次のコードの結果は次のようになります。


$obj = new Example("some", "value", "here"); 
echo json_encode($obj);

// {"publicProperty":"some"}

ご覧のとおり、結果のjsonにはパブリックフィールドのみが含まれていました。
すべてのフィールドが必要な場合はどうしますか？


決定


PHP <5.4の場合：
完成したJSONを返すメソッドをクラスに実装する必要があります。なぜなら クラス内ではすべてのフィールドにアクセスでき、json_encodeのオブジェクトの正しい表現を形成できます  


class Example {
    public $publicProperty;
    protected $protectedProperty;
    private $privateProperty;

    public function __construct($public, $protected, $private)
    {
        $this->publicProperty = $public;
        $this->protectedProperty = $protected;
        $this->privateProperty = $private;
    }

    public function toJson()
    {
        return json_encode([
            'publicProperty' => $this->publicProperty,
            'protectedProperty' => $this->protectedProperty,
            'privateProperty' => $this->privateProperty,
        ]);
    }
}

オブジェクトからjson-aを取得するには、json_encodeをオブジェクトに直接適用するのではなく、toJsonメソッドを使用する必要があります。


$obj = new Example("some", "value", "here"); 
echo $obj->toJson();
