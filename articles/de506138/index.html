Warum verhält sich eine Liste in einem Tupel in Python seltsam?

In Programmiersprachen war ich immer an ihrer internen Struktur interessiert. Wie funktioniert der eine oder andere Bediener? Warum ist es besser so zu schreiben und nicht anders? Solche Fragen helfen nicht immer, das Problem "hier und jetzt" zu lösen, aber auf lange Sicht bilden sie das Gesamtbild der Programmiersprache. Heute möchte ich das Ergebnis eines dieser Tauchgänge teilen und die Frage beantworten, was passiert, wenn tuple'a in list' e geändert wird.

Wir alle wissen, dass Python einen Datentyp hat list:

a = []
a.append(2)

listIst nur ein Array. Sie können Elemente hinzufügen, löschen und ändern. Es unterstützt auch viele verschiedene interessante Operatoren. Zum Beispiel der Operator + = , dem Elemente hinzugefügt werden sollen list. +=ändert die aktuelle Liste, erstellt jedoch keine neue. Dies ist hier deutlich sichtbar:

>>> a = [1,2]
>>> id(a)
4543025032
>>> a += [3,4]
>>> id(a)
4543025032

Python hat einen weiteren großartigen Datentyp: tuple- eine unveränderliche Sammlung. Das Hinzufügen, Löschen oder Ändern von Elementen ist nicht zulässig:

>>> a = (1,2)
>>> a[1] = 3
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

Bei Verwendung des Operators +=wird ein neuer erstellt tuple:

>>> a = (1,2)
>>> id(a)
4536192840
>>> a += (3,4)
>>> id(a)
4542883144

Achtung, die Frage ist: Was macht der folgende Code?

a = (1,2,[3,4])
a[2] += [4,5]

Optionen:

  1. Elemente werden der Liste hinzugefügt.
  2. Die Tupel-Unveränderlichkeitsausnahme wird ausgelöst.
  3. Beide.
  4. Weder der eine noch der andere.

Schreiben Sie Ihre Antwort auf ein Blatt Papier und lassen Sie uns einen kleinen Check machen:

>>> a = (1,2,[3,4])
>>> a[2] += [4,5]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

Na dann! Also haben wir es herausgefunden! Die richtige Antwort ist 2. Warten Sie einen Moment:

>>> a
(1, 2, [3, 4, 4, 5])

Tatsächlich lautet die richtige Antwort 3. Das heißt, die Elemente wurden hinzugefügt und die Ausnahme kam heraus - wat ?!


Mal sehen, warum das passiert. Und ein wunderbares Modul wird uns dabei helfen dis:

import dis

def foo():
    a = (1,2,[3,4])
    a[2] += [4,5]

dis.dis(foo)
  2     0 LOAD_CONST      1 (1)
        3 LOAD_CONST      2 (2)
        6 LOAD_CONST      3 (3)
        9 LOAD_CONST      4 (4)
       12 BUILD_LIST      2
       15 BUILD_TUPLE     3
       18 STORE_FAST      0 (a)

  3    21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)
       27 DUP_TOP_TWO
       28 BINARY_SUBSCR
       29 LOAD_CONST      4 (4)
       32 LOAD_CONST      5 (5)
       35 BUILD_LIST      2
       38 INPLACE_ADD
       39 ROT_THREE
       40 STORE_SUBSCR
       41 LOAD_CONST      0 (None)
       44 RETURN_VALUE

Der erste Block ist dafür verantwortlich tuple, a zu erstellen und in einer Variablen zu speichern a. Dann beginnt der Spaß:

       21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)

Wir laden den Zeiger auf die Variable aund Konstante 2 auf den Stapel .

       27 DUP_TOP_TWO

Dupliziere sie und lege sie in der gleichen Reihenfolge auf den Stapel.

       28 BINARY_SUBSCR

Dieser Operator übernimmt das oberste Element des Stapels (TOS) und das nächste (TOS1). Und schreibt ein neues Element an die Spitze des Stapels TOS = TOS1[TOS]. Also entfernen wir die beiden obersten Werte vom Stapel und fügen einen Link zum zweiten Element tuple'a (unserem Array) ein.

       29 LOAD_CONST      4 (4)
       32 LOAD_CONST      5 (5)
       35 BUILD_LIST      2

Wir erstellen eine Liste der Elemente 4 und 5 und legen sie oben auf den Stapel:

       38 INPLACE_ADD

Auf +=die beiden obersten Elemente des Stapels anwenden (Wichtig! Dies sind zwei Listen! Eine besteht aus 4 und 5, die andere stammt aus tuple). Hier ist alles in Ordnung, die Anweisung wird fehlerfrei ausgeführt. Da +=die ursprüngliche Liste tuplegeändert wird , hat sich die Liste in 'e bereits geändert (in diesem Moment).

       39 ROT_THREE
       40 STORE_SUBSCR

Hier tauschen wir die drei obersten Elemente des Stapels aus (lebt dort tuple, darin den Index des Arrays und das neue Array) und schreiben das neue Array tuplenach Index ein. Hier tritt die Ausnahme auf!

Das war's! Tatsächlich kann die Liste geändert werden, aber alles liegt beim Bediener =.

Lassen Sie uns endlich herausfinden, wie Sie diesen Code ausnahmslos neu schreiben können. Wie wir bereits verstanden haben, müssen Sie nur den Eintrag in entfernen tuple. Hier sind einige Optionen:

>>> a = (1,2,[3,4])
>>> b = a[2]
>>> b += [4,5]
>>> a
(1, 2, [3, 4, 4, 5])

>>> a = (1,2,[3,4])
>>> a[2].extend([4,5])
>>> a
(1, 2, [3, 4, 4, 5])
