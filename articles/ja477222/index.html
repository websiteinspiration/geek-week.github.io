レイジーコンビネーター


ただし、この記事の主な目的は、プリコンパイルされた数式について話すことではありません。為替レート分析のボーダーラインのケースでは、かなり長いリストの順列を計算する必要がありました。突然、Elixir標準ライブラリはターンキーソリューションを提供しなくなりました。まあ、私はRuby（Array#combinationといとこ）から組み合わせの署名をコピーすることにしました。残念ながら、これは長いリストではそれほど簡単ではありませんでした。組み合わせは、リストの30要素の領域で停止しました。順列-より早い段階で。

まあ、それはすでにここにあると予想されます。だから私はストリームを使って遅延実装を始めました。結局、思ったほど簡単ではありませんでした。以下のようなコードを思いついた

            {_, idx3}, :ok when idx3 <= idx2 ->

上記のコードでは、3つの異なるパターンが表示されています。リストを削除する成功したブランチ。空のリストのクイックダンプ。そして、インデックスによるフローの変換。上記のASTを作成しようとしたようです。

これは、使用Kernel.SpecialForms.quote/2が物事を複雑にするだけの可能性が高いまれなケースなので、抵抗を最小限に抑えるための道を歩みました。古き良き裸のASTを彫刻します。

私はquote do:このコードでコンソールを呼び出すことから始めました。そして、私は目の痛いところまで、結果を研究しました。はい、パターンがあります。行こう。

したがって、共通のフレームワークを作成することから始める必要があります。

defmacrop mapper(from, to, fun),
  do: quote(do: Enum.map(Range.new(unquote(from), unquote(to)), unquote(fun)))

@spec combinations(list :: list(), count :: non_neg_integer()) :: {Stream.t(), :ok}
defmacro combinations(l, n) do
  Enum.reduce(n..1, {[mapper(1, n, &var/1)], :ok}, fn i, body ->
    stream_combination_transform_clause(i, l, body)
  end)
end

ここで、繰り返しのテンプレートパーツを見るために、コードではなくASTの観点から考える必要があります。楽しい！

コードを簡略化するヘルパーマクロから始めましょう。

def var(i), do: {:"i_#{i}", [], Elixir}
def idx(i), do: {:"idx_#{i}", [], Elixir}

一般的なビューから引き裂かれたASTインナーピース：

def sink_combination_clause(i) when i > 1 do
  {:->, [],
    [
      [
        {:when, [],
        [
          {​{:_, [], Elixir}, idx(i)},
          :ok,
          {:<=, [context: Elixir, import: Kernel], [idx(i), idx(i - 1)]}
        ]}
      ],
      {[], :ok}
    ]}
end

一緒にすべての内側の部分：

def sink_combination_clauses(1, body) do
  [{:->, [], [[{var(1), idx(1)}, :ok], body]}]
end

def sink_combination_clauses(i, body) when i > 1 do
  Enum.reverse([
    {:->, [], [[{var(i), idx(i)}, :ok], body]}
    | Enum.map(2..i, &sink_combination_clause/1)
  ])
end

そして最後に、そのすべての外側のラッパーです。

def stream_combination_transform_clause(i, l, body) do
  clauses = sink_combination_clauses(i, body)

  {​{​{:., [], [{:__aliases__, [alias: false], [:Stream]}, :transform]}, [],
    [
      {​{:., [], [{:__aliases__, [alias: false], [:Stream]}, :with_index]}, [], [l]},
      :ok,
      {:fn, [], clauses}
    ]}, :ok}
end

すべての置換はほぼ同じように実行され、唯一の変更は内部呼び出しの条件です。簡単でしょ？すべてのコードはリポジトリで表示できます。
