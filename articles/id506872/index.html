Book Spring Boot 2: Praktik Terbaik untuk Profesional

Halo, habrozhiteli! Ingin meningkatkan efektivitas Anda dalam mengembangkan aplikasi enterprise dan cloud Java?

Meningkatkan kecepatan dan kemudahan mengembangkan layanan microser dan aplikasi yang kompleks, menghilangkan kekhawatiran konfigurasi Spring.

Gunakan perangkat kerangka kerja Spring Boot 2 dan Spring 5 seperti WebFlux, Security, Actuator, dan kerangka kerja Micrometer, yang menyediakan cara baru untuk mengumpulkan metrik.


Kerangka Pengujian Musim Semi


Salah satu ide utama dari kerangka kerja Spring adalah untuk mendorong pengembang untuk membuat kelas pemrograman yang sederhana dan longgar, antarmuka pemrograman, sehingga perangkat lunak menjadi lebih andal dan lebih dapat diperluas. Kerangka kerja Spring menyediakan alat yang menyederhanakan pengujian unit dan integrasi (pada kenyataannya, jika Anda benar-benar memprogram antarmuka, Anda tidak perlu menguji fungsionalitas aplikasi Spring). Dengan kata lain, perlu bahwa aplikasi dapat diuji menggunakan sistem pengujian JUnit atau TestNG berdasarkan objek (dibuat hanya menggunakan operator baru - tanpa Pegas atau wadah lainnya).

Kerangka kerja Pegas mencakup beberapa paket yang dirancang untuk pengujian unit atau integrasi aplikasi. Untuk pengujian unit, beberapa objek simulasi dirancang (Lingkungan, PropertySource, JNDI, Servlet; utilitas pengujian reaktif ServerHttpRequest dan ServerHttpResponse), yang dengannya Anda dapat melakukan pengujian isolasi kode.

Salah satu kemampuan pengujian yang paling umum digunakan dalam kerangka Spring adalah pengujian integrasi. Tugas utamanya:
  • Mengelola caching kontainer IoC Musim Semi antara uji coba
  • pengelolaan transaksi;
  • Ketergantungan injeksi contoh objek uji
  • pembuatan kelas dasar yang dirancang khusus untuk Musim Semi.

Kerangka kerja Spring menyediakan cara mudah untuk menguji dengan mengintegrasikan ke dalam tes konteks aplikasi (ApplicationContext). Modul pengujian Musim Semi menawarkan beberapa cara untuk menggunakan ApplicationContext secara terprogram dan menggunakan anotasi.

  • Abstrak BootstrapDengan. Anotasi tingkat kelas untuk mengonfigurasi bootstrap dari kerangka kerja TestContext Spring.
  • @ContextConfiguration annotation. Menentukan metadata tingkat kelas yang menentukan cara memuat dan mengonfigurasi ApplicationContext untuk pengujian integrasi. Ini adalah anotasi yang harus dimiliki untuk kelas Anda, karena ini adalah tempat ApplicationContext memuat semua definisi komponen.
  • Abstrak @ WebAppConfiguration. Anotasi tingkat kelas yang menunjukkan bahwa konteks aplikasi yang dimuat untuk tes integrasi harus WebApplicationContext.
  • Anotasi @ActiveProfile. Anotasi tingkat kelas yang menunjukkan profil definisi komponen mana yang harus aktif saat memuat ApplicationContext untuk pengujian integrasi.
  • Abstrak @TestPropertySource. Anotasi tingkat kelas yang dirancang untuk menentukan lokasi file properti dan properti tertanam yang ditambahkan ke objek PropertySource yang diatur dalam Lingkungan untuk ApplicationContext dimuat untuk pengujian integrasi.
  • Abstract @DirtiesContext. Menunjukkan bahwa ApplicationContext yang digunakan adalah "kotor" selama pengujian (misalnya, dimodifikasi atau rusak dengan mengubah status komponen tunggal) dan harus ditutup.

Ini jauh dari semua penjelasan, ada banyak lainnya (@TestExecutionListeners, Melakukan, Kembalikan, @BeforeTransaction, @AfterTransaction, Sql, SqlKelompok SqlKonfigurasi Waktunya, Ulang, @IfProfileValue, dll.).

Seperti yang Anda lihat, ada banyak opsi pengujian dengan kerangka kerja Spring. Biasanya anotasi @RunWith selalu digunakan, menghubungkan semua elemen kerangka pengujian. Contohnya:

@RunWith(SpringRunner.class)
@ContextConfiguration({"/app-config.xml", "/test-data-access-config.xml"})
@ActiveProfiles("dev")
@Transactional
public class ToDoRepositoryTests {

        @Test
        public void ToDoPersistenceTest(){
                //...
        }
}

Sekarang mari kita lihat bagaimana cara menggunakan Spring Testing Framework dan fitur apa yang disediakan Spring Boot.

Kerangka Pengujian Booting Musim Semi


Spring Boot memanfaatkan kekuatan dari Spring Testing Framework, memperluas yang lama dan menambahkan anotasi dan fitur baru yang membuat pengujian lebih mudah bagi pengembang.

Untuk mulai menggunakan semua fitur pengujian Spring Boot, Anda hanya perlu menambahkan dependensi spring-boot-starter-test ke aplikasi dengan cakupan tes (lingkup pengujian). Layanan Spring Initializr telah menambahkan ketergantungan ini.

Ketergantungan spring-boot-starter-test menyediakan kemampuan untuk menggunakan beberapa kerangka kerja pengujian yang sangat konsisten dengan kemampuan pengujian Spring Boot: JUnit, AssertJ, Hamcrest, Mockito, JSONassert, dan JsonPath. Tentu saja, ada kerangka kerja pengujian lain yang berfungsi baik dengan modul Uji Boot Spring; hanya dependensi yang sesuai perlu ditentukan secara manual.

Spring Boot menyediakan anotasi @SpringBootTest untuk menyederhanakan pengujian aplikasi Spring. Biasanya, saat menguji aplikasi Spring, Anda perlu menambahkan beberapa penjelasan untuk menguji fitur aplikasi atau fungsionalitas, tetapi tidak di Spring Boot - walaupun untuk pengujian Anda masih perlu menentukan penjelasan @RunWith (SpringRunner.class); jika ini tidak dilakukan, anotasi pengujian Boot Musim Semi baru akan diabaikan. Anotasi @SpringBootTest memiliki opsi yang berguna untuk menguji aplikasi web, seperti RANDOM_PORT dan DEFINED_PORT.

Cuplikan kode berikut adalah kerangka uji Spring Boot.

@RunWith(SpringRunner.class)
@SpringBootTest
public class MyTests {

        @Test
        public void exampleTest() {
                 ...
        }
}

Pengujian Titik Akhir Aplikasi Web


Spring Boot menyediakan cara yang nyaman untuk menguji titik akhir: lingkungan simulasi yang disebut kelas MockMvc:

@RunWith(SpringRunner.class)
@SpringBootTest
@AutoConfigureMockMvc
public class MockMvcToDoTests {

        @Autowired
        private MockMvc mvc;

        @Test
        public void toDoTest() throws Exception {
              this.mvc
              .perform(get("/todo"))
              .andExpect(status().isOk())
              .andExpect(content()
                   .contentType(MediaType.APPLICATION_JSON_UTF8));
        }
}

Anda juga dapat menggunakan kelas TestRestTemplate.

@RunWith(SpringRunner.class)
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
public class ToDoSimpleRestTemplateTests {

      @Autowired
      private TestRestTemplate restTemplate;

      @Test
      public void toDoTest() {
            String body = this.restTemplate.getForObject("/todo", String.class);
            assertThat(body).contains("Read a Book");
      }
}

Kode ini menunjukkan tes yang memulai server penuh dan menggunakan turunan dari kelas TestRestTemplate untuk mengakses titik akhir / todo. Di sini kita mengasumsikan bahwa objek String dikembalikan (ini bukan cara terbaik untuk menguji pengembalian JSON; jangan khawatir, kami akan menunjukkan cara menggunakan kelas TestRestTemplate dengan benar).

Komponen imitasi


Modul pengujian Spring Boot menyediakan anotasi @MockBean yang menjelaskan objek simulasi Mockito untuk komponen di ApplicationContext. Dengan kata lain, Anda bisa membuat tiruan komponen Pegas baru atau mengganti definisi yang ada dengan menambahkan anotasi ini. Ingat: semua ini terjadi di dalam ApplicationContext.

@RunWith(SpringRunner.class)
@SpringBootTest
public class ToDoSimpleMockBeanTests {

        @MockBean
        private ToDoRepository repository;

        @Test
        public void toDoTest() {
              given(this.repository.findById("my-id"))
                    .Return(new ToDo("Read a Book"));
              assertThat(
                    this.repository.findById("my-id").getDescription())
                    .isEqualTo("Read a Book");
        }
}

Irisan Tes Booting Musim Semi


Salah satu fitur terpenting dari Spring Boot adalah menjalankan pengujian tanpa memerlukan infrastruktur khusus apa pun. Modul pengujian Boot Musim Semi mencakup apa yang disebut irisan yang dirancang untuk menguji bagian spesifik aplikasi tanpa menggunakan server atau DBMS.

@JsonTest anotasi


Modul pengujian Boot Musim Semi memiliki anotasi @JsonTest yang menyederhanakan serialisasi / deserialisasi objek JSON dan memeriksa untuk melihat apakah semuanya berfungsi dengan benar. @JsonTest secara otomatis mengkonfigurasi alat tampilan JSON yang didukung, tergantung pada pustaka yang ditemukan di jalur kelas: Jackson, GSON, atau JSONB.

@RunWith(SpringRunner.class)
@JsonTest
public class ToDoJsonTests {

      @Autowired
      private JacksonTester<ToDo> json;

      @Test
      public void toDoSerializeTest() throws Exception {
            ToDo toDo = new ToDo("Read a Book");

            assertThat(this.json.write(toDo))
            .isEqualToJson("todo.json");
            assertThat(this.json.write(toDo))
            .hasJsonPathStringValue("@.description");

            assertThat(this.json.write(toDo))
            .extractingJsonPathStringValue("@.description")
            .isEqualTo("Read a Book");
      }

      @Test
      public void toDoDeserializeTest() throws Exception {
            String content = "{\"description\":\"Read a Book\",\"completed\":
                  true }";
            assertThat(this.json.parse(content))
                  .isEqualTo(new ToDo("Read a Book", true));
            assertThat(
               this.json.parseObject(content).getDescription())
            .isEqualTo("Read a Book");
      }
}

Untuk menguji pengontrol tanpa menggunakan server penuh, Anda dapat menggunakan anjuran Spring Boot yang diusulkan @WebMvcTest, yang secara otomatis mengkonfigurasi infrastruktur Spring MVC dan membatasi daftar komponen yang dilihat sebagai berikut: Pengendali, PengendaliSaran, @JsonComponent, Konverter, GenericConverter, Filter, WebMvcConfigurer, dan HandlerMethodArgumentResolver; Berkat ini, Anda akan tahu jika controller Anda berfungsi seperti yang diharapkan.

Penting untuk dipahami bahwa yang ditandai sebagaiKomponen komponen tidak terlihat saat menggunakan anotasi ini, tetapi Anda dapat menggunakan anotasi @MockBean jika perlu.

@RunWith(SpringRunner.class)
@WebMvcTest(ToDoController.class)
public class ToDoWebMvcTest {

      @Autowired
      private MockMvc mvc;

      @MockBean
      private ToDoRepository toDoRepository;

      @Test
      public void toDoControllerTest() throws Exception {
            given(this.toDoRepository.findById("my-id"))
                  .Return(new ToDo("Do Homework", true));
                  this.mvc.perform(get("/todo/my-id").accept(MediaType.APPLICATION_
                     JSON_UTF8))
                        .andExpect(status().isOk()).andExpect(content().
                           string("{\"id\":\"my-id\",\"description\":\"Do Homework\",
                           \"completed\":true}"));
      }
}

@WebFluxTest anotasi


Untuk pengontrol reaktif, Spring Boot menyediakan anotasi @WebFluxTest. Anotasi ini secara otomatis mengkonfigurasi infrastruktur modul Spring WebFlux dan hanya pencarianPengendali, PengendaliSaran, @JsonComponent, Konverter, GenericConverter dan WebFluxConfigurer.

Penting untuk dipahami bahwa yang ditandai sebagaiKomponen komponen tidak terlihat saat menggunakan anotasi ini, tetapi Anda dapat menggunakan anotasi @MockBean jika perlu.

@RunWith(SpringRunner.class)
@WebFluxTest(ToDoFluxController.class)
public class ToDoWebFluxTest {

      @Autowired
      private WebTestClient webClient;

      @MockBean
      private ToDoRepository toDoRepository;

      @Test
      public void testExample() throws Exception {
             given(this.toDoRepository.findAll())
                         .Return(Arrays.asList(new ToDo("Read a Book"), new
                               ToDo("Buy Milk")));
             this.webClient.get().uri("/todo-flux").accept(MediaType.
                  APPLICATION_JSON_UTF8)
                       .exchange()
                       .expectStatus().isOk()
                       .expectBody(List.class);
      }
}

@DataJpaTest anotasi


Untuk menguji aplikasi JPA, modul pengujian Boot Musim Semi menyediakan anotasi @DataJpaTest, yang secara otomatis mengkonfigurasi basis data tertanam yang terletak di RAM. Dia sedang mencariKesatuan dan tidak memuat komponen apa pun Komponen. Selain itu, ia menyediakan kelas pembantu berorientasi tes, TestEntityManager, sangat mirip dengan kelas JPA EntityManager.

@RunWith(SpringRunner.class)
@DataJpaTest
public class TodoDataJpaTests {
  
      @Autowired
      private TestEntityManager entityManager;

      @Autowired
      private ToDoRepository repository;

      @Test
      public void toDoDataTest() throws Exception {
            this.entityManager.persist(new ToDo("Read a Book"));
            Iterable<ToDo> toDos = this.repository.
                   findByDescriptionContains("Read a Book");
            assertThat(toDos.iterator().next()).toString().contains("Read a Book");
      }
}

Harap dicatat bahwa ketika menguji dengan @DataJpaTest, DBMS internal dalam RAM digunakan. Untuk pengujian dengan database nyata, Anda harus memberikan kelas tes dengan penjelasan @AutoConfigureTestDatabase (ganti = Ganti Ganti.NONE).

@RunWith(SpringRunner.class)
@DataJpaTest
@AutoConfigureTestDatabase(replace=Replace.NONE)
public class TodoDataJpaTests {
        //...
}

@JdbcTest anotasi


Anotasi ini sangat mirip dengan @DataJpaTest; satu-satunya perbedaan adalah bahwa ini berjalan secara eksklusif pada tes JDBC. Secara otomatis mengkonfigurasi DBMS built-in yang terletak di RAM dan kelas JdbcTemplate, sambil melewatkan semua anotasiKomponen kelas.

@RunWith(SpringRunner.class)
@JdbcTest
@Transactional(propagation = Propagation.NOT_SUPPORTED)
public class TodoJdbcTests {

      @Autowired
      private NamedParameterJdbcTemplate jdbcTemplate;

      private CommonRepository<ToDo> repository;

        @Test
        public void toDoJdbcTest() {
              ToDo toDo = new ToDo("Read a Book");

            this.repository = new ToDoRepository(jdbcTemplate);
            this.repository.save(toDo);

            ToDo result = this.repository.findById(toDo.getId());
            assertThat(result.getId()).isEqualTo(toDo.getId());
        }
}


@DataMongoTest anotasi


Untuk menguji aplikasi MongoDB, modul pengujian Boot Musim Semi menyediakan anotasi @DataMongoTest. Secara otomatis mengkonfigurasi server Mongo yang terintegrasi dengan memori, jika tersedia; jika tidak, Anda perlu menambahkan properti yang diperlukan dari spring.data.mongodb. *. Itu juga menyetel kelas MongoTemplate dan mencari anotasi @Document. KomponenKomponen dilewati.

@RunWith(SpringRunner.class)
@DataMongoTest
public class TodoMongoTests {

      @Autowired
      private MongoTemplate mongoTemplate;

      @Test
      public void toDoMongoTest() {
          ToDo toDo = new ToDo("Read a Book");
          this.mongoTemplate.save(toDo);

          ToDo result = this.mongoTemplate.findById(toDo.getId(),ToDo.class);
          assertThat(result.getId()).isEqualTo(toDo.getId());

@RunWith(SpringRunner.class)
@DataMongoTest(excludeAutoConfiguration = EmbeddedMongoAutoConfiguration.class)
public class ToDoMongoTests {
        // ...
}

@RestClientTest anotasi


Anotasi penting lainnya adalah @RestClientTest, yang dirancang untuk menguji klien REST. Anotasi ini secara otomatis membuat pengaturan untuk mendukung Jackson, GSON, dan JSONB, dan juga mengatur kelas RestTemplateBuilder dan menambahkan dukungan untuk MockRestServiceServer.

@RunWith(SpringRunner.class)
@RestClientTest(ToDoService.class)
public class ToDoRestClientTests {

      @Autowired
      private ToDoService service;

      @Autowired
      private MockRestServiceServer server;

      @Test
      public void toDoRestClientTest()
                  throws Exception {
             String content = "{\"description\":\"Read a Book\",\"completed\":
                   true }";
            this.server.expect(requestTo("/todo/my-id"))
             .andRespond(withSuccess(content,MediaType.APPLICATION_JSON_UTF8));
            ToDo result = this.service.findById("my-id");
            assertThat(result).isNotNull();
            assertThat(result.getDescription()).contains("Read a Book");
      }
}
