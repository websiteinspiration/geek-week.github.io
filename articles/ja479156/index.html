ESP32 + Arduino Core + FreeRTOS + Blynk =心の始まりを持つ家

プロジェクトの目的


どういうわけか、私は私の家、スケルトンを建てたことがわかりました。私の贅沢なaulにはガスがなく、近い将来に予想されないので、私はスケルトンを選びました-私にとって、電気で加熱することは非常に高価になるでしょう。まあ、それはそれが最も安い技術の一つだからです。
わかった、家の周りにパイプを投げ、バッテリー、ボイラーを掛けた。暖かいように見えたが、何かがおかしい。


自分の声を聞いて、これは好きではないヒキガエルであることに気付きました。家にいないとき（1日12〜16時間）、暖房は機能しています。また、スケルトンにはわずかな慣性があり、すばやく温度を上げることができるため、機能しない場合があります。到着する前にのみオンにします。どこか長い間家を出たときと同じ状況。まあ、一般的に、路上でボイラーのハンドルを温度の変化に合わせてひねることは、どういうわけかコーシャではありません。


自動化がなければ、どこにもボイラーが最もシンプルなボイラーであることが明らかになりましたが、外部制御リレーを接続するための接点があります。もちろん、必要な機能をすべて備えたボイラーをすぐに購入することもできますが、私にとって、そのようなボイラーはなんとなく非人道的です。さらに、私はarduinoバージョンではありますが、頭を使ってしゃがんだり、魂のために何かをおしっこしたり、少しCを学びたいと思っていました。


実際に要件について：


  • セットポイント温度制御
  • 屋外または手動での温度に応じたクーラント温度の制御
  • 設定が異なるタイムゾーン、日中は冷たく、夜はより高温
  • 自動モード、自動昼夜移行
  • 週末の手動モード、自動遷移なし
  • クーラント温度を手動で設定し、ボイラーをオン/オフできる非自動モード
  • ボタンと画面から、およびWebサイト/モバイルアプリケーションを介して、ローカルで加熱制御

それは初めでした、そして私は苦しんで追加しました：


  • 街路灯制御（LEDスポットライト）
  • モーションセンサー、サイレン、街灯に基づく警報システム
  • ボイラーによって消費されるエネルギーを日/月/年ごとに測定+年間の各月ごと
  • ランプのゆっくりとした点滅のみによる警報モード
  • ランプの高速点滅とサイレンの短いビープ音による信号モード
