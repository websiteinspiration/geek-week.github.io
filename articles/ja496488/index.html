IaaSプロバイダーがBGPプロトコルへの攻撃と戦う

いくつかの大規模な組織がMANRSに参加しています。これは、参加者がプロバイダーのネットワーク内のルートに関する誤ったデータの配布を防止しようとしているイニシアチブです。

ルーティングエラーは失敗につながる可能性があります。多くの人が自宅に座っており、インフラストラクチャの負荷が増加し続けている現在、それらを回避することは特に重要です。

クラウドの運用に影響を与えるもの


ここ数か月の間に、より多くの人々が家に座ってリモートで仕事を始めました。この事実により、IaaSプロバイダーのサービスへの負荷が大幅に増加しています。欧米のクラウドベンダーの1人は、厳格な自己分離対策が導入されたイタリアのユーザー数が8倍に増加したと指摘してい ます。この影響の一部は、リモートジョブをセットアップするためのインフラストラクチャを購入する意思のある企業の数の増加によるものです。Europolに よると、ITインフラストラクチャへの負荷とともに、サイバー犯罪の数も増加しています。同様に、状況は米国で進展しています。詐欺師は、フィッシングメールでコロナウイルスに言及することがよくあります。そのような攻撃の成長は警告されました

国土安全保障省でさえ。

また、専門家は、動的ルーティングプロトコルBGPを狙った攻撃の数の増加を記録しています。 DNSと同様に、DNSはインターネットの主要コンポーネントの1つです。プロトコルを使用して、プロバイダールーターはネットワーク可用性データを相互に送信します。 1998年に、ハッカーグループL0phtのメンバーは、BGPへの十分に検討された攻撃により、30分でグローバルインターネットを「切断」できると述べました。 このような状況では、クラウドやインターネットプロバイダーのネットワークが構築されているインフラストラクチャを保護することが重要です。そのため、3月末には、いくつかの大手IaaSベンダーとIT企業（Google、Facebook、Microsoft、Netflixなど）が接続しました

MANRSプロジェクト（ルーティングセキュリティの相互に合意された規範）へ。

MANRSの機能


このプロジェクトは、インターネットエンジニアリングカウンシル（IETF）とインターネットアーキテクチャカウンシル（IAB）の活動に資金を提供するインターネットソサエティ国際機関の支援を受けて2014年に設立されました。そのメンバー-今日では  300人以上がいます -ルーティングのセキュリティを改善するための一連のベストプラクティスに取り組んでいます。 3月に、イニシアチブの一環として、特にクラウドプロバイダーとコンテンツ配信ネットワーク（CDN）の運用を担当する企業を対象とした新しいプログラムが開始されました。それはグーグル、フェイスブックおよび他のオペレーターによってそれに接続されていました。



これらは、ルートハイジャックやIPスプーフィングと戦うためのフィルタリングおよびスプーフィング防止メカニズムの実装に役立ちます。

企業はまた、グローバルルーティングセキュリティのための階層型公開鍵システムであるRPKI（Resource Public Key Infrastructure）を普及させます。これまでのところ、MANRS参加者の10％のみがそれを使用しています。より多くのIT企業がネットワーク監視用のツールを開発します。

展望


Internet SocietyのプロジェクトマネージャーであるAndrei Robachevskyは、GoogleやMicrosoftなどの主要企業がBGPへのハッカー攻撃の数を減らすのに役立つと確信しています。過去数年にわたって、MANRSはすでにある程度の成功を収めています。The Registerに

よると、2017年にアナリストはプロトコルへの干渉に関連して14,000件のインシデントを記録しました。 1年後、組織のおかげで、インシデントの総数は12,000件に減少しました


写真-Chris Dickens -Unsplash

また、大企業が模範となり、他の企業をMANRSに引き付けると考えられています。すでに最初の結果があります-最近China Telecomオペレーターがイニシアチブに参加しました。彼は頻繁なルーティングエラーで非難されました。昨年、同社は誤ってヨーロッパのいくつかの通信事業者のトラフィックをネットワーク経由で送信しました。 Andrei Robachevskyによれば、China Telecom自体がコミュニティに訴え、今後同様のインシデントを防止する手助けをするとのことです。

大企業がMANRSに参加するのはこれが最後ではないという理由があります。

これは、クラウドプロバイダーが接続する唯一のプロジェクトではないことに注意してください。 3月末、80社以上の新会社が加盟Linux Foundationと主要なWestern IaaSプロバイダーのサポートにより形成されたCNCF（Cloud Native Computing Foundation）。組織は、情報セキュリティを確保するためのオープンクラウドサービスとツールの開発に向けてリソースを投入します。
