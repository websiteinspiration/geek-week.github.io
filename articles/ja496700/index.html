公開された後、私は1人のHabrリーダーから連絡を受けました。彼は手紙の中で、過失により私の記事を子供たちに見せ、書かれたとおりにすべてを行うことを約束しなければならなかったと書いています。今、私には2人の子供の父親として、これらの記事のシリーズを終了しないという道徳的権利はありません。したがって、行きましょう。

最初の部分では、仮想サーバーを作成し、最初のMinecraftサーバーをPaperにダウンロードして起動しました。記事の公開直後にすべての手順を実行した人は、その後のいくつかの修正に注意してください。したがって、コメントのアドバイスに従って、仮想サーバー上に別のユーザーを作成しました。したがって、ユーザーを作成した後のすべてのアクションは、彼の下から実行されます。ユーザーのrootアカウントからアプリケーションを実行することは悪い習慣であり、実行中のアプリケーションへのサーバーへの無制限のアクセスを提供するため、単に安全ではありません。

これでMinecraftサーバーが実行されたので、生成された世界地図をサーバーロビーに適したものに置き換えると便利です。ロビーはメインサーバーである名刺であり、プレイヤーのエントリポイントとして機能します。ここから、プレイヤーはミニゲームやサバイバルサーバーなどの他のサーバーにリダイレクトされます。

サーバーにマップをインポートする方法はいくつかあります。

最も簡単なのは、PCからサーバーフォルダーにワールドフォルダーをコピーすることです。この方法は、シングルプレイヤーゲームで自分でマップを作成した場合や、すぐに使用できる形式でインターネットからダウンロードした場合に適しています。コンピューターでゲームによって保存されたマップファイルを見つける方法については、こちらを参照してください。

2番目の方法は、回路図の拡張子を持つカードをインポートすることです。これは、マップの一部を格納し、MCEdit、WorldEdit、FastAsyncWorldEditなどのサードパーティプログラムで使用するためにコミュニティによって開発された形式です。2 番目の方法を使用してマップをインポートします。これは時間がかかるため、将来このプロセスを知る必要があるかもしれません。世界の既製のアーカイブを使用する場合は、このアイテムをスキップして、世界のフォルダーをサーバーフォルダー〜/ PirateBay /にコピーするだけです。



したがって、たとえば、サーバー用に購入したすべてのカードはシェマティック形式で提供されていました。また、この方法では、インポートしたマップを編集できます。たとえば、個々のパーツをカットまたはコピーできます。購入したファイルにBedWarsの4セットのアイランドがあり、それらをサーバーに1セットずつバラバラに分割する必要があるときに必要でした。

回路図ファイルを使用してサーバーマップを作成する場合、マップをインポートする空のワールドが必要です。私は通常、2種類の空の世界を使用します。1つ目は1ブロックの世界、2つ目は海です。最初のマップは、SkyBlock、BedWars、SkyWarsなどの空中の島でゲームの世界を作成するのに適しています。海は水に浮かぶ島々のある地図を作成するのに適しています。
インターネットでは、空の世界または海を作成する方法に関するチュートリアルを見つけることができます。記事のサイズを小さくするために、作成したマップを添付します。

まず、空のマップでサーバーを起動し、FastAsyncWorldEditプラグインを使用して、必要なマップを空の世界にインポートします。

プラグインの追加のインストールと編集の便宜のため、そして記事のコマンド数を減らすために、私たちはサーバーに接続するFTPクライアントをインストールします。FileZillaをインストールしました。

FTPクライアントをインストールしたら、サーバーに接続し、pluginsフォルダーに移動します。私はこれ/ホーム/ dinikin / PirateBay /プラグインを持っています。簡潔さと汎用性のために、〜/ PirateBay /プラグインを作成します。FastAsyncWorldEdit



プラグインのjarファイルをダウンロードしますサーバーのプラグインフォルダーにコピーします。空のVoid.zipカードのアーカイブもダウンロードし、解凍してVoidフォルダーを〜/ PirateBayサーバーのMinecraftフォルダーに配置します。

ターミナルからMinecraftサーバーを直接起動した場合、セッションが閉じられた後、プロセスは自動的に終了し、サーバーは停止します。今後何が起こっても、screenプログラムを使用してサーバーを起動します。

sshを介してサーバーに移動します。

ssh -i ~/minecraft dinikin@ip_address

プログラムがインストールされているかどうかを確認します。

screen --version

プログラムのバージョンが返された場合はインストールされ、それ以外の場合はインストールされます。

sudo apt-get install screen

そして、Minecraftサーバーを起動するためのstart.shファイルを作成します。

cd ~/PirateBay/
nano start.sh

次の内容で：

#!/bin/sh

screen -S PirateBay java -Xms1G -Xmx1G -XX:+UseConcMarkSweepGC -jar Paper-1.15.2.jar

ファイルに実行する権利を与えます：

chmod +x start.sh

端末セッションのサーバーがこの段階ですでに起動されている場合は、stopコマンドを使用してサーバーを停止します。

再起動する前に、サーバーを新しいカードに再構成する必要があります。 FTPクライアントから〜/ PirateBay / server.propertiesファイルを開き、次のパラメーター値を設定します。

level-name=Void
generate-structures=false

PCから1つのゲームのカードを使用した場合、またはインターネットからアーカイブをダウンロードした場合は、Voidの代わりに、〜/ PirateBay /フォルダーにコピーした後、ワールドファイルを含むフォルダーの名前を指定します。

その後、我々は以前に生成されたサーバー上の世界でのフォルダを削除：

それらを残すことができますが、私は、私はいくつかのプラグインや設定ファイルのデフォルトの世界の名前を変更しなかった場合、彼らは将来的には、サーバによって使用されないようにそれらを削除することを好みます。

そして、shスクリプトを使用してサーバーを起動します。

./start.sh

これで、ターミナルセッションが中断した場合、次のコマンドで実行中のサーバーのリストを確認できます。

screen -ls

次のコマンドで接続します。

screen -r PirateBay

または

screen -r 1234

1234は、screen -lsコマンドの結果からのプロセスIDです。

実際、PirateBayの代わりに、実行中のサーバーを明確に理解できるプロセスの名前を指定できます。
