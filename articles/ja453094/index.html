MD5ハッシュの数字頻度分析

私たちは皆、ハッシュがどのように見えるかを知っていますが、特定の文字がハッシュ内でどのくらいの頻度で出現するのか疑問に思ったことはありますか？私は疑問に思いました。そして、チェックすることにしました。私は数えるPythonスクリプトをスケッチしました、そしてそれはそれから来たものです。

まず、ランダムな文字列（長さ0〜1000）を生成しました。

def random_string(from_int, to_int):
    return str(''.join(random.SystemRandom().choice(string.ascii_letters + string.digits + string.punctuation) for _ in range(random.randint(from_int, to_int))))

次に、文字列からMD5ハッシュを取得しました。 

def md5_from_string(string):
    return hashlib.md5(string.encode('utf-8')).hexdigest()

後-ハッシュに含まれる0から9までの桁数を計算しました。1000のハッシュのサンプルで、次のデータを受け取りました。



ここで、最も頻繁に遭遇する数字と最もまれにしか発見されない（差分値）の違いが興味深いです。



さらに、デルタ値の変化を追跡するために、10,000、100,000、1,000,000、10,000,000ハッシュのサンプルを作成しました。



以下は、最小数と最大数の値、およびMD5ハッシュの数が異なるサンプルのデルタ値のリストです。

  • 100-最小：179、最大：230、デルタ：22.17％
  • 1000-最小：1925、最大：2058、デルタ：6.46％
  • 10000-最小：19769、最大：20251、デルタ：2.38％
  • 100000-最小：199297、最大：200846、デルタ：0.77％
  • 1,000,000-最小：1997650、最大：2001690、デルタ：0.20％
  • 10000000-最小：19991830、最大：20004818、デルタ：0.06％

私たちが持っているもの：配列内のハッシュの数が増加すると、デルタ値が減少し、ほぼ同じ確率の数字が配列に分類されます。したがって、サンプルが大きいほど、頻繁に発生する数とほとんど検出されない数の差が小さくなります。したがって、ハッシュで特定の数字を取得する確率は均一になる傾向があります。
この情報は、bepeam.com競争プラットフォームに実装したアルゴリズムの基礎を形成しました
