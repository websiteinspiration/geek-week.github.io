AlertDisting for Elasticsearchを使用してファイルの変更を追跡する

今日、サーバー上の特定のファイルへの変更を監視する必要がありました。たとえば、facebookからのosqueryなど、さまざまな方法がありますが、最近Open Distro for Elasticsearchを使い始めたので、beat'ovの 1つであるエラスティックでファイルを監視することにしました。


ElasticsスタックとAuditbeatのインストールについては説明しません。すべてのマニュアルによると、インストール後、auditbeat.ymlファイルを編集し、監視対象ファイルへのパスをfile_integrityモジュールに追加するだけです。


設定して起動すると、kibanaはauditbeat- *インデックスを表示します



次に、モニタリングを作成し、モニタリングの名前、スキャン間隔、モニタリングのタイプとインデックスファイルを指定します。



抽出の定義クエリで、次のように記述します。


抽出クエリを定義する
{
    "query": {
        "bool": {
            "must": [
                {
                    "match_phrase": {
                        "file.path": {
                            "query": "</  >"

                        }
                    }
                }
            ],
            "filter": [
                {
                    "term": {
                        "event.action": {
                            "value": "attributes_modified"    # ,  created   deleted

                        }
                    }
                },
                {
                    "range": {
                        "@timestamp": {
                            "from": "now-1m" #     

                        }
                    }
                }
            ],
            "adjust_pure_negative": true,
            "boost": 1
        }
    }
}

[実行]ボタンを押してリクエストを確認すると、次のように表示されます。



ターゲットファイルを変更して、もう一度リクエストを実行しようとします。



2で変更されたヒットを確認できるので、[更新]をクリックしてトリガーを作成し、値を変更します。



写真のようにすべてを残します。


次に、通知をスラックまたは別のメッセンジャーで構成できます。
