最初に、この資料の最後に到達したときに達成できることを見てみましょう。

//  '@@'   `foo`  
function @@ foo(a, b, c) {
  return a + b + c;
}
console.log(foo(1, 2)(3)); // 6

関数をカリー化@@できる 構文構造を実装します。この構文は、ジェネレーター関数の作成に使用される構文と似ていますが、この場合、記号の代わりに、キーワードと関数の名前の間に一連の文字が配置されます。その結果、関数を宣言するときに、ビューコンストラクトを使用できます。 上記の例では、関数を操作するときに、その部分的なアプリケーションを使用できます。必要な引数の数よりも少ない数のパラメーターをそれに渡す関数呼び出しは、残りの引数を取ることができる新しい関数を返します。*function@@function @@ name(arg1, arg2)

foo(1, 2, 3); // 6

const bar = foo(1, 2); // (n) => 1 + 2 + n
bar(3); // 6

@@変数名に文字を使用できないため 、文字のシーケンスを正確に選択しました@。これは、ビューコンストラクトも構文的に正しいことを意味しますfunction@@foo(){}。また、デコレータ機能に@は「演算子」を使用しており、まったく新しいものを使いたかった。その結果、私はデザインを選びました。 目標を達成するために、以下のアクションを実行する必要があります。@@

コンパイラについてもっと知りたい人のための素晴らしいリソースがあります。

「コンパイラ」が非常に複雑で理解できないものだと思う場合は、実際にはすべてがそれほど神秘的ではないことを知っておいてください。コンパイルとは、単にコードを解析し、それに基づいて新しいコードを作成することです。これをXXXと呼びます。XXXコードは、マシンコードで表すことができます（おそらく、マシンコードは、コンパイラーについて考えるときに、私たちの頭の中で最初に浮上するものです）。これは、レガシーブラウザと互換性のあるJavaScriptコードである可能性があります。実際、Babelの主な機能の1つは、最新のJSコードを古いブラウザでも理解できるコードにコンパイルすることです。
