CLionニュース：2020.1リリース、5年間のIDE、オンラインQ＆A

こんにちは、ハブル！

今日は、何から始めればよいかを選択するのがさらに難しいことを伝えたいと思います。したがって、最初に、最も重要なものすべてについて簡単に説明します。

  1. CLionは昨日5歳です！休暇を記念して、私たちは小さなビデオストーリー、奇妙な記憶、この5年間にすべてが私たちのために開発されてきたおもしろさについてのストーリーをマウントしました。話をすぐに英語で警告する：


  2. 昨日、この製品の最初の大きなリリースが今年起こった-CLion 2020.1。その中で、CUDAダイアレクトをサポートし、Dataflow AnalysisをClangに書き直し、IDEにClang-clおよびIARコンパイラーと連携するように教え、さらに他の多くの改善と修正を行いました。
  3. 過去数か月間、私たちのチームは会議や関連する展示会がなければ非常に悲しくなりました。そこでは常に会社の立場にいて、ユーザーやコミュニティ全体と多くのことをコミュニケーションできることを嬉しく思っています。そのため、CLion Ask Me Anything（AMA）セッションというチームとのオンラインQ＆Aセッションを開催することにしました。セッションは5月7日にリアルタイムで開催されます。事前の予約が必要です。参加して、製品に関する質問をしてください！

次に、これらのすべてのイベントについて詳しく説明します。

CLion-5つ！


子供の成長は常に驚くべきことであり、両親にとってはしばしば予期せぬことです。ストーリーは製品と同じです。昨日、初期バージョンをリリースし、リリース1.0までカウントダウンしていたようですが、今日はすでにラウンドデートを祝っています。 CLionには現在、BMW、NASA、Tencent、Teslaなどの企業のチームを含む月間ユーザーが20万人以上います。もちろん、GoogleやAndroid Studioとの協力も行っています。

CLionを使用してソフトウェア開発の最初のステップを実行している世界中の多くの学生をサポートできることを嬉しく思います。そして、2019年の決勝のICPCチームの3分の1もCLionを使用したことを知ってとても嬉しいです！



新しいリリースごとの興奮、膨大な数のユーザーへの責任、素晴らしい仕事をするチームの団結-これらすべてが私たちを圧倒し、Habrコミュニティ全体と最初の小さな記念日を共有できることを嬉しく思います！私たちはあなたが映画を楽しんだことを願っています;）

CLion 2020.1


過去5年間の規模としては最大ではないかもしれませんが、昨日は重要なリリースを確実にリリースしました。最も重要な変更について順番に話しましょう：

  • CUDA方言サポート。
  • 組み込み開発者向けに、IARコンパイラサポートおよびPlatformIOとの統合のためのプラグイン。
  • Clangツールとの更新された統合-DFAからClangdへの移行、自動補完の改善、Clang-TidyおよびClangFormatとの密接な連携。
  • Windowsの開発者向け-Clang-clコンパイラとデフォルトデバッガのサポート。
  • シグネチャの変更リファクタリングのデフォルトのパラメータ値。
  • 入力のリダイレクトやIDEマクロのサポートなど、より便利な起動およびデバッグ構成。
  • フォーマッターの更新、エディターの改善など。

クーダ


CLionはCUDA C / C ++方言をサポートするようになりました。サポートに含まれるもの：

  • 正しいコードパーサー（「赤い」コードとコードアナライザーの誤った操作を除外するため）;
  • ナビゲーションやドキュメントの表示など、さまざまなスマートアクションを強調表示します。
  • 新しいプロジェクトを作成するための更新されたウィザード-CUDAプロジェクトのテンプレートが含まれるようになりました。

  • CUDAファイル拡張子のサポート（.cu / .cuh）;
  • CUDAの特別なCMakeターゲット-新しいファイルを追加するためのダイアログは、通常のCMakeコマンドによって作成された適切なターゲットだけでなく、CUDAに固有のファイル（cuda_add_executable/ cuda_add_library）を追加するようになりました。

Qtのような他の特定の領域ではなく、なぜCUDAに重点を置くのかと尋ねられました。要するに、もちろん、私たちはIDEですべてのタイプのC / C ++プロジェクトをサポートしたいと考えています。ただし、リソースが限られている状況では、選択を行い、最も普遍的な機能のサポートに重点を置く必要があります。CUDAなどの方言は、「そのまま」サポートされていない場合、赤のコード、コードアナライザーエラー、およびIDEの基本機能に関するその他の問題につながります。Qtとは異なり、一般的なパーサーはコードを処理し、特定の機能を欠いています。

組み込み開発者向け


組み込みシステムの開発者をサポートし続けます。IAR Systems ABとの協力により、パートナーシップライセンスを取得し、CLionにIARコンパイラサポートを追加することができました。以前は、コンパイラから情報を正しく読み取ることができず、そのようなプロジェクトはCLionで正しくロードされませんでした。今、すべてがうまくいきます！



コンパイラを使用するには、MinGWツールチェーンを使用する必要があることに注意してください。そして、ここでは、IAR Embedded WorkbenchでCMakeを使用する方法に関するいくつかのヒントを見つけることができます。

私たちは引き続きIAR Systems ABと協力していきます。うまくいけば、いつの日かCLionも彼らのデバッガをサポートするでしょう。

最初のCLionリリースから、PlatformIOの作成者と話しました組み込みシステムのプロジェクトを迅速に開始するためのユニークなエコシステム。たとえば、彼女はArduinoで最初のプロジェクトを開始するのを助けました。 2020.1リリースでは、かなり基本的ですが、CLionプラグイン用の便利なPlatformIOを収集しました。彼の最も重要な機会は、CMakeに基づいてPlatformIOのプロジェクトを作成することです。新しいプロジェクトの作成ウィザードでこのタイプのプロジェクトを選択するだけです

更新されたClangツールの統合


既にご存じのとおり、Clangdベースの言語エンジンには、ますます多くの機能を積極的に移植しています。1つ目は、新しい言語機能のサポートをすばやく取得してIDEアクションをそれらに適応させることです。2つ目は、場合によってはエディターのパフォーマンスを向上させることです（残念ながら、Clangdは非常に普遍的なツールです）。

2020.1リリースでは、Clangdベースのオートコンプリートの修正に多くの時間を費やしました（バージョン2019.3で登場）。そして、このアプローチの結果がかなり良いと判断したため、自動補完のオプションがすべてClangdから取得されるデフォルトモードをオンにしました。設定で混合動作モードを有効に戻すことができます。



ただし、2つの異なるプロバイダーの結果を組み合わせると優先順位の問題が発生する可能性があるため、これを正当な理由なしに行うことはお勧めしません。さらに、私たちの測定から判断すると、Clangdに基づく新しいオートコンプリートは遅くはありませんが、多くの場合、組み込みのコンプリートよりも高速です。

Clangdは、このリリースでデータフロー分析（DFA）も移動しました。このアナライザーは、コンパイラーが通常行わないことを実行できるため、別の記事に値します（理論的には、側で再現することは可能ですが、主にそれらを必要としないため）-データが制御フローグラフに沿って移動する方法を分析し、この知識に基づいて、潜在的なエラーを見つけます。たとえば、常にtrueまたはfalseである条件、無限の再帰とループ、欠落した戻り式、未使用の値、パラメーター、およびローカル変数：



今ではすべてClangdで動作します。これにより、プロジェクトのコード分析のパフォーマンスが向上することを願っています。残念ながら、以前は、エディターのパフォーマンスを向上させるために、DFA分析をオフにするようユーザーにアドバイスする必要がありました。この場合は、アナライザーをオンに戻すことをお勧めします。テクニカルサポートとトラッカーでフィードバックをお待ちしています。

さらに、Clangツールは、Clang-TidyおよびClangFormatのサポートの改善に注意する必要があります。

Windows


LinuxおよびmacOSでCLionツールチェーンを選択するのが非常に簡単である場合（GCCまたはClangのいずれかを使用）、Windowsには、Cygwin、MinGW / Mingw-w64、WSL、Visual Studioなど、理解しにくい多くのオプションがあります。同時に、多くの場合、各ツールチェーン内で異なるコンパイラーが使用されます。 WindowsでClangを使用する可能性について長い間尋ねられてきました。 CLion 2020.1では、Clang-clを（公式のLLVM Webサイトから、またはVisual Studioツールと一緒に）取得して、Visual Studioツールチェーン内で使用できるようになりました。



ちなみに、このダイアログでデフォルトで選択されているデバッガに気づきましたか？私たちのチームが開発しており、昨年導入したMicrosoft Visual C ++ツールチェーン用の同じLLDBベースのデバッガーがデフォルトで新しいバージョンに含まれています！ただし、ネイティブビジュアライザーのサポートも使用する場合は、設定で明示的に有効にする必要があります。ビルド、実行、展開| デバッガーのデータビュー| LLDBのNatVisレンダラーを有効にします。

変更署名のリファクタリング


CLionの多くのリファクタリングの中で、署名の変更が際立っています（WindowsおよびLinuxではCtrl + F6、macOSでは⌘F6）。関数名、戻り値の型、パラメーターのリストを変更できます。そうすることで、IDEは、コードの正確さを維持するために、関数のすべての使用を更新します。しかし、新しいパラメーターを追加するとどうなるでしょうか。以前は、CLionはすべての用途（存在する場合）- 0数値nullptr用、ポインター用にtypeのデフォルト値を設定していました。バージョン2020.1以降、リファクタリングダイアログでCLionが関数を直接使用する代わりに値を指定


できるようになりました。何も指定しない場合、動作は以前と同じになります。

起動およびデバッグ構成


CLionのスタートアップおよびデバッグ構成に対する多くの小さな改善により、それらはより使いやすくなりました。

まず、リモートGDBサーバーと組み込みGDBサーバーの構成は、CMakeターゲットだけでなく、カスタムターゲットでも機能するようになりました。したがって、CMakeベースのプロジェクトの場合だけでなく、（プロジェクトのビルドおよびクリーンアップコマンドを指定することによって）カスタムターゲットを構成した他のタイプのプロジェクトについても、リモートマシンまたはマイクロコントローラーでデバッグできるようになりました。

次に、CMake、カスタムビルド、Gradleネイティブアプリケーションなどの構成では、マクロとパス変数を使用できるようになりました。彼らの助けを借りて、あなたは多くの有用な値にアクセスできます：



そして最後に、プログラム入力をリダイレクトするオプションがこれらのタイプの構成に登場しました。Prompt / FilePromptマクロと組み合わせて使用​​すると、入力をファイルからプログラムにリダイレクトするだけでなく、構成の開始時に入力するファイルを選択するためのダイアログを表示することもできます。



フォーマッターの更新、エディターの改善、その他


さまざまなサブシステムにおけるいくつかの小さな点の改善について言及する価値があります。

  • 構造体フィールドとクラスフィールドの個別の命名設定。
  • #pragma region/で強調表示されたコードブロックを折りたたむ機能#pragma endregion。
  • ドキュメント表示ウィンドウは、ショートカットで呼び出すだけでなく、目的のシンボルにカーソルを合わせるだけで表示することもできます。
  • IDEのデフォルトフォントはJetBrains Monoになりました。これは、開発者がコードを読みやすくするために特別に設計されたJetBrainsの新しい等幅フォントです。
  • デフォルトのLightテーマは、3つのプラットフォームすべてで同じになりました-IntelliJ Light。
  • 端末セッションは、水平または垂直にいくつかに分割できます。したがって、1つのタブで複数のセッションを同時に表示できます。
  • IntelliJプラットフォームから、バージョン管理システムとの統合による改善も行われました。
