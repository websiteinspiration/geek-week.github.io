Sesuaikan lingkungan Python Anda dengan pyenv, virtualenvwrapper, tox dan pip-compile





Alat-alat ini akan menyederhanakan pengaturan dan mengotomatiskan operasi rutin. Mereka akan menyelamatkan pengembang dari banyak kesulitan yang mencegah Anda fokus pada pemecahan masalah dan kode penulisan yang nyaman.



Ada banyak cara untuk menyesuaikan lingkungan Python Anda. Artikel ini tentang salah satunya. Tapi ini, tentu saja, bukan satu-satunya solusi.



Python adalah bahasa pemrograman tujuan umum yang sering direkomendasikan untuk pemula. Selama dua puluh tahun keberadaannya, beberapa buku tentang Python telah ditulis. Meskipun sering disebut sebagai sederhana, menyiapkan alat pengembangan Python jauh dari tugas yang mudah.









Menyiapkan lingkungan Python cukup rumit: xkcd



Gunakan pyenv untuk kontrol versi



Menurut pendapat saya, ini adalah manajer versi Python terbaik. Namun, perlu dicatat bahwa pyenv berjalan di Linux, Mac OS X, dan WSL2: yaitu, tiga lingkungan "mirip UNIX".



Instalasi pyenv itu sendiri kadang-kadang bisa sulit. Tetapi penginstal pyenv khusus yang menggunakan curl | dapat membantu. bash untuk bootstrap.



Jika Anda menggunakan Mac (atau sistem lain tempat Anda menjalankan Homebrew), Anda dapat membaca instruksi untuk menginstal dan menggunakan pyenv di sini .



Setelah menginstal dan mengkonfigurasi pyenv, Anda dapat menggunakan pyenv global untuk menginstal versi standar Python. Anda dapat memilih versi "favorit" Anda. Biasanya ini adalah versi stabil terbaru, tetapi tidak akurat -)



Gunakan virtualenvwrapper untuk mengatur lingkungan virtual



Untuk Python, Anda dapat membuat lingkungan virtual, yaitu, menempatkan setiap proyek di lingkungan yang terisolasi. Ini dapat dilakukan dengan menggunakan virtualenvwrapper. Maka menjadi mungkin untuk beralih di antara lingkungan virtual kapan saja.



Baca lebih lanjut tentang pengaturan lingkungan virtual di sini .



Jika Anda perlu berulang kali membuat dan menggunakan lingkungan virtual yang sama, masuk akal untuk mengotomatiskan proses. Hari ini pilihan saya adalah racun.



Gunakan racun untuk mengotomatisasi



Tox adalah alat yang hebat untuk mengotomatisasi tes Anda. Di setiap lingkungan Python, saya membuat file tox.ini. Apa pun sistem yang saya gunakan untuk integrasi berkelanjutan, semuanya akan berfungsi. Dan saya dapat menjalankannya secara lokal dengan virtualenvwrapper: Masalahnya, saya menguji kode saya pada beberapa versi Python dan beberapa versi dependensi perpustakaan. Ini berarti bahwa racun akan bekerja di banyak lingkungan. Beberapa dari mereka akan memiliki dependensi aktual. Dalam beberapa - beku. Saya juga bisa membuatnya secara lokal menggunakan pip-compile. Baru-baru ini, saya telah mempertimbangkan nox sebagai pengganti racun. Dalam artikel ini saya tidak akan menjelaskan alasannya, tetapi Anda harus memperhatikannya.

Gunakan pip-compile untuk mengelola dependensi



Setiap kali Python memuat dependensinya pada tahap eksekusi kode. Memahami dan mengendalikan versi dependensi mana yang digunakan akan membantu untuk menghindari crash yang tidak terduga saat program sedang berjalan. Ini berarti bahwa kita harus memikirkan alat manajemen ketergantungan.



Untuk setiap proyek baru, saya menyertakan file require.in, yang (sebagai aturan) hanya berisi satu karakter:



.







Ya, semuanya benar di sini. Sebuah string dot tunggal. Dalam file setup.pyfile untuk dependensi, saya menulis Twisted> = 17.5 bukannya Twisted == 18.1, yang membuatnya sulit untuk ditingkatkan ke versi perpustakaan yang baru.



"." berarti "direktori saat ini" di mana file setup.py yang sesuai digunakan sebagai sumber informasi ketergantungan.



Ini berarti bahwa ketika menggunakan pip-kompilasi requirement.in> requirement.txt, sebuah file dengan dependensi beku akan dibuat. Anda dapat menggunakan file dependensi ini baik di lingkungan virtual yang dibuat oleh virtualenvwrapper atau di tox.ini.



Terkadang file persyaratan-dev.txt dihasilkan dari requirement-dev.in (konten :. [Dev]) atau requirement-test.txt yang dihasilkan dari requirement-test.in (konten :. [Tes]) dapat berguna.



Atau pip-compile dapat menggunakan dephell . Alat dephell memiliki banyak hal menarik, misalnya, menggunakan permintaan HTTP asinkron untuk memuat dependensi.



Keluaran



Saya menganggap Python bahasa yang kuat dan indah. Namun demikian, untuk bekerja secara produktif, saya menggunakan seperangkat alat tertentu yang telah membuktikan keefektifannya, setidaknya untuk saya. Jadi gunakan empat alat ini - pyenv, virtualenvwrapper, tox dan pip-compile - dan Anda akan senang =)
