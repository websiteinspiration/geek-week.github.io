Goアプリケーション用のMakefileの作成例

このチュートリアルでは、Go開発者がMakefileを使用して独自のアプリケーションを開発する方法を見ていきます。



Makefileとは何ですか？


Makefileは、Goだけでなく他のほとんどのプログラミング言語でもアプリケーションを実行および構築するために使用できる、非常に便利な自動化ツールです。

多くの場合、GithubおよびGitlab上の多くのGoアプリのルートディレクトリに表示されます。多くの場合、開発者に伴うタスクを自動化するためのツールとして広く使用されています。

Goを使用してWebサービスを作成する場合、Makefileは次の問題の解決に役立ちます。

  • コンパイル、開始、停止、監視などの単純なコマンドの呼び出しの自動化
  • プロジェクト固有の環境変数を管理します。.envファイルを含める必要があります。
  • 変更時に自動的にコンパイルされる開発モード。
  • コンパイルエラーを示す開発モード。
  • 特定のプロジェクトのGOPATHを定義して、依存関係をベンダーフォルダーに格納できるようにします。
  • 簡略化されたファイル監視。たとえば、watch run =“ go test。/ ... "

プロジェクトの一般的なディレクトリ構造は次のとおりです。

.env
Makefile
main.go
bin/
src/
vendor/

このディレクトリでmakeコマンドを呼び出すと、次の出力が得られます。

$  make

 Choose a command run in my-web-server:

 install   Install missing dependencies. Runs `go get` internally.
 start     Start in development mode. Auto-starts when code changes.
 stop      Stop development mode.
 compile   Compile the binary.
 watch     Run given command when code changes. e.g; make watch run="go test ./..."
 exec      Run given command, wrapped with custom GOPATH. e.g; make exec run="go test ./..."
 clean     Clean build files. Runs `go clean` internally.

環境変数


Makefileに最初に必要なのは、プロジェクトに定義した環境変数を含めることです。したがって、最初の行は次のようになります。

include .env

次に、プロジェクト名、Goフォルダー/ファイル、pidへのパスを定義します...

PROJECTNAME=$(shell basename "$(PWD)")

# Go .
GOBASE=$(shell pwd)
GOPATH=$(GOBASE)/vendor:$(GOBASE):/home/azer/code/golang  #       .
GOBIN=$(GOBASE)/bin
GOFILES=$(wildcard *.go)

#     ,       .
STDERR=/tmp/.$(PROJECTNAME)-stderr.txt

# PID-    ,      
PID=/tmp/.$(PROJECTNAME)-api-server.pid

# Make     Linux.   silent.
MAKEFLAGS += --silent

Makefileの残りの部分では、GOPATH変数を使用することがよくあります。すべてのチームは特定のプロジェクトのGOPATHに関連付けられている必要があります。関連付けられていない場合、チームは機能しません。これにより、プロジェクトが完全に分離されますが、同時に作業が複雑になります。タスクを簡略化するために、GOPATHで任意のコマンドを実行するexecコマンドを追加できます。

# exec:     GOPATH. :  make exec run = " go test ./...”
exec:
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) $(run)

ただし、makefileに書き込むことができない処理を実行する必要がある場合にのみ、execを使用する必要があることを覚えておいてください。

開発モード


開発モードは次のとおりです。

  • ビルドキャッシュをクリア
  • コードをコンパイルする
  • バックグラウンドでサービスを実行する
  • コードが変更されたら、これらの手順を繰り返します。

それは簡単に聞こえます。ただし、問題は、サービスとファイルウォッチャーの両方を同時に実行することです。新しいプロセスを開始する前に、プロセスが正しく停止することを確認する必要があります。Control-CまたはControl-Dを押したときに、通常のコマンドラインの動作が損なわれないようにする必要があります。

start:
	bash -c "trap 'make stop' EXIT; $(MAKE) compile start-server watch run='make compile start-server'"

stop: stop-server

上記のコードは、次のタスクを解決します。

  • バックグラウンドでサービスをコンパイルして実行します。
  • メインプロセスはバックグラウンドで実行されないため、Control-Cを使用して中断できます。
  • メインプロセスが中断されたときにバックグラウンドプロセスを停止します。トラップはこのためだけに必要です。
  • コードが変更されたときにサーバーを再コンパイルして再起動します。

次のセクションでは、これらのコマンドについて詳しく説明します。

コンパイル


コンパイルコマンドは、バックグラウンドでgo compileを呼び出すだけではありません。エラー出力をクリアし、簡易バージョンを出力します。

これは、破壊的な編集を行ったときのコマンドライン出力です。



compile:
	@-touch $(STDERR)
	@-rm $(STDERR)
	@-$(MAKE) -s go-compile 2> $(STDERR)
	@cat $(STDERR) | sed -e '1s/.*/\nError:\n/'  | sed 's/make\[.*/ /' | sed "/^/s/^/     /" 1>&2

サーバーの起動/停止


start-serverは、バックグラウンドでコンパイルされたバイナリを起動し、そのPIDを一時ファイルに保存します。stop-serverはPIDを読み取り、必要に応じてプロセスを強制終了します。

start-server:
	@echo "  >  $(PROJECTNAME) is available at $(ADDR)"
	@-$(GOBIN)/$(PROJECTNAME) 2>&1 & echo $$! > $(PID)
	@cat $(PID) | sed "/^/s/^/  \>  PID: /"

stop-server:
	@-touch $(PID)
	@-kill `cat $(PID)` 2> /dev/null || true
	@-rm $(PID)

restart-server: stop-server start-server

変更監視


変更を追跡するには、監視ファイルが必要です。たくさん試しましたが、適切なものが見つからなかったので、独自のファイル監視ツール-yoloを作成しました。次のコマンドを使用してインストールします。

$  go get github.com/azer/yolo

インストール後、ベンダーとbinフォルダーを除いて、プロジェクトディレクトリの変更を確認できます。

## watch:      ,  make watch run="echo 'hey'"
watch:
	@yolo -i . -e vendor -e bin -c $(run)

これで、ベンダーディレクトリを除いて、プロジェクトディレクトリへの変更を再帰的に追跡する監視コマンドがあります。実行するコマンドを渡すだけです。
たとえば、コードが変更されると、startはmake-start-serverを呼び出します。

make watch run="make compile start-server"

これを使用して、テストを実行したり、競合状態を自動的にチェックしたりできます。環境変数は実行時に設定されるため、GOPATHを気にする必要はありません。

make watch run="go test ./..."

Yoloの 素晴らしい機能は、そのWebインターフェースです。有効にすると、Webインターフェイスでコマンドの出力をすぐに確認できます。あなたがする必要があるのは-aオプションを渡すことだけです：

yolo -i . -e vendor -e bin -c "go run foobar.go" -a localhost:9001

ブラウザでlocalhost：9001を開き、作業の結果をすぐに確認します。



依存関係のインストール


コードに変更を加える場合は、不足している依存関係をコンパイル前にロードする必要があります。installコマンドは私たちのために仕事をします：

install: go-get

コンパイル前にファイルが変更されたときにインストール呼び出しを自動化するため、依存関係は自動的にインストールされます。依存関係を手動でインストールする場合は、次のコマンドを実行できます。

make install get="github.com/foo/bar"

内部的には、このコマンドは次のように変換されます。

$ GOPATH=~/my-web-server GOBIN=~/my-web-server/bin go get github.com/foo/bar

使い方？高レベルのコマンドを実装するために通常のGoコマンドを追加する次のセクションを参照してください。

Goコマンド


Goエコシステムではまだ正式に解決されていない依存関係管理を簡略化するために、プロジェクトディレクトリにGOPATHをインストールする必要があるため、すべてのGoコマンドをMakefileでラップする必要があります。

go-compile: go-clean go-get go-build

go-build:
	@echo "  >  Building binary..."
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go build -o $(GOBIN)/$(PROJECTNAME) $(GOFILES)

go-generate:
	@echo "  >  Generating dependency files..."
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go generate $(generate)

go-get:
	@echo "  >  Checking if there is any missing dependencies..."
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go get $(get)

go-install:
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go install $(GOFILES)

go-clean:
	@echo "  >  Cleaning build cache"
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go clean

助けて


最後に、利用可能なコマンドのリストを表示するには、helpコマンドが必要です。sedおよびcolumnコマンドを使用して、美しいフォーマットのヘルプ出力を自動的に生成できます。

help: Makefile
	@echo " Choose a command run in "$(PROJECTNAME)":"
	@sed -n 's/^##//p' $< | column -t -s ':' |  sed -e 's/^/ /'

次のコマンドは、Makefileをスキャンして##で始まる行を探し、それらを表示します。このように、特定のコマンドにコメントを付けるだけで、helpコマンドでコメントが表示されます。

いくつかコメントを追加すると：

## install: Install missing dependencies. Runs `go get` internally.
install: go-get

## start: Start in development mode. Auto-starts when code changes.
start:

## stop: Stop development mode.
stop: stop-server

取得します：

$  make help

 Choose a command run in my-web-server:

 install   Install missing dependencies. Runs `go get` internally.
 start     Start in development mode. Auto-starts when code changes.
 stop      Stop development mode.

最終版


include .env

PROJECTNAME=$(shell basename "$(PWD)")

# Go related variables.
GOBASE=$(shell pwd)
GOPATH="$(GOBASE)/vendor:$(GOBASE)"
GOBIN=$(GOBASE)/bin
GOFILES=$(wildcard *.go)

# Redirect error output to a file, so we can show it in development mode.
STDERR=/tmp/.$(PROJECTNAME)-stderr.txt

# PID file will keep the process id of the server
PID=/tmp/.$(PROJECTNAME).pid

# Make is verbose in Linux. Make it silent.
MAKEFLAGS += --silent

## install: Install missing dependencies. Runs `go get` internally. e.g; make install get=github.com/foo/bar
install: go-get

## start: Start in development mode. Auto-starts when code changes.
start:
    bash -c "trap 'make stop' EXIT; $(MAKE) compile start-server watch run='make compile start-server'"

## stop: Stop development mode.
stop: stop-server

start-server: stop-server
	@echo "  >  $(PROJECTNAME) is available at $(ADDR)"
	@-$(GOBIN)/$(PROJECTNAME) 2>&1 & echo $$! > $(PID)
	@cat $(PID) | sed "/^/s/^/  \>  PID: /"

stop-server:
	@-touch $(PID)
	@-kill `cat $(PID)` 2> /dev/null || true
	@-rm $(PID)

## watch: Run given command when code changes. e.g; make watch run="echo 'hey'"
watch:
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) yolo -i . -e vendor -e bin -c "$(run)"

restart-server: stop-server start-server

## compile: Compile the binary.
compile:
	@-touch $(STDERR)
	@-rm $(STDERR)
	@-$(MAKE) -s go-compile 2> $(STDERR)
	@cat $(STDERR) | sed -e '1s/.*/\nError:\n/'  | sed 's/make\[.*/ /' | sed "/^/s/^/     /" 1>&2

## exec: Run given command, wrapped with custom GOPATH. e.g; make exec run="go test ./..."
exec:
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) $(run)

## clean: Clean build files. Runs `go clean` internally.
clean:
	@(MAKEFILE) go-clean

go-compile: go-clean go-get go-build

go-build:
	@echo "  >  Building binary..."
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go build -o $(GOBIN)/$(PROJECTNAME) $(GOFILES)

go-generate:
	@echo "  >  Generating dependency files..."
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go generate $(generate)

go-get:
	@echo "  >  Checking if there is any missing dependencies..."
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go get $(get)

go-install:
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go install $(GOFILES)

go-clean:
	@echo "  >  Cleaning build cache"
	@GOPATH=$(GOPATH) GOBIN=$(GOBIN) go clean

.PHONY: help
all: help
help: Makefile
	@echo
	@echo " Choose a command run in "$(PROJECTNAME)":"
	@echo
	@sed -n 's/^##//p' $< | column -t -s ':' |  sed -e 's/^/ /'
	@echo
