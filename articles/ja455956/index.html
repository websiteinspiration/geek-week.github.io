Chromiumはブラウザであるだけでなく、優れたフレームワークでもあります



ほとんどの人は、Chromiumがブラウザーであり、他のブラウザーの基礎でもあるという事実に慣れています。最近まで私もそう思っていましたが、このトピックを数か月間勉強して、別の素晴らしい世界を発見し始めました。Chromiumは、依存関係システム、クロスプラットフォームビルドシステム、ほとんどすべての状況に対応するコンポーネントなど、すべてが存在する巨大なエコシステムです。それでは、このすべての機能を使用して独自のアプリケーションを作成してみませんか？

カットの下で、これを始める方法についての小さなガイド。

depot_toolsの設定


depot_tools Chromium開発ツールキットです。これをインストールするには、以下を実行する必要があります。

git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git

そして、PATH環境変数にパスを追加します。

export PATH="$PATH:/path/to/depot_tools"

重要：depot_toolsホームフォルダーにダウンロードした場合~は、変数PATHで使用しないでください。使用すると、問題が発生する可能性があります。変数を使用する必要があります$HOME。

export PATH="$PATH:${HOME}/depot_tools"

コード検索


まず、ソース用のフォルダーを作成する必要があります。たとえば、ホームディレクトリでは（約30 GBの空き容量が必要です）：

その後、次の場所fetchからユーティリティを使用してソースをダウンロードできますdepot_tools。

リクエストを送信するには、CreateRequestobject メソッドを使用してオブジェクトをctx作成する必要がありますURLRequest。次のパラメーターが渡されます。

デリゲートは、インターフェイスを実装するクラスですnet::URLRequest::Delegate。このタスクでは、次のようになります。

class MyDelegate : public net::URLRequest::Delegate {
public:
   explicit MyDelegate(base::Closure quit_closure) : quit_closure_(std::move(quit_closure)),
                                                     buf_(base::MakeRefCounted<net::IOBuffer>(BUF_SZ)) {}

   void OnReceivedRedirect(net::URLRequest *request, const net::RedirectInfo &redirect_info,
                           bool *defer_redirect) override {
       std::cerr << "redirect to " << redirect_info.new_url << std::endl;
   }

  void OnAuthRequired(net::URLRequest* request, const net::AuthChallengeInfo& auth_info) override {
       std::cerr << "auth req" << std::endl;
   }

   void OnCertificateRequested(net::URLRequest *request, net::SSLCertRequestInfo *cert_request_info) override {
       std::cerr << "cert req" << std::endl;
   }

   void OnSSLCertificateError(net::URLRequest* request, int net_error, const net::SSLInfo& ssl_info, bool fatal) override {
       std::cerr << "cert err" << std::endl;
   }

   void OnResponseStarted(net::URLRequest *request, int net_error) override {
       std::cerr << "resp started" << std::endl;
       while (true) {
           auto n = request->Read(buf_.get(), BUF_SZ);
           std::cerr << "resp read " << n << std::endl;

           if (n == net::ERR_IO_PENDING)
               return;

           if (n <= 0) {
               OnReadCompleted(request, n);
               return;
           }

           std::cout << std::string(buf_->data(), n) << std::endl;
       }
   }
   void OnReadCompleted(net::URLRequest *request, int bytes_read) override {
       std::cerr << "completed" << std::endl;
       quit_closure_.Run();
   }

private:
   base::Closure quit_closure_;
   scoped_refptr<net::IOBuffer> buf_;
};

すべてのメインロジックはイベントハンドラー内OnResponseStartedにあります。エラーが発生するか、読み取るものがなくなるまで、応答の内容が差し引かれます。応答を読んだ後、アプリケーションを完了する必要があるので、デリゲートはメイン関数に割り込む関数にアクセスできる必要があります。Run loopこの場合、コールバックタイプが使用されbase::Closureます。
