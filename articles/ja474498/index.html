アプリケーションの構造


各アプリケーションは、ステージ（ウィンドウ）、（シーン）、ノード（ノード）のいくつかの主要コンポーネントの階層で構成されています。それらのそれぞれを見てみましょう。

ステージ


ステージは本質的にウィンドウです。アプリケーションには複数のステージコンポーネントがある場合がありますが、少なくとも1つは必要です。

シーン


シーンは、シーン（ステージ）の内容を表示します。各ステージコンポーネントには、切り替え可能な複数のシーンコンポーネントを含めることができます。パフォーマンス中にいくつかのシーンが入れ替わる劇場ステージを想像してみてください。

ノード


各ステージコンポーネントには、ノードと呼ばれるさまざまなコンポーネントを含めることができます。ノードは、ボタンやラベルなどのコントロール、または複数のネストされたコンポーネントを含むことができるレイアウトです。各シーンは1つのサブノードを持つことができますが、複数のコンポーネントを含むことができるレイアウトにすることもできます。ネストはマルチレベルにすることができます-レイアウトには他のレイアウトや共通コンポーネントを含めることができます。

概要


各アプリケーションは複数のウィンドウ（ステージ）を持つことができます。各ステージコンポーネントは、複数のシーンシーンを切り替えることができます。シーンにはノード（レイアウトと共通コンポーネント）が含まれています。

クラスアプリケーション


プログラミングを開始する時間。前の記事に従っている場合、必要な依存関係はすべて準備できています。

各JavaFXアプリケーションには、クラスを拡張するメインクラスが必要です。

javafx.application.Application

さらに、Applicationクラスの抽象メソッドをオーバーライドする必要があります。 

public void start(Stage primaryStage) throws Exception

Cass mainは次のようになります。

import javafx.application.Application;
import javafx.stage.Stage;

public class Main extends Application {

    @Override
    public void start(Stage primaryStage) throws Exception {
        // TODO implement me!
    }
}

主な方法


JavaFxアプリケーションを実行するために、main（）メソッドは必要ありません。JavaFX Packagerツールを使用して、jar実行可能ファイルをパッケージ化できます。ただし、メインメソッドを使用する方がはるかに便利です。

同時に、アプリケーションの実行が簡単になるだけでなく、通常どおりコマンドラインパラメータをアプリケーションに転送できます。

main（）メソッド内では、メソッドを使用してアプリケーションを起動できます。
Application.launch()


これがApplicationクラスの静的メソッドであることが簡単にわかります。メインクラスは指定しませんでしたが、JavaFXは、このメソッドを呼び出すクラスに応じて、これを自動的に決定できます。

ステージ設定


これで、main（）メソッドを使用してアプリケーションを起動する方法がわかりました。ただし、これを行っても何も起こりません。表示したいウィンドウが必要です。ウィンドウはステージと呼ばれます、覚えていますか？実際、プライマリー・ステージはすでに入力パラメーターとしてstartメソッドに渡されています。

public void start (Stage primaryStage)

ステージのこのコンポーネントを使用できます。唯一の問題は、デフォルトで非表示になっていることです。さいわい、primaryStage.show（）メソッドを使用して簡単に表示できます。

@Override
public void start(Stage primaryStage) throws Exception {
    primaryStage.show();
}

これで、アプリケーションを起動すると、次のウィンドウが表示されるはず



です。まず、ウィンドウに素敵な署名を追加しましょう。

primaryStage.setTitle("Hello world Application");

ウィンドウの見栄えをよくするために、ウィンドウの上部パネルに美しいアイコンを追加しましょう。

InputStream iconStream = getClass().getResourceAsStream("/icon.png");
Image image = new Image(iconStream);
primaryStage.getIcons().add(image);

アプリケーションを表すいくつかのアイコンを追加できます。より正確には、サイズの異なる同じアイコン。これにより、アプリケーションのコンテキストに応じて適切なサイズのアイコンを使用できます。

これで、Stageオブジェクトのプロパティと動作を構成できます。次に例を示します。

  • setX（）およびsetY（）を使用して位置を設定する
  • setWidth（）およびsetHeight（）で初期サイズを設定します
  • setMaxHeight（）およびsetMaxWidth（）で最大ウィンドウサイズを制限するか、setResizable（false）でサイズ変更を無効にします
  • setAlwaysOnTop（）を使用してウィンドウモードを常に最前面に設定します
  • setFullScreen（）を使用して全画面モードを設定します
  • そして、はるかに

現在、そのような素晴らしい名前のウィンドウがありますが、それはまだ空です。ステージ（ウィンドウ）にコンポーネントを直接追加できないことはすでに知っています。シーンが必要です。

ただし、シーンデザイナーを起動するには、子ノードを指定する必要があります。最初に簡単なラベルを作成しましょう。次に、このラベルを子ノードとしてシーンを作成します。

Label helloWorldLabel = new Label("Hello world!");
Scene primaryScene = new Scene(helloWorldLabel);

Sceneは1つの子コンポーネントしか許可しないことに気づきました。さらに必要な場合はどうなりますか？レイアウト（レイアウト）を使用する必要があります。これは、複数の子を含め、使用するレイアウトに応じて画面に配置できるコンポーネントです。このシリーズの後半でレイアウトについて説明します。

アプリケーションをもう少し視覚的に魅力的なものにするために、画面の中央にラベルを配置しましょう。

helloWorldLabel.setAlignment(Pos.CENTER);

最後に、Scene for Stageをインストールする必要があります。これはすでに用意されています。

@Override
public void start(Stage primaryStage) throws Exception {
    primaryStage.setTitle("Hello world Application");
    primaryStage.setWidth(300);
    primaryStage.setHeight(200);

    InputStream iconStream = getClass().getResourceAsStream("/icon.png");
    Image image = new Image(iconStream);
    primaryStage.getIcons().add(image);

    Label helloWorldLabel = new Label("Hello world!");
    helloWorldLabel.setAlignment(Pos.CENTER);
    Scene primaryScene = new Scene(helloWorldLabel);
    primaryStage.setScene(primaryScene);

    primaryStage.show();
}

これで、ウィンドウにラベルコンポーネントのあるシーンが含まれました。
