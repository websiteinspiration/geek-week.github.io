デノの最初の考え

Node.js開発者コースの開始に先立って、記事の翻訳が準備されました。


Codegramでは、新しいテクノロジーを試すことが大好きで、Denoは1週間前に登場しました。この新しいJavaScriptランタイムと、Codegramがそれについてどう考えているかについて知りたい場合は、記事を読んでください。

あなたはすでにDenoについて聞いたことがあるかもしれませんが、そうではないかもしれませんが、今やインターネットはそれについての情報でいっぱいで、その理由があります！新しいJavaScriptランタイムがかわいい恐竜のロゴとして登場しました（一部の人々は、ロゴには悲しい靴下があると思いますが...はありません）。最初のリリースの1週間前に彼についてさえツイートしましたが、実を言うと、2年前にRyan Dahlによるこの素晴らしいパフォーマンスを見てからDenoについて初めて知りました。

この記事では、Denoを表面的に見て、私たちの考えを共有します。始めましょう！

デノこんにちは！


DenoはNodeのようなJavaScriptランタイムです。これを使用するには、最初にdeno CLIをインストールする必要があります。これを行うには、deno.landのオペレーティングシステムの指示に従ってください（ドメイン名はお世辞です！）。 DenoはNodeのフォークではなく、Rustでゼロから作成されました。JSプログラマー向けのRustに関する記事を書きました。言語について詳しく知りたい場合は、この記事を読んでください。

CLIをインストールしたら、クラシックプログラム「Hello、World！」を実行できます。次のように：

deno run https://deno.land/std/examples/welcome.ts

TypeScriptをそのまま実行していますか？はい！ TSはDenoの歓迎すべきゲストなので、TSでコードを実行するために他に必要なものはありません。 JSコードも実行できますが、私はTSの大ファンなので、最初にそれについて話しました。

NodeコードはDenoと互換性がないことに注意してください。ただし、現在は互換性の確保に取り組んでおり、将来的にはお気に入りのNPMパッケージを使用できるようになります。

デノはデフォルトでサンドボックスであることを言及する価値があるもう一つのことがあります。つまり、最初はネットワークまたはファイルシステムにアクセスできません。プログラムがアクセスを必要とする場合は、明示的に許可する必要があります（つまり、-allow-netを使用してネットワーク特権を付与します）。

これ以上package.jsonまたはnode_modules
このジョークをインターネットで見たことがない人はいますか？



少し誇張にもかかわらず、このジョークにはまだいくつかの真実があります。アプリケーションが大きくなるにつれて、依存関係のリストが大きくなり、巨大な依存関係ツリーができあがります。問題は、すべてのアプリケーションに、使用しているバージョンのパッケージのコピーがあることです。
Denoの主な目標の1つは、ブラウザをミラーリングすることです。何かをブラウザにインポートしたいときは、タグを書い<script>てURLを使います。 Denoは次のように機能します。

import { assertEquals } from "https://deno.land/std/testing/asserts.ts";
assertEquals("hello", "hello");
assertEquals("world", "world");
console.log("Asserted! ");

私は同意します、それは少し奇妙に見えます。package.jsonがないため、外部依存関係を指定する方法はないので、必要なときにインポートします。Denoは最初の起動時にファイルをダウンロードし、キャッシュに保存します。パッケージを再度ダウンロードする場合は、-reloadフラグを指定できます。

あなたはおそらくこのトピックに関して多くの質問を持っているでしょう（私もそれらを持ちました）ので、このトピックについての詳細を取得するために、DenoマニュアルのこのFAQページにアクセスすることを勧めます。

ブラウザーAPI互換ランタイム


Denoのもう1つの重要な目標は、ブラウザーの互換性でした。そのため、fetchやaddEventListenerなど、グローバルスコープから使用できる一連の機能があります。グローバルウィンドウオブジェクトを使用することもできます（ただし、現時点ではglobalThisの使用をお勧めします）。グローバル関数の詳細については、ドキュメントをご覧ください。

ランタイムには、Web標準ではないAPI用のグローバルDenoもあります。ファイルの読み取り、TCPソケットのオープンなどの低レベルの操作に使用できます。

豊富な標準ライブラリ


Denoの開発者は、すぐに使える標準モジュールのコレクションを作成しました。ライブラリはdeno.landドメインでもホストされているため、次のURLを使用してアプリケーションに任意のモジュールをインポートできます。

import { v4 } from "https://deno.land/std/uuid/mod.ts";

console.log(v4.generate());

標準ライブラリには多くの便利なモジュールが含まれていますが、サードパーティのモジュールも多く、このリストは毎日更新されます。もう1つ良いニュースがあります。パブリックURLでホストされている任意のモジュールをインポートできるため、GitHubからモジュールを簡単にインポートできます。Denoには、モジュールを使用可能にするために使用できるURL書き換えサービスがあります。このメカニズムについて詳しくは、このページをご覧ください。

組み込みツール


Deno CLIには、たとえばテストの実行や、パッケージの作成など、いくつかの一般的なタスクを実行するための組み込みツールも含まれています。これは、Denoがjest、prettier、webpackを置き換えることができることを意味しますか？おそらく違います。Denoはかなり新しいソリューションであり、まだそれほど広範なエコシステムを備えていませんが、1つのバイナリをインストールすることでアクセスできるこのツールキットのすべての存在は、依然として印象的です。

結論


この誇大宣伝の波に飛び乗ろう！デノには本当に感動しました。2年間の開発と成功への長い道のりを経て、新しいJavaScriptランタイムは素晴らしく見え、それを試してみたいと思っています。Nodeは約11年前に登場し、JavaScriptの歴史における新たなマイルストーンとなりました。彼は言語をより良くしました、そして私達は私達が新しいマイルストーンに達したと思います、それはデノの時間です！
