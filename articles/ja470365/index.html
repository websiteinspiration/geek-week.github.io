AndroidでのUSBカスタムHIDの操作


他のデバイスと対話するための最新のAndroidアプリケーションでは、Bluetoothなどのワイヤレスデータ転送プロトコルが最もよく使用されます。一部のデバイスがワイヤレス充電を使用している年には、有線インターフェースの使用を必要とするAndroidデバイスと周辺モジュールの束を想像することは困難です。しかし、そのようなニーズが発生すると、すぐにUSBが思い浮かびます。

仮説を立ててみましょう。顧客が来て次のように言ったとします。「データ収集デバイスを制御し、このデータを画面に表示するには、Androidアプリケーションが必要です。ただし、アプリケーションはAndroidオペレーティングシステムを搭載したシングルボードコンピュータに書き込む必要があり、周辺機器はUSB経由で接続されています”

それは素晴らしいように聞こえますが、時々起こります。そして、ここでは、USBスタックとそのプロトコルの詳細な知識が役立ちますが、この記事はそれについてではありません。この記事では、AndroidデバイスからUSBカスタムHIDプロトコルを使用して周辺機器を制御する方法について説明します。簡単にするために、周辺機器（デバイス）のLEDを制御し、ボタンの状態を受信する（押す）Androidアプリケーション（ホスト）を作成します。ペリフェラルボードのコードは提供しません。興味のある方はコメントに書き込んでください。

それでは始めましょう。

理論。できるだけ短く


最初に、できるだけ短い理論です。これは単純化された最小値であり、コードを理解するには十分ですが、理解を深めるために、このリソースに慣れることをお勧めします。

周辺機器でUSBを介して通信するには、インタラクションインターフェイスを実装する必要があります。さまざまな機能（たとえば、USB HID、USB Mass Strorage、またはUSB CDC）はそれらのインターフェースを実装し、一部はいくつかのインターフェースを備えています。各インターフェースには、エンドポイント-特別な通信チャネル、一種のクリップボードが含まれています。

私の周辺機器には、1つのインターフェイスと2つのエンドポイント（1つは受信用、もう1つは送信用）を持つカスタムHIDがあります。通常、デバイスに存在するインターフェースとエンドポイントの情報は、デバイスの仕様に記載されています。それ以外の場合は、USBlyzerなどの特別なプログラムを使用して決定できます。

USB HIDのデバイスは、レポートを通じて通信します。レポートとは何ですか？データはエンドポイントを介して送信されるため、プロトコルに従って何らかの方法でデータを識別して解析する必要があります。デバイスは、データバイトを互いに投げ合うだけでなく、明確に定義された構造を持つパケットを交換します。これは、特別なレポート記述子でデバイスに記述されています。したがって、レポートの記述子に従って、特定のデータの送信の識別子、構造、サイズ、および頻度を正確に判断できます。パケットは、レポートIDである最初のバイトで識別されます。たとえば、ボタンの状態に関するデータはID = 1のレポートに送信され、ID = 2のレポートを通じてLEDを制御します。

鉄から離れ、Androidに近い


Androidでは、APIバージョン12（Android 3.1）以降、USBデバイスのサポートが登場しました。周辺機器を操作するには、USBホストモードを実装する必要があります。USBの操作については、ドキュメントでかなりよく説明されています。

まず、さまざまなUSBデバイスの中から、接続されているデバイスを特定する必要があります。USBデバイスは、vid（ベンダーID）とpid（製品ID）の組み合わせで識別されます。xmlフォルダーで、次の内容のdevice_filter.xmlファイルを作成します。

<resources>
    <usb-device vendor-id="1155" product-id="22352" />
</resources>

次に、アプリケーションマニフェストで適切なアクセス許可とアクション（必要な場合）を作成する必要があります。

<uses-permission android:name="android.permission.USB_PERMISSION" />
<uses-feature android:name="android.hardware.usb.host" />

<activity android:name=".MainActivity">
            <intent-ilter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
            <meta-data
                android:name="android.hardware.usb.action.USB_DEVICE_ATTACHED"
                android:resource="@xml/device_filter" />
</activity>

android：リソースでは、デバイスに必要なフィルターを含むファイルを指定します。また、先に述べたように、たとえばデバイスを接続した結果として、アプリケーションを起動するためにインテントフィルターを割り当てることができます。

まず、UsbManagerを取得し、デバイス、デバイスのインターフェイスとエンドポイントを見つける必要があります。これは、デバイスが接続されるたびに実行する必要があります。

val usbManager = context.getSystemService(Context.USB_SERVICE) as UsbManager

private var usbConnection: UsbDeviceConnection? = null
private var usbInterface: UsbInterface? = null
private var usbRequest: UsbRequest? = null
private var usbInEndpoint: UsbEndpoint? = null
private var usbOutEndpoint: UsbEndpoint? = null

fun enumerate(): Boolean {
  val deviceList = usbManager.deviceList
    for (device in deviceList.values) {

      /*      VID  PID */
      if ((device.vendorId == VENDOR_ID) and (device.productId == PRODUCT_ID)) {

        /*      */
        usbInterface = device.getInterface(CUSTOM_HID_INTERFACE)

        /*     
                   */
        for (idx in 0 until usbInterface!!.endpointCount) {
          if (usbInterface?.getEndpoint(idx)?.direction == USB_DIR_IN)
            usbInEndpoint = usbInterface?.getEndpoint(idx)
          else
            usbOutEndpoint = usbInterface?.getEndpoint(idx)
          }

           usbConnection = usbManager.openDevice(device)
           usbConnection?.claimInterface(usbInterface, true)

           usbRequest = UsbRequest()
           usbRequest?.initialize(usbConnection, usbInEndpoint)
        }
     }

  /*    */
  return usbConnection != null
}

ここでは、前のセクションで説明したインターフェースとエンドポイントを確認できます。インターフェイス番号がわかっているので、受信と送信の両方のエンドポイントを見つけ、USB接続を開始します。以上で、データを読み取れるようになりました。

先に述べたように、デバイスはレポートを通じて通信します。

    fun sendReport(data: ByteArray) {
        usbConnection?.bulkTransfer(usbOutEndpoint, data, data.size, 0)
    }

    fun getReport(): ByteArray {
        val buffer = ByteBuffer.allocate(REPORT_SIZE)
        val report = ByteArray(buffer.remaining())

        if (usbRequest.queue(buffer, REPORT_SIZE)) {
            usbConnection?.requestWait()

            buffer.rewind()
            buffer.get(report, 0, report.size)
            buffer.clear()
        }

        return report
    }

バイトの配列をsendReportメソッドに送信します。0バイトはレポートIDであり、デバイスへの現在のUSB接続を取得して転送を実行します。パラメータとして、エンドポイント番号、データ、それらのサイズ、および送信タイムアウトをBulkTransferメソッドに渡します。 UsbDeviceConnectionクラスには、USBデバイスとのデータ交換を実装するためのメソッド（bulkTransferおよびcontrolTransferメソッド）があることに注意してください。それらの使用は、エンドポイントがサポートする送信のタイプに依存します。この場合、bulkTransferを使用しますが、コントロールタイプでのエンドポイントの使用はHIDで最も一般的です。しかし、私たちはカスタムHIDを持っているので、必要なことを行います。送信されるデータの量と頻度はデータの送信頻度に依存するため、送信のタイプについては個別に読むことをお勧めします。

データを取得するには、受信したデータのサイズを知る必要があります。これは、事前に知ることも、エンドポイントから取得することもできます。

USB HIDを介してデータを受信する方法は同期的でブロッキングであり、別のスレッドで実行する必要があります。さらに、デバイスからのレポートは常にまたはいつでも送信される可能性があるため、データを見逃さないようにレポートの定期的なポーリングを実装する必要があります。RxJavaを使ってみましょう：

fun receive() {
	Observable.fromCallable<ByteArray> { getReport() }
	                .subscribeOn(Schedulers.io())
	                .observeOn(Schedulers.computation())
	                .repeat()
			.subscribe({
				/* check it[0] (this is report id) and handle data */
			},{
				/* handle exeption */
			})
}

バイトの配列を受け取ったら、ゼロバイトをチェックする必要があります。これはレポートのIDであるため、受け取ったデータを解析するためです。

USBでのすべてのアクションが完了したら、接続を閉じる必要があります。これは、onDestroyアクティビティまたはViewModelのonClearedで実行できます。

    fun close() {
        usbRequest?.close()
        usbConnection?.releaseInterface(usbInterface)
        usbConnection?.close()
    }

結論


この記事では、特定のデバイスの実装に関する非常に小さくてプリミティブな、非常に実証的なコードについて説明します。もちろん、HIDだけでなく、多くのUSBクラスがあります。もちろん、それらのクラスは実装が異なります。ただし、すべての方法はかなり文書化されており、USBスタックを十分に理解しているため、それらの適用方法を簡単に理解できます。
