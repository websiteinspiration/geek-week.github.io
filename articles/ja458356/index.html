カレンダーを勉強

この記事は、私が昨日自問した質問からのものです。
「月曜日に1か月が始まらない年はありますか？」
一見、そうです。1年は任意の曜日から開始でき、毎月は毎週異なる曜日から開始できます。多くのオプションがありますが、おそらく1年以上かかるでしょう。

と思ってから最初の１分と思いました。これは証明されるべきです。たとえば、すべての年を通過します。シンプルで迅速な方法ですが、面白くありません。数学的に証明することははるかに魅力的なアイデアでしたが、私はこれへのアプローチ方法を完全に理解していませんでした。したがって、私は毎月の期間を紙に書き始めました。

ここで言及する価値があるのは、1918年以来私たちが住んでいるグレゴリオ暦についてもっと話すことです。ただし、理由の一部はJulianにも当てはまります。

実際、そのような年は存在しません。その理由を理解しましょう。

パート1.月


まず、各月の日数を覚えておいてください：
ヤン 2月 行進 4月 五月 六月 七月 8月 9月 10月 11月 12月
31 28/29 31 30 31 30 31 31 30 31 30 31
次に、各月の4週間を超える日数を見てみましょう。
ヤン 2月 行進 4月 五月 六月 七月 8月 9月 10月 11月 12月
３ 0/1 ３ 2 ３ 2 ３ ３ 2 ３ 2 ３
この時点で、次のアイデアが生まれます。日付に7日を追加しても、曜日は変更されません。モジュラー演算が機能します。ここからは、4週間よりも1か月の日数が2日多い場合、翌月の最初の日は、当月の最初の日と比較して、週の2日だけずれることになります。とにかく、
月に（28 + N）日がある場合、翌月の最初の日は、現在の月の最初の日の曜日を基準としてN日シフトします。
たとえば、今年の1月は火曜日に始まったため、2月は金曜日に始まりました。火+ 3 =金

ある月の初日の曜日はいくらですか。これを見つけるには、前月すべての4週間の「余剰」日を合計する必要があります。この表は、1月1日の曜日に対するシフトを示しています。最初の行はうるう年ではなく、2行目はうるう年です。
ヤン 2月 行進 4月 五月 六月 七月 8月 9月 10月 11月 12月
0 ３ ３ 6 8 十一 十三 16 十九 21 24 26日
0 ３ 4 7 9 12 14 17 二十 22 25 27日
しかし、これはあまりはっきりとは見えません。7日間シフトしても曜日は変わらないことがわかっています。したがって、シフトの合計を7で割った余りを表に書き込みます。
ヤン 2月 行進 4月 五月 六月 七月 8月 9月 10月 11月 12月
0 ３ ３ 6 1 4 6 2 5 0 ３ 5
0 ３ 4 0 2 5 0 ３ 6 1 4 6
もう一つ！1月1日の曜日がわかれば、月の最初の曜日をどのように決定するかがはっきりとわかります。関心のある月のシフトを追加するだけです。高校時代から2〜3月〜11月のパターンは知っていましたが、他には気づかなかった。

記事の冒頭で質問に対する回答を得ました。
テーブルの年の両方のバリアントについて、すべて0から6へのシフトがあるため、どの年でも、特定の曜日に始まる月があります。
しかし今、あなたは他の質問をすることができます。たとえば、「そのような月は1年間しかあ​​りませんか？」または「そのような月の何年に最大数がありますか？」これを行うには、任意の年の1月1日の曜日を判別できる必要があります。

パート2.年


私がプログラミングを学び、これがPascalABCの学校の10年生であったとき、最初の重大なタスクの1つは、引数として渡された、年間のカレンダーを出力する手順を実装することでした。実装する機能に関するヒントがありました。一般に、参照と現在の2つの日付の間の日数をカウントして、目的の年の1月1日の曜日を決定しました。

このアプローチは機能しましたが、速度は必要な年が参照にどれだけ近いかに依存しました。それは私を動揺させました、しかし私はその時より良いものを思いつくことができませんでした。今、これを完全に理解する絶好の機会が来ました。

グレゴリオ暦のうるう年は次のように割り当てられます。
  • 400の倍数の年はうるう年
  • 残りの年、その数は100の倍数であり、飛躍的ではありません
  • 残りの年は4の倍数で、うるう年
  • 残りの年は飛躍的ではない
この記述は、うるう周期が400年の周期であることを示しています。しかし、そのような400年のサイクルが同じ曜日に始まるかどうかは明らかではありません。

1月1日は年ごとに週の1日から2日にシフトすることに注意してください。
いくつかのコード。 
bool is_leap_year(int year)
{
    if ((year % 400) == 0) return true;

    if ((year % 100) == 0) return false;

    if ((year %   4) == 0) return true;

    return false;
}

void first_weekdays_table()
{
    ofstream file("weekdays.txt", ios_base::out);

    int weekday = 3;

    for (int i = 1801; i <= 3000; ++i)
    {
        file << weekday;

        if ((i % 100) != 0)
        {
            file << " ";
        }
        else
        {
            file << endl;
        }

        weekday += is_leap_year(i) ? 2 : 1;
        weekday %= 7;
    }

    file.close();
}


曜日は、毎年1月1日の1801〜3000で表示されます。月曜日は「0」、火曜日は「1」などと指定されます。2つの完全な400年サイクルと2つの半分の表の形ですべてを表示します。世紀は水平に、これらの世紀の年間の垂直方向に移動します。世紀と年の交差するセルには、今年が始まった曜日が書かれています。たとえば、1997年に始まった曜日は、列「1900」と行「97」の交点です。水曜日です。フルテーブル：パート1、パート2。


この表では、2つのことにすぐに気付くことができます。実際には400年のサイクルが同じ曜日（2001、2401、および2801、月曜日）に始まり、2000の代わりに「1000分の1」があります。後者は、さらに便宜上、故意に行われました。最初の事実により、障害なく進むことができます。
グレゴリオ暦では、400年サイクルすべてが月曜日に始まります。
しかし、最も興味深いのはテーブルの完全版です。400年のサイクル内のすべての世紀は、繰り返される28年のサイクルで構成されていることがわかります。
0 1 2 ３ 5 6 0 1 ３ 4 5 6 1 2 ３ 4 6 0 1 2 4 5 6 0 2 ３ 4 5

最初の世紀は、0に等しいサイクルのシフトで始まり、2番目は4のシフト、3番目は8のシフト、4番目は12のシフトで始まります。このため、表は世紀の「100分の1」があり、ゼロがない形式で表示されます。年間で合計14の異なるオプションがあることは言うまでもありません。28年サイクルでは、各曜日に1回、うるう年の開始は3回、うるう年以外の開始は3回です。

これで、参照日付を使用せずに、任意の日付の曜日を決定できます。これを行うには、その年が400年のサイクルのどの世紀にあるか、そしてそれが今世紀に何を意味するかを理解する必要があります。表によると、年の1月1日の曜日を、記事の最初の部分（月の特定の曜日）を使用して決定します。千の言葉の代わりに
さらにコードを書きます。 
int get_weekday(int year, int month, int day)
{
    int weekdays[] = {0, 1, 2, 3, 5, 6,
                      0, 1, 3, 4, 5, 6,
                         1, 2, 3, 4, 6,
                      0, 1, 2, 4, 5, 6,
                      0, 2, 3, 4, 5};

    int shift_not_leap[] = {0, 3, 3, 6, 1, 4, 6, 2, 5, 0, 3, 5};
    int shift_leap[]     = {0, 3, 4, 0, 2, 5, 0, 3, 6, 1, 4, 6};

    bool is_leap = is_leap_year(year);

    year -= 1;
    year %= 400;

    int century = year / 100;

    year %= 100;

    int index = (year + (4 * century)) % 28;
    int weekday = weekdays[index];

    weekday += is_leap ? shift_leap[month - 1]
                       : shift_not_leap[month - 1];
    weekday += (day - 1);
    weekday %= 7;

    return weekday;
}

これを考慮すると、うるう年の月のオフセットは、うるう年以外のオフセットを介して計算できるという事実と同様に、
次の機能 
int get_weekday_c(int year, int month, int day)
{
    int shifts[] = {0, 3, 3, 6, 1, 4, 6, 2, 5, 0, 3, 5};

    int shift = shifts[month - 1];

    if (is_leap_year(year) and (month > 2))
    {
        shift += 1;
    };

    year = (year - 1) % 400;

    int century = year / 100;

    int index   = ((4 * century) + (year % 100)) % 28;

    int weekday = (index + (index / 4)) + shift + (day - 1);

    return (weekday % 7);
}


したがって、任意の日付の曜日を計算できます。12の数値のみを知っています。つまり、曜日の各月の最初の日へのシフトです。
