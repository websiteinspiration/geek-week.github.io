Game Code Command＆Conquer：90年代のバグ。第1巻


アメリカの会社Electronic Arts Inc（EA）は、コマンド＆コンカー：ティベリアンドーンとコマンド＆コンカー：レッドアラートのゲームのオープンソースコードを投稿しました。このコードは、ゲームコミュニティがModとマップを開発し、カスタムユニットを作成し、ゲームプレイロジックをカスタマイズするのに役立ちます。私たちは皆、開発の歴史に突入するユニークな機会があり、それは現代のものとは非常に異なります。次に、StackOverflowサイト、便利なコードエディタ、強力なコンパイラはありませんでした。そして、静的アナライザーはなく、コミュニティーが最初に直面するのは、コード内の何百ものエラーです。ただし、PVS-Studioチームがこれを支援し、これらのエラーの場所を指摘します。

前書き


Command＆Conquer-リアルタイム戦略のジャンルの一連のコンピューターゲーム。シリーズの最初のゲームは1995年にリリースされました。Electronic Artsは、このゲームの開発スタジオを1998年に買収しました。

それ以来、いくつかのゲームと多くのModがリリースされました。ゲームのソースコードは、Command＆Conquer Remasteredコレクションのリリース時に公開されました。 コードのエラーを見つけるために、PVS-Studioアナライザーを使用しました。これは、C、C ++、C＃、およびJavaで記述されたプログラムのソースコードのエラーおよび潜在的な脆弱性を検出するためのツールです。 コードで見つかった大量の問題のため、エラーのすべての例は2つの記事のサイクルで示されます。

V610未定義の動作。シフト演算子 '<<'を確認してください。右のオペランド（ '（32-bits_to_shift）' = [1..32]）は、昇格した左のオペランドの長さ（ビット単位）以上です。MP.CPP 659

#define UNITSIZE 32

void XMP_Shift_Right_Bits(digit * number, int bits, int precision)
{
  ....
  int digits_to_shift = bits / UNITSIZE;
  int bits_to_shift = bits % UNITSIZE;

  int index;
  for (index = digits_to_shift; index < (precision-1); index++) {
    *number = (*(number + digits_to_shift) >> bits_to_shift) |
      (*(number + (digits_to_shift + 1)) << (UNITSIZE - bits_to_shift));
    number++;
  }
  ....
}

アナライザは、32ビット数がより多くのビット数だけ右にシフトする可能性がある状況を検出しました。方法は次のとおりです。

int bits_to_shift = bits % UNITSIZE;

UNITSIZE定数の値を有する32。

int bits_to_shift = bits % 32;

したがって、bits_to_shift変数の値は、32の倍数であるすべてのビット値に対してゼロになります。 したがって、このコードでは次のようになります。



.... << (UNITSIZE - bits_to_shift) ....

定数32から0を減算すると、32ビットのシフトが発生します。
