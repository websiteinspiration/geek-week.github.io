Seq2seq training



All this is very interesting, but how is RL related? The connection lies in the learning process of the seq2seq model. But before moving on to modern RL approaches to the problem, we must say a few words about the standard way of teaching.



Credibility training



Imagine that you need to create a machine translation system from one language (say, French) to another (English) using seq2seq. We have a good, large set of sentences translated from French into English on which we are going to train our model. How to do it?



The encoding phase is obvious: we apply the encoder to the first sentence in the training pair, creating its encoded representation. A natural candidate for this view would be a hidden state returned from the last RNN application. At the encoding stage, we ignore the RNN output, taking into account only the latent state from the last RNN application. We also expand the offer with a special token, which signals the encoder about the end of the offer. This process is shown in fig. 12.4.









To start decoding, we transfer the encoded representation to the input hidden state of the decoder with a token as a signal to start the process. At this stage, the decoder should return the first token of the translated offer. However, at the beginning of training, when the encoder and decoder are initialized with random weights, the decoder output will be random, and our goal is to push it to the correct conversion using stochastic gradient descent (SGD).



The traditional approach is to consider this problem as a classification problem, when the decoder needs to return the probability distribution by tokens at the current position of the decoded sentence. This is usually done by converting the output of the decoder using a shallow (1-2 layer) fully connected network that returns a vector whose length is equal to the size of our dictionary. Then we take this probability distribution and the standard loss for classification problems ‚Äî cross entropy (also known as maximum likelihood loss).



With the first token in the decoded sequence to be generated by the token specified at the input, everything is clear, but what about the rest of the sequence? There are two options. The first is the supply of tokens from the matched offer. For example, if we have a training pair Je ne mange pas six jours -> I haven't eaten for six days, we feed the tokens (I, haven't, eaten ...) to the decoder and then use the cross-entropy loss between the RNN pin and the next token in the offer. This learning mode is called teacher forcing, and at each step we enter a token from the correct translation, asking RNN for the correct next token. This process is shown in the diagram (Fig. 12.5).









The expression for losses in the previous example will be calculated as follows: 



L = xentropy(p1,"I") + xentropy(p2,"haven't") + xentropy(p3,"eaten") +
   xentropy(p4,"for") + xentropy(p5,"six") + xentropy(p6,"days") +
   xentropy(p7,"<END>")


Since both the encoder and the decoder are differentiable NSs, we can apply the backpropagation algorithm to the error in order to force both of them to better classify this example in the future, in the same way that we train, for example, the image classifier.



Unfortunately, the previous procedure does not solve the seq2seq training problem completely, which is related to how the model will be used after training. During training, we know both the input and the desired output sequence, so we can transfer the actual output sequence to the decoder, which is only required to create the next sequence token.



After training the model, we will not have the target sequence, since it is assumed that this sequence will be created by the model. Thus, the simplest way to use the model is to encode the input sequence with the encoder, and then ask the decoder to generate one output element at a time, by applying the received token to the decoder input.



Passing the previous result into the input may look natural, but there is a danger. During training, we did not ask the decoder to use its own output as input, so a single error during generation can confuse the decoder and lead to garbage output.



To prevent this, there is a second approach to learning seq2seq, called curriculum learning. This method uses the same cross-entropy loss, but instead of transmitting the full target sequence as decoder input, we simply ask it to decode the sequence in the same way that we are going to use it after training. This process is illustrated in the following diagram (Fig. 12.6). This adds reliability to the decoder, which gives the best result in practical application of the model.









The disadvantage of this mode can be very long training, since the decoder learns how to produce the desired output of the token after the token. To compensate for this, in practice, we usually train the model using both teacher-driven and chain learning, simply randomly choosing between them for each training set.



Bilingual evaluation understudy score



Before moving on to the main topic of this chapter (RL for seq2seq), we need to introduce a parameter used to compare the quality of machine translation output and commonly used in NLP tasks. The Bilingual evaluation understudy (BLEU) parameter is one of the standard ways to compare the output sequence generated by the machine with a certain set of reference outputs. It allows you to use several reference outputs (one sentence can be translated in various ways) and, in fact, calculates the ratio of unigrams, bigrams, etc., which are shared by the findings and reference sentences.



There are other alternatives such as CIDEr and ROGUE. In this example, we use BLEU implemented in the nltk library from Python (package nltk.translate.bleu_score).



RL in seq2seq



RL and text generation may look very different, but there are common ground between them that can be used to improve the quality of trained seq2seq models. The first thing you should pay attention to is that our decoder displays the probability distribution at each step, which is very similar to the model based on gradients by strategy. From this point of view, the decoder can be considered as an agent trying to decide which token to create at each stage. There are several advantages to such an interpretation of the decoding process.



First of all, considering the decoding process as stochastic, we can automatically take into account several target sequences. For example, hearing ‚ÄúHello! How are you? ‚Äù, We can answer in many ways, and they are all true. By optimizing the maximum likelihood indicator, our model will try to learn something average for all these answers, but average for the sentences ‚ÄúEverything is in order, thanks!‚Äù and ‚ÄúNot Very Good‚Äù will not necessarily be a meaningful phrase. By returning the probability distribution and choosing the next token from it, our agent could potentially learn to create all possible options, instead of learning some kind of averaged answer.



The second advantage is the optimization of the indicator of interest. When training using maximum likelihood, we minimize cross-entropy between tokens created and taken from the standard, but with machine translation and solving many other NLP problems, we are not particularly worried about maximum likelihood: we want to maximize the BLEU score of the resulting sequence. Unfortunately, the BLEU score is non-differentiable, so we cannot use it as an error in the backpropagation algorithm. However, PG methods such as REINFORCE (see chapter 9) work even when the reward is undifferentiated: we simply increase the probabilities for successful episodes and decrease for unsuccessful ones.



The third advantage that can be used is that we ourselves determine the process of generating the sequence and know its internal components. Introducing stochasticity into the decoding process, we can repeat it several times, collecting various decoding scenarios from a single training example. This can be useful when the training data set is limited, and this is almost always the case, especially if you are not working for Google, Facebook, or another large company.



In order to understand how to switch learning from maximum likelihood to an RL scenario, let us consider both of them from a mathematical point of view. Maximum likelihood score means maximizing the amountby adjusting the model parameter, which is the same as minimizing the Kullback - Leibler distance between the data probability distribution and the probability distribution parameterized by the model, which can be written as maximizing the expression.



At the same time, the goal of the REINFORCE method is to maximize the expression. The relationship is obvious, and the difference between them is only in the scale factor in front of the logarithm and in the way of choosing the actions that are tokens in our dictionary.

Self-critical sequence learning



The described approach, despite its positive aspects, is fraught with a number of difficulties. Learning from scratch is almost useless. Even for simple dialogs, the output sequence usually contains at least five words, each of which is taken from the dictionary of several thousand words. The number of different phrases consisting of five words with a dictionary of 1000 words is 51000, which is slightly less than 10700. Thus, the probability of getting the right answer at the beginning of training, when the weights for the encoder and decoder are random, is negligible. To cope with this, you can combine approaches based on maximum likelihood and RL, and pre-train the model with the maximum likelihood indicator (switching between teacher-led learning and learning by chain), and after it reaches a certain level of quality,Switch to the REINFORCE method for fine tuning. In general, this can be considered as a unified approach to the complex tasks of RL, when a large space of actions makes it impossible to start with an agent whose behavior is random, since the probability that such an agent accidentally reaches the goal is insignificant.
