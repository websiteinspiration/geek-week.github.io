Ludum Dare：开始前一周的清单

Ludum Dare是一项国际计算机游戏加速竞赛，您可以在周末根据主题从头开始创建游戏。在活动开始时将显示该主题。

比赛一年举行几次，下一届Ludum Dare 46比赛将在不到一周的时间里开始，或者说是4月18日星期六，莫斯科时间凌晨4:00。

在这里，我想分享一些技巧，这些技巧将帮助那些正在考虑首次参加此活动的人。这是初学者的清单。

官方网站包含参与规则，以及托管游戏并将其发送给社区其他成员进行评估的指南。我不会讲这些规则。但是，有关托管和发送到该站点的信息写在了下面。

在开始之前，请先介绍一下Jam和Compo。

果酱和康宝


Ludum Dare有两种参与类型：Jam（简单选项）和Compo（高难度模式）。

Jam（创建游戏需要72个小时）是团队和单打比赛的Ludum Dare类别。您可以使用任何工具和库来创建游戏。允许以任何可用的基本代码开头。

Compo（创建游戏需要48小时）是经典的Ludum Dare。Compo游戏完全是由一个人从零开始创建的。游戏及其所有内容必须在48小时内创建，最后您必须提供所有源代码。 

为了更好地了解这些模式的细节和局限性，以及决定参加哪个学科，最好阅读官方指南。

如何准备


制作游戏是一个复杂的创意过程，需要花费时间才能确定游戏的机制，视觉部分和声音。

重要的是要记住，您的游戏将在以下几点上被其他参与者评价：

  1. 创新（novelty）； 
  2. 乐趣（愉悦）；
  3. 主题
  4. 图形;
  5. 音频（声音）；
  6. 幽默（幽默）；
  7. 心情（心情）;
  8. 总体（总体评价）。

在我看来，Ludum Dare是一个活动，最好使用精通您的技术和工具，以便将更多的时间用于开发游戏机制和图形。

卡纸之前


还有一些更好的步骤可以更好地遵循或深思熟虑：

  • 考虑托管一个完成的游戏；
  • 考虑使用版本控制系统和项目托管系统；
  • 确定团队和职责；
  • 事先考虑团队成员之间的沟通方式；

对于我来说，很重要的一点是，要想一想就先想一想，如果不知道，如果不清楚使用哪种风格，或者什么时候感觉一切都不对，那游戏的外观将是不知道的。

在设计学校，人们经常说，如果您没有自己的想法，则需要注意他人的想法。这是所有时间的建议。看看有什么启发，但尝试做出新的变化。实验。

同样重要的是要记住，游戏的印象会在最初的几秒钟内产生。如果游戏不能立即流行，那么很可能很多人都不会欣赏它。

我想回想一句话：
从想象最终目标开始。
这也很重要。您可以逃避小事情，而从不做主要事情。 

卡纸时


根据经验，在卡纸期间很难正确评估时间。似乎需要很多时间，但这是一个陷阱。因此，我建议您以已经证明的解决方案为基础，并将其结合起来。最好制作一个具有1-2个有趣功能的小型完成项目。否则，您可以沉迷于细节，什么也不做。

至于主题，您不需要按字面意义进行操作。更好地提出自己的解释。您可以在网站上的“提交”中，在游戏名称之后添加对您的解释想法的描述。 

规则的主要思想是：
官方不，您不需要使用主题。我们不会因为您无视它而取消您的资格。对我们来说，主题只是一个投票类别。因此，作为一般建议，如果您想在Ludum Dare中获得良好的成绩，请确保以某种方式使用主题。
如果您不坚持该主题，您将不会失去资格。但是主题是投票的类别之一。因此，如果您想在Ludum Dare中获得良好的成绩，我们建议您以某种方式坚持该主题。

以下是发布在官方网站上的游戏托管指南的基本信息。

从Jam / Compo启动到申请时间结束，您都可以随时提交游戏。Compo在主题宣布后48小时结束，而Jam在主题宣布后72小时结束。观看Ludum Dare网站侧边栏中的倒计时时钟。

完成后，Compo和Jam都有一个额外的小时要发送。强烈建议您尽早发送文档，因为可能会遇到各种各样的技术问题。 

在上传到站点之前，您必须具有：

  1. 在Ludum Dare网站上的帐户；
  2. 游戏中的一些屏幕截图；
  3. 您的游戏名称；
  4. 链接到位于*处的游戏；
  5. 链接到位于某处的源代码（仅适用于Compo）**。

*您需要的托管类型取决于您制作的是网络游戏还是台式机游戏。 

官方网站上的指南列出了几种流行的平台。这不是一个详尽的列表，还有许多其他方式来托管您的游戏。

游戏网站：

  • Itch.io
    允许免费托管HTML5，Flash，Java和Unity，以及桌面游戏。
  • GameJolt
    允许您免费托管网络和桌面游戏。
  • Newgrounds
    允许免费托管Flash（swf）和HTML5（zip）上的游戏。
  • Kongregate
    允许您免费在Unity，Flash，HTML5和WebGL上托管游戏。

项目托管系统：

  • Github
  • Gitlab
  • 比特桶

云存储（更适合于托管桌面游戏）：

  • 投寄箱
  • Google云端硬碟

有许多提供免费网络托管的站点，以及许多提供免费试用版的付费站点。如果您决定使用免费的虚拟主机，建议您在启动Ludum Dare之前下载一个测试游戏。

如果您拥有自己的网站或服务器，则可以在此处托管游戏。

**您的游戏的源代码（参与Compo类别）可以放置在上面列出的项目托管系统或云存储中。使用云服务时，请将.zip文件与所有源代码一起放置。

参加Ludum Dare是一种摆脱舒适区，测试自己并获得新体验的好方法。 

我认为，优先级和时间问题仍然是绊脚石，也是许多项目失败的原因，在竞争中，您可以在2-3天的时间内完成大部分开发阶段，并学到很多东西，而在一个真正的项目中，这可能需要几个月或几年的时间。 

此外，Ludum Dare的本质不仅在于运动兴趣和对制作游戏项目的渴望（通常是将来开发更多游戏的原型），而且是与新朋友合作的机会。没有必要组织一个协调良好的团队来参与工作-人们常常在工作过程中被揭露。  

如果您仍然对参与度有疑问，那么我可以说Ludum Dare是一项超出可能范围的实验。 

不要对自己太苛刻。玩得开心！
