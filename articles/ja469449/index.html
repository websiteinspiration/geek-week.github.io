EV SSL証明書：死後の人生はありますか？

Chrome 77とFirefox 70の新しいバージョン（10月22日リリース）では、拡張検証付きのEV証明書がアドレスバーでの通常の場所を失いました。現在、一見、ドメインを検証する通常のDV証明書と違いはありません。会社に関する追加情報は、ロックアイコンを押すだけで公開され、アドレスバーには公開されません。


これが、Firefox 70のEV SSL証明書に関する情報の見方です。

業界の批評家は、「航空取引」が停止することを喜んでいます。しかし、レジストラ自身はEV SSLを埋めるには時期尚早だと考えています。

拡張検証証明書


拡張検証証明書（Extended Validation、EV）-証明機関がドメインを法人にリンクすることにより、ドメイン所有者の追加の検証を実行するHTTPS証明書の一種。手順自体は最大2週間かかります。Know Your Clientバンキング標準と同様に、会社の住所と電話番号がチェックされます。次に、この情報が証明書に埋め込まれ、認証局のデジタル署名によって検証されます。

通常のDV証明書は、証明書で指定されたドメインを所有者が制御していることを確認するだけです。同時に、証明機関には、ドメインを所有していてドメインに連絡できない手掛かりがありません。

CN = whoami.com

これらは実際には匿名の証明書であり、HTTPS経由のトラフィックを暗号化する場合にのみ役立ちます。これらはサイトのセキュリティを示すものではありません。自動化された手順で誰でもこのような証明書を取得できます。

ブラウザーがEV SSLインジケーターを削除した理由


主な理由は、長いアドレスバーがモバイルデバイスの画面に収まらないため、ブラウザーインターフェイスを最適化する開発者の要望です。しかし、彼らは便宜上セキュリティを犠牲にすることはできませんでした。グーグルは特別な調査を開始し、それに基づいてEVの拒絶は依然としてセキュリティを低下させないと結論付けた。この調査では、UIを変更する前に青信号を点灯しました。これは、ユーザーの利便性のためにインターフェイス開発者が必要としていました。

これは、Googleのインターフェース設計とセキュリティの専門家（Chrome Security UXグループ）による調査です。彼らは「EV UIはユーザー保護を適切に提供しない」と結論付けました。

Googleは、「EVバッジは貴重な画面スペースを占有し、ユーザーインターフェースに偽の会社名を表示し、Chromeが安全な接続を示すものではなく中立的な方向に移動するのを防ぐ」と述べています。 Chrome Security UXの専門家のロジックによると、EV証明書との一致はTLSの肯定的な兆候であり、中立的な兆候はユーザーへの影響の点でより効果的です。したがって、今後、HTTPSを使用するサイトでは「ロック」アイコンがなくなり、HTTPSを使用しないサイトではセキュリティ警告が表示されるようになります。これにより、すべてのサイトでSSL証明書をインストールすることができます。

拡張チェックを与えるもの


CA安全保障理事会は、EVをアドレスバーから削除するのは間違いだと考えています。多くの主要な認証局をまとめる組織には、次の理由があります

。1.サイトが特定の会社に属していることを確認できるのはEV証明書だけです。ドメインを所有している会社がわからない場合、ユーザーの機密データを保護することはできません。

2.フィッシングに対する保護。調査によると、攻撃者はDV証明書を積極的に使用しており、自動化された匿名の手続きを通じて数千単位で無料で注文しています。

その結果、フィッシングが大規模にDV証明書に切り替わりました。FBIはすでにフィッシングサイトの半分がそのようなインジケーターを表示しているため、ユーザーはブラウザーのHTTPSロックアイコンまたは緑色のインジケーターを信頼すべきではないという警告を出しました。

同時に、EV証明書を取得することは詐欺師にとって困難です。



2019年2月にSSL証明書を使用した3494のフィッシングサイトを調査した結果は次のとおりです。

  • EV：フィッシングサイト0件（0％）
  • OV：145のフィッシングサイト（4.15％）、主にCloudflareなどのCDNマルチSANマルチドメイン証明書
  • DV：3349のフィッシングサイト（95.85％）

ブラウザーのフィッシング対策フィルターは脅威に対処できません。 2018年10月のNSS Labsの調査によると、Googleセーフブラウジングは最初から79％のフィッシングサイトしか認識していません。この割合は2日間で95％に上昇しますが、この時点までに、これらのサイトのほとんどはタスクを完了した後に機能しなくなります。



3. CA安全保障理事会は、EVインジケーターは間違いを拒否できるというGoogleの主張を考慮しますユーザーがそれを肯定的なセキュリティインジケータとして認識しないという理由で。まず、より多くの資格のあるユーザーは、証明書の種類の違いをまだ知っています。第2に、DVとEVの違いを理解していない、資格のないユーザーに知らせる努力をする価値があります。第3に、ユーザーの認識は状況に依存します。通常の快適な状態の車のシートベルトのような簡単な方法には気づきませんが、これはシートベルトを拒否する理由ではありません。

4. URLのみに依存するだけでは不十分フィッシングの認識。 Googleの調査でも、ユーザーはURLの分析に問題があり、フィッシングサイトのエラーやタイプミスに気付かないことがよくあります。同時に、EVインジケーターはURL分析を必要としません。

CA安全保障理事会は、現在のEV証明書の欠点は、それらを改善するための取り組みの言い訳であり、拒否の理由ではないと考えています。統計によると、EV証明書はフィッシングサイトを排除するのに最適です。これを効果的に使用するには、ブラウザが標準の表示に同意する必要があります、安全保障理事会は、再び自動車の世界からの類推を引用して信じています：一時停止標識が国ごとに、また州ごとに異なり、ドライバーがその意味を理解していない場合、これはすべての道路から一時停止標識を削除する理由にはなりません。彼らは効果がありません。

ブラウザの視覚的な変更にもかかわらず、EVは信頼できるセキュリティインジケータであり続けます。彼はまだ法人の有効性を確認していますが、この情報は別の場所に移動しました。この情報を表示するには、ロックアイコンをクリックする必要があります。

したがって、EV証明書の死亡の噂は時期尚早でした。セキュリティの専門家や設計者が問題を理解し、インターフェイスを調整する可能性があります。現在、この問題はブラウザ開発者と議論されています。
