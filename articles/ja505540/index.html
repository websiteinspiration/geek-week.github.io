2020年に再設計されたReact Nativeアーキテクチャ

React Native（RN）フレームワークは2015年に登場しました。 Reactライブラリを使用してクロスプラットフォームのモバイルアプリケーションを開発するように設計されています。これらのアプリケーションは、それらが作成されたプラットフォームのネイティブ機能をサポートします。元のReact Nativeアーキテクチャにはいくつかの欠陥がありました。しかし、それにもかかわらず、RNはコミュニティからの支持を得ており、特にReactの知名度のおかげで、その人気は徐々に高まっていました。



React Nativeアーキテクチャプロジェクトは2018年に話題になりました。この作業はFacebookチームによって行われます。再設計の目的は、フレームワークをより安定させ、長年の開発でRNに蓄積された最も一般的な問題を解決することです。今日公開している資料は、RNアーキテクチャーの処理によってアプリケーションのパフォーマンスとプログラマーの速度を向上させる方法を検討することに専念しています。

古い建築


React Nativeは、プラットフォームに依存しないと言えるソリューションの例です。その結果、フレームワークの主な目的は、開発者がReactメカニズムを使用してJavaScriptコードを記述できるようにすることです。React Nativeの目標は、Reactコンポーネントのツリーをさまざまなモバイルプラットフォームで機能するものに変換することです。これは、次のことを意味します。

  • 正しいインターフェース形成。
  • さまざまなプラットフォームのネイティブ機能へのアクセスを提供します。

JSとネイティブスレッド間の相互作用を整理するには、ブリッジモジュールを使用します。このモジュールはC ++で書かれており、非同期キューに基づいています。ブリッジは、いずれかのパーティからデータを受信すると、このデータをシリアル化して文字列形式に変換し、キューに渡します。データが宛先に到着すると、デシリアライズされます。

つまり、すべてのスレッドは、ブリッジを介して送信される非同期JSONメッセージに依存しています。各当事者はこれらのメッセージを送信し、将来いつかはこれらのメッセージが応答されることを期待します（ただし保証はありません）。このデータ交換スキームでは、通信チャネルが過負荷になるリスクがあります。

新しいアーキテクチャ


React Nativeアーキテクチャーの再設計中、ブリッジ機能は段階的に廃止され、JavaScriptインターフェース（JSI）と呼ばれる新しいメカニズムに置き換えられます。

JSIを使用すると、いくつかの大きな改善がもたらされます。

そのような最初の改善は、JSバンドルがJSCに依存しなくなったことです。言い換えれば、JSCエンジンは簡単に他のものに置き換えることができ、おそらくより高いパフォーマンスを備えています。たとえば-V8エンジンに。

2つ目の改善点は、新しいReact Nativeアーキテクチャの基礎となるものです。それは、JSIの使用のおかげで、JavaScriptでC ++-オブジェクト（ホストオブジェクト）へのリンクを格納し、これらのオブジェクトのメソッドを呼び出すことができるという事実にあります。その結果、アプリケーションのJavaScriptの部分とネイティブメカニズムは、以前よりもお互いをよく理解していることがわかります。

つまり、JSIはすべてのスレッド間の完全な対話を可能にします。共有所有権の概念を使用することにより、JavaScriptコードはJSストリームから直接ネイティブメソッドを実行できます。 JSONを使用してデータを転送する必要はありません。これにより、キューのオーバーフローの可能性や非同期データ転送に関連するブリッジの使用に固有の問題を取り除くことができます。


新しいReact Nativeアーキテクチャー（ソース）

上記のスキームにより異なるスレッド間の相互作用が大幅に改善されることに加えて、新しいアーキテクチャーはネイティブモジュールの直接制御も可能にします。これは、ネイティブモジュールを使用できることを意味し、アプリケーションが起動したときにすべてを初期化する必要はありません。これにより、アプリケーションの起動速度が大幅に向上します。

この新しいメカニズムは、古いRNアプリケーションでは利用できなかったことができる新しいプロジェクトの開発への道を開くかもしれません。実際のところ、私たちが自由に使えるのはC ++の力でした。つまり、React Nativeに基づいて、以前よりもはるかに多くの種類のアプリケーションを作成できるようになります。

概要


長年の開発の間に、React Nativeインフラストラクチャーには多くのものが蓄積されており、今日では時代遅れで未使用と見なすことができます。 RN開発者が不必要にそれを削除し、それをサポートする能力を向上させたいという願望は、その機能の一部を削除することにつながります。たとえば、WebviewやAsyncStorageなどのカーネルモジュールが徐々にフレームワークから削除され、React Nativeコミュニティによって開発されたリポジトリとして作成されているという事実について話しています。

新しいコンパクトコアと、JavaScriptコードとネイティブコード間の相互作用の印象的なメカニズムのおかげで、React Nativeアーキテクチャの再設計により、RNアプリケーションの生産性が向上し、開発がより速く、より便利になるという事実につながるはずです。

React Nativeのリストラは2020年第4四半期に完了する予定です。RNアーキテクチャの再設計に伴うアプリケーションのパフォーマンスと使いやすさの観点に満足しています。
