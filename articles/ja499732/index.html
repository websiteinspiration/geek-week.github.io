まず、サーバーを見てみましょう。

サーバールーチンは、プロトコルパッケージのデータ構造を使用して、TCPプロトコル上で実行されるデータサーバーとして機能します。

このルーチンは、サーバー、プロトコル、タイプのパッケージを使用します。パッケージ自体にはtcp_server.goデータ構造Serveが含まれています。

type Serve struct {
	Port string
	BufSize int
	ST *types.Settings
}

次のパラメーターを使用できます。

  • データが交換されるネットワークポート
  • JSONサーバー構成ファイル
  • デバッグモードでフラグを実行する（プライベートブロックチェーン）

進捗：

  • JSONファイルから構成を読み取る
  • デバッグモードフラグがチェックされます。設定されている場合、ネットワーク同期スケジューラは起動されず、ブロックチェーンはロードされません。
  • 構成データ構造の初期化とサーバーの起動

プロトコル


プロトコルは、ネットワーク通信中にデータを表示するツールとして機能します。

選択肢（文字列）（文字列、エラー）は、サーバーが受信したデータの初期処理を実行し、データの文字列表現を入力として受け取り、Interprete用に準備された文字列を返します。

  • 入力文字列は、ReqParseN2（str）を使用してヘッドとボディに分割されます。
  • ヘッドは要素に分割され、ReqParseHead（ヘッド）を使用してコマンドスライスに配置されます
  • ではスイッチ（コマンドは[0]）、我々は受信したコマンドを選択します（CMD、キー、アドレスやデフォルトのセクションがトリガされます）
  • cmdでは、2つのスイッチコマンド（コマンド[1]）がチェックされます-長さとgetversion。
  • 長チェックでのデータ型コマンド[2]とに格納するデータ型
  • ことをチェックし、本体は文字列値が含まれています

    len(body) < 1
  • 応答文字列を返します。 

    "result:bufferlength:" + datatype + "/" + body
  • getversionは文字列を返します

    return "result:version/auto"

通訳


これにはInterpreteData構造が含まれ、Choiceから返された文字列の2次処理とInterpreteDataオブジェクトの形成を実行します。

type InterpreteData struct {
	Head string
	Commands []string
	Body string
	IsErr bool
	ErrCode int 
	ErrMessage string
}

関数 

Interprete(str string) (*InterpreteData, error)

文字列の結果を受け入れ、InterpreteDataオブジェクトへの参照を作成します。

進捗：

  • 同様に、ChoiceはReqParseN2（str）を使用して頭と体を取得します
  • ReqParseHead（ヘッド）を使用してヘッドが要素に分割されます
  • InterpreteDataオブジェクトが初期化され、それへのポインターが返されます。

res := &InterpreteData{
	Head: head,
	Commands: commands,
	Body: body,
}
return res, nil

クライアント


クライアントパッケージには含まれていTCPConnectとTCPResponseData機能を。

関数

TCPConnect(s *types.Settings, data []byte, payload []byte)

次のように動作します：

  • 転送された設定オブジェクトで指定された接続に接続されます 

    net.Dial("tcp", s.Host + ":" + s.Port)
  • dataパラメータで渡されたデータが転送されます： 

    conn.Write(data)
  • 答えを読む 

    resp, n, _ := TCPResponseData(conn, s.BufSize)

    そしてコンソールに印刷 

    fmt.Println(string(resp[:n]))
  • 場合ペイロードが送信され、それが送信され、

    conn.Write(payload)

    また、サーバーの応答を読み取り、コンソールに出力します

関数

 TCPResponseData(conn net.Conn, bufsiz int) ([]byte, int, error)

指定されたサイズのバッファを作成し、そこでサーバーの応答を読み取り、このバッファと読み取られたバイト数、およびエラーオブジェクトを返します。

クライアントルーチン


コマンドをノードサーバーに送信し、簡単な統計とテストを取得します。

次のパラメーターを使用できます：JSON形式の構成ファイル、文字列としてサーバーに送信されるデータ、ペイロードに送信するファイルへのパス、ノードスケジューラエミュレーションフラグ、数値として転送されるデータのタイプ。

  • 構成を取得する 

    st := types.ParseConfig(*config)
  • あなたがエミューフラグを与えられている場合はsedulerを起動します
  • ファイルへのパスを示すフラグfが指定されている場合、そのデータをfdbにロードし、コンテンツをサーバーに送信します

    client.TCPConnect(st, []byte(CMD_BUFFER_LENGTH + ":" + strconv.Itoa(*t) + "/" + strconv.Itoa(fdblen)), fdb)
  • ファイルが指定されていない場合、-dフラグからのデータが単に送信されます。

    client.TCPConnect(st, []byte(*data), nil)
