とげから夢のゲームへ-生き物の進歩と進化



みなさん、こんにちは！私たちは、「夢」ゲームのもう1つの方法に取り組んでいます。これは、このテーマの3番目の投稿です。1か月後のリンクから、夢のゲームへのいばらを介して、以前の記事に慣れることができます。私は1か月強の進捗状況については書きませんでした。多くの情報が蓄積されているため、テキストは最小限ですが、写真はたくさんあります。交通を恐れないすべての人に-猫へようこそ。

このアップデートでは、先ほどお話ししたいくつかのことの小さな進化を示したいと思います。



上記は、Hagvosの初期のコンセプトです。創作当時は芸術的描写に対応しており、これで十分でした。しかし、時が経ち、ゲームの開発を開始しました。最初の章の中心的な生き物の1つは単なるブラインドであり、モデリングプロセスの前に、より精巧なコンセプトを得ることができました。





そして、完全に本格的なモデルの少し後、





感情がいっぱい詰まった数少ない瞬間の1つ。とても不思議な感じで、幸福とも言えるでしょう。昔々この生き物について書かれた最初の手紙から、あなたが最終的な結果、すでにゲームに移行したモデルを見たとき。



残りはすべて通常のものであり、まれな例外でそのような感情をもたらすことはありません。

以前の投稿で、建物、住宅のモジュール式システムの開発を開始したことを述べましたが、現在はその進捗状況を共有しています。



現在、これはすでにいくつかのバリエーションで本格的な家を建てることが可能なものです。



そして、はい、モデリングに従事している私たちのチームは、ますます多くの環境オブジェクトを作成し続けています。



もちろん、獣医も補充します。将来の生き物の1つはBolotnikです。





しかし、私たちがすでに何らかのパイプラインを手に入れて、プロセスをコンセプトからキャラクターが登場するまでのプロセスに持ち込み、Bolotnikがより明確な概要を取っていることがより重要です



私たちの宇宙の各キャラクターには、独自のストーリー、独自の運命、独自のキャラクターがあります。したがって、特定のヒーローに関する利用可能な情報に基づいて彼らの外部参照を計算し、外見でさえそれ自体が物語っているようにします。





前回のアップデート以降、ほとんどの場合、GITとUE4を結婚させ、プロジェクトで快適なコラボレーションを実現するために、Unut Muti User Edit Toolが何を食べ、最適化を照らし、カスケードシャドウが「食べる」か、なぜ「凸」テクスチャかを学びました。必ずしも良いとは限らない、彼らは4kカードが万能薬ではないことを認識し、他の同様に興味深いことをしました。

PSS：私たちは常にチームの新しい人々を歓迎します。このような大規模で野心的なプロジェクトでは、あらゆる方向からの専門家が常に必要です。私たちの活動が好きで、あなたが助けることができると思うなら、そして特にあなたが3Dアニメーターであるならば、それから書いてください、私たちは喜んで対話を確立します。
