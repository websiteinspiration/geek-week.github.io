オレンジ色の円周率ゼロのビデオ監視-安く、まったく怒らない

最近、私は次の特徴を持つビデオ監視を実装するタスクを設定しました：

  1. 最小消費電力
  2. カメラ（または複数のカメラ）からビデオを受信するデバイスの最小サイズ。たとえば、10-10 cmの取り付けボックスに詰め込んだり、幅の広いケーブルチャネルに隠したりできるほどの小さいサイズ（！）
  3. IPカメラはwi-fi経由で接続されます。つまり、IPカメラは誰からビデオを視聴/書き込みしているのかを判別できません。オフィスのアクセス照明やソケットなど、一般的なネットワークから電力を供給できます
  4. ビデオ録画（私の場合-microsdカード）が必要です。インターネットを介して世界中のどこからでも、またモバイルアプリケーションを介してビデオを視聴できなければなりません。
  5. もちろん、システム全体をリモートで構成する機能が必要です-新しいカメラの追加、録画のコピー削除、すべての設定へのアクセス権
  6. 最低コスト！しかし、どうですか？それなしではどこにも

このアイデアを実装するには、次のコンポーネントを選択する必要があります。

1）wi-fiを搭載した最も安価なシングルボードコンピューターで、任意のarm linuxディストリビューションをロールバックできます-たとえば、私のお気に入りのdebian。 Malinkaは高価ですが、ebayのオレンジのパイゼロは900ルーブルしかかかりません（数か月前に700ルーブルで購入しました）。ここにebayへのリンクがあります- ここ。



2）実際には、wifi ip-cameras-それらなしではどこにもありません。小さな、ドーム型の天井。そして、少なくとも720p-通常、オフィスポーチでのビデオ監視にはもはや必要ありません。発行価格は、カメラあたり約1,500ルーブルです。ここまでaliexpress- ここまで



3）32 GB microSDカード。なぜ32か、多かれ少なかれですか？私の場合、数日前に2台のカメラからビデオを巻き戻す必要があるためです。さて、それから、別の300-400ルーブル。

4）シングルボードおよびカメラ用の電源アダプタ。

それで、何が行われたか。大切なパッケージを郵送で受け取ったので、そこからオレンジ色の1人払いを取り出して調べましたが、最初に考えました-いまいましいですが、ビデオ出力がありません。 debianおよびビデオ監視ソフトウェアをインストールして初期設定を行う方法は？ラズベリーにはいつも普通のコンポジットAVがあったことをすぐに思い出しましたが、ここにはありません。まあ、それはメーカーがコンポジットを介して接続するような機会を残していないということではあり得ません-私は考えて、そしてGPIOピン配列を見るためにGoogleに行きました。ええ、2番目のピンは地球で、9番目はテレビアウトです。

AVソケットをはんだ付けします。今度はAV-> VGAアダプター（これはaliexpressでも購入できると思います）を使用して、モニターに接続します。ただし、まだ何も含めません。結局、microsdカードはまだ準備していません。

ウェブサイトwww.orangepi.orgにアクセスしますそして、Debian xfceデスクトップディストリビューションをダウンロードし、win32ディスクイメージャなどでカードに書き込みます。 USBフラッシュドライブをオレンジ色に差し込み、モニターの電源を入れ、電源アダプターを接続します。これはまさに、初心者が遭遇する可能性が最も高いものです。シングルボードコンピュータはどのようにも起動せず、黄色のダイオードでモックに点灯するだけです。それは何ですか？正しく記録されていないカード（たぶん）、動作していないシングルボードカード（開梱していないため、そうではない）、ビデオ出力から信号がありませんか？最も一般的な問題は栄養です。ラズベリーや他のシングルボードのものと同様に、オレンジ色のパイは非常に電力を消費します。アダプターが弱く、シングルボードをオンにして最初の数秒間で電力が低下すると、アダプターは機能しなくなり、アダプターは5 V / 2 Aになります。だから、あなたの古いアンドロイドから充電しないで、良い強力な電源を使ってください。

適切な電源アダプターを接続し、debianのダウンロードがどのように行われたかを確認します。デスクトップが見える-すばらしい！次に、デバイスをWi-Fi経由で機能するように構成する必要があります。ネットワーク接続に入り、新しいWi-Fi接続を作成し、ホームオフィスのWi-Fiからユーザー名とパスワードを入力します。これで、デバイスはローカルネットワーク上にあり、インターネットにアクセスできます。次に、モニターを取り除く必要があります。

デスクトップをリモートで表示できるようにするために、vncビューアプログラムは、シングルボードのvncサーバーの場合に役立ちます。通常、このことはすでにそのdebianに組み込まれており、単一支払者のデータの製造元によって配布されます。設定する必要があります-承認パラメーターなどを入力します。ローカルネットワーク内のデスクトップにアクセスできることを確認したら、通常のコンピューターに座ったまま、モニターを取り外し、ボックス/ケーブルチャネル/その他の場所でデバイスを非表示にして、vncビューアーからその他すべてを構成します。

ビデオ監視用のソフトウェアとして、zoneminderが選択されました。なぜ彼は正確に？無料で、柔軟で、設定が簡単で、debianを使用したシングルボードのarmバージョンで利用できます。このソフトウェアはphp / mysqlで動作するため、まず必要なパッケージをインストールする必要があります。

sudo apt-get install php5 mysql-server php-pear php5mysql php5-gd

次に、/ etc / apt / sources.listに次の行を追加する必要があります。

deb http://ftp.debian.org/debian jessie-backports main

更新を実行します。

sudo apt-get update

そして最後に、私たち全員がこれを始めたことについて：

sudo apt-get install zoneminder

コンソールからzoneminderがインストールされたことが通知されたら、それをセットアップします。構成プロセスについては、公式のzoneminder Webサイトで詳しく説明しています。この記事にすべての項目を記載しても意味がありません。親愛なる読者がこれを行ったと判断します。ここで、シングルサブスクライバー/ zmのコンピューターアドレスにブラウザーで入力すると、Webベースのビデオ監視管理パネルに移動します。ここでは、カメラのIPアドレスを追加したり、ビデオ録画のパラメーターを設定したりする必要があります。



これで、有線ネットワークとWi-Fiの両方を介して、内部ネットワークからブラウザーを介してカメラからビデオを見ることができます。
