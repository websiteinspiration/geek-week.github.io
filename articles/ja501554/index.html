セキュリティのためにSaaSサービスの個人アカウントを作成しています



SaaSの急速な発展の時代において、クラウドサービス用の個人アカウントの欠如は、すでに単に不適切と見なされています。しかし、ボット保護サービスやその他のサイバー脅威には別の重要な理由があります。それらは、クラウドストレージ（Azure、AWSなど）、仮想サーバーレンタル（DigitalOcean）、バージョン管理システム（GitHub、Gitlab）などの一般的なサービスのようには見えません。顧客がなぜお金を払うのか理解するのが難しいという理由。この質問に明確に答えることができるのは、高度な統計とグラフだけです。したがって、ヴァリティで個人アカウントを作成するプロセスには、独自の特性がありました。

なぜボット対策ソリューションユーザーの個人アカウントが必要なのですか？

仕事の透明性


Varitiを立ち上げたとき、私たちはこのサービスをユーザーにとってできるだけシンプルでわかりやすいものにしたいと考えていました。私たちの目標は、Azureのような強力な製品のユーザーインターフェイスではなく、コンシューマーサブスクリプションアプリケーションのようなものを作成することでした。なぜなら、顧客は保護が技術的にどのように機能するか気にしないからです。それが機能することが重要であり、保護の費用はその不在からの起こり得る損失をカバーします。

つまり、クライアントは透過的な統計情報を利用できるはずです。これにより、これらすべてを確認できます。

テクニカルサポートのアンロード


当社のセキュリティサービスには、多くの設定とオプションがあります。たとえば、セキュリティドメインの追加、ホワイトリストの編集、L7レベルでのフィルタリングの有効化、CDN統合、TLS 1.3のサポートの追加など。

「wwwサブドメイン」からメインドメイン（www.example.com-> example.com）への自動リダイレクトなど、一部の単純な設定は通常のオン/オフトグルスイッチで切り替えることができます。いくつかは最近書いたテレグラムボットの助けを借りて。。しかし、より複雑な設定では、クライアントからのより多くのアクションとテクニカルサポートが必要です。たとえば、ボットの軽減策-常時フィルタリングのオプションで、保護されたリソースをノックするすべてのボットを常にブロックします。しかし、すべてのボットが悪いわけではないため、ここにはいくつかのニュアンスがあります。たとえば、クライアントに独自の「ホワイト」ボットがある場合があります（データベースの同期、テレグラムボットでの設定、検索ロボットなど）。このようなシナリオはすべて、クライアントと話し合って、不注意でビジネスプロセスを中断させないようにする必要があります。

原則として、これらの問題はすべて、テクニカルサポートの助けを借りてすぐに解決できます。しかし、彼女のリソースは有限であり、複雑な設定の場合、速度が重要な場合、その場ですべての問題を解決する時間がないかもしれません。

たとえば、ハッカーが解析を目的としてボットネット攻撃を仕掛けたとします。顧客はグラフ上でリクエストの増加を確認し、現時点でボットからの保護を有効にするように依頼します（Bot Mitigation）。このオプションは時間がかかる可能性がある理由を上で説明しました。一方、別のクライアントは、単純なセットアップをオンにするか、一定期間の要求に関する統計を取得する必要がありました。

そのため、アカウント内のオプションと設定をできるだけ多く接続し、それらを引き続き追加できるようにしました。たとえば、最近LCでは、ユーザーのサーバーへのバインド（Service Iphash）を使用してバランシングモードをオンまたはオフにする機会がありました。

メインタブ


だから、これがあなたのアカウントにあるものです。

1.ダッシュボード




クライアントリソースのリストを次に示します。それらのそれぞれについて、詳細な統計を含む7つのタブが利用可能です。それらの中で最も人気のあるものを以下に示します。

リクエスト分析-リソースリクエストに関する詳細情報。リクエストフローの参加者を表示します。

  • 人間-ユーザーからのリクエスト。
  • 優れたボット-検索エンジンまたはインスタントメッセンジャーからのリクエスト。
  • 不良ボット-ブロックされたボット要求。
  • 「ホワイトリスト」からのリクエスト-クライアントが独自に追加する検証済みリソースからのリクエスト。

BandWidth-すでにクラスターを通過し、フィルターされたトラフィックに関する情報が含まれています。重要な指標は、パケットチャネルの幅の使用の95パーセンタイルであり、課金の計算の基礎でもあります。ユーザーは追加費用なしで、帯域幅を設定されたレートの5％超えることができます（チャネルが使用されている期間中）。たとえば、まれなトラフィックのバーストがあった場合、それらはこの方法で平滑化され、トラフィックの使用状況をよりリアルに把握できます。 1か月以内に、使用された帯域幅は特定の間隔で、たとえば1分ごとに測定されます。次に、月末に最大値の5％、つまりトラフィックのバーストを差し引きます。残りの95％から、支払いの計算に使用される最大数が選択されます。

応答時間と応答コード -これらのインジケーターは、時間の分布とクライアントリソースサーバーの応答コードの分布を示します。この情報は、サイトのパフォーマンスの「低下」の瞬間を判断するために使用されます。

リクエストの地理 -受信リクエストの地理的分布に関する統計。たとえば、クライアントがロシアのみにサービスを提供し、リクエストのグループがブラジルから来た場合、ボット攻撃について話す可能性が非常に高くなります。

2.サービス




このセクションでは、サービスを自分で構成できます。

たとえば、最初のタブには、メインのトラフィックフィルタリングオプションが表示されます。それらの中で最も興味深いものは次のとおりです。

  • ボットの軽減 -不正な自動リソース要求の継続的なフィルタリング
  • Web Application Firewall -XSSやSQLインジェクションなどの標的型攻撃から保護するためのサービス
  • グローバルホワイトリスト -検索エンジンボットとサイトプレビューシステムのサイトへのアクセス許可

[ エイリアス ]タブでは、domen.example.comやexample.netなどのリソースのエイリアスを追加できます。[発信元]タブでは、フィルタリングされたトラフィックの送信先となるIPアドレスのリストを構成できます。

WhiteListとBlackListには、IPアドレスのリストが含まれています。「ホワイトリスト」からのリクエストは常にサイトにスキップする必要があり、「ブラックリスト」からのリクエストは常に不可能です。それはあなたが彼らがあなたのものであるか、あなたの競争相手に正確に属していると確信しているリソースに使用することができます。

3.トラフィックマークアップ




トラフィックをマーキングする場合、アクティブボット保護テクノロジーが使用されます。事前にマークされたURLでトラフィックの「品質」をチェックします。たとえば、広告バナーを配置し、リンクを介してトラフィックを促進することを約束することが提案されています。実在の人物ではなく、ボットの適切なトラフィックを誘導し、その結果の達成に関する統計を報告する詐欺師に遭遇した場合、このタブでは、ボットまたは人物が誰に来たのかを確認できます。また、この情報は1日のコンテキストで表示できます。

4.テクニカルサポートとヘルプセンター




電話、電子メール、テレグラムに加えて、テクニカルサポートにはサポートセクションで連絡できます。新しいリクエストを作成し、スペシャリストと通信し、以前のリクエストのステータスを追跡します。ヘルプセンターでは、よくある質問への回答が記載されたFAQページが表示されます。

個人アカウントをより便利にする


2020年のSaaSサービスダッシュボードは、高速で、便利で、簡単にスケーラブルなWebアプリケーションでなければなりません。このためのすべてのツールを利用できるため、妥協はありません。これらの考慮事項に基づいて、2019年に個人アカウントのサービスを最新のAngularフレームワークに書き換えました。

これは、SPAアプリケーション（単一ページアプリケーション）を作成するためのトップツールの1つです。 SPAは、ユーザーに「見えないように」情報を更新および追加できるアーキテクチャです。たとえば、新しいステータスや新しいサービスの接続を確認するためにページを更新する必要はありません。技術的な観点から見ると、ページは一度読み込まれ、ユーザーのアクションやスクリプトに応じてコンテンツを動的に更新します。たとえば、[ダッシュボード]タブでは、ページをリロードすることなく、リクエストのグラフと統計がリアルタイムで更新されます。サイトの他のページに移動したときにも同じことが起こります。ページの必要な部分のみが更新されます。このアプローチは、データをより速く取得し、サーバーの負荷を軽減するのに役立ちます。

今年は、アカウントでいくつかの主要な更新を計画しています。すべての更新により、トラフィックフィルタリングサービスの透過的で柔軟な管理が実現します。透過的とは、単一のリクエストまで、レポートの最大の詳細を取得することを意味します。柔軟な管理-特定のURLを除外したり、特定の制限への起点トラフィックの要求を許可したりするなど、トラフィック条件を構成します。

調査結果


主な結論は非常にシンプルでオリジナルではありません。SaaSサービスは、SaaSサービス自体の開発に弾みをつけるため、できるだけ早く個人アカウントの品質と機能を改善する必要があります。例えば、個人口座を書き換えて機能を拡張したところ、3倍以上のお客様が利用し始めました。さらに、高速で便利なため、追加サービスを接続することがよくあります。

2つ目は、最新のUIフレームワーク（React、Angular、Vie）を簡単に実装およびスケーリングできることです。最初に、おそらく、最初の顧客からフィードバックを受け取るための特定のMVPが存在します。開発計画が表示されたらすぐに、すべてを「賢明に」最初からすぐに行う必要があることが重要です。

そして、最後に、絶えずフィードバックを受け取り、優先順位を付ける必要があります。まず、ロードマップの開発中に発明されなかった、要求されている機能を追加します。単純なことですが、開発側では必ずしも明白ではありません。

弊社では、機能を絶えず拡張することで、テクニカルサポートの負担を軽減できます。また、私たちが立ち止まっているわけではないことをお客様に明らかにしています。
