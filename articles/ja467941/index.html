金額の問題

変化を数える必要のあるコインを使った有名なタスクでは、ご存じのように2つの問題があります。
-1
つ目はコインの金種の数、2つ目は変化を表す数字の桁数です。
そして、これらの両方の量は、実際に計算に含まれるチューリングマシンの負荷に指数関数的な影響を与えます。
人が一度に2つの依存関係を持っていることを認識することは、アルコール依存症の社会でも受け入れられません。したがって、私はそれを安全にプレイし、これらの問題の1つを事前に取り除くことを決定しました。これがコインの金種の数になる方法。

一言で言えば、問題の本質は次のように説明されています：指数依存。つまり、これまでの金種の新しい種類の硬貨の発売は、硬貨の組み合わせの数が半分に増えることになる。別の宗派-条件付きの無限大の2倍など。インフレを駆け巡って、新しいコイン/紙幣が頻繁に発行される場合、問題を解決するにはより強力なコンピューターを購入する必要があります。そして、どこでギャロッピングインフレでそれのためにお金を得るのですか？

したがって、実際には非常に単純なソリューションです。
コインの金種に対応するポイントが印刷された0からC（変化）までのセグメントを想像すると、インテリジェントな人なら次の画像が表示さ

れます。
では、赤い点について何が言えるでしょうか。もちろん、この点のいずれかに対応する数は、変化の形で与えることができる量です。そして、コイン1枚付き。多分誰かが何か他のものを見たかもしれませんが、私はアーティストとして、この意味を正確に投資しました。そして、アーティストとして、この画像に、最初の（左から右に）点と同じ点の合計に対応する別の点を描画します（すべて正しく、面の値が2倍になります）。新しいドットは同じことを意味するため、私は同じ色で描画します-この量も変更になる可能性があります。

次に、2番目のポイントが前の合計であっても、最初のポイントを取り、2番目のポイントで要約します（ここではすべてのポイントが等しいため、これは問題ではありません）。もう一度新しいポイントを入れます。

新しいポイントがセグメントの境界を超える場合、何も描画せずに、セグメントの最初に戻り、次のポイント、つまり2番目のポイントを取得します。さらに同じ（左から右へ）。
実際、C点（変更点です）が赤に変わったら、解決策が見つかったと考えられます。そして、あなたは完全なサイクルを通過し、コインの数が最小になるように最適なソリューションを見つけることができます。プログラミングの

観点から見ると、2つのサイクルがあります。最初は0からC / 2までです（2番目のポイントは常に最初のポイントよりも大きく、合計でセグメントの境界を超えるため、最初のポイントをより大きなC / 2にする必要はありません）。 2番目のサイクルは最初のサイクルに組み込まれ、外部サイクルが指すのと同じポイントから始まり、合計がセグメントの境界を離れるまで続きます。
実際、これはバストです。1つのオプションを失うことはなく、最適なソリューションを見つけることが保証されます。そうでない場合、ソリューションはないと結論付けます。ループ内の反復量を

数えましょう。外部サイクルでは-これはC / 2で、内部では-どこか同じです。 C / 2 * C / 2 =（C ^ 2）/ 4.を乗算します。Cの2乗に丸めます。これは、セグメント全体が単に赤い点で構成されている場合の最悪のオプションです。ポイント間にスペースがある場合、反復回数は大幅に減少します。
ご覧のとおり、問題の解決の難しさを判断する際に、硬貨の金種数は使用していません。この値は、ソリューションの複雑さに直接影響しません。これらの宗派の価値は、例えば1セント硬貨に影響を与え、このセグメントを完全に赤くします。したがって、このコインは考慮に入れず、決定の最後に、Cに最も近い赤い点を取り、1セントの上にスケッチします。しかし、これはすでにアルゴリズムの最適化の瞬間であり、この記事の範囲を超えています。
