React Native-それは何ですか？


2015年、FacebookはオープンソースのReact Nativeフレームワークを発表し、リリースしました。以前は、モバイルアプリケーションの開発のために既に内部的に使用されていました。 React Nativeは、React jsライブラリを使用してネイティブに表示されるモバイルアプリケーションをクロスプラットフォームで開発するためのフレームワークです。したがって、これにより、トップのjs-libraryを使用して、ネイティブモバイルアプリケーションのユーザーインターフェイスを作成できます。

Shopifyでは、この考え（そして、ある意味では、現在も）は懐疑的でしたが、多くの人がそれを有望であると考えていました。次のShopify Hackdaysイベント完全にReact Native専用でした。その後、チームはフレームワークのメリットを高く評価しましたが、2015年にはReact Nativeアプリケーションを実装できないと判断しました。これは主にパフォーマンスの問題とAndroid OSの完全なサポートの欠如が原因でした。しかし、私たちは明らかに、リアクティブプログラミングパラダイムとGraphQLが好きであることに気づきました。さらに、React Nativeとのミーティング後、iOS用のレンダラーを開発してレイアウトしました。 2015年、これらのテクノロジーをスタックに実装しましたが、大規模モバイル開発にReact Nativeを使用しませんでした。私たちの最初の実験はここに詳しく書かれています。

今日まで、Shopifyは標準としてネイティブモバイル開発を行ってきました。モバイルツールとファウンデーションの 2つのチームを作成しました。これらはiOSとAndroidに重点を置いており、開発プロセスのスピードアップにも役立ちました。彼らが良い結果を示したという事実にもかかわらず、私たちが次の場合に開発がより効果的になるように思われます。

  • 私たちは、JavaScriptやその他のウェブ技術をモバイル開発に利用しています。
  • クライアントアプリケーションの開発にリアクティブプログラミングパラダイムを導入する。
  • React NativeでのiOSとAndroidの開発を1つの技術スタックに統合します。

React Nativeはどのように機能しますか？


React Nativeを使用すると、JavaScriptを使用して、クロスプラットフォームでネイティブに表示されるモバイルアプリケーションを作成できます。React NativeはReactに似ています。ReactJSの観点から、UI要素のツリーまたは仮想DOM（VDOM）を使用して、JSでユーザーインターフェイスを宣言的に作成します。ReactJSがインターフェイス要素をブラウザーに表示する場合、React NativeはVDOMオブジェクトをネイティブのプラットフォーム固有の視覚要素に変換し、バインダーを使用してモバイルデバイスに表示します。ネイティブコンポーネントはReactでラップされます。私たちはAndroidとiOSのプラットフォームにのみ関心がありますが、Windows、macOS、およびApple tvOS用のReact Nativeの開発に関心がある愛好家がいます。


ReactJSはブラウザーと対話し、React NativeはモバイルAPIと対話します。

React Nativeを使用しないのはいつですか？


Shopifyチームがモバイルアプリケーションの開発にReact Nativeを使用しない状況があります。特に、以下が必要な場合：

  • 古いハードウェア（CPU <1.5 GHz）でアプリケーションを起動する。
  • 洗練された処理;
  • 超高性能;
  • 多くのスレッドがバックグラウンドで実行されています。

注意：さまざまなオープンソースSDKを含む低レベルのライブラリは、完全にネイティブのままです。ハードウェアとより密接に接触したい場合は、独自のネイティブモジュールを作成できます。

なぜ今、React Nativeに切り替えるのですか？


主な理由は3つあります。

  1. 2018年にTictailマーケットプレイス（サービスは最初にモバイルの原則に基づいてReact Nativeで開発されました）を取得した後、React Nativeがどのような開発を受け取り、2019年にさらに3つのプロジェクトに投資しました。
  2. ShopifyはWeb開発でReactを積極的に使用しています。モバイルアプリケーションの開発にノウハウを適用するときが来ました。
  3. かつてRubyを信じていたのと同じように、パフォーマンス曲線がクロールして（たとえば、以前は何ができ、今ではデスクトップMicrosoft Officeと比較してGoogleドキュメントで何ができるか）、React Nativeに賭けていることに気付きました。 Rails、Kubernetes、Rich Media。

2019年のShopifyでのモバイル開発


Shopifyでは、売り手と買い手はいくつかの方法でやり取りできます：Webおよびモバイルアプリケーションを介して。昨年、私たちの3つのチームはReact Nativeを試していました。その結果、彼らは3つのモバイルアプリケーションを開発しました：到着、POS、コンパス。

実験後、次の結論を出しました。

  • ArriveアプリをReact Nativeにコピーすると、チームは、1つのプラットフォーム用にのみ作成した場合でも、ネイティブ開発に比べて生産性が2倍になったことを発見しました。
  • AndroidバージョンのPOSアプリを低電力でテストしたところ、アプリケーションがより低いCPU周波数で動作することがわかりました。1.5GHz（React Nativeで作成）と2 GHz（ネイティブ開発）です。
  • プラットフォームに依存しないコードの割合は約80％になると暫定的に推定しました。そして、実際にはそれがはるかに高いことが判明したときに驚きました：到着では95％、コンパスでは99％。

ただし、予約する価値はあります。ReactNativeで新しいアプリケーションを作成することを決定しましたが、これは古いアプリケーションをすべて書き換えることを意味するものではありません。

到着


2018年の終わりに、最も人気のあるクライアントアプリケーションの1つをReact Native、Arriveに書き直すことにしました。これは、何百万ものダウンロード数を誇る高性能で評価の高いiOSアプリです。Androidバージョンがなかったため、到着を選択しました。そこで、Androidデバイスのユーザーに会いに行きました。現在、iOSバージョンとAndroidバージョンの両方がReact Nativeで記述されており、クロスプラットフォームコード全体の95％を占めています。

書き換えの過程で、次の結果が得られました。

  • iOSのクロスプラットフォームバージョンを実行すると、ネイティブバージョンよりもクラッシュが少なくなります。
  • Android OS用のバージョンをリリースしました。
  • モバイル開発者と非モバイル開発者の両方がプロジェクトに取り組むことができました。

さらに、チームは「作業中」のプルリクエストをすばやく実行するための優れた方法を試すことができました。スマートフォンを使用してGithubコメントからQRコードをスキャンするだけで、アプリケーションのJSバンドルが更新されます。したがって、アプリケーションを即座に更新して、その現在のバージョンを使用することができます。サービスステーションは最近、このプロセスをTwitterで説明しました。

POS


2019年の初めに、主力製品のPOSアプリで 6週間の実験を行い、React Nativeでの書き換えに適した候補かどうかを確認しました。私たちの小売業者はアプリケーションの応答時間を半分にすることができると期待していたので、私たちはこの質問をたくさん研究しました：アプリケーションを使用する顧客サービスのプロセスは売り手の筋肉の記憶に固定されるべきであり、これに気を取られるべきではありません。

小売業者の要件を満たし、同時にPOSデバイス用のReact Nativeの開発を理解するために、iOS用の新しいネイティブモバイルアプリケーションを作成し、React NativeでAndroid OS用のバージョンを作成することにしました。

これらのタスクに2つの別々のチームを割り当てましたが、その理由は次のとおりです。

  1. 私たちはすでに、ネイティブPOSアプリケーションの開発者を含め、iOSに関する強力な専門知識を持つチームを持っていました。
  2. 直接的なアプリケーションのパフォーマンスに加えて、React Nativeでの開発速度をテストして、これらすべてをゴールドスタンダード、つまりネイティブiOS開発と比較したいと考えました。
  3. 売り手の高いパフォーマンス要件を満たすために、すべてのFacebook更新をReact Nativeのアーキテクチャの更新に再作成することにしました（ただし、これはパフォーマンスにとって重要ではないことが判明しました）。2つのプロジェクトで作業する別々のチームを特定することにより、リリース計画の中断に関連するリスクを軽減しました。

Unite 2019 で書き直されたPOSを発表しました。React NativeのiOSとAndroidの両方のバージョンは、2020年に利用可能になります。
