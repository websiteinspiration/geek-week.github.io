無線オブジェクト通信モジュールWISE-4000



プログラミングスキルがなくても、産業レベルのデータ収集システムを展開できますか？この記事では、数回のクリックでJSON APIを介して既製のデジタル信号処理システムを利用できるようにするデバイスについて説明します。WISE-4000

シリーズは、リモートオブジェクトからのデータをログに記録、監視、送信するためのI / Oデバイスです。業界プロトコルMQTT、Modbus / TCPをサポートし、HTTP JSON APIを備えています。クラウドサービスMicrosoft Azure、Dropboxなどとの統合があります。 デバイスは、SCADAの一部としても自律的にも機能します。WiFi経由のサポートされているネットワーク接続。

初期設定はWebインターフェイスを介して行われます。デバイスは、WISE- <モデル<-<MAC_address>形式のネットワーク名（SSID）でアクセスポイントモードに切り替わり、通常のWi-Fiルーターと同様にDHCPサーバーをアクティブにします。 WISE-4000を初期モードにするには、背面カバーのDIPスイッチSW1-P1をオフに設定する必要があります。

セットアップモードに切り替えても、Webインターフェイスのパスワードは工場出荷時のパスワードにリセットされないことに注意してください。設定をリセットするには、Windows Adam / Apaxでユーティリティを使用する必要があります。ネットユーティリティ。

Webインターフェイスでは、Wi-Fiクライアントモード（インフラストラクチャモード）またはアクセスポイントモード（APモード）を設定できます。


クライアントモードでは、デバイスが接続を試みる2つの異なるWi-Fiネットワークを設定できます。2番目のネットワークとして、構成にサービスネットワークを使用すると便利です。サービススタンドにいるときにデバイスが自動的に接続されます。ネットワークの設定後、DIPスイッチSW1-P1をONに設定できます。

入力/出力ポート


Modbus / RTUマスター用の8つのデジタル入力（DI）ポートと1つのRS-485ポートを備え たWISE-4051モデルの例を使用して、I / Oポートの割り当てを調べてみましょう。このモデルの主な目的は、センサーからデータを収集し、リモートオブジェクトを監視することです。

論理レベル

ポート0〜3はドライ接点タイプです。論理ユニットはDCOM0との閉じた接点であり、論理0はそれぞれ開いた開いた接点です。リードスイッチ、リレー、ボタンをこの接点に接続できます。

ポート4〜7はウェットコンタクトタイプです。論理ユニットは10〜30Vと見なされ、論理0はDCOM1の接点とグランドの間で0〜3Vです。

すべてのポートは、ロジックレベルの反転と最大3 kHzの周波数のパルス信号をサポートします。


ポート割り当てWISE-4051

I / Oモード


各デジタル入力は、入力信号を処理する5つのモードの1つで独立して動作できます。選択したモードに応じて、受信信号はさまざまな方法でデバイスのメモリに保存されます。各モードでは、論理0と1を反転できます。


DI-従来のデジタル入力


このモードでは、接点のステータスを低/高で読み取ることができます。さらに、10分の1ミリ秒単位で最小および最大の信号長を設定することにより、干渉フィルターをアクティブにすることができます。


カウンター


カウンターモードは、受信した信号の数を順次カウントし、それらの数を保存します。さらに、変数の開始値を設定できます。このモードは、さまざまなドアの開度を評価するためのリードスイッチの操作など、定期的なイベントのカウントに使用すると便利です。


デジタル入力カウンターモード

アラームモード（ラッチ）


このモードでは、単一の信号が、APIまたは手動で強制的にリセットされるまで、入力状態を操作モードに変更できません。このモードは、アラーム、ヒューズ、および人的介入を必要とする信号に役立ちます。


アラームモードの2つのデジタル接点。ピン0はアラーム状態です。

周波数測定モード


信号周波数の現在の値を0.1〜0.01 Hzの精度で保存します。さらに、値をゼロにする時間間隔を設定できます。このモードは、パルス信号、回転カウンタ、モーター、振動などに使用できます。



ローカルストレージ


サーバーとの接続が失われた場合、または完全に自律的に動作する場合、デバイスは受信信号を内部メモリに保存できます。バックアップバッテリを使用すると、主電源がオフのときにデータを揮発性メモリに保存できます。

ローカルストレージのリクエストは、ウェブインターフェース、Modbus / TCP、HTTP RESTful APIを介して行うことができます。


Webインターフェイスを介したローカルストレージ要求。

したがって、プログラミングスキルや追加のソフトウェアをインストールすることなく、Webインターフェイスのみを使用して、5分で最新のシステムに簡単に統合できるシンプルなデータ収集とストレージデバイスをセットアップできました。この例は非常に原始的であるという事実にもかかわらず、構成の容易さの一般的な理解を提供します。同じ原理により、異種のデータタイプと動作モードを備えた、より複雑なシステムを展開できます。

クラウドプロバイダー


WISE-4000 シリーズデバイスは、クラウドサービスとの統合をサポートしています。収集されたデータは、MQTTプロトコルを介してクラウドストレージに転送するか、ファイルとしてドロップボックスまたは内部サーバーにアップロードできます。Advantech WebAccess SCADAシステムとの統合もサポートします

デモI / Oボード


開発者がハードウェアレベルでI / O信号を個別にシミュレーションできるように、キットにはデモボードが含まれています。2つのポテンショメータは、アナログ出力、ディスクリート入力用のDIPスイッチ、およびリレー出力用の2つのLEDのデバッグ用に設計されています。これにより、余計な配線なしで、完全なテストベンチを職場で得ることができます。



結論


WISE-4000は、データ取得および送信デバイス（DTD）とオブジェクトとのリモート通信用デバイス（USO）の機能を組み合わせています。セットアップにはプログラミングのスキルは必要ありません。スマートフォンからWebインターフェイスを介して最短時間でセットアップできます。最新のプロトコルのサポートにより、デバイスを最新のSCADAおよびクラウドシステムに簡単に統合できます。
