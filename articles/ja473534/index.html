OEM情報の変更を自動化する


ラップトップを購入して初めてWindowsを更新したところ、システムに表示されていたかなりのメーカーのロゴが突然どこかに消えてしまった。

だから私はOEMが何であるかを知りました...

私が失った美しいアイコンは、Windowsの更新後に消去されたOEM情報の一部であることをGoogleはすぐに説明しました（これは実際には論理的です）。Windowsの

OEM（相手先ブランド供給）は、次のパラメーターによって特徴付けられます。

  • メーカー -メーカー
  • モデル -デバイスモデル
  • SupportHours-テクニカルサポートの時間
  • SupportPhone-テクニカルサポートの電話番号
  • SupportURL-テクニカルサポートサイトのアドレス
  • ロゴ -製造元のロゴ（そのすべてが始まったため）

文字列値を使用 してレジストリ[HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows \ CurrentVersion \ OEMInformation] を通じてハンドルを追加または変更できるこのすべての情報。 すべてのパラメーターには1行あたり256文字の制限があり、ロゴは.bmp形式で、解像度は120x120ピクセルにする必要があります。




または、これらの目的のために、既成のユーティリティ（主に自己記述型）を使用できます。

最初のオプションはすべてを正面から行うために提供されたため、私は両方のオプションが好きではありませんでした。2番目のケースでは、サードパーティの認定されていないソフトウェアは自信を刺激しませんでした

。ただし、注意点が1つあります。突然、私と同じ問題に遭遇した人から上記の不利な点がなくなるように、ソースをレイアウトしてください。

数日の間に、割り当てられたすべての機能を実行する最小限のアプリケーションができました。


私も注意してしまうように、「読み込み設定」ボタンをすべて読み込むことができます、OEMの設定ファイルからパラメータ（.exe.configを）。

おそらく、それはこのボタンがなければ、その後のポストがないでしょう。実際のところ、現在の職場ではソフトウェアとハ​​ードウェアを一緒に提供することがあるので、OEM情報の導入は緊急の作業です（ただし、誰も考えたことがないのは奇妙ですが）。

否や言うほどない。ガイドは承認されました。今、この赤ちゃんは良いのために働いているの会社。

この投稿では、OEM情報とは何か、どのように変更できるかを説明するだけでなく、簡単な例を使って簡単な考えを伝えたいと思います。アイデアがあります-実装してください！引き出しに入れないでください。結局のところ、あなたが直面している問題は他の人に関係があるかもしれません。
