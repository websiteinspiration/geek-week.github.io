WPF DataGrid。テンプレートのために戦う

みなさん、こんにちは！

私の記事を通じて、DataGridのXML表現とそのコードコレクションをリンクする問題を明確にしたいと思います。このため、特定の例を使用するのが最善です。したがって、いくつかの構造の要素のリストをテーブルに配置する必要があります。要素自体がコードで次のようになっているとしましょう：

class Element
    {
       public Element (string firstName, string lastName, string phoneNumber, DateTime date)
        {
            this.firstName = firstName;
            this.lastName = lastName;
            this.phoneNumber = phoneNumber;
            this.date = date;
         }
       
        public string FirstName { get { return this.firstName; } set { this.firstName = value; } }
        public string LastName { get { return this.lastName; } set { this.lastName = value; } }
        public string PhoneNumber { get { return this.phoneNumber; } set { this.phoneNumber = value; } }
        public DateTime Date { get { return this.date; } set { this.date = value; } }
        
        private string firstName;
        private string lastName;
        private string phoneNumber;
        private DateTime date;
        
    }

したがって、それらのリストは次のようになります。


List<Element> elements = new List<Element>();

次に、DataGridに移りましょう。これはまだWPFなので、手でフォームに引っ張ることができます。IDEにコードを記述させます。

<DataGrid x:Name="dataGrid" Margin="10,103,0,15" 
                  CellEditEnding="dataGrid_CellEditEnding" Grid.RowSpan="2" Grid.ColumnSpan="2" HorizontalAlignment="Left" Width="728"
                  AutoGenerateColumns="False"/>

ここでは、AutoGenerateColumns =“ False”のみを手動で設定します。将来的には、テーブルの列を自分で編集するためです。

次に、データをテーブルにバインドする必要があります。要素の各フィールドはテーブルの列に対応している必要があります。

<DataGrid.Columns>
                <DataGridTextColumn Header="" Binding="{Binding Path=FirstName}"/>
                <DataGridTextColumn Header="" Binding="{Binding Path=LastName}"/>
                <DataGridTemplateColumn Header="" SortMemberPath="PhoneNumber">
                    <DataGridTemplateColumn.CellTemplate>
                        <DataTemplate>
                            <TextBox Text="{Binding Path=PhoneNumber}"/>
                        </DataTemplate>
                    </DataGridTemplateColumn.CellTemplate>
                    <DataGridTemplateColumn.CellEditingTemplate>
                        <DataTemplate>
                            <TextBox Text="{Binding Path=PhoneNumber}" />
                        </DataTemplate>
                    </DataGridTemplateColumn.CellEditingTemplate>
                </DataGridTemplateColumn>
                <DataGridTemplateColumn Header="" SortMemberPath="Date">
                    <DataGridTemplateColumn.CellTemplate>
                        <DataTemplate >
                            <TextBlock Text="{Binding Path=Date, StringFormat='dd/MM/yyyy'}"/>
                        </DataTemplate>
                    </DataGridTemplateColumn.CellTemplate>
                    <DataGridTemplateColumn.CellEditingTemplate>
                        <DataTemplate>
                            <DatePicker SelectedDate="{Binding Date}"/>
                        </DataTemplate>
                    </DataGridTemplateColumn.CellEditingTemplate>
                </DataGridTemplateColumn>
                
            </DataGrid.Columns>

ここでは、DataGridTextColumnとDataGridTemplateColumnの2つのタイプの列が使用されます。あなたはどこでもそれらのタイプについて読むことができます。ここでの主なことは、DataGridTemplateColumnを使用して任意のテンプレートを設定できることを理解することです。つまり、この列のセルには、任意のコントロール（ボタン、カレンダーなど）を配置できます。たとえば、3番目の列はTextBoxです。

<DataGridTemplateColumn Header="" SortMemberPath="PhoneNumber">
                    <DataGridTemplateColumn.CellTemplate>
                        <DataTemplate>
                            <TextBox Text="{Binding Path=PhoneNumber}"/>
                        </DataTemplate>
                    </DataGridTemplateColumn.CellTemplate>

重要：自動補完が無効になっているため、ヘッダーを登録することを忘れないでください。また、DataGridTemplateColumnの場合、SortMemberPathを登録して、どのフィールドでデータをソートするかがわかるようにする必要があります。

バインドに移りましょう。これはデータバインディング自体です。 Binding Path = PhoneNumberを使用して指定します。ここで考慮すべき主なことは、これがフィールドの値を返すクラスメソッドの名前であることです（phoneNumberではなくPhoneNumber、クラスの説明を参照）。
そして一瞬。通常の状態（CellTemplate）と編集状態（CellEditingTemplate）のセルに異なるタイプのテンプレート（DataTemplate）を入力することができます。通常の状態のセルの4番目の列はTextBoxで、編集状態はDatePickerです。

<DataGridTemplateColumn Header="" SortMemberPath="Date">
                    <DataGridTemplateColumn.CellTemplate>
                        <DataTemplate >
                            <TextBlock Text="{Binding Path=Date, StringFormat='dd/MM/yyyy'}"/>
                        </DataTemplate>
                    </DataGridTemplateColumn.CellTemplate>
                    <DataGridTemplateColumn.CellEditingTemplate>
                        <DataTemplate>
                            <DatePicker SelectedDate="{Binding Date}"/>
                        </DataTemplate>
                    </DataGridTemplateColumn.CellEditingTemplate>
                </DataGridTemplateColumn>

また、日付を操作する場合、Textフィールドにはフォーマット可能なStringFormatプロパティがあることに注意してください。月を正しく表示するには、 "MM"を大きくすることが重要です。

ここで、特定のデータをテーブルにバインドするために、C＃のコードに戻りましょう。
配列に要素を入力したら、データソースとして指定する必要があります。

dataGrid.ItemsSource = elements;

したがって、要素への変更はテーブルに表示され、逆に、テーブルに加えられた変更は要素の変更につながります。ただし、要素に変更を加える場合は、メソッドを呼び出す必要があります
dataGrid.Items.Refresh();
。
C＃バンドルのもう1つの良い例として、XMLを使用して、電話番号のセルが数字のみで埋められていることを確認する例を示します。これを行うには、3列目のコードを少し編集します。

<DataGridTemplateColumn.CellTemplate>
                        <DataTemplate>
                            <TextBox Text="{Binding Path=PhoneNumber}" TextChanged="phN_TextChanged"/>
                        </DataTemplate>
                    </DataGridTemplateColumn.CellTemplate>

TextBoxタグで、テキスト変更イベントのハンドラーを割り当てたことがわかります。コードにこのハンドラーを登録するだけです。

 private void phN_TextChanged(object sender, TextChangedEventArgs e)
        {
            int result = 0;
            TextBox txtx = sender as TextBox;
            if (txtx !=null)
            {
                if (!int.TryParse(txtx.Text, out result)) {
                    txtx.Text = txtx.Text.Substring(0, txtx.Text.Length - 1);
                    txtx.CaretIndex = txtx.Text.Length;
                }
            }
        }

数値のチェックは、正規表現を使用して行うこともできます。
