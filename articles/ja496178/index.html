Swift 5.2。すべての変更の概要

3月末に、Swift 5.2がXcode 11.4用にリリースされました。エラー診断、依存関係分析が改善され、SwiftPM機能が拡張されました。いくつかの変更の概要はHabréで既に公開されています。同じ記事では、使用例を含む言語自体の進化が検討されています。


 

SE-0249関数としてのKeyPath


ショートカットはkeyPathを関数に入れます。ここで、入力パラメーターはオブジェクト自体であり、結果はそのプロパティです。

例を見てみましょう。単純なモデルの配列を作成します。

struct Model {
    let isHidden: Bool
}
 
let modelArray = [Model(isHidden: true), Model(isHidden: false)]

isHiddenプロパティで配列をフィルタリングします。以下は、同じ結果の3つの例です。

// 1
let funcIsHidden: (Model) -> Bool = \.isHidden
modelArray.filter(funcIsHidden)
 
// 2
modelArray.filter(\.isHidden as (Model) -> Bool)
 
// 3
modelArray.filter(\.isHidden)

宣言された機能は、次の例では機能しません。

//   
let selfFunc: (Model) -> Model = \.self
modelArray.compactMap(selfFunc)
 
// ERROR: Expression type '(Model) -> Model' is ambiguous without more context
modelArray.compactMap(\.self as (Model) -> Model)
 
// ERROR: Key path value type 'Optional<_>' cannot be converted to contextual type '_'
modelArray.compactMap(\.self)

また、keyPathとは異なり、オートコンプリートは機能しません。

filter、map、reduce、sortなどの関数で配列を操作する場合に使用すると便利です。

SR-6118の添え字にデフォルト設定が含まれる場合がある


すべての関数パラメーターをデフォルト値に設定できます。要素の配列とそれらにアクセスするための添え字関数を含むBox構造を作成します。

struct Box {
    let items: [String]
    
    subscript(_ index: Int = 0) -> String {
        items[index]
    }
}

ここで、最初の要素にアクセスするには、インデックスを省略できます。

let box = Box(items: ["laptop, , mobile phone"])
let firstItem = box[] // "laptop"

SR-2189ローカル関数は外部の可視性からのデフォルトパラメータをサポートします


日常業務では、ローカル機能はほとんど使用されません。それらの使用を正当化するのは難しい場合があります。それにもかかわらず、私は特定のケースでこれが便利になるかもしれないことを認めます。

例として、ローカル関数を記述する関数を作成します。

func getXPosition() {
    func calculateWidth(x: CGFloat = minX) -> CGFloat { ... }
    let minX: CGFloat = 0
    ...
}

calculateWidth関数のデフォルトパラメータとして、getXPosition関数内の値を使用できます。

SE-0253関数としての値の使用


@dynamicCallableと同様の機能により、値を関数として使用できます。しかし、それは「静的呼び出し」の実装です。

実際には、すべてが非常に簡単です。関数を呼び出すには、値をメソッドとして参照できます。

プレーヤー構造を作成します。

struct Player {
    private(set) var isRunning = false
    
    mutating func callAsFunction() {
        isRunning.toggle()
    }
}

Playerのインスタンスを作成し、それを関数として参照します。

var player = Player()
print(player.isRunning) // false
player()
print(player.isRunning) // true

同時に、キャストすることは禁止されているため、オブジェクトを関数として渡すことができます。

// ERROR: Cannot convert value of type 'Player' to type '() -> Void' in coercion
let playerAsFunc = player as () -> Void

クラス、構造、またはプロトコルには、callAsFunctionと同じ数のメソッドを追加できます。

extension Player {
    func callAsFunction(isRunning: Bool = false) throws { ... }
    func callAsFunction() -> Bool { ... }
}

数学関数、複雑な式である値、または値が1つの主要な関数を持つ場合に適用できます。それでも、誤解を招く可能性があるため、この機能を悪用しないでください。

SR-4206ジェネリックパラメーターでオーバーライドする関数のバグを修正


これで、関数をオーバーライドして、ジェネリック型の制限を変更または追加できなくなりました。例として、CarBuilderクラスを作成し、addメソッドをオーバーライドすることによってそれから継承します。

protocol PowerfullEngine { }
 
class CarBuilder {
  func add<T>(engine: T) {}
}
 
class MercedesBuilder: CarBuilder {
    //  
    override func add<T>(engine: T) {}
    // ERROR: Overridden method 'add' has generic signature <T where T : PowerfullEngine> which is incompatible with base method's generic signature <T>; expected generic signature to be <T>
    override func add<T: PowerfullEngine>(engine: T) {}
}

SR-11298クラス制限なしでプロトコルを拡張すると、この制限が継承されます


Selfに対する制限の場合、拡張機能は制限を適用します。

たとえば、クラス制限付きのメニューと拡張プロトコルを作成します。

protocol Menu {}
 
class SideMenu: Menu {
  var sectionHeight = 0
}
 
extension Menu where Self: SideMenu {
  var menuHeight: Int {
    get { sectionHeight * 20 }
    set { sectionHeight = newValue / 20 }
  }
}

同時に、セッターは、プロトコルにクラス制限があるかのように、変化しません。

SR-11429ラベル付き関数のキャスト


タイプラベルに関数をキャストするときに削除されます。これにより、ラベル付きの関数をキャストできます。以前は、この機能はラベルのない関数でのみ機能していました：

func setX(x: Int) {}
(setX as (Int) -> Void)(5)

ただし、デフォルト値は保存されません：

func setPoint(x: Int, y: Int = 0) {}
(setPoint as (Int, Int) -> Void)(5, 1)

これはジェネリックにも適用されるため、このコードは無効になりました。

typealias Point<T> = T
func set(x: Int) {}
 
//  
(set as Point)(5) 
 
// ERROR: Extraneous argument label 'x:' in call
(set as Point)(x: 5)

SR-11841フィルター（_ :)関数は、遅延コレクションで予期された順序で呼び出されます


レイジーコレクションを作成し、countメソッドを呼び出します。

let lazyArray = [0]
    .lazy
    .filter { _ in print("A"); return true }
    .filter { _ in print("B"); return true }
 
//  A B
_ = lazyArray.count

以前は、呼び出しの結果は反対でした：BA。

SR-2790 UnsafePointer / UnsafeBufferPointerファミリーの多くのタイプ初期化子が警告を発行するようになりました


この制限は、関数呼び出しの外部に存在しない文字列、配列、および入出力パラメーターに適用されます。

初期化中にパラメータとしてUnsafePointerを受け取る構造を作成します。

struct Game {
    let info: UnsafePointer<Int8>
}
 
func createGame() {
    var i: Int8 = 0
    
    // WARNING: Initialization of 'UnsafePointer<Int8>' results in a dangling pointer
    _ = Game(info: .init(&i))
    
    // WARNING: Passing '[Int8]' to parameter, but argument 'character' should be a pointer that outlives the call to 'init(character:)'
    _ = Game(info: [1, 2])
    
    // WARNING: Passing 'String' to parameter, but argument 'character' should be a pointer that outlives the call to 'init(character:)
    _ = Game(info: "")
}

このバージョンでは9つの変更がありましたが、新しい機能が確実に導入されました。現在使用されているものの中で最も使用されているのは、関数としてのKeyPathだと思います。

そして、次のバージョンを楽しみにしています。どうやら、それはWindows上のSwiftをサポートし、selfを介してアクセスせずにクロージャをエスケープする際のselfの使用などの興味深い機能が表示されます。*（SE-0269）およびジェネリック（SE-0267）の機能が拡張されます。
