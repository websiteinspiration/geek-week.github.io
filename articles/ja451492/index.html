7つの予期しないBash変数

あまり知ら れていない bash 関数についての一連のメモを続けて、知らないかもしれない7つの変数を示します。

1） PROMPT_COMMAND


プロンプトプロンプトを操作してさまざまな有用な情報を表示する方法はすでに知っているかもしれませんが、プロンプトを表示するたびにシェルコマンドを実行できることを誰もが知っているわけではありません。

実際、多くの複雑なプロンプトマニピュレータはこの変数を使用してコマンドを実行し、プロンプトに表示される情報を収集します。

これを新しいシェルで実行してみると、セッションで何が起こるかがわかります。

$ PROMPT_COMMAND='echo -n "writing the prompt at " && date'

2） HISTTIMEFORMAT


あなたが実行した場合history、コンソールには、以前のアカウントの下で実行されたコマンドのリストを取得します。

$ HISTTIMEFORMAT='I ran this at: %d/%m/%y %T '

この変数が設定されるとすぐに、新しいレコードはコマンドとともに時間を記録するため、出力は次のようになります。

1871  I ran this at: 01/05/19 13:38:07 cat /etc/resolv.conf
1872  I ran this at: 01/05/19 13:38:19 curl bbc.co.uk
1873  I ran this at: 01/05/19 13:38:41 sudo vi /etc/resolv.conf
1874  I ran this at: 01/05/19 13:39:18 curl -vvv bbc.co.uk
1876  I ran this at: 01/05/19 13:39:25 sudo su -

書式設定は、日付の文字と一致します。

3） CDPATH


コマンドラインの時間を節約するために、この変数を使用して、コマンドを呼び出すのと同じくらい簡単にディレクトリを変更できます。

のようPATHに、変数CDPATHはパスのコロンで区切られたリストです。cd相対パスを指定して（つまり、先頭にスラッシュを付けずに）コマンドを実行すると、デフォルトで、シェルはローカルフォルダーで対応する名前を探します。CDPATHあなたが行きたいディレクトリのためにあなたが与えたパスを検索します。このように

設定CDPATHした場合：

$ CDPATH=/:/lib

次に入力します。

$ cd /home
$ cd tmp

そうすれば/tmp、どこにいても必ず陥ります。

ただし、リスト.でローカル（）フォルダーを指定しないtmpと、通常どおり他のフォルダーを作成してそこに移動できないため、注意してください。

$ cd /home
$ mkdir tmp
$ cd tmp
$ pwd
/tmp

おっとっと！

これは、ローカルフォルダーがおなじみの変数に含まれていないことに気付いたときの混乱と似てPATHいますが、ダウンロードしたコードから偽のコマンドを実行することでだまされる可能性があるため、PATH変数で行う必要があります。

鉱山が出発点として設定されます：

CDPATH=.:/space:/etc:/var/lib:/usr/share:/opt

4） SHLVL


exit現在のbashシェルから別の「親」シェルに移動するのか、コンソールウィンドウを完全に閉じるの か疑問に思ったことはありませんか？

この変数は、bashシェルにどの程度深く埋め込まれているかを追跡します。新しいターミナルを作成すると、1に設定されます。

$ echo $SHLVL
1

次に、別のシェルプロセスを開始すると、数が増加します。

$ bash
$ echo $SHLVL
2

これは、終了するかどうかが不明なスクリプトや、ネストによって現在の場所を追跡するスクリプトで非常に役立ちます。

5） LINENO


また、現在の状態とデバッグを分析するにはLINENO、セッションで現在までに実行されたコマンドの数を報告する変数が役立ちます。

$ bash
$ echo $LINENO
1
$ echo $LINENO
2

これは、スクリプトのデバッグ時に最もよく使用されます。などの行を挿入するecho DEBUG:$LINENOと、スクリプトのどこにいるか（またはいないか）をすばやく判断できます。

6） REPLY


私のように、通常は次のようなコードを書く場合：

$ read input
echo do something with $input

変数の作成についてまったく心配する必要がないのは、驚くかもしれません。

$ read
echo do something with $REPLY

それは同じことです。

7） TMOUT


セキュリティ上の理由で運用サーバーに長時間留まらないようにしたり、誤った端末で誤って危険なものを実行したりしないようにするために、この変数を設定すると保護として機能します。

設定された秒数内に何も入力されない場合、シェルは終了します。

つまり、これは代替手段sleep 1 && exitです。

$ TMOUT=1
