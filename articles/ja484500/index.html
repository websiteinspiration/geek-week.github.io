Google Cloudからconfig sshにサーバーを追加するスクリプト

注釈。サーバーのリストからssh Linuxの構成を作成する非常に単純なスクリプトに関する記事。Ubuntu 18でテストされています。GoodleCloud SDK、Python 2.7、Bashを使用しています。


処理するサーバーの数が急激に増加した後、すべてのIPと詳細を暗記した当時のように、パスワードストレージとCMDBがそのようなオンラインアクセスを提供しないことに気付きました。CMDBがまだ習得していないためかもしれません。それにもかかわらず、SSHを介して多数のサーバーにすばやくアクセスするという問題を解決するには、何らかの方法でそれを行う必要があります。


次に-Linuxターミナルの観点から（Ubuntu 18で実行）。多分それは他のディストリビューションで動作し、おそらくWindowsに類似物さえあるでしょう-私は見ませんでした。


主な要件：


  • 繰り返しやすい。複数の管理者がいて
    、誰もが同じように構成できる必要があります。さらに、リモートでの作業も可能です。少なくともすべてのラップトップに状況がありますが、通常の「長時間調整およびデバッグ」されたコンピューターでは作業できない場合があります。
  • サーバーは追加、削除、変更されたアドレスです。これは考慮されるべきです。

これを行うために、ssh設定でエイリアスホストを使用し、gcloud cli GCPクライアントを介してサーバーのリストを取得し、Python 2.7を使用してこれをすべて自動化することを決定しました（Ubuntuのデフォルトであり、データを操作するために調査することにしました）。カットの下に説明があるスクリプト自体。


問題の定式化

#!/usr/bin/python
import commands
import json

sComputeListOutput = commands.getoutput('gcloud compute instances list --filter="status:running" --format="json(name, status, networkInterfaces[].accessConfigs[])" ')
s = json.loads(sComputeListOutput)
for server in s:
    print 'HOST ',server['name']
    print '   HostName ',server['networkInterfaces'][0]['accessConfigs'][0]['natIP']


このタスクはまだ進行中です。同じスクリプト内でssh / configファイルを読み取り、受け取った値と比較して、結果全体を書き戻す予定です。または、新しいファイルを個別に生成し、なんらかの差分を使用して比較を実行します。これにより、すべての変更を手動で確認できます。
