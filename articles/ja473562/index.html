Androidの暗号化された設定

こんにちは。私はドミトリーと申します。Otusの基本コース「Android Developer」の講師です。今日私は興味深いと思う記事の翻訳を共有することにしました、そしてそれは私たちのブログの多くの読者にとって有用であると思います。






SharedPreferencesへのデータの保存は非常に高速で便利です。また、サイバー犯罪者がSharedPreferencesに保存されているデータを簡単に見られるようになっているので、そこに何を置くか注意してください。データを暗号化された形式で保存する方法を考える必要があるかもしれません。

SqlCipherなどのデータベースエンジンの使用を正当化しない少量のデータの場合、機能が制限されていました。

  • ネイティブの暗号化方法（何をしているのかわかっている場合）
  • Secure-preferences、その他のSecure-preferences、Armadilloなどの既製のソリューション
  • Androidのすべてのバージョンでの奇妙なキーストアAndroidシステムライフサイクルの問題との戦い

これは以前は機能していましたが、今では適切かつ公式な解決策があります。
まだアルファ版ですが、プロジェクトで使用するとしばらくの間はうまく機能しました。EncryptedSharedPreferencesの使用は歓迎されます（またはそれを使用できます）。すべてのユーザーがmin-sdk 23+を使用しています。

それを使用する方法の例を見てみましょう：

EncryptedSharedPreferencesの例


最小SDK


今日23（Android 6.0）

minSdkVersion 23

依存関係を追加


implementation "androidx.security:security-crypto:1.0.0-alpha02"

初期化/開く


Androidキーストアからマスターキーを作成または取得し、それを使用してEncryptedSharedPreferencesインスタンスを初期化/開くだけです。

//  1:    -   / 
val masterKeyAlias = MasterKeys.getOrCreate(MasterKeys.AES256_GCM_SPEC)

//  2.  /   EncryptedSharedPreferences
val sharedPreferences = EncryptedSharedPreferences.create(
    "PreferencesFilename",
    masterKeyAlias,
    applicationContext,
    EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
    EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
)

レコードを保存


SharedPreferencesで常に行ったようにデータを保存します。

//  3.    EncryptedSharedPreferences
sharedPreferences.edit()
    .putString("DATA", saveText.text.toString())
    .apply()


メモを読む


SharedPreferencesでいつも行ったようにデータを読み取ります。
//  3:    EncryptedSharedPreferences
val value = sharedPreferences.getString("DATA", "")

結論


EncryptedSharedPreferencesは、Android 6.0以降向けの非常にシンプルで信頼性の高いソリューションです。

2つの大きな利点があります。

  1. コードに何もエンコードする必要はありません。彼はAndroid Keystoreを使用するだけで、それに対処する必要はありません。
  2. ユーザーはロック画面を設定する必要はありません。EncryptedSharedPreferencesは画面ロックなしでも同様に機能します

これは、SharedPreferencesのほぼ完全な代替品です。EncryptedSharedPreferencesの初期化/オープンがユーザーに悪影響を与えないことを確認してください。

この決定は間違いなく残るでしょう。私はこれを適切なシナリオで使用します。今、私はただAndroidの人たちがそのパフォーマンスを向上させると言いたいので、私たちはもっと心配する必要はありません:)
