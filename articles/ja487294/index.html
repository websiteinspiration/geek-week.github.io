Playwright-Microsoft Playwrightと新しいテストツール



私は10年間コードのテストと作成を行っており、過去4年間はハイゼンバグ会議のプログラム委員会でレポートをテストしています。今日は、エンドツーエンドのテスト、ブラウザーをまたがるアクセシビリティ、およびPlaywrightバージョン0.10.0を使用するスリルについてお話します。

端から端まで


なぜエンドツーエンドのテストが必要なのですか？ブラウザを制御し、ユーザーのアクションをシミュレートします。たとえば、カスタムスクリプトについて説明しましたが、製品の各バージョンでテストする必要があります。すべてのバージョンのすべてのスクリプトを手動でチェックすることは、自動よりもコストと時間がかかります。

さまざまなツールがあります：Selenium、Puppeteer、Protractor、Cypressなど。2週間前に、新しいツールがリリースされました。Playwrightは、Puppeteerの開発者であるAndrey Lushnikovによって開発されました。このライブラリは、クロスブラウザテストを作成する際の問題を完全に解決します。

Playwrightでクロスブラウザーテストを作成するには何が必要ですか？私はnpm、実験的なHabr、ドキュメンテーションを読むための学校の英語、コーヒー1杯と15分半が必要でした。

Playwrightをインストールします。コンソールに次のように記述し
npm i playwright

ます。スクリーンショットHabr。ドキュメントから例をコピーします。

const playwright = require('playwright');

(async () => {
   for (const browserType of ['chromium', 'firefox', 'webkit']) {
       const browser = await playwright[browserType].launch();
       const context = await browser.newContext();
       const page = await context.newPage('http://habr.com/');

       await page.screenshot({ path: `screenshots/example-${browserType}.png` });
       await browser.close();
   }
})();

Puppeteerのコードはほとんど同じです。

const puppeteer = require('puppeteer');

(async () => {
   const browser = await puppeteer.launch();
   const page = await browser.newPage();
   await page.goto('https://habr.com');
   await page.screenshot({path: 'screenshots/example.png'});

   await browser.close();
})();

最初の例のテストは、3つのブラウザーで実行されます。確かに、どの特定のバージョンで、どのバージョンが一般に利用可能かは不明です。ドキュメントには、Playwright がブラウザーのドライバーを自動的にダウンロードすることが記載されています。特定のブラウザーを指定することもできます。

 const browser = await chromium.launch();  // Or 'firefox' or 'webkit'.
 const context = await browser.newContext();

Puppeteerでブラウザーのバージョンを選択するには、ドライバーへのパスを指定する必要があります。 PlayWrightではそのような可能性はありません。私が理解しているように、常に最新バージョンのエンジンが使用されます。これは、GitHubで既に議論されています。

ちなみに、スクリーンショットテストが好きな場合は注意してください。 PlaywrightとPuppeteerは、Chromiumで異なるスクリーンショットを撮ります


。何らかの理由で、フォントのサイズが異なります。GitHubでチケットを作成しました。

Playwrightは、モバイルブラウザーでのテストを意味するWebKitをサポートしています。しかし、すべてのスマートフォンとタブレットは異なる解像度を持っています-テストを実行するときに必要なものをどのように選択するのですか？人気のモデルを含め、deviceDescriptors.tsコードに76台のデバイスが見つかりましたBlackBerryとJioPhone 2.便利なものから：横長モードでiPhone 8とiPhone 8を選択できます。テストコードは、通常のブラウザのテストとほとんど同じです。

const { webkit, devices } = require('playwright');
const deviceType = devices['iPhone 8'];

(async () => {
   const browser = await webkit.launch();
   const context = await browser.newContext({
       viewport: deviceType.viewport,
       userAgent: deviceType.userAgent
   });
   const page = await context.newPage('http://habr.com');
   await page.screenshot({ path: `example-${deviceType.name}.png`});
   await browser.close();
})();

Playwrightの興味深い機能は、地理位置情報を変更できることです。これは、マップをテストするときに役立ちます。ブラウザのコンテキストを初期化するときは、座標を指定する必要があります。

const context = await browser.newContext({
   viewport: iPhone11.viewport,
   userAgent: iPhone11.userAgent,
   geolocation: { longitude: 12.492507, latitude: 41.889938 },
   permissions: { 'https://yandex.ru/maps': ['geolocation'] }
});

これはテストの最初の年ではなく、テストがどのように成功するかを確認したい場合があります。Puppeteerには、headless: falseブラウザウィンドウを起動するオプションがありました。まったく同じオプションがPlaywrightの場合です。

puppeteer.launch({headless: false});
playwright.launch({headless: false});

両方のライブラリのテストコードは非同期であるため、取得することはできません。Puppeteerのドキュメントには段落全体がありますが、Playwrightのドキュメントには何もありません。さらに、Playwrightのドキュメントにはコード例はありませんが、GitHub にはチケットがあります。オープンソースプロジェクトに参加するのは良い理由のようです。そして、PlaywrightとJestに関するこの記事の例をリポジトリに入れました。

移動する時間ですか？


Playwrightの開発者は、機能をすばやく追加してバグを修正する準備ができているようです。たとえば、現在ファイルのダウンロードは機能していませんが、開発者はそれを修正し、Puppeteerの場合よりも悪くなることはないと書いています。テストをプレイライトに移植したい場合は、チケットを開始する準備をしてください。

Playwrightチームは、このライブラリで最高のものをまとめたいと考えています。 GitHubチケットで、APIを改善するためのいくつかのアイデアを見つけました。たとえば、サイプレスのような機能を追加します。

Playwrightは複数のブラウザーをサポートしており、それを使用してクロスブラウザーテストを作成する方が便利です。確かに、現在彼はバージョン0.10.0を持っていますが、ドキュメントは多くなく、SlackOverflowには6つの質問しかありません。Playwrightのリリースを注意深く監視することをお勧めしますが、APIが変更される可能性があるため、今のところ、本番環境での使用を急いではいけません。
