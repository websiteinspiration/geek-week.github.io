FabEx-Hyperledger Fabricのブロックエクスプローラー

こんにちは、FabExプロジェクトについてお話ししたいと思います。これは、Hyperledger Labsによって最近採用された、Hyperledger Fabricのブロックエクスプローラーであり、公式のエクスプローラーに比べていくつかの利点があります。プロジェクトは完全にGolang（公式ではNodejsではなく）で記述され、ブロックチェーンデータをMongoDBまたは選択したCassandra（公式ではPostgreSQLではありません）に格納し、GRPCとREST APIの両方を備えています。拡張は簡単です。たとえば、ストレージの実装を簡単に追加できます。カットの下の詳細。

ブロックエクスプローラーの主なタスクは、ブロックを抽出して解析し、ブロック内の必要な情報を見つけることです。
ブロック構造につ​​いては、Senthil Nathanのブログで詳しく説明されています。 FabExデータベースの個々のレコードには、トランザクションとこのトランザクションが分類されたブロックに関するデータが含まれています。

type Tx struct {

    Hash           string
    PreviousHash   string 
    Blocknum       uint64 
    Payload        string
    ValidationCode int32 
    Time           int64

FabExは、ピアから最後のブロックのハッシュを受け取り、データベースでそのようなブロックを検索します。そうでない場合は、データベースで最後に追加されたブロックを見つけ、欠落している間隔でピアにブロックを要求します。これはリアルタイムで行われ、特別な介入は必要ありません。すべてを保存するベースを選択できます。


ストレージとしてのCassandraまたはMongoDB


FabExはもともとPostgreSQLとMongoDBをサポートしていました。その後、使用可能なメモリよりも多くのブロックが存在する可能性があるため、メモリ内ではなく、迅速な読み取り要求に適したソリューションを探し始めました。その結果、Cassandraがまさに私が必要とするものであると確信している1つの記事：ディスクへのデータの保存、（比較的）高速な読み取り、および便利なクラスタリング。

マイクロサービスとして使用できます


FabExは元々、他のシステムコンポーネントがブロックチェーンに関する必要な情報を取得するために通信できるサービスとして作成されました。彼は自分の財産をCouchDBに入れようとはしませんが、もっと簡単ですが、それを分離して保管します。美しいグラフィックを備えたUIダッシュボードになることはありませんが、明確なGRPC API（およびREST APIとCLI）を提供し、ブロックチェーンからそれを必要とするサービスへのコンパクトで予測可能なデータ転送を可能にします。ただし、最小限のUIも使用できます。

GRPC APIはこれまでのところ（そして将来うまくいけば）シンプルで、fabex.protoに記述されています。

service Fabex {
    rpc Explore(RequestRange) returns (stream Entry);
    rpc Get(Entry) returns (stream Entry);
}


rpc Exploreは、ブロックチェーンからRequestRange 範囲のブロックからトランザクションを取得し、これらのトランザクションでストリームを返します。

rpc Getを使用すると、データベースに対してフィルタリングクエリを実行して、特定の基準（ブロック番号、トランザクションID、トランザクションキー）を満たすすべてのトランザクションを取得できます。

独自のGRPCクライアントを作成するか、github.com / hyperledger - labs / fabex / clientパッケージから既製のGRPCクライアントを使用できます。

たとえば、この方法では、1から15までの範囲のブロックからトランザクションを取得できます。
txs, err := client.Explore(1, 15)


そして、あなたはフィルタリングリクエストを実行することができます：

-トランザクションIDでフィルタリングします：
txs, err := client.Get(&pb.Entry{Txid:"x"})


-ブロック番号でフィルター：
txs, err := client.Get(&pb.Entry{Blocknum:42})


-トランザクションキー：
txs, err := client.Get(&pb.Entry{Payload:"mykey"})


トランザクションをブロックごとにグループ化するには、PackTxsToBlocks ヘルパーを使用できます。
blocks, err := helpers.PackTxsToBlocks(txs)

CLIモード


上記のすべては、CLIを使用してAPIと対話することなく繰り返すことができます。クリーンなデータベースを起動したばかりの場合は、トランザクションでそれを埋める必要があります。
./fabex -task=explore -configpath=tests -configname=config -enrolluser=true -db=mongo



これで、5番目のブロックをリクエストできます。
./fabex -task=getblock -blocknum=5 -configpath=tests -configname=config -enrolluser=true -db=mongo
