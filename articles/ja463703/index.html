攻撃者はドメイン内の特定の構成を使用します


攻撃者は多くのトリックや脆弱性を利用して企業ネットワークに侵入し、特権を増やします。これらの脆弱性の一部は、特定された後に簡単に変更できるドメイン構成パラメーターです。

ADダッシュボードは、あなた（またはシステム管理者）が先月KRBTGTパスワードを変更しなかった場合、または組み込みのデフォルト管理者アカウント（管理者）で誰かが認証された場合、すぐに通知します。これらの2つのアカウントは、ネットワークへの無制限のアクセスを提供します。攻撃者はそれらにアクセスして、特権とアクセス許可の違いを自由に迂回しようとします。その結果、興味のあるデータにアクセスできるようになります。

もちろん、これらの脆弱性を自分で検出できます。たとえば、確認のためにカレンダーにリマインダーを設定したり、PowerShellスクリプトを実行してこの情報を収集したりできます。

Varonisダッシュボードは自動的に更新され、潜在的な脆弱性を強調する主要な指標をすばやく表示して分析できるため、すぐに対応できます。

ドメインレベルの3つの主要なリスク指標


以下は、Varonisダッシュボードで使用可能ないくつかのウィジェットです。これらを使用すると、企業ネットワークとITインフラストラクチャ全体の保護が大幅に強化されます。

1. Kerberosアカウントのパスワードが大幅に変更されていないドメインの数

KRBTGTアカウントは、すべてのKerberosチケットに署名するADの特別なアカウントです。ドメインコントローラー（DC）にアクセスする攻撃者は、このアカウントを使用してゴールデンチケットを作成できます。これにより、企業ネットワーク内のほとんどすべてのシステムに無制限にアクセスできます。ゴールデンチケットを正常に受信した後、攻撃者が2年間組織のネットワークにアクセスできる状況に直面しました。会社のKRBTGTアカウントのパスワードが過去40日間変更されていない場合、ウィジェットはそのことを通知します。

攻撃者がネットワークにアクセスするには、40日は十分な時間です。ただし、このパスワードを定期的に変更するプロセスを提供して標準化すると、攻撃者が企業ネットワークに侵入するタスクが大幅に複雑になります。


MicrosoftによるKerberosプロトコルの実装に従って、 KRBTGT パスワードを 2回変更する必要があることに注意してください。

将来、このADウィジェットは、ネットワーク上のすべてのドメインのKRBTGTパスワードを再度変更する時期を通知します。

2.ビルトインAdministratorアカウント（Administrator）を最近使用したドメインの数最小限の特権

の原則によると、システム管理者には2つのアカウントが提供されます。これは、誰もデフォルトの管理者アカウントを使用してはならないことを意味します。

組み込みの管理者アカウントは、システム管理プロセスを簡略化するためによく使用されます。これは悪い習慣になり、ハッキングにつながる可能性があります。これが組織で発生した場合、このアカウントの適切な使用と潜在的に悪意のあるアクセスを区別することは困難です。


ウィジェットがゼロ以外を表示する場合は、誰かが管理者アカウントで正しく機能していないことを意味します。この場合、ビルトインAdministratorアカウントへのアクセスを修正および制限するための対策を講じる必要があります。
ウィジェットの値がゼロに達した後、システム管理者はこのアカウントを自分の仕事に使用しなくなります。今後、このアカウントの変更は潜在的なサイバー攻撃を示します。

3.保護されたユーザー（保護されたユーザー）のグループがないドメインの数

古いバージョンのADでは、弱いタイプの暗号化-RC4がサポートされていました。ハッカーは何年も前にRC4をクラックしましたが、今ではまだRC4を使用しているアカウントを攻撃者がクラックすることは簡単な作業です。Windows Server 2012で導入されたActive Directoryのバージョンでは、保護されたユーザーグループと呼ばれる新しいタイプのユーザーグループが導入されました。追加のセキュリティツールを提供し、RC4暗号化を使用したユーザー認証を防止します。

このウィジェットは、組織のどのドメインにもそのようなグループがないかどうかを示し、修正できるようにします。保護されたユーザーのグループを有効にし、それを使用してインフラストラクチャを保護します。


攻撃者のための軽い標的


ユーザーアカウントは、攻撃者にとって最大の標的です。最初に侵入しようとする試みから、進行中の特権の昇格とそのアクションの隠蔽までです。攻撃者は、基本的なPowerShellコマンドを使用してネットワーク上の単純なターゲットを探します。これは、検出が難しいことがよくあります。これらのライトターゲットのできるだけ多くをADから削除します。

攻撃者は、無制限のパスワードを持つ（またはパスワードを必要としない）ユーザー、管理者であるテクノロジアカウント、および古いRC4暗号化を使用するアカウントを探します。

これらのアカウントはいずれも、簡単にアクセスできるか、原則として監視されません。攻撃者はこれらのアカウントを乗っ取り、インフラストラクチャ内をシームレスに移動できます。

攻撃者がセキュリティ境界に侵入すると、少なくとも1つのアカウントにアクセスできるようになります。攻撃を検出して排除する前に、機密データへのアクセスを阻止できますか？

Varonis ADダッシュボードは脆弱なユーザーアカウントをポイントするため、事前に問題を修正できます。ネットワーク内に侵入することが困難であるほど、攻撃者が深刻なダメージを与える前に攻撃者を無力化する可能性が高くなります。

ユーザーアカウントの4つの主要なリスク指標

1.有効期限のないパスワードを持つアクティブユーザーの数

攻撃者がこのようなアカウントにアクセスすることは、常に大成功です。パスワードが期限切れになることはないため、攻撃者はネットワーク内で永続的な足場を取得し、これを使用して特権を昇格したり、インフラストラクチャ内を移動したりできます。
攻撃者は、クレデンシャル置換攻撃で使用する数百万のユーザーとパスワードの組み合わせのリストと、その可能性を

有効期限のないパスワードを持つアカウントは管理が簡単ですが、安全ではありません。このウィジェットを使用して、これらのパスワードを持つすべてのアカウントを検索します。この設定を変更して、パスワードを更新します。


このウィジェットの値がゼロになるとすぐに、このパスワードで作成された新しいアカウントがダッシュボードに表示されます。

2. SPN

SPN（サービスプリンシパル名）を持つ管理アカウントの数は、サービス（サービス）のインスタンスの一意の識別子です。このウィジェットは、完全な管理者権限を持つサービスアカウントの数を示します。ウィジェットの値はゼロでなければなりません。管理者権限を持つSPNが発生するのは、そのような権限の付与がソフトウェアプロバイダーやアプリケーション管理者にとって便利であるためですが、これにはセキュリティ上のリスクがあります。

サービスアカウントに管理者権限を付与すると、攻撃者は使用されていないアカウントへのフルアクセスを取得できます。つまり、SPNアカウントにアクセスできる攻撃者は、インフラストラクチャ内で自由に行動できると同時に、自分の行動の監視を回避できます。

この問題は、サービスアカウントの権限を変更することで解決できます。このようなアカウントは、最小限の特権の原則に従い、自分の作業に本当に必要なアクセス権のみを持っている必要があります。


このウィジェットを使用すると、管理者権限を持つすべてのSPNを検索し、そのような特権を排除し、最小限の特権での同じアクセス原理に基づいてSPNをさらに制御できます。

新しく表示されたSPNがダッシュボードに表示され、このプロセスを制御できます。

3. Kerberos事前認証を必要としないユーザーの数

理想的には、KerberosはAES-256暗号化を使用してチケット（チケット）認証を暗号化します。

ただし、古いバージョンのKerberosはRC4暗号化を使用していたため、数分で解読できるようになりました。このウィジェットは、RC4をまだ使用しているユーザーアカウントを示します。 Microsoftは下位互換性のためにRC4をサポートしていますが、ADで使用する必要があるという意味ではありません。


そのようなアカウントを見つけたら、ADの「Kerberos事前承認を要求しない」チェックボックスをオフにして、アカウントがより複雑な暗号化を使用するようにする必要があります。

Varonis ADダッシュボードがない場合、これらのアカウントの自己検出には時間がかかります。実際には、RC4暗号化を使用するように編集されたすべてのアカウントについてタイムリーに通知を受けることは、さらに困難な作業です。

ウィジェットの値が変更された場合、これは不正なアクションを示している可能性があります。

攻撃者は基本的なPowerShellコマンドを使用して、アカウントプロパティのADプロパティから「PASSWD_NOTREQD」フラグを読み取ります。このフラグを使用することは、パスワードの存在または複雑さの要件がないことを意味します。

誰にでも公開されている何千もの機密ファイルの1つが次の財務報告である場合はどうなりますか？

これらのアカウントのパスワードを更新して、この問題を修正してください。
