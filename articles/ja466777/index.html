アプリケーションのフォルダーとファイルを構造化する方法


React Nativeでのアプリケーションの編成は、Reactでのアプリケーションの編成と同じです。したがって、これに精通している場合は、同じロジックに従ってください。ただし、これが当てはまらない場合は、次に概説するロジックが役立ちます。



  • フォルダーをルートに追加し、「app」という名前を付けます。
  • 「アプリ」内にフォルダーを作成します。

assets-その中に、フォント、アイコン、画像を含む最大3つのフォルダがあります。

components-ここに、共有アクセスですべてのReactコンポーネントを配置します。通常、これらは状態ロジックがなく、アプリケーション内で簡単に再利用できるため、「ダミー」と呼ばれるコンポーネントです。

views-これらは私たちのアプリケーション画面です。 Reactコンポーネントもここに格納されます。これは、コンテナーと呼ばれ、独自の状態を格納するためです。

modules-対応するビュー（JCX）を持たないパーツがそこに格納されます。典型的な例は、colorsモジュール（すべてのアプリケーションの色が含まれています）とutilsモジュール（再利用可能なユーティリティ関数が含まれています）です。

services-API呼び出しをラップする関数は次のとおりです。

i18n-さまざまな言語と言語標準を持つユーザー向けに翻訳がここに保存されます。

状態管理ライブラリがある場合は、そのコンポーネント用のフォルダーも必要です。Reduxのの場合には、2つの以上のフォルダが使用されている、のための1つactionsとするための1つreducers。外部パッケージを使用せず、React Context APIを使用する場合は、独自のプロバイダーを作成できます。プロバイダーは、モジュールフォルダーまたは新しいフォルダーに配置できますproviders。

私たちはあなたのニーズに応じてナビゲーションライブラリを選択します


残念ながら、RNにはまだ完全なソリューションがなく、少なくとも古いナビゲーターコンポーネントに代わる価値のあるソリューションがないため、コミュニティがこの問題を解決する方法に焦点を当てます。したがって、プロジェクトを作成して、使用するナビゲーションライブラリを知りたい場合は、このセクションが適しています。

一般に、ナビゲーションライブラリには、JavaScriptナビゲーターとネイティブナビゲーターの2種類があります。最初のものはJavaScriptで記述され、ネイティブライブラリはプラットフォーム（Android、iOS）に応じてネイティブモジュールで記述され、JavaScriptモジュールに接続されてコードにスローされます。前者はセットアップがはるかに簡単ですが、後者ははるかに効率的です。時間をかけてどのタイプがあなたに最も適しているかを見つけ出し、そのタイプの多くのライブラリから選択します。

Spencer Carlyは素晴らしい仕事をして、React Nativeの世界で現在のナビゲーションオプションを開発しました。このトピックに関する彼の記事を読むことができます。現在最も人気のあるソリューションは、JavaScriptソリューションとしてのReact NavigationとネイティブソリューションとしてのReact Native Navigationです。

さまざまな画面サイズのさまざまなデバイスでアプリケーションをどのようにスケーリングするかを事前に決定します。


ほとんどの場合、複数のデバイスと画面サイズのアプリケーションを開発しています。現在、設計とアプリケーション開発には2つのアプローチがあります。

画面サイズに応じて、異なるUX / UIを選択できます。ほとんどのタイプのアプリケーションでは、これがおそらく最良のオプションですが、いくつかの画面オプションを考え出して実装する必要があるため、多くの労力が必要です。画面サイズは、Dimensions APIを使用して決定できます。一方、React Native Responsive UIのようないくつかのツールをそのまま提供するサードパーティのパッケージを使用できます。

または、同じUI / UXを使用して、すべての画面サイズに比例してスケーリングすることもできます。これは、たとえばゲームを開発している場合に最適なオプションです。この場合も、React Native Responsive Screenなどのサードパーティパッケージを使用できます。

アニメーションに注意してください


アニメーションはモバイルアプリケーションに不可欠ですが、React Nativeのパフォーマンスは不十分です。

エラーから保護するには、エミュレータが正しいフィードバックを提供しないため、デバイスでアニメーションをテストする必要があります。またuseNativeDriver、パフォーマンスを向上させるために、可能な限り（trueで）使用する必要があります。
