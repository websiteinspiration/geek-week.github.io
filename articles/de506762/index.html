Sicherheitswoche 25: Das rekordverdächtige Patch-Set von Microsoft

Am vergangenen Dienstag, 9. Juni veröffentlichte Microsoft die nächste Reihe von Patches für seine Lösungen (Review Artikel , Release Notes , Nachrichten auf Habré). Das Update hat den Rekord für die Anzahl der geschlossenen Sicherheitslücken gebrochen: 129 Teile, davon 11 kritische. Nach den Daten aus dieser Überprüfung hat der „Dienstag der Patches“ im Juni die Anzahl der in diesem Jahr geschlossenen Probleme auf 616 erhöht, was etwas weniger als das Ergebnis für das gesamte Jahr 2017 ist.

Das Update behebt sofort drei Schwachstellen im SMB-Protokoll: Alle sind nicht so gefährlich wie zuvor entdeckte Probleme (z. B. das berüchtigte EternalBlue- Loch ). In zwei von drei Fällen sind jedoch Denial-of-Service und Datenlecks möglich. Ein weiteres "Deja Vu" ist ein ProblemDie im .LNK-Dateihandler erkannte Funktion kann zur Ausführung von beliebigem Code mit Benutzerrechten führen. Eine kritische Sicherheitslücke wurde auch in geschlossen den CAB - Datei - Handler . In diesem Szenario könnten Angreifer den Benutzer davon überzeugen, „vorbereitete“ Druckertreiber zu installieren und dann die Kontrolle über das System zu übernehmen.

Das Microsoft-Update löst auch das Problem mit Adobe Flash Player. Dieses Produkt hat einmal regelmäßig zu Sicherheitsnachrichten beigetragen. Da Benutzer und Hersteller diese Technologie ablehnen (ab 2017), nimmt die Anzahl der erkannten Probleme zusammen mit dem Interesse der Angreifer ab. Die Art der kritischen Sicherheitsanfälligkeit, die nach der Verwendung verwendet werden kann, musste jedoch sowohl auf der Adobe-Seite als auch auf der Microsoft-Seite geschlossen werden, da sie sich auf die Implementierung auswirkt, die in den Edge- und Internet Explorer-Browsern enthalten ist. Schließlich schloss Microsoft eine ernsthafte Lücke in SharePoint Server , durch die nicht autorisierte Benutzer die Kontrolle übernehmen konnten.

Die rechtzeitige Erkennung von Schwachstellen ist eher eine positive Entwicklung. Es lohnt sich nicht, die Qualität der Software anhand der Anzahl der Fehler zu beurteilen, insbesondere wenn sie jahrelang nicht dort sitzen, sondern nahe beieinander liegen. Microsoft hat den zweiten Monat in Folge keine Zero-Day-Schwachstellen behoben: Keines der Probleme wurde bei echten Angriffen verwendet. Leider bringen Patches manchmal neue Probleme mit sich. Im Falle des Juni-Patch-Sets melden Benutzer fehlende Anschlüsse für USB-Drucker.

Eine sehr gemischte Geschichte, in der Facebook mit einem Einwohner Kaliforniens zu kämpfen hat, der Teenager im Internet systematisch belästigt hat. Der Täter verwendete die Tails-geschützte Linux-Distribution und versteckte seine echte IP-Adresse über das Tor-Netzwerk. Um dem entgegenzuwirken, baute das soziale Netzwerk ein System zur Verhaltenserkennung neuer Konten auf, das er ständig erstellte. Der Angreifer konnte jedoch erst identifiziert und festgenommen werden, nachdem ein Exploit für den im Distributionskit integrierten Media Player entwickelt worden war. Der Exploit kostete Facebook einen "sechsstelligen Wert" und half bei der Berechnung der tatsächlichen IP. In einem offiziellen Kommentar weisen Vertreter von Facebook auf die Ausnahmesituation hin und bestehen darauf, dass sie solche Methoden in anderen Fällen nicht anwenden werden.
