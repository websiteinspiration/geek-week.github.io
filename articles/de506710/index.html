Verwalten Sie mehrere Adressbücher in der Open-Source-Edition der Zimbra Collaboration Suite

Das globale Adressbuch in Zimbra OSE wird mithilfe eines separaten Galsync-Systemkontos implementiert, das in jeder Zimbra OSE-Maildomäne verfügbar ist. Informationen zu jedem Benutzer in der Domäne wurden zur Kontaktliste dieses Kontos hinzugefügt, während Domänenbenutzer Zugriff auf diese Adressliste haben und damit nach Benutzern suchen und beim Eingeben einer E-Mail-Adresse automatisch vervollständigen können. 

Diese Implementierung des globalen Adressbuchs ermöglicht es, den Import neuer Daten zu vereinfachen, funktioniert jedoch standardmäßig nur in einer Domäne. Große Unternehmen verwenden häufig separate E-Mail-Domänen für verschiedene Abteilungen oder Tochterunternehmen, und häufig ist ein gemeinsames globales Adressbuch erforderlich, das die E-Mail-Adressen und Benutzerdaten mehrerer oder sogar aller Unternehmensdomänen enthält.


Um die Kontaktlisten von Galsync-Konten aus verschiedenen Mail-Domänen zu kombinieren, müssen Sie deren Synchronisierung konfigurieren. Am Beispiel der Domänen msk.company.ru und spb.company.ru konfigurieren wir ein gemeinsames Adressbuch für die Niederlassungen des Unternehmens in Moskau und St. Petersburg.

Standardmäßig hat das Galsync-Konto in Zimbra OSE-Domänen ein Suffix, das aus zufälligen Zahlen und Buchstaben besteht. Dies kann es für uns in Zukunft schwierig machen, damit zu arbeiten. Löschen Sie daher zuerst das Konto galsync.kf9song4@msk.company.ru und erstellen Sie dann mit den folgenden Befehlen ein neues galsync @ msk company.ru :

zmgsautil deleteAccount -a galsync.kf9song4@msk.company.ru
zmgsautil createAccount -a galsync@msk.company.ru -n GAL --domain msk.company.ru -t zimbra -s mail.company.ru -f GAL

Wir machen Sie darauf aufmerksam, dass das Galsync-Konto niemals wie ein reguläres Konto gelöscht werden sollte. Dies führt dazu, dass Zimbra OSE ständig einen Fehler meldet, wenn versucht wird, auf eine Domain zuzugreifen, die kein Galsync-Konto mehr hat.

Fügen Sie anschließend dem Galsync-Konto der Moskauer Domäne eine neue SPB-Datenquelle hinzu, deren Kontakte automatisch zum Ordner _SPB-Kontakte hinzugefügt werden. Dies erfolgt mit dem Befehl zmgsautil addDataSource -a galsync@msk.company.ru -n SPB --domain msk.company.ru -t ldap -f _SPB -p 1d . 

Als Informationsquelle geben wir die Domain spb.company.ru an und legen einen Filter für die Suche nach Kontakten fest:zmprov mds galsync @ msk. = zimbraDistributionList) (! (zimbraHideInGAL = TRUE))) 'zimbraGalSyncLdapSearchBase dc = spb, dc = company, dc = ru zimbraGalSyncLdapURL ldap: //192.168.0.15: 389 .

Jetzt müssen Sie nur noch die von uns hinzugefügte Kontaktquelle aktivieren. Zmprov mds galsync@msk.company.ru SPB zimbraDataSourceEnabled TRUE und die erste Synchronisierung mit dem Befehl zmgsautil forceSync -a galsync@msk.company.ru -n SPB erzwingen .

Um sicherzustellen, dass die Kontaktsynchronisierung zwischen globalen Adressbüchern funktioniert, können Sie den Inhalt des Ordners _SPB überprüfen, den wir gerade erstellt haben.  Zmmailbox -z -m galsync@msk.company.ru gaf . Wenn alles in Ordnung ist, müssen Sie nur den Befehl  zmprov md msk.company.ru zimbraGALMode eingeben  und überprüfen, wie die automatische Vervollständigung funktioniert, wenn Sie Kontakte aus der Domäne St. Petersburg wählen.

Für den Fall, dass Sie Benutzer der Domäne spb.company.ru benötigen, um auch Benutzer der Domäne msk.company.ru sehen zu können, müssen Sie dasselbe tun, aber die Domänen austauschen:

  1. zmgsautil addDataSource -a galsync@spb.company.ru -n MSK --domain spb.company.ru -t ldap -f _MSK -p1d
  2. zmprov mds galsync@spb.company.ru SPB zimbraGalSyncLdapBindDn $zimbra_ldap_userdn zimbraGalSyncLdapBindPassword $zimbra_ldap_password zimbraGalSyncLdapFilter '(|(&(mail=*)(zimbraAccountStatus=active)(!(zimbraHideInGAL=TRUE)))(&(mail=*)(objectClass=zimbraDistributionList)(!(zimbraHideInGAL=TRUE))))' zimbraGalSyncLdapSearchBase dc=msk,dc=company,dc=ru zimbraGalSyncLdapURL ldap://192.168.0.15:389
  3. zmprov mds galsync@spb.company.ru MSK zimbraDataSourceEnabled TRUE
  4. zmgsautil forceSync -a galsync@spb.company.ru -n MSKzmmailbox -z -m galsync@spb.company.ru gaf
  5. zmprov md spb.company.ru zimbraGALMode both

Die Anzahl der Datenquellen in Ihrem Galsync-Konto ist unbegrenzt.

Sie können in der Administrationskonsole neue Kontaktquellen für Ihr Galsync-Konto hinzufügen. Dazu müssen Sie jedoch trotzdem an der Kommandozeile arbeiten. Verwenden Sie den Befehl zmgsautil addDataSource -a galsync@msk.company.ru -n KLG --domain msk.company.ru -t ldap -f _KLG , um beispielsweise Kontakte aus einer Zweigstelle in Kaluga zu galsync einer Moskauer Domäne  hinzuzufügen -p 1d Fügen Sie eine KLG-Datenquelle und einen Ordner mit dem Namen _KLG hinzu.



Gehen Sie danach in der Administratorkonsole zu Einstellungen - Domänen - GAL konfigurieren, wählen Sie im geöffneten globalen Adressbuch-Konfigurationsassistenten die KLG-Quelle aus und geben Sie alle erforderlichen Parameter ein, ähnlich wie in den vorherigen Beispielen.



Sie können Active Directory auch als externe Kontaktquelle verbinden. Dies erfolgt auf die gleiche Weise wie im Fall von normalem LDAP: Es wird eine AD-Datenquelle erstellt - zmgsautil addDataSource -a galsync@msk.company.ru -n AD --domain msk.company.ru -t ldap -f _AD -p 1d Anschließend werden der Suchfilter und die Daten für die automatische Vervollständigung in der Administrationskonsole angezeigt.
