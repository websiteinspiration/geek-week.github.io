Reactアプリケーションの速度が低下する主な理由の1つは、そのグローバルな状態です。これを非常に単純なアプリケーションの例で示します。その後、実際の例に近い例を示します。


犬の名前を入力できる簡単なアプリケーションを次に示します（ウィンドウが機能しない場合は、ここにリンクがあります）。



このアプリケーションで遊ぶと、すぐに動作が遅くなることがわかります。入力フィールドを操作する場合、顕著なパフォーマンスの問題があります。そのような状況では、フォームで救命浮輪を使用React.memoし、遅いレンダリングですべてのコンポーネントでラップすることができます。しかし、この問題を別の方法で解決してみましょう。

function sleep(time) {
  const done = Date.now() + time
  while (done > Date.now()) {
    // ...
  }

function SlowComponent({time, onChange}) {
  sleep(time)
  return (
    <div>
      Wow, that was{' '}
      <input
        value={time}
        type="number"
        onChange={e => onChange(Number(e.target.value))}
      />
      ms slow
    </div>
  )
}

function DogName({time, dog, onChange}) {
  return (
    <div>
      <label htmlFor="dog">Dog Name</label>
      <br />
      <input id="dog" value={dog} onChange={e => onChange(e.target.value)} />
      <p>
        {dog ? `${dog}'s favorite number is ${time}.` : 'enter a dog name'}
      </p>
    </div>
  )
}

function App() {
  //   " " (global state)
  const [dog, setDog] = React.useState('')
  const [time, setTime] = React.useState(200)
  return (
    <div>
      <DogName time={time} dog={dog} onChange={setDog} />
      <SlowComponent time={time} onChange={setTime} />
    </div>
  )
}
