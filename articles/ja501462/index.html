ロボ相撲：知能ロボットの戦い


（c）

毎年、全国から約1000チームがロボットトーナメントで激しい戦いを繰り広げています。このコンテストでは、チェーンソー、トーチ、放電は見られません。つまり、「ロボットの戦い」という言葉を聞いても、何も思い浮かびません。

これは戦いだけではありません-エレガントでシンプル、そして戦略にインテリジェンスを示す能力という特別な資質が評価されるロボット相撲です。競技は直径1.5 mの円形の金属リングで行われ、その周囲に幅5 cmの白い線が描かれます。この境界を横切るロボットは敗者と見なされます。そして、この明らかな単純さには、機会の深淵があります。

「ロボットの戦い」がプログラマーの知的競争になった経緯と、ロボスモイストの組み立て方について詳しく説明します。

国境と敵の検索センサー



（c）

ロボットの目は赤外線センサーです。IR反射の程度は、感光性エレメントによって測定されます。境界線の白い色は偶然に選択されたのではなく、リングの黒い表面よりも多くの光を反射します。

ロボットには、国境の「探索」を担当するいくつかの赤外線センサーが装備されています。概して、フロントセンサーは動きの速度を制御するのに役立ちます-戦闘機は過度の熱意のためにフィールドを離れるべきではありません。リアセンサーはリングの境界への接近を「監視」します。それらからの応答を受信すると、ロボットは方向を変えます。

そして、光線を反射するのに必要な時間を測定することにより、敵の位置を理解することができます。このパラメーターを指定すると、多くのロボット（すべてではない）が方向を変更します。

戦闘ルール



上記の利点に加えて、IRセンサーには欠点もあります。視野が狭いままであり、そのため多くの死角が現れ、攻撃でのその使用は高度に操縦可能なロボットの主な戦略です。

ただし、これは自律型ロボットでのみ機能します。独立したロボット相撲士がラジコンの対戦相手と衝突するとき、オペレーターはロボットよりも多くを見るので、彼はブラインドゾーン攻撃戦略を使用できません。

オートボットは死角を攻撃する代わりに、隅から攻撃戦略を使用します。一般に、レスラーには、回転、押す、ブレーキをかける、停止するなどの多くの操作はありません。

ただし、敵を混乱させるために、一部の戦闘機は船体の片側または両側に白い旗を投げます。敵を旗に誘い込み、彼らはその脆弱な側を攻撃してリングから押し出します。

もちろん、そのような攻撃を防ぐ方法が登場しました。左右に旗が出ているのは、ロボット本体が中央にあることを意味します。ロボットが片側に旗を投げた場合、その体は反対の場所にあります。

一部のボットは敵を誘惑するだけでなく、彼のセンサーから隠れることもできます。これを行うには、それらを黒く塗るか、反射コーティングを追加します。

武装



相撲ロボットの戦いは単なるお尻ではありません。それらの多くは鋭利な刃のブルドーザーブレードを備えています。それはあなたがあなたの対戦相手を上げて、彼をコートから外に動かすか、あるいは深刻なダメージを引き起こすことさえ可能にします。確かに、この武器は戦闘機の機動性を低下させます。

人間の相撲では、戦闘機の主な武器は彼の印象的な質量です。奇妙なことに、ロボットでは、質量も「重要な役割」を果たします。

ルールによれば、ロボットの長さ（どちらの側でも）は20 cmを超えず、重量は3 kg以下でなければならないという事実にもかかわらず、robo-sumoistは数十キロの「重さ」を持っています。この「重さ」は、磁石によって達成されます。はい、リングは金属表面に基づいているため、テクニックが床に実際に「くっつく」ことがあります。

マシンマインド


主な戦術が高い操作性であるロボットを作成するとき、使用される磁石の数は少なくなります。この場合の主な重点は、強力なモーターに置かれています。ただし、戦闘での勝利が鉄のみに依存することはほとんどありません。

相撲ロボットの主なものは、コードの形で書かれた戦略です。鉄の選択もそれに依存します。多くの場合、ロボットは非常に速く移動するため、その動きを追跡することは困難です。ただし、超操作性の動作は厳密なプログラムロジックに基づいています。それが作成された原理を検討してください。

相撲ロボット作成



典型的なロボット相撲作者の作成について徹底的なガイドを作成したhackster.ioサイトの専門家 の経験に目を向けましょう。

プロジェクトでは次のコンポーネントが使用されます。

  • CytronTechnologies ×1 コントローラー ;
  • 赤外線センサー（3-80 cm）×5;
  • シリコーンホイールのセットJSumo（52 x 30 mm）×2;
  • 赤外線追跡モジュール×2;
  • DCギアモーター（12 V、380 rpm）×4;
  • LiPoバッテリー（11.1 V、1300 mAh）×1。


1.コントローラ


Arduino Uno、Mega、Nanoを使用できます。すべてのセンサーを接続するためのボードも必要です。もちろん、独自の拡張カードを作成してすべての接点をArduinoに接続することも、既成のカードを購入してすべての電子部品を接続することもできます。


2.エンジン


これは、ロボットのパフォーマンスが依存する最も重要な部分の1つです。理想的には、高速かつ高トルクのエンジンが必要ですが、この場合、そのサイズは大きくなります。すべてのエンジンとホイールが20 x 20 cmのボックスに収まることを確認する必要

があります。プロジェクトで使用できるエンジンにはいくつかのタイプがあります。

円筒歯車と遊星歯車を備えたエンジンで、出力軸はエンジンと平行です。


ウォームギア モーター。ここの出力シャフトは垂直です。ロボット内には他のコンポーネント用のスペースが増えるため、このオプションの方が適している場合があります。


二輪vs四輪


四輪駆動は間違いなく敵のロボットを押す機会を増やします。ただし、その可動性は二輪車よりも低くなります。それでも、このデモプロジェクトでは四輪駆動が使用されました。

3.ホイール



相撲ロボットに適したゴムの選択肢は、市場にはあまりありません。エンジニアは、Jsumo製のシリコンホイールを選びました。ただし、おもちゃの車のホイールは使用できます。

取り付けに注意してください-選択したホイールがエンジンの出力シャフトに対応する穴のサイズと一致していることを確認してください。このプロジェクトでは、それぞれ6 mmのシャフトを使用し、ホイールには6 mmの穴があります。

4.センサー



物体を検出するには、赤外線センサーと超音波などの両方を使用できます。

専門家は敵との距離が60 cm以下のときに攻撃を開始することを推奨しているため、最大1 mの感度範囲でセンサーを配置することが可能です。


このロボットには、3つの赤外線センサーが取り付けられています（片側に1つ）。


赤外線センサーもロボットの下部に設置する必要があります。これは、リングの端にある白い線を検出するのに役立ちます。運転戦略に応じて、前後に数組のエッジセンサーを設置できます。

5.バッテリー



リチウムポリマーバッテリーは、同じサイズの他のタイプのバッテリーよりも多くの電力を供給するため、通常使用されます。

6.ロボットのシャーシ


ロボットの許容サイズ（20 x 20 cm）に合うCytronの既製のベースを 使用するか、自分でゼロから作成できます。


3D印刷の方が経済的であるため、ダイバーは家庭用印刷用のファイルを共有しています。


7.組み立て


以下の図と表は、相撲ロボットの有線接続の概要を示しています。



最初に2本のワイヤー（推奨サイズは16-18 AWG）をモーター端子にはんだ付けし、それをコントローラーに接続する必要があります。


さらに、すべてのモーターとセンサーはワイヤーではんだ付けする必要があります。


次に、ケース内のアセンブリに目を向けます。M3x10ボルトを使用して、モーターを下部ハウジングに取り付けます。


右側面と左側面に赤外線センサーを取り付けます。


ホイールリムをM4ネジで固定し、シリコンテープを貼ります。

止めネジは、丸いモーターシャフトの平らな部分に固定する必要があります。


次に、セルフタッピングネジを使用して、ブレードの下部にエッジセンサーを取り付けます。


下部ケーシングの前部には、エッジセンサーのワイヤーを敷設するための2つの穴があります。


M6x20のボルトとナットを使用して、ブレードを下部ケーシングに組み立てます。


フロントホルダーに赤外線センサーを取り付けてから、ロワーケースに取り付けます。


すべてのセンサーとモーターを取り付けた後、トラブルシューティングが簡単になるように、各ワイヤーにマークを付けることをお勧めします。


回路基板のサポートサポートを修正します。


ボードに接続する前にバッテリーとセンサーの極性を確認し、ボードに他のコンポーネントを接続する前に必ずバッテリーを取り外してください。


これは全輪駆動のロボットなので、片側に2つのエンジンを接続して同時に制御する必要があります。


既製のコード例を使用する場合、すべてのエンジンとセンサーを図に示す同じポートに接続することをお勧めします。


最後に、バッテリーを接続します。


相撲ロボット完成！

8.センサーのキャリブレーション


重要なステップは、プログラミング前のキャリブレーションです。残念ながら、ほとんどの初心者はロボットを設定するときにこれに十分な注意を払っていません。

センサーが必要な感度範囲内で動作していることを確認してください。

敵の検索センサーの最大感度範囲は80 cmですが、このプロジェクトでは、ロボットは60〜80 cmの距離にある物体に反応しないはずなので、感度範囲は20 cm減少します。


これを行うには、ロボットの「顔」を60 cmの距離にある平らな白い面に置き、LEDインジケータが消えるまでセンサーを調整します。手順はビデオで詳細に示されています。


エッジセンサーは、ブレードの穴を通してドライバーを使用して調整されます。

ソフトウェアを使用して、2つのエッジセンサーの信号をLED D0およびD1に表示できます。


これは、センサーが白い表面を感知したときに高い出力信号を出力し、黒い表面で低い出力信号を出力するようにするために必要です（上のビデオのD0とD1のLEDに注意してください）。

9.プログラミング


URC10ロボットコントローラはArduino UNOと互換性があるため、プログラミングにはArduino IDEが使用されます。既製のコード例を使用できます。

プロジェクトで使用される唯一のライブラリはCytronMotorDriversです。

ライブラリを有効にする方法やサンプルコードをダウンロードする方法がわからない場合は、URC10のマニュアルを参照してください。

相撲ロボットプログラムは、基本的に4つの部分に分けることができます。

  • パフォーマンスの開始。
  • 探す;
  • 攻撃;
  • 退却。

void loop() {
  //   .
  if (!digitalRead(EDGE_L)) {
    //     .
    backoff(RIGHT);
 
    //   .
    searchDir ^= 1;
  }
 
  //   .
  else if (!digitalRead(EDGE_R)) {
    //     .
    backoff(LEFT);
 
    //   .
    searchDir ^= 1;
  }
 
  //   .

    //  ,    .
    if ( digitalRead(OPPONENT_FC) &&
         digitalRead(OPPONENT_FL) &&
         digitalRead(OPPONENT_FR) &&
         digitalRead(OPPONENT_L) &&
         digitalRead(OPPONENT_R) ) {
      search();
    }
 
    // ,      .
    else {
      attack();
    }
  }

  if (!digitalRead(BUTTON)) {
    //  .
    motorL.setSpeed(0);
    motorR.setSpeed(0);
 
    // .
    while (1);
  }
}

演奏開始


スタート戦略は競技のルールに依存します。ロボットは、5秒後に、場合によっては1秒後に動き始めることができます。

ルールにより、リングのどこにでもロボットを配置できるようになります。他の場合では、ロボットは特定の「開始ゾーン」になければなりません。

この例では、ロボットはゲームが始まるとすぐに動き回って対戦相手を側面から攻撃するようにプログラムされています。

void startRoutine() {
  // .
  delay(1000);
 
  //    45 .
  motorL.setSpeed(255);
  motorR.setSpeed(0);
  delay(180);
 
  //  .
  motorL.setSpeed(255);
  motorR.setSpeed(255);
  delay(450);
 
  //  ,     .
  motorL.setSpeed(-0);
  motorR.setSpeed(255);
  uint32_t startTimestamp = millis();
  while (digitalRead(OPPONENT_FC)) {
    // ,        .
    if (millis() - startTimestamp > 400) {
      break;
    }
  }
 
}


探す


スタート直後に敵を攻撃することができなかった場合、ロボットはリングを一周して敵を見つけます。それが検出されると、検索モードが停止し、ロボットが攻撃を開始します。

void search() {
  //   .
  if (searchDir == LEFT) {
    motorL.setSpeed(100);
    motorR.setSpeed(255);
  } else {
    motorL.setSpeed(255);
    motorR.setSpeed(100);
  }
}


攻撃


5つのセンサーの1つで敵が検出されると、ロボットは彼の方向を向き、全速力で攻撃します。

攻撃の正確さは、ロボットが全速力で前進するときに敵をどれだけ明確に追跡できるかに依存します。敵がなんとか脱出できた場合、プログラムは検索モードで作業を再開します。

void attack() {
  uint32_t attackTimestamp = millis();
  //    .
  //     .
  if (!digitalRead(OPPONENT_FC)) {
    motorL.setSpeed(255);
    motorR.setSpeed(255);
  }
 
  //   .
  //  .
  else if (!digitalRead(OPPONENT_FL)) {
    motorL.setSpeed(0);
    motorR.setSpeed(255);
  }
 
  //   .
  //  .
  else if (!digitalRead(OPPONENT_FR)) {
    motorL.setSpeed(255);
    motorR.setSpeed(0);
  }
 
  //   .
  //     ,     .
  else if (!digitalRead(OPPONENT_L)) {
    motorL.setSpeed(-150);
    motorR.setSpeed(150);
    while (digitalRead(OPPONENT_FC)) {
      // ,    .
      if (millis() - attackTimestamp > 400) {
        break;
      }
    }
  }
 
  //   .
  //     ,     .
  else if (digitalRead(OPPONENT_R) == 0) {
    motorL.setSpeed(150);
    motorR.setSpeed(-150);
    while (digitalRead(OPPONENT_FC)) {
      // ,    .
      if (millis() - attackTimestamp > 400) {
        break;
      }
    }
  }
}


退却


エッジセンサーの1つが白い線を検出すると、ロボットは一歩下がって向きを変える必要があります。回転中、ロボットは敵を探し続けます。見えてくると攻撃されます。

void backoff(uint8_t dir) {
  //  .
  motorL.setSpeed(0);
  motorR.setSpeed(0);
  delay(100);
 
  // .
  motorL.setSpeed(-255);
  motorR.setSpeed(-255);
  delay(200);
 
  //  .
  motorL.setSpeed(0);
  motorR.setSpeed(0);
  delay(100);
 
  // .
  if (dir == LEFT) {
    motorL.setSpeed(-150);
    motorR.setSpeed(150);
  } else {
    motorL.setSpeed(150);
    motorR.setSpeed(-150);
  }
  delay(100);
 
  //   
  //  -   .
  uint32_t uTurnTimestamp = millis();
  while (millis() - uTurnTimestamp < 300) {
    //   ,     .
    if ( !digitalRead(OPPONENT_FC) ||
         !digitalRead(OPPONENT_FL) ||
         !digitalRead(OPPONENT_FR) ||
         !digitalRead(OPPONENT_L) ||
         !digitalRead(OPPONENT_R) ) {
      //  .
      motorL.setSpeed(0);
      motorR.setSpeed(0);
      delay(100);
 
      //    .
      return;
    }
  }
 
  //    ,         .
  motorL.setSpeed(255);
  motorR.setSpeed(255);
  delay(200);
}
