単一行の乗算表


写真には、皆さんがおなじみの通常の乗算​​表が表示されています。
それを構築するためのアルゴリズム全体が79文字の1つの標準Python文字列に圧縮されることを除いて、特別なことは何もありません（PEP8を参照）。誰も気にしない、猫の下で歓迎します。

そもそも、「なぜこれなのか」という質問が多かったのです。それはすべて、2つの要因から成り立ちました。1つは、善良な1人の緊急の推奨に基づいて、Pythonの詳細な調査を行いました。2つ目は、デモシーンのコンセプトが好きです。その結果、非常に小さい（理想的には1行で）、視覚的な、何かすべてをPythonコーディングの機能を使用して（もちろん、大声で言って）書きたいという要望がありました。九九を表示することにしました。

私がPython3.7で書いていることは注目に値します。f-lineがサポートされていないため、3.6未満のバージョンは機能せず、その結果、作業スクリプトは79文字を超えます。

8行が1行になった方法


まず、乗算表を表示するコードを書きましたが、コンパクトさはまったく気にしていません。



上場
def MultiTabl():
	tabl = ''
	for y in range(1,11):
		for x in range(1,11):
			tabl += f'{x*y}\t'
		tabl += f'\n'
	return tabl


ジェネレーターを使用してテーブル値を生成し、ループを離れてリストをアンパックできます。このアプローチの欠点は、行数が多いことです。



上場
def MultiTabl():
	nums = [[x*y for x in range(1,11)] for y in range(1,11)]
	tabl = ''
	for line in nums:
		for item in line:
			tabl += f'{item}\t'
		tabl += f'\n'
	return tabl


f-lineを使用してジェネレーターにタブ（ '\ t'）の配置を与えることもできます。

nums = [[f'{x*y}\t' for x in range(1,11)] for y in range(1,11)]

最初のループで抽出されたリストが文字列結合（）メソッドを使用して文字列に接着され、変数の並列割り当てを使用してループを1行に配置すると、コードサイズが大幅に削減されます。



上場
def MultiTabl():
	nums, tabl = [[f'{x*y}\t' for x in range(1,11)] for y in range(1,11)], ''
	for line in nums: tabl += ''.join(line) + '\n'
	return tabl


そして、ジェネレーターにjoin（）と '\ n'を追加すると：

def MultiTabl():
	nums, tabl = [''.join([f'{x*y}\t' for x in range(1,11)])+'\n' for y in range(1,11)], ''
	for line in nums: tabl += line
	return tabl

これで、文字列のリストが自由に使えるようになりました。また、join（）を使用して接着することもできるため、ループを取り除くことができます。
def MultiTabl():
	tabl = ''.join([''.join([f'{x*y}\t' for x in range(1,11)])+'\n' for y in range(1,11)])
	return tabl

まあ、約束されたオプションは1行で（もちろん、印刷物を取り除くことはできません）

print(
''.join([''.join([f'{x*y}\t' for x in range(1,11)])+'\n' for y in range(1,11)])
)

もちろん、Pythonの達人は「それは何ですか？」と言いますが、このアプローチは初心者には明らかではないことに注意してください。
