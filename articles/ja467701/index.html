ディレクトリではなくカテゴリ。ファイルを簡単に保存するためのツール

記事に触発されました


"ディレクトリの代わりのカテゴリ、またはLinux Vitisのセマンティックファイルシステム"、

vitisPowerShell Core用に独自のアナログユーティリティを作成することにしました。


なぜこれを始めたのですか？


まず、vitisLinuxのみです。
次に、PowerShellでパイプを使用します。


クロスプラットフォームユーティリティを作成したかったので、.Net Coreを選択しました。


バックグラウンド


最初は混乱がありました。その後、フォルダがディスクに表示されました。
しかし、混乱はまだ支配しました。そして、タグだけでなく、サブタグとタグの同義語が登場しました。しかし、混乱も彼らを捕らえました。そして、カテゴリーが発明されました。


概念


ファイルはフォルダ階層ではなく、特定のカテゴリのヒープに格納されます。1つのファイルは複数のカテゴリに属する​​ことができます。


ファイルを見つける必要がある場合は、そのファイルが属するカテゴリを入力すると便利です。どのフォルダに保存するよりも覚えやすいです。


しかし、そのようなファイルシステムは、コード、ソースコードなどを格納するのに適したものではありません。
たとえば、家族の写真、音楽、ドキュメントなどを格納することを目的としています。


使用する


さて、デモを始めましょう。


Graphileをインストールします。


# install.ps1      
PS D:\Source\repos\Graphile> .\install.ps1

まず、フォルダー内のGraphileを初期化します。


#   
PS C:\Users\Dell\GraphileTest> Import-Module GraphilePowerShell
#  Graphile  
PS C:\Users\Dell\GraphileTest> Init-Graphile
C:\Users\Dell\GraphileTest\graphile.db

これで、すべての準備が整いました。これで、新しいカテゴリを作成できます。


PS C:\Users\Dell\GraphileTest> New-Category -Name "music"
music

ファイルを追加します。


PS C:\Users\Dell\GraphileTest> Get-ChildItem "D:\Music\Pink Floyd - The Wall" -Recurse -Filter "*.mp3" | foreach { $_.FullName } | foreach {
>> Add-FileToCategory -File $_ -Categories "music"
>> }
