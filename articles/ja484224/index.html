ビットごとのLSDソート（基数ソート）



最近、さまざまな並べ替えアルゴリズムとそれらの比較に関する多くの記事が公開されたので、自分で5セントにすることにしました。

カウント（基数ソート）を使用してLSD（最下位桁-最初に最下位ビット）をビットごとにソートするための私のお気に入りのアルゴリズムについてお話します。古典的なアルゴリズムは、加速と単純さを優先して、いくつかの最適化の方向で作者によって幾分再考されました。

したがって、提案された分類は持続可能です。 32ビット整数をソートします。機能させるには、〜（n + 4KB）の追加メモリが必要ですが、これはやや無駄ですが、パフォーマンスをある程度向上させることができます。

この種のLSDでは、比較と交換は使用されず、アルゴリズムは完全に線形です。計算の複雑さO（N）。

アルゴリズムの主な機能は、高度に混合またはランダムなデータセットの高効率です。ほとんどソートされたセットでは、ゲインはそれほど重要ではないため、他のアルゴリズムを使用することは理にかなっています。数百要素未満の小さな配列ではうまく機能しません。

ソートはローカルで行われ、メモリを節約します。

//==================================================
// RADIX  (  by rebuilder)
//   ,  .
//   (n),   ~(n+4k)
//==================================================
procedure RSort(var m: array of Longword);
//-------------------------------------------------- 
   procedure Sort_step(var source, dest, offset: array of Longword; const num: Byte);
   var i,temp : Longword;
       k : Byte;
   begin
     for i := High(source) downto 0 do
     begin
       temp := source[i];
       k := temp SHR num;
       dec(offset[k]);
       dest[offset[k]] := temp;
     end;
   end;
//-------------------------------------------------- 
//    ,    
var s : array[0..3] of array[0..255] of Longword;  
    i,k : longword;
    //     k
    offset : array[0..3] of byte absolute k;       
    m_temp : array of Longword;
begin
  SetLength(m_temp, Length(m));                
  //       
  FillChar(s[0], 256 * 4 * SizeOf(Longword), 0);   

  //  
  for i := 0 to High(m) do                         
  begin
    k := m[i];
    Inc(s[0,offset[0]]);
    Inc(s[1,offset[1]]);
    Inc(s[2,offset[2]]);
    Inc(s[3,offset[3]]);
  end;

  //    
  for i := 1 to 255 do                             
  begin
    Inc(s[0,i], s[0,i-1]);
    Inc(s[1,i], s[1,i-1]);
    Inc(s[2,i], s[2,i-1]);
    Inc(s[3,i], s[3,i-1]);
  end;

  //        
  Sort_step(m, m_temp, s[0], 0);                   
  Sort_step(m_temp, m, s[1], 8);
  Sort_step(m, m_temp, s[2], 16);
  Sort_step(m_temp, m, s[3], 24);

  SetLength(m_temp, 0);                            
end;
//==================================================

コードはPascalで書かれていますが、便利な言語に移植することは難しくありません。

実行シーケンスは次の2つの段階で構成されます。

  1. 各ブロック（2桁の8桁-1バイト（最適値））について、カウントすることにより、新しい配列内のその位置が計算されます。
  2. 各ブロックについて、重要度の低いものから順に、以前に計算された位置に移動します。

改善点：

  1. オフセットの配列の場合、メモリ内の配置を使用します。ボリュームが小さいため、それはL1（プロセッサキャッシュ）に配置されます。
  2. オフセットの配列は、すべての桁に対してすぐに入力されます。これにより、配列をウォークスルーして1回だけカウントできます。
  3. 位置計算は、配列の先頭からではなく、最後から2つの問題を解決します。
    • 最初のパスでの配列の最後は、すでに「ウォームアップ」されたキャッシュにあります。これは、大きな配列ではわずかな高速化をもたらします。
    • 第2に、ゼロへの下降サイクルは、上昇サイクルに比べて、サイクルの各ステップで1つのアセンブラー命令によって短くなります。
  4. 各反復（4つのうち）では、美しさは劣りますが、ネストされたループは使用されませんが、プロセッサー命令がいくつか保存されます。

単純であるため、コードは速度が32ビットと64ビットの両方のコンパイラビルドとほとんど同じです。必要に応じて、16ビットと64ビットの数値のアルゴリズムのバージョンを想像するのは簡単です。

64ビットプラットフォームでのランダムサンプリングアルゴリズムとクイックソートの比較（平均して、それぞれ10パス）。
