ハードウェアの作成


ハードウェアの作成を開始します。 DMAブロックがクロックサイクルでどれだけ競合するかを理解するには、Avalon-MMバス（Avalon Memory-Mapped）の高負荷時に正確な測定を行う必要があります。アルテラのJTAGからAvalon-MMへのブリッジは、高いバス負荷を提供できないことがすでにわかっています。したがって、今日、バスに高速でアクセスできるように、システムにプロセッサコアを追加する必要があります。これがどのように行われるかは、ここで説明されています。最適化のため、プロセッサコアの両方のキャッシュを無効にしますが、ここで行ったように、1つの強く接続されたバスを作成します。



8キロバイトのプログラムメモリとデータメモリを追加します。メモリはデュアルポートであり、特別な範囲のアドレスを持っている必要があることを覚えておいてください（ジャンプしない-ロックするために、ここですべての理由を説明しました）。



私たちはすでにプロジェクトを1000回作成しているため、作成プロセス自体には特に興味深いものはありません（もしあれば、プロジェクトを作成するためのすべての手順をここで説明します）。

ベースの準備ができました。次に、メモリに配置するデータソースが必要です。理想的なのは、常に刻々と過ぎるタイマーです。何らかの措置の間にDMAブロックがデータを処理できなかった場合、欠損値によってこれがすぐにわかります。まあ、つまり、メモリに値1234と1236がある場合、それは、時計で、タイマーが1235を発行したときに、DMAブロックがデータを転送しなかったことを意味します。ファイルを作成するそのような単純なカウンターを備えたTimer_ST.sv：

module Timer_ST (
  input              clk,
  input              reset,
	
  input  logic       source_ready,
  output logic       source_valid,
  output logic[31:0] source_data
	
);
    logic [31:0] counter;
    always @ (posedge clk, posedge reset)
    if (reset == 1)
    begin
        counter <= 0;
    end else
    begin
        counter <= counter + 1;
    end

    assign source_valid = 1;
    assign source_data [31:24] = counter [7:0];
    assign source_data [23:16] = counter [15:8];
    assign source_data [15:8] = counter [23:16];
    assign source_data [7:0] = counter [31:24];

endmodule

このカウンターは先駆者のようなものです。それは常に準備ができて（出力でsource_validは常に1です）、常にカウントします（リセット状態の瞬間を除いて）。モジュールがこれらの信号を正確に持つ理由- この記事で説明しました。
