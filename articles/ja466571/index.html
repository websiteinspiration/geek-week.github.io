Tesseract OCRのヒント-OCRパフォーマンスを向上させるために独自の語彙を作成します

これは、Tesseractを使用した光学式文字認識（OCR）に関する私の最初の投稿です。Tesserastは、GoogleがサポートするOCR用の非常に人気のあるオープンソースライブラリであり、高精度の結果を提供し、100を超える言語をサポートしています。この投稿では、Tesseract言語モデルの標準辞書を使用して、ニーズに合わせてカスタマイズする方法について説明します。猫の下で誰が気にかけてください。

Tesseract言語モデルと辞書


Tesseractは言語モデルと辞書を使用して、特定の言語のテキストを認識します。言語モデルには、ニューラルネットワークモデルのパラメーターの値とその他の学習データが含まれています。たとえば、英語の言語モデルはeng.traineddataファイルに格納されています。ユーザーはテッセラクトの単語リストを独自に作成して、テッセラクトがそれらを認識できるようにすることができます。

Tesseractを使用すると、独自の単語を追加してサポートされている言語の標準辞書を拡張したり、標準辞書の単語を独自の単語で完全に置き換えて言語モデルをトレーニングしたりできます。

Tesseractは、辞書の単語のさまざまなカテゴリに特別な.dawgファイルを使用します。たとえば、辞書の主要な単語には.word-dawgファイルが使用され、最も一般的な単語にはfreq-dawgファイルが使用されます。この問題の詳細については、こちらをご覧ください。

Tesseract言語モデルのカスタマイズ


このガイドは、Tesseract 3.0.5および4.0.0のバージョンに適用できます。 Tesseract 4.0.0と以前の3.0.xの唯一の違いは、バージョン4ではTesseractがLSTMモデルに基づいており、dawg辞書ファイルの拡張子がlstm-dawgであるということです（バージョンv3.0.5では拡張子-dawg）。たとえば、最も一般的な単語のファイルの拡張子はfreq-dawgではなくlstm-freq-dawgになり、unicharsetファイルには拡張子lstm-unicharset（以前は.unicharset）が付けられました。

まず、Tesseract OCRライブラリをインストールします。このチュートリアルでは、Ubuntu OS（Ubuntu 18.04を使用）とTesseract v4を使用します。 Tesseractをaptパッケージでインストールするだけです：

sudo apt update && sudo apt install tesseract-ocr

Tesseractライブラリ自体に加えて、このチームは言語モデルのトレーニングに必要なすべてのツール（トレーニングツール）もインストールします。

まず、自分の言葉で言語モデルを拡張します。たとえば、英語のモデルを拡張したいとします。wordlistfileというプレーンテキストファイルを作成し、モデル辞書に追加する単語を1行に1つずつ追加します。

次に、tessdataフォルダーに移動します。このシステムフォルダで操作を実行するには、root権限が必要です。

sudo su

wordlistfileをこのフォルダーにコピーします。言語モデルファイルeng.traineddataをtraineddat_backupフォルダに解凍します

combine_tessdata -u eng.traineddata traineddat_backup/eng.

このコマンドは、言語モデルをtraineddat_backupフォルダーにコンパイルするために必要なすべてのファイルを抽出します。

次に、wordlistfileユーティリティを使用して、wordlistfileからeng.lstm-word-dawgファイルを作成します。

wordlist2dawg wordlistfile eng.lstm-word-dawg traineddat_backup/eng.lstm-unicharset

新しいeng.traineddata言語モデルファイルをコンパイルします

combine_tessdata -o eng.traineddata eng.lstm-word-dawg

独自の辞書単語からeng.traineddata言語モデルファイルを取得します。

ここで、標準辞書の単語を独自の単語で完全に置き換えることにより、eng言語モデルを教えます。最初に、traineddat_backupフォルダーにあるすべてのdawgファイル（.lstm-word-dawg、.lstm-freq-dawgなど）をバックアップする必要があります。新しいファイルに置き換えるためです。tmpフォルダーを作成し、すべてのdawgファイルをそこに転送するだけです。

その後、先ほど作成したeng.lstm-word-dawgファイルをtraineddat_backupフォルダーにコピーします。このフォルダに移動して、新しい言語モデルをコンパイルしましょう

combine_tessdata eng.

Tesseract OCR構成ファイル


Tesseractは、ユーザーがOCR結果を制御できるようにする構成ファイル（スペースで区切られた変数とそのKey-Value値を含むプレーンテキストファイル）を使用します。独自の構成（myconf）を作成してtessdataフォルダー内のconfigsフォルダーに配置し、Tesseractを使用するときに構成名を指定できます。

tesseract <image> <options> myconf

ここで、オプション：outは結果を出力するファイルの名前または「stdout」、-lは言語モデル、-psmはページセグメンテーションメソッドです。

Tesseractは、出力をカスタマイズし、OCRの精度を向上させるための多くの制御オプションを提供します。したがって、辞書の使用を制御する変数があります。たとえば、word_dawg / user_wordsにない単語の除外（language_model_penalty_non_freq_dict_wordおよびlanguage_model_penalty_non_dict_word）があります。制御パラメータの詳細については、こちらをご覧ください。

私の設定では次の値を使用しました：

language_model_penalty_non_freq_dict_word 1
language_model_penalty_non_dict_word 1

これにより、私の辞書からいくつかの単語を認識できました。
