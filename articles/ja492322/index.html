Raspberry Pi + Fedora（aarch64）= Wi-Fiホットスポット（または青い帽子のラズベリールーター）

記事「Raspberry Pi + CentOS = Wi-Fi Hotspot（または赤い帽子のラズベリールーター）」では、CentOSオペレーティングシステムを使用してラズベリーをワイヤレスアクセスポイントに変換する方法について説明しました。そのような図面から私のホームルーターを集めて、私は自分の創造的な自我に満足し、私の居心地の良いインフラストラクチャの重要な要素に対する落ち着きの告発を受けました。しかし、不完全な決断と内面的な完全主義の感情は、「仕事の不完全な結果には存在する権利がありません。」 「その理想は達成できるし、達成しなければならない」という考えは、私を1分間去りませんでした。

そして、ある日、テーマフォーラムの1つで、「Malinka」（aarch64とarmhfp）の既存のオペレーティングシステムのビット深度について話し合った。

「ユーザーランド」のお気に入りのARMアーキテクチャ向けCentOSは、急いで最新バージョンのコアに切り替えて64ビットにすることはありませんでした。そして、EPELレポジトリは、デジタル署名なしでどこにあるかを知っていて、私の悩みの夢の悪夢でした...

RPMベースのディストリビューションの支持者と言えば、Fedoraが「ラズベリー」のOSの議論で絶対に忘れられていることに気付いて驚いた！そして、これは28番目のバージョンからのリリースが64ビットバージョンのRaspberry Pi 3B +を公式にサポートしているという事実にもかかわらず
です！


この記事では、Raspberry Pi 3 Model B +にFedora（aarch64）を追加の最小バージョンでインストールする方法について説明します。CentOS 7での以前の構成の試験運用の結果として特定されたWi-Fiアクセスポイントを上げる機能について簡単に説明します。

0.必要なもの


前の記事に記載されているものと同じです：

  • Raspberry Pi 3モデルB +;
  • microSD> = 4GB（後でシステムを2GBドライブに「転送」できます）;
  • microSDカードリーダーを備えたLinuxワークステーション。
  • Linuxを使用したRaspberryとワークステーションの有線ネットワーク接続（この場合、設定に追加のモニターとキーボードは必要ありません）、両方のデバイスからのインターネットアクセス。
  • Linuxの高度なスキル（知っていることと恐れないこと：parted、dd、mkfs）。

独自のLinuxの 反復LFS-アセンブリと同様に、Fedoraディストリビューションイメージが使用され、その後、最小限のシステムが（「ソースからのコンパイル」なしで）そのベースで作成されます。

1.元のディストリビューションをインストールする


Web上のシステムの未加工イメージの座標：
https：//.../fedora-secondary/releases/.../Spins/aarch64/images/Fedora-Minimal-...xz

microSDに書き込まれた後、使用する前に必要です。 ：

  1. ファイルシステムの「ルート」を拡張する（3番目のセクション、ext4）

    parted /dev/mmcblk0 resizepart 3 100%
e2fsck -f /dev/mmcblk0p3; resize2fs /dev/mmcblk0p3; e2fsck -f /dev/mmcblk0p3
for i in 1 2 3; do mkdir -p /mnt/$i; mount /dev/mmcblk0p$i /mnt/$i; done

  2. SELinuxを無効にする

    echo 'SELINUX=disabled' > /mnt/3/etc/selinux/config

  3. 初期セットアップウィザードを削除します。

    find /mnt/3/etc/systemd/ -iname initial-setup.service -delete

  4. sshアクセスを許可：

    mkdir -p /mnt/3/root/.ssh
cp -fv ~/.ssh/id_rsa.pub /mnt/3/root/.ssh/authorized_keys
sed -i 's/#PermitRootLogin.*/PermitRootLogin yes/g' /mnt/3/etc/ssh/sshd_config


これで、microSDから「ラズベリー」をダウンロードして、ネットワーク経由で接続できます。

コールドスタートには約1分半かかります。起動後のTTXシステム：


rpm -qa | wc -l
444

2.最小限のシステムアセンブリ


残念ながら、開発者からの「最小限の配布」は、リソース消費の中で最も控えめなものとはかけ離れています。システムのイメージをさらに小さくすることができます。

これを行うには、「ラズベリー」でスクリプトを実行する必要があります。

#!/bin/bash

. /etc/os-release
P=$(mktemp --directory $(pwd)/$ID-$VERSION_ID.XXX)

dnf --installroot=$P --releasever=$VERSION_ID --setopt=install_weak_deps=false \
--assumeyes install  \
    bcm283x-firmware \
    dnf              \
    grub2-efi-aa64   \
    kernel           \
    openssh-server   \
    shim-aa64

for f in /boot/efi/EFI/fedora/grub.cfg \
         /boot/efi/EFI/fedora/grubenv  \
         /boot/efi/rpi3-u-boot.bin     \
         /etc/default/grub             \
         /etc/fstab
do
  cp -fv $f $P$f
done

rm  -fv $P/dev/*
rm -rfv $P/var/cache/dnf

echo "--------------------------------------------------------------------------------"
du -hs $P

スクリプトを実行した後、新しい最小OSエディションのルートのコンテンツを含むサブディレクトリ（$ P）が現在のディレクトリに作成されます。「ラズベリー」をオフにして、microSDをLinuxワークステーションに戻すことができます。

3.最小システムのインストール


インストールは、OSの最小の「イメージ」（前のステップで取得）のファイルを、適切に作成されたmicroSDの適切なディレクトリにコピーすることで削減されます。

2GBカードとその上の2つのセクションで十分です。

  1. / boot / efi -EFI + FAT32、ブート、100MB;
  2. /（ルート）-EXT4、残りのすべてのスペース。

microSDを準備してファイルをコピーした後、次のものが必要です。

  • ブートOSを修復します。
  • ネットワークをオンにします。
  • ssh経由でアクセスを構成します。

ダウンロードを修復するには、ファイル内のセクションのUUIDを置き換えます。

microSD:/boot/efi/EFI/fedora/grub.cfg
microSD:/boot/efi/EFI/fedora/grubenv

そして、最後のファイルのsaved_entry =パラメータ

ファイル内：

microSD:/etc/fstab

コマンドの出力で、古い値と現在の（現在の）値を見つけることができます。

blkid | grep mmcblk | sort

置き換え後、マウントポイントが新しいパーティションのUUIDに対応するように、fstabの内容をmicroSDに調整する必要もあります。

「ラズベリー」を初めてオンにすると、小さな「松葉杖」でネットワークパフォーマンスを実現できます-リンクを（概略的に）作成します。

ln -s /usr/lib/systemd/system/systemd-networkd.service \
  microSD:/etc/systemd/system/multi-user.target.wants
そしてファイル：
mkdir -p microSD:/etc/systemd/network
cat > microSD:/etc/systemd/network/dhcp.network << EOF
[Match]
Name=*
[Network]
DHCP=ipv4
EOF

ブートが成功したら、systemd-networkdの起動を片付けます。

systemctl disable systemd-networkd
systemctl enable systemd-networkd

sshによるスーパーユーザーアクセスは、項目1と同様に構成されます。

エラーを発生させずにすべてを適切に実行したら、「ラズベリー」でmicroSDを再配置し、最小限のパフォーマンスで64ビットOSの使用を開始できます。

4.レディシステム


上記の手順に従って作成された完成したシステムの「イメージ」は、リンクからダウンロードできます
。Fedora-Tiny-31-5.5.7-200.aarch64

これは、2つのファイルを含むアーカイブになります。インストールスクリプトとOSファイルを含むTGZです。Linuxワークステーションでアーカイブを解凍し、microSD（2GBカードで十分）を挿入し、パラメーター-デバイスの名前を指定してスクリプトを実行する必要があります。

./install /dev/mmcblk0
注意してください！

警告なしで、デバイスがフォーマットされ、オペレーティングシステムがインストールされます。

スクリプトをエラーなしで実行した後、カードを「ラズベリー」に再配置して使用できます。dhcpでキャッチするには、パスワードは「1」です。

システムからすべての種類のIDとキーが削除されるため、新しいインストールごとに一意になります。

繰り返しますが、システムは最小限です！したがって、心配しないでください。DNFを使用できます。その機能を使用するには、正しい/etc/resolv.confを「構成」する必要があります。

ラズベリーのコールドスタートには、約40秒かかります。起動後のTTXシステム：


rpm -qa | wc -l
191

5. Wi-Fi


Wi-Fiアクセスポイントの実装の機能について少し詳しく説明します。詳細については、以前の記事を参照してください。

EPELは不要になりました-すべてのパッケージは公式リポジトリに含まれています。

おそらく、それはdnsmasqを放棄する価値があります。CentOSとは異なり、Fedoraにはかなり最近のsystemd-networkdがあり、そこには通常の組み込みDHCP / DNSサーバーがあります。しかし、RHEL8では、開発者がNM以外でネットワークスタックをサポートすることを拒否したという事実は、プロジェクト（悪者）の明るい未来への自信を刺激しません。要するに、私はそれを試していません。

さらに、組み込みWi-Fiアダプターの現在のドライバーは、Raspbianディストリビューションから「盗む」ことはできませんが、githubから直接ダウンロードできます。

これは、Broadcomファームウェアファイルが私の「ラズベリー」上で（概略的に）どう見えるかです。

それらがないと、5GHz / ACは機能しません。

インターフェイスの数と名前について。ここで、特別な必要がない人は誰でも、ソフトウェアスタック（ブリッジ）の「サービス」に頼らないことを強くお勧めします。これにより、ネットワークスタックに大きな負荷がかかり、ルーティングが阻害されます。複数のワイヤレスアダプターが計画されていない場合は、物理インターフェイスのみを使用する必要があります。私は2つのWi-Fiを使用しているので、それらを組み合わせてソフトウェアブリッジを作成するだけです（ただし、hostapd設定を別の方法で見ることで、それなしで行うことができます）。

そして、私はインターフェースの名前を変更したいです。

これを行うには、Fedoraでシンボリックリンクを作成します。

/etc/systemd/network/99-default.link -> /dev/null

そして、systemd-networkdによってのみ、udevをいじることなく意味のある名前を付けることが可能になります。

ここでは、たとえば、私のルーターではネットワークアダプターと呼ばれます。

1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000

2: wan: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP group default qlen 1000

3: lan: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc noqueue state UP group default qlen 1000

4: int: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel master lan state UP group default qlen 1000

5: ext: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc mq master lan state UP group default qlen 1000

  • int-ビルトイン、ext-外部（USB）Wi-Fiアダプターが「ブリッジ」LANに組み込まれています
  • wan-インターネットが接続されているイーサネットアダプタ。

気づいてますか？fq_codelは本当に良いことです。新しいLinuxカーネルと一緒に使用すると、ワイヤレスの範囲で驚異的に機能します。激しい「トレント品質」は、近隣の速度の急激な低下につながりません。チャンネルがロードされた「無線」の家庭用IP-TVで作業しても、言葉が「崩れる」ことはなく、言葉が「途切れる」こともありません。hostapd

デーモンサービスファイルに小さな変更が加えられました。 これは次のようになります（例として組み込みアダプタを使用）。



[Unit]
Description=Hostapd IEEE 802.11 AP, IEEE 802.1X/WPA/WPA2/EAP/RADIUS Authenticator

[Install]

6.よくある質問


6.1 RaspberryでWi-Fiルーターを作成する理由


「試してみるのはおもしろい」などと簡単に答えることができます。

しかし、実際には、それは私には思えます、トピックは非常に深刻です。 「血まみれの」インターネットの時代では、店内でルーターを購入し、その製造元の人質に留まることは非常に鈍い見通しです。多くの人々は、CVEや有線のバックドアを設置することは不可能であることをすでに理解しています。

もちろん、愛好家からWRTファームウェアに移行できます。それらへの信頼はおそらくより高いでしょうが、それらに依存したくないのであれば、あなた自身の製品のみです。理想的には、世界中のすべてのものを実装できる可能性を備えた本格的なコンピュータ。もちろん、ルーティングに関しては。

したがって、「ラズベリー」の選択は、純粋に経済的な動きです。つまり、実際のコンピューターと同時に安価です。しかし、それも可能です-「デュアル」の内部。

6.2しかし、Raspberryは「サブルーター」です。低速で、イーサネットポートが1つあります。


家庭用Wi-Fiルーターとして、「Malinka」は私よりも適しています。「空中」の速度については、すでに上で述べました。そして、ほぼ同じ方法で、Appleの同様の製品のイーサネットは1つだけです。

でもまじめに、もちろんもっと欲しいです。そして、私の家庭ではすべてのデバイスが無線で接続されているという事実にもかかわらず、時には銅線接続が依然として必要です。そのような場合のために、私は店内に「モバイルハブ」を持っています。

デバイス-など

6.3これがルーターの場合、TCP / IPの「チューニング」については何も言われません。重要なのでです。


ネットワークスタック（tcp_fastopen、YeAHなど）の設定に加えて、この記事と前の記事では、他のニュアンス、特にmicroSDを最適に使用するための準備プロセスを開示していません（ただし、インストーラーはメモリカードを慎重にフォーマットしようとします）。改善のプロセスは無限であり、あなたはただ時間内に停止する必要があります。

6.4 Fedoraを選ぶ理由


好きだから！Fedora-「主流」-オタクのシステム。実際、この記事の対象です。この記事の執筆時点では、64ビットバージョンのRaspberry用の唯一のOSは、開発者のかなりのスタッフによって正式にサポートされていると思われます（5.6カーネルを待ちきれません）。

6.5 Bluetoothは機能しますか？ビデオ/サウンド/ GPIOはどうですか？


私は知らない。この記事では、システムの最小限のインストールと、その後のWi-Fiルーターとしての使用について説明します。

6.6すべてのCentOS / Fedora / RedHat記事がSELinuxを無効にすることから始まるのはなぜですか？


システムは最小限なので、ファイアウォールとそれをセットアップするためのユーティリティすらありません。誰がそれを必要とするか-彼らは必要なすべてのものを追加でインストールすることができます。

6.7。システムは使用できず、パスワードは変更できません-パスワードなし。ping、いいえ！


DNF があります。または、このインストールオプションはあなたのためではありません-開発者からの配布キットを使用してください。

6.8 SWAPはどこにありますか？彼なしでは生きられない！


真実？しかたがない：
