TacacsGUI、構成マネージャー

こんにちは！本日は、TacacsGUIプロジェクトに含まれているプラ​​グインであるConfiguration Managerに注目したいと思います。構成マネージャー とは何ですか？これは、（例えば、構成マネージャである酸化型又はRANCID AAAログ（認証認可アカウンティング）の変化（差分）とエントリを表示するための様々なモードを有します）。したがって、構成間の差異（diff）を制御できるだけでなく、誰がこれらの変更を行ったかを判別することもできます。シスコとジュニパーでテスト済み。内部のデモへのリンク。





それで、私たちの計画はこれです：私たちはメインのインターフェースに慣れます。すべての長所と短所を評価します。次に、この機能を構成する方法を学びます。デモを見る; 結論を導き、コメントを残してください。

次に、図に示されている各ステップをさらに詳しく分析します。

ステップ1.デバイスを追加する


ここではすべてが簡単です。アドレス、プロトコル（ssh、telnet）、ポートを指定します。オプションで、ユーザー（資格情報）とプロンプトを指定できます。このステップでユーザーが指定されていない場合は、デフォルトのユーザー（ステップ4で定義）が使用されます。 「プロンプト」フィールドが入力されていない場合、その値はステップ3（モデルの作成）で決定されます。

それではプロンプトについて話しましょう。 Promptは、Switch>やRouter>などのコマンドを入力するためのデバイスプロンプトです。同じデバイスでも、アクセスレベルに応じて異なるプロンプトが表示される場合があります。たとえば、Switch>、Switch＃、またはSwitch（config）＃です。CMプロンプト表記を使用して、コマンドを入力するタイミングを理解します。可能なすべてのプロンプトオプションをコンマで区切ってリストできます。将来、私はこの説明を一度は参照しません。



ステップ2.ユーザーを作成する


入力する必要がある3つのフィールドがあります。アカウントの名前（資格情報名）、ユーザー名（ユーザー名）、およびそのパスワード（パスワード）です。[資格情報名]フィールドは、選択されたときにユーザーリストに表示され、必須です。[ユーザー名]フィールドはデバイスでの認証に使用されます。たとえば、Telnetでパスワードのみが使用される場合はオプションです。



ステップ3.モデルを作成する


最も責任のあるステップ。ここでは、CMが実行する必要のある手順を示します。プロンプトフィールドへの入力はステップ1で説明したものと同様であり、その値はステップ4で定義されたすべてのデバイスに適用されます。この段階でプロンプトフィールドが空白のままである場合、SMはデフォルトで＃および>文字を使用します。さらに、例を挙げて説明することをお勧めします。


この例では、SMは接続が成功した後、5つのコマンドを実行します。

  1. 招待待ち（期待）：router_15>
    コマンドを送信（送信）：有効
  2. (Expect): assword: ( )
    (Send): ****** ( , /hide)
  3. (Expect): ( , Prompt)
    (Send): terminal length 0 ( more, , more)
  4. (Expect):
    (Send): show runn (Write)
  5. (Expect):
    (Send): exit ( )

モデルとは何かを明らかにしたいと思います。

そして、はい、これは最も一般的な期待です。これらの設定は、Pexpectライブラリと組み合わせて使用​​されます。

ステップ4.リクエストを作成する


その前にオブジェクトを作成した場合は、ここでそれらをまとめます。ファイル（s）、モデル（model）、デバイス（devices）、デフォルトユーザーを保存する場所を選択します。また、出力ファイルをトリミングして、それがどのように表示されるかを確認することもできます（プレビュー、2番目の画像）。各ファイルには、テンプレートで構成される名前が付けられます- デバイス名 __ デバイス-id _ リクエストID。



ファイルは最初と最後の両方からトリミングできます。たとえば、行0から2、および-1（最後）と-2（最後から2番目）をトリミングします。

プレビューを使用すると、モデルが機能するかどうか、ファイルがどのように表示されるかをすぐに確認できます。

結論として、私のアドバイスは、構成を取得できない場合は、自分でこのデバイスに接続して、最初に指定されたコマンドを実行してみることです。これはサーバーから直接行うのが最善です。

これでクエリの設定が完了しました。これで、スケジューラ（cron）がオンになっていることを確認し、CMがオンデマンドですべての構成を収集するのを待つか、指定した時間（強制開始）を待たずにCMを起動します。

今後の計画


  1. 構成のバージョンをダウンロードします（GUIで構成されていない、すでに存在します）。
  2. CMエクスプローラーでファイルの詳細情報を追加します。
  3. モデルテンプレートを追加します。
  4. 特定のファイルを更新します。迅速にリクエストしてください。
  5. スケジューラ。すべての要求を満たすために、1日1回または週1回だけではありません。そして、スケジュールされた要求を満たします。CMを使用すると、構成を収集できるだけでなく、デバイスに構成を保存するなど、最適化などを収集することもできます。
  6. SFTP（SCP）、FTPのサポートを追加します。そのため、デバイスからCMに直接ファイルをドロップできます。
  7. SNMPサポート。これにより、構成が変更され、CMが動作を開始するというメッセージがデバイスからスローされます。
