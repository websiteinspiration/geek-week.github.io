多くの理由がありますが、私たちはそれらのうちの1つ、つまり次のことに関心があります。-
取締役会の50％以上が同じ個人である組織。

この根拠は、弁護士が企業の取締役会のすべての構成をメンバーの50％以上の偶然と比較する必要があるという点で、他とは異なります。

私たちが持っている取締役会のすべての構成が、取締役会と20を超える会社の合計数を含むExcelの表に要約されている



とします。取締役会のメンバーの総数の50％を超える偶然が存在する場合は、表を調べて会社を選択するプログラムが必要です。次に、プログラムは結果を画面またはファイルに表示します。

論理的には、プログラムは次のアルゴリズムを実行します。ユーザーから取締役の名前を受け取ったら、彼女は各会社の取締役の名前を含むExcelの各行とそれらを比較します。つまり、5人のユーザーSDメンバーのうち3人ごとが、テーブルの5人中3人ごとと比較されます。この段階では、プログラムはメンバーが5つを超えるCDを分析せず、スキップします。

プログラムの最初の行は標準です：

import openpyxl
wb = openpyxl.load_workbook('sd3.xlsx')
sheet=wb.get_active_sheet()

ここでは、Excelを操作するためのモジュールをインポートし、テーブルでファイルを開きました。

次に、理事会の5人のメンバーを入力するようユーザーに提供します。プログラムは、比較のために社会でさらに検索します。SDのユーザー入力メンバーをすべて変数a、b、c、e、fに書き込みます。

a=str(input("-1: "))
b=str(input("-2: "))
c=str(input("-3: "))
e=str(input("-4: "))
f=str(input("-5: "))

これは重要なポイントです。監督の名前はスペースなしで入力する必要があります。たとえば、Ivanov I.I. 同じ形式で、ディレクターの名前がExcelテーブルに存在する必要があります。

2つのリストを作成しましょう。1つは空で、もう1つは理事会のメンバーが5人を超える社会です。

found = []
found2=[1,10,11,12,13,14,18,27,31,32] #    sd3,    5 .
h = open('55555.txt','a')

また、結果を保存する55555.txtというテキストファイルを開きました。SDの一致率が50％を超えているということは、5人のうち3人のメンバーが一致している必要があるということです。したがって、プログラムは一致する名前を10回チェックする必要があります。これは、5人のメンバーの糖尿病の組み合わせがいくつあるかを示しています。

したがって、同じコードを10回記述しないようにするには、プログラムを煩雑にするために、関数を作成してから10回呼び出し（実行）します。

コードを分析してみましょう。

B2：L36のテーブルの境界を設定します。セルが空の場合はスキップされます。

次に、関数は最初のディレクターの名前を考慮します。セル内の名前がユーザーが入力した名前に対応している場合、関数はさらに進み、2番目と3番目のディレクターを比較します。最後に、この関数は会社のリストを形成し、それぞれをファイル55555.txtに書き込みます。
関数が作成された後、プログラムによって自動的に実行されることはありません。

呼び出す必要があります：

myfun(a,b,c)

ただし、ここでは、最初の3人のディレクターに対してのみ関数を呼び出しました。残りの組み合わせに対して関数呼び出しを繰り返す必要があります。

myfun(a,b,e)
myfun(a,b,f)
myfun(a,e,f)
myfun(a,c,e)
myfun(a,c,f)
myfun(b,c,e)
myfun(b,c,f)
myfun(b,e,f)
myfun(c,e,f)
h.close()
