useSWRは私のお気に入りのReactライブラリーです

この記事の翻訳は、コース「React.js開発者」の開始前に準備されました。


過去数か月間、私はNextJSのアプリケーションに取り組んできました。毎週どんどん増えます。アプリケーションは、API呼び出しにaxiosを使用し、状態管理にunstated-nextを使用します。 API呼び出しはたくさんありますが、ユーザーにたくさんの読み込み画面を見させたくないです。したがって、axios呼び出しの結果は、記述されていないリポジトリに格納されます。

しかし、問題が発生しました。ボールト自体はますます複雑になっています。ページは複数のAPI呼び出しを必要とする場合があり、それらは他のAPI呼び出しの結果に依存します。日は数週間に、週は月に変わりましたが、一方で私たちの述べられていない保管庫はますます扱いにくくなりました。自家製のキャッシュロジックが予期しない極端なケースに対処するのに苦労していたので、奇妙なエラーに遭遇しました。

次に、パスをより単純にする必要があると考えました。
そして、彼はここにいます。また、NextJSの作成者であるVercelによって開発されました。

SWRについて知る


SWRという名前はstale-while- revalidateに由来します。これは現在、フロントエンド開発で人気を得ているキャッシング方式です。キャッシュされたコンテンツをすぐにダウンロードして、すぐに更新して将来の新しいコンテンツを処理することができます。私たちのケースでは、パフォーマンスとユーザーエクスペリエンスの完璧な妥協点を得ました。

useSWRの使い方


useSWRは、Vercelによって開発されたReactフックライブラリです。APIまたはその他の外部ソースからデータを取得し、それをキャッシュに保存してからレンダリングできます。

まず、JSONサーバーからTODOリストを取得してレンダリングするReactコンポーネントの例を見てみましょう。

import React from "react";
import "./App.css";
const todosEndpoint = "http://localhost:3001/todos";
const TodoApp = () => {
  const [todos, setTodos] = React.useState([]);
  React.useEffect(() => {
    const getData = async () => {
      const response = await fetch(todosEndpoint);
      const data = await response.json();
      setTodos(data);
    };
     getData();
    }, []);
  return (
    <div className="App">
      {todos.map((todo) => (
        <div key={todo.id}>{todo.title}</div>
      ))}
    </div>
  );
};
export default TodoApp;

次に、同じコンポーネントを見てみましょう。ただし、useSWRを使用して書き直しました。

import React from "react";
import useSWR from "swr";
const todosEndpoint = "http://localhost:3001/todos";
const getData = async () => {
  const response = await fetch(todosEndpoint);
  return await response.json();
};
const TodoApp = () => {
  const { data: todos } = useSWR(todosEndpoint, getData);
  
return (
    <div>
      {todos && todos.map(todo => (
        <div key={todo.id}>{todo.title}</div>
      )}
    </div>
  );
};
export default TodoApp;

ご覧のとおり、これは前の実装と非常によく似ています。

この例では、TODOリストを取得するためにuseSWR（キー、フェッチャー、オプション）を使用しています。キャッシュにはuseSWRのキーが必要です。この例では、todosEndpointを使用しました。フェッチャーとして、TODOリストを取得する非同期関数を渡しました。useSWRは、データを取得する方法を認識していない

ことに注意してください。任意の非同期データマイニング戦略を使用できます。あなたは使用することができますフェッチ、axiosあるいはGraphQL。関数が非同期でデータを返す間、useSWRは幸せになります。useSWR

呼び出しは、次のパラメーターを返します。

  • data：フェッチャーを使用して取得した主要なデータ（まだロードされていない場合は未定義）。
  • error：フェッチャーが原因のエラー。
  • isValidating：リクエストまたは再検証が進行中かどうかを示すブール。
  • mutate（data？、shouldRevalidate）：キャッシュされたデータを変更する関数。

より複雑な例


useSWR によって返されるより多くのパラメーターを使用する、より複雑な例を見てみましょう。この例では、TODOリストの1つの要素を取得し、リストからタスクの完全なステータスのチェックボックスをレンダリングします。チェックボックスをクリックすると、我々は更新TODOにPUTリクエストを送信した後、我々は呼んのmutateを更新するuseSWRのキャッシュを。

import React from "react";
import useSWR from "swr";
import "./App.css";
const todosEndpoint = "http://localhost:3001/todos";
const getTodo = async (id) => {
  const response = await fetch(`${todosEndpoint}/${id}`);
  return await response.json();
};
const updateTodo = async (id, todo) => {
  const response = await fetch(`${todosEndpoint}/${id}`, {
    method: "PUT",
    headers: {
      "Content-type": "application/json; charset=UTF-8",
    },
    body: JSON.stringify(todo),
  });
  return await response.json();
};
const TodoApp = () => {
  const todoId = 1;
  const key = `${todosEndpoint}/${todoId}`;
  const { data: todo, mutate } = useSWR(key, () =>
    getTodo(todoId)
  );
  const toggleCompleted = async () => {
    const newTodo = {
      ...todo,
      completed: !todo.completed,
    };
    await updateTodo(todoId, newTodo);
    mutate(newTodo);
  };
  if (!todo) {
    return <div>Loading...</div>;
  }
  return (
    <div>
      <p>{todo.title}</p>
      <div>
        <input
          type="checkbox"
          id="completed"
          name="completed"
          value="completed"
          checked={todo.completed}
          onChange={toggleCompleted}
        />
        <label htmlFor="completed">Completed</label>
      </div>
    </div>
  );
};

mutate を使用すると、Webアプリケーションの目に見えるパフォーマンスを向上させることができます。リモートデータソースが更新されるまで待たずに、ローカルでデータを変更してビューを変更できます。useSWRは、再検証してバックグラウンドの最新データに置き換えます。

これにライブラリが必要なのはなぜですか？


あなたはこの質問をするかもしれません。おそらく、アプリケーションにすでに状態管理メカニズムがあり、サードパーティのライブラリを使用してデータの抽出やキャッシュなどの単純なことを行う意味がわかりません。

そして、私はあなたにデータをキャッシュすることは容易ではないことにお答えします。そして、あなたのアプリケーションが成長するにつれて、それはさらに難しくなります。極端なケースに遭遇し、それらを処理すると、理解するのが難しく、維持するのが難しい複雑なリポジトリとプロバイダーができてしまいます。バグはどこからでも流出します。
「コンピューターサイエンスでは、2つの複雑な問題しかありません：キャッシュの無効化と名前の発明」-Phil Carleton

独自のデータマイニングソリューションを実装する代わりに、Reactエコシステムで最も尊敬されている企業の1つによって開発された実績のあるソリューションを利用してみませんか？

useSWRがどのように役立つか


アプリケーションでのデータ抽出の問題をuseSWRに移植すると、多くの利点があります。

1.彼は私たちにコードを削除することを許可しました

不要なコードを削除するプルリクエストですが、新しいものは追加しません-私のお気に入り。アプリケーションのコードが少ないほど、エラーが発生する可能性が低くなります。そして、これは良いことです。年齢を重ねるにつれて、シンプルさがますます高くなるようになりました。useSWRを使用すると、明記されていないストレージを完全に削除できるようになりました。これにより、アプリケーションが簡素化され、理解しやすくなりました。

2.彼は、useSWR プロジェクトへの新しい開発者の接続を簡素化しました

ほとんどのオープンソースプロジェクトと同様に、優れたドキュメントを誇っています。独自のソリューションを実装するには、独自のドキュメントを作成し、プロジェクトの新しい開発者にデータ抽出の処理方法を教える必要があります。useSWRを使用するようになったので、これを行う必要はありません。

3.彼は難しいことを単純化しました

私たちのアプリケーションは、他のほとんどのアプリケーションと同様に、多くのAPI呼び出しを含んでいます。これらの呼び出しの一部は、他のAPI呼び出しに依存しています。useSWR簡単依存クエリのためのフックを書き込みます。

4.

見た目のパフォーマンスが向上し、アプリケーションの表示が速くなりました。ユーザーはそれを評価し、良いフィードバックを与えました。

5.古いデータを更新します

useSWRは、フォーカスされている古いデータを更新します。つまり、ダウンロード時間を除いて、ユーザーは常に最新バージョンのデータを入手できます。

結論


useSWRはアプリケーションに大きな影響を与えました。このライブラリにより、コードが簡素化され、ユーザーエクスペリエンスが向上しました。useSWRと同じくらい簡単に実装して多くのメリットを実現できる別のライブラリを想像することすらできません。useSWRの詳細については、GitHubのサイトまたはリポジトリにアクセスしてください。

代替ライブラリ


データを取得するには、useSWRだけが優れたライブラリではありません。私たちのケースではそれはうまくいきましたが、おそらく別のライブラリがデータ抽出に適しています。

反応クエリ


react-queryは、Reactで非同期データを取得、キャッシュ、更新するためのフックのライブラリです。書かれたこの人気のソリューション、タナーLinsiは、の範囲で非常によく似てuseSWR。

アポロ


アポロのクライアントがあるGraphQLのクライアント取得し、あなたのデータをキャッシュするだけでなく、ユーザーインターフェースを更新するの面倒を見ます。GraphQLを使用する場合、Apolloは優れたソリューションになります。
