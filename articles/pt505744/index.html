Uma árvore cartesiana é uma estrutura híbrida que combina as propriedades de uma pilha e uma árvore de pesquisa binária. Existem sinônimos para essa árvore que serão usados ​​no artigo:

Árvore de pesquisa binária




A aparência da árvore depende muito de como os elementos da matriz são localizados inicialmente. Com base em dados aleatórios, uma árvore com um alto grau de probabilidade ficará desequilibrada, com um número muito maior de níveis em comparação com o ideal. No nosso exemplo, sete níveis se formaram. Se os elementos da matriz fossem organizados, por exemplo, nesta ordem:

25, 28, 29, 23, 30, 27, 24, 22, 21, 26

, a árvore de pesquisa seria equilibrada, com o número mínimo possível de níveis (= 4). A classificação usando uma árvore balanceada teria complexidade de tempo: O ( n log n ) . Porém, no caso de uma árvore desequilibrada, ela pode se degradar para O ( n 2 ) .

Na habrastatia anterior, considerou-se a classificação por uma árvore de pesquisa binária, bem como sua modificação interessante , que permite a classificação garantida com complexidade de tempo de O ( n log n ) .

Quanto mais desequilibrada for a árvore de pesquisa, mais caro será percorrê-la.

Por outro lado, o armazenamento de dados na árvore de pesquisa é muito conveniente do ponto de vista organizacional, pois permite realizar muitas operações com esses dados, incluindo a classificação.

Um pai em uma árvore de pesquisa binária não é necessariamente mais que um filho. Mas, ao mesmo tempo, o filho esquerdo é sempre menor que o pai, o filho direito é sempre maior que o pai (ou igual a ele).

O elemento máximo da matriz ao criar a árvore de pesquisa fica em algum lugar na subárvore direita. Para extrair esse máximo, a árvore de pesquisa deve ser completamente contornada.

Árvore cartesiana


E, finalmente, a árvore cartesiana. Os relacionamentos são estabelecidos de tal maneira que, por um lado, é muito difícil se você observar os valores dos elementos. Todos os pais são mais que seus descendentes:



Por outro lado, uma estrutura é uma árvore de pesquisa binária quando você olha para os índices dos elementos. Para maior clareza, os valores dos elementos nos nós foram removidos da imagem na árvore, restando apenas índices. Cada descendente esquerdo é menor que o pai, e cada descendente direito é maior que o pai:



Por um lado, temos acesso rápido ao elemento máximo. Como esse é um monte, o máximo está na raiz.

Por outro lado, é possível remover facilmente o máximo da estrutura e trabalhar com os dados restantes. Como essa é uma árvore de pesquisa binária, remover a raiz não é caro. Alterações relativamente pequenas ocorrem nos relacionamentos pai-filho, um novo máximo atinge a raiz da árvore cartesiana e você pode continuar trabalhando.

Cada elemento é caracterizado por dois números - o valor do elemento e o índice do elemento na matriz. O índice pode ser considerado como a coordenada X , o valor como Y e o próprio elemento deve ser interpretado como um ponto no plano de coordenadas. O algoritmo final:

  • I. Com base em uma matriz não classificada, construímos uma árvore cartesiana:
    • I.1 Itere os elementos da matriz da esquerda para a direita.
    • I.1 Se este é o primeiro elemento da matriz, então é simplesmente o primeiro nó da árvore cartesiana.
    • I.2 Se esse não é o primeiro elemento da matriz, considere que seja uma árvore cartesiana que consiste em um elemento. Realizamos a operação de mesclagem dessa árvore com a árvore criada a partir dos elementos anteriores.
  • II Excluímos o elemento máximo atual da árvore, após o qual restauramos a árvore cartesiana.
    • II.1 O elemento máximo atual é a raiz da árvore.
    • II.2 Removemos a raiz da árvore, transferimos o elemento máximo para uma matriz adicional.
    • II.3 Após remover a raiz, a estrutura se dividiu em duas árvores cartesianas.
    • II.4 Para essas duas árvores, realizamos a operação Mesclar . O novo máximo atual é uma das raízes das árvores unidas.
    • II.5 Para uma nova alta, repita as etapas II.1-II.4 novamente.




Implementação em C ++:

/*
 * c++ 11 code to construct a Cartesian tree. The method cartesianTreeSort
 * will sort the contents of the array.
 */

#include <iostream>
#include <vector>
#include <queue>

struct Node{
    int value;
    Node *left, *right;
    Node *parent;

    Node(){
        value = 0;
        parent = NULL;
        left = NULL;
        right = NULL;
    }
};

// Used by priority queue
struct compare{
    bool operator()(Node *left, Node *right){
        return left->value > right->value;
    }
};

class CartesianTree{
private:
    // last pointer to keep track of last node added
    Node *root, *last;

    Node * findLowestNode(Node *node, int x){
        if(node->value < x)
            return node;
        else if(node->parent != NULL)
            return findLowestNode(node->parent, x);
        else
            return NULL;
    }

public:
    Node * getRoot(){
        return root;
    }

    void addNode(int x){
        Node *new_node = new Node;
        new_node->value = x;
        if(root == NULL){
            last = new_node;
            root = new_node;
            return;
        }
        Node *z = findLowestNode(last, x);
        if(z == NULL){
            new_node->left = root;
            root->parent = new_node;
            root = new_node;
        }
        else{
            new_node->left = z->right;
            z->right = new_node;
            new_node->parent = z;
        }
        last = new_node;
    }

    CartesianTree(std::vector<int> ar){
        root = NULL;
        last = NULL;
        // Call addNode function for each element of the array
        for(auto x : ar){
            addNode(x);
        }
    }

    void InorderTraversal(Node *node){
        // To print inorder traversal of the tree
        if(node == NULL)
            return;
        InorderTraversal(node->left);
        std::cout << node->value << ' ';
        InorderTraversal(node->right);
    }

    // Function to sort and store values in array
    void cartesianTreeSort(std::vector<int> &sorted_ar){
        // Initialize input array
        sorted_ar.assign(0, 0);

        // Initialize priority queue
        std::priority_queue<Node *, std::vector<Node *>, compare> p_queue;
        p_queue.push(root);
        Node *temp = NULL;
        while(!p_queue.empty()){
            temp = p_queue.top();
            p_queue.pop();
            sorted_ar.push_back(temp->value);
            if(temp->left){
                p_queue.push(temp->left);
            }
            if(temp->right){
                p_queue.push(temp->right);
            }
        }
    }
};

int main(){
    std::vector<int> ar = {1, 14, 5, 0, 8};

    CartesianTree tree(ar);
    std::cout << "Inorder Traversal\n";
    tree.InorderTraversal(tree.getRoot());
    std::cout << '\n';

    std::vector<int> sorted;
    tree.cartesianTreeSort(sorted);
    std::cout << "Sorted array is\n";
    for(auto x : sorted)
        std::cout << x << ' ';
    std::cout << '\n';
}


Complexidade


Uma árvore cartesiana com custos mínimos processa as seções classificadas e quase classificadas (e não importa, em ascensão ou descida) da matriz. Aqui, por exemplo, está o processo para uma matriz de ordem inversa:



A parte mais cara do algoritmo é a operação de mesclagem, que deve ser realizada duas vezes para cada um dos n elementos - primeiro ao construir o deramide e depois quando ele é desmontado. Em dados aleatórios, uma operação de mesclagem única custa O (log n ) , mas para elementos localizados em regiões ordenadas da matriz, o custo da operação O (1).

Assim, a média e a pior complexidade de tempo da classificação são O ( n log n ) e a melhor é O ( n ).

De acordo com a memória adicional, as coisas estão piores - O ( n) Se a árvore cartesiana fosse apenas uma espécie de pilha, o custo seria O (1). Mas a árvore cartesiana também é uma árvore de pesquisa binária, e é por isso que para todos os n elementos é necessário armazenar os relacionamentos pai-filho separadamente.

Além disso, a implementação neste formulário significa que é uma classificação externa - os elementos classificados são coletados em uma matriz separada. E este é O ( n ) para memória adicional.
