Nullish coalescing


, null undefined, ‘||’.


ヌリッシュ合体前：


const response = someResponse.properties.mayBeUndefined || 'Response';

ただし、属性に「false」値があるとします。


'||'の使用に関する問題：


const someResponse = {properties: { mayBeUndefined: ''}}
const response = someResponse.properties.mayBeUndefined || 'Response';

console.log(response)

結果：


これは望ましくない動作です。この場合、デフォルト値ではなく属性値が必要でした。

Nullish合体この問題はなりません。デフォルト値は、nullまたは未定義の属性に対してのみ返されます。


ヌリッシュ合体の使用：


const someResponse = {properties: { mayBeUndefined: ''}}
const response = someResponse.properties.mayBeUndefined ?? 'Response';

オプションの連鎖


サブ属性にアクセスするには、上記の属性が存在することを確認する必要があります。これまでのところ、各過剰属性の存在は手動で確認する必要があります。


オプションの連鎖の前：


const someObj = {
  property: 'prop',
  otherProperty: {
    name: 'prop2'
  }
};

const property = someObj.NotOtherProperty ? someObj.NotOtherProperty.name: undefined;
console.log(property);

オプションの連鎖の出現により、「?.」演算子を使用できます サブ属性へのオプションのアクセス用。以下のコードは、上記のコードと同等です。


オプションの連鎖の使用：


const someObj = {
  property: 'prop',
  otherProperty: {
    name: 'prop2'
  }
};

const property = someObj.NotOtherProperty?.name;
console.log(property);

特に文字列に多くの属性がある場合は、オプションのチェーンによりコードがよりきれいになります。
