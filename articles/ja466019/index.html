ABBYY Mobile Web Capture：スマートフォンのブラウザーでドキュメントの高品質の写真を直接



私たちのお客様は、携帯電話を使用して文書を撮影し、カーシェアリング会社、ビザセンター、通信事業者、金融会社などに送信することがよくあります。書類の写真は、レンタカーのレンタル、SIMカードの有効化、ローンの申し込みに十分です。しかし、スマートフォンから高品質の画像を取得するのは難しい場合があります。それでも、この問題をなんとか解決しました。

現在、iOSおよびAndroidには、ドキュメントの「モバイルスキャン」用の多くのアプリケーションがあります。しかし、携帯電話にはいくつのモバイルアプリケーションがありますか？これができないのに、なぜ時間を無駄にして新しいものをインストールするのですか？

既にスマートフォンに搭載されているモバイルブラウザで直接ドキュメントを撮影する方がはるかに簡単です。そのため、ABBYY Mobile Web Captureを作成しました。。これはJavaScript APIです。つまり、お客様のWebページやWebベースのアプリケーションに埋め込むためにお客様に提供するSDKです。最も人気のあるモバイルOSのWebブラウザーで適切な画像を直接キャプチャーし、サーバーまたはクラウドに送信してさらに処理することができます。今日は、このテクノロジーがどのように機能するかを説明します。

ABBYY Mobile Web Captureは、モバイルブラウザーからのビデオストリームで目的のドキュメントの写真を撮ることを提供します。アカウント、運転免許証、パスポート、契約書、申請書、申請書-あらゆる書類を処理できます。

新製品は、JavaScriptに転送された画像キャプチャの自動キャプチャにモバイルテクノロジーを使用しています。アルゴリズムのコアはC ++で記述されているため、WebAssemblyテクノロジーを使用してそれをWebブラウザーに転送しました。さらに、カメラの操作を担当するUIコンポーネントを作成しました。これらはJS APIに追加されたため、開発者はビデオストリームからのキャプチャをWebベースのアプリケーションまたはWebサイトに簡単かつ便利に埋め込むことができます。統合を非常に簡単にするために、サンプルWebページコードのソースコードを製品の配布に追加し、APIを正しく使用する方法を示しました。実際、開発者はこのコードを自分のWebサイトにコピーするだけでよく、たとえば、トラフィック分析のメトリックを挿入するだけです。

その後、ブラウザで、ビデオカメラの電源を入れ、ドキュメントに向けます。次に、ABBYYモバイルWebキャプチャは次のように動作します

。1.ドキュメントがフレームに入ると、ビデオストリームでその境界を見つけます。

2。次のステップは、ドキュメントの鮮明な画像を自動的に取り込むことです。これを行うには、その人物がカメラの移動を停止し、目的のドキュメントに既に「向け」ていることを確認する必要があります。私たちは電話のセンサーに依存していません。人が電話を動かさずに持っている可能性は常にありますが、ドキュメントを持っている手は動くからです。これを回避するために、画像が動いているかどうかを評価します。フレーム間のオブジェクト間のオフセットを考慮してください。最小限の場合は、キャプチャを開始できます。さらに、シャープネスも確認します。したがって、SDKは、高品質の画像を取得するためにドキュメントの写真を撮る必要がある瞬間を自動的にキャッチします。ボタンを押す必要はありません。


3.ドキュメントでフレームをキャプチャした後、フレームを境界線に合わせて揃えます



。ABBYYMobile Web Captureは適切な画像を提供します。これは、たとえばABBYY FlexiCaptureなどで認識のために送信でき、確実に処理されます。 ABBYY FlexiCaptureを使用したプロジェクトの例では、顧客がスキャナーよりもスマートフォンを使用してドキュメントを入力する方が便利であることがよくあります。しかし、このようにして得られた画像は、しばしばぼやけていることが判明します-そして、それらは単に適切に処理することができません。次に、電話から写真を送信した人に「写真を撮る」よう求められます。しかし、クライアントがこれを行うのは必ずしも便利ではありません。

ABBYY Mobile Web Captureを開発したとき、ブラウザーでドキュメントをビデオストリームで撮影するのはそれほど簡単ではないことに気付きました。まず、ドキュメントの境界の検索、フレームのオフセットとシャープネスの推定には計算リソースが必要です。ブラウザのビデオストリームが遅くならないように最適化する必要がありました。次に、iOSでは、SafariではHDよりも高い解像度のビデオを取得できないという問題に直面しました。 iOSでキャプチャした写真は、クールなXS iPhoneでさえ、あまり良くありませんでした。 10番目のフォントで入力されたA4ドキュメントを認識するには、フルHD画像以上が必要であるため、それらは単に認識されませんでした。アップルにバグレポートを書いて、Safariから高解像度でカメラを操作できるようにするように依頼しました。そして、彼らはそれをiOS 12.2で修正しました！これがなければ、当社の製品は現在のように機能しません。そして今-SDKはあなたに良い写真を与え、そしてあなたはそれらを使ってあなたがやりたいことを何でもすることができます。

もちろん、SDKを実行したときに、エンドユーザーが解決する必要があるタスクを評価しました。それらについて少し教えてください。

初めて口座を開くか、銀行からローンを借りる、保険を注文する、車やその他のサービスを借りる


あなたが銀行に来ると想像してみてください。そうではありませんが、あなたが来れば、すべてがすでに間違っています。多くの人々は銀行に行くことを嫌います。あなたはそのように考えます：「ええ、今少なくとも30分は今。」したがって、ある種の現金ローンをできるだけ早く取得する必要がある場合は、ほとんどの場合、電話とグーグルで利用可能なオプションを選択します。あなたがローンに関する情報を見つけ、サイトがオンラインでアプリケーションに記入することを提案するとします。アプリケーションではなくサイトでのみ配置するのが便利です。アプリケーションをインストールする必要がありますが、ローンを貸してくれるかどうかはまだわかりません。なぜ事前に不要なアプリケーションで電話を詰まらせるのですか？そのため、ボタンを押してから、個人データを入力する必要があります。

以前は次のようになりました。銀行は、ドキュメントをカメラに持ち込むか、完成した写真をギャラリーからアップロードするように求めています。人々はこれを行いますが、写真がぼやけたり、ドキュメント画像がまったくない場合があります-私たちのプロジェクトの経験によると、人々はしばしば混乱して間違ったファイルをアップロードします。これはすべて銀行に送られますが、画像が収まらず、すべてが無駄になっています。

一部の銀行は、この方法で問題を回避しようとしています。つまり、「即時」の認識を裏付けています。しかし、ここに別の話があります。クライアントはドキュメントを写真に撮って、バックオフィスに送って認識させました。約1分かかります。しかし、携帯電話からサービスを提供する場合、1分は非常に長い時間です。この間、おそらくすべてがスタックしていると判断し、ページを閉じてテクニカルサポートに電話するか、すべてをドロップして別のサイトにアクセスし、ローンを申請します。

ABBYY Mobile Web Captureを使用すると、クライアントは何もインストールする必要がありません。彼はビデオストリームでドキュメントを撮影します。テクノロジーが写真を処理して画像を改善し、銀行に簡単に送信できるようになりました。

政府機関、ビザセンター、銀行に送信するには、携帯電話から写真を処理する必要があります


パスポートはお持ちですか？手元にドキュメントがない場合がありますが、それを撮影し、画像がスマートフォンのギャラリーに保存されています。どこかで登録したり、航空会社のWebサイトでチケットを購入したりする必要がある場合は、常に役に立ちます。パスポートのUターンの写真の送信を求められる場合があります。写真があまり鮮明でない場合、またはカーペットに対して撮影された場合はどうなりますか？彼らはビザセンターでこれを理解していますか？おそらく、リスクを冒さない方が良いでしょう。

ABBYY Mobile Web Captureを使用してこの写真をサイトにアップロードできます。テクノロジーにより、画像内のドキュメントの境界が検出されます。境界線がない場合は、警告を出し、クライアントが何か間違ったものをダウンロードした可能性があるという事実に注意を向けます。さらに、それがさらなる認識に適しているかどうかを理解するために、ドキュメントの品質を評価しようとします。（この機能は現在テクノロジープレビューモードであるため、「試してみる」が、積極的に改善に取り組んでいます。）

不必要な苦痛なしにCRMでカードに記入する必要がある


ここに例があります：私たちは潜在的な顧客-自動車ディーラーを持っています、そして彼は彼が車を売る人々についてのすべてを知る必要があります。これを行うには、従業員はCRMシステムなどのWebポータルで顧客データを入力します。彼らはクライアントに権利を求め、それらをスキャンしてから、データをコンピューターに再入力します。従業員がモバイルアプリではなくWebポータルを使用するのはなぜですか？非常に単純です。ディーラーの主なタスクは、内部アプリケーション用に多くのコードを書くのではなく、自動車を販売して顧客にサービスを提供することです。したがって、すべてのプラットフォームで機能するソリューションを迅速に作成することが会社にとって重要です。

ABBYY Mobile Web Captureを使用すると、このビジネスプロセスを簡略化できます。従業員はスマートフォンでドキュメントの写真を撮り、認識して処理するために他の製品であるABBYY FlexiCaptureを使用して画像を送信するだけです。したがって、最終的には時間を節約し、データの品質を向上させます。

正直なところ、私自身、ブラウザでドキュメントを直接スキャンすることが非常に役立ついくつかの状況に遭遇しました。たとえば、1年前にビザを申請したとき、必要なすべての文書を撮影し、コンピューターに転送し、目的の形式で保存して、サイトにアップロードするのに約1時間かかりました。 15分ですべてを撮影できます。

すぐにABBYY Mobile Web Captureが多くのサイトで使用され、ドキュメントの写真を必要とするタスクを簡略化できるようになることを願っています。
