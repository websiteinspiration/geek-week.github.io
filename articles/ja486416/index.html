.NET Core + RabbitMQ

マイクロサービスを開発する場合、イベントが発生すると、あるサービスから他のサービスに情報を提供するという疑問が生じます。送信者を受信者にバインドしないシステムを持つことが重要です。この場合、Publisher-Subscriberパターンが使用されます。

Azure Service Bus、RabbitMQ、Apache Kafkaなど、パブリッシャー/サブスクライバーパターンをサポートする多くのメッセージング製品が市場に出ています。

最近、Azure Service BusとRabbitMQを使用してイベントベースの通信をすばやく便利にセットアップするための2つのNuGetライブラリを公開しました。この短いハウツー記事では、後者の使用手順を説明しています。

ソース


パブリッシャーは、送信者として機能する.NET Coreアプリケーションです。
サブスクライバーは、受信者として機能する.NET Coreアプリケーションです。

方法


1. パブリッシャーアプリケーションとサブスクライバーアプリケーションに、2つのNuGetライブラリをインストールします。

PM> Install-Package Autofac.Extensions.DependencyInjection
PM> Install-Package EventBus.RabbitMQ.Standard


2. パブリッシャーアプリケーションとサブスクライバーアプリケーションで、構成をappsettings.jsonに追加します。

{
  "RabbitMq": {
    "BrokerName": "test_broker",
    "AutofacScopeName": "test_autofac",
    "QueueName": "test_queue",
    "RetryCount": "5",
    "VirtualHost": "your_virtual_host",
    "Username": "your_username",
    "Password": "your_password",
    "Host": "your_host",
    "DispatchConsumersAsync": true
  }
}

これらの設定はCoudAMQPで利用できます。または、RabbitMQをローカルで使用することもできます（Dockerイメージ）。

3. パブリッシャーアプリケーションとサブスクライバーアプリケーションで、発行され、後で処理されるイベントのクラスを作成します。

public class ItemCreatedIntegrationEvent : IntegrationEvent
{
    public string Title { get; set; }
    public string Description { get; set; }

    public ItemCreatedIntegrationEvent(string title, string description)
    {
        Title = title;
        Description = description;
    }
}


4. サブスクライバーアプリケーションで、ItemCreatedIntegrationEventイベントハンドラーのクラスを作成します。

public class ItemCreatedIntegrationEventHandler : IIntegrationEventHandler<ItemCreatedIntegrationEvent>
{
    public ItemCreatedIntegrationEventHandler()
    {
    }

    public async Task Handle(ItemCreatedIntegrationEvent @event)
    {
        //Handle the ItemCreatedIntegrationEvent event here.
    }
}


5. パブリッシャーアプリケーションとサブスクライバーアプリケーションで、Program.csを更新して1行追加します。

public class Program
{
    public static void Main(string[] args)
    {
        CreateHostBuilder(args).Build().Run();
    }

    public static IHostBuilder CreateHostBuilder(string[] args) =>
        Host.CreateDefaultBuilder(args)
            .UseServiceProviderFactory(new AutofacServiceProviderFactory()) //Add this line.
            .ConfigureWebHostDefaults(webBuilder =>
            {
                webBuilder.UseStartup<Startup>();
            });
}


6. パブリッシャーアプリケーションで、Startup.csのConfigureServicesメソッドを更新します。

public class Startup
{
    public void ConfigureServices(IServiceCollection services)
    {
        ...

        var rabbitMqOptions = Configuration.GetSection("RabbitMq").Get<RabbitMqOptions>();

        services.AddRabbitMqConnection(rabbitMqOptions);
        services.AddRabbitMqRegistration(rabbitMqOptions);

        ...
    }
}


7. サブスクライバーアプリケーションで、EventBusExtension拡張機能を作成します。

public static class EventBusExtension
{
    public static IEnumerable<IIntegrationEventHandler> GetHandlers()
    {
        return new List<IIntegrationEventHandler>
        {
            new ItemCreatedIntegrationEventHandler()
        };
    }

    public static IApplicationBuilder SubscribeToEvents(this IApplicationBuilder app)
    {
        var eventBus = app.ApplicationServices.GetRequiredService<IEventBus>();

        eventBus.Subscribe<ItemCreatedIntegrationEvent, ItemCreatedIntegrationEventHandler>();

        return app;
    }
}


8. サブスクライバ・アプリケーション、更新ConfigureServices及び構成の方法でStartup.csを。

public class Startup
{
    public void ConfigureServices(IServiceCollection services)
    {
        ...

        var rabbitMqOptions = Configuration.GetSection("RabbitMq").Get<RabbitMqOptions>();

        services.AddRabbitMqConnection(rabbitMqOptions);
        services.AddRabbitMqRegistration(rabbitMqOptions);
        services.AddEventBusHandling(EventBusExtension.GetHandlers());

        ...
    }

    public void Configure(IApplicationBuilder app, IWebHostEnvironment env)
    {
        ...

        app.UseAuthorization();

        ...
    }
}


9.デモンストレーションと単純化のために、イベントがパブリッシュされるメソッドを使用してパブリッシャーアプリケーションでコントローラーを作成できます。

public class ItemController : ControllerBase
{
    private readonly IEventBus _eventBus;

    public ItemController(IEventBus eventBus)
    {
        _eventBus = eventBus;
    }

    [HttpPost]
    public IActionResult Publish()
    {
        var message = new ItemCreatedIntegrationEvent("Item title", "Item description");

        _eventBus.Publish(message);

        return Ok();
    }
}


10.これで、ItemCreatedIntegrationEventを公開できます。両方のアプリケーションを起動し、subscriberで POST Publishメソッドを呼び出して、subscriberでイベントを処理します。
