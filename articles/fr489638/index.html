Système de fichiers interplanétaire - Blog IPFS simple utilisant XSLT

Il y a un problème: un site dans IPFS n'a pas la possibilité d'utiliser des scripts côté serveur pour former une page. Si vous utilisez la génération de page avant le chargement, puis en ajoutant un nouvel élément de menu à chaque page, nous changerons le hachage de ces pages. L'ensemble du montage des pages doit donc être effectué par le navigateur.


Forme généralement le contenu des pages à l'aide de JavaScript. Il s'agit d'une technologie familière, mais elle a ses inconvénients.


J'utiliserai XSLT. Il s'agit d'une ancienne technologie de modèle intégrée depuis longtemps dans les navigateurs, mais peu l'utilisent. Peut-être parce que les modèles vous font écrire beaucoup de texte en raison de la confusion avec les espaces de noms et de nombreuses erreurs sans explication claire. De plus, malgré le fait qu'il existe déjà XSLT 3.0 dans les navigateurs, seul XSLT 1.0 est toujours disponible.


XSLT fonctionne comme ceci:


  1. Un utilisateur ouvre un document XML dans un navigateur.
  2. Dans l'en-tête XML, le document contient un lien vers le modèle XSLT.
    <?xml-stylesheet href="xslt/.xslt" type="text/xsl" ?>
  3. Un modèle dans un navigateur basé sur un document XML et d'autres données forme un document xHTML.
  4. Le navigateur affiche le document xHTML résultant.

En liant plusieurs pages à un modèle, vous pouvez modifier le document xHTML affiché sans modifier les documents XML. Ainsi, lors du changement de conception, le hachage des documents XML ne changera pas, ce qui signifie que leurs anciennes copies seront des sources pour les nouvelles dans IPFS.
