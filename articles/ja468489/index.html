HTTPリクエストの密輸-新しいアプローチ

2019年10月7日、PortSwigger Research Director（BurpSuite製造元）は、HTTPリクエストの密輸への新しいアプローチに関する研究を発表しました。彼らの助けを借りて、彼はバグバウンティで約70,000ドルを獲得しました。この記事では、攻撃の本質、ツール、およびこの脆弱性に対して脆弱なWebサーバーを調査する方法について簡単に説明します。


HTTPリクエストスマグリングとは


HTTPリクエストスマグリング-WebサーバーのフロントエンドとWebサーバーのバックエンドの同期を外すことを目的とした攻撃。その結果、攻撃者はHTTPリクエストをフロントエンドサーバーを通過して密輸できます。元の記事の写真は、優れたデモです。



このような攻撃は、他のユーザーのセッションでのXSSの導入、ユーザーをサードパーティのリソースにリダイレクトする、サーバーキャッシュを汚染する、SSRFとの類似性など、さまざまな結果を招く可能性があります。

既存のWebサーバーとプロキシをよりよく理解してトラブルシューティングするには、Dockerを使用してテスト環境をローカルにデプロイするのが賢明です。


テスト環境図の例：



アプリケーションコード：


from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/', methods=['GET', 'POST'])
def main():
    # the next line is required for Transfer-Encoding support in the request
    request.environ['wsgi.input_terminated'] = True
    headers = {}
    for header in request.headers:
        headers[header[0]] = header[1]
    print (request.data)
    print (headers)
    return jsonify(body=str(request.data), headers=headers)
