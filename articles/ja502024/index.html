Androidでのクライアントサーバーアプリケーションの宣言型プログラミング。パート2

前の記事簡単には不可欠と比較して、アンドロイドのためのクライアント・サーバ・アプリケーションの宣言型プログラミングの利点を示しました。

ここで、DeProライブラリの有効性を評価するために、小さいながら十分なプロジェクトを作成します。これは、図書館の教育事例の 1つです。説明するすべての画面のデザインを次の図に示します。

これらの画面から、それらの機能は一般的に理解されています。サイドメニューには「カタログ」と「フィットネス」の2項目があり、項目を選択すると対応する画面が表示されます。 CATALOG画面には、水平の「ニュース」リストが含まれています。 「新製品」をクリックすると、その製品の説明画面が表示されます。説明は、「説明」と「特性」の2つのタブで構成されています。また、CATALOG画面には、ドロップダウンリスト「Catalog」があります。下矢印をクリックすると、ディレクトリが開きます（閉じます）。行全体をクリックすると、PRODUCT_LIST画面が表示され、カタログで選択したアイテムの製品のリストが表示されます。

「フィットネス」をクリックすると、サービス一覧の「フィットネス」画面が表示されます。リストは、スピナーで選択したクラブによって異なります。アプリケーションを終了しようとすると、警告ダイアログが表示されます。

サーバーはjson形式のデータをアプリケーションに転送します。各画面のデータ構造を以下に示します。APIからDeProで記述されたアプリケーションでは、URLのみが使用され、バインディングは名前で実行されるため、データ構造は名前（ID）ビューを正しく設定するためにのみ必要です。データが実際のデータの不完全なスライスであることは注目に値します。したがって、名前、画像に関連がない場合があります。特に、商品画像は合計20枚です。したがって、同じ画像が多くの製品で使用される可能性があります。

次に、アプリケーションを作成するための手順を検討します。

1.スタジオで、新しいプロジェクトを作成します。名前は自由に設定できます。

2.リソースの配備。最初の記事で述べたように、リソースファイル（XML）は従来どおり使用されます。したがって、既知のものに時間を無駄にしないために、必要なすべてのリソースをダウンロードするだけです。

結果のres_example.zipファイルを解凍します。スタジオで、resフォルダーの内容全体を削除します。

解凍したresフォルダーの内容をプロジェクトのresフォルダーに転送します。その後、Android Studioの機能を考慮して、プロジェクトをクリアするか、無効なキャッシュ/再起動コマンドを実行する必要があります。

3.ライブラリの接続

モジュールのbuild.gradleファイルの依存関係セクションで、以下を指定する必要があります。

    implementation 'github.com/deprosystem/depro:compon:3.0.1'

モジュールのbuild.gradleファイルのandroidセクションで、次を指定する必要があります。

    packagingOptions {
        exclude 'META-INF/DEPENDENCIES'
    }
    compileOptions {
        sourceCompatibility JavaVersion.VERSION_1_8
        targetCompatibility JavaVersion.VERSION_1_8
    }

minSdkVersion属性を選択する場合、ライブラリがminSdkVersion = 17をサポートしていることを考慮する必要があります。build.gradleを変更した後、プロジェクトを同期する必要があります。

4.必要なクラス（ファイル）の作成。ライブラリを操作する場合、4つのクラスのみが使用されます。MyDeclareScreens-すべての画面が記述されます。MyParams-アプリケーションに必要なパラメーターを設定します。MyApp-DeProライブラリが開始されます。MainActivity-開始アクティビティ。独自のクラス名を使用できます。

クラスMyDeclareScreens.javaを作成します。その内容は次のとおりです。

public class MyDeclareScreens extends DeclareScreens {

    public final static String
            MAIN = "main", DRAWER = "DRAWER", CATALOG = "CATALOG",
            DESCRIPT = "DESCRIPT", CHARACTERISTIC = "CHARACTERISTIC",
            PRODUCT_LIST = "PRODUCT_LIST", PRODUCT_DESCRIPT = "PRODUCT_DESCRIPT",
            FITNESS = "FITNESS";

    @Override
    public void declare() {
        activity(MAIN, R.layout.activity_main)
                .navigator(finishDialog(R.string.attention, R.string.finishOk))
                .drawer(R.id.drawer, R.id.content_frame, R.id.left_drawer, null, DRAWER);

        fragment(DRAWER, R.layout.fragment_drawer)
                .menu(model(menu), view(R.id.recycler));

        fragment(CATALOG, R.layout.fragment_catalog)
                .navigator(handler(R.id.back, VH.OPEN_DRAWER))
                .component(TC.RECYCLER_HORIZONTAL, model(Api.NEWS_PROD).pagination().progress(R.id.progr),
                        view(R.id.recycler_news, R.layout.item_news_prod),
                        navigator(start(PRODUCT_DESCRIPT)))
                .component(TC.RECYCLER, model(Api.CATALOG),
                        view(R.id.recycler, "expandedLevel", new int[]{R.layout.item_catalog_type_1,
                                R.layout.item_catalog_type_2, R.layout.item_catalog_type_3})
                                .expanded(R.id.expand, R.id.expand, model(Api.CATALOG_EX, "catalog_id")),
                        navigator(start(PRODUCT_LIST)));

        activity(PRODUCT_LIST, R.layout.activity_product_list, "%1$s", "catalog_name").animate(AS.RL)
                .navigator(back(R.id.back))
                .component(TC.RECYCLER, model(Api.PRODUCT_LIST, "expandedLevel,catalog_id"),
                        view(R.id.recycler, R.layout.item_product_list)
                                .visibilityManager(visibility(R.id.bonus_i, "bonus"),
                                        visibility(R.id.gift_i, "gift"),
                                        visibility(R.id.newT, "new_product")),
                        navigator(start(PRODUCT_DESCRIPT)));

        activity(PRODUCT_DESCRIPT, R.layout.activity_product_descript, "%1$s", "catalog_name").animate(AS.RL)
                .navigator(back(R.id.back))
                .setValue(item(R.id.product_name, TS.PARAM, "product_name"))
                .component(TC.PAGER_F, view(R.id.pager, DESCRIPT, CHARACTERISTIC)
                        .setTab(R.id.tabs, R.array.descript_tab_name));

        fragment(DESCRIPT, R.layout.fragment_descript)
                .component(TC.PANEL, model(Api.PRODUCT_ID, "product_id"),
                        view(R.id.panel).visibilityManager(visibility(R.id.bonus, "bonus")))
                .component(TC.RECYCLER, model(Api.ANALOG_ID_PRODUCT,"product_id"),
                        view(R.id.recycler, R.layout.item_product_list).noDataView(R.id.not_analog)
                                .visibilityManager(visibility(R.id.bonus_i, "bonus"),
                                        visibility(R.id.gift_i, "gift"),
                                        visibility(R.id.newT, "new_product")),
                        navigator(start(0, PRODUCT_DESCRIPT, PS.RECORD),
                                handler(0, VH.BACK))) ;

        fragment(CHARACTERISTIC, R.layout.fragment_characteristic)
                .component(TC.RECYCLER, model(Api.CHARACT_ID_PRODUCT, "product_id"),
                        view(R.id.recycler, "2", new int[] {R.layout.item_property, R.layout.item_property_1}));

        fragment(FITNESS, R.layout.fragment_fitness)
                .navigator(handler(R.id.back, VH.OPEN_DRAWER))
                .component(TC.SPINNER, model(JSON, getString(R.string.clubs)),
                        view(R.id.spinner, R.layout.item_spin_drop, R.layout.item_spin_hider))
                .component(TC.RECYCLER, model(Api.FITNESS, "clubId"),
                        view(R.id.recycler, R.layout.item_fitness), null).eventFrom(R.id.spinner);
    }

    Menu menu = new Menu()
            .item(R.drawable.list, R.string.m_catalog, CATALOG, true)
            .divider()
            .item(R.drawable.ic_aura, R.string.fitness, FITNESS);
}

後で使用するすべてのDePro構造について説明します。インポートを接続するには、Alt + Enterキーを使用します。赤は、すべてのリクエストのアドレスが設定されているApiクラスも強調表示されます。内容は後ほど掲載します。

クラスMyParams.javaを作成します。ほとんどの場合、デフォルト設定で十分です。この例では、ベースURLのみを設定します。

public class MyParams extends AppParams {
    @Override
    public void setParams() {
        baseUrl = "https://deprosystem.com/";
    }
}

MainActivityクラスのスタジオによって作成されたコンテンツを次のように変更します。

public class MainActivity extends BaseActivity {
    @Override
    public String getNameScreen() {
        return MyDeclareScreens.MAIN;
    }
}

MainActivityのマニフェストで、縦向きを指定できます。原則として、ライブラリは画面の回転をサポートしますが、このバージョンでは、アクティビティ（）タイプの画面では、最初からポートレート方向も規定されています。

MyAppクラスを作成します。

public class MyApp extends Application {
    private static MyApp instance;
    private Context context;

    public static MyApp getInstance() {
        if (instance == null) {
            instance = new MyApp();
        }
        return instance;
    }

    @Override
    public void onCreate() {
        super.onCreate();
        instance = this;
        context = getApplicationContext();

        DeclareParam.build(context)
                .setAppParams(new MyParams())
                .setDeclareScreens(new MyDeclareScreens());
    }
}

これは、MyParamsとMyDeclareScreensが設定されているonCreateメソッドの通常のシングルトンです。
マニフェストでMyAppを説明することを忘れないでください。

完了するには、Apiクラスを作成します。

public class Api {
    public static final String CATALOG = "depro/cron/catalog",
            NEWS_PROD = "depro/cron/news_prod",
            PRODUCT_LIST = "depro/cron/product_list",
            PRODUCT_ID = "depro/cron/product_id",
            ANALOG_ID_PRODUCT = "depro/cron/product_analog",
            CHARACT_ID_PRODUCT = "depro/cron/product_charact",
            FITNESS = "depro/cron/fitness",
            CATALOG_EX = "depro/cron/catalog_ex";
}

5.これで、アプリケーションを実行することができます。

すべてが正しく入力されている場合、アプリケーションが起動すると、指定されたすべての画面が表示されます（そして機能します）。

ご覧のとおり、アプリケーション全体に対して5つの単純なクラスしかありません。そして、それらの4つは補助です。それらの内容は、画面の数とは無関係です。スクリーンの説明は、一般的には簡単ではありませんが、少しスペースを取ります（80行未満）。ここで、説明が大きくないだけでなく、複雑でもないことを示します。そのために、使用するライブラリコンポーネントの操作について説明します。

最初の記事で示したように、ライブラリの画面はアクティビティまたはフラグメントのいずれかであり、画面名（行）とレイアウトを設定します。画面へのリンクは名前で作成されます。使用する画面のタイプの選択は、通常の方法で実行されます。

メイン画面したがっ

て、デザインから、スタート画面にはサイドメニューとフラグメントのコンテナーがあることがわかります。 R.layout.activity_mainのマークアップでは、標準のDrawerLayoutが設定されています。したがって、MAIN画面の説明では、ドロワーコンポーネントを定義します。ドロワーコンポーネントには、DrawerLayout自体のIDと、サイドバーとフラグメントのコンテナーを渡します。サイドバーに表示される画面（DRAWER）の名前も示します。 finishDialogナビゲーターは、アプリケーションを終了する前に確認ダイアログを発行する必要があることを示します。

DRAWER画面に

は、モデル、およびメニューを表示するタイプRecyclerViewのマークアップ要素のidビューに、Menuタイプのモデルが指定されているメニューコンポーネントが1つだけ含まれています。メニュー自体は、メニューの開始時に「CATALOG」メニュー項目がフラグメントコンテナに表示されることを示しています。

CATALOG画面

水平方向の新しいリストが1つ含まれています。サーバーからデータを受信すると、彼のモデルはページネーションを使用し、R.id.progrに進行状況を表示します。インターネットが速い場合は、進行状況を示すパネルの外観に気付かない場合があります。見たい場合は、遅いインターネットに切り替えるか、R.id.progrの背景色を変更します。この場合、少なくとも点滅します。ページネーションでは、デフォルトでAppParamsで指定されたパラメーターが使用されます。ビューには、RecyclerViewとアイテムのレイアウトとともにR.id.recycler_newsが与えられます。リスト内のアイテムをクリックすると、PRODUCT_DESCRIPT画面が起動します。

ディレクトリリストはドロップダウンです。開示レベルは「expandedLevel」フィールドで定義されます。ソースデータと一緒に送信されない場合、ライブラリ自体がこれを処理します。同じパラメーターは、リストのどのレイアウトを開示の各レベルで使用するかを設定します。リストがドロップダウンであるという事実は、拡張（...）機能を提供します。モデルは、次のレベルのサーバーからデータを受信するためにcatorに設定されます。このモデルは、Api.CATALOG_EXリクエストのアドレスとリクエストパラメータ「catalog_id」の名前を指定します。展開された状態では、R.id.expandも表示されます-リストが展開されるアイテムをクリックすることによるアイテム内のマークアップ要素と、展開時にアニメーションで180度回転される要素のID-リストを閉じます。そして最後に、ナビゲーターはリスト項目をクリックすると、PRODUCT_LIST画面が呼び出されます。

画面全体を参照するナビゲーターは、R​​.id.back要素をクリックするとサイドパネルが開くことを示します。

PRODUCT_LIST画面画面を

指定すると、catalog_nameパラメーターの現在の値がヘッダーに表示され、出力アニメーションが右から左（AS.RL）になることが示されます。リストの説明はすでに私たちによく知られています。プレゼンテーションの唯一のものは、対応するデータの値に応じてマークアップ要素の可視性を制御するvisibilityManagerです。画面全体に関連するナビゲーターは、R​​.id.back要素をクリックすると、前の画面に戻ることを示しています。

PRODUCT_DESCRIPT画面

私たちにとって新しいのは、タイプPAGER_Fのコンポーネントです。通常のViewPagerに関連付けられています。彼は、表示される画面（フラグメント）のリストが表示され、TabLayout（setTab）が接続されています。

PRODUCT_DESCRIPT画面が[説明]タブに表示されます。

PANELタイプのコンポーネントは、モデルによって取得されたデータを表示します。 「アナログ」リストには類似製品のリストが表示され、存在する場合、またはnoDataView（）機能を使用すると、「アナログがありません」というメッセージが表示されます。

CHARACTERISTIC画面

製品の機能一覧を表示します。

フィットネス画面

タイプがSPINNERのコンポーネントには、クラブのリストと、選択するオプションが表示されます。クラブのリストは、モデルにjson文字列として設定されます。

クラスのカテゴリのリストは正常です。eventFrom（...）機能は、R.id.spinner（この場合はスピナー）に関連付けられているコンポーネントを変更するときに、データを更新する必要があることを示しています。
