システムのその他の機能：テレフォニーおよびCRM / ERP /ヘルプデスクの統合。何を、どのように、そしてなぜ？

どの企業でも、サービスの重要かつ有用な機能を向上させるために、成長を目指しています-マネージャーから新規顧客や多額の収入を得て、開発者は自動化システムの開発者です。

そして、それらの目標は相互に関連しています。少なくともこれらの開発のほとんどがお金を稼ぐのに役立つという事実によって-販売および/または顧客とのコミュニケーション。1通のメール、チャット、チケットでうんざりすることはないので、電話でも販売します。また、いくつかのサービス（個別にCRM、個別にテレフォニー）を使用しないようにするため、統合があります。

これを設定する方法は次のとおりです。




何のために？


統合の要求と完成した開発の数が指数関数的に増加しているため、SaaS市場はCISで毎年30％成長しています。ますます多くの企業が自動化とテクノロジーを仕事に取り入れ始めています。

これはほんの一例です
。小さな会社がプラスチック製の窓を設置しています。当初、ビジネス全体は「紙の上」でした。

クライアントは携帯電話でマネージャーに電話をかけました（またはオフィスの都市番号、それは問題ではありません）。彼は彼らの希望と詳細を注意深く聞きました。私が話したことすべてをノートに書き留めました。それから彼はこのすべての情報を渡す必要があった測定者を呼んだ。次に、測定結果、契約、追加の合意...結果として、各注文はさまざまな紙とメモの束になります。

はい、このアプローチは機能し、有益です。しかし、会社は成長しており、ヘッドはこのように続けることができないことを理解し、CRMシステムを仕事に導入しています。

今では、紙の山全体はありません。顧客の情報、注文履歴、その他の必要な情報を格納する顧客の単一のデータベースがあります。通話に関する問題を解決するためだけに残っています。そして、テレフォニーが助けになります。より正確には、システムとの統合。

これで、各通話が自動的に記録され、クライアントのカードに添付されます。入力すると、カードが自動的に開きます-従業員は誰が電話しているかすぐにわかります。通話自体が目的のマネージャーに転送され、クライアントは過去の会話の本質を説明する時間を費やす必要がなくなります。呼び出しの結果（たとえば、測定者の出発のアプリケーション）はワンクリックですぐに残すことができます。

そして、頭は月末に統計をダウンロードし、どのくらいの顧客がどのソースから適用したかを視覚的に確認できるようになります（はい、コールトラッキングについてです）。

そしてもちろん、それだけではありません。

次のことを実行できます

。システムのインターフェースからの呼び出し

一般に、統合の主な機能とタスクの1つ。CRMに保存されている各番号の横に特別なアイコン（またはボタン）を追加すると、ユーザーはワンクリックで電話をかけることができます。また、ダイヤラ付きのWeb電話を実装することをお勧めします。システムに保存されていない番号を呼び出すと便利です。
重要：仮想PBXの内部番号で、IP電話またはソフトフォンを呼び出しを受信および発信するように構成する必要があります。



また、WebRTCテクノロジーを使用して、ブラウザー、システムのページからの呼び出し用のウィジェットを間もなく提供します。これについては、別途説明します。

実装方法：GET / v1 /リクエスト/コールバック/

  • From-ユーザーに割り当てられている3桁のPBX内線番号。
  • Toは、ユーザーが呼び出したい電話番号です。


クライアントポップアップカードを使用した着信および発信の

通知ポップアップコール通知には、次の情報が示される場合があります。

  • 電話番号;
  • 連絡先の名前\会社\リード（番号が以前にCRMに保存されている場合）。
  • 電話が新しいクライアントからのものである場合-会社を作成するためのボタン\連絡先\リード;
  • 担当者の名前（既存の連絡先からの電話の場合）。
  • 着信を受信した仮想番号（Zadarmaに接続されている番号）

通知は、次のイベントの影響を受けます。
会話の開始時：NOTIFY_INTERNAL（PBX内線番号への着信呼び出しの開始）またはNOTIFY_OUT_STARTイベント（PBXからの発信呼び出しの開始）。

受信者がピックアップされ、会話が開始されると、NOTIFY_ANSWER通知がCRM（内線または外線に発信するときの応答）に送信され、ポップアップ通知のトリガーになります。



この通知を受信すると、時間カウントがリアルタイムで通話時間を表示し始める場合があります。必要に応じて、通知タイトル「Call from 442037691880」を「Conversation from 442037691880」に置き換えることができます。

通話が完了すると、NOTIFY_END通知（PBX内線番号への着信通話の終了）またはNOTIFY_OUT_END（PBXからの発信通話の終了）が送信されます。

その後、ポップアップが閉じます。

通話の統計と記録

まず、PBX内線番号の設定でクラウドへの通話の記録を有効にする必要があります。呼び出しが完了すると、NOTIFY_RECORD通知がレコーディングファイルへのリンクとともにCRMに送信されます。

新しいクライアントから電話するときに、連絡先、リード、ディールを自動的に作成します

不明な番号への着信または発信のエンティティまたはイベントを自動的に作成できます。作業の仕組みは次のとおりです。着信コール（または発信コールの開始）を受信すると、CRMデータベースで番号が確認され、番号が見つからない場合は、新しい連絡先、リード、ディール、リクエストを作成できます。

発信通話の場合、通話の開始時（NOTIFY_OUT_START通知）、通話への応答時（NOTIFY_ANSWER通知）、通話の完了時（NOTIFY_OUT_END通知）にエンティティを作成できます。呼び出し先の電話番号は、destinationパラメータで渡されます。

着信コールの場合、CRMユーザーの内部番号でコールを受信したときにエンティティを作成できます（NOTIFY_INTERNAL通知、発信者の番号がcaller_idパラメータで渡されます）

既存のクライアントから責任のあるマネージャーへの着信通話の自動転送

初めてあなたを呼び出していないクライアントの場合-音声挨拶を聞いて、適切な従業員との接続を要求する必要はありません。通話は自動的に担当の従業員に転送されます。 NOTIFY_START通知によって実装されます。

着信があったIP電話にクライアント名を表示する

このオプションを使用すると、クライアント名をCRMから、着信を受けたユーザーのIP電話またはソフトフォンに転送できます。これは、NOTIFY_START通知とその応答を使用して実装されます。通知では、caller_idパラメータで発信者の番号が送信されます。番号がすでにCRMに保存されており、クライアント/連絡先/リードに割り当てられている場合、NOTIFY_STARTへの応答として、クライアントの名前をラテン語で転送できます。

{
"caller_name": "IBM"
}

不在着信のリマインダー

クライアントからの着信時に、NOTIFY_END通知を受信すると、処理パラメーターを調べます。ステータスが応答されない場合、担当マネージャーは、クライアントからの不在着信があったことを示すタスク/リマインダー/通知を作成できます。

通話のブラックリストを使用する機能。

通話を拒否する番号のリストを作成できます。電話設定ページで番号を入力できます。または、ユーザーごとに「ブラックリスト」ボタンを作成できます。

また、NOTIFY_START通知とその応答に基づいて実装されます。（caller_idパラメータからの）発信者の番号がCRMブラックリストにある場合、NOTIFY_STARTへの応答として、以下を送信します。

{
"redirect": "blacklist"
}

拡張機能


標準の統合機能に加えて、テレフォニーをさらに活用できるようにする他の多くの機能があります。

  • 音声メニュー（IVR）とすべての着信コールのルートの完全な制御、システムから直接マルチレベルメニューを備えた秘書ロボットを構築する機能。彼らはそれについてここに書いていた。
  • SMSを受信（SMS Webhook）して送信する機能（メソッド/ v1 / sms / send /）。SMSの送受信が可能な国のリストは、サイトにあります。
  • 無料の通話追跡の接続（UTMタグを含む多くのパラメーターを持つCALL_TRACKING Webhook）。通話追跡の詳細をご覧ください。
