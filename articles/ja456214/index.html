__main__.pyを使用する

なぜ__init__.pyがされて必要な、おそらくどのpythonistは知っているが、何について__main__.py？このマジックファイルを使用しないかなりの数のプロジェクトがGithubで機能しているのを目にしましたが、それらは彼らの生活を楽にするかもしれません。私の意見では、__ main__.py は、複数のファイルで構成されるPythonモジュールと対話するための最良の方法です。


しかし、最初に正しく理解しましょう。ほとんどの人はどのようにしてPythonスクリプトを実行しますか？


インポートしたモジュールとコマンドラインから起動したツールの両方として使用するプログラムを作成したら、この場合、通常何が行われるかを知っていると思います。


if __name__ == '__main__':
    main(sys.argv)

スクリプトをインタープリターに送ると、マジックグローバル変数__name__ は値__main__を取得します。したがって、これはインポートではなく起動であることがわかります。例えば：


python myapp.py

そして、これは単一のファイルに最適です。

そして出来上がり！これで、プロジェクトを通常のモジュールとして簡単に開始できます。


python -m myapp

__main__.py は自動的に実行されます。これは、コマンドラインインターフェイスをホストし、入力引数を処理するのに最適な場所です。
