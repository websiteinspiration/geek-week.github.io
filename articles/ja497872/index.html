企業ネットワークのトラフィックとプロバイダーの作業に影響を与える異常で非常に明白な要因

企業ネットワークのトラフィックスパイク、およびITインフラストラクチャのサイバーセキュリティに関連するいくつかの興味深い依存関係について話しましょう。たとえば、従業員の職場にセカンドモニターを設置するだけでも、ネットワークの負荷が増える可能性があります。


/ Unsplash / Fotis Fotopoulos

より多くのモニター-より多くのトラフィック


ネットワーク監視ソリューションの主要プロバイダーの1つは、昨年のInfosecurity Europeカンファレンスで200人以上の専門家にインタビューして調査を実施しました。回答者の大多数は、自社のワークステーションに2台のモニターを装備して、従業員の生産性を向上させていると述べています。しかし、それらの半分は、非稼働タスク（ストリーミング音楽またはビデオ）に2番目のディスプレイを使用しています。これらのサービスは、企業トラフィックの半分以上を占める場合があります。そのような状況の73％で、企業は追加の負荷に効果的に対処できるインフラストラクチャーを持っていないことがわかりました。

Habréにあるもの：

  • ほとんどの企業ネットワークでは、ハッカーと関連する脆弱性の痕跡があるかもしれません
  • ITコミュニティが新しいIPイニシアチブにどのように対応するか

情報セキュリティの専門家の多くは、非稼働タスクにセカンドモニターを使用している従業員が、攻撃者に新しい攻撃経路を開くことにも注意しています。ストリーミングアプリケーションの脆弱性は、マルウェアが企業ネットワークに侵入する主な理由の1つです。

ほんの1か月前、テレビを見るためのアプリケーションの1つで、ハッカーがオペレーティングシステムを完全に制御できるようにするバグCVE-2020-9380を発見しました。したがって、企業は「セカンドディスプレイ」ポリシーについてもっと注意する必要があります。

長いパッチ-長いリスク管理


Kollectiveのアナリストに よると、大規模組織の半数以上（コンピューターの数が10万ユニットを超える）は、脆弱性の解決に1か月以上費やしています。この事実により、ハッカーの作業が簡略化されます。ほとんどの場合、既知のソフトウェアの脆弱性を使用して企業ネットワークをクラックし、既製のパッチのインストールを引き出す管理者の遅さのために計画を実現します。

したがって、2017年にEquifax局をハッキングするために、攻撃者はApache Strutsフレームワークの脆弱性（CVE-2017-5638）を利用しました。彼女のための「パッチ」は攻撃の2ヶ月前にリリースされました。

パッチのインストールが遅い理由の1つは、官僚主義と不十分なプロセス自動化です。多くの企業が、パッチとその更新に関する情報を通常のオフィスソフトウェアとテーブルで処理しています。Fortune 100の1つの組織には、スプレッドシートに脆弱性情報を入力する専任部門さえあります。Equifaxの場合と同様の状況を回避するために、情報セキュリティの専門家はそのようなプロセスの自動化を推奨しています。

緊急アプリケーション-緊急応答


今日、プロバイダーは45％以上のトラフィックの増加に注目しています。VPNとクラウドサービスの場合も同様です。しかし、これらすべてにより、急激な減少が見られます。


/ Unsplash / Francisco Andreotti

西部のクラウドプロバイダーの1人が興味深いパターンを指摘しました。国の指導者が声明で市民に目を向けると、それぞれの地域のネットワークの負荷が大幅に軽減されます（誰もがテレビやラジオで生のスピーチをライブで見たり聞いたりします）。

3月15日の政府控訴中に、オランダでは交通量の急激な減少が記録されました。フランスでも同様の状況が発生しました-当局が検疫措置の導入を発表したとき、国のトラフィックは約半分に減少しました。

これらすべての点は、それらのいくつかは非常に明白ですが、企業ネットワークのプロバイダーと管理者の作業は常に複雑であることを示しています。私たちは引き続きこのトピックを分析し、モルドバ最大のインターネットプロバイダーであるStarNetのネットワークアーキテクチャの更新に関する小規模なウェビナーを実施します。
