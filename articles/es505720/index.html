Spring Boot + Apache Kafka y SSL en el contenedor Docker

Hola amigos En este artículo, le diré cómo crear una solución para comenzar rápidamente su desarrollo desde una aplicación Java independiente, broker y Kookka Zookeeper, criados en contenedores Docker separados y configurados con autorización de texto sin formato para acceder desde la máquina local y el circuito externo.

El proyecto terminado en github se puede recoger al final del artículo.



Elegir una ventana acoplable Apache Kafka + Zookeeper


Primero tendrá que decidir sobre el proveedor de imágenes de Apache Kafka para nuestro contenedor. Los requisitos de cada proyecto específico varían en el nivel necesario de seguridad y confiabilidad de la solución, en algunos casos, por supuesto, tendrá que ensamblar su propia imagen, pero para la mayoría de los proyectos será aconsejable elegir uno disponible en el Docker Hub. Para 2020, hay tres imágenes principales de proveedores populares kafka + zookeper:

  • imágenes confluentes [subjetivamente, una gran ventaja: son suministradas por los desarrolladores de Apache Kafka y son las más confiables en términos de seguridad de la información]
  • imágenes de bitnami [además, en comparación con confluente, que las imágenes necesitan una configuración mínima y "comienzan desde el primer momento"]
  • wurstmeister [ , github hub.docker, ]

Docker-compose


Para iniciar y configurar imágenes, se utiliza docker-compose , lo que facilita la ejecución de aplicaciones de contenedores múltiples. No puede prescindir de él, porque la configuración mínima de kafka proporciona un montón de zookeeper y al menos un corredor.

ompose está incluido en mac / windows, la distribución Docker, por lo que la instalación del entorno necesario se reduce a la descarga e instalación de la distribución desde docs.docker.com/engine/install .

Pero los usuarios de Linux necesitan instalarlo por separado. Para obtener una guía de instalación de docker + compose, es mejor consultar la documentación oficial docs.docker.com/compose/install .

Para habilitar la autorización, debe transferir el archivo kafka_server_jaas.conf a kafka. En la carpeta con el archivo componer yaml, cree kafka_server_jaas.conf y agregue un usuario administrador con la contraseña secreta de administrador:

KafkaServer {
org.apache.kafka.common.security.plain.PlainLoginModule requerido
username = "admin"
contraseña = "admin-secret"
user_admin = "admin-secret";
};
Cliente {};

Este archivo se transferirá a la imagen con kafka broker a través de:

  1. especificando la ruta al archivo en la configuración de kafka directamente
    KAFKA_OPTS: "-Djava.security.auth.login.config=/etc/kafka/kafka_server_jaas.conf"
  2. mapeo del directorio local con el archivo en / etc / kafka el directorio de imágenes
    volumes:
    - ./:/etc/kafka

Lanzamiento de contenedores


Como resultado, deberíamos obtener 2 archivos que están en el mismo directorio:

  1. docker-compose.yml
  2. kafka_server_jaas.conf

En este directorio debe llamar al comando:

$ docker-compose up -d

El indicador -d le permite comenzar en modo separado y cerrar la consola si es necesario sin desactivar las imágenes.

Cliente Java en Spring Boot


Escribiremos un cliente minimalista con la capacidad de publicar y leer mensajes de kafka. Para hacer esto, cree un nuevo proyecto a través de Spring Initializr o cualquier otra herramienta que le sea familiar. Es suficiente para conectar una dependencia:



prefiero configurar las aplicaciones Spring Boot siempre que sea posible a través de archivos de configuración yaml con una configuración mínima en código Java. Debido a la estructuración incorporada en el formato Yaml, estos archivos son mucho más convenientes para la percepción humana que los archivos .properties y le permiten agrupar la configuración por módulos. El único inconveniente de la configuración .yml es que no es compatible con las pruebas, pero las herramientas de arranque incorporadas lo evitan fácilmente.
Recomiendo tratar de ver cómo cambia su percepción del antiguo archivo .properties (especialmente para proyectos grandes) si lo reescribe en yaml. Esto se puede hacer en un par de clics en www.toyaml.com/index.html .
