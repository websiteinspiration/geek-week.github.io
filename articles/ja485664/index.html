前書き


DevOpsのような略語は、ITコミュニティにとってもはや新しいものではありません。多くの人にとって、「make DevOps」というフレーズは、ある種の魔法のボタンに関連付けられています。クリックすると、アプリケーションコードが自動的にデプロイおよびテストされたアプリケーションになります（実際にはすべてがより複雑ですが、すべてのプロセスから抽象化されています）。


そのため、管理者がワンクリックでアプリケーションをデプロイできるように、このような魔法のボタンを作成するという注文を受けました。このタスクの実装にはさまざまなタイプがあります。メッセンジャーのボットの作成から、別のアプリケーションの開発までです。それでも、これらすべての目的は同じです。つまり、アセンブリの起動とアプリケーションの展開を最も透過的で便利にすることです。

def htmlBuild() {
    html = """
            <html>
            <head>
            <meta charset="windows-1251">
            <style type="text/css">
            div.grayTable {
            text-align: left;
            border-collapse: collapse;
            }
            .divTable.grayTable .divTableCell, .divTable.grayTable .divTableHead {
            padding: 0px 3px;
            }
            .divTable.grayTable .divTableBody .divTableCell {
            font-size: 13px;
            }
            </style>
            </head>
            <body>
        """

    def commitOptions = ""
    getCommitsForMicroservice(MICROSERVICE_NAME).each {
        commitOptions += "<option style='font-style: italic' value='COMMIT=${it.getKey()}'>${it}</option>"
    }
    html += """<p style="display: inline-block;">
        <select id="commit_id" size="1" name="value">
            ${commitOptions}
        </select></p></div>"""

    html += """
            </div>
            </div>
            </div>
            </body>
            </html>
         """
    return html
}

def getCommitsForMicroservice(microserviceRepo) {
    def commits = [:]
    def endpoint = GIT_URL + "/rest/api/1.0/projects/${PROJECT_NAME}/repos/${microserviceRepo}/commits"
    def conn = new URL(endpoint).openConnection()
    conn.setRequestProperty("Authorization", "Basic ${AUTH}")
    def response = new groovy.json.JsonSlurper().parseText(conn.content.text)
    response.values.each {
        commits.put(it.displayId, it.message)
    }
    return commits
}

return htmlBuild()
