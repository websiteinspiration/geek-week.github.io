TS-Serializable 2：継承の代わりにスネークケースとデコレータからのプロパティ変換あり

最近、私はバックエンドがphpで書かれたプロジェクトに参加できて幸運でした。そして、php jsonバックエンドの通常の回答と同様に、それらはヘビのケーススタイルで送信します。その結果、前面のデータを扱うすべての作業が、キャメルケースとヘビケースのスタイルで混同されていました。この問題を解決するために、ts-serializableシリアライゼーションライブラリが改善されました。これで、jsonからデータを受信すると、jsで受け入れられるラクダのケーススタイルに変換し、サーバーに送信すると、ヘビのケースに戻すことができます。



また、最初のバージョンがリリースされた後、一部のユーザーは機能性を望んでいました。これらの願いは、新しいバージョンで実装されています。

ts-serializable ライブラリに慣れていない場合は、まずこの記事を読むことをお勧めします。

プロパティ名の変換


Newtonsoft.Jsonと同様に、シリアル化設定が追加されました。グローバルに、1つのオブジェクトに、または1つの逆シリアル化呼び出しに設定できます。

スネークケースからキャメルケースにデシリアライズされ、シリアライズされるとスネークケースに戻るクラスの例。

import { Serializable, SnakeCaseNamingStrategy, jsonObject, jsonProperty, jsonName } from "ts-serializable";

const json = {
    first_name: "Jack",
    last_name: "Sparrow",
    date_of_birth: "1690-05-05T21:29:43.000Z",
    "very::strange::json:name": "I love jewelry"
};

@jsonObject({ namingStrategy: new SnakeCaseNamingStrategy() })
class User extends Serializable {

    @jsonProperty(String, null)
    public firstName: string | null = null;

    @jsonProperty(String, null)
    public lastName: string | null = null;

    @jsonProperty(Date, null)
    public dateOfBirth: Date | null = null;

    @jsonName("very::strange::json:name")
    @jsonProperty(String, null)
    public veryStrangePropertyName: string | null = null;

}

const user = new User().fromJSON(json);

user.firstName === json.first_name; // true
user.lastName === json.last_name; // true
user.dateOfBirth?.toISOString() === json.date_of_birth; // true
user.veryStrangePropertyName === json["very::strange::json:name"]; // true

ここでは、シリアル化と逆シリアル化の設定がjsonObjectデコレーターに渡されます。また、すぐに使えるのはKebabCaseNamingStrategyとPascalCaseNamingStrategyです。これで十分でない場合は、独自のプロパティ名コンバーターを作成してパラメーターに渡すことができます。

また、jsonNameデコレーターにも注目してください。これは、非常に具体的なプロパティ名で使用できます。

設定を指定


設定はさまざまな場所で設定できます。

  • 世界的に。設定は、オブジェクトまたはメソッドで再定義されていない場合、すべてのオブジェクトに適用されます。
  • オブジェクトごと。メソッドでオーバーライドされていない場合、設定は1つのオブジェクトに適用されます。
  • 1つの方法。設定は1回の逆シリアル化呼び出しに適用され、直列化では機能しません。

//  
Serializable.defaultSettings: SerializationSettings = { ...options };

//   
@jsonObject(settings?: Partial<SerializationSettings>)
class User extends Serializable { ...code }

//    
new User().fromJSON(json: object, settings?: Partial<SerializationSettings>);

継承の代わりにデコレーター


ライブラリの最初のバージョンがリリースされた後、一部のユーザーは、継承が別のオブジェクトによって既に取得されているという事実に問題を抱えていました。モンキーパッチを実装することはお勧めできません。jsでベースオブジェクトを拡張すると、反応ライブラリが壊れます。

しかし、解決策が見つかりました。これで、2番目のパラメーターとしてtrueパラメーターをjsonObjectデコレーターに渡すことにより、オブジェクトを拡張できます。


@jsonObject(void 0, true) //  
class User {

    public fromJSON!: (json: object) => this; //  typescript

    @jsonProperty(String, null)
    public firstName: string | null = null;

    @jsonProperty(String, null)
    public lastName: string | null = null;

}
