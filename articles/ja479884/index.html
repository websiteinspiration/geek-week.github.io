UnityのカスタムCI / CD


次に、小さなゲーム開発会社でのUnityの開発プロセスと、それを改善および自動化する方法について説明します。それでも、2020年は間近で、マウスで運転するのに十分です...

マウスに関しては、これはそのような冗談ではありません。最近まで、私たちと一緒にゲームをテストおよび開発するプロセスは、次のようなものでした。

アンドロイド


  1. 開発者がコードを書く
  2. Androidバージョンへの移行
  3. Googleドライブの共有フォルダへのアップロード
  4. テスターは最新のビルドをダウンロードします
  5. Redmineでタスクをテストしてスローする
  6. 後藤：ステップ1

iOSバージョンは少しトリッキーです：

iOS


  1. 開発者がコードを書く
  2. Xcodeプロジェクトに行く
  3. Postbuildスクリプトはローカリゼーション、SDKなどを追加します
  4. プロジェクトをコンパイルしています
  5. アーカイブ済み
  6. IPAファイルにエクスポートされます（これはすぐには完了しません、ハハ）
  7. その後、IPAをGoogleドライブの共有フォルダーにアップロードできます。
  8. または、数日待つことができるTestFlightにアップロードします。
  9. テスターが最新のIPAをダウンロード
  10. デバイスを接続し、アプリケーションをインストールします
  11. Redmineでタスクをテストしてスローする
  12. 後藤：ステップ1

ここにそのような悲しみがあります。そして、Androidの場合、プロセスはまだなんらかの形で受け入れ可能（いいえ）ですが、iOSの場合、すべてが非常に長く、不便で間違っています。さらに、ほとんどの操作は開発者が手動で（ここではマウスについて）行われ、開発に費やされる可能性のある時間は、コンパイル、Xcodeプロジェクトのエクスポートなどを待つのにかかります。また、全員が同じオフィスにいることを考慮に入れる必要があります。この地獄のプロセス全体を開始する前に、何かを尋ねたり言ったりすることができます。

しかし、一般的に、それは何とか機能し、ゲームをリリースします:)

そしてここで、最近、出願からレオポタム、自転車の建設とすべての最適化の別の愛好家である私は、このプロセスを自動化することにしました。結果は、次のことができるBASHスクリプトです。

  • GITリポジトリから最新の変更をダウンロードする
  • テストを実行する
  • Android向けAPKを開発、リリースします
  • iOS Xcode , develop release
  • IPA
  • HTML
  • sshpass

そして、これらすべてをボタンに触れるだけで行えます。UnityとXcodeを起動せずに。どういうわけかもっと楽しいでしょ？



これで、アセンブリが完了すると、テスターはAndroidおよびiOSバージョンへの2つのリンクを含む通知をTelegramで受け取ります。それらを通過し、デバイスから直接アプリケーションをインストールする必要があります。iOSでも動作し、特別なマニフェストが生成されます。

実際、これはBASHプログラミングでの最初の経験なので、風水であるかどうかはわかりませんが、正しく動作します。いずれの場合でも、スクリプトはプロジェクトのニーズに合わせて更新および変更されます。以下は、実行されていることに関するいくつかの重要なポイントです。

特定のプラットフォーム用のUnityプロジェクトの構築を開始する方法


このマニュアルでは、ビルドに使用できるすべての引数について説明しています。ビルドAPKを実行します。

$UNITY -batchmode -quit -projectPath "$PATH" -executeMethod Game.BuildActions.AndroidDevelopment -buildTarget android -logFile "$LOGS_PATH/android_development.log"

ここで-buildTargetだけでプラットフォームを設定し、 - executeMethodが指定したパラメータでビルドを開始しユニティプロジェクト、関数を呼び出し、それは次のようになります。

static void AndroidDevelopment () {
	PlayerSettings.SetScriptingBackend (BuildTargetGroup.Android, ScriptingImplementation.IL2CPP);
	PlayerSettings.SetScriptingDefineSymbolsForGroup (BuildTargetGroup.Android, "DEV");
	EditorUserBuildSettings.SwitchActiveBuildTarget (BuildTargetGroup.Android, BuildTarget.Android);
	EditorUserBuildSettings.development = true;
	EditorUserBuildSettings.androidETC2Fallback = AndroidETC2Fallback.Quality32Bit;
	BuildReport report = BuildPipeline.BuildPlayer (GetScenes (), ANDROID_DEVELOPMENT_FILE, BuildTarget.Android, BuildOptions.None);
	int code = (report.summary.result == BuildResult.Succeeded) ? 0 : 1;
	EditorApplication.Exit (code);   
}

ここでは、ビルド、条件などのパラメーターを設定できます。結果はBASHスクリプトに返されます。

Xcodeプロジェクトをコンパイルおよびエクスポートする方法


UnityがXcodeプロジェクトを正常に発行した後、それをコンパイル、アーカイブ、およびIPAファイルにエクスポートする必要があります。これは次のように行われます。

xcodebuild -project "$IOS_PATH/Unity-iPhone.xcodeproj" -quiet > "$LOGS_PATH/ios_build_release.log" 2>&1

xcodebuild -project "$IOS_PATH/Unity-iPhone.xcodeproj" -scheme "Unity-iPhone" archive -archivePath "$IOS_RELEASE/Unity-iPhone.xcarchive" -quiet > "$LOGS_PATH/ios_archive_release.log" 2>&1


xcodebuild -exportArchive -archivePath "$IOS_RELEASE/Unity-iPhone.xcarchive" -exportOptionsPlist "$IOS_RELEASE/options.plist" -exportPath $IOS_RELEASE -allowProvisioningUpdates -quiet > "$LOGS_PATH/ios_export_release.log" 2>&1


ここで、options.plistは、エクスポート方法、TeamIDなどを示す特別なマニフェストです。使用可能なすべてのパラメーターは、次のコマンドで表示できます。

xcodebuild -help

これらすべての操作により、Unityが生成する標準のUnity-iPhone.xcodeprojが指定されます。xcworkspaceがある場合は、その可用性と使用状況を確認する必要があります。

BASHスクリプトから電報にメッセージを送信する方法


BotFather ボットを見つけ、彼に/ start、または/ newbotと書き込み、フィールドに入力して、トークンとドキュメントへのリンクを含むメッセージを受信する必要があります。ボットを使用してメッセージを送信するには、次のコマンドを実行する必要があります。

curl $BOT_PROXY https://api.telegram.org/bot$BOT_TOKEN/sendMessage -m 60 -s -X POST -d chat_id=$CHAT_ID -d text="$1" > "$LOGS_PATH/bot.log" 2>&1

ここで、CHAT_IDはメッセージを送信するチャットIDです。たとえば、テスターが座っているグループにボットを追加して、メッセージを読み取る権限を与えることができます。その後、実行します：

https://api.telegram.org/bot[BOT_TOKEN]/getUpdates

そして、何かを書いたグループIDとメンバーを取得します。その後、送信するグループまたは特定のメンバーのIDを指定します。プロキシ経由の送信を見つけて設定する方法は問題ないと思います。

その他の関数、sedを使用してマニフェストテンプレートとHTMLにパッチを適用し、sshpassを使用して必要なファイルをサーバーにアップロードします。
