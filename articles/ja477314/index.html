ViewModelからMVVMのアクティビティ/フラグメントへのイベントの送信

今日は、MVVMでアクティビティ/フラグメントとViewModelの間でイベントを交換する方法について説明します。ViewModelからデータを取得するには、アクティビティ/フラグメントで、ViewModelにあるLiveDataデータをサブスクライブすることをお勧めします。しかし、通知を表示したり、たとえば、別のフラグメントを開いたりするなど、単一の（だけでなく）イベントを送信するにはどうすればよいでしょうか。



みなさん、こんにちは！

私の名前はアレックスです。ホームクレジットバンクのAndroid開発者です。

この記事では、ViewModelsからViews（アクティビティ/フラグメント）へのイベントの送受信方法を紹介します。

私たちのアプリケーション「Home Credit Bankからの割賦商品」では、フラグメントを使用しているため、それらについて説明しますが、すべてがアクティビティにも関連しています。

私達何が欲しいの？


いくつかのViewModelを含むFragmentがあり、データはDataBinding-th によってバインドされています。すべてのユーザーイベントはViewModelを受け取ります。イベントがナビゲーションの場合：あなたは、別のフラグメント/アクティビティを開いて示さなければならないAlertDialog、Snackbarなどのシステム要求権限、そしてこのイベントは、フラグメント内で実行する必要があります。

そして実際、何が問題なのでしょうか？


ビューとViewModelのライフサイクルは無関係です。ViewModelには、フラグメントについて何も知らないはずなので、リスナーにバインドすることは不可能である、とあなたが知っているようにもメモリが「が開始されます、それ以外の場合は、フラグメントへのリンクを含めることはできません喜びます」。

FragmentsとViewModel間の相互作用の標準的なアプローチは、ViewModelにあるをサブスクライブすることLiveDataです。LiveDataこの方法では、イベントが既に完了しているかどうかが考慮されないため、を介してイベントを直接送信することはできません。

存在するソリューション：


1. 長所を使用：イベントは一度だけ実行されます。短所： 1つのイベント-1つ。多数のイベントがあると、N個のイベントオブジェクトがViewModelに表示され、それぞれがフラグメントでサブスクライブされる必要があります。 2.良い例。長所： 1つのイベントも1回実行され、viewModelからフラグメントにデータを転送できます。短所：イベント内のデータが必要ですが、データなしでイベントを実行する必要がある場合SingleLiveEvent

SingleLiveEvent



(val content: T)、別のクラスを作成する必要があります。1つのタイプのイベントを1回実行する問題は解決しません（イベント自体は1回実行されますが、このタイプのイベントは、ViewModelから実行した回数だけ実行されます）。たとえば、Nリクエストは非同期で私たちを離れますが、ネットワークはありません。各リクエストはネットワークエラーで返され、ネットワークエラーに関するN個のイベントのフラグメントを取得します。フラグメントではN個のアラートが開きます。ユーザーはそのような決定を承認しません:)。このエラーのメッセージを一度表示する必要があります。つまり、このタイプのイベントは1回実行する必要があります。

決定


基本としてSingleLiveEvent、イベント処理に関する情報を保存するという考えを採用しています。

可能なイベントタイプを定義する


enum class Type {
    EXECUTE_WITHOUT_LIMITS, //    –   ,            
    EXECUTE_ONCE, //     
    WAIT_OBSERVER_IF_NEEDED,//    ,     -      
    WAIT_OBSERVER_IF_NEEDED_AND_EXECUTE_ONCE //    ,     -          
}

基本イベントクラスを作成する- NavigationEvent


isHandled イベントが受信されたかどうかを示します（フラグメントでObserverが受信された場合に実行されたと考えられます）。

open class NavigationEvent(var isHandled: Boolean = false, var type: Events.Type)

エミッタークラスを作成する- Emitter


イベントエミッタクラスはLiveData< NavigationEvent>を継承します。このクラスは、ViewModelでイベントをディスパッチするために使用されます。

class Emitter : MutableLiveData<NavigationEvent>() {
    private val waitingEvents: ArrayList<NavigationEvent> = ArrayList()
    private var isActive = false

    override fun onInactive() {
        isActive = false
    }

    override fun onActive() {
        isActive = true
        val postingEvents = ArrayList<NavigationEvent>()
        waitingEvents
            .forEach {
                if (hasObservers()) {
                    this.value = it
                    postingEvents.add(it)
                }
            }.also { waitingEvents.removeAll(postingEvents) }
    }

    private fun newEvent(event: NavigationEvent, type: Type) {
        event.type = type
        this.value = when (type) {
            Type.EXECUTE_WITHOUT_LIMITS,
            Type.EXECUTE_ONCE -> if (hasObservers()) event else null

            Type.WAIT_OBSERVER_IF_NEEDED,
            Type.WAIT_OBSERVER_IF_NEEDED_AND_EXECUTE_ONCE -> {
                if (hasObservers() && isActive) event
                else {
                    waitingEvents.add(event)
                    null
                }
            }
        }
    }

    /** Clear All Waiting Events */
    fun clearWaitingEvents() =  waitingEvents.clear()
}

isActive- Emitter少なくとも1つに登録されているかどうかを理解する必要がありObserverます。そして、サブスクライバーが現れ、彼を待っているイベントが蓄積した場合、これらのイベントを送信します。重要な説明：イベントはからthis.postValue(event)ではなく、setterから送信する必要がありますthis.value = event。それ以外の場合、サブスクライバーはリストの最後のイベントのみを受け取ります。

新しいイベント自体を送信するメソッドは、newEvent(event, type)2つのパラメーターを受け取ります。実際には、イベント自体とこのイベントのタイプです。

すべてのタイプのイベント（ロングネーム）を覚えないようにするために、イベント自体のみを受け入れる個別のパブリックメソッドを作成します。

class Emitter : MutableLiveData<NavigationEvent>() {
    …

    /** Default: Emit Event for Execution */
    fun emitAndExecute(event: NavigationEvent) = newEvent(event, Type.EXECUTE_WITHOUT_LIMITS)

    /** Emit Event for Execution Once */
    fun emitAndExecuteOnce(event: NavigationEvent) = newEvent(event, Type.EXECUTE_ONCE)

    /** Wait Observer Available and Emit Event for Execution */
    fun waitAndExecute(event: NavigationEvent) = newEvent(event, Type.WAIT_OBSERVER_IF_NEEDED)

    /** Wait Observer Available and Emit Event for Execution Once */
    fun waitAndExecuteOnce(event: NavigationEvent) = newEvent(event, Type.WAIT_OBSERVER_IF_NEEDED_AND_EXECUTE_ONCE)
}

正式にはEmitter、ViewModelにサブスクライブして、その処理に関係なく（イベントがすでに処理されているかどうかに関係なく）イベントを受け取ることができます。

イベントオブザーバークラスを作成する- EventObserver


class EventObserver(private val handlerBlock: (NavigationEvent) -> Unit) : Observer<NavigationEvent> {
    private val executedEvents: HashSet<String> = hashSetOf()
    
    /** Clear All Executed Events */
    fun clearExecutedEvents() =  executedEvents.clear()
    
    override fun onChanged(it: NavigationEvent?) {
        when (it?.type) {
            Type.EXECUTE_WITHOUT_LIMITS,
            Type.WAIT_OBSERVER_IF_NEEDED -> {
                if (!it.isHandled) {
                    it.isHandled = true
                    it.apply(handlerBlock)
                }
            }
            Type.EXECUTE_ONCE,
            Type.WAIT_OBSERVER_IF_NEEDED_AND_EXECUTE_ONCE -> {
                if (it.javaClass.simpleName !in executedEvents) {
                    if (!it.isHandled) {
                        it.isHandled = true
                        executedEvents.add(it.javaClass.simpleName)
                        it.apply(handlerBlock)
                    }
                }
            }
        }
    }
}

このオブザーバーは、高次関数を入力として受け入れます。イベント処理はフラグメントで記述されます（以下の例）。完了したイベント（一度実行する必要があったもの）をクリアする

方法clearExecutedEvents()。画面の更新時に必要です（例：c）swipeToRefresh()。

実際、onChange()このオブザーバーがサブスクライブする、新しいエミッターデータが受信されたときに発生するメインメソッド。

イベントの実行タイプが無制限である場合は、イベントが実行されたかどうかを確認して処理します。イベントを実行し、受信して処理したことを示します。

if (!it.isHandled) {
    it.isHandled = true
    it.apply(handlerBlock)
}

イベントが1回実行する必要があるタイプの場合は、このイベントのクラスがハッシュテーブルにあるかどうかを確認します。そうでない場合は、イベントを実行し、このイベントのクラスをハッシュテーブルに追加します。

if (it.javaClass.simpleName !in executedEvents) {
    if (!it.isHandled) {
        it.isHandled = true
        executedEvents.add(it.javaClass.simpleName)
        it.apply(handlerBlock)
    }
}

このため、MyFragmentNavigationから継承されNavigationEvent()たクラスで構成されるインターフェースが作成されます。さまざまなクラスが、パラメーターを渡した場合と渡さない場合で以下に作成されます。

interface MyFragmentNavigation {
    class ShowCategoryList : NavigationEvent()
    class OpenProduct(val productId: String, val productName: String) : NavigationEvent()
    class PlayVideo(val url: String) : NavigationEvent()
    class ShowNetworkError : NavigationEvent()
}

class MyViewModel : ViewModel() {
    val emitter = Events.Enitter()
