Ising Modell


Das Ising-Modell wurde eingeführt, um die Natur des Ferromagnetismus zu verstehen und die Untersuchung von Phasenübergängen und kritischen Phänomenen zu beeinflussen. Ferromagnetismus beschreibt das Auftreten einer spontanen Magnetisierung in Ferromagneten unterhalb einer bestimmten Temperatur - dem Curie-Punkt. Am Curie-Punkt (einem engen Temperaturbereich) erfolgt die Ordnung, in diesem Fall die Ausrichtung magnetischer Momente, was einen Phasenübergang zur Folge hat, dh die Eigenschaften der Substanz ändern sich abrupt.

Das Modell wurde ursprünglich 1920 von V. Lenz vorgeschlagen und von Ising für einen eindimensionalen Fall untersucht, zu dessen Ehren es seinen Namen erhielt. In diesem Modell erhält jeder Scheitelpunkt des Kristallgitters eine Zahl: +1 oder -1, - Feld "hoch" oder "runter". Die Zahl, die den Eckpunkten entspricht, wird als Spin bezeichnet. Somit kann sich ein Gitter von N Spins in 2 N Zuständen befinden. Jeder Zustand entspricht der Energie, die aus der paarweisen Wechselwirkung der Spins benachbarter Atome erhalten wird:



Wobei J die Wechselwirkungsenergie benachbarter Spins ist (die Austauschwechselwirkungskonstante ist für alle Paare gleich), sind s i und s j Spins.

Darüber hinaus beschreibt J> 0 das Verhalten eines Ferromagneten, J <0 ist ein Antiferromagnet.

Monte-Carlo-Methode


Wie bereits erwähnt, sind insgesamt 2 N Zustände möglich, und für eine ausreichend große Anzahl von N Spins ist es schwierig, numerische Ergebnisse zu erhalten. Zum Beispiel erhalten wir für N = 10 2 10 Zustände, die nicht so einfach direkt zu modellieren sind, daher wird ein statistischer Ansatz zur Modellierung verwendet.

Bei diesem Ansatz wird das System in einem Zustand des thermodynamischen Gleichgewichts bei einer bestimmten Temperatur T betrachtet. Während des Energieaustauschs mit der Umgebung ändert sich die Energie in der Nähe des Gleichgewichtszustands, und die durchschnittliche Energie eines Teilchens ist proportional zu T. Die Implementierung einer konstanten zufälligen Änderung um den Gleichgewichtszustand verwendet die Monte-Carlo-Methode und Die Modellierung kann in Stufen unterteilt werden:

  1. Spielen Sie die Zustände α i des Systems (zum Beispiel zufällig) mit einem festen T;
  2. Für diese Zustände sind die thermodynamischen Eigenschaften nahe dem Gleichgewicht (Energie E, Magnetisierung M) zu berücksichtigen;
  3. Durchschnitt der erhaltenen Werte


Metropolis-Algorithmus.


In den Problemen der statistischen Mechanik sind die Ausdrücke „Monte-Carlo-Methode“ und „Metropolis-Stichprobenmethode“ fast Synonyme. Wir geben die allgemeinste Form des Metropolis-Algorithmus als Beispiel für ein System von Spins oder Partikeln:

  1. Wir bilden die Erstkonfiguration.
  2. Wir nehmen eine zufällige Teständerung in der Erstkonfiguration vor. Zum Beispiel wählen wir zufällig einen Spin aus und versuchen ihn umzuwerfen. Oder wählen Sie ein zufälliges Partikel aus und versuchen Sie, es auf eine zufällige Entfernung zu verschieben.
  3. Wir berechnen ∆, dh die Änderung der Energie des Systems aufgrund der Änderung der Testkonfiguration.
  4. Wenn ∆ kleiner oder gleich Null ist, akzeptieren wir die neue Konfiguration und fahren mit Schritt 8 fort.
  5. Wenn Δ positiv ist, berechnen wir die "Übergangswahrscheinlichkeit":

  6. Wir erzeugen eine Zufallszahl rnd im Intervall (0, 1)
  7. Wenn rnd ≤ W, akzeptieren wir die neue Konfiguration, andernfalls speichern Sie die vorherige Konfiguration.
  8. Wir bestimmen die Werte der erforderlichen physikalischen Größen.
  9. Wiederholen Sie die Schritte 2 bis 8, um genügend Konfigurationen oder „Tests“ zu erhalten.
  10. Wir berechnen Durchschnittswerte über Konfigurationen, die statistisch unabhängig voneinander sind.

Zweidimensionales Ising-Modell in C ++


Wir werden unser eigenes Programm schreiben, das das zweidimensionale Ising-Modell unter Verwendung der Monte-Carlo-Methode und des Metropolis-Algorithmus implementiert. Wir implementieren die folgenden Funktionen:

  • Der Benutzer legt die Größe der Seite des SIZE-Gitters fest, die maximal zulässige Temperatur T_MAX.
  • ISING SIZE×SIZE, , (0), , , (1), .
  • T ISING. , W S ( ). , – 0,01℃.
  • – . T=2,269℃ .
  • Eine Konfigurationsänderung erfolgt alle 10 ms (Millisekunden) direkt auf dem Bildschirm. Mit jeder Konfigurationsänderung erhöht sich die Anzahl der Monte-Carlo-Schritte.
  • Der Benutzer kann die Temperatur vor jeder neuen Änderung der Konfiguration des Spin-Gitters ändern.
  • Alle 10 s (Sekunden) werden Daten an die Konsole ausgegeben (Akzeptanzkoeffizient, durchschnittliche Energie pro Spin, durchschnittliches Energiequadrat pro Spin, durchschnittliche Magnetisierung, durchschnittliches Magnetisierungsquadrat).

#include <iostream>
#include <random>
#include <locale>
#include "glut.h"

using namespace std;

#define SIZE 256
#define SIZE_PX 800
#define T_MAX 4
#define CURIE_SCALE 1000. 

double quadSize = SIZE_PX / (double)SIZE;

short lattice[SIZE][SIZE];
double w[5];
double T = T_MAX, M, E;

int ratio = 0;
size_t nmcs = 0;
double ecum = 0, e2cum = 0, mcum = 0, m2cum = 0;

void display()
{
	glClear(GL_COLOR_BUFFER_BIT);
	
	glBegin(GL_QUADS);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			glColor3f(lattice[i][j], lattice[i][j], lattice[i][j]);
			glVertex2d(i, j);
			glVertex2d(i, j + 1);
			glVertex2d(i + 1, j + 1);
			glVertex2d(i + 1, j);
		}
	glEnd();

	glBegin(GL_QUADS);
	{
		glColor3f(1, 0, 0);
		glVertex2d(SIZE, 0);
		glVertex2d(SIZE, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, 0);

		glColor3f(1, 1, 1);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
	}
	glEnd();

	glutSwapBuffers();
	glFlush();
}

void calcW()
{
	double e4 = exp(-4 / T), e8 = e4 * e4;
	w[0] = w[4] = e8;
	w[1] = w[3] = e4;
	w[2] = 0;
}

void init()
{
	M = E = 0;
	static std::default_random_engine generator;
	std::uniform_int_distribution<int> distribution(0, RAND_MAX);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			lattice[i][j] = ((distribution(generator) / (double)RAND_MAX >= 0.5) - 1) * 2 + 1;
			M += lattice[i][j];
		}

	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			E += (i + 1 != SIZE) ? lattice[i][j] * lattice[i + 1][j] : 0;
			E += (j + 1 != SIZE) ? lattice[i][j] * lattice[i][j + 1] : 0;
		}

	calcW();
}

void metropolis()
{
	int x, y, sum;
	for (int i = 0; i < SIZE * SIZE; i++)
	{
		x = rand() % SIZE;
		y = rand() % SIZE;
		sum = lattice[(x - 1 + SIZE) % SIZE][y] +
			lattice[(x + 1 + SIZE) % SIZE][y] +
			lattice[x][(y - 1 + SIZE) % SIZE] +
			lattice[x][(y + 1 + SIZE) % SIZE];

		if (sum * lattice[x][y] <= 0 || (rand() / (double)RAND_MAX) < w[sum / 2 + 2])
		{
			lattice[x][y] = -lattice[x][y];
			::ratio++;
			M += 2 * lattice[x][y];
			E -= 2 * lattice[x][y] * sum;
		}
	}
}

void timer1(int)
{
	display();

	metropolis();

	nmcs++;

	ecum += E;
	e2cum += E * E;
	mcum += M;
	m2cum += M * M;


	glutTimerFunc(10, timer1, 0);
}

void outputData(int)
{
	double norm = 1 / (double)(nmcs * SIZE * SIZE);

	cout << "  = " << ::ratio * norm << endl;
	cout << "    = " << ecum * norm << endl;
	cout << "     = " << e2cum * norm << endl;
	cout << "  = " << mcum * norm << endl;
	cout << "   = " << m2cum * norm << endl << endl;

	glutTimerFunc(10000, outputData, 0);
}

void keyboard(unsigned char key, int x, int y)
{
	switch (key)
	{
	case 'w': case 'W':
	{
		T += 0.01;
		T = (T >= T_MAX) ? T_MAX : T;
		calcW();
		break;
	}
	case 's': case 'S':
	{
		T -= 0.01;
		T = (T <= 0.01) ? 0.01 : T;
		calcW();
		break;
	}
	}
}

void motion(int x, int y)
{
	double k = 1 - y / (double)SIZE_PX;
	T = T_MAX * k;
	T = (T >= T_MAX) ? T_MAX : T;
	T = (T <= 0.01) ? 0.01 : T;
	calcW();
}

int main()
{
	setlocale(LC_ALL, "ru");
	init();
	glutInitDisplayMode(GLUT_SINGLE | GLUT_RGB);
	glutInitWindowSize(SIZE_PX + 100, SIZE_PX);
	glutCreateWindow("ISING");
	glMatrixMode(GL_PROJECTION);
	glLoadIdentity();
	gluOrtho2D(0, SIZE + SIZE / 8, 0, SIZE);
	glutDisplayFunc(display);
	glutKeyboardFunc(keyboard);
	glutMotionFunc(motion);
	glutTimerFunc(1, timer1, 0);
	glutTimerFunc(10000, outputData, 0);

	glutMainLoop();
}

Beachten Sie den Zweck des Erstellens einiger Funktionen in diesem Softwareprodukt:

  • init () - Erstkonfiguration erstellen;
  • display () - Darstellung des Drehgitters und der Temperaturskala;
  • calcW () - Berechnung der Übergangswahrscheinlichkeiten;
  • metropolis () - Implementierung des Metropolis-Algorithmus;
  • timer1 () - Aktualisieren der Daten nach jeder Konfigurationsänderung, wodurch die Anzahl der Monte-Carlo-Schritte erhöht wird;
  • outputData () - Berechnung des Durchschnitts pro Drehung, Ausgabe der erforderlichen Daten;
  • Tastatur () und Bewegung () - die Möglichkeit, die Temperatur mithilfe der Skala auf zwei Arten zu ändern - Tasten auf der Tastatur bzw. Mausklick;

Es lohnt sich, ein paar Schlussfolgerungen zu ziehen:

  • Bei einer kritischen Temperatur T = 2,269 ° C tritt ein Phasenübergang von einem ungeordneten in einen geordneten ferromagnetischen Zustand auf.
  • Wenn sich die Temperatur nach unten ändert, ändern sich die folgenden physikalischen und statistischen Eigenschaften wie folgt:
    • Die Akzeptanzrate wird reduziert;
    • Die durchschnittliche Energie pro Drehung nimmt ab;
    • Das durchschnittliche Energiequadrat pro Drehung nimmt zu;
    • Die durchschnittliche Magnetisierung nimmt ab;
    • Das durchschnittliche Quadrat der Magnetisierung nimmt zu.
