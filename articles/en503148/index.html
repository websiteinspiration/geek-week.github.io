Checking Linux Security with Lynis Utility

We figure out how to use one of the most powerful freeware utilities for auditing Linux systems.


Have you ever wondered how secure your Linux system is? There are many Linux distributions (each with its own default settings) on which you run dozens of software packages with different version numbers. In addition, on your OS in the background there are many services that you sometimes do not know or think about.

To get information about the system, that is, check the general status of the software, network and services running on your computer with Linux installed, just run a few commands. But now what to do with the information received? How to understand it? After all, the amount of data that you need to analyze is huge.

It would be much better if you could just run a tool that generates a structured report on the level of security of the system. Lynis is a popular open source tool that audits Linux and Unix based systems.

‚ÄúIt performs a comprehensive scan of security settings and runs on the system itself. The main goal is to test security features and issue recommendations for further improving the level of system security. It also scans system-wide information, information about installed packages, and possible configuration errors. Lynis [usually] is used by system administrators and security experts to assess the level of system security, ‚Äù the project‚Äôs website says.

Install Lynis


You can use package managers to install Lynis. Provided that your Linux system with the relevance of the versions in the repository is in order, the utility can be installed like this:

dnf install lynis

or so:

apt install lynis

However, if the utility version in your repository is not the latest, it is better to install Lynis with GitHub (I use the Red Hat Linux system, but similar commands work on any Linux distribution):

Launch Lynis


Let's get started with Lynis by running help using the -h flag :

$ ./lynis -h

So you can familiarize yourself with the main utility commands. For example, let's find out its version:

$ ./lynis show version
3.0.0
$

For a complete list of Lynis commands, type:

$ ./lynis show commands

Commands:
lynis audit
lynis configure
lynis generate
lynis show
lynis update
lynis upload-only

$

Running an audit of a Linux system


It all starts with the following command:

$ ./lynis audit system

After starting the team we will receive a detailed report. Do not be alarmed, we will figure it out a bit later. Other team work results are also saved, so you can return to them later.

Here Lynis stores the logs:

  Files:
  - Test and debug information      : /var/log/lynis.log
  - Report data                     : /var/log/lynis-report.dat

You can check if all the necessary files were actually created:

$ ls -l /var/log/lynis.log
-rw-r-----. 1 root root 341489 Apr 30 05:52 /var/log/lynis.log
$
$ ls -l /var/log/lynis-report.dat

$

At the end of the report, Lynis makes corrections based on the audit results. Each sentence begins with a description, and the test ( TEST-ID ) that generated it is indicated in parentheses next to it. The next line suggests a solution to the problem, if it exists:

Suggestions (47):
  ----------------------------
  * If not required, consider explicit disabling of core dump in /etc/security/limits.conf file [KRNL-5820]
      https://cisofy.com/lynis/controls/KRNL-5820/

  * Check PAM configuration, add rounds if applicable and expire passwords to encrypt with new values [AUTH-9229]
      https://cisofy.com/lynis/controls/AUTH-9229/

Lynis provides the ability to find additional information about each correction proposal using the show details command , followed by the TEST-ID test number :

./lynis show details TEST-ID

For example, to find out more information for the SSH-7408 test, I enter $ ./lynis show details SSH-7408 and get:
