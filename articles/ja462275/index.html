PVS-Studio Static Analyzer for Javaの紹介

長年にわたり、PVS-Studioチームは同じ名前のスタティックアナライザーを開発してきました。現時点では、アナライザーは複雑なソフトウェアソリューションであり、Windows、Linux、およびmacOSプラットフォームでC、C ++、C＃、Javaなどのプログラミング言語を分析する機能を提供します。最近では、サポートされている多くのプログラミング言語にJavaが追加されています。PVS-StudioアナライザーがC ++およびC＃開発者の間で長年確立されている場合、PVS-StudioはJavaの初心者です。多くの人はアナライザーについてさえ聞いていませんでした、そしてそれを聞いた人は誰でもその機能のすべてについてほとんど知りませんでした。したがって、この記事では、PVS-Studio Javaを紹介し、起動メソッドとその機能について説明します。



一般情報


PVS-Studio Java静的コードアナライザーは、分析を実行するカーネルと、アセンブリシステム（Gradle、Maven）およびIDE（IntelliJ IDEA）に統合するためのプラグインの2つの主要部分で構成されています。

プラグインはプロジェクト構造（ソースファイルとクラスパスのセット）を受け取り、その後この情報をカーネルに転送します。さらに、プラグインはカーネルを分析用にデプロイする責任があります-初回起動時に自動的にダウンロードされます。

ソースとクラスパスをリストすることにより、アナライザーを直接実行することもできます。

コンピュータが次のシステム要件を満たしている場合、分析を実行できます。

  • オペレーティングシステム：Windows、Linux、macOS;
  • アナライザーを実行するJavaの最小バージョン：Java 8（64ビット）。注：分析されたプロジェクトは、任意のバージョンのJavaを使用できます。
  • IntelliJ IDEAの最小バージョンは2017.2です。

Maven用プラグイン


作業中のプロジェクトがMavenビルドシステムに基づいている場合は、pvsstudio-maven-pluginプラグインを使用できます。これを行うには、プロジェクトのpom.xmlファイルに以下を追加します。

<pluginRepositories>
  <pluginRepository>
    <id>pvsstudio-maven-repo</id>
    <url>http://files.viva64.com/java/pvsstudio-maven-repository/</url>
  </pluginRepository>
</pluginRepositories>

<build>
  <plugins>
    <plugin>
      <groupId>com.pvsstudio</groupId>
      <artifactId>pvsstudio-maven-plugin</artifactId>
      <version>{VERSION_PVS_JAVA}</version>
      <configuration>
        <analyzer>
          <outputType>text</outputType>
          <outputFile>path/to/output.txt</outputFile>
          ....
        </analyzer>
      </configuration>
    </plugin>
  </plugins>
</build>

分析を開始する前に、ライセンスデータを入力する必要があります。

mvn pvsstudio:pvsCredentials "-Dpvsstudio.username=USR" "-Dpvsstudio.serial=KEY"

その後、ライセンス情報は、Windowsでは％APPDATA％/ PVS-Studio-Java / PVS-Studio.licに、macOSおよびLinuxでは〜/ .config / PVS-Studio-Java / PVS-Studio.licに保存されます。

その後、分析を実行できます。

$ mvn pvsstudio:pvsAnalyze

<analyzer> ブロックでは、アナライザーを構成できます。すべての設定のリストはここにあります。

Gradleのプラグイン


作業中のプロジェクトがGradleビルドシステムに基づいている場合は、pvsstudio-gradle-pluginプラグインを使用できます。これを行うには、プロジェクトのbuild.gradleファイルに以下を追加します。

buildscript {
  repositories {
    mavenCentral()
    maven {
      url uri('http://files.viva64.com/java/pvsstudio-maven-repository/')
    }
  }
  dependencies {
    classpath group: 'com.pvsstudio',
              name: 'pvsstudio-gradle-plugin',
              version: '{VERSION_PVS_JAVA}'
  }
}

apply plugin: com.pvsstudio.PvsStudioGradlePlugin
pvsstudio {
  outputType = 'text'
  outputFile = 'path/to/output.txt'
  ....
}

分析を開始する前に、ライセンスデータを入力する必要があります。

./gradlew pvsCredentials "-Ppvsstudio.username=USR" "-Ppvsstudio.serial=KEY"

その後、ライセンス情報は、Mavenプラグインとともに、Windows OSの％APPDATA％/ PVS-Studio-Java / PVS-Studio.licまたは〜/ .config / PVS-Studio-Java / PVS-に保存されます。 macOSおよびLinux上のStudio.lic。

その後、分析を実行できます。

$ ./gradlew pvsAnalyze

pvsstudioブロックでは、アナライザーを構成できます。すべての設定のリストはここにあります。

IntelliJ IDEAのプラグイン


PVS-Studio Javaアナライザーは、IntelliJ IDEAのプラグインとしても使用できます。この場合、プロジェクト構造の分析はこのIDEを使用して実行され、プラグインはアナライザーを操作するための便利なグラフィカルインターフェイスを提供します。

IDEA用のPVS-Studioプラグインは、次の場所からインストールできます。

  • プラグインのJetBrains公式リポジトリ。
  • viva64.comのリポジトリから。
  • ダウンロードページから入手できるPVS-Studio for Windowsインストーラー経由。

プラグインをインストールしたら、ライセンスデータを入力する必要があります

。1）分析→PVS-Studio→設定


2）登録タブ：


これで、現在のプロジェクトの分析を開始できます。


ライセンシング


PVS-Studioアナライザーは、開発チームに焦点を当てており、本質的に独自のB2B製品です。アナライザーの機能を理解するために、試用キーをリクエストできます。

オープンソースプロジェクトを開発している場合、または学生などである場合は、 PVS-Studioの無料ライセンスオプションの 1つを使用できます。

誤った警告の抑制


誤検知に対抗するために、アナライザーは一連の異なるメカニズムを提供し

ます。1.コードで特別なコメントを使用します。

void f() {
    int x = 01000; //-V6061
}

2. 抑制ファイルの使用。

3. @SuppressWarningsアノテーション（....）

の使用アナライザーは一部のアノテーションを認識しており、すでにマークアップされているコードに対して警告を生成しない場合があります。例えば：

@SuppressWarnings("OctalInteger")
void f() {
    int x = 01000;
}
