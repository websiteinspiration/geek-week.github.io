攻撃からMikrotikを保護する簡単な方法

外部からの攻撃からネットワークと「見張っている」ネットワークとサービスを保護するためにMikrotikを使用する方法をシンプルで実用的な方法でコミュニティと共有したいと思います。つまり、わずか3つのルールで、ミクロティックにハニーポットを編成します。

したがって、従業員がリモートで作業するための小さなオフィスがあり、その背後にRDPサーバーである外部IPがあるとします。最初のルールはもちろん、外部インターフェイスのポート3389を別のポートに変更することです。しかし、これは長くはありません。数日後、ターミナルサーバーの監査ログには、不明なクライアントからの1秒あたりの失敗した承認がいくつか示され始めます。

別の状況では、もちろん5060 udpポートではなくMikrotikの後ろにアスタリスクが隠されており、2、3日後にパスワードの推測も開始されます...はいはい、わかっています。 4月18日、驚きの状態でfail2banに同じUbuntuディストリビューションの同じボックスからのアスタリスクの実際の設定が含まれていないことに気付きましたが、Googleは既製の「レシピ」のクイック設定をグーグルできません。古いバージョンのレシピは機能しなくなりましたが、新しいレシピはほとんどありません...

つまり、ハニーポットとは簡単に言うと、おとり、つまり私たちの場合、外部IPのいくつかの人気のあるポートであり、外部クライアントからこのポートへの要求があれば、srcアドレスがブラックリストに送信されます。すべて。

/ip firewall filter
add action=add-src-to-address-list address-list="Honeypot Hacker" \
    address-list-timeout=30d0h0m chain=input comment="block honeypot ssh rdp winbox" \
    connection-state=new dst-port=22,3389,8291 in-interface=\
    ether4-wan protocol=tcp
add action=add-src-to-address-list address-list="Honeypot Hacker" \
    address-list-timeout=30d0h0m chain=input comment=\
    "block honeypot asterisk" connection-state=new dst-port=5060 \
    in-interface=ether4-wan protocol=udp 
/ip firewall raw
add action=drop chain=prerouting in-interface=ether4-wan src-address-list=\
    "Honeypot Hacker"

外部ether4-wanインターフェースの人気のあるTCPポート22、3389、8291の最初のルールは、ゲストIPをハニーポットハッカーリストに送信します（ssh、rdp、およびwinboxのポートは事前に無効化されているか、他のポートに変更されています）。 2番目は、人気のあるUDP 5060で同じことを行い

ます。事前ルーティング段階の3番目のルールは、srs-addressが「Honeypot Hacker」に入った「ゲスト」のパケットをドロップします。

私の家のMikrotikの2週間の作業の後、Honeypot Hackerリストには、私のネットワークリソース（家庭の電話、メール、nextcloud、rdp）の乳房に優しいファンのIPアドレスが約1万5千個含まれていました。

職場では、すべてがそれほど単純であるとは限らず、ブルートフォースパスワードでrdpサーバーを破壊し続けています。

明らかに、ポート番号はハニーポットがオンになるずっと前にスキャナーによって決定され、検疫中に100人を超えるユーザーを再構成することはそれほど容易ではなく、そのうち20％は65歳以上です。ポートを変更できない場合は、小さな作業レシピがあります。私はインターネットで同様に会ったが、完成した微調整があります：

4分以内に、リモートクライアントはRDPサーバーに12個の新しい「要求」のみを行うことができます。1回のログイン試行は、1〜4回の「リクエスト」です。12番目の「リクエスト」-15分間のロック。私の場合、攻撃者はサーバーのハッキングを止めず、タイマーに適応しましたが、現在は非常にゆっくりと行っています。このような選択速度により、攻撃効率はゼロになります。企業の従業員は、実際に、行われた作業で不便を経験することはありません。

別の小さなトリック
このルールは、午前1時にスケジュールで有効になり、ライブの人々が正確に寝ている5時にオフになり、自動ピックアップが起きたままになります。

/ip firewall filter 
add action=add-src-to-address-list address-list=rdp_blacklist \
    address-list-timeout=1w0d0h0m chain=forward comment=\
    "night_rdp_blacklist" connection-state=new disabled=\
    yes dst-port=3389 protocol=tcp src-address-list=rdp_stage8

すでに8番目のアカウントで、攻撃者のIPは1週間ブラックリストに登録されています。美しさ！
