Flutter y aplicaciones de escritorio

No es ningún secreto que el equipo de desarrollo de Flutter se esfuerza por garantizar que este marco permita, utilizando una sola base de código, crear aplicaciones para una amplia variedad de plataformas. Esto incluye iOS, Android, Windows, Linux, macOS y la web. Además, tales aplicaciones deben compilarse en un formato específico para cada plataforma, y ​​su apariencia no debe ser inferior a la apariencia de los juegos bien hechos. En Google, el marco Flutter se usa en muchos proyectos: desde Assistant a Stadia , desde Cloud Search a Blogger . Otras compañías usan Flutter, desde ByteDance hasta Grab , desde Nubank hasta MGM Resorts.. Todos ellos se benefician de la flexibilidad y productividad del desarrollo que ofrece Flutter.



Muchos de ustedes están interesados ​​en desarrollar las capacidades de escritorio de Flutter, que incluyen Windows, macOS y Linux. En las encuestas y en GitHub, el desarrollo de escritorio es una de las nuevas características más notables de Flutter. En el futuro cercano vamos a hablar más sobre lo que hacemos. Creemos que vale la pena comenzar con una revisión de lo que han hecho los diferentes equipos responsables de las capacidades de Flutter. Aunque el soporte para aplicaciones de escritorio todavía está en la etapa de versión técnica de prueba, se está trabajando seriamente en las capacidades correspondientes.









Ensamblaje de aplicación



Recientemente lanzamos perfiles de aplicaciones y modos de compilación para Windows y Linux , además de la compatibilidad existente con macOS. Por ejemplo, si usa la última versión de Flutter, las aplicaciones de Flutter se pueden compilar en ejecutables de Windows con el comando flutter build windows



. Para hacer esto, utilizamos nuestro compilador de nivel de producción AOT , que le permite crear código de máquina para la arquitectura x64, que puede ejecutarse en las computadoras correspondientes sin instalar Flutter.



Características de las aplicaciones de escritorio.



Ya sea que esté desarrollando una aplicación representada por un archivo ejecutable independiente o una aplicación web, las aplicaciones de escritorio tienen algunas características y características específicas. Por ejemplo, las ventanas de aplicaciones de escritorio generalmente se muestran en modo horizontal, y su tamaño se puede cambiar. Los usuarios suelen ingresar datos en las aplicaciones utilizando un teclado y un mouse, en lugar de teclados en pantalla y entrada táctil. Los controles se optimizan teniendo en cuenta que las aplicaciones de escritorio utilizan valores especiales de densidad visual al colocar controles.

Actualización del modelo de complemento



Flutter está diseñado para tener un núcleo compacto. Para equipar el marco con características adicionales, la funcionalidad correspondiente se realiza en forma de complementos y paquetes que permiten la integración con los sistemas operativos en los que se ejecuta Flutter. Los complementos y paquetes pueden ser creados no solo por los desarrolladores de Flutter, sino también por otros programadores.



Sin embargo, dado el hecho de que Flutter está mejorando el soporte para plataformas móviles y de escritorio, así como para aplicaciones web, desarrollar complementos para cada plataforma soportada por el marco se está volviendo cada vez más difícil. Al desarrollar un determinado complemento, es muy probable que sea necesario combinar los esfuerzos de diferentes programadores, cada uno de los cuales está bien versado en alguna plataforma.



Aquí, un enfoque puede ayudarnos, según el cual un complemento básico específico describe una interfaz común que se puede implementar de forma independiente en diferentes plataformas. Como resultado, como se describió en el material más reciente sobre el desarrollo moderno de complementos, recientemente implementamos un sistema en Flutter que simplifica la colaboración de varios autores en complementos. Gracias a este sistema, ya es posible indicar explícitamente qué plataformas son compatibles con un complemento en particular.



Comenzamos a usar este modelo cuando trabajamos en algunos complementos básicos. Aquí se pueden encontrar ejemplos de un nuevo enfoque para el desarrollo colaborativo de complementos .



Tenga en cuenta que la API de complementos para Windows y Linux aún no se ha establecido, por lo tanto, aunque recomendamos a los desarrolladores que se familiaricen con las nuevas funciones de Flutter, aún no estamos listos para lanzar estas características en producción. También estamos trabajando para agregar etiquetas de plataforma de escritorio a pub.dev.



Ejecución de programas en Windows: Win32 y UWP



Un aspecto interesante de nuestro trabajo relacionado con Windows es que aquí estamos experimentando con diferentes enfoques de arquitectura. Flutter, en cualquier plataforma, está integrado en una pequeña aplicación de host que es un contenedor ("integrador") utilizando un enfoque similar al utilizado en motores de juegos como Unity. Esta aplicación de inserción, específica para cada plataforma, nos proporciona un punto de entrada al proyecto. Coordina sus acciones con el sistema operativo, trabajando con herramientas de renderizado, con aportes del usuario y con herramientas que aseguran la disponibilidad de contenido. También es compatible con el bucle de eventos.









Windows nos brinda dos formas de crear dicha aplicación. El primero es un modelo de programación Win32 maduro que se puede utilizar como punto de entrada para proyectos de Flutter. Este enfoque ofrece el más alto nivel de compatibilidad con varias plataformas, como Windows 7. Le permite crear archivos EXE estándar que muchos programadores esperan recibir como resultado de trabajar en un proyecto. A diferencia de Win32, el modelo de aplicación UWP moderno se recomienda para Windows 10. Este modelo ofrece características atractivas para crear aplicaciones Flutter para dispositivos especiales, como Xbox y Windows 10X , que saldrá pronto.



Extraoficialmente, trabajamos con varios desarrolladores e investigamos diferentes soluciones. Nos encantaría trabajar más estrechamente con Microsoft para mejorar la calidad de las aplicaciones de Windows hechas con Flutter. Dado que la familia de dispositivos Surface ahora incluye sistemas basados ​​en Android y Windows, creemos que Flutter puede ofrecer a Microsoft una plataforma poderosa que le permite crear atractivas aplicaciones nativas para una amplia variedad de dispositivos.



Experimentos de aplicaciones de escritorio



Dado que las características que se discutieron aún se encuentran en la etapa de versión técnica de prueba, la API y las herramientas auxiliares aún son inestables. Hemos planeado muchas más cosas que deben completarse antes de transferir el soporte para aplicaciones de escritorio a la categoría de características estables. Esto incluye, además, mejorar la accesibilidad y localización del contenido.



Si ya desea experimentar nuevas oportunidades, entonces necesita usar el canal de desarrollo. El soporte para Windows y Linux está disponible en la sucursal master



donde Flutter se está desarrollando activamente. El soporte para MacOS está disponible en la sucursal dev



. Hay características más estables que, sin embargo, no se recomiendan para su uso en producción. Puede cambiar entre canales utilizando los comandos flutter channel master



oflutter channel dev



. Después de eso, debe ejecutar uno de los siguientes comandos para habilitar el soporte para la plataforma que le interesa: 



C:\flutter> flutter config --enable-windows-desktop
$ flutter config --enable-macos-desktop
$ flutter config --enable-linux-desktop
