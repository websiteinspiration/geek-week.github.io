Azúcar de serpiente o escribe tu rango en JavaScript

Mucha gente ama Python ... Los principiantes son admirados por la ausencia de punto y coma, y ​​los avanzados disfrutan de una simplicidad real. Hoy hablaremos sobre cómo implementar en JavaScript una apariencia de la simpleza de Python, y específicamente la función de rango .

En Python, la función de rango puede iterarse o, por ejemplo, convertirse en una lista de matriz (rango (inicio, fin)) .

Pero la pregunta es, ¿es posible que JavaScript cree algo similar y al mismo tiempo haga que la solución parezca nativa y simple?

Lo primero que viene a la mente es escribir una clase similar:

function range(from, to, step = 1){
    this.current = from
    this.to = to
    this.step = step

    this.next = () => this.current++ % to
}

Sí, escribí que a continuación se devuelve el valor recortado, pero este también es un problema que debe resolverse cómo resolverlo: ¿arrojar un error al llegar al máximo o hacer otra cosa?

En general, dicha solución no agregará conveniencia en absoluto y será más fácil escribir un estándar para , por ejemplo, puede agregar la función final y, después de cada iteración, verificar si nos hemos ido al extranjero, pero no tiene sentido desarrollar esta solución.

ES6 presenta un nuevo tipo de datos primitivo, Symbol. Él abre ante nosotros las puertas a la metaprogramación.

Para resolver nuestro problema con el rango usaremos Symbol.iterator . Permite crear un objeto iterador utilizando un protocolo específico.

Para que un objeto se convierta en un iterador, debe definir la función [Symbol.iterator] () en él , pero dado que queremos crear una función común, no una clase, tendremos una función que devuelve un objeto iterador.

Así es como se ve:

function range(from, to, step = 1){
    return {
        [Symbol.iterator](){
            return {
                current: from,
                to: to,
                from: from,
                step,
                next(){
                    const it = { done: this.current >= this.to, value: this.current }
                    this.current += this.step
                    return it
                }
            }
        }
    }
}

¿Qué esta pasando? Tenemos una función de rango , devuelve, de todos modos , nuestro objeto iterador, en el que la función [Symbol.iterator] () devuelve un objeto. Lo más importante es que este objeto contiene la siguiente función, de lo contrario , el objeto no será iterable y aparecerá un error al intentar usarlo. La siguiente función debería (!) Devuelve un objeto con las propiedades done y value , donde done (bool) señala el final de la iteración, y el valor contiene el valor actual.

En principio, todo funciona, y ya puedes escribir algo parecido a Python:

    for(let i of range(0, 10, 2))
        console.log(i)

Conclusión:

0
2
4
6
8

Podríamos parar aquí ... Pero puedes escribir una solución más elegante usando las funciones del generador:

function range(from, to, step = 1){
    return {
        *[Symbol.iterator](){
            for(let val = from; val < to; val += step){
                yield val;
            }
        }
    }
}

Una ventaja adicional será el hecho de que tenemos otras oportunidades para los iteradores. Por ejemplo, puede convertir fácilmente nuestro rango a una matriz:

[...range(0, 10, 2)]
// [ 0, 2, 4, 6, 8 ]
