プッシャーを使用してLaravelにリアルタイム通知を追加する方法

特に「Framework Laravel」コースの学生向けに記事の翻訳が用意されました。



最近のWebユーザーは、アプリケーションで発生するすべてのことについて知らされることを期待しています。通知のドロップダウンリストさえもないWebサイトになりたくないでしょう。これは現在、すべてのソーシャルネットワーキングサイトだけでなく、一般に最近はどこでも見られるようになっています。

幸い、LaravelとPusherを使用すると、この機能の実装は非常に簡単です。

リアルタイム通知


快適なユーザーエクスペリエンスを提供するには、通知をリアルタイムで表示する必要があります。1つのアプローチは、AJAXリクエストを定期的にサーバーに送信し、最新の通知がある場合はそれを受信することです。

最善の方法は、WebSocketの機能を使用して、送信時に通知を受け取ることです。これはまさにこの記事で実装するものです。

プッシャー


プッシャーは、WebSocketsを介してリアルタイムの双方向機能をWebおよびモバイルアプリケーションに統合するためのWebサービスです。

非常にシンプルなAPIを備えていますが、Laravel BroadcastingとLaravel Echoでさらに簡単に使用できるようにします。

この記事では、既存のブログにリアルタイム通知を追加します。

ユーザー登録


ユーザーに互いにサブスクライブする機会を提供したいので、これを実現するには、ユーザー間に多対多の関係を作成する必要があります。

ユーザーをユーザーにリンクするピボットテーブルを作成してみましょう。新しいマイグレーションを作ってみましょうfollowers：

この移行にいくつかのフィールドを追加する必要があります。サブスクライブしているユーザーを表すためuser_idと、follows_idサブスクライブしているユーザーを表すフィールドです。

public function up()

        $table->integer('user_id')->index();

// ...

class extends Authenticatable
{
    // ...

    {
        $this->follows()->attach($userId);
        return $this;
    }

    {
        $this->follows()->detach($userId);
        return $this;

    public function isFollowing($userId) 
    {
        return (boolean) $this->follows()->where('follows_id', $userId)->first(['id']);

}

お知らせ


Laravelは、複数のチャネルに通知を送信するためのAPIを提供します。電子メール、SMS、Web通知、およびその他の種類の通知は、通知クラスを使用して送信できます。

2種類の通知があります。

  • サブスクリプション通知：別のユーザーがサブスクライブしたときにユーザーに送信されます
  • 投稿通知：新しい投稿が公開されたときに、このユーザーのサブスクライバーに送信されます。

サブスクリプション通知


職人コマンドを使用して、通知の移行を生成できます。

php artisan notifications:table

移行を行って、このテーブルを作成しましょう。

php artisan migrate

まず、サブスクリプション通知から始めます。このコマンドを実行して、通知クラスを作成しましょう。

php artisan make:notification UserFollowed

次に、作成した通知クラスファイルを変更します。

class UserFollowed extends Notification implements ShouldQueue
{
    use Queueable;

    protected $follower;

    public function __construct(User $follower)
    {
        $this->follower = $follower;
    }

    public function via($notifiable)
    {
        return ['database'];
    }

    public function toDatabase($notifiable)
    {
        return [
            'follower_id' => $this->follower->id,
            'follower_name' => $this->follower->name,
        ];
    }
}

これらの数行のコードで、すでに多くのことを達成できます。まず、$followerこの通知が生成されたときにインスタンスを実装する必要があります。

このメソッドを使用しviaて、チャネルを通じてこの通知を送信するようにLaravelに指示しますdatabase。 Laravelはこれを検出すると、通知テーブルに新しいエントリを作成します。

user_idそして、type通知が自動的に設定され、加えて、我々は、追加のデータとの通知を拡張することができます。それが目的toDatabaseです。返された配列はdata通知フィールドに追加されます。

そして最後に、実装のおかげでShouldQueue、Laravelはこの通知をバックグラウンドで実行されるキューに自動的に配置し、応答を高速化します。これは、後でPusherを使用するときにHTTP呼び出しを追加するため、理にかなっています。

ユーザー登録通知を実装しましょう。

UserモデルはすでにNotifiableトレイトを使用しているため、notifyメソッドを呼び出すことができます。

通知するモデルは、それを使用して、通知メソッドにアクセスする必要があります。

通知に既読のマークを付けます。

通知には、情報とリソースへのリンクが含まれます。たとえば、ユーザーが新しいメッセージに関する通知を受け取った場合、通知には有益なテキストが含まれ、押されたときにユーザーをメッセージにリダイレクトし、既読としてマークする必要があります。

リクエストにオカレンスがあるかどうかをチェックし、?read=notification_idそれを既読としてマークするレイヤーを作成します。

次のコマンドでこのレイヤーを作成しましょう：

php artisan make:middleware MarkNotificationAsRead

次に、このコードをhandle中間メソッドに配置します。

class MarkNotificationAsRead
{
    public function handle($request, Closure $next)
    {
        if($request->has('read')) {
            $notification = $request->user()->notifications()->where('id', $request->read)->first();
            if($notification) {
                $notification->markAsRead();
            }
        }
        return $next($request);
    }
}

リクエストごとにレイヤーを実行するために、レイヤーをに追加し$middlewareGroupsます。

//...
class Kernel extends HttpKernel
{
    //...
    protected $middlewareGroups = [
        'web' => [
            //...
            \App\Http\Middleware\MarkNotificationAsRead::class,
        ],
        // ...
    ];
    //...
}

その後、通知を表示させましょう。

新しい投稿の通知


ユーザーが新しい投稿を公開したときに購読者に通知します。

通知クラスを作成することから始めましょう。

php artisan make:notification NewPost

生成されたクラスを次のように変更してみましょう。

// ..
use App\Post;
use App\User;
class NewArticle extends Notification implements ShouldQueue
{
    // ..
    protected $following;
    protected $post;
    public function __construct(User $following, Post $post)
    {
        $this->following = $following;
        $this->post = $post;
    }
    public function via($notifiable)
    {
        return ['database'];
    }
    public function toDatabase($notifiable)
    {
        return [
            'following_id' => $this->following->id,
            'following_name' => $this->following->name,
            'post_id' => $this->post->id,
        ];
    }
}

次に、通知を送信する必要があります。これを行うにはいくつかの方法があります。

Eloquentウォッチャーを使用するのが好きです。

Postのオブザーバーを作成して、そのイベントをリッスンしましょう。新しいクラスを作成します。app/Observers/PostObserver.php

namespace App\Observers;
use App\Notifications\NewPost;
use App\Post;
class PostObserver
{
    public function created(Post $post)
    {
        $user = $post->user;
        foreach ($user->followers as $follower) {
            $follower->notify(new NewPost($user, $post));
        }
    }
}

次に、オブザーバーをに登録しAppServiceProviderます。

//...
use App\Observers\PostObserver;
use App\Post;
class AppServiceProvider extends ServiceProvider
{
    //...
    public function boot()
    {
        Post::observe(PostObserver::class);
    }
    //...
}

次に、JSで表示するメッセージをフォーマットするだけです。

// ...

    follow: 'App\\Notifications\\UserFollowed',

    var to = `?read=${notification.id}`;

結論


プッシャーは非常にユーザーフレンドリーなAPIを備えており、リアルタイムイベントの実装を非常に簡単にしています。Laravel通知と組み合わせて、1つの場所から複数のチャネル（メール、SMS、Slackなど）を介して通知を送信できます。このガイドでは、シンプルなブログでユーザーアクティビティを追跡する機能を追加し、上記のツールを使用して改善し、スムーズなリアルタイム機能を実現しました。

PusherとLaravelには、さらに多くの通知があります。タンデムでは、サービスにより、ブラウザ、携帯電話、IOTデバイスにパブ/サブメッセージをリアルタイムで送信できます。オンライン/オフラインのユーザーステータスを取得するためのAPIもあります。

ドキュメントをご覧ください（Pusherのドキュメント、プッシャーチュートリアル、Laravel docs）を使用して、その使用法と真の可能性について詳しく学習します。
