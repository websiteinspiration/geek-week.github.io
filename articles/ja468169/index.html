jQueryの歴史と遺産


jQueryは、世界で最も人気のある JavaScriptライブラリです。Web開発者のコ​​ミュニティが2000年代後半にそれを作成し、その結果、jQueryを内部で使用するサイト、プラグイン、フレームワークの豊富なエコシステムが登場しました。

しかし、近年、ウェブ開発の主要ツールとしての地位が揺らいでいます。jQueryが人気になった理由と時代遅れになった理由を確認してみましょう。また、jQueryを使用して最新のサイトを作成することをお勧めします。

jQueryの簡単な歴史


John Resig（John Resig）は、ライブラリの最初のバージョンを2005年に作成し、BarCampNYCというイベントで2006年に公開しました。上の公式jQueryのウェブサイト、著者は書いています：

jQueryは、モットーに基づいたJavascriptライブラリです。Javascriptでプログラミングするとよいでしょう。jQueryは頻繁に繰り返されるタスクを取り、すべての不要なマークアップを抽出し、それらを短く、エレガントで、わかりやすくします。

jQueryには2つの主な利点があります。1つ目は、Webページを操作するための便利なAPIです。特に、アイテムを選択するための強力な方法を提供します。jQueryでは、IDやクラスだけでなく、複雑な式を記述して、他の要素との関係に基づいて要素を選択することもできます。

// Select every item within the list of people within the contacts element
$('#contacts ul.people li');

時間の経過とともに、選択エンジンは別のシズルライブラリに進化しました。

ライブラリの2番目の利点は、ブラウザ間の違いを抽象化したことです。これらの年には、すべてのブラウザーで確実に機能するコードを作成することが困難でした。

標準化の欠如は、開発者がブラウザーと境界ケースの多くの違いを考慮する必要があることを意味しました。見てみましょう。この初期のjQueryのソースコードをしてjQuery.browserを探します。以下はその一例です。

// If Mozilla is used
if ( jQuery.browser == "mozilla" || jQuery.browser == "opera" ) {
        // Use the handy event callback
        jQuery.event.add( document, "DOMContentLoaded", jQuery.ready );

// If IE is used, use the excellent hack by Matthias Miller
// http://www.outofhanwell.com/blog/index.php?title=the_window_onload_problem_revisited
} else if ( jQuery.browser == "msie" ) {

        // Only works if you document.write() it
        document.write("<scr" + "ipt id=__ie_init defer=true " + 
                "src=javascript:void(0)><\/script>");

        // Use the defer script hack
        var script = document.getElementById("__ie_init");
        script.onreadystatechange = function() {
                if ( this.readyState == "complete" )
                        jQuery.ready();
        };

        // Clear from memory
        script = null;

// If Safari  is used
} else if ( jQuery.browser == "safari" ) {
        // Continually check to see if the document.readyState is valid
        jQuery.safariTimer = setInterval(function(){
                // loaded and complete are both valid states
                if ( document.readyState == "loaded" || 
                        document.readyState == "complete" ) {

                        // If either one are found, remove the timer
                        clearInterval( jQuery.safariTimer );
                        jQuery.safariTimer = null;

                        // and execute any waiting functions
                        jQuery.ready();
                }
        }, 10);
}

そして、jQueryのおかげで、開発者はこれらすべての落とし穴を、ライブラリ開発チームの肩に任せることができました。

jQueryは後に、アニメーションやAjaxなどのより高度な技術の実装を容易にしました。ライブラリは、実際には標準のWebサイト依存関係になっています。そして今日、それはインターネットの大きなシェアの仕事を提供します。 W3Techsは、現在、サイトの74％がjQueryを使用していると推定しています。

jQueryの開発を制御することもより正式になりました。 2011年、チームはjQuery Boardを作成しました。そして2012年、jQuery Board はjQuery Foundationに変わりました。

2015年、jQuery FoundationはDojo Foundation に加わり、JS Foundationを設立しましたその後、2019年にNode.js Foundationと協力してOpenJS Foundationを作成しました。OpenJSFoundationでは、jQueryは「パンチのあるプロジェクト」の1つでした。

状況の変化


ただし、近年、jQueryの人気が高まっています。GitHub は、サイトのフロントエンドからライブラリを削除しました。Bootstrap v5 は、「通常のJavaScriptに対する最大のクライアント依存関係」（現在30 KB、最小化およびパッケージ化）であるため、jQueryを取り除きます。Web開発におけるいくつかの傾向により、必要なツールとしてのjQueryの位置付けが弱まっています。

ブラウザー


多くの理由により、ブラウザの違いや制限はそれほど重要ではなくなりました。まず、標準化が改善されました。主要なブラウザ開発者（Apple、Google、Microsoft、およびMozilla）は、Web Hypertext Application Technology Working Groupの一部として共同でWeb標準を開発しています。 ブラウザーは依然としていくつかの重要な点で互いに異なりますが、ベンダーは少なくともお互いに永続的な戦争をする代わりに、共通の基盤を検索して作成する手段を持っています。したがって、ブラウザAPIは新しい機能を獲得しています。たとえば、Fetch APIはjQueryのAjax関数を置き換えることができます。


// jQuery
$.getJSON('https://api.com/songs.json')
    .done(function (songs) {
        console.log(songs);
    })

// native
fetch('https://api.com/songs.json')
    .then(function (response) {
        return response.json();
    })
    .then(function (songs) {
        console.log(songs);
    });

querySelectorとquerySelectorAll方法は jQueryのからセレクタを複製します：

// jQuery
const fooDivs = $('.foo div');

// native
const fooDivs = document.querySelectorAll('.foo div');

これで、classListを使用して要素クラスを操作できます。

// jQuery
$('#warning').toggleClass('visible');

// native
document.querySelector('#warning').classList.toggle('visible');

あなたはjQueryを必要としないかもしれませんjQuery Webサイトは、jQueryコードをネイティブコードに置き換えることができるいくつかの状況をリストしています。一部の開発者は常に新しいAPIについて知らないため、常にjQueryを使用しますが、それを発見すると、このライブラリの使用頻度を減らします。

ネイティブ機能を使用すると、ページのパフォーマンスを向上させることができます。多くのjQueryアニメーション効果がCSSを使用してはるかに効率的に実装できるようになりました。

2番目の理由は、ブラウザーが以前よりもはるかに速く更新されるためです。それらのほとんどには、「常緑の」更新戦略があります。Apple Safariを除く。これらはユーザーの関与なしにバックグラウンドで更新でき、OSの更新とは関係ありません。

これは、新しいブラウザー機能とバグ修正がはるかに速く広がっていることを意味し、開発者は「使用できる」共有が許容レベルに達するまで待つ必要がありません。彼らは、jQueryやポリファイルを読み込まなくても、自信を持って新しい機能とAPIを使用できます。

3番目の理由は、Internet Explorerがまったく無関係の状態に近づいていることです。 IEは長い間、世界中のWeb開発の惨劇でした。その典型的なバグは広く蔓延しており、IEは2000年代に主流になり、「常緑の」アップデート戦略を使用しなかったため、古いバージョンがまだ一般的です。

2016年、Microsoft は 10番目とそれ以前のバージョンのサポートを停止することでIEの廃止を加速し、IE 11のサポートに限定しました。さらに、Web開発者はIEの互換性を無視するという贅沢を利用できるようになりました。

jQueryでさえ、2013年にリリースされたバージョン2.0以降、IE 8以下のサポートを終了しました。また、古いサイトなどではIEのサポートが必要な場合もありますが、これらの状況はあまり一般的ではありません。

新しいフレームワーク


jQueryのの出現以来、多くのフレームワークは、現代の指導者を含め、作成されたリアクト、角度、およびVueの。 jQueryに比べて2つの重要な利点があります。

まず、ユーザーインターフェイスをコンポーネントに簡単に分割できます。フレームワークは、レンダリングとページの更新を処理するように設計されています。また、jQueryは通常、更新のみに使用され、開始ページを提供するタスクをサーバーに委託します。

一方、React、Angular、およびVueコンポーネントを使用すると、HTML、コード、さらにはCSSまでを密接にリンクできます。コードベースを多くの自己完結型の関数とクラスに分割するため、インターフェースを再利用可能なコンポーネントに分割できるため、複雑なサイトの組み立てとメンテナンスが簡単になります。

2番目の利点は、より最近のフレームワークが宣言的パラダイムに準拠していることです。開発者は、インターフェースがどのように見えるかを記述し、フレームワークに必要なすべての変更の実装を委託して、望ましいものを実現します。このアプローチは、jQueryコードに特有の命令型アプローチと矛盾します。

jQueryでは、変更を加えるための手順を明示的に規定します。そして、宣言型フレームワークでは、「このデータによると、インターフェースは次のようになるはずです。」と言います。これにより、バグのないコードの記述が大幅に容易になります。

開発者はWebサイト開発に新しいアプローチを採用しているため、jQueryの人気は低下しています。

いつjQueryを使用するのですか？


では、jQueryはいつ使用すべきですか？

プロジェクトの複雑さが増す場合は、複雑さを有意義に管理できるようにする別のライブラリまたはフレームワークから開始することをお勧めします。たとえば、インターフェースをコンポーネントに分割します。そのようなサイトでのjQueryの使用は最初は許容できるように見えるかもしれませんが、どのフラグメントがページのどの部分に影響を与えるかわからない場合、スパゲッティコードにすぐにつながります。

私はそのような状況にあり、何かを変えようとすると、困難な作業を感じます。 jQueryセレクターはサーバーによって作成されたHTML構造に依存しているため、何も壊さないという確信はありません。

もう一方の端にあるのは、インタラクティブな機能や動的なコンテンツを少しだけ必要とする単純なサイトです。そのような場合、ネイティブAPIでさらに多くのことができるため、デフォルトではjQueryも使用しません。

私はより強力な何かを必要とする場合であっても、私は、例えば、専門的なライブラリーを探しますaxiosのAjaxまたはのためAnimate.cssアニメーションのために。少しの機能のためにすべてのjQueryをロードするよりも簡単です。

jQueryを使用する最も正当な理由は、jQueryがフロントエンドサイトに包括的な機能を提供することです。さまざまなネイティブAPIや特殊なライブラリを学習する代わりに、jQueryのドキュメントのみを読むことができ、すぐに生産性が高まります。

命令型アプローチは適切にスケーリングされませんが、他のライブラリーの宣言型アプローチよりも学習が容易です。機能が明らかに制限されているサイトの場合、jQueryを使用して静かに作業することをお勧めします。ライブラリは複雑なアセンブリやコンパイルを必要としません。

さらに、jQueryは、時間の経過とともにサイトが複雑にならないことが確実で、ネイティブ機能を気にしない場合はもちろん、jQueryよりも多くのコードを記述する必要があります。

古いバージョンのIEをサポートする必要がある場合は、このライブラリを使用することもできます。そうすれば、jQueryは、IEが最も人気のあるブラウザーであった時代のように役立ちます。

未来への展望


jQueryはすぐには消えません。それは積極的に開発し、多くの開発者でもネイティブメソッドが利用可能で、そのAPIを使用することを好みます。このライブラリは、あらゆる世代の開発者があらゆるブラウザで動作するサイトを作成するのに役立ちました。そして、多くの面で新しいライブラリ、フレームワーク、パラダイムに取って代わられましたが、jQueryは最新のWebを作成する上で非常に前向きな役割を果たしてきました。

jQueryの機能が大幅に変更されない場合、今後数年間でライブラリの使用はゆっくりと、しかし着実に衰退し続ける可能性があります。新しいWebサイトは通常、最初からより新しいフレームワークを使用して作成されており、適切なjQueryの使用シナリオはあまり一般的ではありません。

誰かがWeb開発ツールの陳腐化の激しさを嫌っていますが、私にとってこれは急速な進歩の証拠です。jQueryを使用することで、多くのことを実行できます。同じことが彼女の後継者にも当てはまります。
