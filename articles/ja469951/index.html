日曜大工のセグメント化されたコントロール、iOS 13.0以降

IOS 13はSegmented Controlをアップデートしましたが、Xcodeをアップグレードできない人のために、それは利用できません。そこで、同じセグメント化されたコントロールを自分ですばやく作成する方法を書くことにしました。
行こう！

まず、ストーリーボードに移動します。 View要素をViewControllerに配置します。



必要に応じて、必要な場所に制約を付けて修正します。
背景をペイントし、[境界にクリップ]チェックボックスをオンにします。



別のビューを取得して、最初のビューの中に配置します。最初のビューの中にあることを確認してください。はしごに置く必要があります。



次に、すべての側から2 pixのインデントを持つ定数でそれを修正します。



下の写真のように、最初のビューの中に配置されます。



次に、内部にある白いビューを選択します。先頭と末尾の制約を選択し、優先度を750に変更し



ます。次に、このビューに対して、0 pixでインデントされた左右からさらに2つの定数を作成します



。また、先ほど作成した定数データの編集に切り替えて、優先度= 900と灰色のビューの端からではなくそれらを修正しますが、中心に対して相対的に設定します。すべてを正しく行うと、白いビューは中心の線で組み立てられます。



次に、2つのボタンを選択して灰色のビュー内に配置しますが、ボタンを移動したときにテキストが白いビューと重ならないように、ボタンは階層的に一番下に配置する必要があります。ボタンの半分に対して中央にボタンを固定します。



すべて。次に、ViewControllerを使用して要素をストーリーボードに接続します。
私が呼び出した灰色のビュー-fonView。
ホワイトビューが呼び出されました-blockView。
また、2つのボタンbuttonOneとbuttonTwoは、それらのアクションを作成しました。



また、優先度750で作成した2つの定数をコードに取り込みます。これらに名前を付けます。

@IBOutlet weak var checkButtonOne: NSLayoutConstraint!
@IBOutlet weak var checkButtonTwo: NSLayoutConstraint!



viewDidLoadでは、ビューの丸めのみを規定し、優先度950に等しい定数を1つ作成することで、白いビューが線のように中央になく、選択されたセグメントのように見えます。

override func viewDidLoad() {
        super.viewDidLoad()
        fonView.layer.cornerRadius = 10
        blockView.layer.cornerRadius = 9
        checkButtonOne.priority = UILayoutPriority(rawValue: 950)
    }

フィニッシュライン。アクションボタンには、そのようなコードを記述します。結論としては、優先順位を変更し、必要なサイズだけビューを増やし、定数をアニメーションで更新しています。


@IBAction func buttonOne(_ sender: UIButton) {
        checkButtonOne.priority = UILayoutPriority(rawValue: 950)
        checkButtonTwo.priority = UILayoutPriority(rawValue: 750)
        UIView.animate(withDuration: 0.5) {
            self.view.layoutIfNeeded()
        }
    }
    
@IBAction func buttonTwo(_ sender: UIButton) {
        checkButtonOne.priority = UILayoutPriority(rawValue: 750)
        checkButtonTwo.priority = UILayoutPriority(rawValue: 950)
        UIView.animate(withDuration: 0.5) {
            self.view.layoutIfNeeded()
        }
    }

全部です！小切手！
