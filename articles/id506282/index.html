.NET Core vs Node.js



Tujuan artikel ini bukan untuk memilih yang terbaik dari kedua platform, tetapi untuk mencari kekuatan dan kelemahan masing-masing. Kedua teknologi tersebut sangat mapan di dunia pengembangan web. Masing-masing dari mereka memiliki penggemarnya, yang menerima hasil yang baik dan berbagi prestasi mereka di Internet. Jaringan memiliki banyak ulasan tentang manfaat yang diterima pengembang ketika beralih ke .Net Core atau Node.js. Mari kita coba mencari tahu.

Sedikit konteks


Saya telah bekerja di perusahaan yang telah terlibat dalam pengembangan kustom sejak 2015, pertama kami menulis di .NET, kemudian beralih ke .NET Core dan kemudian secara bertahap mengubah tumpukan ke Node.JS. Alasan utamanya adalah kecepatan perkembangan.

Informasi Umum




.Net Core

.NET Core adalah kerangka kerja tujuan umum sumber terbuka untuk membuat aplikasi lintas platform. Memungkinkan Anda membuat aplikasi untuk berbagai sistem operasi Windows, macOS dan Linux. Mendukung prosesor x64, x86, ARM32 dan ARM64. Rilis resmi pertama dirilis pada 27 Juni 2016.

Node.js

Menurut situs web resmi , Node.js adalah platform perangkat lunak yang dibangun di atas V8. V8, pada gilirannya, adalah mesin JavaScript open source Google (ditulis dalam C ++) yang menerjemahkan JavaScript ke dalam kode mesin. Node.js memungkinkan Anda membuat aplikasi di macOS, Linux, SmartOS, FreeBSD, Microsoft Windows, AIX, dan Android. Rilis pertama adalah pada tahun 2009.

Sistem berbagi pengetahuan Stackoverflow menerbitkan peringkat popularitas kerangka tahun lalu berdasarkan survei lebih dari 90 ribu pengembang, hasil survei adalah sebagai berikut:


Node.js lebih populer menurut survei ini. Namun, hasilnya dapat berubah ketika. Net 5.0 dirilis, sebagai akibatnya tempat kedua dan ketiga pada dasarnya akan bergabung, setelah menerima jumlah total suara yang sebanding dengan Node.js saat ini.

Bahasa pemrograman




Net Core

Mostly C # digunakan - bahasa pemrograman berorientasi objek yang dikembangkan oleh Microsoft pada tahun 1998-2001. Pengetikan ketat memungkinkan Anda untuk menangkap kesalahan pada tahap kompilasi, sebagai hasilnya, mendapatkan loop umpan balik yang lebih cepat, dan mengembangkan aplikasi lebih cepat dengan lebih sedikit kesalahan. Menggunakan kata kunci "dinamis" memungkinkan Anda untuk meniru pengetikan dinamis. JavaScript

Node.js

adalah bahasa yang diartikan, berorientasi objek dengan pengetikan dinamis. Ini juga memiliki sejumlah fitur yang melekat dalam bahasa fungsional.

Bahasa JavaScript standar adalah ECMAScript. JavaScript mudah digunakan, tetapi banyak kesalahan harus didebug saat runtime, tidak seperti C #. Untuk menghilangkan kelemahan ini, Anda bisa menggunakan TypeScript, yang diperkenalkan oleh Microsoft pada 2012. TypeScript adalah add-on JavaScript, kompatibel dengan JavaScript, dan dikompilasi ke dalam yang terakhir. Namun, TypeScript juga tidak menyediakan semua kekuatan dari bahasa berorientasi objek yang diketik kuat yang ditawarkan oleh C #. TypeScript terkait erat dengan JavaScript, yang memiliki tipifikasi struktural, sebagai akibatnya hanya dapat meniru pekerjaan dengan jenis dalam kemiripan C #.

Contoh kesalahan yang tidak muncul pada tahap perakitan:

interface A {
    x: number;
}
 
let a: A = {x: 5}
 
let b: {x: number | string} = a; 
b.x = "unsound";
 
a.x.toFixed(0); // Not a function error occurs.

Masalah dengan kode ini adalah bahwa sebagai hasil dari penugasan, jenis "kapak" akan menjadi string, dan kesalahan hanya akan di runtime.

Modul dan Alat




Net Core

Community mendistribusikan alat melalui paket NuGet . Saat ini ada 205.000 paket. Untuk pengembangan, yang paling umum adalah Visual Studio dan Rider, yang menyederhanakan dan mempercepat pengembangan aplikasi.

Node.js

Toolkit ini adalah NPM, yang memiliki 1.298.879 paket. Lingkungan pengembangan resmi belum diumumkan, tetapi penggunaan paling umum dari Visual Studio Code dengan plugin untuk pengembangan untuk Node.js.
Secara umum, sepertinya Node.js memiliki pilihan alat yang lebih luas, paket alternatif.

Skalabilitas




Mari kita pertimbangkan hanya satu opsi skalabilitas - yaitu horisontal menambahkan server untuk menangani peningkatan beban kerja. Platform

Net Core

menyediakan kemampuan untuk menulis aplikasi multi-utas, sehingga tugas mengunduh server terpisah agak lebih mudah daripada untuk Node.js. Untuk layanan penyebaran, penskalaan, dan ketahanan, Microsoft telah merilis Azure Service Fabric SDK. Anda dapat menggunakan SDK ini di server Anda, di Azure atau AWS. Dokumentasi diposting di sini .

Node.js Satu

instance hanya menggunakan satu utas, jadi untuk memuat semua kernel server Anda harus menggunakan modul “ Cluster ”"Atau kontrol proses eksternal. Untuk skala di beberapa server, Anda memerlukan load balancer (AWS ELB / NGINX). Ada juga alat yang menyediakan kemampuan untuk mengelola layanan dan lalu lintas, misalnya, Istio.

Secara umum, masalah penskalaan diselesaikan berdasarkan kedua teknologi, yang menyediakan semua alat yang diperlukan untuk ini.

Performa




Kinerja tergantung pada banyak parameter, termasuk teknologi yang dipilih, berdasarkan dasar aplikasi tersebut dibangun. Skenario yang dipilih juga memainkan peran mereka, yang hasilnya akan dibandingkan. Oleh karena itu, tugas membandingkan kinerja ke tingkat yang lebih besar adalah untuk meratakan kondisi percobaan di mana pengukuran dilakukan. Saya memilih konfigurasi yang paling dekat dengan kenyataan di antara tolok ukur yang tersedia untuk membandingkan hasilnya.

Dalam semua tes di bawah, parameternya sama dan memiliki nilai berikut: OS - Linux, DB - Postgres, ketik ORM - FULL (mis. Bukan micro ROM dan tidak akses langsung ke DB).

Permintaan basis data tunggal

Dalam pengujian ini, setiap kueri mengambil satu baris dari tabel, maka baris ini diserialisasi dalam JSON dan respons dikirim. Jumlah koneksi konkuren adalah 16 hingga 512.



Beberapa kueri dalam database

Dalam tes ini, setiap kueri melakukan beberapa kueri dalam database, dan masing-masing mengambil banyak baris. Ada 512 senyawa kompetitif dalam tes ini.



Memperbarui data dalam database

Setiap kueri mengambil banyak baris dari database, mengubahnya menjadi objek, mengubah objek, menyimpan perubahan ke database satu per satu. Selanjutnya, serialisasi objek yang diperbarui dan mengirimkannya dalam respons. Ada 512 senyawa kompetitif dalam tes ini.



Uji Hello World

Permintaan dijawab dalam teks biasa "Hello, World".



Menurut tolok ukur, Net Core lebih cepat dari Node.js, tetapi Anda harus selalu mempertimbangkan skenario spesifik dan teknologi yang digunakan.

Kesimpulan


Tabel ringkasan peringkat untuk item yang dibahas di atas.



Perlu dicatat sekali lagi bahwa tidak ada alat universal untuk semua kesempatan. Jika Anda seorang pemula dan ingin cepat membuat aplikasi yang memiliki antarmuka pengguna dan kode server, maka Node.js akan menjadi pilihan yang bagus. Sudah cukup untuk berurusan dengan JavaScript, dan itu bisa menjadi bahasa tunggal untuk server dan sisi klien.

Tugas yang kompleks dan banyak diselesaikan dengan menggunakan .Net Core, ini dalam banyak kasus mengurangi jumlah ketergantungan eksternal, yang akhirnya menyederhanakan dukungan. Node.js memiliki kinerja yang baik, tetapi jika Anda membutuhkan sesuatu yang lebih dan aplikasi Anda tunduk pada beban ekstrem, maka ini adalah kesempatan untuk berpikir tentang menggunakan .Net Core
