DIY通話ボタンパート2. Raspberry Piのテレビ電話

このパートでは、SIPプロトコルを使用してビデオ通話を行うためのRaspberry Pi VoIPデバイスを作成します。

最初のタスクは同じままで、外部からの影響（ボタンを押す）によって電話をかけました。しかし、アイデアの実装は少し変わっています。前のパートと同様に、LinphonecターミナルSIPクライアントを使用しますが、セットアップを簡略化するために、ビデオサポートを含む無料のオンネットコールでZadarma.com VoIPプロバイダーを使用することにしました。ビデオ通話には、手頃なUSBカメラを使用しました。

詳細、およびカットの下の段階的な手順。


システムの最初の部分は、RPI v3_40_intの基本イメージ上で、追加のインストールを行わずにすぐに導入されます。

テストと投稿の作成の過程で、VoIPサーバー（Fresswitch）の使用を中止することにしました。 VoIPサーバーをインストールすると、セットアッププロセスが複雑になりますが、柔軟性が高まります。

例として、Habr VoIPで提示された演算子Zadarmaを使用します。

電話をかけるには、SIPサーバーに2つのアカウントが必要です。1つでは、Raspberry Piとコンソールクライアントが機能し、2つ目はスマートフォン（またはPC）で機能します。登録プロセスは直感的で、説明は必要ありません。

LinphonecコンソールSIPクライアントとLinphonecsh管理ユーティリティのインストール


ソースからパッケージをビルドするには、追加の依存関係をインストールします。

sudo apt-get install cmake automake autoconf libtool intltool yasm libasound2-dev libpulse-dev libv4l-dev nasm git libglew-dev

ホームディレクトリに移動してLinphoneパッケージ自体をダウンロードします。ダウンロードには約20分かかりました。

cd /home/pi/
git clone git://git.linphone.org/linphone-desktop.git -recursive

パッケージをダウンロードするときに作成されたディレクトリに移動し、グラフィカルインターフェイスなしのバージョンのインストールの準備をします。

cd linphone-desktop
sudo ./prepare.py no-ui -DENABLE_OPENH264=ON -DENABLE_WEBRTC_AEC=OFF -DENABLE_UNIT_TESTS=OFF -DENABLE_MKV=OFF -DENABLE_FFMPEG=ON -DENABLE_CXX_WRAPPER=OFF -DENABLE_NON_FREE_CODECS=ON -DENABLE_VCARD=OFF -DENABLE_BV16=OFF -DENABLE_V4L=OFF

–j4属性を使用してビルドします（つまり、4つのスレッドを同時にビルドします：

sudo make -j4

組み立てには30〜40分かかります。

コンパイルされたプログラムファイルは、OUTPUT / no-ui / binディレクトリにあります。プログラムを実行するには、プログラムにアクセスし、プログラムのバージョンを確認します。

cd OUTPUT/no-ui/bin
./linphonec -v

結果は次のようになります：version：3.12.0

ボタンを接続してGPIOを操作する




Raspberry Piには、インターフェースとなる多数のGPIOポート（汎用入出力）があります。これには、さまざまなアクチュエータを接続できる汎用入力と出力が含まれます。この例では単純なボタンです。

ほとんどすべてのGPIOピン（複数の予約済みピンがあります）は、「出力」（別名OUTまたは論理1）または「入力」（INまたは論理0）のいずれかの状態に設定できます。出力電圧は3.3Vです。

ボタンには、一番上の行の一番右の2つのピンを使用します。コネクタ＃38（GPIO20）は「出力」（OUT）に設定され、コネクタ＃40（GPIO21）は「入力」（IN）に設定されます。

BBJジャンパーは、GPIOコームへの接続に使用されます。ラズベリーがネットワークから切断されているときに接続することをお勧めします。

2つのスクリプトを作成してみましょう。1つはGPIOポートの初期化に使用され、2つ目はLinphonecshユーティリティ制御コマンド用に使用されます。

GPIOポートを初期化するBashスクリプト：

sudo nano /usr/local/bin/gpio21.sh

内容を貼り付けます

gpio21.sh
#! /bin/bash
# Set up GPIO20 and set to output
echo 20 > /sys/class/gpio/export
echo out > /sys/class/gpio/gpio20/direction
echo 1 > /sys/class/gpio/gpio20/value
# Set up GPIO21 and set to input
echo 21 > /sys/class/gpio/export
echo in > /sys/class/gpio/gpio21/direction

ファイルを実行可能にします。

sudo chmod +x  /usr/local/bin/gpio21.sh

ボタンをクリックするだけでダイヤルするための2番目のスクリプトは、その後MajorDoMoホームオートメーションシステムと統合するために、PHPで行うことにしました。ボタンを押すだけで、ポート21のステータスが変化し、システムは定期的にポート21の値を読み取り、変化した場合はダイヤリングコマンドをLinphonecshに送信します。

sudo nano /usr/local/bin/dial.php

内容を貼り付けます：

dial.php

次に、起動ファイルの最後に3行を書き込みます。

  1. GPIOポート初期化スクリプト
  2. 自動応答モードおよびビデオ転送サポートでの管理ユーティリティの起動
  3. PHPスクリプトダイヤリング。

crontab -e


@reboot sudo /usr/local/bin/gpio21.sh
@reboot sudo -u pi /home/pi/linphone-desktop/OUTPUT/no-ui/bin/linphonecsh init -c /home/pi/.linphonerc -V -a
@reboot sudo php /usr/local/bin/dial.php

結論としての小さな余談


当初、自分では電話が使えない病気の人とコミュニケーションを図るための装置を作ろうと思っていましたが、その後は必要がなくなったので、仕事を終えることにし、ホームオートメーションのコンポーネントの1つを他の目的に使用する可能性を見ました。

作成するときは、次の基準を満たすか、3つのうち少なくとも2つを満たす必要があります。

  1. 可用性（ここ（今のところ）でコンポーネントを購入する機能）。
  2. 予算（出口でのデバイスの低コスト）。
  3. 実行の単純さ、または少なくとも段階的な指示の完全性（特別な知識と時間のコストなしに、デバイスを作成してそれを使い始めるためにそれを繰り返す必要がある人の能力）。
