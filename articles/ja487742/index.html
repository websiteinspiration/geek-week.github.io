アービトラージ取引（Bellman-Fordアルゴリズム）



取引所での取引は通常、リスクを伴います。これは、ほとんどの取引戦略に当てはまります。これらの場合の取引の成功は、リスクを正しく評価して管理する能力によってのみ決定されます。しかし、すべての取引戦略がそうであるとは限りません。特に調停を含むリスクのない戦略があります。この記事では、アービトレーションとは何か、ベルマンフォードアルゴリズムなどの従来のグラフアルゴリズムを使用してアービトレーションを実装する方法について説明します。

仲裁とは何ですか？


仲裁は、異なる市場での同じまたは関連する資産の価格の違いからの利益を目的とするいくつかの論理的に関連するトランザクションであり（空間的仲裁）、または同じ市場で異なる時点で（一時的な仲裁） 。

簡単な例として、空間的裁定取引を考えます。ニューヨークとロンドンでは、ユーロでドルを購入し、ドルでユーロを購入することができます。これは、ニューヨークでは3ユーロで4ドル、ロンドンでは3ユーロで5ドルのレートで行うことができます。このレートの違いにより、空間的仲裁の可能性が広がります。



4ドルで、ニューヨークでは3ユーロを購入できます。その後、ロンドンで、これらの3ユーロ5ドルで購入します。ご覧のとおり、このような単純なトランザクションのシーケンスでは、投資した4ドルごとに1ドルの利益がもたらされます。したがって、最初に400万ドルあれば、利益はすでに100万ドルになります。

同じ通貨ペアの為替レート（スプレッドは考慮しない）が異なる場合、裁定取引戦略を実装するために必要な一連のトランザクションは非常に簡単です。1つの通貨ペアの為替レートが固定されているが、複数の通貨ペアが並行して取引されている場合、仲裁も可能ですが、トランザクションのシーケンスはすでに重要です。たとえば、4ユーロを5ドル、3ポンドを4ユーロ、6ドルを3ポンドで購入できます。そのような一連のトランザクションからの利益は、投資された5ドルごとに1ドルになります。



何百もの通貨ペアが取引所で取引でき、為替レートは常に変化しています。アルゴリズムのソリューションなしでは、どのトランザクションシーケンスが利益をもたらすかを理解することはすでに不可能です。

アルゴリズム問題への移行


アルゴリズムの形、つまりグラフの形で潜在的な為替取引を想像してみてください。このグラフの頂点は通貨を表し、エッジは潜在的なトレードです。リブの長さは、トランザクションを完了できる為替レートに対応しています。


次の質問は、利益をもたらすような列で一連のトランザクションを見つける方法です。明らかに、シーケンスの最初と最後では同じ通貨である必要があるため、シーケンスは特定のグラフのサイクルに対応している必要があります。次に、2つの通貨が直接交換されず、特定の3番目の通貨（または任意の数の中間操作）を介して交換される場合、2つの通貨間の為替レートを計算する方法を決定する必要があります。ここでもすべてが非常に単純です。このような為替レートは、中間取引の為替レートの積として計算されます。この商品の値が1未満の場合、収益性のあるトランザクションのシーケンスになります。言い換えれば、通貨の単位が同じ通貨の単位未満で購入できる場合。


古典的なグラフアルゴリズムは、エッジの長さの積での作業にはあまり適していません。このようなアルゴリズムは主に、これらの長さの合計として定義されるパスを見つけることを目的としています。ただし、この制限を回避するために、積から和に移動する数学的な方法があります。この方法は対数です。積が対数の下に表示される場合、そのような対数は対数の合計に変換できます。この方程式の右辺では、必要な数は1未満です。つまり、この数の対数はゼロ未満でなければなりません。




このような単純な数学的トリックを使用すると、エッジ長の積が1未満のサイクルを検索することから、エッジ長の合計が0未満のサイクルを見つけることへと移行できます。このようなタスクは、古典的なグラフアルゴリズム、より正確にはベルマンフォードアルゴリズムによってすでに解決可能に見えます。

ベルマン-フォードアルゴリズム


Bellman-Ford アルゴリズムは通常、特定の頂点からグラフの他のすべての頂点までの距離を見つけるために使用されますが、その変更により、負の長さのサイクルを見つけることができます。

このアルゴリズムの基本的な操作は、エッジ緩和です。この操作の本質は次のとおりです。エッジがあると仮定しますa→b、およびピークまでの距離の以前に計算された予備値もわかっています a そして b。エッジリラクゼーションを実行するには、上部までの距離を計算する必要がありますbパスが上を通過した場合 a そしてリブ a→b。この距離は、上部までの距離の合計として計算されます。a とリブの長さ a→b。さらに、この距離が現在の予備距離より短い場合aそれからこれは非常に距離です a計算されたばかりの新しい値に対応して取得します。

アルゴリズムの残りの部分も複雑ではありません。必要N 回（Nは、グラフの頂点の数です）、エッジのリストをバイパスし、各ラウンドに緩和操作を適用します。アルゴリズムの複雑さが得られますN∗M （どこ N -頂点の数、および M-肋骨の数）。負のサイクルのないグラフの場合、さらにエッジを緩和しても、頂点までの距離は変化しません。同時に、負のサイクルを含むグラフの場合、緩和は頂点までの距離を減らし、その後N回り道。このプロパティを使用して、目的のサイクルを見つけることができます。

上記のKotlinでのアルゴリズムの次の小さな実装は、コードの整理に慣れている人に役立つはずです。

fun findNegativeCycle(nodes: Int, edges: Array<Edge>, source: Int): List<Int>? {
    // Initialize distances and prev arrays. All distances but the distance to
    // the source node are infinite, distance to the source node is zero.
    val distances = DoubleArray(nodes) { if (it == source) 0.0 else INFINITY }
    val prev = IntArray(nodes) { -1 }

    // Relax all edges N times where N is the number of nodes.
    repeat(nodes) {
        edges.forEach { it.relax(distances, prev) }
    }

    // Try to relax at least one more edge. If it's possible memorize the node,
    // otherwise return from the method.
    val firstRelaxedEdge = edges.firstOrNull { it.relax(distances, prev) }
    var node = firstRelaxedEdge?.to ?: return null

    // Step back N times where N is the number of nodes. As a result, the node will
    // be in the loop for sure.
    repeat(nodes) {
        node = prev[node]
    }

    // Recover the loop by the node that is inside it and prev links.
    val lastNode = node
    return buildList {
        do {
            add(node)
            node = prev[node]
        } while (node != lastNode)

        reverse()
    }
}

// Edge DTO with implemented relaxation operation.
data class Edge(val from: Int, val to: Int, val length: Double) {

    fun relax(distances: DoubleArray, prev: IntArray): Boolean {
        if (distances[from] + length >= distances[to]) {
            return false
        }

        distances[to] = distances[from] + length
        prev[to] = from
        return true
    }
}

負の長さのサイクルを含む小さなグラフの例を見てみましょう。アルゴリズムが機能するためには、各頂点が現在の既知の距離を維持すること、および前の頂点へのリンクを維持することが必要です。この場合の前の頂点への参照は、肋骨の弛緩の成功によって決まります。緩和操作が成功し、頂点までの距離が更新された場合、この頂点の前の頂点へのリンクも更新され、指定されたエッジのソース頂点の値を取得します。

したがって、最初に、最初の等しい無限大を除くすべての頂点への距離を設定して、頂点を初期化する必要があります。最初の頂点には、ゼロの距離が設定されます。


すべての肋骨の最初のラウンドが続き、それらの弛緩が行われます。肋骨の弛緩を除いて、ほとんどすべての弛緩は結果を与えませんa→b。このリブの緩和により、距離を更新できますb。


これに続いて、グラフのすべてのエッジの第2ラウンドと対応する緩和が続きます。今回、結果は肋骨の弛緩です。b→c、および b→d。ピークまでの距離が更新されます。c そして d。ここで、結果はエッジがトラバースされる順序に依存することに注意してください。


3番目の肋骨では、すでに3つの肋骨、つまり肋骨を正常にリラックスさせることができます。 →d、 d→e、 d→b。この場合、肋骨がリラックスしているとき→d そして d→b すでに記録された距離 d そして b、および以前の頂点への対応するリンク。


第4ラウンドでは、肋骨弛緩操作が正常に完了しました。 b→c そして b→a。この場合、すでに記録されているピークまでの距離の値が再び更新されます。a そして c、および以前の頂点への対応するリンク。


5回目が最後です。この歩行中に肋骨がリラックスする→d、 d→b、 d→e。ここでは、負の長さのサイクルが存在することで、ピークまでの距離の値がすでにある程度調整されていることがわかります。


このツアーの後で、グラフに負の長さのサイクルが含まれていない場合、エッジの緩和によって変更が加えられなかったため、アルゴリズムは完了します。ただし、このグラフでは、負の長さのサイクルが存在するため、緩和によって頂点の1つまでの距離が更新されるエッジを見つけることができます。


緩和によって頂上までの距離が更新されるエッジが見つかります。これは、負の長さのサイクルの存在を確認します。次に、このサイクル自体を見つける必要があります。現在更新されている距離である頂点が、サイクルの内側と外側の両方にあることが重要です。例では、これがトップですeそして彼女はサイクルから外れています。次に、アルゴリズムのすべてのステップで注意深く更新された以前の頂点へのリンクを参照する必要があります。サイクルに入ることが保証されるためには、あなたは一歩下がらなければなりませんNこれらのリンクを使用してピーク。

この例では、遷移は次のようになります。e→d→c→b→d→c。だからトップはc、負の長さのサイクルにあることが保証されています。


さらに、技術の問題。目的のサイクルを返すには、頂点が再び合流するまで、前の頂点へのリンクを使用して再度繰り返す必要がありますc。これは、サイクルが終了したことを意味します。前の頂点へのリンクを介した反復中に反復が逆にされたため、残っているのは順序を逆にすることだけです。

上記のアルゴリズムは、距離が計算される最初の頂点が存在することを前提としています。このような頂点の存在は、アルゴリズムが機能するために必要ではありませんが、元のベルマン-フォードアルゴリズムと一致するように大幅に導入されました。対象の主題が負の長さのサイクルである場合、特定のグラフのすべての頂点が初期であると仮定できます。つまり、すべての頂点までの距離は最初はゼロです。
