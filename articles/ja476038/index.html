Pulumiと.NET Coreを使用して最新のクラウドアプリケーションを作成する

Pulumiの.NET Coreのサポートを発表します！この発表は、C＃、VB.NET、F＃などのお気に入りの.NET言語を使用して、クラウドインフラストラクチャ（Kubernetes、Functions、AppService、仮想マシン、CosmosDBなどのAzureのすべてを含む）を宣言できることを意味します。これにより、すぐに使用できるインフラストラクチャとしてのコードメソッドを使用してコードエディターを離れることなく、クラウド全体を使用できます。



マイクロサービス、コンテナー、サーバー、データウェアハウスなどの最新のクラウド機能がアプリケーションのアーキテクチャに浸透している今日、インフラストラクチャの関連性が高まっています。インフラストラクチャという用語は、アプリケーションに必要なすべてのクラウドリソースをカバーします。現代のアーキテクチャは、アプリケーションを作成するときにそれらについて深く考える必要があります。 Pulumiのアプローチは、インフラストラクチャ開発者とチームが協力して、最新のクラウドが提供するすべてのものを活用する革新的な新機能を作成するのに役立ちます。

Pulumiは1年以上前に発売され、最近は1.0の安定マークに達しています。何百もの企業と協力してクラウドベースのアプリケーションを本番環境にデプロイした後、.NETはすぐにPulumiで最も頻繁にリクエストされるコミュニティ機能の1つに移行しました。特に、Pulumiチームの私たちの多くは初期の.NETエキスパートであるため、今日、クラウド開発のニーズに合わせて.NET CoreでPulumiを利用できることを非常に嬉しく思います。

プルミとは？


Pulumiでは、実際の言語を使用して、「コードとしてのインフラストラクチャ」と呼ばれる強力なメソッドを使用して、アプリケーションのインフラストラクチャニーズを表現できます。インフラストラクチャをコードとして使用して、必要なインフラストラクチャを宣言します。メカニズムにより、インフラストラクチャが自動化され、複製が容易になり、生産要件を満たすのに十分な信頼性が得られます。Pulumiは、このアプローチをさらに一歩進め、実際の言語を使用し、コンテナやサーバーレスプログラムなどの最新のクラウドインフラストラクチャテンプレートを、ファーストクラスでシンプルなものにしています。

  • Kubernetes、Dockerコンテナー、サーバーレス機能などの従来のクラウドインフラストラクチャテクノロジーを使用して、スケーラブルなクラウドアプリケーションを作成します。

例：サーバーレスアプリケーションを備えたグローバルデータベース


Azure CosmosDBをグローバルデータ配布に使用する新しいアプリケーションを作成して、データベースに自動的にスケーリングするサーバーレスC＃アプリケーションを使用して、顧客がどこにいても高いパフォーマンスを提供するとします。通常、JSON、YAML、DSLなどのその他のインフラストラクチャ構築ツールを使用するか、Azureコンソールで手動でポイントアンドクリックします。このアプローチは一般的には正常ですが、恐ろしいです-複雑で未使用であり、開始するにはインフラストラクチャの専門家が必要であることを意味します。

Pulumiのアプローチは、私たちのお気に入りの.NET言語でコードを書くことであり、Pulumiツールは残りの部分を処理できます。たとえば、次のC＃コードは、サーバーレスのAzure AppService FunctionAppを使用してAzure CosmosDBデータベースを作成します。これは、データベースに合わせて自動的にスケーリングされます。

using System;
using System.Collections.Generic;

using Pulumi;
using Pulumi.Azure.AppService;
using Pulumi.Azure.Core;
using Pulumi.Azure.CosmosDB;

class Program
{
    static Task<int> Main(string[] args)
    {
        return Deployment.RunAsync(() => {
            var locations = new[] { "WestUS", "WestEurope", "SouthEastAsia" };

            var rg = new ResourceGroup("myapp-rg", new ResourceGroupArgs {
                Location = locations[0],
            });

            var app = new CosmosApp("myapp", new CosmosAppArgs {
                 ResourceGroup = resourceGroup,
                 Locations = locations,
                 DatabaseName = "pricedb",
                 ContainerName = "prices",
                 Factory = (location, db) => {
                     var func = new ArchiveFunctionApp("myapp-func",
                          new ArchiveFunctionAppArgs {
                              Location = location,
                              Archive = new FileArchive("app"),
                              AppSettings = new Dictionary<string, string> {
                                  ["COSMOSDB_ENDPOINT"] = db.Endpoint,
                              },
                          },
                      );
                      return func.App.ID;
                 },
            });
        });
    }

    //  CosmosApp  ArchiveFunctionApp   .
    //          «app».
    //         .
}
 GitHubでProgram.csを見る

このコードを記述したら、pulumi upコマンドを使用してPulumi CLIを実行します。最初に、作成する予定のインフラストラクチャリソースのプレビューが表示されます。確認後、アプリケーション全体と必要なインフラストラクチャを数分で展開します。



その後、変更を加える必要がある場合は、プログラムを編集してCLIを再実行するだけで、必要な増分変更が行われ、それに応じてインフラストラクチャが更新されます。展開の完全な履歴が記録されるため、変更内容が簡単にわかります。

.NETがインフラストラクチャにも優れているのはなぜですか？


私たちの多くは.NETを使用してアプリケーションを構築したいのですが、それをIaaSに使用してみませんか？実際、上記の利点のいくつかをすでに挙げています。それらの多くは、.NETをすでに知っていて愛している場合はおそらく明白ですが、簡単に繰り返しましょう。

任意の.NET言語を使用すると、IaaSの多くの便利な機能を利用できます。

  • トレーニング：DSLやマークアップテンプレート言語を学ぶ必要はありません。

  • 抽象化：共通のテンプレートをクラスと関数にカプセル化して、複雑さを隠し、同じテンプレートを何度もコピーして貼り付けることを避けます。

Pulumiは、.NETエコシステム全体へのアクセスを可能にします。これは当たり前のことですが、JSON、YAML、DSL、またはCLIスクリプトに基づく他のソリューションでは不可能です。上記のCosmosAppサンプルを追加するには、完全な言語へのアクセスが必要でした。これは、クラス、ループ、ラムダ、さらにはLINQを内部的に使用するカスタムコンポーネントです。このアプローチは、開発者とオペレーターが共通のフレームワークを使用して連携するのにも役立ちます。上記のすべてを一緒に追加すると、より速く、より信頼性の高い結果が得られます。
