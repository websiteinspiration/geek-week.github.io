Rust Embedded。STM32F103C8T6（Black Pill）デバッグボードを使用したCortex-M3プロセッサの開発

こんにちは！Rust Embeddedプロジェクトを紹介したいと思います。組み込みプラットフォーム（Embedded Linux / RTOS / Bare Metal）の開発にRustプログラミング言語を使用できます。


この記事では、Cortex-M3マイクロプロセッサの開発を開始するために必要なコンポーネントについて検討します。その後、簡単な例を記述します-内蔵LEDの点滅。

これを行うには、手頃で安価な中国のデバッグボードSTM32F103C8T6またはBlack Pill（黒色と小型のため）が必要です。ブルーのボードのバージョン-ブルーピルもあります。間違った抵抗があると聞いたのでお勧めしません。USBポートを使用するときに問題が発生します。ピンの配置（結論）のブラックピルとの2番目の違い。しかし、それについては後で詳しく説明します。

STM32デバッグボード用のプログラマーも必要です。この記事では、安価で手頃な価格の中国製ST-Link V2プログラマーを使用します。


Rustコンパイラのバージョン


はじめに、コンパイラのバージョンが1.31以降であることを確認する必要があります。次のコマンドを入力して、コンパイラのバージョンを確認できます。

> rustc --version

コンポーネントのインストール


これで、必要なコンポーネントのインストールを開始できます。

GNU Arm組み込みツールチェーン


ARMチップ用のデバッガー-arm-none-eabi-gdbが必要です。インストーラーを起動し、指示に従います。インストールの最後に、必ず「環境変数へのパスを追加する」オプションをチェックしてください。

ダウンロードリンク

インストール後、コマンドラインで次のように入力して、すべてが正しいかどうかを確認できます。

> arm-none-eabi-gdb -v

すべてが正常であれば、インストールされているコンポーネントのバージョンが表示されます。

ST-Linkドライバー


ST-Linkプログラマー用のドライバーのインストールに移りましょう。

インストーラーの指示に従い、オペレーティングシステムのビット深度に応じて、ドライバーの正しい（64ビットまたは32ビット）バージョンをインストールしてください。

ダウンロードへのリンク

ST-Linkツール


次のステップは、ファームウェアに必要なツールをインストールすることです-ST-Linkツール。アーカイブをダウンロードして任意の場所に解凍します。また、環境変数「PATH」でbinサブフォルダー（stlink-1.3.0 \ bin）へのパスを指定する必要があります。ダウンロードへのリンク



貨物binutils


最後に、cargo-binutilsパッケージをインストールします。これは、コンソールの2つのコマンドによって行われます。

> cargo install cargo-binutils
> rustup component add llvm-tools-preview

これでコンポーネントのインストールは完了です。

プロジェクトの作成と設定


続行するには、「stm32f103c8t6」という名前の新しいプロジェクトを作成する必要があります。思い出させてください、これはコマンドによって行われます：

> cargo new stm32f103c8t6

プロジェクトの依存関係と最適化


Cargo.toml ファイルで必要なライブラリを接続します。

[package]
name = "stm32f103c8t6"
version = "0.1.0"
authors = ["Nick"]
edition = "2018"

#      Cortex-M3
[dependencies]
cortex-m = "*"
cortex-m-rt = "*"
cortex-m-semihosting = "*"
panic-halt = "*"
nb = "0.1.2"
embedded-hal = "0.2.3"

#       stm32f1
[dependencies.stm32f1xx-hal]
version = "0.5.2"
features = ["stm32f100", "rt"]

#   `cargo fix`!
[[bin]]
name = "stm32f103c8t6"
test = false
bench = false

#   
[profile.release]
codegen-units = 1 #  
debug = true #  ,     Flash 
lto = true #

また、ファイルの最後でコードの最適化が有効になっていることがわかります。

cortex-m-rtライブラリでは、プロジェクトのルートディレクトリにファイルを作成する必要があります。このファイルは「memory.x」と呼ばれている必要があります。これは、デバイスに搭載されているメモリの量とそのアドレスを示します。

MEMORY
{
 FLASH : ORIGIN = 0x08000000, LENGTH = 64K
 RAM : ORIGIN = 0x20000000, LENGTH = 20K
}

ターゲットプラットフォームとデフォルトのコンパイル目標の設定


コンパイラのターゲットプラットフォームを設定する必要があります。これは次のコマンドで行います。

> rustup target add thumbv7m-none-eabi

その後、ルートディレクトリに「.cargo」フォルダを作成し、その中に「config」ファイルを作成します。設定

ファイルの内容：

[target.thumbv7m-none-eabi]

[target.'cfg(all(target_arch = "arm", target_os = "none"))']

rustflags = ["-C", "link-arg=-Tlink.x"]

[build]
target = "thumbv7m-none-eabi"  # Cortex-M3

その中で、デフォルトのコンパイル目標を特定しました。これにより、シンプルで使い慣れたコマンドでコードをARM .elfファイルにコンパイルできます。

> cargo build --release

例


それが機能することを確認するために、最も単純な例-LEDを点滅させることを検討してください。main.rs

ファイルの内容：

#![deny(unsafe_code)]
#![no_std]
#![no_main]

use panic_halt as _;

use nb::block;

use stm32f1xx_hal::{
    prelude::*,
    pac,
    timer::Timer,
};
use cortex_m_rt::entry;
use embedded_hal::digital::v2::OutputPin;

//   .
#[entry]
fn main() -> ! {
    
    //     
    let cp = cortex_m::Peripherals::take().unwrap();
    let dp = pac::Peripherals::take().unwrap();
    let mut flash = dp.FLASH.constrain();
    let mut rcc = dp.RCC.constrain();

    let clocks = rcc.cfgr.freeze(&mut flash.acr);
    let mut gpiob = dp.GPIOB.split(&mut rcc.apb2);

    //   b12   .
    //  "crh"      .
    //   0-7,    "crl".
    let mut led = gpiob.pb12.into_push_pull_output(&mut gpiob.crh);
    //        .
    let mut timer = Timer::syst(cp.SYST, &clocks)
    .start_count_down(1.hz());

    //     
    //    .
    loop {
        block!(timer.wait()).unwrap();
        led.set_high().unwrap();
        block!(timer.wait()).unwrap();
        led.set_low().unwrap();
    }
}

最初の行では、-属性#![deny(unsafe_code)]を使用して、安全でないコードを使用する可能性を排除しています。

2行目には、属性を配置しました#![no_std]。これは、ベアアイロン用のアプリケーションを作成しているため必須です。標準ライブラリにはオペレーティングシステムが必要です。

次に#![no_main]、引数ベクトルと戻り値の型を持つデフォルトのデフォルト関数を使用していないことをコンパイラーに伝える属性があります。関数を呼び出して戻り値を処理するオペレーティングシステムや他のランタイムがないため、これは意味がありません。

属性の後に、必要なモジュールをスコープに接続します。

これの後に関数が続きます。この場合、習慣から、私たちはそれを呼び出しました-main()。これはプログラムへのエントリポイントであり、属性によって決定されます#[entry]。

main関数内で、すべての周辺機器を「所有」する周辺オブジェクトへのハンドルを作成します。

その後、ボードに組み込まれたLEDを担当するピンを出力に設定できます。私の場合、それはブラックピルボードなので、その中のLEDはピンB12に接続されています。Blue Pillボードをお持ちの場合は、ピンC13が内蔵LEDの役割を果たします。したがって、ピンB12をプッシュプル出力として設定し、変数に割り当てますled。

これで、システムタイマーの更新イベントトリガー時間を設定し、変数に保存できます。timer。この例では、この時間は1秒です。

次に、無限ループで、プログラムのロジックを記述できます。これは非常に単純で、まずtimer、指定された時間が経過するまでプログラムの実行をブロックします。次のステップは、内蔵LEDのピンをHighに設定して、点灯させることです。その後、同じタイマーによってプログラムの実行が再び1秒間ブロックされます。そして、LEDのピンはローに設定されているため、消灯します。さらに、プログラムは周期的に繰り返されます。

コンパイル


プログラムのコードとロジックを理解したので、次のコマンドを使用してそれを.elfファイルにコンパイルできます。

> cargo build --release

この形式には、バイナリコードだけでなく、いくつかのヘッダーなども含まれています。これは、ファイルがオペレーティングシステムやブートローダーなどの他のソフトウェアによって起動された場合に役立ちます。

ただし、プログラムをベアメタルで実行するには、.elfファイルではなく.binファイルが必要です。.binファイルは、マイクロコントローラのメモリにバイトを書き込むことができるソフトウェアイメージです。.elfファイルは、次のコマンドを使用して.binファイルに変換できます。

> cargo objcopy --bin stm32f103c8t6 --target thumbv7m-none-eabi --release -- -O binary stm32f103c8t6.bin

これで、バイナリファイルのファームウェアの準備が整いました。

接続


ST-Link V2プログラマーを使用してボードをフラッシュします。これを行うには、まずボードに接続する必要があります。

プログラマの場合、コネクタのピン配列があります。コネクタの切り欠きに注意する必要があります。それは図にも表示され、接続の方法を理解できます。プログラマ

の3.3VピンとGNDピンをボード上の対応するピンに接続します。 PingのSWDIOはピンに接続DIO、およびピンSWCLKピンするCLKを。

ボードをプログラマに接続したら、ST-Linkをコンピュータに接続できます。

注意！接続する前に、他のすべての電源がある場合はボードから取り外してください。そうしないと、ボードまたはコンピューターが損傷する可能性があります。

これで接続を確認できます：

> st-info --descr

コンソールに、「F1 Medium-density device」という行が表示されます。

ファームウェア


オプションで、フラッシュする前に、何かが書き込まれている場合は、ボードのメモリをクリアできます。

> st-flash erase

最後に、ファームウェアを起動できます。 

> st-flash write stm32f1.bin 0x8000000

すべてが正常に完了すると、LEDが点滅するはずです。おめでとう！

結論


したがって、ボードをフラッシュするには、次の一連のコマンドを順番に使用する必要があります。

> cargo build --release
> cargo objcopy --bin stm32f103c8t6 --target thumbv7m-none-eabi --release -- -O binary stm32f103c8t6.bin
> st-flash erase
> st-flash write stm32f1.bin 0x8000000

私たちの生活を簡素化するために、これらのコマンドで.batファイルを作成し、コンソールから実行できます。
