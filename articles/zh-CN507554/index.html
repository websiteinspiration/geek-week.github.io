基于JsSIP + FreePBX的Web Sip客户端

FreePBX有一个Web Sip客户端。我想知道它是如何工作的。开始学习WebRTC技术后，我遇到了3个库-这些是JsSIP，其派生SIP.js和sipml5。



第一次无法运行JsSIP和SIP.js，经验不足。但是sipml5最容易使用。下载了其测试版本后，事实证明是第一个调用。



所有3个库均可使用。每个人都有自己的优点和缺点。测试了所有3个库之后，我选择了JsSIP。



我遇到的主要优点和缺点：



1. JsSIP



优点：



  • 轻便
  • 得到开发人员的积极支持。


缺点：



  • 在起步阶段，更难以启动；
  • 我必须编辑源代码才能使用星号。


2. Sip.js



优点：



  • 轻便
  • 重点放在使用星号。


缺点：



  • 无法为“早期”媒体启动声音（未连接声音以用于操作员预消息）。


3. Sipml5



优点：



  • 对于初学者来说相对简单。


缺点：



  • 复杂而令人困惑的代码，没有希望理解它。


训练



Web客户端仅在具有加密支持的页面上工作。我们需要创建一个自签名证书。如何做到这一点，Internet上有现成的文章。



Firefox允许您使用自签名证书。要在Chrome中工作，您首先必须制作一个根证书，并且在该证书的基础上已经签署了证书才能使手机正常工作。根证书必须导入到证书存储中。



在FreePBX上启用对Web SIP客户端的支持



管理员->用户管理->选项卡“组”->选项卡“ UCP”->选项卡“电话”

对于“启用电话”属性，选择“是”。



如果您在FreePBX中有分机号码，例如200，那么当您激活Web sip客户端时，将出现前缀为99的其他号码。99200.



在客户端浏览器上，在授予使用自签名证书的权限之后，您仍然需要授予对星号证书的权限。为此，请在浏览器中输入 



https://192.168.1.70:8089/ws


必须指定运行星号的IP地址。



编辑JsSIP源



在JsSIP库的源代码中，有必要将transport = ws替换为transport = wss。没有此编辑，客户端将无法工作。您无需在Sip.js中进行任何编辑，此刻已在此处提供。

JsSIP.debug.enable('JsSIP:*');
var socket = new JsSIP.WebSocketInterface('wss://192.168.1.70:8089/ws');
var configuration = {
	sockets  : [ socket ],
	//  	
	uri      : '99XXX@192.168.1.70',
	// 
	password : 'passw'
};

var remoteAudio = new window.Audio();
remoteAudio.autoplay = true;

var ua = new JsSIP.UA(configuration);

//   
ua.on('connected', function(e) { /*   */ });
ua.on('disconnected', function(e) { /*   */ });

ua.on('registered', function(e) { /*   */ });
ua.on('unregistered', function(e) { /*   */ });
ua.on('registrationFailed', function(e) { /*   */ });

//  
ua.start();

//   . 
var eventHandlers = {
	'progress': function(e) {
		console.log('call is in progress');

		session.connection.ontrack = function(e) {
			console.log(e);
			remoteAudio.srcObject = e.streams[0];
		};
	},
	'failed': function(e) {
		console.log('call failed with cause: ' + e.cause);
		$('#call').css({'display' : 'flex'});
		$('#hangup').css({'display' : 'none'});
	},
	'ended': function(e) {
		console.log('call ended with cause: ' + e.cause);
		$('#call').css({'display' : 'flex'});
		$('#hangup').css({'display' : 'none'});
	},
	'confirmed': function(e) {
		console.log('call confirmed');
		console.log(e);
	}
};

var options = {
	'eventHandlers'    : eventHandlers,
	'mediaConstraints' : { 'audio': true, 'video': false }
};

//   
$('#call').click(function(e) {
	session = ua.call($('#num').val(), options);
	$('#call').css({'display' : 'none'});
	$('#hangup').css({'display' : 'flex'});
});

//    
$('#hangup').click(function() {
	if (session) {
		session.terminate();
	}

	$('#call').css({'display' : 'flex'});
	$('#hangup').css({'display' : 'none'});
});
</script>

摘要



在过去两年中，使用Web sip客户端时，在更新浏览器时出现了一些问题。



例如，在Firefox的63版本中，我不得不更改SDP协议，这样才能避免会话中断。浏览器停止通讯而没有参数a = mid：0。在最新版本中，它不再相关，该错误已修复，在更高版本中，则没有此问题。



在带有参考的Firefox版本65中。通话过程中对话者的声音开始消失，LAN中没有此类问题。问题仅在通过网关呼出时出现。在chrome中，没有观察到这样的问题，一切都稳定运行。



大多数情况下，用户在Firefox中工作，因此，有必要删除库中SDP协议的重新协商。答案停止后，声音消失的问题不再困扰。
