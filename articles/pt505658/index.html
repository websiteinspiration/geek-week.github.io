Criando o efeito do voo rápido no espaço em 10 minutos no p5.js



Recentemente, sob a inspiração do canal The Coding Train , decidi participar de um dos desafios de 10 minutos, nos quais tive que criar a ilusão de voar pelo espaço em alta velocidade.

Para implementar o projeto, escolhi o conhecido p5.js - uma biblioteca para JavaScript projetada para criar arte de maneira algorítmica. Por que era impossível usar pacotes gráficos padrão da Adobe?

Em primeiro lugar, fazer isso com código é lindo.
Em segundo lugar, o código permite em tempo real, alterando apenas alguns parâmetros, para obter um resultado diferente, a velocidade de experimentos e iterações aumenta significativamente.
E em terceiro lugar, esse código é facilmente integrado a qualquer projeto da Web na forma de um script em JS.

Bem, até o infinito e além ...


Para iniciar um projeto, a maneira mais fácil é acessar o editor on-line em editor.p5js.org

A primeira coisa que devemos fazer é criar estrelas. É claro que precisamos de um grande número de estrelas, embora tecnicamente não seja difícil encontrar espaço no espaço sideral onde você não possa ver estrelas a olho nu, mas nossa ideia desse efeito foi criada usando filmes de ficção científica. Eu acho que você pode negligenciar a precisão por uma questão de eficácia. Assim. Precisamos criar um gerador de estrelas aleatório.

Primeiro, crie uma matriz no topo que armazene nossas estrelas:

let stars = [];


Vamos criar uma nova função chamada Star (), que deve ter quatro parâmetros principais: posição ao longo de x, ye variável z, através dos quais mudaremos todo o resto. Para criar uma ocorrência aleatória dos pontos iniciais, podemos usar a função random () e, como os parâmetros definem os limites da nossa tela em largura e altura.

A função random () funciona com 0, 1 ou 2 argumentos. Sem argumentos - retorna um número aleatório de 0 a 1, com um - de 0 ao argumento, com dois - do primeiro ao segundo.


Como vamos construir uma “fábrica” para a produção de estrelas, vamos introduzir imediatamente a palavra-chave this, que acessará e acessará informações do objeto a partir do qual chamaremos essa função (ou melhor, o método).

function Star() {
  this.x = random(-width, width);
  this.y = random(-height, height);

Pessoas experientes podem ter uma pergunta - vamos fazer 3D de verdade? Mas, na verdade, é muito mais fácil criar pseudo-3d, como nos jogos da velha escola. A variável z é necessária aqui para outros fins. Ela será responsável apenas pela ilusão da terceira dimensão.

Feito, embora as estrelas ainda não estejam visíveis, mas já podemos criá-las. É hora de dar-lhes movimento. Para fazer isso, expandimos nossa função Star () com um novo método (função aninhada) update (), responsável por um ciclo interminável de alterações.

Precisamos determinar a velocidade do movimento através das estrelas. Para fazer isso, crie uma velocidade variável e torne-a igual a 25. A lógica é a seguinte: alteramos o parâmetro z, que afetará os parâmetros x e y, para que, ao alterar um parâmetro, alteremos tudo. Assim que esse parâmetro atingir um valor menor que a unidade, reverteremos os parâmetros novamente para valores iniciais aleatórios, iniciando um novo ciclo de movimento da estrela na tela.

Bem, as estrelas são criadas e, mesmo em movimento, é hora de vê-las. Para fazer isso, usaremos a função show (), também criando uma chamada para o objeto por meio disso e pintando nossas estrelas de branco.

this.show = function() {
    fill("white");
}


Agora precisamos criar uma forma para nossa estrela, de modo a ter uma variedade de trajetórias e tamanho. Faremos isso através da mesma variável z, que é inicialmente igual à largura da tela, mas diminui em 25 com cada quadro até que se torne menor que 1, após o qual o ciclo se repete.

Para obter um belo movimento de estrelas do centro para a borda, vamos ver como a posição deve mudar, ou seja, as variáveis ​​x e y. Cada estrela deve aparecer em um raio imaginário e ir até a borda da tela.

Imagine que o ponto de partida é a coordenada 200.200. Precisamos fazer com que ambos os indicadores mudem gradualmente para baixo

Como z é sempre igual à largura da tela (por exemplo, 600) e x e y variam de -600 a 600, as variáveis ​​x e y no início sempre serão menores ou iguais a z. Isso sugere que podemos converter os valores em valores mais fáceis de gerenciar de 0 a 1, simplesmente dividindo x ou y por z.

Em seguida, podemos criar uma função de mapeamento padrão - para igualar números de 0 a 1 em proporção à largura ou altura da tela. E são precisamente esses valores que já são passados ​​como parâmetros para nossa estrela de bola: ellipse ()

Imagine que largura = 600 é o nosso cosmos. x = -300, y = -300. Nesse caso, o ponto de partida será -300/600 = -0,5, após 4 quadros, quando z diminuir em 4 * 25 = 100, a coordenada já será -300 / 500 = -0,6, ou seja, as coordenadas serão LINEAR longe do centro da tela (coordenadas 0,0) para cima e para a esquerda, até z se tornar menor que 1.

Caso sejam positivos, tudo acontecerá exatamente da mesma forma, apenas o movimento será para baixo e para a direita.

Caso sejam diferentes, as opções são para baixo e esquerda (com y positivo e x negativo) ou para cima e direita (com y negativo e x positivo).

A função elipse usa quatro parâmetros: posição ao longo de x, posição ao longo de y, diâmetro ao longo de x e diâmetro ao longo de y, com base nos quais a elipse é construída.

Bem, é hora de iluminar as estrelas e criar espaço.

Sob partidas e velocidade, criamos uma função padrão que cria o ambiente (funciona uma vez) - setup () Dentro de nós criamos nossa tela - createCanvas (600.600) e iniciamos nossa fábrica para criar estrelas através de um loop for, para começar, nos limitaremos a 800.

function setup() {
  createCanvas(600, 600);
  for (let i = 0; i < 800; i++) {
    stars[i] = new Star();
  }
}


Depois disso, criamos a segunda função padrão - draw (), sua diferença é que ela funciona em um loop infinito. Pintamos o fundo através de background () em preto, após o qual contamos o comprimento da matriz de estrelas (800) através do loop for e, referindo-se a cada estrela da matriz, chamamos a função (neste caso, o método) update () e show ()

function draw() {
  background("black");
  for (let i = 0; i < stars.length; i++) {
    stars[i].update();
    stars[i].show();
  }
}


É isso, voamos, mas há dois problemas. A primeira - como o ponto de referência das coordenadas, por padrão, não está no centro, mas no canto superior esquerdo, vemos apenas a parte das estrelas que não possui um valor de coordenada inicialmente negativo. O resto é deixado de vista.



Para resolver este problema é fácil - basta mover o centro através do comando de conversão (largura / 2, altura / 2).



O segundo problema é que as estrelas não mudam de tamanho quando se aproximam do observador. Em um nível subconsciente, sentimos que algo não está certo.

Para resolver esse problema, voltemos à nossa elipse, crie outra variável r, que também dependerá de z e determinará o terceiro e o quarto parâmetros da elipse, responsáveis ​​pelos diâmetros horizontal e vertical. O código final é o seguinte:

let stars = [];
let speed = 25;

function setup() {
  createCanvas(600, 600);
  for (let i = 0; i < 800; i++) {
    stars[i] = new Star();
  }
}

function draw() {
  background("black");
  translate(width / 2, height / 2);
  for (let i = 0; i < stars.length; i++) {
    stars[i].update();
    stars[i].show();
  }
}

function Star() {
  this.x = random(-width, width);
  this.y = random(-height, height);
  this.z = random(width);
 
  this.update = function() {
    this.z = this.z - speed;
    if (this.z < 1) {
      this.x = random(-width, width);
      this.y = random(-height, height);
      this.z = width;
    }
  };

  this.show = function() {
    fill("white");
    var sx = map(this.x / this.z, 0, 1, 0, width);
    var sy = map(this.y / this.z, 0, 1, 0, height);
    var r = map(this.z, 0, width, 10, 0);
    ellipse(sx, sy, r, r);
  };
}
