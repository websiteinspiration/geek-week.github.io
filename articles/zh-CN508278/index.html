实用函数式编程







本文文字摘自我2016年在LinkedIn上展示的一个演示文稿。该演示文稿试图在不使用“单子”，“不变性”或“副作用”等概念的情况下解释功能编程。相反，她专注于思考组合如何使您无论使用哪种语言都可以成为更好的程序员。



40年前的1977年10月17日，图灵奖授予约翰·巴库斯（John Bakus），以表彰他对高级编程系统（主要是Fortran编程语言）的开发做出的贡献。所有图灵奖获奖者都有机会在获奖期间就所选主题进行演讲。作为Fortran编程语言的创建者，人们希望Bakus演讲有关Fortran的好处以及该语言的未来发展。取而代之的是，他作了题为“是否可以从冯·诺依曼的风格中解放编程”的演讲？他批评当时包括Fortran在内的一些主要语言的缺陷。他还提出了另一种选择：函数式编程风格。



该讲座将传统程序及其“无法有效使用强大的组合形式”与“基于组合形式的使用”的功能性程序进行了对比。在过去的几年中，由于可扩展和并行计算的增长，函数式编程引起了新的兴趣。但是函数式编程的主要优点是您的程序是否可以并行化：函数式编程在结构上更好。



组合是通过组合简单零件来组合复杂行为的能力。在计算机科学课程中，人们非常重视抽象：处理一个大问题并将其分为多个部分。相反地​​，重点放在相反的位置：一旦您意识到了这些小零件，便如何将它们放在一起。似乎某些功能和系统很容易组合在一起，而其他功能和系统则复杂得多。但是我们需要退后一步，问：这些功能和系统的哪些属性有助于它们的布局？哪些属性使其布局困难？阅读足够的代码后，该模式开始出现，该模式是理解函数式编程的关键。

与异步代码相比讨论较少的非组合代码的另一个示例是循环，或更一般而言，返回多个值的函数（例如列表）。让我们看一个例子： 



// ["hello", "world"] -> ["hello!", "world!"]
List<String> addExcitement(List<String> words) {
  List<String> output = new LinkedList<>();
  for (int i = 0; i < words.size(); i++) {
    output.add(words.get(i) + “!”);
  }
  return output;
}

// ["hello", "world"] -> ["hello!!", "world!!"]
List<String> addMoreExcitement(List<String> words) {
  return addExcitement(addExcitement(words));
}

这是一个有点人为的示例，但是如果您想象代码散布在更大的代码库中，那么它说明了一个重要的观点：在大型系统上，当您尝试将事物分解为模块时，对一块数据的操作将不会在一起。您必须在模块化或性能之间进行选择。

现在，我们看了一些示例，让我们回到任务，以找出与其他属性相比，哪些属性使创建某些函数更容易。我们已经看到，诸如空对象，回调和循环之类的东西组合得不好。另一方面，可选类型，承诺和流确实可以很好地堆叠。为什么会这样呢？



答案是，复合示例在您要执行的操作和实际执行的操作之间有明显的区别。



前面的所有示例都有一个共同点。一种实用的处事方式着眼于您希望得到的结果。行动的迭代模式着重于您实际到达目的地的方式，实现细节。事实证明，编写关于如何做事情的迭代指令看起来不如需要做的事情的高级描述。
