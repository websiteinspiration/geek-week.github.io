フレネル反射


エッジの輝きは、さまざまな方法で実現できます。最も一般的なシェーダーコーディングは、よく知られたフレネル反射モデルを使用します。

フレネル反射の基礎となる方程式を理解するには、それがどこで発生するかを視覚化すると役立ちます。下の図は、カメラを通して砂丘がどのように見えるかを示しています（青色）。赤い矢印は、砂丘の上部の通常の表面を示し、鏡像になります。砂丘のすべての端に共通の特性があることが簡単にわかります。N、赤）は、ビューの方向に垂直です（V、青色）。


拡散色についてのパートで行ったのと同様に、スカラー積を使用できます。 N そして Vそれらの並列性の測定値を取得します。この場合N⋅V 等しい 0、2つの単位ベクトルが垂直であるため。代わりに使用できます1−N⋅Vそれらの非並列性の測定値を取得します。

直接使用1−N⋅V反射が強すぎるため、良い結果は得られません。反射をよりシャープにしたい場合は、表現を次第にとることができます。からの等級0 前 1は1つの間隔に制限されたままですが、闇と光の間の遷移がよりシャープになります。

フレネル反射モデルは、光の明るさがI 次のように設定されます。

I=(1−N⋅V)power∗strength(1)


どこ power そして strength-これらは、効果のコントラストと強さを制御するために使用できる2つのパラメーターです。パラメーターpower そして strength鏡面反射光および光沢と呼ばれることもありますが、名前は異なる場合があります。

式（1）をコードに変換するのは非常に簡単です。

float _TerrainRimPower;
float _TerrainRimStrength;
float3 _TerrainRimColor;

float3 RimLighting(float3 N, float3 V)
{
    float rim = 1.0 - saturate(dot(N, V));
    rim = saturate(pow(rim, _TerrainRimPower) * _TerrainRimStrength);
    rim = max(rim, 0); // Never negative
    return rim * _TerrainRimColor;
}

ブリンフォン反射のより詳細な説明は、物理ベースのレンダリングと照明モデルのチュートリアルにあります。以下は、シェーダーコードでの簡単な実装です。

float _OceanSpecularPower;
float _OceanSpecularStrength;
float3 _OceanSpecularColor;

float3 OceanSpecular (float3 N, float3 L, float3 V)
{
    // Blinn-Phong
    float3 H = normalize(V + L); // Half direction
    float NdotH = max(0, dot(N, H));
    float specular = pow(NdotH, _OceanSpecularPower) * _OceanSpecularStrength;
    return specular * _OceanSpecularColor;
}

マイクロフェイスの理論


より正式に状況にアプローチするには、砂丘をランダムな方向を持つ微視的な鏡で構成されているものとして認識する必要があります。このアプローチは、呼び出された微小面理論、これらの微小ミラーのそれぞれが呼ばれるマイクロファセット。最新のシェーディングモデルの数学的基礎は、Unityの標準シェーダーモデルを含むマイクロフェースの理論に基づいています。

最初のステップは、砂丘の表面を微小面に分割し、それぞれの向きを決定することです。すでに述べたように、砂の法線に関するチュートリアルの一部で同様のことを行いました。砂丘の3DモデルのUV位置を使用して、ランダムテクスチャをサンプリングしました。同じアプローチをここで使用して、各マイクロファセットにランダムな方向を付けることができます。各マイクロフェイスのサイズは、テクスチャのスケールとミップテクスチャリングのレベルに依存します。私たちの仕事は、特定の美学を再現することであり、フォトリアリズムへの欲求ではありません。このアプローチで十分です。

実装


上記の光沢効果はreflect、Cgの関数を使用して簡単に実装できます。これにより、計算が非常に簡単になります。R。

sampler2D_float _GlitterTex;
float _GlitterThreshold;
float3 _GlitterColor;

float3 GlitterSpecular (float2 uv, float3 N, float3 L, float3 V)
{
    // Random glitter direction
    float3 G = normalize(tex2D(_GlitterTex, uv).rgb * 2 - 1); // [0,1]->[-1,+1]

    // Light that reflects on the glitter and hits the eye
    float3 R = reflect(L, G);
    float RdotV = max(0, dot(R, V));
	
    // Only the strong ones (= small RdotV)
    if (RdotV > _GlitterThreshold)
        return 0;
	
    return (1 - RdotV) * _GlitterColor;
}

波の実現


waveを実装する方法はたくさんあります。最も安価なのは、単にテクスチャ上にそれらを描画することですが、チュートリアルでは他の何かを実現したいと考えています。その理由は単純です。波は「平坦」ではなく、光と正しく相互作用する必要があるためです。単純にそれらを描くと、カメラ（または太陽）が動いたときにリアルな効果を得ることができなくなります。

波を追加する別の方法は、砂丘モデルのジオメトリを変更することです。ただし、モデル全体のパフォーマンスに大きな影響を与えるため、モデルを複雑にすることはお勧めできません。

砂の法線に関する部分で見たように、法線マップを使用してこの問題を回避できます。実際、これらは従来のテクスチャのように表面に描画されますが、より複雑なジオメトリをシミュレートするための照明計算で使用されます。

つまり、タスクは別の方法に変わりました。これらの法線マップを作成する方法です。手動レンダリングは時間がかかりすぎます。また、砂丘を変更するたびに、波を再描画する必要があります。これにより、多くのテクニカルアーティストが回避しようとしているリソースの作成プロセスが大幅に遅くなります。

より効果的で最適なソリューションは、手続き的な方法で波を追加することです。つまり、砂丘の法線方向は、ローカルジオメトリに基づいて変化し、砂の粒子だけでなく波も考慮されます。

波は3Dサーフェスでシミュレートする必要があるため、各ピクセルの法線方向の変更を実装する方がより論理的です。波のパターンでシームレスな法線マップを使用する方が簡単です。このマップは、砂に使用されていた既存の法線マップと組み合わされます。

実装


以下のコードのスニペットは、法線を接線からワールド空間に変換し、上方向に対する勾配を計算します。

// Calculates normal in world space
float3 N_WORLD = WorldNormalVector(IN, o.Normal);
float3 UP_WORLD = float3(0, 1, 0);

// Calculates "steepness"
// => 0: steep (90 degrees surface)
//  => 1: shallow (flat surface)
float steepness = saturate(dot(N_WORLD, UP_WORLD));

砂丘の勾配を計算したので、それを使用して2つの法線マップを混合できます。両方の法線マップがサンプリングされ、フラットでクールです（以下のコードでは、_ShallowTexおよびと呼ばれています_SteepTex）。次に、それらは値に基づいて混合されますsteepness：

float2 uv = W.xz;

// [0,1]->[-1,+1]
float3 shallow = UnpackNormal(tex2D(_ShallowTex, TRANSFORM_TEX(uv, _ShallowTex)));
float3 steep   = UnpackNormal(tex2D(_SteepTex,   TRANSFORM_TEX(uv, _SteepTex  )));

// Steepness normal
float3 S = normalerp(steep, shallow, steepness);

砂の法線に関する部分で述べたように、法線マップを正しく組み合わせるのは非常に難しく、これはで行うことはできませんlerp。この場合slerp、を使用する方が適切ですが、代わりに、より安価な関数が呼び出されnormalerpます。

波の混合


上記のコードを使用すると、結果に失望する可能性があります。これは、砂丘の勾配が非常に小さいため、2つの通常のテクスチャの混合が多すぎるためです。これを修正するために、急勾配に非線形変換を適用できます。これにより、混合のシャープネスが向上します。

// Steepness to blending
steepness = pow(steepness, _SteepnessSharpnessPower);

2つのテクスチャを混合する場合、シャープネスとコントラストを制御するためによく使用されpowます。物理ベースのレンダリングのチュートリアルで、その仕組みと理由を学びました。

以下に2つのグラデーションを示します。上部には、X軸に沿って線形補間された黒から白の色が表示されc = uv.xます。下部では、同じグラデーションが次のように表されc = pow(uv.x*1.5)*3.0ます。



気づきやすいのでpow、黒と白の間でよりシャープなトランジションを作成できます。テクスチャを操作するとき、これはそれらの重なりを減らし、より鋭いエッジを作成します。
