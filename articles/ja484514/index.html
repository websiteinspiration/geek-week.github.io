Reactアプリケーションのユニバーサルルーティング

Webアプリケーションのフロントエンドのルーティング機能が何で構成されているかを簡単に説明しようとすると、人気のある各フレームワークがこれについてまったく異なる考えを持っているという結論に達することができます。同じフレームワークのバージョンを比較したとしても、関数とルーティングAPIは変更の影響を最も受けやすい（多くの場合、下位互換性がない）と結論付けることができます。たとえば、Reactのルーティングの4番目のバージョンは大幅に再設計されたため、githab.comで人気のあるプロジェクトの一部はこのバージョンに切り替わりませんでした。

これらすべての背景には、一般的な傾向があります。私の意見では、多くの一般的なフロントエンドフレームワークのルーティング機能が過負荷になっているということです。この点で、ルーティングから分離できる他のコンポーネント（ナビゲーション、履歴、リンクなど）と密接に接続されます。したがって、おそらくルーティングの使用が不快になり、その拡張が単に不可能であるという感覚に慣れている方も多いでしょう。柔軟で拡張可能なコンポーネントと比較して、人気のあるフロントエンドフレームワークでのルーティングは、あまり便利ではなく、拡張できないように見えます。これは、Reactのルーティングの最初のバージョン（4番目まで）に特に当てはまります。

この記事では、ルーティングに関するこの状況につながったいくつかの歴史的ポイントと、Reactと一緒にユニバーサルルーターライブラリを使用する方法について説明します。

ルーティングは必要ですか？


技術的には、1ページのWebアプリケーションはルーティングなしで機能します。たとえば、デスクトップアプリケーションにはルーティングがないためです。1ページのWebアプリケーションが同じWebブラウザアプリケーションのままでなければ、すべてがほぼ正常に機能します。つまり、ユーザーはF5キーを押すか、ブラウザの「再読み込み」アイコンをクリックすることで、いつでもページを更新できます。または、「左矢印」アイコンと「右矢印」アイコンをクリックするか、「Backspace」キーを押すと、いつでもストーリーを前後にスクロールできます。

したがって、単一ページのアプリケーションの場合、コンポーネントの変更とアプリケーションの内部状態の変更には、常にURLの変更が伴う必要があります。

なぜルーティングするのですか？


私の意見では、フロントエンドWebアプリケーションの一般的なフレームワークのルーティング機能は、（サーバーレンダリングを使用した）従来のWebアプリケーションのルーティングとの過去の接続に影響されます。

当初、URLは静的Webドキュメントのネットワークアドレスでしたが、非常にシンプルでした。次に、MVCアーキテクチャーのWebへの適応が始まりました：モデル1とモデル2。最後の2つはフロントコントローラーを含み、これは後で2つの部分に分割されます：ルーティング（目的のコントローラーを選択します）とコントローラー自体。モデルとレンダービュー。ご覧のように、従来のWebアプリケーションでは、ルーティングによってアクション（コントローラー）が決定され、間接的に（コントローラーを介して）サーバーでレンダリングされるビューが決定されます。

つまり、デスクトップアーキテクチャは、サーバー上の従来のWebアプリケーションで動作するように適応された後、ルーティングの形でWebアプリケーションの前面に戻り、サーバー側で必要な機能で重み付けされました。

ユニバーサルルーターライブラリは何を提供していますか？


ユニバーサルルーターライブラリは、不要なものをすべて破棄し、クライアントとWebサーバー側（ユニバーサル/同型Webアプリケーション）の両方でレンダリングするときに、フレームワークの有無にかかわらず使用できる部分のみを残すことを提供します。

時間のすべての階層を拒否したユニバーサルルーターは、明確に定義された機能を1つだけ提供します。行に基づいて（履歴、場所などのオブジェクトではなく、行を再度強調します）、非同期関数を呼び出します。非同期関数には、解析されたURL文字列が実際のパラメーターとして送信されます。それで全部です。Reactでは次のようになります。

import React from 'react';
import UniversalRouter from 'universal-router';
import App from './App';
import Link from './Link';

const routes =
    {
        path: '/',
        async action({next}) {
            const children = await next();
            return (
                <App>
                    {children}
                </App>
            );
        },
        children: [
            {
                path: '',
                async action() {
                    return (
                        <div>Root route go to <Link href='/test'>Test</Link></div>
                    );
                },
            },
            {
                path: '/test',
                async action({next}) {
                    const children = await next();
                    return (
                        <App>
                            {children}
                        </App>
                    );
                },
                children: [
                    {
                        path: '',
                        async action() {
                            return (
                                <div>Test route return to <Link href='/'>Root</Link></div>
                            );
                        },
                    },
                ]
            },
        ],
    };

export const basename = '';

const router = new UniversalRouter(routes, {
    baseUrl: basename
});

export default router;

ネストされたルートもサポートされています。それらは子フィールドで定義され、次の（）非同期関数を呼び出すことで取得できます。

そして、これはどのようにReactで機能しますか？


履歴のnavigate（）メソッドを定義しますが、多くの場合、ネイティブのpush（）メソッドを使用するだけで十分です

import { createBrowserHistory } from 'history'
import parse from 'url-parse'
import deepEqual from 'deep-equal'
const isNode = new Function('try {return this===global;}catch(e){return false;}') //eslint-disable-line
let history

if (!isNode()) {
  history = createBrowserHistory()
  history.navigate = function (path, state) {
    const parsedPath = parse(path)
    const location = history.location
    if (parsedPath.pathname === location.pathname &&
      parsedPath.query === location.search &&
      parsedPath.hash === location.hash &&
      deepEqual(state, location.state)) {
      return
    }
    const args = Array.from(arguments)
    args.splice(0, 2)
    return history.push(...[path, state, ...args])
  }
} else {
  history = {}
  history.navigate = function () {}
}

export default history

Linkコンポーネントも作成します。これにより、ナビゲーションが発生します。

import React from 'react';
import {basename} from './router';
import history from './history';

function noOp(){};

const createOnClickAnchor = (callback) => {
    return (e) => {
        e.preventDefault();
        history.navigate(e.currentTarget.getAttribute('href'));
        callback(e);
    };
};

export default ({href, onClick = noOp, children, ...rest}) => (
    <a
        href={basename + href}
        onClick={createOnClickAnchor(onClick)}
        {...rest}
    >
        {children}
    </a>
);
