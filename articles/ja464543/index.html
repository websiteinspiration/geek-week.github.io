プログラミングでは、型変換にはオブジェクトのデータ型を別のものに変更することが含まれます。暗黙的な変換とは、この変更がインタープリターまたはコンパイラーによって自動的に行われることを意味します。たとえば、以前に浮動小数点値が割り当てられていた変数にint値を割り当てます。この変換はGoでは使用できません。変数の宣言時に型が指定されていない場合、リテラルの構文構成に基づいて、int、float、stringなどの適切な型が割り当てられます。次の例では、Goは2つの異なるデータ型を検出し、それらを処理できないため、エラーをスローします。これは、Goコンパイラが暗黙的にに変換intするために発生しfloat64ます。

a := 1.0 // same as float64(1.0)
b := 1 // same as int(1)

fmt.Printf("%f", a*b) 
// invalid operation: a * b (mismatched types float64 and int)

コンストラクタとデストラクタ


デザイナのタスクはオブジェクトの主要な処理と初期化であり、デストラクタはオブジェクトの耐用年数とメモリの解放後にオブジェクトを破棄することです。他のオブジェクト指向のパラダイムとは異なり、Goにはクラスがありません。したがって、コンストラクタとデストラクタの概念も存在しません。

オペレーターのオーバーロード


オペレーターのオーバーロードは、オペレーターがユーザー定義の操作を実行できる方法です。演算子は、渡された引数に従って動作します。たとえば、C ++では、+演算子を使用して文字列を結合したり、2つの整数を追加したりできます。値「+」はユーザーが定義することもでき、プログラムのニーズに応じて変更できます。JavaScriptでは、文字列の優先順位が高いため、タイプ操作の'1' + 1結果として文字列が出力"11"されます。このような定義はGoでは許可されていません。演算子は厳密に機能し、引数の特定のデータ型でのみ操作を実行します。

デフォルト値


Goでは、関数プロトタイプのデフォルト、または関数がオーバーロードされている場合のデフォルトは許可されていません。Go言語の仕様は非常に小さく、特に構文解析を簡素化するためにサポートされています。他の言語とは異なり、デフォルト値/オプションのパラメーターを関数に渡すことができますが、Goでは値が渡されたかどうかしか確認できません。Goのデフォルト値への別のアプローチは次のようなものです。

func Concat1(a string, b int) string {

  if a == "" {
    a = "default-a"
  }
  if b == 0 {
    b = 5
  }

  return fmt.Sprintf("%s%d", a, b)
}

継承


Goはおなじみのオブジェクト指向プログラミングクラスの階層に従っていないため、Goの構造は互いに継承されません。一般に、継承は、1つのクラスがその親のプロパティとクラスメソッドを継承するOOP言語の手順です。継承にはいくつかのレベルがあります。ただし、Goでは、ポインターを提供したり、協調的な構造に埋め込んだりすることで、構造を簡単に構築できます。Goの構成例を以下に示します。Goでは、クラスをインターフェースに置き換えることができます。インターフェースは他の言語でも存在しますが、Goインターフェースは暗黙のうちに満足されます。

type TokenType uint16

type Token struct {
  Type TokenType
  Data string
}

type IntegerConstant struct {
  Token *Token
  Value uint64
}

一般的なプログラミング


ジェネリックプログラミングは、ジェネリックスと呼ばれるパターンをプラグインする形式です。これは、実際のソースコードではありませんが、コンパイラーによってコンパイルされ、ソースコードに変換されます。簡単にパターンを理解してみましょう。プログラミングのパターンをフォームと考えてください。テンプレートの重要な詳細は空白のままにして、後でコンパイル中に入力する必要があるフォームを作成します。次に、このテンプレートから何かを作成する必要がある場合は、タイプなどの詳細を指定するだけです。

template<typename T>
class MyContainer
{
    // Container that deals with an arbitrary type T
};

void main() 
{
    // Make MyContainer take just ints.
    MyContainer<int> intContainer;
}

上記のスニペットはC ++で書かれています。テンプレートにはタイプはありませんが、MyContainerが初期化されるときに提供されます。必要に応じてfloat、doubleなどの他のタイプを指定することもできます。一般化されたテンプレートは、いくつかのタイプのデータのセットに対してアルゴリズムを実行するときに役立ちます。

例外


例外は、アプリケーションが代行受信する必要のある合理的な状態を示します。例外を通じて、プログラムが機能しない状況を解決できます。チェックされた例外は、実行の完全な停止につながるわけではなく、傍受されて処理される可能性があります。Goには例外はなく、インターフェースや組み込みエラーなどのエラーのみがあります。エラーと例外の主な違いは、重大な問題を示し、すぐに解決する必要があるため、Goプログラミングがより厳格になることです。Goのエラーは、発生時に明示的にチェックする必要があります。

マクロ


マクロはマクロコマンドです。これは、特定の入力セットに対して事前定義された出力を定義することにより、反復的なプログラミングタスクを最小限に抑える方法です。たとえば、Cで数値の2乗を取得するx * x場合xは、変数が変数であると書くだけですが、必要になるたびに数値の2乗を返すマクロを定義することもできます。マクロは関数ではありません。Goではマクロは使用できません。

#define square(x) ((x) * (x))

int main() {
    int four = square(2);  // same as 2 * 2
    return 0;
}

機能の注釈


アノテーションは、メタデータを関数パラメーターに関連付ける方法です。Pythonでは、アノテーションは構文的にサポートされており、完全にオプションです。Pythonで注釈を記述するための小さな例を見てみましょう。

def foo(a: int, b: 'description', c: float) -> float: print(a+b+c)

foo(1, 3, 2)  // prints 6.0
foo('Hello ', , 'World!')  // prints Hello World!

上記のコードでは、パラメーターはaでbありc、いくつかのメタデータで注釈が付けられています。aおよびc種類によって注釈付きintとfloatしながら、b文字列の説明を提供します。foo注釈で言及されている引数のタイプに関係なく、特定の出力を出力します。

ローカルストリームストレージ


ローカルストリームストレージは、ストリームにローカライズされた静的またはグローバルメモリを使用するコンピュータープログラミング方式です。これは、プログラム内のストリームごとにデータがコピーされる静的領域です。複数のストリームが同じタスクで同じ静的データを使用する場合、それらは自分で保存するのではなく、ローカルストリームストレージからコピーできます。

結論


囲碁はシンプルさと優雅さに焦点を当てています。構文は高速で、小さく、単純です。他のOOP言語とは異なり、ヘッドに配置する必要のある概念は少なくなっています。Goの作成者は、言語の隣接部分に乗算的な複雑さを追加することなく、言語のシンプルさを選択しました。その結果、Goにはパーサーを遅く、大きくする機能がありません。シンプルさは優れたソフトウェアの鍵です。
