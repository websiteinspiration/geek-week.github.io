コードベース。RecyclerViewの拡張



こんにちは！



私の名前は、オメガRのシニアAndroid開発者であるアントンクニャゼフです。私は過去7年間、モバイルアプリケーションを専門的に開発し、ネイティブ開発の複雑な問題を解決してきました。



私たちのチームと私が開発したRecyclerView拡張メソッドを共有したいと思います。これらは、アプリケーションでカスタムリストを作成するための信頼できるベースになります。



各アプリケーションは、そのアイデア、デザイン、およびスペシャリストのチームにより、独自の方法でユニークです。多くの場合、成功した決定は、あるプロジェクトから別のプロジェクトに転送されます。したがって、単にコピーするのではなく、チーム全体が使用して改善する個別のライブラリを作成するのが合理的です。



チームは、アプリケーション開発を改善および加速する小さなライブラリを作成し、それらをパブリックGitHubリポジトリにアップロードすることを決定しました。これにより、JitPackを介してプロジェクト内のライブラリを簡単に接続できるようになり、コードに「犯罪者」がいないことが保証されます。GitHubに



最初に投稿したライブラリは、RecyclerViewの単純な拡張です。



彼女が解決した問題から始めましょう：



  1. デフォルトのlayoutManagerはありません。同じLinearLayoutManagerを選択する必要があることが多いため、これは不便です。
  2. xmlを使用してディバイダーとアイテムスペースを追加する方法はありません。これは、アイテムレイアウトまたはItemDecoratorのいずれかを追加する必要があるため、不便です。
  3. 単にxmlを介してヘッダーとフッターを追加することはできません。これは、別のViewHolderを通じてのみ可能です。


問題は重大ではありませんが、不便を生じ、開発時間を増加させます。



1.問題：デフォルトのlayoutManagerがない



RecyclerViewの開発者は、デフォルトのLayoutManagerを選択する機能を提供していませんでした。次の方法で、layoutManagerを設定でき



ます。1. app属性のXMLを使用：layoutManager =” LinearLayoutManager”： 



<?xml version="1.0" encoding="utf-8"?>
<androidx.recyclerview.widget.RecyclerView
   ...
    app:layoutManager="LinearLayoutManager"/>


2.コードを通じて： 



recyclerView.layoutManager = LinearLayoutManager(this)


私たちの経験では、ほとんどの場合、LinearLayoutManagerが必要です。



ITProTV、Simple World、およびDexenアプリケーションからのそのようなリストの例をいくつか示します。







解決策：デフォルトのlayoutManagerを追加します



OmegaRecyclerViewに追加されるのは3行のみです。 



 if (layoutManager == null)  {
            layoutManager = LinearLayoutManager(context, attrs, defStyleAttr, 0)
 }


したがって、LinearLayoutManagerが必要な場合、何も追加する必要はありません。つまり、layoutManagerを忘れることができます。 



<?xml version="1.0" encoding="utf-8"?>
<com.omega_r.libs.omegarecyclerview.OmegaRecyclerView
    android:id="@+id/recyclerview"
    android:layout_width="match_parent"
    android:layout_height="match_parent" />


2.問題：XMLを介してディバイダーとアイテムスペースを追加する方法がない



RecyclerViewを使用する場合、分周器を頻繁に追加する必要があります。たとえば、プロジェクト「シンプルワールド」では、画面の1つにこのような非標準の仕切りがありました。







このレイアウトから、次のことがわかります。



  • 要素間および最端で使用される仕切り。
  • 中古品スペース。


これをAndroidに標準的な方法で実装するにはどうすればよいですか？



方法1



最も明白な方法は、ImageView要素としてディバイダーを含めることです。 



 <RelativeLayout
   ...
   android:paddingStart="20dp"
   android:paddingTop="12dp"
   android:paddingEnd="20dp"
   android:paddingBottom="12dp">

...

   <ImageView
       ...
       android:layout_alignParentBottom="true"
       android:src="@drawable/divider"/>

</RelativeLayout>


要素間でのみ除算を行う必要がある場合があります。この場合、最後の仕切りを削除し、アダプターで非表示にするコードを追加する必要があります。



方法2



別の方法は、このディバイダーが描画できるDividerItemDecorationを使用することです。そのためには、さらにドロアブルを作成する必要があります。 



<?xml version="1.0" encoding="utf-8"?>
<layer-list xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:left="32dp">

        <shape android:shape="rectangle">

            <size
                    android:width="1dp"
                    android:height="1dp" />

            <solid android:color="@color/gray_dark" />

        </shape>

    </item>

</layer-list>


インデントを追加するには、ItemDecorationを記述する必要があります。 



class VerticalSpaceItemDecoration(private val verticalSpaceHeight: Int): RecyclerView.ItemDecoration {

    override fun getItemOffsets(outRect: Rect, view: View, parent: RecyclerView,
            state: RecyclerView.State) {
        outRect.bottom = verticalSpaceHeight
    }

}


DividerItemDecorationは単純です。常に各リスト項目の下に仕切りを描画します。

ただし、要件が変更された場合は、別のソリューションを探す必要があります。



解決策：XMLを介してディバイダーとアイテムスペースを追加する機能を追加する



したがって、OmegaRecyclerViewは次の属性を使用してディバイダーを追加できるはずです。



  1. divider – drawable, ;
  2. dividerShow (beginning, middle, end) – , , ;
  3. dividerHeight – divider, ;
  4. dividerPadding, dividerPaddingStart, dividerPaddingEnd – : , , ;
  5. dividerAlpha – ;
  6. itemSpace – .


これらの属性はすべて、2つの特別なItemDecorationを使用してOmegaRecyclerViewに直接適用されます。



ItemDecorationの1つはアイテム間にパディングを追加し、2つ目はディバイダー自体を描画します。要素間にインデントがある場合、2番目のItemDecorationはインデントの中央に仕切りを描画することに注意してください。すべての可能なLayoutManagerオプションをサポートする方法はないため、LinearLayoutManagerおよびGridLayoutManagerのみをサポートします。 LinearLayoutManagerのリストの方向にも注意してください。



コードを簡略化するために、方向と順序（逆または直接）に応じて、Rectで相対データを読み書きする特別なDividerDecorationHelperを選択します。これには、メソッドsetStart、setEndおよびgetStart、getEndがあります。



次の一般的なロジックを含む2つのクラスの基本的なItemDecorationを作成してみましょう。



  1. layoutManagerがLinearLayoutManagerの子孫であることを確認します。
  2. 現在の方向とシーケンスの計算。
  3. 適切なDividerDecorationHelperを決定します。


SpaceItemDecorationでは、インデントを追加する1つのgetItemOffsetメソッドのみをオーバーライドします。 



override fun getItemOffset(outRect: Rect, parent: RecyclerView, helper: DividerDecorationHelper, position: Int, itemCount: Int) {
        if (isShowBeginDivider() || countBeginEndPositions <= position) helper.setStart(outRect, space)
        if (isShowEndDivider() && position == itemCount - countBeginEndPositions) helper.setEnd(outRect, space)
    }


次のDividerItemDecorationは、ディバイダーを直接描画します。要素間のインデントを考慮し、中央に仕切りを描画する必要があります。まず、インデントが指定されていない場合のgetItemOffsetメソッドを再定義しますが、描画には仕切りが必要です。 



    override fun getItemOffset(outRect: Rect, parent: RecyclerView, helper: DividerDecorationHelper, position: Int, itemCount: Int) {
        if (position == 0 && isShowBeginDivider()) {
            helper.setStart(outRect, dviderSize)
        }
        if (position != 0 && isShowMiddleDivider()) {
            helper.setStart(outRect, dividerSize)
        }
        if (position == itemCount - 1 && isShowEndDivider()) {
            helper.setEnd(outRect, dividerSize)
        }
    }


また、DividerItemDecorationが、選択したアイテムの上または下に描画できるかどうかをアダプターに確認できるオプションを追加します。この機能を実装するには、次のメソッドを使用して、標準のアダプターから継承する独自のアダプターを作成します。 



   open fun isDividerAllowedAbove(position: Int): Boolean {
        return true
    }

    open fun isDividerAllowedBelow(position: Int): Boolean {
        return true
    }


次に、onDrawOverメソッドを再定義して、描画された要素の上にディバイダーを描画します。このメソッドでは、画面に表示されているすべての要素を（getChildAtを介して）確認し、必要に応じてこの仕切りを描画する必要があります。高さのない色は、dividerDrawable属性から取得できることにも注意してください。この場合、高さはdividerHeight属性から取得できます。



3.問題：xml経由でヘッダーとフッターを直接追加できない







RecyclerViewでは、xml経由でビューを追加することはできませんが、これを行う方法は他にもあります。



方法1



ビューを追加する1つの明白な方法は、アダプターを使用することです。さらに、viewTypeのIDを入力するときは、アダプターのヘッダーとフッターを区別する必要があります。 



 fun onCreateViewHolder(parent: ViewGroup, viewType: Int): RecyclerView.ViewHolder? {
	  val inflater = LayoutInflater.from(parent.context)
        return when (viewType) {
		 TYPE_HEADER -> {
                val headerView: View = inflater.inflate(R.layout.item_header, parent, false)
                HeaderViewHolder(itemView)
            }
            TYPE_ITEM -> {
                val itemView: View = inflater.inflate(R.layout.item_view, parent, false)
                ItemViewHolder(itemView)
            }
           
            else -> null
        }
    }


方法2



少し異なる方法ですが、アダプターを介しても。recyclerview：1.2.0-alpha02から、MergeAdapterが登場しました。これにより、複数のアダプターを1つに結合して、コードをよりクリーンにすることができます。 



val mergeAdapter = MergeAdapter(headerAdapter, itemAdapter,  footerAdapter)
recyclerView.adapter = mergeAdapter


解決策：xmlを使用してヘッダーとフッターを追加する機能を追加する



最初に行うことは、膨張プロセスの進行中にOmegaRecyclerViewでのビューの追加をインターセプトすることです。これを行うには、addViewメソッドをオーバーライドし、すべてのヘッダーとフッターのビューを自分に追加します。このメソッドは、表示可能なリスト項目を追加するためにRecyclerView自体によって使用されます。ただし、xmlを介して追加されたビューにはViewHolderがなく、最終的にNullPointerExceptionがスローされます。



したがって、インフレ中にビューが追加されるタイミングを決定する必要があります。さいわい、保護されたonFinishInflateメソッドがあり、拡張プロセスの終了時に呼び出されます。したがって、このメソッドを呼び出すと、膨張プロセスが完了したことがマークされます。 



  protected override fun onFinishInflate() {
        super.onFinishInflate()
        finishedInflate = true
    }


したがって、addViewメソッドは次のようになります。 



  override fun addView(view: View, index: Int, params: ViewGroup.LayoutParams) {
        if (finishedInflate) {
            super.addView(view, index, params)
        } else {
            // save header and footer views
        }
    }


次に、これらすべての追加ビューを覚えて、それらをMergeAdapterとして特別なアダプターに転送する必要があります。



また、findViewByIdメソッドが呼び出されたときにビューが返されないという別の問題も解決しました。この問題を解決するには、findViewTraversalメソッドを再定義します。見つかったビューのIDを比較し、一致する場合はビューを返す必要があります。このメソッドは非表示であるため、オーバーライドであることを示すことなく、単純に記述します。OmegaRecyclerView



ライブラリの詳細な説明で、これらの機能やその他の便利な機能について知ることができます。



  1. 2018年に、RecyclerViewからViewPagerを作成しました。さらに、ViewPagerには無限のスクロールがあります。
  2. ExpandableRecyclerView-ドロップダウンリストを追加するための特別なクラスで、拡張アニメーションを選択できます。
  3. StickyHeaderは、アダプターを介して追加できる特定のリスト項目です。


これらはすべて、Omega-Rの経験の結果です。開発者のスキルの進化にはいくつかの段階があります。まず、別のプロジェクトからコードをコピーするか、それに類似したことを行います。次に、蓄積された経験を記録して別のリポジトリを作成する必要がある段階になります。



次の段階では、プロジェクトで行うことをためらった機能を意図的に作成し始めます。これにはかなりの時間がかかる場合がありますが、将来のための予備を作成し、新しいプロジェクトの開発をスピードアップすることができます。開発で困難に遭遇したすべての人に、 Omega-R GitHubリポジトリにある私たちのソリューションを知ってもらうことを勧めます。
