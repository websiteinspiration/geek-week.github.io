Docker Compose：Makefileを使用して簡素化する

数年ごとに、ソフトウェア開発業界でパラダイムシフトが発生しています。これらの現象の1つは、マイクロサービスの概念への関心の高まりです。マイクロサービスは最新のテクノロジーではありませんが、最近その人気が文字通り急上昇しました。

最近の大規模なモノリシックサービスは、独立した自律型マイクロサービスに置き換えられています。マイクロサービスは、単一の非常に特定の目的を果たすアプリケーションと考えることができます。たとえば、リレーショナルDBMS、Expressアプリケーション、Solrサービスなどです。 今日では、マイクロサービスを使用しない新しいソフトウェアシステムの開発を想像することは困難です。そしてこの状況は、今度はDockerプラットフォームにつながります。





Docker


Docker プラットフォームは、マイクロサービスの開発とデプロイにおいて、ほぼ業界標準になっています。プロジェクトのウェブサイトで、Dockerが唯一の独立したコンテナー化プラットフォームであり、組織があらゆるアプリケーションを簡単に作成し、ハイブリッドクラウドからボーダーシステムまで、あらゆる環境で配布および実行できることがわかります。

Docker作成


Docker Compose テクノロジーは、マルチコンテナーアプリケーションを構成するように設計されています。Docker Composeプロジェクトには、このプロジェクトの作成者が必要とするだけの数のDockerコンテナーを含めることができます。

Docker Composeを使用する場合、YAMLファイルを使用してアプリケーションサービスを構成し、相互の相互作用を整理します。したがって、Docker Composeは、Dockerマルチコンテナアプリケーションを記述して実行するためのツールです。


ホストシステムで実行されている2つのコンテナー

GNU Make


プログラムmakeは基本的に、ソースコードからのプログラムとライブラリのアセンブリを自動化するためのツールです。一般に、make特定のソースマテリアルを特定の結果のフォームに特定の目標に変換するための任意のコマンドの実行を含むすべてのプロセスに適用できると言えます。私たちの場合、チームdocker-composeは抽象的な目標（Phonyターゲット）に変換されます。

プログラムmakeに必要なものを伝えるために、ファイルが必要ですMakefile。

我々は、Makefile通常のコマンドが含まれていますdockerし、docker-compose多くの問題を解決するように設計されています。つまり、コンテナーの組み立て、開始、停止、再起動、ユーザーのコンテナーへのログインの整理、コンテナーログの操作、その他の同様の問題の解決について話し合っています。

Docker Composeの一般的な使用法


次のコンポーネントを持つ通常のWebアプリケーションを想像してください。

  • TimescaleDBデータベース（Postgres）。
  • Express.jsアプリケーション。
  • Ping（特別なことを何もしない単なるコンテナー）。

このアプリケーションには、3つのDockerコンテナーと、docker-composeこれらのコンテナーを管理するための指示を含むファイルが必要です。各コンテナには、異なる相互作用ポイントがあります。たとえばtimescale、データベースを操作するのとほとんど同じ方法でコンテナを操作できます。つまり、次のアクションを実行できます。

  • Postgresシェルにログインします。
  • テーブルをインポートおよびエクスポートします。
  • pg_dumpテーブルまたはデータベースを作成します。

Express.jsアプリケーションのコンテナにexpressjsは、次の機能があります。

  • システムログからの新しいデータの出力。
  • シェルに入り、特定のコマンドを実行します。

コンテナの相互作用


Docker Composeを使用してコンテナー間の接続を構成した後、これらのコンテナーとの対話を確立します。Docker Composeシステムの一部として、ファイルをシステムに転送できるdocker-composeオプション-fをサポートするコマンドがありますdocker-compose.yml。

このオプションの機能を使用して、システムとの対話を、ファイルに記載されているコンテナのみに制限できますdocker-compose.yml。

コマンドを使用したときのコンテナの相互作用の様子を見てみましょうdocker-compose。シェルに入る必要があると想像するとpsql、対応するコマンドは次のようになります。

docker-compose -f docker-compose.yml exec timescale psql -Upostgres

同じではないdocker-composeが使用されている同じコマンドは、次のdockerようになります。

docker exec -it  edp_timescale_1 psql -Upostgres

このような場合、コマンドdockerではなくコマンドを使用docker-composeすることをお勧めします。これにより、コンテナの名前を覚えておく必要がなくなるためです。

上記のコマンドはどちらもそれほど複雑ではありません。しかしMakefile、単純なコマンドの形式でインターフェースを提供し、そのような長いコマンドを自分で呼び出すような形式で「ラッパー」を使用した場合、同じ結果が次のように得られます。

make db-shell

これを使用するMakefileと、コンテナーでの作業が大幅に簡略化されます。

実施例


上記のプロジェクト図に基づいて、次のファイルを作成しますdocker-compose.yml。

version: '3.3'
services:
    api:
        build: .
        image: mywebimage:0.0.1
        ports:
            - 8080:8080
        volumes:
            - /app/node_modules/
        depends_on:
            - timescale
        command: npm run dev
        networks:
            - webappnetwork
    timescale:
        image: timescale/timescaledb-postgis:latest-pg11
        environment:
          - POSTGRES_USER=postgres
          - POSTGRES_PASSWORD=postgres
        command: ["postgres", "-c", "log_statement=all", "-c", "log_destination=stderr"]
        volumes:
          - ./create_schema.sql:/docker-entrypoint-initdb.d/create_schema.sql
        networks:
           - webappnetwork
    ping:
       image: willfarrell/ping
       environment:
           HOSTNAME: "localhost"
           TIMEOUT: 300
networks:
   webappnetwork:
       driver: bridge

Docker Compose構成を管理し、それが説明するコンテナーと対話するには、次のファイルを作成しますMakefile。

THIS_FILE := $(lastword $(MAKEFILE_LIST))
.PHONY: help build up start down destroy stop restart logs logs-api ps login-timescale login-api db-shell
help:
        make -pRrq  -f $(THIS_FILE) : 2>/dev/null | awk -v RS= -F: '/^# File/,/^# Finished Make data base/ {if ($$1 !~ "^[#.]") {print $$1}}' | sort | egrep -v -e '^[^[:alnum:]]' -e '^$@$$'
build:
        docker-compose -f docker-compose.yml build $(c)
up:
        docker-compose -f docker-compose.yml up -d $(c)
start:
        docker-compose -f docker-compose.yml start $(c)
down:
        docker-compose -f docker-compose.yml down $(c)
destroy:
        docker-compose -f docker-compose.yml down -v $(c)
stop:
        docker-compose -f docker-compose.yml stop $(c)
restart:
        docker-compose -f docker-compose.yml stop $(c)
        docker-compose -f docker-compose.yml up -d $(c)
logs:
        docker-compose -f docker-compose.yml logs --tail=100 -f $(c)
logs-api:
        docker-compose -f docker-compose.yml logs --tail=100 -f api
ps:
        docker-compose -f docker-compose.yml ps
login-timescale:
        docker-compose -f docker-compose.yml exec timescale /bin/bash
login-api:
        docker-compose -f docker-compose.yml exec api /bin/bash
db-shell:
        docker-compose -f docker-compose.yml exec timescale psql -Upostgres

ここで説明するコマンドのほとんどはすべてのコンテナーに適用されますが、オプションをc=使用すると、コマンドのスコープを1つのコンテナーに制限できます。
