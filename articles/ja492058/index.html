ROS2 vs ROS1。Ubuntu 18.04へのROS2のインストール

ROS（ロボット操作システム）は、ロボット工学の標準において自信を持って主導的地位を占めています。有名な政治家の言葉では、「すべてではない」がすでにそれを使用しています。
この記事では、ROS開発の次の段階である、より「洗練された」開発者に適したROS2システムについて見ていきます。両方のシステムを一般的な用語で比較し、同時にROS2をUbuntu 18.04にインストールします。

前書き。


過去10年間で、ROS（ROS1）は成長し、数回成長しています。現在、
彼はパッケージの膨大なリストを持っています。各パッケージは問題を部分的または
完全に解決するため、ホイールの発明の概念が再びなくなります。コミュニティによって作成されたパッケージは、ロボット工学への完全に新しいアプローチの作成と、既存のシステムのための提案されたインテリジェントな充填をもたらしました。

ROS1の制限事項。


ROS1は複雑なハードウェアおよびソフトウェアコンポーネントとの通信の特定の自由を与えたという事実にもかかわらず、最終製品でのROS1の使用に関連するいくつかの困難があります。これらの問題は、異種ロボット（移動ロボット、ロボットマニピュレータなど）が多数存在するときに発生し始めます。
それらの間の関係を確立することは、ROS1のアーキテクチャ上の制限のために非常に困難です。
最初の制限は、ROS1がマルチマスターの概念をサポートしないことです。
ROS1の2番目の制限は、ノードが問題なく互いに通信できるという事実にもかかわらず、この通信には安全な方法がないということです。マスター（マスターノード）に接続する権限を持つユーザーはだれでもトピックにアクセスし、トピックを利用または変更できます。したがって、ROS1の主な目的は、科学的な目的でコンセプトまたはプロトタイプをテストすることです。
そして3つ目は、おそらくROS1の主な制限は、このシステムがリアルタイムシステムではないことです。これにより、データの受信が遅れ、データが失われます。
上記のすべての欠点がROS2の作成に押しやられました。

ROS2の利点は何ですか？


-システムコンポーネント（ノード）間の安全な接続を確立できます
-相互作用はリアルタイムです-1
つのネットワークへの複数のロボットの接続が簡素化
されます
- ノード間の通信の品質が向上します-ROSアーキテクチャレベルはハードウェアに直接実装されます
-最新のパッケージが使用されます更新（ROS1は頻繁に更新されません）。

ROS2は業界標準に準拠し、いわゆるDDS（データ分散サービス）の概念を通じてリアルタイムの対話を実装します。

ROS2バージョン

。
ROS2の最初の公式安定バージョンは2017年12月にリリースされました。ディストリビューションは「Ardent Apalone」と名付けられました。その後、2018年6月に「Bouncy Bolson」という名前のリリースがありました。Crystal Clemmysは2018年12月に登場しました。DashingDiademata（現在のバージョン）は2019年5月に公開されています。ご覧のように、ディストリビューションはかなり元気に更新されています。

OSサポート

。

ROS2はROS1と同じ動作原理を使用します。同時に、ROS2はROS1の一部ではありません。 ROS2は、新しい依存関係とツールを使用して仲間の限界を超えるために作成されました。
スタックがC ++で記述されたROS1と、C ++およびPythonのクライアントライブラリとは異なり、ROS-2もC言語を使用します。rclcpp、rclpy、 rcljava。

ROS2はネットワーク構成をより適切にサポートし、信頼できる接続を実装します。
ROS2はノードレット（http://wiki.ros.org/nodelet）の概念も使用せず、同時に複数のノードを初期化する機能（マルチノード初期化）をサポートします。ROS1とは異なり、ROS2のもう1つの特徴的な機能は、ノードのステータスを追跡し、ノードまたはトピックが追加または削除された場合にメッセージを受信する機能です。これは、フォールトトレラントシステムを作成するために使用できます。

ROS2をインストールする

。
そのため、Ubuntu 18.04上のDashing Diademata（ダッシュ）の現在のディストリビューションをソースからインストールします

環境を設定します。


$ sudo locale-gen en_US en_US.UTF-8
$ sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
$ export LANG=en_US.UTF-8


ROS2リポジトリを追加します。


$ sudo apt update && sudo apt install curl gnupg2 lsb-release
$ curl -s https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc |
sudo apt-key add -
$ sudo sh -c 'echo "deb [arch=amd64,arm64] http://packages.ros.org/ros2/ubuntu `lsb_release -cs` main" > /etc/apt/sources.list.d/ros2-latest.list'


ROS開発者ツールをインストールします。


$ sudo apt update && sudo apt install -y build-essential cmake git
python3-colcon-common-extensions python3-lark-parser python3-pip
python-rosdep python3-vcstool wget
$ python3 -m pip install -U argcomplete flake8 flake8-blind-except
flake8-builtins flake8-class-newline flake8-comprehensions flake8-
deprecated flake8-docstrings flake8-import-order flake8-quotes
pytest-repeat pytest-rerunfailures pytest pytest-cov pytest-runner
setuptools
$ sudo apt install --no-install-recommends -y libasio-dev libtinyxml2-dev


ワークスペースROS2を作成します。


$ mkdir -p ~/ros2_ws/src
$ cd ~/ros2_ws
$ wget https://raw.githubusercontent.com/ros2/ros2/release-latest/ros2.repos
$ vcs import src < ros2.repos

しばらく時間がかかります。

rosdepを使用して依存関係をインストールします。


$ sudo rosdep init
$ rosdep update
$ rosdep install --from-paths src --ignore-src --rosdistro dashing -y --skip-keys "console_bridge fastcdr fastrtps libopensplice67 libopensplice69
rti-connext-dds-5.3.1 urdfdom_headers"

すべてがうまくいき、ターミナルに「すべての必要なrosdepsが正常にインストールされました」というエントリが表示されたら、次のステップに進むことができます。

コードを収集し、colconをインストールします。


ROS1では、パッケージはcatkinツールを使用して収集されました。ROS2はcolcon（catkin_make、catkin_make_isolated、catkin_tools、およびament_toolsを含むユーティリティ）を使用します。colconは、ROS1パケットとROS2パケットの両方を収集できます。 最後のコマンドの後、お茶を飲みに安全に行くことができます。インストールには1時間ほどかかる場合があります。 正常に完了すると、インストールが成功したことに関するおおよそ次の情報がターミナルに表示されます。
$ sudo apt install python3-colcon-common-extensions
$ cd ~/ros2_ws/
$ colcon build --symlink-install






ROS2の環境をセットアップします（環境）。


ターミナルでROSを開始するたびに「source〜/ ros2_ws / install / local_setup.bash」という長いコマンドを入力しないようにするには、bashrcにエイリアスを追加します。
$ sudo gedit ~/.bashrc

次に、ファイルの最後に行を追加します。ファイルを
alias initros2='source ~/ros2_ws/install/local_setup.bash

保存して終了し、コマンドでもう一度読み取ります再起動システム）：
$ source ~/.bashrc

ROS2の動作を確認してください。
