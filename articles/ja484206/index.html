Dartでのミックスインの使用

Dart言語のミックスイン（不純物）がなぜ必要なのかが明確ではないという質問が同僚から何度か寄せられました。私はこの問題についてインターネット上に何があるかを確認することにしました。見つけることができた記事の悔しさに、彼らは主に不純物の使用方法について話しますが、それらがなぜ必要なのかについては説明していません。この記事は、このギャップを埋める試みです。


ダートとフラッターの不純物のトピックに関するインターネット上の記事が十分あるという事実にもかかわらず、与えられた例は不純物からクラスを構築する純粋なメカニズムを示しているため、それらは私の意見を明確にするものではありません。 。特に、私はそのような例に出会いました。我々は持っています：


class Animal {}
class Dog {}
class Cat {}

そして、何らかの理由で、猫と犬の特性を同時に備えた動物を取得したかったのです。この場合、これを行うことができます：


class CatDog extends Animal with Cat, Dog {}

この例には少なくとも2つの質問があります。


  • なぜ猫と犬の間に十字架が必要なのですか？
  • 猫と犬はなぜ継承しAnimalないのですか？動物じゃないの？

abstract class Paintable {
  void paint(Color color);
  Color get color;
}

class PaintableRect extends Rectangle implements Paintable {
  Color _color;
  @override
  void paint(Color color) {_color = color;}

  @override
  Color get color => _color;
}

class PaintableContainer extends Container implements Paintable {

  void paint(Color color) {_color = color;}

  @override

, , , (mixin):


enum Color {red, yellow, green}

mixin PaintableMixin {
  Color _color;
  void paint(Color color) {_color = color;}
  Color get color => _color;  
}

class PaintableRect extends Rectangle with PaintableMixin {
  @override
  void draw() {
    print('Draw rectangle with color $color');
  }
}

class PaintableContainer extends Container with PaintableMixin {
  @override
  void render() {
    print('Render container with color $color');
  }
}

:


main() {
  PaintableRect()
    ..paint(Color.red)
    ..draw();
  PaintableContainer()
    ..paint(Color.yellow)
    ..render();
}
