Terraformを使用したvm ESXiのクイックデプロイ

みなさん、こんにちは。私の名前はイワンです。私はアルコール依存症のシステム管理者（OPS）です。

Terraformを使用してvCenterなしでESXiに仮想マシンをデプロイする方法をお伝えします。

多くの場合、このアプリケーションまたはそのアプリケーションをテストするには、仮想マシンを展開または再作成する必要があります。怠惰なので、プロセスの自動化について考えました。私の検索は素晴らしいに私を導いたhashicorpの製品、テラフォーム。

多くの人はTerraformが何であるかを知っており、誰も知らないが、これはIasC（コードとしてのインフラストラクチャ）の概念を使用してクラウド、インフラストラクチャ、またはサービスを管理するためのアプリケーションです。

仮想化環境として、ESXiを使用しています。非常にシンプルで、便利で、信頼性があります。
質問があると思います。
vCenter Serverを使用できるのになぜテレフォームするのですか？
もちろんできますが。第一に、これは追加のライセンスです。第二に、この製品は非常に多くのリソースを必要とし、私のホームサーバーに適合しません。第三に、スキルをアップグレードする能力です。

サーバーはIntel NUCプラットフォームです。

CPU: 2 CPUs x Intel(R) Core(TM) i3-4010U CPU @ 1.70GHz
RAM: 8Gb
HDD: 500Gb
ESXi version: ESXi-6.5.0-4564106-standard (VMware, Inc.)

だから、まず最初に。

ここでは、esxiを設定します。つまり、ファイアウォール設定でVNCポートを開きます。

デフォルトでは、ファイルは書き込み保護されています。次の操作を実行します。

chmod 644 /etc/vmware/firewall/service.xml
chmod +t /etc/vmware/firewall/service.xml
vi /etc/vmware/firewall/service.xml

次のブロックをファイルの最後に追加します。

<service id="1000">
  <id>packer-vnc</id>
  <rule id="0000">
    <direction>inbound</direction>
    <protocol>tcp</protocol>
    <porttype>dst</porttype>
    <port>
      <begin>5900</begin>
      <end>6000</end>
    </port>
  </rule>
  <enabled>true</enabled>
  <required>true</required>
</service>

終了して保存します。権限を元に戻し、サービスを再起動します。

chmod 444 /etc/vmware/firewall/service.xml
esxcli network firewall refresh

この場合、仮想マシンに接続し、キックスタートファイルへのパスを指定するには、VNCが必要です。

実際にホストを再起動する前。その後、この操作を繰り返す必要があります。

さらに、同じサーバー上の仮想マシンですべての作業を実行します。

特徴：

OS: Centos 7 x86_64 minimal
RAM: 1GB
HDD: 20GB
Selinux: disable
firewalld: disable

次に、HashiCorpの製品であるpackerが必要です。

「ゴールデン」イメージの自動アセンブリに必要です。将来使用する予定です。

yum install unzip git -y
curl -O https://releases.hashicorp.com/packer/1.5.5/packer_1.5.5_linux_amd64.zip
unzip packer_1.5.5_linux_amd64.zip -d /usr/bin && rm -rf packer_1.5.5_linux_amd64.zip
packer version
Packer v1.5.5

同じ名前のパッケージがRedHatベース である可能性があるため、packer versionステップでエラーが発生する場合があります。

which -a packer
/usr/sbin/packer

解決するには、シンボリックリンクを作成するか、絶対パス/ usr / bin / packerを使用します。

次に、ovftool ダウンロードリンクが必要です。ダウンロードしてサーバーに置き、インストールします。

次へ。

ギータの上で、必要なものをすべて用意しました。

git clone https://github.com/letnab/create-and-deploy-esxi.git && cd create-and-deploy-esxi

iso フォルダーには、オペレーティングシステムのディストリビューションを配置する必要があります。私の場合はcentos 7

です。centos-7-base.jsonファイルを編集することも必要です。

variables:     
iso_urls:  
iso_checksum:

すべての変更後、アセンブリを実行します。

/usr/bin/packer build centos-7-base.json

このプロセスには7〜8分かかります。

正常に完了すると、ovaファイルはフォルダーoutput-packer-centos7-x86_64に配置されます。

Terraformをインストールします。

curl -O https://releases.hashicorp.com/terraform/0.12.24/terraform_0.12.24_linux_amd64.zip
unzip terraform_0.12.24_linux_amd64.zip -d /usr/bin/ && rm -rf terraform_0.12.24_linux_amd64.zip
terraform version
Terraform v0.12.24

TerraformにはESXiのプロバイダーがないため、それをビルドする必要があります。

置く：

cd /tmp
curl -O https://dl.google.com/go/go1.14.2.linux-amd64.tar.gz
tar -C /usr/local -xzf go1.14.2.linux-amd64.tar.gz && rm -rf go1.14.2.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin
go version
go version go1.14.2 linux/amd64

次に、プロバイダーを収集します。

go get -u -v golang.org/x/crypto/ssh
go get -u -v github.com/hashicorp/terraform
go get -u -v github.com/josenk/terraform-provider-esxi
export GOPATH="$HOME/go"
cd $GOPATH/src/github.com/josenk/terraform-provider-esxi
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-w -extldflags "-static"' -o terraform-provider-esxi_`cat version`
cp terraform-provider-esxi_`cat version` /usr/bin

私たちはゴールラインにいます。イメージを展開してみましょう。

フォルダに移動します。

cd /root/create-and-deploy-esxi/centos7

まず、variables.tfファイルを編集します。ESXiサーバーへの接続を指定する必要があります。network_config.cfg

ファイルには、将来の仮想マシンのネットワーク設定が含まれています。ニーズに合わせて、ワンライナーを実行します。

sed -i -e '2d' -e '3i "network": "'$(gzip < network_config.cfg| base64 | tr -d '\n')'",' metadata.json

そうですね、main.tfファイルで、ovaファイルへのパスを独自のパスに変更します（異なる場合）。
