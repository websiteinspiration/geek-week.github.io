ゲームエンジンエディター：プロジェクトファイル構造の視覚化



こんにちは、私の名前はアレクサンダーです。私はフロントエンドの開発者です。プログラマーとしてのキャリアは比較的最近になりましたが、私は特別な教育を受けておらず、長い間、まったく別の分野で働いていました。IT業界は非常に急速に発展しているので、私の背後にある素晴らしい経験を持つ同僚に遅れずについていくために、私は絶えず加速されたペースで勉強しなければなりません。自己啓発のため、私は自由な時間にペットプロジェクトを開始することにしました。

プロジェクトを選択する際に自分にとって重要な側面を確認しました


  1. プロジェクトは、相互に補完および開示する必要がある関連テクノロジーの最大数を使用する必要があります（基本として、Javascript-開発者のフロントエンド/バックエンド/フルスタックの方向で雇用主の要件のサンプルをとることができます）。
  2. それは、開発者としてのあなた自身のためのプレゼンテーションとしてそれをさらに使用できるようにするべきです（ndaの現在の作業場所で）。
  3. この作品は、ペットプロジェクトだけでなく、彼らが興味を持って使用することになる本格的な製品になる可能性があるという見通しを持つ必要があります。

子供の頃から自分のゲームを書きたかったのですが、オンラインロールプレイングゲーム「Lineage II」を初めて知った後、自分はマルチプレイヤーである必要があることに気付きました。私はこれを実装することを試みることを決めました、そしてこの場合の締め切りはずれていて、私にとって最も重要なことは必要なハードスキルを取得することでした。そこで、組み込みのエディターとサーバーを備えたゲームエンジンを作成し始めました。

この記事では、私が遭遇した困難とそれらに対処する方法について説明します


ゲームエンジンのエディターを実装するときに、プロジェクトのファイル構造を表示して、コードファイルの編集、画像の読み込み/変更、新しいディレクトリの作成など、プロジェクトの後続の操作を行う必要がありましたが、実際には構造自体のスケルトンが必要です。

    /**
     *       ,       
     * @param folder
     * @param arrayOfStructures
     * @returns {[Object]}
     */

    readFolder(folder: string, arrayOfStructures: object[]): object[] {
        const projectStructure = {};
        let currentDirectory = fs.readdirSync(folder, 'utf8');
        currentDirectory.forEach(file => {
            let pathOfCurrentItem = path.join(folder, file);

            if (fs.statSync(pathOfCurrentItem).isFile()) {
                arrayOfStructures.push({
                    name: file,
                    path: folder.concat('/' + file),
                    extension: path.extname(file),
                    type: 'file'
                });
            }
            else {
                projectStructure[file] = {name: file, path: folder, type: 'directory', arrayOfStructures: []};
                arrayOfStructures.push(projectStructure[file]);
                this.readFolder(pathOfCurrentItem, projectStructure[file].arrayOfStructures);
            }
        });
        return arrayOfStructures;
    }

このスクリプトは、ネストされた要素に関する情報の収集を開始する必要があるディレクトリと、受信したデータを追加する空の配列を受け取ります。簡単にするために、結果のアイテムを2つのタイプ（ファイルとディレクトリ）に分けます。
fs.readdirSync（フォルダ、 'utf8'）は、ディレクトリの読み取り結果を返します。その後、ディレクトリを調べて、実際にそこにあるものを確認する必要があります。これがファイルの場合は、単にarrayOfStructures配列に追加しますが、受け取ったタイプがディレクトリの場合は、readFolder関数を再帰的に呼び出し、最初から開始します。この場合のみ、新しいディレクトリの名前を渡します。出力では、すでにクライアントに送信できるオブジェクトの配列を取得しています。

    /**
     *        
     * @param request
     */
    sendDirectoryEngine(request: Context<object>): void {
        const folder = '/GameEngine/Client';
        const structure = this.readFolder(folder, [{}]);

        request.options.parentCtx.params.res.writeHead(200, {'Content-Type': 'text/plain'});
        request.options.parentCtx.params.res.end(JSON.stringify({
            data: [{
                name: 'root',
                type: 'directory',
                arrayOfStructures: structure
            }]
        }));
    }

        const projectDirectory = BusinessLogic.getDirectoryProject();
        projectDirectory.then(response => response.json())
            .then(result => {
                result.data[0].arrayOfStructures.sort((a, b) => a.type === 'directory' ? -1 : 1);

                    selectedDirectory: result.data[0]

getDirectory方法は、我々は状態どこに受信したデータを書き込み、ビジネスロジックにアクセスするためのメソッドを呼び出すと、受信した約束で動作します：

  • directoryProjectは、プロジェクトのツリー構造の視覚化のコンポーネントを構築するオブジェクトを実装します。
  • selectedDirectoryは、選択されたディレクトリの視覚化コンポーネントを構築するためのデータを含む配列を実装します（デフォルトでは、これはプロジェクトのルートです）。
