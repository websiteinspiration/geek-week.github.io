Guardar valores en la aplicación .Net en la fase de construcción

Era necesario que al compilar la aplicación pasara un conjunto de constantes para usar en tiempo de ejecución. Por ejemplo, queremos "coser" en la aplicación algunos valores de cadena que se conocerán en el momento del ensamblaje.

En el mundo de C ++, resolví esas cosas de manera muy simple usando las opciones de definición y compilación. Pero en .Net define no tiene valores distintos de verdadero / falso, es decir están identificados o no. Según tengo entendido, su objetivo es la compilación condicional más simple.

A quién le importa la solución, bienvenido a cat.

El primer pensamiento que me visitó fue generar un archivo con constantes por plantilla frente a la compilación, pero me gustaría hacerlo sin involucrar artillería pesada de motores de plantillas.

Después de algunas búsquedas, descubrí un hecho interesante. .Net tiene un mecanismo de atributo. Estos atributos pueden aferrarse a clases, métodos, campos y todo tipo de entidades diferentes. Resultó que se puede conectar a todo el conjunto.

En los archivos de proyecto (.csproj), es posible establecer valores para estos atributos durante el ensamblaje. Y en MSBuild, puede pasar parámetros desde el exterior a través del mecanismo de propiedades. Parece que todo encaja, tienes que intentarlo.

Cree una nueva aplicación de consola:

% mkdir Example && cd Example
% dotnet new console

Creamos el archivo ExampleAttribute.cs con la definición de nuestro atributo.

using System;

namespace Example
{

[AttributeUsage(AttributeTargets.Assembly)] //     
public class ExampleAttribute : Attribute
{
    public string Value { get; set; }

    public ExampleAttribute(string value)
    {
        Value = value;
    }
}
 
}

A continuación, el archivo Example.csproj se reduce al siguiente formulario. Agregué comentarios para que la esencia de los cambios fuera clara.

<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp3.1</TargetFramework>
    <RootNamespace>Example</RootNamespace>

    <ExampleValue>default</ExampleValue> <!--        -->
  </PropertyGroup>

  <!--          -->
  <ItemGroup>
    <AssemblyAttribute Include="Example.ExampleAttribute"> <!--    -->
      <_Parameter1>$(ExampleValue)</_Parameter1> <!--      -->
    </AssemblyAttribute>
  </ItemGroup>

</Project>

Bueno, en realidad, obtener el valor en tiempo de ejecución en Project.cs

using System;
using System.Reflection;

namespace Example
{
    class Program
    {
        static void Main(string[] args)
        {
            var assembly = Assembly.GetExecutingAssembly();
            var attr = (ExampleAttribute) assembly.GetCustomAttribute(typeof(ExampleAttribute));
            Console.WriteLine($"Assembly attribute value = '{attr.Value}'");
        }
    }
}

Entonces, nos reuniremos y correremos para ver lo que tenemos.

% dotnet build .
% dotnet run --no-build .
Assembly attribute value = 'default'

Y ahora con el parámetro:

% dotnet build . /p:ExampleValue="NOT DEFAULT"
% dotnet run --no-build .
Assembly attribute value = 'NOT DEFAULT'

Voila, el objetivo se ha logrado. Uso en salud.
