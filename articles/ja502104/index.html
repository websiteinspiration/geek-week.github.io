メッセンジャーボットメッセンジャーICQ新しい


何よりも、EGD BAGチームはICQ New（Mail.ru Group）からのタスクを処理しました。その参加者は、コロナウイルス感染の広がりについてユーザーに通知するICQ Newメッセンジャー用の情報ボットを開発しました。

ジオタグによると、人々は新旧の市民感染事例に関する情報を受け取り、COVID-19の検査を受ける最寄りの医療機関や研究所の住所、最寄りの薬局やショップの住所を確認できます。また、ボットには、電子パスを受信するための簡略化されたSMSメッセージジェネレーターがあります。

ボットを記述する際の計算フローを調整するために、標準のJavaツールが使用されました。ボットの作業を大幅に簡略化するために、ICQのAPIライブラリが選択されました。実稼働環境でのボットのデプロイメントを簡素化するタスクも解決されました。企業開発の標準が今はDockerであることを知り、EGD BAGチームはDockerイメージを準備しました。

その結果、アップグレードが容易でスケーラブルな製品が生まれました。ボットの機能はパンデミックの後でさえ関連性があります。ラボの検索機能は、フィットネスセンター、レストラン、その他の都市施設の検索に簡単に置き換えられます。

ボットは現在、github.com / airaketa / egdbag- botで入手できます。

非接触型決済


Buckwheat42チームは、X5 Retail Groupのタスクにおいて他のチームよりも優れており、Pyaterochkaスーパーマーケットでの購入に対する非接触型決済のための追加の音声入力機能を開発しました。

プロジェクトはPythonで開発されました。プロトタイプは、スピーチをテキストに変換する（speech-to-text）オープンソーステクノロジーと、受信したテキストを処理および分析するモジュール（自然言語理解）に基づいています。音声をテキストに変換するために利用可能なライブラリーの中で、kaldiが選択されました。kaldiは迅速に動作し、ロシア語だけでなく他の多くの言語でも比較的高い認識品質を提供するためです。

タスクの条件に応じて、現在のキャッシュデスクソフトウェアへのソリューションの統合を容易にするために、購入者の音声要求が現在のキャッシュデスクAPIに基づいてメッセージに送信されます。導入とテストを簡単にするために、プロトタイプはDockerテクノロジーを使用して組み立てられました。
このモジュールは、操作ごとに、ユーザーの意図（意図）を識別し、製品の発音名、バーコード、ポイントカード番号、クーポン、その他の関連情報を取得します。

この機能は、インターネットや外部音声変換サービスにアクセスしなくても機能します。

交渉ロボット


このチームをインポートすると、SberCloudから秘書ロボットを作成するタスクが正常に完了しました。このプログラムは、よく知られているZoom、Skype、Teamsなどのオンライン会話のサービスと組み合わせて使用​​でき、オフラインでの会話にも適しています。

秘書ロボットは音声を認識し、対話者の声を区別し、話し言葉の調性によって人の気分を推測することができます。彼は人々が会議や電話を計画するときに気づき、これらのタスクをカレンダーまたはCRMシステムに保存することができます。秘書ロボットは人間のスピーチをテキストメッセージに変換し、交渉後にこれらのメッセージをdocファイルに保存します。

Import this開発チームは、Pythonプログラミング言語で秘書ロボット（バックエンド）の技術的な部分と、Vue.jsおよびElectronソフトウェアプラットフォームの外部部分（フロントエンド）を作成しました。

テキストと音声の認識のために、Google Cloud Speech APIが一時的に使用されましたが、将来的にはより適切なモデルに置き換えられる予定です。テキストの調性を通じて対話者の感情を分析するために、ドストエフスキーライブラリが使用されました。通話や会議の配置をテキストと区別するために、NLTK自然言語処理ライブラリが使用されました。

障害を持つ人々のための視聴覚コンテンツへのアクセス


Uma.Techは、社会的に重要な機会均等タスクのディレクターでした。そこでは、障害のある人々がプレミアプラットフォームやマッチプレミアウェブサイトとやり取りしたり、投稿されたコンテンツを認識したりできるようにするソリューションの開発を参加チームに求めました。

タスクの一環として、チームは、聴覚、視覚、運動障害、集中力の問題を抱えている人々のために、サイトやメディアへのアクセスを簡素化する方法を考え出しました。チームのハッカソンの複雑さは、問題の正確な説明の問題（可能なオプションの範囲が広すぎる）と、従来の限られた開発期間にありました。したがって、ハッカソンの一環として、陪審員は有効なプロトタイプと十分に開発されたコンセプトの両方を受け入れました。

勝者はXGBeastsチームで、Webページと対話するためのニューロコンピューターインターフェイスを開発しました。サービスの原則は、ユーザーの頭にあるニューラルインターフェースからのデータを処理することです。このデータは、コンピュータ画面上の特定のボタンを押す人の意図の説明です。人工知能アルゴリズムを使用して、このデータが認識され、ユーザーがクリックしたいボタンがプログラムによって押されます。

セラピストチャットボット


他よりも優れているのは、ボットセラピストを作成するMobile Medical Technologies LLCのタスクがDevラボチームによって実行されたことです。そのプログラマーは3つのソリューションを同時に作成しました。

  • COVID-19疾患の可能性を迅速に判断し、推奨事項を提示できるチャットボット（医師に連絡するか救急車を呼ぶ）。
  • 必要な情報を提供したり、適切なタイミングで患者を安心させるオンラインアシスタント。
  • 人から半径500メートル以内の最も正確な疫学的状況を計算するサービス。

Web APIを実装するために、React、Redux、Typescriptの開発スタックが選択されました。技術的な部分はPython Flaskで実装されました。音声インテントの分類は、DeepPavlovディープラーニングライブラリを使用して実装されます。COVID-19に関するよくある質問に回答する機能を実装するために、AzureQnAMakerがデプロイされました。TelegramBotを使用して、パンデミックの蔓延を監視するための便利なサービスが実装されました。

モスクワイノベーションエージェンシーによると、近い将来、メガポリスモスクワトラックの一部として開発されたサービスは、トラックのパートナーと最終決定し、そのインフラストラクチャでパイロットされる予定です。
