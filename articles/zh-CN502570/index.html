Citrix Workspace Environment Management是好的产品，但营销效果不好，还是？

在本文中，我想谈一谈Citrix产品组合中的一种产品，即WEM。大约三年半前，Citrix收购了一家名为Norskale的小型软件制造商及其产品Norskale VUEM。Citrix社区对此次收购的宣布非常满意。RES Software发行了RES ONE Workspace产品的免费版本，称为RES ONE Workspace Core，可能是对当时的Norskale竞争对手的接管。

在过去的三年中，以Citrix为品牌的产品发生了很小的变化。我相信值得关注的是，过去的制造商名称并没有在所有产品控制台中都消失了。

当然，许多读者会发现我的方法是业余的，不是基于制造商的官方统计数据，而是根据我个人的估计，基于与众多同事的交流，WEM并不是很受欢迎。高级或高级Citrix Virtual Apps and Desktops许可证的所有持有人均可使用WEM。

关于WEM架构的几句话


我建议首先对体系结构和该产品的既定目的进行简要回顾。 Citrix提供了三种使用方案：

性能优化 -该方案可基于对系统资源（CPU，内存，I / O）的实时性能评估以及随后对其配置的即时更改，有效利用系统资源。可能节省多达70％的资源。

配置文件管理设置 -此方案涉及将用户环境的各种元素的设置从GPO传输到WEM，例如网络打印机，文件夹，登录脚本。 WEM为Citrix User Profile Management提供图形化自定义。该方案假定登录时间显着减少。
安全 -此方案出现在更高版本中，并且基于两个关键组件：

  • 流程管理 -此功能已从“性能优化”部分移出，表示禁止或许可基于黑名单/白名单启动单个流程。
  • 应用程序安全性为单个用户和组提供了对单个程序，脚本和DLL的更广泛访问（启用/禁用）。应用程序安全性类似于Microsoft AppLocker，甚至允许您导出在那里创建的规则。

Transformer -Norskale将该产品定位为一个独立的解决方案，可以将Windows PC转换为ThinClient（将任何PC转换为瘦客户机）。如果像我一样，您认为我们在谈论某种嵌入式Linux，那么您就错了。变形金刚只是为所谓的Windows Shell隐藏一个机会信息亭模式。 Transformer与Citrix自己的“ Desktop Lock”产品直接竞争。

我认为，共享软件产品有不错的选择集。

-我认为，有几个类似的原因。首先，对于拥有数千或数万个用户的大型基础架构，WEM最初不足。例如，一个经纪人一次只能服务3,000个代理。也许有人会反对我的说法，即借助外部负载平衡，可以将多个WEM代理合并到一个基础架构中。

-我仍然认为，第二个原因是可用性，或者可用性。如上所述，Broker是关键组件，但安装后，两个控制台彼此独立。其中之一（WEM数据库管理实用程序）的目的是在从头开始安装时创建数据库，或者在切换到新版本时进行升级。第二个控制台（WEM基础结构服务配置）设计用于基础结构配置。是什么导致Citrix开发人员无法将两个控制台合并为一个（例如PVS中的“配置向导”）或将设置转移到管理控制台中（例如CVAD控制台中的“向导”）？

管理控制台本身还是不合逻辑的，而且绝对不直观。 “科学oke”的方法无法掌握。为了更容易记住发生的事情，我根据使用场景为自己划分了功能区：优化，常规设置和用户配置文件（UEM）。

-脚本用户环境管理。 WEM可能已过时，因为越来越多的产品不再需要安装，而是作为Web控制台提供给用户。在Windows XP和Windows 7中，需要在“开始”菜单中对程序进行分组很方便。连接多个网络驱动器将替换单个SharePoint或ShareFile页面。到2020年，通过Windows GPO连接大量网络打印机的需求有多紧迫？对于用户而言，真正在不到20秒的时间内登录到底有多重要？虽然我毫不怀疑有这样的事情。

-场景资源优化。在我看来，此功能是WEM中最有趣和最吸引人的部分，至今仍未失去其意义。主要问题在于此功能的定位。服务器可扩展性高达70％的承诺对于许多客户端来说实在是太好了。根据我自己的经验，我可以说WEM确实有效地管理CPU资源，并且可以通过更改用户会话中进程的优先级来做到这一点，因此即使处理器已满载，整个系统仍可正常运行。 WEM还可以非常有效地释放处于空闲状态（从RAM到页面文件）的应用程序占用的内存。

以我的经验，一切正常且可靠。但是要说服客户说他可以减少铁的使用量（减少70％）非常困难。大多数客户不希望盲目地信任和仅依赖一个程序，而是在计算必要的铁时希望根据可靠且行之有效的方案采取行动。

可能的营销错误


Citrix或合作伙伴公司的员工在演讲中打赌，所有授权客户都应关注WEM。WEM始终作为一个完整的解决方案提供，涉及同时使用所有方案。我深信，这正是错误所在。

如何吸引客户使用WEM？


为什么不建议不是针对整个基础架构而是针对那些已经存在性能问题的筒仓使用WEM？有时，如果可以解决本地问题，则尝试同时改善所有方面是没有意义的。

性能优化 -此功能的实施不需要特殊费用，但可以提供快速可见的结果。只有一种选项“快速注销”可以轻松证明产品的实用性。

总而言之，我只能说WEM可以减少基础结构的负载并积极影响用户体验，只是您不需要在所有地方（而是在真正需要的地方）引入该产品。

我总是很高兴收到您的建设性评论以及您对产品使用体验的描述。
