Programmierung für BK0010 mit Android



Die Geschichte passierte im BK00010 Telegrammkanal, ich war ein Teilnehmer.

Es stellte sich die Frage: Wie schreibe ich Programme? Der Emulator unterstützt das Brennen von Discs nicht, daher ist es unwahrscheinlich, dass die Verwendung des Assemblers im Emulator erfolgreich ist. Um das Ganze abzurunden, ist das Bild auf Smartphones sehr klein.

Eine Lösung wurde vorgeschlagen (smaslovski), die ich beschreiben werde. Sie müssen Termux herunterladen, die Version für Android 5.0 finden Sie hier , die Datei termux-app-git-debug.apk. Die Version des Programms ist 0.83 und unterstützt das Anpassen der Tastaturleiste.

Nach der Installation und dem Starten von Termux habe ich die Schlüssel neu konfiguriert. Die Vorgehensweise ist wie folgt:

mkdir $HOME/.termux

echo "extra-keys = [['ESC','/','-','HOME','UP','END','PGUP'],['TAB','CTRL','ALT','LEFT','DOWN','RIGHT','PGDN']]" > $HOME/.termux/termux.properties

termux-reload-settings

Es ist bequemer geworden



Jetzt müssen Sie das erforderliche Finish installieren . MACRO-11 wird als Assembler zum Erstellen von .BIN verwendet:
pkg install build-essential
pkg install git




mkdir src
cd src
git clone https://gitlab.com/Rhialto/macro11.git

Dann habe ich die Shell installiert:

pkg install mc

und Texteditor

pkg install nano

in dem src \ macro11 Ordnern benötigen Sie bitte den beheben Makefile , die Parameter ändern

OBJFORMAT = -DDEFAULT_OBJECTFORMAT_RT11=1

Geben Sie make ein und warten Sie auf das Ergebnis.

Kopieren Sie die empfangenen Dateien

cp macro11 $PREFIX/bin

cp dumpobj $PREFIX/bin

Zum Kompilieren benötigen Sie ein Skript zum Erstellen

cd ~/src 
git clone https://github.com/smaslovski/pdp11dasm.git
cd pdp11dasm/utils
chmod 755 obj2bkb
cp obj2bkb $PREFIX/bin

Kompilierungsskript, das ich compile genannt habe :

#!/data/data/com.termux/files/usr/bin/sh
n=$1
macro11 $n.mac -o $n.obj -l $n.lst
obj2bkb --binary --base=01000 -o $n.bin $n.obj
du -b $n.bin
cp $n.bin /sdcard

(Sie können eine Nano-Kompilierung erstellen. )

Das Skript muss kopiert werden

chmod 755 compile
cp compile $PREFIX/bin

Zum Testen habe ich ein einfaches gg.mac- Programm geschrieben :



.title proga
.psect code
.=1000

 MOV #40000,R0
M: mov r0,(r0)+
 tst r0
 bpl m
 halt
.end

launch: compile gg Das
Programm gg.bin kann im Emulator geöffnet werden.
