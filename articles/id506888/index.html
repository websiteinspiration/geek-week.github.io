Menyortir lembar contekan untuk Ilmu Data



Menyortir data adalah perhatian utama bagi ilmuwan dan insinyur data. Pengguna python dapat memilih yang paling nyaman dari sejumlah pustaka dengan opsi penyortiran bawaan, yang dioptimalkan. Beberapa bahkan bekerja secara paralel dengan GPU. Anehnya, beberapa metode penyortiran tidak menggunakan jenis-jenis algoritma ini, sementara yang lain tidak berfungsi seperti yang diharapkan.

Memilih perpustakaan dan jenis algoritma penyortiran tidak selalu mudah, dan inovasi berubah dengan cepat. Saat ini, dokumentasi Pandas tidak cocok dengan kode (meskipun secara pribadi pembaruan PR saya tentang opsi penyortiran adalah yang terbaru).

Dalam artikel ini saya akan menjelaskan kepada Anda apa, saya akan memberikan beberapa tips untuk membantu Anda mengetahui metode dan berbagi hasil dari tes kecepatan.

UPD 17 Juli 2019: Hasil evaluasi tes kecepatan sekarang termasuk implementasi GPU PyTorch dan TensorFlow. TensorFlow juga mencakup hasil CPU untuk kapan tensorflow==2.0.0-beta1dan kapan tensorflow-gpu==2.0.0-beta1. Pengamatan menarik: GPU PyTorch benar-benar terbang, dan GPU TensorFlow lebih lambat dari CPU TensorFlow.

Konteks


Ada banyak algoritma penyortiran dasar. Beberapa dari mereka memiliki kinerja tinggi dan mengambil lebih sedikit ruang, yang lain bekerja dengan baik dengan sejumlah besar data. Untuk beberapa algoritma, posisi relatif dari elemen data adalah penting. Pada diagram di awal artikel, Anda dapat melihat situasi dalam waktu dan volume untuk algoritma yang paling umum.

Bahkan, Anda tidak perlu menjadi ahli dalam penerapan dasar untuk menyelesaikan sebagian besar masalah penyortiran. Bahkan, optimasi prematur kadang-kadang dianggap sebagai akar kejahatan. Namun, jika Anda perlu mengurutkan sejumlah besar data beberapa kali, akan sangat berguna untuk mengetahui terlebih dahulu perpustakaan mana dan kata kunci mana yang paling baik digunakan. Saya berikan kepada Anda lembar contekan saya .



Selama bertahun-tahun, algoritma pengurutan telah berubah di sebagian besar perpustakaan. Untuk analisis dalam artikel ini, saya mengambil versi perangkat lunak berikut.

Python (vanilla)


Python memiliki dua metode penyortiran bawaan.

  • my_list.sort()mengurutkan daftar di tempat, dengan daftar asli diganti dengan yang diurutkan. sort()pengembalian Tidak ada.
  • sorted(my_list) . sorted() . sort() .

Secara teori, itu sort()harus lebih cepat, karena penyortiran terjadi di tempat. Anehnya, tidak ada banyak perbedaan di antara mereka. Selain itu, sort()perlu digunakan dengan hati-hati, karena itu mengubah data asli tanpa menyimpan.

Semua implementasi untuk Vanilla Python, yang akan kita bahas dalam artikel ini, memiliki urutan pengurutan bawaan, dari terkecil ke terbesar. Namun, sebagian besar metode penyortiran lainnya menggunakan metode top-down. Sangat tidak nyaman untuk Anda dan kepala Anda, tetapi opsi ini bervariasi untuk setiap perpustakaan.

Untuk kasus kami, untuk mengubah urutan untuk turun di Vanilla Python, Anda perlu menentukan reverse=True.

keydapat dinyatakan sebagai kata kunci untuk kriteria unik Anda. Contohnya,sort(key=len)Urutkan daftar item berdasarkan panjangnya.

Satu-satunya algoritma pengurutan yang digunakan dalam Vanilla Python adalah Timsort. Dengan menggunakan algoritma ini, data diurutkan sesuai dengan kriteria utama mereka. Misalnya, jika Anda ingin mengurutkan daftar pendek, penyortiran penyisipan digunakan. Untuk informasi lebih lanjut tentang Timsort, lihat artikel hebat Brandon Skerritt.

Timsort dan, karenanya, Vanilla Python, adalah konstan. Artinya, jika nilai awal sama, maka nilai yang diproses akan serupa dan disusun dalam urutan yang sama.
Untuk mengingatkan perbedaan antara sort () dan sortir (), saya hanya memperhatikan bahwa sortir () adalah perintah yang lebih kompleks daripada sort(), dan bahwa sorted()Ini akan membutuhkan lebih banyak waktu secara obyektif, karena pada saat yang sama data awal dan salinan disimpan. Dan bahkan jika hasil tes ambigu, mnemonik adalah segalanya bagi kita.

Sekarang saya mengusulkan untuk mempertimbangkan penggunaan Numpy.

Numpy


Numpy adalah pustaka Python mendasar untuk komputasi ilmiah. Seperti Vanilla Python, ia memiliki dua opsi untuk implementasi: menyalin atau mengubah larik data sumber:

  • my_array.sort () mengubah array di tempat dan mengembalikan array yang diurutkan;
  • np.sort (my_array) mengembalikan salinan array yang diurutkan, tanpa mengubah data asli.

Argumen yang digunakan secara opsional:

  • axis: int, opsional - sumbu tempat penyortiran dilakukan. Secara default -1 - pengurutan berdasarkan sumbu terakhir.
  • kind: {'quicksort', 'mergesort', 'heapsort', 'stable'} - algoritma pengurutan. Secara default, 'quicksort' digunakan. Selanjutnya saya akan membicarakan hal ini secara lebih rinci.
  • order: str atau daftar str - ketika a adalah array dengan batas yang ditentukan, argumen ini menunjukkan urutan urutan batas ini dibandingkan. Satu bidang dapat ditentukan sebagai string, bidang lebih lanjut mungkin tidak ditentukan. Mereka dalam hal apa pun akan digunakan dalam tipe untuk gangguan.

Terus terang, saat ini, algoritma penyortiran sebenarnya cocok dengan nama mereka. Perintah itu kind=quicksortsecara harfiah berarti penyortiran dimulai dengan penyortiran introspektif . Lebih detail di sini .

Saat mengurutkan berdasarkan satu kolom di Numpy, ini digunakan secara default quicksort. Tensorflow


TensorFlow adalah media paling populer untuk pembelajaran mendalam. Anda dapat mempelajari lebih lanjut tentang pembelajaran mendalam di artikel saya di tautan di sini . Informasi berikut ini relevan untuk GPU TensorFlow 2.0.

tf.sortpada dasarnya menggunakan top_k(). Sebab top_k(), perpustakaan CUB untuk GPU CUDA digunakan, yang menyederhanakan implementasi paralel. Dokumentasi mengatakan bahwa "CUB menyediakan komponen perangkat lunak yang modern dan berulang untuk setiap tingkat model pemrograman CUDA." TensorFlow menggunakan penyortiran dasar GPU melalui CUB ( diskusi ).

Informasi tentang GPU TensorFlow dapat ditemukan di sini . Untuk mengaktifkan GPU dengan TensorFlow 2.0, Anda harus mendaftar !pip3 install tensorflow-gpu==2.0.0-beta1. Akan terlihat di bawah ini bahwa Anda dapat mengikuti jalur tensorflow==2.0.0-beta1jika Anda hanya mengurutkan data (yang tidak mungkin).

Untuk memeriksa kode pada CPU dan GPU, gunakan baris berikut:

Untuk menunjukkan bahwa Anda ingin menggunakan GPU, Anda harus menggunakan blok ini:

with tf.device('/GPU:0'):

tf.sort()Ini adalah metode intuitif ketika bekerja di TensorFlow. Hanya ingat apa yang Anda direction=descendingbutuhkan untuk mengubah urutan pengurutan.

Pytorch


torch.sort(my_tensor)mengembalikan salinan tensor yang diurutkan. Argumen opsional:

  • dim: {int} - volume pemilahan.
  • descending: {bool} - mengontrol urutan sortir (naik atau turun).
  • out: {tuple} - pelacakan keluaran (Tensor, LongTensor), yang dapat digunakan sebagai buffer output.

Jika Anda ingin menggunakan GPU untuk menyortir, pasang .cuda ()ke ujung tensor Anda.

gpu_tensor=my_pytorch_tensor.cuda()
%time torch.sort(gpu_tensor)

Penelitian telah menunjukkan bahwa PyTorch menggunakan pemilahan paralel tersegmentasi melalui Thrust jika menyortir dataset lebih besar dari 1 juta baris per 100.000 kolom.

Sayangnya, saya tidak memiliki cukup memori ketika saya mencoba membuat data sewenang-wenang sebesar 1,1 juta per 100 ribu ukuran melalui Numpy di Google Colab. Setelah itu, saya mencoba GCP dengan 416 MB RAM, dan sekali lagi saya kehabisan memori.
Pemilahan tersegmentasi dan pengurutan berdasarkan lokasi adalah opsi pemilahan menggabungkan kinerja tinggi yang bekerja dengan data acak yang heterogen. Pengurutan tersegmentasi memungkinkan Anda untuk mengurutkan beberapa array dengan panjang variabel secara paralel. - https://moderngpu.github.io/segsort.html
Thrust adalah pustaka algoritma paralel yang menyediakan kompatibilitas antara kinerja GPU dan CPU multi-core. Ini menyediakan kerangka kerja penyortiran yang secara otomatis memilih metode implementasi yang paling efisien. Perpustakaan CUB yang digunakan oleh TensorFlow meringankan beban. PyTorch dan TensorFlow menggunakan algoritma yang sama untuk mengurutkan GPU - apa pun pendekatan yang Anda gunakan.

Sebagai TensorFlow, metode pengurutan di PyTorch cukup mudah untuk diingat: torch.sort(). Satu-satunya hal yang perlu diingat adalah arah nilai yang diurutkan: TensorFlow menggunakan direction, dan PyTorch menggunakan descending. Dan jangan lupa menggunakannya .cuda()untuk memaksimalkan kecepatan saat bekerja dengan data dalam jumlah besar.

Sementara penyortiran GPU mungkin merupakan opsi yang baik untuk kumpulan data yang sangat besar, juga masuk akal untuk menyortir data secara langsung dalam SQL.

SQL


Penyortiran dalam SQL biasanya sangat cepat, terutama jika penyortiran berlangsung langsung di memori.

SQL adalah spesifikasi yang tidak mengharuskan Anda untuk menggunakan algoritma tertentu. Postgres menggunakan penyortiran gabungan disk, piramida atau penyortiran cepat, tergantung pada kondisinya. Jika Anda memiliki cukup memori, penyortiran dapat dilakukan di dalamnya, dan jauh lebih cepat. Anda dapat menambah memori yang tersedia untuk disortir work_mem.

Opsi SQL lainnya menggunakan algoritma pengurutan yang berbeda. Misalnya, Google BigQuery menggunakan penyortiran internal dengan beberapa trik, seperti yang disajikan dalam jawaban Stack Overflow .

Penyortiran dalam SQL dilakukan oleh perintahORDER BY. Sintaks ini berbeda dari Python, di mana mereka lebih suka menggunakan beberapa bentuk pengurutan kata. Secara pribadi, saya ingat bahwa ORDER BY digunakan dalam sintaks SQL, karena agak tidak biasa.

Untuk mengurutkan data dalam urutan menurun, gunakan kata kunci DESC. Dengan demikian, permintaan untuk mengembalikan data dalam urutan abjad dari yang terakhir ke yang pertama akan terlihat seperti ini:

SELECT Names FROM Customers
ORDER BY Names DESC;

Perbandingan


Untuk masing-masing pustaka Python di atas, saya melakukan analisis dengan mengurutkan 1.000.000 titik data dalam satu kolom, array, atau daftar. Saya menggunakan laptop Google Colab Jupyter dengan GPU K80 dan CPU Intel Xeon 2,30 GHz.



Pengamatan


  • PyTorch dengan GPU secepat mungkin.
  • Untuk Numpy dan Pandas, pengurutan di tempat umumnya lebih cepat daripada menyalin data.
  • Quicksort panda secara default cukup cepat.
  • Sebagian besar fitur Pandas relatif lebih lambat daripada rekan-rekan Numpy mereka.
  • Prosesor TensorFlow cukup cepat. Menginstal GPU memperlambat TensorFlow bahkan saat menggunakan CPU. Penyortiran GPU cukup lambat. Opsi ini tidak terlalu efektif.
  • Dalam Vanilla Python, pengurutan di tempat sangat lambat - hampir 100 kali lebih lambat daripada pengurutan dengan dukungan GPU PyTorch. Saya membuat percobaan beberapa kali (dengan data berbeda) untuk mengecek apakah ini bukan anomali.
