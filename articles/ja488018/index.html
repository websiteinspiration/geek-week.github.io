CAPTCHA、特殊なケース：30行のコードでニューラルネットワークを壊す

   どのように記事habr.com/ru/post/464337に出くわしたか覚えていませんが、それは私の脳に沈み込み、最終日まで休息しませんでした。何が起こっているのかを理解しようと何度か試みましたが、数回実行してみましたが、役に立ちませんでした。ニューラルネットワークでは何も完全に理解しておらず、実際のプログラマのようにさえプログラミングしています。


   最後に、数日前に、私はpythonの起動を圧倒し、それがすべてではない理由を決定しました。前述の記事で読んだことをすべて忘れて、私は自分の道を進みました。
   無数のキャプチャされたキャプチャを思い出して、後で確認されたマスクとの平凡な比較によってそれらを解決できることを提案しました。
   まず、手動でテストキャプチャ（83個）を組み立てて、わかりやすい名前を付けました。スクリプトはそれらをビットマップに変えました。

   CAPTCHAの数値は、高さが2サイズで、1ピクセルが異なり、幅が3〜4面です。すべてのキャプチャのすべてのキャラクターのベースラインは同じです。結局のところ、この多様性にはすべて、特定の共通のマスクがあります。これを比較すると、図を一意に識別できます。さまざまなキャプチャから同じ番号をいくつか（最初は-5、次に別の1-2を追加しました。 "4"から、残りよりも時間がかかりました）。paint.netでは、それらを重ねて、各指のすべての面に共通のマスクを取得しました。

   後で大量処理中に唯一の問題を発見しましたが、問題を回避できました

   それで私はグーグルを発見し、自分で完璧なキャプチャを自分でマイニングする必要がありました。後で判明したように、49個の完全に空のファイルを含め、1晩に3224個のファイルが蓄積されました。ガネーシャ、コードをありがとう。

   実際にキャプチャの認識は、退屈なpythonコードの26行に収まります。外部モジュールからはPILのみが必要です。古いCore 2の「4コア4ギグ」では、速度は1分あたり約1000キャプチャ（1分あたり1000キャプチャ）です。より適切な8ストリームでは、当然、スレッドに問題はありませんが、i5は著しく高速です。 100％またはそれに非常に近い認識：スポットチェックはエラーを示しませんでした。

   もちろん、これらすべてはニューラルネットワークや他のブロックチェーンの意味では興味深いものではありませんが、以前に提案されたオプション（速度と精度）に比べて非常に明確な利点があります。captchaのパラメーターの変更（ヘッドセットまたはフォントサイズ、ノイズの種類など）も同様です。-私の決定の完全な操作不能につながります。

from PIL import Image, ImageTk

def recognize(filepath):
  Zlist = [] # [(x1, z1), (x2, z2), (x3, z3), etc.] - position and digit
  captcha = ""
  originalimage = Image.open(filepath).convert('L').point(lambda x : 255 if x > 20 else 0, mode='1').convert('1').convert('RGBA')
  if originalimage.getextrema() == ((0, 0), (0, 0), (0, 0), (255, 255)):
    return("empty image")
  for z in [4, 2, 3, 1, 5, 6, 7, 8, 9]: # reorder to exclude false 1 on 4
    mask = Image.open('mask' + str(z) + '.png').convert('RGBA')
    previ = 0
    for i in range(15, 120): # no digit in left part
      resultimage = Image.alpha_composite(originalimage.crop((i, 0, i + 30, 0 + 50)), mask)
      if resultimage.getextrema() == ((0, 0), (0, 0), (0, 0), (255, 255)):
        if z == 4: # delete 4 to exclude false 1 on 4
          maskx = Image.open('mask4x.png').convert('RGBA') 
          originalimage.paste(Image.alpha_composite(originalimage.crop((i, 0, i + 30, 0 + 50)), maskx), (i, 0))
        if previ == 0 or i > previ + 15: #no digit closer then 15 px
          Zlist.append((i, z))
          if len(Zlist) == 5:
              Zlist.sort()
              for z in Zlist:
                captcha = captcha + str(z[1])
              return(captcha)
          previ = i
          i = i + 15 #skip a little
  Zlist.sort()
  return(str(Zlist)) #if less then 5 digits recognized
