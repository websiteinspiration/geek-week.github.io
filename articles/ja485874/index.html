多くの辞書をリストに保存するか、リストを辞書要素の値として保存する必要がある場合があります。このような複雑な構造を作成することを、埋め込みと呼びます。リスト、辞書内のアイテムのリスト、または別の辞書内の辞書に複数の辞書を埋め込むことができます。次の例が示すように、アタッチメントは非常に強力なメカニズムです。

辞書のリスト


alien_0辞書には、1人のエイリアンに関するさまざまな情報が含まれていますが、エイリアンによって詰まった画面全体は言うまでもなく、2番目のエイリアンに関する情報を保存する場所はありません。侵略艦隊をシミュレートする方法は？たとえば、各要素がエイリアンに関する情報を含む辞書であるエイリアンのリストを作成できます。たとえば、次のコードは3つのエイリアンのリストを作成します。

aliens.py
   alien_0 = {'color': 'green', 'points': 5}
   alien_1 = {'color': 'yellow', 'points': 10}
   alien_2 = {'color': 'red', 'points': 15}
❶ aliens = [alien_0, alien_1, alien_2]
   for alien in aliens:
        print(alien)

まず、3つの辞書が作成され、それぞれが別のエイリアンを表します。ポイントpointで、各辞書はエイリアンの名前でリストされます。最後に、プログラムはリストを反復処理し、各エイリアンを表示します。

{'color': 'green', 'points': 5}
{'color': 'yellow', 'points': 10}
{'color': 'red', 'points': 15}

もちろん、現実的な例では、4つ以上のエイリアンが使用され、自動的に生成されます。次の例では、range（）関数は30人のエイリアンの艦隊を作成します。

すべてのエイリアンは同じ特性を持っていますが、Pythonは各エイリアンを個別のオブジェクトとして扱います。これにより、各所有者の属性を個別に変更できます。

多くの人とどのように連携するのですか？このゲームで何人かのエイリアンが色を変え、より速く動き始めると想像してください。色を変更するときは、forループとifコマンドを使用して色を変更できます。たとえば、最初の3つのエイリアンを黄色に変え、中速で移動し、プレーヤーにそれぞれ10ポイントを与えるには、次のようにします。

aliens = []

#  30  .
for alien_number in range (0,30):
     new_alien = {'color': 'green', 'points': 5, 'speed': 'slow'}

for alien in aliens[0:3]:

最初の3つのエイリアンを変更するには、最初の3つのエイリアンのみを含むセグメントの要素を繰り返します。現時点では、すべてのエイリアンは緑色（「グリーン」）ですが、常にそうであるとは限らないため、ifコマンドを記述して、緑色のエイリアンのみが変更されるようにします。エイリアンが緑の場合、彼の色は黄色（「黄色」）に変わり、速度は中（「中」）に変わり、報酬は10ポイントに増加します。

for alien in aliens[0:3]:

辞書をリストに格納するソリューションは、各辞書に1つのオブジェクトの異なる属性が含まれている場合に非常に一般的です。たとえば、p。のuser.pyで行ったように、サイトのユーザーごとに辞書を作成できます。114、個々の辞書をusersという名前のリストに保存します。リストを反復処理し、各ディクショナリオブジェクトで同じ操作を実行できるように、リスト内のすべてのディクショナリは同じ構造である必要があります。

辞書リスト


辞書をリストに入れる代わりに、リストを辞書に入れると便利な場合があります。プログラムで注文したピザをどのように説明するか想像してみてください。リストのみに制限すると、ピザのトッピングのリストのみを保存できます。辞書を使用する場合、トッピングリストはピザの説明の一部にすぎません。

次の例では、ピザごとに2つのタイプの情報（ベースとトッピングリスト）が保存されます。トッピングリストは、「トッピング」キーに関連付けられた値です。リスト内のアイテムを使用するには、辞書の他の値と同様に、辞書名と「toppings」キーを指定する必要があります。単一の値の代わりに、トッピングのリストが取得されます：

pizza.py
   #     .
❶ pizza = {
         'crust': 'thick',
         'toppings': ['mushrooms', 'extra cheese'],
         }

   #  .
❷ print(f"You ordered a {pizza['crust']}-crust pizza "
        "with the following toppings:")

❸ for topping in pizza['toppings']:
print("\t" + topping)

作業はポイントatから始まり、注文されたピザに関する情報を含む辞書が使用されます。文字列値 'thick'は、 'crust'辞書のキーに関連付けられています。別のキー「トッピング」には、すべての順序付けされたトッピングが格納されるリスト値があります。ポイントpointでは、ピザを作成する前に注文の説明が表示されます。 print（）コールで長い行を分割する必要がある場合は、出力行を分割するポイントを選択し、行を引用符で終了します。次の行をインデントし、開始引用符を追加して、行を続けます。 Pythonは、括弧内にあるすべての文字列を自動的に連結します。追加の結論として、for❸ループが記述されています。トッピングを一覧表示するには、「toppings」キーを使用します。Pythonは、トッピングのリストを辞書から取得します。

次のメッセージは、これから作成するピザについて説明しています。

You ordered a thick-crust pizza with the following toppings:
      mushrooms
      extra cheese

辞書にリストをネストすると、複数の値を1つの辞書キーに関連付ける必要があるたびに使用できます。前の例のプログラミング言語を使用して回答がリストに保存されている場合、調査の1人の参加者が複数のお気に入りの言語を一度に選択できます。辞書を列挙する場合、各人に関連付けられた値は、（1つの言語ではなく）言語のリストになります。辞書のforループでは、別のループが作成され、各参加者に関連付けられている言語のリストを反復処理します。

favorite_languages.py
❶ favorite_languages = {
          'jen': ['python', 'ruby'],
          'sarah': ['c'],
          'edward': ['ruby', 'go'],
          'phil': ['python', 'haskell'],
          }

❷ for name, languages in favorite_languages.items():
        print(f"\n{name.title()}'s favorite languages are:")
❸      for language in languages:
             print(f"\t{language.title()}")

ポイントatで、各名前に関連付けられた値がリストになっていることがわかります。好きなプログラミング言語を持っている参加者もいれば、複数の参加者がいる参加者もいます。ポイントatでディクショナリを反復する場合、言語という名前の変数を使用して、ディクショナリからの各値を格納します。これは、各値がリストになることがわかっているためです。メインサイクルでは、辞書要素に従って、別のサイクル❸が各参加者のお気に入りの言語のリストの要素を反復します。これで、各調査参加者は好きなだけプログラミング言語を指定できます。

Jen's favorite languages are:
      Python
      Ruby

Sarah's favorite languages are:
      C

Phil's favorite languages are:
      Python
      Haskell

Edward's favorite languages are:
      Ruby
      Go

プログラムをさらに改善するには、辞書サイクルの先頭にifコマンドを含めて、参加者が複数のプログラミング言語を選択したかどうかを確認します（チェックはlen（言語）の値に基づいています）。参加者のお気に入りの言語が1つしかない場合、メッセージテキストは単数形に変更されます（たとえば、「Sarahのお気に入りの言語はC」）。
