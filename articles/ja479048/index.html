ダミーのNode.jsストリーム、またはストリームの操作方法

多くの人がNode js Streamsについて2回以上聞いたことがあると思いますが、それを使用したことはありません。ストリームとは何か、パイプ（パイプ）、チャンク（チャンク-データの一部）などすべてを把握しましょう）



Node jsでストリームがどのように機能するかを理解することがなぜ重要なのですか？答えは簡単です。Nodejsの組み込みモジュールの多くは、HTTP要求/応答、fs読み取り/書き込み、zlib、crypto、TCPソケットなどのストリームを実装します。大きなファイルを処理するときや、画像を操作するときなどに、ストリームも必要になります。独自のストリームを作成しているわけではありませんが、それがどのように機能するかを理解すると、より有能な開発者になります。

それで、正確にはストリームとは何ですか（以下、Stream（ストリーム）の代わりに使用します）。ストリームは、小さな部分でデータを処理できる概念であり、これにより少量のRAMを使用できます。また、その助けを借りて、各部分の処理を互いに独立したモジュール（関数またはクラス）に分割できます。たとえば、データの一部をすぐに圧縮してから、暗号化してファイルに書き込むことができます。主なアイデアは、データ全体を処理するのではなく、データの一部を1つずつ処理することです。

Node jsには4種類のストリームがあります。

  • 読みやすい
  • 書き込み可能-書き込み
  • 両面印刷-読み取りと書き込み
  • 変換-データを変更できる一種の二重ストリーム

詳細は公式サイトで確認できますので、早速練習していきましょう。

簡単な例


多くの人が気づかないうちにすでにストリームを使用していると思います。この例では、ファイルをクライアントに送信するだけです。

// 1 - ( )      ,        
const getFile = async (req, res, next) => {
  const fileStream = fs.createReadStream('path to file');

  res.contentType('application/pdf');

  fileStream.pipe(res);
};

// 2 - (  )        
const getFile = async (req, res, next) => {
  const file = fs.readFileSync('path to file');

  res.contentType('application/pdf');

  res.send(file);
};

唯一の違いは、最初のケースでは、ファイルの一部をダウンロードして送信するため、サーバーのRAMをロードしないことです。2番目のケースでは、ファイル全体をすぐにRAMにロードしてから送信します。

さらに記事では、各ストリームを個別に分析します。継承またはコンストラクター関数を使用してストリームを作成できます。

const { Readable } = require('stream');

// 1 -  
const myReadable = new Readable(opt);

// 2 -   
class myReadable extends Readable {
  constructor(opt) {
    super(opt);
  }
}

すべての例で、2つの方法を使用します。

読み取り可能なストリーム


NodeJSでReadableストリームを作成する方法を見てみましょう。

const { Readable } = require('stream');

class myReadable extends Readable {
  constructor(opt) {
    super(opt);
  }

  _read(size) {}
}

上記の例からわかるように、このクラスはパラメータのセットを受け入れます。 Readableストリームの一般的な理解に必要なものだけを考慮し、残りはドキュメントで確認できます。 highWaterMarkパラメータと_readメソッドに関心があります。

highWaterMark-これは、リソースからの読み取りが中断される到達時の内部ストリームバッファー（デフォルトでは16kb）の最大バイト数です。読み取りを続けるには、内部バッファーを解放する必要があります。これを行うには、パイプ、再開メソッドを呼び出すか、データイベントをサブスクライブします。

_readは、Readableクラスの内部メソッドによって呼び出されるプライベートメソッドの実装です。データサイズがhighWaterMarkに達するまで、継続的に呼び出されます。

さて、私たちが興味を持っている最後のメソッドはreadible.pushです。これは、データを内部バッファーに直接追加します。 trueを返しますが、バッファがいっぱいになるとすぐに、このメソッドの呼び出しはfalseを返し始めます。それは、readible._readメソッドで制御できます。

状況を明確にするための例を見てみましょう。

class Counter extends Readable {
  constructor(opt) {
    super(opt);

    this._max = 1000;
    this._index = 0;
  }

  _read() {
    this._index += 1;

    if (this._index > this._max) {
      this.push(null);
    } else {
      const buf = Buffer.from(`${this._index}`, 'utf8');

      console.log(`Added: ${this._index}. Could be added? `, this.push(buf));
    }
  }
}

const counter = new Counter({ highWaterMark: 2 });

console.log(`Received: ${counter.read().toString()}`);

まず、counter.read（）は、クラスで実装した_readではない、と言います。そのメソッドはプライベートであり、これはパブリックであり、内部バッファーからデータを返します。このコードを実行すると、コンソールに次のように表示さ



れます。新しいCounter（{highWaterMark：2}）ストリームを作成するときに、内部バッファーのサイズが2バイトになることを示しました。 2文字を格納できます（1文字= 1バイト）。 counter.read（）を呼び出した後、ストリームは読み取りを開始し、内部バッファーに「1」を書き込んでそれを返します。それから彼は読み続け、「2」を書きます。それが「3」を書き込むと、バッファーはいっぱいになり、readible.pushはfalseを返し、ストリームは内部バッファーが解放されるまで待機します。なぜならこの例では、バッファを解放するロジックはなく、スクリプトは終了します。

前述のように、読み取りが中断されないようにするために、内部バッファーを常にクリアする必要があります。これを行うには、データイベントをサブスクライブします。最後の2行を次のコードに置き換えます。

const counter = new Counter({ highWaterMark: 2 });

counter.on('data', chunk => {
  console.log(`Received: ${chunk.toString()}`);
});

この例を実行すると、すべてが正常に機能し、1から1000までの数字がコンソールに表示されます。

書かれたストリーム


実際、これは読み取り可能なストリームに非常に似ており、データの書き込みのみを目的としています。

const { Writable } = require('stream');

class myWritable extends Writable {
  constructor(opt) {
    super(opt);
  }

  _write(chunk, encoding, callback) {}
}

Readable Streamなどの同様のパラメーターを受け入れます。 highWaterMarkと_writeに関心があります。

_writeは、データの一部を書き込むためにWritableクラスの内部メソッドによって呼び出されるプライベートメソッドです。チャンク（データの一部）、エンコーディング（チャンクが文字列の場合はエンコーディング）、コールバック（書き込みが成功または失敗した後に呼び出される関数）の3つのパラメーターを取ります。

highWaterMarkは、内部ストリームバッファの最大バイト数（デフォルトでは16kb）であり、それに到達すると、stream.writeがfalseを返し始めます。

前の例をカウンターで書き換えましょう。

const { Writable } = require('stream');

class Counter extends Writable {
  _write(chunk, encoding, callback) {
    console.log(chunk.toString());

    callback();
  }
}

const counter = new Counter({ highWaterMark: 2 });

for (let i = 1; i < 1000; i += 1) {
  counter.write(Buffer.from(`${i}`, 'utf8'));
}

実際、すべてがシンプルですが、覚えておく価値のある興味深いニュアンスが1つあります。新しいCounter（{highWaterMark：2}）ストリームを作成するときに、内部バッファーのサイズが2バイトになることを示しました。 2文字を格納できます（1文字= 1バイト）。カウンターが10に達すると、書き込みが呼び出されるたびにバッファーがいっぱいになり、遅いソースで記録が行われた場合、書き込みを呼び出すときの他のすべてのデータはRAMに格納され、オーバーフローする可能性があります（この例では、これはもちろん、私たちのバッファは2バイトなので重要ではありませんが、大きなファイルではこれを覚えておく必要があります）。このような状況が発生した場合、ストリームがデータの現在の部分を書き込み、内部バッファーを解放（ドレインイベントをトリガー）するまで待機する必要があります。その後、データの記録を再開できます。例を書き直してみましょう。

const { Writable } = require('stream');
const { once } = require('events');

class Counter extends Writable {
  _write(chunk, encoding, callback) {
    console.log(chunk.toString());

    callback();
  }
}

const counter = new Counter({ highWaterMark: 2 });

(async () => {
  for (let i = 1; i < 1000; i += 1) {
    const canWrite = counter.write(Buffer.from(`${i}`, 'utf8'));

    console.log(`Can we write bunch of data? ${canWrite}`);

    if (!canWrite) {
      await events.once(counter, 'drain');
      console.log('drain event fired.');
    }
  }
})();

events.onceメソッドはv11.13.0で追加され、プロミスを作成し、特定のイベントが1回実行されるのを待つことができます。この例では、データをストリームに書き込むことができるかどうかを確認し、できない場合は、バッファーが解放されるまで待機して、記録を続行します。

一見すると、これは不要なアクションのように見えるかもしれませんが、たとえば10GBを超えるファイルなどの大量のデータを操作するときに、これを忘れると、メモリリークが発生する可能性があります。

双方向ストリーム


これは、読み取り可能ストリームと書き込み可能ストリームを組み合わせたものです。つまり、2つの_readメソッドと_writeメソッドの実装を作成する必要があります。

const { Duplex } = require('stream');

class myDuplex extends Duplex {
  constructor(opt) {
    super(opt);
  }

  _read(size) {}

  _write(chunk, encoding, callback) {}
}

ここでは、コンストラクターに渡すことができる2つのパラメーターに興味があります。これらは、ReadableHighWaterMarkとwritableHighWaterMarkです。これにより、ReadableストリームとWritableストリームの内部バッファーのサイズをそれぞれ指定できます。これは、Duplexストリームを利用した前の2つの例の実装の様子です。

const { Duplex } = require('stream');
const events = require('events');

class Counter extends Duplex {
  constructor(opt) {
    super(opt);

    this._max = 1000;
    this._index = 0;
  }

  _read() {
    this._index += 1;

    if (this._index > this._max) {
      this.push(null);
    } else {
      const buf = Buffer.from(`${this._index}`, 'utf8');

      this.push(buf);
    }
  }

  _write(chunk, encoding, callback) {
    console.log(chunk.toString());

    callback();
  }
}

const counter = new Counter({ 
  readableHighWaterMark: 2, 
  writableHighWaterMark: 2 
});

(async () => {
  let chunk = counter.read();

  while (chunk !== null) {
    const canWrite = counter.write(chunk);

    console.log(`Can we write bunch of data? ${canWrite}`);

    if (!canWrite) {
      await events.once(counter, 'drain');
      console.log('drain event fired.');
    }

    chunk = counter.read();
  }
})();

このコードは、以前と同じで、1つのクラスのみなので、説明は必要ないと思います。

ストリームの変換


このストリームは二重ストリームです。データのチャンクを変換し、チェーンのさらに下に送信するために必要です。残りのストリームと同じ方法で実装できます。

const { Transform } = require('stream');

class myTransform extends Transform {
  _ transform(chunk, encoding, callback) {}
}

_transformメソッドに興味があります。

_transformは、データのチャンクを変換するためにTransformクラスの内部メソッドによって呼び出されるプライベートメソッドです。チャンク（データの一部）、エンコーディング（チャンクが文字列の場合はエンコーディング）、コールバック（書き込みが成功または失敗した後に呼び出される関数）の3つのパラメーターを取ります。

この方法を使用すると、データの一部に変更が発生します。このメソッド内で、transform.push（）を0回以上呼び出すことができ、変更をコミットします。データ変換が終了したら、コールバックを呼び出す必要があります。コールバックは、追加したすべてをtransform.push（）に送信します。このコールバック関数の最初のパラメーターはエラーです。また、transform.push（）は使用できませんが、2番目のパラメーターによって変更されたデータをコールバック関数に送信します（例：コールバック（null、データ））。このタイプのストリームの使用方法を理解するために、stream.pipeメソッドを分析してみましょう。

stream.pipe-このメソッドは、ReadableストリームをWritableストリームに接続し、ストリームチェーンを作成するために使用されます。これは、データの一部を読み取り、次のストリームに転送して処理できるようにすることを意味します。次に、

各データパーツの先頭と末尾に*記号を追加する変換ストリームを作成してみましょう。

class CounterReader extends Readable {
  constructor(opt) {
    super(opt);

    this._max = 1000;
    this._index = 0;
  }

  _read() {
    this._index += 1;

    if (this._index > this._max) {
      this.push(null);
    } else {
      const buf = Buffer.from(`${this._index}`, 'utf8');

      this.push(buf);
    }
  }
}

class CounterWriter extends Writable {
  _write(chunk, encoding, callback) {
    console.log(chunk.toString());

    callback();
  }
}

class CounterTransform extends Transform {
  _transform(chunk, encoding, callback) {
    try {
      const resultString = `*${chunk.toString('utf8')}*`;

      callback(null, resultString);
    } catch (err) {
      callback(err);
    }
  }
}

const counterReader = new CounterReader({ highWaterMark: 2 });
const counterWriter = new CounterWriter({ highWaterMark: 2 });
const counterTransform = new CounterTransform({ highWaterMark: 2 });

counterReader.pipe(counterTransform).pipe(counterWriter);

この例では、前の例のReadableおよびWritableストリームを使用し、Transformも追加しました。ご覧のとおり、非常に単純なことがわかりました。
