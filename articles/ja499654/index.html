実装とは何ですか？


中小企業がCRMシステムを実装することを決定するとき、それは通常、それを「サービスへの登録、開始」と想像します。この戦術は、サブスクリプションを購入するように説得することに加えて、特に有能でない多数のCRMリセラー（たとえば、主な活動がフィットネスストレッチスタジオである有名なCRMのパートナーなど）によって大幅にウォームアップされます。主にそのようなユーザーの期待のために、CRM市場は失敗したプロジェクトを受け取り、その後、企業は別のCRMに戻る準備ができていません。これは、会社の精神と経営陣に対する従業員の態度を打つ悲しくて費用のかかる経験です。

実装とは、CRMシステムの選択、購入、試運転のプロセスです、正確には会社の要件を満たし、運用および分析活動の問題を解決できるものです。 詳細なスキームを使用したCRMの実装に関する記事

なぜ今、企業はCRMを必要としているのですか？


CRMは、販売用の自己組織化テーブルクロスではなく、魔法の杖ではありません（しかし、私はしたいです！）。これは、会社の技術スタックまたはその技術的基盤のクールなツールです。調査によると、実装を完了した企業は、CRMシステムとCRM戦略に投資された1ドルごとに8ドルのROIを受け取ります。

原則として、CRMを持たない企業は、常に自動化された競合他社よりも生産性が低いですが、なんとかして管理されていました。現在、チームは非常に断片化されており、単一のMS Excelファイルではなく、CRMのような集中管理および運用作業の重要な要素になる可能性があります。

RegionSoft CRMでのリモート作業の方法-技術的側面


System RegionSoft CRMは、インターネットを介してリモートでユーザーを操作する機能をサポートしています。これは、異なる場所に複数のオフィスがある企業や、常時または一時的にリモート作業に切り替えざるを得ないすべての人に適しています。同時に、すべての部門と従業員が同じ情報システム内で集中的に作業します。すべてのデータは1か所に保存され、RegionSoft CRMで提供されるすべての情報セキュリティツールによって確実に保護されます。

リモートで作業するには2つの方法があります。

  • 端末モード;
  • DBMSポートを介した接続。

どちらの方法もオンライン作業をサポートしています。リモートコンピュータにある中央情報データベースにリアルタイムでアクセスできます。

方法1：端末モード（推奨される方法）



ターミナルモードは、RDP（リモートデスクトップ管理）テクノロジを使用した中央コンピューターへの接続です。これまでのところ、この方法は最も安全で便利です。互いに離れた場所にあるサーバーとワークステーションの両方が、少なくとも512 kbit / sの推奨される安定したチャネルでインターネットに接続されている必要があります。ユーザーのPC上のCRMは、RDPプロトコルを使用して（リモートデスクトップユーティリティまたはそのアナログを使用して）サーバーに接続し、システムはサーバー側で起動するか、RemoteAppテクノロジを使用して起動します。また、本格的な作業はオンラインで行っています。リモートユーザーには、システムのすべての機能へのフルアクセスが与えられます。

RegionSoft CRMがサポートするRDPプロトコルは、シンクライアントテクノロジーに基づいています。このテクノロジーの利点は、作業中にアプリケーションが起動され、すべてのデータ操作がサーバー側で実行されることです。これにより、データ伝送チャネルの負荷が大幅に軽減されます。このモードでは、チャネルとリモートワークステーションの負荷を最小限に抑えながら、大量のデータを処理できます。接続すると、ユーザーはオペレーティングシステムによって認証され、適切なセキュリティが確保されます。

端末操作モードを構築する手段として、さまざまなテクノロジーを使用できます。ただし、最も高品質のソリューションと、適切なレベルのセキュリティを確保できるソリューションは、Microsoft CorporationのMicrosoft Windows Serverオペレーティングシステムに含まれているMicrosoftターミナルサービスと見なされます。

現在、リモートユニットまたは従業員を持つRegionSoft CRMクライアントの90％以上が、ターミナル接続方法を使用しています。そして、「自己分離の時代」にCRMを導入しているのは、新しいお客様にとって絶対的な優先事項となっているのは彼です。 

方法2：DBMSポートを介して接続する


DBMSポートを介した接続は、中央データベースへのリモート接続の代替オプションです。これは、リモートステーションからFirebirdポートへの直接アクセスを使用します。リモートユーザーが使用しているデータはクライアント側に送信され、そこで処理されます。したがって、このリモート接続方法では、インターネットチャネルの速度と安定性がより厳しくなります。

互いに離れた場所にあるサーバーとワークステーションの両方が、1 Mb /秒以上の推奨される安定したチャネルでインターネットに接続されている必要があります。ワークステーションからデータベースに接続するための設定では、データベースサーバーの静的IPアドレスとDBMSポートを指定する必要があります（例：「192.168.0.122/3059」）。他のユーティリティは必要ありません。 CRMシステムはリモートワークステーションで実行されます。

作業はオンラインで行われます。リモートユーザーには、システムのすべての機能へのフルアクセスが与えられます。

この方法の欠点は、接続のセキュリティが低下し、低速の接続速度と不安定なインターネットチャネルでデータのダウンロード時間が長くなることです。

リモート接続デバイス


リモート接続のターミナル方式は、次のデバイスから実行できます。

  • Windows、MacOS、Linuxを実行するパーソナルコンピュータ。
  • Android、iOS、Windowsを実行するタブレットとスマートフォン。SmartTVテクノロジーを搭載した最新のTVを含む、RDPプロトコルをサポートするその他のデバイス。

DBMSポートを介したリモート接続の方法では、次のデバイスを使用できます。

  • Windows、Mac OS、Linuxを実行しているパーソナルコンピュータ。

注：端末およびリモートの操作モード、および仮想化テクノロジーの使用に関連する作業については、「Terminal＆Remote＆Mobile Access（TRM）」という接頭辞が付いた特別なRegionSoft CRMライセンスの購入が必要です。
