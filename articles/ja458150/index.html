C ++およびC＃コンパイラでののぞき穴のマイクロ最適化

学校では、方程式を解いたり式を検討したりするときに、最初にそれらを数回削減しようとしZ = X - (Y + X)ましたZ = -Y。最近のコンパイラでは、これはいわゆるピープホール最適化のサブセットであり、大まかに言えば、一連のパターンにより、式を減らし、特定のプロセッサ用の命令をより高速なものに置き換えます。この記事では、LLVM、GCC、および.NET Core（CoreCLR）ソースで見つかったそのような最適化のコレクションをまとめました。


簡単な例から始めましょう：


  X * 1        =>  X
-X * -Y        =>  X * Y 
-(X - Y)       =>  Y - X  
X * Z - Y * Z  =>  Z * (X - Y)

C ++とC＃で最後の例を確認してください：


int Test(int x, int y, int z) {
    return x * z - y * z;  //  =>  z * (x - y)
