配列ベースのコレクションを取得する最良の方法は何ですか？


配列に基づいたコレクションの形成から始めることをお勧めします。

ほとんどの場合、このメソッドが発生します：

Integer[] someArray = {9, 10, 11, 12}; 
List<Integer> list = Arrays.asList(someArray);

それは確かに機能しますが、すべて大丈夫ですか？また、代替ソリューションはありますか？

このアプローチの2つのマイナスが同時に思い浮かびます。

  • まず、Arrays.asListメソッドはListを返します。しかし、コレクションの別の実装が必要な場合はどうなりますか？Arrays.asListはこれを許可しませんが、代替案はもう少し検討されます。
  • 次に、Arrays.asListを呼び出して取得したリストは、サイズ変更をサポートしていません。多くの人がそのようなリストを扱うことから生じる例外を思いついたと思います。

でコレクションインタフェース、あなたはに代わる見つけることができますは、Arrays.asListの -法Collections.addAllの方法を。使用方法は次のとおりです。

//      (List, Set, ...) 
Collection<Integer> collection = ...; 
Integer[] someArray = {9, 10, 8, 7}; 
Collections.addAll(collection, someArray);

または単に： 

Collections.addAll(collection, 11, 12, 13, 14);

Collections.addAll メソッドは、Collectionオブジェクトと配列を入力として受け入れます。配列の代わりに、コンマで区切られた要素を指定することもできます。

メリットは何ですかCollections.addAllオーバーは、Arrays.asListは？

  • まず、Collections.addAll配列に基づいてコレクションを作成すると、入力Arrays.asListを使用したコレクションのaddAllメソッドよりもはるかに高速に動作します。これは、このメソッドのJavaDocにあります。
    この便利なメソッドの動作はc.addAll（Arrays.asList（elements））の動作と同じですが、このメソッドはほとんどの場合非常に高速に実行されます
  • さらに、Collections.addAllはListだけでなく、他のコレクションでも機能します。
  • また、この方法を使用する場合、サイズ変更の問題はありません。

配列、多次元配列、またはコレクションを印刷する最も簡単な方法は何ですか？


配列とコレクションの印刷表現を取得する問題に移りましょう。System.out.println（someArray）を

作成するだけの場合、次のような結果が得られます：[Ljava.lang.Integer; @ 6d06d69c。配列でtoString（） メソッドを使用した場合も、同様の結果が期待されます。 配列を出力するには、Arrays.toString（...）メソッドが役に立ちます。




Integer[] someArray = new Integer[]{1, 2, 3}; 
System.out.println(Arrays.toString(someArray));

この行の出力は次のようになります。 

[1、2、3]

多次元配列の場合は、Arrays.deeptoStringメソッドを使用できます。

int[][] a = { 
    {1, 2, 3}, 
       {4, 5, 6} 
}; 
System.out.println(Arrays.deepToString(a));

このスニペットの出力は次のようになります。 

[[1、2、3]、[4、5、6]]


したがって、要素を表示するために手動でループを介して配列をソートする必要はありません。このメソッドを使用するだけで十分です。Mapの

コレクションや実装は問題ありません。通常、配列を除くすべてのデータ構造が出力されます。 例があるとします：



Collection<Integer> someCollection = new HashSet<>(); 
someCollection.add(1); 
someCollection.add(2); 
System.out.println(someCollection); 

Map<Integer, String> someMap = new HashMap<>(); 
someMap.put(1, "Some 1"); 
someMap.put(2, "Some 2"); 
System.out.println(someMap);

以下の出力では、セットとマップの両方が読みやすい形式で表示されていることに注意してください。

[12] 
{1 =一部1、2 =一部2}


配列同士を比較するのは簡単ですか？


配列を比較する必要がある場合があります。Arraysクラスには、そのような比較を可能にするメソッドがあります。Arrays.equalsメソッドは、要素の数を比較し、対応する要素の同等性をチェックします。1つのフィールドと特定の等号

を持つElementsクラスがあるとしましょう

private class Element { 
    final String name; 
     private Element(String name) { this.name = name; } 

     @Override 
    public boolean equals(Object o) { 
        if (this == o) return true; 
        if (o == null || getClass() != o.getClass()) return false; 
        Element element = (Element) o; 
        return Objects.equals(name, element.name); 
    } 

    @Override 
    public int hashCode() { 
        return Objects.hash(name); 
    } 
}

3つの配列を定義します。 

Element[] firstElementArray = { new Element("a"), new Element("b"), new Element("c") }; 
Element[] secondElementArray = {new Element("c"), new Element("b"), new Element("a") }; 
Element[] thirdElementArray = { new Element("a"), new Element("b"), new Element("c") };

最初と3番目の配列の要素は同じ順序であることに注意してください。
これで、Arrays.equalsメソッドを使用して比較を実行できます。

System.out.println("first equals to second? " 
        + Arrays.equals(firstElementArray, secondElementArray)); 
System.out.println("second equals to third? " 
        + Arrays.equals(secondElementArray, thirdElementArray)); 
System.out.println("first equals to third? " 
        + Arrays.equals(firstElementArray, thirdElementArray));

結果は次のようになります。 

最初は2番目に等しいですか？偽
2番目は3番目と等しいですか？偽
最初は3番目に等しいですか？本当


配列を効率的にコピーする方法は？


多くの場合、ループを使用して手動で配列をコピーするコードを見ることができます。ただし、はるかに高速にコピーするSystem.arraycopyメソッドがあります。

私はそのような単純な例を見ることを提案します：

Element[] elements = { 
        new Element("a"), 
        new Element("b"), 
        new Element("c") 
}; 
Element[] copyOfElements = new Element[elements.length]; 
System.arraycopy(elements, 0, copyOfElements, 0, elements.length); 
System.out.println(Arrays.toString(copyOfElements));

要素の配列があります。同じ長さの空の配列を作成し、最初の要素から2番目の要素にすべての要素をコピーします。その結果、次の結論が得られます。

[要素{名前= 'a'}、要素{名前= 'b'}、要素{名前= 'c'}]


配列またはコレクションをさまざまな方法で並べ替える方法は？


配列は、Arrays.sort（someArray）メソッドを使用してソートできます。配列を逆順にソートする場合は、Collections.reverseOrder（）を2番目のパラメーターとしてこのメソッドの入力に渡すことができます。

たとえば、直接ソートしてから逆順にソートする配列があります。

String[] someArray = new String[]{"b", "a", "c"}; 

Arrays.sort(someArray); 
System.out.println(Arrays.toString(someArray)); 

Arrays.sort(someArray, Collections.reverseOrder()); 
System.out.println(Arrays.toString(someArray));

結論は次のようになります。 

[a、b、c] 
[c、b、a]


直接および逆ソートに加えて、大文字と小文字を区別せずに文字列の配列をソートする必要がある場合があります。これは、2番目のパラメータとしてString.CASE_INSENSITIVE_ORDERをArrays.sortに渡すことで簡単に実行できます。

Collections.sortは、残念ながら、List実装のソートのみを許可しています。

Javaをソートするアルゴリズムは何ですか？


Javaでの並べ替えについて最後に述べることができるのは、Javaでは「単純な並べ替え」が最も単純な型に使用され、「安定したマージ」がオブジェクトに使用されるということです。したがって、プロファイラーがこれが必要であることを示すまで、ソート方法の独自の実装の開発にリソースを費やすべきではありません。

配列があり、メソッドがIterableを受け入れる場合はどうなりますか？


Iterable を必要とするメソッドに配列を渡すなどの質問に進むことを提案します。Iterableは、Iterator （）メソッドを含むインターフェースであり、Iteratorが返す必要があることを思い出してください。

入力でIterableを使用するメソッドがある場合、そのように配列をそこに転送することはできません。forループで配列を反復処理できますが、Iterableではありません。

String[] someArray = new String[]{"a", "b", "c"}; 
for (String currentString : someArray) { 
    ... 
}

この例では、すべてが正常です。しかし、メソッドがある場合：

private static void someIteration(Iterable<String> iterable) { 
    ... 
}

その行はコンパイルされません： 

someIteration(someArray);

この状況での唯一の解決策は、配列をコレクションに変換し、すでにそのようなメソッドにフィードすることです。
