WSLおよびVisual Studio Code Remoteを使用したWindowsでのLinux開発

Linux用のVS Code RemoteおよびWindowsサブシステム（WSL）を使用すると、Windowsがプリインストールされたラップトップまたはデスクトップでフル機能のLinux開発環境を使用できます。この記事では、これらのツールを使用してLinux上でPythonアプリケーションを開発する方法について説明します。



まず最初に、必要なコンポーネントをインストールします


この記事で説明するすべての手順を完了するには、VS Code Remote拡張機能（WSL）がインストールされたWSLおよびVS Codeが必要です。それが何であるか、それをどのようにインストールするかを簡単に見てみましょう。

Linux用のWindowsサブシステム


LinuxツールのWindowsサブシステム（略称WSL）を使用すると、変更されていないLinuxバイナリをWindowsで直接実行できます。つまり、デュアルブートなどの複雑な設定を必要とせずに、お気に入りのLinuxツール、コマンドラインアプリケーション、およびワークフローをWindowsコンピューターで使用できます。そして何よりも、WSLはWindowsとLinuxの間で緊密に統合されたソリューションです。つまり、Linuxが好きな場合は、すべてのWindowsファイルと実行可能ファイルにbashシェルから直接アクセスでき、Windowsが好きな場合は、PowerShellまたはCMDからLinuxバイナリとファイルに簡単にアクセスできます。

WSLが初めてで、さらに詳しく知りたい場合は、次のフルコースを受講できます。LinuxのWindowsサブシステムで作業を開始するか、WSLドキュメントのインストール手順のみを読んでください。

ビジュアルスタジオコード


Visual Studio Codeは、軽量で強力なコードエディターで、デスクトップで実行され、Windows、macOS、Linuxで使用できます。JavaScript、TypeScript、Node.jsのサポートが組み込まれており、他の言語（C ++、C＃、Java、Python、PHP、Goなど）およびランタイム（.NETやUnityなど）向けの拡張機能の豊富なエコシステムもあります。ダウンロードして、VS Codeで旅を始めましょう。

また、VS Code Remote-WSL拡張機能も必要です。これにより、完全に機能する開発環境として、コンテナー、リモートコンピューター、またはLinux用のWindowsサブシステム（WSL）を使用できます。詳細を確認して開始するには、VS Code Remoteのドキュメントページにアクセスしてください。

はじめに、注意、Linux！


WSLでVS Code Remoteを使い始めるにあたって長いセクションを書く代わりに、VS Code Remoteを使用したWSLでのPython開発を示すこの短いビデオを撮影することにしました。


これはPythonだけで隔離されているわけではなく、任意のLinuxプロジェクトでVSCode RemoteとWSLを使用できます。


WSLでVS Code Remoteを使用する場合、コマンドと拡張機能はLinuxディストリビューションで直接実行されるため、バイナリ互換性の問題や他のクロスOSの問題を心配する必要はありません。 WSLでは、Windowsと同じ方法でVSコードを使用できます。

VS Code RemoteとWSLは、WindowsでのLinux開発に関して、ゲームのルールを変更します。これらのツールを使用すると、大量のLinuxアプリケーションをWindowsコンピューターで直接開発、実行、デバッグできるため、WindowsとLinuxの両方の優れた機能を利用できます。
