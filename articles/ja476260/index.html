私たちはrpm WireMockで収集します-UbuntuのPPAのFedora COPRアナログでWebサービスを介してスタブを作成するためのユーティリティ

WireMockは、Webサービスを介してスタブを作成するためのJavaライブラリユーティリティです。実際のWebサービスとして接続できるHTTPサーバーを作成します。


Fedora COPRは、カスタムリポジトリをホストするための無料のホスティングサービスです（Arch LinuxのAURまたはUbuntuのPPAと同様）。機能のうち、PIPとRubyGemsの名前を指定してrpmパッケージを収集する組み込み機能。


この投稿では、Fedora COPRにコミットしてリポジトリからrpmを収集する方法を説明します。


たとえば、specファイルとsystemdユニットファイルを作成したwiremock-rpmリポジトリを見てみましょう。リポジトリを作成するときは、wiremock-rpmをベースにして、必要なものを変更できます。specファイルとsystemdユニットファイルの書き込みは省略します。私はあなたが知っていると思います。

WireMock.


指示に従ってワイヤモックをインストールします。私のプロジェクトのwiremock作業ディレクトリは/usr/lib/wiremock/です。このディレクトリには、wiremock.jarとマッピングディレクトリが含まれています。マッピングディレクトリには、wiremockに送信するリクエストと準備された回答を含むjsonファイルがあります。


http://wiremock.org/docs/running-standalone/の例：


{
    "request": {
        "method": "GET",
        "url": "/api/mytest"
    },
    "response": {
        "status": 200,
        "body": "More content\n"
    }
}

/ api / mytestにリクエストを送信して取得します。


curl http://localhost:8080/api/mytest
More content
