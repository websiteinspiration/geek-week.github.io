コントロールの並べ替え


VoiceOverの制御を渡す順序を正しく設定しないと、ほとんどの場合、必要に応じて要素がバイパスされ、誤った順序で要素が読み取られます。たとえば、これは「閉じる」ボタンと「バスケットへ」ボタンで発生しました。

画面がスクロールし、ボタンが上に表示されますUIScrollView。VoiceOverは最初に内部のすべての要素をバイパスしようとし、UIScrollViewそれから上部のボタンを見つけます。ユーザーにとって、VoiceOverのこの動作は正しくありません。ボタンが一番上にあるため、反復とスコアリングはボタンから始める必要があります。

まず、VoiceOverがコントロールの順序を決定する方法を理解しましょう。彼はこのようにしていますaccessibilityElements。プロパティから要素を取得します。デフォルトではview、それらはすべて存在しisAccessibilityElement = trueます。

これで、次のコードをオーバーライドして、ボタンを最初に配置できますaccessibilityElements。

override var accessibilityElements: [Any]? {
    get {
        var elements = [Any]()
            
        elements.append(contentsOf: [closeButton, cartButton])
        elements.append(contentsOf: contentScrollView.accessibilityElements)
            
        return elements
    }
    set { }
}

見えないフレームを揃える


読み取り順序はフレームごとにカウントされますが、予期しない結果になる場合があります。たとえば、「i」ボタンの枠を拡大してクリックしやすくしました。しかし、ピザの名前のフレームよりも高いことが判明したため、「i」ボタンが最初のフォーカス要素であることが判明しました。小さいですが、右側にあり、一般的にそれほど重要ではありません。



読み順はから変更できますがaccessibilityElements、別の方法を示します。VoiceOverはプロパティを使用しますaccessibilityFrame。デフォルトでは、通常のフレームと一致します。いくつかの解決策があります：

  1. コントロールサブクラスをオーバーライドし、削減された値を返します。
  2. 外に正しいフレームを設定してください。
  3. フレームを調整して、碑文と同一平面になるようにします。

ただし、このフレームが画面の座標内にあることが重要です。単純な変換のための関数がありますUIAccessibility.convertToScreenCoordinates。

また、コントロールの組み合わせにも使用できます。たとえば、スイッチャーとその署名を組み合わせる必要があるため、要素が大きくなり、クリックしやすくなり、不要な重複がなくなります。

override func layoutSubviews() {
        super.layoutSubviews()
        repeatSwitch.accessibilityLabel = repeatLabel.text
        repeatSwitch.accessibilityFrame = UIAccessibility.convertToScreenCoordinates(
                repeatSwitch.frame.union(repeatLabel.frame).insetBy(dx: -12, dy: -12),
                in: repeatSwitch.superview!)
}
