Unity Canvasを使用したUnityの構成のインターフェイス

Unityには、UI Canvasユーザーインターフェイスを作成するための優れたシステムがあります。多くのトレーニング資料が書かれていますが、ほとんどのガイドでは、どのボタンを押すか、すべてのコードをどのコードで機能させるかを説明しています。例として、小さなインターフェイスは通常、メインメニューと設定の2つのウィンドウで構成されます。ただし、ゲームにははるかに多くのウィンドウがあり、すでに少なくとも12のウィンドウがある場合は、何らかの組織の必要性が生じます。この記事の一部として、この問題の解決方法についてお話したいと思います。

まず、インターフェースで個々のウィンドウを選択する必要があります。ウィンドウとは、いくつかのコントロールを備えたパネルを意味します。

ウィンドウの例






各ウィンドウには、すべてのコントロールを含むルートオブジェクトが必要です。このオブジェクトはウィンドウ全体を表します。たとえば、コントロールが配置されているパネルがある場合、これらの要素をパネルに対して子にすることは論理的です。コンポーネントは、各ウィンドウに接続されます。これは、抽象ウィンドウクラスの子孫です。

窓


ウィンドウは、ウィンドウが子ウィンドウを持つことができるという前提で書かれています。現在のウィンドウから同時に開くことができるウィンドウで、同時に1つの子ウィンドウを開くことができ、残りのウィンドウはすべて閉じる必要があります。この目的のために、クラスには、現在開いているウィンドウへのリンクが格納されているCurrentWindowプロパティが含まれています。また、ウィンドウが開いたことを報告するOnOpenイベントもあります。 ChangeCurrentWindow（）メソッドは、子ウィンドウの近くでこのイベントにサブスクライブできるため、1つの子ウィンドウが開かれると、開いている子ウィンドウが閉じられ、現在開いているウィンドウへのリンクが変更されます。以下に、実装例を示します。また、クラスにはSelfClose（）およびSelfOpen（）メソッドがあり、これらのメソッドはウィンドウの開閉方法を制御します。 Awake（）メソッドでは、ウィンドウはUIManagerに登録されます。ウィンドウへのリンクが追加されます。

public abstract class Window : MonoBehaviour
{
    public bool IsOpen { get; private set; }
    public Window CurrentWindow { get; protected set; } = null;

    public delegate void OpenEventHandler(Window sender);
    public event OpenEventHandler OnOpen;

    void Awake()
    {
        UIManager.Instance.Windows.Add(this.gameObject);
    }

    public void Open()
    {
        IsOpen = true;
        if (OnOpen != null)
            OnOpen(this);

        SelfOpen();
    }

    protected abstract void SelfOpen();
    
    public void Close()
    {
        IsOpen = false;

        if (CurrentWindow != null)
            CurrentWindow.Close();
        SelfClose();
    }

    protected abstract void SelfClose();

    protected void ChangeCurrentWindow(Window sender)
    {
        if (CurrentWindow != null)
            CurrentWindow.Close();
        
        CurrentWindow = sender;
    }
}

UIManager


次に、UIManagerクラスに移ります。これはシグルトンクラスであるため、必要に応じてすべてのウィンドウがアクセスできます。ご覧のとおり、Windowsウィンドウのリストがあり、ステージ上のすべてのウィンドウへのリンクが格納されています。InitUI（）は何かを初期化するために必要です。たとえば、プレハブからウィンドウを作成する場合、ここでそれらのインスタンスを実行できます。
Start（）メソッドでは、最初から開く必要があるウィンドウを開くことができます。逆の場合も、不要なウィンドウを閉じることができます。Get（）メソッドを使用すると、特定のウィンドウへのリンクを取得できます。

using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using System;
public class UIManager : MonoBehaviour
{
    public static UIManager Instance = null;

    public List<GameObject> Windows;

    void Awake()
    {
    if (Instance == null)
        Instance = this;
    else if (Instance != this)
        Destroy(gameObject);

    InitUI();
    }

    private void InitUI()
    {
        //to do
    }

    void Start()
    {
        foreach(var window in Windows)
        {
            var windowComponent = window.GetComponent<Window>();
            if (windowComponent is StartWindow)
                windowComponent.Open();
            else
                windowComponent.Close();
        }
    }

    public Window Get<T> () where T : Window
    {
        foreach(var window in Windows)
        {
            var windowComponent = window.GetComponent<Window>();
            if (windowComponent is T)
                return windowComponent;
        }
        return null;
    }
}

設定ウィンドウ


例として、設定ウィンドウの実装を示します。

using System.Collections;
using System.Collections.Generic;
using UnityEngine;
public class SettingsWindow : Window
{
    private VideoSettingsWindow videoSettingsWindow;
    private LanguageSettingsWindow languageSettingsWindow;
    private AudioSettingsWindow audioSettingsWindow;
    private ControlSettingsWindow controlSettingsWindow;
    
    public void Start()
    {
        videoSettingsWindow = UIManager.Instance.GetWindow<VideoSettingsWindow>();
        languageSettingsWindow = UIManager.Instance.GetWindow<LanguageSettingsWindow>();
        audioSettingsWindow = UIManager.Instance.GetWindow<AudioSettingsWindow>();
        controlSettingsWindow = UIManager.Instance.GetWindow<ControlSettingsWindow>();

        videoSettingsWindow.OnOpen += ChangeCurrentWindow;
        languageSettingsWindow.OnOpen += ChangeCurrentWindow;
        audioSettingsWindow.OnOpen += ChangeCurrentWindow;
        controlSettingsWindow.OnOpen += ChangeCurrentWindow;
    }

    protected override void SelfOpen()
    {
        this.gameObject.SetActive(true);
    }

    protected override void SelfClose()
    {
        this.gameObject.SetActive(false);
    }

    public void Apply()
    {

    }

    public void VideoSettings()
    {
        videoSettingsWindow.Open();
    }

    public void LanguageSettings()
    {
        languageSettingsWindow.Open();
    }
   
    public void AudioSettings()
    {
        audioSettingsWindow.Open();
    }

    public void ControlSettings()
    {
        controlSettingsWindow.Open();
    }
}

設定ウィンドウから、他の4つのウィンドウを開いて1つのウィンドウのみを開くことができるように、設定ウィンドウのChangeCurrentWindow（）メソッドを子ウィンドウのOnOpenイベントにサブスクライブするため、開いているウィンドウは他のウィンドウが開いたときに閉じられます。SelfOpen（）およびSelfClose（）の実装は、ウィンドウをアクティブ化または非アクティブ化するだけです。

したがって、簡単に拡張可能なUIシステムが得られます。インスペクターなどのウィンドウへのリンクを手動で追加する必要はありません。新しいウィンドウを追加するには、適切なクラスを作成し、それをWindowから継承するだけです。そのようなシステムにはいくつかの欠点があります。ウィンドウが多いほど、作成する必要のあるクラスが多くなり、ウィンドウへのリンクの検索がリンクの配列を並べ替えることによって行われるという事実ですが、私の意見では、これらの欠点には利点があります。
