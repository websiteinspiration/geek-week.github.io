Unityを使用して別のプラットフォーマーを開発する方法。別のチュートリアル、パート2

最初の記事（ここにあります）の続きで、「2Dゲームのレベルデザインパターン」という記事に基づいてプラットフォーマーを開発し続けています。

最初の記事のリリース後、そこに記述されているボタンコントロールは完全に便利ではないことが明確に決定されました。したがって、ゲームのコントロールがジョイスティックでやり直されました。さらに、残念ながら、ゲームはプレイマーケットでモデレートに合格しませんでした。先週の金曜日、メタデータの収集が原因でプロジェクトが拒否されたという通知を受け取りました。ちなみに、私の最初のプラットフォーマー「Knight Kadavar」も、通話の管理とSMSの表示の許可を求める申し立てがあったため、初めて拒否されました（ボットの一部がまったく愚かでした。ゲームには、許可は必要ありませんでした）。それからGoogleは私になぜそれが必要なのかについて書面による通知を要求しました。しかし、私が気付いたいくつかのエラーを修正してゲームを再モデレートに送ったという事実ですべてが終わりました。彼女は正常にプレイマーケットに追加されました。さて、私はこのゲームでもまったく同じことをするつもりです。

したがって、2番目のシーンが作成されるとすぐに、パターンは自動的に閉じられました

ボーナス


プレイヤーに良い影響を与えるコレクターアイテム私たち

はルーカスに2種類のボーナスを実装します：

  • モンスターを破壊するとドロップする応急処置キット
  • ルーカスが敵に向けて投げることができる剣が入った胸

応急処置キットでボーナスを実装するには、モンスターでプレハブを変更する必要があります。具体的には、ボーナスが表示される場所を示します。



次に、Enemy.csスクリプトを変更します。

スポイラーの見出し
/* ,        */
	[SerializeField]
	private GameObject bonusPref; //  
	[SerializeField]
	private Transform instBonus; //    
	[SerializeField]
	private int isBonus; //    ,    
	/*   */

public void ifDie()
	{
		if (Damage(0) <= 0)
		{
			isBonus = Random.Range(0,3);
			if (isBonus == 0)
			{
				Instantiate(bonusPref, instBonus.position, instBonus.rotation);
			}
			Destroy(this.gameObject);
		}
	}


返されたタイプDamage（0）で関数を呼び出し、ヘルス0が返されるかどうかを確認します。返される場合は、乱数ジェネレータを呼び出します。ジェネレーターが番号0の選択で停止した場合、ボーナスをプレーヤーに投げ、モンスターを破壊します。

次に、このボーナスで何ができるかを説明します。これを行うには、コンポーネントSpriteRenderer、BoxCollider2DおよびRigidbody2Dを使用してプレハブを作成します。また、リンゴがプレーヤーと衝突した場合に必要な処理を担当するスクリプトを作成します。

スポイラーの見出し

これらのコード行はどういう意味ですか？最初に、コライダーの配列を定義し、キューブに落ちたすべてのものをチェックします。

Collider2D[] ED = Physics2D.OverlapBoxAll(Hero.position, new Vector2(attackInBoxX, attackInBoxY), lEnemy);

渡されるパラメーターに注意してください。これらは、OvelapCircleAllで使用されるパラメーターとは大きく異なります。主なパラメータ-

Vector2(attackInBoxX, attackInBoxY)

次に条件が満たされ、コライダー配列が0より大きい場合は、剣の在庫を確認します。剣の数が0の場合、何もせずにメソッドを実行します
ED[i].GetComponent<Enemy>().Damage(1);
通常のヒットのように。0より大きい場合は、剣を解放し、剣の数を1つ減らします。

次に、instantiateSword（）メソッドを実装します。

スポイラーの見出し
 private void instantiateSword()
    {
		GameObject newArrow = Instantiate(swordPref) as GameObject;
		newArrow.transform.position = instSword.transform.position;
		Rigidbody2D rb = newArrow.GetComponent<Rigidbody2D>();
		if (GameObject.Find("Hero").GetComponent<HeroScript>().localScale.x > 0)
		{
			rb.velocity = new Vector3(swordSpeed, 0, 0);
		}
		else
		{
			rb.velocity = new Vector3(-swordSpeed, 0, 0);
			newArrow.transform.Rotate(0,0,-180);
		}
	}


前の記事をよく読んでいると、このコードがヒマワリの撮影を担当するコードのセクションに似ていることに気づいたかもしれません。Lucasのスケールを決定するためのコードのこのセクションに行を追加しました。つまり、彼は左または右に見えます。

(GameObject.Find("Hero").GetComponent<HeroScript>().localScale.x > 0)

newArrow.transform.Rotate(0,0,-180);
