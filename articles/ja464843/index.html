C ++でのASIOの有用または開発に満足


プログラマーでありエレキギター愛好家である私は、音楽ソフトウェアの開発から離れることはできませんでした。ある種のギタープロセッサを使用してエレキギターをWindowsコンピュータに接続しようとしたことのある人なら誰でも、これらの目的でASIOインターフェイスが必要になることが多いことを知っています（他の目的も可能ですが、ASIOが最も人気があります）。これは、オーディオの入出力の処理にかなりの時間が費やされ、その結果、楽器を演奏するときに不快な遅延が聞こえ、ゲームを非常に難しくするという事実によるものです。ASIOを使用すると、サウンドのミキシングの段階をスキップして、遅延を大幅に減らすことができます。詳しくはこちらをご覧ください。

何が問題ですか


このドライバーは人気があり便利なので、ASIOを使用することにしましたが、ここで問題が発生しました。このテクノロジーの開発についての記事はほとんどありません。実際、私が最初に案内できるのは、SDKの公式ドキュメントといくつかの英語フォーラムです。同様の問題がある場合は、この記事が少なくとも少し役立つことを願っています。

次に、音声ストリームをマイクから音声出力デバイスに送る必要があります。これを行うにはいくつかの方法があります。最も簡単な方法は、チャネルを「ミラーリング」することです。

マイク、左イヤホン、右イヤホンの3つのチャンネルがあります。マイクから出力チャンネルに信号を転送するには、関数を使用します

BOOL BASS_ASIO_ChannelEnableMirror(
    DWORD channel -  
    BOOL input2 -   ? (0  1)
    DWORD channel2 –    ( )
);

ここではすべてが簡単です。信号を入力チャネル0から出力チャネル0および1に送信する必要があります。

BASS_ASIO_ChannelEnableMirror( 0, 1, 0 );
BASS_ASIO_ChannelEnableMirror( 1, 1, 0 );

次に、疑問が生じます。信号を送信する必要があるチャネルをどのようにして見つけたのですか。回答-チャネルに関する情報は、BASS_ASIO_ ChannelGetInfo関数を使用して見つけることができます。

a = 0; BASS_ASIO_CHANNELINFO channel_info;
std::cout << "inputs: " << std::endl;	
for (a = 0; BASS_ASIO_ChannelGetInfo(0, a, &channel_info ); a++ )
	std::cout << a << ") " << channel_info.name << " format: " << channel_info.format << std::endl;	
std::cout << "Outputs: " << std::endl;
for (a = 0; BASS_ASIO_ChannelGetInfo(1, a, &channel_info); a++)
	std::cout << a << ") " << channel_info.name << " format: " << channel_info.format << std::endl;
std::cout << "__________" << std::endl;



すべての設定が完了しました。Start- BASS_ASIO_Start。入力に最大サンプル長とストリーム数のパラメーターを指定できますが、このタスクでは、これらのパラメーターをデフォルトのままにすることができます（ゼロで埋めます）。

BASS_ASIO_Start( 0, 0 );

実際、それだけです。ただし、これはコンソールアプリケーションです。システム（「一時停止」）を忘れないでください。起動直後に閉じず、その直後にメモリを解放して、デバイスとドライバーの操作を停止します。

BASS_ASIO_Stop();
BASS_ASIO_Free();
BASS_Stop();
BASS_Free();
