精度計算でのRの使用

ビット精度がとき定期的に、問題があっても、日常の生活の中で、そこにあるfloat64/ int64必要な精度で答えを得るには十分ではありません。別の楽器を探して急いで？オプションもあります。


または、これを行うことはできませんが、好奇心を示し、任意の精度で計算するために、ほとんどすべての言語のラッパーが存在するGNU MPFRライブラリが長い間作られていることがわかります。実践では、このライブラリに精通している人はほとんどいないことを示しています。これは、大学やそれに続くプログラマーの主流である学習プログラムの特殊性が原因である可能性があります。


ライブラリは優れており、少なくともその視野を広げる枠組みの中で注意を払う価値があります。R to itにはRmpfrラッパーがあります。以下では、小学生の問題に関する簡単な例を示し（まあ、NDAの下の設計データには触れないでください）、ほとんどすぐに攻撃されるいくつかの古典的なレーキに触れます。


これまでの刊行物の続きです。


たとえば、量子での2019/2020学年コンテストの問題No. 39を考えてみましょう。
正の数xとyは、左の分数の下の不等式が右の分数より大きいようなものです。xとyのどちらが大きいですか。



当然、これは分析的に解決する必要があります（交互の不等式）。ただし、これをデモンストレーションとして使用しない理由にはなりません。


分数のシーケンスを計算する単純なコードを描画します。すぐに最終値を設定できますが、次にのデモシーンなmpfrので、少しずつ移動します。


標準的な方法で解決します 
library(tidyverse)
library(magrittr)
options(digits=15)

frec <- function(stopval, n, x){
  res <- ifelse(n == stopval, 
                (stopval - 1) + stopval/(stopval + 1 + x), 
                (n - 1 ) + n / (frec(stopval, n + 2, x))
  )
  res
}

frec_wrap <- function(stopval, x){
  res <- frec(stopval = stopval, n = 1, x = x)
  print(glue::glue("{stopval}: {res}"))
  res
}

sol_df <- tibble(stopval = seq(1, 29, by = 2)) %>%
  mutate(val1 = purrr::map_dbl(stopval, frec_wrap, x = 1),
         val2 = purrr::map_dbl(stopval, frec_wrap, x = 5),
         delta = val1 - val2)

library(tidyverse)
library(magrittr)
library(Rmpfr)
frec2 <- function(stopval, n, x){
  if(n == stopval){
    (stopval - 1) + stopval/(stopval + 1 + x)} else {
      (n - 1 ) + n / (frec2(stopval, n + 2, x))
    }
}
frec2_wrap <- function(stopval, x){
  .precision <- 5000
  res <- frec2(stopval = mpfr(stopval, .precision), 
               n = mpfr(1, .precision), 
               x = mpfr(x, .precision)
  )
  print(glue::glue("{stopval}: {formatMpfr(res, drop0trailing = TRUE)}"))
  res
}

sol2_df <- tibble(stopval = seq(1, 29, by = 2)) %>%
  mutate(val1 = purrr::map(stopval, frec2_wrap, x = 1),
         val2 = purrr::map(stopval, frec2_wrap, x = 5))
