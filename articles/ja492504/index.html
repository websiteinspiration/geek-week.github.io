JSモジュールを2回インポートするとどうなりますか？

この記事の質問から始めましょう。ES2015モジュールにincrementは、次のコードが含まれています。

// increment.js
let counter = 0;
counter++;

export default counter;

呼び出す別のモジュールでconsumerは、上記のモジュールが2回インポートされます。

// consumer.js
import counter1 from './increment';
import counter2 from './increment';

counter1; // => ???
counter2; // => ???

そして今、実際に質問。変数counter1とcounter2モジュールの後に何が入りますconsumerか？ この質問に答えるには、JavaScriptがモジュールを実行する方法と、それらがどのようにインポートされるかを理解する必要があります。





モジュールの実行


内部のJavaScriptメカニズムがどのように機能するかを理解するには、仕様を調べると便利です。

仕様によると、各JavaScriptモジュールはモジュールレコードエンティティに関連付けられています。このエントリにはEvaluate()、モジュールが実行するメソッドがあります。

このモジュールがすでに正常に実行されている場合は、undefinedを返します。[...]。それ以外の場合は、このモジュールのすべての依存関係を推移的に実行してから、このモジュールを実行します。
その結果、同じモジュールが1回だけ実行されることがわかります。

残念ながら、私たちの質問に答えるために知っておくべきことはこれに限定されません。指示呼び出しを確実にする方法import 同じパスを使用すると、同じモジュールが返されますか？

インポートコマンドを許可する


抽象操作HostResolveImportedModule（）は、モジュール（指定子）へのパスを特定のモジュールにリンクする責任があります。モジュールのインポートコードは次のようになります。

import module from 'path';

仕様に記載されている内容

は次のとおりです。HostResolveImportedModuleの実装は、次の要件を満たしている必要があります。

  • 通常の戻り値は、モジュールレコードの特定のサブクラスのインスタンスである必要があります。
  • ペアreferencingScriptOrModule、specifierに対応するモジュールレコードエンティティが存在しない場合、またはそのようなエンティティを作成できない場合は、例外がスローされます。
  • この操作が、引数としてreferencingScriptOrModuleと指定子の特定のペアを渡して呼び出されるたびに、通常の実行の場合、モジュールレコードエンティティの同じインスタンスを返す必要があります。

これをより理解しやすい方法で検討してください。

HostResolveImportedModule(referencingScriptOrModule, specifier)-パラメータの組に対応するモジュールを返す抽象オペレーションreferencingScriptOrModule、specifier。

  • パラメータreferencingScriptOrModuleは現在のモジュール、つまりインポートするモジュールです。
  • パラメータspecifierは、命令のモジュールパスに一致する文字列ですimport。

説明の最後HostResolveImportedModule()に、同じパスに対応するモジュールをインポートすると、同じモジュールがインポートされると言われています。

import moduleA from 'path';
import moduleB from 'path';
import moduleC from 'path';

// moduleA, moduleB  moduleC -    

moduleA === moduleB; // => true
moduleB === moduleC; // => true

興味深いことに、この仕様は、ホスト（ブラウザ、Node.js環境、一般に-JavaScriptコードを実行しようとするものすべて）が特定の実装を提供する必要があることを示していますHostResolveImportedModule()。

回答


仕様を注意深く読んだ後、JavaScriptモジュールがインポート中に一度だけ実行されることが明らかになります。また、同じパスを使用してモジュールをインポートすると、モジュールの同じインスタンスが返されます。

質問に戻りましょう。

モジュールはincrement常に1回だけ実行されます。

// increment.js
let counter = 0;
counter++;

export default counter;

モジュールがインポートされた回数に関係なくincrement、式はcounter++1回だけ評価されます。counterデフォルトのエクスポートメカニズムを使用してエクスポートされる変数は重要1です。

モジュールを見てみましょうconsumer：

// consumer.js
import counter1 from './increment';
import counter2 from './increment';

counter1; // => 1
counter2; // => 1

コマンドimport counter1 from './increment'とimport counter2 from './increment'同じパスを使用します- './increment'。その結果、モジュールの同じインスタンスがインポートされていることがわかります。同じ値1 が

変数counter1とcounter2にconsumer書き込まれていることがわかります。

概要


簡単な質問を検討したところ、JavaScriptモジュールがどのように実行およびインポートされるかについてさらに知ることができました。

モジュールのインポート時に使用されるルールは非常に単純です。同じモジュールが1回だけ実行されます。つまり、モジュールレベルのスコープ内で実行されるのは1回だけです。一度実行済みのモジュールを再度インポートすると、再度実行されることはありません。モジュールをインポートするとき、前のセッションの結果として得られたものを使用して、エクスポートするものを正確に見つけます。

モジュールが複数回インポートされても、モジュール指定子（モジュールへのパス）が同じである場合、JavaScript仕様により、モジュールの同じインスタンスがインポートされます。
