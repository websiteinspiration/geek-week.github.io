Waktu terbaik untuk merilis proyek kesayangan

Setiap pengembang melakukan proyek hewan peliharaan yang tidak pernah melihat cahaya hari. Saya punya selusin proyek semacam itu, dan minggu ini saya hampir saja menambah daftar kegagalan, tetapi rilisnya tetap terjadi. Dalam posting ini saya akan membagikan hasil refleksi dan pada akhirnya saya akan menjawab pertanyaan kapan akan merilis rilis pertama dari proyek hewan peliharaan.

Ketika Anda memulai proyek hewan peliharaan berikutnya, Anda harus memahami area subjek yang tidak dikenal. Untuk mewujudkan ide dan mendapatkan kepuasan dari hasil akhirnya, Anda harus mengkonsumsi konten. Ini meningkatkan tingkat kompetensi dalam bidang subjek, tetapi membutuhkan waktu.


Jadwal 1 - perubahan tingkat kompetensi dari waktu ke waktu. Pada sumbu x - waktu, pada sumbu y - tingkat kompetensi dalam sesuatu atau, misalnya, di bidang pemrosesan bahasa alami.

Awalnya, tingkat antusiasme melampaui, tetapi akhirnya berkurang menjadi nol, dan orang tersebut berhenti dari pelajaran. Biarkan saya mengingatkan Anda, kita berbicara tentang proyek hewan peliharaan yang tidak menyiratkan imbalan keuangan atau pemenuhan kewajiban kepada orang lain.


Bagan 2 - perubahan tingkat antusiasme seiring waktu. Sepanjang sumbu x, waktu, sumbu y adalah tingkat antusiasme.

Jika kita meletakkan grafik di atas satu sama lain, kita akan melihat titik persimpangan antusiasme dan kompetensi. Saya pikir poin ini cocok untuk rilis pertama dari proyek hewan peliharaan. Semakin jauh Anda mendorong rilis di sepanjang sumbu x, semakin tinggi risiko bahwa proyek tidak akan melihat cahaya hari. Semakin sedikit antusiasme, semakin Anda rela beralih ke kegiatan lain. Dimungkinkan untuk melepaskan rilis pertama ke titik persimpangan, tetapi hasilnya mungkin tidak memuaskan. Semakin awal rilis, semakin sedikit pengetahuan tentang area subjek.


Jadwal 3 - antusiasme dan kompetensi.

Poin ini sangat terasa ketika melihat kembali jalur panjang proyek kesayangan Anda sendiri. Saya cukup antusias selama seminggu, oleh karena itu, saya harus merencanakan rilis pertama dalam seminggu. Jika saya mengerti bahwa dalam seminggu saya tidak terbiasa dengan bidang subjek baru cukup untuk membuat versi pertama, saya tidak akan mulai mengerjakan proyek, atau saya akan menggunakan trik untuk menjaga tingkat antusiasme.

Setiap orang memiliki waktu sendiri. Jika Anda memiliki cukup antusiasme selama sebulan - jangan rencanakan rilis pertama dalam enam bulan.
