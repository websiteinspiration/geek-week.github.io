2つの機能：Intel Optane Memory H10（パート1）

パート1 >> パート2

SSDキャッシングは古くからあり、高速ストレージデバイスから最大限のパフォーマンスを引き出すことができます。近年、不揮発性の3D XPointメモリを使用するIntel Optane製品は、小型で高価な非常に高速なドライブの領域を支配しています。 Intelの第3世代Optaneメモリキャッシングメモリにより、Optaneのパフォーマンスは新しい製品セグメントに適合します。



最初のOptaneメモリドライブは、低速のSATAドライブ、特に機械式ハードドライブへのアクセスを高速化するように設計された小さなNVMe SSDでした。 Intelは現在、OptaneフラッシュメモリとQLC NANDの組み合わせに重点を置いて、OptaneメモリSSDを使用して他のNVMe SSMをキャッシュし続けています。今、彼らは両方のタイプのSSDを同じM.2モジュールに配置して、新しいOptane Memory H10を作成しています。

Intel Optane Memory H10によって初めて、インテルはOptaneメモリキャッシングテクノロジーをウルトラブックに実装できるようになりました。ウルトラブックでは、SSDは1つしかありません。SATAについて話すことはありません。特にIntelの4ビットフラッシュメモリQLC NANDの高密度により、2つのディスクを片側80 mm M.2モジュールに圧入することが可能になりました。 Intel 660p QLC SSDには、バージョン1 TBおよび512 GBのボード上に多くの空き容量があります。Optaneキャッシュは、QLC NANDのパフォーマンスと耐久性の欠点を潜在的に補うことができます。 1つのモジュールに2種類のメモリを配置しても、H10のデザインは非常にシンプルです。



Optane Memory H10は、OptaneおよびQLCドライブモジュールを単一のデバイスのように見せるために、新しいASICまたはその他のハードウェアを使用していません。キャッシングは完全にソフトウェアで制御され、ホストシステムは独立してOptaneおよびQLC H10にアクセスします。ドライブの各半分には、2つの専用PCIeレーンがあります。以前は、すべてのOptaneメモリSSDはPCIe x2デバイスであったため、新しいデバイスで何も失うことはありませんでした。しかし、Intel 660pは4xコントローラーのシリコンモーションNVMeを使用しています。率直に言って、660pはx2チャネルが提供できる以上の帯域幅をほとんど必要としなかったので、これは実際にはボトルネックではありません。



低速のQLC SSDと高速のOptane SSDを単一のデバイスに搭載したIntelは、ドライブのパフォーマンス特性を決定する際に厳しい決定を下さなければなりませんでした。 2つの大きなH10容量（512 GBと1 TB）の場合、2 GB /秒を超える順次読み取り速度が宣言されています。これは、QLCとOptane H10の両方から同時にデータを抽出するIntel Optane Memoryソフトウェアの機能を反映しています。記録は、異なるタイプのメモリ間で交替することもできますが、最大定格は、デバイスのパフォーマンスの明らかな制限を超えません。 H10のランダムI / Oの仕様は、Optaneメモリを搭載したデバイスのパフォーマンスと660p SSDの中間ですが、Optaneのパフォーマンスに非常に近くなっています。 Intelは理想的なキャッシュヒット率を宣伝しようとはしていませんが、期待しています実際の使用には十分であること。

OptaneキャッシュはQ10 H10が書き込む負荷を軽減するはずですが、Intelは依然としてデバイス全体の耐久性を1日あたり同じ0.16ディスクの書き換えと推定しています。これは、660p QLC SSDと同じ耐久性です。







Intel Optane Memory H10のマーケティング写真は、ドライブの二重の性質を強調するために2色のプリント回路基板を備えたSSDを特徴としていますが、実際にはドライブボードはモノクロです。ボードのレイアウトは、2つのコントローラーと3つのタイプのメモリのおかげでユニークですが、ベースとなる2つのディスクリート製品も明確に示しています。 QLC NANDを備えたドライブの半分はM.2コネクタの近くにあり、SM2263コントローラーと1つのDRAMおよびNANDパッケージを備えています。おなじみのシリコンモーションテスト/デバッグ接続は、NANDとOptaneの境界にあります。 Optane側には、小さなIntel Optaneコントローラ、単一の3D XPointメモリパッケージ、およびほとんどの電源管理コンポーネントが含まれています。 Intel SSD 660p以前のOptane Memoryソリッドステートドライブは、プリント回路基板上に多くの空き容量がありました。Optane Memory H10はフルで、おそらく市場に出回っているM.2 SSDの中でほとんどの部分を占めています。



一見すると、Optane Memoryソフトウェアはほとんど変更されていません。キャッシュするデバイスを選択する際の柔軟性が高まった場合を除きます。 （Intelは、Coreプロセッサで既にサポートされているプラ​​ットフォームでのPentiumおよびCeleronプロセッサの拡張Optaneメモリサポートを発表しました。）ブートボリュームがキャッシュされている場合、Intelソフトウェアでは、選択したファイルとアプリケーションをキャッシュして、それらがキャッシュから削除されないようにすることができます。このオプション以外には、キャッシュ動作の設定はありません。

Optane Memoryを搭載した一部のOEM販売システムは、メモリサイズをDRAMとOptaneの合計として報告しました。プロセッサのメモリコントローラーに接続されたOptane DC読み取り専用メモリモジュールについて話すとき、それは理にかなっています。誤解を招くのは、問題のOptane製品がSSDであるという事実です。

当初、Optane Memory H10はOEMのみの部品になります。新しいSSDは、主にラップトップなどの新しいシステムのコンポーネントとしてのみ利用できます。インテルは、スタンドアロン製品として、およびNUCキットの一部として、小売でH10をリリースする可能性をまだ検討しています。タイミングは発表されていません。彼らのマザーボードパートナーは、ほぼ1年間H10サポートの舞台を設定しており、多くのデスクトップ300シリーズマザーボードは、最新の公開ファームウェアでH10をすでにサポートしています。

プラットフォームの互換性


控えめに言っても、2つのPCIeデバイスを同じM.2カードに配置するのは新しいことです。 Intelはすでに1つの回路基板に2つのSSDコントローラーをインストールしています-企業のハイエンドSSD（P3608やP4608など）ですが、これらのドライブはPCIeスイッチチップを使用して、ボード上の2つのNVMeコントローラーそれぞれのx8ホスト接続を2つのx4ホストに分割します。このアプローチでは、デバイスのTDPが40ワットに膨らみますが、これはM.2の制限内ではまったく役に立ちません。

1つのPCIe x16スロットを介して4つのM.2 PCIe SSDを接続できるいくつかのPCIe拡張カードがあります。これらのカードの一部にはPCIeスイッチが含まれていますが、ほとんどの場合、デュアルPCIeポートをサポートするホストシステムを使用して、1つのx16ポートを4つの独立したx4ポートに分割します。従来のコンシューマCPUは通常これをサポートせず、分割がx8 + x4 + x4またはx8 + x8に限定されており、複数のGPUの使用をサポートするためにラインが異なるスロットにリダイレクトされる場合に限られます。サーバーおよびワークステーション用の最新のプロセッサーは、最大x4までのポート分離をサポートする可能性が高くなりますが、この機能を有効にするマザーボードのサポートがどこにでもあるわけではありません。

x16スロットを4つのx4ポートに分割できるプロセッサーでも、x2ポートまでさらに分離することはまれであるか、まったく不可能です。狭いx2またはx1ポートなど、多くのPCIeレーンをサポートするチップは、ほとんどのマザーボードのサウスブリッジ/ PCHチップです。原則として、x4より広いポートはサポートされていません。これは、これがプロセッサへの接続の通常の幅であるためです。

2つの異なるブランドのPCIe 3.0スイッチを介してH10を接続しました。 Avago PLX PEX8747は、x4ポートまでのデュアルPCIeポートしかサポートしていないため、予想されていたNAND側へのアクセスのみを提供していました。 Microsemi PFX PM8533スイッチはx2までのスプリットをサポートしており、H10の両側にアクセスできることを望んでいましたが、代わりにOptaneの半分にしかアクセスできませんでした。 MicrosemiスイッチとThreadripperマザーボードは、H10の両方の半分で機能するためにファームウェアの更新のみが必要な場合があります。以前の世代のIntel PCHにはこの可能性がありますが、Intelはそのようなアップデートを提供しません。これらのプラットフォームがH10の両方の半分にアクセスできたとしても、それらはIntelのOptane Memoryキャッシングドライバーではサポートされません。真実、サードパーティのキャッシュソフトウェアがあります。

試験システム


消費者向けSSDをテストするための主なシステムは、Skylakeデスクトップコンピューターです。詳細な電力測定用のQuarch XLCプログラマブルパワーモジュールを備えており、IO ATSBトレーステストおよびFIOを使用した模擬テストに使用されます。しかし、私たちのシステムはOptane Memoryよりも古く、Intelとそのマザーボードパートナーは、Skylake世代のシステムでOptane Memoryキャッシュをサポートするファームウェアのアップデートをリリースしたくありませんでした。その結果、テストベンチの助けを借りて、ディスクの半分、つまりQLC NANDにしかアクセスできません。



新しいOptaneメモリのリリースと同様に、Intelは新しいOptaneメモリH10がすでにインストールおよび構成されたシステム全体を送ってくれました。したがって、テストするシステムは、Intel Core i7-8565U Whisky Lakeプロセッサと16 GBのDDR4メモリを搭載したHP Spectre x360 13tラップトップです。インテルは以前、Optane Memory製品をテストするためにデスクトップシステムを提供していましたが、H10の主な利点は、小型システムに適したM.2モジュールが1つであるため、13インチのラップトップが選択されたことです。 IntelはSpectre x360がストレージオプションの1つとしてOptane Memory H10とともにまもなく販売されることを確認しました。

HP Spectre x360 13tにはM.2タイプMスロットが1つしかないため、SATAを使用して複数のディスクまたはオプションのキャッシュ構成をテストするために、インテルが以前のリリースのOptaneメモリに提供したCoffee LakeおよびKaby Lakeシステムを使用しました。 SYSmarkやPCMarkなどのアプリケーションテスト結果は、これらのマシン間のプロセッサパワーとRAMの違いに大きく影響されるため、テストしたディスク構成ごとに3セットの評価をリストする必要があります。それにもかかわらず、AnandTech Storage Bench I / OテストとFIOを使用した合成テストでは、これら3つのシステムでほぼ同じ結果が得られるため、直接比較できます。そのようなテストごとに、ストレージ構成ごとに1セットのポイントをリストします。



インテルOptaneメモリキャッシングソフトウェアはWindows専用であるため、Linuxベースの合成FIOテストをWindowsで実行するように調整する必要がありました。構成とテストの手順は通常の方法論に可能な限り近いですが、いくつかの重要な違いは、このレビューの結果を通常のSSDレビューの結果およびBenchで公開された結果と直接比較できないことを意味します。特に、場合によっては、WindowsからNVMeを安全に消去またはフォーマットすることができませんでした。私たちのテストでは通常、SSDのバックグラウンドガベージコレクションがSLCキャッ​​シュのクリーニングとクリアを完了するのを待たずに、メインフェーズ間でドライブを消去してパフォーマンスを復元します。 Windowsテストに基づく模擬テストのレビューでは、記録されるデータ量が最も少なく、最初に実行され、ディスク全体を埋める必要があるテストが最後に機能しました。

Optaneメモリキャッシングには、Intelドライバーの使用が必要です。 Windowsベースのテストの通常の手順は、Microsoft独自のNVMeドライバーを使用することですが、ベンダー固有のドライバーは使用しません。このレビューのOptaneキャッシング構成テストはIntelドライバーを使用して行われましたが、単一のドライブを使用するすべてのテスト（Optane Memory H10の片側のみのテストを含む）は、デフォルトのWindowsドライバーを使用します。

当社の標準Skylakeテストベンチは、プロセッサーに接続されたメインPCIe x16スロットでNVMe SSDをテストするように設計されています。Optaneメモリキャッシングでは、チップセットを介してドライブを接続する必要があるため、x4 DMIチャネルの過負荷が最速のドライブに影響を与える可能性はほとんどありませんが、H10がこの接続を飽和させることはほとんどありません。

ほぼすべてのパフォーマンステストに詳細な電力測定を含めようとしていますが、このレビューでは、それらの多くをスキップすることを余儀なくされています。当社の最先端の電力測定機器は、ラップトップのM.2スロットに電力を供給することができず、通常のPCIe x4スロットが必要です。

アプリケーションテスト


Intel Optane Memory H10のような高度な階層型ストレージシステムでは、最も正確なベンチマークは、実際のアプリケーションを使用するテストになります。 BAPCoのSYSmark 2018とULのPCMark 10は、自動化されたアプリケーションテストの2つの競合するセットです。どちらにも共通の目標があります-システムの全体的なパフォーマンスの評価と、さまざまな一般的な使用例をカバーするいくつかの追加の評価を提供することです。 PCMark 10はより短いテストであり、さまざまなパラメーターのより詳細な内訳を提供します。また、3Dレンダリングは標準テストスイートに含まれており、一部の3DMarkテストは高度なテストに含まれているため、GPUの負荷がはるかに高くなります。 SYSmark 2018の利点は、人気のあるアプリケーションの完全な商用バージョンを使用できることです。Microsoft OfficeおよびAdobe Creative Suiteを含み、テスト中にシステムの全体的な電力消費を測定する機能が含まれています。これらのテストの欠点は、日常的な使用の最も一般的なケースのみをカバーし、重いマルチタスク負荷をシミュレートしないことです。それらのサブテストはどれもストレージを集中的に使用しないため、高速SSDと低速SSDを比較した場合、スコアがわずかに異なることがよくあります。したがって、高速SSDと低速SSDを比較すると、ポイントがわずかに変化することがよくあります。したがって、高速SSDと低速SSDを比較すると、ポイントがわずかに変化することがよくあります。

消費電力
