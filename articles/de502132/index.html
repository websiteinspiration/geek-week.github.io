So übertragen Sie den OpenVZ 6-Container ohne Kopfschmerzen auf den KVM-Server

Jeder, der mindestens einmal in seinem Leben einen OpenVZ-Container auf einen Server mit vollständiger KVM-Virtualisierung übertragen musste, hatte einige Probleme:


  • Die meisten Informationen sind banal veraltet und waren für den längst vergangenen EOL-Betriebssystemzyklus relevant
  • Für verschiedene Betriebssysteme werden immer unterschiedliche Informationen bereitgestellt, und mögliche Fehler werden bei der Migration niemals berücksichtigt.
  • Manchmal müssen Sie sich mit Konfigurationen befassen, die nach der Migration nicht funktionieren möchten

Wenn Sie 1 Server übertragen, können Sie unterwegs immer etwas reparieren, und wenn Sie einen gesamten Cluster übertragen?


In diesem Artikel werde ich versuchen zu erklären, wie OpenVZ-Container mit minimaler Ausfallzeit und schneller Lösung für alle Probleme korrekt auf KVM migriert werden können.


Ein kleines Bildungsprogramm: Was ist OpenVZ und was ist KVM?


Wir werden uns nicht mit der Terminologie befassen, sondern allgemein:


OpenVZ ist eine Virtualisierung auf Betriebssystemebene. Sie können sie sogar in der Mikrowelle bereitstellen, da auf dem Host-Computer keine CPU-Anweisungen und Virtualisierungstechnologien erforderlich sind.


KVM ist eine vollwertige Virtualisierung, die die gesamte Leistung der CPU nutzt und in der Lage ist, alles zu virtualisieren, unabhängig von Länge und Breite.
