nRF52832の容量性土壌水分センサー用ワイヤレスモジュール

みなさん、こんにちは。今日は、土壌水分センサーをAliexpressでアップグレードすることに決めた方法についてお話します。約1ヶ月前、土壌水分センサーを購入しました。なぜ私がそれを購入したのか、そしておそらく40ルーブルの価格のせいで、自分自身を知らないのでしょう:

)（Arduino Nanoの助けを借りて）センサーを受け取ってテストに成功したので、すでに動作しているMaysensorsベースのシステムのどこに取り付けるかを考え始めました（これについては後で説明します）。センサーは非常に安いので、同様に安くて気取らない解決策を見つけたいと思っています。



センサー回路はTLC555タイマーチップ上に構築されています。電圧レギュレータXC6206P332（データシート）がそれぞれ3.3vで回路に追加され、回路は最大6vの電源から電力を供給できます。電源電圧が3.3Vより低い場合、スタビライザーは出力も提供し、入力で受け取ります。

2ヶ月という早い時期に、EBYTE社のnRF52832の2つのモジュール（E73-2G​​4M04S1B）がアイドル状態でした。非常に安価なモジュール、価格の問題で、他のすべてのnRF52モジュールははるかに遅れています。



しかし、私には2つの大きな欠点があります。最初に、そしてそれほど重要ではないのは、モジュールのサイズです。彼らはかなり大きいです。 2番目のマイナスは、モジュールが魅力の半分を失うために、回路内に2つの小さな要素がないことです。欠落している要素は、DCCおよびDEC4レッグに接続された2つのインダクターです。これは、低電力モード（7-8mA VS 15-16mA）でのモジュールの使用を許可しないため、良くありません。彼らがそれらを配置し始めなかった理由、私は理解できません、スキームが他の要素を節約できるので、「節約のため」のオプションは適合しません。一般的に、インダクタンスのインストールとDC-DCモードの存在をウィッシュリストに追加します。

次に解決すべきことは、センサーの電力管理です。私たちのテーマはバッテリーのテーマなので、一定の電力は悪いオプションです。それ自体がすぐに示唆する最も簡単なことは、キーモードでのトランジスタの使用です。選択は、フィールドpチャネルトランジスタIRLML6402TRPBFに依存しました。

次に考えなければならないのはプログラミングポートです。SWDとシリアルで、パッドを作成しました。もちろん、他の2x3Pデバイスで使用するマイクロコネクタも追加しました。 6ピン| 1.27mm | SMT |ピンヘッダーメスですが、これは純粋にオプションです。

また、クロックボタンと少なくとも1つのLEDを追加して、機能するかどうかを理解しやすくする必要もあります。

次に解決すべきことは、無線モジュールノードと静電容量センサーを接続する方法でした。センサーに取り付けられているソケットとそれに付属しているワイヤーをまったく使用したくありませんでした。ソケットがはんだ付けされているボード上のコネクタの穴ピッチは2.54 mmで、追加の複製列もボードに表示されます。通常の「くし」を2.54の増分で使用することが決定されました。一度に両方の列を使用すると、接続に追加の剛性が与えられます。

それはすべてのようです、まんじゅうから、雨の日のために残したりはんだ付けしたりできるいくつかの要素とワイヤー付きのソケットがあります（それはどこかで役立つでしょう:)）。

ボードは、いつものように、Diptraceプログラムで作成されました。最初のオプションは、実際には、この記事のスピーチにすぎないことが判明したLUTに対して作成されました。その後、工場での注文のためにボードのバリアントが作成されました。



酸洗い、錫メッキ、切断、穴あけ、はんだ付けの後、テストの時間です。一般に、EBYTEのモジュールのセンサーには特に何も期待していませんでした。特に、Aliを備えたある種の外部水分計ではそうでした。しかし、結局私はいくつかの結果に驚きさえしました。データ転送モードでの消費は9 mA（半放電バッテリーの場合）以下であり、測定モードでの消費は5 mA以下でした。スリープモードでの消費量は2.1〜2.2μAでした!!!



センサーが現在できる合計。低電力モードで動作します。 Maysensorsネットワークを介してUDコントローラーを測定し、土壌水分、温度、バッテリー残量、および信号強度の読み取り値を送信します。

void int_temp() {
  for (byte i = 0; i < 10; i++) {
    NRF_TEMP->TASKS_START = 1;
    while (!(NRF_TEMP->EVENTS_DATARDY)) {}
    rawTemperature = NRF_TEMP->TEMP;
    rawTemperature2 = rawTemperature2 + rawTemperature;
    wait(10);
  }
  celsius = ((((float)rawTemperature2) / 10) / 4.0);
  rawTemperature2 = 0;

}

void msm () {
  digitalWrite(6, LOW);
  wait(500);
  for (byte i = 0; i < 10; i++) {
    m_s_m = analogRead(5);
    m_s_m2 = m_s_m2 + m_s_m;
    wait(50);
  }
  m_s_m = m_s_m2 / 10;
  m_s_m2 = 0;
  digitalWrite(6, HIGH);
  wait(50);
  if(m_s_m >3000){
    m_s_m = 3000;
  }
  if(m_s_m <1100){
    m_s_m = 1100;
  }
  m_s_m_calc = map(m_s_m, 3000, 1100, 0, 100);
  flagSendmsm = 1;
}

void sendBatteryStatus() {
  wait(100);
  batteryVoltage = hwCPUVoltage();
  wait(20);

  if (batteryVoltage > battery_vcc_max) {
    currentBatteryPercent = 100;
  }
  else if (batteryVoltage < battery_vcc_min) {
    currentBatteryPercent = 0;
  } else {
    currentBatteryPercent = (100 * (batteryVoltage - battery_vcc_min)) / (battery_vcc_max - battery_vcc_min);
  }
  sendBatteryLevel(currentBatteryPercent, 1);
  wait(3000, C_INTERNAL, I_BATTERY_LEVEL);

  linkQuality = calculationRxQuality();
  wait(50);
  sendSignalStrength(linkQuality, 1);
  wait(2000, 1, V_VAR1);
}

bool sendSignalStrength(const int16_t level, const bool ack)
{
  return _sendRoute(build(_msgTmp, GATEWAY_ADDRESS, SIGNAL_Q_ID, C_SET, V_VAR1,
                          ack).set(level));
}
int16_t calculationRxQuality() {
  int16_t nRFRSSI_temp = transportGetReceivingRSSI();
  int16_t nRFRSSI = map(nRFRSSI_temp, -85, -40, 0, 100);
  if (nRFRSSI < 0) {
    nRFRSSI = 0;
  }
  if (nRFRSSI > 100) {
    nRFRSSI = 100;
  }
  return nRFRSSI;
}
