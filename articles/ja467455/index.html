Milanderコントローラーの例を使用してQtCreatorでマイクロコントローラーをプログラミングするためのQBSテンプレート

少し前まで、彼は1つのあまり知られていない会社-ミランダーのマイクロコントローラーの開発を手伝いました。公式のKeil開発環境。私は好きではありませんでしたが、プロセッサはもう1つのあまり知られていない会社ARMのアーキテクチャに基づいています。また、gccがある場合、QtCreatorをIDEとして使用することもできます。



前書き


最初は、タスクは非常に単純でした。プロジェクトのビルドに必要なファイルを見つけます。stm32f407で既に行われたプロジェクトを取り、ファイルで処理します。しかし、そのようなシンプルな計画の実行中に、当初考えられていたよりも少し興味深いプロジェクトを作りたいという要望がありました。

トレーニング


まず、Cortex-Mシリーズコアに基づくマイクロコントローラー用のコンパイラーとデバッガーをダウンロードする必要があります。最新バージョンはこちら[1]です。QtCreatorも必要です。レイジーでないものはソースからコンパイルでき、レイジーなものはここから取得できます[2]。また、マイクロコントローラーのフラッシュとデバッグにはopenocdも必要です。[3] ARMサイトのCortex-M用GCCコンパイラ[9]

指定されたソフトウェアと、必要に応じてすべての依存関係をインストールします。次に、環境のセットアップに進みます。

マイクロコントローラライブラリ


ミランダーのサイトをざっと検索しても、周辺機器とマイクロコントローラーのコアを操作するためのライブラリの公式アセンブリは作成されませんでした。アセンブリはgccをサポートし、github [4]にあります。アセンブリはstm32f407の完成したプロジェクトに非常に簡単に行き、いくつかの場所でそれを修正した後、マイクロコントローラーをフラッシュすることはすでに喜んで熱心でした。 openocd構成ファイルを編集すると、すべてが点滅します。もう少し錠前屋の仕事とデバッグがすでに動いています。そしてそれはそれだけです-簡単な勝利です。しかし、ライブラリはいくつかのマイクロコントローラをサポートしています。また、異なるコアのマイクロコントローラーも、異なるペリフェラルライブラリを使用します。すぐに1つのプロジェクトを作成して、目的のマイクロコントローラーを簡単に選択でき、環境自体が必要なコアライブラリと周辺機器を選択するようにするというアイデアがありました。このアイデアは、私がstm32f407を習得したときに思いつきました。そのようなかなり興味深いアイデアを実現する機会を逃すことはできませんでした。

図書館とQBSで数泊する[5]肯定的な結果が得られました。MDR1901VC1T（19011）、MDR1986VE1T（1986 BE1T）、MDR1986VE3（1986 BE3）、MDR1986BE4（1986 BE4x）、MDR1986VE8T（1986 BE8x）、MDR1986VE9x（1986 BE）のマイクロコントローラー用にコンパイルを構成できました。ソースライブラリの説明はMDR1986VE8Tをサポートしていませんが、コアライブラリとペリフェラルライブラリファイルの一部がありました。ARMリポジトリで見つかったヘッダーファイルの一部がありませんでした[6]。

最終プロジェクト


その結果、いくつかのマイクロコントローラー用のプロジェクトを作成できる、かなり控えめなテンプレートができました。github [7]にあるテンプレート。ソースライブラリは、いくつかの表面的な変更を経ています。元のプロジェクトとの互換性を維持したかったのですが、[8]をフォークして互換性を失う必要がありましたが、これは汎用性の代償です。次のようにプロジェクトをダウンロードできます。

git clone https://github.com/WicRus/emdr1986x-std-per-lib-qbs.git --recursive

リポジトリのアドレスを指定して再帰的にチェックすることで、QtCreatorからプロジェクトを直接複製することもできます。

育種


主な結果は、さまざまなコアおよび周辺機器ライブラリを備えたいくつかのマイクロコントローラーをサポートするユニバーサルテンプレートです。ただし、注意すべき点がいくつかありました。

  • MDR1986VE1Tでのみファームウェアとデバッグを確認できました。
  • ターゲットのopenocd構成ファイルは、MDR1986VE1TおよびMDR1986VE9xでのみ見つかりました。残りのコントローラーについては、自分でどこかに持って行くか書き込む必要があります。
  • MDR1986VE8Tのライブラリは、問題をあまり理解せずにコンパイルされました。正しい動作を確認する必要があります。
  • ソースライブラリが大幅に変更されました。可能なマージの問題。
  • MDR1901VC1T、MDR1986BE4、MDR1986VE9x用にビルドすると、古いカーネルライブラリに関する警告が表示されます。MDR1986VE8Tと同様にカーネルライブラリを更新し、正しい動作を確認する必要があります。
  • openocd構成を作成する機能。マイクロコントローラーの個々のタイプごとに、QtCreatorに個別のビルドキットが必要になる場合があります。
