最新のWebアプリケーションを作成します。プロジェクトの知識と仕事の準備。パート1


このシリーズの記事では、アプリケーションのクライアント部分を作成する全サイクルを経て、最新のテクノロジースタックを使用してコンポーネントの小さなライブラリを作成します。
この記事は、最初のJavaScriptプロジェクトを作成して全世界に公開したいフロントエンド開発者向けに作成したもので、このシリーズの記事では、最新のほとんどのプロジェクトにある基本的なスタックを選択しました。飽きないように、いつでも独自のものを追加できるので、独自の実装を作成し、記事を読みながらGitHubで作業結果を公開することをお勧めします。たぶん、あなたが試したいテクノロジー、ライブラリ、フレームワーク、ツールがたくさんあります。そのようなペットプロジェクトの開発は、何か新しいことを試す素晴らしいオプションです。

プロジェクトの知識


実装するプロジェクトの主なアイデアは、TypeScriptを使用してReactコンポーネントライブラリを記述し、それをStorybookで文書化して視覚化し、npmでパッケージとして公開することです。また、リンターを構成し、Jestのテストを追加し、Travis CIを使用してテストプロセスを自動化します。おそらく、作業中に何か他のものが追加されます。コメントしてソリューションを提案することをためらわないでください。

記事はいくつかのパートに分かれているので、プロジェクトの各段階を詳しく調べることができます。

仕事の始まり


まず、プロジェクトのGitHubにリポジトリを作成する必要があります。



これが、新しいリポジトリを作成するためのウィンドウの外観です。リポジトリの名前と簡単な説明を考え出す必要があります。私のすべてのペットプロジェクトでは、常にパブリックタイプのリポジトリを選択しますが、チームが3人以下の場合、GitHubは無料でプライベートリポジトリを作成する機会を提供します。私はすぐにMITライセンスも追加しました。これはオープンソースプロジェクトのライセンスの最も単純で最も一般的なバージョンです。ライセンスについて詳しく知りたい場合は、GitHubによって作成されたこのサイトをご覧ください。

新しいリポジトリを複製します。 GitHubでは、SSHまたはHTTPSを使用したクローン作成を推奨しています。私は通常、2番目の方法を使用します。

git clone https://github.com/Alexandrshy/react-ant.git

正常な解凍に関するメッセージが表示されれば、複製は成功しています。

また、すぐにパスワードをキャッシュする必要があります。次回git push、git fetch、git cloneを実行するときにこれを行わない場合は、ユーザー名とパスワードを入力する必要があります（詳細はこちら）。

git config --global credential.helper osxkeychain

package.jsonファイルの作成に移りましょう。これを行うには、次のコマンドを実行します。

npm init -y

リポジトリでコマンドを実行すると、package.jsonファイルにいくつかのフィールドが入力されていることがわかります。私の場合は次のようになります。

{
  "name": "react-ant",
  "version": "1.0.0",
  "description": "A set of light React components ",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "repository": {
    "type": "git",
    "url": "git+https://github.com/Alexandrshy/react-ant.git"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "bugs": {
    "url": "https://github.com/Alexandrshy/react-ant/issues"
  },
  "homepage": "https://github.com/Alexandrshy/react-ant#readme"
}

小さな変更をすぐに加えましょう：

  "keywords": [
    "react",
    "css",
    "UI"
  ],
  "author": {
    "name": "Alex",
    "email": "alexandrshy@gmail.com",
    "url": "https://github.com/Alexandrshy"
  },
  "license": "MIT",


ここではすべてがはっきりしていると思いますが、より詳細な構成については、このドキュメントが必要になる場合があります。

プロジェクトで作業するときに、package.json構成に戻ります。しかし、今こそ最初のコミットをする時です。


git status
git add package.json
git commit -m "Init npm package"
git push

簡単に言えば、私たちは何をしましたか：変更履歴を確認し、変更されたpackage.jsonにインデックスを付け、シンプルで明確なコミットメッセージでコミットし、変更をリモートリポジトリに入れました。これで、package.jsonと新しいコミットに関する情報がリポジトリに表示されました。IDEまたはGUIを使用してGitを操作できますが、コンソールですべてを行う方がより快適です。

リンター


コードをよりクリーンに保つには（複数の人がプロジェクトで作業している場合、これは特に重要です）、間違いなくエラーを分析して特定するツールが必要です。私のプロジェクトでは、ESLintを使用してJavaScriptコードをテストしています。インストールは簡単で、高度にカスタマイズ可能です。

ESLintをインストールします。

npm i -D eslint

構成ファイルを設定します。

./node_modules/.bin/eslint --init

ESLintを手動で構成するか、定義済みのルールセットを使用できます。Airbnbからのガイドが好きです。次の設定を使用しました。


? How would you like to use ESLint? To check syntax, find problems, and enforce code style
? What type of modules does your project use? JavaScript modules (import/export)
? Which framework does your project use? React
? Does your project use TypeScript? Yes
? Where does your code run? Browser
? How would you like to define a style for your project? Use a popular style guide
? Which style guide do you want to follow? Airbnb (https://github.com/airbnb/javascript)
? What format do you want your config file to be in? JavaScript

TypeScriptを使用する予定なので、ダイアログボックスですぐにこの項目を選択しました。そのため、「Typescript」モジュールが見つからないというエラーが発生しました。これは、TypeScriptをまだインストールしていないため論理的です。これをすぐに修正しましょう。


npm i typescript
npm i -D @typescript-eslint/parser

インストール後、eslintrc構成ファイルが表示されます。これはすでに構成されていますが、開発中にいくつかのルールを追加または変更したい場合は、それが役立ちます。

ESLintをテストするには、index.tsファイルを作成して、次のコードをそこに保存します。

var a
console
    .log('a = ' + a)
 
    a = 100

そしてチェックを実行します：


./node_modules/.bin/eslint index.ts

5行のコードには7つのエラーと1つの警告があります。そしてすぐにESLintはこれらのエラーを自動的に修正するように私に提案します、これをやってみましょう：


./node_modules/.bin/eslint index.ts --fix

そして、エラーがなく、console.logの使用に関する警告が1つあるこのコードを取得します。

var a;
console
  .log(`a = ${a}`);
 
a = 100;

自動修正が機能することを確認できるように、エラーは修正されていますが、コードは依然として見苦しく見えます。コードをフォーマットするために、私の意見で最高のツールはPrettierです。それをプロジェクトに追加しましょう：


npm i -D prettier-eslint
npm i -D prettier-eslint-cli


"scripts": {
    "format": "prettier-eslint '**/*.{js,ts,tsx}' --write"
},

すべてのフォーマット済みファイルが上書きされるように--writeオプションを追加しました。結果を確認します。


npm run format

index.ts 

let a;
console.log(`a = ${a}`);
 
a = 100;

すべてがうまくいきます。IDEのプラグインをインストールして、キーボードショートカットを使用するか、変更を保存するときにファイルをフォーマットすることもできます。次に、スクリプトをpackage.jsonに追加します。

"scripts": {
    "eslint": "./node_modules/.bin/eslint '**/*.{js,ts,tsx}'",
    "eslint:fix": "npm run eslint -- --fix",
    "format": "prettier-eslint '**/*.{js,ts,tsx}' --write"
},

新しいプロジェクトを開始するときの理想的なオプションは、すべてのリンターをすぐに構成することです。すでに完了したプロジェクトにそれらを実装しようとすると、多数のエラーが発生する可能性があり、最初にコードの品質に注意を払った場合よりも修正に時間がかかります。

変更内容を保存：


git add .
git commit -m "Added ESLint and Prettier"
git push

事前コミットフック


ESLintとPrettierを構成し、スクリプトを手動で実行するように作成しましたが、コミットする前にこれを自動的に行うと便利です。これには、Gitフックを使用できます。Huskyパッケージを使用すると、 `git commit`を実行する前にスクリプトを実行できます。Lint - stagedパッケージを使用すると、特定のフィルターに対してインデックス付きファイルのみをチェックできます。

npm i -D husky lint-staged

package.jsonに戻り、次のコードを追加しましょう。

  "husky": {
    "hooks": {
      "pre-commit": "lint-staged"
    }
  },
  "lint-staged": {
    "*.(js|jsx|ts|tsx)": [
      "npm run eslint:fix",
      "npm run format",
      "git add"
    ]
  },

ここで、各コミットの前に、変更されたすべてのjsおよびtsファイルに対してESLintおよびPrettierチェックを実行し、フォーマット後、これらのファイルをコミットに追加します。

index.tsファイルをもう一度テストしてみましょう。

var a;
console
  .log(`a = ${a}`);
 
a = 100;

これだけがすぐにこのファイルにインデックスを付けてコミットします：


git add index.ts
git commit -m "Testing pre-commit hook"

ここでindex.tsファイルを確認すると、コードがフォーマットされていることがわかります。変更を保存する前に、ファイルはチェックされ、必要に応じてフォーマットされます。これにより、リポジトリに含まれるファイルが正しいことを確認できます。

##結論

すべての変更を保存する必要があります。その前に、作成.gitignoreファイルに書くnode_modulesを、我々は我々のリポジトリに取得するには、依存関係のフォルダを必要としません。 index.tsファイルを削除することもできます。将来的には不要になります。


git add .
git commit -m "Added husky and lint-staged"
git push

リポジトリで 作業の結果全体を見ることができます。プロジェクトの現在のステータスはマスターブランチであり、記事ごとに個別のブランチを作成します。

私たちはこれで止まります、あなたの注意をありがとう、次の部分でお会いしましょう。
