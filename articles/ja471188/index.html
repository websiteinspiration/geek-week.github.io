Visual Studio CodeのPython-10月の更新

Visual Studio Codeの10月Python拡張機能拡張が利用可能になったことをお知らせします。Python拡張機能は、Marketplaceからダウンロードするか、Visual Studio Codeの拡張機能ギャラリーから直接インストールできます。Python拡張機能がすでにインストールされている場合は、Visual Studioコードを再起動して最新のアップデートを入手することもできます。ドキュメントから、Visual Studio CodeでのPythonサポートについて詳しく知ることができます。

このリリースでは、Jupyter Notebooksのネイティブ編集、ターミナルでPythonファイルを起動するボタン、Python言語サーバーを使用したリントとインポートの改善など、97の問題を確認しました。改善の完全なリストは、変更ログに記載されています。



Jupyterノートブックのネイティブ編集 


VS CodeでのネイティブJupyterノートブック編集の最初のリリースを発表できることを嬉しく思います！Jupyterのネイティブエクスペリエンスは、データサイエンティストと開発者に.ipynbファイルを直接編集し、VS Codeのすべての機能を備えたJupyterノートブックのインタラクティブ性を実現する新しい方法を提供します。Jupyter Notebookのネイティブ編集の詳細については、VS Codeブログをご覧ください。



ボタン「ターミナルでPythonファイルを実行」


このリリースには、[ターミナルでPythonファイルを実行]コマンドを実行するための[再生]ボタンが含まれています。 Python拡張子が付いたPythonファイルを実行するには、ワンクリックで完了します。

Python言語サーバーを使用したリントとインポートの改善 


このリリースには、PyTorchやpandasなどのパッケージのオートコンプリートの大幅な改善に加えて、Python言語サーバーでの3つの新しいリンティングルールも含まれています。

さらに、インポートの解像度が大幅に改善されました。歴史的に、Language Serverはワークスペースのルートをユーザーモジュールのインポートのsys.path（つまり、メインワークスペースのルート）エントリと見なしていました。そのため、srcディレクトリからモジュールをインポートするときに、不正なインポートに関する誤った警告が出されました。このリリースでは、プロジェクト環境にそのようなsrcディレクトリがある場合、言語サーバーは自動的にそのディレクトリを検出して、検索パスのリストに追加します。ドキュメントを参照できます言語サーバーの検索パスの設定について詳しくは、をご覧ください。

その他の変更と改善。


また、ユーザーから要求されたマイナーな改善とバグ修正を追加しました。これにより、Visual Studio CodeでのPythonのエクスペリエンスが向上します。主な変更点は次のとおりです。

  • pytest 5.1以降でのテスト検出の問題を修正しました。（＃6990） 
  • シェル検出の修正。（＃6928） 
  • 言語サーバーの内部者にユーザーを自動的に追加します。（＃7108）
