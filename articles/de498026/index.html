Der Rest wird in einer kurzen Zeilenliste für einen Satz zu den Nachrichten gehen. In einem separaten Block geht es um Releases. Wir freuen uns über Feedback zum neuen Format in den Kommentaren oder privaten Nachrichten.

Git 15-




Die erste Veröffentlichung des Git-Versionskontrollsystems erfolgte am 7. April 2005 - vor 15 Jahren. Git begann als harte Währung für den Linux-Kernel, da die zuvor in BitKeeper verwendete Lizenz geändert wurde. Inzwischen ist Git seiner ursprünglichen Rolle der harten Währung nur für den Kern deutlich entwachsen und wurde zur Grundlage für die Entwicklung fast jeder freien, offenen und sogar proprietären Software auf der ganzen Welt.

„ Seit seiner Einführung im Jahr 2005 hat sich Git zu einem benutzerfreundlichen System entwickelt, wobei die ursprünglichen Eigenschaften beibehalten wurden. Er ist überraschend schnell, arbeitet effizient mit großen Projekten und verfügt über ein hervorragendes Verzweigungssystem für die nichtlineare Entwicklung “, schreiben Scott Chacon und Ben Straub in ihrem Buch„ Git for a Professional Programmer “.

LLHD-Projekt - Universal Hardware Description Language




Auf Habré wird ein interessanter Artikel über die offene universelle Sprache zur Beschreibung von Hardware vorgestellt. Die Autoren zeigten, dass die für Compiler von Programmiersprachen traditionellen Techniken recht erfolgreich auf die Sprachen des "Eisens" angewendet werden können. "Es wurde eine neue Hardwarebeschreibungssprache für Fortgeschrittene entwickelt, Prototypen des Übersetzers von SystemVerilog, einem Referenzinterpreter und LLHD-JIT-Simulator, die eine gute Leistung zeigten ", schreibt der Artikel.

Open Source hat sich als führender Weg der Softwareentwicklung etabliert.




Ungefähr 80% des IT-Stacks in Unternehmen auf der ganzen Welt besteht aus Open Source-Software. JaxEnter hat ein großartiges Interview mit dem Red Hat-Entwickler Jan Wildeboer zu diesem Thema. Es werden Antworten gegeben, was Open Source für Jan persönlich ist, wie der Stand von Open Source heute ist, wie seine Zukunft aussieht, welche ethischen Grundsätze für die Verwendung gelten, welche Unterschiede zwischen freier und Open Source-Software bestehen, wie sich die Verwendung von Open Source auf die internen Prozesse von Red Hat und andere Probleme auswirkt .

Interview mit Alexander Makarov über Open Source, Konferenzen und Yii




Auf Habré gab es ein großes Interview mit dem Entwickler des PHP Yii Frameworks Alexander Makarov. Es wurden verschiedene Themen diskutiert - IT-Konferenzen in Russland, Remote-Arbeit und Arbeit im Ausland, Alexanders persönliches Offline-Geschäft und natürlich das Yii-Framework selbst.
