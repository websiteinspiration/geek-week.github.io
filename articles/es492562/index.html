18 de marzo: cómo entender qué Ignite SQL es adecuado para usted y comenzar a usarlo


Si se enfrenta a restricciones en el tamaño, la escalabilidad o el rendimiento de su base de datos relacional, puede ser hora de actualizar a un sistema distribuido. Apache Ignite es una plataforma distribuida que puede funcionar como una base de datos, proporcionando API de SQL y JCache para trabajar con datos. En este seminario web veremos ejemplos de la vida real y discutiremos los pros y los contras de cada enfoque:

  • Características SQL en Ignite
  • Cómo interactúa SQL con la API de Java (carga a través de la API de JCache a través de SQL y viceversa)
  • Garantías de consistencia (Ignite SQL y JCache API)
  • Las complejidades del SQL distribuido y cómo la colocación de datos puede minimizarlo.

25 de marzo - Computación distribuida con Apache Ignite


Para aprovechar todo el potencial de la plataforma In-Memory, a menudo no es suficiente cargar datos en el clúster y comenzar a trabajar con ellos utilizando el valor clave o la API SQL. También debe colocar los datos de manera eficiente en un clúster y usar computación distribuida que minimice el movimiento de datos a través de la red. En este seminario web, aprenderá a diseñar y realizar cálculos teniendo en cuenta todas las ventajas y desventajas de los sistemas distribuidos.

Consideraremos:

  • Cómo distribuir eficientemente la información en un clúster utilizando el modelado y la colocación adecuados de datos;
  • Cómo se implementa y funciona MapReduce API en Apache Ignite;
  • Cómo minimizar los retrasos en la operación utilizando la colocación de datos con cálculos;
  • Cómo las consultas SQL se pueden ejecutar de manera eficiente en un entorno distribuido.

El seminario web será útil para aquellos que comienzan a familiarizarse con Apache Ignite y desean aprender cómo diseñar y utilizar de manera óptima las capacidades de la informática distribuida en esta plataforma.

14 de abril: Guía de configuración de seguridad para Apache Ignite y GridGain con ejemplos


Apache Ignite y GridGain permiten una informática rápida y consultas de datos distribuidos altamente eficientes. Tanto Ignite como GridGain proporcionan una configuración flexible que ayudará a que las operaciones de clúster sean más seguras. En este seminario web, cubriremos los siguientes temas de seguridad:

  • Conexión segura entre nodos (SSL / TLS);
  • Autenticacion de usuario
  • Autorización de usuario.

Usando ejemplos, consideraremos configuraciones para:

  • SSL / TLS para crear una conexión segura entre todos los nodos, clientes ligeros y conexiones mediante REST;
  • Autenticación a través de Apache Ignite Advanced Security;
  • Complemento de seguridad GridGain.

El seminario web ayudará a los desarrolladores y especialistas en desarrollo a comprender los conceptos básicos de seguridad para Apache Ignite y GridGain, y proporcionará ejemplos de configuración para los casos más comunes.
