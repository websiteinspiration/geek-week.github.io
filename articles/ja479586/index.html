Efros Config Inspectorは、それを使用しない人にもメリットがあります

Efros Config Inspectorソフトウェアパッケージは、アクティブなネットワーク機器構成、仮想化環境、オペレーティングシステムのセキュリティを制御するように設計されており、さらに最近では、ビジネスプロセス自動化ツール（詳細はこちらを参照）が8年以上にわたってさまざまなユーザーによって使用されています。情報技術とサイバー防御ツールの開発の傾向を考えると、開発者はクロスプラットフォームソリューションに特に注意を払っています。

ユーザーが次の冬のリリースで高く評価する構成管理機能を実装するために、OpenSSHクロスプラットフォームパッケージが選択されました。ほとんどのLinuxディストリビューションでサポートされていますが、最近ではWindowsオペレーティングシステムファミリーでもサポートされています。

パッケージの機能を分析した結果、エラーが明らかになりました。クライアント（Ciscoルーター）がサーバーから切断されています。 OpenSSHパッケージのサーバーコードには、さまざまなデバイス、特にシスコデバイスとのいわゆる互換性サポートがあります。使用するデバイスのタイプに応じて、クライアントとサーバー間の相互作用のプロトコルを指定できます。詳細な調査の結果、OpenSSHパッケージをWindowsに移植する際、移植の詳細により、この互換性サポートが無効になっていることが明らかになりました。

脆弱性を排除するために、OpenSSHからのSSHサーバープログラムのソースコードが分析されました。 Windowsへの移植の特殊性を考慮して、クライアント/サーバー相互作用のメカニズムをデバッグするために、対応する改訂が行われました。これは、Ciscoルーターだけでなく、SSHサーバーと対話するすべてのデバイスの問題も解決するように実行されました。作業と内部テストが正常に完了した後、Gazinformserviceはソースコードの変更をWindowsのOpenSSH開発者に転送することを決定しました。

開発コミュニティは、パッケージの開発への貢献を高く評価し、編集が受け入れられました。したがって、Efros Config Inspector開発チームは、オープンソースソリューションを自分の利益のために変更するだけでなく、OpenSSHソフトウェアパッケージのすべての関心を持つユーザーに修正バージョンを提案しました。

オープンソースソフトウェアを使用すると、ビジネスの問題をより早く解決できます。今日、誰もこれに疑いを抱いていません。しかし、ITコミュニティ全体の利益のための「無料」ソフトウェアの開発に対応して、感謝とサポートの面で何かをする必要があります。 Gazinformservice LLC
