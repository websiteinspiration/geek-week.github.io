ZeroTierでゾンビをより効率的に殺す方法



ベッドの下にサーバーがあり、国内にスマートハウスがあり、ガレージにIPカメラがあります。それらすべてを1つのネットワークに統合して、すべてのデバイスにアクセスするにはどうすればよいですか？このような状況では、VPNを使用するのが通例ですが、より効果的なソリューションであるZeroTierがあります。 VPNよりもはるかに簡単に構成できますが、デバイス間にピアツーピア（P2P）ネットワークが作成されます。リモートユーザーサポート用のP2Pネットワークの編成に関する一連の記事

は既に公開しています。その中で、ZeroTierを使用して分散仮想ネットワークを構築するオプションを検討しました。従来のVPNソリューションとは異なり、ZeroTierには中央管理サーバーがなく、デバイス間で直接接続が確立されます。

私は2つのケースでZeroTierをよく使用します。クライアントデバイスに接続して、サポートを提供し、シューティングゲームでゾンビを殺します。ローカルネットワークで別の接続方法を比較することは私にとって興味深いものになりました。

ZeroTierの原理を理解したい場合は、これについて詳細な分析記事を公開しています。

リモートサポート用のRDP接続


ほぼ毎日、何かを解決するためにマシンの反乱を抑えるためにクライアントと接続する必要があります。パンデミックでは、これらは在宅勤務の同僚です。 

会社の内部サービスと連携するには、企業ネットワークからの認証が必要であり、どこかでリモートで何かを行う必要がある場合に備えて、VPNがオフィスルーターで長い間確立されています。 

私にとっては、それに接続してシステムにいくつかのルートを登録することは難しくありませんが、平均的なユーザーにとっては面倒でした。したがって、影響を受けたユーザーが自分のコンピューターに簡単にアクセスできるようにする簡単なソリューションが必要になったときです。

制限とコストのため、当時はTeamviewerやRadminさえ考慮していませんでした。さらに、私はOrthodox RDPのサポーターであり、サードパーティのユーティリティではありません。私は、ソリューションが実装においてできるだけシンプルであり、人が彼の側で最小限のアクションを必要とすることを望みました。そして、私は私の母国の最愛の会社の従業員だけでなく、時にはサードパーティのキャラクターを助けることも心配しているので、企業のVPNサーバーを使用するのは間違っています。

ZeroTierと何を比較したか


ネットワークでは、仮想ネットワークを実装するためのいくつかの簡単なソリューションを見つけることができます。

  • 機能しないタングル。
  • Player.meと統合されたEvolve。
  • ハマチ;
  • Radmin VPN
  • そして、ZeroTierは、市場に出回っています。

上記のソフトウェアのいずれかをインストールし、RDPを有効にするために次のスクリプトをダウンロードして実行するようクライアントに依頼することを計画しました。

# Execute with Administrator rights
$path = $MyInvocation.MyCommand.Path
if ($args[0] -ne $null) {
$path = $args[0]
}
$path = Split-Path -Path $path

$windowsID = [System.Security.Principal.WindowsIdentity]::GetCurrent()
$windowsPrincipal = New-Object System.Security.Principal.WindowsPrincipal($windowsID)
$adminRole=[System.Security.Principal.WindowsBuiltInRole]::Administrator
if ($windowsPrincipal.IsInRole($adminRole)) {
$Host.UI.RawUI.WindowTitle = $myInvocation.MyCommand.Definition + "(Elevated)"
$Host.UI.RawUI.BackgroundColor = "DarkBlue"
clear-host
}
else {
$newProcess = New-Object System.Diagnostics.ProcessStartInfo "PowerShell";
$newProcess.WorkingDirectory = $path;
$newProcess.Arguments = $myInvocation.MyCommand.Definition;
$newProcess.Verb = "runas";
[System.Diagnostics.Process]::Start($newProcess);
exit
}

# Set RDP enabled
Set-ItemProperty -Path "HKLM:\System\CurrentControlSet\Control\Terminal Server" -Name "fDenyTSConnections" –Value 0

New-NetFirewallRule -DisplayName "Remote Desktop Allow UDP" -Program "%SystemRoot%\system32\svchost.exe" -Action Allow -Protocol UDP -LocalPort 3389 -Direction Inbound -Enabled True

  • コストと制限;
  • 機能的;
  • ゲーム（詳細は最後に）。

各パラメーターの私の評価は主観的であり、批判に値します。そのため、テストで測定できるパラメーターに注目しました。

ハマチ


これを書いている時点で、LogMeIn Hamachi for Windows 2.2.0.633プログラムの最新バージョンが2019年3月にリリースされました。無料版は、同じネットワークへの同時接続が5つに制限されています。フルバージョンの費用は年間49ドルからです。

開始するには、次のものが必要です。

  • サービスに登録します。
  • ソフトウェアをインストールする
  • グラフィカルアプリケーションを使用して、完成したネットワークを作成または接続します。

ファイル転送の速度を推定するために、CrystalDiskMarkを使用して、ローカルネットワーク用に以前共有されていたフォルダーで読み取り/書き込みテストを実行し、大きなファイルを簡単にコピーしました。

すべてのテストが同じ構成の2台のPCで実行される（各テストで同じドライブ、同じシステム設定など）と仮定すると、CrystalDiskMarkの結果からデータ転送速度がわかります。

ケース2：オフィスの弱いPCで文化的なリラックスを楽しむ方法


ZeroTierはまた別の機会を提供します-現在人気のある方法でゲームを別のデバイス（La GeForce Now）にストリーミングします。これには、ZeroTierに代わる3つの方法があります。RDP、Parsec、Moonlighrtです。それらをテストしてみましょう。

▍RDP


残念ながら、これには対応していません。私がテストしたゲームのほとんどは、単に正常に動作することを拒否しました。ビデオドライバの非互換性が原因でクラッシュしたか、単にコントロールに応答しませんでした。

▍パーセク


これは、超低レイテンシに重点を置いてPC間でビデオを転送するように設計された専用の無料ソフトウェアです。 TeamviewerやRadminとは異なり、Parsecは、安定した60 FPSをFullHD画像で比較的快適に送信できます（何度もdrakefaceを取得しないようにするため）。



快適に使用するには、ホストに少なくとも30 Mbpsの速度でケーブル接続することをお勧めします。

大多数のゲームでは、大きな入力遅延を経験することなく快適にプレイできますが、残念ながら画質は低下します。圧縮方法により、画像は対照的な要素のピクセルに分割され、目を痛めます。シューティングゲームなどのFPSに依存するゲームは、適切なレベルの快適さではプレイできません。入力ラグは低いですが、時々画像の多少のひねりが目立ちます。これは射撃を妨げることはありませんが、迷惑です。

パーセクが他のソリューションを完全に上回っているのは、地元の協同組合です。Parsecではモニターの画像を共有できるだけなので、どのロケールでも問題なく協力ゲームをプレイできます。また、あなたとの距離は関係ありません。私はノボシビルスクの友達とカップヘッドをするのを楽しんでおり、反応に問題はありませんでした。

パーセクは日常の仕事にも長けています。Teamviewerよりも高速で、より優れた画像を転送し、リモートデスクトップの数に制限はありません。ただし、そのような目的のために、私は常にRDPを使用します。組み込みのOSとローカルドライブを転送する機能のために用意されています。

▍月光


これはオープンソースであり、リモートゲーム用の完全に無料のソリューションです。
私は彼らのサイトを引用します：
Moonlight（旧Limelight）は、NVIDIA GameStreamプロトコルのオープンソース実装です。NVIDIAシールドで使用されるプロトコルを実装し、一連のサードパーティクライアントを作成しました。

Moonlightの最も明白で主な成果は、遅延なしで 4Kビデオを120以上のFPSで放送できることです。

Moonlightは、ホストマシンにNvidiaカードがインストールされている場合にのみ機能します。これは、プロジェクトがNVIDIA Shieldをエミュレートし、ドライバーと直接連携するためです。これにより、ローカルネットワークで遅延なく4K 120FPSをブロードキャストできます。

さらに、Moonlightの2番目の欠点は、タンバリンで追加のダンスを行わないと、インターネットでプレイできないことです。

▍ムーンライト+ハマチ


残念ながら、私にはキャプチャカードがありません。電話のモニターを取り外すことはできません。したがって、私の言葉を信じるか、各ソリューションを個別にテストする必要があります。ハマチの帯域幅は、2K 60-120FPSの快適なゲームに十分です。解像度が高くなると、マイクロフリーズは非常に強くなります。

FPSに依存するゲームをプレイすることは可能ですが、あちこちでゲームが1/4秒間フリーズすることがあります。Parsecは、遅延を最小限に抑えるという点で、まだ改善を続けています。ただし、画質は低下します。ここで、どちらがより重要であるかを決定するのはあなた次第です：最小入力遅延と画像アーチファクト、または完璧な画像ですが、マイクロフリーズですか？

▍Moonlight + Radmin VPN


Radmin VPNはシンセティックと同じように動作しました。再生は比較的快適で、FullHDと60FPSでのみ判明しました。上記の解像度は、耐え難いスライドショーにつながりました。時々、画像は2秒間完全にフリーズしましたが、残念ながら、この束が最悪であることが判明しました。

▍Moonlight + ZeroTier


そして今、私たちはおいしいカップルに行きました。言うまでもなく、公式のMoonlightリポジトリは、P2P接続を構築するためにZeroTierに特にアドバイスしています。リモートで起動したゲームのゲームプレイとローカルPCでの起動を文字通り区別できませんでした。宣言されているすべてのMoonlight機能が動作します。ノボシビルスク出身の私の友人がAMDカードを持っていて、彼の兄弟（これはモスクワの北と南です）を呼び出して同じ都市でテストしなければならなかったのは残念です。

画像は最大4Kのスムーズな解像度で、ブレーキをかける必要はありません。押すとすぐに反応し、このようなシューティングゲームをプレイできます。おそらく、このバンドルはクラウドゲームに最適なオプションです。

調査結果


ZeroTierは、ピアツーピアネットワークを迅速に構築するためのおそらく最も手頃なソリューションです。はい、その機能では複雑なネットワークを構築できず、従来のL2TP / IPsecのさまざまな設定すべてにアクセスできません。ただし、これは単純なタスクでは必ずしも必要ではありません。

Parsecはおそらく、ローカルの協力画面または分割画面をサポートするリモートでゲームをプレイするための唯一のオプションです（私はすぐに危険な時空の恋人を思い出しました）。物理装置の特定のテクノロジーに関連付けられておらず、追加のネットワーク設定を必要としません。ただし、画質が大幅に低下する可能性があります。通常の方法でリモートPCに接続する機能がない場合は、迅速なコンピューター支援に適しています。

月光-NvidiaカードがゲームPCにインストールされている場合、Moonlight + ZeroTierはリモートゲームに理想的なペアです。ZeroTierは、仮想ネットワークを簡単かつエレガントにセットアップする問題を解決します。ムーンライトは、Nvidia Shieldのエミュレーションのおかげで、美しい写真をすぐに転送できます。リモートPCの設定はできませんが、上記の他のソリューションについて説明しました。
