Angularでのディレクティブ機能の遅延使用

最近、コンポーネントライブラリを使用して実装されたツールチップを表示するための古いメカニズムを新しいメカニズムに変更する問題を解決する必要がありました。いつものように、私は自転車を発明しないことに決めました。この問題を解決するために、純粋なJavaScriptで記述された、Angularディレクティブに配置してこのフォームで使用できるオープンソースライブラリの検索を開始しました。 私の場合、私はpopper.jsをよく使用しているので、tippy.jsライブラリを見つけました



同じ開発者によって書かれました。私にとって、そのようなライブラリーは問題に対する完璧な解決策のように見えました。tippy.jsライブラリには、広範な機能セットがあります。その助けを借りて、ツールチップ（ツールチップ要素）や他の多くの要素を作成できます。これらの要素は、テーマを使用してカスタマイズできます。これらは高速で、厳密に型指定されており、コンテンツへのアクセシビリティを提供し、他の多くの便利な機能が異なります。

まず、tippy.jsのラッパーディレクティブを作成します。

@Directive({ selector: '[tooltip]' })
export class TooltipDirective {
  private instance: Instance;
  private _content: string;

  get content() {
    return this._content;
  }

  @Input('tooltip') set content(content: string) {
    this._content = content;
    if (this.instance) this.instance.setContent(content);
  }

  constructor(private host: ElementRef<Element>, private zone: NgZone) {}

  ngAfterViewInit() {
    this.zone.runOutsideAngular(() => {
      this.instance = tippy(this.host.nativeElement, {
        content: this.content,
      });
    });
}

ツールチップは、関数tippyを呼び出して要素hostを渡して作成されcontentます。さらに、変更検出サイクルをトリガーするためにtippyログに記録さtippyれるイベントは必要ないため、Angular Zoneの外側を呼び出します。

すべてのイベントリスナーの概要

コードをこの形式のままにすると、アプリケーションのメモリ消費と最初のレンダリングの時間に影響を与える可能性があります。これは、モバイルデバイスでのページ出力に特に当てはまります。これについて考えましょう。tippyビューポートに表示されないアイテムのインスタンスを作成する必要がありますか？いいえ、必要ありません。

APIIntersectionObserverを使用して、アイテムが画面に表示されるまでツールチップサポートの組み込みを遅らせます。 APIIntersectionObserverに慣れていない場合は、ドキュメントをご覧ください。 

監視IntersectionObserver対象オブジェクトで表されるラッパーを作成します。

const hasSupport = 'IntersectionObserver' in window;

export function inView(
  element: Element,
  options: IntersectionObserverInit = {
    root: null,
    threshold: 0.5,
  }
) {
  return new Observable((subscriber) => {
    if (!hasSupport) {
      subscriber.next(true);
      subscriber.complete();
    }

    const observer = new IntersectionObserver(([entry]) => {
      subscriber.next(entry.isIntersecting);
    }, options);

    observer.observe(element);

    return () => observer.disconnect();
  });
}

要素が特定の領域と交差する瞬間をサブスクライバーに通知する監視可能なオブジェクトを作成しました。また、ここではIntersectionObserverブラウザのサポートを確認しています。ブラウザがをサポートしていない場合はIntersectionObserver、単に発行trueして終了します。IEユーザー自身がその悲惨さのせいにするべきです。

これinViewで、観察されたオブジェクトをツールチップ機能を実装するディレクティブで使用できます。

@Directive({ selector: '[tooltip]' })
export class TooltipDirective {
  ...

  ngAfterViewInit() {
    //   
    inView(this.host.nativeElement).subscribe((inView) => {
      if (inView && !this.instance) {
        this.zone.runOutsideAngular(() => {
          this.instance = tippy(this.host.nativeElement, {
            content: this.content,
          });
        });
      } else if (this.instance) {
        this.instance.destroy();
        this.instance = null;
      }
    });
  }
}
