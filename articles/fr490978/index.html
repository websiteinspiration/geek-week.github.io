Envoyer des commandes à plusieurs appareils en même temps à l'aide de SecureCRT

Les ingénieurs réseau ont souvent pour tâche d'effectuer une série d'actions identiques sur un grand nombre d'équipements. La principale façon de le faire rapidement est d'écrire des actions sur le bloc-notes et de les copier / coller sur chaque appareil. Il existe un moyen d'accélérer ce processus déjà rapide en utilisant le logiciel SecureCRT et la fonctionnalité de la fenêtre de commande. Ce qui vous permet d'envoyer des commandes à plusieurs appareils en même temps.
À quoi sert cet article? Cet article est issu du cycle de démarrage rapide et vise à faire gagner du temps aux ingénieurs réseau lors de la configuration de l'équipement (une tâche) sur plusieurs appareils. Il utilise le logiciel SecureCRT et la fonctionnalité de fenêtre de commande intégrée.

introduction


Lors de la configuration de l'équipement réseau, il est souvent nécessaire d'effectuer des actions simples et identiques sur un grand nombre d'appareils. Pour effectuer ces tâches, vous pouvez utiliser la méthode standard: Bloc-notes + Copier / Coller. Et vous pouvez utiliser la fonctionnalité intégrée SecureCRT - fenêtre de commande - une fenêtre spéciale dans laquelle vous pouvez modifier la commande avant de l'envoyer à l'équipement. Et une fonctionnalité supplémentaire: envoyer une commande non seulement à la session en cours, mais aussi à plusieurs sessions ouvertes.

Fenêtre de commande d'appel, définition des modes de fonctionnement


La fenêtre de commande est appelée via le menu: Menu SecureCRT => Affichage => Fenêtre de commande. Ou Menu SecureCRT => Afficher => Fenêtre de discussion.


Les modes de fonctionnement peuvent être:
  • Mode édition
  • En temps réel ("Envoyer les caractères immédiatement")

Mode d'édition - vous permet de modifier la commande avant de l'envoyer à l'appareil.
Mode temps réel - toute pression sur une touche est automatiquement envoyée à l'appareil - qu'il s'agisse d'un espace ou d'une pression de touche (alphabétique ou Entrée).

Le paramètre de transfert de commande a les modes de fonctionnement suivants:
  • Transférer à la session active (uniquement la session actuelle).
  • Dans toutes les sessions.
  • Vers le groupe (onglet) ou le groupe sélectionné.

Officiellement, les options sont les suivantes:
Active Session — Send commands only to the active session. This is the default.
All Sessions — Send commands to all your open sessions at once.
Selected Tab Group(s) — Send commands to one or more tab groups. This setting is automatically selected when you right-click on a tab and enable Send Commands to This Group. When commands are being sent to selected tab groups, the tabs of the sessions that commands will be sent to contain an asterisk in front of the session name.
Visible Sessions — Send commands to all visible tabbed sessions. When there are multiple tab groups, commands will be sent to the currently active session in each tab group.

Le paramètre de mode en temps réel est activé à l'aide du menu contextuel de la fenêtre de commande. (Cliquez avec le bouton droit sur la fenêtre de commande - un menu contextuel apparaît et sélectionnez l'élément de menu "Envoyer les caractères immédiatement"). Les
paramètres de transfert de commande sont également appelés à l'aide du menu contextuel de la fenêtre de commande. (Faites un clic droit sur la fenêtre de commande - un menu contextuel apparaît, sélectionnez "Envoyer les commandes à", sélectionnez le sous-élément de menu {"Toutes les sessions" | "Onglet sélectionné" | "Session active")


Utilisation de la fenêtre de commande pour configurer plusieurs appareils en même temps


Très souvent, pour fonctionner, vous devez exécuter des commandes simples simultanément sur plusieurs appareils. Pour ce faire, il est très pratique (car rapide) d'utiliser la fenêtre de commande pour envoyer des données à plusieurs appareils simultanément.
Pour configurer plusieurs appareils:
  1. Nous sommes connectés à tous les appareils du même type.
  2. Dans le menu SecureCRT, appelez la fenêtre de commande
  3. Dans le menu contextuel de la fenêtre de commande, sélectionnez Toutes les sessions.
  4. Nous entrons les commandes comme une seule unité ou une à la fois. (Dans un seul échange, les commandes sont entrées via Ctrl + Entrée. - saut de ligne sans transfert vers l'équipement.)

Il s'agit généralement d'un ensemble standard de commandes:
screen-len 0 tem
dis cu
save
y
q

Connexion rapide à plusieurs sessions


Sélectionnez les sessions auxquelles nous voulons nous connecter en même temps et appuyez sur Entrée.

Nous envoyons des commandes à tous les appareils simultanément:
ter len 0
show ip int br

Utilisation de la fenêtre de commande pour configurer plusieurs appareils en temps réel en même temps.


Que peut-on utiliser et vraiment utiliser en temps réel? Réponse: lorsque des paramètres identiques sont configurés, mais avec une légère différence. Exemple de nom d'hôte, d'adresse IP sur l'interface.
A quoi cela ressemble en réalité:
1. Les sessions sont ouvertes où il est nécessaire d'entrer presque les mêmes paramètres.

2. Le mode de fonctionnement de la fenêtre de commande est configuré: "Envoyer immédiatement les caractères" + "Envoyer les commandes à: Toutes les sessions".

3. Les données suivantes sont entrées dans la fenêtre de commande:
interface gi0/0/0.10
dot1q termination vid 10
ip address 172.31.255.

4. Le curseur est transféré à l'aide de la souris de la fenêtre de commande vers la fenêtre de session et l'adresse nécessaire est configurée pour chaque périphérique de la session:
R1 – 1
R2 – 2
R3 – 3

5. Le curseur est transféré à l'aide de la souris dans la fenêtre de commande et le suffixe est ajouté:
 255.255.255.0
quit
dis ip int br


Petit truc. Pour vérifier l'orthographe - je vérifie d'abord l'orthographe en une seule session. Après vérification, la commande est envoyée à toutes les sessions.


Conclusion


Vous devez toujours être prudent.
Dans certains cas, il est plus rapide de se déconnecter de toutes les sessions. Connectez-vous uniquement aux sessions personnalisées. Entrez des commandes dans les fenêtres de commandes. Après vous être connecté à toutes les autres sessions. La connexion à SecureCRT est très rapide. Et la reconnexion ne pose pas de problème.
Très souvent, après avoir entré les commandes de groupe, le travail dans chaque session est vérifié (en utilisant la combinaison [ctrl + tab], nous basculons sur toutes les fenêtres et comparons le résultat de la commande dans chaque fenêtre). Nous vérifions l'exactitude de l'exécution de toutes les commandes, erreurs pouvant survenir au cours d'une même session. Exemple: lorsqu'il est connecté à 3 routeurs Huawei, l'un d'eux par défaut est connecté au mode utilisateur. Pour travailler dans le fil général, j'ai basculé le mode de configuration en mode privilégié.
