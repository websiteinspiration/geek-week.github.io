WebStorm 2020.1: mejoras de interfaz, soporte de Vuex y lanzamiento de Prettier al guardar archivos

¡Hola a todos! Nos complace presentar la primera actualización importante de WebStorm este año. En la nueva versión, encontrará muchas funciones nuevas y mejoras muy esperadas, incluido el soporte de Vuex y una nueva opción para iniciar Prettier al guardar archivos.



Descargue la versión de prueba de 30 días de WebStorm 2020.1 en línea o con la aplicación Toolbox . Los titulares de una suscripción activa a WebStorm o All Products Pack pueden utilizar la versión completa , así como estudiantes y desarrolladores gratuitos de proyectos de código abierto.

Ahora veamos las principales mejoras con más detalle.

Nueva fuente para trabajar con código


Durante el año pasado, hemos desarrollado una fuente especial para trabajar con código que ayudaría a nuestros usuarios a programar con gran comodidad y minimizar la fatiga visual. El resultado de nuestros esfuerzos es JetBrains Mono , una nueva fuente de código abierto. A partir de la versión 2020.1, JetBrains Mono se selecciona de forma predeterminada en WebStorm y nuestros otros IDE. La capacidad de incluir una fuente diferente, por supuesto, también se mantuvo.



Documentación rápida más útil.


WebStorm 2020.1 lo ayuda a encontrar rápidamente la información relevante sobre un símbolo, ya que ahora se muestra una documentación rápida al pasar el mouse sobre ese símbolo, y no solo cuando presiona F1 . Si hay un problema en su código y WebStorm puede ayudarlo con su solución, también se lo informará al mostrar una solución rápida disponible directamente en la ventana emergente de documentación.



Cuando se trabaja con código JavaScript y TypeScript, la documentación rápida mostrará información detallada sobre el tipo y la visibilidad de un carácter, así como también dónde se define ese carácter.



Modo Zen para trabajo enfocado


El nuevo modo Zen te ayuda a concentrarte completamente en tu código. Combinando dos modos, Distraction Free y Full Screen , le permite activarlos rápidamente y aislar temporalmente algo que puede distraer su atención.



Para probar el nuevo modo, vaya a Ver | Apariencia | Ingrese al modo Zen desde el menú principal de WebStorm o use la ventana emergente Cambiar ( Ctrl + `| Modo de visualización | Ingrese al modo Zen ).

Soporte para Vuex y Vue Composition API


Vue.js está ganando popularidad, por lo que estamos trabajando activamente para hacer de WebStorm el IDE más útil para trabajar con este marco. Esta vez agregamos dos mejoras importantes.

Para empezar, cuando trabaje con la biblioteca Vuex, verá opciones para completar el código de los símbolos y módulos de la tienda Vue al editar los componentes de Vuex. WebStorm también lo ayuda a pasar rápidamente a definir captadores, mutaciones y acciones.



En segundo lugar, dado que ya puede usar la API de composición, disponible en la próxima versión de Vue 3, decidimos agregarle soporte en WebStorm 2020.1.



Ejecutando Prettier al guardar archivos


Gracias a la nueva opción Ejecutar al guardar para archivos , puede aplicar el formato Prettier a todos los archivos especificados en la configuración de WebStorm y editados en el proyecto actual, mientras guarda estos archivos; ya no necesita configurar el observador de archivos o usar un complemento de terceros.



Ayuda con errores gramaticales y estilísticos.


A partir de la versión 2020.1, WebStorm viene incluido con Grazie, nuestra herramienta para revisar la ortografía, la gramática y el estilo de texto. Esto lo ayudará a evitar errores gramaticales al agregar comentarios, confirmaciones y diversas construcciones de lenguaje.



De forma predeterminada, Grazie está habilitado solo para el idioma inglés y no verifica todos los tipos de archivos para detectar posibles errores gramaticales. Puede agregar más idiomas y cambiar configuraciones predefinidas en Preferencias / Configuraciones | Editor | Corrección de pruebas (vaya a la sección Gramática si desea configurar el alcance del análisis, consulte las reglas existentes y agregue excepciones).

Nuevas acciones de intención e inspecciones


Como siempre, hemos agregado algunas nuevas acciones de intención y soluciones rápidas para ayudarlo a ahorrar tiempo al trabajar con código JavaScript y TypeScript. Por ejemplo, una de las nuevas acciones intencionales, disponibles presionando Alt + Entrar , convertirá rápidamente el código existente a encadenamiento opcional y / o fusión nula.



Más información sobre las nuevas acciones intencionales y soluciones rápidas está disponible en la publicación del blog de lanzamiento (en inglés).

Soporte para las últimas versiones de tecnologías populares.


En WebStorm 2020.1 encontrará soporte integrado para la funcionalidad TypeScript 3.8, incluidas las importaciones / exportaciones de tipo único, los campos privados y el nivel superior await, que lo ayudarán a trabajar con ellos de manera más eficiente. Además, facilitamos el trabajo con proyectos de Angular 9 al admitir completamente el nuevo formato de metadatos.

Rebajas más convenientes de commits


La acción Interactivamente Rebase from Here , disponible en la pestaña Registros de la ventana Git , le permite editar, fusionar y eliminar confirmaciones anteriores. Esto ayuda a que el historial de confirmaciones sea más lineal y comprensible.



En WebStorm 2020.1, encontrará una versión mejorada y más interactiva del cuadro de diálogo Rebasar compromisos que se abre cuando se llama a la acción Interactivamente Rebase from Here . El cuadro de diálogo actualizado muestra acciones que se pueden aplicar a cada confirmación. También muestra información detallada sobre cada confirmación y le permite ver la diferencia y descartar rápidamente los cambios aplicados.

Usando WebStorm para editar archivos rápidamente


Gracias al nuevo modo LightEdit , puede abrir archivos en una ventana de editor de texto separada, sin crear o cargar un proyecto completo. El modo funcionará si WebStorm aún no se está ejecutando. Si se está ejecutando WebStorm, el archivo se abrirá en él, como de costumbre.

Para probar el nuevo modo en acción, haga clic con el botón derecho en el archivo que desea editar y seleccione WebStorm de la lista de programas propuestos. Alternativamente, también puede configurar el iniciador de línea de comando como se describe aquí y abrir el archivo usando la línea de comando.
