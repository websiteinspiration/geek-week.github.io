Personalize seu ambiente Python com pyenv, virtualenvwrapper, tox e pip-compile





Essas ferramentas simplificarão a configuração e automatizarão as operações de rotina. Eles salvarão o desenvolvedor de muitas dificuldades que impedem que você se concentre na solução de problemas e na escrita confortável de códigos.



Existem várias maneiras de personalizar seu ambiente Python. Este artigo é sobre um deles. Mas esta, é claro, não é a única solução.



Python é uma linguagem de programação de uso geral que é frequentemente recomendada para iniciantes. Ao longo dos vinte anos de sua existência, vários livros sobre Python foram escritos. Embora muitas vezes referido como simples, configurar ferramentas de desenvolvimento Python está longe de ser uma tarefa fácil.









A configuração do ambiente Python é bastante complicada: xkcd



Use pyenv para controle de versão



Na minha opinião, este é o melhor gerenciador de versões do Python. No entanto, é importante notar que o pyenv é executado no Linux, Mac OS X e WSL2: ou seja, três ambientes "semelhantes ao UNIX".



Instalar o próprio pyenv às vezes pode ser difícil. Mas um instalador pyenv especial que usa curl | pode ajudar. bash para inicialização.



Se você estiver usando um Mac (ou outro sistema em que esteja executando o Homebrew), poderá ler as instruções para instalar e usar o pyenv aqui .



Após instalar e configurar o pyenv, você pode usar o pyenv global para instalar a versão padrão do Python. Você pode escolher sua versão "favorita". Geralmente, esta é a versão estável mais recente, mas não é precisa -)



Use virtualenvwrapper para configurar um ambiente virtual



Para Python, você pode criar ambientes virtuais, ou seja, colocar cada projeto em um ambiente isolado. Isso pode ser feito usando o virtualenvwrapper. Então, é possível alternar entre ambientes virtuais a qualquer momento.



Leia mais sobre como configurar um ambiente virtual aqui .



Se você precisar criar e usar repetidamente o mesmo ambiente virtual, faz sentido automatizar o processo. Hoje minha escolha é tox.



Use tox para automatizar



O Tox é uma ótima ferramenta para automatizar seus testes. Em todos os ambientes Python, eu crio um arquivo tox.ini. Não importa qual sistema eu use para integração contínua, tudo funcionará. E posso executá-lo localmente usando o virtualenvwrapper: O problema é que estou testando meu código em várias versões do Python e em várias versões das dependências da biblioteca. Isso significa que o tox funcionará em vários ambientes. Alguns deles terão dependências reais. Em alguns - congelados. Eu também poderia gerá-los localmente usando o pip-compile. Recentemente, estive considerando o nox como um substituto para o tox. Neste artigo, não explicarei os motivos, mas você deve prestar atenção.

Use pip-compile para gerenciar dependências



Cada vez que o Python carrega suas dependências no estágio de execução do código. Compreender e controlar qual versão das dependências é usada ajudará a evitar falhas inesperadas enquanto o programa está sendo executado. Isso significa que devemos pensar nas ferramentas de gerenciamento de dependências.



Para cada novo projeto, incluo o arquivo require.in, que (como regra geral) contém apenas um caractere:



.







Sim, tudo está correto aqui. Uma única sequência de pontos. No arquivo setup.pyfile para dependências, escrevo Twisted> = 17.5 em vez de Twisted == 18.1, o que dificulta a atualização para novas versões da biblioteca.



"." significa "diretório atual" no qual o arquivo setup.py correspondente é usado como fonte de informações de dependência.



Isso significa que, ao usar o pip-compile requirements.in> requirements.txt, um arquivo com dependências congeladas será criado. Você pode usar esse arquivo de dependência no ambiente virtual criado por virtualenvwrapper ou no tox.ini.



Às vezes, o arquivo requirements-dev.txt gerado a partir de requirements-dev.in (conteúdo :. [Dev]) ou requirements-test.txt gerado a partir de requirements-test.in (conteúdo :. [Test]) pode ser útil.



Alternativamente, o pip-compile pode usar dephell . A ferramenta dephell tem muitas coisas interessantes, por exemplo, usando solicitações HTTP assíncronas para carregar dependências.



Resultado



Considero o Python uma linguagem poderosa e bonita. No entanto, para trabalhar produtivamente, uso um certo conjunto de ferramentas que provaram sua eficácia, pelo menos para mim. Portanto, use essas quatro ferramentas - pyenv, virtualenvwrapper, tox e pip-compile - e você será feliz =)
