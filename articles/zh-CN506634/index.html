Webpack 5中的模块联合，一个用于在Javascript应用程序，描述和示例之间交换模块的插件

在Webpack构建器的第五版中，出现了一组用于在Javascript应用程序之间交换模块的插件。

本文是简短说明，并且是一个基于两个基于ReactJS框架构建的小型应用程序的示例。

Module Federation插件允许应用程序将一个或多个模块导出到单独的JS文件中。构建微前端应用程序的好方法。第三方应用程序可以导入现成的模块，例如，对组件进行反应。而且，依赖项的导入Webpack接管了。与NPM的区别在于它会导入到运行时。



webpack配置中的源应用程序明确指示：

  • 出口模块
  • 导出文件名，例如export.js
  • 不需要导出的依赖项，例如export.js

在接收方应用程序的HTML中，源应用程序的JS文件作为常规JS脚本导入： 

<script src="http://source-app.com/export.js">

在webpack配置中，哪些模块是从文件中获取的。

看起来就是这样


源应用程序应在webpack.config.js设置中注册导出，
在插件设置中，指定导出容器的名称，将包含在其中的模块以及如何调用文件。

配置示例 
const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
 
module.exports = {
 ...
 plugins: [
   new ModuleFederationPlugin({
     name: 'home',
     library: { type: 'var', name: 'home' },
     filename: 'export.js',
     exposes: {
       ProductCarousel: './src/ProductCarousel'
     },
     shared: ['react', 'react-dom', '@material-ui/core', '@material-ui/icons']
   }),
 ]
};

收件人申请

  • 在javascript.html中导入源应用程序js文件

    例 
    <!DOCTYPE html>
<html lang="en">
 <head>
   <script src="http://source-app.com/export.js"></script>

  • 在webpack.config.js的插件设置中，指定所需容器的名称，插件将在JS文件中找到它

    配置示例 
    const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
 
module.exports = {
...
 plugins: [
   new ModuleFederationPlugin({
     remotes: {home: 'home' } // 
   }),
 ]
};

  • 在JSX中，从所需容器中导入模块，就好像它是常规本地模块一样

    代码示例 
    import React from 'react';
import ProductCarousel from 'home/ProductCarousel';
 
function App() {
 return (
     <ProductCarousel />
 );
}
 
export default App;
