抜粋。第25章プロジェクトの始まり


この章では、プログラムの作成を開始します。このプロジェクトの目標は、シェルのさまざまな機能を使用してプログラムを作成する方法、そして最も重要なこととして、優れたプログラムを作成する方法を示すことです。

次に、レポートジェネレータを記述します。システムとそのステータスに関するさまざまな情報をHTML形式で表示するため、Webブラウザーで表示できます。

通常、プログラムの作成はいくつかの段階で実行され、それぞれの段階で新しい機能と機能が追加されます。最初の段階の最後に、プログラムは情報なしで最小限のHTMLページを再生します。次のステップでこの情報を追加します。

ステージ1：最小ドキュメント


まず最初に、整形式のHTMLドキュメントの形式がどのようなものかを確認しましょう。次の形式になります。

<html>
       <head>
           <title> </title>
       </head>
       <body>
                .
        </body>
</html>

このテキストをテキストエディターに入力してfoo.htmlというファイルに保存すると、Firefoxで次のURLを入力してファイルを開くことができます：file：///home/username/foo.html。
最初の段階では、このHTMLマークアップを標準出力に出力するプログラムを作成します。そのようなプログラムを書くのはとても簡単です。テキストエディターを開き、〜/ bin / sys_info_pageというファイルを作成します。

[me@linuxbox ~]$ vim ~/bin/sys_info_page

次に、次のプログラムを入力します。

#!/bin/bash
#       
echo "<html>"
echo "  <head>"
echo "         <title>Page Title</title>"
echo "  </head>"
echo "  <body>"
echo "          Page body."
echo "  </body>"
echo "</html>"

最初のバージョンには、シバン行、コメント（ようこそ）、および各行出力に1つずつの一連のエコーコマンドが含まれています。ファイルを保存した後、実行可能にして実行してみます。

[me@linuxbox ~]$ chmod 755 ~/bin/sys_info_page
[me@linuxbox ~]$ sys_info_page

開始後、スクリプト内のエコーコマンドが行を標準出力に送信するため、HTMLドキュメントのテキストが画面に表示されます。プログラムを再度実行し、出力をsys_info_page.htmlファイルにリダイレクトして、Webブラウザーで結果を確認します。

[me@linuxbox ~]$ sys_info_page > sys_info_page.html
[me@linuxbox ~]$ firefox sys_info_page.html

ここまでは順調ですね。

プログラムを開発するときは、常に単純さと明快さについて覚えておく必要があります。プログラムが読みやすく理解しやすい場合は、メンテナンスが容易です。手動入力の量を減らすことができる場合は、プログラムが記述しやすいことは言うまでもありません。プログラムの現在のバージョンは問題なく動作しますが、簡略化することができます。すべてのエコーコマンドを1つに組み合わせると、将来、プログラムの出力に新しい行を追加することが確実に容易になります。したがって、以下のようにプログラムを変更します。

#!/bin/bash

#       

echo "<html>
        <head>
                <title>Page Title</title>
        </head>
        <body>
                Page body.
        </body>
</HTML>"

引用符で囲まれた文字列には、改行を含めることができるため、複数行のテキストを含めることができます。シェルは、終了の引用に遭遇するまでテキストを読み続けます。このルールはコマンドラインにも適用されます：

[me@linuxbox ~]$ echo "<html>
>         <head>
>                 <title>Page Title</title>
>         </head>
>         <body>
>                 Page body.
>         </body>
> </html>"

各行の先頭の>文字は、PS2変数で定義されたシェルに入るためのプロンプトです。複数行の命令が入力されるたびに表示されます。この機能はまだあいまいですが、複数行のプログラム命令に慣れると、その利点が明らかになります。

ステップ2：データを追加する


プログラムが最小限のドキュメントを生成できるようになったので、レポートにデータを追加します。これを行うには、次の変更を行います。

#!/bin/bash

#       
echo "<html>

        <head>
                <title>System Information Report</title>
        </head>
        <body>
                <h1>System Information Report</h1>
        </body>
</html>"

ここに、レポートの本文にページの名前とタイトルが追加されます。

変数と定数


スクリプトに問題がありました。システム情報レポートの行が2回繰り返されていることに気づきましたか？一般に、このような小さなシナリオの場合、これはそれほど大きな問題ではありませんが、この行が何度も繰り返される非常に長いシナリオを想像してください。このようなシナリオで名前を変更する必要がある場合は、多くの場所で変更を加える必要があり、これは多くの手作業です。文字列が一度だけ定義されるようにスクリプトを変更することは可能ですか？これにより、将来のスクリプトのメンテナンスが大幅に簡略化されます。はい、これはたとえば次のように可能です：

#!/bin/bash

#       

title="System Information Report"

echo "<html>
        <head>
                <title>$title</title>
        </head>
        <body>
                <h1>$title</h1>
        </body>
</html>"

title という変数を作成し、それにSystem Information Reportという値を割り当てることで、パラメーターの置換を利用して、文字列をさまざまな場所に配置しました。

しかし、変数を作成する方法は？シンプル-使うだけ。シェルは変数を検出すると、それを自動的に作成します。これは、変数を使用する前に明示的に宣言または定義する必要がある多くのプログラミング言語とは異なります。コマンドシェルは、この点で自由になりすぎて、最終的にいくつかの問題を引き起こします。たとえば、コマンドラインで実行される次のスクリプトを考えてみます。

[me@linuxbox ~]$ foo="yes"
[me@linuxbox ~]$ echo $foo
yes
[me@linuxbox ~]$ echo $fool

[me@linuxbox ~]$

最初に、変数fooに値yesを割り当て、次に、echoコマンドでその値を出力しました。次に、変数の値を再度表示しようとしましたが、名前foolを指定してタイプミスをし、空の文字列を取得しました。この結果は、シェルがfool変数に遭遇するとそれを正常に作成し、それに空のデフォルト値を割り当てたことによって説明されます。この例から、スペルを注意深く監視する必要があることがわかります。この例で実際に何が起こったかを理解することも重要です。置換メカニズムの機能についての以前の知り合いから、チームは

[me@linuxbox ~]$ echo $foo

パラメータ置換のメカニズムに公開され、次の形式になります

[me@linuxbox ~]$ echo yes

一方、チーム

[me@linuxbox ~]$ echo $fool

になる

[me@linuxbox ~]$ echo

空の変数は何も置換されません！これにより、引数を必要とするコマンドでエラーが発生する可能性があります。例えば：

[me@linuxbox ~]$ foo=foo.txt
[me@linuxbox ~]$ foo1=foo1.txt
[me@linuxbox ~]$ cp $foo $fool
cp:  'foo.txt'  ,   
  "cp --help"    .

2つの変数fooとfoo1に値を割り当てました。そして、cpコマンドを実行しようとしましたが、2番目の引数の名前にタイプミスがありました。検索メカニズムによる処理後、cpコマンドは1つの引数しか受け取りませんでしたが、2つ必要です。

以下は、変数の命名に関するいくつかのルールです。

  • 変数名には、英数字（文字と数字）とアンダースコアを使用できます。
  • 変数名の最初の文字は、文字またはアンダースコアのみです。
  • 変数名にスペースや句読点を含めることはできません。

名前変数は、変更可能な値を意味し、多くのアプリケーションでは、変数はそのように使用されます。ただし、アプリケーションのタイトル変数は定数として使用されます。変数と同様に、定数には名前があり、値が含まれています。唯一の違いは、定数の値が変化しないことです。幾何学的計算を実行するアプリケーションでは、プログラム全体でこの数値を使用する代わりに、3.1415の値でPI定数を定義できます。シェルは定数と変数を区別しません。これらの用語は、主にプログラマーの便宜のために使用されます。典型的な規則は、大文字を使用して定数を示し、小文字を真の変数に使用することです。この慣例に沿ってシナリオを変更してみましょう。

#!/bin/bash

#       

TITLE="System Information Report For $HOSTNAME"

echo "<html>
        <head>
                <title>$TITLE</title>
        </head>
        <body>
                <h1>$TITLE</h1>
        </body>
</html>"

途中で、HOSTNAMEシェル変数の値を最後に追加して名前を補足しました。これはマシンのネットワーク名です。

注意


実際、シェルには、定数の不変性を保証するメカニズムがあり、-rパラメーターを指定した組み込みの宣言コマンドの形式で提供されます（読み取り専用-読み取り専用）。以下に示すようにTITLE変数に値を割り当てる場合：

declare -r TITLE="Page Title"

シェルでは、値をTITLE変数に再割り当てできません。このメカニズムは実際にはめったに使用されませんが、使用可能であり、特に厳しいシナリオで使用できます。

変数と定数への値の割り当て


代替メカニズムの動作の特徴についての知識が実を結び始めたときがきました。見てきたように、変数への値の割り当ては次のように行われます：

=

ここで、variableは変数の名前、valueは文字列です。他の一部のプログラミング言語とは異なり、シェルは変数に割り当てられた値のタイプを気にしません。彼女はすべての値を文字列として解釈します。-iオプションを指定したdeclareコマンドを使用して、割り当てられた値の範囲を整数に制限するようにシェルに強制することは可能ですが、読み取り専用変数の宣言と同様に、この機能は実際にはほとんど使用されません。

変数名、等号、値の間の代入演算子にはスペースがないことに注意してください。そして、その意味は何から成っていますか？何からでも文字列に展開できます。

a=z                 #   a  "z".
b="a string"        #      .
c="a string and $b" #     ,
                    # ,   .
d=$(ls -l foo.txt)  #   .
e=$((5 * 7))        #    .
f="\t\ta string\n"  #  ,  
                    #     .

1行で、複数の変数を一度に割り当てることができます。

a=5 b="a string"

置換を使用する場合、変数名をオプションの中括弧{}で囲むことができます。これは、周囲のコンテキストで変数名があいまいになる場合に役立ちます。次の例では、変数を使用してmyfileの名前をmyfile1に変更しようとしています。

[me@linuxbox ~]$ filename="myfile"
[me@linuxbox ~]$ touch $filename
[me@linuxbox ~]$ mv $filename $filename1
mv:  'myfile'  ,   
  "mv --help"    .

シェルがmvコマンドの2番目の引数を新しい（および空の）変数の名前として解釈したため、この試みは失敗しました。この問題を解決する方法を次に示します。

[me@linuxbox ~]$ mv $filename ${filename}1

中括弧を追加することで、シェルが最後の文字1を変数名の一部として解釈しないようにしました。

注意


置換を実行するときは、シェルが行を単語に分割しないように、変数名とコマンドを二重引用符で囲むことをお勧めします。変数にファイル名が含まれる可能性がある場合は、引用符を使用することが特に重要です。
この機会を使用して、レポートに追加のデータ、つまりレポートの日付と時刻、およびレポートを作成したユーザーの名前を追加します。

#!/bin/bash

#       

TITLE="System Information Report For $HOSTNAME"
CURRENT_TIME=$(date +"%x %r %Z")
TIME_STAMP="Generated $CURRENT_TIME, by $USER"

echo "<html>
        <head>
                <title>$TITLE</title>
        </head>
        <body>
                <h1>$TITLE</h1>
                <p>$TIME_STAMP</p>
        </body>
</html>"

埋め込みドキュメント


2つの異なるテキスト出力方法を調べましたが、どちらもechoコマンドを使用しています。ただし、インラインドキュメント（ここではドキュメント）またはインラインスクリプト（ここではスクリプト）と呼ばれる別の3番目の方法があります。埋め込みドキュメントは、スクリプトに埋め込まれたテキストを標準のコマンド入力に渡すI / Oリダイレクトの追加形式です。このリダイレクトは次のように機能します。

 <<

ここで、commandは、標準入力を介して指定されたテキストを受け取るコマンドの名前であり、indicatorは、埋め込みテキストの終わりを示す行です。埋め込まれたドキュメントを使用してシナリオを変更します。

#!/bin/bash

#       

TITLE="System Information Report For $HOSTNAME"
CURRENT_TIME=$(date +"%x %r %Z")
TIME_STAMP="Generated $CURRENT_TIME, by $USER"

cat << _EOF_
<html>
        <head>
                <title>$TITLE</title>
        </head>
        <body>
            <h1>$TITLE</h1>
            <p>$TIME_STAMP</p>
        </body>
</html>
_EOF_

これで、echoコマンドの代わりに、スクリプトはcatコマンドと埋め込みドキュメントを使用します。行_EOF_は、インジケーターの役割（ファイルの終わり - ファイルの終わり、共通の合意を意味する）として選択され、埋め込みテキストの終わりを示します。インジケーターラインは別の行にある必要があり、その後にスペースを入れないでください。

しかし、ここに埋め込まれたドキュメントを使用する利点は何ですか？埋め込まれたドキュメント内の引用符がシェルにとって特別な意味を失うことを除いて、事実上何もありません。以下は、コマンドラインで埋め込みドキュメントを使用する例です。

[me@linuxbox ~]$ foo="some text"
[me@linuxbox ~]$ cat << _EOF_
> $foo
> "$foo"
> '$foo'
> \$foo
> _EOF_
some text
"some text"
'some text'
$foo

ご覧のとおり、シェルは引用符に注意を払っていません。彼女はそれらを普通のキャラクターとして解釈します。そのおかげで、埋め込まれたドキュメントに自由に引用符を挿入できます。この状況は、レポートプログラムを開発するときに使用できます。

埋め込みドキュメントは、標準入力からデータを受け入れる任意のコマンドで使用できます。次の例では、組み込みドキュメントを使用して一連のコマンドをftpプログラムに送信し、リモートFTPサーバーからファイルをダウンロードします。

#!/bin/bash

#     FTP

FTP_SERVER=ftp.nl.debian.org
FTP_PATH=/debian/dists/stretch/main/installer-amd64/current/images/cdrom
REMOTE_FILE=debian-cd_info.tar.gz

ftp -n << _EOF_
open $FTP_SERVER
user anonymous me@linuxbox
cd $FTP_PATH
hash
get $REMOTE_FILE
bye
_EOF_
ls -l $REMOTE_FILE

リダイレクト演算子<<を<<-に置き換えると、コマンドシェルは埋め込みドキュメントの最初のタブ文字を無視します。これにより、埋め込みドキュメントにインデントを追加して読みやすくすることができます。

#!/bin/bash

#     FTP

FTP_SERVER=ftp.nl.debian.org
FTP_PATH=/debian/dists/stretch/main/installer-amd64/current/images/cdrom
REMOTE_FILE=debian-cd_info.tar.gz

ftp -n <<- _EOF_
        open $FTP_SERVER
        user anonymous me@linuxbox
        cd $FTP_PATH
        hash
        get $REMOTE_FILE
        bye
        _EOF_
ls -l $REMOTE_FILE

ただし、インデントのために多くのテキストエディター（およびプログラマー自身）がタブの代わりにスペース文字を使用することを好むので、この機能を使用することは必ずしも便利ではありません。
