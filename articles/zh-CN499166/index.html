, Open Source




FOSS技术已被实施了许多年的Oracle和Microsoft等行业领导者广泛使用，因为这些技术使组织更加高效，并为定制，适应和扩展提供了广泛的环境。 TechRepublic写道，对于像Amazon和IBM这样专注于使用开源作为创建强大的云解决方案的工具的大型公司来说，技术似乎仅限于大联盟，但是中小型企业也逐渐参与到这一过程中。 。许多人意识到开源使他们能够在大型技术公司的水平上竞争，并提供了这些解决方案所提供的灵活性，互操作性和成本节省。但是，如此庞大的参与者还没有遇到困难：寻找有能力的人员，选择合适的项目来使用，运营问题以及缺乏支持。



Open Source 50% 2019. 2020?




根据WhiteSource团队的研究报告，DevOps写道，开源产品中发现的漏洞数量增加的主要原因是此类产品本身的使用量增加。现在有比以往更多的开源项目，代码和社区成员。所有这些善良的人们不仅在通常在主要技术巨头的支持下努力编写更多的代码，而且还在寻找可能使组件用户面临风险的代码漏洞。更多编写的代码以及为这些不可避免的人为错误分析代码的目光的结合最终导致发现更多漏洞。开源组件中漏洞的数量增加将对软件开发产生重大影响。近年来，我们已经看到开源组件在开发人员创建应用程序的过程中扮演着越来越重要的角色。大多数估计表明，在大多数现代应用程序中，开源组件占代码库的60％至80％。当在诸如Apache Struts或Linux内核之类的流行项目中报告了漏洞时，大量开发人员突然面临更新其程序的需求。在大多数现代应用程序中，开源组件占代码库的60％至80％。当在诸如Apache Struts或Linux内核之类的流行项目中报告了漏洞时，大量开发人员突然面临更新其程序的需求。在大多数现代应用程序中，开源组件占代码库的60％至80％。当在诸如Apache Struts或Linux内核之类的流行项目中报告了漏洞时，大量开发人员突然面临更新其程序的需求。



Zoom? Jitsi Open Source




从会议和聚会到约会，我们现在都生活在视频会议应用程序中。但是在Zoom发生一系列隐私和安全漏洞之后，在Covid-19大流行期间，该漏洞几乎成为视频会议的代名词，许多组织和个人都想知道哪种服务最安全地进行我们的对话。埃米尔·伊沃夫（Emil Ivov）说，没有人需要被信任。 Ivov是Jitsi的文本和视频聊天开源软件的创建者，也是8x8的视频协作团队的负责人，该团队于2018年收购了Jitsi。该公司销售基于Jitsi代码的服务，但仍向开发人员付费以支持开源版本。 Jitsi Meet是一款具有便捷功能的视频会议应用程序，例如使用密码保护您的约会或将其他人排除在会议之外的功能。但是，它与大多数著名的视频会议服务的不同之处在于它是免费的，并且可以完全在您自己的设备上运行。





Python 2




Python 2死了吗？并非如此，但是从这次活动中，他正朝着计算机技术历史博物馆的荣誉地位迈出自信的一步。 4月20日，Python 2.7.18的最后最终版本被引入，这标志着对Python 2分支的完全停止支持。正如StackOverflow博客所说，此事件结束了一个时代。如果您还没有升级到版本3，那就该了。但是，第二版将继续通过各个公司的努力来维持生存，例如，Red Hat将在RHEL 6和7发行版的整个生命周期中继续使用Python 2.7维护软件包，对于RHEL 8，它将在Application Stream中生成软件包更新，直到2024年6月。 。如果这不是您的选择-欢迎您查看正式的过渡指南。但是可能要花很长时间，例如Dropbox迁移了3年。
