テスト方法


各サーバーは1週間テストされました。CPU、RAM、ディスクサブシステム、およびネットワークに負荷がかかりました。テストはスケジュールどおりに実行され、cronでホストされます。 

結果は、値を表にして、グラフおよび（または）チャートをプロットすることによって収集および処理されました。以下のツールを使用しました。

模擬テスト：

  • sysbench
  • CPU、一般的なテスト：合計時間のsysbench --test=cpu run (値:）
  • メモリ、一般テスト：（sysbench --test=memory run 値：合計時間）
  • ファイルi / o、テストおよびコマンド（すべてのテストのブロックサイズは4kです。値：転送速度）：

    • シミュレートされたキュー深度32のシングルスレッドシーケンシャルリード： sysbench --num-threads=1 --test=fileio --file-test-mode=seqrd --file-total-size=2G --file-block-size=4K --file-num=32
