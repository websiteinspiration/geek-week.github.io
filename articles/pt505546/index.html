Desenvolvimento de vibração e desktop

A idéia de criar aplicativos multiplataforma está longe de ser nova. O Flutter também oferece a capacidade de fazer isso. Neste artigo, tentarei descrever duas abordagens para iniciar um aplicativo móvel na área de trabalho, que eu mesmo uso para desenvolver aplicativos móveis. Parei de rodar o emulador e o simulador, principalmente porque havia uma oportunidade de ficar sem eles. Para aqueles que estão interessados ​​na idéia, bem-vindo ao gato.

Então, por que se preocupar em escrever uma versão para desktop ignorando emuladores e simuladores?

Primeiro, recursos da estação de trabalho . Meu projeto comercial atual usa muitas animações e vários widgets "pesados". No meu laptop muito bom com i7, 16 GB de RAM e uma placa gráfica GTX1050Ti, um processador com transições e animações muito complexas que salta para 30-35% no modo de depuração, o emulador de RAM para Android "consome" até 2 GB. Os quadros, é claro, são perdidos e as animações movem-se "espasmódicas", o que não nos permite avaliar adequadamente a integridade da imagem. Quando a versão para desktop é iniciada, meu processador "descansa" e é carregado com as mesmas animações de até 3-4% e isso, por um minuto, economiza o processador de 8 a 10 vezes e a RAM de 300 a 400 MB.

Em segundo lugar, a capacidade de visualizar rapidamente o posicionamento correto dos widgets em vários dispositivos. Basta pegar a janela e esticá-la ou apertá-la no tamanho mínimo, e isso economiza muito seu tempo pessoal e o tempo dos colegas testadores.

Plug-in de exemplo para emular tamanhos de janela do emulador


A maneira mais fácil é copiar a pasta Go do repositório de projeto de amostra, por exemplo, em github.com/go-flutter-desktop/examples/tree/master/text_demo

Em seguida, no diretório Go, crie as pastas plugins / desktop_utils e nele um arquivo com nosso futuro plugin, vamos chamá-lo main.go e escreva para ele

código a seguir: 
package desktop_utils

import (
	"github.com/go-flutter-desktop/go-flutter"
	"github.com/go-flutter-desktop/go-flutter/plugin"
	"github.com/go-gl/glfw/v3.3/glfw"
)

const channelName = "go-test/desktop_utils"

type DesktopUtilsFlutterPlugin struct {
	window *glfw.Window
	channel *plugin.MethodChannel
}

var _ flutter.Plugin = &DesktopUtilsFlutterPlugin{}
var _ flutter.PluginGLFW = &DesktopUtilsFlutterPlugin{}

func (p *DesktopUtilsFlutterPlugin) InitPlugin(messenger plugin.BinaryMessenger) error {
	channel := plugin.NewMethodChannel(messenger, channelName, plugin.StandardMethodCodec{})
	
	channel.HandleFunc("set_window_size", p.setWindowSizeHandler)

	return nil
}

func (p *DesktopUtilsFlutterPlugin) InitPluginGLFW(window *glfw.Window) error {
	p.window = window

	return nil
}

func (p *DesktopUtilsFlutterPlugin) setWindowSizeHandler(arguments interface{}) (reply interface{}, err error) {
	argumentsMap := arguments.(map[interface{}]interface{})

	var minW = int(argumentsMap["minW"].(int32))
	var minH = int(argumentsMap["minH"].(int32))
	var maxW = int(argumentsMap["maxW"].(int32))
	var maxH = int(argumentsMap["maxH"].(int32))

	p.window.SetSizeLimits(minW, minH, maxW, maxH);

	return nil, nil
}

channelName — Flutter
minW, minH, maxW, maxH — /

Inicializamos nosso arquivo como um módulo Go: go mod init desktop_utils (o arquivo go.mod aparecerá).

Vamos para o diretório Go, localizamos o arquivo go.mod, adicionamos o diretório local do nosso plugin: substitua desktop_utils => ./plugins/desktop_utils
e adicione nosso plugin ao options.go na linha de opções:

var options = []flutter.Option{
	flutter.WindowInitialDimensions(InitW, InitH),

	flutter.AddPlugin(&desktop_utils.DesktopUtilsFlutterPlugin{}),
}

Nesta parte da Go acabou.

A parte Flutter é a mais simples possível: precisamos criar apenas um canal (MethodChannel) com o nosso nome: go-test / desktop_utils e passar os parâmetros de comprimento e largura necessários:

const MethodChannel _utilsChannel = MethodChannel('go-test/desktop_utils');

_utilsChannel.invokeMethod(
  'set_window_size',
   <String, dynamic>{
     'minW': 400,
     'minH': 600,
     'maxW': 800,
     'maxH': 1200,
   },
 );

Isso é tudo. Nosso plugin está pronto.
