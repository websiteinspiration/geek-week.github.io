Exploring sealed classes in Java 15

We continue to explore new features that appear in Java. In past times, we examined in detail instanceofand records , and today will be the object of study sealedclasses are scheduled for release in the fifteenth version of Java.


The idea of ​​introducing "sealed" types into the language was first described in detail in February 2019 in the document "Data Classes and Sealed Types for Java" , and in July of that year received its JEP 360 , which was kindly translated into Habré by Oleg Chirukhin (olegchir) It is worth noting that the version of JEP'a that Oleg translated was a bit outdated and differs from the current one. In particular, in the new version, sealedtypes are no longer types, but classes and interfaces. This renaming is connected with the fact that the term “type” is too overloaded and can mean not only classes and interfaces, but also their derivatives like arrays ( Object[], String[]) and parameterization of generics ( List<Integer>, List<?>). You can read more about this renaming here .


However, the lyrics are pretty, and let's go directly to our sealedclasses. For programmers familiar with Scala and Kotlin, this kind of class should be familiar. In Java, it means exactly the same thing as there: the modifier sealedlimits the range of classes that can be inherited from this class. But in C # it sealedmeans something completely different . So if you came from C #, be careful: the analogue of the keyword sealedin Java is this final, not sealed.
