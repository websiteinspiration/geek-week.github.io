EXIF + PHPからのデータで写真を並べ替え

PHPのスクリプトを使用して写真を並べ替えた経験を共有したいと思い
ます。写真の数が少なく、悲惨なほど多いときがやってきます。

バックグラウンド


ある日、20年間に蓄積されたデジタル写真のアーカイブ全体を整理することに決めました。その間、435ギガバイトあたり112,000枚の写真が蓄積されていました。

さらに、それらのいくつかは多かれ少なかれ、たとえばデジタル一眼レフカメラからの写真、名前と日付を含むフォルダーによってソートされ、iphone / androidからインポートされた写真の他の部分は名前が付けられず、ソートされません。多くの場合、それは単なる巨大なフォルダーです10ギガバイト、中に数千のファイルがあり、削除して整理するのは残念です。

自動並べ替えツールを探し始めたところ、Picasaなどのすべての優れたサービスは既にGoogleによって購入および閉鎖されていることに気づきました。もちろん、Googleにすべてをアップロードできます。ただし、写真には問題があり、すべてが検索されず、一般に機能の半分が欠落しています。 Picasaにアクセスし、写真が認識されて使用されることにまだ不安がある場合は、Webにアップロードすることはまったくできません。

その結果、すべてを並べ替える小さなスクリプトを書くことになりましたが、最初はシェルスクリプトについて考えましたが、必要なことに気づき、EXIFは古き良きPHPに戻ることにしました。

タスク1-すべてのファイルを日付で展開する


最初に、最も簡単な方法で、すべてのファイルを取得し、作成日を確認して、ネストされたパスに沿って散布します。

$file_list = $files->getDirContents($config['photos.unsorted']);

foreach ($file_list as $key => $value) {
	moveImageFile($value);
}

function moveImageFile($filename) {
    $dt= new DateTime();
    $dt->setTimestamp(filectime($filename));
    $start_path = $this->config['photos'];

    $year = $start_path."\Year".$dt->format('Y');
    if (!is_dir($year)) mkdir($year);

    $month = $year."\\".$dt->format('Y-m-F');
    if (!is_dir($month)) mkdir($month);

    $path = $month."\\".$dt->format('Y-m-d');
    if (!is_dir($path)) mkdir($path);
}
$full_path = getUniqueFilename($filename, $path, $dt, 0);
copy($filename, $full_path);

いくつかの問題がありました：

  • 一部のファイルの作成日が正しくなかった
  • コピーすると、現在の日付で新しいファイルが作成されます。
  • ファイルは同じ作成時刻で重複する可能性があります

基本的に、PHPのライブラリセットには既にexif拡張が含まれているため、超自然現象は予見されませんでした

タスク№3-EXIFジオデータからの国、都市、地域


座標はファイル内で簡単に見つけることができ、GPSLongitudeとGPSLatitudeのExifにありますが、度、分、秒で格納されていることを忘れないでください。そのため、関数を使用して座標を10進数に変換する必要があります。

function getGps($exifCoord, $hemi) {
    $degrees = count($exifCoord) > 0 ? $this->gps2Num($exifCoord[0]) : 0;
    $minutes = count($exifCoord) > 1 ? $this->gps2Num($exifCoord[1]) : 0;
    $seconds = count($exifCoord) > 2 ? $this->gps2Num($exifCoord[2]) : 0;

    $flip = ($hemi == 'W' or $hemi == 'S') ? -1 : 1;
    return $flip * ($degrees + $minutes / 60 + $seconds / 3600);
}

何百万ものクエリでYandexを殺さないようにするために、データをMySqlにキャッシュし、座標を小数点以下3桁に四捨五入します。つまり、43.161-19.182で都市を特定するのに十分であるため、110,000枚の写真では1,500のジオメトリしか取得できませんでした。
