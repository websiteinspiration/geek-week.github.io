APIを試してLinkedInの検索制限を回避する

限定


LinkedInにはそのような制限があります- 商用利用制限。私のように、最近まで、彼に会ったことも、聞いたこともない可能性が高いです。



制限の本質は、連絡先の外の人の検索を頻繁に使用する場合（正確なメトリックが存在しない場合、アルゴリズムはアクションに基づいてそれを解決します-ユーザーを検索して追加した頻度）、検索結果は1000ではなく3つのプロファイルに制限されます（デフォルトは100ページ、1ページあたり10プロファイル）。制限は毎月の初めにリセットされます。当然、プレミアムアカウントにはそのような制限はありません。

しかし、少し前のことですが、あるペットプロジェクトで、LinkedInで検索を何度も試し始め、突然この制限が適用されました。当然、私はこれを商業目的で使用しなかったため、あまり好きではありませんでした。そのため、最初に考えたのは、制限を調べて回避することでした。

[ 重要な説明-記事の資料は、情報提供と教育のみを目的として提示されています。著者は、商業目的での使用を推奨していません。]

重要な更新-LinkedInはこの脆弱性を考慮し、最近修正しました。クロールとプラグインが機能しなくなりました。



私たちは問題を研究します


ページ付けされた10個のプロファイルの代わりに、検索では3つだけが返されます。その後、プレミアムアカウントの「推奨事項」を含むブロックが挿入され、下にぼやけてクリックできないプロファイルが表示されます。

すぐに、開発者のコ​​ンソールに手を差し伸べて、これらの非表示のプロファイルを確認します。おそらく、ブルースを入れるいくつかのスタイルを削除したり、マークアップのブロックから情報を抽出したりできます。ただし、予想どおり、これらのプロファイルは単なるスタブイメージであり、情報を格納しません。



では、[ネットワーク]タブを見て、3つのプロファイルしか返さない別の検索結果が実際に機能するかどうかを確認しましょう。興味のあるクエリ「/ api / search / blended」を見つけ、その答えを確認します。



プロファイルは `included`配列に含まれますが、その中にすでに15のエンティティがあります。この場合、最初の3つのエンティティは追加情報を持つオブジェクトであり、各オブジェクトには特定のプロファイルに関する情報が含まれています（たとえば、プロファイルはプレミアムです）。



次の12は実際のプロファイル-検索結果で、そのうち3つだけが表示されます。もうおわかりのように、追加の情報（最初の3つのオブジェクト）を受け取った人だけが表示されます。たとえば、制限なしでプロファイルから回答を取得すると、28個のエンティティが発生します-10個のオブジェクトとext。情報と18のプロファイル。

結論


したがって、最小限の労力で、弱点を見つけ、制限なしに検索を取り戻すことができました。クエリ自体を調べるには、データとそのパスを分析するのに十分でした。

これはLinkedInにとって脅威ではないため、深刻な問題であるとは言えません。最大はそのような「ラウンド」によるプレミアムの損失であり、プレミアムを支払うことはできません。おそらく、そのようなサーバーの応答は、サイトの他の部分が正しく機能するために必要です。あるいは、これは単に開発者の怠惰であり、うまく機能できないリソースの不足です。（この制限は2015年1月にこの制限がなくなる前に現れました）。
