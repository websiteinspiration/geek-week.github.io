フィッシングメールで送信された最近の悪意のあるdocファイルに遭遇しました。これがリバースエンジニアリングを実践し、Habrで新しい何かを書くのに十分な理由だと私は思いました。猫の下-悪意のあるマクロドロッパーと同様に悪意のあるdllを解析する例。

大きい


ファイルのSha256 - abb052d9b0660f90bcf5fc394db0e16d6edd29f41224f8053ed90a4b8ef1b519。 docファイル自体の最初のページに、このファイルが保護されていることを通知し、マクロを有効にする方法を説明する画像があります。ファイルには、番号の付いた2つの大きなテーブルがあります。数値は10進数で書かれ、最長は10桁で、正と負の両方があります。

今後は、最初のdllは64ビットであり、32ビットシステムでは実行できないと言います。したがって、マクロは正しいバイナリコードアーキテクチャを選択します。

次に、ペイロードが.datファイルから読み取られ、デコードされ（base64）、XORとCRC32を使用してURLから復号化され、復号化されたデータの最初の2バイトが「MZ」であることを確認します。 EXE

Python復号化コード
from binascii import crc32
from base64 import b64decode


def decrypt_payload(payload_b64: bytes, url: str):
    payload_bin = b64decode(payload_b64.decode())
    key = str(crc32(url.encode())).encode()
    decrypted = bytearray()
    for i, b in enumerate(payload_bin):
        decrypted.append(b ^ key[i % len(key)])
    return bytes(decrypted)

CreateProcessW 関数を使用して、保存されたファイルが起動されます。問題がなければ、cmd.exeプロセスはZwTerminateProcessを使用して終了します。何か問題が発生した場合（ネットワークの不足を除く）、すべてが再度繰り返され、最大3回の試行が行われ、datファイルとexeファイルの名前が1つずつ増加します。
