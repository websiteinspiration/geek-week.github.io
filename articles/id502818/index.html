Y messenger: messenger terdesentralisasi di blockchain *


* Ya, kami menggunakan blockchain, tetapi hanya untuk menyimpan informasi tentang pengguna, server, obrolan grup dan saluran. Blockchain tidak cocok untuk menyimpan sesuatu seperti korespondensi pribadi.

Saat ini, banyak pengguna di ponsel pintar telah menginstal beberapa messenger. Masing-masing dari mereka memiliki tugasnya sendiri. Di suatu tempat kita berbicara dengan teman-teman, di suatu tempat dengan saudara, di suatu tempat kita menyelesaikan masalah pekerjaan. Pertanyaannya adalah, mengapa Anda memutuskan untuk memercayai mereka masing-masing? Siapa yang memberi Anda jaminan bahwa pesan / dokumen / foto yang Anda kirim melalui messenger tertentu tidak akan tersedia untuk umum? Jika Anda memiliki komunitas di jejaring sosial, siapa yang memberi tahu Anda bahwa besok Anda tidak akan diambil dari grup ini dan Anda tidak akan kehilangan akses ke audiens Anda? Siapa yang memberi tahu Anda bahwa setelah penghapusan korespondensi, memang akan dihapus dan tidak akan muncul di arsip dalam beberapa tahun? Tetapi ada beberapa kasus.

Di sini kita datang dengan utusan Y kita. Kemungkinan besar Anda tidak tahu satu pun dari tim kami dan Anda tidak akan terbiasa dengan produk kami yang lain, tetapi saya meyakinkan Anda bahwa pada akhir artikel Anda akan mulai mempercayai kami.

Apa yang telah kita lakukan


Kami telah mengembangkan messenger baru. Kami tidak cukup senang dengan bagaimana Internet berkembang, dan kami melihat cara untuk mengubahnya. Anda dapat membaca lebih lanjut tentang motif kami di manifes kami ( tautan ). Singkatnya, tujuan super kami adalah membuat internet yang aman. Utusan adalah tahap pertama.

Utusan kami didesentralisasi. Siapa pun dapat memulai server mereka untuk teman, kolega, dan kenalan. Anda akan menjadi data pengguna Anda akan disimpan di server Anda. Data pribadi dienkripsi dan dikirim ke blockchain. Poin yang sangat penting adalah Anda akan menjadi bagian dari jaringan bersama. Anda dan pengguna Anda akan dapat berkomunikasi tidak hanya di dalam server Anda, tetapi juga dengan pengguna server lain, bergabung dengan obrolan dan saluran server lain. Semacam Internet internal, tetapi bekerja sesuai dengan aturan yang sedikit berbeda.

Utusan Y dibuat untuk aman. Data melewati 3 lapisan enkripsi. Enkripsi saluran komunikasi melalui SSL, enkripsi data asimetris dalam saluran dan enkripsi terminal simetris opsional untuk pesan Anda. Tidak seorang pun kecuali lawan bicara, bahkan admin server, dapat membaca pesan yang dienkripsi akhir. Itu berakhir dengan berbagai trik antarmuka seperti mengunci tangkapan layar ketika ada pesan yang dilindungi di layar. Sistem tanda tangan digital elektronik yang terintegrasi memungkinkan untuk menjamin kepengarangan dan memastikan pekerjaan pemungutan suara dan persetujuan dokumen.

Kami tidak mengontrol server. Semua server dibuat oleh perusahaan dan penggemar. Kami tidak memiliki akses ke server Anda dan tidak dapat mempengaruhi operasinya. Kami tidak tahu siapa dan mengapa menggunakan utusan kami. Sebenarnya, ini bukan urusan kami.

Kode sumber server, komponen enkripsi dan aplikasi akan terbuka. Protokol pertukaran jaringan akan terbuka. Jika Anda memahami C ++ / C # / Kotlin atau Bereaksi, Anda dapat berkenalan dengan produk kami dari dalam, pastikan kebenaran kata-kata kami dan bahkan membuat versi Anda sendiri.

Semua ini memungkinkan kita untuk mengatakan bahwa utusan kita lebih baik daripada banyak yang ada saat ini, terutama dalam konteks privasi dan keterbukaan.

Apa yang akan terjadi selanjutnya?


Utusan adalah titik awal, semoga, jauh. Tujuan global kami jauh lebih jauh daripada pembawa pesan, tetapi kami akan bergerak secara konsisten dan bertahap. Sekarang kami meluncurkan fondasi sistem kami, elemen dasarnya. Setelah itu, kami akan mulai bekerja secara otomatis menyeimbangkan jaringan untuk mengurangi lalu lintas dan beban server, kami akan menyediakan API dan SDK untuk pengembangan komponen server tambahan serta aplikasi mini yang tertanam.

Bagaimana kita menghasilkan ini


Ya, kami ingin mendapat penghasilan dari messenger ini. Saat ini, kami menginvestasikan dana kami sendiri dalam pengembangan dan dukungannya. Sekarang Anda dapat membantu kami dengan memberikan donasi dalam berbagai ukuran melalui halaman khusus.

Di masa depan, kami sedang mempertimbangkan kemungkinan memperkenalkan monetisasi secara berlangganan. Mungkin ini akan menjadi kuota untuk jumlah pengguna, atau mungkin berlangganan bulanan sederhana ke pembaruan server. Belum ada solusi final, kami sedang mempertimbangkan berbagai opsi.

Bagaimana cara mencoba?


Anda dapat menggunakan aplikasi server di hampir semua server virtual atau sesuatu yang lebih serius. Aplikasi server hadir sebagai gambar buruh pelabuhan. Instruksi tersedia di situs web kami https://ymessenger.org/en/user-guide/servers/deployment .
Aplikasi Android dapat diunduh dari Play Market ( Y messenger ). Anda dapat mengunduhnya dan mulai menggunakannya secara gratis.

Aplikasi web tersedia di sini . Versi web tidak mengizinkan otorisasi melalui QR dan tidak menampilkan file terenkripsi, polling, dan pesan suara. Fitur-fitur ini masih beroperasi.

Kami belum memiliki aplikasi iOS asli, kami berencana (yang kami bisa) melakukannya musim gugur ini. Saat ini, kami menawarkan pengguna iOS untuk menggunakan perakitan aplikasi web kami.

Sumber


Kami sedang mempersiapkan kode sumber untuk dipublikasikan di GitHub segera. Kami akan menerbitkan kode sumber aplikasi server, aplikasi Android, klien web dan komponen enkripsi di bawah GPL.
