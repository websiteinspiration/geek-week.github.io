モデムの作成：OFDMとGNU Radioを使用して無線でデジタルデータを送信する

こんにちは、ハブル。

このテキストは、モデム間の通信を確立する方法を扱った「ダイヤルアップモデムの音を分析する」という記事の続きと考えることができます。今日はさらに進んで、実際にデータが送信される方法を確認します。そのために、OFDMとGNU Radioを使用して簡単なモデム実装を作成します。

文字通りの意味でデータを空中で送信します-送受信にはスピーカーとマイクで十分です。

これがどのように機能するかに興味がある人のために、カットの下で続けられました。

したがって、私たちの仕事は、最も単純なモデムがポイントAからポイントBにデータを送信できるようにすることです。デジタル信号を変調する方法はたくさんあります。現代の通信システムで広く使用されている方法であるOFDMを使用します。OFDMでは、フーリエ変換デジタル信号がいくつかの並列サブキャリアに変換されるため、伝送チャネルの高速と効率が保証されます。OFDMは、デジタルテレビやラジオからLTEまで、さまざまな場所で使用されています。私たちのオーディオチャネルは、パラメータがはるかに狭く、適度なものになっているため、速度は遅くなりますが、基本的に、本質は変わりません。厳密には、OFDMは無線でデータを送信するようには設計されていませんが、距離が短い場合は、従来のマイクとスピーカーで十分です。

プレーンテキストファイルがデータソースとして使用され（私の場合、「Hello Habr !!」という文字列が含まれています）、データはブロックに分割され、OFDMトランスミッターに送られます。OFDMはかなり複雑なタイプの変調です。OFDMエンコーダーの設定では、あまり明確ではない多くのパラメーターを指定する必要があります。



値のより詳細な分析はここにあります。もちろん、OFDM Receiverブロックのパラメーターは、送信パラメーターと一致する必要があります。Channel Model

ブロックは、通信チャネル、QT GUI Frequency Sinkブロックのシミュレーションに使用されますスペクトルを視覚化するために使用されます。低いサンプリング周波数（サンプルレート= 2400）にも注意を払う価値があります。これは、オーディオチャネルでの送信に適したスペクトルの小さな帯域を信号が占有するように特別に行われます。

残念ながら、GNU Radioで受信データを表示するための組み込みユニットが見つからなかったため、UDPを使用する必要がありました。UDP Sinkブロックはデータの出力に使用され、Pythonプログラムはデータの受信に使用されます。

import socket

UDP_IP = "127.0.0.1"
UDP_PORT = 40868

sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM) # UDP
sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
sock.bind((UDP_IP, UDP_PORT))
sock.settimeout(0.5)

while True:
    try:
        data, addr = sock.recvfrom(128) 
        print("Msg:", data, 'Hex:', ' '.join('{:02x}'.format(x) for x in data))
    except socket.timeout:
        pass

これにより、受信したデータをコンソールに表示でき、非常に便利です。おそらく、GNU Radioに出力する簡単な方法があるでしょう。

放映する


上記の例では、すべてが非常に単純です。しかし、1つだけ「しかし」あります。GNUラジオは、いわゆる「複雑な信号」で動作します。これは、実際には、位相が90度ずれた数値のペアです。これはコンピュータ処理には便利ですが、複素数を取得してブロードキャストすることはできません。

また、2つのオプションがあります。 SDRトランスミッター（HackRF、LimeSDR、USRPなど）の所有者はこれ以上読むことはできませんが、GNU Radioの既製のブロックを使用するだけで、すべてが「自動的に」行われます。しかしそれ以来サウンドカードを使用して信号を送信します。変換は自分で行う必要があります。そのために、DSPの記事にあるフローチャートを使用します。



ここでOFDMは送信機、Fcキャリア周波数です。信号がスピーカーとマイクの最大感度に収まるように、4KHzに設定します。転送のグラフのフローチャートもそれに応じて複雑です



。テストの便宜上、WAVファイルシンクを使用して、保存したファイルを個別に記録および再生できるようにします。ファイルを開いてそのスペクトルを確認することもできます。中心周波数が本当に4KHzであることを確認できます。



受信


そして最後に、最後のステップ：信号を受信するためのブロック図を作成します。ここでは、実際にはすべてが同じですが、逆の順序でのみです。



元の信号に位相シフトされた参照信号を乗算し、LPFの助けを借りて高周波成分をカットオフします。出力は、さらに処理するために使用できる複雑な信号です。
