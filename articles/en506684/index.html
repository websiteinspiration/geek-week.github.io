How I transferred dedic.ru to github pages and was satisfied

Once upon a time I was quite closely involved in the organization of various kinds of hosting. In technical terms, of course - setting up servers, configuring control panels and other related things.

Then I had the idea to write down various decisions and clever thoughts on the site in order to popularize this topic, since at that time there was nothing about hosting hosting in RuNet. And he called it dedic.ru.

For the site engine, for some reason I chose Drupal rather than Wordpress, later I couldn‚Äôt explain this choice to myself. Then it seemed to me that Drupal is more flexible, convenient and logical.
My first disappointment was at the time of the release of the sixth version of Drupal. I simply could not transfer my current project there - constantly something fell off, then a taxonomy, then a forum, then something else.

As a result, I had to leave dedic.ru in read-only mode and try to continue what I started on other sites. But by that time, quite a lot of sites on this topic had already appeared and my attempts did not take off. Yes, and I myself began to change interests - clouds, security and automation.

And then one evening I thought - and how actually can a site be mothballed in such a way as not to pay for its content?

The solution was found quite quickly - put the site in github pages. This is just a service for publishing static sites, and it‚Äôs completely free.

In order to use it, just create a public repository and change the branch from master to gh-pages. After that, the contents of the repository can be processed as content for the web server and watch it in the browser.

However, the question arose - how to actually transfer content to the repository? Hands? - Stupid!

The solution was found quickly enough: There are two great plugins for Wordpress - FG Drupal to WordPress and WP2Static. The first makes it easy to import content from Drupal into Wordpress through a query to the Drupal database. And the second is to publish a static site from Wordpress to various repositories, including github pages.

Of course, the import did not transfer the taxonomy and the forum, but since I didn‚Äôt think about developing the site further, I decided to simply score some of the content for loss, and I redid the structure manually - since there are not many entries there.

Having exported the site to github pages, I attached the original dedic.ru domain there and was almost satisfied. Two things I didn‚Äôt have were https and some kind of interaction for visitors.

For https, I easily adapted CloudFlare, specifying as the CNAME the internal address of the site on github pages, turning on proxying and specifying the TLS / SSL method as flexible.

For interaction, I didn‚Äôt come up with anything smarter than to put a standard donat from paypal and add the ability for donors to replace the contents of the block with their own via pull-request. Perhaps it would be necessary to add the commenting option through the Disqus service, but to be honest - I was already too lazy to do this further.

I hope this information will be useful to someone, and my content once benefited from setting up servers.
