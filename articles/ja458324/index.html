材料の概要


通常、私は、特定のフレームワークの学習を始めたばかりの人のために、同じ問題を解決するためにあまりにも多くの方法について話さないようにします。しかし、この場合、最初のVueアプリケーションを開発するためのいくつかの方法を示すことが非常に重要だと思います。おそらく、コマンドラインや、グラフィカルインターフェースを備えたツールで作業するのが好きでしょう。どちらもVue開発に適しています。さらに、Vueアプリケーションはコマンドラインに頼ることなく作成できます。これを行うには、HTMLページにタグを追加して、対応するスクリプトをロードします。そして最後に、開発環境のセットアップについて心配する必要はありません。代わりに、不必要な動きをせずに、できるだけ早くVueを試すように努めます。実際、どのような戦略をとっても、ここではVueを使い始めるのに役立ちます。

UeVueは進歩的なフレームワークです


Vue.jsのホームページに アクセスすると、そのページが「Progressive JavaScriptフレームワーク」と呼ばれていることがわかります。一見、これは通常のマーケティング策のように思えるかもしれませんが、Vueの本質を少し理解すれば、これはそうではないことを理解できます。まず、「プログレッシブ」という言葉の意味を調べましょう。それは、徐々にまたは段階的に起こっている、または発展しているものです。それは段階的に行われるものです。

すぐに、Vueアプリケーションを開発するときに、このフレームワークの機能を徐々に導入して作業できることがわかります。たとえば、すでに特定のプロジェクトがある場合、問題なくVueサポートを装備できます。まず、Vueスクリプトをロードし、数行のコードを記述します。

Vue CLI、Vueコマンドラインツールを使用して、必要な機能を備えた新しいプロジェクトをデプロイする場合も、これは問題ではありません。アプリケーションが成長し、ルーティングや状態管理などのかなり高度な機能が必要になると、アプリケーションはこれらの機能を備えることが難しくなります。

すでに述べたように、Vueは手頃な価格のフレームワークです。HTML、CSS、JavaScriptをすでに知っている場合は、Vueを使い始める準備ができていることを意味します。実際、もしあなたが本当にVueアプリケーションを書き始めたいのなら、私はそうすることを提案します。

オプション＃1：ページに接続されたVueスクリプト


私たちが探求しているVueを使用する最初の方法は、タグを使用してVueスクリプトをページに接続すること<script>です。 JavaScriptをすでに使用している場合は、何も新しいことはありません。つまり、一部のJSライブラリに実装されているいくつかの機能を装備する必要があるHTMLページがあるということです。これを行うには、タグを使用してライブラリをページに接続します<script>。ここで検討されているVueの使用方法は、トレーニングアプリケーションの開発に適しているだけではないことに注意してください。実際のプロジェクトでアプリケーションを見つけます。

プログラマが働いている組織などで、実際のプロジェクトに新しいツール、ライブラリ、フレームワークを導入することは通常非常に困難です。たとえば、Vueでの作業を本当に楽しんでいます。当局に行って、プロジェクト全体をVueに書き直すことができるかどうか尋ねます。そのような質問に答えて、ほとんどのオプションはすぐに拒否されます。現在開発中の特定のページでVueを使用することを提案した場合、そのような提案は受け入れられる可能性があります。

次の例では、非常にプリミティブなアプリケーションを開発します。これは、Vueの概念を検討するのに非常に便利であるため、優れています。

これで、最初のVue.jsアプリケーションを作成したことを祝福できます。index.htmlブラウザで開くと「Hello、Vue！」が大きな文字で書かれたページが表示されるはずです。この碑文の下には、現在の日付と時刻を表示する行があります。このバージョンのサンプルの開発と調査中に、Visual Studio CodeとLive Server拡張機能を使用しました。

ここで行ったことは、特に高度なプロジェクトのようには見えません。しかし、Vueの内部では、このプロジェクトが私たちが必要とする方法で機能することを可能にする多くのことが起こっています。フレームワークはDOMとデータ間の接続を確立し、その後アプリケーションは反応しました。これをどのようにチェックしますか？非常に簡単-JSコンソール（アプリケーションページの右側）を開いて、何かを書くだけですapp.message。これにより、ページに表示されるラベルが変更されます。以下はそれがどのように見えるかです。




Vueアプリケーションを調べることで、開発者はコンポーネントのライフサイクルイベントのハンドラーを作成することもできます。これはcreated、mountedやのようなイベントをリッスンできることを意味しdestroyedます。イベントが発生すると、いくつかのコードの実行を整理できます。この資料の主な目的は、Vueアプリケーションを作成するためのさまざまな方法を示すことであり、フレームワークの複雑さをすべて分解することではないため、詳細には触れません。ただし、今でもコンポーネントのライフサイクルイベントの処理がどのように見えるかを確認し、最初のアプリケーションでの作業を終了します。

次のコードのおかげで、変数の値はnow毎秒更新されます。 Vueのマウント中に、使用setInterval、日付と時刻を更新する関数の2番目の呼び出しを構成します。オブジェクトに適切なメソッドを追加することで、ライフサイクルイベントハンドラを作成できますmethods。これは、オブジェクトの操作方法とよく似ていますdata。これがコードです：

<script>
  const app = new Vue({
    el: '#app',
    data: {
      message: 'Hello Vue!',
      now: new Date()
    },
    methods: {
      updateDate() {
        this.now = new Date();
      }
    },
    mounted() {
      setInterval(() => {
        this.updateDate();
      }, 1000);
    }
  })
</script>

このページをブラウザで開くと、外観は変わりませんが、表示される時刻は毎秒更新されます。ただし、DOMを手動で更新する必要はありません。DOMにリンクされたVueデータ。データが変更されると、ページに表示される内容も変更されます。

オプション＃2：Vue CLI


Vueスクリプトをページに接続するだけでアプリケーションを作成しましたが、このアプローチはあまりスケーラブルではありません。本格的な単一ページアプリケーション（単一ページアプリケーション、SPA）を開発する必要があり、同時にVue補助ツールを使用したい場合は、Vue CLIを使用するのが最善です。

コマンドラインでの作業があまり好きではない方は、グラフィカルインターフェイス（Vue UI）を備えた同様の機能を持つツールを使用できます。

まず、Vue CLIをインストールする必要があります。これを行う前に、Node.jsとnpmを取得する必要があります。以前にNode.jsとnpmを使用したことがない場合は、時間をかけてそれらを理解し、少し理解してください。Vue CLIをインストールできるコマンドは次のとおりです。

npm install -g @vue/cli

コマンドを使用して、新しいアプリケーションを作成できますvue create。コマンドをvue使用して、サポートされているコマンドのリストを見つけることができますvue -h。
