Astra Linux "Eagle" Common Edition：Windowsの後の人生はありますか

OSのユーザーの1人から詳細なレビューを受け取りました。これを共有します。

Astra Linuxは、オープンソースソフトウェアに切り替えるためのロシアのイニシアチブの一部として作成されたDebian派生物です。Astra Linuxにはいくつかのバージョンがあり、そのうちの1つは一般的な日常使用を目的としています-Astra Linux "Eagle" Common Edition。誰にとってもロシアのOSは本質的に興味深いものであり、私は毎日3つのオペレーティングシステム（Windows 10、Mac OS High SierraおよびFedora）を使用し、過去13年間Ubuntuに忠実である人の観点からOrelについて話したいと思います。この経験に基づいて、インストール、インターフェース、ソフトウェア、開発者向けの基本機能、さまざまな角度からの利便性の観点からシステムを検討します。Astra Linuxは、より一般的なシステムとどのように比較されますか？そして、彼女は自宅でWindowsを置き換えることができますか？



アストラリナックス


Astra Linuxインストーラーは、Debianインストーラーと多くの共通点があります。ほとんどのパラメータはデフォルトで固定されているため、おそらく最初のものはさらに単純です。それはすべて、高すぎない高層ビルを背景にした一般的なライセンス契約から始まります。おそらく、オレルでも。



インストールの重要なポイントは、ソフトウェアの選択です。これは、システムにデフォルトで付属しています。利用可能なオプションは、標準的なオフィスと仕事のニーズをカバーします（非開発者向け）。



また、最後のウィンドウは、インタープリターのブロック、コンソール、トレース、実行ビットの設定など、追加の設定セットです。これらの単語で何もわからない場合は、どこにもチェックマークを付けないようにしてください。さらに、これらすべては、必要に応じて、後で構成できます。



システムは、適度なリソースを備えた仮想環境内にインストールされました（比較的最近のシステム）。速度とパフォーマンスに関する不満はありませんでした。テストした構成を以下に示します。



インストール手順は通常です：iso-imageをマウントし、システムの標準インストールプロセスを介してインストールし、GRUBブートローダーを書き込みます。



ロード時のシステムはリソースに要求されない-デスクトップモードの起動時に約250〜300 MBのRAM。



代替起動オプション：タブレットおよび電話モード


システムに入るとき、いくつかの起動オプションの1つを選択できます：安全、デスクトップ、モバイルまたはタブレット。



タッチデバイスで作業するには、画面キーボードをオンにします。



さまざまなモードで何が面白いか見てみましょう。デスクトップは、システムがWindowsに似ている通常モードです。



タブレットモードは、大きなタッチスクリーンに適しています。以下のスクリーンショットで確認できる明らかな外部の違いに加えて、インターフェースには他の機能があります。タブレットモードのカーソルは表示されず、アプリケーションの閉じるボタンがタスクバーに配置されます。フルスクリーンアプリケーションの動作は少し異なり、ファイルマネージャーのファイルも異なる方法で選択されます。



モバイルモードについて言及する価値があります。ここではすべてがAndroidとほぼ同じです。Flyグラフィカル環境が使用されます。タッチモードでは、長いタッチが機能し、コンテキストメニューを呼び出すことができます。モバイルモードは、デスクトップやタブレットと比較してわずかに多くのリソースを消費します。





異なる動作モードの存在は便利です。たとえば、プラグインキーボードを備えたタブレットを使用している場合、それに応じてタッチと非タッチの使用例があります。

システムアップデート


システムの使用を開始する前に、システムを更新する必要があります。基本的に、Astra Linux リポジトリには14千のパッケージ（stable、test、および実験ブランチ）があります。試験的ブランチは間もなく不安定な更新を受け取るため、テスト用ブランチをテストします。リポジトリをテストに変更します。



リポジトリの更新とシステムの更新を開始します。これを行うには、左上の[更新]ボタンをクリックし、[すべての更新をマーク]、[適用]の順にクリックします。リブート。

ユーザーポリシー


新しいユーザーは、セキュリティポリシー管理ユーティリティを介してシステムに作成されます。



デフォルトでは、リモートログイン機能が提供されています（コントロールパネル-システム-ログイン）。



通常の個別のリモートセッションに加えて、ネストされたセッションを開始できます（開始-シャットダウン-セッション）。



最初の2つでは、すべてが明確です。ネストされたセッションは、現在のセッションのウィンドウで開始されるセッションです。



ちなみに、セッションは遅延時間の後に完了することができます。長い操作の終了を待たずに、自動シャットダウンを構成するだけです。



Astra Linuxインターフェイスと標準ソフトウェア


Astra Linux Common Editionは、数年前のDebianに似ています。外見上、Astra Linux Common EditionがWindowsに近づこうとしていることは注目に値します。



ファイルシステムのナビゲートと操作は、LinuxよりもWindowsに近いです。システムイメージには、オフィス、ネットワーク、グラフィック、音楽、ビデオなどの標準ソフトウェアセットが付属しています。システム設定もメインメニューにグループ化されています。デフォルトでは4つの画面が使用可能です。


ご覧のように、LibreOfficeはオフィススイートとしてシステムにインストールされており、

コントロールパネルはWindows / Mac /などに似ており、基本設定を1か所にまとめています。



ファイルマネージャには2つのパネルのインターフェースがあり、アーカイブをフォルダとしてマウントできます。





ファイルマネージャーは、GOST R 34.11-2012に準拠したものを含むチェックサムを計算できます。



Mozilla Firefoxが標準ブラウザとしてインストールされます。それはかなり禁欲的に見えますが、同時に非常に適切です。例として、新しいHabrを開いて調べました。ページがレンダリングされ、システムはクラッシュせず、ハングしません。



次のテストはグラフィック編集です。 Habrの記事のタイトルから画像をダウンロードし、GIMPで開くようにシステムに要求しました。ここでも珍しいことはありません。



そして、手首を振るだけで、記事の1つに対する効率のテストを追加します。原則として、標準のLinuxシステムとの違いはありません。



単純なスクリプトを超えて、apt-getを使用して標準パッケージを配置してみましょう。 



インデックスの更新後：
sudo apt-get update

テストでは、python3-pip、zshをインストールし、oh-my-zshをインストールしました（追加のgit依存関係を使用）。システムは正常に動作しました。

ご覧のとおり、システムは平均的なユーザーの標準的な日常のシナリオのフレームワークでうまく機能します。ここでDebian / Ubuntuに馴染みのあるプログラムが表示されると予想される場合は、ハンドルを使用してそれらを追加でインストールする必要があります（たとえば、ack-grepなどのパッケージが必要な場合-それらはcurl / sh経由でインストールされます）。sources.listにリポジトリを追加して、使い慣れたapt-getを使用できます。

ネイティブAstra Linuxユーティリティ


上記のツールは、Astra Linuxユーザーが利用できるもののほんの一部です。さらに、開発者は、システムの更新に使用したのと同じリポジトリを介してインストールできる約100の追加ユーティリティを作成しました。 



ユーティリティを見つけるには、「fly」という単語を検索します。必要なすべてのユーティリティには、このようなプレフィックスが付いています。


 
1つのレビューの枠組みの中ですべてのアプリケーションについて話すのは難しいので、単純なユーザーの観点からいくつかの有用なものを選択します。天気アプリケーションは、ロシアの選択された都市の予測を表示します。これは、ロシア地域向けに最適化されています。



ファイルを検索するためのいくつかのフィルターと設定を備えたシンプルなグラフィカルユーティリティもあります。



バッテリーの充電とさまざまなモードを監視するための独自のユーティリティがあり、その移行はタイマーを介して設定されます-モニターの電源を切る、スリープ、休止状態。



コマンドの実行可能ファイルの選択もグラフィカルシェルでラップされました。たとえば、コマンドの実行時にシステムが選択する「vi」を指定できます。



別の管理ユーティリティを使用すると、システムの起動時に起動するアプリケーションを構成できます。



GPS / GLONASSモニタリングもあり、電話/タブレット（対応するモジュールが通常存在する）でかなり便利です。

主要なシステムと対比


インターフェイスとコントロールのロジックの観点から見ると、システムは従来のWindows XPに似ています。時にはMac OSの個々の要素です。

ユーティリティ、コンソール、ハードウェアの観点から見ると、システムは従来のDebianに似ています。これは、UbuntuとMintedの同じユーザーにはかなりよく馴染んでいますが、最も高度なものは、すべてのリポジトリの通常のパッケージの範囲を見逃します。

潜在的なユーザーのポートレートに経験を課すなら、私は新しいシステムに前向きな期待を持っています。Windows / Macでの経験に基づいて、一般ユーザーはAstra Linux Common Editionに簡単に慣れることができます。そして、標準的なunix-utilitiesを使用するより上級のLinuxユーザーは、自分に合ったものをすべて構成します。

Astra Linuxの現在のバージョンはDebian 9.4に基づいており、Debian 10（4.19）の最新のカーネルも利用できます。 

もちろん、Ubuntuの新しいバージョンがありますが、1つだけ小さいが重要なニュアンスがあります。それらはLTS（長期サポート）ではありません。Ubuntu LTSバージョンは、Astra Linuxのパッケージバージョンと同じです。W ikipediaからAstra Linux（OSバージョンのリリース日を追跡しやすくするためにAstra Linuxスペシャルエディションによって認定されている）のデータを、UTSのLTSバージョンのリリース日と比較しました。
