メソッドとフィールドを実装する

最初の部分では、コンストラクターレベルで依存関係を実装する方法について説明します。さらに、daggerはフィールドとメソッドの依存関係を実装できます。ただし、これらの実装は、絶対に必要な場合に使用する必要があります。

メソッド実装例：

class Car @Inject constructor(private var engine: Engine){
    var key: Key? = null
        @Inject set
}

class Key @Inject constructor()

上記の例では、キーフィールドのsetメソッドの依存関係が

導入されています。フィールドは3つの段階で実装されます。

  1. embedメソッドを抽象ファクトリに追加する
  2. 実装するフィールドを定義する
  3. 依存関係を実装するために、抽象クラスのダガー実装で埋め込みメソッドを使用する

これを視覚的に理解すると、

1の順になります。抽象クラスで、MainActivityの実装メソッドを追加します。

@Component
interface DaggerComponent {
    fun getCar(): Car
    fun getEngine(): Engine
    fun getFuel(): Fuel

    fun inject(act: MainActivity)
}

2. MainActivityで実装する必要があるフィールドを定義します。挿入されたフィールドはlateinit varであり、誰でも見ることができるはずです（パブリック）

@Injected 
lateinit var car: Car

3.抽象クラスのinject（）メソッドを呼び出して、追加したアクティビティフィールドを実装します。

最終的に、MainActivityクラスはトレースのようになります。仕方：

class MainActivity : AppCompatActivity() {
    @Inject
    lateinit var car: Car

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        DaggerDaggerComponent.create().inject(this)
    }
}

短剣での初期化の遅延

ご存知のように、アプリケーションを起動するとき、すべてのクラスのインスタンスが必ずしも必要ではありません。これにより、初回の起動などがスピードアップします。daggerに実装されたオブジェクトの初期化には2つのタイプがあります：プロバイダー<>およびレイジー<>

プロバイダー -オブジェクトに初めてアクセスすると初期化が行われ、呼び出されるたびに
レイジーオブジェクトの新しいインスタンスが返されます-初めてアクセスすると初期化が行われ、次に以前にキャッシュされたインスタンスが返されます

データを適用するには初期化のタイプでは、初期化されたオブジェクトを目的の形式で「ラップ」する必要があります。

プロバイダーの使用例：

class Engine @Inject constructor(private var fuel: Fuel){
    fun start(){
        if(fuel!=null){
            print("Started!")
        }else{
            print("No more fuel!")
        }
    }
}

class Car @Inject constructor(private var engine: Provider<Engine>){
    var key: Key? = null
        @Inject set

    fun startCar(){
        engine.get().start()
    }
}

class Key @Inject constructor()

get（）メソッドが呼び出されるたびに、目的のオブジェクトの新しいインスタンスを取得します。

Lazyの使用例：

class Fuel @Inject constructor() {
    val fuelType = if(BuildConfig.DEBUG){
        "benzine"
    }else{
        "diesel"
    }
}

class Engine @Inject constructor(private var fuel: Lazy<Fuel>){
    fun start(){
        if(fuel!=null){
            print("Started with ${fuel.get().fuelType}")
        }else{
            print("No more fuel!")
        }
    }
}

get（）メソッドが呼び出されるたびに、同じインスタンスを取得します。

import dagger.Lazy

短剣モジュール。短剣が

あなたを理解していない場合短剣があなたの意図を理解していない場合があります。たとえば、CarクラスにはIvanovクラスによって継承されるタイプ（インターフェイス）ドライバー

のフィールドがあります。インターフェイスタイプでフィールドを実装しようとすると、「@アノテーション付きメソッドなしでは提供できません」というエラーが発生します。

この問題を解決するために、ダガーはモジュールの使用を提案しています。モジュールは、ダガーにそれ自体では取得できない追加情報を提供します。モジュールとして、インターフェースまたはオブジェクト（オブジェクト）を使用できます。

上記の問題を解決するには、モジュールを作成します。

@Module
interface DaggerModul {
    @Binds
    fun bindDriver(driver: Ivanov): Driver
}

class Ivanov @Inject constructor(): Driver

bindDriverメソッドでは、インターフェースを初期化する方法を説明します。

また、コンポーネントでは、すべての既存の短剣モジュールをリストする必要があります

@Component(modules = [DaggerModul::class])
interface DaggerComponent {
…
}

サードパーティのライブラリフィールド（インターフェース）がEngineクラスに使用されているとします。ランタイムで初期化されるクラスが明確でない場合、ダガーのそのようなフィールドをどのように説明しますか

これまで、依存関係をどのように注入するかを説明するために注釈を使用してきました。たとえば、外部ライブラリからクラスを作成する方法がわからない場合はどうなりますか？

注釈提供する 初期化するクラスのインスタンスを明示的に記述する必要がある場合について説明します。 

@Module
object DaggerModuleObject {
    @Provides
    @JvmStatic
    fun getBoschCylinder(): Cylinder = BoschCylinder()
}

したがって、シリンダーフィールドを初期化する場合、BoschCylinderクラスのインスタンスが必要であることをダガーに伝えます。

注釈名前付き。同じタイプの複数のインスタンス

異なる設定で同じクラスのインスタンスを作成する必要がある場合があります。この例では、これらはボディとドアで異なる色です。

トレースを使用してプロジェクトをビルドしようとした場合。モジュールは「（私たちのクラス）色が複数回バインドされている」というエラーを受け取ります

@Provides
@JvmStatic
fun getColorRed():Color =  Color("red")

@Provides
@JvmStatic
fun getColorBlue():Color =  Color("blue")

そのような場合を解決するために要約が使用されます。 名前付き。まず、モジュールで、短剣で初期化するための3つの新しいメソッドを作成します

@JvmStatic
@Provides
fun getColor(): Color = Color("")

@Provides
@Named("blueColor")
@JvmStatic
fun getColorBlue(): Color{
    return Color("blue")
}

@JvmStatic
@Named("redColor")
@Provides
fun getColorRed(): Color = Color("red")

最初のメソッドはデフォルトです。それがないと、ダガーはクラスが存在しないことを誓います。「Injectコンストラクターまたは提供アノテーション付きメソッドなしでは提供できません」

次の2つのメソッドは、同じクラスのインスタンスを返します。このクラスの紹介を適切な場所に追加し、注釈を付けて呼び出す必要があります名前付き
