使用Xdebug和PhpStorm调试PHP容器

在Backend PHP Developer课程开始之前，准备了本文的翻译。









Docker指令＃9：xdebug



我将创建一个非常简单的php页面，并使用xdebug和PhpStorm运行它。



可以在以下位置找到源文件：

github - com / ikknd / docker - study在recipe-09文件夹中



1.在“ docker”文件夹中创建一个“ Dockerfile”： 



FROM php:7.2-fpm

#Install xdebug
RUN pecl install xdebug-2.6.1 && docker-php-ext-enable xdebug

CMD ["php-fpm"]


从docker文件夹运行以下命令以创建映像： 



docker build -t php-xdebug-custom -f Dockerfile .


2.在docker文件夹中创建docker-compose.yml文件： 



version: "3.7"

services:

  web:
    image: nginx:1.17
    ports:
      - 80:80
    volumes:
      - /var/www/docker-study.loc/recipe-09/php:/var/www/myapp
      - /var/www/docker-study.loc/recipe-09/docker/site.conf:/etc/nginx/conf.d/site.conf
    depends_on:
      - php

  php:
    image: php-xdebug-custom
    volumes:
      - /var/www/docker-study.loc/recipe-09/php:/var/www/myapp
      - /var/www/docker-study.loc/recipe-09/docker/php.ini:/usr/local/etc/php/php.ini


在这里，我使用php-xdebug-custom图像而不是php：7.2-fpm„



3.在php.ini文件中进行以下设置： 



[xdebug]
zend_extension=xdebug.so
xdebug.profiler_enable=1
xdebug.remote_enable=1
xdebug.remote_handler=dbgp
xdebug.remote_mode=req
xdebug.remote_host=host.docker.internal
xdebug.remote_port=9000
xdebug.remote_autostart=1
xdebug.remote_connect_back=1
xdebug.idekey=PHPSTORM


4.在PhpStorm中配置服务器：



文件->设置->语言和框架-> PHP->

服务器使用+图标添加新服务器并按以下屏幕截图所示进行配置：







确保选择“使用路径映射”（ “使用路径映射”），并将php文件夹映射到/var/www/myapp



。



5.在PhpStorm中配置PHP远程调试器：



运行->编辑配置-> PHP远程调试



添加新配置并为其分配值，如以下屏幕截图所示：







6.在PhpStorm调试面板中选择调试配置







7.转到/var/www/docker-study.loc/recipe-09/docker/



并运行： 



docker-compose up -d


如果现在输入myapp.loc/



浏览器，我将从文件中看到结果index.php



。



我可以设置一个断点，开始在PhpStorm调试面板中侦听连接，然后重新加载页面。



有一个很好的调试！
