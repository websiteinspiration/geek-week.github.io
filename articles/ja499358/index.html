注釈


このPEPは、Python docstringの使用に関連するセマンティクスと規則を文書化しています。

正当化


このPEPの目的は、ドキュメント文字列の高レベルの構造を標準化することです。つまり、ドキュメント文字列に含まれるべき内容を正確に説明し、説明します（実際のマークアップ構文については説明しません）。PEPには厳密なガイドラインは含まれていませんが、推奨事項は次のとおりです。
「従来の慣例は、明快さ、一貫性、メンテナンスの容易さを提供し、優れたプログラミング習慣を育みます。しかし、彼らはあなたにあなたの意に反して行動することを強制しません。これはPythonです！」

2001年6月16日comp.lang.pythonのTim Peters

一般的に受け入れられている契約を回避すると、最悪の場合、目を細めることになります。ただし、契約について知っていて、それに従うことでより良い結果を得ることができるアプリケーション（たとえば、Docutilsなどのドキュメントシステム）があります。

仕様


Docstringとは何ですか？


説明文字列は、モジュール、関数、クラス、またはメソッド定義の最初のステートメントである文字列リテラルです。このような文字列は、このオブジェクトの特別な__doc__属性を処理するときに使用可能になります。

すべてのライブラリ、およびそれらによってエクスポートされる関数とクラスには、docstringが必要です。パブリックメソッド（__ init__コンストラクターを含む）にもドキュメントが必要です。パッケージ自体は、対応するディレクトリにある__init__.pyファイル内に文書化できます。

コードの他の場所にある文字列リテラルも、ドキュメントの役割を果たすことができます。これらはPythonバイトコードコンパイラでは認識されず、実行時にオブジェクト属性として使用できません（つまり、__ doc__の情報が不足しています）。ただし、他のソフトウェアツールを使用して取得されるドキュメント行には、さらに2つのタイプがあります。

  1. モジュール、クラス、または__init__レベルでの単純な割り当ての直後に発生する文字列リテラルは、「属性ドキュメント文字列」と呼ばれます。（属性docstrings）
  2. ドキュメントの別の行の直後に出現する文字列リテラルは、「追加のドキュメント行」と呼ばれます。（追加のドキュメント文字列）

属性と追加のdocstringの詳細については、PEP 258「Docutils Project Specification」を参照してください。

Docstring処理


ドキュメンテーション行処理ツールは、空でないすべての行の最小インデントと同じ数のインデントを2番目から削除する必要があります。ドキュメントの最初の行のインデントは重要ではないため、削除されます。ドキュメント行の後の行の相対的なインデントは保持されます。文書の行の最初と最後から空の行を削除する必要があります。

コードは単語よりもはるかに正確であるため、アルゴリズムの実装は次のとおりです。

def trim(docstring):
    if not docstring:
        return ''
    #     (  Python)
    #      :
    lines = docstring.expandtabs().splitlines()
    #    (   ):
    indent = sys.maxsize
    for line in lines[1:]:
        stripped = line.lstrip()
        if stripped:
            indent = min(indent, len(line) - len(stripped))
    #   (   ):
    trimmed = [lines[0].strip()]
    if indent < sys.maxsize:
        for line in lines[1:]:
            trimmed.append(line[indent:].rstrip())
    #       :
    while trimmed and not trimmed[-1]:
        trimmed.pop()
    while trimmed and not trimmed[0]:
        trimmed.pop(0)
    #    :
    return '\n'.join(trimmed)
