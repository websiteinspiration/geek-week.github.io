なんとなく2つのブラウザがスクロールバーに入り......

スクロールバー（スクロールバー）はシンプルでありながら効果的なメカニズムであり、大きなドキュメントを表示するための主要な手段として機能します。しかし、スクロールバーでできることはそれだけではありません。これらの控えめな主力製品は、ユーザーが操作するドキュメントのサイズをユーザーに伝えるのにも非常に適しています。その結果、スクロールバーは二重の負荷をかけます。彼らは様々な材料での作業を支援し、これらの材料のサイズについてユーザーに通知します。

スクロールバーのシンプルで理解可能なロジックは、スクロールジャッキングメカニズムを実装するときに変更できます。これは、スクロールバーの標準動作が変更されたときに発生します。これは、ドキュメントの長さとスクロールバーの高さの間のユーザーの対応に違反します。

さらに、タッチスクリーンを備えたデバイス用のプログラムは、スクロールバーを非表示にするアイデアを広めました。このアプローチでは、ユーザーが特定の表示領域に収まらない要素のスクロールを開始するまで、スクロールバーは表示されません。その結果、アプリケーションの視覚的な魅力のために、設計者はインターフェースの明瞭さを犠牲にしています。ユーザーは、特定のコンテナのコンテンツをスクロールできることを理解するのに少し時間がかかる場合があります。このようなコンテナは、スクロールをサポートしていないかのように見えたり、単にスクロールを必要としないかのように見える場合があります。

従来のデスクトップオペレーティングシステムは、このモバイルトレンドを採用し、アプリケーション設計における従来のスクロールバーの干渉を最小限に抑えようとしました。

本日翻訳する資料は、Webアプリケーションでのスクロールバーの使用に関するいくつかの機能に特化しています。

いくつかの用語


先に進む前に、ここで使用するいくつかの用語を定義しましょう。つまり、2つのタイプのスクロールバーを区別します。

  • 修正された（目立たない）スクロールバー-画面スペースを占めるスクロールバー。それらはあなたが見ているコンテンツと重なりませんが、代わりにそれらの隣に座っています。
  • 固定されていない（邪魔にならない）スクロールバー-コンテンツと重なるスクロールバー。彼らは、彼らが属しているコンテナにあるものから画面スペースを奪いません。

最新のスクロールバーの標準的な動作


デフォルトでは、iOSとAndroidの両方のスクロールバーは固定されていません。

macOS（特に、この資料の執筆時点では実際のmacOS Mojaveについて話している）では、要素がスクロールを開始するまでスクロールバーは非表示になっています。これは、マウスがコンピュータに接続されていない場合のシステムの標準的な動作です。スクロールバーを表示するには3つのオプションがあります（対応する設定はにありますGeneral > System Preferences）。

CSSでどこまで行けますか？


以下は、クラスが割り当てられるコンテナ要素のoverflowing-elementコードと、スタイルを設定するためのCSSコードです。

<div class="overflowing-element"></div>
.overflowing-element {
  overflow-y: auto;
  -webkit-overflow-scrolling: touch;
  -ms-overflow-style: -ms-autohiding-scrollbar;
}

EdgeHTMLに基づいてInternet ExplorerとEdgeで固定されていないスクロールバーが必要な場合、これはこのプロパティが役立つことを意味します-ms-overflow-style: -ms-autohiding-scrollbar;。これを使用すると、すべてが正常に機能します（十分に単純です-正しいですか？）。

iOS 13が登場したとき、スクロールの物理を改善するために-webkit-overflow-scrolling：touchプロパティは必要ない可能性があることがわかりました。ただし、iOSの古いバージョンをサポートする必要がある場合は、このプロパティの使用を拒否しないことをお勧めします。

スクロールに関連するCSSプロパティについて説明する場合は、overscroll-behaviorプロパティについて読むと便利です。スクロールをサポートする要素の境界に達したときのシステムの動作を制御できます。

▍Firefox


Firefoxブラウザーは、接頭部のないCSSプロパティーをサポートします。これはscrollbar-colorとscrollbar-widthです。

次の例では、わかりやすくするために、Internet Explorer 11でサポートされていないCSS変数を使用しています。

:root {
  --scrollbar-track-color: transparent;
  --scrollbar-color: rgba(0,0,0,.2);

  --scrollbar-width: thin; /* or `auto` or `none` */
}
.overflowing-element {
  scrollbar-width: var(--scrollbar-width); 
  scrollbar-color: var(--scrollbar-color) var(--scrollbar-track-color);
}

▍ChromeとSafari、ChromiumベースのEdgeブラウザ、およびその他のブラウザ


WebkitおよびBlinkベースのブラウザーは、スクロールバーをカスタマイズするために設計されたカスタムの疑似要素をサポートします。

:root {
  --scrollbar-track-color: transparent;
  --scrollbar-color: rgba(0,0,0,.2);

  --scrollbar-size: .375rem;
  --scrollbar-minlength: 1.5rem; /*     ( ,  ) */
}
.overflowing-element::-webkit-scrollbar {
  height: var(--scrollbar-size);
  width: var(--scrollbar-size);
}
.overflowing-element::-webkit-scrollbar-track {
  background-color: var(--scrollbar-track-color);
}
.overflowing-element::-webkit-scrollbar-thumb {
  background-color: var(--scrollbar-color);
  /*   -  :hover :active */
}
.overflowing-element::-webkit-scrollbar-thumb:vertical {
  min-height: var(--scrollbar-minlength);
}
.overflowing-element::-webkit-scrollbar-thumb:horizontal {
  min-width: var(--scrollbar-minlength);
}

これは、CSSによって排他的に実行される、スクロールバーをカスタマイズする機能を示すCodePenの例です。そして、ここでのスクロールバーの標準的な動作を示す例です。それらを比較することができます。

上記のコードには1つの問題があることに注意してください。これは、macOSでプロパティheightまたはwidth疑似要素::-webkit-scrollbarを設定すると、固定されていないスクロールバーが固定されたスクロールバーに置き換えられるという事実にあります（標準設定が再定義されます）。ただし、これは小さなJavaScriptコードで修正するのは難しくありません。

プロジェクトに少量のJavaScriptコードを追加できます。これにより、標準のスクロールバーが修正されているかどうかを確認できます。次のようになります。

parent.setAttribute("style", "width:30px;height:30px;");
parent.classList.add('scrollbar-test');

var child = document.createElement("div");
child.setAttribute("style", "width:100%;height:40px");

//   .

  document.body.classList.add("layout-scrollbar-obtrusive");

document.body.removeChild(parent);

スクロールバーが固定されている場合は、ドキュメント要素にbodyクラスを追加しますlayout-scrollbar-obtrusive。このクラスは、プロパティwidthとheight唯一の固定スクロールバーを構成するために使用できます。これにより、スクロールバーの動作における上記の干渉が回避されます。この介入の間、スクロールバーが変化し、プロジェクトはユーザーが行ったシステム設定から離れます。これがクラスのスタイルですlayout-scrollbar-obtrusive：

.layout-scrollbar-obtrusive .layout-scrollbar::-webkit-scrollbar {
  height: var(--scrollbar-size);
  width: var(--scrollbar-size);
}
