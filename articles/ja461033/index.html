この構成はどこから来たのですか？[Debian / Ubuntu]

この投稿の目的は、システム構成ファイルの「ソース検索」に関連するdebian / ubuntuデバッグ手法を示すことです。


テスト例：インストールされたOSのtar.gzコピーを長時間いじめた後、それを復元してアップデートをインストールした後、次のメッセージが表示されます。


update-initramfs: Generating /boot/initrd.img-4.15.0-54-generic
W: initramfs-tools configuration sets RESUME=/dev/mapper/U1563304817I0-swap
W: but no matching swap device is available.
I: The initramfs will attempt to resume from /dev/dm-1
I: (/dev/mapper/foobar-swap)
I: Set the RESUME variable to override this.

目的：この値（U1563304817I0）がどこから来たのか、およびそれを正しく変更する方法を理解する。これは最初に出くわす例であり、それ自体は特に興味深いものではありませんが、Linuxを操作する実際的な方法を示すのに便利です。


ステップ番号1：RESUMEはどこから来たのですか？


# cd /etc
# grep -r RESUME
initramfs-tools/conf.d/resume:RESUME=/dev/mapper/U1563304817I0-swap

再帰的に（-r）/ etcディレクトリ（ほとんどの設定がある場所）でこの変数の言及を探します。conf.dは、initramfs-toolsパッケージによって明示的に使用されるスニペットです。


このスニペットはどこから来たのですか？


3つのオプションがあります。


  1. 魔法のアーティファクト（誰かが置き忘れた）
  2. パッケージからの構成
  3. システムパッケージからスクリプトによって生成された設定

2番をチェックする（最も簡単なものとして）：


 dpkg -S initramfs-tools/conf.d/resume
dpkg-query: no path found matching pattern *initramfs-tools/conf.d/resume*
