例




假设我需要一个按钮，当单击该按钮时，它将播放经典的盒子动画。但是有一个问题：按钮必须进入非活动状态（以标记为黑色.interactable），直到盒子完成旋转为止。

出于代码清洁的原因，我想使用一个await等待框旋转完成并立即添加一行内容的方法interactable，而不是像创建一个包含协程的对象那样检查每一帧的状态以启动此任务。就可读性而言，在一处拥有明显的线性代码是一大优势。这样的代码很不错。

以下简单，紧凑的代码实现了所需的任务。 （async 方法也出现在Unity委托列表中，请放心。）

using System.Threading.Tasks;
using UnityEngine;
using UnityEngine.UI;

public class SpinBox : MonoBehaviour
{
    public Button button;
    public Animation ani;

    public async void SpinAndDisableButton()
    {
        ani.Play();
        button.interactable = false;
        while (ani.isPlaying == true)
        {
            await Task.Yield();
        }
        button.interactable = true;
    }
}

与常规C＃程序相比


一切与常规C＃程序并不完全相同。查看官方文档await. await意味着返回该对象的调用者Task。调用者可以继续执行，直到他需要执行结果为止，并且此时他再也负担不起做其他事情（任务是否为多线程无关紧要），除了await。这个链条可以继续，直到您最终到达Main。在这里，如果我们也见面async，则编译器会生成另一个等待状态，它会立即请求结果。

现在回到我们的SpinAndDisableButton。退货去哪儿了await？在Unity中，我们没有Main，因为它已深深地隐藏在引擎代码中。实际上，问题与谁执行的问题相同Update，LateUpdate依此类推。这是一个PlayerLoopAPI，它支持游戏周期内的引擎代码，以提供对象的有序渲染以及框架中执行的所有操作。但是以前，这并没有打扰我们，因为到目前为止这些入口点已经返回void。

    {
        Debug.Log($"Update {Time.frameCount}");
