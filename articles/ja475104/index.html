モジュールで飛ぶ：Jetpackを使用したマルチモジュールアプリケーションでのナビゲーション


ほとんどすべての成長プロジェクトは遅かれ早かれマルチモジュラーアーキテクチャに目を向け始めます。開発者は、変更された機能が1つだけの場合に、プロジェクト全体が再アセンブルされるまで待ちたくありません。マルチモジュール性は、アプリケーション機能を互いに分離するのに役立ち、ビルド時間を短縮します。ただし、この分離により、コンポーネントのスコープにいくつかの制限が課されます。 1つのモジュールを持つプロジェクトでJetpackからのナビゲーションを使用する場合、ナビゲーショングラフは任意のアプリケーションパッケージから利用できます。プロジェクトにネストされたフラグメントがある場合、NavControllerが実行するアクションを常に明示的に指定でき、グローバルホストにアクセスすることもできます。しかし、多くのモジュールがある場合、質問が発生します：ナビゲーショングラフを作成する場所、それにアクセスする方法、およびモジュールの依存関係で混乱しないようにする方法。これについてはカットの下で話します。


ナビゲーショングラフ


マルチモジュールアプリケーションを設計するときに覚えておくべき最も重要なことは、依存関係です。モジュールの依存関係ツリーの依存関係は、一方向に向ける必要があります。



マルチモジュールアプリケーションで最も依存するモジュールは、常にアプリモジュールです。彼は他のほとんどすべてのモジュールを知っています。アプリでは、DIは通常、さまざまなフレームワークを使用して実装されます。アプリモジュールのこの依存関係を使用して、メインホストのナビゲーショングラフを実装することができます。


アプリモジュールが実装する機能はできる限り少なくする必要があることを常に覚えておく必要があります。これは、アプリモジュールが最も依存しており、プロジェクトのほとんどすべての変更がアプリモジュールの再アセンブリにつながるためです。

FragmentExt.kt


fun Fragment.navigate(navCommand: NavCommand, hostId: Int? = null) {
   val navController = if (hostId == null) {
       findNavController()
   } else {
       Navigation.findNavController(requireActivity(), hostId)
   }
   navController.navigate(navCommand.action, navCommand.args, navCommand.navOptions)
}
