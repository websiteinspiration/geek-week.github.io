作業スキームは次のとおりです。サーバーから、PC /ラップトップで実行されているプログラムを使用して、データが保存されているファイルを読み取ります。 USB経由のこのプログラムは、コントローラーにデータを送信します。コントローラはデータを受信し、それらに対してアクションを実行します。サーバーを使用するスキームは、インターネットにアクセスできる世界中のどこのスマートフォン/タブレット/ PC /ラップトップからでもサイトにアクセスするだけで、アプリケーションなしでスキームを制御できるという点で魅力的です。

COMポートを介してデータを受信し、それらをコマンドの内部テーブルと比較して、対応するアクションを実行します。

スキームは最初は非常に単純に見えましたが、1つありましたが、インターネット上のファイルを読み取ってCOMポートにデータを送信するプログラムはありませんでした。したがって、私はそのようなプログラムを書かなければなりませんでした。

プログラムはVB6で書かれました。サーバーからファイルを読み取るには、VB6コンポーネント（Microsoft Internet Transfer Control 6.0）を使用します。その助けを借りて、サーバー上のテキストファイルは、単に文字列変数に読み込まれます。この行を読み取った後、VB6コンポーネントを使用してCOMポートに送信されます：Microsoft Comm Control 6.0。ファイルを読み取り、行を送信するプロセス全体は、タイマーを使用してループで読み取られます。タイマー間隔は、プログラム設定で、または操作中に直接変更できます。さらに、ポートの動作モード、ポート番号、インターネット接続の動作モード、および読み取り可能なファイルへのリンクを選択できます。

大きなファイルサイズと小さなギャップがあると、プログラムはフリーズしますが、引き続き機能することを述べておきます。私のプログラムのバッファサイズは512バイトです。 MKのバッファーが少ないことを考えると、これで十分です。

重要なポイント。MKのプログラムはデータを解析できません。シリアルポート経由で入力に送信された文字のみを読み取ることができます。エラーなしで、私はラテンアルファベット（26文字のAZと10桁の0〜9）を受け入れることができました。合計36チームで、アルゴリズムが完成し、データ解析がMKに導入されると、あらゆるデータを転送できます。双方向のデータ交換のためにソフトウェアに「ドーピング」する可能性もあります。

PHPコード、ここでは、LEDをオンにする必要がある場合はコマンドAをファイルに書き込み、LEDが消灯する場合はBを書き込みます。

<?php
$cm = fopen("command.txt", "w");
switch ($_POST['sv']) {
case "On":
fwrite($cm,"a");    
    break;   
case "Off":
fwrite($cm,"b");    
    break;  
}
fclose($cm); 
?>
