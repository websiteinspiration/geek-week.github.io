Jupyter NotebookでSmileをダウンロードする


優れた準備環境！ディレクトリのフォルダーに新しいScalaノートブックを作成し、Smileを機能させるためにMavenからライブラリーをダウンロードする必要があります。

%%classpath add mvn
com.github.haifengl smile-scala_2.12 1.5.2

コードを実行すると、ダウンロードされたjarファイルのリストが出力ブロックに表示されます。

次のステップ：サンプルを機能させるために必要なパッケージをインポートします。

import java.awt.image.BufferedImage
import java.awt.Color
import javax.imageio.ImageIO
import java.io.File
import smile.clustering._

クラスタリングのためのデータの準備


次に、次の問題を解決します。赤、緑、青（R、G、B）の3つの原色のゾーンで構成される画像を生成します。画像の色の1つが優先されます。画像のピクセルをクラスター化し、最も多くのピクセルが存在するクラスターを取得し、それらの色を灰色に変更して、すべてのピクセルから新しい画像を作成します。期待される結果：主な色のゾーンは灰色になり、残りのゾーンの色は変わりません。

//    640  360
val width = 640
val hight = 360

//     
val testImage = new BufferedImage(width, hight, BufferedImage.TYPE_INT_RGB)

//   .    .
for {
    x <- (0 until width)
    y <- (0 until hight)
    color = if (y <= hight / 3 && (x <= width / 3 || x > width / 3 * 2)) Color.RED
    else if (y > hight / 3 * 2 && (x <= width / 3 || x > width / 3 * 2)) Color.GREEN
    else Color.BLUE
} testImage.setRGB(x, y, color.getRGB)

//   
testImage

このコードを実行すると、次の画像が表示されます。



次のステップ：画像をピクセルのセットに変換します。ピクセルとは、次のプロパティを持つエンティティを意味します。

  • ワイドサイド座標（x）;
  • 狭い側の座標（y）;
  • 色の値;
  • クラス/クラスター番号のオプションの値（クラスター化が完了する前は空）。

エンティティとして使用すると便利case classです。

case class Pixel(x: Int, y: Int, rgbArray: Array[Double], clusterNumber: Option[Int] = None)

ここでは、rgbArray赤、緑、青の3つの値の配列が色値に使用されています（たとえば、赤Array(255.0, 0, 0)）。

//      (Pixel)
val pixels = for {
    x <- (0 until testImage.getWidth).toArray
    y <- (0 until testImage.getHeight)
    color = new Color(testImage.getRGB(x, y))
} yield Pixel(x, y, Array(color.getRed.toDouble, color.getGreen.toDouble, color.getBlue.toDouble))

//   10  
pixels.take(10)

これでデータの準備は完了です。

ピクセルカラークラスタリング


したがって、3つの原色のピクセルのコレクションがあるため、ピクセルを3つのクラスにクラスター化します。

//  
val countColors = 3

//  
val clusters = kmeans(pixels.map(_.rgbArray), k = countColors, runs = 20)

ドキュメントrunsでは、10〜20の範囲でパラメーターを設定することを推奨しています。

このコードを実行すると、タイプのオブジェクトが作成されKMeansます。出力ブロックには、クラスタリングの結果に関する情報が含まれます。

K-Means distortion: 0.00000
Clusters of 230400 data points of dimension 3:
  0    50813 (22.1%)
  1    51667 (22.4%)
  2    127920 (55.5%)

クラスターの1つには、他のクラスターよりも多くのピクセルが含まれています。次に、ピクセルのコレクションを0から2までのクラスでマークする必要があります。

//   
val clusteredPixels = (pixels zip clusters.getClusterLabel()).map {case (pixel, cluster) => pixel.copy(clusterNumber = Some(cluster))}

//  10  
clusteredPixels.take(10)

画像を再描画します


残っているのは、最大のピクセル数を持つクラスターを選択し、このクラスターに含まれるすべてのピクセルを灰色に再描画することです（配列の値を変更rgbArray）。

//  
val grayColor = Array(127.0, 127.0, 127.0)

//      
val blueClusterNumber = clusteredPixels.groupBy(pixel => pixel.clusterNumber)
    .map {case (clusterNumber, pixels) => (clusterNumber, pixels.size) }
    .maxBy(_._2)._1

//      
val modifiedPixels = clusteredPixels.map {
    case p: Pixel if p.clusterNumber == blueClusterNumber => p.copy(rgbArray = grayColor)
    case p: Pixel => p
}

//  10     
modifiedPixels.take(10)

複雑なことは何もありません。クラスタ番号（これは私たちのものですOption:[Int]）でグループ化し、各グループの要素の数を数えて、要素の最大数のクラスタを引き出します。次に、見つかったクラスターに属するピクセルのみの色をグレーに変更します。

新しい画像を作成し、結果を保存します。


ピクセルコレクションから新しい画像を収集する：

//      
val modifiedImage = new BufferedImage(width, hight, BufferedImage.TYPE_INT_RGB)

//    
modifiedPixels.foreach { 
    case Pixel(x, y, rgbArray, _) => 
        val r = rgbArray(0).toInt
        val g = rgbArray(1).toInt
        val b = rgbArray(2).toInt
        modifiedImage.setRGB(x, y, new Color(r, g, b).getRGB)
}

//   
modifiedImage

それが結局のところ私たちがしたことです。



両方の画像を保存します。

ImageIO.write(testImage, "png", new File("testImage.png"))
ImageIO.write(modifiedImage, "png", new File("modifiedImage.png"))

結論


Scalaには機械学習が存在します。基本的なアルゴリズムの実装では、巨大なライブラリをドラッグする必要はありません。上記の例は、開発中に通常のツールをあきらめることができないことを示しています。同じJupyter NotebookをScalaと簡単に友達にすることができます。

もちろん、Smileのすべての機能の完全な概要のために、1つの記事は十分ではありません、そして、これは計画に含まれませんでした。主なタスク-Scalaで機械学習の世界への扉を開くこと-は完了したと思います。これらのツールを使用するかどうか、さらには本番環境にドラッグするかどうかは、あなた次第です。
