ダイアログボックスとは何ですか？

ウィキペディアは次のように述べています。
グラフィカルユーザーインターフェイスのダイアログボックスは、特別なインターフェイス要素であり、情報を表示し、ユーザーからの応答を受け取るように設計されたウィンドウです。コンピュータとユーザーの双方向の対話（「ダイアログ」）を実行するため、その名前が付けられました。ユーザーに何かを伝え、ユーザーからの応答を待ちます。

興味がある
彼からの返事を待っている
つまり、モーダルウィンドウを開いてフィードバックを取得し、その後何かを行います。似ていませんか？そして私はそう思いました。

状況を想像してみてください。ユーザー管理のアプリケーションがあります。
シナリオは次のとおりです。

メインページで、ユーザーはモーダルウィンドウを開いて他のユーザーのリストを表示できます。
リストから、ユーザー情報を含むモーダルウィンドウを開くことができます。このウィンドウには、手紙を送信するためのフォームもあります。

レターを送信するとき、ユーザーは正常な送信に関するモーダルウィンドウを開きます。
ユーザーがモーダルウィンドウを閉じると、ユーザーリストのモーダルウィンドウに戻りますが、別の文字を書き込むためのボタンがあり、クリックすると、ユーザーのページに移動します。

シーケンス図の形でこのタスクを想像するのは非常に困難です。



今ではすべてがはるかに簡単です。

コードの観点から見ると、モーダルウィンドウを開くことは同期アクションであり、開くことができます。ウィンドウ内で閉じることができますが、モーダルウィンドウでデータを変更すると、閉じるときに、そこからデータを取得する必要がありますか？

簡単な例として、ユーザーのモーダルウィンドウからデータを変更し、リストのモーダルウィンドウに戻ったら、このユーザーに関する情報を更新する必要があります。

非同期のような匂いがします...

モジュールを開くとき、モジュールが閉じてユーザーが入力したデータを取得するまで待つ必要があります。非同期アクションはpromiseで非常に適切に実装されています。

実際、プロミスはすでに図に組み込まれています。アクションとしてマークするだけです。少しやり直すことができます。



これで、ユーザーがモーダルウィンドウを開いたときにすべてがシンプルになり、作業が完了するまで待機します。その後、promisでresolveが呼び出されます。簡単に聞こえます。始めましょう。

私の主なフレームワークは反応なので、それに基づいてすぐに行います。アプリケーションのどの部分からもモーダルウィンドウを開くことができるようにするために、Context APIを使用します。

まず、コンテキストとそれを格納する場所を作成する必要があります。

// ./Provider.js
export const DialogContext = React.createContext();

export const Provider = ({ children, node, Layout, config }) => {
  const [instances, setInstances] = useState([]);
  const [events, setEvents] = useState([]);

  const context = {
    instances,
    setInstances,
    config,
    events,
    setEvents
  };

  const Component = instances.map(instance => (
    <Layout
      key={instance.instanceName}
      component={config[instance.instanceName]}
      {...instance}
    />
  ));

 const context = {
     instances
     setInstances
  };

  //   state    
  const child = useMemo(() => React.Children.only(children), [children]);

  return (
    <DialogContext.Provider value={context}>
      <>
        {child}
        {createPortal(Component, node)}
      </>
    </DialogContext.Provider>
  );
};

ここではすべてが単純です。最初のuseStateを使用して、開いているモーダルウィンドウの配列を作成します。スタックのようなもの。

2番目のuseStateは、promiseを解決して拒否するための参照を追加するために必要です。これは以下で確認します。

レンダリングはポータルを介してリダイレクトされるので、Z-indexで何かが発生した場合でも戦う必要はありません。

レイアウトは、すべてのモーダルウィンドウの基本コンポーネントとなるコンポーネントです。

configパラメータは単なるオブジェクトであり、キーはモーダルウィンドウの識別子であり、値はモーダルウィンドウのコンポーネントです。

//  config.js
export const exampleInstanceName = 'modal/example';

export default {
  [exampleInstanceName]: React.lazy(() => import('./Example')),
};

次に、モーダルウィンドウを開くメソッドの実装を記述します。

これがフックになります：


export const useDialog = () => {
  const { setEvents, setInstances, config } = useContext(DialogContext);

  const open = instance =>
    new Promise((resolve, reject) => {
      if (instance.instanceName in config) {
        setInstances(prevInstances => [...prevInstances, instance]);
        setEvents(prevEvents => [...prevEvents, { resolve, reject }]);
      } else {
        throw new Error(`${instance['instanceName']} don't exist in modal config`);
      }
    });

  return { open };
};

フックは、モーダルウィンドウを呼び出すために使用できるopen関数を返します。

このオプションでは、モーダルウィンドウが閉じるまで待機することはありません。Promiseを完了するには、メソッドを追加する必要があります。

// ./Provider.js
export const DialogContext = React.createContext();

export const Provider = ({ children, node, Layout, config }) => {
  const [instances, setInstances] = useState([]);
  const [events, setEvents] = useState([]);

  const close = useCallback(() => {
    const { resolve } = events[events.length - 1];
    const resolveParams = { action: actions.close };

    setInstances(prevInstances => prevInstances.filter((_, index) => index !== prevInstances.length - 1));
    setEvents(prevEvents => prevEvents.filter((_, index) => index !== prevEvents.length - 1));

    resolve(resolveParams);
  }, [events]);

  const cancel = useCallback((values): void => {
    const { resolve } = events[events.length - 1];
    const resolveParams = { action: actions.cancel, values };

    setInstances(prevInstances => prevInstances.filter((_el, index) => index !== prevInstances.length - 1));
    setEvents(prevEvents => prevEvents.filter((_el, index) => index !== prevEvents.length - 1));

    resolve(resolveParams);
  },  [events]);

  const success = useCallback((values) => {
      const { resolve } = events[events.length - 1];
      const resolveParams = { action: actions.success, values };

      setInstances(prevInstances => prevInstances.filter((_el, index) => index !== prevInstances.length - 1));
      setEvents(prevEvents => prevEvents.filter((_el, index) => index !== prevEvents.length - 1));

      resolve(resolveParams);
    }, [events]);

  const context = {
    instances,
    setInstances,
    config,
    events,
    setEvents
  };

  const Component = instances.map(instance => (
    <Layout
      key={instance.instanceName}
      component={config[instance.instanceName]}
      cancel={cancel}
      success={success}
      close={close}
      {...instance}
    />
  ));

 const context = {
     instances
     setInstances
  };

  //   state    
  const child = useMemo(() => React.Children.only(children), [children]);

  return (
    <DialogContext.Provider value={context}>
      <>
        {child}
        {createPortal(Component, node)}
      </>
    </DialogContext.Provider>
  );
};

これで、Layoutコンポーネント内で、またはこれらのメソッドをモーダルウィンドウコンポーネントに渡すと、success、cancel、またはcloseメソッドが呼び出され、必要なプロミスで解決されます。ここではアクションなどの概念が追加されていますが、これは対話が完了したステータスで答える行です。これは、モーダルウィンドウの後に何らかのアクションを実行するときに役立ちます。

const { useState } from 'rect';
import { exampleInstanceName } from './config';
import { useDialog } from './useDialog';

const FillFormButton = () => {
  const [disabled, setDisabled] = useState(false); 
  const { open } = useDialog();
  const fillForm = () => open(exampleInstanceName)
    .then(({ action }) => {
      if (action === 'success') setDisabled(true);
    });

  return <button onClick={fillForm} disabled={disabled}>fill form from modal</button>
}

それで全部です。モーダルウィンドウからopen関数のモーダルウィンドウへのパラメーターの転送を追加するために残ります。まあ、私はあなたがこれを自分で処理できると思いますが、もしあなたが怠惰すぎる場合は、プロジェクトで使用できる既製のパッケージがあります。
