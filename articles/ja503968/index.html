エンジンは、StuduinoボードのM2コネクターに接続されているモーターのM1およびD7、D8、D5コネクターに接続されているエンジンの端子D2、D4、D3の論理状態を設定することによって制御されます。 Studuinoボードの開発者がモータードライバードライバーチップを並列に接続したのは、これらの結論に基づいています。そこに回路を接続してこれらの結論を使用し、同時にエンジンを同時に使用すると、失敗します。

M1コネクタに接続されているエンジンの制御方法を検討してください。端子D2とD4に異なる論理状態を設定することにより、エンジンを目的の方向に回転させることができます。例：D2-1およびD4-0の場合、エンジンは時計回りに回転し、D2-0およびD4-1の場合、エンジンは反時計回りに回転します。また、D3ピンにPWM値を設定して、モーターの回転速度を設定する必要があります。0〜255の範囲で設定できます。値が0の場合、モーターは回転せず、値が255の場合、最大回転速度を取得します。エンジンドライバーは、エンジンの回転方向を瞬時に変更することもでき、エンジンブレーキをサポートします。エンジンを減速するには、端子D2とD4で1と同じ論理状態を設定する必要があります。

ちなみに、Studuinoボード上の3つのピン（GND、VCC、SIG）を持つコネクターにはD3およびD5ピンはありませんが、プラットフォームの互換性を維持するために開発者が残した標準のArduinoコネクター上にあります。

同様に、M2コネクタに接続されているモーターが制御されます。彼にとって、回転方向は、端子D7とD8の状態、および端子D5による回転速度によって指定されます。

ロボットを前後に動かすには、DCモーターまたはサーボを使用します。サーボドライブは、ロボットを適切な場所に停止する方法の問題を解決します（サーボの回転角度を正確に設定できます）。このソリューションには制限があります（ホイールが固定されたセットのサーボを使用します）-サーボは180度を超える角度で回転することができないため、ロボットの動きはサーボのホイール回転の長さの半分に制限されますが、通常はもっと必要です。

セットからの減速機付きのDCモーターを使用して、前後に移動します。これらのモーターにはフィードバックがありません。エンジン始動後、ロボットがどこまで移動したかはわかりません。エンジンの動作中にロボットが必要な距離を移動する時間を検出し、これらの遅延をプログラムで使用して、ロボットを適切な場所に停止できます。この方法には重大な欠点が1つあります。エンジン速度は、供給される電圧と必要な労力に依存します。ロボットはバッテリーを使用しているため、しばらくすると少し放電して電圧が下がるので、同時にロボットはより短い距離を移動し始めるので、再度時間を選択する必要があります。

使用できる最も簡単なオプションは、ロボットが目的のポイントを通過してタッチセンサーに触れることです。タッチセンサーは、適切な場所に停止信号になります。

このデザインでは、カウントダウンは紙の黒いストライプで発生します。これらのバンドの数は事前にわかっているため、最後のバンドに到達したら、ゼロ基準点に戻ることができます。 Studuinoのスクラッチプログラムを以下に示します。


StuduinoのスクラッチIRロボットプログラム

プログラムを注意深く見ると、IRセンサーの値と比較される40と50の理解できない値がわかります。これらの数値を理解するには、さまざまなセンサーから読み取ることができる信号について少し覚えておく必要があります。デジタルとアナログのセンサーがあります。デジタルセンサーの位置は固定されており、ある位置から別の位置に急激に移動し、中間値はありません。 2つの位置で主に使用されるセンサー。タッチセンサーは、デジタルセンサーの最も印象的な例です。センサーが押されるまで論理値1が与えられ、押されると論理値0が使用されます（このコンストラクターでは、これらのセンサーではこのスキームが使用されますが、他の方法で行われるスキームもあります：押された-1、押されていない-0）。アナログセンサーは、信号と出力の電圧または電流の間の不可分な関係を表し、固定値はありません。 IRセンサーは、周囲の光、バッテリー電圧、さらには周囲の温度などのさまざまな要因に依存する可能性のあるアナログ光信号を発信します。

アナログ信号は、StuduinoのScratchの場合、0から100の範囲で数値に線形変換されます。これは、エンドユーザーに対する作業を簡略化するために行われます（このプログラミング環境は、子供たちを教えるために最初に開発されました）。 Arduino IDEでは、0から1023までの「正直な」値の範囲が発行されます。これは、Atmega168コントローラーにある10ビットADCコンバーターに対応します。デジタル信号とアナログ信号のグラフの違いを見ると、アナログ信号が値の範囲である理由が明らかになります。

IRセンサー

IR光線が反射されるセンサーの前に障害物を置くと、障害物からの反射がよくなります（障害物までの距離または障害物の種類によって異なります）、IRフォトディテクターによって検出され、センサーはより多く戻ります調査中のユーザーにとっての価値。

数値40と50は経験的に選択されています。私の場合、センサーは白い表面の上に約65-75の値を示しました。黒い表面の上で、センサーは18-25の範囲の値を返しました。数値40はセンサーが白い表面から黒色に切り替わる瞬間であり、数値50は黒い表面から白色に変化する瞬間です。これらの数値は、センサーの測定におけるエラーをブロックするために、わずかなマージンをとって取られています。実際の設計を開発する場合、これらのすべての要因がセンサーの読み取りに影響を与えるため、センサーが読み取りを行う条件（外部照明、センサーの場所など）を考慮する必要があります。外部条件に応じてこれらの値を常に調整するためのアルゴリズムを開発する必要がある場合があります。

小さな結果をまとめます。倉庫内でのロボットの移動の問題に対する2つの可能な解決策が検討されました。これらのオプションはどちらも、ロボットを誘導する追加の外部「タグ」を必要としました。それらなしで行う方法はありますか？たとえば、ロボットがモーターシャフトがどの角度で回転したかを知るために、角度の値に応じて、停止または移動を決定しました。この問題に対する簡単な解決策があります-エンコーダー付きのモーター（角度センサー）の使用です。

セットにはギア付きのDCモーターがありますが、エンコーダーはありません。考えが思い浮かびました。特に、セットのギアがかなり大きなモジュール（歯のサイズ）を持っているので、デザイナーのパーツからシンプルなエンコーダーを作ってみようと思うかもしれません。


エンコーダの主な要素

主な問題は、位置を選択してIRセンサーを固定し、回転時にギアの歯と交差するようにすることでした。その結果、下図のようなテスト設計ができました。


エンコーダーのテストモデル

上の図に示すように、IRセンサーは固定されているため、回転中に歯車がその作動測定領域を歯と交差します。 IRセンサーの反対側（ギアの反対側）に、センサーからより正確なデータを取得するために、赤外線の反射に障害物を追加しました。ギアが回転し、IRセンサーが信号の反射を測定するとき、センサーの前に歯がある場合は大きな値になり、ギアの歯の間に「穴」がある場合は小さな値になります。

エンコーダを操作するためのプログラムは、Arduino IDEで開発されています。私のアイデアの効率をテストするために、エンジンを一定の速度で回転させ、IRセンサーからデバッグコンソールに値を継続的に出力する簡単なプログラムを作成しました。

プログラムテキスト 
#define M1_A        2       //   1   1
#define M1_B        4       //   2   1
#define M1_PWM      3       //       1
#define SENSOR_PIN  A4      //     IR-sensor

void setup() {
  Serial.begin(9600);       //      
  pinMode(M1_A, OUTPUT);    //       " "
  pinMode(M1_B, OUTPUT);    
  analogWrite(M1_PWM, 100); //    
  digitalWrite(M1_A, HIGH); //    
  for (int i=0; i < 2000; i++) {    //   2000    -
    Serial.println(analogRead(SENSOR_PIN));   //      
  }
  digitalWrite(M1_A, LOW);  //   
}

void loop() {
}

このアルゴリズムでギアの歯を数える原理は、IRセンサーからの読み取り値の一定の読み取りと、信号レベルが「エラーバンド」の上限または下限を超えたときのcur_state変数の値の変化に基づいています。値が上限を超えると、変数cur_stateは1に等しくなり、歯車の歯を意味します。下限を超えると、変数cur_stateは0に等しくなり、歯車の歯の間のギャップになります。count変数の追加は、cur_state変数の状態が変化したときにのみ行われます。

このアルゴリズムを使用するプログラムを以下に示します。その中で、エンジンがギアを所定の歯数だけ回転させるまで待機し、その後メインプログラムに制御を移すサブプログラムについて説明しました。

プログラムテキスト 
#define M1_A        2       //   1   1
#define M1_B        4       //   2   1
#define M1_PWM      3       //       1
#define SENSOR_PIN  A4      //     IR-sensor

#define MIDDLE      550     //         
                            // ""
#define WIDTH       50      //      ,   
      //    -     
      // 

int enc_tooth = 0;          // ,    
int cur_state = 0;          //   :  (1)  "" (0)
int prev_state = 0;         //   
int tmp;                    //      

void init_enc() {
  enc_tooth = 0;
  cur_state = 0;
  prev_state = 0;

  //         cur_state
  //    ,   = 1,    ""      
  //   0      
  if (analogRead(SENSOR_PIN)>MIDDLE) {
    prev_state = 1;
    cur_state = 1;
    enc_tooth++;
  }  
}

void wait_by_count(int count) {
  //       count,       IR 
  //           ""
  while (enc_tooth <= count) {
    //    IR-    
    tmp = analogRead(SENSOR_PIN);

    //       +  ,    
    if ((tmp + WIDTH) > MIDDLE) {
      cur_state = 1;
    } else {
    //       -  ,    ""
      if ((tmp - WIDTH) < MIDDLE) {
        cur_state = 0;
      }
    }

    //       ,      
    // ,     ""   -  ""  
    if (cur_state != prev_state) {
    //   ,   0  - ,     
    // 
      enc_tooth += cur_state;
    
    //        
      prev_state = cur_state;
    }
  }
}

void setup() {
  //        
  Serial.begin(115200);

  //        ""
  pinMode(M1_A, OUTPUT);
  pinMode(M1_B, OUTPUT);

  //   
  init_enc();

  //    
  analogWrite(M1_PWM, 100);
 
  //    ,      1 (    
  //  0)
  digitalWrite(M1_A, 1);

  // ,    30  
  wait_by_count(30);
  
  //   
  digitalWrite(M1_B, 1);

  Serial.print("  = ");
  Serial.println(enc_tooth);

}

void loop() {
}
