Vite-在Vue上没有捆绑的潜在客户开发

嗨哈布罗夫斯克 六月，OTUS推出了Vue.js开发人员课程。预期课程的开始，我们通常会准备一份有趣的材料的翻译本。


一天，Ewan Yu（Vue.js的创建者）决定不夜眠，并最终创建了Vite。



Vite允许您使用单个文件组件在Vue上创建应用程序，而无需进行捆绑的组装步骤。浏览器要求将导入作为本机ES模块。开发服务器拦截对.vue文件的请求，并即时对其进行编译。它立即发生。

请注意，Vite仍是实验性的解决方案，我不知道将来会发生什么，以及本文将持续使用多长时间。将其视为开胃酒或有趣的事物。

我们试试吧


创建以下文件：

Comp.vue

<template>
  <button @click="count++"> {{ count }}</button> 
</template>

<script>
export default {
  data: () => ({ count: 0 })
}
</script>

<style scoped>
button{
  font-size: 2rem;
}
</style>

index.html

<div id="app"></div>
<script type="module">
  import { createApp } from 'vue'
  import Comp from './Comp.vue'

  createApp(Comp).mount('#app')
</script>

然后执行以下操作：

npx vite

转到http://localhost:3000，编辑文件.vue以查看即时更改。

生产组装


现在，让我们创建一个可用于生产的应用程序。

创建package.json

{
  "scripts": {
    "build": "vite build"
  }
}

跑：

npm i -D vite
npm run build


用代码检查目录。
我遇到的问题：

  • 错误：找不到模块“ tslib”，通过
    运行npm i -D tslib 解决
  • 错误：ENOENT：没有这样的文件或目录
    通过手动创建一个空目录解决

思想


它真的很容易启动，并且开发速度更快，因为我立即看到了更改的结果。类似于即时编译。因此，看起来确实很有希望，也许过一会儿，我们将继续使用Vue的这种开发方法。

埃文·于，尽你的魔力。

有趣的事实：vue是从法语翻译而来的，而且浏览速度也很快。

似乎目前为止，有关Vite的工作已经完成，并且可能会在Vue 3之后继续进行。最简单的尝试方法是Vue 3 Beta。

我说，这是基于一个完整的从鸣叫埃文羽。但是，看着提交的信息库，我们可以说，这项工作仍在进行中。
