Parsing YouTube, including uploaded data, without the YouTube API

Introduction


To load content data on YouTube, they usually use either Selenium or the YouTube API. However, there are disadvantages everywhere.

  1. Selenium is too slow to parse. Imagine parsing a playlist of ~ 1000 clips by selenium.
  2. The YouTube API, of course, is the best option if you have some kind of your application or project, but there you need to register the application, etc. In the "trial" version you need to constantly log in to use the api, there is also a fast-ending quota.
  3. In our method, I would say very complex data structures generated by YouTube. Parsing YouTube searches is particularly unstable.

How to load data on YouTube?


There is a token for this, which can be found in the html code of the page. Then in the future we will use it as a parameter for a request to YouTube, which gives us new content. YouTube itself downloads content using a request where this token is used.

There are additional outgoing parameters that we will need in the next step.



Getting a token through a script


We compose the headers parameter for the request to YouTube. In addition to the user-agent, we insert two additional ones, which you see below.

headers = {
    'User-Agent': '      ,      ',
    'x-youtube-client-name': '1',
    'x-youtube-client-version': '2.20200529.02.01'
}

We make a request using the requests library. Put a link to the page you want to load, and also add headers.

token_page = requests.get(url, headers=headers)

The token cannot be found by parsers, because it is hidden in the tag tag. To save it to a variable, I write such an ugly code:

nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]

Usually this is a token 80 characters long.

Making a request for content


service = 'https://www.youtube.com/browse_ajax'
params = {
"ctoken": nextDataToken,
"continuation": nextDataToken
}
r = requests.post(service, params=params, headers=headers)

Different types of loaded data have different service links. Ours is suitable for playlists and video from channels.

YouTube sends data in json format. Therefore, we write r.json (), but the list will fly to you, we need the second element of the list, therefore r.json () [1]. Further, we already have data. It remains to parse.

Parsing json response


You can see long chains of dictionaries, but we will shorten them so that it is more or less clear.

r_jsonResponse = r_json['response']
dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]

Here we get a new token for further request. If the loaded data is over, then you will not see the token.
for content in dataContainer["contents"]:
	videoId = content['playlistVideoRenderer']['videoId']

This is how you can extract the id of the video by adding the template part, you will get a link to the video.

To get the following data, you must do the same thing - request a token, parsing and then again.

The fully working code looks like this:

import requests, json

url = input('url: ')
headers = {
		"User-Agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/81.0.4044.122 Safari/537.36",
		'x-youtube-client-name': '1',
		'x-youtube-client-version': '2.20200429.03.00',
		}
token_page = requests.get(url, headers=headers)
nextDataToken = token_page.text.split('"nextContinuationData":{"continuation":"')[1].split('","')[0]
sleep = False #  ,    
ids = []
while not sleep:
	service = 'https://www.youtube.com/browse_ajax'
	params = {
	"ctoken": nextDataToken,
	"continuation": nextDataToken
	}
	r = requests.post(service, params=params, headers=headers)
	r_json = r.json()[1]
	r_jsonResponse = r_json['response']
	dataContainer = r_jsonResponse["continuationContents"]["playlistVideoListContinuation"]
	try: #  
		nextDataToken = dataContainer["continuations"][0]["nextContinuationData"]["continuation"]
	except:
                #  . ,    .    
		sleep = True
	for content in dataContainer["contents"]:
		videoId = content['playlistVideoRenderer']['videoId']
		ids.append(videoId)
print(len(ids))
