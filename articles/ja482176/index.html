@Kubernetesカンファレンスの11月29日の様子：ビデオと結果



Mail.ru Cloud Solutions が主催する@Kubernetesカンファレンスが 11月29日に開催されました。会議は@Kubernetes mitapsから生まれ、シリーズの4番目のイベントとなりました。Mail.ruグループには350人以上の参加者が集まり、私たちと一緒にロシアでKubernetesエコシステムを作る人々と最も差し迫った問題について話し合っています。 会議のビデオレポートのカットシーン-Tinkoff.ruがインフラストラクチャプロバイダーBareMetalを作成した方法、KubernetesがMail.ruグループにアップロードされた方法、HelmがRollingUpdateとインタラクティブに謎を解く-CarPrice、Eldorado.ru、Rosgosstrakh、 Brain4Net、および将来のスピーカー@Kubernetesのコンテスト。




会議のゲストに感謝


私たちに参加してくれた皆さんに感謝します-あなたがいなければ、@ Kubernetesは違っていただろう。



そしてここにビデオがあります：


会議の開幕。Mail.ruクラウドソリューションプラットフォーム責任者、Ilya Letunov氏



プログラムでは、ロシアの景観でK8を使用するための可能な限り幅広いオプションを収集することを目指しました。本番環境でK8を実装し、それを使用するための独自のツールを作成し、エラーを収集して問題を解決する方法を探し、他の人がKubernetesに切り替え、K8sプロバイダーとして開発し、テクノロジーのニュアンスに関する知識を広めるために取り組む人々のストーリーを待っています。これがロシアのKubernetesコミュニティの集合的な体験の1つの写真になることを願っています。

Tinkoff.ruがインフラプロバイダーのBareMetalを書いたとき。Stanislav Halup氏、Tinkoffのアトラクションインフラストラクチャグループヘッド



Stanislav Halupは2年間、Tinkoff.ruでアトラクションインフラストラクチャグループを率いてきましたが、マネージドK8とパブリッククラウドでの生活についてよく話しますが、今回は、Hyperscale Privateプラットフォームをゼロから構築するときに直面した技術的な課題について説明します。

Mail.ru GroupサービスをKubernetesに転送するにはどうすればよいですか。Mail.ruグループ、プラットフォーム、テクニカルプロジェクトマネージャー、ミハイルペトロフ



ミハイルペトロフは、Mail.ruグループのKubernetesコンピテンスセンターを率いており、グループのすべてのサービスのKubernetesへの移行を担当しています。このレポートでは、ミハイルは、クラスターとパイプラインだけでなく、Kubernetesを開始する人が持っているべきスキル、チーム内のプロセスの変革、および妥協点の探求についても話します。

インタラクティブな「開発者の目を通してヘルム。RollingUpdateパズル。 " Dmitry Sugrobov、開発者、Leroy Merlin



Kubernetesは事実上の標準であり、Helmは別のデフォルトの選択肢です。しかし、ここに簡単な質問があります。Helmを使用して起動されたアプリケーションのバージョンを更新するには、何をする必要がありますか？ヘルムアップグレードを呼び出しますか？実際、チュートリアルを超えることは少し価値があります。すぐに多くのニュアンスに出くわします。インタラクティブモードでは、Helmの概要を確認し、そのアーキテクチャとストレージデバイスを確認します。最後に、目的のRollingUpdateを機能させるために何を行う必要があるかという質問に答えます。

Kubernetesプロバイダーとしての進化。Mail.ruクラウドソリューションプロダクトディレクター、Dmitry Lazarenko



Mail.ru Cloud SolutionsがKubernetesで複雑なことがどのように機能するかについて話していることに誰もが慣れています。実際には、Kubernetesを2年以上使用しており、約1年半、お客様へのサービスとして提供してきました。会議の例外を設けました。Kubernetesへの道のり、プロバイダーとしての進化について話し合うことにしました。私たちが経験したこと、私たちが行ったこと、そしてこの複雑なマシンが内部からどのように機能するかです。

海賊行為の精神によるKubernetes。CarPrice、開発責任者、Yuri Buylov氏



今日、GolangとReactで新しいプロジェクトを立ち上げたCarPriceは、自信を持ってKubernetesの舵を握り、象が乗った漁船で「クジラ」を追いかけて溺死しないようにした様子を笑顔で思い出します。この話は、「大きな美しい帆船を運転することではなく、小さな、見苦しい漁船の艦隊についてです-時には錆びているが、速く、とても軽快で危険です。」

プロダクションeldorado.ruでK8をどのように展開したか。Konstantin Rekunov、MIオペレーショングループEldorado.ruの責任者。AGIMA、DevOpsエンジニア、Denis Gurov氏



Eldorado.ruは、比較的小さなクラスターで少数のサービスを使用して、独自のクラウド/ハードウェアでKubernetesを使用していますが、すでに多くの有用なコーンを取得しています。同僚は、ダウンタイムなしでDC間でクラスターを移動する多くの人々にとって興味深いトピックや、問題の運用診断のためのプロセスを構築することが判明した方法など、K8を本番環境で実装した経験を共有します。スピーカーは、ネットワークの問題を解決するための実際的なケースに特別な注意を払いました。これらの問題は、彼らにとって非常に苦痛であり、自明ではないことが判明したためです。

RosgosstrakhでのOpenShiftの実装：DevOpsから本番運用まで。Rosgosstrakh、DevOps責任者Alexander Krylov



戦略的分岐-KubernetesまたはOpenShiftに直面して、Rosgosstrakhの運用は、市場投入までの時間のために後者を選択しました。その結果、OpenShift、Bamboo、Artifactoryを使用してCI / CDを構築し、OpenShiftに基づくサービスの運用を本番環境で確実に実行し、使用されているエンタープライズツールの既存のエコシステムに統合することができました。このスピーチから、同僚が近年経験したさまざまなボックスソリューションの統合の厄介なパスと、それを繰り返すことを望む企業にとっての道の難しさについて学びます。

KubernetesはeBPFとCiliumでネットワークを保護します。カーネルレベルでネットワークを深く操作するにはどうすればよいですか？Alexin Kostrikov、DrainOpsエンジニア、Brain4Net



SDNの会社として、Brain4Netはネットワーク機器だけでなく、ネットワークスタック全体のプログラマビリティを目指しています。Kubernetesは、iptablesルールだけではなく、より柔軟にプログラミングすることもできます。これにより、Ciliumのようなツールを作成できます。CiliumがどのようにeBPFを使用してカーネルコードとユーザー空間を操作するか、またネットワーク、セキュリティ、監視、負荷分散をどのように行うかを見てみましょう。
