脆弱性の最初の警告は、パッチのリリースの前日に配布されましたが、これは標準的ではない理由によるものです。脆弱性を説明する記事は、Cisco TalosとFortinetによって誤って公開されました。これは、明らかに、共同研究プロセスにおけるMicrosoftとの誤解の結果でした。出版物は削除されましたが、予防策として、マイクロソフトはSMBサーバーを保護するための推奨事項を発行しました。 Windows 10（および2つのバージョンのWindows Server）のホールが悪用されて任意のコードが実行される可能性があるため、パッチがリリースされる前に顧客に回避策はありませんでした。

脆弱性は、Windows 10リリース1903や1909などのSMBプロトコルバージョン3.1.1を使用するシステム、および6か月に1回定期的に更新される特別なチャネルを通じて配布される同様のバージョンのWindows Serverの影響を受けます。 Windows Serverの通常のインストール、および以前のユーザーバージョンのWindowsは影響を受けません。明らかに、最近、ファイル共有用のSMBコードに導入されました。公開された

基本的な脆弱性の説明非常に遠隔のフォーティネットとCisco Talosのコンテンツに基づくDuo Security。これは、パッチが適用されていないシステムで任意のコードが実行される原因となるバッファオーバーフローエラーです。 SMBサーバーへの攻撃は、アクセス権を持つ任意のクライアントから実行できます。クライアントソフトウェアを攻撃するには、変更されたサーバーに接続するスクリプトを実装する必要があります。この簡単な説明は、この脆弱性が「ワーム化可能」である可能性があることを示唆しています。つまり、攻撃者はこれを使用して、たとえば企業の境界内など、影響を受けるすべてのシステムに順次感染させます。

したがって、Microsoftの推奨事項：SMBを介したアクセスの制限、外部アクセス用のポートのクローズ。設定は別のドキュメントに記載されています。コンピューターが信頼されていないネットワークに接続されているときに、ポート関連のファイル共有をブロックする。したがって、自宅からのリモート作業や公共のWi-Fiを使用する場合の一般的なシナリオで、従業員のコンピューターへの攻撃のリスクを軽減できます。ファイルサーバーを保護するための最初の推奨事項（データ圧縮を無効にする）は、問題の考えられる原因を示唆しています。
