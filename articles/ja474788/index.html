ララヴェルのルートの整理

こんにちは、ハブル。最近、フリーランスで注文したサイトのレビューに同意しました。私はすべてを実行して200行以上を占有するコントローラー（およびその他の不良コードの兆候）が表示されることを期待していましたが、すべてがかなりまともでした。検証はコントローラーに存在しませんでしたが、これは十分に一般的です。もちろん、PSR-2の不整合が時々ありましたが、ルートを含むファイルを調べるまで、すべてが問題なく見えました。複数の画面を占め、ルートグループについてのコメントがありました。私は開発者に連絡し、彼はすべてが少し「成長した」と言って、それを修正することに同意しました。次のバージョンでは、彼が静的メソッドを使用していくつかのクラスを作成し、その中にコードを転送して、通常行われているように、承認を得てそれらを呼び出したことがわかりました。それから私はすでにこれに遭遇し、そのような議論に会ったことを思い出しました：「ドキュメントには、別のファイルへのルートを取ることについては何も記載されていません。ほんの数分後、記事を見つけて開発者に送りました。 30分後、私は通常の形でルートを受け取り、この記事が多くの初心者に役立つだろうと決めました。したがって、この無料の翻訳が生まれました。

RouteServiceProviderファイルを編集したことがない場合は、catへようこそ。

小さなプロジェクトで作業していて、ルートが2つしかない場合は、問題はありません。ただし、ルートの数が多すぎる場合、特に異なるユーザー、管理者などのページがある場合、1つのファイルでの作業が困難になります。

最初に、laravelは4つのファイルを作成します。

  • api.php
  • console.php
  • channels.php
  • web.php

ユーザーのタイプごとに10ページのプロジェクトを作成するとします。

  • 管理者[サイトの設定、統計など] ]
  • ユーザー[登録、ログイン、プロファイル管理など] ]
  • ゲスト[ブログ、連絡先など] ]

ルート 内に2つのディレクトリを作成します。

  • web-ここに、Webインターフェースに関連付けられたすべてのルートがあります。
  • api-ここにすべてのAPI関連ルートがあります


我々は移動api.phpからのルートにルート/ APIとweb.phpディレクトリ内のルート/ウェブ、及びまま残りconsole.phpとchannels.phpルートでは。

作成admin.phpファイル内のルート/ウェブを。このファイルには、管理者に関連付けられたすべてのWebルートが含まれ、ユーザー関連のルート用にuser.phpが作成されます。



app / Providers / RouteServiceProvider.php-このファイルは、アプリケーションのすべてのルートをダウンロードします。map（）メソッドはmapApiRoutes（）およびmapWebRoutes（）メソッドを呼び出して、すでに移動したweb.phpおよびapi.phpファイルをロードします。そこで、ルートファイルへのパスを修正しましょう。

protected function mapWebRoutes()
{
    Route::middleware('web')
         ->namespace($this->namespace)
         ->group(base_path('routes/web/web.php'));
}

protected function mapApiRoutes()
{
    Route::prefix('api')
         ->middleware('api')
         ->namespace($this->namespace)
         ->group(base_path('routes/api/api.php'));
}

RouteServiceProvider.php内にroute / web / admin.phpおよびroutes / web / user.phpの 新しいメソッドを作成します

protected function mapAdminWebRoutes()
{
    Route::middleware('web')
        ->namespace($this->namespace)
        ->prefix('admin')
        ->group(base_path('routes/web/admin.php'));
}

protected function mapUserWebRoutes()
{
    Route::middleware('web')
        ->namespace($this->namespace)
        ->prefix('user')
        ->group(base_path('routes/web/user.php'));
}

このコードでは、パスに名前空間、ミドルウェア、プレフィックスなどを追加できることに注意してください。

次に、マップ（）から呼び出すだけです。

public function map()
{
    $this->mapApiRoutes();
    $this->mapWebRoutes();
    $this->mapAdminWebRoutes();
    $this->mapUserWebRoutes();
}

最後のステップ：

ルート/ web / user.phpを開き、テストルートを追加します。

Route::get('/test', function () {
    return response(' ', 200);
});

site.local / user / testに移動すると、「Test Route」というテキストが表示されます。
