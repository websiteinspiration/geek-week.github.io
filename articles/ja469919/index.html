PVS-StudioアナライザーによるTelegram Open Networkコードの検証


Telegram Open Network（TON）は、Telegram messengerの作成者によるプラットフォームであり、ブロックチェーンに加えて、多数のサービスが含まれています。開発者は最近、C ++で記述されたプラットフォームコードを公開し、GitHubに投稿しました。正式にリリースする前にプロジェクトを確認したかったのです。

前書き


Telegram Open Networkは、さまざまなサービスのセットです。たとえば、プロジェクトにはグラム暗号化に基づく独自の支払いシステムがあり、仮想マシンTON VMもあります-スマートコントラクトが機能します。メッセージングサービス-TONメッセージがあります。プロジェクト自体はインターネット検閲と戦うことを目的としています。

プロジェクトはCMakeアセンブリシステムを使用しているため、アセンブリと検証に特別な問題はありませんでした。コードはC ++ 14で記述され、21万行あります。


プロジェクトは小規模で高品質であるため、エラーは多くありませんでしたが、注意して修正する価値があります。

int compute_compare(const VarDescr& x, const VarDescr& y, int mode) {
  switch (mode) {
    case 1:  // >
      return x.always_greater(y) ? 1 : (x.always_leq(y) ? 2 : 3);
    case 2:  // =
      return x.always_equal(y) ? 1 : (x.always_neq(y) ? 2 : 3);
    case 3:  // >=
      return x.always_geq(y) ? 1 : (x.always_less(y) ? 2 : 3);
    case 4:  // <
      return x.always_less(y) ? 1 : (x.always_geq(y) ? 2 : 3);
    case 5:  // <>
      return x.always_neq(y) ? 1 : (x.always_equal(y) ? 2 : 3);
    case 6:  // >=
      return x.always_geq(y) ? 1 : (x.always_less(y) ? 2 : 3);
    case 7:  // <=>
      return x.always_less(y)
                 ? 1
                 : (x.always_equal(y)
                        ? 2
                        : (x.always_greater(y)
                               ? 4
                               : (x.always_leq(y)
                                      ? 3
                                      : (x.always_geq(y)
                                            ? 6
                                            : (x.always_neq(y) ? 5 : 7)))));
    default:
      return 7;
  }
}

最終的に


TONコードはそれほど大きくなく、エラーは多くありません。もちろん、それについては、テレグラムチームのプログラマに敬意を表する価値があります。ただし、エラーが発生しても問題はありません。コードアナライザーは、高品質のコードベースであっても初期段階で危険な場所を見つけることができる強力なツールであるため、その使用を無視してはなりません。静的分析は1回限りのチェックのためのツールではなく、開発プロセスの一部です：「プロセスに静的分析を埋め込み、それでバグを探しません」。
