UNIX v0からPDP-7への復元：バックルームの詳細


シリアル番号129のPDP-7の前にいる

フレッドヤリャン元ボーイングのエンジニアであるフレッドヤリャンと、リビングコンピューターのエンジニアであるジェフケイリンに会って、リビングコンピューター：博物館+ラボのラボでPDP-7を復元する作業について話し合いました。 PDP-7（Programmed Data Processor、「programmable data processor」）は、1964

年にDigital Equipment Corporation（DEC）によって導入されました。1960年代初頭、「コンピュータ」という用語は、理解しづらかった巨大で高価なマシンと同義でした。 。 DECは、このような否定的な意味合いを避けるために、マシンにPDPという名前を付けました。

DECは、特にベルの研究所、太平洋岸北西海岸国立研究所、およびジェット推進研究所で、PDP-7の購入者を全国的に見つけました。 PDP-7は大学でも人気があり、特にオックスフォード大学、MIT、ヘルシンキ工科大学、東京大学に送信されました。

今日、PDP-7を正常な状態で見つけることは非常にまれな成功です。長年の間、世界には4つしかないと信じられていました。これらのうち2つは米国外にあり、1つはノルウェーのオスロで修復され、もう1つはオーストラリアの個人コレクションでした。

3つ目はカリフォルニア州マウンテンビューのコンピューター歴史博物館の倉庫にあり、4つ目はシアトルのリビングコンピューター：博物館+実験室にありました。

後者はもともとオレゴン大学の核物理学科に設置されていました。パッケージを受け入れて、Harlan Lefebvre教授と彼の研究室は3年間、彼らの研究を実行するために必要なソフトウェアを書きました。この作業が完了すると、システムは非常に信頼できることが証明されました。次の40年間で60,000時間以上稼働し、23人の大学院生が博士号を取得できるようになりました。

ヤーリャンの所有物で別の車が発見される前、最後のPDP-7は世界で唯一のものと考えられていました。笑顔のイアヤンは、自分の話を初めて聞いたとき、LCM + L博物館に初めて行ったときを思い出します。

「私は博物館に行き、誰かが私に言った：「ああ、これが唯一の実用的なPDP-7です。」そして私は答えた：まあ、実際には、私は地下室でも同じです！

ワシントン大学の電気機械学部を卒業したYaryanは、数十年間、太平洋岸北西部沿岸の国立研究所で電気技師として働いていました。彼は彼のキャリアのかなりの部分をボーイングで過ごし、ケント宇宙センターで宇宙飛行士を訓練するために飛行シミュレーションを開始するのを手伝いました。この間、彼はSDS 930、Varian Data Machines、Tektronixなどのさまざまなコンピューティングシステムを完全にマスターしました。

彼自身が言うように、Earyanを1973年にボーイング開発センターに導いたのは、彼の幅広い知識でした。

この訪問中に、彼は車の中にPDP-7があることに気づきました。どうやら、それはローカルSDS 940に接続され、システム全体が処理されたデータを示すように設計されました。

彼は数年後の1979年にボーイングが販売する余剰店舗で同じPDP-7を見つけることを知りませんでした。彼はプロジェクトのためにワイヤーやその他のものを購入するためによくそこに行きました。

彼が初めて店にPDP-7について尋ねたとき、誰かがペーパーロール印刷システムを含む部品のためにそれを分解したかったので、それは売り物ではないと言われました。しかし、次の訪問の1つで、彼はコンピュータがまだそこに立っていることに気づきました。彼は再び売り手に頼り、最終的に500ドルでこのシステムを購入し、ピックアップトラックに積んで自宅に持ち帰ることができました。

PDP-7はYaryanの地下室に定住し、この移動の後、新しい所有者はコンピューターで問題に遭遇しました。彼はすぐにこれらの問題の原因を発見しました。彼自身が言うように、「問題は定期的に発生しました。 PDP-7は非常に長い間ボーイングに立っていました、そして彼らはその原因を見つけることができませんでした。古いTektronix 45でデバッグしたところ、ターミネーターが間違った場所にあったことがわかりました。」

PDP-7を動作させた後、Yaryanは彼のために別のコンピューターのBASICでプログラムを作成し、それをこのコンピューターにダウンロードして、すべてが正しく動作していることを確認しました。そして、Yaryanが2017年3月にLCM + L博物館を初めて訪問する前に25年が経過したその瞬間から。

この訪問中にYaryanがガイドと話し、自宅にPDP-7があることを彼らに伝えた後、エンジニアのチーム博物館は彼に連絡を取った。

「以前はすでにこの情報に懐疑的でした。以前は博物館に来て、そのようなものがあると言ったことがありますが、結局それは何かではないことがわかりました。まったく機能していません」と博物館のマネージングディレクターであるスティーブンジョーンズは言いました。 「しかし、彼の最初の訪問中にフレッドと話し合った後、私たちは彼が完全な、そしておそらくは作動する機械を持っていることを確認しました。」

この訪問から1年以上経過した2018年11月7日、Jeff Kailin、Cindy Moyu、Stephen Jonesを含む数人の博物館エンジニアがYaryanの家に到着し、PDP-7をワークショップから取り出して、博物館に向かうピックアップトラックに積み込みました。

まもなく、チームはUNIXバージョン0オペレーティングシステムをマシンにロードする作業を開始しました。そうすることで、彼らはケントンプソンが1969年にベルの研究所の古いPDP-7マシンでUNIXバージョン0を書いたという事実に敬意を表したいと思いました。今年、UNIXは50周年を迎え、復旧プロジェクトをタイムリーに行います。

UNIXの読み込みに加えて、エンジニアはPDP-7に直接メモリアクセスとI / Oインターフェイスを追加するように働きました。これにより、データをメインメモリに転送できます。

これは、ボーイングの研究室でこのコンピュータ用に特別に作られた装置に接続されたディスクエミュレータJK09を介して行われました。カーネルが新しいドライブを使用できるように、デバイスドライバー（おそらく、過去45年間で最初のUNIXバージョン0の新しいドライバー！）を追加しました。

そして数え切れないほどの作業時間の後、チームは最初に2019年10月28日月曜日にYaryanが所有するPDP-7にUNIX v0をアップロードしました。1969年にケントンプソンとUNIXを作成したDennis McAleystair Ritchieに敬意を表して、エンジニアはユーザー名dmrでログインしました。


メッセージを含むPDP-7出力：「UNIX v0を実行するPDP-7からの挨拶」

会話の最後に、ヤリャンは自分のPDP-7のレガシーをどのように考えているか、そしてコンピューターの歴史の別の例を人々に示す方法を説明しました。私のキャリアには多くの興味深い話がありました。

今日、訪問者はこのPDP-7と復活プロジェクトをLiving Computers：Museum + Labsで見ることができます。
