Kelola beberapa JDK di Mac OS, Linux, dan Windows WSL2

Halo lagi. Untuk mengantisipasi dimulainya kursus Pengembang Java, kami telah menyiapkan untuk Anda terjemahan dari artikel yang menarik.





Jika Anda, seperti saya, harus bekerja secara bersamaan dengan beberapa proyek (baik yang lama maupun yang modern), dan mungkin Anda sedang mengembangkan semacam perpustakaan atau utilitas untuk pengguna dengan versi Java yang berbeda, maka Anda akan memahami masalah peralihan di antara berbagai JDK.







Saat ini, menginstal, mengelola, dan beralih antar JDK selama pengembangan menjadi tugas yang menakutkan karena sejumlah besar pengembang masih bekerja dengan Java 8, meskipun banyak yang sudah bermigrasi ke Java 11. Masalah ini dapat diselesaikan dengan berbagai cara. Pada artikel ini kita akan membahas beberapa di antaranya.



Cara manual



Anda dapat mengunduh distribusi dari situs vendor JDK dan menginstalnya semua secara manual (atau cukup buka ritsletingnya ke beberapa jenis folder seperti itu $user/jdks



) dan itu saja. Tetapi kemudian Anda harus memeriksa dan memperbarui setiap waktu JAVA_HOME



sehingga JDK yang Anda butuhkan ada di sana.



Ini bisa dilakukan menggunakan skrip bash, fungsi bash, dll. Apa masalahnya? Anda bisa saja lupa memanggil skrip / fungsi dan menyadari bahwa ada yang tidak beres hanya ketika Anda melihatnya UnsupportedClassVersionError



.



Jika Anda lebih suka melakukan semuanya secara manual, Anda dapat menggali melalui Internet untuk menemukan solusi. Dan ini bisa menjadi pilihan yang baik untuk pengguna tingkat lanjut. Tapi saya menggunakan beberapa utilitas untuk ini. Jadi mari kita lihat mereka.



Jalan terbaik



Menurut pendapat saya, solusi ideal adalah dengan menggabungkan dua alat open source yang telah ada selama beberapa waktu, dan mungkin Anda akrab dengan setidaknya satu dari mereka. Tetapi yang tidak Anda ketahui adalah bahwa mereka dapat digunakan bersama.



  • SDKMAN!
  • jEnv


Mari kita lihat sharing mereka.



Instal beberapa JDK





Setelah menginstal SDKMAN! Tipe: 



$ sdk list java


Temukan vendor, versi, dan masukkan: 



$ sdk install java <id>


Mari kita instal empat JDK: dua versi terakhir Oracle OpenJDK dan dua versi LTS dari AdoptOpenJDK.



Saat memasang, jangan pilih JDK default.



  1. AdoptOpenJDK membangun OpenJDK 8u252 - LTS
  2. AdoptOpenJDK membangun OpenJDK 11.0.7 - LTS
  3. Oracle OpenJDK 14 - GA terbaru
  4. Oracle OpenJDK 15 - Build Akses Awal


Keempat versi ini harus memenuhi kebutuhan Anda untuk proyek Java 8 lawas, untuk proyek Java 11 modern, dan untuk bereksperimen dengan inovasi di Jawa 14 dan Jawa 15.



Hebat. Anda memiliki empat JDK yang diinstal secara lokal menggunakan SDKMAN! Saya pikir utilitas ini sangat mudah untuk menginstal JDK dan, saya harap, Anda menyukainya juga.



Semua JDK yang terinstal dapat ditemukan di direktori berikut: 



$ cd /Users/bruno/.sdkman/candidates/java
$ ls
11.0.7.hs-adpt  14.0.1-open     15.ea.20-open   8.0.252.hs-adpt


Jika Anda tidak perlu sering beralih di antara JDK yang berbeda, maka ini sudah cukup. Anda bisa menggunakan SDKMAN! untuk memilih JDK: 



$ sdk current java
Using java version 15.ea.19-open
$ sdk default java 15.ea.20-open
Default java version set to 15.ea.20-open


Anda dapat berhenti pada ini dan hanya menggunakan SDKMAN !, tetapi cepat atau lambat, Anda akan lupa untuk mengganti JDK dengan cara yang sama seperti dengan pendekatan manual. Oleh karena itu, saya lebih suka beralih otomatis JDK.



Kelola beberapa JDK dengan jEnv







Terlepas dari kenyataan bahwa SDKMAN! Adalah nyaman untuk menginstal JDK, tetapi itu tidak membantu ketika berpindah secara otomatis antara JDK yang berbeda ketika berpindah dari satu proyek ke proyek lainnya. Di sini jEnv akan membantu kami .



Setelah Anda menginstal beberapa JDK menggunakan SDKMAN! atau dengan cara lain, Anda perlu menambahkannya ke jEnv: 



$ cd ~/.sdkman/candidates/java
$ ls -1
11.0.7.hs-adpt 
14.0.1-open    
15.ea.20-open
8.0.252.hs-adpt
$ jenv add 15.ea.20-open
openjdk64-15-ea added
15-ea added
15-ea added
...
$ jenv versions
 system
 1.8
 1.8.0.252
 11.0
 11.0.7
 14.0
 14.0.1
* 15-ea (set by /Users/bruno/.jenv/version)
 openjdk64-1.8.0.252
 openjdk64-11.0.7
 openjdk64-14.0.1
 openjdk64-15-ea


Ulangi perintah jenv add



untuk tiga versi JDK yang tersisa.



jEnv sangat nyaman bagi mereka yang terus menggunakan baris perintah. Dia akan membiarkan Anda:



  1. Instal versi Java secara global untuk sistem Anda.
  2. Instal versi Java untuk direktori / proyek saat ini di mana Anda berada.
  3. Instal versi Java untuk shell saat ini.


jEnv menggunakan shim dan secara otomatis mengelola variabel lingkungan JAVA_HOME. Dengan demikian, setelah Anda mengkonfigurasi sistem Anda, jEnv akan mengalihkan versi JDK ke versi yang paling sesuai, berdasarkan prioritas di atas dan direktori saat ini. Keren! Ini memiliki fitur berguna lainnya - lihat situs web untuk informasi lebih lanjut .



Beralih otomatis antara JDK



Anda sekarang sudah menginstal Java 8, 11, 14, dan 15-EA. Jadi bagaimana Anda beralih di antara mereka? Mari sesuaikan versi dan uji sakelar.



Pertama, Anda perlu menentukan JDK global default dalam sistem. Untuk ini, saya biasanya menggunakan Early Access. Setiap kali saya mulai bekerja dengan proyek baru, saya secara otomatis menggunakan rilis yang akan datang, yang membantu saya mengidentifikasi potensi masalah yang dapat saya laporkan ke proyek OpenJDK. Kamu juga melakukannya ... 



$ jenv global 15


Sekarang untuk folder di mana Anda berada, setelah eksekusi java -version



Anda akan menerima OpenJDK 15 EA.



Jika Anda memiliki proyek yang harus menggunakan Java 8, pergi ke folder dan jalankan: 



$ jenv local 1.8




Akibatnya, file dibuat dengan nama .java-version



dengan konten berikut: 



$ cat .java-version
1.8


File ini memberi tahu jEnv JDK mana yang akan digunakan ketika Anda berada di folder ini.



Akhirnya, jika Anda ingin beralih sementara ke versi lain dari JDK, maka gunakan jenv shell



.



Ada demo di sini yang dapat menjelaskan semua hal di atas dan bagaimana pengalihan otomatis bekerja.



Setelah artikel itu diterbitkan di Reddit, ada beberapa komentar menarik yang layak disebutkan di sini:



  • SDKMAN! dari versi 5.8.1+ mendukung sdk env dan peralihan otomatis juga tersedia di sana.
  • direnv sangat bagus dan tidak terikat ke java. Itu melakukan hal yang sama seperti jEnv, kecuali itu tidak membantu Anda beralih JDK dengan mudah. Anda harus mengedit file secara manual.
  • autoenv adalah alternatif direnv lain dengan keterbatasan yang sama.
  • jabba adalah alternatif paling menjanjikan untuk kombinasi SDKMAN + jEnv, karena mendukung semua fungsi mereka dan berfungsi di Windows (PowerShell), yang tidak didukung oleh alat-alat lain yang disebutkan di atas.
