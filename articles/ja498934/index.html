PHPを使用して1分に複数回c​​ronを実行する方法

Linuxオペレーティングシステムでのcronジョブエントリの従来の設定ファイルは次のとおりです。

# ┌─────────────  (0 - 59)
# │ ┌─────────────  (0 - 23)
# │ │ ┌─────────────  (1 - 31)
# │ │ │ ┌─────────────  (1 - 12)
# │ │ │ │ ┌─────────────   (0 - 6)
# │ │ │ │ │
# * * * * * <  >

最初の5つのパラメーターはこのタスクの実行時間を示し、6番目のパラメーターは実行する必要があるコマンド自体です。時間パラメーターは、分、時間、日、月、および曜日です。さらに、すべての数値は整数または特別な構文の形式で表す必要があります。

その結果、コマンドを起動するための可能な最小間隔は1分に1回です。

多くのタスクでは、コマンドの実行がより頻繁に必要になります（たとえば、10秒ごと）。一部のビジネスプロセス自動化タスクでは、最大許容遅延は多くの場合1〜1.5秒です。

もちろん、従来のcronはこれには適していません-改善する必要があります。

以下は、プロセスの再起動に対する追加の保護を使用して、Linuxのクラシックcronに追加機能（PHPで）を作成するステップバイステップの実装です。

タスク設定とcronセットアップ


たとえば、次のタスクを使用します。

  1. フロントエンドでは、ユーザーは[実行]ボタンをクリックして、複雑なタスクの実行を開始できます。
  2. データベースに新しい行を書き込んだ後のバックエンドは、ユーザーに確認を伝えます。
  3. cronを使用して、そのような新しいタスクを「追跡」し、ユーザーが1分以内に結果を得るのではなく、すぐに*を取得できるように、それらをできるだけ早く完了します。

*コマンドの起動を1分ごとに使用する場合、秒が最も近いゼロ（新しい分の始まり）に達するとタスクが開始されます。したがって、クラシック形式では、ユーザーは実行を0〜59秒待つ必要があります。

したがって、cronは最大形式で構成されます。毎分：

* * * * * /usr/bin/php -q /run.php > /dev/null 2>&1
#/usr/bin/php -    PHP   (      )
#/run.php -   PHP   
#> /dev/null 2>&1 - ,            run.php

シングルサイクル


最初に、データベースで新しいタスクを要求する頻度を決定する必要があります-これに応じて、サイクル数と論理スリープ（関数sleep）が変化します。

現在の例では、10秒のステップが使用されています。したがって、サイクル数は60/10 = 6です。合計、一般的なコードは次のとおりです。

for ($cycle = 1; $cycle <= 6; $cycle++) { # 6 
    $all_tasks = get_all_tasks(); #     
    if ($all_tasks) { #    - 
        foreach($all_tasks as $one_task) { #     
            solve_one_task($one_task); #  
        }
    }
    sleep(10); #  ,  ""  10 
}

明確化：この例では、10秒のステップが使用されます。これにより、10秒ごとに1回、スクリプトの最小実行間隔を提供できます。したがって、より頻繁に実行するには、サイクル数と「スリープ時間」を変更する必要があります。

タスクの繰り返し実行を回避する


提示された形式では、1つの不確実性があります。つまり、タスクがすでに開始されている場合、タスクの繰り返し実行です。これは、タスクが「困難」であり、実装に数秒かかる場合に特に当てはまります。

これにより、再実行の問題が発生します。

  • 関数はsolve_one_task()すでに実行されていますが、まだその作業は完了していません。
  • したがって、データベースでは、タスクはまだ未達成としてマークされています。
  • 次のサイクルでこのタスクが再度取得さsolve_one_task()れ、同じタスクで関数が再度実行されます。

もちろん、これは、たとえば、このタスクのためにデータベースのステータスを変更することで解決できます。

しかし、データベースはロードしません。私のテストに基づいて、MYSQLはリクエストを受け入れることができますが、すぐには処理できません。0.5秒でも差があると、実行が繰り返される可能性があります。これは、明らかに適切ではありません。

また、この場合はタスクのステータスについてのみ話しているため、サーバーのファイルシステムを使用することをお勧めします。

基本的な検証モデルはflock、ファイルを設定およびロック解除する機能を使用して構築されています。

PHPの実行では、関数は次のように表すことができます。

$lock_file_abs = 'file'; #  
$fp = fopen($lock_file_abs,"w+"); #   
if (flock($fp, LOCK_EX | LOCK_NB)) { #,    
    solve_one_task($one_task); #  
    flock($fp, LOCK_UN); #   ,     
}
else {
    #,   , ..       
}
fclose($fp); #   
unlink($lock_file_abs); # ,

結果


サイクル全体の概観は次のとおりです。

for ($cycle = 1; $cycle <= 6; $cycle++) {
    $all_tasks = get_all_tasks();
    if ($all_tasks) {
        foreach($all_tasks as $one_task) {
            $lock_file_abs = __DIR__.'/locks/run_'.$one_task['id'];
            $fp = fopen($lock_file_abs,"w+");
            if (flock($fp, LOCK_EX | LOCK_NB)) {
                solve_one_task($one_task);
                flock($fp, LOCK_UN);
            }
            else {
                #    
            }
            fclose($fp);
            unlink($lock_file_abs);
        }
    }
    sleep(10);
}

したがって、このようなアルゴリズムを使用すると、タスク処理の周期的な実行を開始でき、タスクが複数回処理されることを心配しません。

そして今度は、このコードを毎分cronで実行するだけで十分であり、その結果、すでに内部でより小さなサイクルが実行されます。
