Data preparation for tuning and training a neural network for object detection.



We continue the topic of object detection . In order for the algorithm to distinguish between objects or to be able to detect deviations in the process, it must be trained on a data set that corresponds to the overall picture of the analyzed video materials. Preparing a training set is a very time-consuming process, which depends on both the speed of training and the quality of the algorithm. Consider the use of several tools that will help you prepare the data correctly.



The first stage of preparation is the compilation of an initial data set. As a rule, the larger the volume of data and the higher the diversity of the analyzed object, the higher the quality of the algorithm. If you intend to analyze video from DVRs, then compiling a data set is preferable to take screenshots from existing video materials, because in this case, less data and less time for retraining are required. In other cases, the data set is compiled taking into account the viewing angle from which the object is captured, its size, etc., how much the objects from the training set are similar to objects from materials that will be analyzed by the algorithm. The data set must be composed of various variations, i.e. on which both one object and several will be displayed, for example, a person and a phone in one frame.



If the compilation of the data set was carried out using a camera or phone, most likely the images will have a high resolution, which negatively affects the learning process. The main problem with high resolution images will be the ‚ÄúOut of memory‚Äù error, even if the batch_size parameter is small. To reduce the likelihood of this error, it is necessary to reduce the size of the images. The IrfanView program will help us with this. This program allows you to batch process images, i.e. you just need to specify the path to the source images, set the conversion parameters and start the conversion process. As a result, we get ‚Äúlite‚Äù images in the specified folder. To do this, in the start window of the program, press the B key (Russian layout "I") or in the File tab, select "Batch Conversion / Rename",a window appears as follows:

  4. add all images;

Formation of a training sample is a rather tedious process, whether it is the formation of screen shots from video materials or shooting with a camera and phone. To significantly increase the volume of the data set, various augmentation methods are used. Let's look at a few of these methods implemented in the Python programming language using the Pillow library. One of the most popular augmentation methods is ‚Äúmirror reflection‚Äù.



To begin, we import the necessary libraries: 



from PIL import Image, ImageOps
import os
import numpy


We indicate the location of the directory with the source images and the path for the conversion result: 



path_to_origin = "images/" 
output_folder = "Augmentation_result/"
img_names = os.listdir(path_to_origin)

if not os.path.exists(output_folder):
    os.makedirs(output_folder)


We will write a function to convert the image to ‚Äúmirror image‚Äù: 



def mirror(path_to_origin, img_name, output_folder):
    im = Image.open(path_to_origin + img_name)
    ImageOps.mirror(im).save(output_folder+"mirror"+img_name, "JPEG")


Another popular augmentation method is image rotation. It is worth considering with full-sized objects that with this method of augmentation, part of the image is lost. We write a function to rotate the image: 



def rotate(path_to_origin, img_name, output_folder):
    im = Image.open(path_to_origin + img_name)
    im.rotate(-30).save(output_folder+"rotate_"+img_name, "JPEG")


The value (-30) is the angle of rotation of the image, which is selected based on the type of object. For example, a 180-degree rotation, if there is a phone in the image, then this is a good option for augmentation, but if there is a person in the image, then 180 degrees for this image is a bad example, because you are unlikely to see a video in which people walk upside down.



And finally, we will spend 2 types of augmentation for each image: 



for img in img_names:
    rotate(path_to_origin, img, output_folder)
    mirror(path_to_origin, img, output_folder)

There are many different augmentation methods (adding noise, changing contrast, etc.), it all depends on the type of data being processed. Thus, using two augmentation methods, we increased the size of the training data three times. The formation of the training set is completed, it remains to mark the objects in the images.



For markup we will use the LabelImage program, since in my opinion it is quite easy to use and multifunctional. This program is able to return the markup result in two formats (PascalVOC and YOLO). The first format is presented in the form of an XML file in which the image parameters are indicated, as well as the coordinates of the rectangles that describe the location of objects in the image:







Such an XML file is generated for each image in which all marked objects are described. The markup file name is the same as the image name. The YOLO format is presented in the form of a text file in which the class of the object and the coordinates of the rectangles are described, described by the values ‚Äã‚Äãx, y, w, h in normalized form:







In addition to a text file with markup for the YOLO format, a classes.txt file is also generated, in which the names of the objects are indicated. The sequence numbers of the names of objects depend on the order of appearance when marking images. It is worth noting that the classes.txt file can be overwritten when you reopen the application, so you need to follow the sequence of entering names in the program, otherwise coordinate inconsistency will occur. The markup of objects occurs in the following sequence:







  1. Select a directory with images;
  2. Select the directory in which markup files will be saved;
  3. Choose a layout format (PascalVOC or YOLO);
  4. Select the ‚ÄúW key‚Äù object selection tool;
  5. Select an object in the image;
  6. Set the name of the object (during repeated operation, you can select from the list, under the OK button);
  7. Confirm
  8. Save ‚ÄúCtrl + s‚Äù (repeat steps 4-8 for each required object);
  9. Go to the next image.


Upon completion, the number of markup files should correspond to the number of images with identical names. Thus, we examined the tools and methods that allow you to resize images, increase the amount of data, as well as mark objects on images that will be detected by a neural network. The formation and preparation of data is an important and time-consuming step on which success in creating the deep learning algorithm depends.
