パートIII：スマートフォンのDOSエミュレーターを使用してGPSS / PCを起動する


さて、今はMinuteman Softwareからの作成を検討する時が来ました。

GPSS / PCから始めます。これはDOS用のプログラムの古いバージョンです。



私が最初に彼女に会ったとき、プログラムが始まり、それについて考え、PCスピーカーを使用して音を出し、しばらくの間、Sierraのロゴが表示され、子供の頃にお気に入りのSpace Qurst and Leisure Suitラリーに固執するように思えました。



しかし、執着はすぐに過ぎ去った。起動画面の後は、インタラクティブコンソールのみが表示されます。



このコンソールでは、ファイルを作成せずにコマンドをすぐに入力できます。

しかし、コマンドを少し変わった方法で導入することをお勧めします。

私たちの場合、GPSS / PCのシンプルさには1つの大きなプラスがあります。

どのMS-DOSエミュレータでも簡単に実行できます。では、スマートフォン（私の場合はAndroidプラットフォーム）でパフォーマンスを確認してみませんか？

エミュレータをインストールします。 aFreeBox（無料のDOSbox）をインストールしましたが、他のものを選択できます。

私の場合、エミュレータのルートフォルダは、スマートフォンファイルシステムのルートにあるダウンロードフォルダでした。 GPSS / PCからこのフォルダーにファイルを転送しました。



プログラムを開始します（側近のPCスピーカーシミュレーションを有効にすることを忘れないでください）。



これでコマンドを入力できます。

最初に行番号を入力し、スペースバーを押します。コマンドのラベルがない場合は、もう一度スペースバーを押してコマンドの名前を入力します。

ちなみに、自動コード補完を使用することができます-提案されたコマンドの数文字を入力してスペースバーを押します。コンピュータは最も適切なコマンドを印刷します。

コマンドを入力した後も、パラメータを入力してEnterキーを押します。

直接モデリングに移りましょう。

GPSSを使用する場合、重要な要素は想像力であることを覚えておくことが重要です。

たとえば、この最も単純なモデルは次のとおりです。

	
        GENERATE    4,1     ;Create next customer.
	QUEUE       Larek    ;Begin queue time.
	SEIZE       Larek     ;Own or wait for trader.
	DEPART      Larek   ;End queue time.
	ADVANCE     5,2     ;Shopping takes a few minutes.
	RELEASE     Larek   ;Shopping done. Give up the trader.
	TERMINATE   1       ;Customer leaves.

あなたの想像力のおかげで、それは次のようになる可能性があります：回転式改札口の通過のモデル、1人のマスターがいる美容師のモデル、さらにはマシンのセットのモデル、ダンプトラックショベル。



記事のタイトル画像を正当化し、上記のコードがストール内のキューをシミュレートすると想像してください。

時間単位は分として解釈されます。

念のため、ブロックの値を思い出させます：

GENERATE 4,1-システム内にオブジェクトを作成します。私たちの場合、これは屋台に並んで立つ人です。人々は、-1分から+1分までの範囲の偏差で、4分ごとに一度屋台に来ます。
QUEUE -男は屋台に並んでいた。
SEIZE-男は屋台の窓に行った。
DEPART -男は頭を屋台の窓に突き刺しました。つまり、彼は列から削除されるべきです。
ADVANCE 5,2-商品を選択する時間は5±2分です
RELEASE-担当者は販売員（販売者）を解放しました。
TERMINATE 1-人は列を離れ、希少品で家に帰ります。

シミュレーションを開始します。



コマンドを選択しSTART



、値100を入力します。この場合、100人の顧客を「購入」するとすぐに、シミュレーションが完了します。



シミュレーションが完了すると、レポートウィンドウが表示されます。
