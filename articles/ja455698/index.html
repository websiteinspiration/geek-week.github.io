コンストラクタ


第13章では、ファクトリー（関数を返す関数）を使用しました。これで、コンストラクター（関数を含むオブジェクトを返す関数）で同様のことができます。

まず、counterジェネレーターと同様に、counter_constructorを作成します。upとdownの2つのメソッドがあります。

function counter_constructor() {
      let counter = 0;

      function up() {
            counter += 1;
            return counter;
      }

      function down() {
            counter -= 1;
            return counter;
      }

      return Object.freeze({
            up,
            down
      });
}

返されたオブジェクトは凍結されています。破損したり破損したりすることはできません。オブジェクトには状態があります。変数counterは、オブジェクトのプライベートプロパティです。メソッドを通じてのみアクセスできます。そして、これを使用する必要はありません。

これは非常に重要な状況です。オブジェクトインターフェイスは、メソッドのみです。彼は非常に強い殻を持っています。最高のカプセル化を実現します。データへの直接アクセスはありません。これは非常に高品質のモジュール設計です。

コンストラクターは、オブジェクトを返す関数です。パラメータとコンストラクタ変数は、オブジェクトのプライベートプロパティになります。データで構成されるパブリックプロパティはありません。内部関数はオブジェクトメソッドになります。それらはプロパティを閉じたものに変えます。フリーズしたオブジェクトに分類されるメソッドは開いています。

メソッドはトランザクションを実装する必要があります。たとえば、personオブジェクトがあるとします。データが保存されている人の住所を変更する必要がある場合があります。これを行うには、個々の住所要素を変更するための個別の関数セットは必要ありません。変更する必要のある住所のすべての部分を記述できるオブジェクトリテラルを受け取る1つのメソッドが必要です。

JavaScriptの優れたアイデアの1つは、オブジェクトリテラルです。これは、情報をクラスタリングするための優れた表現力のある構文です。データオブジェクトを使用および作成するメソッドを作成することにより、メソッドの数を減らし、オブジェクトの整合性を高めることができます。

2種類のオブジェクトがあることがわかります。

  • ハードオブジェクトにはメソッドのみが含まれます。これらのオブジェクトは、クロージャーに含まれるデータの整合性を保護します。それらは私たちに多態性とカプセル化を提供します。
  • ソフトデータオブジェクトにはデータのみが含まれます。彼らは行動しません。これは、関数を操作できる便利なコレクションです。

OOPは、Kobol言語のレコードに手順を追加することから始まり、それによって何らかの動作が保証されると考えられています。データメソッドとプロパティの組み合わせは重要な一歩だったと思いますが、最後のステップではありません。

ハードオブジェクトを文字列に変換する必要がある場合は、toJSONメソッドを有効にする必要があります。それ以外の場合、JSON.stringifyはそれを空のオブジェクトとして認識し、メソッドと非表示データを無視します（第22章を参照）。

コンストラクタオプション


一度、10個の引数を取るコンストラクタを作成しました。議論の順序を覚えることができなかったので、それを使用することは非常に困難でした。後で、2番目の引数を誰も使用していないことがわかりました。パラメーターのリストから2番目の引数を削除したかったのですが、すでに開発されたすべてのコードが壊れてしまいます。

私が賢明だったら、1つのオブジェクトをパラメーターとしてとるコンストラクターができたでしょう。通常はオブジェクトリテラルから取得されますが、JSONコンテンツなどの他のソースから取得することもできます。

これには多くの利点があります。

  • キーラインは、コードに文書化された外観を与えます。呼び出し元の各引数が何であるかがわかるので、コードは読みやすくなります。
  • 引数は任意の順序で配置できます。
  • 将来的には、既存のコードを損なうことなく新しい引数を追加できます。
  • 無関係なパラメータは無視できます。

ほとんどの場合、パラメーターはプライベートプロパティを初期化するために使用されます。これは次のように行われます。

function my_little_constructor(spec) {
      let {
           name, mana_cost, colors, type, supertypes, types, subtypes, text,
           flavor, power, toughness, loyalty, timeshifted, hand, life
      } = spec;

このコードは、仕様にある同じ名前のプロパティを使用して、15個のプライベート変数を作成および初期化します。specに対応するプロパティがない場合、新しい変数が初期化され、値undefinedが割り当てられます。これにより、すべての欠落値をデフォルト値で埋めることができます。
