Mendengus atau Suricata. Bagian 3: melindungi jaringan kantor

Dalam artikel sebelumnya, kami berbicara tentang cara menjalankan versi stabil Suricata di Ubuntu 18.04 LTS. Mengkonfigurasi IDS pada satu node dan menghubungkan set aturan bebas cukup mudah. Hari ini kita akan melihat bagaimana melindungi jaringan perusahaan menggunakan Suricata yang diinstal pada server virtual, yang merupakan jenis serangan yang paling umum. Untuk ini, kita perlu VDS di Linux dengan dua core. Jumlah RAM tergantung pada beban: 2 GB sudah cukup untuk seseorang, dan untuk tugas yang lebih serius mungkin butuh 4 atau bahkan 6. Ditambah mesin virtual dengan kemungkinan eksperimen: Anda dapat mulai dengan konfigurasi minimal dan menambah sumber daya sesuai kebutuhan.

Menghapus IDS ke mesin virtual mungkin diperlukan terutama untuk pengujian. Jika Anda belum pernah berurusan dengan solusi seperti itu, Anda tidak perlu terburu-buru memesan perangkat keras fisik dan mengubah arsitektur jaringan. Lebih baik menjalankan sistem dengan aman dan tanpa biaya tambahan untuk menentukan kebutuhan komputasi. Penting untuk dipahami bahwa semua lalu lintas korporat harus dilewatkan melalui satu simpul eksternal tunggal: untuk menghubungkan jaringan lokal (atau beberapa jaringan) ke VDS dengan IDS Suricata diinstal, Anda dapat menggunakan SoftEther , server VPN lintas-platform yang mudah dikonfigurasi yang menyediakan enkripsi kuat. Koneksi Internet kantor mungkin tidak memiliki IP asli, jadi lebih baik untuk memutakhirkannya ke VPS. Tidak ada paket yang sudah jadi dalam repositori Ubuntu, Anda harus mengunduh perangkat lunaknya dengansitus proyek , atau dari repositori eksternal pada layanan Launchpad (jika Anda mempercayainya): 



sudo add-apt-repository ppa:paskal-07/softethervpn
sudo apt-get update


Anda dapat melihat daftar paket yang tersedia menggunakan perintah berikut: 



apt-cache search softether






Kita akan memerlukan softether-vpnserver (server dalam konfigurasi tes berjalan pada VDS), serta softether-vpncmd - utilitas baris perintah untuk mengkonfigurasinya. 



sudo apt-get install softether-vpnserver softether-vpncmd


Untuk mengkonfigurasi server, utilitas baris perintah khusus digunakan: 



sudo vpncmd






Kami tidak akan berbicara secara rinci tentang pengaturan: prosedurnya cukup sederhana, itu dijelaskan dengan baik dalam berbagai publikasi dan tidak berlaku langsung pada topik artikel. Singkatnya, setelah memulai vpncmd, pilih item 1 untuk pergi ke konsol manajemen server. Untuk melakukan ini, masukkan nama localhost dan tekan enter alih-alih memasukkan nama hub. Kata sandi administrator diatur di konsol dengan perintah serverpasswordset, hub DEFAULT virtual (perintah hubdelete) dihapus dan yang baru dibuat dengan nama Suricata_VPN, dan kata sandinya juga diatur (perintah hubcreate). Selanjutnya, buka konsol manajemen hub baru menggunakan perintah Suricata_VPN hub untuk membuat grup dan pengguna menggunakan perintah groupcreate dan usercreate. Kata sandi pengguna diatur menggunakan kata sandi pengguna.



SoftEther mendukung dua mode transfer lalu lintas: SecureNAT dan Local Bridge. Yang pertama adalah teknologi eksklusif untuk membangun jaringan pribadi virtual dengan NAT dan DHCP-nya sendiri. SecureNAT tidak memerlukan TUN / TAP, serta mengatur Netfilter atau firewall lain. Routing tidak mempengaruhi inti sistem, dan semua proses tervirtualisasi dan bekerja pada VPS / VDS, terlepas dari hypervisor yang digunakan. Hal ini menyebabkan peningkatan beban prosesor dan kecepatan lebih rendah dibandingkan dengan mode Local Bridge yang menghubungkan hub virtual SoftEther dengan adaptor jaringan fisik atau perangkat TAP.



Pengaturan dalam hal ini rumit karena perutean dilakukan pada level kernel menggunakan Netfilter. VDS kami dibangun di atas Hyper-V, jadi pada langkah terakhir kami membuat jembatan lokal dan mengaktifkan perangkat TAP dengan perintah bridgecreate Suricate_VPN -device: suricate_vpn -tap: yes. Setelah keluar dari konsol manajemen hub, kita akan melihat antarmuka jaringan baru di sistem yang belum diberi IP: 



ifconfig






Selanjutnya, Anda harus mengaktifkan perutean paket antar antarmuka (ip forward) jika tidak aktif: 



sudo nano /etc/sysctl.conf


Batalkan komentar pada baris berikut: 



net.ipv4.ip_forward = 1


Kami menyimpan perubahan pada file, keluar dari editor dan menerapkannya menggunakan perintah berikut: 



sudo sysctl -p


Selanjutnya, kita perlu mendefinisikan subnet dengan IP dummy untuk jaringan virtual (misalnya, 10.0.10.0/24) dan menetapkan alamat ke antarmuka: 



sudo ifconfig tap_suricata_vp 10.0.10.1/24


Maka Anda perlu menjabarkan aturan Netfilter.



1. Jika perlu, izinkan paket masuk ke port mendengarkan (protokol kepemilikan SoftEther menggunakan HTTPS dan port 443) 



sudo iptables -A INPUT -p tcp -m tcp --dport 443 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 992 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 1194 -j ACCEPT
sudo iptables -A INPUT -p udp -m udp --dport 1194 -j ACCEPT
sudo iptables -A INPUT -p tcp -m tcp --dport 5555 -j ACCEPT


2. Konfigurasikan NAT dari subnet 10.0.10.0/24 ke server IP utama 



sudo iptables -t nat -A POSTROUTING -s 10.0.10.0/24 -j SNAT --to-source 45.132.17.140


3. Kami mengizinkan paket yang lewat dari subnet 10.0.10.0/24 



sudo iptables -A FORWARD -s 10.0.10.0/24 -j ACCEPT


4. Kami mengizinkan paket lewat untuk koneksi yang sudah ada 



sudo iptables -A FORWARD -p all -m state --state ESTABLISHED,RELATED -j ACCEPT


Otomatisasi proses ketika sistem dihidupkan ulang menggunakan skrip inisialisasi akan diserahkan kepada pembaca sebagai pekerjaan rumah.



Jika Anda ingin mengeluarkan klien IP secara otomatis, Anda juga harus menginstal semacam layanan DHCP untuk jembatan lokal. Ini menyelesaikan pengaturan server dan Anda dapat pergi ke klien. SoftEther mendukung banyak protokol, penggunaannya tergantung pada kemampuan peralatan LAN. 



netstat -ap |grep vpnserver






Karena router pengujian kami juga berfungsi di Ubuntu, kami akan menginstal paket softether-vpnclient dan softether-vpncmd di dalamnya dari repositori eksternal untuk menggunakan protokol berpemilik. Anda harus memulai klien: 



sudo vpnclient start


Untuk mengkonfigurasi, kami menggunakan utilitas vpncmd, memilih localhost sebagai mesin yang menjalankan vpnclient. Semua perintah dilakukan di konsol: Anda harus membuat antarmuka virtual (NicCreate) dan akun (AccountCreate).



Dalam beberapa kasus, Anda perlu menentukan metode otentikasi menggunakan perintah AccountAnonymousSet, AccountPasswordSet, AccountCertSet, dan AccountSecureCertSet. Karena kami tidak menggunakan DHCP, alamat untuk adaptor virtual diatur secara manual.



Selain itu, kita perlu mengaktifkan ip forward (parameter net.ipv4.ip_forward = 1 dalam file /etc/sysctl.conf) dan mengkonfigurasi rute statis. Jika perlu, pada VDS dengan Suricata, Anda dapat mengonfigurasi port forwarding untuk menggunakan layanan yang diinstal pada jaringan lokal. Dalam hal ini, penyatuan jaringan dapat dianggap lengkap.



Konfigurasi yang kami tawarkan akan terlihat seperti ini:







Sesuaikan Suricata



Pada artikel sebelumnya, kita berbicara tentang dua mode operasi IDS: melalui antrian NFQUEUE (mode NFQ) dan melalui nol salinan (mode AF_PACKET). Yang kedua membutuhkan dua antarmuka, tetapi berbeda dalam kecepatan yang lebih tinggi - kita akan menggunakannya. Parameter diatur secara default di / etc / default / suricata. Kita juga perlu mengedit bagian vars di /etc/suricata/suricata.yaml, mengatur subnet virtual di sana sebagai subnet rumah Anda.







Untuk memulai kembali IDS, gunakan perintah: 



systemctl restart suricata


Solusinya sudah siap, sekarang Anda mungkin perlu memeriksa resistensi untuk tindakan penyerang.



Kami mensimulasikan serangan



Mungkin ada beberapa skenario untuk memerangi penggunaan layanan IDS eksternal:



Perlindungan terhadap serangan DDoS (tujuan utama)



Sulit untuk mengimplementasikan opsi semacam itu dalam jaringan perusahaan, karena paket analisis harus dikirim ke sistem dengan melihat antarmuka Internet. Bahkan jika IDS memblokirnya, lalu lintas palsu dapat menempatkan saluran data. Untuk menghindari hal ini, Anda perlu memesan VPS dengan koneksi Internet yang cukup efisien yang dapat melewati semua lalu lintas di jaringan lokal dan semua lalu lintas eksternal. Ini seringkali lebih mudah dan lebih murah daripada memperluas saluran kantor. Sebagai alternatif, perlu disebutkan layanan khusus untuk perlindungan terhadap DDoS. Biaya layanan mereka sebanding dengan biaya server virtual, tetapi tidak memerlukan pengaturan yang melelahkan, tetapi ada kelemahan - untuk uang klien hanya menerima perlindungan dari DDoS, sementara IDS mereka sendiri dapat dikonfigurasi sesuai keinginan Anda.



Perlindungan terhadap serangan eksternal dari jenis lain



Suricata dapat mengatasi berbagai upaya untuk mengeksploitasi berbagai kerentanan dalam layanan jaringan perusahaan yang dapat diakses dari Internet (server surat, server web dan aplikasi web, dll.). Biasanya, untuk ini, IDS dipasang di dalam LAN setelah perangkat perbatasan, tetapi take-out memiliki hak untuk ada.



Perlindungan terhadap penyusup internal



Meskipun upaya terbaik dari administrator sistem, komputer di jaringan perusahaan mungkin terinfeksi malware. Selain itu, hooligan terkadang muncul di LAN dan mencoba melakukan operasi ilegal tertentu. Suricata dapat membantu memblokir upaya tersebut, meskipun lebih baik untuk menginstalnya di dalam perimeter untuk melindungi jaringan internal dan menggunakannya bersama-sama dengan sakelar yang dikelola yang mampu mencerminkan lalu lintas ke satu port. IDS eksternal dalam hal ini juga tidak sia-sia - setidaknya ia akan dapat menangkap upaya malware yang tinggal di LAN untuk menghubungi server eksternal.



Pertama, kita akan membuat tes serangan VPS lain, dan pada router jaringan lokal kita akan meningkatkan Apache dengan konfigurasi default, setelah itu kita akan meneruskan port ke-80 ke sana dari server IDS. Selanjutnya, kita akan mensimulasikan serangan DDoS dari node yang menyerang. Untuk melakukan ini, unduh dari GitHub, kompilasi dan jalankan program xerxes kecil pada node yang menyerang (mungkin memerlukan instalasi paket gcc): 



git clone https://github.com/Soldie/xerxes-DDos-zanyarjamal-C.git
cd xerxes-DDos-zanyarjamal-C/
gcc xerxes.c -o xerxes 
./xerxes 45.132.17.140 80


Hasil karyanya adalah sebagai berikut:







Suricata memotong penjahat, dan halaman Apache dibuka secara default, terlepas dari serangan dadakan kami dan saluran yang agak mati dari jaringan "kantor" (sebenarnya rumah). Untuk tugas yang lebih serius, Anda harus menggunakan Metasploit Framework . Ini dirancang untuk melakukan tes penetrasi dan memungkinkan Anda untuk mensimulasikan berbagai serangan. Instruksi pemasangan tersedia di situs web proyek. Setelah instalasi, diperlukan pembaruan: 



sudo msfupdate


Untuk pengujian, jalankan msfconsole.







Sayangnya, dalam versi terbaru dari kerangka kerja tidak ada kemungkinan peretasan otomatis, jadi exploit harus diurutkan secara manual dan dijalankan menggunakan perintah use. Pertama, Anda perlu menentukan port terbuka pada mesin yang diserang, misalnya, menggunakan nmap (dalam kasus kami, itu akan sepenuhnya diganti oleh netstat pada node yang diserang), dan kemudian pilih dan gunakan modul Metasploit yang sesuai . 



Ada cara lain untuk menguji resistensi IDS terhadap serangan, termasuk layanan online. Demi rasa ingin tahu, Anda dapat mengatur pengujian stres menggunakan versi percobaan IP Stresser. Untuk memeriksa reaksi terhadap tindakan penyerang internal, ada baiknya memasang alat khusus di salah satu mesin di jaringan lokal. Massa pilihan dan secara berkala harus diterapkan tidak hanya pada tempat pelatihan eksperimental, tetapi juga untuk sistem kerja, tetapi ini adalah cerita yang sama sekali berbeda
