SwiftUIを使用したネオモーフィズム。パート1

敬礼、ハブロバイト！上級コース「IOS Developer」の立ち上げを見越して、興味深い翻訳をもう1つ用意しました。



非モーフィックなデザインはおそらくここ数ヶ月の最も興味深いトレンドですが、実際には、AppleはそれをWWDC18のデザインモチーフとして使用しました。この記事では、SwiftUIを使用して非モーフィックデザインを実装する方法、これを行う理由、そして最も重要なこととして、このデザインを改良してアクセシビリティを向上させる方法について説明します。
重要：ネオモーフィズム（ニューロモーフィズムとも呼ばれる）はアクセシビリティに重大な影響を与えるため、この記事の最初の部分を読み、残りをスキップする誘惑にもかかわらず、記事を最後まで読み、利点と欠点の両方を調べて全体像を確認できるようにしてください。 。


ネオモーフィズムの基礎


コードに移る前に、設計におけるこの方向の2つの基本原則について簡単に説明します。これらは、私たちが進むにつれて関連性があるためです。

  1. ネオモーフィズムは、グレアとシャドウを使用して、画面上のオブジェクトの形状を決定します。
  2. コントラストは低下する傾向があります。完全に白または黒は使用されないため、ハイライトとシャドウを強調できます。

最終結果は、「押し出しプラスチック」を連想させる外観です。ユーザーインターフェースのデザインは、目にぶつかることなく、新鮮で興味深いものに見えます。コントラストを減らし、影を使用して図形を強調表示することは、アクセシビリティに深刻な影響を与えることをもう一度繰り返すしかありません。これについては、後で説明します。

ただし、SwiftUIでネオモーフィズムについて学習するのに費やした時間は、それだけの価値があると私はまだ思います。自分のアプリケーションでそれを使用しなくても、スキルを磨くのに役立つコードを書くコードのようなものです。

さて、十分なアイドルトーク-コードに移りましょう。

非モーフィックマップの作成


最も単純な開始点は、非モーフィックマップを作成することです。いくつかの情報を含む角丸四角形です。次に、これらの原則をSwiftUIの他の部分に転送する方法を見ていきます。

まず、シングルビューアプリテンプレートを使用して新しいiOSプロジェクトを作成します。ユーザーインターフェイスにSwiftUIを使用していることを確認し、プロジェクトにNeumorphismという名前を付けます。

ヒント：XcodeでSwiftUIをプレビューするアクセス権がある場合は、すぐにアクティブにすることをお勧めします-実験がはるかに簡単になります。

まず、クリーミーな色合いを表す色を定義します。これは純粋な灰色ではなく、インターフェイスに少し暖かさや涼しさを加える非常に微妙な色合いです。必要に応じてアセットディレクトリに追加できますが、コードで行う方が簡単になりました。

これをColor構造の外に追加しますContentView：

extension Color {
    static let offWhite = Color(red: 225 / 255, green: 225 / 255, blue: 235 / 255)
}

はい、ほとんど白ですが、必要なときに真の白を輝きのように見せるために十分に暗いです。

これContentViewでZStack、画面全体を占めるを提供することでボディを塗りつぶすことができます。新しい準白色を使用して、スペース全体を塗りつぶします。

struct ContentView: View {
    var body: some View {
        ZStack {
            Color.offWhite
        }
        .edgesIgnoringSafeArea(.all)
    }
}

地図を表すために、解像度300x300の角丸長方形を使用して、画面上で美しく鮮明にします。だからこれをZStack色の下に追加します：

RoundedRectangle(cornerRadius: 25)
    .frame(width: 300, height: 300)

デフォルトでは黒になりますが、ネオモーフィズムの実装ではコントラストを大幅に減らしたいため、背景に使用しているのと同じ色に置き換え、実際には形状を非表示にします。

したがって、次のように変更します。

RoundedRectangle(cornerRadius: 25)
    .fill(Color.offWhite)
    .frame(width: 300, height: 300)

重要なポイント：ライトが画面の左上隅から光線を投影するかのように、1つの暗いライトと1つのライトを使用して形状を決定します。

SwiftUIを使用すると、修飾子を数回適用できるため、ネオモーフィズムの実装が容易になります。次の2つの修飾子を角丸長方形に追加します。

.shadow(color: Color.black.opacity(0.2), radius: 10, x: 10, y: 10)
.shadow(color: Color.white.opacity(0.7), radius: 10, x: -5, y: -5)

これらは、右下隅の暗い影のオフセットと左上隅の明るい影のオフセットを表します。準白色の背景を使用したため、明るい影が表示され、マップが表示されます。

ほんの数行のコードを記述しましたが、すでに非モーフィックなマップがあります。SwiftUIが驚くほどプロセスを容易にすることに同意していただければ幸いです。



単純な非モーフィックボタンの作成


UIのすべての要素の中で、ネオモーフィズムはカードのリスクをかなり低くします。カード内のUIがはっきりしている場合、カードに明確な境界線がなくても、アクセシビリティには影響しません。ボタンは相互作用するように設計されているため、別の問題です。ボタンのコントラストを下げると、効果よりも害が大きくなる可能性があります。

これは、SwiftUIがボタン構成を多くの場所に分散できるようにする独自のボタンスタイルを作成することで対処します。これは、作成する各ボタンに多数の修飾子を追加するよりもはるかに便利です。スタイルを一度定義するだけで、多くの場所で使用できます。

実際には空のボタンスタイルを定義します。SwiftUIはボタンのラベル（テキスト、画像など）を提供し、変更せずに送り返します。

この構造を外側のどこかに追加しますContentView。

struct SimpleButtonStyle: ButtonStyle {
    func makeBody(configuration: Self.Configuration) -> some View {
        configuration.label
    }
}

これconfiguration.labelはボタンのコンテンツを保持するものであり、すぐに何かを追加します。まず、それを使用するボタンを定義して、デザインがどのように進化しているかを確認します。

Button(action: {
    print("Button tapped")
}) {
    Image(systemName: "heart.fill")
        .foregroundColor(.gray)
}
.buttonStyle(SimpleButtonStyle())

画面には何も表示されませんが、ボタンのスタイルに非モーフィック効果を追加することで修正できます。今回は、丸い長方形を使用しません。シンプルなアイコンの場合は円の方が良いためですが、ボタンのクリック領域が大きくて美しいように、インデントを追加する必要があります。インデントを追加し
てメソッドを変更し、makeBody()非モーフィック効果をボタンの背景として配置します。

configuration.label
    .padding(30)
    .background(
        Circle()
            .fill(Color.offWhite)
            .shadow(color: Color.black.opacity(0.2), radius: 10, x: 10, y: 10)
            .shadow(color: Color.white.opacity(0.7), radius: 10, x: -5, y: -5)
    )



これで目的の効果に十分近づきますが、アプリケーションを実行すると、実際には動作がまだ完全ではないことがわかります。ボタンを押しても視覚的に反応せず、奇妙に見えます。

これを修正するにconfiguration.isPressedは、ボタンが現在押されているかどうかを示すカスタムボタンスタイル内のプロパティを読み取る必要があります。これを使用してスタイルを改善し、ボタンが押されているかどうかを視覚的に示すことができます。

簡単なものから始めましょう。Group背景にボタンを使用しconfiguration.isPressed、ボタンが押された場合は平らな円を、そうでない場合は現在の暗い円をチェックして返します。

configuration.label
    .padding(30)
    .background(
        Group {
            if configuration.isPressed {
                Circle()
                    .fill(Color.offWhite)
            } else {
                Circle()
                    .fill(Color.offWhite)
                    .shadow(color: Color.black.opacity(0.2), radius: 10, x: 10, y: 10)
                    .shadow(color: Color.white.opacity(0.7), radius: 10, x: -5, y: -5)
            }
        }
    )

状態isPressedは疑似白色の円を使用しているため、ボタンを押したときに効果が見えなくなります。

警告：SwiftUIがタップ可能な領域を計算する方法が原因で、ボタンのクリック領域を意図せずに非常に小さくしました。今度は、画像自体をタップする必要があり、その周囲の変則的なデザインをタップする必要はありません。これを修正するには、の.contentShape(Circle())直後に修飾子を追加し、.padding(30)SwiftUIが利用可能なタップスペースをすべて使用するようにします。

これで、シャドウを反転することで人工的な凹面のエフェクトを作成できます。shadowベースエフェクトから2つのモディファイヤをコピーし、次に示すように、白と黒のX値とY値を交換します。

if configuration.isPressed {
    Circle()
        .fill(Color.offWhite)
        .shadow(color: Color.black.opacity(0.2), radius: 10, x: -5, y: -5)
        .shadow(color: Color.white.opacity(0.7), radius: 10, x: 10, y: 10)
} else {
    Circle()
        .fill(Color.offWhite)
        .shadow(color: Color.black.opacity(0.2), radius: 10, x: 10, y: 10)
        .shadow(color: Color.white.opacity(0.7), radius: 10, x: -5, y: -5)
}

結果を見積もります。



ボタンクリックの内部シャドウを作成する


私たちの現在のコードは原則としてすでに機能していますが、人々はその効果を別の方法で解釈します-凹型のボタンとして見る人もいれば、ボタンがまだ押されておらず、光が別の角度から来るだけの人もいます。

改善のアイデアは、ボタンを内側に押す効果をシミュレートする内側の影を作成することです。これは標準のSwiftUIキットの一部ではありませんが、非常に簡単に実装できます。

内側のシャドウを作成するには2つの線形グラデーションが必要です。これらは、この記事で使用する多くの内部グラデーションの最初のものにすぎないためLinearGradient、標準のグラデーションの作成を簡略化するための小さな補助拡張をすぐに追加します。

extension LinearGradient {
    init(_ colors: Color...) {
        self.init(gradient: Gradient(colors: colors), startPoint: .topLeading, endPoint: .bottomTrailing)
    }
}

これにより、色の変数リストを提供して、対角線方向の線形グラデーションを取得できます。

ここで重要なポイントについて：押した円に2つの反転した影を追加する代わりに、ぼかし（ストローク）を使用して新しい円をオーバーレイし、グラデーション付きの別の円をマスクとして適用します。これは少しトリッキーですが、少しずつ説明します。

  • 私たちのベースサークルは、疑似白色で満たされた、ネオモーフィック効果を持つ現在のサークルです。
  • その上に灰色のフレームで囲まれた円を置き、少しぼかしてエッジを柔らかくします。
  • 次に、この円の上に別の円を重ねたマスクを適用します。今回は線形グラデーションで塗りつぶします。

あるビューを別のビューのマスクとして適用すると、SwiftUIはマスクのアルファチャネルを使用して、ベースビューに何を表示するかを決定します。

したがって、ぼやけた灰色のストロークを描画し、黒から透明への線形グラデーションでマスクすると、ぼやけたストロークは一方の側では見えなくなり、もう一方の側では徐々に増加します-滑らかな内部グラデーションが得られます。効果をより際立たせるために、陰影を付けた円を両方向に少しシフトできます。少し実験してみましたが、暗いものよりも太い線で明るい影を描くと効果が最大になります。

ネオモーフィズムに奥行き感を与えるために2つの影が使用されていることを思い出してください。1つは明るい色、もう1つは暗い色なので、内側の影のこの効果を異なる色で2回追加します。次のように

円configuration.isPressを変更します。

Circle()
    .fill(Color.offWhite)
    .overlay(
        Circle()
            .stroke(Color.gray, lineWidth: 4)
            .blur(radius: 4)
            .offset(x: 2, y: 2)
            .mask(Circle().fill(LinearGradient(Color.black, Color.clear)))
    )
    .overlay(
        Circle()
            .stroke(Color.white, lineWidth: 8)
            .blur(radius: 4)
            .offset(x: -2, y: -2)
            .mask(Circle().fill(LinearGradient(Color.clear, Color.black)))
    )

アプリケーションを再度実行すると、ボタンを押したときの効果がより顕著になり、見栄えがよくなります。
