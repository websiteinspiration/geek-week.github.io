Warum überhaupt Spiele im Namen der Seite machen?



Ja, vielleicht klingt das nach etwas sehr Seltsamem, aber es ist trotzdem ziemlich interessant. Auf jeden Fall interessanter als das Spiel auf die übliche Weise zu machen.

Wir machen es bequemer



Das erste, was ich tun möchte, ist eine Funktion, die die winAnimation-Logik ersetzt. Es wird eine Meldung angezeigt und dann eine andere Funktion ausgeführt. Als Argument wird ein Objekt mit Texten, Anzahl der Iterationen, Geschwindigkeit und Rückruf verwendet. 



const message = async ({text = [], count = 'infinity', speed = 1000, callback = null}) =>  {
    if(!text[0]) throw new Error('need array for text');
    let i = 0;
    let textIndex = 0;
    while(true) {
        if(count <= i && count !== 'infinity') {
            //    ,   
            if(callback) callback();
            break;
        }
        if(textIndex >= text.length) textIndex = 0;
        //      1    
        document.title = text[textIndex];
        i++;
        textIndex++;
        await sleep(speed)
    }
};
// 
message({text: ['️WINNER', 'WINNER️'], count: 3, speed: 800, callback: mainAnimation})




Ich möchte auch einen Animationsmanager erstellen, mit dem Sie sie ausführen und steuern können. Dies ist eine Klasse mit einem Animations-Repository und mehreren Methoden. 

class Animations {
    animations = [];
    setAnimation(animation, args = {}) {
        //   
        for(const item of this.animations) {
            item.play = false
        }
        //  ,    
        const Animation = this.animations.find(item => item.name === animation);
        if(!Animation) return;
        Animation.play = true;
        Animation.callback(args)
    };
    isPlay(animation) {
        return this.animations.find(item => item.name === animation).play
    }
    createAnimation(name, callback) {
        this.animations.push({
            play: false,
            name,
            callback,
        })
    }
}


Beschreibung der Methoden:

setAnimation - ruft die Animation auf.

isPlay ist eine Bedingungsfunktion zum Überprüfen, ob eine Animation „abgespielt“ wird.

createAnimation - Fügt dem Repository eine Animation hinzu.

Zur Abwechslung fügen wir die Möglichkeit hinzu, einen zufälligen Slot hinzuzufügen.

(Hier nehmen wir einfach Emoji aus der API, fügen es dem Array hinzu und aktivieren die Hauptanimation.) 



const AddSlot = async () => {
    //       API
    animations.setAnimation('message', {text: ['Getting slot','Getting slot'], speed: 500});
    try {
        const data = await fetch(`https://emoji-api.com/emojis?access_key=`);
        const body = await data.json();
        const emoji = await body[Math.floor(Math.random() * body.length -1)];
        slots.push(emoji.character);
    } catch (e) {
        //   
        console.log(e);
        animations.setAnimation('none');
        document.title = 'Error, try again';
        await sleep(800);
    } finally {
        animations.setAnimation('main');
    }
};
