前書き


GJKは、2つの凸形状の交差を決定するために設計されたアルゴリズムです。これは単純で、より一般的なアプローチを使用できる一般化された「補助関数」を使用して実装されます。同様に、楕円などの曲線で構成されるポリゴンやシェイプを処理できます。

必要な情報


ミンコフスキー和


GJKは、ミンコフスキー和と呼ばれる概念を使用します。SMは、2つの数値のすべてのポイントを合計して計算されます。たとえば、次の2つの図を見てください。


図A（緑）：

（0,1） （1、-1） （-1、-1）

図B（紫）：

D E F
（0、-1） （1,1） （-1.1）

図Aと図Bの値を取得すると、ミンコフスキーの合計を計算できます。 これらの値を取得してグラフを作成すると、どの図が結果になるかがわかります。図AとBのミンコフスキー合計：表とグラフのADはA + Dに対応することに注意してください

A + E = (0,1) + (1,1) = (1,2)

A + F = (0,1) + (-1,1) = (-1,2)

B + D = (1,-1) + (0,-1) = (1,-2)

B + F = (1,-1) + (-1,1) = (0,0)

C + E = (-1,-1) + (1,1) = (0,0)

ミンコフスキーの違い


GJKは、ミンコフスキー和のバリエーションを使用します。A+ Bではなく、A-Bをとります。私が読んだソースでは、これは「ミンコフスキー差」と呼ばれています。ミンコフスキー差には興味深い特性があります。2つの数値が重なる/交差する場合、結果のミンコフスキー差には原点が含まれます。そして、これがGJKアルゴリズムの基礎です。

図AとBの値を取得すると、ミンコフスキー差を計算できます。 これらの値を取得してグラフに配置すると、結果の図が表示されます。図AとBのミンコフスキーの違い：表とグラフのADはA-Dを指すことに注意してください

A - D = (0,1) - (0,-1) = (0,2)

A - E = (0,1) - (1,1) = (-1,0)

A - F = (0,1) - (-1,1) = (1,0)

B - D = (1,-1) - (0,-1) = (-1,0)

B - E = (1,-1) - (1,1) = (0,-2)

B - F = (1,-1) - (-1,1) = (2,-2)

C - D = (-1,-1) - (0,-1) = (-1,0)

C - E = (-1,-1) - (1,1) = (-2,-2)

C - F = (-1,-1) - (-1,1) = (0,-2)

アルゴリズム


これらの概念に基づいて、GJKアルゴリズムはそれらを最適化します。ミンコフスキーの合計の計算には、特に多くの点で構成される2つの図形の交点をチェックする場合、時間がかかる場合があります。これを回避するために、GJKはヘルパー関数とシンプレックスという2つの重要な概念を使用しています。

二次機能


補助関数は、図全体を作成せずにミンコフスキー差のエッジ上の点をサンプリングする方法です。ヘルパー関数は、2つの比較された数値と、チェックする必要がある方向を取得します。次に、補助関数は、2つの反対方向から最も遠い点を各図から受け取ります。これらの2つの最も離れた点を使用して、ミンコフスキー差の図上の補助点を計算できます。ミンコフスキー差のポイントを取得するため、反対方向からポイントを取得します。これにより、最大の面積が得られます。つまり、図で原点を囲む可能性が高くなります。ミンコフスキーの違いは a - b、反対方向からサンプリングされた図bの点の存在は、この方向に可能な限り離れた補助点を与えます。

ヘルパー関数の実装は非常に簡単です。

function support(a: IShape, b: IShape, direction: Vector): Vector {
    const aFar = a.FarthestPointInDirection(direction);
    const bFar = b.FarthestPointInDirection(direction.Invert());
    return aFar.Sub(bFar);
}

GJKの利点の1つは、FarthestPointInDirection抽象化してポリゴンやカーブに適用できることです。これFarthestPointInDirectionがポリゴンの実装です。

class Polygon implements IShape {
    public points: Vector[];
    ...
    public FarthestPointInDirection(direction: Vector): Vector {
        let farthestDistance = -Infinity;
        // If there are no points, just return point 0,0
        let farthestPoint: Vector = new Vector(0,0);
        for (const point of this.points) {
            const distanceInDirection = point.Dot(direction);
            if (distanceInDirection > farthestDistance) {
                farthestPoint = point;
                farthestDistance = distanceInDirection;
            }
        }
        return farthestPoint;
    }
}

の（1,0）方向の補助点を計算する方法を次に示します。

  1. 図Aから最も離れた点を取り上げます。それは点B（1、-1）であることがわかります。（これは、上記のアルゴリズムと同様に計算することも、グラフを見るだけで確認することもできます）。
  2. 図Bから最も離れた点を取り上げます。それは点F（-1、1）であることがわかります。
  3. B-Fを計算します。それはポイントBF（2、-2）であることがわかります-それは補助的です。

シンプレックス


シンプレックスは、ミンコフスキー差の図に沿った点のサンプルです。シンプレックスには最大3つのポイントを含めることができます。GJKはそれらを使用して、原点の周りに三角形を作成し、衝突の発生を判断しようとします。

シンプレックス構造


シンプレックスは、さまざまな方向に補助点を追加することにより、繰り返し構築されます。各補助点は新しい方向を指している必要があります。これにより、原点を含むシンプレックスを可能な限り迅速に作成できます。問題は、次の補助点を取得する方向を選択することです。

衝突検出と方向選択


基本的なアルゴリズムは、補助関数を使用してシンプレックスを構築し、図の原点を囲みます。計算された補助点が原点に到達するかどうかを確認することで、衝突/交差がないことを理解できます。到達しない場合、原点はミンコフスキー差の外側にある必要があります。したがって、衝突/交差はありません。

function Calculate(a: IShape, b: IShape): Collision | undefined {
    // Build a new Simplex for determining if a collision has occurred
    const simplex = new Simplex();

    // Choose an arbitrary starting direction
    let direction: Vector | undefined = new Vector(0,1);

    // Get the first support point and add it to the simplex
    const initSupportPoint = support(a, b, direction);
    simplex.Add(initSupportPoint);

    // Flip the direction for the next support point
    direction = direction.Invert();

    // Keep iterating until the direction is undefined, this will occur when
    // 'CalculateDirection' doesn't return a direction, indicating that an 
    // intersection has been detected
    while(direction) {
        const supportPoint = support(a, b, direction);

        // If the support point did not reach as far as the origin,
        // the simplex must not contain the origin and therefore there is no
        // intersection
        if (supportPoint.Dot(direction!) <= 0) {
            // No intersection
            return;
        }

        // Add the simplex and determine a new direction
        simplex.Add(supportPoint);
        direction = simplex.CalculateDirection();
    }
    // No direction calculated, intersection detected
    return new Collision(a, b);
}

アルゴリズムのすべての複雑さと内部動作はにありますsimplex.CalculateDirection。この関数は、原点が現在のシンプレックス内にあるかどうかを判別します-ある場合、戻りundefinedます。それ以外の場合は、シンプレックスに追加する必要がある補助ポイントを取得するために新しい方向を返します。

class Simplex {
    private points: Vector[];
    ...

    CalculateDirection(): Vector | undefined {
        // Get a, the last point added to the simplex
        const a = this.points[this.points.length - 1];
        // Since a was just added, we know that the inverse of a points 
        // towards the origin
        const ao = a.Invert();
        // If the simplex is a triangle
        if (this.points.length == 3) {
            // B is the penultimate in the simplex
            // C is the oldest point in the simplex
            const b = this.points[1];
            const c = this.points[0];

            // Determine a->b and a->c lines
            const ab = b.Sub(a);
            const ac = c.Sub(a);

            // Determine perpendicular of the a->b line
            let abPerp = new Vector(ab.y, -ab.x);

            // Check the handedness of the perpendicular, it should
            // face AWAY from the simplex
            if (abPerp.Dot(c) >= 0) {
                abPerp = abPerp.Invert();
            }

            // If the origin lies outside of the simplex remove the
            // point and determine a new direction in the direction
            // of the perpendicular; aiming to try to encapsulate
            // the origin that lies outside
            if (abPerp.Dot(ao) > 0) {
                this.points.splice(0, 1);
                return abPerp;
            }

            // Determine perpendicular of the a->c line
            let acPerp = new Vector(ac.y, -ac.x);

            // Check the handedness of the perpendicular, it should
            // face AWAY from the simplex
            if (acPerp.Dot(b) >= 0) {
                acPerp = acPerp.Invert();
            }

            // If the origin lies outside of the simplex remove the
            // point and determine a new direction in the direction
            // of the perpendicular; aiming to try to encapsulate
            // the origin that lies outside
            if (acPerp.Dot(ao) > 0) {
                this.points.splice(1, 1);
                return acPerp;
            }
            return undefined;
        }
        // Otherwise the simplex is just a line
        // B is the penultimate point in the simplex,
        // in this case the other end of the line
        const b = this.points[0];
        // Determine a -> b line
        const ab = b.Sub(a);

        // Get the perpendicular of the a->b line
        let abPerp = new Vector(ab.y, -ab.x);

        // Check the handedness of the perpendicular, it should
        // face TOWARDS the origin
        if (abPerp.Dot(ao) <= 0) {
            abPerp = abPerp.Invert();
        }
        return abPerp;
    }
}

疑問に思われるかもしれませんが、なぜBCセグメントをチェックしないのですか？原点が垂直に沿っていることを無条件に除外できるからです。ポイントBとCは既にシンプレックスにあり、追加されているだけでなく、前の反復でチェックされたことがわかります。それらは、三角形の一部として、またはシンプレックスの最初の2つのポイントのセグメントとしてチェックできます-それは問題ではありません。したがって、BCのチェックは安全にスキップできます。
