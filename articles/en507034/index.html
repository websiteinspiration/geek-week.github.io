Webstorm NodeJs interpreter from Docker container

This article describes the process of deploying NodeJs applications using the NodeJs interpreter located in the Docker container as a project (by default). The solution will most likely be relevant only for users of the Linux OS, since in the end the interpreter will be taken from the official node image on DockerHub .


What for?


The application can be deployed with only docker on the host machine, there is no need to install the interpreter yourself, which will allow the developer to have a specific version of the interpreter for a particular project, and when changing the version of the image in the project, the developer will not need to manually configure, a new instance of the interpreter will be used in project automatically.


Decision


For example, an application was created on ReactJs + Typescipt, because if the interpreter is incorrectly configured, at least the code highlighting will break.


1. Create an application from the finished template:


$ npx create-react-app reactjs-typescript-custom-nodejs --template typescript

2. Write a Dockerfile:


FROM node:14
WORKDIR /app

CMD cp /usr/local/bin/node /app/docker/node \
    && npm install \
    && npm run start

CMD, Docker /app/docker/node, /app . .


3. node docker/node .gitignore, :

4. :


docker build -t client . && docker run -it --rm --volume=${PWD}:/app -p 3000:3000 --name client client
