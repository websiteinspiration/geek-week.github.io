2020年のフロントエンド開発の上位5 JSフレームワーク。パート2

Webプロジェクトのクライアント側部分を開発するための5つの最高のJavaScriptツールに特化した資料の翻訳の第2部を公開しています。最初の部分は、ライブラリやフレームワーク角度に反応についてでした。ここでは、Vue、Ember、およびBackboneについて説明します。



3. Vue


Vueの背後にあるアイデアはAngularとReactに基づいていますが、Vueは、多くの点で、これら2つのフロントエンド開発ツールよりも優れています。 Vueは今年、4000万回以上ダウンロードされました。最近のSnyk JavaScript Frameworks Security レポートがリリースされました。それは主にReactとAngularのセキュリティの研究を目的としていますが、他のプロジェクトを研究する場所がありました。 Vueから、このレポートから、修正された即時の脆弱性のうち4つについてのみ知られているものを見つけることができます。

Vueに慣れていない場合は、このフレームワークに関するいくつかの重要な事実を以下に示します。

Vueを使用する場合、コンポーネントのロジック、そのレイアウト、およびスタイルは1つのファイルに保存されます。スタイルを除いて、プロジェクトマテリアルもReactに保存されます。 Vueのコンポーネントの相互作用は、コンポーネントのプロパティと状態を格納するオブジェクトを通じて提供されます。このアプローチも、Vueに登場する前は、Reactで使用されていました。

Angularに似たVueを​​使用すると、HTMLマークアップとJavaScriptコードを混在させることができます。コンポーネントデータをテンプレートに統合するには、Vueディレクティブを使用する必要があります。など、v-bindまたはv-if。

VueがReactに代わる価値のある選択肢と見なされる理由の1つは、アプリケーションの状態がここに整理されているためです。 Reactプロジェクトでは、React + Reduxバンドルを使用すると、アプリケーションのサイズが大きくなるにつれて、その状態を管理するために必要な手順がより複雑になります。これは、プログラマーがアプリケーション自体に取り組むのではなく、Reduxメカニズムのセットアップに多くの時間を費やさなければならないという事実に帰着する可能性があります。 VueはVuexライブラリを使用して状態を管理します。 Fluxに似ており、Vue専用に作成されています。 Reduxよりもはるかに便利です。

VueとAngularのどちらかを選択しようとしている場合、Vueを選択する理由は、Vueと比較してAngularが複雑すぎる大規模プロジェクトのように見え、その性質上、開発者を制限したいということです。VueはAngularよりもはるかに単純で、プログラマーをそれほど制限しません。
AngularとReactに対するVueのもう1つの利点は、このフレームワークで作業するために新しい言語を学ぶ必要がないことです。

the Vueエコシステムの要素


Vueを選択するエコシステムについて簡単に説明します。

  • Vueアプリケーションの状態を制御するために、Fluxに関連する概念に基づいたVuexライブラリーが使用されます。
  • Vueプロジェクトの作成を容易にするChromeとFirefoxの開発者ツールがあります。
  • Vueエコシステムには、vue-loaderVueコンポーネントを単一ファイルコンポーネントとして使用できるようにするWebpackのローダーがあります。
  • サーバーAPIを操作するには、HTTPクライアントvue-resourceとAxiosライブラリーを使用できます。
  • VueはNuxt.jsフレームワークをサポートします。これには、サーバーでレンダリングされるアプリケーションのサポートが含まれます。Nuxt.jsは、Angular Universalの競合相手と見なすことができます。
  • Weexライブラリを使用して、Vueベースのモバイルアプリケーションを開発できます。

Vueは、使いやすさの点で他のユーザーを超える優れたフレームワークです。おそらく、私自身も近い将来にVueに切り替えるでしょう。このフレームワークは、ReactやAngularほど複雑ではなく、さまざまなサイズのアプリケーションの開発にも適しています。

4.エンバー


Ember 3.13 は今年リリースされました。このバージョンのフレームワークには、多くの新しいものが登場しています。 Emberは、BackboneおよびAngularに似ています。また、最も古いJavaScriptフレームワークの1つでもあります。しかし、これにもかかわらず、機会の面では、彼は若い競争相手に遅れをとっていません。たとえば、追跡可能なプロパティ変更テクノロジをサポートしているため、アプリケーションの状態の変化を簡単に監視し、これらの変化を簡単に視覚化できます。

Emberにはかなり洗練されたアーキテクチャがあり、巨大なクライアントアプリケーションをすばやく作成できます。それは典型的なMVCアイデアを実装します。 Emberアプリケーションは、アダプター、コンポーネント、コントローラー、補助オブジェクト、モデル、ルート、サービス、テンプレート、ユーティリティ、アドオンから構築されます。

Emberの最も興味深い機能の1つは、コマンドラインツール（Ember CLI）です。これらのツールは、フロントエンドの開発者が生産的に作業するのに役立ちます。Ember CLIを使用して、プロジェクトテンプレートだけでなく、アプリケーションの構築元であるコントローラー、コンポーネント、およびその他のエンティティのスタブも作成できます。

▍残りの生態系要素


Emberを使用するフロントエンド開発者は、次の主要機能を自由に利用できます。

  • Ember CLIは、アプリケーションのラピッドプロトタイピングと依存関係管理のためのコマンドラインツールです。
  • 標準Ember開発者サーバー。
  • Ember Data-データを操作するためのライブラリ。
  • ハンドルバーは、Emberアプリケーションで使用されるテンプレートエンジンです。
  • QUnitはEmberプロジェクトをテストするためのフレームワークです。
  • Ember Inspector-ChromeおよびFirefox向けの開発ツール。
  • Ember Observer-Ember CLIのアドオンカタログ。

Emberは過小評価されたフレームワークと呼ぶことができるかもしれませんが、それにもかかわらず、複雑なWebプロジェクトを作成するのに最適です。

5. Backbone.js


バックボーンは、MVCと同様のアーキテクチャに基づくJavaScriptフレームワークです。バックボーンでMVCが「コントローラー」と呼ぶものを「ビュー」と呼びます。バックボーンビューでは、さまざまなテンプレートシステムを使用できます。たとえば、Mustache、Handlebars、jQuery-tmplなどです。バックボーンプロジェクトでは、サードパーティのライブラリを使用できます。唯一のハードバックボーン依存関係はUnderscore.jsライブラリーであることに注意してください。

バックボーンは使いやすいフレームワークで、単一ページのアプリケーションをすばやく開発できます。 Backbone.jsと組み合わせて使用​​される補助ツールの中には、Chaplin、Marionette、Thoraxなどがあります。

異なるグループに属するユーザーが機能するアプリケーションを開発する必要がある場合は、Backboneコレクション（配列）を使用してモデルを分離できます。バックボーンモデル、コレクション、ルート、ビューでは、イベントを使用できます。

▍バックボーンエコシステム要素


バックボーンエコシステムの機能には、次のものがあります。

  • バックボーンライブラリには、イベント、モデル、コレクション、ビュー、ルーターが含まれています。
  • Backboneが依存するUnderscore.jsライブラリーには、クロスブラウザーJSコードの作成に役立つ一連のヘルパー関数が含まれています。
  • バックボーンアプリケーションを開発する場合、さまざまなテンプレートシステムを使用できます。
  • Backbone CLIコマンドラインツールは、アプリケーション開発を簡素化します。
  • Marionette、Thorax、およびChaplinライブラリは、特定のアーキテクチャソリューションでアプリケーションを作成するのに役立ちます。

おそらく、Backboneは、この資料で説明されている他のフロントエンド開発ツールの真の競争相手とは言えません。ただし、このフレームワークは開発者の間で非常に人気があります。したがって、Backboneがまさに必要なものであることがわかるでしょう。
