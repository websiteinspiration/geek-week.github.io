VMworld Europe 2019：主な発表と伝説のバックパックの開梱

バルセロナで開催されるVMworld Europe 2019のオープニングで、VMwareのCEOであるPat Gelsingerがカンファレンスとマスコミについて講演しました。彼の歓迎のスピーチで、彼はIT専門職の現代の代表者のための鍵となる概念を概説しました：あらゆる時代の技術、あらゆるものとすべての人の時代の技術。今年の会議には、100か国以上から約5,000社の代表が集まりました。本日の記事では、VMworld Europe 2019カンファレンスの初日の主要な発表をお伝えします。最後に、VMworld Europe 2019を散策して、伝説のバックパックを開梱してください！



パットは彼のスピーチで、以前は少数の愛好家グループだけの運命であったテクノロジーが、今や一般人の生活のあらゆる側面を捉えていると指摘しました。朝のメールをチェックしたり、睡眠や血圧を測定したり、ちょっと待ったりと同じくらい自然なことです。これらはすべてテクノロジーです。

もちろん、生活とテクノロジーのこのような緊密な合併は経済に影響を与えざるを得ません。PatGelsinger氏によると、消費者は映画市場に最大360億ユーロ、モバイルアプリケーション市場にほぼ3倍、最大1,080億ユーロを費やしています。このようにデジタル市場を活用することで、IT企業は新しいテクノロジーを作成できるだけでなく、独自の機能制限を独自に決定して増やし、ユーザーにとってより強力な製品を作成できるようになります。

とりわけ、パットは、将来のテクノロジーに関するVMwareの最新の見解を聴衆と共有しました。パブリックからハイブリッド、プライベートまで、あらゆるクラウドで、あらゆるデジタルデバイスから安全に使用できるアプリケーションが存在する可能性があります。
マルチクラウドをマスターする技術者は、次の10年を所有します。

もう1つの良いニュースは、VMwareとAmazon、Google、IBM、Oracle間の既存のパートナーシップの機能を拡張することです。これにより、パートナー企業の製品をベースにした新しい本格的なアーキテクチャの出現について語ることができます。
Kubernetesは一般にコンテナオーケストレーションと呼ばれます。しかし、それは本当に即興のジャズのようなものです。

感動的なオープニングスピーチの後、VMwareのプリンシパルエンジニアであるJoe BedaとKubernetesの原作者の1人がステージに登場し、Pat は自分のコミュニティのロックスターと個人的に呼びました。

VMware Tanzu


Joe氏によると、現時点でのプロジェクトの使命の1つは、Kubernetesと同じ柔軟性を維持することですが、機能の増加に伴って増加する構成の難易度のしきい値を克服することです。これがまさに、新しいプロジェクト-Kubernetesのコンテナー化されたアプリケーションインフラストラクチャ向けのVMware Tanzuの登場の理由です。実際、Tanzuはオールインワンスイートであり、最新のアプリケーションを構築するための企業のアプローチを変える製品ポートフォリオです。それは、Kubernetesでのアプリケーションのライフサイクル、つまり作成、運用、および管理の3つのクジラを組み合わせて構成します。

Tanzuの仕事は、500万人のプログラマーと世界中のいくつかの企業に広がっています。特に、Bitnami、Pivo​​tal、Springがプロジェクトに関与しています。

Bitnamiは、コンテナと仮想マシンの両方に導入できるコンポーネントのオープンソースカタログを提供します。Pivotalはネイティブクラウドアプリケーションの構築に重点を置いており、Springの基盤は開発者の生産性を大幅に向上させることができます。

VMware Tanzuミッションコントロール


VMwareオンプレミス、パブリッククラウド環境、EKS、IKS、GKEを含むすべてのKubernetesクラスターを1つの「ウィンドウ」から効率的に管理できます。VMware PKSからTanzuへの非常にスムーズな移行が保証されています。

プロジェクトガレオン


もう1つの発表は、Bitnamiカタログの事実上の論理拡張である、BitnamiからのアプリケーションのマーケットプレイスであるProject Galleonのベータ版です。Galleonを使用すると、企業の標準に最適な一連のコンポーネントを組み立て、それらをカスタマイズして、管理と変更が容易な独自のディレクトリを作成できるため、安全で信頼性の高いアプリケーションをより迅速に開発できます。

プロジェクト太平洋


Project Pacificは、VMwareからのもう1つの発表です。プロジェクトの使命は、KubernetesとvSphereを完全に接続することです。これは、vSphere上でKubernetesを起動することではなく、完全な統合についてです。Project Pacificは、VMware vSphere環境をKubernetesクラスターのネイティブプラットフォームに変換するツールスイートです。単一のAPIを通じて、Kubernetes仮想マシンとアセットの両方を管理することが可能になります。統合は、管理レベルだけでなく仮想化にも影響します。Joe Beda氏によると、Kubernetes + vSphereを使用した場合のパフォーマンスは、Linux上の従来の仮想マシンと比較して最大30％、ベアメタルサーバーと比較して8％増加するとのことです。

NSX Advanced Load BalancerおよびNSX Intelligence


VMworld 2019で、VMwareはNSX Advanced Load BalancerおよびNSX-T 2.5リリースの主な宝石であるNSX Intelligenceを発表しました。これはVMworld 2019で大きな影響を与えました。VMwareNSX

Advanced Load Balancer（以前のAvi Networks Platform）-用の分散アプリケーション配信コントローラー（ADC）雲。ソフトウェアロードバランサー、インテリジェントなWebアプリケーションファイアウォール（iWAF）、監視、高度な分析を組み合わせています。

VMware NSXインテリジェンス-NSX-Tに組み込まれた新しい分散分析エンジンは、セキュリティ管理者がデータセンターに共有アクセスできるようにします。NSX Intelligenceは個々の仮想化ワークロードパッケージを検証し、非常に詳細なネットワークポリシーを可能にします。NSX-Tはバージョン2.5も取得します。これにはネイティブクラウド適用モードが含まれており、お客様は独自のクラウドセキュリティコントロールを使用してパブリッククラウドワークロードを保護できます。

  • サービスプロバイダー向けVCF（VCPP）-大規模顧客向けの専用クラウドインフラストラクチャ（専用プライベートクラウド）を構築するための製品の基盤。
  • HCX- . (workloads) on-premises environment Dedicated Private Clouds;

メインシーンに戻ります。 Pat Gelsingerは、VMware Cloud FoundationとVMware vCloud Directorの機能をレビューして紹介スピーチを締めくくった後、Sanjay PunenをCOOステージに招待し、SAP SEの理事であるThomas Saueressigとオープンディスカッションを行いました。 Saueressigは、クラウドテクノロジーの未来と企業市場セグメントとの相互作用に関する会社の戦略的かつ実用的な見解を聴衆に共有しました。次に、Sanjay氏は、SAPは、現在SDxに進化しているソフトウェア定義ソリューションのパイオニアであると述べました。会話の中で、会議の主要な議題の1つであるマルチクラウドに触れました。同社は、AWS、Microsoft Azure、Google Cloud Platform、AliCloudと連携して、この戦略的方向性を積極的に実装しています。また、SAP顧客向けの独自のクラウドにも焦点を当てています。
「私たちはマルチクラウドハイブリッドクラウドソリューションの世界に住んでいるため、消費者はS / 4HANAのリソースを選択する機会が得られ、これは大きな利点です。」

Pat Gelsingerはまた、VMwareの最新のビジョンを共有し、それを同社のパートナーのクラウドビジネスの礎石と呼び、VMwareのタスクの1つは、パートナーとサービスプロバイダーのコミュニティだけでなく、本物の銀河を作ることであると指摘しました。パットによると、このステップは、真のグローバルマルチクラウドの時代に足を踏み入れることを可能にします。


「未来の銀河系の雲を構築するときに力があなたと一緒にいられますように」

新しいMicrosoft統合機能


Sanjay Punenでの別のユーザーケースの後、Pat Gelsingerが舞台に戻り、Microsoftソリューションとの新しい統合オプションについて話しました。企業の協力により、企業の顧客のデジタル変革を加速することを目的とした新しいソリューションと製品のアップデートを発表することができました。特に、以下のイニシアチブが強調されました。

  • VMware Workspace ONEとMicrosoft Endpoint Managerを使用してWindows 10の最新の管理を加速する新しいソリューション。
  • VMware HCXを使用した新しい強化されたクラウド移行機能を備えたAzure VMware Solutionsハイブリッドクラウドサービスのさらなるグローバル展開。
  • クライアントデータセンターのVMware vSphereにおけるMicrosoft SQL Server 2019の認定とサポート。
  • VeloCloudのVMo SD-WANによるAzure周辺機器の拡張。

両社は、Azure VMwareソリューションを世界中で利用できるようにする生産的なコラボレーションに取り組んでいます。Pat Gelsingerは、これらのソリューションを利用できるゾーンの数を増やすことを約束しました。特に、日本、東南アジア、オーストラリアが追加される予定です。

VMware EMC on Dell EMC


VMware Cloud on Dell EMCの発表、およびバルセロナで行われた他のいくつかの発表は、「ホット」とは言えません-今年の4月の初めにすでに発表されていましたが、この決定を回避できませんでした。このソリューションは、パブリッククラウドの柔軟性とローカルインフラストラクチャを保護する機能を組み合わせています。 VMware Cloud on Dell EMCはサービスとして提供されます。このサービスでは、お客様は、ローカルデータセンターとお客様のネットワークの境界のための便利で安全でスケーラブルなインフラストラクチャを受け取ります。このシステムは、Cloud Data Center-as-a-Serviceソリューションに基づいており、顧客独自の機器で機能し、サービスの形でクラウドインフラストラクチャのすべての機能を提供します。この利点は、法律などの特定の理由で、ITシステムをクラウドに置くことはできません。この独自のアプローチにより、イノベーションの導入と競争力の獲得に集中できます。



5GとProject Maestro


Pat Gelsingerのスピーチの一部は、世界中で1か月以上にわたって話題にされてきたトピック（5G）に捧げられました。



トピックの一部として、同社は新しいVMwareソリューションであるProject Maestroの予備レビューを発表しました。その目的は、プロバイダーが最新のクラウドおよびネットワークソリューションの市場投入までの時間を短縮できるようにすることです。Project Maestroは、仮想ネットワークのモデリング、整理、および管理への統一されたアプローチを提供します。その支援により、通信サービスプロバイダーはネットワークサービスを作成および自動化し、NFVおよびTelcoクラウドアーキテクチャの各レベルでの操作の相互作用と最適化を提供できるようになります。ネットワークとクラウドのライフサイクル管理を自動化します。
