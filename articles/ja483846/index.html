Linuxでの代替ウィンドウ管理

私は、Caps Lock切り替えレイアウトを採用する1人です。1つのキーを押すことができるときに、2つのキーを押すのが面倒だからです。2つの不要なキーも必要です。1つは英語のレイアウトをオンにするために使用し、もう1つはロシア語用です。しかし、2つ目の不要なキーはコンテキストメニューの呼び出しです。これは不必要なため、ラップトップの多くのメーカーによってカットされています。だからあなたはあなたが持っているものに満足している必要があります。


また、ウィンドウを切り替えるときは、タスクバーでアイコンを探したり、Alt + Tabをスクロールしたり、デスクトップをスクロールしたりするときに名前を確認したりしたくない。 ）すぐに必要なウィンドウに入ります。たとえば、次のようになります。


  • Alt + F：Firefox
  • Alt + D：Firefox（プライベートブラウジング）
  • Alt + T：ターミナル
  • Alt + M：電卓
  • Alt + E：IntelliJのアイデア
  • 等

さらに、たとえばAlt + Mを押すと、このプログラムが現在実行されているかどうかに関係なく、電卓が表示されます。実行中の場合は、ウィンドウにフォーカスを与える必要があります。そうでない場合は、目的のプログラムを起動し、ロード時にフォーカスを転送します。


前のスクリプトでカバーされていない場合のために、開いているウィンドウを簡単に割り当てることができるユニバーサルキーボードショートカットを用意したいと思います。たとえば、プログラムに関連付けられていないAlt + 1からAlt + 0までの10の組み合わせが割り当てられています。Alt + 1を押すだけで、現在フォーカスされているウィンドウにAlt + 1を押すとフォーカスが移ります。


カットの下で、さらにいくつかの機能の説明とこれを行う方法への回答。ただし、このような「自分用」のカスタマイズは強力な依存症を引き起こし、必要に応じてWindows、Mac OS、またはLinuxを搭載した他の人のコンピューターを使用する場合もあることをすぐに警告します。
