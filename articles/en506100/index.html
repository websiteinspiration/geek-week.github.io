Modeling a two-dimensional Ising model in C ++ (using the OpenGL graphics package)

Ising Model


The Ising model was introduced to understand the nature of ferromagnetism and influenced the study of phase transitions and critical phenomena. Ferromagnetism describes the appearance of spontaneous magnetization in ferromagnets below a certain temperature - the Curie point. At the Curie point (a narrow temperature range), ordering occurs, in this case, the alignment of magnetic moments, which entails a phase transition, that is, the properties of the substance change abruptly.

Initially, the model was proposed by V. Lenz in 1920, and investigated by Ising for a one-dimensional case, in honor of which it got its name. In this model, each vertex of the crystal lattice takes a number: +1 or -1, - field “up” or “down”. The number that matches the vertices is called a spin. Monte Carlo Method


As mentioned earlier, a total of 2 N states are possible, and for a sufficiently large number of N spins, it is difficult to obtain numerical results. For example, for N = 10, we obtain 2 10 states that are not so simple to directly model, therefore, a statistical approach is used for modeling.

In this approach, the system is considered in a state of thermodynamic equilibrium at a certain temperature T. During the exchange of energy with the environment, the energy will change near the equilibrium state, and the average energy of one particle is proportional to T. The implementation of a constant random change around the equilibrium state uses the Monte Carlo method and modeling can be divided into stages:

  1. Play the states α i of the system (for example, randomly) at a fixed T;
  2. To consider for these states the thermodynamic characteristics near equilibrium (energy E, magnetization M);
  3. Average the obtained values


Metropolis Algorithm.


In the problems of statistical mechanics, the expressions “Monte Carlo method” and “Metropolis sampling method” are almost synonyms. We give the most general form of the Metropolis algorithm as an example of a system of spins or particles:

  1. We form the initial configuration.
  2. We make a random test change in the initial configuration. For example, we randomly select a spin and try to knock it over. Or select a random particle and try to move it to a random distance.
  3. We calculate ∆, that is, the change in the energy of the system due to the test configuration change.
  4. If ∆ is less than or equal to zero, then we accept the new configuration and go to step 8.
  5. If Δ is positive, we calculate the "transition probability":

  6. We generate a random number rnd in the interval (0, 1)
  7. If rnd≤W, then we accept the new configuration, otherwise, save the previous configuration.
  8. We determine the values ​​of the required physical quantities.
  9. Repeat steps 2–8 to get enough configurations or “tests.”
  10. We calculate averages over configurations that are statistically independent of each other.

Two-dimensional Ising model in C ++


We will write our own program that implements the two-dimensional Ising model using the Monte Carlo method and the Metropolis algorithm. We implement the following functionality:

  • A configuration change will occur directly on the screen every 10 ms (milliseconds); with each configuration change, the number of Monte Carlo steps will increase.
  • The user can change the temperature before each new change in the configuration of the spin lattice.
  • Every 10 s (seconds), data will be output to the console (acceptance coefficient, average energy per spin, average square of energy per spin, average magnetization, average square of magnetization).

#include <iostream>
#include <random>
#include <locale>
#include "glut.h"

using namespace std;

#define SIZE 256
#define SIZE_PX 800
#define T_MAX 4
#define CURIE_SCALE 1000. 

double quadSize = SIZE_PX / (double)SIZE;

short lattice[SIZE][SIZE];
double w[5];
double T = T_MAX, M, E;

int ratio = 0;
size_t nmcs = 0;
double ecum = 0, e2cum = 0, mcum = 0, m2cum = 0;

void display()
{
	glClear(GL_COLOR_BUFFER_BIT);
	
	glBegin(GL_QUADS);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			glColor3f(lattice[i][j], lattice[i][j], lattice[i][j]);
			glVertex2d(i, j);
			glVertex2d(i, j + 1);
			glVertex2d(i + 1, j + 1);
			glVertex2d(i + 1, j);
		}
	glEnd();

	glBegin(GL_QUADS);
	{
		glColor3f(1, 0, 0);
		glVertex2d(SIZE, 0);
		glVertex2d(SIZE, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, SIZE * T / T_MAX);
		glVertex2d(SIZE + SIZE / 8, 0);

		glColor3f(1, 1, 1);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
		glVertex2d(SIZE, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX + SIZE / CURIE_SCALE);
		glVertex2d(SIZE + SIZE / 8, SIZE * 2.27 / T_MAX - SIZE / CURIE_SCALE);
	}
	glEnd();

	glutSwapBuffers();
	glFlush();
}

void calcW()
{
	double e4 = exp(-4 / T), e8 = e4 * e4;
	w[0] = w[4] = e8;
	w[1] = w[3] = e4;
	w[2] = 0;
}

void init()
{
	M = E = 0;
	static std::default_random_engine generator;
	std::uniform_int_distribution<int> distribution(0, RAND_MAX);
	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			lattice[i][j] = ((distribution(generator) / (double)RAND_MAX >= 0.5) - 1) * 2 + 1;
			M += lattice[i][j];
		}

	for (int i = 0; i < SIZE; i++)
		for (int j = 0; j < SIZE; j++)
		{
			E += (i + 1 != SIZE) ? lattice[i][j] * lattice[i + 1][j] : 0;
			E += (j + 1 != SIZE) ? lattice[i][j] * lattice[i][j + 1] : 0;
		}

	calcW();
}

void metropolis()
{
	int x, y, sum;
	for (int i = 0; i < SIZE * SIZE; i++)
	{
		x = rand() % SIZE;
		y = rand() % SIZE;
		sum = lattice[(x - 1 + SIZE) % SIZE][y] +
			lattice[(x + 1 + SIZE) % SIZE][y] +
			lattice[x][(y - 1 + SIZE) % SIZE] +
			lattice[x][(y + 1 + SIZE) % SIZE];

		if (sum * lattice[x][y] <= 0 || (rand() / (double)RAND_MAX) < w[sum / 2 + 2])
		{
			lattice[x][y] = -lattice[x][y];
			::ratio++;
			M += 2 * lattice[x][y];
			E -= 2 * lattice[x][y] * sum;
		}
	}
}

void timer1(int)
{
	display();

	metropolis();

	nmcs++;

	ecum += E;
	e2cum += E * E;
	mcum += M;
	m2cum += M * M;


	glutTimerFunc(10, timer1, 0);
}

void outputData(int)
{
	double norm = 1 / (double)(nmcs * SIZE * SIZE);

	cout << "  = " << ::ratio * norm << endl;
	cout << "    = " << ecum * norm << endl;
	cout << "     = " << e2cum * norm << endl;
	cout << "  = " << mcum * norm << endl;
	cout << "   = " << m2cum * norm << endl << endl;

	glutTimerFunc(10000, outputData, 0);
}

void keyboard(unsigned char key, int x, int y)
{
	switch (key)
	{
	case 'w': case 'W':
	{
		T += 0.01;
		T = (T >= T_MAX) ? T_MAX : T;
		calcW();
		break;
	}
	case 's': case 'S':
	{
		T -= 0.01;
		T = (T <= 0.01) ? 0.01 : T;
		calcW();
		break;
	}
	}
}

void motion(int x, int y)
{
	double k = 1 - y / (double)SIZE_PX;
	T = T_MAX * k;
	T = (T >= T_MAX) ? T_MAX : T;
	T = (T <= 0.01) ? 0.01 : T;
	calcW();
}

int main()
{
	setlocale(LC_ALL, "ru");
	init();
	glutInitDisplayMode(GLUT_SINGLE | GLUT_RGB);
	glutInitWindowSize(SIZE_PX + 100, SIZE_PX);
	glutCreateWindow("ISING");
	glMatrixMode(GL_PROJECTION);
	glLoadIdentity();
	gluOrtho2D(0, SIZE + SIZE / 8, 0, SIZE);
	glutDisplayFunc(display);
	glutKeyboardFunc(keyboard);
	glutMotionFunc(motion);
	glutTimerFunc(1, timer1, 0);
	glutTimerFunc(10000, outputData, 0);

	glutMainLoop();
}

Note the purpose of creating some functions in this software product:

  • init () - create an initial configuration;
  • display () - rendering of the lattice of spins and temperature scale;
  • calcW () - calculation of transition probabilities;
  • metropolis () - implementation of the Metropolis algorithm;
  • timer1 () - updating data after each configuration change, increasing the number of Monte Carlo steps;
  • outputData () - calculation of the average per spin, output of the necessary data;
  • keyboard () and motion () - the ability to change the temperature using the scale in two ways - buttons on the keyboard and mouse click, respectively;

It is worth making a couple of conclusions:

  • At a critical temperature T = 2.269 ℃, a phase transition occurs from a disordered to an ordered ferromagnetic state.
  • When the temperature changes downward, the following physical and statistical characteristics change as follows:
    • The acceptance rate is reduced;
    • The average energy per spin decreases;
    • The average square of energy per spin increases;
    • The average magnetization decreases;
    • The average square of the magnetization increases.
