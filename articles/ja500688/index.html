私がこの問題を解決したとき、私は今、もう問題は残っていないと思いました。 main.pyの関数を作成するだけで、特定のイベントで、テレグラムからデータベースにデータを書き込みます。まあ、私はモデルのインポートユーザーから接続し、関数と呼ばれる適切な場所に：

def find_user_in_db(username, first_name, last_name):
    """
               
    :param username:
    :param first_name:
    :param last_name:
    :return:
    """
    user = db.session.query(Users).filter(Users.username == f'{username}').all()
    if not user:
        add_users_to_db(username, first_name, last_name)

最後の関数では、最初にuser = db.session.query（Users）.filter（Users.username == f '{username}'）の代わりに、First_or_404（）がUsers.query.filter_by（username = '{username}'）を立っていました。 first_or_404（）。ここでも、データベース内のこのクエリが機能せず、データを受信しないため、データベースへのクエリが送信されないことを理解するのに約30分かかりました。 Googleは、db.session.query（ユーザー）を使用する方がよいと人々が言っ​​た小さな記事を促しました。なぜそうなのか、私は、分析に時間を浪費しませんでした。その後、データがデータベースに書き込まれ始めました。
