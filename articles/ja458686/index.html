\通常の行の 文字には特別な意味があります。\t-これはタブ文字です\r-改行など

この動作を無効にするには、生の文字列を使用できます。その後r'\t'、バックスラッシュとに変わりますt。

当然、'内部では使用できませんr'...'。そして、この制限は助けを借りて回避することができますが\、線\はまだ残ります：

>>> print(r'It\'s insane!')

リストジェネレーターには、複数の式forとif次のペアを含めることができます。

In : [(x, y) for x in range(3) for y in range(3)]
Out: [
    (0, 0), (0, 1), (0, 2),
    (1, 0), (1, 1), (1, 2),
    (2, 0), (2, 1), (2, 2)
]
In : [
    (x, y)
    for x in range(3)
    for y in range(3)
    if x != 0
    if y != 0
]
Out: [(1, 1), (1, 2), (2, 1), (2, 2)]

さらに、任意の式が内部にforあり、if以前に定義されたすべての変数を使用できます。

In : [
    (x, y)
    for x in range(3)
    for y in range(x + 2)
    if x != y
]
Out: [
    (0, 1),
    (1, 0), (1, 2),
    (2, 0), (2, 1), (2, 3)
]

あなたは混在させることができますifし、for独自の裁量で：

In : [
    (x, y)
    for x in range(5)
    if x % 2
    for y in range(x + 2)
    if x != y
]
Out: [
    (1, 0), (1, 2),
    (3, 0), (3, 1), (3, 2), (3, 4)
]


この関数をsorted使用すると、カスタムの並べ替え方法を指定できます。これはkey、後で比較するために元の値を変換する方法を説明する引数を使用して行われます：

>>> x = [dict(name='Vadim', age=29), dict(name='Alex', age=4)]
>>> sorted(x, key=lambda v: v['age'])
[{'age': 4, 'name': 'Alex'}, {'age': 29, 'name': 'Vadim'}]

悲しいかな、比較を行うすべてのライブラリが引数をサポートしているわけではありませんkey。聞いたことがあるのは、heapq（部分的なサポート）とbisect（サポートなし）です。

この状況に陥るには2つの方法があります。適切な比較をサポートするカスタムオブジェクトを使用できます。

>>> class User:
...     def __init__(self, name, age):
...         self.name = name
...         self.age = age
...     def __lt__(self, other):
...         return self.age < other.age
...
>>> x = [User('Vadim', 29), User('Alex', 4)]
>>> [x.name for x in sorted(x)]
['Alex', 'Vadim']

ただし、オブジェクトはさまざまな方法で比較できるため、このようなクラスのいくつかのバージョンを作成する必要がある場合があります。これは不便な場合があるため、2つ目の方法があります。

カスタムオブジェクトを作成する代わりに、比較する値（優先度）と元の値(a, b)であるタプルを使用できます。ab

>>> users = [dict(name='Vadim', age=29), dict(name='Alex', age=4)]
>>> to_sort = [(u['age'], u) for u in users]
>>> [x[1]['name'] for x in sorted(to_sort)]
['Alex', 'Vadim']


定義と関数発生器の違いyieldは、関数本体にキーワードが存在することです。

In : def f():
...:     pass
...:

In : def g():
...:     yield
...:

In : type(f())
Out: NoneType

In : type(g())
Out: generator

つまり、空のジェネレータを作成するには、次のようにする必要があります。

In : def g():
...:     if False:
...:             yield
...:

In : list(g())
Out: []

しかし、yield from単純なイテレータ、つまりより快適なバージョンをサポートしているので：

def g():
    yield from []


Pythonでは、比較演算子のチェーンを作成できます。

>>> 0 < 1 < 2
True
>>> 0 < 1 < 0
False

このような鎖は、数学的に正しいことがありません、あなたは混在させることができます>と<：

>>> 0 < 1 > 2
False
>>> 0 < 1 < 2 > 1 > 0
True

演算子もサポートされてい==ます。isとin：

>>> [] is not 3 in [1, 2, 3]
True

各演算子は、隣接する2つのオペランドに適用されます。a OP1 b OP2 c完全に同等(a OP1 b) AND (b OP2 c)です。比較されa、c実行されません：

class Spy:
    def __init__(self, x):
            self.x = x

    def __eq__(self, other):
            print(f'{self.x} == {other.x}')
            return self.x == other.x

    def __ne__(self, other):
            print(f'{self.x} != {other.x}')
            return self.x != other.x

    def __lt__(self, other):
            print(f'{self.x} < {other.x}')
            return self.x < other.x

    def __le__(self, other):
            print(f'{self.x} <= {other.x}')
            return self.x <= other.x

    def __gt__(self, other):
            print(f'{self.x} > {other.x}')
            return self.x > other.x

    def __ge__(self, other):
            print(f'{self.x} >= {other.x}')
            return self.x >= other.x


s1 = Spy(1)
s2 = Spy(2)
s3 = Spy(3)

print(s1 is s1 < s2 <= s3 == s3)

結果：

1 < 2
2 <= 3
3 == 3
True
