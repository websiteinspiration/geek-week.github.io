こんにちは、ハブル！

私の名前はVitaliy Kotovです。私は多くのテスト自動化を行っており、気に入っています。私は最近、TypeScript + Protractor + Jasmineスタックでゼロからオートメーションを構成するプロジェクトに参加しました。私にとっては、このスタックは新しく、インターネットで必要な情報を検索しました。

私はなんとかして、最も有用で実用的なマニュアルを英語で見つけることができました。私はロシア語でもこれを行う必要があると判断しました。基本的なことだけを説明します。このようなスタックの理由、構成が必要なもの、最も単純なテストの外観を示します。

NodeJS、npm、およびサーバー側のJavaScript全般（特にTypeScript）を使用することはめったにないことを、すぐに言っておく必要があります。用語のどこかで間違いを見つけた場合、または私の決定の一部を改善できる場合は、経験豊富な人からのコメントでそれを知って喜んでいます。

ちなみに、「PHPUnit、Selenium、Composer：PHPUnit、Selenium、Composer」という自動化を数時間でデプロイするという同様の記事がすでにありました。



仕事


まず、どのような問題を解決しているかを考えましょう。 AngularJSを使用して記述されたWebアプリケーションがあります。これは、多くの場合、どのWebプロジェクトが作成されるかに基づいたJavaScriptフレームワークです。

この記事では、AngularJSプロジェクトの長所と短所を考慮しません。これらのプロジェクトのe2eテストの記述に関して、そのようなプロジェクトの機能について少しだけ説明します。

テスト自動化のかなり重要な側面は、ロケーターの助けを借りて行われるページ要素の操作です。ロケーターは、特定のルールに従って構成され、UI要素を識別する行です：1つ以上。

Webでは、CSSとXpathが最も一般的に使用されます。ページ上に一意のIDを持つ要素がある場合は、その要素で検索できることがあります。ただし、WebDriverはまだこのIDを最終的にCSSロケーターに変換し、すでにそれを使用しているようです。

AngularJSプロジェクトのHTMLコードを見ると、従来のHTMLにはない要素の多くの属性



が表示されます。コードは分度器デモページから取得されています。

AngularJSは、ng- *で始まるすべての属性を使用してUIを操作します。かなり典型的な状況は、これらのコントロール属性以外の要素に他の要素がない場合であり、品質ロケーターのコンパイルプロセスが多少複雑になります。

多くの自動化を行う人は、ロケーターを簡単に構築できるUIの価値を知っています。結局のところ、これは大規模なプロジェクトではまれです。:)

実際、このようなプロジェクトでは、テストの自動化も構成する必要があります。行こう！

何が何


まず、スタックの各コンポーネントが必要な理由を理解しましょう。

分度器はWebDriverJSに基づくテストフレームワークです。私たちのブラウザーを起動し、必要なページを開いて必要な要素と対話させるのは彼です。

このフレームワークは、AngularJSプロジェクト用に特別に調整されています。ロケーターを指定する追加の方法を提供します。

element(by.model('first'));
element(by.binding('latest'));
element(by.repeater('some'));

完全なリストはマニュアルページにあります。

これらのメソッドは、プロジェクトでのロケーターの作成とサポートを簡素化します。ただし、「内部」でこれらすべてがいずれにせよcssに変換されることを理解する必要があります。実際のところ、WebDriverでの相互作用に基づいて行われるW3Cプロトコルは、有限のロケーターのセットでのみ機能します。このリストはw3.orgで表示できます。

TypeScriptは、Microsoftによって作成されたプログラミング言語です。 TypeScriptは、変数を入力する機能、完全なクラスの使用のサポート、およびモジュールを接続する機能がJavaScriptと異なります。

V8エンジンで動作するようにTSコードで記述されたコードは、すでに実行されているJSコードに変換されます。この変換中に、コードが準拠しているかどうかがチェックされます。たとえば、intの代わりに文字列が関数のどこかに明示的に渡された場合、「コンパイル」は行われません。

Jasmineは、JavaScriptコードをテストするためのフレームワークです。実際、JSのコードがテストと呼ばれるものに変わったのは彼のおかげです。彼はこれらのテストを管理します。

以下にその機能を示します。

TypeScriptとその設定


TypeScriptをインストールするには、npmが必要です。

npm install -g typescript

インストールされていることを確認してください：

$  tsc -v
Version 3.4.1

すべてが整然としているようです。

次に、TSを操作するための構成を作成する必要があります。また、プロジェクトのルートにあり、tsconfig.jsonと呼ばれている必要があります。

内容は次のようになります。

{
    "compilerOptions": {
      "lib": ["es6"],
      "strict": true,
      "outDir" : "output_js",
      "types" : ["jasmine", "node"]
    },
    "exclude": [
      "node_modules/*"
    ]
}

要するに、この設定で次のように指定しました。

  • 最終的なJSコードを置くディレクトリ（この場合はoutput_jsです）
  • 厳格モードを有効にする
  • 私たちが作業しているフレームワークを示しています
  • node_modulesをコンパイルから除外

TSにはさまざまな設定があります。これらは私たちのプロジェクトには十分です。詳しくはtypescriptlang.orgをご覧ください。

次に、tscコマンドがどのように機能するかを見てみましょう。これにより、TSコードがJSコードに変わります。これを行うには、次の内容で簡単なcheck_tsc.tsファイルを作成します。

saySomething("Hello, world!");

function saySomething(message: string) {
    console.log(message);
}

次に、tscコマンドを実行します（このためには、プロジェクトディレクトリ内にいる必要があります）。

output_jsディレクトリがあり、次の内容の同様のjsファイルが中にあることがわかります。

"use strict";
saySomething("Hello, world!");
function saySomething(message) {
    console.log(message);
}

このファイルは、nodeコマンドを使用してすでに起動できます。

$  node output_js/check_tsc.js
Hello, world!

それで、最初のTypeSciptプログラムを書いて、おめでとうございます。今テストを書いてみましょう。:)

ここに私が得たコードがあります：

import { browser, by, element, protractor } from "protractor";

describe('Search', () => {
    it('Open google and find a text', async () => {
        //      
        let EC = protractor.ExpectedConditions;

        //    AngularJS
        await browser.waitForAngularEnabled(false);

        //   Google
        await browser.get('https://www.google.com/');
        
        //    css = input[role='combobox']
        let input_button = element(by.css("input[role='combobox']"));

        //     ( presenceOf)
        await browser.wait(EC.presenceOf(input_button), 5000);

        //     “protractor”
        await input_button.sendKeys("protractor");

        //      css
        let submit_button = element(by.css(".FPdoLc input[type='submit'][name='btnK']"));

        //      ( ,     input-,   )
        await browser.wait(EC.presenceOf(submit_button), 5000);

        //    
        await submit_button.click();

        // ,  URL    'protractor'
        await browser.wait(EC.urlContains('protractor'), 5000);
    });
});

コードでは、すべてがdescribe（）関数で始まることがわかります。彼女はJasmineフレームワークからやって来ました。これはスクリプトのラッパーです。その中には、すべてのテストの前および各テストの前に操作を実行するbeforeAll（）およびbeforeEach（）関数がある場合があります。それと同じくらい多くの関数（）が実際にはテストです。最後に、定義されている場合、各テストおよびすべてのテストの後で、操作のためにafterAll（）およびafterEach（）が実行されます。

Jasmineのすべての機能については説明しません。Webサイトjasmine.github.ioでそれらについて読むことができ

ます。テストを実行するには、まずTSコードをJSコードに変換してから実行する必要があります。

NPM scripts


作業を簡単にするために、コマンドの一部をnpmエイリアスにすることができます。たとえば、以前のJSファイルのあるディレクトリを削除し、各テストを実行する前に新しいファイルで再作成したいと考えています。

これを行うには、スクリプトアイテムをpackage.jsonに追加します。

{
  "name": "protractor",
  "scripts": {
    "test": "rm -rf output_js/; tsc; protractor output_js/config.js"
  },
  "dependencies": {
    "@types/node": "^10.5.2",
    "@types/jasmine": "^3.3.12",
    "protractor": "^5.4.2",
    "typescript": "^3.4.1"
  }
}

npm test コマンドを入力すると、次のようになります。古いコードを含むoutput_jsディレクトリが削除され、新しく作成されて、新しいJSコードが書き込まれます。その後、テストはすぐに開始されます。

この一連のコマンドの代わりに、個人的に作業する必要があるその他のコマンドを指定できます。たとえば、テスト実行の間にセレンサーバーを起動して停止できます。もちろん、これはテストコード自体の中で制御する方が簡単です。

プロミスについて少し


最後に、Promise、async / await、およびNodeJSでのテストの記述が同じJavaまたはPythonとどのように異なるかについて少しお話します。

JavaScriptは非同期言語です。つまり、コードは常に記述された順序で実行されるわけではありません。これにはHTTPリクエストが含まれ、コードはHTTP経由でSelenium Serverと通信することを覚えています。

プロミス（通常は「プロミス」と呼ばれます）は、非同期コードを編成する便利な方法を提供します。それらについて詳しくは、learn.javascript.ruを参照してください。

実際、これらは、1つのコードを別のコードの実行に依存させるオブジェクトであり、特定の順序を保証します。分度器はこれらのオブジェクトを非常に積極的に操作します。

例を見てみましょう。次のコードを実行するとします。

driver.findElement().getText();

Javaでは、String型のオブジェクトを返すことが期待されています。 Protractorでは、これはまったくそうではありません。Promiseオブジェクトを返します。それと同じように、デバッグ用に出力しても機能しません。

通常、結果の値を出力する必要はありません。この値で既に機能する他のいくつかのメソッドに渡す必要があります。たとえば、テキストが期待どおりかどうかをチェックします。

Protractorの同様のメソッドもPromiseオブジェクトを入力として受け入れるため、問題はありません。しかし、それでも値を見たい場合は、（）が便利です。

これは、Googleページにボタンのテキストを印刷する方法です（これはボタンなので、テキストはvalue属性内にあることに注意してください）。

//  
let submit_button = element(by.css(".FPdoLc input[type='submit'][name='btnK']"));

//   
await browser.wait(EC.presenceOf(submit_button), 5000);
    
//   then()  
await submit_button.getAttribute("value").then((text) => {
    console.log(text);
});

async / awaitキーワードに関しては、これは非同期コードを操作するための少し新しいアプローチです。これにより、多数のネストが原因で以前にコードで形成されたpromise hellを回避できます。それでも、Promiseを完全に取り除くことはできず、Promiseと連携できる必要があります。これは理解可能で詳細な記事は、JavaScriptでの非同期/待機のデザイン：強み、落とし穴、使用上の機能の記事にあります。
