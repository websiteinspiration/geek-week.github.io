プログラマーであるときに色の音楽を作る方法

ある映画を見て、映画の主人公の1人が脚と腕をメロディーのリズムに合わせて巧みに持ち上げたときの瞬間が記憶に刻まれました。だから：ヒーローだけでなく、彼の後ろにいるモニターのセットも注目に値します。


最近、他の多くのように、4つの壁に費やされる時間が大幅に増加し、「そのようなシーンを実現するための最も奇妙な方法は何ですか？」という考えが浮上しました。


これから先は、Webブラウザー、つまりWebRTCとWebAudio APIの使用に選択が下がったと言います。


そこで私は考えて、すぐにオプション（ラジオを手に取り、そのような視覚化を持つプレーヤーを見つける）から長いオプション（ソケットを介して色に関する情報を送信するクライアントサーバーアプリケーションを作成する）に行きました。そして、「ブラウザーには実装に必要なすべてのコンポーネントが揃っている」と思ったので、それを使ってやってみます。


WebRTCは、ブラウザーからブラウザー（ピアツーピア）にデータを転送する方法です。つまり、最初はサーバーを作成する必要がないと思いました。しかし、彼は少し間違っていました。RTCPeerConnectionを作成するには、シグナルとICEの2つのサーバーが必要です。2つ目は、既成のソリューションを使用できます（STUNまたはTURNサーバーは多くのLinuxディストリビューションのリポジトリにあります）。最初に何かをする必要があります。


ドキュメントには、任意の双方向対話プロトコルが信号として機能し、すぐにWebSockets、Longプーリング、または別の何かを実行できることが記載されています。いくつかのライブラリのドキュメントからhello worldを取得するのが最も簡単なオプションのようです。そして、ここにそのような単純なシグナルサーバーがあります：


import os
from aiohttp import web, WSMsgType

routes = web.RouteTableDef()
routes.static("/def", os.path.join(os.path.dirname(os.path.abspath(__file__)), 'static') )

@routes.post('/broadcast')
async def word(request):
    for conn in list(ws_client_connections):
        data = await request.text()
        await conn.send_str(data)
    return web.Response(text="Hello, world")

ws_client_connections = set()

async def websocket_handler(request):
    ws = web.WebSocketResponse(autoping=True)
    await ws.prepare(request)

    ws_client_connections.add(ws) 
    async for msg in ws:
        if msg.type == WSMsgType.TEXT:
            if msg.data == 'close':
                await ws.close()
                # del ws_client_connections[user_id]
            else:
                continue
        elif msg.type == WSMsgType.ERROR:
            print('conn lost')
            # del ws_client_connections[user_id]
    return ws

if __name__ == '__main__':

    app = web.Application()
    app.add_routes(routes)
    app.router.add_get('/ws', websocket_handler)
    web.run_app(app)
