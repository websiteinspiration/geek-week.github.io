Angularのカスタムコントロール

プロローグ


角張った反応型について話し、カスタムコントロールについて、それらを作成、使用、検証する方法を学びましょう。この記事では、すでに角度フレームワークに精通しているものの、その詳細を詳しく知りたいと想定しています。では、始めましょう。

反応型およびテンプレート駆動型フォーム


同じページにいることを確認するためのいくつかの単語。アンギュラーには、テンプレート駆動型フォームと反応型フォームという 2種類のフォームがあります。

テンプレート駆動型フォームは、双方向バインディングベースのフォームです（hi、angularjs）。クラスのフィールド（usernameなど）を指定し、入力タグのhtmlで[（value）] = "username"をバインドします。入力値が変わると、ユーザー名の値も変わります。 2011年、それはいまいましい魔法でした。わかりましたが、ニュアンスがあります。このようにして、複雑な形状を構築することは...困難になります。

Reactive Formsは、単純なフォームと複雑なフォームを作成するための便利なツールです。 「フォームクラスのインスタンスを作成し（FormGroup）、コントロールのインスタンスをそれに渡します（FormControl）そして、それをhtmlで与えるだけで、あとは私が行います。」では、試してみましょう。

class UsefullComponent {
 public control = new FormControl('');
 public formG = new FormGroup({username: control});
}


<form [formGroup]="formG">
 <input type="text" formControlName="username">
</form>

結果として、ブラックジャックを備えた反応的なフォームが得られます。たとえば、formG.valueChangesは、フォームの変更の監視可能な（ストリーム）を提供します。新しいコントロールを追加したり、既存のコントロールを削除したり、検証ルールを変更したり、フォームの値（formG.value）を取得したりすることもできます。そして、これらすべてがformGの 1つのインスタンスで機能します。

そして、上記のクラスのインスタンスを毎回手動で作成しないようにするために、angularの開発者は、上記の例を次のように書き換えることができる便利なFormBuilderを提供してくれました。

class UsefullComponent {
 public formG: FormGroup;

 constructor(private fb: FormBuilder) {
  this.formG = fb.group({
   name: '' //  new FormControl() !!
  })
 }

}

カスタムコントロール


Angularは、「ブロ、ネイティブコントロール（入力、日付、数値など）が十分にない場合、または突然それらが気に入らなくなった場合に、独自のコントロールを作成するためのシンプルで非常に強力なツールを紹介します。」カスタムコントロールは、リアクティブでもテンプレート駆動フォームでも使用でき、ディレクティブを使用して実装されます（驚いた！）。コンポーネントがテンプレート付きのディレクティブ（必要なもの）であることを知って、次のように記述します。

import { Component, Input } from '@angular/core';

@Component({
 selector: 'counter-control',
 template: `
  <button (click)="down()">Down</button>
   {{ value }}
  <button (click)="up()">Up</button>
 `
})
class CounterControlComponent {
  @Input()
  value = 0;
 
 up() {
  this.value++;
 }
 
 down() {
  this.value - ;
 }
}

他のディレクティブと同様に、これはモジュールで宣言する必要があります（記事が大きくならないように、このようなニュアンスは省略します）。

これで、作成したコンポーネントを使用できます。

import { Component } from '@angular/core';

@Component({
 selector: 'parent-component',
 template: `
  <counter-control></counter-control>
 `
})
class ParentComponent {}

もちろんすべてうまくいきますが、フォームはどこにありますか？少なくともテンプレート駆動....
パニックがなければ、すべてはControlValueAccessorに会った後になります。

制御値アクセサー


角度メカニズムがカスタムコントロールと相互作用するためには、特定のインターフェイスを実装するためにこのコントロールが必要です。このインターフェースはControlValueAccessorと呼ばれます。これは、オブジェクト（この場合はコントロール）がインターフェイスを実装し、他のオブジェクト（角度形式）がこのインターフェイスを介してオブジェクトと対話する場合の、OOPからのポリモーフィズムのかなり良い例です。

おかげControlValueAccessorコントロールを操作する方法は1つしかありませんが、これは、カスタムコントロールを作成するときだけではありません。フードの下のAngularもこのインターフェースを使用します。何のために？そして、ネイティブコントロールの動作を一見するだけです。たとえば、入力では、値はvalue属性に含まれ、チェックボックスでは、checked属性によって値が決定されます。したがって、各タイプはそれ自身のコントロールControlValueAccessor有する：DefaultValueAccessor  - input'ovとTEXTAREAため、CheckboxControlValueAccessor  -チェックボックスのため、RadioControlValueAccessorラジオボタンのために、というように。

しかし、ControlValueAccessorを使用してAngularが行うこと？すべてが非常に単純で、モデルからの値をDOM（ビュー）に書き込み、また、FormGroupおよびその他のディレクティブに変更制御イベントを発生させます。ControlValueAccessor

について学習したので、それをコントロールに適用できます。

そのインターフェースを見てみましょう：

interface ControlValueAccessor {
  writeValue(value: any): void
  registerOnChange(fn: any): void
  registerOnTouched(fn: any): void
}

writeValue（値：任意） -ソース（新しいFormControl（ 'I am default value'））またはcontrol.setValueの上の新しい値（ 'I am setted value'）が設定されたときに呼び出されます。

registerOnChange（fn：any） -値が変更されたときに呼び出すハンドラーを定義するメソッド（fnは、このコントロールで値が変更されたことをフォームに通知するコールバックです）。

registerOnTouched（fn：any）  -blurイベントで呼び出されるコールバックを定義します（コントロールにはtouchedのフラグが立てられます）

import { Component, Input } from '@angular/core';
import { ControlValueAccessor } from '@angular/forms';

@Component({
 selector: 'counter-control',
 template: `
  <button (click)="down()">Down</button>
   {{ value }}
  <button (click)="up()">Up</button>
 `
})
class CounterControlComponent implements ControlValueAccessor {
 @Input()
 value = 0;

 onChange(_: any) {}

 up() {
  this.value++;
 }

 down() {
  this.value - ;
 }

 writeValue(value: any) {
  this.value = value;
 }

 registerOnChange(fn) {
  this.onChange = fn;
 }

 registerOnTouched() {}
}


親フォームがコントロールの変更を認識できるようにするには、valueの値が変更されるたびにonChangeメソッドを呼び出す必要があります。各メソッド（アップおよびダウン）でonChange呼び出しを記述しないようにするために、ゲッターとセッターを通じて値フィールドを実装します。

// …
class CounterControlComponent implements ControlValueAccessor {
 private _value;

 get value() {
  return this._value;
 }

 @Input()
 set value(val) {
  this._value = val;
  this.onChange(this._value);
 }

 onChange(_: any) {}
 
 up() {
  this.value++;
 }
 
 down() {
  this.value - ;
 }
 // …
}

現時点では、Angleは、ControlValueAccessorを実装するコンポーネントをカスタムコントロールと見なす必要があることを認識していません。そのことを彼に伝えましょう。

import { Component, Input, forwardRef } from '@angular/core';
import { ControlValueAccessor, NG_VALUE_ACCESSOR } from '@angular/forms';

@Component({
 …
 providers: [{ 
  provide: NG_VALUE_ACCESSOR,
  useExisting: forwardRef(() => CounterControlComponent),
  multi: true
 }]
})
class CounterControlComponent implements ControlValueAccessor {
 …
}

コードのこのセクションでは、「新しいカスタムコントロールを登録したい（提供：NG_VALUE_ACCESSOR）、コンポーネントの既存の（この時点でコンポーネントが初期化される）インスタンスを使用する（useExisting：forwardRef（（）=> CounterControlComponent））」という角度で言います。 。 

multi：trueは、そのようなトークン（NG_VALUE_ACCESSOR）にはいくつかの依存関係が存在する可能性があることを示します。

作成されただけではありません


カスタムコントロールをザイザットする時が来ました。このコントロールが使用されるモジュールのインポートにFormsModule / ReactiveFormsModuleを追加することを忘れないでください。

テンプレート駆動フォームでの使用


ここではすべてが単純です。ngModelを介した双方向バインディングを使用すると、モデルが変更されたときにビューが変更され、その逆も同様です。

import { Component } from '@angular/core';

@Component({
 selector: 'parent-component',
 template: `
  <counter-control [(ngModel)]="controlValue"></counter-control>
 `
})
class ParentComponent {
 controlValue = 10;
}

反応型での使用


記事の冒頭で述べたように、FormBuilderを介してリアクティブフォームのインスタンスを作成し、それをhtmlでレンダリングしてお楽しみください。

import { Component, OnInit } from '@angular/core';
import { FormBuilder } from '@angular/forms';

@Component({
 selector: 'parent-component',
 template: `
  <form [formGroup]="form">
   <counter-control formControlName="counter"></counter-control>
  </form>
 `
})
class ParentComponent implements OnInit {
 form: FormGroup;

 constructor(private fb: FormBuilder) {}

 ngOnInit() {
  this.form = this.fb.group({
   counter: 5
  });
 }
}

これは、カスタムコントロールを備えた本格的な反応型フォームですが、すべてのメカニズムはネイティブコントロールと同じように機能します（ポリモーフィズムについて言及しましたか？）。現在の記事を読み込まないようにするために、次の記事ではカスタムコントロールエラーの検証と表示について説明します。
