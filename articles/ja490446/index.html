Raspberry Piでネットワークを制御する

こんにちは、ハブル！APCマガジンの記事の翻訳にご注意ください。



データとハードウェアの整合性を確保するには、ネットワークセキュリティが不可欠です。間違いなく、ルーターのファイアウォールとソフトウェア保護ツールが必要ですが、ネットワークで何が起こっているかを詳しく知るには、それを監視および制御するツールが必要です。

ハッキングの試みについてコンピュータに信号を送信するネットワーク監視ツールは、SF映画のプロットの要素にすぎないように見えるかもしれませんが、そのようなツールは実際に発生します。つまり、これは、ネットワーク内にあるデバイスがインターネットから切断されたとき（スマートホームやモノのインターネットなど）をすばやく検出し、ルーターまたはネットワークハードウェアへの不正な接続を検出できることを意味します。必要なのは、Raspberry PiのシングルボードコンピューターとNagiosだけです。

Nagiosは、Raspberry Piシングルボードコンピューターと同様に、いくつかのバージョンで利用できます。理解を容易にするために、プログラムをインストールするための2つのオプション、ここではディスクイメージの作成と現在のOSへの手動インストールを検討します。

NEMS（Nagios Enterprise Monitoring Server）は、Nagiosエンタープライズモニタリングサーバーであり、Raspberry Piにインストールされ、bit.ly lxf253nemsから入手できます。 16 GB以上の容量のメモリカードが必要ですが、32 GBを選択することをお勧めします。 Raspberry Pi 3B +を使用して操作することをお勧めしますが、Raspberry Pi1モデルAおよびRaspberry Pi計算モジュール以外のバージョンも適しています。 NEMSパッケージはBitTorrent経由でのみダウンロードできることに注意してください。ただし、他のNagiosディスクイメージは無料で入手できます。パッケージをダウンロードしたら、ツールを使用してディスクイメージを作成し、IMGファイルをSDカードに書き込みます。

シングルボードコンピューターでNEMSを実行する準備ができたら、イーサネットケーブルを介してRaspberry Piをルーターに接続します。サーバーはWi-Fi経由で機能しますが、イーサネットはネットワークを監視するための信頼性の高い手段です。 SDカードを挿入し、Raspberry Piを起動して、NEMSセットアップが完了するまで待ちます。このプロセスではファイルシステムのサイズが自動的に変更されるため、最初のダウンロードには通常より時間がかかります。 Secure Shell Protocol（SSH）を使用し、ユーザー名とパスワードとしてnemsadminを使用してNEMSに接続します。
次に、次のコマンドを入力します。

sudo nems-init

その後、NEMSのインストールが始まります。この段階で、地域設定を実行し、エンコード方法を選択し、アカウントを作成し、通知が送信される電子メールアドレスを追加する必要があります。これを行うには、デスクトップのブラウザーでnems.localリンクを開く（またはRaspberry PiのIPアドレスを使用する）ことで構成を開始します。

手動インストール

または、Raspbianなどの任意のRaspberry PiオペレーティングシステムにNagiosを手動でインストールできます。最良の結果を得るには、クリーンなOSインストールから始めて、SSHセッションを開始します。これを行うには、まずデータベースをアップグレードします。

sudo apt update && sudo apt upgrade

デバイスを再起動します。

sudo reboot

次に、Nagiosをインストールします。

sudo apt install nagios3

ダイアログボックスが管理者アカウントを作成するのを待ち、後で必要になるため、パスワードを作成して書き留めます。このプロセスにはそれほど時間がかかりません。その後、別のデバイスからIPアドレス192.168.1.10/nagios3を介してNagiosにアクセスします。必要に応じて、前に作成したnagiosadminのユーザー名とパスワードを使用します。

ネットワーク監視



Raspberry PiにNagiosをインストールすると、継続的な監視システムを実行する準備が整います。必要に応じてカスタマイズするだけです。プログラムが手動でインストールされた場合、ホストはターミナルを介して構成されます。これを行うには、構成ファイルを作成します。たとえば、これをmonitor.cfgと呼びます。

sudo nano /etc/nagios3/ conf.d/monitor.cfg

追跡するデバイスの詳細を追加します。Raspberry PiにMinecraftサーバーがあるとします。次のコマンドを入力します。

define host {  
use      generic-host
 host_name   minecraft 
 alias     minecraft  
address    192.168.1.22
}

ここでの一般的なホストコマンドはテンプレートです。これは、generic-host_nagios2.cfgフォルダーにあります。テンプレートは、デバイスでモニタリングセッションのインストールを作成するときに時間を節約するために使用されます。これを行うには、特定の定義に基づいて複数のエントリを作成し、host_name（コンピューティングノードの名前）、エイリアス（追加の名前）、および監視するデバイスのアドレス（IPアドレス）を変更するだけです。
Ctrl-Xを押して変更を保存し、終了してからNagiosを再起動します。

sudo service nagios3 reload

構成プロセス中に問題が発生した場合は、次のコマンドを使用してサーバーのエラーを確認する必要があります。

sudo /usr/sbin/nagios3 -v / etc/nagios3/nagios.cfg

彼女は設定ファイルの正確さをチェックします。その後、Nagiosを再起動することを忘れないでください。
通常、ネットワーク監視はブラウザを介して行われます。タブは無期限に開いたままにすることができます。リモート監視は、VNC（Remote Computer Management System）を介してRaspberry Piデスクトップで開始することもできます。

ターミナルに加えて、NagiosはWebブラウザーを介して構成されます。セットアップメニューとNEMSコンフィギュレーターを開き、ノードの説明を手動インストールに必要なものと同じ資格情報と必要な通知で追加するだけです。

Nagiosを監視するには、[レポート]メニューセクションを使用します。プログラムの外部部分は、2つのバージョンで提示できます。最新のAdagiosユーザーインターフェイスと、数年前に作成されたNagios Coreインターフェイスです。どちらも完全に機能しています。

Nagiosは大きな可能性を秘めているので、プログラムの機能に慣れ、デバイスを追加し、ネットワークアクティビティログを確認し、視覚化を有効にするなど、時間をかける価値はあります。NEMSを使用すれば、ブラウザーインターフェイスを介してホストを簡単に構成できます。仕方。

以上で、NagiosがインストールされたRaspberry Piはネットワーク監視の準備ができました。バックグラウンドで動作し、機器の接続に関する問題やネットワーク内の不審なアクティビティを通知します。
