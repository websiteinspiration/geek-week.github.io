Dies ist jedoch nicht die einzige Möglichkeit, eine auf Node-Red basierende Benutzeroberfläche zu erstellen. Es gibt ein ziemlich beliebtes Modul, das eine Reihe von Knoten zum Erstellen von Dashboards in Echtzeit von den Erstellern der Node-Red-Dashboard- Plattform selbst enthält .







Das Vorlagen-Widget kann alle gültigen HTML- und Angular / Angular-Material-Anweisungen enthalten. Mit diesem Knoten kann ein dynamisches Benutzeroberflächenelement erstellt werden, das sein Erscheinungsbild basierend auf der Eingabenachricht ändert und Nachrichten an Node-RED zurücksenden kann. Mit dem Vorlagenknoten können Sie ein Objekt der Scadavis-Bildschirmseite erstellen, die erforderlichen Eigenschaften festlegen, das Verhalten des Objekts emulieren und die Tags (Variablen) der Bildschirmseite der externen Umgebung zuordnen

Der Artikel AngularJS (Node-RED Dashboard Template Examples) zeigt Beispiele dafür, wie der Template-Knoten mit anderen Knoten interagiert, die Nachrichtennachrichten im Kontext von Angular Scope austauschen.

Angenommen, wir übergeben den Tag-Namen als Eigenschaft des Objekts msg.topic und msg.payload als Wert. Und mit jedem msg-Objektänderungsereignis aktualisieren wir das SCADA-Objekt mithilfe der Methoden storeValue und updateValues 



 (function(scope) {
        scope.$watch('msg', function(msg) {
            if (msg) 
                office.storeValue(msg.topic, msg.payload);
                office.updateValues();
            }
        });
        })(scope);


Mithilfe des Beispiels aus der Ressource https://scadavis.io/learn.html und der Funktionalität des Vorlagenknotens können Sie verschiedene Ereignisse des SCADA-Objekts verarbeiten und an andere Knoten übergeben, z. B.: 
(function(scope) {
substationsynoptic.on("click", function (event, tag) {
    var v = substationsynoptic.getValue(tag);
    if (event.currentTarget.id === "TAPUP")
        substationsynoptic.setValue(tag, v + 1, false, false);
    else
        if (event.currentTarget.id === "TAPDOWN")
            substationsynoptic.setValue(tag, v - 1, false, false);
    if (event.currentTarget.id === "XCBROPEN")
        substationsynoptic.setValue(tag, false, false, false);
    else
        if (event.currentTarget.id === "XCBRCLOSE")
            substationsynoptic.setValue(tag, true, false, false);
    if (v === true)
        substationsynoptic.setValue(tag, false, false, false);
    else
        if (v === false)
            substationsynoptic.setValue(tag, true, false, false);
    scope.send( {'payload':{ 'tag': tag, 'value': v }} );
});
})(scope);




Grafikeditor



Ein paar Worte zum Grafikeditor zum Erstellen visueller Seiten für das SCADA-System. Wie oben erwähnt, wird vorgeschlagen, den SCADAvis.io Synoptic Editor außerhalb des Unternehmens zu erwerben. Wieder allein - dies ist nicht die einzige Option. Wie Sie aus der Beschreibung selbst wissen, basiert der Editor auf dem Inkscape SVG-Editor. In der Praxis ist dies Inkscape mit einem Plugin, mit dem Sie verschiedenen Grafikkomponenten einer SVG-Datei Tags (Variablen) und andere Eigenschaften zuweisen können. Jene. Um die Seite einfach im Kontext von Scadavis anzuzeigen, reicht eine SVG-Datei aus, die in jedem praktischen Editor bearbeitet werden kann. Aber das ist nicht alles. Es gibt viele Ähnlichkeiten zwischen dem Herausgeber von ECAVA INTEGRAXOR www.integraxor.comInkscape 0.91 SAGE 4.16 wird kostenlos verteilt (Sie müssen sich nur auf der Website registrieren) und Synoptic Editor. Wie sich herausstellte, ist es mit Inkscape 0.91 SAGE 4.16 wirklich möglich, SVG-Dateien für Scadavis zu erstellen und zu bearbeiten. Der Unterschied wird beim Entwerfen von Skriptanimationen und einigen anderen spezifischen Dingen beobachtet, die in den Objekteigenschaften des SAGE-Editors fehlen.







Gemäß der Beschreibung von SCADAvis.io Synoptic Editor muss die Nummer 2 in der Spalte Limit angegeben werden, um die Farbe von Objekten gemäß der Einschlussbedingung zuzuweisen:



  • 0 - nicht initialisierter Zustand
  • 1 - falscher (Aus) Zustand
  • 2 - wahrer (Ein) Zustand
  • 129 - falscher (Aus) Zustand plus fehlgeschlagener Wert
  • 130 - wahrer (Ein) Zustand plus fehlgeschlagener Wert






SPS-Plugin für Siemens



Das Node-Red-Arsenal verfügt über ein Plug-In für die Arbeit mit der Siemens-SPS - Node-Red-Contrib-S7 , basierend auf dem NodeS7- Framework . Das Plugin ist recht einfach und intuitiv zu bedienen. Es enthält drei Knoten: S7 in, S7 out, S7 control. Die ersten beiden Knoten werden hauptsächlich zum Lesen und Schreiben von Tags in der SPS verwendet. Eine Nuance, die bei der Arbeit mit S7-1200 berücksichtigt werden muss, ist im Plug-In-Dock beschrieben. Ich bin früher darauf gestoßen, als ich die Snap7- Bibliothek verwendet habe .







In der SPS sind einige zusätzliche Konfigurationsschritte erforderlich:



"Optimierter Blockzugriff" sollte für die DB-Blöcke deaktiviert sein, auf die wir zugreifen möchten




Aktivieren Sie im Abschnitt "Schutz" der CPU-Eigenschaften das Kontrollkästchen "Zugriff über PUT / GET zulassen"




Kurzer Abschluss



Die Vorteile der Verwendung von Node-Red als Web-SCADA-ähnliches System liegen auf der Hand. Node-Red ist eine beliebte Open-Source-IoT-Plattform mit einer riesigen Community, vielen verschiedenen Plugins und Lösungen sowie der Möglichkeit, verschiedene Protokolle zu integrieren. Auf der Ressource support.industry.siemens finden Sie außerdem ein Handbuch zur Verwendung von Node-Red und eine Diskussion zur Verwendung der Plattform. Ein SCADA-Framework wie scadavis.io ist ein gutes Beispiel für die Integration eines bekannten Visualisierungsansatzes unter Spezialisten - SCADA-Systemen und der IoT-Edge-Plattform.
