F＃6：タプル

したがって、F＃への旅は続きます。次の記事では、標準の.NET同等物がある場合とない場合があるF＃型に焦点を当てます。この投稿はタプルについてです。

タプルとは


タプルは、名前が付けられていないが順序付けられた値のグループであり、異なるタイプの場合があります。

タプルの作成


タプルの作成は非常に簡単です。以下に示すような処理を行うだけです。数値と文字列の混合タプルパッケージを作成した方法に注意してください。また、型と値の数の両方を組み合わせて照合することもできます。

let a = (1,2)
let b = (1,"cat")
let c = (1,"cat")
let d = (1,"cat", 'c')

ユニバーサルタプル


タプルを受け付ける関数は、ユニバーサルタプルも問題なく受け入れることができます。F＃のような推論システムは、タプルの正しい型の推論をうまく処理します。例を示します。

let someFunction tup = 
    let (x,y) = tup
    printfn "x is %A and y is %A" x y
 
do someFunction ("cat","dog")
do someFunction (11,12)

タプル署名


これまでのところ、F＃シグネチャの理解にはまったく触れていません。実際、これは非常に複雑であると思うので、ブログの投稿全体を取り上げることにしたトピックです。しかし、私たちは現在の場所、つまり今ここにあり、タプルを研究しているので、タプルのシグネチャがどのように見えるかを示したかっただけです。

FSIウィンドウで次のタプルを宣言したとします。
let a = (1,2)
let b = (1,"codfather")
let c = (1,"c", 12.5)

次に、FSIウィンドウでそれらを調べました。次のようなものが表示されます。
val a : int * int = (1, 2)
val b : int * string = (1, "codfather")
val c : int * string * float = (1, "c", 12.5)

これはおもしろい、ここにはいくつかのことがわかります。
  • 括弧は型シグネチャの一部ではありません
  • F＃型システムは、タプル自体に含まれている値に基づいて型を正しく推論できます。
  • コンマは「*」に置き換えられます

したがって、非常に透明なタプルは次のようになります。
let a = (1,2)

次の署名があります： 
int * int


タプルの内訳


それで、タプルを作成する方法を見ましたが、それらを個別の値に分解または分解することについてはどうでしょうか。出来ますか？はい、もちろん。前と同じように、いくつかの例から始めましょう。

let (a,b) = (1,2)
 printfn "(a,b) = (1,2), so value of 'a' should be 1, and it is =%i,\r\n 'b' should be 2, and it is =%i" a b 
//   ,  ,     ,  
//    
let (_,z) = (1,2)
printfn "grabbing last value from (1,2) which is = %i" z 
let (a,b :string) = (1,"cat")
printfn "grabbing (1,\"cat\") which has values = %i %s" a b 
let (a :int,b :string) = (1,"cat")
printfn "grabbing (1,\"cat\") which has values = %i %s" a b 
let (a ,b, c) = (1,"cat", 'c')
printfn "grabbing (1,\"cat\",'c') which has values = %i %s %c" a b c 
let first = fst (1, 2)
printfn "grabbing fst from (1,2) which has values = %i" first 
let second = snd (1, 2)
printfn "grabbing 2nd from (1,2) which has values = %i" second

Letを使用する


しかし、どのようにして個々の部品を手に入れましたか？さて、必要なのは上のコードだけですが、1つの例を見てみましょう。次のようなタプルがあるとします。
(1,2)

そして、タプルの両方の「値」の値をいくつかの新しい個々の値に結び付けたいと思っていたので、これを行うことができます：
let (a,b) = (1,2)

また、本当に必要な値のみを選択することもできます。これは、不要な部分にワイルドカードを使用して行われます。これにより、不要な値のバインドがないことが保証されます。次に例を示します。
let (_,z) = (1,2)


組み込みタプル関数の使用


タプルから最初と2番目の値を取得するための組み込みサポートもあります。何がで行うことができるFSTとSND機能。2番目の要素以外のサポートはありません（これらはおそらく最も一般的なケースです）。「Fst」と「snd」は次のように使用できます。
let first = fst (1, 2)
let second = snd (1, 2)

ここで、個々の値に分解しようとしている値の数と不一致がある特別なケースに注意を向けたいと思います。したがって、これは次の例のようなものになります。
let (a ,b) = (1,"cat", 'c')

タプル自体には実際には3つの値が含まれていることがわかりますが、Letバインディングには2つの値しかないため、以下のスクリーンショットでわかるように、コンパイラーはこれについて警告します。

新しいタプルの作成


あなたは既存のタプルから新しいタプルを作成したいかもしれません、それは十分に簡単です、ここに例があります：
let oldOne = (1,2)
let (x,y) = oldOne
let newOne = (x+1,y+1)
printfn "original = %A, and we did this (x+1,y+1)\r\n to obtain newOne = %A" oldOne newOne



タプルの比較


タプルは等しい場合にのみ等しいと見なされます

  • 同じ数の値があります。
  • すべての値は等しいと見なされます（明らかに、これらはカスタムEqualsメソッドまたはカスタムIEquatable実装などにすることができます）

簡単な例を見てみましょう。

printfn "(1,2) = (3,4) =% b" ((1,2) = (3,4))
printfn "(1,2) = (1,2) =% b" ((1,2) = (1,2))
printfn "('a', 'b') = ('a', 'b') =% b" (('a', 'b') = ('a', 'b'))
printfn "('a', 'b') = ('a', 'c') =% b" (('a', 'b') = ('a', 'c'))



実際、タプルの長さが異なる場合（要素数）、等号演算子「=」を使用してタプルを比較しようとすると、警告が表示されます。


タプルマッチングパターン


マッチングパターンについてはまだ扱っていませんが、後でこのトピックに関する投稿全体が表示されます。それまでの間、これが入力パラメータを再度照合する方法であることを知っておいてください。タプルに対してこれを行うことができます。これは次のように行われます。
let someFunction tup = 
    match tup with
    | 1,2 -> printfn "you passed (1,2)"
    | _,_ -> printfn "you passed some other tuple"
     
do someFunction (11,12)
do someFunction (4,5)
do someFunction (1,2)
do someFunction (13,23)
