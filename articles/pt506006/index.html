Pythonists, pare de usar colchetes para obter valores de um dicionário

Olá Habr! Apresento a você a tradução do artigo “Pare de usar a notação de colchete para obter o valor de um dicionário em Python”, de Jonathan Hsu.

Indo além do método tradicional e protegendo seu código


Um dicionário é uma coleção não ordenada de chaves e valores. Significa que:

  • Cada elemento do dicionário consiste em um termo (chave) e sua definição (significado).
  • As chaves devem ser exclusivas para este dicionário - sem repetições.
  • As chaves no dicionário não possuem a organização explícita dos itens da lista.

Para criar um dicionário, use chaves, dentro das quais escreva os pares chave: valor separados por vírgulas. Como isso:

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}

Uma maneira tradicional (e insegura) de obter significado de um dicionário.

Para acesso normal ao valor de um elemento de um dicionário, colchetes são usados. Ao usar essa entrada, a chave é indicada entre colchetes após o nome do dicionário:

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author['username']) #  Friday1719
print(author['middle_initial']) #   KeyError: 'middle_initial'

Observe que tentar acessar um valor por uma chave que não está no dicionário lançará uma exceção KeyError. Isso pode criar problemas sérios, especialmente ao lidar com dados de trabalho imprevisíveis.

Obviamente, você pode usar a construção try / except ou usar a instrução if. Mas uma atenção tão grande a uma operação tão simples apenas atravessa o código.

author = {}
try:
   print(author['username'])
except KeyError as e:
   print(e) # 'username'

if 'username' in author:
   print(author['username'])

Se você tem experiência em desenvolvimento de JavaScript, poderá obter o valor do dicionário usando notação pontilhada. Mas em Python, isso não vai funcionar.

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author.username)
# AttributeError: 'dict' object has no attribute 'username'

Usamos o método get ()


Quando você precisa obter um valor de um dicionário, a maneira mais segura é usar o método get (). Este método possui dois parâmetros:

  • Primeiro (obrigatório): o nome da chave pela qual queremos obter o valor. Este nome pode ser uma cadeia ou um nome de variável, se a nossa chave puder mudar durante o programa.
  • Segundo (opcional): o valor que será usado se nossa chave não aparecer repentinamente no dicionário.

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}
print(author.get('username')) #   Friday1719
print(author.get('middle_initial', “Monday1917”)) #  Monday1917

Se a chave existir no dicionário, o método get () funcionará exatamente da mesma maneira que acessando a chave entre colchetes. Porém, no caso em que não existe essa chave no dicionário, o método get () retornará o valor padrão, evitando que você precise manipular a exceção.

O valor padrão pode ser qualquer objeto que seja válido nesse contexto. Lembre-se de que este parâmetro é opcional. Portanto, se você não o especificar explicitamente, quando tentar acessar uma chave que não existe no dicionário, o método get () retornará um objeto None.

Usamos o método setdefault ()


Às vezes, você precisará não apenas receber dados do dicionário com segurança, mas também adicionar novos dados com segurança ao dicionário. Para isso, os dicionários possuem um método setdefault (). Ele tem os mesmos parâmetros do método get (), mas, ao contrário do último, ao acessar o dicionário usando uma chave inexistente, ele não apenas retornará o valor padrão, mas também criará um novo elemento no dicionário com essa chave e o valor passado. Se, ao acessar o dicionário usando o método setdefault (), a chave transmitida já estiver no dicionário, esse método deixará o dicionário inalterado.

author = {
   "first_name": "Robinson",
   "last_name": "Crusoe",
   "username": "Friday1719"
}

print (author.setdefault ('nome_de_usuário')) # exibirá Friday1719
print (author.setdefault ('middle_initial', “Monday”)) # # exibirá Monday e criará
# no dicionário um elemento com a chave 'middle_initial' e o valor dessa chave será 'Monday ”

no exemplo acima, vemos que o comportamento do método setdefault () não é diferente do comportamento do método get () ou da utilização de colchetes ao acessar o dicionário usando a chave existente na mesma. Se não houver essa chave no dicionário, o método setdefault () não apenas retornará o valor de seu segundo argumento para o programa (como o método get ()), mas também criará um elemento no dicionário com a chave e o valor passados ​​para ele. Esse é o comportamento do método setdefault () e o distingue do método get ().

Agora, se você seguir o exemplo acima e exibir os elementos do dicionário,então obtemos o seguinte resultado:

print(author)
{
  "first_name": "Robinson",
  "last_name": "Crusoe",
  "username": "Friday1719"
  "middle_initial": "Monday"
}

O uso dos métodos get () e setdefault () é uma técnica de primeira classe para acessar valores de dicionário. Leva apenas tempo para abandonar velhos hábitos e começar a usar essa técnica na prática.

Se você só precisa obter o valor do dicionário, seu auxiliar é o método get ().

Se você precisar adicionar com segurança um novo valor ao dicionário, chame o método setdefault ().
