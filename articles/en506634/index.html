Module Federation in Webpack 5, a plug-in for exchanging modules between Javascript applications, description and example

In the fifth version of the Webpack builder, a set of plugins for the exchange of modules between Javascript applications has appeared.

This article is a short description and an example based on two small applications built on the ReactJS framework.

The Module Federation plugin allows an application to export one or more modules to a separate JS file. A great way to build a microfrontend application. Third-party applications can import ready-made modules, for example, react components. Moreover, the import of dependencies Webpack takes over. The difference from NPM is that it imports into runtime.



The source application in the webpack config explicitly indicates:

  • modules for export
  • export file name e.g. export.js
  • dependencies that do not need to be exported e.g. export.js

In the HTML of the recipient application, the JS file of the source application is imported as a regular JS script: 

<script src="http://source-app.com/export.js">

And in the webpack config, which modules are taken from the file.

This is how it looks


The source application should register the export in the webpack.config.js settings.
In the plugin settings, specify the name of the exported container, which modules will be included in it and how the file will be called.

config example 
const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
 
module.exports = {
 ...
 plugins: [
   new ModuleFederationPlugin({
     name: 'home',
     library: { type: 'var', name: 'home' },
     filename: 'export.js',
     exposes: {
       ProductCarousel: './src/ProductCarousel'
     },
     shared: ['react', 'react-dom', '@material-ui/core', '@material-ui/icons']
   }),
 ]
};

Recipient Application

  • In javascript.html imports the source application js file

    example 
    <!DOCTYPE html>
<html lang="en">
 <head>
   <script src="http://source-app.com/export.js"></script>

  • In webpack.config.js, in the plugin settings, specify the name of the container you need, the plugin will find it in the JS file

    config example 
    const HtmlWebpackPlugin = require('html-webpack-plugin');
const ModuleFederationPlugin = require('webpack/lib/container/ModuleFederationPlugin');
 
module.exports = {
...
 plugins: [
   new ModuleFederationPlugin({
     remotes: {home: 'home' } // 
   }),
 ]
};

  • In JSX, import the module from the desired container, as if it were a regular local module

    code example 
    import React from 'react';
import ProductCarousel from 'home/ProductCarousel';
 
function App() {
 return (
     <ProductCarousel />
 );
}
 
export default App;
