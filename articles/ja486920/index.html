アプリケーションの段階的なガイドを作成する方法（プロジェクトがAngularにある場合）

みなさん、こんにちは。

それほど前のことではありませんが、次のスプリントが終了しました。ユーザーのために必要な時間ではありませんでしたが、同時に興味深い機能である、アプリケーションを操作するためのインタラクティブなガイドも作成しました。

インターネットには既製のソリューションがたくさんあります。それらすべてが確かにこのタスクに適している可能性がありますが、自分でそれを行う方法を見ていきます。

建築


このコンポーネントのアーキテクチャは非常に単純です。

DOMツリーには、ユーザーにボタンなど、何かを伝えたい重要な要素があります。

この要素の周りに暗いレイヤーを描画して、注意を向ける必要があります。
この要素の隣に、重要なメッセージを記載したカードを引く必要があります。



この問題を解決するには、@ angular / cdkが役立ちます。前回の記事では、すでにCDKに依存する@ angular /マテリアルを賞賛しましたが、フレームワークのすべての機能を使用して作成された高品質のコンポーネントの例として残っています。@ angular /マテリアル

ライブラリのメニュー、ダイアログボックス、ツールバーなどのコンポーネントCDK-オーバーレイのコンポーネントを使用して作成。

このコンポーネントのシンプルなインターフェイスを使用すると、アプリケーションの上にレイヤーをすばやく作成できます。これは、画面サイズの変更とスクロールに個別に適応します。すでに理解したように、このコンポーネントを使用して問題を解決することは非常に簡単になります。

まず、ライブラリをインストールします

npm i @angular/cdk @angular/material -S

インストール後、style.cssにスタイルを追加することを忘れないでください

@import '~@angular/cdk/overlay-prebuilt.css';
@import '~@angular/material/prebuilt-themes/deeppurple-amber.css';

ここで、プロジェクトに新しいモジュールを作成します。

ng generate library intro-lib

そしてすぐにディレクティブのテンプレートを生成します：

ng generate directive intro-trigger

このディレクティブは、今後のガイドのトリガーとして機能し、要素のクリックをリッスンして、ページ上で強調表示します。

@Directive({
  selector: '[libIntroTrigger]'
})
export class IntroTriggerDirective {
  @Input() libIntroTrigger: string;

  constructor(private introLibService: IntroLibService, private elementRef: ElementRef) {}

  @HostListener('click') showGuideMessage(): void {
    this.introLibService.show$.emit([this.libIntroTrigger, this.elementRef]);
  }
}

ここでは、ライブラリの作成時に生成されたサービスに目を向けます。主な作業はすべて彼が行います。

まず、サービスで新しいプロパティを宣言します

show$ = new EventEmitter<[string, ElementRef]>();

これらの値を見たように、ディレクティブは私たちに送信します。配列では、最初の要素は説明で、2番目は説明するDOM要素です（このソリューションは例を簡単にするために選択しました）。

@Injectable({
  providedIn: 'root'
})
export class IntroLibService {
  private overlayRef: OverlayRef;
  show$ = new EventEmitter<[string, ElementRef]>();

  constructor(private readonly overlay: Overlay, private readonly ngZone: NgZone, private readonly injector: Injector) {
    this.show$.subscribe(([description, elementRef]: [string, ElementRef]) => {
      this.attach(elementRef, description);
    });
  }
}

EventEmitterからの更新のサブスクリプションを追加してサービスデザイナーを更新します。アタッチ機能が更新を受信し、レイヤーを作成します。

レイヤーを作成するには、Overlay、Injector、NgZoneが必要です。

次のアクションは、いくつかの段階に分けることができます。

  • 現在のオーバーレイを閉じる（存在する場合）
  • PositionStrategyを作成する
  • OverlayRefを作成する
  • PortalInjectorを作成する
  • コンポーネントをレイヤーにアタッチする

最初のポイントは明らかです。これについては、サービスでプロパティを既に宣言しています。PositionStrategy-DOMツリーでのレイヤーの配置方法を管理します。

いくつかの既製の戦略があります：

  1. FlexibleConnectedPositionStrategy
  2. GlobalPositionStrategy

簡単に言うと、

FlexibleConnectedPositionStrategy-特定の要素に従い、構成に応じて、ブラウザーまたはスクロールのサイズを変更するときに、それに固執します。これは、ドロップダウンリスト、メニューの明確な使用例です。

GlobalPositionStrategy-名前が示すように、グローバルに作成されます。作業に要素は必要ありません。使用の明らかな例はモーダルウィンドウです。

調査中の要素の周囲にフローティングウィンドウ戦略を作成するメソッドを追加します。

メソッドを追加してOverlayRefを作成する

createOverlay(elementRef: ElementRef): OverlayRef {
    const config = new OverlayConfig({
      positionStrategy: this.getPositionStrategy(elementRef),
      scrollStrategy: this.overlay.scrollStrategies.reposition()
    });

    return this.overlay.create(config);
  }

そして、コンポーネントをレイヤーにバインドするメソッドを追加します。

  attach(elementRef: ElementRef, description: string): void {
    if (this.overlayRef && this.overlayRef.hasAttached()) {
      this.overlayRef.dispose();
    }

    this.overlayRef = this.createOverlay(elementRef);

    const dataRef = this.ngZone.run(
      () => new DataRef(this.overlay, this.injector, this.overlayRef, elementRef, description)
    ); //   ,      ,    ,  CD    -  

    const injector = new PortalInjector(this.injector, new WeakMap([[DATA_TOKEN, dataRef]]));

    dataRef.overlayRef.attach(new ComponentPortal(IntroLibComponent, null, injector));
  }

これはメッセージを表示するコンポーネントがどのように見えるかです

DataRefを除いて、リスティングで生じるすべてのことについてはすでに説明しました。

DataRefは、コンポーネントのインジェクターに追加する単純なクラスであり、実際には、レンダリングなどのデータを転送するためのものです。たとえば、説明です。

また、その中で、要素を暗くして強調するために別のレイヤーを描くことにしました。この場合、すでにグローバルレイヤー作成戦略を使用します。

export class DataRef {
  shadowOverlayRef: OverlayRef;

  constructor(
    private overlay: Overlay,
    private injector: Injector,
    public overlayRef: OverlayRef,
    public elementRef: ElementRef,
    public description: string
  ) {
    const config = new OverlayConfig({
      positionStrategy: this.overlay.position().global(),
      scrollStrategy: this.overlay.scrollStrategies.block()
    });

    this.shadowOverlayRef = this.overlay.create(config);

    this.shadowOverlayRef.attach(
      new ComponentPortal(
        ShadowOverlayComponent,
        null,
        new PortalInjector(this.injector, new WeakMap([[DATA_TOKEN, this.elementRef]]))
      )
    );
  }
}

ShadowOverlayComponent-コンポーネントを描画し、インジェクターで同じトークンを受け取ります。強調する必要のある要素のみが含まれます。

これをどのように実装したかは、githubのソースで確認できます。これに個別に焦点を当てたくありません。

ここでは、キャンバスを全画面で描画し、要素の周りに図形を描画し、fill（ 'evenodd'）メソッドでコンテキストを塗りつぶします。

合計


最もすばらしいのは、@ angular / cdk / overlayを使用して、好きなだけレイヤーを描画できることです。それらは適応的で柔軟です。画面サイズの変更について心配する必要はありません。そうでなければ、要素が自然な理由でシフトし、オーバーレイがそれを考慮します。

レイヤーの操作方法を理解し、段階的なガイドを作成する作業はそれほど難しくないことに気付きました。

要素を切り替えたり、ビューモードを終了したり、その他の多くのコーナーケースを実行したりする機能を追加して、ライブラリを変更できます。
