SILで関数を作成する

みなさん、こんにちは！

この記事では、SILで関数を作成する方法を説明します。

前書き


SILはSimple Issue Languageの略で、cPrime開発者がAtlassian JiraおよびConfluenceでの手動アクションを自動化するために作成しました。

SILの主な利点は、SILに、Atlassian JiraまたはAtlassian Confluence APIを使用するための知識を必要としない関数が含まれていることです。これにより、SILを入力するためのしきい値が大幅に削減され、SILプログラムコードの容量は、類似のJavaまたはGroovyコードよりも小さくなります。

たとえば、リクエストに応じてJiraからチケットを選択し、チケットに対していくつかのアクションを実行できるようにします。これを行うには、JavaまたはGroovyで、数十行のコードを記述する必要があります。SILでは、これは1行です。

selectIssues(" JQL ");

さらに、JavaまたはGroovyでは、Jira 7とJira 8のコードの互換性を確保する必要があります。SILでは、互換性について考える必要はありません。SelectIssuesは、SILがサポートするJiraのすべてのバージョンで機能します。

SILの詳細については、こちらをご覧ください。
しかし、このようなことをする必要がある場合、SILに関数がないのはなぜですか？たとえば、Table Grid Next Generationプラグインからデータを取得するか、JiraプラグインのPDFエクスポーターを使用してPDFファイルを生成する必要があります。

この場合、2つのオプションがあります。

  1. SIL Groovyコネクタ -このプラグインを使用すると、JavaまたはGroovyでJira Java APIを使用してコードを記述し、このコードをSILスクリプトから呼び出すことができます。
  2. 関数をSILに追加するJiraプラグインを記述し、SILスクリプトで関数を使用します

この記事では、Jiraプラグインを作成してSILの機能を拡張する方法に焦点を当てます。この記事の手順を繰り返すには、Atlassian SDKとgitが必要です。

Mavenアーキタイプを設定する


SILを拡張するプラグインには、SILに関数を追加するロジックが含まれている必要があります。このロジックを毎回プラグインに追加するのではなく、関数の作成のみに集中するために、新しいプラグインで必要なすべてを作成するMavenアーキタイプを作成しました。

まず、リポジトリからアーキタイプを複製します。

git clone https://alex1mmm@bitbucket.org/alex1mmm/sil-extension-archetype.git --branch v1 --single-branch

sil-extenstion-archetypeフォルダーが作成されます。それを見てみましょう：

cd sil-extension-archetype

次に、ローカルのmavenリポジトリーにアーキタイプをインストールします。

atlas-mvn install

プラグインを作成する


上記のフォルダに移動して、新しいプラグインを作成します

cd ..
atlas-mvn archetype:generate -DarchetypeCatalog=local

Mavenを使用してプロジェクトを作成すると、標準的な質問が表示されます。質問に答えます。それらを変更できます。

次に、独自の関数を作成するためにプラグインで何を変更する必要があるかについて話しましょう。

関数のJavaクラスを作成する


SILに追加する各関数には、独自のクラスが必要です。このクラスは、AbstractSILRoutineクラスの子孫である必要があります。

新しいクラスに何を追加する必要があるかを理解するために、利用可能な例を見てみましょう。

SayHello.java


public class SayHello extends AbstractSILRoutine<MutableString> { 
  private static final SILType[][] types = {{ TypeInst.STRING }};

  public SayHello(ClassLoader classLoader, String name) { 
    super(classLoader, name, types);
  }

  @Override 
  public SILType<MutableString> getReturnType() { return TypeInst.STRING; }

  @Override 
  protected SILValue<MutableString> runRoutine(SILContext silContext,                                   List<SILValue<?>> list) { 
    SILValue param = list.get(0); 
    return SILValueFactory.string( "Hello " + param.toStringValue());
 }
  @Override 
  public String getParams() { return "(name)"; }

最初の行：

private static final SILType[][] types = {{ TypeInst.STRING }};

この行は、関数に渡されるパラメーターのタイプと数を定義します。SILの関数は次のようになります。

mymethod(myparameter);

したがって、{{TypeInst.STRING}}は、1つのパラメータがあり、そのタイプが文字列であることを意味します。String型の2つのパラメーターを渡す場合は、次のように文字列を変更する必要があります。

private static final SILType[][] types = {{ TypeInst.STRING, TypeInst.STRING }};

SILでは、次のように関数を呼び出すことができます。

mymethod(myparameter1, myparameter2);

関数が1つまたは2つのパラメーターを受け入れることができる場合。その用語は次のようになります。

private static final SILType[][] types = {{ TypeInst.STRING},  
                                                          {TypeInst.STRING, TypeInst.STRING }};

次が来る：

public class SayHello extends AbstractSILRoutine<MutableString>

AbstractSILRoutineからクラスを継承します。つまり、クラスはString型のパラメーターを返します。

さらに：

public SayHello(ClassLoader classLoader, String name) { 
  super(classLoader, name, types);
}

これはコンストラクタです。おそらく何も変更する必要はありません。

さらに：

@Override 
public SILType<MutableString> getReturnType() { return TypeInst.STRING; }

ここでは、返されたパラメーターのタイプを判別します。この例では、文字列です。

さらに：

@Override 
protected SILValue<MutableString> runRoutine(SILContext silContext, List<SILValue<?>> list) { 
SILValue param = list.get(0); 
return SILValueFactory.string( "Hello " + param.toStringValue());
}

これは、関数のメインエントリポイントです。SILで関数を実行すると、ここに到達します。すべてのロジックはここにあります。

この関数は2つのパラメーターを取ります

。silContext-Jira変数とSIL変数を取得できます。

たとえば、SILスクリプトはポスト関数として機能し、スクリプトが実行されるチケットを受け取りたいとします。

Issue issue = (Issue) silContext.getAllMetaInformation().get("issue");

list-SIL関数に渡されるパラメーターの値を取得できます。

SILValue param = list.get(0);

ライン：

SILValueFactory.string( "Hello " + param.toStringValue());

値を返します。SILが理解できる値を返す必要があるため、SILValueFactoryを使用して値を生成します。

さらに：

@Override 
public String getParams() { return "(name)"; }

これは、SILマネージャの関数のタイプを定義します。SILマネージャーで入力を開始すると、使用可能な関数のヒントが表示されます。私たちの場合、次のようになります。

yourmethod(name)

クラスをSILエンジンに追加する


クラスを作成しました。次に、SILに関数を認識させる必要があります。これはESLauncher.javaファイルで行われます。このファイルには次の行が含まれています。

RoutineRegistry.register(new SayHello( classLoader,"SayHello"));

この行は、SayHelloクラスをSILにSayHelloという名前で追加していることを示しています。クラス名と機能が一致しない場合があります。

次に、次の行を追加します。

RoutineRegistry.unregister("SayHello");

SayHelloを関数の名前に変更します。Jiraからプラグインを削除すると、この行はSILから関数をアンロードします。

SILの詳細


SayHello2クラスでは、リストを返します。

SILValue param1 = list.get(0); 
SILValue param2 = list.get(1); 
List<String> res = new ArrayList<>(); 
res.add("Hello " + param1.toStringValue()); 
res.add("Hello " + param2.toStringValue()); 
return SILValueFactory.stringArray(res);

これが機能するためには、戻り値をTypeInst.STRING_ARRに変更し、KeyableArraySILObjectを指定してAbstractSILRoutineから継承する必要もあります。

SILでは、次のようにSayHello2を呼び出します。

string[] res = SayHello2("Alexey", "Matveev");
runnerLog(res[0]);
runnerLog(res[1]);

SayHello3では、Mapを返します。

SILValue param1 = list.get(0); 
SILValue param2 = list.get(1); 
Map<String, String> res = new HashMap<>(); 
res.put("param1","Hello " + param1.toStringValue()); 
res.put("param2","Hello " + param2.toStringValue()); 
return SILValueFactory.stringArray(res);

SILでは、次のようにSayHello3を呼び出すことができます。

res = SayHello3("Alexey", "Matveev");
runnerLog(res["param1"]);
runnerLog(res["param2"]);

この場合、配列の要素をインデックスではなくキーで取得します。

Javaのコードを見ると、どちらの場合もKeyableArraySILObjectを使用しています。実際、SILリストとマップでは、KeyableArraySILObjectクラスによって実装されます。それら。SILのどの配列にも、インデックスまたはキーのいずれかでアクセスできます。

配列をパラメーターとして渡します


SIL関数から配列を返すことができます。次に、配列をパラメーターとして渡す方法を見てみましょう。

コードは次のようになります。

@Override
protected SILValue<MutableString> runRoutine(SILContext silContext, List<SILValue<?>> list) {
    GenericArraySILObject rowsToUpdate =  (GenericArraySILObject) list.get(0).getObject();
    Map<String, int[]> keys = rowsToUpdate.getKeysMapping()
......
}

最初に、パラメーターからGenericArraySILObjectクラスのオブジェクトを取得し、次にそれからマップを取得します。

SILでは、次のように配列を渡すことができます。

string[] arr;
arr["key1"] = "value1";
arr["key2"] = "value2";
yourmethod(arr);

これでこの記事は終わりです。SILで関数を作成するための十分な知識を身に付けていることを願っています。
