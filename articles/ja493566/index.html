織りテスト-ABテストより100倍高速

A / Bテストは製品管理の主要なツールの1つです。これまでのところ、製品のビジネスメトリックに対する1つの特定の変更の影響を確実に評価し、他のすべての要因からそれを分離する、より信頼性が高く安価な方法は考えられていません。

この記事では、製品の変更をテストする別の方法について説明します。英語の文献では、インターレーステスト-インターリーブテストです。その長所と短所を明らかにするために、従来のA / Bテストと個人的に比較しますが、これは、より高速で正確である新しい、より高度な方法であり、A / Bテストを置き換える必要があるためではありません。これは、アプリケーションの分野が異なり、異なる質問に答える製品マネージャーのための追加ツールです。比較により、織りテストの違いと長所を簡単に示すことができます。

織りテストがA / Bテストよりもはるかに速い理由


織り方の基本的な考え方を同僚や他の製品マネージャーに伝える多くの試みの中で、次の例がそれを最もよく示しているという結論に達しました。少し時間をとってコンテキストを入力してください。例の終わりまでに、これが非常に明確であることに同意することを約束します。

コカコーラまたはペプシなど、できるだけ多くの飲み物を販売するために、バーで提供する必要のあるソーダを決定する必要があるとします。 A / Bテストの観点からこのソリューションにアプローチする場合、一方がコーラのみ、もう一方がペプシのみとなる2つの完全に同一のバーを開き、訪問者をこれらのバーの1つにランダムに誘導する必要があります。



次に、利用可能な飲み物を注文したバーの訪問者を比較し、どの飲み物が最も収益を上げているかを判断します。

私はあなたがすでに問題が何であるかを理解していると思います：彼らがまだ飲みたいので、彼らの好きな飲​​み物が無いそのバーへの多くの訪問者は彼らが与えるものを注文します。そして、ごくわずかしか飲まないか、飲まないほどの好みに基づいています。あまり重要ではない訪問者は、飲み物の好みに対する私たちのテストの感度を低下させます。なぜなら、彼らは実際に私たちに彼らの行動に関するいかなる信号も与えないからです。

織ることで同じ問題を解決する方法は？比較対象の両方のオプションを同時にユーザーに提供し、どちらが好ましいかを確認できるという物理的な能力があれば、ユーザーの真の好みをすばやく特定できます。



バーでメタファーにインターウィービングを適用する場合、カウンターに2つのタップを置き、訪問者が最も注文する飲み物を確認します。 A / Bテストでは注文数の違いのみがシグナルであるのに対し、各注文はどちらか一方のオプションを支持する「投票」になるため、このテストはかなり迅速に有意な結果をもたらすと直感的に感じます。

でNetflixのテックブログの記事A / Bよりも100倍速く織ることがユーザーの好みを決定するという証拠を提供します。残念ながら、私はウィービングに関する個人的な経験を公開することはできませんが、私の場合、この評価は確認されました。ほぼ妥当なトラフィックでウィービングすると、24時間未満で重要な結果が得られます。ただし、サンプルの代表性を確保する必要があるため、テスト期間を1日未満にしても機能しません（朝、午後、夕方の訪問者は異なる行動パターンをとることがあるので、毎週のサイクルは無視してください）。

A / Bテストと織りテストの違いは何ですか


A / Bテストを実施すると、ユーザーエクスペリエンスの変化が関心のあるすべての指標に及ぼす影響を測定できます。これは、単一のユーザーのコンテキストで検討します。変換から販売、サポートコールの数まで。

インターレーステストでは、織り交ぜられたオプションの1つをクリックすると直接関連付けることができるイベントのみを比較できます。ただし、この比較では、「製品でAをBに置き換えるとどうなるか」という質問に答えることはできません。ユーザーがBのランキングしか表示しないとどうなるかわからないためです。ユーザーの独立したバージョンではない組み合わせで測定しました。経験。

したがって、ウィービングは、多くの仮説の中で最も有望な仮説を選択するための予備段階として使用することをお勧めします。そのため、この変更がターゲットメトリックにどのように影響するかを確認するために、より長いA / Bテストを実施することはすでに理にかなっています。

多くの場合、アルゴリズムの改善がビジネス指標に影響を与えなかったことが判明する可能性がありますが、少なくともユーザーエクスペリエンスが向上していることは確かであり、これで、対象とする指標を改善するために最適化する意味がない可能性が最も高いブロックがわかりました。
