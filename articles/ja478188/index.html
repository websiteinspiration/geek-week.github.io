PowerShellを使用してインシデント情報を収集する

PowerShellは、マルウェア開発者と情報セキュリティの専門家の両方がよく使用するかなり一般的な自動化ツールです。
この記事では、情報セキュリティインシデントに対応するときに、PowerShellを使用してリモートでエンドデバイスからデータを収集するオプションについて説明します。これを行うには、ターゲットデバイスで実行するスクリプトを記述する必要があります。その後、このスクリプトの詳細な説明があります。

function CSIRT{
param($path)
if ($psversiontable.psversion.major -ge 5)
	{
	$date = Get-Date -Format dd.MM.yyyy_hh_mm
	$Computer = $env:COMPUTERNAME
	New-Item -Path $path\$computer\$date -ItemType 'Directory' -Force | Out-Null
	$path = "$path\$computer\$date"

	$process = get-ciminstance -classname win32_process | Select-Object creationdate, processname,
	processid, commandline, parentprocessid

	$netTCP = Get-NetTCPConnection | select-object creationtime, localaddress,
	localport, remoteaddress, remoteport, owningprocess, state
	
	$netUDP = Get-NetUDPEndpoint | select-object creationtime, localaddress,
	localport, remoteaddress, remoteport, owningprocess, state

	$task = get-ScheduledTask | Select-Object author, actions, triggers, state, description, taskname|
	where author -notlike '**' | where author -ne $null |
	where author -notlike '*@%systemroot%\*' | where author -notlike '*microsoft*'

	$job = Get-ScheduledJob

	$ADS =  get-item * -stream * | where stream -ne ':$Data'

	$user = quser

	$runUser = Get-ItemProperty "HKCU:\Software\Microsoft\Windows\CurrentVersion\Run\"

	$runMachine =  Get-ItemProperty "HKLM:\Software\Microsoft\Windows\CurrentVersion\Run\"

	$array = $process, $netTCP, $netUDP, $task, $user, $runUser, $runMachine, $job, $ADS
	$arrayName = "Processes", "TCPConnect", "UDPConnect", "TaskScheduled", "Users", "RunUser", "RunMachine",
	"ScheduledJob", "AlternativeDataStream"


	for ($w = 0; $w -lt $array.count; $w++){
		$name = $arrayName[$w]
		$array[$w] >> $path\$name.txt
		}

	}

}

次のように、現在のユーザーに代わって実行中のプロセスのリストを取得します。get-ciminstanceコマンドレットにwin32_processクラスを割り当てて$プロセス変数を作成します。Select-Objectコマンドレットを使用して、追加の出力パラメーターを追加できます。この例では、parentprocessid（親プロセスのPPIDの識別子）、creationdate（プロセスが作成された日付）、processed（プロセス識別子PID）、processname（プロセス名）、commandline（開始コマンド）になります。

$process = get-ciminstance -classname win32_process | Select-Object creationdate, processname, processid, commandline, parentprocessid

すべてのTCPおよびUDP接続のリストを取得するには、変数$ netTCPおよび$ netUDPを、それぞれGet-NetTCPConnectionおよびGet-NetTCPConnectionコマンドレットに割り当てて作成します。

$netTCP = Get-NetTCPConnection | select-object creationtime, localaddress, localport, remoteaddress, remoteport, owningprocess, state

$netUDP = Get-NetUDPEndpoint | select-object creationtime, localaddress, localport, remoteaddress, remoteport, owningprocess, state

スケジュールされたタスクとタスクのリストを知っておくことが重要になります。これを行うには、get-ScheduledTaskおよびGet-ScheduledJobコマンドレットを使用します。変数$ taskと$ jobを割り当てます。システムには多数のスケジュールされたタスクがあるため、悪意のあるアクティビティを特定するには、正当なスケジュールされたタスクを除外する価値があります。これには、Select-Objectコマンドレットが役立ちます。

NTFSファイルシステムには、代替データストリーム（Alternate Data Streams、ADS）などがあります。つまり、NTFSのファイルは、オプションで、任意のサイズの複数のデータストリームに関連付けることができます。ADSを使用すると、標準のシステムチェックでは表示されないデータを非表示にすることができます。これにより、悪意のあるコードを挿入したり、データを非表示にしたりできます。

PowerShellで代替データストリームを表示するには、get-itemコマンドレットと*記号付きの組み込みWindowsストリームツールを使用して、考えられるすべてのフローを表示します。これには、$ ADS変数を作成します。

$ADS = get-item * -stream * | where stream –ne ':$Data'

ログインしているユーザーのリストを見つけることも役立ちます。これを行うには、$ユーザー変数を作成し、それにquserプログラムを割り当てます。

$user = quser

攻撃者がシステムの足掛かりを得るために、自動実行に変更を加えることができます。Get-ItemPropertyコマンドレットを使用して、自動実行でオブジェクトを表示できます。
2つの変数を作成してみましょう。$ runUser-ユーザーに代わってスタートアップを表示するためと$ runMachine-コンピューターに代わってスタートアップを表示するため。

$runUser = Get-ItemProperty 
"HKCU:\Software\Microsoft\Windows\CurrentVersion\Run\"
$runMachine = Get-ItemProperty 
"HKLM:\Software\Microsoft\Windows\CurrentVersion\Run\"
