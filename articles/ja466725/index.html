依存性注入とは何ですか？


依存性注入（依存性の注入または注入）は、あるクラスの別のクラスへの依存性です。それら。1つのクラスを完全に操作するには、他の（それらの）クラスの初期化が必要です。

たとえば、CarクラスはEngineクラスなしでは機能できず、EngineクラスはFuelクラスなしでは機能しません。次のようになります。

class Car(private var engine: Engine)

class Engine ( private var fuel: Fuel)

class Fuel(){
private val fuel = if(BuildConfig.DEBUG){
	“benzine”
} else {
	“diesel”
}
}

この例では、CarクラスはEngineクラスに依存しており、さらにEngineクラスはFuelクラスに依存しています。

ダガー2-はじめに


Daggerは、依存性注入の実装に役立つライブラリです。これはグーグルライブラリです。詳細なドキュメントはこちらから入手できます。

短剣の長所：

  • ボイラープレートコードを少なくする必要があります。
  • 依存関係の構造化に役立ちます。
  • 多くの依存関係がある場合、作業を大幅に簡素化します
  • コードが読みやすくなります。

短剣の短所：

  • 詳細なドキュメントの欠如
  • Daggerは、注釈を使用して開発者の意図を理解しようとします。彼があなたを正しく理解していないとき、それは複雑になります
  • daggerは、見つけるのと同じくらい難しいコードを生成します

Dagger 2の最初の使用


まず、アプリケーションに短剣を追加する必要があります。私はこれを行う方法の2つの方法を知ってい
ます。1. build.gradle（アプリ）を開き、トレースを追加します。

1.1プラグイン宣言セクションの最上部

apply plugin: 'kotlin-kapt'

1.2依存関係セクション

{
...
    kapt "com.google.dagger:dagger-compiler:$dagger_version"
    implementation "com.google.dagger:dagger:$dagger_version"
}

セクションでdaggerのバージョン（dagger_version）を指定します

ext {
    dagger_version = '2.17'
}

まだの場合は、androidセクションの上にセクションを追加します。

2.プロジェクト構造を介してMavenリポジトリーを追加-依存関係-ライブラリの依存関係を追加

プロジェクトを同期したら、短剣を使用して依存関係を実装する準備が整います。

まず、Car、Engine、およびFuelクラスを作成します。


class Car constructor(private var engine: Engine)

class Engine constructor(private var fuel: Fuel)

class Fuel {
    private val fuelType = if(BuildConfig.DEBUG){
        "benzine"
    }else{
        "diesel"
    }
}

Car、Engine、およびFuelクラスのコンストラクターの前に、ダガーのInjectアノテーションを追加します。これにより、必要に応じてこれらのクラスを実装する必要があることをダガーに認識させます。痕跡がつきます。

class Car @Inject constructor(private var engine: Engine)
class Engine @Inject constructor(private var fuel: Fuel)
class Fuel @Inject constructor() {
    private val fuelType = if(BuildConfig.DEBUG){
        "benzine"
    }else{
        "diesel"
    }
}

Daggerは、実装する必要があるすべてのオブジェクトを作成する方法を知っている必要があります。実装（Inject）するすべてのクラスをリストするために、アノテーションが使用されます成分これは、インターフェース（DaggerComponent）に対して宣言されています。

次のようになります。


@Component
interface DaggerComponent {
    fun getCar(): Car
    fun getEngine(): Engine
    fun getFuel(): Fuel
}

コンポーネントメソッドを宣言する場合、重要なのはメソッド名ではなく、それらが返すクラスです。

このステップでは、プロジェクトをビルドする必要があります（ビルド-プロジェクトの再ビルド）。その後、短剣はコンポーネントを初期化するために必要なクラスとファクトリーを生成します。ファクトリーの名前は、ダガーのクラスを初期化するインターフェースの名前と一致しますが、「Dagger」というプレフィックスが追加されます。出力では、DaggerDaggerComponentクラスを取得します。

// Generated by Dagger (https://google.github.io/dagger).
package com.example.dagger2;

public final class DaggerDaggerComponent implements DaggerComponent {
  private DaggerDaggerComponent(Builder builder) {}

  public static Builder builder() {
    return new Builder();
  }

  public static DaggerComponent create() {
    return new Builder().build();
  }

  @Override
  public Car getCar() {
    return new Car(getEngine());
  }

  @Override
  public Engine getEngine() {
    return new Engine(new Fuel());
  }
  @Override
  public Fuel getFuel() {
    return new Fuel();
  }

  public static final class Builder {
    private Builder() {}

    public DaggerComponent build() {
      return new DaggerDaggerComponent(this);
    }
  }
}

すべて準備ができています。MainActivityでCarタイプのcarフィールドを作成してみましょう。

private var car: Car = DaggerDaggerComponent.create().getCar()

それにアクセスする際にアプリケーションを起動することで、あなたは車のフィールドが初期化されていることを確認することができます
