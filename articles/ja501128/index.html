VR開発があなたに合わない理由

みなさん、こんにちは！私の名前はグリゴリーディアディチェンコで、AR / VRに携わっています。（部屋にはポップスがあります）。私はAR / VRに4年以上携わっていますが、それらだけでなく、さまざまなカスタムプロジェクトをずっとやっています。私はスタジオで働いていました。長い間、私はフリーランサーでしたが、1年間でも自分のスタジオを持っていました。今日は、開発者として「VRに行くべきでない理由」とニュアンスをさまざまな観点から説明したいと思います。バーチャルリアリティで遭遇するものに興味がある場合-カットへようこそ！



愛好家のためのVR


一般的に、あなたがテクノロジー愛好家なら、VRが好きですか、あるいはいつか私たち全員が家庭でヘルメットを着用し、より美しく、より強く、よりスマートになることを夢見ていますか？この記事はあなたのためではありません。私は純粋に日常的な問題、ニュアンス、限界について話しますので、マニアにとっては役割を果たしません。それでも、VR開発は基本的に、同じWeb、バックエンド、デスクトップ、またはモバイル開発とは異なります。

非常に高価な機器




少し背景があります。第一に、スタジオでは機器を提供するため、これは開発者にとってそれほどの役割を果たしません。企業で働いていたときは、それについて考える必要がありませんでした。次に、デスクトップVRとモバイルVRを少し分離する必要があります。モバイルVRでは-高価ですが、デスクトップVRでは-非現実的に高価です。

私がフリーランスを始めたとき、私は最新のサムスンのフラッグシップを備えた40kルーブルとGearVRに適したMSIノートパソコンを持っていました。つまり、合計セットアップ費用は約13万ルーブルです。一般に、私の意見では、Unityフリーランサーの最小（最も快適ではない）キットは、40〜5万ルーブルのラップトップです。すでに彼と一緒にプロジェクトを開発することができます。モバイルVRの場合、現時点で取得した場合、別のOculus Questを購入する必要があります。これは約60kルーブルです（つまり、ラップトップよりも高価です）。すでに100kルーブルですが、それほど安価ではありません。特にモスクワはそうではありません。

しかし、最悪のシナリオを考えてみてください。デスクトップVR。私たちは何が必要なのか？コンプ、モニター（少なくとも1つ）、キーボード、マウス、HTC Vive（少なくとも最初）。そして、この最小限のセットは、200-250kルーブルのどこかでかなりの費用がかかります。この場合、開発用の最小グラフィックスカードは1060です。これは直接の有形の金額です。さて、少なくともUnityにさまざまなコントローラーの抽象化があることは良いことです。古代には、それぞれにボタンマッピングを作成する必要がありました。これは、ワイプと言うと非常に不便ですが、オクルスはありません。テストしないでください。

モバイルARの場合、最低限必要なのは良い電話です。これまでのところ、私の経験ではお金に最適な値はRedmi Note 7です。これは10〜11,000ルーブルで見つけることができるので、それほど悪くはありません。彼には問題があり、これは確かにiPhoneではありません。しかし、それはテストに完全に適合します（現在、これはAndroidのメインテストデバイスです）。これはARの研究またはAndroidのみの開発にのみ適しているという警告が本当に1つあります。これはまれです（b2bタスクで発生します）。さらに、快適にするにはMacbookが必要ですが、最初はUnity Cloud Buildを使用して歯を磨くことができます。 +開発者アカウント-年間150ドル（現在のレートで約10〜11k）。これにより、合計で90〜100kになります。なぜiPhoneが必要なのですか？ ARFoundationでも、同じAPIはAndroidとiOSでは動作が異なります。特にマーカーの場合

私は年間を通じて膨大な量の機器を蓄積しています。いくつかのヘルメット、lipmo、電話の束、その他のゴミ。

以上が要約です。
商用Unity開発の最小セットは4万〜5万ルーブルです（SSDやその他の生活設備を加速して購入できるため、快適な開発が可能です）。3Dで作業する場合は、最初にMacBookを購入しないことをお勧めします。タスクがあるので-光を焼く。また、50k対応のMacbookとは異なり、Windowsには通常のビデオカードを搭載したラップトップがあり、これは問題になりません。

そして、はい、それは機器なしでは不可能です。仕事の山がたくさんあったとしても、タスクの一部しか「目で」行うことができません。ゲームに触れた場合は、ヘルメットでテストする必要があります。モバイルおよびWeb開発では、別の電話が必要になるため、真実が常にスムーズに行われるとは限りません。しかし、それはすべてタスクに依存し、少なくともこれは、すでにプロジェクトがあることを意味します。コメントの誰かがあなたの意見では、フロントエンド、バックエンドなどのようなもののための最小限のキットであるものを共有するかもしれませんが 個人的には、私の観点から、少なくともケトルからSSH経由で仮想マシンにアクセスするため、バックエンドの最も単純な最小セットの1つです。さらに、virtualochka、環境をローカルに引き上げ、一般的にMacbookを購入することは、仕事の快適さの問題です。

テスト中




おそらく私の主な痛み。まず、経験豊富な開発者として、私はシステムを作成して入力を抽象化し、作業の90％をエンジンですべて同じように行います。つまり、シューティングゲームですか？さて3Dコントローラー-シュート。テキスト入力？画面にも転送します。複雑なアルゴリズムまたは視覚化？さまざまな入力オプションを実行するテストスクリプト。

そして、スタジオや会社での作業-すべてをテストできる作業領域があります。一般的に、他の開発と比較して、デバイスをピックアップする必要があるときに、UnityやさまざまなIDEの外でしか実行できないタスクにはまだ不満があります。モバイルゲーム開発では、自分がやったことをプレイするのも良いことです。特にUXを理解していない場合は、多くの場合、それほど重要ではありません。たとえば、ターンベースのゲームやパズルです。そして主な問題は、あなたがUXを担当するかどうかです。そうでなければ、あなたはスクラムチームを持っていません-これは必要ではありませんが、便利です。ほとんどのバグ（すべてではない）はフレームワーク内で捕捉されます。

AR / VRのメカニズムは、機器でのみテストされます。 AR / VRの観点から最も便利なインターフェースの1つ-世界中のすべての要素を物理オブジェクトと呼びます。したがって、あなたはあなたの力、物理的な力学などがどのように機能するかを見守る必要があります。さらに、このトピックのガイドラインが完成するまで、フォントが読みやすいと私たちが言うことを監視する必要があります。解像度の問題-グラフィックが波打たないこと。さらに、ユーザーがコンテンツに夢中になることもありません。 1つのプロジェクトでゼロ重力のメカニズムのみ（ユーザーが壁から離れた）速度と減衰によって2週間調整したので、ゲームがおかしくなることはありません。

そして、これらすべてはヘルメットでのみテストするのが現実です。ARでは、テストマーカーや追跡などに関連する多くのニュアンスがまだあります。そのため、他の多くの種類の開発よりもテストが困難になります。もちろん、愛好家にとっては、これはおそらく問題にはなりませんが、これから次の問題が発生します。

VRで作業しているときに旅行を忘れる




今話をするのはちょっと皮肉ですが、それでもいつもそういうわけではなかったので、きっと世界は正常に戻ると思います。旅行中のVRプロジェクトの開発には、25リットルのバックパックが含まれます。これには、ゲーム用ラップトップ、Vive、三脚が含まれ、空港では不審に見えます。そして、あなたが必要とするホテルの部屋や予約は非常に具体的です。理想的には、三脚を置くために少なくとも3x3メートルの空きスペースが必要です。または、サバンナなどの美しい景色を眺めながら、VRで360パノラマを旅することもできます。

フリーランス旅行の場合は難しいですが、私の友人の一部はそれを行います。もちろん、これはインターネットからの写真のようなものではありません。あなたはマイタイとムラートのいるビーチではなく、部屋の中でエアコンの下で、インターネットがうまくいき、すべてがうまくいくことを祈っています。確かに、同じアジアではそれほど高価ではなく、24時間年中無休で仕事をすることができるので、好きなように夜や仕事が終わった時間を過ごすことができます。

また、VRプロジェクトは非常に重要です。私の3年間の平均プロジェクトは2〜3 GBです。ホテルのインターネット上、およびロシア連邦外のインターネット上-そのようなファイルを転送することは困難です。

モバイルバーチャルリアリティを使用すると、少し簡単になります。クエストはそれほど大きくなく、テストのために小さな場所を取ることができます。

VR開発は難しい




私の意見では、これは恐ろしいことではありませんが、それでもそれについて言う価値があります。優れた興味深いVRは非常に難しいものです。非現実的な自由度と、優れた没入感が得られるおかげで、そこには多くの数学と物理学が存在します。 3D開発からの多くの標準的なトリックは適用されません。 3Dでの手の任意の動きは難しいため、ユーザーがどのように世界と対話するかを深く考える必要があります。

テニスのような同じゲームの観点から考えてみましょう。デスクトップの下-これは基本的にボタンが押された状態です。 VRでは、衝突の瞬間にボールに与える加速度と速度の方向を計算する必要があります。これにより、ユーザーは、ボールが飛んでどこで快適にプレーできるかをユーザーが信じられるようにします（これらは常に単純な公式ではありません）。

したがって、従来の開発と比較して、最適化、数学、物理学、シェーダーなどの知識に対する要件が増えます。これは私が好きなものですが、問題になるかもしれません。

このすべてがあなたを怖がらせない場合




一般に、VR / AR開発には1つの大きなプラスがあります。それは知識の欠如のために興味深いものであり、実験の無限の余地があります。多くの場合、美しいものは偶然にかなり判明しました。あなたは「でも何だと思う？」とは思わないので。

しかし、これらのテクノロジーを使って3年間作業した結果、これはテクノロジー愛好家にとってより重要であるという結論に達しました。ご清聴ありがとうございました。この情報がお役に立てば幸いです。さらに、誰かがこれに追加するものを持っているかもしれません。
