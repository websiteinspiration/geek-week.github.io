Wi-Fiの作業を改善します。一般原則と役立つもの


ラジオ受信機で収集、購入、または少なくとも同調した人は、おそらく感度や選択性（選択性）などの言葉を聞いたでしょう。

感度-このパラメーターは、最も離れた場所でも受信機が信号をどれだけうまく受信できるかを示します。

そして、選択性は、他の周波数の影響を受けずにレシーバーが特定の周波数にどれだけうまくチューニングできるかを示しています。これらの「その他の周波数」、つまり、選択された無線局からの信号送信には関係なく、この場合は無線干渉の役割を果たします。

送信機の電力を増やすことで、低感度の受信機にあらゆるコストで信号を受信するように強制します。重要な役割を果たすのは、異なる無線局からの信号が相互に影響し合うことです。これにより、チューニングが複雑になり、無線通信の品質が低下します。

Wi-Fi環境では、データを送信するための媒体としてラジオが使用されます。したがって、過去のラジオエンジニアやアマチュアラジオ事業者、そして最後の1世紀前にさえも操作したことの多くは、今日でも関連しています。

しかし、何かが変わった。アナログフォーマットは、送信信号の性質の変化を伴うデジタル放送に置き換えました。

以下は、IEEE 802.11b / g / n規格のフレームワーク内のWi-Fiワイヤレスネットワークの動作に影響を与える一般的な要因の説明です。

Wi-Fiネットワークのニュアンス


放送では、大規模な集落からはほど遠く、ローカルFMラジオ局と受信機のVHF帯域でマヤクしか受信できない場合、相互影響の問題は発生しません。

もう1つは、2.4および5 GHzの2つの限られた帯域でのみ動作するWi-Fiデバイスです。以下は、克服しなければ、回避する必要があるいくつかの問題です。

最初の問題は、異なる規格が異なる範囲で機能することです。

2.4 GHz帯域では、802.11b / g / nをサポートするデバイスが機能します。5 GHz帯域-802.11aおよび802.11n。

ご覧のように、2.4 GHz帯域と5 GHz帯域の両方で動作できるのは802.11nデバイスのみです。それ以外の場合は、両方の帯域でのブロードキャストをサポートするか、一部のクライアントがネットワークに接続できないという事実に我慢する必要があります。

2番目の問題は、最も近いアクションの半径で動作するWi-Fiデバイスが同じ周波数範囲を使用できることです。

2.4 GHz周波数帯域で動作するデバイスの場合、802.11b / g / n規格の場合は20 MHz、802.11n規格の場合は5 MHz間隔の40 MHzの13のワイヤレスチャネルが利用可能であり、ロシアでの使用が承認されています。

したがって、ワイヤレスデバイス（クライアントまたはアクセスポイント）は隣接チャネルに干渉します。もう1つは、クライアントデバイス（スマートフォンなど）の送信電力が通常のアクセスポイントの送信電力よりも大幅に低いことです。したがって、この記事では、アクセスポイントが相互に及ぼす影響についてのみ説明します。

デフォルトで顧客に提供される最も人気のあるチャネルは6です。しかし、次の桁を選択することで、偽の影響を取り除くことができることを自慢しないでください。チャネル6で動作するアクセスポイントは、チャネル5と7に強い干渉を与え、チャネル4と8に弱い干渉を与えます。チャネル間のギャップが大きくなると、相互の影響は減少します。したがって、相互干渉を最小限に抑えるには、それらの搬送周波数が25 MHz間隔（5チャネル間隔）であることが強く望まれます。

全体的にモデレートのメリット


上記のように、電力を削減することは必ずしも悪いことではありません。さらに、電力が増加すると受信品質が大幅に低下する可能性があり、ここでのポイントはアクセスポイントの「弱点」ではありません。以下では、これがどのような場合に役立つかを検討します。

ラジオダウンロード


接続先のデバイスを選択すると、輻輳の影響を直接見ることができます。 Wi-Fiネットワークのリストに3つから4つ以上の項目が含まれている場合-ラジオのダウンロードについてはすでに説明しています。さらに、各ネットワークは、近隣の干渉源となります。また、干渉はノイズレベルを劇的に増加させるため、ネットワークパフォーマンスに影響を与えます。これにより、常にパケットを再送信する必要が生じます。この場合の主な推奨事項は、アクセスポイントの送信機の電力を下げることです。理想的には、すべてのネイバーが互いに干渉しないように同じことを行うように説得します。

先生が欠席したときの授業の様子は、学校の授業に似ています。各生徒は、デスクパートナーや他のクラスメートと話し始めます。一般的なノイズでは、彼らはお互いに聞き取りが悪く、より大きな声で話し始め、その後さらに大きくなり、最終的には叫び始めます。教師はすぐに教室に行き、懲戒処分を行い、通常の状態に戻ります。教師としての役割がネットワーク管理者であり、小学生-アクセスポイントの所有者としての役割である場合、ほぼ直接的な類似性が得られます。

非対称接続


前述のように、アクセスポイントの送信機の電力は通常、クライアントのモバイルデバイス（タブレット、スマートフォン、ラップトップなど）の2〜3倍です。そのため、クライアントがアクセスポイントから適切な安定した信号を受信し、クライアントからポイントへの送信が「あまり」うまく機能しない「グレーゾーン」が出現する可能性が非常に高くなります。このような接続は非対称と呼ばれます。

良好な品質で安定した接続を維持するには、双方向の受信と送信が非常に効率的に機能する場合に、クライアントデバイスとアクセスポイントの間に対称的な接続があることが非常に望ましいです。

クライアントステアリング


前述のように、説明されている問題は主に2.4 GHz帯域に影響します。
最新のデバイスの所有者は、5 GHzの周波数範囲を使用できます。

利点：

  • チャネル数が増えるため、相互に影響を与えるチャネルを最小限に抑えやすくなります。
  • Bluetoothなどの他のデバイスはこの範囲を使用しません。
  • 20/40/80 MHzの幅を持つチャネルのサポート。

短所：

  • この範囲の無線信号は、障害物をより良く通過します。したがって、1つの「スーパーブレークダウン」ではなく、異なる部屋で信号強度がより穏やかな2つまたは3つのアクセスポイントを用意することが望ましいです。一方、カバレッジの性質は、1つからの信号をキャッチするよりも均一になりますが、「超強力」なカバレッジになります。

ただし、実際には、いつものように、ニュアンスが生じます。たとえば、一部のデバイス、オペレーティングシステム、およびソフトウェアは、デフォルトで接続用に「古き良き」2.4 GHz帯域を提供しています。これは、互換性の問題を減らし、ネットワーク接続アルゴリズムを簡素化するために行われます。接続が自動的に行われる場合、またはユーザーがこの事実に気付く時間がない場合-5 GHz帯域を使用する可能性は傍観者のままです。

デフォルトで5 GHzで接続するクライアントデバイスを提供するクライアントステアリング機能は、この状況を変えるのに役立ちます。この範囲がクライアントでサポートされていない場合でも、2.4 GHzを使用できます。

自動修復


上記は、柔軟な電力制御を支持する多くの議論でした。ただし、妥当な質問が残っています。これはどのように行うことができますか？

このため、Zyxelワイヤレスコントローラーには、自動修復という特別な機能があります。
コントローラは、支援を得て、アクセスポイントのステータスとパフォーマンスをチェックします。それらの1つが機能しないことが判明した場合、隣人は形成された無音ゾーンを埋めるために信号電力を増加するように指示されます。欠落しているアクセスポイントが動作を再開した後、隣接するポイントは、互いの作業を妨げないように信号強度を下げるように指示されます。

この機能は、ワイヤレスネットワークコントローラーの特別なライン（NXC2500およびNXC5500）の一部でもあります。

安全なワイヤレスネットワークエッジ


パラレルネットワークからの隣接するアクセスポイントは、干渉を引き起こすだけでなく、ネットワークへの攻撃の踏み台としても使用できます。

次に、ワイヤレスコントローラはこれに対処する必要があります。NXC2500およびNXC5500コントローラーには、標準のWPA / WPA2エンタープライズ認証、さまざまな拡張認証プロトコル（EAP）の実装、組み込みのファイアウォールなど、多くのツールがあります。

したがって、コントローラは不正なアクセスポイントを検出するだけでなく、悪意のある意図を運ぶ可能性が最も高い企業ネットワーク上の疑わしいアクションもブロックします。

不正AP検出機能（不正AP封じ込め）


まず、ローグAPが何であるかを理解しましょう。

不正APは、ネットワーク管理者によって制御されない外部アクセスポイントです。ただし、企業のWi-Fiネットワークの範囲内にあります。たとえば、オフィスのネットワークソケットに許可なく接続されている従業員の個人用アクセスポイントにすることができます。このタイプのイニシアチブは、ネットワークセキュリティに悪影響を及ぼします。

実際、このようなデバイスは、主要な保護システムをバイパスして、企業ネットワークへのサードパーティ接続のチャネルを形成します。

たとえば、アクセスの外部アクセスポイント（RG）は正式には企業ネットワークに配置されていませんが、正当なアクセスポイントと同じSSID名でワイヤレスネットワークが作成されています。その結果、企業ネットワーククライアントが誤って接続して資格情報を転送しようとしたときに、RGポイントを使用してパスワードやその他の秘密情報を傍受できます。その結果、ユーザー資格情報は「フィッシング」ポイントの所有者に知られます。

ほとんどのZyxelアクセスポイントには、不正ポイントを検出するための組み込みの無線空中スキャン機能があります。

重要！外部ポイントの検出（AP検出）は、これらの「ウォッチドッグ」アクセスポイントの少なくとも1つがネットワーク監視モードで動作するように構成されている場合にのみ機能します。

Zyxelアクセスポイントが監視モードで動作しているときに外部ポイントを検出すると、ブロッキング手順を実行できます。

不正APが合法的なアクセスポイントをエミュレートするとします。上記のように、攻撃者は企業のSSID設定を偽のポイントで複製することができます。次に、Zyxelアクセスポイントは危険なアクティビティを妨害しようとし、ダミーパケットのブロードキャストブロードキャストを介して干渉を引き起こします。これにより、クライアントが不正APに接続して資格情報を傍受することが不可能になります。そして、「スパイ」アクセスポイントはその使命を果たすことができなくなります。

ご覧のように、アクセスポイントの相互影響は、相互に作用するときに厄介な干渉を引き起こすだけでなく、攻撃者からの保護にも使用できます。

結論


小さな記事の枠組みの資料では、すべてのニュアンスについて話すことはできません。しかし、簡単にレビューしても、ワイヤレスネットワークの開発と保守には非常に興味深いニュアンスがあることが明らかになります。一方では、アクセスポイントの電力を削減するなど、信号ソースの相互影響に対抗する必要があります。一方、安定した通信を行うためには、信号レベルを十分に高く保つ必要があります。
