80年代のPC用キラーアプリ：VisiCalcとWordStar

Microsoft OfficeとGoogle Docsの祖先がどのようにして80年代にPCを普及させ、スタッフを大幅に削減し、何年にもわたって一部のライターの心をつかんだ。


写真提供：John Markos O'Neill（はい、このPCは90年代です）/ CC BY-SA

PCの大量適応のプロセスはかなり長いものでした。コンピューターが大学から居間に移動するためには、コンピューターのサイズを縮小して価格を下げるだけでは不十分でした。コンピューターの機能は、聴衆のニーズを満たす必要がありました。たとえば、Microsoft BASICの普及により、プログラミングがよりアクセスしやすくなり、教育用PCアプリケーションの扉が開かれました。その後、Commodore VIC-20などのコンピュータが家庭用コンソールに挑戦しました。しかし、この市場はまだニッチのままでした。

テクノロジーが真にパーソナルであるためには、そのメリットが幅広い人々に明らかにされなければなりません。これを行う最良の方法は、新しい技術ツールを使用して既存のアナログプロセスを最適化することです。1970年代に、そのような最適化のチャンスは、ハーバードプログラムのダニエル（ダン）ブリックリン（ダンブリックリン）のMBAの学生を見ました。

VisiCalcの誕生


ダンのコンピュータとの知り合いは60年代に始まりました。彼はまだ男子生徒でしたが、メインフレームとパンチカードを扱う機会を得ました。放課後、MITに留学し、UNIXの前身であるMULTICSオペレーティングシステムの作成に参加しました。数年の仕事の後、ダンは再び教育プロセスに戻ることを決定し、ハーバードMBAプログラムを選択しました。

1977年にハーバードに入社したダンは、久しぶりに完全に「紙」の環境にありました。そして、ビジネスの世界はまだ高度な技術から遠かったので、不思議ではありません。大企業はR＆Dコンピューティングにメインフレームを使用し、航空会社はメインフレームを使用してチケットを予約しましたが、一般の従業員のデスクにはコンピューターがありませんでした。ダンのクラスメートの大多数が彼らとの経験がなかったことは論理的です。

ハーバードMBAプログラムの多くはケース管理でした。学生は、特定のビジネスの動きを整理し、収益性を計算するように提案されました。この作業は最初から最後まで紙で行われました。

それ以降のすべての計算が不正確になるように、計算で1つの誤りを犯すだけで十分でした。このため、授業の途中で生徒が教育プロセスから「脱落」することがよくあります。

この問題を解決するために、ダンは、変数の値を動的に変更できる財務計算用のプログラムを作成したいと考えていました。したがって、初期データに誤りがあったとしても、すべての計算を数分で再実行できます。彼はMITの友人、プログラマーであるボブフランクストンに連絡し、チームは新製品の開発を開始しました。

このタスクに適したインターフェースを見つけるには、かなりの時間がかかりました。インスピレーションの主な情報源は、ダンが子供の頃から家で見ていた巨大な会計ノートブックでした。彼の父親は中小企業を所有していました。しかし、彼らには大きなマイナスがありました-つまり、特定のセルにアクセスする方法の欠如。

ダンは、問題を解決するためにいくつかのオプションを試しました。セルの連続番号付けから、変数のようにパーソナライズされた名前までです。 VisiCalcの重要な革新の1つは、おなじみのテーブル番号付けシステムです。文字は列を示し、番号は行を示します。したがって、潜在的に無限のテーブルを構築でき、その中で迷子になることはありません。

1979年6月、ニューヨークのNational Computer ConferenceでVisiCalc（Visual Calculator）と呼ばれる完成品が発表されました。会計の分野の財務家や専門家は、プログラムの可能性にすぐに注目を集めました。作成者はこれを「繰り返し計算できる魔法の紙」と表現しました。

会計業界にとっても、VisiCalcの出現は同様の転換点でした。「紙」の会計システムは、多くの場合、相互接続された多くのノートブックで構成されていました。監査プロセスに矛盾があった場合、エラーの検索には数週間ではなく数日かかる可能性があります。

スプレッドシートは、計算の実行にかかる時間を大幅に削減し、その精度を向上させました。新しい現実では、会計に必要なリソースがはるかに少なくなりました。その結果、Planet Moneyポッドキャストによると、1980年以降、米国だけで会計士の数は40万人減少しています。

ワードスター


ワードプロセッサは、PCユーザーが学習する最初のプログラムの1つです。しかし、常にそうであるとは限りませんでした。 Microsoft Wordの前身は、広く使用される権利をめぐって競争しなければなりませんでした。

執筆は常に高い収益を伴うことからはほど遠い-したがって、マイクロコンピュータの時代の初めに、多くの著者は業界のコンピュータ化を贅沢であると考えました。

まず、タイプライターの機能をシミュレートするためだけに、システムユニット、モニター、キーボード、プリンターなどのいくつかのデバイスが必要です。これは重要な投資です。次に、デジタルファイルを交換できるのは、他のコンピューターの所有者だけです。編集するには、結果を印刷する必要があります。壊れていない場合は修正しないでください。ワークフローを変更しない方が安上がりで信頼性が高くなります。


写真Klaas / Unsplash

この考え方は、WordStarプログラムによって変更されました。WordStarプログラムは、すべてのワードプロセッサで使用できる標準関数のセットを形成しました。WordStarは、WYSIWYG機能を備えた最初のワードプロセッサであり、マイクロコンピュータ用に設計され、長いテキストを作成するために「シャープ」になっています。

タイプライターの原則をエミュレートする代わりに、他のワードプロセッサが行うように、WordStarは通常のユーザーパラダイムではなく、デジタル製品に固有の柔軟性に依存していました。これにより、ソフトウェアの作成者は「上級ユーザー」向けの機能を開発および実装できました。

WordStarには、ドキュメントを表示すると表示されるが、印刷すると非表示になる「非表示」コメントのシステムがありました。これにより、必要に応じて戻ることができる仮想ブックマークを作成でき、Emacsでの作業を連想させるホットキーの便利なシステムを使用しました。私たちが慣れ親しんだWordStarの革新の1つは、柔軟なテキスト選択システムです。他のプログラムとは異なり、ユーザーは自発的にテキストを選択し、それをどうするかを決定することができます-辞書をコピー、切り取る、または開くことができます。


このプログラムはSF作家の間で人気を博しています。彼女のフォロワーには、今日までWordStarを使用してきたGame of Thronesの作者George Martinが含まれています。コナンオブライエンが2014年に番組に出演したとき、作家は、プログラムに必要なすべての機能が含まれていたため、「更新」の要点は見当たらないと説明しました。そして、過去5年間で彼がこのスコアについて考えを変えたことはほとんどありません。
