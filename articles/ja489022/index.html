MonbMQパート2でのRabbitMQの使用


（画像から取られた公式のRabbitMQウェブサイト）

で最初の記事、我々は2つのPHPのプログラムを書いたことを利用RabbitMQの：1がメッセージを送信し、第二は、受け取りました。この記事では、かなりの時間を消費するタスクを多数のワーカー（メッセージハンドラー）に分散するキューを作成する方法について説明します。

主なアイデアは、かなりの時間を消費するタスクをすぐに完了することではなく、メッセージの形でキューに追加することです。その後、作業中のワーカーはキューからメッセージを受信し、バックグラウンドであるかのように消費してタスクを実行します。
この概念はWebアプリケーションに非常に関連しています。たとえば、時間のかかるタスクを実行する必要があり、その結果がすぐに必要とされない場合があります。たとえば、電子メールの送信やサードパーティアプリケーションへのHTTPリクエストの作成（たとえば、libcurlを介して） PHP）。

料理


この部分では、ワーカーのタスクを表すメッセージを送信します。画像の処理やpdfファイルの生成などの実際のタスクはないため、sleep（）関数を使用して忙しいと仮定します。メッセージのポイント数をタスクの複雑さとして扱います。各ドットは1秒の作業を意味します。たとえば、Hello ...というメッセージは3秒の作業にかかります。前のレッスンのsend.php

スクリプトを少し変更して、任意のメッセージを送信できるようにします。

try {
   $producer = \MonsterMQ\Client\Producer();

   $producer->connect('127.0.0.1', 5672);
   $producer->logIn('guest', 'guest');

   $producer->queue('test-queue')->declare();

   $message = implode(' ', array_slice($argv, 1));
   $message = empty($message) ? 'Hello world!' : $message;

   $producer->publish($message, 'test-queue');

   echo "\n Sent {$message} \n";
} catch(\Exception $e) {
   var_dump($e);
}

receive.php スクリプトも変更する必要があります。受信メッセージの各ポイントで1秒間動作をシミュレートする必要があります。ファイルの名前をworker.phpに変更して、次のコードを記述します。

try {
   $consumer = \MonsterMQ\Client\Consumer();

   $consumer->connect('127.0.0.1', 5672);
   $consumer->logIn('guest', 'guest');

   $consumer->queue('test-queue')->declare();

   $consumer->consume('test-queue');

   $consumer->wait(function ($message, $channelNumber) use ($consumer){
      echo "\n Received: {$message}";
      sleep(substr_count($message, '.'));
      echo "\n Done";
   });
} catch(\Exception $e) {
   var_dump($e);
}

次に、両方のスクリプトを異なる端末で実行します。

# Shell 1
php worker.php 


# Shell 2
php send.php "A very hard task which takes two seconds.."

タスクキューを使用する利点の1つは、多くのワーカー間で作業を分散できることです。2つの異なる端末で2 つのworker.phpスクリプトを一度に実行してみましょう。そして3番目では、send.phpスクリプトでメッセージを送信します

# Shell 1
php worker.php 


# Shell 2
php worker.php 


# Shell 3
php new_task.php First message.
php new_task.php Second message..
php new_task.php Third message...
php new_task.php Fourth message....
php new_task.php Fifth message.....

次に、労働者が何をもたらしたかを見てみましょう。
# shell 1
php worker.php
# Received: First message.
# Done
# Received: Third message...
# Done
# Received: Fifth message.....
# Done


# shell 2
php worker.php
# Received: Second message..
# Done
# Received: Fourth message....
# Done

デフォルトでは、RabbitMQは後続の各メッセージを次に消費する受信者に順番に送信します。平均して、各受信者は同じ数のメッセージを受信します。メッセージを配信するこの方法は、ラウンドロビン（サークル内）と呼ばれます。3人以上の労働者でこれを試してください。

メッセージ確認


タスクには一定の時間がかかる場合があります。おそらく、このメッセージを最後まで処理できなかった受信者の作業を終了した場合に、メッセージがどうなるかについて興味があるかもしれません。現在のコードでは、メッセージの確認はMonsterMQでデフォルトで有効になっているため、メッセージはキューに返されます。

メッセージ確認を使用する場合、RabbitMQにメッセージが処理され、キューから削除する権利があることを伝えます。受信者が確認を送信せずに作業を完了した場合（たとえば、TCP接続の予期しない終了の結果として）、RabbitMQはメッセージが処理されなかったことを理解し、メッセージをキューに戻し、他の使用可能なワーカーに配信しようとします。したがって、ワーカーが予期せずシャットダウンした場合でも、メッセージが失われないようにすることができます。

MonsterMQでメッセージ確認を無効にするには、消費（）の 2番目の引数としてtrueを渡します。

$consumer->consume('test-queue', true);

受信したメッセージの確認を忘れることはかなり一般的で、簡単に許容できるエラーであり、深刻な結果につながる可能性があります。これが発生すると、メッセージは何度も配信され、キューに蓄積されてメモリを消費します。このエラーをデバッグするには、rabbitmqctlを使用してmessages_unacknowledgedフィールドを表示します

sudo rabbitmqctl list_queues name messages_ready messages_unacknowledged

Windowsでは、sudoをドロップします

rabbitmqctl.bat list_queues name messages_ready messages_unacknowledged

キューのセキュリティ


ワーカーのいずれかが予期せずに作業を終了してもメッセージは失われないという事実にもかかわらず、RabbitMQがシャットダウンされた場合でも、作成されたキューが失われる可能性があります。

キューが失われるのを防ぐには、キューを永続的（永続的、永続的）として宣言する必要があります。キューはべき等なので、同じ名前の宣言メソッドを呼び出してキューを変更または再作成することはできないため、新しいキューを宣言する必要があります。次のようにしましょう

$consumer->queue('new-queue')->setDurable()->declare();

送信者コードのキュー通知コードも変更することを忘れないでください。

メッセージの公平な分配


2人のワーカー間のメッセージがまだ正直に配信されていないことに気づいたかもしれません。たとえば、すべての偶数メッセージに時間がかかり、すべての奇数メッセージが迅速に処理される場合、時間のかかるメッセージを受信する1人のワーカーは常にビジーで、2番目のワーカーはアイドルになります。これを回避するには、サービス品質を使用できます。コードに次の行を追加しましょう

$consumer->qos()->prefetchCount(1)->perConsumer()->apply();

この行は、現在のメッセージを処理して確認するまで受信者にメッセージを送信しないようにRabbitMQに指示します。perConsumer（）は、受信者のすべてのチャネルにサービス品質を適用します。現在のチャネルのみにサービス品質を適用する場合は、perChannel（）メソッドを使用します。

コード全体をまとめる


これは、send.php（送信者）のようになります。

try {
   $producer = \MonsterMQ\Client\Producer();

   $producer->connect('127.0.0.1', 5672);
   $producer->logIn('guest', 'guest');

   $consumer->queue('new-queue')->setDurable()->declare();

   $message = implode(' ', array_slice($argv, 1));
   $message = empty($message) ? 'Hello world!' : $message;

   $producer->publish($message, 'test-queue');

   echo "\n Sent {$message} \n";
} catch(\Exception $e) {
   var_dump($e);
}

そして、受信者worker.php

try {
   $consumer = \MonsterMQ\Client\Consumer();

   $consumer->connect('127.0.0.1', 5672);
   $consumer->logIn('guest', 'guest');

   $consumer->queue('new-queue')->setDurable()->declare();

   $consumer->qos()->prefetchCount(1)->perConsumer()->apply();

   $consumer->consume('test-queue');

   $consumer->wait(function ($message, $channelNumber) use ($consumer){
      echo "\n Received: {$message}";
      sleep(substr_count($message, '.'));
      echo "\n Done";
   });
} catch(\Exception $e) {
   var_dump($e);
}
