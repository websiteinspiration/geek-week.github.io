Arbeiten mit Code: drei offene Tools

Lassen Sie uns über einige Texteditoren und ein Dienstprogramm zum Entwerfen von Code in der Dokumentation sprechen.



Weitere Entwicklertools: APIs und UI-Elemente

Mikro



Dies ist ein Terminal-Texteditor, der von modernen Systemen geschärft wird. Der Quellcode befindet sich auf GitHub und wird unter der MIT-Lizenz vertrieben . Die Entwickler sagen, dass das Produkt der ideologische Nachfolger des Nano- Editors für Unix-ähnliche Betriebssysteme ist.



Unter den interessanten Mikrofunktionen können wir die Unterstützung für die Maus (Ziehen, Doppelklicken, Dreifachklicken) und mehrere Cursor, die automatische Verknüpfung des Codes (Fehlerprüfung) und die Syntaxhervorhebung für mehr als 90 Sprachen herausgreifen . In einem thematischen Thread zu Hacker News stellen die Bewohner fest, dass das Dienstprogramm dem proprietären Sublime Text- Editor ähneltund hat viele seiner Funktionen: mehrzeilige Bearbeitung, Plugin-Verwaltung - sogar Hotkeys sind ähnlich.



Das Tool wurde vor ungefähr drei Jahren veröffentlicht, sodass bei seiner Arbeit immer noch Fehler auftreten . Obwohl einer der Bewohner von HN feststellte, dass die Mikrostabilität in letzter Zeit gewachsen ist. In jedem Fall hat das Tool eine Community (mehr als 14.000 Sterne auf GitHub) und es gibt eine aktive Kommunikation im Gitter- Chat.





Atom



Es ist ein plattformübergreifender Texteditor und ein Entwicklungstool von GitHub. Es basiert auf HTML, JavaScript, CSS, Node.js und verwendet das Electron- Framework .



Es bietet umfangreiche Schnittstellen- und Texteinstellungen - Sie können die Stile, Tastenkombinationen und sogar das Erscheinungsbild des Editors ändern, wenn Sie JavaScript kennen. Fast alle Einstellungen befinden sich in CSON- Dateien (JSON-Format, jedoch für CoffeeScript-Objekte). Da GitHub Atom entwickelt, verfügt es über eine Erweiterung , mit der Sie direkt über einen Texteditor auf das Versionskontrollsystem zugreifen können - Zweige verwalten, Commit- , Push- und Pull- Befehle ausführen . Es gibt auch ein Teletype- Paket- Mit ihm kann Code von mehreren Entwicklern gleichzeitig geschrieben werden.





Screenshot: atom.io / Die Arbeit mit Git in Atom



Atom ist ein relativ junges Tool, daher gibt es Leistungsprobleme. Die Autoren befassen sich jedoch mit der Optimierung .





Kohlenstoff



Mit dem Tool können Sie wunderschöne Screenshots von Code für Präsentationen, Artikel oder Dokumentationen erstellen. Es reicht aus, einen Teil des Programms in das Feld auf der Website zu kopieren und das Bild zu speichern (wenn Sie möchten, können Sie das Bild direkt auf Twitter veröffentlichen). Der Ingenieur Brian Dennis von FOSSA , einem unternehmensweiten Open-Source-Anwendungsmanagementsystem, ist an der Entwicklung von Carbon beteiligt .


Mit Carbon können Sie eine Hintergrundfarbe auswählen und die Syntaxhervorhebung anpassen. Es werden mehr als 30 Sprachen und Frameworks unterstützt: von C und Python bis Vue und R. Hier ist ein Beispiel für ein Bild, das auf einer Site generiert werden kann (der Code ist ein von Carbon angebotener „Stub“):







Die Autoren des Projekts planen, die Anzahl der verfügbaren Sprachen zu erhöhen, eine Version für mobile Geräte zu veröffentlichen und weitere Einstellungen hinzuzufügen. Der Quellcode für Carbon befindet sich im Repository auf GitHub .



Auf dem Gelände rund um das Instrument hat sich eine ziemlich große Gemeinschaft gebildet - mehr als 24.000 Sterne . Seine Mitglieder entwickeln benutzerdefinierte Erweiterungen. Zum Beispiel gibt es Plugins zur Integration von Carbon in die Texteditoren Emacs , Vim und das bereits erwähnte Atom.
