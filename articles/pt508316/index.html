Autorização rápida no Bitrix ou como alternar entre usuários sem inserir uma senha

Olá!



Não faz muito tempo, no trabalho, como parte do teste de um novo processo de negócios, eu precisava fazer o login com diferentes usuários.



Ir para a seção apropriada no painel de administração exigiu uma certa quantidade de tempo, o que não me agradou muito, por isso foi decidido encontrar um método de autorização mais rápido.



Como resultado, o componente QuickAuth apareceu , que será discutido abaixo.

O conjunto de extensões necessárias em si é formado e escrito em $ arResult na classe component.Recentemente,



gosto de usar o Vuejs na frente, pois o Bitrix o suporta imediatamente. Porém, devido ao fato de não haver acesso direto ao código de seleção do usuário, essa opção foi descartada imediatamente e a escolha foi feita em favor do JS nativo, que recebe o ID do main.user.selector e o envia ao servidor usando o método BX.ajax bitrix . runComponentAction : 



// script.js

window.onload = function() {

    let userId;
    
    const input = document.getElementById('select_user');
    const button = document.getElementById('authorize');

    input.addEventListener('change', () => {
        userId = input.value;
    })

    button.addEventListener('click', () => {
        if(!userId) {
            alert('  ');
        } else {
            const request = sendAjax('ramapriya:quick.auth', 'sendUserId', 'class', {
                user: userId
            });

            request.then(response => {
                if(response.result === 'error') {
                    alert(response.error_description)
                } else {
                    window.location.reload(true)
                }
            })
        }
    })

    async function sendAjax(component, action, mode, params = {}) {

        const request = await BX.ajax.runComponentAction(component, action, {
            mode: mode,
            data: params
        });

        return await request.data
    }
    
}


O método sendUserIdAction é responsável por receber dados na classe de componentes . Ele processa a solicitação usando o objeto Request e também chama um método que autoriza o usuário desejado: 



// class.php

private function authorize($userId) {

        $USER = new CUser;

        if($USER->Authorize($userId)) {
            return true;
        } else {
            throw new SystemException(' ');
        }

    }

    public function sendUserIdAction() {

        $request = Context::getCurrent()->getRequest();

        if(!$request['user']) {

            $result = [
                'result' => 'error',
                'error_description' => Loc::getMessage('QA_AJAX_RESPONSE_ERROR_DESCRIPTION')
            ];

        }

        $userId = (int) $request['user'];

        try {
            if($this->authorize($userId)) {
                $result = [
                    'result' => 'success'
                ];
            }
        } catch(SystemException $e) {
            $result = [
                'result' => 'error',
                'error_description' => $e->getMessage()
            ];
        }

        return $result;

    }
