Neutralino.jsについて

neutralino.jsとは何ですか？


このテクノロジーは、現在確立されているElectronおよびNodeWebkitのかなり興味深いアナログです。上記の技術的ソリューションとどう違うのですか？公式ドキュメントで提供されているステートメントを信じる場合、アプリケーション構築自体のアーキテクチャは優れているため、最終ビルドのサイズが十分に小さいWebテクノロジー上でクロスプラットフォームアプリケーションを作成できます。



インストールはどうですか？


公式ドキュメントに基づいて、Webテクノロジー上の独自のクロスプラットフォームアプリケーションの開発において、説明したテクノロジーをインストールおよび使用するための2つのオプションを区別できます。

  1. 完成したポータブルSDKをダウンロードします。
  2. neu-cliという名前の特別なCLIインターフェースを使用します。

この記事では、2番目のインストールオプションを正確に検討します。私自身の主観的な意見では、与えられたソリューションは、アーキテクチャと実際の両方の観点から最も包括的なものだからです。

#  npm
$ ~ npm install -g @neutralinojs/neu

#  yarn
$ ~ yarn global add @neutralinojs/neu

* nixシステムのユーザーへの注意

#  Linux Arch   
$ ~ sudo pacman -S webkit2gtk

#  Debian   
$ ~ sudo apt-get install libwebkit2gtk


主要なアーキテクチャ機能


このテクノロジーは、クライアント/サーバーアーキテクチャに基づいて実装されており、各オペレーティングシステム用に事前に準備されたバイナリファイルに基づく、かなり重要なビルドシステムを使用することも特徴です。
また、ソフトウェアの起動とデバッグにはいくつかのバリエーションがあることにも注意してください。

  1. クラウド-使用されているポートの人気を考慮に入れて、ソフトウェア製品を接続できることを特徴としています。
  2. ブラウザー-ユーザーのコンピューターのメインブラウザーが自動的に開くことを特徴とします。トークンシステムが使用されます。
  3. ウィンドウは、ソフトウェアの配布に使用される主な方法です。以前のバージョンと同様に、HTMLドキュメントに埋め込まれたトークンシステムが使用されます。

このテクノロジーの開発者は、最終的なアプリケーションを最小限の環境に固執しようとします。このことから、興味深いエラーや問題がいくつか発生します。

まず、機能がかなり厳しく制限されています。それが欠けているとどうなりますか？丁度。完全に異なる技術ソリューションを使用する必要があります。
第二に、公式ドキュメントではレンダリングプロセスとサーバープロセスの間に相互作用がありません。この場合、問題を解決するためのオプションは1つしかありません-開発者からのソースコードを表示することです。

第3に、使用するアセンブリシステムは、最小限のデータセットに対応していません。ソフトウェア配布では、* nixシステムとWindowsの両方のファイルを同時に配布する必要があるためです。

第4に、最終ビルドの宣言されたミニマリズムは、テストReactアプリケーションのアセンブリで終わります。最終的なビルドでは、180〜190 MiB程度になります。

結論


この記事で説明するテクノロジーは、Webテクノロジーに基づくデスクトップアプリケーションを作成するための興味深いアーキテクチャソリューションですが、同時に、現時点では、ElectronだけでなくNodeWebkitにも適切な競争を提供することはできません。 。

私の意見では、この問題は、使用されているアーキテクチャソリューションとテクノロジー自体の不十分なドキュメントに正確にあります。最終的なアプリケーションの機能を拡張する必要があるでしょうか？そして、そのようなミニマリズムはある時に必要ですか？結局のところ、RAMの20〜30 MiBの向上は、今日のハードウェアの基準ではかなり小さいものです。
