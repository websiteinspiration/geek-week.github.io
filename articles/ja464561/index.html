Reduxパターンを使用したiOSアプリケーションの作成


最近、私はモバイルよりもフロントエンドの開発に携わっており、すでに知っているが、実際にはそれらを使用していなかった非常に興味深いデザインパターンに出くわしました。

しかし、今やこれらすべてが理にかなっています。Reactでの開発から数週間使用した後は、以前のiOS開発方法に戻ることができなくなりました。私はモバイルアプリケーションを開発するためにJavaScript（別名React Native）に切り替えませんが、ここでいくつかのことを学びました。


iOS開発に戻り、新しいプロジェクトを作成してReSwiftの調査を開始しました。これは、SwiftのFluxおよびReduxパターンの実装です。そして、それは非常に簡単に機能します。JavaScriptアプリケーションのアーキテクチャを数回複製しましたが、今やグローバルな状態になり、コントローラーはこの状態をリッスンします。コントローラ自体は、非常に特定の動作をカプセル化するさまざまなプレゼンテーションコンポーネントで構成されています。



すべての状態変更は、レデューサー内の1か所で行われます。 1つはサブステート用です。すべてのアクションを1か所で確認できます。ネットワークコードや呼び出しコントローラー、ビュー内のオブジェクトの変更は不要です。スパゲッティコードはもう必要ありません。状態は1つだけであり、それが真である場合、さまざまなプレゼンテーションコンポーネント（および私はそれを主張します）は状態のさまざまな部分をサブスクライブし、それに応じて反応します。これは、強力なモデルアプリケーションに最適なアーキテクチャです。

例えば。以前は、ログインビューコントローラーには多数のコード行、さまざまな制御状態、エラー処理などが含まれていました。...これは次のようになります（例として）。

import UIKit
import Base
import ReSwift

class LoginViewController: UIViewController {

    @IBOutlet var usernameField: UITextField!
    @IBOutlet var passwordField: UITextField!

    override func viewDidLoad() {
        super.viewDidLoad()

        store.subscribe(self) {state in
            state.usersState
        }
    }

    @IBAction func onLoginButton(_ sender: Any) {
        store.dispatch(AuthenticatePassword(username: usernameField.text!, password: passwordField.text!))
    }

    @IBAction func onTwitterButton(_ sender: Any) {
        store.dispatch(AuthenticateTwitter())
    }

    @IBAction func onFacebookButton(_ sender: Any) {
        store.dispatch(AuthenticateFacebook(from: self))
    }
}

// MARK: - State management
extension LoginViewController: StoreSubscriber {
    func newState(state: UsersState) {
        if let error = state.authState.error {
            presentError(error: error.type, viewController: self, completion: nil)
        }
        if let _ = state.getCurrentUser {
            self.dismiss(animated: true, completion: nil)
        }
    }
}

コントローラーとグローバル状態でのディスパッチアクションの表現。これらのアクションは実際にはネットワークで機能するか、アプリケーションが新しい状態に変換する必要があるさまざまな部分を起動します。

アクションは別のアクションをトリガーできます。これは、ネットワークリクエストで発生する方法です。たとえば、1つのFetchUserアクション（id：String）と、SetUser（user：User）のように見えるレデューサーでインターセプトする1つのアクションがあります。レデューサーでは、新しいオブジェクトを現在の状態とマージ/マージする必要があります。

最初にstateが必要です。この例ではUserオブジェクトに焦点を当てますので、stateは次のようになります。

struct UsersState {
    var users: [String: User] = [:]
}

ユーザーオブジェクトのすべてのネットワークアクティビティをカプセル化するファイルが必要です。

struct FetchUser: Action {
    init(user: String) {
        GETRequest(path: "users/\(user)").run { (response: APIResponse<UserJSON>) in
            store.dispatch(SetUser(user: response.object))
        }
    }
}

リクエストが完了すると、別のactionが呼び出されます。このアクションは実際には空です。たとえば、UsersActionsで参照する必要があります。このアクションは、レデューサーが状態の変更に依存する必要がある結果を示します。

struct SetUser: Action {
    let user: UserJSON?
}

そして最後に、最も重要な作業はUsersReducerで行われます。アクションをキャッチし、その内容に従っていくつかの作業を行う必要があります。

func usersReducer(state: UsersState?, action: Action) -> UsersState {
    var state = state ?? initialUsersState()

    switch action {
    case let action as SetUser:
        if let user = action.user {
            state.users[user.id] = User(json: user)
        }
    default:
        break
    }
    return state
}

必要なのは、コントローラーまたはビューの状態をサブスクライブ/サブスクライブすることだけです。状態が変化したら、必要な情報を抽出して新しい値を取得します。
