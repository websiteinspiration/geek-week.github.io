純粋な関数 -決定論的で副作用のない関数。



つまり、関数が純粋であるためには、それ

が決定論的である必要があります。つまり、同じ引数のセットを使用するたびに同じ結果が生成されます。

純粋な関数の2番目に重要な品質は、副作用の欠如が

明確に示されることです。 

my_list = [32,3,50,2,29,43]

sort_by_sort関数は、要素の元のリストを変更し、コンソールに何かを表示するため、副作用があります。 



my_list = [32,3,50,2,29,43]

def sort_by_sorted(li):
  return sorted(li)

print(sort_by_sorted(my_list))
print(my_list)

>>>[2, 3, 29, 32, 43, 50]
>>>[32, 3, 50, 2, 29, 43]


前の例とは異なり、sort_by_sorted関数は元の配列を変更せず、それ自体をコンソールに出力せずに結果を返します。

高階関数 -プログラミングにおいて、他の関数を引数として取るか、別の関数を結果として返す関数。





FVPの例 



def func(num):
  return num**2

def higher_order_func(fun, num):
  return fun(num)+fun(num)

print(func(4))
print(higher_order_func(func,4))

>>>16
>>>32

それでは始めましょう



まず、次のことを理解する必要があります-一般に関数型プログラミングとは何か。個人的に、私は日常のプログラミングで最も頻繁に言及される2つのパラダイム、OOPとFPを知っています。



それを完全に簡略化して指で説明すると、次のようにこれら2つのパラダイムを説明できます。



  • OOPはオブジェクト指向プログラミングです。どのオブジェクトをパラメーターとして渡し、値として使用できるかをプログラミングするためのアプローチです。
  • このロジックを使用して、FPがプログラミングへのアプローチであることを確立することができます。どの関数を他の関数にパラメーターとして転送し、他の関数から返される値として使用できるかを使用する場合...答えは名前自体に隠されています。




私の好きな先生が言ったように ズベロクビクター・シェペレフ：「プログラミングにおけるすべての仕事はデータを使った仕事です。「私はデータをいくつか取り、それらをいじって、それを返しました。」



これはFPにも当てはまります。データを取り、関数を取り、遊んで、何かを出しました。



私はすべてをペイントするわけではありません、そうでなければそれはすごく長くなります。この記事の目的は、仕組みと仕組みを説明することではなく理解することです。そのため、ここではFPの主な機能を見ていきます。



ほとんどの場合、FP（私が理解しているとおり）は、単純化されたコードの記述です。関数スタイルで記述されたコードは、品質を損なうことなく通常のスタイルで非常に簡単に書き直すことができますが、より原始的です。FPの目標は、よりシンプルで理解しやすく、保守が容易で、メモリ使用量も少ないコードを書くことです。もちろん、それがなければ、プログラミングの永遠のモラルはDRY（Do n’t Repeat Yourself-Do n't Repeat）です。 。



次に、FP- Lambda関数で使用される主な関数の1つを分析します。



次の記事では、Map、Zip、Filter、Reduceなどの関数を分析します。



ラムダ関数



ラムダは、匿名関数を呼び出すためのPythonおよびその他のプログラミング言語のツールです。ほとんどの場合、これは何も言わず、どのように機能するかを明確にしないため、ラムダ式のメカニズムのみを説明します。



すべてが非常に簡単です。



例を考えてみましょう。たとえば、半径がわかっている円の面積を考慮する関数を記述する必要があります。



サークル面積式は



S = pi*(r**2)







どこ

Sが円の面積である

PIは-数学定数は、我々は標準ライブラリ数学から取得する3.14に等しい

私たちは関数に渡すことを円の半径は唯一の変数である- R







今のpythonでのフォーマットにすべてをしましょう 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

#              
def area_of_circle_simple(radius):
  return pi_const*(radius**2)

print(area_of_circle_simple(5))
print(area_of_circle_simple(12))
print(area_of_circle_simple(26))
>>>78.5
>>>452.16
>>>2122.64




見た目はいいようですが、ラムダで録音すると、見た目がかっこよくなります。 



import math #  math

pi_const = round(math.pi, 2) # pi    
#       
#  3.141592653589793     

print((lambda radius: pi_const*(radius**2))(5))
print((lambda radius: pi_const*(radius**2))(12))
print((lambda radius: pi_const*(radius**2))(26))

>>>78.5
>>>452.16
>>>2122.64


わかりやすくするために、匿名関数呼び出しは、上の例のように、どこでも宣言せずに使用することを意味します。



ラムダ関数は次の原理に従って機能します

2つの入力引数を持つ例を考えます。たとえば、次の式を使用して円錐の体積を計算する必要があります。 



V = (height*pi_const*(radius**2))/3






すべてをpythonで書いてみましょう： 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

#      
def cone_volume(height, radius):
  volume = (height*pi_const*(radius**2))/3
  return volume

print(cone_volume(3, 10))

>>>314.0


そして今それがラムダ形式でどのように見えるか： 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

print((lambda height, radius : (height*pi_const*(radius**2))/3)(3, 10))

>>>314.0




ここでの変数の数は無制限です。たとえば、3つの異なる変数を考慮に入れて、円錐台の体積を計算します。



円錐台の体積は、次の式で計算されます。 



V = (pi_const*height*(r1**2 + r1*r2 + r2**2))/3






そして、これは古典的にPythonでどのように見えるかです 



import math #  math

pi_const = round(math.pi, 2) # pi           3.141592653589793     

#      
def cone_volume(h,r1,r2):
  return (pi_const * h * (r1 ** 2 + r1 * r2 + r2 ** 2))/3

print(cone_volume(12, 8, 5))
print(cone_volume(15, 10, 6))
print(cone_volume(20, 12, 9))
