Rust 1.41.1リリース：修正リリース

Rustチームは、Rustの新しい修正リリース1.41.1を公開しました。Rustは、誰でも信頼性の高い効率的なソフトウェアを作成できるプログラミング言語です。


toolsを使用して以前のバージョンのRustをインストールした場合rustup、バージョン1.41.1にアップグレードするには、次のコマンドを実行するだけです。


rustup update stable

まだインストールしていない場合は、Webサイトの対応するページからインストールrustupできます。


安定版1.41.1に含まれるもの


Rust 1.41.1は、Rust 1.41.0で発生した2つの重大なリグレッションに特化しています。静的ライフタイムの不正確さと、セグメンテーションエラーを引き起こす不正確なコンパイルです。これらのリグレッションはRustの以前のリリースには影響しません。Rust1.41.0のユーザーはできるだけ早くアップグレードすることをお勧めします。Rust 1.0以降、ライフタイム'staticとトレイトの実装間の相互作用に関連する別の問題Copyがあり、このリリースでも修正されています。


static要素検証失敗


Rust 1.41.0では、static値の内部表現が一部変更されたため、借用アナライザーが誤ったプログラムを誤って解決しました。特に、借用アナライザーは、static要素のタイプの正しさをチェックしませんでした。これにより'static、static変数よりも有効期間が短い値を一時的に割り当てることができました。


static mut MY_STATIC: &'static u8 = &0;

fn main() {
    let my_temporary = 42;
    unsafe {
        //     1.41.0:
        MY_STATIC = &my_temporary;
    }
}

1.41.1では、そのようなコードはコンパイルされません。


error[E0597]: `my_temporary` does not live long enough
 --> src/main.rs:6:21
  |

  |         ------------^^^^^^^^^^^^^
  |         |           |
  |         |           borrowed value does not live long enough
  |         assignment requires that `my_temporary` is borrowed for `'static`
7 |     }

このエラーの詳細については、＃69114とそれを修正したPRを参照してください。


Copy生涯の実装'static


Rust 1.0以降、次のエラープログラムが正常にコンパイルされました。


#[derive(Clone)]
struct Foo<'a>(&'a u32);
impl Copy for Foo<'static> {}

fn main() {
    let temporary = 2;
    let foo = (Foo(&temporary),);
    drop(foo.0); //     `foo`.
    drop(foo.0); //     .
}

Rust 1.41.1では、この問題は上記と同じPRで修正されています。プログラムをコンパイルすると、次のエラーが発生します。
