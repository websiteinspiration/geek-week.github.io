ARA：直線上のポイントの最大数を見つけるアルゴリズム

最近、私はインタビューのための古典的なタスクに出くわしました：直線上（平面上、整数座標上）に立つポイントの最大数を検索すること。徹底的な検索のアイデアがすぐに頭に浮かびましたが、これは明らかにO（n ^ 2）に時間の複雑さがありますが、少なくともO（n * log（n））には別の方法があるはずだと思いました。30分後、さらに良いものが見つかりました！



入出力の例を含む問題のより詳細な説明は、GeeksForGeeksとLeetCodeにあります。

最初に、問題を水平線または垂直線についてのみ簡単に解決でき、辞書の各点のX / Yを追加できることに気付きました。そして、他のラインの違いは何ですか？ただの傾斜？..しかし、それは修正可能です！

したがって、ポイントのリスト全体を回転させることで、リスト全体を何度も移動できると判断しました。そしてO（n）の解が得られます！係数は大きいですが。私は自転車を発明しなかったことを本当に望みます:)

# ***  ***

#  
#    = 180/ROT_COUNT 
#
#      12,
#  180*4    (6% ),
#  180*20   (0% ).
# ó    .
#     - .
ROT_COUNT = 180*10

# 
#        ,
#       1 / MULT_COEF.
#    4  .
#   MULT_COEF   ROT_COUNT.
# ó    - .
#     - .
MULT_COEF = 2 ** 3

angles = list(map(lambda x: x*180.0/ROT_COUNT, range(ROT_COUNT)))

def mnp_rotated(points, angle):
	"""Returns Maximum Number of Points on the same line with given rotation"""
	#  
	rad = math.radians(angle)
	co = math.cos(rad)
	si = math.sin(rad)
	#     
	counts = {}

	for pair in points:
		#   
		nx = pair[0]*co - pair[1]*si

		#       ,
		#     
		#      
		#     
		nx = int(nx * MULT_COEF)

		#  
		if nx in counts:
			counts[nx] += 1
		else:
			counts[nx] = 1

	#   
	return max(counts.values())


def mnp(points):
	"""Returns Maximum Number of Points on the same line"""
	res = 0
	best_angle = 0
	#     
	for i in angles:
		current = mnp_rotated(points, i)

		#  ,    
		if current > res:
			res = current
			best_angle = i

	return res

その後の徹底的な検索の実装には、さらに多くのコード行が必要でした。

ヘッダーには、回転アルゴリズムの実行時間の測定値と、ポイントの数に応じて完全に列挙されたグラフが含まれています。

ここでグラフを開く


約150ポイントは、時間の中で線形の複雑さの利点を示しています（上記のコードで使用されている係数を使用）。その結果、このアルゴリズムには次の欠点があります。

  • 仕事の正確さは絶対ではありません。
  • 小さなポイントセットの実行時間は長くなります。
    一般に、これは適切な係数を選択することで簡単に修正できます。単純なセットの場合、2000ではなくROT_COUNT = 36で十分であり、コードを50回以上高速化できます。

そしてそのような利点：

  • 小数座標で落ち着いて動作します。
  • 時間の複雑さは線形であり、大規模なデータセットで顕著です。

ここに 測定値の表があります。アルゴリズムとさまざまなチェックの両方を備えたプロジェクトのソースコード全体がGitHubにあります。
