必要なオブジェクトを検出するためのニューラルネットワークの構成


ニューラルネットワークは、幅広い問題の解決に役立つ複雑な数学的アルゴリズムです。使用する前に、セットアップしてトレーニングする必要があります。ニューラルネットワークの学習は、非常に長い時間とコストのかかる計算プロセスです。ただし、これらの問題を簡略化できるツールがあります。今日は、ニューラルネットワークを再トレーニングして、画像内の特定のオブジェクトを検出するプロセスについて検討します。

例として、YOLOv3アルゴリズムを使用してオブジェクトを検出します。記事の追加資料はgithubページにあります。インストールと構成の例は、Windows 10で実行されます。

まず、このgithubページから、特定のタイプのオブジェクトのニューラルネットワークを再トレーニングするために必要な資料をダウンロードまたは複製する必要があります。アクセスまたは権限の問題を回避するために、アカウントのディレクトリに資料を配置することをお勧めします。例：C：\ Users \ user_name \ Documents \ NN \。

1. yolov3-masterフォルダーに移動し、ターミナル（コマンドライン）を呼び出して、pip install -U -r requirements.txtコマンドを実行します。コマンドを実行したときに、一部のパッケージがインストールされていませんでした。これは、Windowsのpipリポジトリに、この名前のパッケージが存在しないためです。インストールを成功させるには、requirements.txtでいくつかのパッケージ（「torch> = 1.5」と「pycocotools」）をコメント化する必要があります。次に、次のコマンドでpytorchをインストールします。

pip install torch==1.5.0 torchvision==0.6.0 -f https://download.pytorch.org/whl/torch_stable.html

私たちの場合、pycocotoolsパッケージは必要ありません。計算がビデオカードで行われるように、CUDAツールをダウンロードしてインストールすることも必要です。これにより、再トレーニングの速度が向上する場合があります。

2.インストールが正常に完了したら、データの準備を開始できます。アルゴリズムが適切に機能するためには、トレーニング用の写真は、処理が実行される素材から取得する必要があります。たとえば、オブジェクトが大きな角度で配置されている部屋に設置されている監視カメラからのビデオ素材を処理する必要があります。このような状況では、ニューラルネットワークがトレーニングされたインターネット上の画像とは非常に異なるオブジェクトを検出するアルゴリズムをトレーニングすることは非常に困難になります。したがって、トレーニングサンプルは、ビデオ資料を使用したスクリーンショットから、またはカメラによる手動撮影を使用してコンパイルされます。トレーニングサンプルのサイズは大きく異なる可能性があり、すべてタスクの性質によって異なります。この例では、データセットは46の画像で構成され、3種類のオブジェクト（ペン、鉛筆、マーカー）を表しています。表とA4シートを背景に撮影を行いました。

3.次のステップは、画像内のオブジェクトをマークすることです。ただし、マークアップを開始する前に、いくつかの拡張方法を使用してトレーニングサンプルを増やす必要があります。これは、さまざまなバリエーションのオブジェクトを認識するニューラルネットワークの機能を改善するために重要です。このノートブックには、拡張の例がいくつかあります。拡張が完了すると、元の画像と拡張結果を1つのフォルダーに結合します（画像がyolov3_masterフォルダーのルートにあることが重要です。例：C：\ Users \ user_name \ Documents \ NN \ yolov3_master \ images \）。次に、labelImgプログラムを使用してオブジェクトをマークアップし、次の手順を順番に実行します。



  1. 画像のあるフォルダを選択（... \ yolov3_master \ images \）;
  2. . labels, (….\yolov3_master\labels\);
  3. c PascalVOC YOLO;
  4. ;
  5. , ;
  6. (Pen, Pencil, Marker), ;
  7. ;
  8. ;
  9. 4 – 8.

4.次に、準備されたサンプルをトレーニングとテストに分割する必要があります。この場合、2つのテキストドキュメント（train_112.txtおよびtest_24.txt）を作成する必要があります。これは、各画像の完全な場所のパスを示します。ドキュメントの作成例は、ノートブックにあります。

5.目的のオブジェクトに名前を付けるには、ラベルフォルダーのclasses.txtに示されているのと同じ順序で名前をリストする、objects.namesなどの拡張子.namesのファイルを作成する必要があります。これは次のようになります。

6.再トレーニングするには、構成ファイルを構成する必要があります。これを行うには、フォルダー（... \ yolov3_master \ cfg）からファイルyolov3-spp.cfgをフォルダー（... \ yolov3_master \ data）にコピーします。次に、3つの各レイヤーの行（636,643-722,729-809,816）を編集する必要があります。



ここで、filters =（5 + n）* 3およびclasses = nです。 nはクラスの数です（この例では3）。

7.次に、構成済み構成ファイルの重みをダウンロードする必要があります。これを行うには、ページからyolov3-spp.ptファイルをダウンロードして、（... \ yolov3_master \ data）に配置します。たとえば、yolov3-sppアルゴリズムが使用されます。ただし、アルゴリズムにはいくつかのバリエーションがあります。たとえば、yolov3-sppの軽量バージョンであるyolov3-tinyです。ライトバージョンは精度が低くなりますが、材料の処理速度ははるかに速くなります。

8.オブジェクトタイプのアルゴリズムの再トレーニングに直接進みます。これを行うには、フォルダー... \ yolov3_master \でコマンドラインを実行し、次のコマンドを入力します。

python train.py --cfg data/yolov3-spp.cfg --data data/conf.data --weights data/yolov3-spp.pt --nosave

次のタイプのエラーも表示されることがあります。



このエラーは、計算グラフを作成するのに十分なビデオカードメモリがないことを示しています。この問題を解決する1つの方法は、トレーニング中にバッチのサイズを減らすことです。これを行うには、ファイル（... \ yolov3_master \ train.py）で、batch_sizeパラメーターを変更する必要があります。デフォルトは16で、4の倍数の低い値（8または4）に変更し



ます。初期化中にパラメーター値を使用して、この値を明示的に渡すこともできます。

python train.py --cfg data/yolov3-spp.cfg --data data/conf.data --weights data/yolov3-spp.pt --batch-size 4 --nosave

上記の例では、構成ファイル、トレーニングセット、重み、および保存オプションに関する情報を含むファイルを指定しました。

--nosaveパラメーターは、スケールを含む最終的なファイルのみが保存されることを示します。たとえば、400エポックごとに重みを保存する場合は、train.pyファイルで保存モードを設定することもできます。



--epochsパラメーターを指定することも重要です。元号の数は、オブジェクトのタイプ、トレーニングサンプルのサイズ、およびその他の要因によって異なります。この例では、最初は46枚の画像があり、拡張方法を使用してデータセットを136に増やしました。そのうち112枚はトレーニングサンプル、24枚はテストサンプルです。計算は、GTX 1060 6 GBグラフィックスカードで実行されました。最初の試みは300時代に行われ、トレーニングには約2.5時間かかりました。アルゴリズムの非常に低い精度を明らかにしました。 2番目の試みは1200時代に実施され、トレーニングには約12時間かかりました。トレーニングが完了すると、重みがフォルダー（... \ yolov3_master \ weights \）に保存されます-ファイルlast.pt.アルゴリズムの操作性を確認するには、次のコマンドを実行する必要があります。

python detect.py --weights weights/last.pt --cfg data/yolov3-spp.cfg --names data/objects.names

コマンドを実行する前に、アルゴリズムで処理するイメージをフォルダー（... \ yolov3_master \ data \ samples \）に配置する必要があります。結果はフォルダ（... \ yolov3_master \ output \）に表示されます。



したがって、完成したニューラルネットワークの再トレーニングが実行され、このリストに含まれていない新しい3種類のオブジェクトのオブジェクトの80クラスを決定できました。マーカーオブジェクトの場合、アルゴリズムは完全に正確ではなく、信頼度は0.5未満です。理由の1つは、トレーニングサンプルのサイズが小さいことです。この例では、マーカー付きの画像は26％で、残りはペンと鉛筆です。
