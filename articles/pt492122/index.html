Migrando do Gitolite para o GitLab com um Shell Script

O processo de migração geralmente é uma tarefa difícil, especialmente quando a quantidade de informações que precisam ser transferidas é tão grande que se torna mais rentável automatizá-las. Foi a necessidade de migrar do Gitolite para o GitLab que me levou a escrever um artigo sobre minha experiência nesse assunto.


Para migrar repositórios, usarei um computador com o sistema operacional CentOS 7. Instalado, preciso instalar o seguinte conjunto de aplicativos: git, ssh-agent, curl, jq, xargs .


Para começar, precisamos obter a chave de acesso à API do GitLab. Na interface da web, vá para a seção de configurações do usuário. Em seguida, selecione o item de menu “Access Tokens” . No formulário que é aberto, você deve especificar o nome da chave recebida e também pode especificar o período em que a chave estará ativa. Abaixo, marque a caixa ao lado do item da API e clique no botão "Criar token de acesso pessoal" . Essa chave será usada ao enviar solicitações para a API do GitLab.

    # $1 - gitlab remote url
# $2 - api access token
# $3 - remote repository name
# $4 - remote repository namespace identifier
initRemoteRepository() {
    local response=`curl -s -k -X POST --header "Private-Token: $2" --data "name=$3&namespace_id=$4&visibility=private&description=$3" $1/api/v4/projects`
    local id=`echo $response | jq -r ".id"`
    echo "$id"
}

    done

    cd ..
