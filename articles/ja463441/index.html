サンプラー。シェルコマンドの結果を視覚化するコンソールユーティリティ

一般的に、シェルコマンドを使用すると、コードや統合を記述せずに任意のメトリックを取得できます。したがって、コンソールには視覚化のためのシンプルで便利なツールが必要です。



データベースの状態変化の監視、キューのサイズの監視、リモートサーバーからのテレメトリ、スクリプトスクリプトの起動、完了時の通知の受信は、シンプルなYAMLファイルを使用して1分で構成されます。


コードはgithubで入手できます。インストール手順-Linux、macOSおよび（実験的）Windows用。


完全な監視システムがあるのに、なぜこれが必要なのですか？


これは、フルスケールのダッシュボードと監視に代わるものではないことを、すぐに言っておく必要があります。Cは、サンプラー比較プロメテウス+ Grafana比較と同じ- tailとlessを有する弾性スタックまたはSplunkの。


しかし、すずめの大砲のように、タスクの生産監視を上げて構成する場合、おそらくサンプラーが質問への答えになります。これは、ロケールとリモートサーバーのメトリックをプロトタイピング、デモンストレーション、または単に監視するためのツールとして考えられました。

Runchart




runcharts:
  - title: Search engine response time
    rate-ms: 500     # sampling rate, default = 1000
    scale: 2         # number of digits after sample decimal point, default = 1
    legend:
      enabled: true  # enables item labels, default = true
      details: false # enables item statistics: cur/min/max/dlt, default = true
    items:
      - label: GOOGLE
        sample: curl -o /dev/null -s -w '%{time_total}'  https://www.google.com
      - label: YAHOO
        sample: curl -o /dev/null -s -w '%{time_total}'  https://search.yahoo.com
      - label: BING
        sample: curl -o /dev/null -s -w '%{time_total}'  https://www.bing.com

Barchart




barcharts:
  - title: Local network activity
    rate-ms: 500        # sampling rate, default = 1000
    scale: 0            # number of digits after sample decimal point, default = 1
    items:
      - label: UDP bytes in
        sample: nettop -J bytes_in -l 1 -m udp | awk '{sum += $4} END {print sum}'
      - label: UDP bytes out
        sample: nettop -J bytes_out -l 1 -m udp | awk '{sum += $4} END {print sum}'
      - label: TCP bytes in
        sample: nettop -J bytes_in -l 1 -m tcp | awk '{sum += $4} END {print sum}'
      - label: TCP bytes out
        sample: nettop -J bytes_out -l 1 -m tcp | awk '{sum += $4} END {print sum}'

Gauge




gauges:
  - title: Minute progress
    rate-ms: 500        # sampling rate, default = 1000
    scale: 2            # number of digits after sample decimal point, default = 1
    percent-only: false # toggle display of the current value, default = false
    color: 178          # 8-bit color number, default one is chosen from a pre-defined palette
    cur:
      sample: date +%S  # sample script for current value
    max:
      sample: echo 60   # sample script for max value
    min:
      sample: echo 0    # sample script for min value
  - title: Year progress
    cur:
      sample: date +%j
    max:
      sample: echo 365
    min:
      sample: echo 0
