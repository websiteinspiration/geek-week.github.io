1.ソリューションの特徴


1Cでは、ファイルシステムからHTMLフィールドのページにローカルファイルを読み込むことができないため、Webサーバーまたは少なくとも一時記憶域内のこのファイルのアドレスが必要です。1Cの横にTiny jsライブラリを配置し、HTMLドキュメントのヘッダーに読み込みます。不可能です。また、既製のエディターウィンドウを内部的に形成し、HTMLドキュメントに既に接続されているHTMLレイアウトのファイルを作成する必要があります。

一般に、実際には、リポジトリーでライブラリーのアドレスを取得した場合、ライブラリーのロードは、ライブラリーが1つのファイルにコンパイルされている場合に実行できます。また、Tinyは、メイン・ライブラリーに加えて、プラグインjs-files、スタイル・マネージャー、およびスタイルを含み、ファイル自体をクラックします、アイコンなど、あなたが理解しているように、これはタスクを非常に複雑にします。かつては、これらすべてを1つのファイルにコンパイルして問題を知らないという考えさえありましたが、ツールを理解するのに長い時間がかかりました。その後、すべてのjsを1つのファイルにコンパイルするTinyMCE Builderが公式Webサイトで見つかりました。cssで同じことをした場合、小さな勝利になりますが、cssはまだいくつかのフォルダーの横にあります。

TinyMCEは、編集ウィンドウ用のiframeを作成します。これは、ページに埋め込まれたHTMLドキュメントであるため、このiframe 1Cで発生するイベントは追跡できません。

開発時のUT 11.4の構成には互換モード8.3.12があったため、実際には組み込みブラウザーとしてIEを使用していましたが、さらなる更新により、すでにWebキットを使用する新しいバージョンへの移行が示唆されました。

3.決定


ブラウザーと1Cの相互作用に関する問題を解決するために、通常のトリックが適用され、HTMLページに非表示のボタンを作成し、1CがHTML Pressイベントでインターセプトする必要なイベントがページ内で発生したときに、JSからそれをクリックすることを計画しました。しかし、どのようなイベントでプログラムプレスを実施するのでしょうか。

レイアウトは次のようになります。

<body> 
	<textarea id="editor"></textarea>
	<button id="button1c" style="display: none"></button>
</body>

そして、ボタンクリックイベントをトリガーする関数

function clickButton() 
{
  button1c.click();
}

以前は、HTMLフィールドでフォーム1Cのオブジェクトをオブジェクトとして転送することが可能でしたが、IEがCOMを介して1Cに接続しているため、フォーム自体がIEが理解できるオブジェクトに変換され、JSコードから直接フォーム1Cのエクスポート関数を呼び出すことができました。今では不可能です。

Tiny APIのドキュメントを調査したところ、ドキュメントのHTMLフィールドをアクティブ化する必要のあるイベントハンドラーとイベントの適切な組み合わせをインターセプトするための構造が見つかりました

tinymce.activeEditor.on('click', function(e){clickButton(e)});

tinymce.activeEditor.on('KeyDown', function(e){clickButton(e)});

このコードは、Tinyが既に初期化されている場所に配置する必要があります。ウィンドウオブジェクトのonloadハンドラーでこれを行いました。ただし、Tinyをパラメーターで初期化するときに、より美しく実行してコードを配置できます。

init_instance_callback

これですべてが整いました。フォームは正常に機能するはずです

。HTMLフィールドは実際にアクティブになり始めましたが、何らかの理由で一部のクリックでフォーカスがフィールドに移動しなかったということはよくなりました。



このエントリは、カーソルが点滅している行をクリックするとフォーカスが移動することを示しています。フィールドは正常に機能しますが、下をクリックしてもフィールドはアクティブになりません。

それは迷惑なアーティファクトであり、何が起こっているのかを理解しようとしても解決策はありませんでした。ハンドラーにアラートを出力し、アラートが表示されましたが、1CがPRESS Pressイベントに応答しませんでした



突然、そのアイデアが思い浮かびました。1Cがフィールドで発生しているイベントを処理できなかった場合、JSスクリプトの実行を少し遅くして、何が発生しているのかを確認するとどうなるでしょうか。JSの一時停止機能は、promiseとasync awaitを使用してすばやく実装されました。JSで確認できるように、デフォルトでは存在せず、何も住んでいません）。

function sleep(ms){

  return new Promise(resolve => setTimeout(resolve, ms));

}

async function clickButton(e){

  await sleep(1);

  button1c.click();

}

ここでは、ボタンをクリックする前に、遅延を1ミリ秒に設定しています。そして出来上がり、うまくいきました。1Cは、ボタンのクリックを時間内に処理し始めました。
