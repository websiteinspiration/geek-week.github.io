N + 1スキームに従ってTier IVデータセンターを構築する方法

独立したパラレルバス（IPバス）を備えたUPSシステム-データセンターの容量の増加に対する開発者の答え。 IP-Busを備えた多くのデータセンターは、Tier IV Uptime Institute証明書を含むものを含め、すでに世界中で構築されています。そのような決定はロシアの顧客によって監視されています。

データセンターを構築する慣行では、データセンターの拡大に向けた着実な傾向があります。 100 MWの容量を持つオブジェクトが世界に登場しました。ロシアもこの方向にいくらか遅れをとってはいますが、脇に立ちません。 10年前、我が国では、5 MWの容量を持つデータセンターが大規模であると考えられていましたが、今日、いくつかの主要な事業者が2,000ラック以上の設備を建設する計画を発表しました。これは、15 MW以上のエネルギー消費に相当します。

世界の慣行が示しているように、高出力のエンジニアリングシステムの編成では、経済的な観点から最も便利なのは、デバイスの並列N + x（N + 1、N + 2 ...）接続を備えた回路です。さらに、そのようなソリューションで使用できる世界最大のUPS設備-ダイナミックのユニット容量は、UPSでの作業に使用される最大のディーゼルエンジンの出力（およびコスト）によって制限されます。

ただし、効果的なN + x構成を作成する機能を提供するUPSの直接並列接続には、いくつかの重大な欠点があります。

  • 低電圧設備は、最大5 MWのシステムでのみ使用できます。これは、低電圧パッケージデバイス（6300 A）の使用可能な定格の制限と、150 kAを超える可能性がある大きな短絡電流の両方によるものです。
  • 5 MWを超えるシステムの構築を可能にする中電圧ソリューションは、電力システムのコストを増大させ、運用の点で常に顧客に適するとは限りません。
  • システムの一般的なコンポーネント（入力バスと出力バス、バイパス）は、一般的な障害ポイントです。

Tier IVアップタイムインスティテュートのフォールトトレランスのレベルに対応するN + N（2N）スキームは、個別のエネルギーモジュールを構築することで、従来の並列システムの主な欠点を回避することを可能にします。ただし、このアプローチには他にも明らかな欠点があります。

  • 機器の100％の複製、つまり 高い資本コスト;
  • 大きな設置面積;
  • 最大負荷レベル-50％（実際には-40％以下）。
  • 高い運用コスト。

これらの理由により、N + N（2N）構成は、容量が10 MWを超える施設ではめったに使用されません。

2005年に、0.4 kVの低電圧を維持しながら、並列回路の主な利点（N + x回路のUPSモジュールの最適な数）を維持しながら、最大20 MWの容量のシステムを実際に導入するというソリューションが見つかりました。 IP-Bus構成と呼ばれるこのソリューションは、最高レベルのフォールトトレランス（Tier IV Uptime Institute）を満たしています。 IPバスのアイデアは、リングバスを使用して個々のUPSモジュールを接続することに基づいています。各UPSモジュールは、リアクターを使用して分離されています（図1）。

  • UPS間で有効電力を再分配できます。負荷の低いUPSモジュールは、IPバスを介して過剰な電力を伝送することにより、他のモジュールを「支援」します（図2）。

「直接」並列構成とは対照的に、IP-Busシステムでは、各UPS設置が他から独立して出力電圧を制御します-集中制御装置がなく、共通の障害点が排除されます。何らかの理由で1つのUPSからの電力潮流が突然なくなったと仮定すると、その負荷はIPチョークを使用してIPバスに接続されたままであり、バックアップ電源として機能します。このシナリオでは、負荷は自動的かつ中断なくIPバスから電力を受け取ります（図3を参照）。

実際には、IP-Busは通常、図1に示すように、リングの形で作成されます。 4.リターンバスと呼ばれることが多いIPバスの2番目のセグメントは、負荷のバックアップソースとして機能し、個別のスイッチを介してIPバスに直接接続できるようにします。これは、緊急時にも負荷の定格電圧を保証する一種のバイパスです。状況またはサービス作業を実行するとき。上記のように、バイパスが閉じるまで、負荷はIPチョークを介してIPバスから電力を供給され続けるため、このようなバイパスは一般的な障害点ではありません。


図。 4バックアップIPバスから直接ロードNo. 2を操作する例

短絡シナリオでのIP-Busシステムの動作も、「直接」並列構成のプロセスとは大きく異なります。 IP-Bus回路では、IPチョークの使用による短絡の可能性は、負荷への影響がごくわずかです。この場合、短絡電流は100 kAを超えないため、標準のスイッチング、保護、バス機器を使用できます。

UPSの負荷側で短絡が発生した場合（図5を参照）、システム全体へのそのような短絡の影響は、残りの負荷が直列に接続された2つのリアクターによってUPSから分離されているため、比較的重要ではありません。一方、UPSから一般的なIPバスに供給される短絡電流は、IPチョークの抵抗によって制限されます。したがって、影響を受けていない負荷での電圧の変化は重要ではなく、ITI曲線の安全領域（CBEMA）に残ります。


図。 5. UPS 2に接続された負荷電源バスに短絡があるIP-Busシステムの短絡電流の分布と値の例

IPバスで短絡が発生した場合、障害点とUPSまたは負荷の間に配置されるIPリアクターは1つだけです。したがって、このシナリオの負荷間の電圧降下は、負荷分散システムの短絡と比較してはるかに大きくなります。遷移抵抗が低いと、負荷両端の初期電圧降下は30％になります。敏感なサーバー電源の場合、ITI曲線（CBEMA）によると、この電圧降下は最大500 msの間許容されます。セグメント化された方向保護の使用は、IP-Busシステムの要件に特別に適合されており、短絡を選択的に分離することにより60 msの間IP-Busの短絡をクリーニングでき、同時に、直接影響を受けないIP-Busシステムの一部を完全に維持できます。実行可能。

図。 6.回転式UPS Piller UNIBLOCKと「より低い」ターンオンの外部DGUを使用したIPバスシステムの例

Pillerの実際の経験によれば、バックアップエネルギー貯蔵デバイスとしてフライホイール（図6）を備えたダイナミックUPSは、IPバスシステムに最適です。 DIBPの一部としてのキネティックドライブは、瞬間的なエネルギー吸収と瞬間的な放電の両方のモードで動作できます。これは、負荷が変化したときにIPバスシステムの動作パラメーターを安定させるために重要です。

さらに、DIBPのモータージェネレーターは、最大20 x Inomの高い短絡電流を供給することができます。これにより、IP-Busシステムは、隣接する負荷を短絡の悪影響にさらすことなく、非常に長い時間の短絡洗浄に対処できます。
バッテリーを備えたスタティックUPSは、高電流を瞬時に送受信する能力に制限があり、さらにUPS自体の短絡電流は比較的低いです。これらの理由により、スタティックUPSのIP-Busソリューションは実験的なものであり、既存のデータセンターには実際にはありません。

世界初のIP-Busシステムは、2007年にAshburn（米国バージニア州）の36 MWデータセンターに実装されました。施設には2つの独立したIPバスシステムが設置されており、それぞれに14 + 2構成のフライホイールを備えた16台のPiller UNIBLOCK UBT 1670 kVA UPSが含まれています。外部ネットワークが長期間停止した場合、各DIBPは、「低いターンオン」を備えた別個の2810 kVAディーゼル発電機によって予約されます。無停電電源装置と保証された電源装置の両方の負荷で動作します
最初のIPバスシステムの成功に続き、この構成はデータセンター業界で急速に人気を博しました。 IP-Busテクノロジーの開発と認知におけるもう1つのマイルストーンは、2017年9月のTier IV Design＆Facility Uptime Institute証明書のIP-Bus N + 1電源システムを備えたオーストラリアのNEXTDC B2データセンターによる受領でした。

ロシアのデータセンター市場は、10 MWを超える容量の大規模施設の建設段階に入るに過ぎません。ロシアのいくつかのデータセンタープロジェクト（5〜15 MWの容量の範囲）でのIP-Busソリューションの最初の概念計算と予算評価の結果に基づいて、次の結論を導き出すことができます。静的UPSの2N構成と比較して、DIBPに基づくIP-Busソリューションは、初期資本コストがそれほど高くなく、占有スペースが30〜60％増加し、10年間の所有コスト（TCO）が50％以上高くなります。静的および動的UPSの両方に実装されたN + 1分散冗長構成（DR 3 / 2、4 / 3）と比較して、DIBPに基づくIP-Busソリューションは、初期資本コストがそれほど高くありません（容量が10のデータセンターの場合） MW以上）、占有領域で20〜50％の増加、10年間のTCOで50％の利益増加

したがって、ロシアのデータセンターでのIP-Busシステムの実装は時間の問題に過ぎないと確信しています。
