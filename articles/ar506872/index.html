التمهيد الربيع للكتاب 2: أفضل الممارسات للمحترفين

مرحبا ، هابروجيتلي! هل تريد زيادة فعاليتك في تطوير تطبيقات جافا وتطبيقات السحابة؟

زيادة سرعة وسهولة تطوير الخدمات الدقيقة والتطبيقات المعقدة ، والقضاء على مخاوف تكوين الربيع.

استخدم أدوات إطار Spring Boot 2 و Spring 5 مثل WebFlux و Security و Actuator وإطار عمل Micrometer ، الذي يوفر طريقة جديدة لجمع المقاييس.


إطار اختبار الربيع


تتمثل إحدى الأفكار الرئيسية لإطار عمل الربيع في تشجيع المطورين على إنشاء فئات بسيطة ومرتبطة ، وواجهات برمجة ، بحيث يصبح البرنامج أكثر موثوقية وقابلية للتوسيع بشكل أفضل. يوفر إطار عمل الربيع أدوات تبسط اختبار الوحدة والتكامل (في الواقع ، إذا كنت تقوم ببرمجة الواجهات حقًا ، فلن تحتاج إلى اختبار وظائف تطبيقات الربيع). بعبارة أخرى ، من الضروري اختبار التطبيق باستخدام أنظمة اختبار JUnit أو TestNG استنادًا إلى كائنات (تم إنشاؤها ببساطة باستخدام عامل التشغيل الجديد - بدون Spring أو أي حاوية أخرى).

يتضمن إطار الربيع العديد من الحزم المصممة لاختبار الوحدة أو التكامل للتطبيقات. بالنسبة لاختبار الوحدات ، تم تصميم العديد من كائنات المحاكاة (البيئة ، PropertySource ، JNDI ، Servlet ؛ أدوات الاختبار التفاعلي المساعدة ServerHttpRequest و ServerHttpResponse) ، والتي يمكنك من خلالها إجراء اختبار عزل التعليمات البرمجية.

يعتبر اختبار التكامل من أكثر إمكانيات الاختبار استخدامًا في إطار عمل الربيع. مهامها الرئيسية:
  • إدارة التخزين المؤقت لحاوية Spring IoC بين عمليات الاختبار
  • ادارة العمليات التجارية؛
  • حقن التبعية لحالات كائن الاختبار
  • إنشاء فئات أساسية مصممة خصيصًا لفصل الربيع.

يوفر إطار عمل الربيع طريقة سهلة للاختبار من خلال الدمج في اختبارات سياق التطبيق (ApplicationContext). تقدم وحدة اختبار الربيع عدة طرق لاستخدام ApplicationContext برمجياً واستخدام التعليقات التوضيحية.

  • Bootstrap الملخص مع. تعليق توضيحي على مستوى الفصل لتكوين تمهيد التشغيل لإطار الربيع TestContext.
  • ContextConfiguration تعليق توضيحي. يحدد البيانات الوصفية على مستوى الفصل والتي تحدد كيفية تحميل ApplicationContext وتكوينه من أجل اختبارات التكامل. هذا هو التعليق الذي يجب أن يكون لديك لفصولك ، لأن هذا هو المكان الذي يقوم ApplicationContext بتحميل جميع تعريفات المكونات.
  • الملخصWebAppConfiguration. تعليق توضيحي على مستوى الفصل يشير إلى أن سياق التطبيق الذي تم تحميله لاختبار التكامل يجب أن يكون WebApplicationContext.
  • تعليق توضيحيActiveProfile. تعليق توضيحي على مستوى الفصل يشير إلى أي من ملفات تعريف تعريف المكون التي يجب أن تكون نشطة عند تحميل ApplicationContext لاختبار التكامل.
  • الخلاصةTestPropertySource. تعليق توضيحي على مستوى الفئة مصمم لتحديد مواقع ملفات الخصائص والخصائص المضمنة التي تمت إضافتها إلى كائن PropertySource المعيّن في البيئة لـ ApplicationContext الذي تم تحميله لاختبار التكامل.
  • الخلاصة @ DirtiesContext. يشير إلى أن ApplicationContext المستخدم كان "متسخًا" أثناء الاختبار (على سبيل المثال ، تم تعديله أو تالف عن طريق تغيير حالة مكون واحد) ويجب إغلاقه.

هذه ليست كل التعليقات التوضيحية ؛ فهناك العديد من التعليقات الأخرى (TestExecutionListeners ، ارتكب، التراجع،BeforeTransaction ،AfterTransaction ، مزود، مزودمجموعة مزودالتكوين موقوت، كرر،IfProfileValue ، وما إلى ذلك).

كما ترى ، هناك العديد من خيارات الاختبار مع إطار عمل الربيع. عادة ما يتم استخدام التعليق التوضيحيRunWith لربط جميع عناصر إطار الاختبار معًا. على سبيل المثال:

@RunWith(SpringRunner.class)
@ContextConfiguration({"/app-config.xml", "/test-data-access-config.xml"})
@ActiveProfiles("dev")
@Transactional
public class ToDoRepositoryTests {

        @Test
        public void ToDoPersistenceTest(){
                //...
        }
}

الآن دعونا نرى كيفية استخدام Spring Testing Framework والميزات التي يوفرها Spring Boot.

إطار اختبار التمهيد الربيعي


يعمل Spring Boot على الاستفادة من قوة Spring Testing Framework ، والتوسع في القديم وإضافة التعليقات التوضيحية والميزات الجديدة التي تجعل الاختبار أسهل للمطورين.

لبدء استخدام جميع ميزات اختبار Boot Spring ، تحتاج فقط إلى إضافة تبعية اختبار Spring-boot-starter-test إلى التطبيق باستخدام نطاق الاختبار (اختبار النطاق). لقد أضافت خدمة Spring Initializr بالفعل هذه التبعية.

يوفر الاعتماد على اختبار بدء التشغيل الربيعي القدرة على استخدام العديد من أطر الاختبار التي تتوافق تمامًا مع قدرات اختبار Boot Spring: JUnit و AssertJ و Hamcrest و Mockito و JSONassert و JsonPath. بالطبع ، هناك أطر اختبار أخرى تعمل بشكل جيد مع وحدة اختبار التمهيد الربيعي ؛ يجب تحديد التبعيات المقابلة يدويًا فقط.

يوفر Spring Boot التعليق التوضيحيSpringBootTest لتبسيط اختبار تطبيقات Spring. عادة ، عند اختبار تطبيق Spring ، تحتاج إلى إضافة بعض التعليقات التوضيحية لاختبار ميزة أو وظيفة تطبيق معينة ، ولكن ليس في Spring Boot - على الرغم من أنه للاختبار ما زلت بحاجة إلى تحديد التعليق التوضيحيRunWith (SpringRunner.class) ؛ إذا لم يتم ذلك ، فسيتم تجاهل أي تعليقات توضيحية لاختبار Spring Boot جديدة. يحتوي التعليق التوضيحيSpringBootTest على خيارات مفيدة لاختبار تطبيقات الويب ، مثل RANDOM_PORT و DEFINED_PORT.

مقتطف الشفرة التالي هو إطار اختبار Boot Spring.

@RunWith(SpringRunner.class)
@SpringBootTest
public class MyTests {

        @Test

                 ...
        }

اختبار نقطة نهاية تطبيق الويب


يوفر Spring Boot طريقة مناسبة لاختبار نقاط النهاية: بيئة محاكاة تسمى فئة MockMvc:

@RunWith(SpringRunner.class)
@SpringBootTest
@AutoConfigureMockMvc
public class MockMvcToDoTests {

        @Autowired
        private MockMvc mvc;

        @Test
        public void toDoTest() throws Exception {
              this.mvc
              .perform(get("/todo"))
              .andExpect(status().isOk())
              .andExpect(content()
                   .contentType(MediaType.APPLICATION_JSON_UTF8));
        }
}

يمكنك أيضًا استخدام فئة TestRestTemplate.

@RunWith(SpringRunner.class)
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
public class ToDoSimpleRestTemplateTests {

      @Autowired
      private TestRestTemplate restTemplate;

      @Test
      public void toDoTest() {
            String body = this.restTemplate.getForObject("/todo", String.class);
            assertThat(body).contains("Read a Book");
      }
}

يعرض هذا الرمز اختبارًا يبدأ خادمًا كاملاً ويستخدم مثيلًا من فئة TestRestTemplate للوصول إلى نقطة نهاية / todo. هنا نفترض أن كائن String تم إرجاعه (هذه ليست أفضل طريقة لاختبار عودة JSON ؛ لا تقلق ، سنوضح كيفية استخدام فئة TestRestTemplate بشكل صحيح).

تقليد مكون


توفر وحدة اختبار Boot Boot تعليقًا توضيحيًا علىMockBean يصف كائن محاكاة Mockito لمكون في ApplicationContext. بمعنى آخر ، يمكنك إنشاء تقليد لمكون ربيع جديد أو استبدال تعريف موجود بإضافة هذا التعليق التوضيحي. تذكر: كل هذا يحدث داخل ApplicationContext.

@RunWith(SpringRunner.class)
@SpringBootTest
public class ToDoSimpleMockBeanTests {

        @MockBean
        private ToDoRepository repository;

        @Test
        public void toDoTest() {
              given(this.repository.findById("my-id"))
                    .Return(new ToDo("Read a Book"));
              assertThat(
                    this.repository.findById("my-id").getDescription())
                    .isEqualTo("Read a Book");
        }
}

شرائح اختبار التمهيد الربيع


واحدة من أهم ميزات Spring Boot هي إجراء الاختبارات دون الحاجة إلى أي بنية أساسية محددة. تتضمن وحدة اختبار Boot Spring ما يسمى بالشرائح المصممة لاختبار أجزاء معينة من التطبيق دون استخدام خادم أو DBMS.

تعليق توضيحيJsonTest


تحتوي وحدة اختبار Boot Boot على تعليق توضيحيJsonTest يبسط تسلسل / إلغاء تسلسل كائنات JSON والتحقق مما إذا كان كل شيء يعمل بشكل صحيح. يقومJsonTest بتكوين أداة عرض JSON المدعومة تلقائيًا ، اعتمادًا على المكتبة الموجودة في مسار الفصل الدراسي: جاكسون أو GSON أو JSONB.

@RunWith(SpringRunner.class)
@JsonTest
public class ToDoJsonTests {

      @Autowired
      private JacksonTester<ToDo> json;

      @Test
      public void toDoSerializeTest() throws Exception {
            ToDo toDo = new ToDo("Read a Book");

            assertThat(this.json.write(toDo))
            .isEqualToJson("todo.json");
            assertThat(this.json.write(toDo))
            .hasJsonPathStringValue("@.description");

            assertThat(this.json.write(toDo))
            .extractingJsonPathStringValue("@.description")
            .isEqualTo("Read a Book");
      }

      @Test
      public void toDoDeserializeTest() throws Exception {
            String content = "{\"description\":\"Read a Book\",\"completed\":
                  true }";
            assertThat(this.json.parse(content))
                  .isEqualTo(new ToDo("Read a Book", true));
            assertThat(
               this.json.parseObject(content).getDescription())
            .isEqualTo("Read a Book");
      }
}

لاختبار وحدات التحكم بدون استخدام خادم كامل ، يمكنك استخدام التعليق التوضيحي المقترح من Boot BootWebMvcTest ، والذي يقوم تلقائيًا بتكوين البنية الأساسية لـ Spring MVC ويقيد قائمة المكونات التي تم عرضها بما يلي: مراقب، مراقبنصيحة ،JsonComponent ، محول ، GenericConverter ، مرشح ، WebMvcConfigurer و HandlerMethodArgumentResolver ؛ بفضل هذا ، ستعرف ما إذا كانت وحدات التحكم تعمل كما هو متوقع.

من المهم أن نفهم أن تلك التي تم وضع علامة عليهامكون المكونات غير مرئية عند استخدام هذا التعليق التوضيحي ، ولكن يمكنك استخدام التعليق التوضيحيMockBean إذا لزم الأمر.

@RunWith(SpringRunner.class)
@WebMvcTest(ToDoController.class)
public class ToDoWebMvcTest {

      @Autowired
      private MockMvc mvc;

      @MockBean
      private ToDoRepository toDoRepository;

      @Test
      public void toDoControllerTest() throws Exception {
            given(this.toDoRepository.findById("my-id"))
                  .Return(new ToDo("Do Homework", true));
                  this.mvc.perform(get("/todo/my-id").accept(MediaType.APPLICATION_
                     JSON_UTF8))
                        .andExpect(status().isOk()).andExpect(content().
                           string("{\"id\":\"my-id\",\"description\":\"Do Homework\",
                           \"completed\":true}"));
      }
}

التعليق التوضيحيWebFluxTest


بالنسبة إلى وحدات التحكم التفاعلية ، يوفر Spring Boot التعليق التوضيحيWebFluxTest. يعمل هذا التعليق التوضيحي تلقائيًا على تهيئة البنية الأساسية لوحدة Spring WebFlux والبحث فقطمراقب، مراقبنصيحة ،JsonComponent ، محول ، GenericConverter و WebFluxConfigurer.

من المهم أن نفهم أن تلك التي تم وضع علامة عليهامكون المكونات غير مرئية عند استخدام هذا التعليق التوضيحي ، ولكن يمكنك استخدام التعليق التوضيحيMockBean إذا لزم الأمر.

@RunWith(SpringRunner.class)

تعليق توضيحيDataJpaTest


لاختبار تطبيقات JPA ، توفر وحدة اختبار Spring Boot التعليقات التوضيحيةDataJpaTest ، والتي تقوم تلقائيًا بتكوين قواعد البيانات المضمنة الموجودة في ذاكرة الوصول العشوائي. إنها تبحثشخصية ولا يتم تحميل أي مكونات مكون. بالإضافة إلى ذلك ، فإنه يوفر فئة المساعد TestEntityManager الموجهة للاختبار ، وهي مشابهة جدًا لفئة JPA EntityManager.

@RunWith(SpringRunner.class)
@DataJpaTest
public class TodoDataJpaTests {
  
      @Autowired
      private TestEntityManager entityManager;

      @Autowired
      private ToDoRepository repository;

      @Test
      public void toDoDataTest() throws Exception {
            this.entityManager.persist(new ToDo("Read a Book"));
            Iterable<ToDo> toDos = this.repository.
                   findByDescriptionContains("Read a Book");
            assertThat(toDos.iterator().next()).toString().contains("Read a Book");
      }
}

يرجى ملاحظة أنه عند الاختبار باستخدامDataJpaTest ، يتم استخدام DBMSs المضمنة في ذاكرة الوصول العشوائي. للاختبار باستخدام قاعدة بيانات حقيقية ، يجب عليك تزويد فئة الاختبار بالتعليق التوضيحيAutoConfigureTestDatabase (replace = Replace.NONE).

تعليق توضيحيJdbcTest


هذا التعليق التوضيحي مشابه جدًا لـDataJpaTest ؛ والفرق الوحيد هو أنه يعمل حصريًا في اختبارات JDBC. يقوم تلقائيًا بتكوين نظام DBMS المدمج الموجود في ذاكرة الوصول العشوائي وفئة JdbcTemplate ، مع تخطي جميع التعليقات التوضيحيةمكون الطبقات.

@RunWith(SpringRunner.class)
@JdbcTest
@Transactional(propagation = Propagation.NOT_SUPPORTED)
public class TodoJdbcTests {

      @Autowired
      private NamedParameterJdbcTemplate jdbcTemplate;

      private CommonRepository<ToDo> repository;

        @Test
        public void toDoJdbcTest() {
              ToDo toDo = new ToDo("Read a Book");

            this.repository = new ToDoRepository(jdbcTemplate);
            this.repository.save(toDo);

            ToDo result = this.repository.findById(toDo.getId());
            assertThat(result.getId()).isEqualTo(toDo.getId());
        }
}


تعليق توضيحيDataMongoTest


لاختبار تطبيقات MongoDB ، توفر وحدة اختبار Boot Boot التعليقات التوضيحية لـ @ DataMongoTest. يقوم تلقائيًا بتكوين خادم Mongo المدمج القائم على الذاكرة ، إذا كان متاحًا ؛ إذا لم يكن الأمر كذلك ، فأنت بحاجة إلى إضافة الخصائص الضرورية لـ spring.data.mongodb. *. كما أنها تضبط فئة MongoTemplate وتبحث عن التعليقات التوضيحية للوثائق @. مكوناتمكون تم تخطيها.

@RunWith(SpringRunner.class)
@DataMongoTest
public class TodoMongoTests {

      @Autowired
      private MongoTemplate mongoTemplate;

      @Test
      public void toDoMongoTest() {
          ToDo toDo = new ToDo("Read a Book");
          this.mongoTemplate.save(toDo);

          ToDo result = this.mongoTemplate.findById(toDo.getId(),ToDo.class);
          assertThat(result.getId()).isEqualTo(toDo.getId());
      }
}

إذا كنت بحاجة إلى خادم MongoDB خارجي (غير مضمن وداخل الذاكرة) ، فأضف ExclusionAutoConfiguration = EmbeddedMongoAutoConfiguration.class إلى المعلمةDataMongoTest.

@RunWith(SpringRunner.class)
@DataMongoTest(excludeAutoConfiguration = EmbeddedMongoAutoConfiguration.class)
public class ToDoMongoTests {
        // ...
}

تعليق توضيحيRestClientTest


تعليق توضيحي مهم آخر هوRestClientTest ، المصمم لاختبار عملاء REST. يعمل هذا التعليق التوضيحي تلقائيًا على ضبط الإعدادات لدعم Jackson و GSON و JSONB ، كما يقوم بإعداد فئة RestTemplateBuilder ويضيف دعمًا لـ MockRestServiceServer.

@RunWith(SpringRunner.class)
@RestClientTest(ToDoService.class)
public class ToDoRestClientTests {

      @Autowired
      private ToDoService service;

      @Autowired
      private MockRestServiceServer server;

      @Test
      public void toDoRestClientTest()
                  throws Exception {
             String content = "{\"description\":\"Read a Book\",\"completed\":
                   true }";
            this.server.expect(requestTo("/todo/my-id"))
             .andRespond(withSuccess(content,MediaType.APPLICATION_JSON_UTF8));
            ToDo result = this.service.findById("my-id");
            assertThat(result).isNotNull();
            assertThat(result.getDescription()).contains("Read a Book");
      }
}

هناك العديد من الشرائح الأخرى المتاحة للاستخدام. الشيء الرئيسي الذي يجب تذكره: للاختبار ، البنية التحتية الكاملة مع جميع الخوادم قيد التشغيل اختيارية. تعمل الشرائح على تسهيل اختبار تطبيقات Boot Spring.
