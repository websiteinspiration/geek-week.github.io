レイヤー1.ツイストXOR


アンパッカーの重要な部分の前には、無用な指示がたくさんあります。ウイルス作成者は、アンチウイルス製品のエミュレーターを混乱させるために、このような手法に頼ることがよくあります。すべての興味深いことは、WinAPI関数VirtualAllocExを使用して、サイズ0xD20のバッファにメモリを割り当てることから始まります。メモリはPAGE_EXECUTE_READWRITE権限で割り当てられます。これにより、コードを記述して実行できます。



アンパッカーのコンテンツ部分の始まり

調査中のファイルのデータセクションには、解読手順を通過するデータを含む配列が含まれており、結果は割り当てられたメモリに書き込まれます。復号化アルゴリズム：

  • 4バイトを整数として解釈し、
  • シリアル番号を引く
  • 与えられた定数でXOR演算を実行し、
  • 左に7ユニットずつ循環ビット単位シフトを実行します。
  • 与えられた定数でXOR演算を再度実行します。



最初のレイヤーの復号化
このアルゴリズムを省略形SUB-XOR-ROL7-XORで指定して、後で参照できるようにします。
復号化手順の後、変数は順次初期化されます。これは、Cの構造体を次の形式で埋めることで表すことができます。

struct ZOZ {
	HMODULE hkernel32;
	void *aEncodedBlob;
	unsigned int nEncodedBlobSize;
	unsigned int nBlobMagic;
	unsigned int nBlobSize;
};

どこ：

  • hkernel32-ライブラリ記述子kernel32.dll;
  • aEncodedBlob-エンコードされたデータブロックへのポインター。上記のサンプルの視覚的な類似性を参照するときに言及しました。



エンコードされたデータブロック

  • nEncodedBlobSize-エンコードされたデータブロックの4バイトのサイズ。
  • nBlobMagic-データブロックの前にある4バイトの定数。後で戻ります。
  • nBlobSize-デコードされたデータブロックの4バイトのサイズ。

レイヤー2.小さいほど良い


復号化されたコードに制御を移した後の最初のステップは、WinAPI関数GetProcAddress、VirtualQuery、VirtualAlloc、VirtualProtect、VirtualFree、およびLoadLibraryAのアドレスを判別することです。このリストは、シェルコードの作業でよく見られます。結局のところ、それらは後続のペイロードの起動のためにメモリを準備して正しく埋める必要があります。

準備手順が完了すると、送信されたエンコードデータブロックは切り捨てられます。最初の2バイトが破棄され、残りの3バイトが保存



されます。エンコードデータブロックの削減

次に、SUB-XOR-ROL7-XORと呼ばれる復号手順が実行されます。この場合、XOR操作の定数として、構造体で渡された別のnBlobMagic値が使用されますゾズ。



SUB-XOR-ROL7-XORアルゴリズムの再利用

その後、結果の配列は、一連のより複雑な変換が行われる関数に転送されます。定数の特性値によって、これが人気のあるPEラッパーFSG（Fast Small Good）の実装であることを確立するのは簡単です。 FSGパッカーの元のバージョンがPEをセクションに圧縮し、この場合のアルゴリズムの実装が全体としてPEで機能するのは奇妙です。



FSG Packerの実装

この段階で、さらに分析する準備ができている、解凍されたPEターゲットファイルがメモリにあります。シェルコードの残りの部分は、アドレス空間の元のPEを解凍されたバージョンで上書きし、正しく実行します。ロードされたモジュールのエントリポイントを調整するプロセスでは、構造がPEBで操作されることに注意してください。同じPEB構造を使用して取得するのではなく、最初のレイヤーのロジックからkernel32記述子を転送することにした理由は不明です。

結論


したがって、ペイロード解凍アルゴリズム：

  • SUB-XOR-ROL7-XORを使用したシェルコードのデコード
  • ZOZ構造を埋め、シェルコードを呼び出す、
  • 5から3のペイロード削減、
  • SUB-XOR-ROL7-XORを使用したペイロードのデコード、
  • 減圧パッカーFSG。

マルウェアの進化の間に、ロジックが変更されました。たとえば、SUB-XOR-ROL7-XORアルゴリズムの循環シフトの数（検討中のケースでは7つ）が5、9に変更され、x64プラットフォームのパッカーバージョンがリリースされました。ユニークなサイバー犯罪グループ化パッカーは優れていますTA505のツールと機能に関する将来のナレーションへのエピグラフ。
