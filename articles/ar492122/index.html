الترحيل من Gitolite إلى GitLab باستخدام برنامج نصي

غالبًا ما تكون عملية الترحيل مهمة صعبة ، خاصة عندما تكون كمية المعلومات التي يجب نقلها كبيرة جدًا بحيث تصبح أكثر ربحية لأتمتة هذه المعلومات. كانت الحاجة إلى الهجرة من Gitolite إلى GitLab هي التي دفعتني إلى كتابة مقال حول تجربتي في هذا الأمر.


لترحيل المستودعات ، سأستخدم جهاز كمبيوتر مثبت عليه نظام التشغيل CentOS 7. وأحتاج إلى تثبيت مجموعة التطبيقات التالية: git و ssh-agent و curl و jq و xargs .


للبدء ، نحتاج إلى الحصول على مفتاح الوصول إلى GitLab API. في واجهة الويب ، انتقل إلى قسم إعدادات المستخدم. بعد ذلك ، حدد عنصر القائمة "رموز الوصول" . في النموذج الذي يفتح ، يجب عليك تحديد اسم المفتاح المستلم ، ويمكنك أيضًا تحديد الفترة التي سيتم خلالها تنشيط المفتاح. أدناه ، حدد المربع بجوار عنصر واجهة برمجة التطبيقات وانقر على الزر "إنشاء رمز دخول شخصي" . سيتم استخدام هذا المفتاح عند إرسال الطلبات إلى GitLab API.

    # $1 - gitlab remote url
# $2 - api access token
# $3 - remote repository name
# $4 - remote repository namespace identifier
initRemoteRepository() {
    local response=`curl -s -k -X POST --header "Private-Token: $2" --data "name=$3&namespace_id=$4&visibility=private&description=$3" $1/api/v4/projects`
    local id=`echo $response | jq -r ".id"`
    echo "$id"
}

getCanonicalBranchName() {
    local branch=""
    IFS=$2 read -r -a array <<< "$1"
    length=${#array[@]}
    for index in "${!array[@]}"
