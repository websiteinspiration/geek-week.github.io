視覚要素の分離


FXMLに関する 前回の記事では、UIコードを2つに分割することにより、JavaFXがタスクを明確に分離する方法を学びました。コンポーネントとそのプロパティはFXMLファイルで宣言され、相互作用ロジックはコントローラーに明確に割り当てられます。

さらに、3番目の部分であるFXML言語があります。FXML言語は、アプリケーションのコンポーネント、それらのプロパティ、およびそれらが相互に埋め込まれる方法のみを制御します。つまり、フォント、色、背景、インデントなど、コンポーネントの視覚要素を定義しません。一般に、これはFXMLで実現できますが、実現しないでください。代わりに、視覚要素はCSSスタイルシートで明確に定義する必要があります。

したがって、設計は独立し、残りのアプリケーションに影響を与えることなく簡単に交換または変更できます。ユーザーの要求に応じて切り替えることができるいくつかのテーマを実装することもできます。

CSS


Web上のHTMLページのスタイルを設定するために使用されるCSS（Cascading Style Sheets）に慣れていることでしょう。JavaFXには同様のアプローチが実装されていますが、JavaFXは独自のカスタムプロパティのセットを使用します。

例を見てみましょう：

.button {
    -fx-font-size: 15px;
}

ここでは、2つの主要な概念が使用されています。1つ目は.buttonセレクターです。スタイルを適用するコンポーネントを決定します。この例では、スタイルはすべてのボタンに適用されます。

2番目の部分はスタイルの実際のプロパティであり、セレクターに一致するすべてのコンポーネントに適用されます。プロパティはすべて中括弧内にあります。

各プロパティには特定の意味があります。この例では、テキストのサイズを決定する-fx-font-sizeプロパティがあります。この例では、値は15pxですが、この値は他の値にすることができます。

要約すると、すべてのボタンのテキストサイズを15ピクセルにする必要があることを示すルールを作成しました。

セレクター


次に、JavaFXでセレクターがどのように機能するかを詳しく見てみましょう。これは通常のCSSとほぼ同じです。

クラス


CSSのクラスはいくつかの類似した要素を表します。たとえば、ボタンやチェックボックスです。同じクラスのすべての要素に適用されるセレクターは、ドット「。」で始まり、直後にクラスの名前が続きます。クラスの命名規則では、個々の単語を「-」文字で区切ります。次のセレクターは、ラベルクラスを持つすべての要素に適用されます。

.label {
    // Some properties
}

組み込みクラス


良い知らせは、すべてのJavaFX組み込みコンポーネント（LabelやButtonなど）には、事前定義されたクラスがすでにあることです。アプリケーションのすべてのラベルのスタイルをカスタマイズする場合は、ラベルごとにカスタムクラスを追加する必要はありません。各ラベルには、デフォルトでラベルクラスがあります。

コンポーネントからクラス名を決定するのは簡単です：

  • たとえば、Javaコンポーネントクラスの名前を取ります。ラベル
  • 名前を小文字にする
  • 複数の単語で構成される場合は、「-」記号で区切ります

いくつかの例：

  • ラベル→ラベル
  • チェックボックス→チェックボックス

セレクタなどのクラスを使用する場合は、必ず「。」を追加してください。つまり、ラベルクラスのセレクターは.labelです。

カスタムクラス


組み込みクラスでは不十分な場合は、独自のクラスをコンポーネントに追加できます。コンマで区切られたいくつかのクラスを使用できます。

<Label styleClass="my-label,other-class">I am a simple label</Label>

またはJavaで：

Label label = new Label("I am a simple label");
label.getStyleClass().addAll("my-label", "other-class");

この方法でクラスを追加しても、デフォルトではコンポーネントクラス（この場合はlabel）は削除されません。root

と呼ばれる特別なクラスが1つあります。これは、シーンのルートコンポーネントです。これを使用して、シーン内のすべてのスタイルを設定できます（たとえば、グローバルフォントを設定します）。これは、HTMLでbodyタグセレクタを使用するのと似ています。

ID


CSSでコンポーネントを選択する別の方法は、コンポーネント識別子（ID）を使用することです。これは、コンポーネントの一意の識別子です。複数のコンポーネントに割り当てることができるクラスとは異なり、識別子はシーン内で一意である必要があります。

一方、「。」という記号はクラスを示すために使用されます。セレクターの名前の前には、識別子に記号「＃」が付いています。

#my-component {
  ...
}

FXMLでは、fx：idを使用してコンポーネントのCSS識別子を設定できます。

<Label fx:id="foo">I am a simple label</Label>

ただし、注意点が1つあります。同じ識別子は、同じ名前のコントローラーで宣言されたコンポーネントオブジェクトを参照するために使用されます。コントローラーのIDとフィールド名は一致する必要があるため、fx：idはフィールド名のJava命名制限を考慮する必要があります。CSSの命名規則では「-」文字で区切られた個々の単語を定義していますが、これはJavaフィールド名には無効な文字です。したがって、fx：idにいくつかの単語が含まれる場合は、CamelCaseなどの別の命名規則を使用するか、アンダースコアを使用する必要があります。

<!--  This is not valid  -->
<Label fx:id="my-label">I am a simple label</Label>
<!--  This is valid  -->
<Label fx:id="my_label">I am a simple label</Label>
<Label fx:id="MyLabel">I am a simple label</Label>

Javaでは、コンポーネントのsetId（）メソッドを呼び出すだけです。

Label label = new Label("I am a simple label");
label.setId("foo");

プロパティ


JavaFXで使用されるCSSは元のWeb CSSと非常に似ていますが、大きな違いが1つあります。プロパティ名は異なり、多くの新しいJavaFX固有のプロパティがあります。プレフィックスは-fx-です。

ここではいくつかの例を示します。

  • -fx-background-color：背景色
  • -fx-text-fill：テキストの色
  • -fx-font-size：テキストサイズ

すべてのプロパティのリストは、公式設計ガイドに記載されています。

疑似クラス


特定のコンポーネントをマークする通常のクラスに加えて、コンポーネントの状態を示すいわゆる疑似クラスがあります。これは、たとえば、コンポーネントにフォーカスがあること、またはマウスカーソルがそのコンポーネント上にあることをマークするためのクラスである可能性があります。

多くの組み込み疑似クラスがあります。ボタンを見てみましょう。使用できるいくつかの疑似クラスがあります。次に例を示します。

  • ホバー：マウスオーバーボタン
  • フォーカス：ボタンにフォーカスがあります
  • 無効：ボタンは無効です
  • 押された：ボタンが押された

疑似クラスは、CSSセレクターで「：」文字（たとえば、:: hover）で始まります。もちろん、疑似クラスが属しているコンポーネントを示す必要があります-たとえば、button：hover。次の例は、フォーカスを持つすべてのボタンに適用されるセレクターを示しています。

.button:focused {
    -fx-background-color: red;
}

focusやhover などの状態の基本的な疑似クラスしか持たないCSSとは異なり、JavaFXには、コンポーネントのさまざまな状態やプロパティに関連するコンポーネント固有の疑似クラスがあります。

例えば：

  • スクロールバーには水平および垂直の擬似クラスがあります
  • 要素（セル）擬似クラス有する奇数と偶数
  • TitledPaneは疑似クラスを展開および縮小しました

カスタム疑似クラス


組み込みの疑似クラスに加えて、独自の疑似クラスを定義して使用できます。

独自のラベルを作成しましょう（Labelクラスから継承）。これには、shinyという新しい論理プロパティがあります。この場合、ラベルに光沢のある疑似クラスを設定します。

タグには光沢のある疑似クラスがあるため、金タグの背景を設定できます。

.shiny-label:shiny {
    -fx-background-color: gold;
}

次に、クラス自体を作成します。

public class ShinyLabel extends Label {
    private BooleanProperty shiny;

    public ShinyLabel() {
        getStyleClass().add("shiny-label");

        shiny = new SimpleBooleanProperty(false);
        shiny.addListener(e -> {
            pseudoClassStateChanged(PseudoClass.getPseudoClass("shiny"), shiny.get());
        });
    }

    public boolean isShiny() {
        return shiny.get();
    }

    public void setShiny(boolean shiny) {
        this.shiny.set(shiny);
    }
}

ここにはいくつかの重要な部分があります：

  1. BooleanProperty boolean. , shiny observable (), () .
  2. listener (), , shiny shiny.addListener().
  3. shiny , / shiny pseudoClassStateChanged(PseudoClass.getPseudoClass(«shiny»), shiny.get()).
  4. shiny-label, label, . , shiny.


あなた自身がスタイルを提供していなくても、各JavaFXアプリケーションにはすでにいくつかの視覚スタイルがあります。各アプリケーションに適用されるデフォルトのスタイルシートがあります。これはモデナと呼ばれます（JavaFX 8以降、以前はcaspianと呼ばれていました）。

このスタイルシートは、ファイル内で見つけることができます：

jfxrt.jar \ COM \日\ JavaFXの\シーン\制御\皮膚\モデナ\ modena.css

またはファイルを見つけることができるここに。同じディレクトリに、スタイルシートで使用される多くの画像があります。

このスタイルシートはデフォルトのスタイルを提供しますが、他のタイプのスタイルシートよりも優先度が最も低いため、簡単にオーバーライドできます。

シーンスタイルシート


上記のデフォルトのスタイルシートに加えて、もちろん独自のスタイルシートを提供できます。スタイル設定を適用できる最高レベルは、シーン全体です。これをFXMLに実装できます。

<BorderPane xmlns="http://javafx.com/javafx"
            xmlns:fx="http://javafx.com/fxml"
            stylesheets="styles.css"
            ...
            >
  ...
</BorderPane>

またはあなたのJavaコードで：

String stylesheet = getClass().getResource("/styles.css").toExternalForm();
scene.getStylesheets().add(stylesheet);

toExternalForm（） の呼び出しに注意してください。シーンはスタイルシートのコンテンツをファイルではなく文字列として取得することを想定しているため、スタイルシートのコンテンツを文字列として提供する必要があります。

親スタイルシート


シーン全体のスタイルシートに加えて、レイアウトレベルでスタイルを設定すると便利な場合があります。つまり、VBox、HBox、GridPaneなどの個別のコンテナの場合です。すべてのレイアウトの共通の親は親クラスであり、レイアウトレベルでスタイルシートを処理するためのメソッドを定義します。これらのスタイルは、このレイアウトのコンポーネントにのみ適用され、シーン全体には適用されません。レイアウトレベルのスタイルは、シーンレベルのスタイルよりも優先されます。

<HBox stylesheets="styles.css">
    ...
</HBox>

Javaでは、以前と同じように、自分でスタイルシートのコンテンツを自分でロードする必要があります。

HBox box = new HBox();
String stylesheet = getClass().getResource("/styles.css").toExternalForm();
box.getStylesheets().add(stylesheet);

インラインスタイル


これまでは、外部スタイルシートがシーン全体またはレイアウト全体に割り当てられている場合のみを見てきました。ただし、コンポーネントレベルで個々のスタイルプロパティを設定できます。

ここでは、すべてのプロパティが特定のコンポーネントに設定されているため、セレクターについて心配する必要はありません。

セミコロンで区切られたいくつかのプロパティを指定できます。

<Label style="-fx-background-color: blue; -fx-text-fill: white">
  I'm feeling blue.
</Label>

Javaでは、setStyle（）メソッドを使用できます。

Label label = new Label("I'm feeling blue.");
label.setStyle("-fx-background-color: blue; -fx-text-fill: white");

コンポーネントレベルのスタイルは、レイアウトレベルのシーンスタイルや親スタイルよりも優先されます。

それらを避ける必要がある理由


コンポーネントレベルのスタイリングは便利な場合がありますが、すばやく簡単な方法です。CSSの主な利点は、スタイルとコンポーネントを分離することです。次に、視覚要素をコンポーネントに直接厳密にバインドします。必要に応じてスタイルシートを簡単に切り替えることができなくなり、テーマを変更することはできません。

さらに、スタイルが定義される単一の中心的な場所がなくなります。類似したコンポーネントのセットで何かを変更する必要がある場合は、外部スタイルシートの1つの場所だけを編集するのではなく、各コンポーネントを個別に変更する必要があります。したがって、インラインコンポーネントスタイルは避けてください。

スタイルシートの優先順位


シーン、親、インラインスタイルなど、いくつかのレベルでスタイルを提供でき、デフォルトのモデムスタイルシートもあります。同じコンポーネントの同じプロパティを複数のレベルで変更する場合、JavaFXには、使用するスタイルを決定する優先順位設定があります。優先リスト-最高から最低：

  1. インラインスタイル
  2. 親のスタイル
  3. シーンスタイル
  4. デフォルトのスタイル

つまり、インラインレベルとシーンレベルの両方で特定のラベルの背景色を設定した場合、JavaFXは優先度が高いため、インラインスタイルで設定された値を使用します。
