Angular CLIでChrome拡張機能を開発する

私は長い間Chrome用の拡張機能を開発しており、その間、Greasemonkeyのユーザースクリプトから、Chrome拡張シェルで本格的なAngularアプリケーションに至るまでのすべてを行ってきました。私の仕事は、機能しているサイトの一部のプロセスを自動化するために、機能しているサイトにパッチを適用することです。アプリケーションが大規模になり、プレーンなjs拡張をサポートすることが困難になる場合があります（アプリケーションの設定、CRUD機能などが多数あります）。そして、Angularが助けになりました。

この記事では、Chrome拡張機能とAngular CLIを使って友達を作り、開発プロセスを設定した方法と、遭遇した問題とその解決方法について説明します。
新しいフォルダを作成し、その中で新しいアプリケーションを初期化します。

mkdir new-project
cd new-project
ng new frontend --routing=true --skipGit=true --style=scss --skipTests=true

開発段階では、Angularは開発が行われる動的なhtmlファイルを生成します。作業の結果を確認するには、Chrome拡張機能に静的なhtmlファイルを供給する必要があります。もちろん、Angulyarovskプロジェクトを個別にアセンブルしてから、ビルドバージョンを追加登録することもできますが、これが自動的に行われると便利です。

新しいアプリケーションを生成したら、フロントエンドフォルダーに移動し、スクリプトセクションのpackage.jsonファイルに新しいスクリプトを追加してプロジェクトをビルドします。

"developing": "ng build --watch --deploy-url /frontend/dist/frontend/ --base-href /frontend/dist/frontend/index.html?/"

とに注意しdeploy-urlてくださいbase-href。

次に、プロジェクトのルートで、別の拡張フォルダーを作成し、その中にextension.jsファイルを作成します。これが拡張のバックグラウンドスクリプトになります。現在のプロジェクト構造：

| new-project/
| | extension/
| | | extension.js
| | frontend/
| | | ...

コンテンツextension.js

const ANGULAR_HTML_URL = "../../frontend/dist/frontend/index.html";

chrome.browserAction.onClicked.addListener(function () {
    chrome.tabs.create({
        url: chrome.runtime.getURL(ANGULAR_HTML_URL)
    });
});

これはbrowserActionであり、角度のあるアプリケーションで新しいタブを開きます。これは、組み立てられた形式でこのパスに沿ってすでに保存されています。

プロジェクトにmanifest.jsonを追加します
{
  "manifest_version": 2,
  "name": "Simple Chrome Ext",
  "description": "Simple Chrome Extension as an example",
  "version": "1.00",
  "author": "Bogdan",
  "content_security_policy": "script-src 'self' 'unsafe-eval'; object-src 'self'",
  "background": {
    "persistent": true,
    "scripts": ["/extension/extension.js"]
  },
  "browser_action": {
    "default_title": "Chrome ext"
  }
}


これで、プロジェクトを拡張機能にアセンブルできます。私たちは、に行く/ //拡張子：クロム、開発者モードと選択をオンにダウンロード、解凍拡張。

結果

便利なのは、ほとんどホットリロードできることです。監視フラグを使用してngビルドを起動しました。コードが変更されると、プロジェクトが再構築され、jsファイルが置き換えられます。つまり、拡張ページでは、ページをリロードするだけですべての更新が取得されます。これにより、アドオンの開発が大幅にスピードアップします。 私が犬を食べたもう1つのポイントは、ルーティング構成におけるものです。



{
   useHash: true
}

/frontend/dist/frontend/index.html?/として登録したビルド でbase-hrefを返します -最後のスラッシュの前の疑問符に注意してください。実際のところ、一部のシステムでは、Angularアプリケーションのリンクをクリックすると、この記号がないとすべてがクラッシュします。どうやら、Chromeエンジンの静的Webサーバーは、URLの変更を別のファイルへのリクエストとして認識し（useHash：trueであっても）、ページを更新すると404エラーを返しました。この組み合わせでのみ、すべてのシステムで安定した動作を達成することができました。アプリケーション

のpackage.jsonに戻り、別のスクリプトを追加しましょう

"prod": "ng build --sourceMap false --prod true --deploy-url /frontend/dist/frontend/ --base-href /frontend/dist/frontend/index.html?/"

これは、製品版のアプリケーションを構築するためのスクリプトになります。

Chromeウェブストアで、拡張子が付いたzipアーカイブが投稿され、アセンブリを簡略化するために、これらの目的のためにスクリプトを作成しました

update.sh
#!/bin/bash

rm -rf ./prod-build
mkdir -p prod-build/frontend

cd frontend
npm run prod
cd ..

cp -R ./frontend/dist ./prod-build/frontend
cp -R ./extension ./prod-build
cp ./manifest.json ./prod-build

zip -r prod-build{.zip,}
rm -rf ./prod-build

ソースコードはここにあります。

合計：Angular Chrome Extensionの事前開発キットを作成し、適切なルーティングとアプリケーションの便利な更新/ビルドを行いました。

PS：そのような/フロントエンド/ dist /フロントエンド/パスを取り除くために、環境を構成し、本番アセンブリでパスを別の方法で指定できますが、これは重要な点ではありません。
