1.フラグメントでのLiveDataオブザーバーの漏洩


フラグメントには複雑なライフサイクルがあり、フラグメントが切断されてアクティビティに再び参加しても、常に破棄されるわけではありません。たとえば、保存されたフラグメントは、構成の変更中に破棄されません。これが発生すると、フラグメントインスタンスが残り、そのビューのみが破棄さonDestroy()れるため、呼び出されず、DESTROYED状態にはなりません。


これは、onCreateView()後でLiveDataの監視を開始し（通常はonActivityCreated()）、スニペットをLifecycleOwnerとして渡す場合を意味します。


class BooksFragment: Fragment() {

    private lateinit var viewModel: BooksViewModel

    override fun onCreateView(inflater: LayoutInflater, container: ViewGroup?, savedInstanceState: Bundle?): View? {
        return inflater.inflate(R.layout.fragment_books, container)
    }

    override fun onActivityCreated(savedInstanceState: Bundle?) {
        super.onActivityCreated(savedInstanceState)
        viewModel = ViewModelProviders.of(this).get(BooksViewModel::class.java)

        viewModel.liveData.observe(this, Observer { updateViews(it) })  //    LifecycleOwner
    }

    ...
}

ViewModelProviderはViewModelのインスタンスを保存しますが、ViewModelFactoryのインスタンスは保存しません。そのため、次のコードがあるとします。


class MoviesViewModel(
    private val repository: MoviesRepository,
    private val stringProvider: StringProvider,
    private val authorisationService: AuthorisationService
) : ViewModel() {

    ...
}

class MoviesViewModelFactory( 
    private val repository: MoviesRepository,
    private val stringProvider: StringProvider,
    private val authorisationService: AuthorisationService
) : ViewModelProvider.Factory {

    override fun <T : ViewModel> create(modelClass: Class<T>): T { 
        return MoviesViewModel(repository, stringProvider, authorisationService) as T
    }
}

class MoviesActivity : AppCompatActivity() {

    @Inject
    lateinit var viewModelFactory: MoviesViewModelFactory

    private lateinit var viewModel: MoviesViewModel

    override fun onCreate(savedInstanceState: Bundle?) {  
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_movies)

        injectDependencies()

        viewModel = ViewModelProviders.of(this, viewModelFactory).get(MoviesViewModel::class.java)
    }

    ...
}

次に、構成変更が発生するたびに、ViewModelFactoryの新しいインスタンスを作成します。したがって、すべての依存関係の新しいインスタンスを作成する必要はありません（ただし、それらが何らかの形で無制限である場合）。


class MoviesViewModel(
    private val repository: MoviesRepository,
    private val stringProvider: StringProvider,
    private val authorisationService: AuthorisationService
) : ViewModel() {

    ...
}

class MoviesViewModelFactory(
    private val repository: Provider<MoviesRepository>,             
    private val stringProvider: Provider<StringProvider>,         
    private val authorisationService: Provider<AuthorisationService>
) : ViewModelProvider.Factory {

    override fun <T : ViewModel> create(modelClass: Class<T>): T {  
        return MoviesViewModel(repository.get(),                    
                               stringProvider.get(),                
                               authorisationService.get()
                              ) as T
    }
}
