Amazon AuroraサーバーレスのDATA API


従来のデータベースへの接続はどのように機能しますか？接続を開き、それを使用して1つ以上のSQLクエリまたはその他のステートメントを処理してから、接続を閉じます。オペレーティングシステム、プログラミング言語、データベースに固有のクライアントライブラリを使用した可能性があります。ある時点で、接続の作成に時間がかかり、データベースエンジンのメモリを消費することに気付きました。

DATA APIは、長期的なデータベース接続の管理とスケーリングを処理し、簡単に分析できるようにデータをJSON形式で返します。すべてのトラフィックは暗号化されており、HTTPSを介して発生します。

では、DATA APIとは何でしょうか？

ExecuteStatement-単一のSQLクエリを実行します。
BatchExecuteStatement-データ配列に対して1つのSQLクエリを実行します。
BeginTransaction-トランザクションを開始し、トランザクション識別子を返します。
CommitTransaction-トランザクションを完了し、トランザクションで実行されたSQL操作をコミットします。
RollbackTransaction-トランザクションをロールバックします。

唯一の制限は、各関数が1分以内に完了する必要があり、最大1メガバイトのデータを返すことができることです。
