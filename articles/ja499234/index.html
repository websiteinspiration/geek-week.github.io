Flexiant Cloud Orchestratorのネットワーキング

クラウドオーケストラに関する一連の記事の続きで、Flexiantは、仮想マシンとコンテナーのネットワークの組織がこのオーケストラの下でどのように管理されているかについて話したいと思います。

Flexiantは、クライアントが仮想マシンでアクセスできる4種類のネットワークを識別します。

PVIPは、単一の/ 32アドレスがクライアントに割り当てられるネットワークです。このIPアドレスへのルーティングは、
パブリックIP 仮想マシンが配置されているホストマシン（クライアントが個別のVLANおよび/ 29サブネットを受信するネットワーク）で実行されます。これは私たちが使用するメインのネットワークです。そのルーティングは、Flexiant Cloud Orchestratorによって管理される特別なルーターノードで実行されます。

プライベートIP-ネットワーク、前のものと同様の組織。ただし、ルーターノードでは外部に表示されません。クライアントはそれを使用して、仮想マシン間に内部ネットワークを作成できます。 IPアドレスは、インターフェースとマシン自体の両方から割り当てることができます。

インターワーキングVLANも、プロバイダーの参加なしではクライアントが直接利用できないことを除いて、プライベートIPと非常によく似ています。クライアントが既存のL2インフラストラクチャをFlexiant Cloud Orchestratorインフラストラクチャに接続する必要がある場合、このネットワークをクライアントに提供します。

最初のネットワークを除くすべてのネットワークは個別の独立したVLANで動作するため、クライアントはすべてのL2プロトコル（arp、dhcp、lldpなど）を使用できます。 PVIPネットワークはL3で実行され、DDoSから保護を取得したいクライアントに使用します。 IPアドレスを保護する場合は、IPアドレスを使用する必要があります。

各ネットワークは仮想マシンアダプターに割り当てられます。したがって、1つの仮想マシンに同時に複数のタイプのネットワークが存在する可能性があります。

プライベートIPおよびインターワーキングVLANのIPアドレスは、クライアントが任意に選択できます。パブリックIPおよびPVIPのIPアドレスは、コントロールパネルのリストから選択されます。



アダプターがネットワークとオプションでIPアドレスを受信するとすぐに、これに関する情報がオーケストラのデータベースに取り込まれ、対応する設定がルーティングノード（パブリックIPネットワークの場合）、コンピューティングノード（PVIPの場合）、および配置されているDHCPサーバーに入力されます管理ノードで、IPアドレスが設定されている場合。カードの仮想MACアドレスと選択したIPアドレスの間の対応表がPgSQLデータベースに入力され、そこからDHCPサーバーによってさらに取得されます。

ルーターノードでも計算ノードでも、各ノードにFlexiantエージェントがインストールされ、オーケストラからコマンドを受信します。ノード自体の内部では、仮想ルーターが別のネットワーク名前空間で発生します。したがって、ノード上のLinuxネットワークスタックと仮想ルーターネットワークスタックは互いに独立しています。

コイの実装において、パブリックIPの機能がVRRPプロトコルを介して相互に重複することを保証するルーターノード。ノードの1つに障害が発生すると、トラフィックはもう1つのノードを通過します。

evrr / evrlインターフェースのペアは、2つのネットワーク名前空間を相互接続するLinux上の通常のvethです。したがって、仮想ルーターに入る着信トラフィックがどのように特定のインターフェイスにルーティングされるかがわかります。

ファイアウォール管理も仮想ルーターで行われます。特定のネットワークインターフェイスへのトラフィックを許可または禁止する個別のテーブルとチェーンが作成されます。

ルーティングプロトコルとしてOSPFを使用します。コンピューティングとルーターノードの構成はネットワークのコアに到達し、そこからコアルーターに到達します。ノード上のソフトウェアとして、Flexiantによって制御される鳥が使用されます。Flexiantではこれが可能であるため、構成テンプレートにも変更を加えます。Flexiantでは、スタティックおよびbgpをルーティングプロトコルとして使用することもできます。

支払いに関しては、すべてのネットワークが無料です。支払いは、PVIPおよびパブリックIPネットワークのIPアドレスに対してのみ行われます。ただし、PVIPのIPアドレスはDDoS攻撃から保護されているため、より高価です。
