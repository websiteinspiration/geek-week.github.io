JSオブジェクトを操作する際のRestおよびSpreadオペレーターの7つ​​のトリック

こんにちは、ハブル！Joel Thomsによる記事「JavaScriptオブジェクトの休憩と拡散に関する7つの秘訣」の翻訳を紹介します。

みなさん、こんにちは。先日、職場の同僚から、休息演算子とスプレッド演算子を操作するさまざまな方法をリストした英語の記事へのリンクが表示されました。彼女は私にとって役に立ち、私はそれを翻訳することにしました。それでは始めましょう。



RestおよびSpread演算子は、引数を配列に結合したり、配列を結合したりするためだけに使用できるわけではありません。

この記事では、レストオペレーターとスプレッドオペレーターを使用する上で最も知られていない7つのトリックを紹介します。

1.プロパティを追加する


複製されたオブジェクトに新しいオブジェクトを追加しながらオブジェクトを複製します。
以下の例では、ユーザーオブジェクトを複製し、複製されたuserWithPassオブジェクトにpasswordプロパティを追加します。

const user = { id: 100, name: 'Howard Moon'}
const userWithPass = { ...user, password: 'Password!' }

user //=> { id: 100, name: 'Howard Moon' }
userWithPass //=> { id: 100, name: 'Howard Moon', password: 'Password!' }

2.オブジェクトのマージ


分析プロセスのパート1とパート2のオブジェクトmerzhatsya USER1

const part1 = { id: 100, name: 'Howard Moon' }
const part2 = { id: 100, password: 'Password!' }

const user1 = { ...part1, ...part2 }
//=> { id: 100, name: 'Howard Moon', password: 'Password!' }

そのため、この構文を使用すると、同じオブジェクトを隣接させることができます。

const partial = { id: 100, name: 'Howard Moon' }
const user = { ...partial, id: 100, password: 'Password!' }

user //=> { id: 100, name: 'Howard Moon', password: 'Password!' }

3.オブジェクトからプロパティを削除する


オブジェクトのプロパティは、構造化解除ステートメントとRestステートメントの組み合わせを使用して削除できます。以下の例では、restオブジェクトを破棄し、そこからpasswordプロパティを削除しています。

const noPassword = ({ password, ...rest }) => rest
const user = {
  id: 100,
  name: 'Howard Moon',
  password: 'Password!'
}

noPassword(user) //=> { id: 100, name: 'Howard moon' }

4.プロパティの動的削除


以下の例では、removeProperty関数は引数としてpropを取ります。引数で渡されたプロパティ名を使用して、クローンされたオブジェクトから動的にプロパティを除外します。

const user1 = {
  id: 100,
  name: 'Howard Moon',
  password: 'Password!'
}
const removeProperty = prop => ({ [prop]: _, ...rest }) => rest
//                     ----       ------
//                          \   /
//                dynamic destructuring

const removePassword = removeProperty('password')
const removeId = removeProperty('id')

removePassword(user1) //=> { id: 100, name: 'Howard Moon' }
removeId(user1) //=> { name: 'Howard Moon', password: 'Password!' }

5.プロパティの編成または名前による並べ替え


オブジェクトのプロパティが必要な順序になっていない場合があります。いくつかのトリックを使用して、たとえば、プロパティをリストの最上部または中央にプッシュできます。

const user3 = {
  password: 'Password!',
  name: 'Naboo',
  id: 300
}

const organize = object => ({ id: undefined, ...object })
//                            -------------
//                          /
//  move id to the first property

organize(user3)
//=> { id: 300, password: 'Password!', name: 'Naboo' }

パスワードを最後 に置くには、まずオブジェクトを破棄してパスワードを削除し、次にスプレッド演算子を使用してそれを追加する必要があります。

6.デフォルトのプロパティ


デフォルトのプロパティは、クローンされたオブジェクトにない場合に設定されるプロパティです。

以下の例では、引用符フィールドがuser2オブジェクトにありません。setDefaults関数は、quotesフィールドがない場合、デフォルトで値[]を使用して設定します。setDefaults（user2） 関数を呼び出した後、値quotesを持つプロパティを返します：[] setDefaults（user4）を 呼び出すと、user4オブジェクトにはすでにquotesプロパティがあるため、関数は変更されていない値を返します




const user2 = {
  id: 200,
  name: 'Vince Noir'
}

const user4 = {
  id: 400,
  name: 'Bollo',
  quotes: ["I've got a bad feeling about this..."]
}

const setDefaults = ({ quotes = [], ...object}) =>
  ({ ...object, quotes })

setDefaults(user2)
//=> { id: 200, name: 'Vince Noir', quotes: [] }

setDefaults(user4)
//=> {
//=>   id: 400,
//=>   name: 'Bollo',
//=>   quotes: ["I've got a bad feeling about this..."]
//=> }

デフォルト値を最初に表示したい場合は、次のように書くこともできます：

const setDefaults = ({ ...object}) => ({ quotes: [], ...object })

7.プロパティの名前を変更する


上記のトリックを使用して、オブジェクトのプロパティの名前を変更する関数を作成することもできます。

プロパティIDが大文字で書かれたオブジェクトがあり、名前を小文字に変更する必要があるとします。まず、オブジェクトからIDプロパティを削除します。次に、オブジェクトのクローンを作成しているときに、idという名前で再度追加します。

const renamed = ({ ID, ...object }) => ({ id: ID, ...object })

const user = {
  ID: 500,
  name: "Bob Fossil"
}

renamed(user) //=> { id: 500, name: 'Bob Fossil' }

8.ボーナス。条件に基づくプロパティの追加


そのような方法を示して くれた@vinialbanoに感謝します。この例では、password == trueの場合にのみパスワードフィールドを追加します。

const user = { id: 100, name: 'Howard Moon' }
const password = 'Password!'
const userWithPassword = {
  ...user,
  id: 100,
  ...(password && { password })
}

userWithPassword //=> { id: 100, name: 'Howard Moon', password: 'Password!' }
