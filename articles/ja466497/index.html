React Nativeアプリケーションの最新環境

この記事では、expo-cli、Typescript、およびJestを使用してReact Native環境をセットアップする方法を見ていきます。Typescriptは、開発ミスを回避し、より効率的なモバイルアプリケーションを作成するのに役立ちます。 最新の開発では、Typescriptを開発環境に統合できます。 Typescriptを完全にサポートするVS Codeを使用することもできます。 また、統合では、ネイティブが反応私たちの自動補完サービス、コードナビゲーションとリファクタリングを使用する機会を提供します。万博







ネイティブReactアプリケーションの作成を簡素化するツールのセットです。このチュートリアルでは、Expoを使用してネイティブReactアプリケーションをすばやく作成する方法を紹介します。



プロジェクトを作成する


最初に、アプリケーションテンプレートの作成に必要なすべての依存関係を糸からインストールします。

yarn global add expo-cli

次に、React Nativeアプリケーションを初期化し、空のTypescriptテンプレートを選択します。

Tslintの構成


VSCodeまたは別のエディターを正しく使用するようにtslintを構成し、開発段階でエラーを確認します。このようにして、統一されたコードスタイルを確保し、その複雑化を防ぎます。

：まず、グローバルディレクトリにtslintパッケージをインストール
糸グローバル加算tslint

次に、tslint構成を作成：
tslint --init

このコマンドは、以下の構成でtslint.jsonファイルを作成します。
{
  "extends": ["tslint:recommended"],
  "jsRules": {},
  "rules": {},
  "rulesDirectory": []
}

次に、設定を使用してtypescriptファイルを単にチェックします：
tslint 'src / ** / *。Ts'

その後、VS Codeは自動的に設定ファイルを使用して、エディターでコードを検証します。アプリケーションの開発時に、タイプエラーがエディターに表示されます。

また、便宜上、yarnのコマンドを追加します。継続的インテグレーションの段階で、ローカルでの開発または検証に使用できます。
"スクリプト"：{
"lint"： "tslint '* .ts *'"
}

また、tslintはプラグインをインストールすることで検証ルールを拡張する機能をサポートします。この時点で、ルールにeslintサポートを追加します。
yarn add tslint-eslint-rules --dev

tslint構成にプラグインを追加するには、拡張フィールドにプラグイン名を追加するだけです：

"extends": [
    "tslint:recommended",
    "tslint-eslint-rules"
]

JestおよびTypescriptテスト


Jestは、JavaScriptコードをテストするためのフレームワークです。React Nativeアプリケーションのテスト、サポート、開発を容易にします。

最初に、フレームワークをインストールし、typescriptサポートを追加する必要があります。これは、コードベースが静的型付けのサポートで記述されているためです。

yarn add --dev jest ts-jest @ types / jest

追加のパッケージをインストールする価値もあります：
  • ts-jest -JavaScriptでtypescriptテストコードを処理およびコンパイルする
  • @ types / jest -Jest環境からタイプを追加します
  • react-native-testing-library -DOMにバインドせずにReactコンポーネントをレンダリングできます

そして、最後のステップは、テスト用の設定ファイルを作成することです：

糸TS-冗談設定：initは

このコマンドは、テンプレートからの構成ファイルを生成します。React Native環境用に少し変更します。ファイルは次のようになります。

module.exports = {
  jest: {
    preset: "react-native",
    transform: {
      "^.+\\.js$": "./node_modules/react-native/jest/preprocessor.js",
      "\\.(ts|tsx)$": "ts-jest"
    },
    globals: {
      "ts-jest": {
        tsConfig: "tsconfig.json"
      }
    },
    moduleFileExtensions: ["ts", "tsx", "js"],
    testRegex: "(/__tests__/.*|\\.(test|spec))\\.(ts|tsx|js)$"
  }
};

そして最後に、package.jsonにテストを実行するコマンドを追加します：

"test"： "yarn test"

Jestのテストを作成します


Jestのテストの例を書いてみましょう。これを行うには、ファイルApp.test.tsxを作成します。

export const helloFn = (name?: String = "World") => `Hello, ${$name}`;
 
describe("hello function", () => {
  it("should return `hello world`", () => {
    expect(helloFn()).toEqual(`Hello, World`);
  });
 
  it("should return `hello name`", () => {
    expect(helloFn("Zuck")).toEqual(`Hello, Zuck`);
  });
});

テストを実行するには、以前に作成したyarnコマンドを実行します。

yarn test

すべてのテストは成功しました。しかし今のところ、コンポーネントのテストを書くことはできません。React Nativeコンポーネントをテストするための環境を拡張してみましょう。
別のパッケージ、react-test-rendererが必要です。これは、出力にJavascript構造を与えるReactの特別なレンダリングエンジンを提供します。したがって、テスト環境でDOMまたはネイティブモジュールを構成する必要はありません。

yarn add -D react-test-renderer

次に、App.tsxコンポーネントの簡単なテストでApp.test.tsxファイルを更新します。

import React from "react";
import renderer from "react-test-renderer";
 
import App from "./App";
 
describe("App", () => {
  it("should display welcome message", () => {
    const tree = renderer.create(<App />);
 
    expect(tree.toJSON()).toMatchSnapshot();
    expect(tree.root.findByType("Text").children).toContain(
      "Open up App.tsx to start working on your app!"
    );
  });
});

テスト環境でネイティブコンポーネントをテストできます。この例では、Textタグから子の配列を取得しました。これは、React Nativeパッケージのネイティブコンポーネントです。

結論


このテクノロジスタックにより、ネイティブアプリケーションを開発するための環境をすばやく作成できました。静的型を使用するビジネスロジックは、アプリケーションをより安定させます。Typescriptの強い型付けは、コーディングエラーの回避にも役立ちます。
Jest内部でのReact Nativeコンポーネントのテスト開発は、通常のReactアプリケーションの場合とまったく同じです。

この記事が、React Nativeアプリケーションを開発するための環境設定の初期段階を克服するのに役立つことを願っています。
