今年のARKit開発とARKit 4の新機能：ロケーションアンカー



近年、私はAppleプラットフォーム向けのネイティブゲームの開発に非常に積極的に取り組んでおり、ARでのゲームプロセスの統合の可能性に関心があります。したがって、私はWWDCで毎年発表されるすべての更新を監視するようにしています。 SpriteKit、SceneKit、GameplayKitの更新は近年ありませんが、Appleはこれらのゲームツールと統合できるARKitおよびRealityKitを積極的に更新し、宣伝しています。



AppleはWWDCでのプレゼンテーション中に拡張現実にほとんど注意を払っていませんでしたが、今週リリースされた新しいバージョンのソフトウェア開発キット（SDK）は、ARアプリケーションの作成に携わるすべての人にとって非常に便利で興味深いものです。ではARKit 4つのプレゼントプロセッサを搭載したすべてのiOS / iPadOSデバイスで利用可能な開発者のための新しい機会、A12バイオニック以上。



ご覧のように、近年、AR技術の開発へのAppleの投資は正当化されることが証明されています。これは、コンピュータの視覚化、機械学習、および動き検出の機能を使用して、固定カメラを使用してインタラクティブなコンピュータグラフィックスのより複雑な世界を作成できるようになったためです。このテクノロジーの開発は、スマートフォンをベースにした仮想現実システムを作成しようとする競合他社の試みよりもさらに成功し、生産性が高いことがわかりました。



このツールは、Apple LiDARスキャナーを搭載したデバイスで利用できる重要な新しい深度検出機能も追加します（現在は最新のiPad Proでのみ利用可能です）。しかし、おそらくARKit 4で最も重要なのはロケーションバインディングです。これにより、開発者は仮想オブジェクトを現実世界の特定の場所に配置できます。



LiDAR：高度なリモート追跡システム



Appleは、LiDARスキャナーを備えたiPad Proラインの発売後、ARKit 3.5にScene Geometry APIを導入しました。私はAppleがLiDARスキャナーをその次世代のiPhoneに追加することを期待しています、それはそれが今年の終わりまでにリリースすることを計画しているので、この機能は次のローンチの間に最も話題になるでしょう。



新しいScene Geometry APIでは、LiDARスキャナーを使用して情報を収集し、取得したデータに基づいて世界のトポロジマップを作成できます。この情報を使用して、特定の物理オブジェクトを識別し、それらをステージに配置して、現実世界と仮想世界のオブジェクト間の相互作用のゲームシミュレーションを作成できます。



ARKit 4は、仮想オブジェクトを実在する人々の前または後ろに配置し、レイキャスティング機能を使用して現実にイベントをリアルに表示して、実在するコンテンツとデジタルコンテンツの境界をぼかすのに役立ちます。





ではiOSの14、 Appleはより良い仮想環境でそれらとオブジェクト間の距離を決定するためのLiDARスキャナーのサポートをiOSデバイスの機能を拡張します。機械学習を使用すると、デバイスの広角カメラから取得したカラーRGB画像とLiDARスキャナーからの深度インジケーターを組み合わせて、緻密で深い画像を作成できます。この深度データは60Hzの頻度で更新されます。これにより、iOSはオブジェクトをステージに表示するだけでなく、仮想環境でリアルタイムにオブジェクトを配置できるようになります。



LiDARでは、レイキャスティングと呼ばれる機能を改良することもできます。レイキャスティングは、計算ジオメトリを使用して2次元平面に3次元空間を作成するレンダリング方法です。さらに、Appleは以前のバージョンのARKitで追跡機能を提供していましたが、その使用は、正面向きのTrue-Depthカメラを備えたデバイスでのみ利用できました。 ARKit 4は、最近リリースされた次世代のiPhone SEを含む、A12 Bionicプロセッサー以降を搭載したすべてのデバイスでフェイストラッキングを利用できるようにすることで、これらの機能を大幅に拡張します。フェイストラッキングを使用すると、開発者は仮想コンテンツの上に人物の画像を配置し、その表情をリアルタイムで観察するアプリケーションを作成できます。





Location Anchors



もちろん、LiDARスキャナーが提供する機能は、Appleが発表したすべての新機能の中でも非常に印象的です。 ARKit 4の新しいロケーションアンカーテクノロジーにより、高品質のARコンテンツが街路にもたらされ、開発者は仮想オブジェクトの配置の経度と緯度を設定できます。次に、ARKit 4はこれらの座標とApple Mapsからのデータを使用して、ARオブジェクトを特定の場所に、現実世界の特定の高さに配置します。



次世代のARインターフェイス管理プロセスは、ロケーションアンカーと呼ばれます。これにより、環境に関連してデバイスを正確にロックできます。Appleによると、これはGPSを使用するよりもはるかに正確に行うことができます。このプロセス全体は、高度な機械学習技術に基づいてデバイスのローカルで制御されます。



その結果、開発者が現実の世界に仮想オブジェクト、たとえば混雑した広場の仮想彫刻を配置すると、このオブジェクトはこの場所に保存および表示されるため、Apple ARデバイスを使用して表示するすべての人がこれを見ることができます。場所。ロケーションアンカーは、ロサンゼルス、サンフランシスコ、シカゴ、マイアミ、ニューヨークなどの主要都市に最初に表示され、その後、この夏の終わりまでに、さらにいくつかの都市が利用可能になります。



WWDCセッションからのロケーションアンカーの例




ロケーションアンカーの重要性は多くの開発者に評価され、これはAppleがAR関連のデバイスとテクノロジーの開発に向けた広範囲にわたる計画を実証している。今日、拡張現実技術に焦点を当てた多くの新興企業が機能的な機能を開発するために革新的な市場セグメントを占有しようとしています。Appleは今週、ARKit 4に多くの新機能を静かに立ち上げました。



同社は独自の地図データを所有しており、絶えず位置情報を収集する何億ものデバイスを持っているため、多くの開発者の位置を特定することに基づいてAR機能を提供するのに最も有利な立場にあります。これらの新機能により、ARを最終的に最先端のテクノロジーにする次世代アプリケーションの作成が可能になります。



ロケーションアンカーの仕組み



ARKit 4では、座標を設定するだけで、ARコンテンツを現実世界の特定の場所に簡単にバインドできます。以前は画像認識またはコードスキャンに基づいて特定の場所でアクションをトリガーしていたARベースのアプリケーションは、リアルタイムで更新し、場所ベースの機能を利用できるようになりました。



Apple Mapsデータベースの変革の一環として、同社は長い間、世界のさまざまな場所にある街路のカメラと3D LiDARからデータを収集していました。場所にバインドするため、ARKitはクラウドからデバイスを取り巻く仮想マップをダウンロードし、カメラデータにマッピングします。ARKitはGPSを使用して、現実世界での位置をすばやく正確に特定できます。すべての情報処理は、デバイス上で直接機械学習テクノロジーを使用して実行されます。



可用性



すでに述べたように、ロケーションアンカー機能、つまりジオトラッキングは、GPSおよびA12チップを搭載したすべてのデバイスでサポートされています。この機能では、このエリアがすでにAppleカードにマッピングされている必要があるため、この機能は米国の特定の都市でのみ利用できます。 2020年6月現在、サポートエリアは5都市のみでしたが、今後もアップルがサポートエリアを拡大していきたいと思います。



ジオトラッキングは吹き替え新しいARConfiguration、に基づいておりARGeoTrackingConfigurationそれが簡単に互換性のあるデバイスとそれらの可用性をチェックすることができ、。



まず、デバイスがこの機能をサポートしているかどうかを確認します。 



guard ARGeoTrackingConfiguration.isSupported else { return }


これで、デバイスがサポートされている都市にあるかどうかを確認できます。その場合は、ARViewでジオトラッキング構成を開始します。RealityKitを使用する場合、自動構成を適用できないため、手動で開始する必要があります。 




ARGeoTrackingConfiguration.checkAvailability { available, error in
    guard available else { return }
    arView.session.run(ARGeoTrackingConfiguration())
}


重要な注意点。checkAvailability関数には、ARリソースを読み込むためのネットワーク接続が必要です。デバイスがインターネットに接続されていない場合、Availableはfalseになります。



ロケーションアンカーの構築



ロケーションアンカーはどのように機能しますか？結局のところ、ARKitは独自の座標系を使用してデバイスの相対位置を決定することが知られていますが、現実の世界では、位置は緯度と経度で記述されています。ジオトラッキングでは、この問題は単一の座標系を作成することで解決されました。 ARKit軸は自動的にコンパスと一致します。X軸は東方向を示し、Z軸は南方向を示します。ARGeoAnchor



を作成するために知っておく必要があるのは、別個のGPS座標だけです。たとえば、ゴールデンゲートブリッジを見つけるためにスナップを作成する方法は次のとおりです。信頼性を高めるために、小数点以下の正確な座標系を使用しています。 




let coordinate = CLLocationCoordinate2D(latitude: 37.8185, longitude: -122.4738)
let geoAnchor = ARGeoAnchor(name: "Golden Gate Bridge", coordinate: coordinate)


必要に応じて、高さをメートル単位で指定できます。これは、デフォルトで、海抜より上のパラメータを設定することで受け入れられます。 

let geoAnchor = ARGeoAnchor(name: "Golden Gate Bridge", coordinate: coordinate, altitude: 67)


これで、アンカーをメインシーンに追加できます。でRealityKit、これはに基づいて動作しますAnchorEntityのツールからARGeoAnchor。 


arView.session.add(anchor: geoAnchor)
let geoAnchorEntity = AnchorEntity(anchor: geoAnchor)
arView.scene.addAnchor(geoAnchorEntity)


これで、アプリケーションを作成し、選択した場所に関連する他のオブジェクトを追加して、実際の空間で好きなように回転して配置できます。

進歩AR技術は避けられない



ARテクノロジの開発を注意深く監視している多くの人は、このテクノロジセクターの進歩のペースが遅いことに不満を持っています。もちろん、私たちの多くは、たとえば、AppleのARメガネを今日そして今持っていることを拒否しません。実際、これはかなり複雑なテクノロジーであり、ここで重要なのは速度ではなく、正しい実装です。光学、バッテリー寿命、ワイヤレス接続、およびその他の多くの側面に関連するデバイスを作成する際の実際の問題に加えて、変化し続ける現実を深く理解し、反映した優れたARコンテンツも必要です。少数の企業だけが、この問題を単独で処理するためのリソースを持っています。それらの中で、Appleのほか、成功しているのは、最近スタートアップ6D.AIを買収したMicrosoftとNianticです。



もう1つの問題は、ARコンテンツを実行できるハードウェアおよびソフトウェアプラットフォームの不足です。Appleは、ARKit 4とiOS 14を使用して、最新テクノロジーの市場における世界最大のARプラットフォームとしての地位を大幅に強化し、ユーザーが待ち望んでいたARアプリケーションを作成するための新しいツールを開発者に提供できます。
