PHPランタイム


ランタイムAPIを使用すると、Lambdaで任意のランタイムを使用できます。ある意味では、サーバーレスプラットフォームとランタイムの間にHTTP APIがあるため、これはOpenWhiskに似ています。 Lambdaでは、ランタイムが呼び出しデータを受信するためにランタイムがプラットフォームにリクエストを送信するのに対し、OpenWhiskはランタイムが提供するエンドポイントを呼び出すという大きな違いがあります。詳細については、Michael MoussaのAWSブログの投稿を参照してください。

開始するには、Lambda用のPHPランタイムが必要です。これは、PHP実行可能ファイル、サーバーレス関数を呼び出すPHPコード、およびファイルで構成されます。bootstrapプラットフォームの要求に応じて。これらの3つからレイヤーを収集します。レイヤーは別のアカウントで再利用できるため、AWSがPHPアカウントを提供していないことに驚いています。信じられないほどですが、真実です。PHP7.3を使用していないため、独自にビルドする必要があります。プロジェクトの
ディレクトリlayer/phpに配置したすべてのファイル。

PHP実行可能ファイルを構築する


Lambdaコンテナー内で実行されるPHP実行可能ファイルが必要です。これを行う最も簡単な方法は、Lambdaと同じプラットフォームでコンパイルすることです。そのため、EC2を使用します。マイケルの記事では、これを行う方法を説明しています。これらのコマンドをcompile_php.shスクリプトにラップしてから、それをEC2インスタンスにコピーし、実行可能ファイルを実行してコンピューターにコピーします。

$ export AWS_IP=ec2-user@{ipaddress}
$ export SSH_KEY_FILE=~/.ssh/aws-key.rsa
 
$ scp -i $SSH_KEY_FILE compile_php.sh $AWS_IP:doc/compile_php.sh
$ ssh -i $SSH_KEY_FILE -t $AWS_IP "chmod a+x compile_php.sh && ./compile_php.sh 7.3.0"
$ scp -i $SSH_KEY_FILE $AWS_IP:php-7-bin/bin/php layer/php/php


このアプローチにより、再現性が向上し、新しいバージョンのPHPにアップグレードされることを期待しています。

ブートストラップ


ランタイムAPIを使用するため、bootstrapファイルが必要です。Lambda自体は、ファイルを指定するためにそのような名前を必要とします;ループでAPIを呼び出すことにより、画像に対応する関数呼び出しに応答します。

基本的に、/next次に何を呼び出すかを理解し、それを呼び出して、エンドポイントに応答を送信するために、ループ状態にあり、エンドポイントを呼び出す必要があります/response。
AWSはcurlを使用したBASHの例を提供します。

while true
do
  HEADERS="$(mktemp)"
  # Get an event
  EVENT_DATA=$(curl -sS -LD "$HEADERS" -X GET "http://${AWS_LAMBDA_RUNTIME_API}/2018-06-01/runtime/invocation/next")
  REQUEST_ID=$(grep -Fi Lambda-Runtime-Aws-Request-Id "$HEADERS" | tr -d '[:space:]' | cut -d: -f2)
 
  # Execute the handler function from the script
  RESPONSE=$($(echo "$_HANDLER" | cut -d. -f2) "$EVENT_DATA")
 
  # Send the response
  curl -X POST "http://${AWS_LAMBDA_RUNTIME_API}/2018-06-01/runtime/invocation/$REQUEST_ID/response"  -d "$RESPONSE"
done


PHPでも同じようにしたいのですが、自分で書くこともできますが、Pariksit AgnihotriはPHP-Lambda-Runtime / runtime.phpですでに私の前にいるので、これをにコピーしlayer/php/runtime.phpます。私のバージョンでは、いくつかの変更を加え、json_encodingを追加し、エラーハンドラーを改善しました。
このファイルはlayer/php/bootstrap非常にシンプルで、PHP実行可能ファイルをこのファイルで実行するだけです。

#!/bin/sh
cd $LAMBDA_TASK_ROOT
/opt/php /opt/runtime.php


それで全部です。これで、layer / phpに3つのファイルができました。

  • php -PHP実行可能ファイル。
  • runtime.php -作業ファイルランタイムAPI;
  • bootstrap -必要なLambdaファイル。


その結果、これらすべてがLambdaアプリケーションのPHPレイヤー（レイヤー）になります。

サーバーレスフレームワークの構成


サーバーレスフレームワークは、サーバーレスアプリケーションの繰り返し可能な構成と展開を提供します。私はこのコンセプトのファンであり、そのようなツールをもっと使いたいと思っています。 PHP Hello Worldにはサーバーレスフレームワークを使用します。
サーバーレスフレームワークのPHPには、アプリケーション用の便利なテンプレートがないためserverless.yml、プロジェクトのあるディレクトリにファイルを作成するだけです。
まず、最も基本的なもの：

service: php-hello-world
provider:
  name: aws
  runtime: provided
  region: eu-west-2
  memorySize: 128


アプリケーションに名前を付け、php-hello-worldAWSをプロバイダーとして使用します。私は英国にいるので、ロンドン地域を設立しました。大量のメモリは必要ないため、128 MBで十分です。
ランタイムは通常、関数を実行する言語です。runtime APIどのbootstrapファイルを実行するかを使用するには、このフィールドを提供に設定します。
また、以下.gitignoreを含むファイルも必要です。

.serverless


以来git、私たちは、このディレクトリは必要ありません。
次に、レイヤーをserverless.ymlに追加してみましょう。

layers:
  php:
    path: layer/php


これにより、AWSレイヤーが作成さPhpLambdaLayerれ、関数で参照できる名前が付けられます。

関数を記述しましょうHello World
サーバーレスPHP関数を記述できます。これはファイルに入力する必要があります  handler.php 。 

<?php
function hello($eventData) : array
{
    return ["msg" => "hello from PHP " . PHP_VERSION];
}


この関数は、イベントに関する情報を受け取り、連想配列を返します。
関数のデプロイについてサーバーレスフレームワークに通知するには、ファイルに以下を追加する必要がありますserverless.yml。

functions:
  hello:
    handler: handler.hello
    layers:
      - {Ref: PhpLambdaLayer}


サーバーレスフレームワークは、1つのアプリケーションで複数の機能をサポートします。それらのそれぞれには、この場合は名前が  hello あり、私たちの場合は拡張子のないファイル名であるハンドラー、その後にドット、このファイル内の関数の名前が続きます。したがって、ハンドラ  handler.hello は、関数をhello() で実行することを意味  し  handler.php ます。
最後に、PHPコードを実行できるように、PHPレイヤーに関数も報告します。
