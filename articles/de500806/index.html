Begleiten

Als ich vor einigen Jahrzehnten für Dave Weiner in UserLand arbeitete, lernte ich nach einer Hauptversion das Konzept der Wartung kennen.

Wenn Sie ein Anwendungsentwickler sind, scheint es Ihr Ziel zu sein, den Veröffentlichungstag zu erreichen. Füllen Sie die Anwendung aus, stellen Sie sie zur Verfügung, veröffentlichen Sie die Ankündigung und kehren Sie dann zur Programmierung zurück. Dann lass die Welt tun, was sie tut.

Mit anderen Worten: Rauchen Sie einen Joint und machen Sie sich wieder an die Arbeit.

Dies erhöht jedoch nicht die Chancen auf eine gute Veröffentlichung. Sie müssen begleiten - Sie müssen weitermachen.
Zum Beispiel könnten Sie:

  • Veröffentlichen Sie Tipps zur Verwendung der App - eine pro Tag oder so
  • Füllen Sie die Website mit Kommentaren, Bewertungen und guten Bewertungen auf.
  • Seien Sie verfügbar und kommunizieren Sie über die Anwendung
  • Gehe zu einigen Podcasts
  • Schreiben Sie über den Veröffentlichungstag
  • Schreiben Sie über Pläne für Version x.0.1
  • Vielen Dank, dass Sie auf Fehlermeldungen und Funktionsanfragen reagiert haben
  • Vielen Dank an die Rezensenten, die gute Arbeit geleistet haben
  • Bieten Sie Reportern und Gutachtern den einfachsten Zugriff auf die Anwendung und auf Sie
  • Erstellen Sie eine Community von Benutzern in Slack oder ähnlichem

Ich bin sicher, Sie können sich etwas anderes einfallen lassen. Das Obige ist nicht alles und alle Anwendungen sind unterschiedlich.

Aber die Hauptsache ist, dass Sie nicht nur loslassen und aufhören. Zeigen Sie stattdessen, dass Sie reagieren, dass die Anwendung an Dynamik gewinnt, und zeigen Sie, dass sie Sie genug begeistert, um weiterhin angekündigt zu werden.

Zumindest für mich ist das das interessanteste. Ich verstehe, dass dies nicht jedermanns Sache ist. Aber du solltest es trotzdem tun.
