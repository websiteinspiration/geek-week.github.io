8つのコードでのPHP 8

PHP 8には多くの新機能があります。このリストでは、最も優れたものを取り上げます。

免責事項：この記事への英語のリンクは、すでに最後のPHPダイジェストの投稿にありました。言語を十分に理解している場合は、元の言語に進む価値があるかもしれません。他の英語の記事への脚注がたくさんあります。

時間


use \Support\Attributes\ListensTo;

class ProductSubscriber
{
    <<ListensTo(ProductCreated::class)>>
    public function onProductCreated(ProductCreated $event) { /* … */ }

    <<ListensTo(ProductDeleted::class)>>
    public function onProductDeleted(ProductDeleted $event) { /* … */ }
}

docblockの代わりに、属性。

はい、私は知っています、構文はあなたが望んだまたは望んだ方法ではないかもしれません。おそらく、@、@：、またはdocblock、あるいは他の何かを好むでしょう。しかし、彼はそれだけです。構文について言及する価値がある唯一のことは、すべてのオプションが説明されていることであり、この構文が選択された非常に十分な理由があります。RFCでこれの簡単な要約を読むか、内部コンポーネントのリストで RFCに関するすべての議論を読むことができます。


二


public function foo(Foo|Bar $input): int|float;

public function bar(mixed $input): mixed;

タイプを組み合わせると、一度に複数のタイプを指定できます。同時にいくつかのタイプを表す新しい混合タイプもあります。


三


interface Foo
{
    public function bar(): static;
}

戻り値の型にはstaticを指定できます。


四


[JIT]
opcache.jit=5

組み込みJITコンパイラ。

JITとは

「JIT」は「ジャストインタイム」-「適切なタイミング」の略です。PHPはインタプリタ言語であることをご存じでしょう。C、Java、Rustのプログラムとしてコンパイルされていません。代わりに、実行時にプロセッサが理解するマシンコードに変換されます。

JITは、コンパイルされたバージョンを代わりに使用できるように、実行時にコードの一部をコンパイルするメソッドです。

実行時に生成される解釈済みコードの「キャッシュバージョン」と考えてください。

詳細はすでにハブレに書いています

$triggerError = fn() => throw new MyError();

$foo = $bar['offset'] ?? throw new OffsetDoesNotExist('offset');

throwは式で使用できます。


六

セブン

    string $parameterA,
    int $parameterB,

8

str_ends_with('haystack', 'stack');

新しい文字列関数。彼らの名前はそれ自体が物語っていると思います。
だまさないでください。8ブロックのコードでは、PHP 8の優れた新機能をすべて要約するのに十分ではありません。それでは、さらにいくつか追加してみましょう。


ナイン

十
