レッスンを始める前に、私たちのWebサイトnwking.orgで、CCNAコースの学習に関する情報だけでなく、ネットワークの専門家にとって役立つ他の多くのトピックも見つけることができることを言いたいと思います。他のメーカーの製品に関する興味深い情報やさまざまなプログラムのレビューを公開しています。

今日は、2つの重要な問題について説明します。低速接続とポートセキュリティスイッチのセキュリティ機能です。



シスコのネットワーク技術者の責任を引き受ける場合、解決しなければならない最も重要な問題の1つは、接続速度の低下です。会社の従業員が、インターネットが遅い、サーバーへのアクセスが遅い、IPテレフォニーの音声通話が遅いなどの不満を言う場合があります。この問題はオフィスネットワークで非常によく見られ、さまざまな理由で発生する可能性があります。たとえば、トラフィック量が急増し、接続が大幅に遅くなる場合などです。

ネットワークエンジニアは、問題の原因がどこにあるかを見つけるために、問題の最も多様な側面に注意を払う必要があります。ここでは、OSIモデル全体のトラブルシューティングについて話しているわけではありませんが、作業計画と、問題の原因を見つけるために実行する一連のアクションがあり、それが正確に何で構成されるかを理解する必要があります。誰かがあなたに電話して、インターネット接続が遅すぎると言った場合は、自分でこのサイトにアクセスして、コンピューターから確認する必要があります。大規模なソーシャルネットワーキングサイトは、ユーザーの流入が多いために速度が低下する場合があります。その場合、ネットワーク上の問題の解決策を探す必要はありません。したがって、同僚の問題がコンピューターの状態に依存しない場合があるため、低速接続のすべての側面を考慮する必要があります。サイトの読み込みが遅い場合、他のサイトが遅いかどうかを確認する必要があります。すべてのサイトの読み込みが遅い場合、その理由は特定のサイトではなく、インターネット接続の問題であると判断できます。



問題がどの部分にあるかを理解するには、問題を個別のコンポーネントに分割する必要があります。

ほとんどの場合、接続速度が遅い理由は2つあります。スイッチポートの速度とデュプレックスです。最新のスイッチはすべて、ファストイーサネットデバイスの場合は10/100 Mbit /秒、ギガビットイーサネットデバイスの場合は1 Gビット/秒の速度でデータを転送するように設計されています。当然、ギガビットイーサネットデバイスも10/100 Mbpsで実行できます。したがって、異なる速度のデバイスがありますが、4〜5年前に製造されたほとんどのデバイスは1ギガビット/秒をサポートせず、ファストイーサネットのみを提供します。ただし、最近の多くのデバイスは全二重をサポートしていません。
半二重は、デバイスがトラフィックを送信または受信のみできる場合で、全二重は、デバイスが情報を同時に送信および受信できる場合です。コンピューターを半二重モードに設定した場合、つまり、トラフィックを送信または受信でき、コンピューターが接続されているスイッチポートが全二重モードで機能する場合、そのような接続は機能しません。したがって、理想的なケースは、両方のデバイスが同じデュプレックスモード（半二重または全二重、たとえば100 Mbps）で動作する場合です。 1つのポートが10 Mbpsで動作し、もう1つのポートが100 Mbpsで動作する場合、通信が失敗する可能性があります。したがって、これらのことに注意する必要があります。

ほとんどの場合、デバイス構成はデフォルトで自動互換モードに設定されています。つまり、スイッチポートの速度モードとコンピューターのネットワークポートポートは自動デュプレックスモードに設定されています。ただし、速度が100 Mbpsのスイッチのファストイーサネットポートの場合、これにより問題が発生する可能性があります。ほとんどの場合、スイッチポートは速度の互換性を提供しますが、2番目のデバイスと「ネゴシエーション」できないため、それらが誤っている場合があります。この場合、1つのデバイスは半二重モードを選択し、2番目のデバイスは全二重を選択できます。

この場合、ギガビットイーサネットの方が動作が優れており、これについての説明があります。事実は、何年も前にファストイーサネット標準が作成されたとき、大多数のデバイスが半二重モードで動作していたことです。デフォルトでは、デバイスが一致しない場合、自動速度選択と自動デュプレックスに設定されたファストイーサネットスイッチは、100 Mbpsの速度と半二重モードを使用します。

デバイスの1つを全二重モードに設定し、100 Mbpsの速度を割り当てた後、これらのパラメーターをネットワーク上のすべてのスイッチに拡張するとします。ただし、ネットワーク上のデバイスの1つが自動モードになっている可能性があり、構成したデバイスのポートに接続すると、速度は100 Mbpsになりますが、同時に、自動モードではデフォルトで半二重になります。これが発生した場合、あなたは大きな問題に直面しています。

ただし、両方のデバイスがギガビットイーサネット標準を使用し、両方が自動モードの場合、デフォルトでは、1ギガビット/秒、または1000メガビット/秒の速度で全二重の状態になります。

誰かがあなたに電話をかけ、彼のコンピュータが遅いと言った場合、show int <port name>コマンドを使用してスイッチインターフェイスのステータスを表示できます。



ここから、多くの有用な情報を抽出できます。ファストイーサネットが有効になっていることがわかります（ファストイーサネット0/1がアップ）。「アップ」は、物理的にこのポートが実際に機能していることを意味します。さらに、ネットワークプロトコルがアクティブ、回線プロトコルがアップ、つまり接続が確立されて機能していることが報告されています。さらに、ポートの動作モードが全二重で、接続速度が100 Mbpsであることがわかります。

モードが半二重に設定されている場合、何かが故障している可能性があります。この場合、接続の反対側にあるデバイスを確認する必要があります。デバイスが半二重モードでも機能する場合は、コンピューターの動作が遅い理由が異なります。ただし、「全二重-半二重」モードの不整合がある場合、これは多くの問題の原因となる可能性があります。

デバイスが半二重モードの場合、データの送信または受信のみを行います。ただし、全二重モードで動作しているスイッチはそれを認識せず、データの受信と同時にデータを送信しますが、デバイスはトラフィックの送信でビジー状態であるため、データを受信できません。その結果、データは「衝突」し、単に失われます。前のレッスンでわかったように、TCPはフレームのシーケンスをチェックするメカニズムであり、この場合、失われたデータの転送を繰り返し試行します。ただし、宛先に到達しなかったトラフィックの送信を繰り返そうとすると、データ交換が大幅に遅くなります。同時に、トラフィックが受信者に到達していないことを確認すると、TCPは失われたデータの転送を何度も繰り返します。このプロセスは、ネットワークの速度も低下させます。

スイッチの操作を評価するためのもう1つの有用なパラメーターは、操作の最後の5分間のデータ入出力の速度で、この間に送受信されたデータ量をビット/秒またはパケット/秒で示します。スイッチが動作状態にある場合、これらのパラメータにはいくつかの値が含まれている必要があります。このポートを通るトラフィックがゼロであることを意味する0が表示される場合は、何かが間違っています。受信したブロードキャストの数と受信したブロードキャストの数と衝突の数に関する情報も役立ちます。すでに述べたように、スイッチに48個のポートがある場合、衝突の数も48になるはずです。0が表示された場合、このスイッチのどのポートも機能していません。 0に等しいレイトコリジョンの数も、スイッチの誤動作を示します。

これらの構成方法を示すために、Packet Tracerプログラムに移動します。 SW0の設定は、config tコマンドを使用して入力します。一枚の紙を取り、使用するコマンドを書き留めておくことをお勧めします。コマンドを手動で書き留めると、コマンドラインプロンプトを使用してキーボードでコマンドを入力するよりもはるかに速く覚えられると思います。したがって、グローバルコンフィギュレーションモードを開始するには、configure terminalコマンドを入力してから、interface fastEthernet 0/1コマンドを使用して、目的のインターフェイスに移動します。このポートの速度を設定するために、speedと入力すると、システムは3つの可能なパラメーターを生成します。



Packet Tracerにはファストイーサネットスイッチのみがあり、ギガビットイーサネットスイッチがないため、最大速度は100 Mbpsです。速度を10または100に設定できます。これには、speed 100コマンドを使用します。手動で速度を設定しているため、このポートの二重モードも手動で選択する必要があります。

自動速度モードと手動デュプレックス設定モードは使用できません。これらのパラメータのいずれかを自動選択モードに設定するか、両方のパラメータを手動で割り当てる必要があります。

警告：稼働時間中は、稼働中のネットワークでこれらのパラメーターの設定を変更しないでください。これを行うと、スイッチはポートを無効にし、再起動して新しいパラメーターでポートを有効にしますが、再起動する5秒間はネットワーク接続が中断されます。これらの設定は、新しく作成されたネットワークでのみ、または時間外に行うことをお勧めします。

したがって、デュプレックスを設定するには、duplexコマンドを入力する必要があります。その後、システムは3つの可能なモードを表示します。auto、全二重、半二重です。



duplex fullコマンドを発行して、このポートの全二重を有効にします。 f0 / 1インターフェースのパラメーターを見ると、全二重モードと100 Mbpsモードの回線を確認できます。同時に、show runコマンドを使用する現在の構成を見てみましょう。



FastEthernet0 / 1ポートの操作パラメーターは個別にリストされていますが、他のポートには操作モードが指定されていないことがわかります。これは手動で構成したためで、他のすべてのポートはデフォルトで構成されています。

このポートを無効にして、interface interface FastEthernet0 / 1 shutdownコマンドを使用して構成をもう一度確認したい場合は、このポートのパラメーターにシャットダウン行が追加されていることがわかります。したがって、スイッチインターフェイス指定のあるラインの下にシャットダウンラインがない場合、これはこのポートがオン状態であることを意味します。



次に、speed autoおよびduplex autoコマンドを使用して、このポートの動作モードを変更します。



現在の構成を再度確認しても、speed autoおよびduplex autoコマンドがデフォルトのコマンドであり、デフォルトのモードパラメータは現在のデバイス構成に表示されないため、変更はありません。そこで、ポートの速度とデュプレックスを構成する方法を示しました。

パラメーターを手動で調整するという考えは悪くありませんが、それは膨大な作業につながります-作業ネットワークの各スイッチを100 Mbpsの速度と全二重で手動で構成する必要があります。そのため、サーバーやルーターなどの重要なネットワークデバイスに対してのみこれを行うという、書かれていないルールがあります。通常、エンドデバイス、コンピュータ、またはIP電話が接続されているため、ハードポートの設定で予期せぬ事態が発生することはありません。速度を100メガビット/秒または1ギガビット/秒に設定し、全二重モードにすると、このポートに接続されたクライアントデバイスは、選択した接続パラメーターを問題なくサポートします。重要なネットワークデバイスを厳密に構成すると、時間を大幅に節約できます。

しかし、ハッカーであれば、中間者攻撃を簡単に実行できます。デバイスをこのポートに接続し、MACアドレスを変更して、オフィス内のすべてのデバイスがインターネットにアクセスするために作成した偽のゲートウェイを介してトラフィックを送信することを確認します。 Wiresharkまたは任意のパケット分析ソフトウェアを使用して、ハッカーは傍受したトラフィックを復号化し、興味のあるデータ、パスワード、およびその他の機密情報を入手できます。

スイッチのコマンドラインターミナルを入力し、デバイスのすべてのポートのステータスを表示するshow int briefコマンドを入力します。ここに示されているのは、24個のファストイーサネットポートと2個のギガビットポート、およびIPアドレス10.1.1.1のデフォルトの仮想ネットワークVLAN1です。次に、PC0ネットワーク設定を入力します。ここで、すべてのパラメーターを事前に入力しました。



ポートセキュリティを有効にするには、まず次の手順を実行する必要があります。コンピューターが接続されているポートであるため、interface f0 / 1コマンドを入力し、この特定のポートのセキュリティを確保したいと考えています。

ポートセキュリティ機能はアクセスポートに対してのみ有効にでき、トランクポートは別のモードで機能することを覚えておく必要があります。したがって、まず、このポートをアクセスポートに変更する必要があります。これを行うには、switchport mode accessコマンドとswitchport access vlan 1コマンドを順に入力します。次に、ポートセキュリティ機能を有効にして、switchport port-securityコマンドを入力すると、3つのオプションから選択できます。



パラメータのリストの下に記号（正しい）が表示されます。これは、switchport port-securityコマンド自体が有効なコマンドであることを意味します。つまり、コマンドを入力してEnterキーを押すと、ポートセキュリティ機能がオンになります。このポートをクリックすると、ポートセキュリティ機能が有効になっていることがわかります。ただし、これを行う前に、いくつかの設定を行う必要があります。

最初のパラメーターを使用して、MACアドレスを保護できます。これを行うには、switchportort-security mac-addressコマンドを使用します。つまり、このスイッチポートは、接続されているPC0のMACアドレスのみを使用します。
コマンドラインターミナルでコマンドPC> ipconfig / allを入力するコンピューターのIPアドレスの構成を見てみましょう。接続の名前-FastEthernet0が表示され、このコンピューターの物理アドレスは0001.6307.EEAE、つまりMACアドレスです。



パケットトレーサーの代わりにWindowsコマンドラインを使用してコンピューターのネットワーク構成を表示しようとすると、MACアドレスは6つの2文字のグループのセットとして異なる形式で表示されます。

原則として、これは同じアドレスです。MSWindowsが16進数としてそれを提示するだけであり、シスコは3つの4文字のグループを使用することを好みますが、MACアドレス自体はどちらの場合もまったく同じです。



このMACアドレスをコピーして、switchportort-security mac-addressコマンドを使用して行の最後に貼り付けます。つまり、ポートf0 / 1は特定のMACアドレスとのみ通信できます。



このアドレスを手動で入力したくない場合は、do show mac address-tableコマンドを使用して、選択したポートに接続されているデバイスのMACアドレスを確認できます。このポートは現在トラフィックを受け入れていないため、テーブルが空であることがわかります。



ここにデータを表示するには、トラフィックを生成する必要があります。ポートf0 / 1にno shutdownコマンドを指定すると、ポートが赤から緑に変わるはずです。何らかの理由でこれが発生しない場合は、show ip interface briefコマンドを使用してポートのステータスを表示し、ポートf0 / 1がまだダウン状態であることを確認してください。なぜそれがオンにならなかったのかを解明しようとします、そして私はpingコマンドでコンピューターからそれにトラフィックを転送します。接続が確立され、両方のポートが緑色のドットで示されていることがわかります。



show mac address-tableコマンドを再発行すると、コンピュータのMACアドレスがここに表示されていることがわかります。このアドレスを印刷したくない場合や、コンピューターに入力してipconfigコマンドプロンプトウィンドウからコピーしたくない場合は、ここからコピーして、目的のコマンドに貼り付けるだけです。



したがって、2つの方法があります。MACアドレスがわかっている場合は、コマンドの最後に出力するか、スイッチのMACアドレステーブルを参照して、そこから目的のアドレスをコピーします。
ただし、ポートに特定のMACアドレスを通知する最善の方法は、stickyコマンドを使用することです。これは、switchport port-security mac-address stickyのようになります。スイッチを構成し、ポートセキュリティを提供するために新しいオフィスに来たとします。正しいデバイスがスイッチ、つまりこれらのポートに接続する必要があるコンピューターに接続されていると想定します。 stickyコマンドを使用する場合、これは、既存のMACアドレスがこのポートに「スタック」していることを意味し、別のMACアドレスを持つデバイスを接続するには、このポートを手動で再構成する必要があります。

switchportort-security違反コマンドを使用するための3つのオプションがあります。シャットダウンパラメータは、違反が検出されたとき、つまり、不正なMACアドレスを持つデバイスがポートに接続されたとき、たとえば、スイッチが何も知らない新しいコンピュータがオンになったときに、ポートを切断するだけです。この場合、ネットワーク管理者が手動でポートを再起動した後にのみ、ポートは自動的にオフになり、再びオンになります。 theort-security違反関数には、保護と制限の2つのパラメーターがあります。これらのパラメータのいずれかを適用すると、トラフィックはすぐに停止されます。たとえば、保護モードで、既知のMACアドレスを持つコンピューターをスイッチポートから切断し、新しいコンピューターをそれに接続した場合、古いコンピューターをこのポートに再接続するまで、トラフィックの送信は停止されます。

制限モードも同じで、「間違った」MACアドレスを持つコンピュータが接続されている場合はポートを介したトラフィックを停止し、「正しい」コンピュータが接続された後も送信を続けます。制限モードの唯一の違いは、スイッチが接続違反を検出した後にSNMPログを送信することです。不明なMACアドレスを持つデバイスをスイッチポートに接続しようとするたびに、違反ログにそのことが記録されます。したがって、保護と制限の唯一の違いは、後者の場合、違反ログが保持されることです。デフォルトでは、ポートセキュリティ違反はシャットダウンパラメータを使用します。
