ArduinoおよびSTM32のブラウザーのグラフ

みなさん、こんにちは。データの傾向を視覚化するプロジェクト

を継続的に開発しており、ブラウザーでリアルタイムに表示する機能を追加しました。誰に面白いか聞いてみます。


プロジェクトを簡単に教えてください。「SVisual」と呼ばれるソフトウェアシステムです。
3つの部分で構成されています。

  • サーバー「SVMonitor」、これはオンラインモニターでもあります。
  • クライアント-MKからデータを転送するためのライブラリまたはスケッチ。
  • ビューアのアーカイブレコード。

このシステムについては、以前の記事で詳しく説明しています。

すぐにビデオを見て、何が起こったのか：

PCでデスクトップアプリケーションを使用する方が便利なので、電話からの表示に焦点を当てました。

ページは最新のスタック（React + Redux、Boostrapデザイン）で書かれています。
古いタブレットやスマートフォンでは機能しません。

ソフトウェアの機能図。




ここに：

  • MKはCOMポートまたはイーサネットを介してSVMonitorサーバーにデータを転送します。これもhttpサーバーです。
  • ブラウザのユーザーがページを開きます。

ページはhttp経由で送信されます。後でhttpsを実行します。現時点では、たとえばstunnelなどのサードパーティソフトウェアを使用して保護できます。

すべてが前の記事とソフトウェアマニュアルで説明されているため、サーバーの内部キッチンについては詳しく説明しませんでした。


Arduinoの使用例。


1. SVMonitorを実行し、Webサーバーの設定でIPアドレスとポートを指定します。



2.そのようなスケッチをMKに入力します。

#include <SVClient.h>

void setup() {

  bool ok = svisual::connectOfCOM("client");
}

int cnt = 0;
void loop() {

  svisual::addIntValue("dfv", cnt);

  ++cnt;
  if (cnt > 10) cnt = 0;

  bool odd_cnt = cnt % 2;

  svisual::addBoolValue("bFW", odd_cnt );

  svisual::addBoolValue("bBW", !odd_cnt );

  delay(200);
}

3.次に、SVMonitorを再起動してMKに再接続する必要があります。

4.これで、SVMonitorに接続されたMKが表示され



ます。5.ブラウザーを開き、アドレスバーのSVMonitor設定で指定されたアドレスを入力すると、次のページが表示されます。
