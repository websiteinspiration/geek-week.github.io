Vue.js：サードパーティコンポーネントのライフサイクルフック


ライフサイクルフックは、コンポーネントの非常に重要な部分です。私たちのアプリケーションは、コンポーネントが作成、マウント、更新、または破棄されたときに、コンポーネントがどうなるかを知る必要があります。

コンポーネントでは、適切なメソッドを使用してこれらのイベントをキャッチできます。次に例を示します。

  • created-コンポーネントのインスタンスが作成されます
  • マウント済み -コンポーネントのインスタンスがマウントされています
  • 更新済み -データの変更により仮想DOMが更新されました
  • destroy-コンポーネントのインスタンスは破棄されます
  • 等 ドキュメンテーション

コンポーネントのコードでは、次のようになります。

// MyComponent.vue
<template>
  <h1>Hello, {{ who }}</h1>
</template>

<script>
export default {
  name: 'MyComponent',
  data() {
    return {
      who: 'world'
    }
  },
  mounted() {
  //  -     
  }
}
</script>

良い。しかし、たとえばコンポーネントが更新されたことなど、親コンポーネントが子コンポーネントで何が起こっているかを知る必要がある場合はどうでしょうか。子コンポーネントでは、イベントを発生させ、親コ​​ンポーネントでリスナーをハングさせることができます。アプリケーションのコードでは、次のようになります。

//   ChildComponent.vue
<template>
  <div>
    <div>Count: {{ counter }}</div>
    <button @click="add">+1</button>
  </div>
</template>

<script>
export default {
  name: 'ChildComponent',
  data() {
    return {
      counter: 0
    }
  },
  updated() {
    this.$emit('updated')
  },
  methods: {
    add() {
      this.counter++
    }
  }
}
</script>


//   ParentComponent.vue
<template>
  <div>
    <ChildComponent @updated="usefulMethod"/>
  </div>
</template>

<script>
import ChildComponent from 'ChildComponent.vue'

export default {
  name: 'ParentComponent',
  components: {
    ChildComponent
  },
  data() {
    return {}
  },
  methods: {
    usefulMethod() {
    //  -       
    }
  }
}
</script>

すばらしい、子コンポーネントを更新すると、親コンポーネントが更新されたイベントをキャッチし、便利なメソッド（）メソッドが実行されます。

すべてが素晴らしいです！しかし、npmまたはgitからダウンロードしたサードパーティコンポーネントを埋め込み、親コンポーネントで何かを実行する必要がある場合（たとえば、マウントする場合）はどうでしょうか。マウントされたフックでサードパーティコンポーネントのコードにアクセスして、イベントを生成します...良い考えではありません！しかし、私はあなたがする必要がないとあなたに言ったらどうでしょう！代わりに、イベントを聞くだけです！このイベントの名前はライフサイクルのフックの名前になりますが、プレフィックス@hook:はそうなので@hook:___、次のように追加する必要があります@hook:mounted, @hook:updated @hook:destroyed。

コードでは、次のようになります。

//   ParentComponent.vue

<template>
  <div>
    <ThirdPartyComponent @hook:mounted="usefulMethod"/>
  </div>
</template>

<script>
import ThirdPartyComponent from 'vue-third-party-component' //  

export default {
  name: 'ParentComponent',
  components: {
    ThirdPartyComponent
  },
  data() {
    return {}
  },
  methods: {
    usefulMethod() {
    //  -    mounted    
    }
  }
}
</script>

これは、独自のコンポーネントとサードパーティのコンポーネントの両方を操作するための非常に便利な機能のようです。お役に立てれば幸いです。ドキュメンテーションにそれについての単語がなく、いくつかの機能はソースコードを読むことによってのみ見つけることができるのは少し奇妙です。
