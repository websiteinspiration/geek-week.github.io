クラウドとコンテナでのSSL / TLS証明書の管理-人間の作業ではない


Venafiプレゼンテーションから：手動で証明書をインストールすると、アプリケーションの継続的な統合と展開のプロセスがどのように遅くなるか

クラウドサービスとコンテナーは、Webアプリケーションを展開するための事実上の標準になっています。ただし、SSL / TLS証明書のDevOps環境への統合は、依然として複雑で低速です。多くのタスクは依然として手動で実行されており、これは開発者にとって非常に大きな負荷です。コンテナを備えた仮想環境では、ネットワーク内のマシンの数が劇的に増加し、マシン間の接続とそれらの間の通信の保護が依然として必要です。このような環境で証明書と管理手法の発行が十分に確立されていない場合、各マシンの信頼できる認証の欠如により、攻撃対象が増加します。

すべてが手動で行われる場合、開発者はしばしばセキュリティではなく、スピードとシンプルさを優先します。場合によっては、速度を上げるために、より簡単なオプションを選択します。自己署名証明書を使用して独自の認証局（CA）を作成する、弱い暗号化アルゴリズム、信頼できないルート証明書をインポートする、ルートCAおよび中間CAの秘密鍵の保護が不十分です。また、開発者はマシンとコンテナ間の通信を暗号化するためにSSL / TLSをまったく使用しない場合があります。

この問題を解決するために、継続的な統合/配信サイクル（CI / CD）に直接統合してプロセスを自動化するいくつかの新しいサービスが市場に登場しました。

これらのサービスは、セキュリティの向上と開発の生産性の向上、およびPCI-DSS、NIST、HIPAAなどのセキュリティ規制基準への準拠を提供します。また、サポートには数行のコードしか必要ありません。 2017年4月以降のこれらのサービスの1つは、情報セキュリティソリューションを専門とするVenafiによって提供されています。


Venafi CloudをパイプラインCI / CD

サービスに配置するDevOps用のVenafi Cloud-暗号化キーとデジタル証明書のインフラストラクチャを人気のある企業のDevOpsプラットフォームに便利に統合する統合クラウドサービス。同社は最近、Venafi CloudとGlobalSign PKI公開鍵インフラストラクチャの統合を発表した。

Venafi Cloudは、SSL / TLS証明書の管理に役立ちます。無料のベータ版の一部としてプラットフォームをテストできます。

主な機能：


  • すべての外部証明書の追跡。
  • 各内部証明書がインストールされている場所の継続的な監視と表示（軽量スキャナーが使用されます）。
  • 潜在的な脆弱性の特定。

  • REST API, DevOps ACME (Automated Certificate Management Environment).

Venafi Cloudは当初、Hashicorp Terraform、Hashicorp Vault、SaltStack、Ansible、Docker、Jetstack Cert-ManagerなどのDevOpsツールとの統合を提供しています。Venafi CloudとGlobalSign PKI DevOpsは、REST API、オープンソースのVCert SDK（GoとPythonで利用可能）、ACMEなど、十分に文書化された標準インターフェースを提供します。あらゆる規模の企業が、ハイブリッドインフラストラクチャと複数のクラウドで1つのマシン識別サービスを利用できるようになりました。これにより、DevOpsの速度が向上します。
