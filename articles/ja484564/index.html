ステップ3-ドライバー！


それから私は思い出しました-そうそう、あなたはまだドライバーを書かなければなりません。私はまだそれをカーネルにコミットしないことに決めました（質問がそこで提起され始めたので、私は本当に考えなければなりませんでした。ハブロフスク市民の誰かが私に助言してくれるなら、私に書いてください-私は幸せになります！）

pythonを開きます。 （最初はbashでしたが、すぐに気が変わりました。）有名なツール/ dev / hidrawを使用します。

/ dev / hidraw0、/ dev / hidraw1-これらのファイルはどのように機能しますか？単純なI / Oの場合、通常のファイルと同じように使用でき、機能します。しかし、GetFeatureとSetFeatureでは、いじくり回す必要があります...

StackOverflow（または他の誰か、私はまだ覚えていません）は、いくつかのioctlが必要であることを示唆しています。これは、HIDデバイス、端末などの通常とは異なるファイルを操作するための特別な方法です。

Ioctlはこのように動作します。あなたは彼にオープンファイル記述子（それが何であるかに興味がある、Wikipediaへのリンクがあります）、いくつかの番号、そしてバッファを与えます-その後、彼はこのバッファで何かをしてそれを返します。誇張するつもりはありません。多くのことが実装に依存しています。例を挙げましょう：0xC0114806、または既におなじみのSetFeature。それはだ

(6 << 29) | (ord('H') << 8) | (0x06 << 0) | (0x11 << 16).

まず6は、ここでファイルを書き込み、読み込みのためにオープンすることを意味します。なぜ読書がそれほど明確ではないのですか-しかし、それはそうするように書かれており、おそらくそうです。ここではOrd（ 'H'）-省略されたHID。ファイルによっては、他のことを意味するかもしれません。この場合、HID。
0x06-チーム自体。 HIDを持つ6番目のチームはSetFeatureです。最後の部分はバッファの長さです。

これらの値を使用してioctlを呼び出すだけで、出力はドライバーです。彼は働いています。
