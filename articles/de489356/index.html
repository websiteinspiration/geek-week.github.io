CI / CD unter AWS, Azure und Gitlab. Neuer Kurs von OTUS

Beachtung! Dieser Artikel ist kein Ingenieurartikel und richtet sich an Leser, die sich für eine Ausbildung im Bereich CI / CD interessieren. Wenn Sie nicht am Lernen interessiert sind, ist dieses Material höchstwahrscheinlich nicht für Sie von Interesse.



Wenn Sie ein Entwickler oder Administrator sind, der für die Einrichtung von Prozessen für kontinuierliche Integration / kontinuierliche Bereitstellung verantwortlich ist , hat OTUS eine Reihe von Kursen speziell für Sie eröffnet: einen praktischen Intensivkurs über die beliebte Methode zur kontinuierlichen Entwicklung und Bereitstellung von Software für kontinuierliche Integration und kontinuierliche Bereitstellung auf den Plattformen Amazon Web Service, Azure, GitLab und Jenkins .

Während der Schulung lernen die Schüler, wie sie den Montage- und Testprozess der Anwendung und den Installationsprozess bei den drei führenden Anbietern konfigurieren, die Architektur von Cloud-Anbietern verstehen und die Automatisierung der Codeanalyse und der Schwachstellensuche erlernen.

Als Ergebnis der Schulung erstellt jeder Schüler eine Abschlussarbeit, bei der CI / CD-Prozesse für jedes OpenSource-Projekt implementiert werden. Nach dem Training erhält jeder Schüler natürlich Materialien für alle Klassen, ein Abschlusszertifikat und vor allem die Zusammenstellung und das Testen der Anwendung, um Schwachstellen zu finden.



Natürlich ist dieser Kurs nicht jedermanns Sache. Aber wenn Sie Erfahrung haben :

  • Arbeite mit Git
  • Linux- oder Windows-Systemadministration
  • Entwicklung oder Betrieb
  • Arbeiten Sie mit einem Cloud-Anbieter zusammen

dann wartet OTUS auf dich! Sie können einen Eingangstest durchführen, um festzustellen, ob Sie über ausreichende Kenntnisse verfügen, um den CI / CD-Kurs in AWS, Azure und Gitlab abzuschließen.

Am Vorabend des Starts des CI / CD-Kurses für AWS, Azure und Gitlab veranstaltete OTUS am 17. Februar den Tag der offenen Tür. Der Lehrer sprach ausführlicher über den Lehrplan, beantwortete Fragen der Schüler und beschrieb den Lernprozess.


Ebenfalls offen ist der Zugang zu einer kostenlosen Ansicht des offenen Webinars zum Thema „Verwenden von Jenkins mit K8S“, das von Boris Nikolaev , Dozent im CI / CD- Kurs zu AWS, Azure und Gitlab, durchgeführt wurde :


Der Schulungsprozess für den Kurs "CI / CD auf AWS, Azure und Gitlab" findet im Format von Online-Webinaren statt. Während des gesamten Trainings (und es dauert 3 Monate) können die Schüler erfahrenen Lehrern Fragen stellen, die immer in Kontakt sind. Die Übungsaufgaben werden mit der Google Cloud Platform (GCP), dem Amazon Web Service und Microsoft Azure durchgeführt.

Das Kursprogramm besteht aus vier Hauptmodulen:

  1. Entwicklung in der Cloud (Code)
  2. Automatisierung von Montage und Prüfung (Continuous Integration)
  3. Installationsautomatisierung (kontinuierliche Lieferung)
  4. Zusammenfassungsmodul

Jeder von ihnen wird in den Klassen im Format von Online-Webinaren gründlich verstanden, und Hausaufgaben helfen dabei, das erworbene Wissen zu festigen, wonach Sie bei Bedarf ein detailliertes Feedback von Lehrern erhalten können.

Viele Experten bezeichnen CI / CD als eine der besten Softwareentwicklungstechniken für moderne Aufgaben. Stimmen Sie dieser Aussage zu?
