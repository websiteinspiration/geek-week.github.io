ターンベースの戦略のためにAIを書いたとき

みなさん、こんにちは。見出しから、人工知能（以下、単にAI）の作成、どの決定が行われたか、最終的に何が起こったかについて話すことは明らかだと思います。しかし、最初にあなたはあなたをスピードアップする必要があります。

ゲームはLuaプログラミング言語で記述されているため、この言語でのコードの例を示します。

AIにとって重要なゲームの詳細について説明します。

  1. ゲームはターンベースの戦略です。最初にプレーヤーが歩き、次にAIが各国に対してアクションを実行します。AIは、[次の移動]をクリックしたときにのみ機能し、別のときに何が起こるかわかりません。
  2. ゲームには、兵士を募集/移動/配備できる地図があります。AIはそれを分析し、必要な決定を行う必要があります。
  3. ゲームでは、平和を作ったり、戦争を宣言したり、非侵略協定に署名したり、同盟を結んだり、終了したりできます。AIはこれに対処できなければなりません。
  4. テクノロジーと政治制度は、プレイヤーのみが利用できます。プレイヤーとは異なり、AIボーナスはゲームの最初から変わりません。

次の移動の開始を担当する関数は次のようになります。

function next_step()
	--    
	--   ,   
	for k, v in pairs(game_data.lands) do
		calc_ai(k)
	end
	-- 
End

あなたcalc_aiはボットのすべての行動を司る魔法の機能に気づいたでしょう。興味深い詳細が1つあります。機能は国ごとに交互に実行されます。

次に、で何が起こっているかという質問に答えcalc_aiます。

function calc_ai(land)
	if land == game_data.player_land or land == "Undeveloped_land" 
	or not game_data.lands[land] then
		return
	end
	--    ,    
	for k, v in pairs(game_data.lands) do 
		local option = math.random()
		--  ,       .
		if k == game_data.player_land then
			ai_data = fixed_ai_data[game_data.difficulty].player
		else
			ai_data = fixed_ai_data[game_data.difficulty]
		end
  	end
	move_army( math.random(), land) -- 
	balance_army( math.random(), land) --  
end

AIが政治においてどのように決定を下すかを深く理解するまで、次のことに注意してください。

  1. AIが誤ってプレーヤーまたは理解できないUndeveloped_landに移動しないことを確認します。これは国のない土地です（実際、州には常にそれを所有する国がある必要があるため、Undeveloped_landは主にゲームの指定です）。
  2. AIは、プレイヤーがコントロールするかどうかに応じて、国とのやり取りにさまざまなテーブルを選択します。
  3. AIは最初に軍を移動させ、次に採用します。

fixed_ai_data-テーブル自体。変更されないために名前が付けられています（より正確には、変数は常にルートテーブルを参照します）。
ai_data-テーブル自体。ボットがプレーヤーを処理するかどうかによって変わります（変数は目的のテーブルを単に参照します）。

standard-難易度。難易度にはいくつかのレベルがあり、表によって値は異なります。

peace、war-国の状態。テーブルはpeace任意のアクションに使用され、テーブルwarは敵とのアクション専用です。

3番目のテーブルplayerは、テーブルpeaceとwar変更された値で構成されます。ご存知のように、ゲームではAIはプレイヤーとボットに対して異なる態度を持つことができます。

テーブルの値は、名前から明らかだと思います。しかし、いずれにせよ、それらのいくつかを検討します。

それまでの間、AIがゲームでどのように機能するかを理解する必要があります。ここではすべてが簡単で、状態に応じて、ボットはリストから任意の値を選択できます。上記の数値は、各イベントの発生確率です。例を考えてみましょう：
pact = 0.005-0.005の確率で、国は別の国に非侵略協定を締結することを提案することを意味します。はい、すべてが簡単です。「AIがすべてをランダムに行うことを知って、どのようにプレイできますか？」実際はそうではありません。後で分析します。

それまでの間、次の関数を見てください。

function get_answer(option,state)
	local sum_option = 0
	for k, v in pairs(state) do
		sum_option = sum_option + v
		if option < sum_option  then
			return k
		end
	end
	if sum_option > 1 then
		print("AI Error, sum_option > 1")
	end
end

option-0から1までの乱数
state-国の状態
関数はget_answer単にランダムなアクションを返します。なぜそれが必要なのか、なぜoption<行動のチャンスをチェックできないのか、説明する必要はないと思います。

この機能は、AIのすべての可能なアクションでチェックされます。

if __ and get_answer(option, ai_data.peace) == "pact" then
	_
end

すべてが簡単です。これは、完了したように思えますが、できません。

いくつかの点を明確にしましょう：

  1. 異なる可能性を持つAIは、隣人と非隣人に戦争を宣言します（war、war_neighbour）
  2. 小さな国は大きな国よりも組合に参加する可能性が高い（alliance、alliance_small）
  3. AIは羨望の的です（envy）。名前は完全に明確ではありませんが、これは国がゲームのパワーの基準によって戦争が強すぎると宣言するチャンスです。
  4. ボーナステーブルを忘れました（bonuses）。人口増加に対するボーナスや州を改善する機会など、他の国に対する態度とは関係のない値を格納します。

興味深いことはそこで終わりませんでした（もちろん、あなたが興味を持っていない限り）。国が理由もなく戦争を宣言したときは奇妙であり、この戦争の準備ができていないときはさらに奇妙です。したがって、ゲーム内のAIには戦略があります。突然の攻撃の代わりに、ほとんどの場合、国は戦略を開始することを説明します。

保存ファイルの各国には、次のフィールドがあります。

  • strategy-戦略識別子。それが起こります：
    1. retreat -国はお金を節約するために軍を削減しています。
    2. recruit -国は最大の軍隊を雇います。
  • target- 目標。これは、私たちのAIが戦争を宣言することを決定したが、戦略を開始した国です。
  • ultimatum -nil、最後通しがない場合は、最後通しに関するデータ（存在する場合）。
  • value-宣戦布告前の移動回数。申し訳ありませんが、より適切な変数名を見つけることができませんでした。

つまり、AIが戦略を開始する場合、彼は次のことを行います。

  1. 彼の軍隊を削減し、お金を節約します。
  2. 可能な限り最高の軍隊を雇う。
  3. プレイヤーに戦争を宣言します。



あるチャンスがあれば、国は最後の最後通告を出すことができ、その履行で戦争は中止されます。例：
リペツク州の軍を5,000に減らします。それ以外の場合、私たちはあなたに戦争を宣言します。

おかしなところから：国が州の軍隊を削減するように要求したときに、興味深いバグが1つ、または欠陥が1つあり、プレイヤーは、満了するまで最終通告を失いました。 AIは州を考慮せず、最後通告が尊重されない場合は戦争を宣言します。



プレイヤーをサポートするチャンス（ゴールドをプレイヤーに転送する）を担う変数は2つあります。最初のものは、それが計算された国との戦争状態を除いて、常に機能します。また、共通の敵がいる場合は、非常に重要な2番目の要素が考慮されます。

恐怖などのさまざまな変数も追加されました（AIが世界について考え始めるために敵軍がどれだけ超える必要があるか）。これはすべて、AIのアクションをより論理的で理解しやすいものにするために行われます。しかし、とにかく、これは実際にはこれが普通の変装した事故であることを知っています。

私はまた、プレーヤーがAIに彼には珍しいいくつかの機能を本当に与えていることに気づきました。たとえば、ある時点でプレイヤーが複数の国で戦争を宣言した場合、彼はまずこれが彼に対する陰謀であり、単なる偶然ではないと考えます。そして、プレイヤーが非侵略協定を締結する提案を送られたら、隣人は自分の土地を恐れていると思います。しかし、残念ながらそうではありません。現実は失望でいっぱいです。
