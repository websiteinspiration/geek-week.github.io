Modificar e remover VMs do Azure usando o PowerShell

Usando o PowerShell, engenheiros e administradores de TI automatizam com êxito várias tarefas ao trabalhar não apenas no local, mas também com infraestruturas de nuvem, em particular com o Azure. Trabalhar no PowerShell em alguns casos é muito mais conveniente e mais rápido do que no portal do Azure. Graças aos seus recursos de plataforma cruzada, o PowerShell pode ser usado para qualquer sistema operacional.



Não importa se você trabalha com Ubuntu, Red Hat ou Windows - o PowerShell ajudará a controlar os recursos da nuvem. Usando o módulo Azure PowerShell , por exemplo, você pode definir quaisquer propriedades de máquinas virtuais.



Neste artigo, veremos como você pode usar o PowerShell para redimensionar VMs na nuvem do Azure e também para remover VMs e seus objetos associados.







Importante! Não se esqueça de limpar as mãos com um desinfetante para se preparar para o trabalho:



  • Você vai precisar do Módulo PowerShell Azure - você pode baixá-lo a partir da Galeria PowerShell com o comando Install-Module Az



    .
  • Você precisa se autenticar no Azure nuvem, onde a máquina virtual está girando executando o comando Connect-AzAccount



    .


Primeiro, crie um script que redimensione a VM do Azure. Abra o VS Code e salve um novo script do PowerShell chamado Resize-AzVirtualMachine.ps1 - adicionaremos trechos de código a ele à medida que avançamos no exemplo.



Solicitamos os tamanhos de VM disponíveis



Antes de alterar o tamanho da VM, você precisa descobrir quais tamanhos são geralmente aceitáveis ​​para máquinas virtuais na nuvem do Azure. Para fazer isso, execute o comando Get-AzVMSize



.



Portanto, para a máquina virtual devvm01 do grupo de recursos dev , solicitamos todos os tipos de tamanhos permitidos: 



Get-AzVMSize -ResourceGroupName dev -VMName devvm01


(Em tarefas reais, é claro, em vez de ResourceGroupName = dev e VMName = devvm01, você especificará seus valores para esses parâmetros.)



O comando retornará uma lista como esta:







Todos esses tamanhos são possíveis para essa máquina virtual.



Redimensionar carro



Por exemplo, redimensionaremos para o novo tamanho Standard_B1ls - ele está em primeiro lugar na lista acima. (Nas tarefas reais, é claro, você escolhe o tamanho que precisa.)



  1. Primeiro, usando o comando, Get-AzVM



    obtemos informações sobre nosso objeto (máquina virtual), salvando-os em uma variável $virtualMachine



    : 



    $virtualMachine = Get-AzVM -ResourceGroupName dev -VMName devvm01
  2. Em seguida, pegamos uma propriedade desse objeto .HardwareProfile.VmSize



    e definimos o novo valor desejado: 



    $virtualMachine.HardwareProfile.VmSize = "Standard_B1ls"
  3. E agora basta executar o comando update VM - Update-AzVm



    : 



    Update-AzVM -VM devvm01 -ResourceGroupName dev
  4. Garantimos que tudo correu bem - para isso, solicitamos novamente informações sobre nosso objeto e analisamos a propriedade $virtualMachine.HardwareProfile



    : 



    $virtualMachine = Get-AzVM -ResourceGroupName dev -VMName devvm01
$virtualMachine.HardwareProfile


Se virmos Standard_B1ls lá , tudo está em ordem, o tamanho da máquina foi alterado. Você pode ir além e desenvolver o sucesso redimensionando várias VMs de uma vez usando uma matriz.



Que tal remover VMs no Azure?



Com a remoção, nem tudo é tão simples e direto quanto parece. Afinal, você precisa excluir vários recursos associados a esta máquina, incluindo:



  • Armazenar contêineres de armazenamento de diagnóstico
  • Interfaces de rede
  • Endereços IP públicos
  • Disco e blob do sistema em que seu status está armazenado
  • Discos de dados


Portanto, criaremos uma função e o nomearemos Remove-AzrVirtualMachine



- e ela excluirá não apenas a VM do Azure, mas também todas as opções acima.



Seguimos o caminho padrão e primeiro temos nosso objeto (VM) usando o comando Get-AzVm



. Por exemplo, seja a máquina WINSRV19 do grupo de recursos MyTestVMs .



Nós salvamos este objeto com todas as suas propriedades em uma variável $vm



: 



$vm = Get-AzVm -Name WINSRV19 -ResourceGroupName MyTestVMs


Excluir o contêiner com arquivos de diagnóstico de inicialização



Ao criar uma VM no Azure, o usuário também é convidado a criar um contêiner para armazenar diagnósticos de inicialização (contêiner de diagnóstico de inicialização), para que, se houver problemas com a inicialização, houver algo a que recorrer para solucionar problemas. No entanto, quando a VM é removida, esse contêiner permanece para continuar sua existência agora sem propósito. Corrija esta situação.



  1. Primeiro, descubra a qual conta de armazenamento esse contêiner pertence - para isso, precisamos encontrar a propriedade storageUri



    nas entranhas do objeto da DiagnosticsProfile



    nossa VM. Para isso, eu uso esta expressão regular: 



    $diagSa = [regex]::match($vm.DiagnosticsProfile.bootDiagnostics.storageUri, '^http[s]?://(.+?)\\.').groups[1].value
  2. Agora precisamos descobrir o nome do contêiner e, para isso, precisamos obter o ID da VM usando o comando Get-AzResource



    : 



    
if ($vm.Name.Length -gt 9) {
    $i = 9
} else {
    $i = $vm.Name.Length - 1
}
 
$azResourceParams = @{
    'ResourceName' = WINSRV
    'ResourceType' = 'Microsoft.Compute/virtualMachines'
    'ResourceGroupName' = MyTestVMs
}
 
$vmResource = Get-AzResource @azResourceParams
$vmId = $vmResource.Properties.VmId
$diagContainerName = ('bootdiagnostics-{0}-{1}' -f $vm.Name.ToLower().Substring(0, $i), $vmId)
  3. Em seguida, obtemos o nome do grupo de recursos ao qual o contêiner pertence: 



    $diagSaRg = (Get-AzStorageAccount | where { $_.StorageAccountName -eq $diagSa }).ResourceGroupName
  4. E agora temos tudo o que precisamos para remover o contêiner com o comando Remove-AzStorageContainer



    : 



    $saParams = @{
    'ResourceGroupName' = $diagSaRg
    'Name' = $diagSa
}
 
Get-AzStorageAccount @saParams | Get-AzStorageContainer | where { $_.Name-eq $diagContainerName } | Remove-AzStorageContainer -Force


Excluir VM



Agora vamos excluir a própria máquina virtual, pois já criamos uma variável $vm



para o objeto correspondente. Bem, execute o comando Remove-AzVm



: 



$null = $vm | Remove-AzVM -Force


Exclua a interface de rede e o endereço IP público



Nossa VM possui uma (ou mesmo várias) interfaces de rede (NICs) - para removê-las como desnecessárias, passe pela propriedade do NetworkInterfaces



objeto VM e exclua o comando NIC Remove-AzNetworkInterface



. Caso haja mais de uma interface de rede, usamos um loop. Ao mesmo tempo, para cada NIC, verificamos a propriedade IpConfiguration



para ver se a interface possui um endereço IP público. Se for encontrado, exclua-o com um comando Remove-AzPublicIpAddress



.



Aqui está um exemplo desse código, em que percorremos todas as NICs, as excluímos e verificamos se há um IP público. Se houver, analisaremos a propriedade PublicIpAddress



, obteremos o nome do recurso correspondente por ID e o excluiremos: 




foreach($nicUri in $vm.NetworkProfile.NetworkInterfaces.Id) {
    $nic = Get-AzNetworkInterface -ResourceGroupName $vm.ResourceGroupName -Name $nicUri.Split('/')[-1]
    Remove-AzNetworkInterface -Name $nic.Name -ResourceGroupName $vm.ResourceGroupName -Force

    foreach($ipConfig in $nic.IpConfigurations) {
        if($ipConfig.PublicIpAddress -ne $null) {
            Remove-AzPublicIpAddress -ResourceGroupName $vm.ResourceGroupName -Name $ipConfig.PublicIpAddress.Id.Split('/')[-1] -Force
        }
    }
}


Excluir a unidade do sistema



O disco do SO é um blob, cuja remoção existe um comando Remove-AzStorageBlob



- mas antes de executá-lo, você precisará definir os valores necessários para seus parâmetros. Para isso, em particular, você precisa obter o nome do contêiner de armazenamento que contém o disco do sistema e transferi-lo para esse comando junto com a conta de armazenamento correspondente. 



$osDiskUri = $vm.StorageProfile.OSDisk.Vhd.Uri
$osDiskContainerName = $osDiskUri.Split('/')[-2]
$osDiskStorageAcct = Get-AzStorageAccount | where { $_.StorageAccountName -eq $osDiskUri.Split('/')[2].Split('.')[0] }
$osDiskStorageAcct | Remove-AzStorageBlob -Container $osDiskContainerName -Blob $osDiskUri.Split('/')[-1]


Excluir status do disco do sistema de blob



Para fazer isso, como você provavelmente já adivinhou, pegamos o contêiner de armazenamento em que esse disco está armazenado e, assumindo que o blob contenha no final status



, passamos os parâmetros apropriados para o comando delete Remove-AzStorageBlob



: 



$osDiskStorageAcct | Get-AzStorageBlob -Container $osDiskContainerName -Blob "$($vm.Name)*.status" | Remove-AzStorageBlob


E, finalmente, exclua os discos de dados



Nossa VM pode ter discos com dados anexados. Se não forem necessários, também os removeremos. Primeiro, analisamos StorageProfile



nossa VM e encontramos a propriedade Uri



. Se houver vários discos, organizamos um ciclo por URI



. Para cada URI, encontraremos a conta armazenada correspondente Get-AzStorageAccount



. Em seguida, vamos analisar o URI de armazenamento para obter o nome do blob necessário e passá-lo para o comando delete Remove-AzStorageBlob



junto com sua conta. Aqui está como ficaria no código: 



if ($vm.DataDiskNames.Count -gt 0) {
    foreach ($uri in $vm.StorageProfile.DataDisks.Vhd.Uri) {
        $dataDiskStorageAcct = Get-AzStorageAccount -Name $uri.Split('/')[2].Split('.')[0]
        $dataDiskStorageAcct | Remove-AzStorageBlob -Container $uri.Split('/')[-2] -Blob $uri.Split('/')[-1]
    }
}
