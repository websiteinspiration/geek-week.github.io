Raspberry Piを有益に使用する5つの方法

こんにちはHabr。

ほとんどの人が家にRaspberry Piを持っています。私は、多くの人がそれをアイドル状態に置いていることを提案します。しかし、ラズベリーは貴重な毛皮であるだけでなく、 Linuxを備えた非常に強力なファンレスコンピューターでもあります。今日は、Raspberry Piの便利な機能を見ていきます。この機能を使用するために、コードを記述する必要はまったくありません。

興味のある方のために、詳細はカットの下にあります。記事は初心者向けです。

注：この記事は初心者を対象としていますが、IPアドレスとは何か、パテまたはその他の端末を使用してSSH経由でRaspberry Piにログインする方法、およびnanoエディターでファイルを編集する方法の少なくとも基本的な概念を持っています。実験として、今回はPythonコードでリーダーを「ロード」しないため、プログラミングはまったく行われません。以下のすべてについて、コマンドラインだけで十分です。このようなフォーマットがどれだけ必要とされているかについて、テキストの見積もりを見ていきます。

もちろん、FTPサーバーやネットワークボールのような非常に明白なことは考慮しません。以下では、多かれ少なかれ有用で独創的なものを強調してみました。

何かをインストールする前に、重要なヒント：Raspberry Piの安定した動作には、正しい電源（電話機からの無充電ではなく、2.5Aの方がブランドが優れています）とプロセッサーのヒートシンクが非常に重要です。これがないと、Raspberryがフリーズしたり、ファイルコピーエラーなどが発生したりすることがあります。このようなエラーは、CPU負荷のピーク時やSDカードに大きなファイルが書き込まれているときなど、まれにしか発生しません。

コンポーネントをインストールする前に、システムを更新することをお勧めします。そうしないと、aptコマンドの古いアドレスが機能しない可能性があります。

sudo apt-get update

これで、インストールと構成の準備ができました。

1. WiFiアクセスポイント


Raspberry Piは簡単にワイヤレスアクセスポイントに変えることができ、他に何かを購入する必要はありません。すでにWiFiが「搭載」されています。これを行うには、hostapd（ホストアクセスポイントデーモン、アクセスポイントサービス）とdnsmasq（DNS / DHCPサーバー）の2つのコンポーネントをインストールします。

dnsmasqとhostapdをインストールします。

sudo apt-get install dnsmasq hostapd

Raspberry PiがWiFiネットワーク上で持つ静的IPアドレスを設定します。これを行うには、コマンドsudo nano /etc/dhcpcd.confを入力して、dhcpcd.confファイルを編集します。次の行をファイルに追加します。

interface wlan0
  static ip_address=198.51.100.100/24
  nohook wpa_supplicant

ご覧のように、WiFiネットワークでは、Raspberry Piのアドレスは198.51.100.100になります（サーバーが動作するかどうかを覚えておくことが重要です。アドレスはブラウザーに入力する必要があります）。

次に、IP転送をアクティブ化する必要があります。そのために、sudo nano /etc/sysctl.confコマンドを実行し、net.ipv4.ip_forward = 1行のコメントを外します。

次にDHCPサーバーを構成する必要があります-DHCPサーバーは接続されたデバイスにIPアドレスを配布します。コマンドsudo nano /etc/dnsmasq.confを入力して、次の行を追加します。

interface=wlan0
dhcp-range=198.51.100.1,198.51.100.99,255.255.255.0,24h

ご覧のとおり、接続されたデバイスのIPアドレスは198.51.100.1 ... 198.51.100.99の範囲になります。

最後に、Wi-Fiを設定します。/ etc / default / hostapdファイルを編集し、そこにDAEMON_CONF = "/etc/hostapd/hostapd.conf"という行を入力します。次に、コマンドsudo nano /etc/hostapd/hostapd.confを入力して、hostapd.confファイルを編集します。
アクセスポイントのパラメータを入力します。

interface=wlan0
driver=nl80211
ssid=Raspberry Pi
hw_mode=g
channel=7
wmm_enabled=0
macaddr_acl=0
auth_algs=1
ignore_broadcast_ssid=0
wpa=2
wpa_passphrase=12345678
wpa_key_mgmt=WPA-PSK
wpa_pairwise=TKIP
rsn_pairwise=CCMP

パラメータ「ssid」（アクセスポイント名）、「wpa_passphrase」（パスワード）、「channel」（チャネル番号）、「hw_mode」（動作モード、a = IEEE 802.11a、5 GHz、b = IEEE 802.11）に注意することが重要ですb、2.4 GHz、g = IEEE 802.11g、2.4 GHz）。残念ながら、自動チャネル選択はないため、最も使用率の低いWiFiチャネルを自分で選択する必要があります。

重要：このテスト例では、パスワードは12345678です。実際のアクセスポイントでは、より複雑なものを使用する必要があります。辞書にパスワードを列挙するプログラムがあり、単純なパスワードを持つアクセスポイントがクラックされる可能性があります。さて、現代の法律の下で部外者とインターネットを共有することは、失敗する可能性があります。

すべての準備が整ったら、すべてのサービスをアクティブ化できます。

sudo systemctl unmask hostapd
sudo systemctl enable hostapd
sudo systemctl start hostapd
sudo systemctl reload dnsmasq

これで、ネットワークのリストに新しいWiFiアクセスポイントが表示されます。ただし、インターネットが表示されるようにするには、イーサネットからWLANへのパケット転送をアクティブにする必要があります。これには、sudo nano /etc/rc.localコマンドを入力して、iptables構成行を追加します。

sudo iptables -t nat -A  POSTROUTING -o eth0 -j MASQUERADE

それでおしまい。Raspberry Piを再起動し、すべてが正しく行われた場合は、アクセスポイントを確認して接続できます。



ご覧のとおり、速度はそれほど悪くなく、そのようなWiFiを使用できます。

ちなみに、ちょっとしたヒント：sudo raspi-configコマンドを実行して、Raspberry Piのネットワーク名を変更できます。デフォルトでは、（驚き:) raspberrypiです。これはおそらくよく知られています。ただし、この名前がローカルネットワークでも使用できることを誰もが知っているわけではありませんが、「。local」を追加する必要があります。たとえば、putty pi@raspberrypi.localと入力して、SSH経由でRaspberry Piにログインできます。真実があります。注意点が1つあります。これはWindowsとLinuxでは機能しますが、Androidでは機能しません。IPアドレスを手動で入力する必要があります。

2.メディアサーバー


Raspberry Piにメディアサーバーを作成する方法は1001ありますが、私は最も単純なものだけを検討します。 MP3ファイルのお気に入りのコレクションがあり、すべてのメディアデバイスがローカルネットワークで利用できるようにしたいとします。 Raspberry PiにMiniDLNAサーバーを配置します。これを行うことができます。

インストールするには、コマンドsudo apt-get install minidlnaを入力します。次に、コマンドsudo nano /etc/minidlna.confを入力して構成を構成する必要があります。そこで、ファイルへのパスを示す1行のみ追加する必要があります。media_dir= / home / pi / MP3（もちろん、パスは異なる場合があります）。ファイルを閉じた後、サービスを再起動します

。sudosystemctl restart minidlna

すべてが正しければ、ローカルネットワーク上に既製のメディアサーバーが作成され、そこからWiFiテーブルトップラジオまたはAndroid VLC-Playerを介して音楽を再生できます。



ヒント：WinSCPを使用してRaspberry Piにファイルをダウンロードすることは非常に便利です。 RPiフォルダーの操作は、ローカルフォルダーの操作と同じくらい簡単です。



3. SDRレシーバー


RTL-SDRまたはSDRPlayレシーバーがある場合は、GQRXまたはCubicSDRプログラムを使用してRaspberry Piで使用できます。これにより、24時間体制でも機能するスタンドアロンのサイレントSDRレシーバーを使用できます。

お詫び申し上げます。RTL-SDRまたはSDRPlayを使用すると、最大1 GHz（少しでも高い）の周波数でさまざまな無線信号を受信することが可能です。たとえば、通常のFMラジオだけでなく、パイロットや他のサービス間の交渉も聞くことができます。ちなみに、Raspberry Piを使用するラジオ愛好家は、WSPR信号や他のデジタルモードを非常によく受信、デコード、送信できます。

4.「スマートホーム」のサーバー


自宅をより「スマート」にしたい人は、無料のOpenHABプログラムを使用できます。



これは単なるプログラムではなく、さまざまなプラグイン、さまざまなデバイス（Z-Wave、Philips Hueなど）を制御できるスクリプトを備えたフレームワーク全体です。興味のある方は、about.site https://www.openhab.orgでさらに詳しく学ぶことができます。

ちなみに、ここでは「スマートホーム」について説明しているため、さまざまなローカルデバイスで使用できるMQTTサーバーがRaspberry Piでうまく機能する可能性があります。

航空愛好家で、FlightRadarのカバレッジが不十分な地域に住んでいる場合は、受信機を設置することで、コミュニティとすべての旅行者を支援できます。必要なのは、RTL-SDRレシーバーとRaspberry Piだけです。ボーナスとして、FlightRadar24 Proアカウントに無料でアクセスできます。
