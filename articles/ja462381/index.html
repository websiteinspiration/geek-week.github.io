したがって、深いスキームは設計プロセスを容易にします。しかし、それらは設計だけでなく支援にもなります。非常に浅い回路でいくつかの関数を計算するには、深いものよりも指数関数的に多くの要素を使用する必要があるという数学的証拠があります。例えば、科学論文の有名なシリーズがあります1980年代、ビットセットのパリティの計算には、浅い回路を備えた指数関数的に多数のゲートが必要であることが示されていました。一方、ディープスキームを使用する場合、小さなスキームを使用してパリティを計算する方が簡単です。ビットのペアのパリティを計算し、その結果を使用してビットのペアのパリティを計算するなど、一般的なパリティにすばやく到達します。したがって、深いスキームは浅いスキームよりもはるかに強力です。

これまでのところ、この本では、クレイジークライアントのリクエストと同様に、ニューラルネットワーク（NS）へのアプローチを使用しています。私たちが取り組んだほとんどすべてのネットワークには、ニューロンの単一の隠れ層（および入力層と出力層）がありました。



これらの単純なネットワークは非常に便利であることがわかりました。前の章では、そのようなネットワークを使用して、98％を超える精度で手書きの数字を分類しました。ただし、多くの隠れ層を持つネットワークの方がはるかに強力であることは直感的に明らかです。



このようなネットワークでは、ブールスキームの場合と同様に、中間層を使用して多くのレベルの抽象化を作成できます。たとえば、パターン認識の場合、第1層のニューロンは、顔、第2層のニューロン-顔から作成された三角形や四角形などのより複雑な形状を認識することを学習できます。次に、3番目のレイヤーはさらに複雑な形状を認識できるようになります。等。抽象化のこれらの多くの層は、複雑なパターンを認識する問題を解決する際に、深いネットワークに説得力のある利点を与える可能性があります。さらに、回路の場合と同様に、深いネットワークには本質的に浅いネットワークよりも多くの機能があることを確認する理論的な結果があります。

これらのディープニューラルネットワーク（GNS）をどのようにトレーニングしますか？この章では、トレーニングアルゴリズムの中の主力である確率的勾配後方伝搬降下法を使用して、STSのトレーニングを試みます。ただし、問題が発生します。GNSは、（それを超えるとしても）浅いGNSよりもうまく機能しません。

この失敗は、上記の議論に照らして奇妙に思われます。しかし、STSをあきらめるのではなく、問題を深く掘り下げて、STSのトレーニングが難しい理由を理解しようとします。この問題を詳しく見ると、STSの異なるレイヤーが非常に異なる速度で学習していることがわかります。特に、ネットワークの最後の層が適切にトレーニングされている場合、最初の層はトレーニング中に行き詰まり、ほとんど何も学習しません。そしてそれは不運だけではありません。勾配ベースの学習手法の使用に関連する、学習を遅くする基本的な理由を見つけます。

この問題を深く掘り下げてみると、逆の現象も発生する可能性があることがわかります。初期の層はうまく学習でき、後の層は動けなくなる可能性があります。実際、深い多層NSの勾配降下訓練に関連する内部の不安定性を発見します。そして、この不安定性のために、初期または後期の層のいずれかがトレーニングで行き詰まることがよくあります。

これはすべてかなり不快に聞こえます。しかし、これらの困難に突き当たって、STSの効果的なトレーニングのために何をする必要があるかについてのアイデアを開発し始めることができます。したがって、これらの研究は、画像認識の問題にアプローチするためにディープラーニングを使用する次の章の良い準備になります。

グラデーションのフェージングの問題


では、深いネットワークをトレーニングしようとすると何がうまくいかないのでしょうか？

この質問に答えるために、隠れ層を1つだけ含むネットワークに戻ります。いつものように、トレーニングと実験のサンドボックスとしてMNIST桁分類問題を使用します。

コンピューターでこれらすべての手順を繰り返す場合は、Python 2.7をインストールし、Numpyライブラリーと、リポジトリーから取得できるコードのコピーが必要です。

git clone https://github.com/mnielsen/neural-networks-and-deep-learning.git

データとコードをダウンロード するだけで、gitなしで実行できます。srcサブディレクトリに移動し、PythonシェルからMNISTデータをロードします。

>>> import mnist_loader
>>> training_data, validation_data, test_data = \
... mnist_loader.load_data_wrapper()

ネットワークを構成します。

>>> import network2
>>> net = network2.Network([784, 30, 10])

このようなネットワークの入力レイヤーには784個のニューロンがあり、入力画像の28×28 = 784ピクセルに対応しています。数値MNIST（ '0'、 '1'、 '2'、...、 '9'）の10の可能な分類オプションに対応する30の非表示ニューロンと10の週末を使用します。

学習速度η= 0.1および正則化パラメーターλ= 5.0で、一度に10個のトレーニング例のミニパッケージを使用して、ネットワーク全体を30エポックにトレーニングしてみましょう。トレーニング中、validation_dataを通じて分類の正確さを追跡します。

>>> net.SGD(training_data, 30, 10, 0.1, lmbda=5.0, 
... evaluation_data=validation_data, monitor_evaluation_accuracy=True)

類似の設定を使用した以前の結果と比較して、分類精度は96.48％（つまり、数値はリリースごとに異なります）になります。

同じく30個のニューロンを含む別の非表示レイヤーを追加して、同じハイパーパラメーターでネットワークをトレーニングしてみましょう。

>>> net = network2.Network([784, 30, 30, 10])
>>> net.SGD(training_data, 30, 10, 0.1, lmbda=5.0, 
... evaluation_data=validation_data, monitor_evaluation_accuracy=True)

分類精度は96.90％に向上します。それは刺激的です-深さのわずかな増加が役立ちます。30個のニューロンからなる別の隠れ層を追加しましょう：

>>> net = network2.Network([784, 30, 30, 30, 10])
>>> net.SGD(training_data, 30, 10, 0.1, lmbda=5.0, 
... evaluation_data=validation_data, monitor_evaluation_accuracy=True)

役に立たなかった。結果は、元の浅いネットワークに近い値である96.57％にまで低下しました。そして、別の隠しレイヤーを追加すると：

>>> net = network2.Network([784, 30, 30, 30, 30, 10])
>>> net.SGD(training_data, 30, 10, 0.1, lmbda=5.0, 
... evaluation_data=validation_data, monitor_evaluation_accuracy=True)

その後、分類精度は再び96.53％に下がります。統計的には、この低下はわずかであると思われますが、それについて良いことは何もありません。

この動作は奇妙に見えます。追加の隠れ層は、ネットワークがより複雑な分類関数を学習し、タスクにうまく対処するのに役立つはずだと直感的に思われます。もちろん、最悪の場合、追加のレイヤーは何もしないため、結果は悪化しないはずです。ただし、これは起こりません。

では、何が起こっているのでしょうか。追加の非表示レイヤーが原則的に役立つ可能性があり、問題は、トレーニングアルゴリズムが重みとオフセットの正しい値を見つけられないことであると仮定しましょう。アルゴリズムの何が問題で、どのように改善するかを理解したいと思います。

問題の原因を理解するために、ネットワークの学習プロセスを視覚化してみましょう。以下に、ネットワークの一部を構築しました[784,30,30,10]。2つの隠れ層があり、それぞれに30個の隠れニューロンがあります。図では、各ニューロンには、ネットワークを学習するプロセスの変化率を示すバーがあります。大きなバーはニューロンの重みと変位が急速に変化することを意味し、小さなバーはゆっくり変化することを意味します。より正確には、バーはニューロンの∂C/∂b勾配、つまり変位に関する値の変化率を示します。では、第2章この勾配値は、トレーニング中の変位の変化率だけでなく、入力ニューロンの重みの変化率も制御することがわかりました。これらの詳細を思い出せなくても心配しないでください。これらのバーは、ネットワークの学習中にニューロンの重みと変位がどれだけ速く変化するかを示していることだけを覚えておく必要があります。

フェージンググラデーションの問題があるのはなぜですか？それを避ける方法はありますか？ STSをトレーニングする場合、どのように対処しますか？実際、代替案は彼女にとって魅力的ではないように見えますが、すぐにそれが避けられないことがすぐにわかります。最初のレイヤーでは、勾配がはるかに大きくなることがあります。これはすでに爆発的な勾配の成長の問題であり、勾配が消える問題と同じくらいそれで良いです。一般に、STSの勾配は不安定であり、爆発的な成長または最初の層での消失の傾向があることがわかります。この不安定性は、勾配GNSトレーニングの基本的な問題です。これは私たちが理解し、おそらくそれを何らかの方法で解決する必要があることです。

フェージング（または不安定）勾配に対する反応の1つは、これが実際に深刻な問題であるかどうかを考えることです。 NSから少し気を散らして、1つの変数から関数f（x）を数値的に最小化しようとしていると想像してください。導関数f ′（x）が小さければ、すばらしいと思いませんか？これは、私たちがすでに極端に近いことを意味しているのではないでしょうか？同様に、GNSの最初の層の小さな勾配は、重みと変位を大幅に調整する必要がなくなったことを意味しませんか？

もちろん違います。ネットワークの重みとオフセットをランダムに初期化したことを思い出してください。元の重みとブレンドがネットワークに必要なものとうまく機能する可能性はほとんどありません。具体的な例として、ネットワークの重みの最初の層[784,30,30,30,10]を考えます。これは、MNISTの数を分類します。ランダムな初期化とは、最初のレイヤーが着信画像に関するほとんどの情報を排出することを意味します。後の層が慎重にトレーニングされたとしても、単に情報が不足しているために、着信メッセージを特定することは非常に困難です。したがって、最初の層をトレーニングする必要がないだけだと想像することは絶対に不可能です。STSをトレーニングする場合、勾配が消える問題を解決する方法を理解する必要があります。
