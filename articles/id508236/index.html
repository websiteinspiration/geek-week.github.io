Aku membawakanmu grafonia! Bagaimana jaringan saraf dapat meningkatkan resolusi pada game-game lama menjadi HD





Artikel ini adalah terjemahan gratis artikel saya di Media .



Saya, sebagai seorang anak, saya suka bermain di komputer. Saya menemukan permainan kecil pada kaset ZS Spectrum cukup kecil, tetapi penemuan penuh warna tahun 90-an menjadi penemuan nyata. Kemudian mayoritas genre yang ada muncul. Setelah sedikit nostalgia, saya memutuskan untuk mengingat masa muda saya dan menjalankan salah satu game lama di emulator Dosbox dan secara tidak menyenangkan terkena piksel raksasa dan resolusi rendah. Meskipun grafis lama pixel besar mungkin memiliki daya tarik sendiri, banyak sekarang tidak senang dengan kualitas ini.



Untuk meningkatkan resolusi dan menghilangkan angularitas dalam game, berbagai algoritma postprocessing dan anti-aliasing saat ini digunakan (untuk lebih jelasnya, lihat di sinizen.yandex.ru/media/id/5c993c6021b68f00b3fe919c/kak-rabotaet-sglajivanie-v-kompiuternyh-igrah-5c9b3e76d82a083cc9a0f1a7 ), tetapi algoritma anti-aliasing mengarah pada semua preferensi yang penuh kebencian.



Pada saat yang sama, meningkatkan kualitas grafis seringkali penting bagi gamer. Penggambaran tekstur untuk game Heroes 3 HD membawa Ubisoft sekitar enam bulan pada tahun 2014 dan menyebabkan gelombang minat pada game ini. Dari berita terbaru - rilis ulang CNC pertama dalam grafis hd. Secara terperinci, Anda dapat melihat minat yang tumbuh pada diterbitkan kembali dalam grafis hd pada tren google .



Tidak untuk setiap game, masuk akal untuk merepotkan dan menerbitkan ulang dengan kualitas tinggi - menggambar ulang tekstur adalah latihan yang mahal. Tetapi Anda dapat mencoba meningkatkan kualitas grafis menggunakan teknologi superresolution. Gagasan superresolusi adalah untuk meningkatkan resolusi gambar dengan menggambar piksel yang hilang di jaringan saraf berdasarkan data yang tersedia. Hasil yang mengesankan sekarang telah dicapai yang dikaitkan dengan adegan meme-shot untuk meningkatkan gambar dari pembuat film bladerunner





Teknologi Superresolution meningkatkan persepsi visual gambar, misalnya, github.com/tg-bomze/Face-Depixelizer , tetapi membawa informasi baru ke gambar. Dan dapat digunakan untuk meningkatkan kualitas film:





Namun, sebagian besar algoritma bersifat intensif sumber daya, dan saya ingin membuat skrip untuk meningkatkan permainan secara real time.



Sedikit teori



Semua yang berikut ini akan berlaku untuk jaringan saraf convolutional - subtipe jaringan saraf yang digunakan untuk bekerja dengan gambar. Pertama, mari kita lihat bagaimana jaringan saraf bekerja untuk menyelesaikan masalah superresolusi. Tugas ini sangat mirip dengan solusi dari masalah auto encoder. Penting untuk mengirimkan gambar ke input jaringan, untuk mendapatkan gambar yang sama di output. Namun, autoencoder biasanya digunakan untuk menyelesaikan masalah kompresi data yang efisien, oleh karena itu, fitur arsitektur mereka adalah Bottleneck - bottleneck, yaitu lapisan jaringan dengan sejumlah kecil neuron. Kehadiran lapisan seperti itu membuat bagian-bagian yang tersisa belajar untuk pengkodean dan penguraian informasi yang efisien. Untuk melatih jaringan superresolusi, resolusi gambar berkualitas tinggi pertama-tama sengaja dikurangi dan dimasukkan ke input jaringan saraf. Outputnya diharapkan gambar sumber dalam kualitas tinggi.



Tugas superresolution mendefinisikan arsitektur jaringan yang digunakan:

biasanya mereka memiliki koneksi antara input dan output data (lewati-koneksi), yang sangat mempercepat pembelajaran. Ukuran piksel dari data input meningkat dan ditambahkan ke output jaringan convolutional. Jadi, sebenarnya, Anda tidak perlu melatih jaringan untuk mengubah gambar menjadi hampir sama. Anda hanya perlu melatihnya untuk menggambar perbedaan antara yang diperbesar dengan meningkatkan ukuran piksel dan gambar nyata. Gagasan memiliki lompatan koneksi dari berbagai tingkatan dan melalui sejumlah lapisan yang berbeda sangat efektif dan telah menyebabkan munculnya kelas jaringan Residual Networks. Saat ini, koneksi semacam itu digunakan di hampir semua arsitektur populer. Tinjauan yang baik tentang arsitektur canggih untuk memecahkan masalah superresolusi dapat ditemukan di sini.. Tugas saya adalah membuat jaringan saraf untuk menyelesaikan masalah superresolusi secara real time.



Pertama, saya memilih arsitektur EDSR dengan 4 blok (biasanya lebih dari 16 lapisan blok digunakan) ResNet dengan peningkatan resolusi 4 kali lipat. Di sini, saya menggunakan sebagian perkembangan dari github.com/krasserm/super-resolution dan pembuat data dari proyek yang sama.



Arsitektur jaringan umum ditunjukkan dalam diagram. Setiap blok adalah gambar X * Y * N, di mana lebarnya sesuai dengan jumlah saluran. Transisi - sesuai dengan konvolusi 3x3 (dalam kasus blok res diikuti oleh aktivasi ReLU untuk non-linearitas). Langkah peningkatan skala - peningkatan dimensi karena perataan saluran.







Jika Anda melihat dengan cermat pada jaringan sekecil itu, Anda akan melihat bahwa blok terakhir adalah linier dan adanya beberapa langkah peningkatan tidak dibenarkan. Anda dapat menggantinya dengan satu lapisan dengan filter jauh lebih sedikit tanpa kehilangan kualitas tanpa kehilangan kualitas jaringan:







Ternyata jaringan seperti itu mengatasi tugas superresolusi dengan cara yang hampir sama, tetapi jauh lebih cepat dan dapat diluncurkan untuk memproses resolusi video kecil secara real time .



Foto berkualitas tinggi telah diunduh dari data.vision.ee.ethz.ch/cvl/DIV2K . Pada prinsipnya, Anda dapat menggunakan foto apa pun untuk pelatihan. 



import tensorflow as tf
from model import resolve_single
from model. rtsr import rtsr
import numpy as np
from utils import load_image, plot_sample
from data import DIV2K
from tensorflow.keras.optimizers.schedules import PiecewiseConstantDecay
# Model creation
rtsr_model= rtsr(scale=4, num_res_blocks=4)
#Data loading and generator creation
train_loader = DIV2K(scale=4,             
                     downgrade='bicubic',
                     subset='train')      
                     


train_ds = train_loader.dataset(batch_size=16,
                                random_transform=True,
                                repeat_count=None)  
valid_loader = DIV2K(scale=4,  
                     downgrade='bicubic',
                     subset='valid')  
                     

valid_ds = valid_loader.dataset(batch_size=16,           # use batch size of 1 as DIV2K images have different size
                                random_transform=True, # use DIV2K images in original size
                                repeat_count=40)         # 1 epoch
#Saving checkpoints during training
cbcks=tf.keras.callbacks.ModelCheckpoint('callback_x4{epoch:02d}-{loss:.2f}', monitor='loss', verbose=0, save_best_only=False)

learning_rate=PiecewiseConstantDecay(boundaries=[20000], values=[1e-4, 5e-5])
#model compilation before training
rtsr_model.compile(loss='mae', optimizer=tf.keras.optimizers.Adam(learning_rate=learning_rate))
#Training
rtsr_model.fit(train_ds,validation_data=valid_ds, steps_per_epoch=3000, epochs=100, validation_steps=10,callbacks=[cbcks])
#saving weights
rtsr_model.save_weights('weights/ myedsr-lastdiv2_4block-x4_weights.h5')


Outputnya adalah jaringan saraf yang relatif kecil (2,6 MB) dengan arsitektur sederhana. Pada saat yang sama, cek memberikan perbedaan yang halus dari jaringan 16 blok pra-dilatih:







Gambar di sebelah kiri adalah yang asli, di sebelah kanan ditingkatkan menggunakan edsr 16-layer, di tengah - menggunakan rtsr. Perbedaan antara 2 dan 3 tidak signifikan dari sudut pandang saya.

Bagaimana memulai?



Untuk bekerja dengan cepat, Anda memerlukan kartu video dengan dukungan untuk cuda dan cudnn (https://developer.nvidia.com/cuda-gpus) dan pustaka cuda / cudnn yang diinstal. Anda perlu menginstal python 3.7 dan tensorflow (versi lebih tinggi dari 2.0 dengan dukungan GPU). Ini bisa menjadi tugas yang menakutkan dan masalah kompatibilitas mungkin muncul .



Cara termudah adalah dengan menginstal distribusi Anaconda (https://www.anaconda.com/products/individual) dan kemudian menginstal tensorflow-gpu menggunakan conda: 



conda install tensorflow-gpu


Jika tidak berhasil karena konflik, maka Anda dapat mencoba 



conda install cudnn
pip install tensorflow-gpu


harus bekerja.



Perpustakaan lain dapat diinstal menggunakan pip: 



pip install opencv-python
pip install pywin32
pip install mss


Selanjutnya, Anda perlu menjalankan skrip dengan perintah: 



python superres.py


Jendela tangkap dikontrol menggunakan tombol wsad dan ijkl (resize). q- menutup jendela. 0 - nyalakan / matikan mode resolusi tinggi. Angka 1 dan 2 - mode gambar (2 secara default berarti bahwa permainan menggunakan peningkatan karena peningkatan piksel sebanyak 2 kali).
