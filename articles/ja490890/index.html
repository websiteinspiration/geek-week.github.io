Herokuとは何ですか？なぜ彼が必要なのですか？


Herokuは、多くのプログラミング言語をサポートするクラウドベースのサービスとしてのプラットフォーム（PaaS）プラットフォームです（そしてそれは本当に誇りであり、際立っています）。Herokuの歴史は2007年に始まり、Rubyは最初のプログラミング言語でした。現在、Java、Node.js、Scala、Clojure、Python、PHP、Goをサポートしています。

なぜこのクラウドが必要なのですか？安価なホスティングを購入できます


はい、ホスティングでこのサービスがサポートされている場合は、ホスティングを購入してそこにNode.jsサーバーをインストールできます。ただし、クラウドプラットフォームには、弾力性や消費量測定などの品質があります。多くのユーザーがサービスにアクセスすると、プラットフォーム（またはプラットフォームが提供するツールを使用しているユーザー）がフローを自動的に拡大または縮小します。消費の会計処理とは、必要なリソースに対してのみ支払うことを意味します。クラウドプラットフォームには他にも多くの利点があります。完全なリストはこちらにあります。さて、私たちは直接展開に行きます。

ファビコンを作成する

データベースで作業がないのに 、Node.jsサーバーが必要なのはなぜですか？サーバーを使用して、ファビコンと残りのすべてのコードを提供します。当社ではリアクトアプリケーションに行くのパブリックフォルダおよびそこからテンプレートのfavicon.icoを削除します。ここからアイコンを取り出し、パブリックフォルダに転送します。

Expressサーバーを作成する

次に、ビルドを処理するExpressサーバーを作成します。アプリケーションフォルダーに直接、server.jsバックエンドの作業が展開されるファイルを作成します。

次のように記述します。

 const express = require('express');
 const favicon = require('express-favicon');
 const path = require('path');
 const port = process.env.PORT || 8080;
 
 //           
 const app = express();
 app.use(favicon(__dirname + '/build/favicon.png')); 
 
 //    
 app.use(express.static(__dirname));
 app.use(express.static(path.join(__dirname, 'build')));
 
 //  
 app.get('/ping', function (req, res) {
  return res.send('pong');
 });
 
 // html
 app.get('/*', function (req, res) {
 res.sendFile(path.join(__dirname, 'build', 'index.html'));
 });
 app.listen(port);

ここでは、express、express-favicon、およびpathパッケージを使用しているため、それらをインストールする必要があります。

 yarn add express express-favicon path -D

ではpackage.json、開始コマンドを次のように変更します。

 "start": "node server.js",

ビルドを実行する

次に、次のコマンドを使用してアプリケーションをビルドする必要があります。

 yarn build

アプリケーションが正しく動作していることをテストするとよいでしょう。これを行うには、yarn startそれがどのように正しく機能するかを採用して評価します。
