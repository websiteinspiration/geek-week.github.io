パート1.巻雲


ゲームがオープンスペースで行われる場合、世界の信頼性を決定する要因の1つは空です。考えてみてください。ほとんどの場合、空は文字通り画面全体の約40-50％を占めます。空は美しいグラデーション以上のものです。星、太陽、月、そして最後に雲があります。

現在の傾向はレイマーチングを使用した雲のボリュームレンダリング（この記事を参照）にあるようですが、The Witcher 3の雲は完全にテクスチャベースです。私は以前にそれらをすでに調べましたが、それらを使用すると、すべてが当初の予想よりも複雑であることがわかりました。私の一連の記事をフォローすると、Blood and Wine DLCとゲームの残りの部分に違いがあることがわかります。そして、ご想像のとおり、DLCでのクラウドの処理にはいくつかの変更点があります。

ウィッチャー3には雲の層がいくつかあります。天候に応じて、それは巻雲、高積雲、層状雲のファミリーからのいくつかの雲（たとえば、嵐の間）のみになる可能性があります。結局、雲はまったくないかもしれません。

一部のレイヤーは、それらをレンダリングするために使用されるテクスチャーとシェーダーの点で異なります。明らかに、これはピクセルシェーダーのアセンブラコードの複雑さと長さに影響します。

このすべての多様性にもかかわらず、ウィッチャー3で雲をレンダリングするときに観察できるいくつかの一般的なパターンがあります。最初に、それらはすべてプロアクティブなパスでレンダリングします。これは完璧な選択です。それらはすべてミキシングを使用します（以下を参照）。これにより、別のレイヤーが空を覆う方法を制御するのがはるかに簡単になります。これは、ピクセルシェーダーからのアルファ値の影響を受けます。


さらに興味深いことに、一部のレイヤーは同じパラメーターで2回レンダリングされます。

コードを確認した後、（1）完全なリバースエンジニアリングを実行する可能性が最も高い、（2）すべての側面を理解するために、最も短いシェーダーを選択しました。

うわー、それは何ですか？

cb [7] .xおよびcb [8] .xの値は2000.0および7000.0です。

これはlinstep関数を使用した結果であることがわかります。

彼女は3つのパラメーターを受け取ります：最小/最大 -間隔およびv-値。

これは以下のような作品：場合vが区間[ている最小 - 最大 ]、区間[0.0 - 1.0]の補間、線形、関数リターンします。一方、vが範囲外の場合、linstepは0.0または1.0を返します。

簡単な例：

linstep( 1000.0, 2000.0, 999.0) = 0.0
linstep( 1000.0, 2000.0, 1500.0) = 0.5
linstep( 1000.0, 2000.0, 2000.0) = 1.0

つまり、エルミート補間の代わりに線形が実行されることを除いて、HLSL のスムースステップに非常に似ています。

LinstepはHLSLの機能ではありませんが、非常に便利です。ツールキットに入れておく価値はあります。

 // linstep:  
 //  
 // Returns a linear interpolation between 0 and 1 if t is in the range [min, max]   
 // if "v" is <= min, the output is 0  
 // if "v" i >= max, the output is 1  
   
 float linstep( float min, float max, float v )  
 {  
   return saturate( (v - min) / (max - min) );  
 }

シェーダーは、ピクセルが空にあるかどうかを判断することから始まります。ピクセルが空にある場合（深度== 1.0）、シェーダーは黒を返します。ピクセルがシーン内にある場合（深度<1.0）、深度バッファー（7〜11行目）を使用してワールド内の位置を再作成し、フォグの計算を続けます。

4.まとめ


主な問題は、不透明度機能のある部分にあります。それをデコードした後、他のすべては非常に簡単に理解できます。

ピクセルシェーダーは少し複雑すぎると上記で述べました。実際には、r2.xに格納されている不透明度（x）関数の値（49行目から）のみが重要です。ただし、アセンブラコードのopacity関数は、3つの追加の変数、minRange（r2.y）、maxRange（r2.z）、およびvalue（r2.w）を作成します。これらはすべて、不透明度（x）が使用されていないときに不透明度を計算するために使用されるパラメーターです

lerp( minRange, maxRange, smoothstep(0, 1, value) );

。実際、最終的な不透明度の値は、55行目の条件付きブランチで取得されます（入力値がxの場合）。[controlPoint0-controlPoint3]の範囲にある場合、これは不透明度関数が使用されることを意味するため、r2.xが選択されます。それ以外の場合、xが間隔の外にあるとき、不透明度はr0.xから、つまり上記の方程式に従って計算されます。

間隔[controlPoint0-controlPoint3]の外側の数ピクセルをデバッグしたところ、最終的な不透明度は常にゼロでした。
