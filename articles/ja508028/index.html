プロトタイプのIT製品とは何ですか？どんな感じですか？



プロトタイプは、インターフェースからのユーザーインタラクションをシミュレートする動的な製品モデルであり、これを使用すると、顧客または対象読者の代表からフィードバックを迅速かつ安価に受け取ることができます。







プロトタイプは、サイト（モーションキャプチャ）または配線フレーム（将来の製品のブロックのセット）の設計レイアウトではないことに注意することが重要です。完成品とは視覚的に異なる場合があります。プロトタイプで最も重要なことは、それが仮説とロジックをテストし、ユーザースクリプトを作成するための作業ツールであることです。つまり、プロトタイプはクリック可能でなければなりません。

プロトタイピングは、非常に高価で時間がかかるコミュニケーション設計の形式です。また、プロトタイプは、インターフェイスを理解するために何らかの努力を「読み取る」必要があるため、プロトタイプは通常、想像できる最良のドキュメントではないことにも注意してください。ただし、大規模プロジェクトを実現する場合、プロトタイプの助けを借りて実際に将来の製品を検証し、そのすべての利点と欠点を分析し、潜在的なユーザーから最も正確なフィードバックを得て、関係のない機能の完成にさらに多くのお金を節約できるため、プロジェクトなしでは実現できません。

UI / UXとは何ですか？



UX / UIデザインは古くから存在し、アプリケーションやサイトだけに適用されません。以前は別様に呼ばれていただけです。たとえば、シンガーミシンです。結局のところ、19世紀の他の誰かが、その外観と使用方法を考え出しました。同時に、ユーザーが達成しなければならない目標は、何かを速く、美しく、効率的に縫うことでした。







では、UIおよびUXデザインとは何でしょうか。



UI-ユーザーインターフェースの略で、「ユーザーインターフェース」を翻訳します。この用語は通常、画面のフォームとインターフェースの視覚化として理解されています。 UIはグラフィックデザインに関するものです。 UIデザイナーは、色、構成、アニメーション、フォントを操作します。ドラフトの回路図スケッチ（ワイヤーフレーム）に基づいて、UIデザイナーは最終的なデザインレイアウトを収集します。



Uxまたはユーザーエクスペリエンスは「ユーザーエクスペリエンス」と解釈されます。つまり、ユーザーが目標を達成できるかどうかに関係なく、インターフェイスをどのように操作するかが簡単か、難しいかです。UXデザイナーは、クライアントエクスペリエンスをインターフェイスに転送し、システムとのユーザーインタラクションの仮説を構築およびテストします。UX / UIデザインの



下で、ユーザビリティが外観と同じくらい重要であるデザインユーザーインターフェイスを実現します。 UXとUIのデザインは、並んでいると混乱することがよくありますが、パスの最後で1つの全体にマージする必要がある異なる概念です。









プロトタイプとUX / UIデザインのリンク



UX / UI情報システムのプロトタイプはデザインですか？私の意見では、です。

UI / UXデザインはプロトタイプですか？はい。ただし、ビジネスロジックが何らかの方法でユーザーインターフェイスに重ねられている場合のみ。ユーザーエクスペリエンスが紙のどこかに記載されており、インターフェイスでクライアントにそれを示すことができない場合、UIとUXのデザインは別々ですが、プロトタイプはありません。







ソフトウェア開発のないプロトタイプの良い例があります。サイトのページは紙に描かれています。実験者はユーザーの前に座り、サイトのメインページが記載されたシートをユーザーの前に置きます。ユーザーがボタンの上にボタンを押し込むか、テキストフィールドをポイントして、そこに何を入力したかを言います。ユーザーのアクションごとに、実験者は前のシートを新しいシートに置き換え、ユーザーのアクションに対するシステムの応答を表示します。



UIは-ボタンが描画された紙のシートです。 UX is-これはユーザーアクションです。開発は-これらは実験者の行動です。このようにしてユーザーエクスペリエンスを確認し、クライアントからフィードバックを得ることができるため、プロトタイプがあります。



しかし、実験者を削除し、ユーザーに紙をそのままにしておくとどうなりますか？UX / UIデザインは残り、プロトタイプは機能しません。したがって、システムのロジックをテストすることは不可能です。

figmaとは？



figmaはWebデザイナー向けのプログラムで、プロトタイプだけでなく、サイトやアプリケーションの最終的なインターフェースも作成できます。プログラムでの作業プロセスは直感的であり、プロジェクトでの同時同時作業の可能性により、Figmaでプロトタイプを実装するチームがますます増えています。



figmaの長所と短所



Figmaで作業する利点は次のとおりです。



  1. プロジェクトで作業するユーザーが2人以下の場合は無料です。
  2. 1つのプログラムでレイアウトを作成および描画する
  3. 多数の既製のWebサイトテンプレートがあります。
  4. Figmaを使用するには、新しいブラウザーバージョンが必要です。デスクトップバージョンには、強力で最新のコンピューターは必要ありません。
  5. 無料バージョンで最大30日間のプロジェクトバージョンの保存


短所figma：



  1. ロシア化の欠如。figmaは英語で働かなければなりません。
  2. Figmaはオフラインで動作しますが、変更内容を同僚が確認するにはクラウドとの同期が必要です。


Figmaで最も単純なプロトタイプを作成する例



Figmaで最も単純なプロトタイプは学生でも作成できます。以下では、ツールを使用する例として、個人用ページを作成するプロセスについて説明します。これは2ページのサイトになります。最初のページには小さなテキストブロックと写真を配置し、2番目のページには連絡先、メニュ​​ーヘッダーも作成します。



ステップ1.登録



Figmaに別のアカウントを登録する必要はありません。Googleメールがあれば、そのサービスを利用してサービスにアクセスできます。最初のオープニングで、フィグマはすぐにプログラムの使用方法に関する簡単なトレーニングを受けることを提案します。



ステップ2.プロジェクトを作成する



新しいプロジェクトを作成するには、画面の右上隅にある「+」をクリックするだけです。そこで、以前にダウンロードしたテンプレートの1つを開くことができます。



手順3.ページワークスペースを作成する



将来のサイトまたはアプリケーション用のワークスペースを作成する前に、それを表示するデバイスを決定する必要があります。モニター、スマートフォン、タブレットのモデルが異なると、レイアウトが異なって表示され、写真が別の場所に移動する可能性があるため、これは重要です。



しかし、ここではアダプティブデザインについては触れていませんが、ワークスペースを作成することで、将来のプロトタイプのインターフェースの基礎を実際に作成しています。選択したデバイスタイプのユーザーには、Figmaのようにレイアウトが表示されます。これとは別に、Figmaではサイトの作業領域をフレームと呼びます。



将来のページの基礎を作成するには、Fキーを押して、右側に表示されるメニューで、上で選択したデバイスのタイプをダブルクリックします。

サイズ、背景、その他のパラメータを変更できる白いシートがあり、その上にページインターフェイスが作成されます。







ステップ4.画像を追加する



Figmaでは、写真をどんな形にでも追加したり、切り抜いたりするのはとても簡単です。これを行うには、目的の形状と画像をワークスペースに追加し、Ctrlキーの組み合わせを使用して両方のオブジェクトを選択し、Ctrl + Alt + Mを押します。絵が図より高いことが重要です。ワークスペースと左側のパネルの両方をクリックして、シェイプを選択できます。







ステップ5.テキストフィールドを追加する



テキストを追加するには、「T」の文字を含むキーを押して、ワークスペース上のテキストフィールドの場所を選択する必要があります。テキストブロックオプションは、右側のリボンで変更できます。たとえば、ブロックを作業領域の中央に配置できます。







ステップ6.メニューを作成する



この記事では、読者にFigmaでの作業のすべての側面を教えることを追求しているわけではないので、メニューは最もシンプルな2つのボタンです。ボタンは、テキストボックスと組み合わされた長方形です。CTRL + Gを押してオブジェクトをグループにまとめ、テキストフィールドをボタンの中央に配置できます。



ステップ7.ページのベースを複製する



ページのベースには、ワークスペース、メニュー、テキストボックスが含まれます。コピーのコピーを作成するには、最初のページのオブジェクトのグループの外側にあるイメージマスクを削除できます。これを行うには、マスク名を左クリックしてアイコンを上にドラッグします。次に、オブジェクトのリストで、最初のページのベースに含まれるオブジェクトのグループを選択し、CTRL + Dを押します。







ステップ8.外部リンクを追加する



多くの場合、単一ページのサイトは、メインサイトへのトラフィックを増やすという1つの目的のためにのみ作成されます。つまり、1ページは実際にはサードパーティのリソースへのリンクです。残念ながら、Figmaではテキストへのリンクしか配置できません。これを行うには、テキストフィールドを選択し、画面上部のリンクアイコンをクリックして、サードパーティのサイトのアドレスを入力します。







ステップ9.ページ間のリンクを追加する



Figma内のオブジェクトへのリンクの追加は、外部オブジェクトへのリンクの追加と似ていますが、内部オブジェクトへのリンクをどこかに取る必要がある点が異なります。これを行うには、オブジェクトを右クリックし、ドロップダウンメニューから[コピー/貼り付け]および[リンクをコピー]を選択します。







ステップ10.プロトタイプのデモ



最後のステップは、プロトタイプを顧客に示すことです。最も簡単な方法は、画面の右上隅にある[共有]ボタンをクリックし、ポップアップウィンドウで[リンクをコピー]を選択することです。
