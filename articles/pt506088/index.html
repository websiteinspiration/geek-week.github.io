Explique-me como você descobriu a digitação de modelos para si mesmo - todos são embaçados


Quando eu era iniciante, podia escrever aplicativos simples em C # e C ++. Joguei por um longo tempo com programas de console, senti a área de trabalho e, em algum momento, queria criar um site. Uma grande surpresa me esperava - para criar sites, uma dessas coisas não é suficiente. Você também precisa conhecer ws, html, css e outras porcarias da linha de frente. Passei cerca de uma semana nessas coisas e percebi - não a minha. Eu poderia escrever algum tipo de código javascript, mas não continha tipos, e não conseguia entender como abordar isso. Isso é algum tipo de programação de brinquedos. Bem, abandonado para o inferno.


Mais tarde, trabalhando no terceiro, fui transferido para o departamento onde eles criaram a web. Eu estava pensando em sair, mas eles me explicaram - existe um script de tempo, um script de tempo é um cookie para o navegador.


Eu concordei, estudei e agora é um dos meus PLs favoritos. Mas. Texto datilografado - este não é um sysharp. Esta é uma linguagem com um sistema de tipos fundamentalmente diferente. Complexo, poderoso, mas diferente.


Existem vários parâmetros de tipificação em si.


Há digitação estática / dinâmica. Estático é quando os tipos de dados são conhecidos no estágio de compilação. O compilador conhece os tipos de variáveis ​​e, com base nas mesmas, verifica a correção do programa. Dinâmico é quando os tipos são conhecidos apenas em tempo de execução, e o compilador não verifica nada durante a montagem.
