女王を引き換える：Varonisは、急速に普及したランサムウェアランサムウェア「SaveTheQueen」を調査します



ランサムウェアウイルスクラスの新しいタイプのマルウェアは、ファイルを暗号化し、.SaveTheQueen拡張子をファイルに追加して、Active DirectoryドメインコントローラーのSYSVOLシステムネットワークフォルダーを介して拡散します。

お客様が最近このマルウェアに遭遇しました。以下に、完全な分析、その結果、および結論を示します。

検出


お客様の1人が、環境内の新しい暗号化ファイルに拡張子「.SaveTheQueen」を追加した新しいタイプのランサムウェアランサムウェアに出会った後、私たちに連絡しました。

調査中、または感染源を検索する段階で、感染した被害者の配布と追跡が、お客様のドメインコントローラのSYSVOLネットワークフォルダを使用して実行されていることがわかりました。

SYSVOLは、グループポリシーオブジェクト（GPO）とログインおよびログアウトスクリプトをドメインコンピューターに配信するために使用される各ドメインコントローラーのキーフォルダーです。このフォルダーの内容はドメインコントローラー間でレプリケートされ、組織のサイトでこのデータを同期します。SYSVOLへの書き込みには高いドメイン権限が必要ですが、侵害された後、この資産は攻撃者が悪意のある負荷を迅速かつ効率的にドメイン全体に分散させることができる強力なツールになります。

Varonis監査チェーンは、以下を迅速に特定するのに役立ちました。

  • 感染したユーザーアカウントは、SYSVOLに「hourly」という名前のファイルを作成しました
  • SYSVOLに多数のログファイルが作成されました-それぞれログファイルはドメインデバイス名に対応しています
  • 多くの異なるIPが時間単位のファイルにアクセスしました。

ログファイルは新しいデバイスでの感染プロセスの追跡に使用され、「毎時」はPowershellスクリプトを使用して新しいデバイスで悪意のある負荷を実行するスケジュールされたタスクであると結論付けました-サンプル「v3」と「v4」。

攻撃者はおそらくSYSVOLにファイルを書き込むためにドメイン管理者権限を取得して使用しました。感染したノードで、攻撃者はPowerShellコードを起動しました。これにより、マルウェアを開き、解読し、起動するためのスケジュールタスクが作成されました。

ドロッパーは保護されていない通常の.NETファイルでした。DNSpyを使用してソースコードを読んだところ、その唯一の目的はwinlogon.exeプロセスにシェルコードを挿入することであることがわかりました。





シェルコードまたは単純な問題


私たちは、Hexacorn原作者ツールを使用- shellcode2exeを順番にシェルコード「コンパイル」を実行可能ファイルにデバッグおよび分析のため。その後、32ビットと64ビットの両方のマシンで動作することがわかりました。



アセンブラーからのネイティブ翻訳で単純なシェルコードを書くことさえ難しい場合があります。両方のタイプのシステムで実行される完全なシェルコードを書くことはエリートスキルを必要とするため、攻撃者の巧妙さに驚かれるようになりました。x64dbg

を使用してコンパイルされたシェルコードを解析したところ、.NET動的ライブラリがロードされていることに気付きましたclr.dllやmscoreei.dllなど。これは奇妙に思えました。通常、攻撃者はシェルコードをできるだけ小さくしようとし、OSのネイティブ関数をロードする代わりに呼び出します。Windowsの機能をオンデマンドで直接呼び出すのではなく、シェルコードに埋め込む必要があるのはなぜですか。

判明したように、マルウェアの作成者はこの複雑なシェルコードをまったく記述していません。このタスクに特徴的なソフトウェアは、実行可能ファイルとスクリプトをシェルコードに変換するために使用されました。ドーナツ

ツールを見つけました。これは、私たちには、同様のシェルコードをコンパイルできるように思えました。これがGitHubからの彼の説明です。

ドーナツは、VBScript、JScript、EXE、DLL（.NETアセンブリを含む）からx86またはx64シェルコードを生成します。このシェルコードは、
RAM で実行するために任意のWindowsプロセスに埋め込むことができます。

理論を確認するために、Donutを使用して独自のコードをコンパイルし、サンプルと比較しました。その後、元の.NET実行可能ファイルを抽出して分析することができました。

コード保護


このファイルは、ConfuserExで難読化されています





。ConfuserExは、他の設計からコードを保護するためのオープンソースの.NETプロジェクトです。このクラスのソフトウェアを使用すると、開発者は、文字の置換、制御コマンドのフローのマスキング、参照メソッドの非表示などの方法を使用して、リバースエンジニアリングからコードを保護できます。マルウェアの作成者は難読化ツールを使用して、検出を回避し、リバースエンジニアリングのタスクをより困難にします。ElektroKill Unpackerの

おかげで、コードを解凍しました。



結論-ペイロード


変換の結果として生じるペイロードは、非常に単純なランサムウェアウイルスです。システム内に存在することを保証するメカニズムはなく、コマンドセンターへの接続もありません。これらの犠牲者を判読不能にするための古き良き非対称暗号化だけです。

main関数は、次の行をパラメーターとして選択します。

  • 暗号化後に使用するファイル拡張子（SaveTheQueen）
  • 身代金ノートファイルに入力する作成者の電子メール
  • ファイルの暗号化に使用される公開鍵



プロセス自体は次のとおりです。

  1. マルウェアは、被害者のデバイス上のローカルドライブとマップされたドライブをスキャンします


  2. 暗号化ファイルを探す


  3. 暗号化しようとしているファイルを使用してプロセスを終了しようとします。
  4. MoveFile関数を使用してファイルの名前を「Source_file_name.SaveTheQueenING」に変更し、暗号化します
  5. ファイルが作成者の公開鍵で暗号化された後、マルウェアは再びファイル名を「Original_file_name.SaveTheQueen」に変更します
  6. 同じフォルダーに身代金要求が記録されているファイル



ネイティブのCreateDecryptor関数の使用に基づくと、マルウェア関数の1つに、秘密鍵をパラメーターとして必要とする復号化メカニズムが含まれているようです。

暗号化ウイルスは、ディレクトリに保存されているファイルを暗号化しません：

C：\ windows
C：\ Program Files
C：\ Program Files（x86）
C：\ Users \\ AppData
C：\ inetpub

また、次のファイルタイプは暗号化しません： EXE、DLL 、MSI、ISO、SYS、CA​​B。

まとめと結論


ランサムウェアウイルス自体には異常な機能が含まれていなかったという事実にもかかわらず、攻撃者はActive Directoryを巧みに使用してドロッパーを配布し、マルウェア自体が分析中に興味深い、しかし複雑ではない障害を提供しました。

マルウェアの作成者は次のとおりです。

  1. 私は、winlogon.exeプロセスに組み込みの実装と、
    ファイルの暗号化および復号化機能を備えたランサムウェアウイルスを作成しました
  2. ConfuserExを使用して悪意のあるコードを偽装し、ドーナツを使用して結果を変換し、さらにbase64 Gzipドロッパーを隠しました
  3. 被害者のドメインで特権を取得し、それらを使用して
    暗号化されたマルウェアとスケジュールされたタスクをドメインコントローラのSYSVOLネットワークフォルダにコピーしました
  4. マルウェアを拡散させ、SYSVOLのログに攻撃の進行状況を書き込むために、ドメインデバイスでPowerShellスクリプトを起動しました
