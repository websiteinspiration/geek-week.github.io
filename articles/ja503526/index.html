初心者のためのDjango ORM | リクエストを最適化



Django ORM（オブジェクトリレーショナルマッピング）は、Djangoの最も強力な機能の1つです。これにより、SQLではなくPythonコードを使用してデータベースを操作できます。

実例として、次のモデルについて説明します。

from django.db import models

class Blog(models.Model):
    name = models.CharField(max_length=250)
    url = models.URLField()

    def __str__(self):
        return self.name

class Author(models.Model):
    name = models.CharField(max_length=250)

    def __str__(self):
        return self.name

class Post(models.Model):
    title = models.CharField(max_length=250)
    content = models.TextField()
    published = models.BooleanField(default=True)
    blog = models.ForeignKey(Blog, on_delete=models.CASCADE)
    authors = models.ManyToManyField(Author, related_name="posts")

2. OneToManyリレーション


OneToMany関係を使用する場合、ForeignKeyフィールドを使用し、クエリは次のようになります。

>>> post = Post.objects.get(id=1)
SELECT "blog_post"."id",
       "blog_post"."title",
       "blog_post"."content",
       "blog_post"."blog_id"
  FROM "blog_post"
 WHERE "blog_post"."id" = 1
 LIMIT 21
Execution time: 0.000161s [Database: default]

そして、postオブジェクトからblogオブジェクトにアクセスしたい場合は、次のようにします。

>>> post.blog
SELECT "blog_blog"."id",
       "blog_blog"."name",
       "blog_blog"."url"
  FROM "blog_blog"
 WHERE "blog_blog"."id" = 1
 LIMIT 21
Execution time: 0.000211s [Database: default]
<Blog: Django tutorials>

ただし、これにより、ブログから情報を取得する新しい要求がトリガーされました。そのため、select_relatedを使用してこれを回避してください。これを使用するには、元のリクエストを更新します。

>>> post = Post.objects.select_related("blog").get(id=1)
SELECT "blog_post"."id",
       "blog_post"."title",
       "blog_post"."content",
       "blog_post"."blog_id",
       "blog_blog"."id",
       "blog_blog"."name",
       "blog_blog"."url"
  FROM "blog_post"
 INNER JOIN "blog_blog"
    ON ("blog_post"."blog_id" = "blog_blog"."id")
 WHERE "blog_post"."id" = 1
 LIMIT 21
Execution time: 0.000159s [Database: default]

Djangoが現在JOINを使用していることに注意してください！また、クエリの実行時間は以前よりも短くなっています。さらに、post.blogがキャッシュされます！

4.プリフェッチオブジェクト

つまり、リクエストの数を減らすためにprefetch_relatedを使用しましたが、実際にはそれを増やしました。これを回避するには、Prefetchオブジェクトを使用してリクエストをカスタマイズします。

>>> authors = Author.objects.prefetch_related(
...     Prefetch(
...             "posts",
...             queryset=Post.objects.filter(published=True),
...             to_attr="published_posts",
...     )
... )
>>> for author in authors:
...     print(author.published_posts)
...
