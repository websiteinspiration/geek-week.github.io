クリーンでスケーラブルなJavaScriptコードの作成：12のヒント

JavaScriptは初期のWebからのものです。最初は、サイトのページを「盛り上げる」簡単なスクリプトが書かれていました。現在、JSはサーバーサイドプロジェクトの開発にも使用できる本格的なプログラミング言語になっています。

最新のWebアプリケーションは、JavaScriptに大きく依存しています。これは、シングルページアプリケーション（シングルページアプリケーション、SPA）に特に当てはまります。ライブラリーやフレームワーク（React、Angular、Vueなど）の登場により、JavaScriptはWebアプリケーションの主要なビルディングブロックの1つになりました。



このようなアプリケーションのスケーリングは、クライアントパーツであろうとサーバーパーツであろうと、困難な作業になる可能性があります。そのようなアプリケーションが十分に考えられていないアーキテクチャに基づいている場合、開発者は遅かれ早かれ特定の制限に直面するでしょう。彼らは不愉快な驚きの海で溺れています。

本日の翻訳を公開している記事の著者は、純粋なJavaScriptコードを書くためのヒントを共有したいと考えています。彼の記事はあらゆるスキルレベルのJSプログラマー向けであると彼は言います。しかし、少なくとも中級レベルでJavaScriptに精通している人にとっては特に便利です。

1.コードの分離


コードを簡単に読み取れるようにプロジェクトのコードベースをクリーンに保つには、コードフラグメントを目的に基づいて個別のブロックに分離することをお勧めします。これらのブロックは通常関数です。私はこの勧告を私が与えることができる最も重要なものと考えています。関数を作成する場合、この関数は単一の問題を解決することを目的としているという事実にすぐに焦点を合わせる必要があります。関数は、いくつかの問題を解決するように設計されるべきではありません。

さらに、関数呼び出しが副作用につながらないようにする必要があります。ほとんどの場合、これは関数がその外部で宣言されているものを変更してはならないことを意味します。データはパラメータを介して受信されます。彼女は他のことをするべきではありません。キーワードを使用して関数を返す必要がありますreturn。

2.コードをモジュールに分解


同様の方法で使用される関数や、同様のアクションを実行する関数は、1つのモジュール（または必要に応じて別のクラス）にグループ化できます。プロジェクトでさまざまな計算を実行する必要があるとします。このような状況では、そのような計算のさまざまな段階を個別の関数（分離されたブロック）の形で表現することは理にかなっており、その呼び出しをチェーンに組み合わせることができます。ただし、これらの関数はすべて1つのファイル（つまり、モジュール）で宣言できます。以下は、calculation.js同様の機能を含むモジュールの例です。

function add(a, b) {
    return a + b   
}

function subtract(a, b) {
    return a - b   
}

module.exports = {
    add,
    subtract
}

そして、このモジュールが別のファイルでどのように使用されるかを示します（これをと呼びましょうindex.js）：

const { add, subtract } = require('./calculations')

console.log(subtract(5, add(3, 2))

フロントエンドアプリケーション開発者には、次の推奨事項を与えることができます。モジュールで宣言された最も重要なエンティティをエクスポートするには、デフォルトのエクスポートオプションを使用します。セカンダリエンティティについては、名前付きエクスポートを使用できます。

3.パラメータを持つ単一のオブジェクトの代わりに複数の関数パラメータを使用する


関数を宣言するときは、パラメーターを持つ単一のオブジェクトではなく、いくつかのパラメーターを使用するように努めるべきです。次に例をいくつか示します。

// 
function displayUser(firstName, lastName, age) {
    console.log(`This is ${firstName} ${lastName}. She is ${age} years old.`)
}

// 
function displayUser(user) {
    console.log(`This is ${user.firstName} ${user.lastName}. She is ${user.age} years old.`)
}

複数のパラメーターを持つ関数の存在により、その宣言の最初の行を見ることで、何を渡す必要があるかをすぐに見つけることができます。これがまさに私がこの勧告をする理由です。

関数を開発するとき、それぞれが1つの問題のみを確実に解決するように努力する必要があるという事実にもかかわらず、関数コードのサイズは非常に大きくなる可能性があります。関数がパラメーター付きの単一のオブジェクトを受け入れる場合、関数が期待するものを正確に見つけるために、多くの時間を費やしてすべてのコードを調べる必要がある場合があります。関数を使用する場合は、パラメータを持つ単一のオブジェクトを使用する方がはるかに簡単であるように見えることがあります。ただし、アプリケーションの将来のスケーリングを考慮して関数を作成する場合は、いくつかのパラメーターを使用することをお勧めします。

個々のパラメータを使用するとその意味が失われる一定の制限があることに注意してください。私の場合、これらは4〜5個のパラメーターです。関数が多くの入力を必要とする場合、プログラマーはパラメーターを持つオブジェクトの使用を検討する必要があります。

この推奨事項の主な理由は、関数が予期する個々のパラメーターを特定の順序でそれに渡す必要があるためです。いくつかのパラメータがオプションである場合、そのようなものundefinedかは、代わりに関数に渡さなければなりませんnull。パラメータ付きのオブジェクトを使用する場合、オブジェクト内のパラメータの順序は重要ではありません。このアプローチでは、オプションのパラメーターcを設定せずに行うことができますundefined。

4.リストラ


再構築は、ES6に登場した便利なメカニズムです。指定したフィールドをオブジェクトから抽出して、すぐに変数に書き込むことができます。オブジェクトとモジュールを操作するときに使用できます。

//   
const { add, subtract } = require('./calculations')

特に、モジュールを操作する場合、モジュール全体ではなく必要な関数のみをファイルにインポートして、それらにわかりやすい名前を付けることは理にかなっています。それ以外の場合は、モジュールを表す変数を使用して関数にアクセスする必要があります。

単一のオブジェクトが関数パラメーターとして使用される場合にも、同様のアプローチが適用されます。これにより、関数の最初の行を見て、パラメーター付きのオブジェクトとして受け取ることが正確に何を期待しているのかを即座に見つけることができます。

function logCountry({name, code, language, currency, population, continent}) {
    let msg = `The official language of ${name} `
    if(code) msg += `(${code}) `
    msg += `is ${language}. ${population} inhabitants pay in ${currency}.`
    if(contintent) msg += ` The country is located in ${continent}`
}

logCountry({
    name: 'Germany',
    code: 'DE',
    language 'german',
    currency: 'Euro',
    population: '82 Million',
})


logCountry({
    name: 'China',
    language 'mandarin',
    currency: 'Renminbi',
    population: '1.4 Billion',
    continent: 'Asia',
})

ご覧のとおり、関数がパラメーターを持つ単一のオブジェクトを受け入れるという事実にもかかわらず、その構造化により、関数が呼び出されたときに何を配置する必要があるかを正確に見つけることができます。次のヒントは、関数が期待するものをユーザーにもっと正確に伝える方法です。

ちなみに、再構築は、Reactの機能コンポーネントを操作するときにも使用できます。

5.デフォルトの関数パラメーター値を設定する


関数のパラメーターの標準値、パラメーターのデフォルト値は、パラメーターを使用してオブジェクトを分解する場合、および関数がパラメーターのリストを受け入れる場合に使用すると便利です。まず、それはプログラマーに関数に渡せるものの例を提供します。次に、必須のパラメーターとオプションのパラメーターを見つけることができます。前の例の関数宣言を標準パラメーター値で補足します。

function logCountry({
    name = 'United States', 
    code, 
    language = 'English', 
    currency = 'USD', 
    population = '327 Million', 
    continent,
}) {
    let msg = `The official language of ${name} `
    if(code) msg += `(${code}) `
    msg += `is ${language}. ${population} inhabitants pay in ${currency}.`
    if(contintent) msg += ` The country is located in ${continent}`
}

logCountry({
    name: 'Germany',
    code: 'DE',
    language 'german',
    currency: 'Euro',
    population: '82 Million',
})


logCountry({
    name: 'China',
    language 'mandarin',
    currency: 'Renminbi',
    population: '1.4 Billion',
    continent: 'Asia',
})

場合によっては、関数を呼び出すときに関数パラメーターが渡されなかった場合、このパラメーターの標準値を使用するのではなく、エラーを返す必要があることは明らかです。ただし、多くの場合、ここで説明する手法は非常に役立ちます。

6.不要なデータを関数に渡さないでください


以前の推奨事項は、興味深い結論に導きます。それは、関数が必要としないデータを転送する必要がないという事実にあります。このルールに従うと、機能の開発にさらに時間がかかる場合があります。しかし、長い目で見れば、このアプローチにより、読みやすさが優れているコードベースが形成されます。さらに、プログラムの特定の場所ごとにどのようなデータが使用されているかを知ることは非常に役立ちます。

7.ファイルの行数とコードネストの最大レベルの制限


プログラムコードを含む大きなファイルを見たことがあります。とても大きい。 3,000を超える行があったものもあります。そのようなファイルでは、ナビゲートするのが非常に困難です。

結果として、コード行で測定されるファイルのサイズを制限することをお勧めします。私は通常、ファイルのサイズが100行を超えないように努めています。特定のロジックを小さなフラグメントに分割することが難しい場合、ファイルのサイズが200〜300行に達することがあります。そして、非常にまれにそ​​れらのサイズが400行に達する。この制限を超えるファイルは、読み取りや保守が困難です。

プロジェクトでの作業中に、新しいモジュールとフォルダーを大胆に作成します。プロジェクト構造は、ツリー（モジュールのグループとモジュールファイル）とブランチ（モジュールのセクション）で構成されるフォレストに似ている必要があります。プロジェクトが山脈のように見えないようにしてください。

コードでファイル自体の外観について話す場合、それらは低い丘のある地形に似ているはずです。大規模なコードのネストを回避することです。コードのネストが4レベルを超えないようにするために努力する価値があります。

12. console.logの使用を避けます


コマンドconsole.logは、プログラムをデバッグするためのシンプルで高速かつ便利なツールです。もちろん、この種のより高度なツールもありますが、console.logほとんどすべてのプログラマーがまだそれを使用していると思います。console.logデバッグに使用して、不要になったこのコマンドの呼び出しを時間どおりにクリーンアップしないと、コンソールはすぐに完全に混乱します。完全に作業の準備ができているプロジェクトのコードでさえ、一部のロギングチームを残すことが理にかなっていることに注意する必要があります。たとえば、エラーメッセージと警告を表示するコマンドです。

その結果、デバッグの目的で使用することはかなり可能であると言えますconsole.log、および作業チームでのロギングチームの使用が計画されている場合は、専門のライブラリに頼ることが理にかなっています。その中には、ログレベルとウィンストンがあります。さらに、ESLintを使用して、不要なロギングコマンドに対抗することができます。これにより、そのようなコマンドのグローバルな検索と削除が可能になります。
