Windowsターミナルプレビューv0.8がリリースされました

Windowsターミナルの新しいバージョンがリリースされました。ターミナルは、Microsoft StoreまたはGitHubのリリースページからダウンロードできます。イノベーションを見てみましょう！

探す


ターミナルに検索機能が追加されました。デフォルトでは、CTRL + Shift + Fキーの組み合わせ（{"command"： "find"、 "keys"：["ctrl + shift + f"]}）を使用して検索文字列を呼び出しますが、別の組み合わせを希望する場合は、ためらわずに、profiles.jsonファイルの中で最も便利なファイルに置き換えてください。

これとは別に、検索文字列を使用すると、大文字と小文字を照合するだけでなく、バ​​ッファを上下に検索できることに注意してください。



レトロ効果（実験機能）


CRTモニターを操作する昔のことをお見逃しですか？もしそうなら、このバージョンのターミナルはあなたのためです！@ironymanコミュニティのメンバーのおかげで、CRTモニターの効果をWindowsターミナルで直接有効にすることができます。これはこれまでの実験的な機能であり、これを有効にするには、プロファイルに次のコードスニペットを追加する必要があります。

"experimental.retroTerminalEffect": true



強化されたカスタマイズオプション


パネルとタブ


新しいパネルまたはタブを開くキーボードショートカットを作成するときに、プロファイル名「profile」：「profile-name」、guid 「profile」：「profile-guid」またはインデックス「index」：profile-indexを指定して、使用するプロファイルを選択できるようになりました。指定しない場合、デフォルトのプロファイルが使用されます。

さらに、プロファイルのいくつかの側面を上書きできます。たとえば、プロファイルコマンドライン実行可能ファイル「commandline」：「path / to / my.exe」、開始ディレクトリ「startingDirectory」：「my / path」、またはtabTitleタブのタイトル：「新しいタイトル "。

革新の例をいくつか示します。

{"keys": ["ctrl+a"], "command": {"action": "splitPane", "split": "vertical"}}

デフォルトのプロファイルを新しい垂直パネルで開きます。

{"keys": ["ctrl+b"], "command": {"action": "splitPane", "split": "vertical", "index": 0}}

プロファイルのドロップダウンリストから、新しい垂直パネルの最初のプロファイルを開きます。

{"keys": ["ctrl+c"], "command": {"action": "splitPane", "split": "horizontal", "profile": "{00000000-0000-0000-0000-000000000000}", "commandline": "foo.exe"}}

foo.exeコマンドライン実行可能ファイルを使用して、新しい水平パネルでguid 00000000-0000-0000-0000-000000000000のプロファイルを開きます。

{"keys": ["ctrl+d"], "command": {"action": "newTab", "profile": "profile1", "startingDirectory": "c:\\foo"}}

ディレクトリc：\ fooで始まる、profile1という名前の新しいタブでプロファイルを開きます。

{"keys": ["ctrl+e"], "command": {"action": "newTab", "index": 1, "tabTitle": "bar", "startingDirectory": "c:\\foo", "commandline":"foo.exe"}}

ディレクトリc：\ fooで始まる行のタブのタイトルでfoo.exeコマンドライン実行可能ファイルを使用して、プロファイルのドロップダウンリストから新しいタブに2番目のプロファイルを開きます。

ユーザー設定


これで、独自のデフォルトのプロファイル設定ができるように、profiles.jsonファイルを変更できます。新しいアーキテクチャーのおかげで、プロパティを一度設定してすべてのプロファイルに適用できます。この革新により、プロファイル間の冗長な設定を最小限に抑えることができます。この関数を追加するには、以下の例のように、「defaults 」プロパティと「list」プロパティを追加して、profiles.jsonのプロファイルオブジェクトを変更する必要があります。

"profiles": {
    "defaults": {
        "fontFace": "Cascadia Code",
        "colorScheme": "Vintage"
    },
    "list": [
        {
            "commandLine": "cmd.exe",
            "guid": "{00000000-0000-0000-0000-000000000000}",
            "name": "cmd"
        },
        {
            "guid": "{11111111-1111-1111-1111-111111111111}",
            "name": "PowerShell Core",
            "source": "Windows.Terminal.PowershellCore"
        }
    ]
},

上記のコードスニペットを使用すると、すべてのプロファイルでCascadia Codeフォントが使用され、ヴィンテージの配色になります。

注：プロファイルの設定に使用できるすべてのプロパティは、こちらで確認できます。表示されるすべてのプロパティは、「デフォルト」に適用できます。guidを除くすべて。

UIの改善


タブ


これで、タブの幅の動作を変更することができます。tabWidthModeというオプションが追加されました。このパラメーターは、「等しい」と「titleLength」の 2つの異なるタブ幅動作を提供します。「等しい」は、すべてのタブの幅を同じにして、ブラウザのタブと同様に、追加のタブが追加されるとタブを減らします。「TitleLength」は、タブタイトルの長さに応じて各タブのサイズを決定します。

当初、ターミナルのタブ幅の動作は"titleLength"に設定されていました。同じリリースでは、デフォルトの動作が「等しい」に変更されます。「titleLength」に戻りたい場合、次に、profiles.jsonファイルの「globals」プロパティに次のコードスニペットを追加します。

"tabWidthMode": "titleLength"
