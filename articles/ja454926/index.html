あなたがword2vecについて知っていたすべては真実ではありません

元の科学記事と無数のブログ投稿での負のサンプルのスキップグラムアーキテクチャとしてのword2vecの古典的な説明は、次のようになります。

while(1) {
   1. vf = vector of focus word
   2. vc = vector of focus word
   3. train such that (vc . vf = 1)
   4. for(0 <= i <= negative samples):
           vneg = vector of word *not* in context
           train such that (vf . vneg = 0)
}

実際、[word2vec skipgram]をググると、次のようになります。

  • 高レベルでアルゴリズムを説明するウィキペディアのページ
  • 同じ説明のTensorflowページ
  • 同じアルゴリズムの説明を含むデータサイエンスブログに向けて、リストは続きます。

しかし、これらの実装はすべて間違っています。

Cでのword2vecの元の実装は、動作が異なり、根本的に異なります。word2vecの単語埋め込みを使用してシステムを専門的に実装する人は、次のいずれかを実行します。

  1. Cの元の実装を直接呼び出す。
  2. それらは、変数の名前が一致する範囲で、ソースCから音訳された実装gensimを使用します。

実際、私がCgensimで知っている唯一の真の実装は私に知られています。

Cの実装


Cの実装は、実際には各単語に対して2つのベクトルをサポートしています。単語の1つのベクトルにフォーカスがあり、2番目のベクトルはコンテキスト内の単語にあります。（それはおなじみのようですか？確かに、GloVe開発者はこの事実を言及せずにword2vecからアイデアを借用しました！）

Cコードの実装は非常に有能です：

繰り返しますが、これは元の記事ではまったく説明されておらず、インターネットでもどこにも説明されていないため、推測しかできません。

仮説は、負のサンプルがテキスト全体からのものであり、頻度で重み付けされていない場合、任意の単語を選択でき、ほとんどの場合、ベクトルがまったくトレーニングされていない単語です。このベクトルに意味がある場合は、本当に重要な単語にランダムにシフトします。

重要なのは、すべての負の例をゼロに設定して、多かれ少なかれ一般的なベクトルのみが別のベクトルの表示に影響するようにすることです。

これは実際にはかなりトリッキーであり、初期化戦略がどれほど重要であるか考えたことはありませんでした。

なぜ私はこれを書いているのですか


私は生涯の2か月を費やして、元の科学出版物やインターネット上の無数の記事に記載されているword2vecを再現しようとしましたが、失敗しました。最善を尽くしましたが、word2vecと同じ結果は得られませんでした。

出版物の著者が文字通り機能しないアルゴリズムを文字通り作り上げているのに、実装がまったく違うことをしているとは思えませんでした。

結局、私はソースを研究することにしました。文字通りインターネット上の誰もが異なる実装について話し合ったので、3日間、コードを誤解していると確信していました。

元の出版物やインターネット上の記事がword2vec の実際のメカニズムについて何も述べていない理由がわからないので、この情報を自分で公開することにしました。

これはまた、否定的なコンテキストに対して別々のベクトルを設定するというGloVeの根本的な選択を説明しています-彼らはword2vecがすることをしただけで、それについて人々に話しました:)

これは科学的なトリックですか？わかりません、難しい質問です。しかし、正直なところ、私は信じられないほど怒っています。おそらく、機械学習のアルゴリズムの説明を真剣に受け止めることは二度とできないでしょう。次回は、すぐにソースを見ていきます。
