YouTubeを視聴するためにYandex。Stationを完成させます

Yandex.Stationsでは、YouTubeを視聴するのは不便です。おすすめ、サブスクリプションはなく、検索も正常に機能しません。そのため、ボットにビデオを送信するための電報をボットに書き込みました。



カットの下では、公式のオープンAPIがないという事実にもかかわらず、ストーリーは私がどのようにそれをしたかです。

どのようにしてすべてが始まったのですか？


私はエンジニアです。私は絶えずさまざまなテクノロジーや物事がどのように機能するかを研究しており、私自身も多くの興味深いプロジェクトを行っています。友達がYandex.Stationをくれたとき、私はアクティベーションプロトコルを逆にして、すごい効果を重視したデータ転送のアイデアを開発しました。

私は愚かな（スマートではない）テレビを持っており、メインのメディアコンソールとしてStationを使用しています。すべてが順調ですが、YouTubeだけがそれを見るのに完全に不快です。 YouTubeアカウントにログインできないため、おすすめや定期購入はありません。さらに、ステーションでのビデオの検索は、私が理解しているように、Yandex.Videoを通じて実行されます。残念ながら、そのようなスキームはあまりうまく機能しません。文字通り名前を発音しても動画がない場合があり、Yandex検索エンジンがインデックスを作成するまで、新しい動画はまったく視聴できません。

ステーションでYouTubeを見ることができないという事実にほとんど同意しましたが、数週間前にすべてが変わりました。

集める


リクエストからすべての「余分な」フィールドを削除して、操作可能なままにしました。本文とPOSTリクエストヘッダーでステーションにビデオを送信するには、4つのパラメーターのみを指定する必要があります。

  • SessionID-Yandexでの認証
  • x-csrf-token
  • provider_item_id-動画へのリンク（または一部のサービスの識別子）
  • device-以前に受け取ったデバイスの識別子

x-csrf-tokenとは何ですか？今は深く入りません。Yandexで承認されている場合は、frontend.vh.yandex.ru / csrf_tokenへのGETリクエストで簡単に取得できます。

この時点で、私はすでにすべてをPythonスクリプトでラップし始めていました。その結果、ステーションにビデオを送信するための関数は次のようになります。

def sendToScreen(video_url):

    # Auth and getting Session_id

    auth_data = {
            'login': config.login, 
            'passwd': config.password
            }

    s = requests.Session()
    s.get("https://passport.yandex.ru/")
    s.post("https://passport.yandex.ru/passport?mode=auth&retpath=https://yandex.ru", data=auth_data)
    
    Session_id = s.cookies["Session_id"]
    
    # Getting x-csrf-token
    token = s.get('https://frontend.vh.yandex.ru/csrf_token').text

    # Getting devices info TODO: device selection here
    devices_online_stats = s.get("https://quasar.yandex.ru/devices_online_stats").text
    devices = json.loads(devices_online_stats)["items"]

    # Preparing request
    headers = {
        "x-csrf-token": token,
    }

    data = {
        "msg": {
            "provider_item_id": video_url
        },
        "device": devices[0]["id"]
    }

    if "https://www.youtube" in video_url:
        data["msg"]["player_id"] = "youtube"

    # Sending command with video to device
    res = s.post("https://yandex.ru/video/station", data=json.dumps(data), headers=headers)

    return res.text

YouTubeからのリンクが送信された場合にplayer_idフィールドを追加することに気づいたかもしれません。事実、ステーションにはyoutube、vh、ottのコードを持つプレイヤーが何人かいます。デフォルトではvhが使用されますが、ビデオのプレビューとタイトルが壊れます。さらに、ムービーが変更されても状態がリセットされないため、多くの場合エラーが発生します（おそらく、リクエストのすべてのフィールドが「冗長」であったわけではありません）。私が理解しているように、ottプレーヤーはストリーミングサービスに使用されます。つまり、将来はステーションを通じてIPTVを視聴できるようになります。

結果はどうですか？


これで、YouTubeからStationにビデオを送信するボットができました。YouTubeアプリで[共有]をクリックして、ボットにリンクを送信するだけです。ちなみに私はそれを「ボックス」と呼んでロゴを作りました。


ログインとパスワードを収集しないように、私はそれを公開しませんでした。ただし、同じものを自分でデプロイしたり、OAuth認証用に変更したり、他のサイトからビデオを送信したりできます。すべてのソースはGitHubで入手できます。
