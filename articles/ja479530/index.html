超音波レシーバー







図。2-超音波レシーバーの開発（上から下）。

当初、レシーバーは1つのモジュールで構成されていて、部屋のカバレッジエリアが非常に狭かったため、当時は信号をデジタル化せず、通常のコントローラーが提供した長さをセンサーから単純に受信していました。

次に、カバレッジを拡大するために、3つのセンサーのモジュールを作成することにしました。データは同時にデジタル化され、PCに送信されて、距離と座標が計算されました。3コンポーネントレシーバーをテストした後、信号を自分で処理する場合は1つのレシーバーで十分であることがわかりました。

その結果、受信機は再び1コンポーネントになりましたが、現在信号を処理しているため、カバレッジエリアが非常に大きくなっています。信号はデジタル化後にSTM32で処理され、距離だけが出力に出力されます。

構造：
  • STM32-超音波信号をデジタル化し、エミッタまでの距離を計算するために使用されます。
  • HC-SR04-少し変更して、入力から信号を受信できるようになりました（図3を参照）。
  • RS485-エミッタまでの距離をPCに転送します。

超音波エミッタ







図。5-エミッターの開発（上から下）。

図5を見るとわかるように、最初はHC-SR04センサーのTrigフットを引っ張っただけでした。これは、作成できる超音波の数を決定できず、出力を変更できず、このエミッターが最適なシステムではありませんでした。十分にかさばった。

次に、2番目のバージョンを作成しましたが、これははるかに扱いにくいものでしたが、超音波スピーカーに直接接続されたSTM32とL293Dですべてを調整することができました。これで、標準の5ボルトの代わりに、17を適用しました。これにより、作業領域が大幅に増加しました。無線モジュールとRS485インターフェイスも追加し、このモジュールをスタンドアロンにしました。

一連のテストを行った後、最後の写真にある結果から、エミッターを適切なサイズに縮小して、より密に組み立てることにしました。

構造：
  • STM32-無線チャネルを介してTrigコマンド（距離の測定を開始）を受信し、超音波信号を発信します。また、無線モジュールから受信したコマンドをRS485に送信します（モバイルプラットフォームのリモートコントロールに使用します）。
  • RS485-ユーザーのためのデバイスの情報インターフェース。
  • DC-DCブースト-エミッターの電源を5Vから17Vに変換します。
  • L293D-私は17Vの電圧で強力な超音波信号を生成するために使用します。
  • nRF24-無線チャネル。
  • エミッター-HC-SR04モジュールから引き出される6つのエミッター。

基本モジュール



図。6-基本モジュール。

構造：
  • STM32-Trigをエミッタとレシーバに送信し、PCからUSBコマンドを受信して​​、無線チャネルを介してエミッタ（別名モバイルモジュール）に送信します。
  • nRF24-Trigをエミッターに送信し、コマンドをエミッターに送信します。

PCモジュール


PCはRS485アダプターを介してすべての超音波受信機に接続され、このインターフェイスでそれらからすべての長さを受信し、モバイルモジュールの座標を計算します。USB経由でベースモジュールに接続され、モバイルモジュールに制御コマンドを転送します。

超音波座標決定システムの仕事のアルゴリズム


  1. 基本モジュールは、Trigコマンドを（無線で）エミッターと（有線で）レシーバーに送信します。
  2. エミッターが放出し始め、レシーバーがブロードキャストを聴き始めます。
  3. 超音波信号を受信した各受信機は、Trigコマンドとこの信号の受信時刻との間の時間を記録し、それを距離に変換します。
  4. RS485レシーバーは、エミッターからPC上の自身までの距離を送信します。
  5. PCがエミッターの座標を計算します。

システム運用


これをすべて実行すると、オブジェクトの座標が取得されます。精度は部屋の場所によって異なります。最良の場所では、エラーは数センチメートルを超えず、最悪の場合は...図9ですべてを確認できます


。9-部屋の周りにエミッタを移動します。

この図には4つの座標セットがあります。これは、4つのレシーバーがあり、3次元空間で座標を決定するために必要なのは3つだけである場合、レシーバーの4つの組み合わせがあります。

図からわかるように、一般にシステムは座標を正しく決定しますが、場所によって誤差が異なります。エミッターをX軸とY軸に平行に移動しました。メーターの線の間隔がどこか、どこか半分のメーターのところにあります。これは図で確認できます。最高の精度は中央にあり、端では超音波信号がレシーバーに大きな角度で落ちるので、すべてが不良です。これにより、デジタル化された信号に歪みが生じます。

実際の問題


現在、いくつかの緊急の問題があり、誰かが私に何か言うことができる場合は、コメントまたはVKに書いてください：
vk.com/b__s__v

1. 超音波信号を受信する最良の方法は何ですか？私が現在超音波信号を受信して​​増幅するために使用しているHC-SR04センサーは、最良の選択肢ではありません。まず、それらは常に1つの周波数のみの信号を受信し、それらから信号源の周波数分離を行うことができないため（これは私が試すのは興味深いことです）、それらもかさばりすぎて、非常に依存しています。
2. 部屋のさまざまなコーナーの座標の歪みを削除するにはどうすればよいですか？床に塗ったグリッドでは、モバイルモジュールを常にスムーズに動かしているのに、直線がずれ始めることがあり、どういうわけかそれを取り除く必要があります。
3. nRF24の場合、信号は消えます。モジュールが近くにある場合は問題はありませんが、メッセージが1通に届くように、モジュールを数メートル広げて自分の間に置くことは価値があります。このチャネルでクロック信号を送信するので、メッセージを再送信する必要はありません。また、それらは常に同じ秒で来る必要があります。外部アンテナ付きのモジュールがありますが、ベースモジュール上のモジュールは、アンテナなしの場合よりもアンテナなしの方がうまく機能します。何もわかりませんが、力は最大です。
4. プロジェクトの推進場所。誰かが英語でHabrのような国際的なサイトを知っている場合は、リンクを破棄してください。そうでない場合、適切なものは見つかりませんでしたが、stackoverflowは、単なる質問と回答であり、類似の記事を投稿するのではありません。
5。コメントと提案。誰かが発言したい場合、私は批判や提案を喜んでします。

これは私のプロジェクトの終わりにはほど遠いですが、間もなく終わりますが、興味があれば、このシステムに基づいて私の部屋の特定の座標で移動ロボットがどのように移動するかについて記事を書きます。
