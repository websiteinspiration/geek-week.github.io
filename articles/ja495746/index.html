条件から始めて、SwiftUIでViewを形成します

条件によっては、SwiftUIビューを作成する必要がある場合があります。例えば、上記のコードでは、我々は定義HomeView含んでいてもよい、ProfileViewの場合、LogInManagerを有するloggedInUserを。標準のifステートメントを使用してこれを実装しようとしています。

struct HomeView: View {
    @ObservedObject var loginManager: LoginManager

    var body: some View {
        VStack {
            if let user = loginManager.loggedInUser {
                ProfileView(user: user)
            }

            ...
        }
    }
}

残念ながら、このコードはコンパイル時にエラーをスローします。
制御フローステートメントを含むクロージャーは、関数ビルダーViewBuilderでは使用できません。

ここでは関数クローザーは使用されず、関数ビルダーが使用されるため、HStackまたはVStackを形成するためにそれらに任意のコードを挿入することはできません。では、この状況からどうやって抜け出すのでしょうか？

1つの方法は、そのようなオプションの処理を、作成しているビューに直接渡すことです。たとえば、特定のUser値ではなくProfileViewを渡すことができますが、それをオプションにすることができます。

struct ProfileView: View {
    var user: User?

    var body: some View {
        guard let user = user else {
            // We have to use 'AnyView' to perform type erasure here,
            // in order to give our 'body' a single return type:
            return AnyView(EmptyView())
        }

        return AnyView(VStack {
            Text(user.name)
            ...
        })
    }
}

このコードは機能しますが、特に美しくはありません。ユーザーnilのProfileViewを作成しても意味がありません。別のアプローチを考えてみましょう。オプションのユーザーへのマップを使用して、それをProfileViewに変換します。

struct HomeView: View {
    @ObservedObject var loginManager: LoginManager

    var body: some View {
        VStack {
            loginManager.loggedInUser.map { user in
                ProfileView(user: user)
            }
            ...
        }
    }
}

これはすでにかなり上品です。ユーザーに値がない場合、EmptyViewを手動で指定する必要はありません。オプションではなく、特定の値をProfileViewに渡すことができます。もっと良いことはできますか？@ViewBuilder

の良い点は、これはSwiftUIのプライベート実装ではなく、独自の関数とクロージャに注釈を付けることができるアクセス可能な属性であることです。

この属性を使用して、ビューのUnwrapをコンパイルできます。これは、オプションの値をパラメーターとして受け取り、@ ViewBuilderタグ付きクロージャーを使用して、nil以外の値をViewに変換します。

struct Unwrap<Value, Content: View>: View {
    private let value: Value?
    private let contentProvider: (Value) -> Content

    init(_ value: Value?,
         @ViewBuilder content: @escaping (Value) -> Content) {
        self.value = value
        self.contentProvider = content
    }

    var body: some View {
        value.map(contentProvider)
    }
}

この設計を使用して、HomeViewの構造全体を完全に再設計できます。

struct HomeView: View {
    @ObservedObject var loginManager: LoginManager

    var body: some View {
        VStack {
            Unwrap(loginManager.loggedInUser) { user in
                HStack {
                    Text("Logged in as:")
                    ProfileView(user: user)
                }
            }
            ...
        }
    }
}
