新しいVisual Studioターミナルに会う

最近発表されたDeveloper PowerShellのテーマを開発して、新しいVisual Studioターミナルの最初のプレビューバージョンをお届けします。この関数はVisual Studioバージョン16.3プレビュー3



の一部です。VisualStudio ターミナルはゼロから作成されていませんが、カーネルのほとんどをWindowsターミナルと共有しています。あなたにとって、これはターミナルのより信頼できる操作と新しい機能のより速い導入を意味します。

Visual Studioで新しいターミナルを有効にする


端末のプレビューバージョンをテストするには、まず[プレビュー機能]ページにアクセスして、ターミナルを有効にする必要があります。これを行うには、[ツール]> [オプション]> [プレビュー機能]に移動し、[実験用VSターミナル]の横のボックスをオンにして、Visual Studioを再起動します。



オンにすると、[表示]> [ターミナルウィンドウ]または検索から呼び出すことができます。



端末プロファイルの作成


ターミナルが起動すると、組み込みのPowerShellインスタンスが自動的に開きます。ただし、シェルプロファイルを使用して起動モードを設定できます。

シェルプロファイルを使用すると、さまざまなタイプのシェルを使用したり、固有の引数を使用してそれらを呼び出したり、必要に応じてデフォルトのシェルを設定したりできます。

今後のアップデートでは、ターミナルにいくつかの基本的なプロファイルを追加する予定です。ただし、オプションページ（ターミナルアイテム）で手動でプロファイルを追加できるようになりました。



一般的なオプションのプロファイルを設定する方法の例：

開発者コマンドプロンプト


シェルの場所：

C:\Windows\System32\cmd.exe

引数：

/k "C:\Program Files (x86)\Microsoft Visual Studio\2019\IntPreview\Common7\Tools\VsDevCmd.bat"

開発者PowerShell


シェルの場所：

C:\WINDOWS\system32\WindowsPowerShell\v1.0\powershell.exe

引数：

-NoExit -Command "& { Import-Module 'C:\Program Files (x86)\Microsoft Visual Studio\2019\Preview_master\Common7\Tools\vsdevshell\Microsoft.VisualStudio.DevShell.dll'; Enter-VsDevShell -InstanceId f86c8b33}"

注：特定の構成に合わせて、上記の引数を調整する必要があります。引数の情報を取得するには、Developer PowerShellプロパティの[ショートカット]タブの[ターゲット]行を確認します。



WSL


シェルの場所：

C:\WINDOWS\sysnative\wsl.exe
