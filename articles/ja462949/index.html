PVS-Studioが... PVS-Studioで使用されているライブラリでエラーを検出した方法のストーリー


これは、PVS-Studioを使用して、PVS-Studioで使用されるライブラリのソースコードにエラーを見つけることができた方法についての短編です。さらに、理論的なものではなく実際のものです。アナライザーでライブラリーを使用すると、実際にエラーが発生しました。

PVS-Studio_Cmd（およびその他のユーティリティ）では、コマンドライン引数を解析するための特別なライブラリであるCommandLineを使用します。

今日、私はPVS-Studio_Cmdの新しいモードのサポートに携わっていましたが、たまたまこの引数の解析ライブラリを使用しなければなりませんでした。コードの作成過程では、なじみのないAPIを使用する必要があるため、コードのデバッグも行っています。

したがって、コードは記述、コンパイル、実行のために実行されます。ii...



コードの実行はライブラリー内で行われ、そこでNullReferenceException型の例外が発生します。側面からはあまり明確ではありません-メソッドに明示的なnull参照を渡さないでください。

念のため、呼び出されたメソッドのコメントを確認します。それらがNullReferenceException型の例外の条件を記述することはほとんどありません（通常、私には思えるように、この型の例外は予測されないため）。



メソッドへのコメントには、NullReferenceExceptionに関する情報はありません（ただし、これは予期されています）。

例外の原因（および場所）を正確に確認するために、プロジェクトのソースコードをダウンロードし、ライブラリのデバッグバージョンを収集してアナライザーに接続することにしました。プロジェクトのソースコードはGitHubで入手できます。アナライザーで現在使用されているのはまさにこのタイプであるため、バージョン1.9.71が必要です。

適切なバージョンのソースコードをロードし、収集し、デバッグライブラリをアナライザーに接続し、コードを実行して実行します。



したがって、例外の場所は明らかです。helpInfoがnullであり、Leftインスタンスプロパティにアクセスすると、タイプNullReferenceExceptionの例外が発生します。 そして、私は思慮深くなりました。最近、PVS-Studio for C＃は、nullの可能性がある参照の逆参照の検索の分野を含む、さまざまな分野でそれほど改善されていません。特に、手続き間の分析が改善されました。そのため、PVS-Studioが議論中のエラーを見つけることができるかどうかを確認するためにソースコードをチェックすることは、すぐに興味深くなりました。 私はソースコードをチェックし、他の警告の中で、私が望んでいたものを正確に見ました。PVS-Studio警告：V3080





'helpInfo.Left'のメソッド内でnull参照が発生する可能性があります。2番目の引数、helpInfoを調べることを検討してください。Parser.cs 405

アナライザーは、DisplayHelpVerbTextメソッドを呼び出すときに警告メッセージを生成し、2番目の引数helpInfoについて警告します。このメソッドはifステートメントのthenブランチにあることに注意してください。条件式は、then-分岐が変数の次の値で実行できるように構成されています：

  • helpInfo == null ;
  • _settings.HelpWriter！= null ;

DisplayHelpVerbText メソッドの本体を見てみましょう。

private void DisplayHelpVerbText(
  object options, Pair<MethodInfo, 
  HelpVerbOptionAttribute> helpInfo, string verb)
{
  string helpText;
  if (verb == null)
  {
    HelpVerbOptionAttribute.InvokeMethod(options, helpInfo, null, out helpText);
  }
  else
  {
    HelpVerbOptionAttribute.InvokeMethod(options, helpInfo, verb, out helpText);
  }

  if (_settings.HelpWriter != null)
  {
    _settings.HelpWriter.Write(helpText);
  }
}

以来== nullの動詞我々が興味を持っている（メソッドの呼び出しを参照してください）then-の枝if文。elseブランチの状況は似ていますが、特定のケースでは実行されたため、then-ブランチを検討します。helpInfoはnullにできることに注意してください。

次に、HelpVerbOptionAttribute.InvokeMethodメソッドの本体を見てみましょう。実際、上記のスクリーンショットですでに見ています。

internal static void InvokeMethod(
    object target,
    Pair<MethodInfo, HelpVerbOptionAttribute> helpInfo,
    string verb,
    out string text)
{
  text = null;
  var method = helpInfo.Left;
  if (!CheckMethodSignature(method))
  {
    throw new MemberAccessException(
      SR.MemberAccessException_BadSignatureForHelpVerbOptionAttribute
        .FormatInvariant(method.Name));
  }

  text = (string)method.Invoke(target, new object[] { verb });
}

helpInfo.Leftは無条件に呼び出されますが、helpInfoはnullにすることもできます。アナライザーはこれについて警告し、これが起こりました。

結論

PVS-Studioの助けを借りて、PVS-Studioで使用されるライブラリのコードでエラーを見つけることができたのはおかしいことでした。これは、「PVS-StudioはPVS-Studioコードのエラーを見つけるのですか？」という質問への回答の一種の続きだと思います。 :) PVS-Studioコードだけでなく、使用するライブラリのコードでもエラーを見つけることができます。
