ReduxをReactコンテキストとフックで置き換えるための効果的なソリューションを実装する方法に関する記事の無料翻訳を紹介します。翻訳やテキストの誤りを指摘することは大歓迎です。観るのを楽しむ。


React 16.3.0での新しいContext APIのリリース以来、多くの人々は、新しいAPIがReduxの代替品と見なすのに十分かどうかを自問しています。同じことを考えましたが、フック付きのバージョン16.8.0がリリースされた後も完全には理解できませんでした。私は人気のあるテクノロジーを使用しようとしますが、そのパスはそれらが解決するすべての問題を常に理解しているわけではないため、Reduxにも慣れています。

Button冗談を受け取るアクションジェネレーター（おおよそアクションクリエーター。翻訳は、Reduxのドキュメンテーションから取得されます）を受け取る必要があります。DadJoke状態を受け取りApp、プロバイダーコンテキストを使用して両方のコンポーネントを表示する必要があります。

次に、カスタムコンポーネントを作成して名前を付けますDadJokeProvider。これにより、内部で状態が管理され、子コンポーネントがコンテキストプロバイダーにラップされます。上記のReactでの子の最適化により、状態を更新してもアプリケーション全体が再レンダリングされないことに注意してください。

したがって、ファイルを作成して名前を付けますcontexts/dad-joke.js。

import React from 'react'

const DadJokeContext = React.createContext()

export function DadJokeContextProvider({ children }) {
  const state = { dadJoke: null }
  const actions = {
    fetchDadJoke: () => {},
  }
  return (
    <DadJokeContext.Provider value={{ state, actions }}>
      {children}
    </DadJokeContext.Provider>
  )
}

また、コンテキストから値を取得するために2つのフックをエクスポートします。 

export function useDadJokeState() {
  return React.useContext(DadJokeContext).state
}

export function useDadJokeActions() {
  return React.useContext(DadJokeContext).actions
}

これを実装できます：

import React from 'react'
import {
  DadJokeProvider,
  useDadJokeState,
  useDadJokeActions,
} from './contexts/dad-joke'

function Button() {
  const { fetchDadJoke } = useDadJokeActions()
  console.count('Button')
  return (
    <button type="button" onClick={fetchDadJoke}>
      Fetch dad joke
    </button>
  )
}

function DadJoke() {
  const { dadJoke } = useDadJokeState()
  console.count('DadJoke')
  return (
    <p>{dadJoke}</p>
  )
}

function App() {
  console.count('App')
  return (
    <DadJokeProvider>
      <Button />
      <DadJoke />
    </DadJokeProvider>
  )
}

export default App

ここに！フックを使用して作成したAPIのおかげで。今後、このファイルに変更を加えることはありません。

stateから始めて、コンテキストでファイルに機能を追加しましょうDadJokeProvider。はい、フックを使用することもできますが、useState代わりにreducer、よく知られている機能を追加するだけで状態を管理しましょうRedux。

function reducer(state, action) {
  switch (action.type) {
    case 'SET_DAD_JOKE':
      return {
        ...state,
        dadJoke: action.payload,
      }
    default:
      return new Error();
  }
}

これで、このレデューサーをフックに渡しuseReducerて、APIでジョークを取得できます。


export function DadJokeProvider({ children }) {
  const [state, dispatch] = React.useReducer(reducer, { dadJoke: null })

  async function fetchDadJoke() {
    const response = await fetch('https://icanhazdadjoke.com', {
      headers: {
        accept: 'application/json',
      },
    })
    const data = await response.json()
    dispatch({
      type: 'SET_DAD_JOKE',
      payload: data.joke,
    })
  }

  const actions = {
    fetchDadJoke,
  }

  return (
    <DadJokeContext.Provider value={{ state, actions }}>
      {children}
    </DadJokeContext.Provider>
  )
}

これは、関数fetchDadJokeが時間の経過とともに変化するものに依存せず、他のアクションジェネレーターにも依存しないことを意味します。したがって、最初のレンダリング時にアクションオブジェクトを作成する必要があるのは1回だけです。

const actions = React.useMemo(() => ({
    fetchDadJoke,
  }), [])

アクション付きのキャッシュされたオブジェクトがあるので、コンテキスト値を最適化できますか？実際、いいえ。値オブジェクトをどれだけ適切に最適化しても、状態が変化するため、毎回新しい値オブジェクトを作成する必要があります。ただし、アクションオブジェクトを既存のコンテキストから新しいコンテキストに移動するとどうなるでしょうか。コンテキストは1つしか持てないと言ったのは誰ですか？

const DadJokeStateContext = React.createContext()
const DadJokeActionsContext = React.createContext()

両方のコンテキストを組み合わせることができますDadJokeProvider。

 return (

      <DadJokeActionsContext.Provider value={actions}>

そして、フックを微調整します：

export function useDadJokeState() {
  return React.useContext(DadJokeStateContext)
}

export function useDadJokeActions() {

}
