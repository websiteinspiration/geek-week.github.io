Travailler avec du code: trois outils ouverts

Parlons de quelques éditeurs de texte et d'un utilitaire pour concevoir du code dans la documentation.



Autres outils de développement: API et éléments d'interface utilisateur





Photo - ThisisEngineering RAEng - Unsplash





Micro



Il s'agit d'un éditeur de texte terminal, aiguisé par les systèmes modernes. Le code source est sur GitHub et est distribué sous la licence MIT . Les développeurs affirment que le produit est le successeur idéologique de l'éditeur Nano pour les systèmes d'exploitation de type Unix.



Parmi les micro-fonctions intéressantes, nous pouvons souligner le support de la souris (glisser, double-cliquer, triple-cliquer) et plusieurs curseurs, la liaison automatique du code (vérification des erreurs) et la mise en évidence de la syntaxe pour plus de 90 langues . Dans un fil conducteur sur Hacker News, les résidents notent que l'utilitaire est similaire à l'éditeur propriétaire Sublime Textet possède plusieurs de ses fonctions: édition multi-ligne, gestion des plugins - même les raccourcis clavier sont similaires.



L'outil est apparu il y a environ trois ans, donc des bugs sont toujours rencontrés dans son travail . Bien que l'un des résidents de HN ait noté que la microstabilité s'est récemment accrue. Dans tous les cas, l'outil a une communauté (plus de 14 000 étoiles sur GitHub), et il y a une communication active dans le chat Gitter .





Atome



Il s'agit d'un éditeur de texte multiplateforme et d'un outil de développement fourni par GitHub. Il est construit sur HTML, JavaScript, CSS, Node.js et utilise le framework Electron .



Il offre des paramètres étendus pour l'interface et le texte - vous pouvez changer les styles, les raccourcis clavier, même l'apparence de l'éditeur, si vous connaissez JavaScript. Presque tous les paramètres sont dans des fichiers CSON (format JSON, mais pour les objets CoffeeScript). Puisque GitHub développe Atom, il a une extension qui vous permet d'accéder au système de contrôle de version directement à partir d'un éditeur de texte - gérer les branches, exécuter les commandes commit , push et pull . Il existe également un package Teletype- En l'utilisant, le code peut être écrit par plusieurs développeurs en même temps.





Capture d'écran: atom.io / Travailler avec Git dans Atom



Atom est un outil relativement jeune, donc il y a des problèmes de performances, cependant, les auteurs sont engagés dans l'optimisation .





Carbone



L'outil vous permet de créer de belles captures d'écran de code pour des présentations, des articles ou de la documentation. Il suffit de copier un morceau du programme sur le terrain sur le site et de sauvegarder l'image (si vous le souhaitez, vous pouvez poster la photo directement sur Twitter). L'ingénieur Brian Dennis de FOSSA , un système de gestion d'applications open source à l' échelle de l' entreprise, est impliqué dans le développement de Carbon .


En utilisant Carbon, vous pouvez choisir une couleur d'arrière-plan et ajuster la coloration syntaxique - plus de 30 langages et frameworks sont pris en charge: de C et Python à Vue et R. Une communauté assez importante s'est formée sur le site autour de l'instrument - plus de 24 000 étoiles . Ses membres développent des extensions personnalisées. Par exemple, il existe des plugins pour intégrer Carbon aux éditeurs de texte Emacs , Vim et Atom déjà mentionnés.
