WebStorm 2020.1: peningkatan antarmuka, dukungan Vuex, dan meluncurkan Prettier saat menyimpan file

Halo semuanya! Kami senang mempersembahkan pembaruan WebStorm besar pertama tahun ini. Di versi baru Anda akan menemukan banyak fitur baru dan perbaikan yang sudah lama ditunggu-tunggu, termasuk dukungan Vuex dan opsi baru untuk meluncurkan Prettier saat menyimpan file.



Unduh versi percobaan 30 hari WebStorm 2020.1 online atau menggunakan Aplikasi Toolbox . Versi lengkap dapat digunakan oleh pemegang langganan aktif ke WebStorm atau All Products Pack, serta siswa gratis dan pengembang proyek sumber terbuka.

Sekarang mari kita lihat peningkatan utama lebih terinci.

Font baru untuk bekerja dengan kode


Sepanjang tahun lalu, kami telah mengembangkan font khusus untuk bekerja dengan kode yang akan membantu pengguna kami memprogram dengan sangat nyaman dan meminimalkan ketegangan mata. Hasil dari upaya kami adalah JetBrains Mono , font open source baru. Dimulai dengan versi 2020.1, JetBrains Mono dipilih secara default di WebStorm dan IDE kami yang lain. Kemampuan untuk memasukkan font yang berbeda, tentu saja, juga tetap ada.



Dokumentasi cepat yang lebih bermanfaat.


WebStorm 2020.1 akan membantu Anda dengan cepat menemukan informasi yang relevan tentang simbol, karena dokumentasi cepat sekarang ditampilkan ketika Anda mengarahkan kursor ke simbol itu, dan bukan hanya ketika Anda menekan F1 . Jika kode Anda memiliki masalah dan WebStorm dapat membantu dengan solusinya, itu juga akan memberi tahu Anda tentang hal itu dengan menampilkan perbaikan cepat yang tersedia tepat di jendela pop-up dokumentasi.



Saat bekerja dengan kode JavaScript dan TypeScript, dokumentasi cepat akan menampilkan informasi terperinci tentang jenis dan visibilitas karakter, serta di mana karakter itu didefinisikan.



Mode Zen untuk pekerjaan terfokus


Mode Zen baru membantu Anda fokus sepenuhnya pada kode Anda. Menggabungkan dua mode, Distraction Free dan Full Screen , memungkinkan Anda untuk dengan cepat mengaktifkan keduanya dan untuk sementara mengisolasi sesuatu yang dapat mengalihkan perhatian Anda.



Untuk mencoba mode baru, buka Lihat | Penampilan | Masuk ke Mode Zen dari menu utama WebStorm atau gunakan jendela pop-up Switch ( Ctrl + `| Mode tampilan | Masuk ke Mode Zen ).

Dukungan untuk Vuex dan Vue Composition API


Vue.js semakin populer, jadi kami secara aktif berupaya menjadikan WebStorm sebagai IDE paling berguna untuk bekerja dengan kerangka kerja ini. Kali ini kami menambahkan dua peningkatan besar.

Untuk mulai dengan, ketika bekerja dengan perpustakaan Vuex, Anda akan melihat opsi untuk penyelesaian kode untuk simbol dan modul Vue store saat mengedit komponen Vue. WebStorm juga membantu Anda dengan cepat beralih ke mendefinisikan getter, mutasi, dan tindakan.



Kedua, karena sudah dimungkinkan untuk menggunakan API Komposisi, tersedia dalam rilis Vue 3 yang akan datang, kami memutuskan untuk menambahkan dukungannya di WebStorm 2020.1.



Menjalankan Prettier saat menyimpan file


Berkat opsi Jalankan di simpan untuk file yang baru , Anda dapat menerapkan pemformatan Prettier ke semua file yang ditentukan dalam pengaturan WebStorm dan diedit dalam proyek saat ini, sambil menyimpan file-file ini - Anda tidak perlu lagi mengkonfigurasi pengamat file atau menggunakan plugin pihak ketiga.



Membantu kesalahan tata bahasa dan gaya bahasa


Dimulai dengan versi 2020.1, WebStorm dibundel dengan Grazie, alat kami untuk memeriksa ejaan, tata bahasa, dan gaya teks. Ini akan membantu Anda menghindari kesalahan tata bahasa saat menambahkan komentar, komit, dan berbagai konstruksi bahasa.



Secara default, Grazie hanya diaktifkan untuk bahasa Inggris dan tidak memeriksa semua jenis file untuk kemungkinan kesalahan tata bahasa. Anda dapat menambahkan lebih banyak bahasa dan mengubah pengaturan yang telah ditentukan di Preferensi / Pengaturan | Editor | Proofreading (buka bagian Tata Bahasa jika Anda ingin mengonfigurasi ruang lingkup pemindaian, lihat aturan yang ada dan tambahkan pengecualian).

Tindakan dan inspeksi niat baru


Seperti biasa, kami telah menambahkan beberapa tindakan niat baru dan perbaikan cepat untuk membantu Anda menghemat waktu saat bekerja dengan kode JavaScript dan TypeScript. Misalnya, salah satu tindakan niat baru, tersedia dengan menekan Alt + Enter , akan dengan cepat mengkonversi kode yang ada ke rantai opsional dan / atau null penggabungan.



Informasi lebih lanjut tentang tindakan-niat baru dan perbaikan cepat tersedia di posting blog rilis (dalam bahasa Inggris).

Dukungan untuk versi terbaru dari teknologi populer


Di WebStorm 2020.1 Anda akan menemukan dukungan bawaan untuk fungsionalitas TypeScript 3.8, termasuk hanya impor / ekspor, bidang pribadi, dan tingkat atas await, yang akan membantu Anda bekerja dengannya secara lebih efisien. Selain itu, kami membuat bekerja dengan proyek Angular 9 lebih mudah dengan sepenuhnya mendukung format metadata baru.

Pengikatan komitmen yang lebih nyaman


Tindakan interaktif Rebase dari sini , tersedia di tab log dari jendela Git , memungkinkan Anda untuk mengedit, menggabungkan, dan menghapus komit sebelumnya. Ini membantu membuat sejarah komit lebih linier dan dapat dimengerti.



Dalam WebStorm 2020,1, Anda akan menemukan ditingkatkan, versi yang lebih interaktif dari Rebasing Komit kotak dialog yang terbuka ketika Interaktif Rebase dari sini tindakan disebut . Kotak dialog yang diperbarui menampilkan tindakan yang dapat diterapkan untuk setiap komit. Itu juga menunjukkan informasi terperinci tentang setiap komit dan memungkinkan Anda untuk melihat perbedaan dan dengan cepat membuang perubahan yang diterapkan.

Menggunakan WebStorm untuk mengedit file dengan cepat


Berkat mode LightEdit baru , Anda dapat membuka file di jendela editor teks yang terpisah, tanpa membuat atau mengunduh seluruh proyek. Mode ini akan berfungsi jika WebStorm belum berjalan. Jika WebStorm berjalan, file akan terbuka di dalamnya, seperti biasa.

Untuk mencoba mode baru dalam aksi, klik kanan pada file yang ingin Anda edit dan pilih WebStorm dari daftar program yang diusulkan. Atau, Anda juga dapat mengkonfigurasi peluncur baris perintah seperti yang dijelaskan di sini dan membuka file menggunakan baris perintah.



Itu saja untuk sekali. Terima kasih sudah membaca sampai akhir! Anda akan menemukan lebih banyak detail di posting blog rilis .(dalam Bahasa Inggris). Pertanyaan, keinginan, dan pikiran hanya diungkapkan dalam komentar. Kami, seperti biasa, akan dengan senang hati menjawab. Laporan bug dapat dibuat di sini .
