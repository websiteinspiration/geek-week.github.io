.NETを使用したWeb開発：Blazor WebAssembly 3.2.0の紹介

こんにちは、ハブル！Blazor WebAssemblyが正式にリリースされ、誰でも利用できるようになりました。これはBlazor WebAssemblyのフル機能でサポートされているリリースであり、本番環境で使用できます。.NETによるフルスタックWeb開発！カットの下の詳細。



仕事の始まり


Blazor WebAssemblyの開始は非常に簡単です。blazor.netにアクセスし、Blazor WebAssemblyアプリケーションを作成して実行するために必要なすべてを含む最新バージョンの.NET Core SDK（3.1.300以降）をインストールします。

次に、次のコマンドを実行して、最初のBlazor WebAssemblyアプリケーションを作成して実行できます。

dotnet new blazorwasm -o BlazorApp1
cd BlazorApp1
dotnet run

アドレス「https：// localhost：5001」に移動して、出来上がり！最初のBlazor WebAssemblyアプリケーションを作成して起動しました。

Blazorのパフォーマンスを最大化するには、選択したプラットフォームに対応したバージョンのVisual Studioをインストールしてください。

  • Windows：Visual Studio 2019 16.6
  • macOS：Visual Studio for Mac 8.6
  • Windows / macOS / Linux：C＃拡張子の付いたVisual Studio Code

blazor.netで 追加のドキュメントと例を見つけることができます。

既存のプロジェクトを更新する方法


すでにBlazor WebAssemblyプロジェクトがある場合は、次の手順で、プロジェクトをプレビューバージョン3.2.0からリリースバージョン3.2.0にアップグレードできます。

  • Microsoft.AspNetCore.Components.WebAssembly。およびSystem.Net.Http.Jsonパッケージへのすべてのリンクをバージョン3.2.0に更新します。

これですべてです。すべての準備が整いました。

Blazor WebAssemblyとは何ですか？


Blazorについて初めて聞く場合は、Blazor WebAssemblyとは何かを教えてください。

Blazorは、JavaScriptの代わりに.NETとC＃を使用して単一ページのアプリケーションを作成するためのオープンソースのクロスプラットフォームユーザーインターフェイスフレームワークです。 Blazorは、多機能のインタラクティブなWebインターフェイスを作成するための強力で柔軟なコンポーネントモデルに基づいています。 .NETコードとRazor構文の組み合わせ、つまりHTMLとC＃のエレガントな組み合わせを使用して、Blazorユーザーインターフェイスコンポーネントを実装します。 Blazorコンポーネントは、ユーザーインターフェイスイベントをシームレスに処理し、ユーザー入力と通信し、ユーザーインターフェイスの更新を効率的に表示できます。

Blazorコンポーネントをさまざまな方法でホストして、Webアプリケーションを作成できます。最初にサポートされる方法は、Blazor Serverと呼ばれます。 Blazor Serverアプリケーションでは、コンポーネントは.NET Coreを使用してサーバー上で実行されます。すべてのユーザーインターフェイスの操作と更新は、ブラウザーとのリアルタイムWebSocket接続を使用して処理されます。 Blazor Serverアプリケーションは迅速にロードされ、簡単に実装できます。 Blazor Serverのサポートは、.NET Core 3.1 LTSで利用できます。

現在、Blazor WebAssemblyは、Blazorコンポーネントをホストする2番目のサポートされている方法です。WebAssemblyに基づく.NETランタイムを使用するブラウザーのクライアント側。 Blazor WebAssemblyには、Webの標準化されたバイトコードであるWebAssemblyで実装された適切な.NETランタイムが含まれています。この.NETランタイムはBlazor WebAssemblyアプリケーションと共にロードされ、通常の.NETコードをブラウザーで直接実行できます。プラグインやコードは必要ありません。 Blazor WebAssemblyは、デスクトップとモバイルの両方のすべての最新のWebブラウザーで動作します。 JavaScriptと同様に、Blazor WebAssemblyアプリケーションは、ブラウザーのセキュリティサンドボックスからユーザーのデバイスで確実に実行されます。これらのアプリケーションは、.NETサーバーコンポーネントなしで完全に自律的な静的サイトとして展開するか、ASP.NET Coreに接続して、コードをクライアントとサーバーに簡単に渡すことができる完全な.NET Web開発を提供できます。 。

フル機能


Blazor WebAssemblyには、次のWebアプリケーションプロジェクトで生産的に作業するのに役立つ多くの機能が付属しています。

  • 実行時にC＃パフォーマンスと厳密な型指定を使用します。
  • 安定した成熟した.NETエコシステムを構築します。
  • クライアントとサーバーでコードと既存の.NET標準ライブラリを簡単に再利用できます。
  • Blazor Serverアプリケーションとの共通コンポーネントモデル。
  • アプリケーションをスタンドアロンの静的サイトとしてデプロイするか、ASP.NET Coreでホストします。
  • スタンドアロン機能と統合OS統合を備えた高度なWebアプリケーション（PWA）を作成します。
  • 組み込み認証サポート。
  • グローバリゼーションとローカリゼーションの統合サポート。
  • 環境ベースの構成。
  • 組み立て中のILのトリミングと事前圧縮。
  • フルスタックデバッグ。
  • Visual Studio、Visual Studio for Mac、およびVisual Studio Codeとの優れた相互作用。

すぐに使えるコンポーネント


もちろん、どのWebアプリケーションにも、美しく機能豊富なコンポーネントが必要です。さまざまなBlazorユーザーインターフェイスコンポーネントが、Blazor WebAssemblyアプリケーションを含むあらゆるBlazorアプリケーションで優れた機能を発揮する素晴らしいパートナーから入手できます。

オープンソースコミュニティ


Blazorには、クールなオープンソースコミュニティとエコシステムもあります。コミュニティメンバー（あなたと同じように！）多くの優れたコンポーネントライブラリ、インタラクションライブラリ、テスト環境などを作成し、それらを自由に使用できるようにしました。ここにいくつかの興味深い例があります：

  • ブラゾライズ

これらのコミュニティプロジェクトなどは、GitHubのBlazorリポジトリで見つけることができます。

LTSまたは現在？


Blazor WebAssembly 3.2.0は、.NET Core Support Policyに従って完全にサポートされたリリースです。これはBlazor WebAssemblyの最初のリリースであるため、これは現在のリリースであり、LTS（長期サポート）リリースではありません。LTS .NET Core 3.1のステータスを継承しません。つまり、今年後半にBlazor WebAssemblyが.NET 5をリリースした後、それを.NET 5にアップグレードする必要があります。.NET5のBlazorは非常に高い互換性があると予想されます。

次は何ですか？


Blazor WebAssemblyを公開したので、次に.NET 5に注目します。BlazorWebAssemblyを.NET 5で利用できるようにする作業はすでに始まっており、来月の新しいプレビューで完了する予定です。

.NET 5および6で検討しているBlazorの機能と改善点についてのアイデアがあります。GitHubの.NET 5 のBlazorロードマップで実装する主な機能のリストを確認できます。このリストは非常に重要であると考えています。しかし、それでも、これらの改善をすべて作成することを望んでいますが、多くは不明なままであり、計画はもちろん、前進するにつれて変更される可能性があります。また、多くの小さな改善が行われることも期待しています。これについても説明します。

また、Xamarinチームの友人と引き続き協力して、Mobile Blazor Bindingsプロジェクトの一部としてBlazorを使用した独自のユーザーインターフェイスの作成を実験的にサポートします。この作業には、いわば、ネイティブアプリケーション用のハイブリッドユーザーインターフェイスの作成を研究するための「初期」の取り組みが含まれます。これについては、すぐにお話ししたいと思います。
