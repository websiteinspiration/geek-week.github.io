Get-Dateを使用しているときに、さまざまな目盛りについて、またはPowershellでエラーを取得しない方法について一言

こんにちは、Powershell愛好家。

私は彼を愛しています。そして今日、私はこの投稿を書く動機となった奇妙なことに気づきました。あなたも興味があると思います。過剰ティックの場合。興味があれば、猫へようこそ：

奇妙さの本質は何ですか？

1回の処理では、選択の都合上、月末までの日数が必要でした。

これは、次のような1行で計算されました。

((Get-date -Day 1).AddMonths(1)-(get-date)).days-1

その計算の特徴は、さまざまな結果をもたらす可能性があります。

最初は、コードまたはPowershellバージョンに問題があると思いました。

数台の車で確認したところ、再現性があることがわかりました。

したがって、私は座って関数を書き、書きました。

私はこれを得ました、コメントはすぐにコードに行きます：

    $CurrentDate = get-date       #  
    $CurrentDay = $CurrentDate.day #

Get-DaysToEndOfMonths(1)

まあ、私が書いた後、私は座って考え、この行に来ました：

[int]((((Get-Date -day 01).AddMonths(1)).AddDays(-1)).Day-(Get-date).Day)

彼女はエラーを出しませんでしたが、それは可能でした。なぜなら

、私たちは時間\分\秒、またはプロセッサーのクロックの刻みを考慮に入れていないことがわかりました。
Get-dateはミリ秒単位の値を返すことに注意してください。

ただし、計算を実行するときに、最初と2番目のGet-date呼び出しが1ティックに落ちた場合、次のような値になります。

(Get-date -day 1).AddMonth(1) = 1.12.2019 15:33:00:500
Get-date = 1.11.2019 15:33:00:500

引き算して30 00：00：00：000を
取得しますが、2番目のGet-dateの呼び出しが次のティックに該当する場合、その値は
=>になります。
1.11.2019 15:33:00:501

そして、私たちは値を取得します 
29 23:59:59:999

問題が見つかったので、これを行うことができます。


#    
((((Get-Date -day 01).AddMonths(1)).AddDays(-1))-(Get-date -Hour 0 -Minute 0 -Second 0 -Millisecond 0)).Days

そして、私たちは常に同じ価値を持ちます。

気をつけて、良い週末を！
