広告ブロッカーは、いわゆるフィルターを使用します。これは、ネットワーク要求をブロックし、ページ上の要素を非表示にすることができるコード行です。餌の要素を作成することにより、BlockAdBlockはこれらのフィルターを特別に起動します。

このシンプルな防御により、BlockAdBlockは、uBlock Origin、AdBlock Plus、Ghosteryなどのすべての主要な広告ブロッカーに対して効果的です。これに対抗するには、BlockAdBlockを実行するサイトでのみアクティブになる独自のフィルターを作成する必要があります。

ブロッカー用のフィルターの作成は少し複雑です。ロード後に生成されるページの要素をブロックするコンテンツフィルターが必要です。ベイトには識別子があるため、要素を非表示にbanner_adする例外を作成しますマークされ#@#、すべての要素のための#識別子でbanner_ad、私たちのブロッカーのユーザフィルタのリストにそれを置きます。

BlockAdBlockは広告ブロッカーの盲点を悪用します。フィルターが上記の識別子をすべてスキップすると、実際の広告もスキップされます。したがって、BlockAdBlock は広告ブロッカーを強制的に無効にします。

ブロッカーでは、ページをロードする前に任意のスクリプトを実行できます。事前にBlockAdBlockオブジェクトを削除してみることができます。ただし、このためには、起動するたびにランダム化されるBlockAdBlockオブジェクトの名前が必要です。

uBlock Originは別のアプローチを採用しました。コードは関数によって実行されるため、BlockAdBlockが見つかった場合に実行をブロックevalする独自の関数evalを定義できます。 JSでは、オブジェクトがこれProxyを行うことができます。つまり、任意のオブジェクトの任意のプロパティとメソッドを置き換えることができます。

これは、最初のBlockAdBlockペイロードを渡さずevalに直接使用することで回避できるため、エントリポイントのプロキシである呼び出しも使用しますsetTimeout。それがされているのでsetTimeout、文字列ではなく、関数によって渡され、私たちはこの文字列を確認してください。

function scriptExists(href) {
    if (href) href = href.substr(href.length - 15);
    var scripts = document.getElementsByTagName('script');
    for (var i = scripts.length; i--;) {
        var src = String(scripts[i].src);
        if (src) src = src.substr(src.length - 15);
        if (src === href) return true
    };
    return false
};

結論


誰が広告ブロッカーと広告ブロッカーをブロックするアンチブロッカーの間の武装競争に勝つでしょうか？時間だけが教えてくれます。軍拡競争が進展するにつれて、BlockAdBlockの進化が示すように、アンチブロッカーはより洗練された方法と特別なコードを使用する必要があります。

一方、ブロッカーには、JavaScriptへのアクセスだけでなく、安定したシステムとフィルターリストを介した強力なフィルターツールの利点があります。このようなシステムでは、1人で敵を倒す方法を理解し、新しいサイトでフィルターリストを更新するだけで十分です。

BlockAdBlockの進化と広告ブロッカーのさまざまな応答を分析することにより、BlockAdBlockとブロッカー間の小さな戦争の絵を描くことができました。その過程で、私たちは敵対行為でどのような方法が使われているかを学びました。
