Mash programming language



http://mash-project.org

https://github.com/RoPi0n/mash-lang



Mash?



It is an imperative programming language with dynamic typing, garbage collection, OOP, and multithreading support.







Interesting? Then under the cut!



How completed is the project?







At the moment, the language is at the Pre-Release stage, its functionality and performance is not yet enough to solve absolutely any tasks with its help, but this is what I strive for.



In the repository there is an assembly for Win64, in which all the functionality of the language is debugged and works without bugs.



The functionality of Mash is similar to Python.



The language is designed as simple and fully functional as possible.



Fully supports OOP, introspection, reflection, higher-order functions, multithreading, synchronization, parallelization and code synchronization within the body of methods.



As the language runtime environment, a stacked virtual machine (VM) is used that supports multithreading and garbage collection (Reference Counting).



Implemented a language translator (with the construction of AST, everything is just like in books) into abstract code for VMs and in the future into other languages ‚Äã‚Äã(but because of the complexity of the language, these are only plans for the future).



For convenience, a small IDE (FPC + SynEdit) is implemented to familiarize yourself with the language and work with it.



Embeddability



The VM of the language has an API, the functionality of the language can be expanded by writing libraries for VMs in any native language (FPC, Delphi, C / C ++, Rust, etc.), and the language can also be built into any of your projects, and at the same time get at your disposal all the functionality of Mash and its native libraries.



Debug?



The VM has the ability to work in conjunction with a debugger (which is still very crude).



The language supports exception tracing.







Platform support



The language is completely written in Free Pascal, which in turn supports a huge list of platforms for which Mash can be built.



There are no dependencies on any libraries.



The project has already been assembled and tested on Windows, Linux and Android.



What is Mash capable of right now?



At the moment I am working on the implementation of a standard set of libraries for working with files, math, I / O, multithreading, GUI, network, databases, cryptography, etc.











In the project repository (and in the Pre-Release build) you can find small demo applications.

Mash contains a snake, an asynchronous web server / framework (like Flask), drawing graphs in Cartesian and polar coordinate systems, the Lorenz attractor, rotation of simple 3D models like a cube, as well as a version of the Mash translator written in Mash 'e.
