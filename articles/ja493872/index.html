Node.jsについて一言


Node.jsは、イベントループやイベント指向アーキテクチャなどの概念に基づく非同期JavaScriptランタイムです。Node.jsプラットフォームと標準のNPMパッケージマネージャーを使用すると、Webから機械学習まで、さまざまな分野の効果的なアプリケーションを作成できます。

最も単純なWebサーバーの例（node server.js）：

const http = require('http')
const hostname = '127.0.0.1'
const port = 3000
const server = http.createServer((req, res) => {
res.statusCode = 200
res.setHeader('Content-Type', 'text/plain')
res.end('Hello World\n')
})
server.listen(port, hostname, () => {
console.log(`Server running at http://${hostname}:${port}/`)
})

上記のコードを見ると、Node.jsの次の機能に注目できます。

  1. JavaScriptコードをサーバーで直接実行できます。つまり、nodeコマンドを使用してJavaScriptファイルを実行できます。
  2. 依存関係とESモジュールをロードするためのCommonJS形式のモジュールがサポートされており、両方の形式を使用できます。
  3. HTTPがその一部である標準モジュールライブラリがサポートされています。
  4. APIは非同期コールバックパターンに基づいていますが、Promiseもサポートされています。
  5. ES2015構文がサポートされています。ちなみに、これは、JS機能がサポートされているNodeのどのバージョンでいつでも確認できる便利なリンクです。問題が発生した場合は、問題を比較して理解できます。

また、小さなデモ（VSCode + Chrome Debug）もお見逃しなく。

歴史への短い遠足


Node.jsプラットフォームは2009年に登場し、プロジェクトの主な開発者および作成者はRyan Dahlでした。彼の主なアイデアは、JavaScriptを使用して、サーバーの非ブロッキングI / O（入出力）を作成することでした。当時、そのようなアプローチはあまり一般的ではなく、JavaScriptは最良のソリューションの1つでした。

Node.jsは、Chromiumエンジンを内部で使用します。つまり、JavaScript- V8を解釈するその部分を使用します。一度にインタプリタが個別のV8プロジェクトに分離されたという事実により、その周りにエコシステムを簡単に作成することが可能になりました。厳密に言えば、これはまさにNode.jsのRyan Dahlの作成者が行ったことです。ちなみに、今日、プロジェクトを支援する組織であるNode.js Foundationがあります。
