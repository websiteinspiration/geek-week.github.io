就職の面接のためにデータサイエンティストからよく寄せられる5つのSQLの質問

SQLクエリのコンパイルはデータサイエンティストにとって最も興味深いものではありませんが、SQLをよく理解することは、データ処理に関連するアクティビティで成功したい人にとって非常に重要です。ここでのポイントは、SQLだけではないということであるSELECT、FROMとWHERE。スペシャリストが知っているSQLの構成が多ければ多いほど、必要なすべてのものをデータベースから取得するためのリクエストを作成するのが簡単になります。 本日の翻訳を公開している記事の著者は、次の2つの問題を解決することを目的としていると述べています。





  1. 基本的なSQLの知識を超えるメカニズムを探索します。
  2. SQLを操作するためのいくつかの実用的なタスクについての考察。

この記事では、Leetcodeからの5つのSQL質問について説明します。これらは、インタビューでよく遭遇する実際的なタスクを表しています。

質問1：給与の面で2位


SQLクエリを記述して、従業員の給与情報（Employee）を含むテーブルから2番目に高い給与を含むエントリを取得します。

たとえば、以下のテーブルに対して実行されたそのようなクエリはを返し200ます。テーブルに最高の給与より低い値がない場合、リクエストはを返しnullます。

+----+--------+
| Id | Salary |
+----+--------+
| 1  | 100    |
| 2  | 200    |
| 3  | 300    |
+----+--------+

▍決定A：使用IFNULLしてOFFSET


この問題のこのソリューションで使用される主なメカニズムは次のとおりです。

  • IFNULL(expression, alt)：この関数は、引数expressionが等しくない場合に引数を返しますnull。それ以外の場合、引数が返されaltます。nullテーブルに必要な値が含まれていない場合は、この関数を使用して戻ります。
  • OFFSET：この演算子は式で使用されORDER BY、最初のn行を破棄します。これは、結果の2行目（つまり、2番目に大きい給与、テーブル内のデータ）に関心があるという理由で役立ちます。

これは既製のリクエストです：

SELECT
    IFNULL(
        (SELECT DISTINCT Salary
        FROM Employee
        ORDER BY Salary DESC
        LIMIT 1 OFFSET 1
        ), null) as SecondHighestSalary
FROM Employee
LIMIT 1

▍ソリューションB：使用 MAX


以下のクエリは関数を使用していますMAX。ここでは、最高の給与値が選択されています。これは、テーブル全体で受け取った最大給与とは異なります。その結果、必要なもの、つまり2番目に大きな給与が得られます。

SELECT MAX(salary) AS SecondHighestSalary
FROM Employee
WHERE salary != (SELECT MAX(salary) FROM Employee)

質問2：メールアドレスの重複


テーブル内のPersonすべての重複する電子メールアドレスを検出するSQLクエリを記述します。

+----+---------+
| Id | Email   |
+----+---------+
| 1  | a@b.com |
| 2  | c@d.com |
| 3  | a@b.com |
+----+---------+

▍決定A：COUNTサブクエリ内


最初に、テーブル内の各アドレスの出現頻度が決定されるサブクエリを作成します。次に、サブクエリによって返された結果は、命令を使用してフィルタリングされますWHERE count > 1。クエリは、ソーステーブルで見つかったアドレスに関する情報を複数回返します。

SELECT Email
FROM (
    SELECT Email, count(Email) AS count
    FROM Person
    GROUP BY Email
) as email_count
WHERE count > 1

▍ソリューションB：式 HAVING


  • HAVING：これは、式で命令を使用できるようWHEREにする式GROUP BYです。

SELECT Email
FROM Person
GROUP BY Email
HAVING count(Email) > 1

質問番号3：温度の上昇


Weather温度がそれより前の日付の温度よりも高くなるすべての日付（日付識別子）を テーブルで検索するSQLクエリを記述します。つまり、「今日」の気温が「昨日」の気温よりも高い日付に関心があります。

+---------+------------------+------------------+
| Id(INT) | RecordDate(DATE) | Temperature(INT) |
+---------+------------------+------------------+
|       1 |       2015-01-01 |               10 |
|       2 |       2015-01-02 |               25 |
|       3 |       2015-01-03 |               20 |
|       4 |       2015-01-04 |               30 |
+---------+------------------+------------------+

▍解決策： DATEDIFF


  • DATEDIFF：この関数は、2つの日付の差を計算します。正確に「今日」と「昨日」の温度を比較するために使用されます。

次のクエリを通常の言語で定式化すると、次のアイデアを表すことがわかります。それらが表す日付に対応する温度が、それらに関する「昨日の」日付の温度よりも高くなるように識別子を選択する必要があります。

SELECT DISTINCT a.Id
FROM Weather a, Weather b
WHERE a.Temperature > b.Temperature
AND DATEDIFF(a.Recorddate, b.Recorddate) = 1

質問番号4：単元の最高給


テーブルにEmployeeは、会社の従業員に関する情報が格納されます。このテーブルの各レコードにIdは、従業員の識別子（）、氏名（Name）、給与（Salary）、および勤務先の会社の部門（）に関する情報が含まれていますDepartment。

+----+-------+--------+--------------+
| Id | Name  | Salary | DepartmentId |
+----+-------+--------+--------------+
| 1  | Joe   | 70000  | 1            |
| 2  | Jim   | 90000  | 1            |
| 3  | Henry | 80000  | 2            |
| 4  | Sam   | 60000  | 2            |
| 5  | Max   | 90000  | 1            |
+----+-------+--------+--------------+

この表Departmentには、会社の部門に関する情報が含まれています。

+----+----------+
| Id | Name     |
+----+----------+
| 1  | IT       |
| 2  | Sales    |
+----+----------+

給与が最大の従業員の各部門で検索するSQLクエリを記述します。たとえば、上記のテーブルの場合、同様のクエリは次のテーブルで表される結果を返します（テーブル内の行の順序は重要ではありません）。

+------------+----------+--------+
| Department | Employee | Salary |
+------------+----------+--------+
| IT         | Max      | 90000  |

lutionソリューション：チーム IN


このコマンドを使用すると、いくつかのコマンドの使用に対応する条件INを命令に設定できWHEREますOR。たとえば、次の2つの構成は同じです。

WHERE country = ‘Canada’ OR country = ‘USA’
WHERE country IN (‘Canada’, ’USA’).

ここでは、部門Departmentの名前（Employee）、従業員の名前（）、およびその給与（Salary）を含むテーブルを取得します。これを行うには、ユニット識別子（DepartmentID）とこのユニットの最大給与に関する情報を含むテーブルを作成します。その後、我々は、以下の場合にのみ該当したテーブル内のどのエントリに応じて条件テーブルの上に2を組み合わせるDepartmentIDとSalary、前のテーブルに形成されています。

SELECT
    Department.name AS 'Department',
    Employee.name AS 'Employee',
    Salary
FROM Employee
INNER JOIN Department ON Employee.DepartmentId = Department.Id
WHERE (DepartmentId , Salary) 
    IN
    (   SELECT
            DepartmentId, MAX(Salary)
        FROM
            Employee
        GROUP BY DepartmentId
 )

質問5：生徒の移植


メアリーは高校の先生です。彼女は、seat生徒の名前と教室内の場所に関する情報を格納するテーブルを持っています。idこの表の値は常に増加しています。メアリーは隣の生徒を入れ替えたいと思っています。

学生の初期配置の表は次のとおりです。

+---------+---------+
|    id   | student |
+---------+---------+
|    1    | Abbot   |
|    2    | Doris   |
|    3    | Emerson |
|    4    | Green   |
|    5    | Jeames  |
+---------+---------+

近隣の生徒を移植した後の状況は次のとおりです。

+---------+---------+
|    id   | student |
+---------+---------+
|    1    | Doris   |
|    2    | Abbot   |
|    3    | Green   |
|    4    | Emerson |
|    5    | Jeames  |
+---------+---------+

教師が上記の問題を解決できるようにするリクエストを作成します。

学生数が奇数の場合は、最後の学生をどこかに転校させる必要はありません。

▍解決策：演算子を使用する WHEN


SQL構文CASE WHEN THENはif、プログラミングでは演算子と見なすことができます。

この例では、最初の演算子をWHEN使用して、奇数の識別子がテーブルの最後の行に割り当てられているかどうかを確認します。その場合、ラインは変更されることはありません。2番目の演算子WHENは、各奇数識別子に1を加算し（たとえば、1、3、5は2、4、6に変わります）、各偶数識別子から1を減算します（2、4、6は1、3、5に変わります）。

SELECT 
    CASE 
        WHEN((SELECT MAX(id) FROM seat)%2 = 1) AND id = (SELECT MAX(id) FROM seat) THEN id
        WHEN id%2 = 1 THEN id + 1
        ELSE id - 1
    END AS id, student
FROM seat
ORDER BY id
