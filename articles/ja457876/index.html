2019年2月に、非営利組織UWB Allianceが作成されました。この提携には、アップル、ヒュンダイ、起亜、ゼブラ、デカウェーブ、アルテロス、ノベルダ、ユビセンスなどが含まれます。

作業を調整して、超広帯域（UWBまたはUWB）で無線信号を送信する技術を促進および改善するため。この組織内で、IEEE 802.15.4z標準を開発するためのワーキンググループが作成されました。

こんにちは。私の名前はユージーンです。私はRealTrac Technologiesで働いています。IEEE 802.15.4z標準開発グループの作業に関するレポートの翻訳をあなたの法廷に提案します。また、ウルトラワイドバンドテクノロジー、開発パス、およびアプリケーションの現在の状況についての質問にも答えさせていただきます。

前書き


このドキュメントでは、802.15.4-2015 HRPおよびLRP PHYを使用して、同じ範囲で動作する他の802ワイヤレス標準に関してP802.15.4zによって修正されたシステムのパフォーマンスを評価するために実行された簡単な互換性分析を提供します。 。

1.用語


このドキュメントで使用されている用語の意味は次のとおりです。

「基本標準」とは、標準の802.15.4-2015およびこのドキュメントの作成時に承認されたすべての修正を意味します。

「802.15.4」は基本規格を意味します。

「この修正」とは、修正P802.15.4zを意味します：低速ワイヤレスネットワークの規格の修正：拡張高速パルス（HRP）、低速パルス（LRP）、物理環境のUWB（PHY）、および関連する周波数範囲の決定方法。

2.概要


802.15.4 UWBに基づくシステムは世界中で広く使用されています。 802.15.4a-2007の最初のバージョンにはHRP UWB PHYが標準に含まれており、LRP UWB PHYが802.15.4f-2010に追加されました。修正P802.5.4zは、UWB PHYと新規および既存のアプリケーションの両方に適用されます。現在のUWBシステムは、非常に低い電力スペクトル密度で世界中で動作しています。このドキュメントでは、802.15.4の古いバージョンや現在の802.11システムなど、他の802標準ワイヤレスシステムとの互換性を分析します。

UWB互換性情報のソースは多数あります。このドキュメントで使用されている方法は、同じ周波数範囲で動作可能な802ワイヤレスシステム間の互換性に関する結果をまとめたものです。CAD 802.15.4a [8]と802.15.4f [9]への参照と研究[10] [11]は、UWB PHYと以下のシステムとの互換性について説明しています。

  • オーバーラップする周波数帯域で動作する802.15.4 PHY
  • 3.4から3.8 GHzの範囲で動作する802.16
  • 5 GHzおよび6 GHz帯域で動作する802.11 OFDM。

世界中で、UWBシステムは通常、パワースペクトル密度（PSD）の制限によって制限される非常に低い電力で送信し、意図しないエミッターに対して確立されたスプリアスおよび/または意図しない電磁放射と一致しています。たとえば、米国およびアジアとヨーロッパの多くの地域では、PSDの制限は-41.3 dBmです。

2.1.5 MAC


この修正によって追加された新しいMAC機能は、既存のMAC機能を使用して、レガシー802.15.4デバイスとの互換性を確保し、標準によって提供されるテスト済みの互換性機能を保持します。
MACは、この修正により、次のように伝送媒体の距離を測定する際に使用される規定で補足されます。

  • ブロードキャスト/マルチキャスト：ブロードキャストのスケジューリングとマルチキャスト通信
  • 関連情報の交換距離を測定するために使用される情報を送信するための新しい情報要素
  • 完全性チェックが改善された距離測定モニタリング用のMAC機能
  • 新しいランキングおよびデータ共有ツールをサポートするためのSAP MACの変更

チャネルステータスの評価とデータ送信の開始に使用されるチャネルアクセス方法は、これらの追加のMAC機能によって変更されません。互換性への影響は最小限です。

2.2 802.15.4の互換性メカニズムの概要


802.15.4節の互換性メカニズムは、[8]と[9]で説明されています。[13]で説明されているように、MACアーキテクチャのおかげで、固有の802.15.4短いデューティサイクルによって互換性も強化されています。

この修正に加えられたMACの変更は、互換性への影響が最小限になります。

  • 新しいスケジューリング機能は、既存のチャネルアクセス制御メカニズム（CSMA-CA）と類似しており、互換性があります。
  • 新しい機能は、[8]で説明されているように、負荷の軽減、効率的なデューティサイクル、およびチャネルへのアクセス制御の点で互換性を維持します。

UWB PHYは、非常に低い電力で動作し、通常はバックグラウンドの電磁放射の限界以下です。これは通常、他のシステムに対するUWBエミッターの影響を制限します。

2.3互換性分析の方法論


このドキュメントで言及されている互換性の研究は、一般に[12]で説明されている方法論に従って行われ、各システムを影響の主体と対象の両方として考慮します。このドキュメントでは、互換性分析が現在の802標準への準拠について検討されており、ここで関連する調査結果を示します。802ワイヤレス規格は変化しているため、802.15.4 UWBと802.11システムの間の互換性を具体的に評価する研究がさらに行われ、普及してきました。これらの研究の結果は、このホワイトペーパーに記載されています。

このペーパーで引用されている互換性の研究[10]と[11]は、モンテカルロシミュレーション法を使用して、スペクトルを共有する際の潜在的な影響を評価しています。

3.同じ周波数帯の異なるシステム


この節では、同じ周波数範囲で動作する他の802システムとの互換性係数を示します。この条項では、さまざまな意味で、802.15.4 LRPまたはHRP PHY仕様に従って動作するIR-UWBとは異なることを意味します。

3.1 802.11の互換性


付録Eの[5]および[6]で詳細に説明されているように、802.11システムは、表1に示すように、チャネル間隔が5 MHz〜160 MHzのさまざまな範囲で動作できます。802.11 WLANデバイスは、一部の地域では最大1000 mW（30 dBm）の比較的高いEIIRP電力で動作できます。UWBデバイスは、-41.3 dBm / MHzに制限された平均EIRP値で動作します。802.15.4 UWBデバイスは、500 MHz以上の公称帯域幅を使用します。

研究[10]と[11]は、802.15.4 UWBに基づくシステムの近くで動作する802.11システムの影響を示すシミュレーション結果を示しています。この調査では、さまざまな展開シナリオと動作条件を調べます。

3.1.1 802.15.4 UWBに対するWLANの影響


[10]と[11]で説明されているシナリオの結果は、潜在的な影響を示しています。 WGSE PT45調査[10]は、アクティブな信号測定データと組み合わせたシミュレーション手法を使用して、個別干渉と総干渉の両方を考慮しています。結果は、最大946メートルの802.11ワイヤレスLANからの干渉により、UWB通信および位置追跡システムで3 dBを超える感度損失が生じることを示しています。サウンディングの場合、対応する距離は212メートルです。モンテカルロシミュレーション中の干渉の総合評価では、WLANデューティサイクルが1.97％の場合、UWB通信と位置追跡デバイスへの感度が低下する確率は3 dBを超え、範囲は5〜15％です。

センサーの場合、感度が3 dB以上低下する可能性は3〜6％です。[11]でモデリング手法を使用して、追加の構成とシナリオが調査されます。研究は、通信と範囲/場所の両方に大きな影響を与えます。この調査には、相互運用性を改善するための緩和に関する推奨事項も含まれています。

3.2 802.15.4互換システム


RCC PHYは、表1に示す範囲で動作できます。明らかに、RCC PHYは、UWBシステムに近接して動作することはありません。RCCは主に屋外や線路の近くで使用されます。

3.3その他の802ワイヤレスシステム


リンク[8]では、802.15.4 UWBと802.16に基づくシステム間の互換性プロパティについて詳しく説明しています。結果は、間隔距離1 mでPER値が1％を下回り、間隔距離が6.9 mを超える場合、UWB LRP信号からの802.16への影響は無視できる程度になることを示しています。

結果は、802.16システムを干渉源として使用し、HRP UWBシステムを影響の対象として使用する場合、44 mの分離距離でインジケーターが1％未満に低下し、140 mを超える分離距離では重要ではなくなることを示しています。

LRPシンボルの信号構造、帯域幅、およびパワースペクトル密度はHRP信号と非常に似ているため、LRPの結果はリファレンス[8]に示すものと同様になります。

4. 802.15.4 UWBシステム


この規定では、この修正と既存の802.15.4 UWBシステムの互換性状況について説明します。

4.1 HRP


古い802.15.4a HRPおよび新しい802.15.4z HRPモードは、プリアンブルシーケンスを使用して距離を同期および測定します。どちらのシーケンス標準も、干渉条件下での信頼性が高くなるように設計されています。両方の標準のシーケンスは、別の標準のシーケンスとの相関が非常に低くなります。プリアンブル間の標準間干渉は、標準内干渉とほとんど同じです。どちらの規格も500 MHzの帯域幅を使用します。どちらも〜7 Mbpsでの動作に128 nsシンボルを使用し、〜30 Mb / sでの動作に32 nsシンボルを使用します。 802.15.4z HRPは、802.15.4z HRPよりも高いPRFを使用します。ピークスペクトルの制限により、送信電力の自動調整はわずかに異なる場合があります。それにもかかわらず、標準間干渉は、基本的に標準内干渉と同じです。

4.2 LRP


この修正の変更は、同じチャネルアクセス方法に依存しており、無線の影響分野で追加のレガシーLRPデバイスが利用可能になるのと同じ影響があると予想されます。[9]で説明されている互換性メカニズムは同じです。グラスファイバーシステムは、非常に低いデューティサイクルで動作することが期待されています。

5。結論


この修正で説明されているUWBシステムは、無線の影響の分野で動作している他の802ワイヤレスシステムへの影響が最小限か、まったくありません。低信号電力と低デューティサイクルは、UWB以外のシステムでのUWB信号からの干渉の影響を低減します。特に、802.15.4と802.11に基づく他のシステムへの影響は、通常、同じ無線範囲で検出されません。

レガシーUWB 802.15.4システムと同じ無線の影響範囲で作業する場合、この修正に従って動作するシステムの影響は、追加のレガシーデバイスの影響と同じかそれ以下です。プリアンブルとSTSを追加すると、レガシーシステムで信号を認識せず、ノイズレベルを下回るため、レガシーUWBへの影響が軽減されます。

干渉の対象として、この修正で説明されているUWBシステムは、それらからの信号が認識されて適切に処理されるため、従来のUWBシステムと互換性があります。近接して動作する802.11ベースのシステムが存在する場合、より高い電力を使用するため、UWBシステムに大きな影響が予想されます。影響の程度は、802.11システムの動作サイクルに最も依存します。物理的な分離により露出が減少します。
