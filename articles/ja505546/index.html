Flutterとデスクトップ開発

マルチプラットフォームアプリケーションを作成するという考えは、決して新しいものではありません。Flutterはこれを行う機能も提供します。この記事では、私が自分でモバイルアプリケーションの開発に使用しているデスクトップでモバイルアプリケーションを起動する2つの方法について説明します。エミュレーターとシミュレーターの実行を停止したのは、主にそれらなしで実行する機会があったためです。アイデアに興味のある方は猫へようこそ。

では、なぜエミュレータとシミュレータをバイパスしてデスクトップバージョンを作成する必要があるのでしょうか。

まず、ワークステーションのリソース。現在の商用プロジェクトでは、多くのアニメーションとさまざまな「重い」ウィジェットを使用しています。 i7、16GB RAM、GTX1050Tiグラフィックスカードを備えた非常に優れたラップトップでは、非常に複雑な移行とアニメーションを備えたプロセッサーがデバッグモードで30〜35％にジャンプし、RAMアンドロイドエミュレーターは最大2GBを消費します。もちろん、フレームは失われ、アニメーションは「ぎくしゃく」動きます。これにより、画像の完全性を適切に評価できません。デスクトップバージョンが起動すると、私のプロセッサは「休止」し、同じアニメーションが最大3〜4％読み込まれます。これにより、1分間でプロセッサが8〜10倍、RAMが300〜400 MB節約されます。

第二に、さまざまなデバイスでウィジェットの正しい配置をすばやく表示する機能。窓を取り、それを伸ばすか、最小サイズに絞ると、個人的な時間と他のテスターの時間を大幅に節約できます。

エミュレーターウィンドウサイズをエミュレートするためのサンプルプラグイン


最も簡単な方法は、たとえばgithub.com/go-flutter-desktop/examples/tree/master/text_demoなどのサンプルプロジェクトリポジトリからGoフォルダーをコピーすることです。

次に、Goディレクトリにplugins / desktop_utilsフォルダーを作成し、その中に将来のプラグインを含むファイルに名前を付けます。 main.goに書き込み

次のコード： 
package desktop_utils

import (
	"github.com/go-flutter-desktop/go-flutter"
	"github.com/go-flutter-desktop/go-flutter/plugin"
	"github.com/go-gl/glfw/v3.3/glfw"
)

const channelName = "go-test/desktop_utils"

type DesktopUtilsFlutterPlugin struct {
	window *glfw.Window
	channel *plugin.MethodChannel
}

var _ flutter.Plugin = &DesktopUtilsFlutterPlugin{}
var _ flutter.PluginGLFW = &DesktopUtilsFlutterPlugin{}

func (p *DesktopUtilsFlutterPlugin) InitPlugin(messenger plugin.BinaryMessenger) error {
	channel := plugin.NewMethodChannel(messenger, channelName, plugin.StandardMethodCodec{})
	
	channel.HandleFunc("set_window_size", p.setWindowSizeHandler)

	return nil
}

func (p *DesktopUtilsFlutterPlugin) InitPluginGLFW(window *glfw.Window) error {
	p.window = window

	return nil
}

func (p *DesktopUtilsFlutterPlugin) setWindowSizeHandler(arguments interface{}) (reply interface{}, err error) {
	argumentsMap := arguments.(map[interface{}]interface{})

	var minW = int(argumentsMap["minW"].(int32))
	var minH = int(argumentsMap["minH"].(int32))
	var maxW = int(argumentsMap["maxW"].(int32))
	var maxH = int(argumentsMap["maxH"].(int32))

	p.window.SetSizeLimits(minW, minH, maxW, maxH);

	return nil, nil
}

channelName — Flutter
minW, minH, maxW, maxH — /

ファイルをGoモジュールとして初期化します：go mod init desktop_utils（go.modファイルが表示されます）。

Goディレクトリに移動し、go.modファイルを見つけて、プラグインのローカルディレクトリを追加します。replace_desktop_utils => ./plugins/desktop_utils
を置き換え、オプション行のoptions.goにプラグインを追加します。

var options = []flutter.Option{
	flutter.WindowInitialDimensions(InitW, InitH),

	flutter.AddPlugin(&desktop_utils.DesktopUtilsFlutterPlugin{}),
}

これでゴーパートは終わりました。

Flutterパーツはできるだけシンプルです。必要なのは、go-test / desktop_utilsという名前でチャネル（MethodChannel）を作成し、それに必要な長さと幅のパラメーターを渡すことだけです。

const MethodChannel _utilsChannel = MethodChannel('go-test/desktop_utils');

_utilsChannel.invokeMethod(
  'set_window_size',
   <String, dynamic>{
     'minW': 400,
     'minH': 600,
     'maxW': 800,
     'maxH': 1200,
   },
 );

それで全部です。プラグインの準備が整いました。
