同時に実行するアクションを指定する機能を備えたRLアルゴリズムを実装するためのニューラルネットワークアーキテクチャ

RLアルゴリズムを実装するための古典的なニューラルネットワークスキームの1つは次のとおりです。


ここで：入力-ニューラルネットワークへの入力。 FC-（完全に接続された）隠しレイヤーアーキテクチャまたはCNN-FC-アーキテクチャ隠しレイヤーアーキテクチャ（入力への入力に応じて）;出力-ネットワーク出力。多くの場合、ネットワーク出力はすべての可能なアクションのセットからのアクションの1つの確率を与えるソフトマックスレイヤーです。

このアーキテクチャの欠点は、同時に実行されるいくつかのアクションの選択を実装することが難しいことです。

この問題を解決するために、マスク層を備えたアーキテクチャが提案されている。提案されたアーキテクチャは次のとおりです。


このアーキテクチャは、古典的なアーキテクチャと完全に一致していますが、アクションマスクレイヤーも含まれています。このアーキテクチャから出る方法は1つだけです。これは、アクション値（同時に実行されるアクションのグループ）の値です。アクションマスクレイヤーは、以下の擬似コードに従って実装できます。

import numpy as np

class Layer:
    def __init__(self, items, item_size, extra_size):
        assert(items > 0)
        assert(item_size > 0)
        assert(extra_size >= 0)
        self.items = items
        self.item_size = item_size
        self.extra_size = extra_size

    def build(self):
        self._expand_op = np.zeros((self.items, self.items*self.item_size), \
		dtype=np.float32)
        for i in range(self.items):
            self._expand_op[i,i*self.item_size:(i+1)*self.item_size] = np.float32(1.0)

    def call(self, inputs, ops):
        op_mask_part = inputs[:self.items*self.item_size]
        if self.extra_size > 0:
            ext_part = inputs[self.items*self.item_size:]
        else:
            ext_part = None
        # if ops in [-0.5, 0.5] or [-0.5 .. 0.5]:
        ops1 = np.add(ops, np.float(0.5)) # optional
        extended_op = np.matmul(ops1, self._expand_op)
        if self.extra_size > 0:
            return np.concatenate((np.multiply(op_mask_part, extended_op), ext_part))
        else:
            return np.multiply(op_mask_part,extended_op)

このコードを使用すると、次のコードスニペットが示されます。

items = 5
item_size = 10
extra_size = 20
l = Layer(items=items, item_size=item_size, extra_size=extra_size)
l.build()
inputs = np.random.rand(items*item_size+extra_size)
ops = np.random.randint(0, 2, (items,), dtype="int")
ops = ops.astype(dtype=np.float32) - np.float32(0.5)
result = l.call(inputs,ops)

レイヤーコードから、各アクションに対してニューラルネットワークが一連の重みの形でアクションの表現を形成することを学習することがわかります。そして、これらの表現は、マスク層の後にネットワークに行くか、そうでないかのどちらかです。操作に応じて、これらのウェイトは、アクションウェイトのグループ全体に対するいくつかの操作のタスクで発生します（[0,1]による乗算だけでなく）。したがって、アクションのタスクは、ネットワークによって実行されるアクションのグループの値を計算するために形成されます。（古典的なケースでは、softmaxレイヤーがすべてのアクションの値を計算しました。提案されたアーキテクチャでは、ニューラルネットワークが選択されたアクションのグループの値を計算します。）

（たとえば、R.S。Sutton、EG Barto、強化学習を参照してください。）

提案されたアーキテクチャの使用例


テトリスゲーム


テトリスをプレイするためにこのアーキテクチャを使用する考え方は次のとおりです。入力では、テトリスゲームのガラスの画像（1ピクセル1正方形）を送信します。個々のアクションをアクショングループにグループ化します。ニューラルネットワークの1つのアクションの評価は、グラス内の図の最終的な位置のマスクです。図は、ニューラルネットワークのアクションマスクのレイヤーにあるアクションマスクの四角形で定義されます。アクションのグループを選択するには、現在の図のすべての終了位置のリストからアクション（終了）の最大評価を選択します。


お絵かき。フィールド（青いセル）と落下する形状（薄い灰色のセル）が表示されます。フィギュアの最終的な位置は、ゲームのルール（図示せず）に従ってフィギュアが移動できないすべての可能な位置です。

車の動きをシミュレートするエージェント


この場合、基本的なアクションとして、各加速アクション（数個の加速）、ブレーキ（ブレーキ中のいくつかの可能な加速）、および数度の回転がモデル化されました。同時に、回転および加速のうちの１つ、または１つ​​の回転動作または１つ​​の加速動作のみが関与し得ることが理解される。この場合、アーキテクチャでは、複数の基本アクションを同時に指定して、複雑なアクションを形成できます。


お絵かき。車のモデル（駐車目標が赤と緑の線で示されている）でアクションを実行するためのフィールドに加えて、ニューラルネットワークの入力（下）と、モデルのこの状態で可能なすべてのアクションに対するアクションの評価の値も表示されます。

アーキテクチャの他の可能なアプリケーション


同様に、ゲームでテトリスを使用すると、アーキテクチャは他のゲームにも使用でき、一連の数字といくつかのアクションを同時にフィールドで指定できます（たとえば、競技場内を移動する）。

ロボット工学では、このアーキテクチャは、個々の構造要素を共通のアンサンブルに調整するメタネットワークとして機能します。

また、このアーキテクチャでは、転移学習を使用してCNNパーツを事前トレーニングすることができます。逆の場合も同様に、最初にニューラルネットワークのRLパーツをトレーニングしてから、モデルデータのトレーニング済みのRLネットワークでCNNパーツをトレーニングします。この例では、テトリスゲームをプログラミングするときに、CNNパートとFCパートの最初にトレーニングを適用して転移学習をアクションマスクレイヤー（結果として得られるネットワークに転移されるもの）に適用しました。パーキングタスクでは、RLパート（つまり、最初のチェリー）を学習した後でCNNトレーニングを適用する予定です。
