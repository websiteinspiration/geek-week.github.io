Bonne après-midi!

Dans la partie précédente, nous avons créé avec succès notre centre de certification. Comment peut-il être utile à nos fins?

Avec l'aide d'un centre de certification local, nous pouvons émettre des certificats, ainsi que vérifier la signature de ces certificats.

Lors de l'émission d'un certificat à un utilisateur, l'autorité de certification utilise une demande spéciale pour émettre un certificat Pkcs # 10, qui a le format de fichier «.csr». Cette demande contient une séquence codée que l'autorité de certification sait analyser correctement. La requête contient à la fois la clé publique de l'utilisateur et les données de création du certificat (tableau associatif avec les données utilisateur).

Comment obtenir une demande de certificat, nous examinerons dans le prochain article, et dans ce document, je veux donner les principales commandes du centre de certification qui nous aideront à terminer notre tâche côté backend.

Donc, nous devons d'abord créer un certificat. Pour ce faire, utilisez la commande:

openssl ca -batch -in user.csr -out user.crt

ca - commande openSSL, qui fait référence au centre de certification,
-batch - annule les demandes de confirmation lors de la génération d'un certificat.
user.csr - demande de création d'un certificat (fichier au format .csr).
user.crt - certificat (le résultat de la commande).

Pour que cette commande fonctionne, le centre de certification doit être configuré exactement comme décrit dans la partie précédente de l'article . Sinon, vous devrez également spécifier l'emplacement du certificat racine de l'autorité de certification.

Commande de vérification de certificat:

openssl cms -verify -in authenticate.cms -inform PEM -CAfile /Users/……/demoCA/ca.crt -out data.file

cms est une commande openSSL qui est utilisée pour signer, vérifier, chiffrer les données et autres opérations cryptographiques à l'aide d'openSSL.

-verify - dans ce cas, nous vérifions le certificat.

authenticate.cms - un fichier contenant les données signées avec un certificat émis par la commande précédente.

-inform PEM - utilise le format PEM.

-CAfile /Users/DP/demoCA/ca.crt - chemin vers le certificat racine. (sans cela, la commande n'a pas fonctionné pour moi, bien que les chemins vers ca.crt soient écrits dans le fichier openssl.cfg)

-out data.file - J'envoie les données décryptées dans le fichier data.file.
