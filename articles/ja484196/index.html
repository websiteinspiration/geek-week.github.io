マイクからのJSサウンドまたは音声コメントの録音


少し前まで、企業のWebアプリケーションを1つ開発しているときに、顧客は音声コメントを残せることを望んでいました。以前は、メディアコンテンツの作成に遭遇することはなく、興味を持ってこのトピックを研究し始めました。

ネットワークは、この種のコンテンツの作成と処理に関するトピックに関する十分な背景情報を提供しましたが、単純で完全に機能する例は見つかりませんでした。お客様がタスクを実装した後、音声コメントの録音と保存の最も簡単な例を公開して記事を書くことにしました。おそらく、この資料は誰かにとって有用であり、研究に役立つでしょう。

問題の定式化


ブラウザで実行するミニアプリケーションを開発するタスクを設定します。これにより、音声コメントを録音し、録音をサーバーに送信できます。サーバーは録音を保存します。成功した場合、作成されたファイルの名前を含む応答を返し、オブジェクトをページに表示して録音を聞くことができます。

ブラウザに音声を録音する


サウンドレコーディングは、MediaStream Recording Web APIを使用して実装することが決定されました。録音には、MediaRecorder（）インターフェイスを使用します。しかし、最初にインターフェースを作成します。最も基本的なタグのみを含むindex.htmlを作成してみましょう。タグの本文には、将来のJavaScript voice.jsを含むファイルを含めます。

<!DOCTYPE html>
<head>
    <meta charset="UTF-8">
    <title>Voice comments</title>
</head>
<body>
    <script src="voice.js"></script>
</body>
</html>

voice.jsファイルを作成し、録音されたサウンドを受け取るスクリプトへのリンクを含むURL定数を定義します。次に、開始ボタンと停止ボタンを作成して、サウンドレコーディングを開始および停止します。また、保存されたレコードが表示されるdivブロックも作成します。これでインターフェースが準備できました。直接サウンドレコーディングに進むことができます。

録音についてはすでに述べたように、MediaRecorder（）インターフェイスを使用します（インターフェイスの詳細については、ドキュメントを参照してください）。その操作では、サウンドを取得するメディアストリームを決定し、オーディオトラックのみが必要な場合にのみ初期化する必要があります。

navigator.mediaDevices.getUserMedia({ audio: true})
    .then(stream => {
        const mediaRecorder = new MediaRecorder(stream)});

これで、インターフェイスのインスタンスを含む定数mediaRecorderが作成されました。これを引き続き使用します。

録音を開始するには、MediaRecorder.start（）メソッドを呼び出し、録音を停止するには、MediaRecorder.stop（）メソッドを呼び出す必要があります。この場合、MediaRecorder.stop（）はdataavailableイベントを生成し、それを介してバイナリ配列の形式でデジタル化されたサウンドレコーディングにアクセスします。

そして、上記のイベントについて説明し、音声[]配列を宣言し、受信したデータをそこに書き込みます。

navigator.mediaDevices.getUserMedia({ audio: true})
    .then(stream => {
        const mediaRecorder = new MediaRecorder(stream);
        let voice = [];
        document.querySelector('#start').addEventListener('click', function(){
            mediaRecorder.start();
        });

        mediaRecorder.addEventListener("dataavailable",function(event) {
            voice.push(event.data);
        });

        document.querySelector('#stop').addEventListener('click', function(){
            mediaRecorder.stop();
        });
    });

次に、受信したデータを送信する準備をします。これを行うには、停止イベントによってBLOBインスタンスを作成し、受信したデータをその中に入れ、MIMEデータタイプを指定します。私たちの場合は、audio / wavになります。

mediaRecorder.addEventListener("stop", function() {
    const voiceBlob = new Blob(voice, {
        type: 'audio/wav'
    });

その結果、音声メッセージを記録する将来のwavファイルの内容であるvoiceBlob定数が得られます。

レコードをサーバーに送信する


レコードをサーバーに送信するために、私はfetch（）メソッドを使用することにしました。このメソッドは最新のものであり、サーバーへのリクエストを行うための改良されたインターフェースを提供します。タスクの一部として、サーバーに保存するために将来のファイルのコンテンツを送信する本文でPOSTリクエストを開始する必要があります（fetch（）メソッドの仕組みと、ドキュメントに記載されている機能の詳細）。音声フィールドを使用して新しいフォームを作成し、レコードの内容をそこに挿入します。

let fd = new FormData();
fd.append('voice', voiceBlob);

サーバーにメッセージを送信して応答を受信し、オーディオオブジェクトを表示して既に保存されているファイルを再生するための非同期関数を作成します。引数として、関数は上記で作成された形式を取ります。

サーバーリクエストを開始します。

let promise = await fetch(URL, {
    method: 'POST',
    body: form});

サーバーからのHTTP応答にエラーコードが含まれていない場合（応答コードの範囲は200〜299）、応答をデコードし、ページに新しいオーディオオブジェクトを作成し、そのプロパティを定義して表示する必要があります。答えがどのように形成されるかについては、以下で説明します。

サーバーにファイルを保存する


音声メッセージを含むPOSTリクエストを受信するサーバー上にスクリプトを作成しましょう。送信したサウンドレコーディングは、基本的にはフォーム内のファイルであるため、サーバーで受信します。

$uploadDir = 'voice/';
$typeFile = explode('/', $_FILES['voice']['type']);
$uploadFile = $uploadDir . basename(md5($_FILES['voice']['tmp_name'].time()).'.'.$typeFile[1]);
if (move_uploaded_file($_FILES['voice']['tmp_name'], $uploadFile)) {
    $response = ['result'=>'OK', 'data'=>'../'.$uploadFile];
} else {
    $response = ['result'=>'ERROR', 'data'=>''];
}
echo json_encode($response);

ネットワーク上で受信したファイルを処理する、PHPコードの同様の例がたくさん見つかります。まず、変数を初期化します。$ uploadDir-受信したファイルが保存されるディレクトリ、この場合のファイルタイプとtypeFileは、wavとディレクトリを含む完全なファイル名になります。この場合のファイル名は、「一時」ファイル名と、md5メソッドを使用して暗号化された現在時刻の文字列値を組み合わせて形成されます。音声メッセージ付きのファイルを指定したディレクトリに正常に保存した場合、結果に応じて「OK」または「ERROR」に等しい結果フィールドと、処理が成功した場合に保存されたファイルへのリンクを含む「データ」フィールドを含む配列の形式で応答を形成します。

便宜上、配列をJSONオブジェクトに変換し、それを応答として送信します。
