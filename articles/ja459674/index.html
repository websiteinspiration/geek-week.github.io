脅威インテリジェンスエピックサガ



情報セキュリティの成熟度のある段階で、多くの企業は、関連するサイバー脅威に関する情報を取得して使用する方法について考え始めます。組織の業界仕様に応じて、さまざまな種類の脅威が関心を引き起こす可能性があります。そのような情報の使用へのアプローチは、材料であるインテリジェンスドリブンディフェンスでロッキードマーティンによって形成されました。

幸いなことに、ISサービスには、それらを取得するための多くのソースがあり、別のクラスのソリューションである脅威インテリジェンスプラットフォーム（TIP）さえ持っているため、受信、生成、およびセキュリティツールへの統合のプロセスを管理できます。

情報セキュリティインシデントの監視と対応の中心として、私たちが受け取って生成したサイバー脅威に関する情報が、関連性があり、適用可能であり、重要なことに管理可能であることが非常に重要です。結局のところ、インフラストラクチャの保護を私たちに委託している組織のセキュリティは、これに依存しています。

Jet CSIRTでのTIのビジョンを共有し、潜在的に有用なさまざまなアプローチをサイバー脅威インテリジェンスの管理に適応させる試みについて話し合うことにしました。

サイバーセキュリティの分野では、セット・アンド・フォーゲットの原則にほとんど取り組みません。ファイアウォールは、フィルターが設定されるまでパケットをブロックしません。IPSは、署名がダウンロードされるまで、トラフィックの悪意のあるアクティビティの兆候を検出しません。SIEMは、相関ルールを個別に記述し、誤検知を判定し始めます。脅威インテリジェンスも例外ではありません。

脅威インテリジェンスの概念を真に反映するソリューションの実装の複雑さは、その定義そのものにあります。

脅威インテリジェンスは、サイバーセキュリティを強化し、情報セキュリティの意識を高めるための対策を実行するために、現在のサイバー脅威に関する情報を取得して蓄積するための特定の情報源の調査と分析のプロセスです。特定のIBコミュニティ。

妥協の指標の優先順位付け


別の問題は、応答の優先順位付けです。比較的言えば、侵害の特定の指標を検出するときにどのようなSLAが得られるか。実際、脅威フィードプロバイダーは、含まれるエンティティに優先順位を付けません。消費者を支援するために、カスペルスキーのフィードで行われているように、エンティティの有害性にある程度の信頼を加えることができ



ます。

Jet CSIRTでこの問題に対処するために、COUNTERMEASURE 2016でRyan Kazanciyanによって概説されたアプローチを採用しました。その本質は、インフラストラクチャで検出できる侵害のすべての指標が、システムドメインとデータドメイン。

データドメインは3つのカテゴリに分類されます。

  • ソースでのリアルタイムアクティビティ（現在メモリに保存されているもの。情報セキュリティイベントをリアルタイムで分析することによって検出されます）：
    • プロセスの実行、レジストリキーの変更、ファイルの作成。
    • ネットワークアクティビティ、アクティブな接続;
    • 生成されたばかりの他のイベント。

    このカテゴリのインジケータが検出された場合、ISサービスからの応答時間は最小限です。
  • 過去の活動（すでに何が起こっているか、遡及チェック中に検出された）：
    • 履歴ログ。
    • テレメトリー;
    • トリガーされたアラート。

つまり、データドメインは、侵害インジケータが検出された分析データの状態です。

システムドメインは、インフラストラクチャサブシステムの1つに対する侵害インジケータのソースの所属です。

  • ワークステーション。ワークステーション、ラップトップ、タブレット、スマートフォン、端末（VoIP、VKS、IM）、アプリケーションプログラム（CRM、ERPなど）を使用してユーザーが毎日の作業を直接行うソース。
  • サーバー。これは、インフラストラクチャを提供する他のデバイスを指します。ITコンプレックスの操作用デバイス：SZI（FW、IDS / IPS、AV、EDR、DLP）、ネットワークデバイス、ファイル/ Web /プロキシサーバー、ストレージシステム、ACS、環境制御。環境など



この情報を妥協インジケーター（アトム、コンポジット）の兆候の構成と組み合わせることで、許容可能な反応時間に応じて、インシデントが検出されたときの優先度を定式化できます。



  1. 至急。インジケーターの検出には、対応チームからの迅速な対応が必要です。
  2. 限られた。インディケーターの検出には、インシデントの状況を明確にし、さらなるアクションを決定するための追加の分析が必要です。
  3. 無制限。指標の検出には、徹底的な調査と攻撃者の活動に関するレポートの作成が必要です。通常、このような調査結果はフォレンジックのフレームワークで調査され、何年も続く可能性があります。

どこ：

  • RC-リアルタイムの複合指標検出;
  • RA-リアルタイムでの原子インジケーターの検出;
  • HC-遡及チェックの一部としての複合指標の検出。
  • HA-遡及検証の一部としての原子インジケーターの検出。
  • DC-長い未使用のソースでの複合指標の検出。
  • DA-長い未使用のソースでのアトミックインジケーターの検出。

優先順位は、インジケーターの検出の重要性を損なうものではなく、インフラストラクチャの侵害の可能性を防ぐために必要なおおよその時間を示しています。

また、そのようなアプローチは、観察されたインフラストラクチャから切り離して使用することはできないことに注意してください。これに戻ります。

妥協指標の寿命の監視


妥協の兆候を永遠に残すいくつかの悪意のあるエンティティがあります。このような情報を長期間にわたって削除することはお勧めしません。これは、遡及監査（NA / HA）や、長期間未使用のソース（DC / DA）をチェックする場合に関連することがよくあります。

一部の監視センターと妥協の指標のサプライヤーは、一般的にすべての指標の寿命を制御する必要はないと考えています。ただし、実際には、このアプローチは効果がなくなります。



実際、悪意のあるファイルのハッシュ、マルウェアによって生成されたレジストリキー、ノードの感染に使用されたURLなどの侵害の兆候が正当なエンティティになることはありません。それらの有効性は制限されません。

適例：C＆Cサーバーとデータを交換するためのカプセル化されたSOAPプロトコルを使用したRAT Vermin ファイルの合計のSHA-256分析。分析は、ファイルが2015年に作成されたことを示しています。最近、お客様のファイルサーバーの1つで見つかりました。



ただし、悪意のあるキャンペーンのためにしばらくの間作成または「借用」されたエンティティのタイプは、まったく異なる状況になります。つまり、実際には、侵入者の制御下にあるエンドポイントです。

このようなエンティティは、所有者が侵害されたノードをクリーンアップした後、または攻撃者が次のインフラストラクチャの使用を停止したときに、再び正当なものになる可能性があります。

これらの要因を考慮に入れると、データドメイン、それらの関連性の期間、およびそれらのライフサイクルを制御する必要性に関して、妥協の指標の近似表を作成することができます。



この表のタスクは、3つの質問に答えることです。

  1. この指標は、時間の経過とともに正当になる可能性がありますか？
  2. 分析されたデータの状態に応じて、インジケーターの関連性の最小期間はどれくらいですか？
  3. このインジケーターの寿命を制御する必要はありますか？

IP C＆Cサーバーの例を使用して、このアプローチを検討してください。現在、攻撃者は分散インフラストラクチャを利用することを好み、多くの場合、アドレスを変更して、気付かれないようにし、プロバイダーからの妨害を回避しています。同時に、たとえばEmotetの場合のように、ハッキングされたノードにC＆Cがデプロイされることがよくあります。しかし、ボットネットが清算され、悪役が捕らえられているため、C＆CサーバーのIPアドレスなどのインジケーターが正当なエンティティになり、その寿命を制御できるようになります。

リアルタイム（RA / RC）でIP C＆Cサーバーへの呼び出しが見つかった場合、その関連性の期間は日数で計算されます。結局のところ、発見の翌日、このアドレスがC＆Cをホストしなくなることはまずありません。

通常、より長い間隔（数週間/月ごとに1回）がある遡及的チェック（HA / HC）でこのようなインジケーターが検出されると、対応する間隔に等しい最小の関連期間も示されます。同時に、C＆C自体はアクティブではなくなっている可能性がありますが、インフラストラクチャに循環の事実が見つかれば、指標は私たちに関連しています。

同じロジックが他のタイプのインジケーターにも適用されます。例外は、ハッシュ量、レジストリキー、Directory Authority（DA）TorネットワークノードおよびURLです。

ハッシュとレジストリ値を使用すると、すべてが簡単です。自然から削除することはできないため、その寿命を制御することは意味がありません。ただし、悪意のあるURLを削除することはできます。もちろん、それらは正当なものにはなりませんが、非アクティブになります。ただし、これらも独特であり、悪意のあるキャンペーンのために特別に作成されているため、正当なものになることはできません。

TorネットワークのDAノードのIPアドレスはよく知られており、変更されていません。それらの寿命はTorネットワーク自体の寿命によってのみ制限されるため、そのようなインジケーターは常に関連しています。
