これは通常のHelloWorldとは少し異なります。私は、規則としてすぐに採用するのが最善であるいくつかの複雑化に故意に行きました。特に、これは典型的な3層アーキテクチャです：View-Domain-DataAccess。

生命とは関係のない、人工的なものを示すよりも、すぐに従う方が良いと思います。実践が示すように、多くの開発者はHelloWorldレベルで立ち往生し、先験的に大きなアプリケーションを設計するときに見たものを使用します（すべてのアプリケーションコードが1つのファイルに含まれている元の国の最もグリーンな銀行とそのService.csにこんにちは、冗談ではありません）。

レイヤー間でデータをマッピングする方法を説明します。コードを何行も書く必要はありません。また、すべてをAzure DevOpsにデプロイします。はい、これは開発プロセスの必要な部分です。今日、全員がオンプレミス、AzureまたはAWSでクラウドを利用しています。

また、最初の記事がVisual Studioで 1つのボタンを押す方法についてのありふれた話にならないようにするために、小さなチャレンジを準備してLinuxで初めて何かを試すのが良いと思いました。

ASP.NET、C＃に関する最小限の知識があることを期待しています。これはHelloWorldですが、Program.csについては触れません。

。.Net Core 3.1のインストールから始めましょう

.Net Core 3.1をインストールする


.NETのコア3.1.1 SDKが利用可能な最新バージョンのSDK、それが利用可能であるUbuntuのリポジトリ。Blazorは非常に迅速に変更されるため、Blazor WebAssemblyを使用するには、最新の安定バージョンである3.1.1を使用します。

最初に、Microsoftキーとそのリポジトリを登録する必要があります。これは、ターミナルアプリケーションで次のコマンドを使用して行います（[ Win ]ボタンをクリックしてTermと入力します。使用可能なもののリストに表示されます）。



  1. wget -q https://packages.microsoft.com/config/ubuntu/19.10/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
  2. sudo dpkg -i packages-microsoft-prod.deb

次に、.Net Core SDKのインストールに直接進みます。

  1. sudo apt-get update
  2. sudo apt-get install apt-transport-https
  3. sudo apt-get update
  4. sudo apt-get install dotnet-sdk-3.1

Blazor WebAssemblyテンプレートをインストールする


.Net Core のインストールが完了したら、今度はBlazor WebAssemblyテンプレートをインストールする必要があります。これは、5月にすぐにリリースされるプレビュー段階にあるときに通知します。記事を書き始める前日に、更新されたバージョン3.2.0-preview1.20073.1（h）が1月28日にリリースされました

dotnet new -i Microsoft.AspNetCore.Blazor.Templates::3.2.0-preview1.20073.1

Gitのインストール


まだGitをインストールしていない場合は、今すぐインストールしてください。厳密に言えば、最初のコマンドはまったく必要ありません。Ubuntuはパスワードの入力を求めます。コマンドを実行するために特権が必要な場合は、それを予期します=）：

  1. sudo su
  2. add-apt-repository ppa:git-core/ppa
  3. apt update; apt install git

それだけではありません。Blazorはブラウザで実行されるときにMonoを使用します。つまり、Blazorもインストールする必要があります。

モノのインストール


最初に、Monoリポジトリをシステムに追加します。

  1. sudo apt install gnupg ca-certificates
  2. sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys 3FA7E0328081BFF6A14DA29AA6A19B38D3D831EF
  3. echo "deb https://download.mono-project.com/repo/ubuntu stable-bionic main" | sudo tee /etc/apt/sources.list.d/mono-official-stable.list
  4. sudo apt update

Mono 自体を直接インストールします。

sudo apt install mono-complete

これは、Core-i5 6200Uプロセッサーを搭載したラップトップで約10分かかった素早いプロセスです。ソースのコンパイルがコンピューターで直接行われるためです。

おめでとう！Blazor WebAssemblyプロジェクトの開発に必要なものがすべてインストールされています。
これで、プロジェクト自体に直接移動できます。

Dotnetユーティリティ


さらに、.Net Core SDK-dotnetのユーティリティを積極的に使用します。

それはあなたが多くの便利なことをすることを可能にします、私たちはいくつかに興味があります：

dotnet build

上記のコマンドはソリューションを収集します。ソリューションのあるフォルダーから開始する場合、引数は必要ありません。

dotnet new blazorwasm -ho -o BlazingPizza

これにより、newという単語の後に続くテンプレートに基づいて新しいプロジェクトを作成できます。この場合はblazorwasmです。たとえば、ここではclasslibまたはconsoleを指定できます。

次は、テンプレートの特定のパラメーターです。この場合は-ho、このパラメーターは、アプリケーションがASP.NET Coreアプリケーションのサーバー側でホストされることを示します。また、.Netがコントローラーを作成します。Web APIを使用して、Blazorアプリケーションからだけでなく、他のクライアントからもアクセスできます。

最後のオプション-oは、作成されたソリューションが配置されるフォルダーを示します。デフォルトでは、これは現在のディレクトリです。

dotnet run

名前からわかるように、便利な別のチームがアプリケーションを起動します。

そして、おなじみのチームを実行してソリューションを生成します。これには、Rider IDEが作成し、パス/ home / {userName} にあるRiderProjectsフォルダーなど、知っている場所を選択します。

dotnet new blazorwasm -ho -o BlazingPizza
