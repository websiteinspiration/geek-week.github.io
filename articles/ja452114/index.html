HolyJS 2019：SEMrushからの報告（パート1）



5月24〜25日にサンクトペテルブルクで開催されたHolyJS JavaScript開発者向けの定例会議で、当社のブースは皆に新しいタスクを提供しました。今回は3本！タスクが順番に与えられ、後続の各ソリューションの解決には記章が使用されました（JS Brave> JS Adept> JS Master）。これは停止しないことの良い動機として役立ちました。合計で約900の回答を収集し、最も人気のあるユニークなソリューションの分析を共有するために急いでいます。

提案されたテストは、言語の基本的な「機能」の勇気と理解、およびECMAScript 2019の新機能への認識から期待されています（実際、後者は必要ありません）。これらのタスクは面接用ではなく、実用的ではなく、単に楽しむために考えられていることが重要です。

タスク1〜カウントダウン式


式は何を返しますか？任意の1文字を

  1. 式が2を返しました
  2. 式が1を返しました

+(_ => [,,~1])().length

さらに、順列によって0を取得することは可能ですか？

式は何を返しますか？


ここで考えるのは長くありません。式は3を返します。無名関数には3つの要素の配列を返すだけで、最初の2つは空（empty）です。関数呼び出しはこの配列を提供し、そこから長さ（length）を取得し、単項プラスは何も解決しません。

式は2を返します


迅速な修正により、返された配列の要素数が減少したようです。これを行うには、カンマを1つ捨てます。

[,~1].length // 2

そのような記号を捨てることは不可能です。元の式の別の場所に再配置する必要があります。ただし、配列のlengthプロパティは、1つの場合を除いて、配列リテラルにリストされた空の要素を考慮に入れることを知っています。

要素が配列の最後で省略されている場合、その要素は配列の長さに影響しません。

つまり、空の要素が配列の最後にある場合、それは無視されます。

[,10,] // [empty, 10]

したがって、修正された式は次のようになります。

+(_ => [,~1,])().length // 2

このカンマを取り除く別のオプションはありますか？または運転しました。

式は1を返します


少なくとも2つの置換を行う必要があるため、配列のサイズを小さくして1を取得することはできなくなりました。別のオプションを探す必要があります。

関数自体が決定を示唆します。JavaScriptの関数は、独自のプロパティとメソッドを持つオブジェクトであることを思い出してください。また、プロパティの1つはlengthでもあり、これは関数が予期する引数の数を決定します。私たちの場合、関数には1つの引数（下線）があります-必要なもの！

ためには長さがしていない配列から、関数から取られ、あなたは括弧を通してそれを呼び出しを停止する必要があります。これらの括弧の1つが置換の候補であることが明らかになります。そして、いくつかのオプションが思い浮かびます：

+((_ => [,,~1])).length // 1
+(_ => ([,,~1])).length // 1

たぶん他に何かありますか？または次のレベル。

式は0を返します


追加のタスクでは、順列の数は制限されません。しかし、最小限の動きでそれを実現しようとすることができます。関数オブジェクトから長さを使用し

て以前のエクスペリエンスを開発すると、すぐに解決策を見つけることができます。

+(() => [,_,~1]).length // 0

いくつかの派生物がありますが、全体の要点は、関数の引数の数をゼロに減らしたことです。これを行うには、3つの順列が必要でした。2つの角かっこと、配列の要素となるアンダースコア文字です。

まあ、でも、推論で加算（+）演算子とビットごとのNOT（〜）演算子を無視するのをやめる時が来たのではないでしょうか。この問題の算術は私たちの手に渡るようです。最初に戻らないようにするために、これが元の式です。

+(_ => [,,~1])().length

まず、〜1を計算します。x のビット単位のNOTは-（x + 1）を返します。つまり、〜1 = -2です。また、式には加算演算子もあります。これは、別の場所でさらに2つを見つける必要があり、すべてが機能することを示唆しています。 最近、配列リテラルの最後の空の要素がそのサイズに及ぼす「影響なし」を思い出しました。これは、デュースがどこかにあることを意味します。



[,,].length // 2

そして、すべてが非常にうまく加算されます。配列から要素〜1を取得し、その長さを2に減らして、式の先頭に追加するための最初のオペランドとして追加します。

~1+(_ => [,,])().length // 0

したがって、すでに2つの順列で目標を達成しています！

しかし、これが唯一の選択肢ではない場合はどうでしょうか？少しドラムロール...

+(_ => [,,~1.])(),length // 0

ユニット後の点（数値タイプのみであるため、これは、可能である。また、2個の順列が必要数）と前コンマ長さ。ばかげているように見えますが、「時々」機能します。なぜ時々？

この場合、コンマ演算子による式は2つの式に分割され、結果は2番目の計算値になります。しかし、2番目の式は長さです。実際、ここではグローバルコンテキストで変数の値にアクセスしています。ランタイムがブラウザーの場合、window.length。そして、ウィンドウオブジェクトは、実際に持っている長さのプロパティフレームの数を返します（フレームを）ページ上。ドキュメントが空の場合、lengthは0を返します。はい、想定を使用したオプションです。したがって、前のオプションについて詳しく説明します。

そして、ここで発見されたいくつかのより興味深いオプションがあります（すでに異なる順列の数に対応しています）：

(_ => [,,].length+~1)() // 0
+(~([,,].len_gth) >= 1) // 0
~(_ => 1)()+[,,].length // 0
~(_ => 1)().length,+[,] // 0
~[,,]+(_ => 1()).length // 0

コメントはありません。誰かがもっと楽しいものを見つけますか？

動機


「マッチを1つまたは2つ並べ替えて正方形にする」という昔ながらのタスク。ロジックの開発と創造的思考のためのよく知られたパズルは、そのようなJavaScriptバリエーションではあいまいなものに変わります。これは役に立ちましたか？はいよりもないでしょう。ソリューションの根底に到達するために、言語の多くの機能に触れ、一部はかなり概念的なものにさえ触れました。しかし、実際のプロジェクトは、機能や発現からステロイドまでの長さについてのものではありません。このタスクは、JavaScriptとは何かを思い出すための中毒性のあるトレーニングの一種として会議で提案されました。

評価の組み合わせ論


考えられるすべての回答が考慮されたわけではありませんが、まったく見逃さないように、実際のJavaScriptを見てみましょう！あなたがそれらを自分で見つけようとすることに興味がある場合は、上記の十分なヒントがあり、それ以上読み進めない方が良いでしょう。



したがって、23文字の式があり、その文字列レコードで順列を行います。全体で、元の式レコードでn *（n-1）= 506順列を実行して、（問題1と2の条件で必要とされる）1つの置換シンボルを持つすべてのバリアントを取得する必要があります。

関数を定義組み合わせますこれは、入力式を文字列および述語として受け取り、この式を実行して取得した値の適合性をテストします。この関数は、すべての可能なオプションをブルートフォースとによって式を評価しますevalを：オブジェクトに結果を保存し、キー -受信した値、値 -この値の私たちの表現の変異のリストを。このようなことが起こりました：

const combine = (expr, cond) => {
  let res = {};
  let indices = [...Array(expr.length).keys()];
  indices.forEach(i => indices.forEach(j => {
    if (i !== j) {
      let perm = replace(expr, i, j);
      try {
        let val = eval(perm);
        if (cond(val)) {
          (res[val] = res[val] || []).push(perm);
        } 
      } catch (e) { /* do nothing */ }
    }
  }));
  return res;
}

式の渡された文字列から のreplace関数は、位置iから位置jに文字を再配置した新しい文字列を返します。そして今、あまり恐れずに、私たちはやります：

console.dir(combine('+(_ => [,,~1])().length', val => typeof val === 'number' && !isNaN(val)));

その結果、一連のソリューションが得られました。

{
"1": [
  "+(_ => [,,~1]()).length",
  "+((_ => [,,~1])).length",
  "+(_ =>( [,,~1])).length", 
  "+(_ => ([,,~1])).length"
],
"2": [
  "+(_ => [,~1,])().length"
]
"3": [/* ... */]
"-4": [/* ... */]
}

3と-4の解は私たちには興味がなく、2つは唯一の解を見つけたので、ユニットには[,,〜1]（）の興味深い新しいケースがあります。TypeErrorでない理由：bla-blaは関数ではありませんか？そして、すべてが単純です。この式は構文パーサーにエラーがなく、実行時に関数が呼び出されないため、実行されません。

ご覧のとおり、1つの順列ではゼロの問題を解決することはできません。二人でやってみませんか？このような徹底的な検索によって問題を解決すると、この場合、文字列の長さの複雑さO（n ^ 4）が得られ、何度も「評価」され、追究され罰せられますが、好奇心は広まります。結合機能を個別に変更することは難しくありません または、特定の表現を考慮に入れて、より適切な検索を作成します。 

console.dir(combine2('+(_ => [,,~1])().length', val => val === 0));

結局、ゼロの解のセットは次のようになります。

{
"0": [
  "+(_ => [,~.1])(),length",
  "+(_ => [,~1.])(),length",
  "~1+(_ => [,,])().length"
]
}

推論では、ユニットの後にドットを再配置したことは奇妙ですが、ユニットの前のドットも可能であることを忘れていました。ゼロを省略して、0.1を記録しました。

2つの文字それぞれの可能なすべての順列を実行すると、3から-4の範囲の値に対して非常に多くの答えがあることがわかります：

{ "3": 198, "2": 35, "1": 150, "0": 3, "-1": 129, "-2": 118, "-3": 15, "-4": 64 }

したがって、2つの順列でのカウントダウン式ソリューションパスは、提案されたパス3から1への0より長くなる可能性があります。
