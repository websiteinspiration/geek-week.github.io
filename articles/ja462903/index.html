サーバーとユーザーネットワークのペアリング


ユーザーファクトリでは、さまざまな目的で個別のVRFが使用されています。ユーザーワークステーション、ビデオ監視システム、会議室のマルチメディアシステムを接続したり、スタンドやデモゾーンを整理したり

するために使用されます。サーバーファクトリでは、さまざまなVRFセットが作成されています。

  • 企業サービスが展開されている通常のサーバーを接続するため。
  • 別のVRF。インターネットからアクセスできるサーバーが展開されます。
  • 他のサーバー（アプリケーションサーバーなど）のみがアクセスするデータベースサーバー用の個別のVRF。
  • メールシステム（MS Exchange + Skype for Business）用の個別のVRF。

したがって、ユーザーファクトリ側からのVRFセットとサーバーファクトリ側からのVRFセットがあります。どちらのセットも企業ファイアウォール（ME）でクラスター化されています。MEは、サーバーファクトリとユーザーファクトリの両方の境界スイッチに接続されます。

移行はどのように行われましたか


移行中、一時的なトランクを介して、チャネルレベルで新旧のサーバーファクトリを接続しました。特定のVLANにあるサーバーを移行するために、古いサーバーファクトリのVLANと新しいサーバーファクトリのVXLANを含む別個のブリッジドメインを作成しました。

構成は次のようになります。キーは最後の2行です。

bridge-domain 22
 vxlan vni 600022
 evpn 
  route-distinguisher 10.xxx.xxx.xxx:60022
  vpn-target 6xxxx:60022 export-extcommunity
  vpn-target 6xxxx:60022 import-extcommunity

interface Eth-Trunk1
 mode lacp-static
 dfs-group 1 m-lag 1

interface Eth-Trunk1.1022 mode l2
 encapsulation dot1q vid 22
 bridge-domain 22
