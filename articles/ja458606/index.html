DockerでOpenVPNを2秒で実行する

こんにちは、ハブロフスク市民！事実上別の都市、国、または別の大陸に旅行したいという状況に遭遇したことがありますか？私はこのようなニーズを頻繁に抱えているため、数秒でどこでも起動できる独自のVPNサーバーを使用できる可能性は非常に深刻でした。この記事では、既成のソリューション（この場合はdocker）を探していたときに計画したプロジェクトについてお話したいと思います。最低限の設定と許容可能なレベルのセキュリティで、OpenVPNサーバーをすばやく立ち上げることができるイメージ。


バックグラウンド

任意のマシンでサービスを実行する機能：物理サーバーでも仮想プライベートサーバーでも、一般的には別のコンテナー管理システム内のコンテナースペースでも、重要でした。私の視線はすぐにDockerに落ちました。第1に、このサービスの人気が高まっているため、プレインストール済みのターンキーソリューションを提供するプロバイダーが増えています。次に、イメージの集中リポジトリがあり、そこからターミナルで1つのコマンドでサービスをダウンロードして開始できます。そのようなプロジェクトはすでに存在しているはずだという考えが私を訪れ、私はしつこく検索しました。しかし、私が見つけたほとんどのプロジェクトは、面倒すぎる（永続的なデータストレージ用のコンテナーを作成し、異なるパラメーターを持つアプリケーションでコンテナーを数回実行する必要があった）か、まともなドキュメントなしでした。許容できるものを見つけられなかったので、私は自分のプロジェクトに取り組み始めました。ドキュメントの研究、コードの記述、デバッグの前に眠れない夜がありましたが、結局、私のサービスは光を見て、ルーターのモノクロLEDパネルのすべての色で遊んでいました。だから愛してください-Docker-OpenVPN。私はロゴ（上、カットの前）も思いつきましたが、私はデザイナーではないので、厳密には判断していません（既に）。試行錯誤により、これらの基準の最適なバランスを見つけましたが、セキュリティのために展開速度を犠牲にする必要があり、最小限の設定では移植性を支払う必要がありました。現在の構成では、あるサーバーで作成されたコンテナーを別のサーバーに転送して実行することはできません。たとえば、クライアントとサーバーのすべての証明書はサービスの開始時に生成され、これには約2秒かかります。ただし、Defi Hellmanファイルの生成はビルド時に行う必要がありました。Dockerイメージのビルド中に作成され、最大10分間続く可能性があります。尊敬されるコミュニティから、このようなソリューションのセキュリティ監査を受けたいと思います。

発売

サービスを開始するには、いくつかのものが必要です。
  1. サーバー：物理または仮想。理論的には、docker-in-dockerモードで実行できますが、私はこのオプションの広範なテストを実施していません。
  2. 実際にはDocker。多くのホスティングプロバイダーは、Dockerを「オンボード」でターンキーソリューションに提供しています。
  3. パブリックIPアドレス。
すべての詳細が整っている場合は、サーバーのコンソールで次のコマンドを実行する必要があります。
docker run --cap-add=NET_ADMIN \
-it -p 1194:1194/udp -p 80:8080/tcp \
-e HOST_ADDR=$(curl -s https://api.ipify.org) \
alekslitvinenk/openvpn
注意深い読者は、サーバーのIPアドレスがipify.orgを使用して自動的に決定されることに注意してください。何らかの理由でこれが機能しない場合は、アドレスを手動で指定できます。前のすべての手順が正しく完了している場合は、コンソールに同様のものが表示されます。
Sun Jun  9 08:56:11 2019 Initialization Sequence Completed
Sun Jun  9 08:56:12 2019 Client.ovpn file has been generated
Sun Jun  9 08:56:12 2019 Config server started, download your client.ovpn config at http://example.com/
Sun Jun  9 08:56:12 2019 NOTE: After you download you client config, http server will be shut down!
これで目標に近づきました。今度はexample.comをコピーし（この場合はサーバーのアドレスが表示されます）、ブラウザーのアドレスバーに貼り付けます。Enterキーを押すと、client.ovpnファイルがダウンロードされ、httpサーバー自体が忘却されます。この解決策が疑わしい場合は、次のトリックを使用できます。前のコマンドを実行し、zpフラグとパスワードを追加します。これで、生成されたリンクをブラウザーウィンドウに貼り付けると、パスワード付きのzipアーカイブが届きますクライアント構成のファイルがある場合は、任意の適切なクライアントを使用できます。Mac用のTunnelblickを使用しています。
