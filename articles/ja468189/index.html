コンソールトリックを使用してJavaScriptデバッグスキルを向上させる

これは、Medium.comのBetter Programming Blogの記事の翻訳です。著者であるIndrek Lasnは、JavaScriptが提供するコードデバッグツールについて語っています。



JavaScriptコードの通常のデバッグは、メソッドを使用して結果を出力することconsole.logです。ただし、この方法は機能しますが、最適とは言えません。デバッグをもっと便利にしてみませんか？



オブジェクトconsoleは、ブラウザのデバッグコンソールへのアクセスを提供します。 JavaScriptコードがブラウザーで起動された場合にのみ使用できます（つまり、これはクライアントパーツのコードであり、サーバーパーツのコードではありません）。実装consoleはブラウザによって異なる場合がありますが、原則として、常にいくつかの基本的な機能があります。デバッグオペレーターの最も良い点は、すべてのライブラリーとフレームワークを操作できることです（それらは言語のコアに組み込まれているため）。

最も簡単な使用例console.logはコード出力です。たとえば、次のコードを見てください。

function sayHello(name) {
  console.log(name)
}

sayHello('Indrek')

関数に渡される名前を出力しますsayHello。


関数に渡された名前を表示し

、我々は関数が呼び出された回数を知りたい場合はsayHello？複雑なことは何もありません。これに使用していconsole.count()ます。

console.count


この関数count()は、指定されたパラメータを持つ呼び出しの数を表示しますlabel。label指定しない場合、デフォルト設定の通話数が表示されます。

function sayHello(name) {
  console.count()
  console.log(name)
}

sayHello("Indrek")
sayHello("William")
sayHello("Kelly")

上記のコードは以下を表示します


。sayHello関数の呼び出し数のカウント

この方法で、関数の呼び出しの総数を確認できます。そして、もしあなたが同じ名前の関数呼び出しの数を知る必要があるなら？それは簡単です-引数nameをメソッドに渡すだけcountです。

function sayHello(name) {
  console.count(name)
}

sayHello("Indrek")
sayHello("William")
sayHello("Kelly")
sayHello("Indrek")

出来上がり！この関数は、ある名前または別の名前で関数を呼び出した回数を追跡します。


各名前の言及の数を数える

console.warn


このメソッドはコンソールに警告を表示します-APIまたは開発ツールを使用するときに役立ちます。console.warnそれは常に何かがうまくいっていないことを示すことができます：引数が欠落している、APIのバージョンが古いなどです。

function sayHello(name) {
  if(!name) {
    console.warn("No name given")
  }
}

sayHello()

このコードは、引数が関数に渡されたかどうかを確認しますname。これが発生しない場合、警告がコンソールに表示されます。


名前が転送されないという警告

console.table


配列やオブジェクトを操作する場合、データを表示する場合、このメソッドは非常に役立ちconsole.tableます。そのおかげで、テーブルの各要素に個別の行が割り当てられます。

果物の配列の例でこのメソッドの操作を見てみましょう。配列をに渡すconsole.tableと、次のようになります。

const fruits = ["kiwi", "banana", "strawberry"]

console.table(fruits)

コンソールを見ると、配列を説明する表が表示されます。


テーブルの形式で表示された

ます配列の操作は簡単です。しかし、配列ではなくオブジェクトの場合はどうでしょうか？

const pets = {
  name: "Simon",
  type: "cat"
};

console.table(pets);

したがって、配列の代わりに、ペットのname（名前）とtype（タイプ）の2つのキーを含むオブジェクトがあります。



以前は、テーブルには値のみが表示されていましたが、現在は値とキーが含まれています。そして、別のオブジェクトを取り、それをテーブルに追加しようとした場合はどうでしょうか。

const pets = {
  name: "Simon",
  type: "cat"
};

const person = {
  firstName: "Indrek",
  lastName: "Lasn"
}

console.table(pets, person);

予想通り、2つの別々のオブジェクトが2つの異なるテーブルに表示されます。


2つのオブジェクト

これらを1つのテーブルに結合する必要があるため、両方のオブジェクトを配列に配置します。

const pets = {
  name: "Simon",
  type: "cat"
};

const person = {
  firstName: "Indrek",
  lastName: "Lasn"
}

console.table([pets, person]);

次に、テーブル内のオブジェクトをグループ化します。


オブジェクトを配列に配置してグループ化する

console.group


セットまたは関連データを操作する場合、ネストされたグループを使用できます。これは、コンソールに視覚的に関連するメッセージを表示できるので便利です。新しいネストされたグループを作成するには、を呼び出す必要がありますconsole.group()。

console.log("This is the first level");
console.group();
console.log("Level 2");
console.group();
console.log("Level 3");
console.warn("More of level 3");
console.groupEnd();
console.log("Back to level 2");
console.groupEnd();
console.log("Back to the first level");

このコードは、コンソールに情報を含む埋め込みブロックを表示します。これは、表形式で提示されたデータを操作するときに役立ちます。



メソッドconsole.groupCollapsed()は同様に機能しますが、新しいブロックはデフォルトで折りたたまれています。それを読むには、デプロイボタンをクリックする必要があります。
