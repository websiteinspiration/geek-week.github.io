Apolloオンラインチャット、Node.jsのバックエンドアプリケーションの作成

以前、私はモバイルアプリケーションに取り組んでいました。その機能には、便利なオンラインチャットが含まれていました。そして今度は、バックエンドでapolloサーバーとnode.jsを使用してチャットを作成する方法、およびクライアント側でネイティブクライアントとapolloクライアントを反応させる方法についての簡単な説明を含む記事を書くことにしました。

記事は読みやすくするために2つの部分に分かれています。最初の部分にはアプリケーションバックエンドを作成するためのガイドが含まれ、2番目の部分にはアプリケーションフロントエンドを作成するためのガイドが含まれます。

読むのが面倒な場合は、Github'eのコードがこことここにすぐに表示されます。

実装の主なテクノロジーとして、node.js koaフレームワーク、postgresqlデータベース、GraphQLサーバー-apollo -server-koaを選択しました。

まず最初に、空のkoa2プロジェクトが生成されました。このため、ターミナルでコマンドを実行して簡単なkoa-generatorを使用しました。 

$ koa <project name>

次に、必要な依存関係がインストールされました。これは、yarnを使用して行いますが、npmを使用できます。

$ yarn add apollo-server-koa knex objection pg

必要なすべてのライブラリがインストールされ、コードを記述できるようになりました


データベースに接続するには、2つのファイルを記述する必要があります。1つ目はdb.jsです。これはknexクライアントインスタンスをエクスポートし、モデルがデータベースからのデータを操作できるようにします。2つ目はknexfile.jsで、作成とローリングマイグレーション。

以下にdb.jsコードについて説明します。すべての設定は環境変数から取得されることに注意してください。

const db = require('knex')({
 client: 'pg',
 connection: {
   host : process.env.POSTGRES_HOST,
   port: process.env.POSTGRES_PORT,
   user : process.env.POSTGRES_USER,
   password : process.env.POSTGRES_PASSWORD,
   database : process.env.POSTGRES_DATABASE
 }
});

module.exports = db;

コードknexfile.jsはこちらから入手できます。

これで、移行について説明して、必要な2つのテーブルを作成できます。


テーブル自体は可能な限り単純で、最小限必要なフィールドセットのみが含まれます。それらを作成するコマンドは次のとおりです。

$ knex migrate:make migration_name

ここで 移行ファイルを表示できます。

次に、エンティティモデルのメッセージとユーザーを作成します。


class Message extends Model {
 static get tableName() {
   return 'messages';
 }

 $beforeInsert() {
   this.created_at = new Date().toISOString();
 }

 static get relationMappings() {
   return {
     user: {
       relation: Model.BelongsToOneRelation,
       modelClass: User,
       join: {
         from: 'messages.user_id',
         to: 'users.id'
       }
     }
   };
 }
}

最も興味深いものは残りました-apollo-server-koaの接続と構成、graphqlスキームとリゾルバーの説明。

apollo-server-koaに接続するには、次のコード行を追加するだけです


app.js：

const { ApolloServer } = require('apollo-server-koa');
const graphqlSchema = require('./graphqlSchema');
…
const apolloServer = new ApolloServer(graphqlSchema);

apolloServer.applyMiddleware({ app });

www：

var { app, apolloServer } = require('../app');
...
apolloServer.installSubscriptionHandlers(server);

apollo-server-koaの接続に加えて、サブスクリプションを使用して、チャットに新しいメッセージが届いたことを顧客に通知する機能が含まれています。

Apollo-server-koaが接続されています。次のステップは、チャットに必要なタイプを含むgraphql-schemeの説明です


input UserInput {
 username: String!
}
input MessageInput {
 text: String!
 user_id: ID!
}
type User {
 id: ID
 username: String
}
type Message {
 id: ID
 text: String
 created_at: String
 user: User
}
type Query {
 getLast100Messages: [Message]
}
type Mutation {
 findOrCreateUser(user: UserInput!): User
 createMessage(message: MessageInput!): Message
}
type Subscription {
 messageCreated: Message
}

回路の準備ができました。レゾルバについて説明します


新しいメッセージを送信するためのリゾルバーの例：

const Message = require('../../models/message');
const { pubsub, MESSAGE_CREATED } = require('../../utils');

module.exports = {
 createMessage: async (obj, { message }, context, info) => {
   const createdMessage = await Message
     .query()
     .insert(message);

   const resultMessage = await Message
     .query()
     .eager('user')
     .findById(createdMessage.id);
   pubsub.publish(MESSAGE_CREATED, { messageCreated: resultMessage });
   return resultMessage;
 },
};

興味深い点は、メッセージをデータベースに保存することに加えて、publish（）関数がここで呼び出され、MESSAGE_CREATEDイベントについてすべてのサブスクライバーに通知し、新しいメッセージオブジェクトを送信します（注意深い読者は、送信者にも新しいメッセージが通知されることに気づき、これをさらに処理します）クライアントにとって、実際のプロジェクトでは、これをバックエンド側で処理して、異なるクライアント間でロジックが重複しないようにするのは理にかなっています）。
