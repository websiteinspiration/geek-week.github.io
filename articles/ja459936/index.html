Visual Studioを操作するための9つのトリック

最近では、Visual Studioを操作するための18のトリッ​​クについて話しました。本日、ツールを使用した作業の生産性を高めるのに役立つ9つのヒントを選択しました。

1.最後に編集したコード行に戻りたいですか？最後の編集場所に移動するには、Ctrl + Shift + Bkspceを試してください。

2. Set Next Statementを使用して、デバッグ中に命令ポインタ（黄色の矢印）を簡単に移動します。



3. Ctrl + Shift + F12を使用して、エラーリストにリストされている次のエラーのソースに移動します。

4.選択した単語のすべての出現箇所を選択するには、Ctrl + Shift +セミコロンを押して複数の挿入ポイントのサポートを使用し、それらをすべて同時に選択して編集可能にします。

5. CancellationTokenからJTF.SwitchToMainThreadAsync（）に移行するときは注意してください。直感に反して、これはタスク全体のキャンセルではなく、スイッチのキャンセル...なので、コードはバックグラウンドで動作し続けます。

6. Visual StudioがSurface Dialで動作することを知っていますか？無料のSurface Dialツール拡張機能を使用して、ブックマーク、エラー、デバッグ、スケーリングなどを簡単にナビゲートできます。



7。Ctrl + Shift + Alt + F12をダブルクリックして、Visual Studioがガベージコレクションを自分で実行するようにします。8

. 開発アシスタントを使用してVisual Studioでコードスニペットを検索します。



9.「私のファイルはどこにありますか？」デフォルトでは、Visual Studioプロジェクトのファイルパスは非常に長いです。Windowsエクスプローラーからプロジェクトのサブフォルダーまたはファイルにアクセスする場合、またはコードでファイルへのパスを指定する場合は、目的のファイルの場所を探して、フォルダーを長い間上下に移動できます。コードエディターウィンドウのタブを右クリックするだけで、ファイルパスを開いたり、エクスプローラーでフォルダーを開いたりできることをご存知ですか？これをチェックして。
