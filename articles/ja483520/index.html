Cでの類似パターンオブザーバーの使用

最近、Javaのデザインパターンに関する本を読んだときのことを思い出しましたが、OOPのメモリからこれを行う方法がまだわからないため、Cで実装することにしました。

私はこのパターンを実装する方法について、このJavaブックをもう一度見ることにしました。私は章全体を読み、コードを調べましたが、頭には何も残っていませんでした。何故ですか？ ）おっとスタイルで考えるのは正しいですか？たとえば、JavaとC ++については少し知っていますが、実装を快適にして忘れられないようにするにはどうすればよいでしょうか。また、実験のために、これがC ++でどのように行われるかを見ました。正直なところ、そのようなコードを見るのは難しいので、すべての行を読んで考えなければなりません。

それで、将来プログラミングに役立つ可能性のあることは何ですか。 GitHubで実装を満たします。私が書いたものの使いやすさを説明したいと思います。この実装では、データを格納する構造を作成する必要はありません。根底にある構造自体が舞台裏にあります。たとえば、TRADEパブリッシャーを1つのファイルで購読したいとします。たとえば、この発行元は製品の割引を提供します。私はオブザーバーを通常の関数と考え、ルールに従って宣言しています。

void sub_trade ( void *event, void *data ) {
  /*        . */
}

次に、パブリッシャーを作成し、イベントをサブスクライブしたいと思います。この場合、余分なものを呼び出す必要はありません。サブスクライバーを転送するパブリッシャーに示す必要があるだけです。

enum { TRADE };
init_publisher ( TRADE, sub_trade, NULL );

以上です。私たちは出版社を購読しました。たとえば、別のファイルで、オンラインチャットでニュースレターをやりたいと思っています。OOPでこれを行うのがどれほど便利か、さらにはJavaでそれがさらに便利かどうかはわかりませんが、私の実装の助けを借りて、これは単純に行われます。ちなみに、Javaはおそらく静的メソッドを使用する必要があります。

別のサブスクライバー関数を宣言しています。

enum { TRADE };
void notify_chat_trade ( void *event, void *data ) {
  /*       */
}
...
init_publisher ( TRADE, notify_chat_trade, NULL );

これで、新しいデータが受信および解析されるファイルに、割引データが入ります。このような関数を呼び出すだけで、必要に応じてデータが転送されます。

/*      10,      . */
send_event ( TRADE, 10 );

また、使いやすくて使いやすく、目的の結果を得ることができます。うまくできたと思います。
