TONテストクライアントのインストール手順

先日、ご存じのとおり、Telegram Open Network テストクライアントがパブリックドメインにリリースされました（TONの詳細についてはこことここ、テストネットワークの操作についてはここを参照）。私はすぐにそれを着始めました。READMEに記載されているインストール手順は幸運にも単純で、励みになりました。しかし、問題はすぐに始まりました。

当初、総会は全体の約3分の1に達し、エラーで落ちました。 libssl-devパッケージがインストールされているにもかかわらず、tests.cppファイルにエラーが送信され、本質は「sha256はこのスコープで宣言されていません」でした。私は認め、私はこの問題の解決に多くの時間を費やしました。パブリックテレグラムチャットのユーザーの1人がヘルプに応答し、次のコマンドを実行するように勧められました。

sudo apt install zlib1g-dev

そして、驚いたことに、問題は解決されました！もう一度私は組み立てを始めました、しかし今、彼女は完了する運命にありませんでした。アセンブリの83％で新しいエラーが発生し



ました。インターネットでこの問題の解決策が見つからず、何か別のバージョンのUbuntuでビルドするように言われました（19.04を使用して、18.04で試すことにしました）。

ライブCDから起動しましたが、他にもいくつかの問題がありました（すでに小さくなっています）。まず、不足しているコンポーネントをインストールする必要がありました。

sudo apt install build-essential
sudo apt install libssl-dev
sudo apt install zlib1g-dev
sudo apt install cmake

アセンブリを開始しましたが、エラーが最初に発生しました：cmakeがgperfパッケージがないことについて不平を言った、そしてコマンド

sudo apt install gperf

データベースにそのようなパッケージがないため、実行されませんでした。これは、次のスプリング（/etc/apt/sources.list）を追加することで解決しましたdeb http://cz.archive.ubuntu.com/ubuntu cosmic main universe。

もう一度インストールを始めたところ95％に達しましたが、やはり苦労しました。今回は、この現象が何らかの理由で私のコンピュータでのみ発生したようですが、実際には、アセンブリの95％がハングしました。さらに、システム全体がハングしました。彼女は単にどんな行動にも反応しなくなった。初めてUSBフラッシュドライブからのブートとして書き留めたので、ソケットから少し外れただけで、システム全体がクラッシュする可能性があります。私は再起動し、再試行しましたが、今回はフラッシュドライブがコネクタにしっかりと装着されていることを確認しました。しかし、この状況は不運な95％でも繰り返されました！その結果、ハードドライブで19.04から18.04にロールバックして、もう一度試す以外に方法はありませんでした。幸い、今回はすべてが正常に終了し、クライアントが起動しました。

これらすべての問題を解決するのに長い時間がかかり、インターネットでそれらの解決策を見つけることができなかったので、すべての問題を解決したチームのセットをオープンアクセスにしたいと考えています。私はこれが誰かが時間を節約するのに役立つと信じたいです。

sudo apt install cmake
sudo apt install libssl-dev
sudo apt install zlib1g-dev
sudo apt install build-essential
sudo xdg-open /etc/apt/sources.list  #   ,

test.ton.org/download.html からアーカイブをダウンロードし、ホームフォルダーに解凍して、ton-test-liteclient-fullではなくlite-clientフォルダーが含まれるようにします。端末を再起動します。

mkdir ~/liteclient-build
cd ~/liteclient-build
cmake ~/lite-client
cmake --build . --target test-lite-client
cmake --build . --target fift
wget https://test.ton.org/ton-lite-client-test1.config.json
./test-lite-client -C ton-lite-client-test1.config.json  #

インストールが成功するかどうかは、ディストリビューションのバージョン、およびシステムがインストールされているか、Liveから起動しているかに依存するという問題があったことを思い出してください。私の成功した結果は、Ubuntuディスクにインストールされたバージョン18.04で得られました。
