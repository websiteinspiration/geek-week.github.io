サプライズバスケット-プッシュを保存

TLDR：MercurialからGitへのBitbucketリポジトリの移行を自動化する一連のスクリプトを作成しました。

ある晴れた日、私のお気に入りのBitbucketリポジトリホスティングサービスは、Gitを支持してMercurialリポジトリのサポートをすぐに停止し、その後すべてのMercurialリポジトリが削除されることを発表しました。



私はBitbucketに多くのプライベートMercurialリポジトリを持っています。コミット履歴のように、それらを失いたくありませんでした。別のホスティングに移動することもオプションではありません-私はBitbucketに慣れています。奇妙なことに、Bitbucket自体はイ​​ンプレースコンバーターを作成できませんでした。彼らは、変換するための段階的な指示を書いておらず、コミュニティのフォーラムに全員を送りました-むしろイエズス会の言葉遣いで（私たちはあなたの移行をサポートさせていただきます、そして私たちの専用コミュニティスレッドで利用可能なオプションについての議論を見つけることができます）-彼らは言う私たちはあなたのリポジトリをGitの下に移動することを嬉しく思います、そしてその方法-フォーラムで自分でそれを議論してください。ただし、同じ投稿では、hg-fast-exportとhg-gitコンバーターのペアへのリンクを残しました。

1つ目は個別のPythonスクリプトで、2つ目はMercurialから直接Gitリポジトリを操作するためのプラグインです。 Googleはまた、Gitの下でMercurialをドラッグアンドドロップするタスクを他の人がどのように解決したかを示す多数の例を見つけました：lo、behold、lo。また、hg-fast-exportまたはhg-gitのいずれかを使用しました。 2番目はすでに実証済みのソリューションでした-MercurialからGithubにコミットするときにhg-gitを使用することもありました-Gitで奇妙なことに、Gitからコミットする方がよいことに気づくまでしたがって、私はhg-gitを選択しました。

ここにリポジトリの移行を自動化するためのPowershellスクリプトが提供されており、これが私のソリューションの出発点となりましたが、Linuxの場合はBashに書き直す必要がありました。さらに、元のスクリプトはローカルのGitリポジトリにのみ移行するので、変換されたGitリポジトリのコンテンツをそこのBitbucketリポジトリにプッシュする機能を追加しました。結果のスクリプトは次のようになります。

これで既製のソリューションが手に入りましたが、手動での操作が必要でした-ソリューションを起動する前に、まずBitbucketにGitリポジトリを作成して、結果をプッシュするための多くのことを行う必要がありました。変換する100,500のリポジトリを見て、私はそれらを長時間処理することに気づきました。変換されたもの用のGitリポジトリをスクリプト自体で作成したいと思います。さらに良いことに、Mercurialリポジトリのリストを受け取って作業します。これは明らかに純粋なMercurialのスキルを超えたものであり、ここではBitbucket自体の機能、つまりBitbucket APIが必要でした。

さまざまなプログラミング言語、特にPython用のBitbucket APIを使用するための既成のライブラリがあります。明らかに公式のSDKです。（古くなった放棄と混同しないでください。ただし、Bitbucket自体にある公式のSDKと同じです）。Java、NodeJS、およびPHP用のBitbucket APIクライアントもあります。私が選んだ最後のもの。

ただし、詳細に調査したところ、ユーザーリポジトリのリストを取得して新しいリポジトリを作成することは実装されていませんでした。幸い、ライブラリの作成者は、プライベートメソッドを保護することでクラスを拡張する可能性を提供しました。そのため、APIクライアントのクラスを継承することで、必要なことを行う独自のメソッドを追加できます。

class ExtendedClient extends Client
{
    /**
     * @return \Bitbucket\Api\Repositories
     */
    public function repositories()
    {
        return new ExtendedRepositories($this->getHttpClient());
    }
}

class ExtendedRepositories extends Repositories
{    
    public function listWorkspace(string $workspaceName, array $params = [])
    {
        $path = $this->buildRepositoriesPath($workspaceName);
        return $this->get($path, $params);
    }

    public function create(string $workspaceName, string $repoName, array $params = [])
    {
        $path = $this->buildRepositoriesPath($workspaceName, $repoName);

        return $this->post($path, $params);
    }
}
