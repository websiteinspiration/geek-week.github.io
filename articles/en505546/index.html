Flutter and desktop development

The idea of ‚Äã‚Äãwriting multi-platform applications is far from new. Flutter also provides the ability to do this. In this article I will try to describe two approaches to launching a mobile application on the desktop, which I myself use to develop mobile applications. I stopped running the emulator and simulator largely because there was an opportunity to do without them. For those who are interested in the idea, welcome to cat.

So why bother writing a desktop version bypassing emulators and simulators?

First, workstation resources . My current commercial project uses a lot of animations and various ‚Äúheavy‚Äù widgets. On my very good laptop with i7, 16GB of RAM and a GTX1050Ti graphics card, the processor jumps to 30-35% in debug mode on very complex transitions and animations, the android emulator eats up to 2GB of RAM. Frames, of course, are lost and animations move ‚Äújerkily‚Äù, which does not allow us to properly assess the completeness of the picture. When the desktop version is launched, my processor ‚Äúrests‚Äù and is loaded with the same animations up to 3-4% and this, for a minute, saves the processor 8-10 times and the RAM 300-400MB.

Secondly, the ability to quickly view the correct placement of widgets on various devices. Just take a window and stretch it, or squeeze it to the minimum size and this is a huge saving of your personal time and the time of fellow testers.

The first option to launch the desktop version


Using, now part of Flutter - flutter-desktop-embedding.

Everything is simple here. We copy the windows, linux, macos folders to ourselves in the working project. We write in the console the permission command to use the OS libraries:

flutter config --enable-windows-desktop
flutter config --enable-linux-desktop
flutter config --enable-macos-desktop

All. Now, when choosing a device, a desktop emulator with the name Linux, Macos, Windows appears, depending on the platform. We select, launch, work.
We get the platform identical to the emulator with Hot Reload and Hot Restart.

Pros:

  1. the most necessary plugins, for example, file_chooser (selecting folders in the system), window_size (resizing windows) are already there
  2. everything works out of the box
  3. release

Sample plugin for emulating emulator window sizes


The easiest way is to copy the Go folder from the sample project repository, for example, from github.com/go-flutter-desktop/examples/tree/master/text_demo

Next, in the Go directory, create the plugins / desktop_utils folders and in it a file with our future plugin, let's name it main.go and write to it

following code: 
package desktop_utils

import (
	"github.com/go-flutter-desktop/go-flutter"
	"github.com/go-flutter-desktop/go-flutter/plugin"
	"github.com/go-gl/glfw/v3.3/glfw"
)

const channelName = "go-test/desktop_utils"

type DesktopUtilsFlutterPlugin struct {
	window *glfw.Window
	channel *plugin.MethodChannel
}

var _ flutter.Plugin = &DesktopUtilsFlutterPlugin{}
var _ flutter.PluginGLFW = &DesktopUtilsFlutterPlugin{}

func (p *DesktopUtilsFlutterPlugin) InitPlugin(messenger plugin.BinaryMessenger) error {
	channel := plugin.NewMethodChannel(messenger, channelName, plugin.StandardMethodCodec{})
	
	channel.HandleFunc("set_window_size", p.setWindowSizeHandler)

	return nil
}

func (p *DesktopUtilsFlutterPlugin) InitPluginGLFW(window *glfw.Window) error {
	p.window = window

	return nil
}

func (p *DesktopUtilsFlutterPlugin) setWindowSizeHandler(arguments interface{}) (reply interface{}, err error) {
	argumentsMap := arguments.(map[interface{}]interface{})

	var minW = int(argumentsMap["minW"].(int32))
	var minH = int(argumentsMap["minH"].(int32))
	var maxW = int(argumentsMap["maxW"].(int32))
	var maxH = int(argumentsMap["maxH"].(int32))

	p.window.SetSizeLimits(minW, minH, maxW, maxH);

	return nil, nil
}

channelName ‚Äî Flutter
minW, minH, maxW, maxH ‚Äî /

We initialize our file as a Go-module: go mod init desktop_utils (the go.mod file will appear).

We go to the Go directory, find the go.mod file, add the local directory of our plugin: replace desktop_utils => ./plugins/desktop_utils
and add our plugin to options.go in the options line:

var options = []flutter.Option{
	flutter.WindowInitialDimensions(InitW, InitH),

	flutter.AddPlugin(&desktop_utils.DesktopUtilsFlutterPlugin{}),
}

On this Go-part is over.

The Flutter part is as simple as possible: we only need to create a channel (MethodChannel) with our name: go-test / desktop_utils and pass the length and width parameters we need to it:

const MethodChannel _utilsChannel = MethodChannel('go-test/desktop_utils');

_utilsChannel.invokeMethod(
  'set_window_size',
   <String, dynamic>{
     'minW': 400,
     'minH': 600,
     'maxW': 800,
     'maxH': 1200,
   },
 );

That's all. Our plugin is ready.
