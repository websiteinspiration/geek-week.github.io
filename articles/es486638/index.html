Blazor WebAssembly 3.2.0 Preview 1 ya está disponible

Hoy lanzamos una nueva actualización de vista previa para Blazor WebAssembly con muchas nuevas funciones y mejoras.

Estas son las novedades de esta versión:

  • Versión actualizada a 3.2.
  • Lanzamiento simplificado
  • Descargar mejoras de tamaño
  • .NET SignalR Client Support



comienzo


Para comenzar con Blazor WebAssembly 3.2.0 Preview 1,  instale .NET Core 3.1 SDK  y luego ejecute el siguiente comando:

dotnet new -i Microsoft.AspNetCore.Blazor.Templates::3.2.0-preview1.20073.1

¡Eso es todo! Puede encontrar documentos y ejemplos adicionales en  https://blazor.net .

Actualizar proyecto existente


Para actualizar una aplicación Blazor WebAssembly existente de 3.1.0 Vista previa 4 a 3.2.0 Vista previa 1:

  • Actualice todos los enlaces a los paquetes Microsoft.AspNetCore.Blazor * a 3.2.0-preview1.20073.1.
  • En  Program.cs  en el proyecto del cliente Blazor WebAssembly, reemplace  BlazorWebAssemblyHost.CreateDefaultBuilder() con  WebAssemblyHostBuilder.CreateDefault().
  • Mover los registros de componentes de la raíz en el proyecto de cliente Blazor WebAssembly a partir  Startup.Configure de  Program.cs por  llamar  builder.RootComponents.Add<TComponent>( ).
  • Blazor WebAssembly  Startup.ConfigureServices  Program.cs   builder.Services.
  •  Startup.cs  Blazor WebAssembly client project.
  • Blazor WebAssembly ASP.NET Core,  Server   app.UseClientSideBlazorFiles<Client.Startup>(...)  app.UseClientSideBlazorFiles<Client.Program>(...).

3.2


En esta versión, actualizamos las versiones del paquete Blazor WebAssembly a 3.2 para distinguirlas de la versión reciente de .NET Core 3.1 Long Term Support (LTS). No hay una versión correspondiente de .NET Core 3.2: la nueva versión 3.2 se aplica solo a Blazor WebAssembly. Blazor WebAssembly se basa actualmente en .NET Core 3.1, pero no hereda el estado de .NET Core 3.1 LTS. En cambio, la versión inicial de Blazor WebAssembly, programada para mayo de este año, será Actual , que se "mantiene durante tres meses después de la próxima versión de Actual o LTS", como se describe en la Política de soporte de .NET Core . La próxima versión próxima de Blazor WebAssembly después de la versión 3.2 en mayo será con .NET 5. Esto significa que después del lanzamiento de .NET 5, deberá actualizar sus aplicaciones Blazor WebAssembly a .NET 5 para admitirlo.

Lanzamiento simplificado


En esta versión, hemos simplificado el lanzamiento y el alojamiento de las API para Blazor WebAssembly. Inicialmente, las API de lanzamiento e implementación de Blazor WebAssembly se diseñaron para reflejar los patrones utilizados en ASP.NET Core, pero no todos los conceptos eran relevantes. Las API actualizadas también permiten utilizar algunos escenarios nuevos.

Así es como se ve el nuevo código de inicio en Program.cs :

public class Program
{
    public static async Task Main(string[] args)
    {
        var builder = WebAssemblyHostBuilder.CreateDefault(args);
        builder.RootComponents.Add<App>("app");

        await builder.Build().RunAsync();
    }
}

Las aplicaciones Blazor WebAssembly ahora admiten Mainmétodos asincrónicos para el punto de entrada a la aplicación.

Para crear un host predeterminado, llame WebAssemblyHostBuilder.CreateDefault(). Los componentes y servicios raíz se configuran utilizando el enlazador; Startupya no se necesita una clase separada . Se agrega el siguiente ejemplo WeatherService, por lo que está disponible a través de la inyección de dependencia (DI):

public class Program
{
    public static async Task Main(string[] args)
    {
        var builder = WebAssemblyHostBuilder.CreateDefault(args);
        builder.Services.AddSingleton<WeatherService>();
        builder.RootComponents.Add<App>("app");

        await builder.Build().RunAsync();
    }
}

Después de crear el host, puede acceder a los servicios desde el DI raíz antes de presentar cualquier componente. Esto puede ser útil si necesita ejecutar alguna lógica de inicialización antes de que se muestre algo:

public class Program
{
    public static async Task Main(string[] args)
    {
        var builder = WebAssemblyHostBuilder.CreateDefault(args);
        builder.Services.AddSingleton<WeatherService>();
        builder.RootComponents.Add<App>("app");

        var host = builder.Build();

        var weatherService = host.Services.GetRequiredService<WeatherService>();
        await weatherService.InitializeWeatherAsync();

        await host.RunAsync();
    }
}

El host ahora también proporciona una instancia de la configuración central de la aplicación. La configuración no se rellena con ningún dato predeterminado, pero puede rellenarla según sea necesario en su aplicación.

public class Program
{
    public static async Task Main(string[] args)
    {
        var builder = WebAssemblyHostBuilder.CreateDefault(args);
        builder.Services.AddSingleton<WeatherService>();
        builder.RootComponents.Add<App>("app");

        var host = builder.Build();

        var weatherService = host.Services.GetRequiredService<WeatherService>();
        await weatherService.InitializeWeatherAsync(host.Configuration["WeatherServiceUrl"]);

        await host.RunAsync();
    }
}

Descargar mejoras de tamaño


Las aplicaciones Blazor WebAssembly ejecutan el enlazador .NET IL en cada ensamblaje para eliminar el código no utilizado de la aplicación. En versiones anteriores, solo se cortaban las bibliotecas principales de framework. A partir de esta versión, los ensamblajes Blazor también se recortan, lo que resulta en una ligera reducción de tamaño, alrededor de 100 KB. Como antes, si alguna vez necesita deshabilitar el enlace, agregue el suyo propio <BlazorLinkOnBuild>false</BlazorLinkOnBuild> a su archivo de proyecto.

.NET SignalR Client Support


Ahora puede usar SignalR desde sus aplicaciones Blazor WebAssembly usando el cliente .NET SignalR.

Para probar SignalR en su aplicación Blazor WebAssembly:

  1. Cree una aplicación Blazor WebAssembly alojada en ASP.NET Core.

    dotnet new blazorwasm -ho -o BlazorSignalRApp
  2. Agregue el paquete del cliente ASP.NET Core SignalR al proyecto del Cliente .

    cd BlazorSignalRApp
dotnet add Client package Microsoft.AspNetCore.SignalR.Client
  3. En el proyecto Servidor , agregue la siguiente clase Hub / ChatHub.cs .

    using System.Threading.Tasks;
using Microsoft.AspNetCore.SignalR;

namespace BlazorSignalRApp.Server.Hubs
{
    public class ChatHub : Hub
    {
        public async Task SendMessage(string user, string message)
        {
            await Clients.All.SendAsync("ReceiveMessage", user, message);
        }
    }
}
  4. En el proyecto Servidor , agregue los servicios SignalR en el método Startup.ConfigureServices.

    services.AddSignalR();
  5. Agregue también el punto final para ChatHubat Startup.Configure.

    .UseEndpoints(endpoints =>
{
    endpoints.MapDefaultControllerRoute();
    endpoints.MapHub<ChatHub>("/chatHub");
    endpoints.MapFallbackToClientSideBlazor<Client.Program>("index.html");
});
  6. Actualice Pages / Index.razor en el proyecto del Cliente con el siguiente marcado.

    @using Microsoft.AspNetCore.SignalR.Client
@page "/"
@inject NavigationManager NavigationManager

<div>
    <label for="userInput">User:</label>
    <input id="userInput" @bind="userInput" />
</div>
<div class="form-group">
    <label for="messageInput">Message:</label>
    <input id="messageInput" @bind="messageInput" />
</div>
<button @onclick="Send" disabled="@(!IsConnected)">Send Message</button>

<hr />

<ul id="messagesList">
    @foreach (var message in messages)
    {
        <li>@message</li>
    }
</ul>

@code {
    HubConnection hubConnection;
    List<string> messages = new List<string>();
    string userInput;
    string messageInput;

    protected override async Task OnInitializedAsync()
    {
        hubConnection = new HubConnectionBuilder()
            .WithUrl(NavigationManager.ToAbsoluteUri("/chatHub"))
            .Build();

        hubConnection.On<string, string>("ReceiveMessage", (user, message) =>
        {
            var encodedMsg = user + " says " + message;
            messages.Add(encodedMsg);
            StateHasChanged();
        });

        await hubConnection.StartAsync();
    }

    Task Send() => hubConnection.SendAsync("SendMessage", userInput, messageInput);

    public bool IsConnected => hubConnection.State == HubConnectionState.Connected;
}
  7. Crear y ejecutar un proyecto de servidor

    cd Server
dotnet run
  8. Abra la aplicación en dos pestañas separadas del navegador para comunicarse en tiempo real a través de SignalR.

Problemas conocidos


La siguiente es una lista de problemas conocidos en esta versión que se solucionarán en una actualización futura.

  • Al iniciar una nueva aplicación Blazor WebAssembly alojada en ASP.NET Core desde la línea de comandos, se genera una advertencia:

    CSC : warning CS8034: Unable to load Analyzer assembly C:\Users\user\.nuget\packages\microsoft.aspnetcore.components.analyzers\3.1.0\analyzers\dotnet\cs\Microsoft.AspNetCore.Components.Analyzers.dll : Assembly with same name is already loaded.
    • Solución: esta advertencia se puede ignorar o eliminar mediante la propiedad  <DisableImplicitComponentsAnalyzers>true</DisableImplicitComponentsAnalyzers> MSBuild.
