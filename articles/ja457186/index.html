Visual Studio CodeのPython-6月のリリース

2019年6月のVisual Studio Code用のPython拡張機能がすでに利用可能であることをお知らせいたします。Python拡張機能は、Marketplaceからダウンロードするか、Visual Studio Codeの拡張機能ギャラリーから直接インストールできます。Python拡張機能がすでにインストールされている場合は、Visual Studio Codeを再起動するだけで最新のアップデートを入手することもできます。Visual Studio CodeのPythonサポートについて詳しくは、ドキュメントをご覧ください。

このリリースでは、変更ログに記載されている改善を行い、Pythonインタラクティブウィンドウを使用したグラフビューアーやpytestを使用した並列テストに関連する問題など、合計70の問題を解決しました。猫の下のすべての変更についてお読みください。



Pythonウィンドウでのビューアーのプロット


グラフは一般的にデータを視覚化するために使用されます。Pythonインタラクティブウィンドウの最も人気のある機能の1つは、作成されたグラフをより詳細に確認できる機能です（たとえば、スケーリング、パン、エクスポートなど）。6月の更新には、matplotlibライブラリなどからのグラフ画像の操作に使用できる完全に新しいビューアが含まれています。

ビューアを開くには、グラフをダブルクリックするか、Pythonインタラクティブウィンドウでグラフィック画像にカーソルを合わせると表示される[画像の最大化]ボタンをクリックします。



グラフビューアを使用すると、グラフの移動、ズームインとズームアウト、現在のセッションでのグラフの切り替え、PDF、SVGまたはPNGへのエクスポートが可能です。

pytestによる並列テスト


テストの実行、特にpytestと並行してテストを実行する場合に表示される統計の信頼性が向上しました。

pytestと並行してテストを実行するには、pytest-xdistパッケージをインストールし、構成ファイルに「-n <プロセッサ数>」を追加します。たとえば、4つのプロセッサの場合、プロジェクトフォルダにpytest.iniファイルを作成し、それに次のコンテンツを追加できます。

[pytest]
addopts=-n4

これで、テストを実行してデバッグすると、並行して実行されます。Python拡張機能でのテストサポートの詳細については



、ドキュメントを参照してください。

端末での選択的開始時にインデント


私たちのGitHubリポジトリでは、VS Code Python関数が非常に人気があり、「Run Selection / Line in Python Terminal」コマンドが実行されたときに端末に送信する前に、選択したコードの一般的なインデントを削除しました。このリリース以降、チームは、選択したコードの最初の空でない行に基づいて、削除された共通インデントを含むターミナルコードを送信します。



その他の変更と改善。


また、マイナーな改善を追加し、ユーザーから報告された問題を修正しました。以下は、いくつかの注目すべき変更です。

  • jupyterを使用したリモート認証にパスワードが許可されています。（＃3624）
  • Data Viewerは、大きなデータフレームを正しく処理し、数値列の式によるフィルタリングをサポートします（多かれ少なかれ等しい）（＃5469）
  • インポートされたラップトップをインタラクティブなPythonウィンドウでプレビューします。（＃5675）
  • テストのデバッグ時にサブプロセスをデバッグするためのサポートが追加されました。（＃4525）
  • PowerShellでconda環境をアクティブ化するためのサポートが追加されました。（＃668）
  • 現在のセルを起動するキーボードショートカットとして「Ctrl + Enter」を追加します。（＃5673）

上記のイノベーションを評価するには、Visual Studio Code用のPython拡張機能を今すぐインストールして ください。問題が発生した場合は、Python VS Code GitHubページで報告してください。
