Ausführen, PHPUnit, Ausführen: Wie ich die Testausführungszeit optimiert habe

Von Beginn der Arbeit an einem der Webprojekte an strebten wir eine hohe Codeabdeckung mit Tests an, und in der Anfangsphase der Entwicklung dachte ich nicht daran, die Geschwindigkeit ihrer Ausführung zu optimieren. Infolgedessen stieg mit dem Wachstum des Projekts, der immer größeren Abdeckung seiner Tests und dem Wachstum des Teams die Testausführungszeit von einigen Sekunden auf einige zehn Minuten. Schnelle Tests können genauso wichtig sein wie die Leistung der gesamten Anwendung.


Wie habe ich das bekämpft und was ist am Ende passiert?


Was kam vor den Optimierungen


Die Anwendung wird in PHP 7.2 und Symfony 4.2 entwickelt. Zum Schreiben von Tests verwenden wir PHPUnit. Die meisten Tests sind Integrationstests für die REST-API.


Führen Sie die Tests aus:


> bin/phpunit
...
Time: 20.17 minutes, Memory: 400.25 MB
OK (1494 tests, 5536 assertions)

Alle Tests dauerten ca. 20 Minuten. Dies führte zu folgenden Problemen:


  • Der automatische CI-Prozess wurde für die Dauer der Tests blockiert, was zur Erschöpfung der freien CI-Läufer führte.
  • Entwickler mussten sich in einer virtuellen Warteschlange anstellen und auf die Fertigstellung der CI-Pipelines warten.
  • Während der Dauer aller Tests konnten Sie vor Ort eine Tasse Kaffee kochen und in den Nachrichten blättern, was nur den Wunsch weckte, den Start einfach zu ignorieren.


Fangen Sie an, nach den wichtigsten Engpässen zu suchen und zu experimentieren!

#!/bin/sh

php_no_xdebug() {
  temporaryPath="$(mktemp -t php.XXXXXX).ini"

  # Using awk to ensure that files ending without newlines do not lead to configuration error
  php -i | grep "\.ini" | grep -o -e '\(/[a-z0-9._-]\+\)\+\.ini' | grep -v xdebug | xargs awk 'FNR==1{print ""}1' | grep -v xdebug >"$temporaryPath"

  php -n -c "$temporaryPath" "$@"
  rm -f "$temporaryPath"
}
