RFC (Request For Comments) zur Verbesserung der Ember Registry API


Chris Cricho ( @chriskrycho ) schlug vor, eine neue objektorientierte API für alle Registrierungsschnittstellen hinzuzufügen. Deklarieren die aktuelle API zeichenkettenbasierten microsyntax und eine Eigenschaft einführen capabilities, um die vorhandene Erkennungs-Engine sicher zu unterstützen .


RFC: Enhanced Ember Registry API beschreibt aktuelle Probleme mit Mikrosyntax und mangelnde TypeScript-Unterstützung. RFC empfiehlt das Hinzufügen einer neuen API Identifier , die Felder enthalten name, typeund gegebenenfalls namespaceanstelle des aktuellen microsyntax. Zum Beispiel:


stattdessen


getOwner(this).lookup('service:session');

wird sein


getOwner(this).lookup({ type: 'service', name: 'session' })

aber stattdessen


class Example {
  @service('global@session')
  session;
}

wird sein


class Example {
  @service({ namespace: 'global', name: 'session' })
  session;
}

Wenn Sie mehr wissen oder Ihre eigene Bewertung hinzufügen möchten, schauen Sie sich RFC auf GitHub an !
