スネークシュガーまたはJavaScriptで範囲を記述

多くの人々はPythonを愛しています...初心者はセミコロンがないことに賞賛され、上級者は真のシンプルさを楽しんでいます。今日は、JavaScriptでPythonの非常に単純さの類似物、特にrange関数を実装する方法について説明します。

Pythonでは、範囲関数を反復するか、たとえば配列- リスト（範囲（開始、終了））に変換できます。

しかし問題は、JavaScriptが同様のものを作成し、同時にソリューションをネイティブでシンプルに見えるようにすることは可能ですか？

最初に頭に浮かぶのは、同様のクラスを書くことです。

function range(from, to, step = 1){
    this.current = from
    this.to = to
    this.step = step

    this.next = () => this.current++ % to
}

はい、次はトリミングされた値を返すと書きましたが、これは解決方法を理解する必要がある問題でもあります。最大値に切り替えるか、何か他のことをするとエラーがスローされますか？

一般に、このようなソリューションは、すべての利便性を追加しません、標準を書くことが容易になりますため、たとえば、あなたが追加することができ、エンド機能を、各反復のチェックの後、我々は海外に行っている場合は、このソリューションを開発するにはポイントがありません。

ES6では、新しいプリミティブデータ型であるシンボルが導入されています。彼は私たちの前にメタプログラミングへの扉を開きます。範囲に関する

問題を解決するには、Symbol.iteratorを使用します。特定のプロトコルを使用してイテレータオブジェクトを作成することができます。

オブジェクトをイテレータにするためには、その中に[Symbol.iterator]（）関数を定義する必要がありますが、クラスではなく共通の関数を作成するため、イテレータオブジェクトを返す関数を作成します。

これは次のようになります。

function range(from, to, step = 1){
    return {
        [Symbol.iterator](){
            return {
                current: from,
                to: to,
                from: from,
                step,
                next(){
                    const it = { done: this.current >= this.to, value: this.current }
                    this.current += this.step
                    return it
                }
            }
        }
    }
}

何が起こっていますか？範囲関数があり、同じように[Symbol.iterator]（）関数がオブジェクトを返すイテレータオブジェクトを返します。最も重要なことは、このオブジェクトに次の関数が含まれていることです。そうでない場合、オブジェクトは反復可能ではなく、使用しようとするとエラーが表示されます。次の関数は、（！）を持つオブジェクト返す必要が行わと値の特性、行わ（BOOL）の反復の終了を通知し、値は現在の値を含みます。

原則として、すべてが機能し、Pythonのようなものをすでに記述できます。

    for(let i of range(0, 10, 2))
        console.log(i)

結論：

0
2
4
6
8

ここで止めることができます...しかし、ジェネレーター関数を使用して、よりエレガントなソリューションを作成できます。

function range(from, to, step = 1){
    return {
        *[Symbol.iterator](){
            for(let val = from; val < to; val += step){
                yield val;
            }
        }
    }
}

追加のプラスは、イテレータに他の機会があるという事実です。たとえば、範囲を配列に非常に簡単に変換できます。

[...range(0, 10, 2)]
// [ 0, 2, 4, 6, 8 ]

結論として、私は当初、類似の類似の実装のリストを含む記事を書くつもりでしたが、他には何も思いつかなかったと言いたいです。そして、いきなりアイデアが浮かんだら、これをサイクルとして続けていきます。
