匿名の内部クラス


Javaでは、内部クラスはクラスのメンバーとして定義されたクラスです。内部クラスには4種類あります。

  • 静的ネスト
  • 内部
  • ローカル（メソッドローカル）
  • 匿名の

匿名内部クラスは、既存のクラスの実装を提供する名前のないクラスです。このようなクラスは、イベント指向プログラミングでイベントを処理するために広く使用されています。通常、匿名の内部クラスは、抽象クラスのオンザフライの実装を提供します。ただし、これは必須ではありません。匿名の内部クラスは、非抽象クラスから作成できます。

匿名の内部階級に関しては、十分に理解されていない点があると思います。実際には、プログラマは実際にソースクラスのサブクラスを作成します。このサブクラスには、外部クラスを表す名前Class$XとClass、外部クラスでX内部クラスのインスタンスが作成される順序を表す番号が付けられます。例えば、AnonDemo$3-で作成された3番目の内部クラスAnonDemo。これらのクラスを通常の方法で呼び出すことはできません。また、他のタイプの内部クラスとは異なり、匿名の内部クラスは常に、それが作成された元になったタイプに基づいて暗黙的に子クラスになりますvar（これについては後で説明します）。
例を見てみましょう。

/* AnonDemo.java */
class Anon { };

public class AnonDemo {
   public static void main (String[] args) {
       Anon anonInner = new Anon () {
           public String toString() {
               return "Overriden";
           };
           public void doSomething() {
               System.out.println("Blah");
           };
       };
       System.out.println(anonInner.toString());
       anonInner.doSomething(); //  !
   };
};

この例では、Anonクラスに基づいて匿名の内部クラスをインスタンス化しました。基本的に、特定のクラスの名前のないサブクラスを作成しました。Java 10より前は、匿名の内部クラスはほとんどの場合暗黙的にポリモーフィックでした。もちろん、このような非ポリモーフィックなコードが実行されるため、「ほぼ」と言います。

new Anon() { public void foo() { System.out.println("Woah"); } }.foo();

ただし、匿名の内部クラスのインスタンスを作成した結果を元の型に割り当てたい場合、そのような操作は本質的に多態性になります。これは、匿名の内部クラスのソースとして指定したクラスのサブクラスを暗黙的に作成し、特定のオブジェクトタイプ（Class$X）にアクセスしてソースコードで指定できないためです。

ポリモーフィズムと匿名の内部クラス、実際の結果


上記のコードに気づきましたか？サブクラスオブジェクトへの基本クラス参照を使用するため、多態性の法則に従って、1）基本クラスで定義されたメソッド、または2）サブクラスでオーバーライドされた仮想メソッドのみを参照できます。

したがって、前のコードフラグメントではtoString()、匿名の内部クラスオブジェクトを呼び出すと、オーバーライドされた「オーバーライドされた」値が返されますが、呼び出しdoSomething()によってコンパイルエラーが発生します。理由は何ですか？

基本クラスのタイプへの参照を持つサブクラスオブジェクトは、基本クラスへのこの参照を通じてサブクラスのメンバーにアクセスできません。このルールの唯一の例外は、サブクラスが基本クラスのメソッドをオーバーライドする場合です。この場合、Javaはその多態性に忠実であり、Dynamic Method Dispatchを使用して、実行時に仮想サブクラスメソッドのバージョンを選択します。

まだ知らない場合、仮想メソッドはオーバーライドできるメソッドです。Javaでは、すべての非final、非private、非staticメソッドはデフォルトで仮想です。私はデフォルトで話しますが、暗黙ではありません。異なるjvmがこれを変更する可能性のある最適化を実行できるためです。

Java 10はそれとどのような関係がありますか？


型推論と呼ばれる小さな機能。次の例を見てください。

/* AnonDemo.java */
class Anon { };
public class AnonDemo {
  public static void main(String[] args) {
    var anonInner = new Anon() {
      public void hello() {
       System.out.println(
       "New method here, and you can easily access me in Java 10!\n" +
       "The class is:  " + this.getClass()
       );
      };
    };

    anonInner.hello(); // !!
  }
}

機能しましたhello()。悪魔は細部にあります。に精通している場合はvar、ここで何が行われているのかすでに理解しています。予約された型名を使用してvar、Javaは匿名内部クラスの正確な型を判別できました。したがって、サブクラスオブジェクトにアクセスするために基本クラスを参照することに限定されなくなりました。
