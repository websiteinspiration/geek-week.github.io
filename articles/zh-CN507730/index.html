如何在插件中使用第三方程序集

问题



我，而且我认为许多MS Dynamic CRM开发人员（在我的情况下，这是前提条件下的2016年）遇到了一个问题，为了使用第三方程序集，您需要将它们与主程序集组合在一起，然后在“插件注册实用程序”中进行注册。



例如，许多使用ILMerge或ILRepack。



搜索



我在各个站点上搜索了解决方案选项，但是除了使用ILMerge或ILRepack之外，其他地方都找不到其他选项。我想实现该任务，以便程序集可以与数据库一起移动到另一台服务器，并且一切正常。



没有发现



我认为，我描述的选项不会带来什么特别的新变化，但一方面，我的朋友和同事尤里·尤尔科夫斯基（Yuriy Yurkovskiy）帮助我找到了一个简单的解决方案。

他建议将必要的程序集存储在数据库中，并使用该模块在站点级别加载它们。

using System;

using System.Data.SqlClient;

using System.Reflection;

using System.Web;



namespace CrmExternalAssemblyLoader

{

public class AssemblyLoaderModule: IHttpModule

{

private static bool IsLoaded;

private readonly object _lock = new object();

public void Dispose()

{

throw new NotImplementedException();

}



public void Init(HttpApplication context)

{

lock (_lock)

{

if (!IsLoaded)

{

AppDomain.CurrentDomain.AssemblyResolve += CurrentDomain_AssemblyResolve;

IsLoaded = true;

}

}

}



private Assembly CurrentDomain_AssemblyResolve(object sender, ResolveEventArgs args)

{

var assemblyName = new AssemblyName(args.Name);

var assembly = GetAssembly(assemblyName.Name + ".dll");

if (assembly != null)

{

return Assembly.Load(assembly);

}

return null;

}



public byte[] GetAssembly(string assemblyName)

{

var sqlConnString = @«Data Source=[database]; Initial Catalog=[catalog]; Integrated Security=true;»;

var query = @«select Lib

from CrmExternalAssambly

where Name = name»;

using (var connection = new SqlConnection(sqlConnString))

{

using (var command = new SqlCommand(query, connection))

{

command.Parameters.Add(new SqlParameter("@name", assemblyName));

connection.Open();

var reader = command.ExecuteReader();

if (reader.HasRows)

{

while (reader.Read())

{

return (byte[])reader[0];

}

}

return null;

}

}

}

}

}

做完了



当插件执行开始时，CRM将尝试在内存中找到程序集，而不是找到它，然后爬到AppDomain.CurrentDomain.AssemblyResolve中，在该程序集中从数据库中减去程序集并将其加载到域内存中。程序集将一直存在，直到重新加载该站点为止。



优点缺点



+您不需要将程序集合并到插件中，因此程序集的大小不会增加，插件启动时SQL不会有太大的负载，尤其是如果插件未注册时。

+对程序集的访问速度更高，因为它已被加载到域中。

+所有第三方程序集都在您的数据库中，并随基础迁移。

+能够存储不同版本的程序集（此刻未经充分测试！因此仍然需要检查）



-也许在更新CRM时，您将需要再次将模块行添加到web.config CRM。
