「悪いプログラマーはコードについて考えます。優れたプログラマーは、データ構造とそれらの関係について考えています。
私たちは、プログラミングにおけるあらゆる問題の解決策が適切なデータ構造の選択に帰着することが非常に多いという公理を採用しています。この公理は証明することができますが、それは長い時間であり、記事は別のものについて少しです。今日は、最も単純な理論的構造の1つであるペアについて話します。

ペアは、任意の2つのデータ要素をそれ自体に格納するデータ構造であり、何らかの形でそれらを論理的に1つに結合したいことを意味します。この問題を正確に解決します。いくつかのデータ要素がありb、フォームbでそれらを提示する必要がある場合、ペアにこの構築を実装できます。

これまでに一度も遭遇したことがない、または遭遇しないと思われる場合、これはそうではありません。

問題の状態を理解しましょう！数学的エンティティの抽象化を実装する必要があるため、元のソースを参照するのが妥当です。もちろん、経験豊富なプログラマーがこのテキストを読んだら、数学の分数に関する記事を読まずに、コード内のソリューションをすぐに想像できたはずですが、私たちは想定します。分数がどのように機能するかをぼんやりと想像し、推論の全過程を示しています。

ウィキペディアの資料によると、分数は通常（1/4）と小数（0.1）です。明らかに、問題の条件により、通常の表示形式で分数を処理する必要があります。

また、通常の分数は2つの論理和です。数値（分子と分母）、これは、このタスクのデータ構造としてペアを使用する必要性についての最も確実なシグナルです。

コードでは、この構造を次のように説明できます。

次に、そのような計画の割合を8 / 4、6 / 9にして状況を分析する必要があります。タスクの状態は、正規化を提供する必要性について述べています。分数の正規化は、最大公約数（GCD）をその分から取り除くことです。

GCDの2つの数値を検索する関数を実装します。

//Gcd     
const getGcd = (a, b) => ((a % b) ? getGcd(b, a % b) : Math.abs(b));

関数のテキストが明確でない場合は、再帰について読むことをお勧めします。

分数を正規化するには、分子と分母をGCDで除算する必要があります。正規化関数を作成します。

const normalize = (n1, n2) => {
  const commonDivisor = getGcd(n1, n2);
  return [n1 / commonDivisor, n2 / commonDivisor];
};

makeRationalコンストラクターに正規化を入れて、分数を作成するときに常にデータを正規化するのが合理的です。 

const makeRational = (num, denom) => {
  const [normalizeNum, normalizeDenom] = normalize(num, denom);
  return new Map([
    ['num', normalizeNum],
    ['denom', normalizeDenom],
  ]);
};

次に、操作インターフェースについて説明します。

1）

加算2つの通常の分数を加算するには、それらを共通の分母に持ってくる必要があります。次に、分子を追加し、分母は変更しません。

//..       ,
//       ,
//         
const add = (rat1, rat2) => (
  makeRational(
    getNumer(rat1) * getDenom(rat2) + getNumer(rat2) * getDenom(rat1),
    getDenom(rat1) * getDenom(rat2),
  ));

2）減算

分数の差を取得するには、それらも共通の分母に削減する必要があります。その後、分子を減算し、分母を変更しないでおく必要があります。

//    
//     
const sub = (rat1, rat2) => (
  makeRational(
    getNumer(rat1) * getDenom(rat2) - getNumer(rat2) * getDenom(rat1),
    getDenom(rat1) * getDenom(rat2),
  ));

3）乗算

2つの通常の分数を乗算するには、それらの分子と分母を乗算する必要があります。

const multi = (rat1, rat2) => (
  makeRational(
    getNumer(rat1) * getNumer(rat2),
    getDenom(rat1) * getDenom(rat2),
  ));

4）

除算通常の分数を別の分数に分割するには、最初の分数に2番目の分数の逆数を掛ける必要があります。逆は、分子が元の分母に等しい分数と呼ばれ、分母は元の分子です。

const div = (rat1, rat2) => (
  makeRational(
    getNumer(rat1) * getDenom(rat2),
    getDenom(rat1) * getNumer(rat2),
  ));

ご覧のとおり、各操作で新しいエンティティを作成します。これは不変性ルールの結果です。数学演算では、元の数を変更せず、新規に作成することに注意してください1 + 2 = 3。

注意を払う必要があります。実装makeRationalを他のものに変更できますが、呼び出し元のコードはそれを認識せず、引き続き機能します。これは、抽象化を正しく実装し、直接ではなくインターフェイスを介してそのコンポーネントを操作するという事実の結果です。
