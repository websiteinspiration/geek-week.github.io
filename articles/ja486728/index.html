ドイツのデータ保護機関：Windows 10 1909 Enterpriseのテレメトリは完全に無効にできます

マイクロソフトがWindows 10の最初のバージョンをリリースして以来、テレメトリは長い間熱く議論されてきました。マイクロソフトはデータ収集をオペレーティングシステムに深く統合することを決定しました。そのため、Windows 10 HomeおよびWindows 10 Proのユーザーエディションにはテレメトリを完全に無効にするオプションがありませんでした。

EnterpriseおよびEducationエディションにもテレメトリが含まれていましたが、これらのエディション専用の特別な設定が付属しています。

HomeユーザーとProユーザーは、「完全」と「基本」のテレメトリレベルを選択できます。 Basicは、Microsoftへの情報の転送を制限しますが、送信を完全にブロックしません。 EnterpriseおよびEducationエディションには、セキュリティテレメトリオプションがあり、データの収集と送信を大幅に制限します。



バイエルン州データ保護監督局は、2019年の活動に関するレポートを公開しています。レポートには、Windows 10 Enterpriseバージョン1909

のデータ収集分析に関する情報が含まれています。レポートは、こちらからダウンロードできます。（ドイツ語、.PDF）、Windows 10 EnterpriseおよびWindows 10 Educationのエディションを購入できる組織は、Windows 10バージョン1909でテレメトリデータの送信を完全に無効にできると記載されています。

テストのセットアップは、送信されたデータを見つけるために行われた他のテストと同じでした。Windows10。Windows10 Enterpriseバージョン1909がラボにインストールされ、すべてのシステムトラフィック（受信と送信の両方）が記録および分析されました。

Windows 10デバイスのテレメトリレベルは「セキュリティ」に設定され、Microsoftが提供するツール/設定はテレメトリアクションを排除するために使用されました。

経営陣は、実験室の条件では、Windows 10 Enterprise（およびEducation）バージョン1909を実行しているコンピューターでテレメトリデータの送信を完全に無効にできる可能性があることを発見しました。経営陣に

よると、テスト結果は実際の条件で検証する必要があります。実際のテストで確認されれば、テレメトリはWindows 10 EnterpriseまたはEducationを実行しているシステムのデータ保護の障害にはなりません。

これらのリリースではデータ収集を完全に無効にすることはできないため、管理者はWindows 10 Pro（およびHome）システムでは状況が異なると指摘しています。

# Turn off "Connected User Experiences and Telemetry" service
#   "      "
Get-Service -Name DiagTrack | Stop-Service -Force
Get-Service -Name DiagTrack | Set-Service -StartupType Disabled

# Turn off per-user services
#  c   
$services = @(
	# Contact Data
	#   
	"PimIndexMaintenanceSvc_*"
	# User Data Storage
	#    
	"UnistoreSvc_*"
	# User Data Access
	#     
	"UserDataSvc_*"
)
Get-Service -Name $services | Stop-Service -Force
New-ItemProperty -Path HKLM:\System\CurrentControlSet\Services\PimIndexMaintenanceSvc -Name Start -PropertyType DWord -Value 4 -Force
New-ItemProperty -Path HKLM:\System\CurrentControlSet\Services\PimIndexMaintenanceSvc -Name UserServiceFlags -PropertyType DWord -Value 0 -Force
New-ItemProperty -Path HKLM:\System\CurrentControlSet\Services\UnistoreSvc -Name Start -PropertyType DWord -Value 4 -Force
New-ItemProperty -Path HKLM:\System\CurrentControlSet\Services\UnistoreSvc -Name UserServiceFlags -PropertyType DWord -Value 0 -Force
New-ItemProperty -Path HKLM:\System\CurrentControlSet\Services\UserDataSvc -Name Start -PropertyType DWord -Value 4 -Force
New-ItemProperty -Path HKLM:\System\CurrentControlSet\Services\UserDataSvc -Name UserServiceFlags -PropertyType DWord -Value 0 -Force

# Stop event trace sessions
#

# Change Windows Feedback frequency to "Never"

if (-not (Test-Path -Path HKCU:\Software\Microsoft\Siuf\Rules))
{
	New-Item -Path HKCU:\Software\Microsoft\Siuf\Rules -Force

# Turn off diagnostics tracking scheduled tasks
#
