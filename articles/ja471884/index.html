Microsoft SQL Serverデータベースを管理するための10の無料ApexSQLユーティリティ



こんにちは、ハブル！私たちはQuest Softwareとよく協力しており、今年彼らはMicrosoft SQL Serverデータベース管理および監視ソリューションのメーカーであるApexSQLを買収しました。ロシアでは、これらの男についてほとんど知られていないようです。彼らのホームページには、「SQL Server用のキラーツール」と書かれています。威嚇するような音。私たちはこのベンダーを紹介し、彼らの決定で何が殺人的であるかを理解する考えを持っていました。誇大広告の部分から始めましょう-無料の製品。その後、商用製品について簡単に説明します。これらのプログラムがお役に立てば幸いです。猫をお願いします。

記事上で：

  1. ApexSQL Compareは、SQLコード、ファイル、フォルダを比較するためのツールです。SSMSまたはVisual Studioの拡張機能としても機能します。
  2. ApexSQL Decrypt — SQL Server: , , (view). SSMS Visual Studio.
  3. ApexSQL Discover — SQL Server SSRS, SSAS SSIS.
  4. ApexSQL Refactor — SQL-. SSMS Visual Studio.
  5. ApexSQL Model — SQL Server. SSMS Visual Studio.
  6. ApexSQL Plan — Execution plans. SSMS.
  7. ApexSQL Complete-ツールはSQLステートメントを自動的に完了し、独自のスニペット（オートコンプリートのキーボードショートカット）を追加できます。SSMSまたはVisual Studioの拡張機能としても機能します。
  8. ApexSQL Propagateは、一度に複数のデータベースでSQLコードを実行するためのツールです。
  9. ApexSQL Searchは、SQL Serverの内部でデータとオブジェクトを検索するためのユーティリティです。SSMSまたはVisual Studioの拡張機能として機能します。
  10. ApexSQL DevOps Toolkit -CI / CDパイプラインを作成するためのツール。ここにリストされている唯一の製品にはWebコンソールがあります。

各製品は、ApexSQL有料ソリューションの機能の一部です。記事の最後で、商用ソリューションから得られるものについて簡単に説明します。

1. ApexSQLの比較




仕事のメカニズムと機会に関する短いビデオ


比較では、ファイル、フォルダー、およびSQL Serverオブジェクトを比較できます。さまざまなオブジェクトを格納するために、双方向の比較とスクリプトの生成がサポートされています。インターフェイスでは、ユーティリティはC＃、C ++、HTML、JavaScript、PostgreSQL、Python、T-SQL、Visual Basic、XMLの構文を強調表示できます。



主な機能はSSMSとVisual Studioへの統合であり、オブジェクトエクスプローラーから直接比較するためにさまざまなデータベースインスタンスからオブジェクトを呼び出し、CLIを介して作業をサポートします。

2. ApexSQL復号化




仕事のメカニズムと機会に関する短いビデオ


復号化では、SSMSまたはVisual Studioインターフェイスからワンクリックでプロシージャ、関数、トリガー、またはビューを復号化できます。



インターフェースでは、DDL復号化スクリプトを表示できます。SQL Serverの複数のインスタンスへの同時接続をサポートします。

3. ApexSQL Discover




SQL Server、SSRS、SSAS、およびSSISの既存のインスタンスをIP範囲で識別し、詳細情報を表示します。



インターフェイスから、データベース検出をスケジュールに従って構成できます。ApexSQL Discoverは、検出結果を含む通知をメールで送信できます。

4. ApexSQLリファクタリング




仕事のメカニズムと機会に関する短いビデオ


ボックスプロファイルに基づいてコードをフォーマットするか、コードをカスタマイズして見慣れたものにすることができます。リファクタリングは、コード内の未使用の変数を強調表示し、コードブロックをプロシージャにカプセル化し、「*」を完全なリストに展開できます。



便利な機能の1つは、テーブルをパーティション分割して、最も使用頻度の低い列を分離することです。リファクタリングは、この分離の影響を受けるオブジェクトを報告します。



また、リファクターインターフェースでは、すべての依存関係の変更に伴い、オブジェクトの名前を安全に変更できます。



これらの機能やその他の機能については、ユーティリティの説明ページで詳しく説明しています。

5. ApexSQLモデル




ユーティリティは、SQL Serverオブジェクトとそれらの間の関係を視覚化します。同じインターフェースで、オブジェクトの構成を変更し、対応するDDLコードを生成できます。すべての視覚エフェクトは画像としてエクスポートできます。

機能モデルは、商用製品のDiffおよびDocで拡張されます。最初の製品を使用して、モデルをデータベース内の現在の状態と比較したり、2つの異なるデータベースのモデルを比較したりできます。2番目の製品は、データベースオブジェクトを文書化するために必要です。

6. ApexSQLプラン




仕事のメカニズムと機会に関する短いビデオ


これは、パフォーマンスを最適化するためのツールです。ApexSQLプランを使用して、HTMLおよびXML形式でレポートにプランをエクスポートすることもできます。このユーティリティーの頻繁な使用例は、最新計画と変更された計画をさまざまな特性について比較することです。



ApexSQLプランは、クエリストアからのアンロードと、これらのクエリの特性をダッシュ​​ボードとして表示することをサポートしています。

7. ApexSQLの完了




仕事のメカニズムと機会に関する短いビデオ


コードライティングアクセラレータ。ユーティリティインターフェイスでは、エイリアス、スニペット、コードの強調表示を設定できます。

8. ApexSQL伝播




ApexSQL Propagateでは、さまざまなSQL Serverデータベースでのコード実行を設定し、実行の順序を決定できます。



実行されたコードの結果は、ユーティリティによって解析され、実行結果を表示するために記録されます。

9. ApexSQL検索




仕事のメカニズムと機会に関する短いビデオ


ツールはデータベース構造を検索します。ここから、オブジェクトの名前を安全に変更し、それらの関係を追跡できます。検索結果はHTML、CSV、Excelにエクスポートされます。

10. ApexSQL DevOpsツールキット




仕事のメカニズムと機会に関する短いビデオ


ユーティリティの内部では、PowerShellコードが実行されています。ビルド、レビュー、ドキュメント、およびデプロイは、ApexSQL DevOps Toolkitインターフェースから開発者が利用できます。ここでは、合成データを生成し、ユニットテストを自動的に実行し、変更を文書化できます。プラグインとしてTeamCity、Jenkinsなどに接続できます。
