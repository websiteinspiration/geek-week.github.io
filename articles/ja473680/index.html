エアコンIT。パートII

この部分では、WiFi <=> IRゲートウェイを使用して、エアコン（および、一般に、IRを介して制御されるほとんどすべてのデバイス）の「シンプルな」統合のオプションについて説明します。

もっと面白くするために、私は人気のゲートウェイを選びました（ちなみに、Yandexは彼女のアリスにそれを使用しています）。あなたはアリで1つ買うことができます、価格は約1200ルーブルです（Yandexはより高価です）。

デバイスは非常にコンパクト（手のひらに簡単に収まる）で、目に見える器官から1つの小さな目立たない青いLEDとペアリング用のボタンがあります。他のすべては、黒い赤外線透過ケースの後ろに隠されています。電源として利用可能なUSBポート（電源、コンピュータなど）を使用できます。ゲートウェイは、microUSBケーブル（付属または任意の方法）を使用してゲートウェイに接続されます。

このデバイスの特殊性は、「狭い円でよく知られている」esp8266に基づいて構築されていることです。つまり、非常に簡単に「飼いならす」ことができます。

「最小の」オプション


デバイスを変更する前に、私はそれを通常モードで使用しようとしました：TuyaSmartアプリケーションを電話にインストールしました（AndroidとiPhoneの両方にバージョンがあります）、そしてすべてが簡単です：

  • アプリケーションに登録し、
  • 小さなウィザードを実行して新しいデバイスを接続します（このためには、ゲートウェイを電源に接続し、1つのボタンを使用して「ペアリング」モードにする必要があります。
  • ゲートウェイがアプリケーションに表示されたら、新しい「コントロールパネル」を作成します。別の小さなウィザードでデバイスのタイプ（私の場合-エアコン）を選択し、次に製造元を選択します（アプリケーションにElectroluxはありませんが、Mideaを選択するとうまくいきました）。すぐに新しいコンソールの「名前」が思いつきます（「Mr. White」を選択しました）。

それで全部です。その後、次のコントロールパネルが電話に表示され

ます。これを使用すると、基本的な基本機能を制御できます。

  • オンオフ
  • 温度設定
  • 動作モードの選択（「自動」、「冷却」、「加熱」、「ファン」）、
  • ファン速度の選択（「自動」、「低」、「中」、「高」）。

一般に、リモートコントロールに使用することはすでに可能ですが、「スマートホーム」と統合はどこにありますか？

この方法は私には不必要に複雑に見え（ファームウェアがはんだ付けするのに4本のワイヤーしか必要としない場合、非常に多くの動作が必要）、内部を見るのは興味深いものでした。

次に、モジュールをフラッシュする必要があります。元のファームウェアは上書きされ、復元できません。

ファームウェアは非常に簡単に実行されます。私たちはArduino環境（またはAtomなど）を使用します。Tasmotaファームウェアを含むスケッチを開発環境にダウンロードします。次に、任意のUSBシリアルからTYWE3S RX、TX、GND、VCCモジュールに接続します（混同しないでください。3.3Vの電力と対応する信号レベルが必要です。5Vがある場合は、新しいゲートウェイを注文する必要があります）。電源を投入する前に、モジュールのGPIO0を（便利な方法で）クローズして、モジュールをファームウェアモードにすることを忘れないでください。点滅します。

ファームウェアを構成する必要はありません。すべてを「そのまま」コードに残すことができます。モジュールを「飼いならす」必要があるだけです。

すべてが正しく行われた場合（もちろん、ファームウェアのワイヤを取り外した後）、ゲートウェイをオンにします。携帯電話またはコンピュータで、WiFiネットワークの検索をオンにします。新しいネットワークが、sonoff-xxxx（xxxxは数字）という名前で表示されます。

このネットワークに接続し、アドレス192.168.4.1にアクセスします-これはゲートウェイのWebインターフェイスです。

まず、ゲートウェイ設定で、ホームWiFiネットワークとそのパスワードを指定します（デバイスが再起動してホームネットワークに接続します）。さらに、すべてのアクションがその中で実行する方が簡単です。当然、ホームネットワークでゲートウェイが受信したアドレスを、便利な方法で見つける必要があります。Tasmotaファームウェアリリースページに

エアコン+ Domoticz =永遠の友情！


私はDomoticzをメインの自動化システムとして使用しているので、この特定のシステムと「友達」になります。

作業計画は次のとおりです。

  1. 複数の仮想デバイスを作成します：スイッチ、2つのセレクター（モードと空気流量を選択するため）、およびサーモスタット。
  2. 仮想デバイスでのアクションによって、ゲートウェイにコマンドを送信するいくつかのスクリプトを追加します。
  3. すべてが適切に管理されていることを確認します。

仮想デバイス


デバイスを作成するときは、その名前を指定する必要があります。私は持ってきます。他のものを選択できます。後でそれらのスクリプトを修正することを忘れないでください。

「HVAC」という名前の通常のスイッチ（タイプ「スイッチ」）を作成します。このスイッチは、1つのボタンのオン/オフを切り替えるために使用されます。

エアコンの動作モードを選択するには、次の特性を持つ仮想スイッチ「HVAC-Mode」（タイプ「セレクタースイッチ」）を追加します。


ここでは、「加熱」と「冷却」の2つのモードのみを作成したことに注意してください。エアコンには「オート」と「ファン」モードがまだありますが、私にとっては使い物になりません。1つ目は不思議で家計にとって不快で、2つ目はまったく必要ありませんでした。これらのモードを使用する場合、それらをセレクターに追加するだけです。

同様に、「HVAC-FanSpeed」セレクターを作成します。


これで、HVAC-Tempサーモスタットのみが作成されます（デバイスタイプはサーモスタットセットポイントです）。

スクリプト


Lua（デバイス）のDomoticzスクリプトに何とか近づいているので、より便利なことをしています。

明確にするために、ホームネットワークのIRゲートウェイのIPアドレスは10.40.20.254であり、スクリプトの1つに表示されるのはこのアドレスです。ゲートウェイのアドレスで修正する必要があります。

次の3つのスクリプトを作成します。

HVACauto：

commandArray = {}

for deviceName,deviceValue in pairs(devicechanged) do
    if (deviceName=='HVAC-Temp') then
        if (tonumber(deviceValue) < tonumber(otherdevices_temperature[''])) then
            commandArray['HVAC-Mode'] = 'Set Level: 10' --"Cool"
        else
            commandArray['HVAC-Mode'] = 'Set Level: 20' --"Heat"
        end
            commandArray['HVAC'] = "On"
    end
end

return commandArray

このスクリプトは、エアコンの設定温度と「部屋」センサーからの温度を比較して、正しいモード「冷却」または「暖房」を選択します。

HVACpower：

commandArray = {}

for deviceName,deviceValue in pairs(devicechanged) do
    if (deviceName == 'HVAC') then
        if deviceValue == "Off" then
            print('  ')
            commandArray['HVAC-Mode'] = 'Set Level: 0' --"Off"
        else
            if (tonumber(otherdevices['HVAC-Temp']) < tonumber(otherdevices_temperature[''])) then
                commandArray['HVAC-Mode'] = 'Set Level: 10' --"Cool"
            else
                commandArray['HVAC-Mode'] = 'Set Level: 20' --"Heat"
            end
        end
    end
end

return commandArray

このスクリプトは、エアコンの「一般」スイッチのステータスを監視し、エアコンをオンまたはオフにするときに目的の動作モードを選択します。

そして最後のHVACmainスクリプト：

commandArray = {}

local cmnd

for deviceName,deviceValue in pairs(devicechanged) do
    if (deviceName=='HVAC-Mode' or deviceName=='HVAC-FanSpeed' or deviceName=='HVAC-Temp') then
        if (otherdevices['HVAC-Mode']=='Off') then
            cmnd = 'http://10.40.20.254/cm?cmnd=IRhvac{"Protocol":"COOLIX","Power":"Off"}'
        else
            cmnd = 'http://10.40.20.254/cm?cmnd=IRhvac{"Protocol":"COOLIX","Power":"On","Mode":"'..otherdevices['HVAC-Mode']..'","FanSpeed":"'..otherdevices['HVAC-FanSpeed']..'","Temp":'..otherdevices['HVAC-Temp']..'}'
        end
        print(cmnd)
        os.execute("wget '"..cmnd.."'")
    end
end

return commandArray

しかし、このスクリプトが最も重要です。必要なパラメーターを使用して正しいコマンドを生成し、ゲートウェイへのGET要求を実装します。ゲートウェイのIPアドレスを修正することを忘れないでください。

実際、それだけです。Domoticzインターフェースを対応するタブと「押しボタン」に切り替えて、空調を制御できます。すべてが正しく行われていれば、必要に応じて空調システムが制御されます。


特徴


IRゲートウェイを介したこの制御オプションには、フィードバックの欠如という1つの欠点があります。それら。誰かが標準のリモートコントロールを使用している場合、Domoticzのステータスは変更されず、関連のなくなった情報が表示されます。

一般に、これは実際には干渉しません（特にIRリモコンを使用せず、UDの機能のみを使用する場合）。

もちろん、IRゲートウェイでどのコマンドがIRを介して送信されるか（ゲートウェイはMQTTを介してこの情報を送信します）を監視し、標準のリモートコントロールを使用している場合でも、現在のステータスとパラメーターがUDシステムで設定されるように少しプログラムすることができます。

しかし、そのための時間はなく、特別な必要もありません（ビデオ監視システムを「調べて」、エアコンがオンになっているかどうかを確認する方が簡単です）。
