as constコンストラクトとinferキーワードを使用したTypeScriptによる型推論

TypeScriptを使用すると、この言語を使用せずに開発者が独自に解決しなければならない多くのタスクを自動化できます。ただし、TypeScriptを使用する場合、型注釈を常に使用する必要はありません。実際のところ、コンパイラーはコード実行のコンテキストに基づいて型推論の素晴らしい仕事をします。本日翻訳する記事は、キーワードinferと構文が使用される型推論のかなり複雑なケースに当てられていas constます。



型推論の基本


まず、最も単純な型推論の例を見てみましょう。

let variable;

この方法で宣言された変数はタイプanyです。コンパイラーをどのように使用するかについてのヒントはコンパイラーに提供しませんでした。

let variable = 'Hello!';

ここでは、変数を宣言し、すぐに値を書き込みました。TypeScriptは、この変数がtype stringであると推測できるようになったため、完全に許容される型付き変数ができました。

同様のアプローチが関数に適用されます。

function getRandomInteger(max: number) {
  return Math.floor(Math.random() * max);
}

このコードでは、関数getRandomIntegerが数値を返すことを示していません。しかし、TypeScriptコンパイラはこれをよく知っています。

ジェネリックの型推論


上記の概念は、ユニバーサルタイプ（ジェネリック）に関連しています。ジェネリックについてもっと知りたい場合は、これとこの資料を見てください。

ジェネリック型を作成する場合、多くの便利なことができます。型推論により、ユニバーサル型での作業がより便利になり、簡素化されます。

function getProperty<ObjectType, KeyType extends keyof ObjectType>(
  object: ObjectType, key: KeyType
) {
  return object[key];
}

上記のジェネリック関数を使用する場合、タイプを明示的に指定する必要はありません。

const dog = {
  name: 'Fluffy'
};
getProperty(dog, 'name');

この手法は、とりわけ、ユニバーサルReactコンポーネントを作成する際に非常に役立ちます。こちらがその素材です。

キーワード推論の使用


型推論について話すときに頭に浮かぶ最も高度なTypeScript機能の1つはキーワードinferです。

例を考えてみましょう。次の関数を作成します。

function call<ReturnType>(
  functionToCall: (...args: any[]) => ReturnType, ...args: any[]
): ReturnType {
  return functionToCall(...args);
}

この関数の助けを借りて、別の関数を呼び出し、それが定数に返すものを書きます：

const randomNumber = call(getRandomInteger, 100);

前の式を使用するとgetRandomInteger、入力を受け取った関数が、ランダム整数の上限100として返されたものを取得できます。真、小さな問題が1つあります。それは、関数の引数の型を無視することを妨げるものは何もないという事実にありますgetRandomInteger。

const randomNumber = call(getRandomInteger, '100'); //

TypeScriptは高次関数のスプレッドパラメータとレストパラメータをサポートしているため、この問題を次のように解決できます。

function call<ArgumentsType extends any[], ReturnType>(
  functionToCall: (...args: ArgumentsType) => ReturnType, ...args: ArgumentsType
): ReturnType {
  return functionToCall(...args);
}

これで、関数callは任意の形式の引数の配列を処理できること、および引数は渡される関数の期待と一致する必要があることを指摘しました。

ここで、正しくない関数呼び出しをもう一度実行してみましょう。

const randomNumber = call(getRandomInteger, '100');

その結果、エラーメッセージが表示されます。

Argument of type ‘”100″‘ is not assignable to parameter of type ‘number’.

実際、上記の手順に従って、タプルを作成しました。TypeScriptのタプルは固定長配列であり、その値の型はわかっていますが、同じである必要はありません。

type Option = [string, boolean];
const option: Option = ['lowercase', true];

キーワードの機能から推測


ここで、関数が返すものを取得することではなく、関数に返されるデータのタイプに関する情報を取得することだけが目的であると想像してみましょう。

type FunctionReturnType<FunctionType extends (...args: any) => ?> = ?;

上記のタイプはまだ使用できません。戻り値を決定する方法の問題を解決する必要があります。ここではすべてを手動で説明できますが、これは私たちの目標に反します。

type FunctionReturnType<ReturnType, FunctionType extends (...args: any) => ReturnType> = ReturnType;
FunctionReturnType<number, typeof getRandomInteger>;

これを自分で行う代わりに、TypeScriptに戻りの型を出力するように依頼できます。キーワードinferは条件付きタイプでのみ使用できます。そのため、コードが乱雑になる場合があります。

type FunctionReturnType<FunctionType extends (args: any) => any> = FunctionType extends (...args: any) => infer ReturnType ? ReturnType : any;

これがこのコードで何が起こるかです：

  • それは言うFunctionTypeここに拡大します(args: any) => any。
  • FunctionReturnTypeこれは条件付きタイプであることを指摘します。
  • 拡大するか確認しますFunctionType (...args: any) => infer ReturnType。

これがすべて終わったら、関数の戻り値の型を抽出できます。

FunctionReturnType<typeof getRandomInteger>; // number

上記は非常に一般的なタスクであるため、TypeScriptには、この問題を解決するために設計された組み込みのユーティリティReturnTypeがあります。

constとして構築


型推論に関連する別の問題は、キーワードconstとの違いであり、let定数と変数を宣言するときに使用されます。

let fruit = 'Banana';
const carrot = 'Carrot';

変数fruit-タイプがありstringます。つまり、任意の文字列値を格納できます。

そして定数carrotは文字列リテラルです。サブタイプの例として考えることができますstring。文字列リテラルの次の説明は、このPRに記載されています。「文字列リテラルタイプは、期待される値が、文字列リテラルと同じ内容のテキストコンテンツを持つ文字列であるタイプです。」

この動作は変更できます。 TypeScript 3.4では、構成の使用を提供するconstアサーションと呼ばれる興味深い新機能が導入されていますas const。その使用方法は次のとおりです。

let fruit = 'Banana' as const;

これfruitは文字列リテラルです。この設計as constは、エンティティを不変にする必要がある場合にも便利です。次のオブジェクトを考えます。

const user = {
  name: 'John',
  role: 'admin'
};

JavaScriptでは、このキーワードconstは、定数に格納されているものを上書きできないことを意味しますuser。ただし、その一方で、この定数に記録されているオブジェクトの内部構造を変更できます。

これで、オブジェクトには次のタイプが格納されます。

const user: {
  name: string,
  role: string
};

システムがこのオブジェクトを不変として認識するためには、次の設計を使用できますas const。

const user = {
  name: 'John',
  role: 'admin'
} as const;

これでタイプが変更されました。文字列は通常の文字列ではなく文字列リテラルになりました。しかし、それだけが変わったわけではありません。これで、プロパティは読み取り専用になります。

const user: {
  readonly name: 'John',
  readonly role: 'admin'
};

そして、配列を操作するとき、さらに強力な可能性が私たちの前に開かれます：

const list = ['one', 'two', 3, 4];

この配列のタイプは(string | number)[]です。この配列を使用as constすると、それをタプルに変換できます。

const list = ['one', 'two', 3, 4] as const;

この配列の型は次のようになります。

readonly ['one', 'two', 3, 4]

これはすべて、より複雑な構造に適用されます。Anders HalesbergがTSConf 2019でのスピーチで述べた例を考えてみましょう。

const colors = [
  { color: 'red', code: { rgb: [255, 0, 0], hex: '#FF0000' } },
  { color: 'green', code: { rgb: [0, 255, 0], hex: '#00FF00' } },
  { color: 'blue', code: { rgb: [0, 0, 255], hex: '#0000FF' } },
] as const;

これで配列がcolors変更から保護され、その要素も変更から保護されます。

const colors: readonly [
    {
        readonly color: 'red';
        readonly code: {
            readonly rgb: readonly [255, 0, 0];
            readonly hex: '#FF0000';
        };
    },
    /// ...
]

概要


この記事では、TypeScriptで高度な型推論メカニズムを使用する例をいくつか取り上げました。ここではキーワードinferとメカニズムが使用されますas const。これらのツールは、特に困難な状況で非常に役立ちます。たとえば、不変のエンティティを操作する必要がある場合や、関数形式でプログラムを作成する場合などです。このトピックについて引き続き理解を深めたい場合は、この資料をご覧ください。
