C ++を使用してサイトを作成する

出来ますか？


はい。

私はそれが奇妙に聞こえ、より価値のない運動のように聞こえることは知っていますが、そうではありません。
この記事では、サイト開発でC ++を使用する方法を説明します。これにはいくつかの理由があります。

それは興味深いだけでなく、非常に実用的でもあります。

ホスティング


あなたはおそらく考えます：そのような珍しい構成は特別な環境でのみ実装することができます。ただし、これは通常は当てはまりません。通常または仮想の特別なサーバーが多くのサイト（C ++サイトだけでなく）にとって理想的なソリューションであるという事実にもかかわらず、ほとんどの場合、共有ホスティングサービスでC ++を使用できます。
CGI（つまり、原則としてすべて）をサポートするWebホスティングはすべて、C ++でサイトをサポートできます。プロバイダーに応じて、サイトをローカルでコンパイルするかどうかにかかわらず、アカウントにコンパイラーを含める必要があります。Webサーバーで直接SSH経由で編集およびコンパイルする場合は、これらを確認してください。

簡単な例


上記の例では、cPanelホスティングアカウントを使用しています。彼らは手頃な価格、安価で一貫しています。ただし、Apache構成にいくつかの簡単な変更を加えるだけで、これらのメソッドを仮想サーバーまたは専用サーバーまたはAmazon EC2インスタンスに簡単に適合させることができます。

cPanelはcgi-binフォルダを提供しますが、必要はありません。ほとんどの場合、拡張子が.cgiのファイルは、適切な権限（通常は0755）があれば自動的に処理されます。必要なファイルを次に示します（Makefileでは必ずTABを使用してください）

Makefile：

all:
        g++ -O3 -s hello.cpp -o hello.cgi
clean:
        rm -f hello.cgi

hello.cpp：

#include <iostream>
#include <string>
#include <stdlib.h>
using namespace std;
void set_content_type(string content_type) {
  cout << "Content-type: " << content_type << "\r\n\r\n";
}
void set_page_title(string title) {
  cout << "<title>" << title << "</title>\n";
}
void h1_text(string text) {
  cout << text << "\n";
}
int main() {
  set_content_type("text/html");
  // Output HTML boilerplate
  cout << "<!doctype html>\n";
  cout << "<html lang=\"en\">\n";
  cout << "<head>\n";
  set_page_title("Hello, World!");
  cout << "</head>\n";
  cout << "<body>\n";
  h1_text("Hello, World!");
  cout << "</body>\n";
  cout << "</html>";
  return 0;
}

アカウントにコンパイラーが含まれている場合（サポートチームにWebホスティングを依頼する必要がある場合があります）、アカウントでSSHを使用し、これらのファイルをpublic_htmlフォルダーに配置して、次のようにします。

make

hello.cgiファイルが実行されます。たとえば、次のようにブラウザでファイルアドレスを入力すると、

http://your-test-site.com/hello.cgi

次に、your-test-site.comをドメイン名またはホスティングURLに置き換えます。「Hello World」が画面に表示されるはず

ですコード自体に飛び込む前に、Webサーバーでどのように機能するかを見てみましょう。Apacheがリクエストを受け入れると、まずApacheが内部ハンドラーまたは書き換えルールを調べ、次にディスク上のリクエストに一致するファイルを探します。この例では、hello.cgiを見つけて取得します。私たちのプログラムは入力を受け付けず、「Hello world」というメッセージを表示するだけです。次に、Apacheはこのデータを受け取り、ユーザーに返します。

コードについて言えば、それはより簡単かもしれないと言及する必要があります。個別の関数set_content_type、set_page_title、h1_textを含める必要はありませんでした。メイン関数でクリーンを維持する単純なヘルパーがあります。これらすべてを単にメイン関数に持ち込むことができ、まったく同じように動作します。

しかし、私はあなたがこれらの機能を強調することの利点を見ることを望みます。HTML要素ごとに関数を作成する必要がある場合は、次のコードを使用して、プログラムにクリーンな応答を作成できます。

void p(string text) {
  cout << "<p>" << text << "</p>\n";
}

次に、次のようなものを使用します：

p("This would be paragraph text.");

段落を表示します。

このアイデアを開発して、p、h1_textなどのヘルパー関数が、coutを介して標準出力に直接出力する代わりにテキストを返すようにすることもできます。同時に、テンプレートシステムを作成するか、回答をネストして、非常に合理化された効率的なC ++コードで複雑なページを作成できます。

この例はできるだけ単純で、要点のみを示しています。各応答ヘッダーに完全にアクセスできるため、応答ループを完全に制御できます。

入力についての詳細


この例では入力を受け入れず、「Hello World」のみを返します。しかし、Apacheは環境変数を介して大量の情報で各要求に応答します。C標準ライブラリのgetenv（）関数を使用して、それらの値を取得できます（コードの先頭に#include <stdlib.h>を追加することを忘れないでください）。たとえば、完全なリクエストURIを調べる必要がある場合は、次を使用します。

string request_uri = getenv("REQUEST_URI");

この値を取得します。その他の有用な変数：

  • REMOTE_ADDR-訪問者のIPアドレスを取得する
  • REQUEST_METHOD-メソッドを返します（GET、POSTなど）
  • DOCUMENT_ROOT-サイトのルート（通常、共有システムでは〜/ public_htmlまたは仮想/専用サーバーでは/ var / www / html）。
  • QUERY_STRING-GET変数を取得するためのクエリ文字列。

より明確な例


もちろん、GET変数を手動で解析することは可能であり、POST変数の処理は標準入力をチェックすることで実行できます。リクエストヘッダーとレスポンスヘッダーを変更することで、Cookieを受信して​​設定することもできます。しかし、どちらの方法も面倒です。

独自のシェルを作成するか、既成のGNU cgiccライブラリを使用できます。HTMLを変更してフォームを処理するためのヘルパー関数が含まれています。大規模なプロジェクトで作業する場合、そのようなライブラリを使用すると時間をかなり節約できます。

DebianとUbuntuでは、次のコマンドでライブラリとヘッダーをインストールできます。

apt install libcgicc5 libcgicc5-dev

ただし、CentOS / RHELには独自のパッケージはありません。それらにインストールするには、次を実行：

cd /usr/local/src
wget ftp://ftp.gnu.org/gnu/cgicc/cgicc-3.2.19.tar.gz
tar xfz cgicc*.tar.gz
cd cgicc*
./configure — prefix=/usr
make
make install

注：3.2.19は執筆時点での最新バージョンですが、ftp.gnu.org / gnu / cgiccで新しいコピーを確認できます。ライブラリバインディングの問題を回避するために、プレフィックスとして/ usrも使用しました。どちらかといえば、落ち着いて変更してください。

cgiccをインストールしたら、コンパイルできます。フォームから入力を取り、それをブラウザーに表示する次の例を試してください：

Makefile：

all:
        g++ -O3 -s hello.cpp -o hello.cgi
        g++ -O3 -s cgicc.cpp -o cgicc.cgi /usr/lib/libcgicc.a
clean:
        rm -f hello.cgi cgicc.cgi

cgicc.html：

<!doctype html>
<html lang="en">
<head>
  <title>cgicc Test</title>
</head>
<body>
  <form method="POST" action="cgicc.cgi">
    <label for="name">Name</label>
    <input name="name" type="text" value="">
    <input name="submit" type="submit" value="Submit">
  </form>
</body>
</html>

cgicc.cpp：

#include <iostream>
#include <string>
#include <stdio.h>
#include <stdlib.h>
#include <cgicc/CgiDefs.h>
#include <cgicc/Cgicc.h>
#include <cgicc/HTTPHTMLHeader.h>
#include <cgicc/HTMLClasses.h>
using namespace std;
using namespace cgicc;
void set_content_type(string content_type) {
  cout << "Content-type: " << content_type << "\r\n\r\n";
}
void set_page_title(string title) {
  cout << "<title>" << title << "</title>\n";
}
void h1_text(string text) {
  cout << text << "\n";
}
int main() {
  Cgicc cgi;
  string name;
  set_content_type("text/html");
  cout << "<!doctype html>\n";
  cout << "<html lang=\"en\">\n";
  cout << "<head>\n";
  set_page_title("cgicc Test");
  cout << "</head>\n";
  cout << "<body>\n";
  cout << "<p>";
  // Grab the "name" variable from the form
  name = cgi("name");
  // Check to make sure it isn’t empty.
  if (!name.empty()) {
    cout << "Name is " << name << "\n";
  } else {
    cout << "Name was not provided.";
  }
  cout << "</p>\n";
  cout << "</body>\n";
  cout << "</html>";
  return 0;
}

Makefileでcgiccライブラリーを静的にリンクしたことに気づいたかもしれません。これは必須ではありませんが（-Icgiccで置き換えることができます）、サーバーに送信するバイナリコードを静的にリンクして、プログラムの送信に必要なすべてのものがセットとして送信されるようにします。

この例では、cgiccライブラリーがPOST変数のハードワークを実行し、「名前」を返します。

私はPOST入力変数をエスケープしませんでしたが、特にデータベースと対話する場合は特に、本番サイトで推奨されています。

これはcgiccの可能性のほんの一部にすぎません。ここで完全なドキュメントを読むことができます。

パフォーマンス/パフォーマンス


C ++は、優れたコードの処理が非常に高速です。CGIインターフェースは少し遅くなりますが、それでもPHPのようなインタプリタ言語よりも実行が良くなります。
彼らが言うように、努力することは常にあります。特殊な環境または仮想環境では、NginxまたはApacheのFastCGIサポートを使用して、プログラムのロード時のわずかな（そしてサーバーに大きな負荷がかかっていない場合は目立たない）遅延を減らすことができます。私のテストでは遅延はありませんでしたが、トラフィックがロードされたWebサイトで作業する場合は、これらのソリューションを調査して実行をさらに高速化/迅速に行うことが有用です。

さらに


C ++プログラムをシンDockerコンテナーにラップするのは簡単です。これにより、サイトを配置するときに大きな柔軟性が得られます。プログラムにC / C ++ MySQL開発ヘッダーを含めることで、MySQLデータベースにアクセスすることもできます。PHPでMySQLを使用することに慣れている場合は、変数名が非常に似ていることに気付くでしょう。

画像処理用の追加のコマンドラインプロセスを作成する代わりに、ImageMagick C ++ヘッダーを含めて、これらの画像をプログラムで直接処理できます。

長くて奇妙な旅


C ++でサイトを作成することは、特にパフォーマンスが重要な場合は実用的です。これはブログや個人サイトにはお勧めしません。WordPressで簡単に実行できます。しかし、スピードに対する極端なニーズがあり、打たれたパスでうまく記述したい場合は、Web上の次のニッチなプロジェクトのためにC ++を検討してください。
