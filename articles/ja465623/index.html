アコーディオン、FAQ、スポイラー、その他のドロップダウンウィジェット



DivとJavaScriptを使用して、アコーディオン、FAQ、スポイラーなどを作成できます。
しかし、より良い：詳細と要約

すべての最新のブラウザーでサポートされており、これは意味的に正しくフォーマットされたコードであり、これを使用すると次のような利点があります。

  1. 障害を持つ人々はあなたのサイトを使いやすくなります！彼らのソフトウェア（スクリーンリーダーなど）はhtml5タグを完全に理解し、それらを正しく処理して、コンテンツについて人々に正しく通知します。
  2. テキストの一貫性が向上し、検索エンジンはサイトのインデックスをより適切に作成できるようになります。これは、テキストの表示と非表示がどのように関連しているかをよりよく理解するためです。
  3. キーボードやその他のデバイスから要素を制御するために使用できます。
  4. 読み込まれるJavaScriptコードの量が減り、ページの読み込み速度、処理速度、正確さが向上します。
  5. Lighthouse、Google PageSpeed、その他の同様のツールでのパフォーマンスの向上。
  6. JavaScriptがオフの場合に機能します。

マイナス：

  1. 古いブラウザはそのようなタグを知らず、情報を隠しません。

見た目はそれほど美しくありませんが、ニュートラルで多くのデザインに簡単に適合します。ちなみに、Detailsタグのデフォルトビューは、Habrのスポイラーと非常によく似ています。少し塗り直して、下線を引いて、意味的に正しいものにするだけです。



残念ながら、デフォルトのマーカーには2つの欠点があります。

  1. 古いブラウザはそれを見ません。
  2. Webkitブラウザーでは、マーカーシンボルを変更できません。

このため、デフォルトのマーカーを非表示にし、独自のマーカーを作成する必要があります。


テキストマーカーを変更した最初の詳細/概要の例を考えてみましょう：



CSS：

summary::-webkit-details-marker{display:none;}
summary::-moz-list-bullet{list-style-type:none;}
summary::marker{display:none;} 
summary {
   display:inline-block;
   padding: .3em .5em .3em .4em;
   font-size:1.4em;
   cursor: pointer;
}
summary:before {  
  content: "+";
  margin-right: .3em;
}
details[open] > summary:before {
  content: "–";
}
summary ~ * {
   padding:0 1em 0 1em;
}
summary:focus {
  outline:0;
  box-shadow: inset 0 0 1px rgba(0,0,0,0.3), inset 0 0 2px rgba(0,0,0,0.3);
}
details{
  display:block;
  margin-bottom: .5rem;
}

デフォルトのマーカーを表示します：なし、サマリーを使用して別のマーカーを表示します：before {content： "+";}

サマリー：フォーカス-ボックスシャドウを使用したスト​​ローク。これは、アクティブな要素が表示され、タブキーとスペースで開閉します。

概要タグの場合、display：inline-blockを設定します。これにより、全幅に拡大されず、クリックできるのは行全体ではなく単語のみになります。

  100%  {opacity: 1;}
}

  display:block;
  margin-bottom: .5rem;
}

新しい例では、summary：beforeの代わりに、summer：afterをマーカーに使用して、右側に表示されるようにします。

変換を使用したマーカーアニメーション：スケール（1、-1）;

要約後のすべての要素に、アニメーションを使用してアニメーションをスムーズな外観に設定します。スイープ.5sイーズインアウト。

これで、純粋なHTML5とCSSで、JavaScriptなしで美しいアコーディオン、スポイラー、およびFAQを作成できます。

アウトラインを削除する前に、どのように交換できるかを約100倍考えて、人がフォーカスを確認してキーボードや他のデバイスから移動できるようにします。

1つのスポイラーを開き、他のスポイラーを閉じる場合は、javascriptを使用する必要があります。以下に、この問題を解決する簡単なJSコードの例を示します。

JavaScript：

var details = document.querySelectorAll("details");
for(i=0;i<details.length;i++) {
  details[i].addEventListener("toggle", accordion);
}
function accordion(event) {
  if (!event.target.open) return;
    var details = event.target.parentNode.children;
    for(i=0;i<details.length;i++) {
      if (details[i].tagName != "DETAILS" || 
         !details[i].hasAttribute('open') || 
         event.target == details[i]) {
         continue;
      }
      details[i].removeAttribute("open");
    }
}
