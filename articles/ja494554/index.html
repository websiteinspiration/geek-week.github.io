SDL_ttfを使用したフォントのダウンロード


このチュートリアルでは、SDL_ttf拡張ライブラリを使用してTrue Typeフォントを表示する方法を学びます。ライブラリのインストールは、レッスン3の SDL_imageの設定と同じですが、「イメージ」の代わりに「ttf」と書くだけです（Windowsユーザーは、含まれているfreetype dllもコピーする必要があります）。だから、ダウンロードSDL_ttfは見とり、ドキュメントをしてチュートリアルを開始！

SDL_ttfで作業するために最初に必要なのは、ttfフォントです。BitFontMakerを使用して、著者はリポジトリからダウンロードできるかなりひどいフォントを作成しましたただし、独自の美しいフォントがすでにある場合は、それを使用できます。このフォントは単純なASCII文字のみを提供するため、ASCII以外の文字を表示しようとすると、ほとんどの場合成功しません。このレッスンのコードは、通常はGithubからダウンロードできるように、レッスン5に基づいて記述されます。このチュートリアルでは、画像、スライスされたスプライト、描画が変更されます。

テキストを表示する


SDL_ttfは、さまざまな速度と品質のフォントを表示するいくつかの方法と、UTF8およびUnicode文字を表示する機能を提供します。ドキュメントには、さまざまな表示方法の概要がわかりやすいので、品質と速度の要件に応じて、どの方法をどのような方法で使用したいかを読んで学ぶ必要があります。このチュートリアルでは、TTF_RenderText_Blendedを使用します。時間制限がなく、テキストを美しくしたいからです。さまざまなテキスト表示関数がRGB SDL_Color colorを受け入れます。これを使用して、描画するテキストの色を選択できます。

残念ながら、SDL_ttfは表面にしか表示できないため、テキストをレンダリングした後で、レンダラーを使用して描画できるテクスチャを作成するために、追加の手順を実行する必要があります。もちろん、私たちが使用するフォントをダウンロードする必要があります。これはTTF_OpenFontを使用して行われます。ここで、希望のフォントサイズを指定することもできます。

renderText関数の記述


作業を簡単にするために、テキストを受け取るrenderText関数、TTFフォント、色、必要なサイズ、および最終的なテクスチャをロードするレンダラーを含むファイルを作成します。この関数はフォントを開き、テキストを表示し、それをテクスチャに変換して、テクスチャを返します。問題が発生する可能性があるため、ライブラリの呼び出しごとにエラーを確認して正しく処理する必要があります。たとえば、リソースをクリーンアップし、エラーをログに記録してnullptrを返すことで、何か問題が発生したことがわかります。SDL_ttfはすべてのエラーをSDL_GetErrorを通じて報告するため、引き続きlogSDLErrorを使用してエラーを記録できます。

これらの要件を念頭に置いて、renderText関数を記述しましょう。

/**
*      
* @param message ,   
* @param fontFile ,   
* @param color ,     
* @param fontSize  
* @param renderer ,     
* @return SDL_Texture,     nullptr,  -   
*/
SDL_Texture* renderText(const std::string &message, const std::string &fontFile,
        SDL_Color color, int fontSize, SDL_Renderer *renderer)
{
        // 
        TTF_Font *font = TTF_OpenFont(fontFile.c_str(), fontSize);
        if (font == nullptr){
                logSDLError(std::cout, "TTF_OpenFont");
                return nullptr;
        }       
        //       TTF_RenderText,
        //    
        SDL_Surface *surf = TTF_RenderText_Blended(font, message.c_str(), color);
        if (surf == nullptr){
                TTF_CloseFont(font);
                logSDLError(std::cout, "TTF_RenderText");
                return nullptr;
        }
        SDL_Texture *texture = SDL_CreateTextureFromSurface(renderer, surf);
        if (texture == nullptr){
                logSDLError(std::cout, "CreateTexture");
        }
        //   
        SDL_FreeSurface(surf);
        TTF_CloseFont(font);
        return texture;
}

パフォーマンス警告


テキストを表示するたびに、フォントを再度開いて閉じます。これは、テキストを1回だけ表示するため、この場合に適していますが、大量のテキストを表示したり、頻繁に表示したりする場合は、そのままにしておくことをお勧めします。必要なだけフォントを開きます。このより頻繁なケースのためのrenderTextのバージョンは、フォントファイルの寿命が個別に調整されるため、フォントファイルの名前の代わりにTTF_Font *を受け入れ、開閉しません。

SDL_ttfを初期化する


SDLと同様に、使用する前にライブラリを初期化する必要があります。これはTTF_Initを使用して行われ、成功すると0を返します。SDL_ttfを初期化するには、SDLを初期化した後でこの関数を呼び出し、戻り値をチェックしてすべてが正常であることを確認します。

if (TTF_Init() != 0){
        logSDLError(std::cout, "TTF_Init");
        SDL_Quit();
        return 1;
}

renderTextの使用


便利なrenderText関数を使用すると、1回の呼び出しでテキストを表示できます。このチュートリアルでは、「TTFフォントはかっこいい！」というテキストを表示します。以前ダウンロードしたフォントの白、サイズ64。次に、他のテクスチャと同じように幅と高さを要求し、x / y座標を計算して、ウィンドウの中央にメッセージを描画します。

const std::string resPath = getResourcePath("Lesson6");
//   "TTF fonts are cool!"
//   RGBA
SDL_Color color = { 255, 255, 255, 255 };
SDL_Texture *image = renderText("TTF fonts are cool!", resPath + "sample.ttf",
        color, 64, renderer);
if (image == nullptr){
        cleanup(renderer, window);
        TTF_Quit();
        SDL_Quit();
        return 1;
}
//    ,     
int iW, iH;
SDL_QueryTexture(image, NULL, NULL, &iW, &iH);
int x = SCREEN_WIDTH / 2 - iW / 2;
int y = SCREEN_HEIGHT / 2 - iH / 2;

テキストを描く


最後に、renderTexture関数を使用する前に行ったように、テクスチャを描画できます。

//:     
SDL_RenderClear(renderer);
//    ,     ,  
//    
renderTexture(image, renderer, x, y);
SDL_RenderPresent(renderer);
