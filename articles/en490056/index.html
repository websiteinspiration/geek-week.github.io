Introduction


I started using Groovy in 2009.


Then I began research on how to structure and standardize logging in programs.


Problem


There is a methodological gap in the approach and best practices for integrating logging into program code.


In short, there is no generally accepted set of rules to help programmers understand:


  • What data to log
  • How to structure logged data
  • When to log
  • Where to place the logging code in the program code
  • How will created logs be used
  • Why spend time writing a logging code at all?

Also, it would seem with the existence of AST API (Groovy) and CGLIB (Java) - logging automation tools should have already appeared - but no:


  • Groovy has a couple of annotations (@ Slf4j, Log, etc.) which only implement the declaration and initialization of the logger variable.
