.NET CoreとNode.js



この記事の目的は、2つのプラットフォームのうち最適なものを選択することではなく、それぞれの長所と短所を探すことです。どちらのテクノロジーも、Web開発の世界では十分に確立されています。それぞれにファンがいて、良い結果を得て、インターネットでその成果を共有しています。ネットワークには、.Net CoreまたはNode.jsに切り替えたときに開発者が享受したメリットに関する多数のレビューがあります。それを理解してみましょう。

少しコンテキスト


私は2015年からカスタム開発に従事している会社で働いています。最初に.NETで書き込み、次に.NET Coreに切り替えてから、徐々にスタックをNode.JSに変更しました。主な理由は開発のスピードでした。

一般情報




.NET Core

.NET Coreは、クロスプラットフォームアプリケーションを作成するためのオープンソースの汎用フレームワークです。 Windows、macOS、Linuxなどのさまざまなオペレーティングシステム用のアプリケーションを作成できます。サポートのx64、x86の、ARM32およびARM64プロセッサを。最初の公式リリースは2016年6月27日にリリースされました。

Node.js

公式ウェブサイトによると、Node.jsはV8上に構築されたソフトウェアプラットフォームです。 V8は、JavaScriptをマシンコードに変換するGoogleのオープンソースJavaScriptエンジン（C ++で記述）です。 Node.jsを使用すると、macOS、Linux、SmartOS、FreeBSD、Microsoft Windows、AIX、Androidでアプリケーションを作成できます。最初のリリースは2009年でした。

プログラミング言語




Net Core

ほとんどの場合C＃が使用されます-1998〜2001年にMicrosoftによって開発されたオブジェクト指向プログラミング言語。厳密な型指定により、コンパイル段階でエラーをキャッチし、その結果、フィードバックループを高速化し、エラーの少ないアプリケーションをより迅速に開発できます。キーワード「動的」を使用すると、動的型付けをエミュレートできます。

Node.js

JavaScriptは、動的型付けを備えたインタープリター型オブジェクト指向言語です。また、関数型言語に固有の多くの機能があります。

標準のJavaScript言語はECMAScriptです。 JavaScriptは使いやすいですが、C＃とは異なり、実行時に多くのエラーをデバッグする必要があります。この欠点を解消するには、2012年にMicrosoftによって導入されたTypeScriptを使用できます。 TypeScriptはJavaScriptアドオンであり、JavaScriptと下位互換性があり、JavaScriptにコンパイルされています。ただし、TypeScriptは、C＃が提供する厳密に型指定されたオブジェクト指向言語の能力をすべて提供するわけではありません。 TypeScriptはJavaScriptと密接にリンクされており、JavaScriptは構造的に分類されているため、C＃に似たタイプの作業のみを模倣できます。

アセンブリ段階で表示されないエラーの例：

interface A {
    x: number;
}
 
let a: A = {x: 5}
 
let b: {x: number | string} = a; 
b.x = "unsound";
 
a.x.toFixed(0); // Not a function error occurs.

このコードの問題は、割り当ての結果として、「ax」のタイプが文字列になり、エラーが実行時にのみ発生することです。

Node.js

ツールキットはNPMで、1,298,879個のパッケージがあります。公式の開発環境は発表されていませんが、Node.js開発用のプラグインを備えたVisual Studio Codeの最も一般的な使用法は発表されていません。
一般的に、Node.jsにはツール、代替パッケージの幅広い選択肢があるようです。

結論
