SIP-Anschluss Telegramm + Voximplant = Anrufe an Mobiltelefone und Softphones


Für das Telegramm wurde ein SIP-Anschluss (@siptg) entwickelt. Voximplant ist eine Telefonieplattform. Das stimmt, sie können perfekt zusammenarbeiten und heute werden wir zeigen, wie. Willkommen bei Katze!

Registrieren Sie das Softphone in Voximplant


Der SIP-Anschluss kann in zwei Modi arbeiten: Softphone und SIP-Gateway. Wir werden die erste Option in Betracht ziehen, da Sie damit den Bot "unter dem Deckmantel" des internen Benutzers Ihrer Voximplant-Anwendung registrieren können. Aus diesem Grund müssen Sie zuerst die Voximplant-Anwendung und die darin enthaltenen Benutzer erstellen.

  1. Melden Sie sich in Ihrem Konto unter https://manage.voximplant.com/ an .
  2. Gehen Sie zum Abschnitt Anwendungen und erstellen Sie eine Anwendung namens sip.
  3. Wechseln Sie in der Anwendung zur Registerkarte "Benutzer" und erstellen Sie einen Benutzer-Telegramm-Benutzer. Vergessen Sie das Passwort nicht, es wird bald benötigt:

  4. Erstellen Sie hier einen Userbob-Benutzer (dies wird sich später als nützlich erweisen).

. @siptg : . Voximplant : usertelegram, userbob .. camelCase, , .
Jetzt gehen wir zur Seite des Boten. Das Softphone selbst steht unter dem Namen @siptg und seine Einstellungen unter dem Namen @siptg_bot - öffnen Sie sie. Wir drücken Start, geben den Befehl / tariff ein und wählen Free aus (zur Demonstration ist dies mehr als ausreichend, weist jedoch Einschränkungen auf. Beispielsweise tritt nach 5 erfolglosen Anrufen eine vorübergehende Blockierung auf. Die genaue Entsperrzeit kann auch mit dem Befehl / tariff angezeigt werden).

Verwenden Sie dann den Befehl / softphone und klicken Sie auf die Schaltfläche Neu. Geben Sie hier nacheinander die Voximplant-Benutzerdaten ein und klicken Sie auf Übernehmen. Das Serverfeld besteht aus sip. + Voximplant-Kontoname + .voximplant.com:


Wenn alles korrekt eingegeben wurde, wird der registrierte Client grün hervorgehoben.


Nun können wir überprüfen, ob sie zu Telegrammen kommen ...

… eingehende Anrufe


Dazu müssen Sie auf der Voximplant-Seite in die sip-Anwendung gehen, um ein Skript und eine Regel für die Verarbeitung von Anrufen zu erstellen. Das Szenario ist einfach: Wenn ein Anruf vom Benutzer der sip-Anwendung in die Cloud kommt, ruft die Cloud den zweiten Benutzer derselben Anwendung an und verbindet sie. Es sieht aus wie das:

VoxEngine.addEventListener(AppEvents.CallAlerting, e => {
   const inc = e.call
 
   const out = VoxEngine.callUser(e.destination, e.callerid)
 
   VoxEngine.easyProcess(inc, out)
})

Speichern Sie das Skript als callUser. Wenn Sie dort bleiben, wechseln Sie in der sip-Anwendung zur Registerkarte "Regeln" und erstellen Sie eine Anrufbenutzerregel mit diesen Einstellungen (beachten Sie die Maske):


Erinnern Sie sich an den Benutzererb-Benutzer, der am Anfang erstellt wurde? Großartig, melden Sie sich mit seinem Benutzernamen und Passwort in unserem Webphone an :


Geben Sie usertelegram ein und klicken Sie auf "Call" - Sie erhalten einen eingehenden Anruf vom @ siptg-Bot auf allen Geräten, auf denen Sie einen Instant Messenger haben.


Ausgehende Anrufe an Voximplant-Benutzer


Das Szenario und die Regel sind identisch, es muss nichts Neues erstellt werden.
Die einzige Nuance, die ich oben erwähnt habe, ist, dass Voximplant-Benutzernamen in Kleinbuchstaben geschrieben werden müssen. Wenn der Benutzername userBob ist, konvertiert der @ siptg-Bot ihn bei der Angabe schließlich in userbob, und Ihre Voximplant-Anwendung enthält einfach keinen solchen Benutzer. Infolgedessen ein Einwahlfehler.
Öffnen Sie den @ siptg-Bot, senden Sie ihm eine Benutzer-Bob-Nachricht, und Sie sehen einen angeblich eingehenden Anruf vom Bot (intuitiv, aber so wie er ist).


Klicken Sie auf das grüne Mobilteil und danach wird der Anruf an das Softphone im Browser weitergeleitet, dh an userbob.


Ausgehende Anrufe an Zellennummern


Damit der Telegrammbot Festnetznummern anrufen kann, benötigen Sie:

  • noch ein Szenario und eine Regel;
  • Gültige Rückruf-Telefonnummer (Anrufer-ID). Dies kann entweder Ihre persönliche Telefonnummer sein, die Sie im Panel bestätigt haben , oder die bei Voximplant gekaufte Nummer. Bitte beachten Sie, dass Testnummern für diesen Zweck nicht geeignet sind.

Erstellen Sie in der sip-Anwendung ein callPSTN-Skript mit diesem Code (beachten Sie Zeile 4 - hier müssen Sie eine gültige Nummer ersetzen):

VoxEngine.addEventListener(AppEvents.CallAlerting, e => {
   const inc = e.call
 
   const out = VoxEngine.callPSTN(e.destination, "real callback number")
 
   VoxEngine.easyProcess(inc, out)
})

Erstellen Sie dann eine PSTN-Regel für Anrufregeln. Sie können die Maske standardmäßig verlassen:


Wichtig: Verschieben Sie diese neue Regel in der Liste der Regeln nach unten. Daher nimmt seine Priorität ab und es funktioniert nur, wenn die Regel für den Benutzer nicht befolgt wird. *


Ok, jetzt können Sie anrufen: Senden Sie die Zieltelefonnummer an den @ siptg-Bot, es zeigt Ihnen die Anrufschnittstelle, drücken Sie das grüne Mobilteil und der Anruf geht an die angegebene Mobil- / Städtenummer.

Fazit


Die Integration eines Telegramms in die Cloud-Telefonie kann für Unternehmen praktisch sein, da dies eine weitere Möglichkeit ist, Kunden zu erreichen - lesen Sie Omnichannel. Es kann auch intern angewendet werden, wenn das Telegramm Ihr Unternehmensbotschafter ist. Vielleicht ist dies für persönliche Bedürfnisse auch gefragt. Die Möglichkeit, Telegrammanrufe zu tätigen, ist in jedem Fall relevant, wenn Sie diesen Messenger verwenden, und derjenige, den Sie anrufen möchten, wird nicht verwendet. Infolgedessen müssen Sie die Kommunikationsumgebung nicht ändern. Der Anruf wird direkt vom Messenger weitergeleitet, und für Ihren Gesprächspartner ändert sich nichts. Er erhält einen Anruf an sein Softphone / Telefon.
