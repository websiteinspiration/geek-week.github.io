仮想マシンに対するLXCの利点


サーバーリソースの不足、さまざまなプロジェクトのアクセス権を区別することが困難、1つの仮想マシンでのソフトウェアの競合が発生したときに、コンテナーに切り替えました。仮想マシンに対するLXCの主な利点は次のとおりです。

  1. CPUパフォーマンス、ディスク操作、およびRAMの損失が少ない。コンテナーで使用できるほとんどすべてのものがサーバーの速度で動作します。
  2. OSカーネル、ビデオメモリ、ディスクバッファなどにRAMを割り当てる必要はありません。
  3. クイックスタート。実際、コンテナが機能するために必要なアプリケーションのみが起動されます。
  4. システム全体ではなく、コンテナ内の個々のアプリケーションの起動をサポートします。コンテナーにはプロジェクトが1つしかない（仮想マシンには複数のプロジェクトがあった）ため、ユーザーを追加するだけで（chrootではなく）アクセスを許可できるので便利です。
  5. 各コンテナのリソースを管理する機能。
  6. サーバー間でコンテナをすばやく簡単に移動します。

rsyncを使用します。新しいサーバーで同じ名前のコンテナーを作成してコピーするだけです。

rsync -alvz :/var/lib/lxd/container/test/ /var/lib/lxd/container/test/

5番目の段落について詳しく説明し、リソース管理について説明する必要があります。作業に十分な量のリソースをコンテナーに提供する必要があります。同時に、コンテナーが不要なリソースを消費しないようにして、システムの他の部分の作業を妨害しないようにする必要があります。

これらの統計によると、コンテナが消費するリソースの数を追跡し、必要に応じて、特別なコマンドを使用してそれらの消費を制限できます。

  • lxc config set test limits.memory 2048M＃メモリ制限を設定します。
  • lxc config set test limits.cpu 1,2,3,8＃コンテナーをCPUコアにバインドします。
  • lxc config set test cpu.allowance 10％＃CPU消費を制限します。
  • lxc config set test root size 50GB＃コンテナが使用するディスク容量を制限します（ZFSまたはbtrfsでのみ機能します）。
  • lxc config get test limits.memory-設定された制限値を表示

その結果、コンテナーを使用すると、十分なパフォーマンスを維持しながら、非常に標準的なデスクトップで数十のコンテナーを持ち上げることができます。
