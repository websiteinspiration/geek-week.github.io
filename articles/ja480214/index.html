PNGをDXT5形式のDDSに変換し、Pythonを使用して結果をその場で表示

DXT5形式に何が期待できるかを理解している場合は、自分で確認してください。スポイラーでは、アニメーションは左側にある4x4ピクセルのPNG画像で、右側はDDS（DXT5）に変換され
ています。

はい、DXT5への変換がどのように行われるかを正確に知っています。


DXT5に変換した後にPNG画像がどうなるかを確認するためのツールや機会をその場で探していた場合-記事へようこそ。

目的


DXT5形式の概要と、変換結果をその場で確認できるツール。

この記事はプロ向けではなく、ガイダンスのみを目的としています。

誰のため


  • テクスチャの作成に直面しなかった人
  • ビットマップの描画：pixelart / 2D
  • 誰がそのようなS3TC / DXTのような獣の種類を疑問に思いました
  • ゲームを作成する簡単な方法を探していない人
  • 有名なフレームワーク（Unityなど）なしでゲームを作成しようとしている人
  • UNIXシステムを使用します（後で検討するツールはUbuntuでのみテストされたため）。

S3TC / DXTとは


非常に簡単に言うと、これは画像圧縮であるため、次のようになります。b）（v）RAMに高速にロードされます。

ここで、プレートへのリンクを残すことを許可します。ここには、フォーマットの簡単な概要と、ここでそれらを使用する理由とタイミングがあります。

この形式は、ゲームで使用する方法について、より詳細に読みたい場合は、ここでは英語での4部では記事の、またはあなたは、それ自体が圧縮を数式ものを読むことを好む場合は、ここでの記事のロシア語、英語で。

問題


DXT5（現在のプロジェクトで使用しているため、例として取り上げました）は非可逆圧縮形式なので、色を犠牲にします。まず、色は8ビットRGB形式ではありません。 RGB888ではなく、RGB565形式です。これにはまだ問題の半分があります。しかし、（私の意見では）主な問題は、4 x 4ピクセルの1つのテクセル内で、出力が最大4色（基本2つと追加2つ）になることです。

あなたがこれまでにゲーム開発に遭遇したことがなく（例として）開発者（またはデザイナー/アーティスト）であり、pixelart / 2Dのスタイルで美しいマルチカラースプライトを作成するか、または背景に画像のみ（もう一度、ベクトルではなくビットマップ）を作成することを想像してください。 。

マウス/タブレットを拾い、描いた。それは美しいようです（それが美しいと仮定しましょう）。



DDSに変換するには、GIMP、Photoshopのプラグインなどを使用する必要があります。ただし、結果のファイルを開いて初めて結果が表示されます。

何のために
