LinuxのPython 3.7仮想環境でのSCIPおよびGLPKを使用したor-toolsのインストール

コンテストに参加することにしました。最適化の問題を解決するには、彼がパッケージを使用する必要がありました。私は、さまざまなソルバー（またはソルバー？ソルバー？）のインターフェースであるGoogleツールまたはツールを選択しました。いくつかの最適化ツールが含まれていますが、作成者は商用のGurobiやCPLEXを含むいくつかの外部パッケージのサポートを主張しています。ただし、私たちは裕福な人々ではなく、無料のSCIPとGLPK（ちなみに国内の開発者からのもの）を使用します。驚いたことに、私は、インターネットの英語圏と私たちの両方のオープンスペースで、仮想Python環境でこのすべての素晴らしさを機能させる方法に関する指示を見つけられなかったので、少しいじくり回す必要がありました。お急ぎの方はこちら、残りの部分は、Googleソフトウェアのインストールの浮き沈みを説明する、私の書記狂気の怪しげな品質に精通することをお勧めします。はい、私はすべてがManjaro Linuxにインストールされたことをすぐに言うのを忘れていました。


最初のステップ


3つのパッケージはすべてpythonリポジトリに存在するため、pipからインストールします。

それぞれに独自のインターフェースがあり、or-toolsはプログラムを大幅に書き換えることなくさまざまなパッケージを試すために選択されたため、or-toolsがこれらのソルバーを取得してそれ自体を実行できるようにする必要があります。そして、あなたはどう思いますか？ pipまたは-toolsを介してインストールされた場合、pipを介してインストールされたソルバーは表示されません。そのため、あなたはまだ彼らがドキュメントに書いているものを見る必要があります。
ドックでは、or-toolsが外部パッケージを取得できるように、ソースから収集する必要があるものを記述します。収集しようとすると、pipを介してインストールされたソルバーが表示されないため、何も出力されません。
したがって、pipを介してすべてのパッケージをインストールする最も簡単な方法は機能しません。or-toolsはソルバーをプルせず、ソルバーはor-toolsが望むようにライブラリを配置しないため、誰もがエラーを出し、愚かであり、あらゆる方法で笑います。

さらにドックでは、ソースコードからソルバーを収集する必要があり、次にソース自体を収集するためにソースから収集する必要があると書いています。否や言うほどない！ SCIPをインストールし、GLPKをインストールしましたが、ドキュメントに記載されていない質問は、or-toolsを仮想環境に配置する方法でした。

ドックでの唯一のmake pythonコマンドは、パッケージをシステムにグローバルにインストールします（驚くべきことに、ハァッ？）

しかし、私は幸運でした：エンドレスモンキーメソッドを使用したエンジニアリング検索中に、誤ってmake helpと入力し、すべてのor-toolsビルドオプションのリストを受け取りました。その中には、wheelパッケージをビルドする機会がありました。これは、make python_packageコマンドです（その場合は、最後にもう一度繰り返します）。それでは、ビルドして起動する時が来ましたか？



問題を探しています


or-toolsのビルド段階では何​​も機能せず、悪いGLPKライブラリを誓います。



ドキュメントのパラメータを使用してSCIPでビルドすることは成功しますが、or-toolsはインポート時に機能しません。



私は自分でこれらの問題の解決策を見つけることができなかったので、私はgithubに行き、開発者の指示に従ってインストールすると、コード内のどこかにエラーが発生する理由を尋ねました。


うーん、明確化のためのおかげで。

私はgithubの上の答えを待っていましたが、私は私を助ける何かを見つけることを期待して、インターネットを中心にrummaged。そして、それは驚くことではないので、私はそれを見つけました。それは彼がGLPKをor-toolsとリンクする方法について書いたプロジェクトチームのリードブログでした。そして、この方法がついにうまくいきました！これで、SCIPを使用せずにGLPKを使用してすべてをビルドすると、これで作業することもできます。すでに進行中です。

ただし、or-toolsチームのリーダーは、GLPKの使用について助言せず、結果は重要ではないと述べ、SCIPを無効にして機能させたいと考えています。ブログの隣の投稿は、Googleリリースパッケージが特定のバージョンで動作するという考えに出くわしました。リリース変更のリストを見て、そこにバージョン6.0.0の明示的な表示があることを確認します。makefileにバージョン6.0.2が示されているという事実にもかかわらず、私たちは古いバージョンと出来上がりをインストールしようとしています！

インストール手順


  1. ていることを確認して、アーカイブSCIPのダウンロードがバージョンのための指示されたサポートは、あなたが（私のケースでダウンロードすることを明らかにdevelopers.google.com/optimization/support/release_notes#support-for-scip-6.0.0）およびアンパック。
  2. GLPK（最新バージョン4.65の投稿時点）をダウンロードして解凍します。
  3. SCIPコマンドのインストール 

    make GMP=false READLINE=false TPI=tny ZIMPL=false scipoptlib install INSTALLDIR=<path>/scipoptsuite-6.0.0
  4. 次のコマンドでGLPKをインストールします。

    ./configure --prefix=<install_path> --with-pic --enable-shared
make
make install
  5. githubからORツールをダウンロード

    git clone https://github.com/google/or-tools
  6. 次のコマンドを使用してORツールをインストールします。

    make third_party
make python_package

    最後のコマンドが機能しない場合は、次のコマンドでwheelをインストールします。

    pip install wheel
  7. 仮想環境をアクティブ化するには、Pythonパッケージのあるフォルダーに移動し、そこからインストールします。

    仮想環境ではなく、システム全体に対してグローバルにインストールする必要がある場合：

    make python_package

    次のコマンドを実行します。

    make python
