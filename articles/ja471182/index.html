GitHubデスクトップ2.2：新機能



Gitを使用する人なら誰でも、彼がクールな学習ラインを持っていることを知っています。私たちは開発者から、同僚、教授、友人、またはYouTubeビデオでさえ、ほとんどの人が知人から通常学ぶことを学びました。GitHub Desktop 2.2では、GitとGitHubのインタラクティブチュートリアルの最初のバージョンを追加します。これは、あなたの友達になり、始めるのに役立ちます。デスクトップを初めて使用する場合は、desktop.github.comからチュートリアルをダウンロードして試すことができます  。 さて、詳細はカットの下にあります。

仕事の始まり


はじめに、リポジトリの作成とエディタの接続という2つの主要な段階を経るお手伝いをします。プログラムを最初に開くと、新しいオプション「トレーニングリポジトリの作成」を含むウェルカムページが表示されます。この機能を使用して、GitHubデスクトップを使用したGitの操作の基本概念をガイドするトレーニングリポジトリが作成されます。



GitとGitHubを使い始めるには、多くのツールが必要です。これらの中で最も重要なのはコードエディターです。チュートリアルの最初の段階で、まだエディターがない場合は、エディターをインストールするよう求められます。



GitHubを探索する


次に、GitHubデスクトップを使用してコードをローカルで変更し、作業をGitHubに送信する方法を説明します。新しいブランチを作成し、ファイルに変更を加えてコミットし、GitHubに送信して、最初のプルリクエストを開きます。

また、新しいユーザーは最初にGit、GitHub、およびGitHub Desktopの間で混乱を経験すると聞きました。マニュアルでこれらの違いを見て、それらが通常の説明をサポートすることを確認します。



自分のプロジェクトを続行します。


GitHub Desktop 1.6では、リポジトリの状態に応じて「可能な次のステップ」機能が導入されました。チュートリアルが終了したので、次のステップも提供します。貢献したいGitHubプロジェクトを探索する、新しいプロジェクトを作成する、または既存のプロジェクトをデスクトップに追加する。私たちは、GitHubデスクトップを、作業中か、初心者の開発者かに関係なく、次のステップを明確にするツールにしたいと考えています。



次は何ですか？


GitHub Desktop 2.2に取り組んでいるため、初心者が製品にアクセスしやすくなっています。フィードバックに基づいてチュートリアルを改善し、GitHubとローカルコンピューター間の接続を改善し続けます。何かの作成を開始したいが、方法がわからない場合は、GitHubデスクトップを友達として始めてみてください。
