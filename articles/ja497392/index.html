学生からソフトウェアエンジニアに成長するには？

こんにちは！私の名前はデニスドブジェンコです。数年前から、C / C ++ソフトウェアエンジニアの候補者との技術面接を行っています。シニアSWエンジニア以上の候補者との主な会話が実務経験である場合、将来のインターンとジュニアの選択は、大学で取得した知識と同じレベルの学問の問題を解決する能力に基づいています。主要な技術大学でさえ、彼らは「そうではない」と「そうではない」ことを教えていることが判明したのはここです。そして、ほとんどのトレーニングの最初の日から、学生の状態からエンジニアの状態への移行の準備を始めるのは素晴らしいことです。

この記事では、サンクトペテルブルグ開発センターのデルテクノロジーズが、学生がソフトウェアエンジニアの進路に沿って成功するための準備にどのように役立つかについて説明します。



教育、またはどこでどのように勉強しますか？


学校では、すべてのタスクに適切な解決策があるという事実に慣れています。学士号の終わりまでに、それが唯一のものではないかもしれないと感じる学生もいますが、問題の条件に最適な解決策を選択するための基準はまだ理解されていません。とにかく、学士の頭には、ソフトウェアエンジニアが訓練を受けた猿（コードモンキー）にほかならないという絶対的な自信があります。結局のところ、それはまさに実験室の仕事によって開発されたそのようなスキルです：斜めにタスクを読んで、すぐにいっぱいになります。しかし、どのような学生がセッションの1週間前にラボを開始しますか？そして突然、インタビューで、邪悪な叔父は彼に少なくとも最小限のアルゴリズム文化を要求し、いくつかの「開発プロセス」について話し、彼らに複雑な問題を解決するよう強制します...学士のジュニアステップが高すぎるため、マスターはまだ十分なステータスを持っていないようです。

私の意見では、問題はITの専門分野でさえ、プログラミング教育は実際のタスクから切り離されているということです。カリキュラムでは、その分野は「言語によるプログラミング...」と呼ばれることもありますが、実際には、教室では、遠くまで出てきた例を使用した構文は長くて退屈であり、プログラミングの芸術はほとんどありません。多くの場合、大学のコースでは、コードがコンピュータによって実行されるという理解はまったくありませんが、人々はそれを読みます！

「プログラミング」はコンピュータのハードウェアとは切り離して研究され、ソフトウェアアーキテクチャは上級コースに移行し、参照条件を読む能力に煩わされることはほとんどありません。もちろん例外もあります。たとえば、私の同僚であるA.V. Martynovがサンクトペテルブルクポリテクニックで数年間教えてきた「C ++での産業プログラミング」というコースがあります。



その結果、インタビュー中に、また学生プロジェクトの参加者によって書かれたコードでさえ、筆者が考えられる制限について単純に考えていないのに、「教科書から」の例の再現にしばしば気付きました。たとえば、C / C ++の場合、ほとんどの上級生でさえ、静的、動的に割り当てられたメモリとスタックメモリの違いを明確に説明できず、言語のキーワードとストレージ期間およびリンケージとの関係についての質問は単にそれらを恐怖に陥れます。インタビューでは、以下のような分析の例を挙げようとしました（注、できるだけ簡単に）が、今はそうしません。 f2（）関数を呼び出すとプログラムがSEGFAULTでクラッシュする理由は、ほとんどの学生にとって理解度をはるかに超えています。

#include <iostream>

using namespace std;

extern void populate(int* data, unsigned long size);

const unsigned long ARRAY_SIZE = 2048 * 2048;

int d1 [ARRAY_SIZE];

long f1()
{
  populate(d1, ARRAY_SIZE);
  long ret = 0;
  for (unsigned long i = 0; i < ARRAY_SIZE; ++i) {
    ret += d1[i];
  }
  return ret;
}

long f2()
{
  int d2 [ARRAY_SIZE];
  populate(d2, ARRAY_SIZE);
  long ret = 0;
  for (unsigned long i = 0; i < ARRAY_SIZE; ++i) {
    ret += d2[i];
  }
  return ret;
}

int main()
{
  cout << f1() << endl;
  cout << f2() << endl;
  return 0;
}

  • 雇用主は「はい、あなたはようやく、数年にわたって再訓練する必要がない、またはこの時期に上級レベル以上の従業員に費やす再訓練する必要さえない通常の卒業生を準備し始めます」と叫びます。

資格のある人材の流入に関心のある企業も、学生のトレーニングに責任を持つべきです。記事の第2部では、サンクトペテルブルクのR＆Dデルテクノロジーセンターの例を使用して、ITにおける専門家としてのキャリアを構築するために小学生と生徒をサポートするいくつかのプログラムについて説明します。

企業はどのようにして人員を「偽造」できますか？


人気のある知恵は、「彼がベンチを横切っている間に子供を育てる必要がある」と言います。この格言に基づいて、Dell Technologiesは将来のエンジニアと対話するための多段階システムを開発しました。高校生のトレーニングからセンターエンジニアの学部生や修士課程の保護の委員会への参加までです。

2018年以来、私たちはデジタルバナナの学生向けのITコースの主催者と友達になっています。昨年、私たちのエンジニアは小学生のためのハッカソンの実施を手伝い、デジタルジャングルフェスティバルで講義やマスタークラスを行いました。高校生向けのもう1つの教育プログラム、Student STARは国際的にサポートされています。開発センターがこのプログラムに参加してから8年目がすでに過ぎています。毎年6月に、物理学10年生の約30人の卒業生がソフトウェアエンジニアリングの世界に1週間滞在します。そして、Digital Bananaとの協力の測定可能な結果について話すのが早すぎる場合、主要な技術大学の学生の間で、過去数年のStudent STARの卒業生をすでに目にしています。

新入社員の間で有望な将来のエンジニアを探し続けています。 Dell Technologiesの関連ネットワーク（Polytech、SPbSU、ITMO、LETI、SUAI、PSUPS）に所属する6つの大学のLeonard Euler Foundationと共同で、最初のセッションは、パーソナライズされた「Successful Freshman Scholarship」を持つ優秀な学生に授与されます。そして、2年目には、フェローはメンタリングプログラムに参加する機会があります。ちなみに、数年前にこの記事がハブレのこのブログに掲載されました。加えて、近年の写真でも同じ顔が再び見られます。私たちのメンティーの多くは大学を卒業し、Dell Technologiesの従業員になり、メンター、教師、キュレーターの役割ですでにメンタープログラムに戻っています。

次の段階は、学生が大学で直接R＆Dプロジェクトに参加する機会を与える「共同学生プロジェクトのプログラム」です。そして、そのようなプロジェクトのすべての参加者が同じ大学の学生である必要はまったくありません。他の学生、つまり条件付きの「フェンスを構築する」大学でさえ、彼らに参加することができます。共同プロジェクトのタスクは、実際のバックログ、つまりプロジェクトでやりたいタスクからすでに取得されていますが、より優先度の高いタスクのプレッシャーの下で「後で」延期される可能性が高くなります。共同プロジェクトでは、チームリーダーの役割を大学の従業員が担い、エンジニアがお客様の役割を果たします。共同学生プロジェクトに参加することによる追加のボーナスは、優秀な学生は主にインターンの地位の候補者と見なされることです。また、共同プロジェクトの結果は、定期論文や論文の資料として使用できます。

大学との共同プロジェクトの歴史には、確かに多くの成功事例があります。たとえば、Unityストレージのユーティリティの1つの開発は、SUAIとの共同プロジェクトとして始まりました。また、ITMOの3年生（このプロジェクトのキュレーターの1人として自分の責任の一部を押し出そうとしている）にとって、共同プロジェクトの参照条件が複雑すぎると判明した奇妙なケースもありましたが、ファイナルまで続いた参加者は、困難に対処し、実際に対処する能力を示すことができました全力でインターンになった。現在、彼らはすでに本格的なエンジニアです。

共同学生プロジェクトの後にチームに参加したDell Technologiesエンジニア：アナスタシアジンナー、ニキータティカチェフ、ウラジスラフアレクセーエフ

大学は、3〜4年生の学生に可能な限りあらゆる方法で実際のプロジェクトに参加するように学生を刺激し始めます（刺激は、古代ローマの馬を駆り立てるために使用された尖った棒であることを覚えています）。そのとき、平均的な学生の頭の中に「インターンシップ」という魔法の単語がポップアップします。これは、蛹の学生から蝶へのエンジニアへの変換を瞬時に自動的に完了するはずです。 （インターンシップの申請以外に）追加の作業を必要としない別の移行は、Isaac Asimovによって説明されているのと同じくらい素晴らしいです。実生活では、瞬間的な変化はありません。インターンシップは長いプロセスです。デルテクノロジーズでは、インターンは実際にはジュニアの役割を果たし、6か月から1年以内にインターンの役割を果たしてから、会社のフルタイムの従業員になります。

明らかに、メンタープログラムと学生の共同プロジェクトの参加者は、インターンポジションに採用する際に一定のメリットがあります。最低でも、メンターやプロジェクトキュレーターからオープンポジションに関する情報を最初に受け取る人です。そしてインタビューでは、学生プログラムの参加者は通常、より強力な候補者であることがわかります。一方、学生プロジェクトに合格しなかった候補者への扉は閉めません。

この記事の範囲外で、学生と対話する方法は他にもたくさんあります。たとえば、学生フォーラムでの講義や、私たちのオフィスで定期的に開催される「テクノロジーデー」など、誰でも参加できます。労働市場の現状を考えると、企業は社内外の人材の育成に時間とお金を費やさなければなりません。私たちの行動は、私たち自身のために資格のあるスペシャリストを準備することを可能にするだけでなく、エンジニアの成長と業界全体の発展にも貢献すると信じています。
