Bellman-Fordアルゴリズム

「開発者のためのアルゴリズム」というコースの開始を見越して、彼らは興味深い記事の別の翻訳を準備しました。



問題：グラフとグラフの最初の頂点srcが与えられた場合、srcから特定のグラフのすべての頂点への最短経路を見つける必要があります。グラフには、負の重みを持つエッジが含まれている場合があります。

この問題を解決する方法として、ダイクストラのアルゴリズムについてはすでに説明しました。ダイクストラのアルゴリズムは貪欲なアルゴリズムであり、その複雑さはO（VLogV）（フィボナッチヒープを使用）です。ただし、ダイクストラは、エッジの重みが負のグラフでは機能しませんが、ベルマンフォードは完全に機能します。Bellman-Fordアルゴリズムは、ダイクストラアルゴリズムよりもさらに単純で、分散システムに適しています。同時に、その複雑さはO（VE）であり、ダイクストラのアルゴリズムのインジケーターを超えています。

推奨事項：ソリューションの表示に進む前に、練習してみてください 自分で。

アルゴリズム


以下の手順について詳しく説明します。

入力：グラフを作成し、頂点を開始しsrcます。
出力：srcからすべての頂点までの最短距離。負の重みのサイクルが発生した場合、最短距離は計算されず、そのようなサイクルの存在を示すメッセージが表示されます。

  1. このステップでは、元の頂点から他のすべての頂点までの距離は無限に初期化され、src自体までの距離は0と見なされます。要素を除いて、無限大に等しいすべての値でdist[]サイズの配列が作成され|V|ます。dist[src]ただし、srcは初期頂点です。
  2. 2番目のステップでは、最短距離を計算します。次の手順は|V|-1回実行する必要があり|V|ます。ここで、はこのグラフの頂点の数です。
    • u-v:
      dist[v] > dist[u] + uv, dist[v]
      dist [v] = dist [u] + uv
  3. , . u-v :

    • dist[v] > dist[u] + uv, .

ステップ3の考え方は、グラフに負の重みのサイクルが含まれていない場合、ステップ2が最短距離を保証するということです。もう一度すべてのエッジに行き、いずれかの頂点のパスを短くすると、これは負のウェイトサイクルが存在することを示します。

使い方？他の動的プログラミングタスクと同様に、アルゴリズムはボトムアップで最短パスを計算します。まず、最短距離、つまり、長さが1エッジ以下のパスを計算します。次に、長さが2エッジ以下の最短経路を計算します。外部ループのi番目の反復の後、計算されたiエッジ以下の最短経路長。単純なパスでは、最大| V | -1のエッジが存在する可能性があるため、外側のループは正確に実行されます| V | -1回。アイデアは、エッジがi以下の最短パスを計算する場合、すべてのエッジを反復することで、エッジがi + 1以下の最短パスを取得できることを証明します（証明は非常に単純です。MITのこの講義またはビデオ講義を参照できます））

最初の頂点を0にしますsrc。それ自体までの距離を除いて、すべての距離を無限として受け入れます。グラフの頂点の総数は5であるため、すべてのエッジを4回トラバースする必要があります。



（B、E）、（D、B）、（B、D）、（A、B）、（A、C）、（D、C）、（B、C）、（ E、D）。初めて肋骨に沿った通路が完成したとき、以下の距離が得られます。 1行目は初期距離を示し、2行目はエッジ（B、E）、（D、B）、（B、D）および（A、B）が処理されたときの距離を示します。 3行目は、処理距離（A、C）を示しています。 4行目は、（D、C）、（B、C）および（E、D）が処理されたときに何が起こるかを示しています。



最初の反復により、すべての最短パスが1エッジのパスよりも長くならないことが保証されます。すべてのエッジの2番目のパスが完了すると、次の距離が得られます（最後の行は最終的な値を示しています）。



2回目の反復により、すべての最短パスの長さが最大2エッジになることが保証されます。アルゴリズムは、すべてのエッジに沿ってさらに2回実行されます。2回目の反復の後、距離は最小化されるため、3回目と4回目の反復では距離値は更新されません。

実装：

# Python program for Bellman-Ford's single source 
# shortest path algorithm. 

from collections import defaultdict 

# Class to represent a graph 
class Graph: 

	def __init__(self, vertices): 
		self.V = vertices # No. of vertices 
		self.graph = [] # default dictionary to store graph 

	# function to add an edge to graph 
	def addEdge(self, u, v, w): 
		self.graph.append([u, v, w]) 
		
	# utility function used to print the solution 
	def printArr(self, dist): 
		print("Vertex Distance from Source") 
		for i in range(self.V): 
			print("% d \t\t % d" % (i, dist[i])) 
	
	# The main function that finds shortest distances from src to 
	# all other vertices using Bellman-Ford algorithm. The function 
	# also detects negative weight cycle 
	def BellmanFord(self, src): 

		# Step 1: Initialize distances from src to all other vertices 
		# as INFINITE 
		dist = [float("Inf")] * self.V 
		dist[src] = 0


		# Step 2: Relax all edges |V| - 1 times. A simple shortest 
		# path from src to any other vertex can have at-most |V| - 1 
		# edges 
		for i in range(self.V - 1): 
			# Update dist value and parent index of the adjacent vertices of 
			# the picked vertex. Consider only those vertices which are still in 
			# queue 
			for u, v, w in self.graph: 
				if dist[u] != float("Inf") and dist[u] + w < dist[v]: 
						dist[v] = dist[u] + w 

		# Step 3: check for negative-weight cycles. The above step 
		# guarantees shortest distances if graph doesn't contain 
		# negative weight cycle. If we get a shorter path, then there 
		# is a cycle. 

		for u, v, w in self.graph: 
				if dist[u] != float("Inf") and dist[u] + w < dist[v]: 
						print "Graph contains negative weight cycle"
						return
						
		# print all distance 
		self.printArr(dist) 

g = Graph(5) 
g.addEdge(0, 1, -1) 
g.addEdge(0, 2, 4) 
g.addEdge(1, 2, 3) 
g.addEdge(1, 3, 2) 
g.addEdge(1, 4, 2) 
g.addEdge(3, 2, 5) 
g.addEdge(3, 1, 1) 
g.addEdge(4, 3, -3) 

# Print the solution 
g.BellmanFord(0)
