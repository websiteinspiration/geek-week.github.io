ハイブリッドアプリケーションの自動テストのセットアップ

この記事では、Androidでのモバイルアプリケーションの自動テストの起動に関する問題を解決した経験を紹介します。

私が言うこと：


  • Appiumを使用して自動テスト環境を実行する方法
    （PC OS-Windows 10）
  • ハイブリッドアプリケーションでテストを実行しようとしたときに発生するいくつかのエラーを回避する方法
  • 最も単純なテストを実行するためのいくつかの基本的なコマンド。JavaとJavaScriptを使用します

仕事


ハイブリッドモバイルアプリのテストを自動化します。

テストするアプリケーションについて：
プラットホーム アンドロイド
フレームワーク Angular、Cordova、Ionic
ブラウザ 横断歩道

おそらくこれらは、テストを実行するためのPCの構成に影響を与えた重要なポイントです。

テスト環境


Appiumを自動テストツールとして使用します。
簡単に言うと、これはサーバーを使用してクライアントにコマンドを送信して、何をするのか、そしてその理由を可能にするツールです。

完全な情報は開発者の Webサイトで入手できます。

始めましょう。

ディスク上の都合の良い場所に「Appium Automation」というフォルダを作成します。

例えば：

C:\Appium automation

さらにこのフォルダー（C：\ Appium Automation）に、さらにいくつかのフォルダーを作成します。
Android SDK C：\ Appiumオートメーション\ Android SDK
Android Studio C：\ Appiumオートメーション\ Android Studio
APK C：\ Appiumオートメーション\ APK
Appium lib C：\ Appiumオートメーション\ Appium Lib
セレンlib C：\ Appiumオートメーション\ Selenium Lib

インストールしたアプリケーション/コンポーネントのいずれかである場合は、指示に従って関係のロジックを理解するだけです。すべてを再インストールする必要はありません。

作成したフォルダを埋め、他のコンポーネントをロードする

6. Androidデバイスのセットアップ


実際のデバイス
デバイスをコンピューターに接続し、スマートフォン自体を直接接続するには、USBケーブルが必要です。

開始するには、スマートフォンの[設定]-[スマートフォンについて]に移動します。

さらに、スマートフォンごとにすべてが異なりますが、一番下の行は、「シリアル番号」を見つけて5〜7回クリックし、トースターに「あなたは開発者になりました」と表示されるまでです（通知テキストはモデルごとに異なる場合があります）。

この操作の後スマートフォンの設定に「開発者向け」セクションが表示されます。また、たとえば、「設定」-「開発者向け」、「設定」-「特殊機能」-「開発者向け」など、さまざまなセクションでモデル間を非表示にすることもできます。 。

「開発者向け」セクションに移動し、「USBデバッグ」をアクティブにする必要があります。

スマートフォンをUSB経由で接続すると、USB経由のデバッグ許可に関するメッセージが表示されます。[このデバイスを信頼する]をオンにして[OK]をクリックすると、次に電話がこのPCに接続されたときにこのメッセージが再び表示されなくなります。
すべてのスマートフォンが自動テストで動作する準備ができています。

仮想デバイス

Android Studioを使用して仮想デバイスを作成することもできます。その場合、実際のデバイスをコンピューターに接続する必要はありません。

しかし、仮想デバイスでは、テストは実際のデバイスよりも遅くなる可能性があります。

作成するには、Android Studioを開き、次の手順を実行します。

  • 「ツール」-「AVDマネージャー」-「+仮想デバイスの作成...」
  • デバイスを選択し、[次へ]をクリックします
  • Androidの目的のバージョンを選択し、[次へ]をクリックします
  • デバイスの名前を指定し、[完了]をクリックします

「ツール」-「AVDマネージャー」セクションでの操作が完了すると、作成されたデバイスが表示されます。緑の「再生」アイコンを使用して、デバイスを起動できます。

PCが接続されたデバイスを見たことを確認するにはどうすればよいですか？

コマンドライン（「スタート」-「ユーティリティ-Windows」-「コマンドライン」）
を実行し、コマンドadb devicesを入力します。

その結果、デバイスが「USBデバッグ」機能を有効にして接続されている場合、接続されたデバイスとそのUDID（シリアル番号）を受け取ります。

List of devices attached
UDID   device

7.プロジェクトを作成しています


インポートを追加します。

package name;

import java.util.*;
import io.appium.java_client.android.AndroidDriver;
import io.appium.java_client.android.AndroidElement;
import io.appium.java_client.remote.MobileCapabilityType;
import org.openqa.selenium.*;
import org.openqa.selenium.remote.DesiredCapabilities;
import org.openqa.selenium.JavascriptExecutor;
import java.net.MalformedURLException;
import java.net.URL;
import java.util.concurrent.TimeUnit;

サーバーとリクエストの送信先を登録します

。Androidドライバーをインストールします。

AndroidDriver<AndroidElement> driver = null;

パラメータ（プロパティ）を設定します。

DesiredCapabilities capabilities = new DesiredCapabilities();

デバイスの名前（実際のデバイスの場合は、[設定]-[電話について]と仮想の[ツール]-[AVDマネージャー]-[名前]フィールドで確認できます）：

capabilities.setCapability(MobileCapabilityType.DEVICE_NAME, "MyPhone");

起動する必要があるAPKへのリンク（.apkはデバッグビルドである必要があるため、ユーザーとappiumはアプリケーションを検査できます）：

capabilities.setCapability(MobileCapabilityType.APP, "C:\\Appium automation\\APK\\My-debug-Apk.apk");

Appium Driverを起動します。

try {
            driver = new AndroidDriver<>(new URL("http://127.0.0.1:4723/wd/hub"), capabilities);
            driver.manage().timeouts().implicitlyWait(10, TimeUnit.SECONDS);
        } catch (MalformedURLException e) {
            System.out.println(e.getMessage());
        }

このコードは、接続されたデバイスでアプリケーションを起動するのに十分です。

追加コマンド

プロジェクトを特定の時間、たとえば8秒間一時停止するため。（値はミリ秒で示されます）。ページまたはアプリケーションが読み込まれるのを待つ必要がある場合は、次のようにしてください。

Thread.sleep(8000);

ContextHandlesを取得します。ここで、現在作業しているもの（NATIVE_APPまたはWEBVIEW）を理解しています。

Set<String> contextNames = driver.getContextHandles();
        for (String contextName : contextNames) {
            System.out.println(contextName); 
        }

配列で受け取った最後の値にコンテキストを割り当てます。これにより、WEBVIEW、tkに正しく​​切り替えることができます。デフォルトのコンテキスト= NATIVE_APP：

driver.context((String) contextNames.toArray()[contextNames.toArray().length - 1]);

マネージドアプリケーションインタフェース

ので、ハイブリッドアプリケーションで作業しています。アプリケーション要素はNATIVE_APPではなくWEBVIEWでレンダリングされるため、ほとんどの場合、findElementByCssSelector要素に対して1つのタイプの検索を使用する必要があります。

[OK]ボタンをクリックします。

driver.findElementByCssSelector(".button-ok").click();

-.button-okは要素クラスです。要素IDを正確に使用することもできます。

フィールドに値を送信します。たとえば、検索してそこに「Cinema」という値を送信するとします。

driver.findElementByCssSelector(".search-input").sendKeys("");


クリックして検索：

driver.findElementByCssSelector(".search-button").click();

このようにして、アプリケーションをすでに制御できます。

遅かれ早かれ、それに遭遇します。すべてのコンテンツがスマートフォンの画面に配置されるわけではありません。スクロールする必要があります。テスト中に要素が画面に表示されない場合、Appiumは要素が表示されないというエラーをスローし、ほとんどすべてのテストはエラーでカバーされます。

画面をスクロールするには、JavaでJS関数scrollIntoViewを実行します。

((JavascriptExecutor) driver).executeScript("document.querySelector('.button-ok').scrollIntoView({block: \"end\", behavior: \"smooth\"});");

私の場合、これで[OK]ボタンに移動できます。

なんらかの形で正しく書き込んだのに、スクロールが実行されない場合があります。

画面上の他の要素にバインドしてみてください。最初に、Chromeのコンソールからこれを実行することをお勧めします-検査。この要素にバインドすると、スクロールが機能することを理解できることが保証されます。

ステップ8に進むことができます。すべてがうまくいったら、これはすばらしいことです。
しかし、別の落とし穴がありました。
彼の名前はCrosswalk Browserです。

私がプロジェクトを始めたとき、私は常にエラーを注いだ：

“Make sure the app has its webview configured for debugging”

したがって、1つのテストは実行されませんでした。

この場合に確認する必要があるもの：

  • デバッグビルドがあることを確認してください
  • デバッグビルドがInspect-aアプリケーションを許可することを確認してください
  • 固定Chromedriverをインストールする
    • リンクhttps://github.com/blutter/appium-crosswalk-fixに従ってください
    • インストールはnpmを介して行われるため、事前にPCにインストールしてください（https://www.npmjs.com/get-npm）

これらの操作の後、プロジェクトにもう1つの機能を追加します。

capabilities.setCapability("chromedriverExecutable", "C:\\node_modules\\appium-with-crosswalk-fix\\chromedriver\\2.28\\chromedriver.exe");

-ここで、C：\\ node_modules \\ appium-with-crosswalk-fix \\ chromedriver \\ 2.28 \\ chromedriver.exe
は、修正されたchromedriverのパスです

。プロジェクトは次のようになります。

package name;

import java.util.*;
import io.appium.java_client.android.AndroidDriver;
import io.appium.java_client.android.AndroidElement;
import io.appium.java_client.remote.MobileCapabilityType;
import org.openqa.selenium.*;
import org.openqa.selenium.remote.DesiredCapabilities;
import org.openqa.selenium.JavascriptExecutor;
import java.net.MalformedURLException;
import java.net.URL;
import java.util.concurrent.TimeUnit;

public class Main {

public static void main(String[] args) throws InterruptedException {

            AndroidDriver<AndroidElement> driver = null;
            DesiredCapabilities capabilities = new DesiredCapabilities(); 

            capabilities.setCapability(MobileCapabilityType.DEVICE_NAME, "MyPhone");
            capabilities.setCapability(MobileCapabilityType.APP, "C:\\Appium automation\\APK\\My-debug-apk.apk"); 
            capabilities.setCapability("chromedriverExecutable", "C:\\node_modules\\appium-with-crosswalk-fix\\chromedriver\\2.28\\chromedriver.exe");

            try {
                        driver = new AndroidDriver<>(new URL("http://127.0.0.1:4723/wd/hub"), capabilities);
                        driver.manage().timeouts().implicitlyWait(10, TimeUnit.SECONDS);
            } catch (MalformedURLException e) {
                        System.out.println(e.getMessage());
            }
            
            Set<String> contextNames = driver.getContextHandles();
                    for (String contextName : contextNames) {
                        System.out.println(contextName); 
                    }

            driver.context((String) contextNames.toArray()[contextNames.toArray().length - 1]);          

            Thread.sleep(8000);
            driver.findElementByCssSelector(".search-input").sendKeys("");
            driver.findElementByCssSelector(".search-button").click();

}
