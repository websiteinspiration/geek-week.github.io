CC2531を使用してHomebridgeにZigBeeデバイスを追加する

Xiaomi Gatewayなどのブランドハブを使用せずに、スマートホームコントロールシステムを備えたさまざまなメーカーのZigBeeデバイスを作成する方法に関するHabr に関する記事はすでに あります。しかし、Habréだけでなく他のインターネットオープンスペースにもHomebridgeの指示は見つかりませんでした。したがって、ZigBeeデバイスをhomebridgeに追加した経験を共有したいと思います。リソースzigbee2mqtt.ioの作成者に多くの感謝を表明する必要があります。CC2531モデムをフラッシュし、ホームアシスタント、Domoticz、Majordomoなどのスマートホームコントロールシステムに接続する方法については、すでにそこで説明されています。この記事は、すでにインストールおよび構成されているユーザーを対象としています。ホームブリッジ。


記事のすべてのアクションは、次の機器のセットで実行されます。

  • Raspberry Pi 3 B +（Raspbian OS）
  • CC2531ドングル（ステッチ）
  • Xiaomiデバイス（ワイヤレススイッチ、温度および湿度センサー...）
  • MacBook Pro（OSX 10.14）

ロシア語のファームウェアCC2531の手順は、こちら（Pi経由）とこちら（CCDebuger経由）にあります。

概略的に、一連の作業全体は次のように表すことができます



。

モデムをフラッシュした後、MQTTブローカーをインストールする必要があります（まだインストールされていない場合）。

私はこのようにインストールされたMosquittoを使用します：

sudo apt install -y mosquitto mosquitto-clients #
sudo services mosquitto start # 
sudo systemctl enable mosquitto.service #

より詳細な設定手順はここにあります。

次に、CC2531モデムと対話してデータをMosquittoに転送するzigbee2mqttサービスをインストールする必要があります。

sudo git clone https://github.com/Koenkk/zigbee2mqtt.git /opt/zigbee2mqtt
sudo chown -R pi:pi /opt/zigbee2mqtt
cd /opt/zigbee2mqtt
npm install

zigbee2mqttをサービスとして実行するには、ファイルを作成する必要があります。
/etc/systemd/system/zigbee2mqtt.service

次の内容で：
[Unit]
Description=zigbee2mqtt
After=network.target

[Service]
ExecStart=/usr/bin/npm start
WorkingDirectory=/opt/zigbee2mqtt
StandardOutput=inherit
StandardError=inherit
Restart=always
User=pi

[Install]
WantedBy=multi-user.target


すでにMQTTブローカーをインストールしている場合は、そのファイルに正しい接続設定をファイルに登録する必要があります。

/opt/zigbee2mqtt/data/configuration.yaml

サービスを開始し、自動実行に追加します。

sudo systemctl start zigbee2mqtt
sudo systemctl enable zigbee2mqtt.service

ログを確認します。

sudo journalctl -u zigbee2mqtt.service -f

すべてが正しく構成されている場合は、次の行が表示されます。
`permit_join` set to  `true` in configuration.yaml.
Allowing new devices to join.
Set `permit_join` to `false` once you joined all devices.

Connecting to MQTT server at mqtt://localhost
zigbee-shepherd ready
Connected to MQTT server


Homebridgeとの統合を開始できます。zigbee2mqtt.ioリソースには、サポートされているデバイスのリストとペアを作成するための手順が含まれています。たとえば、Xiaomiワイヤレススイッチの追加を検討してください。



Mosquittoでイベントを表示するには、MQTT Explorerプログラムを使用します。まず、MQTTサーバーのアドレスを入力します192.168.1.104。私の場合は、アドレスにあります。すべてが正しくインストールされている場合、ツリーにはzigbee2mqttブランチが表示されます。それに入る。 Xiaomiワイヤレススイッチで、ダイオードが点滅するまでリセットボタンを押し続けます（約5秒）。 zigbee2mqttブランチで約10秒後、デバイスとデバイスが送信するデータが表示されます。

これが発生しない場合は、ダイオードが点滅した後、毎秒の頻度でリセットを1回押してみてください。次のように



なります。データを分析しましょう。

{
  "linkquality":110,
  "battery":100,
  "voltage":3052,
  "click":"single"
}

クリックフィールドにはクリックのタイプが含まれています。クリックのタイプは5つあります。
Click, double click, triple click, long, long_release

データを受信する方法を学びました。次に、Xiaomiワイヤレススイッチをホームブリッジに追加します。MQTTと対話するには、homebridge-mqttthingモジュールを使用します。他と同じ方法でインストールします。

(sudo) npm install -g homebridge-mqttthing

このモジュールは、スイッチからサーモスタットまで、多数のデバイスクラスをサポートしています。Xiaomiワイヤレススイッチには、このクラスが最適StatelessProgramableSwitchです。追加するには、トピック(0x00158d00022c85ea)とチーム名を知っている必要があります。この種の構成が判明しました：

{
       "accessory": "mqttthing",
       "type": "statelessProgrammableSwitch",
        "name": "Xiaomi Single Button",
        "topics": {
            "getSwitch": {
               "topic": "zigbee2mqtt/0x00158d00022c85ea",
                "apply": "return JSON.parse(message).click;"
            }
        },
        "switchValues": [
            "single",
            "double",
             "long_release"
        ]
}
