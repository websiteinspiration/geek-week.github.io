18 GitLab features go open source


We open the source code of features from the stages Plan, Create, Verify, Package, Release, Configure and Defend.


Sid Sijbrandij spent some time studying the GitLab features and came to the conclusion that, according to the strategy for determining GitLab plans aimed at the buyer , the source code of eighteen features from seven different stages of the DevOps life cycle should be open.


When in 2018 we introduced a new model of GitLab plans, we decided that features will be distributed between the four plans based on who is the consumer of this feature. Features intended for an individual user fall into the Core / Free plan, features for managers fall into Starter / Bronze, for directors into Premium / Silver and for senior management into Ultimate / Gold. On our pricing page, we explain:


A feature falls into the plan based on who it really will be important to . This makes sense, since a more expensive plan needs a higher-level buyer.
