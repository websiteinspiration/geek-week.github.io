必ずPouetに関する作業のアーカイブをダウンロードし、紹介（April Fools）の記事とコメントを読んでください。2バイトコードがx86でどのように機能するかを実際に示す最初のビデオをご覧ください。そして、それからはじめて、以下のテキスト全体をマスターしてください。


はい、2Bかどうかにかかわらず、2Bは実際にはさまざまなサイズコーディング作業を開始するための環境であり、非常にシンプルで、おそらく最小のものです。ただし、独自の要件と制限があります。

他の誰かがそれをキャッチしていない場合は、ツール2b.comが DOS（DOSBox、FreeDOS、MS-DOS）から起動され、コマンドラインエリア（PSPセグメントのオフセット$ 82 *）にジャンプして、実行に転送されたコードを起動します。バイナリのコマンドライン。実際、このコードは、キーボードで入力できる文字列の形式（つまり、33から126のコードを持つASCII文字で構成される）である可能性がありますが、後で詳しく説明します。

* Pascal表記$ XXで表記する16進数。これは便利で、fasmを使用してこれを行うことができます。

知っておくべき重要なことは何ですか？


Null、fasmをコンパイラーとして使用することをお勧めします。すべてのツールは専用に作成されています。

まず、メインコードの最大サイズは125バイト（これらはコマンドラインの長さの制限です）で、通常のCOMプログラムとして開始され、通常のように$ 100ではなく、$ 82のオフセットで開始されます。メインコードの直後に、コード13（$ 0D）のキャリッジリターン（CR）記号が自動的に追加され、コマンドjmp short $82（$ EB、$ 80）が$ 100に配置されます。

第2に、起動はBATファイル（まあ、またはコマンドラインインタプリタから）から行われることが想定されているため、コードに一部の文字を含めることはできません。まず、これらはI / Oリダイレクション文字（ "<"、 ">"、 "|"）と、パラメーター置換の文字と環境変数（ "％"）です。一部のシステム（V86からのDOSプログラムの起動をサポートするWindowsを含む）では、文字「＆」、「^」も特別な意味を持っています。 32までのコードの特殊文字は、すべてのDOSでサポートされているわけではなく、一部またはすべてでサポートされていない（DOSBoxには特にわずかなセットがある）ため、これらの文字もすべて除外されます。

3番目に、すべてのレジスタとフラグの開始値は、COMプログラムを開始するときと同じです。 DOSの大部分では、最初は次のようになります：ax = bx = 0（ほとんどの場合それです）、cx = $ FF、dx = cs = ds = es = ss、si = $ 100、di = sp = $ FFFE（十分な操作可能メモリ）、bp = $ 9XX（下位バイトはどこでも異なりますが、その高いテトラッド、つまりニブル、通常= 1）、フラグcf = df = 0。それを使用するかどうかはあなた次第です。

何よりも、「2番目」のポイントはここでは混乱しますね。
次のように記述する必要があるとします。

   mov ah,0
   int $16
   cmp al,27
   je x

次に、すぐに5つの禁止文字が一度に表示されます。0インチmov ah,0、$ 16 int $16インチ、$ 3C（文字 "<"）、27（$ 1B）でcmp al,27あり、コード内のどこかにコード<32 in が配置されているje x場合xは、コード<32の数字です。

何をすべきか？他のコマンドで置き換えることができるものは、次のものに置き換えられます。

  • 代わりmov ah,0に、xor ah,ahまたはcbw（可能であれば）を書き込みます。
  • 代わりcmp al,27に、not al+ sub al,not 27またはxor al,not 27+ を記述しinc al、さらに優れています（ここでは、キーが押されるのを待ち、受け取ったコードをESCキーコードと比較する必要があるため）- dec ah。

Cは、int $16より複雑な、しかし、あなたはそれについて考える場合は、xor ah,ah+の構造はint $16と、例えば、交換することができますmov ah,$83+ ror ah,1+ int $21。

残りje $+10ます。少なくとも2つの方法があります。（十分な距離まで）後方にジャンプするか、そこから前方に移動します。または、コードのバイトを置き換えます。たとえば、と書くことができますがz: je ($*2+3)-x、上のどこかに：not byte [si-($100-(z+1))]。

その結果、以下が得られます。

   not byte [si-($100-(z+1))]  ;  2-  ( ) je  si=$100
   mov ah,$83
   rol ah,1  ; ah=7
   int $21  ;   ,    al
   not al
   sub al,not 27  ; cmp al,27
z: je ($*2+3)-x  ;    x (   )

代替ソリューション


もちろん、100バイト以上の禁止文字の最後のイントロにはかなりの数（たとえば、15〜20など）が存在する可能性があり、そのような操作を行うたびにかなり退屈な作業が必要になるだけでなく、多くの場合、コード長が長くなります。

したがって、暗号化に頼ることができます。コード全体、または個別の場所。2b_life.asmの例では、各バイトに$ ACを追加してすべてのコードを暗号化します。最初の暗号化の後、約4つの禁止文字が残っています。他のコマンドに置き換えることで解決できます。もちろん、暗号化方法（add、sub、xor、notなど）とキーの選択にも時間がかかりますが、これはすべての悪の中でより少ないです。デコーダーのコードはわずか8バイトです-これはこの状況ではまったく問題ありません。暗号化はディレクティブを使用して自動的に行われます。repeat、loadおよびstore（すなわち、我々は、コードがすでに暗号化され得ます）。


2b_note.asmの 例では、個々の場所が暗号化されています。ここで、もう一度、の助けを借りてrepeat、loadそしてstore$ 3Dは、いくつかのバイトに追加された値、およびこれらのバイトのアドレスのリスト別々に格納される（ような各バイトのアドレスの1バイト）。合計で、20バイトを暗号化し、13バイトが復号化機能によって占有されます。はい、最初の方法はより経済的でした:)

記事の冒頭で、コードについて説明することを約束しました。これは、33から126までのコードを持つASCII文字で構成される文字列の形式をとることができます（そのため、たとえばキーボードで特別な問題なしに入力できます）。これは、たとえば、コードが16進文字などを使用して暗号化されている場合に可能です。はい、これは無駄ですが、BASE64方式を使用して暗号化すると、デコーダーがそのような文字のみで構成されるため、費用がさらに高くなる可能性があります。
