C ++でのプロパティの実装

みなさん、こんにちは！プロジェクトを実装する場合、開発者はフィールドのみを含み、関数を持たないクラスを必要とすることがよくあります。そのようなクラスは、必要な情報とその後の操作を保存するのに役立ちます。

このようなクラスを実装する最初のアプローチは、C構造体の使用に基づいています。このアプローチの欠点として、すべてのフィールドが書き込みと読み取りが可能であり、常に良いとは限りません。

struct person {
   int id;
   human type;

   std::string name;
   std::string address[5];

   bool merried;
};

2番目のアプローチは、すべてのフィールドを非表示にし、フィールドのゲッターとセッターを提供することに基づいています。このアプローチは、Java言語で鮮明に使用されます。利点として、フィールドへのアクセスを制御できます。短所としては、クラスが大きくなること、ゲッターとセッターが論理的な負担を負わないことなどがあります。

class person {
public:
   void set_id(int id) {
      this->id = id;
   }

   int get_id() const {
      return id;
   }

   void set_merried(bool merried) {
       this->merried = merried;
   }

   bool is_merried() const {
       return merried;
   }

   void set_type(human type) {
       this->type = type;
   }

   human get_type() const {
       return type;
   }

   void set_name(const std::string& name) {
       this->name = name;
   }

   const std::string& get_name() const {
       return name;
   }
private:
   int id;
   human type;

   std::string name;
   std::string address[5];

   bool merried;
};

フィールドがクラスタイプの場合、オブジェクトを値、左辺値リンク、右辺値リンクで設定する必要がある場合があります。また、const / volatile修飾子も使用します。

class person {
public:
   void set_name(const std::string& name) {
       this->name = name;
   }

   void set_name(std::string&& name) {
       this->name = std::move(name);
   }

   const std::string& get_name() const {
       return name;
   }

private:
   int id;
   human type;

   std::string name;
   std::string address[5];

   bool merried;
};

多くの言語は、言語機能としてプロパティをサポートしています。コードの記述がより簡潔で信頼性の高いものになります。ゲッターとセッターの記述を簡単にするために、マクロを使用してコードを生成できます。

#define SETTER_PRIM(type, name)	 \
	void set_##name(type value) {	 \	
		this->name = value;		 \
	}

#define GETTER_PRIM(type, name) 	\
	type get_##name() const { 	\
		return name; 			\
	}

ただし、マクロの使用は危険です。型（type）または間違った型（name）の変数を正しく示すことができない場合があります。最良のケースでは、ゲッターとセッターを使用するとランタイムエラーが発生します。最悪の場合、エラーは残ります。

ゲッターとセッターを生成できるようにしたかったのですが、同時にタイプ（タイプ）の正確さ、変数（名前）を使用したタイプの正確さ、およびタイプが等しいことを確認できました。そして、これは、type_traits標準ライブラリを使用して、C ++ 11以降で実行できます。


#define SETTER_PRIM(type, name)				\
    void set_##name(type value)	{				\
	using T1 = type;                                                \
        using T2 = decltype(name);                                \
	static_assert(std::is_fundamental<T1>::value, 	\
			  "only primitive types");			\
	static_assert(std::is_fundamental<T2>::value,	\
			  "variable must be primitive");		\
	static_assert(std::is_same<T1, T2>::value,  	\
			  "both types must be same");		\
	this->name = value;						\
   }

#define GETTER_PRIM(type, name)			       \
    type get_##name() const { 				       \
        using T1 = type;                                               \
        using T2 = decltype(name);                               \
	static_assert(std::is_fundamental<T1>::value, 	\
			  "only primitive types");			\
	static_assert(std::is_fundamental<T2>::value,	\
			  "variable must be primitive");		\
	static_assert(std::is_same<T1, T2>::value,  	\
			  "both types must be same");		\
	return name; 							\
    }

このアプローチを使用すると、すべてのタイプのクラスフィールドにゲッターとセッターを実装できます。

  • プリミティブ型
  • オブジェクトタイプ
  • 乗り換え
  • アレイ
  • ポインタ
  • リンク

ヘッダーのみのライブラリの形式で実装されたゲッターおよびセッターのすべてのマクロ。ヘッダーファイルを1つだけ接続することにより、必要なゲッターとセッターをすべて備えた日付クラスを簡単に実装できます。

#include "property.hpp"

class person {
public:
   person() = default;
   ~person() = default;

   SETTER_PRIM(int, id);
   SETTER_FLAG(bool, merried);
   SETTER_ENUM(human, type);

   SETTER_PTR(int, next);
   SETTER_ARR(std::string, address, 3);

   SETTER_OBJ_LR(std::string,  name);
   SETTER_OBJ_CLR(std::string, name);
   SETTER_OBJ_RR(std::string,  name);

   GETTER_PRIM(int, id);
   GETTER_FLAG(bool, merried);
   GETTER_ENUM(human, type);

   GETTER_OBJ_LR(std::string,  name);
   GETTER_OBJ_CLR(std::string, name);

   GETTER_PTR(int, next);
   GETTER_ARR(std::string, address);

private:
   int id;
   human type;

   std::string name;
   std::string address[5];

   bool merried;
   int* next;
};
