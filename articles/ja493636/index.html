I.年間収益の計算：ツール開発


すぐに次の式が思い浮かびます
[利益] / [投資された資金] / [日数] * 365 = [利益率（％）/年間]
彼女はお金を投資したとき本当に働き、しばらくするともっと離陸しました。

複数回に分けて入金する場合や定期的に支払いを行う場合はどうすればよいですか？この状況では、アカウントにどれだけの時間が費やされたかを理解することは困難です。当然のことながら、このような問題に遭遇したのは私が最初ではなく、現在2つの解決策があります。

  1. . , . : , .
  2. IRR . , . ?

すべてのキャッシュフローを合計した場合（投資を差し引き、引き出しを追加した場合）、結果として生じる差は収入です。たとえば、彼らは銀行に10,000ルーブルを投資しました。1年後、彼らは10,600ルーブルを削除しましたが、収入は600ルーブルでした。また、投資額に金利を掛けることで収入を得ることができます。たとえば、10,000ルーブルに6％の銀行金利を掛けると、600ルーブルになります。

さらに興味深いことに、利益を補う数値でキャッシュフローを変更します。これはより有能な説明ですが、本質的には全体のロジックは次の式にあります。

IRRは不明な年間収益です。例の値を銀行預金で置き換えます：-10,000ルーブル/（1 + 6％）^ 0 + 10,600ルーブル/（1 + 6％）^ 1 =0。すべてが収束します。これで、IRRを表現し、ループまたはExcelの関数PUREを使用して、プログラムですでに計算しています。

function XIRR(values, dates, guess) {
  // Credits: algorithm inspired by Apache OpenOffice
  
  // Calculates the resulting amount
  var irrResult = function(values, dates, rate) {
    var r = rate + 1;
    var result = values[0];
    for (var i = 1; i < values.length; i++) {
      // result += values[i] / Math.pow(r, moment(dates[i]).diff(moment(dates[0]), 'days') / 365);
      result += values[i] / Math.pow(r, daysLag(dates[i],dates[0]) / 365);
    }
    return result;
  }

  // Calculates the first derivation
  var irrResultDeriv = function(values, dates, rate) {
    var r = rate + 1;
    var result = 0;
    for (var i = 1; i < values.length; i++) {
      // var frac = moment(dates[i]).diff(moment(dates[0]), 'days') / 365;
      var frac = daysLag(dates[i], dates[0]) / 365;
      result -= frac * values[i] / Math.pow(r, frac + 1);
    }
    return result;
  }

  // Check that values contains at least one positive value and one negative value
  var positive = false;
  var negative = false;
  for (var i = 0; i < values.length; i++) {
    if (values[i] > 0) positive = true;
    if (values[i] < 0) negative = true;
  }
  
  // Return error if values does not contain at least one positive value and one negative value
  if (!positive || !negative) return '#NUM!';

  // Initialize guess and resultRate
  var guess = (typeof guess === 'undefined') ? 0.1 : guess;
  var resultRate = guess;
  
  // Set maximum epsilon for end of iteration
  var epsMax = 1e-10;
  
  // Set maximum number of iterations
  var iterMax = 50;

  // Implement Newton's method
  var newRate, epsRate, resultValue;
  var iteration = 0;
  var contLoop = true;
  do {
    resultValue = irrResult(values, dates, resultRate);
    newRate = resultRate - resultValue / irrResultDeriv(values, dates, resultRate);
    epsRate = Math.abs(newRate - resultRate);
    resultRate = newRate;
    contLoop = (epsRate > epsMax) && (Math.abs(resultValue) > epsMax);
  } while(contLoop && (++iteration < iterMax));

  if(contLoop) return '#NUM!';

  // Return internal rate of return
  return resultRate;
}

以前受け取ったリクエストに基づいて否定的なフレーズを取り入れます。誰かが特定の方法、コースの著者、またはまったく異なる分野の事柄を探しています。たとえば、「アセット1とアセット2の比較」というクエリは、財務に関するものではなく、時計のモデルに関するものであることがわかりました。さらに、インターネットには否定的なフレーズの普遍的なリストがあります。私の場合、これはコース、卒業証書、タスクなどです。この段階では、無関係な要求をすべて切り離すことはできないため、分析と新しいフレーズを常に監視する必要があります。

詳細設定では、詳細な地域ターゲティングを残して、Yandexに基づくオペレーターとのチャットを追加します。今では、できるだけユーザーとコミュニケーションをとることが重要です。 Yandex-Dialogs、チャットの作成->ビジネス向けチャットに移動します。



サイトと同じアイコンを追加することが重要です。 2サイズの画像を作成しています。 favicon.pngサイト用の64x64ピクセル。サイトのあるフォルダーにドロップする必要があります。ダイアログの224x224も同じです。



[ウィジェット]タブで保存した後、コードをコピーしてサイトに追加し、検索エンジンだけでなくサイトでもオペレーターとのダイアログを開くことができるようにする必要があります。次に、[全般]タブで、ダイアログの識別子をコピーして、Yandex.Direct広告会社に貼り付けます。



「サイトがダウンしたときに広告を停止する」という項目も含めて、広告キャンペーンを保存します。

Excelテーブルでさらに編集を行うため、例としてグループとお知らせのみを作成するインターフェイスを作成します。追加->グループ。ここでは、キーフレーズに任意の行を記述する必要があります。保存後、Directは広告の作成を提案します。

私たちは、あらゆる見出し、一次および二次を叩きます。アナウンスのテキストはすぐに普遍的に書くことができます。画像とビデオは、クリエイティブを使用する時間がなくなるまで追加されません。テキスト広告のみが表示され、コンバージョンも増加します。

連絡先情報と仮想名刺がありません。移行ページに情報を追加できます。たとえば、site.ru /＃invest-programaです。別のページは表示されませんが、広告のサイズが大きくなり、見やすくなります。各リクエストに対応する名前のリンクがある場合、ユーザーにとってより関連性が高くなります。

絞り込みとクイックリンクを追加します。これは、以前に短縮ツールで作成された追跡リンクが必要になる場所です。以下を作成します。

  • エクセルスプレッドシート
  • ウェブアプリケーション
  • iOSアプリ
  • Androidアプリ

それらのうち3つは、テスト用に同じ形式のレコーディングにつながるので、これは、ユーザーがどのアプリケーションフォーマットを望んでいるかを同時に見つける機会です。リンクに説明を追加します-もう一度、広告のサイズを大きくします。セーブ。

Excelによる広告の追加


より正確な設定を行うには、キーワードフレーズごとに広告を作成することをお勧めします。私はそれらを100以上持っています。さらに2つのフレーズ、2つのテキストを作成し、組み合わせて、各キーフレーズに対して4つの広告を取得することをお勧めします。すでに、アルゴリズムは経験的に最も変換の多いものを特定しています。このような量をインターフェイスで行うのは非現実的であるため、[ツール]-> [Excelで管理]に移動します。表をダウンロードしてください。



恐ろしいように見えますが、1つのグループと1つの広告が事前に入力されているため、それを理解することができます。グループID、フレーズ、お知らせを削除し、Yandex.Directにアップロードすると、新しいお知らせが作成され、更新されません。

以前に収集したキーフレーズに戻ります。ここであなたは汗をかく必要があります。コンテキストに合わないものはすべて削除します。



荷下ろしに戻ります。広告行は、キーフレーズが残っている限り何度でもコピーする必要があります。次に、グループ名、フレーズ、タイトルの3つの場所にキーフレーズの列を挿入します。はい、ユーザーのリクエストは広告のヘッダーに完全に複製されるため、最大のコンバージョンが得られます。 35文字を超える場合は、トリミングする必要があります。それだけです、次の写真が判明しました。先ほど述べたように、必要に応じて、広告の組み合わせをいくつか作成できます。



保存してDirectに再度読み込みます。それは多くのグループであり、それぞれに1つのキーフレーズと1つの広告があります。原則として、いくつかのグループを作成し、それぞれに意味が共通するいくつかのキーフレーズと、対応する見出しを持つ同じ数の広告を作成することは可能ですが、Yandexは常に広告を正しく選択するとは限りません。これが何に関係しているのかわかりません。

キャンペーンを支払い、実行できます。このような低頻度の広告では、すべてが正しく行われていれば、最初は10,000ルーブル以内に保つことができます。私は1500を投げました。

結果


そのような予算を持つユーザーはほとんどいないので、実際に各ユーザーを個別に追跡する機会を得ました。

メトリカを使用して、私は彼が入力したクエリを調べ、そこから問題の新しい領域を学び、広告に否定的なフレーズを追加しました。 Webvisorを通じて、サイト上のすべてのアクションを追跡しました。リンクナビゲーションパスとGoogle Appsログのエラーについて調べました。

数字に移りましょう。





コンバージョンは本当に良いですが、繰り返しますが、私は少数のスーパーターゲットユーザーに焦点を合わせました。カバレッジを増やす場合は、戦略を変更し、予算を引き上げ、コンバージョンの減少を許容する必要があります。
