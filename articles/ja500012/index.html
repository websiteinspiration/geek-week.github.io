前書き


最新のASP NET CoreアプリケーションのほとんどはEntity Framework Coreを使用しています。Entity Framework Coreは、Microsoftからデータベースにアクセスするためのテクノロジーです。データベーステーブルではなく、エンティティ、つまりクラスとNETオブジェクトを使用してDBMSと対話できます。これはC＃で最も有名で機能的なORMツールです。ORMはオブジェクトリレーショナルマッピングです。データを実際のオブジェクトにマッピングします。

たとえば、開発者がデータベースを直接操作する場合、プログラマはSQLとSQLパラメータの接続、準備、クエリとトランザクションの送信方法について検討する必要があります。そして、Entity Framework Coreの助けを借りて、これはすべて自動的に行われます-開発者はNETクラスを直接操作します。

ORMアプローチ


ORMにはいくつかのアプローチがあります。

最初はコードファーストです。これは、Cコードが最初に記述され、次にこのコードを使用してデータベースが作成されることを意味します。このアプローチでは、データベースに格納されるモデルまたはエンティティのクラスを定義し、それをC＃クラスにモデルとして記述し、使用するデータベースで機能するコンテキストクラスを作成することが非常に重要です。 Code Firstアプローチは、C＃プログラマーによって最も頻繁に使用されます。

第二のアプローチ-Database-First-はSQLをよく知っている人に適していますが、この場合はC＃をよく知る必要はありません。最初にデータベースが作成され、次にEDMXデータベースモデルが生成されます。.edmxファイル内のこのXMLには、データベースの構造、データモデル、およびそれらの相互のマッピングに関する情報が含まれています。Visual Studioには、.edmx

Model-First を使用できるグラフィックデザイナーがあり、これが3番目の ORM アプローチです。このアプローチではSQL構文もC＃構文も知ることができないため、これはアーキテクトがよく使用します。この場合、最初にグラフィカルEDMXモデルが作成され、この時点でC＃モデルクラスがバックグラウンドで作成され、次にEDMXダイアグラムに基づいてデータベースが生成されます。

エンティティフレームワークのコアモデル


Entity Frameworkでは、すべてのデータベーステーブルは、原則として、1つのテーブル（ユーザーなど）、NETの1つのクラス（ユーザーなど）の原則に従って、モデルまたはエンティティのクラスとして定義されます。このようなペアは規約と呼ばれ、データコンテキストクラスでDbSetセットとして定義されます。このアプローチはデフォルトで機能します。

Fluent APIやデータアノテーションなどのメカニズムは存在しますが、これらの規則や追加の構成ルールをオーバーライドすることが可能です。

マイグレーション


開発プロセス中に、Entity Frameworkモデルクラスが変更された可能性が非常に高いため、モデルのコンプライアンスを維持するためにデータベースを削除する必要があります。ただし、データベースを削除すると、そのデータベースのすべてのデータも削除されます。

モデルが変更されたときにデータを保存するために、Entity Framework Coreには移行機能があります。データベースにスキーマ変更を一貫して適用して、データベースをデータモデルと同期させることができます。

移行では、列とテーブル、外部キーの削除、追加、列設定の変更、データの追加、削除、変更などを実行できる操作があります。マイグレーションを作成すると、クラスが自動的に作成され、Up（）マイグレーションを適用してそれをDown（）メソッドに戻すために必要な操作が実行されます。

LINQ


LINQは、NETのEntity Frameworkと密接にリンクしています。LINQは、Language Integrated QueryまたはIntralingual Queryです。これは、構造化データベースクエリを記述できるようにするNETの一連の関数であるテクノロジです。

Entity Framework Coreでの作業には、テクノロジーLINQ to Entitiesを使用します。LINQは、SQLに似たC＃式を使用してデータベースからデータを取得します。リレーショナルデータベースはSQLクエリを通じて機能し、Entity Framework CoreはLINQ to Entities式を、使用するデータベースで理解できるSQLクエリに変換します。

結論


したがって、Entity Framework Coreの機能について簡単に説明しました。あなたが見たように、それは本当に非常に強力であり、それでそれを扱うプログラマーはSQLを知る必要すらありません。また、Entity Framework Coreは、NETの世界のORMの中で1位を占めています。
