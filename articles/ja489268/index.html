「自動テスト」の開発中に、システムからファイルをダウンロード（ダウンロード）し、その後、ファイルの内容を分析するタスクがありました。テストしたシステムからファイルを「アップロード」するには、次のソリューションを使用します。
  • 「ローカル」起動モード（作業PCで直接起動）-「ローカル」ブラウザが初期化されると、一時ローカルフォルダの名前がパラメータとして渡されます。次に、後続の分析のためにファイルがローカルフォルダーから直接読み取られます。
  • 「リモート」モードの場合（Bamboo経由）-ファイルは、ソレノイドサーバー機能を介してブラウザーからコンテナーから「取得」されました：selenoid-host.example.com：4444 /ダウンロード/ {SESSION_ID} / {FILE_NAME}


詳細はドキュメントに記載されています。

どちらのモードでも、ファイルにアクセスするには、その名前を知っている必要がありました。これは驚きでした。「ダウンロード」ファイルの名前に関するデータの欠如の問題は次のとおりです。
  • 「ファイルのダウンロード」ボタンを「クリック」した後、ブラウザーはファイルをコンテナー内の適切なローカルフォルダーにダウンロードしました。
  • この場合、ファイル名は動的に形成され、事前にそれを決定する方法は見つかりませんでした。これはテストシステムの機能でした。
  • さらに、このファイルをコンテナから「引き出し」、その内容のビジネス検証を実行する必要があります。
  • ファイルを「引き出す」には、その名前を知っている必要がありますが、名前は動的に生成され、リンクに値がなかったため、名前はわかりませんでした。


さらに、一部のテストは非常に複雑なため、テスト自体の構成とは異なる独立したテストスクリプトの一部としていくつかのファイルがテスト中にアンロードされる可能性があるという事実により、状況はさらに悪化しました。

Chromeからファイル名を取得：//ダウンロード/ページ


この方法は、ローカルモードでもリモートモードでも同じように機能します。作業のスキームは非常に簡単です：
  • javaスクリプトを実行してChromeウィンドウを開きます：//ダウンロード/;
  • 通常の方法でウィンドウ内のデータを処理します。リストの最初のファイルが読み込まれるまで待ち、その名前を確認します。
  • chromeを閉じます：//ダウンロード/ウィンドウを開き、探しているファイルの名前を返します。


Chromeを使用するという考えをグーグルで検索しました：//ダウンロード/そして、残念ながら、ソースへのリンクを提供することはできません。ファイル名を取得するためのクラスの実際の実装を以下に示します。

chrome経由でダウンロードしたファイルの名前を取得するクラス：//ダウンロード/
	private static final long DOWNLOADS_PAGE_LOAD_TIMEOUT = 5_000;
	private static final long MAX_GET_FILE_NAME_ATTEMPT = 5;
 
	public static String getLastDownloadedFilename() throws DownloaderException {
        String[] filename = new String[1];
        Exception[] ex = new Exception[1];
        WebDriver driver = WebDriverRunner.getWebDriver();
        JavascriptExecutor executor = (JavascriptExecutor) driver;
        Utils.driver.openNewTabCheckAndClose(
                () -> {
                    executor.executeScript("window.open('','_blank');");
                },
                () -> {
                    driver.get("chrome://downloads/");
 
                    for (int i = 0; i < MAX_GET_FILE_NAME_ATTEMPT; i++) {
                    	ex[0] = null;
                        sleep();
                    	try {
                        	WebElement element = (WebElement) executor.executeScript(
                                    "return document.querySelector('downloads-manager').shadowRoot.querySelector('downloads-item').shadowRoot.getElementById('file-link')");
                        	filename[0] = element.getText();
                            LOGGER.info("Attempt get file name " + i + ". Name = '" + filename[0] + "'");
                    	} catch (WebDriverException e) {
                        	ex[0] = e;
                            LOGGER.info("Failed attempt "+ i + " to get filename text: " + e.getMessage());
                        	continue;
                    	}
                    	if (filename[0] != null && !filename[0].isEmpty()) {
                        	 //             
                         	executor.executeScript(
                                    "document.querySelector('downloads-manager').shadowRoot.querySelector('downloads-item').shadowRoot.getElementById('remove').click()");
                        	break;
                    	}
                    }
                }
    	);
 
    	if (filename[0] != null && !filename[0].isEmpty()) {
            return filename[0];
    	}
 
    	String message = "Timeout. Can not get last downloaded file name from chrome://downloads/. File name is '" + filename[0] + "'. Exception: " + ex[0].getMessage();
        LOGGER.warning(message);
    	throw new DownloaderException(message, ex[0]);
	}
