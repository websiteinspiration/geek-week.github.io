Tunnel VPN IPsec IPIP entre la machine Linux et Mikrotik derrière le fournisseur NAT

Linux: Ubuntu 18.04.4 LTS (GNU / Linux 4.15.0-91-x86_64 générique)


  • Eth0 1.1.1.1/32 IP externe
  • ipip-ipsec0 192.168.0.1/30 sera notre tunnel

Miktoik: CCR 1009, RouterOS 6.46.5


  • Eth0 10.0.0.2/30 IP interne du fournisseur. Le NAT IP externe du fournisseur est dynamique.
  • ipip-ipsec0 192.168.0.2/30 sera notre tunnel

Nous allons augmenter le tunnel IPsec sur la machine Linux en utilisant racoon. Je ne décrirai pas les détails, il y a un bon article dansvvpoloskin.


Installez les packages nécessaires:


sudo install racoon ipsec-tools

Nous configurons racoon, il agira conditionnellement comme un serveur ipsec. Étant donné que mikrotik en mode principal ne peut pas transmettre un identifiant client supplémentaire et que l'adresse IP externe par laquelle il se connecte à Linux est dynamique, vous ne pouvez pas utiliser la clé pré-partagée (autorisation de mot de passe), car le mot de passe doit être mappé soit à l'adresse IP de l'hôte qui se connecte, soit à identifiant.


Nous utiliserons l'autorisation par clés RSA.
