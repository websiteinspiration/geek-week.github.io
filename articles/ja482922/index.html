モジュール


12月に登場したいくつかのモジュール：

Drupal 7


► ファイルオーガナイザー -ファイルシステムのマウントフォルダー[ルート]からフォルダー[ルート/年/月]にファイルを移動します。このモジュールは、ファイルの配置を整理したり、IMCEなどのファイルマネージャを使用してファイルを管理したりする場合に役立ちます。このモジュールは、一連のパラメーターを使用してdrush file-organizerコマンドを実装し、次のように機能します。1）file_managedテーブル内のファイルシステム（または特定のフォルダー）のルートにあるファイルを探し、2）タイムスタンプに基づいてファイルを新しいフォルダーに移動します。3）file_managedテーブルを更新します。 4）フィールドfieldbodyでファイルリンクを探して更新します。注：モジュールはfield_bodyフィールドでのみ機能します。

►TS Google Cloud Storage-Google Cloud Storageとの統合。このモジュールを使用すると、ローカルファイルシステムをGoogle Cloudストレージに置き換えることができます。これは、「ファイル」および「画像」タイプのすべてのフィールドのデフォルトファイルシステムとして使用できます。または、各フィールドを個別にGoogle Cloud Storageを使用するように設定できます。



Drupal 8


► アカマイメディアビューア -アカマイメディアビューア（AMV）画像ギャラリーとの統合。

► ライブラリをアタッチ -ライブラリをビュー表示、ブロック、ドキュメント、または用語ページにアタッチします。



► オートコンプリートエンティティID-標準のオートコンプリートフィールドの拡張機能で、タイトルだけでなく、オブジェクトの識別子を入力して関係を作成できます。



► バックアップと移行Yandexのディスク -バックアップストレージとしてYandex.Diskを使用することができますバックアップと移行モジュールの拡張。



► 基本的な透かし - 透かしの画像に追加する画像スタイルの効果。



► ボックスアクセス-box.comサービスとの統合。ユーザーは、このサービスでフォルダーを作成し、ファイルを操作できます。

► CKEditorバージョンPastefromgdocs -マークアップを保存するとCKEditorバージョンにGoogleドキュメントからの挿入データ。



► CKEditorバージョンPastefromword -マークアップを保存するとCKEditorバージョンにMicrosoft Wordから挿入データ。

► Clockify - Clockifyとの統合。

► CloudFlareのイメージスタイル -のCloudFlare CDN経由で適用されるスタイルでのアップロード画像のためにCloudFlareとの統合とCDNが使用されていないローカル環境での標準的な画像管理への復帰。

► コマースの重量別カスタム配送-Drupal Commerceモジュールの拡張。パッケージの重量に基づいて配送コストを計算できます。

► 設定バッチエクスポート -遅いVPSまたは小さなhttpタイムアウトでのホスティングで設定アーカイブを便利にダウンロードします。たとえば、サイトでCDNを使用している場合、CDNはサーバーの応答時間を30秒に制限します。多くの構成ファイルがある場合、アーカイブの生成プロセスには時間がかかり、エラーが表示されます。モジュールの利点：1）速度、2）メモリ使用量が少ない、3）応答を待つための時間制限がない、4）アーカイブの便利なダウンロード、および最初のダウンロード後のアーカイブファイルの削除によるセキュリティ。



► コンフィグスタイルシート-ブロックの可視性と同じロジックを使用してサイトのセクションに適用できるCSSの設定オブジェクト。ページに添付されているCSSは、CSSインポートリストの最後に配置されます。これにより、テーマに存在するスタイルを重ね合わせることができます。

► カスタムアカウントリンク -ユーザーモジュールは、ユーザーメニューにパス/ユーザーを含むマイアカウントリンクを作成します。このモジュールを使用すると、リンクのタイトルを上書きしたり、現在のユーザーの名前をタイトルとして使用したりできます。標準リンクをオフにして、希望のタイトルで新しいリンクを作成できます。

► メールフィールドフォーマッタ-E-mailタイプの標準フィールドのフォーマッター。これにより、次のことが可能になります。郵送先住所をmailto：リンクとしてレンダリングする;;切り捨て後、リンクを特定の数の文字と省略記号で切り捨てます。独自のテキスト、HTMLフォーマット、およびFont Awesomeアイコンを追加します。

► ブロッカーをメールで送信 -送信メールメッセージをブロックします。このモジュールは、サイトからメッセージを送信する必要がなく、送信メッセージが問題を引き起こす可能性がある開発環境で役立ちます。たとえば、サイトがcronでレターを送信し、サイトの開発バージョンから送信されるべきではない顧客に請求を送信する場合。

► エンティティのオートコンプリート階層 - オートコンプリートのあるフィールドに階層を表示します。分類用語はデフォルトでサポートされています。



►エンティティテーマエンジン -サイトインターフェイスを介したTwigテンプレートの編集。 block_contentやノードを含むすべてのコンテンツオブジェクトがサポートされます。



► エンティティ使用状況ビューフィールド -参照しているオブジェクトの数を示すビューフィールド。



► エンティティの訪問者 -ページの訪問者を表示します。モジュールは3つのブロックで配布されます：1）ユーザーがプロファイルブロックを表示（プロファイルを入力したユーザーのリスト）、2）ユーザーがページブロックを表示（ページを訪問したユーザーのリスト）、3）ユーザーがノードブロックを表示（ドキュメントを入力したユーザーのリスト） 。パラメータを使用すると、誰かが自分のプロファイルを見に来た場合のユーザーのアラートを設定できます。



► 電子出版ビューア-標準タイプの「ファイル」のフィールドフォーマッター。訪問者がWebサイト上の本を.epub形式で読むことができます。



► ステータスレポートの公開-/ admin / reports / statusページのデータをJSONファイルに公開します。複数のサイトを管理していて、このモジュールによって表示されるデータを1か所に収集することにより、それらのステータスの追跡を設定する場合は、モジュールが役立ちます。

► Fakelink -パス＃との虚偽のリンクを作成します。レコードを使用してリンクパスを示し、リンクのhrefは＃に変換されます。



► 柔軟なビュー-いくつかの新しい機能をビューに追加して、ビューの人間工学を改善：ビュー形式プラグイン：フレキシブルテーブル、公開フィルター：表示列セレクター、公開フォームスタイル：手動選択。柔軟なテーブルと列選択プラグインを使用して、ユーザーはフォームに表示するデータと非表示にする必要があるデータを選択できます。



► グーグルWebfontsヘルパー - Googleのフォントサービスとの統合。このモジュールでは、ローカルで使用するためにGoogleサービスからフォントをダウンロードすることをお勧めします。必要なファイルをダウンロードし、それらに関連付けられたライブラリを生成します。したがって、サイトのフォントをサーバーまたはCDNからビジターに配信できるため、ビジター側のサイトのロードが高速化されます。



►HTML インポーター -サーバー上のHTMLファイルからドキュメントを送信します。



►Infogramチャート -Infogramサービスとの統合。このモジュールを使用すると、標準のサービスコードとテキストフィルターを介してサイトにInfogramグラフィックを埋め込むことができます。



► 国際電話 -intl-tel-inputライブラリに基づく国際電話番号コードの入力と検証。



► 軽量グループ-ユーザーグループおよびコンテンツとしての分類用語の使用（このモジュールは、Groupsモジュールの簡易バージョンに似ています）。アイデアは次のとおりです。1）ユーザー接続があり、用語分類法がある場合、これは実際にはユーザーのグループです。 2）コンテンツと用語分類法の間に関連がある場合、これはコンテンツのグループです。 3）一致する場合、このコンテンツはこのユーザーグループに属しています。モジュールは、グループの作業のためのさまざまなパラメーターを提供します。

► ログイン警告-アカウントログインフォームが表示される前に警告/通知メッセージを表示します。ログインフォームを表示するには、訪問者は[OK]をクリックしてメッセージを受け入れる必要があります。通常、このようなメッセージは、サイトの使用条件についてユーザーに通知し、ユーザーの同意を得るために使用されます。

► 数学操作は -モジュールは、引数を入力するための2つのテキストフィールドが含まれていると、あなたが入力したデータと算術演算子に基づいて合計値を計算することができますサイトへのフィールドの新しいタイプを追加（+、 - 、*、/）。

►Microsoft Azure Blob Storage - Microsoft Azure Blob Storageとの統合。

► モーダルブロック-モーダルウィンドウでのブロックの表示。モジュールには、表示前の遅延、リフレッシュレート、ブラウザでタブを閉じようとしたときの表示など、カスタマイズ可能な多数のパラメータがあります。



►D8のモジュールリスト -サイトにインストールされているモジュールを表形式で示し、カスタムモジュールに分割され、drupal.orgモジュールからダウンロードされます。



► NfP365 CRM API - Microsoft DynamicsのNPOのための365と統合するためのAPI。

► ノードリンクレポート-モジュールは、このドキュメントで使用されているすべてのリンクに関するレポートが表示されるブロックを生成します。ドキュメントのコンテンツ内のリンクは、テキストエディターのあるフィールド、リンクのフィールド、テキストフィールド、リンクのフィールドなどのさまざまな場所に配置できるため、検出および検証が困難になるため、便利です。このモジュールを使用すると、コンテンツで使用されているすべてのリンクが機能していることを確認できます。

► 通知バー -サイトにカスタムアラートを表示します。通知は、モジュール設定ページでオン/オフを切り替えることができます。また、モジュールによって生成される通知バーブロックを使用して表示することもできます。



► ページ固有のクラス -サイトページのパスに基づいて、本文のHTML要素にクラスを追加します。



►PHPランタイムメモリ -php.iniファイルを変更せずにPHP環境の構成を変更します。モジュールパラメータを使用すると、ファイルアップロード制限、メモリ制限、最大実行時間制限、入力変数制限などのPHPパラメータを設定できます。



► ノード削除の防止 -別のドキュメントに接続されているとき、または通信フィールドを介してオブジェクトで使用されているときのドキュメントの削除を防止および制限します。

► セキュリティの質問-Drupal 8-モジュールは、プロファイル、サイトでの登録プロセス、およびパスワード回復プロセスにセキュリティの質問を追加します。

► 送信して続行 -フォームにボタンを追加してフォームデータを送信し、ユーザーを同じフォームにリダイレクトして、新しいアクションを実行できるようにします。

► 分類用語集-分類法の辞書を用語集として使用。モジュールの機能：用語または文字で検索するためのブロックの生成（簡単な用語集）、コンテンツのスキャンとタグ付け（フィールドtext_with_summary、text_long、string_long、text、entity_reference、entity_reference_revisionsはデフォルトでスキャンされます）、JavaScriptを使用して、モーダルウィンドウに用語を表示します。
