Javaでフローを制御するために例外の使用を避ける理由

Javaは、特定のタスクのための多くの代替ソリューションを備えた汎用プログラミング言語です。それにもかかわらず、従わなければならない優れたアプローチがあり、私たちがまだほとんどの部分で使用しているいくつかの失敗したアプローチもあります。

最も一般的な失敗したアプローチの1つは、例外を使用して実行のフローを制御することです。これは、次の2つの理由で回避する必要があります。

  1. これにより、コードのパフォーマンスと速度が低下します。
  2. これにより、コードが読みにくくなります。

例を見てみましょう。ここでは、例外を使用して実行のフローを制御します。

   public static int findAge(String name) {
       try {
           String ageAsString = findUser(name);
           return ageAsString.length();
       } catch (NameNotFoundException e) {
           return 0;
       }
   }
   private static String findUser(String name) {
       if(name==null) {
           throw new NameNotFoundException();
       }
       return name;
   }

ユーザーが名前にゼロ以外の値を指定した場合、findAgeメソッドはこの名前の長さを返しますが、ユーザー名がnullの場合、findUserメソッドはNameNotFoundExceptionをスローし、この場合、findAgeメソッドは0を返し

ます。例外を使用せずにこのコードを変換するにはどうすればよいですか？一般に、多くのオプションがあります。次のいずれかを検討してください。

public static int findAgeNoEx(String name) {
       String ageAsString = findUserNoEx(name);
       return ageAsString.length();
   }
   private static String findUserNoEx(String name) {
       if(name==null) {
           return "";
       }
       return name;
   }

例外がパフォーマンスに及ぼす影響を確認するために、例外のある場合とない場合の両方のメソッドを1000万回呼び出す次のコードを準備しました。 

public class ControlFlowWithExceptionOrNot {
   public static class NameNotFoundException extends RuntimeException {
       private static final long serialVersionUID = 3L;
   }
   private static final int TRIAL = 10000000;
   public static void main(String[] args) throws InterruptedException {
       long start = System.currentTimeMillis();
       for (int i = 0; i < TRIAL; i++) {
           findAgeNoEx(null);
       }
       System.out.println("Duration :" + (System.currentTimeMillis() - start));
       long start2 = System.currentTimeMillis();
       for (int i = 0; i < TRIAL; i++) {
           findAge(null);
       }
       System.out.println("Duration :" + (System.currentTimeMillis() - start2));
   };
   public static int findAge(String name) {
       try {
           String ageAsString = findUser(name);
           return ageAsString.length();
       } catch (NameNotFoundException e) {
           return 0;
       }
   }
   private static String findUser(String name) {
       if (name == null) {
           throw new NameNotFoundException();
       }
       return name;
   }
   public static int findAgeNoEx(String name) {
       String ageAsString = findUserNoEx(name);
       return ageAsString.length();
   }
   private static String findUserNoEx(String name) {
       if (name == null) {
           return "";
       }
       return name;
   }
}

結局のところ、実際に例外が発生して必要な場合に例外を使用する方がよいでしょう。例外を使用して実行のフローを制御すると、プログラムのパフォーマンスが低下し、コードが読みにくくなります。
