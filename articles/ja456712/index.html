SOAPサービスの選択的なトラフィックロギング

カスタマーサービスと統合するプロジェクトのほとんどで、SOAPサービスが使用されています。そして、そのようなすべてのプロジェクトには、情報を記録するタスクがあります。私たちは特に、ソリューションの業務に関連するリクエストのロギングに関心があります。たとえば、従業員が誤ったデータを入力し、サービスがエラーを返しました。このエラーの詳細を知り、できるだけ早く修正するか、お客様との話し合いのために送信します。

発生した問題を整理するのに役立ちますので、クライアントのリクエストとサービスの回答は変更しないでください。

この記事では、SOAPサービスの選択的なトラフィックロギングを設定する方法を示します。



問題


まず、Springには、構成を通じて有効になる組み込みの要求と応答のロギングがあります。

logging.level.org.springframework.ws.client.MessageTracing.sent=TRACE
logging.level.org.springframework.ws.client.MessageTracing.received=TRACE

問題は、これらの設定にすべてのSOAPトラフィックのロギングが含まれていることです。必要なメソッドはほんの数個で、完全ではありません。たとえば、これは大量のデータであるため、ログのファイルをダウンロードするリクエストを表示したくありません。

Spring Frameworkでは、クライアントソープを構築するための事実上の標準は、WebServiceGatewaySupportを使用することです。この場合、ClientInterceptorを介して要求と応答の処理を追加することもできます。しかし、彼のメソッドでは、クライアントのどのメソッドが呼び出しを開始したかを理解することは困難です。また、要求のみ、応答のみ、またはすべてを一度に記録する必要があるかどうかは明確ではありません。

決定


標準のClientInterceptorを使用しますが、メソッドシグネチャに関する不足している情報と、リクエストとレスポンスをログに記録するかどうかを決定できる「入力」、「出力」フラグを提供します。

そのようなソリューションはストリーミングシリアライゼーション/デシリアライゼーションでは機能しないことをすぐに予約します。しかし、デフォルトで「ペイロードキャッシュ」オプションが有効になっているAxiomを使用しており、このソリューションは機能します。

情報を格納するための構造：

import lombok.Data;
@Data
public class SoapLoggingInfo {
    private String method;
    private boolean input = false;
    private boolean output = false;
}

情報を保存するためのコンテキスト：

public class SoapLoggingContext {
    private static final ThreadLocal<SoapLoggingInfo> holder = new ThreadLocal<>();
    public static void set(SoapLoggingInfo value) {
        holder.set(value);
    }
    public static SoapLoggingInfo get() {
        return holder.get();
    }
}

コンテキスト内に情報を配置するために、注釈付きメソッドの「スライス」といくつかの「ヒント」を伴うAOPアプローチを使用します。メソッドが呼び出される前と後です。

メソッドの注釈：

@Target({ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Component
public @interface SoapLoggable {
    boolean value() default true;
    boolean input() default true;
    boolean output() default true;
}

面自体は猫の下に置かれました。

側面
@Aspect
@Component
@Slf4j
public class SoapLoggingAspect {

    @Value("${logging.soap.request.enabled:false}")
    private Boolean enabled;


    @Pointcut("execution(@ru.trueengineering.agentapi.logging.SoapLoggable * *(..))")
    public void soapLoggableMethod() {}

    @Before("soapLoggableMethod()")
    public void beforeSoapLoggable(JoinPoint joinPoint) {
        if (!enabled) {
            return;
        }
        SoapLoggable soapRequestLogger = getAnnotation(joinPoint);
        if (soapRequestLogger.value()) {
            SoapLoggingInfo loggingInfo = new SoapLoggingInfo();
            loggingInfo.setInput(soapRequestLogger.input());
            loggingInfo.setOutput(soapRequestLogger.output());
            final Class<?> aClass = joinPoint.getTarget().getClass();
            final Signature signature = joinPoint.getSignature();
            final String name = signature.getName();
            loggingInfo.setMethod(aClass.getSimpleName() + "." + name);
            SoapLoggingContext.set(loggingInfo);
        }
    }

    @After("soapLoggableMethod()")
    public void afterSoapLoggable(JoinPoint joinPoint) {
        SoapLoggingContext.set(null);
    }

    private SoapLoggable getAnnotation(JoinPoint joinPoint) {
        MethodSignature signature = (MethodSignature) joinPoint.getSignature();
        Method method = signature.getMethod();
        return  method.getAnnotation(SoapLoggable.class);
    }

}


分解してみましょう


スライスは、「SoapLogableによって注釈が付けられたすべてのメソッド」を意味する式です。AspectJの機能を使用します。

@Pointcut("execution(@ru.trueengineering.agentapi.logging.SoapLoggable * *(..))")
public void soapLoggableMethod() {}

スライスに該当するメソッドの前に呼び出されるアドバイスがあります。

@Before("soapLoggableMethod()")
public void beforeSoapLoggable(JoinPoint joinPoint) {}

このメソッドでは、アノテーションを取得し、メソッドシグネチャとメタ情報をアノテーションから抽出し、ClientInterceptorのオブジェクトを形成して、コンテキストに配置します。

スライスに該当するメソッドを呼び出した後に呼び出されるヒントがあります。

@After("soapLoggableMethod()")
public void afterSoapLoggable(JoinPoint joinPoint) {}

コンテキストをクリアするだけです。実際、猫の下でログを記録します：

ロギング
@Component
@Slf4j
public class SoapLoggingInterceptor extends ClientInterceptorAdapter {
    @Override
    public boolean handleRequest(MessageContext messageContext) throws WebServiceClientException {
        SoapLoggingInfo info = SoapLoggingContext.get();
        if (info != null && info.isInput()) {
            ByteArrayOutputStream xml = new ByteArrayOutputStream();
            try {
                messageContext.getRequest().writeTo(xml);
                log.debug(": " + info.getMethod() + ", :" + xml.toString(StandardCharsets.UTF_8));
            } catch (IOException e) {
                log.error("  SOAP request", e);
            }
        }
        return true;
    }

    @Override
    public boolean handleResponse(MessageContext messageContext) throws WebServiceClientException {
        return handleResponseOrFault(messageContext);
    }

    @Override
    public boolean handleFault(MessageContext messageContext) throws WebServiceClientException {
        return handleResponseOrFault(messageContext);
    }

    private boolean handleResponseOrFault(MessageContext messageContext) {
        SoapLoggingInfo info = SoapLoggingContext.get();
        if (info != null && info.isOutput()) {
            ByteArrayOutputStream xml = new ByteArrayOutputStream();
            try {
                messageContext.getResponse().writeTo(xml);
                log.debug(": " + info.getMethod() + ", :" + xml.toString(StandardCharsets.UTF_8));
            } catch (IOException e) {
                log.error("  SOAP response", e);
            }
        }
        return true;
    }
}


SOAPリクエストを「インターセプト」して処理する標準的な方法を使用しますが、コンテキストからの情報を使用して、必要なメソッドのみを選択的にログに記録します。
