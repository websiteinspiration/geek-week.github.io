上記の例では、フォームコンポーネントの「input」プロパティのデータを「text」変数に記録しました。今度は、これらをグリーティングコンポーネントとログアウトコンポーネントのプロパティに表示し、users_array配列に新しいコンテナーを作成する必要があります。カスタムイベントが必要な理由を理解するために、まずこれを使用せずにさまざまな方法でこれを実行してみましょう。



1.方法-フォームのクリックメソッド自体で、各コンポーネントに直接移動し、textプロパティを設定します。 




        event.preventDefault();

このアプローチの欠点は明らかです。このプロパティのデータをどこかで取得するとどうなるでしょうか。たとえば、localstorageに基づいて、それらは自動的にロードされます。

この場合、コードの重複を回避することに成功しましたが、問題は、コンポーネントのプロパティを操作するメソッドがその内部ではなく、外部関数にあることです。プロパティを設定する前に、コンポーネントごとに異なる方法でプロパティをフォーマットする必要がある場合、または1つのプロパティではなく複数のプロパティを設定する必要がある場合は、コンポーネント自体がプロパティを処理する場合に適しています。



3.方法は、プロパティを操作するすべてのロジックをコンポーネントに転送し、特定のアクションを呼び出すメソッドをコンポーネント内に作成することです。

これでコンポーネントがカプセル化されましたが、コンポーネントには名前でアクセスします。後でコンポーネントまたはメソッドの名前を変更する場合、またはコンポーネントを完全に削除する場合、または同じプロパティをリッスンするそのようなコンポーネントが多数ある場合はどうなるでしょうか。または、そのようなプロパティを多数持つため、そのようなメソッドを多数作成する必要があります。



このタスクを容易にするために、カスタムイベントを使用できます。コンポーネントはイベントをサブスクライブし、発生すると、そのプロパティを使用して何かを行います。



カスタムイベントを使用して、コードを編集してみましょう。 



var StateMap = {

   eventEmiters: {	
             //   -  	
             ["emiter-entry-user"]: {prop: ""}, 		
   },
   form: {
	container: "form",
	props: ["input", "click"],
	methods: {
			
		click: function(){
		    event.preventDefault();
		    var text = this.parent.props.input.getProp();

		     //        
		    this.rootLink.eventProps["emiter-entry-user"].setEventProp(text); 	
		    window.localStorage.setItem('user_name', text);							
		}			
	},		
},	
greeting: {
		
	container: "greeting",
	props: [ "user_name", ['listen_entry_user', "emiter-entry-user", "" ] ], //   "emiter-entry-user"
	methods: {
			
		listen_entry_user: function(){
		   //      
		   this.parent.props.user_name.setProp( this.emiter.getEventProp() );
		},	
	},		
},
logout: { 
		
	container: "logout",
	props: [ "user_name", ["listen_entry_user", "emiter-entry-user", "" ]], //   "emiter-entry-user"
	methods: {
			
		listen_entry_user: function(){
		     //       
		     this.parent.props.user_name.setProp( this.emiter.getEventProp() );
		},			
	},	
},
    //      , ..            
users_array: { 
		
	arrayProps: [ ['listen_entry_user', "emiter-entry-user", ""] ], //   "emiter-entry-user"
	   arrayMethods: {
			
		listen_entry_user: function(){
			//       
			this.parent.add( {user_name: this.emiter.getEventProp()} );
		},			
	},		
	container: "user",
	props: [ "user_name",   ],
	methods: {						
		}
   },        	


}
window.onload = function(){
	
		
	var HM = new HTMLixState(StateMap);
	var name = window.localStorage.getItem('user_name');
	
	if(name != null)HM.eventProps["emiter-entry-user"].setEventProp(name); ///  "emiter-entry-user"        	
}


したがって、中間メソッド「entryUser」を削除し、新しいデータをフォームに渡し、ページが読み込まれた後にイベントを発生させます。これで、コンポーネントの何かを変更したり、削除したりする必要がある場合、各関数のコードを修正する必要はありません。これは任意の数にすることができます。また、イベントのサブスクライブを解除するには、コンポーネントのリスナーを削除するか、プロパティ-subscriberのdisableEvent（）メソッドを使用してリスナーを一時的に無効にします。
