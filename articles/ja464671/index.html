ViberおよびTelegramのインスタントメッセンジャーに定期的にSMSを受信します（GoIPゲートウェイを使用）

企業の通信にIPテレフォニーを使用し、GSMゲートウェイを使用する多くの企業では、番号からSMSを送受信するという疑問がよく出てきます。

中国のGOIPモデルの場合、優れたGOIP-SMS-Serverソリューションがあり、インターネットやHabrコミュニティでも繰り返し言及されています。これはシンプルで無料で便利な製品ですが、特にモバイルからのWEB経由のSMSはあまり便利ではありません。

メッセンジャーを使用してSMSを送受信するというアイデアがあり、シンプルなPython3.6スクリプトとTelegramボットを使用して1時間で実装されました（2か月後にViberに転送されました）

ロジックは次のとおりです：リクエストを処理して解析し、S​​MSの本文を取得してボットに送信し、個人のチャットに送信します-SMSを取得します。このトピックは、SIMが1つあるiPhoneの所有者にとって特に興味深いものです。オペレーターの補充：

したがって、最初の部分、つまり、テレグラムでGSMネットワークから直接本格的なSMSを受信する方法を検討します。

  • GoIP SMSサーバー
  • ボットを登録します（すべてのボットのお父さん@BotFatherからトークンを取得します）
  • Gmailに新しいメールボックスを登録し、IMAP受信を有効にする

スクリプトの例を分析してみましょう。最初に、受信者の電子メールアドレスをSMSサーバーに追加します（いくつかの電子メールアドレスはコンマで区切ることができます）。次

mail.login('yougmail@gmail.com', 'gmailpassword') #   @gmail
mail.list() #     
mail.select('INBOX') #    
result, data = mail.uid('search', None, 'UNSEEN') #  

i = len(data[0].split())
for x in range(i):
 latest_email_uid = data[0].split()[x]
