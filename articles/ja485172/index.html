大規模プロジェクトにおけるアーキテクチャの問題

モバイルアプリの開発は、かなり単純な作業のようです。そこで何をすればいいのでしょうか？私はいくつかのビューを投げ、いくつかのアーキテクチャに油をそそぎました、そしてそれで終わりです。プロジェクトの準備ができました。アプリケーションをステーションに送信できます。一連の記事で、大規模銀行向けのアプリケーションを開発するときに出会った機能を紹介します。

典型的なケースは、新しいCoordinatorの初期化とクロージャーfinishFlowの実装です。キャプチャweak coordinatorは必須です。それ以外の場合、コーディネーターはそれ自体を参照し、AuthCoordinatorの形式でリークが発生します。


    func runAuthFlow() {
        let coordinator = AuthCoordinator(router: MainRouter())
        coordinator.finishFlow = { [weak self, weak coordinator] in
            self?.removeDependency(coordinator)
        }
        self.addDependency(coordinator)
        coordinator.start()
    }
