Bit of theory



All of the following will apply to convolutional neural networks ‚Äî a subtype of neural networks used to work with images. First, let's look at how a neural network works to solve the superresolution problem. The task is very similar to the solution of the auto encoder problem. It is necessary to submit an image to the network input, to get the same image at the output. However, autoencoders are usually used to solve the problem of efficient data compression, therefore, a feature of their architecture is Bottleneck - a bottleneck, that is, a network layer with a small number of neurons. The presence of such a layer makes the remaining parts learn for efficient coding and decoding of information. To train superresolution networks, the resolution of high-quality images is first intentionally reduced and fed to the input of the neural network. The output is expected source image in high quality.



The superresolution task defines the architecture of the networks used:

usually there is a connection between input and output data (skip-connection), which greatly accelerates learning. The pixel size of the input data increases and is added to the output of the convolutional network. Thus, in fact, you do not need to train the network to turn the image into almost the same. You just need to train her to draw up the difference between the enlarged by increasing the size of the pixel and the real image. The idea of ‚Äã‚Äãhaving skip connections at various levels and through a different number of layers is extremely effective and has led to the emergence of the Residual Networks class of networks. Nowadays, such connections are used in almost all popular architectures. A good overview of state-of-art architectures for solving superresolution problems can be found here.. My task was to create a neural network to solve the superresolution problem in real time.



First, I chose the edsr architecture with 4 blocks (usually more than 16 layers of blocks are used) ResNet with a 4-fold increase in resolution. Here I partially took advantage of the developments from github.com/krasserm/super-resolution and the data generator from the same project.



The general network architecture is shown in the diagram. Each block is an X * Y * N image, where the width corresponds to the number of channels. Transitions - correspond to 3x3 convolutions (in the case of res blocks followed by activation of ReLU for non-linearity). Upscaling step - increase of dimension due to flattening of channels.







If you look closely at such a small network, you will notice that the last blocks are linear and the presence of several upscaling steps is unjustified. You can replace them with one layer with much fewer filters without losing quality without losing network quality:







It turned out that such a network copes with the superresolution task almost the same way, but much faster and can be launched to process small real-time video by resolution .



High-quality photos have been downloaded from data.vision.ee.ethz.ch/cvl/DIV2K . In principle, you can use any photo for training. 



import tensorflow as tf
from model import resolve_single
from model. rtsr import rtsr
import numpy as np
from utils import load_image, plot_sample
from data import DIV2K
from tensorflow.keras.optimizers.schedules import PiecewiseConstantDecay
# Model creation
rtsr_model= rtsr(scale=4, num_res_blocks=4)
#Data loading and generator creation
train_loader = DIV2K(scale=4,             
                     downgrade='bicubic',
                     subset='train')      
                     


train_ds = train_loader.dataset(batch_size=16,
                                random_transform=True,
                                repeat_count=None)  
valid_loader = DIV2K(scale=4,  
                     downgrade='bicubic',
                     subset='valid')  
                     

valid_ds = valid_loader.dataset(batch_size=16,           # use batch size of 1 as DIV2K images have different size
                                random_transform=True, # use DIV2K images in original size
                                repeat_count=40)         # 1 epoch
#Saving checkpoints during training
cbcks=tf.keras.callbacks.ModelCheckpoint('callback_x4{epoch:02d}-{loss:.2f}', monitor='loss', verbose=0, save_best_only=False)

learning_rate=PiecewiseConstantDecay(boundaries=[20000], values=[1e-4, 5e-5])
#model compilation before training
rtsr_model.compile(loss='mae', optimizer=tf.keras.optimizers.Adam(learning_rate=learning_rate))
#Training
rtsr_model.fit(train_ds,validation_data=valid_ds, steps_per_epoch=3000, epochs=100, validation_steps=10,callbacks=[cbcks])
#saving weights
rtsr_model.save_weights('weights/ myedsr-lastdiv2_4block-x4_weights.h5')


The output is a relatively small (2.6 MB) neural network with a simple architecture. At the same time, the check gives a subtle difference from the pre-trained 16 block network:







The image on the left is the original one, on the right is improved using the 16-layer edsr, in the middle - using rtsr. The difference between 2 and 3 is not significant from my point of view.



Interference



github.com/Alexankharin/RTSR We will



launch the resulting network on a video card (I have GTX 1060) with cudnn support (https://developer.nvidia.com/cudnn) for high performance.



Pipeline for inference is as follows:



  1. Capturing an image from an area or window
  2. Image enhancement
  3. Rendering enhanced image in a new window


During testing, I found that when most games are launched in the dosbox emulator, the resolution is 640x480 pixels, but most often these are just 320x240 resolution pixels (and later found details at www.dosgamers.com/dos/dosbox-dos-emulator/screen- resolution ), which leads to the need in some cases to do downscale 2 times before processing.

Capture screenshots using the mss library (for OS ubuntu) or d3dshot (faster for windows).



Mapping - using opencv-python. To close the window, make it active and press the q key. Controlling the capture of the area to be improved using the WSAD and IKJL keys.



The script is written in the superres_win.py file.

How to start?



To work quickly, you need a video card with support for cuda and cudnn (https://developer.nvidia.com/cuda-gpus) and the installed cuda / cudnn libraries. You need installed python 3.7 and tensorflow (version higher than 2.0 with gpu support). This can be a daunting task and compatibility issues may arise .



The easiest way would be to install the Anaconda distribution (https://www.anaconda.com/products/individual) and then to install tensorflow-gpu using conda: 



conda install tensorflow-gpu


If it doesn‚Äôt work out due to conflicts, then you can try 



conda install cudnn
pip install tensorflow-gpu


should work.



Other libraries can be installed using pip: 



pip install opencv-python
pip install pywin32
pip install mss


Next, you need to run the script with the command: 



python superres.py


The capture window is controlled using the wsad and ijkl keys (resize). q- closing the window. 0 - turn on / off superresolution mode. Numbers 1 and 2 - image mode (2 by default means that the game uses an increase due to an increase in the pixel by 2 times).
