Nouvelles fonctionnalités de Python 3.9

Un examen des meilleures fonctionnalités incluses dans la dernière itération Python.



Le moment est venu, la sortie d'une nouvelle version de Python est inévitable. Il est actuellement en version bêta (3.9.0b3), mais nous verrons bientôt la version complète de Python 3.9.

Certaines des dernières fonctionnalités sont incroyablement intéressantes, et il sera étonnant de voir leur utilisation après leur sortie. Nous tiendrons compte des éléments suivants:

  • Opérateurs de combinaison de dictionnaires
  • Indication de type
  • Deux nouvelles méthodes de chaîne
  • Le nouvel analyseur Python est très cool

Voyons d'abord les nouvelles fonctionnalités et comment nous les utiliserons.

Combinaison de dictionnaires


L'une des nouvelles fonctionnalités et déjà ma préférée avec la syntaxe. Si nous avons deux dictionnaires aet bque nous devons combiner, nous utilisons maintenant les opérateurs d'union.

Nous avons l'opérateur de fusion « |»:

a = {1: 'a', 2: 'b', 3: 'c'}
b = {4: 'd', 5: 'e'}
c = a | b
print(c)

[Out]: {1: 'a', 2: 'b', 3: 'c', 4: 'd', 5: 'e'}

Et l'instruction de mise à jour « |=», qui met à jour le dictionnaire source:

a = {1: 'a', 2: 'b', 3: 'c'}
b = {4: 'd', 5: 'e'}
a |= b
print(a)

[Out]: {1: 'a', 2: 'b', 3: 'c', 4: 'd', 5: 'e'}

Si nos dictionnaires ont une clé commune , une paire clé-valeur du deuxième dictionnaire sera utilisée:

a = {1: 'a', 2: 'b', 3: 'c', 6: 'in both'}
b = {4: 'd', 5: 'e', 6: 'but different'}
print(a | b)

[Out]: {1: 'a', 2: 'b', 3: 'c', 6: 'but different', 4: 'd', 5: 'e'}

Mise à jour du dictionnaire à l'aide d'itérations


Un autre comportement intéressant de l'opérateur « |=» est la possibilité de mettre à jour le dictionnaire avec de nouvelles paires clé-valeur en utilisant un objet itératif - par exemple, une liste ou un générateur:

a = {'a': 'one', 'b': 'two'}
b = ((i, i**2) for i in range(3))
a |= b
print(a)

[Out]: {'a': 'one', 'b': 'two', 0: 0, 1: 1, 2: 4}

Si nous essayons de répéter la même chose avec l'opérateur d'union standard « |», nous obtenons TypeError, car il n'autorisera que les jointures entre les types dict.
