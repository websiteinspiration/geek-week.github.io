HTMLとCSSを作成するときに開発者が犯す最も一般的な6つの間違い

こんにちは、ハブル！Stas Melnikovによる「HTMLとCSSを書くときに開発者が犯す最も一般的な6つの間違い」という記事の翻訳を紹介します。

label要素の代わりにplaceholder属性を使用する


多くの場合、開発者はlabel要素の代わりにplaceholder属性を使用します。ただし、この場合、スクリーンリーダーはプレースホルダー属性からテキストを読み取ることができないため、スクリーンリーダーのユーザー（スクリーンリーダー）はフィールドに入力できません。

<input type="email" placeholder="Enter your email">

したがって、名前フィールドにはlabel要素を使用し、ユーザーが入力する必要のあるデータなどのプレースホルダー属性を使用することをお勧めします。

<label>
  <span>Enter your email</span>
  <input type="email" placeholder="e.g. example@gmail.com">
</label>

img要素を使用して装飾的なグラフィックをレイアウトする


開発者が装飾的なグラフィックとコンテンツ画像を混同しているのをよく見ます。たとえば、img要素を使用してソーシャルアイコンをマークアップします。

<a href="https://twitter.com" class="social">
  <img class="social__icon" src="twitter.svg" alt>
  <span class="social__name">Twitter</span>
</a>

しかし、ソーシャルネットワークのアイコンは、ユーザーがテキストを読まなくても要素の意味をすばやく理解できるようにする装飾的なアイコンです。アイコンを削除しても、要素の値は失われないため、background-imageプロパティを使用できます。

<a href="https://twitter.com" class="social">
  <span class="social__name">Twitter</span>
</a>

.social::before {
  background-image: url("twitter.svg");
}

resizeプロパティの使用


resizeプロパティを使用してテキスト領域のサイズ変更を無効にすると、アクセシビリティが低下します。したがって、ユーザーは便利な方法でデータを入力できません。

textarea {
  width: 100%;
  height: 200px;
  resize: none;
}

要素のサイズを制限するプロパティmin-width、max-width、min-height、max-heightを使用でき、ユーザーは便利な方法でフィールドに入力できます。

textarea {
  min-width: 100%;
  max-width: 100%;
  min-height: 200px;
  max-height: 400px;
}

表示：ブロックと位置：絶対（固定）を一緒に使用。


開発者が表示と位置のプロパティを次のように使用するのをよく見ます。

.button::before {
  content: "";
  display: block;
  position: absolute;
  top: 0;
  left: 0;
}

ただし、ブラウザはデフォルトのブロック値を設定します。したがって、絶対位置または固定位置の要素にこの値を追加する必要はありません。したがって、次のコードは前のコードと同じ結果になります。

.button::before {
  content: "";
  position: absolute;
  top: 0;
  left: 0;
}

構造プロパティの値がありません


キーボードを使用してWebサイトを操作できません。リンクをたどることができません。登録できません。これは、アウトラインプロパティに何も追加しないと、開発者が要素へのフォーカスをオフにするためです。

.button:focus {
  outline: none;
}

/* or */

.button:focus {
  outline: 0;
}

デフォルトでフォーカスをオフにする必要がある場合は、必ず代替フォーカスの状態にしてください。

.button:focus {
  outline: none;
  box-shadow: 0 0 3px 0 blue;
}

空のアイテム


多くの場合、開発者は空のHTML要素を使用して要素のスタイルを設定します。たとえば、空のdivまたはspan要素を使用してハンバーガーメニューをマークアップします。

<button class="hamburger">
  <span></span>
  <span></span>
  <span></span>
</button>

.hamburger {
  width: 60px;
  height: 45px;
  position: relative;
}

.hamburger span {
  width: 100%;
  height: 9px;

  background-color: #d3531a;
  border-radius: 9px;

  position: absolute;
  left: 0;
}

.hamburger span:nth-child(1) {
  top: 0;
}

.hamburger span:nth-child(2) {
  top: 18px;
}

.hamburger span:nth-child(3) {
  top: 36px;
}

ただし、疑似要素:: beforeおよび:: afterを使用して、同様の結果を得ることができます。

<button class="hamburger">
  <span class="hamburger__text">
    <span class="visually-hidden">Open menu</span>
  </span>
</button>

.hamburger {
  width: 60px;
  height: 45px;
  position: relative;
}

.hamburger::before,
.hamburger::after,
.hamburger__text::before {
  content: "";
  width: 100%;
  height: 9px;

  background-color: #d3531a;
  border-radius: 9px;

  position: absolute;
  left: 0;
}

.hamburger::before {
  top: 0;
}

.hamburger::after {
  top: 18px;
}

.hamburger__text::before {
  top: 36px;
}

.visually-hidden {
  position: absolute !important;
  clip: rect(1px, 1px, 1px, 1px);
  width: 1px !important; 
  height: 1px !important; 
  overflow: hidden;
}
