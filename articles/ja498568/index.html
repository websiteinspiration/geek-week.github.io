マルチチップセクショナルマイクロプロセッサー


複数のプロセッサチップがどのように連携し、任意の長さの単語をサポートできるか疑問に思われるかもしれません。要するに、マイクロプロセッサセクション（MS）はプロセッサ全体ではなくビルディングブロックであり、命令をデコードしてシステムを制御するために個別の回路が必要です。 MSにはレジスタがあり、データを使って算術演算と論理演算を実行し、制御チップ（Am2901など）がMSに何をすべきかを伝えました。各機械語命令は、ROMマイクロコードに保存されたマイクロ命令である小さなステップに分割されました。この場合、命令セットはAm2901ではなくマイクロコードによって決定されたため、ほとんどすべての命令セットをサポートできます。

そのようなプロセッサのセクションは互いに完全に独立しているわけではないため、プロセッサの動作中に特定の問題が発生します。たとえば、2つの数値を加算する場合、1つのセクションからの転送を別のセクションに転送する必要があります。また、いくつかのセクションの共同作業には、符号の確認やゼロの結果の確認などの操作が必要です。 Am2901チップには、これらの機能をサポートする特別な出力があります。

Am2901はマイクロプロセッサですか？私の観点から見ると、Am2901はプロセッサの一部にすぎませんが、すべては「マイクロプロセッサ」を決定する方法に依存します（これについての私の考えを別の記事で詳しく説明しました））興味深いことに、ソビエト連邦では、米国よりもセクションマイクロプロセッサに偏っていました。そして西側で「マイクロプロセッサ」という言葉が通常シングルチップ上のプロセッサを意味する場合、シングルチップ上のUSSRプロセッサまたは複数のセクションからのプロセッサは通常区別されませんでした。

マルチチップセクショナルマイクロプロセッサー（MSM）は、マイクロプロセッサーチップと単純なTTLで作られたコンピューターの中間にありました。チップ。当時、TTLチップからコンピューターを組み立てるのは、マイクロプロセッサーを作るよりもはるかに高速でしたが、これにはチップを搭載した多くのボードが必要でした。MSMを使用すると、使用するチップの数を減らしながら、速度の利点を維持できます。また、MSMはマイクロプロセッサに比べて柔軟性が高く、設計者は一連の命令やその他のアーキテクチャ機能をカスタマイズできます。

プレビュー付きの転送では、GenerateおよびPropagate信号を使用して、各位置のビットが独自の転送を生成するか、着信転送を渡すかを決定します。たとえば、0 + 0 + C（Cは転送）を追加すると、サイズに関係なく、この追加から転送を削除することはできません。一方、1 + 1 + Cを追加すると、Cに関係なく転送が表示されます。最後に、0 + 1 + C（または1 + 0 + C）の場合、転送はさらに転送されますCがゼロ以外の場合。その結果、単純な論理ゲートは、両方のビットが1に等しい場合は各ビットのG（生成）信号を作成し、両方のビットが0に等しくない場合はP（伝播）信号を作成します。

転送式はビットの場所によって異なります。たとえば、ビット0からビット1への転送を考えてみます。これは、P0フラグが設定されている（つまり、転送が開始または送信された）場合、および転送がこのビットに現れた場合、または他のビットから転送された場合に起こります。次に、C1 = P0 AND（Cin OR G0）。高次のハイフネーションでは、オプションの数が増えており、その複雑さは絶えず増加しています。たとえば、ビット2への転送を考えます。最初に、転送がビット1から行われるようにP1をコッキングする必要があります。さらに、転送はビット1によって作成されたか、ビット0から転送されました。最後に、最初の転送もどこかから行われなければなりませんでしたそれを取りなさい：それはビット0から来た転送、またはビット1によって生成された転送のいずれかでした。これをすべて式に入れると、Am2901で使用される関数が得られます：C2 = P1 AND（G1 OR P0）AND（C0 OR G0 OR G1）。さまざまな転送と外部PおよびGの式は、仕様、図 9.

論理スキームは、非常に異なる方法で作成できます。最近のほとんどすべてのコンピュータは、CMOSロジックシステム（相補型金属酸化膜半導体構造）を使用しており、バルブはMOSトランジスタで構成されています。ミニコンピュータの時代には、TTLが非常に人気がありました。 ESLはより高速でしたが、あまり一般的ではありませんでした。 ESLの欠点は、エネルギー消費量が多かったことです（1985年のCray-2スーパーコンピューターは速度を上げるためにESLバルブを使用していましたが、液体フレオンで冷却する必要がありました）。

速度におけるESLの利点のほとんどは、トランジスタが完全にオンになっていなかったためです。これにより、トランジスタは非常に迅速に電流経路を変更できました。さらに、値0と1の電圧の差は小さく（0.8 Vのオーダー）、信号は非常に高速に切り替えられました。たとえば、TTLバルブでは、電圧差は約3.2 Vです（信号はナノ秒あたり約1 Vの速度で切り替わるため、電圧差が大きい場合、数ナノ秒の遅延があります）。一方、小さな電圧差により、ESLの電気ノイズに対する感度が向上しました。

Am2901の最初のバージョンはTTLを使用していましたが、1979年にAMDはより高速なバージョンのAm2901Cを発表しました。 Am2901Cは内部でESLを使用して速度を上げていましたが、外側ではTTL電圧を維持していたため、TTLコンピューターで簡単に使用できました。この投稿では、Am2901Cバリアントについて説明します。

結論


Am2901Cチップは、比較的まれな論理ファミリである高速ESLの例として注目されています。ALUチップはチップの下部に配置され、8つの異なる機能を実装し、プレビュー転送を使用して作業を高速化します。チップは非常に複雑ですが、顕微鏡で注意深く調べると、その動作を理解するのに役立ちます。

Am2901などのマルチチップセクショナルプロセッサは、1970年代と1980年代にマイクロコンピュータや他の多くのシステムで使用されていました。しかし、結局のところ、CMOSテクノロジーの改善により、シングルチップに高速プロセッサを実装できるようになり、このテクノロジーは時代遅れになりました。そして、Am2901はおそらく約1000のトランジスタを含み、16 MHzで動作しますが、現在AMDは数十億のトランジスタを含み、4 GHzで動作するプロセッサを製造しています。
