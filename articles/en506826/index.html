We make a scintillation spectrometer ourselves from ... a radiometer

In the spring I was quarantined and a little time appeared to solder something interesting. The choice fell on this device .

Of course, I wanted to expand the capabilities of the radiometer and find out what specific radioactive isotopes lie in wait in my everyday life in Kiev, which is very close to the ChEZ.



In the article I will tell how I assembled the device and what I changed in circuitry and firmware.

The first thing to do is order the boards. In the materials for the original article there are gerber files, so everything is simple. The order was made on PCBWay and JLCPCB to compare quality. I can‚Äôt recommend the first one: the delivery took 3 months, the mounting holes on the boards were less than necessary. Of the 5 motherboards, 2 turned out to be defective (which they reported to me in the letter). With JLCPCB everything turned out well and there is nothing to complain about.

I ordered components on Mouser and sets of capacitors and resistors on Ali (laziness began to select everything by capacity and decided to just order a set). As SiPM used MicroFC 60035 - this is the most expensive part of the device. At the time of ordering, it cost $ 70 on a Mouser. With a smaller and cheaper 30035, he decided not to mess with it, afraid that it would be more difficult to solder and assemble it.

The second main component of the device, in addition to the photodetector, is a scintillation crystal. And here is a big field for modifications. I could not find the CsI (Tl) used by the author of small sizes for less than $ 90. Therefore, I stopped at NaI (Tl) 10x40mm c ebeya for 32 dollars with delivery. The search for such a crystal is an exciting activity in itself, the main thing here is not to rush. All Google searches led me to Eugene from Ukraine, but he simply did not have transparent crystals for spectrometry. Everything he sent had an unpleasant yellow hue of urine.

And now, all the parts and boards came, you can begin to solder. The first thing I decided was to solder the analog board. There is no adventure here, the main thing is not to forget to solder the resistor, the place for which is not divorced (carefully read the assembly tips for the original article).

The following changes had to be made to the system board: According to the LM2733Y datasheet, the output voltage does not depend on the input voltage, so there is nothing to adjust. We take the formula R1 = R2 X (VOUT / 1.23 - 1) from the same datasheet and from what was found, set R13 = 1.8K, R12 = 12K, R11 = 300K. The output is stably 28.18V (I tried to supply 2.5V, 5V - everything is stable at the output). After connecting the display, the device began to give out a much larger number of pulses than it actually is. We managed to fix it by changing the display power circuit: we connect the input of the DA6 switch to the input of DA3. At the output of DA6, we set the converter to 5V (I had the pololu cj7032 at my fingertips) and already feed the display from it. With this connection, all interference immediately went away.

I took nx4024t032 as the display: it is smaller, cheaper, consumes less, and most importantly, it was available in the local store. I still planned to change the firmware, more on that below.

After soldering, I washed the circuit with isopropyl alcohol in an ultrasonic bath. After alcohol, it is worth washing with distilled water in the same bath and drying in the oven at a temperature of about 70-80 degrees.

Now it's time to do the most interesting: connect the sensor and see what happens. MicroFC 60035 is almost perfectly soldered to a piece of the 3x3 hole breadboard: we tin the corner holes and solder the sensor with a hairdryer. Solder the wires from the back of the breadboard. The time has come for the most interesting part - changes in the firmware. But we want to make a scintillation detector, and not just a radiometer. To do this, we need to use DMA with ADC (there is only one ADC in this processor, but there is an input switch). And we have two inputs: SP and battery voltage. DMA is needed to speed up the whole process. I also want to pay attention to the number of measurement cycles ADC_SampleTime, with 48 or more, I did not succeed. 4 cycles showed the most stable result.

Change the ADC initialization code as follows:

void initADC(void) {
	/* PWR_CTRL and CHG_STAT clock enable */
	RCC_AHBPeriphClockCmd(RCC_AHBPeriph_GPIOB, ENABLE);

	/* UBAT input pin configuration */
	GPIO_InitStructure.GPIO_Pin = GPIO_Pin_12|GPIO_Pin_15;
	GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AN;
	GPIO_InitStructure.GPIO_PuPd = GPIO_PuPd_NOPULL;
	GPIO_InitStructure.GPIO_Speed = GPIO_Speed_40MHz;
	GPIO_Init(GPIOB, &GPIO_InitStructure);

	/*------------------------ DMA1 configuration ------------------------------*/
	/* Enable DMA1 clock */
	RCC_AHBPeriphClockCmd(RCC_AHBPeriph_DMA1, ENABLE);
	/* DMA1 channel1 configuration */
	DMA_DeInit(DMA1_Channel1);
	DMA_InitStructure.DMA_PeripheralBaseAddr = ADC1_DR_ADDRESS;
	DMA_InitStructure.DMA_MemoryBaseAddr = (uint32_t)&ADC_ConvertedValue[0];
	DMA_InitStructure.DMA_DIR = DMA_DIR_PeripheralSRC;
	DMA_InitStructure.DMA_BufferSize = 2;
	DMA_InitStructure.DMA_PeripheralInc = DMA_PeripheralInc_Disable;
	DMA_InitStructure.DMA_MemoryInc = DMA_MemoryInc_Enable;
	DMA_InitStructure.DMA_PeripheralDataSize = DMA_PeripheralDataSize_HalfWord;
	DMA_InitStructure.DMA_MemoryDataSize = DMA_MemoryDataSize_HalfWord;
	DMA_InitStructure.DMA_Mode = DMA_Mode_Circular;
	DMA_InitStructure.DMA_Priority = DMA_Priority_High;
	DMA_InitStructure.DMA_M2M = DMA_M2M_Disable;
	DMA_Init(DMA1_Channel1, &DMA_InitStructure);
	/* Enable DMA1 channel1 */
	DMA_Cmd(DMA1_Channel1, ENABLE);
	/*----------------- ADC1 configuration with DMA enabled --------------------*/
	/* Enable The HSI (16Mhz) */
	RCC_HSICmd(ENABLE);
	/* Check that HSI oscillator is ready */
	while
	(!RCC_GetFlagStatus(RCC_FLAG_HSIRDY));
	/* Enable ADC1 clock */
	RCC_APB2PeriphClockCmd(RCC_APB2Periph_ADC1, ENABLE);
	/* ADC1 Configuration -----------------------------------------------------*/
	ADC_InitStructure.ADC_Resolution = ADC_Resolution_12b;
	ADC_InitStructure.ADC_ScanConvMode = ENABLE;
	ADC_InitStructure.ADC_ContinuousConvMode = DISABLE;
	ADC_InitStructure.ADC_ExternalTrigConvEdge = ADC_ExternalTrigConvEdge_None;
	ADC_InitStructure.ADC_ExternalTrigConv = 0;
	ADC_InitStructure.ADC_DataAlign = ADC_DataAlign_Right;
	ADC_InitStructure.ADC_NbrOfConversion = 2;
	ADC_Init(ADC1, &ADC_InitStructure);
	/* Enable temperature sensor and Vref */
	ADC_TempSensorVrefintCmd(ENABLE);
	//ADC_TempSensorVrefintCmd(DISABLE);
	/* ADC1 regular channel configuration */
	ADC_RegularChannelConfig(ADC1, SP_ADC_CHANNEL, 1, ADC_SampleTime_4Cycles);
	ADC_RegularChannelConfig(ADC1, UBAT_ADC_CHANNEL, 2, ADC_SampleTime_4Cycles);

	/* Enable the request after last transfer for DMA Circular mode */
	ADC_DMARequestAfterLastTransferCmd(ADC1, ENABLE); /* Causes problem.. */
	/* Define delay between ADC1 conversions */
	ADC_DelaySelectionConfig(ADC1, ADC_DelayLength_None);
	/* Enable ADC1 Power Down during Delay */
	ADC_PowerDownCmd(ADC1, ADC_PowerDown_Idle_Delay, ENABLE);
	/* Enable ADC1 DMA */
	ADC_DMACmd(ADC1, ENABLE);
	/* Enable ADC1 */
	ADC_Cmd(ADC1, ENABLE);

	while(!ADC_GetFlagStatus(ADC1, ADC_FLAG_ADONS));

	ADC_SoftwareStartConv(ADC1);
}

Here we start the measurement, increase the pulse counter and save the result of the measurement in an array for further processing and display.

In addition, in the firmware I changed the buttons: up / down changes the brightness of the display, the menu button shows the spectrum, the last button turns the sound on and off. We have only two screens: the main search screen with a graph of the intensity of the counting pulses and a blank screen for displaying the spectrogram. We immediately display the spectrogram on a linear and logarithmic scale, it‚Äôs more convenient to watch.
