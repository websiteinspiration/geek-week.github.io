現実的な苛性反射


ほとんどのテクニカルアーティストは、キャリアのある時点で、コースティクスのもっともらしい反射を作成しようとします。ゲーム開発者の場合、Twitterを読む主な理由の1つは、Twitterから引き出せるインスピレーションの無限のストリームです。数日前、Florian Gelzenlichter（twitterのkolyaTQ）が、シェーダーを使用してUnityで作成したコースティック効果のGIFを投稿しました。投稿（下に表示）はすぐに1.5のいいね！を獲得しました。これは、このタイプのコンテンツに対する誠実な関心を示しています。


私は通常、より長く技術的に複雑な一連の記事（たとえば、光の体積大気散乱 [ Habréの翻訳 ]と逆運動学 [ Habréの翻訳の最初と2番目の部分] ）に惹かれますが、短くてかわいいチュートリアルを書く誘惑に抵抗できませんでしたフロリアンの影響について。

この記事の最後に、Unityパッケージと必要なすべてのアセットをダウンロードするためのリンクがあります。

苛性アルカリとは


コースティクスの 概念を認識していない場合がありますが、この影響は毎日発生します。コースティックスは、曲面によって引き起こされる光の反射です。一般的なケースでは、曲面はレンズのように振る舞い、いくつかの点で光を集め、他の点で散乱します。そのような効果を提供する最も一般的な媒体は、ガラスと水であり、いわゆる苛性波を生成します（以下を参照）。


コースティクスは他の形式をとることができます。たとえば、レインボーは、雨滴で光が屈折するときに発生する光学現象です。したがって、厳密には苛性アルカリです。

効果の解剖学


苛性波の認識できる特徴は、それが動く方法です。プールの底を見れば、おそらく彼に会ったでしょう。多くの光線のシミュレーションを必要とするため、実際のコースティクスの再作成は非常にコストがかかります。

フロリアンは、1つのコースティックテクスチャから始めて、もっともらしい効果を生み出しました。チュートリアルを作成するために、以下に示すOpenGameArtから取得したテクスチャを使用しました。


この効果を実現するための重要な特性は、上記のコースティックパターンがシームレスであることです。これは、2つの画像を隣り合わせに配置でき、それらの間に目立つ継ぎ目がないことを意味します。大きな表面にこの効果を使用したいので、錯覚を破壊する可能性のある涙なしでこのテクスチャを引き伸ばす機会があることが重要です。

テクスチャを受け取ったら、Florianは3つのステップを実行することを提案します。

  • コースティックパターンをモデルの表面に2回適用し、毎回異なるサイズと速度を使用します
  • 演算子で2つのパターンを混ぜる min
  • サンプリング時にRGBチャネルを分離します。

Unityの各ステップを実装する方法を見てみましょう。

シェーダー作成


最初のステップは、新しいシェーダーを作成することです。この効果は、実際の照明も持つ3Dゲームで使用される可能性が高いため、サーフェスシェーダーから始めるのが最善です。サーフェスシェーダーは、Unityでサポートされている多くのタイプのシェーダーの1つです（消灯しているマテリアルの頂点シェーダーとフラグメントシェーダー、ポストプロセッシングエフェクトのスクリーンシェーダー、オフスクリーンシミュレーションの計算シェーダーなど）。

新しいサーフェスシェーダーには、いくつかの機能しかありません。このエフェクトを作成するには、情報をシェーダーに転送する必要があります。1つはコースティックテクスチャです。第二に、これはそれをスケーリングおよびオフセットするために使用されるパラメーターです。

サンプリングテクスチャ


各サーフェスシェーダーには、一般にと呼ばれる関数が含まれ、surfレンダリングされた各ピクセルの色を決定するために使用されます。「標準」関数surfは次のようになります。

void surf (Input IN, inout SurfaceOutputStandard o)
{
    // Albedo comes from a texture tinted by color
    fixed4 c = tex2D (_MainTex, IN.uv_MainTex) * _Color;
    o.Albedo = c.rgb;
    // Metallic and smoothness come from slider variables
    o.Metallic = _Metallic;
    o.Smoothness = _Glossiness;
    o.Alpha = c.a;
}

最終的な色は、シェーダーが初期化してと呼ばれる構造体で返す必要があるいくつかのフィールドによって決定されSurfaceOutputStandardます。Albedo白色光で照らされたオブジェクトの色にほぼ一致するように変更する必要があります。

新しく作成されたサーフェイスシェーダーでは、アルベドはというテクスチャから取得され_MainTexます。コースティックス効果は既存のテクスチャの上に重ねられるため、でテクスチャの追加のサンプリングを実行する必要があります_CausticsTex。UVオーバーレイ

と呼ばれる手法により、レンダリングする必要があるジオメトリの部分に応じて、テクスチャのどの部分をサンプリングする必要があるかを理解できます。これは、3Dモデルの各頂点に格納され、テクスチャの座標を示す変数を使用して行われます。uv_MainTexfloat2

私たちのアイデアは_Caustics_ST、スケーリングとシフトuv_MainTexにコースティクスを使用して、モデル全体でテクスチャをストレッチおよび移動することです。

void surf (Input IN, inout SurfaceOutputStandard o)
{
    // Albedo comes from a texture tinted by color
    fixed4 c = tex2D (_MainTex, IN.uv_MainTex) * _Color;
    o.Albedo = c.rgb;

    // Caustics sampling
    fixed2 uv = IN.uv_MainTex * _Caustics_ST.xy + _Caustics_ST.zw;
    fixed3 caustics = tex2D(_CausticsTex, uv).rgb;

    // Add
    o.Albedo.rgb += caustics;

    // Metallic and smoothness come from slider variables
    o.Metallic = _Metallic;
    o.Smoothness = _Glossiness;
    o.Alpha = c.a;
}

アニメーションされたコースティクス


コースティックスの最も重要な機能の1つは、その動き方です。現時点では、それらはモデルの表面に2番目のテクスチャとして単に静的に投影されています。

シェーダー内のマテリアルのアニメーションは、Unity提供のプロパティを使用して実装できます_Time。現在のゲーム時間にアクセスするために使用できます。つまり、方程式に時間を追加します。

最も簡単な方法は、現在の時間に基づいてテクスチャを単にオフセットすることです。

// Caustics UV
fixed2 uv = IN.uv_MainTex * _Caustics_ST.xy + _Caustics_ST.zw;
uv += _CausticsSpeed * _Time.y;

// Sampling
fixed3 caustics = tex2D(_CausticsTex, uv).rgb;

// Add
o.Albedo.rgb += caustics;

このフィールドに_Time.yは、現在の再生時間が秒単位で表示されます。反射の動きが速すぎる場合は、係数を掛けることができます。このために、上記のコードでは_CausticsSpeed型変数が使用されていますfloat2。

目的に応じて、正弦波のコースティックテクスチャを振動させる必要がある場合があります。ここでは、効果を実現する標準的な方法がないことを理解することが重要です。必要に応じて、コースティクス反射をまったく異なる方法で動かすことができます。

以下に示す結果は、まだかなり平凡です。これは正常です。反射を美しく見せるためには、まだやらなければならないことがたくさんあります。


複数サンプリング


コースティックステクスチャを1回ではなく2回サンプリングすると、効果が活きてきます。それらを互いに重ねて、異なる速度で移動すると、結果は完全に異なります。

まず第一に、我々は、プロパティを複製_Caustics_STし、_CausticsSpeed2つのテクスチャのサンプルは、異なるスケール、変位と速度を持っているので、こと：

[Header(Caustics)]
_CausticsTex("Caustics (RGB)", 2D) = "white" {}
		
// Tiling X, Tiling Y, Offset X, Offset Y
_Caustics1_ST("Caustics 1 ST", Vector) = (1,1,0,0)
_Caustics2_ST("Caustics 1 ST", Vector) = (1,1,0,0)

// Speed X, Speed Y
_Caustics1_Speed("Caustics 1 Speed", Vector) = (1, 1, 0 ,0)
_Caustics2_Speed("Caustics 2 Speed", Vector) = (1, 1, 0 ,0)

これで2つの苛性アルカリサンプルができたので、演算子を使用してそれらを混合できますmin。平均値だけをとると、結果はあまり良くありません。

// Caustics samplings
fixed3 caustics1 = ...
fixed3 caustics2 = ...

// Blend
o.Albedo.rgb += min(caustics1, caustics2);

このような小さな変更により、大きな違いが生まれます。


コードを美しく保つために、コースティックサンプリングコードを独自の関数でラップすることもできます。

// Caustics
fixed3 c1 = causticsSample(_CausticsTex, IN.uv_MainTex, _Caustics1_ST, _Caustics1_Speed);
fixed3 c2 = causticsSample(_CausticsTex, IN.uv_MainTex, _Caustics2_ST, _Caustics2_Speed);

o.Albedo.rgb += min(c1, c2);

RGB分離


コースティクス反射を美しく見せるために、最後のトリックを行う必要があります。スライスを通過すると、異なる波長の光は異なる方法で屈折します。これは、水の中を移動するときに、ライトがさまざまな色に「分割」できることを意味します。

この効果をシミュレートするために、各苛性アルカリサンプルを3つに分割できます（各カラーチャネルに1つ）。赤、緑、青のチャネルをわずかにバイアスしてサンプリングすると、色の不一致が発生します。まず、溶解効果の強さを示す

プロパティ_SplitRGBを追加します。

// Caustics UV
fixed2 uv = IN.uv_MainTex * _Caustics_ST.xy + _Caustics_ST.zw;
uv += _CausticsSpeed * _Time.y;

// RGB split
fixed s = _SplitRGB;
fixed r = tex2D(tex, uv + fixed2(+s, +s)).r;
fixed g = tex2D(tex, uv + fixed2(+s, -s)).g;
fixed b = tex2D(tex, uv + fixed2(-s, -s)).b;

fixed3 caustics = fixed3(r, g, b);

RGBチャネルオフセットは任意に選択できますが、この単純なオフセットを使用しても、非常に説得力のある画像が得られます。
