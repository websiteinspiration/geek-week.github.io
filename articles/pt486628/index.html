Mas a questão com a distribuição desigual de dados ao longo do eixo permaneceu aberta.
A decisão foi formada no processo de vagar por fóruns e artigos - numere as linhas .

A opção mais fácil é numerar manualmente.

A segunda opção é numerar automaticamente:

 nl -v 0 -s ';' -w 1 data.csv >> data_nums.csv

Se você precisar excluir linhas comentadas, o comando assumirá o formato:

grep -v "^#" data.csv | nl -v 0 -s ';' -w 1  >> data_nums.csv

Ao trabalhar com esse conjunto de dados, o gnuplot distribuirá os dados ao longo do eixo em intervalos regulares:



Script de resumo:

#!/usr/bin/gnuplot -persist

set encoding utf8
set terminal pngcairo size 640,400 enhanced monochrome
set output "img.png"

set ylabel "Y"
set xlabel "X"
set key left top samplen 5 spacing 1.15 width 2
set xrange [-0.5:24.5]
set mytics 2
set xtics rotate by -60
set grid
set pointsize 0.5
set datafile separator ";"
set style line 1 linetype 1 pointtype 7 linewidth 2 linecolor black
set style line 2 linetype 2 pointtype 7 linewidth 2 linecolor black

Label(String,Size) = sprintf("{/=%d %s}", Size, String)

plot "data_nums.csv" using 1:4:xtic(3) title "1" w lp ls 1 ,\
     "data_nums.csv" using 1:5 title "2" w lp ls 2 ,\
     "data_nums.csv" using 1:($4>$5?$4+20:$5+20):(Label(stringcolumn(2), 10)) with labels notitle


update:
Graças à atenção plenaDeosise uma referência exata à documentação (parágrafo Pseudocolunas ), descobriu-se que você pode fazer isso sem numeração artificial.
Para distribuição uniforme dos dados ao longo do eixo, você pode usar a coluna nula ($ 0), que fornece índices de linha.

Nesse caso, é possível usar o conjunto de dados original - data.csv e fazer as seguintes alterações no script:
plot "data.csv" using 0:3:xtic(2) title "1" w lp ls 1 ,\
     "data.csv" using 0:4 title "2" w lp ls 2 ,\
     "data.csv" using 0:($3>$4?$3+20:$4+20):(Label(stringcolumn(2), 12)) with labels notitle
