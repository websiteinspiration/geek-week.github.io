Why does a list in a tuple behave strangely in Python?

In programming languages, I have always been interested in their internal structure. How does one or another operator work? Why is it better to write like this and not otherwise? Such questions do not always help to solve the problem "here and now", but in the long run they form the overall picture of the programming language. Today I want to share the result of one of these dives and answer the question of what happens when modifying tuple'a in list' e.

We all know that Python has a data type list:

a = []
a.append(2)

listIs just an array. It allows you to add, delete and modify items. It also supports many different interesting operators. For example, the + = operator to add items to list. +=changes the current list, but does not create a new one. This is clearly visible here:

>>> a = [1,2]
>>> id(a)
4543025032
>>> a += [3,4]
>>> id(a)
4543025032

Python has another great data type: tuple- an immutable collection. It does not allow adding, deleting or changing elements:

>>> a = (1,2)
>>> a[1] = 3
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

When using the operator +=, a new one is created tuple:

>>> a = (1,2)
>>> id(a)
4536192840
>>> a += (3,4)
>>> id(a)
4542883144

Attention, the question is: what will the following code do?

a = (1,2,[3,4])
a[2] += [4,5]

Options:

  1. Items are added to the list.
  2. The tuple immutability exception will be thrown.
  3. Both.
  4. Neither one nor the other.

Write your answer on a piece of paper and let's do a little check:

>>> a = (1,2,[3,4])
>>> a[2] += [4,5]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

Well then! So we figured it out! The correct answer is 2. Although, wait a moment:

>>> a
(1, 2, [3, 4, 4, 5])

In fact, the correct answer is 3. That is, the elements were added, and the exception came out - wat ?!


Let's see why this happens. And a wonderful module will help us with this dis:

import dis

def foo():
    a = (1,2,[3,4])
    a[2] += [4,5]

dis.dis(foo)

        9 LOAD_CONST      4 (4)
       12 BUILD_LIST      2
       15 BUILD_TUPLE     3
       18 STORE_FAST      0 (a)

  3    21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)
       27 DUP_TOP_TWO

The first block is responsible for building tuple'a and storing it in a variable a. Then the fun begins:

       21 LOAD_FAST       0 (a)
       24 LOAD_CONST      2 (2)

We load the pointer to the variable aand constant 2 onto the stack .

       27 DUP_TOP_TWO

Duplicate them and put them on the stack in the same order.

       28 BINARY_SUBSCR

This operator takes the top element of the stack (TOS) and the next one (TOS1). And writes a new element to the top of the stack TOS = TOS1[TOS]. So we remove the top two values ‚Äã‚Äãfrom the stack and put in it a link to the second element tuple'a (our array).

       29 LOAD_CONST      4 (4)
       32 LOAD_CONST      5 (5)
       35 BUILD_LIST      2

We build a list of elements 4 and 5 and put it on the top of the stack:

       38 INPLACE_ADD

Apply +=to the top two elements of the stack (Important! These are two lists! One consists of 4 and 5, and the other is taken from tuple). Everything is fine here, the instruction is executed without errors. Since it +=modifies the original list, the list in tuple'e has already changed (at that very moment).

       39 ROT_THREE
       40 STORE_SUBSCR

Here we swap the top three elements of the stack (lives there tuple, in it the index of the array and the new array) and write the new array in tupleby index. This is where the exception occurs!

Well, that‚Äôs it! In fact, the list can be changed, but everything falls on the operator =.

Let‚Äôs finally figure out how to rewrite this code without exception. As we already understood, you just need to remove the entry in tuple. Here are a couple of options:

>>> a = (1,2,[3,4])
>>> b = a[2]
>>> b += [4,5]
>>> a
(1, 2, [3, 4, 4, 5])

>>> a = (1,2,[3,4])
>>> a[2].extend([4,5])
>>> a
(1, 2, [3, 4, 4, 5])
