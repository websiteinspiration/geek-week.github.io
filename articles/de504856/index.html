1. Oracle bietet eine

Cloud der zweiten Generation Die Cloud-Infrastruktur von Oracle bietet Tools und Architekturen, mit denen Unternehmen nahtlos von lokal in die Cloud migrieren können. Dabei werden fortschrittliche Automatisierung und integrierte Sicherheit verwendet, um Bedrohungen zu minimieren und letztendlich eine überlegene Migration und Kosteneffizienz zu unterstützen.

Oracle Cloud-Infrastruktur für Innovation konzipiert. Dies umfasst branchenführende Skalierbarkeit und Verfügbarkeit, integrierte Verwaltung und Kontrolle sowie Zuverlässigkeit, die durch End-to-End-SLAs unterstützt wird. Die Cloud-Mission von Oracle umfasst die Unterstützung neuer Technologien wie künstliche Intelligenz, maschinelles Lernen, Internet der Dinge, Blockchain und menschenorientierte Schnittstellen.

Oracle-Unternehmenskunden haben sich vom Experimentieren mit diesen Technologien zum Einsatz für unternehmenskritische Anwendungen, zum Aufbau neuer Geschäftsmodelle und zur Schaffung neuer Geschäftswerte entwickelt.

Oracle Cloud Infrastructure adressiert die wichtigsten Herausforderungen von Cloud-Lösungen der ersten Generation, die nicht für die Arbeit mit großen Finanzsystemen ausgelegt sind, die von Workloads oder Anwendungen angetrieben werden, die große Datenmengen erfordern. Cloud-Lösungen der ersten Generation basieren auf Technologien aus einem Jahrzehnt zuvor, in denen die Einstellungen für Leistung, Sicherheit und Migration immer zu spät waren. Die Oracle Cloud Infrastructure der nächsten Generation wurde speziell für die Anforderungen heutiger Unternehmen entwickelt und bietet eine schnellere und vorhersehbarere Leistung, einen besseren Preis und eine bessere Sicherheit sowie eine verbesserte Kompatibilität mit Unternehmens-Workloads.

Oracle ist der einzige Anbieter von IaaS-, PaaS- und SaaS-Diensten im Rahmen seines Cloud-Angebots der zweiten Generation. Eigenständige Oracle-Datenbankdienste verwenden dasselbe Hochgeschwindigkeitsnetzwerk wie andere Dienste, sodass Sie schnell geschäftskritische Anwendungen bereitstellen können, um kontinuierliche Innovationen zu unterstützen.

2.Oracle Cloud unterstützt Ihre langfristigen Pläne

Oracle Cloud Infrastructure ist eine grundlegend neue Open Cloud-Architektur der zweiten Generation, die als Grundlage für Oracle Cloud dient. Diese Infrastruktur wurde speziell entwickelt, um vorhersehbare Leistung, Sicherheit vom Kern des Systems bis zur Spitze und die zur Unterstützung kritischer und intensiver Workloads erforderlichen Kontrollen bereitzustellen.

Oracle Cloud bietet Funktionen und Technologien für Computer, Data Warehousing, Datenbanken, Netzwerk- und Plattformdienste, die für zuverlässige Geschäftsergebnisse erforderlich sind, sodass Sie die Anforderungen des Rechenzentrums überdenken können.

Die Oracle Cloud-Infrastruktur übertrifft AWS bei der End-to-End-Workload- Leistung ( Andrew Reichman, „Oracle testet eine bessere Leistung als Amazon Web Services“, Oracle, 18. September 2018 ).

3. Möglichkeiten zur Leistungsoptimierung

Oracle IaaS bietet eine breite Palette von Funktionen, die in der Branche ihresgleichen suchen - von einer Plattform der zweiten Generation und einer Reihe von Diensten für Plattformen ohne Betriebssystem (Bare Metal) bis hin zu RDMA (Remote Direct Memory Access) für technische Computercluster. Diese Unterscheidung stellt sicher, dass die Oracle-Cloud-Infrastruktur sowohl eine vorhersehbare Leistung als auch eine gute Isolation der Cloud-Clients aufweist.
Die Cloud-Infrastruktur von Oracle öffnet auch die Tür zu innovativen Innovationen wie Oracle Autonomous Data Warehouse und Oracle Autonomous Transaction Processing.

Eigenständige Workloads funktionieren am besten in einer Infrastruktur, die für geringe Latenz, hohe Verfügbarkeit, Fehlertoleranz und konsistente Leistung ausgelegt ist. Dies bedeutet, dass Ihre Daten im Oracle-Objektspeicher gespeichert werden und mehrere Kopien automatisch repliziert werden, wodurch eine hohe Verfügbarkeit und Fehlertoleranz gewährleistet wird. Die in sich geschlossenen Selbstheilungsfunktionen von Oracle stellen sicher, dass Ihre Daten "fehlerfrei" bleiben und Benutzer immer auf das neueste bekannte Replikat zugreifen können.

4. Tiefenschutz

Sicherheit ist ein zentrales Designprinzip innerhalb der Oracle Cloud-Infrastruktur. Oracle bietet Schutz vom Kern des Systems bis zu seinen neuesten Technologien wie Client-Isolation, Datensicherheit, Erkennung interner Bedrohungen und deren hochautomatisierte Beseitigung. Oracle Infrastructure isoliert Computer- und Netzwerkressourcen, um Ihre persönlichen Daten und den Datenverkehr vor anderen Benutzern zu schützen. Außerdem werden Code, Daten und Ressourcen von Kontrollmaschinen getrennt, sodass Angreifer keine Daten in der Cloud stehlen oder manipulieren können.

Oracle Cloud bietet nur sehr begrenzte Angriffsmöglichkeiten durch granulare Client-Isolation. Sicherheitsstufen mit integrierten Firewalls, DDoS-Angriffsdetektoren und Verschlüsselung erkennen und stoppen Bedrohungen proaktiv. Kunden können eine Identität auf dem neuen Perimeter festlegen und mithilfe der adaptiven Authentifizierung automatisch zusätzliche Überprüfungen hinzufügen, wenn Benutzeraktivitäten auf ein höheres Risiko hinweisen. Schließlich verfügt Oracle über eines der branchenweit breitesten Portfolios an Cloud- und lokalen Lösungen.

5. Optimierung für Workloads von Oracle

Oracle Cloud Infrastructure bietet verschiedene einzigartige Funktionen und Tools, die die effiziente Migration und den Start des Oracle-Portfolios an Datenbank- und Geschäftsanwendungen unterstützen. Das Verschieben von Oracle-Anwendungen erfordert nur minimale Änderungen, wodurch sich die Kosten und die Dauer der Migration in die Cloud verringern. Die Priorität von Oracle besteht darin, die neuesten Geräte und Technologien bereitzustellen, um die Produktivität der Kunden kontinuierlich zu steigern.

Oracle-Datenbanken lassen sich in Bezug auf Speicher und Leistung der Wettbewerber um ein Vielfaches skalieren und erreichen eine Kapazität von bis zu 40 Terabyte und Millionen von E / A-Vorgängen pro Sekunde (IOPS) pro Instanz. Wenn die maximale Anzahl von E / A-Vorgängen erforderlich ist, um die komplexesten autonomen Datenbank-Workloads auszuführen, bietet Oracle die branchenweit größte Menge an lokalem nichtflüchtigem All-Flash-Express-Speicher (NVMe) (bis zu 51,2 TB pro Instanz). Durch das Starten von Oracle RAC-Clustern (Oracle Real Applications Clusters) in der Oracle Cloud Infrastructure werden HA-Datenbanken (High Availability) mit Failover in Sekundenschnelle bereitgestellt, die Leistung auf Hunderttausende von E / A skaliert und reibungslose Vorgänge basierend auf fortlaufenden Patches und Updates durchgeführt.Diese Technologie führt einen neuen Cloud-Standard für Produktionsdatenbankanwendungen ein.

Mit Oracle Exadata Cloud können Kunden Exadata-Formfaktor-Computer in denselben erweiterten Regionen und virtuellen Cloud-Netzwerken ausführen wie Bare-Metal-Computing und andere Oracle-Cloud-Infrastrukturdienste, die auf demselben System wie das vollständige System oder auf dem gesamten Rack verwaltet werden. Management-Toolkit und Zugriff über dieselbe Konsole / API.

Die Cloud-Infrastruktur von Oracle bietet eine Reihe von Migrations-, Bereitstellungs- und Verwaltungstools für wichtige Oracle-Anwendungen, einschließlich Oracle E-Business Suite und Oracle PeopleSoft, Siebel und JD Edwards, sowie Anpassungs- und zugehörige Datenbanken, mit denen Kunden den Übergang in die Cloud beschleunigen können. Insbesondere für eine eigenständige Datenbank optimiert die Cloud-Infrastruktur von Oracle die Transaktionsanalyse und die Workloads für die Transaktionsverarbeitung. Es wurde speziell für die Ausführung von ML-Workloads entwickelt, um Informationen basierend auf künstlicher Intelligenz bereitzustellen. Dadurch kann der Client mehrere Datenquellen verbinden und Schlüsseldaten kombinieren, um fundierte Geschäftsentscheidungen zu unterstützen. Die Cloud-Infrastruktur von Oracle ist auch die Grundlage für neue erweiterte Dienste, einschließlich Integration, Container, Microservices und Blockchain.und bietet verbesserte Skalierbarkeit, um die Bereitstellung des Internet der Dinge zu unterstützen.

6. Volle Unterstützung für die Anwendungsentwicklung nutzen

Oracle ist schnell innovativ und bietet eine offene, standardbasierte und vollständig integrierte Anwendungsentwicklungsplattform, mit der Sie moderne Cloud-basierte Anwendungen auf Basis von APIs und Mobilgeräten kostengünstig erstellen, bereitstellen und verwalten können. Darüber hinaus bietet Oracle Unterstützung für die Container- und Cloud-Entwicklung mit einer geringen Menge an modifizierbarem Code.

Die Oracle AppDev-Plattform bietet eine umfassende DevOps-Umgebung für die kontinuierliche Integration (CI) / Continuous Delivery (CD), eine eingehende Diagnose von Java-Anwendungen und eine vereinfachte Integration in SaaS und lokale Software.

7. Bestes Preis-Leistungsverhältnis

Oracle Cloud Infrastructure bietet das beste derzeit auf dem Markt verfügbare Preis-Leistungs-Verhältnis. In der Oracle Cloud-Infrastruktur bereitgestellte Workloads erfordern häufig weniger Computerserver und blockieren Speichervolumen, wodurch die Kosten für die Optimierung der Workload-Leistung gesenkt werden.

Die nicht blockierende Cloud-Infrastrukturarchitektur von Oracle trägt nicht nur zur Minimierung der Latenz bei, sondern verringert auch das Risiko, dass andere Netzwerkbenutzer Auswirkungen auf Clientumgebungen haben, und wird auch durch das umfassendste, finanziell sicherste End-to-End-SLA der Branche unterstützt.

8. Oracle Cloud übertrifft AWS

Die Oracle Cloud bietet optimale Leistung zu niedrigsten Kosten und ist damit die beste Option für die Einführung des Industriestandards Oracle Database für unternehmenskritische Unternehmensanwendungen. AWS bietet weder Oracle RAC noch Oracle Autonomous Data Warehouse oder Oracle Autonomous Transaction Processing an. Amazon Aurora schließt den Benutzer in der AWS-Cloud, während die Oracle-Datenbank Flexibilität verschiedener Hersteller bietet und kombiniert.

Oracle Computing-Server wie Oracle Virtual Machines und Bare Metal helfen Unternehmen dabei, im Vergleich zu konkurrierenden virtuellen Maschinen wie AWS erhebliche Kosteneinsparungen (25-65%) zu erzielen.

Oracle-Datenbanken sind schneller und kosten im Vergleich zu AWS bis zu 66% weniger. Auf diese Weise können Sie effizienter und kostengünstiger Ergebnisse auf der Grundlage von Geschäftsdaten erzielen ( Accenture, „Enterprise Workloads Meet the Cloud: Grundlegendes zu Oracle-Datenbanken in der Cloud“ , ”, Storage Review,“ Oracle Cloud Infrastructure Compute Bare Metal Instances Review ” .



9. Oracle Cloud unterstützt das Wachstum von KMU

Oracle Cloud wurde entwickelt, um die Interessen aller Unternehmen zu erfüllen, einschließlich kleiner und mittlerer Unternehmen. Dies ist auf die Tatsache zurückzuführen, dass kleine Unternehmen immer größer werden und häufig einen erheblichen Anstieg der Anwendungsimplementierung verzeichnen. Sie haben jedoch möglicherweise immer noch die gleichen Probleme wie größere Unternehmen. Oracle Cloud unterstützt jedes mögliche Szenario: Planen und verwenden Sie traditionelle und moderne Tools für Entwickler und für geschäftskritische Workloads.

10. Oracle und Microsoft stellen verbundene Cloud-Systeme bereit

Die Technologiepartnerschaften von Oracle und Microsoft für die Interaktion mit den Clouds des anderen unterstützen sowohl reguläre als auch geschäftskritische Anwendungen. Kunden können jetzt in die Cloud migrieren oder neue Anwendungen erstellen, indem sie das Beste aus Oracle Cloud, einschließlich Oracle Autonomous Database, und das Beste aus Microsoft Azure mit nahtloser Kompatibilität verwenden.

Diese strategische Partnerschaft bietet eine Reihe neuer Funktionen, mit denen gemeinsame Kunden einfacher auf die Cloud umsteigen können, darunter:

  • Cloudübergreifende Verbindungen,
  • Integrierte Cloud-Dienste,
  • Modell für kollaboratives Support
  • Flexibilität bei der Anwendungsbereitstellung.
