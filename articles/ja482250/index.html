VueJSの単純な状態マシン



私は最近、フルスタックラジオ- ステートマシンを使用したUIコンポーネントの強化に関する興味深いディスカッションに遭遇しました。ポイントは、ステートマシンの概念がVueコンポーネントの開発に役立つということでした。私は既製のソリューションを検討し始めましたが、それらはそれほど単純ではなく、より単純な実装を作成したかったので、この記事で説明します。この記事は、Vueを使用するユーザーだけでなく、Angular、Reactなどのユーザーや、他の「宗派」のプログラマーにとっても役立つ場合があります。

ディスカッションでは、コンポーネントを作成したすべての人に近いトピックに触れました。想像してみてください。データを要求するコンポーネントを作成していて、クエリの実行中にスピナーを表示する必要があること。通常、このような場合、ブール変数isLoadingを作成します。最初はisLoading = falseで、データを要求する前にisLoading変数をtrueに設定します。データが到着したら、再びfalseに設定します。スピナーの可視性はisLoadingに関連付けられています。

このアプローチはうまくいきます。しかし、コンポーネントがそれほど単純であることはめったにありません。そして、作業の過程で、さまざまな状態を格納する論理変数を作成する必要があります。問題は、1つの状態変数がある場合、2つの状態が生成され、2つの論理変数にはすでに4つ、3つ、8つなどの状態があることです。そして問題は、プログラマーが間違いを犯し、システムが発生する可能性のある状態を処理しない可能性があることです。そして、これはエラーにつながります。さらに、そのようなコンポーネントを理解することはかなり難しく、条件を作成する必要があります。

答えは状態機械です。原理は単純です。システムが存在する可能性のある状態と、状態間の遷移を記述します。このようなマシンがあれば、多くのエラーを回避できるだけでなく、システムの状態を宣言的に決定できます。

このテーマに関する記事はかなりあります（たとえば、Vueの有限状態マシンですが、それらのほとんどは、簡単な理論の後で、既製のライブラリの1つを接続することを推奨しています。たとえば、davidkpiano / xstateです。

Xstateはかさばるように見え、自分の自転車を試したかったのです。また、これは私にとって新しいパターンを作成する大きな理由です。

これがあなたがプレイできるコードです-Vueバージョン2のステートマシン。そして、これは少し異なるアプローチの生バージョンです-Vueを備えた状態マシン、バージョン1。
記事State Machinesから始めました。

したがって、ステートマシン：

class StateMachine {
    constructor (initialState, transitions) {
        this.state = initialState
        this.transitions = transitions
    }
    transition (nextState, method, params) {
  	const transitionsArray = this.transitions[this.state]	
        if(transitionsArray.indexOf(nextState) === -1) return this.state
        if(method) method(...params)
        this.state = nextState
        return this.state
    }
}

次のように初期化します。

const machine = new StateMachine('idle', {
    idle: ['waitingConfirmation'],
    waitingConfirmation: ['idle','waitingData'],
    waitingData: ['dataReady', 'dataProblem'],
    dataReady: ['waitingConfirmation'],
    dataProblem: ['waitingConfirmation']
})

コンポーネントメソッドで、遷移関数を作成します。

transition(nextState, method = null, params = []) {
    this.machineState = machine.transition(nextState, method, params)
}

イベントでは、行きたい場所をすぐに書きます：

@click=“transition('waitingConfirmation')"

同時にコンポーネントメソッドを呼び出す必要がある場合は、次のように記述します。

@click="transition('waitingData', getData, [222])”

2番目のパラメーターは、この遷移で呼び出す必要があるコンポーネントメソッドです。
3番目のパラメーターは、このメソッドの変数の配列です。

メソッド内では、呼び出しは次のとおりです。

this.transition('dataReady')
