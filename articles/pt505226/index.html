Sobrepor uma textura 2D em um objeto 3d usando p5.js (parte 1 - criando um padrão)


A imposição de um desenho ou padrão plano a um objeto tridimensional é sempre uma tarefa não trivial e, se você deseja fazê-lo com algum tipo de animação, a complexidade aumenta às vezes.

Ao mesmo tempo, existe uma biblioteca p5.js bastante simples, com a qual você pode criar padrões espetaculares complexos em apenas algumas dezenas de linhas de código. Eu criei recentemente esse projeto e agora quero dar um exemplo.

A maneira mais fácil de começar é criar um novo projeto no editor.p5js.org online

Por padrão, o novo projeto é o seguinte:

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
}

função setup () - define a função que é executada apenas quando a
função draw () - se repete em um loop infinito

Começamos criando um padrão radial 2D, que depois aplicamos ao nosso cubo. Ao criá-lo, primeiro usaremos as coordenadas polares e só depois as traduziremos para as coordenadas cartesianas da forma x, y familiar para a maioria.

O que é importante entender: a coordenada polar de um ponto é determinada por dois números:

r é a distância do ponto de origem (por exemplo, o centro da tela) ao ponto de destino.
a é o ângulo da linha imaginária em relação ao ponto inicial em que o ponto alvo está localizado

, portanto, o primeiro parâmetro pode variar de 0 ao infinito e o segundo - de 0 a 360.

A beleza dessa abordagem é que ela permite desenhar padrões bastante complexos com apenas uma fórmula.



A rosa polar é dada pela equação r (φ) = 2 \ sin φ

Para criar variáveis, vamos adicionar duas variáveis ​​no início do projeto: a primeira responsável pela distância do centro e a segunda pelo ângulo: let r = 10 let a = 0

Em seguida, na função draw (), que roda em um loop infinito, adicionamos a mudança de variável: a = a + 0,4

Como resultado, você obtém o seguinte código:

let r =10
let a =0
function setup() {
  createCanvas(400, 400);

}

function draw() {
  background(220);
    a = a + 0.4
}

Agora começamos a traduzir as coordenadas nas coordenadas cartesianas x y familiares para nós. Se você quiser entender o que exatamente está acontecendo no código, é melhor ler o tutorial de trigonometria, mas como isso não é tão importante para o resultado, você pode simplesmente colar o código na mesma função draw ():

let x = r*cos(a)
let y = r*sin(a)

Para finalmente ver os resultados de nossos cálculos matemáticos, precisamos fazer duas coisas:

  • pinte objetos com cores (você pode usar o dicionário de cores interno ou especificar o código de formato RGB usual - três números)
  • defina a forma da distribuição dos pontos. Por exemplo, uma elipse.

function draw() {
  background(220);
  
  
  let x = r*cos(a);
  let y = r*sin(a);
  fill(255,0,0);
  ellipse (x,y,10,10);
  r = r + 0.4
    
}



Os parâmetros da elipse (posição ao longo de X, posição ao longo de Y, diâmetro ao longo de X, diâmetro ao longo de Y) são definidos usando variáveis ​​ao longo de xe y. Como a variável x depende da variável r, que em nosso ciclo aumenta constantemente em 0,4, o círculo vermelho que vemos se move constantemente no canto superior esquerdo.

Para mover o ponto de referência para o centro, adicione a função de conversão (200.200) antes da variável x. Agora o círculo aparecerá no meio da nossa tela.

E vamos resolver outro problema - agora todos os círculos que aparecem são apagados a cada novo ciclo. Para sair desse círculo vicioso, precisamos mover a criação de nossa tela () além da função draw () e transferi-la para a função setup (), responsável pela criação do ambiente.

Após essa alteração, veremos algo semelhante a um círculo de carregamento. O principal problema é que agora eles estão localizados muito próximos um do outro. Vamos adicionar um pouco de espaço, definindo uma variável de ordem superior C com um valor de 20 e adicionando-a para alterar cada ciclo por 0,6 (ao lado de a). Bem, vamos introduzi-lo em nossas fórmulas trigonométricas: x = r + c * cos (a) e y = r + c * sin (a)
