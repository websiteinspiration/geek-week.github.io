Ubuntu 20.04の新機能


4月23日、Ubuntuバージョン20.04のコードネームFocal Fossaのリリースが開催されました。これは、長期サポート（LTS）のためのUbuntuの次のリリースであり、2018年にリリースされたUbuntu 18.04 LTSの続きです。

コード名について少し。 「Focal」という言葉は「中心点」または「最も重要な部分」を意味します。つまり、焦点の概念に関連付けられており、プロパティ、現象、イベントの中心にあり、「Fossa」には「FOSS」というルートがあります（フリーでオープンソースのソフトウェア-動物の名前でUbuntuのバージョンに名前を付けるという伝統の中で、フォッサはマダガスカル島のワイバーダー家の最大の略奪的な哺乳類です。
開発者は、Ubuntu 20.04を重要で成功したアップデートとして位置付け、今後5年間のデスクトップコンピューターとサーバーのサポートをサポートしています。
Ubuntu 20.04は、Ubuntu 19.04「Disco Dingo」とUbuntu 19.10「Eoan Ermine」の論理的な継続でした。デスクトップ版では、最新のトレンドに従って、暗いテーマが登場しました。したがって、Ubuntu 20.04には、標準のYaruテーマの3つのバリエーションがあります。

  • 光
  • 闇
  • 標準。

Amazonアプリもアンインストールされました。Ubuntu 20.04では、最新のGNOME 3.36をデフォルトのグラフィカルシェルとして使用しています。


主な変更点


Ubuntu 20.04は、2019年11月24日にリリースされた5.4カーネルに基づいています。このバージョンでは、いくつかの重要な革新が実装されました。これについては、以下で説明します。

lz4


Canonicalエンジニアは、カーネルと初期ブートイメージのinitramfsのさまざまな圧縮アルゴリズムをテストし、最適な圧縮（ファイルサイズが小さい）と解凍時間の妥協点を見つけようとしました。ロスレス圧縮アルゴリズムlz4は最も顕著な結果を示し、Ubuntu 19.10に追加されました。これにより、以前のリリース（Ubuntu 18.04および19.04）と比較して起動時間が短縮されました。同じアルゴリズムはUbuntu 20.04にも残ります。

Linuxロックダウンカーネル


Lockdown関数は、ユーザープロセスによって提供されるコードを介して任意のコードを実行できるようにする関数へのアクセスを制限することにより、Linuxカーネルのセキュリティを強化します。簡単に言うと、rootスーパーユーザーアカウントでもカーネルコードを変更することはできません。これにより、rootアカウントが侵害された場合でも、潜在的な攻撃による被害を減らすことができます。したがって、オペレーティングシステムの全体的なセキュリティも向上します。

exFAT


Microsoft FATファイルシステムでは、4 GBを超えるファイルを転送できません。この制限を克服するために、マイクロソフトはexFATファイルシステムを作成しました（英語の拡張FAT-「拡張FAT」から）。たとえば、組み込みの exFAT ファイルシステムサポートを使用して、USBドライブをexFATにフォーマットできます。

ワイヤーガード


Ubuntu 20.04は5.6カーネルを使用しませんが、少なくともすぐには、5.4カーネルのWireGuardバックポートをすでに使用しています。WireGuardはVPN業界における新しい言葉です。そのため、カーネルにWireGuardを含めることで、Ubuntu 20.04にクラウドの利点がもたらされます。CFSクォータのバグを

修正し、マルチスレッドアプリケーションがより高速に実行できるようになりました。Ryzenシリーズのプロセッサーの温度および電圧センサーを操作できるドライバーが追加されました。 これらすべては、5.4カーネルに登場したすべての革新とはかけ離れています。詳細なレビューは、上で見つけることができkernelnewbies.orgリソース（英語）と上OpenNETのフォーラム（ロシア語で）。



Kubernetesの使用


Canonicalは、Ubuntu 20.04で実装され、Kubernetes 1.18を完全にサポートし、Charmed Kubernetes、MicroK8s、kubeadmをサポートしています。

Ubuntu 20.04にKubectlをインストールします。

# snap install kubectl --classic

kubectl 1.18.0 from Canonical ✓ installed

SNAPの使用


Canonicalは引き続き、ユニバーサルパッケージフォーマット（snap）を宣伝しています。これは、Ubuntu 20.04リリースでさらに明白になります。インストールされていないプログラムを実行しようとすると、まず、次のコマンドを使用してインストールするように提案されます。

# snap install <package>


改善されたZFSサポート


が、Linus Torvalds氏は、ZFS好まないだろう、それはまだ一般的なファイルシステムであり、実験的なサポートは、Ubuntuの19.10で追加されました。
それはデータストレージ、仕事での同じホームアーカイブまたはサーバーストレージ（箱から出して同じLVMより多くを行うことができる）のために十分に便利で安定しています。 ZFSは、最大256兆ゼタバイト（したがって、名前の文字「Z」）のパーティションサイズをサポートし、最大16エクサバイトのファイルを処理できます。

ZFSは、ディスクへの配置方法に基づいてデータ整合性監視を実行します。記録中のコピー機能は、使用されるデータが上書きされないことを保証します。代わりに、新しい情報が新しいブロックに書き込まれ、ファイルシステムのメタデータがそれを指すように更新されます。 ZFSを使用すると、ファイルシステムに加えられた変更を追跡するスナップショット（ファイルシステムのスナップショット）を作成し、ファイルシステムとデータを交換してディスク領域を節約できます。

ZFSはディスク上の各ファイルにチェックサムを割り当て、そのステータスを常にチェックします。ファイルが破損していることがわかった場合は、自動的に復元しようとします。ZFSを使用できるようにする個別のアイテムがUbuntuインストーラーに表示されました。ZFSの歴史とその機能の詳細については、It's FOSSブログをご覧ください。

さようならPython 2.X


Pythonの3番目のバージョンは2008年に導入されましたが、Python 2プロジェクトをそれに適応させるには12年も足りませんでした。
Ubuntu 15.10に戻ると、Python 2を放棄する試みが行われましたが、そのサポートは継続されました。そして、2020年4月20日、Python 2ブランチの最後のリリースであるPython 2.7.18がリリースされました。

Ubuntu 20.04はPython 2をサポートしなくなり、PythonのデフォルトバージョンとしてPython 3.8を使用します。残念ながら、世界中には多くのPython 2プロジェクトが残っており、それらにとって、Ubuntu 20.04への切り替えは骨の折れる作業です。

1つのコマンドで最新バージョンのPython 2を配置できます。

# apt install python2.7

Python 3.8に加えて、開発者は更新されたツールキットを評価できます。

  • MySQL 8
  • glibc 2.31
  • OpenJDK 11
  • PHP 7.4、
  • Perl 5.30、
  • Golang 1.14。

さようなら32ビット


Ubuntuは数年間、32ビットコンピュータ用のISOイメージを提供していませんでした。現在、32ビットバージョンのUbuntuの既存のユーザーはUbuntu 18.04にアップグレードできますが、Ubuntu 20.04にアップグレードすることはできません。つまり、現在32ビットのUbuntu 18.04を使用している場合は、2023年4月までそのまま使用できます。

アップグレードする方法


以前のバージョンからUbuntu 20.04へのアップグレードは、次のコマンドを実行するのと同じくらい簡単です。

# sudo apt update && sudo apt upgrade
# sudo do-release-upgrade

Ubuntu 20.04 LTS（Focal Fossa）が、当社のクラウドプラットフォームの仮想マシンのイメージとしてすでに利用可能であることをお知らせいたします。最新のソフトウェアを使用して、独自の仮想ITインフラストラクチャを作成してください！

UPD： Ubuntu 19.10ユーザーは20.04にアップグレードできるようになり、Ubuntu 18.04ユーザーは2020年7月23日にリリース予定の20.04.1のリリース後にアップグレードできるようになります。
