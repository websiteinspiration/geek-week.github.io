JSでの命名について簡単に

こんにちは、ハブル！私はJavascriptでエンティティを命名するトピックに触れることにしました。職場では、私はインターンと多くの交流をしていて、十分に見てきました。そのため、JavaScriptコミュニティで今日採用されているエンティティの命名規則を1つの小さなメモにまとめておくとよいと思いました。たぶん全部集めなかったので、コメントで補足してくれたらありがたいです。

エンティティの命名


命名はソフトウェア開発において非常に重要です。ご存知のように、コードは主にそれを読む人（プログラマー向け）向けに書かれています。不適切な名前を付けると、コードの読み取りに余分な時間がかかるため、プロジェクトの開発またはサポートの人件費が大幅に増加する可能性があります。ネーミングが悪いと、「コードの内容は何か」の解釈が難しくなります。

名前にはさまざまな構文形式があり、たくさんありますが、一部は使用されていません。jsで最も一般的なものは次のとおりです。

  • ラクダ記法（CamelCase）：MyClass
  • Snake_case：my_const
  • バーベキュー表記（ケバブケース）：my-data

ケースを選択するときは、現在受け入れられている標準を考慮することが重要です。今日のjs では、snake_caseとkebab -caseは受け入れられませんが、たとえばPythonやRubyで見つかります。

1文字の識別子


教育機関では、コードで1文字の識別子を使用することがよくあります。私はこのコーディングスタイルを高校を卒業した男の半分に見かけます。これは非常に悪質な行為です。名前は本質を明確に説明する必要があります。今日では、1文字の識別子を使用することは悪い兆候です。例外は、カウンターとインデックスです。1つの文字が本質の本質を伝えるのに十分以上の状況。

名前に変換


文字変換を使用することも学生の間で非常に人気があります。もちろん、これは悪い味と悪いコードの兆候でもあります。ネーミングに文字変換はないはずです。プログラミングの共通言語は英語です。コードを開発するときは、各国の専門家が知っている国際的な言語を使用する必要があります。ロシア語の文字変換はそれらには適用されません。

変数とクラスの命名


変数の名前はlower camelCase次のとおりです。

const maxCount = 10;

クラスの名前はCamelCase次のとおりです。

class EnumerableCollection {
//some code
}

行動


アクション（関数など）の命名には動詞を使用することが非常に重要です。そのような動詞を選択する必要があります。これはアクションのタイプに対応しています。
例えば：

const checkNumberIsEven = (number) => (number % 2 === 0);

checkNumberIsEvenいい名前です。関数がパリティの数をチェックすることはすぐに明らかです。

また、良い名前です isEven -この関数が/helpers/number.jsにある場合、このような短い名前でも十分です。ディレクトリ自体は、数値を操作するための関数が含まれていることを示しています（ただし、この関数を使用するファイルには大量のコードが含まれている可能性があるため、ここでも最初のオプションを使用できます。真ん中。）

機能は常にアクションとはほど遠いので、これを理解することも重要です。
例えば、

const arifmeticalProgression = (start, depth, maxLength = 10) => {
  const progression = [start];
  const iter = (acc) => {
    if (acc.length >= maxLength) {
      return acc;
    }
    const newIndex = acc.length;
    const newItem = start + newIndex * depth;
    const newProgresion = [].concat(acc, newItem);
    return iter(newProgresion);
  };
  return iter(progression);
};

この関数は算術的な進行を生成しますが、アクションではありません。その宣言的な性質を考慮して、算術的進行の定義が考慮されます。この瞬間を区別できることが重要です。これには、定数を定義する関数も含まれます。

const defaultCollection = () => ([]);

述語


上記では、関数について説明しました

const checkNumberIsEven = (number) => (number % 2 === 0);

このタイプの関数は、述語と呼ばれます。述語は何かについての声明です。いわゆる関数が「本質は何か」というチェックを実行しています。プログラミング述語は常にブール値を返します。

原則として、述語は、英語の補助動詞の3人称単数形を通じてと呼ばれます。です。

const isEven = (number) => (number % 2 === 0);

一部の述語は、エンティティ内の目的の要素（プロパティまたはメソッドまたはitem'a）の出現（存在）を決定します。そのような述語。通常、英語の動詞が持つ（3人称単数動詞が持つ）で始まります。たとえば、安全なフォームObject.prototype.hasOwnPropertyは次のようになります。

const hasProp = (obj, key) => (Object.prototype.hasOwnProperty.call(obj, key));

エンティティが何かの量である場合はcount 、名前に単語を使用する必要があります。
