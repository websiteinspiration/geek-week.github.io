ニューラルネットワーク上のボット：仮想アシスタントのしくみと学習方法

2016年、チャットボットの人気のピーク時に、私たちのチームはビジネス用のボタンアシスタントを作成しました。「ニューラルネットワークを使用してクライアントサポートを自動化するとどうなるでしょうか？」チャットロボットが自然言語を理解し、快適になることを最終的に学びたいと思っていました。

4人の数学者、6か月のハードワーク、1つの新しいプログラミング言語、多くのミスを要しました。そして、誰もがAIで仮想アシスタントを組み立てることができるコンストラクターを作成しました。

チャットボットと仮想アシスタント


高度なチャットボットは、60年代にキーワードを強調し、人間の会話を模倣することができました。ヒッピーはLSDに身を投じ、ビートルズはスタジアムを集め、ジョセフウェイゼンバウムは対談心理学者のエリザを作成しました。
たとえば、「父は私を憎んでいる」という文の中で、エリザは「父」というキーワードに応答して、「家族の他に誰があなたを憎んでいるのですか」と尋ねました。しかし、ロボ心理療法士は問題の本質を理解していませんでした。最新のチャットボットも機能します：キーワード、線形スクリプト、およびライブの対話のパロディ。

しかし、60年代から何かが変わっています。今では、機械学習とNLPテクノロジーのおかげで、チャットボットに自然言語とコンテキストを理解するように教えることができます。これはまだ模倣ですが、より意味があります。

違いを確認するために、チャットボットとアシスタントを比較してみましょう。仮想ソムリエを作成して、顧客がワインを選択できるようにする必要があるとします。

ステージ1


チャットボットと仮想アシスタントの最初の段階は1つです。ユーザーの要求を見積もり、ユーザーが記述できるフレーズを考え出します。そして、ロボットがどのように反応するかを規定します。

基本的な要求は明確です-ワインをピックアップしてください。ただし、このリクエストには、理由、価格、国、色、ブドウの品種など、多くのパラメータが含まれる可能性があります。おそらく、ユーザーはすぐに検索するワインの名前を書き込みます。または、彼は詳細を明らかにします：「どこで作られますか？」あるいは、好奇心から、「世界で最も高いボトルはどれですか？」という質問でボットを投げ始めます。等さらに、ケースのリクエストの他に、「こんにちは」、「さようなら」、「お元気ですか」など、尋ねる必要のある小さなトークフレーズがあります。

あなたは無限に空想することができますが、それでも私たちはユーザーのすべての質問を予測することはできません。説明されている状況でリクエストの98％をカバーするのに十分であると思われる場合は、停止します（ただし、厳しい現実が解消され、最大で80％がカバーされることがわかります）。

次に、特定のリクエスト-意図に対するユーザーのニーズに関する仮定を分散させます。意図は、ユーザーが何を望んでいるかを示しますが、ユーザーがそれについて書いた方法に関する情報は破棄します。このステージは、チャットボットとアシスタントの場合と同じです。

アルゴリズムはいくつかのフレーズを他のフレーズと区別できますが、ロボットが言語を本当に理解していると言えるでしょうか？

この質問に答えるために、フレーズの語彙的意味の比較に戻りましょう。コンピュータの意味は、文字列、数値、およびそれらの組み合わせなどの理解可能なデータ型です。したがって、プログラマーは、ソーステキストを比較の数学演算に適した形式（ベクトル）に変換する作業に直面します。

vectorize(" ") = (0.004, 17.43, -0.021, ..., 18.68)
vectorize("  ") = (0.004, 19.73, -0.001, ..., 25.28)
vectorize("  ") = (-8.203, 15.22, -9.253, ..., 10.11)
vectorize(" ") = (89.23, -68.99, -10.62, ..., -0.982)

私たちのタスクでは、語彙的に近いフレーズのベクトルは数学的に互いに近く、語彙的に離れたフレーズのベクトルは遠く離れている必要があり、別のオペラのフレーズのベクトルは遠く離れている必要があります。たとえば、「ワインが欲しい」は「ワインが欲しい」よりも「白ワインが欲しい」に近いです。そして「火星の攻撃」とは程遠い。

適切にトレーニングされたニューラルネットワークは、これらのベクトルで語彙の意味を結論付けることができます。 2つのフレーズの意味を比較するには、それらのベクトルを比較する必要があります。

したがって、「ロボットは言語を理解していますか？」という質問への回答。それはこのようになるでしょう。彼らは人がどういうものかを理解していません、彼らは単に語彙の意味を比較する方法を知っており、温かさとソフトを混同しません。しかし、アルゴリズムが示唆に富む改良を提供し、結論を導き出すことができるとき、私たちは正直に言って：はい、理解が到着しました。それまでの間、「言語を理解する」は、単なる美しいマーケティングフレーズです。

実際、ロボットは3歳の子供と同じように機能します。ただし、十分な数の例を子供に与えると、子供は知識人になりすまして話し合いを進めることができます。サポートの最初の行の「ライブ」オペレーターは同じように機能します。それらは一連の状況の概要を示し、それらの中でどのように動作するかを伝えます。したがって、仮想アシスタントはサポートの自動化に適しています。

コンテキストを理解するようにロボットに教えた方法：lialang


通常のサポートの場合、ロボットは自然言語をほとんど「理解」する必要はありません。ロボットが質問に答えて状況を把握できることが重要です。これを行うために、スクリプトを記述してロボットに渡すことができるダイアログマークアップ言語であるlialangを作成しました。

lialangプログラマーの主なタスクは、人と機械の間の対話で起こり得るすべての状況を説明することです。これを行うには、私たちの言語で意図と行動の名前を関連付けることができます。

簡単な例を考えてみましょう-挨拶：

if intent() {
    reaction(_)
}

通常のコードのように見えますが、ニューラルグリッドはインテント（...）コンストラクトの背後で機能します。lialangは、通常のプログラミングコンストラクトを使用して、一般的なパターン（「何かを求められた場合」）でダイアログを記述します。もちろん、これが機能するためには、機械学習とNLUテクノロジーを適用する必要があります。これは、ユーザーが希望どおりにリクエストを作成できるためです。

そして、ここに文脈状況を説明する方法があります。

if intent() {

        reaction(___)
    } else {
        reaction(_)
    }
}

リアクションは、リアが意図に応じて実行する必要があるアクションです。95％の場合、これは単なるテキストです。しかし、ロボットはコード内の関数を呼び出したり、通信をオペレーターに切り替えたり、その他の複雑なアクションを実行したりすることもできます。

テキストと関数を送信するためのコードは、言語とは別に存在します-言語は状況をできるだけ簡単に記述します。

次に、もっと複雑なことを書いてみましょう-銀行とのチャットで、顧客はしばしば詳細を尋ねます。lialangを使用してチャットやメールに送信する方法を学びましょう。

ここでは2つの状況について説明します。

  1. Lia、詳細の送信を求められた場合は、チャットに送信してください。彼らが「郵送する必要がある」と尋ねた後、彼らを郵便に送ります。
  2. Lia、詳細をメールで送信するようにすぐに求められた場合-詳細をメールで送信してください。

したがって、lialangはその役割を果たします-コンテキスト内で機能します。「メールでお願いします」と書いても、ロボットが細部について話していることを理解します。

Liaは複雑なシナリオをサポートすることを学びました-ユーザーにとって必要なときに、CRMとの間でデータの取得/送信、SMSの送信、支払いの支援、または人生についての話をします。

徐々に言語を改善しました。変数、関数、エンティティ（日付、住所、電話番号、名前など）、状態、その他の便利な構成を追加しました。したがって、それを書くことはさらに便利になりました。

ケースチェック：3つの銀行でサポートを自動化した方法


テクノロジーを作成したらすぐに、それを早急に完成させる必要がありました。最初のお客様がいました。 VTBは、起業家のための新しいインターネット銀行でのサポートを自動化する必要がありました。

特に4か月で作成された製品の場合、私たちはかなり成功しました。 VTB用のハイブリッドロボットはニューラルネットワークに基づいており、すぐに効果的でした。800以上の質問に答え、いくつかの複雑なシナリオ（ステートメント、関税の変更、ユーザー設定）をサポートし、人のように話しました。その結果、2か月でLiaはサポートの負荷を74％削減しました。それは明らかになりました：サポート作業を自動化するという考え。



さらに、Liaに基づいて、「Rocketbank」と「DeloBank」のFAQを自動化し、2週間でオペレーターのいないアプリケーションの32％を閉じました。

少年たちは成功したようだ。しかし、最初の顧客の後、コンセプトを変更する必要があることが明らかになりました。それは地獄でした-スクリプトを手動で変更し、編集し、ブランチを開発する必要がありました。シンプルなボットでも同じことが言えますが、より難しく、より多くの強度が必要です。この状況では、拡張が困難でした。

それから、クライアント自身が複雑なアシスタントを組み立てることができるツールを作ることにしました。また、チュートリアルのサポートとユーザーの教育のみを行います。

Liaプラットフォームとインターフェース用エンジン


そこで、開発が分からない人のためのプラットフォームを作ることにしました。 lialangに含まれる設計は10個未満ですが、すべてのマネージャーが独自のボットを作成するように教えるわけではありません。マネージャーはマウスが大好きです。

したがって、私たちはlialangができるすべてのことを実行できるようなインターフェースについて考え始めました。彼は、ネストされたブランチ、スクリプトから別のスクリプトへの遷移、そして最も重要なことに-私たちのプログラマーだけでなく、スクリプトを作成したい人は誰も問題を抱えません。

それがどのように見えるかを見てください：


ダイアログは非線形のものであり、あらゆる種類の会話スキームのためのユニバーサルエンジンを書くことは非常に困難です。しかし、スキームについて考える前に、すでにlialangがありました-エンジンになりました。

インターフェース設計者が思いついたものは何でも、私たちはそのための頭脳をコーディングしませんが、インターフェースからlialangコードへのマークアップの小さなトランスレータのみを書きます。インターフェースがやり直された場合、トランスレータを変更するだけで済みます。これにより、インターフェースコマンドとコアコマンドを別々に存在させることができます。

仮想アシスタントを作成するためのプラットフォームはどのように機能しますか？


Liaで独自の仮想アシスタントを組み立てるには、ユーザーは3つの段階を経る必要があります。

ステップ1.ロボットが主なシナリオを理解して強調できるように、ユーザーとのチャット履歴をダウンロードする


クライアントにユーザーとの会話の履歴がある場合は、クライアントをシステムにアップロードして、最も人気のあるクエリからクラスターを取得できます。それらから意図を作成することは非常に便利です。



時間の経過とともに、Liaの効果は増加します。ロボットが応答できなかったメッセージは、クラスターに再度配信されます。たとえば、「あなたは誰ですか？」、「あなたは誰ですか？」、「あなたの名前は何ですか？」などです。そしてあなたはロボットですか？1つのグループに分類されます。したがって、プラットフォームは半自動で自己学習します。ユーザーはギャップがどこにあるかを確認してそれを閉じ、新しいシナリオを追加します。その結果、カバーされるリクエストの割合が6か月で30％から70％に増加します。

通信履歴がない場合は、この手順をスキップして、すぐに2番目の手順から始めます。ユーザーがアシスタントに依頼する要求を予測します。

ステップ2.目的を書き留める：最も頻繁に発生するユーザー要求に対する10〜20のオプション


2番目の段階では、10〜20の最も人気のあるクエリに基づいて意図を指定します。ニューラルネットワークのおかげで、この数のオプションで十分です。したがって、「ワインを注文したい」などのフレーズは、ボットに類似のユーザーリクエストを認識するように指示します。たとえば、「ワインを入手」や「ワインを注文」などです。



別のアシスタントがエンティティを理解して抽出します：都市名、電話番号、住所、タイムスタンプ、期間、日付、呪い-「疲れた、明日のワインの箱を持っていく」と言っても。

さらに、ユーザーは独自のオブジェクトを追加し、トレーニングフレーズに手動でマークを付けることができるため、アシスタントはより速く学習できます。これはプラットフォームの最も強力な機能の1つであり、これにより、人間のオペレーターと同じように機能するロボットを作成できます。

ここで、ユーザーはユーザーの要求に対する答えを考え出す必要があります。平均的なプロジェクトの95％は通常、簡単な質問と回答のシナリオ、つまり一般的な質問への回答によって行われます。



ちなみに、アシスタントは写真、動画、音声ファイルで応答し、必要に応じて位置情報を送信できます。

構築へのジャンプは特に便利です-そのおかげで、リアはあるシナリオから別のシナリオに切り替えてから戻って、いくつかの問題を解決できます。これは、会話から離れて正しいトラックに戻る必要がある場合に役立ちます。たとえば、明確な質問をします。

ダイアログジャンプの例：

アシスタント：受信者が応答しないため、配送先住所を明確にする必要があります。
クライアント：そして何について？何の順番？ （順序を明確にするためのジャンプ先）
アシスタント：8月21日の注文、Antigal、 "Uno" Malbec。
アシスタント：では、現在の配送先住所を明確にできますか？（メインスクリプトに戻ります）

作成者は、アシスタントがテキストで応答するだけでなくアクションを実行することを望んでいる場合、プログラマーを呼び出して専門家向けのJSスニペットを作成することができます。スクリプトの実行中にJavaScriptを直接実行できます：外部APIへのアクセス、電子メールの送信、または別の複雑なアクションの実行。

4.まとめ


アシスタントは何とでも統合できます。電話で、インスタントメッセンジャーで、またはクライアントがサイトに投稿するウィジェットで、人と話します。

プラットフォームでスマートアシスタントを組み立てるには、数時間から1か月かかります。次に、ロボットは要求を現実的に理解し、パターンを認識することを学びます-約6か月かかります（この間、生きている人が監視します）。ビジネスは、不正なLiaガールの日常的なタスクのほとんどを実行できます。配達の調整からタクシーの呼び出し、クライアントのコンサルティングまで。



ちなみに、銀行との完了したプロジェクトはすでにプラットフォームに転送されています。それらも同様に機能しますが、より便利に管理されます。

近い将来、ユーザーがより複雑なデータを抽出できる抽出機能を追加する予定です（たとえば、ロボットは「明日の翌日の昼食後」というフレーズを理解します）。また、クライアントがプロジェクトのバージョンをすばやくロールバックできるように、バージョン管理を完成させます。また、組織の役割システムを解き放ちます。

私たちはガートナーの予測が正しいことを期待しています-そして2022年には、すべての顧客とのやり取りの最大70％が何らかのAIを通過するでしょう。私たちの考えによると、リアのようなデザイナーは、クライアントサービスをロボットにさらに速く転送するのに役立ちます。
