DDoS保護会社自身がDDoS攻撃を仕掛けた、その創設者は認めた


2016年までに、vDosはDDoS攻撃を注文するための世界で最も人気のあるサービスになりました。

陰謀説によれば、ウイルス対策会社自体がウイルスを拡散し、DDoS保護サービス自体がこれらの攻撃を開始します。もちろん、これはフィクションです...

2020年1月16日、ニュージャージー州連邦地方裁判所は、ジョージア州メイコンの居住者である22歳のタッカープレストンに、「プログラム、コード、またはコマンドの転送による保護されたコンピューターへの損害」のいずれかの罪で有罪判決を下しました。 Tuckerは、DDoS攻撃に対する保護を提供するBackConnect Security LLCの共同創設者です。若いビジネスマンは、手に負えない顧客に復讐する誘惑に抵抗できませんでした。

タッカープレストンの悲しい話は、ハッカーのティーンエイジャーが友人のマーシャルウェッブと一緒にBackConnect Security LLCを設立したときに始まり、その後、BackConnect、Inc. 2016年9月、この会社はvDosサービスを閉鎖する運用中に注目を浴びました。vDosサービスは、当時、DDoS攻撃を注文する世界で最も人気のあるサービスと見なされていました。次に、BackConnectはvDosを介して自分自身を攻撃し、異常な「反撃」を実行し、BGPインターセプト（BGPハイジャック）によって敵のIPアドレスを255個キャプチャしました。このような攻撃を行って自分の利益を守ることは、情報セキュリティコミュニティで対立する意見を生み出しています。多くの人が、BackConnectが限界を超えたと信じていました。

単純なBGPインターセプトは、他の誰かのプレフィックスを自分のものとしてアナウンスすることによって実行されます。アップリンク/ピアはそれを受け入れ、ウェブ上に広がり始めます。たとえば、2017年に、ソフトウェア障害の結果として、Rostelecom（AS12389）は、プレフィックス Mastercard（AS26380）、Visa、およびその他の金融機関を発表し始めました。 BackConnectは、ブルガリアのホストVerdina.netからIPアドレスを取得した場合とほぼ同じように機能しました。

BackConnectのCEOブライアントタウンゼントは後に言い訳をしましたネットワークオペレーター向けのNANOGメーリングリスト。敵のアドレス空間を攻撃するという決定は容易ではなかったと彼は言ったが、彼らは彼らの行動に対して答える用意ができていた：「私たちは自分の行動を隠す機会があったが、それは間違っていると感じた。 「私はこの決定について、そしてそれが一部の人々の目には会社と私にどのように悪影響を与える可能性があるかについて考えることに多くの時間を費やしましたが、結局私はそれを支持しました。」

結局のところ、BackConnectがBGPインターセプトを使用するのは初めてではありませんが、会社には一般的に暗い歴史があります。ただし、BGPインターセプトは必ずしも悪意のある目的で使用されるわけではありません。ブライアン・クレブスは書いています彼はProlexic Communications（現在はAkamai Technologiesの一部）のサービスを使用してDDoSから保護しています。 BGPハイジャックを使用してDDoS攻撃から保護する方法を思いついたのは彼女でした。

DDoS攻撃の被害者がProlexicに助けを求めると、後者はクライアントのIPアドレスをそれ自体に変換します。これにより、着信トラフィックの分析とフィルタリングが可能になります。

BackConnectがDDoS保護サービスを提供したため、クライアントの利益のために正当であると見なすことができるBGPインターセプトと、疑わしいと思われる分析が行われました。これには、他の誰かのアドレスのキャプチャ期間、他の誰かのプレフィックスが自分のアドレスとしてどれだけ広く発表されているか、クライアントとの合意が確認されているかどうかなどが考慮されます。



どうやら、被害者の1人がBackConnectを訴えました。プレストンの告白（pdf）には、裁判所が被害者を見つけた会社の名前は含まれていません。被害者は、ドキュメントでは被害者1と呼ばれています。

上記のとおり、BackConnectの調査はvDosサービスをハッキングした後に始まりました。その後、サービス管理者の名前、およびvDosデータベース（登録ユーザーと、DDoS攻撃を行うためにvDosを支払った顧客の記録を含む）が判明しました。

これらの記録は、vDos Webサイトのアカウントの1つが、Tucker Prestonの名前で登録されているドメインに関連付けられたメールアドレスに対して開かれていることを示しています。このアカウントは、Free Software Foundation（FSF）に属する多数のネットワーク攻撃を含む、多数のターゲットに対する攻撃を開始しました。

2016年、元FSFのシステム管理者は、非営利組織がBackConnectとの連携を検討していると述べ、FSFがDDoSから保護する別の会社を探すと言った直後に攻撃が始まりました。声明に
