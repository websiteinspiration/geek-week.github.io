少し前に、自分の目でReportPortalとは何かを確認することにしましたが、このメモは、それがどのような種類のモンスターであるか、または何ができるか、または何ができないかについてではなく、「知る」および「感じる」ためにすばやく展開する方法についてです。

レポートポート自体のwikiにインストールドキュメントがあり、私が行動しました。 Dockerは指示に従ってWin10 +に段階的にインストールされましたが、何かが「うまくいきませんでした」。このプロセスは数回繰り返されましたが、それでも結局すべてが判明しました。一般に、長い間苦しめられていて、ドッカー自身も彼の神経を痛めました（まあ、Unixのために書かれました）。結果として、すべてのアクションが機能した後、私はこのビジネスを終了し、仮想マシンですべてを行うことにしました。

3.bオプション。デフォルトでは、レポートポートはポート8080で「リッスン」します。別のサービス（selenoid-ui）にこのポートが必要だったため、レポートポートに別のポートを「与える」ことにしました。

これを行うには、docker-compose.ymlを編集します。

nano docker-compose.yml

セクションを探しています：


  gateway:
    image: traefik:1.6.6
    ports:
      - "8080:8080" # HTTP exposed
    #      - "8081:8081" # HTTP Administration exposed
    command:
      - --consulcatalog.endpoint=registry:8500
      - --defaultEntryPoints=http
      - --entryPoints=Name:http Address::8080
      - --web
      - --web.address=:8081
    restart: always

次のように編集します。


  gateway:
    image: traefik:1.6.6
    ports:
      - "<NEW_PORT>:8080"
    expose:
      - "8080"
    #      - "8081:8081" # HTTP Administration exposed
    command:
      - --consulcatalog.endpoint=registry:8500
      - --defaultEntryPoints=http
      - --entryPoints=Name:http Address::8080
      - --web
      - --web.address=:8081
    restart: always

ここで、<NEW_PORT>は、レポートポートに指定するポート番号です。「8888」の

3秒を入力しました。レポートポートをインストールします。

docker-compose -p reportportal up -d --force-recreate

画像がダウンロードされ、Dockerコンテナーが作成され、自分で起動されます。コンソールでのプロセス全体が終了したら、少し待って、すべてが正常に開始するようにします。実際のマシンのブラウザーで、仮想マシンのIPと選択したポート（または標準の8080）に関するレポートを開くことができます。



組み込みの管理者にログインします： `superadmin \ erebus`
