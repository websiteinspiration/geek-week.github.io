コンピュータービジョンサマーキャンプ-インテルコンピュータービジョンサマースクール



国連の彼らに基づいて7月3日から16日まで。N.I. ロバチェフスキーは、インテルコンピュータビジョンの大学間サマースクール-コンピュータビジョンサマーキャンプに合格し、100人以上の学生が参加しました。この学校は、コンピュータービジョン、ディープラーニング、ニューラルネットワーク、Intel OpenVINO、OpenCVに関心のあるニジニノヴゴロド大学の専門技術の学生を対象としています。

この記事では、学校の選択がどのように行われたか、私たちが何を研究したか、実際の部分で男が何をしたかを共有し、また、防御について提示されたいくつかのプロジェクトについて話します。

選考プロセスと参加フォーム


私たちは子供たちに、フルタイムとパートタイムの2つの形式のトレーニングに申し込むことを選択することにしました。学生はフルタイムの通信の選択に合格せず、すぐに登録されました。彼らは平日の午前中だけ講義に出席した。彼らはまた、実用的なタスクを完了し、教師によるテストのためにそれらをGitHubに送信する機会がありました。

フルタイムで行くには、委員会との面接のためにインテルのオフィスに来なければなりませんでした。パートタイムのフォームとの違いは、講義に加えて、キャンプの参加者はキュレーター、UNNの教師やIntelのエンジニアと一緒に実践的なタスクを整理したことです。 2週目には、実際のタスクが終了し、プロジェクトが開始され、参加者は3人のグループで作業しました。

インタビューでは、数学とプログラミングに関する質問が行われ、その場で解決する必要のある課題が与えられました。委員会がソフトウェアエンジニア、アルゴリズムエンジニア、大学教授で構成されていたことは注目に値します。 N.I.ロバチェフスキー、それでインタビューは多国間で卓越していることがわかりました。インタビュイーの観点から、コンピュータービジョンに適用できる学生の基本的な技術知識を見つけることは興味深いため、C ++ / STL、OOP、基本的なアルゴリズムとデータ構造、線形代数、数学的分析、離散数学などのトピックが尋ねられました。タスクから、学生の推論を学ぶことが優先されました。委員会は彼らがどこで勉強したかにも興味がありました、この学校の前に彼らがどのような経験をしたか（たとえば、科学的活動）、それをコンピュータビジョンの分野に直接適用する方法。

フルタイムの選考には合計78人の学生が参加しましたが、24のフルタイムの場所があり、コンテストは場所ごとに3人の学生でした。以下の表に、参加者の統計と、フルタイムとパートタイムの参加形態の視覚的な違いを示します。



彼らは2週間何をしましたか？


理論と実践の学生は、コンピュータビジョンの主要なタスクである画像の分類、オブジェクトの検出、およびそれらの伴奏に慣れました。原則として、各トピックの講義コンポーネントには、コンピュータービジョンの問題を解決するための古典的な方法と、機械学習とニューラルネットワークを使用した現代的な解決方法の開発への歴史的エクスカーションが含まれていました。理論の後には、学生が人気のあるニューラルネットワークモデルをダウンロードした後、OpenCVライブラリのDNNモジュールを使用してそれらを起動し、ユーザーアプリケーションを作成する練習が続きました。

すべての講義のプレゼンテーションがGithubのパブリックリポジトリに投稿されました生徒が放課後も含め、必要な情報をいつでも開いて確認できるようにします。講師、教師、インテルのエンジニアとのチャットは、Gitterでライブでもチャットでも可能でした。プロジェクトの週の時間も成功したことがわかりました。それは水曜日に始まり、週末を講義なしで過ごすことができ、チームの意思決定を改善しました。最も責任のある参加者は、土曜日の半分をインテルのオフィスで過ごしました。彼らは、その日の予定外の遠足に励まされました。

プロジェクトの保護はどうでしたか？


各チームには、プロジェクト中に何をしていたか、何をしたかを伝えるために10分が与えられました。この時間の後、5分間が始まりました。会社のエンジニアが彼らに質問し、プロジェクトの改善や将来の間違いを防ぐのに役立つ少しのアドバイスをしました。男たちはそれぞれ、スピーカーとして自分自身を試し、コンピュータービジョンの分野での知識を示し、プロジェクトの作成への彼の貢献を確認しました。これは、学校の各参加者について考え、結論を導くのに役立ちました。防御は3時間行われましたが、スタッフの面倒を見て、小さなコーヒーブレークで状況を解消しました。そこでは、スタッフが一息つき、主要なIntelスペシャリストと問題について話し合うことができました。

1日の終わりに、1位、2位、3位の3位を割り当てました。各チーム、各プロジェクトには独自のフレーバーがあり、プレゼンテーションの独創性によって区別されたため、選択するのは非常に困難でした。

発表されたプロジェクト


スマートグローブ




空間での視覚的なナビゲーションのためのOpenCVを使用した検出器とトラッカーの使用。チームはさらに、2台のカメラを使用して深度を決定する機能を追加しました。Microsoft Speech APIは、管理インターフェースとして使用されます。

受容体




食品を検出し、見つかった材料を含む完成した料理のレシピを選択します。彼らはタスクを恐れず、1週間、自分で十分な数の画像をマークアウトし、TensorFlow Object Detection APIを使用して検出器をトレーニングし、レシピを見つけるロジックを追加しました。シンプルで上品！

エディター2.0




プロジェクト参加者は、特定の人物がいる長いビデオでフラグメントを検索するタスクの一部として、顔認識にニューラルネットワークのセット（顔検索、キーポイントによる顔画像の正規化、顔画像記述子の計算）を使用しました。開発されたシステムは、ビデオ編集のヘルプシステムとして使用でき、必要なフラグメントを探してビデオを自分で見る必要がなくなります。チームはOpenVINOモデルライブラリのニューラルネットワークを使用して、高いアプリケーション速度を達成しました。インテルCore i5プロセッサーを搭載したラップトップでは、ビデオ処理速度は58フレーム/秒でした。

アノニマイザー




人の顔にメガネとマスクを描く。顔と重要なポイントを検出するために、MTCNNネットワークが使用されました。

匿名の




人格の隠蔽のトピックに関する別の興味深い作品。このチームは、ぼかしやピクセレーションなど、いくつかの顔の歪みオプションを導入しました。1週間、彼らはタスクを理解しただけでなく、特定の人物に匿名化モードを提供しました（顔認識付き）。

準備し始める


「ウォームアップ」プロジェクトチームは、頭を傾けるためのスポーツアシスタントを作成する問題を解決しました。そして、このアプリケーションの最終的なアプリケーションがまだ物議を醸している場合でも、さまざまな顔検出アルゴリズムを比較する広範な調査が行われました：Haarカスケード、TensorFlow、OpenCV、OpenVINOのネットワーク。体だけでなく精神的にもウォームアップ！

800以下




学校があったニジニ・ノヴゴロドは、2年で800年の歴史を迎えます。つまり、興味深いプロジェクトを実施するのに十分な時間があります。私たちは子供たちにガイドを作成するタスクについて考えるように促しました。ガイドは、オブジェクトが画像に表されているものと、建物のファサードの画像からそれについてどのような事実がわかるかについての情報を提供できます。私たちの意見では、古典的なコンピュータービジョンに関連するため、このタスクは最も困難なものの1つでしたが、チームはまともな結果を示しました。

最初はグー、じゃんけん


設計作業の実装には厳しい時間制限がありますが、このチームは、有名なゲームで手の位置を分類するために、独自のニューラルネットワークをトレーニングする実験を行うことを恐れていませんでした。
