Djangoでプロジェクトを開始するときに最初に行うこと

コミュニティへの挨拶として、ソーシャルネットワークを作成するという課題に直面しました。そして、私が多かれ少なかれPythonを話すプログラミング言語なので、私はプロジェクトを実装するためにDjangoフレームワークを選びました。HTMLテンプレートとCSSスタイルの作成に時間を費やさないように、私はこれらのファイルをインターネット上で無料とマークしています。これらは主にColorlibのテンプレートです。

行け。

PyCharmを使用していることをすぐに予約するため、manage.pyファイルのすべてのコマンドがわからない場合があります。

他のDjangoプロジェクトと同様に、環境でプロジェクトを作成した後、最初に行うことは、settings.pyファイルに変更を加えることです。

  1. 2つの変数を

    LANGUAGE_CODE = 'ru-ru'
TIME_ZONE = '  / ' #: Europe/Moscow
  2. 静的ファイルとメディアファイル用のフォルダーを追加する

    # Static files
STATIC_URL = '/static/'
STATIC_ROOT = os.path.join(BASE_DIR, 'static')

# Media files
MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

次に、コマンドでアプリケーションを作成します

python manage.py startapp my_app

私の知る限り、Djangoでアプリケーションを作成するための優れた方法は、プロジェクトを個別のアプリケーションに分割することです。そのアプリケーションでは、すでに必要な機能について説明しています。このアプローチは、あるプロジェクトから別のプロジェクトへのアプリケーションの転送を容易にし、非常に便利です。さらに進みましょう。

アプリケーションが作成されたら、使用を開始するためにアプリケーションを登録する必要があります。使用したもののリストにアプリケーションを追加します。

INSTALLED_APPS = [
    'django.contrib.admin',
    ...
    'django.contrib.staticfiles',
    'my_app',# 
]

次に、アプリケーションのURLを決定する必要があります。これも、優れた開発トーンです。アプリケーションフォルダー（プロジェクトフォルダーと混同しないでください）で、最初にurls.pyファイルを作成する必要があります。次に、urls.pyファイルのプロジェクトフォルダーにinclude関数をインポートし、アドレスを指定します。これは、ファイル自体のコメントにも書き込まれます。

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('account.urls'))
]

この時点で、コマンドでサーバーを起動する場合 

python manage.py runserver

コマンドライン（PyCharmの[実行]または[デバッグ]ボタン）では、アプリケーションフォルダーにurls.pyファイル（以下-アカウント/ urls.py）が空であるため、エラーが発生します。

次のコードを入力します。

from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='login'),#     ''( ),    ( Django) log
]

これで、account / views.pyが空であるためエラーが発生します。修理する。

from django.shortcuts import render # 
# Create your views here.
def index(request):#  
    context = {}#          HTML
    return render(request, 'my_app/index.html', context)#   HTML

ほとんど準備ができています。すでに行われていること：

  1. プロジェクトにアプリケーションを作成しました。
  2. 設定を変更しました。
  3. メインのurls.pyプロジェクトのアドレスが登録されます。
  4. urls.pyファイルがアプリケーションで作成され、ビューがそこに書き込まれます。
  5. ビューには、住所のURLとHTMLテンプレートを処理するための関数が作成されています。

テンプレートを作成します。

プロジェクトを作成すると、「templates」フォルダーが自動的に作成されます。その中で、my_appフォルダーを作成する必要があります。my_app内にindex.htmlファイルを作成します。bodyタグをチェックインするには、次のように記述します。

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title></title>
</head>
<body>
    <h2> .</h2>

</body>
</html>

サーバーを起動できます。

その結果、次のようになります（画像）。



サイトでCSSスタイルまたはJavaScriptファイルが使用されており、これらが10ケースのうち10ケースである場合、コマンドを使用してDOCTYPE行の前にそれらをロードする必要があります。

{%load staticfiles%}

この場合、アプリケーションフォルダーに「静的」フォルダーを作成し、CSSやJSなどを内部に保持します。そして、HTMLテンプレートで、次のように、hrefパラメータを使用してlinkタグでファイルへの絶対パスを指定します。

<link href="{% static 'css/style.css' %}" rel="stylesheet">

合計、ファイルは次のようになります。


{%load staticfiles%}
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title></title>
    <link href="{% static 'css/style.css' %}" rel="stylesheet">
</head>
<body>
    <h2> .</h2>

</body>
</html>

一般に、ここで紹介が終了します。さらに、すべてのアクションは類似しており、本質のみが異なります。アプリケーションのforms.pyファイルにはさまざまなフォーム（HTMLのフォームタグ）があり（このファイルを最初に作成する必要があります）、views.pyで説明されているさまざまなアプリケーションロジックと、models.pyで説明されているデータベース内のモデルがあります。しかし、アクションのアルゴリズムは同じです

  1. 住所を作成します。
  2. ビュー（ロジック）を作成します。
  3. HTMLテンプレートを作成します。
  4. 私たちはすべてを結びつけます。
