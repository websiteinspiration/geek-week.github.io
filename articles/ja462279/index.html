Yandex.Metrica Logs APIから取得したデータに基づく、R言語での行動漏斗の構築

販売目標到達プロセスの分析は、インターネットマーケティング、特にeコマースの典型的なタスクです。その助けを借りて、次のことができます。


  • 潜在的な顧客を失っている購入へのステップのいずれかを見つけます。
  • 購入の途中で各ステップが拡大した場合に、追加の収入の流入量をシミュレートするため。
  • さまざまな広告プラットフォームで購入したトラフィックの品質を評価します。
  • 各マネージャーの着信アプリケーション処理の品質を評価します。

この記事では、R言語でYandex Metrics Logs APIからデータをリクエストし、それらに基づいてファネルを作成して視覚化する方法について説明します。


R言語の主な利点の1つは、その基本的な機能を拡張する多数のパッケージが存在することです。この記事では、パッケージを検討するrym、funneljoinとggplot2。


rymこれを使用して、Logs APIからデータをロードfunneljoinし、行動ファネルを構築してggplot2、結果を視覚化します。

my_funnel %>%
  mutate(padding = (sum(my_funnel$nb_step) - nb_step) / 2) %>%
  gather(key = "variable", value = "val", -moment_type) %>%
  filter(variable %in% c("nb_step", "padding")) %>%
