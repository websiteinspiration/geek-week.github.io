RealWorld：aiohttp、Tortoise ORM

Real Worldでのaiohttpの例はなく、私はそれを作ることにしました。経験豊富な開発者は、これを行う時間がないようです、そしてaiohttpの初心者はそれを正しく行う方法を理解していません。Tortoise ORMで始めました。認証を開始している間。


私はこのプロジェクトを正しくしたいので、経験豊富なaiohttp開発者には多くの質問があります。


プロジェクトのインストール


リポジトリー


リポジトリのクローンを作成します。


git clone git@github.com：nomhoi / aiohttp-realworld-example-app.git

開発では、Python 3.8を使用しました。依存関係をインストールします。


pip install -r requirements.txt

データベースを初期化します。


python init_db.py

サーバーを起動します。


python -mコンジット

リポジトリには、PostmanコレクションRealWorld.postman_collection.jsonがあります。それをPostmanにインポートすると、結果のAPIを試すことができます。


PostgreSQL


デフォルトでは、SQLiteデータベースが使用されます。


PostrgeSQLを使用するには、/ conduit / settings.pyのDB_URL変数を変更する必要があります。


DB_URL = "postgres：// postgres：postgres@0.0.0.0：5432 / postgres"

データベースサービスを開始します。


docker-compose up -d

その後、データベースを初期化します。


python init_db.py

Tortoise ORM


class TimestampedMixin:
    created_at = fields.DatetimeField(auto_now_add=True)
    updated_at = fields.DatetimeField(auto_now=True)

class AbstractBaseModel(Model):
    id = fields.IntField(pk=True)

    class Meta:
        abstract = True

class User(TimestampedMixin, AbstractBaseModel):
    username = fields.CharField(db_index=True, max_length=255, unique=True)
    email = fields.CharField(db_index=True, max_length=255, unique=True)
    password = fields.CharField(max_length=128)

    class Meta:
        table = "user"

    def __str__(self):
        return self.username
