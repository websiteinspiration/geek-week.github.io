Javaファイルの上書き


プロジェクトを開発するとき、その実行中にアプリケーションによって作成されたファイルを削除することが必要になりました。しかし、ファイルがセッションの終わり、PCの作業ではなく、ユーザーの要求で削除されることが必要でした。

そして問題ないようです。Java標準ライブラリのjava.io.Fileクラスには、ファイルを削除するためのdelete（）メソッドがあります。

File file = new File("path/to/file");

if (file.delete()) {  
	System.out.println(file.getName() + " deleted");   
} else {
	System.out.println(file.getName() + " not deleted");   
}

OSによっては、java.io.Fileクラスのdelete（）メソッドが内部でネイティブ関数を呼び出し、ファイルを削除します。また、最近のオペレーティングシステムでは、ファイルを削除してもすぐにはファイルが削除されず、ファイル名だけが削除されます。ファイルの内容は残り、削除済みのファイルが使用していたメモリは将来再利用される可能性があります。しかし、すべて同じですが、しばらくの間、すでに削除されたファイルが使用可能であるように思われます。

インターネットを見ると、削除されたファイルを復元するためのプログラムがたくさんあります。たとえばRecuvaです。

しかし、私は将来的にそれらを復元する可能性なしにファイルが削除されることを望みます。インターネットの検索を開始すると、復元（上書き）せずにファイルを削除することは簡単な作業ではありません。また、このようなタスクを実装する場合、特定のOSでファイルを操作する機能を考慮する必要があります。同時に、手動で作成したネイティブAPIまたはなんらかのネイティブライブラリを使用する必要があります。

アプリケーションがUbuntuで開発されたため、このOSはコマンドラインユーティリティの形式で多くの既製のソリューションを提供します。たとえば、secure-deleteユーティリティを使用すると、さまざまな方法で復元せずにファイルを削除できます。

$ srm -vz private/*

アプリケーションは、ユーティリティがインストールされているかどうかを確認し、見つからない場合はエラーを表示する必要があります。ただし、別のOSで使用する場合は、同様のユーティリティを使用する必要があります。

これはあまり便利ではないので、これらの問題を回避したいと思います。secure-deleteユーティリティのソースコードを見ると、さまざまなオペレーティングシステムで作業することができます。それはC99で書かれており、さまざまな前処理マジックとプラットフォーム固有のAPIを使用しています。エラーが発生した場合にそのようなネイティブコードをデバッグすることは非常に難しく、それがタスクです。

secure-deleteユーティリティがどのように機能するかを理解している場合は、次の手順を区別できます。

  • 最初に、ファイルが存在するかどうか、および権限が正しいかどうかを確認します。
  • 指定されたアルゴリズムに応じて、ファイルの内容を上書きします。
  • ファイルサイズをゼロバイトに減らします。
  • ファイルの名前をランダムな文字シーケンスに変更します。
  • ファイルを削除します。

secure-deleteを使用すると、さまざまなアルゴリズムでファイルの内容を上書きできます。

  • 単純なアルゴリズム-1を0x00バイトで上書きします。
  • DOEアルゴリズム-ランダム、ランダム、「DoE」の3つのパスで上書きします。
  • RCMPアルゴリズム-3つのパスで上書き0x00、0xFF、「RCMP」。
  • OPENBSDアルゴリズム-3パスで上書き0xFF、0x00、0xFFバイト。
  • DODアルゴリズム-7パスを上書きします。
  • グットマンアルゴリズム-35パスを上書きします。

コードをプラットフォームに依存せず、さまざまなオペレーティングシステムで動作させたい。最新のC ++を見ると、Secure-Deleteがファイルを上書きするために行うすべての手順を実行できます。

ファイルが存在するかどうか、および正しい権限があるかどうかを確認するには、C ++ 17で追加されたstd :: filesystemを使用できます。

ファイルの内容を上書きするには、選択したアルゴリズムに応じて、実装をsecure-deleteのままにすることができます。

bool kl::erase_content::overwrite() {
	switch (entry.get_mode()) {
	case kl::overwrite_mode::SIMPLE_MODE:
		if (!overwrite_byte(1, 0x00)) { return false; }
		break;
	case kl::overwrite_mode::DOE_MODE:
		if (!overwrite_random(1)) { return false; }
		if (!overwrite_random(2)) { return false; }
	        if (!overwrite_bytes(3, "DoE")) { return false; }
		break;
	case kl::overwrite_mode::OPENBSD_MODE:
		/* override OPENBSD_MODE method */
		break;
	case kl::overwrite_mode::RCMP_MODE:
		/* override RCMP_MODE method */
		break;
	case kl::overwrite_mode::DOD_MODE:
		/* override DOD_MODE method */
		break;
	case kl::overwrite_mode::GUTMAN_MODE:
		/* override GUTMAN_MODE method */
		break;
	default:
		std::cerr << "overwrite mode doesn't choose" << std::endl;
	}

	return true;
}

アルゴリズムに応じて、特定のサイズのバッファーは特定のデータセットで満たされ、最後に達するまでこのバッファーをファイルに書き込みます。

bool kl::erase_content::overwrite_byte(const int pass, const uint8_t byte) {
	const auto& [file_name, file_size, buffer_size, mode] = entry;

	this->buffer = std::make_unique<uint8_t[]>(buffer_size);
	std::memset(buffer.get(), byte, buffer_size);

	this->file = kl::fs_util::make_open_file(file_name, "r+b");

	if (!overwrite_data(pass)) {
		return false;
	}

	return true;
}
	
bool kl::erase_content::overwrite_data(const int pass) {
	const auto& [file_name, file_size, buffer_size, mode] = entry;

	const size_t count = file_size / buffer_size;
	const size_t tail  = file_size % buffer_size;
	size_t writted = 0;		

	if (fseek(file.get(), 0, SEEK_SET) != 0) {
		std::cerr << "couldn't seek in file" << std::endl;
		return false;
	}

	writted = write_buffer(count, tail);

	if (writted != file_size) {
		std::cerr << "couldn't write buffer in file" << std::endl;
		return false;
	}

	fflush(file.get());

	if (fseek(file.get(), 0, SEEK_SET) != 0) {
		std::cerr << "couldn't seek in file" << std::endl;
		return false;
	}

	file.reset();

	return true;
}

ネイティブ実装は標準C ++ライブラリのみを使用するため、他のプラットフォームへの移植が容易になります。そして最も重要なのは、エラーが発生した場合にデバッグするのがそれほど簡単ではない、プリプロセッサマクロマジックがないことです。

標準のC ++ 17は、MVSC、Clang、GCCなどのすべての一般的なコンパイラですでにサポートされています。
