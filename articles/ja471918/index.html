SwiftUI：知人

このチュートリアルでは、Viewを使用してUIアプリケーションを計画する方法と、State変数を使用してUIを変更する方法を学習します。

推定読書時間：25分。

SwiftUIを使用すると、インターフェイスビルダー（IB）とストーリーボードを完全に忘れることができます。IBとXcodeはXcode 4より前の個別のアプリケーションでしたが、IBActionまたはIBOutletの名前を編集するとアプリケーション間のクラッシュが発生するため、IBはコードの変更について何も知りません。または、セグエまたはテーブルセルに識別子を設定したが、文字列であるため、Xcodeがそれらをチェックできない場合。

SwiftUIが助けになります！コードを入力するとすぐに、ビューの変更がすぐにわかります。一方の変更は他方の更新につながるため、常に関連しています。誤っている可能性のある文字列識別子はありません。これがすべてのコードですが、UIKitを使用して記述した場合よりもはるかに小さいため、理解、編集、デバッグが簡単です。教えてください、それは素晴らしいことではありませんか？

変数「@State」の使用


SwiftUIでは、通常の変数を使用できますが、変数の変更をUIに反映させる場合は、変数に'@State'のようなマークを付けます。このアプリケーションでは、色を選択しているため、選択した色に影響を与えるすべての変数は「@State」変数です。body宣言の前に、

次の行をstruct ContentView内に追加します。

let rTarget = Double.random(in: 0..<1)
let gTarget = Double.random(in: 0..<1)
let bTarget = Double.random(in: 0..<1)
@State var rGuess: Double
@State var gGuess: Double
@State var bGuess: Double

R、G、Bの値は0から1の間にあります。目的の値をランダムな値で初期化します。選択した値を0.5に初期化することもできますが、この場合は何をする必要があるかを示すため、ここでは初期化しないでください。

さんがに下少し行こう構造体ContentView_Previews ContentViewインスタンスを初期化し、プレビューのために。これで、イニシャライザは選択した値の初期値を必要とします。ContentView（）を次のように変更します。

ContentView(rGuess: 0.5, gGuess: 0.5, bGuess: 0.5)

スライダーを作成すると、プレビューではそれらの値が中央に表示されます。

また、シーン関数（_：willConnectTo：options :)のSceneDelegateの初期化子を修正する必要があります-ContentView（）を次のように置き換えます。

window.rootViewController = UIHostingController(rootView:
  ContentView(rGuess: 0.5, gGuess: 0.5, bGuess: 0.5))

アプリケーションが読み込まれると、スライダーポインターが中央に表示されます。

次に、ターゲットの四角形に色修飾子を追加します。

  .foregroundColor(Color(red: rTarget, green: gTarget, blue: bTarget, opacity: 1.0))

.foregroundColor 修飾子は、ランダムに生成されたRGB値によって指定された色で新しい四角形ビューを作成します。

アラートを表示


スライダーを正しい位置に置いた後、ユーザーはHit Meボタンを押します。その後、概算でアラートが表示されます。

最初に、スコアを計算するメソッドをContentViewに追加します。変数 '@State'と
本体の間に、このメソッドを追加します。

func computeScore() -> Int {
  let rDiff = rGuess - rTarget
  let gDiff = gGuess - gTarget
  let bDiff = bGuess - bTarget
  let diff = sqrt(rDiff * rDiff + gDiff * gDiff + bDiff * bDiff)
  return Int((1.0 - diff) * 100.0 + 0.5)
}

差分 値は、3次元空間内の2点間の距離、つまりユーザーのエラーの値です。見積もりを取得するには、diffを1から差し引き、その値を0から100の範囲に減らします。diffが小さいほど、見積もりは高くなります。

次に、Text（ "Hit me button"）スタブを次のコードに置き換えます。

Button(action: {

}) {
  Text("Hit Me!")
}

ボタンには、UIButtonのようにアクションとラベルがあります。アクションでアラートビューをトリガーする必要があります。ただし、アクションボタンでアラートを作成しても、何も起こりません。

代わりに、ContentViewのAlert部分を作成し、Boolタイプの '@State'変数を追加します。次に、アラートをアクションボタンに表示する場所で、この変数をtrueに設定します。ユーザーがアラートを非表示にすると、値はfalseにリセットされ、アラートが非表示になります。

この「@State」変数を追加します。

@State var showAlert = false

次に、このコードをアクションボタンとして追加します。

self.showAlert = true

showAlertはクロージャーの中にあるので、私たちは自分自身が必要です。

最後に、アラート修飾子をボタンに追加すると、ボタンは完全に次のようになります。

Button(action: {
  self.showAlert = true
}) {
  Text("Hit Me!")
}
.alert(isPresented: $showAlert) {
  Alert(title: Text("Your Score"), message: Text("\(computeScore())"))
}

$ showAlertをバインディングとして渡します。これは、ユーザーがアラートを非表示にした瞬間にこの変数の値が変更され、この変更によりビューが更新されるためです。

SwiftUIには、アラートビュー用の単純な初期化子があります。デフォルトでは[OK]ボタンがあるため、パラメータとして設定する必要もありません。

ライブプレビューをオンにして、スライダーを動かし、[ヒット]ボタンを押します。出来上がり！
