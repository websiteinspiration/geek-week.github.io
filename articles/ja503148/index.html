Lynisユーティリティを使用したLinuxセキュリティの確認

Linuxシステムの監査に最も強力なフリーウェアユーティリティの1つを使用する方法を理解しています。


Linuxシステムがどれほど安全かを考えたことはありますか？多くのLinuxディストリビューション（それぞれ独自のデフォルト設定を使用）があり、バージョン番号の異なる数十のソフトウェアパッケージを実行できます。また、バックグラウンドのOSには、知らない、または考えていないサービスがたくさんあります。

システムに関する情報を取得するには、つまり、Linuxがインストールされているコンピューターで実行されているソフトウェア、ネットワーク、サービスの一般的なステータスを確認するには、いくつかのコマンドを実行します。しかし今、受け取った情報をどうするか？それを理解するには？結局のところ、分析する必要があるデータの量は膨大です。

システムのセキュリティレベルに関する構造化されたレポートを生成するツールを実行できれば、はるかに優れています。Lynisは、LinuxおよびUnixベースのシステムを監査する人気のあるオープンソースツールです。

「セキュリティ設定の包括的なスキャンを実行し、システム自体で実行されます。主な目標は、セキュリティ機能をテストし、システムセキュリティのレベルをさらに向上させるための推奨事項を発行することです。また、システム全体の情報、インストール済みパッケージに関する情報、および考えられる構成エラーもスキャンします。Lynisは[通常]システム管理者とセキュリティ専門家がシステムセキュリティのレベルを評価するために使用します。」とプロジェクトのWebサイトは述べています。

Lynisをインストールする


Lynisをインストールするには、パッケージマネージャーを使用できます。リポジトリ内のバージョンに関連するLinuxシステムが適切であれば、ユーティリティは次のようにインストールできます。

dnf install lynis

とか、ぐらい：

apt install lynis

ただし、リポジトリのユーティリティバージョンが最新でない場合は、GitHubを使用してLynisをインストールすることをお勧めします（私はRed Hat Linuxシステムを使用していますが、同様のコマンドがどのLinuxディストリビューションでも機能します）。

Lynisを起動する


-h フラグを使用してヘルプを実行し、Lynisを始めましょう。

$ ./lynis -h

そのため、メインユーティリティコマンドに慣れることができます。たとえば、そのバージョンを調べてみましょう。

$ ./lynis show version
3.0.0
$

Lynisコマンドの完全なリストを表示するには、次のように入力します。

$ ./lynis show commands

Commands:
lynis audit
lynis configure
lynis generate
lynis show
lynis update
lynis upload-only

$

Linuxシステムの監査の実行


それはすべて次のコマンドで始まります：

$ ./lynis audit system

チームを開始した後、詳細なレポートを受け取ります。心配しないでください、後でわかります。他のチームワークの結果も保存されるので、後でそれらに戻ることができます。

ここでLynisはログを保存します：

  Files:
  - Test and debug information      : /var/log/lynis.log
  - Report data                     : /var/log/lynis-report.dat

必要なすべてのファイルが実際に作成されたかどうかを確認できます。

$ ls -l /var/log/lynis.log
-rw-r-----. 1 root root 341489 Apr 30 05:52 /var/log/lynis.log
$
$ ls -l /var/log/lynis-report.dat
-rw-r-----. 1 root root 638 Apr 30 05:55 /var/log/lynis-report.dat
$

より明確にするために、Lynisは色の規則を使用しています。

  • 緑：すべて順調
  • 黄色：必要なものがすべて見つからなかったか、物議を醸す状況が発生した
  • 赤：これは問題なので、対処する価値があります

私の場合、ほとんどの問題（赤いマーク）はカーネルの強化セクションで見つかりました。カーネルには、その動作を決定するさまざまな構成可能なパラメーターがあり、これらのパラメーターの一部はセキュリティを担っています。ディストリビューションは、さまざまな理由でデフォルトでそれらをインストールしない場合がありますが、それぞれを調べて、セキュリティ要件に応じてその値を変更する必要があるかどうかを確認する必要があります。

レポートの最後に、Lynisは監査結果に基づいて修正を行います。各文は説明で始まり、それを生成したテスト（TEST-ID）はその横の括弧内に示されています。次の行は、問題が存在する場合、その解決策を提案します。

Suggestions (47):
  ----------------------------
  * If not required, consider explicit disabling of core dump in /etc/security/limits.conf file [KRNL-5820]
      https://cisofy.com/lynis/controls/KRNL-5820/

  * Check PAM configuration, add rounds if applicable and expire passwords to encrypt with new values [AUTH-9229]
      https://cisofy.com/lynis/controls/AUTH-9229/

Lynisでは、show detailsコマンドを使用し、その後にTEST-IDテスト番号を指定することで、各修正案に関する追加情報を見つけることができます。

./lynis show details TEST-ID

たとえば、SSH-7408テストの詳細情報を見つけるには、$ ./lynis show details SSH-7408と入力して、次の情報を取得します。

Linuxシステムのセキュリティについて詳しく知りたい場合は、Lynisをチェックしてください。そして、このユーティリティがどのように機能するか、すべての情報をどのように収集するかについて詳しく知りたい場合は、シェルスクリプトを調べてください。他のユーティリティと同様に、まず仮想マシンで試してみるのが理にかなっています。
