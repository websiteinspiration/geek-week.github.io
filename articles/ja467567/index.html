一連のssh + tmux + neovimのターミナルでの24ビットカラーのサポート



Windowsでは、私は常にSecureCRTを使用してsshで作業し、悲しみを知りませんでした。しかし、私は、24ビットのターミナルを試して、リモートサーバー上のvimの美しいテーマを使用してsshで作業する必要があることを知りました。

ステップ1. 24ビットの端末プログラム


残念ながら、現時点ではSecureCRTは24ビット画面をサポートしていません。フォーラムでスレッドのみがハングし、遠い将来のサポートが追加されます。

それで私は検索してみたところ、この件に関してそのようなgitが見つかりました。True ColorをサポートするLinux / Mac / Windows用のさまざまな端末のリストがあります。繰り返し、好きなものを見ることができます。私はWindowsのMobaXtermに落ち着きましたが、SecureCRTの構成が選択されても、通常のSecureCRTに少し似ています。ポータブルバージョンがあり、ここからダウンロードできます

。mobaxterm.mobatek.net

開始後、次のコマンドを入力すると、

lscolors

記事の冒頭のスクリーンショットのように、テストプレートが表示されます。

ステップ2. NeoVim


次に、私はMobaXtermのターミナルを使用して、ssh経由で接続するリモートマシンにneovimをインストールしました。

ここにneovimのインストール手順があります：

github.com/neovim/neovim/wiki/Installing-Neovim

そしてvim-plugをインストールしました：

github.com/junegunn/vim-plug

ステップ3. True Colorをサポートするテーマchallenger_deep






True Colorをサポート するテーマchallenger_deepを選択しましたが、個人的には気に入りました。しかし、それらの多くがあり、あなたはあなたの好みに合わせて選択することができます、あなたはTrue Colorサポートを探す必要があります。

そして、neovimの最小構成：

call plug#begin('~/.local/share/nvim/plugged')
Plug 'vim-airline/vim-airline'
Plug 'itchyny/lightline.vim'
Plug 'challenger-deep-theme/vim'
call plug#end()

syntax on
filetype on
filetype plugin on
filetype plugin indent on
set termguicolors
colorscheme challenger_deep

ファイル〜/ .config / nvim / init.vimは、クラシックvimの.vimrcに類似しています。

さらに：

:PlugInstall

そして、実際には、すべてがスクリーンショットのようになるはずです。

ステップ4. tmuxでの色のサポート


デフォルトでは、tmuxセッション（画面のアナログ）を開始すると、色はフェードし、通常の退屈なコースに戻ります。これを無効にして配色を返す方法を次に示します。

以下の内容で〜/ xterm-24bit.terminfoファイルを作成します。


xterm-24bit|xterm with 24-bit direct color mode,
   use=xterm-256color,
   sitm=\E[3m,
   ritm=\E[23m,
   setb24=\E[48;2;%p1%{65536}%/%d;%p1%{256}%/%{255}%&%d;%p1%{255}%&%dm,
   setf24=\E[38;2;%p1%{65536}%/%d;%p1%{256}%/%{255}%&%d;%p1%{255}%&%dm,

ファイルの最後に空の行があることを確認してください。

次のコマンドを実行します。


tic -x -o ~/.terminfo xterm-24bit.terminfo

そしてさらに：

export TERM=xterm-24bit

.bashrcに置くことができます
。.tmux.conf構成で、以下を追加します。


set -g default-terminal "xterm-24bit"
set -g terminal-overrides ',xterm-24bit:Tc'

tmuxを起動してお楽しみください！
