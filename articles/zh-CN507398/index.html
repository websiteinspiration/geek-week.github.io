寻找APT30：经过数年的沉默，我们注意到该小组的一项新活动







网络组织APT30知名度已经有一段时间了-2015年，我们来自FireEye的同事描述了它的活动。该小组的成员通常攻击南亚和东南亚的政府机构（印度，泰国，马来西亚和其他国家），以进行网络间谍活动，其工具至少是从2005年开始开发的。



在过去的几年中，APT30并不十分活跃，但是在2020年春季，我们能够检测到由黑客开发的新恶意软件的痕迹。完整的研究仅供参考，在本文中，我们将探讨其要点。



介绍



2020年4月8日，我们来自PT专家安全中心的专家发现了一个著名的网络犯罪集团的活动。流行的用于动态恶意软件分析的资源触发了描述APT30活动的网络签名，有关APT30的活动已很久没有听到。这是我们研究的起点。



事实证明，黑客不仅继续支持其十年前的旧工具，而且还坚持其组织网络资源的方法。



BACKSPACE和NETEAGLE后门



2019年8月25日，来自马来西亚的名为AGENDA.scr 的文件（MD5：f4f8f64fd66a62fc456da00dd25def0d）被上传到VirusTotal 。这是UPX打包的基于x86的可执行PE文件。该示例具有一个办公室文档图标以欺骗用户，并且资源包含两个以上的加密对象。







第一个文件（MD5：634e79070ba21e1e8f08aba995c98112）使用Microsoft Office模板- ％APPDATA％\ Microsoft \ Windows \ Templates \ AGENDA.docx写入目录，然后启动。这是一份办公文件，其中包含在马来西亚政府一个部门中开会的计划。该文件的目的是引起用户的注意。







第二个文件（MD5：56725556d1ac8a58525ae91b6b02cf2c）位于启动目录中％APPDATA％\ Microsoft \ Windows \开始菜单\程序\ Startup \ WINWORD.EXE。在创建时，该文件不会启动（攻击者希望它在重新启动系统后起作用，例如，以免引起注意）。这是NETEAGLE系列的后门，在FireEye同事的报告中详细讨论了其修改。奇怪的是，2015年的样本中有特色的NetEagle系列（用作恶意软件家族的名称）现在被JokerPlay取代了。







2015年样本中的NetEagle字符串JokerPlay







样本中的2019年字符串



根据获得的指标，发现了另外两个后门（MD5：d9c42dacfae73996ccdab58e429548c0和MD5：101bda268bf8277d84b79fe52e25fee4）。根据编译日期，它们是在2019年10月21日创建的，其中一个也从2020年5月才从马来西亚上传到VirusTotal。这些恶意软件属于BACKSPACE家族，FireEye专家的报告中也考虑了对它们的修改。我们给出了每个样本的解密行以及解码算法。



后门RHttpCtrl



表示一个基于x86的可执行PE文件。首先，恶意软件尝试提取注册表分支HKCU \ Software \ HttpDiv的随机密钥的值。如果失败，则使用WinAPI函数GetSystemTimeAsFileTime，将获得系统时间-然后用作任意数字的种子生成器。生成的随机数将存储在注册表中，并在以后使用。



通过对地址hxxp的GET请求：//www.kabadefender.com/plugins/r.exe，该恶意软件可以接收并存储合法的WinRAR存档器（其CLI组件4fdfe014bed72317fa40e4a425350288）。然后，它创建系统的指纹，收集计算机名称，IP地址和系统版本，并将POST请求发送到该地址hxxp://www.kabadefender.com/clntsignin.php



。



后门RCtrl



它是使用MFC库开发并使用UPX打包的基于x86的可执行PE文件。



该恶意软件使用数字0x23的单字节XOR解密主攻击者服务器的地址：103.233.10 \ .152。通过端口4433上的TCP转向他，检查连接是否成功。如果连接失败，它将使用其他数据来获取服务器地址。



其他数据是地址，hxxp://www.gordeneyes.com/infos/p



并hxxp://www.techmicrost.com/infos/p



以数字0x25在单字节XOR中编码。解密地址后，恶意软件会尝试通过GET请求交替连接它们。服务器响应期望8个字节：服务器IP地址和端口。







获取攻击者服务器的地址：'0xAC 0xF7 0xC5 0xBD'→'172247197189'，'0xBB 0x01 0x00 0x00'→0x1BB→443



收到攻击者服务器的有效IP地址后，恶意软件会重新连接到服务器，并期望从中接收到Jo * Po * Hello字符串。此字符串使用单字节XOR（数字0x24）编码在恶意软件的主体中。一个奇怪的窍门：通常，木马本身会启动数据交换。



如果收到该行，则会创建系统指纹：系统版本，IP地址，有关处理器品牌和频率的信息以及磁盘空间量。收集的数据使用基于位循环移位和XOR（即左移4 + 3 = 7位，XOR为0x23）的独特算法加密，然后发送到攻击者服务器。



然后创建一个单独的流，每30秒将相同的数据缓冲区发送到服务器。接下来，控件接收命令处理功能，该功能对接收到的数据进行解密（解密算法由与所考虑的加密算法相反的动作组成）并检索命令编号。



结论



这并不是说攻击者创建的软件以高级程序代码，某些技巧隐藏在系统中或绕过检测而著称。目标似乎并没有改变：也许它们不需要更复杂的恶意武库。在谈到新工具包时，我们注意到了它的不完整。也许该小组正在战斗中测试新软件，以发现缺陷。因此，将来我们应该期望出现改进的后门RHttpCtrl和RCtrl，可能使用隐藏或使分析复杂化的技术。



完整的报告可在此处获得。
