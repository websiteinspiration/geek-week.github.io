サイプレス+ストーリーブック。テストケース、データ、コンポーネントのレンダリングを1か所に保存

当初、サイプレスはe2eテストツールとして認識されていました。Seleniumが一生を統治するトピックで、フロントエンドエンジニアの関心の高まりを観察することは興味深いことでした。当時、サイプレスの機能を説明する一般的なビデオまたは記事は、ランダムに選択されたサイトとデータ入力用のAPIに関するお世辞に値するレビューをさまようことに限定されていました。


私たちの多くは、Cypressを使用して、Storybook / Styleguidist / Doczなどの環境によって提供されるコンポーネントを分離してテストすることを推測しています。良い例は、Stefano Magniの記事「サイプレスとストーリーブックを使用した仮想リストコンポーネントのテスト」です。 Storybook Storyを作成してコンポーネントを配置し、テストに役立つグローバル変数データを配置することを提案します。このアプローチは適切ですが、テストはStorybookとCypressの間で引き裂かれています。多くのコンポーネントがある場合、そのようなテストは読み取りや保守が困難になります。


この記事では、もう少し先に進み、サイプレスでJavaScriptを実行する機能を最大限に活用する方法を紹介します。動作を確認するには、アドレスからソースコードをダウンロードし、npm iおよびnpm run testコマンドを実行してください。


tl; dr：


  • ウィンドウにStorybook Storyのコンポーネントへのリンクを配置して、サイプレスで全体をテストできます（テストロジックをいくつかの部分に分解することなく）。
  • サイプレスは私たちのチームにとって非常に強力であるように見え、UIコンポーネントをテストするために内部でjs-domを使用するツールを完全に放棄しました。
