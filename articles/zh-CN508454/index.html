让我们从她解决的问题开始：



  1. 没有默认的layoutManager-这很不方便，因为您通常必须选择相同的LinearLayoutManager。
  2. 无法通过xml添加分隔符和项目空间-这也很不方便，因为您需要添加到项目布局或通过ItemDecorator；
  3. 您不能简单地通过xml添加页眉和页脚-这只能通过单独的ViewHolder进行。


这些问题并不重要，但会带来不便，并增加了开发时间。



1.问题：没有默认的layoutManager



RecyclerView的开发人员无法选择默认的LayoutManager。您可以通过以下方式设置layoutManager：



1.通过XML在app属性中：layoutManager =“ LinearLayoutManager”： 



<?xml version="1.0" encoding="utf-8"?>
<androidx.recyclerview.widget.RecyclerView
   ...
    app:layoutManager="LinearLayoutManager"/>


2.通过代码： 



recyclerView.layoutManager = LinearLayoutManager(this)


根据我们的经验，在大多数情况下，需要LinearLayoutManager。



以下是我们的ITProTV，Simple World和Dexen应用程序中此类清单的一些示例：







解决方案：添加默认的layoutManager



仅3行被添加到OmegaRecyclerView： 



 if (layoutManager == null)  {
            layoutManager = LinearLayoutManager(context, attrs, defStyleAttr, 0)
 }


因此，当需要LinearLayoutManager时，则无需添加任何内容，也就是说，您可以忽略layoutManager。 



<?xml version="1.0" encoding="utf-8"?>
<com.omega_r.libs.omegarecyclerview.OmegaRecyclerView
    android:id="@+id/recyclerview"
    android:layout_width="match_parent"
    android:layout_height="match_parent" />


2.问题：无法通过xml添加分隔符和项目空间



使用RecyclerView时，您必须经常添加分隔线。例如，在“简单世界”项目中，其中一个屏幕带有这样的非标准分隔线：







从这种布局可以看出：



  • 在元素之间和最后使用了分隔线；
  • 使用的项目空间。


如何以标准方式在Android中实现？



方法一



最明显的方法是将除法器包含为ImageView元素： 



 <RelativeLayout
   ...
   android:paddingStart="20dp"
   android:paddingTop="12dp"
   android:paddingEnd="20dp"
   android:paddingBottom="12dp">

...

   <ImageView
       ...
       android:layout_alignParentBottom="true"
       android:src="@drawable/divider"/>

</RelativeLayout>


可能需要仅在元素之间进行除法。在这种情况下，您将必须删除最后一个分隔线并添加代码以将其隐藏在适配器中。



方法2



另一种方法是使用该分隔线可以绘制的DividerItemDecoration。为此，您必须另外创建一个可绘制对象： 



<?xml version="1.0" encoding="utf-8"?>
<layer-list xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:left="32dp">

        <shape android:shape="rectangle">

            <size
                    android:width="1dp"
                    android:height="1dp" />

            <solid android:color="@color/gray_dark" />

        </shape>

    </item>

</layer-list>


要添加缩进，您需要编写ItemDecoration： 



class VerticalSpaceItemDecoration(private val verticalSpaceHeight: Int): RecyclerView.ItemDecoration {

    override fun getItemOffsets(outRect: Rect, view: View, parent: RecyclerView,
            state: RecyclerView.State) {
        outRect.bottom = verticalSpaceHeight
    }

}


DividerItemDecoration很简单：它总是在每个列表项下绘制一个分隔线。

但是，如果需求发生变化，则必须寻找另一种解决方案。



解决方案：添加通过XML添加分隔符和项目空间的功能



因此，我们的OmegaRecyclerView应该能够使用以下属性添加分隔线：



  1. divider – drawable, ;
  2. dividerShow (beginning, middle, end) – , , ;
  3. dividerHeight – divider, ;
  4. dividerPadding, dividerPaddingStart, dividerPaddingEnd – : , , ;
  5. dividerAlpha – ;
  6. itemSpace – .


所有这些属性都使用两个特殊的ItemDecoration直接应用于我们的OmegaRecyclerView。



其中一个ItemDecoration在项目之间添加填充，第二个绘制分隔符本身。请记住，如果元素之间存在缩进，则第二个ItemDecoration会在缩进的中间绘制一个分隔线。无法支持所有可能的LayoutManager选项，因此我们仅支持LinearLayoutManager和GridLayoutManager。还要注意LinearLayoutManager的列表方向。



为了简化代码，我们选择一个特殊的DividerDecorationHelper，它将根据方向和顺序（反向或直接）在Rect中读取和写入相关数据。它将具有方法setStart，setEnd和getStart，getEnd。



让我们为两个类创建一个基本的ItemDecoration，其中将包含常规逻辑：



  1. 检查layoutManager是LinearLayoutManager的后代；
  2. 计算当前方向和顺序；
  3. 确定适当的DividerDecorationHelper。


在SpaceItemDecoration中，我们仅覆盖一个getItemOffset方法，该方法将添加缩进： 



override fun getItemOffset(outRect: Rect, parent: RecyclerView, helper: DividerDecorationHelper, position: Int, itemCount: Int) {
        if (isShowBeginDivider() || countBeginEndPositions <= position) helper.setStart(outRect, space)
        if (isShowEndDivider() && position == itemCount - countBeginEndPositions) helper.setEnd(outRect, space)
    }


下一个DividerItemDecoration将直接绘制分隔线。它应该考虑元素之间的缩进，并在中间绘制分隔线。首先，针对未指定缩进但绘制需要分隔线的情况，重新定义getItemOffset方法。 



    override fun getItemOffset(outRect: Rect, parent: RecyclerView, helper: DividerDecorationHelper, position: Int, itemCount: Int) {
        if (position == 0 && isShowBeginDivider()) {
            helper.setStart(outRect, dviderSize)
        }
        if (position != 0 && isShowMiddleDivider()) {
            helper.setStart(outRect, dividerSize)
        }
        if (position == itemCount - 1 && isShowEndDivider()) {
            helper.setEnd(outRect, dividerSize)
        }
    }


我们还添加了一个选项，该选项允许DividerItemDecoration询问适配器是否可以在所选项目的上方或下方绘制。要实现此功能，请使用以下方法创建从标准适配器继承的自己的适配器： 



   open fun isDividerAllowedAbove(position: Int): Boolean {
        return true
    }

    open fun isDividerAllowedBelow(position: Int): Boolean {
        return true
    }


接下来，重新定义onDrawOver方法以在绘制的元素上方绘制分隔线。在此方法中，您需要遍历屏幕上所有可见的元素（通过getChildAt），并在必要时绘制此分隔线。还应注意，不具有高度的颜色可以来自dividerDrawable属性。对于这种情况，可以从dividerHeight属性获取高度。



3.问题：您不能直接通过xml添加页眉和页脚







在RecyclerView中，无法通过xml添加视图，但是还有其他方法可以做到这一点。



方法一



添加视图的一种显而易见的方法是通过适配器。此外，在输入viewType的标识符时，有必要在适配器的页眉和页脚之间进行区分。 



 fun onCreateViewHolder(parent: ViewGroup, viewType: Int): RecyclerView.ViewHolder? {
	  val inflater = LayoutInflater.from(parent.context)
        return when (viewType) {
		 TYPE_HEADER -> {
                val headerView: View = inflater.inflate(R.layout.item_header, parent, false)
                HeaderViewHolder(itemView)
            }
            TYPE_ITEM -> {
                val itemView: View = inflater.inflate(R.layout.item_view, parent, false)
                ItemViewHolder(itemView)
            }
           
            else -> null
        }
    }


方法2



稍有不同的方式，也可以通过适配器。从recyclerview：1.2.0-alpha02开始，出现了MergeAdapter，它使您可以将多个适配器组合为一个，从而使代码更简洁。 



val mergeAdapter = MergeAdapter(headerAdapter, itemAdapter,  footerAdapter)
recyclerView.adapter = mergeAdapter


解决方案：添加通过XML简单添加页眉和页脚的功能



要做的第一件事是在进行充气过程时在OmegaRecyclerView中拦截视图的添加。为此，请覆盖addView方法并将所有页眉和页脚视图添加到您自己。RecyclerView本身使用此方法来添加可见列表项。但是通过xml添加的视图将没有ViewHolder，这最终将引发NullPointerException。



因此，我们需要确定在充气期间何时添加视图。幸运的是，有一个受保护的onFinishInflate方法，当充气过程结束时会调用该方法。因此，在调用此方法时，我们标记为充气过程已完成。 



  protected override fun onFinishInflate() {
        super.onFinishInflate()
        finishedInflate = true
    }


因此，addView方法将如下所示： 



  override fun addView(view: View, index: Int, params: ViewGroup.LayoutParams) {
        if (finishedInflate) {
            super.addView(view, index, params)
        } else {
            // save header and footer views
        }
    }


接下来，您需要记住所有这些其他视图，并将它们传输到特殊的适配器作为MergeAdapter。



我们还设法解决了另一个问题：当调用findViewById方法时，将不会返回我们的视图。为了解决这个问题，我们重新定义了findViewTraversal方法：有必要比较找到的视图的ID，并在匹配时返回该视图。由于此方法是隐藏的，因此我们仅编写它而不指示它被覆盖。



您可以通过OmegaRecyclerView库中的详细说明来熟悉这些功能和其他有用的功能：



  1. 在2018年，我们从RecyclerView创建了ViewPager。而且，在我们的ViewPager中，有无尽的滚动；
  2. ExpandableRecyclerView-用于添加下拉列表的特殊类，能够选择扩展动画；
  3. StickyHeader是可以通过适配器添加的特定列表项。
