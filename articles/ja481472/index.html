DNSの履歴：ドメイン名が支払われる時期

以前、ドメインネームシステムの履歴を伝え始め、DNSで行われた変更について話しました。次のステップは、ITコミュニティが最良の方法で対応していない変更です。

有料トップレベルドメイン


では最初の部分、私たちは、どのように言及した1983年に、ポール・モカペトリスが提案されている新しいホスト識別子の構造を-名前と特別なカテゴリで。この構造は、最初のドメインが.com、.edu、.net、.org、.int、.gov、.milのgTLD（一般的なトップレベルドメイン）の分類の基礎となりました。その後、.info、.biz、.nameがこのリストに追加されました。

1990年代初頭、Network Solutions Inc.はトップレベルドメインの作成と管理の問題に対処しました。 （NSI）米国政府が採用。これらの目的のために、同社は子会社のInterNICを開設しました。その後、DNSを管理するタスクは、米国国防総省との契約の下で機能するIANA組織に割り当てられました。それは、モカペトリスの同僚であり、最初のDNS仕様の共著者であるジョンポステルが率いるものでした。ジョンの死後、gTLDは別の国際組織であるICANNに移管されました。

最初の数年間、Network Solutionsは米国国防情報システム局（DISA）から資金提供を受けました。しかし1995年、別の政府機関であるアメリカ国立科学財団（NSF）が技術開発を担当し、NSIがドメイン登録料を請求することを許可しました。同社は2年間の所有権ごとに100ドルの価格を設定しました。

最初の代替サーバーはAlterNICで、1995年にエンジニアのEugene Kashpureffによって設立されました。 AlterNICのフレームワーク内でgTLDがリリースされました：.exp、.llc、.lnx、.ltd、.medなど。 1997年、彼はInterNICをハッキングし、リソーストラフィックを自分のサイトに3日間リダイレクトしました。そのような訴訟は、いくつかの訴訟と刑事訴追を伴いました。その結果、AlterNICの創設者が逮捕され、会社は閉鎖されました。

AlterNICに加えて、90年代にはeDNS、Iperdomeなどのルートサーバーがありました。パーソナルドメインサービスとNew.Netは、2002年まで西太平洋で唯一機能していたルートサーバーでした。サービスの創設者たちは、ネットワークソリューションと競争しようとはしませんでしたが、プロジェクトはとにかく閉じられました。

代替サーバー市場は衰退しておらず、今日、そのようなソリューションは進化し​​続けています。たとえば、.thanks、.joy、.wise、.truthなどのドメインを持つdot.loveがあります。単一の.berlinドメインを管理するdotBERLINもあります。オープンコミュニティの参加者もDNSサーバーを構成します。GNUネームシステムは.gnuドメインで機能し、分散型EmerCoinシステムは.coin、.lib、.emc、.bazarを管理します。

代替ルートサーバーの急増にもかかわらず、グローバルな専門家は、その存在はインターネットに有害であると信じています。 IETFのエンジニアは、特別なRFC2826で懸念についても説明しました。彼らは、DNSエコシステムの断片化がリゾルバーの作業を妨げ、ユーザーに必要なリソースの検索を遅くし、セキュリティ更新の配布を困難にしていると指摘しています。

ただし、すべての代替サービスがDNS構造を複雑にするわけではありません。Open Root Server Networkのような、利益指向ではなく、ルートゾーンを追加のトップレベルドメインで拡張しないソリューションがあります。彼らの目標は、インターネットコミュニティのICANNへの依存を減らすことです（現在、ICANNはgTLDを担当しています）。

代替DNSサーバーの状況は、ドメインネームシステムの歴史における「戦争」の一例にすぎません。次回は、DNS機能を拡張するためのEDNS仕様の実装に関する問題と、DoH / DoTへの移行に関する議論について話します。
