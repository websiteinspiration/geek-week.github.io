実装の選択


セグメンテーションに関する新しい情報の流れは圧倒的でした。本能的に、もっと単純なものに追いつきたかった。私は内部の禅がネットワークのアーキテクチャを理解し、学習に時間を費やすことを感じませんでした。しかし、PyImageSearchの記事には、OpenCV-DNNと互換性のある形式の既製のトレーニング済みニューラルネットワークがありました。

したがって、選択は最小の抵抗に向けて行われました。

使い方は非常に簡単です：（
最も心配なのは、ネットワークが1024x512の画像でトレーニングされていることです。これは、第一に、カメラがRaspberryで提供する以上のものであり、第二に、この量のデータを処理するために必要なパフォーマンスがやや混乱しています。その結果、主な問題は正確に判明しますその中で）。

ファイルからニューラルネットワークを読み取ります（1つはモデル自体、他のクラス名は3番目、色）。

def load_segment_model():
    try:
        classes = None
        with open(PiConf.SEGMENT_CLASSES) as f:
            classes = f.read().strip().split("\n")
        colors = None
        with open(PiConf.SEGMENT_COLORS) as f:
            colors= f.read().strip().split("\n")
        colors = [np.array(c.split(",")).astype("int") for c in colors]
        colors = np.array(colors, dtype="uint8")
        print("[INFO] loading model...")
        net = cv2.dnn.readNet(PiConf.SEGMENT_MODEL)
        return net, classes, colors
    except Exception as e:
        logging.exception("Cannot load segment model")
    return None, None, None

def segment_image(image_path, seg_net, seg_classes, seg_colors):

    image0 = cv2.imread(image_path)
    image = cv2.resize(image0, (1024, 512),interpolation=cv2.INTER_NEAREST)
    blob = cv2.dnn.blobFromImage(image, 1 / 255.0, (1024, 512), 0, swapRB=True, crop=False)

    seg_net.setInput(blob)
    start = time.time()
    output = seg_net.forward()
    end = time.time()

    print("[INFO] inference took {:.4f} seconds".format(end - start))

    (numClasses, height, width) = output.shape[1:4]

    classMap = np.argmax(output[0], axis=0)

    mask = seg_colors[classMap]

    mask = cv2.resize(mask, (image0.shape[1], image0.shape[0]),interpolation=cv2.INTER_NEAREST)
    classMap = cv2.resize(classMap, (image0.shape[1], image0.shape[0]), interpolation=cv2.INTER_NEAREST)

方向ニューラルネットワーク（実際には、分類器）は、左に移動するように命令されます。あまり正確ではないが、耐えられる。

2

ただし、ほとんどの道路はまだ認識されています。



分類器はまっすぐ行くことを提案しています。

３

応用


タンクのファームウェアを少し意識して、カラー道路検出器をセグメンテーションニューラルネットワークに置き換えました。

これらすべてをRaspberry Piで起動したときに最初に出てきたのは、パフォーマンスの低下でした。
1つの画像を分割するのに6秒かかります-この間、タンクは勢いのあるトロットですべてのターンをすり抜けます。

実際のテストでは、歩道と制御ニューラルネットワークからの正しいコマンドがほぼ完全に認識されているにもかかわらず、画像が処理されている間、タンクはなんとか横になりました。



一般に、このサイズの画像はRaspberryでは消化できません。
あなたはまだ専門的なニューラルネットワークの訓練をしなければならないようです。
