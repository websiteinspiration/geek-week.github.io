アメリカの旗の並べ替え


この「マルチバンド」ソーティングの動作原理を理解するには、3つのストライプのあるフラグの例から始めるのが簡単です。また、3色のフラグを簡単に処理するには、まず2色のフラグの例でどのように機能するかを確認することをお勧めします。そして2色に対応するために...

最初に、並べ替えられた配列の数値が2ビットで分布している場合を考えます。簡単にするために、ゼロ（低次）と1（高次）の配列があると仮定します。

したがって、「バンド」は2つしかありません。1つは最下位ビット（ゼロ）を移動し、もう1つは最上位ビット（単位）を移動します。 2色のフラグはすべてデモンストレーションに使用されます。たとえば、ウクライナの旗。



何が起きてる？最初の段階では、ゼロの数とユニットの数が不明であるため、各「バンド」の最終的なサイズが不明です。したがって、配列のキーに2つのポインターを置きます。低次の場合、ポインタは配列の最初に設定され、高次の場合、最後に設定されます。次に、配列を左から右に1回パスし、各ビット要素を調べます。

通過中に要素が最高次数に等しい場合、2番目のポインタはこの要素の転送先を指示します（交換を行います）。次の要素を挿入するためのポインタが左に移動し、上位の桁の「ストリップ」が拡大しました。

それが最下位桁と等しい場合、そのポインタを1要素右に移動します。2桁しかないため、要素を転送する必要はありません。すでに配置されています。若いカテゴリーの「ストリップ」は当然より広くなりました。

その結果、互いに向かって移動する2つのポインターが1点で衝突し、放電は「バンド」内で順序付けられます。配列全体を通過する必要はありません。ポインタが途中で出会うと、アルゴリズムが機能します。

オランダ国旗問題::オランダ国旗問題



タスクを少し複雑にします。2桁ではなく3桁を考慮してください。配列の要素が最下位（ゼロ）、中間（単位）、および上位（2）桁に属しているとしましょう。

デモでは、フランス、 ロシアのルクセンブルク 、オランダのシュレースヴィヒゴールドシュタインの 3方向の赤白青の旗を取り上げます。なぜオランダの旗なのか？エドガーダイクストラがこのフラグの例に関する講義で、対応するアルゴリズムを調べたためです。これは、「オランダ国旗のタスク」と呼ばれていました。



ご覧のとおり、特に新しいものはありません。各カテゴリには独自のポインタがあります。最初は、ジュニアとミドルのラベルが配列の先頭の開始位置を占め、対応する要素が見つかると右に移動します。上位のポインターは、最初に配列の最後にあり、左に移動します。

配列の通過も実際には不完全です。ビットが多かれ少なかれ均等に分散されている場合、アルゴリズムは配列の2/3を通過してから、「バンド」内のすべての要素を分散します。

アメリカ国旗ソート::アメリカ国旗ソート



今、私たちの説明では、マルチバンドのアメリカ国旗に移ることができます。

2桁ではなく、3桁ではなく任意の桁数がある場合、各桁の開始位置（「バンド」）を修正し、「バンド」内の要素を再描画します。

このアルゴリズムでは、数値は通常10進数ではなく、異なるビット深度であると見なされます。ほとんどの場合、2の累乗です。多くの場合、256という数値はビット深度の基礎として使用されます（128よりも少し少ないことが多い）。これにより、並べ替えを効率的に調整して文字列を配置できます。数値の場合、ビット深度は小さい2 n（2、4、16など）を取る方が便利です。これにより、10進数を比較するときに累乗するのではなく、ビットごとにシフトして比較できます。

スカソート::スカソート


ドイツのプログラマーであるMalte Skarupkeは、新しいソートアルゴリズムを開発したと発表しました。これは根本的に改善された「アメリカの旗」であり、std ::ソートの平均を2回上回ります（std ::ソート- 内省的ソートとも呼ばれるアルゴリズム）- クイックソートとヒープによるソートのハイブリッド）。

  1. 配列は再帰的にソートされ、再帰の最初のレベルでは、配列全体がサブ配列として扱われます。
  2. サブ配列の要素数が128未満の場合、std :: sortが呼び出されます。
  3. サブ配列に128から1024の要素が含まれている場合、そのためにアメリカのフラグの並べ替えが呼び出されます。
  4. サブ配列に1024を超える要素がある場合は、スカソートが呼び出されます。
  5. また、最悪のケースを回避するために、再帰的なネストが大きすぎる（16レベルを超える）場合、サブアレイに128を超える要素がある場合でも、アルゴリズムはstd :: sortに切り替えます。

どうやら、それは非常に効果的なアルゴリズムであり、同時に非常に複雑です-その作者の実装は、約1.5行を必要とします。おそらく、いつかはこの分類を検討するでしょうが、今はその分類には入りません。興味のある方は下のリンクをクリックしてください。
