自己暗号化SSD：それはどのように機能しますか？


自己暗号化ドライブ（SED /自己暗号化ドライブ）は、市場に長年存在していますが、実際には、これらのドライブの機能を完全に利用する人はほとんどいません。このようなSSDは、ドライブコントローラに組み込まれた暗号化メカニズムを使用して、フラッシュメモリセルに保存されている各ファイルを暗号化します。このハードウェアベースの暗号化方式は、高レベルのデータセキュリティを提供し、ユーザーには見えません。無効にすることはできず、パフォーマンスに影響を与えません。



最新の自己暗号化ドライブは、業界標準のSSC V1.0 Trusted Computing Group（TCG）Enterpriseに基づいています。古い製品では、ハードウェア暗号化メカニズムは、ユーザーが提供した暗号化キーを使用するだけでした。TCG Opal 2.0の場合、ハードウェアメカニズムは、乱数ジェネレーターによって作成されたランダムな「キー暗号化キー」（KEK）を使用します。このキーは外部インターフェイスでは使用できず、MEKキー（または「マルチメディア暗号化キー」）の暗号化に使用されます。これらのキーは常に暗号化された形式でSED SSDコントローラー内に保存されます。

TCG Opalとは何ですか？


TCGセキュリティサブシステムクラスは、自己暗号化SED SSDのハードウェア機能であり、ドライブ上のデータ暗号化の高速化に役立ちます。ソフトウェアとは異なり、ハードウェア暗号化はプロセッサまたはオペレーティングシステムを暗号化および復号化プロセスの負荷から解放するため、全体的なパフォーマンスは低下しません。 Opal SSC仕様セット自体は、データがインストールされているストレージデバイスまたはホストシステムにアクセスする悪質な人々によるデータの盗難や改ざんからデータを保護する一連のセキュリティ管理標準を提供します。 TCG Opal機能はそれ自体では機能しないが、ドライブへの特別なソフトウェアのインストールを提供することは注目に値します（Symantec、McAfee、WinMagicおよびその他の企業から）。コンピュータの電源を入れるたびにセキュリティ設定とユーザー初期化を提供します。

Opal SSC仕様は、ストレージデバイスがオフになっていて、ユーザーがシステムからログアウトしたときに、保存されているデータを保護するように設計されています。この場合、ユーザーがシステムを終了しないときに、アイドル状態を「スリープモード」に関連付けないでください。その結果、StandByモードの便利な使用を妥協して放棄する必要があります。 SED SSDは、有効な資格情報を使用してストレージデバイスがロック解除された後のデータアクセスから保護するように設計されていないことも考慮する価値があります。



静止時には、ディスクのメイン領域は完全にロックされており、アクセスできません。ただし、システムが起動すると、暗号化されたディスクが起動前の識別のためにメインブートパーティションのシャドウコピーを起動します。このシャドウMBRは、ドライブからのパスワードをユーザーに要求する小さなオペレーティングシステムで、OPALコマンドを介してSSDコントローラーに転送されます。パスワードが有効な場合、ディスクのロックが解除され、実際のオペレーティングシステムが起動します。その結果、許可されたユーザーのみが、パスワード保護を追加したデバイス上のデータにアクセスできます。これにより、盗難、改ざん、またはデータ損失の可能性が最小限に抑えられます。

SSDからデータを完全に消去する方法は？


自己暗号化SSDを含む一部のSSDは、このハードウェア暗号化自体のために完全に消去することはできません。同時に、PSID Revertの操作を通じて情報のいわゆる暗号化「消去」を可能にする効果的なソリューションがあります。実際、クリーニング手順全体は、すべての暗号化キーを破棄するプロセスに帰着します。したがって、データを解読することはできません。この方法は、TCG OpalがサポートされていないSSDでは使用できないことに注意してください。また、TCG OpalおよびeDriveオプションがアクティブ化されていないと機能しません。キングストンドライブに関するTCG Opalの使用に関するより詳細なマニュアルは、会社の公式Webサイトにあります。。この記事では、Opal互換ドライブを使用する前に行う必要のある事前設定の複雑さについては触れずに、SSDからのデータの完全な削除についてのみ触れます。



だから…、言葉から行動へ。 Revertメソッドを使用してKingston KC2000 500GB SSDからすべてのデータを消去するには、Windows 8、10、またはWindows Server 2012で上記の「ドライブ」をセカンダリドライブとして指定する必要があります。準備が整ったら、メーカーの公式WebサイトからユーティリティをダウンロードするだけですキングストンSSDマネージャー、メインドライブにインストールして実行します（他のメーカーのドライブにも同様の機能を備えた緊急ドライブが用意されているため、暗号消去方法も同様です）。ディスクを保守するための設定の中には、ドライブをリセットする必要があるオプションがあります。



これは[セキュリティ]タブにあり、TCGの[元に戻す]コマンド（「TCGを元の設定に戻す」）を選択する必要があります。 SSDへのアクセスを確認するには、TCG Revertの右側にあるテキストボックスにPSID番号を入力します。一意の32桁のSSD識別子（ドライブ自体にあります）で、リセットオプションを有効にします。リセット操作が完了すると、プログラムは操作の成功に関するメッセージを表示します。それ以外の場合（メッセージが表示されなかった場合：たとえば、入力を間違え、間違ったPSID番号を入力した場合）、設定を再度リセットする必要があります。



手順の結果に基づいて、ドライブ上のすべてのデータが暗号的に削除され、SSDが工場出荷時の設定にリセットされます。IEEE1667サポートを無効にするために残っており、ドライブの情報が復元されることを心配することなく、ドライブを再利用する準備ができています。さらに、互換性のあるセキュリティアプリケーションを使用して、そのようなSSDでOPALまたはeDriveを再アクティブ化できます。
