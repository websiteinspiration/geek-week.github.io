プリンター言語とテクノロジー



今日、プリンターはコンピューターと大差ありません。プロセッサ、メモリ、オペレーティングシステム、読み取り専用メモリを備えています。プリンターがネットワークプリンターの場合は、ネットワークカードと、管理を可能にするWebサーバーがあります。プリンターは、さまざまなネットワークプロトコルをサポートするだけでなく、ページやドキュメントのレイアウトを記述する特定の言語のタスクも受け入れます。したがって、大企業では、印刷ソフトウェアは印刷プロトコルのさまざまなハードウェア実装と対話する必要があります。プリンター印刷の設定は、最初に表示されるよりも難しいです。ユーザーは当然のことながら印刷を行いますが、高品質の結果を得るには、複数のステップを実行する必要があります。

この記事では、新しいプリンターを選択するとき、ドキュメントのプリプレス準備を行うとき、またはドキュメントの電子PDFコピーを操作するときに遭遇する印刷技術を検討して比較することを提案します。そしてもちろん、あなたが印刷ジョブを傍受して分析しようとしているなら。

プリンター言語について


印刷ジョブとは何ですか？これは特別なプログラミング言語-ページ記述言語（PDL）で書かれたプログラムです。

印刷されたページはPDLでエンコードされ、サイズが小さいため、生の画像よりも高速に送信されます。 PDLは、デバイス自体と解像度の両方から独立しています。プリンターは、特殊な言語のジョブをデバイスフレンドリーな形式に変換します。つまり、プリンターには言語インタープリターが含まれています。プログラミング言語「アラJava」のように、これらの言語には複数のバージョンがあり、それぞれが独自の方法で機能します。

PDLからビットマップへの変換は、プロセッサプログラム（ビットマッププロセッサ、ラスターイメージプロセッサ、またはRIPのみ）によって行われます。

最も有名なPDL言語：Postscript、PCL5、PCL6

多くのプリンタは複数の言語を理解しています。しかし、プログラムを別の言語で送信すると、印刷を完了できません-コンパイラは同じではありません。

PostScriptが最も一般的です。もともとアドビによって開発されました。使用するにはライセンスが必要なため、現在は高価格帯の高性能デバイスで主に使用されています。ページを構成するほとんどすべてのプログラムがPostScriptジョブを生成できます。これは完全に機能するプログラミング言語です。作成されたプログラムは、テキストエディタを使用して表示できます。多くの括弧と/％文字があります！

PSこれらの文字は、印刷ジョブを認識するためにインタープリターによって検索されます。

PostScriptの例：

%!PS-Adobe-3.0 EPSF-3.0
   %%Creator: [generally the program that generated the postscript]
   %%Title: [descriptive name or just the file name]
   %%CreationDate: [date the file was created]
   %%DocumentData: Clean7Bit
   %%Origin: [eg: 0 0]
   %%BoundingBox: xmin ymin xmax ymax
   %%LanguageLevel: 2 [could be 1 2 or 3]
   %%Pages: 1
   %%Page: 1 1
   %%EOF

PostScriptはMACの標準でもあり、プロフェッショナルな標準でもあります。

PCL-またはPrinter Common Language-は、Hewlett Packard（以降HP）のPostScriptの代替です。この言語は、他のメーカーのプリンターでも理解できます。一部のメーカーのプリンターしか使用できません。 PCLはプログラミング言語ではなく、単にページを印刷する方法をプリンターに指示するだけです。 PCLのジョブはバイナリであり、人間には理解できませんが、サイズはPostScriptよりも短くなります。

PostscriptをPCLに変換するフィルターがあります。 PCLのバージョンはPostScriptほど大きくは異なりませんが、煩わしいほど十分です。ジョブは、プリンターモデルによって印刷方法が少し異なります。その理由は、特別なチームを持つ方言にあります。 PostScriptとは異なり、PCLはもともとページの移植性ではなく、プリンターを管理するために特別に設計されていたため、最高の印刷結果を得るには、対応するプリンターのコマンドを使用する必要があります。プリンタモデルがオペレーティングシステム（OS）で示されているのはこのためです。そうでない場合、誤って解釈されるか完全に無視される他のPCLコマンドが生成されます。

実際、問題はハードウェアだけではありません。いわゆるエミュレーションがあります。

PCLエミュレーション-これは、標準の開発者（HPなど）がPCL互換性について製造元のプリンターのライセンスを取得またはテストしていないことを意味します。

PostScriptエミュレーション -AdobeはPostScriptインタープリターの使用料を受け取りませんでしたが、一部のベンダーは独自のコードを作成しました。ライセンスポリシーにより、言語の方言が似ていますが、元の言語を正確に繰り返すことはできませんでした。実際には、どちらもエラーで失敗する可能性がありますが、これはめったに起こりません。

完全に混乱させるために、HPは2つの言語ファミリ、PLC5（5eは白黒、5cはカラー）とPCL6（PCL / XL）を識別しました。新しいHPプリンターは両方をサポートしています。以前はPCL4が存在していましたが、現在は古すぎます。PCL5 5eから、600 dpi解像度のサポート、プリンターとコンピューター間の双方向データ交換、Microsoft Windowsの新しいフォントなどの革新的な機能も導入されました。

PDFは別のアドビ開発であるPortable Document Formatです。これは、発行およびOfficeプログラムの基盤であるPostScriptの機能の一部を使用するドキュメント形式です。 PDF文書はOSおよびプラットフォームに依存しません。多くの場合、この形式は、表示および印刷機能を備えたドキュメントの交換に使用されます。 PDFはページではなく、ドキュメントを記述するための言語です。ページだけでなく、ドキュメントの全体的な構造、章、テキスト列の相互関係、編集なども説明できます。さらに、マルチメディア機能の束。

PDFを直接解釈するプリンターがあります。 PDFをPostScriptなどに変換する機能を備えた多くのトランスレーターやビジュアルエディターがあります。この変換はユーザーから隠すこともできます。

XHTML-比較的最近登場しました。プリンターは、XHTML印刷Webページを記述する言語でデータストリームを受信し、ジョブ表現を生成します（異なるブラウザーが異なるようにページを反映するように、異なるプリンターは異なるジョブを生成します）。

HP-GL / 2 -Hewlett-Packard Graphics Language-ドキュメントの一部としてベクターグラフィックスを印刷するために使用されます。

HPGLはプロッタサポート言語です。ほとんどすべてのHPプリンターでサポートされています。

PJL-プリンタージョブ言語。プリンターのジョブ言語であるHPのメタ言語は、ジョブに使用するPDL、用紙サイズ、印刷する必要がある部数、片面ジョブまたは両面ジョブなどを記述しています。

PJLの例：

@PJL SET PAGEPROTECT=OFF
@PJL SET COPIES=3
@PJL ENTER LANGUAGE =PCL

ドライバーについて


プリンタードライバーとサポート-ファイルをプリンターに適したファイルに変換するソフトウェア。ドライバーのタスクと機能は、「プリンターがすべての言語をサポートしていない場合はどうなりますか？」、「ポストスクリプトジョブがあり、プリンターはPCL 5Eのみを認識します。 「PDFを印刷する必要がありますが、プリンターで解釈できない場合はどうなりますか？」

システムはすべてを独自に実行できます（PDLファイルの言語を見つけ、変換を実行します）。ファイルを手動で変換することもできます。ブラウザはHTMLをPostScriptまたはPDFに変換できます。 Open Officeは.docをPDFに変換できます。ポストスクリプトからは、PCLを含むほとんどすべての形式に変換できます。

Gdi-別の印刷ジョブをPCで一元的に表示および解釈できます。完成した処理済みビットマップ画像を「知能なしで」プリンタに送信することもできます。これが、Windows GDIプリンターの数です。このようなプリンターには、非常に少数の論理命令があり、PDLインタープリターはまったくありません。代わりに、ラスター化はホストコンピューターによって実行されます。GDIと対話するための情報の一部は、特許の下でWindowsコードに隠されています。この秘密性は、Linuxシステムで対応するものを開発することを困難にし、本質的には利点です。同様に、状況は最新のプリンターモデルのサポートによって発展しています。ただし、リバースエンジニアリングを使用する多くのWinプリンターをサポートするCUPSデーモンが存在するため、状況は変化しています。

プリントサーバーについて


CUPSについてのちょっとした楽しみと、それを介した印刷ジョブの送信。このプリントサーバーは、よく知られているHTTPオーバーロードであるIPPに基づいています。プリンターのステータスに関する問い合わせは、HTTP GET操作として送信され、HTTP POSTを介してジョブを印刷します。したがって、CUPSは実際にはWebサーバーですが、80ではなくポート631でのみ通信します。CUPSはおそらく地球上で最も完全なIPP実装であり、その構成ファイルはApache Webサーバーファイルと非常に似ています。そしてもちろん、印刷システムが認証、アクセス制御、暗号化技術を使用できるようにします。

Windowsでは、IPPを介した印刷がWindows 2000以降に登場しました。Windows7以降のクライアントでは、通常IPPサポートがすでにインストールされています。インターネット印刷もあります-IPPによるインターネットプリントサーバーのWindows実装。これをインストールするには、最初にMS IIS Webサーバーサービスをインストールする必要があります。

プリンターのセキュリティについて


もちろん、ネットワークプリンタはリモートコントロールシステムをサポートしています。設定と監視は主にIPPプロトコル、SNMP、またはWebブラウザからHTTPを介して実行されます。特に、プリンターにアクセスすると、IPアドレス、標準ゲートウェイ、Syslogサーバー、SNMPグループ名（パスワード）、プロトコルオプション、管理者パスワードなどのパラメーターを取得できます。ここでは、SNMPおよびDDOS攻撃を介した不正アクセス攻撃が可能です。リモートで管理できるほとんどすべてのプリンターは安全ではなく、インストール時に指定されたパスワードまたはSNMP名を使用します。

プリンターはどのオペレーティングシステムを使用していますか？一部のモデルには、Linuxベースのディストリビューションが搭載されています。製造元のドキュメントを掘り下げることによってのみ、プリンターにインストールされているものを理解したり、デフォルトのパスワードを変更したりできます。オペレーティングシステムとの混乱は、グラフィカルな管理ツールが製造元間の違いに関する情報を隠す傾向があるという事実によって悪化します。
