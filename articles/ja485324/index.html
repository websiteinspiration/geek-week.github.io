Qtウィジェットのマルチスレッド

ウィンドウ化されたインターフェースでアプリケーションを実行する場合、フリーズがないことを確認することが重要です。このため、複雑な計算は別のスレッドで実行する必要があります。マルチスレッドアプリケーションのコンセプトは、run（）メソッドを再定義する必要がなく、Qtシグナルスロットアプローチに適しています。

本旨。マルチスレッドのアプリケーションでは、計算は別のスレッドで実行され、最後にシグナルが発行され、その結果が引数で送信されます。 MainWindowがすでに所有しているスロットが呼び出されます。計算結果はスロットの引数に含まれ、出力することは難しくありません。

マイクロコントローラーとの類似点を描くと、信号は割り込みベクトルに沿った遷移であり、スロットはそれ自体が割り込みハンドラーです。 「割り込み」が発生するためには、シグナルを発行する必要があります。その後、Qtは彼を「ハンドラー」（スロット）で探し始めます。 Qtでは、1つの信号に多くのスロットを掛けることができます。

次の図は、デモアプリケーションのアルゴリズムを示しています。伝説によれば、USB HIDデバイスは処理スレッドが毎秒ポーリングします。次に、発行シグナルが発行されます。これは、MainWindowがすでに所有しているスロットを処理し、それに応じてフォームに描画する機能を備えています。



したがって、信号スロットアプローチを使用できるクラスがあります。これを行うには、その宣言でQ_OBJECTマクロを使用します。

class Worker : public QObject
{
    Q_OBJECT  //   -  

public:
    QTimer *timerDeviceRead;  //      GuiUpdateCallback
    Worker();

public slots:
    void updateElectropipData();

signals:
    void GuiUpdatePlease(uint8_t const *arrptr,size_t);
};

class MainWindow : public QMainWindow  //   GUI
{
    Q_OBJECT  
public slots:
    void GuiUpdateCallback(uint8_t const *arrptr, size_t);
private:
    Ui::MainWindow *ui;  //      
    QThread *thread;

引数に計算結果を渡すには、引数に使用する型を登録する必要があります。これはさまざまな場所で行うことができますが、通常はクラスのコンストラクターで行うことができます。これにより、これらの型で機能します。

Worker::Worker(){
    qRegisterMetaType<std::size_t>("size_t");  
    qRegisterMetaType<uint8_t const *>("uint8_t const *");


次に、処理スレッドのコンストラクターにタイマーが作成されます。毎秒、updateUSBDataCallback処理スロットが呼び出されます。Qt5の信号スロット接続の構文に注意してください。

    this->timerDeviceRead = new QTimer();
    connect(Worker::timerDeviceRead, &QTimer::timeout, this, &Worker::updateUSBDataCallback);
    this->timerDeviceRead->start();

以下は、処理スレッドスロットの本体です。すべての長い間考え抜かれたコードがここにあるはずです。

void Worker::updateUSBDataCallback(){
    size_t mysize = 65;
    uint8_t buf[65] = { "I like USB HID" };
    emit GuiUpdatePlease(buf,mysize);  // 
	//:
    for(int i =0;i<64;i++){
        buf[i]=i+'0';
        if (i+'0' > 250) i=0;
    }
}

ここで説明するために、信号がMainWindowスロットに送信された直後に、配列の内容が勇敢に変更されます。また、配列はポインタによって渡されるため、ダーティリードが取得されます。この状況を回避するには、処理スレッドからの信号を特定の方法でGuiUpdateCallback（）スロットに接続する必要があります。

MainWindow::MainWindow{
connect(worker, &Worker::GuiUpdatePlease, this, &MainWindow::GuiUpdateCallback, Qt::BlockingQueuedConnection);

この場合、シグナルを発信すると、GuiUpdateCallback（）スロットの終わりまで処理スレッドがブロックされます。

長い「uint8_t const *」がプログラムのテキストを混乱させる場合、同義語TU8PTRを取得できます。

typedef uint8_t const * TU8PTR;
