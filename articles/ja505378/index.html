なぜこれを読む必要があるのですか？


  1. ITには「どうやって」のテクニックや解説はたくさんありますが、私の考えでは「なぜ」「なぜ」「誰のため」なのかはほとんどわかりません。
  2. ITは市場でも顧客でも機能します-したがって（試してみてください）すべての市場法がここで機能します。
  3. ITのマーケティングに携わっている場合、同僚の経験について読むのは興味深いと思います。
  4. あなたが開発者である場合、マーケティング担当者が誰であるか、なぜマーケティングが単なるプロモーションではないのか、なぜコンテンツの請求書を求めて研究に参加するように頼むのかを知ることは興味深いでしょう。また、製品と会社を少し異なる視点から見ます（たとえば、ビジネス側。理想的にはクライアント）。

2つのストーリーから始めます


歴史＃1。アプリケーションについて。靴がたくさん売れている人気サイトでスニーカーを注文したかったのですが。以前は、アプリケーションを介していくつかのサイズを注文し、店舗で試して、気に入ったペアを購入することができました。今、彼らの店は閉まっています、アプリケーションを通して購入します。最初にサイズを選択します（推測できる靴を使用できますが、推測することはできません）。その後、支払いを行い、靴を持ち帰ります。そうでない場合はどうなりますか？返金しますか？もう一週間待つ？ 3足の靴を注文して支払い、それから合わなかった2足を返却しますか？..私はそれを理解して手紙を書くことにしました。アプリケーションにはチャットボットもオンラインコンサルタントもいません。「申し立て」セクションに書き、問題を説明し、続行するための最良の方法を尋ねました。答えは（引用）「配達先の都市を指定してください」でした。そして、答えのフィールドはありません。物理的に。 2番目のアピールを書いた、彼女がそれに注意を払ったところ-彼らは私に答えませんでした。同社は、マーケティング、広告に多くのお金を費やし、売り手を教えており、アプリケーションのアピールに対応するためのフィールドは、当然のことながら存在しません。アプリケーションは、企業とクライアント間の明白な連絡先です。連絡に失敗しました。ささいなこと？番号。スニーカーは買わなかった。

#2。警備員について。先日、薬局に行ってハンドクリームを買いました。薬局は広く有名なネットワークです。広い部屋で、私だけが買いました。探していたクリームがなかったので、新しいものを選びました。そして、新しいものを選ぶことは全体の話です。そのため、文字通り警備員が後ろを向いていることに気づき、さまざまな企業の製品を見始めました。私はラックにいます-彼は私の後ろにいます。クリームを買いたくなくて、うんざりして逃げたかった。しかし、クリームなしで-子供を連れて行きました。レジに立っています。私は警備員の私の印象を売り手と共有します。そして彼らは答えました-はい、彼らは常に彼について不満を言っています、彼はいつもそのようです。みんな！これは薬局です。ここはとてもデリケートな場所です-人々はクリームだけでなくここに来ます。薬の場合、脱毛症やインポテンスの治療法は、脱毛を意味します。そのような親密なことを警備員と共有したくはありません。残念ながら、これも連絡窓口です。この薬局に戻りますか？結構です。

これらは、接点がどのように異なるか、およびクライアントに関連して些細なことはないという2つの物語です。

なぜこれがITにあるのですか？はい、多分問題は技術的ではなく純粋なマーケティングですが、製品エンジニア、両側で作業するプロジェクトマネージャーにとって、複雑なIT開発の決定であってもロボットではなく人によって行われることを覚えておくことが重要であるように思えます（現時点では）。また、薬局や靴屋の窓口に関連するすべてのものは、ITセグメントにも関連しています。

連絡先とは？


これらはすべて、顧客がオンライン、オフライン、電話、またはその他の方法で会社や製品に出会う「場所」です。

マーケティング担当者のIgor Mannによると、これらはグループ全体である可能性があります。

  1. 会社の連絡先（ウェブサイト、看板、文書、広告、印刷など）。
  2. 製品の連絡先：パッケージ、レイアウト、デザイン、名前、バーコード、ユーザーマニュアル、保証書。
  3. サービスの連絡先：名前、プレゼンテーション、カスタマーレビュー、ケース、出版物。
  4. 従業員の連絡先：身長、体格、笑顔、きちんとした外観、髪型、制服、バッジ、姿勢、音声パターン。

それらは制御できますが、優先順位はありません。

「ゼロ」の接点


これらは3つのブロックです。

  • ポジショニング
  • ブランドブック会社
  • 会社と製品の説明

これらは基本的なものであり、それがなければ他のものと一緒に作業することは不可能です。顧客に配信するメッセージと、このメッセージの外観を決定します。急いでメディアやソーシャルネットワークに書き込んだり、完成する前に自分について言及したりすることは意味がありません。製品の理解は断片化され、無秩序であり、視覚的な部分がないためにその認識は低くなります。
「世界で最も安全な車」とは異なるボルボのキャッチフレーズを思い出してください。
または、Sberbankの別のコーポレートカラーまたはロゴ。

連絡先の練習には数分かかります。たとえば、企業のメールの署名など​​です。一部（たとえば、製品自体の後にサイトを主要なポイントとして再開した）-数か月。

連絡先をどのように操作しますか？


  1. クライアントがあなたの会社、製品、従業員に出くわす可能性があるすべての場所について考えてください。
  2. リスト全体をグループ別に記録します。
  3. 連絡先を、最も重要なものから最も重要でないもの、または将来的に考える必要があるものに並べ替えます。
  4. ポイントの状態を簡単に説明してください。改善する必要がある場合は、正確に実行する必要があることを記述します。
  5. 連絡先を割り当てます。
  6. 更新スケジュールをスケジュールします。たとえば、6か月に1度-グローバルな監査を行い、サイトのフォームをチェックするか、週に1度「クライアントの目を通して」コールセンターに電話をかけます。
