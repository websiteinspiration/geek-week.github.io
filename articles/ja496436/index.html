サーバー上のIPFS。ラップトップからサイトをホストする



静的なページやWebサイト、Webフォームやレイアウトを使ったデモを公開する必要があることがよくあります。毎回Jsfiddleのような場所に注ぐことは常に便利であるとは限らず、ローカルホストで静的を編集する方がはるかに高速で快適です。誰かに私の作品を見せてもらうか、電話から同じページを開くだけで問題が発生します。これらの無限の作業オプションとスケッチをすべてホストしなければなりません。アップロードファイルごとに、vhostsをねじ込みます。

IPFSを使用すると、ラップトップから直接インターネット上のサイトをホストできます。ローカルファイルのすべての更新はすぐにインターネットに適用され、どこかにアップロードする必要はありません。ラップトップをネットワークから切断しても、サイトは引き続き利用できます。IPFSはBittorrentに似ており、Web向けです。

この記事では、サーバーにIPFSノードを展開し、このテクノロジーを実際に試します。

IPFSとは何ですか？


IPFSは、ファイル共有サービス、Webアーカイブ、またはBittorrentの代替として使用される、大規模な分散型p2pネットワークです。実際のプロジェクトでIPFSを使用する優れた例はすべて、awesome.ipfs.ioの公式Webサイトの名誉の殿堂にあります。

簡単に言えば、これは次のように機能します。格納されたファイルはマルチキャッシュを取得し、ブロックに分割され、すべての対象ノードに分散されます。DHTはノード上で同期されます。ファイルがダウンロードされると、ブロックは異なる（理論的には最も近い）ノードから収集されます。さらに、ノードを上げる必要のないファイルまたはディレクトリにアクセスするために、それらはすべてブラウザーからアクセスできるため、人気のある機能にアクセスできます。IPFSでサイトを無料でホストできます。ただし、静的であるだけで、リンクを形成する同じハッシュがファイルを変更するたびに変更されるため、数分ごとに複数回データを更新するのは不便です（永続的なIPNSネームシステムはありますが、低速です）。ただし、これによってOrbitdbの担当者がデータベースをIPFSにフラッシュするのを止めることはできませんでしたが、いくつかのニュアンスがあります。ネットワークデバイスの詳細については、こちらをご覧ください。。


静的サイトを無料で配布するためのノードがあるとします。自分でそれらのサイトにアクセスし、ときどき同僚に見てもらいます。データの総量は、私のディスクのサイズと家庭用インターネットによるダウンロード速度によってのみ制限されます。しかし、いくつかの問題があります。まず、IPFS自体はかなりの量のインターネットと堅固なプロセッサを必要とし、トラフィックがなくても、DHT同期には常にいくつかのリソースを必要とします。次に、主にラップトップで作業し、すべてのファイルをその上に保持するため、自家製のハーフギガバイトのファイバーを手元に置いていることはいつも遠くからです。 IPFSの短期間の中断は問題ではありません。DHTにキャッシュを数時間保持しますが、数日間離れるにはどこかにコストがかかります。これで、すべてのプロジェクトがネットワークから十分にスリープ状態になりました。デスクトップでファイルを踏み付ける（「記憶する」）ことができますが、これによりトラフィックが少なくとも2倍になります。これもまた、失敗ではありません。何をすべきか？サーバーのノードを上げてラップトップをアンロードしようとしましたが、それでも通常のホスティングのように手動でファイルをアップロードする必要がありました。その結果、ドックとAPIをスモークし、ローカルの静的データをサーバーと同期する簡単なユーティリティを作成しました。



IPFSには、go-ipfsとjs-ipfsの2つの個別の実装があります。JSは私に近いので、私はそれについて書きました。ユーティリティで自分のサイトのフォルダーを取得し、作業中に定期的にそれらをネットワークにアップロードできるようにしたいと考えていました。サーバー側は、フォルダーのハッシュをキャッチしてそれらをキックして、ファイルが失われないようにする必要があります。

インストールと起動


npm install ipfs -g

Js-ipfsには例を含むかなり詳細なチュートリアルがあるので、

git clone https://github.com/ipfs/js-ipfs.git
cd js-ipfs
npm install
npm run build

ノードは複数の行で実行されます。

const IPFS = require('ipfs')
async function main () {
  const node = await IPFS.create()
}
main()

そのための構成とシードはすぐに登録されますが、これで十分です。

結果はどうですか？


  • 座ってコードを書くと、自動的にIPFSに保存されます
  • すべてのバージョンは、常にパーソナルリンク/ ipfs / Qmを介して利用できます...
  • 一連のリンクをクライアントに送信しないように、サイトをIPNSで公開できます
  • ローカルノードは、ファイルをアップロードしてサーバーに接続するためだけに起動し、その後、スリープ状態に戻ります。
  • ローカルでは、サイトのコピーが1つしかありません。新しいバージョンの単一ダウンロードへのトラフィックは、IPFSノードのバックグラウンドトラフィックよりもはるかに少ないです。
  • 私はようやくバージョン管理で頭痛なしにローカルホストで作業できます
