永遠のサーバー-毎月のホスティング支払いを取り除く方法



-サービスを拡張するのに十分な資金がアカウントにありません。10年間のすべての作業は2営業日以内に破棄されます。


ホスティングの料金が間違ったタイミングで発生する状況は誰もが知っています：カードのお金が使い果たされたか、有効期限が切れ、通知レターがスパムに陥り、あなたはこの瞬間に動き、猫を産みました。その結果、サービスがブロックされ、すべてが機能しなくなり、支払いのために実行する必要があります。また、危機的な状況では、コロナウイルスで死なないようにする必要があり、個人的なプロジェクトでは、ソバと有料サーバーのどちらかを選択する必要があります。

各ドメインゾーンには、ルールを設定する独自の組織があります。たとえば、.ruゾーンは以前はRosNIIROS（ロシア公共ネットワーク開発研究所）によって管理されていましたが、現在は別の組織であるNational Internet Domainの調整センターが作成されています。

ドメインを10年間購入する方法は？


ドメインの最大更新期間は、ドメインゾーンの規制によって決まります。 .RUゾーンの場合は1年、.COMおよび.NETゾーンの場合は10年です。ただし、すべてのレジストラが10年の延長を提供するわけではありません。一部のレジストラは、ドメインゾーンルールが許可するよりも長い期間ドメイン更新サービスを提供することに注意することが重要です。これはマーケティングの策略であり、実際には、いくつかの更新の料金を請求し、通知なしに密かにドメインを更新します。ドメインが更新される実際の用語は、有効期限の行のwhoisに示されます。


実際のドメインの有効期限は、whoisでのみ指定されます

更新の価格と条件が変更される可能性があるため、レジストラが更新の義務を確認できるため、実際のドメインの有効期限をレジストラがパネルに表示するものと区別できることが重要です。また、ドメインを別のレジストラに移管するときは、実際の更新期間のみが考慮されます。

そこで、できるだけ長い期間ドメインを更新することにしました。.COMと.NETは10年間延長できることを忘れないでください。これを行うには、この期間にドメインを本当に更新するレジストラを選択する必要があります。それらの数はそれほど多くありませんが、ほとんどの大規模な外国のレジストラがこれを行うことができます。たとえば、uniregistry.com、domain.comなどです。Cloudflareサービスドメインをドメインに登録することはできませんが、10年の延長でドメインを自分に移管することができます。

永久SSL証明書




毎年、ブラウザとオペレーティングシステムは、暗号化されていないHTTPプロトコルに対してますます厳しくなっています。新しいバージョンのGoogle Chromeは、サイトがHTTPSをサポートしていない場合、プロトコルを使用して安全でないサイトにアクセスしているという警告を表示することを約束します。 HTTPSを使用しないサイトで下位のサイトを検索します。したがって、SSL証明書がないと難しいことは明らかです。

SSL証明書の絶え間ない期限切れや新しい証明書の購入に悩まされないようにするにはどうすればよいですか？もちろんLets Encryptを使用してください！これは、SSL証明書を自動的に発行するための人気のある無料サービスです。 SSL証明書を永久に解放することを忘れて、自動化に任せることができます。彼にとっては、セットアップ手順と同様に、多くの実装とクライアントがあります。最も人気のあるcertbotクライアントであるnginxウェブサーバーとのバンドルの場合。ほとんどの場合、長い間暗号化されており、今後10年間でそのプロトコルと条件が大きく変わることはほとんどありません。

let暗号化の使用法の説明はこの記事の範囲を超えているため、重要なポイントのみを取り上げます。証明書を発行するときに、オプションで電子メールを指定するように求められます。証明書がなんらかの理由で再発行できなかった場合、このアドレスに通知が送信されます。暗号化を実行できるACMEプロトコルは時々変更され、クライアントの更新が必要です。古いクライアントを使用しているという通知もこのメールで送信されます。


Lets Encryptからの非推奨のプロトコル通知レター

Lets Encryptが存在する間、そのプロトコルは何度か修正されており、これは今後も続くことが予想されます。したがって、証明書が突然破られないように、彼のリマインダーを追跡することが重要です。

ドメインなしのSSL証明書


ドメインなしでSSL証明書を発行することは不可能ですが、内部APIまたはテスト用にサーバーをデプロイし、証明書のためだけにドメインを購入したくない場合はどうでしょうか。そして、あなたはすでにあなたのサーバーにバインドされたドメインを持っているので、それは必要ではありません！SSLip.ioサービスは、世界中のすべての既存のIPアドレスに対して事前に生成されたドメインを提供します。

サーバーのIPアドレスを特別なサブドメインエントリに変換するだけです。

ここでは次のようになります。

サーバーのIPアドレスがある場合：123.45.67.89
ドメインは次のようになります。123-45-67-89.sslip.io

チェックこれがtrueの場合：

$ host 123-45-67-89.sslip.io
123-45-67-89.sslip.io has address 123.45.67.89

したがって、ドメインやDNSを構成する必要なく、新しいサーバーの証明書をすぐに発行できます。SSLip.ioは、サーバーの外部IPアドレスの自動検出と組み合わせてスクリプトで使用でき、初回起動時に完全に構​​成されたサーバーを生成します。

自動セキュリティ更新


ゾンビの黙示録が来ると、忙しくてセキュリティや新しい脆弱性に関する定期的なレポートを読むことができなくなる可能性があります。バンカーでの生存闘争の中で、サーバー上のソフトウェアが脆弱になっていることに簡単に気付くことはありません。

時間内に脆弱性にパッチを適用しないと、すぐにハッキングされ、暗号通貨マイナーが起動されるか、DDoSボットネットに追加されます。この場合、悪意のあるアクティビティまたはリソースのオーバーランによってブロックされる可能性が高くなります。

このような状況に対処するために、自動セキュリティアップデートがあります。 Ubuntuでは、これは無人アップグレードと呼ばれていますが、他のディストリビューションにも類似物があります。

Ubuntuでのセットアップを検討してください。

#   
apt-get install unattended-upgrades

#     
dpkg-reconfigure  unattended-upgrades


自動セキュリティ更新の

有効化デフォルトでは、システムはセキュリティ更新を1日に1回チェックし、利用可能な更新がある場合は自動的にインストールします。これを行うには、サーバーを自動的に再起動できるため、必要なサービスを起動時に自動的に開始するように構成することが重要です。無人アップグレードでは重要なセキュリティ更新のみがインストールされ、インストールされているプログラムとディストリビューションのバージョンは変更されないことに注意してください。
