Berikutnya> Mode pengoperasian server surat

Untuk dapat bekerja dengan email, Anda memerlukan klien email. Ini bisa berupa klien web, katakanlah gmail, owa, roundcube, atau aplikasi di komputer - pandangan, burung petir, dll. Misalkan Anda sudah terdaftar di beberapa layanan email dan Anda perlu mengonfigurasi klien email Anda. Anda membuka program dan meminta Anda untuk data: nama akun, alamat email dan kata sandi Anda.



Setelah Anda memasukkan data ini, klien email Anda akan mencoba mencari informasi tentang server email Anda. Ini dilakukan untuk menyederhanakan konfigurasi koneksi ke server, karena sebagian besar pengguna tidak mengetahui alamat dan protokol koneksi. Untuk melakukan ini, klien email menggunakan berbagai metode mencari informasi tentang pengaturan server dan koneksi. Metode ini dapat bervariasi tergantung pada klien email.



Misalnya, Outlook menggunakan metode "autodiscover", klien menghubungi server dns dan meminta entri autodiscover tertentu yang dikaitkan dengan domain email yang Anda tentukan dalam pengaturan klien email Anda. Jika administrator telah mengonfigurasi catatan ini di server DNS, itu menunjuk ke server web.



Setelah klien email menemukan alamat server web, ia menghubungi dan menemukan file yang sudah disiapkan dengan pengaturan untuk menghubungkan ke server email dalam format XML.



Dalam kasus Thunderbird, klien email membuang dengan pencarian DNS untuk autodiscover dan segera mencoba untuk terhubung ke server web autoconfig. dan nama domain yang ditentukan. Dan dia juga mencoba mencari file pengaturan koneksi dalam format XML di server web.



Jika klien email tidak menemukan file dengan pengaturan yang diperlukan, ia akan mencoba menebak pengaturan di antara yang sering digunakan. Misalnya, jika domain disebut example.com, maka server surat akan memeriksa apakah ada server dengan nama imap.example.com dan smtp.example.com. Jika ditemukan, ia akan mendaftarkannya di pengaturan. Jika klien surat tidak dapat menentukan alamat server surat dengan cara apa pun, itu akan meminta pengguna untuk memasukkan data koneksi sendiri.



Kemudian Anda akan melihat 2 bidang untuk server - alamat server surat masuk dan alamat server surat keluar. Sebagai aturan, dalam organisasi kecil alamat ini bertepatan, bahkan jika mereka ditunjukkan melalui nama DNS yang berbeda, dan di perusahaan besar mereka bisa menjadi server yang berbeda. Tetapi tidak masalah apakah itu server yang sama atau tidak - layanan di belakang mereka berbeda. Salah satu bundel layanan email yang paling populer adalah Postfix & Dovecot. Di mana Postfix bertindak sebagai server surat keluar (MTA - mail transfer agent), dan Dovecot bertindak sebagai server email masuk (MDA - mail delivery agent). Berdasarkan namanya, Anda dapat menebak bahwa Postfix digunakan untuk mengirim surat, dan Dovecot digunakan untuk menerima surat oleh klien surat. Server mail sendiri berkomunikasi satu sama lain melalui protokol SMTP - mis. Dovecot (MDA) diperlukan untuk pengguna.



Misalkan kita mengkonfigurasi koneksi ke server email kami. Mari kita coba mengirim pesan. Dalam pesan tersebut kami menunjukkan alamat Anda dan alamat penerima. Sekarang, untuk mengirim pesan, klien email Anda akan mengirim pesan ke server email keluar Anda.



Ketika server Anda menerima pesan, ia akan mencoba mencari siapa yang mengirim pesan ini. Server Anda tidak dapat mengetahui alamat semua server email dengan hati, sehingga beralih ke DNS untuk menemukan catatan MX khusus - menunjuk ke server mail untuk domain ini. Entri ini dapat bervariasi untuk subdomain yang berbeda.



Setelah ia menemukan alamat server penerima, ia mengirim pesan Anda melalui SMTP ke alamat ini, di mana server surat penerima (MTA) akan menerima pesan dan meletakkannya di direktori khusus, yang juga melihat layanan yang bertanggung jawab untuk menerima pesan (MDA) oleh klien.



Lain kali klien email penerima meminta server pesan masuk untuk pesan baru, MDA akan mengiriminya pesan Anda.

Tetapi karena server e-mail bekerja di Internet dan siapa pun dapat terhubung ke mereka dan mengirim pesan, serta server e-mail banyak digunakan oleh berbagai perusahaan untuk bertukar data penting, ini adalah berita menarik bagi penyerang, terutama untuk spammer. Oleh karena itu, di server surat modern ada banyak langkah tambahan untuk mengonfirmasi pengirim, memeriksa spam, dll. Dan saya akan mencoba mengungkap banyak dari topik ini di bagian berikut.
