3DでのJavaScript：Three.jsの概要

こんにちは、ハブル！Bret Cameronによる記事「JavaScript in 3D：An Introduction to Three.js」の翻訳を紹介します。

前書き


Three.jsは強力なツールです。許容できるパフォーマンスのブラウザで3D設計を使用すると役立ちます。はじめにThree.jsは、特にこれまでに3Dプログラミングの世界に突入したことがない場合は、注意が必要です。

UnityとC＃のゲームエンジンの基本的な経験はありますが、それでも多くの概念は初めての私です。初心者の開発者向けのリソースは非常に少ないという結論に達したので、この記事を書くことにしました。その中で、ポリゴンメッシュやマテリアルからジオメトリ、ローダーなど、Three.jsシーンの基本的な要素を検討します。

この記事の最後では、将来のWebプロジェクトに次元を追加するために必要な基本的な側面をしっかりと理解できます。Ben Houston、Thomas Diewald、StrykerDoesAnimationによる

Three.jsの例。

ベクターとコンテナー-基本的なビルディングブロック


Three.jsには、Vector3とBox3の 2つの主要なクラスがあることがよくあります。3Dに慣れていない場合、これは少し抽象的に聞こえるかもしれませんが、何度も会うことになります。

Vector3


3つの数値x、y、zを含む最も基本的な3Dクラス。数値は、3D空間内の点の座標または方向と長さです。例えば：

const vect = new THREE.Vector3(1, 1, 1);

Three.jsでのコンストラクタのほとんどは、型のオブジェクト受け入れるのVector3例えば、入力引数としてをBOX3

ボックス3


このクラスは、直方体（3Dコンテナー）を表します。その主なタスクは、他のオブジェクトの周りにコンテナを作成することです。これが、3Dオブジェクトが収まる最小の直方体です。各Box3は、x、y、z軸を中心に配置されます。Vector3を使用してコンテナを作成する方法の例：

const vect = new THREE.Vector3(1, 1, 1);
const box = new THREE.Box3(vect);

既存の3Dオブジェクトの周囲にコンテナーを作成する方法の例：

const box = new THREE.Box3();
box.setFromObject(object);

この詳細な知識がなくてもグリッドを作成できますが、モデルの作成または変更を開始するとすぐに、これらのクラスは間違いなく役立ちます。ここで、抽象化から離れて、より目に見えるものに移動します。

ポリゴンメッシュ


Three.jsでは、ステージの主な視覚要素はMeshです。これは、三角形の長方形（多角形メッシュ）で構成される3Dオブジェクトです。これは2つのオブジェクトを使用して構築されます。
ジオメトリ -形状を決定し、マテリアル -外観を決定します。

それらの定義は少し混乱しているように見えるかもしれません（たとえば、Geometryクラスに色に関する情報が含まれている可能性があります）が、主な違いは正確にそれです。

ジオメトリ


達成したいタスクに基づいて、Three.js内でジオメトリを定義するか、ファイルから別のジオメトリをインポートすることができます。THREE.TorusKnotGeometryなどの

関数を使用すると、1行のコードで複雑なオブジェクトを作成できます。これについてはすぐに説明しますが、まずはより単純なフォームについて検討します。 最も単純な3D形状（直方体またはコンテナ）は、width、height、およびdepthパラメータで指定できます。


const geometry = new THREE.BoxGeometry( 20, 20, 20 );


球の場合、radius、widthSegments、およびheightSegmentsパラメータの値は最小です。最後の2つの変数は、球体を表すためにモデルが使用する必要がある三角形の数を示します。数値が大きいほど、見た目が滑らかになります。

const geometry = new THREE.SphereGeometry( 20, 64, 64 );


鋭い形状や三角形の形状を作りたい場合は、円錐を使用できます。彼の議論は、前の2つの図の議論を組み合わせたものです。以下では、radius、widthSegments、radialSegmentsを規定しています。

 const geometry = new THREE.ConeBufferGeometry( 5, 20, 32 );


これは、最も一般的な数値の一部にすぎません。Three.jsには、ボックス内に多くの形状があり、ドキュメントに記載されています。この記事では、TorusKnotGeometryメソッドに基づいて構築されたより興味深いフォームを見ていきます。

なぜこれらの形状は見た目とまったく同じように見えるのですか？この質問はこの記事の範囲を超えていますが、1行のコードで非常に興味深い形状を得ることができるため、パラメーター値を試してみることをお勧めします。

const geometry = new THREE.TorusKnotGeometry(10, 1.3, 500, 6, 6, 20);

材料


ジオメトリは3Dオブジェクトの形状を定義しますが、外観は定義しません。これを修正するには、材料が必要です。

Three.jsは、箱から出してすぐに10のマテリアルを提供します。それぞれに独自の利点とカスタマイズ可能なパラメーターがあります。最も有用なものの一部のみを検討します。



MeshNormalMaterial


クイックスタートに便利ですし、開始します。

私たちは始まりますMeshNormalMaterial、我々は上記の例で使用される多色材料。これは、RGBパネルの法線ベクトルに対応します。つまり、3D空間でのベクトルの位置を決定するために色が使用されます。

const material = new THREE.MeshNormalMaterial();

マテリアルの色を変更したい場合は、CSSフィルターを使用して彩度を変更できます。 
 filter: hue-rotate(90deg) .


私の経験では、この資料は素早い階層と起動に役立ちます。オブジェクトをより細かく制御するには、別のものを使用することをお勧めします。

Meshbasicmaterial


スケルトンのみを表示する

場合に便利です。図を単一の色にしたい場合は、ライティングが適用されていない場合にのみMeshBasicMaterialを使用できます。モデルのスケルトンをレンダリングする際にそのマテリアルを使用すると便利だと思いました。スケルトンのみを描画するには、{wireframe：true}をパラメーターとして渡す必要があります。

const material = new THREE.MeshBasicMaterial({ 
  wireframe: true, 
  color: 0xdaa520
});

このマテリアルの主な欠点は、マテリアルの深さに関する情報が完全に失われることです。各マテリアルにはスケルトンのみを表示するオプションがありますが、1つのマテリアルのみが深さ不足の問題を解決します-MeshDepthMaterial


MeshLambertMaterial


パフォーマンスは高くなりますが、精度

は低くなります。これは光を考慮した最初のマテリアルなので、シーンに光を追加する必要があります。以下のコードでは、黄色の色合いのスポットライトを追加して、暖かい効果を作成しています。

const scene = new THREE.Scene();
const frontSpot = new THREE.SpotLight(0xeeeece);
frontSpot.position.set(1000, 1000, 1000);
scene.add(frontSpot);
const frontSpot2 = new THREE.SpotLight(0xddddce);
frontSpot2.position.set(-500, -500, -500);
scene.add(frontSpot2);

次に、図の材料を追加します。私たちの姿は飾りのようなものなので、もっと金色を加えることをお勧めします。別のパラメーター、emissiveは、オブジェクト自体（光源なし）からのオブジェクトの色です。多くの場合、暗い色としてよりよく機能します。たとえば、以下の例のように、灰色の濃い色合いです。

const material = new THREE.MeshLambertMaterial({
  color: 0xdaa520,
  emissive: 0x111111,
});


下の例でわかるように、色は多少正確ですが、ライトとの相互作用の方法はリアルさを増しません。これを修正するには、MeshPhongMaterialまたはMeshStandardMaterialを使用する必要があります。

MeshPhongMaterial


中程度のパフォーマンスと中程度の精度に役立ちます。

このマテリアルは、パフォーマンスとレンダリング精度の間の妥協点を提供するため、MeshLambertMaterialよりも正確なレンダリングとともに生産性を必要とするアプリケーションに適したオプションです。

これで、表面反射の明るさと色に影響を与える鏡面反射光プロパティを変更できます。通常、放射プロパティが暗い場合、鏡面反射光は明るい色に最適です。以下ではライトグレーを使用しています。

const material = new THREE.MeshPhongMaterial({
  color: 0xdaa520,
  emissive: 0x000000,
  specular: 0xbcbcbc,
});


視覚的には、上からの画像はより説得力のある光を反射しますが、それでも完璧ではありません。白色光が明るすぎて、マテリアルがメタリックよりもリブのように見えます（このために努力しています）。MeshStandardMaterialを使用すると、より良い結果を得ることができます。

MeshStandartMaterial


高精度に役立ちますが、生産性は低くなります。

これは、最も正確な材料ですが、使用するとより多くの電力を使用するコストが発生します。MeshStandartMaterialは、追加のmetalnessパラメータと荒さパラメータとともに使用されます。それぞれのパラメータは0〜1の値を取ります。metalness

パラメータは、オブジェクトの反射に影響を与え、金属の性質に近づきます。これは、セラミックなどの誘電体とは異なり、金属などの導電性材料は反射特性が異なるためです。粗さは、カスタマイズ用のレイヤーを追加します。光沢の反対として想像できます。0-非常に光沢があり、1-非常にマットです。



const material = new THREE.MeshStandardMaterial({
  color: 0xfcc742,
  emissive: 0x111111,
  specular: 0xffffff,
  metalness: 1,
  roughness: 0.55,
});


これは、Three.jsで提供されるすべての中で最も現実的な資料ですが、

以下で説明する最もリソースを消費する資料も、私が最も頻繁に会った資料であり、ドックのすべてのオプションを見ることができます。

ローダー


上で説明したように、手動でジオメトリとポリゴンメッシュを定義できます。実際には、ファイルからジオメトリをロードする可能性が高くなります。さいわい、Three.jsには、多くの3D形式をサポートするダウンローダーがほとんどありません。

メインObjectLoaderは、JSONオブジェクト/シーン形式を使用してJSONファイルをロードします。ほとんどのブートローダーは手動でインポートする必要があります。サポートされているブートローダーの完全なリストをここで見つけてインポートできます。以下は、インポートできるものの小さなリストです。

// GLTF
import { GLTFLoader } from 'three/examples/jsm/loaders/GLTFLoader.js';
// OBJ
import { OBJLoader } from 'three/examples/jsm/loaders/OBJLoader.js';
// STL
import { STLLoader } from 'three/examples/jsm/loaders/STLLoader.js';
// FBX
import { FBXLoader } from 'three/examples/jsm/loaders/FBXLoader.js';
// 3MF
import { 3MFLoader } from 'three/examples/jsm/loaders/3MFLoader.js';

オンラインでの表示に推奨される形式はGLTFです。これは、この形式が「実行時にアセットを配信することを目的としており、転送用にコンパクトで、ダウンロード用に高速であるため」です。

もちろん、特定の種類のファイルを選択する理由はたくさんあります（たとえば、品質が優先される場合や、3Dプリントに正確さが必要な場合）。最高のオンラインパフォーマンスは、GLTFをインポートするときです。

import { GLTFLoader } from 'three/examples/jsm/loaders/GLTFLoader.js';
import model from '../models/sample.gltf';
let loader = new GLTFLoader();
loader.load(model, function (geometry) {
  // if the model is loaded successfully, add it to your scene here
}, undefined, function (err) {
  console.error(err);
});

フレームワークを使用する必要がありますか？


最後に、お気に入りのフレームワークでThree.jsを使用するかどうかについて説明します。現時点では、Reactに適した3つのファイバーの反応パッケージがあります。Reactユーザーにとって、このようなパッケージを使用することには明らかな利点があります-コードを再利用できるようにするコンポーネントを操作するための構造を維持します。Three.jsについて書かれたほとんどのオンライン資料はVanila JSのThree.jsに関連している

ため、初心者には、通常のVanila JSから始めることをお勧めします。私の学習経験に基づくと、これはパッケージを通じて学習するのが混乱し、難しい場合があります。たとえば、Three.jsオブジェクトとメソッドをコンポーネントとプロップに変換する必要があります。（Three.jsを学ぶとすぐに、任意のパッケージを使用できます）。

フレームワークにThree.jsを追加する方法


Three.jsは、アプリケーションの任意のHTMLオブジェクトに追加できるHTMLオブジェクト（ほとんどの場合、renderer.domElementと呼ばれます）を提供します。たとえば、あなたが持っている場合はdiv要素でIDを=「threejs」単にあなたのThree.jsコードに次のコードを含めることができます。

document.getElementById('threejs').appendChild(renderer.domElement);

一部のフレームワークには、ツリーのDOMノードにアクセスするための優先パスがあります。たとえば、REFでは、リアクト$ REFのVueかでngRef角度では、それはDOM要素への直接アクセスを背景に巨大なプラスのように見えます。例として、Reactの簡単な実装を見てみましょう。

Reactの戦略


Reactを使用する場合、Three.jsファイルをコンポーネントの1つに埋め込む方法が1つあります。ThreeEntryPoint.jsファイルに、次のコードを記述します。

export default function ThreeEntryPoint(sceneRef) {
  let renderer = new THREE.WebGLRenderer(); 
  // ...
  sceneRef.appendChild(renderer.domElement);
}

これを、1つの引数（コンポーネント内の要素への参照）を取る関数としてエクスポートします。これでコンポーネントを作成できます

import React, { Component } from 'react';
import ThreeEntryPoint from './threejs/ThreeEntryPoint';
export default class ThreeContainer extends Component {
componentDidMount() {
    ThreeEntryPoint(this.scene);
  }
render() {
    return (
      <>
        <div ref={element => this.scene = element} />
      </>
    );
  }
}

インポートしたThreeEntryPoint関数をcomponentDidMountメソッドで呼び出し、リンクを使用して新しいdivを引数として渡す必要があります。
このアプローチの実際の例として、リポジトリを複製して自分で試すことができます：https : //github.com/BretCameron/three-js-sample。
