新規および削除


メモリの割り当ては静的および動的にすることができます。静的メモリ割り当ては、プログラムのコンパイル中の1回限りのメモリ割り当てと呼ばれ、静的メモリの量は実行時に変化しません。古典的な例は、整数変数または配列の宣言です。しかし、プログラマがコンテナ内に必要な要素の数を事前に知らない場合はどうでしょうか。
必要に応じてプログラムのニーズに合わせてメモリの割り当てを整理する必要がある場合は、動的メモリの使用をお勧めします。new
演算子はC ++ で動的メモリを割り当てる責任があり、deleteはそれをクリアする責任があります。new 演算子は、その操作の結果、クラスの新しいインスタンスへのポインターを返します。

構文は次のとおりです。

| データ型（T1）ポインター | * | ポインター名 | = 新規 | タイプT1 |; new

演算子の後、たとえば、コンストラクタを使用してクラスのフィールドを初期化できます。 プログラマが割り当てを制御できなくなったときに、同じメモリリークが発生することに注意してください。

ポインタ


ポインタは、C ++で動的メモリを操作する責任があります。これは、食欲が初心者を台無しにするトピックです。*

演算子を使用してポインタを宣言できます。デフォルトでは、メモリのランダムな領域を指します。必要なメモリ領域にアクセスできるように、目的の変数にリンク（演算子＆）を渡す必要があります。 ポインタ自体は単にメモリセルのアドレスであり、このセルに格納されているデータにアクセスするには、逆参照する必要があります。


重要な後退


逆参照せずにポインタを表示しようとすると、ポインタが指すメモリ領域の値の代わりに、このメモリ領域のアドレスが表示されます。
ポインターを逆参照するには、名前の前に*演算子を置くだけです。

このようにして、メモリを自由に操作できます。だからこそ、記憶を操作するときに、「足で自分を撃つ」ことができます。値自体はコピーされず、特定のアドレスへのリンクのみが割り当てられるため、ポインタの操作がはるかに高速になることに注意してください。

プログラマーに戻りましょう。今、私たちは中クラスのフィールドにする必要があるとのプライベートセクションを、ふさわしくOOPからカプセル化の原則として、我々はする必要がゲッターこれらのフィールドへの読み取りアクセスを取得します。しかし、2つのフィールドではなく100のフィールドがあるとします。このために、それぞれに独自のアクセサーを作成する必要がありますか？

ネタバレ
, .

これを行うには、void型の「アクセサ」を作成し、参照によって引数を渡します。引数を参照渡しすることの意味は、引数の値はコピーされず、実引数のアドレスのみが送信されることです。したがって、そのような引数の値を変更すると、現在の引数のメモリセルのデータも変更されます。
参照による引数の受け渡しは、値による受け渡しよりも速いため、これは全体的なパフォーマンスにも影響します。そして、これは要素の大規模なコレクションについては言うまでもありません。

たとえば、内部のgetParamsメソッドは受信引数を変更し、スコープ内を含め、呼び出し元の値を変更します。
ポインタは、配列をナビゲートするのに役立ちます。データ構造の理論から、配列はメモリの連続した領域であり、要素が次々に配置されていることがわかります。
つまり、ポインタの値を配列内の要素が占めるバイト数に変更すると、ポインタが配列の境界を超えるまで各要素に到達できます。プログラマ
配列の最初の要素を指す別のポインタを作成します。

class Programmers{
public:
    Programmers(){}
    Programmers(int iWeight, int iAge){
        this->weight = iWeight;
        this->age = iAge;
    }
    //    ,   main    
    void getParams(int &w, int &a){
        w=weight;
        a=age;
    }

private:
    int weight;
    int age;
};

int main()
{
    int size = 9;
    Programmers *prog [size];
    Programmers *ptr=nullptr;

    for (int i =0;i<size;i++) {
        ptr=new Programmers(i+100,i);
        prog[i]=ptr;
    }

    int w,a;
    int count = 9;
//   
//       
    Programmers **iter = prog;
   
    for (int i=0;i<count;i++)
    {
        ptr = *iter++;
        ptr->getParams(w,a);
        if(*(iter-1) != nullptr){
            delete *(iter-1);
            ptr = nullptr;
        }
        cout<<w<<"\t"<<a<<endl;
    }
    return 0;
}



この例では、ポインターのアドレスの値を変更すると、メモリの別の領域にアクセスできるという事実の本質を伝えたいと思います。

リスト、ベクターなどのデータ構造 ポインタに基づくため、動的データ構造と呼ばれます。そして、それらを反復するためには、反復子を使用する方が正しいです。イテレータはデータ構造の要素へのポインタであり、コンテナの要素へのアクセスを提供します。
