「Guide to Competitive Programming」は英語からロシア語に翻訳されました。英語とロシア語に加えて、韓国語の出版物がリリースされました。

この本の著者は、プログラミングとアルゴリズムの教育に豊富な経験を持つヘルシンキ大学アアルト（フィンランド）[3]の教師兼研究者であり、国際プログラミングコンクールでフィンランドチームのコーチを務めるAntti Laaxsonenです。彼の評価は2347で、Codeforcesポータルでは「pllk」というニックネームで「国際マスター」のステータスを獲得しています[4]。 2008年、彼のA. Laaxsonenは、自国の情報学オリンピックの主催者の1人になりました。 2016年-情報学におけるバルト海オリンピックの科学監督。

この本の対象読者はすべて、オリンピアードプログラミングの分野に関心を持っているか、またはその両方で働いています。ただし、資料を完全に同化するには、プログラミングの基礎に関する知識が必要ですが、作者は読者にアルゴリズムの設計やコンテストへの参加の経験を要求しません。これらすべてにより、私たちはこの作品を幅広い興味のある読者に推薦することができます。初心者にとって、それは現代のオリンピックプログラミングの紹介となり、経験豊富なスペシャリストが知識の体系化を支援し、それらのリファレンスになります。

本の資料の提出は、単純なものから複雑なものまで行われます。最初に、彼は本の目標と目的、Olympiadプログラミング、CSESのタスクのコレクション[5]、およびOlympiadプログラミングに関するその他の関連する本を理解します。

必要な理論的基礎を受け取った読者は、練習に進む準備ができています。プログラミング手法は次のトピックです。その中に著者は、本のすべての例を実装するC ++言語（C11標準）の基本を含めました。再帰的アルゴリズムとビット単位の操作の分析。

例として、「アルゴリズム設計の選択された質問」の章からタスクの例を示します。

数字を並行して表示するアルゴリズムは、数値の個々の数字をビット単位の演算を使用して並行して操作できるという事実に基づいています。したがって、アルゴリズムを設計する方法の1つは、アルゴリズムのステップをビットごとの演算を使用して効果的に実装できるように提示することです。

ハミング距離ハミング距離


同じ長さのラインaとbの間のハミング（a、b）は、これらのラインが異なる位置の数です。例：

ハミング（01101、11001）=2。

次の問題を検討してください：長さkのnビット文字列が与えられ、2つの文字列間の最小ハミング距離を計算します。たとえば、行[00111、01101、11110]の場合、答えは2です。

  • ハミング（00111、01101）= 2;
  • ハミング（00111、11110）= 3;
  • ハミング（01101、11110）= 3。

この問題は、2行ごとのハミング距離を計算することにより、「正面から」解決できます。そのようなアルゴリズムの時間の複雑さは（n

O(n2k)2

k）。行aとbの間の距離を計算するには、次の関数を使用します。

int hamming(string a, string b) {
int d = 0
for (int i = 0; i < k; i++) {
if (a[i] != b[i]) d++;
}
return d;
}

しかし、文字列はビットで構成されているため、文字列を整数として保存し、ビットごとの演算を使用してそれらの間の距離を計算することで、ソリューションを最適化できます。特に、k≤32の場合、文字列はint値として保存でき、距離を計算するには次の関数を使用します：

int hamming(int a, int b) {
return __builtin_popcount(a^b);
}

ここで、EXCLUSIVE OR演算は、ユニットがaとbが異なる位置にある行を作成します。次に、__ builtin_popcount関数によって単位の桁数が計算されます。この表は、元のアルゴリズムとアルゴリズムを、最新のコンピューターでの実行時間に関してビットの並列ビューと比較した結果を示しています。ビットを並列表示するアルゴリズムは、約20倍高速であることが判明しました。
