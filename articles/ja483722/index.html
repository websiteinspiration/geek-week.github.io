小売業のジオアナリティクス、パート1：ビジネスの場所を選択するプロセスを自動化します。2GIS + MS Azure + ML



みなさん、こんにちは！

私の名前はセルゲイコンコフです。私はビジネスレポート＆データのデータアーキテクトです。
この記事では、小売企業での地理分析と機械学習の可能性について説明します。これは、クライアントの実際のプロジェクトである美容サロンの大規模ネットワークに基づいています。コードや技術的な詳細はあまりありません。テクノロジー、アプリケーション、統合のレビューに注意が払われます。

仕事


サービス部門には活発に発展している会社があります。同社は主にショッピングセンター（TC）にいくつかの美容サロンを毎月オープンしています。

タスクは、発見に最適な場所を選択し、意思決定の時間とコストを削減することです。

実装計画の例


  1. 場所の魅力を評価するのに役立つショッピングセンターのデータを受け取ります。
  2. 得られたデータと、会社がすでに運営しているショッピングセンターのサロンの結果との関連を見つけてみましょう。
  3. 新しいショッピングセンターの魅力を評価する方法論を作成します。
  4. 私たちの会社がまだ存在しないショッピングセンターの推定値を計算するための機械学習モデルを作成します。
  5. 私たちは、小売店を開く場所を選択する際の専門家の作業を自動化するためのツールを作成します。

それは、「経験豊富な」人がビジネスのための場所を決める必要がある人について考えていることです。上級者は分析も自由に利用でき、ExcelまたはSQLではこれらすべての要因に基づいて適切な正当化を行うことができます。

このデータの収集と処理には時間がかかります。これを最適化してみましょう。

データはどこで入手できますか？市場に出回っているジオサービスを見てみましょう。主なものは、Yandex Maps、2GIS、Google Mapsです。機能を比較した後、2GISで解決しました。
私たちの意見では、サービスの主な利点：データの正確性、関連性、完全性、ユーザーフレンドリーなAPI。ただし、他のメーカーのソリューションを使用できます。

2GIS APIへのアクセスは、サービスWebサイトのフォームから取得できます。アクセスは有料であり、リクエストの数によって異なりますが、2GISとの契約でデータの使用方法を明記する必要があります。これが、データをリクエストする理由、それをどうするかなどです。

2GIS APIの操作方法について少し説明します。

このサービスでは、特定の地域（都市、地区、通りなど）での組織の場所を選択できます。フィルターとして、組織のルーブリック（活動のタイプ）を指定できます。ショッピングセンターのカテゴリを持つすべての組織の検索をリクエストします。その結果、テリトリー内のショッピングセンターのリストが表示されます。各ショッピングセンターの応答では、いわゆる住宅識別子-building_idも返されます。受信したbuilding_idの1つを使用して、建物内にあるすべての組織にリクエストを送信します。このショッピングセンターにあるすべての組織の名前と見出しを含むリストを取得します。



したがって、特定のショッピングセンターにあるすべてのショップ、サロン、レストラン、その他の組織のリストを受け取りました。興味のあるショッピングセンターごとにそのようなリクエストを出すことができます。

プロジェクトインフラストラクチャ


実装には、MS Azureのクラウドを選択しました。つまり、必要なものがすべてあります。

  • データウェアハウス-Azure SQLデータベース ;
  • ETL統合ツール- データファクトリ ;
  • OLAP- Analysis Services Azureのサービス。
  • MLサービス-Azure Machine Learning ;
  • ボット開発用のエンジン-Azure Bot Service（記事の後半で必要になります）。

2GISから取得したすべてのデータは、Azure SQLデータベーステーブルに読み込まれます。
ダウンロード用のすべてのスクリプトをAzure Data Factoryパッケージに入れました。これにより、データの関連性を維持するためのスケジュールに従って、また特定の必要なショッピングセンターのリクエストに応じて、データを一括で（たとえば、特定の都市のすべてのショッピングセンターについて）ダウンロードできます。

データを操作する


2GISからのデータとサロンの実際の結果との関係の検索を開始できます。これを行うには、ショッピングセンターのデータを、会社がすでに存在するストアにアップロードします。そこで、ERPからのサロンの仕事に関するデータ（収益、チェックの数、マージン、訪問者の数）をアップロードします。

この段階で、スペシャリストとデータサイエンスプロジェクトを結び付けます。そのために、収集したデータとAzure Machine Learning環境に基づいてOLAPキューブを展開します。

データサイエンティストが実施した調査によると、次の要因がサロンの仕事の結果に最も強く影響します（収益と限界）。

  • ショッピングセンターがある都市。
  • ショッピングセンター内の店舗数。
  • ショッピングセンターの競合他社の数。
  • 特定のブランドの存在;
  • 半径2 km以内のその他のショッピングセンターの数。

ショッピングセンターの魅力を評価するために、1から10までのスケールを導入します。魅力評価の透明性のために、どの要素とそれらが最終的な評価にどのように影響したかをユーザーに正確に示す必要があります。以下は、ショッピングセンターの1つの評価の例です。
