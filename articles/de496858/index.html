6 Open Source Tools für RPA





Gartner nannte RPA (Robotic Process Automation) 2018 das am schnellsten wachsende Segment von Unternehmenssoftware mit einem weltweiten Umsatzwachstum von 63%, schreibt EnterprisersProject. Für den Build können Sie Ihre eigenen Bots von Grund auf neu schreiben, vorausgesetzt, Sie haben die richtigen Leute und das richtige Budget. Mit dem Problem „Build versus Purchase“ ist jedoch etwas gemeinsam: Derzeit laufen mehrere Open-Source-RPA-Projekte.Dies gibt IT-Managern und Fachleuten die Möglichkeit, RPAs zu recherchieren, ohne selbst von vorne anfangen zu müssen oder einen Deal mit einem kommerziellen Lieferanten abzuschließen, bevor es wirklich funktioniert, um eine Strategie zu entwickeln. Chrome 81-Version




Google hat die Veröffentlichung des Chrome 81-Webbrowsers eingeführt. Gleichzeitig ist laut OpenNET eine stabile Version des kostenlosen Chromium-Projekts verfügbar, das als Grundlage für Chrome dient. Die Veröffentlichung erinnert daher daran, dass sich der Chrome-Browser durch die Verwendung von Google-Logos, das Vorhandensein eines Benachrichtigungssystems im Falle eines Absturzes, die Möglichkeit zum Laden des Flash-Moduls bei Bedarf, Module zum Abspielen von geschützten Videoinhalten (DRM) und ein System zum automatischen Installieren von Updates und Übertragen bei der Suche nach RLZ-Parametern auszeichnet. Ursprünglich war die Veröffentlichung von Chrome 81 am 17. März geplant. Aufgrund der SARS-CoV-2-Coronavirus-Pandemie und der Übertragung von Entwicklern auf die Arbeit zu Hause verzögerte sich die Veröffentlichung jedoch. Die nächste Version von Chrome 82 wird übersprungen, die Veröffentlichung von Chrome 83 ist für den 19. Mai geplant. Einige Neuerungen:

  1. Die FTP-Unterstützung ist deaktiviert.
  2. Für alle Benutzer ist die Funktion zum Gruppieren von Registerkarten aktiviert, mit der Sie mehrere Registerkarten mit ähnlichen Zwecken zu visuell getrennten Gruppen kombinieren können.
  3. Es wurden Änderungen an den Nutzungsbedingungen von Google vorgenommen, in denen ein separater Abschnitt für Google Chrome und Chrome OS angezeigt wurde.
  4. Es ist stabilisiert und die Benutzeroberfläche des Badging-Programms ist jetzt außerhalb von Origin Trials verteilt, sodass Webanwendungen Indikatoren erstellen können, die auf einem Bedienfeld oder Startbildschirm angezeigt werden
  5. Verbesserungen der Tools für Webentwickler;
  6. Das Deaktivieren der TLS 1.0- und TLS 1.1-Unterstützung wurde bis zur Veröffentlichung von Chrome 84 verzögert.

Das Chrome OS-Update wurde ebenfalls veröffentlicht. Es ist mit einfachen Navigationsgesten und einem neuen Quick Shelf-Dock ausgestattet, berichtet CNet.

FlowPrint, ein Toolkit zum Identifizieren von Anwendungen basierend auf verschlüsseltem Datenverkehr, ist verfügbar





Der Code des FlowPrint-Toolkits wurde veröffentlicht, mit dem mobile Netzwerkanwendungen ermittelt werden können, indem der verschlüsselte Datenverkehr analysiert wird, der während des Betriebs der Anwendung erstellt wurde, berichtet OpenNET. Es ist möglich, sowohl typische Programme zu identifizieren, für die Statistiken gesammelt werden, als auch die Aktivität neuer Anwendungen zu identifizieren. Der Code ist in Python geschrieben und wird unter der MIT-Lizenz vertrieben. Das Programm implementiert eine statistische Methode, die die Merkmale des Datenaustauschs definiert, die für verschiedene Anwendungen charakteristisch sind (Verzögerungen zwischen Paketen, Merkmale von Datenflüssen, Änderungen der Paketgröße, Merkmale einer TLS-Sitzung usw.). Für mobile Android- und iOS-Anwendungen beträgt die Erkennungsgenauigkeit der Anwendung 89,2%. In den ersten fünf Minuten der Analyse des Datenaustauschs können 72,3% der Anträge ermittelt werden. Die Genauigkeit der Identifizierung neuer Anwendungen, die zuvor noch nicht gesehen wurden,beträgt 93,5%.

"Die Linux Foundation wird die seL4 Foundation und die Community unterstützen, indem sie Fachwissen und Dienstleistungen bereitstellt, um das Engagement der Community und ihrer Mitglieder zu erhöhen und das Betriebssystem des Ökosystems auf die nächste Ebene zu bringen ", sagte Michael Dolan, Vizepräsident für strategische Programme bei der Linux Foundation.

Linux Exec-Systemaufrufe sollten in zukünftigen Kerneln weniger anfällig für Deadlocks sein





Kontinuierliche Arbeit an Exec-Code unter Linux sollte es weniger anfällig für Deadlocks in zukünftigen Kernel-Versionen machen. Die aktuelle Exec-Funktionalität im Kernel ist „extrem anfällig für Deadlocks“, aber Eric Biederman und andere haben daran gearbeitet, diesen Code zu löschen und in einen besseren Zustand zu bringen, um mögliche Deadlocks zu vermeiden. Die Bearbeitung für den Linux 5.7-Kernel war der erste Teil der Exec-Verfeinerung, die es einfacher macht, komplexere Fälle zu identifizieren, und es besteht die Hoffnung, dass für Linux 5.8 der Code zum Lösen von Exec-Deadlocks bereit ist. Linus Torvalds akzeptierte Änderungen für 5.7, reagierte jedoch nicht sehr schmeichelhaft.

Microsoft schlug ein Linux-Kernelmodul vor, um die Systemintegrität zu überprüfen





Entwickler von Microsoft haben den IPE-Mechanismus (Integrity Policy Enforcement) eingeführt, der als LSM-Modul (Linux Security Module) für den Linux-Kernel implementiert ist. Mit dem Modul können Sie eine allgemeine Integritätsrichtlinie für das gesamte System definieren, die angibt, welche Vorgänge gültig sind und auf welche Weise die Authentizität der Komponenten überprüft werden soll. Mit IPE können Sie angeben, welche ausführbaren Dateien ausgeführt werden dürfen, und sicherstellen, dass diese Dateien mit der von einer vertrauenswürdigen Quelle bereitgestellten Version identisch sind. Der Code ist unter der MIT-Lizenz geöffnet.
