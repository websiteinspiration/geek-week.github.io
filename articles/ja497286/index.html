Ludum Dare：開始の1週間前のチェックリスト

Ludum Dareは、週末にテーマを基にしてゼロからゲームを作成する、国際的なコンピュータゲーム加速型コンテストです。トピックはイベントの最初に明らかにされます。

コンテストは年に数回行われ、次のLudum Dare 46は1週間足らずで、つまり4月18日土曜日のモスクワ時間午前4時に始まります。

ここでは、このイベントへの参加を初めて検討している人に役立つヒントを紹介したいと思います。初心者向けのチェックリストです。

公式Webサイトには、参加のルール、およびゲームをホストし、評価のためにコミュニティの他のメンバーに送信するためのガイドが含まれています。私はルールについては詳しく述べません。しかし、サイトのホスティングと送信については、この記事の下部に記載されています。

始める前に、JamとCompoについて少し説明します。

ジャムとコンポ


Ludum Dareへの参加には、Jam-簡単なオプションとCompo-「高難易度モード」の2つのカテゴリがあります。

ジャム（ゲームを作成するための72時間）は、チームとシングルのLudum Dareカテゴリーです。任意のツールとライブラリを使用してゲームを作成できます。使用可能な任意のベースコードで開始できます。

Compo（ゲームを作成するための48時間）は、古典的なLudum Dareです。Compoのゲームは、1人で完全にゼロから作成します。ゲームとそのすべてのコンテンツは48時間以内に作成する必要があり、最終的にはすべてのソースコードを提供する必要があります。 

これらのモードの詳細と制限をよりよく理解し、参加する分野を決定するには、公式ガイドを読むことをお勧めします。

準備する方法


ゲームの作成は複雑な創造的なプロセスであり、ゲームのメカニズム、視覚的な部分、サウンドを理解するには時間がかかります。

あなたのゲームは次の点で他の参加者によって評価されることを覚えておくことが重要です：

  1. イノベーション（目新しさ）; 
  2. 楽しい（楽しみ）。
  3. テーマ
  4. グラフィック;
  5. オーディオ（サウンド）;
  6. ユーモア（ユーモア）;
  7. 気分（気分）;
  8. 全体（総合評価）。

私の意見では、Ludum Dareは、ゲームメカニクスとグラフィックスの開発により多くの時間を費やすには、よく知っているテクノロジーとツールを使用する方が良いイベントです。

ジャムの前


ジャムを実行するためにフォローするか、じっくり考えるのに適したいくつかのステップ：

  • Ludum Dare Webサイトでアカウントを登録します。
  • 参加するモード（Jam / Compo）を決定します。
  • 完成したゲームをホストすることを考えてください。
  • バージョン管理システムとプロジェクトホスティングシステムの使用を検討してください。
  • チームと責任を決定します。
  • チームメンバー間のコミュニケーション手段を事前に考えます。

デザインスクールでは、アイデアがない場合は他の人が何をしているかを見る必要があるとよく言われました。これは常にアドバイスです。インスピレーションの源を見るが、新しいバリエーションを作ってみる。実験。

また、ゲームの印象が最初の数秒で発生することを覚えておくことも重要です。ゲームがすぐにキャッチされない場合、それはおそらくそれは多くの人々に認められないでしょう。

ジャム中


経験上、ジャム中に時間を正確に評価することは非常に困難です。長い時間のようですが、これは罠です。したがって、すでに実証済みのソリューションに基づいて構築し、それらを組み合わせることをお勧めします。1-2個の興味深い機能を備えた小さな完成したプロジェクトを作成することをお勧めします。それ以外の場合は、詳細に気を取って何もすることができません。

トピックに関しては、文字通りそれに従う必要はありません。自分の解釈を考えた方がよい。解釈のアイデアの説明は、サイトの[送信]でゲームの名前の後に追加できます。 

ルールからの主なアイデア：
正式にはいいえ、テーマを使用する必要はありません。それを無視しても失格にはなりません。私たちにとって、テーマは単一の投票カテゴリにすぎません。一般的な提案として、Ludum Dareで良いスコアを獲得したい場合は、テーマを何らかの方法で使用してください。
トピックに固執しない場合は失格にはなりません。しかし、テーマは投票のカテゴリの1つです。したがって、Ludum Dareで良い成績を取得したい場合は、何らかの方法でトピックに固執することをお勧めします。

以下は公式サイトに掲載されているゲームホスティングガイドの基本情報です。

Jam / Compoの起動から申し込み時間の終了まで、いつでもゲームを提出できます。Compoはトピックが発表されてから48時間後に終了し、Jamはトピックが発表されてから72時間後に終了します。Ludum Dareウェブサイトのサイドバーにあるカウントダウンクロックをご覧ください。

CompoとJamはどちらも、完了後に送信する追加の時間を持っています。発生する可能性のあるあらゆる種類の技術的な問題があるため、ドキュメントを早めに送信することを強くお勧めします。 

サイトにアップロードする前に、次のものが必要です。

  1. Ludum Dareウェブサイトのアカウント。
  2. ゲームのいくつかのスクリーンショット。
  3. ゲームの名前。
  4. どこかにあるゲームへのリンク*;
  5. どこかにあるソースコードへのリンク（Compoのみ）**。

*必要なホスティングのタイプは、ネットワークゲームかデスクトップゲームかによって異なります。 

公式サイトのガイドには、いくつかの人気のあるプラットフォームがリストされています。これは完全なリストではなく、ゲームをホストする方法は他にもたくさんあります。

ゲームサイト：

  • Itch.io
     HTML5、Flash、Java、Unity、およびデスクトップゲームの無料ホスティングを可能にします。
  • GameJolt
    を使用すると、ウェブゲームやデスクトップゲームを無料でホストできます。
  • Newgrounds
     Flash（swf）およびHTML5（zip）でゲームを無料でホスティングできます。
  • Kongregate
     Unity、Flash、HTML5、WebGLで無料でゲームをホストできます。

プロジェクトホスティングシステム：

  • Github
  • Gitlab
  • BitBucket

クラウドストレージ（デスクトップゲームのホスティングに適しています）：

  • ドロップボックス
  • グーグルドライブ

無料のWebホスティングを提供している多くのサイトや、無料の試用版を提供している多くの有料サイトがあります。無料のWebホスティングを使用する場合は、Ludum Dareを起動する前にテストゲームをダウンロードすることをお勧めします。

独自のWebサイトまたはサーバーがある場合は、そこでゲームをホストできます。

**ゲームのソースコード（コンポカテゴリに参加しているもの）は、上記のプロジェクトホスティングシステムまたはクラウドストレージに配置できます。クラウドサービスを使用する場合は、すべてのソースコードを含む.zipファイルを配置します。

Ludum Dareに参加することは、快適ゾーンから抜け出し、自分自身をテストして、新しい体験を得る素晴らしい方法です。 

私の意見では、優先順位と時間の問題は依然として障害であり、多くのプロジェクトが失敗する理由です。コンテストでは、開発段階のほとんどを2〜3日で通過し、実際のプロジェクトでは数か月または数年かかる可能性がある多くのことを学ぶことができます。 

さらに、すべてのLudum Dareの本質は、スポーツへの関心と、ゲームプロジェクトを作成したいという願望だけでなく、多くの場合、将来の何かのプロトタイプであるだけでなく、新しい人々と仕事をする機会にもあります。よく調整されたチームが参加する必要はありません-多くの場合、人々は仕事の過程で明らかにされます。  

参加についてまだ疑問がある場合は、Ludum Dareは可能な範囲を超えた実験であると言えます。 

自分に無理をしないでください。楽しんで！
