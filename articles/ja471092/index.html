この記事では、コピー時のワイルドカードの使用、コピーcp時のコマンドのあいまいな動作、および漏れやクラッシュなしに膨大な数のファイルを正しくコピーする方法に関連するいくつかの明らかでないことが明らかになります。

/ sourceフォルダーから/ targetフォルダーにすべてをコピーする必要があるとします。

最初に頭に浮かぶのは：

cp /source/* /target

このコマンドをすぐに修正します。

cp -a /source/* /target

キー-aは、すべての属性、権限のコピーを追加し、再帰を追加します。権利を正確に複製する必要がない場合は、鍵で十分-rです。

コピーした後、すべてのファイルがコピーされたわけではないことがわかりました-などの点から始まるファイルは無視されました 。 なぜこれが起こったのですか？ ワイルドカードはシェルを処理するため（通常）。デフォルトでは、bashは隠しファイルとして扱うため、ピリオドで始まるすべてのファイルを無視します。この動作を回避するには、次のコマンドを使用して動作を変更する必要があります。

.profile
.local
.mc

shopt -s dotglob

再起動後もこの動作の変化を維持するには、このコマンドを使用してwildcard.shファイルをフォルダーに作成します/etc/profile.d（フォルダーに別のフォルダーが含まれている場合があります）。

また、ソースディレクトリにファイルがない場合、シェルはアスタリスクを何も置換できず、コピーも失敗します。この状況に対して、オプションfailglobとがありますnullglob。を設定する必要がありfailglobます。これにより、チームは実行できなくなります。nullglob一致しないワイルドカードを含む文字列を（長さが0の）空の文字列に変換しcp、エラーが発生するため、機能しません。

ただし、フォルダーに数千以上のファイルがある場合、ワイルドカードを使用するアプローチは完全に破棄する必要があります。事実はbash ワイルドカードを次のような非常に長いコマンドラインに展開します。

cp -a /souce/a /source/b /source/c …… /target

コマンドラインの長さに制限があります。これは、次のコマンドを使用して確認できます。

ですから、ワイルドカードなしでやりましょう。

ただ書こう

cp -a /source /target

そして、ここで私たちは振る舞いの曖昧さに直面していますcp。/ターゲットフォルダが存在しない場合は、必要なものを取得します。

ただし、ターゲットフォルダが存在する場合、ファイルは/ target / sourceフォルダにコピーされます。

/ targetフォルダーは、必要なファイルが含まれている可能性があるため、事前に削除できるとは限りません。たとえば、/ sourceのファイルを使用して/ targetにファイルを追加することが目的です。

たとえば、ソースフォルダーと宛先フォルダーの名前が同じである場合、/ sourceから/ home / sourceにコピーする場合は、次のコマンドを使用できます。

cp -a /source /home

コピー後、/ home / source内のファイルは、/ sourceからのファイルで補完されます。

これは論理的なタスクです。フォルダーの名前が同じであればファイルを宛先ディレクトリに追加できますが、フォルダーが異なる場合、ソースフォルダーはレシーバー内に配置されます。ワイルドカードなしでcpを使用して/ソースから/ターゲットにファイルをコピーする方法

この有害な制限を回避するために、明白ではない解決策を使用します。

cp -a /source/. /target

DOSとLinuxに精通している人はすでにすべてを理解しています。各フォルダー内には2つの見えないフォルダー "。"があります。と「..」は、現在のディレクトリと上位のディレクトリへの疑似フォルダリンクです。

  • コピーするとき、それcpは存在をチェックして、/ターゲット/を作成しようとします。
  • そのようなディレクトリが存在し、それが/ターゲットです
  • /ソースから/ターゲットに正しくコピーされたファイル。

だから、私たちは思い出の中で、または壁に太字のフレームでぶら下がっています：

cp -a /source/. /target

このコマンドの動作は明確です。100万個のファイルがあるかどうかに関係なく、すべてがエラーなしで機能します。

調査結果


1つのフォルダーから別のフォルダーにすべてのファイル をコピーする場合は、ワイルドカードを使用しません。代わりcpに、ソースフォルダーの最後にあるドットと組み合わせて使用することをお勧めします。これにより、非表示のファイルを含むすべてのファイルがコピーされ、数百万のファイルや完全なファイルの不存在で失敗することはありません。
