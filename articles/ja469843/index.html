「私たちを地球の表面から拭き取る若いパンクはどこにあるのですか？」

コミュニティの1つでの次のラウンドのディスカッションの後、最初のWebバックエンド開発者にSQLの知識が必要かどうか、またはORMがすべてを実行するかどうかについて、Grebenshchikovの文言に見出しを付けて実存的な質問をしました。私はORMとSQLについてだけではなく少し広い範囲で答えを検索することに決めました。そして、原則として、現在、中小規模の開発職で面接を受けようとしている人々、彼らの歴史、彼らが住んでいる世界を体系化しようとします。一般的に私は意見を述べましたが、それは採用における個人的な経験によって形成され、地元の市場に合わせて明確に調整されました。一般的に、それは好奇心をそそられました。これが私がなんとか見つけたものです。

グローバルデベロッパー人口


どういうわけかこの問題に取り組むために、私は、世界で現在何人の開発者が原理的に存在し、この人口が時間とともにどのように変化しているかに関するデータを検索することから始めることにしました。
さまざまな情報源の見積もりでは、フォークの1200万人から3000万人の数値を参照しています。私はSlashDataからのデータに立ち止まることにしました。なぜなら、それらの方法論は私には非常にバランスが取れており、私のニーズに適しているように思えたからです。評価では、Github上のアカウントとリポジトリの数、StackOverflow上のアカウントの数、npmアカウント、および米国とヨーロッパでの雇用に関する公式情報源を考慮に入れました。また、調査ごとに約2万人を対象とした独自の16件の調査で数値を調整しました。

SlashDataによると、2018年の第4四半期には世界中に約1,890万人の開発者がおり、そのうちの1290万人がプロであり、つまりプログラミングによって生計を立てていることがわかりました。現在プロの開発者ではない人とは、プログラミングが趣味の人と、現在職業を勉強している人（さまざまな学生や独学）です。ええと、それは私が興味を持っているグループの規模（600万人）のヒントです。正直なところ、これは思った以上です。

私にとって2つ目の驚きは、プログラマー数の増加率でした。2017年の第2四半期から2018年の第4四半期にかけて、14.7億から1890万に増加し、2018年には21％増加しました。プログラマ数の増加率を見積もると、年に5％程度の増加で、年に少しずつ増加します。そして、ここでそれは20％もの高さになります。

さらに、SlashDataは2030年までに人口が4500万人に達すると推定しています。これは年間20％ではなく、毎年8％強の増加を意味することを計算するのは簡単ですが、インターネット浸透を考慮した修正を参照しています（現在、Statistaによると、世界では約57％））および1人あたりの開発者数など、他のいくつかの要因。地理的には、インドと中国の開発者数が最も増加しており、2023年までにインドが開発者数で米国を追い抜くと予想されています（これはすでにC＃コーナーデータです）。

一般に、需要が高まっているため、多くのプログラマーがいます。需要といえば。

需要は何ですか？


需要を推定するために、2018年と2019年の HackerRankデータを使用しました。

プログラミング言語によって、JavaScript、Python、およびJavaに対する最大の需要は、コンピューターハードウェアを除いて、ほぼすべての業界にあります。後者では、理解できるC / C ++の最大の需要であり、ハードウェアプロジェクトでは、対応するソフトウェアのリソース強度と生産性に対する要件が依然として存在します。



フレームワークによると、AngularJS、Node.js、Reactは最大の需要にあり、それらに応じて最大の需給ギャップがあります。これは、たとえばExpressJSでは、供給がすでに需要を超えているため、JavaScriptエコシステムが変化する速度によって説明されているようです。



コンピテンシー別では、雇用主はまず候補者に問題解決スキルを期待します。雇用主の約95％がこれらのスキルを重要だと述べています。 56％で2位のプログラミング言語の熟練度。ちなみに、アルゴリズム、データ構造、およびその他のコンピュータサイエンスの基本的な知識を備えた行は、アンケートにないか、学術的な大規模な知識は必要ありません。

データベース設計には、100人までの企業の23.2％、および1000人を超える企業の18.8％が必要です。ええ、ここではORMとSQLについてです！論理的な私見の説明では、大企業では専用のDBAの役割があり、これがこの側面に関与しているため、開発者の要件を緩和し、より早く採用することが可能です。しかし、システムデザインでは、逆に、小さなものでは37.0％、大きなものでは44.1％です。大規模な建築家は献身的であるように思われるかもしれませんが、おそらく彼らは、生成されたシステムの数をカバーすることはできません。または、システム設計に非常に基本的なアルゴリズムとデータ構造に投資すると同時に、少し明確になります。

中小企業には、前述のシステム設計よりも多くのフレームワーク能力が必要であり、そこから、スタートアップができるだけ早く機能する製品を発売することが重要であり、明日は明日になるという船長の結論を引き出すことができます。



学生は何を学びますか？


ここでは、別のHackerRank調査のデータを利用しました。
何らかの形でプログラミングが大学で教えられているという事実（つまり、コンピュータサイエンスの専門分野）にもかかわらず、半数以上の回答者は、自己教育にも携わっていると回答しました。

現代の学生はYouTubeで学ぶことを好みますが、古い開発者はチュートリアルや本を好む傾向があります。どちらもStackOverflowを積極的に使用しています。これは、ビデオがジェネレーションZの通常のメディアチャネルであり、ジェネレーションYの代表がまだブロガーのいない時代を見つけたことに起因します。

雇用主が主張しているものを学ぶ：JavaScript、Java、Python。彼らはC / C ++を知っていることを示していますが、これはおそらくこれらの言語が大学での教育に使用されているためです。JSフレームワークは教えられていますが、需要は供給よりも大幅に高いため、明らかに最初の仕事を見つけるために積極的に教えられています。



一般に、教えられることが期待されているのは、需要があることです。

最初の仕事の学生は、主に専門的な成長を期待し、2番目に（最初のいくつかの国では）ワークライフバランス、3番目の興味深い仕事を期待しています。

プログラミング言語とソフトウェアのタイプによる開発者の人口動態




Webアプリケーションは最初に推定1690万人の開発者がいます。これもSlashDataデータです。次は、バックエンドサービス（1360万）、モバイルアプリケーション（1310万）、デスクトップ（1230万）です。 AR / VRおよびIoTセクターは徐々に人気を博し、AI / ML /データサイエンスは過去2年間で大幅に成長しました。

Javascriptは最も急速に成長しており、そのコミュニティはすでに最大であり、2018年だけで250万成長しました。彼らはそれをIoTやMLの分野でも書こうとしています。
2018年のPythonは、MLの人気が高まっているため、伝統的に強力であるだけでなく、開発の容易さと言語の利便性のために220万台増加しました。

Java、C / C ++およびC＃は、一般的な開発者の人口よりもゆっくりと成長しています。それらが、人々が最初から好むプログラミング言語であることはほとんどありません。ここでの開発者の需要は、供給と多かれ少なかれバランスが取れています。AndroidでなければJavaはさらに遅くなると思います。

PHPは2番目に人気のあるWebアプリケーションプログラミング言語であり、大幅に成長しています（2018年には32％増加しています）。そのコミュニティは590万人の開発者と推定されています。PHPの評判についての世論には反対ですが、学ぶことは簡単で、広く普及しています。

現代の若い候補者が過去の世代と比較してどのように勉強するか


HackerRankデータを再び。現在38歳から53歳の人々は、最初のプロジェクトのゲームを示しています。

ちなみに、私は最初の多かれ少なかれ作業しているプロジェクトが無制限のフィールドを持つ5つまでの「三目並べ」であり、2番目は15歳のゲームだったことを確認しました。BC010-01にこれをすべて書きました。ビリニュスBASIC（BASICとも呼ばれます） 86とフォーカル。ええ

現代の初心者プログラマー（21歳以前）は、最初のプロジェクトとして電卓とWebサイトを作成しています。

X世代の代表者のうち、ほぼ半数が16歳より前にコードを書き始め、その多くは一般的に5〜10歳（主に35〜45歳）です。それは多かれ少なかれ理解できる理由です。情報源がほとんどなく、プログラマーになるためには、これを本当に望んでいる必要があり、本当に早くプログラミングを始めたいと思っている人々が必要でした。そんなに欲しくなかった人は、今のところ別の職業にいる可能性が高いので、社会学の絵はそれだけです。



今日の若い候補者はわずか20％のケースで、16歳より前にプログラミングを開始します。ほとんどの候補者は16日から20日の間にいます。しかし、彼らが学ぶのははるかに簡単で、今ではよりアクセスしやすくなっています。

調査結果


今日、初心者のWebバックエンド開発者SQLが必要かどうかという質問に対する具体的な回答は見つかりませんでしたが、彼は現代のプログラマー集団に関する彼の考えを修正しました。

次世代の開発者-以前の住宅問題を幾分思い出させる普通の人々は、彼らを台無しにしただけでした。彼らは雇用主によって作成された需要を満たします。すぐに結果を出すことができる便利なツールとフレームワークにより、職業に入るしきい値が低くなりました。現在、より多くの人々がプログラマーになり、デジタル世代（Generation Z）は誕生以来テクノロジーに住んでいます。彼らにとって、それは普通の職業であり、他の人より劣っていません。

Latensi L1キャッシュは約4サイクルであり、キャッシュラインを不必要に撃たない方がよいことを知っている人は、総人口サイズのパーセンテージとして小さくなります。ただし、雇用について心配する必要はありません。結局のところ、これが必要な場合は、低レベルのことを書いてください。心配する必要はありません。システム設計に深い基礎知識があり、流血の実用的な戦いでそれらを習得し、貨物カルトに追従するだけではありません。チームでフレームワークを「コードを書く」だけで「使用する」ことができるようになると、フレームワークを「無意味に費やした年に耐え難いほど痛みを感じない」ようにするため（c）、そのような人々だけでバランスを取る必要があります。

望ましいというカテゴリのソフトスキルは、次第に必須に移行します（これを確認するために、私は客観的なデータはなく、実際的な観察のみです）。プログラマーの数は増え続けており、直接でも間接的にでも、結果を達成するためにすべてのプログラマーに指示する必要がありますが、これにはソフトスキルが必要です。

「ITに入る」というのは、プログラマーの収入が「ITではない」と同等の資格を持つ専門家の収入と大きく異なる場所に特徴的な、地方の歴史であるように思えます。私が住んでいるミンスクでは、これは一般的に大衆運動です。毎日、切望されているITにアクセスする方法に関する次のコースの広告を目にし、清掃会社はプログラマーをターゲットにして「この図のコードを理解していますか？だからアパートを掃除しないで済むのなら、私たちはあなたのために全力を尽くします。」同じことが、ある種のインドでも起こっているようです。これの証拠として、私はどちらも持っていません。

一般的に、プログラマーの人口は私の意見では脅かされていません。実際のプログラマーは昼夜を問わず満足しておらず、候補者は常に「何も知らない」と言っても意味がありません。彼らは「実際のプログラマー」よりも賢くて有能で、おそらくもっと賢くて有能です。彼らは、急成長中の市場が彼らから必要とする知識を取得し、彼らがまだ必要とせず、現時点で利益をもたらすことはない将来のために取っておきます。彼らはまだ学びたいので、彼らは必要なときに学びます。おそらく、誰もがこれを実行できるわけではありませんが、誰もがそれを必要とするわけではありません。近い将来、市場は、あるフレームワークで別のアプリケーションコンセプトをすばやく組み立てることができる人々を受け入れやすくします。
