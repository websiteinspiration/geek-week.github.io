Ahmad Shadidによる 「RTL Styling 101-CSSでのRTLのスタイル設定方法に関する広範なガイド」の翻訳。

世界中で2億9200万人以上の人々が母語としてアラビア語を話します。私は彼らに属しているので、テキストの書き込みの両方向をサポートする必要があるサイトを開発することがあります。左から右（LTR-左から右）と右から左（右から左）です。

RTLスタイリングの概要


CSSはデフォルトでLTR方向を使用します。html要素のブラウザースタイルを確認すると、dir（または "direction"）プロパティの場合、ltr値がデフォルトであることがわかります。以下は、LTRマークアップとRTLマークアップの違いを示すための基本的な例です。



RTLブロックに注意してください-LTRとは異なり、その中のテキストは右から左に読み取られます。この簡単な例では、ブラウザはそれを正しく表示しました。ドキュメント言語の方向を変更するには、ルート要素にdir属性を追加する必要があります。

<html dir="rtl">...</html>

dir値が変更されると、次にネストされたすべての要素が自動的に切り替わります：見出し、段落、リンク、画像、フォーム。

dir属性を「auto」に設定できることにも言及する価値があります。これにより、要素の内容の分析に基づいて要素の方向が自動的に変更されます。HTML仕様によると：
著者は、テキストの方向が本当に不明であり、サーバー側でこれを効果的に決定する方法がない場合にのみ、この値を最後の手段として使用することを強く勧めます。


HTML要素のdir = rtl属性の設定に加えて、CSSスタイルとしてdirection：rtlを追加することもできます。

.element { direction: rtl; }

ただし、CSSWG（CSSワーキンググループ）では、CSSが存在するかどうかに関係なく、マークアップが正しいことを確認するために、HTMLルート要素の属性として方向を正確に定義することをお勧めします。

マーキング方向変更例


LTRからRTLにマーキング方向を変更する方法をよりよく理解するために、より詳細な例を見てみましょう。



<article class="media">
  <img class="media__photo" src="blueberry-cheesecake.jpg" alt="">
  <div class="media__content">
    <h2>Blueberry Cheesecake</h2>
    <p>...</p>
    <p><a href="#" class="link">View Recipe</a></p>
  </div>
</article>

最初は、古き良きフロートを使用して、LTRマークアップの画像を左揃えにし、もちろん、clearfixを適用しました。

.media:after {
  content: "";
  display: block;
  clear: both;
}
 
.media img {
  float: left;
  width: 200px;
  margin-right: 16px;
}

次に、アラビア語のテキストを含む要素にdir =“ rtl”を追加します。次の結果になります：



画像を除くすべての要素が右側に配置されます。これは、imageプロパティがfloat：leftおよびmargin-right：16pxに設定されていたために発生しました。これを修正するには、次のスタイルをオーバーライドします

.media[dir="rtl"] img {
  float: right;
  margin-right: 0;
  margin-left: 16px;
}

@supports (text-decoration-color: red){
  .link-3 {
    box-shadow: none;
  }

CSSブールプロパティ


MDNによると：
論理プロパティと値CSSは、物理的な方向と寸法ではなく論理を使用してレイアウトを制御する機能を提供する論理プロパティと値を表すモジュールです。

簡単な例を見てみましょう。テキストの行を右に揃える必要があるとします。以下を追加します。
.page-header {
  text-align: right;
}

そしてRTLの場合：
[dir="rtl"] .nav-item {
  text-align: left;
}

ページの方向を考慮して、配置側を変更するtext-alignプロパティに値を1つだけ追加する方法があった場合はどうなりますか？CSSの論理プロパティが役に立ちます！
.page-header {
  text-align: end;
}


これを使用する場合、text-alignプロパティで指定された辺は、ページの方向に基づきます。デモ

これがどのように機能するかについての概要を理解したところで、CSSの論理プロパティを適用する追加の例と方法を見てみましょう。

論理パディング




右側に検索アイコンのある入力フィールドがあるとします。両側にパディングを設定する必要があります。右側のパディングは、テキストにアイコンが重ならないように少し大きくなっています。

.input--search {  
  padding-inline-start: 1rem;
  padding-inline-end: 2.5rem;
}

論理マージン



アイコンの右側のマージンは論理的である必要があるため、これにはmargin-inline-endを使用します。
.page-header__avatar {  
  margin-inline-end: 1rem;
}

論理フレームワーク（境界）



多くの場合、フレームはアクティブなナビゲーション項目を示すために使用されます。上記の例では、各要素の左側にフレームが設定されています。論理プロパティを使用してこれを実装する方法は？

.nav__item {  
  border-inline-start: 3px solid transparent;
}
 
.nav__item.is-active {
  border-color: #1e9ada;
}

CSSの命名規則


要素に添付されているCSSクラス名を指定しないでください。再利用可能なコンポーネントに抽出できる名前を使用してください。次の例を考えてみます。
<div class="c-section">
  <p><a href="#" class="see-link">See more</a></p>
</div>
 
<div class="c-section">
  <p><a href="#" class="see-link">Learn more</a></p>
</div>


両方のブロックのリンクは同じですが、それらの名前は異なります。2番目のブロックでは、see-linkクラスは意味がありません。良い名前はc-linkかもしれません。接頭辞c–はコンポーネントの代わりに使用されます;私はITCSSフレームワークからこの方法を採用しました。

自動化ツール


LTRおよびRTLマークアップを簡単に操作できる優れたツールがあります。

1. Bi-App-Sass


Anas NakawaのBi-App-Sassを使用すると、LRT用とRTLマークアップ用の2つの異なるスタイルシートにコンパイルするスタイルの1つのバージョンを作成できます。

このツールは、大規模なプロジェクトに役立ちます。その結果、書く方向ごとに多くのスタイルシートが存在する可能性があります。次の例を考えてみます。
.elem {
  display: flex;
  @include margin-left(10px);
  @include border-right(2px solid #000);
}


結果のCSSコードは次のようになります。

App-ltr.cssファイル：
.elem {
  display: flex;
  margin-left: 10px;
  border-right: 2px solid #000;
}


App-rtl.cssファイル：
.elem {
  display: flex;
  margin-right: 10px;
  border-left: 2px solid #000;
}


ただし、GitHub上のリポジトリへの最後のコミットは4年前（2015年11月）であることに注意してください。

2. RTLCSS


Mohamed JonasによるRTLCSSは、RTLでLTRスタイルシートを変換するためのフレームワークです。

このツールの違いは、既に組み立てられたバージョンのCSSファイルに適用されることです。たとえば、プロジェクトに50以上のSassコンポーネントがある場合、RTLCSSはコンパイルされたCSSファイルを解析し、そこからRTLバージョンを作成するのに役立ちます。
