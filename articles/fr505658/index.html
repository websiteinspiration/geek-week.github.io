Créer l'effet d'un vol rapide dans l'espace en 10 minutes sur p5.js



Récemment, sous l'inspiration de la chaîne The Coding Train , j'ai décidé de participer à l'un des défis de 10 minutes dans lequel j'ai dû créer l'illusion de voler à travers l'espace à grande vitesse.

Pour mettre en œuvre le projet, j'ai choisi le bien connu p5.js - une bibliothèque pour JavaScript conçue pour créer de l'art de manière algorithmique. Pourquoi était-il impossible d'utiliser des packages graphiques standard d'Adobe?

Tout d'abord, faire cela avec du code est magnifique.
Deuxièmement, le code permet à la volée, en ne modifiant que quelques paramètres, d'obtenir un résultat différent, la vitesse des expériences et des itérations augmente considérablement.
Et troisièmement, ce code est ensuite facilement intégré dans n'importe quel projet Web sous la forme d'un script dans JS.

Pour démarrer un projet, le plus simple est d'aller dans l'éditeur en ligne à editor.p5js.org.

La première chose à faire est de créer des étoiles. Il est clair que nous avons besoin d'un grand nombre d'étoiles, bien qu'il ne soit techniquement pas difficile de trouver de l'espace dans l'espace où vous ne pouvez pas voir du tout les étoiles à l'œil nu, mais notre idée de cet effet a été créée à l'aide de films de science-fiction. Je pense que vous pouvez négliger la précision pour des raisons d'efficacité. Donc. Nous devons faire un générateur d'étoiles aléatoire.

Tout d'abord, créez un tableau tout en haut qui stockera nos étoiles:

Créons une nouvelle fonction appelée Star (). Elle devrait avoir quatre paramètres clés: la position x, y et la variable z, par laquelle nous changerons tout le reste. Pour créer une occurrence aléatoire des points initiaux, nous pouvons utiliser la fonction random () et, en tant que paramètres, définir les limites de notre écran en largeur et en hauteur.

La fonction random () fonctionne avec 0, 1 ou 2 arguments. Sans arguments - renvoie un nombre aléatoire de 0 à 1, avec un - de 0 à l'argument, avec deux - du premier au second.


Puisque nous allons construire une «usine» pour la production d'étoiles, introduisons immédiatement le mot-clé this, qui accèdera et accèdera aux informations de l'objet à partir duquel nous appellerons cette fonction (ou plutôt, la méthode).

Les personnes expérimentées peuvent avoir une question - allons-nous faire de la vraie 3D? Mais en fait, il est beaucoup plus facile de faire de la pseudo-3d, comme dans les jeux à l'ancienne. La variable z est nécessaire ici à d'autres fins. Elle ne sera responsable que de l'illusion de la troisième dimension.

C'est fait, bien que les étoiles ne soient pas encore visibles, mais on peut déjà les créer. Il est temps de leur donner du mouvement. Pour ce faire, nous développons notre fonction Star () avec une nouvelle méthode (fonction imbriquée) update (), qui est responsable d'un cycle sans fin de changements.

Nous devons déterminer la vitesse de déplacement à travers les étoiles. Pour ce faire, créez une vitesse variable et rendez-la égale à 25. La logique est la suivante: nous modifions le paramètre z, ce qui affectera les paramètres x et y, de sorte qu'en changeant un paramètre nous changerons tout. Dès que ce paramètre atteint une valeur inférieure à l'unité, nous rétablissons les paramètres à des valeurs initiales aléatoires, commençant un nouveau cycle de mouvement des étoiles sur l'écran.

Eh bien, les étoiles sont créées et même en mouvement, il est temps de les voir. Pour ce faire, nous utiliserons la fonction show (), créant également un appel à l'objet à travers cela et peignant nos étoiles en blanc.

Maintenant, nous devons créer une forme pour notre étoile, afin d'avoir une variété de trajectoires et de tailles. Nous le ferons à travers la même variable z, qui est initialement égale à la largeur de l'écran, mais diminue de 25 à chaque image jusqu'à ce qu'elle devienne inférieure à 1, après quoi le cycle se répète.

Pour obtenir un beau mouvement d'étoiles du centre vers le bord, voyons comment la position doit changer, à savoir les variables x et y. Chaque étoile doit apparaître sur un rayon imaginaire et aller le long du bord de l'écran.

Imaginez que le point de départ est la coordonnée 200 200. Nous devons faire évoluer progressivement ces deux indicateurs vers le bas

Étant donné que z est toujours égal à la largeur de l'écran (par exemple, 600) et que x et y vont de -600 à 600, les variables x et y au début seront toujours inférieures ou égales à z. Cela suggère que nous pouvons traduire les valeurs en valeurs plus faciles à gérer de 0 à 1 en divisant simplement x ou y par z.

Ensuite, nous pouvons créer une fonction de mappage standard - pour assimiler des nombres de 0 à 1 proportionnellement à la largeur ou la hauteur de l'écran. Et ce sont précisément ces valeurs qui sont déjà passées en paramètres à notre étoile boule: ellipse ()

Imaginez que la largeur = 600 est notre cosmos. x = -300, y = -300. Dans ce cas, le point de départ sera -300/600 = -0,5, après 4 images, lorsque z diminue de 4 * 25 = 100, la coordonnée sera déjà -300 / 500 = -0,6, c'est-à-dire que les coordonnées seront LINÉAIRE à l'écart du centre de l'écran (coordonnées 0,0) vers le haut et vers la gauche, jusqu'à ce que z devienne inférieur à 1.

Dans le cas où ils sont positifs, tout se passera exactement de la même manière, seul le mouvement sera en bas et à droite.

Dans le cas où ils diffèrent, les options sont en bas et à gauche (avec y positif et x négatif) ou en haut et à droite (avec y négatif et x positif).

La fonction ellipse prend quatre paramètres: position le long de x, position le long de y, diamètre le long de x et diamètre le long de y, sur la base desquels l'ellipse est construite.

Eh bien, il est temps d'éclairer les étoiles et de créer de l'espace.

Sous les démarrages et la vitesse, nous créons une fonction standard qui crée l'environnement (cela fonctionne une fois) - setup () À l'intérieur, nous créons notre toile - createCanvas (600,600) et démarrons notre usine pour créer des étoiles via une boucle for, pour commencer nous nous limiterons à 800.

function setup() {
  createCanvas(600, 600);
  for (let i = 0; i < 800; i++) {
    stars[i] = new Star();
  }
}


Après cela, nous créons la deuxième fonction standard - draw (), sa différence est qu'elle fonctionne dans une boucle infinie. Nous peignons l'arrière-plan en passant par background () en noir, après quoi nous comptons la longueur du tableau des étoiles (800) à travers la boucle for et, en référence à chaque étoile du tableau, nous appelons la fonction (dans ce cas, la méthode) update () et show ()

function draw() {
  background("black");
  for (let i = 0; i < stars.length; i++) {
    stars[i].update();
    stars[i].show();
  }
}


Voilà, nous avons volé, mais il y a deux problèmes. La première - puisque le point de référence des coordonnées par défaut n'est pas au centre, mais dans le coin supérieur gauche, nous ne voyons que la partie des étoiles qui n'a pas de valeur de coordonnées initialement négative. Le reste est laissé hors de vue.



Pour résoudre ce problème est assez facile - il vous suffit de déplacer le centre via la commande de traduction (largeur / 2, hauteur / 2).



Le deuxième problème est que les étoiles ne changent pas de taille à l'approche de l'observateur. À un niveau subconscient, nous pensons que quelque chose ne va pas.

Afin de résoudre ce problème, revenons à notre ellipse, créons une autre variable r, qui dépendra également de z et déterminera les troisième et quatrième paramètres de l'ellipse, qui sont responsables des diamètres horizontal et vertical. Le code final est le suivant:

let stars = [];
let speed = 25;

function setup() {
  createCanvas(600, 600);
  for (let i = 0; i < 800; i++) {
    stars[i] = new Star();
  }
}

function draw() {
  background("black");
  translate(width / 2, height / 2);
  for (let i = 0; i < stars.length; i++) {
    stars[i].update();
    stars[i].show();
  }
}

function Star() {
  this.x = random(-width, width);
  this.y = random(-height, height);
  this.z = random(width);
 
  this.update = function() {
    this.z = this.z - speed;
    if (this.z < 1) {
      this.x = random(-width, width);
      this.y = random(-height, height);
      this.z = width;
    }
  };

  this.show = function() {
    fill("white");
    var sx = map(this.x / this.z, 0, 1, 0, width);
    var sy = map(this.y / this.z, 0, 1, 0, height);
    var r = map(this.z, 0, width, 10, 0);
    ellipse(sx, sy, r, r);
  };
}
