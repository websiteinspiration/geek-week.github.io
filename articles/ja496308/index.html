シームレスなノイズマップを作成する


Photoshopでシームレスな画像を作成するのは簡単です。画像をトリミングし、トリミングした右と下の部分を取り、フェードツールを使用して左と上に接着します。ただし、シームレスなノイズマップを適切に実装するには、慎重に検討する必要があります。パーリンノイズの基本的な知識が

あれば、補間された乱数で構成されていることがわかります。主に2次元で使用されます。ただし、1次元（移動時など）、3次元（3Dオブジェクトの円筒形および球形変換）、さらには4次元または5次元でも役立ちます。 4次元ノイズを使用して、シームレスな2D画像を作成できます。 4次元で考えることはあまり一般的ではないので、一度に1つの次元を取ります。



で、私の例では、私が使用し、二オクターブシンプレックスノイズを。シンプレックスノイズは大きな次元で高速であり、その三角形の性質により、見栄えがよくなります。

drawNoiseキャンバスを作成し、ループでピクセル配列を処理 する小さな関数を書きました。

一次元シームレスノイズ


1次元では、ノイズは無限の滑らかな線です（ノイズの実装は2で始まるため、2番目のパラメーターとして定数を使用します）。ここでは、これらが補間された乱数であることがわかります。

// one dimensional line
fNoiseScale = .02;
drawNoise(function(i,x,y){
    var v = Simplex.noise(
         123+x*fNoiseScale
        ,137 // we just need one dimension so this parameter is a constant
    );
    return v*iSize>y?255:0;
}).img();


1次元のノイズ

これをアニメーションで使用して、ミリ秒ごとにノイズ値を再計算できますが、ループを作成してすべての値を事前に計算することもできます。上の画像の値は、エッジの周りをループしていません。しかし、再現性を実装するのは非常に簡単です。そのためには、もう1つの次元とループ...または円が必要です。

1次元ループを作成するために円でノイズ。

コードでは、次のようになります。

// one dimensional loop
drawNoise(function(i,x,y){
    var fNX = x/iSize // we let the x-offset define the circle
        ,fRdx = fNX*2*Math.PI // a full circle is two pi radians
        ,a = fRdsSin*Math.sin(fRdx)
        ,b = fRdsSin*Math.cos(fRdx)
        ,v = Simplex.noise(
             123+a*fNoiseScale
            ,132+b*fNoiseScale
        )
    ;
    return v*iSize>y?255:0;
}).img().div(2);


おそらく、あなたは私たちがこれから何をするのかをすでに理解しています。2次元画像をループするには、3次元（少なくとも）ノイズマップが必要です。

円筒カード


Noise Perlinはもともと、継続的な3Dテクスチャリング（映画「トロン」）のために作成されました。イメージマップは、オブジェクトの周りに巻かれた1枚の紙ではなく、3次元のノイズフィールド内での位置によって計算されます。したがって、オブジェクトを切り取るときに、新しく作成されたサーフェスのマップを計算できます。

シームレスな画像という最終的な目標に到達する前に、まず左と右をシームレスに結合する画像を作成します。これは、1次元ループの2次元円に似ていますが、1つの追加の次元である円柱があります。

// three dimensional cylindrical map
drawNoise(function(i,x,y){
    var fNX = x/iSize
        ,fRdx = fNX*2*Math.PI
        ,a = fRdsSin*Math.sin(fRdx)
        ,b = fRdsSin*Math.cos(fRdx)
        ,v = Simplex.noise(
             123+a*fNoiseScale
            ,132+b*fNoiseScale
            ,312+y*fNoiseScale // similar to the one dimensional loop but we add a third dimension defined by the image y-offset
        )
    ;
    return v*255<<0;
}).img().div(2);

球状マップ画像


球を使用してシームレスなイメージを作成すると便利だと思うかもしれませんが、それは誤りです。

少し脱線して、球状のイメージマップがどのように計算され、どのように見えるかを示します。

// three dimensional spherical map
document.body.addChild('h2').innerText = 'three dimensional spherical map';
fNoiseScale = .1;
var oSpherical = drawNoise(function(i,x,y){
    var  fNX = (x+.5)/iSize // added half a pixel to get the center of the pixel instead of the top-left
        ,fNY = (y+.5)/iSize
        ,fRdx = fNX*2*Math.PI
        ,fRdy = fNY*Math.PI // the vertical offset of a 3D sphere spans only half a circle, so that is one Pi radians
        ,fYSin = Math.sin(fRdy+Math.PI) // a 3D sphere can be seen as a bunch of cicles stacked onto each other, the radius of each of these is defined by the vertical position (again one Pi radians)
        ,a = fRdsSin*Math.sin(fRdx)*fYSin
        ,b = fRdsSin*Math.cos(fRdx)*fYSin
        ,c = fRdsSin*Math.cos(fRdy)
        ,v = Simplex.noise(
             123+a*fNoiseScale
            ,132+b*fNoiseScale
            ,312+c*fNoiseScale
        )
    ;
    return v*255<<0;
}).img();

ノイズのある球

キュービックパノラママップ


作成した球体は、球体の中心にカメラを配置した場合、パノラマとして使用することもできます。ただし、顔の数がはるかに少ないため、立方パノラマを使用するのが最善の方法です。このスケッチに示すように、球は立方体の6つの側面に投影されます。


立方体に球体を重ねる立方体

の表面の各ピクセルについて、中心の視点Cと球体の間の交差を計算する必要があります。複雑に見えるかもしれませんが、実際には非常に単純です。

CAラインをベクトルと見なすことができます。また、ベクトルは、方向が変わらないように正規化できますが、長さは1に減少します。これにより、すべてのベクトルが一緒に球のように見えます。

正規化も非常に単純です。ベクトルの値をベクトルの全長でxyzで除算するだけです。ベクトルの長さは、ピタゴラスの定理を使用して計算できます。

以下のコードでは、最初に正規化計算が1つの面で実行されます。次に、6つのエッジすべてについてノイズが同時に計算されます。次の面の位置を取得するには、xyzに沿って値を反転する必要があるだけです。

// 3D panoramical cube map
document.body.addChild('h2').innerText = '3D panoramical cube map';
// we're not using the drawNoise function because our canvas is rectangular
var mCubemap = document.createElement('canvas')
    ,iW = 6*iSize;
mCubemap.width = iW;
mCubemap.height = iSize;
var  iHSize = iSize/2 // half the size of the cube
    ,oCtx = mCubemap.getContext('2d')
    ,oImgData = oCtx.getImageData(0,0,iW,iSize)
    ,aPixels = oImgData.data
    ,aa = 123
    ,bb = 231
    ,cc = 321
;
for (var i=0,l=iSize*iSize;i<l;i++) {
    var  x = i%iSize        // x position in image
        ,y = (i/iSize)<<0    // y position in image
        ,a = -iHSize + x+.5    // x position on the cube plane, the added .5 is to get the center of the pixel
        ,b = -iHSize + y+.5 // y position on the cube plane
        ,c = -iHSize        // z position of the cube plane
        ,fDistanceAB = Math.sqrt(a*a+b*b) // to calculate the vectors length we use Pythagoras twice
        ,fDistanceABC = Math.sqrt(fDistanceAB*fDistanceAB+c*c)
        ,fDrds = .5*fDistanceABC // adjust the distance a bit to get a better radius in the noise field
        ,v = 1
    ;
    a /= fDrds; // normalize the vector
    b /= fDrds; // normalize the vector
    c /= fDrds; // normalize the vector
    //
    // since we now know the spherical position for one plane we can derive the positions for the other five planes simply by switching the x, y and z values (the a, b and c variables)
    var aNoisePositions = [
         [a,b,c]    // back
        ,[-c,b,a]    // right
        ,[-a,b,-c]    // front
        ,[c,b,-a]    // left
        ,[a,c,-b]    // top
        ,[a,-c,b]    // bottom
    ];
    for (var j=0;j<6;j++) {
        v = Simplex.noise(
             aa + aNoisePositions[j][0]
            ,bb + aNoisePositions[j][1]
            ,cc + aNoisePositions[j][2]
        );
        var pos = 4*(y*iW+j*iSize+x); // the final position of the rgba pixel
        aPixels[pos] = aPixels[pos+1] = aPixels[pos+2] = v*255<<0;
        aPixels[pos+3] = 255;
    }
}
oCtx.putImageData(oImgData,0,0);
document.body.addChild('img',{src:mCubemap.toDataURL("image/jpeg")});

これは、1つの画像の6つの側面と、立方体から見たときの外観のスクリーンショットです。ソースコードにはthreejsで書かれた3Dの例があります。


キュービックパノラママップ


シームレスな2D画像


シームレスな2D画像を実装するのは簡単に思えるかもしれませんが、それを理解するには4次元で考える必要があるため、これは記事で説明した中で最も難しいように思えます。これに最も近いのは円筒形のマップ（水平方向の繰り返し）だったので、これをベースとして使用します。円筒マップでは、円の画像の水平位置を使用しました。つまり、画像の水平位置は、ノイズフィールドxyzで2つの座標xとyを与えます。画像の垂直位置は、ノイズフィールドのzに対応します。

画像をシームレスで垂直にしたいので、別の次元を追加する場合は、それを使用して2番目の円を作成し、zフィールドの線形値を置き換えることができます。これは、4次元フィールドで2つの円柱を作成するのと似ています。私はこれをスケッチで視覚化しようとしましたが、それは不正確ですが、4次元の円柱を描くのではなく、一般的な原理を伝えようとしました。


4次元

の2つの円柱のスケッチコードは非常に単純です。これらは4次元のノイズ空間にある2つの円です。

// four dimensional tile
fNoiseScale = .003;
drawNoise(function(i,x,y){
    var  fNX = x/iSize
        ,fNY = y/iSize
        ,fRdx = fNX*2*Math.PI
        ,fRdy = fNY*2*Math.PI
        ,a = fRds*Math.sin(fRdx)
        ,b = fRds*Math.cos(fRdx)
        ,c = fRds*Math.sin(fRdy)
        ,d = fRds*Math.cos(fRdy)
        ,v = Simplex.noise(
             123+a*fNoiseScale
            ,231+b*fNoiseScale
            ,312+c*fNoiseScale
            ,273+d*fNoiseScale
        )
    ;
    return (Math.min(Math.max(2*(v -.5)+.5,0),1)*255)<<0;
}).img().div(2,2);
