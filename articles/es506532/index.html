Hardware


Entonces, para crear un prototipo que funcione mínimamente, necesitamos:

  1. Un tablero o cualquier cosa a la que pueda unir todos los componentes
  2. Casi cualquier microcontrolador. Tomé Arduino Uno como la opción más fácil
  3. Módulo GPS (tomamos la fecha, hora y coordenadas del mismo). Teóricamente, puede tomar el módulo de reloj en tiempo real, pero luego deberá ingresar sus coordenadas manualmente y el dispositivo no funcionará de inmediato, pero el tiempo del arranque en frío se reducirá en gran medida
  4. Dos servos, o mejor aún, dos motores paso a paso.
  5. Puntero láser
  6. Una bagatela diferente: un soldador, adhesivo termofusible, una placa de pruebas, cables, un botón, un condensador y manos directas.

No hay trucos para armar: simplemente fije todos los componentes en la base con adhesivo termofusible.
Solo trate de asegurarse de que los ejes de los servos y el láser estén tan perpendiculares entre sí como sea posible, esto reducirá el error de puntería.

Conexión


Nada complicado aquí tampoco:

  • El GPS está conectado a través de uart (solo se necesita Rx, ya que no necesitamos enviar nada al módulo
  • servos - en los pines 10 (eje de acimut) y 11 (eje de altura)
  • botón - 2 pines
  • alimentación a todos los módulos
  • opcional - condensador de potencia

int Days(int d, int m, int y)// ,        (21.03)
{
  int days = 0;
  int yearNotLeap[12] = { 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31 };
  int yearIsLeap[12] = { 31, 29, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31 };

  if (((y % 4 == 0) && (y % 100 != 0)) || (y % 400 == 0))
  {
    for (int i = 0; i < m - 1; i++)
    {
      days += yearIsLeap[i];
    }
  }
  else
  {
    for (int i = 0; i < m - 1; i++)
    {
      days += yearNotLeap[i];
    }
  }
  if (m == 1) {

    return d - 81;
  }
  else
    return days + d - 81;
}
void GpsGetData()
{
  while (Serial.available() > 0)
    gps.encode(Serial.read());
  if (gps.location.isUpdated())
  {
    phi = gps.location.lat(); //   (double)
    lambda = gps.location.lng(); //    (double)
    yr = gps.date.year(); //  (2000+) (u16)
    mo = gps.date.month(); //  (1-12) (u8)
    d = gps.date.day(); //  (1-31) (u8)
    h = gps.time.hour(); //  (0-23) (u8)
    m = gps.time.minute(); //  (0-59) (u8)
    s = gps.time.second(); //  (0-59) (u8)
  }
}
void CalculateParams()
{
  float _time = (h + (float)m / 60 + (float)s / 3600) * 360.0 / 24.0;//   
  float sunlambda = (float)Days(d, mo, yr) * 360 / 365; //      
  sundelta = asin(0.398749 * sin(sunlambda * 2 * PI / 360));//    
  sunalpha = asin(tan(sundelta) / 0.434812); //      
  float tS = _time + lambda - 180 + sunalpha * 360 / (2 * PI) - alpha; //     
  height = 360 / 2 / PI * asin(sin(phi * 2 * PI / 360) * sin(delta * 2 * PI / 360) + cos(phi * 2 * PI / 360) * cos(delta * 2 * PI / 360) * cos(tS * 2 * PI / 360));// 
  az = 360 / 2 / PI * asin(sin(tS * 2 * PI / 360) * cos(delta * 2 * PI / 360) / cos(height * 2 * PI / 360)); //   (   )
  if (tS > 90 || tS < -90)
  {
    az = 180 - az;
  }
 // if (az > 180 && az < 270)
  //  az = az - 360;
}

void setup() {
  Serial.begin(9600);
  yaw.attach(10, 90);
  pitch.attach(11, 180);
  yaw.setSpeed(20);
  yaw.setAccel(0.1);
  pitch.setSpeed(20);
  pitch.setAccel(0.1);
  yaw.setAutoDetach(false);
  pitch.setAutoDetach(false);
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  but.tick();
  yaw.tick();
  pitch.tick();
  if (but.isSingle())
  {
    counter += 1;
    if (counter == 9)
      counter = 0;
  }
  alpha = Coordinates[counter][0];
  delta = Coordinates[counter][1];
  GpsGetData();
  CalculateParams();
  digitalWrite(LED_BUILTIN, gps.location.isValid());
  if (az > 90 && az < 180)
  {
    yaw.setTargetDeg(270 - az);
    pitch.setTargetDeg(height);
  }
  if (az < -90 && az > -180)
  {
    yaw.setTargetDeg(-90 - az);
    pitch.setTargetDeg(height);
  }
  if (az < 90 && az > -90)
  {
    yaw.setTargetDeg(90.0 - az);
    pitch.setTargetDeg(180 - height);
  }
}

... y solo hay un momento: la función CalculateParams ()
Qué debe hacer esta función: aceptar las coordenadas de la estrella en el sistema ecuatorial (ascensión recta y declinación), el tiempo y las coordenadas del observador, y dar la altura y el acimut del objeto, es decir esencialmente traduce las coordenadas de la estrella del sistema ecuatorial (en el que las estrellas están inmóviles) a horizontal (en el que las estrellas se mueven durante el día).

Esto se realiza utilizando las fórmulas de trigonometría esférica , así como la astronomía esférica en el vacío .

El algoritmo es como sigue:

  1. calcular la longitud eclíptica del sol
  2. calcular la declinación solar
  3. calcular la salida directa del sol
  4. calcular el ángulo horario de una estrella
  5. calcular altitud y acimut

Así es como se implementa:

void CalculateParams()
{
  float _time = (h + (float)m / 60 + (float)s / 3600) * 360.0 / 24.0;//

Estimación de error


Permítanos evaluar la contribución de cada factor al error (ordenado por contribución a la inexactitud)

  1. Curvatura de la mano: este es el factor más importante para mí
  2. Posición de horizonte imprecisa
  3. posición inexacta de azimut cero (recuerdo que los astrónomos cuentan el azimut desde el sur en sentido horario)
  4. El hecho de que los servos no pueden rotar en un ángulo fraccional
  5. imprecisiones en el algoritmo de traducción (la posición del sol no se determina con mucha precisión: por ejemplo, la longitud eclíptica del sol se considera mucho más complicada (una versión simplificada en el código); el día del equinoccio vernal no siempre ocurre al mismo tiempo;
