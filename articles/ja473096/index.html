「方向性非循環グラフ？私はこれについて聞いたことがありません。私のことをすべて知っているとは思わないでください！」と言うことはできますが、バージョン管理を可能にするのはこのグラフです。はい、Gitは非循環グラフです。この記事では、有向非巡回グラフ（DAG）の知識を共有し、独自のグラフの作成方法を示します。

DAGとは何ですか？


それはどういう意味ですか？DAGは、要素を子と見なすことができない一方向のグラフです。私たちの多くは、リンクリスト、ツリー、グラフに精通しています。3番目の実装では、DAGは1番目と2番目に非常に似ています。 DAGは


ツリーのように見えますが、完全ではありません

。DAGには、最小限の形で4つのコンポーネントがあります。

  1. ノード。データを保存します。
  2. 方向エッジ：一方向を指す矢印（このデータ構造が他とは異なるもの）。
  3. 親のない1つの「素晴らしい」祖先ノード。（楽しい事実：ほとんどの祖先ツリーは、実際にはツリーではなく、非循環グラフに向けられています。これは、「いとこ同士が結婚することもあるからです。」）
  4. 葉。または、子ノードのないノード。

書いて、コードを書いてみましょう。まず、2つのプロパティを持つコンストラクタを作成し、DAGという名前を付けます。

function DAG() {
 this.nodes = [];
 this.vertices = {};
}

ノードを追加するメソッドを作成します。ここで私がしたことを見てください。

DAG.prototype.add = function(node) {
  if (!node) { return; }
  if (this.vertices[node]) {
  return this.vertices[node];
  }
  const vertex = {
    node: node, 
    incoming: {}, 
    incomingNodes: [], 
    hasOutgoing: false, 
    value: null
  };
  this.vertices[node] = vertex;
  this.nodes.push(node);
  return vertex;
};

使い方？オブジェクトvertexはすべての魔法が発生する場所です。ノード、着信ノードを持つオブジェクト、すべての名前を含む配列、ノードが何かを指しているかどうかを示すブール型の変数、およびその値を追加します。これについては後で説明します。

次に、いくつかのエッジを追加して、ノードを接続します。これを行う前に、このノードにアクセスしたかどうかをチェックするヘルパー関数を作成する必要があります。彼女に電話しましょうvisit。

function visit(vertex, fn, visited, path) {
  const name = vertex.name,
  vertices = vertex.incoming,
  names = vertex.incomingNames,
  len = names.length,
  i;
  if (!visited) {
    visited = {};
  }
  if (!path) {
    path = [];
  }
  if (visited.hasOwnProperty(name)) {
    return;
  }
  path.push(name);
  visited[name] = true;
  for (i = 0; i < len; i++) {
    visit(vertices[names[i]], fn, visited, path);
  }
  fn(vertex, path);
  path.pop();
}
