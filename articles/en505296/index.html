Koyaaniskatsi - like a cry from the soul!


In contrast to the well-known Intel processors , whose brand name is quite easy to remember and pronounce, Koyaniskatsi was conceived as a processor whose name is more difficult to pronounce than to understand its architecture and learn how to program it at the most elementary level of machine code by dump editors to check Is it really worth bending under amateur radio-technical specifications and is it impossible to compose a beautiful command system, understandable at the level of intuition?

The style code “O'Key, prototsi, calculate me the factorial mean” is not meant, but a machine code was planned with a minimum level of getting into the basics of the art of programming it, which does not require hard cramming and memorizing of all teams.

Processor architecture


The entire processor is based on a register file of four functional groups of ten cells each. Although ALU operations are possible on any cells, the rules for correctly operating with them must be observed more or less strictly. Here you just need to remember the logic and purpose of these functional groups:
Group Appointment Note
A₀ ... ₉ Battery A₀ stores ALU status flags
B₀ ... ₉ Base B₀ stores the return base from the subroutine
C₀ ... ₉ Counter C₀ stores the return offset from the subroutine
D₀ ... ₉ Device You can organize a cache

Since the processor is quite simple and does not have built-in hardware mechanisms for organizing stack operations, the B₀: C₀ registers store the address where the operation of accessing the subprogram was placed. If necessary, the programmer himself must take care of all operations with the stack and describe them with an algorithm.

Addition of values ​​from the registers A₁ and B₂ can traditionally be represented by the expression "A1 + = B2" or the mnemonic notation "ADD A1, B2" , which should be executed in machine code. Since the processor architecture is extremely simple and organized in Polish, it is literally necessary to first predetermine the operands used in the operation, in order to then perform the particular calculation operation itself.

Since the combination of the registers “A1” and “B2” in the order “An, Bn” is used, then the machine code AB₁₆ should be specified for their definition. The choice of indices on specific operands is performed by codes A1₁₆ and B2₁₆, which indicates the literality of the codes. Since the index of the right operand is two, then in the code of the ALU operation “Sum” it should be indicated explicitly in the left notebook as 2A₁₆. Thus, it becomes apparent that the code of the whole expression is placed as “A1 AB B2 2A”. But since the index B₂ is indicated explicitly in the ALU operation, the byte of the B2 command can be excluded from the chain, since the B₂ register itself is not the receiver of the result and the code will be reduced to one of the two combinations of the record “A1 AB 2A” or “AB A1 2B ”, Since the result does not change on the location of prefixes before operations, and everything depends on the programmer’s style.

The expression "A1 = A1 + B2 - B3 & B4 | B5 ^ B6 ”can be described by the sequence“ A1 AB 2A 3B 4C 5D 6E ”, since each subsequent operation code also explicitly indicates the index of the right operand, which allows you to write code compactly and efficiently.
