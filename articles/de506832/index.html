Wie viele Prozessoranweisungen verwendet der Compiler?

Vor einem Monat habe ich versucht zu zählen, wie viele verschiedene Anweisungen von modernen Prozessoren unterstützt werden, und 945 in Ice Lake gezählt. Kommentatoren stellten eine interessante Frage : Welcher Teil dieser Vielfalt wird tatsächlich von Compilern genutzt? Zum Beispiel berechnete ein bestimmter Pepijn de Vos im Jahr 2016 , wie viele verschiedene Anweisungen in den Binärdateien in seinem / usr / bin enthalten sind, und zählte 411 - d. H. Etwa ein Drittel aller zu diesem Zeitpunkt vorhandenen x86_64-Anweisungen wurde in keinem der Standardprogramme des Betriebssystems verwendet. Sein anderer merkwürdiger Fund ist, dass der Code für x86_64 aus Anweisungen für ein Drittel besteht mov. (Im Allgemeinen ist bekannt, dass Anweisungen allein movausreichen, um ein Programm zu schreiben.)

Ich habe mich entschlossen, die De-Vos-Forschung mit dem LLVM / Clang-Compiler als „Referenzcode“ zu entwickeln. Es hat mehrere Vorteile gegenüber dem Inhalt von / usr / bin einer unbenannten Version eines unbenannten Betriebssystems:

  1. Es ist praktisch, damit zu arbeiten: Dies ist eine riesige Binärdatei, deren Größe mit allen Inhalten von / usr / bin eines durchschnittlichen Linux vergleichbar ist.
  2. Sie können verschiedene ISAs vergleichen: Offizielle Versionen für x86, ARM, SPARC, MIPS und PowerPC finden Sie unter release.llvm.org/download.html ;
  3. Damit können Sie historische Trends verfolgen: Seit 2003 sind für alle Releases offizielle Binärdateien verfügbar.
  4. Schließlich ist es bei der Untersuchung von Compilern logisch, den Compiler als experimentelles Objekt zu verwenden :-)
