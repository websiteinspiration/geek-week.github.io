テクニカルマーケット分析のためのOracleデータベースの使用

この記事では、Oracleデータベース、PL / SQL、SQL、MATCH_RECOGNIZE、MODEL句、集計関数、パイプライン関数について説明します。

市場のテクニカル分析（TA）が機能領域として使用されました。最初に、市場での取引の小さな表面的な導入、次に計算。

テクニカル分析-過去の価格変動のパターンに基づいて、起こりそうな価格変動を予測するための一連のツール。理論的には、テクニカル分析はあらゆる市場に適用できます。しかし、非常に流動性の高い自由市場、たとえば取引所で受け取られた最も広範囲なテクニカル分析。


多くのSQLステートメント、プロシージャー、およびグラフが開発されています。記事の下部にあるリンクのGitHubの一部-以下、完全なコード。

テクニカルマーケットインジケーター（TIR）- 基本価格チャートに含まれる値の再計算に基づいて形成された価格チャートの追加チャート。通常、これらはさまざまなタイプの平均化です（グラフの次のポイントは、移動平均など、以前の複数の価格値の平均値として計算されます）、関係（次のポイントは、複数の以前の価格を比較した結果-それらの差、期間にわたる変化の微分）、またはラグ（遅延）。インジケーターは、価格変動の統計に含まれるいくつかの非自明な情報を明確に示し、取引注文に関する推奨を形成することができます-購入/販売。インジケーターには少なくとも1つの変数パラメーターがあり、その値から結果が変化します。実際の取引を開始するには、通常、トレーダーの裁量で、いくつかのインジケーターと追加情報を組み合わせて使用​​します。

テクニカル分析の仮定の1つは「履歴が繰り返される」です。同様の状況にある市場参加者は、ほぼ同じように行動し、価格変動の同様のダイナミクスを形成します。そして、将来の市場行動は、主に過去に出現したパターンを繰り返すと想定するのは当然です。この声明に続いて、投資家は、TIRの可能なパラメーターから、過去の期間で最もよく証明されたパラメーターを選択できます。

この記事は、TIRパラメーターの値に各指標を使用するときのトレードパフォーマンスのグラフを作成します。これは2次元の表現です。しかし、実際には、注文の実行における遅延の結果への影響も評価するために、3次元表現を構築します。そのため、パラメーターはX軸（横座標）、遅延はY軸（縦座標）、結果はZ軸（適用）になります。これは、「スリップ」の影響を評価しようとする試みの1つです。 Y軸に沿った実際の「スリッページ」の代わりに、1〜5期間のBUY / SELL注文の実行を延期します。

このグラフでグローバル最大値が強調表示されており、グラフ自体が帽子のように見える場合（一般に、これは「正規分布」と呼ばれますが、垂直軸に関して厳密な対称性を想定しています）、円錐、ピラミッドも適しています-これは、特定のパラメーター値を選択できることを意味します、これは最高の結果を示し、このインジケーターを使用して取引を試みることができます。パラメータに応じてパフォーマンスチャートが「ストッケード」に似ている場合、最適なパラメータ値を選択することは不可能であり、このTIRを取引に使用しても意味がありません。

この記事では、指標の「ヒット率」のみを計算していると言えます。問題は、あなたがどれだけ「稼ぐ」ことができるかを評価するために、あえてしませんでした。

記載された検証が正常に完了することは、効果的な取引を行うために必要な要素の1つですが、十分ではありません。

移動平均


移動平均には少なくとも3つの主要なタイプがあります。

  • 線形（単純移動平均、SMA）、
  • 指数移動平均（EMA）および
  • 線形加重（加重移動平均、WMA）。

それらは構成部品の重量が異なります。線形移動平均の場合、重みは等しくなります。指数関数的および線形的に重み付けされた重みの場合、コンポーネントがウィンドウの右端から遠ざかるにつれて、指数関数的または線形的に減少します。

線形移動平均が最も簡単に計算できます。 Oracleでは、これは関数avg (VALUE) over (partition by STOCK_NAME order by ADATE rows between 9 preceding and current row)であり、10個の値の平均ウィンドウを持つ移動平均です。

線形移動平均には欠点があります。第一に、そのような平均は市場の逆転にゆっくりと反応します。多くの値が平均化され、それぞれに等しい重みが与えられているため、平均的な反応は、多くの場合、資産価格の反転後のいくつかの時間枠で発生します。

また、線形移動平均はシグナルに2回応答するため、あまり効果的ではありません。インジケーターがスライディングウィンドウに入り、それを離れたときです。残りはインジケーターの入力にのみ反応し、ウィンドウの右端から左に移動するとスムーズに計算から削除されます。TIRの

計算では、3種類すべての移動平均が使用されます。この記事では、指数関数的かつ線形的に重み付けされた分析関数で使用される集約関数とが開発されています。さらに、これらの移動平均は、再帰またはモデル（フレーズ）によって計算できます。 Oracleデータベースでは、分析のみで再帰やモデルカなしにEMAおよびWMAを計算することは、明らかに不可能です。 しかし、移動平均に関する予約はまだあります。EMAWMAMODEL

ほとんどすべてのインジケーターについて、いくつかの計算方法が提供されます：CALC-PL / SQLコードを使用した計算、SIMP-1つのオペレーターによる計算またはRECU-再帰による計算、AGRF-集約関数を使用した計算、MODE-モデルによる計算。

いくつかの方法を開発する理由があります。まず、いくつかの方法を使用してTIRを計算し、インジケーターを比較することで、インジケーターが同じ場合、計算が正しく実行されたことを確認できます（NULL値と "0"値を丸め、処理するさまざまな方法があります）。この記事では、さまざまな方法のサンプルのハッシュを比較するので、ビットまでのマッチングとすべてのアルゴリズムによる同じ処理が保証されます。

OracleはTIRの開発を開始すると思いますSIMP法で必要-一人のオペレーターによる計算。これが行われると、oracleoidの頭には計画と計算アルゴリズムがあり、PL / SQLまたは別の手続き型言語に簡単に転送できます。

ここでPL / SQLのCALC計算方法は、計算全体がカーソルに沿って1回のパスで実行できる場合、SIMP方法（1つの演算子を使用）よりも高速になることがわかります。しかし、計算のために一時テーブルまたはコレクションを作成する必要がある場合、またはカーソル上に複数のパスを作成する必要がある場合、「1人のオペレーター」による方法の方が速く、リソースをあまり消費しないことがわかります。

SIMPメソッド（「1つの演算子」）を含むすべてのメソッドで、演算子を関数に配置して、計算をパラメーター付きで呼び出して最適な値を選択できるようにします。

7つのTIRについて計算されます：移動平均の交差点（EMASIMPLE）、ゴールデンクロスとデッドリークロス（CROSSES）、バランスボリューム（OBV）、ケルトナーチャネル（KELTNER）、価格とボリュームトレンド（PVT）、アームの動きやすさのインジケーター（EMV）、商品チャネルインデックス（CCI）。

これら2つの記事の7つのインジケーターには、「平均ウィンドウサイズ」という1つのパラメーターと、2番目のパラメーター-シフト（ラグ）があります。シフトは、BUY / SELL注文の実行を延期するために必要なバーの数-終値をシフトするバーの数を示します（すべての注文はバーの終値で行われます）。これは「スリッページ」のようなものですが、実際には「スリッページ」ではありません。通常、スリッページはクライアントに有利ではなく、私たちの遅れはクライアントに有利ではなく、有利になる可能性があります。それでも、一部のインジケーターで1〜5バーのラグを使用したモデリングを使用すると、ずれが結果に大きな影響を与えることがわかります。また、一部の指標では、ラグとスリッページはそれほど重要ではありません。

SIMP（「単一ステートメント」）およびRECU（「再帰」）メソッドは、このフレーズを積極的に使用します。MATCH_RECOGNIZE、指定された範囲内の計算されたTIRの出入りに基づいてBUY / SELLトレーディングシグナルを生成するか、または移動平均に対するその動作。

指数移動平均の交点


指数移動平均を交差させる方法は、最も単純なTIRです。

この方法には、価格値がボトムアップ指数移動平均（ESA）を超える場合に買い（ロングポジションを開く）が含まれます。価格がESSを上から下に横切った場合の売り（ロングポジションのクローズ）。

この記事では、ショートポジションおよびマージン取引全般については考慮していません。

将来的には、他との比較の基準として使用できるのはこの指標です。このTIRとの比較は、バイアンドホールド戦略との比較よりも優れています。これは、この戦略が下落市場では収益性がないためです。

唯一のTIRパラメータは、移動平均の長さです。

PL / SQLを使用した計算

create or replace function HABR_MARKETINDEXES_EMASIMPLE_F_CALC (p_averaging_window_width integer)

return HABR_MARKETINDEXES_RESULT_LIST_T pipelined is

    l_result HABR_MARKETINDEXES_RESULT_LIST_T;

    EMA number;

    prev_EMA number;

    prev_TYPICAL_PRICE number;

    retval HABR_MARKETINDEXES_RESULT_T := HABR_MARKETINDEXES_RESULT_T (null, null, null, null, null, null, null, null, null);
    prev_STOCK_NAME varchar2(256);
    l_alpha number;
begin

    l_alpha := 2 / (p_averaging_window_width + 1);

    for c1 in (select STOCK_NAME, ADATE, TYPICAL_PRICE, ACLOSE from LOAD_YAHOO_V order by 1, 2)
    loop
   
        retval.ADATE        := c1.ADATE;
        retval.ACLOSE       := c1.ACLOSE;

        if prev_STOCK_NAME is null or prev_STOCK_NAME <> c1.STOCK_NAME
        then
            retval.STOCK_NAME   := c1.STOCK_NAME;
            EMA                 := c1.TYPICAL_PRICE;
            prev_EMA            := null;
        else
            EMA := round (c1.TYPICAL_PRICE * l_alpha + EMA * (1 - l_alpha), 20);
        end if;
       
        if    prev_TYPICAL_PRICE < prev_EMA and c1.TYPICAL_PRICE > EMA then retval.AACTION := 'BUY';
        elsif prev_TYPICAL_PRICE > prev_EMA and c1.TYPICAL_PRICE < EMA then retval.AACTION := 'SELL';
        else  retval.AACTION := null;
        end if;

        retval.IND_VALUE  := EMA;
       
        pipe row (retval);
       
        prev_STOCK_NAME := c1.STOCK_NAME;
        prev_EMA := EMA;
        prev_TYPICAL_PRICE := c1.TYPICAL_PRICE;
       
    end loop;
end;


1つの再帰演算子による計算

create or replace function HABR_MARKETINDEXES_EMASIMPLE_F_RECU (p_averaging_window_width integer)

return HABR_MARKETINDEXES_RESULT_LIST_T is

    l_result HABR_MARKETINDEXES_RESULT_LIST_T;

begin

    with
      T1 (STOCK_NAME, ADATE, TYPICAL_PRICE, EMA, ACLOSE, RN) as
           (select STOCK_NAME, ADATE, TYPICAL_PRICE, round (TYPICAL_PRICE, 20), ACLOSE, RN from LOAD_YAHOO_V where RN = 1
            union all
            select b.STOCK_NAME
                 , b.ADATE
                 , b.TYPICAL_PRICE
                 , round (b.TYPICAL_PRICE * 2 / (p_averaging_window_width + 1) + a.EMA * (1 - 2 / (p_averaging_window_width + 1)), 20)
                 , b.ACLOSE
                 , b.RN
            from T1 a, LOAD_YAHOO_V b
            where b.RN = a.RN + 1 and b.STOCK_NAME = a.STOCK_NAME)
    select HABR_MARKETINDEXES_RESULT_T (STOCK_NAME, ADATE, ACLOSE, EMA, null, null, null, null, AACTION)
    bulk collect into l_result
    from T1 match_recognize (partition by STOCK_NAME order by ADATE
                             measures classifier() as AACTION
                             all rows per match with unmatched rows
                             pattern (BUY+ | SELL+)
                             define BUY  as (prev (TYPICAL_PRICE) < prev (EMA) and TYPICAL_PRICE > EMA)
                                  , SELL as (prev (TYPICAL_PRICE) > prev (EMA) and TYPICAL_PRICE < EMA)
                             ) MR;

    return l_result;
end;


フレーズMODELを使用した計算
機能テキストHABR_MARKETINDEXES_EMASIMPLE_F_MODE
create or replace function HABR_MARKETINDEXES_EMASIMPLE_F_MODE (p_averaging_window_width integer)
return HABR_MARKETINDEXES_RESULT_LIST_T is
    l_result HABR_MARKETINDEXES_RESULT_LIST_T;
begin

    with
      T1 as (select * from LOAD_YAHOO_V
             model dimension by (STOCK_NAME, RN) measures (ADATE, TYPICAL_PRICE, ACLOSE, to_number(null) as EMA)
             rules (EMA[any, any] = round (TYPICAL_PRICE [cv(), cv()] * 2 / (p_averaging_window_width     + 1) + nvl(EMA [cv(), cv() - 1], TYPICAL_PRICE [cv(), cv()]) * (1 - 2 / (p_averaging_window_width     + 1)), 20)))
    , T2 as (select STOCK_NAME, ADATE, ACLOSE
                  , TYPICAL_PRICE, LAG (TYPICAL_PRICE) over (partition by STOCK_NAME order by ADATE) as PREV_TYPICAL_PRICE
                  , EMA, lag (EMA) over (partition by STOCK_NAME order by ADATE) as PREV_EMA
             from T1)
    select HABR_MARKETINDEXES_RESULT_T (STOCK_NAME, ADATE, ACLOSE, EMA, null, null, null, null
                                      , case when prev_TYPICAL_PRICE < prev_EMA and TYPICAL_PRICE > EMA then 'BUY'
                                             when prev_TYPICAL_PRICE > prev_EMA and TYPICAL_PRICE < EMA then 'SELL' end)
    bulk collect into l_result
    from T2 order by STOCK_NAME, ADATE;
    
    return l_result;

end;

create or replace function HABR_MARKETINDEXES_EMASIMPLE_F_AGRF (p_averaging_window_width integer)
return HABR_MARKETINDEXES_RESULT_LIST_T is
    l_result HABR_MARKETINDEXES_RESULT_LIST_T;
begin

    with
      T1 as (select STOCK_NAME, ADATE, TYPICAL_PRICE, ACLOSE
                  , round (EMA (EMA_DATA_T (TYPICAL_PRICE, p_averaging_window_width)) over (partition by STOCK_NAME order by ADATE), 20) as EMA
             from LOAD_YAHOO_V)
    select HABR_MARKETINDEXES_RESULT_T (STOCK_NAME, ADATE, ACLOSE, EMA, null, null, null, null, AACTION)
    bulk collect into l_result
    from T1 match_recognize (partition by STOCK_NAME order by ADATE
                             measures classifier() as AACTION
                             all rows per match with unmatched rows
                             pattern (BUY+ | SELL+)
                             define BUY  as (prev (TYPICAL_PRICE) < prev (EMA) and TYPICAL_PRICE > EMA)
                                  , SELL as (prev (TYPICAL_PRICE) > prev (EMA) and TYPICAL_PRICE < EMA)
                             ) MR;

    return l_result;
end;
