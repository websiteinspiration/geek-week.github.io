初心者向けAndroid開発者向けのReactネイティブチュートリアル（サンプルアプリケーション付き）

hackernoon.comで公開されたNikhil Sachdevaによる記事の翻訳を提示します。著者は、React Nativeを使用してモバイルアプリケーションを開発した経験を共有し、このフレームワークを使用して独自のアプリケーションを作成することを提案しています。



私はAndroid開発者であり、かなり長い間Javaを使用していました。最近、React Nativeを使用してモバイルアプリケーションを作成しようとしました。これにより、控えめに言っても、開発プロセスを再検討しました。私の記事の目的は、アプリケーション開発でこれら2つのフレームワークを使用したときに気付いた違いを示すことです。

React Nativeとは何ですか？


公式ウェブサイトからの情報：
「React Nativeを使用すると、JavaScriptのみを使用して、Reactと同じ構造のモバイルアプリケーションを作成できます。これにより、宣言的なコンポーネントを使用して多機能のモバイルUIを作成することができます。

これらは、SwiftまたはJava / Kotlinで作成されたネイティブiOSまたはAndroidアプリと同じアプリです。
「React Nativeで作成するアプリケーションはモバイルWebアプリケーションではありません。ReactNativeは通常のiOSおよびAndroidアプリケーションと同じコンポーネントを使用するためです。Swift、Kotlin、またはJavaを使用する代わりに、JavaScriptとReactを使用してこれらのコンポーネントを構築します。

つまり、React NativeはReact.jsに基づいたフレームワークであり、これによりAndroidとiOSの両方のクロスプラットフォームアプリケーションを開発できるようになります。

なぜおなじみのJavaから離れて、JavaScriptとReact.jsを学ぶのでしょうか。これらの言語を使用するいくつかの利点を次に示します。

長所：あなたが勝つもの


1.クロスプラットフォーム開発


開発者の主な目標は、顧客にサービスを提供することです。開発者が他のプラットフォーム用のアプリケーションを作成できないので、ユーザーが1つのプラットフォームだけに制限されることを望まないでしょう。したがって、開発者自身が特定の開発ツールを使いこなせるという理由だけで、自分の能力を制限するべきではありません。

React Nativeフレームワークは移植可能です。つまり、JavaScriptで記述された単一のコードベースは、AndroidとiOSの両方のモジュールを作成します。

2. Reactをマスターする


React NativeとJavaScriptを習得することで、たとえばWebサイトのフロントエンド開発の新しい世界を発見できます。React NativeフレームワークはReactと同じコンポーネントに基づいているため、ここで得られるスキルはモバイルアプリケーションの開発に限定されません。

3.ビルド時間がAndroid Studioよりも速い


基本的な機能をテスト/修正するためにビルドに2〜3分以上費やし、同時にバグ修正が何時間も行われたことがありますか？ソリューションはReact Nativeになります。これにより、組み立てにかかる時間が大幅に短縮されます。ホットリロードなどの機能により、ユーザーインターフェイスの設計とテストは簡単です。この機能により、JSファイルが保存されるたびにアプリケーションが再起動します。

4. JavaScriptは、ネットワーク経由でデータを送信するのに便利です。


React Nativeでは、APIの呼び出し、URLによる画像のレンダリング、その他のプロセスは非常に簡単です。Retrofit、OkHttp、Picassoなどを使用する必要がなくなりました。セットアップにかかる時間が大幅に短縮されます。データがAndroidプラットフォームのAPIから取得される場合、最初にPOJOモデルに変換され、UI要素でのみ使用されます。しかし、React Nativeで受信したJSONデータはJavaScriptに便利であり、UIをプレビューするために直接使用できます。これにより、REST APIからのGETまたはPOSTリクエスト用のウェブインターフェースを簡単に取得できます。

5. UI開発


React Nativeでは、Android XMLマークアップと競合するflexboxモジュールがUI マークアップとして機能します。FlexboxはWeb開発者のコ​​ミュニティで非常に人気があります。React Nativeでは、UI要素は主にゼロから開発する必要がありますが、Androidのネイティブ開発では、Google Design Support Libraryがすでに接続されています。これにより、インタラクティブでレスポンシブなデザインを開発者に自由に与えることができます。

短所：何を失う可能性が高いですか


1.おそらくJavaScriptが嫌い


多くの人々は、この言語がJavaやC ++などの従来の言語のように見えないという理由だけでJavaScriptを好きではありません。詳細な否定的なレビューはこことここにあります。

2.多くのサードパーティライブラリ


React Nativeコミュニティはまだ初期段階であり、ネイティブのAndroidライブラリほど人気のないサードパーティのライブラリをサポートしています（ちなみに、私のAndroidライブラリのスライドショーを評価してください）。

、今のプロジェクトを開きます。テキストエディタで次のようにApp.jsを調整します。

export default class App extends Component<Props> {
constructor(props){
super(props);
this.state ={ isLoading: true}
}
componentDidMount(){
return fetch('https://jsonplaceholder.typicode.com/photos')
.then((response) => response.json())
.then((responseJson) => {
this.setState({
isLoading: false,
dataSource: responseJson,
}, function(){
});
})
.catch((error) =>{
console.error(error);
});
}
render() {
    if(this.state.isLoading){
return(
<View style={{flex: 1, padding: 20}}>
<ActivityIndicator/>
</View>
)}
return(
<View style={{flex: 1, paddingTop:20}}>
<Text style={styles.header}>Mock App</Text>
<FlatList
data={this.state.dataSource}
renderItem={({item}) => <View style={styles.card}><Text style={styles.id}> {item.id}</Text><Text style={styles.title}>{item.title}</Text><Image source ={{uri : item.url }}  style={styles.image}/></View>}
keyExtractor={({id}, index) => id}
/>
</View>
);
}

このコードには多くの情報があります。コード内のすべての主要なキーワードの概要を説明しますが、理解できない各キーワードの検索を続けます。しばらく時間がかかります。

1. componentDidMount。これは、アプリケーションのライフサイクルにおけるReact Nativeの一部です。componentDidMount（）は、コンポーネントがコンポーネントツリーに追加された直後に起動されます。

2.フェッチ。ネットワークリクエストを処理するために、React NativeにはFetch APIがあります。

3.状態変数（isLoading、dataSource）。 isLoadingは、APIデータがロードされているかどうかを示すブール型変数です。 dataSourceは、フェッチコマンドからのJSON応答を格納する変数です。

4. FlatList。これは、React NativeのRecyclerViewに相当しますが、はるかに単純です。 FlatListコンポーネントには、共通の構造がありますが、変更可能なデータのスクロールシートが表示されます。FlatListは、項目の数が時間の経過とともに変化する可能性がある長いリストを操作するのに最適です。

ご覧のとおり、データリストでは、要素ごとに変数を区切る必要はありません。これが動的言語、つまりJavaScript言語の美点です。変数のタイプはリアルタイムで決定されるため、大量のデータを操作することも簡単な作業です。

ご覧のとおり、各コンポーネントのスタイルはすでに定義されています。変数の値を設定する必要があります。これは次の方法で実行できます。
