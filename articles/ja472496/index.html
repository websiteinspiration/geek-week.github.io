CSSグリッドでのスクラップブックレイアウトの作成

最近、就学前のトレーニングレッスンで、息子に1週間テディベアの世話をするように指示されました。これは、クマを冒険に連れて行く必要があることを示唆し、彼の思い出をアルバムに追加しました。私はこのアルバムを本当に楽しんでいて、CSSグリッドを使用してこのようなことをどのように行うのかを考えるように勧めました！




複合メッシュ


Andy ClarkeがState of the Browserカンファレンスで素晴らしいプレゼンテーション「Inspired by CSS Grid Technology」を行いました。これは、私のようなデザインのルーツを持つ開発者にとって本当に洞察に富んだものでした。彼の講演では、印刷デザインのアイデアをWebで使用して素晴らしいレイアウトを作成する方法と、CSSグリッドがこれを可能にするだけでなく、これまでよりもはるかに簡単にする方法について話しました。これらの原則の1つは、複合メッシュの使用でした。

私たちのほとんどは、おそらくWebの設計と開発にグリッドを使用することに慣れています。私が開発を担当したほとんどすべてのサイトレイアウトは、同じ幅の列を持つ標準の12列（場合によっては24列）のグリッドに分割されました。これまでのところ、すべてがかなり予測可能です。

一方、複合メッシュは、2つ以上のメッシュを互いに重ね合わせることによって作成されます。たとえば、4カラムのグリッドに重ねられた5カラムのグリッドを比較すると、リズミカルなパターンが作成され、通常のグリッドよりも動的なレイアウトを構築する可能性が広がります。


図2-4列と5列のグリッドから始めます図3- グリッドは互いに重なり合っています


。その結果、複合グリッドが得られます

これは心理的および技術的な観点の両方から当てはまります。複合グリッドを使用しているにもかかわらず、最も普通のレイアウトを作成できます。Andyは、複合グリッドについて詳しく説明する詳細な記事「Inspired Design Decisions：Pressing Matters」を執筆しました。State of the Browserカンファレンスからの彼のプレゼンテーションも利用できます。

複合メッシュジェネレータ


frユニットを使用すると、CSSグリッドに複合グリッドを簡単に実装できます。私はウェブデザインで複合グリッドを使用するアイデアが好きですが、それらを計算するプロセス（特に、より複雑なグリッド）は非常に時間がかかると感じました。複合メッシュをすばやく簡単に作成できるようにしたかったので、Andyのレポートに触発されて、袖をまくり、それらを作成して視覚化するための小さなツールを作成しました。 2つのグリッドの列数を入力し（それぞれ最大10列）、ジェネレーターがそれらを組み合わせて、プロパティに設定できる最終的な値を与えますgrid-template-columns。たとえば、4列のグリッドに5列のグリッドを追加すると、値が得られます4fr 1fr 3fr 2fr 2fr 3fr 1fr 4fr。



このツールはCodepenで提供されています。なので、自由に使ったり、ニーズに合わせたりしてください。

アルバムレイアウトのグリッドを作成する


コンポジットグリッドは、アルバムのレイアウトに最適です。アルバムのレイアウトを少し予測しにくくしますが、リズムとバランスの感覚は維持します。ジェネレーターでの少数の実験の後、私は複合グリッド6/5に落ち着きました、それは私には思えたように、さらなる操作のために正しい数の列を与えました。これにより、元のグリッドで作業できます：

.grid {
	display: grid;
	grid-template-columns: 5fr 1fr 4fr 2fr 3fr 3fr 2fr 4fr 1fr 5fr;
	gap: 1rem;
}

グリッド行の定義


グリッド線の定義はより困難であり、もう少し試行錯誤が必要でした。グリッド内の各写真は互いに重なり合う必要があります。何本の線が必要になるかを理解するために、紙の上にグリッドを描くのが便利であることがわかりました。

縦のリズム感を保つために、写真はある程度重ねるようにしました。このオーバーレイサイズを変数に割り当てて、ページ全体で使用できるようにし、必要に応じて更新できるようにしました（図4）。


図4-垂直画像オーバーレイ

.grid {
	--verticalPadding: 2rem;
	--overlap: 6rem;
}

各画像には、付随するテキストもあります。このため、前の写真と重ならないように、上下に十分なスペースが必要です。これには、キャプションの上下にグリッド線を追加することが含まれます。これは「パディング」として機能します。これで、各画像は少なくともグリッドの4行をカバーするはずです。また、上下に重なる画像は5行を占めるはずです。


図5-「パディング」の役割を果たす行により、テキストブロックの終わりと次の画像の始まりの間の最小間隔を維持できます。

しかし、グリッドの設計はまだ完了していません。画像の縦横比を固定することにしました。縦向きの写真もあれば、横向きの写真もあります。写真のアスペクト比やテキストの長さに関係なくグリッドレイアウトが機能するようにしたいので、グリッドラインが順応できることが必要でした。

オーバーレイまたはパディングとして機能する文字列に固定値を使用する代わりに、minmax（）関数を使用してこれらのトラックを柔軟にすることができます。これにより、ライントラックのサイズは最小になりますが、コンテンツに対して必要に応じて拡大されます。

minmax(var(--padding, auto));

要素の配置


グリッドのスケルトンができたので、要素の配置を処理します。それらをあらゆる構造のグリッドに配置するための最良の方法を理解して選択することは、難しい場合があります。行番号、スパンキーワード、名前付きの行または領域など、さまざまなオプションがあります。それらの一部は、特定の状況で他よりもうまく機能します。しかし、正しいまたは間違った選択肢はなく、多くの場合、それはあなたに最も適した方法を見つけることになります。

すべてが正常に機能している限り、間違ったアプローチはありません

グリッド線を使用した配置


多くの場合、開始値と終了値（通常は最初と最後の行の数）を使用して要素を配置することから始めますが、要素がカバーするトラックの正確な数がわかっている場合は、代わりに「スパン」キーワードを使用します。グリッド線に名前を付けて重要なランドマークを追加することもありますが（たとえば、wrapper-startおよびwrapper-end）、グリッド線に名前を付けたり、グリッド内の各要素に直接グリッド領域を作成したりすることはほとんどありません。グリッドの最後に要素を配置したい状況で、マイナスのグリッド線を表示することは、非常に役立ちます。これについては別の記事で書いた。ほとんどの場合（使用するグリッドの場合）、列の数は既知であり、固定されているため、列軸で負のグリッド線を最も頻繁に使用します。値が1 / -1の

プロパティgrid-columnを介して配置された要素は、グリッドの最初から最後までのすべての列をカバーします。

.item {
	grid-column: 1 / -1;
}

グリッドに非常に多数のトラックがある状況では、グリッド線に名前を付ける傾向があります。検討中のケースでは10列のトラックしかないため、行番号で要素を配置すると、さらに操作しやすくなります。

正/負のグリッド線と「スパン」値の混合を使用して、要素を列の軸に沿って配置するだけです。Firefoxの開発者パネルでグリッドインスペクターをオンにすると、行番号を確認できるため、非常に役立ちます。

グリッド領域を使用した配置


グリッドを見れば。行数がかなり多いことがわかります。


図7は、開発者パネルのFirefoxグリッドインスペクターのスクリーンショットであり、グリッドの列と行を示しています。

行軸に行番号で要素を配置し始めましたが、すぐに管理が難しくなります。要素は重複している必要があり、1つの要素が終了し、もう1つの要素が開始するトラックを追跡するのは難しいようです。また、将来的にレイアウトを補完したい可能性があるため、マイナスのグリッド線は使いたくありません。最終的にグリッドに明示的な行を追加すると、負の行番号が無効になり、多くのレイアウトエラーが発生する可能性があります。

これは、行軸に名前付きグリッド領域を作成することを決めたときです。グリッド領域は次の2つの方法で作成されます。

  1. grid-template-areasグリッドレイアウトをASCII描画として効果的に「描画」できるプロパティを使用する
  2. 名前付きグリッド線の使用-start、および-end線名の接尾辞としての使用

このプロパティでgrid-template-areasは、重なり合う要素の領域を定義できないため、この特定のレイアウトでは実際には役立ちません。ただし、名前付きグリッド領域を使用すると、間違いなくタスクが簡単になります。

行軸と列軸の両方の線に名前を付けると、グリッド領域が得られます（図8）。


図8-行名のサフィックスco -startと-endグリッド領域を作成します。

プロパティを使用して要素を配置するときに、この領域を参照できますgrid-area：

.item {
	grid-area: image;
}

これにより、grid-columnプロパティとgrid-rowプロパティを使用して行名をリストするよりもコードが簡潔で読みやすくなります。

.item {
	grid-row: image-start / image-end;
	grid-column: image-start / image-end;
}

ただし、この場合は、行軸に名前付きグリッド領域のみが必要です。grid-rowプロパティを使用して参照できるため、これは正常です。

.item {
	grid-row: image;
}

行数が多いgrid-template-rowsので、ページの構造を反映するようにプロパティを垂直方向に記述する方がはるかに簡単であるように思えました。

grid-template-rows:

グリッド構造を視覚化できるため、適切な場所にライン名を追加することがより簡単になります。

grid-template-rows:

	auto
	[fig1-start]
	3rem
	[header-end]
	minmax(var(--verticalPadding), auto)
	[p1-start]
	minmax(0, auto)
	[p1-end]
	minmax(var(--verticalPadding), auto)
	[fig2-start]
	var(--overlap)
	[fig1-end]
	minmax(var(--verticalPadding), auto)
	[p2-start]
	minmax(0, auto)
	[p2-end]
	minmax(var(--verticalPadding), auto)
	[fig3-start]
	var(--overlap)
	[fig2-end]
	minmax(var(--verticalPadding), auto)
	[p3-start]
	minmax(0, auto)
	[p3-end]
	minmax(var(--verticalPadding), auto)

あとは、グリッド要素を配置したときに行軸上の領域の名前を参照するだけです。

.fig--1 {
	grid-column: span 5 / -1;
	grid-row: fig1;
}

.fig--2 {
	grid-column: 1 / span 7;
	grid-row: fig2;
}

.fig--3 {
	grid-column: span 5 / -2;
	grid-row: fig3;
