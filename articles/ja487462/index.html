ステップ1.フォームを使用してデータを編集する


このステップでは、テーブルとフォームが描画されている「ダッシュボード」タブを操作します。テーブルコードはtable.jsファイルにあり、フォームコードはform.jsにあります。

記事「フォームの操作」では、フォームの検証によって補足されたadd（）メソッドを使用して新しいエントリが追加されました。これで、フォームはエントリの編集に使用されます。

ウィジェットテーブルonAfterSelectイベントを追加します。
このイベントがトリガーされると、テーブルからデータを受信して​​フォームフィールドに転送します。

const table = {
    view:"datatable", 
    id:"film_list",
    scroll:"y",
    select:true,
    url:"data/data.js",
    hover:"myhover",
    columns:[
        { id:"rank", header:"", width:50, css:"rank"},
        { id:"title", header:"Film title", fillspace:true},
        { id:"year",  header:"Released", width:100},
        { id:"votes", header:"Votes", width:100},
        { id:"rating", header:"Rating", width:100}
    ],
    on:{
        onAfterSelect(id){
            let values = $$("film_list").getItem(id);
            $$("film_form").setValues(values)
        }
    }
}

上記のコード のsetValues（）メソッドは、受け取った値をフォームに渡す責任があります。

ここでプロセスを複雑にします。以前と同様に、新しいレコードがテーブルに追加されます。テーブルからデータを取得して編集した場合、これらの変更を保存します。saveItem

関数を書き換えて、新しいメソッドを追加します。

let saveItem = () => {
    let form = $$( "film_form" );  
    let list = $$( "film_list" );  
    let item_data = $$("film_form").getValues();    
    if( form.isDirty() && form.validate() ){
        if( item_data.id ) 
            list.updateItem(item_data.id, item_data);
        else 
            list.add( item_data );
    }
}

この関数には、次のアルゴリズムがあります。
  • 最初に、フォームデータが検証に合格したかどうか、およびフォームが変更されたかどうかの2つの条件のチェックが開始されます。フォームの変更はisDirty（）メソッドによって監視されます。
  • その後、条件「item_data.id」により、新しいレコードを定義できます。文字列は、IDを含むすべてのデータとともにフォームに送信されます。IDにはフォームにテキストフィールドがありませんが、データには必須です。フォームは送信されたすべての値を保存して読み取ることができるため、idを使用して確認します。
  • レコードを調整した後、add（）メソッドによって新しいレコードが追加されるか、現在の要素が新しいプロパティで更新されます。


編集したデータを更新するには、updateItem（）メソッドを使用します。このメソッドは2つのパラメータを取ります。選択された要素のIDと、フォームから取得された新しいプロパティのセットです。関数が呼び出されると、現在のデータが置き換えられ、新しいデータで補足されます。

ステップ2.組み込みエディターのインストール


前のステップでは、フォームを使用してテーブルデータを編集するオプションを検討しました。ただし、ページ上のフォームは必ずしも必要ではないため、要素に直接編集メソッドを実装します。この方法は、表など、大量のデータを扱うためのその他のコンポーネント（リストなど）の両方に適用できます。

「製品」タブのツリービューから始めます。テーブルのコードは、products_module.jsファイルにあります。Treetable

ウィジェットではテーブルの内容を編集できますが、この機能はデフォルトで無効になっています。編集は、テーブル構成editable:true、構成、および各列の構成の2つの場所で構成されます。スピーカーが設置されましたeditor:”text”。 「text」属性は、エディターのタイプを設定します。エディタータイプの詳細については、ドキュメンテーション。

const products = {
    editable:true,
    view:"treetable",
    scrollX:false,
    columns:[
        { id:"id", header:"", width:50 },
        { id:"title", header:"Title", fillspace:true, 
        template:"{common.treetable()} #title#", editor:"text" },
        { id:"price", header:"Price", width:200, editor:"text" }
    ],
    select:"row",
    url:"data/products.js",
    rules:{
        "price":webix.rules.isNotEmpty,
        "title":webix.rules.isNotEmpty
    }
}

ステップ3.リストとグラフにデータを追加する


記事のこの部分では、リストとチャートのウィジェットが使用され、そのコードはusers_module.jsファイルにあり、「ユーザー」タブにレンダリングされます。

チャートウィジェットとリストウィジェットは同じデータ、つまりJSON配列を使用します。したがって、データを追加する機能を実装する場合は、両方のコンポーネントに分類する必要があります。

これを行うには、usersモジュールで、「Add new person」ボタンを描画します。

{
    cols:[
        { 
        view:"button", 
        id:"btn_add_person", 
        value:"Add new person", 
        width:150, css:"webix_primary", 
        click:addPerson
        },
       {}
    ]
}

ボタンをクリックすると、addPerson関数が呼び出され、新しいレコードがリストに追加されます。

グラフには各レコードの経過時間が表示されるため、より大きなスプレッドに対して乱数が生成されます。

新しいレコードを追加する関数：

    let obj = {

ステップ4.アイテムの削除


要素の削除については、[リスト]ウィジェットを[ユーザー]タブとして、[表]ウィジェットを[ ダッシュボード ]タブとして使用して説明します。

まず、テーブルウィジェットで新しい列を作成し、そこにアイコン付きのhtmlタグを配置します。

アイコンのクリックを追跡するには、onClickテーブルプロパティを使用する必要があります。これは、CSSクラスの1つに従って要素に関連付けられたハンドラーを格納するオブジェクトです-私はそれを持ってい"delete_icon"ます。

アイコンのクリックを処理するために、私はそれらのdelete_iconクラスを取得し、それをキーとしてonClickオブジェクトに書き込みます。このキーの値は関数（ハンドラー）になります。引数の中にはレコードのIDが含まれるため、remove（）メソッドを使用してレコードが削除されるのを妨げるものはありません。

テーブルウィジェットコード：

const table = {
    view:"datatable", 
    id:"film_list",
    scroll:"y",
    select:true,
    url:"data/data.js",
    hover:"myhover",
    columns:[
        { id:"rank", header:"", width:50, css:"rank"},
        { id:"title", header:"Film title", fillspace:true},
        { id:"year",  header:"Released", width:100},
        { id:"votes", header:"Votes", width:100},
        { id:"rating", header:"Rating", width:100},
        { header:"", template:"<span class='webix_icon wxi-close delete_icon'></span>", 
        width:35}
    ],
    onClick:{
        delete_icon(e, id){
        this.remove(id);
        return false;
        }
    },
    on:{
        onAfterSelect(id){
            let values = $$("film_list").getItem(id);
            $$("film_form").setValues(values)
        }
    }
}

リストウィジェットの行削除アルゴリズムは、テーブルウィジェットの場合と同じです。
