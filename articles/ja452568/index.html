React 16+の新機能の詳細分析、パート1：一般情報

本日、この素晴らしい資料の翻訳の最初の部分をお伝えします。これは、バージョン16以降、このライブラリに登場した新しいReact機能の詳細な分析に専念しています。この資料の作成者は、5年間プログラミングに携わっています。過去にはデザイナーであり、イラストレーターとしても活躍した。彼は記事にコミックを含めました、彼の意見では、読者が議論されていることをよりよく理解するのに役立ちます。 オリジナルでは、この記事のタイトルは「Reactの再紹介：v16以降のすべてのReactの更新がわかりやすくなった」です。タイトルは、おそらくすでにReactに精通している読者が、このライブラリーをもう一度知るように招待されていることを示しています。このReactの再紹介は、彼がすでに知っていることを整理し、新しい何かを発見するのに役立ちます。

誰のために、そしてなぜこの資料は書かれたのですか？


私は4年ほど前に最初のReactアプリを作成しました。それ以来、Reactの基本原則は変わっていません。同じ宣言コードと同じコンポーネントがあります。これらはすべて良好ですが、最新のReactアプリケーションの記述方法は以前とは異なります。長年にわたって、Reactには多くの新しいものが登場し、このライブラリの一部は削除されました。

Reactを長い間研究してきた場合、このライブラリーの新機能に特に精通していない可能性があります。さらに、いわばイノベーションに追随する人でも、いわば「迷子」になる可能性があります。古いものを破棄する必要があり、新しいものを受け入れる必要があることを認識して、状況を修正する方法は？特に役立つと思われる最新のReact機能をどのように強調しますか？

経験豊富なプログラマーでも、必要に応じて、古いものを忘れ、代わりに新しいものを学ぶために迷うことがあります。これは、完全に新しい概念を学ぶのと同じくらい難しいように見えるかもしれません。

あなたもこのようなものを感じているなら、私の仕事があなたがすべてをその場所に置くのを助けることを願っています。

同じことは、Reactを学び始めたばかりの人にも言えます。

周りには古い情報がたくさんあります。かなり古い出版物からReactを学ぶなら、確かに基本をマスターすることができます。しかし、最近のReact開発には、注目に値する多くの新しい興味深い機能があります。これらの機能については、古い作業方法を後で忘れて新しい方法を学ぶのではなく、できるだけ早く学ぶことが最善です。

実際のところ、ここでは、React 16+で登場したすべての新機能を詳細に分析します。

React 16+の新機能


一見すると、バージョン16のリリース後、Reactではそれほど多くの変更が加えられていないように見えます。しかし、実際にはそうではありません。これが私たちが話すことです：

  1. 新しいコンポーネントのライフサイクルメソッド。
  2. Context APIを使用した簡略化された状態管理。
  3. contextTypeプロパティとContext API機能。
  4. プロファイラーアプリケーション：図、相互作用分析。
  5. React.lazyとSuspenseを使用したレイジーマテリアルの読み込み。
  6. React.memoを使用して作成されたPureComponentベースのコンポーネントの機能バージョン。
  7. フックを使用してReactアプリケーションを簡素化します。
  8. 高度なフックベースのReactアプリケーション設計パターン。

このリストの各項目は、個別の大きなセクションとして表示されます。最初のセクションでは、コンポーネントのライフサイクルの新しい方法について説明します。
