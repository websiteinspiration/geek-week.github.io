Menginstal dan Mengkonfigurasi LXD pada OpenNebula

Terjemahan artikel disiapkan sebelum peluncuran kursus Administrator Linux .









Pada artikel ini, Anda akan menemukan panduan tentang dasar-dasar sistem orkestrasi berbasis cloud OpenNebula di LXD. Pertama, kita akan menggunakan alat MiniONE sederhana untuk membuat lingkungan cloud node tunggal dalam mesin fisik atau virtual untuk pengguna yang ingin mencoba fungsionalitas OpenNebula, kemudian melihat sesi kerja reguler dengan GUI dan CLI, dan akhirnya memberikan beberapa tautan yang akan membantu dalam menciptakan lingkungan produksi terdistribusi .



Penerapan Otomatis dengan MiniONE



MiniONE adalah alat yang mengonfigurasikan host fisik atau mesin virtual sebagai cloud node tunggal untuk dengan cepat menggunakan case uji yang sederhana namun berfungsi penuh.



Untuk mengevaluasi LXD, MiniONE membutuhkan mesin virtual khusus atau host fisik dengan instalasi baru Ubuntu 18.04 atau 18.10 dengan setidaknya 2GB RAM dan 20GB ruang disk kosong, serta akses root istimewa. Sebagai contoh, MiniONE memudahkan untuk membangun lingkungan LXD / OpenNebula pada mesin virtual Amazon. Ukuran minimum yang disarankan paling mungkin t2.medium



. Sederhana memberinya 25 GB ruang disk dan akses ke port TCP 9869, di mana WebUI berjalan.



Jalankan skrip MiniONE pada sistem khusus: 



wget https://github.com/OpenNebula/minione/releases/download/v5.8.0/minione
chmod u+x minione
sudo minione --lxd


Mengenal GUI



Setelah langkah dengan MiniONE, Anda akan menerima instruksi untuk menghubungkan ke antarmuka web seperti berikut: 



### Report
OpenNebula 5.8 was installed
Sunstone (the webui) is runninng on:
  http://192.168.100.101:9869/
Use following to login:
  user: oneadmin
  password: o6ARsMAdGe


Setelah menyelesaikan pekerjaan MiniONE, hal pertama yang akan kita lakukan adalah login sebagai oneadmin dan lihat Admin View di Sunstone , di mana akan ada lebih banyak opsi daripada antarmuka lain untuk pengguna biasa. Lihatlah semua sumber daya cloud yang diunduh.







Di Tampilan Admin, Anda dapat melakukan apa saja dengan OpenNebula, tetapi Anda tidak perlu memberikan kesempatan yang sama kepada pengguna akhir! Beralihlah ke Cloud View dan lihat apa yang akan dilihat pengguna akhir di OpenNebula.







Antarmuka Cloud Viewjauh lebih sederhana karena ditargetkan pada pengguna akhir. Buat mesin virtual baru dengan mengklik tombol +. Pilih satu-satunya templat yang tersedia dan klik tombol Buat. Setelah mengklik tombol Buat , Anda akan dibawa ke dasbor, tempat Anda dapat melihat semua mesin virtual yang sedang berjalan.







Untuk membuka panel kontrol, Anda dapat mengklik mesin virtual Anda. Di sana Anda dapat mengaksesnya dari VNC, menyimpan statusnya, me-reboot-nya, dll.:







Dengan mengeklik ikon konsol, Anda masuk ke konsol sebagai root sebagai pengguna melalui VNC.







Menggunakan peran oneadmin, Anda dapat mengonfigurasi apa yang dapat mereka lihat dan lakukan. pengguna cloud Anda.



Temui CLI



OpenNebula dimulai sebagai oneadmin, dan administrator utama harus mengeksekusi perintah atas nama pengguna ini, jadi hal pertama yang perlu Anda lakukan adalah beralih ke oneadmin : 



su - oneadmin




Dari akun oneadmin Anda dapat melihat sumber yang dimuat:

Kami memiliki satu simpul visualisasi: 



onehost list


Dibuat Gambar CentOS 



oneimage list


Templat Mesin Virtual Terdaftar: 



onetemplate list


Anda dapat melihat konfigurasi templat lebih detail. 



onetemplate show 0
