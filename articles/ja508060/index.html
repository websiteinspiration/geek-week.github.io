KotlinとJava

またあったね。新しいKotlinバックエンド開発コースの立ち上げを見越して、KotlinとJavaの違いを説明する記事の翻訳を用意しました。

KotlinはJetbrainsによって開発されたプログラミング言語であり、IntelliJ、WebStorm、PhpStorm、およびVisual StudioのReSharperプラグインで有名です。彼らは、Javaより優れているが、機能的にはJavaと互換性のある新しいプログラミング言語を探しました。Java、C＃、Scala、Groovy言語に触発されて、Jetbrainsの人々はチームを結成して新しいプログラミング言語を開発しました。Kotlinは、Javaのすべての苦痛を経験した人々によって設計されました。



Kotlinとは何ですか？



Kotlinが最初に導入されたのは2011年で、2016年2月にバージョン1.0の安定版がリリースされ、その後3月に1.1がリリースされました。オープンソースのプログラミング言語は、JVM（Java Virtual Machine）、Android、JavaScriptでコンパイルされています。したがって、KotlinはJVMデバイスとAndroidデバイスで同時に使用できます（相互運用性）。JavaScriptを使用してフロントエンドで実行することもできます。Googleは5月のI / O会議で、Kotlinが公式にAndroid開発用にサポートされる言語になることを正式に発表しました。それ以来、言語、アプリケーション、コミュニティへの関心が大幅に高まりました。

Mediaan Kotlin



2015年と2016年にロンドンで開催されたDroidcon、およびアムステルダムでGDG DevFest 2016に関する他のKotlinレポートを訪れた後、モバイル開発チームは新しい言語を検討するときがきたと判断しました。私たちは2016年10月にそれを使用し始め、ちょうどそれに夢中になりました。最初の新しいAndroidプロジェクトは、すでに100％Kotlinで記述されていました。それ以来、JavaでのAndroid開発に戻っていません。



ここで、Javaでの経験を振り返り、Kotlinがどのように使用されているかを確認すると、次の結論を導き出すことができます。



  • コードはより簡潔です。つまり、作成して保守するコードを少なくすることができます。
  • 言語は、（非）可変性、null可能性などを考えさせるため、コードはより安全です。コードの記述中。
  • 上記も開発を加速します。複雑なことをするのが簡単になります。
  • 優れたIDEサポート（私たちはAndroid Studioを使用しています）;
  • 大きく成長しているコミュニティ。既存のライブラリがサポートされているか、Kotlinに移行されています。インターネットには、ブログの投稿、ビデオ、プレゼンテーション、そしてもちろん

    StackOverflow に関する質問/回答が満載です。
  • Kotlinは本番稼働の準備ができています。私たちはそれを制作だけでなく、Square、Pinterest、Trello、Googleなどの多くの企業で使用しています。


さて、GoogleはKotlinがAndroid開発の公式言語であることを発表しました。この言語を調べて使用し始める理由は事実上ありません。



それで、あなたはコトリンから始めるか、それに行きますか？



その開発に役立つ多くのリンクがあります：



  • 公式サイト ;
  • ドキュメント（マニュアル、ガイド、書籍など）。
  • オンラインプログラミング ;
  • Android開発 ;
  • YouTube、Medium、SlideShareでのスピーチ、ブログ、プレゼンテーション。


さらに、Android Studioのプラグインには、現在開いているJavaファイルをKotlinにすばやく変換する機能があります。それはかなりうまくいきます。コードでいくつかの小さなことを微調整する必要があるかもしれませんが、比較と研究のためにJavaコードのKotlinバージョンを即座に取得します。別の驚くべき可能性もあります。Kotlinのファイルで作業し、Javaコードの一部を貼り付けると、挿入時に自動的にKotlinコードに変換されます。



既存のプロジェクトでKotlinを使用するか、既存のプロジェクトを完全にそれに転送するには、次のアプローチをお勧めします。



  • 相互運用性を使用します（KotlinとJavaは共存できます）。
  • プラグインを使用して、ファイルとコードスニペットを変換します。
  • Kotlinのすべての新機能を記述します。
  • 編集したものすべてをJavaからKotlinに変換します。


今後について



JVM、Android、JavaScriptのサポートに加えて、Kotlinはより多くのプラットフォームのサポートに取り組んでいます。マシンコードのサポートは次の大きなステップです。たとえば、RaspBerry Pi（ベータ版）でコードを既に実行できます。JetbrainsはiOS、MacOS、およびWindowsのサポートの追加に取り組んでいます。これは、Kotlinがいつかアプリケーションの主要な新しいクロスプラットフォームソリューションになる可能性があることを意味します。今年の終わりにサンフランシスコで開催される独自のカンファレンスであるKotlinConfで、さらなる開発についてさらに学ぶことができます。
