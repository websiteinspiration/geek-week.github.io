Webpacker


WebpackerはRuby on Railsに同梱されている宝石です。

Webpackerは、Webpackのラッパーを提供します：作業用の構成を接続および開始するためのコマンド。Webpackerデファクトは、Ruby on Railsでフロントエンドを操作するための標準を設定し、最新のJavaScript機能の使用とコードを操作するための最新の原則（構造、モジュール性、アセンブリなど）を促進します。

Webpackerは、開始に必要な一般的な構成とアプリケーションの構造を定義します。これにより、確実性が高まり、さまざまな開発者によるプロジェクトの理解が簡素化されます。JavaScriptコードの場合app/javascript/、プライマリファイルのあるフォルダーが割り当てられapp/javascript/packs/application.jsます。

Webpackerは、新しいアプリケーションの作成プロセス中にデフォルトで起動し、その設定を実行します。Stimulus、Vue、Typescript、または提供されているリストからの別の構成を追加して、すぐにアプリケーションを作成できます。

rails new myapp --webpack=stimulus

または、アプリケーションの作成後に追加の構成をインストールします。

bundle exec rails webpacker:install:stimulus

Ruby on Railsフレームワークでフロントエンドを開発する=最新のJavaScript開発アプローチを使用してください。最新のJavaScript標準を使用するすべての利便性は、Ruby on Railsとよく統合されています。Webpackを使用するために必要な構成が用意されているため、プロジェクトの正しい編成に煩わされることなく、使い慣れた環境を使用して一般的なタスクの解決に集中できます。

ターボリンク


Turbolinksは、Ruby on Railsに付属するJavaScriptライブラリです。

Turbolinksの優先タスクは、サーバーの負荷を軽減し、アプリケーションURLに移動するときの「継ぎ目」を減らすことです。この機能は、ページ間の単純な単純な遷移ではなく、ブラウザでコンテンツの再レンダリングの印象を与えるため、SPAと比較されることがよくあります。

Turbolinksの動作原理：新しいアドレスへの標準的な遷移ではなく、ajaxを介して「バックグラウンドで」このアドレスへのリクエストを実行し、JavaScriptで応答をロードして、ページのコンテンツを新しいアドレスに置き換えることにより、ページ間を移動します。このプロセスには、ページ間の遷移に機能を追加したり、前のページに戻ることができる特別なイベントが伴います。例えば、
  • 別のアドレスへの移行を開始しますturbolinks:click, turbolinks:before-visit, turbolinks:visit。
  • または新しいページに対する要求の処理：turbolinks:request-start, turbolinks:request-end;
  • または新しいページを表示するプロセスturbolinks:before-render, turbolinks:render, turbolinks:load。

さらに、Turbolinksには進行状況バーがあり、ロードされたページの履歴をキャッシュし、更新されていないページ要素を指定できます。

アクションケーブル


ActionCableは、Ruby on Railsフレームワークの一部です。ActionCableはWebソケットを操作します。サーバー上のチャンネルを一覧表示するためにapp/channels/、プライマリファイルchannel.rbを含むフォルダーが提供されconnection.rbます。これらのチャネルへの接続を実装するには- app/javascript/channels/ファイルindex.jsとを含むフォルダconsumer.js。

例として、すぐにActionCableの可能性について知っておくとよいでしょう。その助けを借りてWebソケットへの最も単純な接続は、わずか数ステップで実装できます。

  1. チャンネルファイルを作成

    app / channels / hello_channel.rb 
    # app/channels/hello_channel.rb
class HelloChannel < ApplicationCable::Channel
  def subscribed
    stream_from "hello_1"
  end
end

  2. このチャンネルのサブスクリプションを作成します
    app / javascript / channels / hello_channel.js 
    // app/javascript/channels/hello_channel.js
import consumer from "./consumer"

consumer.subscriptions.create({ channel: "HelloChannel" }, {
  received(data) {
    console.log("Data received", data);
    document.body.innerText += `\nHello, ${data.name}!`
  }
})


これで、Webソケットへの接続の準備が整いました。

これを確認するには、単純なページと送信のためのアクションが必要です。これを行うには、コントローラーを作成し、そのアドレスをルートに追加します

app / controllers / hello_controller.rb 
# app/controllers/hello_controller.rb
class HelloController < ApplicationController
  def index
    render :html => "Hello page...", :layout => true
  end

  def new
    ActionCable.server.broadcast("hello_1", name: params[:name])
    head 200
  end
end


config /routes.rb 
# config/routes.rb
get "/hello" => "hello#index"
get "/hello/new" => "hello#new"


アプリケーションを起動し、アドレス127.0.0.1haps000 / helloに移動してブラウザーコンソールを開きます。この中に、Webソケット経由でサーバーから送信されたメッセージのログが表示されます。

次に、アクションメーリングのリクエストを送信します。

curl http://127.0.0.1:3000/hello/new?name=World

そして/ helloページとそのコンソールの出力を見てください。

Stimulus-Ruby on Rails開発者によるJavaScriptフレームワーク。

Stimulusは珍しいフレームワークの1つであり、独自の方法で独自のものです。これは、新しいJavaScriptアプローチを使用してフロントエンド開発を実装する一方で、すべてのアクションを制御しようとするものではなく、フロントエンドをバックエンドから分離するよう強制するものではありません。

Stimulusの基本的なタスクは、ハンドラーをイベントにバインドすることです。 Stimulusによると、ソースコードはコントローラークラスに配置する必要があり、それらのメソッドをハンドラーとして使用する必要があります。デフォルトではapp/javascript/controllers/、プライマリファイルを含むフォルダーは、プロジェクトのスティミュラスコントローラーに割り当てられますindex.js。ここでコントローラーを追加できます。これには、サフィックスが付いたファイルを作成する必要があります。_controller.js次に例を示します。articles_controller.js。次に、Stimulusローダーがそのようなファイルをインポートし、コントローラーをページ上の対応するブロックに接続します。

Stimulusのコントローラーには、コントローラーオブジェクトの初期化（initialize）、ブロック内の要素へのアクセス用ヘルパー（targets、ターゲット）、コントローラーオブジェクトのブロックへのアタッチconnectとブロックからの切断（disconnect）、ブロックのデータ属性へのアクセス（）などの追加機器がありますthis.data.get。以下は、Stimulusで記述されたアクティブ/非アクティブなステートフルブロックの例です。

// app/javascript/controllers/home_controller.js
import { Controller } from "stimulus"

export default class extends Controller {
  //  
  static targets = [ "text" ]

  initialize() {
    //      
    this.activeText = this.data.get("active-text");
    this.deactiveText = this.data.get("deactive-text");
  }

  connect() {
    //   /
    this.active = this.data.get("active") == "true";
    //  
    this.refresh();
  }

  disconnect() {
    //     
    this.element.setAttribute("data-home-active", !!this.active);
  }

  click() {
    //  
    this.active = !this.active;
    //  
    this.refresh();
  }

  //    
  refresh(){
    //     
    this.element.style.background =   this.active ? "none" : "#EEE";
    //  
    this.textTarget.innerHTML =   this.active ? this.activeText : this.deactiveText;
    //   
    this.textTarget.style.color = this.active ? "black" : "#777";
  }
}

Turbolinksがページをロードするとすぐに刺激がアクティブ化されます;刺激をページのブロックに接続するために追加の操作は必要ありません。

Turbolinksは、ページのロードを容易にするだけでなく、遷移時にそのコンテンツをキャッシュします。履歴からキャッシュされたページに戻ると、新しいページをロードするときのように、Stimulusが自動的にアクティブになります。ユニットからコントローラーを切断する前にいくつかの値を保存したい場合は、このメソッドを使用できます。disconnectその後、コントローラーを返して接続すると、最後の状態を復元できます。刺激の作業の最初の例のコードでは、（切断時どのように見ることができるdisconnect）、値は、コントローラブロックのデータ属性に固定されthis.active、そして（接続時connect）には、バック抽出します。

ページを介して戻るは、アプリケーションを操作するための便利な方法です。たとえば、フォームを操作する場合、別のページのディレクトリに値を入力/編集してから、新しいデータに戻って選択する必要がある場合があります。

別の例：Webチャットの設定ページで値を編集または表示する必要があるとします。会話でユニットからコントローラを切断するときに、最後のメッセージの識別子を覚えておくと便利です。そうすると、戻るときに、最初に欠落している新しいメッセージを要求し、次にWebソケット経由で接続を開きます。

したがって、ページ履歴に戻ることは、Webアプリケーションを操作する便利な方法として使用できます。
