Astra Linux 1.6（スモレンスク）。システムは通常のユーザーと動作する準備ができていますか？松葉杖の例


ユーザーの無力化と新しいOSのインストールプロセス

こんにちは、Habr。今日は、輸入代替要件の実装の一環として、1つの組織（以下、お客様）を国内OSに移行した経験を共有したいと思います。お客様が自分でこのOSを選択して購入したことを示す必要があります。しかし、このOSを導入、最適化し、情報保護の要件を満たすことができました。

仕事


条件付きで同じタイプのユーザーが約1000人、第2のセキュリティクラス（以下、GIS）の状態情報システムがあり、公式タスクの範囲が厳密に定義されています。GIS自体は、ローカルデータベースを備えた約30のサイトに分散されたWebサービスで構成されています。集落には遠隔地の仕事もあります。同時に、共通のドメインがあります（どこにでもあるわけではありません）。ローカル/企業のボールがあります。一部の場所では、コンピュータとオフィス機器の公園は最初の鮮度ではなく、オブジェクト間の伝送チャネルはそれほど熱くありません。ローカルシステム管理者とチーフ管理者がいます。一般的に、国内ソフトウェアへの大規模な移行前の、中規模の国家機関に特徴的な標準的な状況。

問題、テスト、研究



1. Astrovグラフィックファイルマネージャー（fly-fm）を介してボールを接続しました。 「ウィンドウ」ボールにロシア語のファイル名を表示しても問題はありません。しかし、これらの名前は、これらのファイルをデスクトップやディスク上の他の場所にコピーすると、人が読めない文字のセットに変わります。この問題について開発者と協力しましたが、修正はありますが、質問は複雑です。

Astrovソフトウェアの機能に関係なく、Linuxを使用してボールをマウントする方法があります。メソッドは機能しますが、実際の条件でテストするには時間がかかります。予備テストでは、ファイルを表示およびコピーしても歪んでいないことが示されていますが、一部のバージョンのWindows Serverには互換性の問題があります。 Linuxクライアントを接続するときにボール上のファイルへのアクセスを制限する問題の本質は、ボールのマウントを担当するcifs-toolsパッケージの開発者が推奨するさまざまなオプションで接続をテストすることです。 autofsを使用することもあれば、慣れていることから「行う必要がない」ことを指示することもあります。

2.最初に関連付けられています。 Astrov fly-fmからボールを​​マウントする場合、ユーザーはファイルをFirefoxから直接「ボール」に保存できません。彼らは2つの操作を行う必要があります。Firefoxからデスクトップに保存してから、ドキュメントをボールに投げます。同じことをファイルをサイトに配置するには、まずボールからデスクトップに移動し、次にファイルをデータベースにドロップします。

これは、Firefoxが独自のダイアログを使用してファイルを開いたり書き込んだりするためです。このダイアログでは、Astrov fly-fmで作成したボールのシンボリックリンクを指す方法がありません。

アストラが彼のグラフィック環境を使用しなかった場合、この問題は存在しませんが、Linuxで最も人気のあるもののいくつか。現時点では、autofsを使用するか、ファイルのコピーとダウンロードのルールについて従業員に指示を書き込む必要があります。

3. USBプリンターをネットワークで共有します。 2つの標準オプション：プリントサーバーのcupsとsambaを介して、プロトコルは標準設定で起動しませんでした。我々はそれを回した唯一の注意点と、クライアント・モードに、別のコンピュータのCUPSにそれを接続し、それが働いた- 1つのクライアントは1台のだけプリントサーバとそのローカルプリンタを使用することはできませんクライアントモードで構成されたコンピュータを持つことができ、そのプリントサーバが詰まっているので。

4.教育機関は、以前のオフィススイートのドキュメントでマクロを使用することがよくあります。これらのマクロは、Visual BasicではなくJavaで記述されているため、Libre Officeのマクロと互換性がありません。

さらに、Astraの作成者はJavaに関連するすべてのものを配布キットから捨てました。つまり、Astrovオフィスでマクロを書くことはまだ不可能です。唯一のオプションは、エミュレーションまたはWindowsを備えた近くのコンピューターです。

結論と決定


インフラの現状分析から、国産OSへの切り替えプロセスの自動化が想定外のトラブルにつながる可能性があると判断した。このアプローチは、管理（セントラルオフィス）にのみ適用できます。その場合、迅速に対応することができました。地区の地方施設にはかなり典型的な職場と技術的プロセスがありましたが、これは完全な標準化と「サイレントインストール」の可能性を与えません。

ネットワークを介した同時インストールの起動をまだ妨げているもの：

  • 共通ドメインの欠如;
  • 鉄と周辺の動物園;
  • ネットワークからのダウンロードは優先事項ではありませんが、リモートワークステーションにアクセスしてダウンロードの順序を変更する必要があります（一元化されたドメインがないため、事前設定のためにエージェントをインストールする運用機能はありません）。
  • リモートワークステーションとのオブジェクト間の通信チャネルが不十分です。
  • ほとんどの部分は標準ですが、実際にはすべてのユーザーが転送されたわけではありませんが、重要な部分にはWindowsでのみ実行できるタスクがあるため、技術プロセス。
  • 追加のユーザー設定なしですべてのコンピューターにOSを同時に再インストールすると、組織全体の作業が麻痺し、これは受け入れられません。

その結果、現場で展開するための最も普遍的なディストリビューションを手動で作成することが決定されました。さらに、現場でのシステム管理者向けの技術サポートのチャネルを編成し、公式タスクに関するコンサルティングと放送体験を行う必要がありました。

一般に、国内向けソフトウェアへの切り替えについては、現時点では、多くのアプリケーションの移行に関する問題は解決されていません。完全に標準的でエキゾチックです（自己作成を含む）。したがって、新しいOSへの移行には、とりわけ、公式タスクを新たに実行するための最適なソリューションを見つけることが含まれ、組織の既存の技術プロセスに大幅な変更が加えられる可能性があります。これは、設計および認証文書の更新/改訂の必要性、および全国の情報セキュリティのトピックに関する他の哲学的考察を伴う可能性があります。

検討中の組織での移行は続行されます。ステージの概要が示され、次のオブジェクトと他のワークスペースが新しいOSへの移行のために選択されています。また、このプロジェクトは、他のお客様および新規のお客様の国内ソフトウェアへの移行のために上から開始された計画と関連しています。プロセスは困難に進んでいますが、時には緊急ですが、さまざまなアプローチがテストされ、適用されます。分析、ソリューションの選択のための情報はたくさんありますが、可能な限り可能な限りカバーします。

フラッシュドライブの準備とインストール


1.インストールするには、USBフラッシュドライブまたは他の外部NTFS USBドライブ（
少なくとも16GBの容量）が必要です。 Porteusのイメージの重量は270MBです。これは基本的にモバイルLinuxであり、インストーラースクリプトを実行するためにのみ必要です。インストールされているAstra Linuxのイメージの重量は8GBです。必要に応じて、ギガバイトのフロアでイメージを縮小できます。8ギガバイトのフラッシュドライブで十分です。

2.少し変更したPorteusポータブルOS のシステムイメージとディレクトリツリーをコピーします。

3.スクリプトを実行して、フラッシュドライブを起動可能にします。

/boot/Porteus-installer-for-Linux.com —  Linux
/boot/Porteus-installer-for-Windows.exe —  Windows

4.準備したドライブから起動し、指示に従います



。Porteusの場合、インタラクティブモードでAstra Linuxを展開するためのbashスクリプトを含むモジュールを作成しました。



このスクリプトは、ブートドライブをスキャンしてシステムイメージを探し、システムをインストールするディスクを選択してイメージをディスクに展開し、システムが最初に起動する前にいくつかの設定を行うことができます。

脚本
#!/bin/sh
# unmount swap disks
for i in `cat /proc/swaps | grep dev | awk -F" " '{print $1}'`; do swapoff $i; done
#variables
DIALOG=${DIALOG=dialog}
tempfile=`mktemp 2>/dev/null` || tempfile=/tmp/temp$$
tempfile1=`mktemp 2>/dev/null` || tempfile1=/tmp/temp$$
tempfile2=`mktemp 2>/dev/null` || tempfile2=/tmp/temp$$
tempfile3=`mktemp 2>/dev/null` || tempfile3=/tmp/temp$$
tempfile4=`mktemp 2>/dev/null` || tempfile4=/tmp/temp$$
#image selection
setimage () {
mdevs=`mount | grep fuseblk | awk -F" " '{print $3}'`
cd /
$DIALOG --backtitle "   :" \
--menu " " 15 80 5 \
`find $mdevs -name *.di -type f | awk '{print $0,"image"}'` 2> $tempfile
retval=$?
case $retval in
0)
image=`cat $tempfile | awk -F/ '{print $6}'`
input=`cat $tempfile`
echo "  : $image"
sleep 2;;
1)
echo "  ."
exit 0;;
255)
echo "  ESC."
exit 0;;
esac
}
#disk selection
setdisk () {
sdisk=`mount | grep fuseblk | awk -F" " '{print $1}' | awk -F/ '{print $3}' | tr -d [0-9]`
$DIALOG --backtitle "     $image :" \
--menu "`parted -l | grep -E "| \/"`" 15 80 5 \
`fdisk -l | grep " /"| grep "/dev/s" | awk -F" " '{print $2,$1}'| awk -F: '{print $1,$2}' | sed "/$sdisk/d"` 2>
$tempfile1
retval=$?
case $retval in
0)
disk=`cat $tempfile1 | awk -F/ '{print $3}'`
output=`cat $tempfile1`
echo " $disk       $image"
sleep 2;;
1)
echo "  ."
exit 0;;
255)
echo "  ESC."
exit 0;;
esac
}
#cloning (dd process)
cloning () {
bs=`hdparm -I $output | grep "cache/buffer size" | awk -F" " '{print $4}'`
umount -l $output"*"
dd if=/dev/zero of=$output bs=512 count=1
clear
sleep 1
echo "  $image   $disk..."
#dd if=$input of=$output bs=$bs 2>/tmp/error & pid=$!
dd if=$input of=$output bs=8M 2>/tmp/error & pid=$!
sleep 2
while [ -d /proc/$pid ];
do kill -USR1 $pid && tail -n 2 /tmp/error | grep GB && sleep 10 && clear;
done
clear
tail -n 3 /tmp/error
echo "  ."
rm /tmp/error
sleep 2
}
#resize second partition
resizepart () {
echo "     :"
fdisk -l $output"2" | head -n 1
echo "    $disk..."
parted $output rm 2
parted $output mkpart primary 2GB 100%
resize2fs -f $output"2"
e2fsck $output"2"
echo "   :"
fdisk -l $output"2" | head -n 1
sleep 2
}
#hostname
sethost () {
$DIALOG --inputbox "  :" 10 80 2> $tempfile2
retval=$?
case $retval in
0)
hostn=`cat $tempfile2`
mount $output"2"
echo $hostn >/mnt/$disk"2"/etc/hostname
hosts=/mnt/$disk"2"/etc/hosts
/bin/cat << ENDFILE >$hosts
127.0.0.1 localhost
127.0.1.1 $hostn
# The following lines are desirable for IPv6 capable hosts
::1 localhost ip6-localhost ip6-loopback
ff02::1 ip-allnodes
ff02::2 ip6-allrouters
ENDFILE
echo "   : `cat /mnt/$disk"2"/etc/hostname`"
sleep 2;;
1)
echo "  ."
exit 0;;
255)
echo "  ESC."
exit 0;;
esac
}
#adduser
adduser () {
$DIALOG --inputbox "  :" 8 40 --inputbox " :" 8 40 2> $tempfile3
retval=$?
case $retval in
0)
name=`awk '{print $1}' $tempfile3`
pass=`awk '{print $2}' $tempfile3`
passf=/mnt/$disk"2"/tmp/pass
/bin/cat << ENDFILE >$passf
parsec_pam del
useradd -m -N -s /bin/bash -G cdrom,audio,video,floppy,plugdev,dialout,users $name
echo -e "$pass\n$pass\n" | passwd $name
passwd -w 14 -x 90 $name
parsec_pam add
ENDFILE
chmod 755 $passf
chroot /mnt/$disk"2" /tmp/pass
echo " : $name"
echo " : $pass"
sleep 2;;
1)
echo "  ."
exit 0;;
255)
echo "  ESC."
exit 0;;
esac
}
#kasperftp
kasper () {
$DIALOG --inputbox " ip  ftp    :" 10 80 2> $tempfile4
retval=$?
case $retval in
0)
kftp=`cat $tempfile4`
obnovftp=/mnt/$disk"2"/opt/kaspersky/kesl/bin/kesl-obnovftp
/bin/cat << ENDFILE >$obnovftp
SourceType=Custom
ConnectionTimeout=10
[CustomSources.item_1]
URL=ftp://ftpuser:1qwertyuiop_0@$kftp/Updates
Enabled=Yes
ENDFILE
echo "ip  ftp    : $kftp"
sleep 2;;
1)
echo "  ."
exit 0;;
255)
echo "  ESC."
exit 0;;
esac
} autofs () {
mcedit /mnt/$disk"2"/etc/auto.share
}
#statement
state () {
echo " $image    $disk."
echo "     $disk:"
fdisk -l /dev/$disk
echo "ip  ftp    : `cat /mnt/$disk"2"/opt/kaspersky/kesl/bin/kesl-obnovftp |
grep URL | awk -F@ '{print $2}' | awk -F/ '{print $1}'`"
echo " : `cat /mnt/$disk"2"/etc/hostname`"
echo " : $name"
echo " : $pass"
echo "  :"
cat /mnt/$disk"2"/etc/group | grep $name | awk -F: '{print $1}'
echo "  .  reboot   Ctrl+Alt+Del   ."
}
#program start:
setimage
setdisk
cloning
resizepart
sethost
adduser
kasper
autofs
state
exit 0

プリンターの接続。

プリンターは、ブラウザーのCUPS構成ページ（127.0.0.1：631またはlocalhost：631）で構成されます。この

ページはロシア語に翻訳されており、直感的です。プリンターをインストールするには、システム管理者権限が必要です。ネットワーク上でプリンターを公開するには、サーバー設定で2つのアイテムをアクティブにする必要があります。

ネットワークプリンターを追加するときは、通常、AppSocket / HP JetDirectまたはIPアドレス付きのインターネット印刷プロトコル（ipp）が使用されます。

ネットワークプリンターがこれらの方法を使用して接続しなかった場合は、ファイルを編集して、ローカルCUPSプリントサーバーを別のサーバーのクライアントにすることができます。/etc/cups/client.confで、プリンターが接続されているマシンのネットワークアドレスを指定します。この場合、ドライバーをインストールする必要はありません。 CUPSデータベースには膨大な数のドライバーがありますが、ブランドのドライバーのインストールが必要になる場合があります。

一部のXeroxネットワークプリンターでは、独自のドライバーとコンフィギュレーターを次のコマンドで既にインストールしています。

xeroxprtmgr
