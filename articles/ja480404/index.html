MikrotikとLinux。ルーチンと自動化

自己紹介、私の名前はアンドリューです。

最初のタスクは次のとおりです。Mikrotikの数百の構成を作成し、各ovpnで証明書を生成し、構成を数百のMikrotik構成と証明書およびキーにアップロードします。

各証明書のパスワードは一意です。何百もの設定を手で作成するのはナンセンスです。明日は新しい設定を作成するように指示するか、Mikrotikで他のいくつかの変更を記入したいとします。

私が達成したかったこと：

  1. 多くのスクリプトを自動的に作成します。
  2. 必要な機器に自動的に送ります。
  3. エラーなしで動作することを確認するか、設定が正しく実行されなかったことを確認してください。


BashとFTPを使用して、すべてがうまくいきました。

理論のビット
RouterOS ( - .auto.rsc). , FTP , , '/import' ( FTP).
, . .auto.log

AWK — C- (, ) ( ). .

Expect — Unix, Don Libes - Tcl, , telnet, ftp, passwd, fsck, rlogin, tip, ssh, .


まず、3つのファイルを作成しました。

  1. その中のList.txtファイルには、OVPN接続の名前（私の場合はストアの名前）とキーのパスワード（タブで区切られている）が記録されていました。
  2. sh1.sh-list.txt +パイプラインでの行ごとの解析コマンドを含むbashファイル。
  3. sh2.sh-Echoと変数を使用して構成ファイルを作成し、それに設定を書き込むbashファイル。

そう：

vi list.txt
gamazin1	pass1
gamazin2	pass2
gamazin3	pass3


vi sh1.sh
#!/bin/bash
awk '{print "./sh2.sh "$1 " "$2}' ./list.txt | /bin/bash 
sudo chmod +x sh1.sh

vi sh2.sh
#!/bin/bash
echo /certificate > rsc/$1.auto.rsc
echo import file-name=$1.crt passphrase=$2 >> rsc/$1.auto.rsc
echo import file-name=$1.key passphrase=$2 >> rsc/$1.auto.rsc
echo import file-name=ca.crt passphrase=$2 >> rsc/$1.auto.rsc
echo /interface ovpn-client >> rsc/$1.auto.rsc
echo add certificate=$1.crt_0 connect-to=10.10.10.10 name=$1_ovpn port=1111 user=$1 profile=default auth=sha1 cipher=blowfish128 >> rsc/$1.auto.rsc
#      ehco.
sudo ./sh1.sh

注意点が1つあります。

$ 1.crtと$ 1.keyは、同じlist.txtを使用して事前に作成しました。これは、ファイル名がストアの名前でもあるためです。

100の構成を作成しました。次に、同じAWKとExpect + FTPを使用して送信する必要があります。

4つのファイルを作成します。

  1. list2.txt – Mikrotik IP| | | . .
  2. newsh1.sh – list2.txt +
  3. newsh2.sh – FTP Expect , .
  4. $1.auto.rsc – $1, . gamazin1.auto.rsc.

vi list2.txt
192.168.1.1 admin   password1	gamazin1
192.168.2.1 admin   password2	gamazin2
192.168.3.1 admin   password3	gamazin3
#  1  list.txt       

vi newsh1.sh
#!/bin/bash
awk '{print "./newsh2.sh "$1 " "$2 " "$3 " "$4}' ./list2.txt | /bin/bash

sudo chmod +x newsh1.sh

Sudo apt-get install expect
vi newsh2.sh
#!/bin/bash
Mikrotik=$(expect -c "
    set timeout 3
    spawn ftp $1
    expect \"Name*\"
    send \"$2\n\"
    expect \"?assword:\"
    send \"$3\n\"
    expect \"ftp>\"
    send \"put $4.crt\n\" 
    expect \"ftp>\"
    send \"put $4.key\n\" 
    expect \"ftp>\"
    send \"put ca.crt\n\" 
    expect \"ftp>\"
    send \"put $4.auto.rsc\n\" 
    expect \"ftp>\"
    send \"bye\n\"
")
sudo ./newsh1.sh


最初に、キーと一緒に証明書を送信し、次に構成のみを送信しました（逆の順序で送信した場合、そのような証明書がないことを誓い始めます）。

設定や証明書がホットケーキのように散在しています。FTP経由の.auto.rscという名前のため、Mikrotikが自動的にそれらをドライブし、設定が適用されました。

上記の設定を使用して1行を変更することで、すべてが優れていることを知りたいと思います。

着る

send \"get $4.auto.log\n\"
#  .log    .rsc


私たちはすべてのファイルをダウンロードし、ファイルを並べ替えました。0バイトのファイルサイズでは、Mikrotikに問題があります。ファイル名から、list2.txtファイルでIPを確認できます。また、適切なバージョンでは、ファイルを開くと、「スクリプトファイルが正常にロードされて実行された」ことがわかります。

これらのファイルのスクリプトを作成することもできますが、ファイルを重みで並べ替える必要がありました。

私は別のスクリプトも作成しましたが、FTPを閉じたり、その逆を行ったりするこれらすべてのアクションの後に、SSHを介して開きます。FTPをSSHに変更して、...

FTPで遊ぶ
/ip service set ftp disabled=yes
#  SSH   
#      
/file remove $4.auto.rsc


すべてを一度に1つの構成にプッシュしようとしましたが、コマンドの最後にあるファイルの削除では結果が得られなかったため、このコマンドを受け取ったファイルを消去できませんでした。ニュアンスはまだありました。

そして最後に...幸せ！個々のMikrotikリストを管理することも、一度にすべてを管理することもできます。これで、FTPを開いて閉じることで、コマンドを送信し、コマンドが完了したことを確認できます。しかし、私の意見はFTPより便利です。SSHは途中で落ちることはありませんでした。ログフォルダーを確認することで、すべてが順調に進んだことを確認できます。

この記事ではツールを紹介しましたが、自分でそれらを使用する方法を考えることができます...私
はUbuntu 16.04ですべてを行いました。
