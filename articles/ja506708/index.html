VPN / Mikrotik 2要素認証-シンプルでスケーラブル

こんにちは！

この記事を書くために、ユーザーの記事の同様のコンテンツを読むように促されましたnkusnetsov。ビューの数は、コミュニティがこのトピックに興味を持っていることを示しています。

したがって、私は以前に私が実装した独自のソリューションをあなたと共有することにしました。

  • 低レベルのエントリとコードのシンプルさ（他の従業員による理解/デバッグ用）
  • 単純なROSスクリプトは負荷を作成せず、hAP Liteでも機能します
  • スケーラビリティ-負荷または地理的な分散を軽減するために多数のVPNゲートウェイを接続する機能
  • Mikrotik CHRをVPNサーバーとして使用する機能
  • 「1xN」-負荷の増加に伴う拡張の可能性がある無制限の数のルーター用の1つのSMSゲートウェイ
  • 別のルーターを「特定の」モデムにバインドする機能（何のために？-詳細は後ほど）
  • リモートサーバーで1つのphpスクリプトを使用する
  • VPN接続を開始したデバイス、SMSからのリンクによる認証は関係ありません
  • サーバーに従業員の承認を記録する機能
  • 複数のモデムからランダムにSMSを送信することにより、フォールトトレランスを向上させ、システムの負荷を軽減します


UPD 1.0-PHPコードが更新されました。これには、スクリプトへのすべての要求のロギングと、複数のゲートウェイが指定されている場合は複数のゲートウェイを介してランダムにメッセージを送信することが含まれます。


問題と解決策の説明


タスク1


このソリューションの開発プロセスでは、USBモデムの数を最小限に抑えるようにタスクを設定しました-所有コストを削減し、管理を簡略化し、モデムを1か所にローカライズして、システムの保守性を向上させます。

「信頼できるチャネル」にインストールされた1台のルーターを基本的なSMSゲートウェイとして使用することが決定されました。私が知らなかった唯一のことは、システムの負荷、モデムが対応し、オペレーターがそれをブロックするかどうかでした。したがって、私はいくつかのモデムを接続し、システムをさらにグループに分割することを可能にしました。さらに、追加のモデムを単純なUSBハブを介してベースゲートウェイに接続できます。ただし、完全に機能するには、ルーターの数に関係なく、最小限の機能で1つのモデムが必要です。

タスク2


ルーターの設置エリアでローカルSIMカードを使用できるようにするため。
例：カザフスタンに複数の店舗がある広い支店ネットワーク。ロシア連邦からのSMSメッセージの送信は非常に費用がかかります。このソリューションにより、カザフスタンの従業員は市内番号からSMSを受信できます。

しかし、反省の過程で、解決策はすでにタスク1で見つかり実装されていることがわかりました。

タスク3


承認済みデバイスに物理的にいる必要がないモバイルデバイスからの承認のトンネル。

目標は、ユーザートンネルだけでなく、VPN接続（Mikrotik-> Mikrotik、Server-> Mikrotikなど）も承認する機能です。この場合、これらのトンネルを担当するユーザーは、SMSメッセージからのリンクをたどるだけで済みます。ログインしたいトンネルが表示されます。

このように認証を行う可能性があるため、スクリプトを外部リソース（どこからでもアクセスできるサーバー）に転送し、認証コードを使用してファイアウォールのアドレスリストをマークし、その後ロックを解除する必要がありました。

この問題の解決策は、RouterOS API（またはSSH）の使用をすでに明確に暗示しています。APIが最も簡単なオプションとして勝ちました。

ロジック


  1. ユーザーは事前に追加されたユーザー名とパスワードでログインします
  2. VPNゲートウェイはそのIPをアドレス待機リストに入れ、サーバーにPOST要求を送信します
  3. サーバーはデータを受信し、確認コードを生成し、認証コードを生成して、フォームの連絡先番号にSMSを送信します。
  4. ユーザーは任意のデバイスからリンクをたどります
  5. サーバーはルーターにリクエストを送信し、コメントに認証コードが含まれるエントリのアドレスリストに存在するかどうかを確認します。レコードが見つかった場合、それを削除し、ユーザーは成功したログインページを表示します。

他のすべてのケースでは、何かがテストに合格しない場合、ユーザーはリクエストエラーを受け取ります。

セットアップとコード


次に、概念的な部分からMikrotikのセットアップ、コードとその説明に移りましょう
説明：これらのスクリプトとphpコードは、あらゆるタイプのVPNサーバーで機能します。サーバーの種類（PPP / L2TP / SSTP）を選択し、プロファイルとトンネルを設定します。「好みに応じて」選択しますが、クイックセットアップの一般的な例を示します。

Mikrotikに追加：


ファイアウォール


許可されたリソースのリストを必ず作成してください。その中には、独自のMTアドレス、パブリックDNS、認証が行われるサーバーアドレスがあります。 
/ip firewall address-list
add address=10.10.0.1 list=Allow-list
add address=8.8.8.8 list=Allow-list
add address=synome.ru list=Allow-list


待機リストから許可されていないホストへのVPNユーザーのトラフィックをブロックするルールを追加する 
/ip firewall raw
add action=drop chain=prerouting dst-address-list=!Allow-list src-address-list=VPN-blocked disabled=no


PPPプロファイル


アイドル接続タイムアウトを設定したプロファイルを作成します。時間は、次のOn Upスクリプトで指定した時間よりも短くする必要があります。それ以外の場合は、認証が行われていない場合、リストをアドレスリストから削除した後、ユーザーはその差（アイドルタイムアウト-アドレスリストタイムアウト）に等しい時間アクセスできます。

プロフィールを追加 
/ppp profile
add dns-server=10.10.0.1 idle-timeout=59m local-address=10.10.1.1 name=2F-VPN use-compression=no use-encryption=no use-mpls=no


次に、スクリプトの最後のタブに以下を追加します。

上に 
:global ruid "#ROUTERLOGIN#";
:global pass "#ROUTER_PASSWORD#";
:local userip [/ppp active get [find name=$user] address];
# if phone number stored in comment
#:local userphone [/ppp secret get [find name=$user] comment];
# if phone number = username
:local userphone $user;

:local authkey [/tool fetch http-method=post http-data="ruid=$ruid&pass=$pass&tel=$userphone" url="https://yourwebsite.ru/" mode=https as-value output=user];

/ip firewall address-list remove [find address=$userip];
/ip firewall address-list add address=$userip list=VPN-blocked timeout=1h comment=($authkey->"data");

:log info message="User connect:";
:log info message=$userphone;
:log info message=$userip;
:log info message=($authkey->"data");
