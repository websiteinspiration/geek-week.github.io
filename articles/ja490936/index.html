iOSでMLを作成する



人工知能は毎年近づいてきており、昨日だけ限られた数の人々しか利用できませんでした。今日、私たちは機械学習モデルを自分で作成し、プロジェクトで使用できます。この記事では、ML作成ツールを紹介し、画像を分類するためのモデルのトレーニングを試みます。

しかし、始める前に、ちょっとした歴史と説明。 Appleは、開発者向けの年次会議WWDC2018で、機械学習Create MLを操作するためのツールを紹介しました。 Create MLでトレーニングされたモデルは、一連のトレーニングデータに機械学習アルゴリズムを適用した結果です。モデルはスペースをあまり取らない（約3Mb）ので、プロジェクトに保存できます。当初、Xcode 10のプレイグラウンドを使用してモデルをトレーニングすることが提案され、画像、テキスト、およびテーブルでの作業がサポートされていました。プロジェクトの遊び場を起動するとき、CreateMLライブラリをインポートしてMLImageClassifierBuilderを実行する必要がありました（画像を分類するためにモデルをトレーニングした場合）。

import CreateMLUI

let builder = MLImageClassifierBuilder()
builder.showInLineView()

2019年、Appleは機械学習モデルを操作するためのまったく新しいツールを導入し、Playgroundsから分離しました。コードを1行も追加せずにモデルをトレーニングできるようになりました。データタイプのリストには、サウンドとアクティビティが追加されています。合計で、5つのデータタイプと9つのモデルテンプレートが表示されます。各データ型と、結果として得られるモデルを見てみましょう。

画像




画像を使用してトレーニングする場合は、画像分類子（画像分類子）とオブジェクト検出器（オブジェクト検出器）の2つのテンプレートを使用できます。

画像分類子は、その内容に基づいて画像を分類するために使用されます。モデルをトレーニングしたら、画像内の1つのオブジェクトを識別し、それをあるタイプに関連付けることができます。たとえば、画像で動物を識別し、写真に描かれている花を区別することもできます。 MLの作成では、画像分類子テンプレートは転送トレーニングを使用します。転送トレーニングは、事前トレーニング済みのモデルをユーザーデータと組み合わせる機能です。これにより、少数の画像でモデルをトレーニングできます。

Object Detectorは、深層学習モデルです。人、建物、車など、画像内のいくつかのオブジェクトを識別できます。そして、これらすべてを1つの画像に収めます。Object Detectorモデルをトレーニングするときに、コンピューターのGPUが使用されることも追加する必要があります。

音




サウンドを操作する場合、モデルの作成に使用できるテンプレートは、サウンド分類子（サウンド分類子）の1つだけです。サウンド分類子は新しいテンプレートであり、オーディオストリームで最も支配的なサウンドを決定できます。たとえば、オーディオで鳴る楽器を特定できます。Sound Classifierは、Image Classifierと同様に転送トレーニングを使用します。

アクティビティ



アクティビティは、1つのテンプレート（アクティビティ分類子）だけで表されます。活動分類子はCreate MLの新しいテンプレートであり、加速度計とジャイロスコープからのモーションデータを分類するために使用されます。

このようなモデルはディープラーニングモデルであり、コンピューターのGPUを使用してトレーニングされます。

テキスト




テキストは、テキスト分類子（テキスト分類子）とワードタガー（ワードマーカー）の2つのテンプレートで表されます。

テキスト分類子は、文章、段落、さらにはその内容に基づいて記事全体を識別するために使用されます。Word Taggerは、単語のマークやテキスト内の名前の認識に使用されます。

テーブル




テーブルは3つのテンプレートで表されます。表形式のリグレッサー（テーブルリグレッサー）、表形式の分類子（テーブル分類子）、およびレコメンダー（推奨）。

表形式リグレッサーは、いくつかのデータに基づいて数値を予測できるモデルです。たとえば、天気を予測するモデルをトレーニングできます。

表分類子は、その機能に基づいてオブジェクトを分類します。たとえば、さまざまなユーザーパラメータに基づいてオンラインストアの服のサイズを決定するモデルをトレーニングできます。モデルは、ターゲットが離散値である場合、integer、double、string、longなどのタイプで機能します。

Recommenderを使用すると、ユーザーの行動に基づいてコンテンツを推奨でき、すべての計算はデバイス上で直接実行されます。

利用可能なすべてのテンプレートに慣れたので、最初の機械学習モデルの作成を開始できます。画像分類子から始めましょう。

画像分類器


HBOシリコンバレーシリーズを見ましたか？第4シーズンのシリーズの1つで、彼らは画像内の「ホットドッグ」と「ホットドッグではない」を認識するアプリケーションを示しました。画像分類子を使用して、このアプリケーションの類似物を作成できます。ただし、最初に、モデルをトレーニングするためのデータを準備する必要があります。画像付きのアーカイブはこちら。

画像分類子モデルをトレーニングするためのいくつかの要件。画像は299 x 299ピクセル以上でなければなりません。トレーニングキットには少なくとも10枚の画像が含まれている必要がありますが、多ければ多いほど良いです。これで、画像分類子をトレーニングするときに転送トレーニングが使用されることがわかりました。これにより、少数の画像でモデルをトレーニングできます。各セットの画像の数は同じでなければなりません。そうでない場合、カテゴリの1つにシフトします。画像の約80％がモデルのトレーニングに使用され、残りの20％がテストに使用されます。また、画像は繰り返さないでください。つまり、トレーニング用の画像が含まれるフォルダーには、さらにテストに使用する画像を含めないでください。トレーニング用のフォルダーの名前は、それぞれのクラスの識別子として使用されます。モデルを使用する場合。私の例では、HotDogまたはNotHotDogという名前を使用しています。画像自体の名前は何の役割も果たしません。好きなように呼び出すことができます。シリアル番号で指定しました。

データの準備ができたので、モデルのトレーニングを開始できます。 Create MLを使用するには、Xcode 11以降が必要です。 Xcode 11.3.1を使用します。 MLの作成ツールを実行するには、Xcode->開発者ツールを開く-> MLの作成に移動する必要があります。ウィンドウが表示され、既存のプロジェクトを選択するか、新しいプロジェクトを作成できます。 [新しいドキュメント]を選択します。次に、テンプレートを選択する必要があります。この例では、画像分類子を使用します。プロジェクトに名前を付け、[次へ]をクリックして、プロジェクトを保存する場所を選択します。モデルを操作するための作業環境が開きます。



ご覧のとおり、これは完全に別のプロジェクトであり、複数のモデルを作成してトレーニングできます。左側のインスペクターにプロジェクト名（Project）が表示され、クリックするとプロジェクトに関する情報が表示され、編集できるようになります。さらに下の「モデルソース」には、プロジェクトで使用されるすべてのモデルが含まれています。次に、メインツールウィンドウの[データ入力]セクションを見てみましょう。

トレーニングデータ-ここでは、トレーニングモデルの画像を追加する必要があります。検証データ-モデルのトレーニングをテストするために画像のセットを追加できます（私の場合、私はAutoを残しました）。テストデータ-ここでは、モデルのトレーニングに関与しなかったモデルをテストするための画像を追加する必要があります。

[パラメータ]セクションでは、モデルのトレーニング中に単一の画像に対して実行される反復の最大数を指定できます。また、拡張セクションでは、画像に重ねる効果を選択できます。

Create MLについて理解したので、最初のモデルのトレーニングを開始できます。トレーニングデータフォルダー（名前は任意）をトレーニングデータタブにドラッグするだけです。次に、[トレーニング]をクリックします。学習プロセスが始まります。



そのため、トレーニングに約27秒かかり、モデルの準備はほぼ完了しました。トレーニング後、モデルがトレーニングデータセットから画像をどの程度適切に分類するかを評価できます。モデルはこれらの画像でトレーニングされているため、よく分類されます。私の場合、モデルはトレーニングセットのすべての画像とテストセットの画像の97％を識別しました。テストの時間が来ました。



[テスト]タブに移動します。モデルをトレーニングするためのデータと同様に、Testing DataフォルダーをTesting Dataウィンドウにドラッグするだけです。実際、トレーニングを開始する前に、テスト用の画像を含むフォルダを設定することができました。ただし、プロセスの線形性については、別の方法で行います。 [モデルのテスト]をクリックします。テスト結果によると、モデルは98％で対処しました。

簡単ですよね？次に、分類要求を作成、処理、および実行する必要があります。

private func makeClassifierRequest(for model: VNCoreMLModel, ciImage: CIImage) {
    let request = VNCoreMLRequest(model: model) { [weak self] request, error in
      self?.handleClassifierResults(request.results)
    }
    
    let handler = VNImageRequestHandler(ciImage: ciImage)
    DispatchQueue.global(qos: .userInteractive).async {
      do {
        try handler.perform([request])
      } catch {
        self.onDidUpdateState?(.requestFailed)
      }
    }
  }

結果を処理するだけです。

  private func handleClassifierResults(_ results: [Any]?) {
    guard let results = results as? [VNClassificationObservation],
      let firstResult = results.first else {
      onDidUpdateState?(.requestFailed)
      return
    }
    
    DispatchQueue.main.async { [weak self] in
      let confidence = (firstResult.confidence * 100).rounded()
      let resultModel = ClassifierResultModel(identifier: firstResult.identifier, confidence: Int(confidence))
      self?.onDidUpdateState?(.receiveResult(resultModel: resultModel))
    }
  }
