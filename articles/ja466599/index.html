ロシアのサウンドモデルであるPocketsphinxのインストール


*Habré に関する記事があり、サウンドはサウンド出力リダイレクトを介してオンラインでGoogle翻訳に送られます。そして、これがすべてこのケースでうまくいったなら、これはこの投稿を終えることができます。

Pocketsphinx自体をWindows（およびLinuxにも）にインストールすることはそれほど複雑ではありません。ダウンロードしてインストールしてください。

デフォルトでは、pocketsphinxには英語、音響モデル、辞書が付属しているため、ロシア語についても同様に必要です。

ロシア語版をダウンロード- リンク。

ファイル構造でロシア語モデルを解凍した後、次のpythonコードを使用して、テスト.wavファイルdecode-text.wavを試すことができます。

import os
from pocketsphinx import AudioFile, get_model_path, get_data_path
#from pocketsphinx import Pocketsphinx

model_path = get_model_path()
data_path = get_data_path()

config = {
    'verbose': False,
    'audio_file': os.path.join(data_path, 'C://python3//decoder-test.wav'),
    'buffer_size': 2048,
    'no_search': False,
    'full_utt': False,
    'hmm': os.path.join(model_path, 'C://python3//zero_ru_cont_8k_v3//zero_ru.cd_cont_4000'),
    'lm': os.path.join(model_path, 'C://python3//zero_ru_cont_8k_v3//ru.lm'),
    'dict': os.path.join(model_path, 'C://python3//zero_ru_cont_8k_v3//ru.dic')    
    
}

audio = AudioFile(**config)
for phrase in audio:
    print(phrase)

オーディオファイルの内容は、「Ilya Ilf Evgeny Petrov Golden Calf」という行に表示されます。
（私の状況のように）表示されない場合は、decoder-test.wavを別のオーディオ形式に変換する必要があります。

これにはffmpegが必要です。

辞書


独自の辞書が必要になります。この場合、ロシア語のアルファベットのすべての文字（b、s、bを除く）と数字で構成されます。

すべての文字は、UTF-8エンコーディングの各行に1つずつ、プレーンテキストファイルに配置する必要があります。

次に、辞書を変換する必要があります。
perlをインストールする必要があります（コンバーターが機能するために必要です）。
次に、ru4sphinxを変換するためのプロジェクトをダウンロードします。

以前に作成した辞書を変換します。

 C:\ru4sphinx-master\ru4sphinx-master\text2dict> perl dict2transcript.pl my_dictionary.txt my_dictionary_out.txt.

出力で、作業用の辞書を取得します



。辞書の拡張子を.txtから.dic形式に名前変更する必要があり、ファイル自体をアクセス可能な場所に配置する必要があります。

それははるかに速く動作します-1秒未満、さらにすべての文字が定義されています。
しかし、ここで少し注意が必要です。

すべての文字が正しく認識されるわけではありません。正しい文字の代わりに別の文字が表示される場合は、以前に作成した.dic辞書を、文字の対応を照合することによって手動で修正できます。

たとえば、文字aの代わりにeを表示します。辞書e：から行を取り、それ
r y
を転送（古いものを削除）して文字を変更
r y
する必要があります。ただし、文字「a」はすでに辞書にあるので、文字に「（2）」（または3.4）を追加する必要があります。通常、序数辞書に既にあるサウンドの数に応じて、数：
a(2) r y

辞書を再度変換する必要はありません。そのような単純な方法で、ほとんどすべての文字の音素を「拾う」ことができます。
