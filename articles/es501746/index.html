Quarkus: actualizaciones de aplicaciones utilizando el ejemplo helloworld de JBoss EAP Quickstart (continuación)

Hola a todos, esta es la quinta publicación de nuestra serie sobre Quarkus. (Por cierto, regístrese y vaya a nuestro seminario web " Este es el marco Java nativo de Quarkus - Kubernetes ", que se llevará a cabo el 27 de mayo. Le mostraremos cómo comenzar desde cero o transferir soluciones ya preparadas).



En una publicación anterior, examinamos la modernización de aplicaciones Java utilizando Quarkus compatible tecnologías (CDI y Servlet 3) que utilizan el programa helloworld del repositorio de inicio rápido de Red Hat JBoss Enterprise Application Platform (JBoss EAP) . Hoy continuaremos con el tema de la modernización y discutiremos el tema del consumo de memoria.

La medición del rendimiento es la base fundamental de casi cualquier actualización, y los informes de uso de memoria son una parte importante del proceso de análisis de rendimiento. Hoy veremos las herramientas de medición relevantes con las que puede cuantificar las mejoras realizadas como resultado de la modernización de las aplicaciones Java.

Para obtener más información sobre la medición del uso de la memoria, consulte el manual de Quarkus titulado Medición del rendimiento: ¿cómo medimos el uso de la memoria?

Y a continuación solo mostramos cómo comparar los datos de uso de memoria con tres tipos diferentes de aplicaciones (JBoss EAP, paquete JAR y ejecutable), recolectando estos datos en Linux usando las utilidades pmap y ps.

Jboss eap


Lanzamos una instancia de la aplicación JBoss EAP (consulte la sección "Implementación de helloworld" en la publicación anterior ) y luego miramos su identificador de proceso PID (en nuestro ejemplo, es 7268) usando el siguiente comando:

$ pgrep -lf jboss
7268 java

Nota. La opción –a le permite extraer la línea de comando completa (es decir: $ pgrep -af jboss).

Ahora use PID 7268 en los comandos ps y pmap.

Me gusta esto:

Paquete de tarro


Inicie la aplicación JAR (consulte la sección "Iniciar helloworld empaquetado en el JAR" en la publicación anterior ):

$ java -jar ./target/helloworld-<version>-runner.jar

Nuevamente miramos el PID usando el comando pgrep (esta vez usando la opción -a descrita anteriormente):

$ pgrep -af helloworld
6408 java -jar ./target/helloworld-<version>-runner.jar

Archivo ejecutable


Lanzamos el nativo (consulte la sección "Lanzamos el archivo ejecutable helloworld nativo" en la publicación anterior ):

$ ./target/helloworld-<version>-runner

De nuevo, mira su PID:

$ pgrep -af helloworld
6948 ./target/helloworld-<version>-runner

Y luego use el identificador de proceso resultante (6948) en los comandos ps y pmap.

Comparar el consumo de memoria


Entonces, obtuvimos los siguientes números sobre el uso de la memoria:

  • JBoss EAP - 650 MB.
  • Paquete JAR - 130 MB.
  • El archivo ejecutable es de 20 MB.

Obviamente, el ejecutable ocupa mucho menos memoria.
