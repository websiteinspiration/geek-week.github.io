こんにちはHabr。

Software Defined Radioに関する以前の記事で、GNU Radioを使用してRDSをデコードする方法を尋ねられました。 RDSデコーダーはゼロから作成するのは簡単ではありませんが、幸いなことに、GNUラジオに既に組み込まれているため、RTL-SDRレシーバーだけで、コードを1行も書かなくても簡単に動作を確認できます。



それがどのように機能するか、カットの下で続けました。

私はGNUラジオとその設置について繰り返すことはしません、それがすでに記載されている以前に。つまり、Windows用のディストリビューションはここにあります。 RDSに移りましょう。

誰かが突然知らなくなった場合、RDS（Radio Data System）は、追加のデジタル情報（時間、為替レート、天候など）を送信するために従来のFM放送で使用されるプロトコルです。伝送は、57KHz（パイロットトーンの3次高調波）の周波数で1187.5bpsの速度で行われ、BPSK位相エンコーディング（バイナリ位相シフトキーイング）を使用した位相最小化が使用されます。ビットレベルのRDSは以前からハブで検討されていましたが、まあ、GNU Radioに移ります。

GNU Radioのデコーダーは約10年前に作られました、しかしそのバージョンを見るのは無意味であり、多くのブロックの名前が変更されており、そこで与えられた例はもはや機能しません。更新されたフォークはここから取得できます。

ちなみに、Windowsのソースからgr-rdsを収集することはできませんでした-cmostはインストールされていますが、Boostがないとエラーを出しますちなみに、長年のcmakeとboostの存在において、cmakeはWindowsでパスを見つける方法を学習しなかったことは明らかではありません。たとえば、ディスク上のフォルダーを見つけることはまったく科学的なものではありません（解決策を知っている人がいる場合はコメントに書き込んでください。StackOverflowで判断すると、問題は何年も存在していますそして、すべては気にしない）。しかし、RDSデコーダーはすでにGNU Radioに追加されているため、必要ないことが判明しました。githubのプロジェクトでは、appsフォルダーにある例のみが必要です。

例で囲まれたブロック図は非常に巨大であり、それを開くとエラーが発生します（ブロックは赤でマークされています）。



ただし、実際にはすべてが単純です。回路の2/3はFM用のステレオプレーヤーで、L + RチャネルとLRチャネルが元の信号から抽出され、処理されてサウンドカードに送られます。これは今のところ私たちには関係がないので、これらのブロックは削除できます（なぜかそのままでは機能しませんでしたが、理解するのが面倒だったためです）。エラーはグリッド位置パラメーターが原因で発生します。これは明らかにWindowsバージョンではサポートされていませんが、問題なく削除でき、機能には影響しません。

「不要なもの」をすべて削除すると、RDSデコーダー回路は次のよう



になります。何があるか見てみましょう。

元の信号はRTL-SDRソースから発信され、周波​​数はWX GUIスライダータイプのfreqパラメーターによって設定されます（そうです、GNU Radioで独自のUIを作成でき、基本的なコントロールがあります）。中心のゼロ周波数でのピークを回避するために、freq_offsetパラメーターが使用され、Frequency Xlatingブロックはこの値で周波数をシフトします。 WBFM受信ブロックは、その名前が示すように、FM復調を実行した後、周波数を再度シフトして57KHz RDS自体を強調表示します。 Root Cosined Filterブロックは狭い周波数を選択しますが、パラメーター2のMPSKデコーダーはBPSKデコードを実行します（RDS自体は2つの状態の位相変調を使用して送信されます）。 RDSは差分エンコーディングを使用しますしたがって、対応する差分デコーダーが呼び出され、最後に、完成したバイナリストリームがRDSデコーダーブロックに供給されます（そのソースはgithubで表示できます）。デコーダの次に重要な部分はRDSパーサーです。RDSには非常に多くのパケットタイプがあり、パーサーはそれらを復号化するためのすべての作業を行います。

実際にはそれだけです。 KDPVのデコーダーの結果とスクリーンショット。







誰かがこのプログラムを非UIモードで使用する必要がある場合は、FR Tapブロックを使用できます。詳細についてはこちらをご覧ください。個人的には試していません。 RDSのより低いビットレベルが興味深い場合、私は以前にそれを検討しました。
