通过Helm v2分er器打破Kubernetes集群







Helm是Kubernetes（类似于apt-get



Ubuntu ）的软件包管理器。在本说明中，我们将看到带有默认分till服务的头盔（v2）的早期版本，通过该服务，我们可以访问群集。







准备集群，为此，我们运行命令：







kubectl run --rm --restart=Never -it --image=madhuakula/k8s-goat-helm-tiller -- bash








示范



  • 如果未进行其他任何配置，则helm v2将启动分till服务，该服务具有具有完全群集管理员权限的RBAC。
  • 安装后在命名空间中kube-system



    出现tiller-deploy



    并打开端口44134，绑定到0.0.0.0。可以使用telnet进行验证。


$ telnet tiller-deploy.kube-system 44134








  • 现在，您可以连接到分er服务。与分er服务进行通讯时，我们将使用helm二进制文件进行操作：


$ helm --host tiller-deploy.kube-system:44134 version
