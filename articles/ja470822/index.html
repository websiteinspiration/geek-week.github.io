企業内の温度監視


工場で温度を表示および制御するための「何か」を考案するという課題が浮上しました。PLC 160コントローラーが既にインストールされており、RS-485インターフェース（Wikipedia）を介して温度センサーが接続されています。

コントローラーとセンサーは私の前に設置されました。

アイデアはこのようにして生まれました。MySQLデータベースと連携してWEBサイトを作成し、そこに気温と事故に関する情報を保存することです。

将来、次のことが明らかになりました。

最小値と最大値は、事故を制御するには不十分です。

クリティカルな最大値とクリティカルな最小値、および温度が正常に戻る時間を追加しました。

  2. 温度が臨界最小値または臨界最大値を超えた場合、これはすぐに重大な事故です。

function KillProcess(ExeName: string): LongBool;
var
  B: BOOL;
  ProcList: THandle;
  PE: TProcessEntry32;
begin
  Result := False;
  ProcList := CreateToolHelp32Snapshot(TH32CS_SNAPPROCESS, 0);
  PE.dwSize := SizeOf(PE);
  B := Process32First(ProcList, PE);
  while B do begin
    if (UpperCase(PE.szExeFile) = UpperCase(ExtractFileName(ExeName))) then
      Result := TerminateProcess(OpenProcess($0001, False, PE.th32ProcessID), 0);
    B:= Process32Next(ProcList, PE);
  end;
  CloseHandle(ProcList);
end;

procedure TForm1.RestartThisApp;
begin
  ShellExecute(Handle, nil, PChar(Application.ExeName), nil, nil, SW_SHOWNORMAL);
  Application.Terminate; // or, if this is the main form, simply Close;
end;
