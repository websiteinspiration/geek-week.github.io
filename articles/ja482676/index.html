Structタイプのデータの効果的なソート

来た誰もエリクシール / Erlangの方法を比較演算子に関するいくつかの期待を持っている可能性が高い他の言語から、などが動作するはずです>、一つはそれを期待するなど、（これはそう確かにあります）。原則として、比較は正常に機能していると言えます。しかしいつもではない。<==1 < 2


ではエリクシール / アーラン、あなたは何を比較することができます。一般的に。同じタイプの2つのオペランドの場合、上記の例のように結果は期待できませんが、異なるタイプの2つのオペランドを比較すると、予想外の結果が生じます。タイプ自体が「比較のために順序付けられている」ためです。この方法では：


number < atom < reference < function < port < pid < tuple < map < list < bitstring

これは突然、完全に正当な比較に42 < nil戻りtrueます。


さらに、maps（そして結果として、structs実際には内部で同じですmaps）、フィールドに従ってアルファベット順に比較されます。言い換えると、日付（Elixir構造に実装されているDate）は、フィールドの値day、次にmonth、そしての順で順番に比較されますが、yearこれはおそらく私たちが望んでいることとは異なります。


から始めてv1.10.0、構造が関数をエクスポートする場合、Elixirは標準による便利なソートを提供Enum.sort/2しますcompare/2：


defmodule User do
  defstruct [:name]
  def compare(%User{name: n1}, %User{name: n2}) when n1 < n2,
    do: :lt
  def compare(%User{name: n1}, %User{name: n2}) when n1 > n2,
    do: :gt
  def compare(%User{}, %User{}), do: :eq
end

users = [
  %User{name: "john"},
  %User{name: "joe"},
  %User{name: "jane"}
]

Enum.sort(users, {:asc, User})
#⇒ [%User{name: "jane"},
#   %User{name: "joe"},
#   %User{name: "john"}]

構造体を定義し、関数をエクスポートするモジュールは、そのまま、またはタプルcompare/2として、2番目のパラメーターEnum.sort/2として呼び出しに渡すことができます{:asc | :desc, StructModule}。モジュールを転送Enum.sort/2できるようになりました。以下は、この特定の機能を実装するモジュールからの抜粋です。 compare/2Enum.sort/2Enum


...
defp to_sort_fun(module) when is_atom(module),
  do: &(module.compare(&1, &2) != :gt)
defp to_sort_fun({:asc, module}) when is_atom(module),
  do: &(module.compare(&1, &2) != :gt)
defp to_sort_fun({:desc, module}) when is_atom(module),
  do: &(module.compare(&1, &2) != :lt)

これにより、上記の構造の例のDate.compare/2ように、ユーザー定義の構造（compare/2もちろん、エクスポートする場合）と同様に、以下の例の日付（既存のものに長い間感謝している）で日付を正しくソートできますUser。


dates = [~D[2019-01-01], ~D[2020-03-02], ~D[2019-06-06]]

Enum.sort(dates) # wrong
#⇒ [~D[2019-01-01], ~D[2020-03-02], ~D[2019-06-06]]

Enum.sort(dates, {:asc, Date}) # correct
#⇒ [~D[2019-01-01], ~D[2019-06-06], ~D[2020-03-02]]
