Bot em nenhuma ferramenta de código. Detalhes da implementação

Continuamos a experimentar o paradigma sem código (e código parcialmente baixo).

Eles montaram um chatbot que ajuda a ler grandes arquivos de artigos.

No artigo, mostraremos quais ferramentas eles usaram e quais limitações eles encontraram.



O material do artigo e o rascunho são coletados juntamente com roman_nebel.

Aviso Legal. Sim, também é mais fácil escrever código limpo. Mas a essência do exercício nocódico é coletar serviços úteis de ferramentas prontas. Começamos muito tempo, continuamos, gostamos. Contaremos tudo gradualmente.

O público de Habr, aparentemente, está mais interessado em detalhes técnicos do que em detalhes do produto. Se fizer sentido falar sobre a camada de trabalho do produto em projetos sem código, sem entrar nos detalhes da implementação, informe-me nos comentários.

O que você queria fazer


Eles queriam ajudar as pessoas (e elas mesmas):

  • Acostumado a ler em inglês;
  • deixe para o almoço: "Aqui está um artigo, leia à mesa."

Eles decidiram fazer esse microproduto através do bot do Telegram:

  1. Uma vez por dia envia um link para um artigo do arquivo. Para testar, arquivei um arquivo de artigos sobre o design do ReadingDesign . Ainda será.
  2. Ele pergunta ao leitor se o artigo é ou não adequado. Adicionamos os resultados, salvamos os dados e, em seguida, podemos encontrar os artigos mais interessantes e avaliar a atividade do usuário.

Você pode sentir o bot : @bracho_read_bot
Há erros . Idéias para recursos também. E há um plano para um novo lançamento.
Em resumo, tudo está lá, mas muito - não um pouco, então jogue seus pensamentos.

O que usado

Platforma . Um dos muitos designers de bate-papo (escolhidos). Não é perfeito, mas pelo menos de alguma forma faz o que você precisa. Sobre a seleção abaixo.

Integromat . Ele transfere dados de um lado para outro de acordo com as regras e condições. A seleção padrão para nenhum projeto de código.

Airtable . Banco de dados com uma interface avançada e boa integração com outras ferramentas. Essa também é a opção padrão (e a usamos há muito tempo na automação de nossos processos de negócios, mas isso é outra questão).

Peças de ferramenta


Pai bot


Primeiro, precisamos do bot em si, ou melhor, da "conta" no Telegram. A maneira mais fácil é encontrar o Pai dos Bots ( @BotFather ) no Telegram e registrar um novo bot nele. Uma descrição passo a passo é o google para todos os gostos, em poucas palavras: criamos o nome do bot, seu nome de usuário e é isso. Após o sucesso, você receberá um token (uma chave para identificar e conectar o bot ao mundo exterior), que precisávamos neste estágio.

Platforma


A rede possui vários serviços para criar bots simples para redes sociais. Mas todos eles sofrem ferimentos: foram projetados para funis de vendas. E, portanto, simples e burro.

O padrão mais comum: um cliente veio, cumprimentou-o, perguntou o que era necessário, lambeu e começou a enviar spam obsessivamente. Tudo isso é feito com a ajuda de garfos de texto simples e integração de dados (por exemplo, com Bitrix24). Sem interatividade.

Obviamente, existem soluções como o DialogFlow no Google Cloud, mas o que entender é mais fácil escrever tudo com canetas. Portanto, tomamos a opção mais simples.

Tomou Chatforma. De toda a diversidade, ele tem pelo menos uma API e suporte Webhook. E, nesse caso, podemos confiar à plataforma chatbot a função de enviar diretamente e configurar a lógica em outro lugar.

Sim, fizemos o projeto anterior no code no AIMyLogic . Infelizmente, a integração com serviços externos é definida como zero com um plus. Então, infelizmente, não desta vez.

Airtable


Um local para armazenar dados. No nosso caso, este é um arquivo de artigos que o bot envia, uma lista de usuários (somos muito curiosos e queremos saber seus nomes) e uma classificação com pontos para artigos lidos.

Sim, também era possível no Planilhas Google (já as concluímos e estamos executando um projeto comercial sem código, de repente este é um livro de tarefas de design ). E no SQL era possível. E há muito mais.

Há duas razões para escolher Airtable : é agradável no rosto e se integra maravilhosamente com tudo e com tudo. Você obtém o token da API diretamente no aplicativo e direciona os dados nas duas direções que você quiser e onde quiser. Fácil e agradável. E queríamos que fosse fácil e agradável, pelo menos em algum lugar e em alguma coisa.

Integromat


E, finalmente, cérebro. É difícil nomear as vantagens sobre o mesmo Zapier , há sabor puro. Já utilizamos o Integromat em projetos anteriores, é de alguma forma mais familiar. A propósito, também coletamos uma lista de artigos do arquivo usando-o.

Um pouco de comércio duro


Todos os instrumentos têm diferentes graus de carga. Micropagamentos dentro de 1000 rublos por mês não nos pairam. No final, para a nossa mega corporação , é um dinheiro acessível. Mas sim, se para você ferramentas gratuitas é uma questão de princípio, é melhor não jogar brinquedos sem código, é preciso desatar um centavo bonito para quase tudo.

Como fizemos


Configurações básicas


Problema número 0 - no Chatforma para obter um novo bot, de modo que era. Registramos na plataforma, estamos contentes por termos apenas 14 dias gratuitos. Vamos para a aba bots e encontramos o precioso botão "Adicionar bot". Nós o nomeamos (o nome é apenas para Chatforma, o bot já foi nomeado) e selecionamos a rede social (Telegram).

Depois disso, vamos imediatamente para o novo bot e, na seção de configurações, prescrevemos o mesmo token que o Pai dos Bots nos deu, nada funcionará sem ele.


A configuração do token está oculta, você ainda precisa procurar

A primeira coisa que um botânico que se preze deve fazer é ensinar o bot a dizer olá. Tudo é simples: vamos ao Designer e vemos duas mensagens preparadas: início e padrão. Qual é a diferença: primeiro - esta mensagem será exibida se você acabou de lançar o bot (clique em "Iniciar"). O segundo é um esboço universal para qualquer mensagem que o bot não entenda.


E aí você pode adicionar um teclado embutido, mas isso é bem mimado

Coletamos usuários


Para levar em conta as estatísticas e distribuir links exclusivos, cada um de nós precisará de uma lista de usuários que lançaram o bot. Está na seção "Usuários", mas não possui benefícios práticos, exceto pelo fato de o ego ser excitado pelo número de signatários.


No cabeçalho da tabela, você pode determinar facilmente sua utilidade

: usaremos Integromat, Airtable e solicitação de API. No Integromat, extraímos a lista de usuários com essa solicitação e a escrevemos na tabela desejada em Airtable. Para não produzir duplicatas, antes da gravação, verificamos a presença do usuário na tabela por ID. Se você já possui um, filtramos e anotamos o restante.


Fetiche separado em Integromat - tentando alinhar tudo lá

Configurar um boletim informativo


Mais interessante. Todos os dias, precisamos gerar um boletim informativo para todos os participantes, inserindo um link aleatório da Airtable lá. Melhor ainda, um link aleatório para todos. Melhor ainda, escreva algo na frente dela. Algo pessoal e não repetindo todos os dias. Vamos para a seção "Boletins" na esperança de implementar nosso plano ... Tudo está muito ruim.


Vamos agora explicar por que razão

é essa. O Chatforma pode criar dois tipos de boletins: links regulares e automáticos. Os primeiros são adequados se você precisar enviar um determinado texto, imagem, pesquisa e assim por diante para todos (ou alguns) participantes de uma só vez. O segundo permite que você configure o boletim de acordo com a programação e os dias da semana, mas a funcionalidade de envio é limitada apenas por texto e mídia; as pesquisas não podem ser enviadas assim.


Apenas enviando


avtorassylki

Voltar ao boletim. O que a plataforma oferece não nos convém. Não podemos enviar correspondências personalizadas e inserir dinamicamente nenhuma informação lá. Apenas pré-preparado. E não tenho vontade de fazê-lo com canetas. Quando o bot tem dois usuários, isso não é nada, mas se já existem três ... Portanto, resolvemos o problema usando o Integromat.

Criamos um cenário com a seguinte lógica: Obtemos uma lista de artigos da Airtable - A partir daí, pegamos a lista de usuários que inserimos anteriormente - Cada usuário passa por uma iteração na qual um número aleatório é gerado e um artigo com esse número de linha é enviado por uma solicitação POST ao Chatforma. Para fazer isso, tive que entrar na documentação da API e encontrar o método necessário lá.

Para impedir que o usuário receba o mesmo link duas vezes, carregue o banco de dados com os links já enviados, classifique-o por usuário e verifique a conformidade dos links. Se houvesse esse link, retornamos à etapa anterior e repetimos novamente; se não existia, enviamos para o usuário e o inserimos na tabela com os enviados.

Também precisamos simular uma mensagem interativa antes do link. Adicionamos opções de mensagem ao banco de dados e formamos o final usando o nome de usuário no telegrama, uma frase aleatória e um link aleatório.


Uma variação no tópico da lógica de negócios, e

é o que acontece depois no lado do Chatforma. Quando a Integromat enviou a mensagem, o Chatforma a recebe e forma uma única newsletter para um usuário específico. No texto, tudo é o mesmo que a Integromat formou. Parece funcionar.

Um problema foi descoberto imediatamente: não apenas houve um atraso no recebimento de dados externos, mas também o Chatforma passou muito tempo (até 15 minutos) em correspondências já formadas. Acontece que um boletim informativo de dez horas pode chegar às 10h01 e 10h16. O suporte técnico estuda esse fenômeno há 4 dias - e sem muito sucesso.


Mas o suporte é responsivo e é bom

Votação

Todo mundo pegou o link, alguém até leu o conteúdo. Agora você precisa avaliar este artigo. A API Chatforma possui duas limitações que não nos dão espaço para criatividade: só podemos enviar respostas recebidas de serviços de terceiros de uma entidade como "Enquete" para serviços de terceiros. Pela API, podemos retornar apenas texto, podemos criar a pesquisa de que precisamos apenas dentro do Chatforma, o que significa que podemos usar apenas os tipos de correspondência descritos anteriormente.

Como resultado, eles fizeram o seguinte: preencheram pesquisas sobre a qualidade do artigo para todos os dias para todos os assinantes e o definiram como 11:00. Depois que o usuário vota, o resultado via webhook vai para Integromat e, com base na resposta do último artigo enviado, os pontos são colocados para o usuário desejado. O cálculo é o seguinte: para "Disparar" +1 ponto no votante e +1 no artigo, para "Não muito" -1 ponto no artigo e +1 usuário. Para "eu não li" - nada. Eles também planejavam enviar ao usuário uma mensagem de devolução após a votação, mas abandonaram essa ideia. Porque a velocidade de entrega está no espírito do Russian Post nos seus melhores anos.


O PE não interferirá nesta votação

Total do instrumento


Airtable


Sem problemas para integrar, gravar e recuperar dados.

Integromat


O serviço faz um bom trabalho com operações lineares. Se a lógica if / else e a comparação de dados aparecerem, o esquema é bastante confuso. Mas funciona.
É necessário otimizar muito os processos, pois todas as tarifas são limitadas pelo número de itens concluídos.

Platforma


O serviço é bruto, há muitas deficiências, eles estão com raiva. Ao mesmo tempo, o suporte técnico é muito bem organizado, é rápido e competente o suficiente. Um grande número de instruções e webinars sobre funcionalidade são relevantes.

Falhas encontradas.

  1. Restrição dos tipos de conteúdo, dependendo do tipo de distribuição.
  2. O conteúdo de um boletim informativo da API é limitado apenas ao texto.
  3. O Webhook pode enviar apenas resultados de pesquisas, não ações do usuário.
  4. Grande atraso no envio de boletins.
  5. Erros com um atraso na realização de alterações após o salvamento.
  6. O log não armazena mensagens enviadas pelo usuário, apenas respostas a pesquisas.
  7. Pagar pelo serviço é uma missão completa e você realmente deseja fazer isso.
