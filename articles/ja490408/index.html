FreeBSD：GNU / Linuxよりもはるかに優れている

FreeBSDとGNU / Linuxシステムを比較してください。これらはすべて主観的な意見です！これの多くは単なる私の経験です。私がコンピューターに費やしている時間のほぼ100％とUnixシステムのみで、FreeBSDやGNU / Linuxとほぼ同じで、20年以上になります。

使用上のNetBSD、OpenBSDの、DragonflyBSD：FreeBSDの利点のほとんどは、他の人気のある無料のBSDシステムに適用されます。これらのシステムの何かがより良くなるでしょう、多くのことがFreeBSDで初めて現れませんでしたが、それに移行しました。 FreeBSDは、NetBSD（さまざまなドライバーとサブシステム）、OpenBSD（ネットワークテクノロジー）、DragonflyBSD（マルチプロセッサープラットフォームでの最高のスケーラビリティ）、Solaris（DTrace、Jails、ZFSのアイデア）から多くを継承し、すべての最高のものを組み合わせています。

つまり、FreeBSDは、高品質、信頼性、使いやすさ、使いやすさを意味します。GNU / Linuxは動物園、疎結合コードのガベージダンプ、最後まで完成させることはほとんどできない、ドキュメントの欠如、カオス、バザーです。

BSDは、全体として開発された完全な完全なOSです。Zstd圧縮アルゴリズムのサポートを追加する場合、これはコンプレッサーだけでなく、syslogローテーションやその他の場所にも表示されます。統計を発行するコマンドがある場合、それらはすべてlibxoなどのライブラリを使用して、さまざまな出力オプションを統合します。すべてのチームは1つの櫛の下に作成されます。
 GNUは、その名前、著作権者、ドキュメントの作成およびアセンブリの構成に関するヒントで結ばれた、単なるソフトウェアのセットです。 Linuxは、HurdまたはFreeBSDとともに、カーネルオプションの1つです。数百（数千？）のディストリビューションがあり、これらすべてを組み合わせて、完全に異なる、異なる要件、パッケージマネージャー、アプローチ、品質管理などを備えています。ディストリビューションがあり、GNUソフトウェアはほとんどありません。 Linuxを使用していないディストリビューションがあります。

ソフトウェアでのドキュメント、構成、情報出力の統合は実際にはありません。どこでもどこでも、この小さなプログラム/ユーティリティが一人の人間によって書かれていること、そしてこれが別の人間によって書かれていることがはっきりとはっきりとわかります。どこでも、どこでも、すべてに対して異なるアプローチがあります。1つはそう思う、もう1つはそう思う。

基本的に、すべてを単一の統一された外観にすることへの懸念は、ソフトウェアを編集する必要があるディストリビューションメンテナーの肩にかかっています。提供されるソフトウェアの利便性と品質（きちんとレイアウトされたドキュメント、ログローテーションシステムでのデーモンのアカウンティング、ロギングシステム、定期的なタスクなど）は、メンテナーによって異なります。

実際には、最新のUbuntuディストリビューションのいずれかを現在使用している場合、それを最初のハードドライブにインストールできないという事実ではなく、GRUBブートローダーをインストールできるという事実ではなく、IPv6を構成できるという事実ではなく、最初のパッケージ更新後の事実ではありませんインストール中に、すべてが起動します。最近、UbuntuのネットワークインターフェイスでMTUを適切に構成できないという問題に直面しました（真剣に！？）。私が実際に経験したこのすべてとそのような些細な行動は定期的に失敗し、機能しません。

GNU / Linuxディストリビューションを使用すると、GNOMEやNetworkManagerなどと一緒に数千のソフトウェアコンポーネントを入手でき、それらをオフにしてアンインストールできます。また、IPv6を実行できるコンパイラやnetcatユーティリティさえも含まないディストリビューションを入手できます（そして、telnetもsocatも）、OSネットワークの操作性をチェックするのにも適したものはほとんどありません。
BSDシステムソフトウェアの品質ははるかに優れています。開発者がそのような機能が産業利用の準備ができていると言うなら、これはそうです。長年にわたり、これは実験的なものであり、開発者の時間と信頼性を確認した後にのみこのマークが削除されるという大きな声で、OSに多くの機能を含めることができます。

GNU / Linuxには本当にメリットがないのですか？いくつかの重要なGNUソフトウェアはBSDオプションよりもはるかに高速であるとすぐに言えます：sedおよびgrep私はGNU-shnyを入れました。これは多くの場合、コードやあらゆる種類のトリックを複雑にするという犠牲を払っていますが、パフォーマンスを完全に引き下げることはできません。ちなみに、OpenBSDは自慢できないものです-それは複数のプロセッサでうまくスケーリングしません。
