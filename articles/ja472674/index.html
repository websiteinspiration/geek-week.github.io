Pythonプロジェクトの環境変数


Webアプリケーションまたはボットを開発する場合、私たちは秘密情報、さまざまなトークン、およびパスワード（APIキー、Webフォームの秘密）を扱うことがよくあります。この情報をハードコードすることは、公的に利用可能なバージョン管理システムに保存することはもちろんのこと、非常に悪い考えです。

設定ファイル


この問題を解決する最も簡単な方法は、すべての機密情報を含む個別の構成ファイルを作成し、それをに追加すること.gitignoreです。このアプローチの欠点は、gitでは構成ファイルテンプレートを保持する必要があり、定期的に更新することを忘れないことです。


#  .
from config import API_KEY
app = Flask(__name__)
app.config['API_KEY'] = API_KEY

環境変数


より高度なアプローチは、環境変数を使用することです。環境変数は、プログラムを実行して使用できるテキスト情報を含む名前付き変数です。たとえば、フラスコアプリケーションを起動するには、まず環境変数にFLASK_APPアプリケーションの名前を指定する必要があります。


$ export FLASK_APP=hello.py
$ flask run
 * Running on http://127.0.0.1:5000/

環境変数を使用すると、さまざまなアプリケーションパラメータと秘密鍵を取得できます。


import os
app.config['API_KEY'] = os.environ.get('API_KEY')

サンプルファイル設定ファイル .envrc

これは.envrc、フラスコアプリケーションを開発するために構成されたファイルのように見えます。


export VIRTUAL_ENV=venv
layout python-venv
export FLASK_APP=app.py
export FLASK_DEBUG=1

これにより、プロジェクトフォルダーに入ったときに、仮想環境を自動的にアクティブにし、環境変数を読み込むことができます。
