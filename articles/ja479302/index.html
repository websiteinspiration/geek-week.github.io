Unityシェーダーグラフの基本


シェーダーは、GPUの命令を含む小さなプログラムです。特定の素材の画面の色を計算する方法について説明します。

Unityには標準シェーダーがありますが、標準シェーダーではできないエフェクトを実装する必要がある場合があります。

以前は、これにはCgやHLSLなどの特定のシェーダー言語の知識が必要であり、それらのアプローチは通常のゲームプレイスクリプトの作成とは少し異なりました。多くの人々にとって、シェーダーを書くことはゲーム開発の人気のない側面です。なぜなら、それは追加の学習曲線の開発を必要とするからです。

UnityはShader Graphを導入しました。これにより、ほとんどコードなしでシェーダーを簡単に作成できます。何よりも、シェーダーグラフを使用すると、視覚的なインタラクティブインターフェイスを操作できます。

このチュートリアルでは、最初のUnityシェーダーを作成します。

働き始める


このチュートリアルでは、Unityバージョン2019.1以降を使用します。こちらからUnityの新しいバージョンをダウンロードしてください。

注：このチュートリアルはシェーダーグラフの初心者を対象としていますが、Unityでの開発の基本を理解し、エンジンのインターフェースをマスターしていることを前提としています。Unityを初めて使用する場合は、優れたGetting Started In Unityチュートリアルをご覧ください。

こちら からチュートリアル資料をダウンロードしてください。次に、コンテンツを解凍し、UnityでIntro to Shader Graph Starterを開きます。プロジェクトウィンドウ

  • フォント：シーンで使用されるフォント。
  • マテリアル：シーンのマテリアル。
  • モデル：ゲームの形状と背景用の3Dメッシュ。
  • PostFX：シーンレンダリング用の後処理エフェクト。
  • プレハブ：さまざまな事前作成コンポーネント。
  • シーン：ゲームシーン。

シェーダーグラフのパイプラインパラメーターの確認


シェーダーグラフは、比較的新しいスクリプト可能なレンダーパイプライン、つまり高精細レンダーパイプラインまたは軽量レンダーパイプラインでのみ機能します。

シェーダーグラフを操作するための新しいプロジェクトを作成するときは、正しいテンプレートを選択してください。


サンプルプロジェクトは、Lightweight Render Pipelineで動作するようにすでに構成されています。まず、ウィンドウ►PackageManagerを選択して、Lightweight RPとShaderGraphがインストールされていることをPackageManagerで確認します。 次に、使用可能なバージョンの1つを選択し、必要に応じて[ 更新 ]ボタンをクリックします。通常、最良のオプションは最新の検証済みバージョンです。




パッケージを更新したら、[編集]►[プロジェクト設定]でパイプラインパラメーターが正しいかどうかを注意深く確認します。

[ グラフィックス ]タブで、[ スクリプト可能なレンダリングパイプライン設定]パラメーターをLWRP-HighQualityに設定する必要があります。このおかげで、プロジェクトはLightweight Render Pipelineの最も高いデフォルト設定を使用します。スクリプト可能なレンダーパイプライン

アセットの1つを使用していることを確認したら、このウィンドウを閉じます。

PBRグラフの作成


3Dメッシュを画面にレンダリングする場合、マテリアルとシェーダーは常に連携します。したがって、シェーダーの作成を始める前に、マテリアルが必要です。

まず、プロジェクトウィンドウの[ 作成 ]ボタンをクリックして、RW /マテリアルフォルダに新しいマテリアルを生成します。 [ 作成]► [ マテリアル]を選択し、名前をGlow_Matに変更します。

次に、RW / Shadersフォルダーで、[ 作成]►[シェーダー]►[PBRグラフ]を選択して、PBRグラフを作成します。これは、物理的に正確なレンダリング、つまりPBR をサポートするシェーダーグラフです。HighlightShaderGraphという

名前を付けます。


マテリアルGlow_Matは、LightweightRenderPipeline / Litと呼ばれる標準のLightweightRenderPipelineシェーダーをまだ使用しています。


新しく作成した新しいグラフシェーダーに変更します。Glow_Mat選択をクリックして、シェーダのドロップダウンメニューインスペクタの上部にあると選択シェーダグラフ►HighlightShaderGraphを。

Glow_Matは少し明るいグレーの色調になりますが、それ以外はかなり鈍いままになります。心配しないでください。すぐに修正されます。

次に、HighlightShaderGraphアセットをダブルクリックするか、インスペクタでOpen Shader Editorをクリックします。これにより、Shader Graphウィンドウが開きます。


このインターフェースの主要部分を確認してください。

  • メインワークスペースは、グラフ操作が保存される濃い灰色の領域です。ワークスペースを右クリックして、コンテキストメニューを開くことができます。
  • ノードは、グラフの一つの要素です。各ノードには、ポートに応じて、入力、出力、または操作があります。ノードは、エッジを使用して相互に接続されます。
  • マスターノードはグラフの最後の出力ノードです。この例では、PBRマスターノードとも呼ばれる、物理的に正確なレンダリングのオプションを使用します。標準のシェーダーで使用されているアルベド、ノーマル、エミッション、メタリックなどの特定のプロパティに気付く場合があります。
  • Blackboardでは、インスペクタのグラフの特定の部分にアクセスできます。これにより、ユーザーはグラフ自体を直接編集せずにパラメーターを構成できます。
  • メインプレビューは、シェーダーの現在の出力を球としてインタラクティブに表示します。

さまざまなノードを接続することで、UnityがコンパイルしてGPUに転送するシェーダーグラフを作成できます。

ハイライトスクリプトの作成


私たちはその姿が絶え間なく輝いてほしくありません。マウスの位置によってのみ、グローをオンにする必要があります。

マウスカーソルが形状の上にあるとき、Glow_Matマテリアルを有効にする必要があります。他の場合では、図は標準マテリアルWood_Matで表示されます。

まず、RW / Scriptsで新しいC＃スクリプトを作成し、それをHighlighterと呼びます。彼は、プログラムの実行中に2つの資料を切り替えるのに役立ちます。スクリプトのすべての行を次のコードに置き換えます。

using UnityEngine;

// 1
[RequireComponent(typeof(MeshRenderer))]
[RequireComponent(typeof(Collider))]
public class Highlighter : MonoBehaviour
{
    // 2
    // reference to MeshRenderer component
    private MeshRenderer meshRenderer;

    [SerializeField]
    private Material originalMaterial;

    [SerializeField]
    private Material highlightedMaterial;

    void Start()
    {
        // 3
        // cache a reference to the MeshRenderer
        meshRenderer = GetComponent<MeshRenderer>();

        // 4
        // use non-highlighted material by default
        EnableHighlight(false);
    }

    // toggle betweeen the original and highlighted materials
    public void EnableHighlight(bool onOff)
    {
        // 5
        if (meshRenderer != null && originalMaterial != null && 
            highlightedMaterial != null)
        {
            // 6
            meshRenderer.material = onOff ? highlightedMaterial : originalMaterial;
        }
    }
}

このスクリプトを解析してみましょう。

  1. スクリプトは、MeshRendererおよびコンポーネントを含むオブジェクトにのみ適用できますCollider。これは[RequireComponent]、スクリプトの先頭に属性を追加することによって制御されます。
  2. これらはMeshRenderer、originalMaterialおよびへのリンクhighlightedMaterialです。マテリアルには属性が付け[SerializeField]られ、インスペクターから割り当てることができます。
  3. このメソッドでStartは、を自動的MeshRendererに入力しGetComponentます。
  4. と呼ばれるEnableHighlight(false)。これにより、デフォルトでは、強調表示されていないコンテンツが表示されます。以下は、EnableHighlightレンダラーマテリアルを切り替えるパブリックメソッドです。onOffバックライトの状態を決定するブールパラメータを受け取ります。
  5. すべてのNullReferenceエラーの防止
  6. スペースを節約するために、三項演算子を使用します。

マウスイベントを追加する


このスクリプトはを含む図形に適用されるMeshColliderため、組み込みのメソッドOnMouseOverとを使用できますOnMouseExit。メソッドの後にEnableHighlight次のコードを追加します。

    private void OnMouseOver()
    {
        EnableHighlight(true);
    }

    private void OnMouseExit()
    {
        EnableHighlight(false);
    }

マウスが図の上にあるとき、それはを呼び出しますEnableHighlight(true)。同様に、マウスがColliderから離れると、が呼び出されますEnableHighlight(false)。

それで全部です！スクリプトを保存します。

フレネルエフェクトパワーを 使用して、ハローを増減できます。値を1.5に下げると、広い緑色に光ります。

このゲームの例では、4から5までの値が適切ですが、実際の値を試すことができます。

黒板のプロパティを追加


グロー効果の外観を変更する場合は、シェーダーグラフエディターウィンドウに戻り、これらの変更を行う必要があります。たとえば、フレネルエフェクトパワーを使用して、ハローバンドを拡大または縮小します。

さまざまな変更をテストする場合、これはあまり便利ではありません。幸いにも、シェーダーグラフの概念にはプロパティがあります（プロパティにはがあります）。

グラフの一部をインスペクターで公開して、インタラクティブに小さな変更を加えることができます。これは、Blackboardインターフェイスを使用して行われます。

シェーダーグラフに戻り、Blackboardディスプレイをオンにします。非表示になっている場合は、右上隅の[ 黒板 ]ボタンをクリックします。

ベーステクスチャと法線マッププロパティの追加


次に、ベーステクスチャと法線マップの両方を開いて、インスペクタからアクセスできるようにします。Blackboardの右上隅に

ある+アイコンをクリックします。ドロップダウンリストから[ テクスチャ2D]を選択します。アイテムが黒板に表示されます。名前をBaseTextureに変更します。


公開 チェックボックスがオンになっていることを確認します。プロパティを開くと、パブリックになり、インスペクターで使用できるようになります。


プロパティをグラフに追加するには、ラベルのそばにあるプロパティをワークスペースにドラッグするだけです。サンプルテクスチャ2Dノードの左側のどこかに残します。

接続BaseTexture用にポートをテクスチャの入力ポートSampleTextureの2Dノードアルベドに接続されています。これにより、以前の設定値が置き換えられます。法線マップ

についても同じプロセスを繰り返します。+アイコンをクリックして、新しいテクスチャ2Dを作成します。名前をNormal Mapに変更します。 プロパティをワークスペースにドラッグし、法線マップのサンプルテクスチャ2Dにアタッチします。




[ アセットを保存]をクリックして、メインエディターウィンドウに戻ります。Glow_Mat

マテリアルを選択し、インスペクターに 2つの新しいフィールド、Base TextureとNormal Mapがあることに注意してください。

テクスチャがまだ設定されていないため、灰色の球の上に緑色のハイライトがプレビューウィンドウに表示されます。


BaseTextureおよびNormalMapテクスチャWoodAlbedoおよびWoodNormalを 選択します。

これで、明るいエッジの下で、木のテクスチャが正しく表示されます。


パブリックプロパティを使用すると、シェーダーグラフを編集する必要なく、データを直接シェーダーに入力できます。さまざまな基本テクスチャと法線マップの選択を自分で試してください。

プロパティグローサイズとグローカラーを追加します


次に、Blackboardの他のタイプのプロパティを表示します。たとえば、フレネルエフェクトパワーの値を変化させると便利です。Blackboard

の+アイコンをクリックし、Vector1プロパティを作成します。これは、float型の単一のパラメーターを示します。 名前をGlowSizeに変更します。 このプロパティをスライダーに変換することで、このプロパティに入力する値の範囲を制限できます。[ モード]を[ スライダー ]に切り替え、[ 最小]を0.05に、[ 最大]を6に設定して範囲を指定します。デフォルト値



5を割り当てます。


GlowSize プロパティをワークスペースにドラッグします。出力ポートをフレネルエフェクト電源入力に接続します。

次に、ユーザーがグローの色を設定できるようにします。Blackboardでプロパティを作成する代わりに、グラフの既存のノードを変換します。


選択色のノードを、そして右クリックし、選択プロパティに変換します。カラーノードはれるの色プロパティに変換黒板もはやグラフで直接変更することができます。プロパティの名前をGlowColorに変更します。




[ アセットを保存]をクリックして、メインエディターウィンドウに戻ります。


プロジェクトウィンドウでGlow_Matマテリアルを選択します。GlowSizeスライダーとGlowColorフィールドがインスペクターで使用できるようになっているはずです。


素材の値を好みに合わせて変更してください。プレイモードを起動して、作業をテストします。
