Red Hat Ansible Engine 2.8：新しいAnsibleコンテンツ処理手法に向けて


ユーザーのリクエストに応えて、Ansible Engine 2.8 は、公式リリースに含まれないコンテンツの処理に関する変更を導入しました。これらの変更により、ユーザーにコンテンツを配信する新しい方法を実装できます。これは、Ansibleコンテンツとプラットフォーム自体のコードの両方を管理するときに、Ansibleメンテナーに依存しません。

新しいデザインになる


Becomeコンストラクトはかなり前に登場しましたが、バージョン2.8から、特権が昇格したときにパスワードを要求するためにAnsible EngineでデフォルトでBECOMEという単語が使用されています（* nixシステムでのsudo特権またはネットワークデバイスの有効モード）。言い換えれば、現在BECOMEは、それ自体がすでにターゲットシステムを理解している、権限を高めるための標準ツールです。

以下はその使用例です。

ansible-playbook --become --ask-become-pass site.yml
BECOME password:

さらに、BECOMEプラグインはAnsible Engine 2.8で登場しました。これはLinuxのdoasやWindowsのrunasのように機能し、特定のユーザーに代わってアクションを実行できるようにします。ネットワークデバイスの権限を高めるには、enable enableプラグインを使用します。

Pythonインタープリター検索


このエラーが発生した可能性があります：

/usr/bin/python: bad interpreter: No such file or directory

実際のところ、Ansible Engineの以前のバージョンでは、メイン（デフォルトで使用）のPythonインタープリターは/ usr / bin / pythonフォルダーにあると想定されていました。Ansibleはバージョン2.8以降、各ターゲットシステムでインタープリターを検索します。最初に、さまざまなディストリビューションのメインPythonインタープリターの実行可能ファイルのパスと名前のテーブルを調べ、次に順序付けられたフォールバックリストを使用します。Ansible2.8移植ガイドを参照してください。。

再試行ファイルはデフォルトでは作成されなくなりました


ディスク上の.retryファイルを長い間探していましたか？Ansibleを長い間使用している場合、明らかにそれらの多くがあり、それらは有用な場所しか占めていません。バージョン2.8以降、デフォルトでAnsible Engineはこれらのファイルを作成しなくなりました（デフォルトのansible.cfgファイルを編集することでキャンセルできます）。

Playの要約を更新


Ansible 2.8移植ガイド からの抜粋：

Play Recapは、デフォルトのコールバックプラグインの2つの新しい統計カウンターのおかげで、無視されて救出された状態のタスク、およびok、変更、到達不能、失敗、スキップされたタスクをカウントするようになりました。ignore_errors：yesフラグが設定されている失敗したタスクは、無視されたものとしてカウントされます。その後、レスキューセクションが作成された失敗したタスクは、レスキューされたものとしてカウントされます。Ansibleの以前のバージョンとは異なり、救出されたタスクは失敗としてカウントされなくなりました。

また、Play Recapプレートでは、プレイブックの実行後、ホストには追加の列がスキップされ、救出され、無視されます。

Red Hat Ansible Tower 3.5：さらなる自動化


5月末以降、新しいバージョンのRed Hat Ansible Tower 3.5が利用可能になり、自動化のためのいくつかの重大な改善が同時に提供されます。自分で確認する時間がない場合は、それらについて説明します。

Red Hat Enterprise Linux 8のサポート


Red Hat Enterprise Linux（RHEL）は、たとえばハイブリッドクラウドを構築するための信頼性が高く用途の広い基盤であると繰り返し言います。Ansible Tower 3.5（Ansible Engine 2.8と同様）はRHEL 8ホスト管理を提供し、Red Hat Ansible AutomationシステムのホストとしてRed Hat Enterprise Linux 8でも実行できます。

外部資格情報ストレージ


Ansible Tower 3.5では、組み込みの認証情報ストレージに加えて、外部の認証情報ストレージを使用できるようになりました。分散アプリケーションで認証情報にアクセスしやすくする必要がある場合があるためです。したがって、新しいバージョンのAnsible Towerは、パスワードとキーを保存するためのさまざまなソリューションと直接連携できます。次に例を示します。
