CI / CD en AWS, Azure y Gitlab. Nuevo curso de OTUS

¡Atención! Este artículo no es de ingeniería y está dirigido a lectores interesados ​​en la educación en el campo de CI / CD. Lo más probable es que si no está interesado en aprender, este material no le interesará.



Si usted es un desarrollador o administrador responsable de configurar procesos de integración continua / entrega continua , OTUS ha abierto un conjunto de cursos específicamente para usted: un curso práctico intensivo sobre el método popular de desarrollo continuo y entrega de software de Integración continua y Entrega continua en las plataformas Amazon Web Service, Azure, GitLab y Jenkins .

Durante la capacitación, los estudiantes aprenderán cómo configurar el proceso de ensamblaje y prueba de la aplicación y el proceso de instalación con tres proveedores líderes, así como comprender la arquitectura de los proveedores de la nube y aprenderán la automatización del análisis de código y la búsqueda de vulnerabilidades.

Como resultado de la capacitación, cada estudiante creará un trabajo de graduación, que será la implementación de procesos de CI / CD para cualquier proyecto de código abierto para elegir. Después de la capacitación, por supuesto, cada estudiante recibirá materiales para todas las clases, un certificado de finalización y, lo más importante, configurará el ensamblaje y las pruebas de la aplicación y podrá encontrar vulnerabilidades.



Por supuesto, este curso no es para todos. Pero si tienes experiencia :

  • Trabajar con git
  • Administración del sistema Linux o Windows
  • Desarrollo u operación
  • Trabaja con un proveedor de la nube

¡entonces OTUS te está esperando! Puede realizar una prueba de acceso para determinar si tiene los conocimientos suficientes para completar el curso CI / CD en AWS, Azure y Gitlab.

En la víspera del inicio del curso "CI / CD para AWS, Azure y Gitlab" , OTUS organizó la jornada de puertas abiertas. El maestro habló sobre el programa del curso con más detalle, respondió preguntas de los estudiantes y también describió el proceso de aprendizaje.


También está abierto el acceso a una visualización gratuita del seminario web abierto sobre el tema "Uso de Jenkins con K8S", realizado por Boris Nikolaev , profesor en el curso de CI / CD sobre AWS, Azure y Gitlab :


El proceso de capacitación en el curso "CI / CD en AWS, Azure y Gitlab" se lleva a cabo en formato de seminarios web en línea. A lo largo de la capacitación (y dura 3 meses), los estudiantes pueden preguntar a maestros experimentados que siempre están en contacto. Los ejercicios de práctica se completarán con Google Cloud Platform (GCP), Amazon Web Service y Microsoft Azure.

El programa del curso consta de cuatro módulos principales:

  1. Desarrollo en la nube (Código)
  2. Automatización de ensamblaje y prueba (Integración continua)
  3. Automatización de la instalación (entrega continua)
  4. Módulo resumen

Cada uno de ellos se comprenderá a fondo en las clases en formato de seminarios web en línea, y la tarea ayudará a consolidar el conocimiento adquirido, de acuerdo con el cual, si es necesario, puede obtener comentarios detallados de los maestros.

Muchos expertos llaman a CI / CD una de las mejores técnicas de desarrollo de software para tareas modernas. ¿Estás de acuerdo con esta afirmación?
