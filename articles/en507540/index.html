Working with custom events in htmlix.js

This article will examine the basic principles of working with user events in Htmlix and create a small application of four components: a form for entering a username, a greeting, an exit button for an application, and an array with application users.



A ready-made example can be clicked here .



You can read the htmlix api here.



After entering the user name in the form, the application will display its name in the greeting, exit button and create a new container in the array with users, and also save the data in localStorage. After reloading the page, it will take the name from localStorage. Clicking the ‚Äúexit profile‚Äù button will delete the data from all components and the variable from localStorage.



Let's create four components: the login form - form, the greeting - greeting, the logout button - logout and the array with all visitors - users_array, and see how we would create communication between them if we did not use user events.

In the example above, we recorded the data of the ‚Äúinput‚Äù property from the form component into the ‚Äútext‚Äù variable. Now we need to display them in the properties of the greeting and logout components, and also create a new container in the users_array array. To understand why custom events are needed, let's first try to do this in various ways without using them.



1. way - in the form's click method itself, go directly to each component and set the text property: 




        event.preventDefault();
        var text = this.parent.props.input.getProp();
                //  
                this.rootLink.state["greeting"].props.user_name.setProp(text);
                this.rootLink.state["logout"].props.user_name.setProp(text);
                  
                  ///  
                 this.rootLink.state["users_array"].add({user_name: text});


The disadvantage of this approach is obvious, what if we get data for this property somewhere else? For example, based on localstorage, they will load automatically.



2. The way is to create a method in the stateMethods object and call it when data changes: 




ststeMethods: {

         entryUser: function(text){

                this.state["greeting"].props.user_name.setProp(text);
                this.state["logout"].props.user_name.setProp(text);
                 this.state["users_array"].add({user_name: text});
       }
}

window.onload = function(){
             var name = window.localStorage.getItem('user_name');

             if(name)HM.stateMethods.entryUser.call.(HM, name); ///    = HM
}

// 
click: function(){
                  event.preventDefault();
		  var text = this.parent.props.input.getProp();
                  this.rootLink.stateMethods.entryUser.call.(this.rootLink, text); 
}


In this case, we managed to avoid duplication of code, but the problem is that the methods for working with component properties are not inside it, but in external functions. What if before setting a property we need to somehow format it, differently for each component, or we need to set not one property, but several, therefore it is better when the components themselves work with their properties.



3. the way is to transfer all the logic of working with properties to components, and create methods in them with which to call certain actions. 



	
	greeting: {
		
		container: "greeting",

                //     
		props: [ "user_name", ["greet_user", "aux"] ], 
		methods: {

                                this.props.user_name.setProp(name+" !!!");
                     }
	        }
	},
	logout: { 		
		container: "logout",

		props: [ "user_name", ["set_name", "aux"]], //

                                this.props.user_name.setProp(name);
                     }
                          
		},	
	},
	users_array: { 
		arrayProps: [["entry_user", "aux"]],  ///      
                arrayMethods: {
                       entry_user: function(name){
                              this.add({user_name: name})
                       }
               }	
		container: "user",
		props: [ "user_name", ],
		methods: {		  
		},        	
	},
    stateMethods: {   //            
         entryUser: function(text){

                this.state["greeting"].methods.greet_user(text);
                this.state["logout"].methods.set_name(text);
                this.state["users_array"].methods.entry_user(text);
        }
   }
   //     entryUser


Now the components are encapsulated, however, we access the component by its name. What if we later want to change the name of the component or method, or to delete the component altogether, or we will have many such components that listen to the same property. Or we will have many such properties, so we will have to create many such methods.



To facilitate this task, you can use custom events. The component subscribes to the event, and when it occurs, does something with its properties.



Let's edit the code, now using custom events: 



var StateMap = {

   eventEmiters: {	
             //   -  	
             ["emiter-entry-user"]: {prop: ""}, 		
   },
   form: {
	container: "form",
	props: ["input", "click"],
	methods: {
			
		click: function(){
		    event.preventDefault();
		    var text = this.parent.props.input.getProp();

		     //        
		    this.rootLink.eventProps["emiter-entry-user"].setEventProp(text); 	
		    window.localStorage.setItem('user_name', text);							
		}			
	},		
},	
greeting: {
		
	container: "greeting",
	props: [ "user_name", ['listen_entry_user', "emiter-entry-user", "" ] ], //   "emiter-entry-user"
	methods: {
			
		listen_entry_user: function(){
		   //      
		   this.parent.props.user_name.setProp( this.emiter.getEventProp() );
		},	
	},		
},
logout: { 
		
	container: "logout",
	props: [ "user_name", ["listen_entry_user", "emiter-entry-user", "" ]], //   "emiter-entry-user"
	methods: {
			
		listen_entry_user: function(){
		     //       
		     this.parent.props.user_name.setProp( this.emiter.getEventProp() );
		},			
	},	
},
    //      , ..            
users_array: { 
		
	arrayProps: [ ['listen_entry_user', "emiter-entry-user", ""] ], //   "emiter-entry-user"
	   arrayMethods: {
			
		listen_entry_user: function(){
			//       
			this.parent.add( {user_name: this.emiter.getEventProp()} );
		},			
	},		
	container: "user",
	props: [ "user_name",   ],
	methods: {						
		}
   },        	


}
window.onload = function(){
	
		
	var HM = new HTMLixState(StateMap);
	var name = window.localStorage.getItem('user_name');
	
	if(name != null)HM.eventProps["emiter-entry-user"].setEventProp(name); ///  "emiter-entry-user"        	
}


Thus, we got rid of the intermediate method "entryUser" and simply raise the event by passing new data into it in the form and after the page is loaded. Now, if we need to change something in the component, or delete it altogether, we do not need to correct the code in each function, which can be any number. And to unsubscribe from any event, you can simply delete the listener in the component, or temporarily disable it using the disableEvent () method in the property - subscriber.
