    Zookepperの起動を確認するには、

    ZookeeperとKafkaサービスを仕事用に構成します。最初に、/ etc / systemd / system / zookeeper.serviceファイル（以下のファイルの内容）を編集/作成します。

    [Unit]
Description=Apache Zookeeper server
Documentation=http://zookeeper.apache.org
Requires=network.target remote-fs.target
After=network.target remote-fs.target

[Service]
Type=simple
ExecStart=/home/ubuntu/kafka/bin/zookeeper-server-start.sh /home/ubuntu/kafka/config/zookeeper.properties
ExecStop=/home/ubuntu/kafka/bin/zookeeper-server-stop.sh

[Install]
WantedBy=multi-user.target

    次に、Kafkaの場合、ファイル/etc/systemd/system/kafka.serviceを編集/作成します（以下のファイルの内容）。

    [Unit]
Description=Apache Kafka server (broker)
Documentation=http://kafka.apache.org/documentation.html
Requires=zookeeper.service

[Service]
Type=simple
ExecStart=/home/ubuntu/kafka/bin/kafka-server-start.sh /home/ubuntu/kafka/config/server.properties
ExecStop=/home/ubuntu/kafka/bin/kafka-server-stop.sh

[Install]
WantedBy=multi-user.target

最初の喜び


組み立てられたkafkaサーバーで最初のトピックを作成します。

  • server.properties構成ファイルで指定したように、「host01：2181」への接続を使用することが重要です。
  • トピックにデータを書き込みます。

    kafka-console-producer.sh --broker-list host01:9092 --topic first_topic

    Ctrl-C-トピックコンソールを終了します。

  • 次に、トピックからデータを読み取ってみます。

    kafka-console-consumer.sh --bootstrap-server host01:9092 --topic last_topic --from-beginning

  • カフカのトピックのリストを見てみましょう。

    bin/kafka-topics.sh --zookeeper spark01:2181 --list

  • 単一クラスターセットアップ用にチューニングするためのkafkaサーバーパラメーターの編集＃ISR
    パラメーターを1に変更する必要があります。

    bin/kafka-topics.sh --zookeeper spark01:2181 --config min.insync.replicas=1 --topic __consumer_offsets --alter

  • Kafkaサーバーを再起動し、消費者のオームを再度接続しようとします

  • トピックリストを見てみましょう。

    bin/kafka-topics.sh --zookeeper host01:2181 --list

単一ノードのクラスターでApache Sparkを構成する


AWSにZookeeperとKafkaサービスがインストールされたノードインスタンスを準備しました。これを行うには、Apache Sparkをインストールする必要があります

。ApacheSparkディストリビューションの最新バージョンをダウンロードします。

wget https://archive.apache.org/dist/spark/spark-2.4.0/spark-2.4.0-bin-hadoop2.6.tgz


  • ディストリビューションを解凍し、sparkのシンボリックリンクを作成し、不要なアーカイブファイルを削除します。

    tar -xvf spark-2.4.0-bin-hadoop2.6.tgz
ln -s spark-2.4.0-bin-hadoop2.6 spark
rm spark*.tgz

  • sbinディレクトリに移動し、sparkウィザードを実行します。

    ./start-master.sh

  • Webブラウザーを使用して、ポート8080でSparkサーバーに接続します。

  • 同じノードでスパークスレーブを実行する

    ./start-slave.sh spark://host01:7077

  • host01のマスターでSparkシェルを実行します。

    ./spark-shell --master spark://host01:7077

  • 起動が機能しない場合は、bashでSparkへのパスを追加します。

    vi ~/.bashrc
#     
SPARK_HOME=/home/ubuntu/spark
export PATH=$SPARK_HOME/bin:$PATH

    source ~/.bashrc

  • host01でマスターを使用してもう一度Sparkシェルを実行します。

    ./spark-shell --master spark://host01:7077

    Kafka、Zookeeper、およびSparkを備えた単一ノードのクラスターが機能します。やったー！
