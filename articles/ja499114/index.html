連邦通信局の番号計画をリレーショナルデータベースにエクスポートする

連邦通信庁は定期的に公開されている番号計画を更新します。この計画を使用して、ダイヤルプランの地域または加入者プロバイダーを決定する場合、この情報の関連性に関心がある可能性が最も高くなります。一見すると、データをダウンロードして処理し、データベースに送信するアプリケーションを書くのに複雑なことは何もありませんが、それを実装し始めると、必然的に落とし穴に遭遇します。これについては、これから説明します。

ダイヤルプランは、csv形式の4つの表形式のファイルで構成されています

rossvyaz.ru/data/ABC-3xx.csv
rossvyaz.ru/data/ABC-4xx.csv
rossvyaz.ru/data/ABC-8xx.csv
rossvyaz.ru/data/ DEF-9xx.csv

URLが変更されることがあります。

すべてのテーブルの構造は同じです：

/ DEF;;;;;

[ 地域]フィールドでは、一部のプロバイダーに追加のセミコロンがあります。

 ; | ;

アプリケーションでテーブルを読み取る際のエラーを回避するには、予想されるフィールド数のチェックを無効にするか、正しくないレコードを正しいレコードに置き換える必要があります。

次の構造を持つテーブルにデータを書き込みます。

first BIGINT PRIMARY KEY NOT NULL  //  
last BIGINT UNIQUE NOT NULL        //  
provider TEXT                      // 
source_region TEXT                 //        
region INT NOT NULL                //

テーブル内のほとんどの領域は、部分文字列の出現によって明確に識別できます。たとえば、サラトフリージョンのすべてのエントリには、部分文字列Saratが含まれています。

明確に決定できない領域については、以下に示す追加のロジックが必要です。

例1：

  • ヤマル・ネネツ自治管区
  • ネネツ自治管区

ヤマロネネツ自治管区はチュメニ地域の一部であり、ネネツ自治管区はアルハンゲリスクの一部です。地域の数値コードを決定する際の問題は、部分文字列Nenets AOがYamalo-Nenets AOに完全に含まれていることです。これは、2つのリージョンがNenets自律エリアのサブストリングを含むエントリに対応することを意味します。

この問題を解決するには、部分文字列（部分文字列を除く）がないかどうかのチェックを追加する必要があります。換言すれば、ヤマロ・ネネツ自治管区でのエントリによって決定されるであろうYamaloのエントリによって、及びネネツ自治管区ネネツの不在Yamalo。

例2：

  • g.o. クラスノゴルスク
  • ライアントール
  • パルツァ

不明な地域の例。Rossvyazのテーブルには、そのようなレコードが多数あります。FinderはそのGOを見つけるのに役立ちます クラスノゴルスク -モスクワ地方、リャントール -ハンティマンシ自治管区、パルツの入植地 -モルドヴィア共和国。解決策は簡単で、目的の部分文字列が文字列の配列に変換され、ループ内での出現を確認します。排他的な部分文字列も配列に変換されます。

例3：

  • シベリア連邦管区、極東連邦管区
  • クラスノヤルスク地域、ハカシア共和国、モスクワ、サンクトペテルブルク
  • ロシア連邦

この例では、地域を一意に識別することはできません。リストから最も人口の多いものを選択するか、そのようなレコードに特別なコードを割り当てることができます。実際、いくつかの地域に対応するすべてのレコードは番号8〜80 [0〜9]であり、通話料は請求されません。そのようなレコードに200から210の範囲でコードを割り当てました。それらが私にとって役立つとは思いません。

アプリケーションコードはここにあります。
Linux、macos、およびWindowsプラットフォームの実行可能ファイルは、binディレクトリにあります（Windowsプラットフォームではアプリケーションをテストしていません）。Config.yml

設定ファイル実行可能ファイルと同じディレクトリにある必要があります。必要に応じて、アプリケーションにフラグサポートを実装して、コマンドライン引数で構成へのパスを示すことができます。プルリクエストは歓迎されます。

data_source フィールドには、Rossvyazテーブルファイルへのパスが含まれています。パスがhttpで始まる場合、アプリケーションは組み込みのWebクライアントを使用してテーブルをロードします。そうでない場合、ファイルシステムでテーブルを見つけようとします。

コメントアウトされた行に注意してください。これらは、リポジトリに格納されているRossvyazテーブルファイルへのパスです。最速のインターネット接続がない場合、アプリケーションを起動するたびにこれらのファイルが再度ダウンロードされます。より高速なアプリケーションでは、これらの行のコメントを外して、ハイパーリンクをコメント化します。アプリケーションを思ったより頻繁に実行する必要があるかもしれません。後で例を使用して理由を説明します。

フィールドの例外と領域修正を含むファイルへのパスとリージョンの説明が含まれています。それらについて少し後で。db

セクションでは、MySQL DBMSに接続するためのパラメーターについて説明します。別のDBMSを使用するには、コード内のドライバーと、サービス構造のコンストラクター内の接続文字列テンプレートを置き換える必要があります。

db, err := dbr.Open("mysql",
		fmt.Sprintf("%s:%s@tcp(%s)/%s", c.DB.User, c.DB.Password, c.DB.Host, c.DB.Name),
		nil)

ただし、DBMSタイプフィールドを構成に追加し、これらのタイプのサポートをアプリケーションに追加するのに複雑なことは何もありません。プルリクエストも歓迎します。exceptions.yml

ファイルには修正のリストが含まれています。Rossvyazのテーブルファイルで見つかった、コロンの前のexceptions.ymlの部分文字列のすべての一致は、コロンに続く部分文字列に置き換えられます。regions.yml ファイルには、オカレンスの配列を含むリージョンの数値コードのリストが含まれ、特定のリージョンを決定するために使用されるサブストリングは除外されます。いずれかのレコードで一致が見つからない場合、または2つ以上の一致がある場合、このレコードはデータベースに送られますが、地域フィールドには値0が含まれます（地域は定義されていません）。

これで、すべての設定が完了しました。アプリケーションを起動します。
