レトロフィットAPIの説明


ライブラリ開発者のWebサイトの説明によると、これはAndroid用の型指定されたhttpクライアントです。公式ドキュメントはこちらから入手できます。認識を容易にするために、いくつかの概念を説明する必要があります。

  1. ネットワークデータはどのように交換されますか？
  2. REST Webサーバーとは
  3. 後付けの説明
  4. HTTPメソッド

  1. ネットワーク上のすべてのデバイスは、交換プロトコル（TCP / IP、Http、Https、Ftpなど）を介して相互にデータを交換します。プロトコルは、データのパッケージ化、アドレス指定、送信、および受信に関して一般的に合意されたルールです。これらのプロトコルを使用して、クライアントとサーバー（Webサーバー）は互いにデータを交換します
  2. Restは、httpおよびhttpsプロトコルでデータを交換するために使用されるWebサーバーアーキテクチャです。Restは、HTTPメソッドを実行するための一連のルールです。
  3. Retrofitは、残りのWebサーバーとの通信を簡素化するJava APIです。HTTPメソッドをJava言語で直接実行できます。
  4. Httpメソッドは、クライアントがWebサーバーに実行を要求する標準化された手順の形式です。httpメソッド：Get、Head、Post、Put、Delete、Trace、Options、Connect、Patch。後付けは最初の5つのみをサポートします

HTTPメソッド


1. GetおよびHeadメソッドは、最も安全なメソッドと見なされていますどこでも変更せずにデータを受信します。最初のメソッドは応答のタイトルと本文を取得し、2番目のメソッドはタイトルのみを取得します。サーバーからの応答は、値のコレクションの形式で送られます。リクエストに応じて、選択用の追加パラメーターを指定できます。
たとえば、リクエストsomeadress.ru?list=10&name=ivanovで、選択パラメーターリスト= 10および名前= ivanovを渡します

2.削除-メソッドは同じ名前の機能を実行し、サーバー上のエントリを削除します。メソッドはリクエストボディを必要としません

3. PostおよびPutメソッドは、サーバー上のデータを作成または変更します。これらの他のメソッドとは異なり、リクエストボディの存在は必須です。Putメソッドは、レコードがない場合はサーバーにレコードを作成し、レコードがある場合は変更します。Postはエントリを作成し、サーバーでプロシージャを実行します。次の式を覚えておいてください。Put-データを送信し、Post-サーバーにメッセージを送信します。

例はAndroid Studio IDEに示されています。

クライアントとサーバー間の交換が同じ形式で行われるように交換を構成する必要があります。主に3つのデータ形式が使用されます。以下は、3つの形式でのマッピングの例です

。JSON

{
"id":"1234",
"name":"Vlad"
}

XML

<root>
<id>1234</id>
<name>Vlad</name>
</root>

プロトコルバッファ

{
id:"1234",
name:"Vlad"
}

このリストに、次のようなJavaクラスのフォーマットがないのは残念です。

public class root{
private string id;
private string name;
…
}

Javaオブジェクトを任意の言語で機能するサーバーに転送するために、レトロフィットではデータコンバーターを使用します。これらのコンバーターは、データシリアル化ライブラリとも呼ばれます。Retrofitには、Gson、Jackson、Moshi、Protobuf、WireおよびSimple XMLの最も一般的なシリアル化ライブラリを使用する6つのコンバーターが含まれています。使用するコンバーターは、サーバーが受け入れる形式によって異なります。
たとえば、私はGSonコンバーターを使用しています 使いやすい
