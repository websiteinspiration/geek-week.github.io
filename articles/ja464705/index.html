PythonでのAPIの記述（FlaskおよびRapidAPIを使用）



この記事を読んでいる方なら、おそらくAPI（アプリケーションプログラミングインターフェース）を使用して開く可能性についてすでにご存じでしょう。

多くのオープンAPIの1つをアプリケーションに追加することにより、このアプリケーションの機能を拡張したり、必要なデータを追加したりできます。しかし、コミュニティと共有したい独自の機能を開発したらどうなるでしょうか。

答えは簡単です。独自のAPIを作成する必要があります。

最初はこれは困難な作業のように見えますが、実際には簡単です。Pythonでこれを行う方法を紹介します。

始めるために必要なもの


必要なAPIを開発するには

  • Python 3
  • Flask -Webアプリケーションを作成するためのシンプルで使いやすいフレームワーク。
  • Flask-RESTfulは、最小限の設定でREST APIをすばやく開発できるFlaskの拡張機能です。

インストールは次のコマンドで実行されます。

pip install flask-restful

始める前に

基本的なCRUID機能を備えたRESTful APIを開発します。

問題を完全に理解するために、上記の2つの用語を見てみましょう。

RESTとは何ですか？

REST API（Representational State Transfer）は、HTTP要求を使用してデータを交換するAPIです。

REST APIは特定の基準を満たしている必要があります。

  • クライアントサーバーアーキテクチャ：クライアントはユーザーインターフェイスとやり取りし、サーバーはバックエンドとデータウェアハウスとやり取りします。クライアントとサーバーは独立しており、それらのいずれかを別のものから個別に置き換えることができます。
  • ステートレス-クライアントデータはサーバーに保存されません。セッション状態はクライアント側に保存されます。
  • キャッシュ機能—クライアントはサーバーの応答をキャッシュして、全体的なパフォーマンスを向上させることができます。

CRUDとは何ですか？

CRUDは、4つの基本的なアクション（作成、読み取り、更新、削除）を説明するプログラミング概念です。

REST APIでは、要求タイプと要求メソッドが、post、get、put、deleteなどのアクションを担当します。

基本的な用語を理解したので、APIの作成を開始できます。

開発


人工知能に関する引用のリポジトリを作成しましょう。AIは今日最も急速に成長しているテクノロジーの1つであり、PythonはAIと連携するための一般的なツールです。

このAPIを使用すると、Python開発者はAIに関する情報をすばやく取得し、新しい開発に刺激を受けることができます。開発者がこのトピックについて貴重な考えを持っている場合、彼はそれらをリポジトリに追加することができます。

まず、必要なモジュールをインポートしてFlaskを構成します。

from flask import Flask
from flask_restful import Api, Resource, reqparse
import random
app = Flask(__name__)
api = Api(app)

このスニペットでは、Flask、Api、Resourceが必要なクラスです。

ReqparseはFlask-RESTfulリクエスト解析インターフェースです...ランダムな引用を表示するにはrandomモジュールも必要です。

次に、リソースクラスQuoteを作成する必要があります。これは、APIのエンドポイントのオペレーションを決定します。クラス内では、get、post、put、deleteの4つのメソッドを宣言する必要があります。

GETメソッドから始めましょう。

このメソッドを使用すると、IDを指定することで特定の見積もりを取得できます。

class Quote(Resource):
    def get(self, id=0):
        if id == 0:
            return random.choice(ai_quotes), 200
        for quote in ai_quotes:
            if(quote["id"] == id):
                return quote, 200
        return "Quote not found", 404

IDにデフォルト値が含まれている場合、GETメソッドはランダムな引用を返します。メソッドを呼び出すときに、IDが設定されていませんでした。

設定されている場合、メソッドは引用符の中を検索し、指定されたIDを含む引用符を見つけます。何も見つからない場合は、「見積書が見つかりません、404」というメッセージが表示されます。

要確認：このメソッドは、リクエストが成功した場合は200、レコードが見つからない場合は404のHTTPステータスを返します。

次に、新しい見積もりをリポジトリに追加するPOSTメソッドを作成します。

このメソッドは、入力時に新しい見積もりの​​IDを受け取ります。さらに、POSTはreqparseを使用して、リクエスト本文（作成者と引用テキスト）に含まれるパラメーターを解析します。

def post(self, id):
      parser = reqparse.RequestParser()
      parser.add_argument("author")
      parser.add_argument("quote")
      params = parser.parse_args()
      for quote in ai_quotes:
          if(id == quote["id"]):
              return f"Quote with id {id} already exists", 400
      quote = {
          "id": int(id),
          "author": params["author"],
          "quote": params["quote"]
      }
      ai_quotes.append(quote)
      return quote, 201

上記のコードでは、POSTメソッドが見積IDを受け入れました。次に、reqparseを使用して、クエリから著者と見積もりを取得し、params辞書に保存しました。

指定されたIDの見積もりが既に存在する場合、メソッドは対応するメッセージとコード400を表示します。

指定されたIDの見積もりがまだ作成されていない場合、メソッドは指定されたIDと作成者、およびその他のパラメーターで新しいレコードを作成します。次に、ai_quotesリストにエントリを追加し、201コードとともに新しい引用符でエントリを返します。

次に、PUTメソッドを作成して、リポジトリ内の既存の引用符を変更します

def put(self, id):
      parser = reqparse.RequestParser()
      parser.add_argument("author")
      parser.add_argument("quote")
      params = parser.parse_args()
      for quote in ai_quotes:
          if(id == quote["id"]):
              quote["author"] = params["author"]
              quote["quote"] = params["quote"]
              return quote, 200
      
      quote = {
          "id": id,
          "author": params["author"],
          "quote": params["quote"]
      }
      
      ai_quotes.append(quote)
      return quote, 201

前の例と同様に、PUTメソッドはIDと入力を取り、reqparseを使用して引用パラメーターを解析します。

指定されたIDの見積もりが存在する場合、メソッドはそれを新しいパラメーターで更新し、更新された見積もりをコード200で表示します。指定されたIDの見積もりがない場合、新しいレコードがコード201で作成されます。

最後に、DELETEメソッドを作成して見積もりを削除します。それはもはや刺激的ではありません

def delete(self, id):
      global ai_quotes
      ai_quotes = [qoute for qoute in ai_quotes if qoute["id"] != id]
      return f"Quote with id {id} is deleted.", 200

このメソッドは、入力時に見積もりIDを取得し、共通リストを使用してai_quotesリストを更新します。

これですべてのメソッドが作成されたので、APIにリソースを追加し、パスを設定してFlaskを実行するだけです。

api.add_resource(Quote, "/ai-quotes", "/ai-quotes/", "/ai-quotes/<int:id>")
if __name__ == '__main__':
    app.run(debug=True)

python3 app.py

APIのテスト

APIが作成されたら、テストする必要があります。

これは、curlコンソールユーティリティまたはInsomnia RESTクライアントを使用するか、APIをRapid APIに公開することで実行できます。



API

RapidAPI を公​​開しています。これは、10,000を超えるAPI（および約100万の開発者）を抱える世界最大の市場です。

RapidAPIは、サードパーティのAPIを操作するための単一のインターフェースを提供するだけでなく、独自のAPIをすばやく簡単に公開する機能も提供します。

にこれを行うには、まずネットワーク上のサーバーに公開する必要があります。この例では、Herokuを使用します。彼と一緒に作業しても問題は発生しません（彼について詳しくは、こちらをご覧ください）。

HerokuでAPIを公開する方法

1. Herokuをインストールします。

最初のステップは、Heroku Command Line Interface（CLI）を登録してインストールすることです。これはUbuntu 16以降で動作します。

sudo snap install heroku --classic

次にログインします：

heroku login

2.必要なファイルを追加します。

次に、公開するファイルをアプリケーションのフォルダーに追加する必要があります。

  • 必要なPythonモジュールのリストを含むrequirements.txt。
  • Procfile。アプリケーションを実行するために実行する必要があるコマンドを示します。
  • .gitignore-サーバーで不要なファイルを除外します。

requirements.txtファイルには、次の行が含まれます。

  • フラスコ
  • フラスコ休息
  • Gunicorn

注：サーバー上でアプリケーションを実行する必要があるため、リストにgunicorn（Python WSGI HTTPサーバー）を追加しました。

Procfileには以下が含まれます：

web：gunicorn app：app

.gitignore content：

*.pyc
__pycache__/

ファイルが作成されたので、git repoを初期化してコミットします。

git init
git add
git commit -m "First API commit"

3.新しいHerokuアプリケーションを作成します。

heroku create

マスターブランチをHerokuリモートリポジトリに送信します。

git push heroku master

これで、次のコマンドを使用してAPIサービスを開くことができます。

heroku ps:scale web=1
heroku open

APIはyour-random-heroku-name.herokuapp.com/ai-quotesで入手できます。

無料のアカウントを登録します-これはFacebook、Google、GitHubを使用して行うことができます。



2. APIをコントロールパネルに追加します。



3.次に、APIに関する一般情報を入力します。



4. [APIの追加]をクリックすると、APIに関する情報を入力できる新しいページが表示されます。



5.これで、APIエンドポイントを手動で入力するか、OpenAPIを使用してswagger ファイルをダウンロードできます。



さて、ここで、エンドポイントページでAPIのエンドポイントを設定する必要があります。私たちの場合、エンドポイントはCRUD（get、post、put、delete）の概念に対応しています。



次に、エンドポイントGET AI見積もりを作成する必要があります。これは、ランダムな見積もり（IDがデフォルトの場合）または指定されたIDの見積もりを表示します。

エンドポイントを作成するには、「エンドポイントの作成」ボタンをクリックします。
