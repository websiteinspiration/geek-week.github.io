简要介绍如何教Brother MFP在Windows Server中进行扫描



这甚至不是文章，而是有关如何在Microsoft Windows Server环境中使用Brother MFP扫描仪的网络功能的说明。这将与设备驱动程序的完整安装有关。将以Brother MFC-L2700DRN为例。正式地，扫描功能仅在台式机操作系统上受支持，服务器操作系统只能发送到打印。在寻找解决方案的过程中，我经常遇到与该特定制造商存在类似问题的论坛。全部归结为使用FTP，SMB或电子邮件。

因此，下面将给出通用解决方案。

第1部分。安装程序的修改


安装扫描仪需要完整的驱动程序包。由于我的目标操作系统是Windows Server 2016，因此安装程序将适用于Windows 10。下载，解压缩。
接下来，您需要找出哪个驱动程序负责扫描程序。在安装/ MFC-L2700DN_series文件夹包含Brinst_Common.ini文件。在其中，我们在[Specification]部分中找到了EnableServer = 0行，并将其值更改为1。此参数禁用对服务器系统的检查。保存文件。接下来，在[Driver]部分中找到ScannerInfName =行。该驱动程序将被修改。 驱动程序文件位于install \ driver \ gdi \ 32_64文件夹中

。必须执行以下操作：找到[Manufacturer]部分，并将NTamd64.XY添加到小数点后的列表末尾，其中X和Y对应于您的OS版本。您可以在这里查看。对于Server 2016对应于Windows 10并等于NTamd64.10.0

第2部分。服务器和安装


安装的一个小功能是需要两次重新引导。因此，需要将服务器置于“测试模式”以禁用驱动程序的数字签名验证。安装后，重新打开所有设备，它将可以使用。

打开命令行或PS（根据需要），然后输入bcdedit -set TESTSIGNING ON，然后重启系统。

以标准方式，在安装\ MFC-L2700DN_series \ setup.exe下运行Brother安装程序！不要安装网络打印机驱动程序，否则安装程序将抛出错误，您将必须手动删除未知设备（通过添加打印机进行安装）。其余的要品尝。



安装完成后，打开命令行或PS并输入bcdedit -set TESTSIGNING OFF，然后重新引导系统。检查扫描仪。
