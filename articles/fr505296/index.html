Architecture du processeur


L'ensemble du processeur est basé sur un fichier de registre de quatre groupes fonctionnels de dix cellules chacun. Bien que les opérations ALU soient possibles sur toutes les cellules, les règles de fonctionnement correct avec elles doivent être observées plus ou moins strictement. Ici, il vous suffit de vous rappeler la logique et le but de ces groupes fonctionnels:
Groupe Rendez-vous Remarque
A₀ ... ₉ Batterie A₀ stocke les drapeaux d'état ALU
B₀ ... ₉ Base B₀ stocke la base de retour du sous-programme
C₀ ... ₉ Compteur C₀ stocke le décalage de retour du sous-programme
D₀ ... ₉ Dispositif Vous pouvez organiser un cache

Étant donné que le processeur est assez simple et ne dispose pas de mécanismes matériels intégrés pour organiser les opérations de pile, les registres B₀: C₀ stockent l'adresse où l'opération d'accès au sous-programme a été placée. Si nécessaire, le programmeur doit prendre en charge toutes les opérations avec la pile et les décrire avec un algorithme.

Étant donné que l'index de l'opérande de droite est deux, dans le code de l'opération ALU «Sum», il doit être indiqué explicitement dans le cahier de gauche comme 2A₁₆.
