Visual Studioコードスペースの紹介：どこにいてもクラウド開発

私たち全員が社会的距離や遠隔作業などの慣行を実施するにつれて、開発チームはより分散されてきました。私たち自身のチームは、この移行の一部であるいくつかの課題に直面しました。このため、開発者がどこからでも生産性を維持できるようにするための取り組みを強化することが求められました。まず、ユーザーがブラウザからライブ共有セッションに参加できるようにすることで、リモートコラボレーションを簡素化しました。今日は、Visual Studio Onlineに関するニュースをお届けします。これは、どこにいても生産的な作業を支援します。



Visual Studioコードスペースの紹介


11月、Public Preview Visual Studio Onlineのリリースを発表しました。過去数か月にわたって、私たちは膨大な数のレビューを受け取り、このツールにうまく取り組んできました。私たちが見た中で最も頻繁に引用されるコメントの1つは、クラウドベースの開発環境を作成する機能は、以前考えていたよりも多くの要因にプラスの影響を与えるということです。特に、ユーザーは次のことを信じています。

  • 新しいプロジェクト（または既存のプロジェクトの新しいタスク）に接続するために必要な時間ははるかに短い
  • クラウドベースの開発環境のパフォーマンスは、オンプレミス環境よりも多くの点で優れています。
  • Visual Studio Code、ブラウザーエディター、またはVisual Studio IDE（プライベートプレビュー）を簡単に切り替えることができるため、ほぼすべてのデバイスから作業できます。
  • Live Shareは共同開発、編集、デバッグに使用できるため、同僚との共同作業は簡単です。

開発者はVisual Studio Onlineが単なる「ブラウザーエディター」以上のものであると考えていることを学びました。彼らは、「このクラウドベースの開発環境の機能により、すべてのコードを記述したい場所になる」と述べています。

この気分とサービスの真の価値をよりよく一致させるために、Visual Studio Onlineの名前をVisual Studio Codespacesに変更します。あなたの大きなプロジェクトでクールな経験をしたいですか？コードスペースで実行してください。新しい関数のプロトタイプをすばやく作成したり、短期的なタスク（たとえば、プルリクエストの操作）を実行したりする必要がありますか？コードスペースを作成します。コードスペースは、単に最も生産的なコードスペースです。

価格が低くなりました


Visual Studio Online Codespacesは、市場で最高のリモート開発エクスペリエンスを提供すると私たちは信じています。特に現在私たち全員が直面している問題を考慮して、できるだけ多くの人がコードスペースにアクセスして使用できるようにしたいと考えています。これを実現するために、価格を下げました。

いつものように、私たちはあなたが使用した時間とリソースに対してのみあなたに課金し続けます。Codespaceでアクティブな間、Azureコンピューティングリソースを使用します。アクティブな各コードスペースの新しい価格は、Azureの価格ページまたは価格計算ツールで確認できます。

必要な力だけ


最後に、本日、1つの新機能も発表します。開発者はクラウドのコードスペースで見られるパフォーマンスを気に入っていますが、あまり多くのパワーを必要としないシナリオや、マルチタスクモードで作業できるようにいくつかの小さなコードスペース（コードスペース）を実行したいシナリオもたくさんあることがわかりました。 。

これらのシナリオをより適切に実装し、人々が初めてリモート開発を試すのを支援するために、2つの仮想コア、4 GBのRAMと64 GBのSSDストレージを備えた基本インスタンスタイプを提示します。この構成で多くのテストを行いましたが、エクスペリエンスは標準およびプレミアムのタイプと同じくらい素晴らしいですが、低コストです。基本的なインスタンスタイプは現在利用可能です。

リモート開発。今まで一番


過去数か月の間に、他のクールな機能をサービスに追加しました！私の個人的なお気に入りは、独自のDockerファイルまたはイメージを作成するためのサポートです。この機能を使用すると、Docker Hubなどのレジストリからイメージをポイントするか、Visual Studioコードスペースを起動したときに接続するリポジトリに独自のDockerfileを配置することにより、Codespaceを仕様に適合させることができます。

私たちは最近、ユーザーが環境内のインスタンスのサイズを変更できるようにする小さいながらも重要な機能をリリースしました。これは、ほとんどの時間を低電力消費と低コストの環境で費やし、必要な場合にのみ「電力を増やす」ことができることを意味します-状態を失うことなく、または新しい環境を作成する必要はありません。これは、Basicのインスタンスタイプもサポートします。

最近嬉しかった最後の追加は、セルフホスト環境の機能に加えられた大幅な改善でした。スタンドアロン環境では、任意のコンピューターをVisual Studio Codespacesに登録し、VS Codeまたはブラウザーからそれに接続できます。さらに、それは100％無料です。私の同僚のアリソンはすべての詳細を説明しました。 -詳細については、彼女の投稿を読むことを忘れないでください。
