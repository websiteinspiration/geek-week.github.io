ファームウェアをダウンロードする


私たちは成功から一歩離れています。ファームウェアをダウンロードするだけです。ターミナルを開き、ファームウェアのあるフォルダ（gnuk.elf）に移動します。

次のコマンドを実行します。

$ openocd -f interface/stlink-v2.cfg -f target/stm32f1x.cfg -c 'program build/gnuk.elf verify reset exit'

これで、ファームウェアがデバイスにアップロードされました。マイクロコントローラのメモリの読み取りを禁止するために、残りのステップが1つありました。注意！これは非常に重要な段階であり、キーを盗んだ人がそこから秘密情報を抽出することはできません。

これを行うには、次のコマンドを実行します。

openocd -f interface/stlink-v2.cfg -f target/stm32f1x.cfg -c init -c "reset halt" -c "stm32f1x lock 0" -c reset -c exit

これですべて準備が整いました。

新しいSSH鍵の生成


キーはコンピューターではなくデバイス自体で生成されることを理解することが重要です。したがって、そのようなキーがデバイスから出ることはありません。

gpgに移動します。


$ gpg --card-edit

gpgインタラクティブモードが開き、adminコマンドで管理モードを有効にします。


gpg/card> admin

次に、新しい鍵生成コマンドを実行します。


gpg/card> generate

以下は、標準のgpgキー生成手順です。その間、バックアップキーをディスクに保存するように求められます。開発者はバックアップを作成することをお勧めしますが、あなたが決定します。
唯一のことは、2048ビットまでのRSAキーのハードウェア生成がサポートされていることです。4096が必要な場合は、コンピューターでキーを生成してから、デバイス自体にインポートする必要があります。
次に、ピンコードが必要になります。デフォルトでは、次のPINコードがファームウェアで保護されています

。CARD PIN-123456
ADMIN PIN-12345678

今後、変更する必要があります。

キーは、コンピューター上で直接生成した場合よりも少し長く生成できるため、しばらくお待ちください。

標準のピンコードを変更する方法


同様に重要なステップは、独自のPINコードをインストールすることです。最初に、2つの異なるPINコードが使用されていることを理解する必要があります。

  • PINコード-このコードは、スマートカードでの日常的な操作に使用されます-何かを暗号化したり、署名したりするためなど
  • 管理者のPINコード-このコードは、キーを変更/削除し、あらゆる種類の「管理者」操作を実行するために必要です
  • コードのリセット-PINコードの入力を3回間違えた後、トークンのロックを解除できるコード。その使用はオプションなので、それはあなた次第です

コードごとに3回しか試行できません。その後、トークンはブロックされます。

それでは始めましょう。これを行うには、再びインタラクティブGPGモードに入ります。


$ gpg --card-edit

そしてpasswdコマンドを入力します：


gpg/card> passwd

トークンからPINコードを変更できるウィンドウが開きます。

次に、管理者PINを変更する必要があります。これを行うには、管理者モードに移動します。


gpg/card> admin

passwdコマンドをもう一度入力します。


gpg/card> passwd

ただし、現在は拡張モードで動作し、いくつかのオプションが提供されます。


1 - change PIN
2 - unblock PIN
3 - change Admin PIN
4 - set the Reset Code
Q - quit

Your selection?

管理者PINの変更を選択し、標準スキームに従って管理者パスワードを設定します。デフォルトの管理PINは12345678です。

デバイスのセキュリティについて


もちろん、3ドルでは200ドル以上のキーレベルのセキュリティは実現できません。ただし、組み立てられたデバイスは、エントリレベルのキーと見なすことができます。私の知る限り、Nitrokey Startファクトリーキーは同じファームウェアを使用しています。いずれの場合でも、このようなデバイスを使用すると、セキュリティが少なくとも1レベル高くなります。したがって、これはハードウェアキーの使用を開始するための優れた方法です。

次に、このキーを紛失した場合にどうなるかについて少し話しましょう。キー自体はピンコードによって保護されています。数回失敗した後、キーはロックされます。デバイス自体のメモリは読み取りから保護されているため、デバイスに書き込まれたキーを直接読み取ることはできません。
チップ自体への攻撃にはいくつかの種類があります（たとえば、メモリを含むチップ内の任意の場所で電圧を測定する微小電極の準備と接続）が、非常に高価です。

もう一度繰り返しますが、これはエントリーレベルの優れたオプションであり、それを使用しても、セキュリティはまったく新しいレベルに到達します。
