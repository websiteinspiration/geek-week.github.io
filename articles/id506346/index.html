Berikut adalah contoh fungsi untuk mengirim surat:
public function email($emails, $body, $subject)
{
    if (empty($emails)) {
        return false;
    }

    $prefix = getenv('NATS_PREFIX');
    $encoder = new JSONEncoder();
    $options = new ConnectionOptions([
        'token' => getenv('NATS_TOKEN')
    ]);
    $client = new EncodedConnection($options, $encoder);

    try {
        $client->connect();
        $client->publish(
            $prefix . 'email',
            [
                'Emails' => $emails,
                'HtmlBody' => $body,
                'TextBody' => strip_tags($body),
                'Subject' => $subject,
                'Sender' => 'info@' . getenv('DOMAIN_NAME')
            ]
        );
    } catch (\Exception $exception) {
        return false;
    }

    return true;
}

Produk kami berorientasi ke Eropa, oleh karena itu harganya dalam Euro.

Kesulitan apa yang kita hadapi:

  1. Anda dapat mengandalkan gateway pembayaran jari yang memiliki terminal mata uang. Apa itu? Ini adalah hal yang memungkinkan Anda menerima pembayaran dalam EURO, bukan rubel. Mengapa ini sangat penting? Ya, karena Eropa waspada terhadap Rusia, ada banyak scammer. Saat membayar, rubel akan mengurangi konversi dan menambahkan tiket ke dukungan, di mana Anda harus menjelaskan siapa Anda dan apa Anda.
  2. Koordinasi terminal mata uang membutuhkan waktu. Bersiaplah dari 5 hari ke atas. Dalam praktiknya, sekitar 10. Karena Anda akan diperiksa dengan cermat, dan orang Eropa enggan untuk karantina. Juga, pertanyaan akan muncul dari bank domestik kami, mengapa Anda membutuhkannya?
  3. 3d secure belum dibatalkan (SMS, yang dilengkapi dengan kode konfirmasi pembayaran). Karena itu, jika sebuah bank di Eropa tidak mendukung kemungkinan ini untuk klien di Eropa, maka ia tidak akan dapat membayar. Dalam hal ini, Anda harus mengalahkan klien dengan pena di PayPal
  4. Koordinasi situs hanya terjadi ketika semua konten sudah ada di sana. Itu situs ini sepenuhnya siap, termasuk semua dokumen hukum seperti penawaran. Karena kami tidak memiliki toko online dan secara umum solusi atipikal - kami beralih ke pengacara untuk membuat penawaran untuk produk kami dan terjemahan berikutnya

Kebetulan kami pertama kali menerima pembayaran dalam rubel. Setelah menyetujui gateway pembayaran, mereka beralih ke euro. Ini sedikit mengurangi konversi di awal.

Organisasi Pengaliran


Jika semuanya sederhana dengan otorisasi, dan penagihan dapat dikacaukan sendiri. Tugas seperti itu sering ditemukan di pengembang statistik rata-rata. Tetapi dengan streaming, semuanya tidak sejelas seperti yang terlihat pada pandangan pertama. Apa yang salah?

  1. Kami tidak pernah berurusan dengannya. Saya membutuhkan server + klien untuk guru
  2. Gambar perlu dicerminkan, mis. melakukan tampilan vertikal (fitur video tarian)
  3. Kami tidak tahu persis berapa banyak peserta di kamp. Dan kami tidak dapat membeli akun untuk bulan \ tahun \ jumlah lalu lintas di mana saja (dalam solusi streaming yang sudah jadi). Itu akan menelan semua keuntungan
  4. Saya membutuhkan perlindungan iframe terhadap pencurian konten setidaknya oleh perujuk

Sebagai solusinya, kami memilih aplikasi larix broadcastr -> nginx + rtmp -> cdn -> klien.

Selama pencarian, aplikasi Larix Broadcaster untuk menyiarkan RTMP dari perangkat seluler ternyata paling andal dan stabil. Pada server virtual kecil yang terpisah, kami menaikkan nginx + rtmp untuk streaming lebih jauh ke CDN.

Skema ini adalah yang termurah. Jauh lebih murah daripada menggunakan layanan platform streaming yang sudah jadi. Anda tidak perlu membeli akun premium \ gold \ super selama satu bulan \ tahun. Anda hanya membayar untuk lalu lintas yang dihabiskan. Ini adalah solusi sempurna dalam situasi kami.

Kesimpulan


Apa hasilnya?

Kami menghabiskan 2 kamp online.

Hanya menerima umpan balik positif dari pengguna pertama.

Di masa karantina yang sulit, inilah tepatnya yang dibutuhkan orang di rumah.

Total pendapatan berjumlah lebih dari 1.000.000 rubel .

Jika kita berbicara sedikit tentang castdev, lalu berfokus pada umpan balik dari kamp pertama, kami menyelesaikan sistem

  1. Mereka memungkinkan untuk membeli tiket untuk satu hari (dan bukan hanya tiket penuh) - ini meningkatkan pendapatan, karena banyak yang tidak ingin membeli tiket penuh, tetapi hanya ingin membeli kelas master satu atau dua guru
  2. Kami membuat obrolan sederhana di dekat pemutar video di socket.io - ini menambahkan interaktivitas ke pelajaran dan memungkinkan kami untuk mendapatkan umpan balik waktu nyata
  3. Karena kamp internasional dan banyak orang menjadi bingung dalam zona waktu - kami menambahkan kemampuan untuk mengubah zona waktu dalam jadwal dan menonton jadwal sesuai dengan zona waktu kami

Kesimpulan apa yang bisa dibuat:

  1. Tidak perlu memiliki investasi awal dan tim besar untuk menguji ide tersebut
  2. Pimpin pengembangan secara fleksibel, dengarkan pengguna Anda. Tambahkan hanya apa yang benar-benar dibutuhkan orang
  3. Jangan langsung menulis monolit besar dengan banyak fungsi, tidak perlu menguji hipotesis. Jangan simpan itu. hutang. Belajarlah untuk menemukan keseimbangan antara kode "buruk" dan fungsionalitas "berlebihan". Gunakan setiap alat untuk tujuan yang dimaksudkan.
  4. Gunakan pengalaman teman-teman Anda. Bagaimanapun, sebagian besar tugas telah diselesaikan. Cukup menemukan alat yang diperlukan, melakukan penelitian, menggabungkannya menjadi satu sistem
