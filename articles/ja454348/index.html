今日は、Reactフックを調べ、アプリケーションのグローバル状態を管理するためのカスタムフックを開発します。これは、Redux実装よりも簡単で、Context APIよりも生産的です。

React Hooks Basics


フックに慣れている場合は、この部分をスキップできます。

useState（）


フックが出現する前は、機能コンポーネントにはローカル状態を設定する機能がありませんでした。状況は出現とともに変化しましたuseState()。



この呼び出しは配列を返します。その最初の要素は、状態値へのアクセスを提供する変数です。2番目の要素は、状態を更新し、変更を反映するためにコンポーネントを再描画する関数です。

import React, { useState } from 'react';

function Example() {
  const [state, setState] = useState({counter:0});
  const add1ToCounter = () => {
    const newCounterValue = state.counter + 1;
    setState({ counter: newCounterValue});
  }

  return (
    <div>
      <p>You clicked {state.counter} times</p>
      <button onClick={add1ToCounter}>
        Click me
      </button>
    </div>
  );
}

useEffect（）


クラスコンポーネントは、などのライフサイクルメソッドを使用して副作用に応答しますcomponentDidMount()。フックをuseEffect()使用すると、機能コンポーネントでも同じことができます。

デフォルトでは、各再描画後に効果がトリガーされます。ただし、特定の変数の値を変更し、配列の形式で2番目のオプションのパラメーターを渡した後でのみ、それらが実行されるようにすることができます。

//    
useEffect(() => {
  console.log('     ');
});

//   
useEffect(() => {
  console.log('     valueA');
}, [valueA]);

同様の結果を得るにはcomponentDidMount()、2番目のパラメーターとして空の配列を渡します。空の配列の内容は常に変更されないままであるため、エフェクトは一度だけ実行されます。

//    
useEffect(() => {
  console.log('    ');
}, []);

状態共有


フックの状態は、クラスコンポーネントの状態と同じように機能することがわかりました。各コンポーネントインスタンスには、独自の内部状態があります。

コンポーネント間で状態を共有するために、独自のフックを作成します。



アイデアは、リスナーの配列と1つの状態のみを作成することです。コンポーネントの状態が変化するたびに、サブスクライブされたすべてのコンポーネントはそれらを呼び出し、getState()これにより更新されます。

これはuseState()、カスタムフック内で呼び出すことで実現できます。ただし、関数を返す代わりにsetState()、リスナーの配列に追加して、状態オブジェクトを内部的に更新してすべてのリスナーを呼び出す関数を返します。

最初のバージョン


import { useState, useEffect } from 'react';

let listeners = [];
let state = { counter: 0 };

const setState = (newState) => {
  state = { ...state, ...newState };
  listeners.forEach((listener) => {
    listener(state);
  });
};

const useCustom = () => {
  const newListener = useState()[1];
  useEffect(() => {
    listeners.push(newListener);
  }, []);
  return [state, setState];
};

export default useCustom;

コンポーネントをアンマウントする直前に関数を呼び出す


useEffect(function, [])空の配列 を使用した呼び出しが同じように機能することはすでにわかっていcomponentDidMount()ます。ただし、最初のパラメーターで渡された関数が別の関数を返す場合、コンポーネントをアンマウントする直前に2番目の関数が呼び出されます。のようにcomponentWillUnmount()。

したがって、2番目の関数のコードでは、リスナーの配列からコンポーネントを削除するロジックを記述できます。

const useCustom = () => {
  const newListener = useState()[1];
  useEffect(() => {
    //    
    listeners.push(newListener);
    return () => {
      //    
      listeners = listeners.filter(listener => listener !== newListener);
    };
  }, []);
  return [state, setState];
};

2番目のバージョン


このアップデートに加えて、次のことも計画しています。

  • Reactパラメータを渡してインポートを削除します。
  • customHookではなく、指定されたでcustomHookを返す関数をエクスポートしますinitalState。
  • store値stateと関数を含むオブジェクトを作成しますsetState()。
  • 通常のものでで矢印の機能を交換setState()し、useCustom()それが関連することができるようにstoreとthis。

function setState(newState) {
  this.state = { ...this.state, ...newState };
  this.listeners.forEach((listener) => {
    listener(this.state);
  });
}

function useCustom(React) {
  const newListener = React.useState()[1];
  React.useEffect(() => {
    //    
    this.listeners.push(newListener);
    return () => {
      //    
      this.listeners = this.listeners.filter(listener => listener !== newListener);
    };
  }, []);
  return [this.state, this.setState];
}

const useGlobalHook = (React, initialState) => {
  const store = { state: initialState, listeners: [] };
  store.setState = setState.bind(store);
  return useCustom.bind(store, React);
};

export default useGlobalHook;

アクションをコンポーネントから分離する


複雑な状態管理ライブラリを使用したことがある場合、コンポーネントからグローバル状態を操作することは良い考えではないことがわかります。

状態を変更するアクションを作成して、ビジネスロジックを分離する方が正しいでしょう。したがって、最新バージョンのパッケージで、コンポーネントへのアクセスではなくsetState()、一連のアクションへのアクセスを提供する必要があります。

これを行うには、useGlobalHook(React, initialState, actions)3番目の引数を指定します。コメントをいくつか追加したいだけです。

  • アクションはにアクセスできますstore。このようにして、アクションは内容を読み取り、store.state呼び出しによって状態を更新し、store.setState()さらにを参照して他のアクションを呼び出すこともできますstore.actions。
  • お粥を避けるために、アクションオブジェクトにはサブオブジェクトを含めることができます。したがって、actions.addToCounter(amountすべてのekshenomカウンターを持つサブオブジェクト内で）を移動できますactions.counter.add(amount)。

最終版


次のスニペットは、NPMパッケージの現在のバージョンですuse-global-hook。

function setState(newState) {
  this.state = { ...this.state, ...newState };
  this.listeners.forEach((listener) => {
    listener(this.state);
  });
}

function useCustom(React) {
  const newListener = React.useState()[1];
  React.useEffect(() => {
    this.listeners.push(newListener);
    return () => {
      this.listeners = this.listeners.filter(listener => listener !== newListener);
    };
  }, []);
  return [this.state, this.actions];
}

function associateActions(store, actions) {
  const associatedActions = {};
  Object.keys(actions).forEach((key) => {
    if (typeof actions[key] === 'function') {
      associatedActions[key] = actions[key].bind(null, store);
    }
    if (typeof actions[key] === 'object') {
      associatedActions[key] = associateActions(store, actions[key]);
    }
  });
  return associatedActions;
}

const useGlobalHook = (React, initialState, actions) => {
  const store = { state: initialState, listeners: [] };
  store.setState = setState.bind(store);
  store.actions = associateActions(store, actions);
  return useCustom.bind(store, React);
};

export default useGlobalHook;
