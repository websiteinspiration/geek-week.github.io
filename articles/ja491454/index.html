Linux NVMeリグ

良い一日。

1つのシステムで複数のNVMe SSDを使用する場合、コミュニティの注目をLinuxの特徴的な機能に向けたかったのです。NVMeからソフトウェアRAIDアレイを作成したい人にとっては特に重要です。

以下の情報があなたのデータを保護し、迷惑なエラーからあなたを救うことを願っています。

ブロックデバイスを操作するときは、次のLinuxロジックに慣れています。
デバイスが/ dev / sdaと呼ばれる場合、その上のパーティションは/ dev / sda1、/ dev / sda2などになります。
SMART属性を表示するには、smartctl -a / dev / sdaなどを使用しますが、フォーマットして、/ dev / sda1などの配列にパーティションを追加します。

/ dev / sda1が/ dev / sdaにあるという公理に慣れています。そして、ある日SMARTが/ dev / sdaがほとんど死んでいることを示した場合、交換のためにRAIDアレイから捨てるのは/ dev / sda1です。

NVMe名前空間を使用する場合、このルールは機能しないことがわかります。証明：

nvme list && ( smartctl -a /dev/nvme0 && smartctl -a /dev/nvme1  && smartctl -a /dev/nvme2 ) | grep Serial

シリアル番号を比較するときの注意深い読者は、/ dev / nvme1n1が実際に/ dev / nvme2に配置されていること、およびその逆に気付くでしょう。

追伸

RAIDアレイから最後のライブNVMe SSDを削除しないでください。
