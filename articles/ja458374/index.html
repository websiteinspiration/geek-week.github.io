IBM Watsonサービスの例としてのTJBOT

こんにちは、ハブル！ 2019年の春に、次のThink Developers Workshopが開催され、誰もがIBM Watson Servicesを実行するTJBotaボール紙ロボットを組み立てることができました。そのようなロボットを組み立てる方法と方法に関する詳細な説明、ワトソンサービスの認知機能のいくつかを示す便利なリンクと簡単なレシピ、IBMのモスクワオフィスでのワトソンサービスに関する7月の2つのセミナーの小さな発表があります。



IBM Watsonサービスは、自然言語を処理し、画像を認識し、学習できる認知システムです。アプリケーションでこれらのサービスを便利に使用するために、APIが存在します。

TJBotは、Watsonサービスへのアクセスを支援するために設計されたオープンソースプロジェクトです。これは、すべてのラズベリーpiおよび既製の人工知能が作成できるロボットです。TJBotはレシピで復活させることができます。

レシピは、TJBotをSpeech to Text、Visual Recognition、Language TranslatorなどのWatsonサービスに接続するのに役立つステップバイステップの指示です。レシピはRaspberry Piに基づいています。

プログラムの準備


最初のステップはパッケージをインストールすることです：

curl -sL http://ibm.biz/tjbot-bootstrap | sudo sh –

githubから既製のレシピをダウンロードします。

git clone https://github.com/ibmtjbot/tjbot.git

レシピのあるディレクトリに移動します。

cd tjbot / recipes / speech_to_text

このフォルダーには、config.js構成ファイルと、実行可能スクリプトstt.jsを含むファイルが含まれています。

npmをインストールします。

sudo apt-get install npm

Watsonサービス接続


Watsonサービスを使用するには、以下の手順を実行する必要があります。

このサイトに移動します。



登録してディレクトリに移動します。カタログでは「スピーチからテキストへ」を探しています。スピーチからテキストへの変換は、スピーチをテキストに変換するために使用されるサービスです。 APIへのアクセスについては、こちらをご覧ください。



テキストから音声への変換と視覚認識も、画像認識で作業するときに必要になります。スピーチトゥテキストをクリックすると、このコンポーネントの説明と使用計画が記載されたページが表示されます。



無料プランで十分です。 [作成]をクリックし、左側のメニューで[サービス資格情報]に移動します。



ここから、認証情報とAPIKEYをコピーして、config.jsファイルに貼り付ける必要があります。

// Create the credentials object for export
exports.credentials = {};

// Watson Speech to Text
// https://www.ibm.com/watson/services/speech-to-text/
exports.credentials.speech_to_text = {
  "apikey": "...",
  "iam_apikey_description": "...",
  "iam_apikey_name": "...",
  "iam_role_crn": "...",
  "iam_serviceid_crn": "...",
  "url": "https://gateway-lon.watsonplatform.net/speech-to-text/api"
};

ここで、別のWatsonサービスを追加する場合は、構成ファイルで、次の構造にラップされた各サービスにapikeyおよびurlを含むブロックを追加する必要があります。

exports.credentials.[ text_to_speech/visual_recognition/speech_to_text ] = { … };

TjBotaの復活


ボット実行可能スクリプトstt.jsを含むファイルを考えます。Watsonサービスを使用せずにボットの動作をチェックする既製のdiskoParty（）関数があります。この関数は、ボットダイオードを異なる色で点滅させます。

function discoParty() {
    for (i = 0; i < 30; i++) {
        setTimeout(function() {
            var randIdx = Math.floor(Math.random() * tjColors.length);
            var randColor = tjColors[randIdx];
            tj.shine(randColor);
        }, i * 250);
    }
}

discoParty();

同じスクリプトに、開発者のスピーチを使用してダイオードの色を切り替えることができる関数があります。

スクリプトを実行します。

sudo node stt.js

ダイオードの色を青に切り替えるにはボットに「ライトブルーをオン」に、ダイオードをオンにするには「ライトをオン」に、オフにするには「ライトをオフ」にボットに伝えます。認識のためにサポートされている色（これまでのところ英語のみがサポートされています）：黄色、緑、オレンジ、紫、マゼンタ、赤、青、水色、白。

TjBotには多くの基本機能があります。たとえば、サーボをチェックするには、tj.wave（）関数を使用できます。これにより、ボットがハンドルの波で挨拶します。これらの短い説明関数はここにあります。

次に、視覚認識とテキスト読み上げの両方を使用する次のシナリオを考えます。

Text to Speechは、さまざまな音声、キー、言語を使用して、印刷されたテキストを音声に変換するサービスです。そのAPIは次のリンクにあります。視覚認識サービスを使用すると、画像に表示されているものを説明できます。おおよその年齢と性別、食べ物、料理、オブジェクトの定義を持つ人々の顔を認識し、類似の画像を検索できます。このサービスのAPIはここにあります。これらのサービスを使用して、ボットに見たり話したりすることを教えます。カメラから受け取った画像に基づいて、Watsonサービス（視覚認識）は、jsonオブジェクトに画像タグを付けた応答を送信します。また、テキストを音声に変換することで、それらに声をかけることができます。

まず、cloud.ibm.comに資格情報を作成します。それらをコピーして、config.js構成ファイルに貼り付けます。

次に、実行可能スクリプトstt.jsを編集します。その中に次の行があります。

// these are the hardware capabilities that our TJ needs for this recipe
var hardware = ['led', 'microphone'];

ハードウェアアレイには、使用されたボットデバイスが含まれています。スクリプトでサーボを使用する場合は、「サーボ」を配列にサインインする必要があります。カメラが必要な場合は、「カメラ」を配列に追加して、「スピーカー」と書く列を使用します。
したがって、スクリプトではそれぞれ列とカメラを使用し、これをハードウェア配列にサインインします。

// these are the hardware capabilities that our TJ needs for this recipe
var hardware = ['led', 'servo', 'camera', 'speaker', 'microphone'];

// set up TJBot's configuration
var tjConfig = {
    log: {
        level: 'verbose'
    },
    speak: {
        language: 'en-US', // see TJBot.prototype.languages.speak
        voice: undefined, // use a specific voice; if undefined, a voice is chosen based on robot.gender and speak.language
        speakerDeviceId: "plughw:0,0" // plugged-in USB card 1, device 0; see aplay -l for a list of playback devices
    },
    listen: {
        microphoneDeviceId: "plughw:1,0", // plugged-in USB card 1, device 0; see arecord -l for a list of recording devices
        inactivityTimeout: -1, // -1 to never timeout or break the connection. Set this to a value in seconds e.g 120 to end connection after 120 seconds of silence
        language: 'en-US' // see TJBot.prototype.languages.listen
    },
};

tj-libraryの基本関数から、関数tj.see（）およびtj.speak（）が必要です。

tj.see（）関数は写真を作成し（オブジェクトはtmpフォルダーに保存されます）、Watsonサービスを使用してクラウドに送信し、画像を分析して、タグ-写真を説明する単語で構成されるjsonオブジェクトを生成します（さまざまな説明と信頼度を選択できます）。これらのタグの信頼性の割合。サービスの応答内容をコンソールに出力します。

tj.speak（）関数は、Watsonサービスを使用してテキストをサウンドファイルに変換し、再生できます。また、Watsonサービスを使用している場合、写真で人物が検出された場合、TJBotがペンを振ります。

// instantiate our TJBot!
var tj = new TJBot(hardware, tjConfig, credentials);

tj.see().then(function(objects){
    var tags = objects.map(function(object){
        return object.class;
    });
    if (tags.includes('person')){
        tj.wave();
    }
    console.log(tags);
    for(var i=0;i<tags.length;i++){
        tj.speak(tags[i]);
    }
});
