8.重複する要素をコンポーネントに変換する


重複する要素を再利用に適したコンポーネントに変換することは、そのような要素の「コンポーネント化」と呼ぶことができます。

各開発者には、Reactコードを重複して作成する理由があります。これは故意の行動かもしれませんし、偶然かもしれません。

アプリケーションに同じコードフラグメントが現れる理由が何であれ、プログラマーは状況を改善する方法を考える必要があります。

たとえば、誰かが重複を取り除く習慣をつけていない場合、それらはおそらく彼のプロジェクトで何度も発生します。これを行うのはどのようなチームプレーヤーですか？重複するコードに遭遇すると混乱する同僚の将来の生活を複雑にするだけです。同様のコードフラグメントを編集する必要がある場合、特別な「ギフト」を受け取ります。

次の例を見て、それを改善する方法を考えてください。

const SomeComponent = () => (

    <Divider />

あなたは今からグリッド設定を変更する必要がある場合xs={12} sm={6}にxs={12} sm={4}は、このタスクは、特に快適ではありません。実際には、このためにコードを4か所で編集する必要があります。

コンポーネントアプローチの優れている点は、コードを1か所だけ変更することで、上記と同様の問題を解決できることです。私たちの場合、この変更はグリッドが使用されているすべての場所に反映されます。

ここで示されている最小限のレベルのコード変換でさえ、このコードを読んでサポートするという点ではるかに便利です。同時に、彼は彼に割り当てられたタスクを解決するための完全に適切な方法です。

9.コンポーネントをできるだけシンプルに保つよう努めます。


販売アプリケーションで作業しているときに、コンポーネントを単純化するために努力する必要はなく、コンポーネントが複雑になりすぎる状況を回避する必要に出くわしたことがあります。

これは、不必要に複雑なコンポーネントの例です。次のファイルで表されますConfirmAvailability.js。

import React from 'react'
import Grid from '@material-ui/core/Grid'
import Typography from '@material-ui/core/Typography'
import MenuItem from '@material-ui/core/MenuItem'
import Select from '@material-ui/core/Select'
import Time from 'util/time'

/**
 *     .    ,     , ,  , 
 * ,    .        .     -     .
 *
 * :     Date().getTimezoneOffset().     :
 *      1.      .
 *      2.     UTC -     ,   UTC   .  ,      
 *     UTC.
 */
export default class TimeZonePicker extends React.Component {
  state = {
    time: new Date(),
    offset: -(new Date().getTimezoneOffset() / 60),
  }

  componentDidMount() {
    this.props.setOffset(this.state.offset)
  }

  handleChange = (event) => {
    const d = new Date()
    d.setTime(
      d.getTime() +
        d.getTimezoneOffset() * 60 * 1000 +
        event.target.value * 3600 * 1000,
    )
    this.setState({
      time: d,
      offset: event.target.value,
    })
    this.props.setOffset(event.target.value)
  }

  render() {
    const timezones = []
    for (let i = -12; i <= 14; i++) {
      timezones.push(
        <MenuItem key={i} value={i}>
          {i > 0 ? '+' : null}
          {i}
        </MenuItem>,
      )
    }

    return (
      <React.Fragment>
        <Grid container justify="space-between">
          <div>
            <Typography>Current time</Typography>
            <Typography variant="h6" gutterBottom>
              {Time.formatTime(this.state.time)}
            </Typography>
          </div>
          <div>
            <Typography>Set timezone</Typography>
            <Select value={this.state.offset} onChange={this.handleChange}>
              {timezones}
            </Select>
          </div>
        </Grid>
      </React.Fragment>
    )
  }
}

このコンポーネントは単純なメカニズムとして考えられましたが、関連性の高いロジックが含まれているため、いくつかの問題を解決します。このコードが作成された時点では、Reactフックはまだリリースされていませんでしたが、Reactには高次コンポーネントやレンダープロップなどのテクノロジーが含まれていました。これは、これらのパターンの1つを使用してコンポーネントを単純化できることを意味します。これにより、既存の機能を変更せずにコンポーネントを簡素化するアプローチを示すことができます。

10. useStateを複雑にする場合はuseReducerを使用します


プロジェクトで処理しなければならない状態の断片が多いほど、使用は複雑になりますuseState。

たとえば、これは次のようになります。

import React from 'react'
import axios from 'axios'

const useFrogs = () => {
  const [fetching, setFetching] = React.useState(false)
  const [fetched, setFetched] = React.useState(false)
  const [fetchError, setFetchError] = React.useState(null)
  const [timedOut, setTimedOut] = React.useState(false)
  const [frogs, setFrogs] = React.useState(null)
  const [params, setParams] = React.useState({ limit: 50 })
  const timedOutRef = React.useRef()

  function updateParams(newParams) {
    if (newParams != undefined) {
      setParams(newParams)
    } else {
      console.warn(
        'You tried to update state.params but the parameters were null or undefined',
      )
    }
  }

  function formatFrogs(newFrogs) {
    const formattedFrogs = newFrogs.reduce((acc, frog) => {
      const { name, age, size, children } = frog
      if (!(name in acc)) {
        acc[name] = {
          age,
          size,
          children: children.map((child) => ({
            name: child.name,
            age: child.age,
            size: child.size,
          })),
        }
      }
      return acc
    }, {})
    return formattedFrogs
  }

  function addFrog(name, frog) {
    const nextFrogs = {
      ...frogs,
      [name]: frog,
    }
    setFrogs(nextFrogs)
  }

  function removeFrog(name) {
    const nextFrogs = { ...frogs }
    if (name in nextFrogs) delete nextFrogs[name]
    setFrogs(nextFrogs)
  }

  React.useEffect(() => {
    if (frogs === null) {
      if (timedOutRef.current) clearTimeout(timedOutRef.current)

      setFetching(true)

      timedOutRef.current = setTimeout(() => {
        setTimedOut(true)
      }, 20000)

      axios
        .get('https://somefrogsaspi.com/api/v1/frogs_list/', { params })
        .then((response) => {
          if (timedOutRef.current) clearTimeout(timedOutRef.current)
          setFetching(false)
          setFetched(true)
          if (timedOut) setTimedOut(false)
          if (fetchError) setFetchError(null)
          setFrogs(formatFrogs(response.data))
        })
        .catch((error) => {
          if (timedOutRef.current) clearTimeout(timedOutRef.current)
          console.error(error)
          setFetching(false)
          if (timedOut) setTimedOut(false)
          setFetchError(error)
        })
    }
  }, [])

  return {
    fetching,
    fetched,
    fetchError,
    timedOut,
    frogs,
    params,
    addFrog,
    removeFrog,
  }
}

export default useFrogs

このコードを使用するように翻訳すると、誰もがこれで作業することがはるかに便利になりますuseReducer。

import React from 'react'
import axios from 'axios'

const initialFetchState = {
  fetching: false
  fetched: false
  fetchError: null
  timedOut: false
}

const initialState = {
  ...initialFetchState,
  frogs: null
  params: { limit: 50 }
}

const reducer = (state, action) => {
  switch (action.type) {
    case 'fetching':
      return { ...state, ...initialFetchState, fetching: true }
    case 'fetched':
      return { ...state, ...initialFetchState, fetched: true, frogs: action.frogs }
    case 'fetch-error':
      return { ...state, ...initialFetchState, fetchError: action.error }
    case 'set-timed-out':
      return { ...state, ...initialFetchState, timedOut: true }
    case 'set-frogs':
      return { ...state, ...initialFetchState, fetched: true, frogs: action.frogs }
    case 'add-frog':
      return { ...state, frogs: { ...state.frogs, [action.name]: action.frog }}
    case 'remove-frog': {
      const nextFrogs = { ...state.frogs }
      if (action.name in nextFrogs) delete nextFrogs[action.name]
      return { ...state, frogs: nextFrogs }
    }
    case 'set-params':
      return { ...state, params: { ...state.params, ...action.params } }
      default:
        return state
  }
}

const useFrogs = () => {
  const [state, dispatch] = React.useReducer(reducer, initialState)
  const timedOutRef = React.useRef()

  function updateParams(params) {
    if (newParams != undefined) {
      dispatch({ type: 'set-params', params })
    } else {
      console.warn(
        'You tried to update state.params but the parameters were null or undefined',
      )
    }
  }

  function formatFrogs(newFrogs) {
    const formattedFrogs = newFrogs.reduce((acc, frog) => {
      const { name, age, size, children } = frog
      if (!(name in acc)) {
        acc[name] = {
          age,
          size,
          children: children.map((child) => ({
            name: child.name,
            age: child.age,
            size: child.size,
          })),
        }
      }
      return acc
    }, {})
    return formattedFrogs
  }

  function addFrog(name, frog) {
    dispatch({ type: 'add-frog', name, frog })
  }

  function removeFrog(name) {
    dispatch({ type: 'remove-frog', name })
  }

  React.useEffect(() => {
    if (frogs === null) {
      if (timedOutRef.current) clearTimeout(timedOutRef.current)

      timedOutRef.current = setTimeout(() => {
        setTimedOut(true)
      }, 20000)

      axios
        .get('https://somefrogsaspi.com/api/v1/frogs_list/', { params })
        .then((response) => {
          if (timedOutRef.current) clearTimeout(timedOutRef.current)
          const frogs = formatFrogs(response.data)
          dispatch({ type: 'set-frogs', frogs })
        })
        .catch((error) => {
          if (timedOutRef.current) clearTimeout(timedOutRef.current)
          console.error(error)
          dispatch({ type: 'fetch-error', error })
        })
    }
  }, [])

  return {
    fetching,
    fetched,
    fetchError,
    timedOut,
    frogs,
    params,
    addFrog,
    removeFrog,
  }
}

export default useFrogs

このアプローチはを使用するよりもおそらくクリーンではありませんがuseState、コードを見るとわかるように、新しいコードは保守が簡単です。これはuseReducer、これらのすべての操作が内の1つの場所で定義されているため、適用時に、フックのさまざまな部分での状態の更新についてプログラマが心配する必要がないためreducerです。使用する

コードのバージョンではuseState、ロジックを記述するだけでなく、状態の次の部分が何であるかを見つけるために、フック内で関数を宣言する必要があります。そしてuseReducerこれを使うときあなたはする必要はありません。代わりに、すべてが関数に分類されますreducer。適切なタイプのアクションをトリガーするだけでよく、それが実際に心配する必要のあるすべてです。

11.問題のある状況で関数宣言を使用する


この推奨事項を使用する良い例は、クリーニングメカニズムを作成することですuseEffect。

React.useEffect(() => {
  setMounted(true)

  return () => {
    setMounted(false)
  }
}, [])

経験豊富なReact開発者は、返される関数の役割を知っているので、そのようなコードを簡単に理解できます。ただし、このコードにあまり詳しくない人がコードを読むことを想像する場合useEffectは、コード内で意図をできるだけ明確に表現することをお勧めします。意味のある名前を付けることができる関数宣言を使用することです。たとえば、このコードは次のように書き換えることができます。

React.useEffect(() => {
  setMounted(true)

  return function cleanup() {
    setMounted(false)
  }
}, [])

このアプローチにより、返された関数が果たす役割を明確に説明できます。

12. Prettierを使用する


Prettierは、個々の開発者やチームが、コードのフォーマットに対して一貫したアプローチを維持するのに役立ちます。このツールは、時間と労力を節約するのに役立ちます。プログラムのスタイルを議論する理由の数を減らすことにより、コードレビューの実行を簡素化します。Prettierはまた、プログラマーにクリーンなコード作成手法を使用するように勧めています。このツールによって適用されるルールは編集可能です。その結果、誰もが自分の好みに合わせてカスタマイズできることがわかりました。

14.コードを書くときは、要素の配置の特定の順序に従ってください。


コードを書くとき、いくつかのコマンドを特定の順序で配置することを好みます。たとえば、ファイルをインポートするときにこれを行います（唯一の例外はimportですreact）。

このコードを見ると、特別な注文はここでは守られていないと思われるかもしれません。結局のところ、インポートされたエンティティはアルファベット順に並べ替えさえされていません。しかし、アルファベット順に並べることは、私が使用するコマンドの順序付けスキームの一部にすぎません。

プロジェクトのコードをクリーンに保つために、私は次のルールを使用します。

  1. Reactをインポートします。
  2. ライブラリをアルファベット順でインポートします。
  3. プロジェクトからエンティティをインポートする絶対コマンド（アルファベット順）。
  4. 相対インポートコマンド（アルファベット順）。
  5. コマンドを表示しますimport * as。
  6. コマンドを表示しますimport './<some file>.<some ext>'。
