JsSIP + FreePBXに基づくWeb SIPクライアント

FreePBXにはWeb Sipクライアントがあります。それがどのように機能するのか疑問に思いました。WebRTCテクノロジーを学び始めて、私は3つのライブラリーに出会いました-これらはJsSIP、そのフォークSIP.js、そしてsipml5です。



JsSIPとSIP.jsは初めて実行できなかったため、十分な経験がありませんでした。しかし、sipml5が最も簡単に使用できました。テストバージョンをダウンロードした後、最初の呼び出しを行うことがわかりました。



3つのライブラリはすべて動作しています。それぞれに長所と短所があります。3つのライブラリすべてをテストしたので、JsSIPを選択しました。

トレーニング



Webクライアントは、暗号化がサポートされているページでのみ機能します。自己署名証明書を作成する必要があります。これはどのように行われるのですか？インターネット上に既成の記事があります。



Firefoxでは、自己署名証明書を使用できます。Chromeで作業するには、最初にルート証明書を実行する必要があります。これに基づいて、電話が機能するための証明書にすでに署名しています。ルート証明書を証明書ストアにインポートする必要があります。



FreePBXでのWeb SIPクライアントのサポートの有効化



管理者->ユーザー管理->タブ「グループ」->タブ「UCP」->タブ「電話」「電話を

有効にする」プロパティで、「はい」を選択します。



FreePBXに内部番号（たとえば200）があった場合、Web SIPクライアントをアクティブ化すると、プレフィックスが99の追加の番号が表示されます。99200。



クライアントブラウザで、自己署名証明書の使用を許可した後も、アスタリスク証明書の許可を与える必要があります。これを行うには、ブラウザに入力します 



https://192.168.1.70:8089/ws


アスタリスクが実行されているIPアドレスを指定する必要があります。



JsSIPソースの編集



JsSIPライブラリのソースでは、transport = wsをtransport = wssに置き換える必要があります。この編集がないと、クライアントは機能しません。Sip.jsでは何も編集する必要はありません。この瞬間のために用意されています。

var socket = new JsSIP.WebSocketInterface('wss://192.168.1.70:8089/ws');

ua.on('disconnected', function(e) { /*   */ });

var eventHandlers = {

概要



Web SIPクライアントを使用してから2年間、ブラウザの更新時に問題が発生しました。



たとえば、Firefoxのバージョン63では、セッションが中断されないようにSDPプロトコルを変更する必要がありました。ブラウザーはパラメーターa = mid：0なしで通信を停止しました。最新バージョンでは関係がなくなり、エラーが修正されました。それ以降のバージョンではそのような問題はありません。



Firefoxのバージョン65ではref。通話中に対談者の声が消え始め、LANにはそのような問題はありませんでした。問題は、ゲートウェイを介して呼び出すときだけです。クロムではこのような問題は見られず、すべてが安定して動作します。



ほとんどのユーザーはFirefoxで作業するため、ライブラリ内のSDPプロトコルの再ネゴシエーションを削除する必要がありました。回答後の音の損失の問題は、もはや気になりません。
