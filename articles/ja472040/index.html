赤黒の木のバランス調整-3つのケース

バイナリ検索ツリーは、クイック検索機能を備えたアイテムを格納するためのデータ構造です。アイデアはシンプルで独創的です。これが単純さが終わり、ツリーのバランスをとる複雑な問題が始まるため、ツリーが長いブランチにならないようにします。



この記事では、定義を定義し、要素を赤黒ツリーに配置するためのルールをリストし、バランシングアルゴリズムを検討し、例で述べたことをまとめます。「開発者のためのアルゴリズム」コースでは、このトピックと他のタイプの二分探索木についてさらに詳しく学習します。



赤黒ツリーは、ノードの数からのツリーの高さの対数の増加と、ノードの追加、削除、検索などの検索ツリーの基本操作の迅速な実行を保証する自己バランス型の二分探索ツリーです。

赤黒の木は完全にバランスが取れていません。場所によっては、高さがほぼ2倍に変わることがあります。このような仮定は、要素の検索速度に漸近的に影響することはありませんが、要素を追加するたびにツリーを「振る」必要がないため、新しい要素の配置を大幅にスピードアップします。 。



赤黒木に要素を配置するためのルール


  1. 各ノードは赤または黒で、NILリーフは常に黒です。
  2. 木の根は常に黒です
  3. 各赤いノードの両方の子孫は黒です
  4. ノードから子孫の葉までのパスには、同じ数の黒いノードが含まれています。

新しい要素を挿入すると、赤色が割り当てられます。最初の2つのルールを満たすには、新しい頂点を目的の色で再ペイントするだけです。

3点と4点の実装のバランス規則を検討してください。
各図では、3ルールに違反する要素Xをすでに追加しており、3ルールが実行され、4ルールに違反しないようにツリーの構造を変更する必要があると想定しています。

頂点の凡例：

  • X-ルールの3段落に違反する追加要素。
  • P-お父さんのアイテムX
  • G-要素Xの祖父、要素Pの父
  • Uは要素Xの叔父、要素Pの兄弟、要素Gの次の息子です。

ケース1-赤いおじ




父親と叔父の両方が赤の場合、図に示すように、祖父のレベルから父親のレベルまで黒色を「下げ」、ノードを再描画できます。この場合、「黒の高さ」は同じままですが、要素Gのルール3に違反する可能性があるため、このノードに対してさらにバランスを再帰的に呼び出す必要があります。

ケース2-黒いおじ-父と祖父が別の方向に。




この構造は、お父さんとおじいさんが同じ方向に向かう3番目のケースに持ち込まれなければなりません。これを行うには、息子Xから父親に向かって小さなターンを行い（この記事ではターンのルールは考慮されていません）、要素Rに対して3つのケースを呼び出す必要があります。

ケース3-黒いおじ-お父さんとおじいさんが同じ側に




この場合、父親から祖父を経て黒人の叔父に大きな転換をして、Pを黒に、Gを赤に塗り直すことができます。このローテーションの結果、3つのルールが満たされます。

4番目のルールも満たされていることを確認してください。大きなターンの前に、要素Gの黒の高さがN + 2であったと仮定します。次に、「一時停止」要素の高さは次のようになります：

A = N + 1、
B = N + 1、
C = N + 1、
D = N、
E =N。

回転後、4つのルールがすべての要素に当てはまることを確認してください。

具体例


次に、要素1、2、3、4、5、6を順に挿入して赤黒ツリーを形成するプロセスを検討します。



要素1はルートなので、ルール1を満たすように再ペイントするだけです。



要素2を追加した後、すべてのルールが実行されます。



要素3を追加するときに、規則3に違反しました。黒の叔父と祖父と父がいるので、3番目のケースを使用します-ターンして再ペイントします。



要素4を追加すると、再び規則3に違反します。今回はおじが赤くなっているので、最初のケースに塗り直しを適用します-木の黒い高さが1ずつ増えます。バランシングアルゴリズムが祖父の要素2に追加して起動されることを確認してください。要素2は、ルートと同様に、単に黒で再描画されます。



要素5を挿入するときは、3のケースを再度適用します。大きな回転をして、頂点を再描画します。黒の高さが変更されていないことに注意してください。



要素6を追加すると、再び規則3に違反しました。おじさんが赤なので、最初のケースを塗り直し、黒の高さは変わっていません。この要素はルールに違反していないので、4つの要素のバランシングコールはツリーの何も変更しませんでした。
