遅かれ早かれ、電報ボット開発者の生活の中で、ボットのための多かれ少なかれ本格的なメニューを作成するためにすべてが必然的に必要であるという明確な理解があります。

タスクの本質


一見-タスクは簡単ですが、どのように関係なく。問題全体は、API自体、つまりUpdateオブジェクトにあります。
もう少し詳しく：getUpdateメソッドを使用して、Updateオブジェクトを取得します。したがって、このオブジェクトには、メッセージに関する情報を含む、多くの有用なものが含まれていますが、最後のものだけです。これが問題です
ウェブフックのリクエストを処理して、追加データを格納することなく、...私たちのために輝いていません-以上を踏まえ、以下では最も快適な結論ではありません

、まあを自分自身を洗浄し、仕事に行きます。

コンセプトは、その後、理解できなくなった…私は明確にしようと思います。理解のために、あるレベルの抽象化に進む必要があります。つまり、ボットの全体構造が、ユーザーが切り替えるさまざまなメニューで構成されていると想像してください（Androidアクティビティとの類似性を描くことができます）。ボットがアクティブになると、ユーザーは「初期メニュー」に移動します。このメニューから、たとえば「設定」、「情報」などのメニューに移動できます。



したがって、ボットの構造は少しだけ変化しますが

、最初はリクエストが存在するかどうかを確認します

。YES）「リクエストハンドラ」を実行します
NO）「初期メニュー」に留まり、利用可能なコマンドでユーザーの応答を確認します。

*これは非常に簡略化されたバージョンです。これにより、バーがノックされます。例を使用すると、簡単になります。

続けましょう ..？


インストールから始めましょう。インストールにはcompserを使用します（「composer.json」を作成し、以下の行をコピーして、off.docsで詳細を読んでください）。

{
  "require" : {  
	"s0d3s/tg-long-req": ">=1"   
  }
}

そして接続する

include_once('path/to/autoload.php');
use TgLongReq\TgLongReq;

または手動で接続（プロジェクトフォルダーにダウンロード）

include_once('TgLongReq.php');
use TgLongReq\TgLongReq;

みんな集まった！


電報ボットの作成については最低限の知識があると想定します。-％記事％ではなく、「ボットコードの作成」という項目まで読んでください。件名に関するGoogleの最初のリンクです。自分で検索できます。主なことは、tg自体でボットを作成し、設定する方法です。彼をあなたのサーバーに。興味深い場合は、ローカルサーバーで動作するようにWebhookを構成する方法についての記事を書きます（〜NEW！httpsで「問題」を回避するためのソリューション）。

この例を実現します。スペースを「botmain.php」（メインメニューと処理）と「varfunc.php」（リクエスト処理関数）の2つのファイルに分割します。例を簡単にするために、私たちはtelegram-bot-sdk（作成者のおかげで小規模プロジェクトに便利な、よくできて設計されたプロジェクト）を使用します。

一般的に、リスト：

composer.json

{
  "require" : {  
	"irazasyed/telegram-bot-sdk": ">=3",
	"s0d3s/tg-long-req": ">=1",
	"php":">=7.1"    
  }
}


次に、プロジェクトフォルダーでコンソールを開き、...

>composer install
...

UPD：Habrのフレームワーク内では、ソースコードはグローバルに編集しないと読めなくなります。このため、例に興味がある場合は、githubに切り替えることをお勧めします。
