RFC (Request For Comments) pour améliorer l'API Ember Registry


Chris Cricho ( @chriskrycho ) a proposé d'ajouter une nouvelle API orientée objet pour toutes les interfaces de registre; Déclarez l'API actuelle à l'aide d' une microsyntaxe basée sur des chaînes et introduire une propriété capabilitiespour prendre en charge en toute sécurité le moteur de reconnaissance existant .


RFC: Enhanced Ember Registry API décrit les problèmes actuels de microsyntaxe et le manque de prise en charge de TypeScript. RFC recommande d' ajouter une nouvelle API identifiant qui contiendra les champs name, typeet, le cas échéant, au namespacelieu de la microsyntax actuelle. Par exemple:


au lieu


getOwner(this).lookup('service:session');

sera


getOwner(this).lookup({ type: 'service', name: 'session' })

mais plutôt


class Example {
  @service('global@session')
  session;
}

sera


class Example {
  @service({ namespace: 'global', name: 'session' })
  session;
}

Si vous voulez en savoir plus ou ajouter votre propre avis, consultez RFC sur GitHub !
