Bagaimana cara: Manet atau Monet? Jaringan saraf merespons

Para seniman Eduard Manet dan Claude Monet bingung selama hidup mereka (di sini ada artikel yang sangat menarik tentang Arzamas). Yang tidak mengejutkan, karena mereka berdua adalah pendiri impresionisme dan menulis dengan cara yang sama. Mendengarkan kursus tentang jaringan saraf convolutional di coursera , saya memutuskan untuk mencoba membuat model yang menentukan seniman mana yang melukis gambar itu.

Sebelum mulai bekerja, saya hanya tahu bahwa Manet menulis "skandal" "Olympia" dan "Sarapan di Rumput" . Dalam proses pengumpulan data, menjadi jelas bahwa:

  • Gaya penulisan sangat mirip
  • Manet melukis lebih banyak potret, dan Monet melukis pemandangan
  • Monet memiliki serangkaian lukisan serupa (misalnya, lanskap yang sama, dilukis pada waktu yang berbeda dalam sehari)
  • Monet melukis lebih banyak lukisan daripada Manet.

Kumpulan data lengkap ternyata kecil untuk kedua tugas pemrosesan gambar: sedikit lebih dari 100 gambar dalam satu kelas (Manet) dan sedikit lebih dari 400 gambar di kelas lain (Monet, meskipun ini bukan semua lukisannya). Oleh karena itu, saya tidak mengharapkan akurasi klasifikasi yang tinggi yang biasanya dapat dilihat pada masalah seperti itu. Tapi itu menarik level apa yang bisa dicapai.

Model sederhana tanpa pemrosesan data tambahan


Untuk mulai dengan, kami akan menggunakan model CNN sederhana, dan kami tidak akan memproses data (kecuali untuk dijual kembali).

model_simple_cnn_wo_augm = tf.keras.models.Sequential([
    tf.keras.layers.Conv2D(16, (3,3), activation='relu', input_shape=(input_size, input_size, 3)),
    tf.keras.layers.MaxPooling2D(2, 2),
    tf.keras.layers.Conv2D(32, (3,3), activation='relu'),
    tf.keras.layers.MaxPooling2D(2,2),
    tf.keras.layers.Conv2D(64, (3,3), activation='relu'),
    tf.keras.layers.MaxPooling2D(2,2),
    tf.keras.layers.Flatten(),
    tf.keras.layers.Dense(512, activation='relu'),
    tf.keras.layers.Dense(1, activation='sigmoid')
])

train_datagen_wo_augm = ImageDataGenerator(rescale=1.0/255.)

Untuk mengatasi overfitting, mari kita coba menambahkan preprocessing data.

Model preprocessing data sederhana


Untuk preprocessing, kami menggunakan fungsi ImageDataGenerator dalam keras. Kami menambahkan rotasi, pergeseran vertikal dan horizontal, "peregangan" dan penskalaan gambar, perubahan kecerahan dan warna, serta tampilan horizontal.

train_datagen_with_augm = ImageDataGenerator(rescale=1.0/255.,
                                  rotation_range=40,
      width_shift_range=0.2,
      height_shift_range=0.2,
      shear_range=0.2,
      zoom_range=0.2,
      brightness_range = [0.5, 1.5],
      channel_shift_range = 100,
      horizontal_flip=True,
      fill_mode='nearest')

Kami meninggalkan model dan semua parameter pelatihan sama. Menurut hasil pelatihan, jelas bahwa nilai metrik (AUC) untuk sampel pelatihan dan validasi cukup dekat, yaitu pra-pemrosesan membantu menghilangkan overfitting.



Akibatnya, AUC pada sampel uji adalah 0,919, pada validasi - 0,909.

Model sederhana dengan preprocessing data dan bobot


Menambahkan preprocessing menghilangkan overfitting. Dan meskipun AUC tidak buruk, ketidakseimbangan data memotivasi saya untuk mencoba menambah bobot pada model. Saya menggunakan pendekatan untuk menghitung bobot dari manual keras :

weight_for_manet = (1 / manet_test_size)*(manet_test_size + monet_test_size)/2.0 
weight_for_monet = (1 / monet_test_size)*(manet_test_size + monet_test_size)/2.0

class_weight = {0: weight_for_manet, 1: weight_for_monet}

Sebagai hasil dari perhitungan tersebut, bobot untuk kelas 0 (Manet) 2.50, untuk kelas 1 (Monet) 0.62.

Variabel class_weight kemudian ditambahkan ke model.fit ().

Namun, hasil pelatihan tidak berbeda secara signifikan dari versi tanpa bobot: AUC pada set pelatihan adalah 0,892, pada validasi - 0,915.

Transfer pembelajaran: menggunakan model Inception V3


Kumpulan data kami kecil untuk tugas pembelajaran mesin, dan bahkan pembelajaran yang lebih mendalam. Dalam kasus seperti itu, pendekatan transfer pembelajaran dapat membantu.

Idenya adalah untuk mengambil model siap pakai yang telah dilatih tentang dataset besar, dan karenanya memiliki kesempatan untuk mempelajari lebih banyak pola dalam data. Setelah itu, lapisan terakhir dari jaringan diganti dengan yang sesuai dengan tugas kita, dan model tersebut melatih pada set data kita.

Saya menggunakan model Inception V3 ( wiki dan artikel tentang "Habr" ). Hasilnya baik: AUC pada sampel pelatihan 0,971, pada validasi - 0,970.
