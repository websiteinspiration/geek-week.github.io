クライアント/サーバーユーティリティWindowsの機能を備えたソフトウェアの作成、パート01

こんにちは。

本日は、Telnet、TFTPなどの純粋なJavaでの標準的なWindowsユーティリティの機能を実行するクライアントサーバーアプリケーションを作成するプロセスを分析したいと思います。私が新しいものをもたらすことはないことは明らかです。これらのユーティリティはすべて1年以上にわたって正常に機能していますが、誰もが内部で何が起こるかを知っているわけではありません。

これはカットの下で議論されるものです。

この記事では、それを遅らせないために、一般的な情報に加えて、Telnetサーバーについてのみ説明しますが、現時点では、他のユーティリティに関する資料がまだあります。これは、サイクルのさらに後の段階になります。

まず、Telnetとは何か、なぜ必要なのか、Telnetと一緒に何を食べるのかを理解する必要があります。私はソースをそのまま引用しません（必要に応じて、記事の最後に関連資料へのリンクを添付します）。Telnetがデバイスのコマンドラインへのリモートアクセスを提供していると言えます。ほとんどの場合、その機能はここで終わります（私は静かに静かにサーバーポートに意識的に接続しました。詳細は後で説明します）。したがって、その実装では、クライアントの行を受け入れ、サーバーに転送し、コマンドラインに転送し、コマンドラインの応答を読み取り、もしあれば、クライアントに転送して表示するか、発生した場合はそれを表示する必要があります。エラーの場合は、ユーザーに何かが間違っていることを理解させます。

したがって、上記を実装するには、2つの作業クラスと、サーバーを起動し、クライアントが動作するテストクラスが必要です。
したがって、現時点では、アプリケーションの構造には次のものが含まれます。

  • Telnetclient
  • TelnetClientTester
  • TelnetServer
  • TelnetServerTester

それぞれについて見ていきましょう

。TelnetClient

このクラスで実行できることは、受信したコマンドを送信し、受信した回答を表示することだけです。さらに、リモートデバイスの任意の（上記のように）ポートに接続して切断できる必要があります。

これを行うために、次の関数が実装されました：

引数としてソケットアドレスを取り、接続を開き、入出力ストリームを開始する関数（ストリーム変数は上記で宣言されており、記事の最後に完全なソースです）。

 public void run(String ip, int port)
    {
        try {
            Socket socket = new Socket(ip, port);
            InputStream sin = socket.getInputStream();
            OutputStream sout = socket.getOutputStream();
            Scanner keyboard = new Scanner(System.in);
            reader = new Thread(()->read(keyboard, sout));
            writer = new Thread(()->write(sin));
            reader.start();
            writer.start();
        }
        catch (Exception e) {
            System.out.println(e.getMessage());
        }
    }

デフォルトのポートに接続して同じ機能をオーバーロード-Telnetの場合、これは23です。


    public void run(String ip)
    {
        run(ip, 23);
    }

関数は、キーボードから文字を読み取り、それらを出力ソケットに送信します。これは、文字モードではなく小文字で通常行われます。


    private void read(Scanner keyboard, OutputStream sout)
    {
        try {
            String input = new String();
            while (true) {
                input = keyboard.nextLine();
                for (char i : (input + " \n").toCharArray())
                    sout.write(i);
            }
        }
        catch (Exception e) {
            System.out.println(e.getMessage());
        }
    }

関数はソケットからデータを受け取り、画面に表示します。

    private void write(InputStream sin)
    {
        try {
            int tmp;
            while (true){
                tmp = sin.read();
                System.out.print((char)tmp);
            }
        }
        catch (Exception e) {
            System.out.println(e.getMessage());
        }
    }

関数はデータの送受信を停止します。

    public void stop()
    {
        reader.stop();
        writer.stop();
    }
}

TelnetServer

このクラスには、ソケットからコマンドを受け入れ、実行のために送信し、コマンドからの応答をソケットに送信する機能が必要です。プログラムは入力データを意図的にチェックしません。最初に「ボックスtelnet」でサーバーのディスクをフォーマットすることが可能であり、2番目に、この記事のセキュリティ問題は原則として省略されているため、暗号化またはSSL

関数は2つしかなく（そのうちの1つはオーバーロードされています）、一般にこれは良い方法ではありませんが、このタスクの一部として、すべてをそのままにしておくのが適切と思われました。

 boolean isRunning = true;
    public void run(int port)    {

        (new Thread(()->{ try {
            ServerSocket ss = new ServerSocket(port); //         
            System.out.println("Port "+port+" is waiting for connections");

            Socket socket = ss.accept();
            System.out.println("Connected");
            System.out.println();

            //      ,       .
            InputStream sin = socket.getInputStream();
            OutputStream sout = socket.getOutputStream();

            Map<String, String> env = System.getenv();
            String wayToTemp = env.get("TEMP") + "\\tmp.txt";
            for (int i :("Connected\n\n\r".toCharArray()))
                sout.write(i);
            sout.flush();

            String buffer = new String();
            while (isRunning) {

                int intReader = 0;
                while ((char) intReader != '\n') {
                    intReader = sin.read();
                    buffer += (char) intReader;
                }


                final String inputToSubThread = "cmd /c " + buffer.substring(0, buffer.length()-2) + " 2>&1";


                new Thread(()-> {
                    try {

                        Process p = Runtime.getRuntime().exec(inputToSubThread);
                        InputStream out = p.getInputStream();
                        Scanner fromProcess = new Scanner(out);
                        try {

                            while (fromProcess.hasNextLine()) {
                                String temp = fromProcess.nextLine();
                                System.out.println(temp);
                                for (char i : temp.toCharArray())
                                    sout.write(i);
                                sout.write('\n');
                                sout.write('\r');
                            }
                        }
                        catch (Exception e) {
                            String output = "Something gets wrong... Err code: "+ e.getStackTrace();
                            System.out.println(output);
                            for (char i : output.toCharArray())
                                sout.write(i);
                            sout.write('\n');
                            sout.write('\r');
                        }

                        p.getErrorStream().close();
                        p.getOutputStream().close();
                        p.getInputStream().close();
                        sout.flush();

                    }
                    catch (Exception e) {
                        System.out.println("Error: " + e.getMessage());
                    }
                }).start();
                System.out.println(buffer);
                buffer = "";

            }
        }
        catch(Exception x) {
            System.out.println(x.getMessage());
        }})).start();

    }

プログラムは、サーバーのポートを開き、データを読み取り、コマンドの終了文字が見つかるまでコマンドを新しいプロセスに転送し、プロセスからの出力はソケットにリダイレクトされます。すべてがちょうどカラシニコフ突撃銃のようです。

したがって、この関数では、デフォルトのポートで過負荷が発生します。
