あなたのコアはtypescriptでコンポーネントを反応させます


標準の反応コンポーネントはtypescriptでどのように見えますか？それをJavaScriptのreactコンポーネントと比較してみましょう。

import React from 'react'
import PropTypes from 'prop-types'

export function StandardComponent({ children, title = 'Dr.' }) {
  return (
    <div>
      {title}: {children}
    </div>
  )
}

StandardComponent.propTypes = {
  title: PropTypes.string,
  children: PropTypes.node.isRequired,
}

そして今、typescriptバージョン：

import * as React from 'react'

export interface StandardComponentProps {
  title?: string
  children: React.ReactNode
}

export function StandardComponent({
  children,
  title = 'Dr.',
}: StandardComponentProps) {
  return (
    <div>
      {title}: {children}
    </div>
  )
}

とても似ていますよね？propTypesインターフェイスに置き換えましたtypescript。

タイトルpropはオプションのままですが、相続人の小道具は引き続き必要です。別のコンポーネントがインターフェースへのリンクを必要とする場合に備えて、インターフェースをエクスポートしました。

標準HTML属性の拡張


私たちは親コンポーネントには、追加型指定された属性を提供することができるようにしたい場合divなど、aria-hidden、styleまたはをclassName、私たちはそれらを定義することができinterface、内蔵のインターフェイスを拡張しますか。以下の例では、コンポーネントdivがヘッダーと子孫に加えてすべての標準プロパティを受け入れると言います。

import * as React from 'react'

export interface SpreadingExampleProps
  extends React.HTMLAttributes<HTMLDivElement> {
  title?: string
  children: React.ReactNode
}

export function SpreadingExample({
  children,
  title = 'Dr.',
  ...other
}: SpreadingExampleProps) {
  return (
    <div {...other}>
      {title}: {children}
    </div>
  )
}

イベント処理


イベントハンドラーをタイプ指定して、イベント引数が正しいタイプであることを確認できます。以下の例は、この目標を達成するためのさまざまな方法を示しています。

export interface EventHandlerProps {
  onClick: (e: React.MouseEvent) => void
}

export function EventHandler({ onClick }: EventHandlerProps) {
  // handle focus events in a separate function
  function onFocus(e: React.FocusEvent) {
    console.log('Focused!', e.currentTarget)
  }

  return (
    <button
      onClick={onClick}
      onFocus={onFocus}
      onKeyDown={e => {
        // When using an inline function, the appropriate argument signature
        // is provided for us
      }}
    >
      Click me!
    </button>
  )
}

使用する引数の署名がわからない？エディターで、イベントハンドラーの対応するプロパティにカーソルを合わせます。

反応コンポーネントでジェネリックを使用する


これはより高度な機能ですが、非常に強力です。通常、データ型は、特定の属性を持つコンポーネントの反応で定義します。コンポーネントにオブジェクトが必要だとしますprofile。

interface ProfileType {
  name: string
  image: string
  age: number | null
}

interface ProfilesProps {
  profiles: Array<ProfileType>
}

function Profiles(props: ProfilesProps) {
  // render a set of profiles
}

ここで、任意のタイプの配列を受け入れることができるコンポーネントがあるとします。ジェネリックは小包を郵送するようなものです。クーリエ（コンポーネント）は送信するパッケージの内容を知る必要はありませんが、送信者（親コンポーネント）は受信者が送信したコンテンツを受信することを期待しています。

次のように実装します。

interface GenericsExampleProps<T> {
  children: (item: T) => React.ReactNode
  items: Array<T>
}

export function GenericsExample<T>({
  items,
  children,
}: GenericsExampleProps<T>) {
  return (
    <div>
      {items.map(item => {
        return children(item)
      })}
    </div>
  )
}

少し奇妙な例...それにもかかわらず、それは本質を示しています。コンポーネントは、任意のタイプの要素の配列を受け入れ、それを通過childrenし、配列要素を持つレンダリング関数として関数を呼び出します。親コンポーネントがレンダーコールバックを継承として提供する場合、要素は正しく入力されます！

理解されていない？これは正常です。私自身、最後までジェネリックを把握していませんが、ジェネリックを完全に理解する必要はほとんどありません。ただし、より多くの作業をtypescriptするほど、意味がわかります。

タイピングフック


フックはほとんどそのままで機能します。2つの例外はuseRef、およびのみuseReducerです。次の例は、refを入力する方法を示しています。

import * as React from 'react'

interface HooksExampleProps {}

export function HooksExample(props: HooksExampleProps) {
  const [count, setCount] = React.useState(0)
  const ref = React.useRef<HTMLDivElement | null>(null)

  // start our timer
  React.useEffect(
    () => {
      const timer = setInterval(() => {
        setCount(count + 1)
      }, 1000)

      return () => clearTimeout(timer)
    },
    [count]
  )

  // measure our element
  React.useEffect(
    () => {
      if (ref.current) {
        console.log(ref.current.getBoundingClientRect())
      }
    },
    [ref]
  )

  return <div ref={ref}>Count: {count}</div>
}

状態は自動的に型付けされnullますが、問題があるか要素を含むことを示すためにrefを手動で入力しましたdiv。関数内useEffectでrefを参照するときは、それが等しくないことを確認する必要がありnullます。

ギアボックスのタイピング


ギアボックスを使用すると、もう少し複雑になりますが、適切に入力されている場合、これは素晴らしいことです。

// Yeah, I don't understand this either. But it gives us nice typing
// for our reducer actions.
type Action<K, V = void> = V extends void ? { type: K } : { type: K } & V

// our search response type
interface Response {
  id: number
  title: string
}

// reducer actions. These are what you'll "dispatch"
export type ActionType =
  | Action<'QUERY', { value: string }>
  | Action<'SEARCH', { value: Array<Response> }>

// the form that our reducer state takes
interface StateType {
  searchResponse: Array<Response>
  query: string
}

// our default state
const initialState: StateType = {
  searchResponse: [],
  query: '',
}

// the actual reducer
function reducer(state: StateType, action: ActionType) {
  switch (action.type) {
    case 'QUERY':
      return {
        ...state,
        query: action.value,
      }

    case 'SEARCH':
      return {
        ...state,
        searchResponse: action.value,
      }
  }
}

interface ReducerExampleProps {
  query: string
}

export function ReducerExample({ query }: ReducerExampleProps) {
  const [state, dispatch] = React.useReducer(reducer, initialState)

  React.useEffect(
    () => {
      if (query) {
        // emulate async query
        setTimeout(() => {
          dispatch({
            type: 'SEARCH',
            value: [{ id: 1, title: 'Hello world' }],
          })
        }, 1000)
      }
    },
    [query]
  )

  return state.searchResponse.map(response => (
    <div key={response.id}>{response.title}</div>
  ))
}

typeofとkeyofを使用してコンポーネントオプションを分類する


異なる外観のボタンが必要であると想定します。それぞれのボタンは、キーとスタイルのセットを持つオブジェクトで定義されています。次に例を示します。

const styles = {
  primary: {
    color: 'blue',
  },
  danger: {
    color: 'red',
  },
}

ボタンコンポーネントはプロパティを受け入れる必要がありますtype。これは
、オブジェクトの任意のキーにすることができますstyles（たとえば、「primary」または「danger」）。とても簡単にタイプできます：

const styles = {
  primary: {
    color: 'blue',
  },
  danger: {
    color: 'red',
  },
}

// creates a reusable type from the styles object
type StylesType = typeof styles

// ButtonType = any key in styles
export type ButtonType = keyof StylesType

interface ButtonProps {
  type: ButtonType
}

export function Button({ type = 'primary' }: ButtonProps) {
  return <button style={styles[type]}>My styled button</button>
}
