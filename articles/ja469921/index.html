【事例】コテージビレッジでの大気質モニタリング



記事では、エコセンサーのネットワークを使用して村の大気質を監視し、アドバンテックの技術ソリューションに基づいてリアルタイムでユーザーに情報を提供するソリューションを開発した経験を紹介します。

スマートシティと大気汚染の問題


ロシアの人口の74％以上が大都市に住んでおり、都市人口の増加は続いています。人口密度と生産量が高いと、大気質が悪化します。主な情報源は、産業、火力、自動車輸送です。世界保健機関によると、最も危険な汚染物質は、呼吸時に肺に入り、一生を通じて排泄されないため、細かい粉塵です。

2019年のNAFIの調査によると、市民の約71％が環境状況が失敗したと考え、これを自動車や産業からの排出に起因しているとしています。問題は、地域の汚染源をタイムリーに特定することです。

2019〜2024年の国家プロジェクト「デジタルエコノミー」のタスクの1つ。主に国内開発を中心としたデジタル技術とプラットフォームソリューションの導入による、都市経済を含む経済および社会的領域の優先分野の変革です。このプロジェクトの枠組みの中で、2019年3月4日に建設住宅サービス省は、都市経済のデジタル化の一環として実施された活動のリストを含むスマートシティ基準を承認しました。

この標準には、28方向で10万人以上の人口を抱える都市でのソリューションの実装が含まれます。その中には、環境の安全性の向上につながる対策、大気質の状態の制御の確保、および市の予算収入の管理を含む「オンライン大気監視システム」の活動があります。環境保護の観点から、市民の都市の魅力のレベルを高めます。2018年12月24日の戦略的開発と国家プロジェクトに関する大統領評議会の議長の決定によりパスポートが承認され
た国家プロジェクト「エコロジー」の枠組みの中で、連邦プロジェクト「クリーンエア」が実施されています」、RD 52.04.667-2005の推奨に従って決定された、大気汚染の程度が「増加」のレベルまで低下する12の工業地帯の大気の質を改善することを目的としています。

このプロジェクトのタスク1.5によると、大気質を監視するために、近代化された州観測ネットワーク、地域およびローカルシステムからのデータを統合する予定です。これにより、高空間分解能の高密度センサーネットワークによって取得された大気質データに基づいて、地域の大気汚染源を特定し、排出を制御することができます。
ただし、小都市や町の住民はこれらの大規模な連邦プロジェクトに関与しておらず、自分のリソースにのみ依存することができます。工業中心地から離れた集落の大気質ははるかに高いという事実にもかかわらず、それにもかかわらず、汚染の局所発生源を迅速に特定する問題は、人口密集地域と同じくらい深刻です。

大気質監視用センサーネットワーク


村のインフラ施設や大規模なレクリエーションの場所にエコセンサーを配置することで、住民、地方自治体、規制当局に情報を提供することで、大気質を管理し、環境の安全性を向上させることができます。WISE-PaaSデジタルプラットフォームを使用すると、エコセンサーからデータを収集し、その変動性を分析して、大気汚染のリスクを予測することができます。これらすべてが、生活の質の向上と、居住者とゲストにとっての村の魅力につながります。
センサー自体は、実際には、アドバンテックのハードウェアを使用して実装された、実世界とそのデジタル対応物との間の接続リンクです。それらは空気中のダスト含有量とその温度を測定し、メッセージパケットを形成し、LoRaWANプロトコルに従って868 MHzの範囲でワイヤレスで基地局に送信します。そこでデータはJSONテキストメッセージにパックされ、MQTTプロトコルに送信されてWISE-PaaSデジタルプラットフォーム。アプリケーションサーバーによって既に処理されています。



LoRaゲートウェイとして、WISE-6610デバイスを使用しますアドバンテックチェコ共和国製。これは、外部アンテナ、LAN、および電源を接続するためのコネクタを備えた、DINレールまたは壁に取り付けられたかなりコンパクトなデバイスです。電源コネクタの4つの接点のうち、実際に使用されているのは、赤（+）と黒（-）の2つだけです。デフォルトでは、ゲートウェイはルーターモードで動作し、LoRaモジュールはオフになっています。デバイスをアクティブにするには、イーサネットケーブルを介してデバイスに接続し、ゲートウェイの設定Webインターフェイスが使用できるアドレスをブラウザーのアドレスバーに示す必要があります。

http://192.168.1.1

承認リクエストで、デフォルトのパラメーター（ログイン：root、パスワード：root）を指定し、図に示すインターフェースに入ります。



ここで、「ユーザーモジュール」の項目が最も重要であり、LoRaゲートウェイとローカルNode-REDサーバーを構成できます。 LoRaゲートウェイの設定に進み、LoRaモジュールを有効にし、LoRaゲートウェイの無線チャネルの周波数を指定し、ネットワークサーバーの動作パラメーターを設定し、ホスト、ポート、ユーザー名、およびパスワードを指定してMQTTブローカーへのメッセージ送信を構成します。



エコセンサーをLoRaゲートウェイにバインドするだけです。これを行うには、ABP（Activation-by-personalization）メソッドを使用します。ネットワークサーバー設定パネルに移動します。

https://192.168.1.1:8443

[Devices-Activated（Nodes）]を選択し、[+ Create]ボタンをクリックして、表示されるウィンドウで、NwkSKey（ネットワークキー）、AppSKey（アプリケーションキー）、DevAddr（センサーアドレス）の3つのパラメーターを指定します。エコセンサーの設定でも同じパラメーターを示しています。WISE-6610をローカルネットワークに



接続するには、WISE-6610をDHCPクライアントモードにする必要があります。その後、デバイス自体がローカルネットワーク上のDHCPサーバーを検出し、IPアドレスを取得してインターネットに接続します。これは、メインWebインターフェースを介して行われます。「Configuration-LAN」メニュー項目を選択し、「DHCPクライアント」フィールドで「有効」を示します。



WISE-PaaSデジタルプラットフォーム

今年の初めに、モノのインターネットのための150以上のデジタルプラットフォームが知られています。それぞれに独自の特性、利点、欠点があります。WISE-PaaSプラットフォームを使用すると、センサーネットワークデータの収集と視覚化の問題を解決できます。
デジタルプラットフォームWISE-PaaS上のセンサーからのデータの処理は、いくつかの段階で実行されます。

  1. MQTTブローカーがLoRaゲートウェイからJSONメッセージを受信し、アプリケーションに転送する。
  2. PythonスクリプトによるJSONメッセージの受信、データ処理、およびデータベースへの書き込み。
  3. PostgreSQL DBMSを使用したデータの記録、保存、提供。
  4. WISE-PaaS /ダッシュボードを使用したデータの視覚化;



WISE-PaaSで新しい仮想スペースを作成すると、MQTTブローカーRabbitMQがデフォルトで起動され、使用可能なサービスのリストに表示されます。ブローカーのパラメーター（ホスト、ポート、ユーザー名、パスワード）は、Webインターフェースから取得できます。

MQTTブローカーから測定データを受信して​​データベースに書き込むアプリケーションを作成するには、Advantechが用意した例に基づいて段階的な手順を使用できます。

作業ディレクトリを作成します。

mkdir ~/wisepaas/

それに行く：

cd ~/wisepaas/

MQTTデータを取得してPostgreSQLデータベースにデータを書き込むために、Githubを使用した例を複製しています。

git clone https://github.com/WISE-PaaS/example-py-iothub-postgresql.git

index.py実行可能アプリケーションを準備するには、いくつかのパラメーターの値をWISE-PaaSコントロールパネルで指定された実際の値に変更します。

…
# MQTT(rabbitmq)
service_name = 'p-rabbitmq-innoworks'
…
# Postgresql
service_name = 'postgresql-innoworks'
…
schema = 'projectname'
table = 'tablename'
group = 'groupname'
…
def on_connect(client, userdata, flags, rc):
	client.subscribe("#")
…

manifest.yml ファイルには、アプリケーションパラメータ（名前、使用可能なRAMのサイズ、ディスククォータ、アプリケーション起動コマンド、使用されているサービスのリスト）が含まれています。

---
applications:
  - name: py-postgresql-meteolab
    memory: 256MB
    disk_quota: 256MB
    buildpack: python_buildpack
    command: python index.py
services:
  - rabbitmq

CloudFoundryクライアントを使用して、 ユーザーuserをパスワードpasswordでapi.wise-paas.ioサーバーに接続します。

cf login -a api.wise-paas.io -u user@advantech.com -p password

CloudFoundryへのクライアント接続を確認します。

cf target

起動せずにアプリケーションをサーバーにコピーする：

cf push py-postgresql-meteolab --no-start

グループを使用してアプリケーションをデータベースにバインドします。

cf bs py-postgresql-meteolab postgresql -c '{\"group\":\"groupname\"}'

アプリケーションをMQTTブローカーにバインドします。

cf bs py-postgresql-meteolab rabbitmq

サーバーでアプリケーションを実行する：

cf start py-postgresql-meteolab

アプリケーションの動作制御は、アプリケーションの実行ログを確認することで実行されます。

cf logs py-postgresql-meteolab --recent

すべてのアプリケーション環境変数をローカルのJSONファイルにコピーすると便利です。

cf env py-postgresql-meteolab > env.json

仮想空間の作成時に、PostgreSQLサーバーもデフォルトで起動されます。WISE-PaaSにはデータベースを管理するためのWebインターフェースがないため、pgAdminなどのクライアントをインストールする必要があります。次に、データベーステーブルが作成され、Webインターフェイスを使用してセンサーからのデータを記録するプロセスが制御されます。



情報視覚化ツール



WISE-PaaS デジタルプラットフォームにはWISE-PaaS /ダッシュボードが含まれており、さまざまな表示オプションを持つ任意の数のブロックを含むグラフィックパネルを作成できます。テキスト、グラフ、チャート、マップなど、さまざまなものがあります。



ブロックは「編集」メニューから設定されます。表示されるデータのソースは、データベースへのSQLクエリです。さらに、「編集」メニューの表示を調整するためのさまざまな構成可能なオプションがあります。





さらに細かく調整する場合は、「パネルJSON」を使用して、JSON形式で設定を編集できます。 「編集」パネルからすべてのパラメーターを編集することはできないため、これが必要になる場合があります。



各パネルには共通の設定があります。ここでは、パネル名、説明、タグの割り当て、時間設定、背景の変更を設定できます。



さらに、WISE-PaaS /ダッシュボードには、すべてのパネルに共通の設定があります。ここでは、データソースの追加、グループとユーザーの管理、プラグインの管理（追加、アクティブ化、削除）、画像のアップロードなどを行うことができます。



結論


コテージの村で大気質監視システムを使用すると、汚染源をすばやく特定し、街路清掃を計画し、最終的に人々の生活の質を向上させることができます。このシステムをビデオ監視システム、アクセス制御および管理、街路照明と統合することで、コテージビレッジの運営効率を高める「スマート」なデジタルソリューションを作成できます。
