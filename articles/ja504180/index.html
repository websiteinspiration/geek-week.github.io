これはどういう意味ですか？店舗のバスケットは、通貨、タイムゾーン、税金、顧客の種類、アイテムの種類、注文など、すべての結果を伴う世界のさまざまな地域でのオンライン販売に関するデータの一定のストリームを生成します。実際、同じことが任意のオンラインストアで生成されますが、おそらく、注文のパラメーターのオプションがわずかに異なるだけです。


このストリームに座って、特定の集計ダッシュボードを発行する必要があります。このダッシュボードで、企業は現在の瞬間のオンラインの主要なインジケーターをオンラインで（または定期的に）監視できます。さらに、これらのダッシュボードは、さまざまな詳細度と専門性で多少異なることが望ましいです。

def merge_tables(temp_table, base_table, schema, head, merge_param):
    merge_query = “MERGE into “+base_table+” as T USING “+temp_table+” as S \
         ON T.”+merge_param+” = S.”+merge_param+ \
         “ WHEN MATCHED then UPDATE SET “+ \
         ‘,’.join([‘T.’+item+’=S.’+item for item in head])+ \
         “ WHEN NOT MATCHED then INSERT \
         (“+’,’.join(head)+”) VALUES (“+’,’.join([‘S.’+item for item in head])+”);”
