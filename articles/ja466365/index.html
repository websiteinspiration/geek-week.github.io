備考1.スケジューラは単純すぎる。

前回の記事の公開以降、コードは大幅に変更されました。本格的な核が形成され、小核でなくなった。初期RAMディスク（カーネルファイルシステム）のサポート、仮想メモリ（MMU）のサポートがあります。カーネルがカスタムプログラムを書くことが可能になりました。システムコールとclibライブラリ（stdio、string）がありました。したがって、/ initrd / sh.elfシェルは、elfブートローダーによって解析され、プロセスとして起動される別個のプログラムです。

シェルコマンドのリスト：exec <file.elf>、ps、kill、exit。この場合、シェルはシェル自体から起動できます（マルチタスクのデモ）。

行け。パート1.システムを構築して起動する


記事では、主要なステップのみをリストします。詳細な説明については、この記事のビデオチュートリアルを参照してください。

Linuxが必要です。懐かしいメイクを集めます。カーネル実行可能ファイルをコンパイルするには、次のフラグが必要です。

CC_FLAGS=-g -m32 -isystem $(IDIR) -I include -DKERNEL=1 -fno-stack-protector -Wall -Werror -fno-pie
AS_FLAGS=-g --32
LD_FLAGS=-m elf_i386

ハードウェアエミュレーターとして、qemuエミュレーターをLinuxにインストールします。次のようにカーネルを実行します。

qemu-system-i386 -kernel ./bin/kernel.elf

リンカ用の小さなスクリプトも必要です。セクションを正しいアドレスで正しい順序でロードするために必要です。また、エントリポイントを示します。

OUTPUT_FORMAT(elf32-i386)
ENTRY(start)
SECTIONS
 {
   . = 0x100000;
   .text : { *(.text) }
   .data : { *(.data) }
   .bss  : { *(.bss)  }
 }

カーネルはマルチブート仕様に従って読み込まれるため、コードセクションの先頭にヘッダーが必要です。

.code32
.text
    # multiboot spec
    .align 4
multiboot:
    .long 0x1BADB002 # magic
    .long 0x00 # flags
    .long -(0x1BADB002 + 0x00) # checksum. m+f+c should be zero

すぐに独自のスタックに切り替えて、グローバル記述子テーブルをセットアップすることをお勧めします。独自のスタックを定義します。

.bss
.fill 8192,1 # 8Kb
stack:

エントリーポイントを書いてみましょう。gnuアセンブラー構文が異常な場合があります。私はかつてIntel構文を好み、それを試してLinuxのソースコードを掘り下げ、AT＆T構文を完全に好みました。残りは直感的です。

start:
    cli
    movl $stack,%esp
    push %esp
    push %ebx /* address of struct multiboot_t */
    call kernel_start /* should never return */
    hlt

これでボイラープレートコードが完成しました。楽しみが始まります。これで、Cでコードを記述できます。最初に、挨拶メッセージを定義します。

char *hello = "Hello world!";
int screen_size = 80 * 25;

次に、アセンブラコードが制御を移す先のエントリポイント自体を記述します。

/*
 * Api - Kernel entry point
 */
extern void kernel_start(struct multiboot_t* multiboot, void* kstack)
{
    char *video = (char*)0xB8000;
    char buff[screen_size + 1];
    video[screen_size] = '\0';

    memset(buff, ' ', screen_size);
    strext(video, buff, 0x7);
    strext(video, hello, 0x7);
}

ここでは、単にメッセージを画面に表示します。原則として、最初のレッスンのコードはテンプレートであり、変更されることはないため、完全にコピーできます。画面に何かを表示するには、それをビデオメモリーに直接書き込んで、属性記号で各文字を補足する必要があります。これを行うには、独自のC言語ライブラリが必要になります。これは、ニーズに合わせて独自に作成します。したがって、プロセスの制御が容易になります。たとえば、今日では、各文字の後に属性バイトを挿入するために、使い慣れた2つの関数（strcpy、memcpy）と独自のstrextの1つを自由に使用できます。
