MSリモートデスクトップゲートウェイ、HAProxyおよびパスワード推測

よう皆！

自宅からオフィスの職場に接続する方法はたくさんあります。それらの1つは、Microsoftリモートデスクトップゲートウェイを使用することです。これはRDP over HTTPです。 RDGW自体の構成については触れたくありません。なぜそれが良いのか悪いのかについては議論したくありません。リモートアクセスツールの1つとして扱いましょう。 RDGWサーバーを悪質なインターネットから保護することについてお話したいと思います。 RDGWサーバーをセットアップすると、すぐに保護、特にパスワード保護に夢中になりました。これを行う方法に関するインターネット上の記事を見つけられなかったことに驚きました。まあ、あなたはそれを自分でしなければなりません。

RDGW自体は保護されていません。はい、裸のレールインターフェイスを白いネットで公開することは可能で、問題なく機能します。しかし、適切な管理者またはIB'shnikはこれから落ち着きません。さらに、怠慢な従業員が自宅のコンピューターで企業アカウントのパスワードを覚えてからパスワードを変更した場合に、アカウントをブロックする状況を回避できます。

内部リソースを外部環境から保護する良い方法は、さまざまなプロキシ、公開システム、およびその他のWAFを使用することです。 RDGWはすべて同じhttpであることを思い出してください。それから、内部サーバーとインターネットの間に専用のソリューションを直接設置するように頼みます。

クールなF5、A10、Netscaler（ADC）があることは知っています。これらのシステムのいずれかの管理者として、これらのシステムの破壊に対する保護を設定することも可能だと言います。そして、はい、これらのシステムは、途中での合成洪水からあなたを守ります。

しかし、すべての企業がそのようなソリューションを購入する（そしてそのようなシステムの管理者を見つける）ことができるわけではありませんが、セキュリティを管理することができます！

無料のオペレーティングシステムに無料バージョンのHAProxyをインストールすることができます。 Debian 10でhaproxy 1.8.19の安定したリポジトリバージョンでテストしました。また、テストリポジトリからバージョン2.0.xxを確認しました。

debian自体の設定は、この記事の範囲を超えています。簡単に言うと、白いインターフェースでは443ポートを除くすべてを閉じ、灰色のインターフェースでは-たとえば、ポリシーに従って、22ポートを除くすべてを閉じます。作業に必要なものだけを開きます（たとえば、VRIPなどのVRRP）。

最初に、haproxyをSSLブリッジモード（別名モードhttp）に構成し、ロギングをオンにして、RDP内の内容を確認しました。いわば、真ん中を登りました。そのため、RDGatewayの構成に関する「すべて」の記事で指定されている/ RDWebパスがありません。 /rpc/rpcproxy.dllと/ remoteDesktopGateway /があります。この場合、標準のGET / POSTリクエストは使用されず、独自のリクエストタイプはRDG_IN_DATA、RDG_OUT_DATAです。

多くはないが、少なくとも何か。

テストしてみましょう。

mstscを開始し、サーバーに移動して、ログに4つの401エラー（無許可）を表示し、ログイン/パスワードを入力すると、回答200が表示されます。

電源をオフにして、もう一度開始すると、ログに同じ4つの401エラーが表示されます。間違ったユーザー名/パスワードを入力すると、4つ表示されますエラー401。必要なもの。これは私たちがキャッチするものです。

ログインURLを特定できなかったため、haproxyで401エラーをキャッチする方法がわからないため、すべての4xxエラーをキャッチします（実際にはキャッチせず、カウントします）。また、問題を解決してうれしいです。

保護の本質は、単位時間あたりの（バックエンドでの）4xxエラーの数をカウントし、それが指定された制限を超える場合、指定された時間の間、このIPからのすべての接続を（フロントエンドで）拒否することです。

技術的には、これはパスワード保護ではなく、4xxエラー保護になります。たとえば、存在しないURL（404）を頻繁に要求する場合、保護も機能します。

Windowsリモートデスクトップクライアント（mstsc）については、TLS1.2（少なくともWindows 7では）をサポートしていないことは注目に値するため、TLS1を離れる必要がありました。現在の暗号をサポートしていないので、古いものも残さなければなりませんでした。

haproxy.conf 
global
        log /dev/log    local0
        log /dev/log    local1 notice
        chroot /var/lib/haproxy
        stats socket /run/haproxy/admin.sock mode 660 level admin expose-fd listeners
        stats timeout 30s
        user haproxy
        group haproxy
        daemon

        # Default SSL material locations
        ca-base /etc/ssl/certs
        crt-base /etc/ssl/private

        # See: https://ssl-config.mozilla.org/#server=haproxy&server-version=2.0.3&config=intermediate
        #ssl-default-bind-ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE
-RSA-AES256-GCM-SHA384
        ssl-default-bind-ciphers ECDH+AESGCM:DH+AESGCM:ECDH+AES256:DH+AES256:ECDH+AES128:DH+AES:RSA+AESGCM:RSA+AES:!aNULL:!MD5:!DSS
        ssl-default-bind-ciphersuites TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256
        #ssl-default-bind-options ssl-min-ver TLSv1.2 no-tls-tickets
        ssl-default-bind-options no-sslv3
        ssl-server-verify none


defaults
        log     global
        mode    http
        option  httplog
        option  dontlognull
        timeout connect 5000
        timeout client  15m
        timeout server  15m
        errorfile 400 /etc/haproxy/errors/400.http
        errorfile 403 /etc/haproxy/errors/403.http
        errorfile 408 /etc/haproxy/errors/408.http
        errorfile 500 /etc/haproxy/errors/500.http
        errorfile 502 /etc/haproxy/errors/502.http
        errorfile 503 /etc/haproxy/errors/503.http
        errorfile 504 /etc/haproxy/errors/504.http


frontend fe_rdp_tsc
    bind *:443 ssl crt /etc/haproxy/cert/dektop.example.com.pem
    mode http
    capture request header Host len 32
    log global
    option httplog
    timeout client 300s
    maxconn 1000

    stick-table type ip size 1k expire 15s store gpc0
    tcp-request connection track-sc0 src
    tcp-request connection reject if { sc0_get_gpc0 gt 0 }

    acl rdweb_domain hdr(host) -i beg dektop.example.com
    http-request deny deny_status 400 if !rdweb_domain
    default_backend be_rdp_tsc


backend be_rdp_tsc
    balance source
    mode http
    log global

    stick-table type ip size 1k expire 15s store http_err_rate(10s)
    acl errors_too_fast sc1_http_err_rate gt 8
    acl mark_as_abuser sc0_inc_gpc0(fe_rdp_tsc) gt 0
    acl clear_as_abuser sc0_clr_gpc0(fe_rdp_tsc) ge 0
    tcp-request content track-sc1 src
    tcp-request content reject if errors_too_fast mark_as_abuser
    tcp-request content accept if !errors_too_fast clear_as_abuser

    option forwardfor
    http-request add-header X-CLIENT-IP %[src]

    option httpchk GET /
    cookie RDPWEB insert nocache
    default-server inter 3s    rise 2  fall 3
    server rdgw01 192.168.1.33:443 maxconn 1000 weight 10 ssl check cookie rdgw01
    server rdgw02 192.168.2.33:443 maxconn 1000 weight 10 ssl check cookie rdgw02
