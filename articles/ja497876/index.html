Webサイトを作成するときにフロントエンド開発者が下す必要がある決定の1つは、Webページに画像を含める方法の選択に関するものです。このメソッドは、HTMLタグを使用する場合があります<img>。これは、CSSプロパティbackgroundまたは<image>要素タグにすることができます<svg>。プロジェクトのパフォーマンスと可用性に大きな影響を与える可能性があるため、画像を操作する適切な方法を選択することは非常に重要です。 今日私たちが公開している翻訳は、ウェブページに画像を含めるさまざまな方法の研究に当てられています。ここでは、これらの方法の長所と短所について説明します。さらに、それらが通常使用される時期と理由についても説明します。

CSSCSSを使用して画像を非表示にする


CSSを使用して画像を非表示にすることができます。確かに、そのような画像はページが読み込まれたときにも読み込まれます。したがって、その際には注意が必要です。特定の画像を非表示にする必要がある場合は、装飾目的でのみ使用できます。

これは画像を隠すCSSコードです：

img {
    display: none;
}

私が繰り返します：このアプローチでは、ブラウザーは画像をダウンロードし、見えなくてもこれを実行します。ここでの要点は、要素<img>は交換可能な要素と見なされるため、CSSからそのような要素を管理する機能には制限があります。

<svg>で<image>を使用する


これが私たちの問題の最も興味深い解決策だと思います。私は新しい Facebookのデザインを研究しているときにこのトリックを発見しました。

ここにマークアップがあります：

<svg role="none" style="height: 36px; width: 36px;">
  <mask id="avatar">
    <circle cx="18" cy="18" fill="white" r="18"></circle>
  </mask>
  <g mask="url(#avatar)">
    <image x="0" y="0" height="100%" preserveAspectRatio="xMidYMid slice" width="100%" xlink:href="avatar.jpg" style="height: 36px; width: 36px;"></image>
    <circle cx="18" cy="18" r="18"></circle>
  </g>
</svg>

まず、このコードを解析してみましょう。これは次のとおりです。

  1. 円形の画像をトリミングするマスク。
  2. マスクが適用されるグループ。
  3. 属性を持つ画像自体preserveAspectRatio=«xMidYMid»。
  4. 内部の境界として使用される円。

これはスタイルの仕方です：

circle {
  stroke-width: 2;
  stroke: rgba(0, 0, 0, 0.1);
  fill: none;
}

アイコン付きの入力フィールド

printing印刷用のCSSスタイル


ページの訪問者は、プリンタで印刷する必要がある場合があります。そのページに料理のレシピがあり、キッチンの電話やコンピュータを常に見る必要がないように、それを印刷する必要があるとします。

写真で示されているレシピにステップがある場合、それらが印刷バージョンに該当することが重要です。そうしないと、ページを印刷する人がレシピを使用できなくなります。
