Spring Boot 2.2の新機能

記事の翻訳は、コース「Developer on the Spring Framework」の学生のために特別に準備されました。



2019年10月16日、Spring Boot 2.2がリリースされました。

この投稿では、バージョン2.2が提供する多くの新しい利点について学びます。

メイベン


Spring Boot 2.2を使い始める。アプリケーションで、新しいバージョンのstarter-parentにアップグレードします。

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
  <modelVersion>4.0.0</modelVersion>
  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.2.0.RELEASE</version>
    <relativePath/> <!-- lookup parent from repository -->
  </parent>
  <groupId>com.example</groupId>
  <artifactId>your-very-cool-project</artifactId>
  <version>0.0.1-SNAPSHOT</version>
  <name>your-very-cool-project</name>
  <description>Spring Boot 2.2 Project</description>

  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter</artifactId>
    </dependency>

    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-test</artifactId>
      <scope>test</scope>
      <exclusions>
        <exclusion>
          <groupId>org.junit.vintage</groupId>
          <artifactId>junit-vintage-engine</artifactId>
        </exclusion>
      </exclusions>
    </dependency>
  </dependencies>
</project>


アプリケーションを最初から開始する場合は、start.spring.ioでMavenプロジェクトまたはGradeプロジェクトを作成します。



グローバルレイジーBean初期化


Beanコンポーネントの遅延初期化は、Spring Frameworkで長い間サポートされてきました。

遅延して初期化されたBeanは、IoCコンテナに、起動時ではなく、最初のリクエスト時にBeanをインスタンス化するように指示します。


Spring Boot 2.2で導入された新機能は、Beanコンポーネントのグローバルな遅延初期化のサポートです（この機能はデフォルトでは無効になっています）。

グローバルレイジーBean初期化をオンにするとどうなりますか？

  • すべてのSpring Beanとその依存関係の初期化は、必要になるまで延期されます。


アプリケーションの起動時間を短縮するために、次のコマンドを使用して、設定プロパティでSpring Beanのグローバルな遅延初期化を有効にできるようになりました。

spring.main.lazy.initialization=true


またはyml設定の場合：

spring:
  main:
    lazy-initialization: true


レイジーBeanの初期化を使用することに不利な点はありますか？もちろん！結果を理解することは非常に重要です。この解決策を考慮せずにBeanコンポーネントのグローバル初期化を含めないでください。考慮すべきいくつかのトレードオフがあります。

  • 遅延初期化が発生している間は、HTTPリクエストの処理に時間がかかる場合があります。後続のリクエストは影響を受けません。
  • 起動時に通常発生する障害（アプリケーションコンテキストの作成時にSpring Beanとその依存関係が作成されるため）がすぐに発生しなくなりました。したがって、アプリケーションは明らかなスタートアップクラッシュを生成しなくなります。その結果、問題が最初に発生する可能性があります（豆の配線）。


Bean（spring.main.lazy.initialization = false）の遅延初期化をグローバルレベルで有効にしたくない場合は、アノテーションを使用して特定のコンポーネントの遅延初期化を設定することを検討できます@Lazy。

一方、Beanコンポーネント（spring.main.lazy.initialization = true）の遅延初期化をグローバルに有効にし、特定のコンポーネントの遅延初期化を明示的に無効にすることもできます。

JMXはデフォルトで無効になっています


Spring Boot 2.2以降、JMXはデフォルトで無効になっています。これにより、アプリケーションの起動時間が短縮され、実行時に大量のリソースが無駄になりません。それでもJMXに依存している場合は、再度有効にすることができます。

spring.jmx.enabled=true


または：

spring:
  jmx:
    enabled: true


構成プロパティの改善


Spring Boot 2.2には、構成プロパティにいくつかの素晴らしい改善が加えられています。

  • サポートスキャンクラスパスについて@ConfigurationProperties
  • 不変バインディング @ConfigurationProperties


クラスパスのスキャンのサポートについて@ConfigurationProperties


Spring Bootは@ConfigurationProperties、クラスパスをスキャンするときに見つかった注釈付き構成クラスごとにBeanを作成します。これは、

  • プロパティクラスをバインドするクラスの@EnableConfigurationProperties
  • または@Component、構成クラスでアノテーションを使用してBeanにします。


と@Component同様にアノテーションが付けられた2つのBeanコンポーネントを構成クラス用に作成できることに注意してください@ConfigurationProperties。このような場合@Componentは、構成クラスから削除する必要があります。

不変バインディング @ConfigurationProperties


コンストラクターベースのプロパティバインディングの使用で、不変の構成プロパティクラスがサポートされるようになりました。コンストラクタベースのバインディングは、クラス@ConfigurationPropertiesまたはそのコンストラクタの1つにで注釈を付けることで有効にできます@ConstructorBinding。

例えば：

package com.example.immutable.configuration.binding;

import lombok.Getter;
import org.springframework.boot.context.properties.ConfigurationProperties;
import org.springframework.boot.context.properties.ConstructorBinding;

@Getter
@ConfigurationProperties("stock.quote.subscription")
public class ImmutableStockQuoteSubscriptionProperties {

  private final String endpoint;
  private final String apiKey;
  private final SubscriptionType subscriptionType;
  private final boolean enabled;

  @ConstructorBinding
  public ImmutableStockQuoteSubscriptionProperties(String endpoint, String apiKey, SubscriptionType subscriptionType, boolean enabled) {
    this.endpoint = endpoint;
    this.apiKey = apiKey;
    this.subscriptionType = subscriptionType;
    this.enabled = enabled;
  }

  enum SubscriptionType {

    /**
     * Real time stock quotes
     */
    REALTIME,

    /**
     * Stock quotes that are delayed by 15 minutes
     */
    DELAYED,
  }
}


私のapplication.ymlのプロパティ：

stock:
  quote:
    subscription:
      endpoint: http://stockgeeks.io/api
      enabled: true
      apiKey: secret
      subscriptionType: realtime

RSocketサポート


RSocketは、バイトストリームの転送に使用されるバイナリ通信プロトコルです。これは、単一チャネルを介した非同期メッセージングを介した対称相互作用モデルを可能にします。

RSocketの新しいスターターとともに、広範な自動構成が追加されました。

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-rsocket</artifactId>
</dependency>


詳細については、RSocket Spring Bootのドキュメントをご覧ください。

バナー


Spring Bootにはデフォルトのバナーが付属しており、アプリケーションの起動直後にコンソールに表示されます。



これはかなりマニアックな機能ですが、Spring Bootはすでに（アニメーション化された）カスタムバナーをサポートしています。

Spring Boot 2.2以降では、バナーをさらに見栄えよくすることができます。

  • ASCIIバナーファイルで、{AnsiColor.NNN}（NNNはカラーコード）を使用してANSI 256カラーコントロール文字を使用できるようになりました。
  • （アニメーション）グラフィックバナーの場合、プロパティspring.banner.image.bitdepthをequalに設定できます8。また、プロパティspring.banner.image.pixelmodeはblockブロックASCII文字を使用することです。

Java EEからJakarta EEへの移行


可能な場合、Spring（Boot）コマンドは、Java EEの依存関係からグループ識別子に切り替えましたjavax。Jakarta EEの同等の依存関係とグループ識別子jakarta.Spring Bootスターター。

さらに、Jakarta EE依存関係APIの依存関係管理が、Java EE依存関係APIの既存の依存関係管理と共に追加されました。

Java EE API依存関係管理はSpring Bootの将来のバージョンで削除されることに注意してください。JakartaEE API依存関係の使用を開始することをお勧めします。

Spring Boot 2.2へのアップグレード


2019年8月1日以降、Spring Boot 1.xはライフサイクルを完了しました。Spring Boot 1.xアプリケーションをまだ使用している場合は、アップグレードする時が来ました！

Spring Boot 2.1で廃止されたクラス、メソッド、およびプロパティは、Spring Boot 2.2で削除されたことを覚えておいてください。アップグレードする前に、廃止されたメソッドを呼び出さないようにしてください。Spring Boot 2.1で非推奨となったリリースノートを確認してください。
