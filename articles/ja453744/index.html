疑似クラスの原則を研究します。「純粋なCSSでテーブルのアクティブな行を強調表示する」タスクの例を使用して（）

数日前、私の友人は私に彼のインタビューであった問題の解決を手伝うように頼んだ。インタビュアーは彼に次のコードスニペットを与えました：


tbody:hover tr {
  opacity: 0.24;
}

また、「テーブルにカーソルを合わせると、すべての行が薄くなります。そして、ポイントしたラインがアクティブなままであることを確認する必要があります。」



決定


現在のソリューションでは、tbody：hover要素にカーソルを合わせると、すべてのtr要素がすぐに不透明度を0.24に変更します。


tbody:hover tr {
  opacity: 0.24;
}

セレクターを変更して、指定したものを除くすべてのtr要素に不透明度が適用されるようにする必要があります。


このような実装では、疑似クラスではなくホバーを使用する必要があります。ホバーの助けを借りて、ブラウザは線がポイントされたと判断し、そうでないの助けを借りて、スタイルが適用されるべきではないことを理解します。


tbody:hover tr:not(:hover) {
  opacity: 0.24;
}

宿題


まとめるために、テキストで画像を表示する際のエラーを修正する必要がある宿題を用意しました。次のマークアップとスタイルを用意しました。


<main class="content">
  <p>Some text</p>
  <img src="picture.jpg" alt="some alt">
  <p>Some text</p>
</main>


.content img {
  margin-top: 35px;
  margin-bottom: 35px;
}

間違いは、画像が最初の要素である場合でも、上部にマージンがあることですが、それが存在しないことを確認する必要があります。


<main class="content">
  <img src="picture.jpg" alt="some alt">
  <p>Some text</p>
  <p>Some text</p>
</main>

したがって、画像が最後の要素である場合は、下からインデントを削除する必要があります。


<main class="content">
  <p>Some text</p>
  <p>Some text</p>
  <img src="picture.jpg" alt="some alt">
</main>
