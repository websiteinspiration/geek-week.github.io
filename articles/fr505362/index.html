Vite - Développement de leads sans bundles sur Vue

Salut, Habrovsk. En Juin, OTUS lance le développeur Vue.js cours . En prévision du début du cours, nous avons traditionnellement préparé une traduction de matériel intéressant.


Un jour, Ewan Yu (créateur de Vue.js) a décidé de ne pas dormir la nuit et a finalement créé Vite .



Vite vous permet de créer des applications sur Vue avec des composants à fichier unique sans l'étape d'assemblage des bundles. Les importations sont demandées par le navigateur en tant que modules ES natifs. Le serveur de développement intercepte les demandes de fichiers .vue et les compile à la volée. Et cela arrive instantanément.

Veuillez noter que Vite est toujours une solution expérimentale, et je ne sais pas ce qui se passera à l'avenir et combien de temps cet article restera utile. Considérez-le comme un apéritif ou tout simplement comme quelque chose d'amusant.

essayons


Créez les fichiers suivants:

Comp.vue

<template>
  <button @click="count++"> {{ count }}</button> 
</template>

<script>
export default {
  data: () => ({ count: 0 })
}
</script>

<style scoped>
button{
  font-size: 2rem;
}
</style>

index.html

<div id="app"></div>
<script type="module">
  import { createApp } from 'vue'
  import Comp from './Comp.vue'

  createApp(Comp).mount('#app')
</script>

Ensuite, procédez comme

npx vite

suit : allez dans http://localhost:3000, modifiez le fichier .vuepour voir les changements instantanés.

Assemblage de production


Créons maintenant une application prête pour la production .

Créerpackage.json

{
  "scripts": {
    "build": "vite build"
  }
}

Courir:

npm i -D vite
npm run build


Vérifiez le répertoire avec le code.
Problèmes rencontrés:

  • Erreur: impossible de trouver le module «tslib» résolu en
    exécutant npm i -D tslib
  • Erreur: ENOENT: aucun fichier ou directeur de ce type
    résolu en créant manuellement un répertoire vide

Pensées


Le démarrage a été très simple et le développement s'est accéléré, car j'ai immédiatement vu le résultat des changements. Quelque chose comme une compilation à la volée. Cela semble donc très prometteur, peut-être qu'après un certain temps, nous passerons à une telle méthode de développement pour Vue.

Evan Yu, fais ta magie.

Fait amusant: vue est traduite du français comme une vue, et vite aussi vite.

Il semble qu'à ce stade, le travail sur Vite soit terminé et puisse se poursuivre après Vue 3. La façon la plus simple d'essayer est Vue 3 beta.

J'ai dit qu'il a été réalisé sur la base d'un tweet d' Evan Yu. Cependant, en regardant les commits dans le référentiel, nous pouvons dire que le travail est toujours en cours.
