Windows ServerでスキャンするようにブラザーMFPを教える方法について簡単に



これは記事ではありませんが、Microsoft Windows Server環境でのブラザーMFP Scannerのネットワーク機能の使用方法に関する注記です。これは、デバイスドライバーの完全インストールについてです。ブラザーMFC-L2700DRNの例を検討します。正式には、スキャン機能はデスクトップOSでのみサポートされ、サーバーOSは印刷にのみ送信できます。解決策の検索中に、この特定の製造元の同様の問題を抱えたフォーラムによく出くわしました。それはすべて、FTP、SMB、または電子メールの使用に帰着します。

したがって、普遍的な解決策を以下に示します。

パート1.インストーラーの変更


スキャナーをインストールするには、完全なドライバーパッケージが必要です。私の場合のターゲットOSはWindows Server 2016なので、インストーラーはWindows 10用です。ダウンロードして解凍します。
次に、スキャナーを担当するドライバーを見つける必要があります。インストールフォルダ/ MFC-L2700DN_series含まBrinst_Common.iniのファイルを。その中で、[Specification]セクションにEnableServer = 0という行を見つけ、値を1に変更します。このパラメーターは、サーバーシステムのチェックを無効にします。ファイルを保存します。次に、[Driver]セクションでScannerInfName =という行を見つけます。このドライバーは変更されます。 ドライバーファイルはinstall \ driver \ gdi \ 32_64フォルダーにあります

。その中で次のことを行う必要があります。[Manufacturer]セクションを見つけ、リストの最後の小数点以下にNTamd64.XYを追加します。XとYはOSバージョンに対応します。こちらでチェックできます。サーバー2016はWindows 10に対応し、NTamd64.10.0と同じです

パート2.サーバーとインストール


インストールの小さな機能として、2回の再起動が必要です。これは、ドライバーのデジタル署名検証を無効にするためにサーバーを「テストモード」にする必要があるためです。インストール後、すべてをオンに戻すと機能します。

コマンドラインまたはPSを（必要に応じて）開き、bcdedit -set TESTSIGNING ONと入力してから、システムを再起動します。

標準的な方法で、インストール\ MFC-L2700DN_series \ setup.exeでブラザーインストーラーを実行します。ネットワークプリンタードライバーをインストールしないでください。インストールすると、インストーラーがエラーをスローし、不明なデバイスを手動で削除する必要があります（プリンターを追加することでインストールされます）。味わう残りのアイテム。



インストールが完了したら、コマンドラインまたはPSを開き、次のように入力します。bcdedit -set TESTSIGNING OFF、システムを再起動します。スキャナーをチェックしています。
