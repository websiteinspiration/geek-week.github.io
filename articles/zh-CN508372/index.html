改善Zimbra Collaboration Suite开源版中的SSL安全设置

将信息系统用于企业时，加密的可靠性是最重要的指标之一，因为它们每天都参与大量机密信息的传输。评估SSL连接质量的常用工具是Qualys SSL Labs进行的独立测试。由于该测试可以由任何人进行，因此对于SaaS提供者而言，该测试的最高分尤其重要。不仅SaaS提供商，而且普通企业也要注意SSL连接的质量。对于他们来说，此测试是一次很好的机会，可以提前发现潜在漏洞并弥补网络犯罪分子的所有漏洞。





在Zimbra OSE中，允许两种类型的SSL证书。第一个是自签名证书，该证书会在安装过程中自动添加。该证书是免费的，使用时间不受限制，这意味着它非常适合测试Zimbra OSE或仅在内部网络中使用。但是，进入Web客户端时，用户将在浏览器中看到警告，提示该证书不可靠，并且来自Qualys SSL Labs的测试肯定会使您的服务器失败。



第二个是由证书颁发机构签署的商业SSL证书。这样的证书容易被浏览器察觉，并且通常在Zimbra OSE的商业操作中使用。正确安装商业证书后，Zimbra OSE 8.8.15立即在Qualys SSL Labs的测试中显示A级。这是一个极好的结果，但是我们的目标是达到A +的结果。











为了在使用Zimbra Collaboration Suite开源版时在Qualys SSL Labs的测试中获得最高分，您需要执行以下步骤：



1.增加Diffie-Hellman协议的参数



默认情况下，在所有使用OpenSSL的Zimbra OSE 8.8.15组件中，Diffie-Hellman协议设置均为2048位。原则上，这足以在Qualys SSL Labs的测试中获得A +分数。但是，如果要从旧版本升级，则参数的值可能会更低。因此，建议更新完成后，运行命令zmdhparam set -new 2048，它将Diffie-Hellman协议参数增加到可接受的2048位，并且如果需要，可以使用同一命令将参数值增加到3072或4096位，一方面该值会增加但是，生成时间将对邮件服务器的安全级别产生积极影响。



2.包括推荐的使用密码列表



默认情况下，Zimbra Collaborataion Suite开源版支持各种强密码和弱密码，这些密码对通过安全连接传递的数据进行加密。但是，在检查SSL连接的安全性时，弱密码的使用是严重的缺点。为了避免这种情况，您必须配置使用的密码列表。



为此，请使用命令。zmprov mcf zimbraReverseProxySSLCiphers 'ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-AES256-GCM-SHA384:DHE-RSA-AES128-GCM-SHA256:DHE-DSS-AES128-GCM-SHA256:kEDH+AESGCM:ECDHE-RSA-AES128-SHA256:ECDHE-ECDSA-AES128-SHA256:ECDHE-RSA-AES128-SHA:ECDHE-ECDSA-AES128-SHA:ECDHE-RSA-AES256-SHA384:ECDHE-ECDSA-AES256-SHA384:ECDHE-RSA-AES256-SHA:ECDHE-ECDSA-AES256-SHA:DHE-RSA-AES128-SHA256:DHE-RSA-AES128-SHA:DHE-DSS-AES128-SHA256:DHE-RSA-AES256-SHA256:DHE-DSS-AES256-SHA:DHE-RSA-AES256-SHA:AES128-GCM-SHA256:AES256-GCM-SHA384:AES128:AES256:HIGH:!aNULL:!eNULL:!EXPORT:!DES:!MD5:!PSK:!RC4'







此命令将立即包含这组推荐密码，并且由于此原因，该命令可以立即在列表中包括可靠密码，并排除不可靠的密码。现在剩下的就是使用zmproxyctl restart命令重新启动具有反向代理的节点。重新启动后，更改将生效。



如果此列表由于某种原因不适合您，则可以使用命令从其中删除许多弱密码zmprov mcf +zimbraSSLExcludeCipherSuites



。因此，例如，一个zmprov mcf +zimbraSSLExcludeCipherSuites TLS_RSA_WITH_RC4_128_MD5 +zimbraSSLExcludeCipherSuites TLS_RSA_WITH_RC4_128_SHA +zimbraSSLExcludeCipherSuites SSL_RSA_WITH_RC4_128_MD5 +zimbraSSLExcludeCipherSuites SSL_RSA_WITH_RC4_128_SHA +zimbraSSLExcludeCipherSuites TLS_ECDHE_RSA_WITH_RC4_128_SHA



完全消除了RC4密码使用的团队。您可以对AES和3DES密码执行相同的操作。



3.启用HSTS启用



连接加密和TLS会话恢复的机制也是从Qualys SSL Labs获得测试最高分数的先决条件。要启用它们，您必须输入命令zmprov mcf +zimbraResponseHeader "Strict-Transport-Security: max-age=31536000"



。此命令将在配置中添加必要的标头，并且要使新设置生效，您将必须使用zmcontrol restart命令重新启动Zimbra OSE 。



Qualys SSL Labs的测试已经在这个阶段证明了A +分数，但是，如果您想进一步提高服务器的安全性，可以采取多种措施。







例如，您可以启用进程间连接的强制加密，以及在连接到Zimbra OSE服务时启用强制加密。要验证进程间连接，请输入以下命令： 



zmlocalconfig -e ldap_starttls_supported=1
zmlocalconfig -e zimbra_require_interprocess_security=1
zmlocalconfig -e ldap_starttls_required=true


要启用强制加密，必须输入： 



zmprov gs `zmhostname` zimbraReverseProxyMailMode
zmprov ms `zmhostname` zimbraReverseProxyMailMode https

zmprov gs `zmhostname` zimbraMailMode
zmprov ms `zmhostname` zimbraMailMode https

zmprov gs `zmhostname` zimbraReverseProxySSLToUpstreamEnabled
zmprov ms `zmhostname` zimbraReverseProxySSLToUpstreamEnabled TRUE




多亏了这些命令，所有与代理服务器和邮件服务器的连接都将被加密，并且所有这些连接都将被代理。







因此，按照我们的建议，您不仅可以在测试中获得SSL连接安全性的最高分，而且可以显着提高整个Z​​imbra OSE基础架构的安全性。
