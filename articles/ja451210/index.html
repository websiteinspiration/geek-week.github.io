JiraとGitLabの統合

目的


gitにコミットするときに、コメントでJiraからのタスクに名前を付けます。その後、2つのことが起こります。

  • GitLabでは、タスクの名前がJiraでのアクティブなリンクに変わります

  • Jiraでは、コメントがタスクに追加され、コミットとそれをコミットしたユーザーへのリンクと、言及のテキストが追加されます

カスタマイズ


  1. 書き込み権限を持つJiraユーザーが必要です。既存のものを使用できます。gitからのタスクについて言及する場合、Jiraのすべてのコメントはこのユーザーに代わるものであることを覚えておくことが重要です。そのため、新しいものを作成し、GitLabなどの名前を付けて、すべてのプロジェクトで書き込み権限を持つJiraに追加することをお勧めします。
  2. 接続する各プロジェクトで管理者権限を持つGitLabユーザーが必要です。プロジェクトごとに、統合は個別に構成されます。
  3. GitLabでプロジェクトを開き、[設定] -> [ 統合]に移動します。下にスクロールすると、接続できるサービスの長いリストを含むプロジェクトサービスが表示されます。

  4. このリストでJiraを見つけると、フォームが表示されます
