Google Cloud RunでのLaravel 6の継続的インテグレーションウォークスルー

みなさん、こんにちは！「Framework Laravel」および「PHPのバックエンド開発者」コースの開始を期待して、このトピックに関する興味深い資料を引き続き共有します。行け。



過去数年間で、Laravelの人気は劇的に高まりました。 Laravelコミュニティーは、LaravelがPHPでの作業プロセスを以前よりも面白くしたとの見方さえしています。 Laravel 6にはクールな新機能がいくつかあります。アプリケーションの非常にスケーラブルな作業用URLを取得するには、数日ではなく数時間かかります。そして、Kubernetesのようなものを作成することは困難な作業です。このような場合にGoogle Cloud Runが最適なのは、コンテナアプリケーションのHTTPs URLを数分で取得できるためです。

Google Cloud Runサーバーレスで、Googleが完全に管理しています。スーパースケーリング、1秒あたりの支払い、HTTPs URL、および独自のドメインマッピングを取得します。

ステートレスコンテナーを実行する場合は、Cloud runが最も簡単な方法です。この記事では、Laravel 6アプリケーションをGoogleクラウドで実行し、Continous Integration（CI）で実行する方法を詳しく説明します。

必要な要件


  • PHP / Composerに精通していて、Laravelについて知っている（そうでなければ、ここにいないと思います）。
  • コンソールからGitを使用する方法を知っています。
  • CI / CDコードはGitHubでホストされており、その使用方法を知っています。
  • あなたはDockerに精通していて、多段階のアセンブリにさえ精通しているかもしれません。
  • 有効なGoogle Cloudアカウントを持っている（彼らはあなたに利息を与える- 1年間$ 300の無料ローン、それを開始しない理由はない）。

Cloud Runが初心者に最適な理由


以下に2つの理由で説明します。

  • DockerやCI / CDなどのベストプラクティスとソフトウェアについて学びます。
  • メインフレームは、ボタンを押した後、2つの選択と5分の待機を選択すると機能し、HTTPS URLを取得できます。どのくらい簡単ですか？

これは単なるデモなので、実行できますが、実際のアプリケーションでは、秘密に注意してください。実稼働対応アプリケーションの場合、デバッグおよび開発に関連するその他のことを無効にします。

そのようなDockerfileをプロジェクトのルートに追加します。

FROM composer:1.9.0 as build
WORKDIR /app
COPY . /app
RUN composer global require hirak/prestissimo && composer install

FROM php:7.3-apache-stretch
RUN docker-php-ext-install pdo pdo_mysql

EXPOSE 8080
COPY --from=build /app /var/www/
COPY docker/000-default.conf /etc/apache2/sites-available/000-default.conf
COPY .env.example /var/www/.env
RUN chmod 777 -R /var/www/storage/ && \
    echo "Listen 8080" >> /etc/apache2/ports.conf && \
    chown -R www-data:www-data /var/www/ && \
    a2enmod rewrite

次のファイルをに追加しますdocker/000-default.conf。

<VirtualHost *:8080>

  ServerAdmin webmaster@localhost
  DocumentRoot /var/www/public/

  <Directory /var/www/>
    AllowOverride All
    Require all granted
  </Directory>

  ErrorLog ${APACHE_LOG_DIR}/error.log
  CustomLog ${APACHE_LOG_DIR}/access.log combined

</VirtualHost>

その後追加しdocker-compose.ymlます：

version: '3'
services:
  app:
    build:
      context: ./
    volumes:
      - .:/var/www
    ports:
      - "8080:8080"
    environment:
      - APP_ENV=local

すべてを主要なものに減らしましょう。

ここに書かれているすべてを理解しようとしている場合、それは網羅的である可能性があるので、主な部分について教えてください：

  • Laravelの実行には、公式のPHP Apache Dockerイメージを使用します。これには、PHPバージョン7.3が含まれています。
  • マルチステージビルドを使用してComposerとの依存関係を取得し、それらをPHP 7.3とApacheを含むメインのDockerイメージにコピーします。
  • Google Cloud Runは、ポート8080でリッスンするWebサーバーを必要とするため、000-default.confこれを構成するために使用します。
  • docker-composeを実行しやすくするには、docker-compose upコマンドを1つだけ入力します。
  • この時点まで読んだので、ルートからdocker-composeを実行すると、すべてがアドレスhttp://localhost:8080で機能するはずです。この場合、Laravel 6がDockerでローカルに実行されていることがわかります。
