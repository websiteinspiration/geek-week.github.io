PythonのBefungeコンパイラ

4年生向けの「コンパイラの基礎」コースの準備として、さまざまな難解なプログラミング言語を勉強しました。これはこのトピックに関する良い記事です。Befunge言語（Chris Press、1993年）は、記事の中で私にとって最も興味深いように思われました。特に、その3つの機能に注目します。

  1. プログラムフィールドは2次元トーラスです。物理的には、これは、上（下）の境界と左（右）の列に沿って閉じたシンボルコマンドの長方形の行列です。コマンドポインタはフィールド内を移動し（各コマンドはx、y座標を持つ特定の文字です）、コマンドを実行し、次に進みます。移動は4つの方向すべてに可能です（デフォルトでは、ポイント0,0から右）。「フィールド」を超えると、ポインターが反対側に表示されます。
  2. 言語自体には、フィールド自体を変更する2つのコマンド（p、g）があります。プログラムは実行中に「自己書き換え」します。開始時のプログラムコードは、終了時のコードと異なる場合があります。プログラム「123pgpg ## @」が開始し、プログラム「ABC @ 1 @ 2 @ 3.14」（正しい例ではありません）が動作を終了しました。
  3. Chris Pressyは、コンパイルが可能な限り複雑な言語を作成したいと述べた。事実、それは事実であり、exeファイルをプログラムにとって非常に困難にするコンパイラを作成し、誰かがCでそれを実行できるという情報を見つけました...言語からPythonコードへのトランスレータを作成するのが最善です。単純化のためのコンパイラ。

class Pointer:
    def __init__(self, x=0, y=0, vector=2, value=None):
        self.x = x
        self.y = y
        self.vector = vector
        self.value = value
        self.stack = []
        self.stack_sf = 0

    def __str__(self):
        return 'Point ({},{}) vektor:{} value:{} stack_sf:{} stack:{}'.format(self.x, self.y, self.vector, self.value, self.stack_sf, self.stack)

    def step(self):
        if self.vector == 1:
            self.x -= 1
        elif self.vector == 2:
            self.y += 1
        elif self.vector == 3:
            self.x += 1
        elif self.vector == 4:
            self.y -= 1
