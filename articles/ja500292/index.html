レスポンシブ画像の出力には、かなり複雑なロジックの使用が含まれます。これには、特に、表示される画像の決定や、ユーザーが高解像度画面で作業しているかどうかの決定が含まれます。幸いなことに、ブラウザは、特定の各ユーザーに最適な画像を判断できる人よりも優れています。私たちがする必要があるのは、彼らにいくつかの手がかりを与えることです。この属性srcsetは、ブラウザにグラフィックリソースのリストを提供し、そこから最適な画像を選択できます。この属性をsizes使用すると、特定の場合に表示する画像のサイズをブラウザに指示できます。



ちなみに、レスポンシブな画像を使用すれば、このテクノロジーのブラウザサポートについて心配する必要はありません。私たちが興味を持っている属性は、優れたブラウザサポートを楽しんでいます。さらに、IE11などの古いブラウザ用に設計されたバックアップメカニズムを自由に利用できます。

この属性にsizesは、コンマで区切られたメディア条件のリストが含まれています。メディア条件は、メディアクエリのサブセットです。ここでは、条件（printまたはscreen）が適用される環境のタイプを指定することはできませんが、表示領域の幅に関連するメディアクエリを使用できます。

各リストエントリには、ビューポートウィンドウの幅と対応する画像の幅が含まれます。リストアイテムの形式は(min-width: 1023px) 780pxです。このようなレコードは、ビューポートの幅が1023ピクセル（またはそれ以上）の場合、幅が780ピクセルの画像を使用する必要があることをブラウザに伝えます。

ここでは、さらに、幅の相対的な単位を使用できます。たとえば、次のようなもの50vw。これは、画像の幅がビューポートの幅の50％になることをブラウザに伝えます。ここでは、より複雑な状況で関数を使用することもできますcalc。たとえば、ビューのデザインcalc(50vw — 2rem)は、画像の幅がビューポートの幅から50％を引いたものになることをブラウザに通知し2remます。おそらく、幅はこのように設定されて、インデントまたは境界の幅を考慮に入れます。

sizesサイズ属性に入力する必要がある値は？


属性sizesに入力する必要がある値を理解するには、CSSを分析し、さまざまな条件で表示される画像の幅を理解する必要があります。

これは画像自体の幅によって決まる場合があります。

img {
  width 320px;
}

@media screen and (min-width: 37.5em) {
  width: 640px;
}

この場合、2つの固定画像サイズオプションがあります。この事実は、属性に直接反映できますsizes。

<img
  alt="Ferrari"
  src="ferrari.jpg"
  srcset="ferrari-s.jpg 320w,
          ferrari-m.jpg 960w,
          ferrari-l.jpg 1920w"
  sizes="(min-width: 37.5em) 640px, 320px">

ただし、多くの場合、画像のサイズは柔軟です。多くの場合、イメージはコンテナからサイズを継承します。

img {
  height: auto;
  width: 100%;
}

.container {
  padding: 1rem;
  width: 100%;
}

@media screen and (min-width: 37.5em) {
  .container {
    width: 50%;
  }
}

この例では（画像の幅に影響を与える唯一の要素がであると想定.container）、コンテナーの幅は画像の幅と見なすことができます。ここでは、内側のインデントの幅がコンテナの幅から差し引かれていることに注意する価値があります。あなたはこれを考慮する必要があるかもしれませんが、多分そうではありません。それはすべて、インデントが画像の最終的な幅にどの程度影響するかに依存します。

<img
  alt="Ferrari"
  src="ferrari.jpg"
  srcset="ferrari-s.jpg 320w,
          ferrari-m.jpg 960w,
          ferrari-l.jpg 1920w"
  sizes="(min-width: 37.5em) calc(50vw - 2rem),
         calc(100vw - 2rem)">

ご覧のとおり、属性の設定sizesは特定のレイアウトに大きく依存しています。通常、私はブラウザー開発者ツールで画像を調査し、どのオプションが画像のサイズに影響を与えるかを理解することから始めて、作業を構築します。

レスポンシブ画像の正しい設定を確認するにはどうすればよいですか？


レスポンシブな画像の正しい設定をテストするのは難しく、時間がかかることは容易に想像できます。しかし、私たちの幸せのために、そのような問題の解決を簡単にするツールがあります。これはレスポンシブ画像リンターです。

これは、ブラウザに追加して自分のサイトで使用できるブックマークレットです。呼び出されると、さまざまなサイズの表示領域と画面のピクセル密度を使用してページを自動的にスキャンし、画像をテストします。

次に、ページのすべての画像に関する情報を含むレポートを表示し、サイズの管理が正しく構成されているかどうかを確認します。ページのテスト中に問題が発生した場合は、そのことが通知され、問題の修正方法についてもアドバイスされます。
