Pleskレビュー-ホスティングおよびサイトコントロールパネル

Pleskは、サイトとWebアプリケーションまたはWebホスティングを管理するための日常業務をすべてすばやく効率的に実行するための強力で便利なユニバーサルツールです。「世界の6％のサイトはPleskパネルを通じて管理されています」と開発会社はプラットフォームについてHabréの企業ブログで述べています。この便利でおそらく最も人気のあるホスティングプラットフォームの概要を紹介します。このライセンスは、RUVDSのVPSサーバーを使用して年末まで無料で購入できます。



panelsパネル、ブランド、会社について


Pleskはノボシビルスクで開発されたプロプライエタリソフトウェアで、2001年に米国で最初にリリースされました。ほぼ20年間、このプラットフォームの権利は、ブランドや名前を変更したさまざまな企業によって次々と取得されました。2015年以降、Pleskは複数の支店（ノボシビルスクを含む）と約500人のスタッフ（本社と支店の両方のロシアの専門家を含む）を持つ独立したスイスの会社です。 

3つの最新バージョン： 

  • Plesk 12.5（2015）
  • Plesk Onix（2016年〜2019年）
  • Plesk黒曜石（2020）

パネルは多言語です。 PHP、C、C ++で書かれています。 PHPの複数のバージョン、Ruby、Python、NodeJSのサポート。 Gitの完全サポート。 Docker統合SEOツールキット。各Pleskインスタンスは、SSL / TLSを使用して自動的に保護されます。 

サポートされるOS：WindowsおよびLinuxのさまざまなバージョン。以下に、これらのOSの要件を示します。


Linux


Windows 

プログラムはさまざまなアセンブリでリリースされ、それぞれが独自のユーザー向けに設計されています。たとえば、このパネルを使用すると、管理者は単一のWebインターフェースを使用してすべてのシステムサービスを集中管理でき、メンテナンスコストを削減して、必要なレベルの柔軟性と制御を提供できます。また、仮想および専用ホスティングを販売している企業の場合、このパネルを使用すると、サーバーリソースをパッケージに編成し、これらのパッケージを顧客（インターネットでサイトをホストしたいが、これに必要なITインフラストラクチャがない企業または個人）に提供できます。

インターフェース


Pleskでは、各ユーザグループに、ニーズに合わせた独自のインターフェースがあります。ホスティングプロバイダーのインターフェイス-事業を自動化するための統合請求システムを含む、ホスティングを提供するためのツール。プラットフォームを使用して独自のWebインフラストラクチャを管理する企業は、システムの復旧やWebサーバーのセットアップなど、さまざまなサーバー管理操作を使用できます。プラットフォームの最後の2つのバージョン（Plesk OnyxとPlesk Obsidian）をWeb管理者の目を通して見てみましょう。

webウェブ管理者向けの機能


ユーザーアカウント。独自の資格情報を持つ個別のユーザーアカウントを作成します。各ユーザーまたはユーザーグループのユーザーロールとサブスクリプションを定義します。

定期購入サービスプランに関連する特定のリソースとサービスのセットでサブスクリプションを作成し、ユーザーの役割に従ってユーザーにアクセスを許可します。特定のサブスクリプションで使用できるシステムリソース（CPU、RAM、ディスクI / O）の量を制限します。

ユーザーの役割個々のユーザーの機能とアイコンのオンとオフを切り替えます。同じサブスクリプションレベルで、異なるユーザーに異なるアクセスレベルを提供します。

サービスプラン。リソースの配分を決定するサービスプランを作成します。たとえば、ディスクスペース、帯域幅、顧客に提供されるその他の機能などです。 

メールサーバーのサポート。デフォルトでは、PostfixメールサーバとCourier IMAPはPlesk for Linuxに、MailEnableはPlesk for Windowsにインストールされています。

DKIM、SPF、DMARC保護。 Pleskはメール認証用にDKIM、SPF、SRS、DMARCをサポートしています。

対応OS Plesk for Linux / Unixの最新バージョンは、Debian、Ubuntu、CentOS、Red Hat Linux、CloudLinuxなどのいくつかのプラットフォームをサポートしています。

データベース管理。サポートされているデータベースのスキャン、復元、レポート、修復。

そのままでPCI DSS規格に準拠。サーバーを保護し、LinuxサーバーでPCI DSSコンプライアンスを実現します。 

タスク計画。特定のコマンドまたはタスクを実行する日時を設定します。

システムアップデート。サーバー上で利用可能なすべてのシステムパッケージを、コンソールを開かずに手動または自動で更新します。

Plesk移行。コマンドラインを使用する必要のない移行。サポートされているソース：cPanel、Confixx、DirectAdminなど。

サーバー管理者は、必要に応じて、サーバー管理パネルの外観、コントロール、さらにはロゴを変更することができます。インターフェース設定を変更するマーケティングの目的でも、単に仕事の便宜のためでも可能です。独自のテーマを使用できます。管理者向けガイドの詳細。
