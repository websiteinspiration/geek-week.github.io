組み込みモード「拡大鏡」について


システムのタブバーには、各ロングタップタブのプレビューが表示されます。「利用可能な」テキストサイズを含めてから、システムタブの任意のタブ（たとえば、音楽や写真）を長押しします。次のポップアップが画面に表示されます。


タブバーに加えて、「虫眼鏡」モードは、ナビゲーションバーとツールバーのボタンでサポートされています。

また、iOS 13以降、同じ動作をコントロールに簡単に追加できます。

UILargeContentViewerItemをアプリケーションに固定する方法


残念ながら、このプロトコルはドキュメントではかなり不十分に説明されていますが、APIは非常にシンプルに見えるため、これは問題ではありません。

@available(iOS 13.0, *)
public protocol UILargeContentViewerItem : NSObjectProtocol {
    var showsLargeContentViewer: Bool { get }

    var largeContentTitle: String? { get }

    var largeContentImage: UIImage? { get }

    var scalesLargeContentImage: Bool { get }

    var largeContentImageInsets: UIEdgeInsets { get }
}

これらのメソッドのいくつかを実装するだけで十分です。これらのメソッドの名前はそれ自体がわかります。コントロールを長押しすると大きなプレビューが表示されます。

非常にシンプルな実装


アプリケーションのいくつかのボタンでこのプロトコルを試してみましょう。たとえばi、製品カードのボタンについて：


3行のコードを追加します。

nutritionButton.showsLargeContentViewer = true
nutritionButton.addInteraction(UILargeContentViewerInteraction())
nutritionButton.largeContentTitle = nutritionButton.accessibilityValue


できました。いいね？

ここでのみ、デフォルトで引っ張られる偶数アイコンがありますが、私は好きではありません。2色ではなく、文字の形を切り抜いた1色の方が良いでしょうi。これは単に.highlightedボタンの状態に使用され  ます。

nutritionButton.largeContentImage = nutritionButton.image(for: .highlighted)


ノルム

そして今、拡張機能でまとめましょう：

extension UIView {
    public func enableLargeContent(title: String? = nil,
                                   image: UIImage? = nil,
                                   scales: Bool = true,
                                   insets: UIEdgeInsets = .zero) {
        guard !showsLargeContentViewer else { return }
        
        showsLargeContentViewer = true
        addInteraction(UILargeContentViewerInteraction())
        
        largeContentTitle = title
        largeContentImage = image
        scalesLargeContentImage = scales
        largeContentImageInsets = insets
    }
}

nutritionButton.enableLargeContent(title: nutritionButton.accessibilityValue,
                                   image: nutritionButton.image(for: .highlighted))

実際、多くの標準iOSコントロールは、すでにこのプロトコルのいくつかのメソッドを実装しています。拡大鏡の後ろにすべてを隠すのではなく、コンテンツを実際に増やす必要があるため、このモードはオフになっています。

言い換えると、この機能は、コントロールをまったく増やせない最後の手段としてのみ使用してください。
インターフェースを開くと、このすべてのドキュメントがXcodeで利用可能になりますUILargeContentViewerItem。
