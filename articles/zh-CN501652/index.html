Nuxt作为全栈服务器：前端+后端API服务器（第1部分）



  • 第1部分：Nuxt作为全栈服务器：前端+后端API服务器
  • 第2部分：Nuxt全栈服务器的其他SSR性能

Nuxt 开发人员提供了三种访问API的方法：

  1. 内置Nuxt Server 连接并使用serverMiddleware
  2. 集成框架（Express，Koa，Hapi等）
  3. 外部服务器API

我将通过一个简单的示例演示如何在我们与前端使用的同一Nuxt实例上使用serverMiddleware来组织API服务器。

这种方法有什么好处：

  • 快速启动和最小化技术堆栈，仅需要Nuxt即可，它将成为一个全栈服务器：呈现前端页面并充当后端API服务器

  • , . , , .
  • Nuxt Connect, Node () 55 . , Nuxt' (Express, Koa, Hapi ..) .
  • 尽管有极简主义，但我们有完全的自由来扩展应用程序，可以轻松部署与一个数据库交互的多个全栈Nuxt服务器，还可以部署nginx来平衡多个全栈Nuxt服务器之间的负载。同样，通过小的修改，每个Nuxt实例都可以充当唯一的API服务器（唯一的后端服务器），并且可以充当前端服务器（唯一的前端服务器）。
  • 和巨大的奖金！当在前端和后端使用一个实例时，可以显着加快服务器页面呈现（SSR），正如我将在第2部分中讨论的那样

    export default async (req, res, next) => {
  let url = req._parsedUrl.pathname.replace(/^\/+|\/+$|\.+/g, "");
  url = url.split("/");
  let method = url.pop();
  let controller = url.slice(1).join("/");
  let api = require("../api/" + controller);
  let result = await api[method](req.params);
  res.end(JSON.stringify(result));
};

        const url = require("url");
        req.query = url.parse(req.url, true).query;
        req.params = { ...req.query, ...req.body };
        next();
      }

async function getDataFromDB() {
  return (await require("axios").get(`https://reqres.in/api/users?page=1`))
    .data;
}

async function index() {
  let res = await getDataFromDB();

  // some business logic
  let ret = res.data.map(el => {
    return {
      id: el.id,
      full_name: el.first_name + " " + el.last_name,
      highlight: el.first_name.charAt(0) === "J"
    };
  });
  ret = ret.sort(() => 0.5 - Math.random());

  return ret;
}

export { index };

    /plugins/api-context.js
    export default (context, inject) => {
    inject('api', async (controller, method, params) => {
        try {
            return await context.$axios['$' + (params ? 'post' : 'get')]('/api/' + controller + '/' + method, params);
        } catch (e) {
            console.error(e);
            throw e;
        }
    })
}
