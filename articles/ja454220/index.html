2桁温度計



著者は、この2桁のLED温度計を友人の息子への誕生日プレゼントとして作成しました。彼はまだ2歳で、すでに数字を読んでいますが、文字はそうではありません。これで彼は自分で窓の外の温度を知ることができます。温度計のセンサーは、1-Wireプロトコルで動作するDS18B20チップであり、マイクロコントローラーはATtiny84タイプです。ボードは一辺が25 mmの正方形で、サイズは50ペンスのコインに匹敵します。著者は、ボードを防水ケースに入れ、窓の外に置くことを計画しています。表示は24秒ごとに短時間オンになり、CR2032バッテリーは約1年間持続します。
温度計は-19から+99°Cの範囲で動作します。必要に応じて、マイナスと1を同時に上位に表示します。範囲外では、文字LoまたはHiが表示されます。ドットがマイナスになっているセグメントを使用して、デバイスを-19°C未満の温度で表示するように「教える」ことができます。

このスキームによると、デバイスは以前にブレッドボードに組み立てられていました。



マイクロコントローラーのすべての出力が関与し、組み込みの8 MHzクロックジェネレーターが使用されていました。プロトタイプは次のとおりです。



プロトタイプでは、TO-92パッケージのDS18B20、PDIPパッケージのATtiny84、および3.6インチの3621ASインジケーターを使用しました。次に、著者はイーグルでボードを開発し、PCBwayで注文しました。ここでは、マイクロコントローラーはすでにSOICケースにあり、センサーはµSOPケースにあり、抵抗器、コンデンサー、およびディスプレイはサイズが0805です。ディスプレイ以外のすべてのものは、250℃の温度でYouyue 858D +ドライヤーではんだ付けされています。

プロトタイプとプリント基板の両方に、アノードが共通のインジケータがあります。このデバイスは、赤と黄色のインジケーターが付いた2つのバージョンで作られています。赤-KDPVの場合、黄色-ここ：



20 mmリチウム電池のホルダー（指定が20で始まるもの、つまり2016、2025、2032のいずれか）が裏面にはんだ付けされています。



ファームウェアは、マイクロコントローラーがほとんどの場合スリープモードになり、ウォッチドッグタイマーからの割り込みでウェイクアップするように書き込まれます。1-Wireインタフェースの実装には、同じ作者のこの動作時間が含まれます。1 MHzの周波数で動作するマイクロコントローラーの16ビットタイマーカウンターは時間がかかります。

void OneWireSetup () {
  TCCR1A = 0<<WGM10;                   // Normal mode
  TCCR1B = 0<<WGM12 | 2<<CS10;         // Normal mode, divide clock by 8
}

DelayMicros（）サブルーチンは、OCR0A出力比較レジスターに基づいて、指定されたマイクロ秒数の遅延を提供します。

void DelayMicros (unsigned int micro) {
  TCNT1 = 0; TIFR1 = 1<<OCF1A;
  OCR1A = micro;
  while ((TIFR1 & 1<<OCF1A) == 0);
}

DisplayTemperature（）ルーチンは、センサーから温度値を読み取り、それを表示します。バスにはセンサーが1つしかないので、シリアル番号を無視して、Skip ROMコマンドを発行するだけで済みます。その後、次のすべてのコマンドが任意のデバイスに送信されます。

void DisplayTemperature () {
  cli();                                  // No interrupts
  if (OneWireReset() != 0) {
    sei();
    DisplayError(0);                      // Device not found
  } else {
    OneWireWrite(SkipROM);
    OneWireWrite(ConvertT);
    while (OneWireRead() != 0xFF);
    OneWireReset();
    OneWireWrite(SkipROM);
    OneWireWrite(ReadScratchpad);
    OneWireReadBytes(9);
    sei();                                // Interrupts
    if (OneWireCRC(9) == 0) {
      int temp = DataWords[0];
      Display((temp+8)>>4);               // Round to nearest degree
    } else DisplayError(1);               // CRC error
  }
}

センサーは、リクエストに応答して、1/16度に等しい単位の16ビット符号付き整数の形式で温度値を返します。数値は最も近い整数度に丸められ、Display（）ルーチンを呼び出すことによって表示されます。

DisplayError（）サブルーチンは、1-Wireバス上のセンサーとマイクロコントローラーの相互作用におけるエラーを表示します。

void DisplayError (int no) {
  Buffer[0] = Error;
  Buffer[1] = no;
}

E0-センサーが検出されなかった、E1-CRCエラー。

動的表示のデータは、Buffer []配列から取得されます。たとえば、20という数字を表示するには、次のようにする必要があります。

Buffer[0]=2; Buffer[1]=0;

タイマーカウンター0は、125 Hzの周波数で割り込みを生成します。これは、フリッカーを除去するのに十分です。最初に、タイマーはセットアップで設定されます（） "

TCCR0A = 2<<WGM00;                      // CTC mode; count up to OCR0A
  TCCR0B = 0<<WGM02 | 4<<CS00;            // Divide by 256
  OCR0A = 250-1;                          // Compare match at 125Hz
  TIMSK0 = 0;                             // Interrupts initially off

比較中の対応する割り込み処理ルーチンは、DisplayNextDigit（）ルーチンを呼び出してから、反対方向にカウントします。

ISR(TIM0_COMPA_vect) {
  DisplayNextDigit();
  Ticks--;
}

DisplayNextDigit（）ルーチンは、Buffer []配列の対応するセルからデータを読み取り、対応する表示ビットに目的のセグメントを含めます。プログラムは#defineを使用して、共通のカソードまたはアノードを持つインジケーターから選択します。電源投入直後にすべてのセグメントが点灯している場合、ディスプレイのタイプはファームウェアで指定されているものと一致しません。共通カソードの場合、サブルーチンを次のものに置き換える必要があります。

void DisplayNextDigit () {
  PORTB = PORTB | 1<<digit;                    // Turn old digit off
  digit = digit ^ 1;                           // Toggle between 0 and 1
  char segs = charArray[Buffer[digit]];
  PORTA = segs;                                // Lit segments high
  PORTB = PORTB & ~(1<<digit);                 // Turn new digit on
}

最後に、Display（）ルーチンは、Buffer []配列に書き込む2桁の数値を生成します。

void Display (int n) {
  int units = n % 10;
  int tens = n / 10;
  int temp0 = tens;
  int temp1 = abs(units);
  if (tens < -1) {temp0 = Lo; temp1 = Lo+1; }
  else if (tens > 9) {temp0 = Hi; temp1 = Hi+1; }
  else if (tens == -1) temp0 = Minus1;
  else if ((tens == 0) && (units >= 0)) temp0 = Blank;
  else if ((tens == 0) && (units < 0)) temp0 = Minus;
  Buffer[0] = temp0;
  Buffer[1] = temp1;
}

また、マイナスと上位の単位を一緒に表示する場合や、範囲外の温度に関するメッセージも考慮されます。

最大限のエネルギー節約のために、ADC、USI、およびADCクロックジェネレーターは無効になり、PWR_DOWNスリープモードが有効になります。

  ADCSRA &= ~(1<<ADEN);                   // Disable ADC to save power
  PRR = 1<<PRUSI | 1<<PRADC;              // Turn off clocks to USI & ADC to save power
  set_sleep_mode(SLEEP_MODE_PWR_DOWN);

メインプログラムは、温度を1/10秒表示してから、スリープモードをオンにします。これは、読み取りに便利な最小表示時間であることがわかりました。温度が表示される2秒前に、ドットが短く点滅します。

void loop () {
  Buffer[0] = DP; Buffer[1] = Blank;
  DisplayOn(12);
  WDDelay(6);                              // Sleep for 1 second
  Buffer[0] = Blank; Buffer[1] = DP;
  DisplayOn(12);
  WDDelay(6);                              // Sleep for 1 second
  DisplayTemperature();
  DisplayOn(12);
  WDDelay(9);                              // Sleep for 8 seconds
  WDDelay(9);                              // Sleep for 16 seconds
  WDDelay(9);                              // Sleep for 24 seconds
}

ウォッチドッグコールがそれぞれ8秒間行われるため、ディスプレイは24秒間オフのままです。インジケーターの実行中、消費電流は6.6 mA、スリープモードでは4.7μA、平均消費電流は1/240 * 6.6 mAです。 CR2032セルの標準的な容量は225 mAhであるため、（225 / 6.6）x 240/24 = 340日-1年弱です。

コンポーネントの温度範囲は次のとおりです：マイクロコントローラーおよびインジケーター--40〜+ 85°C、抵抗器およびコンデンサー-55〜+125°C、バッテリー--20〜+70°C。拡張温度範囲BR2032のエレメントは、-30〜+85°Cの範囲で動作します。

マイクロコントローラーがArduinoをこの開発に対応スペンス・コンデ。IDEのボードメニューのATTinyCoreセクションでATtiny24 / 44/84を選択します。次に、残りの部分に注意を払うことなく、次のオプションを設定する必要があります。

Chip: "ATtiny84"
Clock: "8 MHz (internal)"
B.O.D: "B.O.D. Disabled"
Pin Mapping: "Clockwise (like damellis core)"

プログラムは、Pomonaテストクリップデバイスを使用してアップロードされ、マイクロコントローラーの上に配置され、SparkFun Tiny AVR Programmerに接続されます。最初にBurn Bootloaderを選択し、次にUploadを選択する必要があります。
