AWS +テレグラムを使用してネイティブフィードバックフォームに反応する



最後のプロジェクトの1つにフィードバックフォームを統合する前に、問題を解決するための非標準的なアプローチを見つけようとしました。電子メールには何らかのSMTPサーバーが必要であり、特にメッセージフローが低い新興企業にとっては、退屈なだけです。

メッセンジャーズ、これは素晴らしい代替手段であり、これは私が選択した道です。 Telegramは、ボットを作成する最も簡単な方法の1つを提供します（最も簡単でない場合）。その代わりに、他のメッセンジャー、電話（Amazon Connect）、Yandex Dialogsなど、何でもかまいません。

通信用のチャットボットやインターフェースを作成するのではなく、送信者が即時の応答を期待しない場合、これを通常のフィードバックフォームのように見せたいです。次の例では、受信者は電子メールでユーザーに返信できます。電子メールはフォームフィールドに残されます。

ステップ2：仲介者としてのラムダ


Telegramを設定するためのオプションの1つは、React Nativeアプリケーションコード自体にTelegram APIを使用してリクエストと会話コード全体を埋め込むことです。これはテストに適したオプションです。私の場合、いくつかのアクティブなLambda関数を含むAPIがすでに構成されているため、私のオプションには、アプリケーションとTelegram APIの間の中間層としてLambdaが含まれています。以下のコードと手順は、React Nativeプロジェクト自体で使用できます。

サーバーレスフレームワークを使用して、以下のパラメーター（format yaml）で新しい関数を作成しました。

sendTelegramMessage:
  description: used for contact form (${self:custom.stage})
  handler: lib/additional/sendMessageTelegram.handler
  environment: ${file(env_contact.yml):${self:custom.stage}}
  events:
    - http:
      path: contact
      method: POST
      private: false
  package:
    include:
      - lib/additional/sendMessageTelegram.js

トークンを 忘れchat_idないようにしてください。この例では、Lambdaで使用される環境変数ファイルを使用しています。上記の行からわかるように、サーバーレスはゲートウェイAPIをPOSTリクエスト用に構成します。また、この場合、APIは制限なく誰でも利用できることに注意してください。

そして、これがリクエストを受け入れるLambdaコードの一部です。私たちのフロントエンドは、情報（電子メールとメッセージ）を収集し、Telegram APIでリクエストを処理する関数にリクエストを送信します。

exports.handler = async (event) => {
  if (!event.body) {
    return createResponse(400, " ");
  }

  const data = JSON.parse(event.body);

  if (!data.email || !data.message) {
    return createResponse(400, " 'email'  'message' ");
  }

  try {
    const telegram = getTelegramConfig();

    await fetch(`${telegram.url}/bot${telegram.token}/sendMessage?${telegram.params(data.email, data.message)}`)

    return createResponse(200, " ");
  } catch (e) {
    return createResponse(500, e.message);
  }
};

getTelegramConfigリクエストURLを作成するため、およびcreateResponseステータスコードとメッセージを含む標準レスポンスを生成するための 2つのヘルパー関数。

const getTelegramConfig = () => {
  const token = process.env.TELEGRAM_BOT_TOKEN;
  const chatId = process.env.TELEGRAM_MAIN_CONTACT;

  if (!token || !chatId) {
    throw new Error("  ");
  }

  return {
    url: "https://api.telegram.org",
    token,
    params: (email, message) => qp.encode({
      chat_id: chatId,
      parse_mode: "Markdown",
      text: "  " + email + ": ```" + message + "```"
    })
  };
};

const createResponse = (statusCode, message) => ({
  statusCode,
  body: JSON.stringify({ message }, null, 2)
});

qp.encodeオブジェクトをなどの文字列に変換するために使用されます?param1=value&param2=value。

ここで、以下の内容でPOSTリクエストを行うと、Telegramでメッセージを受信します。

{
    "email": "privet@gmail.com",
    "message": "   .  . , ,  ,   . ,        .         ,   .  ,  .  .   ,     .          .  .    ,   "
}
