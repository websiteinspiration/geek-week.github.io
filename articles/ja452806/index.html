Swiftでジェネリックを説明しますか？


ジェネリック（ジェネリックテンプレート）を使用すると、柔軟で再利用可能な関数と、任意の型で機能する型を作成できます。重複を回避し、その目的を明確で抽象的な方法で表現するコードを記述できます。

Swiftの配列型と辞書型はユニバーサルコレクション（ジェネリック）です。
以下のコードでは、2つの値を交換するための汎用関数が文字列と整数に使用されています。これは再利用可能なコードの例です。

func swapTwoValues<T>(_ a: inout T, _ b: inout T) {
 let temporaryA = a
 a = b
 b = temporaryA
}
var num1 = 4
var num2 = 5
var str1 = “a”
var str2 = “b”
swapTwoValues(&num1,&num2)
swapTwoValues(&str1,&str2)
print (“num1:”, num1) //output: 5
print (“num2:”, num2) //output: 4
print (“str1:”, str1) //output: b
print (“str2:”, str2) //output: a

Swiftのオプションのチェーンとは何ですか？


「nil」という値を持つオプションのプロパティ、添え字、およびメソッドを要求、呼び出すプロセスは、オプションのシーケンス（オプションのチェーン）として定義されます。

オプションのシーケンスは2つの値を返します-
  • オプションに「値」が含まれている場合、それに関連付けられているプロパティ、メソッド、および添え字が呼び出されると、値が返されます
  • オプションが「nil」を含む場合、それに関連付けられているすべてのプロパティ、メソッド、および添え字はnilを返します

オプションのシーケンスは、強制アンパックの代替手段です。

強制アンラップとは何ですか？


強制アンパックは、オプションに含まれる値を取得する方法です。基本的にコンパイラーに指示しているため、この操作は危険です。このオプションには実際の値が含まれているはずです。抽出してください！

let value: Int? = 1
let newValue: Int = value! //  newValue  1 
let anotherOptionalInt: Int? = nil
let anotherInt = anotherOptionalInt! // Output:fatal error:  nil    .

何がGuard、その利点は何ですか？


オペレーターguardはシンプルで強力です。いくつかの条件をチェックし、それがfalseと評価された場合は、elseステートメントが実行され、通常はメソッドが終了します。

func addStudent(student: [String: String]) {
guard let name = student["name"] else {
return
}
print("Added \(name)!")
guard let rollNo = student ["rollNo"] else {
print("roll No not assigned")
return
}
print("Assigned roll no is \(rollNo).")
}
addStudent(student: ["name": "Ravi"])
//  "Added Ravi!"
//  "roll No not assigned"
addStudent(student: ["name": "Ravi", "rollNo": "1"])
//  "Added Ravi!"
//  "Assigned roll no is 1"

利点guardは、実行が速いことです。Guardブロックは、条件が偽である場合にのみ実行され、ユニットの出力は、以下のような、制御用伝送オペレータを介して行われるreturn、break、continueまたはthrown。これにより、早期終了とブラケットの削減が実現します。早期終了は実行の高速化を意味します。
