API门户：设计时要寻找的内容。罢工经验



使用公共API可以帮助公司增加其自身资源的价值，创建独特的内容并满足各种业务任务的要求。抢劫也不例外。已经基于Wrike API创建了3万多个应用程序。产品的用户数量在增长，这意味着对门户的需求每天都在增加。

在本文中，我将分享我在重新设计Wrike开发人员门户的界面方面的经验，并告诉您应注意的事项。

在重新设计期间，我们能够：

  1. 创建直观的导航。
  2. 创建网格以在台式机和移动设备上最佳显示内容。
  3. 根据当前设计系统更新门户网站设计，并创建缺少的组件。
  4. 符合可访问性 AA级的要求并介绍主题。


1.直观的导航


我们对门户的主要用户（后端开发人员和用户支持团队）进行了一系列采访，这些采访给了我们来自外部用户的反馈。

在采访中，我们发现，除了通知和信息更新方面的技术问题之外，门户网站还存在导航方面的问题：用户很难找到重要的信息-参考和文档。困难在于到达方法之后的部分。例如，要找到“ BI导出”部分，用户必须立即查看所有方法；无法折叠该部分。


在门户的先前版本上导航

为了优化门户的结构，我们区分了主要部分和子部分，并将技术内容与信息内容分开。因此，在新门户中，我们确定了5个主要部分：简介，API文档，API参考，BI导出和支持，并通过侧面板底部的单独链接保护了API社区部分，以便用户在必要时可以快速获得帮助。

在侧边栏中放置导航可以帮助用户立即不仅查看主要部分，还可以查看子部分。隐藏方法的能力简化了与面板的交互。如有必要，用户可以折叠不必要的部分，或者反过来钻研所需的部分。徽标，切换和搜索（开发中）保留在顶部菜单中。



在门户网站上更新导航

为了简化交互，最好让用户以不超过两个的嵌套级别显示门户的结构。其他功能（例如搜索）可以移至顶部菜单。如果用户需要使用门户的授权，则登录按钮也可以与主导航分开。这样，您可以快速找到所需的信息，并轻松在页面之间导航。

新门户网站上的方法页面

重要的是提供不同分辨率的网格行为，以便从计算机，笔记本电脑和平板电脑的屏幕上正确显示门户网站。无需在屏幕的整个宽度上拉伸内容部分，因为读取宽度为2560 px的文本很不方便。同时，打破表也不值得。

通过设置内容区域的最大和最小宽度来创建流体网格。设置内容部分的最小缩进量并进行中心对齐。内容将随屏幕宽度成比例变化，但仍将方便阅读文本和表格。

还值得添加隐藏侧边栏的功能，以便在必要时可以增加内容区域。当在屏幕宽度为1024像素的设备上工作时，这是非常正确的。



新门户上的网格原理

大多数开发人员在工作中都会使用宽屏设备。但是最好提供与门户网站和平板电脑一起使用的功能。在内容区域上方的移动设备上打开边栏。这将使信息可读。

3.根据当前设计系统更新门户的设计并创建缺少的组件


Portal API实体不同于标准网站实体。我重点介绍了几个在设计方面需要特别注意的组件。

HTTP方法


HTTP方法（GET，HEAD，POST，PUT，DELETE，CONNECT，OPTIONS，TRACE，PATCH）是最重要的API实体之一。根据操作的类型，实体具有特定的HTTP方法。在设计HTTP方法时，请尝试遵循这些准则。

  1. 为9个现有HTTP方法中的每个方法分配一个颜色标识符，以便您可以在视觉上区分它们，而不必强迫用户长时间关注。
  2. HTTP URL, : .
  3. . API Wrike , , .
  4. , : Query Tasks, Create Task, Modify Task  —  . : Contacts, Users, Tasks .. .


HTTP


发送到服务器的所有请求都以具有大量数据的表的形式呈现。

门户网站早期版本上的表

避免堆积信息。在以前的版本中，强制性参数以带有铭文的徽章的形式表示：这不仅造成视觉噪音，而且还违反了网站的可访问性要求。在新版本中，此参数突出显示了单独的“必需”列。


新门户网站上的表

尝试使嵌套最明显，并指出它所引用的参数。在早期版本的门户中，尽管嵌套与参数整体相关，但只有描述列用彩色突出显示。


嵌套在门户的先前版本上

在新的门户网站上，V形符号引用参数本身，并帮助用户了解内容隐藏在内部。

值得注意的是，嵌套表的列最好放在父表的列下。这有助于更快地了解信息。

重点放在突出显示的信息上很重要。用户在处理大量数据时确实很难集中精力。为了视觉上的方便，将鼠标悬停在一行上时，值得用颜色突出显示它。嵌套表也更好地突出显示，同时保留了选择行及其内部的功能。

4.遵守情况和专题要求


网站不仅应合乎逻辑，方便且美观，还应可供残障人士使用。如上所述，大多数门户网站组件均符合AA可访问性和包容性设计原则。

纯文本，标题和图形元素的对比符合WCAG标准。

该网站提供了制表的可能性，即键盘站点的可控制性和可访问性。

每个图形对象都有一个描述。因此，对于用于将信息复制到缓冲区的按钮，文本“复制”位于图标旁边，当您单击按钮时，文本将变为“复制的CURL示例”。

如果在应用程序注册表中出现错误，除了文本消息并以红色突出显示输入内容之外，输入字段中还会显示图形符号，以便用户了解错误所属的字段，并可以轻松地纠正输入的值。

我还要强调提供给用户的界面主题选择。由于以下几个原因，它已成为新门户的一大优势：

  1. 这是一个友好的解决方案，符合包容性设计原则之一-选择原则。
  2. 暗模式是开发人员最常使用的模式。
  3. 暗模式可调节屏幕的亮度以适应照明条件并减少眼睛疲劳。
  4. 提供更好的对比度使视力受损或色盲的人可以访问该站点。

总结


要创建方便的开发人员门户界面：

  • 通过将技术内容与信息内容分开，使导航变得逻辑，方便且易于理解。
  • 立即向用户显示门户的结构，其嵌套级别不超过两个。
  • 提供各种分辨率的网格行为，以实现最佳的内容显示。
  • 通过突出显示颜色来注意方法的设计。
  • 消除表格中图标形式的视觉干扰-文本比约定的效果更好。
  • 继续关注突出显示的信息。
  • 符合可访问性要求和包容性设计原则。
