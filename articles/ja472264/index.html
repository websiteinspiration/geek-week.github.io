デジタル考古学と仮想現実、またはBIMとVRで友達を作ろうとした方法

すべての新しいファッショナブルなテクノロジーには古いものからの大規模な基盤があり、時々それらを使用するには、歴史的なレイヤーの厚さを徹底的に掘り下げ、そこに便利で非常にタイムリーなツールを見つけることに驚きます。



VRでBIMモデルを表示する、またはRevitからエクスポートするというタスクがありました。最終結果の観点から、UnityまたはA-Frameの2つのオプションがありました。エクスポートには、IFCのみであるオープンなフォーマットが必要でした。したがって、私は最初にxBIMプロジェクトを調べました。彼らは独自のwexBIMフォーマットを備えたxBIM WeXplorer Webビューアープロジェクトを持っていました。しかし、そこではWebGLを介して独自のレンダーを使用しており、現在のバージョンのXbimXplorerではwexBIMへのエクスポートが機能せず、フォーマットの説明が見つかりませんでした。

次に、Tree.jsでエクスポートを使用するというアイデアがありましたが、一度にそれをいろいろと試しましたが、AフレームはTree.jsのラッパーにすぎません。私は、ことがわかったジェレミーTammik私はすでに同僚と同様のものを実装しました-RevitからTree.jsにjson形式でエクスポートします。githubには、Revitからエクスポートするための2つのリポジトリ、CustomExporterAdnMeshJsonとRvtVa3cがあります。

gltf / glbを作成するために、C＃-SharpGLTFを含む多数のユーティリティがKhronosグループで作成されました。

gltfを作成するには、次のオブジェクトを作成する必要があります。モデル<=シーン<=ノード<=メッシュ<=マテリアル

            var material1 = new MaterialBuilder()
                .WithAlpha(AlphaMode.MASK)
                .WithDoubleSide(true)
                .WithSpecularGlossinessShader()          
                .WithChannelParam("BaseColor", new Vector4(1, 0, 0, 0.1f));

            var ch = material1.UseChannel("MetallicRoughness");

            var mesh = new MeshBuilder<VERTEX>("mesh");

            var prim = mesh.UsePrimitive(material1);
            VERTEX tv = new VERTEX(-10, 0, 0);
            prim.AddTriangle(tv, new VERTEX(10, 0, 0), new VERTEX(0, 10, 0));
            prim.AddTriangle(new VERTEX(10, 0, 0), tv, new VERTEX(0, -10, 0));

            var mesh2 = new MeshBuilder<VERTEX>("mesh");

            // create a new gltf model
            var model = ModelRoot.CreateModel();

            // create a scene, a node, and assign the first mesh
            Scene scene = model.UseScene("Default");
            scene.CreateNode().WithMesh(model.CreateMeshes(mesh)[0]);

            // save the model in different formats
            model.SaveAsWavefront("mesh.obj");
            model.SaveGLB("mesh.glb");
            model.SaveGLTF("mesh.gltf");

Revitからのエクスポートを作成するだけです。これについては、Jeremy Tammikユーティリティを研究しました。エクスポートの場合、IExportContextインターフェイスを実装するクラスを作成する必要があります。モデルがエクスポートされると、そのメソッドが特定の順序で呼び出されます。メインチェーンStart => OnViewBegin => OnElementBegin => OnPolymeshに関心があります。Startメソッドでgltfモデルを作成し、OnElementBeginメソッドでメッシュを作成します-ここでは要素のジオメトリを取得できますが、これはbrepモデルであり、gltfには適していません。次のOnPolymeshメソッドでは、既に保存されているメッシュと同じジオメトリを取得しますgltf。

OnInstanceBeginメソッドが実行されます-要素がファミリの挿入の場合、そのジオメトリは独自の座標系で取得され、それをワールド座標系に変換する必要があります。これらの座標系を格納するスタックが必要です。

OnMaterialメソッドは、マテリアルを変更するときに実行されます。毎回新しいマテリアルを作成するのではなく、マテリアルのディクショナリを作成し、そこから必要なマテリアルを取得することをお勧めします。

OnLinkBeginメソッドは、外部リンク（実際にはネストされたモデル）を処理するときに実行されます。

OnLightメソッドは、光源を処理するときに実行されます

。OnFaceBeginメソッドは、要素の個々の面を処理するときに実行されます。このメソッドは、IncludeGeometricObjects = trueプロパティの場合にのみ呼び出されます

原則として、モデル全体を1つのノードに書き込むことができますが、将来的に要素を属性データに関連付けたいので、ElementIdを名前として使用して各要素をノードに書き込みます
