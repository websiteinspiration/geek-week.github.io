サポートを安くして品質を落とさないようにする

ハイパーバイザーレベルから直接RDPなしでVPSに接続できる緊急モード（IPKVMとも呼ばれます）により、週あたり15〜20分節約できます。

最初にそして最も重要なことは人々を激怒させないことです。世界中でサポートがラインに分かれており、従業員は最初に典型的な解決策を試す必要があります。タスクが制限を超えた場合は、2行目をパスしてください。そのため、VDS管理者の間には、考え方を知っている人がかなりいます。他の多くのサポートとは異なり。まあ、少なくともかなり頻繁に。また、チケットを適切に構成し、必要なすべてのものをすぐに説明します。最初の行で「目がぼやけている」と彼らが誤ってこれに応じてオンとオフを切り替えるように要求した場合、これは大失敗です。

タスクは非常に単純です。最小限のコストでVDSホスティングのサポートを適切にすることです。私たちはホスティングプロバイダーの世界のファーストフードです。特別な「舐め」、低価格、通常の品質はありません。以前は、 Instagram nyashkiがアカウント管理を自動化しようとしていて、リモートアカウンティングを使用して中小企業の所有者と、テクノロジーにあまり興奮していない残りの人々が、「管理者のいる管理者のような」コミュニケーションがうまく機能しなくなったという話がありました。コミュニケーションの言語を変えなければなりませんでした。

次に、プロセスについてもう少し詳しく説明します。プロセスとの不可避な問題についても説明します。

人を激怒させないでください1


すべてのサポートはコンベアの生産です。アプリケーションが到着すると、最初の行の従業員はすぐに、1000回発生した、1000回発生する典型的な状況を認識しようとします。確率は90％で、アプリケーションは標準的なものであり、ボタンを数回押すだけで答えることができ、テンプレートを置き換えることができます。通常、テンプレートにいくつかの単語を入力する必要があります-これで完了です。または、コントロールインターフェイスに移動して、そこでいくつかのボタンをクリックします。より複雑なケース（ゾーンからゾーンへの転送など）では、アルゴリズムに従ってアクションを実行する必要があります。

他のサポートの質に関係なく、人々を最も苛立たせるのは、非典型的なアプリケーションに対する典型的な反応です。すべてが詳細に説明されているチケットが届き、3つの質問に必要なデータがたくさんあります。クライアントはダイアログを予期します...そして最初の言葉によると、オートパイロットのサポート担当者はコードをピックアップして、「再起動してみてください、助けてください」というテンプレートに置き換えます。

これは人々に直接脳を開くものであり、最も否定的なレビューと怒りのコメントが残っているのはそのような状況の後です。私たちが間違っていたことは明らかであり、そこから統計を知ることができます。一般的に、私たちはさまざまな点で誤解されていましたが、そのようなケースは常に単なるワイルドです。私たち自身のために。もちろん、これがまったく起こらないようにしたいと思います。しかし、これは実際にはあまり可能ではありません。数週間ごとに、単調さに飽きてしまった従業員は、いやいや、変なボタンを押すことさえあります。

人々2を激怒させないでください


同等の成功で脳を開く2つ目のことは、だれもチケットに十分な時間答えない場合です。ヨーロッパでは、この種のサポート行動は正常です。インシデントが機能する3日前は、通常よりも多くなっています。あなたが非常に緊急で、何かが燃えている場合でも、ソーシャルネットワーク、電話、メッセンジャーはなく、メールを送信して順番を待つだけです。ロシアではこれはそれほど一般的ではありませんが、一部のチケットは「忘れられ」ています。最初に、SLAは15分間最初の反応に置かれました。そして、これは24時間年中無休です。VDSホスティングが大きくなると表示されることは明らかです。しかし、怪しいサービスプロバイダーはそうしません。そして、当初、私たちは怪しげで、多かれ少なかれ大きくなりました。わかりました、多かれ少なかれ平均。

最初の行は、スクリプトを与えられ、典型的な状況に対応するように教えられたオペレーターです。彼らは迅速かつ迅速に問題を分類し、15分以内に通常のアクションで応答するか、チケットが機能していることを報告して2番目に転送します。

2行目はすでに管理者をホストしており、ほとんどすべてを手作業で行うことができます。すべてともう少しできるサポートマネージャーもいます。3行目はすでに開発者であり、「インターフェイスでこれを修正する」または「そのようなパラメーターが誤って考慮されている」などのチケットを取得します。

アプリケーションの数を減らす


明白な理由から、サポートを安価に提供したい場合は、最初の行を増やす必要はありません。そうすることで、人々はスクリプトにすばやく対応できますが、自動化は向上します。そのため、スクリプトを持つ人の代わりに、実際のスクリプトがあります。したがって、最初に実行したことの1つは、仮想マシンの起動、リソース（ディスクの上下を含むがプロセッサの周波数は含まない）によるスケーリングなどのプロセスを自動化することでした。ユーザーがインターフェースからより多くのことができるほど、それは最初の行を生きるのがより簡単になり、それが少なくなることができます。ユーザーが自分の個人アカウントにあるものを処理するときは、それを実行し、それを個別に実行する方法を伝える必要があります。

サポートが必要ない場合は問題ありません。

多くの時間を節約する2番目の機能は、ナレッジベースの長い入力です。サポートされているアクションのリストに含まれていない問題がある場合（ほとんどの場合、「Minecraftサーバーのインストール方法」や「WinサーバーでのVPSの構成場所」などの質問です）、記事がナレッジベースに書き込まれます。同じ詳細な記事は、すべての奇妙なクエリに対して書かれています。たとえば、ユーザーが組み込みのWindows Server Firewallを削除するためのサポートを要求した場合、実際に無効にした場合にどうなるか、および選択したソフトウェアのアクセス許可のみをリセットする方法についての読み取りを送信します。問題は通常、ファイアウォールが原因ではなく、設定が原因で何かが接続できないことです。しかし、対話の中で毎回これを説明することは非常に困難です。そして、どういうわけかファイアウォールを無効にしたくありません。仮想マシンまたはクライアントのいずれかがすぐに失われるからです。

ナレッジベースのアプリケーションソフトウェアに関する何かが頻繁に訪れるようになった場合は、配布パッケージをマーケットプレイスに追加して、「これをインストール済みのサーバーを上げる」サービスが表示されるようにすることができます。実際、それはDockerで、Minecraftサーバーでも同様になりました。繰り返しになりますが、インターフェースの1つの「よくやる」ボタンで、年間数百枚のチケットを節約できます。

緊急モード


これらのアクションの後、手動での作業を必要とする最も深刻な故障は、ユーザーが何らかの理由でハイパーバイザーのゲストOSへのリモートアクセスの手段を失ったことです。最も一般的なケースは、不正なファイアウォール設定です。2番目に頻繁に発生するのは、Winが正常に起動できず、セーフモードで再起動するように強制するバグです。また、セーフモードでは、RDPはデフォルトでは使用できません。

この場合のために緊急モードを作成しました。実際、通常、VDSマシンにアクセスするには、リモートで作業するためのクライアントが必要です。ほとんどの場合、コンソールアクセス、RDP、VNCなどについて話します。これらの方法の欠点は、OSなしでは機能しないことです。しかし、ハイパーバイザーレベルでは、画面に画像を取得して、そこでキーストロークを送信できます。確かに、これは問題ないわけではありません。（ビデオの実際のブロードキャストにより）プロセッサがロードされますが、目的の結果を得ることができます。

そのため、すべてのユーザーに緊急モードへのアクセスを許可しましたが、連続使用の期間によって制限されています。幸いなことに、実践が示すように、この時間は再起動して何かを修正するには十分です。

その結果、サポートチケットはさらに少なくなります。また、管理者が自分でチケットを修正できる場合、サポートはあなたの手に渡って理解する必要はありません。

残っている問題


たいていの場合、ユーザーはサポートに何かがあると考えています。悲しいかな、これでは何もできません（まあ、それとも思いつきませんでした）。最も一般的な2つの例は、リソース制限とDDoS保護です。

各仮想マシンには、ディスクの負荷、メモリ、および許容トラフィックに制限があります。制限を設定する可能性はオファーで指定されています。制限自体は、ほとんどのユーザーが知らなくても静かに作業できるように選択されています。しかし、突然チャネルとディスクをいじり始めると、アルゴリズムは自動的にユーザーに警告します。昨年4月からオートロックを外しました。代わりに-可変期間のソフト制限を設定します。

これは以前は次のようなものでした。警告で、ユーザーが注意しなければ、自動的にブロックされます。そしてその瞬間、人々は不快に思いました：「なぜあなたは、これはあなたのシステムにバグがあり、何もなかったのです！」 -そして、あなたはアプリケーションソフトウェアを理解しようと試みるか、関税プランを増やすことを申し出ることができます。これはサポートの範囲を超えているため、アプリケーションソフトウェアの動作を理解する方法はありません。最初のいくつかのケースはユーザーによって分解されましたが。特にYouTubeの詐欺師がトロイの木馬を組み込んでいて、このトロイの木馬がメモリフローを持っていたものを覚えています。その結果、彼らはこれらがハイゼンバッグではなく、ユーザーの問題であるという結論に達しました。しかし、一人でも関税を超える可能性があることを認めていません。

DDoSについても同様です。親愛なるユーザーであるあなたが攻撃を受けていると書いています。保護を接続してください。そしてユーザー：「はい、私を攻撃するのはあなたです！」もちろん、300ルーブルを作成するために、DDoSを持つユーザーを1人だけ配置しました。収益性の高いビジネス。はい、私はより高価なカテゴリの多くの大規模なホスティング会社がこの保護を関税に含めることを知っていますが、それを行うことはできません。ファーストフード経済は他の最低価格を決定します。

多くの場合、私たちが削除したデータを持つ人々はサポートに不満を持っています。有償期間終了後に正当に削除されるという意味で。誰かがVDSレンタルを延長しない場合、次に何が起こるかを説明する通知がいくつか送信されます。支払いの終わりに、仮想マシンは停止しますが、そのイメージは保存されます。別の通知が届き、その後さらに2つ通知が届きます。画像はさらに7日間保存され、その後完全に削除されます。それで、これに非常に不満な人々のカテゴリーがあります。 「管理者が辞め、通知が彼のメールに送信され、復元されました」から始まり、詐欺の告発と身体的暴力の脅迫で終わりました。その理由は、他のすべてのユーザーにとってはすべて同じ価格です。 1か月続けると、さらにストレージが必要になります。これは、個々の顧客にとって素晴らしい価格を意味します。そして、ファーストフードの経済...さて、あなたは理解しています。その結果、「お金を盗み、データを削除し、詐欺師」の精神でフォーラムにフィードバックを受け取ります。

私は一連のプレミアム料金を持っていることに注意してください。もちろん、状況は異なります。クライアントの希望を考慮し、支払われない場合は制限と削除の両方を柔軟に調整するためです（ブロックしないためにマイナスにします）。本当にすべてが起こり、大規模な常連客を維持することは費用がかかるため、そこではすでに経済的に実現可能です。

ユーザーは悪意のある場合があります。私たちのシステムでは、クライアントの明らかに不正なアクションが原因で、数百の仮想マシンがブロックされるという障害が何度かありました。実際、このような状況が原因で、ネットワークアクティビティを監視し、ユーザーがサーバーからの攻撃を実行していないことを確認するために、独自のネットワークドライバーが必要でした。このような計画を監視することは、隣接する仮想マシンの境界が暴力的な男に違反されないようにするために重要です。

スパム、マイニング、またはその他の方法でオファーに違反する人がいます。それから彼はサポートをノックし、何が悪かったのか、なぜ車がロックされているのかを尋ねます。スクリーンショットのチケットにあるプロセスが「spam remover.exe」と呼ばれている場合は、おそらく何か問題が発生しています。 2週間ごとのどこかで、SonyまたはLucasfilm（現在のDisney）から、IPアドレスの範囲内にある仮想マシンの誰かが偽の映画を配信しているという苦情を受けています。そのようなブロックを一度にして、オファーのアカウントに残っているお金を返却します（私は覚えています：量子化は1秒あたりです。つまり、残高は常に正確です）。そして、お金を返すためには、法律の下でパスポートを提示する必要があります。これはマネーロンダリング防止です。何らかの理由で、海賊はパスポートを提示する代わりに、状況の一部を明らかにするのを忘れて、彼らからお金を搾り取ったと書いています。

そうそう。1年で最も良い要求があります。「購入前に、1か月あたり30ルーブルの割合で仮想マシンを数日間テストできますか？」

合計


最初の行はチケットをソートし、典型的なアクションで応答します。不満のほとんどはここにあります。修正の基礎は自動化のホスティング、つまり巨大なバックログにあるため、すべて同じように修正しても機能しません。はい、私たちは市場に多くを持っていますが、それでも十分ではありません。したがって、実行できる最善の方法は、最初の回線の監視を確立することです。サポートデスクの監視-最初の行のKPI実装。リアルタイムで、SLAの遅延が表示されます。このようなアラートによる注文が失われることはありません。はい、トピック外のテンプレートを使用してチケットに応答できますが、フィードバックでこれを知っています。

クライアントが要求した場合、2行目のスペシャリストはサーバーにアクセスして、そこで必要なことを実行できます（条件は、サーバーに入力するためのデータを提供する手紙による確認です）。

ユーザーデータが損傷しないことを保証したいので、これを行うことは非常にまれであり、この作業を最大限に信頼します。最高のサポートの2行目です。

最初の行には、複雑に見えるように送信できるナレッジベースがあります。

豊富な個人アカウントとナレッジベース-そして今では、コール数をクライアントあたり平均で年間1〜1.5に減らすことができました。

2行目は通常、手作業を必要とする複雑なアプリケーションを処理します。特徴的なのは、料金プランが高額になるほど、仮想マシンごとにそのようなアプリケーションが少なくなることです。通常、高額な料金を払うことができる人は州の専門家を持っているか、問題の半分だけがすべてに十分な構成があるという事実のために発生しないためです。 256 MBのRAMを備えた構成で最も古いWindowsサーバーをインストールしなかったヒーローを今でも覚えています。

2行目には、一連の配布と一連の自動化スクリプトがあります。どちらも必要に応じて更新できます。

2行目および個人のVIP関税マネージャーは、クライアントプロファイルにメモを追加できます。彼がLinux管理者であれば、それを書き留めます。これが最初の行のヒントになります。ユーザーはこれが脚のショットではなく、破壊を制御することを確実に知っています。

3行目は最も奇妙なルールです。たとえば、Firefoxで個人アカウントの機能の1つに到達できないというバグがありました。ユーザーは直接脅迫しました。「12時間以内に修正しない場合は、すべてのホストレビューに書き込みます。」結局のところ、問題はカスタム広告ブロックにありました。ユーザー側では、奇妙なことに十分です。多くの場合、複雑なエラーは詳細なしで発生し、それを繰り返すことはできません。スクリーンショット付きの刑事があります：「なぜ彼の月を修正しているのですか？」 -「はい、いつもバグを探していました」「まあ、今日また遭遇しましたが、もう一度繰り返すことはできませんでした」...

一般的に、サポートとの対話のスクリーンショットがどこに表示されるか、そして誰かがサポートをノックするかどうかはわかりません、それから彼は問題を抱えています。あなたは態度を改善することができます。少なくともそれを試してください。

はい、サポートは完璧ではありませんが、私が信じたいのは、十分な速度と十分な品質を組み合わせたサポートです。そして、それなしでできる人のための関税価格を引き上げません。
