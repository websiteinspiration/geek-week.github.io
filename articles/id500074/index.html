Anda memutuskan untuk bergabung dengan pengembangan produk sumber terbuka. Apa yang harus dipersiapkan?

Halo, saya terinspirasi untuk menulis artikel ini berdasarkan pengalaman saya berpartisipasi dalam proyek open source Apache Cloudstack, di mana saya secara berkala mengirim fitur dan perbaikan bug . Saya tidak dapat disebut sebagai kontributor aktif, karena saya hanya memberikan kontribusi dari waktu ke waktu ketika saya membutuhkan sesuatu dari produk atau saya menemukan bug yang membuat cluster saya gatal.


Pengalaman yang dijelaskan dalam artikel ini adalah murni pribadi, dan itu khas untuk produk Apache Cloudstack.


CloudStack - produk untuk menciptakan lingkungan cloud menengah dan besar jenis pribadi, publik dan campuran.

Apache CloudStack digunakan oleh sejumlah besar perusahaan, termasuk, misalnya, bagian dari yang terdaftar dalam Pimpinan dan Penantang kuadran Gartner.



Setahu saya, setidaknya BT (British Telecom), Interoute, KDDI, Datapipe, Leaseweb menggunakan CloudStack. Daftar pengguna terkenal yang tidak lengkap dapat dilihat pada tautan di Wikipedia .


Semua komunitas berbeda - saya meminta Anda untuk tidak memproyeksikan pengalaman saya pada semua proyek, masing-masing akan memiliki kekhasan dan nuansa tersendiri.
