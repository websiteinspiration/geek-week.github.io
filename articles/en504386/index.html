Vassbotn H. Class Virtual Variables

Greetings to all who began to read this article! I want to invite you (and to a large extent to fans of Object Pascal, lovers of programming tricks, techniques and syntactic refinements) to familiarize yourself with the translation of the rather old (2007) post "Hack # 17: Virtual class variables, Part I" and "Hack # 17: Virtual class variables, Part II , by Hallvard Vassbotn, a renowned developer and author of numerous Delphi-oriented techniques.


Hallward's message is devoted to one of the most interesting topics - the possibility of placing and using data associated with a particular class of application objects. If you thought about constants or class variables, then you will be right, but only partially.
This article is about class virtual var., in Russian, this phrase sounds like a virtual variable of a class (it is precisely this phrase that is placed in the name of this translation). You will say that there is no such syntactic construction in Object Pascal, and you will be absolutely right. Moreover, I am sure that experts and developers in other OO programming languages ​​will declare: in my practice, I have not heard, encountered or used anything like this. Like all I "do not see a gopher ...", but I will argue that it can exist. And this will be discussed in the text of the proposed translation.

class procedure TBasicObject.InitVMTPlaceholders;
begin
  // First, check if the VMT-mapping came thru the compiler alright :
  if Pointer(ClassGetClassInfo) = Addr(TBasicObject.VMT_Placeholder1) then
  begin
    // Now, empty the variable default, 
    // very important for later code !
    PatchCodeDWORD(@PBasicObjectOverlay(Self).OurClassInfo, DWORD(nil));

    // Now check that we see a cleaned up variable :

    // has already been done - there _has_ to be a nil here :
