オープンソースを使用する中小企業にとっての主な障壁と利点




FOSSテクノロジーは、組織をより効率的にし、カスタマイズ、適応、スケーリングの幅広い可能性を備えた環境を提供するため、長年にわたって実装してきたOracleやMicrosoftなどの業界リーダーによって広く使用されています。 TechRepublicは、AmazonやIBMなどの強力なクラウドソリューションを作成するためのツールとしてオープンソースを使用することに重点を置いている大規模な企業では、テクノロジーがメジャーリーグに限定されているように見えるかもしれませんが、中小企業も徐々にこのプロセスに関与し始めています。 。多くの人々は、オープンソースがより大きなテクノロジー企業のレベルでの競争を可能にし、これらのソリューションが提供する柔軟性、相互運用性、およびコスト削減を提供することを認識しています。しかし、このような大企業が直面していない困難もあります。有能な人材を探す必要、適切なプロジェクトを選択すること、運用上の問題、サポートの不足。



Open Source 50% 2019. 2020?




WhiteSourceチームの調査レポートによると、オープンソース製品で発見された脆弱性の数の増加の主な原因は、そのような製品自体の使用の増加であるとDevOpsは書いています。現在、これまで以上に多くのオープンソースプロジェクト、コード、コミュニティメンバーがいます。これらすべての親切な人々は、多くの場合主要な大手技術者のサポートを得て、より多くのコードを書くだけでなく、コンポーネントのユーザーを危険にさらす可能性のあるコードの脆弱性を探すためにも一生懸命取り組んでいます。より多くの記述されたコードとこれらの不可避の人為的エラーのコードを分析する目の数の組み合わせは、最終的にはより多くの脆弱性の発見につながります。オープンソースコンポーネントの脆弱性の数の増加は、ソフトウェア開発に大きな影響を与えます。近年、開発者がアプリケーションを作成する方法において、オープンソースコンポーネントがますます重要な役割を果たすことがわかりました。ほとんどの見積もりでは、最新のアプリケーションのほとんどで、オープンソースコンポーネントがコードベースの60〜80％を占めているとされています。 Apache StrutsやLinuxカーネルなどの人気のあるプロジェクトで脆弱性が報告されると、膨大な数の開発者がプロ​​グラムを更新する必要性に突然直面します。これらのオープンソースコンポーネントは、最新のほとんどのアプリケーションのコードベースの60〜80％を占めています。 Apache StrutsやLinuxカーネルなどの人気のあるプロジェクトで脆弱性が報告されると、膨大な数の開発者が突然プログラムを更新する必要に直面します。これらのオープンソースコンポーネントは、最新のほとんどのアプリケーションのコードベースの60〜80％を占めています。 Apache StrutsやLinuxカーネルなどの人気のあるプロジェクトで脆弱性が報告されると、膨大な数の開発者が突然プログラムを更新する必要に直面します。



Zoom? Jitsi Open Source




Wiredは、会議やパーティーからデートまで、今や全員がビデオ会議アプリケーションを利用しています。しかし、Covid-19の大流行の際にビデオ会議とほぼ同義になったZoomでの一連のプライバシーとセキュリティ違反の後、多くの組織や個人は、どのサービスが私たちの会話にとって最も安全であるか疑問に思っています。 Emil Ivovは誰も信用する必要はないと言っています。 Ivovは、Jitsiのテキストおよびビデオチャット用オープンソースソフトウェアの作成者であり、2018年にJitsiを買収した8x8のビデオコラボレーションチームの責任者です。同社はJitsiコードに基づくサービスを販売していますが、オープンソースバージョンをサポートするために開発者に支払いを行っています。 Jitsi Meetは便利な機能を備えたビデオ会議アプリです。あなたの予定をパスワードで保護したり、会議から人々を除外したりする機能など。しかし、最も有名なビデオ会議サービスとの違いは、無料であり、自分の機器で完全に動作できることです。





Python 2




Python 2は死んでいますか？実際にはそうではありませんが、このイベントから、彼はコンピューター技術の歴史の博物館で自分の名誉の地に向けて自信を持って足を踏み入れています。 4月20日に、Python 2.7.18の最後の最終リリースが導入されました。これにより、Python 2ブランチのサポートが完全に停止し、OpenNETが作成されました。StackOverflowブログが書いているように、このイベントは時代を完成させます。それでもバージョン3にアップグレードしていない場合は、時間です。ただし、2番目のバージョンは、個々の企業の努力を通じて引き続き存続します。たとえば、Red Hatは、RHEL 6および7ディストリビューションのライフサイクルを通じてPython 2.7でパッケージを維持し続け、RHEL 8の場合、2024年6月までアプリケーションストリームでパッケージの更新を生成します。 。これが選択肢でない場合- 公式の移行ガイドをご覧ください。ただし、Dropboxが3年間移行されるなど、時間がかかる場合があります。
