プログラミング言語の選択


Javaは現在、Androidでのモバイル開発に最も人気のあるプログラミング言語です。 Googleは、Javaに代わる言語としてKotlinを積極的に宣伝していますが。 Androidアプリケーションは他の言語でも書かれています。

さまざまな種類の言語を研究することで、特定の種類のアプリケーションを作成するために選択する必要があるプログラミング言語を明確に理解することができます。

ツールと言語の詳細


Android Studio


Androidアプリを開発する最良の方法は、Android Studioをインストールすることです。Android Studioは、Androidプラットフォームと連携するための統合開発環境（IDE）です。



JetBrains IntelliJ IDEAソフトウェアを搭載したAndroid Studioは、公式のAndroidアプリ開発ツールです。2017年5月17日、Googleは毎年開催されるGoogle I / O会議で、JavaおよびC ++に加えて、Androidプラットフォームの公式プログラミング言語としてAndroid Studioで使用されるKotlin言語のサポートを発表しました。

Android Studioは、IDE、または統合開発環境と呼ばれるソフトウェアの一部です。これは、Android SDKのパッケージとして提供されます。これは、Androidの開発を容易にするために使用されるツールのセットに過ぎません。Android用のアプリケーションの作成を開始するために必要なすべてのものがここに集中しています。ビジュアルデザイナーなどの機能によりプロセスがスムーズになり、開発者がクラウドストレージなどにアクセスできるように、高度で強力な機能が常に追加されます。

ジャワ


Android上のモバイルアプリケーションの開発者にとって、JavaはGoogleによってサポートされており、Google Play上のほとんどのアプリケーションはその上に構築されているため、Javaが最初で最も推奨されるプログラミング言語であり続けます。



Java自体は1995年にSun Microsystems（後にOracleに買収）によって開発され、現在でも幅広いソフトウェアアプリケーションで使用されています。 Javaコードは、Androidデバイスで実行され、コードを解釈する仮想マシンによって実行されます。

初心者にとって、Javaは少し複雑に見えるかもしれません。これは、Android向けアプリケーションの開発に乗り出すことを計画している人々が直面する障害になっています。

ただし、初心者でゲームを作りたい場合、または単に学習のために学習を開始する場合は、簡単なものから始めて、少し経験を積んだらすぐにJavaに戻ることをお勧めします。

コトリン


Kotlinは最近、Android開発の「その他」の公用語として登場しました。Javaと同様に、KotlinはJava仮想マシン上で実行されます。Javaと完全に互換性があり、障害やファイルサイズの増加を引き起こしません。



主な違いは、Kotlinが必要とする「ボイラープレート」コードが少ないことです。読みやすいシステム。また、ヌルポインター例外などのエラーも排除され、セミコロンで各行を終了する必要がなくなります。これは、Android向けのアプリケーション開発を学んでいるだけの場合に最適なプログラミング言語です。

Kotlinは初心者にとってより簡単な出発点であり、Android Studioを使用できるという事実は大きなプラスです。

C / C ++


初心者がAndroid向けのアプリケーションを開発するために実際に選択すべきではないのは、C / C ++です。Android Studioは、Android Native Development Kitを使用したC / C ++をサポートしています。つまり、記述されたコードはJava仮想マシンではなくデバイス自体で機能し、メモリなどをより詳細に制御できます。たとえば、3Dゲームの場合、これはデバイスから追加のパフォーマンスを絞り出します。

WITH＃


C＃は、MicrosoftのCおよびC +開発のより単純で純粋なオブジェクト指向バージョンです。Microsoftの主な目標は、C ++の能力とVisual Basicのシンプルさを組み合わせることでした。このAndroid開発言語は、多くの開発者、特にC＃とUnityの組み合わせでよく利用されています。

ユニティ


Unityは、コンピューターゲーム用のクロスプラットフォーム開発環境です。 Unityを使用すると、パーソナルコンピューター、ゲームコンソール、モバイルデバイス、インターネットアプリケーションなど、20以上の異なるオペレーティングシステムで実行されるアプリケーションを作成できます。 Unityは2005年に発売され、それ以来絶え間なく開発が続けられています。



Unityの主な利点は、ビジュアル開発環境、クロスプラットフォームサポート、およびコンポーネントのモジュラーシステムの存在です。短所としては、マルチコンポーネント回路を操作するときに問題が発生したり、外部ライブラリを接続するときに問題が発生したりすることがあります。

これは、独自のゲームを非常に簡単に作成できるオープンソースツールです。ほんの数行のコードで、1時間もかからないゲームを作成できます。

さらに、この場合のオブジェクトは実際にはほとんどの場合オブジェクトであるため、この方法での開発はオブジェクト指向コーディングを学習するための非常に実用的な方法を提供します。

C＃は、Visual Studioを介してXamarinでも使用できます。これは、AndroidとiOSのコードベースを備えたクロスプラットフォームの利点を備えた従来のAndroid開発に似ています。

ベーシック


C＃について学んだことは、Cの能力とVisual BASICの使いやすさを組み合わせる試みでした。これは、BASIC（初心者向け多目的シンボリック命令コード）が非常に楽しく使用でき、コーディング学習の完全な出発点となるためです。
残念ながら、Android Studioでは正式にサポートされておらず、Unityでは使用できません。ただし、B4A（BASIC 4 Android）と呼ばれるBASICでAndroidアプリケーションを開発するためのあまり知られていないオプションがあります。これにより、BASICを使用してAndroidアプリケーションを作成できます。



コロナ


Coronaは、Androidアプリケーションを開発する別の簡単な方法を提供します。コーディングはLUAで行います。これは、Javaよりもはるかに簡単です。独自のライブラリをすべてサポートしているため、複数のプラットフォームでデータを公開できます。

コロナは、比較的シンプルなものを作成したい人で、コーディングスキルの開発や専門家になることにそれほど関心がない人に適しています。アプリ内購入などの機能を使用したい場合は、支払う必要があります。ネイティブAndroid APIを使用する場合も同様です。

電話ギャップ
