ifuseを使用してOSXでiOSをマウントする

先日、iPadをOSX 10.14の外付けドライブとしてマウントし、コンソールからファイル操作を実行する機能に直面しました。ifuseファイルシステムを使用してこれを行う方法をインターネット上でかなりの数の説明を見つけました。残念ながら、iOS 12では何も出てきませんでした。デバイスのロックが解除され、データへのアクセスが許可されましたが、これにもかかわらずエラーが発生しました：

Failed to connect to lockdownd service on the device.
Try again. If it still fails try rebooting your device.

この記事は、同じ問題を抱えている人のために時間を節約するために書かれています。

上githubのが、リポジトリ内のソースから直接ifuseとメインの依存関係を構築するための命令であるが、主な依存関係はより多くの依存関係を持っているように、これは、かなり骨の折れる方法であり、彼らはまた、そのため、私たちが使用するなどを持っている自作のパッケージ管理システムを。

オフサイトでスクリプトを使用してインストールされます。

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

スクリプト自体は、Xcodeコマンドラインツールを含め、homebrewが必要とするすべてのものをダウンロードします。ifuseにはOSXFuseが必要です。OSXFuseは、オフサイトからダウンロードするか、次のコマンドを使用してインストールできます。

brew cask install osxfuse

次に、主な依存関係をインストールする必要があります。

brew uninstall --ignore-dependencies libimobiledevice
brew uninstall --ignore-dependencies usbmuxd
# libimobiledevice  usbmuxd   
#    
brew install --HEAD usbmuxd
brew unlink usbmuxd
brew link usbmuxd
brew install --HEAD libimobiledevice

重要： libimobiledeviceとusbmuxdの安定したバージョンがすでにインストールされている場合は、それらを削除し、-HEADスイッチを使用してdevバージョンをインストールする必要があります。そうしないと、デバイスをiOS 12に接続するときにエラーが発生します。

最後に、iFuseをインストールします。

brew install ifuse

すべての準備が整っています。1つのデバイスが接続されている場合は、次のコマンドでマウントできます。

ifuse ~/_ifuse_mount_point

どこ〜/ _ifuse_mount_pointがあるマウントポイントへのパス。



ifuseは、ファイルアクセスAPIをサポートするアプリケーションディレクトリだけでなく、パブリックにアクセス可能なファイルシステム（またはルートがある場合は全体）をマウントできます。

複数のデバイスが接続されている場合、マウントするデバイスを選択するには、-uスイッチを使用する必要があります。

ifuse ~/ifuse_mnt -u <UniqueDeviceID>

UniqueDeviceIDを見つけるには、次のコマンドを使用できます。

ideviceinfo



Spotlightから始めるには、AppleScriptを作成してアプリケーションとして保存します。

try
	do shell script "/usr/local/bin/ifuse ~/_ifuse_mount_point"
on error errMsg
	display dialog "ERROR: " & errMsg
end try
