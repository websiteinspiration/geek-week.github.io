Breaking Kubernetes cluster through Helm v2 tiller







Helm is a package manager for Kubernetes, something like apt-get



for Ubuntu. In this note, we will see the previous version of helm (v2) with the default tiller service, through which we will gain access to the cluster.







Prepare the cluster, for this we run the command:







kubectl run --rm --restart=Never -it --image=madhuakula/k8s-goat-helm-tiller -- bash








Demonstration



  • If nothing is additionally configured, helm v2 starts the tiller service, which has RBAC with full cluster administrator rights.
  • After installation in namespace , port 44134 kube-system



    appears tiller-deploy



    and opens, bound to 0.0.0.0. This can be verified using telnet.


$ telnet tiller-deploy.kube-system 44134








  • Now you can connect to the tiller service. We will use the helm binary for operations when communicating with the tiller service:


$ helm --host tiller-deploy.kube-system:44134 version








  • Let's try to get the secrets of the Kubernetes cluster from namespace kube-system



    :


$ kubectl get secrets -n kube-system
